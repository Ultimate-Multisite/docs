# Webhooks (v2) குறித்த முதல் பார்வை

_**கவனம்: இந்த அம்சம் அல்லது கட்டுரை மேம்பட்ட பயனர்களுக்கானது என்பதை கவனிக்கவும்.**_

**webhook** என்பது Ultimate Multisite போன்ற செயலி அல்லது மென்பொருள், பிற பயன்பாடுகளுக்கு நிகழ்நேரத் தகவலை வழங்கும் ஒரு வழி. webhook, தரவு அல்லது payloads-ஐ நிகழும் போதே பிற பயன்பாடுகளுக்கு வழங்குகிறது. அதாவது, நீங்கள் **தரவை உடனே பெறுகிறீர்கள்.**

Ultimate Multisite-இலிருந்து வேறு CRM அல்லது அமைப்பிற்கு, ஒவ்வொரு முறை ஒரு நிகழ்வு தூண்டப்படும்போதும் குறிப்பிட்ட தரவை ஒருங்கிணைக்க அல்லது அனுப்ப வேண்டியிருந்தால் இது உதவும். எடுத்துக்காட்டாக, புதிய பயனர் கணக்கு உருவாக்கப்படும் ஒவ்வொரு முறையும் பயனரின் பெயர் மற்றும் மின்னஞ்சல் முகவரியை mailing list-க்கு அனுப்ப வேண்டும்.

## webhook உருவாக்குவது எப்படி {#how-to-create-a-webhook}

webhook உருவாக்க, உங்கள் network admin dashboard-க்கு செல்லவும். **Ultimate Multisite > Webhooks > Add New Webhook** என்பதை கிளிக் செய்யவும்.

![Add New Webhook பொத்தானுடன் காலியான Webhooks பட்டியல் பக்கம்](/img/admin/webhooks-list-empty.png)

பின்னர் webhook உள்ளமைவைத் திருத்தலாம்:

![Name, Event, URL புலங்களுடன் Add New Webhook படிவம்](/img/admin/webhook-add-modal.png)

புதிய webhook உருவாக்கும்போது **Name, URL,** மற்றும் **Event** போன்ற தகவல்கள் கேட்கப்படும். உங்கள் webhook-க்கு நீங்கள் விரும்பும் எந்த பெயரையும் பயன்படுத்தலாம். மிக முக்கியமான புலங்கள் URL மற்றும் Event ஆகும்.

![URL புலம் மற்றும் payload முன்னோட்டத்தை காட்டும் webhook திருத்த இடைமுகம்](/img/admin/webhook-url-field.png)

URL என்பது Ultimate Multisite **payload அல்லது தரவை** அனுப்பும் **endpoint அல்லது இலக்கு** ஆகும். தரவைப் பெறும் பயன்பாடு இதுவே.

3rd party பயன்பாட்டுடன் ஒருங்கிணைப்பை எளிதாக்க பயனர் பயன்படுத்தும் மிகவும் பொதுவான தீர்வு Zapier ஆகும். Zapier போன்ற தளம் இல்லாமல், தரவைப் பெற்று அதை செயலாக்கும் தனிப்பயன் function-ஐ நீங்கள் கைமுறையாக உருவாக்க வேண்டும். **Ultimate Multisite webhook-ஐ Zapier உடன் எப்படி பயன்படுத்துவது** குறித்த இந்த கட்டுரையைப் பார்க்கவும்.

இந்த கட்டுரையில், webhook எவ்வாறு செயல்படுகிறது என்பதன் அடிப்படை கருத்தையும் Ultimate Multisite-இல் கிடைக்கும் நிகழ்வுகளையும் பார்க்கிறோம். [requestbin.com](https://requestbin.com/) என்ற 3rd party தளத்தைப் பயன்படுத்துவோம். எந்த coding-மும் செய்யாமல் endpoint உருவாக்கி payload-ஐப் பெற இந்த தளம் அனுமதிக்கும். _**பொறுப்புத்துறப்பு: இது செய்யுவது தரவு பெறப்பட்டதை நமக்குக் காட்டுவதே.**_ payload மீது எந்த செயலாக்கமும் அல்லது எந்த வகையான செயலும் செய்யப்படாது.

[requestbin.com](https://requestbin.com/) செல்லவும் மற்றும் Create Request Bin என்பதை கிளிக் செய்யவும்.

அந்த பொத்தானைக் கிளிக் செய்த பிறகு, உங்களிடம் ஏற்கனவே Account இருந்தால் உள்நுழையவோ அல்லது பதிவு செய்யவோ அது கேட்கும். உங்களிடம் ஏற்கனவே Account இருந்தால், அது உங்களை நேரடியாக அவர்களின் dashboard-க்கு அழைத்துச் செல்லும். அவர்களின் dashboard-இல், உங்கள் Ultimate Multisite webhook உருவாக்குவதில் பயன்படுத்தக்கூடிய endpoint அல்லது URL-ஐ நீங்கள் உடனே காண்பீர்கள்.

URL-ஐ நகலெடுத்து Ultimate Multisite-க்கு திரும்பவும். URL புலத்தில் endpoint-ஐ வைத்து, dropdown-இலிருந்து ஒரு நிகழ்வைத் தேர்ந்தெடுக்கவும். இந்த எடுத்துக்காட்டில், **Payment Received** என்பதைத் தேர்ந்தெடுக்கிறோம்.

பயனர் கட்டணம் செலுத்தும் போதெல்லாம் இந்த நிகழ்வு தூண்டப்படும். கிடைக்கும் அனைத்து நிகழ்வுகளும், அவற்றின் விளக்கமும், payloads-உம் பக்கத்தின் கீழே பட்டியலிடப்பட்டுள்ளன. webhook-ஐ சேமிக்க **Add New Webhook** பொத்தானைக் கிளிக் செய்யவும்.

![Payment Received தேர்ந்தெடுக்கப்பட்ட webhook நிகழ்வு dropdown](/img/admin/webhook-event-picker.png)

நாம் உருவாக்கிய webhook செயல்படுகிறதா என்பதைப் பார்க்க, இப்போது endpoint-க்கு ஒரு சோதனை நிகழ்வை அனுப்பலாம். நாம் உருவாக்கிய webhook-ன் கீழ் **Send Test Event** என்பதை கிளிக் செய்வதன் மூலம் இதைச் செய்யலாம்.

![ஒரு உள்ளமைக்கப்பட்ட webhook மற்றும் Send Test செயலை காட்டும் Webhooks பட்டியல்](/img/admin/webhooks-list-populated.png)

சோதனை வெற்றிகரமாக இருந்தது என்று கூறும் உறுதிப்படுத்தல் சாளரத்தை இது காட்டுகிறது.

![சோதனை payload அனுப்பிய பின் webhook சோதனை நிகழ்வு முடிவு](/img/admin/webhook-test-result.png)

இப்போது _Requestbin_ தளத்திற்கு திரும்பினால், சில சோதனைத் தரவுகளைக் கொண்ட payload பெறப்பட்டிருப்பதைப் பார்க்கலாம்.

webhook மற்றும் endpoints எவ்வாறு செயல்படுகின்றன என்பதன் அடிப்படை கொள்கை இதுவே. நீங்கள் தனிப்பயன் endpoint உருவாக்க வேண்டுமெனில், Ultimate Multisite-இலிருந்து நீங்கள் பெறும் தரவை செயலாக்க ஒரு தனிப்பயன் function உருவாக்க வேண்டும்.
