# Gratis AI Agent அமைப்புகள்

Gratis AI Agent இல் உள்ள **Settings → Advanced** திரை, பின்தள ஒருங்கிணைப்புகளுக்கான நிர்வாக நிலை உள்ளமைவை வழங்குகிறது. இந்தப் பக்கம் கருத்து அனுப்புதல், தேடல் வழங்குநர் keys, நிர்வகிக்கப்படும் Superdav சேவை அமைப்பு, Google Calendar கட்டுப்பாடுகள், TextBee SMS அமைப்புகள், மற்றும் வலையமைப்பு முழுவதற்குமான அம்சக் கொடிகளை ஆவணப்படுத்துகிறது.

## அமைப்புகளை அணுகுதல்

1. WordPress admin-இல், **Gratis AI Agent → Settings** என்பதற்கு செல்லவும்.
2. **Advanced** தாவலைக் கிளிக் செய்யவும்.

## கருத்து endpoint உள்ளமைவு

பயனர் thumbs-down பொத்தான், auto-prompt banner, அல்லது `/report-issue` command மூலம் கருத்தைச் சமர்ப்பிக்கும் போதெல்லாம், feedback endpoint AI agent இலிருந்து POST கோரிக்கைகளைப் பெறுகிறது.

| புலம் | விளக்கம் |
|---|---|
| **Feedback Endpoint URL** | JSON body உடன் HTTP POST கோரிக்கைகளாக கருத்துச் சமர்ப்பிப்புகளைப் பெறும் URL. |
| **Feedback API Key** | ஒவ்வொரு கருத்துக் கோரிக்கையின் `Authorization` header-இல் அனுப்பப்படும் bearer token. உங்கள் endpoint-க்கு அங்கீகாரம் தேவையில்லை என்றால் காலியாக விடவும். |

### எதிர்பார்க்கப்படும் JSON Payload

உங்கள் feedback endpoint, குறைந்தபட்சம் பின்வரும் புலங்களைக் கொண்ட JSON body-ஐ ஏற்க வேண்டும்:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

உரையாடல் சூழலைப் பொறுத்து payload-இல் கூடுதல் புலங்கள் இருக்கலாம்.

### `triage_category` மதிப்புகள்

Payload-ஐ அனுப்புவதற்கு முன், AI triage அடுக்கு `triage_category`-க்கு பின்வரும் மதிப்புகளில் ஒன்றை ஒதுக்குகிறது:

| மதிப்பு | பொருள் |
|---|---|
| `factual_error` | உதவியாளர் தவறான உண்மைத் தகவலை வழங்கியது. |
| `unhelpful_answer` | பதில் தொழில்நுட்ப ரீதியாக சரியானது, ஆனால் பயனுள்ளதாக இல்லை. |
| `inappropriate_content` | பயனர்களுக்குக் காட்டப்படக் கூடாத உள்ளடக்கம் பதிலில் இருந்தது. |
| `other` | கருத்து அறியப்பட்ட வகையுடன் பொருந்தவில்லை. |

### அங்கீகாரம்

உங்கள் endpoint-க்கு அங்கீகாரம் தேவைப்பட்டால், **Feedback API Key** புலத்தை உங்கள் bearer token ஆக அமைக்கவும். Agent அனுப்புவது:

```
Authorization: Bearer <your-api-key>
```

**Feedback API Key** புலம் காலியாக இருந்தால், `Authorization` header அனுப்பப்படாது.

### கருத்துச் சேகரிப்பை முடக்குதல்

**Feedback Endpoint URL** மற்றும் **Feedback API Key** ஆகிய இரு புலங்களையும் காலியாக விடவும். Thumbs-down பொத்தானும் கருத்து UI-யும் பயனர்களுக்குத் தொடர்ந்து தெரியும், ஆனால் சமர்ப்பிப்புகள் எந்த வெளிப்புற சேவைக்கும் அனுப்பப்படாது.

## Brave Search API Key

**Advanced** தாவலிலேயே, **Brave Search API Key** புலம் [இணையத் தேடல்](../configuration/internet-search) திறனை இயக்குகிறது.

| புலம் | விளக்கம் |
|---|---|
| **Brave Search API Key** | Brave Search developer dashboard இலிருந்து கிடைக்கும் உங்கள் API key. AI assistant-இல் இணையத் தேடலை இயக்க இது தேவை. |

புல லேபலில் Brave Search API பதிவு பக்கத்திற்கான கிளிக் செய்யக்கூடிய இணைப்பு உள்ளது. இணையத் தேடலை முடக்க காலியாக விடவும்.

இந்த அம்சத்திற்கான இறுதி பயனர் ஆவணத்திற்கு [இணையத் தேடல்](../configuration/internet-search) பார்க்கவும்.

## நிர்வகிக்கப்படும் Superdav சேவை

Superdav AI Agent v1.18.0, ஆதரிக்கப்படும் தளங்களுக்கு நிர்வகிக்கப்படும் Superdav சேவை endpoints மற்றும் தானியங்கி இணைப்பு provisioning-ஐ சேர்க்கிறது. உங்கள் தளம் கைமுறையாக உள்ளமைக்கப்பட்ட சேவை endpoint-க்கு பதிலாக hosted provider-உடன் இணைக்கப்பட வேண்டுமெனில், இந்தக் கட்டுப்பாடுகளைப் பயன்படுத்தவும்.

| புலம் | விளக்கம் |
|---|---|
| **Managed Superdav Service** | ஆதரிக்கப்படும் தளங்களுக்கு hosted Superdav சேவை இணைப்பை இயக்குகிறது. |
| **Provision Connection** | தானியங்கி endpoint மற்றும் credential provisioning-ஐத் தொடங்குகிறது. தளம் managed provider-ஐப் பயன்படுத்த வேண்டும் என்பதை உறுதிப்படுத்திய பிறகு இதைப் பயன்படுத்தவும். |
| **Service Endpoint / Connection Status** | Provisioning முடிந்த பிறகு தற்போதைய endpoint அல்லது இணைப்பு நிலையை காட்டுகிறது. |

Provisioning முடிந்த பிறகு, managed-service பணிப்பாய்வுகளை நம்புவதற்கு முன் அமைப்புகளைச் சேமித்து இணைப்பு நிலையைச் சரிபார்க்கவும். Provisioning தோல்வியடைந்தால், காட்டப்படும் மீண்டும் முயற்சி வழிகாட்டுதலைப் பார்வையிட்டு, hosted provider-ஐப் பயன்படுத்த தளத்திற்கு அனுமதி உள்ளதா என்பதை உறுதிப்படுத்தவும்.

## Google Calendar உள்ளமைவு

Superdav AI Agent v1.18.0 calendar அம்சங்கள் இயக்கப்பட்டிருக்கும்போது, agent உள்ளமைக்கப்பட்ட calendars மற்றும் event விவரங்களைப் படிக்க முடியும். Calendar கருவிகள் படிப்பை மையமாகக் கொண்டவை. அட்டவணை அறிந்த நினைவூட்டல்கள், பங்கேற்பாளர் தொடர்ச்சி, மற்றும் தொடர்பு பொருத்துதல் ஆகியவற்றுக்கு அவை பயனுள்ளதாக இருக்கும்.

| புலம் | விளக்கம் |
|---|---|
| **Google Calendar Credentials** | Calendar தரவைப் படிக்க தேவையான credentials அல்லது token இணைப்பைச் சேமிக்கிறது. |
| **Calendar Selection** | Agent பரிசோதிக்கக்கூடிய உள்ளமைக்கப்பட்ட calendars எவை என்பதை வரையறுக்கிறது. |
| **Calendar Connection Status** | தற்போதைய credentials, calendars மற்றும் events-ஐப் படிக்க முடியுமா என்பதை உறுதிப்படுத்துகிறது. |

Calendar credentials-ஐ agent-க்கு தேவையான calendars மட்டுமாக வரையறுக்கவும். நிலை காலாவதியான token-ஐக் காட்டினால், credentials-ஐ மீண்டும் இணைக்கவும் அல்லது மாற்றவும்.

## TextBee SMS அறிவிப்புகள்

Superdav AI Agent v1.18.0, உள்ளமைக்கப்பட்ட அறிவிப்பு பணிப்பாய்வுகளுக்கான SMS வழங்குநராக TextBee-யைச் சேர்க்கிறது. நுணுக்கமான அல்லது பயனர்களை நோக்கிய செய்திகளுக்கு, SMS அறிவிப்புகள் மனித ஒப்புதல் வாயில்களுடன் இணைக்கப்பட வேண்டும்.

| புலம் | விளக்கம் |
|---|---|
| **TextBee API Key** | TextBee SMS வழங்குநருக்கான கோரிக்கைகளை அங்கீகரிக்கிறது. |
| **TextBee Device / Sender** | வழங்குநர் தேவைப்படும்போது, வெளியேறும் செய்திகளுக்கு பயன்படுத்தப்படும் TextBee sender அல்லது device-ஐத் தேர்ந்தெடுக்கிறது. |
| **SMS Notifications Enabled** | ஒப்புதல் பெற்ற பணிப்பாய்வுகள் text-message அறிவிப்புகளை அனுப்ப அனுமதிக்கிறது. SMS அனுப்புவதைத் தடுக்க முடக்கப்பட்டபடியே விடவும். |

நிர்வாகியுடைய எண்ணுக்கு மட்டும் சோதனைச் செய்தி அனுப்பவும். பின்னர் அட்டவணைப்படுத்தப்பட்ட அல்லது பங்கேற்பாளர்களை நோக்கிய நினைவூட்டல்களை இயக்குவதற்கு முன் approval-gate நடத்தை உறுதிப்படுத்தவும்.

## அம்சக் கொடிகள்

v1.9.0-இல் அறிமுகப்படுத்தப்பட்ட **Settings → Feature Flags** தாவல், விருப்ப செயல்பாடுகளுக்கான toggle switches-ஐ வழங்குகிறது. ஒவ்வொரு கொடியும் வலையமைப்பு முழுவதும் இயக்கப்பட்டதோ முடக்கப்பட்டதோ ஆக இருக்கும்; இப்போது தனித் தளம் வாரியான override இல்லை.

### அம்சக் கொடிகளை அணுகுதல்

1. WordPress admin-இல், **Gratis AI Agent → Settings** என்பதற்கு செல்லவும்.
2. **Feature Flags** தாவலைக் கிளிக் செய்யவும்.

### அணுகல் கட்டுப்பாட்டு கொடிகள்

| குறி | இயல்புநிலை | விளக்கம் |
|---|---|---|
| **Administrators-க்கு மட்டுப்படுத்து** | Off | இயக்கப்பட்டால், `administrator` பங்கு உள்ள பயனர்கள் மட்டுமே AI Agent அரட்டை பலகத்தைத் திறக்க முடியும். மற்ற எல்லா பங்குகளுக்கும் அதற்கு பதிலாக "உங்கள் administrator-ஐ தொடர்புகொள்ளவும்" என்ற செய்தி தெரியும். |
| **Network Admins-க்கு மட்டுப்படுத்து** | Off | multisite வலையமைப்பில் இயக்கப்பட்டால், Super Admins மட்டுமே agent-ஐ பயன்படுத்த முடியும். தனிப்பட்ட site admins தடுக்கப்படுவார்கள். இரண்டும் இயக்கப்பட்டிருந்தால் "Administrators-க்கு மட்டுப்படுத்து" என்பதைக் காட்டிலும் இதற்கு முன்னுரிமை உண்டு. |
| **Subscriber அணுகலை அனுமதி** | Off | இயக்கப்பட்டால், `subscriber` பங்கு உள்ள பயனர்கள் அரட்டை இடைமுகத்தைப் பயன்படுத்த முடியும். ஆனால் படிக்க மட்டும் கூடிய திறன்களுக்கே வரையறுக்கப்படுவார்கள் (பதிவு உருவாக்கம் அல்லது அமைப்பு மாற்றங்கள் இல்லை). |
| **உறுப்பினரல்லாதவர்களுக்கு முடக்கு** | Off | Ultimate Multisite உறுப்பினர் நிலைமைக்குடன் ஒருங்கிணைக்கிறது. இயக்கப்பட்டால், செயலில் உள்ள உறுப்பினர் நிலை இல்லாத தளங்களுக்கு அரட்டை மறைக்கப்படும். |

### பிராண்டிங் குறிகள்

| குறி | இயல்புநிலை | விளக்கம் |
|---|---|---|
