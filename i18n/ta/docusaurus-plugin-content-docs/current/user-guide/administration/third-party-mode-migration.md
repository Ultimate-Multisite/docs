# மூன்றாம் தரப்பு முறை இடம்பெயர்வு

Superdav AI Agent v1.12.0, மூன்றாம் தரப்பு திறன்களை (third-party abilities) கையாளும் விதத்தில் மாற்றங்களைச் செய்துள்ளது. **மூன்றாம் தரப்பு முறை இப்போது இயல்புநிலை (default) ஆக 'auto' என்று அமைக்கப்பட்டுள்ளது**, இது WordPress 7.0+ இல் கையேடு உள்ளமைவு (manual configuration) இல்லாமல், native WordPress Abilities API ஒருங்கிணைப்பை செயல்படுத்துகிறது.

## என்ன மாறியுள்ளது? {#what-changed}

### v1.12.0 க்கு முன் {#before-v1120}

மூன்றாம் தரப்பு திறன்களைப் பயன்படுத்த கையேடு உள்ளமைவு தேவைப்பட்டது:

- நீங்கள் வெளிப்படையாக "third-party mode" என்பதை இயக்க வேண்டும்.
- திறன்கள் ஒரு தனிப்பயன் பதிவேற்றத்தில் (custom registry) இருந்து ஏற்றப்பட்டன.
- WordPress Abilities API உடன் ஒருங்கிணைப்பது விருப்பத்தேர்வாக இருந்தது.
- பழைய முறை (Legacy mode) இயல்புநிலையாக இருந்தது.

### v1.12.0 க்குப் பிறகு {#after-v1120}

மூன்றாம் தரப்பு திறன்கள் தானாகவே வேலை செய்கின்றன:

- மூன்றாம் தரப்பு முறை இயல்புநிலை 'auto' ஆக உள்ளது.
- திறன்கள் WordPress Abilities API உடன் இயல்பாகவே ஒருங்கிணைக்கப்படுகின்றன.
- WordPress 7.0+ இல் கையேடு உள்ளமைவு தேவையில்லை.
- பழைய WordPress பதிப்புகளுக்கு பழைய முறை (Legacy mode) இன்னும் கிடைக்கிறது.

## யாருக்குப் பொருந்தும்? {#who-is-affected}

### புதிய நிறுவல்கள் (WordPress 7.0+) {#new-installations-wordpress-70}

**எதுவும் செய்யத் தேவையில்லை.** மூன்றாம் தரப்பு முறை தானாகவே "auto" என்று அமைக்கப்பட்டுள்ளது, மேலும் திறன்கள் உடனடியாக வேலை செய்யும்.

### ஏற்கனவே உள்ள நிறுவல்கள் {#existing-installations}

**உங்கள் அமைப்புகள் பாதுகாக்கப்படுகின்றன.** நீங்கள் பயன்படுத்தியிருந்தால்:

- **Legacy mode**: நீங்கள் பழைய முறையிலேயே இருப்பீர்கள் (மாற்றம் இல்லை)
- **Manual third-party mode**: நீங்கள் கையேடு முறையில் இருப்பீர்கள் (மாற்றம் இல்லை)
- **Auto mode**: நீங்கள் ஆட்டோ முறையில் தொடர்வீர்கள் (மாற்றம் இல்லை)

### 7.0-க்கு முந்தைய WordPress பதிப்புகள் {#wordpress-versions-before-70}

**பழைய முறை (Legacy mode) இன்னும் கிடைக்கிறது.** நீங்கள் WordPress 6.x அல்லது அதற்கு முந்தைய பதிப்பில் இருந்தால்:

- மூன்றாம் தரப்பு முறை இயல்புநிலை "legacy" ஆக உள்ளது.
- தேவைப்பட்டால், நீங்கள் மூன்றாம் தரப்பு முறையை கையேடாக இயக்கலாம்.
- native Abilities API ஐப் பயன்படுத்த WordPress 7.0+ க்கு மேம்படுத்தவும்.

## முறைகளைப் புரிந்துகொள்ளுதல் {#understanding-the-modes}

### Auto Mode (புதிய இயல்புநிலை) {#auto-mode-new-default}

**Auto mode** என்பது native WordPress Abilities API ஒருங்கிணைப்பைப் பயன்படுத்துகிறது:

- திறன்கள் WordPress hooks மூலம் பதிவு செய்யப்படுகின்றன.
- WordPress 7.0+ Abilities API உடன் முழு இணக்கத்தன்மை.
- மூன்றாம் தரப்பு திறன்களை தானாகவே கண்டறிதல்.
- கையேடு உள்ளமைவு தேவையில்லை.

**எப்போது பயன்படுத்த வேண்டும்**: WordPress 7.0+ உடன் மூன்றாம் தரப்பு திறன்கள் இருக்கும்போது.

### Manual Mode {#manual-mode}

**Manual mode** கையேடு உள்ளமைவைக் கோருகிறது:

- நீங்கள் எந்த மூன்றாம் தரப்பு திறன்களை ஏற்ற வேண்டும் என்று குறிப்பிடுகிறீர்கள்.
- சோதனை செய்வதற்கோ அல்லது குறிப்பிட்ட திறன்களை மட்டும் ஏற்றவோ இது பயனுள்ளதாக இருக்கும்.
- உள்ளமைவு கோப்புகளை (configuration files) திருத்த வேண்டும்.
- அதிக கட்டுப்பாடு, ஆனால் அதிக அமைப்பு தேவை.

**எப்போது பயன்படுத்த வேண்டும்**: சோதனை, குறிப்பிட்ட திறன்களை மட்டும் ஏற்றுதல் அல்லது தனிப்பயன் உள்ளமைவுகள் தேவைப்படும்போது.

### Legacy Mode {#legacy-mode}

**Legacy mode** பழைய மூன்றாம் தரப்பு திறன்கள் அமைப்பைப் பயன்படுத்துகிறது:

- தனிப்பயன் திறன் பதிவேற்றம் (Custom ability registry) (இது WordPress Abilities API அல்ல).
- பழைய WordPress பதிப்புகளுடன் பின்னோக்கி இணக்கமானது (Backward compatible).
- native WordPress ஒருங்கிணைப்பு இல்லை.
- காலாவதியானது (Deprecated) ஆனால் இன்னும் ஆதரிக்கப்படுகிறது.

**எப்போது பயன்படுத்த வேண்டும்**: WordPress 6.x அல்லது அதற்கு முந்தைய பதிப்புகள், அல்லது பழைய இணக்கத்தன்மை தேவைப்படும்போது.

## உங்கள் தற்போதைய முறையைச் சரிபார்த்தல் {#checking-your-current-mode}

### Admin Panel வழியாக {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** க்குச் செல்லவும்.
2. **Third-Party Mode** அமைப்பைக் கவனிக்கவும்.
3. உங்கள் தற்போதைய முறை மற்றும் அதை மாற்றும் விருப்பங்கள் உங்களுக்குத் தெரியும்.

### Code வழியாக {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', அல்லது 'legacy'
```

## உங்கள் முறையை மாற்றுதல் {#changing-your-mode}

### Auto Mode க்கு மாறுதல் {#switch-to-auto-mode}

நீங்கள் WordPress 7.0+ இல் இருந்து, auto mode ஐப் பயன்படுத்த விரும்பினால்:

1. **Superdav AI Agent** → **Settings** க்குச் செல்லவும்.
2. **Third-Party Mode** ஐக் கண்டறியவும்.
3. **Auto (WordPress Abilities API)** என்பதைத் தேர்ந்தெடுக்கவும்.
4. **Save** என்பதைக் கிளிக் செய்யவும்.

Superdav AI Agent தானாகவே மூன்றாம் தரப்பு திறன்களைக் கண்டறிந்து பதிவு செய்யும்.

### Manual Mode க்கு மாறுதல் {#switch-to-manual-mode}

எந்த திறன்கள் ஏற்றப்பட வேண்டும் என்பதை நீங்கள் கையேடாகக் கட்டுப்படுத்த விரும்பினால்:

1. **Superdav AI Agent** → **Settings** க்குச் செல்லவும்.
2. **Third-Party Mode** ஐக் கண்டறியவும்.
3. **Manual** என்பதைத் தேர்ந்தெடுக்கவும்.
4. **Save** என்பதைக் கிளிக் செய்யவும்.
5. எந்த திறன்களை ஏற்ற வேண்டும் என்று குறிப்பிded உங்கள் உள்ளமைவு கோப்பை (configuration file) திருத்தவும்.

### Legacy Mode க்கு மாறுதல் {#switch-to-legacy-mode}

உங்களுக்கு பழைய இணக்கத்தன்மை தேவைப்பட்டால்:

1. **Superdav AI Agent** → **Settings** க்குச் செல்லவும்.
2. **Third-Party Mode** ஐக் கண்டறியவும்.
3. **Legacy** என்பதைத் தேர்ந்தெடுக்கவும்.
4. **Save** என்பதைக் கிளிக் செய்யவும்.

## Auto Mode இன் நன்மைகள் {#benefits-of-auto-mode}

### தானியங்கி கண்டறிதல் (Automatic Discovery) {#automatic-discovery}

திறன்கள் தானாகவே இவற்றிலிருந்து கண்டறியப்படுகின்றன:

- நிறுவப்பட்ட plugins
- செயலில் உள்ள theme
- must-use plugins
- drop-in plugins

கையேடுப் பதிவு தேவையில்லை.

### native ஒருங்கிணைப்பு (Native Integration) {#native-integration}

திறன்கள் WordPress Abilities API உடன் ஒருங்கிணைக்கப்படுகின்றன:

- WordPress core உடன் இணக்கமானது.
- WordPress admin உடன் வேலை செய்கிறது.
- Abilities API ஐப் பயன்படுத்தும் பிற plugins உடன் இணக்கமானது.
- WordPress வளர்ச்சியடையும்போது எதிர்காலத்திற்கான பாதுகாப்பு (Future-proof).

### எளிமைப்படுத்தப்பட்ட மேலாண்மை (Simplified Management) {#simplified-management}

- திருத்த வேண்டிய உள்ளமைவு கோப்புகள் இல்லை.
- கையேடு திறன்ப் பதிவு தேவையில்லை.
- Ability Visibility கட்டுப்பாடுகள் தானாகவே வேலை செய்கின்றன.
- நிர்வாக அறிவிப்புகள் (Admin notices) வகைப்படுத்தப்படாத திறன்களை உங்களுக்கு எச்சரிக்கின்றன.

### சிறந்த செயல்திறன் (Better Performance) {#better-performance}

- திறன்கள் கேச் செய்யப்படுகின்றன (cached).
- தேவைப்படும்போது சோம்பேறி முறையில் ஏற்றப்படுகின்றன (Lazy-loaded).
- WordPress 7.0+ க்காக உகந்ததாக்கப்பட்டுள்ளது.

## இடம்பெயர்வு பாதை (Migration Path) {#migration-path}

### நீங்கள் WordPress 6.x இல் இருந்தால் {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ க்கு மேம்படுத்தவும்** (தயாரானதும்).
2. **Superdav AI Agent** ஐ v1.12.0+ க்கு மேம்படுத்தவும்.
3. **third-party mode ஐ Auto ஆக மாற்றவும்** (விருப்பத்தேர்வு; legacy mode இன்னும் வேலை செய்யும்).
4. சரியான அணுகல் கட்டுப்பாடுகளை உறுதிப்படுத்த **ability visibility** ஐ மதிப்பாய்வு செய்யவும்.

### நீங்கள் WordPress 7.0+ இல் இருந்தால் {#if-youre-on-wordpress-70}

1. **Superdav AI Agent** ஐ v1.12.0+ க்கு மேம்படுத்தவும்.
2. **third-party mode Auto ஆக அமைக்கப்பட்டுள்ளதா என்பதைச் சரிபார்க்கவும்** (இது இயல்புநிலையாக இருக்க வேண்டும்).
3. சரியான அணுகல் கட்டுப்பாடுகளை உறுதிப்படுத்த **ability visibility** ஐ மதிப்பாய்வு செய்யவும்.
4. அவை வேலை செய்கின்றன என்பதை உறுதிப்படுத்த **third-party abilities** ஐச் சோதிக்கவும்.

## சிக்கல் தீர்க்கும் வழிகாட்டி (Troubleshooting) {#troubleshooting}

### auto mode இல் திறன்கள் ஏற்றப்படவில்லை {#abilities-arent-loading-in-auto-mode}

- நீங்கள் WordPress 7.0+ இல் இருக்கிறீர்களா என்பதைச் சரிபார்க்கவும்.
- third-party mode "Auto" என்று அமைக்கப்பட்டுள்ளதா என்பதைச் சரிபார்க்கவும்.
- திறனை வழங்கும் plugin செயலில் உள்ளதா என்பதைச் சரிபார்க்கவும்.
- பதிவு பிழைகளுக்காக WordPress error logs ஐச் சரிபார்க்கவும்.

### நான் legacy mode ஐத் தக்கவைக்க விரும்புகிறேன் {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** க்குச் செல்லவும்.
- **Legacy** என்பதைத் தேர்ந்தெடுக்கவும்.
- **Save** என்பதைக் கிளிக் செய்யவும்.
- legacy mode தொடர்ந்து வேலை செய்யும்.

### எனது தனிப்பயன் திறன்கள் காட்டப்படவில்லை {#my-custom-abilities-arent-showing}

- அவை WordPress hooks மூலம் பதிவு செய்யப்பட்டுள்ளதா என்பதைச் சரிபார்க்கவும்.
- அவை Abilities API ஐச் சரியாகச் செயல்படுத்துகின்றனவா என்பதைச் சரிபார்க்கவும்.
- WordPress error logs ஐ மதிப்பாய்வு செய்யவும்.
- அனைத்து பதிவு செய்யப்பட்ட திறன்களையும் பார்க்க **Ability Visibility** admin பக்கத்தைப் பயன்படுத்தவும்.

### எனக்கு "unclassified ability" அறிவிப்புகள் வருகின்றன {#im-getting-unclassified-ability-notices}

- இது புதிய மூன்றாம் தரப்பு திறன்களுக்கு இயல்பானது.
- நிர்வாக அறிவிப்பில் அவற்றை மதிப்பாய்வு செய்து வகைப்படுத்தவும் (classify).
- வகைப்படுத்துதல் பற்றிய விவரங்களுக்கு **Ability Visibility** ஐப் பார்க்கவும்.

## பின்னோக்கு இணக்கத்தன்மை (Backward Compatibility) {#backward-compatibility}

### ஏற்கனவே உள்ள உள்ளமைவுகள் {#existing-configurations}

உங்களிடம் ஏற்கனவே மூன்றாம் தரப்பு திறன் உள்ளமைவுகள் இருந்தால்:

- **Legacy mode**: உங்கள் உள்ளமைவு வேலை செய்யத் தொடரும்.
- **Manual mode**: உங்கள் உள்ளமைவு வேலை செய்யத் தொடரும்.
- **Auto mode**: உங்கள் உள்ளமைவு புறக்கணிக்கப்படும் (auto mode பொறுப்பேற்கிறது).

உங்கள் தனிப்பயன் உள்ளமைவைப் பாதுகாக்க, Manual அல்லது Legacy முறையில் இருங்கள்.

### காலாவதி காலவரிசை (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: Legacy மற்றும் Manual முறைகள் இன்னும் முழுமையாக ஆதரிக்கப்படுகின்றன.
- **v1.13.0+**: Legacy mode காலாவதி அறிவிப்புகளைக் காட்டக்கூடும்.
- **v2.0.0**: Legacy mode நீக்கப்படலாம் (TBD).

## சிறந்த நடைமுறைகள் (Best Practices) {#best-practices}

### புதிய நிறுவல்களுக்கு {#for-new-installations}

- Auto mode ஐப் பயன்படுத்தவும் (இது இயல்புநிலை).
- Superdav AI Agent திறன்களை தானாகவே கண்டறிய அனுமதிக்கவும்.
- அணுகலைக் கட்டுப்படுத்த Ability Visibility ஐப் பயன்படுத்தவும்.

### ஏற்கனவே உள்ள நிறுவல்களுக்கு {#for-existing-installations}

- முடிந்தால் WordPress 7.0+ க்கு மேம்படுத்தவும்.
- எளிமைப்படுத்தப்பட்ட மேலாண்மைக்காக Auto mode க்கு மாறவும்.
- Ability Visibility ஐப் பயன்படுத்தி திறன்களை மதிப்பாய்வு செய்து வகைப்படுத்தவும்.

### தனிப்பயன் திறன்களுக்கு {#for-custom-abilities}

- WordPress hooks (Abilities API) மூலம் திறன்களைப் பதிவு செய்யவும்.
- தனிப்பயன் திறன் பதிவேற்றங்களைத் தவிர்க்கவும்.
- Auto mode உடன் WordPress 7.0+ இல் சோதிக்கவும்.

## அடுத்த படிகள் {#next-steps}

1. **உங்கள் WordPress பதிப்பைச் சரிபார்க்கவும்**: Auto mode க்காக நீங்கள் 7.0+ இல் இருக்கிறீர்களா என்பதை உறுதிப்படுத்தவும்.
2. **உங்கள் third-party mode ஐ மதிப்பாய்வு செய்யவும்**: Settings க்குச் சென்று உங்கள் தற்போதைய முறையைச் சரிபார்க்கவும்.
3. **தேவைப்பட்டால் மேம்படுத்தவும்**: நீங்கள் WordPress 7.0+ இல் இருந்தால் Auto mode க்கு மாறவும்.
4. **திறன்களை வகைப்படுத்தவும்**: எந்தவொரு வகைப்படுத்தப்படாத திறன்களையும் மதிப்பாய்வு செய்து வகைப்படுத்தவும்.
5. **சோதிக்கவும்**: உங்கள் மூன்றாம் தரப்பு திறன்கள் சரியாக வேலை செய்கின்றனவா என்பதை உறுதிப்படுத்தவும்.
