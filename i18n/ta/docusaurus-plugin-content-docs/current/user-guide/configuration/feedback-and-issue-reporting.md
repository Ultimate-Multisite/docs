# வாடிக்கையாளர் கருத்து மற்றும் சிக்கல் அறிக்கை {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 ஆனது, உள்ளமைக்கப்பட்ட கருத்து மற்றும் சிக்கல் அறிக்கை அமைப்பை அறிமுகப்படுத்துகிறது. இது இறுதிப் பயனர்கள் உதவியற்ற பதில்களைக் குறிக்கவும், சாட் இடைமுகத்திலிருந்தே சிக்கல்களைப் புகாரளிக்கவும் அனுமதிக்கிறது. இந்த அமைப்பு, சம்மத மேலாண்மை (consent management), தானியங்கி அறிக்கை கட்டளை (automated reporting command), மற்றும் பின்னணியில் AI-உதவி பெற்ற முன்கூட்டியே வகைப்படுத்துதல் (AI-assisted triage) ஆகியவற்றை உள்ளடக்கியது.

## கட்டைவிரல் கீழ் பொத்தான் (Thumbs-Down Button) {#thumbs-down-button}

AI உதவியாளர் அனுப்பும் ஒவ்வொரு செய்தியிலும் ஒரு **கட்டைவிரல் கீழ் பொத்தான்** (👎) காட்டப்படும். ஒரு பயனர் இதை கிளிக் செய்யும்போது, அந்தப் பதிலை உதவியற்றதாகவோ அல்லது தவறானதாகவோ குறிக்க முடியும்.

- இந்த பொத்தான், ஒவ்வொரு உதவியாளர் செய்தியின் அருகிலும், நீங்கள் அதன் மீது கர்சரை வைக்கும்போது (on hover) தோன்றும்.
- இதை கிளிக் செய்வது **கருத்து சம்மத மாடலை** (Feedback Consent Modal) திறக்கும்.
- இந்த கருத்து, உரையாடல் நூல் (conversation thread) மற்றும் குறிப்பிட்ட செய்தியுடன் இணைக்கப்படும்.

## கருத்து சம்மத மாடல் (Feedback Consent Modal) {#feedback-consent-modal}

பயனர் கட்டைவிரல் கீழ் பொத்தானை கிளிக் செய்யும் போது, எந்தத் தரவும் அனுப்பப்படுவதற்கு முன் ஒரு சம்மத மாடல் தோன்றும். இந்த மாடல்:

- என்னென்ன தகவல்கள் சேகரிக்கப்படும் என்பதை விளக்குகிறது (உரையாடல் பகுதி, பிரவுசர் சூழல்).
- இந்தத் தரவைப் பகிர்வதற்கு நீங்கள் சம்மதம் அளிக்கிறீர்களா என்பதை உறுதிப்படுத்த கேட்கிறது.
- என்ன தவறு நடந்தது என்பதை விவரிக்க ஒரு இலவச உரைத் புலத்தை (free-text field) வழங்குகிறது.
- சமர்ப்பிக்காமல் மூட ஒரு **Cancel** விருப்பத்தை வழங்குகிறது.

பயனர் வெளிப்படையாக உறுதிப்படுத்தாத வரை எந்தக் கருத்தும் பதிவு செய்யப்படாது.

## ஆட்டோ-ப்ராம்ட் கருத்து பதாகை (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

ஒரு உரையாடல் முடிந்ததும், உதவியாளர் ஒரு **ஆட்டோ-ப்ராம்ட் கருத்து பதாகையை** (auto-prompt feedback banner) காட்டலாம் — இது ஒரு தொந்தரவு இல்லாத செய்தி. இந்தச் செய்தி, அமர்வு உதவியாக இருந்ததா என்று கேட்கும்.

இந்த பதாகை, உரையாடலின் நீளம் மற்றும் விளைவுக்கான உத்திகளை (heuristics) அடிப்படையாகக் கொண்டு தானாகவே தோன்றும். இது கட்டைவிரல் கீழ் பொத்தானைப் போலவே அதே கருத்துப் பணிப்பாய்வுக்கு (feedback flow) இணைக்கிறது. பயனர்கள் கருத்து கொடுக்காமல் இந்த பதாகையை மூடலாம்.

## /report-issue ஸ்லாஷ் கட்டளை (Slash Command) {#report-issue-slash-command}

பயனர்கள் சாட் உள்ளீட்டில் `/report-issue` என்று டைப் செய்வதன் மூலம் நேரடியாக கருத்துப் பணிப்பாய்வைத் தூண்ட முடியும். இந்த கட்டளை:

- உடனடியாக கருத்து சம்மத மாடலை திறக்கும்.
- தற்போதைய உரையாடலைப் பற்றிய சூழலுடன் (context) விளக்கம் புலத்தை (description field) தானாக நிரப்பும்.
- சமர்ப்பிப்பதற்கு முன் கூடுதல் விவரங்களைச் சேர்க்க பயனர்களை அனுமதிக்கிறது.

`/report-issue` கட்டளை அனைத்து சாட் முறைகளிலும் (inline, floating widget, full-screen) கிடைக்கும்.

## AI-உதவி பெற்ற வகைப்படுத்துதல் (AI-Assisted Triage) {#ai-assisted-triage}

சமர்ப்பிக்கப்பட்ட கருத்துக்கள், பின்வரும் செயல்பாடுகளைச் செய்யும் ஒரு AI வகைப்படுத்துதல் அடுக்குக்கு (AI triage layer) அனுப்பப்படும்:

- அறிக்கையை வகைப்படுத்துகிறது (உண்மைத் தவறு, உதவியற்ற பதில், பொருத்தமற்ற உள்ளடக்கம், போன்றவை).
- தொடர்புடைய உரையாடல் மெட்டாடேட்டாவை (metadata) இணைக்கிறது.
- வகைப்படுத்தல் சுருக்கத்தை (triage summary) கட்டமைக்கப்பட்ட **Feedback Endpoint**-க்கு அனுப்புகிறது (இங்கே [Settings > Advanced](#settings-advanced) பார்க்கவும்).

இது மிகவும் முக்கியமான சிக்கல்களை முதலில் வெளிப்படுத்துவதன் மூலம் கையேடு மதிப்பாய்வு நேரத்தைக் குறைக்கிறது.

## Settings > Advanced {#settings-advanced}

கருத்து பின்னணியை (feedback backend) இயக்க, **Gratis AI Agent → Settings → Advanced** என்ற பிரிவின் கீழ் பின்வரும் புலங்களை உள்ளமைக்கவும்:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | கருத்துப் சுமை (feedback payloads) கொண்ட POST கோரிக்கைகளைப் பெறும் URL (JSON). |
| **Feedback API Key** | கருத்து சமர்ப்பிப்புகளை அங்கீகரிக்க `Authorization` தலைப்பில் அனுப்பப்படும் Bearer token. |

இரண்டு புலங்களையும் காலியாக விடுவதன் மூலம் கருத்து சேகரிப்பை முடக்கலாம். தனிப்பட்ட கருத்து பொத்தான்கள் பயனர்களுக்குத் தொடர்ந்து தெரியும், ஆனால் சமர்ப்பிப்புகள் அனுப்பப்படாது.

:::tip
கருத்து Endpoint ஆனது குறைந்தபட்சம் `message_id`, `conversation_id`, `feedback_text`, மற்றும் `triage_category` புலங்களைக் கொண்ட JSON body-ஐ ஏற்க வேண்டும். எதிர்பார்க்கப்படும் ஸ்கீமாவுக்காக உங்கள் endpoint வழங்குநரின் ஆவணங்களைப் பார்க்கவும்.
:::
