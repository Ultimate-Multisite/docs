# டொமைன் மேப்பிங்கை எவ்வாறு கட்டமைப்பது (v2)

_**முக்கிய குறிப்பு: இந்தக் கட்டுரை Ultimate Multisite பதிப்பு 2.x ஐக் குறிக்கிறது.**_

பிரீமியம் நெட்வொர்க்கின் மிகவும் சக்திவாய்ந்த அம்சங்களில் ஒன்று, நமது வாடிக்கையாளர்களுக்கு அவர்களின் தளங்களில் ஒரு top-level domain ஐ இணைக்கும் வாய்ப்பை வழங்குவது. எல்லாவற்றிற்கும் மேலாக, எது மிகவும் தொழில்முறையாகத் தெரிகிறது: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) அல்லது [_**joesbikeshop.com**_](http://joesbikeshop.com)? அதனால்தான் Ultimate Multisite இந்த அம்சத்தை உள்ளமைத்து வழங்குகிறது, மூன்றாம் தரப்பு plugin-களைப் பயன்படுத்த வேண்டிய அவசியமில்லை.

## டொமைன் மேப்பிங் என்றால் என்ன?

பெயர் குறிப்பிடுவது போல், டொமைன் மேப்பிங் என்பது ஒரு தனிப்பயன் டொமைனுக்கான கோரிக்கையை எடுத்து, அந்த குறிப்பிட்ட டொமைன் இணைக்கப்பட்ட நெட்வொர்க்கில் உள்ள தொடர்புடைய தளத்திற்கு அந்தக் கோரிக்கையை மேப் செய்யும் திறன் ஆகும், இது Ultimate Multisite வழங்குகிறது.

### உங்கள் Ultimate Multisite நெட்வொர்க்கில் டொமைன் மேப்பிங்கை எவ்வாறு அமைப்பது

டொமைன் மேப்பிங் வேலை செய்ய உங்கள் பக்கத்தில் சில அமைப்புகள் தேவை. அதிர்ஷ்டவசமாக, Ultimate Multisite கடினமான வேலையை உங்களுக்காக தானியங்குபடுத்துகிறது, எனவே நீங்கள் தேவைகளை எளிதாக பூர்த்தி செய்யலாம்.

Ultimate Multisite நிறுவலின் போது, wizard தானாகவே **sunrise.php** ஐ நியமிக்கப்பட்ட folder-க்கு நகலெடுத்து நிறுவும். **இந்த படி முடியும் வரை wizard உங்களை தொடர அனுமதிக்காது**.

![sunrise.php படியுடன் Ultimate Multisite நிறுவல் wizard](/img/config/settings-domain-mapping.png)

இதன் பொருள், Ultimate Multisite நிறுவல் wizard உங்கள் நெட்வொர்க்கை அமைத்து முடித்தவுடன், நீங்கள் உடனடியாக தனிப்பயன் டொமைனை மேப் செய்யத் தொடங்கலாம்.

Ultimate Multisite-ல் டொமைன் மேப்பிங் கட்டாயமில்லை என்பதை நினைவில் கொள்ளுங்கள். WordPress Multisite-ன் சொந்த டொமைன் மேப்பிங் செயல்பாடு அல்லது வேறு எந்த டொமைன் மேப்பிங் தீர்வையும் பயன்படுத்த உங்களுக்கு விருப்பம் உள்ளது.

மற்ற டொமைன் மேப்பிங் தீர்வுகளுக்கு வழிவிட Ultimate Multisite டொமைன் மேப்பிங்கை முடக்க வேண்டுமானால், **Ultimate Multisite > Settings > Domain Mapping** என்பதன் கீழ் இந்த அம்சத்தை முடக்கலாம்.

![enable toggle-உடன் Domain Mapping அமைப்புகள்](/img/config/settings-domain-mapping.png)

இந்த விருப்பத்திற்கு கீழே, **Force Admin Redirect** என்ற விருப்பத்தையும் நீங்கள் காணலாம். இந்த விருப்பம், உங்கள் வாடிக்கையாளர்கள் தங்கள் admin dashboard-ஐ அவர்களின் தனிப்பயன் டொமைன் மற்றும் subdomain இரண்டிலும் அணுக முடியுமா அல்லது ஒன்றில் மட்டும் அணுக முடியுமா என்பதைக் கட்டுப்படுத்த உங்களை அனுமதிக்கிறது.

நீங்கள் **Force redirect to mapped domain** ஐத் தேர்வு செய்தால், உங்கள் வாடிக்கையாளர்கள் தங்கள் admin dashboard-ஐ அவர்களின் தனிப்பயன் டொமைன்களில் மட்டுமே அணுக முடியும்.

**Force redirect to network domain** விருப்பம் சரியாக எதிர்மாறாகச் செய்யும் - உங்கள் வாடிக்கையாளர்கள் தங்கள் தனிப்பயன் டொமைன்களில் உள்நுழைய முயற்சித்தாலும், அவர்களின் subdomain-ல் மட்டுமே தங்கள் dashboard-களை அணுக அனுமதிக்கப்படுவார்கள்.

மேலும் **Allow access to the admin by both mapped domain and network domain** விருப்பம், subdomain மற்றும் தனிப்பயன் டொமைன் இரண்டிலும் தங்கள் admin dashboard-களை அணுக அவர்களை அனுமதிக்கிறது.

![டொமைன் மேப்பிங்கிற்கான Force Admin Redirect விருப்பங்கள்](/img/config/settings-domain-mapping.png)

தனிப்பயன் டொமைனை மேப் செய்ய இரண்டு வழிகள் உள்ளன. முதலாவது super admin-ஆக உங்கள் network admin dashboard-லிருந்து டொமைன் பெயரை மேப் செய்வது, இரண்டாவது account page-ன் கீழ் subsite admin dashboard மூலம் செய்வது.

ஆனால் உங்கள் நெட்வொர்க்கில் உள்ள subsite-களில் ஒன்றிற்கு தனிப்பயன் டொமைனை மேப் செய்யத் தொடங்குவதற்கு முன், டொமைன் பெயரின் **DNS அமைப்புகள்** சரியாக கட்டமைக்கப்பட்டுள்ளதா என்பதை உறுதிசெய்ய வேண்டும்.

### 

### டொமைன் DNS அமைப்புகள் சரியாக கட்டமைக்கப்பட்டுள்ளதை உறுதிசெய்தல்

மேப்பிங் வேலை செய்ய, நீங்கள் மேப் செய்ய திட்டமிடும் டொமைன் உங்கள் நெட்வொர்க்கின் IP முகவரியை சுட்டிக்காட்டுகிறது என்பதை உறுதிசெய்ய வேண்டும். உங்களுக்கு நெட்வொர்க் IP முகவரி தேவை - Ultimate Multisite நிறுவப்பட்ட டொமைனின் IP முகவரி - நீங்கள் மேப் செய்ய விரும்பும் தனிப்பயன் டொமைனின் IP முகவரி அல்ல என்பதை நினைவில் கொள்ளுங்கள். ஒரு குறிப்பிட்ட டொமைனின் IP முகவரியைத் தேட, எடுத்துக்காட்டாக [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) க்குச் செல்ல பரிந்துரைக்கிறோம்.

டொமைனை சரியாக மேப் செய்ய, அந்த **IP முகவரியை** சுட்டிக்காட்டும் உங்கள் **DNS** கட்டமைப்பில் ஒரு **A RECORD** சேர்க்க வேண்டும். DNS மேலாண்மை வெவ்வேறு domain registrar-களுக்கு இடையே பெரிதும் மாறுபடுகிறது, ஆனால் "_XXXX-ல் A Record உருவாக்குதல்_" என்று தேடினால், XXXX என்பது உங்கள் domain registrar (எ.கா.: "_GoDaddy-ல் A Record உருவாக்குதல்_"), இதை உள்ளடக்கிய ஏராளமான tutorials ஆன்லைனில் உள்ளன.

இது வேலை செய்வதில் சிக்கல் இருந்தால், **உங்கள் domain registrar support-ஐ தொடர்பு கொள்ளுங்கள்**, அவர்கள் இந்த பகுதியில் உங்களுக்கு உதவ முடியும்.

உங்கள் வாடிக்கையாளர்கள் தங்கள் சொந்த டொமைன்களை மேப் செய்ய அனுமதிக்க திட்டமிட்டால், அவர்கள் இந்த பகுதியில் வேலையை தாங்களே செய்ய வேண்டும். A Record உருவாக்க இயலாவிட்டால், அவர்களின் registrar support system-ஐ அவர்களுக்கு சுட்டிக்காட்டுங்கள்.

### Super Admin-ஆக தனிப்பயன் டொமைன் பெயரை மேப் செய்தல்

உங்கள் நெட்வொர்க்கில் super admin-ஆக உள்நுழைந்திருக்கும்போது, **Ultimate Multisite > Domains** க்குச் சென்று தனிப்பயன் டொமைன் பெயர்களை எளிதாக சேர்க்கலாம் மற்றும் நிர்வகிக்கலாம்.

![Ultimate Multisite-ல் Domains list page](/img/admin/domains-list.png)

இந்த page-ன் கீழ், மேலே உள்ள **Add Domain** பொத்தானைக் கிளிக் செய்யலாம், இது ஒரு modal window-ஐக் கொண்டு வரும், அங்கு நீங்கள் **தனிப்பயன் டொமைன் பெயர்**, தனிப்பயன் டொமைன் பெயரைப் பயன்படுத்த விரும்பும் **subsite**, மற்றும் அதை **முதன்மை டொமைன்** பெயராக அமைக்க விரும்புகிறீர்களா என்பதை அமைக்கலாம் மற்றும் நிரப்பலாம் (நீங்கள் **ஒரு subsite-க்கு பல டொமைன் பெயர்களை** மேப் செய்யலாம் என்பதை நினைவில் கொள்ளுங்கள்).

![டொமைன் பெயர் மற்றும் site புலங்களுடன் Add Domain modal](/img/admin/domains-list.png)

அனைத்து தகவல்களையும் உள்ளிட்ட பிறகு, கீழே உள்ள **Add Existing Domain** பொத்தானைக் கிளிக் செய்யலாம்.

இது தனிப்பயன் டொமைனின் DNS தகவலை சரிபார்த்து பெறும் செயல்முறையைத் தொடங்கும். செயல்முறையைப் பின்பற்ற page-ன் கீழே ஒரு log-ஐயும் காண்பீர்கள். இந்த செயல்முறை முடிவடைய சில நிமிடங்கள் ஆகலாம்.

எல்லாம் சரியாக அமைக்கப்பட்டிருந்தால், **Stage** அல்லது நிலை **Checking DNS**-லிருந்து **Ready**-க்கு மாற வேண்டும்.

![Checking DNS-லிருந்து Ready-க்கு மாறும் Domain stage](/img/admin/domains-list.png)

![domains list-ல் Ready நிலையுடன் கூடிய Domain](/img/admin/domains-list.png)

டொமைன் பெயரைக் கிளிக் செய்தால், அதனுள் சில விருப்பங்களைக் காணலாம். அவற்றை விரைவாகப் பார்ப்போம்:

![stage, site, active மற்றும் SSL விருப்பங்களுடன் Domain detail page](/img/admin/domains-list.png)

**Stage:** டொமைன் இருக்கும் நிலை இது. நீங்கள் முதலில் டொமைனைச் சேர்க்கும்போது, அது **Checking DNS** நிலையில் இருக்கும். செயல்முறை DNS entries-ஐ சரிபார்த்து அவை சரியானவை என்பதை உறுதிப்படுத்தும். பின்னர், டொமைன் **Checking SSL** நிலையில் வைக்கப்படும். Ultimate Multisite டொமைனுக்கு SSL உள்ளதா இல்லையா என்பதை சரிபார்த்து, உங்கள் டொமைனை **Ready** அல்லது **Ready (without SSL)** என வகைப்படுத்தும்.

**Site:** இந்த டொமைனுடன் தொடர்புடைய subdomain. மேப் செய்யப்பட்ட டொமைன் இந்த குறிப்பிட்ட தளத்தின் உள்ளடக்கத்தைக் காண்பிக்கும்.

**Active:** டொமைனை செயல்படுத்த அல்லது செயலிழக்க இந்த விருப்பத்தை on அல்லது off செய்யலாம்.

**Is Primary Domain?:** உங்கள் வாடிக்கையாளர்கள் ஒவ்வொரு தளத்திற்கும் ஒன்றுக்கு மேற்பட்ட மேப் செய்யப்பட்ட டொமைன்களை வைத்திருக்கலாம். குறிப்பிட்ட தளத்திற்கு இது முதன்மை டொமைனா என்பதைத் தேர்ந்தெடுக்க இந்த விருப்பத்தைப் பயன்படுத்தவும்.

**Is Secure?:** Ultimate Multisite டொமைனை செயல்படுத்துவதற்கு முன் அதற்கு SSL certificate உள்ளதா இல்லையா என்பதை சரிபார்த்தாலும், SSL certificate-உடன் அல்லது இல்லாமல் டொமைனை load செய்ய நீங்கள் கைமுறையாகத் தேர்ந்தெடுக்கலாம். வலைத்தளத்தில் SSL certificate இல்லை மற்றும் SSL-உடன் load செய்ய முயற்சித்தால், அது பிழைகளை ஏற்படுத்தலாம் என்பதை நினைவில் கொள்ளுங்கள்.

### Subsite பயனராக தனிப்பயன் டொமைன் பெயரை மேப் செய்தல்

Subsite நிர்வாகிகளும் தங்கள் subsite admin dashboard-லிருந்து தனிப்பயன் டொமைன் பெயர்களை மேப் செய்யலாம்.

முதலில், **Domain mapping** அமைப்புகளின் கீழ் இந்த விருப்பத்தை நீங்கள் செயல்படுத்துவதை உறுதிசெய்ய வேண்டும். கீழே உள்ள screenshot-ஐ பாருங்கள்.

![subsite பயனர்கள் டொமைன்களை மேப் செய்ய அனுமதிக்கும் Domain mapping அமைப்புகள்](/img/config/settings-domain-mapping.png)

**Ultimate Multisite > Products**-ல் உள்ள **Plan** நிலை அல்லது product விருப்பங்களின் கீழும் இந்த விருப்பத்தை அமைக்கலாம் அல்லது கட்டமைக்கலாம்.

### Domain Syncing பற்றி

Domain Syncing என்பது **டொமைன் மேப்பிங் வேலை செய்ய** Ultimate Multisite தனிப்பயன் டொமைன் பெயரை உங்கள் hosting கணக்கில் add-on domain-ஆக சேர்க்கும் ஒரு செயல்முறை ஆகும்.

உங்கள் hosting provider-க்கு Ultimate Multisite டொமைன் மேப்பிங் அம்சத்துடன் integration இருந்தால், Domain syncing தானாக நிகழும். தற்போது, இந்த hosting provider-கள் _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ மற்றும் _Cpanel._

Ultimate Multisite அமைப்புகளில் **Integration** tab-ன் கீழ் இந்த integration-ஐ செயல்படுத்த வேண்டும்.

![Ultimate Multisite அமைப்புகளில் Integration tab](/img/config/settings-domain-mapping.png)

![domain syncing-க்கான Hosting provider integration அமைப்புகள்](/img/config/settings-domain-mapping.png)

_உங்கள் hosting provider மேலே குறிப்பிடப்பட்ட provider-களில் ஒன்றாக இல்லாவிட்டால், **டொமைன் பெயரை உங்கள் hosting கணக்கில் கைமுறையாக sync செய்ய அல்லது சேர்க்க வேண்டும்** என்பதை நினைவில் கொள்ளுங்கள்._
