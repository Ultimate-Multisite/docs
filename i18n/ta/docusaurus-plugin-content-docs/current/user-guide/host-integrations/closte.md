# Closte ஒருங்கிணைப்பு

## மேலோட்டம்
Closte என்பது Google Cloud உள்கட்டமைப்பில் கட்டமைக்கப்பட்ட ஒரு managed WordPress hosting தளமாகும். இந்த ஒருங்கிணைப்பு Ultimate Multisite மற்றும் Closte இடையே தானியங்கி domain ஒத்திசைவு மற்றும் SSL சான்றிதழ் மேலாண்மையை செயல்படுத்துகிறது.

## அம்சங்கள்
- தானியங்கி domain ஒத்திசைவு
- SSL சான்றிதழ் மேலாண்மை
- Wildcard domain ஆதரவு
- Closte-ல் இயங்கினால் எந்த கட்டமைப்பும் தேவையில்லை

## தேவைகள்
நீங்கள் Closte பயன்படுத்துகிறீர்கள் என்றால், உங்கள் `wp-config.php` கோப்பில் பின்வரும் constant வரையறுக்கப்பட வேண்டும்:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

நீங்கள் Closte-ல் hosting செய்கிறீர்கள் என்றால், இந்த constant பொதுவாக ஏற்கனவே வரையறுக்கப்பட்டிருக்கும்.

## அமைவு வழிமுறைகள்

### 1. உங்கள் Closte API Key-ஐ சரிபார்க்கவும்

நீங்கள் Closte-ல் hosting செய்கிறீர்கள் என்றால், `CLOSTE_CLIENT_API_KEY` constant ஏற்கனவே உங்கள் `wp-config.php` கோப்பில் வரையறுக்கப்பட்டிருக்க வேண்டும். உங்கள் `wp-config.php` கோப்பைச் சரிபார்த்து இதை உறுதிசெய்யலாம்.

### 2. ஒருங்கிணைப்பை இயக்கவும்

1. உங்கள் WordPress admin-ல், Ultimate Multisite > Settings என்பதற்குச் செல்லவும்
2. "Domain Mapping" tab-க்கு செல்லவும்
3. "Host Integrations" பகுதிக்கு கீழே scroll செய்யவும்
4. Closte ஒருங்கிணைப்பை இயக்கவும்
5. "Save Changes" என்பதைக் கிளிக் செய்யவும்

## இது எப்படி வேலை செய்கிறது

Ultimate Multisite-ல் ஒரு domain map செய்யப்படும்போது:

1. உங்கள் application-ல் domain-ஐ சேர்க்க ஒருங்கிணைப்பு Closte-ன் API-க்கு ஒரு request அனுப்புகிறது
2. SSL சான்றிதழ் வழங்குதலை Closte தானாகவே கையாளுகிறது
3. ஒரு domain mapping நீக்கப்படும்போது, ஒருங்கிணைப்பு அந்த domain-ஐ Closte-லிருந்து நீக்கிவிடும்

Ultimate Multisite-ல் உள்ள DNS சரிபார்ப்பு இடைவெளி அமைப்புடனும் இந்த ஒருங்கிணைப்பு வேலை செய்கிறது, இது DNS propagation மற்றும் SSL சான்றிதழ் வழங்குதலை எவ்வளவு அடிக்கடி சரிபார்க்க வேண்டும் என்பதை கட்டமைக்க உங்களை அனுமதிக்கிறது.

## Domain Record உருவாக்கம்

ஒரு site உருவாக்கப்படும்போது அல்லது duplicate செய்யப்படும்போது, domain record தானாகவே உருவாக்கப்படுவதை இந்த ஒருங்கிணைப்பு உறுதிசெய்கிறது. Closte ஒருங்கிணைப்புக்கு இது மிகவும் முக்கியம், ஏனெனில் domain record உருவாக்கம் Closte API-ஐ trigger செய்து domain மற்றும் SSL சான்றிதழை உருவாக்குகிறது.

## சிக்கல் தீர்வு

### API இணைப்பு சிக்கல்கள்
- உங்கள் Closte API key சரியானதா என்பதை சரிபார்க்கவும்
- உங்கள் Closte கணக்கில் தேவையான அனுமதிகள் உள்ளனவா என்பதை உறுதிசெய்யவும்

### SSL சான்றிதழ் சிக்கல்கள்
- SSL சான்றிதழ்களை வழங்க Closte-க்கு சிறிது நேரம் ஆகலாம் (பொதுவாக 5-10 நிமிடங்கள்)
- உங்கள் domains சரியாக உங்கள் Closte server-ன் IP முகவரியை சுட்டிக்காட்டுகின்றனவா என்பதை சரிபார்க்கவும்
- உங்கள் domain-க்கான DNS records சரியாக கட்டமைக்கப்பட்டுள்ளனவா என்பதை சரிபார்க்கவும்

### Domain சேர்க்கப்படவில்லை
- ஏதேனும் error messages உள்ளனவா என்று Ultimate Multisite logs-ஐ சரிபார்க்கவும்
- அந்த domain ஏற்கனவே Closte-ல் சேர்க்கப்படவில்லை என்பதை உறுதிசெய்யவும்
- உங்கள் domain-ன் DNS records சரியாக கட்டமைக்கப்பட்டுள்ளனவா என்பதை உறுதிசெய்யவும்

### DNS சரிபார்ப்பு இடைவெளி
- SSL சான்றிதழ்கள் வழங்க அதிக நேரம் எடுத்தால், Domain Mapping settings-ல் DNS சரிபார்ப்பு இடைவெளியை மாற்றலாம்
- இயல்புநிலை இடைவெளி 300 விநாடிகள் (5 நிமிடங்கள்), ஆனால் testing-ன் போது வேகமான சரிபார்ப்புக்கு 10 விநாடிகள் வரை குறைக்கலாம்
