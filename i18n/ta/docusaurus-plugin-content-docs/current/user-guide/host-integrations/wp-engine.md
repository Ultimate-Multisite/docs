# WP Engine ஒருங்கிணைப்பு

## மேலோட்டம்
WP Engine என்பது WordPress தளங்களுக்கு உகந்த செயல்திறன், பாதுகாப்பு மற்றும் அளவிடுதல் திறனை வழங்கும் ஒரு பிரீமியம் நிர்வகிக்கப்படும் WordPress hosting தளமாகும். இந்த ஒருங்கிணைப்பு Ultimate Multisite மற்றும் WP Engine இடையே தானியங்கி domain ஒத்திசைவை செயல்படுத்துகிறது.

## அம்சங்கள்
- தானியங்கி domain ஒத்திசைவு
- Multisite நிறுவல்களுக்கு subdomain ஆதரவு
- WP Engine-இன் தற்போதைய அமைப்புகளுடன் தடையற்ற ஒருங்கிணைப்பு

## தேவைகள்
நீங்கள் WP Engine-இல் hosting செய்கிறீர்களா என்பதை ஒருங்கிணைப்பு தானாகவே கண்டறிந்து உள்ளமைந்த WP Engine API-ஐ பயன்படுத்துகிறது. WP Engine plugin செயலில் இருந்து சரியாக கட்டமைக்கப்பட்டிருந்தால் கூடுதல் உள்ளமைவு தேவையில்லை.

எனினும், நீங்கள் ஒருங்கிணைப்பை கைமுறையாக கட்டமைக்க வேண்டுமெனில், உங்கள் `wp-config.php` கோப்பில் இந்த constant-களில் ஒன்றை வரையறுக்கலாம்:

```php
define('WPE_APIKEY', 'your_api_key'); // விருப்பமான முறை
// OR
define('WPE_API', 'your_api_key'); // மாற்று முறை
```

## அமைவு வழிமுறைகள்

### 1. WP Engine Plugin-ஐ சரிபார்க்கவும்

நீங்கள் WP Engine-இல் hosting செய்கிறீர்கள் என்றால், WP Engine plugin ஏற்கனவே நிறுவப்பட்டு செயல்படுத்தப்பட்டிருக்க வேண்டும். பின்வருவனவற்றை சரிபார்க்கவும்:

1. WP Engine plugin செயலில் உள்ளது
2. `wp-content/mu-plugins/wpengine-common/class-wpeapi.php` கோப்பு உள்ளது

### 2. ஒருங்கிணைப்பை இயக்கவும்

1. உங்கள் WordPress admin-இல், Ultimate Multisite > Settings என்பதற்குச் செல்லவும்
2. "Domain Mapping" tab-க்கு செல்லவும்
3. "Host Integrations" பகுதிக்கு கீழே உருட்டவும்
4. WP Engine ஒருங்கிணைப்பை இயக்கவும்
5. "Save Changes" என்பதைக் கிளிக் செய்யவும்

## இது எப்படி செயல்படுகிறது

### Domain ஒத்திசைவு

Ultimate Multisite-இல் ஒரு domain mapping செய்யப்படும்போது:

1. ஒருங்கிணைப்பு WP Engine API-ஐ பயன்படுத்தி உங்கள் WP Engine நிறுவலில் domain-ஐ சேர்க்கிறது
2. WP Engine domain கட்டமைப்பு மற்றும் SSL சான்றிதழ் வழங்குதலை கையாளுகிறது
3. Domain mapping நீக்கப்படும்போது, ஒருங்கிணைப்பு WP Engine-இலிருந்து domain-ஐ நீக்கும்

### Subdomain ஆதரவு

Subdomain multisite நிறுவல்களுக்கு:

1. புதிய தளம் உருவாக்கப்படும்போது ஒருங்கிணைப்பு ஒவ்வொரு subdomain-ஐயும் WP Engine-இல் சேர்க்கிறது
2. WP Engine subdomain கட்டமைப்பை கையாளுகிறது
3. ஒரு தளம் நீக்கப்படும்போது, ஒருங்கிணைப்பு WP Engine-இலிருந்து subdomain-ஐ நீக்கும்

## முக்கியமான குறிப்புகள்

### Wildcard Domains

Subdomain multisite நிறுவல்களுக்கு, wildcard domain கட்டமைப்பை கோர WP Engine support-ஐ தொடர்பு கொள்ள பரிந்துரைக்கப்படுகிறது. இது ஒவ்வொன்றையும் தனித்தனியாக சேர்க்காமல் எல்லா subdomain-களும் தானாகவே செயல்பட அனுமதிக்கிறது.

### SSL சான்றிதழ்கள்

இந்த ஒருங்கிணைப்பு மூலம் சேர்க்கப்படும் அனைத்து domain-களுக்கும் SSL சான்றிதழ் வழங்குதல் மற்றும் புதுப்பித்தலை WP Engine தானாகவே கையாளுகிறது. கூடுதல் கட்டமைப்பு தேவையில்லை.

## சிக்கல் தீர்வு

### API இணைப்பு சிக்கல்கள்
- WP Engine plugin செயலில் உள்ளது மற்றும் சரியாக கட்டமைக்கப்பட்டுள்ளது என்பதை சரிபார்க்கவும்
- நீங்கள் API key-ஐ கைமுறையாக வரையறுத்திருந்தால், அது சரியானதா என்று சரிபார்க்கவும்
- API-இல் சிக்கல் இருந்தால் WP Engine support-ஐ தொடர்பு கொள்ளவும்

### Domain சேர்க்கப்படவில்லை
- ஏதேனும் பிழை செய்திகளுக்கு Ultimate Multisite logs-ஐ சரிபார்க்கவும்
- Domain ஏற்கனவே WP Engine-இல் சேர்க்கப்படவில்லை என்பதை உறுதிப்படுத்தவும்
- நீங்கள் சேர்க்கும் domain-களின் எண்ணிக்கையை உங்கள் WP Engine plan ஆதரிக்கிறது என்பதை உறுதிப்படுத்தவும்

### Subdomain சிக்கல்கள்
- Subdomain-கள் செயல்படவில்லை என்றால், wildcard domain கட்டமைப்பை கோர WP Engine support-ஐ தொடர்பு கொள்ளவும்
- உங்கள் DNS அமைப்புகள் முதன்மை domain மற்றும் subdomain-களுக்கு சரியாக கட்டமைக்கப்பட்டுள்ளதா சரிபார்க்கவும்
