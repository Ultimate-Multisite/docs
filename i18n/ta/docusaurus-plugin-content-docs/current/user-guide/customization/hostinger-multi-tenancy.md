# ஹோஸ்டிங்கர் மல்டி-டென்டன்சி

Ultimate Multisite: Multi-Tenancy 1.2.0 ஆனது ஒரு ஹோஸ்டிங்கர் மல்டி-டென்டன்சி திறனைச் சேர்க்கிறது. இதன் மூலம், ஏற்கனவே உள்ள ஹோஸ்டிங்கர் டொமைன் மேப்பிங் ஒருங்கிணைப்புடன், ஹோஸ்ட் செய்யப்பட்ட டொமைன்களும் தனித்தனி டென்டண்டுகளை (sovereign tenant) வழங்குவதில் பங்கேற்க முடியும்.

ஹோஸ்டிங்கர் hPanel-இல் டென்டண்ட் டொமைன் மற்றும் தனிமைப்படுத்தப்பட்ட டென்டண்ட் உள்கட்டமைப்பை நிர்வகிக்க வேண்டியிருக்கும் போது இந்த வசதியை பயன்படுத்தவும்.

## அமைப்பதற்கான குறிப்புகள்

1. **Ultimate Multisite > Settings > Host Integrations**-க்குள் கோர் ஹோஸ்டிங்கர் ஒருங்கிணைப்பை (core Hostinger integration) உள்ளமைக்கவும்.
2. இலக்கு டொமைன் அல்லது सबடொமைனை நிர்வகிக்க ஹோஸ்டிங்கர் API டோக்கன் பயன்படுத்த முடியுமா என்பதை உறுதிப்படுத்தவும்.
3. மல்டி-டென்டன்சி addon-ஐ இயக்கவும்.
4. டென்டண்டை வெளியிடுவதற்கு (publishing) முன்பு டென்டண்ட் தனிமைப்படுத்தல் உத்தியை உள்ளமைக்கவும்.
5. உற்பத்தி டிராஃபிக்கை டென்ண்டண்டிற்கு அனுப்பும் முன், மிகிரேஷன் சரிபார்ப்பு பணிப்பாய்வை (migration verification workflow) இயக்கவும்.

ஹோஸ்டிங்கர் வசதி ஹோஸ்ட் பக்க செயல்பாடுகளுக்கு பகிரப்பட்ட ஹோஸ்டிங்கர் இணைப்பை (shared Hostinger connection) பயன்படுத்துகிறது. இருப்பினும், DNS சரியான ஹோஸ்டிங்கர் கணக்கை (Hostinger account) சுட்டிக்காட்ட வேண்டும், மேலும் hPanel கணக்கு வரம்புகள் இன்னும் பொருந்தும்.

## வசதி சார்ந்த மாற்றங்கள்

- ஹோஸ்ட்-அறிவார்ந்த டொமைன் செயல்பாடுகளுடன் தனித்தனி டென்டண்டுகளை உருவாக்க முடியும்.
- அனுமதி சரிபார்ப்புக்கு முன்பு ஒரே இயந்திரத்தில் உள்ள தரவுத்தள ஹோஸ்ட் சரங்கள் (database host strings) இயல்பாக்கப்படும் (normalized).
- WordPress运行时க்கு உள்ளூர் மேலெழுதல்கள் (local override) தேவைப்படாவிட்டால், Hostinger மூலம் நிர்வகிக்கப்படும் டென்டண்டுகள் hPanel-இல் காட்டப்படும் தரவுத்தள ஹோஸ்ட் மதிப்பை பயன்படுத்த வேண்டும்.
- SSO வருகைகள் (SSO visits) டென்டண்ட் டொமைன் ஹோஸ்டிங்கர் மூலம் ஹோஸ்ட் செய்யப்படும் டென்டன்டை சுட்டிக்காட்டுவதை நம்பியுள்ளன.

## ஹோஸ்டிங்கர் டென்டண்டுகளைத் தீர்க்கும் வழிமுறைகள் (Troubleshooting Hostinger tenants)

- ஒரு டென்டண்ட் நிறுவல் தோல்வியடைந்தால், டொமைன் ஏற்கனவே ஹோஸ்டிங்கர் கணக்கில் இணைக்கப்பட்டுள்ளதா என்பதைச் சரிபார்க்கவும்.
- தரவுத்தள சரிபார்ப்பு தோல்வியடைந்தால், hPanel உடன் டென்டண்டின் DB பயனர் பெயர் (username), தரவுத்தளப் பெயர் (database name) மற்றும் ஹோஸ்ட் பிணைப்பை (host binding) ஒப்பிட்டுப் பார்க்கவும்.
- **Visit (SSO)** தோல்வியடைந்தால், டென்டண்ட் டொமைனுக்கு DNS மற்றும் SSL செயலில் உள்ளதா என்பதை உறுதிப்படுத்தவும்.
- நீக்கும்போது (teardown), காப்புப்பிரதிகளைச் சரிபார்த்த பிறகு, hPanel-இல் மீதமுள்ள தரவுத்தளங்கள், பயனர்கள் அல்லது கோப்புறைகளை அகற்றவும்.
