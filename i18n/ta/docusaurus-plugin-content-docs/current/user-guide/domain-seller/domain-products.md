# டொமைன் தயாரிப்புகள் மற்றும் விலை நிர்ணயம்

டொமைன் தயாரிப்புகள் (Domain products) மூலம் நீங்கள் விலை நிர்ணயம், TLDகள் (Top-Level Domains), WHOIS தனியுரிமை மற்றும் வழங்குநரை (provider) தேர்வு செய்வதை கட்டுப்படுத்தலாம். ஒவ்வொரு டொமைன் தயாரிப்பும் ஒரு நிலையான Multisite Ultimate தயாரிப்பாகும், அதனுடன் கூடுதலாக ஒரு **Domain Settings** டேப் சேர்க்கப்பட்டுள்ளது.

## டொமைன் தயாரிப்பை உருவாக்குதல்

1. **Network Admin › Ultimate Multisite › Products** என்பதற்குச் செல்லவும்
2. **Add New** என்பதைக் கிளிக் செய்யவும்
3. தயாரிப்பு வகையை (product type) **Domain** என்று அமைக்கவும்
4. **Domain Settings** டேப்பை உள்ளமைக்கவும் (கீழே பார்க்கவும்)
5. சேமிக்கவும் (Save)

டொமைன் தயாரிப்புகள், தயாரிப்புப் பட்டியலில் ஊதா நிற **Domain** பேட்ஜுடன் தோன்றும், மேலும் **Domain Products** டேப்பைப் பயன்படுத்தி வடிகட்ட (filter) முடியும்.

## Domain settings டேப்

### Provider (வழங்குநர்)

இந்த தயாரிப்புக்கான பதிவை (registration) எந்த ரெஜிஸ்ட்ரர் (registrar) கையாள வேண்டும் என்பதைத் தேர்ந்தெடுக்கவும். இது பொதுவாக Domain Seller அமைப்புகளில் உள்ள உலகளாவிய **Default provider** என்பதை இயல்புநிலையாகப் பயன்படுத்தும்.

### Supported TLDs (ஆதரவு TLDகள்)

**catch-all product** உருவாக்க இதை காலியாக விடவும். இது வேறு எந்த தயாரிப்பாலும் பொருந்தாத அனைத்து TLDகளுக்கும் பொருந்தும்.

ஒரு காற்புள்ளியால் பிரிக்கப்பட்ட TLDகளின் பட்டியலை உள்ளிடவும் (உதாரணமாக, `.com, .net, .org`) இதைச் செய்வதன் மூலம், அந்த நீட்டிப்புகளுக்கு மட்டுமே பொருந்தும் ஒரு **TLD-specific product** உருவாக்கப்படும்.

**தயாரிப்பு பொருத்தம் எப்படி வேலை செய்கிறது:** ஒரு வாடிக்கையாளர் ஒரு டொமைனைத் தேடும்போது, addon மிகவும் குறிப்பிட்ட பொருத்தமான தயாரிப்பைத் தேர்ந்தெடுக்கும். அதன் TLD பட்டியலில் `.com` உள்ள ஒரு தயாரிப்பு, catch-all தயாரிப்பை விட அதிக முன்னுரிமை பெறுகிறது. எந்த TLD-specific தயாரிப்பும் பொருந்தவில்லை என்றால், catch-all பயன்படுத்தப்படும். எந்த தயாரிப்புகளும் இல்லையென்றால், டொமைன் தேடல் காட்டப்படாது.

### Markup type (அதிகப்படியான விலை வகை)

மொத்த செலவில் இருந்து உங்கள் சில்லறை விலை (retail price) எப்படி கணக்கிடப்படுகிறது என்பதை மூன்று முறைகள் கட்டுப்படுத்துகின்றன:

| Mode (முறை) | இது எப்படி வேலை செய்கிறது |
|---|---|
| **Percentage (சதவீதம்)** | மொத்த செலவின் மேல் ஒரு சதவீதத்தை சேர்க்கிறது. $10 மொத்த டொமைன் மீது 20% Markup என்றால், $12 கிடைக்கும். |
| **Fixed markup (நிலையான Markup)** | ஒரு நிலையான டாலர் தொகையை சேர்க்கிறது. $10 டொமைன் மீது $5 Markup என்றால், $15 கிடைக்கும். |
| **Fixed price (நிலையான விலை)** | மொத்த செலவை முற்றிலும் புறக்கணிக்கிறது. நீங்கள் உள்ளிடும் தொகையை எப்போதும் வசூலிக்கும். |

### Introductory pricing (அறிமுக விலை நிர்ணயம்)

தள்ளுபடி செய்யப்பட்ட முதல் ஆண்டு விலையை வழங்க இதை இயக்கவும். வழக்கமான **Renewal price** (இரண்டாம் ஆண்டு மற்றும் அதற்கு மேல்) உடன் தனி **Introductory price** (முதல் ஆண்டு விலை) அமைக்கவும். வாடிக்கையாளருக்கு இரண்டு விலைகளும் checkout செய்யும் போது காட்டப்படும், இதனால் Renewal சமயத்தில் என்ன எதிர்பார்க்க வேண்டும் என்று அவர்களுக்குத் தெரியும்.

### WHOIS privacy (WHOIS தனியுரிமை)

இந்த தயாரிப்பு மூலம் பதிவு செய்யப்படும் டொமைன்களுக்கு WHOIS தனியுரிமை பாதுகாப்பு வழங்கப்படுகிறதா என்பதைக் கட்டுப்படுத்துகிறது.

| Setting (அமைப்பு) | Behaviour (நடத்தை) |
|---|---|
| **Disabled (செயலிழக்கப்பட்டது)** | WHOIS தனியுரிமை ஒருபோதும் வழங்கப்படவோ அல்லது இயக்கப்படவோ செய்யாது. |
| **Always Included (எப்போதும் சேர்க்கப்பட்டது)** | WHOIS தனியுரிமை, எந்த கட்டணமும் இல்லாமல், பதிவின் போது தானாகவே இயக்கப்படும். |
| **Customer Choice (வாடிக்கையாளர் தேர்வு)** | checkout செய்யும் போது ஒரு checkbox தோன்றும். ஆண்டுக்கு வசூலிக்க **Privacy price** என்பதை அமைக்கவும், அல்லது இலவசமாக வழங்க $0 என்று விடவும். |

Namecheap-க்கு, WHOIS தனியுரிமை WhoisGuard-ஐப் பயன்படுத்துகிறது (எப்போதும் இலவசம்). OpenSRS-க்கு, இது OpenSRS தனியுரிமை சேவையைப் பயன்படுத்துகிறது (மொத்த செலவில் கட்டணம் இருக்கலாம்).

## TLD இறக்குமதி மற்றும் ஒத்திசைவு (TLD import and sync)

டொமைன் தயாரிப்புகள், இணைக்கப்பட்ட வழங்குநரிடமிருந்து பெறப்பட்ட நிகழ்நேர மொத்த விலையைக் காண்பிக்கும். இது வேலை செய்ய, TLDகள் இறக்குமதி செய்யப்பட வேண்டும்.

- **Manual sync (கையால் ஒத்திசைவு):** Settings › Domain Seller › Sync TLDs (ஒவ்வொரு வழங்குநரிடமும்)
- **Automatic sync (தானியங்கி ஒத்திசைவு):** இது அனைத்து கட்டமைக்கப்பட்ட வழங்குநர்கள் மூலமாகவும் தினசரி ஒரு scheduled cron job மூலம் இயங்கும்.

ஒத்திசைவுக்குப் பிறகு, எந்த டொமைன் தயாரிப்பின் Domain Settings டேப்பிற்குச் சென்று, கிடைக்கக்கூடிய TLDகள் மற்றும் அவற்றின் தற்போதைய மொத்த விலைகளைக் காண TLD picker-ஐப் பயன்படுத்தவும்.

## Auto-renewal (தானியங்கி புதுப்பித்தல்)

டொமைன் புதுப்பித்தல்கள் வாடிக்கையாளரின் உறுப்பினர் நிலையைப் பொறுத்தது:

- ஒரு உறுப்பினர் புதுப்பிக்கப்பட்டு, ஒரு டொமைன் இணைக்கப்பட்டால், டொமைன் புதுப்பித்தல் தானாகவே வரிசைப்படுத்தப்படும் (queued).
- புதுப்பித்தல் முயற்சிகள் வாடிக்கையாளரின் செயலில் உள்ள Payment Gateway-ஐப் பயன்படுத்துகின்றன.
- தோல்வியடைந்த புதுப்பித்தல்கள் exponential backoff உடன் தானாகவே மீண்டும் முயற்சி செய்யப்படும்.
- வெற்றிகரமான புதுப்பித்தல்கள், தோல்விகள் மற்றும் வரவிருக்கும் காலாவதிகள் குறித்து மின்னஞ்சல் அறிவிப்புகள் அனுப்பப்படும்.

டொமைன் வாழ்க்கைச் சுழற்சி நிகழ்வுகளுக்கான மின்னஞ்சல் template IDகள்:

| Event (நிகழ்வு) | Template ID |
|---|---|
| Domain registered (டொமைன் பதிவு செய்யப்பட்டது) | `domain_registered` |
| Domain renewed (டொமைன் புதுப்பிக்கப்பட்டது) | `domain_renewed` |
| Renewal failed (புதுப்பித்தல் தோல்வியடைந்தது) | `domain_renewal_failed` |
| Domain expiring soon (டொமைன் விரைவில் காலாவதியாகும்) | `domain_expiring_soon` |

## Admin: Manual domain registration (நிர்வாகம்: கையேடு டொமைன் பதிவு)

வாடிக்கையாளர் checkout செய்வதில்லாமல், அவர்களின் சார்பாக ஒரு டொமைனைப் பதிவு செய்ய:

1. **Network Admin › Ultimate Multisite › Register Domain** என்பதற்குச் செல்லவும்
2. வாடிக்கையாளரைத் தேர்ந்தெடுத்து டொமைன் பெயரை உள்ளிடவும்
3. பதிவாளரின் தொடர்பு விவரங்களை நிரப்பவும் (பெயர், முகவரி, தொலைபேசி)
4. **Register** என்பதைக் கிளிக் செய்யவும்

டொமைன் பதிவு உருவாக்கப்பட்டு, வாடிக்கையாளரின் கணக்குடன் இணைக்கப்படும்.
