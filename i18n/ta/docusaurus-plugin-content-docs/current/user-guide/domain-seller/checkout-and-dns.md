# Checkout புலமும் வாடிக்கையாளர் DNS மேலாண்மையும் {#checkout-field-and-customer-dns-management}

## டொமைன் தேர்வு checkout புலம் {#the-domain-selection-checkout-field}

**டொமைன் தேர்வு** புலம் என்பது, வாடிக்கையாளர்கள் தங்கள் தளத்தின் டொமைனை எவ்வாறு பெறுவது என்பதைத் தேர்வுசெய்ய உதவும் checkout கூறாகும். டொமைன் விற்பனையை இயக்க, இதை எந்த checkout படிவத்திலும் சேர்க்கவும்.

### checkout படிவத்தில் புலத்தைச் சேர்த்தல் {#adding-the-field-to-a-checkout-form}

1. **நெட்வொர்க் நிர்வாகம் › Ultimate Multisite › Checkout படிவங்கள்** என்பதற்கு செல்லவும்
2. checkout படிவத்தைத் திறக்கவும் அல்லது உருவாக்கவும்
3. checkout திருத்தியில், **புலம் சேர்க்கவும்** என்பதைக் கிளிக் செய்யவும்
4. புலப் பட்டியலில் இருந்து **டொமைன் தேர்வு** என்பதைத் தேர்வுசெய்யவும்
5. புல விருப்பங்களை அமைக்கவும் (கீழே பார்க்கவும்)
6. படிவத்தைச் சேமிக்கவும்

### புல விருப்பங்கள் {#field-options}

**டொமைன் முறைகள்** — வாடிக்கையாளர் எந்த தாவல்களைப் பார்க்க வேண்டும் என்பதைத் தேர்வுசெய்யவும். ஒவ்வொரு முறையையும் தனித்தனியாக இயக்கலாம் அல்லது முடக்கலாம்:

| முறை | அது என்ன செய்கிறது |
|---|---|
| **துணைடொமைன்** | வாடிக்கையாளர் உங்கள் நெட்வொர்க்கில் இலவச துணைடொமைனைப் பயன்படுத்துகிறார் (எ.கா., `mysite.yournetwork.com`). கட்டணம் தேவையில்லை. |
| **புதிய டொமைனைப் பதிவு செய்யவும்** | வாடிக்கையாளர் புதிய டொமைனைத் தேடி, நீங்கள் அமைத்த வழங்குநர் மூலம் அதை பதிவு செய்கிறார். விலைக்காக பொருந்தும் டொமைன் தயாரிப்பைப் பயன்படுத்துகிறது. |
| **ஏற்கனவே உள்ள டொமைன்** | வாடிக்கையாளர் ஏற்கனவே வைத்திருக்கும் டொமைனை மேப் செய்கிறார். பதிவு கட்டணம் இல்லை. டொமைன் தானாகவே அவர்களின் தளத்துடன் மேப் செய்யப்படும். |

**இயல்புநிலை முறை** — மூன்று முறைகளும் இயக்கப்பட்டிருக்கும்போது, முதலில் திறக்கும் தாவல் எது. டொமைன் பதிவு விருப்பத் தேர்வாக இருக்க **துணைடொமைன்** என அமைக்கவும், அல்லது வாங்குதலை ஊக்குவிக்க **புதிய டொமைனைப் பதிவு செய்யவும்** என அமைக்கவும்.

**டொமைன் தயாரிப்பு** — விருப்பமாக இந்தப் புலத்தை குறிப்பிட்ட டொமைன் தயாரிப்புடன் இணைக்கவும். அமைக்கப்படவில்லை என்றால், வாடிக்கையாளர் தேடும் TLD அடிப்படையில் பொருந்தும் தயாரிப்பை addon தானாகத் தேர்ந்தெடுக்கும்.

### பதிவுதாரர் தொடர்பு புலங்கள் {#registrant-contact-fields}

வாடிக்கையாளர் **புதிய டொமைனைப் பதிவு செய்யவும்** தாவலைத் தேர்வுசெய்யும்போது, checkout படிவம் பதிவுதாரர் தொடர்பு புலங்களை அதே இடத்தில் சேர்க்கும்:

- முதல் பெயர் / கடைசி பெயர்
- மின்னஞ்சல் முகவரி
- முகவரி (வரி 1, நகரம், மாநிலம்/மாகாணம், அஞ்சல் குறியீடு, நாடு)
- தொலைபேசி எண்

இவை எல்லா பதிவு நிறுவனங்களாலும் தேவைப்படுகின்றன, மேலும் பதிவு API அழைப்பு செய்யப்படுவதற்கு முன் சரிபார்க்கப்படுகின்றன. தொலைபேசி எண்கள் பதிவு நிறுவனங்கள் எதிர்பார்க்கும் `+CC.NNN` சர்வதேச வடிவத்திற்கு தானாக வடிவமைக்கப்படுகின்றன.

### தானாக உருவாக்கப்படும் தள URL {#auto-generated-site-url}

வாடிக்கையாளர் ஒரு டொமைனைப் பதிவு செய்யும்போது அல்லது மேப் செய்யும்போது, தேர்ந்தெடுக்கப்பட்ட டொமைனிலிருந்து தள URL புலம் தானாக நிரப்பப்படும். வாடிக்கையாளர்கள் தனி URL புலத்தை நிரப்ப வேண்டியதில்லை.

### தேடல் நடத்தை {#search-behaviour}

- வாடிக்கையாளர் தட்டச்சு செய்யும்போதே AJAX மூலம் டொமைன் கிடைப்பது நேரடியாகச் சரிபார்க்கப்படும்
- விரும்பிய டொமைன் கிடைக்காதபோது மாற்று TLD பரிந்துரைகள் காட்டப்படும்
- விலை நேரடியாகப் பெறப்பட்டு தெளிவாகக் காட்டப்படும் (பதிவு விலை, புதுப்பிப்பு விலை, விருப்ப WHOIS தனியுரிமைக் கட்டணம்)
- கூப்பன் குறியீடுகள் மற்ற தயாரிப்புகளைப் போலவே டொமைன் தயாரிப்புகளுக்கும் பொருந்தும்

**தேடல் பதிலளிக்கும் வேகத்தைச் சரிசெய்தல்:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**டொமைன் தேடல் படிவத்தில் தனிப்பயன் புலங்களைச் சேர்த்தல்:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## வாடிக்கையாளர் DNS மேலாண்மை {#customer-dns-management}

வாடிக்கையாளர்கள் தங்கள் பதிவு செய்யப்பட்ட டொமைன்களுக்கான DNS பதிவுகளை **என் Account** பக்கத்தில், அவர்களின் டொமைன் உள்ளீட்டின் கீழ் நிர்வகிக்கலாம்.

### ஆதரிக்கப்படும் பதிவு வகைகள் {#supported-record-types}

| வகை | பயன்பாடு |
|---|---|
| **A** | ஹோஸ்ட்பெயரை IPv4 முகவரிக்கு மேப் செய்யவும் |
| **AAAA** | ஹோஸ்ட்பெயரை IPv6 முகவரிக்கு மேப் செய்யவும் |
| **CNAME** | மற்றொரு ஹோஸ்ட்பெயரைச் சுட்டும் மாற்றுப்பெயரை உருவாக்கவும் |
| **MX** | மெயில் பரிமாற்ற சேவையகத்தை அமைக்கவும் |
| **TXT** | SPF, DMARC, சரிபார்ப்பு அல்லது பிற உரை பதிவுகளைச் சேர்க்கவும் |

### எந்த வழங்குநர்கள் DNS மேலாண்மையை ஆதரிக்கிறார்கள்? {#which-providers-support-dns-management}

DNS மேலாண்மை (பதிவுகளைச் சேர்த்தல், திருத்தல், நீக்குதல்) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, மற்றும் **Openprovider** உடன் கிடைக்கிறது. **Hostinger** டொமைன்கள் Domain Seller மூலம் nameservers-ஐப் புதுப்பிக்கலாம்; ஹோஸ்ட் செய்யப்பட்ட டொமைன்களுக்கான DNS பதிவுகள் மைய Hostinger டொமைன்-மேப்பிங் ஒருங்கிணைப்பால் நிர்வகிக்கப்படுகின்றன. Namecheap, GoDaddy, மற்றும் NameSilo டொமைன்கள் நிலை மற்றும் காலாவதி தகவலைக் காட்டும், ஆனால் DNS பதிவு நிறுவனத்தின் கட்டுப்பாட்டு பலகையிலேயே நேரடியாக நிர்வகிக்கப்பட வேண்டும்.

### இயல்புநிலை DNS பதிவுகள் {#default-dns-records}

ஒரு டொமைன் பதிவு செய்யப்படும்போது தானாகப் பயன்படுத்தப்படும் இயல்புநிலை DNS பதிவுகளை நீங்கள் அமைக்கலாம். **அமைப்புகள் › Domain Seller › இயல்புநிலை DNS பதிவுகள்** என்பதற்கு செல்லவும்.

இயல்புநிலை பதிவு மதிப்புகள் இரண்டு டோக்கன்களை ஆதரிக்கின்றன:

| டோக்கன் | இதனால் மாற்றப்படும் |
|---|---|
| `{DOMAIN}` | பதிவு செய்யப்பட்ட டொமைன் பெயர் (எ.கா., `example.com`) |
| `{SITE_URL}` | வாடிக்கையாளரின் தளத்திற்கான WordPress தள URL |

**உதாரணம் — apex டொமைன் மற்றும் www-ஐ உங்கள் சேவையக IP-க்கு சுட்டவும்:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### நிர்வாகி: DNS-ஐப் பார்வையிடுதல் மற்றும் திருத்துதல் {#admin-viewing-and-editing-dns}

நெட்வொர்க் நிர்வாகிகள் எந்த வாடிக்கையாளர் டொமைனுக்கான DNS பதிவுகளையும் **நெட்வொர்க் நிர்வாகம் › Ultimate Multisite › டொமைன்கள்** இல் உள்ள டொமைனின் திருத்துப் பக்கத்திலிருந்து பார்வையிட்டு திருத்தலாம்.
