# செக்அவுட் ஃபீல்ட் மற்றும் வாடிக்கையாளர் DNS மேலாண்மை

## டொமைன் தேர்வு செக்அவுட் ஃபீல்ட்

**Domain Selection** ஃபீல்ட் என்பது ஒரு செக்அவுட் உறுப்பு (checkout element). இது வாடிக்கையாளர்களுக்கு தங்கள் தளத்திற்கான டொமைனை எப்படிப் பெற வேண்டும் என்ற தேர்வை வழங்குகிறது. டொமைன் விற்பனையை செயல்படுத்த இதை எந்தவொரு செக்அவுட் படிவத்திலும் சேர்க்கலாம்.

### செக்அவுட் படிவத்தில் ஃபீல்டைச் சேர்ப்பது

1. **Network Admin › Ultimate Multisite › Checkout Forms** என்பதற்குச் செல்லவும்
2. ஒரு செக்அவுட் படிவத்தை திறக்கவும் அல்லது உருவாக்கவும்
3. செக்அவுட் எடிட்டரில், **Add Field** என்பதைக் கிளிக் செய்யவும்
4. ஃபீல்ட் பட்டியலில் இருந்து **Domain Selection** என்பதைத் தேர்ந்தெடுக்கவும்
5. ஃபீல்ட் விருப்பங்களை உள்ளமைக்கவும் (கீழே பார்க்கவும்)
6. படிவத்தை சேமிக்கவும்

### ஃபீல்ட் விருப்பங்கள்

**Domain modes** — வாடிக்கையாளர் எந்தெந்த டேப்களைப் பார்ப்பார் என்பதை நீங்கள் தேர்வு செய்யலாம். ஒவ்வொரு முறையையும் தனித்தனியாக இயக்கவோ அல்லது முடக்கவோ முடியும்:

| Mode | என்ன செய்யும் |
|---|---|
| **Subdomain** | வாடிக்கையாளர் உங்கள் நெட்வொர்க்கில் உள்ள இலவச சப்-டொமைனைப் பயன்படுத்துவார் (உதாரணமாக, `mysite.yournetwork.com`). பணம் செலுத்தத் தேவையில்லை. |
| **Register New Domain** | வாடிக்கையாளர் ஒரு புதிய டொமைனைத் தேடி, உங்கள் கட்டமைக்கப்பட்ட வழங்குநரின் (provider) மூலம் அதைப் பதிவு செய்வார். விலையிடலுக்குப் பொருத்தமான டொமைன் தயாரிப்பைப் பயன்படுத்துகிறது. |
| **Existing Domain** | வாடிக்கையாளர் தனக்கு ஏற்கனவே சொந்தமான டொமைனை இணைப்பார். பதிவு கட்டணம் இல்லை. டொமைன் தானாகவே அவர்களின் தளத்துடன் இணைக்கப்படும். |

**Default mode** — மூன்று முறைகளும் இயக்கப்பட்டிருக்கும்போது, எந்த டேப் முதலில் திறக்கும். டொமைன் பதிவு விருப்பமாக இருக்க வேண்டுமென்றால் **Subdomain** என்று அமைக்கவும், அல்லது வாங்குதல்களை ஊக்குவிக்க **Register New Domain** என்று அமைக்கவும்.

**Domain product** — விருப்பப்பட்டால், இந்த ஃபீல்டை ஒரு குறிப்பிட்ட டொமைன் தயாரிப்புடன் இணைக்கலாம். இது அமைக்கப்படவில்லை என்றால், வாடிக்கையாளர் தேடும் TLD-இன் அடிப்படையில் addon தானாகவே பொருத்தமான தயாரிப்பைத் தேர்ந்தெடுக்கும்.

### பதிவாளர் தொடர்பு ஃபீல்ட்கள் (Registrant contact fields)

ஒரு வாடிக்கையாளர் **Register New Domain** டேப்பைத் தேர்ந்தெடுக்கும்போது, செக்அவுட் படிவம் தானாகவே பதிவாளர் தொடர்பு ஃபீல்ட்களை உள்ளே சேர்க்கும்:

- முதல் பெயர் / கடைசிப் பெயர்
- மின்னஞ்சல் முகவரி
- முகவரி (வரி 1, நகரம், மாநிலம்/மாகாணம், அஞ்சல் குறியீடு, நாடு)
- தொலைபேசி எண்

இவை அனைத்துப் பதிவாளர்களாலும் தேவைப்படுபவை மற்றும் பதிவு API அழைப்பு செய்வதற்கு முன் சரிபார்க்கப்படும். தொலைபேசி எண்கள் பதிவாளர்கள் எதிர்பார்க்கும் `+CC.NNN` என்ற சர்வதேச வடிவத்திற்கு தானாகவே வடிவமைக்கப்படும்.

### தானாக உருவாக்கப்பட்ட தள URL (Auto-generated site URL)

ஒரு வாடிக்கையாளர் டொமைனைப் பதிவு செய்தாலோ அல்லது இணைத்தாலோ, தேர்ந்தெடுக்கப்பட்ட டொமைனிலிருந்து தள URL ஃபீல்ட் தானாகவே நிரப்பப்படும். வாடிக்கையாளர்கள் தனி URL ஃபீல்டை நிரப்ப வேண்டியதில்லை.

### தேடல் நடத்தை (Search behaviour)

- வாடிக்கையாளர் டைப் செய்யும்போதே AJAX மூலம் டொமைன் கிடைக்கும் தன்மை நிகழ்நேரத்தில் சரிபார்க்கப்படும்
- விரும்பிய டொமைன் கிடைக்கவில்லை என்றால் மாற்று TLD பரிந்துரைகள் காட்டப்படும்
- விலை நிர்ணயம் நேரலையில் பெறப்பட்டு தெளிவாகக் காட்டப்படும் (பதிவு விலை, புதுப்பிப்பு விலை, விருப்பமான WHOIS தனியுரிமை கட்டணம்)
- கூப்பன் குறியீடுகள் எந்தவொரு தயாரிப்பைப் போலவே டொமைன் தயாரிப்புகளுக்கும் பொருந்தும்

**தேடல் பதிலளிப்பைச் சரிசெய்தல் (Tuning search responsiveness):**

```php
// API அழைப்புகளைக் குறைக்க, debounce தாமதத்தை (milliseconds) அதிகரிக்கவும்
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**டொமைன் தேடல் படிவத்தில் தனிப்பயன் ஃபீல்ட்களைச் சேர்ப்பது:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'கூடுதல் குறிப்புகள்',
    ];
    return $fields;
});
```

## வாடிக்கையாளர் DNS மேலாண்மை

வாடிக்கையாளர்கள் தங்கள் பதிவு செய்யப்பட்ட டொமைன்களுக்கான DNS பதிவுகளை **My Account** பக்கத்தில், தங்கள் டொமைனின் உள்ளீட்டின் கீழ் இருந்து நிர்வகிக்க முடியும்.

### ஆதரிக்கப்படும் பதிவு வகைகள் (Supported record types)

| Type | பயன்பாடு |
|---|---|
| **A** | ஹோஸ்ட்டேமை IPv4 முகவரிக்கு இணைத்தல் |
| **AAAA** | ஹோஸ்ட்டேமை IPv6 முகவரிக்கு இணைத்தல் |
| **CNAME** | மற்றொரு ஹோஸ்ட்டேமை சுட்டிக்காட்டும் ஒரு alias-ஐ உருவாக்குதல் |
| **MX** | மின்னஞ்சல் பரிமாற்ற சேவையகத்தை அமைத்தல் |
| **TXT** | SPF, DMARC, சரிபார்ப்பு அல்லது பிற உரை பதிவுகளைச் சேர்த்தல் |

### எந்த வழங்குநர்கள் DNS மேலாண்மையை ஆதரிக்கின்றன?

DNS மேலாண்மை (பதிவுகளைச் சேர்ப்பது, திருத்துவது, நீக்குவது) **OpenSRS**, **ResellerClub**, மற்றும் **Enom** ஆகியவற்றுடன் கிடைக்கிறது. Namecheap, GoDaddy, மற்றும் NameSilo டொமைன்கள் நிலை மற்றும் காலாவதித் தகவலைக் காண்பிしますが, DNS பதிவுகள் பதிவாளரின் கட்டுப்பாட்டுப் பலகத்தில் (control panel) நேரடியாக நிர்வகிக்கப்பட வேண்டும்.

### இயல்புநிலை DNS பதிவுகள் (Default DNS records)

ஒரு டொமைன் பதிவு செய்யப்படும்போது தானாகவே பயன்படுத்தப்படும் இயல்புநிலை DNS பதிவுகளை நீங்கள் கட்டமைக்கலாம். **Settings › Domain Seller › Default DNS Records** என்பதற்குச் செல்லவும்.

இயல்புநிலை பதிவு மதிப்புகள் இரண்டு டோக்கன்களை ஆதரிக்கின்றன:

| Token | எதனால் மாற்றப்படும் |
|---|---|
| `{DOMAIN}` | பதிவு செய்யப்பட்ட டொமைன் பெயர் (உதாரணமாக, `example.com`) |
| `{SITE_URL}` | வாடிக்கையாளரின் தளத்திற்கான WordPress தள URL |

**உதாரணம் — உச்ச டொமைன் மற்றும் www-ஐ உங்கள் சர்வர் IP-க்கு இணைத்தல்:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### நிர்வாகி: DNS பார்ப்பதும் திருத்துவதும்

Network admins, **Network Admin › Ultimate Multisite › Domains** இல் உள்ள டொமைனின் திருத்தும் பக்கத்திலிருந்து எந்தவொரு வாடிக்கையாளர் டொமைனுக்கான DNS பதிவுகளையும் பார்க்கவும் திருத்தவும் முடியும்.
