# வெளியீட்டு குறிப்புகள்

## Version 2.12.0 — 2026-05-15 அன்று வெளியிடப்பட்டது

- புதியது: Hostinger (hPanel)-ஐ டொமைன் மேப்பிங் ஒருங்கிணைப்புடன் ஆதரிக்கப்படும் ஹோஸ்ட் வழங்குநராகச் சேர்த்துள்ளோம்.
- புதியது: நெட்வொர்க் முழுவதும் தளத்தை மீட்டெடுப்பதை எளிதாக்க, Site Exporter இப்போது நெட்வொர்க் இறக்குமதி தொகுப்புகளைக் கையாளுகிறது.
- திருத்தம்: BCC பிராட்காஸ்ட் மின்னஞ்சல்கள் இப்போது பெறுநர்களின் முகவரிகள் வெளிப்படுவதைத் தடுக்க, ஒரு undisclosed-recipients தலைப்பைப் பயன்படுத்துகின்றன.
- திருத்தம்: ஒரு தேதி மதிப்பு இல்லாதபோது சேமிக்கும்போது Membership காலாவதி தேதி இனி கெட்டுப் போவதில்லை.
- திருத்தம்: Stripe Membership புதுப்பிப்புகள், deprecated deleteDiscount API-ஐ அழைக்காமல், தள்ளுபடிகளைச் சரியாக நீக்குகின்றன.
- திருத்தம்: டொமைன்-மேப் செய்யப்பட்ட தளங்களில் SSO திருப்பிவிசை (redirects) இப்போது முடிவற்ற திருப்பிவிசை சுழற்சிகளைத் தடுக்க கட்டுப்படுத்தப்பட்டுள்ளது.
- திருத்தம்: Setup wizard image picker தேர்வு இப்போது அடிப்படை தரவு மாதிரியைச் சரியாகப் புதுப்பிக்கிறது.
- திருத்தம்: Site Exporter CLI இப்போது சரியான இயல்புநிலை நெட்வொர்க் தளத் தேர்வைச் சேமிக்கிறது.
- மேம்படுத்தப்பட்டது: பிளக்-இன் தொகுப்பிலிருந்து bundled wp-cli நீக்கப்பட்டுள்ளது, இது பிளக்-இன் அளவைக் குறைக்கிறது.

## Version 2.11.0 — 2026-05-11 அன்று வெளியிடப்பட்டது

- புதியது: Site ஏற்றுமதிகள் இப்போது ஒரு சுய-பூட் செய்யும் `index.php`-ஐ தொகுக்கின்றன. இதனால், தனி プラഗിன் நிறுவல் இல்லாமல் புதிய ஹோஸ்டில் ZIP-ஐ நிறுவ முடியும்.
- புதியது: Network export மூலம் நிர்வாகியால் Site Export நிர்வாகப் பக்கத்திலிருந்து அனைத்து துணைத் தளங்களையும் ஒரே ஆர்கைவில் ஏற்றுமதி செய்ய முடியும்.
- புதியது: Site Templates plan toggle இப்போது ஒரு fallback chain மூலம் செயல்படுத்தப்படுகிறது, இது plan வரம்புகளுக்கு ஏற்ப template கிடைப்பதைச் சரியாகக் கட்டுப்படுத்துகிறது.
- புதியது: Checkout form editor-இல், ஒரு கட்டாயப் புலம் (required field) உள்ளமைக்கப்படாமல் ஒரு தயாரிப்பு சேர்க்கப்பட்டால் எச்சரிக்கை காட்டுகிறது.
- புதியது: Import/Export அமைப்புகள் tab இப்போது அதன் வரம்பை தெளிவாக விவரிக்கிறது மற்றும் Site Export கருவிக்கு நேரடியாக இணைக்கிறது.

- திருத்தம்: Site override உள்ளீடுகள் சேமிப்பதற்கு முன் சரிபார்க்கப்படுகின்றன (validated).

- திருத்தம்: Currents lazy-loaded before init hook மூலம் நேரப் பிரச்சினைகள் தடுக்கப்படுகின்றன.

- திருத்தம்: காலியான site identity விருப்பங்கள் சேமிக்கப்படும்போது பாதுகாக்கப்படுகின்றன.
