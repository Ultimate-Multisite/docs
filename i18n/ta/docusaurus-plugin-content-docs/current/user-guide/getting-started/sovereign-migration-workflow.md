# இறையாண்மை இடம்பெயர்வு பணிப்பாய்வு (Sovereign Migration Workflow) {#sovereign-migration-workflow}

Ultimate Multisite: Multi-Tenancy 1.2.0 ஆனது ஒரு நிலையான துணைதளத்தை இறையாண்மை டெண்டன்ட் கட்டமைப்பில் மாற்றுவதற்கான இடம்பெயர்வு சரிபார்ப்பு வாயில்களை (migration verification gates) சேர்க்கிறது.

## நீங்கள் தொடங்குவதற்கு முன் {#before-you-start}

உங்களிடம் பின்வருபவை இருப்பதை உறுதிப்படுத்திக் கொள்ளுங்கள்:

- மூல தளத்தின் தற்போதைய காப்புப்பிரதி (backup).
- Multi-Tenancy addon செயலில் உள்ளது.
- இலக்கு டெண்டன்ட் தரவுத்தளம், கோப்பு முறைமை வேர் (filesystem root), மற்றும் டொமைன் தயாராக உள்ளன.
- இலக்கு சூழலுக்குப் பொருந்தக்கூடிய தரவுத்தள ஹோஸ்ட் பிணைப்பு (database host binding).
- நெட்வொர்க்கிற்கு WP-CLI கட்டளைகளை இயக்க அணுகல்.

## பரிந்துரைக்கப்பட்ட பணிப்பாய்வு (Recommended workflow) {#recommended-workflow}

1. இலக்கு டெண்டன்ட் தரவுத்தளம் மற்றும் கோப்பு முறைமையை தயார் செய்யவும்.
2. டெண்டன்ட் தனிமைப்படுத்தல் அமைப்புகளைப் பதிவு செய்யவும் அல்லது புதுப்பிக்கவும்.
3. டெண்டன்ட் இடம்பெயர்ப்பை இயக்கவும்.
4. டெண்டன்ட் பயனர்களை வழங்கவும் அல்லது சரிபார்க்கவும்.
5. அசிங்க் (async) இடம்பெயர்வு வேலைகளை வெளியேற்றவும் (Drain).
6. இறையாண்மை இடம்பெயர்வு சரிபார்ப்பைச் செய்யவும்.
7. SSO உடன் டெண்டன்ட்டை அணுகவும்.
8. சரிபார்ப்பு வெற்றிகரமாக முடிந்த பின்னரே DNS அல்லது ரூட்டிங்கை மாற்றவும்.

## சரிபார்ப்பு வாயில்கள் (Verification gates) {#verification-gates}

சரிபார்ப்பு பணிப்பாய்வு பல கோணங்களில் இடம்பெயர்ப்பைச் சரிபார்க்கிறது:

- டெண்டன்ட் ஸ்கீமா (schema) இருக்க வேண்டும், மேலும் டெண்டன்ட் தரவுத்தள எழுத்தாளரால் (database writer) எழுதப்பட முடிந்திருக்க வேண்டும்.
- கட்டமைக்கப்பட்ட தரவுத்தள ஹோஸ்டுக்கு சரியான அனுமதிகள் (grants) இருக்க வேண்டும்.
- டெண்டன்ட் பயனர்கள் இருக்க வேண்டும் மற்றும் எதிர்பார்க்கப்படும் இறையாண்மை நிறுவப்பட்ட பயனர் எண்ணிக்கையுடன் பொருந்த வேண்டும்.
- அசிங்க் புஷ் வரிசையை (async push queue) வெற்றிகரமாக வெளியேற்ற முடியும்.
- இறையாண்மை டெண்டனுக்கு பழைய நெட்வொர்க் பக்க தரவுப் பாதைகள் இனி தேவையில்லை.

சரிபார்ப்பு தோல்விகளை வெளியீட்டிற்கு முன் தடுக்கக்கூடிய தடைகளாகக் கருதுங்கள். புகாரளிக்கப்பட்ட தரவுத்தளம், பயனர், வரிசை அல்லது ரூட்டிங் சிக்கலைச் சரிசெய்த பிறகு, டெண்டன்ட்டை வாடிக்கையாளர்களுக்கு வெளிப்படுத்துவதற்கு முன்பு மீண்டும் சரிபார்க்கவும்.

## முதல் தயாரிப்பு வருகை (First production visit) {#first-production-visit}

சரிபார்ப்பு வெற்றிகரமாக முடிந்த பிறகு, முதல் நிர்வாக வருகைக்கு **Visit (SSO)** ஐ தள மேலாண்மை திரையிலிருந்து பயன்படுத்தவும். இது ஒரு கட்டுப்படுத்தப்பட்ட படிநிலையில் டெண்டன்ட் ரூட்டிங், SSO டோக்கன் கையாளுதல், originate pinning மற்றும் டெண்டன்ட் பக்க பயனர் வழங்குதலை உறுதி செய்கிறது.
