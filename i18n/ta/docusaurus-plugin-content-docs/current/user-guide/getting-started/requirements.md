# Ultimate Multisite தேவைகள்

Ultimate Multisite வாங்குவதற்கு முன் நீங்கள் முதலில் கேட்கக்கூடிய கேள்வி என்னவென்றால், எங்கள் plugin-ஐ இயக்க என்ன தேவைகள் என்பதுதான். அதிர்ஷ்டவசமாக, Ultimate Multisite-ஐ இயக்க சில அடிப்படை தேவைகள் மட்டுமே உள்ளன: ஒரு **domain பெயர்**, ஒரு நல்ல **hosting வழங்குநர்** மற்றும் WordPress **Multisite** நிறுவல். இவ்வளவுதான் உங்களுக்குத் தேவை!

## Domain பெயர்

முதலில், domain பெயரைப் பற்றி பேசுவோம்.

Domain பெயரைத் தேர்ந்தெடுக்கும்போது, உங்கள் குறிப்பிட்ட niche என்ன, Ultimate Multisite மூலம் என்ன உருவாக்கப் போகிறீர்கள் என்பதை மனதில் கொள்ளுங்கள்: நீங்கள் ஒரு agency உருவாக்க விரும்புகிறீர்களா? shop போன்ற websites விற்க விரும்புகிறீர்களா? அல்லது WordPress web hosting? Ultimate Multisite மூலம் எல்லாமே சாத்தியம்.

எங்கள் plugin subscription சேவைகள், பணம் வசூலிப்பு, checkout படிவங்கள், தள்ளுபடி voucher-கள் மற்றும் வாடிக்கையாளர் தொடர்புகளை வழங்குகிறது. WordPress Multisite நிறுவ, கட்டமைக்க மற்றும் பராமரிக்க தேவையான பெரும்பாலான வேலைகளை Ultimate Multisite எளிதாக்குகிறது.

நீங்கள் websites-ஐ சேவையாக விற்கும் வரை, நீங்கள் தேர்ந்தெடுக்கும் எந்த niche-ஐயும் எங்கள் plugin மூலம் நிர்வகிக்கலாம்! உங்கள் வணிகத் திட்டத்துக்கு பொருந்தும் domain பெயரைத் தேர்ந்தெடுக்க மறக்காதீர்கள்.

## Hosting வழங்குநர்

இப்போது உங்கள் niche மற்றும் வணிகத்தை பிரதிபலிக்கும் கவர்ச்சியான domain பெயர் உங்களிடம் உள்ளது, Ultimate Multisite உடன் உங்கள் network-ஐ host செய்ய ஒரு நல்ல web hosting நிறுவனத்தைத் தேர்ந்தெடுக்க வேண்டும். சந்தையில் சில நல்ல விருப்பங்கள் உள்ளன. சில hosting வழங்குநர்கள் Ultimate Multisite உடன் நன்றாக ஒருங்கிணைக்கின்றன, நீங்கள் என்ன உருவாக்க திட்டமிடுகிறீர்கள் என்பதைப் பொறுத்து இது ஒரு முக்கிய காரணியாக இருக்கலாம்.

Ultimate Multisite எந்த hosting வழங்குநரிடமும் வேலை செய்யும். ஒரே வேறுபாடு என்னவென்றால், _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ மற்றும் _Runcloud_ போன்ற சில hosting நிறுவனங்கள் domain mapping-க்கு நெருக்கமான ஒருங்கிணைப்பைக் கொண்டுள்ளன. மற்ற hosting-க்கு, custom domain-களை கைமுறையாக map செய்ய வேண்டியிருக்கலாம். AutoSSL ஆதரிக்கும் hosting வழங்குநரைப் பெறுவதும் நல்ல யோசனை. _Closte_, _Runcloud_ மற்றும் _cPanel_ இதை ஆதரிக்கின்றன.

Server விவரக்குறிப்புகளைப் பொறுத்தவரை, நீங்கள் வழங்கும் அம்சங்கள் மற்றும் செயல்பாடுகள் மற்றும் உங்கள் network-இன் அளவைப் பொறுத்து இது மாறுபடும் என்பதால் குறிப்பிட்ட சூத்திரம் எதுவும் இல்லை. Shared hosting தொடங்குவதற்கு நல்லதாக இருக்கலாம், ஆனால் உங்கள் network வளர ஆரம்பித்தவுடன், உங்களுக்கு ஒரு dedicated VPS தேவைப்படலாம்.

## WordPress Multisite

இப்போது, WordPress Multisite நிறுவலைப் பற்றி சிறிது பேசுவோம்.

WordPress Multisite ஒரே நிறுவலில் sites-இன் network வைத்திருக்க உங்களை அனுமதிக்கிறது. இது ஒரு உள்ளமைந்த அம்சம், ஆனால் இது இயல்பாக செயலில் இல்லை. உங்கள் WordPress நிறுவல் மற்றும் web hosting control panel-இல் இந்த அம்சத்தை நீங்கள் இயக்க வேண்டும்.

Multisite இயக்கப்பட்ட WordPress **Ultimate Multisite-க்கான அடிப்படை ஆகும்**. Ultimate Multisite-இன் முக்கிய பயன்பாடு பயனர்களுக்கு websites-ஐ சேவையாக நிர்வகிக்கவும் விற்கவும் உதவுவதால், இது Multisite இயக்கப்பட்ட WordPress-இல் செய்யப்பட வேண்டும்.

உங்கள் WordPress நிறுவலில் Multisite அம்சத்தை இயக்க என்ன செய்ய வேண்டும் என்பதை படிப்படியாகக் கற்றுக்கொடுக்கும் **விரிவான கட்டுரை** எங்களிடம் உள்ளது.

WordPress-இல் Multisite இயக்கப்பட்ட பிறகு, Ultimate Multisite நிறுவ வேண்டிய நேரம். Ultimate Multisite-ஐ சரியாக நிறுவ, உங்களிடம் இருக்க வேண்டியவை:

  * WordPress: v5.3+ (பரிந்துரை: கடைசி நிலையான பதிப்பு)

  * PHP: 7.4.x (> 8.0 ஆதரவு விரைவில் வருகிறது)

  * MySQL: v5+ (பரிந்துரை: 5.6, உங்கள் hosting வழங்குநரிடம் 8.0 கிடைக்கவில்லை என்றால்)

Ultimate Multisite-க்கான ஒரே software தேவைகள் இவைதான்.

Ultimate Multisite PHP 8.0 உடன் வேலை செய்யலாம், ஆனால் PHP 7.4.x உடன் இயக்க பரிந்துரைக்கிறோம் என்பதை நினைவில் கொள்ளுங்கள்.

மேலும், **முக்கிய domain-களில்** நிறுவ பரிந்துரைக்கிறோம், subdomain-கள் அல்லது subdirectory-களில் அல்ல. Subdomain-கள்/subdirectory-களில் வேலை செய்யலாம், ஆனால் சில பிழைகள் ஏற்படலாம்.

Ultimate Multisite-ஐ எவ்வாறு நிறுவுவது என்பதற்கான விரிவான வழிகாட்டிக்கு, **இங்கே கிளிக் செய்யுங்கள்**.
