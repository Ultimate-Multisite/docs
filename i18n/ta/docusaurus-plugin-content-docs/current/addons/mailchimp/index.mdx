# Mailchimp {#mailchimp}

Multisite Ultimate: Mailchimp ஒருங்கிணைப்பு (Integration)

உங்கள் Multisite Ultimate நெட்வொர்க்கை உங்கள் Mailchimp உடன் இணைக்கலாம். சில திட்டங்களை (plans) Mailchimp Lists உடன் இணைத்து, பயனர் பதிவு செய்யும்போதே தானாகவே அந்தப் பட்டியல்களில் (lists) சந்தாதாரர்களைச் சேர்க்க முடியும்.

## நிறுவுதல் (Installation) {#installation}

1. 'ultimate-multisite-mailchimp' என்பதை '/wp-content/plugins/' கோப்பகத்தில் (directory) பதிவேற்றவும்.
2. உங்கள் WordPress Network Admin-ல் உள்ள 'Plugins' மெனு மூலம் இந்த plugin-ஐ Activate செய்யவும்.

## பயன்பாட்டு வழிகாட்டி (Usage Guide) {#usage-guide}

எங்கள் Ultimate Multisite: MailChimp Integration Add-on மூலம் உங்கள் சந்தாதாரர்களின் மின்னஞ்சல்களை உங்கள் MailChimp பட்டியல்களுடன் (lists) ஒத்திசைப்பது (sync) மிகவும் எளிமையானது. எல்லாவற்றையும் எப்படி அமைப்பது என்று இந்த வழிகாட்டி உங்களுக்கு உதவும்!

## API key பெறுவது எப்படி {#getting-an-api-key}

வெவ்வேறு அமைப்புகள் (settings) பற்றிப் பேசத் தொடங்குவதற்கு முன், நமக்கு ஒரு MailChimp API key தேவை. பயனர்கள் பதிவு செய்யும்போதோ அல்லது வெளியேறும்போதோ, பட்டியல்கள், குழுக்கள் (groups) மற்றும் மின்னஞ்சல் முகவரிகளைச் சேகரிக்க MailChimp API உடன் பேச Ultimate Multisite-க்கு இது உதவும்.

உங்கள் சொந்த MailChimp API key-ஐப் பெற இந்த வழிகாட்டியைப் பின்பற்றவும்: [About API Keys | MailChimp](https://kb.mailchimp.com/integrations/api-integrations/about-api-keys)

_உங்கள் MailChimp API key-ஐ_ _**Ultimate Multisite Settings - > Add-on Settings -> MailChimp**_ _ல் சேர்க்கவும்_

## ஒருங்கிணைப்பு முறை (Integration Mode) {#integration-mode}

MailChimp Integration add-on இரண்டு "ஒருங்கிணைப்பு முறைகளை" (integration modes) வழங்குகிறது. அதாவது, மின்னஞ்சல் முகவரிகளை ஒத்திசைக்க நீங்கள் இரண்டு வெவ்வேறு வழிகளைப் பயன்படுத்தலாம்.

<!-- Screenshot unavailable: MailChimp integration mode selection with Multiple Lists and Single List options -->

_அதிக நெகிழ்வுத்தன்மைக்கு இரண்டு ஒருங்கிணைப்பு முறைகள்!_

### முறை 1: பல பட்டியல்கள் (Multiple Lists) {#mode-1-multiple-lists}

இயல்புநிலை முறை **பல பட்டியல்களைப்** பயன்படுத்துகிறது. இந்த முறை, ஒவ்வொரு திட்டத்திற்கும் (plan) பல MailChimp பட்டியல்களைத் தேர்ந்தெடுக்க உங்களை அனுமதிக்கும். உங்கள் பயனர்களை வெவ்வேறு பட்டியல்களாகப் பிரித்து (segment) வைப்பவர்களுக்கு இதுவே சிறந்த தேர்வாகும்.

உதாரணமாக, உங்கள் MailChimp கணக்கில் **Newsletter**, **Plan A**, மற்றும் **Plan B** என 3 வெவ்வேறு பட்டியல்கள் இருப்பதாக வைத்துக்கொள்வோம். உங்கள் திட்டம் எதுவாக இருந்தாலும், அனைத்துப் பயனர்களும் Newsletter பட்டியலில் சேர்க்கப்பட வேண்டும், அப்போதுதான் உங்கள் புதுப்பித்தல்களை அவர்கள் தவறவிட மாட்டார்கள்.

இதை அடைய, Plan A MailChimp Settings-ல் **Newsletter** மற்றும் **Plan A** ஆகிய இரண்டு பட்டியல்களையும் சேர்க்கவும், மற்றும் Plan B MailChimp Settings-ல் **Newsletter** மற்றும் **Plan B** ஆகிய இரண்டு பட்டியல்களையும் சேர்க்கவும், Multiple Lists முறையைப் பயன்படுத்தலாம்.

<!-- Screenshot unavailable: MailChimp default lists selection for new accounts and canceled accounts -->

_புதிய கணக்குகளுக்கும், ரத்து செய்யப்பட்ட கணக்குகளுக்கும் இயல்புநிலை பட்டியல்களைத் தேர்ந்தெடுக்கவும்_

### முறை 2: ஒற்றைப் பட்டியல் + பல குழுக்கள் (Single List + Multiple Groups) {#mode-2-single-list--multiple-groups}

MailChimp உங்கள் பட்டியலை **குழுக்கள்** எனப்படும் சிறிய பயனர் குழுக்களாகப் பிரிக்க அனுமதிக்கிறது. இது உங்களுக்கு அதிக கட்டுப்பாட்டைக் கொடுக்கிறது. ஏனெனில், நீங்கள் ஒரே பிரச்சாரத்தை (campaign) MailChimp-ல் பல பிரதிகள் உருவாக்க வேண்டியதில்லை (இது அந்தப் பிரச்சாரத்தின் Metrics-களைப் பகுப்பாய்வு செய்வதை கடினமாக்கும்).

உதாரணமாக, **My Network Subscribers** என்ற ஒற்றைப் பட்டியல் இருக்கலாம், மேலும் அதற்குள் **Plan A** மற்றும் **Plan B** க்கான குழுக்கள் இருக்கலாம்.

<!-- Screenshot unavailable: MailChimp single list and groups selection for default lists -->

_“ஒற்றைப் பட்டியல் + பல குழுக்கள்” ஒருங்கிணைப்பு முறையைப் பயன்படுத்தினால், இயல்புநிலை பட்டியல்கள் மற்றும் குழுக்களைத் தேர்ந்தெடுக்கவும்_

## இயல்புநிலை பட்டியல்கள் மற்றும் குழுக்கள் (Default Lists and Groups) {#default-lists-and-groups}

நீங்கள் எந்த ஒருங்கிணைப்பு முறை உங்கள் குறிப்பிட்ட பயன்பாட்டிற்கு மிகவும் பொருத்தமானது என்பதைத் தேர்ந்தெடுத்த பிறகு, இயல்புநிலை பட்டியல்களை (மற்றும் முறையைப் பொறுத்து குழுக்களையும்) தேர்ந்தெடுக்க வேண்டிய நேரம் இது. ஒரு குறிப்பிட்ட திட்டம்க்கு (plan) தனிப்பட்ட பட்டியல் தேர்ந்தெடுக்கப்படவில்லை என்றால், இயல்புநிலை பட்டியல் பயன்படுத்தப்படும்.

மேலும், ரத்து செய்யப்பட்ட கணக்குகளுக்கான இயல்புநிலை பட்டியலையும் நீங்கள் தேர்ந்தெடுக்கலாம். ஒரு பயனர் தனது சந்தாவை முடித்துவிட்டு, தனது கணக்கை உங்கள் நெட்வொர்க்கிலிருந்து நீக்கும்போதெல்லாம், அவருடைய மின்னஞ்சல் முகவரி ரத்து செய்யப்பட்ட கணக்குகளுக்கான பட்டியலில் மாற்றப்படும்.

## இரட்டை சந்தா (Double Subscription) இயக்குதல் {#enabling-double-subscription}

அதே அமைப்புகள் பக்கத்தில், நீங்கள் **Double Subscription** என்பதை இயக்க முடியும். இந்த விருப்பம், பயனரை நீங்கள் தேர்ந்தெடுத்த இயல்புநிலை பட்டியலிலும் (default list) AND அந்தப் பயனரின் திட்டத்திற்கான (plan) பட்டியலில் சேர்க்கும். இந்த விருப்பம் இயக்கப்படவில்லை என்றால், பயனர் அந்தப் பயனரின் திட்டத்திற்காகத் தேர்ந்தெடுக்கப்பட்ட பட்டியலில் மட்டுமே சேர்க்கப்படுவார். அந்த குறிப்பிட்ட திட்டத்திற்கு எதுவும் அமைக்கப்படவில்லை என்றால், பயனர் தேர்ந்தெடுக்கப்பட்ட இயல்புநிலை பட்டியலில் சேர்க்கப்படுவார்.

<!-- Screenshot unavailable: MailChimp double subscription toggle setting -->

_பதிவு செய்யும்போதே உங்கள் பயனர்கள் இயல்புநிலை பட்டியலிலும், திட்டத்தின் பட்டியலிலும் சந்தாதாரர்களாக இருக்க முடியும்_

## ஒவ்வொரு திட்டத்திற்கான பட்டியல்கள் மற்றும் குழுக்களைத் தேர்ந்தெடுத்தல் (Selecting Lists and Groups for each of the Plans) {#selecting-lists-and-groups-for-each-of-the-plans}

உங்கள் **Plans** பகுதிக்குச் சென்று, நீங்கள் எடிட் செய்ய விரும்பும் திட்டத்தைத் (plan) தேர்ந்தெடுக்கவும். **Advanced Options** meta-box-ல், MailChimp என்ற புதிய Tab கிடைக்கும் என்பதைக் கவனியுங்கள்.

அந்த Tab-ல், அந்தத் திட்டத்திற்கான பல பட்டியல்கள் அல்லது ஒற்றைப் பட்டியலையும் பல குழுக்களையும் (ஒருங்கிணைப்பு முறையைப் பொறுத்து) நீங்கள் தேர்ந்தெடுக்க முடியும்.

<!-- Screenshot unavailable: MailChimp tab on plan edit page showing lists and groups selection -->

_உங்கள் ஒவ்வொரு திட்டத்திற்கும் பயன்படுத்த விரும்பும் பட்டியல்கள் மற்றும் குழுக்களைத் தேர்ந்தெடுக்கவும்_

ஒரு புதிய பயனர் அந்தத் திட்டத்திற்காகப் பதிவு செய்யும்போதெல்லாம், அவருடைய மின்னஞ்சல் முகவரி தேர்ந்தெடுக்கப்பட்ட பட்டியல்கள் மற்றும் குழுக்களில் சேர்க்கப்படும்!
