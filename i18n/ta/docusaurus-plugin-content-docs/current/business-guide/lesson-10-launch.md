# பாடம் 10: துவக்க நாள் (Launch Day)

எல்லாவற்றையும் கட்டமைத்துவிட்டீர்கள். கதவுகளைத் திறப்பதற்கு முன், எதுவும் உடைந்ததா, காணாமல் போனதா அல்லது சங்கடமானதா என்பதை உறுதிப்படுத்த, இந்த பாடம் துவங்குவதற்கு முன் செய்ய வேண்டிய சரிபார்ப்புப் பட்டியலை (pre-launch checklist) உங்களுக்கு வழங்குகிறது.

## எங்கு நிறுத்தினோம் {#where-we-left-off}

FitSite-ல் டெம்ப்ளேட்டுகள், திட்டங்கள் (plans), பணம் செலுத்தும் முறை (checkout), பிராண்டிங், புதிய பயனர்களை அறிமுகப்படுத்தும் முறை (onboarding), மற்றும் விலை நிர்ணயம் ஆகியவை அனைத்தும் கட்டமைக்கப்பட்டுவிட்டன. இப்போது எல்லாமே சரியாக வேலை செய்கிறதா என்று சரிபார்த்து, தளத்தை பொதுமக்களுக்கு வெளியிடுவோம்.

## துவங்குவதற்கு முன் செய்ய வேண்டிய சரிபார்ப்புப் பட்டியல் {#pre-launch-checklist}

ஒவ்வொரு அம்சத்தையும் கடந்து செல்லுங்கள். எதையும் தவிர்க்காதீர்கள்.

### தளத்தின் அடிப்படை அமைப்பு (Platform Infrastructure) {#platform-infrastructure}

- [ ] ஹோஸ்டிங் (Hosting) நிலையாகவும், அதிக சுமையின் கீழ் சிறப்பாக செயல்படுவதாகவும் இருக்க வேண்டும்
- [ ] Wildcard SSL செயல்பட வேண்டும், மேலும் அனைத்து subdomain-களும் HTTPS வழியாகவே இயங்க வேண்டும்
- [ ] Domain mapping வேலை செய்ய வேண்டும் -- ஒரு தளத்தை உருவாக்கி, ஒரு தனிப்பயன் டொமைனை (custom domain) வரைபடமாக்குவது (mapping) சோதிக்கவும்
- [ ] காப்புப்பிரதிகள் (Backups) கட்டமைக்கப்பட்டு சோதிக்கப்பட்டிருக்க வேண்டும் (குறைந்தது ஒன்றையாவது மீட்டெடுத்து (restore) சரிபார்க்கவும்)
- [ ] கண்காணிப்பு (Monitoring) அமைக்கப்பட்டிருக்க வேண்டும் -- தளம் செயலிழக்கும் பட்சத்தில் உங்களுக்குத் தெரிய வேண்டும்

### டெம்ப்ளேட்டுகள் (Templates) {#templates}

- [ ] மூன்று டெம்ப்ளேட்டுகளும் புதிய தளங்களில் சரியாக ஏற்றப்பட வேண்டும்
- [ ] மாதிரி உள்ளடக்கம் (Placeholder content) உதவியாகவும், எழுத்துப் பிழைகள் இல்லாமலும் இருக்க வேண்டும்
- [ ] அனைத்து படங்களும் சரியாக உரிமம் பெற்றிருக்க வேண்டும் (ஸ்டாக் போட்டோ வாட்டர்மார்க்குகள் இருக்கக்கூடாது)
- [ ] ஒவ்வொரு டெம்ப்ளேட் பக்கத்திலும் மொபைல் சாதனங்களில் இயங்குதிறன் (Mobile responsiveness) வேலை செய்ய வேண்டும்
- [ ] பக்கத்தின் ஏற்ற வேகம் ஏற்றுக்கொள்ளக்கூடியதாக இருக்க வேண்டும் (GTmetrix அல்லது PageSpeed Insights போன்ற கருவியைப் பயன்படுத்திச் சோதிக்கவும்)
- [ ] எந்த டெம்ப்ளேட்டிலும் உடைந்த இணைப்புகள் (broken links) அல்லது காணாமல் போன சொத்துக்கள் (missing assets) இருக்கக்கூடாது

### திட்டங்கள் மற்றும் தயாரிப்புகள் (Plans and Products) {#plans-and-products}

- [ ] மூன்று திட்டங்களும் செயலில் மற்றும் கண்ணுக்குத் தெரிய வேண்டும்
- [ ] திட்ட விளக்கங்கள் துல்லியமாகவும், குறிப்பிட்ட துறை சார்ந்ததாகவும் இருக்க வேண்டும்
- [ ] விலை நிர்ணயம் சரியாக இருக்க வேண்டும் (மாதாந்திரம் மற்றும் ஆண்டுக்கு)
- [ ] அமைப்பு கட்டணங்கள் (Setup fees) சரியான திட்டங்களில் கட்டமைக்கப்பட வேண்டும்
- [ ] ஸ்டார்ட்டர் திட்டத்தில் (Starter plan) சோதனை காலம் (Trial period) வேலை செய்ய வேண்டும்
- [ ] பணம் செலுத்தும் போது (checkout) ஆர்டர் பம்ப்ஸ் (Order bumps) சரியாகத் தோன்ற வேண்டும்
- [ ] ஒவ்வொரு திட்டத்திற்கும் プラгіன் மற்றும் தீம் வரம்புகள் (Plugin and theme limitations) சரியாக அமல்படுத்தப்பட்டிருக்க வேண்டும்

### பணம் செலுத்தும் செயல்முறை (Checkout Flow) {#checkout-flow}

- [ ] ஒவ்வொரு திட்டத்திலும் முழுமையான சோதனைப் பதிவு (test signup) செய்யுங்கள் (சோதனை கட்டண முறையைப் பயன்படுத்தவும்)
- [ ] டெம்ப்ளேட் தேர்வு, ஒவ்வொரு திட்டத்திற்கும் சரியான டெம்ப்ளேட்டுகளைக் காட்ட வேண்டும்
- [ ] பணம் செலுத்தும் செயல்முறை வெற்றிகரமாக நடக்க வேண்டும்
- [ ] பதிவு செய்த பிறகு வாடிக்கையாளருக்கு வரவேற்பு மின்னஞ்சல் (welcome email) கிடைக்க வேண்டும்
- [ ] சரியான டெம்ப்ளேட்டுடன் புதிய தளம் உருவாக்கப்பட வேண்டும்
- [ ] வாடிக்கையாளர் உடனடியாக தங்கள் புதிய தளத்தில் உள்நுழைய (log in) இருக்க வேண்டும்
- [ ] தள்ளுபடி குறியீடுகள் (Discount codes) சரியாக வேலை செய்ய வேண்டும்

### பிராண்டிங் (Branding) {#branding}

- [ ] உள்நுழைவுப் பக்கத்தில் FitSite பிராண்டிங் காட்ட வேண்டும்
- [ ] Admin dashboard-ல் FitSite பிராண்டிங் காட்ட வேண்டும்
- [ ] அனைத்து சிஸ்டம் மின்னஞ்சல்களும் FitSite பிராண்டிங் மற்றும் ஃபிட்னஸ் சார்ந்த மொழியைப் பயன்படுத்த வேண்டும்
- [ ] விலைப்பட்டியல்கள் (Invoices) உங்கள் வணிக விவரங்களுடன் சரியாகக் காட்ட வேண்டும்
- [ ] மார்க்கெட்டிங் தளம் (Marketing site) இயங்கிக் கொண்டிருக்க வேண்டும் மற்றும் பணம் செலுத்தும் படிவத்துடன் (checkout form) இணைக்கப்பட்டிருக்க வேண்டும்

### புதிய பயனர்களை அறிமுகப்படுத்துதல் (Onboarding) {#onboarding}

- [ ] Quick Start widget புதிய வாடிக்கையாளர் dashboard-ல் தோன்ற வேண்டும்
- [ ] அனைத்து Quick Start இணைப்புகளும் சரியான பக்கங்களை நோக்கிச் சுட்டிக்காட்ட வேண்டும்
- [ ] வரவேற்பு மின்னஞ்சல் தொடர் (Welcome email sequence) கட்டமைக்கப்பட்டு சோதிக்கப்பட்டிருக்க வேண்டும்
- [ ] அறிவுத் தளம் (Knowledge base) கட்டுரைகள் வெளியிடப்பட்டு அணுகக்கூடியதாக இருக்க வேண்டும்
- [ ] கணக்குப் பக்கம் சரியான திட்டத் தகவல்களையும் மேம்படுத்தும் விருப்பங்களையும் (upgrade options) காட்ட வேண்டும்

### சட்ட மற்றும் வணிகம் (Legal and Business) {#legal-and-business}

- [ ] சேவை விதிமுறைகள் (Terms of service) வெளியிடப்பட்டு, checkout இலிருந்து இணைக்கப்பட்டிருக்க வேண்டும்
- [ ] தனியுரிமைக் கொள்கை (Privacy policy) வெளியிடப்பட்டு அணுகக்கூடியதாக இருக்க வேண்டும்
- [ ] பணத்தைத் திரும்பப் பெறும் கொள்கை (Refund policy) வரையறுக்கப்பட்டு ஆவணப்படுத்தப்பட்டிருக்க வேண்டும்
- [ ] பணம் பெறுவதற்காக வணிக நிறுவனம் (Business entity) அமைக்கப்பட வேண்டும்
- [ ] பணம் செலுத்தும் நுழைவாயில் (Payment gateway) நேரடி முறையில் (live mode) இருக்க வேண்டும் (சோதனை முறையில் அல்ல)
- [ ] உங்கள் அதிகார வரம்பிற்கு (jurisdiction) ஏற்ற வரி அமைப்பு (Tax configuration) சரியாக இருக்க வேண்டும்

## மென்மையான துவக்கம் vs. கடினமான துவக்கம் (Soft Launch vs. Hard Launch) {#soft-launch-vs-hard-launch}

இரண்டு கட்ட துவக்கத்தைக் கருத்தில் கொள்ளுங்கள்:

### கட்டம் 1: மென்மையான துவக்கம் (Soft Launch) {#phase-1-soft-launch}

பொது வெளியீட்டிற்கு முன் 5-10 ஃபிட்னஸ் ஸ்டுடியோ உரிமையாளர்களைப் பதிவு செய்ய அழைக்கவும். இவர்களே உங்கள் பீட்டா வாடிக்கையாளர்கள். அவர்களுக்குப் பதிலாக, பின்வருவனவற்றிற்காக அவர்களிடம் இருந்து ஒரு குறிப்பிடத்தக்க தள்ளுபடியை (வாழ்நாள் முழுவதும் 50% தள்ளுபடி, அல்லது 3 மாதங்கள் இலவசம்) வழங்குங்கள்:

- பதிவு மற்றும் புதிய பயனர்களை அறிமுகப்படுத்தும் அனுபவத்தின் மீது நேர்மையான கருத்து.
- அவர்களின் தளத்தை ஒரு காட்சி எடுத்துக்காட்டாகப் பயன்படுத்த அனுமதி.
- அவர்கள் சந்திக்கும் ஏதேனும் பிழைகள் அல்லது சிக்கல்களைப் புகாரளிப்பது.

இது பொதுமக்களுக்குத் திறப்பதற்கு முன் உங்களுக்கு உண்மையான வாடிக்கையாளர் கருத்துக்களையும், காட்சிப்படுத்த சில நேரடி தளங்களையும் அளிக்கிறது.

### கட்டம் 2: பொது வெளியீடு (Public Launch) {#phase-2-public-launch}

மென்மையான துவக்கத்தின் கருத்துக்கள் இணைக்கப்பட்ட பிறகு:

- பணம் செலுத்தும் நுழைவாயிலை நேரடி முறைக்கு (live mode) மாற்றவும்
- உங்கள் மார்க்கெட்டிங் தளத்தை வெளியிடவும்
- வாடிக்கையாளர் கையகப்படுத்துதலைத் தொடங்கவும் (பாடம் 11)
- தொடர்புடைய ஃபிட்னஸ் தொழில்துறை சேனல்களில் அறிவிக்கவும்

## துவக்க நாள் நடவடிக்கைகள் (Launch Day Actions) {#launch-day-actions}

நீங்கள் பொதுமக்களுக்கு வெளியிடும் நாளில்:

1. **நேரடி பணம் செலுத்துதலுக்கு மாறுங்கள்** -- உங்கள் பணம் செலுத்தும் நுழைவாயிலில் சோதனை முறையை (test mode) முடக்கவும்
2. **மீண்டும் சரிபார்க்கவும்** -- உண்மையான பணம் செலுத்துதலுடன் முழுமையான சோதனைப் பதிவைச் செய்யுங்கள் (பின்னர் உங்களுக்கு நீங்களே பணத்தைத் திரும்பப் பெறுங்கள்)
3. **கவனமாக கண்காணிக்கவும்** -- பிழைகள், தோல்வியுற்ற பதிவுகள் அல்லது பணம் செலுத்தும் சிக்கல்களைக் கவனிக்கவும்
4. **கிடைையாய் இருக்கவும்** -- உங்கள் முதல் உண்மையான வாடிக்கையாளர்கள் உதவி தேவைப்படலாம், விரைவான பதில் நம்பிக்கையை உருவாக்கும்
5. **சிறு கொண்டாட்டம்** -- பிறகு மீண்டும் வேலைக்குத் திரும்புங்கள்

## என்ன தவறாகப் போகலாம் (What Can Go Wrong) {#what-can-go-wrong}

இதற்கு நீங்கள் தயாராக இருக்க வேண்டும்:

- **பணம் செலுத்தும் நுழைவாயில் சிக்கல்கள்**: உங்கள் நுழைவாயில் வழங்குநரின் ஆதரவு தொடர்பை தயாராக வைத்திருங்கள்
- **SSL சான்றிதழ் சிக்கல்கள்**: சான்றிதழ்களைச் சரிபார்த்து புதுப்பிப்பது எப்படி என்று தெரிந்து கொள்ளுங்கள்
- **மின்னஞ்சல் விநியோக தோல்விகள்**: மின்னஞ்சல்கள் உண்மையில் வருகிறதா என்று சோதிக்கவும் (ஸ்பேம் கோப்புறைகளைச் சரிபார்க்கவும்)
- **சுமையின் கீழ் செயல்திறன்**: உங்களுக்கு அதிக போக்குவரத்து ஏற்பட்டால், உங்கள் ஹோஸ்டிங்கை எப்படி விரிவாக்குவது என்று தெரிந்து கொள்ளுங்கள்
- **வாடிக்கையாளர் குழப்பம்**: உங்கள் அறிவுத் தளம் மற்றும் ஆதரவு சேனல்களைத் தயாராக வைத்திருங்கள்
