# cPanel ਇੰਟੀਗ੍ਰੇਸ਼ਨ

## ਸੰਖੇਪ ਜਾਣਕਾਰੀ
cPanel ਸਭ ਤੋਂ ਮਸ਼ਹੂਰ ਵੈੱਬ ਹੋਸਟਿੰਗ ਕੰਟਰੋਲ ਪੈਨਲਾਂ ਵਿੱਚੋਂ ਇੱਕ ਹੈ ਜੋ ਬਹੁਤ ਸਾਰੇ ਸ਼ੇਅਰਡ ਅਤੇ ਡੈਡੀਕੇਟਿਡ ਹੋਸਟਿੰਗ ਪ੍ਰੋਵਾਈਡਰ ਵਰਤਦੇ ਹਨ। ਇਹ ਇੰਟੀਗ੍ਰੇਸ਼ਨ Ultimate Multisite ਅਤੇ cPanel ਵਿਚਕਾਰ ਆਟੋਮੈਟਿਕ ਡੋਮੇਨ ਸਿੰਕਿੰਗ ਨੂੰ ਸਮਰੱਥ ਬਣਾਉਂਦੀ ਹੈ, ਜਿਸ ਨਾਲ ਤੁਸੀਂ ਆਪਣੇ cPanel ਖਾਤੇ ਵਿੱਚ ਡੋਮੇਨ ਅਲਾਇਸ ਅਤੇ ਸਬਡੋਮੇਨ ਆਪਣੇ ਆਪ ਜੋੜ ਸਕਦੇ ਹੋ।

## ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ
- cPanel ਵਿੱਚ ਆਟੋਮੈਟਿਕ addon ਡੋਮੇਨ ਬਣਾਉਣਾ
- cPanel ਵਿੱਚ ਆਟੋਮੈਟਿਕ ਸਬਡੋਮੇਨ ਬਣਾਉਣਾ (ਸਬਡੋਮੇਨ ਮਲਟੀਸਾਈਟ ਇੰਸਟਾਲੇਸ਼ਨਾਂ ਲਈ)
- ਮੈਪਿੰਗ ਮਿਟਾਉਣ 'ਤੇ ਡੋਮੇਨ ਹਟਾਉਣਾ

## ਲੋੜਾਂ
ਹੇਠ ਲਿਖੇ constants ਤੁਹਾਡੀ `wp-config.php` ਫਾਈਲ ਵਿੱਚ ਪਰਿਭਾਸ਼ਿਤ ਹੋਣੇ ਚਾਹੀਦੇ ਹਨ:

```php
define('WU_CPANEL_USERNAME', 'your_cpanel_username');
define('WU_CPANEL_PASSWORD', 'your_cpanel_password');
define('WU_CPANEL_HOST', 'your_cpanel_host');
```

ਵਿਕਲਪਿਕ ਤੌਰ 'ਤੇ, ਤੁਸੀਂ ਇਹ ਵੀ ਪਰਿਭਾਸ਼ਿਤ ਕਰ ਸਕਦੇ ਹੋ:

```php
define('WU_CPANEL_PORT', 2083); // ਡਿਫਾਲਟ 2083 ਹੈ
define('WU_CPANEL_ROOT_DIR', '/public_html'); // ਡਿਫਾਲਟ /public_html ਹੈ
```

## ਸੈੱਟਅੱਪ ਨਿਰਦੇਸ਼

### 1. ਆਪਣੇ cPanel ਕ੍ਰੈਡੈਂਸ਼ੀਅਲ ਲਵੋ

1. ਆਪਣੇ ਹੋਸਟਿੰਗ ਪ੍ਰੋਵਾਈਡਰ ਤੋਂ ਆਪਣਾ cPanel ਯੂਜ਼ਰਨੇਮ ਅਤੇ ਪਾਸਵਰਡ ਪ੍ਰਾਪਤ ਕਰੋ
2. ਆਪਣਾ cPanel ਹੋਸਟ ਪਤਾ ਕਰੋ (ਆਮ ਤੌਰ 'ਤੇ `cpanel.yourdomain.com` ਜਾਂ `yourdomain.com:2083`)

### 2. wp-config.php ਵਿੱਚ Constants ਜੋੜੋ

ਆਪਣੀ `wp-config.php` ਫਾਈਲ ਵਿੱਚ ਹੇਠ ਲਿਖੇ constants ਜੋੜੋ:

```php
define('WU_CPANEL_USERNAME', 'your_cpanel_username');
define('WU_CPANEL_PASSWORD', 'your_cpanel_password');
define('WU_CPANEL_HOST', 'your_cpanel_host');
```

ਵਿਕਲਪਿਕ ਤੌਰ 'ਤੇ, ਤੁਸੀਂ ਪੋਰਟ ਅਤੇ ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਨੂੰ ਕਸਟਮਾਈਜ਼ ਕਰ ਸਕਦੇ ਹੋ:

```php
define('WU_CPANEL_PORT', 2083); // ਜੇ ਤੁਹਾਡਾ cPanel ਵੱਖਰਾ ਪੋਰਟ ਵਰਤਦਾ ਹੈ ਤਾਂ ਬਦਲੋ
define('WU_CPANEL_ROOT_DIR', '/public_html'); // ਜੇ ਤੁਹਾਡੀ document root ਵੱਖਰੀ ਹੈ ਤਾਂ ਬਦਲੋ
```

### 3. ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਚਾਲੂ ਕਰੋ

1. ਆਪਣੇ WordPress admin ਵਿੱਚ, Ultimate Multisite > Settings 'ਤੇ ਜਾਓ
2. "Domain Mapping" ਟੈਬ 'ਤੇ ਨੈਵੀਗੇਟ ਕਰੋ
3. "Host Integrations" ਤੱਕ ਸਕ੍ਰੋਲ ਕਰੋ
4. cPanel ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਨੂੰ ਚਾਲੂ ਕਰੋ
5. "Save Changes" 'ਤੇ ਕਲਿੱਕ ਕਰੋ

## ਇਹ ਕਿਵੇਂ ਕੰਮ ਕਰਦੀ ਹੈ

### Addon ਡੋਮੇਨ

ਜਦੋਂ Ultimate Multisite ਵਿੱਚ ਕੋਈ ਡੋਮੇਨ ਮੈਪ ਕੀਤਾ ਜਾਂਦਾ ਹੈ:

1. ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਡੋਮੇਨ ਨੂੰ addon ਡੋਮੇਨ ਵਜੋਂ ਜੋੜਨ ਲਈ cPanel ਦੀ API ਨੂੰ ਰਿਕੁਐਸਟ ਭੇਜਦੀ ਹੈ
2. ਡੋਮੇਨ ਨੂੰ ਤੁਹਾਡੀ ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਵੱਲ ਪੁਆਇੰਟ ਕਰਨ ਲਈ ਕੌਂਫਿਗਰ ਕੀਤਾ ਜਾਂਦਾ ਹੈ
3. ਜਦੋਂ ਕੋਈ ਡੋਮੇਨ ਮੈਪਿੰਗ ਹਟਾਈ ਜਾਂਦੀ ਹੈ, ਤਾਂ ਇੰਟੀਗ੍ਰੇਸ਼ਨ cPanel ਤੋਂ addon ਡੋਮੇਨ ਹਟਾ ਦੇਵੇਗੀ

### ਸਬਡੋਮੇਨ

ਸਬਡੋਮੇਨ ਮਲਟੀਸਾਈਟ ਇੰਸਟਾਲੇਸ਼ਨਾਂ ਲਈ, ਜਦੋਂ ਨਵੀਂ ਸਾਈਟ ਬਣਾਈ ਜਾਂਦੀ ਹੈ:

1. ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਪੂਰੇ ਡੋਮੇਨ ਤੋਂ ਸਬਡੋਮੇਨ ਵਾਲਾ ਹਿੱਸਾ ਕੱਢਦੀ ਹੈ
2. ਇਹ ਸਬਡੋਮੇਨ ਜੋੜਨ ਲਈ cPanel ਦੀ API ਨੂੰ ਰਿਕੁਐਸਟ ਭੇਜਦੀ ਹੈ
3. ਸਬਡੋਮੇਨ ਨੂੰ ਤੁਹਾਡੀ ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਵੱਲ ਪੁਆਇੰਟ ਕਰਨ ਲਈ ਕੌਂਫਿਗਰ ਕੀਤਾ ਜਾਂਦਾ ਹੈ

## ਮਹੱਤਵਪੂਰਨ ਨੋਟਸ

- ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਤੁਹਾਡੇ cPanel ਖਾਤੇ ਨਾਲ ਸੰਚਾਰ ਕਰਨ ਲਈ cPanel ਦੀ API2 ਵਰਤਦੀ ਹੈ
- ਤੁਹਾਡੇ cPanel ਖਾਤੇ ਕੋਲ addon ਡੋਮੇਨ ਅਤੇ ਸਬਡੋਮੇਨ ਜੋੜਨ ਦੀ ਇਜਾਜ਼ਤ ਹੋਣੀ ਚਾਹੀਦੀ ਹੈ
- ਕੁਝ ਹੋਸਟਿੰਗ ਪ੍ਰੋਵਾਈਡਰ ਤੁਹਾਡੇ ਦੁਆਰਾ ਬਣਾਏ ਜਾ ਸਕਣ ਵਾਲੇ addon ਡੋਮੇਨ ਜਾਂ ਸਬਡੋਮੇਨ ਦੀ ਗਿਣਤੀ ਸੀਮਤ ਕਰ ਸਕਦੇ ਹਨ
- ਇੰਟੀਗ੍ਰੇਸ਼ਨ DNS ਕੌਂਫਿਗਰੇਸ਼ਨ ਨਹੀਂ ਸੰਭਾਲਦੀ; ਤੁਹਾਨੂੰ ਫਿਰ ਵੀ ਆਪਣੇ ਡੋਮੇਨਾਂ ਨੂੰ ਆਪਣੇ ਸਰਵਰ ਦੇ IP ਐਡਰੈੱਸ ਵੱਲ ਪੁਆਇੰਟ ਕਰਨ ਦੀ ਲੋੜ ਹੈ

## ਸਮੱਸਿਆ ਨਿਵਾਰਨ

### API ਕਨੈਕਸ਼ਨ ਸਮੱਸਿਆਵਾਂ
- ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡਾ cPanel ਯੂਜ਼ਰਨੇਮ ਅਤੇ ਪਾਸਵਰਡ ਸਹੀ ਹੈ
- ਚੈੱਕ ਕਰੋ ਕਿ ਤੁਹਾਡਾ cPanel ਹੋਸਟ ਸਹੀ ਅਤੇ ਪਹੁੰਚਯੋਗ ਹੈ
- ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡੇ cPanel ਖਾਤੇ ਕੋਲ ਲੋੜੀਂਦੀਆਂ ਇਜਾਜ਼ਤਾਂ ਹਨ
- ਹੋਸਟ ਲਈ ਪੂਰਾ URL ਵਰਤ ਕੇ ਦੇਖੋ (ਜਿਵੇਂ, `https://cpanel.yourdomain.com`)

### ਡੋਮੇਨ ਨਹੀਂ ਜੋੜਿਆ ਗਿਆ
- ਕਿਸੇ ਵੀ ਗਲਤੀ ਸੁਨੇਹੇ ਲਈ Ultimate Multisite ਲੌਗ ਚੈੱਕ ਕਰੋ
- ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਡੋਮੇਨ ਪਹਿਲਾਂ ਤੋਂ cPanel ਵਿੱਚ ਨਹੀਂ ਜੋੜਿਆ ਗਿਆ
- ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਹਾਡਾ cPanel ਖਾਤਾ addon ਡੋਮੇਨ ਜਾਂ ਸਬਡੋਮੇਨ ਦੀ ਸੀਮਾ ਤੱਕ ਨਹੀਂ ਪਹੁੰਚਿਆ

### SSL ਸਰਟੀਫਿਕੇਟ ਸਮੱਸਿਆਵਾਂ
- ਇੰਟੀਗ੍ਰੇਸ਼ਨ SSL ਸਰਟੀਫਿਕੇਟ ਜਾਰੀ ਕਰਨਾ ਨਹੀਂ ਸੰਭਾਲਦੀ
- ਤੁਹਾਨੂੰ ਆਪਣੇ ਡੋਮੇਨਾਂ ਲਈ SSL ਸਰਟੀਫਿਕੇਟ ਜਾਰੀ ਕਰਨ ਲਈ cPanel ਦੇ SSL/TLS ਟੂਲ ਜਾਂ AutoSSL ਫੀਚਰ ਵਰਤਣ ਦੀ ਲੋੜ ਹੋਵੇਗੀ
- ਬਦਲ ਵਜੋਂ, ਤੁਸੀਂ cPanel ਦੇ AutoSSL ਨਾਲ Let's Encrypt ਵਰਗੀ ਸੇਵਾ ਵਰਤ ਸਕਦੇ ਹੋ
