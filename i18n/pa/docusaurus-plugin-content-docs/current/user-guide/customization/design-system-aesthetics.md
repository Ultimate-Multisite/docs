# ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਸਕਿੱਲ

**ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਸਕਿੱਲ** ਇੱਕ ਗਾਈਡਡ ਪਹੁੰਚ ਹੈ ਜੋ ਤੁਹਾਡੀ ਸਾਈਟ ਦੀ ਵਿਜ਼ੂਅਲ ਪਛਾਣ ਨੂੰ ਸੁਧਾਰਨ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ। ਇਹ ਤੁਹਾਨੂੰ ਟਾਈਪੋਗ੍ਰਾਫੀ, ਰੰਗ, ਸਪੇਸਿੰਗ, ਅਤੇ ਮੋਸ਼ਨ ਟੋਕਨ ਬਾਰੇ ਇਕਸਾਰ ਫੈਸਲੇ ਲੈਣ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ ਜੋ ਤੁਹਾਡੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਨੂੰ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦੇ ਹਨ।

## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਕੀ ਹੈ?

ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਇੱਕ ਢਾਂਚਾਗਤ ਸਕਿੱਲ ਹੈ ਜੋ ਇਹ ਕਵਰ ਕਰਦਾ ਹੈ:

- **ਟਾਈਪੋਗ੍ਰਾਫੀ**: ਫੌਂਟ ਫੈਮਿਲੀ, ਸਾਈਜ਼, ਵੇਟ, ਅਤੇ ਲਾਈਨ ਹਾਈਟ
- **ਰੰਗ**: ਪ੍ਰਾਇਮਰੀ, ਸੈਕੰਡਰੀ, ਐਕਸੈਂਟ, ਅਤੇ ਨਿਊਟਰਲ ਪੈਲੇਟ
- **ਸਪੇਸਿੰਗ**: ਪੈਡਿੰਗ, ਮਾਰਜਿਨ, ਅਤੇ ਗੈਪ ਸਕੇਲ
- **ਬਾਰਡਰ**: ਰੇਡੀਅਸ ਅਤੇ ਵਿਡਥ ਟੋਕਨ
- **ਸ਼ੈਡੋਜ਼**: ਐਲੀਵੇਸ਼ਨ ਅਤੇ ਡੈਪਥ ਟੋਕਨ
- **ਐਨੀਮੇਸ਼ਨ**: ਐਨੀਮੇਸ਼ਨ ਅਤੇ ਟ੍ਰਾਂਜ਼ੀਸ਼ਨ

ਇਹ ਫੈਸਲੇ ਤੁਹਾਡੇ ਥੀਮ ਦੇ `theme.json` ਫਾਈਲ ਵਿੱਚ ਕੈਪਚਰ ਕੀਤੇ ਜਾਂਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਇੱਕ ਇਕਜੁੱਟ ਵਿਜ਼ੂਅਲ ਸਿਸਟਮ ਬਣਦਾ ਹੈ।

## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਕਿਉਂ ਵਰਤਣਾ ਚਾਹੀਦਾ ਹੈ?

### ਇਕਸਾਰਤਾ (Consistency)

ਇੱਕ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦਾ ਹੈ:

- ਸਾਰੇ ਟੈਕਸਟ ਇੱਕੋ ਟਾਈਪੋਗ੍ਰਾਫੀ ਸਕੇਲ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਨ
- ਰੰਗ ਸਾਈਟ ਦੇ ਪੂਰੇ ਖੇਤਰ ਵਿੱਚ ਇਕਸਾਰ ਤੌਰ 'ਤੇ ਵਰਤੇ ਜਾਂਦੇ ਹਨ
- ਸਪੇਸਿੰਗ ਇੱਕ ਪੂਰਵ-ਭਾਵਨਾਤਮਕ ਪੈਟਰਨ ਦੀ ਪਾਲਣਾ ਕਰਦੀ ਹੈ
- ਐਨੀਮੇਸ਼ਨ ਇਕਜੁੱਟ ਮਹਿਸੂਸ ਹੁੰਦੇ ਹਨ

### ਕੁਸ਼ਲਤਾ (Efficiency)

ਇੱਕ-ਇੱਕ ਪੰਨੇ 'ਤੇ ਡਿਜ਼ਾਈਨ ਫੈਸਲੇ ਲੈਣ ਦੀ ਬਜਾਏ, ਤੁਸੀਂ:

- ਟੋਕਨ ਇੱਕ ਵਾਰ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦੇ ਹੋ
- ਉਹਨਾਂ ਨੂੰ ਹਰ ਜਗ੍ਹਾ ਲਾਗੂ ਕਰਦੇ ਹੋ
- ਇੱਕ ਮੁੱਲ ਬਦਲ ਕੇ ਗਲੋਬਲੀ ਅਪਡੇਟ ਕਰਦੇ ਹੋ

### ਲਚਕੀਲਾਪਣ (Flexibility)

ਤੁਸੀਂ ਇਹ ਕਰ ਸਕਦੇ ਹੋ:

- ਆਪਣੇ ਪੂਰੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਨੂੰ ਜਲਦੀ ਨਾਲ ਐਡਜਸਟ ਕਰਨਾ
- ਵੱਖ-ਵੱਖ ਐਸਥੈਟਿਕਸ ਨਾਲ ਪ੍ਰਯੋਗ ਕਰਨਾ
- ਵਿਕਸਤ ਹੋਣ ਦੇ ਨਾਲ ਬ੍ਰਾਂਡ ਦੀ ਇਕਸਾਰਤਾ ਬਣਾਈ ਰੱਖਣਾ

## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਸਕਿੱਲ ਨੂੰ ਟ੍ਰਿਗਰ ਕਰਨਾ

### ਹੱਥੀਂ ਐਕਟੀਵੇਸ਼ਨ (Manual Activation)

ਤੁਸੀਂ ਕਿਸੇ ਵੀ ਸਮੇਂ ਇਸ ਸਕਿੱਲ ਨੂੰ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹੋ:

```
"Help me refine my design system"
```

ਜਾਂ

```
"Let's improve my site's aesthetics"
```

ਜਾਂ

```
"Guide me through design system decisions"
```

### ਆਟੋਮੈਟਿਕ ਸੁਝਾਅ (Automatic Suggestions)

ਏਜੰਟ ਤੁਹਾਨੂੰ ਇਹਨਾਂ ਸਥਿਤੀਆਂ ਵਿੱਚ ਸਕਿੱਲ ਵਰਤਣ ਦਾ ਸੁਝਾਅ ਦੇ ਸਕਦੇ ਹਨ:

- ਜਦੋਂ ਤੁਸੀਂ ਡਿਜ਼ਾਈਨ ਬਦਲਾਅ ਬਾਰੇ ਪੁੱਛਦੇ ਹੋ
- ਜਦੋਂ ਤੁਸੀਂ "ਮਾਡਰਨ" ਜਾਂ "ਪ੍ਰੋਫੈਸ਼ਨਲ" ਰੀਡਿਜ਼ਾਈਨ ਦੀ ਮੰਗ ਕਰਦੇ ਹੋ
- ਜਦੋਂ ਤੁਸੀਂ ਵਿਜ਼ੂਅਲ ਇਕਸਾਰਤਾ ਸੁਧਾਰਨਾ ਚਾਹੁੰਦੇ ਹੋ
- ਜਦੋਂ ਤੁਸੀਂ ਆਪਣੀ ਸਾਈਟ ਲਾਂਚ ਕਰਨ ਦੀ ਤਿਆਰੀ ਕਰ ਰਹੇ ਹੁੰਦੇ ਹੋ

## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਪ੍ਰਕਿਰਿਆ

### ਕਦਮ 1: ਟਾਈਪੋਗ੍ਰਾਫੀ

ਏਜੰਟ ਤੁਹਾਡੇ ਫੌਂਟ ਚੋਣਾਂ ਬਾਰੇ ਪੁੱਛਦਾ ਹੈ:

```
What's your preferred typography style?
- Serif (traditional, elegant)
- Sans-serif (modern, clean)
- Monospace (technical, code-focused)

For your heading font:
- Do you want a distinct heading font, or use the same as body text?
- Preference: bold, elegant, playful, minimal?

For your body font:
- Readability is key. Do you prefer:
  - Larger, more spacious text
  - Compact, efficient text
  - Standard sizing
```

ਫਿਰ ਏਜੰਟ ਇਹ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ:

- **Heading font**: ਸਿਰਲੇਖਾਂ ਅਤੇ ਹੈਡਿੰਗਾਂ ਲਈ ਪ੍ਰਾਇਮਰੀ ਫੌਂਟ
- **Body font**: ਪੈਰਾਗ੍ਰਾਫ ਅਤੇ ਬਾਡੀ ਟੈਕਸਟ ਲਈ ਫੌਂਟ
- **Monospace font**: ਕੋਡ ਅਤੇ ਤਕਨੀਕੀ ਸਮੱਗਰੀ ਲਈ ਫੌਂਟ
- **Size scale**: ਪਹਿਲਾਂ ਤੋਂ ਪਰਿਭਾਸ਼ਿਤ ਸਾਈਜ਼ (small, base, large, XL, ਆਦਿ)
- **Weight scale**: ਫੌਂਟ ਵੇਟਸ (regular, medium, bold, ਆਦਿ)
- **Line height**: ਪੜ੍ਹਨਯੋਗਤਾ ਲਈ ਲਾਈਨਾਂ ਵਿਚਕਾਰ ਸਪੇਸਿੰਗ

### ਕਦਮ 2: ਰੰਗ ਪੈਲੇਟ (Color Palette)

```
Let's define your color palette.

Primary color (your brand color):
- Current: [shows existing color]
- Change to: [color picker or hex code]

Secondary color (supporting color):
- Current: [shows existing color]
- Change to: [color picker or hex code]

Accent color (highlights and CTAs):
- Current: [shows existing color]
- Change to: [color picker or hex code]

Neutral palette (grays for text, borders, backgrounds):
- Light: [color]
- Medium: [color]
- Dark: [color]
```

ਏਜੰਟ ਇੱਕ ਪੂਰਾ ਪੈਲੇਟ ਬਣਾਉਂਦਾ ਹੈ ਜਿਸ ਵਿੱਚ ਸ਼ਾਮਲ ਹਨ:

- ਪ੍ਰਾਇਮਰੀ, ਸੈਕੰਡਰੀ, ਅਤੇ ਐਕਸੈਂਟ ਰੰਗ
- ਨਿਊਟਰਲ ਗ੍ਰੇ (ਹਲਕਾ, ਮੱਧਮ, ਗੂੜ੍ਹਾ)
- ਸੈਮੈਂਟਿਕ ਰੰਗ (success, warning, error)
- ਹੋਵਰ ਅਤੇ ਐਕਟਿਵ ਸਟੇਟਸ

### ਕਦਮ 3: ਸਪੇਸਿੰਗ (Spacing)

```
How much breathing room do you want?

Spacing scale preference:
- Compact (tight, efficient layouts)
- Normal (balanced spacing)
- Spacious (generous whitespace)

This affects:
- Padding inside buttons and cards
- Margins between sections
- Gap between grid items
```

ਏਜੰਟ ਸਪੇਸਿੰਗ ਟੋਕਨ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ:

- Base unit (ਆਮ ਤੌਰ 'ਤੇ 4px ਜਾਂ 8px)
- Scale: xs, sm, md, lg, xl, 2xl
- ਪੈਡਿੰਗ, ਮਾਰਜਿਨ, ਗੈਪ ਲਈ ਖਾਸ ਮੁੱਲ

### ਕਦਮ 4: ਬਾਰਡਰ ਅਤੇ ਸ਼ੈਡੋਜ਼ (Borders and Shadows)

```
Visual depth and definition:

Border radius preference:
- Sharp (no rounding)
- Subtle (small radius)
- Rounded (medium radius)
- Very rounded (large radius)

Shadow depth:
- Flat (no shadows)
- Subtle (light shadows)
- Pronounced (strong shadows)
```

ਏਜੰਟ ਇਹ ਬਣਾਉਂਦਾ ਹੈ:

- ਬਾਰਡਰ ਰੇਡੀਅਸ ਟੋਕਨ (ਬਟਨਾਂ, ਕਾਰਡਾਂ, ਇਨਪੁੱਟਾਂ ਲਈ)
- ਐਲੀਵੇਸ਼ਨ ਲੈਵਲਾਂ ਲਈ ਸ਼ੈਡੋ ਟੋਕਨ
- ਬਾਰਡਰ ਵਿਡਥ ਟੋਕਨ

### ਕਦਮ 5: ਐਨੀਮੇਸ਼ਨ ਅਤੇ ਮੋਸ਼ਨ (Motion and Animation)

```
How should your site feel when interactive?

Animation preference:
- Minimal (no animations)
- Subtle (gentle transitions)
- Playful (noticeable animations)

Specific animations:
- Page transitions: fade, slide, or none?
- Button hover: scale, color change, or both?
- Loading states: spinner, skeleton, or progress bar?
```

ਏਜੰਟ ਇਹ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ:

- ਟ੍ਰਾਂਜ਼ੀਸ਼ਨ ਡਿਊਰੇਸ਼ਨ (fast, normal, slow)
- ਈਜ਼ਿੰਗ ਫੰਕਸ਼ਨ (ease-in, ease-out, ease-in-out)
- ਆਮ ਇੰਟਰੈਕਸ਼ਨਾਂ ਲਈ ਐਨੀਮੇਸ਼ਨ ਕੀਫਰੇਮਜ਼

## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਲਾਗੂ ਕਰਨਾ

### ਆਟੋਮੈਟਿਕ ਐਪਲੀਕੇਸ਼ਨ (Automatic Application)

ਜਦੋਂ ਤੁਸੀਂ ਸਕਿੱਲ ਪੂਰਾ ਕਰ ਲੈਂਦੇ ਹੋ, ਤਾਂ ਏਜੰਟ:

1. ਤੁਹਾਡੇ ਥੀਮ ਦੇ `theme.json` ਨੂੰ ਸਾਰੇ ਟੋਕਨ ਨਾਲ ਅਪਡੇਟ ਕਰਦਾ ਹੈ
2. ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਨੂੰ ਤੁਹਾਡੇ ਐਕਟਿਵ ਥੀਮ 'ਤੇ ਲਾਗੂ ਕਰਦਾ ਹੈ
3. ਨਵੇਂ ਸਿਸਟਮ ਨਾਲ ਮੇਲ ਖਾਂਦੇ ਹੋਣ ਲਈ ਬਲਾਕ ਸਟਾਈਲਜ਼ ਨੂੰ ਰੀਜਨਰੇਟ ਕਰਦਾ ਹੈ
4. ਅਪਡੇਟ ਕੀਤੇ ਥੀਮ ਨੂੰ ਐਕਟੀਵੇਟ ਕਰਦਾ ਹੈ

### ਹੱਥੀਂ ਐਪਲੀਕੇਸ਼ਨ (Manual Application)

ਤੁਸੀਂ ਸਿੱਧੇ ਤੌਰ 'ਤੇ `theme.json` ਨੂੰ ਵੀ ਐਡਿਟ ਕਰ ਸਕਦੇ ਹੋ:

```json
{
  "version": 3,
  "settings": {
    "color": {
      "palette": [
        {
          "color": "#0066CC",
          "name": "Primary",
          "slug": "primary"
        }
      ]
    },
    "typography": {
      "fontFamilies": [
        {
          "fontFamily": "Inter, sans-serif",
          "name": "Body",
          "slug": "body"
        }
      ]
    },
    "spacing": {
      "spacingSizes": [
        {
          "size": "0.5rem",
          "name": "Small",
          "slug": "sm"
        }
      ]
    }
  }
}
```

ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਐਸਥੈਟਿਕਸ ਸਕਿੱਲ ਨੂੰ ਦੁਬਾਰਾ ਚਲਾਓ:

```
"Let's redesign my entire design system"
```

ਇਹ ਤੁਹਾਨੂੰ ਤੁਹਾਡੇ ਮੌਜੂਦਾ ਮੁੱਲਾਂ ਨਾਲ ਸ਼ੁਰੂ ਕਰਦੇ ਹੋਏ, ਸਾਰੇ ਫੈਸਲਿਆਂ ਵਿੱਚ ਦੁਬਾਰਾ ਗਾਈਡ ਕਰੇਗਾ।


## ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਦੀਆਂ ਸਭ ਤੋਂ ਵਧੀਆ ਪ੍ਰੈਕਟਿਸਾਂ (Best Practices)

### ਇਕਸਾਰਤਾ (Consistency)

- ਹਰ ਜਗ੍ਹਾ ਇੱਕੋ ਟੋਕਨ ਵਰਤੋ
- ਇੱਕ-ਇੱਕ ਵਾਰ ਦੇ ਰੰਗ ਜਾਂ ਸਾਈਜ਼ ਨਾ ਬਣਾਓ
- ਮੁੱਲਾਂ ਨੂੰ ਹਾਰਡਕੋਡ ਕਰਨ ਦੀ ਬਜਾਏ ਟੋਕਨ ਦਾ ਹਵਾਲਾ ਲਓ

### ਨਾਮਕਰਨ (Naming)

ਸਪੱਸ਼ਟ, ਸੈਮੈਂਟਿਕ ਨਾਮ ਵਰਤੋ:

- ✓ `primary`, `secondary`, `accent`
- ✗ `blue`, `color-1`, `main`

- ✓ `heading-lg`, `body-sm`
- ✗ `font-size-24`, `text-large`

### ਸਕੇਲੇਬਿਲਟੀ (Scalability)

ਆਪਣੇ ਸਿਸਟਮ ਨੂੰ ਸਕੇਲ ਕਰਨ ਲਈ ਡਿਜ਼ਾਈਨ ਕਰੋ:

- ਪਿਕਸਲ ਦੀ ਬਜਾਏ ਰਿਲੇਟਿਵ ਯੂਨਿਟਸ (rem, em) ਦੀ ਵਰਤੋਂ ਕਰੋ
- ਮਨਮਾਨੇ ਮੁੱਲਾਂ ਦੀ ਬਜਾਏ ਸਕੇਲ ਬਣਾਓ (xs, sm, md, lg, xl)
- ਭਵਿੱਖ ਦੇ ਜੋੜਾਂ ਲਈ ਯੋਜਨਾ ਬਣਾਓ

### ਦਸਤਾਵੇਜ਼ੀਕਰਨ (Documentation)

ਆਪਣੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਨੂੰ ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਕਰੋ:

- ਤੁਸੀਂ ਖਾਸ ਰੰਗ ਕਿਉਂ ਚੁਣੇ
- ਹਰੇਕ ਟੋਕਨ ਕਦੋਂ ਵਰਤਣਾ ਹੈ
- ਛੋਟੇ-ਮੋਟੇ ਅਪਵਾਦ ਅਤੇ ਕਿਨਰੇ ਕੇਸ

## ਆਮ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਪੈਟਰਨ (Common Design System Patterns)

### ਮਾਡਰਨ ਮਿਨੀਮਾਲਿਸਟ (Modern Minimalist)

- ਸੈਨਸ-ਸੈਰਿਫ ਟਾਈਪੋਗ੍ਰਾਫੀ (Inter, Helvetica)
- ਸੀਮਤ ਰੰਗ ਪੈਲੇਟ (2-3 ਰੰਗ)
- ਵਿਸ਼ਾਲ ਸਪੇਸਿੰਗ
- ਸੂਖਮ ਸ਼ੈਡੋਜ਼
- ਸੁਹਾਵਣ, ਤੇਜ਼ ਐਨੀਮੇਸ਼ਨ

### ਗਰਮ ਅਤੇ ਦੋਸਤਾਨਾ (Warm and Friendly)

- ਸੈਰਿਫ ਅਤੇ ਸੈਨਸ-ਸੈਰਿਫ ਦਾ ਮਿਸ਼ਰਣ
- ਗਰਮ ਰੰਗ ਪੈਲੇਟ (ਨਾਰੰਝਾ, ਗਰਮ ਗ੍ਰੇ)
- ਗੋਲੇ ਵਰਗੇ ਕੋਨੇ (Rounded corners)
- ਨਰਮ ਸ਼ੈਡੋਜ਼
- ਖੇਡਣ ਵਾਲੇ ਐਨੀਮੇਸ਼ਨ

### ਪ੍ਰੋਫੈਸ਼ਨਲ ਕਾਰਪੋਰੇਟ (Professional Corporate)

- ਸਾਫ਼ ਸੈਨਸ-ਸੈਰਿਫ (Roboto, Open Sans)
- ਐਕਸੈਂਟ ਰੰਗ ਵਾਲਾ ਨਿਊਟਰਲ ਪੈਲੇਟ
- ਢਾਂਚਾਗਤ ਸਪੇਸਿੰਗ
- ਘੱਟ ਸ਼ੈਡੋਜ਼
- ਸੂਖਮ ਟ੍ਰਾਂਜ਼ੀਸ਼ਨ

### ਰਚਨਾਤਮਕ ਅਤੇ ਬੋਲਡ (Creative and Bold)

- ਵਿਲੱਖਣ ਟਾਈਪੋਗ੍ਰਾਫੀ
- ਬੋਲਡ ਰੰਗ ਪੈਲੇਟ
- ਵੱਖਰਾ ਸਪੇਸਿੰਗ
- ਮਜ਼ਬੂਤ ਸ਼ੈਡੋਜ਼
- ਨੋਟਿਸਯेबल ਐਨੀਮੇਸ਼ਨ

## ਸਮੱਸਿਆ ਨਿਪਟਾਰਾ (Troubleshooting)

**ਮੇਰੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਬਦਲਾਅ ਦਿਖ ਨਹੀਂ ਰਹੇ**
- ਆਪਣੇ ਬ੍ਰਾਊਜ਼ਰ ਕੈਸ਼ ਨੂੰ ਸਾਫ਼ ਕਰੋ
- ਜੇ ਤੁਸੀਂ ਸਟੈਟਿਕ ਜਨਰੇਟਰ ਵਰਤ ਰਹੇ ਹੋ ਤਾਂ ਆਪਣੀ ਸਾਈਟ ਨੂੰ ਰੀਬਿਲਡ ਕਰੋ
- ਜਾਂਚ ਕਰੋ ਕਿ `theme.json` ਵੈਲਿਡ JSON ਹੈ
- ਪੁਸ਼ਟੀ ਕਰੋ ਕਿ ਥੀਮ ਐਕਟਿਵ ਹੈ

**ਵੱਖ-ਵੱਖ ਪੰਨਿਆਂ 'ਤੇ ਰੰਗ ਵੱਖਰੇ ਲੱਗ ਰਹੇ ਹਨ**
- ਪਲੱਗਇਨਾਂ ਵਿੱਚ ਟਕਰਾਅ ਵਾਲੇ CSS ਦੀ ਜਾਂਚ ਕਰੋ
- ਪੁਸ਼ਟੀ ਕਰੋ ਕਿ ਸਾਰੇ ਪੰਨੇ ਇੱਕੋ ਥੀਮ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਨ
- ਕਿਸੇ ਵੀ ਕੈਸ਼ਿੰਗ ਪਲੱਗਇਨ ਨੂੰ ਸਾਫ਼ ਕਰੋ

**ਮੈਂ ਪਿਛਲੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ 'ਤੇ ਵਾਪਸ ਜਾਣਾ ਚਾਹੁੰਦਾ ਹਾਂ**
- ਏਜੰਟ ਨੂੰ ਪੁੱਛੋ: "Show me my design system history"
- ਪਿਛਲੇ ਮੁੱਲਾਂ ਲਈ theme.json ਨੂੰ ਹੱਥੀਂ ਐਡਿਟ ਕਰੋ
- ਵੱਖਰੇ ਚੋਣਾਂ ਨਾਲ ਸਕਿੱਲ ਨੂੰ ਦੁਬਾਰਾ ਚਲਾਓ

## ਅਗਲੇ ਕਦਮ (Next Steps)

ਆਪਣਾ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਪਰਿਭਾਸ਼ਿਤ ਕਰਨ ਤੋਂ ਬਾਅਦ:

1. **ਆਪਣੀ ਸਾਈਟ ਦੀ ਸਮੀਖਿਆ ਕਰੋ**: ਨਵਾਂ ਡਿਜ਼ਾਈਨ ਦੇਖਣ ਲਈ ਆਪਣੀ ਸਾਈਟ 'ਤੇ ਜਾਓ
2. **ਹੋਰ ਸੁਧਾਰ ਕਰੋ**: ਸਕਿੱਲ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਹੋਰ ਐਡਜਸਟਮੈਂਟ ਕਰੋ
3. **ਟੈਂਪਲੇਟ ਬਣਾਓ**: ਆਪਣੇ ਡਿਜ਼ਾਈਨ ਟੋਕਨ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਕਸਟਮ ਬਲਾਕ ਟੈਂਪਲੇਟ ਬਣਾਓ
4. **ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਕਰੋ**: ਆਪਣੇ ਡਿਜ਼ਾਈਨ ਸਿਸਟਮ ਨੂੰ ਟੀਮ ਮੈਂਬਰਾਂ ਨਾਲ ਸਾਂਝਾ ਕਰੋ
