# చెక్అవుట్ ఫీల్డ్ మరియు కస్టమర్ DNS నిర్వహణ

## డొమైన్ సెలెక్షన్ చెక్అవుట్ ఫీల్డ్

**Domain Selection** ఫీల్డ్ అనేది ఒక చెక్అవుట్ ఎలిమెంట్, ఇది కస్టమర్‌కు వారి సైట్ యొక్క డొమైన్ ఎలా పొందాలనే ఎంపిక ఇస్తుంది. డొమైన్ అమ్మకాలను ప్రారంభించడానికి దీనిని ఏ చెక్అవుట్ ఫారమ్‌లోనైనా జోడించవచ్చు.

### చెక్అవుట్ ఫారమ్‌లో ఫీల్డ్‌ను జోడించడం

1. **Network Admin › Ultimate Multisite › Checkout Forms**కి వెళ్లండి
2. ఒక చెక్అవుట్ ఫారమ్‌ను తెరవండి లేదా సృష్టించండి
3. చెక్అవుట్ ఎడిటర్‌లో, **Add Field** పై క్లిక్ చేయండి
4. ఫీల్డ్ జాబితా నుండి **Domain Selection** ఎంచుకోండి
5. ఫీల్డ్ ఆప్షన్‌లను కాన్ఫిగర్ చేయండి (కింద చూడండి)
6. ఫారమ్‌ను సేవ్ చేయండి

### ఫీల్డ్ ఆప్షన్‌లు

**Domain modes** — కస్టమర్ ఏ ట్యాబ్‌లను చూస్తారో ఎంచుకోండి. ప్రతి మోడ్‌ను స్వతంత్రంగా ఎనేబుల్ లేదా డిసేబుల్ చేయవచ్చు:

| Mode | ఏమి చేస్తుంది |
|---|---|
| **Subdomain** | కస్టమర్ మీ నెట్‌వర్క్‌లో ఉచిత సబ్-డొమైన్ ఉపయోగిస్తాడు (ఉదాహరణకు, `mysite.yournetwork.com`). చెల్లింపు అవసరం లేదు. |
| **Register New Domain** | కస్టమర్ కొత్త డొమైన్‌ను శోధిస్తాడు మరియు మీ కాన్ఫిగర్ చేసిన ప్రొవైడర్ ద్వారా దాన్ని రిజిస్టర్ చేస్తాడు. ధరల కోసం సరిపోలే డొమైన్ ఉత్పత్తిని ఉపయోగిస్తుంది. |
| **Existing Domain** | కస్టమర్ తాను ఇప్పటికే కలిగి ఉన్న డొమైన్‌ను మ్యాప్ చేస్తాడు. రిజిస్ట్రేషన్ ఫీజు లేదు. డొమైన్ ఆటోమేటిక్‌గా వారి సైట్‌కు మ్యాప్ చేయబడుతుంది. |

**Default mode** — మూడు మోడ్‌లు ఎనేబుల్ అయినప్పుడు, ఏ ట్యాబ్ మొదట ఓపెన్ అవుతుంది. డొమైన్ రిజిస్ట్రేషన్‌ను ఐచ్ఛికంగా ఉంచడానికి **Subdomain**కి సెట్ చేయండి, లేదా కొనుగోళ్లను ప్రోత్సహించడానికి **Register New Domain**కి సెట్ చేయండి.

**Domain product** — ఐచ్ఛికంగా ఈ ఫీల్డ్‌ను ఒక నిర్దిష్ట డొమైన్ ఉత్పత్తికి పిన్ చేయండి. సెట్ చేయకపోతే, అడాన్ ఆటోమేటిక్‌గా కస్టమర్ శోధించిన TLD ఆధారంగా సరిపోలే ఉత్పత్తిని ఎంచుకుంటుంది.

### రిజిస్ట్రెంట్ కాంటాక్ట్ ఫీల్డ్‌లు

కస్టమర్ **Register New Domain** ట్యాబ్‌ను ఎంచుకున్నప్పుడు, చెక్అవుట్ ఫారమ్ రిజిస్ట్రెంట్ కాంటాక్ట్ ఫీల్డ్‌లను ఇన్‌లైన్‌లో జోడిస్తుంది:

- ఫస్ట్ నేమ్ / లాస్ట్ నేమ్
- ఇమెయిల్ అడ్రస్
- అడ్రస్ (లైన్ 1, సిటీ, స్టేట్/ప్రావిన్స్, పోస్టల్ కోడ్, కంట్రీ)
- ఫోన్ నంబర్

ఇవి అన్ని రిజిస్ట్రార్ల ద్వారా అవసరం మరియు రిజిస్ట్రేషన్ API కాల్ చేయబడే ముందు ధృవీకరించబడతాయి. ఫోన్ నంబర్‌లను రిజిస్ట్రార్ల ద్వారా ఆశించిన `+CC.NNN` అంతర్జాతీయ ఫార్మాట్‌లోకి ఆటోమేటిక్‌గా ఫార్మాట్ చేస్తారు.

### ఆటో-జనరేటెడ్ సైట్ URL

కస్టమర్ డొమైన్‌ను రిజిస్టర్ చేసినా లేదా మ్యాప్ చేసినా, సైట్ URL ఫీల్డ్ ఎంచుకున్న డొమైన్ నుండి ఆటోమేటిక్‌గా నింపబడుతుంది. కస్టమర్‌లు వేరే URL ఫీల్డ్‌ను నింపవలసిన అవసరం లేదు.

### శోధన ప్రవర్తన (Search behaviour)

- కస్టమర్ టైప్ చేస్తున్నప్పుడు డొమైన్ లభ్యత AJAX తో రియల్-టైమ్‌లో తనిఖీ చేయబడుతుంది
- ఇష్టపడే డొమైన్ అందుబాటులో లేనప్పుడు ప్రత్యామ్నాయ TLD సూచనలు చూపబడతాయి
- ధరలు లైవ్‌గా తీసుకుని స్పష్టంగా ప్రదర్శించబడతాయి (రిజిస్ట్రేషన్ ధర, పునరుద్ధరణ ధర, ఐచ్ఛిక WHOIS ప్రైవసీ ఫీజు)
- కూపన్ కోడ్‌లు ఏ ఇతర ఉత్పత్తి మాదిరిగానే డొమైన్ ఉత్పత్తులకు వర్తిస్తాయి

**శోధన ప్రతిస్పందనను ట్యూన్ చేయడం:**

```php
// స్లో కనెక్షన్లపై API కాల్‌లను తగ్గించడానికి డీబౌన్స్ డిలే (మిల్లీసెకన్లు) పెంచండి
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**డొమైన్ శోధన ఫారమ్‌కు కస్టమ్ ఫీల్డ్‌లను జోడించడం:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## కస్టమర్ DNS నిర్వహణ

కస్టమర్‌లు తమ రిజిస్టర్డ్ డొమైన్‌ల కోసం DNS రికార్డులను **My Account** పేజీ నుండి, వారి డొమైన్ ఎంట్రీ కింద నిర్వహించవచ్చు.

### మద్దతు ఉన్న రికార్డ్ రకాలు

| Type | ఉపయోగం |
|---|---|
| **A** | హోస్ట్‌నేమ్‌ను IPv4 చిరునామాకు మ్యాప్ చేయండి |
| **AAAA** | హోస్ట్‌నేమ్‌ను IPv6 చిరునామాకు మ్యాప్ చేయండి |
| **CNAME** | మరొక హోస్ట్‌నేమ్‌ను చూపించే ఒక మారుపేరు (alias) సృష్టించండి |
| **MX** | మెయిల్ ఎక్స్ఛేంజ్ సర్వర్‌ను సెట్ చేయండి |
| **TXT** | SPF, DMARC, ధృవీకరణ లేదా ఇతర టెక్స్ట్ రికార్డులను జోడించండి |

### ఏ ప్రొవైడర్లు DNS నిర్వహణకు మద్దతు ఇస్తాయి?

DNS నిర్వహణ (రికార్డులను జోడించడం, సవరించడం, తొలగించడం) **OpenSRS**, **ResellerClub**, మరియు **Enom** తో అందుబాటులో ఉంది. Namecheap, GoDaddy, మరియు NameSilo డొమైన్‌లు స్టేటస్ మరియు గడువు సమాచారాన్ని ప్రదర్శిస్తాయి, కానీ DNSని రిజిస్ట్రార్ యొక్క కంట్రోల్ ప్యానెల్‌లో నేరుగా నిర్వహించాలి.

### డిఫాల్ట్ DNS రికార్డులు

డొమైన్ రిజిస్టర్ అయినప్పుడు ఆటోమేటిక్‌గా వర్తించే డిఫాల్ట్ DNS రికార్డులను మీరు కాన్ఫిగర్ చేయవచ్చు. **Settings › Domain Seller › Default DNS Records**కి వెళ్లండి.

డిఫాల్ట్ రికార్డ్ విలువలు రెండు టోకెన్‌లకు మద్దతు ఇస్తాయి:

| Token | దేనితో భర్తీ చేయబడుతుంది |
|---|---|
| `{DOMAIN}` | రిజిస్టర్ చేయబడిన డొమైన్ పేరు (ఉదా. `example.com`) |
| `{SITE_URL}` | కస్టమర్ సైట్ కోసం వర్డ్‌ప్రెస్ సైట్ URL |

**ఉదాహరణ — అపెక్స్ డొమైన్ మరియు wwwని మీ సర్వర్ IPకి పాయింట్ చేయడం:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### అడ్మిన్: DNS వీక్షణ మరియు సవరణ

**Network Admin › Ultimate Multisite › Domains** లోని డొమైన్ ఎడిట్ పేజీ నుండి నెట్‌వర్క్ అడ్మిన్‌లు ఏ కస్టమర్ డొమైన్ యొక్క DNS రికార్డులను కూడా వీక్షించవచ్చు మరియు సవరించవచ్చు.
