# ਚੈਕਆਉਟ ਫੀਲਡ ਅਤੇ ਗਾਹਕ DNS ਪ੍ਰਬੰਧਨ

## ਡੋਮੇਨ ਚੋਣ ਚੈਕਆਉਟ ਫੀਲਡ

**ਡੋਮੇਨ ਚੋਣ** ਫੀਲਡ ਇੱਕ ਚੈਕਆਉਟ ਤੱਤ ਹੈ ਜੋ ਗਾਹਕਾਂ ਨੂੰ ਇਹ ਚੋਣ ਦਿੰਦਾ ਹੈ ਕਿ ਉਹ ਆਪਣੀ ਸਾਈਟ ਦਾ ਡੋਮੇਨ ਕਿਵੇਂ ਲੈਣ। ਡੋਮੇਨ ਵੇਚਣ ਨੂੰ ਸਮਰੱਥ ਕਰਨ ਲਈ ਇਸਨੂੰ ਕਿਸੇ ਵੀ ਚੈਕਆਉਟ ਫਾਰਮ ਵਿੱਚ ਜੋੜੋ।

### ਚੈਕਆਉਟ ਫਾਰਮ ਵਿੱਚ ਫੀਲਡ ਜੋੜਨਾ

1. **Network Admin › Ultimate Multisite › Checkout Forms** ’ਤੇ ਜਾਓ
2. ਚੈਕਆਉਟ ਫਾਰਮ ਖੋਲ੍ਹੋ ਜਾਂ ਬਣਾਓ
3. ਚੈਕਆਉਟ ਐਡੀਟਰ ਵਿੱਚ, **ਫੀਲਡ ਜੋੜੋ** ’ਤੇ ਕਲਿੱਕ ਕਰੋ
4. ਫੀਲਡ ਸੂਚੀ ਵਿੱਚੋਂ **ਡੋਮੇਨ ਚੋਣ** ਚੁਣੋ
5. ਫੀਲਡ ਵਿਕਲਪਾਂ ਨੂੰ ਕਨਫਿਗਰ ਕਰੋ (ਹੇਠਾਂ ਵੇਖੋ)
6. ਫਾਰਮ ਸੇਵ ਕਰੋ

### ਫੀਲਡ ਵਿਕਲਪ

**ਡੋਮੇਨ ਮੋਡ** — ਚੁਣੋ ਕਿ ਗਾਹਕ ਕਿਹੜੀਆਂ ਟੈਬਾਂ ਵੇਖਦਾ ਹੈ। ਹਰ ਮੋਡ ਨੂੰ ਵੱਖ-ਵੱਖ ਸਮਰੱਥ ਜਾਂ ਅਸਮਰੱਥ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ:

| ਮੋਡ | ਇਹ ਕੀ ਕਰਦਾ ਹੈ |
|---|---|
| **ਸਬਡੋਮੇਨ** | ਗਾਹਕ ਤੁਹਾਡੇ ਨੈੱਟਵਰਕ ’ਤੇ ਮੁਫ਼ਤ ਸਬਡੋਮੇਨ ਵਰਤਦਾ ਹੈ (ਉਦਾਹਰਨ ਲਈ, `mysite.yournetwork.com`)। ਭੁਗਤਾਨ ਦੀ ਲੋੜ ਨਹੀਂ। |
| **ਨਵਾਂ ਡੋਮੇਨ ਰਜਿਸਟਰ ਕਰੋ** | ਗਾਹਕ ਨਵੇਂ ਡੋਮੇਨ ਦੀ ਖੋਜ ਕਰਦਾ ਹੈ ਅਤੇ ਇਸਨੂੰ ਤੁਹਾਡੇ ਕਨਫਿਗਰ ਕੀਤੇ ਪ੍ਰਦਾਤਾ ਰਾਹੀਂ ਰਜਿਸਟਰ ਕਰਦਾ ਹੈ। ਕੀਮਤ ਲਈ ਮਿਲਦੇ ਡੋਮੇਨ ਉਤਪਾਦ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ। |
| **ਮੌਜੂਦਾ ਡੋਮੇਨ** | ਗਾਹਕ ਉਹ ਡੋਮੇਨ ਮੈਪ ਕਰਦਾ ਹੈ ਜੋ ਉਸਦੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ ਹੈ। ਕੋਈ ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਫੀਸ ਨਹੀਂ। ਡੋਮੇਨ ਆਪਣੇ ਆਪ ਉਸਦੀ ਸਾਈਟ ਨਾਲ ਮੈਪ ਹੋ ਜਾਂਦਾ ਹੈ। |

**ਡਿਫਾਲਟ ਮੋਡ** — ਜਦੋਂ ਤਿੰਨੇ ਮੋਡ ਸਮਰੱਥ ਹੋਣ, ਕਿਹੜੀ ਟੈਬ ਪਹਿਲਾਂ ਖੁੱਲ੍ਹਦੀ ਹੈ। ਡੋਮੇਨ ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਨੂੰ ਵਿਕਲਪਿਕ ਰੱਖਣ ਲਈ **ਸਬਡੋਮੇਨ** ’ਤੇ ਸੈੱਟ ਕਰੋ, ਜਾਂ ਖਰੀਦਾਂ ਨੂੰ ਉਤਸ਼ਾਹਿਤ ਕਰਨ ਲਈ **ਨਵਾਂ ਡੋਮੇਨ ਰਜਿਸਟਰ ਕਰੋ** ’ਤੇ ਸੈੱਟ ਕਰੋ।

**ਡੋਮੇਨ ਉਤਪਾਦ** — ਵਿਕਲਪਕ ਤੌਰ ’ਤੇ ਇਸ ਫੀਲਡ ਨੂੰ ਕਿਸੇ ਖਾਸ ਡੋਮੇਨ ਉਤਪਾਦ ਨਾਲ ਪਿੰਨ ਕਰੋ। ਜੇ ਸੈੱਟ ਨਹੀਂ ਕੀਤਾ ਗਿਆ, ਤਾਂ ਐਡਆਨ ਆਪਣੇ ਆਪ ਗਾਹਕ ਵੱਲੋਂ ਖੋਜੇ TLD ਦੇ ਆਧਾਰ ’ਤੇ ਮਿਲਦਾ ਉਤਪਾਦ ਚੁਣਦਾ ਹੈ।

### ਰਜਿਸਟ੍ਰੈਂਟ ਸੰਪਰਕ ਫੀਲਡ

ਜਦੋਂ ਗਾਹਕ **ਨਵਾਂ ਡੋਮੇਨ ਰਜਿਸਟਰ ਕਰੋ** ਟੈਬ ਚੁਣਦਾ ਹੈ, ਚੈਕਆਉਟ ਫਾਰਮ ਰਜਿਸਟ੍ਰੈਂਟ ਸੰਪਰਕ ਫੀਲਡਾਂ ਨੂੰ ਇਨਲਾਈਨ ਜੋੜਦਾ ਹੈ:

- ਪਹਿਲਾ ਨਾਮ / ਆਖਰੀ ਨਾਮ
- ਈਮੇਲ ਪਤਾ
- ਪਤਾ (ਲਾਈਨ 1, ਸ਼ਹਿਰ, ਰਾਜ/ਸੂਬਾ, ਡਾਕ ਕੋਡ, ਦੇਸ਼)
- ਫ਼ੋਨ ਨੰਬਰ

ਇਹ ਸਭ ਰਜਿਸਟ੍ਰਾਰਾਂ ਵੱਲੋਂ ਲਾਜ਼ਮੀ ਹਨ ਅਤੇ ਰਜਿਸਟ੍ਰੇਸ਼ਨ API ਕਾਲ ਕੀਤੇ ਜਾਣ ਤੋਂ ਪਹਿਲਾਂ ਵੈਲੀਡੇਟ ਕੀਤੇ ਜਾਂਦੇ ਹਨ। ਫ਼ੋਨ ਨੰਬਰ ਆਪਣੇ ਆਪ ਰਜਿਸਟ੍ਰਾਰਾਂ ਵੱਲੋਂ ਉਮੀਦ ਕੀਤੇ `+CC.NNN` ਅੰਤਰਰਾਸ਼ਟਰੀ ਫਾਰਮੈਟ ਵਿੱਚ ਫਾਰਮੈਟ ਕੀਤੇ ਜਾਂਦੇ ਹਨ।

### ਆਪਣੇ ਆਪ ਬਣਿਆ ਸਾਈਟ URL

ਜਦੋਂ ਗਾਹਕ ਡੋਮੇਨ ਰਜਿਸਟਰ ਜਾਂ ਮੈਪ ਕਰਦਾ ਹੈ, ਸਾਈਟ URL ਫੀਲਡ ਚੁਣੇ ਡੋਮੇਨ ਤੋਂ ਆਪਣੇ ਆਪ ਭਰ ਜਾਂਦਾ ਹੈ। ਗਾਹਕਾਂ ਨੂੰ ਵੱਖਰੀ URL ਫੀਲਡ ਭਰਨ ਦੀ ਲੋੜ ਨਹੀਂ ਹੁੰਦੀ।

### ਖੋਜ ਵਰਤਾਅ

- ਗਾਹਕ ਦੇ ਟਾਈਪ ਕਰਨ ਨਾਲ ਹੀ ਡੋਮੇਨ ਉਪਲਬਧਤਾ AJAX ਨਾਲ ਰੀਅਲ-ਟਾਈਮ ਵਿੱਚ ਜਾਂਚੀ ਜਾਂਦੀ ਹੈ
- ਜਦੋਂ ਤਰਜੀਹੀ ਡੋਮੇਨ ਉਪਲਬਧ ਨਹੀਂ ਹੁੰਦਾ ਤਾਂ ਵਿਕਲਪਿਕ TLD ਸੁਝਾਅ ਦਿਖਾਏ ਜਾਂਦੇ ਹਨ
- ਕੀਮਤ ਲਾਈਵ ਪ੍ਰਾਪਤ ਕੀਤੀ ਜਾਂਦੀ ਹੈ ਅਤੇ ਸਪਸ਼ਟ ਤੌਰ ’ਤੇ ਦਿਖਾਈ ਜਾਂਦੀ ਹੈ (ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਕੀਮਤ, ਨਵੀਨੀਕਰਨ ਕੀਮਤ, ਵਿਕਲਪਿਕ WHOIS ਪਰਦੇਦਾਰੀ ਫੀਸ)
- ਕੂਪਨ ਕੋਡ ਡੋਮੇਨ ਉਤਪਾਦਾਂ ’ਤੇ ਕਿਸੇ ਵੀ ਹੋਰ ਉਤਪਾਦ ਵਾਂਗ ਹੀ ਲਾਗੂ ਹੁੰਦੇ ਹਨ

**ਖੋਜ ਪ੍ਰਤੀਕਿਰਿਆਸ਼ੀਲਤਾ ਟਿਊਨ ਕਰਨਾ:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ਡੋਮੇਨ ਖੋਜ ਫਾਰਮ ਵਿੱਚ ਕਸਟਮ ਫੀਲਡਾਂ ਜੋੜਨਾ:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ਗਾਹਕ DNS ਪ੍ਰਬੰਧਨ

ਗਾਹਕ ਆਪਣੇ ਰਜਿਸਟਰ ਕੀਤੇ ਡੋਮੇਨਾਂ ਲਈ DNS ਰਿਕਾਰਡ **My Account** ਪੰਨੇ ਤੋਂ, ਆਪਣੇ ਡੋਮੇਨ ਦੀ ਐਂਟਰੀ ਹੇਠਾਂ, ਪ੍ਰਬੰਧਿਤ ਕਰ ਸਕਦੇ ਹਨ।

### ਸਮਰਥਿਤ ਰਿਕਾਰਡ ਕਿਸਮਾਂ

| ਕਿਸਮ | ਵਰਤੋਂ |
|---|---|
| **A** | ਹੋਸਟਨਾਮ ਨੂੰ IPv4 ਪਤੇ ਨਾਲ ਮੈਪ ਕਰੋ |
| **AAAA** | ਹੋਸਟਨਾਮ ਨੂੰ IPv6 ਪਤੇ ਨਾਲ ਮੈਪ ਕਰੋ |
| **CNAME** | ਕਿਸੇ ਹੋਰ ਹੋਸਟਨਾਮ ਵੱਲ ਇਸ਼ਾਰਾ ਕਰਦਾ ਉਪਨਾਮ ਬਣਾਓ |
| **MX** | ਮੇਲ ਐਕਸਚੇਂਜ ਸਰਵਰ ਸੈੱਟ ਕਰੋ |
| **TXT** | SPF, DMARC, ਵੇਰੀਫਿਕੇਸ਼ਨ, ਜਾਂ ਹੋਰ ਟੈਕਸਟ ਰਿਕਾਰਡ ਜੋੜੋ |

### ਕਿਹੜੇ ਪ੍ਰਦਾਤਾ DNS ਪ੍ਰਬੰਧਨ ਦਾ ਸਮਰਥਨ ਕਰਦੇ ਹਨ?

DNS ਪ੍ਰਬੰਧਨ (ਰਿਕਾਰਡ ਜੋੜਨਾ, ਸੰਪਾਦਿਤ ਕਰਨਾ, ਮਿਟਾਉਣਾ) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, ਅਤੇ **Openprovider** ਨਾਲ ਉਪਲਬਧ ਹੈ। **Hostinger** ਡੋਮੇਨ Domain Seller ਰਾਹੀਂ ਨੇਮਸਰਵਰ ਅੱਪਡੇਟ ਕਰ ਸਕਦੇ ਹਨ; ਹੋਸਟ ਕੀਤੇ ਡੋਮੇਨਾਂ ਲਈ DNS ਰਿਕਾਰਡ ਮੁੱਖ Hostinger ਡੋਮੇਨ-ਮੈਪਿੰਗ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਵੱਲੋਂ ਪ੍ਰਬੰਧਿਤ ਕੀਤੇ ਜਾਂਦੇ ਹਨ। Namecheap, GoDaddy, ਅਤੇ NameSilo ਡੋਮੇਨ ਸਥਿਤੀ ਅਤੇ ਸਮਾਪਤੀ ਜਾਣਕਾਰੀ ਦਿਖਾਉਂਦੇ ਹਨ ਪਰ DNS ਨੂੰ ਸਿੱਧਾ ਰਜਿਸਟ੍ਰਾਰ ਦੇ ਕੰਟਰੋਲ ਪੈਨਲ ਵਿੱਚ ਪ੍ਰਬੰਧਿਤ ਕਰਨਾ ਲਾਜ਼ਮੀ ਹੈ।

### ਡਿਫਾਲਟ DNS ਰਿਕਾਰਡ

ਤੁਸੀਂ ਡਿਫਾਲਟ DNS ਰਿਕਾਰਡ ਕਨਫਿਗਰ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਡੋਮੇਨ ਰਜਿਸਟਰ ਹੋਣ ’ਤੇ ਆਪਣੇ ਆਪ ਲਾਗੂ ਕੀਤੇ ਜਾਂਦੇ ਹਨ। **Settings › Domain Seller › Default DNS Records** ’ਤੇ ਜਾਓ।

ਡਿਫਾਲਟ ਰਿਕਾਰਡ ਮੁੱਲ ਦੋ ਟੋਕਨਾਂ ਦਾ ਸਮਰਥਨ ਕਰਦੇ ਹਨ:

| ਟੋਕਨ | ਇਸ ਨਾਲ ਬਦਲਿਆ ਜਾਂਦਾ ਹੈ |
|---|---|
| `{DOMAIN}` | ਰਜਿਸਟਰ ਕੀਤਾ ਡੋਮੇਨ ਨਾਮ (ਉਦਾਹਰਨ ਲਈ, `example.com`) |
| `{SITE_URL}` | ਗਾਹਕ ਦੀ ਸਾਈਟ ਲਈ WordPress ਸਾਈਟ URL |

**ਉਦਾਹਰਨ — apex ਡੋਮੇਨ ਅਤੇ www ਨੂੰ ਆਪਣੇ ਸਰਵਰ IP ਵੱਲ ਪੁਆਇੰਟ ਕਰੋ:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ਐਡਮਿਨ: DNS ਦੇਖਣਾ ਅਤੇ ਸੰਪਾਦਿਤ ਕਰਨਾ

ਨੈੱਟਵਰਕ ਐਡਮਿਨ **Network Admin › Ultimate Multisite › Domains** ਵਿੱਚ ਡੋਮੇਨ ਦੇ ਸੰਪਾਦਨ ਪੰਨੇ ਤੋਂ ਕਿਸੇ ਵੀ ਗਾਹਕ ਡੋਮੇਨ ਲਈ DNS ਰਿਕਾਰਡ ਵੇਖ ਅਤੇ ਸੰਪਾਦਿਤ ਕਰ ਸਕਦੇ ਹਨ।
