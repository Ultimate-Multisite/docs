# ਮੁੱਢਲੀਆਂ ਧਾਰਨਾਵਾਂ

ਜੇਕਰ ਤੁਸੀਂ WordPress Multisite ਨੂੰ ਨਵੇਂ ਵਰਤਣ ਲੱਗੇ ਹੋ ਅਤੇ ਹੁਣੇ ਹੀ Ultimate Multisite ਵਰਤਣੀ ਸ਼ੁਰੂ ਕੀਤੀ ਹੈ, ਤਾਂ ਸ਼ੁਰੂ ਵਿੱਚ ਬਹੁਤ ਸਾਰੇ ਨਵੇਂ ਸ਼ਬਦ ਅਤੇ ਵਾਕੰਸ਼ ਸਿੱਖਣੇ ਪੈ ਸਕਦੇ ਹਨ। ਇਹਨਾਂ ਨੂੰ ਸਮਝਣਾ ਜ਼ਰੂਰੀ ਹੈ ਕਿਉਂਕਿ ਤੁਹਾਨੂੰ ਇਸ ਪਲੇਟਫਾਰਮ ਅਤੇ ਇਸ ਦੇ ਕੰਮ ਕਰਨ ਦੇ ਤਰੀਕੇ ਨੂੰ ਪੂਰੀ ਤਰ੍ਹਾਂ ਸਮਝਣਾ ਪਵੇਗਾ।

ਇਸ ਲੇਖ ਵਿੱਚ, ਅਸੀਂ WordPress ਦੀਆਂ ਕੁਝ ਮੁੱਖ ਧਾਰਨਾਵਾਂ ਨੂੰ ਸਮਝਾਉਣ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰਾਂਗੇ। ਇਹਨਾਂ ਵਿੱਚੋਂ ਕੁਝ ਵਰਤੋਂਕਾਰਾਂ ਲਈ ਵੱਧ ਕੰਮ ਆਉਣਗੀਆਂ, ਕੁਝ ਡਿਵੈਲਪਰਾਂ ਲਈ, ਅਤੇ ਕੁਝ ਦੋਵਾਂ ਲਈ।

## WordPress Multisite

WordPress **Multisite** ਇੱਕ ਕਿਸਮ ਦੀ WordPress ਇੰਸਟਾਲੇਸ਼ਨ ਹੈ ਜੋ ਤੁਹਾਨੂੰ ਇੱਕ ਹੀ WordPress dashboard ਤੋਂ ਕਈ ਵੈੱਬਸਾਈਟਾਂ ਦਾ ਨੈੱਟਵਰਕ ਬਣਾਉਣ ਅਤੇ ਸੰਭਾਲਣ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ। ਤੁਸੀਂ ਸਾਈਟਾਂ ਦੀ ਗਿਣਤੀ, ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ, themes, ਅਤੇ ਵਰਤੋਂਕਾਰ ਭੂਮਿਕਾਵਾਂ ਸਮੇਤ ਸਭ ਕੁਝ ਸੰਭਾਲ ਸਕਦੇ ਹੋ। ਸੈਂਕੜੇ ਅਤੇ ਹਜ਼ਾਰਾਂ ਸਾਈਟਾਂ ਨੂੰ ਸੰਭਾਲਣਾ ਵੀ ਸੰਭਵ ਹੈ।

## Network

WordPress ਦੀ ਭਾਸ਼ਾ ਵਿੱਚ, ਮਲਟੀਸਾਈਟ ਨੈੱਟਵਰਕ ਉਹ ਹੈ ਜਿੱਥੇ ਕਈ **ਸਬਸਾਈਟਾਂ** ਨੂੰ ਇੱਕ ਹੀ dashboard ਤੋਂ ਸੰਭਾਲਿਆ ਜਾ ਸਕਦਾ ਹੈ। ਭਾਵੇਂ ਮਲਟੀਸਾਈਟ ਨੈੱਟਵਰਕ ਬਣਾਉਣ ਦਾ ਤਰੀਕਾ ਵੱਖ-ਵੱਖ ਹੋਸਟਿੰਗ ਪ੍ਰਦਾਤਾਵਾਂ ਲਈ ਵੱਖਰਾ ਹੁੰਦਾ ਹੈ, ਪਰ ਆਖਰੀ ਨਤੀਜਾ ਆਮ ਤੌਰ 'ਤੇ **wp-config.php** ਫਾਈਲ ਵਿੱਚ ਕੁਝ ਵਾਧੂ ਨਿਰਦੇਸ਼ ਹੁੰਦੇ ਹਨ ਜੋ WordPress ਨੂੰ ਦੱਸਦੇ ਹਨ ਕਿ ਇਹ ਇਸ ਖਾਸ ਮੋਡ ਵਿੱਚ ਕੰਮ ਕਰ ਰਿਹਾ ਹੈ।

ਮਲਟੀਸਾਈਟ ਨੈੱਟਵਰਕ ਅਤੇ ਸਟੈਂਡ-ਅਲੋਨ WordPress ਇੰਸਟਾਲੇਸ਼ਨ ਵਿਚਕਾਰ ਕਈ ਸਪੱਸ਼ਟ ਫਰਕ ਹਨ ਜਿਨ੍ਹਾਂ ਬਾਰੇ ਅਸੀਂ ਸੰਖੇਪ ਵਿੱਚ ਗੱਲ ਕਰਾਂਗੇ।

## Database

Database ਡੇਟਾ ਦਾ ਇੱਕ ਸੰਗਠਿਤ ਅਤੇ ਵਿਵਸਥਿਤ ਸੈੱਟ ਹੈ। ਕੰਪਿਊਟਿੰਗ ਦੀ ਭਾਸ਼ਾ ਵਿੱਚ, database ਉਸ ਸਾਫਟਵੇਅਰ ਨੂੰ ਕਹਿੰਦੇ ਹਨ ਜੋ ਡੇਟਾ ਨੂੰ ਸਟੋਰ ਅਤੇ ਸੰਗਠਿਤ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ। ਇਸਨੂੰ ਇੱਕ ਫਾਈਲ ਕੈਬਿਨੇਟ ਸਮਝੋ ਜਿੱਥੇ ਤੁਸੀਂ ਡੇਟਾ ਨੂੰ ਵੱਖ-ਵੱਖ ਹਿੱਸਿਆਂ ਵਿੱਚ ਸਟੋਰ ਕਰਦੇ ਹੋ ਜਿਨ੍ਹਾਂ ਨੂੰ tables ਕਹਿੰਦੇ ਹਨ।

WordPress Multisite ਇੱਕ database ਵਰਤਦਾ ਹੈ ਅਤੇ ਹਰ ਸਬਸਾਈਟ ਨੂੰ prefix ਵਿੱਚ blog id ਨਾਲ ਆਪਣੀਆਂ tables ਮਿਲਦੀਆਂ ਹਨ, ਇਸ ਲਈ ਜਦੋਂ ਤੁਸੀਂ ਨੈੱਟਵਰਕ ਇੰਸਟਾਲੇਸ਼ਨ ਆਪਣਾ database ਬਣਾਉਂਦੇ ਹੋ ਅਤੇ ਸਬਸਾਈਟ ਬਣਾਉਂਦੇ ਹੋ, ਤੁਹਾਡੇ ਕੋਲ ਇਹ tables ਹੋਣੀਆਂ ਚਾਹੀਦੀਆਂ ਹਨ:

_wp_1_options_ \- ਪਹਿਲੀ ਸਬਸਾਈਟ ਲਈ options table

_wp_2_options_ \- ਦੂਜੀ ਸਬਸਾਈਟ ਲਈ options table

## Hosting provider

Hosting provider ਇੱਕ ਕੰਪਨੀ ਹੈ ਜੋ ਕਾਰੋਬਾਰਾਂ ਅਤੇ ਵਿਅਕਤੀਆਂ ਨੂੰ ਆਪਣੀਆਂ ਵੈੱਬਸਾਈਟਾਂ World Wide Web ਰਾਹੀਂ ਉਪਲਬਧ ਕਰਵਾਉਣ ਦੀ ਸਹੂਲਤ ਦਿੰਦੀ ਹੈ। Web hosting providers ਜੋ ਸੇਵਾਵਾਂ ਦਿੰਦੇ ਹਨ ਉਹ ਵੱਖ-ਵੱਖ ਹੁੰਦੀਆਂ ਹਨ ਪਰ ਆਮ ਤੌਰ 'ਤੇ ਇਸ ਵਿੱਚ ਵੈੱਬਸਾਈਟ ਡਿਜ਼ਾਈਨ, host 'ਤੇ ਸਟੋਰੇਜ ਸਪੇਸ, ਅਤੇ ਇੰਟਰਨੈੱਟ ਨਾਲ ਕਨੈਕਟੀਵਿਟੀ ਸ਼ਾਮਲ ਹੁੰਦੀ ਹੈ।

## Domain

Domain name ਉਹ ਪਤਾ ਹੈ ਜੋ ਲੋਕ ਤੁਹਾਡੀ ਸਾਈਟ 'ਤੇ ਜਾਣ ਲਈ ਵਰਤਦੇ ਹਨ। ਇਹ ਵੈੱਬ ਬ੍ਰਾਊਜ਼ਰ ਨੂੰ ਦੱਸਦਾ ਹੈ ਕਿ ਤੁਹਾਡੀ ਸਾਈਟ ਕਿੱਥੇ ਲੱਭਣੀ ਹੈ। ਜਿਵੇਂ ਗਲੀ ਦਾ ਪਤਾ ਹੁੰਦਾ ਹੈ, ਉਵੇਂ ਹੀ domain ਉਹ ਤਰੀਕਾ ਹੈ ਜਿਸ ਨਾਲ ਲੋਕ ਤੁਹਾਡੀ ਵੈੱਬਸਾਈਟ 'ਤੇ ਆਨਲਾਈਨ ਆਉਂਦੇ ਹਨ। ਅਤੇ, ਜਿਵੇਂ ਤੁਹਾਡੀ ਦੁਕਾਨ ਅੱਗੇ ਬੋਰਡ ਲੱਗਾ ਹੁੰਦਾ ਹੈ। ਜੇਕਰ ਤੁਸੀਂ ਸਾਡੀ ਵੈੱਬਸਾਈਟ 'ਤੇ ਆਉਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤੁਹਾਨੂੰ ਆਪਣੇ ਬ੍ਰਾਊਜ਼ਰ ਦੇ ਐਡਰੈੱਸ ਬਾਰ ਵਿੱਚ ਸਾਡਾ ਵੈੱਬ ਐਡਰੈੱਸ ਟਾਈਪ ਕਰਨਾ ਹੋਵੇਗਾ ਜੋ ਹੈ [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) _,_ ਜਿੱਥੇ [**ultimatemultisite.com**](http://ultimatemultisite.com) domain name ਹੈ।

## Subdomain

Subdomain ਮੁੱਖ domain ਦੇ ਹੇਠਾਂ ਇੱਕ ਕਿਸਮ ਦੀ ਵੈੱਬਸਾਈਟ ਲੜੀ ਹੈ, ਪਰ ਵੈੱਬਸਾਈਟ 'ਤੇ ਸਮੱਗਰੀ ਨੂੰ ਫੋਲਡਰਾਂ ਨਾਲ ਸੰਗਠਿਤ ਕਰਨ ਦੀ ਬਜਾਏ, ਇਸਨੂੰ ਆਪਣੀ ਵੱਖਰੀ ਵੈੱਬਸਾਈਟ ਮਿਲ ਜਾਂਦੀ ਹੈ। ਇਹ [**https://site1.domain.com/**](https://site1.domain.com/) ਵਜੋਂ ਦਿਖਾਈ ਦਿੰਦਾ ਹੈ ਜਿੱਥੇ _site1_ subdomain name ਹੈ ਅਤੇ [_domain.com_](http://domain.com) ਮੁੱਖ domain ਹੈ।

## Subdirectory

Subdirectory ਇੱਕ root domain ਦੇ ਹੇਠਾਂ ਇੱਕ ਕਿਸਮ ਦੀ ਵੈੱਬਸਾਈਟ ਲੜੀ ਹੈ ਜੋ ਵੈੱਬਸਾਈਟ 'ਤੇ ਸਮੱਗਰੀ ਨੂੰ ਸੰਗਠਿਤ ਕਰਨ ਲਈ ਫੋਲਡਰਾਂ ਦੀ ਵਰਤੋਂ ਕਰਦੀ ਹੈ। Subdirectory ਅਤੇ subfolder ਇੱਕੋ ਚੀਜ਼ ਹੈ ਅਤੇ ਦੋਵੇਂ ਨਾਂ ਇੱਕ ਦੂਜੇ ਦੀ ਥਾਂ ਵਰਤੇ ਜਾ ਸਕਦੇ ਹਨ। ਇਹ [**https://domain.com/site1**](https://domain.com/site1) ਵਜੋਂ ਦਿਖਾਈ ਦਿੰਦਾ ਹੈ ਜਿੱਥੇ _site1_ subdirectory name ਹੈ ਅਤੇ [_domain.com_](http://domain.com) ਮੁੱਖ domain ਹੈ।

## Subsite

Subsite ਇੱਕ ਚਾਈਲਡ ਸਾਈਟ ਹੈ ਜੋ ਤੁਸੀਂ Multisite ਨੈੱਟਵਰਕ 'ਤੇ ਬਣਾਉਂਦੇ ਹੋ। ਇਹ ਤੁਹਾਡੀ WordPress Multisite ਇੰਸਟਾਲੇਸ਼ਨ ਦੀ ਸੈਟਿੰਗ ਦੇ ਆਧਾਰ 'ਤੇ **subdomain** ਜਾਂ **subdirectory** ਹੋ ਸਕਦੀ ਹੈ।

## Super Admin

WordPress Super Admin ਇੱਕ ਵਰਤੋਂਕਾਰ ਭੂਮਿਕਾ ਹੈ ਜਿਸ ਕੋਲ Multisite ਨੈੱਟਵਰਕ 'ਤੇ ਸਾਰੀਆਂ ਸਬਸਾਈਟਾਂ ਨੂੰ ਸੰਭਾਲਣ ਦੀਆਂ ਪੂਰੀਆਂ ਸ਼ਕਤੀਆਂ ਹੁੰਦੀਆਂ ਹਨ। Multisite ਵਰਤੋਂਕਾਰਾਂ ਲਈ, ਇਹ ਉਹ **ਸਭ ਤੋਂ ਉੱਚਾ ਪੱਧਰ ਦੀ ਪਹੁੰਚ** ਹੈ ਜੋ ਤੁਸੀਂ ਆਪਣੀ WordPress ਇੰਸਟਾਲੇਸ਼ਨ ਨੂੰ ਦੇ ਸਕਦੇ ਹੋ।

## Plugin

ਆਮ ਤੌਰ 'ਤੇ, plugin ਕੋਡ ਦਾ ਇੱਕ ਸੈੱਟ ਹੈ ਜੋ ਤੁਹਾਡੀ WordPress ਸਾਈਟ ਵਿੱਚ ਵਾਧੂ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਜੋੜਦਾ ਹੈ। ਇਹ ਲੌਗਿਨ ਲੋਗੋ ਬਦਲਣ ਜਿੰਨਾ ਸਧਾਰਨ ਜਾਂ ਈ-ਕਾਮਰਸ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਜੋੜਨ ਜਿੰਨਾ ਗੁੰਝਲਦਾਰ ਹੋ ਸਕਦਾ ਹੈ। _Woocommerce ਅਤੇ Contact Form_ plugin ਦੀਆਂ ਉਦਾਹਰਨਾਂ ਹਨ।

WordPress Multisite 'ਤੇ, plugins ਸਿਰਫ਼ Super Admin ਦੁਆਰਾ network admin dashboard ਤੋਂ ਹੀ ਇੰਸਟਾਲ ਕੀਤੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ। Subsite Admins ਸਿਰਫ਼ ਆਪਣੀ ਸਬਸਾਈਟ ਦੇ ਅੰਦਰ plugins ਨੂੰ ਐਕਟੀਵੇਟ ਅਤੇ ਡੀਐਕਟੀਵੇਟ ਕਰ ਸਕਦੇ ਹਨ।

## Themes

WordPress theme ਫਾਈਲਾਂ ਦਾ ਇੱਕ ਸਮੂਹ ਹੈ (_ਗ੍ਰਾਫਿਕਸ, ਸਟਾਈਲ ਸ਼ੀਟਾਂ, ਅਤੇ ਕੋਡ_) ਜੋ ਸਾਈਟ ਦੀ ਸਮੁੱਚੀ ਦਿੱਖ ਨੂੰ ਨਿਰਧਾਰਤ ਕਰਦਾ ਹੈ। ਇਹ ਸਾਰੀ front-end ਸਟਾਈਲਿੰਗ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ ਜਿਵੇਂ ਫੌਂਟ ਸਟਾਈਲਿੰਗ, ਪੇਜ ਲੇਆਊਟ, ਰੰਗ, ਆਦਿ।

Plugins ਵਾਂਗ ਹੀ, WordPress Multisite ਵਿੱਚ themes ਸਿਰਫ਼ Super Admin ਦੁਆਰਾ ਇੰਸਟਾਲ ਕੀਤੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ ਅਤੇ ਸਬਸਾਈਟ admins ਦੁਆਰਾ ਸਬਸਾਈਟ ਪੱਧਰ 'ਤੇ ਐਕਟੀਵੇਟ ਕੀਤੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ।

## Site Template

**Site Template** ਇੱਕ ਬੁਨਿਆਦੀ ਸਾਈਟ ਹੈ ਜੋ ਤੁਹਾਡੇ ਨੈੱਟਵਰਕ ਵਿੱਚ ਨਵੀਆਂ ਸਾਈਟਾਂ ਬਣਾਉਣ ਵੇਲੇ ਅਧਾਰ ਵਜੋਂ ਵਰਤੀ ਜਾ ਸਕਦੀ ਹੈ।

ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਤੁਸੀਂ ਇੱਕ ਬੇਸ ਸਾਈਟ ਬਣਾ ਸਕਦੇ ਹੋ, ਵੱਖ-ਵੱਖ plugins ਐਕਟੀਵੇਟ ਕਰ ਸਕਦੇ ਹੋ, ਇੱਕ ਐਕਟਿਵ theme ਸੈੱਟ ਕਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਇਸਨੂੰ ਆਪਣੀ ਮਰਜ਼ੀ ਨਾਲ ਕਸਟਮਾਈਜ਼ ਕਰ ਸਕਦੇ ਹੋ। ਫਿਰ, ਜਦੋਂ ਤੁਹਾਡਾ ਗਾਹਕ ਨਵਾਂ ਖਾਤਾ ਬਣਾਉਂਦਾ ਹੈ, ਤਾਂ ਬਿਨਾਂ ਕਿਸੇ ਅਰਥਪੂਰਨ ਸਮੱਗਰੀ ਵਾਲੀ ਡਿਫਾਲਟ WordPress ਸਾਈਟ ਮਿਲਣ ਦੀ ਬਜਾਏ, ਉਹਨਾਂ ਨੂੰ ਤੁਹਾਡੀ ਬੇਸ ਸਾਈਟ ਦੀ ਕਾਪੀ ਮਿਲੇਗੀ ਜਿਸ ਵਿੱਚ ਸਾਰੀਆਂ ਕਸਟਮਾਈਜ਼ੇਸ਼ਨਾਂ ਅਤੇ ਸਮੱਗਰੀ ਪਹਿਲਾਂ ਤੋਂ ਹੀ ਮੌਜੂਦ ਹੋਵੇਗੀ।

## Domain Mapping

WordPress ਨਾਲ **Domain mapping** ਵਰਤੋਂਕਾਰਾਂ ਨੂੰ ਵੈੱਬਸਾਈਟ ਦੇ ਐਡਰੈੱਸ ਰਾਹੀਂ ਸਹੀ host 'ਤੇ ਰੀਡਾਇਰੈਕਟ ਕਰਨ ਦਾ ਇੱਕ ਤਰੀਕਾ ਹੈ। WordPress Multisite ਵਿੱਚ, ਸਬਸਾਈਟਾਂ subdirectory ਜਾਂ subdomain ਵਰਤ ਕੇ ਬਣਾਈਆਂ ਜਾਂਦੀਆਂ ਹਨ। Domain mapping ਜੋ ਕਰਦਾ ਹੈ ਉਹ ਇਹ ਹੈ ਕਿ ਇਹ ਸਬਸਾਈਟ ਵਰਤੋਂਕਾਰਾਂ ਨੂੰ [**joesbikeshop.com**](http://joesbikeshop.com) ਵਰਗਾ top-level domain ਵਰਤਣ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ ਤਾਂ ਜੋ ਉਹਨਾਂ ਦੀ ਸਾਈਟ ਦਾ ਐਡਰੈੱਸ ਵਧੇਰੇ ਪੇਸ਼ੇਵਰ ਲੱਗੇ।

## SSL

SSL ਦਾ ਮਤਲਬ ਹੈ **Secure Sockets Layer**। ਇਹ ਇੱਕ ਡਿਜੀਟਲ ਸਰਟੀਫਿਕੇਟ ਹੈ ਜੋ ਵੈੱਬ ਸਾਈਟ ਦੀ ਪਛਾਣ ਪ੍ਰਮਾਣਿਤ ਕਰਦਾ ਹੈ ਅਤੇ ਇੱਕ ਐਨਕ੍ਰਿਪਟਿਡ ਕਨੈਕਸ਼ਨ ਯੋਗ ਕਰਦਾ ਹੈ। ਅੱਜਕੱਲ੍ਹ ਇਹ ਇੰਟਰਨੈੱਟ ਕਨੈਕਸ਼ਨ ਨੂੰ ਸੁਰੱਖਿਅਤ ਰੱਖਣ ਅਤੇ ਦੋ ਸਿਸਟਮਾਂ ਵਿਚਕਾਰ ਭੇਜੇ ਜਾ ਰਹੇ ਕਿਸੇ ਵੀ ਸੰਵੇਦਨਸ਼ੀਲ ਡੇਟਾ ਦੀ ਸੁਰੱਖਿਆ ਲਈ ਮਿਆਰੀ ਤਕਨੀਕ ਵਜੋਂ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਜੋ ਅਪਰਾਧੀਆਂ ਨੂੰ ਟ੍ਰਾਂਸਫਰ ਕੀਤੀ ਕੋਈ ਵੀ ਜਾਣਕਾਰੀ ਪੜ੍ਹਨ ਅਤੇ ਬਦਲਣ ਤੋਂ ਰੋਕਦਾ ਹੈ, ਜਿਸ ਵਿੱਚ ਸੰਭਾਵੀ ਨਿੱਜੀ ਵੇਰਵੇ ਸ਼ਾਮਲ ਹਨ। ਆਧੁਨਿਕ ਬ੍ਰਾਊਜ਼ਰਾਂ ਲਈ SSL ਜ਼ਰੂਰੀ ਹੈ ਜੋ ਇਸਨੂੰ ਵੈੱਬਸਾਈਟ ਬਣਾਉਣ ਅਤੇ ਚਲਾਉਣ ਵੇਲੇ ਲਾਜ਼ਮੀ ਬਣਾਉਂਦਾ ਹੈ।

## Media

Media ਉਹ ਚਿੱਤਰ, ਆਡੀਓ, ਵੀਡੀਓ, ਅਤੇ ਹੋਰ ਫਾਈਲਾਂ ਹਨ ਜੋ ਵੈੱਬਸਾਈਟ ਬਣਾਉਂਦੀਆਂ ਹਨ।

WordPress Multisite ਵਿੱਚ ਨੈੱਟਵਰਕ ਸਾਈਟਾਂ ਇੱਕ ਹੀ database ਸਾਂਝਾ ਕਰਦੀਆਂ ਹਨ, ਪਰ ਉਹ media ਫਾਈਲਾਂ ਲਈ ਫਾਈਲਸਿਸਟਮ 'ਤੇ ਵੱਖਰੇ ਪਾਥ ਰੱਖਦੀਆਂ ਹਨ।

ਮਿਆਰੀ WordPress ਲੋਕੇਸ਼ਨ (wp-content/uploads) ਉਹੀ ਰਹਿੰਦੀ ਹੈ; ਹਾਲਾਂਕਿ, ਇਸਦਾ ਪਾਥ ਨੈੱਟਵਰਕ ਸਾਈਟ ਦੀ ਵਿਲੱਖਣ ID ਦਿਖਾਉਣ ਲਈ ਬਦਲ ਦਿੱਤਾ ਜਾਂਦਾ ਹੈ। ਨਤੀਜੇ ਵਜੋਂ ਨੈੱਟਵਰਕ ਸਾਈਟ ਲਈ media ਫਾਈਲਾਂ wp-contents/uploads/site/[id] ਵਜੋਂ ਦਿਖਾਈ ਦਿੰਦੀਆਂ ਹਨ।

## Permalinks

Permalinks ਤੁਹਾਡੀ ਸਾਈਟ ਦੇ ਅੰਦਰ ਤੁਹਾਡੀਆਂ ਵਿਅਕਤੀਗਤ ਬਲੌਗ ਪੋਸਟਾਂ ਜਾਂ ਪੇਜਾਂ ਦੇ ਸਥਾਈ URLs ਹਨ। Permalinks ਨੂੰ **pretty links** ਵੀ ਕਿਹਾ ਜਾਂਦਾ ਹੈ। ਡਿਫਾਲਟ ਤੌਰ 'ਤੇ, WordPress URLs query string ਫਾਰਮੈਟ ਵਰਤਦੇ ਹਨ ਜੋ ਕੁਝ ਇਸ ਤਰ੍ਹਾਂ ਦਿਖਾਈ ਦਿੰਦਾ ਹੈ:

[**http://www.example.com/registration**](http://www.example.com/registration)

## Ultimate Multisite

Ultimate Multisite ਇੱਕ WordPress plugin ਹੈ, ਜੋ WordPress Multisite ਇੰਸਟਾਲਾਂ ਲਈ ਬਣਾਈ ਗਈ ਹੈ, ਜੋ ਤੁਹਾਡੀ WordPress ਇੰਸਟਾਲ ਨੂੰ ਸਾਈਟਾਂ ਦੇ ਪ੍ਰੀਮੀਅਮ ਨੈੱਟਵਰਕ ਵਿੱਚ ਬਦਲ ਦਿੰਦੀ ਹੈ – ਜਿਵੇਂ [WordPress.com](https://WordPress.com) – ਜੋ ਗਾਹਕਾਂ ਨੂੰ ਮਹੀਨਾਵਾਰ, ਤਿਮਾਹੀ, ਜਾਂ ਸਾਲਾਨਾ ਫੀਸਾਂ ਰਾਹੀਂ ਸਾਈਟਾਂ ਬਣਾਉਣ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ (ਤੁਸੀਂ ਮੁਫ਼ਤ ਪਲਾਨ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ)।

## Checkout Form

Checkout Form ਇੱਕ ਸਿੰਗਲ ਜਾਂ ਮਲਟੀ-ਸਟੈਪ ਆਰਡਰ ਫਾਰਮ ਹੈ ਜਿਸ ਵਿੱਚ Ultimate Multisite ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਰਾਹੀਂ ਸਬਸਾਈਟ, ਮੈਂਬਰਸ਼ਿਪ, ਅਤੇ ਵਰਤੋਂਕਾਰ ਖਾਤੇ ਬਣਾਉਣੇ ਸ਼ਾਮਲ ਹਨ। ਇਸ ਵਿੱਚ ਵੱਖ-ਵੱਖ ਫੀਲਡਾਂ ਅਤੇ ਭੁਗਤਾਨ ਫਾਰਮ ਹੁੰਦੇ ਹਨ ਜੋ ਵਰਤੋਂਕਾਰ ਨੂੰ ਸਾਈਨ-ਅੱਪ ਪ੍ਰਕਿਰਿਆ ਦੌਰਾਨ ਜਮ੍ਹਾਂ ਕਰਨੇ ਪੈਂਦੇ ਹਨ।

## Webhook

Webhook (ਜਿਸਨੂੰ web callback ਜਾਂ HTTP push API ਵੀ ਕਿਹਾ ਜਾਂਦਾ ਹੈ) ਇੱਕ ਐਪ ਲਈ ਹੋਰ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਰੀਅਲ-ਟਾਈਮ ਜਾਣਕਾਰੀ ਪ੍ਰਦਾਨ ਕਰਨ ਦਾ ਤਰੀਕਾ ਹੈ। Webhook ਜਿਵੇਂ ਹੀ ਡੇਟਾ ਆਉਂਦਾ ਹੈ ਉਸੇ ਵੇਲੇ ਹੋਰ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਡਿਲੀਵਰ ਕਰ ਦਿੰਦਾ ਹੈ, ਮਤਲਬ ਤੁਹਾਨੂੰ ਡੇਟਾ ਤੁਰੰਤ ਮਿਲ ਜਾਂਦਾ ਹੈ।

**Ultimate Multisite webhooks** ਅਨੰਤ ਸੰਭਾਵਨਾਵਾਂ ਖੋਲ੍ਹਦੇ ਹਨ, ਜੋ ਨੈੱਟਵਰਕ admins ਨੂੰ ਹਰ ਤਰ੍ਹਾਂ ਦੇ ਅਜੀਬ-ਪਰ-ਉਪਯੋਗੀ integrations ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ, ਖਾਸ ਕਰਕੇ ਜੇ _Zapier ਅਤੇ IFTTT_ ਵਰਗੀਆਂ ਸੇਵਾਵਾਂ ਨਾਲ ਵਰਤੇ ਜਾਣ।

## Events

Event ਇੱਕ ਕਾਰਵਾਈ ਹੈ ਜੋ ਵਰਤੋਂਕਾਰ ਜਾਂ ਕਿਸੇ ਹੋਰ ਸਰੋਤ ਦੀ ਕਾਰਵਾਈ ਦੇ ਨਤੀਜੇ ਵਜੋਂ ਹੁੰਦੀ ਹੈ, ਜਿਵੇਂ ਮਾਊਸ ਕਲਿੱਕ। Ultimate Multisite ਤੁਹਾਡੇ ਪੂਰੇ ਨੈੱਟਵਰਕ ਵਿੱਚ ਹੋ ਰਹੀਆਂ ਸਾਰੀਆਂ events ਅਤੇ logs ਦਾ ਰਿਕਾਰਡ ਰੱਖਦਾ ਹੈ। ਇਹ ਤੁਹਾਡੀ ਮਲਟੀਸਾਈਟ ਵਿੱਚ ਹੋ ਰਹੀਆਂ ਵੱਖ-ਵੱਖ ਗਤੀਵਿਧੀਆਂ ਨੂੰ ਟਰੈਕ ਕਰਦਾ ਹੈ, ਜਿਵੇਂ ਪਲਾਨ ਵਿੱਚ ਤਬਦੀਲੀਆਂ।
