# ਈਮੇਲਾਂ ਅਤੇ ਬ੍ਰਾਡਕਾਸਟ ਭੇਜਣਾ (v2)

_**ਮਹੱਤਵਪੂਰਨ ਨੋਟ: ਇਹ ਲੇਖ Ultimate Multisite ਵਰਜ਼ਨ 2.x ਬਾਰੇ ਹੈ।**_

Ultimate Multisite ਵਿੱਚ ਇੱਕ ਵਿਸ਼ੇਸ਼ਤਾ ਹੈ ਜੋ ਤੁਹਾਨੂੰ ਆਪਣੇ ਗਾਹਕਾਂ ਨਾਲ ਸੰਪਰਕ ਕਰਨ ਦੀ ਇਜਾਜ਼ਤ ਦਿੰਦੀ ਹੈ - ਤੁਸੀਂ ਕਿਸੇ ਖਾਸ ਯੂਜ਼ਰ ਜਾਂ ਯੂਜ਼ਰਾਂ ਦੇ ਸਮੂਹ ਨੂੰ ਈਮੇਲ ਭੇਜ ਸਕਦੇ ਹੋ, ਅਤੇ ਨਾਲ ਹੀ ਉਨ੍ਹਾਂ ਦੇ admin dashboard 'ਤੇ ਨੋਟਿਸ ਭੇਜ ਕੇ ਐਲਾਨ ਕਰ ਸਕਦੇ ਹੋ

## Broadcasts ਨਾਲ ਆਪਣੇ ਗਾਹਕਾਂ ਦੇ dashboard 'ਤੇ admin ਨੋਟਿਸ ਜੋੜੋ

Ultimate Multisite ਦੀ broadcast ਵਿਸ਼ੇਸ਼ਤਾ ਵਰਤ ਕੇ, ਤੁਸੀਂ ਆਪਣੇ ਯੂਜ਼ਰਾਂ ਦੇ subsite admin dashboard 'ਤੇ **admin ਨੋਟਿਸ** ਜੋੜ ਸਕਦੇ ਹੋ।

ਇਹ ਬਹੁਤ ਫਾਇਦੇਮੰਦ ਹੈ ਜੇਕਰ ਤੁਹਾਨੂੰ ਕੋਈ ਐਲਾਨ ਕਰਨਾ ਹੋਵੇ ਜਿਵੇਂ ਕਿ ਸਿਸਟਮ ਮੇਨਟੇਨੈਂਸ ਜਾਂ ਆਪਣੇ ਮੌਜੂਦਾ ਯੂਜ਼ਰਾਂ ਨੂੰ ਨਵੇਂ ਉਤਪਾਦ ਜਾਂ ਸੇਵਾਵਾਂ ਦੀ ਪੇਸ਼ਕਸ਼ ਕਰਨੀ ਹੋਵੇ। ਤੁਹਾਡੇ ਯੂਜ਼ਰ ਦੇ dashboard 'ਤੇ admin ਨੋਟਿਸ ਇਸ ਤਰ੍ਹਾਂ ਦਿਖਾਈ ਦੇਵੇਗਾ।

![ਗਾਹਕ dashboard 'ਤੇ ਦਿਖਾਇਆ ਗਿਆ Admin ਨੋਟਿਸ broadcast](/img/admin/broadcasts-list.png)

admin ਨੋਟਿਸ ਸ਼ੁਰੂ ਕਰਨ ਲਈ, ਆਪਣੇ network admin dashboard 'ਤੇ ਜਾਓ ਅਤੇ **Ultimate Multisite** ਮੀਨੂ ਦੇ ਹੇਠਾਂ, ਤੁਹਾਨੂੰ **Broadcasts** ਵਿਕਲਪ ਮਿਲੇਗਾ।

![Ultimate Multisite admin ਵਿੱਚ Broadcasts ਮੀਨੂ](/img/admin/broadcasts-list.png)

ਇਸ ਪੇਜ ਤੋਂ, ਉੱਪਰ **Add Broadcast** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ।

ਇਹ Add broadcast ਮੋਡਲ ਵਿੰਡੋ ਖੋਲ੍ਹੇਗਾ ਜਿੱਥੇ ਤੁਸੀਂ ਚੁਣ ਸਕਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕਿਸ ਕਿਸਮ ਦਾ broadcast ਭੇਜਣਾ ਚਾਹੁੰਦੇ ਹੋ।

ਅੱਗੇ ਵਧੋ ਅਤੇ **Message** ਚੁਣੋ ਫਿਰ **Next Step** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ।

![Message ਕਿਸਮ ਚੁਣੀ ਹੋਈ Add broadcast ਮੋਡਲ](/img/admin/broadcasts-list.png)

ਅਗਲੀ ਵਿੰਡੋ ਤੁਹਾਨੂੰ **Target customer** ਜਾਂ **Target product** ਬਾਰੇ ਪੁੱਛੇਗੀ। ਯਾਦ ਰੱਖੋ ਕਿ ਤੁਸੀਂ ਇੱਕ ਤੋਂ ਵੱਧ ਯੂਜ਼ਰ ਜਾਂ ਇੱਕ ਤੋਂ ਵੱਧ ਉਤਪਾਦ ਚੁਣ ਸਕਦੇ ਹੋ।

ਕਿਸੇ ਯੂਜ਼ਰ ਖਾਤੇ ਜਾਂ ਉਤਪਾਦ ਨੂੰ ਲੱਭਣ ਲਈ, ਤੁਹਾਨੂੰ ਫੀਲਡ ਵਿੱਚ ਕੀਵਰਡ ਟਾਈਪ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰਨਾ ਹੋਵੇਗਾ।

**Message type** ਫੀਲਡ ਦੇ ਹੇਠਾਂ, ਤੁਸੀਂ ਨੋਟਿਸ ਦਾ ਰੰਗ ਚੁਣ ਸਕਦੇ ਹੋ। ਇਹ ਤੁਹਾਡੇ ਸੁਨੇਹੇ ਦੀ ਜ਼ਰੂਰਤ 'ਤੇ ਜ਼ੋਰ ਦੇਵੇਗਾ।

ਫਿਰ ਤੁਸੀਂ **Next Step** 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ।

![Broadcast ਲਈ Target customer ਅਤੇ product ਚੋਣ](/img/admin/broadcasts-list.png)

ਅਗਲੀ ਵਿੰਡੋ ਵਿੱਚ ਤੁਸੀਂ ਵਿਸ਼ਾ ਅਤੇ ਸਮੱਗਰੀ/ਸੁਨੇਹਾ ਦਾਖਲ ਕਰਕੇ ਆਪਣਾ ਸੁਨੇਹਾ ਲਿਖਣਾ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਤੁਸੀਂ ਯੂਜ਼ਰਾਂ ਨੂੰ broadcast ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।

![Broadcast ਸੁਨੇਹੇ ਦਾ ਵਿਸ਼ਾ ਅਤੇ ਸਮੱਗਰੀ ਸੰਪਾਦਕ](/img/admin/broadcasts-list.png)

ਆਪਣਾ ਸੁਨੇਹਾ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ **Send** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ।

ਬੱਸ ਇਹੀ ਹੈ। admin ਨੋਟਿਸ ਤੁਰੰਤ ਤੁਹਾਡੇ ਯੂਜ਼ਰ ਦੇ dashboard 'ਤੇ ਦਿਖਾਈ ਦੇਣਾ ਚਾਹੀਦਾ ਹੈ।

## ਆਪਣੇ ਗਾਹਕਾਂ ਨੂੰ ਈਮੇਲਾਂ ਭੇਜੋ

Ultimate Multisite ਦੀ broadcast ਵਿਸ਼ੇਸ਼ਤਾ ਵਰਤ ਕੇ, ਤੁਸੀਂ ਆਪਣੇ ਯੂਜ਼ਰਾਂ ਨੂੰ ਈਮੇਲ ਭੇਜ ਸਕਦੇ ਹੋ। ਤੁਹਾਡੇ ਕੋਲ ਸਿਰਫ਼ ਖਾਸ ਯੂਜ਼ਰਾਂ ਨੂੰ ਈਮੇਲ ਭੇਜਣ ਦਾ ਵਿਕਲਪ ਹੈ ਜਾਂ ਉਸ ਉਤਪਾਦ ਜਾਂ ਯੋਜਨਾ ਦੇ ਆਧਾਰ 'ਤੇ ਖਾਸ ਯੂਜ਼ਰ ਸਮੂਹ ਨੂੰ ਨਿਸ਼ਾਨਾ ਬਣਾਉਣ ਦਾ ਜਿਸ ਦੇ ਹੇਠਾਂ ਉਹ ਸਬਸਕ੍ਰਾਈਬ ਹਨ।

ਈਮੇਲ broadcast ਸ਼ੁਰੂ ਕਰਨ ਲਈ, ਆਪਣੇ network admin dashboard 'ਤੇ ਜਾਓ ਅਤੇ Ultimate Multisite ਮੀਨੂ ਦੇ ਹੇਠਾਂ, ਤੁਹਾਨੂੰ Broadcast ਵਿਕਲਪ ਮਿਲੇਗਾ।

![Ultimate Multisite admin ਵਿੱਚ Broadcasts ਪੇਜ](/img/admin/broadcasts-list.png)

ਇਸ ਪੇਜ ਤੋਂ, ਉੱਪਰ **Add broadcast** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ।

ਇਹ Add broadcast ਮੋਡਲ ਵਿੰਡੋ ਖੋਲ੍ਹੇਗਾ ਜਿੱਥੇ ਤੁਸੀਂ ਚੁਣ ਸਕਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕਿਸ ਕਿਸਮ ਦਾ broadcast ਭੇਜਣਾ ਚਾਹੁੰਦੇ ਹੋ। ਅੱਗੇ ਵਧੋ ਅਤੇ **Email** ਚੁਣੋ ਫਿਰ **Next Step** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ।

![Email ਕਿਸਮ ਚੁਣੀ ਹੋਈ Add broadcast ਮੋਡਲ](/img/admin/broadcasts-list.png)

ਅਗਲੀ ਵਿੰਡੋ ਤੁਹਾਨੂੰ **Target customer** ਜਾਂ **Target product** ਬਾਰੇ ਪੁੱਛੇਗੀ। ਯਾਦ ਰੱਖੋ ਕਿ ਤੁਸੀਂ ਇੱਕ ਤੋਂ ਵੱਧ ਯੂਜ਼ਰ ਜਾਂ ਇੱਕ ਤੋਂ ਵੱਧ ਉਤਪਾਦ ਚੁਣ ਸਕਦੇ ਹੋ।

ਕਿਸੇ ਯੂਜ਼ਰ ਖਾਤੇ ਜਾਂ ਉਤਪਾਦ ਨੂੰ ਲੱਭਣ ਲਈ, ਤੁਹਾਨੂੰ ਫੀਲਡ ਵਿੱਚ ਕੀਵਰਡ ਟਾਈਪ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰਨਾ ਹੋਵੇਗਾ।

ਇੱਕ ਵਾਰ ਤੁਹਾਡੀ ਨਿਸ਼ਾਨਾ ਔਡੀਐਂਸ ਚੁਣੀ ਜਾਂਦੀ ਹੈ, ਤੁਸੀਂ **Next Step** 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ।

![Email broadcast ਲਈ Target customer ਅਤੇ product ਚੋਣ](/img/admin/broadcasts-list.png)

ਅਗਲੀ ਵਿੰਡੋ ਵਿੱਚ ਤੁਸੀਂ ਵਿਸ਼ਾ ਅਤੇ ਸਮੱਗਰੀ/ਸੁਨੇਹਾ ਦਾਖਲ ਕਰਕੇ ਆਪਣੀ ਈਮੇਲ ਲਿਖਣੀ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਤੁਸੀਂ ਯੂਜ਼ਰਾਂ ਨੂੰ ਭੇਜਣਾ ਚਾਹੁੰਦੇ ਹੋ।

![Email broadcast ਵਿਸ਼ਾ ਅਤੇ ਸਮੱਗਰੀ ਸੰਪਾਦਕ](/img/admin/broadcasts-list.png)

ਆਪਣਾ ਸੁਨੇਹਾ ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ **Send** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ।

ਅਤੇ broadcast ਵਿਸ਼ੇਸ਼ਤਾ ਵਰਤ ਕੇ ਆਪਣੇ ਅੰਤਿਮ-ਯੂਜ਼ਰਾਂ ਨੂੰ ਈਮੇਲ ਭੇਜਣਾ ਇੰਨਾ ਆਸਾਨ ਹੈ।

## ਸਿਸਟਮ ਈਮੇਲਾਂ

Ultimate Multisite ਵਿੱਚ ਸਿਸਟਮ ਈਮੇਲਾਂ ਉਹ **ਆਟੋਮੈਟਿਕ ਸੂਚਨਾਵਾਂ** ਹਨ ਜੋ ਸਿਸਟਮ ਦੁਆਰਾ ਕੁਝ ਕਾਰਵਾਈਆਂ ਜਿਵੇਂ ਕਿ ਰਜਿਸਟ੍ਰੇਸ਼ਨ, ਭੁਗਤਾਨ, domain mapping, ਆਦਿ ਤੋਂ ਬਾਅਦ ਭੇਜੀਆਂ ਜਾਂਦੀਆਂ ਹਨ। ਇਹ ਈਮੇਲਾਂ Ultimate Multisite ਸੈਟਿੰਗਾਂ ਤੋਂ ਸੰਪਾਦਿਤ ਜਾਂ ਸੋਧੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ। ਇਸ ਵਿੱਚ ਇੱਕ ਵਿਸ਼ੇਸ਼ਤਾ ਵੀ ਹੈ ਜੋ ਤੁਹਾਨੂੰ ਰੀਸੈੱਟ ਕਰਨ ਅਤੇ ਕਿਸੇ ਹੋਰ Ultimate Multisite ਇੰਸਟਾਲੇਸ਼ਨ ਤੋਂ ਮੌਜੂਦਾ ਸੈਟਿੰਗਾਂ ਨੂੰ ਇੰਪੋਰਟ ਕਰਨ ਦੀ ਇਜਾਜ਼ਤ ਦਿੰਦੀ ਹੈ।

### ਰੀਸੈੱਟ ਕਰਨਾ ਅਤੇ ਇੰਪੋਰਟ ਕਰਨਾ

ਨਵੇਂ Ultimate Multisite ਵਰਜ਼ਨ, ਅਤੇ ਨਾਲ ਹੀ add-ons, ਸਮੇਂ-ਸਮੇਂ 'ਤੇ ਨਵੀਆਂ ਈਮੇਲਾਂ ਰਜਿਸਟਰ ਕਰ ਸਕਦੇ ਹਨ ਅਤੇ ਕਰਨਗੇ।

ਟਕਰਾਵਾਂ ਅਤੇ ਹੋਰ ਮੁੱਦਿਆਂ ਨੂੰ ਰੋਕਣ ਲਈ, **ਅਸੀਂ ਤੁਹਾਡੀ ਇੰਸਟਾਲੇਸ਼ਨ 'ਤੇ ਨਵੇਂ ਈਮੇਲ ਟੈਂਪਲੇਟ ਆਪਣੇ ਆਪ System Emails ਵਜੋਂ ਨਹੀਂ ਜੋੜਾਂਗੇ**, ਜਦੋਂ ਤੱਕ ਉਹ ਕਿਸੇ ਖਾਸ ਵਿਸ਼ੇਸ਼ਤਾ ਦੇ ਸਹੀ ਕੰਮ ਕਰਨ ਲਈ ਜ਼ਰੂਰੀ ਨਾ ਹੋਣ।

ਹਾਲਾਂਕਿ, super admins ਅਤੇ agents ਇੰਪੋਰਟਰ ਟੂਲ ਰਾਹੀਂ ਇਹ ਨਵੀਆਂ ਰਜਿਸਟਰਡ ਈਮੇਲਾਂ ਨੂੰ ਇੰਪੋਰਟ ਕਰ ਸਕਦੇ ਹਨ। ਇਹ ਪ੍ਰਕਿਰਿਆ ਨਵੇਂ ਈਮੇਲ ਟੈਂਪਲੇਟ ਦੀ ਸਮੱਗਰੀ ਅਤੇ ਸੰਰਚਨਾ ਨਾਲ ਇੱਕ ਨਵੀਂ ਸਿਸਟਮ ਈਮੇਲ ਬਣਾਏਗੀ, ਜਿਸ ਨਾਲ super admin ਨੂੰ ਜੋ ਵੀ ਸੋਧਾਂ ਚਾਹੀਦੀਆਂ ਹਨ ਕਰ ਸਕਦੇ ਹਨ ਜਾਂ ਉਨ੍ਹਾਂ ਨੂੰ ਜਿਵੇਂ ਹਨ ਰੱਖ ਸਕਦੇ ਹਨ।

#### ਸਿਸਟਮ ਈਮੇਲਾਂ ਕਿਵੇਂ ਇੰਪੋਰਟ ਕਰੀਏ

ਆਪਣੇ Ultimate Multisite Settings ਪੇਜ 'ਤੇ ਜਾਓ ਅਤੇ **Emails** ਟੈਬ 'ਤੇ ਜਾਓ।

![Ultimate Multisite ਸੈਟਿੰਗਾਂ ਵਿੱਚ Emails ਟੈਬ](/img/config/settings-emails.png)

ਫਿਰ, ਸਾਈਡਬਾਰ 'ਤੇ, **Customize System Emails** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ।

![ਸਾਈਡਬਾਰ 'ਤੇ Customize System Emails ਬਟਨ](/img/config/settings-emails.png)

System Emails ਪੇਜ 'ਤੇ, ਤੁਸੀਂ ਉੱਪਰ **Reset & Import** ਐਕਸ਼ਨ ਬਟਨ ਦੇਖੋਗੇ। ਉਸ ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰਨ ਨਾਲ ਇੰਪੋਰਟ ਅਤੇ ਰੀਸੈੱਟ ਮੋਡਲ ਵਿੰਡੋ ਖੁੱਲ੍ਹਣੀ ਚਾਹੀਦੀ ਹੈ।

![System Emails ਪੇਜ 'ਤੇ Reset ਅਤੇ Import ਐਕਸ਼ਨ ਬਟਨ](/img/config/settings-emails.png)

ਫਿਰ, ਤੁਸੀਂ Import Emails ਵਿਕਲਪਾਂ ਨੂੰ ਟੌਗਲ ਕਰ ਸਕਦੇ ਹੋ ਇਹ ਦੇਖਣ ਲਈ ਕਿ ਕਿਹੜੀਆਂ ਸਿਸਟਮ ਈਮੇਲਾਂ ਇੰਪੋਰਟ ਕਰਨ ਲਈ ਉਪਲਬਧ ਹਨ।

![ਉਪਲਬਧ ਸਿਸਟਮ ਈਮੇਲਾਂ ਦਿਖਾਉਂਦੇ Import Emails ਵਿਕਲਪ](/img/config/settings-emails.png)

#### ਸਿਸਟਮ ਈਮੇਲਾਂ ਨੂੰ ਰੀਸੈੱਟ ਕਰਨਾ

ਕਈ ਵਾਰ, ਤੁਹਾਨੂੰ ਪਤਾ ਲੱਗੇਗਾ ਕਿ ਤੁਸੀਂ ਕਿਸੇ ਈਮੇਲ ਟੈਂਪਲੇਟ ਵਿੱਚ ਜੋ ਬਦਲਾਅ ਕੀਤੇ ਹਨ ਉਹ ਹੁਣ ਤੁਹਾਡੇ ਲਈ ਕੰਮ ਨਹੀਂ ਕਰ ਰਹੇ ਅਤੇ ਤੁਸੀਂ ਇਸਨੂੰ ਉਨ੍ਹਾਂ ਦੀ **ਡਿਫਾਲਟ ਸਥਿਤੀ** ਵਿੱਚ ਰੀਸੈੱਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।

ਅਜਿਹੇ ਮਾਮਲਿਆਂ ਵਿੱਚ, ਤੁਹਾਡੇ ਕੋਲ ਦੋ ਵਿਕਲਪ ਹਨ: ਤੁਸੀਂ ਸਿੱਧੇ ਸਿਸਟਮ ਈਮੇਲ ਨੂੰ ਮਿਟਾ ਸਕਦੇ ਹੋ ਅਤੇ ਇਸਨੂੰ ਵਾਪਸ ਇੰਪੋਰਟ ਕਰ ਸਕਦੇ ਹੋ (ਉੱਪਰ ਦਿੱਤੀਆਂ ਹਦਾਇਤਾਂ ਵਰਤ ਕੇ) - ਜੋ send metrics ਅਤੇ ਹੋਰ ਚੀਜ਼ਾਂ ਨੂੰ ਮਿਟਾ ਦੇਵੇਗਾ, ਜਿਸ ਕਾਰਨ ਇਹ ਤਰੀਕਾ ਘੱਟ ਪਸੰਦੀਦਾ ਹੈ।

ਜਾਂ ਤੁਸੀਂ ਉਸ ਈਮੇਲ ਟੈਂਪਲੇਟ ਨੂੰ ਰੀਸੈੱਟ ਕਰਨ ਲਈ **Reset & Import ਟੂਲ** ਵਰਤ ਸਕਦੇ ਹੋ।

ਕਿਸੇ ਈਮੇਲ ਟੈਂਪਲੇਟ ਨੂੰ ਰੀਸੈੱਟ ਕਰਨ ਲਈ, ਤੁਸੀਂ ਉੱਪਰ ਦਿੱਤੇ ਕਦਮਾਂ ਦੀ ਪਾਲਣਾ ਕਰ ਸਕਦੇ ਹੋ ਜਦੋਂ ਤੱਕ ਤੁਸੀਂ Reset & Import ਟੂਲ ਤੱਕ ਨਹੀਂ ਪਹੁੰਚ ਜਾਂਦੇ, ਅਤੇ ਫਿਰ, **Reset** ਵਿਕਲਪ ਨੂੰ ਟੌਗਲ ਕਰੋ ਅਤੇ ਉਹ ਈਮੇਲਾਂ ਚੁਣੋ ਜਿਨ੍ਹਾਂ ਨੂੰ ਤੁਸੀਂ ਉਨ੍ਹਾਂ ਦੀ ਡਿਫਾਲਟ ਸਮੱਗਰੀ 'ਤੇ ਵਾਪਸ ਰੀਸੈੱਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।
