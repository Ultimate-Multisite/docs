# ਡੋਮੇਨ ਮੈਪਿੰਗ ਕਿਵੇਂ ਸੈੱਟ ਕਰੀਏ (v2)

_**ਮਹੱਤਵਪੂਰਨ ਨੋਟ: ਇਹ ਲੇਖ Ultimate Multisite ਵਰਜ਼ਨ 2.x ਬਾਰੇ ਹੈ।**_

ਪ੍ਰੀਮੀਅਮ ਨੈੱਟਵਰਕ ਦੀਆਂ ਸਭ ਤੋਂ ਸ਼ਕਤੀਸ਼ਾਲੀ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਵਿੱਚੋਂ ਇੱਕ ਹੈ ਆਪਣੇ ਗਾਹਕਾਂ ਨੂੰ ਉਨ੍ਹਾਂ ਦੀਆਂ ਸਾਈਟਾਂ ਨਾਲ ਟੌਪ-ਲੈਵਲ ਡੋਮੇਨ ਜੋੜਨ ਦਾ ਮੌਕਾ ਦੇਣਾ। ਆਖ਼ਰਕਾਰ, ਕਿਹੜਾ ਜ਼ਿਆਦਾ ਪੇਸ਼ੇਵਰ ਲੱਗਦਾ ਹੈ: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) ਜਾਂ [_**joesbikeshop.com**_](http://joesbikeshop.com)? ਇਸੇ ਲਈ Ultimate Multisite ਇਹ ਵਿਸ਼ੇਸ਼ਤਾ ਬਿਲਟ-ਇਨ ਦਿੰਦਾ ਹੈ, ਬਿਨਾਂ ਕਿਸੇ ਥਰਡ-ਪਾਰਟੀ plugin ਦੀ ਲੋੜ।

## ਡੋਮੇਨ ਮੈਪਿੰਗ ਕੀ ਹੈ?

ਜਿਵੇਂ ਕਿ ਨਾਮ ਤੋਂ ਪਤਾ ਲੱਗਦਾ ਹੈ, ਡੋਮੇਨ ਮੈਪਿੰਗ Ultimate Multisite ਦੀ ਉਹ ਸਮਰੱਥਾ ਹੈ ਜੋ ਕਸਟਮ ਡੋਮੇਨ ਲਈ ਆਉਣ ਵਾਲੀ ਬੇਨਤੀ ਨੂੰ ਲੈ ਕੇ ਉਸ ਨੂੰ ਨੈੱਟਵਰਕ ਵਿੱਚ ਉਸ ਖਾਸ ਡੋਮੇਨ ਵਾਲੀ ਸੰਬੰਧਿਤ ਸਾਈਟ ਨਾਲ ਜੋੜਦੀ ਹੈ।

### ਆਪਣੇ Ultimate Multisite ਨੈੱਟਵਰਕ 'ਤੇ ਡੋਮੇਨ ਮੈਪਿੰਗ ਕਿਵੇਂ ਸੈੱਟ ਕਰੀਏ

ਡੋਮੇਨ ਮੈਪਿੰਗ ਕੰਮ ਕਰਨ ਲਈ ਤੁਹਾਨੂੰ ਕੁਝ ਸੈਟਿੰਗਾਂ ਕਰਨੀਆਂ ਪੈਣਗੀਆਂ। ਸ਼ੁਕਰ ਹੈ, Ultimate Multisite ਔਖਾ ਕੰਮ ਤੁਹਾਡੇ ਲਈ ਆਪਣੇ ਆਪ ਕਰ ਦਿੰਦਾ ਹੈ ਤਾਂ ਜੋ ਤੁਸੀਂ ਆਸਾਨੀ ਨਾਲ ਲੋੜਾਂ ਪੂਰੀਆਂ ਕਰ ਸਕੋ।

Ultimate Multisite ਇੰਸਟਾਲੇਸ਼ਨ ਦੌਰਾਨ, ਵਿਜ਼ਾਰਡ ਆਪਣੇ ਆਪ **sunrise.php** ਨੂੰ ਨਿਰਧਾਰਿਤ ਫੋਲਡਰ ਵਿੱਚ ਕਾਪੀ ਅਤੇ ਇੰਸਟਾਲ ਕਰ ਦੇਵੇਗਾ। **ਵਿਜ਼ਾਰਡ ਤੁਹਾਨੂੰ ਅੱਗੇ ਨਹੀਂ ਵਧਣ ਦੇਵੇਗਾ ਜਦੋਂ ਤੱਕ ਇਹ ਸਟੈਪ ਪੂਰਾ ਨਹੀਂ ਹੋ ਜਾਂਦਾ**।

![sunrise.php ਸਟੈਪ ਵਾਲਾ Ultimate Multisite ਇੰਸਟਾਲੇਸ਼ਨ ਵਿਜ਼ਾਰਡ](/img/config/settings-domain-mapping.png)

ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਜਦੋਂ Ultimate Multisite ਇੰਸਟਾਲੇਸ਼ਨ ਵਿਜ਼ਾਰਡ ਤੁਹਾਡਾ ਨੈੱਟਵਰਕ ਸੈੱਟ ਕਰ ਲੈਂਦਾ ਹੈ, ਤੁਸੀਂ ਤੁਰੰਤ ਕਸਟਮ ਡੋਮੇਨ ਮੈਪ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹੋ।

ਧਿਆਨ ਰੱਖੋ ਕਿ Ultimate Multisite ਵਿੱਚ ਡੋਮੇਨ ਮੈਪਿੰਗ ਲਾਜ਼ਮੀ ਨਹੀਂ ਹੈ। ਤੁਹਾਡੇ ਕੋਲ WordPress Multisite ਦੀ ਨੇਟਿਵ ਡੋਮੇਨ ਮੈਪਿੰਗ ਵਰਤਣ ਜਾਂ ਕੋਈ ਹੋਰ ਡੋਮੇਨ ਮੈਪਿੰਗ ਹੱਲ ਵਰਤਣ ਦਾ ਵਿਕਲਪ ਹੈ।

ਜੇ ਤੁਹਾਨੂੰ ਹੋਰ ਡੋਮੇਨ ਮੈਪਿੰਗ ਹੱਲਾਂ ਲਈ Ultimate Multisite ਡੋਮੇਨ ਮੈਪਿੰਗ ਬੰਦ ਕਰਨੀ ਪਵੇ, ਤੁਸੀਂ ਇਹ ਵਿਸ਼ੇਸ਼ਤਾ **Ultimate Multisite > Settings > Domain Mapping** ਹੇਠ ਬੰਦ ਕਰ ਸਕਦੇ ਹੋ।

![ਐਨੇਬਲ ਟੌਗਲ ਵਾਲੀਆਂ Domain Mapping ਸੈਟਿੰਗਾਂ](/img/config/settings-domain-mapping.png)

ਇਸ ਵਿਕਲਪ ਦੇ ਬਿਲਕੁਲ ਹੇਠਾਂ, ਤੁਸੀਂ **Force Admin Redirect** ਵਿਕਲਪ ਵੀ ਦੇਖ ਸਕਦੇ ਹੋ। ਇਹ ਵਿਕਲਪ ਤੁਹਾਨੂੰ ਕੰਟਰੋਲ ਕਰਨ ਦਿੰਦਾ ਹੈ ਕਿ ਤੁਹਾਡੇ ਗਾਹਕ ਆਪਣੇ admin dashboard ਤੱਕ ਕਸਟਮ ਡੋਮੇਨ ਅਤੇ subdomain ਦੋਵਾਂ 'ਤੇ ਪਹੁੰਚ ਸਕਣਗੇ ਜਾਂ ਸਿਰਫ਼ ਇੱਕ 'ਤੇ।

ਜੇ ਤੁਸੀਂ **Force redirect to mapped domain** ਚੁਣਦੇ ਹੋ, ਤੁਹਾਡੇ ਗਾਹਕ ਆਪਣੇ admin dashboard ਤੱਕ ਸਿਰਫ਼ ਆਪਣੇ ਕਸਟਮ ਡੋਮੇਨ 'ਤੇ ਹੀ ਪਹੁੰਚ ਸਕਣਗੇ।

**Force redirect to** **network domain** ਵਿਕਲਪ ਬਿਲਕੁਲ ਉਲਟ ਕਰੇਗਾ - ਤੁਹਾਡੇ ਗਾਹਕ ਆਪਣੇ dashboards ਤੱਕ ਸਿਰਫ਼ ਆਪਣੇ subdomain 'ਤੇ ਹੀ ਪਹੁੰਚ ਸਕਣਗੇ, ਭਾਵੇਂ ਉਹ ਆਪਣੇ ਕਸਟਮ ਡੋਮੇਨ 'ਤੇ ਸਾਈਨ ਇਨ ਕਰਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰਨ।

ਅਤੇ **Allow access to the admin by both mapped domain domain and network domain** ਵਿਕਲਪ ਉਨ੍ਹਾਂ ਨੂੰ ਆਪਣੇ admin dashboards ਤੱਕ subdomain ਅਤੇ ਕਸਟਮ ਡੋਮੇਨ ਦੋਵਾਂ 'ਤੇ ਪਹੁੰਚ ਕਰਨ ਦਿੰਦਾ ਹੈ।

![ਡੋਮੇਨ ਮੈਪਿੰਗ ਲਈ Force Admin Redirect ਵਿਕਲਪ](/img/config/settings-domain-mapping.png)

ਕਸਟਮ ਡੋਮੇਨ ਮੈਪ ਕਰਨ ਦੇ ਦੋ ਤਰੀਕੇ ਹਨ। ਪਹਿਲਾ ਹੈ super admin ਵਜੋਂ ਆਪਣੇ network admin dashboard ਤੋਂ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰਨਾ ਅਤੇ ਦੂਜਾ ਹੈ account page ਹੇਠ subsite admin dashboard ਰਾਹੀਂ।

ਪਰ ਆਪਣੇ ਨੈੱਟਵਰਕ ਦੀਆਂ subsites ਵਿੱਚੋਂ ਕਿਸੇ ਇੱਕ ਨਾਲ ਕਸਟਮ ਡੋਮੇਨ ਮੈਪ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਤੁਹਾਨੂੰ ਯਕੀਨੀ ਬਣਾਉਣਾ ਪਵੇਗਾ ਕਿ ਡੋਮੇਨ ਨਾਮ ਦੀਆਂ **DNS ਸੈਟਿੰਗਾਂ** ਸਹੀ ਢੰਗ ਨਾਲ ਕੌਂਫਿਗਰ ਕੀਤੀਆਂ ਹੋਈਆਂ ਹਨ।

### 

### ਯਕੀਨੀ ਬਣਾਉਣਾ ਕਿ ਡੋਮੇਨ DNS ਸੈਟਿੰਗਾਂ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਕੌਂਫਿਗਰ ਹਨ

ਮੈਪਿੰਗ ਕੰਮ ਕਰੇ ਇਸ ਲਈ ਤੁਹਾਨੂੰ ਯਕੀਨੀ ਬਣਾਉਣਾ ਪਵੇਗਾ ਕਿ ਜਿਹੜਾ ਡੋਮੇਨ ਤੁਸੀਂ ਮੈਪ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ ਉਹ ਤੁਹਾਡੇ ਨੈੱਟਵਰਕ ਦੇ IP ਐਡਰੈੱਸ ਵੱਲ ਪੁਆਇੰਟ ਕਰ ਰਿਹਾ ਹੈ। ਧਿਆਨ ਰੱਖੋ ਕਿ ਤੁਹਾਨੂੰ ਨੈੱਟਵਰਕ IP ਐਡਰੈੱਸ ਚਾਹੀਦਾ ਹੈ - ਉਸ ਡੋਮੇਨ ਦਾ IP ਐਡਰੈੱਸ ਜਿੱਥੇ Ultimate Multisite ਇੰਸਟਾਲ ਹੈ - ਨਾ ਕਿ ਉਸ ਕਸਟਮ ਡੋਮੇਨ ਦਾ IP ਐਡਰੈੱਸ ਜੋ ਤੁਸੀਂ ਮੈਪ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ। ਕਿਸੇ ਖਾਸ ਡੋਮੇਨ ਦਾ IP ਐਡਰੈੱਸ ਲੱਭਣ ਲਈ, ਅਸੀਂ [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) 'ਤੇ ਜਾਣ ਦਾ ਸੁਝਾਅ ਦਿੰਦੇ ਹਾਂ, ਉਦਾਹਰਨ ਵਜੋਂ।

ਡੋਮੇਨ ਨੂੰ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਮੈਪ ਕਰਨ ਲਈ, ਤੁਹਾਨੂੰ ਆਪਣੀ **DNS** ਕੌਂਫਿਗਰੇਸ਼ਨ 'ਤੇ ਉਸ **IP ਐਡਰੈੱਸ** ਵੱਲ ਪੁਆਇੰਟ ਕਰਦਾ ਇੱਕ **A RECORD** ਜੋੜਨਾ ਪਵੇਗਾ। DNS ਮੈਨੇਜਮੈਂਟ ਵੱਖ-ਵੱਖ domain registrars ਵਿੱਚ ਕਾਫ਼ੀ ਵੱਖਰੀ ਹੁੰਦੀ ਹੈ, ਪਰ ਜੇ ਤੁਸੀਂ "_Creating A Record on XXXX_" ਸਰਚ ਕਰੋ ਜਿੱਥੇ XXXX ਤੁਹਾਡਾ domain registrar ਹੈ (ਜਿਵੇਂ: "_Creating A Record on_ _GoDaddy_") ਤਾਂ ਔਨਲਾਈਨ ਬਹੁਤ ਸਾਰੇ ਟਿਊਟੋਰੀਅਲ ਮਿਲ ਜਾਣਗੇ।

ਜੇ ਇਹ ਕੰਮ ਕਰਵਾਉਣ ਵਿੱਚ ਤੁਹਾਨੂੰ ਮੁਸ਼ਕਲ ਆ ਰਹੀ ਹੈ, **ਆਪਣੇ domain registrar ਸਪੋਰਟ ਨਾਲ ਸੰਪਰਕ ਕਰੋ** ਅਤੇ ਉਹ ਇਸ ਹਿੱਸੇ ਵਿੱਚ ਤੁਹਾਡੀ ਮਦਦ ਕਰ ਸਕਣਗੇ।

ਜੇ ਤੁਸੀਂ ਆਪਣੇ ਗਾਹਕਾਂ ਨੂੰ ਆਪਣੇ ਡੋਮੇਨ ਖੁਦ ਮੈਪ ਕਰਨ ਦੀ ਇਜਾਜ਼ਤ ਦੇਣ ਦੀ ਯੋਜਨਾ ਬਣਾ ਰਹੇ ਹੋ, ਤਾਂ ਉਨ੍ਹਾਂ ਨੂੰ ਇਸ ਹਿੱਸੇ ਦਾ ਕੰਮ ਖੁਦ ਕਰਨਾ ਪਵੇਗਾ। ਜੇ ਉਹ A Record ਬਣਾਉਣ ਵਿੱਚ ਅਸਮਰੱਥ ਹਨ ਤਾਂ ਉਨ੍ਹਾਂ ਨੂੰ ਉਨ੍ਹਾਂ ਦੇ registrar ਸਪੋਰਟ ਸਿਸਟਮ ਵੱਲ ਭੇਜੋ।

### Super Admin ਵਜੋਂ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰਨਾ

ਜਦੋਂ ਤੁਸੀਂ ਆਪਣੇ ਨੈੱਟਵਰਕ 'ਤੇ super admin ਵਜੋਂ ਲੌਗਇਨ ਹੋ, ਤੁਸੀਂ **Ultimate Multisite > Domains** ਹੇਠ ਜਾ ਕੇ ਆਸਾਨੀ ਨਾਲ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਜੋੜ ਅਤੇ ਮੈਨੇਜ ਕਰ ਸਕਦੇ ਹੋ।

![Ultimate Multisite ਵਿੱਚ Domains ਲਿਸਟ ਪੇਜ](/img/admin/domains-list.png)

ਇਸ ਪੇਜ ਹੇਠ, ਤੁਸੀਂ ਉੱਪਰ **Add Domain** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ ਅਤੇ ਇਹ ਇੱਕ modal ਵਿੰਡੋ ਖੋਲ੍ਹੇਗਾ ਜਿੱਥੇ ਤੁਸੀਂ **ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ**, **ਉਹ subsite** ਜਿਸ 'ਤੇ ਤੁਸੀਂ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਲਾਗੂ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਸੈੱਟ ਅਤੇ ਭਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਫੈਸਲਾ ਕਰ ਸਕਦੇ ਹੋ ਕਿ ਇਸਨੂੰ **primary domain** ਨਾਮ ਵਜੋਂ ਸੈੱਟ ਕਰਨਾ ਹੈ ਜਾਂ ਨਹੀਂ (ਧਿਆਨ ਰੱਖੋ ਕਿ ਤੁਸੀਂ **ਇੱਕ subsite ਨਾਲ ਕਈ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰ ਸਕਦੇ ਹੋ**)।

![ਡੋਮੇਨ ਨਾਮ ਅਤੇ ਸਾਈਟ ਫੀਲਡਾਂ ਵਾਲਾ Add Domain modal](/img/admin/domains-list.png)

ਸਾਰੀ ਜਾਣਕਾਰੀ ਭਰਨ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਹੇਠਾਂ **Add Existing Domain** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦੇ ਹੋ।

ਇਹ ਕਸਟਮ ਡੋਮੇਨ ਦੀ DNS ਜਾਣਕਾਰੀ ਵੈਰੀਫਾਈ ਕਰਨ ਅਤੇ ਫੈੱਚ ਕਰਨ ਦੀ ਪ੍ਰਕਿਰਿਆ ਸ਼ੁਰੂ ਕਰੇਗਾ। ਤੁਸੀਂ ਪੇਜ ਦੇ ਹੇਠਾਂ ਇੱਕ ਲੌਗ ਵੀ ਦੇਖੋਗੇ ਜਿੱਥੇ ਤੁਸੀਂ ਚੱਲ ਰਹੀ ਪ੍ਰਕਿਰਿਆ ਨੂੰ ਫੌਲੋ ਕਰ ਸਕਦੇ ਹੋ। ਇਹ ਪ੍ਰਕਿਰਿਆ ਪੂਰੀ ਹੋਣ ਵਿੱਚ ਕੁਝ ਮਿੰਟ ਲੱਗ ਸਕਦੇ ਹਨ।

ਜੇ ਸਭ ਕੁਝ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਸੈੱਟ ਕੀਤਾ ਹੋਇਆ ਹੈ ਤਾਂ **Stage** ਜਾਂ ਸਥਿਤੀ **Checking DNS** ਤੋਂ **Ready** ਵਿੱਚ ਬਦਲ ਜਾਣੀ ਚਾਹੀਦੀ ਹੈ।

![ਡੋਮੇਨ ਸਟੇਜ Checking DNS ਤੋਂ Ready ਵਿੱਚ ਬਦਲ ਰਹੀ ਹੈ](/img/admin/domains-list.png)

![Domains ਲਿਸਟ ਵਿੱਚ Ready ਸਥਿਤੀ ਵਾਲਾ ਡੋਮੇਨ](/img/admin/domains-list.png)

ਜੇ ਤੁਸੀਂ ਡੋਮੇਨ ਨਾਮ 'ਤੇ ਕਲਿੱਕ ਕਰੋ, ਤੁਸੀਂ ਇਸ ਅੰਦਰ ਕੁਝ ਵਿਕਲਪ ਦੇਖ ਸਕੋਗੇ। ਆਓ ਇਨ੍ਹਾਂ 'ਤੇ ਇੱਕ ਝਾਤ ਮਾਰੀਏ:

![stage, site, active, ਅਤੇ SSL ਵਿਕਲਪਾਂ ਵਾਲਾ Domain ਡੀਟੇਲ ਪੇਜ](/img/admin/domains-list.png)

**Stage:** ਇਹ ਉਹ ਸਟੇਜ ਹੈ ਜਿਸ 'ਤੇ ਡੋਮੇਨ ਹੈ। ਜਦੋਂ ਤੁਸੀਂ ਪਹਿਲੀ ਵਾਰ ਡੋਮੇਨ ਜੋੜਦੇ ਹੋ, ਇਹ ਸ਼ਾਇਦ **Checking DNS** ਸਟੇਜ 'ਤੇ ਹੋਵੇਗਾ। ਪ੍ਰਕਿਰਿਆ DNS ਐਂਟਰੀਆਂ ਚੈੱਕ ਕਰੇਗੀ ਅਤੇ ਪੁਸ਼ਟੀ ਕਰੇਗੀ ਕਿ ਉਹ ਸਹੀ ਹਨ। ਫਿਰ, ਡੋਮੇਨ **Checking SSL** ਸਟੇਜ 'ਤੇ ਰੱਖਿਆ ਜਾਵੇਗਾ। Ultimate Multisite ਚੈੱਕ ਕਰੇਗਾ ਕਿ ਡੋਮੇਨ ਕੋਲ SSL ਹੈ ਜਾਂ ਨਹੀਂ ਅਤੇ ਤੁਹਾਡੇ ਡੋਮੇਨ ਨੂੰ **Ready** ਜਾਂ **Ready (without SSL)** ਵਜੋਂ ਵਰਗੀਕ੍ਰਿਤ ਕਰੇਗਾ।

**Site:** ਇਸ ਡੋਮੇਨ ਨਾਲ ਜੁੜਿਆ subdomain। ਮੈਪ ਕੀਤਾ ਡੋਮੇਨ ਇਸ ਖਾਸ ਸਾਈਟ ਦੀ ਸਮੱਗਰੀ ਦਿਖਾਏਗਾ।

**Active:** ਤੁਸੀਂ ਇਸ ਵਿਕਲਪ ਨੂੰ ਆਨ ਜਾਂ ਆਫ਼ ਟੌਗਲ ਕਰਕੇ ਡੋਮੇਨ ਨੂੰ ਐਕਟਿਵੇਟ ਜਾਂ ਡੀਐਕਟਿਵੇਟ ਕਰ ਸਕਦੇ ਹੋ।

**Is Primary Domain?:** ਤੁਹਾਡੇ ਗਾਹਕਾਂ ਕੋਲ ਹਰ ਸਾਈਟ ਲਈ ਇੱਕ ਤੋਂ ਵੱਧ ਮੈਪ ਕੀਤੇ ਡੋਮੇਨ ਹੋ ਸਕਦੇ ਹਨ। ਇਸ ਵਿਕਲਪ ਦੀ ਵਰਤੋਂ ਕਰੋ ਇਹ ਚੁਣਨ ਲਈ ਕਿ ਇਹ ਖਾਸ ਸਾਈਟ ਲਈ primary domain ਹੈ ਜਾਂ ਨਹੀਂ।

**Is Secure?:** ਭਾਵੇਂ Ultimate Multisite ਡੋਮੇਨ ਐਨੇਬਲ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਚੈੱਕ ਕਰਦਾ ਹੈ ਕਿ ਡੋਮੇਨ ਕੋਲ SSL ਸਰਟੀਫਿਕੇਟ ਹੈ ਜਾਂ ਨਹੀਂ, ਤੁਸੀਂ ਮੈਨੂਅਲੀ ਚੁਣ ਸਕਦੇ ਹੋ ਕਿ ਡੋਮੇਨ ਨੂੰ SSL ਸਰਟੀਫਿਕੇਟ ਨਾਲ ਲੋਡ ਕਰਨਾ ਹੈ ਜਾਂ ਬਿਨਾਂ। ਧਿਆਨ ਰੱਖੋ ਕਿ ਜੇ ਵੈੱਬਸਾਈਟ ਕੋਲ SSL ਸਰਟੀਫਿਕੇਟ ਨਹੀਂ ਹੈ ਅਤੇ ਤੁਸੀਂ ਜ਼ਬਰਦਸਤੀ SSL ਨਾਲ ਲੋਡ ਕਰਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰਦੇ ਹੋ, ਤਾਂ ਇਹ ਤੁਹਾਨੂੰ ਗਲਤੀਆਂ ਦੇ ਸਕਦਾ ਹੈ।

### Subsite ਯੂਜ਼ਰ ਵਜੋਂ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰਨਾ

Subsite administrators ਵੀ ਆਪਣੇ subsite admin dashboard ਤੋਂ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰ ਸਕਦੇ ਹਨ।

ਪਹਿਲਾਂ, ਤੁਹਾਨੂੰ ਯਕੀਨੀ ਬਣਾਉਣਾ ਪਵੇਗਾ ਕਿ ਤੁਸੀਂ **Domain mapping** ਸੈਟਿੰਗਾਂ ਹੇਠ ਇਹ ਵਿਕਲਪ ਐਨੇਬਲ ਕਰੋ। ਹੇਠਾਂ ਸਕ੍ਰੀਨਸ਼ਾਟ ਦੇਖੋ।

![Subsite ਯੂਜ਼ਰਾਂ ਨੂੰ ਡੋਮੇਨ ਮੈਪ ਕਰਨ ਦੀ ਇਜਾਜ਼ਤ ਦੇਣ ਵਾਲੀਆਂ Domain mapping ਸੈਟਿੰਗਾਂ](/img/config/settings-domain-mapping.png)

ਤੁਸੀਂ ਇਸ ਵਿਕਲਪ ਨੂੰ **Ultimate Multisite > Products** 'ਤੇ **Plan** ਲੈਵਲ ਜਾਂ product ਵਿਕਲਪਾਂ ਹੇਠ ਵੀ ਸੈੱਟ ਜਾਂ ਕੌਂਫਿਗਰ ਕਰ ਸਕਦੇ ਹੋ।

![Product ਸੈਟਿੰਗਾਂ ਵਿੱਚ Product custom domains ਵਿਕਲਪ](/img/config/settings-domain-mapping.png)

ਜਦੋਂ ਇਨ੍ਹਾਂ ਵਿੱਚੋਂ ਕੋਈ ਵੀ ਵਿਕਲਪ ਐਨੇਬਲ ਹੋਵੇ ਅਤੇ subsite ਯੂਜ਼ਰ ਨੂੰ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਮੈਪ ਕਰਨ ਦੀ ਇਜਾਜ਼ਤ ਹੋਵੇ, ਤਾਂ subsite ਯੂਜ਼ਰ ਨੂੰ **Account** ਪੇਜ ਹੇਠ **Domains** ਨਾਮ ਦਾ ਇੱਕ metabox ਦਿਖਣਾ ਚਾਹੀਦਾ ਹੈ।

![Subsite account ਪੇਜ 'ਤੇ Domains metabox](/img/admin/domains-list.png)

ਯੂਜ਼ਰ **Add Domain** ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦਾ ਹੈ ਅਤੇ ਇਹ ਕੁਝ ਹਦਾਇਤਾਂ ਵਾਲੀ ਇੱਕ modal ਵਿੰਡੋ ਖੋਲ੍ਹੇਗਾ।

![Subsite ਯੂਜ਼ਰਾਂ ਲਈ DNS ਹਦਾਇਤਾਂ ਵਾਲਾ Add Domain modal](/img/admin/domains-list.png)

ਫਿਰ ਯੂਜ਼ਰ **Next Step** 'ਤੇ ਕਲਿੱਕ ਕਰ ਸਕਦਾ ਹੈ ਅਤੇ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਜੋੜਨ ਲਈ ਅੱਗੇ ਵਧ ਸਕਦਾ ਹੈ। ਉਹ ਇਹ ਵੀ ਚੁਣ ਸਕਦੇ ਹਨ ਕਿ ਇਹ primary domain ਹੋਵੇਗਾ ਜਾਂ ਨਹੀਂ।

![Primary domain ਵਿਕਲਪ ਵਾਲਾ Add custom domain name ਫੀਲਡ](/img/admin/domains-list.png)

![DNS ਵੈਰੀਫਿਕੇਸ਼ਨ ਸ਼ੁਰੂ ਕਰਨ ਲਈ Add Domain 'ਤੇ ਕਲਿੱਕ ਕਰੋ](/img/admin/domains-list.png)**Add Domain** 'ਤੇ ਕਲਿੱਕ ਕਰਨਾ ਕਸਟਮ ਡੋਮੇਨ ਦੀ DNS ਜਾਣਕਾਰੀ ਵੈਰੀਫਾਈ ਕਰਨ ਅਤੇ ਫੈੱਚ ਕਰਨ ਦੀ ਪ੍ਰਕਿਰਿਆ ਸ਼ੁਰੂ ਕਰੇਗਾ।

### Domain Syncing ਬਾਰੇ

Domain Syncing ਇੱਕ ਪ੍ਰਕਿਰਿਆ ਹੈ ਜਿੱਥੇ Ultimate Multisite ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਨੂੰ ਤੁਹਾਡੇ hosting account ਵਿੱਚ ਇੱਕ add-on domain ਵਜੋਂ ਜੋੜਦਾ ਹੈ **ਤਾਂ ਜੋ ਡੋਮੇਨ ਮੈਪਿੰਗ ਕੰਮ ਕਰ ਸਕੇ**।

Domain syncing ਆਪਣੇ ਆਪ ਹੁੰਦੀ ਹੈ ਜੇ ਤੁਹਾਡੇ hosting provider ਦਾ Ultimate Multisite ਡੋਮੇਨ ਮੈਪਿੰਗ ਵਿਸ਼ੇਸ਼ਤਾ ਨਾਲ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਹੈ। ਇਸ ਸਮੇਂ, ਇਹ hosting providers ਹਨ _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ ਅਤੇ _Cpanel._

ਤੁਹਾਨੂੰ ਇਹ ਇੰਟੀਗ੍ਰੇਸ਼ਨ **Integration** ਟੈਬ ਹੇਠ Ultimate Multisite ਸੈਟਿੰਗਾਂ 'ਤੇ ਐਕਟਿਵੇਟ ਕਰਨੀ ਪਵੇਗੀ।

![Ultimate Multisite ਸੈਟਿੰਗਾਂ ਵਿੱਚ Integration ਟੈਬ](/img/config/settings-domain-mapping.png)

![Domain syncing ਲਈ Hosting provider ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਸੈਟਿੰਗਾਂ](/img/config/settings-domain-mapping.png)

_ਧਿਆਨ ਰੱਖੋ ਕਿ ਜੇ ਤੁਹਾਡਾ hosting provider ਉੱਪਰ ਦੱਸੇ ਗਏ providers ਵਿੱਚੋਂ ਨਹੀਂ ਹੈ, **ਤੁਹਾਨੂੰ ਡੋਮੇਨ ਨਾਮ ਨੂੰ ਮੈਨੂਅਲੀ sync ਜਾਂ ਜੋੜਨਾ ਪਵੇਗਾ** ਆਪਣੇ hosting account ਵਿੱਚ।_
