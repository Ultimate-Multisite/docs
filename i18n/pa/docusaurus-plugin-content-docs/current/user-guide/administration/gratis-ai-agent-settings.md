# Gratis AI Agent ਸੈਟਿੰਗਾਂ

Gratis AI Agent ਵਿੱਚ **Settings → Advanced** ਸਕ੍ਰੀਨ ਬੈਕਐਂਡ ਇੰਟੀਗ੍ਰੇਸ਼ਨਾਂ ਲਈ ਪ੍ਰਸ਼ਾਸਕ-ਪੱਧਰ ਦੀ ਸੰਰਚਨਾ ਪ੍ਰਦਾਨ ਕਰਦੀ ਹੈ। ਇਹ ਸਫ਼ਾ ਫੀਡਬੈਕ ਫਾਰਵਰਡਿੰਗ, ਖੋਜ ਪ੍ਰਦਾਤਾ ਕੁੰਜੀਆਂ, ਪ੍ਰਬੰਧਿਤ Superdav ਸੇਵਾ ਸੈੱਟਅੱਪ, Google Calendar ਕੰਟਰੋਲ, TextBee SMS ਸੈਟਿੰਗਾਂ, ਅਤੇ ਨੈੱਟਵਰਕ-ਵਿਆਪੀ ਵਿਸ਼ੇਸ਼ਤਾ ਝੰਡਿਆਂ ਦਾ ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਕਰਦਾ ਹੈ।

## ਸੈਟਿੰਗਾਂ ਤੱਕ ਪਹੁੰਚਣਾ {#accessing-settings}

1. WordPress ਐਡਮਿਨ ਵਿੱਚ, **Gratis AI Agent → Settings** ਤੇ ਜਾਓ।
2. **Advanced** ਟੈਬ ਤੇ ਕਲਿੱਕ ਕਰੋ।

## ਫੀਡਬੈਕ ਐਂਡਪੌਇੰਟ ਸੰਰਚਨਾ {#feedback-endpoint-configuration}

ਫੀਡਬੈਕ ਐਂਡਪੌਇੰਟ AI ਏਜੰਟ ਤੋਂ POST ਬੇਨਤੀਆਂ ਪ੍ਰਾਪਤ ਕਰਦਾ ਹੈ ਜਦੋਂ ਵੀ ਕੋਈ ਵਰਤੋਂਕਾਰ ਥੰਬਜ਼-ਡਾਊਨ ਬਟਨ, ਆਟੋ-ਪ੍ਰੌਮਪਟ ਬੈਨਰ, ਜਾਂ `/report-issue` ਕਮਾਂਡ ਰਾਹੀਂ ਫੀਡਬੈਕ ਜਮ੍ਹਾਂ ਕਰਦਾ ਹੈ।

| ਫੀਲਡ | ਵੇਰਵਾ |
|---|---|
| **ਫੀਡਬੈਕ ਐਂਡਪੌਇੰਟ URL** | ਉਹ URL ਜੋ JSON ਬਾਡੀ ਨਾਲ HTTP POST ਬੇਨਤੀਆਂ ਵਜੋਂ ਫੀਡਬੈਕ ਜਮ੍ਹਾਂਕਰਨ ਪ੍ਰਾਪਤ ਕਰਦਾ ਹੈ। |
| **Feedback API Key** | ਹਰ ਫੀਡਬੈਕ ਬੇਨਤੀ ਦੇ `Authorization` header ਵਿੱਚ ਭੇਜਿਆ ਗਿਆ bearer token। ਜੇ ਤੁਹਾਡੇ ਐਂਡਪੌਇੰਟ ਨੂੰ ਪ੍ਰਮਾਣੀਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ ਹੈ ਤਾਂ ਖਾਲੀ ਛੱਡੋ। |

### ਉਮੀਦ ਕੀਤਾ JSON ਪੇਲੋਡ {#expected-json-payload}

ਤੁਹਾਡੇ ਫੀਡਬੈਕ ਐਂਡਪੌਇੰਟ ਨੂੰ ਘੱਟੋ-ਘੱਟ ਹੇਠਾਂ ਦਿੱਤੇ ਫੀਲਡਾਂ ਨਾਲ JSON ਬਾਡੀ ਸਵੀਕਾਰ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

ਗੱਲਬਾਤ ਦੇ ਸੰਦਰਭ ਦੇ ਆਧਾਰ ਤੇ ਪੇਲੋਡ ਵਿੱਚ ਵਾਧੂ ਫੀਲਡ ਮੌਜੂਦ ਹੋ ਸਕਦੇ ਹਨ।

### `triage_category` ਮੁੱਲ {#triagecategory-values}

AI ਟ੍ਰਾਇਏਜ ਪਰਤ ਪੇਲੋਡ ਨੂੰ ਅੱਗੇ ਭੇਜਣ ਤੋਂ ਪਹਿਲਾਂ `triage_category` ਨੂੰ ਹੇਠਾਂ ਦਿੱਤੇ ਮੁੱਲਾਂ ਵਿੱਚੋਂ ਇੱਕ ਨਿਰਧਾਰਤ ਕਰਦੀ ਹੈ:

| ਮੁੱਲ | ਅਰਥ |
|---|---|
| `factual_error` | ਸਹਾਇਕ ਨੇ ਗਲਤ ਤੱਥਾਤਮਕ ਜਾਣਕਾਰੀ ਦਿੱਤੀ। |
| `unhelpful_answer` | ਜਵਾਬ ਤਕਨੀਕੀ ਤੌਰ ਤੇ ਸਹੀ ਸੀ ਪਰ ਲਾਭਦਾਇਕ ਨਹੀਂ ਸੀ। |
| `inappropriate_content` | ਜਵਾਬ ਵਿੱਚ ਅਜਿਹੀ ਸਮੱਗਰੀ ਸੀ ਜੋ ਵਰਤੋਂਕਾਰਾਂ ਨੂੰ ਨਹੀਂ ਦਿਖਾਈ ਜਾਣੀ ਚਾਹੀਦੀ। |
| `other` | ਫੀਡਬੈਕ ਕਿਸੇ ਜਾਣੀ-ਪਛਾਣੀ ਸ਼੍ਰੇਣੀ ਨਾਲ ਮੇਲ ਨਹੀਂ ਖਾਂਦਾ ਸੀ। |

### ਪ੍ਰਮਾਣੀਕਰਨ {#authentication}

ਜੇ ਤੁਹਾਡੇ ਐਂਡਪੌਇੰਟ ਨੂੰ ਪ੍ਰਮਾਣੀਕਰਨ ਦੀ ਲੋੜ ਹੈ, ਤਾਂ **Feedback API Key** ਫੀਲਡ ਨੂੰ ਆਪਣੇ bearer token ਤੇ ਸੈੱਟ ਕਰੋ। ਏਜੰਟ ਭੇਜਦਾ ਹੈ:

```
Authorization: Bearer <your-api-key>
```

ਜੇ **Feedback API Key** ਫੀਲਡ ਖਾਲੀ ਹੈ, ਤਾਂ ਕੋਈ `Authorization` header ਨਹੀਂ ਭੇਜਿਆ ਜਾਂਦਾ।

### ਫੀਡਬੈਕ ਇਕੱਠਾ ਕਰਨਾ ਅਯੋਗ ਕਰਨਾ {#disabling-feedback-collection}

**ਫੀਡਬੈਕ ਐਂਡਪੌਇੰਟ URL** ਅਤੇ **Feedback API Key** ਦੋਵੇਂ ਫੀਲਡਾਂ ਖਾਲੀ ਛੱਡੋ। ਥੰਬਜ਼-ਡਾਊਨ ਬਟਨ ਅਤੇ ਫੀਡਬੈਕ UI ਵਰਤੋਂਕਾਰਾਂ ਨੂੰ ਦਿਖਾਈ ਦਿੰਦੇ ਰਹਿੰਦੇ ਹਨ, ਪਰ ਜਮ੍ਹਾਂਕਰਨ ਕਿਸੇ ਬਾਹਰੀ ਸੇਵਾ ਨੂੰ ਅੱਗੇ ਨਹੀਂ ਭੇਜੇ ਜਾਂਦੇ।

## Brave Search API Key {#brave-search-api-key}

**Advanced** ਟੈਬ ਤੇ ਹੀ, **Brave Search API Key** ਫੀਲਡ [ਇੰਟਰਨੈੱਟ ਖੋਜ](../configuration/internet-search) ਯੋਗਤਾ ਨੂੰ ਸਮਰੱਥ ਕਰਦੀ ਹੈ।

| ਫੀਲਡ | ਵੇਰਵਾ |
|---|---|
| **Brave Search API Key** | Brave Search ਡਿਵੈਲਪਰ Dashboard ਤੋਂ ਤੁਹਾਡੀ API ਕੁੰਜੀ। AI ਸਹਾਇਕ ਵਿੱਚ ਇੰਟਰਨੈੱਟ ਖੋਜ ਸਮਰੱਥ ਕਰਨ ਲਈ ਲੋੜੀਂਦੀ ਹੈ। |

ਫੀਲਡ ਲੇਬਲ ਵਿੱਚ Brave Search API ਸਾਈਨ-ਅੱਪ ਸਫ਼ੇ ਲਈ ਕਲਿੱਕਯੋਗ ਲਿੰਕ ਸ਼ਾਮਲ ਹੈ। ਇੰਟਰਨੈੱਟ ਖੋਜ ਅਯੋਗ ਕਰਨ ਲਈ ਖਾਲੀ ਛੱਡੋ।

ਇਸ ਵਿਸ਼ੇਸ਼ਤਾ ਬਾਰੇ ਅੰਤਿਮ ਵਰਤੋਂਕਾਰ ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਲਈ [ਇੰਟਰਨੈੱਟ ਖੋਜ](../configuration/internet-search) ਵੇਖੋ।

## ਪ੍ਰਬੰਧਿਤ Superdav ਸੇਵਾ {#managed-superdav-service}

Superdav AI Agent v1.18.0 ਸਮਰਥਿਤ ਸਾਈਟਾਂ ਲਈ ਪ੍ਰਬੰਧਿਤ Superdav ਸੇਵਾ ਐਂਡਪੌਇੰਟ ਅਤੇ ਆਟੋਮੈਟਿਕ ਕਨੈਕਸ਼ਨ ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਜੋੜਦਾ ਹੈ। ਇਹ ਕੰਟਰੋਲ ਉਸ ਵੇਲੇ ਵਰਤੋ ਜਦੋਂ ਤੁਹਾਡੀ ਸਾਈਟ ਨੂੰ ਹੱਥੋਂ ਸੰਰਚਿਤ ਕੀਤੇ ਸੇਵਾ ਐਂਡਪੌਇੰਟ ਦੀ ਥਾਂ ਹੋਸਟ ਕੀਤੇ ਪ੍ਰਦਾਤਾ ਨਾਲ ਜੁੜਨਾ ਚਾਹੀਦਾ ਹੋਵੇ।

| ਫੀਲਡ | ਵੇਰਵਾ |
|---|---|
| **ਪ੍ਰਬੰਧਿਤ Superdav ਸੇਵਾ** | ਸਮਰਥਿਤ ਸਾਈਟਾਂ ਲਈ ਹੋਸਟ ਕੀਤੀ Superdav ਸੇਵਾ ਕਨੈਕਸ਼ਨ ਨੂੰ ਸਮਰੱਥ ਕਰਦੀ ਹੈ। |
| **ਕਨੈਕਸ਼ਨ ਪ੍ਰੋਵਿਜ਼ਨ ਕਰੋ** | ਆਟੋਮੈਟਿਕ ਐਂਡਪੌਇੰਟ ਅਤੇ ਕ੍ਰੈਡੈਂਸ਼ਲ ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਸ਼ੁਰੂ ਕਰਦਾ ਹੈ। ਇਹ ਸਾਈਟ ਦੁਆਰਾ ਪ੍ਰਬੰਧਿਤ ਪ੍ਰਦਾਤਾ ਵਰਤਣ ਦੀ ਪੁਸ਼ਟੀ ਕਰਨ ਤੋਂ ਬਾਅਦ ਵਰਤੋ। |
| **ਸੇਵਾ ਐਂਡਪੌਇੰਟ / ਕਨੈਕਸ਼ਨ ਸਥਿਤੀ** | ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਤੋਂ ਬਾਅਦ ਮੌਜੂਦਾ ਐਂਡਪੌਇੰਟ ਜਾਂ ਕਨੈਕਸ਼ਨ ਸਥਿਤੀ ਦਿਖਾਉਂਦਾ ਹੈ। |

ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਤੋਂ ਬਾਅਦ, ਸੈਟਿੰਗਾਂ ਸੰਭਾਲੋ ਅਤੇ ਪ੍ਰਬੰਧਿਤ-ਸੇਵਾ ਵਰਕਫਲੋਜ਼ ਤੇ ਨਿਰਭਰ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਕਨੈਕਸ਼ਨ ਸਥਿਤੀ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ। ਜੇ ਪ੍ਰੋਵਿਜ਼ਨਿੰਗ ਅਸਫਲ ਰਹਿੰਦੀ ਹੈ, ਤਾਂ ਦਿਖਾਈ ਗਈ ਮੁੜ-ਕੋਸ਼ਿਸ਼ ਰਹਿਨੁਮਾਈ ਦੀ ਸਮੀਖਿਆ ਕਰੋ ਅਤੇ ਪੁਸ਼ਟੀ ਕਰੋ ਕਿ ਸਾਈਟ ਕੋਲ ਹੋਸਟ ਕੀਤੇ ਪ੍ਰਦਾਤਾ ਨੂੰ ਵਰਤਣ ਦੀ ਇਜਾਜ਼ਤ ਹੈ।

## Google Calendar ਸੰਰਚਨਾ {#google-calendar-configuration}

ਜਦੋਂ Superdav AI Agent v1.18.0 ਕੈਲੰਡਰ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਸਮਰੱਥ ਹੁੰਦੀਆਂ ਹਨ, ਏਜੰਟ ਸੰਰਚਿਤ ਕੈਲੰਡਰ ਅਤੇ ਇਵੈਂਟ ਵੇਰਵੇ ਪੜ੍ਹ ਸਕਦਾ ਹੈ। ਕੈਲੰਡਰ ਟੂਲ ਪੜ੍ਹਨ-ਕੇਂਦਰਿਤ ਹਨ ਅਤੇ ਸਮਾਂ-ਸੂਚੀ-ਸਚੇਤ ਯਾਦ-ਦਿਹਾਣੀਆਂ, ਹਾਜ਼ਰੀਨ ਫਾਲੋ-ਅੱਪ, ਅਤੇ ਸੰਪਰਕ ਮਿਲਾਣ ਲਈ ਲਾਭਦਾਇਕ ਹਨ।

| ਫੀਲਡ | ਵੇਰਵਾ |
|---|---|
| **Google Calendar ਕ੍ਰੈਡੈਂਸ਼ਲ** | ਕੈਲੰਡਰ ਡਾਟਾ ਪੜ੍ਹਨ ਲਈ ਲੋੜੀਂਦੇ ਕ੍ਰੈਡੈਂਸ਼ਲ ਜਾਂ ਟੋਕਨ ਕਨੈਕਸ਼ਨ ਨੂੰ ਸਟੋਰ ਕਰਦਾ ਹੈ। |
| **ਕੈਲੰਡਰ ਚੋਣ** | ਸੀਮਿਤ ਕਰਦਾ ਹੈ ਕਿ ਏਜੰਟ ਕਿਹੜੇ ਸੰਰਚਿਤ ਕੈਲੰਡਰਾਂ ਦੀ ਜਾਂਚ ਕਰ ਸਕਦਾ ਹੈ। |
| **ਕੈਲੰਡਰ ਕਨੈਕਸ਼ਨ ਸਥਿਤੀ** | ਪੁਸ਼ਟੀ ਕਰਦੀ ਹੈ ਕਿ ਮੌਜੂਦਾ ਕ੍ਰੈਡੈਂਸ਼ਲ ਕੈਲੰਡਰ ਅਤੇ ਇਵੈਂਟ ਪੜ੍ਹ ਸਕਦੇ ਹਨ ਜਾਂ ਨਹੀਂ। |

ਕੈਲੰਡਰ ਕ੍ਰੈਡੈਂਸ਼ਲ ਉਹਨਾਂ ਕੈਲੰਡਰਾਂ ਤੱਕ ਸੀਮਿਤ ਰੱਖੋ ਜਿਨ੍ਹਾਂ ਦੀ ਏਜੰਟ ਨੂੰ ਲੋੜ ਹੈ। ਜੇ ਸਥਿਤੀ ਮਿਆਦ ਪੁੱਗੇ ਟੋਕਨ ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ ਤਾਂ ਕ੍ਰੈਡੈਂਸ਼ਲ ਮੁੜ ਜੋੜੋ ਜਾਂ ਘੁੰਮਾਓ।

## TextBee SMS ਸੂਚਨਾਵਾਂ {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 ਸੰਰਚਿਤ ਸੂਚਨਾ ਵਰਕਫਲੋਜ਼ ਲਈ TextBee ਨੂੰ SMS ਪ੍ਰਦਾਤਾ ਵਜੋਂ ਜੋੜਦਾ ਹੈ। ਸੰਵੇਦਨਸ਼ੀਲ ਜਾਂ ਵਰਤੋਂਕਾਰ-ਸਾਹਮਣੇ ਸੁਨੇਹਿਆਂ ਲਈ SMS ਸੂਚਨਾਵਾਂ ਨੂੰ ਮਨੁੱਖੀ ਮਨਜ਼ੂਰੀ ਦਰਵਾਜ਼ਿਆਂ ਨਾਲ ਜੋੜਿਆ ਜਾਣਾ ਚਾਹੀਦਾ ਹੈ।

| ਫੀਲਡ | ਵੇਰਵਾ |
|---|---|
| **TextBee API ਕੁੰਜੀ** | TextBee SMS ਪ੍ਰਦਾਤਾ ਲਈ ਬੇਨਤੀਆਂ ਦਾ ਪ੍ਰਮਾਣੀਕਰਨ ਕਰਦੀ ਹੈ। |
| **TextBee ਡਿਵਾਈਸ / ਭੇਜਣ ਵਾਲਾ** | ਪ੍ਰਦਾਤਾ ਦੁਆਰਾ ਲੋੜ ਹੋਣ ਤੇ ਬਾਹਰ ਜਾਣ ਵਾਲੇ ਸੁਨੇਹਿਆਂ ਲਈ ਵਰਤਿਆ ਜਾਣ ਵਾਲਾ TextBee ਭੇਜਣ ਵਾਲਾ ਜਾਂ ਡਿਵਾਈਸ ਚੁਣਦਾ ਹੈ। |
| **SMS ਸੂਚਨਾਵਾਂ ਸਮਰੱਥ** | ਮਨਜ਼ੂਰਸ਼ੁਦਾ ਵਰਕਫਲੋਜ਼ ਨੂੰ ਟੈਕਸਟ-ਸੁਨੇਹਾ ਸੂਚਨਾਵਾਂ ਭੇਜਣ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ। SMS ਭੇਜਣ ਤੋਂ ਰੋਕਣ ਲਈ ਅਯੋਗ ਛੱਡੋ। |

ਕੇਵਲ ਪ੍ਰਸ਼ਾਸਕ-ਮਾਲਕਾਨਾ ਨੰਬਰ ਤੇ ਇੱਕ ਟੈਸਟ ਸੁਨੇਹਾ ਭੇਜੋ, ਫਿਰ ਨਿਰਧਾਰਤ ਜਾਂ ਹਾਜ਼ਰੀਨ-ਸਾਹਮਣੇ ਯਾਦ-ਦਿਹਾਣੀਆਂ ਸਮਰੱਥ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਮਨਜ਼ੂਰੀ-ਦਰਵਾਜ਼ਾ ਵਿਵਹਾਰ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ।

## ਵਿਸ਼ੇਸ਼ਤਾ ਝੰਡੇ {#feature-flags}

v1.9.0 ਵਿੱਚ ਵੀ ਪੇਸ਼ ਕੀਤੀ ਗਈ, **Settings → Feature Flags** ਟੈਬ ਵਿਕਲਪਿਕ ਕਾਰਗੁਜ਼ਾਰੀ ਲਈ ਟੌਗਲ ਸਵਿੱਚ ਪ੍ਰਦਾਨ ਕਰਦੀ ਹੈ। ਹਰ ਝੰਡਾ ਨੈੱਟਵਰਕ-ਵਿਆਪੀ ਤੌਰ ਤੇ ਜਾਂ ਤਾਂ ਸਮਰੱਥ ਜਾਂ ਅਯੋਗ ਹੁੰਦਾ ਹੈ; ਇਸ ਵੇਲੇ ਪ੍ਰਤੀ-ਸਾਈਟ ਓਵਰਰਾਈਡ ਨਹੀਂ ਹੈ।

### ਵਿਸ਼ੇਸ਼ਤਾ ਝੰਡਿਆਂ ਤੱਕ ਪਹੁੰਚਣਾ {#accessing-feature-flags}

1. WordPress ਐਡਮਿਨ ਵਿੱਚ, **Gratis AI Agent → Settings** ਤੇ ਜਾਓ।
2. **Feature Flags** ਟੈਬ ਤੇ ਕਲਿੱਕ ਕਰੋ।

### ਆਟੋਮੇਸ਼ਨ ਸੁਰੱਖਿਆ ਫਲੈਗ {#automation-safety-flags}

### ਬਦਲਾਅ ਲਾਗੂ ਕਰਨਾ {#applying-changes}

ਕਿਸੇ ਵੀ ਫਲੈਗ ਨੂੰ ਟੌਗਲ ਕਰਨ ਤੋਂ ਬਾਅਦ **ਸੈਟਿੰਗਾਂ ਸੇਵ ਕਰੋ** 'ਤੇ ਕਲਿੱਕ ਕਰੋ। ਬਦਲਾਅ ਤੁਰੰਤ ਪ੍ਰਭਾਵੀ ਹੋ ਜਾਂਦੇ ਹਨ — ਕੋਈ ਕੈਸ਼ ਫਲਸ਼ ਜਾਂ plugin ਮੁੜ-ਸਰਗਰਮੀ ਲੋੜੀਂਦੀ ਨਹੀਂ ਹੈ।
