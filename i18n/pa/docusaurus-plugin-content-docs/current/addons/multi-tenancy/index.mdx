# ਬਹੁ-ਕਿਰਾਏਦਾਰੀ {#multi-tenancy}

ਕਾਰਗੁਜ਼ਾਰੀ ਅਤੇ ਸੁਰੱਖਿਆ ਲਈ ਸਬਸਾਈਟਾਂ ਨੂੰ ਸਰਵਰਾਂ ਅਤੇ ਡਾਟਾਬੇਸਾਂ ਵਿੱਚ ਵੰਡੋ

## ਵਰਜਨ 1.2.0 ਰਿਲੀਜ਼ ਦੀਆਂ ਮੁੱਖ ਗੱਲਾਂ {#version-120-release-highlights}

ਬਹੁ-ਕਿਰਾਏਦਾਰੀ 1.2.0 ਉਤਪਾਦਨ ਅਲੱਗਾਵ ਵਰਕਫਲੋ ਲਈ ਸਰਬਭੌਮ ਕਿਰਾਏਦਾਰ ਬੁਨਿਆਦ ਜੋੜਦਾ ਹੈ:

- ਛੋਟੀ ਮਿਆਦ ਵਾਲੀਆਂ, ਮੁੜ-ਚਲਾਉਣ ਤੋਂ ਸੁਰੱਖਿਅਤ SSO ਮੁਲਾਕਾਤਾਂ ਨਾਲ ਸਟੇਟਲੈੱਸ ਕਿਰਾਏਦਾਰ ਆਟੋਲੌਗਇਨ।
- ਸਰਬਭੌਮ ਕਿਰਾਏਦਾਰਾਂ ਲਈ ਪ੍ਰਤੀ-ਸਬਸਾਈਟ ਡਾਟਾਬੇਸ ਅਤੇ ਫਾਈਲਸਿਸਟਮ ਅਲੱਗਾਵ।
- ਹੋਸਟ-ਸਚੇਤ ਕਿਰਾਏਦਾਰ ਪ੍ਰੋਵੀਜ਼ਨਿੰਗ ਲਈ Hostinger ਬਹੁ-ਕਿਰਾਏਦਾਰੀ ਸਮਰੱਥਾ।
- ਪੁਰਾਣੇ ਡਾਟਾ ਜਾਂਚਾਂ ਅਤੇ ਅਸਿੰਕ ਪੁਸ਼ ਪ੍ਰੋਸੈਸਿੰਗ ਲਈ ਸਰਬਭੌਮ ਮਾਈਗ੍ਰੇਸ਼ਨ ਪੁਸ਼ਟੀਕਰਨ ਗੇਟ।
- Bedrock, FrankenPHP, ਅਤੇ ਲੋਕਲਹੋਸਟ ਇੰਸਟਾਲਾਂ ਲਈ ਸੰਰਚਿਤ ਕੀਤੀ ਜਾ ਸਕਣ ਵਾਲੀ ਇੱਕੋ-ਮਸ਼ੀਨ ਡਾਟਾਬੇਸ ਹੋਸਟ ਬਾਈਂਡਿੰਗ।
- ਕਿਰਾਏਦਾਰ ਬੂਟਸਟ੍ਰੈਪ, ਡਾਟਾਬੇਸ ਰਾਈਟਰ, ਸਕੀਮਾ ਬੂਟਸਟ੍ਰੈਪ, ਅਤੇ ਯੂਜ਼ਰ ਪ੍ਰੋਵੀਜ਼ਨਿੰਗ ਸੁਧਾਰ।
- ਹੋਰ ਸੁਰੱਖਿਅਤ ਸਰਬਭੌਮ ਟੀਅਰਡਾਊਨ ਜੋ ਕਿਰਾਏਦਾਰ ਕਰੈਡੈਂਸ਼ਲ ਸਾਫ਼ ਕਰਦਾ ਹੈ।

[ਸਰਬਭੌਮ ਮਾਈਗ੍ਰੇਸ਼ਨ ਵਰਕਫਲੋ](/user-guide/getting-started/sovereign-migration-workflow), [ਬਹੁ-ਕਿਰਾਏਦਾਰੀ ਅਲੱਗਾਵ](/user-guide/configuration/multi-tenancy-isolation), ਅਤੇ [ਸਰਬਭੌਮ ਕਿਰਾਏਦਾਰ ਓਪਰੇਸ਼ਨ](/user-guide/administration/sovereign-tenant-operations) ਲਈ ਯੂਜ਼ਰ ਗਾਈਡਾਂ ਵੇਖੋ।

## ਲੋੜਾਂ {#requirements}

- WordPress 5.3 ਜਾਂ ਇਸ ਤੋਂ ਉੱਚਾ
- PHP 7.4 ਜਾਂ ਇਸ ਤੋਂ ਉੱਚਾ
- Multisite Ultimate plugin (ਸਰਗਰਮ)

## ਇੰਸਟਾਲੇਸ਼ਨ {#installation}

1. ਐਡਆਨ ਫਾਈਲਾਂ ਨੂੰ ਆਪਣੀ `/wp-content/plugins/` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਅੱਪਲੋਡ ਕਰੋ
2. WordPress ਵਿੱਚ 'Plugins' ਮੀਨੂ ਰਾਹੀਂ plugin ਨੂੰ ਸਰਗਰਮ ਕਰੋ
3. ਨੈੱਟਵਰਕ ਐਡਮਿਨ ਵਿੱਚ ਐਡਆਨ ਸੈਟਿੰਗਾਂ ਸੰਰਚਿਤ ਕਰੋ

## ਵਿਕਾਸ {#development}

### ਸੈਟਅੱਪ {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### ਟੈਸਟਿੰਗ {#testing}

ਐਡਆਨ ਵਿੱਚ ਇੱਕ ਵਿਸਤ੍ਰਿਤ ਟੈਸਟਿੰਗ ਫਰੇਮਵਰਕ ਸ਼ਾਮਲ ਹੈ:

- **ਯੂਨਿਟ ਟੈਸਟ**: WordPress ਟੈਸਟ ਸੂਟ ਨਾਲ PHPUnit-ਆਧਾਰਿਤ ਟੈਸਟ
- **ਬੇਸ ਟੈਸਟ ਕਲਾਸ**: ਆਮ ਟੈਸਟ ਸਥਿਤੀਆਂ ਲਈ ਸਹਾਇਕ ਵਿਧੀਆਂ ਪ੍ਰਦਾਨ ਕਰਦੀ ਹੈ
- **ਟੈਸਟ ਕਵਰੇਜ**: ਕੋਡ ਕਵਰੇਜ ਟ੍ਰੈਕ ਕਰਨ ਲਈ ਸੰਰਚਿਤ
- **ਮਲਟੀਸਾਈਟ ਟੈਸਟਿੰਗ**: ਟੈਸਟ ਮਲਟੀਸਾਈਟ ਵਾਤਾਵਰਣ ਵਿੱਚ ਚੱਲਦੇ ਹਨ

### ਕੋਡ ਮਿਆਰ {#code-standards}

- **WordPress ਕੋਡਿੰਗ ਮਿਆਰ**: PHPCS ਰਾਹੀਂ ਲਾਗੂ
- **PHP 7.4+ ਅਨੁਕੂਲਤਾ**: ਆਧੁਨਿਕ PHP ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਸਮਰਥਿਤ
- **ਸਟੈਟਿਕ ਵਿਸ਼ਲੇਸ਼ਣ**: ਕਿਸਮ ਜਾਂਚ ਲਈ PHPStan ਇੰਟੀਗ੍ਰੇਸ਼ਨ
- **ਕੋਡ ਆਧੁਨਿਕੀਕਰਨ**: PHP ਅੱਪਗ੍ਰੇਡਾਂ ਲਈ Rector ਨਿਯਮ

### ਬਿਲਡ ਸਿਸਟਮ {#build-system}

- **ਐਸੈਟ ਪ੍ਰੋਸੈਸਿੰਗ**: CSS/JS ਫਾਈਲਾਂ ਲਈ ਮਿਨੀਫਿਕੇਸ਼ਨ
- **ਅਨੁਵਾਦ**: POT ਫਾਈਲ ਬਣਾਉਣਾ
- **ਪੈਕੇਜਿੰਗ**: ਵੰਡ ਲਈ ਆਰਕਾਈਵ ਬਣਾਉਣਾ
- **ਵਿਕਾਸ ਮੋਡ**: ਡੀਬੱਗਿੰਗ ਲਈ ਗੈਰ-ਮਿਨੀਫਾਈਡ ਐਸੈਟ

## ਸੰਰਚਨਾ {#configuration}

ਐਡਆਨ ਨੈੱਟਵਰਕ ਐਡਮਿਨ ਇੰਟਰਫੇਸ ਰਾਹੀਂ ਉਪਲਬਧ ਵੱਖ-ਵੱਖ ਸੰਰਚਨਾ ਵਿਕਲਪਾਂ ਦਾ ਸਮਰਥਨ ਕਰਦਾ ਹੈ।

### ਸਰਬਭੌਮ ਕਿਰਾਏਦਾਰ ਸੰਰਚਨਾ {#sovereign-tenant-configuration}

ਸਰਬਭੌਮ ਕਿਰਾਏਦਾਰਾਂ ਲਈ, ਕਿਰਾਏਦਾਰ ਡਾਟਾਬੇਸ, ਫਾਈਲਸਿਸਟਮ ਰੂਟ, ਡਾਟਾਬੇਸ ਹੋਸਟ ਬਾਈਂਡਿੰਗ, ਅਤੇ ਹੋਸਟ ਸਮਰੱਥਾ ਨੂੰ ਟ੍ਰੈਫਿਕ ਕਿਰਾਏਦਾਰ ਵੱਲ ਰੂਟ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਸੰਰਚਿਤ ਕਰੋ। ਡਾਟਾਬੇਸ ਕਰੈਡੈਂਸ਼ਲ, ਹੋਸਟ ਬਾਈਂਡਿੰਗ, ਰੂਟਿੰਗ, ਜਾਂ ਯੂਜ਼ਰ ਪ੍ਰੋਵੀਜ਼ਨਿੰਗ ਨੂੰ ਪ੍ਰਭਾਵਿਤ ਕਰਨ ਵਾਲੇ ਹਰ ਬਦਲਾਅ ਤੋਂ ਬਾਅਦ ਮਾਈਗ੍ਰੇਸ਼ਨ ਪੁਸ਼ਟੀਕਰਨ ਚਲਾਓ।

`localhost` ਸਿਰਫ਼ ਤਦੋਂ ਵਰਤੋ ਜਦੋਂ ਕਿਰਾਏਦਾਰ ਰਨਟਾਈਮ ਅਤੇ ਡਾਟਾਬੇਸ ਗ੍ਰਾਂਟਾਂ ਲੋਕਲ ਸਾਕਟ ਜਾਂ ਇੱਕੋ-ਮਸ਼ੀਨ ਪਹੁੰਚ ਦੀ ਉਮੀਦ ਕਰਦੀਆਂ ਹਨ। `127.0.0.1` ਜਾਂ ਕੋਈ ਸੇਵਾ ਹੋਸਟਨਾਮ ਤਦੋਂ ਵਰਤੋ ਜਦੋਂ ਉਹ ਤੁਹਾਡੇ ਪ੍ਰਦਾਤਾ ਵੱਲੋਂ ਸੰਰਚਿਤ ਕੀਤਾ ਗਿਆ ਗ੍ਰਾਂਟ ਹੋਸਟ ਹੋਵੇ।

### Hostinger ਸਮਰੱਥਾ {#hostinger-capability}

Hostinger ਬਹੁ-ਕਿਰਾਏਦਾਰੀ ਸਮਰੱਥਾ ਸਾਂਝੇ Hostinger ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਟੋਕਨ ਨਾਲ ਕੰਮ ਕਰਦੀ ਹੈ। ਪੁਸ਼ਟੀਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਡੋਮੇਨ hPanel ਵਿੱਚ ਮੌਜੂਦ ਹੈ, DNS Hostinger Account ਵੱਲ ਇਸ਼ਾਰਾ ਕਰਦਾ ਹੈ, ਅਤੇ ਕਿਰਾਏਦਾਰ ਡਾਟਾਬੇਸ ਸੈਟਿੰਗਾਂ hPanel ਨਾਲ ਮਿਲਦੀਆਂ ਹਨ।

## ਹੁੱਕ ਅਤੇ ਫਿਲਟਰ {#hooks-and-filters}

### ਐਕਸ਼ਨ {#actions}

- `ultimate-multisite-addon-template_init` - ਐਡਆਨ ਸ਼ੁਰੂਆਤ ਤੋਂ ਬਾਅਦ ਚਲਾਇਆ ਜਾਂਦਾ ਹੈ
- `ultimate-multisite-addon-template_loaded` - ਸਾਰੀਆਂ ਐਡਆਨ ਫਾਈਲਾਂ ਲੋਡ ਹੋਣ ਤੋਂ ਬਾਅਦ ਚਲਾਇਆ ਜਾਂਦਾ ਹੈ

### ਫਿਲਟਰ {#filters}

- `ultimate-multisite-addon-template_settings` - ਐਡਆਨ ਸੈਟਿੰਗਾਂ ਸੋਧੋ
- `ultimate-multisite-addon-template_enabled` - ਐਡਆਨ ਸਮਰੱਥ/ਅਸਮਰੱਥ ਸਥਿਤੀ ਨੂੰ ਓਵਰਰਾਈਡ ਕਰੋ
