# Multi-Currency

Ultimate Multisite: Multi-Currency ਤੁਹਾਡੇ ਨੈੱਟਵਰਕ ਨੂੰ ਕਈ ਮੁਦਰਾਵਾਂ ਵਿੱਚ ਭੁਗਤਾਨ ਸਵੀਕਾਰ ਕਰਨ ਦੇ ਯੋਗ ਬਣਾਉਂਦਾ ਹੈ। ਇਹ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਲਾਈਵ ਜਾਂ ਮੈਨੂਅਲ ਐਕਸਚੇਂਜ ਰੇਟਸ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਕੀਮਤਾਂ ਨੂੰ ਬਦਲਦਾ ਹੈ, ਤੁਹਾਨੂੰ ਕਿਸੇ ਵੀ ਕਰੰਸੀ ਲਈ ਪ੍ਰੋਡਕਟ-ਵਾਰ ਸਹੀ ਕੀਮਤ ਸੈੱਟ ਕਰਨ ਦਿੰਦਾ ਹੈ, ਅਤੇ ਜੀਓ-ਲੋਕੇਸ਼ਨ ਰਾਹੀਂ ਹਰੇਕ ਵਿਜ਼ਟਰ ਦੀ ਪਸੰਦੀਦਾ ਕਰੰਸੀ ਪਛਾਣਦਾ ਹੈ — ਇਹ ਸਭ ਕੁਝ ਇੱਕ ਹੀ admin dashboard ਤੋਂ ਸੰਭਾਲਿਆ ਜਾਂਦਾ ਹੈ।

## Key Features

- **Automatic exchange rate conversion** — ਤੁਹਾਡੇ ਬੇਸ ਕਰੰਸੀ ਤੋਂ ਕੀਮਤਾਂ ਨੂੰ ਤੁਹਾਡੇ ਚੁਣੇ ਹੋਏ ਪ੍ਰੋਵਾਈਡਰ ਤੋਂ ਮਿਲੇ ਰੇਟਸ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਬਦਲਿਆ ਜਾਂਦਾ ਹੈ।
- **Per-product pricing overrides** — ਆਟੋਮੈਟਿਕ ਕਨਵਰਸ਼ਨ ਨੂੰ ਪਾਸ ਕਰਦੇ ਹੋਏ ਕਿਸੇ ਵੀ ਪ੍ਰੋਡਕਟ 'ਤੇ ਕਰੰਸੀ-ਵਿਸ਼ੇਸ਼ ਕੀਮਤਾਂ ਸੈੱਟ ਕਰੋ।
- **Currency selector signup field** — ਗਾਹਕਾਂ ਨੂੰ ਚੈੱਕਆਉਟ/ਸਾਈਨਅੱਪ ਦੌਰਾਨ ਆਪਣੀ ਪਸੰਦੀਦਾ ਕਰੰਸੀ ਚੁਣਨ ਦਿੰਦਾ ਹੈ।
- **Multiple exchange rate providers** — ECB, Frankfurter, Stripe, ਜਾਂ ਪੂਰੀ ਤਰ੍ਹਾਂ ਮੈਨੂਅਲ ਰੇਟਸ।
- **Geolocation-based currency detection** — ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਉਸ ਕਰੰਸੀ ਨੂੰ ਪਹਿਲਾਂ ਹੀ ਚੁਣ ਲੈਂਦਾ ਹੈ ਜੋ ਵਿਜ਼ਟਰ ਦੇ ਦੇਸ਼ ਨਾਲ ਮੇਲ ਖਾਂਦੀ ਹੈ।
- **Admin dashboard** — ਇੱਕ ਜਗ੍ਹਾ ਤੋਂ ਸਰਗਰਮ ਕਰੰਸੀਆਂ, ਐਕਸਚੇਂਜ ਰੇਟਸ, ਅਤੇ ਪ੍ਰੋਵਾਈਡਰ ਸੈਟਿੰਗਾਂ ਦਾ ਪ੍ਰਬੰਧਨ ਕਰੋ।

## Requirements

- Ultimate Multisite 2.0.0 ਜਾਂ ਇਸ ਤੋਂ ਵੱਧ
- WordPress Multisite installation
- PHP 7.4 ਜਾਂ ਇਸ ਤੋਂ ਵੱਧ

## Installation

1. ਪਲੱਗਇਨ ਫਾਈਲਾਂ ਨੂੰ `/wp-content/plugins/ultimate-multisite-multi-currency` 'ਤੇ ਅਪਲੋਡ ਕਰੋ।
2. WordPress ਵਿੱਚ **Plugins** ਸਕ੍ਰੀਨ ਰਾਹੀਂ ਪਲੱਗਇਨ ਨੂੰ ਐਕਟਿਵੇਟ ਕਰੋ (Network Activate)।
3. admin dashboard ਖੋਲ੍ਹਣ ਲਈ **Ultimate Multisite → Multi-Currency** 'ਤੇ ਜਾਓ।
4. ਉਹ ਕਰੰਸੀ ਜੋ ਤੁਸੀਂ ਸਵੀਕਾਰ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ ਉਹ ਜੋੜੋ ਅਤੇ ਆਪਣਾ ਐਕਸਚੇਂਜ ਰੇਟ ਪ੍ਰੋਵਾਈਡਰ ਕੰフィਗਰ ਕਰੋ।

## Configuration

### Admin Dashboard

Multi-Currency admin dashboard (**Ultimate Multisite → Multi-Currency**) ਉਹ ਕੇਂਦਰੀ ਜਗ੍ਹਾ ਹੈ ਜਿੱਥੇ ਤੁਸੀਂ:

- ਕਰੰਸੀ ਜੋੜ ਸਕਦੇ ਹੋ, ਐਨੇਬਲ ਕਰ ਸਕਦੇ ਹੋ, ਜਾਂ ਡਿਸੇਬਲ ਕਰ ਸਕਦੇ ਹੋ।
- ਇੱਕ ਐਕਸਚੇਂਜ ਰੇਟ ਪ੍ਰੋਵਾਈਡਰ ਚੁਣ ਸਕਦੇ ਹੋ ਅਤੇ ਲੋੜ ਪੈਣ 'ਤੇ ਰੇਟਸ ਨੂੰ ਰਿਫਰੈਸ਼ ਕਰ ਸਕਦੇ ਹੋ।
- ਸਾਰੀਆਂ ਸਰਗਰਮ ਕਰੰਸੀਆਂ ਲਈ ਮੌਜੂਦਾ ਰੇਟਾਂ ਦੀ ਸਮੀਖਿਆ ਕਰ ਸਕਦੇ ਹੋ।
- ਮੈਨੂਅਲ ਰੇਟ ਸੈੱਟ ਕਰ ਸਕਦੇ ਹੋ ਜਦੋਂ ਆਟੋਮੈਟਿਕ ਪ੍ਰੋਵਾਈਡਰ ਸਹੀ ਨਾ ਹੋਣ।

### Exchange Rate Providers

| Provider | Description |
|---|---|
| **ECB** | European Central Bank — EUR-based ਨੈੱਟਵਰਕ ਲਈ ਮੁਫ਼ਤ ਰੋਜ਼ਾਨਾ ਰੇਟਸ। |
| **Frankfurter** | Open-source ECB mirror API — ਮੁਫ਼ਤ, ਕੋਈ API key ਜ਼ਰੂਰੀ ਨਹੀਂ। |
| **Stripe** | Stripe ਦੇ ਲਾਈਵ FX ਰੇਟਸ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ — ਇੱਕ ਸਰਗਰਮ Stripe integration ਦੀ ਲੋੜ ਹੈ। |
| **Manual** | ਤੁਸੀਂ ਸਿੱਧੇ ਰੇਟ ਸਾਂਝੇ ਕਰਦੇ ਹੋ; ਕੋਈ ਬਾਹਰੀ API ਕਾਲ ਨਹੀਂ। |

ਰੇਟਸ ਨੂੰ **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** ਹੇਠਾਂ ਆਪਣਾ ਪ੍ਰੋਵਾਈਡਰ ਚੁਣੋ। ਰੇਟਸ ਨੂੰ ਕਿਸੇ ਵੀ ਸਮੇਂ ਮੈਨੂਅਲੀ ਰਿਫਰੈਸ਼ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ ਜਾਂ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਅਪਡੇਟ ਹੋਣ ਲਈ ਸੈੱਡ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ।

### Per-Product Pricing Overrides

ਜਦੋਂ ਆਟੋਮੈਟਿਕ ਕਨਵਰਸ਼ਨ ਕਾਫ਼ੀ ਸਹੀ ਨਹੀਂ ਹੁੰਦੀ, ਤਾਂ ਤੁਸੀਂ ਕਿਸੇ ਵੀ ਪ੍ਰੋਡਕਟ 'ਤੇ ਕਿਸੇ ਖਾਸ ਕਰੰਸੀ ਲਈ ਸਹੀ ਕੀਮਤ ਸੈੱਟ ਕਰ ਸਕਦੇ ਹੋ:

1. ਪ੍ਰੋਡਕਟ ਐਡੀਟਰ ਖੋਲ੍ਹੋ (**Ultimate Multisite → Products → Edit**)।
2. **Currency Pricing** ਸੈਕਸ਼ਨ ਤੱਕ ਸਕ੍ਰੋਲ ਕਰੋ।
3. ਹਰੇਕ ਕਰੰਸੀ ਲਈ ਸਹੀ ਕੀਮਤ ਸਾਂਝੀ ਕਰੋ ਜਿਸਨੂੰ ਤੁਸੀਂ ਓਵਰਰਾਈਡ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।
4. ਪ੍ਰੋਡਕਟ ਨੂੰ ਸੇਵ ਕਰੋ।

ਓਵਰਰਾਈਡ ਤੋਂ ਬਿਨਾਂ ਪ੍ਰੋਡਕਟ ਤੁਹਾਡੀ ਬੇਸ ਕਰੰਸੀ ਤੋਂ ਬਦਲੀ ਹੋਈ ਕੀਮਤ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਨ।

### Currency Selector Signup Field

ਕਰੰਸੀ ਸెలెక్టర్ ਫੀਲਡ ਨੂੰ ਤੁਹਾਡੇ ਸਾਈਨਅੱਪ/ਚੈੱਕਆਉਟ ਫਲੋ ਵਿੱਚ ਜੋੜਿਆ ਜਾ ਸਕਦਾ ਹੈ ਤਾਂ ਜੋ ਗਾਹਕ ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਦੌਰਾਨ ਆਪਣੀ ਪਸੰਦੀਦਾ ਕਰੰਸੀ ਚੁਣ ਸਕਣ:

1. **Ultimate Multisite → Multi-Currency → Settings** 'ਤੇ ਜਾਓ।
2. **Show currency selector on signup** ਨੂੰ ਐਨੇਬਲ ਕਰੋ।
3. ਵਿਕਲਪਿਕ ਤੌਰ 'ਤੇ ਇੱਕ ਡਿਫਾਲਟ ਕਰੰਸੀ ਸੈੱਟ ਕਰੋ ਜਾਂ ਜੀਓ-ਲੋਕੇਸ਼ਨ ਨੂੰ ਪਹਿਲਾਂ ਚੁਣਿਆ ਗਿਆ ਮੁੱਲ ਨਿਰਧਾਰਿਤ ਕਰਨ ਦਿਓ।

ਚੁਣੀ ਗਈ ਕਰੰਸੀ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਨਾਲ ਸਟੋਰ ਕਰ ਲਈ ਜਾਂਦੀ ਹੈ ਅਤੇ ਸਾਰੀ ਬਾਅਦ ਦੀ ਬਿਲਿੰਗ ਲਈ ਵਰਤੀ ਜਾਂਦੀ ਹੈ।

### Geolocation-Based Currency Detection

ਜਦੋਂ ਜੀਓ-ਲੋਕੇਸ਼ਨ ਡਿਟੈਕਸ਼ਨ ਐਨੇਬਲ ਹੁੰਦੀ ਹੈ, ਤਾਂ ਇਹ ਐਡਨੋਨ ਵਿਜ਼ਟਰ ਦੇ ਦੇਸ਼ ਨੂੰ ਇੱਕ ਕਰੰਸੀ ਨਾਲ ਜੋੜਦਾ ਹੈ ਅਤੇ ਇਸਨੂੰ ਕਰੰਸੀ ਸెలెక్టర్ ਫੀਲਡ ਵਿੱਚ ਪਹਿਲਾਂ ਹੀ ਚੁਣ ਦਿੰਦਾ ਹੈ। ਐਨੇਬਲ ਕਰਨ ਲਈ:

1. **Ultimate Multisite → Multi-Currency → Settings** 'ਤੇ ਜਾਓ।
2. **Detect currency from visitor location** ਨੂੰ ਐਨੇਬਲ ਕਰੋ।
3. **Geolocation Mapping** ਟੇਬਲ ਵਿੱਚ ਦੇਸ਼ਾਂ ਨੂੰ ਕਰੰਸੀ ਨਾਲ ਮੈਪ ਕਰੋ।

ਜੇ ਵਿਜ਼ਟਰ ਦੇ ਦੇਸ਼ ਲਈ ਕੋਈ ਮੈਪਿੰਗ ਨਹੀਂ ਹੈ, ਤਾਂ ਨੈੱਟਵਰਕ ਦੀ ਬੇਸ ਕਰੰਸੀ ਫਾਲਬੈਕ ਵਜੋਂ ਵਰਤੀ ਜਾਂਦੀ ਹੈ।

## Frequently Asked Questions

### What happens if an exchange rate provider is unavailable?

ਐਡਨੋਨ ਸਭ ਤੋਂ ਪਿਛਲੇ ਸਫਲਤਾਪੂਰਵਕ ਫੈਚ ਕੀਤੇ ਰੇਟਸ 'ਤੇ ਵਾਪਸ (fallback) ਜਾਂਦਾ ਹੈ। ਜੇ ਕਦੇ ਕੋਈ ਰੇਟ ਫੈਚ ਨਹੀਂ ਹੋਏ, ਤਾਂ ਇਹ ਤੁਹਾਡੇ ਦੁਆਰਾ ਕੰフィਗਰ ਕੀਤੇ ਮੈਨੂਅਲ ਰੇਟਸ 'ਤੇ ਵਾਪਸ ਜਾਂਦਾ ਹੈ। ਸੁਰੱਖਿਆ ਨੈੱਟ ਵਜੋਂ ਮੈਨੂਅਲ ਰੇਟਸ ਕੰフィਗਰ ਕਰਨਾ ਸਿਫਾਰਸ਼ ਕੀਤਾ ਜਾਂਦਾ ਹੈ।

### Can I use different providers for different currencies?

ਨਹੀਂ — ਸਾਰੀਆਂ ਕਰੰਸੀਆਂ ਲਈ ਇੱਕ ਵਾਰ ਵਿੱਚ ਇੱਕ ਪ੍ਰੋਵਾਈਡਰ ਸਰਗਰਮ ਹੁੰਦਾ ਹੈ। ਜੇ ਤੁਹਾਨੂੰ ਵਿਅਕਤੀਗਤ ਰੇਟਾਂ 'ਤੇ ਪੂਰਾ ਕੰਟਰੋਲ ਚਾਹੀਦਾ ਹੈ ਤਾਂ **Manual** ਦੀ ਵਰਤੋਂ ਕਰੋ।

### Does geolocation require a separate plugin?

ਕਿਸੇ ਵੱਖਰੇ ਜੀਓ-ਲੋਕੇਸ਼ਨ ਪਲੱਗਇਨ ਦੀ ਲੋੜ ਨਹੀਂ ਹੈ। ਐਡਨੋਨ ਵਰਡਪ੍ਰੈਸ ਦੇ 内-ਬਿਲਟ IP-to-country ਡਿਟੈਕਸ਼ਨ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ। ਵੱਧ ਸਹੀਤਾ ਲਈ, ਪ੍ਰਦਾਨ ਕੀਤੇ ਫਿਲਟਰ ਹੁਕਸ ਰਾਹੀਂ ਇੱਕ ਸਮਰਪਿਤ ਜੀਓ-ਲੋਕੇਸ਼ਨ ਸੇਵਾ ਨੂੰ ਇੰਟੀਗੇਟ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ।

### Are per-product overrides applied before or after tax?

ਪ੍ਰੋਡਕਟ-ਵਾਰ ਓਵਰਰਾਈਡ ਟਾਰਗੇਟ ਕਰੰਸੀ ਵਿੱਚ ਟੈਕਸ ਤੋਂ ਪਹਿਲਾਂ ਦੀ ਕੀਮਤ ਸੈੱਟ ਕਰਦੇ ਹਨ। ਟੈਕਸ ਤੁਹਾਡੀ ਮੌਜੂਦਾ ਟੈਕਸ ਕੰフィਗਰੇਸ਼ਨ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਓਵਰਰਾਈਡ ਕੀਮਤ 'ਤੇ ਗਣਨਾ ਕੀਤਾ ਜਾਂਦਾ ਹੈ।
