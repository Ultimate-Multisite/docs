# გაკვეთილი 10: გაშვების დღე

ყველაფერი აწყობილია. სანამ კარს გახსნით, ეს გაკვეთილი გაშვებამდე შესამოწმებელ სიას გაგატარებთ, რათა დარწმუნდეთ, რომ არაფერი არის გაფუჭებული, გამოტოვებული ან უხერხული.

## სად გავჩერდით

FitSite-ს უკვე კონფიგურირებული აქვს შაბლონები, გეგმები, გადახდა, ბრენდინგი, პირველადი დაყენება და ფასები. ახლა ვამოწმებთ, რომ ყველაფერი მუშაობს და გავდივართ პირდაპირ რეჟიმში.

## გაშვებამდე შესამოწმებელი სია

გაიარეთ თითოეული პუნქტი. არცერთი არ გამოტოვოთ.

### პლატფორმის ინფრასტრუქტურა

- [ ] ჰოსტინგი სტაბილურია და დატვირთვის ქვეშ კარგად მუშაობს
- [ ] Wildcard SSL აქტიურია და ყველა ქვედომენი HTTPS-ით იტვირთება
- [ ] დომენის მიბმა მუშაობს -- შეამოწმეთ საიტის შექმნა და custom დომენის მიბმა
- [ ] სარეზერვო ასლები კონფიგურირებული და შემოწმებულია (აღადგინეთ მინიმუმ ერთი, რომ გადაამოწმოთ)
- [ ] მონიტორინგი დაყენებულია -- გაიგებთ, თუ პლატფორმა გაითიშება

### შაბლონები

- [ ] სამივე შაბლონი სწორად იტვირთება ახალ საიტებზე
- [ ] დროებითი კონტენტი სასარგებლოა და ბეჭდვითი შეცდომებისგან თავისუფალია
- [ ] ყველა სურათს შესაბამისი ლიცენზია აქვს (stock photo-ს წყლის ნიშნების გარეშე)
- [ ] მობილურზე ადაპტირება მუშაობს შაბლონის ყველა გვერდზე
- [ ] გვერდის ჩატვირთვის სიჩქარე მისაღებია (შეამოწმეთ ისეთი ინსტრუმენტით, როგორიცაა GTmetrix ან PageSpeed Insights)
- [ ] არცერთ შაბლონზე არ არის გაფუჭებული ბმულები ან დაკარგული რესურსები

### გეგმები და პროდუქტები

- [ ] სამივე გეგმა აქტიურია და ჩანს
- [ ] გეგმების აღწერები ზუსტია და ნიშაზე მორგებული
- [ ] ფასები სწორია (თვიური და წლიური)
- [ ] დაყენების საფასურები სწორ გეგმებზეა კონფიგურირებული
- [ ] საცდელი პერიოდი მუშაობს Starter გეგმაზე
- [ ] შეკვეთაზე დამატებითი შეთავაზებები სწორად ჩანს გადახდისას
- [ ] plugin-ის და theme-ის შეზღუდვები თითოეული გეგმის მიხედვით სწორად სრულდება

### გადახდის პროცესი

- [ ] თითოეულ გეგმაზე შეასრულეთ სრული სატესტო რეგისტრაცია (გამოიყენეთ სატესტო გადახდის რეჟიმი)
- [ ] შაბლონის არჩევა აჩვენებს სწორ შაბლონებს თითოეული გეგმისთვის
- [ ] გადახდა წარმატებით მუშავდება
- [ ] მომხმარებელი რეგისტრაციის შემდეგ იღებს მისასალმებელ ელფოსტას
- [ ] ახალი საიტი იქმნება სწორი შაბლონით
- [ ] მომხმარებელს შეუძლია თავის ახალ საიტში დაუყოვნებლივ შესვლა
- [ ] ფასდაკლების კოდები სწორად მუშაობს

### ბრენდინგი

- [ ] შესვლის გვერდი აჩვენებს FitSite ბრენდინგს
- [ ] Admin Dashboard აჩვენებს FitSite ბრენდინგს
- [ ] ყველა სისტემური ელფოსტა იყენებს FitSite ბრენდინგს და ფიტნესზე მორგებულ ენას
- [ ] ინვოისები სწორად ჩანს თქვენი ბიზნესის დეტალებით
- [ ] მარკეტინგული საიტი პირდაპირ რეჟიმშია და გადახდის ფორმაზე ბმულებს შეიცავს

### პირველადი დაყენება

- [ ] სწრაფი დაწყების ვიჯეტი ჩანს ახალი მომხმარებლების Dashboard-ებზე
- [ ] სწრაფი დაწყების ყველა ბმული სწორ გვერდებზე მიუთითებს
- [ ] მისასალმებელი ელფოსტების სერია კონფიგურირებული და შემოწმებულია
- [ ] ცოდნის ბაზის სტატიები გამოქვეყნებულია და ხელმისაწვდომია
- [ ] Account გვერდი აჩვენებს სწორ ინფორმაციას გეგმის შესახებ და განახლების ვარიანტებს

### იურიდიული და ბიზნეს საკითხები

- [ ] მომსახურების პირობები გამოქვეყნებულია და გადახდის გვერდიდან მიბმულია
- [ ] კონფიდენციალურობის პოლიტიკა გამოქვეყნებულია და ხელმისაწვდომია
- [ ] თანხის დაბრუნების პოლიტიკა განსაზღვრული და დოკუმენტირებულია
- [ ] ბიზნეს სუბიექტი დაყენებულია გადახდების მისაღებად
- [ ] გადახდის gateway პირდაპირ რეჟიმშია (არა სატესტო რეჟიმში)
- [ ] საგადასახადო კონფიგურაცია სწორია თქვენი იურისდიქციისთვის

## რბილი გაშვება თუ სრული გაშვება

განიხილეთ ორფაზიანი გაშვება:

### ფაზა 1: რბილი გაშვება

საჯარო გაშვებამდე მოიწვიეთ 5-10 ფიტნეს სტუდიის მფლობელი რეგისტრაციისთვის. ესენი თქვენი ბეტა მომხმარებლები არიან. შესთავაზეთ მათ მნიშვნელოვანი ფასდაკლება (50% ფასდაკლება სამუდამოდ, ან 3 თვე უფასოდ) სანაცვლოდ:

- გულწრფელი უკუკავშირის მისაღებად რეგისტრაციისა და პირველადი დაყენების გამოცდილებაზე
- ნებართვისთვის, რომ მათი საიტი საჩვენებელ მაგალითად გამოიყენოთ
- ნებისმიერი ხარვეზის ან პრობლემის შეტყობინებისთვის, რომელსაც შეხვდებიან

ეს გაძლევთ რეალურ მომხმარებელთა უკუკავშირს და ცოცხალ საიტებს საჩვენებლად, სანამ საჯაროდ გაიხსნებით.

### ფაზა 2: საჯარო გაშვება

როცა რბილი გაშვების უკუკავშირი გათვალისწინებული იქნება:

- გადართეთ გადახდის gateway პირდაპირ რეჟიმში
- გამოაქვეყნეთ თქვენი მარკეტინგული საიტი
- დაიწყეთ მომხმარებლების მოზიდვა (გაკვეთილი 11)
- გამოაცხადეთ შესაბამის ფიტნეს ინდუსტრიის არხებზე

## გაშვების დღის მოქმედებები

იმ დღეს, როცა საჯაროდ გახვალთ:

1. **გადართეთ პირდაპირ გადახდებზე** -- გამორთეთ სატესტო რეჟიმი თქვენს გადახდის gateway-ზე
2. **კიდევ ერთხელ გადაამოწმეთ** -- შეასრულეთ სრული სატესტო რეგისტრაცია რეალური გადახდით (შემდეგ საკუთარ თავს თანხა დაუბრუნეთ)
3. **ყურადღებით აკონტროლეთ** -- დააკვირდით შეცდომებს, წარუმატებელ რეგისტრაციებს ან გადახდის პრობლემებს
4. **იყავით ხელმისაწვდომი** -- თქვენს პირველ რეალურ მომხმარებლებს შეიძლება დახმარება დასჭირდეთ, სწრაფი პასუხი კი ნდობას აყალიბებს
5. **მოკლედ იზეიმეთ** -- შემდეგ ისევ საქმეს დაუბრუნდით

## რა შეიძლება წავიდეს არასწორად

მზად იყავით შემდეგისთვის:

- **გადახდის gateway-ის პრობლემები**: მზად გქონდეთ თქვენი gateway-ის პროვაიდერის მხარდაჭერის საკონტაქტო ინფორმაცია
- **SSL სერტიფიკატის პრობლემები**: იცოდეთ, როგორ შეამოწმოთ და განაახლოთ სერტიფიკატები
- **ელფოსტის მიწოდების ჩავარდნები**: შეამოწმეთ, რომ ელფოსტები ნამდვილად მიდის (შეამოწმეთ სპამის საქაღალდეები)
- **დატვირთვის ქვეშ წარმადობა**: თუ ტრაფიკის მკვეთრი მატება გექნებათ, იცოდეთ, როგორ გააფართოოთ თქვენი ჰოსტინგი
- **მომხმარებლის დაბნეულობა**: მზად გქონდეთ ცოდნის ბაზა და მხარდაჭერის არხები
