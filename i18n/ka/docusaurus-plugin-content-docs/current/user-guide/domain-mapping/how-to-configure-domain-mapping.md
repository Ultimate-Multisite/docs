# დომეინის മാპინგის კონფიგურირება (v2)

_**მნიშვნელოვანი შენიშვნა: ეს სტატია განკუთვნილი Ultimate Multisite-ის ვერსია 2.x-ისთვის.**_

პრემიუმ ქსელის ერთ-ერთი ყველაზე ძლიერი ფუნქცია არის ჩვენი კლიენტებისთვის შესაძლებლობის შეთავაზება, რომ მათ საიტებზე ტოპ-ლეველ დომენი დააკავშირონ. ბოლოს და ბოლოს, რომელი გამოიყურება უფრო პროფესიონალურად: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) თუ [_**joesbikeshop.com**_](http://joesbikeshop.com)? სწორედ ამიტომ, Ultimate Multisite გთავაზობთ ამ ფუნქციას ჩაშენებულად, მესამე მხარის პლაგინების გამოყენების საჭიროების გარეშე.

## რა არის დომეინის മാპინგი?

როგორც სახელი მიუთითებს, დომეინის മാპინგი არის Ultimate Multisite-ის საშუალება, რომელიც ითხოვს დამატებითი დომენისთვის და ამ მოთხოვნას ქსელის შესაბამის საიტთან ასოცირებას უწევს იმ კონკრეტული დომენით.

### როგორ დავაყენოთ დომეინის മാპინგი თქვენს Ultimate Multisite ქსელში

დომეინის മാპინგისთვის საჭიროა თქვენი მხრიდან გარკვეული პარამეტრების დაყენება. საბედნიეროდ, Ultimate Multisite ავტომატიზაციით ამ რთულ სამუშაოს თქვენთვის აკეთებს, რათა ადვილად შევასრულოთ მოთხოვნები.

Ultimate Multisite-ის ინსტალაციის დროს, ვიზუალური მეგზური (wizard) ავტომატურად იკოპირებს და აყენებს **sunrise.php**-ს მითითებულ საქაღალდეში. **ვიზუალური მეგზური არ გაუშვებს შემდეგ ნაბიჯს მანამ, სანამ ეს ნაბიჯი არ დასრულდება**.

<!-- Screenshot unavailable: Ultimate Multisite ინსტალაციის ვიზუალური მეგზური sunrise.php-ით -->

ეს ნიშნავს, რომ როგორც კი Ultimate Multisite-ის ინსტალაციის ვიზუალური მეგზური თქვენი ქსელის დაყენებას დაასრულებს, შეგიძლიათ დაუყოვნებლივ დაიწყოთ კერძო დომენის മാპინგი.

შენიშვნა: Ultimate Multisite-ში დომეინის മാპინგი სავალდებულო არ არის. თქვენ გაქვთ ვარიანტი WordPress Multisite-ის ნატურ დომეინის മാპინგის ფუნქციის გამოყენების ან სხვა დომეინის മാპინგის ნებისმიერი გადაწყვეტის გამოყენების.

თუ გჭირდება Ultimate Multisite-ის დომეინის മാპინგის გამორთვა სხვა დომეინის მართვის გადაწყვეტილებების ადგილზე გასაცემად, შეგიძლიათ ეს ფუნქცია გაამორთოთ **Ultimate Multisite > Settings > Domain Mapping**-ში.

![Domain Mapping settings page showing admin redirect, mapping message and DNS options](/img/config/domain-mapping-settings.png)

ამ ვარიანტის ქვემოთ ასევე ნახავთ ვარიანტს **Force Admin Redirect**. ეს საშუალებას გაძლევთ დააკონტროლოთ, შეძლებს თუ არა თქვენი მომხმარებლები ადმინ პანელზე წვდომას როგორც საკუთარ დომეინსა და 서ბდომეინზე, თუ მხოლოდ ერთზე.

თუ აირჩევთ **Force redirect to mapped domain**-ს, თქვენს მომხმარებლებს შეეძლება ადმინ პანელის მიღება მხოლოდ საკუთარ დომეინებზე.

ვარიანტი **Force redirect to network domain** გააკეთებს ზუსტად საპირისპირო - თქვენს მომხმარებლებმა შეეძლება ადმინ პანელზე წვდომა მხოლოდ 서ბდომეინზე, მაშინაც კი, თუ ცდილობენ შესვლას საკუთარ დომეინებზე.

და ვარიანტი **Allow access to the admin by both mapped domain domain and network domain** მათ საშუალებას აძლევს ადმინ პანელზე წვდომა როგორც 서ბდომეინზე, ასევე საკუთარ დომეინზე.

![Admin Redirect dropdown expanded showing the three redirect options](/img/config/domain-mapping-redirect-options.png)

არსებობს ორი გზა საკუთარი დომეინის മാპინგისთვის. პირველი არის დომეინის სახელის დამატება თქვენი ქსელის ადმინ პანელიდან სუპერ ადმინისტრად, ხოლო მეორე არის 서ბსაიტის ადმინ პანელში ანგარიშის გვერდის ქვეშ.

მაგრამ სანამ დაიწყებთ საკუთარი დომეინის დამატებას თქვენი ქსელის რომელიმე 서ბსაიტზე, საჭიროა დარწმუნდეთ, რომ დომეინის **DNS პარამეტრები** სწორად არის კონფიგურირებული.

###

### დომეინის DNS პარამეტრების სწორი კონფიგურაციის უზრუნველყოფა

იმისათვის, რომ მაცნობა იმუშაოს, თქვენ უნდა დარწმუნდეთ, რომ დომენი, რომლის მიმართაც გეგმავთ დამაპროტესტაციას (mapping), მიუთითებს თქვენი ქსელის IP მისამართზე. გაითვალისწინეთ, რომ გჭირდებათ ქსელის IP მისამართი - ეს არის იმ დომენის IP მისამართი, სადაც Ultimate Multisiteა დაინსტალირებული, და არა თქვენი საკუთარი დომეინის (custom domain) IP მისამართი, რომლის მიმართაც გსურთ მაცნობის გაკეთება. კონკრეტული დომენის IP მისამართის ძებნისთვის, გირჩევთ გადახვიდეთ [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html), მაგალითად.

დომენის სწორად მაცნობისთვის, თქვენ უნდა დაამატოთ **A RECORD** თქვენს **DNS** კონფიგურაციაში, რომელიც მიუთითებს ამ **IP მისამართზე**. DNS-ის მართვა ძალიან განსხვავებულია სხვადასხვა დომენი რეგისტრატორის მიხედვით, მაგრამ ინტერნეტში ბევრი ტუტორიალი არსებობს ამ საკითხზე, თუ მოძებნით " _Creating A Record on XXXX_ ", სადაც XXXX არის თქვენი დომენი რეგისტრატორი (მაგალითად: " _Creating A Record on_ _GoDaddy_ ").

თუ გეწყებათ პრობლემა ამის დადებისას, **დაუკავშირდით თქვენი დომენი რეგისტრატორის მხარდაჭერას** და მათ შეძლებენ დაგეხმარონ ამ ნაწილში.

თუ გეგმავთ, რომ თქვენს კლიენტებს თავიანთ დომენების მაცნობის უფლება მისცეთ, მათ ეს ნაწილი თავად უნდა გააკეთონ. თუ ვერ ახერხებენ A Record-ის შექმნას, მიმართეთ მათ რეგისტრატორის მხარდაჭერის სისტემას.

### საკუთარი დომეინის მაცნობა სუპერ ადმინისტრად

როდესაც თქვენ შედიხართ ქსელში სუპერ ადმინისტრად, შეგიძლიათ მარტივად დაამატოთ და მართოთ საკუთარი დომენები **Ultimate Multisite > Domains**-ის ქვეშ.

![Domains list page in Ultimate Multisite](/img/admin/domains-list.png)

ამ გვერდის ქვეშ შეგიძლიათ ზემოთ არსებული **Add Domain** ღილაკზე დაწკაპუნოთ, რაც გამოგიტანთ მოდალურ ფანჯარას, სადაც შეგიძლიათ დაადგინოთ და შეავსოთ **custom domain name** (სპეციალური დომენის სახელი), **the subsite** (ქვესაიტი), რომელზეც გსურთ სპეციალური დომენის გამოყენება, და გადაწყვიტოთ, დადოთ ის როგორც **primary domain** (ძირითადი დომენი) თუ არა (შენიშვნა: შეგიძლიათ ერთი ქვესაიტზე რამდენიმე დომენი დაუკავშიროთ).

![Add Domain მოდალი დომენის სახელით, საიტის არჩევის და ძირითადი დომენის გადამრთველთან ერთად](/img/admin/domain-add-modal.png)

ყველა ინფორმაციის შეყვანის შემდეგ შეგიძლიათ დააწკაპუნოთ ქვედა ნაწილში არსებულ **Add Existing Domain** ღილაკზე.

ეს დაიწყებს პროცესს, რომელიც მოიცავს სპეციალური დომენის DNS ინფორმაციის გადამოწმებასა და აღებას. გვერდის ბოლოში ასევე გამოჩნდება лог (log), რომელშიც შეგიძლიათ მიჰყვეთ პროცესს. ამ პროცესს შეიძლება რამდენიმე წუთი დასჭირდეს დასრულებისთვის.

Ultimate Multisite v2.13.0 ავტომატურად ქმნის შინაგან დომენის ჩანაწერს ახალი საიტის შექმნისას ჰოსტზე, რომელიც უნდა განიხილებოდეს როგორც per-site (საიტზე დაფუძნებული) დომენი. თუ ჰოსტი არის ქსელის ძირითადი დომენი ან ერთ-ერთი საერთო checkout-form ბაზის დომენები, რომლებიც კონფიგურირებულია **Site URL** (საიტის მისამართი) ველიში, ავტომატურად დაუკავშირდება დომენი ჩანაწერი გამოტოვდება, რათა საერთო ბაზის დომენი ხელმისაწვდომი დარჩეს ყველა საიტისთვის, რომელიც მას იყენებს.

**Stage** (ეტაპი) ან სტატუსი უნდა შეიცვალოს **Checking DNS**-დან **Ready**-ზე, თუ ყველაფერი სწორად არის დაყენებული.

თუ დომენის სახელს დააწკაპუნებთ, შეძლებთ მის შიგნით რამდენიმე ვარიანტის ნახვას. მოდით, სწრაფად გადავხედოთ მათ:

![Domain detail page with stage, site, active, primary and SSL toggles](/img/admin/domain-edit.png)

**სტადი:** ეს არის ის სტადია, რომელზეც დომენი იმყოფება. როდესაც პირველად დაამატებთ დომენს, სავარაუდოდ, ის იქნება **DNS-ის შემოწმების** სტადიაზე. პროცესი შეამოწმებს DNS-ის ჩანაწერებს და დაადასტურებს, რომ ისინი სწორია. შემდეგ, დომენი გადავა **SSL-ის შემოწმების** სტადიაზე. Ultimate Multisite შეამოწმებს, აქვს თუ არა დომენს SSL, და დაასახელებს თქვენს დომენს როგორც **Ready** (გამზავნი) ან **Ready (without SSL)** (გამზავნი SSL-ის გარეშე).

**საიტი:** ეს არის ქვედომენი, რომელიც დაკავშირებულია ამ დომენთან. დაუკავშირებელი დომენი აჩვენებს ამ კონკრეტული საიტის შინაარსს.

**აქტიური (Active):** შეგიძლიათ ჩართოთ ან გამორთოთ ეს ვარიანტი დომენის გასააქტიურებლად ან გათიშვისთვის.

**არის პრიმიარული დომენი? (Is Primary Domain?):** თქვენი მომხმარებლებისთვის შეიძლება იყოს ერთი და მეტი დაკავშირებული დომენი თითოეულ საიტსთან. გამოიყენეთ ეს ვარიანტი იმის შესარჩევად, არის თუ არა ეს პრიმიარული დომენი კონკრეტული საიტისთვის.

**აქვს თუ არა უსაფრთხოება? (Is Secure?):** მიუხედავად იმისა, რომ Ultimate Multisite იწყებს შემოწმებას, აქვს თუ არა დომენს SSL-ის სერტიფიკატი და მისი გააქტიურების საშუალება, თქვენ შეგიძლიათ ხელით აირჩიოთ დომენის ჩატვირთვა SSL-ით ან მის გარეშე. ყურადღება მიაქციეთ, რომ თუ ვებგვერდზე არ არის SSL სერტიფიკატი და ცდილობთ ის ძალით ჩაიტვირთოთ SSL-ით, შესაძლოა შეცდომები გამოგიტანოს.

### მორგებული დომენის სახელი ქვესაიტად მომხმარებლისთვის

### დომეინების სინქრონიზაცია (About Domain Syncing)

დომეინების სინქრონიზაცია არის პროცესი, რომლის დროსაც Ultimate Multisite თქვენს ჰოსტინგის ანგარიშში додаებს საკუთარ დომეინს როგორც add-on დომე인을 **ტომეინის მუშაობისთვის**.

დომეინების ავტომატური სინქრონიზაცია ხდება მაშინ, როდესაც თქვენს ჰოსტინგ პროვაიდერთან Ultimate Multisite-ის დომეინის მითითების ფუნქციას აქვს ინტეგრაცია. ამჟამად, ეს ჰოსტინგ პროვაიდერები არიან _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways_ და _Cpanel._

როდესაც ჰოსტინგ პროვაიდერთან ინტეგრაცია აქტიურია, Ultimate Multisite ასევე შეძლებს ახლად შექმნილ საიტებისთვის პროვაიდერის მხარის DNS-ის ან ქვედომეინის შექმნის დავალების ჩატვირთვას (enqueue). თუ არცერთი ინტეგრაცია არ უსმენს ამ დავალებას, ბექგრაუნდული ჯოგი გამოტოვდება, რათა თავიდან იქნეს აცილებული no-op რიგის ნივთები. მითითებულ დომეინებზე DNS და SSL შემოწმებები კვლავ გაგრძელდება ნორმალურ დომეინის სტედის პროცესის გავლით.

ეს ინტეგრაცია უნდა ჩართოთ Ultimate Multisite-ის პარამეტრებში **Integration** (ინტეგრაციები) ჩან்தა.
