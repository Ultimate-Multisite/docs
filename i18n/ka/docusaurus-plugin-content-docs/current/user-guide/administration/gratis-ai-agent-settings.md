# უფასო AI Agent პარამეტრები

**Settings → Advanced** صفحه Gratis AI Agent-ში ადმინისტრატორებისთვის განკუთვნილია ბექენდ ინტეგრაციების კონფიგურირებისთვის, რომლებიც შევიდა v1.5.0-ით. ეს გვერდი დოკუმენტირებულია **Feedback Endpoint** ველი და მისი მოსალოდნელი ფორმატის შესახებ.

## პარამეტრებზე წვდომა

1. WordPress ადმინში გადადით **Gratis AI Agent → Settings**-ზე.
2. დააწკაპუნეთ **Advanced** ტაბზე.

## Feedback Endpoint კონფიგურაცია

Feedback endpoint იღებს POST მოთხოვნებს AI agent-ისგან ყოველთვის, როდესაც მომხმარებელი აძლევს უკუკავშირს თითების ქვემოთ არსებული ღილაკით, ავტომატური პრომპტის ბანერით ან `/report-issue` ბრძანებით.

| ველი | აღწერა |
|---|---|
| **Feedback Endpoint URL** | URL, რომელიც იღებს უკუკავშირის შეტანას HTTP POST მოთხოვნების სახით JSON სხეულით. |
| **Feedback API Key** | ბერერი ტოკენი, რომელიც იგზავნება თითოეულ უკუკავშირზე `Authorization` ჰედერში. დატოვეთ ცარიელი, თუ თქვენს endpoint-ს ავთენტიფიკაცია არ სჭირდება. |

### მოსალოდნელი JSON პაილოდი (Payload)

თქვენს feedback endpoint-მა უნდა მიიღოს JSON სხეული, რომელშიც მინიმუმ შემდეგი ველი იქნება შეტანილი:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "პასუხი არ იყო სწორი ფასებზე.",
  "triage_category": "factual_error"
}
```

დამატებითი ველი შეიძლება იყოს პაილოდში კონტექსტის მიხედვით.

### `triage_category` მნიშვნელობები

AI ტრიჟის შრე (triage layer) პაილოდის გადაცემის წინ `triage_category`-ს ერთ-ერთ შემდეგ მნიშვნელობას ანიჭებს:

| მნიშვნელობა | მნიშვნელობა |
|---|---|
| `factual_error` | ასისტენტმა არასწორი ფაქტობრივი ინფორმაცია მოგვაწოდა. |
| `unhelpful_answer` | პასუხი ტექნიკურად სწორი იყო, მაგრამ სასარგებლო არ იყო. |
| `inappropriate_content` | პასუხში შეიცავდა კონტენტს, რომელიც მომხმარებლებისთვის არ უნდა იყოს ნაჩვენები. |
| `other` | უკუკავშირმა არ შეესაბამა ცნობილ კატეგორიას. |

### ავთენტიფიკაცია

თუ თქვენი endpoinტი ავтентификаციას მოითხოვს, დაადეთ **Feedback API Key** ველი თქვენს bearer token-ზე. აგენტი გამოგიგზავნით:

```
Authorization: Bearer <your-api-key>
```

თუ **Feedback API Key** ველი ცარიელია, `Authorization` ჰედერი არ გაიგზავნება.

### Feedback-ის შეწყვიტვა

დატოვეთ როგორც **Feedback Endpoint URL**, ასევე **Feedback API Key** ველი ცარიელი. მომხმარებლებისთვის თითების ქვედა ღილაკი და feedback UI კვლავ იქნება ხილული, მაგრამ მონაცემები არ გადაეცემა რაიმე გარე სერვისს.

## Brave Search API Key

ასევე **Advanced** ტაბში, **Brave Search API Key** ველი ირთავს [Internet Search-ის](https://yourdomain.com/configuration/internet-search) შესაძლებლობას.

| ველი | აღწერა |
|---|---|
| **Brave Search API Key** | თქვენი API გასაღები Brave Search დეველოპერების დაფიდან. საჭიროა ინტერნეტ ძებნის ჩართვისთვის AI ასისტენტში. |

ველის წარწერაში მოცემულია გადა link-ი Brave Search-ის სალარმო გვერდზე. ცარიელი დატოვეთ, რომ ინტერნეტ ძებნა გამორთოთ.

ამ ფუნქციაზე end-user დოკუმენტაციისთვის ნახეთ [Internet Search](../configuration/internet-search).

## Feature Flags (ფუნქციური დროის გადამრთველები)

ეს ფუნქცია შევიდა v1.9.0-თან ერთად, **Settings → Feature Flags** ტაბი გთავაზობთ optional ფუნქციონალისთვის toggle switch-ებს. თითოეული флаგი either ჩართულია ან გამორთულია მთელ ქსელში; ამ ეტაპზე არ არსებობს per-site გადალახვის საშუალება.

### Feature Flags-ის მიღება

1. WordPress ადმინში, გადადით **Gratis AI Agent → Settings**-ზე.
2. დააწკაპუნეთ **Feature Flags** ტაბზე.

### Access Control Flags (წვდომის კონტროლის ფლაგები)

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Ограничить администраторами** | Выкл | При включении только пользователи с ролью `administrator` смогут открыть панель чата AI Agent. Остальные роли увидят сообщение «Свяжитесь с вашим администратором». |
| **Ограничить сетевыми администраторами** | Выкл | В многосайтовой сети, если этот флаг включен, агент могут использовать только Суперадминистраторы. Администраторы отдельных сайтов блокируются. Этот параметр имеет приоритет над «Ограничить администраторами», если оба включены. |
| **Разрешить доступ подписчикам** | Выкл | При включении пользователи с ролью `subscriber` смогут пользоваться частым интерфейсом, но их возможности будут ограничены только чтением (нет создания постов или изменения настроек). |
| **Отключить для не-участников** | Выкл | Интегрируется со статусом членства Ultimate Multisite. При включении чат скрывается на сайтах, у которых нет активного членства. |

### Флаги брендинга

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Скрыть подвал «Powered by Gratis AI Agent»** | Выкл | Удаляет строку с указанием бренда внизу виджета чата. Рекомендуется для развертываний с белым брендом (white-label). |
| **Название агента по умолчанию** | *(пусто)* | Заменяет стандартную метку «Gratis AI Agent» в заголовке чата и меню администратора на ваше собственное название продукта. Оставьте пустым, чтобы использовать значение по умолчанию. |
| **Скрыть селектор агентов** | Выкл | При включении пользователи не смогут переключаться между пятью встроенными агентами. Текущий агент будет зафиксирован на том, который настроен как основной в Настройках → Общие. |
| **Использовать иконку сайта в качестве аватара чата** | Выкл | Заменяет стандартную иконку AI в заголовке виджета чата на иконку вашего WordPress-сайта (настраивается в Внешний вид → Настроить → Идентичность сайта). |

### Применение изменений

Нажмите **Сохранить настройки** после переключения любого флага. Изменения вступают в силу немедленно — перезагрузка кэша или повторная активация плагина не требуются.
