# Gratis AI Agent-ის პარამეტრები

**პარამეტრები → გაფართოებული** ეკრანი Gratis AI Agent-ში უზრუნველყოფს ადმინისტრატორის დონის კონფიგურაციას backend ინტეგრაციებისთვის. ეს გვერდი აღწერს feedback-ის გადამისამართებას, ძიების პროვაიდერის გასაღებებს, მართულ Superdav სერვისის დაყენებას, Google Calendar-ის კონტროლებს, TextBee SMS პარამეტრებს და ქსელის მასშტაბით მოქმედ ფუნქციურ ალმებს.

## პარამეტრებზე წვდომა

1. WordPress-ის ადმინისტრაციაში გადადით **Gratis AI Agent → პარამეტრები**.
2. დააწკაპუნეთ **გაფართოებული** ჩანართზე.

## Feedback endpoint-ის კონფიგურაცია

Feedback endpoint იღებს POST მოთხოვნებს AI აგენტისგან, როდესაც მომხმარებელი feedback-ს აგზავნის thumbs-down ღილაკით, ავტომატური prompt-ის ბანერით ან `/report-issue` ბრძანებით.

| ველი | აღწერა |
|---|---|
| **Feedback Endpoint URL** | URL, რომელიც იღებს feedback გაგზავნებს HTTP POST მოთხოვნების სახით JSON body-ით. |
| **Feedback API Key** | bearer token, რომელიც იგზავნება თითოეული feedback მოთხოვნის `Authorization` header-ში. დატოვეთ ცარიელი, თუ თქვენს endpoint-ს ავთენტიფიკაცია არ სჭირდება. |

### მოსალოდნელი JSON Payload

თქვენმა feedback endpoint-მა უნდა მიიღოს JSON body მინიმუმ შემდეგი ველებით:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

payload-ში შეიძლება იყოს დამატებითი ველები საუბრის კონტექსტის მიხედვით.

### `triage_category` მნიშვნელობები

AI triage ფენა payload-ის გადამისამართებამდე `triage_category`-ს ანიჭებს ერთ-ერთ შემდეგ მნიშვნელობას:

| მნიშვნელობა | მნიშვნელობა |
|---|---|
| `factual_error` | ასისტენტმა მიაწოდა არასწორი ფაქტობრივი ინფორმაცია. |
| `unhelpful_answer` | პასუხი ტექნიკურად სწორი იყო, მაგრამ სასარგებლო არ იყო. |
| `inappropriate_content` | პასუხი შეიცავდა კონტენტს, რომელიც მომხმარებლებს არ უნდა ეჩვენოთ. |
| `other` | feedback არ დაემთხვა ცნობილ კატეგორიას. |

### ავთენტიფიკაცია

თუ თქვენს endpoint-ს ავთენტიფიკაცია სჭირდება, **Feedback API Key** ველში მიუთითეთ თქვენი bearer token. აგენტი აგზავნის:

```
Authorization: Bearer <your-api-key>
```

თუ **Feedback API Key** ველი ცარიელია, `Authorization` header არ იგზავნება.

### Feedback-ის შეგროვების გამორთვა

ორივე ველი, **Feedback Endpoint URL** და **Feedback API Key**, დატოვეთ ცარიელი. thumbs-down ღილაკი და feedback-ის UI მომხმარებლებისთვის ხილული დარჩება, მაგრამ გაგზავნები არცერთ გარე სერვისზე არ გადაიგზავნება.

## Brave Search API Key

ასევე **გაფართოებული** ჩანართზე, **Brave Search API Key** ველი ააქტიურებს [ინტერნეტში ძიების](../configuration/internet-search) შესაძლებლობას.

| ველი | აღწერა |
|---|---|
| **Brave Search API Key** | თქვენი API გასაღები Brave Search-ის დეველოპერის Dashboard-იდან. საჭიროა AI ასისტენტში ინტერნეტში ძიების ჩასართავად. |

ველის ეტიკეტი შეიცავს დაწკაპებად ბმულს Brave Search API-ის რეგისტრაციის გვერდზე. დატოვეთ ცარიელი ინტერნეტში ძიების გამოსართავად.

ამ ფუნქციის საბოლოო მომხმარებლის დოკუმენტაციისთვის იხილეთ [ინტერნეტში ძიება](../configuration/internet-search).

## მართული Superdav სერვისი

Superdav AI Agent v1.18.0 ამატებს მართული Superdav სერვისის endpoint-ებს და ავტომატურ კავშირის provision-ს მხარდაჭერილი საიტებისთვის. გამოიყენეთ ეს კონტროლები, როდესაც თქვენი საიტი უნდა დაუკავშირდეს hosted პროვაიდერს ხელით კონფიგურირებული სერვისის endpoint-ის ნაცვლად.

| ველი | აღწერა |
|---|---|
| **მართული Superdav სერვისი** | რთავს hosted Superdav სერვისის კავშირს მხარდაჭერილი საიტებისთვის. |
| **კავშირის Provision** | იწყებს endpoint-ისა და credential-ების ავტომატურ provision-ს. გამოიყენეთ ეს მას შემდეგ, რაც დაადასტურებთ, რომ საიტმა მართული პროვაიდერი უნდა გამოიყენოს. |
| **სერვისის Endpoint / კავშირის სტატუსი** | აჩვენებს მიმდინარე endpoint-ს ან კავშირის მდგომარეობას provision-ის შემდეგ. |

provision-ის შემდეგ შეინახეთ პარამეტრები და გადაამოწმეთ კავშირის სტატუსი, სანამ მართული სერვისის workflow-ებზე დაყრდნობას დაიწყებთ. თუ provision ვერ შესრულდა, გადახედეთ ნაჩვენებ ხელახლა ცდის მითითებებს და დაადასტურეთ, რომ საიტს hosted პროვაიდერის გამოყენების ნებართვა აქვს.

## Google Calendar-ის კონფიგურაცია

როდესაც Superdav AI Agent v1.18.0-ის კალენდრის ფუნქციები ჩართულია, აგენტს შეუძლია კონფიგურირებული კალენდრებისა და ღონისძიების დეტალების წაკითხვა. კალენდრის ხელსაწყოები წაკითხვაზეა ორიენტირებული და სასარგებლოა განრიგის გათვალისწინებით შეხსენებებისთვის, მონაწილეებთან follow-up-ისთვის და კონტაქტების შესატყვისად.

| ველი | აღწერა |
|---|---|
| **Google Calendar-ის Credential-ები** | ინახავს credential-ებს ან token-ის კავშირს, რომელიც საჭიროა კალენდრის მონაცემების წასაკითხად. |
| **კალენდრის არჩევა** | ზღუდავს, რომელ კონფიგურირებულ კალენდრებს შეიძლება გაეცნოს აგენტი. |
| **კალენდრის კავშირის სტატუსი** | ადასტურებს, შეუძლია თუ არა მიმდინარე credential-ებს კალენდრებისა და ღონისძიებების წაკითხვა. |

კალენდრის credential-ები შეზღუდეთ მხოლოდ იმ კალენდრებით, რომლებიც აგენტს სჭირდება. ხელახლა დააკავშირეთ ან შეცვალეთ credential-ები, თუ სტატუსი მიუთითებს ვადაგასულ token-ზე.

## TextBee SMS შეტყობინებები

Superdav AI Agent v1.18.0 ამატებს TextBee-ს, როგორც SMS პროვაიდერს კონფიგურირებული შეტყობინებების workflow-ებისთვის. SMS შეტყობინებები უნდა დაწყვილდეს ადამიანის დამტკიცების კარიბჭეებთან მგრძნობიარე ან მომხმარებლისთვის ხილული შეტყობინებებისთვის.

| ველი | აღწერა |
|---|---|
| **TextBee API Key** | ახორციელებს მოთხოვნების ავთენტიფიკაციას TextBee SMS პროვაიდერთან. |
| **TextBee მოწყობილობა / გამგზავნი** | ირჩევს TextBee-ის გამგზავნს ან მოწყობილობას გამავალი შეტყობინებებისთვის, როდესაც ამას პროვაიდერი მოითხოვს. |
| **SMS შეტყობინებები ჩართულია** | დამტკიცებულ workflow-ებს აძლევს ტექსტური შეტყობინებების გაგზავნის საშუალებას. დატოვეთ გამორთული SMS გაგზავნების ასარიდებლად. |

გაგზავნეთ საცდელი შეტყობინება მხოლოდ ადმინისტრატორის მფლობელობაში არსებულ ნომერზე, შემდეგ დაადასტურეთ approval-gate-ის ქცევა, სანამ დაგეგმილ ან მონაწილეებისთვის განკუთვნილ შეხსენებებს ჩართავთ.

## ფუნქციური ალმები

ასევე v1.9.0-ში წარმოდგენილი **პარამეტრები → ფუნქციური ალმები** ჩანართი უზრუნველყოფს გადამრთველებს არასავალდებულო ფუნქციონალისთვის. თითოეული ალამი ან ჩართულია, ან გამორთულია ქსელის მასშტაბით; ამჟამად ცალკეული საიტისთვის override არ არსებობს.

### ფუნქციურ ალმებზე წვდომა

1. WordPress-ის ადმინისტრაციაში გადადით **Gratis AI Agent → პარამეტრები**.
2. დააწკაპუნეთ **ფუნქციური ალმები** ჩანართზე.

### წვდომის კონტროლის ალმები

| დროშა | ნაგულისხმევი | აღწერა |
|---|---|---|
| **მხოლოდ ადმინისტრატორებისთვის შეზღუდვა** | გამორთულია | ჩართვისას AI Agent ჩატის პანელის გახსნა მხოლოდ `administrator` როლის მქონე მომხმარებლებს შეუძლიათ. ყველა სხვა როლი ამის ნაცვლად ხედავს შეტყობინებას „დაუკავშირდით თქვენს ადმინისტრატორს“. |

| **არაწევრებისთვის გამორთვა** | გამორთულია | ინტეგრირდება Ultimate Multisite წევრობის სტატუსთან. ჩართვისას ჩატი იმ საიტებისთვის იმალება, რომლებსაც აქტიური წევრობა არ აქვთ. |
