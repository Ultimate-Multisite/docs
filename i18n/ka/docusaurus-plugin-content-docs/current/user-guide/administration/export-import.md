# ექსპორტი და იმპორტი {#export--import}

Ultimate Multisite 2.9.0-ის მეორე დონეზე, **Tools > Export & Import**-ის ქვეშ დაემატა ერთი-საიტური ინსტრუმენტი **Export & Import**. გამოიყენეთ ის მაშინ, როდესაც გჭირდებათ ერთი WordPress საიტის შეფუთვა ZIP ფაილად, ამ ZIP-ის აღდგენა ან საიტის გადატანა თავსებადი Ultimate Multisite და სინგლ-საიტური WordPress ინს stallაციებს შორის.

## საჭირო უფლებები {#required-permissions}

თქვენ უნდა შეხვიდეთ ადმინისტრატორის სახელით, რომელსაც აქვს წვდომა WordPress-ის **Tools** მენიუზე იმ საიტზე, რომლის ექსპორტიც გჭირდებათ ან იმ საიტზე, რომლის იმპორტიც გჭირდებათ. მულტისೈტური ქსელისთვის, როდესაც იყენებთ ქსელის დონის Ultimate Multisite ინსტრუმენტებს სუბსაიტების ექსპორტის ან იმპორტისას, გამოიყენეთ ქსელის ადმინისტრატორის ანგარიში.

Export ZIP-ის ჩამოტვირთვები მიწოდებულია ავთენტიფიცირებულ ჩამოტვირთვის終პოინტზე, ამიტომ შეინარჩუნეთ ადმინისტრაციის სესია აქტიური მანამ, სანამ ჩამოტვირთვა არ დასრულდება და არ გააზიარებთ გენერირებულ ჩამოტვირთვის URL-ებს საჯაროდ.

## საიტის ZIP-ად ექსპორტირება {#exporting-a-site-to-a-zip}

1. იმ WordPress ადმინში გადადით, რომელ საიტზეც გსურთ კოპირება, **Tools > Export & Import**-ზე.
2. გახსენით ექსპორტის ზონა და აირჩიეთ ის საიტი, რომლის შეფუთვაც გსურთ.
3. აირჩიეთ სურვილისამებრ კონტენტი, რომელიც უნდა შევიდეს, მაგალითად, აப்ლოუდები, პლაგინები და თემები, როდესაც ეს ვარიანტები ხელმისაწვდომია.
4. დაიწყეთ ექსპორტი და დაელოდეთ პროცესის დასრულებას. დიდი საიტებისთვის შეიძლება საჭირო იყოს ფონური პროცესების დასრულება ZIP-ის მზადებისთვის.
5. ჩამოტვირთეთ დასრულებული ZIP ექსპორტის სიაში.

შეინახეთ ZIP უსაფრთხო ადგილას. მასში შეიძლება შევიდეს საიტის კონტენტი, პარამეტრები, მედია ფაილები და შერჩეული კოდის აქტივები.

## რა შედის ექსპორტში {#what-the-export-includes}

ექსპორტის ZIP-ში შეიძლება შევიდეს:

- არჩეული საიტის მონაცემთა ბაზის კონტენტი და კონფიგურაცია.
- აப்ლოუდური მედია ფაილები, თუ ჩართულია აப்ლოუდები.
- პლაგინები და თემები, თუ ეს ვარიანტები შერჩეულია.
- იმპორტის მეტამონაცემები, რომელსაც Export & Import ინსტრუმენტი იყენებს საიტის აღდგენისთვის მიზნობრივ ინს stallაციაზე.

ზუსტი ZIP ფაილის ზომა დამოკიდებულია მედიის რაოდენობაზე, შერჩეულ პლაგინებზე და თემებზე, ასევე საიტის მონაცემთა ბაზის ცხრილების ზომაზე.

## საიტის იმპორტი ZIP-დან {#importing-a-site-from-a-zip}

1. გადადით **Tools > Export & Import**-ზე მიზნობრივი WordPress საიტზე.
2. გახსენით იმპორტის განყოფილება და ატვირთეთ ZIP ფაილი, რომელიც Export & Import ინსტრუმენტი შექმნა.
3. შეიყვანეთ სასურველი URL-ის ჩანაცვლება, თუ საიტმა ახალი მისამართი უნდა გამოიყენოს, ან დატოვეთ ველი ცარიელი, რომ ორიგინალი URL შენარჩუნდეს.
4. აირჩიეთ, წაიშალოს თუ არა ატვირთული ZIP ფაილი იმპორტის დასრულების შემდეგ.
5. დააჭირეთ **Begin Import**-ს (იმპორტის დაწყებას).
6. აკონტროლეთ შეფერხებული იმპორტი, სანამ ის არ დასრულდება. გამოიყენეთ **Cancel Import**, მხოლოდ იმ შემთხვევაში, თუ გსურთ პროცესის გაჩერება დასრულებამდე.
7. გადახედეთ იმპორტირებულ საიტს დაუშვით ნორმალური ტრაფიკი ან მომხმარებლების წვდომა.

ერთ-საიტზე განთავსებული WordPress ინსტალაციაში, ZIP ფაილის იმპორტი არსებულ საიტს ჩანაცვლებს იმპორტირებულ საიტთან. დაიწყეთ მიზნობრივი საიტის სრული ბექაპის შექმნა და თავიდან აიცილეთ ერთი და იმავე საიტისთვის ერთდროულად მრავალი იმპორტის დაწყება.

## შეზღუდვები და თავსებადობის შენიშვნები {#limitations-and-compatibility-notes}

ძალიან დიდი აპლოუდების დირექტორიები ან მედია ბიბლიოთეკები შეიძლება წარმოქმნას დიდი ZIP ფაილები. დიდი საიტების ექსპორტის ან იმპორტის წინ დაადასტურეთ PHP-ის აპლოუდების ლიმიტები, შესრულების ლიმიტები, დისკზე თავისუფალი ადგილი, მეხსიერება და სერვერის ტაი-აუტი (timeout) პარამეტრები.
ძალიან დიდი მედია ბიბლიოთეკების გადატანა შესაძლოა საჭირო იყოს დაბალტრაფიკის შენარჩუნებისას შენარჩუნებული დროის ფანჯრის დროს.
მიზნობრივი WordPress ინსტალაცია უნდა გაუშვებდეს თავსებად WordPress-ს, PHP-ს, Ultimate Multisite-ს, პლაგინებსა და თემებს.
ერთ-საიტური იმპორტი იცვლის მიზნობic საიტს. ეს არ არის შერწყმის ინსტრუმენტი.
Multisite-დან single-site-ზე გადასვლები და single-site-დან Multisite-ზე გადასვლები მხარდაჭერილია მხოლოდ იმ შემთხვევაში, თუ მიზნობრივი გარემო შეუძლია გაუშვას ექსპორტირებული საიტის პლაგინები, თემები, URL-ები და საჭირო Ultimate Multisite კომპონენტები.
ZIP ფაილს დააკავშირეთ. მას შეიძლება აერთიანებდეს მონაცემთა ბაზის შინაარსს, ატვირთულ ფაილებს და ექსპორტირებული საიტის კონფიგურაციის დეტალებს.

უფრო ძველ ქსელური ექსპორტის სამუშაო პროცესებზე იხილეთ [Site Export](/user-guide/administration/site-export).
