# საიტის ექსპორტი

**Site Export** ადმინ პრეზენტაცია საშუალებას გაძლევთ ერთ საიტს ან მთელ ქსელს შექმნათ ჩამოტვირთვისთვის ხელმისაწვდომ არქივი, რაც საჭიროა მიგრაციისთვის, ბექაპისთვის ან გადაცემის სამუშაოებისთვის.

## ერთი საიტის ექსპორტი {#exporting-one-site}

გადადით **Ultimate Multisite > Site Export**-ზე და აირჩიეთ **Generate new Site Export**. შემდეგ აირჩიეთ ქვე-საიტი, რომლის ექსპორტიც გსურთ, და განსაზღვრეთ, შეიცავს თუ არა არქივში აப்ლოუდებს (uploads), პლაგინებს და თემებს.

ექსპორტის დასრულების შემდეგ, ჩამოტვირთეთ ZIP ფაილი **Existing Exports** სიიდან. ამ ექსპორტი ZIP-ს ახლა აქვს საკუთარი ავტომატური გაშვებისთვის საჭირო `index.php` და `readme.txt`, რათა არქივი შეძლოს ახალ ჰოსტზე ატვირთვა და პირველივე uruchომლებით, სანამ ცალკე იმპორტის პლაგინის ინსტალირებას მოგიწევთ.

## მთელი ქსელის ექსპორტი {#exporting-the-whole-network}

გამოიყენეთ **Network Export** Site Export გვერდზე, როდესაც გჭირდებათ ერთი არქაივი, რომელიც შეიცავს ქსელის ყველა ქვე-საიტს. ეს სასარგებლოა ჰოსტის მიგრაციის წინ, კატასტროფის აღდგენის სავარჯიშოებისთვის ან სტეიჯინგის გადაკეთების დროს, როდესაც თითოეულ ქვე-საიტს ერთად უნდა მო travel-ი.

რადგან ქსელის ექსპორტი შეიძლება გაცილებით დიდი იყოს, ვიდრე ერთი საიტის ექსპორტი, ატვირთვებისთვის, პლაგინებისთვის, თემებისთვის და გენერირებული ZIP ფაილებისთვის საკმარისი თავისუფალი ადგილის არსებობის დადასტურება განახლებადი ტარგი (low-traffic window) დროს გააკეთეთ.

### ქსელის იმპორტის ბუნდლები (Network Import Bundles) {#network-import-bundles}

Ultimate Multisite 2.12.0-დან, Site Exporter-მა შეძლებს **network import bundles**-ის გენერირებას — სპეციალიზებულ არქივებს, რომლებიც დახვეწილია ქსელური საიტების აღდგენისთვის. ქსელის იმპორტის ბუნდლში მოთავსებულია ყველა საჭირო ფაილი და მეტამონაცემები, რათა რამდენიმე საიტი ახალ ქსელურ ინსტალაციაში აღდგენილად იქნას.

#### ქსელის იმპორტის ბუნდლის გენერირება {#generating-a-network-import-bundle}

1. გადადით **Ultimate Multisite > Site Export**-ზე
2. დააჭირეთ **Generate new Network Export**-ს
3. აირჩიეთ **Network Import Bundle** როგორც ექსპორტის ტიპი
4. აირჩიეთ, შეტანილ უნდა იქნას თუ არა აப்ლოუდები (uploads), პლაგინები და თემები (themes)
5. დააჭირეთ **Generate**-ს
6. ჩამოტვირთეთ ბუნდელი ZIP ფაილი **Existing Exports** სიაში

#### აღდგენა Network Import Bundle-დან {#restoring-from-a-network-import-bundle}

საიტების აღდგენისთვის network import bundle-ის გამოყენება:

1. დაინსტალირეთ Ultimate Multisite თქვენს მიზნობრივ ჰოსტზე
2. შეასრულეთ multisite-ის დაყენების ვიზუალური გზამკია (wizard)
3. გადადით ახალ network-ზე **Ultimate Multisite > Site Export**-ზე
4. დააჭირეთ **Import Network Bundle**-ს
5. ატვირთეთ network import bundle ZIP ფაილი
6. მიჰყევით ეკრანის ინსტრუქციებს იმპორტის პროცესისთვის
7. იმპორტი აღადგენს ყველა საიტს, მათ კონტენტს და კონფიგურაციას

Network import bundles ინარჩუნებენ:
- საიტის კონტენტს (პოსტები, გვერდები, კस्टम პოსტ ტიპები)
- საიტის პარამეტრებსა და ოფციებს
- მომხმარებლის როლებსა და უფლებებს
- პლაგინებსა და თემებს (თუ ისინი ბუნდელში შედის)
- მედია აப்ლოუდებს (თუ ისინი ბუნდელში შედის)
- კस्टम მონაცემთა ბაზის ცხრილებსა და მონაცემებს

## თვითდაბეჭდი ZIP ფაილის ინსტალაცია fresh host-ზე {#installing-a-self-booting-export-zip}

თვითდაბეჭდილ ZIP-ის აღდგენისთვის ახალ ჰოსტზე:

1. ატვირთეთ ექსპორტირებული ZIP-ის შინაარსი მიზნობრივი ვებ-რუტის (web root) içine
2. გახსენით ატვირთული `index.php` ბრაუზერში
3. მიჰყევით ინსტრუქციებს ეკრანზე გამოჩენილ ინსტალატორის შესახებ ბუნდელი ექსპორტის პაკეტისგან
4. შეამოწმეთ ბუნდელი `readme.txt` ექსპორტის სპეციფიკური შენიშვნებისთვის, სანამ დროებით ფაილებს წაშლით.

აპლუგინების კონკრეტული ინსტალაციისა და იმპორტის დეტალებისთვის იხილეთ [Site Exporter addon documentation](/addons/site-exporter).

Ultimate Multisite 2.9.0-ში დამატებული სინგლ-საიტის ინსტრუმენტისთვის, იხილეთ [Export & Import](/user-guide/administration/export-import).
