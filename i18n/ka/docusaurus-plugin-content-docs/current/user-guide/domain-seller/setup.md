# doimen Seller: დაყენება და провайდერის კონფიგურაცია

Domain Seller add-on მოჰყვება მითითებულ სცენარს (guided setup wizard), რომელიც გიყვება ყველა საჭირო ნაბიჯს. ეს გვერდი აღწერს სცენარის ნაკადს და იმას, თუ როგორ უნდა შევცვალოთ ან გადავაკონფიგურიროთ провайდერები ამის შემდეგ.

## მოთხოვნები {#requirements}

- **Multisite Ultimate** v2.4.12 ან უფრო მაღალი ვერსია, ქსელზე აქტიური
- **PHP** 7.4+
- მინიმუმ ერთი მხარდაჭერადი რეგისტრატორის API საკრედენციალები

## პირველი uruchომილობის სცენარი (First-run setup wizard) {#first-run-setup-wizard}

სცენარი ავტომატურად იწყება პირველად, როდესაც პლაგინი ქსელზე აქტიურდება. ის ხელმისაწვდომია ნებისმიერ დროს **Network Admin › Ultimate Multisite › Domain Seller Setup**-დან.

### ნაბიჯი 1 — პროвайდერის შერჩევა {#step-1--choose-a-provider}

აირჩიეთ რეგისტრატორი, რომელთანაც გსურთ დაკავშირება. მხარდაჭერილი ვარიანტები:

| პროвайდერი | DNS მართვა | WHOIS კონფიდენციალურობა |
|---|---|---|
| OpenSRS | დიახ | დიახ |
| Namecheap | არა | დიახ (WhoisGuard, უფასო) |
| HostAfrica | დიახ | დიახ (ID დაცვა) |
| Openprovider | დიახ | დიახ |
| Hostinger | Hostinger-ის ძირითადი დომეინის മാპინგის საშუალებით ჰოსტირებული დომეინებისთვის | დიახ |
| GoDaddy | არა | არა |
| ResellerClub | დიახ | არა |
| NameSilo | არა | არა |
| Enom | დიახ | არა |

### ნაბიჯი 2 — საკრედენციალების შეყვანა {#step-2--enter-credentials}

თითოეულ პროвайდერს აქვს განსხვავებული მონაცემთა ველი:

**OpenSRS** — მომხმარებლის სახელი და პრივატული გასაღები (OpenSRS Reseller Control Panel-დან)

**Namecheap** — მომხმარებლის სახელი და API გასაღები (Account › Tools › API Access-დან)

**HostAfrica** — დომეინების resell API终პოინტი და საკრედენციალები HostAfrica resell მოდულიდან. ამჟამად ცალკე სੈਂდბოქსის終პოინტი არ არის დოკუმენტირებული; რეალურ რეგისტრაციებზე გაშვებამდე შეამოწმეთ უსაფრთხო წაკითხვის მხოლოდ (read-only) ტესტები.

**Openprovider** — მომხმარებლის სახელი და პაროლი API-ით ხელმისაწვდომობით. სურვილისამებრ, სੈਂდბოქსის რეჟიმი იყენებს Openprovider-ის სੈਂდბოქსის API-ს, ხოლო სურვილისამებრ, დეფულტური მომხმარებლის სახელის (customer handle) გამოყენება შესაძლებელია რეგისტრაციებისთვის.

**Hostinger** — საერთო Hostinger hPanel API токен ძირითადი Hostinger ინტეგრაციიდან. იგივე ტოკენი აფუძნებს დომეინის ძირითად 맵ინგს და დომეინის გამყიდველის რეგისტრაციის ოპერაციებს.

**GoDaddy** — API გასაღები და სეკრეტული კლავიში (developer.godaddy.com-დან)

**ResellerClub** — Reseller ID და API გასაღები (ResellerClub კონტროლის პანელიდან)

**NameSilo** — API გასაღები (namesilo.com › Account › API Manager-იდან)

**Enom** — ანგარიშის ID და API ტოკენი

შეამოწმეთ **Sandbox mode**, თუ ხელმისაწვდომია, რომ ტესტი გააკეთოთ პროვაიდერის ტესტებთან გარემოში რეალურ ოპერაციებზე გადასSebelum.

### ნაბიჯი 3 — შეამოწმეთ კავშირი {#step-3--test-the-connection}

დააჭირეთ **Test Connection**. ვიზარდი აგზავნის მსუბუქ API მოთხოვნას, რათა დაადასტუროს მონაცემები და კავშირი. გააკეთეთ ნებისმიერი საკრედენციო პრობლემის გამოსწორება გაგრძელებამდე.

### ნაბიჯი 4 — TLD-ების იმპორტი {#step-4--import-tlds}

დააჭირეთ **Import TLDs**, რომ დაუკავშირებულ პროვაიდერისგან ყველა ხელმისაწვდომი TLD და საბითუღლური ფასები ჩამოტვირთოთ. ეს შეავსებს დომეინის პროდუქტებისთვის გამოყენებულ TLD-ების სიას. იმპორტი შეიძლება გაგრძელდეს 30–60 წამით დიდ TLD კატალოგის მქონე პროვაიდერებისთვის.

TLD-ები ავტომატურად გადაკრიბდება ყოველდღიურად ერთი დანახვის საშუალებით დაგეგმილი cron job-ის მეშვეობით.

### ნაბიჯი 5 — დომეინის პროდუქტის შექმნა {#step-5--create-a-domain-product}

ვიზარდი ქმნის დეფოლტულ, ყველაფერს მოიცავენ ("catch-all") დომეინის პროდუქტს 10%-იანი მარკაპით. შეგიძლიათ დაამუშაოთ ეს პროდუქტი დაუყოვნებლივ ან გამოტოვოთ და შექმნათ პროდუქტები ხელით **Ultimate Multisite › Products**-ში.

 முழுமையான პროდუქტის კონფიგურაციის გზაზე ნახეთ [Domain Products and Pricing](./domain-products).

## პროვაიდერის ხელახლა კონფიგურირება {#reconfiguring-a-provider}

წექეცით **Network Admin › Ultimate Multisite › Settings › Domain Seller** (ან დააჭირეთ **Settings** პლაგინების სიაში).

პარამეტრების გვერდზე მოცემულია:

- **Domain-ის გაყიდვების ჩართვა** — გადართეთ მთელი ფუნქცია ჩართვის/გამორთვისთვის
- **ნაგულისხმევი პროვაიდერი** — ის პროვაიდერი, რომელსაც იყენებთ დომეინების ძებნისას და ახალი პროდუქტებისთვის
- **მაქსიმალური TLD-ები თითო ძიებაში** — რამდენი TLD უნდა შეამოწმოთ მომხმარებლის მოთხოვნისას; უფრო მაღალი მნიშვნელობები მეტ ვარიანტს აჩვენებს, მაგრამ ნელი იქნება
- **Availability cache-ის ხანგრძლივობა** — რამდენ ხანს უნდა დავაკეტიოთ ხელმისაწვდომობისა და ფასების შედეგები; დაბალი მნიშვნელობები უფრო ზუსტია, მაგრამ API მოთხოვნების რაოდენობას ზრდის
- **დომენი პროდუქტების მართვა** — სწრაფი ბმული პროდუქტების სიაზე
- **პროვაიდერების კონფიგურაცია** — ხსნის Integration Wizard-ს, რომლის მეშვეობითაც შეგიძლიათ პროვაიდერების დამატება ან ხელახლა კონფიგურირება

### მეორე პროვაიდერის დამატება {#adding-a-second-provider}

დააჭირეთ **Configure providers**-ს და გაუშვით ვიზარდი ახალი რეგისტრატორისთვის. თქვენ შეგიძლიათ ერთდროულად რამდენიმე პროვაიდერის კონფიგურაცია. თითოეულ დომენი პროდუქტს მიანიჭეთ კონკრეტული პროვაიდერი, ან დატოვეთ ნაგულისხმევზე.

### TLD-ების მექანიკური სინქრონიზაცია {#syncing-tlds-manually}

Settings გვერდზე, დააჭირეთ **Sync TLDs**-ს ნებისმიერი კონფიგურირებული პროვაიდერის გვერდით ბოლო ფასების მისაღებად. ეს სასარგებლოა მაშინ, როდესაც პროვაიდერი განახლებს საბითუღლოს ფასებს ან ახალ TLD-ებს დაამატებს.

## Примечания о возможностях провайдеров {#provider-capability-notes}

Не все API регистраторов предоставляют одни и те же операции. Дополнение показывает неподдерживаемые операции с понятными ошибками для администратора, вместо того чтобы молча завершаться неудачей.

- **HostAfrica** поддерживает самый широкий рабочий процесс реселлера в реальном времени, включая поиск, синхронизацию TLD/цены, регистрацию, продление, перевод, обновление имен серверов, DNS-записи, коды EPP, блокировку регистратора и защиту ID.
- **Openprovider** поддерживает синхронизацию TLD по цене реселлера, регистрацию, продление, переводы, обновление имен серверов, DNS-зоны, коды EPP, блокировку регистратора и конфиденциальность WHOIS. Он аутентифицируется с помощью краткосрочного токена bearer, который дополнение автоматически обновляет.
- **Hostinger** поддерживает поиск доступности, регистрацию, поиск портфолио, обновление имен серверов, блокировку регистратора и конфиденциальность WHOIS через общий токен API hPanel. Публичный API доменов Hostinger не предоставляет информацию о ценах реселлера/оптовых цен, входящем переводе, явном продлении или получении кодов EPP; продления происходят только автоматически.
