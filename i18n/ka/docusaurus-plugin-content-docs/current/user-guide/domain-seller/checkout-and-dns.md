# Checkout πεდი და მომხმარებლის DNS მართვა

## დომეინის შერჩევის checkout ველი {#the-domain-selection-checkout-field}

**Domain Selection** ველი არის checkout ელემენტი, რომელიც მომხმარებლებს აძლევს საშუალებას აირჩიონ, როგორ მიიღონ თავიანთი საიტის დომენი. დაამატეთ ის ნებისმიერ checkout ფორმას, რათა გააქტიურდეს დომეინის გაყიდვა.

### ველი checkout ფორმაში დამატება {#adding-the-field-to-a-checkout-form}

1. გადადით **Network Admin › Ultimate Multisite › Checkout Forms**-ზე
2. გახსენით ან შექმენით checkout ფორმა
3. checkout რედაქტორში დააწექით **Add Field**-ს (ველის დამატება)
4. ველიდან აირჩიეთ **Domain Selection**
5. დააკონფიგურირეთ ველის ვარიანტები (იხ. ქვემოთ)
6. შეინახეთ ფორმა

### ველის ვარიანტები {#field-options}

**Domain modes** — აირჩიეთ, თუ რომელი ტაბები დაინახავს მომხმარებელი. თითოეული რეჟიმი შეიძლება ჩართული იყოს ან გამორთული:

| Mode | რა ხდება |
|---|---|
| **Subdomain** (ქვედა დომენი) | მომხმარებელი იყენებს უფასო ქვედა დომენს თქვენს ქსელზე (მაგალითად, `mysite.yournetwork.com`). გადახდა არ არის საჭირო. |
| **Register New Domain** (ახალი დომეინის რეგისტრაცია) | მომხმარებელი ეძებს ახალ დომე인을 და აკონტროლებს მას თქვენი კონფიგურირებული პროვაიდერის მეშვეობით. ფასწარმოქმნისთვის გამოიყენება შესაბამისი დომეინის პროდუქტი. |
| **Existing Domain** (არსებული დომეინის გამოყენება) | მომხმარებელი აკავშირებს იმ დომეინს, რომელიც უკვე საკუთრებაში აქვს. არ არის რეგისტრაციის საფასური. დომენი ავტომატურად მიბმულია მათ საიტზე. |

**Default mode** (ნაგულისხმევი რეჟიმი) — როდესაც ყველა სამი რეჟიმი ჩართულია, რომელი ტაბა იხსნება პირველ რიგში? დაადეთ **Subdomain**, რათა დომეინის რეგისტრაცია არ იყოს ვალდებული, ან **Register New Domain**, რათა წაახალისოთ ყიდვები.

**Domain product** (დომეინის პროდუქტი) — შესაძლებელია ამ ველის დამაგრება კონკრეტულ დომეინის პროდუქტზე. თუ არ არის დაყენებული, add-on ავტომატურად აირჩევს შესაბამის პროდუქტს მომხმარებლის მიერ ძიებული TLD-ის (Top Level Domain) საფუძველზე.

### რეგისტრანტის საკონტაქტო ველი {#registrant-contact-fields}

როდესაც მომხმარებელი აირჩევს **Register New Domain** ტაბას, checkout ფორმა ავტომატურად დაამატებს რეგისტრანტის საკონტაქტო ველებს:

- სახელი / გვარი
- ელფოსტის მისამართი
- მისამართი (პირველი ხაზი, ქალაქი, შტატი/პროVINცია, საფოსტო ინდექსი, ქვეყანა)
- ტელეფონის ნომერი

ეს არის საჭირო ყველა რეგისტრატორისთვის და ვერიფიცირდება სერვისის API-ს გამოძახების წინ. ტელეფონის ნომრები ავტომატურად ფორმატირდება საერთაშორისო ფორმატში `+CC.NNN`, რომელსაც რეგისტრატორები ელოდებიან.

### ავტომატურად გენერირებული საიტის URL-ი {#auto-generated-site-url}

როდესაც მომხმარებელი აკეთებს რეგისტრაციას ან დომე인을 ამოჭრის (მაპინგს), საიტის URL-ის ველი ავტომატურად ივსება შერჩეული დომეინის საფუძველზე. მომხმარებელს არ სჭირდება ცალკე URL-ის შევსება.

### ძიების ქცევა (Search behaviour) {#search-behaviour}

- დომეინის ხელმისაწვდომობა რეალურ დროში AJAX-ით იმოწმებს, როდესაც მომხმარებელი ტეგავს
- ალტერნატიული TLD-ების შეთავაზებები ჩანს, როდესაც სასურველი დომენი ხელმისაწვდომი არ არის
- ფასები იღება პირდაპირ და ნათლად გამოტანილია (რეგისტრაციის ფასი, განახლების ფასი, სურვილისამებრ WHOIS კონფიდენციური საფასური)
- კუპონის კოდები ვრცელდება დომეინის პროდუქტებზე ისევე, როგორც ნებისმიერ სხვა პროდუქტზე

**ძიების პასუხის რეაგირების დახვეწა:**

```php
// გაზრდის დებონსის დაყოვნებას (மில்ლიწამებში) API-ს გამოძახებების შესამცირებლად ნელი კავშირებისთვის
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // დეფოლტი: 500
});
```

**დომეინის საძიებო ფორმის დამატება საკუთარი ველით:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'დამატებითი შენიშვნები',
    ];
    return $fields;
});
```

## მომხმარებლის DNS მართვა (Customer DNS management) {#customer-dns-management}

მომხმარებელს შეუძლია დაარეგულიროს DNS ჩანაწერები თავისი დარეგისტრირებული დომეინებისთვის **My Account** გვერდიდან, დომეინის შესავლის ქვეშ.

### მხარდაჭერილი ჩანაწერების ტიპები (Supported record types) {#supported-record-types}

| ტიპი | გამოყენება |
|---|---|
| **A** | ჰოსტнейმის მიბმა IPv4 მისამართზე |
| **AAAA** | ჰოსტнейმის მიბმა IPv6 მისამართზე |
| **CNAME** | ალIASის შექმნა სხვა ჰოსტнейmesine მიმართვისთვის |
| **MX** | მეილის გაცვლის სერვერის დაყენება |
| **TXT** | SPF, DMARC, ვერიფიკაცია ან სხვა ტექსტური ჩანაწერების დამატება |

### რომელი პროვაიდერები მხარს უჭერენ DNS მართვას? {#which-providers-support-dns-management}

DNS-ის მართვა (რეკორდების დამატება, რედაქტირება, წაშლა) შესაძლებელია **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** და **Openprovider**-ით. **Hostinger** დომეინებს შეუძლიათ ნეიმსერვერების განახლება Domain Seller-ის საშუალებით; მასპინძლობით მიღებულ დომეინებისთვის DNS რეკორდები მართപ്പെടുന്നു Hostinger-ის ძირითადი დომეინის დამაპარტეტირების ინტეგრაციით. Namecheap, GoDaddy და NameSilo დომეინები აჩვენებს სტატუსს და ვადის გასვლის ინფორმაციას, მაგრამ DNS პირდაპირ რეგისტრატორის კონტროლის პანელში უნდა მართოთ.

### ნაგულისხმევი DNS რეკორდები {#default-dns-records}

თქვენ შეგიძლიათ დააყენოთ ნაგულისხმევი DNS რეკორდები, რომლებიც ავტომატურად გამოიყენება დომეინის ჩარიცხვისას. გადადით **Settings › Domain Seller › Default DNS Records**-ზე.

ნაგულისხმევი რეკორდების მნიშვნელობებს მხარს უჭერენ ორი ტოკენი:

| ტოკენი | შეცვლილი ხდება |
|---|---|
| `{DOMAIN}` | ჩარიცხული დომეინის სახელი (მაგალითად, `example.com`) |
| `{SITE_URL}` | WordPress-ის საიტის URL მომხმარებლისთვის |

**მაგალითი — აპექს და www-ს მიმართეთ თქვენი სერვერის IP- address:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ადმინი: DNS-ის ნახვა და რედაქტირება {#admin-viewing-and-editing-dns}

ნეტიური ადმინისტრატორებს შეუძლიათ ნახონ და შეცვალონ ნებისმიერი მომხმარებლის დომეინის DNS რეკორდები დომეინის რედაქტვის გვერდიდან **Network Admin › Ultimate Multisite › Domains**-ში.
