# ჰედერის დომენის ძიება {#header-domain-search}

გამოიყენეთ ეს კონფიგურაცია, როცა გჭირდებათ პატარა ფორმა ჰედერში, რომელიც იწყებს დომენის ძიებას და შემდეგ მომხმარებელს საშუალებას აძლევს, Ultimate Multisite გადახდის პროცესში ხელმისაწვდომი დომენებიდან აირჩიოს.

## მოთხოვნები {#requirements}

- Ultimate Multisite ქსელში აქტიური.
- Multisite Ultimate Domain Seller ქსელში აქტიური.
- მინიმუმ ერთი აქტიური დომენის რეგისტრაციის პროდუქტი, რომელსაც აქვს:
  - `domain_provider` დაყენებული კონფიგურირებულ პროვაიდერზე.
  - მხარდაჭერილი TLD-ები კონფიგურირებული, მაგალითად `com`, `net` და `org`.
- მოქმედი გადახდის ფორმა, რომელიც შეიცავს **დომენის არჩევის** ველს.

## გადახდის ფორმა {#checkout-form}

1. შექმენით ან დაარედაქტირეთ რეგისტრაციის გვერდისთვის გამოყენებული გადახდის ფორმა.
2. დაამატეთ ჩვეულებრივი აუცილებელი გადახდის/Account ველები, მათ შორის **მომხმარებლის სახელი**. გადახდის ფორმა, რომელიც მხოლოდ დომენის ველს შეიცავს, Ultimate Multisite ვალიდაციით უარყოფილია.
3. დაამატეთ **დომენის არჩევის** ველი.
4. დააყენეთ დომენის არჩევის რეჟიმი **მხოლოდ რეგისტრაცია**-ზე, როცა ნაკადი რეგისტრირებულ დომენებზე უნდა იყოს ფოკუსირებული და არა უფასო ქვედომენებზე ან არსებულ დომენებზე.
5. მიანიჭეთ დომენის რეგისტრაციის პროდუქტი ამ ველს.

რეგისტრაციის გვერდმა უნდა გამოიტანოს გადახდის ფორმა, მაგალითად:

```text
[wu_checkout slug="domain-form"]
```

## ჰედერის ფორმა {#header-form}

დაამატეთ პატარა `GET` ფორმა საიტის ჰედერში, რომელიც შეყვანილ საძიებო ტერმინს გადახდის გვერდზე `domain_name`-ად აგზავნის:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

არ აირჩიოთ დომენი წინასწარ მორგებულ ჰედერის JavaScript-ში. ჰედერმა მხოლოდ საძიებო ტერმინი უნდა გადასცეს. Domain Seller გადახდის სკრიპტი კითხულობს `?domain_name=example`-ს, ავსებს გადახდის საძიებო ველს და უშვებს ხელმისაწვდომობის ძიებას, რათა მომხმარებელმა დაბრუნებული დომენებიდან აირჩიოს.

## მოსალოდნელი ქცევა {#expected-behaviour}

ჰედერში `example`-ის ძიებამ უნდა გახსნას:

```text
/register/?domain_name=example
```

შემდეგ გადახდამ უნდა აჩვენოს ასარჩევი შედეგები, როგორიცაა:

- `example.com`
- `example.net`
- `example.org` მიუწვდომელია
- სხვა პროვაიდერის მიერ მხარდაჭერილი TLD-ები

ხელმისაწვდომი შედეგის არჩევის შემდეგ, შეკვეთის შეჯამება უნდა შეიცავდეს დომენის რეგისტრაციის პროდუქტს და არჩეულ დომენის სახელს.

## შემოწმება {#verification}

1. გახსენით მთავარი გვერდი.
2. მოძებნეთ შიშველი სახელი, მაგალითად `example`.
3. დაადასტურეთ, რომ გადახდის URL შეიცავს `?domain_name=example`-ს.
4. დაადასტურეთ, რომ გადახდის დომენის ველი შეიცავს `example`-ს.
5. დაადასტურეთ, რომ დომენის ვარიანტების სია გამოჩნდება.
6. დააწკაპუნეთ **არჩევა**-ზე ხელმისაწვდომი დომენისთვის.
7. დაადასტურეთ, რომ შეკვეთის შეჯამება შეიცავს `Domain Registration - example.com`-ს ან არჩეულ დომენს.

## პრობლემების მოგვარება {#troubleshooting}

- თუ სია არ გამოჩნდება, შეამოწმეთ ბრაუზერის ქსელის ჩანართში `admin-ajax.php?action=wu_domain_search` და დაადასტურეთ, რომ ის აბრუნებს არა ცარიელ `domains`-ს ან `results`-ს.
- თუ გადახდის ფორმა შენახვისას ვალიდაციას ვერ გადის, დაამატეთ აუცილებელი Account ველები, როგორიცაა **მომხმარებლის სახელი**.
- თუ დომენის არჩევა კალათას არ განაახლებს, დაადასტურეთ, რომ `window.wu_checkout_form` არსებობს და რომ Domain Seller გადახდის რესურსები ჩატვირთულია გადახდის გვერდზე.
