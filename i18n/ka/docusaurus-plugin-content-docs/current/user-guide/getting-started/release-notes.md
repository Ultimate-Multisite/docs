## ვერსია 2.13.0 — გამოშვებული 2026-06-05 {#version-2130--released-on-2026-06-05}

ახალი: დამატებულია სუვერენული ტენანტების მხარდაჭერა მომხმარებლის ანგარიშებისთვის, შეკვეთის პროცესისთვის, ბილინგის (გადახდების) დამატებით, საიტის, ინვოისებისა და შაბლონების გადართვის ნაკადებისთვის, რათა ტენანტული ქსელები მომხმარებლებს მართული მოქმედებებისას თავიანთ მთავარ საიტზე მიმართონ.
ახალი: დამატებულია განახლების საკრედenco შემოწმებები განმეორებადი წევრობისთვის, რათა გეითვეებს შეუძლიათ ავტომატური განახლება გამორთონ მაშინ, როდესაც დარჩა შენახული გადახდის შეთანხმება, გამოწერა ან ვੌულტების ტოკენი.
ახალი: დამატებულია HMAC-ვერიფიცირებული loopback pubblicishing (გამოქვეყნებისთვის) დაუყოვნებლივი საიტის შესაქმნელად, რათა შეკვეთის პროცესის და საიტზე გადასვლის პროცესი უფრო სანდო იყოს იმ ჰოსტებზე, სადაც ბექგრაუნდული ამოცანები დაგვიანებულია.
ახალი: დამატებულია დეველოპერებისთვის გაფართო წერტილები SSO URL-ებისთვის, შეკვეთის ფორმის ძირითადი დომეინებისთვის და ავტომატური დომეინის ჩანაწერების შესაქმნელად.
გამოსწორება: SSO უფრო სანდოა დაუკავშირებელ დომეინებს, ანონიმურ ბროკერის ვიზიტებს, გამთიშვას (logout) და პლაგინების ურთიერთდამოკიდებულებების კონფლიქტებს შორის.
გამოსწორება: ახლა საიტის შესაქმნელმა პროცესმა აღადგინა სტალინური pubblicishingフラגים-დან და თავიდან აიცილა მომხმარებლების ჩაკრევა საიტის შექმნის ეკრანზე.
გამოსწორება: დომეინის ჩანაწერები აღარ იქმნება საერთო შეკვეთის ფორმის ძირითადი დომეინებისთვის, და არ გამოიყენება ჰოსტ-პროვაიდერის ბექგრაუნდული ამოცანები, თუ არ არის აქტიური ინტეგრაცია.
გამოსწორება: შეკვეთების პროცესი, გადახდის მისამართი, პაროლის გადაყენება, ელფოსტის ვერიფიკაცია, შაბლონების გადართვა, ტურები და მომხმარებლის დეშბორტის ამოცანები აღარ ბლოკავს ნორმალურ მომხმარებლის ნაკადებს.
გამოსწორება: ბროკასტული ელფოსტები ახლა ინარჩუნებს მიმღებთა კონფიდენციალურობას, đồng thời ა თავისთავად ა გვერდის ავლინებს SMTP/პლაგინის ფატალურ შეცდომებს, როდესაც მიმღებთა სიები ან მეილის ტრანსპორტი იბლოკება.
გამოსწორება: წევრობის განახლებების, ვადის გასვლის ჩვენებისა და გადახდის შეგროვების ამოცანები ახლა თავიდან აცილებს მყისიერ გაქრებას, კრ meaningful-ს (კრ meaningful-ს) და საჭირო გადასახდელების გამოტოვებას.
გაუმჯობესება: WordPress-თან თავსებადობა შემოწმებულია 7.0-მდე, პროდუქციული Vue აქტივები იქმნება npm წყაროებიდან, ხოლო Cypress end-to-end კვლევამ ახლა უფრო მეტ შეკვეთის, დაყენების, SSO და გეითვეების ნაკადები ატარებს.
