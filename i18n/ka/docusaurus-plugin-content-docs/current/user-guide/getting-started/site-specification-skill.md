# საიტის სპეციფიკაციის უნარი

**საიტის სპეციფიკაციის უნარი (Site Specification skill)** არის სტრუქტურირებული მიდგომა თქვენი საიტის მიზნების, აუდიტორიისა და ბრენდის იდენტობის აღსადგენად. ეს ინფორმაცია ინახება თქვენს **site_brief** მეხსიერებაში, რომელსაც აგენტები სხვადასხვა სესიებს შორის ეყრდნობიან თანმიმდევრულ და კონტექსტის გათვალისწინებულ დახმარებისთვის.

## რა არის საიტის სპეციფიკაცია?

საიტის სპეციფიკაცია არის დოკუმენტირება შემდეგში:

- **საიტის მიზანი**: რას აკეთებს თქვენი საიტი და რატომ არსებობს
- **სამიზნე აუდიტორია**: ვინ ეწვია თქვენს საიტს და რა სჭირდება მათ
- **ბრენდის იდენტობა**: თქვენი ფერები, ტონი და ვიზუალური სტილი
- **Geschäftული მიზნები**: წარმატება თქვენი საიტისთვის როგორ გამოიყურება
- **შინაარსის სტრუქტურა**: როგორ არის ორგანიზებული თქვენი საიტი

ეს სპეციფიკაცია ხდება თქვენი **site_brief**, მდგრადი მეხსიერება, რომელსაც აგენტები იყენებენ თქვენი საიტის კონტექსტის გასაგებად.

## რატომ გამოვიყენოთ საიტის სპეციფიკაცია?

### თანმიმდევრულობა სესიებს შორის

site_brief-ის გარეშე, ყოველ ჯერზე ახალი სესიის დაწყებისას თქვენ დაგჭირდებოდათ საიტის მიზნის ხელახლა განმარტება. მასთან ერთად, აგენტები დაუყოვნებლივ გესმით:

- თქვენი საიტის მიზნები და აუდიტორია
- თქვენი ბრენდის ფერები და ტონი
- თქვენი შინაარსის სტრუქტურა
- თქვენიGeschäftული მიზნები

### უკეთესი რეკომენდაციები

აგენტები იყენებენ თქვენს site_brief-ს შემდეგში:

- გთავაზობთ ფუნქციებს, რომლებიც შეესაბამება თქვენი საიტის მიზანს
- გირჩევთ შინაარსის სტრუქტურებს, რომლებიც ემთხვევა თქვენს მიზნებს
- გვთავაზებთ დიზაინებს, რომლებიც თანხვედრაშია თქვენს ბრენდთან
- თავიდან აგაცილებთ შეუსაბამო ფუნქციების შემოთავაზებას

### სწრაფი ინტეგრირება (Onboarding)

ახალი აგენტები (ან ახალ სესიებში არსებული აგენტები) სწრაფად ითვისებენ ინფორმაციას თქვენი site_brief-ის წაკითხვით და არა დამატებითი კითხვების დასმვით.

## საიტის სპეციფიკაციის ინიცირება

### თემა ბილდერის ინტეგრირების დროს (During Theme Builder Onboarding)

საიტის სპეციფიკაციის უნარი ავტომატურად იწყება **Theme Builder-ის ინტეგრირების პროცესში**. დაყენების ასისტენტი (Setup Assistant agent) კითხვებს დასვამს და თქვენს site_brief-ს აგებს.

### მექანიკური ინიცირება (Manual Initiation)

თქვენ შეგიძლიათ დაიწყოთ საიტის სპეციფიკაციის დადგენა ნებისმიერ დროს:

```
"დავადგინოთ ჩემი საიტის სპეციფიკაცია"
```

ან

```
"დამეხმარეთ საიტის ბრიფის შექმნაში"
```

## საიტის სპეციფიკაციის პროცესი

### ნაბიჯი 1: საიტის მიზანი

აგენტი იკითხავს:

```
რა არის თქვენი საიტის მთავარი მიზანი?
- ელექტრონული კომერციული მაღაზია (E-commerce store)
- ბლოგი ან კონტენტის საიტი
- პორტფოლიო ან გამოფენა
- SaaS აპლიკაცია
- თემთა შეკრება ან ფორუმი
- სხვა: [აღწერეთ]
```

თქვენ შეგიძლიათ აირჩიოთ კატეგორია ან აღწეროთ თქვენი მიზანი.

### ნაბიჯი 2: სამიზნე აუდიტორია

```
ვინ არის თქვენი მთავარი აუდიტორია?
- მომხმარებლები / ზოგადი Öffentlichkeit
- ბიზნეს პროფესიონალები
- დეველოპერები / ტექნიკური მომხმარებლები
- სტუდენტები / განათლებების სპეციალისტები
- სხვა: [აღწერეთ]

რა არის მათი ძირითადი საჭიროებები?
```

### ნაბიჯი 3: ბრენდის იდენტობა

```
რა არიან თქვენი ბრენდის ფერები?
- პრიმატული ფერი: [ფერის შერჩევა ან ჰექს კოდი]
- მეორადი ფერი: [ფერის შერჩევა ან ჰექს კოდი]
- აქცენტური ფერი: [სურვილისამებრ]

როგორ აღწერთ თქვენი ბრენდის ტონს?
- პროფესიული / korporative (კორპორატიული)
- შემოქმედებითი / მხატვრული
- სახალისო / არაფორმალური
- მინიმალისტური / თანამედროვე
- თბილი / მეგობრული
```

### ნაბიჯი 4: ბიზნეს მიზნები

```
რა გამოიყურება წარმატებამ თქვენს საიტზე?
- ლიდების გენერირება (Generate leads)
- პროდუქტების გაყიდვა
- თემთა შეკრების მშენებლობა
- ცოდნის გაზიარება
- ავტორიტეტის დამკვიდრება
- სხვა: [აღწერეთ]

რა არის თქვენი მთავარი მეტრიკა?
- შემოსავალი (Revenue)
- მომხმარებლის ჩართულობა (User engagement)
- კონტენტის მიღწევადობა (Content reach)
- კონვერტაციები (Conversions)

```

### ნაბიჯი 5: კონტენტის სტრუქტურა

```
როგორ არის ორგანიზებული თქვენი კონტენტი?
- ბრტყელი (ყველა კონტენტი ერთ დონეზე)
- იერარქიული (კატეგორიები და ქვეკატეგორიები)
- ქრონოლოგიური (ბლოგის სტილი)
- პროდუქტის მიხედვით (каталог)
- სხვა: [აღწერეთ]

რა ტიპის კონტენტს იყენებთ?
- ბლოგ პოსტები
- პროდუქტის გვერდები
- კეის სტადიები (Case studies)
- დოკუმენტაცია
- ვიდეოები
- სხვა
```

## თქვენი site_brief მეხსიერება

საიტის სპეციფიკაციის დასრულების შემდეგ, თქვენი ინფორმაცია ინახება როგორც **site\_brief** თქვენს აგენტის მეხსიერებაში. ეს არის სტრუქტურირებული ჩანაწერი, რომელიც შეიცავს:

## თქვენი საიტის brief-ის ნახვა და განახლება

### თქვენი site_brief-ის ნახვა

გთხოვთ ჰკითხოთ აგენტს:

```
"Show me my site brief"
```

ან

```
"What do you know about my site?"
```

აგენტი გამოიტანს თქვენს შენახულ სპეციფიკაციას.

### თქვენი site_brief-ის განახლება

თუ თქვენი საიტი იცვლება, შეგიძლიათ მისი განახლება:

```
"Update my site brief: we're now targeting B2B customers"
```

ან

```
"Refresh my site specification"
```

ეს ხელახლა გაუშვებს Site Specification-ს ფუნქციას თქვენ არსებული ინფორმაციის გამოყენებით საწყისი წერტილიდან.

## როგორ იყენებენ აგენტები site_brief-ს

### დიზაინის რეკომენდაციები

როდესაც გთხოვთ დიზაინის ცვლილებებს, აგენტები ეყრდნობიან თქვენს site_brief-ს:

```
თქვენ: "Redesign my homepage"
აგენტი: "თქვენი site brief-ის მიხედვით, თქვენ 25-45 წლის ქალებს
მიზნად ისახავთ და გჭირდებათ თბილი, ხელნაკეთი ტონი. მე შევქმნი დიზაინს, რომელიც
დახაზს თქვენს ხელნაკეთ სამკაულებსა და მდგრად ღირებულებებს."
```

### ფუნქციების შეთავაზებები

აგენტები გთავაზობენ ფუნქციებს, რომლებიც համապატარებელია თქვენს მიზნებთან:

```
თქვენ: "What features should I add?"
აგენტი: "კლიენტთა ერთგულების მიმართ ორიენტირებული ელექტრონული კომერციისთვის
ვრწმუნდები შემდეგზე: ერთგულების პროგრამა, კლიენტების მიმოხილვები, ელექტრონული ბიულეტენი და პროდუქტის რეკომენდაციები."
```

### კონტენტის სტრუქტურა

აგენტები 제 შეთავაზებენ კონტენტის ორგანიზებას თქვენი სტრუქტურის მიხედვით:

როგორ დავაწყებ ჩემს ბლოგს?
პასუხი: რადგან თქვენ იყენებთ პროდუქტის კატალოგს ბლოგისთვის, გირჩევთ პოსტების ორგანიზებას პროდუქტის კატეგორიების მიხედვით და შექმნათ „სტილის სახელმძღვანელოები“, რომლებიც რამდენიმე პროდუქტს ერთად წარმოაჩენს.

## საუკეთესო პრაქტიკები

### იყავით კონკრეტული

„ზოგადი აუდიტორიის“ ნაცვლად, აღწერეთ თქვენი რეალური აუდიტორია:

- ✓ „25-45 წლის ქალები, რომლებიც დაინტერესებულნი არიან მდგრადი ფეშენით“
- ✗ „ყველას“

### განაახლეთ რეგულარულად

როდესაც თქვენი საიტი იცვლება, განაახლეთ თქვენი `site_brief`:

- როდესაც ახალ აუდიტორიაზე გადადიხართ
- როდესაც ახალ პროდუქტის ხაზებს ამატებთ
- როდესაც თქვენი ბრენდის იდენტობა იცვლება
- როდესაც თქვენი ბიზნეს მიზნები იცვლება

### გამოიყენეთ თანმიმდევრული ტერმინოლოგია

გამოიყენეთ ერთი და იგივე ტერმინები ყველა სესიის განმავლობაში:

- ✓ ყოველთვის თქვით „მდგრადი სამკაულები“ (არა „ეკო-ფრენდული სამკაულები“ და „ნ绿色 პროდუქტები“)
- ✓ აუდიტორიას თანაბარი სახით მიმართეთ

### ჩართეთ კონტექსტი

მოწოდეთ ფონი, რომელიც ეხმარება აგენტებს თქვენი გადაწყვეტილებების გაგებაში:

- „ჩვენ ვכוობთ პროფესიონალებს, რომლებიც ფასზე მეტს უმატებენ ხარისხს“
- „ჩვენი აუდიტორია ტექნოლოგიურად არის მომზადებული და ელის თანამედროვე დიზაინს“
- „ჩვენ ვართ ბუასტროპული სტარტაპი, ამიტომ გვჭირდება ღირებულების ეფექტური გადაწყვეტილებები“

## ურთიერთობა თემა ბილდერის ინტეგრაციასთან (Relationship to Theme Builder Onboarding)

Site Specification-ის უნარი ინტეგრირებულია **Theme Builder-ის ჩართვის პროცესში**. როდესაც ჩართვას დაასრულებთ, თქვენი `site_brief` ავტომატურად შეიქმნება იმ ინფორმაციით, რაც მოგვაწოდეთ.

თქვენ ასევე შეგიძლიათ Site Specification-ის გაშვება დამოუკიდებლად, თუ გსურთ:

- თქვენი სპეციფიკაციის გაუმჯობესება პირველ კონფიგურაციიდან
- `site_brief`-ის განახლება თქვენი საიტის განვითარებისას
- Theme Builder-ის დაწყებამდე დეტალური სპეციფიკაციის შექმნა

## პრობლემების მოხსნა (Troubleshooting)

**ჩემი site_brief არ გამოიყენება**
- დაადასტურეთ, რომ აგენტს აქვს წვდომა მეხსიერებაზე (memory)
- სთხოვეთ აგენტს „დაასახელოს ჩემი site brief“ (recall my site brief)
- შეამოწმეთ, არის თუ არა მეხსიერება ჩართული თქვენს პარამეტრებში

**მინდა ახალი site\_brief-ით დავიწყო**
- სთხოვეთ აგენტს: "წაშალე ჩემი site brief და თავიდან დავიწყოთ"
- შემდეგ გაუშვით Site Specification ხელახლა

**აგენტი გთავაზობს რეკომენდაციებს, რომლებიც არ ემთხვევა ჩემს site\_brief-ს**
- სთხოვეთ აგენტს: "გადახედე ჩემს site brief-ს"
- განაახლეთ თქვენი site\_brief, თუ ის აღარ არის აქტუალური
- მიაწოდეთ დამატებით კონტექსტი თქვენს მოთხოვნებში

## შემდეგი ნაბიჯები

თქვენს site specification-ის განსაზღვრის შემდეგ:

1. **გამოიყენეთ Theme Builder**: შექმენით საკუთარი თემა (theme) თქვენი site\_brief-ის საფუძველზე
2. **დახვეწეთ დიზაინი**: გამოიყენეთ Design System Aesthetics უნარი დეტალური დიზაინის სამუშაოდ
3. **გეგმავთ კონტენტს**: სთხოვეთ აგენტებს კონტენტის სტრუქტურის რეკომენდაციების შესახებ
4. **შექმენით ფუნქციები**: მოითხოვეთ ფუნქციები, რომლებიც შეესაბამება თქვენს ბიზნეს მიზნებს
