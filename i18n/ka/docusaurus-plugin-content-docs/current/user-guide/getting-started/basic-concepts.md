# ძირითადი ცნებები {#basic-concepts}

ახალი WordPress Multisite მომხმარებლისთვის და მათთვის, ვინც ახლახან დაიწყო Ultimate Multisite-ის გამოყენება, თავიდან შეიძლება ბევრი ახალი სიტყვა და ფრაზა შეგეძლოთ. მათი სწავლა მნიშვნელოვანი ამოცანაა, რადგან საჭირო იქნება პლატფორმის და მთლიანი სისტემის გაგება.

ამ სტატიაში ჩვენ შევეცდებით განვსაზღვროთ და განვმარტოთ WordPress-ის ზოგიერთი საკვანძო ცნება. ზოგი მათგანი უფრო მომხმარებლებისთვის, ზოგი დეველებისთვის და ზოგი კი ორივესთვისაა შესაბური.

## WordPress Multisite {#wordpress-multisite}

WordPress **Multisite** არის WordPress-ის ინსტალაციის ტიპი, რომელიც საშუალებას გაძლევთ შექმნათ და მართოთ მრავალი ვებსაიტების ქსელი ერთი WordPress dashboard-იდან. თქვენ შეგიძლიათ მართოთ ყველაფერი, მათ შორის საიტების რაოდენობა, ფუნქციები, თემები და მომხმარებლის როლები. შესაძლებელია ასობით და ათასობით საიტის მართვაც.

## ქსელი (Network) {#network}

WordPress-ის ტერმინოლოგიით, Multisite ქსელში რამდენიმე **ქვესაიტი** შეიძლება მართული იყოს ერთი dashboard-იდან. მიუხედავად იმისა, რომ Multisite ქსელის შექმნა განსხვავებულია ჰოსტინგ პროვაიდერებს შორის, საბოლოო შედეგი ჩვეულებრივ არის რამდენიმე დამატებითი დირექტივა **wp-config.php** ფაილში, რათა WordPress-მა იცოდეს, რომ ის ამ კონკრეტულ რეჟიმში მუშაობს.

ქსელის დამოუკიდებლად არსებულ WordPress ინსტალაციასთან Multisite ქსელს შორის რამდენიმე განსხვავება არსებობს, რაც მოკლედ განვიხილავთ.

## მონაცემთა ბაზა (Database) {#database}

მონაცემთა ბაზა არის სტრუქტურირებული და ორგანიზებული მონაცემების ნაკრები. კომპიუტერული ტერმინოლოგიაში, მონაცემთა ბაზა გულისხმობს პროგრამულ უზრუნველყოფას, რომელიც გამოიყენება მონაცემების შესანახად და ორგანიზებისთვის. წარმოიდგინეთ ეს როგორც ფაილის კარადა, სადაც მონაცემებს სხვადასხვა განყოფილებებში, რომლებსაც ტაბულები ეწოდება, ინახავთ.

WordPress Multisite იყენებს ერთ მონაცემთა ბაზას და თითოეული ქვესაიტს ექნება საკუთარი ტაბულები, სადაც ბლოგის ID-ს აქვს პ rifექსი (წინდახედი), ამიტომ, როგორც კი ქსელის ინსტალაციას და შექმნით ქვესაიტს, თქვენ უნდა გქონდეთ ეს ტაბულები:

_wp_1_options_ - ოფციების ცხრილი პირველი სუბსೈტისთვის

_wp_2_options_ - ოფციების ცხრილი მეორე სუბსೈტისთვის

## ჰოსტინგის პროვაიდერი (Hosting provider) {#hosting-provider}

ჰოსტინგის პროვაიდერი არის კომპანია, რომელიც ბიზნესებს და ინდივიდებს საშუალებას აძლევს თავიანთ ვებგვერდებს გახადონ ხელმისაწვდომი მსოფლიო ვებ-სრული ქსელში. ვებჰოსტინგის პროვაიდერების მიერ  მოწოდებული სერვისები განსხვავებულია, მაგრამ ჩვეულებრივ მოიცავს ვებსაიტის დიზაინს, ჰოსტის შენახვის სივრცეს და ინტერნეტთან კავშირს.

## დომენი (Domain) {#domain}

დომენი არის მისამართი, რომელსაც ადამიანები იყენებენ თქვენი საიტის სანახავად. ის ეუბნება ვებ-ბრაუзерს, სად უნდა მოძებნოს თქვენი საიტი. ისევე როგორც ქუჩის მისამართი, დომენი არის ის, თუ როგორ სტუმრობენ ადამიანები თქვენს ვებსაიტს ონლაინ. და, ისევე როგორც მაღაზიის წინ არსებული პანneau. თუ გსურთ ჩვენი ვებგვერდის მონახულება, თქვენ უნდა ჩაეწეროთ ჩვენი ვებ-მისამართი ბრაუზერის მისამართში, რომელიც არის [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) _, სადაც [**ultimatemultisite.com**](http://ultimatemultisite.com) არის დომენის სახელი.

## სუბდომენი (Subdomain) {#subdomain}

სუბდომენი არის ვებგვერდების იერარქიის ერთ-ერთი ტიპი მთავარი დომენის ქვეშ, მაგრამ ნაცვლად ფოლდერების გამოყენებისა კონტენტის ორგანიზებისთვის ვებსაიტს საკუთარი არსებობა აქვს. ის წარმოდგენილია როგორც [**https://site1.domain.com/**](https://site1.domain.com/), სადაც _site1_ არის სუბდომენის სახელი და [_domain.com_](http://domain.com) არის მთავარი დომენი.

## სუბდი rikt (Subdirectory) {#subdirectory}

სუბდი rikt არის ვებგვერდების იერარქიის ერთ-ერთი ტიპი ძირითადი დომენის ქვეშ, რომელიც იყენებს ფოლდერებს კონტენტის ორგანიზებისთვის ვებსაიტზე. სუბდი rikt არის სუბფოლდერის მსგავსი და სახელები შეიძლება ურთიერთშრიანად გამოიყენებოდეს. ის წარმოდგენილია როგორც [**https://domain.com/site1**](https://domain.com/site1), სადაც _site1_ არის სუბდი riktის სახელი და [_domain.com_](http://domain.com) არის მთავარი დომენი.

## სუბსೈტი (Subsite) {#subsite}

ქვესაი, ქვესაი არის შვილი საიტი, რომელსაც თქვენ ქსელში (Multisite network) აყრდნით შექმნით. ის შეიძლება იყოს **ქვედა დომენი** (subdomain) ან **ქვედა დირექტორია** (subdirectory), რაც დამოკიდებულია იმაზე, თუ როგორ არის კონფიგურირებული თქვენი WordPress Multisite ინსტალაცია.

## სუპერ ადმინი (Super Admin) {#super-admin}

WordPress-ის სუპერ ადმინი არის მომხმარებლის როლი, რომელსაც აქვს სრული შესაძლებლობა მართოს მთელი ქვესაიტები Multisite ქსელში. Multisite მომხმარებლებისთვის ეს არის **ყველაზე მაღალი დონის წვდომა**, რაც შეგიძლიათ მიანიჭოთ თქვენს WordPress ინსტალაციას.

## პლაგინი (Plugin) {#plugin}

ზოგადად, პლაგინი არის კოდის ნაკრები, რომელიც დამატებით ფუნქციონალს ანიჭებს თქვენს WordPress საიტს. ეს შეიძლება იყოს როგორც მარტივი – მაგალითად, შესვლის ლოგოს შეცვლა, ასევე რთული – მაგალითად, ელექტრონული კომერციის ფუნქციონალის დამატება. _Woocommerce and Contact Form_ არის პლაგინების მაგალითები.

WordPress Multisite-ზე პლაგინებს შეუძლიათ ჩატვირთვა მხოლოდ სუპერ ადმინის დ^{*ბორდიდან* ქსელის ადმინისტრაციის საშუალებით. ქვესაიტის ადმინებს შეუძლიათ პლაგინების გააქტიურება და გაუქმება მხოლოდ საკუთარ ქვესაიტზე.

## თემები (Themes) {#themes}

WordPress-ის თემა არის ფაილების ჯგუფი (_გრაფიკა, სტილის ფაილები და კოდი_), რომელიც განსაზღვრავს საიტის საერთო გარეგნობას. ის უზრუნველყოფს ყველა ანიმაციურ და ვიზუალურ ელემენტს, როგორიცაა შრიფტის სტილი, გვერდის განლაგება, ფერები და ა.შ.

პლაგინების მსგავსად, WordPress Multisite-ზე თემებს შეუძლიათ ჩატვირთვა მხოლოდ სუპერ ადმინის მიერ და მათი გააქტიურება შესაძლებელია ქვესაიტის დონეზე ქვესაიტის ადმინების მიერ.

## საიტის შაბლონი (Site Template) {#site-template}

**საიტის შაბლონი** არის ბაზისური საიტი, რომელსაც შეიძლება გამოყენებულ იქნას როგორც საფუძველი ახალი საიტების შექმნისას თქვენს ქსელში.

ეს ნიშნავს, რომ შეგიძლიათ შექმნათ საბაზისო საიტი, გააქტიუროთ სხვადასხვა პლაგინები, დადოთ აქტიური თემა და დააკონტროლოთ ის istediğiniz სახით. შემდეგ, როდესაც თქვენი მომხმარებელი ახალ ანგარიშს ქმნის, მას არ მიეცეს სტანდარტული WordPress საიტი შინაარსის გარეშე, არამედ მიეცეს თქვენი საბაზისო საიტის ასლი, რომელშიც უკვე დანერგილია ყველა ცვლილება და კონტენტი.

## დომენის ამოჭრა (Domain Mapping) {#domain-mapping}

**Domain mapping** with WordPress არის გზა მომხმარებლების გადამისამართებისთვის სწორ ჰოსტზე, ვებსაიტის მისამართიდან. WordPress-ის მულტი-საიტში (Multisite) ქვესაიტები იქმნება ან დირექტორიის ქვეშ ან დომეინის ქვეშ. Domain mapping კი საშუალებას აძლევს ქვესაიტების მომხმარებლებს გამოიყენონ ტოპ-ლეველ დომენი, როგორიცაა [**joesbikeshop.com**](http://joesbikeshop.com), რათა მათი ვებ-მისამართი უფრო პროფესიონალურად გამოიყურებოდეს.

## SSL {#ssl}

SSL ნიშნავს **Secure Sockets Layer**. ეს არის ციფრული სერტიფიკატი, რომელიც ვებსაიტის იდენტობას ატუპირებს და უზრუნველყოფს დაშიფრულ კავშირს. დღეს ის გამოიყენება ინტერნეტ კავშირის დაცვის სტანდარტული ტექნოლოგია და ნებისმიერი მგრძნობიარე მონაცემების დაცვისთვის, რომელიც ორ სისტემას შორის გადადის, რაც ბოროტმოქმედებს აკონტროლებს ინფორმაციის წასაკითხობასა და შეცვლას, მათ შორის პერსონალურ მონაცემებს. თანამედროვე ბრაუზერები მოითხოვენ SSL-ს, რაც აუცილებელია ვებგვერდის შესაქმნელისა და გაშვებისთვის.

## Media (მედია) {#media}

Media არის სურათები, აუდიო, ვიდეო და სხვა ფაილები, რომლებიც ვებსაიტს ქმნის.

ქსელური საიტები WordPress Multisite-ში ერთ მონაცემთა ბაზას იზიარებენ და მედია ფაილებისთვის ცალკე გზებს ინახავენ ფაილის სისტემაში (filesystem).

სტანდარტული WordPress-ის მდებარეობა (wp-content/uploads) შენარჩუნებულია; თუმცა, მისი გზა იცვლება ქსელური საიტის უნიკალური ID-ს ასახვისთვის. შესაბამისად, ქსელური საიტის მედია ფაილები ჩნდება როგორც wp-contents/uploads/site/[id].

## Permalinks (პერმალინკები) {#permalinks}

Permalinks არის თქვენი ინდივიდუალური ბლოგის პოსტის ან გვერდის მუდმივი URL-ები თქვენს საიტზე. Permalinks ასევე ცნობილია როგორც **pretty links** (ყველაზე ლამაზი ბმულები). ნაგულისხმევად, WordPress-ის URL-ები იყენებს query string ფორმატს, რომელიც ასე გამოიყურება:

## Ultimate Multisite {#ultimate-multisite}

Ultimate Multisite არის WordPress плагин, რომელიც შექმნილია WordPress Multisite-ის ინსტალებებისთვის და გარდაქმნის თქვენს WordPress-ის ინსტალაციას პრემიუმ საიტების ქსელად – მაგალითად [WordPress.com]-ის მსგავსად – რაც საშუალებას აძლევს კლიენტებს შექმნან საიტები ყოველთვიური, კვარტალური ან წლიური საფასურებით (შეგიძლიათ ასევე შექმნათ უფასო პლანები).

## Checkout Form (გადახდის ფორმა) {#checkout-form}

Checkout Form არის ერთი ან მრავალეტაპიანი შეკვეთის ფორმა, რომელიც გულისხმობს ქვესაიტის, წევრობისა და მომხმარებლის ანგარიშების შექმნას Ultimate Multisite-ის რეგისტრაციის მეშვეობით. ის შედგება სხვადასხვა ველიდან და გადახდის ფორმებიდან, რომელთა შევსებასაც მომხმარებელი უნდა ჩაატაროს გაწევრიანების პროცესში.

## Webhook (ჰუკ) {#webhook}

Webhook (რომელსაც ასევე უწოდებენ web callback-ს ან HTTP push API-ს) არის გზა იმისთვის, რომ აპლიკაციამ სხვა აპლიკაციებს რეალურ დროში ინფორმაცია მიაწოდოს. Webhook მონაცემებს სხვა აპლიკაციებში მაშინვე გადასცემს, რაც ნიშნავს, რომ თქვენ მიიღებთ მონაცემებს დაუყოვნებლივ.

**Ultimate Multisite webhooks** უ infinite შესაძლებლობებს ხსნის, რაც ქსელის ადმინისტრატორებს საშუალებას აძლევს შეასრულონ ყველა სახის გენიალური და სასარგებლო ინტეგრაციები, განსაკუთრებით თუ მათ გამოიყენებენ Zapier-სა და IFTTT-ს მსგავს სერვისებთან ერთად.

## Events (ღონისძიებები) {#events}

Event არის მოქმედება, რომელიც ხდება მომხმარებლის ან სხვა წყაროს მოქმედების შედეგად, მაგალითად მაუსის დაჭერისას. Ultimate Multisite ინახავს ყველა ღონისძიებისა და ლოგის ჩანაწერს თქვენს მთელ ქსელში. ის აკონტროლებს სხვადასხვა აქტივობას თქვენს მულტი-საიტზე, როგორიცაა გეგმის ცვლილებები.
