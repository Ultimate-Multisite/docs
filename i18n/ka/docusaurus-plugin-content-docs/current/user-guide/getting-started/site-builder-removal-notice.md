# Site Builder Mode-ის აღმოფხვრის შესახებ შეტყობინება

**Site Builder რეჟიმი მოხსნილია Superdav AI Agent v1.12.0-ში.** თუ თქვენ იყენებდით Site Builder რეჟიმს, გთხოვთ გადახვიდეთ **Setup Assistant agent**-ზე თემის შექმნისა და საიტის დაყენებისთვის.

## რა მოხდა?

### Site Builder რეჟიმი (ძველი)

Site Builder რეჟიმი იყო ვიზუალური ინტერფეისი, რომელიც გამოიყენებოდა შემდეგისთვის:

- ტემპლეტებიდან საიტების შექმნა
- ძირითადი პარამეტრების კონფიგურაცია
- თემის შერჩევა
- თავდაპირველი კონტენტის დაყენება

### რა შეცვალა ამას?

**Setup Assistant agent** ახლა აკონტროლებს Site Builder ფუნქციონალის ყველა ნაწილს შემდეგნაირად:

- უფრო მოქნილი, აგენტზე ორიენტირებული დაყენება
- უკეთესი თემის მორგების ვარიანტები
- Theme Builder-თან ინტეგრაცია
- მომავალი სესიებისთვის `site_brief` მეხსიერების შენარჩუნება

## თუ Site Builder რეჟიმს იყენებდით

### თქვენი საიტები უსაფრთხოა

- Site Builder რეჟიმით შექმნილი არსებული საიტები weiterhin მუშაობს
- არანაირი მონაცემთა დაკარგვა ან საიტის გაჩერება
- შეგიძლიათ თქვენი საიტების ნორმალურად მართვა

### გადადით Setup Assistant Agent-ზე

ახალი საიტის დაყენებისთვის ან თემის ცვლილებებისთვის გამოიყენეთ Setup Assistant agent:

```
"Help me set up a new site"
```

ან

```
"Start the Theme Builder onboarding"
```

Setup Assistant agent იგივე ფუნქციონალს უფრო მეტი მოქნილობით გთავაზობთ.

## შედარება: Site Builder vs. Setup Assistant

| მახასიათებელი | Site Builder (მოხსნილი) | Setup Assistant (ახალი) |
|---------|----------------------|----------------------|
| დაყენების მეთოდი | ვიზუალური ფორმა (Wizard form) | აგენტის დიალოგი (Agent conversation) |
| თემის შერჩევა | წინასწარ განსაზღვრული ტემპლეტები | საკუთარი გენერაცია |
| მორგება | შეზღუდული ვარიანტები | სრული დიზაინის სისტემა |
| საიტის ბრიფ (Site brief) | არ ინახება | მეხსიერების შენარჩუნება |
| მომავალი სესიები | განმეორებით დაყენება | გამოყენებული `site_brief`-ის გამოყენება |
| მოქნილობა | ფიქსირებული სამუშაო პროცესი | ადაპტური დიალოგი |

## Setup Assistant Agent-ზე გადასვლა

### ახალი საიტებისთვის

შეიდაქმენი სೈტი Site Builder რეჟიმის გამოყენების ნაცვლად:

1. მოთხოვნა: "დამეხმარეთ ახალი საიტის დაყენებაში"
2. დაყენების ასისტენტ აგენტი გიხელგატავთ შემდეგში:
   - საიტის მიზნები და გეგმები
   - სამიზნე აუდიტორია
   - ბრენდის იდენტობა
   - თემის გენერირება
   - საწყისი კონფიგურაცია

### არსებული საიტებისთვის

თუ თქვენ გაქვთ არსებული საიტი Site Builder რეჟიმიდან:

1. შეგიძლიათ გამოიყენოთ ის როგორც არის
2. თემის განახლებისთვის, მოითხოვეთ: "დაასწორეთ ჩემი საიტი"
3. დაყენების ასისტენტ აგენტი დაგეხმარებათ ახალი თემის შექმნაში
4. თქვენი საიტის მონაცემები უცვლელი დარჩება

### თემა ცვლილებებისთვის

Site Builder რეჟიმის თემის არჩევის ნაცვლად:

1. მოთხოვნა: "შეცვალეთ ჩემი თემა"
2. დაყენების ასისტენტ აგენტი გააკეთებს შემდეგს:
   - შემოგთავაზებთ თქვენს დიზაინის პრეფერენციებს
   - შექმნის მორგებულ თემას
   - აქტიურებს მას თქვენს საიტზე

## ძირითადი განსხვავებები

### Site Builder რეჟიმი

```
1. აირჩიეთ ტემპლეტი
2. აირჩიეთ თემა
3. დააყენეთ საბაზისო პარამეტრები
4. დასრულებულია
```

### დაყენების ასისტენტ აგენტი

```
1. აღწერეთ თქვენი საიტის მიზანი
2. განსაზღვრეთ თქვენი სამიზნე აუდიტორია
3. აირჩიეთ დიზაინის პრეფერენციები
4. აგენტი გენერირებს მორგებულ თემას
5. აგენტი აქტიურებს თემას
6. საიტის შეჯამება ინახება მომავალი სესიებისთვის
```

## დაყენების ასისტენტ აგენტის უპირატესობები

### უფრო მოქნილი

- აღწერეთ თქვენი საიტი ბუნებრივი ენით
- მიიღეთ მორგებული რეკომენდაციები
- შეესაბამება თქვენს კონკრეტულ საჭიროებებს

### უკეთესი მორგება

- მორგებული თემის გენერირება
- დიზაინის სისტემის გადაწყვეტილებები
- მდგრადი დიზაინის ტოკენები (Persistent design tokens)

### მდგრადი მეხსიერება

- თქვენი `site_brief` ინახება
- მომავალი აგენტებს გააცნობთ თქვენს საიტს
- არ არის საჭირო დაყენების ინფორმაციის განმეორება

### ინტეგრირებული სამუშაო პროცესი (Integrated Workflow)

- თემის მშენებლობის ინტროექცია (Theme Builder onboarding)
- დიზაინის სისტემის ესთეტიკის უნარი
- ხილვადობის კონტროლის შესაძლებლობა
- ყველაფერი ერთმანეთთან უპრობლემოდ მუშაობს

## პრობლემების მოგვარება (Troubleshooting)

### ვერ ვპოულობ Site Builder რეჟიმს

Site Builder რეჟიმი აღარ არსებობს. გამოიყენეთ დაყენების ასისტენტ აგენტი:

"დამეხმარეთ ახალი საიტის დაყენებაში"

### მე მინდა საიტის აღდგენა Site Builder-იდან

თქვენ შეგიძლიათ ეს იგივე შექმნათ Setup Assistant აგენტს გამოყენებით:

1. მოთხოვნა: "დამეხმარეთ ახალი საიტის დაყენებაში"
2. აღწერეთ თქვენი ორიგინალური საიტის მიზანი და დიზაინი
3. აგენტი შექმნის მსგავს თემას
4. თქვენი `site_brief` შენახული იქნება მომავალი მითითებისთვის

### ჩემი არსებული Site Builder-ის საიტი არ მუშაობს

Site Builder რეჟიმში შექმნილი არსებული საიტები weiterhin მუშაობს. თუ პრობლემებს განიცდით:

1. შეამოწმეთ, რომ თქვენს თემას ჯერ კიდევ აქვს აქტიური
2. დაადასტურეთ, რომ თქვენი პლაგინები ჩართულია
3. შეამოწმეთ WordPress-ის შეცდომების ჟურნალები (error logs)
4. დაუკავშირდით მხარდაჭერას პრობლემები გაგრძელების შემთხვევაში

### შემიძლია თუ არა ჩემი ძველი Site Builder-ის შაბლონების გამოყენება?

Site Builder-ის შაბლონები აღარ არის ხელმისაწვდომი. თუმცა:

- თქვენი არსებული საიტები weiterhin მუშაობს
- შეგიძლიათ მსგავსი საიტების შექმნა Setup Assistant აგენტის მეშვეობით
- Setup Assistant აგენტი უფრო მეტ tùyzijბის შესაძლებლობას გთავაზობთ

## შემდეგი ნაბიჯები

1. **ახალი საიტებისთვის**: გამოიყენეთ Setup Assistant აგენტი
2. **არსებული საიტებისთვის**: გააგრძელეთ მათი გამოყენება ისე, როგორც არის
3. **თემის ცვლილებებისთვის**: მოითხოვეთ დახმარება Setup Assistant აგენტისგან
4. **დიზაინის დახვეწისთვის**: გამოიყენეთ Design System Aesthetics სკილი

## დაკავშირებული თემები

- **Theme Builder Onboarding**: მითითებითი დაყენება საკუთარი თემებისთვის
- **Setup Assistant Agent**: საიტის დაყენების აგენტის მიერ ხელმძღვანელობა
- **Site Specification Skill**: განსაზღვრეთ თქვენი საიტის მიზნები და აუდიტორია
- **Design System Aesthetics Skill**: დახვეწეთ თქვენი საიტის ვიზუალური იდენტობა
