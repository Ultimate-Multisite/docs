# Theme Builder Onboarding Flow {#theme-builder-onboarding-flow}

Superdav AI Agent v1.12.0-ით ახალი, წარმართული **Theme Builder onboarding flow** მოგご紹介თ, რომელიც დაგეხმარება თქვენი პირველი პარამეტრების დროს საკუთარი ბლოკ თემის შექმნაში. ეს ძველ Site Builder რეჟიმს უტივებს უფრო მოქნილ, აგენტი-დახმარებით მიდგომას.

## რა არის Theme Builder Onboarding Flow? {#what-is-the-theme-builder-onboarding-flow}

Theme Builder onboarding flow არის ინტერაქტიული დაყენების მითითებელი (wizard), რომელიც:

- გიხელმძღვანელებთ დიზაინის გადაწყვეტილებებში (ფერთა, ტიპოგრაფიის, განლაგების შესახებ)
- იწერს თქვენი ვებსაიტის ვიზუალური იდენტობის პრეფერენციებს
- გენერირებს საკუთარ ბლოკ თემას, რომელიც შექმნილი იქნება თქვენი საჭიროებებისთვის
- ავტომატურად ააქტიურებს თემას და სრულდება მისი შექმნა

ეს პროცესი მართავს **Setup Assistant agent**, რომელიც კითხვების დასმვით და incremental (ნაწილ-ნაწილ) თემის მშენებლობით გეხმარებათ.

## Theme Builder Onboarding-ის დაწყება {#starting-the-theme-builder-onboarding}

### პირველი uruchომბე (First-Run Setup) {#first-run-setup}

როდესაც პირველად გაუშვებთ Superdav AI Agent-ს ახალ WordPress ინსტალაციაზე, თქვენ დაინახავთ:

```
გამარჯობა Superdav AI Agent!

რას გსურთ?
1. შექმნა საკუთარი თემა (Theme Builder)
2. არსებული თემის გამოყენება
3. ამ ეტაპზე დაყენების გამოტოვება
```

აირჩიეთ **"Build a custom theme"** (შექმნა საკუთარი თემა), რომ გადახვიდეთ Theme Builder onboarding flow-ში.

### მექანიკური გააქტიურება (Manual Activation) {#manual-activation}

თქვენ შეგიძლიათ Theme Builder onboarding-ის დაწყება ნებისმიერ დროს, თუ მოითხოვთ:

```
"Start the Theme Builder onboarding"
```

ან

```
"Help me create a custom theme"
```

## Onboarding-ის ნაბიჯები {#the-onboarding-steps}

### ნაბიჯი 1: რეჟიმის შერჩევა (Mode Selection) {#step-1-mode-selection}

Setup Assistant agent გეკითხება თქვენს პრეფერენციაზე:

```
როგორ გსურთ თემის შექმნა?
- მითითებული (მე დაგეკითხებით და მე შევქმნი მას თქვენთვის)
- პრაქტიკული (მე გამოგიტანთ ვარიანტებს და თქვენ გადაწყვეტთ)
```

**მითითებულ რეჟიმს (Guided mode)** ყველაზე მეტ მომხმარებელს ვურჩევთ; აგენტი დიზაინის რეკომენდაციებს გააკეთებს თქვენი ინდუსტრიისა და მიზნების საფუძველზე.

### ნაბიჯი 2: საიტის სპეციფიკაცია (Site Specification) {#step-2-site-specification}

გეკითხება თქვენს შესახებ ინფორმაცია:

- **საიტის მიზანი**: ელექტრონული კომერცია (E-commerce), ბლოგი, პორტფოლიო, SaaS და ა.შ.
- **სამიზნე აუდიტორია**: ვინ არიან თქვენი ვიზიტორები?
- **ბრენდის ფერები**: ძირითადი და მეორადი ფერები (ან "არ ვიცი")
- **ტონი**: პროფესიული, კრეატიული, მხიარული, მინიმალისტური და ა.შ.

ეს ინფორმაცია ინახება თქვენს **site_brief** მეხსიერებაში, რომელსაც აგენტები მომავალ სესიებში იყენებენ.

### ეტაპი 3: დიზაინის სისტემის გადაწყვეტილებები (Design System Decisions) {#step-3-design-system-decisions}

აგენტი გიხელმძღვანელებთ დიზაინის ტოკენების (design tokens) შერ выбоრაში:

- **ტიპოგრაფი**: შრიფტის ოჯახი (serif, sans-serif, monospace) და ზომის სკეილი
- **ფერების პალიტრა**: ძირითადი, მეორადი, აქცენტისა და ნეიტრალური ფერები
- **სივრცე (Spacing)**: კომპაქტური, ნორმალური ან ღია განლაგებები
- **მოძრაობა (Motion)**: ანიმაციები და ტრანზიციები (თუ გსურთ)

### ეტაპი 4: თემის გენერაცია (Theme Generation) {#step-4-theme-generation}

Setup Assistant აგენტი თქვენს საკუთარ ბლოკ თემას ქმნის შემდეგ ელემენტებთან ერთად:

- `theme.json`, რომელიც შეიცავს ყველა თქვენს დიზაინის ტოკენს
- ბლოკის შაბლონები საერთო განლაგებებისთვის (სახლის გვერდი, ბლოგი, კონტაქტი)
- საკუთარი დიზაინის სისტემის შესაბამისი კस्टम ბლოკების სტილები
- თემის მეტამონაცემები და WordPress მხარდაჭერის დეკლარაციები

### ეტაპი 5: აქტივაცია და ვერიფიკაცია (Activation and Verification) {#step-5-activation-and-verification}

თემა ავტომატურად იაქტიურება, და თქვენ ნახავთ შემდეგს:

```
✓ თქვენი კერძო თემა ახლა მზადაა!
  თემის სახელი: [თქვენი საიტის სახელი] Theme
  ფარები: [Primary] / [Secondary]
  ტიპოგრაფი: [Font Family]

  გახსენით თქვენი საიტი და ნახეთ ახალი დიზაინი.
```

შემდეგ შეგიძლიათ შეხვიდეთ თქვენს საიტზე, რომ გადაამოწმოთ თემის სწორად გამოჩენა.

## საიტის სპეციფიკაცია და site_brief მეხსიერება (Site Specification and site_brief Memory) {#site-specification-and-sitebrief-memory}

ონბორдинგის დროს აგენტი იწერს თქვენი საიტის სპეციფიკაციას **site_brief** მეხსიერების კატეგორიაში. ეს მოიცავს:

- საიტის მიზნებსა და მიზნებს
- სამიზნე აუდიტორიას
- ბრენდის ფერებსა და ტონს
- დიზაინის პრეფერენციებს
- კონტენტის სტრუქტურას

### რატომ არის მნიშვნელოვანი site_brief {#why-sitebrief-matters}

მომავალ სესიებში, აგენტები იყენებენ თქვენს site_brief-ს შემდეგ:

დიზაინის თანმიმდევრულობის შენარჩუნება ცვლილებებისას
შეუმატებელი ფუნქციების შეთავაზება თქვენი საიტის მიზნಕ್ಕೆ შესაბამისად
კონტექსტის გათვალისწინებული რეკომენდაციები
გამოსაყენებელი და სერვისის შეკითხვის თავიდან აცილება

### თქვენი site_brief-ის ნახვა {#viewing-your-sitebrief}

შეგიძლიათ ჰკითხოთ აგენტს:

```
"Show me my site brief"
```

ან

```
"What do you know about my site?"
```

აგენტი გამოიტანს თქვენს შენახულ საიტის სპეციფიკაციას.

## ინტეგრირების შემდეგ დამატებითი კონფიგურაცია {#customizing-after-onboarding}

თემის აწყობის (Theme Builder) პროცესი დასრულების შემდეგ, შეგიძლიათ:

### გამოიყენოთ Design System Aesthetics-ის უნარი {#use-the-design-system-aesthetics-skill}
მოითხოვეთ დიზაინის გაუმჯობესება:

```
"Refine the typography to be more modern"
```

ან

```
"Adjust the color palette to be warmer"
```

**Design System Aesthetics-ის უნარი** გიხელთთავებთ მიზნობრივი დიზაინის განახლებების გავლის.

### შეცვალოთ theme.json პირდაპირ {#edit-themejson-directly}
უფრო მოწინავე მომხმარებლებისთვის, შეცვალეთ `/wp-content/themes/[theme-name]/theme.json` იმის შესაცために:

- ფერების ტოკენები (Color tokens)
- ტიპოგრაფიის მასშტაბები (Typography scales)
- ინტერვალები (Spacing values)
- საზღვრების და ჩრდილების განსაზღვრა (Border and shadow definitions)

### შექმენით საკუთარი ბლოკ ტემპლეატები {#create-custom-block-templates}
WordPress block editor-ს გამოიყენეთ საკუთარი ტემპლატების შესაქმნელად:

- მთავარი გვერდებისთვის (Homepage layouts)
- ბლოგის პოსტებისთვის (Blog post pages)
- პროდუქტის გვერდებისთვის (Product pages)
- კონტაქტის ფორმებისთვის (Contact forms)

## შედარება: ძველი და ახალი ინტეგრირება {#comparison-old-vs-new-onboarding}

| მახასიათებელი | Site Builder (ძველი) | Theme Builder (ახალი) |
|---|---|---|
| დაყენების მეთოდი | ვიზარდით შევსებული ფორმა | აგენტის მიერ წარმართული დიალოგი |
| თემის გენერაცია | შეზღუდული ტემპლეატები | საკუთარი სტრუქტურის (Custom scaffolding) შექმნა |
| დიზაინის ტოკენები | ხელით შეყვანა | წარმართული გადაწყვეტილებები |
| მოქნილობა | ფიქსირებული ვარიანტები | მორგებადი |
| მომავალი განახლებები | არ არის მითითებული | შენახულია site_brief-ში |

## პრობლემების გადაჭრა (Troubleshooting) {#troubleshooting}

**ინტეგრირების პროცესი არ დასრულდა**
- ხელახლა დაიწყეთ პროცესი: "Start the Theme Builder onboarding"
- შეამოწმეთ, რომ თქვენი WordPress ინსტალაცია განახლებულია
- დაადასტურეთ, რომ Setup Assistant აგენტი ჩართულია

**ჩემი `site_brief` გამოყენებული არ არის**
- დაადასტურეთ, რომ აგენტს აქვს მეხსიერება (memory)
- სთხოვეთ აგენტს: "დაასახელოს ჩემი საიტის ბრიფე" (recall my site brief)
- შეამოწმეთ, გააქტიურებულია თუ არა მეხსიერება თქვენს პარამეტრებში

**გენერირებული თემა არ ემთხვევა ჩემს პრეფერენციებს**
- გამოიყენეთ Design System Aesthetics სკილი (skill) მისი დახვეწისთვის
- სთხოვეთ აგენტს: "დაასახელოს თემის გენერაცია [კონკრეტული ცვლილებებით]" (regenerate the theme with [specific changes])
- შეცვალეთ `theme.json` პირდაპირ ზუსტი კონტროლისთვის

## შემდეგი ნაბიჯები {#next-steps}

თემა ბილდერის (Theme Builder) ინტეგრირების დასრულების შემდეგ:

1. **შეამოწმეთ თქვენი საიტი**: გადადით თქვენს საიტზე და ნახეთ ახალი თემა
2. **დახვეწეთ დიზაინი**: გამოიყენეთ Design System Aesthetics სკილი ცვლილებებისთვის
3. **დაამატეთ კონტენტი**: დაიწყეთ თქვენი საიტის კონტენტის აგება
4. **გამოიკვლიეთ შესაძლებლობები**: გაეცანით სხვა აგენტის შესაძლებლობებს, როგორიცაა `scaffold-block-theme` და `activate-theme`
