# Multisite დაყენების მეგზური (Wizard)

Ultimate Multisite-ს აქვს ჩაშენებული მეგზური, რომელიც ავტომატურად გარდაქმნის სტანდარტულ WordPress ინსტალაციას WordPress Multisite ქსელად. ეს გიშლით საჭიროებას, რომ ხელით შეცვალოთ `wp-config.php` ან გაუშვათ მონაცემთა ბაზის ბრძანებები.

:::tip
თუ თქვენი WordPress ინსტალაცია უკვე მუშაობს Multisite ქსელად, შეგიძლიათ ეს ნაბიჯი სრულად გამოტოვოთ. მეგზური მხოლოდ მაშინ ჩნდება, როდესაც Multisite ჯერ არ არის გააქტიურებული.
:::

## როდის ჩნდება მეგზური?

როდესაც თქვენ ააქტიურებთ Ultimate Multisite-ს სტანდარტულ (არასტანდარტულ) WordPress ინსტალაციაზე, პლაგინი აღმოაჩენს, რომ Multisite არ არის გააქტიურებული და ავტომატურად გადაგითანებთ Multisite Setup Wizard-ზე, ნაცვლად ჩვეულებრივი დაყენების მეგზურის.

ამის შესაძლებლობა შეგიძლიათ პირდაპირ მიიღოთ: **WP Admin > Ultimate Multisite > Multisite Setup**.

## წინაპირობები

მეგზურის გაშვებამდე დარწმუნდით, რომ:

- თქვენ გაქვთ **administrator წვდომა** თქვენს WordPress ინსტალაციაზე
- თქვენი სერვერის `wp-config.php` ფაილი არის **წერით доступ (writable)** ვებ-서ვერის მიერ
- თქვენ გაქვთ თქვენი ფაილებისა და მონაცემთა ბაზის **ახალი ბექაპი**

:::warning
მეგზური ცვლის თქვენს `wp-config.php` ფაილს და ქმნის ახალ მონაცემთა ბაზის ცხრილებს. ყოველთვის შექმენით ბექაპი სანამ გაგრძელებთ.
:::

## ნაბიჯი 1: მისასალმებელი (Welcome)

პირველი ეკრანი გ説明ებს, რა არის WordPress Multisite და რას აკეთებს მეგზური:

- ჩართეთ Multisite ფუნქცია თქვენს WordPress კონფიგურაციაში
- შექმენით საჭირო ქსელური მონაცემთა ბაზის ცხრილები
- დაამატეთ საჭირო multisite констанტები `wp-config.php`-ში
- გააქტიურეთ Ultimate Multisite ქსელზე მთელი ქსელის გავლით

![Multisite Setup Wizard - Welcome step](/img/installation/multisite-wizard/wizard-welcome.png)

დააჭირეთ **Continue** (გაგრძელება) წინსვლისთვის.

## ნაბიჯი 2: ქსელის კონფიგურაცია (Network Configuration)

ამ ნაბიჯში გთხოვთ, დააკონფიგურიროთ თქვენი ქსელური პარამეტრები.

### საიტის სტრუქტურა

როგორ გსურთ თქვენი ქსელური საიტების ორგანიზება?

- **ქვესაიდები** (რეკომენდებულია) — თითოეულ საიტს ექნება საკუთარი ქვესაიდი, მაგალითად `site1.yourdomain.com`
- **ქვედირექტორიები** — საიტები იქმნება როგორც გზები, მაგალითად `yourdomain.com/site1`

:::note
თუ თქვენ აირჩევთ ქვესაიდებს, დაგჭირდებათ კონფიგურირება **wildcard DNS-ის** და **wildcard SSL სერტიფიკატის** თქვენი დომეინისთვის. უმეტეს მართული WordPress ჰოსტები ამას ავტომატურად აკეთებს. დეტალური შედარებისთვის იხილეთ [Ultimate Multisite 101](./ultimate-multisite-101).
:::

### ქსელის სათაური (Network Title)

შეიყვანეთ სახელი თქვენი ქსელისთვის. ეს ნაგულისხმევად არის თქვენი ამჟამინდელი საიტის სათაური და მას ემატება "Network". შეგიძლიათ შეცვალოთ ეს დროLater ქსელის პარამეტრებიდან.

### ქსელის ადმინისტრატორის ელფოსტა (Network Admin Email)

ელფოსტის მისამართი, რომელიც გამოიყენება ქსელის ადმინისტრირების შეტყობინებებისთვის. ნაგულისხმევი არის ამჟამინდელი მომხმარებლის ელფოსტის მისამართი.

![Multisite Setup Wizard - Network Configuration](/img/installation/multisite-wizard/wizard-network-configuration.png)

ველების შევსების შემდეგ, დააჭირეთ **Continue**-ს პროცესის გასაგრძელებლად.

## ნაბიჯი 3: ინსტალაცია (Step 3: Installation)

დააჭირეთ ღილაკს **Install**, რომ დაიწყოთ. ვიზარდი ავტომატურად აკეთებს ხუთი ნაბიჯის შესრულებას თანმიმდევრობით, თითოეული მათგანი რეალურ დროში გ покаზავნით პროგრესს:

![Multisite Setup Wizard - Installation pending](/img/installation/multisite-wizard/wizard-installation-pending.png)

| ეტაპი | აღწერა |
|---|---|
| **Multisite-ის გააქტიურება** | დაამატეთ `WP_ALLOW_MULTISITE` კონსტანტა `wp-config.php`-ში |
| **ქსელის შექმნა** | შექმენით მულტი-საიტის მონაცემთა ცხრილები (`wp_site`, `wp_sitemeta`, `wp_blogs` და ა.შ.) და შეავსეთ ისინი თქვენი ქსელის კონფიგურაციით |
| **კონფიგურაციის განახლება** | დაამატეთ საბოლოო მულტი-საიტის კონსტანტები `wp-config.php`-ში (`MULTISITE`, `SUBDOMAIN_INSTALL`, `DOMAIN_CURRENT_SITE` და ა.შ.) |
| **კუקיების გამოსწორება** | დარწმუნდით, რომ საიტის URL სწორია ქსელის მეტამონაცემებში, რათა თავიდან აიცილოთ ავთენტიფიკაციის პრობლემები activations-ის შემდეგ |
| **ქსელის პლაგინის გააქტიურება** | გააქტიურეთ Ultimate Multisite ქსელში, რომ ის მთელი ქსელის მასშტაბით იმუშაოს |

ყოველი ეტაპი აჩვენებს ერთ-ერთ ამ სტატუსს:

- **Pending (დალოდებით)** — დამუშავებისთვის ელოდება
- **Installing... (და càiდება...)** — მიმდინარეობს
- **Success! (წარმატებულია!)** — წარმატებით დასრულდა
- **Error message (შეცდომის შეტყობინება)** — წარმოიშვა შეცდომა (შეტყობინება აღწერს პრობლემას)

მას შემდეგ, რაც ყველა ეტაპი წარმატებით დასრულდება, თითოეული ელემენტისთვის დაინახავთ მწვანე "Success!" სტატუსს:

![Multisite Setup Wizard - Installation complete](/img/installation/multisite-wizard/wizard-installation-complete.png)

შემდეგ ვიზუალური გზით ავტომატურად გადახვალთ დასრულების ეკრანზე.

## ეტაპი 4: დასრულება

ინსტალაცია დასრულების შემდეგ, თქვენ დაინახავთ წარმატების შეტყობინებას, რომელიც დაადასტურებს WordPress Multisite-ის გააქტიურებას.

![Multisite Setup Wizard - Setup Complete](/img/installation/multisite-wizard/wizard-complete.png)

ახლა შეგიძლიათ გააგრძელოთ Ultimate Multisite-ის სერვისის კონფიგურირება თქვენი WaaS პლატფორმისთვის (კომპანიის დეტალები, ნაგულისხმევი კონტენტი, გადახდის გეითვეები და ა.შ.).

:::note
მულტისაიტის ინსტალაციის დასრულების შემდეგ, თქვენი ბრაუზერი გადაამისამართებს ახლად გააქტიურებულ ქსელური ადმინისტრატორზე. შესაძლოა, დაგჭირდეთ ხელახლა შესვლა, რადგან ავтенტიზაციის ქუქიები განახლებულია მულტისაიტის გარემოსთვის.
:::

## მექანიკური დაყენების სარეზ dest (Manual Setup Fallback)

თუ ვიზარდი ვერ წერს თქვენს `wp-config.php` ფაილზე (ფაილის ნებართვის ან სერვერის შეზღუდვების გამო), ის აჩვენებს ზუსტ კოდს, რომელიც საჭიროა ხელით დამატება:

1. **wp-config.php კონსტანტები**, რომლებიც უნდა დაემატოს `/* That's all, stop editing! */` სტრიქონის ზემოთ
2. **.htaccess გადაწერის წესები** თქვენთვის შერჩეული საიტის სტრუქტურისთვის (ქვესაიტი ან ქვედირექტორია)

მექანიკური ცვლილებების შეტანის შემდეგ, განახლეთ გვერდი და ვიზარდი აღმოაჩენს, რომ მულტისაითი ახლა აქტიურია.

## პრობლემების მოგვარება (Troubleshooting)

### ვიზარდი ამბობს, რომ wp-config.php არ არის წucible

თქვენს ვებ-서ვერის პროცესს სჭირდება წვდომის უფლება `wp-config.php` ფაილზე. თქვენ შეგიძლიათ:

- დროებით შეცვალოთ ფაილის ნებართვები `644`-ზე ან `666`-ზე
- გამოიყენოთ ვიზარდის მიერ მოწოდებული მექანიკური დაყენების სარეზ dest ინსტრუქციები
- დაუკავშირდეთ თქვენს ჰოსტინგ პროვაიდერს დახმარებისთვის

### საიტებზე წვდომის შეფერხება დაყენების შემდეგ (ქვესაიტები)

თუ თქვენ აირჩიეთ ქვესაიტის სტრუქტურა, თქვენ გჭირდებათ **wildcard DNS** დომეინისთვის. დაამატეთ DNS ჩანაწერი:

```
Type: A (ან CNAME)
Host: *
Value: [თქვენი სერვერის IP]
```

შეამოწმეთ თქვენს ჰოსტინგ პროვაიდერთან, თუ როგორ უნდა დააკონფიგურიროთ ეს, თუ დარდებული არ ხართ.

### ავтенტიზაციის პრობლემები დაყენების შემდეგ

თუ გამოხვიდით სისტემიდან ან განიცდით ქუქიების შეცდომებს მულტისაიტის დაყენების შემდეგ:

1. გაასუფთავეთ თქვენი ბრაუზერის ქუქიები საიტისთვის
2. შესვლა ပြုულ하십시오Again `yourdomain.com/wp-login.php`-ზე
3. თუ პრობლემა გრძელდება, შეამოწმეთ, რომ თქვენს `wp-config.php`-ში არ არის დაყენებული `COOKIE_DOMAIN` მნიშვნელობა `false` — ეს ცნობილი პრობლემაა ქვესაიტის მულტისაიტის ინსტალაციებზე

### ინსტალაციის დროს ერთი ნაბიჯი ჩავარდა

თუ ინსტალაციის ნებისმიერ ეტაპზე შეცდომა ჩნდება:

1. დააკვირდით გამოტანილ შეცდომის സന്ദേശას.
2. დაბრუნდით კონფიგურაციის ეტაპზე და სცადეთ ხელახლა.
3. თუ შეცდომა კვლავ გრძელდება, შეამოწმეთ თქვენი სერვერის PHP შეცდომების лог (error log) მეტი დეტალებისთვის.
4. შემდეგ გაშვებებზე გამოტანილ ეტაპებს გამოვრიცხავთ (ინსტალერი იდენტურია).
