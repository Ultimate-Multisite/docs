# Hostinger Multi-Tenancy

Ultimate Multisite: Multi-Tenancy 1.2.0-ის დამატებით viene Hostinger multi-tenancy ფუნქცია, რაც საშუალებას იძლევა, რომ ჰოსტირებული დომეინებს მონაწილეობა მიანიჭონ სუვერენული ტენანტების პროვაიდინგში არსებული Hostinger დომეინის მართვის ინტეგრაციასთან ერთად.

გამოიყენეთ ეს შესაძლებლობა, როდესაც ტენანტის დომეინებს და იზოლირებულ ტენანტის ინფრასტრუქტურას აკონტროლებთ Hostinger hPanel-ის საშუალებით.

## დაყენების შენიშვნები

1. დააკონფიგურიруეთ ძირითადი Hostinger ინტეგრაცია **Ultimate Multisite > Settings > Host Integrations**-ში.
2. დაადასტურეთ, რომ Hostinger API ტოკენს შეუძლია მიზნობრივი დომეინის ან 서브დომეინის მართვა.
3. გააქტიურეთ Multi-Tenancy addon.
4. დააკონფიგურიруეთ ტენანტის იზოლაციის სტრატეგია ტენანტის გამოქვეყნებამდე.
5. გაუშვით მიგრაციის გადამოწმების სამუშაო პროცესი, სანამ პროდუქციულ ტრაფიკს ტენანტში გააგზავნით.

Hostinger-ის შესაძლებლობა იყენებს საერთო Hostinger-ის კავშირს ჰოსტის მხარეს ოპერაციებისთვის. DNS-მა მაინც უნდა მიუთითებდეს სწორ Hostinger ანგარიშზე, და hPanel-ის ანგარიშის ლიმიტები მაინც ძალაშია.

## სპეციფიკური ცვლილებები შესაძლებლობის შესახებ

- სუვერენული ტენანტები შეიძლება შეიქმნას ჰოსტის ცნობილი დომეინის ოპერაციებით.
-Same-machine database host strings ნორმალიზდება დადასტურების სანამ მიენიჭება.
- Hostinger-ის მართული ტენანტები უნდა გამოიყენონ მონაცემთა ბაზის ჰოსტის მნიშვნელობა, რომელიც காட்டப்பட்டுள்ளது hPanel-ში, თუ WordPress runtime არ საჭიროებს ადგილობრივ გადალახვას (override).
- SSO ვიზიტები ეყრდნობა იმას, რომ ტენანტის დომენი მიუთითებს Hostinger-ზე განთავსებულ ტენანტს.

## Hostinger ტენანტების პრობლემების მოგვარება

- თუ ტენანტის ინსტალაცია ჩავარდა, გადაამოწმეთ, არის თუ არა დომენი უკვე მიმაგრებული Hostinger ანგარიშთან.
- თუ მონაცემთა ბაზის ვერიფიკაცია ჩაიშალა, შეადარეთ ტენანტის DB მომხმარებელი, მონაცემთა ბაზის სახელი და ჰოსტბინდინგი hPanel-ს.
- თუ **Visit (SSO)** ჩავარდა, დაადასტურეთ DNS-ისა და SSL-ის აქტიურობა ტენანტის დომეინისთვის.
- თუ გათიშვისას ჰოსტის რესურსები დარჩება, მოაშორეთ ყველა დარჩენილი მონაცემთა ბაზა, მომხმარებელი ან საქაღალდე hPanel-დან სარეზერვოების დადასტურების შემდეგ.
