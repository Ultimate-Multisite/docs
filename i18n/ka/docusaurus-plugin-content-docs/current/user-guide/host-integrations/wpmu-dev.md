# WPMU DEV ინტეგრაცია {#wpmu-dev-integration}

## მიმოხილვა {#overview}
WPMU DEV არის ყოვლისმომცველი WordPress პლატფორმა, რომელიც სთავაზობს ჰოსტინგს,プラグინებს და სერვისებს WordPress-ის საიტებისთვის. ეს ინტეგრაცია უზრუნველყოფს დომეინის ავტომატურ სინქრონიზაციასა და SSL სერტიფიკატების მართვას Ultimate Multisite-სა და WPMU DEV ჰოსტინგს შორის.

## ფუნქციები {#features}
- დომეინის ავტომატური სინქრონიზაცია
- SSL სერტიფიკატების მართვა
- გაფართო SSL სერტიფიკატის ვერიფიკაციის მცდელობები

## მოთხოვნები {#requirements}
ინტეგრაცია ავტომატურად აღმოაჩენს, თუ WPMU DEV-ზე ხართ ჰოსტინგით და იყენებს ჩაშენებულ API-ს. თუ WPMU DEV-ზე ხართ ჰოსტინგით, დამატებითი კონფიგურაცია არ არის საჭირო.

ინტეგრაცია ითხოვს `WPMUDEV_HOSTING_SITE_ID` კონსტანტის არსებობას, რომელიც ავტომატურად განისაზღვრება WPMU DEV-ზე ჰოსტინგით.

## დაყენების ინსტრუქციები {#setup-instructions}

### 1. WPMU DEV ჰოსტინგის დადასტურება {#1-verify-wpmu-dev-hosting}

თუ WPMU DEV-ზე ხართ ჰოსტინგით, საჭირო კონსტანტები უკვე უნდა იყოს განსაზღვრული. შეამოწმეთ:

1. `WPMUDEV_HOSTING_SITE_ID` კონსტანტი განისაზღვრულია თქვენს გარემოში
2. გაქვთ აქტიური WPMU DEV-ის წევრი და API-ზე წვდომა

### 2. ინტეგრაციის ჩართვა {#2-enable-the-integration}

1. WordPress ადმინში გადადით Ultimate Multisite > Settings-ზე
2. გადადით "Domain Mapping" (დომეინის მაცნობა) ტაბულზე
3. დაწევთ ქვემოთ "Host Integrations" (ჰოსტინგის ინტეგრაციები)-ს
4. ჩართეთ WPMU DEV ინტეგრაცია
5. დააჭირეთ "Save Changes" (შეცვლილი შენახვა)

## როგორ მუშაობს {#how-it-works}

### დომეინის სინქრონიზაცია {#domain-syncing}

როდესაც Ultimate Multisite-ში დომენი მაცნობთ:

1. ინტეგრაცია იყენებს WPMU DEV API-ს, რომ დომენი დაამატოს თქვენს ჰოსტინგის ანგარიშში
2. ის ავტომატურად ამატებს დომენის www ვერსიასაც
3. WPMU DEV აკონტროლებს დომეინის კონფიგურაციასა და SSL სერტიფიკატების გაცემას

### SSL სერტიფიკატების მართვა {#ssl-certificate-management}

ინტეგრაცია დაყენებულია იმისთვის, რომ WPMU DEV ჰოსტინგის მხარდაჭერისთვის SSL სერტიფიკატების ვერიფიკაციის ცდების რაოდენობა გაიზარდოს, რადგან SSL სერტიფიკატების გამოცემის და დაყენების პროცესს შეიძლება დრო დასჭირდეს. ნაგულისხმევად, ის ცდილობს SSL სერტიფიკატის ვერიფიკაციას 10-ჯერ, რაც სტანდარტულ 5 ცდებთან შედარებით მეტია.

## მნიშვნელოვანი შენიშვნები {#important-notes}

### დომეინის მოხსნა {#domain-removal}

ამჟამად, WPMU DEV API არ აქვს საშუალება დომეინების წაშლისთვის. როდესაც Ultimate Multisite-ში დომეინის მაცნობა იშლება, დომენი დარჩება თქვენს WPMU DEV ჰოსტინგის ანგარიშში. საჭიროების შემთხვევაში, დაგჭირდებათ მისი ხელით მოხსნა WPMU DEV ჰოსტინგის პანელებიდან.

### API ავтентификаაცია {#api-authentication}

ინტეგრაცია იყენებს WPMU DEV API გასაღებს, რომელიც ინახება თქვენს WordPress მონაცემთა ბაზაში `wpmudev_apikey` ოფციონად. ეს ავტომატურად დაყენებულია მაშინ, როდესაც თქვენი საიტი WPMU DEV-თან უკავშირდება.

## პრობლემების მოგვარება (Troubleshooting) {#troubleshooting}

### API-ის კავშირის პრობლემები {#api-connection-issues}
- შეამოწმეთ, რომ თქვენი საიტი სწორად არის დაკავშირებული WPMU DEV-თან.
- გადაამოწმეთ, რომ `wpmudev_apikey` ოფცია დაყენებულია თქვენს WordPress მონაცემთა ბაზაში.
- დარწმუნდით, რომ თქვენი WPMU DEV წევრობა აქტიურია.

### SSL სერტიფიკატების პრობლემები {#ssl-certificate-issues}
- WPMU DEV-ს შეიძლება დასჭირდეს დრო SSL სერტიფიკატების გამოცემისათვის (ჩვეულებრივ 5-15 წუთი).
- ინტეგრაცია დაყენებულია SSL სერტიფიკატების შესამოწმებლად 10-ჯერ.
- თუ მრავალი ცდის შემდეგაც SSL სერტიფიკატები არ იცემა, დაუკავშირდით WPMU DEV მხარდაჭერას.

### დომეინის დამატება ვერ მოხერხდა {#domain-not-added}
- შეამოწმეთ Ultimate Multisite-ის лоgs (ლოგები) შეცდომის შეტყობინებების მიხედვით.
- გადაამოწმეთ, რომ დომენი უკვე არ არის დამატებული WPMU DEV-ში.
- დარწმუნდით, რომ თქვენი WPMU DEV ჰოსტინგის პლანი მხარს უჭერს იმ დომეინების რაოდენობას, რომელთა დამატებასაც ახორციელებთ.
