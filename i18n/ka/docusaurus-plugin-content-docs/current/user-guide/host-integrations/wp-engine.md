# WP Engine ინტეგრაცია {#wp-engine-integration}

## მიმოხილვა {#overview}
WP Engine არის პრემიუმ მართული WordPress ჰოსტინგ პლატფორმა, რომელიც უზრუნველყოფს WordPress საიტებისთვის ოპტიმიზებულ მუშაობას, უსაფრთხოებას და მასშტაბურობას. ეს ინტეგრაცია ივარდებს ავტომატურ დომეინის სინქრონიზაციას Ultimate Multisite-სა და WP Engine-ს შორის.

## ფუნქციები {#features}
- ავტომატური დომეინის სინქრონიზაცია
- ქვედომეინების მხარდაჭერა მულტი-საიტის ინსტალაციებისთვის
- უპრობლემო ინტეგრაცია WP Engine-ის არსებულ სისტემებთან

## მოთხოვნები {#requirements}
ინტეგრაცია ავტომატურად აღმოაჩენს, თუ WP Engine-ზე ხართ ჰოსტირებული და იყენებს ჩაშენებულ WP Engine API-ს. არ არის საჭირო დამატებითი კონფიგურაცია, თუ WP Engine პლაგინი აქტიური და სწორად არის დაყენებული.

თუმცა, თუ გჭირდებათ ინტეგრაციის ხელით კონფიგურაცია, შეგიძლიათ განსაზღვროთ ერთ-ერთი ეს константа თქვენს `wp-config.php` ფაილში:

```php
define('WPE_APIKEY', 'your_api_key'); // სასურველი მეთოდი
// ან
define('WPE_API', 'your_api_key'); // ალტერნატიული მეთოდი
```

## დაყენების ინსტრუქციები {#setup-instructions}

### 1. WP Engine პლაგინის შემოწმება {#1-verify-wp-engine-plugin}

თუ WP Engine-ზე ხართ ჰოსტირებული, WP Engine პლაგინი უკვე უნდა იყოს დაინსტალირებული და აქტიური. გადაამოწმეთ:

1. WP Engine პლაგინი აქტიურია
2. ფაილი `wp-content/mu-plugins/wpengine-common/class-wpeapi.php` არსებობს

### 2. ინტეგრაციის გააქტიურება {#2-enable-the-integration}

1. თქვენს WordPress ადმინში გადადით Ultimate Multisite > Settings-ზე
2. გადადით "Domain Mapping" (დომეინის და매პინირების) ჩანართზე
3. გადით ქვემოთ და იპოვეთ "Host Integrations" (ჰოსტის ინტეგრაციები)
4. გააქტიურეთ WP Engine ინტეგრაცია
5. დააჭირეთ "Save Changes" (შეცვლილი შენახვა)

## როგორ მუშაობს {#how-it-works}

### დომეინის სინქრონიზაცია {#domain-syncing}

როდესაც Ultimate Multisite-ში დომენი და매პინირდება:

1. ინტეგრაცია იყენებს WP Engine API-ს, რომ დომენი დაამატოს თქვენს WP Engine ინსტალაციაში
2. WP Engine აკონტროლებს დომეინის კონფიგურაციას და SSL სერტიფიკატის გაცემას
3. როდესაც დომეინის და매პინირება მოიხსნება, ინტეგრაცია დომენს WP Engine-დან წაშლის

### ქვედომეინების მხარდაჭერა {#subdomain-support}

ქვედად დომეინების მრავალსაიტის ინსტალაციებისთვის:

1. ინტეგრაცია ყოველ ახალი საიტის შექმნისას თითოეულ ქვედომეინს WP Engine-ში აკავშირებს.
2. WP Engine-ი აკონტროლებს ქვედომეინების კონფიგურაციას.
3. როდესაც საიტი წაიშლება, ინტეგრაცია ქვედომეინს WP Engine-დან მოხსნის.

## მნიშვნელოვანი შენიშვნები {#important-notes}

### Wildcard დომეინები (Wildcard Domains) {#wildcard-domains}

ქვედომეინების მრავალსაიტის ინსტალაციისთვის რეკომენდებულია WP Engine-თან მხარდაჭერის დაContacto, რომ მოითხოვოთ wildcard დომეინის კონფიგურაცია. ეს საშუალებას გაძლევთ, ყველა ქვედომეინი ავტომატურად იმუშაოს, გარეშე თითოეული მათგანის ინდივიდუალური დამატების საჭიროების გარეშე.

### SSL სერტიifikატები (SSL Certificates) {#ssl-certificates}

WP Engine-ი ავტომატურად უზრუნველყოფს SSL სერტიფიკატების გამოცემასა და განახლებას ყველა დომეინისთვის, რომელიც დაემატება ამ ინტეგრაციით. დამატებითი კონფიგურაცია საჭირო არ არის.

## პრობლემების მოხსნა (Troubleshooting) {#troubleshooting}

### API-ის კავშირის პრობლემები {#api-connection-issues}
- შეამოწმეთ, აქტიურია თუ არა WP Engine პლაგინი და სწორად არის კონფიგურირებული.
- თუ თქვენ თავად განსაზღვრეთ API key, გადაამოწმეთ, სწორია თუ არა ის.
- დაუკავშირდით WP Engine-ს მხარდაჭერას, თუ API-თან დაკავშირებით პრობლემები გაქვთ.

### დომეინის დამატება ვერ მოხერხდა {#domain-not-added}
- შეამოწმეთ Ultimate Multisite-ის лоgs (ლოგები) ნებისმიერი შეცდომის შესახებ.
- გადაამოწმეთ, რომ დომენი უკვე არ არის დამატებული WP Engine-ში.
- დარწმუნდით, რომ თქვენს WP Engine პლანზე მხარდაჭერა გაქვთ იმ დომეინების რაოდენობისთვის, რომელთა დამატებასაც აპირებთ.

### ქვედომეინების პრობლემები {#subdomain-issues}
- თუ ქვედომეინები არ მუშაობს, დაუკავშირდით WP Engine-ს მხარდაჭერას wildcard დომეინის კონფიგურაციისთვის.
- გადაამოწმეთ, სწორად არის კონფიგურირებული თქვენი DNS პარამეტრები მთავარი დომეინისა და ქვედომეინებისთვის.
