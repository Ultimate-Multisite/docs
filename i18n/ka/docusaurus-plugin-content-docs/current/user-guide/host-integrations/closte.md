# ინტეგრაცია Closte-ის

## მიმოხილვა
Closte არის მართული WordPress ჰოსტინგ პლატფორმა, რომელიც აგებულია Google Cloud-ის ინფრასტრუქტურაზე. ეს ინტეგრაცია უზრუნველყოფს დომეინის ავტომატურ სინქრონიზაციას და SSL სერტიფიკატების მართვას Ultimate Multisite-სა და Closte-ს შორის.

## ფუნქციები
- დომეინის ავტომატური სინქრონიზაცია
- SSL სერტიფიკატების მართვა
- Wildcard დომეინის მხარდაჭერა
- თუ Closte-ზე გაქვთ დაყენებული, კონფიგურაციის საჭიროება არ გექნებათ

## მოთხოვნები
თუ Closte-ს იყენებთ, თქვენს `wp-config.php` ფაილში უნდა განსაზღვრული იყოს შემდეგი константа:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

ეს константа ჩვეულებრივ უკვე განსაზღვრულია, თუ Closte-ზე გაქვთ ჰოსტინგი.

## დაყენების ინსტრუქციები

### 1. შეამოწმეთ თქვენი Closte API გასაღები (API Key)
თუ Closte-ზე ხართ ჰოსტინგის, `CLOSTE_CLIENT_API_KEY` константа უკვე უნდა იყოს განსაზღვრული თქვენს `wp-config.php` ფაილში. შეგიძლიათ ეს შეამოწმოთ თქვენი `wp-config.php` ფაილის საშუალებით.

### 2. ჩართეთ ინტეგრაცია
1. WordPress ადმინში გადადით Ultimate Multisite > Settings-ზე
2. გადადით "Domain Mapping" (დომეინის დამაப்பிჭებელი) ტაბულზე
3. გადით ქვემოთ და იპოვეთ "Host Integrations" (ჰოსტის ინტეგრაციები)
4. ჩართეთ Closte ინტეგრაცია
5. დააჭირეთ "Save Changes" (შეცვლილი შენახვა)

## როგორ მუშაობს
როდესაც დომენი Ultimate Multisite-ში დამაப்பிჭებელი ხდება:

1. ინტეგრაცია აგზავნის მოთხოვნას Closte-ის API-ს, რომ დომეინი თქვენს აპლიკაციაში დაამატოს
2. Closte ავტომატურად უზრუნველყოფს SSL სერტიფიკატების პროβისირებას (provisioning)
3. როდესაც დომეინის დამაப்பிჭებელს მოხდება, ინტეგრაცია დომეინს Closte-დან წაშლის

ინტეგრაცია ასევე მუშაობს Ultimate Multisite-ის DNS შემოწმების ინტერვალის პარამეტრებთან ერთად, რაც საშუალებას გაძლევთ დაარეგულიროთ, რამდენად ხშირად სისტემა იწყებს DNS-ის გავრცელებისა და SSL სერტიფიკატების გამოცემას.

## დომეინის ჩანაწერის შექმნა

ეს ინტეგრაცია უზრუნველყოფს, რომ როდესაც ვებსაიტი იქმნება ან დუბლიკდება, დომეინის ჩანაწერი ავტომატურად შეიქმნება. ეს განსაკუთრებით მნიშვნელოვანია Closte-ის ინტეგრაციისთვის, რადგან დომეინის ჩანაწერის შექმნა ააქტიურებს Closte API-ს, რომ ის დომენი და SSL სერტიფიკატი შექმნას.

## პრობლემების მოგვარება (Troubleshooting)

### API-ს კავშირის პრობლემები
- გადაამოწმეთ, სწორია თუ არა თქვენი Closte API გასაღები (key).
- დარწმუნდით, რომ თქვენს Closte ანგარიშს აქვს საჭირო უფლებები.

### SSL სერტიფიკატების პრობლემები
- Closte-ს შეიძლება დასჭირდეს დრო SSL სერტიფიკატების გასაცემად (ჩვეულებრივ 5-10 წუთი).
- გადაამოწმეთ, სწორად არის თუ არა თქვენი დომეინები მიმართული თქვენი Closte სერვერის IP მისამართისკენ.
- შეამოწმეთ DNS ჩანაწერები თქვენი დომეინისთვის და დარწმუნდით, რომ ისინი სწორად არის კონფიგურირებული.

### დომეინის დამატება არ ხდება
- გადაამოწმეთ Ultimate Multisite-ის лоgs (ლოგები) შეცდომის შეტყობინებებისთვის.
- დაადასტურეთ, რომ დომენი უკვე დამატებული არ არის Closte-ში.
- დარწმუნდით, რომ თქვენი დომეინის DNS ჩანაწერები სწორად არის კონფიგურირებული.

### DNS შემოწმების ინტერვალი
- თუ SSL სერტიფიკატების გასაცემას ძალიან ბევრი დრო სჭირდება, შეგიძლიათ შეცვალოთ DNS შემოწმების ინტერვალი Domain Mapping-ის პარამეტრებში.
- ნაგულისხმევი ინტერვალია 300 წამი (5 წუთი), მაგრამ შეგიძლიათ დააყენოთ ის 10 წამამდე, რათა ტესტირების დროს უფრო სწრაფად შეამოწმოთ.
