# ცნობილი შეუთავსებლობები სხვა პლაგინებთან

Ultimate Multisite-ის დაყენებისა და აქტივაციის შემდეგ, შესაძლოა პრობლემები წარმოიშვას სხვა პლაგინებთან არსებული და არ მოგვარებული შეუთავსებლობების გამო.

ჩვენ აქტიურად ვმუშაობთ ამ სიის შემცირებაზე ყოველი ახალი გამოშვებით, მაგრამ თუ პლაგინი აქ ჩამოთვლილია, შესაძლოა საჭირო გახდეს მისი დეაქტივაცია იმისთვის, რომ Ultimate Multisite-ის ქსელი სრულად ფუნქციონირებდეს.

აი მათი სია და რა უნდა გააკეთოთ ამ შემთხვევაში:

**ზოგიერთი WPMU DEV პლაგინი** ზოგიერთი პლაგინი, რომელიც აღარ არის მხარდაჭერაში WPMU DEV-ის მიერ, ცნობილია იმით, რომ Ultimate Multisite-ის რეგისტრაციის პროცესი ბოლო ნაბიჯზე (მომხმარებელს არ შეუძლია ანგარიშის შექმნა) წყდება. აი პლაგინების სია, რომლებიც ამ პრობლემებს იწვევს:

  * Signup Language: _Ultimate Multisite-ს აქვს დამატებითი ფუნქციონალი, რომელიც იგივე შესაძლებლობას გთავაზობს და ეწოდება Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _ეს ჩაშენებულია Ultimate Multisite-ში, ამიტომ დამატებითი პლაგინის საჭიროება არ არის._

  * New Blog Templates: _ეს ჩაშენებულია Ultimate Multisite-ში, ამიტომ დამატებითი პლაგინის საჭიროება არ არის._

  * Set Password on Multisite Blog Creation: _ეს ჩაშენებულია Ultimate Multisite-ში, ამიტომ დამატებითი პლაგინის საჭიროება არ არის._

  * Signup TOS: _ეს ჩაშენებულია Ultimate Multisite-ში, ამიტომ დამატებითი პლაგინის საჭიროება არ არის._

  * Anti-Splog: მიუხედავად იმისა, რომ ნაკლებად ძლიერია, ესეც _ჩაშენებულია Ultimate Multisite-ში._

**შენიშვნა**: ყველა სხვა WPMU Dev პლაგინი შეიძლება ნორმალურად გამოყენებულ იქნას Ultimate Multisite-თან ერთად. მაგალითები მოიცავს _Smush_ , _Forminator_ , _Defender,_ და ა.შ.
