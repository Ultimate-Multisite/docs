# Sunrise ფაილის ინსტალირების შეცდომა {#error-installing-the-sunrise-file}

`sunrise.php` ფაილი არის სპეციალური ფაილი, რომელსაც WordPress იძებს თავისი დაწყებისას (bootstraps). WordPress-ს იმისთვის, რომ `sunrise.php` ფაილი აღმოჩნდეს, ის უნდა იყოს განთავსებული **wp-content საქაღალდეში**.

როდესაც Ultimate Multisite-ს ააქტიურებთ და დაყენების მითითებით მიჰყვებით პროცესს (როგორც სკრინშუტშია), Ultimate Multisite ცდილობს ჩვენი `sunrise.php` ფაილის நகლას `wp-content` საქაღალდეში.

უმეტეს შემთხვევაში, ჩვენ შეგვიძლია ფაილის წარმატებით நகლვა და ყველაფერი იმუშავებს. თუმცა, თუ რამე არ არის სწორად დაყენებული (მაგალითად, საქაღალდის პermissions), შეიძლება აღმოჩნდეთ ასეთი სცენარი, რომლის დროსაც Ultimate Multisite-ს ფაილის நகლვა ვერ შეეძლება.

<!-- სკრინშუტი ხელმისაწვდომია: შეცდომის സന്Message, რომელიც აჩვენებს Sunrise-ის კოპირების წარუმატებლობას -->

ამის გამოსასწორებლად, შეგიძლიათ უბრალოდ நகლოთ `sunrise.php` ფაილი `wp-ultimo` პლაგინის საქაღალდეში და ჩასვათ თქვენს `wp-content` საქაღალდეში. ამის შემდეგ, გადატვირთეთ მითითების გვერდი და შემოწმებები უნდა წარმატებით დასრულდეს.

<!-- სკრინშუტი ხელმისაწვდომია: ფაილის მენეჯერი, რომელიც აჩვენებს sunrise.php-ს wp-ultimo პლაგინის საქაღალდეში --> ნებისმიერ შემთხვევაში, ეს შეიძლება მოითხოვოს თქვენი საქაღალდის perms-ების ზოგადი შემოწმებას მომავალი პრობლემების თავიდან ასაცილებლად (არა მხოლოდ Ultimate Multisite-თან, არამედ სხვა პლაგინებთან და თემებთანაც).

WordPress-ის ნაწილია **Health Check tool** (შეამოწმების ინსტრument) (მას შეგიძლიათ მიწვდეთ თქვენი მთავარი საიტის **admin panel > Tools > Health Check**-დან), რომელიც შეძლებს გითხრათ, არის თუ არა საქაღალდის perms-ები ისეთი მნიშვნელობისა, რომელსაც შეიძლება პრობლემები შეუქმნას WordPress-ს.
