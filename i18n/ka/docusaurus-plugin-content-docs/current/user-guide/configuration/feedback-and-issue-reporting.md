# მომხმარებლის უკუკავშირი და პრობლემების შესახებ შეტყობინება

Gratis AI Agent v1.5.0-ს აქვს ჩაშენებული სისტემა უკუკავშირისა და პრობლემების შესახებ შეტყობინებისთვის, რომელიც საშუალებას აძლევს საბოლოო მომხმარებლებს პირდაპირ чат ინტერფეისიდან აღნიშნონ დაუჭერელი პასუხები და დაუყოვნებლივ მიაწოდონ პრობლემები. ეს სისტემა მოიცავს თანხმობის მართვას, ავტომატურ შეტყობინების ბრძანებას და ბექენდზე AI-ით დამხმარე ტრიაჟს (triage).

## თითის ქვედა ღილაკი (Thumbs-Down Button)

AI ასისტენტის მიერ გაგზავნილ ყველა സന്ദേശთან თან ახლავს **თითის ქვედა** (👎) ღილაკი. როდესაც მომხმარებელი მას დააჭერს, შეუძლია პასუხის აღნიშნოს როგორც არასასასტიკი ან არასწორი.

- ღილაკი ჩნდება ყოველი ასისტენტის സന്ദേശის გვერდით მაუსით გადატანის (hover) დროს.
- მისი დაჭერით იხსნება **Feedback Consent Modal** (უკუკავშირის თანხმობის მოდალი).
- უკუკავშირი დაკავშირებულია საუბრის თემატური შლედთან და კონკრეტულ സന്ദേശთან.

## Feedback Consent Modal (უკუკავშირის თანხმობის მოდალი)

როდესაც მომხმარებელი აჭერს თითის ქვედა ღილაკს, მონაცემების გაგზავნამდე ჩნდება თანხმობის მოდალი. ეს მოდალი:

- განმარტავს, თუ რა ინფორმაცია იქნება შეგროვებული (საუბრის ნაწილი, ბრაუზერის კონტექსტი).
- სთხოვს მომხმარებელს დაადასტუროს, რომ თანახმა ხართ ამ მონაცემების გაზიარებაზე.
- უზრუნველყოფს ტექსტის შეყვანის ველი, რომლის საშუალებითაც მომხმარებელი აღწერს, რა პრობლემა მოხდა.
- გვთავაზობს **Cancel** (გაუქმება) ვარიანტს, რომლითაც შეუძლია დახურვა senza გაგზავნის.

უკუკავშირი არ ჩაიწერება მანამ, სანამ მომხმარებელი პირდაპირ არ დაადასტურებს.

## Auto-Prompt Feedback Banner (ავტომატური შეტყობინების ბანერი)

საუბრის დასრულებისას ასისტენტი შეიძლება გამოიტანოს **ავტომატური შეტყობინების ბანერი** — ეს არის არასასასტიკი შეტყობინება, რომელიც ეკითხება, იყო თუ არა სესია სასარგებლო.

ეს ბანერი ჩნდება ავტომატურად საუბრის სიგრძისა და შედეგების მიხედვით. ის უკავშირდება იმავე უკუკავშირის პროცესს, რაც თითის ქვედა ღილაკიდან იხსნება. მომხმარებელს შეუძლია ბანერის დახურვა უკუკავშირის გაგზავნის გარეშე.

## /report-issue სлэშ ბრძანება (/report-issue Slash Command)

მომხმარებლებს შეუძლიათ პირდაპირ შეტყობინების პროცესის დაწყება чаტის შეყვანში `/report-issue` ტექსტის ჩ입ვით. ეს ბრძანება:

- დაუყოვნებლივ იხსნის Feedback Consent Modal-ს.
- აღწერილობის ველი წინასწარ შევსებულია მიმდინარე საუბრის კონტექსტით.
- მომხმარებლებს შეუძლიათ დამატებითი დეტალების დამატება გაგზავნამდე.

`/report-issue` ბრძანება ხელმისაწვდომია ყველა ჩატის რეჟიმში (inline, floating widget, full-screen).

## AI-ით ტრიაჟირება (AI-Assisted Triage)

გაგზავნილი უკუკავშირი მიმართულია AI ტრიაჟირების შრისკენ, რომელიც:

- კატეგორიზაციას აძლევს რეპორტს (ფაქტობრივი შეცდომა, დახმარების შეუძლებელი პასუხი, მიუღებელი კონტენტი და ა.შ.).
- მიბATTACH-ავს შესაბამის საუბრის მეტამონაცემებს.
- ტრიაჟირების შეჯამებას გადასცემს კონფიგურირებულ **Feedback Endpoint**-ს (იხ. [Settings > Advanced](#settings-advanced)).

ეს პროცესი ამცირებს ხელით მიმოხილვის დროს, რადგან პირველ რიგში გამოჩნდება ყველაზე კრიტიკული საკითხები.

## Settings > Advanced {#settings-advanced}

უკუკავშირის ბექენდის ჩართვისთვის, დააკონფიგურირეთ შემდეგი ველი **Gratis AI Agent → Settings → Advanced**-ის ქვეშ:

| ველი | აღწერა |
|---|---|
| **Feedback Endpoint URL** | URL, რომელიც იღებს POST მოთხოვნებს უკუკავშირის მონაცემებით (JSON). |
| **Feedback API Key** | Bearer ტოკენი, რომელიც გაგზავნილია `Authorization` ჰედერში უკუკავშირის წარდგენის ავთენტიფიკაციისთვის. |

ორივე ველი ცარიელი დატოვეთ, რომ უკუკავშირის შეგროვება გამორთოთ. ინდივიდუალური უკუკავშირის ღილაკები მომხმარებლებისთვის ნაჩვენები რჩება, მაგრამ წარდგენილი მონაცემები არ გადაეცემა.

:::tip
უკუკავშირის ენდპოინტმა უნდა მიიღოს JSON სხეული, რომელშიც მინიმუმ `message_id`, `conversation_id`, `feedback_text` და `triage_category` ველი უნდა იყოს შევსებული. მოsiehe თქვენი ენდპოინტის პროვაიდერის დოკუმენტაცია მოსალოდნელი სქემის შესახებ.
:::
