# ჩაშენებული აგენტები

Gratis AI Agent v1.9.0-ს მოჰყვება ხუთი ჩაშენებული აგენტი, რომელთაგან თითოეული წინასწარ კონფიგურირებულია სპეციალიზებულ ინსტრუმენტებით, დახვეწილი სისტემური промпტებითა და საწყისი შეთავაზებებით, რომლებიც შეესაბამება იმ სფეროს საერთო ამოცანებს. აგენტებს შორის გადართვა ცვლის იმას, რისი გაკეთება შეუძლია ასისტენტს და როგორ უპასუხებს — თქვენი მხრიდან არანაირი კონფიგურაცია არ არის საჭირო.

## რა არის Agent?

თითოეული აგენტი არის სახელწოდებით კონფიგურირებული პროფილის პრეფილი, რომელიც აერთიანებს:

- **Tools** (ინსტრუმენტები) — ეს არის შესაძლებლობები, რომელთა გამოძახებაც აგენტი შეიძლება გააკეთოს (მაგალითად, Content Writer-ს აქვს წერის და რედაქტირების შესაძლებლობა პოსტების, გვერდებისა და საკონტაქტო ფორმების შესაქმნელთან დაკავშირებით; Design Studio-ს აქვს CSS-ისა და theme.json-ის გამოყენების შესაძლებლობა).
- **System prompt** (სისტემური промпტი) — ინსტრუქციები, რომლებიც განსაზღვრავს აგენტის ტონს, პრიორიტეტებსა და შეზღუდვებს.
- **Suggestions** (შთავაზებები) — წინასწარ დაწერილი промпტები, რომლებიც ჩატის ინტერფეისში გამოჩნდება, რათა სწრაფად დაიწყოთ მუშაობა.

## აგენტის არჩევის შესაძლებლობის მიღება

1. გახსენით **Gratis AI Agent** პანელი WordPress-ის ადმინ პანელ გვერდზე.
2. დააწექით ჩატის ჰედერის ზედა მარცხენა კუთხეში არსებულ **აგენტის ხატულას** (ხატულა შეიცვლება აქტიური აგენტის ასახვისთვის).
3. **Agent Picker** გაიხსნება როგორც ფორმის-ტაბლეტისზე გადაფარული ინტერფეისი. თითოეული აგენტი ჩამოთვლილია თავისი ხატულასთან, სახელით და ერთ სტრიქონით აღწერილობით.
4. დააწექით აგენტის სტრიქონს მის გასააქტიურებლად. ჩატის ჰედერი მყისიერად განახლდება.

შეგიძლიათ აგენტების გადართვა საუბრის შუაშიც — ახალი აგენტის სისტემური промпტი ძალაში შედის შემდეგ შეტყობინებაში.

## ხუთი ჩაშენებული აგენტი

### Content Writer (შინაარსის შემქმნელი)

**ფოკუსი:** პოსტების, გვერდებისა და საკონტაქტო ფორმების შექმნა და რედაქტირება.

**ხელმისაწვდომი ინსტრუმენტები:** `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form`, `get_option`, `list_post_types`

**რა კეთდება კარგად:**
- ბლოგის პოსტების მომზადება და გამოქვეყნება მოკლე აღწერილობის ან სტრუქტურის საფუძველზე
- ახალი საიტისთვის ლენდინგ გვერდების ბატჩების შექმნა
- საკონტაქტო და問い合わせ ფორმების აგება
- პოსტებზე ფეიჩერული სურათების დამატება URL-დან ან ძებნით

**საწყისი რეკომენდაციები:**
- *დაწერეთ 500 სიტვემდე ბლოგი, რომელიც ეხება WordPress multisite-ის უპირატესობებს.*
- *შექმენით About, Services და Contact გვერდები და გამოაქვეყნეთ.*
- *Contact გვერდზე დაამატეთ დაჯავშნის შეკითხვის ფორმა.*

---

### Site Builder (საიტის კონსტრუქტორი)

**ფოკუსი:** ერთი ბრძანებით სრულ ციკლის ვებგვერდის შექმნა.

**ხელმისაწვდომი ინსტრუმენტები:** `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`, `create_post`, `batch_create_posts`, `register_post_type`, `create_menu`, `add_menu_item`, `inject_custom_css`, `apply_theme_json_preset`, `install_ability`, `recommend_plugin`, `create_contact_form`

**რა კეთდება კარგად:**
- აღწერილი ბიზნეს ტიპისთვის მრავალფაზიანი საიტის აგების გეგმის გენერირება
- თითოეული ფაზის ავტონომიური შესრულება — სტრუქტურა, კონტენტი, ნავიგაცია, დიზაინი
- გეგმის შუაში შეცდომებიდან აღდგენა მექანიკურად და ყოველგვარი ხელით ჩარევის გარეშე
- აგების პროცესში რეკომენდებული პლაგინების ინსტალაცია
- საკონტაქტო ფორმების პირდაპირი შექმნა чаტის ინტერფეისიდან (Superdav AI Agent v1.10.0+)

**საწყისი რეკომენდაციები:**
- *შექმენით ფოტოპორტფოლიოს საიტი, რომელიც მოიცავს გალერეის პოსტ ტიპს, დაჯავშნის გვერდს და საკონტაქტო ფორმას.*
- *შექმენით რესტორნის ვებგვერდი, რომელზეც იქნება ონლაინ მენიუ, სალარო დროები და მაგიდის დაჯავშნის შეკითხვის ფორმა.*
- *დაამატეთ საკონტაქტო ფორმა Contact გვერდზე Site Builder-ის გამოყენებით.*

### Design Studio (დიზაინის სტუდია)

**ფოკუსი:** ვიზუალური კസ്റ്റმაიზაცია — ფერები, ტიპოგრაფია, CSS და ბლოკების შაბლონები.

**ხელმისაწვდომი ინსტრუმენტები:** `inject_custom_css`, `apply_theme_json_preset`, `get_global_styles`, `set_global_styles`, `reset_global_styles`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `capture_screenshot`, `review_page_design`

**რა არის კარგი:**
- ნიმუშების (preset) გამოყენება (minimal-dark, warm-editorial, corporate-blue, vibrant-startup, classic-blog).
- გლობალური ტიპოგრაფიისა და ფერების დახვეწა theme.json-ის საშუალებით.
- ბრენდს სპეციფიკური გადაფარვისთვის კस्टम CSS-ის ჩასმა.
- გვერდის სკრინშოტების გადაღება და დიზაინის პრობლემების შესამოწმებლად.

**საწყისი შეთავაზებები:**
- *გამოიყენეთ warm-editorial პრესეტები და შემდეგ დააყენეთ ძირითადი ფერი #2d6a4f.*
- *გაიღეთ სკრინშოტი მთავარი გვერდის და მითხარით, თუ რა გაუმჯობესებთ.*
- *შექმენით ხელახლა გამოყენებადი hero block pattern ფართო ფონის სურათით და ცენტრში განლაგებული სათაურით.*

### Plugin Manager (პლაგინების მენეჯერი)

**ფოკუსი:** WordPress-ის პლაგინების აღმოჩენა, ინსტალაცია და მართვა.

**ხელმისაწვდომი ინსტრუმენტები:** `list_available_abilities`, `install_ability`, `recommend_plugin`, `get_option`, `set_option`

**რა არის კარგი:**
- საუკეთესო პლაგინის რეკომენდაცია აღწერილი გამოყენების შემთხვევისთვის.
- რეესტრებიდან ability packs-ის ინსტალაცია.
- ხელმისაწვდომი ability catalogue-ის კატეგორიების მიხედვით przeglądვა.

**საწყისი შეთავაზებები:**
- *რა არის საუკეთესო პლაგინი წევრების დირექტორიისთვის?*
- *დააინსტალირეთ WooCommerce abilities pack.*
- *მჩვენეთ ყველა ხელმისაწვდომი ელექტრონული კომერციის ability pack.*

### Support Assistant (ხელმძღვანელი მხარდაჭერა)

**ფოკუსი:** კითხვებზე პასუხის გაცემა საიტის კონტენტზე, პარამეტრებზე და WordPress-ის კონფიგურაციაზე.

**ხელმისაწვდომი ინსტრუმენტები:** `get_option`, `list_options`, `list_post_types`, `list_taxonomies`, `list_menus`, `list_available_abilities`

**რა კეთდება კარგად:**
- მიმდინარე საიტის პარამეტრებისა და ვარიანტების შემოწმება.
- იმის ახსნა, თუ რა ტიპის პოსტები (post types), ტაქსონომიები (taxonomies) და მენიუებია კონფიგურირებული საიტზე.
- "რა აკეთებს ეს პარამეტრი?" კითხვებზე პასუხის გაცემა ცოცხალი მნიშვნელობების წაკითხვით.
- ცვლილებების შეტანამდე მხოლოდ წაკითხვისთვის დიაგნოსტიკური შრე (read-only diagnostic layer) ფუნქციონირება.

**საწყისი შეთავაზებები:**
- *რა პლაგინები და პარამეტრებია ამ საიტზე ამჟამად აქტიური?*
- *დაასახელეთ ყველა კस्टम post type, რომელიც რეგისტრირებულია ამ საიტზე.*
- *რა ნავიგაციური მენიუები არსებობს და სად არის დანიშნული ისინი?*

## აგენტების კონფიგურაცია (Customising Agents)

ყოველი ჩაშენებული აგენტი შეიძლება გაფართოებული ან შეცვლილი იყოს `gratis_ai_agent_agents` ფილტრის გამოყენებით.

### არსებულ აгенტს კერძო სისტემური промпტის დამატება (Adding a custom system prompt to an existing agent)

```php
add_filter( 'gratis_ai_agent_agents', function ( array $agents ): array {
    if ( isset( $agents['content-writer'] ) ) {
        $agents['content-writer']['system_prompt'] .= "\n\nAlways write in British English and use the Oxford comma.";
    }
    return $agents;
} );
```

### ახალი აგენტის რეგისტრაცია (Registering a new agent)

```php
add_filter( 'gratis_ai_agent_agents', function ( array $agents ): array {
    $agents['seo-specialist'] = [
        'name'          => 'SEO Specialist',
        'description'   => 'საიტის პოსტებისა და გვერდების ოპტიმიზაცია საძიებო სისტემებისთვის.',
        'icon'          => 'dashicons-search',
        'tools'         => [ 'get_option', 'set_option', 'create_post', 'update_post', 'list_post_types' ],
        'system_prompt' => 'შენ ხარ SEO სპეციალისტი. ფოკუსირდი საკვანძო სიტყვების ოპტიმიზაციაზე, მეტა აღწერილობებზე და სტრუქტურირებულ მონაცემებზე.',
        'suggestions'   => [
            'შეამოწმე მთავარი გვერდის სათაური და მეტა აღწერა.',
            'დაასახელე საუკეთესო 5 პოსტისთვის სათაურის ტეგების გაუმჯობესება.',
        ],
    ];
    return $agents;
} );
```

ახალი აგენტი გამოჩნდება Agent Picker-ში ფილტრის გაშვებისთანავე.

### ჩაშენებული აგენტის მოხსნა

```php
add_filter( 'gratis_ai_agent_agents', function ( array $agents ): array {
    unset( $agents['plugin-manager'] );
    return $agents;
} );
```
