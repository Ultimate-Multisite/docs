# Domain Seller

**Multisite Ultimate: Domain Seller** თქვენს WordPress Multisite ქსელს მძლავრ დომენების გაყიდვის პლატფორმად გარდაქმნის. მიეცით მომხმარებლებს საშუალება, რეგისტრაციისას ან თავიანთი Dashboard-დან პირდაპირ დაარეგისტრირონ მორგებული დომენები, ხელმისაწვდომობის რეალურ დროში შემოწმებითა და გამჭვირვალე ფასებით.

### რატომ Domain Seller?

დომენის რეგისტრაცია ერთ-ერთი ყველაზე მოთხოვნადი დამატებითი გაყიდვაა ნებისმიერი WaaS (Website as a Service) პლატფორმისთვის. იმის ნაცვლად, რომ მომხმარებლები მესამე მხარის რეგისტრატორთან გაუშვათ, შეგიძლიათ დომენები პირდაპირ შესთავაზოთ — მეტი შემოსავალი შეინარჩუნოთ თქვენს ქსელში და მომხმარებლებს შეუფერხებელი გამოცდილება მისცეთ.

### ძირითადი შესაძლებლობები

#### მრავალი დომენის რეგისტრატორის მხარდაჭერა
* **OpenSRS** — სრული რეგისტრაცია, განახლება და მართვა
* **Namecheap** — სრული ინტეგრაცია WhoisGuard-თან და sandbox ტესტირებასთან
* **HostAfrica** — დომენის ძიება, ფასები, რეგისტრაცია, გადატანა, DNS და ID დაცვა Domains Reseller API-ის მეშვეობით
* **Openprovider** — გადამყიდველის ფასები, რეგისტრაცია, განახლება, გადატანები, DNS, WHOIS კონფიდენციალურობა და TLD სინქრონიზაცია
* **Hostinger** — დომენის ძიება და რეგისტრაცია საერთო Hostinger ინტეგრაციის token-ის გამოყენებით
* **გაფართოებადი არქიტექტურა** — აგებულია Integration Registry ნიმუშზე, რაც მეტი პროვაიდერის დამატებას ამარტივებს
* **ჩაშენებული კავშირის ტესტირება** — გადაამოწმეთ პროვაიდერის მონაცემები და კავშირი live რეჟიმში გადასვლამდე

#### მართვადი დაყენების ოსტატი
* **ერთკლიკიანი დაწყება** — გატარებთ პროვაიდერის კონფიგურაციაში, TLD-ის იმპორტსა და დომენის პროდუქტის შექმნაში
* **ავტომატური TLD სინქრონიზაცია** — იმპორტირებს ყველა ხელმისაწვდომ TLD-ს და საბითუმო ფასებს თქვენი დაკავშირებული პროვაიდერებიდან
* **ნაგულისხმევი პროდუქტის შექმნა** — ქმნის გამოსაყენებლად მზა დომენის პროდუქტს გონივრული ნაგულისხმევი მარკაფებით

#### მოქნილი დომენის ფასები
* **სამი მარკაფის რეჟიმი** — პროცენტული მარკაფი, ფიქსირებული მარკაფი ან ფიქსირებული ფასი თითო TLD-ზე
* **TLD-ზე მორგებული პროდუქტები** — შექმენით ცალკე დომენის პროდუქტები სხვადასხვა TLD ჯგუფებისთვის (.com/.net, .io, ქვეყნების კოდები და ა.შ.)
* **საწყისი ფასები** — შესთავაზეთ ფასდაკლებული პირველი წლის ფასი ახალი რეგისტრაციების მოსაზიდად
* **რეალურ დროში ფასები** — live საბითუმო ფასები პროვაიდერებიდან ჭკვიანი ქეშირებით
* **კუპონების მხარდაჭერა** — სრული თავსებადობა Multisite Ultimate-ის ფასდაკლების კოდებთან

#### WHOIS კონფიდენციალურობის დაცვა
* **თითო პროდუქტის კონფიგურაცია** — დააყენეთ WHOIS კონფიდენციალურობა როგორც გამორთული, ყოველთვის ჩართული ან მომხმარებლის არჩევანი
* **მომხმარებლის თანხმობით ჩართვა** — როდესაც დაყენებულია მომხმარებლის არჩევანზე, checkout-ის checkbox მომხმარებლებს აძლევს კონფიდენციალურობის დაცვის დამატების საშუალებას
* **კონფიგურირებადი ფასები** — დააყენეთ წლიური ფასი კონფიდენციალურობის add-on-ისთვის ან შესთავაზეთ უფასოდ
* **რეგისტრატორთან ინტეგრაცია** — რეგისტრაციის დროს ავტომატურად რთავს WhoisGuard-ს (Namecheap) ან WHOIS კონფიდენციალურობას (OpenSRS)

#### ერთიანი Checkout გამოცდილება
* **სამი დომენის რეჟიმი** — მომხმარებლები ირჩევენ ქვედომენს (უფასო), ახალი დომენის რეგისტრაციას ან უკვე საკუთრებაში არსებული დომენის მიბმას
* **დომენის რეალურ დროში ძიება** — AJAX-ზე დაფუძნებული ხელმისაწვდომობის შემოწმება გვერდის გადატვირთვის გარეშე
* **ჭკვიანი შემოთავაზებები** — ალტერნატიული TLD შეთავაზებები, როდესაც სასურველი დომენი დაკავებულია
* **გამჭვირვალე ფასები** — რეგისტრაციის ფასის, განახლების ფასისა და არასავალდებულო WHOIS კონფიდენციალურობის საფასურის მკაფიო ჩვენება
* **რეგისტრანტის საკონტაქტო ველები** — აგროვებს საჭირო რეგისტრანტის ინფორმაციას (სახელი, მისამართი, ტელეფონი) checkout-ის დროს inline რეჟიმში
* **ავტომატურად გენერირებული საიტის URL** — საიტის მისამართი ავტომატურად წარმოიქმნება არჩეული დომენიდან

#### ავტომატური განახლების სისტემა
* **Membership-თან მიბმული განახლებები** — დომენები ავტომატურად ახლდება, სანამ მომხმარებლის membership აქტიურია
* **ჭკვიანი დაგეგმვა** — განახლების დავალებები რიგში დგება დომენის ვადის გასვლის თარიღების მიხედვით
* **ხელახალი მცდელობის ლოგიკა** — ჭკვიანი ხელახალი მცდელობები წარუმატებელი განახლების მცდელობებისთვის
* **ელფოსტის შეტყობინებები** — კონფიგურირებადი გაფრთხილებები მომავალი განახლებებისთვის, წარმატებული განახლებებისთვის, წარუმატებლობებისა და ვადის გასვლებისთვის

#### მომხმარებლის DNS მართვა
* **თვითმომსახურების DNS** — მომხმარებლებს შეუძლიათ თავიანთი Dashboard-დან დაამატონ, შეცვალონ და წაშალონ DNS ჩანაწერები (A, AAAA, CNAME, MX, TXT)
* **დომენის სტატუსი** — რეალურ დროში რეგისტრაციის სტატუსი, ვადის გასვლის თარიღები და nameserver ინფორმაცია
* **დომენის ავტომატური მიბმა** — რეგისტრირებული ან არსებული დომენები ავტომატურად ებმება მომხმარებლის საიტს

#### ადმინისტრატორის ხელსაწყოები
* **დომენის ხელით რეგისტრაცია** — დაარეგისტრირეთ დომენები მომხმარებლების სახელით ადმინისტრატორის პანელიდან
* **დომენის მონიტორინგი** — ნახეთ ყველა მომხმარებლის დომენი, სტატუსი და ვადის გასვლის თარიღი ერთ გვერდზე
* **პროვაიდერზე მორგებული ლოგირება** — ცალკე ლოგის არხები თითოეული რეგისტრატორისთვის (მაგ., domain-seller-namecheap.log)
* **რეგისტრაციის დეტალები** — ნახეთ რეგისტრანტის ინფორმაცია და რეგისტრაციის მონაცემები ძირითად დომენის რედაქტირების გვერდზე

### მხარდაჭერილი პროვაიდერები

* **OpenSRS** — სრული ინტეგრაცია რეგისტრაციასთან, განახლებასთან, DNS მართვასთან და WHOIS კონფიდენციალურობასთან
* **Namecheap** — სრული ინტეგრაცია რეგისტრაციასთან, განახლებასთან, ხელმისაწვდომობის ჯგუფურ შემოწმებასთან და უფასო WhoisGuard-თან
* **HostAfrica** — რეგისტრაცია, განახლება, გადატანები, DNS მართვა, EPP კოდის მიღება, რეგისტრატორის lock და ID დაცვა
* **Openprovider** — რეგისტრაცია, განახლება, გადატანები, DNS მართვა, EPP კოდის მიღება, რეგისტრატორის lock, WHOIS კონფიდენციალურობა და გადამყიდველის TLD სინქრონიზაცია
* **Hostinger** — რეგისტრაცია, დომენის მოძიება, nameserver-ების განახლებები, რეგისტრატორის lock და WHOIS კონფიდენციალურობა; განახლებებს ამუშავებს Hostinger-ის ავტომატური განახლება
* **GoDaddy** — რეგისტრაცია და განახლება
* **ResellerClub** — რეგისტრაცია, განახლება და DNS მართვა
* **NameSilo** — რეგისტრაცია და განახლება
* **Enom** — რეგისტრაცია, განახლება და DNS მართვა

### ტექნიკური დეტალები

* **მოითხოვს:** PHP 7.4+, Multisite Ultimate 2.4.12+
* **არქიტექტურა:** Integration Registry ნიმუში Domain_Selling_Capability interface-ით
* **ავტოჩატვირთვა:** Jetpack autoloader PSR-4 class mapping-ით
* **სტანდარტები:** WordPress-ის კოდირების სტანდარტები, მკაცრი ტიპიზაცია, მომზადებული statements

## ინსტალაცია

1. დარწმუნდით, რომ **Multisite Ultimate v2.4.12+** დაინსტალირებულია და ქსელურად გააქტიურებულია
2. ატვირთეთ plugin მისამართზე `/wp-content/plugins/ultimate-multisite-domain-seller/`
3. ქსელურად გააქტიურეთ plugin **Plugins**-დან WordPress Network Admin-ში
4. **Setup Wizard** ავტომატურად გაეშვება პირველი აქტივაციისას — მიჰყევით ნაბიჯებს, რათა:
   * დააკავშიროთ თქვენი დომენის რეგისტრატორი
   * შეამოწმოთ კავშირი
   * შემოიტანოთ ხელმისაწვდომი TLD-ები და საბითუმო ფასები
   * შექმნათ თქვენი პირველი დომენის პროდუქტი ფასის ნამატით
5. დაამატეთ **Domain Selection** ველი თქვენს checkout ფორმას **Ultimate Multisite > Checkout Forms**-ში

## ხშირად დასმული კითხვები

### Multisite Ultimate-ის რომელი ვერსიაა საჭირო?

ვერსია 2.4.12 ან უფრო მაღალი. addon თავსებადი არ არის WP Ultimo v1.x-თან ან Ultimate Multisite-ის 2.4.12-ზე დაბალ ვერსიებთან.

### რომელი დომენის პროვაიდერებია მხარდაჭერილი?

მხარდაჭერილია ცხრა რეგისტრატორი: **OpenSRS**, **Namecheap**, **HostAfrica**, **Openprovider**, **Hostinger**, **GoDaddy**, **ResellerClub**, **NameSilo** და **Enom**. შესაძლებლობები პროვაიდერის მიხედვით განსხვავდება, რადგან რეგისტრატორების API-ები სხვადასხვა ოპერაციებს ამჟღავნებს; პროვაიდერების მიხედვით მიმოხილვისთვის იხილეთ დაყენების გზამკვლევი. addon-ის Integration Registry არქიტექტურა მარტივს ხდის მორგებული პროვაიდერების დამატებას — იხილეთ [დეველოპერის გზამკვლევი](/developer/integration-guide/custom-registrar).

### შემიძლია WHOIS კონფიდენციალურობის დაცვა შევთავაზო?

დიახ. თითოეულ დომენის პროდუქტს აქვს WHOIS Privacy პარამეტრი სამი რეჟიმით: გამორთული, ყოველთვის ჩართული ან მომხმარებლის არჩევანი. როდესაც დაყენებულია მომხმარებლის არჩევანზე, მომხმარებლები checkout-ის დროს ხედავენ მოსანიშნ ველს ფასით (ან "უფასო" Namecheap-ის WhoisGuard-ისთვის). კონფიდენციალურობის ალამი რეგისტრაციის დროს ავტომატურად გადაეცემა რეგისტრატორს.

### შემიძლია სხვადასხვა TLD-ებისთვის განსხვავებული ფასები დავაყენო?

დიახ. შექმენით ცალკეული დომენის პროდუქტები სხვადასხვა TLD ჯგუფებისთვის. თითოეულ პროდუქტს შეიძლება ჰქონდეს საკუთარი ნამატის ტიპი (პროცენტი, ფიქსირებული ნამატი ან ფიქსირებული ფასი), მხარდაჭერილი TLD-ები და შესავალი ფასები.

### რა checkout ვარიანტებს ხედავენ მომხმარებლები?

Domain Selection ველი მომხმარებლებს აძლევს სამ ჩანართს: **Subdomain** (თქვენს ქსელში უფასო ქვედომენის გამოყენება), **Register New Domain** (ახალი დომენის ძიება და რეგისტრაცია) და **Existing Domain** (მათ მიერ უკვე ფლობილი დომენის მიბმა). გამოჩნდება მხოლოდ ის რეჟიმები, რომლებსაც ჩართავთ.

### მხარს უჭერს ავტომატურ განახლებებს?

დიახ. დომენები შეიძლება დაყენდეს ავტომატურ განახლებაზე და დაკავშირებულია მომხმარებლის membership სტატუსთან. განახლების გადახდები მუშავდება მომხმარებლის არსებული გადახდის gateway-ის მეშვეობით.

### შეუძლიათ მომხმარებლებს საკუთარი DNS ჩანაწერების მართვა?

დიახ. მომხმარებლები თავიანთ dashboard-ში იღებენ DNS მართვის ინტერფეისს A, AAAA, CNAME, MX და TXT ჩანაწერების დასამატებლად, რედაქტირებისა და წასაშლელად. DNS მართვა ხელმისაწვდომია OpenSRS, ResellerClub, Enom, HostAfrica და Openprovider-თან. Hostinger DNS ჰოსტირებული დომენებისთვის მუშავდება ძირითადი Hostinger დომენის-მიბმის ინტეგრაციით.

### არსებობს დაყენების ოსტატი?

დიახ. დაყენების ოსტატი ავტომატურად ეშვება პირველი აქტივაციისას და ნაბიჯ-ნაბიჯ გატარებთ რეგისტრატორის დაკავშირებას, TLD-ების შემოტანასა და თქვენი პირველი დომენის პროდუქტის შექმნას. ასევე შეგიძლიათ ოსტატი ხელახლა გაუშვათ addon-ის პარამეტრებიდან.
