# Loco Translate {#loco-translate}

Ultimate Multisite: Loco Translate Integration დამატება თქვენს WordPress multisite ქსელში კონკრეტული საიტისთვის განკუთვნილ თარგმნის შესაძლებლობებს ამატებს. თუ ოდესმე დაგჭირვებიათ, რომ თქვენს ქსელში სხვადასხვა საიტს ერთი და იგივე plugin-ის ან theme-ის განსხვავებული თარგმანები ჰქონოდა, ეს დამატება იდეალური გადაწყვეტაა.

**რას აკეთებს ეს დამატება**

ეს დამატება შეუფერხებლად აერთიანებს Ultimate Multisite-ს პოპულარულ Loco Translate plugin-თან და საშუალებას აძლევს თქვენს ქსელში თითოეულ ქვესაიტს, ჰქონდეს საკუთარი მორგებული თარგმანები. მართავთ მრავალრეგიონულ პლატფორმას, ემსახურებით სხვადასხვა ენობრივ საზოგადოებებს თუ უბრალოდ გჭირდებათ თარგმნის მოქნილობა თქვენს ქსელში, ეს დამატება ყველაფერს მარტივს ხდის.

**იდეალურია შემდეგისთვის:**

* მრავალრეგიონული ბიზნესები განსხვავებული ლოკალიზაციის საჭიროებებით
* საგანმანათლებლო დაწესებულებები, რომლებიც მრავალფეროვან საზოგადოებებს ემსახურებიან
* SaaS პლატფორმები, რომლებიც მრავალენოვან მხარდაჭერას სთავაზობენ
* ვებ სააგენტოები, რომლებიც მრავალი კლიენტის საიტს მართავენ
* ნებისმიერი multisite ქსელი, რომელსაც თარგმნის მოქნილობა სჭირდება

## ინსტალაცია {#installation}

1. დარწმუნდით, რომ Ultimate Multisite დაყენებულია და ქსელში გააქტიურებულია
2. დააყენეთ და გაააქტიურეთ Loco Translate plugin
3. ატვირთეთ ეს დამატება თქვენს plugins დირექტორიაში
4. ქსელში გაააქტიურეთ Loco Translate Integration დამატება
5. დაიწყეთ კონკრეტული საიტისთვის განკუთვნილი თარგმანების შექმნა Loco Translate-ის მეშვეობით ნებისმიერ ქვესაიტზე

სულ ეს არის! ინტეგრაცია ავტომატურად მუშაობს დამატებითი კონფიგურაციის გარეშე.

## ხშირად დასმული კითხვები {#frequently-asked-questions}

## ინსტალაციის შემდეგ რამე უნდა დავაკონფიგურირო? {#do-i-need-to-configure-anything-after-installation}

არა! ინტეგრაცია ავტომატურია. როგორც კი დამატებაც და Loco Translate-იც აქტიური იქნება, ქვესაიტებს დაუყოვნებლივ შეეძლებათ საკუთარი მორგებული თარგმანების შექმნის დაწყება.

## რა მოხდება, თუ ქვესაიტი მორგებულ თარგმანებს არ შექმნის? {#what-happens-if-a-subsite-doesnt-create-custom-translations}

საიტი ავტომატურად გამოიყენებს ქსელის მასშტაბით არსებულ თარგმანებს, როგორც სარეზერვო ვარიანტს. არაფერი გაფუჭდება და ყველაფერი შეუფერხებლად იმუშავებს.

## შეუძლიათ თუ არა ქვესაიტებს ქსელის მასშტაბით არსებულ თარგმანებზე წვდომა? {#can-subsites-access-network-wide-translations}

ჩვეულებრივი საიტის ადმინისტრატორები მხოლოდ საკუთარი საიტის თარგმანებს ხედავენ და არედაქტირებენ. სუპერ ადმინისტრატორებს შეუძლიათ ჰქონდეთ წვდომა როგორც ქსელის მასშტაბით არსებულ, ისე კონკრეტული საიტისთვის განკუთვნილ თარგმანებზე, იმის მიხედვით, რომელ საიტს ათვალიერებენ.

## მუშაობს ეს როგორც plugins-ებთან, ისე themes-ებთან? {#does-this-work-with-both-plugins-and-themes}

დიახ! კონკრეტული საიტისთვის განკუთვნილი თარგმანები მუშაობს როგორც WordPress plugins-ებისთვის, ისე themes-ებისთვის.

## შეანელებს ეს ჩემს multisite ქსელს? {#will-this-slow-down-my-multisite-network}

არა. დამატება იყენებს ფაილების ეფექტიან ჩატვირთვას, რომელიც კონკრეტული საიტისთვის განკუთვნილ თარგმანებს მხოლოდ საჭიროებისას ამოწმებს. წარმადობაზე შესამჩნევი გავლენა არ არის.

## შემიძლია თუ არა ქსელის თარგმანების კონკრეტულ საიტზე საწყის წერტილად კოპირება? {#can-i-copy-network-translations-to-a-specific-site-as-a-starting-point}

დიახ! Loco Translate-ში თარგმანების რედაქტირებისას შეგიძლიათ მარტივად დაიწყოთ ქსელის თარგმანებით და შემდეგ მოარგოთ ისინი თითოეულ საიტს.

## მუშაობს ეს WooCommerce-თან და სხვა plugins-ებთან? {#does-this-work-with-woocommerce-and-other-plugins}

რა თქმა უნდა! ნებისმიერი plugin ან theme, რომელიც WordPress-ის სტანდარტულ თარგმნის ფუნქციებს იყენებს, იმუშავებს კონკრეტული საიტისთვის განკუთვნილ თარგმანებთან.
