# Admin Page Creator

## შინაარსი
Admin Page Creator საშუალებას მოგცემთ მარტივად დაამატოთ ახალი ადმინისტრატორის გვერდები და ქვე-გვერდები WordPress-ის ადმინისტრატორის Dashboard-ში. მისი გამოყენება შეიძლება მისასალმებელი გვერდის ან სასწავლო გვერდის შესაქმნელად, რომელიც ახალი მომხმარებლებისთვის საწყისი გვერდის როლს შეასრულებს. გვერდების მარტივად დასაპროექტებლად შეგიძლიათ გამოიყენოთ გვერდების კონსტრუქტორები, როგორიცაა Beaver Builder, Elementor, Brizy და Oxygen.

ის ასევე გთავაზობთ მორგებული HTML-ის, CSS-ისა და JavaScript-ის მხარდაჭერას, ასევე წვდომის კონტროლს გეგმისა და მომხმარებლის როლის მიხედვით!

## ფუნქციები
* მორგებული HTML-ისა და CSS-ის მხარდაჭერა
* JavaScript-ის მხარდაჭერა
* PHP შინაარსის რეჟიმი
* ნებართვები მომხმარებლის როლის მიხედვით
* გვერდების კონსტრუქტორების მხარდაჭერა

## დოკუმენტაცია
[მორგებული ადმინისტრატორის გვერდებისა და ქვე-გვერდების შექმნა Multisite Ultimate: Admin Page Creator-ის გამოყენებით](https://help.multisiteultimate.com/article/305-creating-custom-admin-pages-and-sub-pages-using-ultimate-multisite-admin-page-creator)

## ინსტალაცია

1. ატვირთეთ **ultimate-multisite-admin-page-creator** **/wp-content/plugins/** დირექტორიაში
2. გაააქტიურეთ plugin **Plugins** მენიუს მეშვეობით თქვენს WordPress Network Admin-ში

## გამოყენების გზამკვლევი

მიუხედავად იმისა, რომ WordPress ძალიან ამარტივებს შინაარსის დამატებას საიტების წინა ნაწილში, ადმინისტრატორის მხარეს გვერდებისა და ქვე-გვერდების შექმნა გარკვეულ საბაზისო კოდირების უნარებს მოითხოვს. Ultimate Multisite: Admin Page Creator მიზნად ისახავს, თქვენი ქსელის საიტების უკანა ნაწილშიც შემოიტანოს გამოყენების ისივე სიმარტივე, რაც წინა ნაწილს აქვს.

## რატომ გამოვიყენოთ Admin Page Creator?

შესაძლოა გსურთ თქვენს კლიენტებს ადმინისტრატორის პანელზე შესთავაზოთ მორგებული გვერდი სასწავლო მასალებით, ან დაამატოთ მორგებული გვერდი ხშირად დასმული კითხვებით და საკონტაქტო ვიჯეტით. Admin Page Creator-ის შესაძლებლობები უსასრულოა და სწორედ ამას განვიხილავთ დღევანდელ სახელმძღვანელოში.

## თქვენი პირველი მორგებული ადმინისტრატორის გვერდის შექმნა

Ultimate Multisite: Admin Page Creator-ის ინსტალაციისა და გააქტიურების შემდეგ გადადით თქვენი ქსელის ადმინისტრატორის მენიუში და გახსენით Admin Pages.

![მორგებული ადმინისტრატორის გვერდების სია Network Admin-ში](/img/admin/admin-pages-list.png)

_მორგებული ადმინისტრატორის გვერდების სია Network Admin-ში_

ნახავთ აქამდე შექმნილი ყველა ადმინისტრატორის გვერდის სიას (რადგან ეს თქვენი პირველი ვიზიტია, სია ცარიელი იქნება). გააგრძელეთ და დააწკაპუნეთ **ახალი Admin Page-ის დამატება**. გადამისამართდებით Admin Page რედაქტორზე.

## მორგებული ადმინისტრატორის გვერდის ელემენტები

Admin Page რედაქტორში გექნებათ რამდენიმე განსხვავებული ვარიანტი თქვენი გვერდის მოსარგებად.

### გვერდის სათაური

ეს არის სათაური, რომელიც გამოჩნდება გვერდის ზედა ნაწილში და არა ადმინისტრატორის მენიუს წარწერა.

_შეიყვანეთ გვერდის სათაური_

## მენიუს პარამეტრები

გვერდით, General Options მეტა-ბოქსში, შეძლებთ მენიუს ტიპის, წარწერის და კიდევ ბევრი სხვა რამის კონფიგურაციას.

### მენიუს ტიპები

თქვენს ახალ მორგებულ ადმინისტრატორის გვერდს შეიძლება ჰქონდეს ორი განსხვავებული მენიუს ტიპი: ზედა დონის მენიუ და ქვემენიუ.

ზედა დონის მენიუს გვერდი, როგორც სახელიდან ჩანს, თქვენს ახლად შექმნილ გვერდს ადმინისტრატორის მენიუში ნაგულისხმევი WordPress გვერდების გვერდით დაამატებს.

თუ ზედა დონის ვარიანტს აირჩევთ, ასევე შეძლებთ მენიუს რიგითობისა და მენიუს ხატულის არჩევას.

თუ გსურთ ეს გვერდი ქვემენიუს გვერდად აქციოთ, შეძლებთ მშობელი გვერდის არჩევას, რომელსაც ეს გვერდი მიებმება. შეგიძლიათ თქვენი მორგებული გვერდი დაამატოთ ნაგულისხმევ WordPress გვერდებს ან თქვენს ზედა დონის მორგებულ ადმინისტრატორის გვერდებს.

## გაფართოებული პარამეტრები

თქვენ უნდა შეძლოთ ყველანაირი შინაარსის დამატება თქვენს მორგებულ ადმინისტრატორის გვერდებზე. ეს მოიცავს CSS წესებსა და ფაილებს, JavaScript-ს და გარე ბიბლიოთეკებსაც. ამის გაკეთება შეგიძლიათ Advanced Options ჩანართის გამოყენებით.

### მორგებული CSS

CSS ჩანართზე შეგიძლიათ დაამატოთ თქვენი საკუთარი მორგებული CSS კოდი, ასევე შემოიტანოთ CSS ფაილები დისტანციური საიტებიდან (როგორიცაა CDN-ები, Google Fonts და ა.შ.).

### მორგებული JavaScript

იგივე ვრცელდება JavaScript-ზეც. შეგიძლიათ დაამატოთ გარე ბიბლიოთეკები თქვენს მორგებულ JavaScript კოდში გამოსაყენებლად.

## ნებართვები

Advanced Options-შიც შეძლებთ აირჩიოთ, რომელ ქვე-საიტებს ექნებათ წვდომა ამ გვერდზე. ეს სასარგებლოა, თუ გეგმავთ სხვადასხვა გეგმის დონეებისთვის განსხვავებული ექსკლუზიური შინაარსის მიწოდებას, მაგალითად.

<!-- ეკრანის სურათი მიუწვდომელია: Admin Page Creator ნებართვების პარამეტრები გეგმისა და როლზე დაფუძნებული წვდომის კონტროლისთვის -->

_თქვენ გაქვთ სრული კონტროლი იმაზე, სად გამოჩნდება ეს გვერდი_

**მნიშვნელოვანია** : გვერდის გამოსაჩენად ორივე პირობა უნდა შესრულდეს. ეს ნიშნავს, რომ თუ მომხმარებელი მოცემული გეგმის კლიენტია, მაგრამ არ აქვს ერთ-ერთი არჩეული როლი (ან პირიქით), გვერდი მის პანელზე არ გამოჩნდება.

## აქტიური

თუ გსურთ გამორთოთ მორგებული ადმინისტრატორის გვერდი მისი წაშლის ან ნებართვებში ჩარევის გარეშე, უბრალოდ მოხსენით მონიშვნა Active პარამეტრს.

ზედა დონის გვერდის დეაქტივაცია ავტომატურად დეაქტივაციას გაუკეთებს ყველა ქვე-გვერდს, რომელიც ამ მშობელ გვერდს უკავშირდება.
