# Multi-Network

Ultimate Multisite: Multinetwork დამატება აფართოებს თქვენს Ultimate Multisite პლატფორმას ძლიერი multinetwork ფუნქციონალით. ეს დამატება თქვენს მომხმარებლებს აძლევს საშუალებას შეიძინონ და მართონ მთელი WordPress multisite ქსელები, და არა მხოლოდ ცალკეული საიტები.

## ძირითადი შესაძლებლობები {#key-features}

* **ქსელის მომზადება**: ავტომატურად შექმენით და დააკონფიგურირეთ სრული WordPress multisite ქსელები თქვენი მომხმარებლებისთვის
* **ქსელის შაბლონები**: მიეცით მომხმარებლებს საშუალება დაათვალიერონ, წინასწარ ნახონ და აირჩიონ წინასწარ მომზადებული ქსელის შაბლონი რეგისტრაციამდე
* **ქსელის მართვა**: სრული მართვის შესაძლებლობები multinetwork გარემოებისთვის
* **იზოლირებული ქსელები**: თითოეული მომხმარებლის ქსელი დამოუკიდებლად მუშაობს საკუთარი მონაცემთა ბაზის ცხრილებითა და კონფიგურაციით
* **ქსელებს შორის SSO**: მიეცით მომხმარებლებს უწყვეტი წვდომა მათ ქვე-ქსელებზე magic-link ავთენტიფიკაციით
* **მასშტაბირებადი არქიტექტურა**: ეფექტურად დაამუშავეთ მრავალი ქსელი ოპტიმიზებული რესურსების განაწილებით
* **ინტეგრაცია**: შეუფერხებლად ინტეგრირდება Ultimate Multisite-ის გამოწერისა და ბილინგის სისტემებთან

## გამოყენების შემთხვევები {#use-cases}

* **White Label ჰოსტინგი**: მიაწოდეთ მომხმარებლებს საკუთარი ბრენდირებული multisite ქსელები
* **სააგენტოებისთვის გადაწყვეტილებები**: მისცეთ სააგენტოებს საშუალება მართონ მრავალი კლიენტის ქსელი ერთი პლატფორმიდან
* **საწარმოს დანერგვები**: მხარი დაუჭირეთ დიდ ორგანიზაციებს რთული მრავალქსელიანი მოთხოვნებით
* **საგანმანათლებლო პლატფორმები**: შექმენით ცალკეული ქსელები სხვადასხვა სკოლისთვის ან დეპარტამენტისთვის

## მოთხოვნები {#requirements}

* Ultimate Multisite 2.0.0 ან უფრო მაღალი
* WordPress Multisite ინსტალაცია
* PHP 7.4 ან უფრო მაღალი
* MySQL 5.6 ან უფრო მაღალი

## ინსტალაცია {#installation}

1. ატვირთეთ plugin-ის ფაილები `/wp-content/plugins/ultimate-multisite-multinetwork` დირექტორიაში
2. გაააქტიურეთ plugin WordPress-ის 'Plugins' ეკრანიდან
3. დააკონფიგურირეთ multinetwork პარამეტრები Ultimate Multisite → Settings-ის მეშვეობით

## ქსელის შაბლონები {#network-templates}

ვერსია 1.3.0 ამატებს შაბლონების აღმოჩენისა და არჩევის პროცესს მომხმარებლებისთვის, რომლებიც ახალ ქსელებს ყიდულობენ. გამოიყენეთ ქსელის შაბლონები, როდესაც გსურთ, რომ თითოეული ახალი მომხმარებლის ქსელი დაიწყოს მომზადებული სტრუქტურით, დიზაინით, კონტენტის ბიბლიოთეკით ან კონკრეტულ მიმართულებაზე მორგებული კონფიგურაციით.

### შაბლონის წინასწარი დათვალიერება {#template-previewer}

ქსელის შაბლონის წინასწარი დათვალიერების ინსტრუმენტი მომხმარებლებს აძლევს საშუალებას დაათვალიერონ ხელმისაწვდომი შაბლონები გადახდამდე. თითოეული შაბლონი შეიძლება გაიხსნას ცოცხალი წინასწარი დათვალიერების პანელში, რათა მომხმარებლებმა შეადარონ განლაგებები და კონტენტი ქსელის შეძენამდე.

გამოიყენეთ წინასწარი დათვალიერების ინსტრუმენტი, რათა დაეხმაროთ მომხმარებლებს:

* გადახედონ თითოეული ხელმისაწვდომი ქსელის შაბლონის იერსახესა და სტრუქტურას
* შეადარონ რამდენიმე შაბლონი შეძენის პროცესის დატოვების გარეშე
* აირჩიონ შაბლონი, რომელიც საუკეთესოდ შეესაბამება მათ გამოყენების შემთხვევას რეგისტრაციამდე

შეინარჩუნეთ შაბლონების სახელები და აღწერები მომხმარებლისთვის გასაგებ ფორმაში. წინასწარი დათვალიერების ინსტრუმენტი ყველაზე სასარგებლოა მაშინ, როდესაც მომხმარებლებს შეუძლიათ გაიგონ განკუთვნილი გამოყენების შემთხვევა, მაგალითად სააგენტოების ქსელები, სკოლების ქსელები, ფრენჩაიზის ქსელები ან white-label ჰოსტინგის პაკეტები.

### შაბლონის ამრჩევი გადახდისას {#template-picker-at-checkout}

შაბლონის ამრჩევი შეიძლება დაემატოს რეგისტრაციის გადახდის პროცესს როგორც ველი. დაკონფიგურირებისას მომხმარებლები გადახდის დროს ირჩევენ სასურველ ქსელის შაბლონს, ხოლო არჩეული შაბლონი გამოიყენება მათი ახალი ქსელის მომზადებისას.

ამრჩევის გამოყენება გადახდის პროცესში:

1. მოამზადეთ ქსელები, რომელთა შეთავაზებაც გსურთ შაბლონებად.
2. ჩართეთ Multinetwork შაბლონის არჩევის ველი შესაბამის რეგისტრაციის ან გადახდის ფორმაში.
3. აირჩიეთ, რომელი შაბლონები უნდა იყოს ხელმისაწვდომი ამ გადახდის პროცესისთვის.
4. გამოაქვეყნეთ გადახდის პროცესი და დატესტეთ რეგისტრაცია, რათა დაადასტუროთ, რომ არჩეული შაბლონი გამოიყენება ახალ ქსელზე.

თუ გადახდის პროცესი მხოლოდ ერთ ქსელურ პროდუქტს სთავაზობს, ამრჩევი მაინც შეიძლება სასარგებლო იყოს როგორც მართვადი onboarding ნაბიჯი. მრავალგეგმარიანი funnel-ებისთვის გამოაჩინეთ მხოლოდ ის შაბლონები, რომლებიც თავსებადია არჩეულ პროდუქტთან ან გეგმასთან.

## ქსელებს შორის Magic Link SSO {#cross-network-magic-link-sso}

ქსელებს შორის magic link SSO მომხმარებლებს აძლევს საშუალებას გადაადგილდნენ თავიანთ მთავარ Account არესა და ქვე-ქსელებს შორის ხელახლა ხელით შესვლის გარეშე. როდესაც მომხმარებელი მიჰყვება ქსელებს შორის წვდომის ბმულს, Ultimate Multisite ქმნის დროით შეზღუდულ magic link-ს და მომხმარებელს ავთენტიფიკაციას უტარებს დანიშნულების ქსელში.

ეს აუმჯობესებს გამოცდილებას მომხმარებლებისთვის, რომლებიც ერთზე მეტ ქსელს მართავენ ან რომლებსაც შეძენისთანავე სჭირდებათ ახლად კლონირებულ შაბლონურ ქსელებზე წვდომა.

ადმინისტრატორის საკონტროლო სია:

* დაადასტურეთ, რომ Multinetwork დამატება აქტიურია მთელ WordPress multisite ინსტალაციაში.
* დარწმუნდით, რომ მომხმარებლებს აქვთ სწორი ქსელური წვდომა, სანამ მათ ქსელებს შორის ბმულებს შესთავაზებთ.
* სწორად დააკონფიგურირეთ ქსელის დომენები და მიბმული დომენები, რათა magic link-ები დანიშნულების სწორ ქსელზე გადავიდეს.
* დატესტეთ მომხმარებლის dashboard და ნებისმიერი გადახდის შემდგომი წვდომის ღილაკი ქსელის, დომენის ან გადახდის პარამეტრების შეცვლის შემდეგ.

Magic link-ები უნდა განიხილებოდეს როგორც შესვლის ბმულები. ისინი შექმნილია მომხმარებლის უწყვეტი წვდომისთვის და არ უნდა გადაიგზავნოს ან ხელახლა გამოიყენოს განკუთვნილი სესიის ფარგლებს გარეთ.

## ხშირად დასმული კითხვები {#frequently-asked-questions}

### რა არის multinetwork? {#what-is-multinetwork}

Multinetwork არის WordPress-ის შესაძლებლობა, რომელიც გაძლევთ საშუალებას ერთ WordPress ინსტალაციაზე გაუშვათ მრავალი დამოუკიდებელი multisite ქსელი.

### რით განსხვავდება ეს ჩვეულებრივი multisite-ისგან? {#how-is-this-different-from-regular-multisite}

ჩვეულებრივი multisite იძლევა რამდენიმე საიტის არსებობის საშუალებას ერთ ქსელში. Multinetwork იძლევა რამდენიმე ქსელის არსებობის საშუალებას, თითოეული საკუთარი საიტებით, ყველა ერთ WordPress ინსტალაციაში.

### სჭირდება ამას სპეციალური სერვერის კონფიგურაცია? {#does-this-require-special-server-configuration}

დიახ, multinetwork საჭიროებს სერვერის სპეციფიკურ მომზადებასა და კონფიგურაციას. დეტალური დაყენების ინსტრუქციებისთვის, გთხოვთ, გაეცნოთ დოკუმენტაციას.
