# Multi-Tenancy

ქვესაიტების განაწილება სერვერებსა და მონაცემთა ბაზებზე წარმადობისა და უსაფრთხოებისთვის

## ვერსია 1.2.0-ის გამოშვების მთავარი სიახლეები {#version-120-release-highlights}

Multi-Tenancy 1.2.0 ამატებს სუვერენული დამქირავებლის საფუძველს საწარმოო იზოლაციის სამუშაო პროცესებისთვის:

- უსტატუსო დამქირავებლის ავტომატური შესვლა მოკლევადიანი, განმეორებითი გამოყენებისგან დაცული SSO ვიზიტებით.
- თითოეული ქვესაიტისთვის მონაცემთა ბაზისა და ფაილური სისტემის იზოლაცია სუვერენული დამქირავებლებისთვის.
- Hostinger multi-tenancy შესაძლებლობა ჰოსტის გათვალისწინებით დამქირავებლის მოსამზადებლად.
- სუვერენული მიგრაციის ვერიფიკაციის კარიბჭეები ძველი მონაცემების შემოწმებისა და ასინქრონული გაგზავნის დამუშავებისთვის.
- კონფიგურირებადი იმავე მანქანის მონაცემთა ბაზის ჰოსტის მიბმა Bedrock, FrankenPHP და localhost ინსტალაციებისთვის.
- დამქირავებლის საწყისი ჩატვირთვის, მონაცემთა ბაზის ჩამწერის, სქემის საწყისი ჩატვირთვისა და მომხმარებლის მომზადების გაუმჯობესებები.
- უფრო უსაფრთხო სუვერენული დაშლა, რომელიც ასუფთავებს დამქირავებლის ავტორიზაციის მონაცემებს.

იხილეთ მომხმარებლის სახელმძღვანელოები [სუვერენული მიგრაციის სამუშაო პროცესისთვის](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy იზოლაციისთვის](/user-guide/configuration/multi-tenancy-isolation) და [სუვერენული დამქირავებლის ოპერაციებისთვის](/user-guide/administration/sovereign-tenant-operations).

## მოთხოვნები {#requirements}

- WordPress 5.3 ან უფრო ახალი
- PHP 7.4 ან უფრო ახალი
- Multisite Ultimate მოდული (აქტიური)

## ინსტალაცია {#installation}

1. ატვირთეთ დამატების ფაილები თქვენს `/wp-content/plugins/` დირექტორიაში
2. გაააქტიურეთ მოდული WordPress-ში 'Plugins' მენიუდან
3. დააკონფიგურირეთ დამატების პარამეტრები ქსელის ადმინისტრაციაში

## განვითარება {#development}

### დაყენება {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### ტესტირება {#testing}

დამატება მოიცავს სრულყოფილ ტესტირების ჩარჩოს:

- **ერთეულოვანი ტესტები**: PHPUnit-ზე დაფუძნებული ტესტები WordPress-ის ტესტების ნაკრებით
- **საბაზისო ტესტის კლასი**: უზრუნველყოფს დამხმარე მეთოდებს ხშირი ტესტირების სცენარებისთვის
- **ტესტების დაფარვა**: კონფიგურირებულია კოდის დაფარვის თვალყურის სადევნებლად
- **Multisite ტესტირება**: ტესტები ეშვება multisite გარემოში

### კოდის სტანდარტები {#code-standards}

- **WordPress-ის კოდირების სტანდარტები**: უზრუნველყოფილია PHPCS-ის მეშვეობით
- **PHP 7.4+ თავსებადობა**: მხარდაჭერილია თანამედროვე PHP შესაძლებლობები
- **სტატიკური ანალიზი**: PHPStan ინტეგრაცია ტიპების შესამოწმებლად
- **კოდის მოდერნიზაცია**: Rector წესები PHP განახლებებისთვის

### აწყობის სისტემა {#build-system}

- **რესურსების დამუშავება**: მინიფიკაცია CSS/JS ფაილებისთვის
- **თარგმანი**: POT ფაილის გენერაცია
- **შეფუთვა**: არქივის შექმნა გავრცელებისთვის
- **განვითარების რეჟიმი**: არამინიფიცირებული რესურსები გამართვისთვის

## კონფიგურაცია {#configuration}

დამატება მხარს უჭერს სხვადასხვა კონფიგურაციის ვარიანტს, რომლებიც ხელმისაწვდომია ქსელის ადმინისტრაციის ინტერფეისიდან.

### სუვერენული დამქირავებლის კონფიგურაცია {#sovereign-tenant-configuration}

სუვერენული დამქირავებლებისთვის, ტრაფიკის დამქირავებელზე მარშრუტიზაციამდე დააკონფიგურირეთ დამქირავებლის მონაცემთა ბაზა, ფაილური სისტემის ძირეული საქაღალდე, მონაცემთა ბაზის ჰოსტის მიბმა და ჰოსტის შესაძლებლობა. გაუშვით მიგრაციის ვერიფიკაცია ყოველი ცვლილების შემდეგ, რომელიც გავლენას ახდენს მონაცემთა ბაზის ავტორიზაციის მონაცემებზე, ჰოსტის მიბმაზე, მარშრუტიზაციაზე ან მომხმარებლის მომზადებაზე.

გამოიყენეთ `localhost` მხოლოდ მაშინ, როდესაც დამქირავებლის გაშვების გარემო და მონაცემთა ბაზის ნებართვები ელოდება ლოკალურ სოკეტს ან იმავე მანქანაზე წვდომას. გამოიყენეთ `127.0.0.1` ან სერვისის ჰოსტის სახელი, როდესაც ეს არის თქვენი პროვაიდერის მიერ კონფიგურირებული ნებართვის ჰოსტი.

### Hostinger შესაძლებლობა {#hostinger-capability}

Hostinger multi-tenancy შესაძლებლობა მუშაობს გაზიარებულ Hostinger ინტეგრაციის ტოკენთან. ვერიფიკაციამდე დაადასტურეთ, რომ დომენი არსებობს hPanel-ში, DNS მიუთითებს Hostinger Account-ზე, და დამქირავებლის მონაცემთა ბაზის პარამეტრები ემთხვევა hPanel-ს.

## ჰუკები და ფილტრები {#hooks-and-filters}

### ქმედებები {#actions}

- `ultimate-multisite-addon-template_init` - გაეშვება დამატების ინიციალიზაციის შემდეგ
- `ultimate-multisite-addon-template_loaded` - გაეშვება ყველა დამატების ფაილის ჩატვირთვის შემდეგ

### ფილტრები {#filters}

- `ultimate-multisite-addon-template_settings` - დამატების პარამეტრების შეცვლა
- `ultimate-multisite-addon-template_enabled` - დამატების ჩართვის/გამორთვის სტატუსის გადაფარვა
