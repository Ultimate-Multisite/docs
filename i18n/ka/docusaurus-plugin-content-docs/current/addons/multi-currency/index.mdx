# მრავალვალუტა

Ultimate Multisite: მრავალვალუტა თქვენს ქსელს საშუალებას აძლევს მიიღოს გადახდები მრავალ ვალუტაში. ის ავტომატურად გარდაქმნის ფასებს ცოცხალი ან ხელით მითითებული გაცვლითი კურსების გამოყენებით, გაძლევთ საშუალებას დააყენოთ ზუსტი თითოეული პროდუქტის ფასები ნებისმიერი ვალუტისთვის და გეოლოკაციით ამოიცნობს თითოეული ვიზიტორის სასურველ ვალუტას — ყველაფერი იმართება ერთი ადმინისტრატორის Dashboard-იდან.

## ძირითადი შესაძლებლობები

- **გაცვლითი კურსის ავტომატური კონვერტაცია** — ფასები გარდაიქმნება თქვენი საბაზისო ვალუტიდან თქვენი არჩეული მომწოდებლისგან მიღებული კურსების გამოყენებით
- **თითოეული პროდუქტის ფასების გადაფარვები** — დააყენეთ ზუსტი, ვალუტაზე მორგებული ფასები ნებისმიერ პროდუქტზე ავტომატური კონვერტაციის გვერდის ავლით
- **ვალუტის ამრჩევი რეგისტრაციის ველი** — მომხმარებლებს საშუალებას აძლევს გადახდის გაფორმების/რეგისტრაციის დროს აირჩიონ სასურველი ვალუტა
- **გაცვლითი კურსის მრავალი მომწოდებელი** — ECB, Frankfurter, Stripe ან სრულად ხელით მითითებული კურსები
- **გეოლოკაციაზე დაფუძნებული ვალუტის ამოცნობა** — ავტომატურად წინასწარ ირჩევს ვალუტას, რომელიც ვიზიტორის ქვეყანას შეესაბამება
- **ადმინისტრატორის Dashboard** — მართეთ აქტიური ვალუტები, გაცვლითი კურსები და მომწოდებლის პარამეტრები ერთი ადგილიდან

## მოთხოვნები

- Ultimate Multisite 2.0.0 ან უფრო ახალი
- WordPress Multisite ინსტალაცია
- PHP 7.4 ან უფრო ახალი

## ინსტალაცია

1. ატვირთეთ მოდულის ფაილები `/wp-content/plugins/ultimate-multisite-multi-currency`-ში
2. გაააქტიურეთ მოდული WordPress-ის **მოდულების** ეკრანიდან (ქსელში გააქტიურება)
3. გადადით **Ultimate Multisite → მრავალვალუტა**-ზე ადმინისტრატორის Dashboard-ის გასახსნელად
4. დაამატეთ ვალუტები, რომელთა მიღებაც გსურთ, და დააკონფიგურირეთ თქვენი გაცვლითი კურსის მომწოდებელი

## კონფიგურაცია

### ადმინისტრატორის Dashboard

მრავალვალუტის ადმინისტრატორის Dashboard (**Ultimate Multisite → მრავალვალუტა**) არის ცენტრალური ადგილი, სადაც შეგიძლიათ:

- დაამატოთ, ჩართოთ ან გამორთოთ ვალუტები
- აირჩიოთ გაცვლითი კურსის მომწოდებელი და მოთხოვნისამებრ განაახლოთ კურსები
- გადახედოთ მიმდინარე კურსებს ყველა აქტიური ვალუტისთვის
- დააყენოთ ხელით კურსები, როცა ავტომატური მომწოდებლები შესაფერისი არ არის

### გაცვლითი კურსის მომწოდებლები

| მომწოდებელი | აღწერა |
|---|---|
| **ECB** | ევროპის ცენტრალური ბანკი — უფასო დღიური კურსები EUR-ზე დაფუძნებული ქსელებისთვის |
| **Frankfurter** | ღია კოდის ECB სარკის API — უფასო, API გასაღები არ არის საჭირო |
| **Stripe** | იყენებს Stripe-ის ცოცხალ სავალუტო კურსებს — საჭიროა აქტიური Stripe ინტეგრაცია |
| **ხელით** | კურსებს პირდაპირ თქვენ შეყვანთ; გარე API გამოძახებები არ არის |

აირჩიეთ თქვენი მომწოდებელი **Ultimate Multisite → მრავალვალუტა → პარამეტრები → გაცვლითი კურსის მომწოდებელი**-ში. კურსების ხელით განახლება შესაძლებელია ნებისმიერ დროს ან ავტომატური განახლებისთვის დაგეგმვა.

### თითოეული პროდუქტის ფასების გადაფარვები

როცა ავტომატური კონვერტაცია საკმარისად ზუსტი არ არის, შეგიძლიათ ნებისმიერ პროდუქტზე კონკრეტული ვალუტისთვის ზუსტი ფასი დააყენოთ:

1. გახსენით პროდუქტის რედაქტორი (**Ultimate Multisite → პროდუქტები → რედაქტირება**)
2. ჩამოსქროლეთ **ვალუტის ფასდადების** სექციამდე
3. შეიყვანეთ ზუსტი ფასი თითოეული ვალუტისთვის, რომლის გადაფარვაც გსურთ
4. შეინახეთ პროდუქტი

პროდუქტები გადაფარვის გარეშე იყენებენ თქვენი საბაზისო ვალუტიდან კონვერტირებულ ფასს.

### ვალუტის ამრჩევი რეგისტრაციის ველი

ვალუტის ამრჩევი ველი შეიძლება დაემატოს თქვენს რეგისტრაციის/გადახდის გაფორმების ნაკადს, რათა მომხმარებლებმა რეგისტრაციისას აირჩიონ სასურველი ვალუტა:

1. გადადით **Ultimate Multisite → მრავალვალუტა → პარამეტრები**-ზე
2. ჩართეთ **ვალუტის ამრჩევის ჩვენება რეგისტრაციაზე**
3. სურვილისამებრ დააყენეთ ნაგულისხმევი ვალუტა ან მიეცით გეოლოკაციას წინასწარ არჩეული მნიშვნელობის განსაზღვრის საშუალება

არჩეული ვალუტა ინახება გამოწერასთან ერთად და გამოიყენება ყველა შემდგომი ბილინგისთვის.

### გეოლოკაციაზე დაფუძნებული ვალუტის ამოცნობა

როცა გეოლოკაციის ამოცნობა ჩართულია, დამატება ვიზიტორის ქვეყანას ვალუტას უკავშირებს და მას წინასწარ ირჩევს ვალუტის ამრჩევ ველში. ჩასართავად:

1. გადადით **Ultimate Multisite → მრავალვალუტა → პარამეტრები**-ზე
2. ჩართეთ **ვიზიტორის მდებარეობიდან ვალუტის ამოცნობა**
3. ქვეყნებს ვალუტებთან დააკავშირეთ **გეოლოკაციის შესაბამისობის** ცხრილში

თუ ვიზიტორის ქვეყნისთვის შესაბამისობა არ არსებობს, სათადარიგო მნიშვნელობად გამოიყენება ქსელის საბაზისო ვალუტა.

## ხშირად დასმული კითხვები

### რა მოხდება, თუ გაცვლითი კურსის მომწოდებელი მიუწვდომელია?

დამატება უბრუნდება ბოლოს წარმატებით მიღებულ კურსებს. თუ კურსები არასდროს ყოფილა მიღებული, ის უბრუნდება თქვენს მიერ დაკონფიგურირებულ ხელით კურსებს. რეკომენდებულია ხელით კურსების კონფიგურაცია უსაფრთხოების ბადის სახით.

### შემიძლია სხვადასხვა ვალუტისთვის სხვადასხვა მომწოდებელი გამოვიყენო?

არა — ყველა ვალუტისთვის ერთდროულად აქტიურია ერთი მომწოდებელი. გამოიყენეთ **ხელით**, თუ ინდივიდუალურ კურსებზე სრული კონტროლი გჭირდებათ.

### გეოლოკაციას ცალკე მოდული სჭირდება?

გარე გეოლოკაციის მოდული საჭირო არ არის. დამატება იყენებს WordPress-ის ჩაშენებულ IP-დან-ქვეყნის ამოცნობას. უფრო მაღალი სიზუსტისთვის, გამოყოფილი გეოლოკაციის სერვისის ინტეგრირება შესაძლებელია მოწოდებული ფილტრის ჰუკების მეშვეობით.

### თითოეული პროდუქტის გადაფარვები გადასახადამდე გამოიყენება თუ შემდეგ?

თითოეული პროდუქტის გადაფარვები სამიზნე ვალუტაში გადასახადამდე ფასს ადგენს. გადასახადი გამოითვლება გადაფარვის ფასზე ზემოდან, თქვენი არსებული საგადასახადო კონფიგურაციის გამოყენებით.

## ცვლილებების ჟურნალი

### ვერსია 1.0.0 — გამოშვებულია 2026-04-02

- ახალი: მრავალვალუტის მხარდაჭერა გაცვლითი კურსის ავტომატური კონვერტაციით
- ახალი: თითოეული პროდუქტის ფასების გადაფარვები ზუსტი, ვალუტაზე მორგებული ფასებისთვის
- ახალი: ვალუტის ამრჩევი რეგისტრაციის ველი გადახდის გაფორმების ნაკადისთვის
- ახალი: გაცვლითი კურსის მომწოდებლები: ECB, Frankfurter, Stripe და ხელით
- ახალი: გეოლოკაციაზე დაფუძნებული ვალუტის ამოცნობა
- ახალი: ადმინისტრატორის Dashboard ვალუტებისა და გაცვლითი კურსების სამართავად
- შესწორება: Dashboard ყოველთვის აჩვენებდა ფასებს საბაზისო ვალუტაში, არჩეული ვალუტის მიუხედავად
