# ცვლილებების ჟურნალი {#changelog}

## 1.9.0 — გამოშვებულია 2026-04-28 {#190--released-on-2026-04-28}

### ახალი {#new}

- **`create_contact_form` შესაძლებლობა** — ქმნის საკონტაქტო ფორმას აქტიური ფორმების plugin-ის გამოყენებით (Contact Form 7, WPForms, Fluent Forms ან Gravity Forms) და აბრუნებს shortcode-ს, რომელიც მზადაა ნებისმიერ ჩანაწერში ან გვერდზე ჩასასმელად.
- **`set_featured_image` შესაძლებლობა** — ანიჭებს გამორჩეულ სურათს ჩანაწერს ან გვერდს არსებული Media Library-ის დანართის ID-დან ან დისტანციური URL-დან; URL-ის მიწოდებისას სურათს ავტომატურად იმპორტირებს.
- **`batch_create_posts` შესაძლებლობა** — ქმნის რამდენიმე ჩანაწერს ერთი შესაძლებლობის გამოძახებით. მხარს უჭერს იგივე პარამეტრებს, რასაც `create_post`, თითოეული ჩანაწერისთვის აბრუნებს წარმატება/წარუმატებლობას და აქვს არასავალდებულო `stop_on_error` რეჟიმი.
- **`page_template` პარამეტრი** — დაემატა `create_post`-სა და `update_post`-ს. შექმნის ან განახლების დროს ანიჭებს PHP გვერდის შაბლონის ფაილს (მაგ. `page-full-width.php`). `update_post`-ში ცარიელი სტრიქონის გადაცემა დააბრუნებს theme-ის ნაგულისხმევ მნიშვნელობას.
- **კლიენტის მხარის ეკრანის ანაბეჭდების შესაძლებლობები** — `capture_screenshot`, `compare_screenshots` და `review_page_design`. სერვერის მხარის headless ბრაუზერის მეშვეობით იღებს ცოცხალი გვერდების სრულ ან viewport ეკრანის ანაბეჭდებს, ადარებს ორ ეკრანის ანაბეჭდს და იღებს AI-ით გენერირებულ დიზაინის მიმოხილვას, რომელიც მოიცავს განლაგებას, ტიპოგრაფიას, ფერსა და ხელმისაწვდომობას.
- **ხუთი ჩაშენებული აგენტი** — კონტენტის ავტორი, საიტის ამწყობი, დიზაინის სტუდია, plugin-ის მენეჯერი და მხარდაჭერის ასისტენტი. თითოეულ აგენტს აქვს ინსტრუმენტების გამოყოფილი ნაკრები, მორგებული სისტემური prompt და საწყისი შეთავაზებები. გადართვა შესაძლებელია ახალი **აგენტის ამრჩევის** მეშვეობით ჩატის სათაურში. იხილეთ [ჩაშენებული აგენტები](../../user-guide/configuration/built-in-agents).
- **ფუნქციების ალმები** — ახალი **პარამეტრები → ფუნქციების ალმები** ჩანართი წვდომის კონტროლის გადამრთველებით (შეზღუდვა ადმინისტრატორებზე, შეზღუდვა ქსელის ადმინისტრატორებზე, გამომწერის წვდომა, არაწევრებისთვის გამორთვა) და ბრენდინგის პარამეტრებით (ფუტერის ატრიბუციის დამალვა, აგენტის მორგებული სახელი, აგენტის ამრჩევის დამალვა, საიტის ხატულის გამოყენება ჩატის ავატარად). იხილეთ [Gratis AI Agent-ის პარამეტრები](../../user-guide/administration/gratis-ai-agent-settings).
- **ბოლო სესიის აღდგენა** — ჩატის პანელი ახლა გვერდის ჩატვირთვისას და ვიჯეტის გახსნისას ავტომატურად ხელახლა ტვირთავს უახლეს საუბარს, ამიტომ კონტექსტი გვერდებს შორის ნავიგაციისას აღარ იკარგება.
- **plugin-ის მოქმედებების ბმულები** — პარამეტრებსა და შესაძლებლობების რეესტრზე სწრაფი ბმულები ახლა ჩანს WordPress-ის **Plugins → Installed Plugins** ეკრანზე, plugin-ის აღწერის ქვემოთ.

### გაუმჯობესებული {#improved}

- **სურათის წყაროს ხელახალი ცდა** — აგენტი ახლა ჩამოტვირთვის წარუმატებლობისას AI-ით გენერირებულ სურათზე გადასვლამდე თავიდან ცდის ყველა კონფიგურირებულ უფასო სურათის წყაროს.
- **მოდელის ინფორმაციის პანელი** — ყოველთვის ჩანს ჩატის სათაურში; პირველი შეტყობინების შემდეგ აღარ იმალება.
- **ავტომატური გადახვევის ქცევა** — ავტომატური გადახვევა ჩერდება, როცა მომხმარებელი წასაკითხად ზემოთ ახვევს; ჩნდება მცურავი **ბოლოში გადასვლა** ღილაკი და ავტომატურად ქრება, როცა მომხმარებელი უახლეს შეტყობინებას მიაღწევს.
- **აგენტის ამრჩევის ინტერფეისი** — ხელახლა შეიქმნა როგორც form-table გადაფარვა თითოეული აგენტის ხატულებით, რაც აგენტების ერთი შეხედვით ამოცნობასა და გადართვას ამარტივებს.
- **ზარმაცად ჩატვირთული JS ნაწილები** — ჩატის ვიჯეტის საწყისი JavaScript პაკეტი ახლა დაყოფილია ზარმაცად ჩატვირთულ ნაწილებად, რაც საწყისი პაკეტის ზომებს 75–90%-ით ამცირებს.
- **ჩატის ვიჯეტის ხელახალი დიზაინი** — ერთიანი AI ხატულა ცვლის წინა მორგებულ ავატარს; შეესაბამება ჩაშენებული აგენტების სისტემას.
- **URL-ების ბმულად ქცევა** — სისტემურ შეტყობინებებსა და შეცდომის შეტყობინების ბუშტებში გამოჩენილი URL-ები ახლა კლიკებად ბმულებად გამოისახება.

### გამოსწორებული {#fixed}

- **შესაძლებლობების აღმოჩენადობა** — გასწორდა აღწერები, სისტემური prompt-ის მითითებები და namespace-ის შესაბამისობა, რათა ყველა შესაძლებლობა საიმედოდ გამოჩნდეს აგენტის ინსტრუმენტების სიაში.
- **პროვაიდერების ქეში** — პროვაიდერები ახლა საიტის მასშტაბით ქეშირდება ვერსიის მრიცხველის მეშვეობით, რაც multisite ქსელებში მოძველებული პროვაიდერების პრობლემებს აფერხებს.
- **`ability_invalid_output`** — მოგვარდა 12 შესაძლებლობის დამმუშავებელში; ყველა აბრუნებს სწორად სტრუქტურირებულ JSON პასუხებს.
- **`pending_client_tool_calls` pipeline** — სრულად შეერთდა ისე, რომ კლიენტის მხარის ინსტრუმენტების გამოძახებები სწორად სრულდება და შედეგებს მოდელს უბრუნებს.
- **ისტორიის უჯრა** — არდაბრუნებადი ცვლილებები გამორიცხულია დაბრუნების სიიდან; **სრული ისტორიის ნახვა** ბმული ახლა სწორად მუშაობს.
- **ცვლილებების/დაბრუნების სისტემა** — გასწორდა ხუთი ცალკეული შეცდომა და გაერთიანდა ახალი ადმინისტრატორის ინტერფეისის ქვეშ.
- **პარამეტრების შენახვის toast** — snackbar შეტყობინება ახლა **პარამეტრების შენახვა**-ზე დაწკაპუნების შემდეგ საიმედოდ ჩნდება.
- **ნაგვის კონტექსტური მენიუ** — დაემატა **სამუდამოდ წაშლა** ვარიანტი, რათა ელემენტები ნაგვის ხედის დატოვების გარეშე საბოლოოდ წაიშალოს.
- **რედაქტირება და ხელახლა გაგზავნა** — რედაქტირების ხატულაზე დაწკაპუნება ახლა რედაქტირების რეჟიმში გადაჰყავს მხოლოდ დაწკაპუნებულ შეტყობინებას და არა მთელ თემაში არსებულ ყველა შეტყობინებას.
- **ჩატის განლაგების სიმაღლე** — ჩატის პანელი თავის სიმაღლეს ადაპტირებს, როცა plugin-ის მიერ ჩასმული კონტენტი (ადმინისტრატორის შეტყობინებები, ბანერები) გვერდის ზემოთ ჩნდება, რაც შეყვანის არეს ეკრანს გარეთ გაწევას აფერხებს.

## 1.4.0 — გამოშვებულია 2026-04-09 {#140--released-on-2026-04-09}

### ახალი {#new-1}

- **WP-CLI benchmark command** (`wp gratis-ai-agent benchmark`) — გაუშვებს Agent Capabilities v1 benchmark suite-ს ბრძანების ხაზიდან CI pipeline-ებისა და მოდელის შეფასების სამუშაო პროცესებისთვის. მხარს უჭერს თითოეული შეკითხვის გაშვებებს, provider/model გადაფარვებს და JSON/CSV გამოტანას.
- **Agent Capabilities v1 benchmark suite** — რთული, მრავალსაფეხურიანი prompt-ების სტრუქტურირებული ნაკრები, რომელიც ამოწმებს შესაძლებლობების სრულ ზედაპირს, შეფასებითა და token/ხანგრძლივობის ანგარიშგებით.
- **Custom Post Type შესაძლებლობები** — `register_post_type`, `list_post_types`, `delete_post_type`. რეგისტრაციები გადატვირთვებს შორის ნარჩუნდება options ცხრილის მეშვეობით.
- **Custom Taxonomy შესაძლებლობები** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. მხარს უჭერს იერარქიულ და ბრტყელ ტაქსონომიებს არასავალდებულო rewrite slug-ებით.
- **Design System შესაძლებლობები** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. მოიცავს ხუთ შერჩეულ preset-ს: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **Global Styles შესაძლებლობები** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. კითხულობს და წერს theme.json მნიშვნელობებს WordPress Global Styles API-ის მეშვეობით.
- **Navigation Menu მართვის შესაძლებლობა** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. მხარს უჭერს ჩადგმულ მენიუებს და theme location-ის მინიჭებას.
- **Options Management შესაძლებლობა** — `get_option`, `set_option`, `delete_option`, `list_options`. მოიცავს ჩაშენებულ უსაფრთხოების blocklist-ს, რომელიც იცავს კრიტიკულ WordPress options-ს შეცვლისგან.
- **Installable Abilities Registry** — `list_available_abilities`, `install_ability`, `recommend_plugin`. აღმოაჩინეთ და გაააქტიურეთ WordPress plugin-ებად გავრცელებული შესაძლებლობების პაკეტები.
- **Site Builder Orchestration v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. მრავალფაზიანი საიტის აგების გეგმები plugin-ის აღმოჩენით, ნაბიჯის გამოტანის მითითებებით, პროგრესის თვალყურის დევნებით და შეცდომების ავტონომიური აღდგენით.
- **Restaurant website benchmark question** (`q-restaurant-website`) — end-to-end ტესტი, რომელიც მოიცავს CPT რეგისტრაციას, design system-ს, ნავიგაციას და plugin-ის აღმოჩენას.
- **AI provider connector plugins** დაემატა WordPress Playground blueprint-ებს უფრო სწრაფი ადგილობრივი განვითარების დაყენებისთვის.

### გაუმჯობესდა {#improved-1}

- README განახლდა AI provider connector-ის დოკუმენტაციითა და დაყენების ინსტრუქციებით.

### გასწორდა {#fixed-1}

- `main` განშტოებაზე 25 PHPUnit ტესტის ხარვეზი მოგვარდა.
- GitHub releases URL-ის ფორმატი `blueprint.json`-ში გასწორდა.
- ამოცანების ID-ების ხელახლა დანომრვა ძველ ID-ებთან კონფლიქტების თავიდან ასაცილებლად.
