# GoCardless {#gocardless}

Multisite Ultimate: GoCardless გადახდა

მიეცით თქვენს მომხმარებლებს საშუალება, GoCardless-ის გამოყენებით პირდაპირი დებეტით გადაიხადონ Multisite Ultimate წევრობებისა და პროდუქტებისთვის!

## ინსტალაცია {#installation}

1. ატვირთეთ 'ultimate-multisite-gocardless' '/wp-content/plugins/' დირექტორიაში
2. გაააქტიურეთ პლაგინი WordPress-ში 'პლაგინები' მენიუდან
3. მიჰყევით ეტაპობრივ Wizard-ს პლაგინის დასაყენებლად

## გამოყენების გზამკვლევი {#usage-guide}

_GoCardless_ არის გადახდის კარიბჭე, რომელიც ხელმისაწვდომია **ევროპასა და გაერთიანებულ სამეფოში** და იძლევა **პირდაპირი დებეტით** გადახდების საშუალებას.

**Ultimate Multisite: GoCardless Gateway** ამატებს _GoCardless_-ის მხარდაჭერას Ultimate Multisite-ში, რაც საშუალებას გაძლევთ, ამ რეგიონში თქვენს მომხმარებლებს თანხა პირდაპირი დებეტით დააკისროთ.

ეს სახელმძღვანელო აგიხსნით, როგორ დააყენოთ _GoCardless_, რათა დაიწყოთ მისი მეშვეობით გადახდების მიღება თქვენს Ultimate Multisite ქსელში.

_**შენიშვნა: GoCardless მუშაობს მხოლოდ შემდეგ ვალუტებთან:**_ **EUR, GBP, AUD, SEK და DKK.**

## დამატების ინსტალაცია {#installing-the-add-on}

პირველ რიგში, Ultimate Multisite-ისთვის GoCardless დამატება უნდა დააინსტალიროთ. ამის გაკეთება შეგიძლიათ **Ultimate Multisite > Settings > დამატებები (გვერდით ზოლში)** გადასვლით და ამ სიიდან GoCardless დამატების არჩევით.

დამატების ინსტალაციისა და გააქტიურების შემდეგ გადადით **Ultimate Multisite > Settings > Payments**, ჩართეთ GoCardless ვარიანტი და შეინახეთ.

![გადახდის კარიბჭის პარამეტრები](/img/config/settings-payment-gateways.png)

## Access Token-ის მიღება {#getting-an-access-token}

შექმენით ანგარიში _GoCardless_-ზე (ან _GoCardless_ sandbox-ზე, თუ გსურთ **ჯერ მისი ტესტირება** , რაც რეკომენდებულია). შემდეგ, შესვლის შემდეგ, გადადით **დეველოპერები** მენიუს პუნქტზე.

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless Dashboard, სადაც ნაჩვენებია დეველოპერების მენიუს მდებარეობა -->

**დეველოპერები** გვერდზე დააწკაპუნეთ ეკრანის ზედა მარჯვენა კუთხეში მდებარე **შექმნა** ღილაკზე, შემდეგ კი დააწკაპუნეთ **Access Token** ვარიანტზე.

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless-ის დეველოპერების გვერდი Create ღილაკითა და Access Token ვარიანტით -->

ეს გახსნის ახალ მოდალურ ფანჯარას, რომელიც საშუალებას მოგცემთ, შექმნათ ახალი ტოკენი. მიანიჭეთ მას ადვილად ამოსაცნობი სახელი და დარწმუნდით, რომ **Scope** მენიუში არჩეული გაქვთ **Read-write access** ვარიანტი.

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless-ის მოდალური ფანჯარა ახალი access token-ის შესაქმნელად Read-write scope-ით -->

Access Token ეკრანზე გამოჩნდება. დააკოპირეთ ის და ჩასვით თქვენს **Ultimate Multisite > Settings > Payments > GoCardless** მენიუში.

დარწმუნდით, რომ ის სწორ ველში მოათავსეთ. თუ ეს **live toke** **n** არის, ტოკენის ჩასმამდე **sandbox რეჟიმი გამორთეთ**.

![გადახდის კარიბჭის პარამეტრები GoCardless-ით](/img/config/settings-payment-gateways.png)

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless ტოკენის ველები Ultimate Multisite-ის გადახდის პარამეტრებში -->

პირველი ნაბიჯისთვის სულ ეს არის. შემდეგ, webhook listener უნდა დააყენოთ.

## Webhook Listener-ის დაყენება {#setting-up-the-webhook-listener}

გამოწერებთან მუშაობისას Ultimate Multisite-ს უნდა ეცნობოს გამოწერის სტატუსის ცვლილებების, ახალი გადახდების შესრულებისა და სხვა მსგავსი საკითხების შესახებ.

გადახდის კარიბჭეები Ultimate Multisite-ს ამ ცვლილებების შესახებ **webhook გამოძახებების** გამოყენებით ატყობინებენ. ამ მიზეზით, იმისათვის, რომ ინტეგრაციამ 100%-ით იმუშაოს, _GoCardless_-ს უნდა ვუთხრათ, რომელ URL-ს დაურეკოს მთელი ამ ინფორმაციით.

დაბრუნდით თქვენს **GoCardless > დეველოპერები** პანელში და კვლავ დააწკაპუნეთ Create ღილაკზე, მაგრამ ამჯერად აირჩიეთ **Webhook endpoint** ვარიანტი.

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless-ის დეველოპერების გვერდი Webhook endpoint ვარიანტით -->გაიხსნება ახალი ფანჯარა webhook endpoint-ის ვარიანტებით. ის webhook URL-ს ითხოვს, ამიტომ დაბრუნდით თქვენს **Ultimate Multisite პარამეტრები > Payments > GoCardless** გვერდზე და დააკოპირეთ იქ მითითებული webhook URL:

<!-- ეკრანის სურათი მიუწვდომელია: Ultimate Multisite GoCardless პარამეტრები, სადაც ნაჩვენებია webhook URL -->

დამატებით, შეგიძლიათ შეიყვანოთ **webhook secret**, რომელიც თქვენი webhook გამოძახებების ხელმოსაწერად იქნება გამოყენებული. რასაც ამ ველში შეიყვანთ, ის ასევე უნდა შეიყვანოთ **Ultimate Multisite Webhook Secret ველში**.

<!-- ეკრანის სურათი მიუწვდომელია: GoCardless webhook endpoint-ის კონფიგურაციის ფორმა URL-ისა და secret ველებით -->

<!-- ეკრანის სურათი მიუწვდომელია: Ultimate Multisite GoCardless webhook secret ველი -->

დაასრულეთ endpoint-ის შენახვით, რისთვისაც დააწკაპუნეთ **Create webhook endpoint** ღილაკზე და შემდეგ **თქვენი Ultimate Multisite პარამეტრებიც შეინახეთ**.

ახლა მზად უნდა იყოთ _GoCardless_-ის მისაღებად!
