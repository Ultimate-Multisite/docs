# WooCommerce ინტეგრაცია {#woocommerce-integration}

WooCommerce ინტეგრაცია მუშაობს ძალიან მსგავსად იმისა, როგორც ჩვენი Manual Gateway მუშაობს. ის თქვენს მომხმარებლებს დიდ მოქნილობას სთავაზობს, აძლევს მათ შესაძლებლობას აირჩიონ გადახდის ნებისმიერი ფორმა, რომელიც ხელმისაწვდომია თქვენს WooCommerce ინსტალაციაზე. და ეს მარტივია: როგორც კი მომხმარებლის ბილინგის პერიოდი დასრულდება, მომხმარებლის გამოწერა შეჩერდება. ამავდროულად, ქსელის მთავარი საიტის WooCommerce ინსტალაციაზე შეიქმნება შეკვეთა. შემდეგ მომხმარებელი ელფოსტაზე მიიღებს ინვოისს, გადახდის შესასრულებელი ბმულით. როგორც კი გადახდა მიღებული იქნება, გამოწერა განახლდება.

## ფუნქციები {#features}

- მარტივი ინტეგრაცია Ultimate Multisite-სა და WooCommerce-ს შორის
- ხელმისაწვდომია რამდენიმე გადახდის მეთოდი
- ავტომატიზებული განმეორებადი ბილინგი ჩაშენებული გამოწერების ბირთვის გამოყენებით, სურვილისამებრ WooCommerce Subscriptions plugin-ის გაუმჯობესებებით
- გადახდის ჯამებში Ultimate Multisite-ის გადასახადები შედის ცალკე, დაუბეგრავ WooCommerce საკომისიო ხაზებად, რათა მომხმარებლებმა გადახდამდე შეძლონ გადასახადის თანხების გადახედვა

## დოკუმენტაცია {#documentation}

## ინსტალაცია {#installation}

1. ატვირთეთ 'ultimate-multisite-woocommerce' '/wp-content/plugins/' დირექტორიაში
2. გაააქტიურეთ plugin WordPress-ის 'Plugins' მენიუდან
3. მიჰყევით ეტაპობრივ Wizard-ს plugin-ის დასაყენებლად

## გამოყენების გზამკვლევი {#usage-guide}

_**მნიშვნელოვანი შენიშვნა: ეს სტატია ეხება Ultimate Multisite-ის 2.x ვერსიას.**_

_**ყურადღება:** Ultimate Multisite: WooCommerce Integration მოითხოვს, რომ WooCommerce გააქტიურებული იყოს მინიმუმ თქვენს მთავარ საიტზე._

გვესმის, რომ _Stripe_ და _PayPal_ ზოგიერთ ქვეყანაში ხელმისაწვდომი არ არის, რაც Ultimate Multisite-ის მომხმარებლებს უზღუდავს ან უშლის ხელს ჩვენი plugin-ის ეფექტურად გამოყენებაში. ამიტომ შევქმენით **add-on _WooCommerce_-ის ინტეგრაციისთვის**.

_WooCommerce_ ძალიან პოპულარული eCommerce plugin-ია. დეველოპერებმა მთელ მსოფლიოში შექმნეს add-on-ები სხვადასხვა გადახდის gateway-ების მასში ინტეგრირებისთვის. ჩვენ ეს გამოვიყენეთ, რათა გაგვეფართოებინა გადახდის gateway-ები, რომლებიც შეგიძლიათ გამოიყენოთ Ultimate Multisite-ის ბილინგის სისტემაში.

## მისი დაყენება {#setting-it-up}

პირველ რიგში, უნდა დააყენოთ WooCommerce Integration add-on, რომელიც **უფასოა ლიცენზირებული მომხმარებლებისთვის**. მის დასაყენებლად, დააწკაპუნეთ Ultimate Multisite-ზე თქვენს super admin dashboard-ში და აირჩიეთ **Add-ons**.

![Add-ons გვერდი](/img/admin/addons-page.png)

აქ შეგიძლიათ იპოვოთ Ultimate Multisite-ის ყველა add-on. დააწკაპუნეთ **Ultimate Multisite: WooCommerce Integration** add-on-ზე.

![Add-ons გვერდი, სადაც ნაჩვენებია WooCommerce Integration](/img/admin/addons-page.png)

გაიხსნება ფანჯარა add-on-ის დეტალებით. უბრალოდ დააწკაპუნეთ **Install Now** *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

ინსტალაციის დასრულების შემდეგ, გადამისამართდებით plugins გვერდზე. აქ უბრალოდ დააწკაპუნეთ **Network Activate**-ზე და _WooCommerce_ add-on გააქტიურდება თქვენს ქსელში.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

მისი გააქტიურების შემდეგ, თუ თქვენს ვებსაიტზე ჯერ კიდევ არ გაქვთ WooCommerce plugin დაყენებული და გააქტიურებული, **მიიღებთ შეხსენებას**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

დაყენების შემდეგ, Ultimate Multisite-ის გადახდების პარამეტრებში დამატებით ვარიანტს ნახავთ. შემდეგ შეგიძლიათ იქიდან **ჩართოთ** _WooCommerce_ ვარიანტი.

![Ultimate Multisite-ის გადახდის gateway პარამეტრები ჩართული WooCommerce-ით](/img/addons/woocommerce-payment-gateway.png)

შეგიძლიათ დააკონფიგურიროთ ზოგიერთი ძირითადი პარამეტრი, როგორიცაა გამოსაჩენი სახელი, რომელიც გამოჩნდება front end-ში, და ინსტრუქციები, რომლებიც თქვენს მომხმარებლებს checkout პროცესში გაუძღვება. შემდეგ შეგიძლიათ გადახვიდეთ თქვენი **მთავარი საიტის _WooCommerce_ პარამეტრებში**, რათა ჩართოთ და დააკონფიგურიროთ გადახდის gateway, რომლის გამოყენებაც გსურთ.

### Checkout-ის გადასახადის საკომისიო ხაზები {#checkout-tax-fee-lines}

როდესაც Ultimate Multisite checkout შეიცავს გადასახადებს, ინტეგრაცია WooCommerce checkout-ში თითოეულ Ultimate Multisite გადასახადის თანხას ამატებს ცალკე, დაუბეგრავ საკომისიო ხაზად. ეს გადასახადის ჯამს გადახდამდე ხილულს ტოვებს და ამავდროულად ხელს უშლის WooCommerce-ს, გადასახადის საკომისიო ხელახლა დაბეგროს.

## როგორ მართოთ გადახდა {#how-to-manage-the-payment}

WooCommerce ინტეგრაცია მუშაობს ძალიან მსგავსად იმისა, როგორც ჩვენი **Manual Gateway** მუშაობს. ნაკლი ის არის, რომ ბილინგის პერიოდის ბოლოს თითოეული გადახდა ხელით უნდა გადაიხადოს. აი, როგორ მუშაობს პროცესი:

  * როგორც კი მომხმარებლის ბილინგის ციკლი დასრულდება, წევრობა **შეჩერდება**. ამავდროულად, თქვენს ქსელის მთავარ საიტზე _WooCommerce_ ინსტალაციაზე შეიქმნება შეკვეთა.

  * შემდეგ მომხმარებლები ელფოსტაზე მიიღებენ ინვოისს, გადახდის შესასრულებელი ბმულით (ბმული ასევე გამოჩნდება მათ ბილინგის ისტორიაში მათი subsite dashboard-იდან). ეს ბმული მომხმარებელს გადაამისამართებს _**WooCommerce**_ **checkout**-ზე, სადაც მომხმარებლებს შეეძლებათ აირჩიონ ერთ-ერთი **ხელმისაწვდომი _WooCommerce_ gateway** თავიანთი გადახდის დასამუშავებლად.

  * როგორც კი გადახდა მიღებული იქნება, წევრობა **განახლდება**, გადახდა ჩაიწერება Ultimate Multisite-ში და შეკვეთა დასრულებულად მოინიშნება.

### რამდენიმე მნიშვნელოვანი შენიშვნა: {#some-important-notes}

  * Ultimate Multisite-ის ნაგულისხმევი ფუნქციონალი ამ ინტეგრაციის ფორმასთან კვლავ მუშაობს. მაგალითად, **თანხის დაბრუნების გაცემა** **Subscription Management** ეკრანიდან დაუკავშირდება WooCommerce-ს, რათა თანხის დაბრუნება არჩეული გადახდის ფორმით გასცეს.

  * Ultimate Multisite: WooCommerce Integration მუშაობისთვის WP-Cron-ს ეყრდნობა. ეს უმეტეს შემთხვევაში ძალიან კარგად მუშაობს, მაგრამ თუ საიტს დაბალი ტრაფიკი აქვს, cron არ იძახება დაგეგმილი მოვლენების შესამოწმებლად და მოვლენა გამოტოვებულია, რამაც შეიძლება შესრულების დაყოვნებები გამოიწვიოს.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

ყოველთვიურად გადახდის ხელით დაწყება საბოლოო მომხმარებლებისთვის იდეალური არ არის, რადგან მოსალოდნელია, რომ გამოწერები ყოველ ბილინგის ციკლში ავტომატურად განმეორდეს. WooCommerce Integration add-on ახლა მოიცავს ძირითად გამოწერების კოდს, ამიტომ განმეორებადი პროდუქტები განმეორებადი ბილინგისთვის ყოველთვის იყენებს გამოწერებს.

თქვენ **არ** გჭირდებათ ცალკე [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) შეძენა ან დაყენება, რათა ამ ინტეგრაციის საშუალებით გაყიდოთ განმეორებადი პროდუქტები. თუ ეს plugin შეძენილი, დაყენებული და ხელმისაწვდომია თქვენს მთავარ საიტზე, Ultimate Multisite მას გამოიყენებს და WooCommerce დამატებით გამოწერის მართვის ფუნქციებს გამოაჩენს. როდესაც ის ხელმისაწვდომი არ არის, ჩაშენებული გამოწერების ბირთვი მაინც ამუშავებს განმეორებად პროდუქტებს.

![WooCommerce გამოწერის ინტეგრაციის პარამეტრები](/img/addons/woocommerce-subscription-integration.png)

სულ ეს არის. თქვენს საბოლოო მომხმარებლებს აღარ დასჭირდებათ ბილინგის ციკლის ბოლოს განმეორებადი პროდუქტებისთვის გადახდის ხელით განხორციელება.

გარდა იმისა, რომ შეკვეთა თქვენს მთავარ საიტზე _WooCommerce_-ის ქვეშ შეიქმნება, თითოეული Account-ისთვის ასევე შეიქმნება გამოწერა.

### წარუმატებელი განახლების შეჩერების ქცევა {#failed-renewal-suspension-behavior}

ნაგულისხმევად, წევრობები აქტიური რჩება, სანამ WooCommerce Subscriptions წარუმატებელი გადახდის ხელახლა ცდის განრიგს ამუშავებს. ეს ინარჩუნებს ჩვეულებრივ WooCommerce Subscriptions საშეღავათო ფანჯარას, რათა აღდგენილმა განახლებამ შეძლოს გამოწერისა და წევრობის ხელახლა სინქრონიზაცია წვდომის ნაადრევად შეწყვეტის გარეშე.

ადმინისტრატორებს, რომლებსაც უფრო მკაცრი წვდომის კონტროლი სურთ, შეუძლიათ WooCommerce Integration-ის პარამეტრებში ჩართონ **წევრობების დაუყოვნებლივ შეჩერება წარუმატებელი განახლებებისას**. როდესაც ეს opt-in პარამეტრი ჩართულია, Ultimate Multisite დაკავშირებულ წევრობას აჩერებს მაშინვე, როგორც კი განახლების გადახდა ვერ შესრულდება, WooCommerce Subscriptions-ის ხელახლა ცდის დამუშავების დასრულების ლოდინის ნაცვლად.

დეველოპერებს შეუძლიათ იგივე გადაწყვეტილების გადაფარვა `wu_woo_suspend_on_payment_failure` ფილტრით. Hook-ის საცნობარო გვერდები add-on-ის წყაროდან გენერირდება, ამიტომ ეს მიმოხილვა მხოლოდ ფილტრის სახელს გამოყოფს მორგებული შეჩერების პოლიტიკებისთვის.
