# Ultimate AI Connector თავსებადი endpoint-ებისთვის {#ultimate-ai-connector-for-compatible-endpoints}

Ultimate AI Connector თავსებადი endpoint-ებისთვის OpenAI-თან თავსებადი AI პროვაიდერების API-ებს აერთიანებს თქვენს WordPress multisite ქსელთან. დააკონფიგურირეთ რამდენიმე AI პროვაიდერი — თითოეული საკუთარი endpoint URL-ით, API გასაღებით და ნაგულისხმევი მოდელით — და გადაამისამართეთ მოთხოვნები ავტომატური სარეზერვო გადართვით, როდესაც პროვაიდერი მიუწვდომელია.

## ძირითადი შესაძლებლობები {#key-features}

- **რამდენიმე პროვაიდერის მხარდაჭერა**: დაამატეთ რამდენიმე AI პროვაიდერის ბარათი, თითოეული საკუთარი endpoint URL-ით, API გასაღებით და ნაგულისხმევი მოდელით
- **გადათრევით გადალაგება**: დაალაგეთ პროვაიდერები პრიორიტეტის მიხედვით თითოეულ ბარათზე არსებული ჩასაჭიდი სახელურის გამოყენებით; მოთხოვნები პროვაიდერებს რიგის მიხედვით იყენებს
- **ავტომატური სარეზერვო გადამისამართება**: თუ ძირითადი პროვაიდერი მიუწვდომელია, მოთხოვნები ავტომატურად გადადის შემდეგ დაკონფიგურირებულ პროვაიდერზე
- **ინდივიდუალური კონფიგურაცია თითოეული პროვაიდერისთვის**: თითოეული პროვაიდერი დამოუკიდებლად კონფიგურდება საკუთარი მონაცემებით და მოდელის არჩევანით
- **ბარათებზე დაფუძნებული პარამეტრების UI**: აგებულია სტაბილურ WordPress 6.9+ Card კომპონენტებზე (CardBody, CardHeader, CardDivider)
- **დამატებისას ავტომატური გაშლა**: ახალი პროვაიდერის ბარათები ავტომატურად იშლება, რათა დეტალები დაუყოვნებლივ შეავსოთ
- **სკრიპტის ქეშის განახლება**: პროვაიდერის სკრიპტებზე plugin-ის განახლებისას ქეში უქმდება, რათა უახლესი ვერსიის ჩატვირთვა იყოს უზრუნველყოფილი

## მოთხოვნები {#requirements}

- WordPress 5.3 ან უფრო ახალი
- PHP 7.4 ან უფრო ახალი
- Ultimate Multisite plugin (აქტიური)
- OpenAI-თან თავსებადი API endpoint (თვითჰოსტინგზე ან მესამე მხარის)

## ინსტალაცია {#installation}

1. ატვირთეთ addon-ის ფაილები თქვენს `/wp-content/plugins/` დირექტორიაში
2. გაააქტიურეთ plugin WordPress-ის „Plugins“ მენიუდან
3. გადადით **Ultimate Multisite → AI Connector** გვერდზე addon-ის დასაკონფიგურირებლად

## კონფიგურაცია {#configuration}

### რამდენიმე პროვაიდერის დაყენება {#multi-provider-setup}

კონფიგურაციის ეკრანი იყენებს ბარათებზე დაფუძნებულ განლაგებას, სადაც თითოეული ბარათი ერთ AI პროვაიდერს წარმოადგენს. შეგიძლიათ დაამატოთ იმდენი პროვაიდერი, რამდენიც გჭირდებათ.

#### პროვაიდერის დამატება {#adding-a-provider}

1. ქსელის ადმინისტრირებაში გადადით **Ultimate Multisite → AI Connector** გვერდზე
2. დააწკაპუნეთ **პროვაიდერის დამატება**
3. გამოჩნდება ახალი პროვაიდერის ბარათი და ავტომატურად გაიშლება
4. შეავსეთ პროვაიდერის დეტალები:
   - **პროვაიდერის სახელი** — იარლიყი ამ პროვაიდერის ამოსაცნობად (მაგ. "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL** — OpenAI-თან თავსებადი API-ის საბაზისო URL (მაგ. `https://api.openai.com/v1`)
   - **API გასაღები** — თქვენი API გასაღები ამ პროვაიდერისთვის
   - **ნაგულისხმევი მოდელი** — მოდელის იდენტიფიკატორი, რომელიც ნაგულისხმევად გამოიყენება ამ პროვაიდერისთვის (მაგ. `gpt-4o`, `mistral-small`)
5. დააწკაპუნეთ **პარამეტრების შენახვა**

#### პროვაიდერის წაშლა {#removing-a-provider}

დააწკაპუნეთ **წაშლა** ღილაკზე ნებისმიერი პროვაიდერის ბარათზე მის წასაშლელად. ცვლილების გამოსაყენებლად შეინახეთ პარამეტრები.

#### პროვაიდერების გადალაგება {#reordering-providers}

გადააადგილეთ ჩასაჭიდი ხატულა (⣿) ნებისმიერი პროვაიდერის ბარათის მარცხენა მხარეს მის გადასალაგებლად. რიგი განსაზღვრავს სარეზერვო გადართვის პრიორიტეტს: მოთხოვნები იგზავნება პირველ პროვაიდერთან და მოთხოვნის წარუმატებლობის შემთხვევაში გადადის შემდეგ პროვაიდერებზე.

### სარეზერვო გადამისამართება {#fallback-routing}

როდესაც ძირითად პროვაიდერთან მოთხოვნა ვერ სრულდება — ვადის ამოწურვის, სიხშირის ლიმიტის ან სერვისის გათიშვის გამო — connector ავტომატურად ხელახლა ცდის სიაში შემდეგ პროვაიდერთან. ეს გრძელდება სიის ბოლომდე, სანამ რომელიმე პროვაიდერი წარმატებით უპასუხებს ან ყველა პროვაიდერი გამოიცდება.

| სცენარი | ქცევა |
|----------|-----------|
| ძირითადი პროვაიდერი პასუხობს | მოთხოვნა ჩვეულებრივ მუშავდება |
| ძირითადი პროვაიდერი ვერ სრულდება | მოთხოვნა ხელახლა ცდის შემდეგ პროვაიდერთან |
| ყველა პროვაიდერი ვერ სრულდება | შეცდომა უბრუნდება გამომძახებელს |

სარეზერვო გადართვა გამჭვირვალეა გამომძახებელი plugin-ის ან theme-ისთვის — ისინი იღებენ ან წარმატებულ პასუხს, ან საბოლოო შეცდომას, იმის მიუხედავად, რამდენი პროვაიდერი გამოიცადა.

### პროვაიდერის კონფიგურაციის ცნობარი {#provider-configuration-reference}

| ველი | სავალდებულო | აღწერა |
|-------|----------|-------------|
| პროვაიდერის სახელი | დიახ | საჩვენებელი იარლიყი ამ პროვაიდერისთვის |
| Endpoint URL | დიახ | OpenAI-თან თავსებადი API-ის საბაზისო URL |
| API გასაღები | დიახ | ავთენტიფიკაციის გასაღები ამ პროვაიდერისთვის |
| ნაგულისხმევი მოდელი | დიახ | მოდელის იდენტიფიკატორი, რომელიც გამოიყენება, როდესაც მოთხოვნაში მოდელი მითითებული არ არის |

## პრობლემების მოგვარება {#troubleshooting}

### პროვაიდერები არ არის ჩამოთვლილი {#no-providers-are-listed}

თუ შენახვის შემდეგ პროვაიდერების სია ცარიელია, დაადასტურეთ, რომ:

- პროვაიდერების დამატების შემდეგ დააწკაპუნეთ **პარამეტრების შენახვა**
- თქვენს ბრაუზერს არ დაუბლოკავს პარამეტრების გვერდის შენახვა (შეამოწმეთ ბრაუზერის კონსოლი JavaScript შეცდომებისთვის)
- plugin აქტიურია და თქვენ ქსელის ადმინისტრირების ეკრანზე ხართ

### მოთხოვნები ყველა პროვაიდერთან ვერ სრულდება {#requests-are-failing-for-all-providers}

- გადაამოწმეთ, რომ თითოეული პროვაიდერის endpoint URL ხელმისაწვდომია თქვენი სერვერიდან (და არა მხოლოდ თქვენი ბრაუზერიდან)
- შეამოწმეთ, რომ თითოეული API გასაღები ვალიდურია და ვადა არ გასვლია
- გადახედეთ პროვაიდერის სიხშირის ლიმიტისა და კვოტის სტატუსს მათ dashboard-ში
- ჩართეთ WordPress-ის გამართვის ლოგირება (`WP_DEBUG_LOG`), რათა connector-იდან დეტალური შეცდომის შეტყობინებები ჩაიწეროს

### პროვაიდერი სარეზერვო გადართვისთვის არ გამოიყენება {#a-provider-is-not-being-used-for-fallback}

გადათრევით გადალაგება ცვლის სარეზერვო გადართვის რიგს. დაადასტურეთ, რომ პროვაიდერი, რომლისგანაც სარეზერვო მოთხოვნების მიღებას ელით, სიაში სწორად არის განთავსებული და რომ მისი endpoint URL და API გასაღები სწორია.

### ახალი პროვაიდერის ბარათი ავტომატურად არ იშლება {#new-provider-card-does-not-expand-automatically}

ავტომატური გაშლა მოითხოვს, რომ თქვენს ბრაუზერში JavaScript ჩართული იყოს. თუ ბარათი დაწკაპუნებისას არ იშლება, შეამოწმეთ ბრაუზერის კონსოლი შეცდომებისთვის. ბარათის შიგთავსი ყოველთვის ხელმისაწვდომია ბარათის სათაურზე დაწკაპუნებით, მაშინაც კი, თუ ავტომატური გაშლა არ ამოქმედდება.
