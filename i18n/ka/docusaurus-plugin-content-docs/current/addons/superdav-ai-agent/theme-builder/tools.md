# ხელსაწყოები და კონფიგურაცია {#tools-and-configuration}

Theme Builder იყენებს ხელსაწყოების ნაკრებს თქვენი WordPress საიტის ანალიზისთვის, დიზაინისთვის და ასაწყობად. ვერსიაში 1.16.0, **sd-ai-agent/site-scrape ახლა Tier 1 ხელსაწყოა**, რაც მას Theme Builder-ში ნაგულისხმევად ხელმისაწვდომს ხდის. Superdav AI Agent v1.18.0 ასევე წარმოადგენს ცალკე გავრცელებულ Advanced თანმხლებ პაკეტს სანდო დეველოპერული სამუშაო პროცესებისთვის; ეს ხელსაწყოები WordPress.org-ის ძირითადი პაკეტის ნაწილი არ არის და ცალკე უნდა დაინსტალირდეს და ავტორიზდეს.

## ხელმისაწვდომი ხელსაწყოები {#available-tools}

### Tier 1 ხელსაწყოები (ყოველთვის ხელმისაწვდომი) {#tier-1-tools-always-available}

Tier 1 ხელსაწყოები Theme Builder-ში ნაგულისხმევად ხელმისაწვდომია დამატებითი კონფიგურაციის გარეშე.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**დანიშნულება:** არსებული ვებსაიტების ანალიზი დიზაინის შთაგონების, შიგთავსის სტრუქტურისა და ფუნქციონალის ამოსაღებად.

**შესაძლებლობები:**

- **ვებსაიტის ანალიზი** — კონკურენტი ან შთაგონებისთვის განკუთვნილი ვებსაიტების scrape და ანალიზი
- **დიზაინის ამოღება** — ფერების, ფონტებისა და განლაგების შაბლონების იდენტიფიცირება
- **შიგთავსის სტრუქტურა** — გვერდების ორგანიზაციისა და იერარქიის გაგება
- **ფუნქციების აღმოჩენა** — plugin-ებისა და ფუნქციონალის იდენტიფიცირება
- **წარმადობის ანალიზი** — გვერდის სიჩქარისა და ოპტიმიზაციის შემოწმება
- **SEO ანალიზი** — meta tag-ებისა და structured data-ს გადახედვა

**გამოყენება:**

```
გააანალიზე example.com-ის დიზაინი და გამოიყენე ის შთაგონებად ჩემი საიტისთვის.
```

**რას იღებს:**

- ფერთა პალიტრა და ფერების გამოყენება
- ტიპოგრაფია (ფონტები და ზომები)
- განლაგების სტრუქტურა და დაშორებები
- ნავიგაციის შაბლონები
- შიგთავსის ორგანიზაცია
- სურათებისა და მედიის გამოყენება
- ინტერაქტიული ელემენტები
- მობილურ მოწყობილობებზე ადაპტირება

**შეზღუდვები:**

- ვერ ასკანერებს პაროლით დაცულ საიტებს
- იცავს robots.txt-სა და საიტის პოლიტიკებს
- შესაძლოა ვერ აღბეჭდოს დინამიკური შიგთავსი
- JavaScript-ზე ძლიერად დამოკიდებულ საიტებს შეიძლება შეზღუდული ანალიზი ჰქონდეთ
- დიდი საიტების ანალიზს შეიძლება მეტი დრო დასჭირდეს

### Tier 2 ხელსაწყოები (არასავალდებულო) {#tier-2-tools-optional}

Tier 2 ხელსაწყოები ხელმისაწვდომია, როდესაც ისინი Theme Builder-ის პარამეტრებში აშკარად ჩართულია.

#### გაფართოებული ანალიტიკა {#advanced-analytics}

უზრუნველყოფს დეტალურ წარმადობის მეტრიკებს:

- გვერდის ჩატვირთვის დროები
- Core Web Vitals
- SEO ქულა
- ხელმისაწვდომობის ქულა
- საუკეთესო პრაქტიკების ქულა

#### შიგთავსის ოპტიმიზატორი {#content-optimizer}

აანალიზებს და გვთავაზობს გაუმჯობესებებს შემდეგისთვის:

- წაკითხვადობა
- SEO ოპტიმიზაცია
- საკვანძო სიტყვების გამოყენება
- შიგთავსის სტრუქტურა
- მოწოდება მოქმედებისკენ განთავსება

### Tier 3 ხელსაწყოები (Premium) {#tier-3-tools-premium}

Tier 3 ხელსაწყოები დამატებით კონფიგურაციას ან premium ფუნქციებს მოითხოვს.

#### AI შიგთავსის გენერატორი {#ai-content-generator}

აგენერირებს შიგთავსს შემდეგისთვის:

- პროდუქტის აღწერილობები
- სერვისის გვერდები
- ბლოგის პოსტები
- meta აღწერილობები
- მოწოდება მოქმედებისკენ ტექსტი

#### გაფართოებული დიზაინის ხელსაწყოები {#advanced-design-tools}

უზრუნველყოფს გაფართოებულ დიზაინ შესაძლებლობებს:

- მორგებული CSS-ის გენერაცია
- ანიმაციის შექმნა
- ინტერაქტიული ელემენტების დიზაინი
- გაფართოებული ფერთა თეორია
- ტიპოგრაფიის ოპტიმიზაცია

### Advanced თანმხლები ხელსაწყოები (ცალკე პაკეტი) {#advanced-companion-tools-separate-package}

Advanced თანმხლები პაკეტი ძირითადი Superdav AI Agent გამოშვებისგან ცალკე ვრცელდება. ის განკუთვნილია სანდო დეველოპერული და საიტის მფლობელის გარემოებისთვის, სადაც ადმინისტრატორები აგენტს აშკარად აძლევენ მაღალზემოქმედებიანი ხელსაწყოების გამოყენების უფლებას.

Advanced თანმხლები შესაძლებლობები შეიძლება მოიცავდეს:

- **ფაილური სისტემის ხელსაწყოები** — დამტკიცებული ფაილების დათვალიერება და შეცვლა სანდო გარემოებში
- **მონაცემთა ბაზის ხელსაწყოები** — საიტის მონაცემების მოთხოვნა ან განახლება, როდესაც ეს აშკარად ავტორიზებულია
- **WP-CLI ხელსაწყოები** — WordPress-ის მოვლისა და შემოწმების ბრძანებების გაშვება
- **REST dispatcher ხელსაწყოები** — რეგისტრირებული REST endpoint-ების გამოძახება კონტროლირებადი dispatch სამუშაო პროცესებით
- **Plugin builder ხელსაწყოები** — plugin კოდის საწყისი სტრუქტურის შექმნა და იტერაცია
- **Git snapshot ხელსაწყოები** — snapshot-ების შექმნა სარისკო დეველოპერულ ოპერაციებამდე
- **მომხმარებლების მართვის ხელსაწყოები** — ნებადართულ ადგილებში მომხმარებლის ჩანაწერების გადახედვაში ან კორექტირებაში დახმარება
- **Benchmark ხელსაწყოები** — წარმადობის ან შესაძლებლობების benchmark-ების გაშვება დეველოპერული გადახედვისთვის

არ დაადოკუმენტიროთ ისინი როგორც ზოგადად ხელმისაწვდომი Theme Builder ხელსაწყოები ყველა ინსტალაციისთვის. ისინი ხელმისაწვდომია მხოლოდ მაშინ, როდესაც Advanced თანმხლები პაკეტი დაინსტალირებულია, ჩართულია და შეზღუდულია სანდო ადმინისტრატორებისთვის.

## ხელსაწყოების კონფიგურაცია {#tool-configuration}

### ხელსაწყოების ჩართვა {#enabling-tools}

Theme Builder-ში დამატებითი ხელსაწყოების ჩასართავად:

1. გადადით **Gratis AI Agent → Settings**
2. გადადით **Theme Builder → Tools**
3. საჭიროებისამებრ ჩართეთ/გამორთეთ ხელსაწყოები
4. შეინახეთ პარამეტრები

### ხელსაწყოების ნებართვები {#tool-permissions}

ზოგიერთ ხელსაწყოს სჭირდება ნებართვები:

- **საიტის scraping** — მოითხოვს ინტერნეტზე წვდომას
- **ანალიტიკა** — მოითხოვს Google Analytics ინტეგრაციას
- **შიგთავსის გენერაცია** — მოითხოვს API keys
- **გაფართოებული ფუნქციები** — შეიძლება მოითხოვდეს premium subscription-ს
- **Advanced თანმხლები ხელსაწყოები** — მოითხოვს ცალკე გავრცელებულ Advanced პაკეტს და ადმინისტრატორის აშკარა ნდობას

### API Keys და სერტიფიკატები {#api-keys-and-credentials}

დააკონფიგურირეთ API keys იმ ხელსაწყოებისთვის, რომლებიც ამას მოითხოვს:

1. გადადით **Gratis AI Agent → Settings → API Keys**
2. შეიყვანეთ სერტიფიკატები თითოეული ხელსაწყოსთვის
3. შეამოწმეთ კავშირი
4. შეინახეთ უსაფრთხოდ

### Advanced თანმხლები პაკეტის ინსტალაცია {#installing-the-advanced-companion-package}

Advanced თანმხლები პაკეტი WordPress.org-ის ძირითადი ZIP-ისგან ცალკე გამოდის. დააინსტალირეთ ის მხოლოდ ოფიციალური პროექტის გავრცელების არხიდან შესაბამისი გამოშვებისთვის, შემდეგ შეზღუდეთ წვდომა სანდო ადმინისტრატორებზე. production საიტზე ფაილური სისტემის, მონაცემთა ბაზის, WP-CLI-ის, REST dispatcher-ის, plugin builder-ის, მომხმარებლების მართვის ან benchmark ხელსაწყოების ჩართვამდე გადახედეთ ადამიანური დამტკიცების კარიბჭეებსა და git snapshot ქცევას.

## sd-ai-agent/site-scrape-ის გამოყენება {#using-sd-ai-agentsite-scrape}

### საბაზისო გამოყენება {#basic-usage}

სთხოვეთ Theme Builder-ს ვებსაიტის ანალიზი:

```
გააანალიზე ჩემი კონკურენტის საიტის დიზაინი competitor.com-ზე
და შემომთავაზე დიზაინის გაუმჯობესებები ჩემი საიტისთვის.
```

### კონკრეტული ანალიზი {#specific-analysis}

მოითხოვეთ კონკრეტული ტიპის ანალიზი:

```
ამოიღე ფერთა პალიტრა example.com-იდან და გამოიყენე ის შთაგონებად.
```

```
გააანალიზე example.com-ის ნავიგაციის სტრუქტურა და მსგავსი
ორგანიზაცია გამოიყენე ჩემს საიტზე.
```

```
შეამოწმე example.com-ის მობილურ მოწყობილობებზე ადაპტირება და დარწმუნდი, რომ ჩემი საიტიც
ასეთივე ადაპტირებულია.
```

### შედარებითი ანალიზი {#comparison-analysis}

შეადარეთ რამდენიმე საიტი:

```
შეადარე site1.com-ისა და site2.com-ის დიზაინები და შექმენი დიზაინი,
რომელიც ორივეს საუკეთესო ელემენტებს აერთიანებს.
```

## ხელსაწყოების შეზღუდვები და გასათვალისწინებელი საკითხები {#tool-limitations-and-considerations}

### სიხშირის შეზღუდვა {#rate-limiting}

- Scraping შეზღუდულია სიხშირით, რათა თავიდან აიცილოს სერვერის გადატვირთვა
- ერთსა და იმავე საიტზე მრავალჯერადი მოთხოვნა შეიძლება შეიზღუდოს
- დიდი საიტების ანალიზს შეიძლება მეტი დრო დასჭირდეს

### შიგთავსის სიზუსტე {#content-accuracy}

- დინამიკური შიგთავსი შეიძლება სრულად არ იყოს აღბეჭდილი
- JavaScript-ით რენდერირებული შიგთავსი შეიძლება არასრული იყოს
- რეალური დროის მონაცემები შეიძლება არ აისახოს
- ზოგიერთი შიგთავსი შეიძლება ფასიანი წვდომის მიღმა იყოს

### სამართლებრივი და ეთიკური მოსაზრებები {#legal-and-ethical-considerations}

- პატივი ეცით robots.txt-ს და საიტის პოლიტიკებს
- ნუ მოიპოვებთ საავტორო უფლებებით დაცულ შიგთავსს რეპროდუცირებისთვის
- გამოიყენეთ ანალიზი შთაგონებისთვის და არა კოპირებისთვის
- გადაამოწმეთ, რომ გაქვთ ამოღებული შიგთავსის გამოყენების უფლებები
- დაიცავით გაანალიზებული საიტების მომსახურების პირობები

### წარმადობაზე გავლენა {#performance-impact}

- დიდი საიტების Scraping-ს შეიძლება დრო დასჭირდეს
- რამდენიმე ერთდროული Scraping შეიძლება უფრო ნელი იყოს
- ქსელთან კავშირი გავლენას ახდენს სიჩქარეზე
- დიდ მონაცემთა ნაკრებებს შეიძლება მეტი დამუშავება დასჭირდეს

## საუკეთესო პრაქტიკები {#best-practices}

### საიტის ანალიზის გამოყენება {#using-site-analysis}

1. **გაანალიზეთ რამდენიმე საიტი** — შეაგროვეთ შთაგონება რამდენიმე წყაროდან
2. **ფოკუსირდით სტრუქტურაზე** — შეისწავლეთ განლაგებისა და ორგანიზების ნიმუშები
3. **ამოიღეთ ფერები** — გამოიყენეთ ფერთა პალიტრები საწყის წერტილებად
4. **შეისწავლეთ ტიპოგრაფია** — გამოავლინეთ შრიფტების კომბინაციები, რომლებიც მოგწონთ
5. **გადახედეთ ნავიგაციას** — გაიგეთ მენიუს ორგანიზება

### ეთიკური Scraping {#ethical-scraping}

1. **პატივი ეცით robots.txt-ს** — დაიცავით საიტის მითითებები
2. **არ დააკოპიროთ შიგთავსი** — გამოიყენეთ ანალიზი მხოლოდ შთაგონებისთვის
3. **გადაამოწმეთ უფლებები** — დარწმუნდით, რომ შეგიძლიათ ამოღებული შიგთავსის გამოყენება
4. **მიუთითეთ წყაროები** — მიუთითეთ შთაგონების წყაროები
5. **დაიცავით პირობები** — შეასრულეთ საიტის მომსახურების პირობები

### ინსტრუმენტის ეფექტიანობის მაქსიმიზაცია {#maximizing-tool-effectiveness}

1. **იყავით კონკრეტული** — მოითხოვეთ კონკრეტული ტიპის ანალიზი
2. **მიაწოდეთ კონტექსტი** — ახსენით თქვენი საიტის დანიშნულება
3. **განსაზღვრეთ მოლოდინები** — აღწერეთ თქვენი დიზაინის მიზნები
4. **გაიმეორეთ პროცესი** — დახვეწეთ შედეგების საფუძველზე
5. **შეათავსეთ ინსტრუმენტები** — გამოიყენეთ რამდენიმე ინსტრუმენტი ყოვლისმომცველი ანალიზისთვის

## პრობლემების მოგვარება {#troubleshooting}

### საიტი არ ექვემდებარება Scraping-ს {#site-wont-scrape}

- შეამოწმეთ, არის თუ არა საიტი საჯაროდ ხელმისაწვდომი
- გადაამოწმეთ, რომ robots.txt Scraping-ს უშვებს
- სცადეთ სხვა საიტი
- შეამოწმეთ ინტერნეტკავშირი
- თუ პრობლემა გაგრძელდა, დაუკავშირდით მხარდაჭერას

### არასრული ანალიზი {#incomplete-analysis}

- საიტს შეიძლება ჰქონდეს დინამიკური შიგთავსი
- JavaScript შეიძლება სრულად არ იყოს რენდერირებული
- დიდ საიტებს შეიძლება დრო ამოეწუროთ
- სცადეთ კონკრეტული გვერდების ანალიზი
- მოითხოვეთ კონკრეტული ტიპის ანალიზი

### ნელი წარმადობა {#slow-performance}

- დიდი საიტების ანალიზს მეტი დრო სჭირდება
- რამდენიმე ერთდროული Scraping უფრო ნელია
- ქსელთან კავშირი გავლენას ახდენს სიჩქარეზე
- სცადეთ არაპიკურ საათებში
- ჯერ გააანალიზეთ უფრო მცირე სექციები
