# მენიუს შექმნა

**მენიუს შექმნის** შესაძლებლობა ქმნის WordPress ნავიგაციის მენიუებს, გვერდების სათაურებისგან განსხვავებული ცალკეული ნავიგაციის ეტიკეტების მხარდაჭერით.

## მიმოხილვა

ეს შესაძლებლობა აფართოებს მენიუს შექმნის სტანდარტულ ფუნქციონალს `navigation_label` პარამეტრის მითითების შესაძლებლობით. ეს გაძლევთ საშუალებას შექმნათ მენიუები, სადაც ნავიგაციაში ნაჩვენები ეტიკეტი განსხვავდება გვერდის სათაურისგან, რაც უზრუნველყოფს მეტ მოქნილობას საიტის სტრუქტურასა და მომხმარებლის გამოცდილებაში.

## პარამეტრები

| პარამეტრი | ტიპი | სავალდებულო | აღწერა |
|-----------|------|----------|-------------|
| `name` | string | კი | მენიუს სახელი, მაგ. `Primary Navigation` |
| `location` | string | არა | Theme-ის მდებარეობა, რომელსაც ეს მენიუ უნდა მიენიჭოს, მაგ. `primary` |
| `navigation_label` | string | არა | ეტიკეტი, რომელიც ნავიგაციაში გამოჩნდება (გვერდის სათაურისგან განსხვავებული) |

## დაბრუნებული მნიშვნელობა

```json
{
  "success": true,
  "menu_id": 7,
  "name": "Primary Navigation",
  "navigation_label": "Main Menu"
}
```

## ნავიგაციის ეტიკეტი გვერდის სათაურთან შედარებით

`navigation_label` პარამეტრი გაძლევთ საშუალებას განაცალკევოთ შიდა მენიუს სახელი იმ ეტიკეტისგან, რომელიც მომხმარებლებს ეჩვენებათ:

- **`name`** — შიდა მენიუს იდენტიფიკატორი, რომელსაც WordPress იყენებს (მაგ., "Primary Navigation")
- **`navigation_label`** — ეტიკეტი, რომელიც საიტის ვიზიტორებს ნავიგაციაში ეჩვენებათ (მაგ., "Main Menu")

ეს სასარგებლოა, როდესაც:
- თქვენი შიდა სახელდების კონვენცია განსხვავდება მომხმარებლისთვის ნაჩვენები ეტიკეტებისგან
- გსურთ ნავიგაციაში უფრო მოკლე ეტიკეტები, ვიდრე admin პანელში
- გჭირდებათ მრავალი ენის მხარდაჭერა განსხვავებული ეტიკეტის სიგრძეებით
- ქმნით მენიუებს კონკრეტული რეგიონებისთვის ან მომხმარებელთა ჯგუფებისთვის

## გამოყენების მაგალითები

### მაგალითი 1: მარტივი მენიუ ნავიგაციის ეტიკეტით

**მოთხოვნა:**
```
შექმენი ძირითადი ნავიგაციის მენიუ სახელად "Main Navigation" ნავიგაციის ეტიკეტით "Menu".
```

**შედეგი:**
```json
{
  "success": true,
  "menu_id": 7,
  "name": "Main Navigation",
  "navigation_label": "Menu"
}
```

### მაგალითი 2: მენიუ კონკრეტული Theme-ის მდებარეობისთვის

**მოთხოვნა:**
```
შექმენი ქვედა კოლონტიტულის მენიუ სახელად "Footer Links" ნავიგაციის ეტიკეტით "Quick Links" და მიანიჭე ის ქვედა კოლონტიტულის მდებარეობას.
```

**შედეგი:**
```json
{
  "success": true,
  "menu_id": 8,
  "name": "Footer Links",
  "navigation_label": "Quick Links",
  "location": "footer"
}
```

## ინტეგრაცია Theme Builder-თან

Theme Builder-ის გამოყენებისას, მენიუს შექმნის შესაძლებლობა:

1. ავტომატურად ამოიცნობს ხელმისაწვდომ Theme-ის მენიუს მდებარეობებს
2. ქმნის მენიუებს თქვენი დიზაინისთვის შესაბამისი ნავიგაციის ეტიკეტებით
3. მენიუებს ანიჭებს სწორ Theme-ის მდებარეობებს
4. მხარს უჭერს მენიუს ელემენტების დამატებას შექმნის შემდეგ

## დაკავშირებული შესაძლებლობები

- **`add_menu_item`** — ელემენტების დამატება არსებულ მენიუში
- **`update_menu`** — მენიუს გადარქმევა ან მისი ხელახლა მინიჭება Theme-ის მდებარეობაზე
- **`delete_menu`** — მენიუს წაშლა თქვენი საიტიდან

## საუკეთესო პრაქტიკები

- **გამოიყენეთ მკაფიო ნავიგაციის ეტიკეტები** — ეტიკეტები მომხმარებლებისთვის მოკლე და აღწერითი იყოს
- **შეუსაბამეთ Theme-ის მდებარეობებს** — სწორი ჩვენებისთვის მენიუები მიანიჭეთ შესაბამის Theme-ის მდებარეობას
- **დაგეგმეთ მენიუს სტრუქტურა** — ელემენტების შექმნამდე გადაწყვიტეთ თქვენი მენიუს იერარქია
- **შეამოწმეთ ადაპტიურობა** — დარწმუნდით, რომ მენიუები სწორად ჩანს მობილურ მოწყობილობებზე
- **ეტიკეტების ლოკალიზაცია** — გამოიყენეთ განსხვავებული ნავიგაციის ეტიკეტები სხვადასხვა ენობრივი ვერსიისთვის

## შეზღუდვები

- ნავიგაციის ეტიკეტები მხოლოდ ჩვენებისთვისაა; შიდა `name` კვლავ გამოიყენება WordPress-ის იდენტიფიკაციისთვის
- Theme-ის მხარდაჭერა განსხვავდება; ყველა Theme არ უჭერს მხარს მენიუს ყველა მდებარეობას
- მენიუს ელემენტები ცალკე უნდა დაემატოს მენიუს შექმნის შემდეგ
- ნავიგაციის ეტიკეტის შეცვლა მოითხოვს მენიუს განახლებას
