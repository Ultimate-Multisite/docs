# GoCardless

Multisite Ultimate: Pagamento GoCardless

Permetti ai tuoi clienti di pagare per le iscrizioni e i prodotti di Multisite Ultimate con addebito diretto tramite GoCardless!

## Installation

1. Carica 'ultimate-multisite-gocardless' nella directory '/wp-content/plugins/'
2. Attiva il plugin tramite il menu 'Plugins' in WordPress
3. Segui la procedura guidata passo passo per configurare il plugin

## Usage Guide

_GoCardless_ è un gateway di pagamento disponibile in **Europa e nel Regno Unito** che consente pagamenti tramite **addebito diretto**.

Il **Ultimate Multisite: GoCardless Gateway** aggiunge il supporto di _GoCardless_ a Ultimate Multisite, consentendoti di addebitare ai tuoi clienti in quella regione tramite addebito diretto.

Questo tutorial spiega come configurare _GoCardless_ per iniziare ad accettare pagamenti tramite esso sulla tua rete Ultimate Multisite.

_**Nota: GoCardless funziona solo con le seguenti valute:**_ **EUR, GBP, AUD, SEK e DKK.**

## Installing the Add-on

Per prima cosa, dovrai installare l'add-on GoCardless per Ultimate Multisite. Puoi farlo andando su **Ultimate Multisite > Settings > Add-ons (sulla barra laterale)**. E selezionando l'add-on GoCardless da quell'elenco.

Dopo che l'add-on è stato installato e attivato, vai su **Ultimate Multisite > Settings > Payments**, attiva l'opzione GoCardless e salva.

![Payment gateway settings](/img/config/settings-payment-gateways.png)

## Getting an Access Token

Crea un account su _GoCardless_ (o sul sandbox di _GoCardless_, se vuoi **testarlo prima**, come raccomandato). Poi, dopo aver effettuato l'accesso, vai alla voce di menu **Developers**.

<!-- Screenshot unavailable: GoCardless dashboard showing Developers menu position -->

Nella pagina **Developers**, fai clic sul pulsante **Create** nell'angolo in alto a destra dello schermo, quindi fai clic sull'opzione **Access Token**.

<!-- Screenshot unavailable: GoCardless Developers page with Create button and Access Token option -->

Si aprirà una nuova finestra modale, che ti permetterà di creare un nuovo token. Assegna un nome riconoscibile e assicurati di selezionare l'opzione **Read-write access** nel menu **Scope**.

<!-- Screenshot unavailable: GoCardless modal for creating new access token with Read-write scope -->

Il token di accesso apparirà sullo schermo. Copialo e incollalo nel menu **Ultimate Multisite > Settings > Payments > GoCardless**.

Assicurati di posizionarlo nel campo corretto. Se è un **token live**, disattiva la **modalità sandbox** prima di incollare il token.

![Payment gateway settings with GoCardless](/img/config/settings-payment-gateways.png)

<!-- Screenshot unavailable: GoCardless token fields in Ultimate Multisite payment settings -->

Ecco fatto per il primo passo. Successivamente, devi configurare un listener webhook.

## Setting up the Webhook Listener

Quando si gestiscono abbonamenti, Ultimate Multisite deve essere informato di cambiamenti nello stato dell'abbonamento, nuovi pagamenti effettuati e altre cose simili.

Il modo in cui i gateway di pagamento informano Ultimate Multisite di tali cambiamenti è tramite **webhook calls**. Per questo motivo, affinché l'integrazione funzioni al 100%, dobbiamo indicare a _GoCardless_ quale URL chiamare con tutte queste informazioni.

Torna al tuo pannello **GoCardless > Developers** e fai clic nuovamente sul pulsante Create, ma questa volta seleziona l'opzione **Webhook endpoint**.

<!-- Screenshot unavailable: GoCardless Developers page with Webhook endpoint option -->A new window will open with the webhook endpoint options. It asks for the webhook URL, so head back to your **Ultimate Multisite settings > Payments > GoCardless** and copy the webhook URL listed in there:

<!-- Screenshot unavailable: Ultimate Multisite GoCardless settings showing webhook URL -->

Inoltre, puoi inserire un **webhook secret** che verrà usato per firmare le tue chiamate webhook. Qualunque cosa inserisci in quel campo, dovrai inserire anche nel campo **Ultimate Multisite Webhook Secret**.

<!-- Screenshot unavailable: GoCardless webhook endpoint configuration form with URL and secret fields -->

<!-- Screenshot unavailable: Ultimate Multisite GoCardless webhook secret field -->

Concludi salvando l'endpoint facendo clic sul pulsante **Create webhook endpoint** e poi **salva anche le tue impostazioni Ultimate Multisite**.

Ora dovresti essere pronto a accettare _GoCardless_!
