# إعدادات وكيل الذكاء الاصطناعي المجاني (Gratis AI Agent)

توفر شاشة **الإعدادات → المتقدم** (Settings → Advanced) في وكيل الذكاء الاصطناعي المجاني (Gratis AI Agent) إعدادات خاصة بمستوى المسؤول (administrator-level configuration) للتكاملات الخلفية التي تم تقديمها في الإصدار v1.5.0. توثق هذه الصفحة حقول **نقطة نهاية الملاحظات** (Feedback Endpoint) وتنسيقها المتوقع.

## الوصول إلى الإعدادات

1. في لوحة تحكم WordPress، انتقل إلى **Gratis AI Agent → الإعدادات**.
2. انقر على علامة التبويب **المتقدم** (Advanced).

## إعداد نقطة نهاية الملاحظات (Feedback Endpoint Configuration)

تستقبل نقطة نهاية الملاحظات طلبات POST من وكيل الذكاء الاصطناعي كلما قام المستخدم بتقديم ملاحظات عبر زر الإبهام للأسفل، أو عبر لافتة المطالبة التلقائية (auto-prompt banner)، أو عبر الأمر `/report-issue`.

| الحقل | الوصف |
|---|---|
| **عنوان URL لنقطة نهاية الملاحظات** (Feedback Endpoint URL) | عنوان URL الذي يستقبل ملاحظات المستخدمين كطلبات HTTP POST بجسم بيانات بصيغة JSON. |
| **مفتاح واجهة برمجة تطبيقات الملاحظات** (Feedback API Key) | رمز حامل (bearer token) يتم إرساله في ترويسة `Authorization` لكل طلب ملاحظات. اتركه فارغًا إذا لم يتطلب عنوان URL الخاص بك مصادقة. |

### حمولة JSON المتوقعة (Expected JSON Payload)

يجب أن يقبل عنوان URL الخاص بالملاحظات الخاص بك جسم بيانات بصيغة JSON يحتوي على الحقول التالية على الأقل:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "كان الرد غير صحيح بخصوص التسعير.",
  "triage_category": "factual_error"
}
```

قد تكون هناك حقول إضافية في الحمولة اعتمادًا على سياق المحادثة.

### قيم `triage_category`

تقوم طبقة الفرز (triage layer) الخاصة بالذكاء الاصطناعي بتعيين إحدى القيم التالية لـ `triage_category` قبل إعادة توجيه الحمولة:

| القيمة | المعنى |
|---|---|
| `factual_error` | قدم المساعد معلومات واقعية غير صحيحة. |
| `unhelpful_answer` | كان الرد صحيحًا من الناحية التقنية ولكنه غير مفيد. |
| `inappropriate_content` | احتوى الرد على محتوى لا ينبغي عرضه للمستخدمين. |
| `other` | لم تتطابق الملاحظات مع أي فئة معروفة. |

### المصادقة (Authentication)

إذا كان عنوان URL الخاص بك يتطلب مصادقة، فقم بتعيين حقل **مفتاح واجهة برمجة تطبيقات الملاحظات** (Feedback API Key) برمز الحامل الخاص بك. يرسل الوكيل:

```
Authorization: Bearer <your-api-key>
```

إذا كان حقل **مفتاح واجهة برمجة تطبيقات الملاحظات** فارغًا، فلن يتم إرسال ترويسة `Authorization`.

### تعطيل جمع الملاحظات (Disabling Feedback Collection)

اترك حقلي **عنوان URL لنقطة نهاية الملاحظات** و **مفتاح واجهة برمجة تطبيقات الملاحظات** فارغين. يظل زر الإبهام للأسفل وواجهة المستخدم الخاصة بالملاحظات مرئيين للمستخدمين، ولكن لن يتم إعادة توجيه أي ملاحظات إلى أي خدمة خارجية.

## مفتاح واجهة برمجة تطبيقات Brave Search

في علامة التبويب **المتقدم** أيضًا، يتيح حقل **مفتاح واجهة برمجة تطبيقات Brave Search** (Brave Search API Key) القدرة على [البحث في الإنترنت](../configuration/internet-search).

| الحقل | الوصف |
|---|---|
| **مفتاح واجهة برمجة تطبيقات Brave Search** (Brave Search API Key) | مفتاح واجهة برمجة التطبيقات الخاص بك من لوحة تحكم مطوري Brave Search. مطلوب لتمكين البحث في الإنترنت في مساعد الذكاء الاصطناعي. |

يتضمن تسمية الحقل رابطًا قابلاً للنقر إلى صفحة التسجيل في واجهة برمجة تطبيقات Brave Search. اتركه فارغًا لتعطيل البحث في الإنترنت.

راجع [البحث في الإنترنت](../configuration/internet-search) للحصول على وثائق المستخدم النهائي حول هذه الميزة.

## أعلام الميزات (Feature Flags)

تم تقديم **أعلام الميزات** (Feature Flags) أيضًا في الإصدار v1.9.0، وتوفر علامة التبويب **الإعدادات → أعلام الميزات** (Settings → Feature Flags) مفاتيح تبديل للوظائف الاختيارية. كل علم إما أنه مُفعَّل أو مُعطَّل على مستوى الشبكة؛ ولا يوجد تجاوز على مستوى الموقع في الوقت الحالي.

### الوصول إلى أعلام الميزات

1. في لوحة تحكم WordPress، انتقل إلى **Gratis AI Agent → الإعدادات**.
2. انقر على علامة التبويب **أعلام الميزات** (Feature Flags).

### أعلام التحكم في الوصول (Access Control Flags)

| العلم | الافتراضي | الوصف |
|---|---|---|
| **التقييد على المسؤولين** (Restrict to Administrators) | مُعطَّل (Off) | عند التفعيل، يمكن فقط للمستخدمين الذين يمتلكون دور `administrator` فتح لوحة دردشة وكيل الذكاء الاصطناعي. ترى جميع الأدوار الأخرى رسالة "يرجى الاتصال بمسؤولك". |
| **التقييد على مسؤولي الشبكة** (Restrict to Network Admins) | مُعطَّل (Off) | عند التفعيل على شبكة متعددة المواقع (multisite)، يمكن فقط للمسؤولين الفائقين (Super Admins) استخدام الوكيل. يتم حظر مسؤولي المواقع الفردية. له الأسبقية على "التقييد على المسؤولين" إذا تم تفعيل كليهما. |
| **السماح بالوصول للمشتركين** (Allow Subscriber Access) | مُعطَّل (Off) | عند التفعيل، يمكن للمستخدمين الذين يمتلكون دور `subscriber` استخدام واجهة الدردشة ولكنهم يقتصرون على القدرات للقراءة فقط (لا يمكنهم إنشاء منشورات أو تغيير الإعدادات). |
| **التعطيل لغير الأعضاء** (Disable for Non-Members) | مُعطَّل (Off) | يتكامل مع حالة العضوية في Ultimate Multisite. عند التفعيل، يتم إخفاء الدردشة للمواقع التي لا تحتوي على عضوية نشطة. |

### أعلام العلامة التجارية (Branding Flags)

| العلم | الافتراضي | الوصف |
|---|---|---|
| **إخفاء تذييل "مدعوم بواسطة Gratis AI Agent"** (Hide "Powered by Gratis AI Agent" Footer) | مُعطَّل (Off) | يزيل سطر الإشارة إلى العلامة التجارية المعروض في أسفل أداة الدردشة. موصى به لعمليات التسمية البيضاء (white-label deployments). |
| **اسم الوكيل المخصص** (Custom Agent Name) | *(فارغ)* | يحل محل التسمية الافتراضية "Gratis AI Agent" في رأس الدردشة وقائمة المسؤول بلوحة التحكم باسم منتجك الخاص. اتركه فارغًا لاستخدام الافتراضي. |
| **إخفاء محدد الوكيل** (Hide Agent Picker) | مُعطَّل (Off) | عند التفعيل، لا يمكن للمستخدمين التبديل بين الوكلاء الخمسة المدمجين. يتم تثبيت الوكيل الحالي على ما يتم تكوينه كافتراضي في الإعدادات → عام (Settings → General). |
| **استخدام أيقونة الموقع كصورة رمزية للدردشة** (Use Site Icon as Chat Avatar) | مُعطَّل (Off) | يحل محل الأيقونة الافتراضية للذكاء الاصطناعي في رأس أداة الدردشة بأيقونة موقع WordPress (المحددة تحت المظهر → تخصيص → هوية الموقع). |

### تطبيق التغييرات

انقر على **حفظ الإعدادات** (Save Settings) بعد تبديل أي علم. تدخل التغييرات حيز التنفيذ على الفور — ولا يتطلب الأمر مسح ذاكرة التخزين المؤقت (cache flush) أو إعادة تنشيط المكون الإضافي (plugin reactivation).
