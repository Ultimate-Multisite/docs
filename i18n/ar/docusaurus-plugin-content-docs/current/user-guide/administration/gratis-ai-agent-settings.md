# إعدادات Gratis AI Agent

توفر شاشة **الإعدادات → متقدم** في Gratis AI Agent إعدادات بمستوى المدير لتكاملات الواجهة الخلفية. توثق هذه الصفحة إعادة توجيه الملاحظات، ومفاتيح مزودي البحث، وإعداد خدمة Superdav المُدارة، وعناصر التحكم في Google Calendar، وإعدادات رسائل TextBee SMS، وعلامات الميزات على مستوى الشبكة.

## الوصول إلى الإعدادات

1. في لوحة إدارة WordPress، انتقل إلى **Gratis AI Agent → الإعدادات**.
2. انقر على تبويب **متقدم**.

## إعداد نقطة نهاية الملاحظات

تستقبل نقطة نهاية الملاحظات طلبات POST من وكيل الذكاء الاصطناعي كلما أرسل مستخدم ملاحظات عبر زر عدم الإعجاب، أو لافتة المطالبة التلقائية، أو أمر `/report-issue`.

| الحقل | الوصف |
|---|---|
| **عنوان URL لنقطة نهاية الملاحظات** | عنوان URL الذي يستقبل إرساليات الملاحظات كطلبات HTTP POST مع جسم JSON. |
| **Feedback API Key** | bearer token يُرسل في `Authorization` header لكل طلب ملاحظات. اتركه فارغًا إذا كانت نقطة النهاية لديك لا تتطلب مصادقة. |

### حمولة JSON المتوقعة

يجب أن تقبل نقطة نهاية الملاحظات لديك جسم JSON يحتوي على الحقول التالية على الأقل:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

قد توجد حقول إضافية في الحمولة حسب سياق المحادثة.

### قيم `triage_category`

تُسنِد طبقة فرز الذكاء الاصطناعي إحدى القيم التالية إلى `triage_category` قبل إعادة توجيه الحمولة:

| القيمة | المعنى |
|---|---|
| `factual_error` | قدّم المساعد معلومات واقعية غير صحيحة. |
| `unhelpful_answer` | كانت الاستجابة صحيحة تقنيًا لكنها غير مفيدة. |
| `inappropriate_content` | احتوت الاستجابة على محتوى لا ينبغي عرضه للمستخدمين. |
| `other` | لم تطابق الملاحظات فئة معروفة. |

### المصادقة

إذا كانت نقطة النهاية لديك تتطلب مصادقة، فاضبط حقل **Feedback API Key** على bearer token الخاص بك. يرسل الوكيل:

```
Authorization: Bearer <your-api-key>
```

إذا كان حقل **Feedback API Key** فارغًا، فلن يتم إرسال أي `Authorization` header.

### تعطيل جمع الملاحظات

اترك حقلي **عنوان URL لنقطة نهاية الملاحظات** و**Feedback API Key** فارغين. يظل زر عدم الإعجاب وواجهة مستخدم الملاحظات مرئيين للمستخدمين، لكن لا تتم إعادة توجيه الإرساليات إلى أي خدمة خارجية.

## Brave Search API Key

أيضًا في تبويب **متقدم**، يفعّل حقل **Brave Search API Key** قدرة [البحث في الإنترنت](../configuration/internet-search).

| الحقل | الوصف |
|---|---|
| **Brave Search API Key** | مفتاح API الخاص بك من لوحة مطوري Brave Search. مطلوب لتفعيل البحث في الإنترنت داخل مساعد الذكاء الاصطناعي. |

يتضمن عنوان الحقل رابطًا قابلًا للنقر إلى صفحة التسجيل في Brave Search API. اتركه فارغًا لتعطيل البحث في الإنترنت.

راجع [البحث في الإنترنت](../configuration/internet-search) للاطلاع على وثائق المستخدم النهائي حول هذه الميزة.

## خدمة Superdav المُدارة

يضيف Superdav AI Agent v1.18.0 نقاط نهاية خدمة Superdav المُدارة وتوفير الاتصال التلقائي للمواقع المدعومة. استخدم عناصر التحكم هذه عندما ينبغي لموقعك الاتصال بالمزود المستضاف بدلًا من نقطة نهاية خدمة مُعدّة يدويًا.

| الحقل | الوصف |
|---|---|
| **خدمة Superdav المُدارة** | تفعّل اتصال خدمة Superdav المستضافة للمواقع المدعومة. |
| **توفير الاتصال** | يبدأ توفير نقطة النهاية وبيانات الاعتماد تلقائيًا. استخدم هذا بعد تأكيد أن الموقع يجب أن يستخدم المزود المُدار. |
| **نقطة نهاية الخدمة / حالة الاتصال** | يعرض نقطة النهاية الحالية أو حالة الاتصال بعد التوفير. |

بعد التوفير، احفظ الإعدادات وتحقق من حالة الاتصال قبل الاعتماد على سير عمل الخدمة المُدارة. إذا فشل التوفير، فراجع إرشادات إعادة المحاولة المعروضة وتأكد من أن الموقع لديه إذن لاستخدام المزود المستضاف.

## إعداد Google Calendar

عند تفعيل ميزات التقويم في Superdav AI Agent v1.18.0، يمكن للوكيل قراءة التقويمات المُعدّة وتفاصيل الأحداث. أدوات التقويم موجهة للقراءة ومفيدة للتذكيرات الواعية بالجدول، ومتابعة الحضور، ومطابقة جهات الاتصال.

| الحقل | الوصف |
|---|---|
| **بيانات اعتماد Google Calendar** | تخزن بيانات الاعتماد أو اتصال الرمز المطلوب لقراءة بيانات التقويم. |
| **اختيار التقويم** | يحدد التقويمات المُعدّة التي يجوز للوكيل فحصها. |
| **حالة اتصال التقويم** | تؤكد ما إذا كانت بيانات الاعتماد الحالية تستطيع قراءة التقويمات والأحداث. |

أبقِ بيانات اعتماد التقويم مقتصرة على التقويمات التي يحتاجها الوكيل. أعد الاتصال أو بدّل بيانات الاعتماد إذا أشارت الحالة إلى رمز منتهي الصلاحية.

## إشعارات TextBee SMS

يضيف Superdav AI Agent v1.18.0 خدمة TextBee كمزود SMS لسير عمل الإشعارات المُعدّة. ينبغي إقران إشعارات SMS ببوابات موافقة بشرية للرسائل الحساسة أو الموجهة للمستخدمين.

| الحقل | الوصف |
|---|---|
| **مفتاح TextBee API** | يصادق الطلبات إلى مزود TextBee SMS. |
| **جهاز / مرسل TextBee** | يحدد مرسل TextBee أو الجهاز المستخدم للرسائل الصادرة، عندما يطلب المزود ذلك. |
| **تفعيل إشعارات SMS** | يسمح لسير العمل المعتمد بإرسال إشعارات الرسائل النصية. اتركه معطلًا لمنع إرسال SMS. |

أرسل رسالة اختبار فقط إلى رقم مملوك للمدير، ثم أكد سلوك بوابة الموافقة قبل تفعيل التذكيرات المجدولة أو الموجهة للحضور.

## علامات الميزات

قُدّم أيضًا في v1.9.0 تبويب **الإعدادات → علامات الميزات** الذي يوفر مفاتيح تبديل للوظائف الاختيارية. تكون كل علامة إما مفعّلة أو معطّلة على مستوى الشبكة؛ ولا يوجد تجاوز لكل موقع في الوقت الحالي.

### الوصول إلى علامات الميزات

1. في لوحة إدارة WordPress، انتقل إلى **Gratis AI Agent → الإعدادات**.
2. انقر على تبويب **علامات الميزات**.

### علامات التحكم في الوصول

| العلامة | الافتراضي | الوصف |
|---|---|---|
| **التقييد على المسؤولين** | إيقاف | عند التفعيل، يمكن فقط للمستخدمين ذوي دور `administrator` فتح لوحة دردشة وكيل الذكاء الاصطناعي. ترى جميع الأدوار الأخرى رسالة "اتصل بالمسؤول لديك" بدلًا من ذلك. |
| **التقييد على مسؤولي الشبكة** | إيقاف | عند التفعيل على شبكة متعددة المواقع، يمكن فقط للمسؤولين الفائقين استخدام الوكيل. يتم حظر مسؤولي المواقع الفردية. تكون له الأولوية على "التقييد على المسؤولين" إذا كان كلاهما مفعّلًا. |
| **السماح بوصول المشتركين** | إيقاف | عند التفعيل، يمكن للمستخدمين ذوي دور `subscriber` استخدام واجهة الدردشة لكنهم يقتصرون على إمكانات القراءة فقط (لا إنشاء منشورات أو تغييرات في الإعدادات). |
| **التعطيل لغير الأعضاء** | إيقاف | يتكامل مع حالة عضوية Ultimate Multisite. عند التفعيل، يتم إخفاء الدردشة للمواقع التي لا تملك عضوية نشطة. |

### علامات العلامة التجارية

| العلامة | الافتراضي | الوصف |
|---|---|---|
| **إخفاء تذييل "Powered by Gratis AI Agent"** | إيقاف | يزيل سطر إسناد العلامة التجارية المعروض أسفل ودجت الدردشة. موصى به لعمليات النشر ذات العلامة البيضاء. |
| **اسم الوكيل المخصص** | *(فارغ)* | يستبدل تسمية "Gratis AI Agent" الافتراضية في ترويسة الدردشة وقائمة الإدارة باسم المنتج الخاص بك. اتركه فارغًا لاستخدام الافتراضي. |
| **إخفاء منتقي الوكلاء** | إيقاف | عند التفعيل، لا يستطيع المستخدمون التبديل بين الوكلاء الخمسة المدمجين. يتم تثبيت الوكيل الحالي على أيٍّ كان مضبوطًا كافتراضي في Settings → General. |
| **استخدام أيقونة الموقع كصورة رمزية للدردشة** | إيقاف | يستبدل أيقونة الذكاء الاصطناعي الافتراضية في ترويسة ودجت الدردشة بأيقونة موقع WordPress (المضبوطة ضمن Appearance → Customize → Site Identity). |

### علامات أمان الأتمتة

يقدّم Superdav AI Agent v1.18.0 بوابات موافقة بشرية وسجلات تذكير لتشغيل الأتمتة بشكل أكثر أمانًا. قد تظهر عناصر التحكم هذه في علامات الميزات أو إعدادات الأتمتة المتقدمة، اعتمادًا على الحزمة المثبتة.

| العلامة | الافتراضي | الوصف |
|---|---|---|
| **طلب موافقة بشرية** | موصى به قيد التشغيل | يوقف الأتمتات الحساسة مؤقتًا حتى يراجع مستخدم مخوّل الإجراء المقترح ويؤكده. |
| **إزالة تكرار التذكيرات** | تشغيل | يسجل التذكيرات المرسلة حتى لا تؤدي إعادة المحاولة أو عمليات التشغيل المجدولة إلى إرسال إشعارات مكررة. |
| **تمكين أدوات التقويم** | إيقاف حتى يتم التكوين | يسمح للوكيل بقراءة تقويمات Google والأحداث المكوّنة. |
| **تمكين إشعارات الرسائل النصية القصيرة** | إيقاف حتى يتم التكوين | يسمح لسير العمل المعتمدة بإرسال إشعارات TextBee عبر الرسائل النصية القصيرة بعد حفظ بيانات الاعتماد. |

### تطبيق التغييرات

انقر على **حفظ الإعدادات** بعد تبديل أي علامة. تسري التغييرات فورًا — لا يلزم تفريغ ذاكرة التخزين المؤقت أو إعادة تفعيل plugin.
