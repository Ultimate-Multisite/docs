# Domain Seller: الإعداد وتكوين المزوّد

تأتي إضافة Domain Seller بمعالج إعداد إرشادي يرشدك خلال كل خطوة مطلوبة. تغطي هذه الصفحة تدفق المعالج وكيفية تكوين المزوّدين أو إعادة تكوينهم لاحقًا.

## المتطلبات

- **Multisite Ultimate** الإصدار v2.4.12 أو أعلى، مفعّل على الشبكة
- **PHP** 7.4+
- بيانات اعتماد API لمُسجّل مدعوم واحد على الأقل

## معالج الإعداد عند التشغيل الأول

يبدأ معالج الإعداد تلقائيًا في المرة الأولى التي تفعّل فيها الإضافة على مستوى الشبكة. وهو متاح أيضًا في أي وقت من **مسؤول الشبكة › Ultimate Multisite › إعداد Domain Seller**.

### الخطوة 1 — اختر مزوّدًا

حدّد المُسجّل الذي تريد الاتصال به. الخيارات المدعومة:

| المزوّد | إدارة DNS | خصوصية WHOIS |
|---|---|---|
| OpenSRS | نعم | نعم |
| Namecheap | لا | نعم (WhoisGuard، مجاني) |
| HostAfrica | نعم | نعم (حماية الهوية) |
| Openprovider | نعم | نعم |
| Hostinger | عبر ربط نطاقات Hostinger الأساسي للنطاقات المستضافة | نعم |
| GoDaddy | لا | لا |
| ResellerClub | نعم | لا |
| NameSilo | لا | لا |
| Enom | نعم | لا |

### الخطوة 2 — أدخل بيانات الاعتماد

لكل مزوّد حقول بيانات اعتماد مختلفة:

**OpenSRS** — اسم المستخدم والمفتاح الخاص (من لوحة تحكم موزّع OpenSRS)

**Namecheap** — اسم المستخدم ومفتاح API (من Account › الأدوات › الوصول إلى API)

**HostAfrica** — endpoint API موزّع النطاقات وبيانات الاعتماد من وحدة موزّع HostAfrica. لا يوجد endpoint sandbox منفصل موثّق حاليًا؛ اختبر باستخدام فحوصات آمنة للقراءة فقط قبل تشغيل التسجيلات الحية.

**Openprovider** — اسم المستخدم وكلمة المرور مع تمكين الوصول إلى API. يستخدم وضع sandbox الاختياري Openprovider sandbox API، ويمكن إعادة استخدام معرّف عميل افتراضي اختياري للتسجيلات.

**Hostinger** — رمز API المشترك لـ Hostinger hPanel من تكامل Hostinger الأساسي. يشغّل الرمز نفسه ربط النطاقات الأساسي وعمليات تسجيل Domain Seller.

**GoDaddy** — مفتاح API والسر (من developer.godaddy.com)

**ResellerClub** — معرّف الموزّع ومفتاح API (من لوحة تحكم ResellerClub)

**NameSilo** — مفتاح API (من namesilo.com › Account › مدير API)

**Enom** — معرّف Account ورمز API

حدّد **وضع Sandbox** حيثما كان متاحًا للاختبار مقابل بيئة اختبار المزوّد قبل الانتقال إلى التشغيل الحي.

### الخطوة 3 — اختبر الاتصال

انقر **اختبار الاتصال**. يرسل المعالج استدعاء API خفيفًا للتحقق من بيانات الاعتماد والاتصال. أصلح أي مشكلات في بيانات الاعتماد قبل المتابعة.

### الخطوة 4 — استورد TLDs

انقر **استيراد TLDs** لجلب كل TLDs المتاحة وأسعار الجملة من المزوّد المتصل. يملأ هذا قائمة TLD المستخدمة بواسطة منتجات النطاقات. قد يستغرق الاستيراد 30–60 ثانية للمزوّدين الذين لديهم كتالوجات TLD كبيرة.

تتم أيضًا إعادة مزامنة TLDs تلقائيًا مرة واحدة يوميًا عبر مهمة cron مجدولة.

### الخطوة 5 — أنشئ منتج نطاق

ينشئ المعالج منتج نطاق افتراضيًا شاملاً بهامش ربح 10%. يمكنك تعديل هذا المنتج فورًا أو التخطي وإنشاء المنتجات يدويًا ضمن **Ultimate Multisite › المنتجات**.

راجع [منتجات النطاقات والتسعير](./domain-products) للحصول على دليل تكوين المنتج الكامل.

## إعادة تكوين مزوّد

انتقل إلى **مسؤول الشبكة › Ultimate Multisite › الإعدادات › Domain Seller** (أو انقر **الإعدادات** في قائمة الإضافات).

تحتوي صفحة الإعدادات على:

- **تمكين بيع النطاقات** — تبديل الميزة بالكامل تشغيلًا/إيقافًا
- **المزوّد الافتراضي** — المزوّد المستخدم لعمليات البحث عن النطاقات والمنتجات الجديدة
- **الحد الأقصى لعدد TLDs لكل بحث** — عدد TLDs التي يتم فحصها عندما يبحث العميل؛ تعرض القيم الأعلى خيارات أكثر لكنها أبطأ
- **مدة تخزين التوفر مؤقتًا** — مدة تخزين نتائج التوفر والتسعير مؤقتًا؛ القيم الأقل أكثر دقة لكنها تزيد استدعاءات API
- **إدارة منتجات النطاقات** — رابط سريع إلى قائمة المنتجات
- **تكوين المزوّدين** — يفتح معالج التكامل لإضافة المزوّدين أو إعادة تكوينهم

### إضافة مزوّد ثانٍ

انقر **تكوين المزوّدين** وشغّل المعالج مرة أخرى للمُسجّل الجديد. يمكنك تكوين عدة مزوّدين في الوقت نفسه. عيّن كل منتج نطاق إلى مزوّد محدد، أو اتركه على الإعداد الافتراضي.

### مزامنة TLDs يدويًا

في صفحة الإعدادات، انقر **مزامنة TLDs** بجوار أي مزوّد مكوّن لجلب أحدث الأسعار. هذا مفيد بعد أن يحدّث مزوّد أسعار الجملة أو يضيف TLDs جديدة.

## السجلات

يكتب كل مزوّد إلى قناة سجل خاصة به. يمكن عرض السجلات ضمن **مسؤول الشبكة › Ultimate Multisite › السجلات**:

| قناة السجل | المحتويات |
|---|---|
| `domain-seller-registration` | كل محاولات التسجيل (النجاح والفشل) |
| `domain-seller-renewal` | نتائج مهمة التجديد |
| `domain-seller-opensrs` | نشاط OpenSRS API الخام |
| `domain-seller-namecheap` | نشاط Namecheap API الخام |
| `domain-seller-hostafrica` | نشاط HostAfrica API الخام |
| `domain-seller-openprovider` | نشاط Openprovider API الخام |
| `domain-seller-hostinger` | نشاط Hostinger API الخام |
| `domain-seller-godaddy` | نشاط GoDaddy API الخام |
| `domain-seller-resellerclub` | نشاط ResellerClub API الخام |
| `domain-seller-namesilo` | نشاط NameSilo API الخام |
| `domain-seller-enom` | نشاط Enom API الخام |

## ملاحظات حول إمكانات المزوّد

لا تكشف كل واجهة API لمُسجّل عن العمليات نفسها. تعرض الإضافة العمليات غير المدعومة بأخطاء واضحة موجهة للمسؤول بدلًا من الفشل بصمت.

- **HostAfrica** يدعم أوسع سير عمل مباشر للموزعين، بما في ذلك البحث، ومزامنة TLD/التسعير، والتسجيل، والتجديد، والنقل، وتحديثات nameserver، وسجلات DNS، وأكواد EPP، وقفل المسجّل، وحماية الهوية.
- **Openprovider** يدعم مزامنة TLD بأسعار الموزعين، والتسجيل، والتجديد، وعمليات النقل، وتحديثات nameserver، ومناطق DNS، وأكواد EPP، وقفل المسجّل، وخصوصية WHOIS. ويصادق باستخدام bearer token قصير الأجل يقوم الملحق بتحديثه تلقائيًا.
- **Hostinger** يدعم البحث عن التوافر، والتسجيل، والبحث في المحفظة، وتحديثات nameserver، وقفل المسجّل، وخصوصية WHOIS عبر رمز hPanel API المشترك. لا تتيح Domains API العامة لدى Hostinger تسعير الموزعين/الجملة، أو النقل الوارد، أو التجديد الصريح، أو استرجاع أكواد EPP؛ وتكون التجديدات تلقائية فقط.
