# بائع النطاقات: الإعداد وتكوين المزود

يأتي إضافة (Addon) بائع النطاقات (Domain Seller) مع معالج إعداد موجه يمر بك خطوة بخطوة عبر جميع الخطوات المطلوبة. تغطي هذه الصفحة سير عمل المعالج وكيفية تكوين أو إعادة تكوين المزودين بعد ذلك.

## المتطلبات

- **Ultimate Multisite** الإصدار v2.4.12 أو أعلى، ومفعل على مستوى الشبكة (network-activated)
- **PHP** 7.4+
- بيانات اعتماد واجهة برمجة التطبيقات (API credentials) لجهة تسجيل (registrar) مدعومة واحدة على الأقل

## معالج الإعداد عند التشغيل الأول

يتم تشغيل معالج الإعداد تلقائيًا في المرة الأولى التي تقوم فيها بتفعيل الإضافة على مستوى الشبكة. وهو متاح أيضًا في أي وقت من **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### الخطوة 1 — اختيار المزود

اختر جهة التسجيل (registrar) التي تريد الاتصال بها. الخيارات المدعومة هي:

| المزود (Provider) | إدارة نظام أسماء النطاقات (DNS management) | خصوصية WHOIS |
|---|---|---|
| OpenSRS | نعم | نعم |
| Namecheap | لا | نعم (WhoisGuard، مجاني) |
| GoDaddy | لا | لا |
| ResellerClub | نعم | لا |
| NameSilo | لا | لا |
| Enom | نعم | لا |

### الخطوة 2 — إدخال بيانات الاعتماد

يحتوي كل مزود على حقول بيانات اعتماد مختلفة:

**OpenSRS** — اسم المستخدم والمفتاح الخاص (من لوحة تحكم Reseller Control Panel الخاصة بـ OpenSRS)

**Namecheap** — اسم المستخدم ومفتاح API (من Account › Tools › API Access)

**GoDaddy** — مفتاح API والسر (من developer.godaddy.com)

**ResellerClub** — معرف الموزع (Reseller ID) ومفتاح API (من لوحة تحكم ResellerClub)

**NameSilo** — مفتاح API (من namesilo.com › Account › API Manager)

**Enom** — معرف الحساب (Account ID) ورمز API (API token)

تحقق من **Sandbox mode** (وضع الاختبار) حيثما كان متاحًا للاختبار مقابل بيئة الاختبار الخاصة بالمزود قبل الانتقال إلى العمل الفعلي.

### الخطوة 3 — اختبار الاتصال

انقر على **Test Connection** (اختبار الاتصال). يرسل المعالج استدعاء API خفيف الوزن للتحقق من بيانات الاعتماد وقابلية الاتصال. قم بإصلاح أي مشكلات في بيانات الاعتماد قبل المتابعة.

### الخطوة 4 — استيراد TLDs

انقر على **Import TLDs** (استيراد TLDs) لسحب جميع TLDs المتاحة والتسعير بالجملة من المزود المتصل. يؤدي هذا إلى ملء قائمة TLDs المستخدمة في منتجات النطاقات. قد يستغرق الاستيراد ما بين 30 و 60 ثانية للمزودين الذين لديهم كتالوجات TLDs كبيرة.

يتم أيضًا مزامنة TLDs تلقائيًا مرة واحدة يوميًا عبر مهمة مجدولة (cron job).

### الخطوة 5 — إنشاء منتج نطاق

ينشئ المعالج منتج نطاق افتراضي شامل (catch-all) به هامش ربح بنسبة 10%. يمكنك تعديل هذا المنتج على الفور أو تخطي هذه الخطوة وإنشاء المنتجات يدويًا ضمن **Ultimate Multisite › Products**.

راجع [Domain Products and Pricing](./domain-products) للحصول على دليل التكوين الكامل للمنتجات.

## إعادة تكوين المزود

اذهب إلى **Network Admin › Ultimate Multisite › Settings › Domain Seller** (أو انقر على **Settings** في قائمة الإضافات).

تحتوي صفحة الإعدادات على ما يلي:

- **Enable domain selling** (تمكين بيع النطاقات) — لتبديل الميزة بالكامل تشغيل/إيقاف
- **Default provider** (المزود الافتراضي) — المزود المستخدم لعمليات البحث عن النطاقات والمنتجات الجديدة
- **Max TLDs per search** (الحد الأقصى لـ TLDs لكل بحث) — عدد TLDs التي سيتم التحقق منها عندما يبحث العميل؛ القيم الأعلى تعرض خيارات أكثر ولكنها أبطأ
- **Availability cache duration** (مدة ذاكرة التخزين المؤقت للتوافر) — المدة التي يتم فيها تخزين نتائج التوافر والتسعير؛ القيم الأقل أكثر دقة ولكنها تزيد من استدعاءات API
- **Manage domain products** (إدارة منتجات النطاقات) — رابط سريع إلى قائمة المنتجات
- **Configure providers** (تكوين المزودين) — يفتح معالج التكامل (Integration Wizard) لإضافة أو إعادة تكوين المزودين

### إضافة مزود ثانٍ

انقر على **Configure providers** وقم بتشغيل المعالج مرة أخرى لجهة التسجيل الجديدة. يمكنك تكوين عدة مزودين في وقت واحد. قم بتعيين كل منتج نطاق لمزود معين، أو اتركه على الافتراضي.

### مزامنة TLDs يدويًا

في صفحة الإعدادات، انقر على **Sync TLDs** (مزامنة TLDs) بجوار أي مزود مُكوّن لسحب أحدث الأسعار. هذا مفيد بعد أن يقوم المزود بتحديث الأسعار بالجملة أو يضيف TLDs جديدة.

## السجلات (Logs)

يقوم كل مزود بالكتابة في قناة سجل خاصة به. يمكن عرض السجلات ضمن **Network Admin › Ultimate Multisite › Logs**:

| قناة السجل (Log channel) | المحتويات |
|---|---|
| `domain-seller-registration` | جميع محاولات التسجيل (الناجحة والفاشلة) |
| `domain-seller-renewal` | نتائج مهمة التجديد |
| `domain-seller-opensrs` | نشاط OpenSRS الخام لواجهة برمجة التطبيقات (API) |
| `domain-seller-namecheap` | نشاط Namecheap الخام لواجهة برمجة التطبيقات (API) |
| `domain-seller-godaddy` | نشاط GoDaddy الخام لواجهة برمجة التطبيقات (API) |
| `domain-seller-resellerclub` | نشاط ResellerClub الخام لواجهة برمجة التطبيقات (API) |
| `domain-seller-namesilo` | نشاط NameSilo الخام لواجهة برمجة التطبيقات (API) |
| `domain-seller-enom` | نشاط Enom الخام لواجهة برمجة التطبيقات (API) |
