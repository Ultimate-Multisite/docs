# حقل الدفع وإدارة DNS للعملاء {#checkout-field-and-customer-dns-management}

## حقل اختيار النطاق في الدفع {#the-domain-selection-checkout-field}

حقل **اختيار النطاق** هو عنصر دفع يمنح العملاء خيارًا لطريقة الحصول على نطاق موقعهم. أضِفه إلى أي نموذج دفع لتمكين بيع النطاقات.

### إضافة الحقل إلى نموذج دفع {#adding-the-field-to-a-checkout-form}

1. انتقل إلى **إدارة الشبكة › Ultimate Multisite › نماذج الدفع**
2. افتح نموذج دفع أو أنشئ واحدًا
3. في محرّر الدفع، انقر على **إضافة حقل**
4. اختر **اختيار النطاق** من قائمة الحقول
5. اضبط خيارات الحقل (انظر أدناه)
6. احفظ النموذج

### خيارات الحقل {#field-options}

**أوضاع النطاق** — اختر علامات التبويب التي يراها العميل. يمكن تفعيل كل وضع أو تعطيله بشكل مستقل:

| الوضع | ماذا يفعل |
|---|---|
| **النطاق الفرعي** | يستخدم العميل نطاقًا فرعيًا مجانيًا على شبكتك (مثل `mysite.yournetwork.com`). لا حاجة إلى دفع. |
| **تسجيل نطاق جديد** | يبحث العميل عن نطاق جديد ويسجله عبر المزوّد الذي قمت بإعداده. يستخدم منتج النطاق المطابق للتسعير. |
| **نطاق موجود** | يربط العميل نطاقًا يملكه مسبقًا. لا توجد رسوم تسجيل. يتم ربط النطاق تلقائيًا بموقعه. |

**الوضع الافتراضي** — عند تفعيل الأوضاع الثلاثة كلها، أي علامة تبويب تُفتح أولًا. اضبطه على **النطاق الفرعي** لإبقاء تسجيل النطاق اختياريًا، أو **تسجيل نطاق جديد** لتشجيع عمليات الشراء.

**منتج النطاق** — يمكنك اختياريًا تثبيت هذا الحقل على منتج نطاق محدد. إذا لم يتم ضبطه، فسيختار الملحق تلقائيًا المنتج المطابق بناءً على TLD الذي يبحث عنه العميل.

### حقول جهة اتصال المسجّل {#registrant-contact-fields}

عندما يختار العميل علامة تبويب **تسجيل نطاق جديد**، يضيف نموذج الدفع حقول جهة اتصال المسجّل ضمن النموذج:

- الاسم الأول / اسم العائلة
- عنوان البريد الإلكتروني
- العنوان (السطر 1، المدينة، الولاية/المقاطعة، الرمز البريدي، البلد)
- رقم الهاتف

هذه الحقول مطلوبة من جميع المسجلين ويتم التحقق منها قبل إجراء استدعاء API التسجيل. يتم تنسيق أرقام الهواتف تلقائيًا إلى الصيغة الدولية `+CC.NNN` التي يتوقعها المسجلون.

### عنوان URL للموقع مُنشأ تلقائيًا {#auto-generated-site-url}

عندما يسجل العميل نطاقًا أو يربطه، تتم تعبئة حقل عنوان URL للموقع تلقائيًا من النطاق المختار. لا يحتاج العملاء إلى ملء حقل URL منفصل.

### سلوك البحث {#search-behaviour}

- يتم التحقق من توفر النطاق في الوقت الفعلي باستخدام AJAX أثناء كتابة العميل
- تُعرض اقتراحات TLD بديلة عندما يكون النطاق المفضل غير متاح
- يتم جلب الأسعار مباشرةً وعرضها بوضوح (سعر التسجيل، سعر التجديد، رسوم خصوصية WHOIS الاختيارية)
- تنطبق أكواد القسائم على منتجات النطاقات مثل أي منتج آخر

**ضبط استجابة البحث:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**إضافة حقول مخصصة إلى نموذج البحث عن النطاق:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## إدارة DNS للعملاء {#customer-dns-management}

يمكن للعملاء إدارة سجلات DNS لنطاقاتهم المسجلة من صفحة **Account الخاص بي**، ضمن إدخال نطاقهم.

### أنواع السجلات المدعومة {#supported-record-types}

| النوع | الاستخدام |
|---|---|
| **A** | ربط اسم المضيف بعنوان IPv4 |
| **AAAA** | ربط اسم المضيف بعنوان IPv6 |
| **CNAME** | إنشاء اسم مستعار يشير إلى اسم مضيف آخر |
| **MX** | ضبط خادم تبادل البريد |
| **TXT** | إضافة SPF أو DMARC أو التحقق أو سجلات نصية أخرى |

### أي المزوّدين يدعمون إدارة DNS؟ {#which-providers-support-dns-management}

إدارة DNS (إضافة السجلات وتعديلها وحذفها) متاحة مع **OpenSRS** و**ResellerClub** و**Enom** و**HostAfrica** و**Openprovider**. يمكن لنطاقات **Hostinger** تحديث خوادم الأسماء عبر Domain Seller؛ تتم إدارة سجلات DNS للنطاقات المستضافة بواسطة تكامل ربط نطاقات Hostinger الأساسي. تعرض نطاقات Namecheap وGoDaddy وNameSilo معلومات الحالة والانتهاء، ولكن يجب إدارة DNS مباشرةً من لوحة تحكم المسجّل.

### سجلات DNS الافتراضية {#default-dns-records}

يمكنك إعداد سجلات DNS افتراضية يتم تطبيقها تلقائيًا عند تسجيل نطاق. انتقل إلى **الإعدادات › Domain Seller › سجلات DNS الافتراضية**.

تدعم قيم السجلات الافتراضية رمزين:

| الرمز | يُستبدل بـ |
|---|---|
| `{DOMAIN}` | اسم النطاق المسجل (مثل `example.com`) |
| `{SITE_URL}` | عنوان URL لموقع WordPress الخاص بموقع العميل |

**مثال — وجّه النطاق الجذر وwww إلى عنوان IP الخاص بخادمك:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### المسؤول: عرض DNS وتعديله {#admin-viewing-and-editing-dns}

يمكن لمسؤولي الشبكة عرض سجلات DNS وتعديلها لأي نطاق عميل من صفحة تعديل النطاق في **إدارة الشبكة › Ultimate Multisite › النطاقات**.
