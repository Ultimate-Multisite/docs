# إدارة حقول الدفع وإدارة DNS للعملاء

## حقل اختيار النطاق (Domain Selection)

يُعد حقل **Domain Selection** عنصرًا في صفحة الدفع يمنح العملاء خيارًا بشأن كيفية الحصول على نطاق موقعهم. أضفه إلى أي نموذج دفع لتمكين بيع النطاقات.

### إضافة الحقل إلى نموذج الدفع

1. اذهب إلى **Network Admin › Ultimate Multisite › Checkout Forms**
2. افتح أو أنشئ نموذج دفع
3. في محرر الدفع، انقر على **Add Field**
4. اختر **Domain Selection** من قائمة الحقول
5. قم بتكوين خيارات الحقل (انظر أدناه)
6. احفظ النموذج

### خيارات الحقل

**أوضاع النطاق (Domain modes)** — اختر علامات التبويب التي يراها العميل. يمكن تفعيل أو إلغاء تفعيل كل وضع بشكل مستقل:

| الوضع | ما يفعله |
|---|---|
| **Subdomain** | يستخدم العميل نطاقًا فرعيًا مجانيًا على شبكتك (مثل: `mysite.yournetwork.com`). لا يتطلب دفعًا. |
| **Register New Domain** | يبحث العميل عن نطاق جديد ويسجله من خلال المزود الذي قمت بتكوينه. يستخدم المنتج المطابق للنطاق لتحديد الأسعار. |
| **Existing Domain** | يربط العميل نطاقًا يمتلكه بالفعل. لا توجد رسوم تسجيل. يتم ربط النطاق تلقائيًا بموقعه. |

**الوضع الافتراضي (Default mode)** — عندما تكون جميع الأوضاع الثلاثة مفعلة، ما هي علامة التبويب التي تفتح أولاً. اضبطه على **Subdomain** لإبقاء تسجيل النطاق اختياريًا، أو **Register New Domain** لتشجيع عمليات الشراء.

**منتج النطاق (Domain product)** — يمكنك اختياريًا تثبيت هذا الحقل على منتج نطاق معين. إذا لم يتم تعيينه، يقوم الإضافة تلقائيًا باختيار المنتج المطابق بناءً على TLD الذي يبحث عنه العميل.

### حقول جهة الاتصال المسجلة (Registrant contact fields)

عندما يختار العميل علامة التبويب **Register New Domain**، يضيف نموذج الدفع حقول جهة الاتصال المسجلة بشكل متكامل:

- الاسم الأول / اسم العائلة
- عنوان البريد الإلكتروني
- العنوان (السطر 1، المدينة، الولاية/المقاطعة، الرمز البريدي، الدولة)
- رقم الهاتف

تُعد هذه الحقول إلزامية من قبل جميع مسجلي النطاقات ويتم التحقق منها قبل إجراء استدعاء API للتسجيل. يتم تنسيق أرقام الهواتف تلقائيًا إلى التنسيق الدولي `+CC.NNN` الذي يتوقعه مسجلو النطاقات.

### عنوان URL للموقع المُنشأ تلقائيًا

عندما يسجل العميل نطاقًا أو يربطه، يتم ملء حقل عنوان URL للموقع تلقائيًا من النطاق المختار. لا يحتاج العملاء إلى ملء حقل URL منفصل.

### سلوك البحث

- يتم التحقق من توفر النطاق في الوقت الفعلي باستخدام AJAX أثناء كتابة العميل
- يتم عرض اقتراحات TLD بديلة عندما يكون النطاق المفضل غير متاح
- يتم جلب الأسعار وعرضها بوضوح (سعر التسجيل، سعر التجديد، رسوم الخصوصية الاختيارية WHOIS)
- تنطبق رموز القسائم على منتجات النطاق تمامًا مثل أي منتج آخر

**ضبط استجابة البحث:**

```php
// زيادة تأخير التخفيف (debounce delay) (بالمللي ثانية) لتقليل استدعاءات API على الاتصالات البطيئة
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**إضافة حقول مخصصة إلى نموذج بحث النطاق:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'ملاحظات إضافية',
    ];
    return $fields;
});
```

## إدارة DNS للعملاء

يمكن للعملاء إدارة سجلات DNS لنطاقاتهم المسجلة من صفحة **My Account**، ضمن إدخال النطاق الخاص بهم.

### أنواع السجلات المدعومة

| النوع | الاستخدام |
|---|---|
| **A** | ربط اسم المضيف بعنوان IPv4 |
| **AAAA** | ربط اسم المضيف بعنوان IPv6 |
| **CNAME** | إنشاء اسم مستعار يشير إلى اسم مضيف آخر |
| **MX** | تعيين خادم تبادل البريد (Mail exchange server) |
| **TXT** | إضافة سجلات SPF أو DMARC أو التحقق أو سجلات نصية أخرى |

### أي المزودين يدعمون إدارة DNS؟

تتوفر إدارة DNS (إضافة، تعديل، حذف السجلات) مع **OpenSRS** و **ResellerClub** و **Enom**. تعرض نطاقات Namecheap و GoDaddy و NameSilo حالة وتاريخ انتهاء الصلاحية، ولكن يجب إدارة DNS مباشرة في لوحة تحكم المسجل.

### سجلات DNS الافتراضية

يمكنك تكوين سجلات DNS الافتراضية التي يتم تطبيقها تلقائيًا عند تسجيل نطاق. اذهب إلى **Settings › Domain Seller › Default DNS Records**.

تدعم قيم السجلات الافتراضية رمزين (tokens):

| الرمز (Token) | يتم استبداله بـ |
|---|---|
| `{DOMAIN}` | اسم النطاق المسجل (مثل: `example.com`) |
| `{SITE_URL}` | عنوان URL لموقع WordPress الخاص بالعميل |

**مثال — توجيه النطاق الرئيسي و www إلى عنوان IP الخاص بخادمك:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### المسؤول: عرض وتحرير DNS

يمكن لمسؤولي الشبكة عرض وتحرير سجلات DNS لأي نطاق عميل من صفحة تحرير النطاق في **Network Admin › Ultimate Multisite › Domains**.
