# ملاحظات الإصدار

## الإصدار 2.12.0 — صدر في 2026-05-15

*   **جديد:** تمت إضافة Hostinger (hPanel) كمزود استضافة مدعوم مع تكامل تعيين النطاق.
*   **جديد:** أداة Site Exporter تتعامل الآن مع حزم استيراد الشبكة لإعادة استعادة الموقع على مستوى الشبكة بسلاسة.
*   **إصلاح:** رسائل البريد الإلكتروني الموزعة عبر BCC تستخدم الآن رأس `undisclosed-recipients` لمنع كشف عناوين المستلمين.
*   **إصلاح:** لم يعد تاريخ انتهاء العضوية يتلف عند الحفظ بقيمة ليست تاريخًا.
*   **إصلاح:** تحديثات عضوية Stripe الآن تقوم بمسح الخصومات بشكل صحيح دون استدعاء واجهة برمجة التطبيقات المهملة `deleteDiscount` API.
*   **إصلاح:** تم تحديد إعادة توجيهات تسجيل الدخول الموحد (SSO) على المواقع ذات النطاق المعين لمنع حلقات إعادة التوجيه اللانهائية.
*   **إصلاح:** اختيار أداة التقاط الصور في معالج الإعداد يقوم الآن بتحديث نموذج البيانات الأساسي بشكل صحيح.
*   **إصلاح:** الآن، يحافظ سطر الأوامر (CLI) الخاص بـ Site Exporter على اختيار موقع الشبكة الافتراضي الصحيح.
*   **تحسين:** تمت إزالة حزمة `wp-cli` المضمنة من حزمة المكون الإضافي، مما يقلل من حجم المكون الإضافي.

## الإصدار 2.11.0 — صدر في 2026-05-11

*   **جديد:** تقوم الآن تصديرات الموقع بتضمين ملف `index.php` قابل للتشغيل الذاتي، بحيث يمكن تثبيت ملف ZIP على استضافة جديدة دون الحاجة لتثبيت مكون إضافي منفصل.
*   **جديد:** يسمح تصدير الشبكة للمسؤولين بتصدير جميع المواقع الفرعية في أرشيف واحد من صفحة إدارة تصدير الموقع.
*   **جديد:** أصبح الآن تطبيق خيار "السماح بتبديل قوالب الموقع" (Allow Site Templates plan toggle) مفروضًا عبر سلسلة احتياطية، مما يقيد توافر القالب بشكل صحيح بالنسبة لحدود الخطة.
*   **جديد:** يقوم محرر نموذج الدفع بتنبيه المستخدم عندما تتم إضافة منتج دون تكوين حقل مطلوب.
*   **جديد:** علامة تبويب إعدادات الاستيراد/التصدير تصف نطاقها بوضوح وترتبط مباشرة بأداة Site Export.

## الإصدار 2.10.0 — صدر في 2026-05-05

*   **جديد:** معالج الإعداد الموجه لـ PayPal لإدخال بيانات الاعتماد يدويًا، مع بوابة علامة OAuth لتكوين البوابة بسلاسة.
*   **جديد:** أُعيد تصميم لوحة عميل تبديل القالب لتشمل بطاقة القالب الحالي، وشبكة دائمة، وزر **إعادة تعيين القالب الحالي**.
*   **إصلاح:** لم يعد تبديل القالب يعلق واجهة المستخدم في حالة فشل AJAX.
*   **إصلاح:** تم تأمين حالات أذونات تبديل القالب ضد الوصول غير المصرح به.
*   **إصلاح:** يتم التحقق من مدخلات تجاوز الموقع (Site override inputs) قبل الحفظ.
*   **إصلاح:** يتم عرض مطالبة عنوان الفواتير الآن عندما يكون العنوان فارغًا.
*   **إصلاح:** تم حل إشعارات إهمال تحويل القيم الفارغة إلى سلاسل نصية (null-to-string deprecation notices) في PHP 8.1.
*   **إصلاح:** يتم تحميل `Currents` بشكل كسول قبل خطاف `init` لمنع مشاكل التوقيت.
*   **إصلاح:** يتم احترام مسار تسجيل الدخول الموحد (SSO) المصفى عبر جميع تدفقات تسجيل الدخول.
*   **إصلاح:** يتم الاحتفاظ بخيارات هوية الموقع الفارغة عند الحفظ.

## الإصدار 2.9.0 — صدر في 2026-04-30

*   **جديد:** تمت إضافة التصدير والاستيراد للموقع الواحد تحت **Tools > Export & Import**.
*   **إصلاح:** يتم الآن تقديم ملفات ZIP للتصدير عبر نقطة نهاية تنزيل تتطلب المصادقة.
*   **إصلاح:** تم تصحيح مخاطر حقن SQL ومشاكل الاستعلام في استعلامات التصدير/الاستيراد المعلقة.
*   **إصلاح:** لم يعد الموقع المعلق (pending site) ينشر عندما يقوم المسؤول بالتحقق يدويًا من بريد العميل الإلكتروني.
*   **إصلاح:** تم تنظيف سجلات `pending_site` اليتيمة عند فقدان العضوية.
*   **إصلاح:** تم تصحيح حشوة التنقل في الإعدادات وتصفح مرساة البحث.
*   **إصلاح:** يتم عرض المواقع المعلقة الآن أولاً في عرض "جميع المواقع".
*   **إصلاح:** تمت إضافة رأس وكيل المستخدم (User-Agent) لمزود لقطات الشاشة (mShots) لمنع أخطاء 403.
*   **إصلاح:** تم حل التبعية الدائرية لجدولة مهمة Cron الخاصة بالاستيراد.
*   **إصلاح:** تم تطبيع معرفات الجولات (Tour IDs) إلى الشرطات السفلية في مفاتيح إعدادات المستخدم.
*   **تحسين:** تم استخدام `ZipArchive` الآن بدلاً من Alchemy/Zippy لتحسين التوافق.

## الإصدار 2.8.0 — صدر في 2026-04-29

*   **جديد:** تمت إضافة مفتاح تبديل "Enable Jumper" إلى واجهة إعدادات "Other Options".
*   **جديد:** تمت إضافة عمود الحالة إلى جدول قائمة نماذج الدفع.
*   **جديد:** مُحمّل ملف `sunrise` الإضافي لتوسعات المكون الإضافي المخصص MU-plugin sunrise.
*   **تحسين:** تمت إزالة خيار الاشتراك في إعدادات الإبلاغ عن الأخطاء من صفحة الإعدادات.
*   **إصلاح:** بطاقة الموقع في صفحة الشكر — أصبحت الصورة الآن مقيدة ويتم تنسيق الروابط بشكل صحيح.
*   **إصلاح:** تم تغيير مزود لقطات الشاشة من thum.io إلى WordPress.com mShots.
*   **إصلاح:** يتم تعيين قيم افتراضية صحيحة لـ "Enable Registration" و "Default Role" عند التثبيت الجديد.
*   **إصلاح:** لم يعد `get_site_url()` يعيد قيمة فارغة عندما يتضمن النطاق منفذًا (port).
*   **إصلاح:** يتم الآن نسخ ملفات الوسائط المنسوخة بشكل صحيح عندما كانت إعدادات `copy_media` فارغة.
*   **إصلاح:** يتم إبطال ذاكرة التخزين المؤقت للكائنات (Object cache) بشكل صحيح بعد كتابة بيانات الموقع في `network-activate sitemeta`.
*   **إصلاح:** يتم ترقية النطاق المخصص تلقائيًا إلى الأساسي عند التحقق من DNS للنطاقات المكونة من 3 أجزاء.
*   **إصلاح:** يتم إلغاء العضوية المعلقة عند تنظيف الدفع منتهي الصلاحية.
*   **إصلاح:** يتم إعادة ربط مدقق قوة كلمة المرور بعد إغلاق مطالبة تسجيل الدخول المضمنة.
*   **إصلاح:** توقف إعادة تحميل الصفحة اللانهائية في صفحة الشكر عندما يكون الموقع قد تم إنشاؤه بالفعل.
*   **إصلاح:** تتم مزامنة خيار تسجيل الدخول في WP core عند تفعيل المكون الإضافي وحفظ الإعدادات.
*   **إصلاح:** تمت إضافة حماية انتهاء الصلاحية (Null expiration guard) في `calculate_expiration` للتوافق مع PHP 8.4.
*   **إصلاح:** يتم حظر التسجيلات المكررة عندما يكون لدى العميل بالفعل عضوية نشطة.
*   **إصلاح:** تمت إضافة فحص للقيم الفارغة لـ `date_expiration` في صفحة الدفع.
*   **إصلاح:** تم تعزيز توفير الموقع (Site provisioning) — بما في ذلك القيود، واستنتاج العضوية، وترقية النطاق.
*   **إصلاح:** تم تصحيح تسمية حالة فحص ما قبل التثبيت إلى "غير مُفعّل" (NOT Activated) عند فشل الفحص.
*   **إصلاح:** يتم استخدام نطاق الدفع لروابط التحقق من البريد الإلكتروني.
*   **إصلاح:** تسجيل الدخول التلقائي بعد الدفع عندما لا يتوفر حقل كلمة المرور.
*   **إصلاح:** لم تعد العضويات المجانية تنتهي صلاحيتها — ويتم التعامل معها على أنها مدى الحياة.
*   **إصلاح:** تحتفظ بوابة التحقق من البريد الإلكتروني بالموقع حتى يتحقق العميل من بريده الإلكتروني.
*   **إصلاح:** تم تصحيح المسار الأساسي لنقطة نهاية API الخاصة بـ SES v2 ومسار الهوية.
*   **إصلاح:** يتم إصدار الخطاف `wu_inline_login_error` في كتلة التقاط ما قبل الإرسال.
