# البريد الإلكتروني

يتيح إضافة Ultimate Multisite: Emails لمالكي الشبكة توفير وإدارة حسابات البريد الإلكتروني للعملاء جنبًا إلى جنب مع المواقع والعضويات. تضيف النسخة 1.1.0 إدارة حساب البريد الإلكتروني الموجهة للعملاء، وتسجيل الدخول الموحد (SSO) للبريد الإلكتروني عبر الويب، ودعم إعادة التوجيه (forwarder) الخاص بـ CyberPanel، وتحسينات التوفير في وقت إتمام عملية الشراء.

## الميزات الرئيسية

* **أداة حساب البريد الإلكتروني للعميل**: يمكن للعملاء إنشاء وإدارة حسابات البريد الإلكتروني من صفحة حسابهم دون الحاجة لفتح تذكرة دعم.
* **تسجيل الدخول الموحد (SSO) للبريد الإلكتروني عبر الويب والروابط السحرية**: يمكن للعملاء فتح البريد الإلكتروني عبر الويب مباشرة من صفحة الحساب، مع مصادقة بنقرة واحدة عندما يدعم المزود ذلك.
* **تغيير كلمات المرور**: يمكن للعملاء والمشرفين تحديث كلمات مرور صندوق البريد من أداة إدارة البريد الإلكتروني.
* **توفير الحساب عند إتمام الشراء**: يمكن إنشاء حساب البريد الإلكتروني للعميل أثناء عملية الدفع، بحيث يكون صندوق البريد جاهزًا بمجرد اكتمال مسار الشراء.
* **مُعيدات التوجيه (Forwarders) الخاصة بـ CyberPanel**: يمكن للشبكات المدعومة بـ CyberPanel إنشاء وإدارة مُعيدات التوجيه للبريد الإلكتروني كجزء من مسار خدمة البريد الإلكتروني.
* **إعدادات واعية بالمزود**: تقوم صفحة الإعدادات بتصفية المزودين لعرض التكاملات النشطة والمُعدة، بحيث لا يرى المشرفون سوى الخيارات القابلة للاستخدام.

## المتطلبات

* Ultimate Multisite 2.0.0 أو أعلى
* تثبيت WordPress Multisite
* تكامل مزود بريد إلكتروني واحد مُعد على الأقل
* بيانات اعتماد واجهة برمجة تطبيقات (API) للمزود مع صلاحية إنشاء صناديق البريد، وتغيير كلمات المرور، وإصدار روابط الوصول للبريد الإلكتروني عبر الويب حيثما كان ذلك مدعومًا

## التثبيت

1. قم بتحميل ملفات الـ plugin إلى `/wp-content/plugins/ultimate-multisite-emails`.
2. قم بتفعيل الإضافة (addon) على مستوى الشبكة من **Plugins** في لوحة تحكم مسؤول شبكة WordPress.
3. انتقل إلى **Ultimate Multisite** > **Settings** وافتح إعدادات البريد الإلكتروني الخاصة بالإضافة.
4. قم بتمكين تكاملات مزود البريد الإلكتروني التي تريد أن يستخدمها العملاء.
5. احفظ الإعدادات، ثم اختبر إنشاء الحساب باستخدام عميل غير إنتاجي أو موقع تجريبي (staging site).

## إدارة البريد الإلكتروني لحساب العميل

تضيف النسخة 1.1.0 أداة إدارة البريد الإلكتروني إلى صفحة حساب العميل. استخدمها عندما يحتاج العملاء إلى خدمة ذاتية لإنشاء وصيانة صناديق البريد الخاصة بمواقعهم.

من صفحة الحساب، يمكن للعملاء:

1. عرض حسابات البريد الإلكتروني المرتبطة بالعضوية أو الموقع الحالي الخاص بهم.
2. إنشاء صندوق بريد جديد عندما تسمح خطتهم بحسابات إضافية.
3. تغيير كلمة مرور صندوق بريد موجود.
4. فتح مزودي البريد الإلكتروني عبر الويب المدعومين من خلال SSO أو تسجيل الدخول باستخدام رابط سحري (magic-link).

تستخدم الأداة نظام Ultimate Multisite الحالي لصفحة الحساب، لذا يتم تحديد الإجراءات ضمن سياق الحساب الذي يشاهده العميل. هذا يمنع العملاء من إدارة حسابات بريد إلكتروني تخص عضوية أو موقعًا آخر.

## تسجيل الدخول الموحد (SSO) والروابط السحرية للبريد الإلكتروني عبر الويب

يوفر تسجيل الدخول الموحد (SSO) للبريد الإلكتروني عبر الويب إجراءً مباشرًا بعنوان **Open webmail** من صفحة الحساب. عندما يدعم المزود النشط ذلك، تطلب الإضافة عنوان URL وصول قصير الأجل وتعيد توجيه العميل إلى البريد الإلكتروني عبر الويب دون مطالبته بنسخ بيانات اعتماد صندوق البريد.

قد يختلف سلوك المزودين:

* المزودون الذين لديهم واجهات برمجة تطبيقات (APIs) لتسجيل الدخول الموحد (SSO) الأصلية يمكنهم إنشاء جلسة بريد إلكتروني عبر الويب موثقة ومباشرة.
* المزودون الذين يدعمون الروابط السحرية يمكنهم إنشاء رابط تسجيل دخول محدود بوقت.
* المزودون الذين لا يدعمون SSO أو الروابط السحرية يجب أن يعرضوا لا يزالون تفاصيل صندوق البريد وإجراءات إدارة كلمات المرور، ولكن بدون زر بريد إلكتروني عبر الويب بنقرة واحدة.

قم بتمكين الوصول إلى البريد الإلكتروني عبر الويب فقط للمزودين الذين قمت بتكوين بيانات اعتماد واجهة برمجة تطبيقات المطلوبة لهم. إذا كان الإجراء مفقودًا لمزود معين، فيرجى التأكد أولاً من أن المزود نشط ومُعد ويدعم روابط تسجيل الدخول للبريد الإلكتروني عبر الويب.

## توفير الحساب عند إتمام الشراء

يمكن للإضافة إنشاء حسابات البريد الإلكتروني للعملاء كجزء من عملية الدفع. هذا مفيد عندما تتضمن الخطة صندوق بريد افتراضيًا أو عندما يجمع نموذج الدفع تفاصيل حساب البريد الإلكتروني.

التطبيق الموصى به:

1. قم بتكوين حدود المزود والخطة قبل تمكين توفير الحساب عند إتمام الشراء.
2. اختبر عملية شراء باستخدام منتج تجريبي (staging product) وتأكد من إنشاء صندوق البريد بعد إتمام عملية الشراء.
3. راجع صفحة حساب العميل للتأكد من ظهور صندوق البريد الجديد في أداة البريد الإلكتروني.
4. أكد إجراءات تغيير كلمة المرور والبريد الإلكتروني عبر الويب قبل تمكين المسار للمنتجات الإنتاجية.

إذا فشل التوفير، يمكن أن يستمر وجود حساب العميل والعضوية بينما يتطلب صندوق البريد متابعة من المشرف. تحقق من سجلات نشاط Ultimate Multisite وسجلات المزود قبل المحاولة مرة أخرى.

## مُعيدات التوجيه (Forwarders) الخاصة بـ CyberPanel للبريد الإلكتروني

بالنسبة للشبكات المدعومة بـ CyberPanel، تضيف النسخة 1.1.0 دعم مُعيدات التوجيه للبريد الإلكتروني. استخدم مُعيدات التوجيه عندما يحتاج العملاء إلى عناوين مثل `info@customer-domain.test` لإعادة توجيه البريد إلى صندوق وارد آخر دون إنشاء صندوق بريد كامل.

قبل تقديم مُعيدات التوجيه، تأكد من تكوين وتوفر تكامل CyberPanel. راجع دليل [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) لمعرفة الثوابت المطلوبة للمضيف واسم المستخدم وكلمة المرور والحزمة.

## الأسئلة الشائعة

## هل يمكن للعملاء إنشاء حسابات بريد إلكتروني خاصة بهم؟

نعم. عندما يتم تمكين أداة حساب العميل وتسمح الخطة بحسابات البريد الإلكتروني، يمكن للعملاء إنشاء وإدارة صناديق البريد الخاصة بهم من صفحة الحساب.

## هل يدعم كل مزود تسجيل الدخول الموحد (SSO) للبريد الإلكتروني عبر الويب؟

لا. يعتمد دعم SSO والروابط السحرية على مزود البريد الإلكتروني النشط. لا يزال بإمكان المزودين الذين لا يملكون واجهة برمجة تطبيقات للوصول (access-link API) دعم إنشاء صناديق البريد وتغيير كلمات المرور.

## هل يمكن للمشرفين إنشاء حسابات بريد إلكتروني للعملاء؟

نعم. يمكن للمشرفين إنشاء حسابات البريد الإلكتروني من واجهة المشرف، بما في ذلك تعيين كلمة المرور الأولية عند إنشاء الحساب.

## هل مُعيدات التوجيه (Forwarders) هي نفسها صناديق البريد (Mailboxes)؟

لا. يخزن صندوق البريد الرسائل ويحتوي على بيانات اعتماد تسجيل الدخول. أما مُعيد التوجيه فيقوم بإعادة توجيه البريد إلى عنوان آخر ولا يوفر صندوق وارد منفصل.
