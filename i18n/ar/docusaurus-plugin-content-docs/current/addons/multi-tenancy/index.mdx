# تعدد المستأجرين

وزّع المواقع الفرعية عبر الخوادم وقواعد البيانات لتحسين الأداء والأمان

## أبرز ما في إصدار 1.2.0

يضيف Multi-Tenancy 1.2.0 أساس المستأجر السيادي لتدفقات عمل العزل الإنتاجي:

- تسجيل دخول تلقائي للمستأجر بلا حالة مع زيارات SSO قصيرة العمر ومحمية من إعادة التشغيل.
- عزل قاعدة البيانات ونظام الملفات لكل موقع فرعي للمستأجرين السياديين.
- قدرة تعدد المستأجرين في Hostinger لتوفير المستأجرين مع مراعاة المضيف.
- بوابات تحقق الهجرة السيادية لفحوصات البيانات القديمة ومعالجة الدفع غير المتزامنة.
- ربط قابل للتكوين لمضيف قاعدة البيانات على الجهاز نفسه لتثبيتات Bedrock وFrankenPHP وlocalhost.
- تحسينات في تمهيد المستأجر، وكاتب قاعدة البيانات، وتمهيد المخطط، وتوفير المستخدمين.
- إزالة سيادية أكثر أمانًا تنظّف بيانات اعتماد المستأجر.

راجع أدلة المستخدم حول [تدفق عمل الهجرة السيادية](/user-guide/getting-started/sovereign-migration-workflow)، و[عزل تعدد المستأجرين](/user-guide/configuration/multi-tenancy-isolation)، و[عمليات المستأجر السيادي](/user-guide/administration/sovereign-tenant-operations).

## المتطلبات

- WordPress 5.3 أو أعلى
- PHP 7.4 أو أعلى
- إضافة Multisite Ultimate (نشطة)

## التثبيت

1. ارفع ملفات الإضافة إلى دليل `/wp-content/plugins/` لديك
2. فعّل الإضافة من خلال قائمة 'Plugins' في WordPress
3. اضبط إعدادات الإضافة في إدارة الشبكة

### الاختبار

تتضمن الإضافة إطار اختبار شاملًا:

- **اختبارات الوحدة**: اختبارات قائمة على PHPUnit مع حزمة اختبار WordPress
- **صنف الاختبار الأساسي**: يوفر طرقًا مساعدة لسيناريوهات الاختبار الشائعة
- **تغطية الاختبار**: مهيأة لتتبع تغطية الكود
- **اختبار المواقع المتعددة**: تعمل الاختبارات في بيئة مواقع متعددة

### معايير الكود

- **معايير ترميز WordPress**: تُفرض عبر PHPCS
- **توافق PHP 7.4+**: دعم ميزات PHP الحديثة
- **التحليل الثابت**: تكامل PHPStan للتحقق من الأنواع
- **تحديث الكود**: قواعد Rector لترقيات PHP

### نظام البناء

- **معالجة الأصول**: تصغير ملفات CSS/JS
- **الترجمة**: إنشاء ملف POT
- **التغليف**: إنشاء أرشيف للتوزيع
- **وضع التطوير**: أصول غير مصغّرة لتصحيح الأخطاء

## التهيئة

تدعم الإضافة خيارات تهيئة متنوعة يمكن الوصول إليها من خلال واجهة إدارة الشبكة.

### تهيئة المستأجر السيادي

بالنسبة إلى المستأجرين السياديين، اضبط قاعدة بيانات المستأجر، وجذر نظام الملفات، وربط مضيف قاعدة البيانات، وقدرة المضيف قبل توجيه الزيارات إلى المستأجر. شغّل تحقق الهجرة بعد كل تغيير يؤثر في بيانات اعتماد قاعدة البيانات، أو ربط المضيف، أو التوجيه، أو توفير المستخدمين.

استخدم `localhost` فقط عندما يتوقع وقت تشغيل المستأجر ومنح قاعدة البيانات وصولًا عبر مقبس محلي أو على الجهاز نفسه. استخدم `127.0.0.1` أو اسم مضيف خدمة عندما يكون ذلك هو مضيف المنح الذي هيّأه مزودك.

### قدرة Hostinger

تعمل قدرة تعدد المستأجرين في Hostinger مع رمز تكامل Hostinger المشترك. تأكد من وجود النطاق في hPanel، وأن DNS يشير إلى حساب Hostinger، وأن إعدادات قاعدة بيانات المستأجر تطابق hPanel قبل التحقق.

## الخطافات والمرشحات

### الإجراءات

- `ultimate-multisite-addon-template_init` - يُشغَّل بعد تهيئة الإضافة
- `ultimate-multisite-addon-template_loaded` - يُشغَّل بعد تحميل جميع ملفات الإضافة

### المرشحات

- `ultimate-multisite-addon-template_settings` - تعديل إعدادات الإضافة
- `ultimate-multisite-addon-template_enabled` - تجاوز حالة تفعيل/تعطيل الإضافة
