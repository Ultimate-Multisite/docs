# موصل الذكاء الاصطناعي الأقصى لنقاط النهاية المتوافقة

يدمج موصل الذكاء الاصطناعي الأقصى لنقاط النهاية المتوافقة واجهات برمجة تطبيقات مزودي الذكاء الاصطناعي المتوافقة مع OpenAI مع شبكة المواقع المتعددة (multisite) الخاصة بـ WordPress. يمكنك إعداد العديد من مزودي الذكاء الاصطناعي — لكل منهم عنوان URL لنقطة النهاية، ومفتاح API، ونموذج افتراضي خاص به — وتوجيه الطلبات مع خاصية التراجع التلقائي (automatic fallback) عندما يتوقف أحد المزودين عن العمل.

## الميزات الرئيسية {#key-features}

- **دعم مزودي الخدمة المتعددين**: أضف بطاقات لمزودي الذكاء الاصطناعي المتعددين، ولكل منها عنوان URL لنقطة النهاية، ومفتاح API، ونموذج افتراضي خاص به.
- **إعادة الترتيب بالسحب**: قم بترتيب المزودين حسب الأولوية باستخدام مقبض السحب الموجود على كل بطاقة؛ حيث يتم إرسال الطلبات إلى المزودين بالترتيب.
- **التوجيه بالتراجع التلقائي**: إذا كان المزود الأساسي غير متاح، سيتم إعادة توجيه الطلبات تلقائيًا إلى المزود التالي المُعد.
- **إعدادات خاصة بكل مزود**: يتم إعداد كل مزود بشكل مستقل باستخدام بيانات الاعتماد واختيار النموذج الخاص به.
- **واجهة مستخدم قائمة على البطاقات**: مبنية على مكونات Card المستقرة في WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **التوسع التلقائي عند الإضافة**: تتوسع بطاقات المزود الجديدة تلقائيًا حتى تتمكن من ملء التفاصيل على الفور.
- **كسر ذاكرة التخزين المؤقت للسكريبت**: يتم كسر ذاكرة التخزين المؤقت لسكريبتات المزود عند تحديث الإضافة لضمان تحميل أحدث إصدار.

## المتطلبات {#requirements}

- WordPress 5.3 أو أحدث
- PHP 7.4 أو أحدث
- إضافة Ultimate Multisite (نشطة)
- نقطة نهاية API متوافقة مع OpenAI (مستضافة ذاتيًا أو طرف ثالث)

## التثبيت {#installation}

1. قم بتحميل ملفات الإضافة إلى دليل `/wp-content/plugins/` الخاص بك.
2. قم بتفعيل الإضافة عبر قائمة "الإضافات" (Plugins) في WordPress.
3. انتقل إلى **Ultimate Multisite → AI Connector** لإعداد الإضافة.

## الإعداد {#configuration}

### إعداد مزودي الخدمة المتعددين {#multi-provider-setup}

تستخدم شاشة الإعداد تخطيطًا قائمًا على البطاقات، حيث تمثل كل بطاقة مزود ذكاء اصطناعي واحد. يمكنك إضافة أكبر عدد من المزودين الذين تحتاجهم.

#### إضافة مزود خدمة {#adding-a-provider}

1. انتقل إلى **Ultimate Multisite → AI Connector** في لوحة تحكم الشبكة.
2. انقر على **Add Provider** (إضافة مزود خدمة).
3. تظهر بطاقة مزود خدمة جديدة وتتوسع تلقائيًا.
4. املأ تفاصيل المزود:
   - **Provider Name** (اسم المزود) — تسمية لتحديد هذا المزود (مثل: "OpenAI"، "Mistral"، "Local Ollama").
   - **Endpoint URL** (عنوان URL لنقطة النهاية) — عنوان URL الأساسي لواجهة برمجة التطبيقات المتوافقة مع OpenAI (مثل: `https://api.openai.com/v1`).
   - **API Key** (مفتاح API) — مفتاح API الخاص بك لهذا المزود.
   - **Default Model** (النموذج الافتراضي) — مُعرّف النموذج الذي سيتم استخدامه افتراضيًا لهذا المزود (مثل: `gpt-4o`، `mistral-small`).
5. انقر على **Save Settings** (حفظ الإعدادات).

#### إزالة مزود خدمة {#removing-a-provider}

انقر على زر **Remove** (إزالة) الموجود على أي بطاقة مزود خدمة لحذفها. احفظ الإعدادات لتطبيق التغيير.

#### إعادة ترتيب المزودين {#reordering-providers}

اسحب أيقونة المقبض (⣿) الموجودة على الجانب الأيسر من أي بطاقة مزود خدمة لإعادة ترتيبها. يحدد الترتيب أولوية التراجع: يتم إرسال الطلبات إلى المزود الأول، ويتم التراجع إلى المزودين اللاحقين إذا فشل الطلب.

### التوجيه بالتراجع التلقائي (Fallback Routing) {#fallback-routing}

عندما يفشل الطلب المرسل إلى المزود الأساسي — بسبب انتهاء المهلة، أو تجاوز الحد الأقصى للمعدل، أو انقطاع الخدمة — يقوم الموصل تلقائيًا بالمحاولة مرة أخرى باستخدام المزود التالي في القائمة. يستمر هذا التتابع نزولاً في القائمة حتى ينجح أحد المزودين أو يتم تجربة جميع المزودين.

| السيناريو | السلوك |
|----------|-----------|
| استجابة المزود الأساسي | يتم تلبية الطلب بشكل طبيعي |
| فشل المزود الأساسي | يتم إعادة محاولة الطلب باستخدام المزود التالي |
| فشل جميع المزودين | يتم إرجاع خطأ إلى المُستدعي |

يظل التراجع التلقائي شفافًا بالنسبة للإضافة أو القالب المُستدعي — حيث يتلقى إما استجابة ناجحة أو خطأ نهائيًا، بغض النظر عن عدد المزودين الذين تم تجربتهم.

### مرجع إعدادات المزود {#provider-configuration-reference}

| الحقل | مطلوب | الوصف |
|-------|----------|-------------|
| Provider Name | نعم | التسمية المعروضة لهذا المزود |
| Endpoint URL | نعم | عنوان URL الأساسي لواجهة برمجة التطبيقات المتوافقة مع OpenAI |
| API Key | نعم | مفتاح المصادقة لهذا المزود |
| Default Model | نعم | مُعرّف النموذج المستخدم عندما لا يتم تحديد نموذج في الطلب |

## استكشاف الأخطاء وإصلاحها {#troubleshooting}

### لا يتم عرض أي مزود خدمة {#no-providers-are-listed}

إذا كانت قائمة المزودين فارغة بعد الحفظ، يرجى التأكد من ما يلي:

- أنك نقرت على **Save Settings** (حفظ الإعدادات) بعد إضافة المزودين.
- أن متصفحك لم يمنع حفظ صفحة الإعدادات (تحقق من وحدة تحكم المتصفح بحثًا عن أخطاء JavaScript).
- أن الإضافة نشطة وأنك في شاشة لوحة تحكم الشبكة.

### فشل الطلبات لجميع المزودين {#requests-are-failing-for-all-providers}

- تحقق من أن عنوان URL لنقطة نهاية كل مزود قابل للوصول إليه من خادمك (وليس فقط من متصفحك).
- تأكد من أن كل مفتاح API صالح ولم تنتهِ صلاحيته.
- راجع حالة الحد الأقصى للمعدل والحصة (quota) الخاصة بالمزود في لوحة التحكم الخاصة به.
- قم بتمكين تسجيل أخطاء WordPress (`WP_DEBUG_LOG`) لالتقاط رسائل الخطأ التفصيلية من الموصل.

### لا يتم استخدام مزود خدمة للتراجع التلقائي {#a-provider-is-not-being-used-for-fallback}

يغير السحب لإعادة الترتيب ترتيب التراجع. تأكد من أن المزود الذي تتوقع منه تلقي طلبات التراجع موضوع بشكل صحيح في القائمة، وأن عنوان URL لنقطة النهاية ومفتاح API الخاص به صحيحان.

### لا تتوسع بطاقة المزود الجديدة تلقائيًا {#new-provider-card-does-not-expand-automatically}

يتطلب التوسع التلقائي تمكين JavaScript في متصفحك. إذا لم تتوسع البطاقة عند النقر، تحقق من وحدة تحكم المتصفح بحثًا عن الأخطاء. تظل محتويات البطاقة متاحة دائمًا عن طريق النقر على رأس البطاقة حتى لو لم يتم تشغيل التوسع التلقائي.
