# سجل التغييرات {#changelog}

## 1.9.0 — صدر في 2026-04-28 {#190--released-on-2026-04-28}

### جديد {#new}

- **قدرة `create_contact_form`** — تنشئ نموذج اتصال باستخدام إضافة النماذج النشطة (Contact Form 7، WPForms، Fluent Forms، أو Gravity Forms) وتعيد شورت كود (shortcode) جاهز للتضمين في أي منشور أو صفحة.
- **قدرة `set_featured_image`** — تعيّن صورة مميزة لمنشور أو صفحة باستخدام معرف مرفق موجود في مكتبة الوسائط (Media Library) أو رابط URL خارجي؛ وتقوم باستيراد الصورة تلقائيًا عند توفير رابط URL.
- **قدرة `batch_create_posts`** — تنشئ عدة منشورات في استدعاء قدرة واحد. تدعم نفس المعلمات المستخدمة في `create_post`، وتُبلغ عن نجاح/فشل كل منشور على حدة، وتحتوي على وضع اختياري يسمى `stop_on_error` (التوقف عند الخطأ).
- **معامل `page_template`** — تمت إضافته إلى `create_post` و `update_post`. يُعيّن ملف قالب صفحة PHP (مثل `page-full-width.php`) وقت الإنشاء أو التحديث. مرر سلسلة نصية فارغة إلى `update_post` للعودة إلى القالب الافتراضي للمظهر (theme default).
- **قدرات لقطات الشاشة من جانب العميل (Client-side screenshot abilities)** — تشمل `capture_screenshot`، و`compare_screenshots`، و`review_page_design`. تلتقط لقطات شاشة كاملة أو لقطات عرض الشاشة (viewport) للصفحات الحية عبر متصفح بدون واجهة (headless browser) يعمل من جانب الخادم، وتقارن بين لقطتي شاشة، وتحصل على مراجعة تصميم مُولّدة بواسطة الذكاء الاصطناعي تغطي التخطيط، والطباعة، والألوان، وإمكانية الوصول.
- **خمس وكلاء مدمجين (Five built-in agents)** — كاتب المحتوى (Content Writer)، وباني الموقع (Site Builder)، واستوديو التصميم (Design Studio)، ومدير الإضافات (Plugin Manager)، ومساعد الدعم (Support Assistant). يمتلك كل وكيل مجموعة مخصصة من الأدوات، وموجه نظام (system prompt) مُصمم خصيصًا، واقتراحات بدء التشغيل. يمكن التبديل بينها عبر **محدد الوكيل (Agent Picker)** الجديد في رأس المحادثة. اطلع على [الوكلاء المدمجين](../../user-guide/configuration/built-in-agents).
- **أعلام الميزات (Feature flags)** — علامة تبويب جديدة في **الإعدادات → أعلام الميزات (Settings → Feature Flags)** مع مفاتيح التحكم في الوصول (تقييد على المسؤولين، تقييد على مسؤولي الشبكة، وصول المشتركين، التعطيل لغير الأعضاء) وخيارات العلامة التجارية (إخفاء إسناد التذييل، اسم الوكيل المخصص، إخفاء محدد الوكيل، استخدام أيقونة الموقع كصورة رمزية للدردشة). اطلع على [إعدادات وكيل Gratis AI](../../user-guide/administration/gratis-ai-agent-settings).
- **استعادة آخر جلسة (Restore last session)** — تقوم لوحة الدردشة الآن بإعادة تحميل أحدث محادثة تلقائيًا عند تحميل الصفحة وعند فتح الأداة (widget)، بحيث لا يضيع السياق أبدًا عبر التنقل بين الصفحات.
- **روابط إجراءات الإضافات (Plugin action links)** — تظهر الآن روابط سريعة إلى الإعدادات وسجل القدرات (Abilities Registry) في شاشة **الإضافات → الإضافات المثبتة (Plugins → Installed Plugins)** في ووردبريس، أسفل وصف الإضافة.

### تحسينات {#improved}

- **إعادة محاولة مصدر الصورة (Image source retry)** — يحاول الوكيل الآن إعادة محاولة جميع مصادر الصور المجانية المُعدة قبل العودة إلى صورة مُولّدة بواسطة الذكاء الاصطناعي في حالة فشل التنزيل.
- **لوحة معلومات النموذج (Model info panel)** — مرئية دائمًا في رأس المحادثة؛ ولم تعد مخفية بعد الرسالة الأولى.
- **سلوك التمرير التلقائي (Auto-scroll behaviour)** — يتوقف التمرير التلقائي عندما يقوم المستخدم بالتمرير للأعلى للقراءة؛ ويظهر زر عائم **التمرير إلى الأسفل (Scroll to bottom)** ويختفي تلقائيًا عندما يصل المستخدم إلى أحدث رسالة.
- **واجهة مستخدم محدد الوكيل (Agent Picker UI)** — أُعيد تصميمها لتكون تراكباً على شكل جدول نموذج (form-table overlay) مع أيقونات لكل وكيل، مما يسهل تحديد الوكلاء والتبديل بينهم بنظرة سريعة.
- **أجزاء JavaScript المُحمّلة بكسل (Lazy-loaded JS chunks)** — تم تقسيم حزمة JavaScript الأولية لأداة الدردشة الآن إلى أجزاء مُحمّلة بكسل، مما يقلل أحجام الحزمة الأولية بنسبة 75-90%.
- **إعادة تصميم أداة الدردشة (Chat widget redesign)** — يحل أيقونة الذكاء الاصطناعي الموحدة محل الصورة الرمزية المخصصة السابقة؛ وهي متسقة مع نظام الوكلاء المدمجين.
- **ربط الروابط (URL linkification)** — يتم الآن عرض عناوين URL التي تظهر في رسائل النظام وفقاعات رسائل الخطأ كروابط قابلة للنقر.

### إصلاحات {#fixed}

- **اكتشاف القدرات (Ability discoverability)** — تم تصحيح الأوصاف، ومراجع موجه النظام (system prompt)، ومحاذاة مساحة الاسم (namespace) بحيث تظهر جميع القدرات في قائمة أدوات الوكيل بشكل موثوق.
- **ذاكرة التخزين المؤقت للمزودين (Providers cache)** — يتم الآن تخزين بيانات المزودين مؤقتًا على مستوى الموقع بالكامل عبر عداد إصدار، مما يمنع مشاكل المزودات القديمة في شبكات المواقع المتعددة (multisite networks).
- **`ability_invalid_output`** — تم حلها عبر 12 معالج قدرة (ability handlers)؛ حيث تعيد جميعها استجابات JSON ذات هيكل صحيح.
- **خط أنابيب `pending_client_tool_calls`** — تم توصيله من طرف إلى طرف بحيث تكتمل استدعاءات الأدوات من جانب العميل بشكل صحيح وتعيد النتائج إلى النموذج.
- **درج السجل (History drawer)** — يتم استثناء التغييرات غير القابلة للتراجع من قائمة التراجع؛ وأصبح رابط **عرض السجل الكامل (View full history)** يعمل بشكل صحيح الآن.
- **نظام التغييرات/التراجع (Changes/revert system)** — تم إصلاح خمسة أخطاء منفصلة وتوحيدها تحت واجهة الإدارة الجديدة.
- **إشعار حفظ الإعدادات (Save Settings toast)** — يظهر الآن إشعار شريط التنبيه (snackbar notification) بشكل موثوق بعد النقر على **حفظ الإعدادات (Save Settings)**.
- **قائمة السياق في سلة المهملات (Trash context menu)** — تمت إضافة خيار **الحذف نهائيًا (Delete Permanently)** حتى يمكن حذف العناصر بشكل قاطع دون تركها في عرض سلة المهملات.
- **تحرير وإعادة الإرسال (Edit & resend)** — النقر على أيقونة التحرير يدخل الآن وضع التحرير للرسالة التي تم النقر عليها فقط، وليس جميع الرسائل في الموضوع.
- **ارتفاع تخطيط الدردشة (Chat layout height)** — تتكيف لوحة الدردشة مع ارتفاعها عندما يظهر محتوى مُحقون من الإضافة (إشعارات المسؤول، اللافتات) فوق الصفحة، مما يمنع منطقة الإدخال من الاندفاع خارج الشاشة.

## 1.4.0 — صدر في 2026-04-09 {#140--released-on-2026-04-09}

### جديد {#new-1}

- **أمر قياس الأداء لـ WP-CLI** (`wp gratis-ai-agent benchmark`) — يقوم بتشغيل مجموعة اختبارات قياس قدرات الوكيل (Agent Capabilities v1) من سطر الأوامر لخطوط أنابيب CI وعمليات تقييم النماذج. يدعم التشغيل لكل سؤال، وتجاوزات المزود/النموذج، ومخرجات JSON/CSV.
- **مجموعة اختبارات قياس قدرات الوكيل v1 (Agent Capabilities v1 benchmark suite)** — مجموعة منظمة من الموجهات المعقدة متعددة الخطوات التي تختبر كامل سطح القدرات، مع تسجيل النقاط وإعداد تقارير الرموز/المدة.
- **قدرات أنواع المحتوى المخصصة (Custom Post Type abilities)** — `register_post_type`، `list_post_types`، `delete_post_type`. تستمر التسجيلات عبر عمليات إعادة التشغيل عبر جدول الخيارات (options table).
- **قدرات التصنيفات المخصصة (Custom Taxonomy abilities)** — `register_taxonomy`، `list_taxonomies`، `delete_taxonomy`. يدعم التصنيفات الهرمية والمسطحة مع مسارات إعادة كتابة (rewrite slugs) اختيارية.
- **قدرات نظام التصميم (Design System abilities)** — `inject_custom_css`، `add_block_pattern`، `list_block_patterns`، `set_site_logo`، `apply_theme_json_preset`. يتضمن خمسة إعدادات مسبقة مُنسقة: `minimal-dark`، `warm-editorial`، `corporate-blue`، `vibrant-startup`، `classic-blog`.
- **قدرات الأنماط العامة (Global Styles abilities)** — `get_global_styles`، `set_global_styles`، `reset_global_styles`. يقرأ ويكتب قيم theme.json عبر واجهة برمجة تطبيقات الأنماط العامة في ووردبريس (WordPress Global Styles API).
- **قدرة إدارة قوائم التنقل (Navigation Menu management ability)** — `create_menu`، `update_menu`، `add_menu_item`، `remove_menu_item`، `list_menus`. يدعم القوائم المتداخلة وتعيين مواقع المظهر (theme location).
- **قدرة إدارة الخيارات (Options Management ability)** — `get_option`، `set_option`، `delete_option`، `list_options`. يتضمن قائمة حظر أمان مدمجة تحمي خيارات ووردبريس الحيوية من التعديل.
- **سجل القدرات القابل للتثبيت (Installable Abilities Registry)** — `list_available_abilities`، `install_ability`، `recommend_plugin`. اكتشف وتفعيل حزم القدرات الموزعة كإضافات ووردبريس (WordPress plugins).
- **تنسيق بناء الموقع v2 (Site Builder Orchestration v2)** — `create_site_plan`، `execute_site_plan`، `get_plan_progress`، `handle_plan_error`. خطط بناء الموقع متعددة المراحل مع اكتشاف الإضافات، ومراجع مخرجات الخطوات، وتتبع التقدم، والاسترداد التلقائي للأخطاء.
- **سؤال اختبار موقع المطعم (Restaurant website benchmark question)** (`q-restaurant-website`) — اختبار شامل يغطي تسجيل أنواع المحتوى المخصصة، ونظام التصميم، والتنقل، واكتشاف الإضافات.
- **إضافات موصل الذكاء الاصطناعي للمزودين (AI provider connector plugins)** تمت إضافتها إلى مخططات (blueprints) ملعب ووردبريس (WordPress Playground) لإعداد تطوير محلي أسرع.

### تحسينات {#improved-1}

- تم تحديث ملف README بتوثيق إعدادات موصل الذكاء الاصطناعي للمزودين.

### إصلاحات {#fixed-1}

- تم حل 25 فشل لاختبار PHPUnit في الفرع الرئيسي (main branch).
- تم تصحيح تنسيق رابط إصدارات GitHub في `blueprint.json`.
- إعادة ترقيم معرف المهمة (Task ID) لتجنب التعارض مع المعرفات القديمة.
