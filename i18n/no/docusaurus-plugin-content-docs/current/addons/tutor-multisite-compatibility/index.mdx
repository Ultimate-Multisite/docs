# Tutor Multisite Compatibility

Et WordPress-plugin som legger til kompatibilitetsfikser for Tutor LMS i fler-nettsted-miljøer (multisite).

## Beskrivelse {#description}

Dette pluginet gir essensielle kompatibilitetsfikser for å kjøre Tutor LMS på WordPress multisite-nettverk. Det håndterer brukerpåmelding, instruktørstyring, kursbegrensninger og annen funksjonalitet spesifikk for multisite.

## Funksjoner {#features}

- **Databasehåndtering**: Oppdager og oppretter automatisk manglende Tutor LMS-tabeller som feiler på grunn av multisite-fremmednøkkelbegrensninger
- **Multisite Brukerpåmelding**: Tilpassede AJAX-håndterere for å administrere brukerpåmelding på tvers av nettverkets nettsteder
- **Instruktørrollehåndtering**: Automatisk tildeling og administrasjon av instruktørroller i multisite-miljøer
- **Kursbegrensninger**: Integrasjon med Ultimate Multisite for å administrere begrensninger på kurs- og funnel-opprettelse basert på nettstedsplaner
- **Tilpassede Admin-visninger**: Multisite-bevisst liste og grensesnitt for instruktørstyring
- **Admin-verktøy**: Grensesnitt for databasehåndtering for nettverks- og nettstedsadministratorer

## Krav {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Tutor LMS plugin (påkrevet automatisk via `Requires Plugins` header)
- WordPress Multisite-installasjon
- Ultimate Multisite plugin (for funksjonaliteten knyttet til kursbegrensninger)

## Installasjon {#installation}

1. Last opp pluginfilene til `/wp-content/plugins/tutor-multisite-compatibility/`
2. Forsikre deg om at Tutor LMS er installert og aktivert
3. Aktiver pluginet via 'Plugins'-menyen i WordPress
4. Pluginet vil automatisk integrere seg med Tutor LMS

## Databasehåndtering {#database-management}

### Problemet {#the-problem}

I WordPress multisite-miljøer feiler opprettelsen av Tutor LMS-tabeller for tabeller med fremmednøkkelbegrensninger til brukertabellen. Dette skjer fordi:

- Nettstedsspesifikke tabeller bruker prefikser som `wp_2_tutor_carts`
- Fremmednøklene refererer til `wp_2_users` (som ikke eksisterer)
- Den faktiske brukertabellen er `wp_users` (delt på tvers av hele nettverket)

### Løsningen {#the-solution}

Dette pluginet gjør automatisk følgende:

1. **Oppdager Manglende Tabeller**: Sjekker etter tabeller som feilet å opprettes på grunn av fremmednøkkelbegrensninger
2. **Oppretter Tabeller med Korrekte Referanser**: Bruker det riktige navnet på brukertabellen (`wp_users` i multisite)
3. **Håndterer Flere Scenarier**:
   - Pluginaktivering/reaktivering
   - Opprettelse av nytt nettsted i nettverket
   - Manuell sjekk av tabeller via admin-grensesnittet

### Berørte Tabeller {#affected-tables}

- `tutor_coupon_usages` - Inneholder fremmednøkkel til brukertabellen
- `tutor_carts` - Inneholder fremmednøkkel til brukertabellen
- `tutor_notification_preferences` - TutorPro-tabell med fremmednøkkel til brukertabellen

### Admin-grensesnitt {#admin-interface}

Få tilgang til verktøyene for databasehåndtering på:

- **Nettverksadmin**: Innstillinger → Tutor Multisite DB
- **Nettstedsadmin**: Verktøy → Tutor Multisite DB

Grensesnittet lar deg:
- Se manglende tabeller på tvers av alle nettsteder
- Manuelt utløse opprettelse av tabeller
- Sjekke databasekonfigurasjonen
- Se detaljert statusinformasjon
