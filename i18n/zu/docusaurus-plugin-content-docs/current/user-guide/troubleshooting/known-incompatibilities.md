# Ukungahambisani okwaziwayo namanye ama-Plugins

Ngemva kokufaka nokwenza kusebenze i-Ultimate Multisite, ungase uhlangabezane nezinkinga ngenxa yokungahambisani okungakalungiswa namanye ama-plugins.

Sisebenza ngenkuthalo ukunciphisa lolu hlu kukho konke ukukhishwa okusha, kodwa uma i-plugin ibhalwe lapha, kungase kudingeke uyicime ukuze ube ne-Ultimate Multisite Network esebenza ngokugcwele.

Nalu uhlu lwawo nokuthi yini okufanele uyenze kulezi zimo:

**Amanye ama-plugin e-WPMU DEV** Amanye ama-plugin _angasagcinwa_ yi-WPMU DEV aziwa ngokubangela ukuthi ukugeleza kokubhalisa kwe-Ultimate Multisite kume esinyathelweni sokugcina (umsebenzisi akakwazi ukudala i-account). Nalu uhlu lwama-plugin abangela lezo zinkinga:

  * Signup Language: _I-Ultimate Multisite ine-add-on enikeza ukusebenza okufanayo, ebizwa ngokuthi Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Lokhu kwakhelwe ngaphakathi ku-Ultimate Multisite, ngakho asikho isidingo se-plugin eyengeziwe._

  * New Blog Templates: _Lokhu kwakhelwe ngaphakathi ku-Ultimate Multisite, ngakho asikho isidingo se-plugin eyengeziwe._

  * Set Password on Multisite Blog Creation: _Lokhu kwakhelwe ngaphakathi ku-Ultimate Multisite, ngakho asikho isidingo se-plugin eyengeziwe._

  * Signup TOS: _Lokhu kwakhelwe ngaphakathi ku-Ultimate Multisite, ngakho asikho isidingo se-plugin eyengeziwe._

  * Anti-Splog: Nakuba kunamandla amancane, lokhu nakho _kwakhelwe ngaphakathi ku-Ultimate Multisite._

**Qaphela** : Wonke amanye ama-plugin e-WPMU Dev angasetshenziswa ngokujwayelekile eceleni kwe-Ultimate Multisite. Izibonelo zifaka _Smush_ , _Forminator_ , _Defender,_ njll.
