# Капча үзгәрешләр журналы

Версия: 1.5.0 - 2026-05-22 чыгарылды
* Яңа: Каты туктатучы ешлык чикләгече — captcha белән сакланган өслекләрдәге һәр GET һәм POST соравын саный (wp-login / register / lost-password / comments, WooCommerce my-account / checkout / pay-for-order, Ultimate Multisite checkout / inline-login) һәм HTTP 429, `Retry-After` header, шулай ук очраклы tarpit тоткарлавы (1–5с, каты чик 15с) белән җавап бирә.
* Яңа: tarpit тәрәзәсен көйләү өчен `cap_rate_limit_tarpit_min` / `cap_rate_limit_tarpit_max` көйләүләре.
* Яңа: ышанычлы IP диапазоннарын искәрмә итү өчен `wu_cap_rate_limit_whitelist_ip` фильтры.
* Яңа: каты туктату җавабы җибәрелер алдыннан ук эшли торган `wu_cap_rate_limit_will_block` гамәле.
* Яңа: Алдауга чыдам клиент-IP ачыклау. `Captcha_Core::get_client_ip()` (ешлык чикләү bucket ачкычлары, captcha siteverify `remoteip`, һәм статистика IP хэшлары өчен хакыйкать чыганагы) хәзер катгый ышаныч моделен мәҗбүри итә: REMOTE_ADDR нигез булып тора, `CF-Connecting-IP` бары тик турыдан-туры peer хәзерге Cloudflare IP диапазоны эчендә булганда гына исәпкә алына, һәм `X-Forwarded-For` бары тик турыдан-туры peer admin тарафыннан көйләнгән ышанычлы-proxy исемлегендә булганда гына исәпкә алына; анда килүче IP-ына тукталганчы ышанычлы/CF арадаш күчешләрен калдырып, уңнан сулга йөрү башкарыла.
* Яңа: `cap_trust_cloudflare_headers` көйләве (килешү буенча OFF) — Cloudflare артында булганда `CF-Connecting-IP` ышанычын кушу. Плагин эченә кертелгән Cloudflare CIDR рәсеме белән килә һәм яңарту уңышсыз булса, эчке fallback кулланып, аны атна саен wp-cron аша яңарта.
* Яңа: `cap_trusted_proxies` көйләве — үзегезнең алгы сызыктагы proxy / йөк-тигезләгечләрне күрсәткән CIDRлар яки ялан IPлар өчен textarea (һәр юлга бер, `#` комментарийлары рөхсәт ителә). Моннан башка, ешлык чикләгече кушылган булса да, `X-Forwarded-For` игътибарга алынмый.
* Яңа: Cloudflare / proxy торышын беренче кушканда авто-ачыклау, бер басу белән "Ачыкланган көйләүләрне куллану" admin белдерүе. Плагин сезнең сакланган кыйммәтләрне беркайчан да өстән язмый; әгәр соңрак трафик конфигурациягез инде чынбарлыкка туры килмәвен күрсәтсә (мәсәлән, Cloudflare CIDR диапазоннарын үзгәрткән һәм сезнең proxy CIDR хәзер искергән), кире кагып булмый торган туры килмәү белдерүе тәкъдим ителгән яңартуны күрсәтә.
* Төзәтелде: Күренмәс режим инде `cap_security_level` кыйммәтен тавышсыз FAST дәрәҗәсенә төшерми — admin көйләгән дәрәҗә хөрмәт ителә. Үзенчәлекле логика теләгән сайтлар өчен яңа `wu_cap_server_security_level` фильтры бар.
* Төзәтелде: Статистикадагы `rate_limits_triggered` санагы хәзер сирәк очрый торган уңыштан-соң backstop юлы белән генә түгел, һәр блокта арта.
* Төзәтелде: `Captcha_Core::get_client_ip()` хәзер ешлык чикләгече, captcha тәэмин итүчеләре (reCAPTCHA + hCaptcha `siteverify`) һәм статистика буенча килүче IP-ына нисбәтләү өчен бердәнбер хакыйкать чыганагы — ялган `CF-Connecting-IP` header алып килгән туры origin-server сораулары чын peer урынына ялган IP буенча bucket-ка эләгә торган алдау векторын яба.
* Төзәтелде: WooCommerce классик checkout ешлык-чикләү капкасы хәзер `woocommerce_before_checkout_form` урынына `template_redirect` (priority 1) вакытында эшли. Кәрзин буш булганда форма-дәрәҗә hook беркайчан да эшләми, шуңа продукт өстәми торган flood трафик чикләгечне тулысынча читләтеп үтә иде.
* Төзәтелде: WooCommerce pay-for-order ешлык-чикләү капкасы хәзер `woocommerce_before_pay_action` урынына `template_redirect` вакытында эшли. Соңгысы бары тик `wp_verify_nonce('woocommerce-pay')` уңышлы булгач кына эшли, димәк unauth һөҗүмчеләр (чын куркыныч моделе) чикләгечне беркайчан да кузгатмый иде.
* Төзәтелде: WooCommerce Store API (блоклар) checkout ешлык-чикләү капкасы хәзер `woocommerce_store_api_checkout_update_order_from_request` урынына `rest_pre_dispatch` вакытында эшли. Соңгысы бары тик Store API кәрзинне һәм исәп-хисап кырларын тикшергәннән соң гына эшли, шуңа unauth ботлар тикшерүчедән 400 ала һәм чикләгечне беркайчан да кузгатмый иде.
* Төзәтелде: Ultimate Multisite inline-login ешлык-чикләү капкасы хәзер `wu_before_inline_login` урынына `wu_ajax_nopriv_wu_inline_login` priority 1 (һәм logged-in көзгесе) вакытында эшли. Соңгысы бары тик `check_ajax_referer('wu_checkout')` уңышлы булгач кына эшли, шуңа гамәлдәге wu_checkout nonce булмаган unauth ботлар 403 ала һәм чикләгечне беркайчан да кузгатмый иде.
* Төзәтелде: `Rate_Limiter::enforce()` хәзер `surface|ip` буенча ачкычланган, бер сорауга бер тапкыр эшләү сакчысын куллана, шуңа бер render эчендә ике тапкыр эшли торган upstream hookлар (аеруча Ultimate Multisite эчендәге `wu_setup_checkout`) нәтиҗәле ешлык-чикләү чиген инде яртылаш киметми.
* Төзәтелде: Ешлык-чикләү өслеге капкалары инде `Captcha_Core::is_whitelisted()` (`wu_captcha_whitelisted` фильтры) белән киңәшләшми. Бу фильтр "captcha инде башка өслек тарафыннан эшкәртелгән" дигәнне белдерә һәм flood саклавына бәйсез — WooCommerce интеграциясе аны Woo nonce булганда WordPress login captchaсын үткәреп җибәрү өчен hook итеп куллана иде, бу ешлык санауга үтеп кереп, Woo POSTларын чикләгечтән качырырга мөмкинлек бирде. Хәзер кулланыла торган бердәнбер читләтеп үтү — ешлык-чикләүгә махсус `wu_cap_rate_limit_whitelist_ip` фильтры.
