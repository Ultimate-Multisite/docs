# Captcha

**Ботларны туктатыгыз. Керемне саклагыз. Көйләү мәшәкате юк.**

Махсус доменнар белән multisite челтәре алып барасызмы? Авыртуын беләсез: Google reCAPTCHA һәм hCaptcha үзләренең админ консолендә *һәрбер доменны* теркәүне таләп итә. Яңа клиент сайты өстисезме? Капча көйләнешен яңартыгыз. Махсус домен бәйлисезме? Тагын яңартыгыз. Бу беркайчан да бетми.

**Ultimate Multisite: Captcha барысын да үзгәртә.**

## Cap Captcha: үзеңдә урнаштырылган, үзе эшли торган саклау

Безнең яңа **Cap Captcha** провайдеры multisite челтәрләре өчен уен кагыйдәләрен үзгәртә:

* **Көйләү юк** — өстәмәне активлаштырыгыз, һәм сез сакланган. API ачкычлары юк, Google аккаунты юк, hCaptcha Dashboard юк, доменнарны ак исемлеккә кертү юк.
* **Һәр доменда автоматик эшли** — субдоменнар, бәйләнгән доменнар, клиент доменнары — барысы да шунда ук саклана.
* **Үзеңдә урнаштырылган һәм хосусый** — мәгълүматыгыз серверыгыздан чыкмый. Өченче як күзәтүе булмаган килеш тулысынча GDPR таләпләренә туры килә.
* **Күренми торган Proof-of-Work** — кулланучылар исәпләү табышмагын фонда чишә. Янгын гидрантларына басу да, ачу китерә торган рәсем челтәрләре дә юк.
* **Җиңел** — нибары 20KB. Checkout-ыгызны акрынайтмаячак.

## WooCommerce карталарын сынаудан саклау

Карталарны сынау һөҗүмнәре сәүдәгәрләргә ел саен миллиардларча зыян китерә. Мошенниклар урланган кредит карталарын WooCommerce checkout-ыгыз аша тикшерү өчен ботлар куллана — эшкәртү түләүләрен сез түлисез, алар исә гамәлдәге карта номерларын ала.

**Без иң мөһим урынны саклыйбыз:**

* **Store API саклавы** — гадәти форма капчаларын урап үтә торган `/wp-json/wc/store/v1/checkout` адресындагы бот һөҗүмнәрен блоклый
* **PayPal Payments белән ярашлы** — WooCommerce PayPal Payments белән җайлы эшли
* **Реаль вакыт статистикасы** — ничә һөҗүмне блоклавыгызны төгәл күрегез

## Әле дә Google reCAPTCHA яки hCaptcha кирәкме?

Без сезне кайгырттык. Тулы ярдәм:

* Google reCAPTCHA v2 (Checkbox)
* Google reCAPTCHA v2 (Invisible)
* Google reCAPTCHA v3 (Score-based)
* hCaptcha (Standard)
* hCaptcha (Invisible)

Провайдерлар арасында теләсә кайсы вакытта челтәр көйләүләреннән күчегез.

## Масштаб өчен төзелгән

* Һәр сайт өчен аерым үзгәртүләр белән челтәр күләмендәге көйләүләр
* Саклауны күзәтү өчен статистика Dashboard-ы
* Көйләнә торган куркынычсызлык дәрәҗәләре (Тиз, Уртача, Максималь)
* Киңәйтелә торган провайдер архитектурасы белән эшләүчеләр өчен уңайлы

## Урнаштыру

1. `ultimate-multisite-captcha` ны `/wp-content/plugins/` директориясенә йөкләгез
2. Плагинны челтәр күләмендә активлаштырыгыз
3. **Шул гына. Сез сакланган.**

Cap Captcha автоматик активлаша — көйлисе көйләүләр дә, кертәсе API ачкычлары да юк. Бөтен челтәрегез буенча һәр ярдәм ителгән endpoint сез активлаштырган мизгелдә үк саклана.

**Ихтыярый:** Аның урынына Google reCAPTCHA яки hCaptcha куллану өчен, **Ultimate Multisite → Settings → Captcha** бүлегенә күчегез һәм өстенлекле провайдерыгызны сайлагыз.

## Еш бирелә торган сораулар

## Миңа нәрсә дә булса көйләргә кирәкме?

Юк! Cap Captcha өстәмәне активлаштырганда автоматик кабызыла. Бөтен челтәрегез шунда ук саклана — көйлисе көйләүләр дә, кертәсе API ачкычлары да юк.

## Cap Captcha өчен API ачкычлары кирәкме?

Юк. Cap Captcha тулысынча үзеңдә урнаштырылган. Тышкы аккаунтлар юк, домен теркәү юк, көйләү баш авыртулары юк.

## Бу checkout-ымны акрынайтачакмы?

Cap Captcha нибары 20KB һәм proof-of-work сынауларын фонда башкара. Клиентларыгыз бернинди тоткарлык та сизмәячәк.

## Бу карталарны сынау һөҗүмнәреннән саклыймы?

Әйе! Бары тик HTML формаларын гына саклый торган гадәти капчалардан аермалы буларак, безнең WooCommerce интеграциясе Store API эченә турыдан-туры тоташа һәм бот һөҗүмнәрен API дәрәҗәсендә блоклый — карталарны сынау һөҗүмнәренең күбесе нәкъ шунда була.

## Төрле сайтларда төрле капча провайдерларын куллана аламмы?

Капча провайдеры челтәр күләмендә билгеләнә, ләкин сез ихтыяҗларыгызга карап катлаулылык көйләүләрен үзгәртә аласыз.

## Бу GDPR таләпләренә туры киләме?

Cap Captcha 100% үзеңдә урнаштырылган, тышкы хезмәтләргә мәгълүмат җибәрелми. Google reCAPTCHA һәм hCaptcha өчен, зинһар, аларның тиешле хосусыйлык сәясәтләрен карагыз.

## Rate Limiting

Ultimate Multisite: Captcha v1.5.0 челтәрегезне brute-force һәм карталарны сынау һөҗүмнәреннән саклый торган катгый туктатучы rate limiter кертә.

### Ул ничек эшли

Rate limiter капча белән сакланган өслекләрдә һәр GET һәм POST соравын саный:

* **WordPress керү endpoint-лары** — wp-login, register, lost-password, comments
* **WooCommerce checkout** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

Килүче rate limit-ны арттырганда, плагин түбәндәгечә җавап бирә:

* **HTTP 429 (Too Many Requests)** статус коды
* Кабат кайчан сынап карарга икәнен күрсәтүче **Retry-After** header-ы
* Һөҗүм итүчеләрне акрынайту өчен **очраклы tarpit йоклатуы** (килешү буенча 1–5 секунд, катгый рәвештә 15 секунд белән чикләнгән)

### Көйләү

#### Tarpit вакыты

Rate limit-ка эләккән сорауларга кулланыла торган очраклы тоткарлыкны идарә итегез:

* **`cap_rate_limit_tarpit_min`** — секундларда минималь йокы дәвамлылыгы (килешү буенча: 1)
* **`cap_rate_limit_tarpit_max`** — секундларда максималь йокы дәвамлылыгы (килешү буенча: 5, катгый рәвештә 15 белән чикләнгән)

Бу көйләүләр **Ultimate Multisite → Settings → Captcha** эчендә бар.

### Киңәйтелүчәнлек

#### IP ак исемлек фильтры

Ышанычлы IP диапазоннарын rate limiting-тан азат итегез:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // Whitelist your monitoring service
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### Блоклау алдыннан гамәл

Rate-limit блокы җибәрелгәнче үк махсус логиканы башкарыгыз:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // Log the block, notify admins, or update custom metrics
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## Клиент IP-ен ачыклау һәм ышанычлы проксилар

Ultimate Multisite: Captcha v1.5.0 spoof-ка чыдам клиент-IP ачыклауны кертә. `Captcha_Core::get_client_ip()` методы хәзер килүчеләрнең IP-ен билгеләү өчен бердәнбер хакыйкать чыганагы булып тора:

* Тизлекне чикләү bucket ачкычлары
* Captcha тәэмин итүчесенең `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* Статистика IP хэшлары

### Ышаныч моделе

Плагин катгый IP ышаныч иерархиясен куллана:

1. **REMOTE_ADDR** — Һәрвакыт ышанычлы (турыдан-туры тоташкан якның IP-адресы)
2. **CF-Connecting-IP** — Түбәндәге очракларда гына исәпкә алына:
   - Турыдан-туры тоташкан як хәзерге Cloudflare IP диапазоны эчендә булса
   - `cap_trust_cloudflare_headers` кушылган булса (килешү буенча: СҮНДЕРЕЛГӘН)
3. **X-Forwarded-For** — Түбәндәге очракларда гына исәпкә алына:
   - Турыдан-туры тоташкан як сез admin көйләгән ышанычлы прокси исемлегендә булса
   - Header уңнан сулга таба тикшерелә, ышанычлы/Cloudflare арадаш узеллары үткәреп җибәрелә

### Көйләү

#### Cloudflare Header ышанычы

Cloudflare-ның `CF-Connecting-IP` header-ына ышанычны кабызыгыз:

* **`cap_trust_cloudflare_headers`** — `CF-Connecting-IP` ышанычын куллану өчен ON итеп куегыз
  - Килешү буенча: OFF (сүндерелгән)
  - Плагин эченә кертелгән Cloudflare CIDR снимогы белән килә
  - CIDR диапазоннары атна саен wp-cron аша яңартыла
  - Яңарту уңышсыз булса, эчкә кертелгән резерв кулланыла

#### Ышанычлы прокси көйләве

Алгы сызыктагы проксиларны һәм йөкләнеш баланслагычларын билгеләгез:

* **`cap_trusted_proxies`** — CIDRлар яки гади IPлар өчен текст кыры (һәр юлга бер)
  - Аңлатмалар рөхсәт ителә (`#` белән башланган юллар)
  - Бу көйләү булмаса, тизлек чикләгеч кушылган булса да, `X-Forwarded-For` игътибарга алынмый
  - Мисал:
    ```
    # Our load balancer
    192.0.2.0/24

    # Backup proxy
    198.51.100.50
    ```

#### Беренче тапкыр кабызганда автоматик ачыклау

Тизлек чикләгечен беренче тапкыр кабызганда, плагин сезнең ихтимал Cloudflare һәм прокси торышын ачыклый һәм бер тапкыр басып була торган "Ачыкланган көйләүләрне куллану" admin белдерүен күрсәтә.

* Плагин сакланган кыйммәтләрегезне **беркайчан да өстеннән язмый**
* Әгәр алдагы трафик конфигурациягезнең чынбарлыкка инде туры килмәвен күрсәтсә (мәсәлән, Cloudflare CIDR диапазоннарын үзгәрткән булса), кире кагып булмый торган туры килмәү белдерүе тәкъдим ителгән яңартуны күрсәтә

### Ни өчен бу мөһим

Катгый IP ышаныч моделе мөһим ялганлау юлын яба: ялган `CF-Connecting-IP` header-ы белән турыдан-туры origin-серверга килгән запрослар элек чын тоташкан як урынына ялган IP буенча bucket-ка салына иде. Бу бигрәк тә түбәндәгеләр өчен мөһим:

* **Тизлекне чикләү** — Һөҗүм итүчеләр IPларны ялганлап чикләрне урап уза алмый
* **Статистика** — Сезнең һөҗүм метрикалары ялган түгел, ә чын керүче IPларын күрсәтә
* **Captcha тикшерүе** — Тәэмин итүчеләр хәвефне бәяләү өчен дөрес керүче IPын ала
