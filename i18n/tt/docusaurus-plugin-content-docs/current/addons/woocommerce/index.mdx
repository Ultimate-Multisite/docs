# WooCommerce интеграциясе {#woocommerce-integration}

WooCommerce интеграциясе безнең Manual Gateway эшләгән ысулга бик охшаш итеп эшли. Ул кулланучыларыгызга зур сыгылмалылык бирә, аларга WooCommerce урнаштыруыгызда булган теләсә кайсы түләү формасын сайларга мөмкинлек бирә. Һәм бу гади: кулланучының исәп-хисап чоры тәмамлангач, кулланучы язылуы тоткарлауга куела. Шул ук вакытта челтәрнең төп сайтының WooCommerce урнаштыруында заказ булдырылачак. Аннары кулланучы үз электрон почтасына түләүне башкару өчен сылтама белән инвойс ала. Түләү алынгач, язылу яңартыла.

## Мөмкинлекләр {#features}

- Ultimate Multisite һәм WooCommerce арасында җиңел интеграция
- Берничә түләү ысулы мөмкин
- Беркетелгән язылулар үзәге ярдәмендә автоматлаштырылган кабатланучы исәп-хисап, өстәмә WooCommerce Subscriptions плагины камилләштерүләре белән
- Checkout йомгаклары Ultimate Multisite салымнарын аерым, салым салынмый торган WooCommerce җыем юллары итеп кертә, шуңа күрә клиентлар түләү алдыннан салым суммаларын карый ала

## Документация {#documentation}

**WooCommerce интеграциясен көйләү**

**Ultimate Multisite белән түләү вариантларыгызны киңәйтү: WooCommerce интеграциясе**

## Урнаштыру {#installation}

1. 'ultimate-multisite-woocommerce' ны '/wp-content/plugins/' директориясенә йөкләгез
2. Плагинны WordPress эчендәге 'Plugins' менюсы аша активлаштырыгыз
3. Плагинны көйләү өчен адымлап Wizard күрсәтмәләрен үтәгез

## Куллану кулланмасы {#usage-guide}

_**МӨҺИМ ИСКӘРМӘ: Бу мәкалә Ultimate Multisite 2.x версиясенә карый.**_

_**ИГЪТИБАР:** Ultimate Multisite: WooCommerce Integration өчен WooCommerce кимендә төп сайтыгызда активлаштырылган булырга тиеш._

Без аңлыйбыз: _Stripe_ һәм _PayPal_ кайбер илләрдә мөмкин түгел, бу Ultimate Multisite кулланучыларына безнең плагинны нәтиҗәле куллануны чикли яки кыенлаштыра. Шуңа күрә без **_WooCommerce_ белән интеграцияләү өчен өстәмә** булдырдык.

_WooCommerce_ — бик популяр eCommerce плагины. Дөнья буйлап эшләүчеләр аңа төрле түләү шлюзларын интеграцияләү өчен өстәмәләр булдырды. Без моннан файдаланып, Ultimate Multisite исәп-хисап системасында куллана алган түләү шлюзларын киңәйттек.

## Аны көйләү {#setting-it-up}

Башта WooCommerce Integration өстәмәсен урнаштырырга кирәк, ул **лицензияле кулланучылар өчен бушлай**. Аны урнаштыру өчен, super admin Dashboard эчендә Ultimate Multisite-ка басыгыз һәм **Өстәмәләр**не сайлагыз.

Монда сез барлык Ultimate Multisite өстәмәләрен таба аласыз. **Ultimate Multisite: WooCommerce Integration** өстәмәсенә басыгыз.

![WooCommerce Integration күрсәтелгән өстәмәләр бите](/img/admin/addons-page.png)

Өстәмә мәгълүматлары белән тәрәзә ачылачак. Бары тик **Хәзер урнаштыру** *.*

Урнаштыру тәмамлангач, сез плагиннар битенә юнәлтеләчәксез. Монда бары тик **Челтәрдә активлаштыру**га басыгыз, һәм _WooCommerce_ өстәмәсе челтәрегездә активлаштырылачак.

### Checkout салым җыемы юллары {#checkout-tax-fee-lines}

Ultimate Multisite checkout салымнарны үз эченә алганда, интеграция һәр Ultimate Multisite салым суммасын WooCommerce checkout-ка аерым, салым салынмый торган җыем юлы итеп өсти. Бу түләү алдыннан салым йомгагын күренеп торырлык итә һәм шул ук вакытта WooCommerce-ның салым җыемына кабат салым салуын булдырмый.

## Түләүне ничек идарә итәргә {#how-to-manage-the-payment}

WooCommerce интеграциясе безнең **Manual Gateway** эшләгән ысулга бик охшаш итеп эшли. Кимчелеге — һәр түләү исәп-хисап чоры ахырында кулдан түләнергә тиеш. Агым ничек эшли:

  * Кулланучының исәп-хисап циклы тәмамлангач, әгъзалык **тоткарлауга** куела. Шул ук вакытта челтәрегезнең төп сайтындагы _WooCommerce_ урнаштыруында заказ булдырылачак.

  * Аннары кулланучылар үз электрон почтасына түләүне башкару өчен сылтама белән инвойс ала (сылтама шулай ук аларның subsite Dashboard-ындагы исәп-хисап тарихында да күренә). Бу сылтама кулланучыны _**WooCommerce**_ **checkout** битенә юнәлтә, анда кулланучылар түләүләрен эшкәртү өчен **мөмкин булган _WooCommerce_ шлюзларының** берсен сайлый алачак.

  * Түләү алынгач, әгъзалык **яңартыла** , түләү Ultimate Multisite-та теркәлә һәм заказ тәмамланган итеп билгеләнә.

### Кайбер мөһим искәрмәләр: {#some-important-notes}

  * Ultimate Multisite-ның стандарт функциональлеге бу интеграция формасы белән дә эшли. Мәсәлән, **Subscription Management** экраныннан **кире түләүләр чыгару** сайланган түләү формасын кулланып кире түләүне чыгару өчен WooCommerce белән элемтәгә керәчәк.

  * Ultimate Multisite: WooCommerce Integration эшләү өчен WP-Cron-га таяна. Бу күпчелек очракта бик яхшы эшли, ләкин сайтта трафик аз булса, cron планлаштырылган вакыйгаларны тикшерү өчен эшләтеп җибәрелми һәм вакыйга үткәреп җибәрелә, бу башкару тоткарлыкларына китерергә мөмкин.

## WooCommerce Subscriptions плагины {#woocommerce-subscriptions-plugin}

Һәр ай саен түләүне кулдан башлау ахыргы кулланучылар өчен уңайлы түгел, чөнки язылулар һәр хисаплау циклы саен автомат рәвештә кабатланыр дип көтелә. WooCommerce Integration өстәмәсе хәзер төп язылулар кодын үз эченә ала, шуңа күрә кабатланучы продуктлар кабатланучы түләү өчен һәрвакыт язылуларны куллана.

Бу интеграция аша кабатланучы продуктлар сату өчен аерым [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) сатып алу яки урнаштыру **кирәк түгел**. Әгәр ул плагин сатып алынган, урнаштырылган һәм төп сайтыгызда мөмкин булса, Ultimate Multisite аны кулланачак, ә WooCommerce өстәмә язылу белән идарә итү мөмкинлекләрен күрсәтәчәк. Ул мөмкин булмаганда, эчкә кертелгән язылулар үзәге барыбер кабатланучы продуктларны эшкәртә.

![WooCommerce язылу интеграциясе көйләүләре](/img/addons/woocommerce-subscription-integration.png)

Һәм шуның белән шул. Сезнең ахыргы кулланучыларга кабатланучы продуктлар өчен хисаплау циклы ахырында түләүне кулдан башкарырга кирәк булмаячак.

Төп сайтыгызда _WooCommerce_ астында заказ булдырылудан тыш, ул һәр Account өчен язылу да булдырачак.

### Яңарту уңышсыз булганда туктатып тору тәртибе {#failed-renewal-suspension-behavior}

Килешү буенча, WooCommerce Subscriptions уңышсыз түләүне кабатлау җәдвәлен эшкәрткәндә, әгъзалыклар актив булып кала. Бу гадәти WooCommerce Subscriptions ташламалы вакыт тәрәзәсен саклый, шуңа күрә уңышлы торгызылган яңарту керү мөмкинлеген вакытыннан алда өзмичә язылуны һәм әгъзалыкны яңадан синхронлаштыра ала.

Керүне катгыйрак контрольдә тотарга теләгән администраторлар WooCommerce Integration көйләүләрендә **Яңартулар уңышсыз булганда әгъзалыкларны шунда ук туктатып тору** параметрын кабыза ала. Бу ихтыяри көйләү кабызылганда, Ultimate Multisite яңарту түләве уңышсыз булган мизгелдә, WooCommerce Subscriptions кабатлап эшкәртүенең тәмамлануын көтмичә, бәйле әгъзалыкны туктатып тора.

Эшләүчеләр шул ук карарны `wu_woo_suspend_on_payment_failure` фильтры белән үзгәртә ала. Hook белешмә битләре өстәмә чыганагыннан булдырыла, шуңа күрә бу күзәтү махсус туктатып тору сәясәтләре өчен фильтр исемен генә күрсәтә.
