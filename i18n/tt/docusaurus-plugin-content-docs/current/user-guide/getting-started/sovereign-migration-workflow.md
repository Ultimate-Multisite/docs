# Суверен Миграция Рабочий Процесс

Ultimate Multisite: Multi-Tenancy 1.2.0 добавляет ворота проверки миграции для переноса стандартного субсайта в инфраструктуру суверенного арендатора (tenant).

## Прежде чем начать

Убедитесь, что у вас есть:

- Текущая резервная копия исходного сайта.
- Добавленный плагин Multi-Tenancy активен.
- Целевая база данных арендатора, корневая файловая система и домен готовы.
- Привязка хоста базы данных, соответствующая целевой среде.
- Доступ для выполнения команд WP-CLI для сети.

## Рекомендуемый рабочий процесс

1. Подготовьте базу данных и файловую систему целевого арендатора.
2. Зарегистрируйте или обновите настройки изоляции арендаторов (tenant isolation settings).
3. Выполните миграцию арендатора.
4. Разверните или проверьте пользователей арендатора.
5. Очистите очередь асинхронных миграционных задач (async migration jobs).
6. Запустите проверку суверенной миграции (sovereign migration verification).
7. Посетите арендатора с SSO.
8. Переключите DNS или маршрутизацию только после успешного прохождения проверки.

## Ворота проверки

Рабочий процесс проверки проверяет миграцию с нескольких сторон:

- Схема арендатора существует и может быть записана писателем базы данных арендатора.
- Настроенный хост базы данных имеет действительные права доступа (grants).
- Пользователи арендатора присутствуют и соответствуют ожидаемому количеству пользователей установки суверенного окружения.
- Очередь асинхронной отправки (async push queue) может быть успешно очищена.
- Устаревшие пути данных на стороне сети больше не требуются для суверенного арендатора.

Рассматривайте сбои проверки как блокирующие факторы перед запуском. Исправьте сообщаемую проблему с базой данных, пользователем, очередью или маршрутизацией, а затем снова запустите проверку, прежде чем открывать арендатора для клиентов.

## Первый посещение в продакшене

После успешного прохождения проверки используйте **Visit (SSO)** из экрана управления сайтом для первого административного посещения. Это подтверждает маршрутизацию арендатора, обработку токенов SSO, привязку к источнику и настройку пользователей на стороне арендатора за один контролируемый шаг.
