# Клиентский отзыв и отчетность об ошибках

Gratis AI Agent v1.5.0-da built-in (встроенная) система обратной связи и отчетности о проблемах, которая позволяет конечным пользователям помечать неинформативные ответы и сообщать о проблемах прямо из чата. Эта система включает управление согласием, автоматическую команду для отчетности и помощь ИИ при сортировке проблем на бэкенде.

## Кнопка "Не нравится" (Thumbs-Down Button)

Каждое сообщение, отправленное ассистентом ИИ, отображает кнопку **«Не нравится»** (👎). Когда пользователь нажимает на нее, он может пометить ответ как неинформативный или неправильный.

- Кнопка появляется при наведении курсора рядом с каждым сообщением ассистента.
- При нажатии открывается модальное окно **"Согласие на обратную связь" (Feedback Consent Modal)**.
- Обратная связь связывается с текущей беседой и конкретным сообщением.

## Модальное окно "Согласие на обратную связь" (Feedback Consent Modal)

Когда пользователь нажимает кнопку «Не нравится», перед отправкой каких-либо данных появляется модальное окно согласия. Это окно:

- Объясняет, какую информацию будут собирать (фрагмент беседы, контекст браузера).
- Просит пользователя подтвердить свое согласие на обмен этими данными.
- Предоставляет поле для свободного ввода, где пользователь может описать, что пошло не так.
- Предлагает опцию **"Отмена" (Cancel)**, чтобы закрыть окно без отправки данных.

Обратная связь записывается только после явного подтверждения пользователем.

## Баннер с автоматическим запросом обратной связи (Auto-Prompt Feedback Banner)

В конце беседы ассистент может отобразить **баннер с автоматическим запросом обратной связи** — ненавязчивое сообщение, спрашивающее, была ли сессия полезной.

Этот баннер появляется автоматически в зависимости от длины беседы и определенных правил (heuristics). Он ведет к тому же процессу получения обратной связи, что и кнопка «Не нравится». Пользователи могут закрыть этот баннер, не предоставляя отзыв.

## Команда /report-issue

Пользователи могут инициировать процесс получения обратной связи напрямую, набрав `/report-issue` в поле ввода чата. Эта команда:

- Обратная связь на модальное окно Согласия открывается немедленно.
- Поле описания предварительно заполняется контекстом текущего диалога.
- Пользователям разрешается добавить дополнительную информацию перед отправкой.

Команда `/report-issue` доступна во всех режимах чата (встроенный, плавающий виджет, полноэкранный).

## AI-Assisted Triage (Сортировка с помощью ИИ)

Отправленная обратная связь направляется на уровень сортировки с помощью ИИ, который:

- Категоризирует отчет (фактическая ошибка, бесполезный ответ, неподобающий контент и т. д.).
- Прикрепляет соответствующую метаинформацию из диалога.
- Пересылает сводку по сортировке на настроенную **Feedback Endpoint** (см. [Settings > Advanced](#settings-advanced)).

Это сокращает время ручного просмотра, выявляя самые критичные проблемы в первую очередь.

## Settings > Advanced {#settings-advanced} (Настройки > Дополнительно)

Чтобы включить бэкенд обратной связи, настройте следующие поля в разделе **Gratis AI Agent → Settings → Advanced** (Бесплатный ИИ Агент → Настройки → Дополнительно):

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** (URL конечной точки обратной связи) | URL, который принимает POST-запросы с полезной нагрузкой обратной связи (JSON). |
| **Feedback API Key** (API ключ обратной связи) | Токен Bearer, отправляемый в заголовке `Authorization` для аутентификации отправки отзывов. |

Оставьте оба поля пустыми, чтобы отключить сбор обратной связи. Индивидуальные кнопки обратной связи остаются видимыми для пользователей, но отправленные данные не будут перенаправлены.

:::tip
Конечная точка обратной связи должна принимать JSON-тело с полями как минимум `message_id`, `conversation_id`, `feedback_text` и `triage_category`. Посмотрите документацию вашего провайдера конечной точки для ожидаемой схемы.
