# Gratis AI Agent көйләүләре {#gratis-ai-agent-settings}

Gratis AI Agent эчендәге **Settings → Advanced** экраны backend интеграцияләре өчен администратор дәрәҗәсендәге конфигурация бирә. Бу бит фикерне юнәлтүне, эзләү провайдеры ачкычларын, идарә ителә торган Superdav сервисын көйләүне, Google Calendar идарәләрен, TextBee SMS көйләүләрен һәм челтәр буенча функция флагларын документлаштыра.

## Көйләүләргә керү {#accessing-settings}

1. WordPress админ өлешендә **Gratis AI Agent → Settings** бүлегенә керегез.
2. **Advanced** вкладкасына басыгыз.

## Фикер endpoint конфигурациясе {#feedback-endpoint-configuration}

Фикер endpoint-ы AI agent-тан POST сорауларын кулланучы бармак-аска төймәсе, auto-prompt баннеры яки `/report-issue` командасы аша фикер җибәргән саен кабул итә.

| Кыр | Тасвирлама |
|---|---|
| **Feedback Endpoint URL** | Фикер җибәрүләрен JSON гәүдәсе белән HTTP POST сораулары буларак кабул итә торган URL. |
| **Feedback API Key** | Һәр фикер соравының `Authorization` header-ында җибәрелә торган bearer token. Әгәр endpoint-ыгыз аутентификация таләп итмәсә, буш калдырыгыз. |

### Көтелгән JSON йөкләмәсе {#expected-json-payload}

Сезнең фикер endpoint-ыгыз кимендә түбәндәге кырлар булган JSON гәүдәсен кабул итәргә тиеш:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Сөйләшү контекстына карап, йөкләмәдә өстәмә кырлар булырга мөмкин.

### `triage_category` кыйммәтләре {#triagecategory-values}

AI triage катламы йөкләмәне алга җибәргәнче `triage_category` өчен түбәндәге кыйммәтләрнең берсен билгели:

| Кыйммәт | Мәгънә |
|---|---|
| `factual_error` | Ярдәмче дөрес булмаган фактик мәгълүмат бирде. |
| `unhelpful_answer` | Җавап техник яктан дөрес иде, ләкин файдалы түгел. |
| `inappropriate_content` | Җавапта кулланучыларга күрсәтелергә тиеш булмаган эчтәлек бар иде. |
| `other` | Фикер билгеле категориягә туры килмәде. |

### Аутентификация {#authentication}

Әгәр endpoint-ыгыз аутентификация таләп итсә, **Feedback API Key** кырын bearer token-ыгызга куегыз. Agent җибәрә:

```
Authorization: Bearer <your-api-key>
```

Әгәр **Feedback API Key** кыры буш булса, `Authorization` header җибәрелми.

### Фикер җыюны сүндерү {#disabling-feedback-collection}

**Feedback Endpoint URL** һәм **Feedback API Key** кырларының икесен дә буш калдырыгыз. Бармак-аска төймәсе һәм фикер UI кулланучыларга күренә булып кала, ләкин җибәрүләр бернинди тышкы сервиска да юнәлтелми.

## Brave Search API Key {#brave-search-api-key}

Шулай ук **Advanced** вкладкасында **Brave Search API Key** кыры [Интернет эзләү](../configuration/internet-search) мөмкинлеген кабыза.

| Кыр | Тасвирлама |
|---|---|
| **Brave Search API Key** | Brave Search эшләүче Dashboard-ынан API ачкычыгыз. AI ярдәмчедә интернет эзләүне кабызу өчен кирәк. |

Кыр ярлыгында Brave Search API теркәлү битенә басыла торган сылтама бар. Интернет эзләүне сүндерү өчен буш калдырыгыз.

Бу функция буенча ахыргы кулланучы документлары өчен [Интернет эзләү](../configuration/internet-search) карагыз.

## Идарә ителә торган Superdav сервисы {#managed-superdav-service}

Superdav AI Agent v1.18.0 хупланган сайтлар өчен идарә ителә торган Superdav сервис endpoint-ларын һәм автоматик тоташу тәэмин итүне өсти. Сайтыгыз кул белән конфигурацияләнгән сервис endpoint урынына хостингтагы провайдерга тоташырга тиеш булганда, бу идарәләрне кулланыгыз.

| Кыр | Тасвирлама |
|---|---|
| **Managed Superdav Service** | Хупланган сайтлар өчен хостингтагы Superdav сервис тоташуын кабыза. |
| **Provision Connection** | Автоматик endpoint һәм credential тәэмин итүне башлый. Сайт идарә ителә торган провайдерны кулланырга тиешлеген раслаганнан соң моны кулланыгыз. |
| **Service Endpoint / Connection Status** | Тәэмин итүдән соң хәзерге endpoint-ны яки тоташу халәтен күрсәтә. |

Тәэмин итүдән соң, идарә ителә торган сервис эш агымнарына таянганчы көйләүләрне саклагыз һәм тоташу халәтен тикшерегез. Әгәр тәэмин итү уңышсыз булса, күрсәтелгән кабатлау күрсәтмәләрен карагыз һәм сайтның хостингтагы провайдерны куллану рөхсәте барлыгын раслагыз.

## Google Calendar конфигурациясе {#google-calendar-configuration}

Superdav AI Agent v1.18.0 календарь функцияләре кабызылганда, agent конфигурацияләнгән календарьларны һәм вакыйга детальләрен укый ала. Календарь кораллары укуга юнәлтелгән һәм расписаниене исәпкә алган искәртүләр, катнашучылар белән алга таба элемтә һәм контактларны туры китерү өчен файдалы.

| Кыр | Тасвирлама |
|---|---|
| **Google Calendar Credentials** | Календарь мәгълүматын уку өчен кирәкле credential-ларны яки token тоташуын саклый. |
| **Calendar Selection** | Agent тикшерә ала торган конфигурацияләнгән календарьларны чикли. |
| **Calendar Connection Status** | Хәзерге credential-лар календарьларны һәм вакыйгаларны укый аламы-юкмы икәнен раслый. |

Календарь credential-ларын agent-ка кирәк булган календарьлар белән генә чикләгез. Халәт token вакыты чыкканын күрсәтсә, credential-ларны яңадан тоташтырыгыз яки алыштырыгыз.

## TextBee SMS белдерүләре {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 TextBee-ны конфигурацияләнгән белдерү эш агымнары өчен SMS провайдеры буларак өсти. SMS белдерүләр сизгер яки кулланучыларга юнәлтелгән хәбәрләр өчен кеше раслау капкалары белән парлаштырылырга тиеш.

| Кыр | Тасвирлама |
|---|---|
| **TextBee API Key** | TextBee SMS провайдерына сорауларны аутентификацияли. |
| **TextBee Device / Sender** | Провайдер таләп иткәндә, чыга торган хәбәрләр өчен кулланыла торган TextBee җибәрүчесен яки җайланмасын сайлый. |
| **SMS Notifications Enabled** | Расланган эш агымнарына текст-хәбәр белдерүләрен җибәрергә рөхсәт итә. SMS җибәрүләрен булдырмау өчен сүндерелгән килеш калдырыгыз. |

Тест хәбәрен бары администраторга караган номерга гына җибәрегез, аннары планлаштырылган яки катнашучыларга юнәлтелгән искәртүләрне кабызганчы раслау капкасы тәртибен раслагыз.

## Функция флаглары {#feature-flags}

Шулай ук v1.9.0 версиясендә кертелгән **Settings → Feature Flags** вкладкасы өстәмә функциональлек өчен күчерү төймәләре бирә. Һәр флаг челтәр буенча йә кабызылган, йә сүндерелгән; хәзерге вакытта аерым сайт өчен өстен кую юк.

### Функция флагларына керү {#accessing-feature-flags}

1. WordPress админ өлешендә **Gratis AI Agent → Settings** бүлегенә керегез.
2. **Feature Flags** вкладкасына басыгыз.

### Керү контроле флаглары {#access-control-flags}

| Флаг | Килешенгәнчә | Тасвирлама |
|---|---|---|
| **Администраторлар белән чикләү** | Сүнек | Кабызылганда, AI Agent чат панелен бары тик `administrator` ролендәге кулланучылар гына ача ала. Башка барлык рольләр моның урынына "Администраторыгыз белән элемтәгә керегез" хәбәрен күрә. |
| **Network Admins белән чикләү** | Сүнек | Multisite челтәрендә кабызылганда, агентны бары тик Super Admins гына куллана ала. Аерым сайт администраторлары тыела. Икесе дә кабызылган булса, "Администраторлар белән чикләү" көйләвеннән өстенрәк була. |
| **Subscriber керүен рөхсәт итү** | Сүнек | Кабызылганда, `subscriber` ролендәге кулланучылар чат интерфейсын куллана ала, әмма бары тик уку гына мөмкинлекләре белән чикләнә (язма булдыру яки көйләүләрне үзгәртү юк). |
| **Әгъза булмаганнар өчен сүндерү** | Сүнек | Ultimate Multisite әгъзалык статусы белән интеграцияләнә. Кабызылганда, актив әгъзалыгы булмаган сайтлар өчен чат яшерелә. |

### Брендинг флаглары {#branding-flags}

| Флаг | Килешенгәнчә | Тасвирлама |
|---|---|---|
| **"Powered by Gratis AI Agent" аскы өлешен яшерү** | Сүнек | Чат виджетының аскы өлешендә күрсәтелгән брендинг атрибуциясе юлын бетерә. White-label урнаштырулар өчен тәкъдим ителә. |
| **Үз Agent исеме** | *(буш)* | Чат башлыгындагы һәм админ менюсындагы килешенгән "Gratis AI Agent" тамгасын үз продукт исемегез белән алыштыра. Килешенгәнне куллану өчен буш калдырыгыз. |
| **Agent сайлагычын яшерү** | Сүнек | Кабызылганда, кулланучылар биш эчке агент арасында күчә алмый. Агымдагы агент Settings → General бүлегендә килешенгән итеп көйләнгән агентка беркетелә. |
| **Сайт иконасын чат аватары итеп куллану** | Сүнек | Чат виджеты башлыгындагы килешенгән AI иконасын WordPress сайт иконасы белән алыштыра (Appearance → Customize → Site Identity астында куела). |

### Автоматлаштыру иминлеге флаглары {#automation-safety-flags}

Superdav AI Agent v1.18.0 хәвефсезрәк автоматлаштыру эшләтеп җибәрүләре өчен кеше раславы капкаларын һәм искәртү язмаларын кертә. Бу контрольләр, урнаштырылган пакетка карап, функция флагларында яки алга киткән автоматлаштыру көйләүләрендә күренергә мөмкин.

| Флаг | Килешенгәнчә | Тасвирлама |
|---|---|---|
| **Кеше раславын таләп итү** | Кабызылган булуы тәкъдим ителә | Вәкаләтле кулланучы тәкъдим ителгән гамәлне карап раслаганчы, сизгер автоматлаштыруларны туктатып тора. |
| **Искәртүләрне кабатланудан саклау** | Кабызылган | Кабат омтылышлар яки расписание буенча эшләтеп җибәрүләр кабат хәбәрнамәләр җибәрмәсен өчен, җибәрелгән искәртүләрне терки. |
| **Календарь коралларын кабызу** | Көйләнгәнче сүнек | Агентка көйләнгән Google календарьларын һәм вакыйгаларын укырга рөхсәт итә. |
| **SMS хәбәрнамәләрен кабызу** | Көйләнгәнче сүнек | Расланган эш агымнарына, учет мәгълүматлары сакланганнан соң, TextBee SMS хәбәрнамәләрен җибәрергә рөхсәт итә. |

### Үзгәрешләрне куллану {#applying-changes}

Теләсә кайсы флагны күчергәннән соң **Көйләүләрне саклау** төймәсенә басыгыз. Үзгәрешләр шунда ук көченә керә — кэшны чистарту яки pluginны яңадан активлаштыру таләп ителми.
