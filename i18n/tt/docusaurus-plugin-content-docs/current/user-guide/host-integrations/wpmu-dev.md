# WPMU DEV Интеграция

## Обзор {#overview}
WPMU DEV — это комплексная платформа для WordPress, которая предлагает хостинг, плагины и услуги для сайтов на WordPress. Эта интеграция позволяет автоматически синхронизировать домены и управлять SSL-сертификатами между Ultimate Multisite и хостингом WPMU DEV.

## Возможности {#features}
- Автоматическая синхронизация доменов
- Управление SSL-сертификатами
- Расширенные попытки проверки SSL-сертификатов

## Требования {#requirements}
Интеграция автоматически определяет, размещаете ли вы сайт на WPMU DEV, и использует встроенный API. Если вы размещаете сайт на WPMU DEV, дополнительная настройка не требуется.

Интеграция проверяет наличие константы `WPMUDEV_HOSTING_SITE_ID`, которая автоматически определяется при размещении на WPMU DEV.

## Инструкции по настройке {#setup-instructions}

### 1. Проверка хостинга на WPMU DEV {#1-verify-wpmu-dev-hosting}

Если вы размещаете сайт на WPMU DEV, необходимые константы уже должны быть определены. Убедитесь, что:

1. Константа `WPMUDEV_HOSTING_SITE_ID` определена в вашей среде
2. У вас есть активная подписка на WPMU DEV с доступом к API

### 2. Включение интеграции {#2-enable-the-integration}

1. В админке WordPress перейдите в Ultimate Multisite > Settings (Настройки)
2. Перейдите на вкладку "Domain Mapping" (Сопоставление доменов)
3. Прокрутите вниз до раздела "Host Integrations" (Интеграции хостинга)
4. Включите интеграцию WPMU DEV
5. Нажмите "Save Changes" (Сохранить изменения)

## Как это работает {#how-it-works}

### Синхронизация доменов {#domain-syncing}

Когда домен сопоставляется в Ultimate Multisite:

1. Интеграция использует API WPMU DEV для добавления домена в ваш аккаунт хостинга
2. Она также автоматически добавляет версию с www домена
3. WPMU DEV занимается настройкой домена и выдачей SSL-сертификатов

### Управление SSL-сертификатами {#ssl-certificate-management}

Интеграция настроена так, чтобы увеличить количество попыток проверки SSL-сертификатов для хостинга WPMU DEV, так как выдача и установка SSL-сертификатов может занять некоторое время. По умолчанию она будет пытаться до 10 раз проверить SSL-сертификат, в сравнении со стандартными 5 попытками.

## Важные замечания {#important-notes}

### Удаление домена {#domain-removal}

В настоящее время API WPMU DEV не предоставляет способа удаления доменов. Когда привязка домена удаляется в Ultimate Multisite, этот домен останется в вашем аккаунте хостинга WPMU DEV. Если это необходимо, вам нужно будет удалить его вручную из панели управления хостингом WPMU DEV.

### Аутентификация API {#api-authentication}

Интеграция использует ключ API WPMU DEV, который хранится в вашей базе данных WordPress как опция `wpmudev_apikey`. Это настраивается автоматически при подключении вашего сайта к WPMU DEV.

## Устранение неполадок {#troubleshooting}

### Проблемы с подключением к API {#api-connection-issues}
- Проверьте, что ваш сайт правильно подключен к WPMU DEV
- Проверьте, что опция `wpmudev_apikey` установлена в вашей базе данных WordPress
- Убедитесь, что ваша подписка на WPMU DEV активна

### Проблемы с SSL-сертификатами {#ssl-certificate-issues}
- WPMU DEV может занять некоторое время для выдачи SSL-сертификатов (обычно 5-15 минут)
- Интеграция настроена так, чтобы проверять SSL-сертификаты до 10 раз
- Если после нескольких попыток SSL-сертификаты все еще не выдаются, свяжитесь со службой поддержки WPMU DEV

### Домен не добавлен {#domain-not-added}
- Проверьте логи Ultimate Multisite на наличие сообщений об ошибках
- Убедитесь, что домен еще не добавлен в WPMU DEV
- Убедитесь, что ваш тариф хостинга WPMU DEV поддерживает количество добавляемых вами доменов
