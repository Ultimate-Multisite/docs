# Headerда домен эзләү

Бу көйләнешне домен эзләүне башлый торган кечкенә header формасы кирәк булганда кулланыгыз; аннары клиент Ultimate Multisite түләү рәсмиләштерүе эчендә мөмкин булган доменнардан сайлый ала.

## Таләпләр {#requirements}

- Ultimate Multisite челтәр буенча активлаштырылган.
- Multisite Ultimate Domain Seller челтәр буенча активлаштырылган.
- Кимендә бер актив домен теркәү продукты булырга тиеш:
  - `domain_provider` көйләнгән провайдерга куелган.
  - Хуплана торган TLDлар көйләнгән, мәсәлән `com`, `net` һәм `org`.
- **Домен сайлау** кырын үз эченә алган гамәлдәге түләү рәсмиләштерү формасы.

## Түләү рәсмиләштерү формасы {#checkout-form}

1. Теркәү бите куллана торган түләү рәсмиләштерү формасын булдырыгыз яки үзгәртегез.
2. Гадәти мәҗбүри түләү рәсмиләштерү/Account кырларын, шул исәптән **Кулланучы исеме**н, өстәгез. Бары тик домен кырын гына үз эченә алган түләү рәсмиләштерү формасы Ultimate Multisite тикшерүе тарафыннан кире кагыла.
3. **Домен сайлау** кырын өстәгез.
4. Агым бушлай субдоменнарга яки булган доменнарга түгел, ә теркәлә торган доменнарга юнәлергә тиеш булганда, Домен сайлау режимын **Тик теркәү** итеп куегыз.
5. Домен теркәү продуктын шул кырга билгеләгез.

Теркәү бите түләү рәсмиләштерү формасын күрсәтергә тиеш, мәсәлән:

```text
[wu_checkout slug="domain-form"]
```

## Header формасы {#header-form}

Сайт headerына кертелгән эзләү сүзен түләү рәсмиләштерү битенә `domain_name` буларак җибәрә торган кечкенә `GET` формасын өстәгез:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

Махсус header JavaScriptында доменны алдан сайламагыз. Header бары тик эзләү сүзен генә тапшырырга тиеш. Domain Seller түләү рәсмиләштерү скрипты `?domain_name=example` укый, түләү рәсмиләштерү эзләү тартмасын тутыра һәм мөмкинлекне эзләүне эшләтеп җибәрә, шулай итеп клиент кайтарылган доменнардан сайлый ала.

## Көтелгән тәртип {#expected-behaviour}

Headerда `example` эзләү түбәндәгене ачарга тиеш:

```text
/register/?domain_name=example
```

Аннары түләү рәсмиләштерү сайлап була торган нәтиҗәләрне күрсәтергә тиеш, мәсәлән:

- `example.com`
- `example.net`
- `example.org` мөмкин түгел
- провайдер хуплый торган башка TLDлар

Мөмкин булган нәтиҗәне сайлаганнан соң, заказ йомгагында домен теркәү продукты һәм сайланган домен исеме булырга тиеш.

## Тикшерү {#verification}

1. Баш битне ачыгыз.
2. Ялангач исем эзләгез, мәсәлән `example`.
3. Түләү рәсмиләштерү URLында `?domain_name=example` булуын раслагыз.
4. Түләү рәсмиләштерү домен кырында `example` булуын раслагыз.
5. Домен сайлау исемлеге күренүен раслагыз.
6. Мөмкин булган домен өчен **Сайлау**га басыгыз.
7. Заказ йомгагында `Domain Registration - example.com` яки сайланган домен булуын раслагыз.

## Кыенлыкларны чишү {#troubleshooting}

- Әгәр исемлек күренмәсә, браузерның челтәр кыстыргычында `admin-ajax.php?action=wu_domain_search` тикшерегез һәм аның буш булмаган `domains` яки `results` кайтаруын раслагыз.
- Әгәр түләү рәсмиләштерү формасы саклаганда тикшерүне үтә алмаса, **Кулланучы исеме** кебек мәҗбүри Account кырларын өстәгез.
- Әгәр домен сайлау кәрзинне яңартмаса, `window.wu_checkout_form` булуын һәм Domain Seller түләү рәсмиләштерү ресурслары түләү рәсмиләштерү битендә йөкләнгән булуын раслагыз.
