# Доменные продукты и ценообразование {#domain-products-and-pricing}

Доменные продукты — это то, как вы контролируете ценообразование, домены верхнего уровня (TLD), конфиденциальность WHOIS и выбор провайдера. Каждый доменный продукт является стандартным продуктом Ultimate Multisite с дополнительной вкладкой **Domain Settings** (Настройки домена).

## Создание доменного продукта {#creating-a-domain-product}

1. Перейдите в **Network Admin › Ultimate Multisite › Products** (Администрирование сети › Ultimate Multisite › Продукты).
2. Нажмите **Add New** (Добавить новый).
3. Установите тип продукта как **Domain** (Домен).
4. Настройте вкладку **Domain Settings** (Настройки домена) (см. ниже).
5. Сохраните.

Доменные продукты отображаются с фиолетовым значком **Domain** (Домен) в списке продуктов и могут фильтроваться с помощью вкладки **Domain Products** (Доменные продукты).

## Вкладка Domain settings (Настройки домена) {#domain-settings-tab}

### Провайдер {#provider}

Выберите, какой регистратор будет обрабатывать регистрацию для этого продукта. По умолчанию используется глобальный **Default provider** (По умолчанию провайдер), установленный в настройках продавца доменов.

### Поддерживаемые TLD {#supported-tlds}

Оставьте поле пустым, чтобы создать **catch-all product** (продукт "на все"), который применяется ко всем TLD, не совпавшим с другим продуктом.

Введите список TLD через запятую (например, `.com, .net, .org`), чтобы создать **TLD-specific product** (специфичный продукт для TLD), который применяется только к этим расширениям.

**Как работает сопоставление продуктов:** Когда клиент ищет домен, аддон выбирает наиболее подходящий продукт. Продукт с `.com` в списке TLD имеет приоритет над продуктом "на все". Если ни один специфичный продукт для TLD не совпадает, используется общий продукт ("catch-all"). Если продуктов нет, поиск по домену не отображается.

### Тип наценки (Markup type) {#markup-type}

Три режима управляют тем, как ваша розничная цена рассчитывается от оптовой стоимости:

| Режим | Как это работает |
|---|---|
| **Percentage** (Процентная) | Добавляет процент к оптовой стоимости. Наценка в 20% на домен с оптовой стоимостью $10 даст $12. |
| **Fixed markup** (Фиксированная наценка) | Добавляет фиксированную сумму в долларах. Наценка в $5 на домен стоимостью $10 даст $15. |
| **Fixed price** (Фиксированная цена) | Полностью игнорирует оптовую стоимость. Всегда взимается сумма, которую вы вводите. |

### Вводная цена (Introductory pricing) {#introductory-pricing}

### Скидка на первый год {#whois-privacy}

Возможность предложить цену со скидкой за первый год. Установите отдельную **Вводную цену** (цену за первый год) рядом с обычной ценой **Повторного продления** (цена за второй год и далее). Клиент видит обе цены во время оформления заказа, чтобы знать, чего ожидать при продлении.

### Конфиденциальность WHOIS {#tld-import-and-sync}

Контролирует, предлагается ли защита конфиденциальности WHOIS для доменов, зарегистрированных через этот продукт.

| Настройка | Поведение |
|---|---|
| **Отключено** | Защита WHOIS никогда не предлагается и не включается. |
| **Всегда включено** | Защита WHOIS автоматически включается при регистрации без какой-либо платы. |
| **Выбор клиента** | Во время оформления заказа появляется флажок. Установите **Цену конфиденциальности** для взимания платы за год или оставьте $0, чтобы предложить ее бесплатно. |

Для Namecheap защита WHOIS использует WhoisGuard (всегда бесплатно). Для OpenSRS используется услуга конфиденциальности OpenSRS (может иметь стоимость при оптовой продаже).

## Импорт и синхронизация TLD {#auto-renewal}

Продукты доменов отображают оптовые цены в режиме реального времени, полученные от подключенного провайдера. Чтобы это работало, необходимо импортировать TLD.

- **Ручная синхронизация:** Настройки › Продавец домена › Синхронизировать TLD (для каждого провайдера)
- **Автоматическая синхронизация:** Выполняется ежедневно через запланированную задачу cron для всех настроенных провайдеров.

После синхронизации перейдите на вкладку Настройки домена любого продукта и используйте селектор TLD, чтобы увидеть доступные TLD с их текущими оптовыми ценами.

## Автопродление {#admin-manual-domain-registration}

Продление доменов привязано к статусу членства клиента:

- Когда членство продлевается и домен связан, продление домена автоматически ставится в очередь
- Попытки продления используют активный платежный шлюз клиента
- Неудачные продления повторяются автоматически с экспоненциальной задержкой (exponential backoff)
- Отправляются электронные уведомления об успешных продлениях, неудачах и предстоящем истечении срока действия

Идентификаторы шаблонов электронной почты для событий жизненного цикла домена:

| Событие | Шаблон ID |
|---|---|
| Домен зарегистрирован | `domain_registered` |
| Домен продлен | `domain_renewed` |
| Продление не удалось | `domain_renewal_failed` |
| Домен скоро истечет | `domain_expiring_soon` |

## Администратор: Ручная регистрация домена

Клиент через оформление заказа без прохождения его через него зарегистрировать домен можно так:

1. Перейдите в **Network Admin › Ultimate Multisite › Register Domain** (Администрирование сети › Ultimate Multisite › Зарегистрировать домен)
2. Выберите клиента и введите имя домена
3. Заполните контактные данные регистратора (имя, адрес, телефон)
4. Нажмите **Register** (Зарегистрировать)

Запись о домене будет создана и привязана к учетной записи клиента.
