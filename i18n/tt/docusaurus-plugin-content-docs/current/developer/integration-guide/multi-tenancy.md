# Multi-Tenancy интеграциясе

Ultimate Multisite: Multi-Tenancy 1.2.0 суверен tenantлар, миграцияне тикшерү һәм tenant тормыш циклы автоматлаштыруы өчен берничә интеграция ноктасын үзгәртә.

## Tenant башлангыч көйләү агымы {#tenant-bootstrap-flow}

Tenantларны булдыручы яки үзгәртүче интеграцияләр түбәндәге тәртипне үтәргә тиеш:

1. Tenant реестры язмасын һәм изоляция моделен билгеләгез.
2. Tenant мәгълүмат базасы язучысын булдырыгыз яки тикшерегез.
3. Tenant схемасын башлангыч көйләгез.
4. Tenant кулланучыларын тәэмин итегез.
5. Tenant маршрутлаштыруын һәм файл системасы юлларын теркәгез.
6. Tenantны ачык иткәнче миграция тикшерүен эшләтегез.

Суверен tenant челтәр мәгълүмат базасы тоташуын кабат куллана ала дип фараз итмәгез. Addon тарафыннан бирелгән tenant реестрын һәм язучы абстракцияләрен кулланыгыз.

## SSO һәм REST элгечләре {#sso-and-rest-hooks}

Хәлсез tenant autologin кыска гомерле токеннарны purpose claim, JTI кабат кулланудан саклау, гамәлдән чыгу чиге һәм origin беркетү белән куллана. Керү төймәләре яки ерактан идарә итү сылтамалары өстәүче интеграцияләр tenant керү URLларын турыдан-туры төзү урынына, хупланган SSO агымы аша tenant керүләрен булдырырга тиеш.

Челтәр ягындагы API аудит вакыйгалары һәм көндәлек йомгаклар суверен tenant gatewayлары өчен бар. Tenant тормыш циклы endpointларына мөрәҗәгать итүче тышкы системаларны төзәткәндә шул журналларны кулланыгыз.

## Суверен клиент гамәл URLлары {#sovereign-customer-action-urls}

Ultimate Multisite v2.13.0 суверен-tenant клиент гамәлләрен Account, checkout, исәп-хисап, invoice, сайт, шаблон алыштыру һәм domain-mapping агымнары өчен төп сайтка кире маршрутлый. Tenant ягында идарә итү сылтамаларын чыгара торган интеграцияләр бу гамәлләрне төп сайттагы клиент панеленә юнәлтергә һәм кулланучы гамәлне тәмамлаганнан соң tenantка кире кайта алырга тиеш булганда тикшерелгән кайту максатын кертергә тиеш.

Доменнар аша идарә итү сылтамалары өчен төп SSO төргечен кулланыгыз:

```php
$url = wu_with_sso($main_site_customer_url);
```

Булдырылган URL `wu_sso_url` аша фильтрлана ала булып кала; ул SSO URLын, хәзерге кулланучыны, максат сайт IDсын һәм юнәлтү контекстын ала. Add-onнар бу фильтрны провайдерга хас контекст өстәү яки Ultimate Multisite токен тикшерүен саклаган хәлдә broker URLын алыштыру өчен куллана ала.

Суверен tenant эчендә әгъзалык, invoice, billing-address, шаблон яки домен-идарә итү халәтен кабатламагыз. Tenant Dashboardын эшләтеп җибәрүче итеп, ә төп сайт клиент панелен идарә ителә торган гамәлләр өчен төп язма системасы итеп карагыз.

## Миграцияне тикшерү {#migration-verification}

Миграция яки тормыш циклы интеграциясе tenant мәгълүматларын үзгәрткәннән соң, тикшерү капкаларын эшләтегез:

- `wp tenant verify-no-legacy --site=<site-id>` tenantның инде иске челтәр ягы юлларына бәйле булмавын раслый.
- `wp tenant verify-sovereign-push --site=<site-id>` суверен push эшләре эшкәртә һәм бушата алуын раслый.

Интеграцияләр уңышсыз тикшерүне урнаштыруны туктатучы киртә итеп карарга һәм хата хәл ителгәнче tenantны live дип билгеләүдән сакланырга тиеш.

## Tenant бетерү {#tenant-deletion}

Бетерү агымнары addon teardown юлын чакырырга тиеш, шуның белән tenant мәгълүмат базасы credentialлары чистартыла. Тышкы интеграцияләр teardown уңышлы тәмамланганнан соң провайдер ресурсларын бетерә ала, ләкин тикшерү яки async push эшләре әле эшләгәндә host мәгълүмат базаларын яки папкаларны бетерергә тиеш түгел.

## Искергән мәгълүмат базасы роутеры {#deprecated-database-router}

Иске `Database_Router` deprecation stub белән алыштырылды. Яңа интеграцияләр иске router classка таяну урынына tenantларны хәзерге сайт routerы һәм tenant registry APIлары аша билгеләргә тиеш.
