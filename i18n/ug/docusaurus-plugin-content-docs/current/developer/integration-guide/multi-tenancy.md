# كۆپ-ئىجارىدارلىق بىرلەشتۈرۈش {#multi-tenancy-integration}

Ultimate Multisite: كۆپ-ئىجارىدارلىق 1.2.0 مۇستەقىل ئىجارىدارلار، كۆچۈرۈشنى دەلىللەش ۋە ئىجارىدار ھايات دەۋرىنى ئاپتوماتلاشتۇرۇش ئۈچۈن بىر قانچە بىرلەشتۈرۈش ئۇچرىشىش نۇقتىسىنى ئۆزگەرتىدۇ.

## ئىجارىدارنى دەسلەپلەشتۈرۈش ئېقىمى {#tenant-bootstrap-flow}

ئىجارىدارلارنى قۇرىدىغان ياكى ئۆزگەرتىدىغان بىرلەشتۈرۈشلەر بۇ تەرتىپكە ئەمەل قىلىشى كېرەك:

1. ئىجارىدار تىزىملاش خاتىرىسى ۋە ئايرىش مودېلىنى ھەل قىلىڭ.
2. ئىجارىدار ساندان يازغۇچىسىنى قۇرۇڭ ياكى دەلىللەڭ.
3. ئىجارىدار سىخېمىسىنى دەسلەپلەشتۈرۈڭ.
4. ئىجارىدار ئىشلەتكۈچىلىرىنى تەمىنلەڭ.
5. ئىجارىدار يۆنىلىشلىرى ۋە ھۆججەت سىستېمىسى يوللىرىنى تىزىملىتىڭ.
6. ئىجارىدارنى ئاشكارىلاشتىن بۇرۇن كۆچۈرۈشنى دەلىللەشنى ئىجرا قىلىڭ.

مۇستەقىل ئىجارىدار تور ساندان ئۇلىنىشىنى قايتا ئىشلىتەلەيدۇ دەپ پەرەز قىلماڭ. قوشۇلما تەمىنلىگەن ئىجارىدار تىزىملىكى ۋە يازغۇچى ئابستراكسىيەلىرىنى ئىشلىتىڭ.

## SSO ۋە REST ئىلمەكلىرى {#sso-and-rest-hooks}

ھالەتسىز ئىجارىدار ئاپتوماتىك كىرىشى قىسقا مۇددەتلىك tokenلارنى ئىشلىتىدۇ، ئۇلاردا مەقسەت تەلىپى، JTI قايتا ئىشلىتىشتىن قوغداش، ۋاقتى توشۇش چەكلىمىسى ۋە مەنبەگە باغلاش بار. كىرىش كۇنۇپكىلىرى ياكى يىراقتىن باشقۇرۇش ئۇلانمىلىرىنى قوشىدىغان بىرلەشتۈرۈشلەر ئىجارىدار كىرىش URLلىرىنى بىۋاسىتە قۇراشنىڭ ئورنىغا، قوللايدىغان SSO ئېقىمى ئارقىلىق ئىجارىدار زىيارەتلىرىنى ھاسىل قىلىشى كېرەك.

تور تەرەپتىكى API تەكشۈرۈش ھادىسىلىرى ۋە كۈندىلىك خۇلاسىلەر مۇستەقىل ئىجارىدار دەرۋازىلىرى ئۈچۈن تەمىنلىنىدۇ. ئىجارىدار ھايات دەۋرى endpointلىرىنى چاقىرىدىغان سىرتقى سىستېمىلارنى سازلىغاندا بۇ خاتىرىلەرنى ئىشلىتىڭ.

## مۇستەقىل خېرىدار مەشغۇلات URLلىرى {#sovereign-customer-action-urls}

Ultimate Multisite v2.13.0 مۇستەقىل-ئىجارىدار خېرىدار مەشغۇلاتلىرىنى Account، checkout، ھېسابات چىقىمى، invoice، site، قېلىپ ئالماشتۇرۇش ۋە دائىرە خەرىتىلەش ئېقىملىرى ئۈچۈن ئاساسىي siteقا قايتا يۆنىلىشلەيدۇ. ئىجارىدار تەرەپتىكى باشقۇرۇش ئۇلانمىلىرىنى كۆرسىتىدىغان بىرلەشتۈرۈشلەر بۇ مەشغۇلاتلارنى ئاساسىي-site خېرىدار تاختىسىغا كۆرسىتىشى ۋە ئىشلەتكۈچى مەشغۇلاتنى تاماملىغاندىن كېيىن ئىجارىدارغا قايتا يۆتكىلىشى كېرەك بولغاندا دەلىللەنگەن قايتىش نىشانىنى ئۆز ئىچىگە ئېلىشى كېرەك.

دائىرە ھالقىغان باشقۇرۇش ئۇلانمىلىرى ئۈچۈن يادرولۇق SSO ئورغۇچىسىنى ئىشلىتىڭ:

```php
$url = wu_with_sso($main_site_customer_url);
```

ھاسىل قىلىنغان URL يەنىلا `wu_sso_url` ئارقىلىق filterلىنالايدۇ؛ ئۇ SSO URL، نۆۋەتتىكى ئىشلەتكۈچى، نىشان site ID ۋە قايتا يۆنىلىشلەش مەزمۇنىنى قوبۇل قىلىدۇ. قوشۇلمىلار بۇ filterنى ئىشلىتىپ تەمىنلىگۈچىگە خاس مەزمۇننى قوشالايدۇ ياكى Ultimate Multisiteنىڭ token دەلىللەشىنى ساقلاپ قېلىپ ۋاسىتىچى URLنى ئالماشتۇرالايدۇ.

ئەزالىق، invoice، ھېسابات-ئادرېسى، قېلىپ ياكى دائىرە-باشقۇرۇش ھالىتىنى مۇستەقىل ئىجارىدار ئىچىدە كۆپەيتمەڭ. ئىجارىدار Dashboardىنى قوزغاتقۇچ، ئاساسىي-site خېرىدار تاختىسىنى بولسا باشقۇرۇلىدىغان مەشغۇلاتلارنىڭ سىستېما خاتىرە مەنبەسى دەپ قاراڭ.

## كۆچۈرۈشنى دەلىللەش {#migration-verification}

كۆچۈرۈش ياكى ھايات دەۋرى بىرلەشتۈرۈشى ئىجارىدار سانلىق مەلۇماتلىرىنى ئۆزگەرتكەندىن كېيىن، دەلىللەش دەرۋازىلىرىنى ئىجرا قىلىڭ:

- `wp tenant verify-no-legacy --site=<site-id>` ئىجارىدارنىڭ ئەمدى كونا تور-تەرەپ يوللىرىغا تايانمايدىغانلىقىنى جەزملەشتۈرىدۇ.
- `wp tenant verify-sovereign-push --site=<site-id>` مۇستەقىل push ۋەزىپىلىرىنىڭ بىر تەرەپ قىلىپ قۇرۇتالايدىغانلىقىنى جەزملەشتۈرىدۇ.

بىرلەشتۈرۈشلەر دەلىللەش مەغلۇپ بولسا ئۇنى تارقىتىش توسقۇچى دەپ قارىشى ۋە خاتالىق ھەل قىلىنمىغۇچە ئىجارىدارنى live دەپ بەلگىلەشتىن ساقلىنىشى كېرەك.

## ئىجارىدارنى ئۆچۈرۈش {#tenant-deletion}

ئۆچۈرۈش ئېقىملىرى قوشۇلما teardown يولىنى چاقىرىشى كېرەك، شۇنداق بولغاندا ئىجارىدار ساندان كىملىك ئۇچۇرلىرى تازىلىنىدۇ. سىرتقى بىرلەشتۈرۈشلەر teardown مۇۋەپپەقىيەتلىك بولغاندىن كېيىن تەمىنلىگۈچى مەنبەلىرىنى ئۆچۈرەلەيدۇ، ئەمما دەلىللەش ياكى async push ۋەزىپىلىرى تېخى ئىجرا بولۇۋاتقاندا host ساندانلىرى ياكى قىسقۇچلارنى ئۆچۈرمەسلىكى كېرەك.

## ۋاقتى ئۆتكەن ساندان يۆنىلەندۈرگۈچى {#deprecated-database-router}

كونا `Database_Router` ۋاقتى ئۆتكەنلىك stubى بىلەن ئالماشتۇرۇلدى. يېڭى بىرلەشتۈرۈشلەر كونا يۆنىلەندۈرگۈچى classقا تايىنىشنىڭ ئورنىغا، نۆۋەتتىكى site يۆنىلەندۈرگۈچىسى ۋە ئىجارىدار تىزىملىك APIلىرى ئارقىلىق ئىجارىدارلارنى ھەل قىلىشى كېرەك.
