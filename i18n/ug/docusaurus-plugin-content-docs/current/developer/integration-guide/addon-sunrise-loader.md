# قوشۇمچە دېتال Sunrise ھۆججەت يۈكلىگۈچىسى

Ultimate Multisite 2.8.0 قوشۇمچە دېتاللار ۋە خاس MU-plugin بىرلەشتۈرۈشلىرى ئۈچۈن sunrise كېڭەيتىلمە يۈكلىگۈچىسى قوشىدۇ؛ بۇلار WordPress sunrise دەسلەپكى قوزغىتىلىشى جەريانىدا، ھاسىل قىلىنغان `wp-content/sunrise.php` ھۆججىتىنى تەھرىرلىمەي ئىجرا قىلىنىشى كېرەك.

## قاچان ئىشلىتىش كېرەك

بىرلەشتۈرۈشىڭىز ئادەتتىكى قىستۇرمىلار يۈكلىنىشتىن بۇرۇن ئىجرا قىلىنىشى زۆرۈر بولغاندا، مەسىلەن خاس دائىرە يۆنىلىش بەلگىلەش، host-قا خاس تەلەپ بىر تەرەپ قىلىش ياكى بالدۇر تور دەسلەپكى قوزغىتىش تەڭشەكلىرى ئۈچۈن sunrise كېڭەيتىلمىسىنى ئىشلىتىڭ.

ئادەتتىكى بىرلەشتۈرۈشلەر ئۈچۈن ئادەتتىكى WordPress قىستۇرمىلىرى، MU-plugins ۋە ھۆججەتلەشتۈرۈلگەن Ultimate Multisite ئىلمەكلىرىنى ئەۋزەل كۆرۈڭ. Sunrise كودى ناھايىتى بالدۇر ئىجرا قىلىنىدۇ، شۇڭا كىچىك، مۇداپىئەچان ۋە بېقىندىلىقسىز بولۇشى كېرەك.

## ھۆججەت نام بېرىش قائىدىسى

نامى `ultimate-multisite-` بىلەن باشلىنىدىغان قوشۇمچە دېتال مۇندەرىجىسىدە `sunrise.php` ناملىق PHP ھۆججەت قۇرۇڭ:

```text
wp-content/plugins/ultimate-multisite-example-addon/sunrise.php
```

يۈكلىگۈچى قىستۇرمىلار مۇندەرىجىسىنى بۇ ئەندىزە بويىچە تەكشۈرىدۇ:

```text
wp-content/plugins/ultimate-multisite-*/sunrise.php
```

ماس كېلىدىغان ھۆججەتلەر قوشۇمچە دېتال يولى بويىچە ئېلىپبە تەرتىپىدە يۈكلىنىدۇ.

## ھۆججەتنى قەيەرگە قويۇش كېرەك

ھۆججەتنى sunrise ھەرىكىتىگە ئىگە بولغان قوشۇمچە دېتالنىڭ يىلتىز مۇندەرىجىسىگە قويۇڭ:

```text
wp-content/
└── plugins/
    └── ultimate-multisite-example-addon/
        ├── ultimate-multisite-example-addon.php
        └── sunrise.php
```

تەكشۈرۈش ھازىرقى `WP_PLUGIN_DIR` قىممىتىگە ئەمەس، `WP_CONTENT_DIR` غا نىسبەتەن ھەل قىلىنىدۇ. بۇ كۆپ ئىجارىلىك ياكى باشقا بالدۇر دەسلەپكى قوزغىتىش كودى sunrise جەريانىدا قىستۇرما مۇندەرىجە تۇراقلىقلىرىنى ئۆزگەرتكەندە بايقاشنى مۇقىم ساقلايدۇ.

ھاسىل قىلىنغان `wp-content/sunrise.php` ھۆججىتىنى بىۋاسىتە تەھرىرلىمەڭ. يۈكلىگۈچى Ultimate Multisite ئورنىتىدىغان ۋە يېڭىلايدىغان يادرولۇق sunrise ھۆججىتىنى ئايرىماي تۇرۇپ، خاس كودنىڭ sunrise ھەرىكىتىنى كېڭەيتىشىگە يول قويىدۇ.

## ئىشلەتكىلى بولىدىغان ئىلمەكلەر ۋە سۈزگۈچلەر

قوشۇمچە دېتال sunrise ھۆججەتلىرى Ultimate Multisite دائىرە خەرىتىلەشنى يۈكلىگەندىن كېيىن ۋە WordPress `ms_loaded` نى قوزغىتىشتىن بۇرۇن ئىجرا قىلىنىدۇ. بۇ نۇقتىدا sunrise ھۆججىتى تۆۋەندىكىلەرنى قىلالايدۇ:

- read or override `$current_site` and `$current_blog`;
- access `$wpdb` after database configuration has loaded;
- زۆرۈر بولغاندا `BLOG_ID_CURRENT_SITE` قاتارلىق sunrise-ۋاقتى تۇراقلىقلىرىنى ئېنىقلاش؛
- كۆپ ئىجارىلىك بىرلەشتۈرۈشلەردە ئىشلىتىلىدىغان يۆنىلىش ھالىتىنى ئۆز ئىچىگە ئالغان Ultimate Multisite sunrise ياردەمچى ھالىتىنى ئوقۇش.

Ultimate Multisite ئۆزىنىڭ sunrise يۈكلىگۈچىسى تاماملانغاندىن كېيىن `wu_sunrise_loaded` نى قوزغىتىدۇ. Sunrise دەسلەپكى قوزغىتىشى تاماملانغاندىن كېيىن ئىجرا قىلىنىشى كېرەك، ئەمما يەنىلا sunrise ھايات دەۋرىگە تەۋە بولغان كود ئۈچۈن شۇ action نى ئىشلىتىڭ.

پەقەت sunrise باسقۇچىدا ئاللىبۇرۇن يۈكلەنگەن فۇنكسىيەلەرنىلا چاقىرىڭ. ساندانغا ئېغىر خىزمەت، قېلىپ كۆرسىتىش، HTTP تەلەپلىرى ۋە ئادەتتىكى قىستۇرما يۈكلىنىش تەرتىپى تاماملاندى دەپ پەرەز قىلىدىغان كودتىن ساقلىنىڭ.

## ئەڭ كىچىك مىسال

```php
<?php
/**
 * Sunrise extension for a custom host integration.
 */

if (! defined('ABSPATH')) {
    exit;
}

if (isset($current_blog) && $current_blog instanceof stdClass) {
    // Adjust early routing or constants before ms_loaded.
}
```

ھۆججەتنى ئورۇنلاشتۇرغاندىن كېيىن، خەرىتىلەنگەن دائىرە ۋە خەرىتىلەنمىگەن ئاساسىي تور بېكەت URL نى يۈكلەپ، ھەر ئىككى يولنىڭ يەنىلا توغرا دەسلەپكى قوزغىتىلىدىغانلىقىنى جەزملەشتۈرۈڭ.
