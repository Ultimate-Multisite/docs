# باش قىسىمدىكى دومېن ئىزدەش

Ultimate Multisite ھېسابات قىلىش جەريانى ئىچىدە خېرىدارنىڭ بار دومېنلاردىن تاللىشىغا يول قويىدىغان، دومېن ئىزدەشنى باشلايدىغان كىچىك باش قىسىم جەدۋىلى لازىم بولغاندا بۇ تەڭشەكنى ئىشلىتىڭ.

## تەلەپلەر {#requirements}

- Ultimate Multisite توردا ئاكتىپ بولۇشى كېرەك.
- Multisite Ultimate Domain Seller توردا ئاكتىپ بولۇشى كېرەك.
- تۆۋەندىكىلەرگە ئىگە ئاز دېگەندە بىر ئاكتىپ دومېن تىزىملاش مەھسۇلاتى بولۇشى كېرەك:
  - `domain_provider` تەڭشەلگەن تەمىنلىگۈچىگە بەلگىلەنگەن بولۇشى كېرەك.
  - قوللايدىغان TLD لار تەڭشەلگەن بولۇشى كېرەك، مەسىلەن `com`، `net` ۋە `org`.
- **دومېن تاللاش** بۆلىكىنى ئۆز ئىچىگە ئالغان ئىناۋەتلىك ھېسابات قىلىش جەدۋىلى.

## ھېسابات قىلىش جەدۋىلى {#checkout-form}

1. تىزىملىتىش بېتى ئىشلىتىدىغان ھېسابات قىلىش جەدۋىلىنى قۇرۇڭ ياكى تەھرىرلەڭ.
2. **ئىشلەتكۈچى نامى** نىمۇ ئۆز ئىچىگە ئالغان ئادەتتىكى زۆرۈر ھېسابات قىلىش/Account بۆلەكلىرىنى قوشۇڭ. پەقەت دومېن بۆلىكىلا بار ھېسابات قىلىش جەدۋىلى Ultimate Multisite دەلىللەش تەرىپىدىن رەت قىلىنىدۇ.
3. **دومېن تاللاش** بۆلىكىنى قوشۇڭ.
4. ئېقىم ئەركىن تارماق دومېنلار ياكى مەۋجۇت دومېنلارغا ئەمەس، تىزىملىتىلغان دومېنلارغا مەركەزلەشكەندە، دومېن تاللاش ھالىتىنى **پەقەت تىزىملاش** قىلىپ بەلگىلەڭ.
5. دومېن تىزىملاش مەھسۇلاتىنى شۇ بۆلەككە تەقسىملەڭ.

تىزىملىتىش بېتى ھېسابات قىلىش جەدۋىلىنى كۆرسىتىشى كېرەك، مەسىلەن:

```text
[wu_checkout slug="domain-form"]
```

## باش قىسىم جەدۋىلى {#header-form}

تور بېكەتنىڭ باش قىسمىغا كىرگۈزۈلگەن ئىزدەش ئاتالغۇسىنى ھېسابات قىلىش بېتىگە `domain_name` سۈپىتىدە ئەۋەتىدىغان كىچىك `GET` جەدۋىلى قوشۇڭ:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

ئىختىيارىي باش قىسىم JavaScript ئىچىدە دومېننى ئالدىن تاللىماڭ. باش قىسىم پەقەت ئىزدەش ئاتالغۇسىنىلا ئۆتكۈزۈشى كېرەك. Domain Seller ھېسابات قىلىش قوليازمىسى `?domain_name=example` نى ئوقۇيدۇ، ھېسابات قىلىش ئىزدەش قۇتىسىنى تولدۇرىدۇ ۋە بار-يوقلۇقىنى ئىزدەشنى ئىجرا قىلىدۇ، شۇنىڭ بىلەن خېرىدار قايتۇرۇلغان دومېنلاردىن تاللىيالايدۇ.

## كۈتۈلگەن ھەرىكەت {#expected-behaviour}

باش قىسمىدا `example` نى ئىزدىگەندە تۆۋەندىكىنى ئېچىشى كېرەك:

```text
/register/?domain_name=example
```

ئاندىن ھېسابات قىلىش تۆۋەندىكىدەك تاللىغىلى بولىدىغان نەتىجىلەرنى كۆرسىتىشى كېرەك:

- `example.com`
- `example.net`
- `example.org` ئىشلەتكىلى بولمايدۇ
- تەمىنلىگۈچى قوللايدىغان باشقا TLD لار

ئىشلەتكىلى بولىدىغان نەتىجە تاللانغاندىن كېيىن، زاكاز خۇلاسىسى دومېن تىزىملاش مەھسۇلاتى ۋە تاللانغان دومېن نامىنى ئۆز ئىچىگە ئېلىشى كېرەك.

## دەلىللەش {#verification}

1. باش بەتنى ئېچىڭ.
2. يالىڭاچ نامنى ئىزدەڭ، مەسىلەن `example`.
3. ھېسابات قىلىش URL ئىچىدە `?domain_name=example` بارلىقىنى جەزملەشتۈرۈڭ.
4. ھېسابات قىلىش دومېن بۆلىكىدە `example` بارلىقىنى جەزملەشتۈرۈڭ.
5. دومېن تاللاش تىزىملىكىنىڭ كۆرۈنگەنلىكىنى جەزملەشتۈرۈڭ.
6. ئىشلەتكىلى بولىدىغان دومېن ئۈچۈن **تاللاش** نى چېكىڭ.
7. زاكاز خۇلاسىسىدە `Domain Registration - example.com` ياكى تاللانغان دومېن بارلىقىنى جەزملەشتۈرۈڭ.

## مەسىلىلەرنى ھەل قىلىش {#troubleshooting}

- ئەگەر تىزىملىك كۆرۈنمىسە، توركۆرگۈچنىڭ تور بەتكۈچىدىن `admin-ajax.php?action=wu_domain_search` نى تەكشۈرۈپ، ئۇنىڭ بوش بولمىغان `domains` ياكى `results` قايتۇرىدىغانلىقىنى جەزملەشتۈرۈڭ.
- ئەگەر ھېسابات قىلىش جەدۋىلى ساقلاش جەريانىدا دەلىللەشتىن ئۆتەلمىسە، **ئىشلەتكۈچى نامى** قاتارلىق زۆرۈر Account بۆلەكلىرىنى قوشۇڭ.
- ئەگەر دومېن تاللاش ھارۋىنى يېڭىلىمىسا، `window.wu_checkout_form` نىڭ مەۋجۇتلۇقىنى ۋە Domain Seller ھېسابات قىلىش مەنبەلىرىنىڭ ھېسابات قىلىش بېتىدە يۈكلەنگەنلىكىنى جەزملەشتۈرۈڭ.
