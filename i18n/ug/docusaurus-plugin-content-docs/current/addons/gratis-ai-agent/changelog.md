# ئۆزگىرىشلەر خاتىرىسى

## 1.9.0 — 2026-04-28 كۈنى ئېلان قىلىندى {#190--released-on-2026-04-28}

### يېڭى {#new}

- **`create_contact_form` ئىقتىدارى** — ئاكتىپ فورما pluginى (Contact Form 7، WPForms، Fluent Forms ياكى Gravity Forms) ئارقىلىق ئالاقە فورمىسى قۇرىدۇ ۋە ھەرقانداق يازما ياكى بەتكە قىستۇرۇشقا تەييار shortcode قايتۇرىدۇ.
- **`set_featured_image` ئىقتىدارى** — مەۋجۇت Media Library قوشۇمچە ھۆججەت IDسى ياكى يىراقتىكى URLدىن يازما ياكى بەتكە تاللانغان سۈرەت بەلگىلەيدۇ؛ URL تەمىنلەنگەندە سۈرەتنى ئاپتوماتىك ئىمپورت قىلىدۇ.
- **`batch_create_posts` ئىقتىدارى** — بىرلا ئىقتىدار چاقىرىقىدا كۆپ يازما قۇرىدۇ. `create_post` بىلەن ئوخشاش پارامېتىرلارنى قوللايدۇ، ھەر بىر يازمىنىڭ مۇۋەپپەقىيەت/مەغلۇبىيىتىنى دوكلات قىلىدۇ، ھەمدە ئىختىيارىي `stop_on_error` ھالىتى بار.
- **`page_template` پارامېتىرى** — `create_post` ۋە `update_post` غا قوشۇلدى. قۇرۇش ياكى يېڭىلاش ۋاقتىدا PHP بەت قېلىپ ھۆججىتىنى (مەسىلەن `page-full-width.php`) بەلگىلەيدۇ. themeنىڭ كۆڭۈلدىكى قىممىتىگە قايتۇرۇش ئۈچۈن `update_post` قا بوش ھەرپ تىزىقى يوللاڭ.
- **خېرىدار تەرەپ screenshot ئىقتىدارلىرى** — `capture_screenshot`، `compare_screenshots` ۋە `review_page_design`. مۇلازىمېتىر تەرەپتىكى headless توركۆرگۈ ئارقىلىق جانلىق بەتلەرنىڭ تولۇق ياكى viewport screenshotلىرىنى تۇتىدۇ، ئىككى screenshotنى سېلىشتۇرىدۇ، ھەمدە ئورۇنلاشتۇرۇش، خەت نۇسخىسى، رەڭ ۋە قولايلىقلىقنى ئۆز ئىچىگە ئالغان AI ھاسىل قىلغان لايىھە باھالاشنى ئالىدۇ.
- **بەش ئىچكى ۋاكالەتچى** — مەزمۇن يازغۇچى، بېكەت قۇرغۇچى، لايىھە ستۇدىيىسى، Plugin باشقۇرغۇچ ۋە قوللاش ياردەمچىسى. ھەر بىر ۋاكالەتچىنىڭ مەخسۇس قوراللار توپلىمى، ماسلاشتۇرۇلغان سىستېما promptى ۋە باشلاش تەكلىپلىرى بار. سۆھبەت بېشىدىكى يېڭى **ۋاكالەتچى تاللىغۇچ** ئارقىلىق ئالماشتۇرغىلى بولىدۇ. [ئىچكى ۋاكالەتچىلەر](../../user-guide/configuration/built-in-agents)نى كۆرۈڭ.
- **ئىقتىدار بايراقلىرى** — يېڭى **تەڭشەكلەر → ئىقتىدار بايراقلىرى** بەتكۈچى، زىيارەت كونترولىنى ئالماشتۇرغۇچلار (پەقەت باشقۇرغۇچىلارغا چەكلەش، پەقەت network باشقۇرغۇچىلىرىغا چەكلەش، مۇشتەرى زىيارىتى، ئەزا ئەمەسلەر ئۈچۈن چەكلەش) ۋە ماركا تاللاشلىرى (ئاستى بەت ئىزاھاتىنى يوشۇرۇش، خاس ۋاكالەتچى ئىسمى، ۋاكالەتچى تاللىغۇچنى يوشۇرۇش، بېكەت سىنبەلگىسىنى سۆھبەت avatarى قىلىپ ئىشلىتىش) بىلەن. [Gratis AI Agent تەڭشەكلىرى](../../user-guide/administration/gratis-ai-agent-settings)نى كۆرۈڭ.
- **ئاخىرقى سەشننى ئەسلىگە كەلتۈرۈش** — سۆھبەت تاختىسى ھازىر بەت يۈكلەنگەندە ۋە widget ئېچىلغاندا ئەڭ يېقىنقى سۆھبەتنى ئاپتوماتىك قايتا يۈكلەيدۇ، شۇڭا بەتلەر ئارا يۆتكىلىشتە context ھەرگىز يوقالمايدۇ.
- **Plugin مەشغۇلات ئۇلانمىلىرى** — تەڭشەكلەر ۋە ئىقتىدارلار تىزىملىكىگە تېز ئۇلانمىلار ھازىر WordPress **Plugins → ئورنىتىلغان Plugins** ئېكرانىدا plugin چۈشەندۈرۈشىنىڭ ئاستىدا كۆرۈنىدۇ.

### ياخشىلاندى {#improved}

- **سۈرەت مەنبەسىنى قايتا سىناش** — ۋاكالەتچى ھازىر چۈشۈرۈش مەغلۇب بولغاندا AI ھاسىل قىلغان سۈرەتكە قايتىشتىن بۇرۇن تەڭشەلگەن بارلىق ھەقسىز سۈرەت مەنبەلىرىنى قايتا سىنايدۇ.
- **Model ئۇچۇر تاختىسى** — سۆھبەت بېشىدا دائىم كۆرۈنۈپ تۇرىدۇ؛ ئەمدى بىرىنچى ئۇچۇردىن كېيىن يوشۇرۇلمايدۇ.
- **ئاپتوماتىك سىيرىش ھەرىكىتى** — ئىشلەتكۈچى ئوقۇش ئۈچۈن ئۈستىگە سىيرىغاندا ئاپتوماتىك سىيرىش توختاپ تۇرىدۇ؛ لەيلىمە **ئاستىغا سىيرىش** كۇنۇپكىسى كۆرۈنىدۇ ۋە ئىشلەتكۈچى ئەڭ يېڭى ئۇچۇرغا يەتكەندە ئاپتوماتىك يوقىلىدۇ.
- **ۋاكالەتچى تاللىغۇچ UI** — ھەر بىر ۋاكالەتچىگە ئايرىم سىنبەلگىلەر بار form-table قاپلىمىسى قىلىپ قايتا لايىھەلەندى، ۋاكالەتچىلەرنى بىر قاراشتىلا تونۇش ۋە ئالماشتۇرۇش ئاسانلاشتى.
- **ھورۇن يۈكلىنىدىغان JS پارچىلىرى** — سۆھبەت widgetىنىڭ دەسلەپكى JavaScript bundleى ھازىر ھورۇن يۈكلىنىدىغان پارچىلارغا بۆلۈندى، دەسلەپكى bundle چوڭلۇقلىرىنى 75–90% ئازايتىدۇ.
- **سۆھبەت widgetىنى قايتا لايىھەلەش** — بىرلەشتۈرۈلگەن AI سىنبەلگىسى ئىلگىرىكى خاس avatarنىڭ ئورنىنى ئالدى؛ ئىچكى ۋاكالەتچى سىستېمىسى بىلەن بىردەك.
- **URL ئۇلانمىغا ئايلاندۇرۇش** — سىستېما ئۇچۇرلىرى ۋە خاتالىق ئۇچۇرى كۆپۈكچىلىرىدە كۆرۈنگەن URLلار ھازىر چېكىشكە بولىدىغان ئۇلانما سۈپىتىدە كۆرسىتىلىدۇ.

### تۈزىتىلدى {#fixed}

- **ئىقتىدارلارنى بايقىغىلى بولۇش** — چۈشەندۈرۈشلەر، سىستېما prompt پايدىلانمىلىرى ۋە namespace ماسلىشىشى تۈزىتىلدى، شۇنىڭ بىلەن بارلىق ئىقتىدارلار ۋاكالەتچىنىڭ قورال تىزىملىكىدە ئىشەنچلىك كۆرۈنىدۇ.
- **تەمىنلىگۈچىلەر cacheى** — تەمىنلىگۈچىلەر ھازىر نەشر سانى ئارقىلىق پۈتكۈل بېكەت بويىچە cache قىلىنىدۇ، multisite تورلىرىدا كونىراپ كەتكەن تەمىنلىگۈچى مەسىلىلىرىنىڭ ئالدىنى ئالىدۇ.
- **`ability_invalid_output`** — 12 ئىقتىدار بىر تەرەپ قىلغۇچىسى بويىچە ھەل قىلىندى؛ ھەممىسى توغرا قۇرۇلمىلىق JSON جاۋابلىرىنى قايتۇرىدۇ.
- **`pending_client_tool_calls` تۇرۇبا لىنىيەسى** — باشتىن ئاخىر ئۇلاندى، شۇنىڭ بىلەن خېرىدار تەرەپ قورال چاقىرىقلىرى توغرا تاماملىنىپ نەتىجىلەرنى modelغا قايتۇرىدۇ.
- **تارىخ تارتمىسى** — قايتۇرغىلى بولمايدىغان ئۆزگىرىشلەر قايتۇرۇش تىزىملىكىدىن چىقىرىۋېتىلدى؛ **تولۇق تارىخنى كۆرۈش** ئۇلانمىسى ھازىر توغرا ئىشلەيدۇ.
- **ئۆزگىرىشلەر/قايتۇرۇش سىستېمىسى** — بەش ئايرىم خاتالىق تۈزىتىلدى ۋە يېڭى admin ئارايۈزى ئاستىدا بىرلەشتۈرۈلدى.
- **تەڭشەكلەرنى ساقلاش toastى** — **تەڭشەكلەرنى ساقلاش**نى چەككەندىن كېيىن snackbar ئۇقتۇرۇشى ھازىر ئىشەنچلىك كۆرۈنىدۇ.
- **ئەخلەت ساندۇقى context تىزىملىكى** — تۈرلەرنى ئەخلەت كۆرۈنۈشىدىن چىقمايلا قاتتىق ئۆچۈرگىلى بولۇشى ئۈچۈن **مەڭگۈلۈك ئۆچۈرۈش** تاللىشى قوشۇلدى.
- **تەھرىرلەش ۋە قايتا ئەۋەتىش** — تەھرىرلەش سىنبەلگىسىنى چەككەندە ھازىر پەقەت چېكىلگەن ئۇچۇرلا تەھرىرلەش ھالىتىگە كىرىدۇ، threadتىكى بارلىق ئۇچۇرلار ئەمەس.
- **سۆھبەت ئورۇنلاشتۇرۇش ئېگىزلىكى** — بەتنىڭ ئۈستىدە plugin قىستۇرغان مەزمۇن (admin ئۇقتۇرۇشلىرى، bannerلار) كۆرۈنگەندە سۆھبەت تاختىسى ئېگىزلىكىنى ماسلاشتۇرىدۇ، كىرگۈزۈش رايونىنىڭ ئېكراندىن سىرتقا ئىتتىرىلىپ كېتىشىنىڭ ئالدىنى ئالىدۇ.

## 1.4.0 — 2026-04-09 كۈنى ئېلان قىلىندى {#140--released-on-2026-04-09}

### يېڭى {#new-1}

- **WP-CLI benchmark command** (`wp gratis-ai-agent benchmark`) — CI pipelines ۋە مودېل باھالاش خىزمەت ئېقىملىرى ئۈچۈن بۇيرۇق قۇرىدىن Agent Capabilities v1 benchmark suite نى ئىجرا قىلىدۇ. ھەر-سوئال بويىچە ئىجرا قىلىش، تەمىنلىگۈچى/مودېل قاپلاشلىرى ۋە JSON/CSV چىقىرىشنى قوللايدۇ.
- **Agent Capabilities v1 benchmark suite** — نومۇرلاش ۋە token/داۋاملىشىش ۋاقتى دوكلاتى بىلەن، تولۇق ئىقتىدار دائىرىسىنى سىنايدىغان مۇرەككەپ، كۆپ باسقۇچلۇق prompt لارنىڭ قۇرۇلمىلىق توپلىمى.
- **Custom Post Type ئىقتىدارلىرى** — `register_post_type`, `list_post_types`, `delete_post_type`. تىزىملاشلار options جەدۋىلى ئارقىلىق قايتا قوزغىتىشلاردىن كېيىنمۇ ساقلىنىدۇ.
- **Custom Taxonomy ئىقتىدارلىرى** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. ئىختىيارىي rewrite slug لار بىلەن دەرىجىلىك ۋە تەكشى taxonomy لارنى قوللايدۇ.
- **Design System ئىقتىدارلىرى** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. بەش تال تاللانما preset نى ئۆز ئىچىگە ئالىدۇ: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **Global Styles ئىقتىدارلىرى** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. WordPress Global Styles API ئارقىلىق theme.json قىممەتلىرىنى ئوقۇيدۇ ۋە يازىدۇ.
- **Navigation Menu باشقۇرۇش ئىقتىدارى** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. ئىچكى قەۋەتلىك تىزىملىكلەر ۋە theme ئورۇن تەقسىملەشنى قوللايدۇ.
- **Options Management ئىقتىدارى** — `get_option`, `set_option`, `delete_option`, `list_options`. مۇھىم WordPress تاللانمىلىرىنى ئۆزگەرتىشتىن قوغدايدىغان ئىچىگە ئورنىتىلغان بىخەتەرلىك blocklist نى ئۆز ئىچىگە ئالىدۇ.
- **Installable Abilities Registry** — `list_available_abilities`, `install_ability`, `recommend_plugin`. WordPress plugin لىرى سۈپىتىدە تارقىتىلغان ئىقتىدار بولاقلىرىنى بايقاش ۋە ئاكتىپلاش.
- **Site Builder Orchestration v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. plugin بايقاش، باسقۇچ چىقىرىش پايدىلانمىلىرى، ئىلگىرىلەشنى ئىز قوغلاش ۋە ئاپتونوم خاتالىق ئەسلىگە كەلتۈرۈش بىلەن كۆپ باسقۇچلۇق تور بېكەت قۇرۇش پىلانلىرى.
- **رېستوران تور بېكىتى benchmark سوئالى** (`q-restaurant-website`) — CPT تىزىملاش، design system، navigation ۋە plugin بايقاشنى قاپلايدىغان باشتىن-ئاخىر سىناق.
- تېخىمۇ تېز يەرلىك تەرەققىيات تەڭشەش ئۈچۈن **AI provider connector plugin لىرى** WordPress Playground blueprints قا قوشۇلدى.

### ياخشىلاندى {#improved-1}

- README، AI provider connector ھۆججەتلىرى ۋە تەڭشەش كۆرسەتمىلىرى بىلەن يېڭىلاندى.

### تۈزىتىلدى {#fixed-1}

- `main` تارمىقىدىكى 25 دانە PHPUnit سىناق مەغلۇبىيىتى ھەل قىلىندى.

- كونا ID لار بىلەن توقۇنۇشتىن ساقلىنىش ئۈچۈن ۋەزىپە ID لىرى قايتا نومۇرلاندى.
