# ماس كېلىدىغان ئاخىرقى نۇقتىلار ئۈچۈن Ultimate AI Connector

ماس كېلىدىغان ئاخىرقى نۇقتىلار ئۈچۈن Ultimate AI Connector، OpenAI بىلەن ماس كېلىدىغان AI تەمىنلىگۈچى APIلىرىنى WordPress multisite تورىڭىز بىلەن بىرلەشتۈرىدۇ. كۆپلىگەن AI تەمىنلىگۈچىلەرنى — ھەر بىرىنىڭ ئۆزىگە خاس ئاخىرقى نۇقتا URLى، API ئاچقۇچى ۋە كۆڭۈلدىكى مودېلى بىلەن — سەپلەپ، تەمىنلىگۈچى ئىشلىتىلمەيدىغان بولغاندا ئاپتوماتىك fallback ئارقىلىق تەلەپلەرنى يوللاڭ.

## ئاساسلىق ئىقتىدارلار

- **كۆپ تەمىنلىگۈچىنى قوللاش**: كۆپلىگەن AI تەمىنلىگۈچى كارتىلىرىنى قوشۇڭ، ھەر بىرىنىڭ ئۆزىگە خاس ئاخىرقى نۇقتا URLى، API ئاچقۇچى ۋە كۆڭۈلدىكى مودېلى بولىدۇ
- **سۆرەپ تەرتىپلەش**: ھەر بىر كارتىدىكى تۇتقۇچنى ئىشلىتىپ تەمىنلىگۈچىلەرنى ئالدىنلىق بويىچە رەتلەڭ؛ تەلەپلەر تەمىنلىگۈچىلەرنى تەرتىپ بويىچە ئىشلىتىدۇ
- **ئاپتوماتىك fallback يوللاش**: ئاساسىي تەمىنلىگۈچى ئىشلىتىلمەيدىغان بولسا، تەلەپلەر ئاپتوماتىك ھالدا كېيىنكى سەپلەنگەن تەمىنلىگۈچىگە fallback قىلىدۇ
- **ھەر تەمىنلىگۈچى بويىچە سەپلىمە**: ھەر بىر تەمىنلىگۈچى ئۆزىنىڭ كىرىش ئۇچۇرلىرى ۋە مودېل تاللىشى بىلەن مۇستەقىل سەپلىنىدۇ
- **كارتا ئاساسىدىكى تەڭشەك UI**: مۇقىم WordPress 6.9+ Card زاپچاسلىرى (CardBody, CardHeader, CardDivider) ئۈستىگە قۇرۇلغان
- **قوشقاندا ئاپتوماتىك كېڭىيىش**: يېڭى تەمىنلىگۈچى كارتىلىرى ئاپتوماتىك كېڭىيىدۇ، شۇڭا تەپسىلاتلارنى دەرھال تولدۇرالايسىز
- **Script cache busting**: تەمىنلىگۈچى scriptلىرى plugin يېڭىلانغاندا cache-busted قىلىنىپ، ئەڭ يېڭى نەشرىنىڭ يۈكلىنىشىگە كاپالەتلىك قىلىنىدۇ

## تەلەپلەر

- WordPress 5.3 ياكى ئۇنىڭدىن يۇقىرى
- PHP 7.4 ياكى ئۇنىڭدىن يۇقىرى
- Ultimate Multisite plugin (ئاكتىپ)
- OpenAI بىلەن ماس كېلىدىغان API ئاخىرقى نۇقتىسى (ئۆزىڭىز ساھىبخانلىق قىلغان ياكى ئۈچىنچى تەرەپ)

## ئورنىتىش

1. addon ھۆججەتلىرىنى `/wp-content/plugins/` مۇندەرىجىڭىزگە يۈكلەڭ
2. WordPress ئىچىدىكى 'Plugins' تىزىملىكى ئارقىلىق pluginنى ئاكتىپلاڭ
3. addonنى سەپلەش ئۈچۈن **Ultimate Multisite → AI Connector** غا يۆتكىلىڭ

## سەپلىمە

### كۆپ تەمىنلىگۈچىنى تەڭشەش

سەپلىمە ئېكرانى كارتا ئاساسىدىكى ئورۇنلاشتۇرۇشنى ئىشلىتىدۇ، بۇ يەردە ھەر بىر كارتا بىر AI تەمىنلىگۈچىگە ۋەكىللىك قىلىدۇ. ئېھتىياجىڭىزغا قاراپ خالىغانچە تەمىنلىگۈچى قوشالايسىز.

#### تەمىنلىگۈچى قوشۇش

1. تور باشقۇرغۇچىسىدا **Ultimate Multisite → AI Connector** غا كىرىڭ
2. **تەمىنلىگۈچى قوشۇش** نى چېكىڭ
3. يېڭى تەمىنلىگۈچى كارتىسى كۆرۈنىدۇ ۋە ئاپتوماتىك كېڭىيىدۇ
4. تەمىنلىگۈچى تەپسىلاتلىرىنى تولدۇرۇڭ:
   - **تەمىنلىگۈچى نامى** — بۇ تەمىنلىگۈچىنى پەرقلەندۈرۈش ئۈچۈن بەلگە (مەسىلەن، "OpenAI"، "Mistral"، "Local Ollama")
   - **ئاخىرقى نۇقتا URLى** — OpenAI بىلەن ماس كېلىدىغان APIنىڭ ئاساسىي URLى (مەسىلەن، `https://api.openai.com/v1`)
   - **API ئاچقۇچى** — بۇ تەمىنلىگۈچى ئۈچۈن API ئاچقۇچىڭىز
   - **كۆڭۈلدىكى مودېل** — بۇ تەمىنلىگۈچى ئۈچۈن كۆڭۈلدىكى ھالدا ئىشلىتىلىدىغان مودېل كىملىكى (مەسىلەن، `gpt-4o`، `mistral-small`)
5. **تەڭشەكلەرنى ساقلاش** نى چېكىڭ

#### تەمىنلىگۈچىنى ئۆچۈرۈش

ھەرقانداق تەمىنلىگۈچى كارتىسىدىكى **ئۆچۈرۈش** كۇنۇپكىسىنى چېكىپ ئۇنى ئۆچۈرۈڭ. ئۆزگىرىشنى قوللىنىش ئۈچۈن تەڭشەكلەرنى ساقلاڭ.

#### تەمىنلىگۈچىلەرنى قايتا رەتلەش

ھەرقانداق تەمىنلىگۈچى كارتىسىنىڭ سول تەرىپىدىكى تۇتقۇچ سىنبەلگىسىنى (⣿) سۆرەپ، ئۇنى قايتا رەتلەڭ. تەرتىپ fallback ئالدىنلىقىنى بەلگىلەيدۇ: تەلەپلەر بىرىنچى تەمىنلىگۈچىگە ئەۋەتىلىدۇ، تەلەپ مەغلۇپ بولسا كېيىنكى تەمىنلىگۈچىلەرگە fallback قىلىدۇ.

### Fallback يوللاش

ئاساسىي تەمىنلىگۈچىگە بولغان تەلەپ مەغلۇپ بولغاندا — ۋاقىت ئېشىپ كېتىش، چەكلىمە ياكى مۇلازىمەت ئۈزۈلۈشى سەۋەبىدىن — connector تىزىملىكتىكى كېيىنكى تەمىنلىگۈچى بىلەن ئاپتوماتىك قايتا سىنايدۇ. بۇ بىر تەمىنلىگۈچى مۇۋەپپەقىيەتلىك بولغۇچە ياكى بارلىق تەمىنلىگۈچىلەر سىناپ بولغۇچە تىزىملىك بويىچە داۋاملىشىدۇ.

| ئەھۋال | ھەرىكەت |
|----------|-----------|
| ئاساسىي تەمىنلىگۈچى جاۋاب قايتۇرىدۇ | تەلەپ نورمال تەمىنلىنىدۇ |
| ئاساسىي تەمىنلىگۈچى مەغلۇپ بولىدۇ | تەلەپ كېيىنكى تەمىنلىگۈچى بىلەن قايتا سىنىلىدۇ |
| بارلىق تەمىنلىگۈچىلەر مەغلۇپ بولىدۇ | خاتالىق چاقىرغۇچىغا قايتۇرۇلىدۇ |

Fallback چاقىرغۇچى plugin ياكى theme ئۈچۈن ئېنىق كۆرۈنمەيدۇ — قانچە تەمىنلىگۈچى سىناپ بېقىلغان بولۇشىدىن قەتئىينەزەر، ئۇلار مۇۋەپپەقىيەتلىك جاۋاب ياكى ئاخىرقى خاتالىقنى تاپشۇرۇۋالىدۇ.

### تەمىنلىگۈچى سەپلىمىسى پايدىلىنىشى

| ساھە | زۆرۈر | چۈشەندۈرۈش |
|-------|----------|-------------|
| تەمىنلىگۈچى نامى | ھەئە | بۇ تەمىنلىگۈچى ئۈچۈن كۆرسىتىش بەلگىسى |
| ئاخىرقى نۇقتا URLى | ھەئە | OpenAI بىلەن ماس كېلىدىغان APIنىڭ ئاساسىي URLى |
| API ئاچقۇچى | ھەئە | بۇ تەمىنلىگۈچى ئۈچۈن دەلىللەش ئاچقۇچى |
| كۆڭۈلدىكى مودېل | ھەئە | تەلەپتە مودېل كۆرسىتىلمىگەندە ئىشلىتىلىدىغان مودېل كىملىكى |

## مەسىلە ھەل قىلىش

### ھېچقانداق تەمىنلىگۈچى كۆرسىتىلمىدى

ساقلىغاندىن كېيىن تەمىنلىگۈچى تىزىملىكى بوش بولسا، تۆۋەندىكىلەرنى جەزملەشتۈرۈڭ:

- تەمىنلىگۈچىلەرنى قوشقاندىن كېيىن **تەڭشەكلەرنى ساقلاش** نى چەكتىڭىز
- توركۆرگۈچىڭىز تەڭشەك بېتىنىڭ ساقلىشىنى توسۇپ قالمىدى (JavaScript خاتالىقلىرى ئۈچۈن توركۆرگۈچ consoleىنى تەكشۈرۈڭ)
- plugin ئاكتىپ، ھەمدە سىز تور باشقۇرغۇچى ئېكرانىدا تۇرۇۋاتىسىز

### بارلىق تەمىنلىگۈچىلەر ئۈچۈن تەلەپلەر مەغلۇپ بولۇۋاتىدۇ

- ھەر بىر تەمىنلىگۈچىنىڭ ئاخىرقى نۇقتا URLىنىڭ مۇلازىمېتىرىڭىزدىن يېتىپ بارغىلى بولىدىغانلىقىنى تەكشۈرۈڭ (پەقەت توركۆرگۈچىڭىزدىنلا ئەمەس)
- ھەر بىر API ئاچقۇچىنىڭ ئىناۋەتلىك ئىكەنلىكىنى ۋە ۋاقتى ئۆتۈپ كەتمىگەنلىكىنى تەكشۈرۈڭ
- تەمىنلىگۈچىنىڭ dashboardىدا ئۇنىڭ چەكلىمە ۋە quota ھالىتىنى كۆرۈپ چىقىڭ
- connectorدىن كەلگەن تەپسىلىي خاتالىق ئۇچۇرلىرىنى تۇتۇش ئۈچۈن WordPress debug logging (`WP_DEBUG_LOG`) نى قوزغىتىڭ

### بىر تەمىنلىگۈچى fallback ئۈچۈن ئىشلىتىلمەيۋاتىدۇ

سۆرەپ تەرتىپلەش fallback تەرتىپىنى ئۆزگەرتىدۇ. fallback تەلەپلىرىنى قوبۇل قىلىشىنى كۈتكەن تەمىنلىگۈچىنىڭ تىزىملىكتە توغرا ئورۇنلاشتۇرۇلغانلىقىنى، شۇنداقلا ئۇنىڭ ئاخىرقى نۇقتا URLى ۋە API ئاچقۇچىنىڭ توغرا ئىكەنلىكىنى جەزملەشتۈرۈڭ.

### يېڭى تەمىنلىگۈچى كارتىسى ئاپتوماتىك كېڭەيمەيدۇ

ئاپتوماتىك كېڭىيىش ئۈچۈن توركۆرگۈچىڭىزدە JavaScript قوزغىتىلغان بولۇشى كېرەك. كارتا چەككەندە كېڭەيمىسە، خاتالىقلار ئۈچۈن توركۆرگۈچ consoleىنى تەكشۈرۈڭ. ئاپتوماتىك كېڭىيىش قوزغالمىغان تەقدىردىمۇ، كارتا ماۋزۇسىنى چېكىش ئارقىلىق كارتا مەزمۇنلىرىغا ھەمىشە كىرگىلى بولىدۇ.
