# سبق 7: ان کي پنهنجو بڻائڻ

توهان جي گراهڪن کي ڪڏهن به ائين محسوس نه ٿيڻ گهرجي ته اهي "ڪو WordPress پلگ ان" استعمال ڪري رهيا آهن. انهن کي ائين محسوس ٿيڻ گهرجي ته اهي FitSite استعمال ڪري رهيا آهن -- هڪ پليٽفارم جيڪو سندن صنعت لاءِ ٺاهيو ويو آهي. هي سبق برانڊنگ، وائيٽ-ليبلنگ، ۽ پليٽفارم کي هڪ پراڊڪٽ جهڙو محسوس ڪرائڻ بابت آهي.

## اسان ڪٿي ڇڏيو هو {#where-we-left-off}

FitSite وٽ هڪ ڪم ڪندڙ checkout وهڪرو آهي جيڪو فٽنيس اسٽوڊيو مالڪن کي منصوبي جي چونڊ کان وٺي زنده سائيٽ تائين وٺي وڃي ٿو. هاڻي اسان سڄي تجربي کي هڪ مربوط، برانڊ ٿيل پراڊڪٽ جهڙو بڻايون ٿا.

## توهان جي پليٽفارم جو ڊومين {#your-platform-domain}

توهان جي برانڊ جو بنياد توهان جو ڊومين آهي. FitSite لاءِ:

- **مکيه ڊومين**: `fitsite.com` (توهان جي مارڪيٽنگ سائيٽ ۽ نيٽ ورڪ روٽ)
- **گراهڪن جون سائيٽون**: `studioname.fitsite.com` (سب ڊومين)
- **ڪسٽم ڊومين**: Growth ۽ Pro منصوبن وارا گراهڪ پنهنجو ڊومين ميپ ڪري سگهن ٿا

### پنهنجو ڊومين سيٽ ڪرڻ {#setting-up-your-domain}

1. پنهنجو پليٽفارم ڊومين رجسٽر ڪريو
2. ان کي پنهنجي هوسٽنگ فراهم ڪندڙ ڏانهن اشارو ڪريو
3. گراهڪن جي سب ڊومينن لاءِ wildcard DNS (`*.fitsite.com`) ترتيب ڏيو
4. پڪ ڪريو ته wildcard SSL فعال آهي

تفصيلي هدايتن لاءِ [Domain Mapping ڪيئن ترتيب ڏجي](/user-guide/domain-mapping/how-to-configure-domain-mapping) ڏسو.

## ايڊمن تجربي جي وائيٽ-ليبلنگ {#white-labeling-the-admin-experience}

جڏهن ڪو فٽنيس اسٽوڊيو مالڪ پنهنجي سائيٽ Dashboard ۾ لاگ ان ٿئي، ته هن کي توهان جو برانڊ ڏسڻ گهرجي، WordPress يا Ultimate Multisite جي برانڊنگ نه.

### ڪسٽم لاگ ان صفحو {#custom-login-page}

WordPress لاگ ان صفحي کي هن ريت ڪسٽمائيز ڪريو ته ڏيکاري:

- توهان جو FitSite لوگو
- فٽنيس سان مناسبت رکندڙ پسمنظر جون تصويرون
- توهان جا برانڊ رنگ

### Dashboard برانڊنگ {#dashboard-branding}

[Admin Page Creator](/addons/admin-page-creator) addon يا ڪسٽم CSS استعمال ڪريو ته:

- WordPress لوگو کي توهان جي FitSite لوگو سان مٽايو
- ايڊمن رنگ اسڪيم کي پنهنجي برانڊ سان ملائڻ لاءِ ڪسٽمائيز ڪريو
- فٽنيس-مخصوص تڪڙين لنڪن ۽ مدد وسيلن سان هڪ ڪسٽم dashboard widget شامل ڪريو

### ڪسٽم ايڊمن صفحا {#custom-admin-pages}

ڪسٽم ايڊمن صفحا ٺاهڻ تي غور ڪريو جيڪي فٽنيس اسٽوڊيو مالڪن لاءِ سڀ کان لاڳاپيل عمل مٿاڇري تي آڻين:

- "پنهنجو ڪلاس شيڊيول ايڊٽ ڪريو"
- "ٽرينر پروفائلز اپڊيٽ ڪريو"
- "پنهنجي سائيٽ ڏسو"

هي سکيا واري وڪڙ کي گهٽائي ٿو، ڇاڪاڻ⁠تہ نچ سان لاڳاپيل عملن کي معياري WordPress مينيو ۾ لڪائڻ بدران سامهون ۽ وچ ۾ رکي ٿو.

## پنهنجي رابطن جي برانڊنگ {#branding-your-communications}

هر اي ميل، انوائس، ۽ اطلاع توهان جي برانڊ کي مضبوط ڪرڻ گهرجي.

### سسٽم اي ميلون {#system-emails}

**Ultimate Multisite > Settings > Emails** ڏانهن وڃو ۽ سڀ سسٽم اي ميلون ڪسٽمائيز ڪريو:

- **موڪليندڙ جو نالو**: FitSite
- **موڪليندڙ اي ميل**: hello@fitsite.com
- **اي ميل ٽيمپليٽس**: پنهنجا برانڊ رنگ ۽ لوگو استعمال ڪريو
- **ٻولي**: سڄي جاءِ تي فٽنيس-مخصوص

ڪسٽمائيز ڪرڻ لاءِ اهم اي ميلون:

| اي ميل | عام ورزن | FitSite ورزن |
|-------|----------------|-----------------|
| ڀليڪار | "توهان جي نئين سائيٽ تيار آهي" | "توهان جي فٽنيس اسٽوڊيو ويب سائيٽ زنده آهي" |
| ادائيگي جي رسيد | "ادائيگي وصول ٿي" | "FitSite سبسڪرپشن ادائيگي جي تصديق ٿي" |
| ٽرائل ختم ٿيڻ | "توهان جو ٽرائل جلد ختم ٿي رهيو آهي" | "توهان جو FitSite ٽرائل 3 ڏينهن ۾ ختم ٿيندو -- پنهنجي اسٽوڊيو ويب سائيٽ زنده رکو" |

### انوائسز {#invoices}

انوائس ٽيمپليٽس کي هنن شين سان ڪسٽمائيز ڪريو:

- توهان جو FitSite لوگو ۽ برانڊ رنگ
- توهان جي ڪاروباري تفصيل
- فٽنيس-مخصوص پراڊڪٽ نالا (عام منصوبي IDs نه)

## گراهڪن کي نظر ايندڙ سائيٽ {#the-customer-facing-site}

توهان جي مکيه ڊومين (`fitsite.com`) کي هڪ مارڪيٽنگ سائيٽ جي ضرورت آهي جيڪا پليٽفارم وڪڻي. هي Ultimate Multisite نيٽ ورڪ ايڊمن کان الڳ آهي -- اهو توهان جي ڪاروبار جو عوامي چهرو آهي.

اهم صفحا:

- **هوم پيج**: فٽنيس ڪاروبارن لاءِ واضح قدر جي پيشڪش
- **خاصيتون**: فٽنيس اصطلاحن ۾ FitSite ڇا ڪري ٿو
- **قيمت**: توهان جا ٽي منصوبا، نچ-مخصوص خاصيتن جي ڀيٽ سان
- **مثال**: پليٽفارم تي ٺهيل سائيٽون ڏيکاريو
- **سائن اپ**: توهان جي checkout فارم ڏانهن لنڪس

:::tip توهان جي مارڪيٽنگ سائيٽ نيٽ ورڪ سائيٽ ٿي سگهي ٿي
پنهنجي مارڪيٽنگ سائيٽ کي پنهنجي ئي نيٽ ورڪ اندر هڪ سائيٽ طور ٺاهيو. هي توهان کي ان کي ساڳئي Dashboard مان منظم ڪرڻ ڏئي ٿو ۽ توهان جي پنهنجي پليٽفارم جي صلاحيتن جو مظاهرو ڪري ٿو.
:::

## گراهڪن لاءِ ڪسٽم ڊومين {#custom-domain-for-customers}

انهن منصوبن وارن گراهڪن لاءِ جن ۾ ڪسٽم ڊومين شامل آهن، عمل کي صاف نموني دستاويز ڪريو:

1. گراهڪ پنهنجو ڊومين ڪنهن registrar وٽ رجسٽر يا ٽرانسفر ڪري ٿو
2. گراهڪ DNS کي توهان جي پليٽفارم ڏانهن اشارو ڪرڻ لاءِ اپڊيٽ ڪري ٿو (درست records ڏيو)
3. Ultimate Multisite ڊومين mapping ۽ SSL سنڀالي ٿو

هن عمل لاءِ خاص طور تي هڪ مدد آرٽيڪل يا knowledge base entry ٺاهيو، جيڪا غير ٽيڪنيڪل فٽنيس اسٽوڊيو مالڪن لاءِ لکيل هجي.

## هن وقت تائين FitSite نيٽ ورڪ {#the-fitsite-network-so-far}

```
FitSite نيٽ ورڪ
├── WordPress Multisite (subdomain موڊ)
├── Ultimate Multisite (ترتيب ڏنل + برانڊ ٿيل)
├── پليٽفارم ڊومين (fitsite.com + wildcard SSL)
├── سائيٽ ٽيمپليٽس (Studio Essential, Gym Pro, Fitness Chain)
├── پراڊڪٽس (Starter, Growth, Pro + Order Bumps)
├── Checkout وهڪرو (نچ-مخصوص، ٽيسٽ ٿيل)
├── برانڊنگ
│   ├── ڪسٽم لاگ ان صفحو
│   ├── برانڊ ٿيل ايڊمن Dashboard
│   ├── نچ-مخصوص سسٽم اي ميلون
│   ├── برانڊ ٿيل انوائسز
│   └── fitsite.com تي مارڪيٽنگ سائيٽ
└── onboarding وهڪري لاءِ تيار (اڳيون سبق)
```

## هن سبق ۾ اسان ڇا ٺاهيو {#what-we-built-this-lesson}

- **پليٽفارم ڊومين ۽ DNS** برانڊ ٿيل تجربي لاءِ ترتيب ڏنل
- **وائيٽ-ليبل ٿيل ايڊمن** سڄي جڳهه FitSite برانڊنگ سان
- **ڪسٽمائيز ٿيل رابطا** -- اي ميلون، انوائسز، ۽ اطلاع سڀ برانڊ مطابق
- **هڪ مارڪيٽنگ سائيٽ** جيڪا FitSite کي فٽنيس اسٽوڊيو مالڪن کي وڪڻي ٿي
- **ڪسٽم ڊومين دستاويز** انهن گراهڪن لاءِ جيڪي پنهنجو ڊومين چاهين ٿا
