# مطابقت رکندڙ ڇيڙن لاءِ Ultimate AI Connector

مطابقت رکندڙ ڇيڙن لاءِ Ultimate AI Connector، OpenAI-مطابقت رکندڙ AI مهيا ڪندڙن جي APIs کي توهان جي WordPress multisite نيٽ ورڪ سان ضم ڪري ٿو. ڪيترائي AI مهيا ڪندڙ ترتيب ڏيو — هر هڪ پنهنجي endpoint URL، API چاٻي، ۽ ڊفالٽ ماڊل سان — ۽ جڏهن ڪو مهيا ڪندڙ موجود نه هجي ته خودڪار fallback سان درخواستن کي روٽ ڪريو.

## اهم خاصيتون

- **گهڻن مهيا ڪندڙن جي حمايت**: ڪيترائي AI مهيا ڪندڙ ڪارڊ شامل ڪريو، هر هڪ پنهنجي endpoint URL، API چاٻي، ۽ ڊفالٽ ماڊل سان
- **ڇڪي ٻيهر ترتيب ڏيڻ**: هر ڪارڊ تي grip handle استعمال ڪري مهيا ڪندڙن کي ترجيح موجب ترتيب ڏيو؛ درخواستون ترتيب موجب مهيا ڪندڙ استعمال ڪن ٿيون
- **خودڪار fallback routing**: جيڪڏهن بنيادي مهيا ڪندڙ موجود نه هجي، درخواستون خودڪار طور ايندڙ ترتيب ڏنل مهيا ڪندڙ ڏانهن fallback ٿين ٿيون
- **هر مهيا ڪندڙ لاءِ ترتيب**: هر مهيا ڪندڙ پنهنجي سندن ۽ ماڊل چونڊ سان الڳ الڳ ترتيب ڏنل آهي
- **ڪارڊ تي ٻڌل سيٽنگن جو UI**: مستحڪم WordPress 6.9+ Card جزن (CardBody, CardHeader, CardDivider) تي ٺهيل
- **شامل ڪرڻ تي خودڪار کولڻ**: نوان مهيا ڪندڙ ڪارڊ خودڪار طور کلي وڃن ٿا ته جيئن توهان فوري طور تفصيل ڀري سگهو
- **اسڪرپٽ cache busting**: plugin update تي مهيا ڪندڙ اسڪرپٽس کي cache-busted ڪيو وڃي ٿو ته جيئن تازو نسخو لوڊ ٿيڻ کي يقيني بڻايو وڃي

## گهرجون

- WordPress 5.3 يا وڌيڪ
- PHP 7.4 يا وڌيڪ
- Ultimate Multisite plugin (فعال)
- هڪ OpenAI-مطابقت رکندڙ API endpoint (پاڻ ميزبان ڪيل يا ٽئين ڌر جو)

## انسٽاليشن

1. addon فائلون پنهنجي `/wp-content/plugins/` ڊاريڪٽري ۾ اپلوڊ ڪريو
2. WordPress ۾ 'Plugins' مينيو ذريعي plugin فعال ڪريو
3. addon ترتيب ڏيڻ لاءِ **Ultimate Multisite → AI Connector** ڏانهن وڃو

## ترتيب

### گهڻن مهيا ڪندڙن جي سيٽ اپ

ترتيب واري اسڪرين ڪارڊ تي ٻڌل ترتيب استعمال ڪري ٿي، جتي هر ڪارڊ هڪ AI مهيا ڪندڙ جي نمائندگي ڪري ٿو. توهان پنهنجي ضرورت موجب اوترا مهيا ڪندڙ شامل ڪري سگهو ٿا.

#### مهيا ڪندڙ شامل ڪرڻ

1. نيٽ ورڪ admin ۾ **Ultimate Multisite → AI Connector** ڏانهن وڃو
2. **مهيا ڪندڙ شامل ڪريو** تي ڪلڪ ڪريو
3. نئون مهيا ڪندڙ ڪارڊ ظاهر ٿئي ٿو ۽ خودڪار طور کلي وڃي ٿو
4. مهيا ڪندڙ جا تفصيل ڀريو:
   - **مهيا ڪندڙ جو نالو** — هن مهيا ڪندڙ کي سڃاڻڻ لاءِ هڪ ليبل (مثال طور "OpenAI"، "Mistral"، "Local Ollama")
   - **Endpoint URL** — OpenAI-مطابقت رکندڙ API جو بنيادي URL (مثال طور `https://api.openai.com/v1`)
   - **API چاٻي** — هن مهيا ڪندڙ لاءِ توهان جي API چاٻي
   - **ڊفالٽ ماڊل** — هن مهيا ڪندڙ لاءِ ڊفالٽ طور استعمال ٿيڻ وارو ماڊل سڃاڻپ ڪندڙ (مثال طور `gpt-4o`، `mistral-small`)
5. **سيٽنگون محفوظ ڪريو** تي ڪلڪ ڪريو

#### مهيا ڪندڙ هٽائڻ

ڪنهن به مهيا ڪندڙ ڪارڊ تي **هٽايو** بٽڻ تي ڪلڪ ڪري ان کي حذف ڪريو. تبديلي لاڳو ڪرڻ لاءِ سيٽنگون محفوظ ڪريو.

#### مهيا ڪندڙن کي ٻيهر ترتيب ڏيڻ

ڪنهن به مهيا ڪندڙ ڪارڊ جي کاٻي پاسي grip icon (⣿) کي ڇڪي ان کي ٻيهر ترتيب ڏيو. ترتيب fallback ترجيح طئي ڪري ٿي: درخواستون پهرين مهيا ڪندڙ ڏانهن موڪليون وڃن ٿيون ۽ جيڪڏهن درخواست ناڪام ٿئي ته پوءِ ايندڙ مهيا ڪندڙن ڏانهن fallback ٿين ٿيون.

### Fallback Routing

جڏهن بنيادي مهيا ڪندڙ ڏانهن درخواست ناڪام ٿئي — timeout، rate limit، يا service outage سبب — connector خودڪار طور فهرست ۾ ايندڙ مهيا ڪندڙ سان ٻيهر ڪوشش ڪري ٿو. اهو فهرست ۾ هيٺ هلندو رهي ٿو جيستائين ڪو مهيا ڪندڙ ڪامياب ٿئي يا سڀ مهيا ڪندڙ آزمائجي وڃن.

| صورتحال | رويو |
|----------|-----------|
| بنيادي مهيا ڪندڙ جواب ڏئي ٿو | درخواست عام طور پوري ڪئي وڃي ٿي |
| بنيادي مهيا ڪندڙ ناڪام ٿئي ٿو | درخواست ايندڙ مهيا ڪندڙ سان ٻيهر آزمائي وڃي ٿي |
| سڀ مهيا ڪندڙ ناڪام ٿين ٿا | غلطي caller ڏانهن واپس ڪئي وڃي ٿي |

Fallback سڏيندڙ plugin يا theme لاءِ شفاف آهي — انهن کي يا ته ڪامياب جواب ملي ٿو يا آخري غلطي، قطع نظر ته ڪيترا مهيا ڪندڙ آزمايا ويا.

### مهيا ڪندڙ جي ترتيب جو حوالو

| فيلڊ | گهربل | وضاحت |
|-------|----------|-------------|
| مهيا ڪندڙ جو نالو | ها | هن مهيا ڪندڙ لاءِ ڏيکارڻ وارو ليبل |
| Endpoint URL | ها | OpenAI-مطابقت رکندڙ API جو بنيادي URL |
| API چاٻي | ها | هن مهيا ڪندڙ لاءِ تصديق واري چاٻي |
| ڊفالٽ ماڊل | ها | جڏهن درخواست ۾ ڪو ماڊل بيان نه ڪيو وڃي ته استعمال ٿيندڙ ماڊل سڃاڻپ ڪندڙ |

## مسئلا حل ڪرڻ

### ڪو به مهيا ڪندڙ فهرست ۾ نه آهي

جيڪڏهن محفوظ ڪرڻ کان پوءِ مهيا ڪندڙن جي فهرست خالي هجي، ته تصديق ڪريو ته:

- مهيا ڪندڙ شامل ڪرڻ کان پوءِ توهان **سيٽنگون محفوظ ڪريو** تي ڪلڪ ڪيو
- توهان جي browser سيٽنگن واري صفحي کي محفوظ ٿيڻ کان نه روڪيو (JavaScript غلطين لاءِ browser console چيڪ ڪريو)
- plugin فعال آهي ۽ توهان نيٽ ورڪ admin اسڪرين تي آهيو

### سڀني مهيا ڪندڙن لاءِ درخواستون ناڪام ٿي رهيون آهن

- تصديق ڪريو ته هر مهيا ڪندڙ جو endpoint URL توهان جي server مان پهچ ۾ آهي (رڳو توهان جي browser مان نه)
- چيڪ ڪريو ته هر API چاٻي صحيح آهي ۽ ختم نه ٿي آهي
- انهن جي dashboard ۾ مهيا ڪندڙ جي rate-limit ۽ quota حالت جو جائزو وٺو
- connector مان تفصيلي غلطي پيغام پڪڙڻ لاءِ WordPress debug logging (`WP_DEBUG_LOG`) فعال ڪريو

### ڪو مهيا ڪندڙ fallback لاءِ استعمال نه ٿي رهيو آهي

Drag-to-reorder fallback ترتيب تبديل ڪري ٿو. تصديق ڪريو ته جنهن مهيا ڪندڙ کي توهان fallback درخواستون حاصل ڪرڻ جي اميد رکو ٿا، اهو فهرست ۾ صحيح جاءِ تي آهي ۽ ان جو endpoint URL ۽ API چاٻي صحيح آهن.

### نئون مهيا ڪندڙ ڪارڊ خودڪار طور نٿو کلي

خودڪار کولڻ لاءِ توهان جي browser ۾ JavaScript فعال هجڻ ضروري آهي. جيڪڏهن ڪارڊ ڪلڪ ڪرڻ تي نه کلي، ته غلطين لاءِ browser console چيڪ ڪريو. ڪارڊ جو مواد هميشه ڪارڊ header تي ڪلڪ ڪري رسائي ۾ هوندو آهي، ڀلي auto-expand trigger نه ٿئي.
