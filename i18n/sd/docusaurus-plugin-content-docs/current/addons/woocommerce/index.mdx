# WooCommerce انٽيگريشن {#woocommerce-integration}

WooCommerce انٽيگريشن بلڪل اهڙي ئي طريقي سان ڪم ڪري ٿي، جيئن اسان جو Manual Gateway ڪم ڪري ٿو. اها توهان جي يوزرن کي تمام گهڻي لچڪ فراهم ڪري ٿي، جنهن سان اهي توهان جي WooCommerce انسٽال تي موجود ڪنهن به ادائيگي فارم کي چونڊي سگهن ٿا. ۽ اهو سادو آهي: جڏهن يوزر جي بلنگ مدت ختم ٿئي ٿي، ته يوزر سبسڪرپشن هولڊ تي رکي وڃي ٿي. ساڳئي وقت، نيٽ ورڪ جي مکيه سائيٽ جي WooCommerce انسٽال تي هڪ آرڊر ٺاهيو ويندو. پوءِ يوزر کي سندن اي ميل ۾ انوائس ملي ٿي، جنهن ۾ ادائيگي ڪرڻ لاءِ لنڪ هوندي آهي. جڏهن ادائيگي وصول ٿي وڃي ٿي، ته سبسڪرپشن نئين ڪئي ويندي آهي.

## خاصيتون {#features}

- Ultimate Multisite ۽ WooCommerce جي وچ ۾ آسان انٽيگريشن
- ڪيترائي ادائيگي طريقا موجود
- شامل ڪيل سبسڪرپشنز ڪور استعمال ڪندي پاڻمرادو ورجائتي بلنگ، اختياري WooCommerce Subscriptions plugin بهتري سان
- چيڪ آئوٽ ٽوٽلز ۾ Ultimate Multisite ٽيڪس الڳ، غير ٽيڪس لائق WooCommerce فيس لائينز طور شامل هوندا آهن، ته جيئن گراهڪ ادائيگي کان اڳ ٽيڪس رقمون ڏسي سگهن

## دستاويز {#documentation}

**WooCommerce انٽيگريشن سيٽ اپ ڪرڻ**

**Ultimate Multisite سان پنهنجا ادائيگي اختيار وڌائڻ: WooCommerce انٽيگريشن**

## انسٽاليشن {#installation}

1. 'ultimate-multisite-woocommerce' کي '/wp-content/plugins/' ڊائريڪٽري ۾ اپلوڊ ڪريو
2. WordPress ۾ 'Plugins' مينيو ذريعي plugin کي چالو ڪريو
3. plugin سيٽ اپ ڪرڻ لاءِ قدم بہ قدم Wizard جي پيروي ڪريو

## استعمال گائيڊ {#usage-guide}

_**اهم نوٽ: هي مضمون Ultimate Multisite ورزن 2.x بابت آهي.**_

_**ڌيان:** Ultimate Multisite: WooCommerce انٽيگريشن لاءِ ضروري آهي ته WooCommerce گهٽ ۾ گهٽ توهان جي مکيه سائيٽ تي چالو هجي._

اسان سمجهون ٿا ته _Stripe_ ۽ _PayPal_ ڪجهه ملڪن ۾ موجود ناهن، جنهن ڪري Ultimate Multisite يوزرن لاءِ اسان جو plugin اثرائتي نموني استعمال ڪرڻ محدود يا ڏکيو ٿي وڃي ٿو. تنهنڪري اسان **_WooCommerce_ کي ضم ڪرڻ لاءِ هڪ add-on** ٺاهيو.

_WooCommerce_ هڪ تمام مشهور eCommerce plugin آهي. دنيا ڀر جي ڊولپرن ان سان مختلف ادائيگي گيٽ ويز ضم ڪرڻ لاءِ add-ons ٺاهيا. اسان انهيءَ مان فائدو وٺي اهي ادائيگي گيٽ ويز وڌايا، جيڪي توهان Ultimate Multisite بلنگ سسٽم ۾ استعمال ڪري سگهو ٿا.

پهريان، توهان کي WooCommerce Integration add-on انسٽال ڪرڻو پوندو، جيڪو **لائسنس يافته يوزرن لاءِ مفت آهي**. ان کي انسٽال ڪرڻ لاءِ، پنهنجي super admin Dashboard تي Ultimate Multisite تي ڪلڪ ڪريو ۽ **Add-ons** چونڊيو.

![Add-ons صفحو جيڪو WooCommerce Integration ڏيکاري ٿو](/img/admin/addons-page.png)

add-on تفصيلن سان هڪ ونڊو کلي ايندي. صرف **Install Now** تي ڪلڪ ڪريو *.*

### چيڪ آئوٽ ٽيڪس فيس لائينز {#checkout-tax-fee-lines}

جڏهن Ultimate Multisite چيڪ آئوٽ ۾ ٽيڪس شامل هوندا آهن، ته انٽيگريشن هر Ultimate Multisite ٽيڪس رقم کي WooCommerce چيڪ آئوٽ ۾ هڪ الڳ، غير ٽيڪس لائق فيس لائين طور شامل ڪري ٿي. هي ادائيگي کان اڳ ٽيڪس ٽوٽل کي ظاهر رکي ٿو، جڏهن ته WooCommerce کي ٽيڪس فيس تي ٻيهر ٽيڪس لڳائڻ کان روڪي ٿو.

## ادائيگي کي ڪيئن منظم ڪجي {#how-to-manage-the-payment}

WooCommerce انٽيگريشن بلڪل اهڙي ئي طريقي سان ڪم ڪري ٿي، جيئن اسان جو **Manual Gateway** ڪم ڪري ٿو. ڪمزوري اها آهي ته هر ادائيگي بلنگ مدت جي آخر ۾ هٿ سان ادا ڪرڻي پوندي آهي. وهڪرو هن ريت ڪم ڪري ٿو:

  * جڏهن يوزر جو بلنگ چڪر ختم ٿئي ٿو، ته ميمبرشپ **on hold** رکي وڃي ٿي. ساڳئي وقت، توهان جي نيٽ ورڪ جي مکيه سائيٽ تي _WooCommerce_ انسٽال ۾ هڪ آرڊر ٺاهيو ويندو.

  * پوءِ يوزرن کي سندن اي ميل ۾ انوائس ملي ٿي، جنهن ۾ ادائيگي ڪرڻ لاءِ لنڪ هوندي آهي (اهو لنڪ سندن سب سائيٽ Dashboard مان سندن بلنگ تاريخ ۾ به ظاهر ٿئي ٿو). اهو لنڪ يوزر کي _**WooCommerce**_ **checkout** ڏانهن ري ڊائريڪٽ ڪري ٿو، جتي يوزر پنهنجي ادائيگي پروسيس ڪرڻ لاءِ **موجود _WooCommerce_ گيٽ ويز** مان هڪ چونڊي سگهندا.

  * جڏهن ادائيگي وصول ٿي وڃي ٿي، ته ميمبرشپ **نئين** ڪئي وڃي ٿي، ادائيگي Ultimate Multisite تي لاگ ڪئي وڃي ٿي ۽ آرڊر مڪمل طور سيٽ ڪيو وڃي ٿو.

### ڪجهه اهم نوٽس: {#some-important-notes}

  * Ultimate Multisite جي ڊفالٽ فنڪشنلٽي اڃا به هن انٽيگريشن فارم سان ڪم ڪري ٿي. مثال طور، **Subscription Management** اسڪرين مان **ريفنڊ جاري ڪرڻ** لاءِ WooCommerce سان رابطو ڪيو ويندو، ته جيئن چونڊيل ادائيگي فارم استعمال ڪندي ريفنڊ جاري ڪري سگهجي.

  * Ultimate Multisite: WooCommerce Integration ڪم ڪرڻ لاءِ WP-Cron تي ڀاڙي ٿي. اڪثر وقت اهو تمام سٺو ڪم ڪري ٿو، پر جيڪڏهن سائيٽ تي ٽريفڪ گهٽ هجي ته cron شيڊول ڪيل واقعن جي چڪاس لاءِ متحرڪ نٿو ٿئي ۽ واقعو رهجي وڃي ٿو، جنهن ڪري عملدرآمد ۾ دير ٿي سگهي ٿي.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

هر مهيني ادائيگي کي هٿ سان شروع ڪرڻ آخري استعمال ڪندڙن لاءِ مثالي نه آهي، ڇاڪاڻ ته subscriptions کان توقع هوندي آهي ته اهي هر بلنگ چڪر ۾ پاڻمرادو ٻيهر ٿين. WooCommerce Integration add-on هاڻي بنيادي subscriptions ڪوڊ کي گڏ ڪري ٿو، تنهنڪري recurring products هميشه recurring billing لاءِ subscriptions استعمال ڪن ٿا.

توهان کي هن integration ذريعي recurring products وڪڻڻ لاءِ الڳ [WooCommerce Subscriptions پلگ ان](https://woocommerce.com/products/woocommerce-subscriptions/) خريد يا انسٽال ڪرڻ جي **ضرورت نه** آهي. جيڪڏهن اهو پلگ ان خريد، انسٽال، ۽ توهان جي مکيه سائيٽ تي موجود هجي، ته Ultimate Multisite ان کي استعمال ڪندو ۽ WooCommerce اضافي subscription management خاصيتون ظاهر ڪندو. جڏهن اهو موجود نه هجي، bundled subscriptions core اڃا به recurring products کي سنڀاليندو آهي.

![WooCommerce subscription integration سيٽنگون](/img/addons/woocommerce-subscription-integration.png)

۽ بس اهو ئي آهي. توهان جي آخري استعمال ڪندڙن کي recurring products لاءِ پنهنجي بلنگ چڪر جي آخر ۾ هاڻي هٿ سان ادائيگي ڪرڻ جي ضرورت نه پوندي.

توهان جي مکيه سائيٽ تي _WooCommerce_ هيٺ هڪ order ٺهڻ کان علاوه، اهو هر Account لاءِ هڪ subscription پڻ ٺاهيندو.

### ناڪام renewal suspension رويو {#failed-renewal-suspension-behavior}

ڊفالٽ طور، memberships فعال رهنديون آهن جڏهن WooCommerce Subscriptions پنهنجي ناڪام ادائيگي جي ٻيهر ڪوشش واري شيڊول کي عمل ۾ آڻي ٿو. هي عام WooCommerce Subscriptions grace window کي برقرار رکي ٿو، ته جيئن بحال ٿيل renewal subscription ۽ membership کي وقت کان اڳ رسائي ۾ خلل وجهڻ کان سواءِ ٻيهر هم وقت ڪري سگهي.

اهي administrators جيڪي وڌيڪ سخت رسائي ڪنٽرول چاهين ٿا، WooCommerce Integration سيٽنگن ۾ **Suspend Memberships Immediately on Failed Renewals** فعال ڪري سگهن ٿا. جڏهن هي opt-in سيٽنگ فعال هجي، Ultimate Multisite renewal payment ناڪام ٿيندي ئي لاڳاپيل membership کي suspend ڪري ٿو، WooCommerce Subscriptions retry handling جي مڪمل ٿيڻ جو انتظار ڪرڻ بدران.

Developers ساڳئي فيصلي کي `wu_woo_suspend_on_payment_failure` filter سان override ڪري سگهن ٿا. Hook reference صفحا add-on source مان generate ڪيا ويندا آهن، تنهنڪري هي overview custom suspension policies لاءِ صرف filter name کي نمايان ڪري ٿو.
