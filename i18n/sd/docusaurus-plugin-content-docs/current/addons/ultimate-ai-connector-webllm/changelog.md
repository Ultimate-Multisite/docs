# Ultimate AI Connector for WebLLM تبديلي لاگ

## نسخو 1.1.0 — 2026-04-09 تي جاري ڪيو ويو {#version-110--released-on-2026-04-09}

* نئون: ايڊمن-بار حالت اشارڪ سان ترندڙ چيٽ وجٽ — ڪو به لاگ اِن ٿيل استعمال ڪندڙ فرنٽ اينڊ مان سڌو برائوزر-پاسي LLM کي پرامپٽ ڪري سگهي ٿو.
* نئون: SharedWorker رن ٽائيم — MLC انجن هاڻي SharedWorker ۾ هلندو آهي، تنهنڪري گهڻا ٽيب هڪ GPU سيشن شيئر ڪن ٿا، ان تي وڙهڻ بدران.
* نئون: apiFetch وچولو انٽرسيپٽر — WordPress REST درخواستون جيڪي AI Client SDK نموني سان ملن ٿيون، شفاف نموني مقامي WebLLM بروڪر ڏانهن موڪليون وڃن ٿيون، لوپ بيڪ HTTP رائونڊ-ٽرپ جي ضرورت ناهي.
* نئون: Connector پينل ۾ وجٽ سيٽنگون UI، چيٽ وجٽ کي آن/آف ڪرڻ ۽ آٽو-پرامپٽ رويو ترتيب ڏيڻ لاءِ.
* درستگي: IndexedDB ڪيش بيڪ اينڊ کي مجبور ڪريو ته جيئن ماڊل وزن ڊائون لوڊون HuggingFace xet CDN ريڊائريڪٽس کان پوءِ به برقرار رهن، جيڪي ڊفالٽ Cache API رستي کي ٽوڙين ٿيون.
* درستگي: embedding ماڊلن لاءِ context_window KV-cache اووررائيڊ کي ڇڏي ڏيو (انهن وٽ decoder ناهي ۽ اووررائيڊ رن ٽائيم غلطي جو سبب بڻيو).
* درستگي: ورڪر ٽيب لوڊ ٿيڻ کان اڳ /webllm/v1/models ۾ cold-start اميدوار ماڊل جو اعلان ڪريو، ته جيئن SDK صارفين کي فوراً ماڊل نظر اچي.
* درستگي: WebLLM کي wpai_preferred_text_models filter ۾ Hook ڪريو، ته جيئن AI Experiments خصوصيت ترتيب ڏنل ماڊل هئڻ تي برائوزر انجن ڏانهن رُخ ڪري.
* بهتر ڪيو ويو: Cache-busting، مواد جي نارملائزيشن، ۽ هارڊويئر-حوالو درستگيون end-to-end ٽيسٽنگ دوران سامهون آيون.
