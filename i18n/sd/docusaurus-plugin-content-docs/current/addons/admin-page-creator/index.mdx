# ايڊمن صفحو ٺاهيندڙ

## مواد {#content}
ايڊمن صفحو ٺاهيندڙ توهان کي WordPress ايڊمن Dashboard اندر نوان ايڊمن صفحا ۽ ذيلي صفحا آساني سان شامل ڪرڻ جي اجازت ڏيندو. ان کي ڀليڪار صفحو يا سکيا وارو صفحو ٺاهڻ لاءِ استعمال ڪري سگهجي ٿو، جيڪو نون صارفن لاءِ لينڊنگ صفحي طور ڪم ڪري سگهي ٿو. توهان صفحن کي بنا تڪليف ڊزائن ڪرڻ لاءِ Beaver Builder، Elementor، Brizy، ۽ Oxygen جهڙا صفحو ٺاهيندڙ استعمال ڪري سگهو ٿا.

اهو ڪسٽم HTML، CSS ۽ JavaScript سپورٽ، گڏوگڏ پلان ۽ صارف ڪردار جي بنياد تي رسائي ڪنٽرول پڻ پيش ڪري ٿو!

## خاصيتون {#features}
* ڪسٽم HTML ۽ CSS جي سپورٽ
* JavaScript سپورٽ
* PHP مواد موڊ
* صارف ڪردار جي بنياد تي اجازت
* صفحو ٺاهيندڙن لاءِ سپورٽ

2. پنهنجي WordPress نيٽورڪ ايڊمن ۾ **پلگ اِنز** مينيو ذريعي پلگ اِن کي فعال ڪريو

## استعمال جي رهنمائي {#usage-guide}

جڏهن ته WordPress سائيٽن جي سامهون واري حصي ۾ مواد شامل ڪرڻ کي واقعي آسان بڻائي ٿو، ايڊمن پاسي صفحا ۽ ذيلي صفحا ٺاهڻ لاءِ ڪجهه بنيادي ڪوڊنگ مهارتن جي ضرورت پوي ٿي. Ultimate Multisite: ايڊمن صفحو ٺاهيندڙ جو مقصد توهان جي نيٽورڪ سائيٽن جي پٺئين حصي ۾ سامهون واري حصي جهڙي ساڳي آساني آڻڻ آهي.

## ايڊمن صفحو ٺاهيندڙ ڇو استعمال ڪجي؟ {#why-use-admin-page-creator}

شايد توهان پنهنجي ڪلائنٽن جي ايڊمن پينل تي سکيا وارن مواد سان هڪ ڪسٽم صفحو پيش ڪرڻ چاهيو ٿا، شايد توهان FAQs ۽ رابطي واري وجٽ سان هڪ ڪسٽم صفحو شامل ڪرڻ چاهيو ٿا. ايڊمن صفحو ٺاهيندڙ سان امڪان بي انتها آهن، ۽ اهو ئي اسان اڄ جي سکيا واري مواد ۾ ڏسنداسين.

## پنهنجو پهريون ڪسٽم ايڊمن صفحو ٺاهڻ {#creating-your-first-custom-admin-page}

Ultimate Multisite: ايڊمن صفحو ٺاهيندڙ انسٽال ۽ فعال ڪرڻ کان پوءِ، پنهنجي نيٽورڪ ايڊمن مينيو ڏانهن وڃو ۽ ايڊمن صفحن ڏانهن وڃو.

![نيٽورڪ ايڊمن تي ڪسٽم ايڊمن صفحن جي فهرست](/img/admin/admin-pages-list.png)

_نيٽورڪ ايڊمن تي ڪسٽم ايڊمن صفحن جي فهرست_

توهان کي هينئر تائين ٺاهيل سڀني ايڊمن صفحن جي فهرست نظر ايندي (جيئن ته هي توهان جو پهريون دورو آهي، فهرست خالي هوندي). اڳتي وڌو ۽ **نئون ايڊمن صفحو شامل ڪريو** تي ڪلڪ ڪريو. توهان کي ايڊمن صفحي جي ايڊيٽر ڏانهن موڪليو ويندو.

## ڪسٽم ايڊمن صفحي جا عنصر {#the-elements-of-a-custom-admin-page}

ايڊمن صفحي جي ايڊيٽر تي، توهان وٽ پنهنجي صفحي کي ڪسٽمائيز ڪرڻ لاءِ مختلف اختيارن جو انگ هوندو.

### صفحي جو عنوان {#page-title}

هي اهو عنوان آهي جيڪو صفحي جي مٿي تي ڏيکاريو ويندو، ايڊمن مينيو ليبل نه.

_صفحي جو عنوان داخل ڪريو_

### مواد جي قسم چونڊيندڙ {#content-type-selector}

Ultimate Multisite: ايڊمن صفحو ٺاهيندڙ توهان کي صفحي جي مواد جي حوالي سان ٻه اختيار ڏئي ٿو: توهان يا ته ڊفالٽ WordPress WYSIWYG ايڊيٽر استعمال ڪري سگهو ٿا يا HTML ايڊيٽر ڏانهن تبديل ٿي سگهو ٿا.

جيڪڏهن توهان HTML سان واقف نه آهيو، ته ڊفالٽ WordPress ايڊيٽر تي قائم رهو. توهان صفحي جي مواد کي بلڪل اهڙي طرح ايڊٽ ڪري سگهندا جيئن توهان عام پوسٽ يا صفحي سان ڪندا آهيو.

_ڊفالٽ WordPress WYSIWYG ايڊيٽر استعمال ڪندي_

## مينيو جا اختيار {#menu-options}

پاسي تي، General Options ميٽا-باڪس ۾، توهان مينيو جو قسم، ليبل، ۽ گهڻو ڪجهه ترتيب ڏئي سگهندا.

### مينيو جا قسم {#menu-types}

توهان جي نئين ڪسٽم ايڊمن صفحي وٽ ٻه مختلف مينيو قسم ٿي سگهن ٿا: مٿئين سطح جو مينيو ۽ ذيلي مينيو.

مٿئين سطح جو مينيو صفحو، جيئن نالو ٻڌائي ٿو، توهان جي نئين ٺاهيل صفحي کي ڊفالٽ WordPress صفحن سان گڏ ايڊمن مينيو ۾ شامل ڪندو.

جيڪڏهن توهان مٿئين سطح وارو اختيار چونڊيو ٿا، ته توهان مينيو جي ترتيب ۽ مينيو آئڪن پڻ چونڊي سگهندا.

جيڪڏهن توهان هن صفحي کي ذيلي مينيو صفحو بڻائڻ چاهيو ٿا، ته توهان هڪ والدين صفحو چونڊي سگهندا جنهن سان هي صفحو ڳنڍجي. توهان پنهنجو ڪسٽم صفحو ڊفالٽ WordPress صفحن يا پنهنجي مٿئين سطح جي ڪسٽم ايڊمن صفحن ۾ شامل ڪري سگهو ٿا.

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ مينيو قسم جا اختيار ليبل، ترتيب، ۽ آئڪن سيٽنگن سان -->

_هن ڪسٽم صفحي لاءِ مينيو آئٽم کي مڪمل طور ڪسٽمائيز ڪريو_

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ ذيلي مينيو والدين صفحي جي چونڊ -->

_توهان هي صفحو ٻين مينيو صفحن جي ذيلي مينيو طور پڻ شامل ڪري سگهو ٿا، جن ۾ توهان جا پنهنجا ڪسٽم مٿئين سطح وارا صفحا پڻ شامل آهن._

## ترقي يافته اختيار {#advanced-options}

توهان کي پنهنجي ڪسٽم ايڊمن صفحن ۾ هر قسم جو مواد شامل ڪرڻ جي قابل هئڻ گهرجي. ان ۾ CSS قاعدا ۽ فائلون، JavaScript، ۽ ٻاهريون لائبريريون پڻ شامل آهن. توهان اهو Advanced Options ٽيب استعمال ڪندي ڪري سگهو ٿا.

### ڪسٽم CSS {#custom-css}

CSS ٽيب تي، توهان پنهنجو ڪسٽم CSS ڪوڊ شامل ڪري سگهو ٿا، گڏوگڏ پري وارين سائيٽن مان CSS فائلون امپورٽ ڪري سگهو ٿا (جهڙوڪ CDNs، Google Fonts، وغيره).

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ ڪسٽم CSS ايڊيٽر ٻاهرين stylesheet امپورٽ سان -->

_ڪسٽم CSS پڻ سپورٽ ٿيل آهي. توهان ٻاهريون اسٽائل فائلون پڻ شامل ڪري سگهو ٿا!_

### ڪسٽم JavaScript {#custom-javascript}

ساڳيو JavaScript لاءِ به درست آهي. توهان پنهنجي ڪسٽم JavaScript ڪوڊ ۾ استعمال ڪرڻ لاءِ ٻاهريون لائبريريون شامل ڪري سگهو ٿا.

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ ڪسٽم JavaScript ايڊيٽر ٻاهرين لائبريري امپورٽ سان -->

_پنهنجو JavaScript ڪوڊ شامل ڪريو!_

## اجازتون {#permissions}

Advanced Options ۾ پڻ، توهان چونڊي سگهندا ته ڪهڙين ذيلي سائيٽن کي هن صفحي تائين رسائي هوندي. اهو مفيد آهي جيڪڏهن توهان مختلف پلان سطحن کي مختلف خاص مواد فراهم ڪرڻ جو ارادو رکو ٿا، مثال طور.

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ اجازت سيٽنگون پلان ۽ ڪردار جي بنياد تي رسائي ڪنٽرول لاءِ -->

_توهان وٽ مڪمل ڪنٽرول آهي ته هي صفحو ڪٿي ظاهر ٿيندو_

**اهم** : صفحي کي ڏيکارڻ لاءِ ٻئي شرطون پوريون ٿيڻ گهرجن. ان جو مطلب آهي ته جيڪڏهن استعمال ڪندڙ ڪنهن ڏنل پلان جو ڪلائنٽ آهي، پر چونڊيل ڪردارن مان ڪو هڪ ڪردار نٿو رکي (يا ان جي ابتڙ) ته صفحو سندس پينل تي ظاهر نه ٿيندو.

## فعال {#active}

جيڪڏهن توهان ڪنهن ڪسٽم ايڊمن صفحي کي حذف ڪرڻ يا اجازتن سان ڇيڙڇاڙ ڪرڻ کانسواءِ غيرفعال ڪرڻ چاهيو ٿا، ته بس فعال آپشن کي اَن چيڪ ڪريو.

مٿئين سطح جي صفحي کي غيرفعال ڪرڻ سان ان والدين صفحي سان ڳنڍيل سڀ ذيلي صفحا پاڻمرادو غيرفعال ٿي ويندا.

_استعمال ڪندڙ جي پينل تي آخري نتيجو!_
