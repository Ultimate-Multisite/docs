# થીમ બિલ્ડર ઓનબોર્ડિંગ ફ્લો (Theme Builder Onboarding Flow)

Superdav AI Agent v1.12.0 માં એક માર્ગદર્શિત **થીમ બિલ્ડર ઓનબોર્ડિંગ ફ્લો** રજૂ કરવામાં આવ્યું છે જે તમને તમારી શરૂઆતની સેટઅપ દરમિયાન કસ્ટમ બ્લોક થીમ બનાવવામાં મદદ કરે છે. આ જૂના Site Builder મોડને વધુ લવચીક, એજન્ટ-સહાયિત અભિગમ સાથે બદલે છે.

## થીમ બિલ્ડર ઓનબોર્ડિંગ ફ્લો શું છે? {#what-is-the-theme-builder-onboarding-flow}

થીમ બિલ્ડર ઓનબોર્ડિંગ ફ્લો એક ઇન્ટરેક્ટિવ સેટઅપ વિઝાર્ડ (setup wizard) છે જે:

- ડિઝાઇનના નિર્ણયો (રંગો, ફોન્ટ સ્ટાઇલ, લેઆઉટ) માં તમને માર્ગદર્શન આપે છે
- તમારી સાઇટની દ્રશ્ય ઓળખની પસંદગીઓ પકડે છે
- તમારી જરૂરિયાતો અનુસાર કસ્ટમ બ્લોક થીમ બનાવે છે
- પૂર્ણ થતાં તે થીમને આપમેળે સક્રિય કરે છે

આ ફ્લો **Setup Assistant agent** દ્વારા ચલાવવામાં આવે છે, જે સ્પષ્ટતા માટેના પ્રશ્નો પૂછે છે અને તમારા થીમને ક્રમશઃ બનાવે છે.

## થીમ બિલ્ડર ઓનબોર્ડિંગ શરૂ કરવું {#starting-the-theme-builder-onboarding}

### પ્રથમ વખત સેટઅપ (First-Run Setup) {#first-run-setup}

જ્યારે તમે નવી WordPress ઇન્સ્ટોલેશન પર Superdav AI Agent ને પહેલીવાર લોન્ચ કરશો, ત્યારે તમને આ દેખાશે:

```
Superdav AI Agent માં આપનું સ્વાગત છે!

તમે શું કરવા માંગો છો?
1. કસ્ટમ થીમ બનાવો (Theme Builder)
2. હાલની થીમનો ઉપયોગ કરો
3. અત્યારે સેટઅપ છોડી દો
```

થીમ બિલ્ડર ઓનબોર્ડિંગ ફ્લોમાં પ્રવેશવા માટે **"Build a custom theme"** પસંદ કરો.

### મેન્યુઅલ સક્રિયકરણ (Manual Activation) {#manual-activation}

તમે કોઈપણ સમયે નીચે મુજબ વિનંતી કરીને થીમ બિલ્ડર ઓનબોર્ડિંગ શરૂ કરી શકો છો:

```
"Start the Theme Builder onboarding"
```

અથવા

```
"Help me create a custom theme"
```

## ઓનબોર્ડિંગના પગલાં (The Onboarding Steps) {#the-onboarding-steps}

### પગલું ૧: મોડની પસંદગી (Mode Selection) {#step-1-mode-selection}

Setup Assistant agent તમારી પસંદગી વિશે પૂછશે:

```
તમે તમારી થીમ કેવી રીતે બનાવવી માંગો છો?
- માર્ગદર્શિત (Guided) (હું પ્રશ્નો પૂછીશ અને તમારા માટે તેને બનાવીશ)
- હાથથી કરવું (Hands-on) (હું તમને વિકલ્પો બતાવીશ અને તમે નિર્ણય લેશો)
```

મોટાભાગના વપરાશકર્તાઓ માટે **Guided mode** ભલામણ કરેલ છે; એજન્ટ તમારી ઉદ્યોગ અને લક્ષ્યોના આધારે ડિઝાઇન સંબંધિત સૂચનો કરશે.

### પગલું ૨: સાઇટની વિશિષ્ટતાઓ (Site Specification) {#step-2-site-specification}

તમને તમારી સાઇટ વિશે પૂછવામાં આવશે:

- **वेबसाइटको उद्देश्य**: ई-कमर्स, ब्लग, पोर्टफोलियो, SaaS आदि।
- **लक्ष्य दर्शक**: तपाईंका आगन्तुक को हुन्?
- **ब्राण्डका रंग**: प्राथमिक र द्वितीयक रंग (वा "मलाई थाहा छैन")
- **स्वर (Tone)**: व्यावसायिक, रचनात्मक, चञ्चल, न्यूनतम, आदि।

यो जानकारी तपाईंको **site_brief** मेमोरीमा भण्डारण गरिएको हुन्छ, जसलाई भविष्यका सत्रहरूमा एजेन्टहरूले सन्दर्भ गर्छन्।

### चरण ३: डिजाइन सिस्टम निर्णयहरू {#step-3-design-system-decisions}

एजेन्टले तपाईंलाई डिजाइन टोकन (design token) चयन गर्न मार्गदर्शन गर्छ:

- **टाइपोग्राफी**: फन्ट परिवार (serif, sans-serif, monospace) र साइजको क्रम
- **रंग योजना**: प्राथमिक, द्वितीयक, उच्चारण (accent), र तटस्थ रंगहरू
- **स्पेसिंग (Spacing)**: सानो, सामान्य, वा प्रशस्त लेआउटहरू
- **मोशन (Motion)**: एनिमेसन र ट्रान्जिशनहरू (यदि चाहनुहुन्छ भने)

### चरण ४: थीम निर्माण {#step-4-theme-generation}

सेटअप असिस्टेन्ट एजेन्टले तपाईंको कस्टम ब्लक थीमलाई निम्न कुराहरूसँग ढाँचा (scaffold) दिन्छ:

- सबै डिजाइन टोकनहरू समावेश भएको `theme.json`
- सामान्य लेआउटहरूको लागि ब्लक टेम्प्लेट (होमपेज, ब्लग, सम्पर्क)
- तपाईंको डिजाइन सिस्टमसँग मेल खाने कस्टम ब्लक शैलीहरू
- थीम मेटाडेटा र वर्डप्रेस समर्थन घोषणाहरू

### चरण ५: सक्रियता र प्रमाणीकरण (Activation and Verification) {#step-5-activation-and-verification}

थीम स्वचालित रूपमा सक्रिय हुन्छ, र तपाईंले निम्न कुराहरू देख्नुहुनेछ:

```
✓ तपाईंको कस्टम थीम अब चल्दैछ!
  Theme name: [Your Site Name] Theme
  Colors: [Primary] / [Secondary]
  Typography: [Font Family]

  नयाँ डिजाइन हेर्न आफ्नो साइटमा जानुहोस्।
```

त्यसपछि, तपाईंले थीम सही तरिकाले प्रदर्शन गरिरहेको छ कि छैन भनेर पुष्टि गर्न आफ्नो साइटमा जान सक्नुहुन्छ।

## साइट विवरण र site_brief मेमोरी {#site-specification-and-sitebrief-memory}

ऑनबोर्डिङको समयमा, एजेन्टले तपाईंको साइटको विवरणलाई **site_brief** मेमोरी श्रेणीमा लिन्छ। यसमा निम्न कुराहरू समावेश हुन्छन्:

- साइटको उद्देश्य र लक्ष्यहरू
- लक्षित दर्शक
- ब्राण्डका रंग र स्वर
- डिजाइन प्राथमिकताहरू
- सामग्री संरचना (Content structure)

### site_brief किन महत्त्वपूर्ण छ? {#why-sitebrief-matters}

भविष्यका सत्रहरूमा, एजेन्टहरूले तपाईंको site_brief लाई निम्न कुराहरूको लागि सन्दर्भ गर्छन्:

### आफ्नो साइटको संक्षिप्त जानकारी हेर्नुहोस् (Viewing Your site_brief) {#viewing-your-sitebrief}

तपाईं एजेन्टलाई यसो सोध्न सक्नुहुन्छ:

```
"Show me my site brief"
```

वा

```
"What do you know about my site?"
```

एजेन्टले तपाईंको भण्डार गरिएको साइट विवरण देखाउनेछ।

## सेटअप पछि अनुकूलन गर्ने (Customizing After Onboarding) {#customizing-after-onboarding}

Theme Builder को सेटअप पूरा भएपछि, तपाईं निम्न कुराहरू गर्न सक्नुहुन्छ:

### डिजाइन सिस्टम एस्थेटिक्स स्किल प्रयोग गर्नुहोस् (Use the Design System Aesthetics Skill) {#use-the-design-system-aesthetics-skill}

डिजाइन सुधारका लागि अनुरोध गर्नुहोस्:

```
"Refine the typography to be more modern"
```

वा

```
"Adjust the color palette to be warmer"
```

**Design System Aesthetics skill** ले तपाईंलाई लक्षित डिजाइन अपडेटहरूमा मार्गदर्शन गर्छ।

### theme.json मा सिधै सम्पादन गर्नुहोस् (Edit theme.json Directly) {#edit-themejson-directly}

अत्याधुनिक प्रयोगकर्ताहरूको लागि, रंग टोकनहरू, टाइपोग्राफी स्केलहरू, स्पेसिसङ मानहरू, र बर्डर तथा शैडो परिभाषाहरू मिलाउन `/wp-content/themes/[theme-name]/theme.json` लाई सम्पादन गर्नुहोस्।

### कस्टम ब्लक टेम्प्लेटहरू बनाउनुहोस् (Create Custom Block Templates) {#create-custom-block-templates}

WordPress block editor प्रयोग गरेर निम्नका लागि कस्टम टेम्प्लेटहरू बनाउन सक्नुहुन्छ:

- होमपेज लेआउटहरू
- ब्लग पोस्ट पेजहरू
- उत्पादन पेजहरू
- सम्पर्क फारमहरू

## तुलना: पुरानो बनाम नयाँ सेटअप (Comparison: Old vs. New Onboarding) {#comparison-old-vs-new-onboarding}

| विशेषता | साइट बिल्डर (पुरानो) | थीम बिल्डर (नयाँ) |
|---|---|---|
| सेटअप विधि | विजार्ड-आधारित फारम | एजेन्ट-निर्देशित कुराकानी |
| थीम निर्माण | सीमित टेम्प्लेटहरू | कस्टम स्क्याफोल्डिङ |
| डिजाइन टोकनहरू | म्यानुअल प्रविष्टि | निर्देशित निर्णयहरू |
| लचिलोपन | निश्चित विकल्पहरू | अनुकूलन गर्न सकिने |
| भविष्यका अपडेटहरू | सन्दर्भ गरिएका छैनन् | site_brief मा भण्डार गरिएको छ |

## समस्या समाधान (Troubleshooting) {#troubleshooting}

**सेटअप प्रवाह पूरा भएन (The onboarding flow didn't complete)**
- प्रवाह पुनः सुरु गर्नुहोस्: "Start the Theme Builder onboarding"
- तपाईंको WordPress इन्स्टलेसन अद्यावधिक भएको छ कि छैन जाँच गर्नुहोस्।
- सेटअप असिस्टेन्ट एजेन्ट सक्रिय छ कि छैन प्रमाणित गर्नुहोस्।

**मेरो साइट_ब्रीफको प्रयोग भइरहेको छैन**
- पुष्टि गर्नुहोस् कि एजेन्टलाई मेमोरीमा पहुँच छ।
- एजेन्टलाई "मेरो साइट ब्रीफ सम्झनुहोस्" (recall my site brief) भनेर सोध्नुहोस्।
- तपाईंको सेटिङहरूमा मेमोरी सक्रिय छ वा छैन, त्यो जाँच गर्नुहोस्।

**उत्पन्न गरिएको थीम मेरो प्राथमिकताहरूसँग मेल खाँदैन**
- यसलाई परिष्कृत गर्न Design System Aesthetics skill प्रयोग गर्नुहोस्।
- एजेन्टलाई "[विशिष्ट परिवर्तनहरू] सहित थीम पुनः उत्पन्न गर्नुहोस्" (regenerate the theme with [specific changes]) भनेर सोध्नुहोस्।
- सटीक नियन्त्रणका लागि `theme.json` लाई सिधै सम्पादन गर्नुहोस्।

## अर्को चरणहरू {#next-steps}

Theme Builder को सेटअप पूरा गरेपछि:

1. **तपाईंको साइट प्रमाणित गर्नुहोस्**: नयाँ थीम हेर्न आफ्नो साइटमा जानुहोस्।
2. **डिजाइन परिष्कृत गर्नुहोस्**: समायोजन गर्न Design System Aesthetics skill प्रयोग गर्नुहोस्।
3. **सामग्री थप्नुहोस्**: आफ्नो साइटको सामग्री बनाउन सुरु गर्नुहोस्।
4. **क्षमताहरू अन्वेषण गर्नुहोस्**: scaffold-block-theme र activate-theme जस्ता अन्य एजेन्ट क्षमताहरूको बारेमा सिक्नुहोस्।
