# ڊومين ميپنگ ڪيئن ترتيب ڏجي (v2)

_**اهم نوٽ: هي مضمون Ultimate Multisite ورزن 2.x بابت آهي.**_

هڪ پريميئم نيٽ ورڪ جي سڀ کان طاقتور خصوصيتن مان هڪ اها صلاحيت آهي ته اسان پنهنجن گراهڪن کي سندن سائيٽن سان ٽاپ-ليول ڊومين ڳنڍڻ جو موقعو ڏيون. آخرڪار، ڪهڙو وڌيڪ پروفيشنل لڳي ٿو: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) يا [_**joesbikeshop.com**_](http://joesbikeshop.com)؟ انهيءَ ڪري Ultimate Multisite اها خصوصيت اندر ئي فراهم ڪري ٿو، ٽئين ڌر جي plugins استعمال ڪرڻ جي ضرورت کان سواءِ.

## ڊومين ميپنگ ڇا آهي؟ {#whats-domain-mapping}

جيئن نالي مان ظاهر آهي، ڊومين ميپنگ اها صلاحيت آهي جيڪا Ultimate Multisite طرفان فراهم ڪئي ويندي آهي ته اها ڪسٽم ڊومين لاءِ درخواست وٺي ۽ ان درخواست کي نيٽ ورڪ ۾ لاڳاپيل سائيٽ سان ميپ ڪري، جنهن سان اهو خاص ڊومين ڳنڍيل هجي.

### پنهنجي Ultimate Multisite نيٽ ورڪ تي ڊومين ميپنگ ڪيئن سيٽ اپ ڪجي {#how-to-setup-domain-mapping-on-your-ultimate-multisite-network}

ڊومين ميپنگ کي ڪم ڪرڻ لاءِ توهان جي طرفان ڪجهه سيٽنگ ڪرڻي پوندي. خوش قسمتيءَ سان، Ultimate Multisite توهان لاءِ ڏکيو ڪم پاڻمرادو ڪري ٿو، ته جيئن توهان آسانيءَ سان گهرجون پوريون ڪري سگهو.

Ultimate Multisite انسٽاليشن دوران، وزارڊ پاڻمرادو **sunrise.php** کي مقرر ڪيل فولڊر ۾ ڪاپي ۽ انسٽال ڪندو. **وزارڊ توهان کي اڳتي وڌڻ نه ڏيندو جيستائين هي قدم مڪمل نه ٿئي**.

<!-- اسڪرين شاٽ موجود نه آهي: Ultimate Multisite انسٽاليشن وزارڊ sunrise.php قدم سان -->

ان جو مطلب آهي ته جڏهن Ultimate Multisite انسٽاليشن وزارڊ توهان جو نيٽ ورڪ سيٽ اپ ڪرڻ مڪمل ڪري ٿو، ته توهان فوراً ڪسٽم ڊومين ميپ ڪرڻ شروع ڪري سگهو ٿا.

نوٽ ڪريو ته Ultimate Multisite ۾ ڊومين ميپنگ لازمي نه آهي. توهان وٽ WordPress Multisite جي اصل ڊومين ميپنگ فنڪشن يا ڪنهن ٻي ڊومين ميپنگ حل کي استعمال ڪرڻ جو اختيار آهي.

جيڪڏهن توهان کي ٻين ڊومين ميپنگ حلن لاءِ Ultimate Multisite ڊومين ميپنگ غير فعال ڪرڻي پوي، ته توهان هي خصوصيت **Ultimate Multisite > Settings > Domain Mapping** هيٺ غير فعال ڪري سگهو ٿا.

![ڊومين ميپنگ سيٽنگن جو صفحو، جنهن ۾ ايڊمن ريڊائريڪٽ، ميپنگ پيغام ۽ DNS اختيار ڏيکاريل آهن](/img/config/domain-mapping-settings.png)

هن اختيار جي بلڪل هيٺ، توهان **Force Admin Redirect** اختيار به ڏسي سگهو ٿا. هي اختيار توهان کي ڪنٽرول ڪرڻ جي اجازت ڏئي ٿو ته ڇا توهان جا گراهڪ پنهنجو ايڊمن Dashboard پنهنجي ڪسٽم ڊومين ۽ سب ڊومين ٻنهي تي رسائي ڪري سگهندا يا صرف انهن مان هڪ تي.

جيڪڏهن توهان **Force redirect to mapped domain** چونڊيو ٿا، ته توهان جا گراهڪ صرف پنهنجن ڪسٽم ڊومينن تي پنهنجو ايڊمن Dashboard رسائي ڪري سگهندا.

اختيار **Force redirect to** **network domain** بلڪل ان جي ابتڙ ڪندو - توهان جا گراهڪ صرف پنهنجي سب ڊومين تي پنهنجا Dashboard رسائي ڪرڻ جي اجازت رکندڙ هوندا، ڀلي اهي پنهنجن ڪسٽم ڊومينن تي سائن اِن ڪرڻ جي ڪوشش ڪن.

۽ اختيار **Allow access to the admin by both mapped domain domain and network domain** انهن کي سب ڊومين ۽ ڪسٽم ڊومين ٻنهي تي پنهنجا ايڊمن Dashboard رسائي ڪرڻ جي اجازت ڏئي ٿو.

![ايڊمن ريڊائريڪٽ ڊراپ ڊائون کليل، جنهن ۾ ٽي ريڊائريڪٽ اختيار ڏيکاريل آهن](/img/config/domain-mapping-redirect-options.png)

ڪسٽم ڊومين ميپ ڪرڻ جا ٻه طريقا آهن. پهريون طريقو اهو آهي ته توهان پنهنجي نيٽ ورڪ ايڊمن Dashboard مان سپر ايڊمن طور ڊومين نالو ميپ ڪريو، ۽ ٻيو طريقو سب سائيٽ ايڊمن Dashboard ذريعي Account صفحي هيٺ آهي.

پر توهان پنهنجي نيٽ ورڪ جي ڪنهن سب سائيٽ سان ڪسٽم ڊومين ميپ ڪرڻ شروع ڪرڻ کان اڳ، توهان کي پڪ ڪرڻي پوندي ته ڊومين نالي جون **DNS settings** صحيح ترتيب ڏنل آهن.

###

### پڪ ڪرڻ ته ڊومين جون DNS سيٽنگون صحيح ترتيب ڏنل آهن {#making-sure-the-domain-dns-settings-are-properly-configured}

ميپنگ کي ڪم ڪرڻ لاءِ، توهان کي پڪ ڪرڻي پوندي ته جنهن ڊومين کي توهان ميپ ڪرڻ جو ارادو رکو ٿا، اهو توهان جي نيٽ ورڪ جي IP پتي ڏانهن اشارو ڪري رهيو آهي. نوٽ ڪريو ته توهان کي نيٽ ورڪ جو IP پتو گهرجي - يعني ان ڊومين جو IP پتو جتي Ultimate Multisite انسٽال ٿيل آهي - نه ته ان ڪسٽم ڊومين جو IP پتو جنهن کي توهان ميپ ڪرڻ چاهيو ٿا. ڪنهن خاص ڊومين جو IP پتو ڳولڻ لاءِ، مثال طور، اسان [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) تي وڃڻ جي صلاح ڏيون ٿا.

ڊومين کي صحيح طرح ميپ ڪرڻ لاءِ، توهان کي پنهنجي **DNS** ترتيب ۾ هڪ **A RECORD** شامل ڪرڻي پوندي، جيڪا انهي **IP address** ڏانهن اشارو ڪري. DNS انتظام مختلف ڊومين رجسٽرارن جي وچ ۾ تمام گهڻو مختلف هوندو آهي، پر جيڪڏهن توهان " _Creating A Record on XXXX_ " ڳوليندا، جتي XXXX توهان جو ڊومين رجسٽرار آهي (مثال: " _Creating A Record on_ _GoDaddy_ ")، ته ان بابت آن لائن گهڻا ٽيوٽوريل موجود آهن.

جيڪڏهن توهان کي هي ڪم ڪرائڻ ۾ ڏکيائي ٿئي، ته **پنهنجي ڊومين رجسٽرار سپورٽ سان رابطو ڪريو** ۽ اهي هن حصي ۾ توهان جي مدد ڪري سگهندا.

جيڪڏهن توهان پنهنجن گراهڪن کي پنهنجا ڊومين پاڻ ميپ ڪرڻ جي اجازت ڏيڻ جو ارادو رکو ٿا، ته هن حصي جو ڪم انهن کي پاڻ ڪرڻو پوندو. جيڪڏهن اهي A Record ٺاهڻ ۾ ناڪام ٿين، ته کين سندن رجسٽرار جي سپورٽ سسٽم ڏانهن رهنمائي ڪريو.

### سپر ايڊمن طور ڪسٽم ڊومين نالو ميپ ڪرڻ {#mapping-custom-domain-name-as-super-admin}

جڏهن توهان پنهنجي نيٽ ورڪ تي سپر ايڊمن طور لاگ اِن ٿيل آهيو، ته توهان **Ultimate Multisite > Domains** هيٺ وڃي آسانيءَ سان ڪسٽم ڊومين نالا شامل ۽ منظم ڪري سگهو ٿا.

![Ultimate Multisite ۾ ڊومينن جي فهرست وارو صفحو](/img/admin/domains-list.png)

هن صفحي هيٺ، توهان مٿي **Add Domain** بٽڻ تي ڪلڪ ڪري سگهو ٿا ۽ اهو هڪ موڊل ونڊو کوليندو، جتي توهان **custom domain name**، **the subsite** جنهن تي توهان ڪسٽم ڊومين نالو لاڳو ڪرڻ چاهيو ٿا، سيٽ ۽ ڀري سگهو ٿا، ۽ فيصلو ڪري سگهو ٿا ته ڇا توهان ان کي **primary domain** نالو بڻائڻ چاهيو ٿا يا نه (نوٽ ڪريو ته توهان **multiple domain names to one subsite** ميپ ڪري سگهو ٿا).

![Add Domain موڊل، جنهن ۾ ڊومين نالو، سائيٽ چونڊيندڙ ۽ بنيادي ڊومين ٽوگل آهي](/img/admin/domain-add-modal.png)

سڀ معلومات داخل ڪرڻ کان پوءِ، توهان پوءِ هيٺان **Add Existing Domain** بٽڻ تي ڪلڪ ڪري سگهو ٿا.

اهو ڪسٽم ڊومين جي DNS معلومات جي تصديق ۽ حاصل ڪرڻ جو عمل شروع ڪندو. توهان صفحي جي هيٺان هڪ لاگ به ڏسندا، ته جيئن توهان ان عمل جي پيروي ڪري سگهو جيڪو اهو گذري رهيو آهي. هي عمل مڪمل ٿيڻ ۾ ڪجهه منٽ وٺي سگهي ٿو.

Ultimate Multisite v2.13.0 پڻ اندروني ڊومين رڪارڊ پاڻمرادو ٺاهي ٿو جڏهن نئين سائيٽ اهڙي host تي ٺهي ٿي جنهن کي هر سائيٽ واري ڊومين طور ورتو وڃي. جيڪڏهن host نيٽ ورڪ جو بنيادي ڊومين آهي، يا **سائيٽ URL** فيلڊ تي ترتيب ڏنل گڏيل checkout-form بنيادي ڊومينن مان هڪ آهي، ته پاڻمرادو mapped-domain رڪارڊ ڇڏيو وڃي ٿو ته جيئن گڏيل بنيادي ڊومين ان کي استعمال ڪندڙ هر سائيٽ لاءِ موجود رهي.

جڏهن ڪو گراهڪ Domain Seller v1.3.0 يا ان کان نئين ذريعي نئون ڊومين رجسٽر ڪري ٿو، Ultimate Multisite ڊفالٽ طور رجسٽر ٿيل ڊومين کي گراهڪ جي نيٽ ورڪ سائيٽ سان پاڻمرادو ميپ ڪري ٿو. منتظمين کي ڪامياب رجسٽريشن کان پوءِ هاڻي الڳ mapped-domain رڪارڊ شامل ڪرڻ جي ضرورت ناهي، جيستائين اهي primary-domain flag، activation state، يا SSL handling جهڙن اختيارن کي ترتيب ڏيڻ نه چاهين.

**مرحلو** يا اسٽيٽس **DNS چيڪ ٿي رهيو آهي** کان **تيار** ۾ تبديل ٿيڻ گهرجي جيڪڏهن سڀ ڪجهه صحيح طرح سيٽ اپ ٿيل آهي.

<!-- اسڪرين شاٽ دستياب ناهي: ڊومينن جي فهرست ۾ Checking DNS مرحلو ڏيکاريندڙ ڊومين قطار -->

<!-- اسڪرين شاٽ دستياب ناهي: سائي اسٽيٽس اشارو سان Ready مرحلو ڏيکاريندڙ ڊومين قطار -->

جيڪڏهن توهان ڊومين نالي تي ڪلڪ ڪريو، ته توهان ان جي اندر ڪجهه اختيار ڏسي سگهندا. اچو ته انهن تي تڪڙي نظر وجهون:

![مرحلي، سائيٽ، فعال، بنيادي ۽ SSL ٽوگلز سان ڊومين تفصيل صفحو](/img/admin/domain-edit.png)

**مرحلو:** هي اهو مرحلو آهي جنهن تي ڊومين موجود آهي. جڏهن توهان پهريون ڀيرو ڊومين شامل ڪندا، ته اهو شايد **DNS چيڪ ٿي رهيو آهي** مرحلي تي هوندو. عمل DNS entries چيڪ ڪندو ۽ تصديق ڪندو ته اهي درست آهن. پوءِ، ڊومين کي **SSL چيڪ ٿي رهيو آهي** مرحلي تي رکيو ويندو. Ultimate Multisite چيڪ ڪندو ته ڊومين وٽ SSL آهي يا نه ۽ توهان جي ڊومين کي **تيار** يا **تيار (SSL کان سواءِ)** طور درجيبند ڪندو.

**سائيٽ:** اهو subdomain جيڪو هن ڊومين سان لاڳاپيل آهي. ميپ ٿيل ڊومين هن مخصوص سائيٽ جو مواد ڏيکاريندو.

**فعال:** توهان هن اختيار کي آن يا آف ڪري ڊومين کي فعال يا غيرفعال ڪري سگهو ٿا.

**ڇا بنيادي ڊومين آهي؟:** توهان جي گراهڪن وٽ هر سائيٽ لاءِ هڪ کان وڌيڪ ميپ ٿيل ڊومين ٿي سگهن ٿا. هي اختيار استعمال ڪريو ته چونڊيو ڇا هي مخصوص سائيٽ لاءِ بنيادي ڊومين آهي.

**ڇا محفوظ آهي؟:** جيتوڻيڪ Ultimate Multisite ڊومين کي فعال ڪرڻ کان اڳ چيڪ ڪري ٿو ته ڊومين وٽ SSL certificate آهي يا نه، توهان هٿ سان چونڊي سگهو ٿا ته ڊومين کي SSL certificate سان يا بغير لوڊ ڪيو وڃي. ياد رکو ته جيڪڏهن ويب سائيٽ وٽ SSL certificate ناهي ۽ توهان ان کي SSL سان زبردستي لوڊ ڪرڻ جي ڪوشش ڪريو، ته اهو توهان کي غلطيون ڏئي سگهي ٿو.

### ذيلي سائيٽ يوزر طور custom domain name ميپ ڪرڻ {#mapping-custom-domain-name-as-subsite-user}

ذيلي سائيٽ منتظمين پڻ پنهنجي ذيلي سائيٽ admin Dashboard مان custom domain names ميپ ڪري سگهن ٿا.

پهريان، توهان کي پڪ ڪرڻي آهي ته توهان هي اختيار **ڊومين ميپنگ** سيٽنگن هيٺ فعال ڪيو. هيٺ اسڪرين شاٽ ڏسو.

<!-- اسڪرين شاٽ دستياب ناهي: ڊومين ميپنگ سيٽنگون جيڪي ذيلي سائيٽ يوزرن کي Customer DNS Management ٽوگل ذريعي ڊومين ميپ ڪرڻ جي اجازت ڏين ٿيون -->

توهان هي اختيار **Plan** سطح يا **Ultimate Multisite > Products** تي پراڊڪٽ اختيارن هيٺ به سيٽ يا ترتيب ڏئي سگهو ٿا.

![پراڊڪٽ ايڊٽ صفحي تي Custom Domains سيڪشن](/img/config/product-custom-domains.png)

جڏهن انهن مان ڪو به اختيار فعال هجي ۽ ذيلي سائيٽ يوزر کي custom domain names ميپ ڪرڻ جي اجازت هجي، ته ذيلي سائيٽ يوزر کي **Account** صفحي هيٺ **ڊومينز** نالي metabox ڏسڻ گهرجي.

<!-- اسڪرين شاٽ دستياب ناهي: ذيلي سائيٽ Account صفحي تي Domains metabox جنهن ۾ Add Domain بٽڻ آهي -->

يوزر **ڊومين شامل ڪريو** بٽڻ تي ڪلڪ ڪري سگهي ٿو ۽ اهو ڪجهه هدايتن سان هڪ modal window کولي ڇڏيندو.

<!-- اسڪرين شاٽ دستياب ناهي: Add Domain modal جيڪو ذيلي سائيٽ يوزرن لاءِ DNS A-record هدايتون ڏيکاري ٿو -->

يوزر پوءِ **اڳيون قدم** تي ڪلڪ ڪري custom domain name شامل ڪرڻ لاءِ اڳتي وڌي سگهي ٿو. اهي اهو به چونڊي سگهن ٿا ته ڇا هي بنيادي ڊومين هوندو يا نه.

**ڊومين شامل ڪريو** تي ڪلڪ ڪرڻ custom domain جي DNS معلومات جي تصديق ۽ حاصل ڪرڻ جو عمل شروع ڪندو.

### Domain Syncing بابت {#about-domain-syncing}

Domain Syncing اهو عمل آهي جنهن ۾ Ultimate Multisite توهان جي hosting account ۾ custom domain name کي add-on domain طور شامل ڪري ٿو **ته جيئن domain mapping ڪم ڪري**.

Domain syncing پاڻمرادو ٿئي ٿي جيڪڏهن توهان جي hosting provider وٽ Ultimate Multisite domain mapping فيچر سان integration آهي. هن وقت، اهي hosting providers _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ ۽ _Cpanel_ آهن.

جڏهن host-provider integration فعال هجي، Ultimate Multisite نئين ٺهيل سائيٽن لاءِ provider-side DNS يا subdomain creation task کي به enqueue ڪري سگهي ٿو. جيڪڏهن ڪا integration انهي task لاءِ ٻڌندڙ نه هجي، ته background job ڇڏيو وڃي ٿو ته جيئن no-op queue entries کان بچي سگهجي. ميپ ٿيل ڊومينن لاءِ DNS ۽ SSL checks عام domain-stage process ذريعي هلندا رهن ٿا.

توهان کي Ultimate Multisite سيٽنگن ۾ **Integration** tab هيٺ هي integration فعال ڪرڻي پوندي.

![Ultimate Multisite سيٽنگن ۾ Integrations tab جيڪو hosting providers ڏيکاري ٿو](/img/config/integrations-tab.png)

<!-- اسڪرين شاٽ دستياب ناهي: Integrations سيٽنگز tab تي hosting provider Configuration links -->

_ياد رکو ته جيڪڏهن توهان جو hosting provider مٿي ذڪر ڪيل providers مان ناهي، **ته توهان کي پنهنجي hosting account ۾ ڊومين نالو هٿ سان sync يا شامل ڪرڻو پوندو**._
