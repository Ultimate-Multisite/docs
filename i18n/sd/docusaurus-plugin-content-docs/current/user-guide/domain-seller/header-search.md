# هيڊر ڊومين ڳولا {#header-domain-search}

هي سيٽ اپ تڏهن استعمال ڪريو جڏهن توهان هڪ ننڍو هيڊر فارم چاهيو ٿا جيڪو ڊومين ڳولا شروع ڪري، پوءِ گراهڪ کي Ultimate Multisite چيڪ آئوٽ اندر موجود ڊومينن مان چونڊڻ ڏئي.

## گهرجون {#requirements}

- Ultimate Multisite نيٽ ورڪ-فعال.
- Multisite Ultimate Domain Seller نيٽ ورڪ-فعال.
- گهٽ ۾ گهٽ هڪ فعال ڊومين رجسٽريشن پيداوار جنهن ۾:
  - `domain_provider` هڪ ترتيب ڏنل فراهم ڪندڙ تي سيٽ ٿيل هجي.
  - حمايت ٿيل TLDs ترتيب ڏنل هجن، مثال طور `com`، `net`، ۽ `org`.
- هڪ صحيح چيڪ آئوٽ فارم جنهن ۾ **ڊومين چونڊ** خانو شامل هجي.

## چيڪ آئوٽ فارم {#checkout-form}

1. رجسٽريشن صفحي لاءِ استعمال ٿيندڙ چيڪ آئوٽ فارم ٺاهيو يا تبديل ڪريو.
2. عام گهربل چيڪ آئوٽ/Account خانا شامل ڪريو، جن ۾ **استعمال ڪندڙ نالو** پڻ شامل هجي. صرف ڊومين خاني تي مشتمل چيڪ آئوٽ فارم Ultimate Multisite تصديق طرفان رد ڪيو ويندو آهي.
3. هڪ **ڊومين چونڊ** خانو شامل ڪريو.
4. جڏهن وهڪري جو ڌيان مفت ذيلي ڊومينن يا موجوده ڊومينن بدران رجسٽر ٿيل ڊومينن تي هئڻ گهرجي، ته ڊومين چونڊ موڊ کي **صرف رجسٽر** تي سيٽ ڪريو.
5. ڊومين رجسٽريشن پيداوار کي انهيءَ خاني سان منسوب ڪريو.

رجسٽريشن صفحي کي چيڪ آئوٽ فارم ڏيکارڻ گهرجي، مثال طور:

```text
[wu_checkout slug="domain-form"]
```

## هيڊر فارم {#header-form}

سائيٽ هيڊر ۾ هڪ ننڍو `GET` فارم شامل ڪريو جيڪو داخل ڪيل ڳولا لفظ کي چيڪ آئوٽ صفحي ڏانهن `domain_name` طور موڪلي:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

ڪسٽم هيڊر JavaScript ۾ ڊومين اڳواٽ چونڊيو نه. هيڊر کي صرف ڳولا لفظ موڪلڻ گهرجي. Domain Seller چيڪ آئوٽ اسڪرپٽ `?domain_name=example` پڙهي ٿو، چيڪ آئوٽ ڳولا باڪس ڀري ٿو، ۽ دستيابي ڳولا هلائي ٿو ته جيئن گراهڪ واپس آيل ڊومينن مان چونڊي سگهي.

## متوقع رويو {#expected-behaviour}

هيڊر ۾ `example` ڳولڻ سان هي کُلڻ گهرجي:

```text
/register/?domain_name=example
```

پوءِ چيڪ آئوٽ کي چونڊڻ لائق نتيجا ڏيکارڻ گهرجن جهڙوڪ:

- `example.com`
- `example.net`
- `example.org` موجود ناهي
- فراهم ڪندڙ طرفان حمايت ڪيل ٻيا TLDs

موجود نتيجو چونڊڻ کان پوءِ، آرڊر خلاصو ۾ ڊومين رجسٽريشن پيداوار ۽ چونڊيل ڊومين نالو شامل هئڻ گهرجي.

## تصديق {#verification}

1. هوم صفحو کوليو.
2. هڪ سادو نالو ڳوليو، مثال طور `example`.
3. تصديق ڪريو ته چيڪ آئوٽ URL ۾ `?domain_name=example` شامل آهي.
4. تصديق ڪريو ته چيڪ آئوٽ ڊومين خاني ۾ `example` موجود آهي.
5. تصديق ڪريو ته ڊومين چونڊن جي فهرست ظاهر ٿئي ٿي.
6. موجود ڊومين لاءِ **چونڊيو** تي ڪلڪ ڪريو.
7. تصديق ڪريو ته آرڊر خلاصو ۾ `Domain Registration - example.com` يا چونڊيل ڊومين شامل آهي.

## مسئلا حل ڪرڻ {#troubleshooting}

- جيڪڏهن ڪا فهرست ظاهر نه ٿئي، ته برائوزر نيٽ ورڪ ٽيب ۾ `admin-ajax.php?action=wu_domain_search` چيڪ ڪريو ۽ تصديق ڪريو ته اهو غير خالي `domains` يا `results` واپس ڪري ٿو.
- جيڪڏهن محفوظ ڪرڻ وقت چيڪ آئوٽ فارم تصديق ۾ ناڪام ٿئي، ته گهربل Account خانا شامل ڪريو جهڙوڪ **استعمال ڪندڙ نالو**.
- جيڪڏهن ڊومين چونڊڻ سان cart اپڊيٽ نٿو ٿئي، ته تصديق ڪريو `window.wu_checkout_form` موجود آهي ۽ Domain Seller چيڪ آئوٽ اثاثا چيڪ آئوٽ صفحي تي لوڊ ٿيل آهن.
