# Դաս 10․ Գործարկման օր

Ամեն ինչ կառուցված է։ Դռները բացելուց առաջ այս դասը կանցնի նախագործարկման ստուգաթերթով՝ համոզվելու, որ ոչինչ կոտրված, բացակայող կամ ամոթալի չէ։

## Որտեղ կանգ առանք

FitSite-ում շաբլոնները, պլանները, վճարման գործընթացը, բրենդավորումը, onboarding-ը և գնագոյացումը բոլորը կարգավորված են։ Այժմ ստուգում ենք, որ ամեն ինչ աշխատում է, և գործարկում ենք։

## Նախագործարկման ստուգաթերթ

Անցեք յուրաքանչյուր կետով։ Ոչ մեկը բաց մի թողեք։

### Հարթակի ենթակառուցվածք

- [ ] Hosting-ը կայուն է և ծանրաբեռնվածության տակ լավ է աշխատում
- [ ] Wildcard SSL-ը ակտիվ է, և բոլոր ենթադոմեյնները սպասարկվում են HTTPS-ով
- [ ] Դոմեյնի կապակցումն աշխատում է -- փորձարկեք կայք ստեղծելը և անհատական դոմեյն կապակցելը
- [ ] Պահուստավորումները կարգավորված և փորձարկված են (վերականգնեք առնվազն մեկը՝ հաստատելու համար)
- [ ] Մոնիթորինգը ներդրված է -- դուք կիմանաք, եթե հարթակը անջատվի

### Շաբլոններ

- [ ] Բոլոր երեք շաբլոնները ճիշտ են բեռնվում նոր կայքերում
- [ ] Փոխարինիչ բովանդակությունը օգտակար է և զերծ է տառասխալներից
- [ ] Բոլոր պատկերները պատշաճ լիցենզավորված են (առանց stock photo ջրանշանների)
- [ ] Բջջային հարմարվողականությունն աշխատում է յուրաքանչյուր շաբլոնի էջում
- [ ] Էջի բեռնման արագությունը ընդունելի է (փորձարկեք GTmetrix-ի կամ PageSpeed Insights-ի նման գործիքով)
- [ ] Ոչ մի շաբլոնում չկան կոտրված հղումներ կամ բացակայող ռեսուրսներ

### Պլաններ և ապրանքներ

- [ ] Բոլոր երեք պլանները ակտիվ և տեսանելի են
- [ ] Պլանների նկարագրությունները ճշգրիտ են և համապատասխան նիշային ոլորտին
- [ ] Գները ճիշտ են (ամսական և տարեկան)
- [ ] Կարգավորման վճարները սահմանված են ճիշտ պլանների վրա
- [ ] Փորձաշրջանն աշխատում է Starter պլանում
- [ ] Order bump-երը ճիշտ են հայտնվում վճարման ընթացքում
- [ ] Plugin-ի և theme-ի սահմանափակումները ճիշտ են կիրառվում ըստ պլանի

### Վճարման հոսք

- [ ] Յուրաքանչյուր պլանի համար կատարեք լիարժեք փորձնական գրանցում (օգտագործեք փորձնական վճարման ռեժիմ)
- [ ] Շաբլոնի ընտրությունը ցույց է տալիս ճիշտ շաբլոնները ըստ պլանի
- [ ] Վճարումը հաջողությամբ մշակվում է
- [ ] Հաճախորդը գրանցումից հետո ստանում է ողջույնի էլ․ նամակ
- [ ] Նոր կայքը ստեղծվում է ճիշտ շաբլոնով
- [ ] Հաճախորդը կարող է անմիջապես մուտք գործել իր նոր կայք
- [ ] Զեղչի կոդերը ճիշտ են աշխատում

### Բրենդավորում

- [ ] Մուտքի էջը ցուցադրում է FitSite բրենդավորումը
- [ ] Admin dashboard-ը ցուցադրում է FitSite բրենդավորումը
- [ ] Բոլոր համակարգային էլ․ նամակներն օգտագործում են FitSite բրենդավորումը և ֆիթնեսին հատուկ լեզու
- [ ] Հաշիվ-ապրանքագրերը ճիշտ են ցուցադրվում ձեր բիզնեսի տվյալներով
- [ ] Մարքեթինգային կայքը ակտիվ է և հղում է դեպի վճարման ձև

### Onboarding

- [ ] Quick Start widget-ը հայտնվում է նոր հաճախորդների Dashboard-ներում
- [ ] Quick Start-ի բոլոր հղումները տանում են ճիշտ էջեր
- [ ] Ողջույնի էլ․ նամակների հաջորդականությունը կարգավորված և փորձարկված է
- [ ] Գիտելիքների բազայի հոդվածները հրապարակված և հասանելի են
- [ ] Account էջը ցույց է տալիս ճիշտ պլանի տեղեկությունները և բարելավման տարբերակները

### Իրավական և բիզնես

- [ ] Ծառայությունների պայմանները հրապարակված են և կապված են վճարման էջից
- [ ] Գաղտնիության քաղաքականությունը հրապարակված և հասանելի է
- [ ] Վերադարձի քաղաքականությունը սահմանված և փաստաթղթավորված է
- [ ] Բիզնես սուբյեկտը կարգավորված է վճարումներ ստանալու համար
- [ ] Վճարային gateway-ը live ռեժիմում է (ոչ թե test ռեժիմում)
- [ ] Հարկային կարգավորումը ճիշտ է ձեր իրավասության տարածքի համար

## Փափուկ գործարկում ընդդեմ կոշտ գործարկման

Դիտարկեք երկփուլ գործարկում․

### Փուլ 1․ Փափուկ գործարկում

Հրավիրեք 5-10 ֆիթնես ստուդիայի սեփականատերերի գրանցվել մինչև հանրային գործարկումը։ Սրանք ձեր բետա հաճախորդներն են։ Առաջարկեք նրանց զգալի զեղչ (50% զեղչ ամբողջ կյանքի համար կամ 3 ամիս անվճար) հետևյալի դիմաց․

- Ազնիվ կարծիք գրանցման և onboarding փորձառության մասին
- Թույլտվություն օգտագործելու իրենց կայքը որպես ցուցադրական օրինակ
- Հայտնել ցանկացած bug-ի կամ խնդրի մասին, որին նրանք կհանդիպեն

Սա ձեզ տալիս է իրական հաճախորդների կարծիք և կենդանի կայքեր՝ ցուցադրելու համար նախքան հանրության առաջ բացվելը։

### Փուլ 2․ Հանրային գործարկում

Երբ փափուկ գործարկման կարծիքները ներառված լինեն․

- Վճարային gateway-ը փոխեք live ռեժիմի
- Հրապարակեք ձեր մարքեթինգային կայքը
- Սկսեք հաճախորդների ձեռքբերումը (Դաս 11)
- Հայտարարեք համապատասխան ֆիթնես ոլորտի ալիքներում

## Գործարկման օրվա գործողություններ

Այն օրը, երբ հանրային եք դառնում․

1. **Անցեք live վճարումների** -- անջատեք test ռեժիմը ձեր վճարային gateway-ում
2. **Մեկ անգամ ևս ստուգեք** -- կատարեք ամբողջական փորձնական գրանցում իրական վճարմամբ (հետո ինքներդ ձեզ վերադարձրեք գումարը)
3. **Ուշադիր մոնիթորինգ արեք** -- հետևեք սխալներին, ձախողված գրանցումներին կամ վճարման խնդիրներին
4. **Հասանելի եղեք** -- ձեր առաջին իրական հաճախորդները կարող են օգնության կարիք ունենալ, իսկ արագ արձագանքը վստահություն է կառուցում
5. **Կարճ նշեք հաջողությունը** -- հետո վերադարձեք աշխատանքին

## Ինչ կարող է սխալ գնալ

Պատրաստ եղեք հետևյալին․

- **Վճարային gateway-ի խնդիրներ**: Ձեր gateway մատակարարի աջակցության կոնտակտը պատրաստ ունեցեք
- **SSL սերտիֆիկատի խնդիրներ**: Իմացեք՝ ինչպես ստուգել և թարմացնել սերտիֆիկատները
- **Էլ․ նամակների առաքման ձախողումներ**: Փորձարկեք, որ էլ․ նամակները իրականում հասնում են (ստուգեք spam պանակները)
- **Աշխատողականություն ծանրաբեռնվածության տակ**: Եթե ունենաք traffic-ի կտրուկ աճ, իմացեք՝ ինչպես մասշտաբավորել ձեր hosting-ը
- **Հաճախորդների շփոթություն**: Ձեր գիտելիքների բազան և աջակցության ալիքները պատրաստ ունեցեք

## Ինչ կառուցեցինք այս դասում

- **Համապարփակ նախագործարկման ստուգաթերթ**, որը ընդգրկում է ենթակառուցվածքը, բովանդակությունը, վճարումները և իրավական հարցերը
- **Փափուկ գործարկման ռազմավարություն**՝ իրական կարծիք ստանալու համար նախքան հանրային դառնալը
- **Գործարկման օրվա ընթացակարգեր**՝ վստահությամբ live անցնելու համար
- **Արտակարգ իրավիճակների պլան**՝ գործարկման օրվա տարածված խնդիրների համար
