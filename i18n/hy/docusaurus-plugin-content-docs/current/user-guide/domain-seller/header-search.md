# Վերնագրի դոմենի որոնում

Օգտագործեք այս կարգավորումը, երբ ցանկանում եք փոքր վերնագրի ձև, որը սկսում է դոմենի որոնում, ապա թույլ է տալիս հաճախորդին ընտրել հասանելի դոմեններից Ultimate Multisite-ի պատվերի ձևակերպման մեջ։

## Պահանջներ

- Ultimate Multisite-ը՝ ցանցում ակտիվացված։
- Multisite Ultimate Domain Seller-ը՝ ցանցում ակտիվացված։
- Առնվազն մեկ ակտիվ դոմենի գրանցման ապրանք՝ հետևյալով.
  - `domain_provider`-ը սահմանված է կարգավորված մատակարարի։
  - Աջակցվող TLD-ները կարգավորված են, օրինակ՝ `com`, `net` և `org`։
- Վավեր պատվերի ձևակերպման ձև, որը պարունակում է **Դոմենի ընտրություն** դաշտը։

## Պատվերի ձևակերպման ձև

1. Ստեղծեք կամ խմբագրեք գրանցման էջի կողմից օգտագործվող պատվերի ձևակերպման ձևը։
2. Ավելացրեք սովորական պարտադիր պատվերի ձևակերպման/account դաշտերը, ներառյալ **Օգտանուն**։ Միայն դոմենի դաշտ պարունակող պատվերի ձևակերպման ձևը մերժվում է Ultimate Multisite-ի վավերացման կողմից։
3. Ավելացրեք **Դոմենի ընտրություն** դաշտ։
4. Սահմանեք Դոմենի ընտրության ռեժիմը **Միայն գրանցում**, երբ հոսքը պետք է կենտրոնանա գրանցված դոմենների վրա՝ անվճար ենթադոմենների կամ գոյություն ունեցող դոմենների փոխարեն։
5. Կցեք դոմենի գրանցման ապրանքը այդ դաշտին։

Գրանցման էջը պետք է արտապատկերի պատվերի ձևակերպման ձևը, օրինակ՝

```text
[wu_checkout slug="domain-form"]
```

## Վերնագրի ձև

Ավելացրեք փոքր `GET` ձև կայքի վերնագրում, որը մուտքագրված որոնման տերմինը ուղարկում է պատվերի ձևակերպման էջ՝ որպես `domain_name`՝

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

Մի նախընտրեք դոմենը custom վերնագրի JavaScript-ում։ Վերնագիրը պետք է միայն փոխանցի որոնման տերմինը։ Domain Seller-ի պատվերի ձևակերպման սկրիպտը կարդում է `?domain_name=example`-ը, լրացնում է պատվերի ձևակերպման որոնման դաշտը և գործարկում հասանելիության որոնումը, որպեսզի հաճախորդը կարողանա ընտրել վերադարձված դոմեններից։

## Սպասվող վարքագիծ

Վերնագրում `example` որոնելը պետք է բացի՝

```text
/register/?domain_name=example
```

Այնուհետև պատվերի ձևակերպումը պետք է ցուցադրի ընտրելի արդյունքներ, ինչպիսիք են՝

- `example.com`
- `example.net`
- `example.org` հասանելի չէ
- մատակարարի կողմից աջակցվող այլ TLD-ներ

Հասանելի արդյունք ընտրելուց հետո պատվերի ամփոփագիրը պետք է ներառի դոմենի գրանցման ապրանքը և ընտրված դոմենի անունը։

## Ստուգում

1. Բացեք գլխավոր էջը։
2. Որոնեք պարզ անուն, օրինակ՝ `example`։
3. Հաստատեք, որ պատվերի ձևակերպման URL-ը ներառում է `?domain_name=example`։
4. Հաստատեք, որ պատվերի ձևակերպման դոմենի դաշտը պարունակում է `example`։
5. Հաստատեք, որ դոմենի ընտրությունների ցանկ է հայտնվում։
6. Սեղմեք **Ընտրել** հասանելի դոմենի համար։
7. Հաստատեք, որ պատվերի ամփոփագիրը պարունակում է `Domain Registration - example.com` կամ ընտրված դոմենը։

## Խնդիրների լուծում

- Եթե որևէ ցանկ չի հայտնվում, ստուգեք դիտարկիչի ցանցի ներդիրը `admin-ajax.php?action=wu_domain_search`-ի համար և հաստատեք, որ այն վերադարձնում է ոչ դատարկ `domains` կամ `results`։
- Եթե պատվերի ձևակերպման ձևը պահպանելիս չի անցնում վավերացումը, ավելացրեք պարտադիր account դաշտերը, ինչպիսիք են **Օգտանուն**։
- Եթե դոմեն ընտրելը չի թարմացնում զամբյուղը, հաստատեք, որ `window.wu_checkout_form` գոյություն ունի, և որ Domain Seller-ի պատվերի ձևակերպման ռեսուրսները բեռնված են պատվերի ձևակերպման էջում։
