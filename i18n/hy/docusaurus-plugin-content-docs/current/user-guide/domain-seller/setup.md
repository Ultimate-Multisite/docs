# Դոմենի վաճառող՝ Կարգավորում և Պրովայդերի Կազմաձևում {#domain-seller-setup-and-provider-configuration}

Domain Seller add-on-ը տեղադրվում է ուղեցույցային կարգավորման գործիքով, որը ձեզ անցնում է բոլոր անհրաժեշտ քայլերը։ Այս էջը ներկայացնում է այս գործիքի հոսքը և թե ինչպես կարելի է հետո կազմաձևել կամ վերակազմաձևել պրովայդերը։

## Պահանջներ {#requirements}

- **Multisite Ultimate** v2.4.12 կամ ավելի բարձր, ցանցով ակտիվացված
- **PHP** 7.4+
-  առնվազն մեկ աջակցվող գրանցիչի API տվյալներ

## Առաջին անգամ օգտագործման կարգավորման գործիք (First-run setup wizard) {#first-run-setup-wizard}

Կարգավորման գործիքը ավտոմատ կերպով բացվում է առաջին անգամ, երբ դուք պլագինը ցանցով ակտիվացնում եք։ Այն հասանելի է նաև ցանկացած պահի **Network Admin › Ultimate Multisite › Domain Seller Setup**-ից։

### Քայլ 1 — Ընտրել պրովայդեր {#step-1--choose-a-provider}

Ընտրեք այն գրանցիչը, որին ցանկանում եք կապվել։ Աջակցվող տարբերակներ.

| Provider | DNS-ի կառավարում | WHOIS գաղտնիություն |
|---|---|---|
| OpenSRS | Այո | Այո |
| Namecheap | Ոչ | Այո (WhoisGuard, անվճար) |
| HostAfrica | Այո | Այո (ID պաշտպանություն) |
| Openprovider | Այո | Այո |
| Hostinger | Հոնկված ինքնուրույն տիրույթների համար Hostinger-ի հիմնական դոմենային քարտեզագրման միջոցով | Այո |
| GoDaddy | Ոչ | Ոչ |
| ResellerClub | Այո | Ոչ |
| NameSilo | Ոչ | Ոչ |
| Enom | Այո | Ոչ |

### Քայլ 2 — Մուտքագրել տվյալները (Credentials) {#step-2--enter-credentials}

Յուրաքանչյուր պրովայդերի տարբեր դաշտեր կան տվյալների համար.

**OpenSRS** — Օգտատիրոջ անունը և գաղտնի բանալին (OpenSRS Reseller Control Panel-ից)

**Namecheap** — Օգտատիրոջ անունը և API բանալին (Account › Tools › API Access-ից)

**HostAfrica** — Դոմենների վաճառողի API ծայրակետը և տվյալները HostAfrica վաճառողի մոդուլից։ Այս պահին առանձին sandbox ծայրակետ չի փաստագրված, փորձարկեք անվտանգ ընթերցման միայն գործողությունների համար նախքան իրական գրանցումները կատարելը։

**Openprovider** — Օգտատիրոջ անունը և գաղտնի բանալին API-ով մուտքով։ Ընտրովի sandbox ռեժիմը օգտագործում է Openprovider sandbox API-ն, և ընտրովի լռելյայն հաճախորդի համար կարող է վերանօգտագործվել գրանցումների համար։

**Hostinger** — Core Hostinger integration-ից ստացված կիսատ Hostinger hPanel API token։ Այս տոքենը աշխատում է հիմնական տիրույթների քարտեզագրման և Domain Seller գրանցման գործողությունների համար։

**GoDaddy** — API key և secret (developer.godaddy.com-ից)

**ResellerClub** — Reseller ID և API key (ResellerClub control panel-ից)

**NameSilo** — API key (namesilo.com › Account › API Manager-ից)

**Enom** — Account ID և API token

Ստուգեք **Sandbox mode**-ը, եթե հնարավոր է, որպեսզի ստուգեք առաջ գնալուց առաջ պրովայդერის թեստային միջավայրում։

### Քայլ 3 — Ստուգել կապը {#step-3--test-the-connection}

Սեղմեք **Test Connection**-ի վրա։ Վիզարդը փոքր API զանգ է ուղարկում՝ հավաստելու համար մուտքային տվյալները և կապը։ Շարունակելուց առաջ շտկեք ցանկացած մուտքային տվյալի խնդիրներ։

### Քայլ 4 — TLD-ների ներմուծում {#step-4--import-tlds}

Սեղմեք **Import TLDs**-ի վրա՝ բոլոր հասանելի TLD-ները և լողավարձային գները ստանալու համար կապված պրովայդերից։ Սա լրացնում է տիրույթների ցուցակը, որը օգտագործվում է դոմենի արտադրանքների համար։ Ներմուծումը կարող է տևել 30–60 վայրկյան՝ մեծ TLD կատալոգներ ունեցող պրովայդերների համար։

TLD-ները ավտոմատ կերպով նաև համաժամանակյշում են մեկ անգամ օրական՝ սխեդուլավորված cron job-ի միջոցով։

### Քայլ 5 — Ստեղծել տիրույթային արտադրանք (domain product) {#step-5--create-a-domain-product}

Վիզարդը ստեղծում է լռելյայն ընդհանուր դոմենային արտադրանք՝ 10% գումարով։ Դուք կարող եք այս արտադրանքը խմբագրել անմիջապես կամ բաց թողնել և ձեռքով ստեղծել արտադրանքներ **Ultimate Multisite › Products**-ի ներքո։

Ամբողջական արտադրանքի կոնֆիգուրացիայի ուղեցույցը տեսեք [Domain Products and Pricing](./domain-products)-ում։

## Պրովայդერის վերակազմակերպում (Reconfiguring a provider) {#reconfiguring-a-provider}

Գնացեք **Network Admin › Ultimate Multisite › Settings › Domain Seller** (կամ սեղմեք **Settings**-ը pluginների ցուցակում)։

Կարգավորումների էջում կտեսնեք.

- **Domain վաճառքի թույլատրում** — այս ամբողջու հնարավորությունը 켜/off անել։
- **Միանշանակ մատակարար (Default provider)** — այն մատակարարը, որը օգտագործվում է տիրույթների որոնման և նոր արտադրանքների համար։
- **Առավելագույն TLD-ներ որոնման ժամանակ** — թե քանի TLD ստուգել՝ գնորդը փնտրում է, ավելի բարձր արժեքները ավելի շատ տարբերակներ են ցույց տալիս, բայց դրանք ավելի դանդաղ են։
- **Հասանելիության կեշի (Availability cache) ժամկետ** — թե որքան ժամանակ է պահվում հասանելիության և գների արդյունքները կեշով, ավելի ցածր արժեքները ավելի ճշգրիտ են, բայց մեծացնում են API կանչերը։
- **Կառավարել տիրույթային արտադրանքները (Manage domain products)** — Արագ հղում՝ Արտադրանքների ցուցակին։
- **Կազմաձևել մատակարարները (Configure providers)** — բացում է Integration Wizard-ը՝ մատակարարներ ավելացնելու կամ վերաձևավորելու համար։

### Երկրորդ մատակարարի ավելացում {#adding-a-second-provider}

Սեղմեք **Configure providers**-ին և նոր գրանցիչի համար նույն Wizard-ը կատարեք։ Դուք կարող եք միաժամանակ կազմաձևել բազմաթիվ մատակարարներ։ Աջակցեք յուրաքանչյուր տիրույթային արտադրանք հատուկ մատակարարին, կամ թողեք դեֆոլտի վրա։

### TLD-ների ձեռքով սինխրոնի (Syncing TLDs manually) {#syncing-tlds-manually}

Կարգավորումների էջում ցանկացած կազմաձևված մատակարարի աջ կողմում սեղմեք **Sync TLDs**-ին՝ ամենաթարմ գները ներմուծելու համար։ Սա հữu ích է, երբ մատակարարը թարմացնում է խոշտագույն գները կամ ավելացնում նոր TLD-ներ։

## Ծառայության կարողությունների նշումներ (Provider capability notes) {#provider-capability-notes}

Չունեն բոլոր գրանցիչների API-ները նույն գործողությունները։ Ավելացուցիչը (addon)-ը ներկայացնում է աջակցվող չլինող գործողությունները հստակ ადմինի համար տեսանելի սխալներով, այլ ոչ թե լռելյայն ձախողում։

- **HostAfrica**-ը աջակցում է ամենամ rộngելի կենդանի վաճառող (reseller) գործընթացին, ներառյալ որոնումը, TLD/գինու համաժամեցումը, գրանցումը, վերանորոգումը, փոխանցումը, nameserver-ների թարմացումները, DNS գրառումները, EPP կոդերը, գրանցչի լոկը և ID-ի պաշտպանությունը։
- **Openprovider**-ը աջակցում է վաճառողի գնով TLD համաժամեցմանը, գրանցմանը, վերանորոգմանը, փոխանցումներին, nameserver-ների թարմացումներին, DNS զոնաներին, EPP կոդերին, գրանչի լոկին և WHOIS գաղտնիությանը։ Այն ճիշտ է հաստատվում կարճ ժամկետով բերման (short-lived bearer token) միջոցով, որը աջակիցը ավտոմատ կերպով թարմացնում է։
- **Hostinger**-ը աջակցում է հասանելիության որոնմանը, գրանցմանը, պորտֆոլիոյի որոնմանը, nameserver-ների թարմացումներին, գրանչի լոկին և WHOIS գաղտնիությանը՝ օգտագործելով ընդհանուր hPanel API token-ը։ Hostinger-ի հանրային Domains API-ն չի ցուցադրում վաճառողական/խորհրդային գները, ներքայացման փոխանցումը, բացահայտ վերանորոգումը կամ EPP-կոդերի ստացումը. վերանորոգումները ավտոմատ վերանորոգվում են միայն։
