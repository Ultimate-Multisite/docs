# Դոմենի ապրանքներ և գնանշումներ {#domain-products-and-pricing}

Դոմենի ապրանքները թույլ են տալիս վերահսկել գները, TLD-ները (Top-Level Domains), WHOIS գաղտնիությունը և մատակարարի ընտրությունը։ Յուրաքանչյուր դոմենի ապրանքը ստանդարտ Multisite Ultimate ապրանք է՝ լրացուցիչ **Domain Settings** ներդիրով։

## Դոմենի ապրանքի ստեղծում {#creating-a-domain-product}

1. Գնացեք **Network Admin › Ultimate Multisite › Products**-ին։
2. Սեղմեք **Add New**-ի վրա։
3. Ապրանքի տեսակը սահմանեք որպես **Domain**։
4. Կազմաձևեք **Domain Settings** ներդիրը (տես ներքևում)։
5. Պահպանեք։

Դոմենի ապրանքները հայտնվում են արտադրանքների ցուցակում ճեղքով կապույտ **Domain** բուլետով և կարելի է զտել՝ օգտագործելով **Domain Products** ներդիրը։

## Domain settings ներդիրը {#domain-settings-tab}

### Մատակարար (Provider) {#provider}

Ընտրեք, թե որ գրանցիչը կկառավարի այս ապրանքի համար գրանցումը։ Սխալ값ով սահմանված է գլոբալ **Default provider**-ը Domain Seller կարգավորումներում։

### Աջակցվող TLD-ներ (Supported TLDs) {#supported-tlds}

Դատարկ թողեք՝ ստեղծելու համար **catch-all product**, որը կիրառվում է բոլոր այն TLD-ների համար, որոնք չեն համապատասխանում մյուս ապրանքներին։

Մուտքագրեք TLD-ների կոմայով բաժանված ցուցակ (օրինակ՝ `.com, .net, .org`), որպեսզի ստեղծեք **TLD-specific product**, որը կիրառվում է միայն այդ ընդլայնումների համար։

**Ինչպես է ապրանքի համապատասխանությունը աշխատում:** Երբ հաճախորհուրդը փնտրում է դոմեն, ավելացուցիչը (addon) ընտրում է ամենահամապատասխանող ապրանքը։ Այն ապրանքը, որը TLD-ների ցուցակում ունի `.com`, առաջնահերթություն է ստանում՝ համեմատած catch-all ապրանքի հետ։ Եթե չկա TLD-հատուկ ապրանք, որը համապատասխանում է, օգտագործվում է catch-all-ը։ Եթե ոչ մի ապրանք չկա, դոմենի որոնումը չի ցուցվի։

### Մարկաի տեսակ (Markup type) {#markup-type}

Երեք մոդը վերահսկում են ձեր մանրածախ գնի հաշվարկը մաքսային արժեքից (wholesale cost).

| Մոդ (Mode) | Ինչպես է աշխատում |
|---|---|
| **Percentage** | Ավելացնում է տոկոս մաքսային արժեքի վրա։ $10-ի մաքսային արժեքի վրա 20% մարկաուպը տալիս է $12։ |
| **Fixed markup** | Ավելացնում է հաստատուն դոլար գումար։ $10-ի դոմենի վրա $5 մարկաուպը տալիս է $15։ |
| **Fixed price** | Բացառում է մաքսային արժեքը։ Միշտ հաշվում է այն գումարը, որը դուք մուտքագրում եք։ |

### Ներածական գին (Introductory pricing) {#introductory-pricing}

### Գինակ առաջարկելու թույլտվություն {#whois-privacy}

Թույլ տալ՝ առաջարկել առաջին տարվա զեղչ գինը։ Սահմանեք առանձին **Ներածական գին** (1-ին տարվա գին)՝ սովորական **Վերականգնման գնի** (2-րդ տարի և այլ) կողքին։ Հաճախորդը տեսնում է երկու գները գրանցման ժամանակ, որպեսզի իմանա ինչի են սպասել վերականգնման ժամանակ։

### WHOIS գաղտնիություն (privacy) {#tld-import-and-sync}

Կառավարում է այն, թե արդյոք տվյալ արտադրանքով գրանցված ադմիններ համար առաջարկվում է WHOIS գաղտնիության պաշտպանություն։

| Սահմանում | Գործողություն |
|---|---|
| **Անջատված** (Disabled) | WHOIS գաղտնիությունը երբեք չի առաջարկվում կամ ակտիվացվում։ |
| **Մշտապես ներառված** (Always Included) | WHOIS գաղտնիությունը ավտոմատ կերպով ակտիվանում է գրանցման ժամանակ առանց վճարման։ |
| **Հաճախորդի ընտրություն** (Customer Choice) | Վճարումների ժամանակ հայտնվում է ստուգում։ Սահմանեք **Գաղտնիության գինը**՝ ըստ տարվա վճարելու, կամ թողեք այն $0-ով՝ անվճար առաջարկելու համար։ |

Namecheap-ի համար WHOIS գաղտնիությունը օգտագործում է WhoisGuard (մշտապես անվճար)։ OpenSRS-ի համար օգտագործվում է OpenSRS գաղտնիության ծառայությունը (համարձակ է վաճառքի դեպքում)։

## TLD ներմուծում և սինխրոնիզացիա {#auto-renewal}

Դոմենի արտադրանքները ցուցադրում են իրական ժամանակի վաճառքի գներ՝ կապված համապատասխան պրովայդերի հետ։ Սա աշխատելու համար անհրաժեշտ է TLD-ների ներմուծում։

- **Մանրամասն սինխրոնիզացիա** (Manual sync): Settings › Domain Seller › Sync TLDs (յուրաքանչյուր պրովայդերի համար)
- **Ավտոմատ սինխրոնիզացիա** (Automatic sync): Կատարվում է ամեն օր սխեդուլավորված cron job-ի միջոցով բոլոր կազմաձևված պրովայդերի համար։

Սինխրոնիզացիայի հետո, գնացեք ցանկացած դոմենի արտադրանքի Domain Settings տաբը և օգտագործեք TLD ընտրիչը՝ հասանելի TLD-ները տեսնելու իրենց ներկայիս վաճառքի գներով։

## Ավտոմատ թարմացում (Auto-renewal) {#admin-manual-domain-registration}

Դոմենի վերականգնումները կապված են հաճախորդի անդամակցության կարգավիճակի հետ.

- Երբ անդամակցությունը թարմացվում է և դոմենը կապվում է, դոմենի վերականգնումը ավտոմատ կերպով տեղադրվում է։
- Վերականգնման փորձերը օգտագործում են հաճախորդի ակտիվ վճարումների գեյթવેը (payment gateway)։
- Հասանելի չհաջողված վերականգնումները ավտոմատ կերպով փորձարկվում են աստիճանաբար մեծացող հետընթացի (exponential backoff) միջոցով։
- Էլ. նամակային տեղեկություններ ուղարկվում են հաջող վերականգնումների, ձախողումների և առաջիկա ժամկետներին մաս կապված իրադարձությունների համար։

Դոմենի ցիկլի իրադարձությունների համար էլ. նամակի տեխնոլոգիական ID-ները.

| Sự kiện | Template ID |
|---|---|
| Դոմենը գրանցվել է | `domain_registered` |
| Դոմենը թարմացվել է | `domain_renewed` |
| Թարմացումը ձախողվել է | `domain_renewal_failed` |
| Դոմենը շուտով կավարտվի | `domain_expiring_soon` |

## Տնօրեն՝ Ձեռքով դոմեն գրանցում

Հաճախորդի անունից դոմեն գրանցել, առանց նրան հասնելու checkout-ի միջոցով.

1. Գնացեք **Network Admin › Ultimate Multisite › Register Domain** (Ցանցի Տնօրեն › Ultimate Multisite › Դոմեն գրանցել)
2. Ընտրեք հաճախորդը և մուտքագրեք դոմենի անունը
3. Լրացրեք գրանցվողի կոնտակտային տվյալները (անուն, հասցե, հեռախոս)
4. Սեղմեք **Register** (Գրանցել)

Դոմենի գրանցումը ստեղծվում է և կապվում է հաճախորդի հաշվի հետ։
