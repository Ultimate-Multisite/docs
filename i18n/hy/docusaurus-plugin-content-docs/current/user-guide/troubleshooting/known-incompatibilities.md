# Այլ Պլագինների հետ Հայտնի անհամապատասխանություններ

Ultimate Multisite-ը տեղադրելու և ակտիվացնելուց հետո կարող եք խնդիրներ հանդիպել այլ պլագինների հետ, որոնք չեն լուծվել։

Մենք ակտիվորեն փորձում ենք այս ցուցակը կրճատել յուրաքանչյուր նոր թարմացման հետ, բայց եթե պլագինը գտնվում է այստեղ, կարող եք անհրաժեշտ լինի դադարեցնել այն՝ Ultimate Multisite-ի ամբողջական ֆունկցիոնալ ցանցը ստանալու համար։

Ահա դրանց ցուցակը և ինչ պետք է անել այդ դեպքերում.

**Որոշ WPMU DEV պլագիններ** WPMU DEV-ի կողմից այլևս թարմացված չեն որոշ պլագինները հայտնի են, որ խանգարում են Ultimate Multisite-ի գրանցման ընթացքը վերջին քայլում (օգտվար չի կարող ստեղծել հաշիվ)։ Ահա այն պլագինների ցուցակը, որոնք առաջացնում են այդ խնդիրները.

  * Signup Language: _Ultimate Multisite-ի մեջ կա լրացուցիչ տարբերակ՝ Ultimate Multisite: Language Switcher_, որը նույն ֆունկցիոնալությունը ապահովում է։

  * Set Blog Description on Blog Creation: _Սա ներառված է Ultimate Multisite-ի մեջ, այսինքն՝ լրացուցիչ պլագինի անհրաժեշտություն չկա_։

  * New Blog Templates: _Սա ներառված է Ultimate Multisite-ի մեջ, այսինքն՝ լրացուցիչ պլագինի անհրաժեշտություն չկա_։

  * Set Password on Multisite Blog Creation: _Սա ներառված է Ultimate Multisite-ի մեջ, այսինքն՝ լրացուցիչ պլագինի անհրաժեշտություն չկա_։

  * Signup TOS: _Սա ներառված է Ultimate Multisite-ի մեջ, այսինքն՝ լրացուցիչ պլագինի անհրաժեշտություն չկա_։

  * Anti-Splog: Թեև ավելի քիչ հզոր է, այն նույնպես _ներառված է Ultimate Multisite-ում_։

**Նշում.** Բոլոր մյուս WPMU Dev պլագինները կարող են սովորականորեն օգտագործվել Ultimate Multisite-ի հետ։ Օրինակներն են _Smush_, _Forminator_, _Defender_ և այլն։
