# V1-ից անցում

## Ultimate Multisite-ը իր սկզբնական 1.x հաջորդականությունից 2.x հաջորդականության վերածվել է։ {#ultimate-multisite-has-switched-from-its-original-1x-family-of-releases-to-the-2x-family-of-releases}

Ultimate Multisite-ի 2.0 և ավելի տարբերակները կոդի ամբողջական վերծանում են, ինչը նշանակում է, որ հին տարբերակի և նորի միջև շատ քիչ համընկնում կա։ Այս պատճառով, 1.x-ից 2.x-ի թարմացման ժամանակ ձեր տվյալները պետք է փոխակերպվեն այն ձևաչափի, որը կարող են հասկանալ նոր տարբերակները։

Բարի լինելու համար Ultimate Multisite 2.0+ **ներառում է մեկ migrаtor**, որը ներկառուցված է կոռը և կարող է հայտնաբերել հին տարբերակի տվյալները և դրանք փոխակերպել նոր ձևաչափի։ Այս մIGRացիան տեղի է ունենում 2.0+ տարբերակի **Setup Wizard**-ի ընթացքում։

Այս դասը ներկայացնում է, թե ինչպես է աշխատում migrаtor-ը, ինչ անել դեպքերի դեպքում ձախողման և այս գործընթացի ընթացքում առաջանալու խնդիրները լուծելու համար։

_**ԿԱՐԵՎՈՐ. Ավելի 1.x-ից 2.0 տարբերակին անցնելուց առաջ համոզվեք, որ ստեղծել եք ձեր կայքի տվյալների բանկ (backup)**_

## Առաջին քայլերը {#first-steps}

Առաջին քայլը՝ ներբեռնել plugin .zip ֆայլը և 2.0 տարբերակը տեղադրել ձեր network admin dashboard-ի վրա։

Երբ դուք [տեղադրեք և ակտիվացրեք 2.0-ը](../getting-started/installing-ultimate-multisite.md), համակարգը ավտոմատ կերպով կհայտնաբերի, որ ձեր Multisite-ը աշխատում է հին տարբերակի վրա, և դուք այս հաղորդագրությունը կտեսնեք plugin-ի էջի վերևում։

_**Նշում. Եթե ձեր Multisite-ում արդեն տեղադրված է Ultimate Multisite 1.x, ապա ունեք տարբերակ այն փոխարինել plugin-ը՝ այն տարբերակով, որը նոր եք ներբեռնել։ Խնդրում ենք անցնել և սեղմել **Replace current with uploaded** (Փոխարինել ընթացիկով վերբեռնվածով)։**_

<!-- Անկանոն պատկեր. Plugin-ի էջը, որտեղ երևում է v1-ը փոխարինելու տարբերակը v2-ով -->

Հաջորդ էջը կտա տեղեկություն այն հավաքածուի մասին, որոնք դուք ունեք տեղադրված՝ version 1.x-ի հետ համատեղ։ Այն կներառի ցուցումներ նրան, թե արդյոք օգտագործվող տարբերակը համատեղելի է version 2.0-ի հետ, կամ թե անհրաժեշտ է հավելվածքի թարմացված տարբերակ տեղադրել մենեջմենթից հետո։

<!-- Screenshot unavailable: Message on the top of the plugins page about updating to v2.0, with list of add-ons that need updating -->

Երբ պատրաստ լինեք շարունակելու, կարող եք սեղմել այն կոճակը, որում գրված է **Visit the Installer to finish the upgrade** (Այցելեք ինստալլատորը՝ թարմացման համար)։

<!-- Screenshot unavailable: Button saying Visit the Installer to finish the upgrade -->

Դա ձեզ կուղղի տեղադրման վարձակազմի էջ, որտեղ կլինեն մի քանի ողջույնի հաղորդագրություններ։ Ձեզ պետք է սեղմել **Get Started** (Սկսել)՝ հաջորդ էջը բացելու համար։

<!-- Screenshot unavailable: Setup Wizard welcoming page with Get Started button -->

**Get Started**-ը սեղմելուց հետո այն ձեզ կուղղի Pre-install Checks_.- Այստեղ կտեսնեք ձեր համակարգային տեղեկությունները և WordPress-ի տեղադրումը, ինչպես նաև կասի, արդյոք դա բավարարում է **Ultimate Multisite**-ի պահանջներին։

<!-- Screenshot unavailable: Pre-install Checks page showing system requirements confirmation -->

Հաջորդ քայլը ձեր Ultimate Multisite լիցենզիայի բանալին մուտքագրելն ու պլագինը ակտիվացնելն է։ Սա կապահովի, որ ձեր սайտի վրա հասանելի լինեն բոլոր հնարավորությունները, ներառյալ add-on-ները։

<!-- Screenshot unavailable: License activation page with license key input and Agree and Activate button -->

Բանալին մուտքագրելուց հետո սեղմեք **Agree & Activate** (Համաձայնություն և ակտիվացում)։

Լիցենզիայի ակտիվացումից հետո կարող եք սկսել իրական տեղադրումը՝ հաջորդ էջում **Install**-ը սեղմելով։ Սա ավտոմատ կերպով կստեղծի version 2.0-ի համար անհրաժեշտ բոլոր անհրաժեշտ ֆայլերը և տվյալների բազան (database)՝ աշխատելու համար։

## Այսուհետ՝ մIGRATION (Տվյալների փոխանցում) {#now-the-migration}

Միգրատորը ներկառուցված է անվտանգության միջոցով. այն ստուգում է ձեր ամբողջ Ultimate Multisite-ը, որպեսզի համոզվի, որ բոլոր տվյալները կարելի է փոխանցել առանց որևէ խնդրի։ Սեղմեք **Run Check** (Ստուգում կատարել) ღილակը՝ գործընթացը սկսելու համար։

Այս ստուգումից հետո երկու տարբերակ կունենաք. արդյունքը կարող է լինել **հետ** սխալով կամ **առանց** սխալի։

### Սխալով (With Error) {#with-error}

Եթե ստանում եք սխալի հաղորդագրություն, անհրաժեշտ կլինեք դիմել մեր աջակցության թիմին՝ նրանց օգնության համար սխալը շտկելու հարցում։ Համոզվեք, որ **տրամադրեք սխալի ֆայլը (error log)**, երբ ստեղծում եք ticket-ը։ Դուք կարող եք ներբեռնել տվյալները կամ սեղմել այն հղումը, որը գրված է «կապվեք մեր աջակցության թիմի հետ» (contact our support team)։ Սա բաց կտուի օգնության վի젯ը ձեր էջի աջ կողմում՝ նախապես լրացված դաշտերով, որոնք ներառում են սխալի ֆայլերը՝ նկարագրության տակ։

_**Քանի որ համակարգը գտել է սխալ, դուք չեք կարող առաջ շարժվել 2.0 տարբերակի փոխանցման համար։ Այնուհետև կարող եք հետ գնալ 1.x տարբերակին՝ ձեր ցանցը շարունակելու համար մինչև սխալի շտկումը։**_

### Առանց Սխալի (Without Error) {#without-error}

Եթե համակարգը որևէ սխալ չի գտնում, դուք կտեսնեք հաջողության հաղորդագրություն և ներքևում **Migrate** (Փոխանցել) ღի թիվը, որը թույլ կտա շարունակել փոխանցումը։ Այս էջում ձեզ հուշվում է՝ նախքան առաջ շարժվելու համար ստեղծել ապահովագրություն (backup) ձեր բազայի, ինչը մենք խիստ խորհուրդ ենք տալիս։ Եթե արդեն ունեք backup-ի պատճեն, սեղմեք **Migrate**։

Այս ամենը բավարար է։

Դու կարող ես շարունակել Wizard-ի հաստատումը՝ ձեր լոգոն թարմացնելու և այլ բաների անելիքների համար, կամ սկսել գործառնության մեջ մտնել Ultimate Multisite version 2.0-ի մենյու և նոր վերադասավորված ինտերֆեյսով։ Անցիր և զվարճացիր։
