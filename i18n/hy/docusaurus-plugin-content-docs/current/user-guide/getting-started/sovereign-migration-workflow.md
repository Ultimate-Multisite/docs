# Sovereign Migration Workflow

Ultimate Multisite: Multi-Tenancy 1.2.0-ը հավելում է մուտքի ստուգման դռներ (migration verification gates)՝ ստանդարտ ենթասայթը տեղափոխելիս սովրեն տնօրինական ենթատերանիի ինֆրամագետային կառուցվածքի մեջ։

## Ավելի սկսելուց առաջ {#before-you-start}

Հաստատեք, որ ունեք.

- Կայուն բաց පිටсиտի (source site) պատկերակազմ։
- Multi-Tenancy addon-ը ակտիվ է։
- Գնդակային տեղամասի (destination tenant) բազան, ֆայլային համակարգի արմատը և ինտերնետային անունը պատրաստ են։
- Տվյալների հոස්թի կապը համապատասխանում է նպատակային միջավայրին (destination environment)։
- Ցանցի համար WP-CLI հրամաններ գործարկելու մուտք։

## Խորհուրդ տրվող ընթացակարգ {#recommended-workflow}

1. Պատրաստեք նպատակային ենթատերանիի բազան և ֆայլային համակարգը։
2. Գրանցեք կամ թարմացրեք ենթատերանիի առանձնացման կարգավորումները (tenant isolation settings)։
3. Գործարկեք ենթատերանիի մուտքագրումը (tenant migration)։
4. Հաստատեք կամ ստեղծեք ենթատերանիի օգտվողներին (tenant users)։
5. Դատարկեք ասինխրոն մուտքագրման առաջադրանքները (async migration jobs)։
6. Գործարկեք սովրեն մուտքագրման վերահսկումը (sovereign migration verification)։
7. Դիտեք ենթատերան SSO-ով (Single Sign-On)։
8. DNS կամ ռութինգը փոխեք միայն այն դեպքում, երբ ստուգումը հաջող է անցել։

## Ստուգման դռներ (Verification gates) {#verification-gates}

Ստուգման ընթացակարգը մուտքագրումը ստուգում է մի քանի անկյունից.

- Տեղատիրույթի սխեման գոյություն ունի և կարող է գրվել ենթատերանիի բազայի կողմից։
- Կազմաձևված բազայի հոස්թը ճիշտ թույլտվություններ (grants) ունի։
- Ենթատերանիի օգտվողները ներկայացված են և համապատասխանում են սպասվող սովրեն տեղադրման օգտվողների քանակին։
- Ասինխրոն ճակատային հերթը (async push queue) կարող է հաջողությամբ դատարկվել։
- Սուբյեկտիվ ենթատերանիի համար արդեն անհրաժեշտ չեն հնչեղ ցանցային տվյալների ուղիները (legacy network-side data paths)։

Ստուգման ձախողումները դիտարկեք որպես գործարկումից առաջ խոչընդոտներ։ Ուղղեք հայտնաբերված բազայի, օգտվողի, հերթի կամ ռութինգի խնդիրը, այնուհետև նորից անցկացրեք ստուգումը՝ ենթատերան մշակողներին դիմելուց առաջ։

## Առաջին արտադրական այցելություն (First production visit) {#first-production-visit}

Ստուգումը հաջող անցնելուց հետո, օգտագործեք կայքի կառավարման էկրանից **Visit (SSO)**-ն առաջին ադմին մուտքի համար։ Սա մեկ վերահսկելի քայլով հաստատում է ենթատերանիի ռութինգը, SSO տոկենների մշակումը, սկզբնական կետի (origin pinning) և ենթատերան կողմից օգտվողների պատրաստումը։
