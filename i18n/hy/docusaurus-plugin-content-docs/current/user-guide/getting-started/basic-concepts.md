# Հիմնական հասկացություններ {#basic-concepts}

Նոր WordPress Multisite օգտվողի և ովս սկսում է Ultimate Multisite-ը օգտագործել, առաջին քայլերում կարող է շատ նոր բառեր ու արտահայտություններ հանդիպել։ Դրանք սովորելը կարևոր խնդիր է, քանի որ անհրաժեշտ կլինի հասկանալ պլատֆորմը և դրա ամբողջական աշխատանքը։

Այս հոդվածում մենք փորձում ենք սահմանել և բացատրել WordPress-ի մի քանի հիմնական հասկացություններ։ Դրանցից որոշները ավելի համապատասխանում են օգտվողներին, այլ որոշը՝ մշակողներին, և ինչպես երկուսին։

## WordPress Multisite {#wordpress-multisite}

WordPress **Multisite**-ը WordPress-ի տեսակ է, որը թույլ է տալիս ստեղծել և կառավարել բազմաթիվ կայքերի ցանց՝ մեկ WordPress dashboard-ից։ Դուք կարող եք կառավարել ամեն ինչ ներառյալ կայքերի քանակը, հնարավորությունները, թեմաները և օգտատերերի դերերը։ Հնարավոր է կառավարել հարյուրավոր և հազարավոր կայքեր։

## Ցանց (Network) {#network}

WordPress-ի տերմինաբանությամբ, multisite network-ը այն վայրն է, որտեղ մեկ dashboard-ից կարելի է կառավարել մի շարք **subsites**։ Չնայած multisite network ստեղծելու մեթոդները տարբեր են հոස්ինգի պրովայդերների միջև, վերջնական արդյունքը սովորաբար այն է՝ **wp-config.php** ֆայլում որոշ լրացուցիչ ուղեցույցներ տեղադրել WordPress-ին, որպեսզի այն իմանա, որ գործում է այս հատուկ ռեժիմով։

Multisite network-ի և առանձին (stand-alone) WordPress-ի միջև կան մի քանի տարբերություններ, որոնք մենք կարճ քննարկենք։

## Տվյալների բազա (Database) {#database}

Տվյալների բազան (database) կառուցվածքային և կազմակերպված տվյալների հավաքածու է։ Համակարգչային տերմինաբանության մեջ database-ը այն ծրագրային ապահովումն է, որն օգտագործվում է տվյալները պահելու և կազմակերպելու համար։ Այն պատկերացրեք որպես գրառների գրառնող գրառան (file cabinet), որտեղ դուք տվյալները պահում եք տարբեր բաժիններով՝ ասված են աղյուսակներ (tables)։

WordPress Multisite-ը օգտագործում է մեկ database, և յուրաքանչյուր subsite ստանում է իր առանձին աղյուսակները՝ blog id-ն պրեֆիքսով (prefix) ներառված, այսինքն որ երբ դուք տեղադրում եք network installation և ստեղծում subsite, ունենաք հետևյալ աղյուսակները.

_wp_1_options_ - առաջին ենթաసైట్‌ի op-տեղեկությունների աղյուսակ

_wp_2_options_ - երկրորդ ենթաసైట్‌ի op-տեղեկությունների աղյուսակ

## Հոස්տինգային խմիչք (Hosting provider) {#hosting-provider}

Հոස්տինգային խմիչքը ընկերություն է, որն թույլ է տալիս բիզնեսներին և անհատներին իրենց වෙබ්անայերը հասանելի դարձնել աշխարհաքաղաքային ցանցի միջոցով։ Վեբ-호ස්տինգային խմիչքների կողմից առաջարկվող ծառայությունները տարբեր են, բայց սովորաբար ներառում են වෙබ්անայերի դիզայնը, հյուպիտենտի վրա պահելու տարածքը և ինտերնետի հետ կապը։

## Դոմեն (Domain) {#domain}

Դոմեն անունը այն հասցեն է, որով մարդիկ օգտագործում են ձեր սайտը այցելելու համար։ Այն ցույց է տալիս վեբ-բրաուզերին, թե որտեղ պետք է փնտրի ձեր սайտը։ Հենց ինչպես փողոցային հասցեն, դոմենը այն է, թե ինչպես մարդիկ օնլայն այցելում են ձեր կայքը։ Եվ նման է խանութի առջև ցուցապակ տեղադրելուն։ Եթե ցանկանում եք այցելել մեր կայքը, պետք է ձեր բրաուզերի հասցեում գրեք մեր վեբ-հասցեն՝ [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) _, որտեղ [**ultimatemultisite.com**](http://ultimatemultisite.com) է դոմենի անունը։

## ենթադոմեն (Subdomain) {#subdomain}

Ենթադոմենը հիմնական Դոմեինտի ներքին տեսակի կառուցվածք է, բայց այն փոխարեն օգտագործում է թղթապանակներ՝ կոնտենտը կազմակերպելու համար։ Այն ներկայացվում է որպես [**https://site1.domain.com/**](https://site1.domain.com/)՝ որտեղ _site1_-ը ենթադոմեանի անունն է, իսկ [_domain.com_](http://domain.com) -ը հիմնական դոմենն է։

## ենթատղղի (Subdirectory) {#subdirectory}

Ենթատղիը հիմնական Դոմեինտի ներքին տեսակի կառուցվածք է, որը օգտագործում է թղթապանակներ՝ կոնտենտը կազմակերպելու համար։ Ենթատղիը նույնն է, ինչ ենթաթղիկ (subfolder), և անունները կարելի է փոխարինել։ Այն ներկայացվում է որպես [**https://domain.com/site1**](https://domain.com/site1)՝ որտեղ _site1_ -ը ենթատղու անունն է, իսկ [_domain.com_](http://domain.com) -ը հիմնական դոմենն է։

## ենթաայծ (Subsite) {#subsite}

Սա ենթասայտ է՝ այն ստեղծում եք Multisite ցանցի մեջ։ Այն կարող է լինել **ենթադիր տիրույթ (subdomain)** կամ **ենթածառ (subdirectory)**, կախված նրանից, թե ինչպես է կարգավորված ձեր WordPress Multisite-ի ներդրումը։

## Սուպեր ադմին (Super Admin) {#super-admin}

WordPress Super Admin-ը օգտվողի դերը, որն ամբողջ հնարավորություններով է կառավարում բոլոր ենթասայտերը Multisite ցանցում։ Multisite օգտատերերի համար այն **ամենաբարձր մուտքի մակարդակն** է, որը կարող եք տրամադրել ձեր WordPress-ինստալացիայի համար։

## Plugin (Պլագին) {#plugin}

Ընդհանուր առմամբ, պլագինը կոդի հավաքածու է, որը ավելի լրացուցիչ ֆունկցիոնալություն է ավելացնում ձեր WordPress-ի վեբсайట్‌ին։ Սա կարող է լինել որքան պարզ՝ մուտքի լոգոյի փոփոխություն, կամ այնքան բարդ, որքան է էլ էլեկտրոնական առևտրային ֆունկցիոնալություն ավելացնելը։ _Woocommerce and Contact Form_-ը պլագինների օրինակներ են։

WordPress Multisite-ում պլագինները կարելի է տեղադրել միայն Super Admin-ի կողմից՝ ցանցի ադմինի վիճակագրից (network admin dashboard)։ Ենթասայտերի ադմինները կարող են ակտիվացնել և անջատել պլագիններ միայն իրենց ենթասայտում։

## Themes (Թեմաներ) {#themes}

WordPress-ի թեմաները (_գրաֆիկա, ոճային տեսքեր և կոդ_) այն ֆայլերի խումբն են, որոնք որոշում են վեբсайթի ընդհանուր տեսքը։ Այն ապահովում է բոլոր առջևի (front-end) ոճային կարգավորումները, ինչպիսիք են տառատեսակի ոճը, էջերի դասավորությունը, գույները և այլն։

Պլագինների նման, WordPress Multisite-ում թեմաները կարելի է տեղադրել միայն Super Admin-ի կողմից և ակտիվացնել ենթասայտի մակարդակով՝ ենթասայտերի ադմինների կողմից։

## Site Template (Վեբсайթի Շաբլոն) {#site-template}

**Site Template**-ը այն ստանդարտ վեբсайթն է, որը կարելի է օգտագործել որպես հիմք՝ ձեր ցանցում նոր վեբсайטים ստեղծելիս։

Սա նշանակում է, որ դուք կարող եք ստեղծել հիմնական վեբсайт, ակտիվացնել տարբեր պլագիններ, ընտրել ակտիվ թեմա և այն ցանկացած ձևով անհատականացնել։ Այնուհետև, երբ ձեր հաճախորդը ստեղծում է նոր հաշիվ, նրան տրվում է ձեր հիմնական վեբсайթի պատճենը՝ բոլոր անհատականացումներով և բովանդակությամբ արդեն լրացված։

## Domain Mapping (Տիրույթի Մ్యాփինգ) {#domain-mapping}

WordPress-ի հետ **Domain mapping** (Տիրույթի քարտեզագրումը) այն միջոց է, որով կարող եք օգտատերին ուղղել ճիշտ հոස්տին՝ կայքի հասցեի միջոցով։ WordPress Multisite-ում են ստեղծվում են ենթակայքերը՝ օգտագործելով թղթապանակ (subdirectory) կամ ենթադարական տիրույթ (subdomain)։ Domain mapping-ը թույլ է տալիս ենթակայքի օգտատերերին օգտագործել վերին մակարդակի ծավալուն տիրույթ, ինչպես [**joesbikeshop.com**](http://joesbikeshop.com)՝ իրենց կայքի հասցեն ավելի պրոֆեսիոնալ դարձնելու համար։

## SSL {#ssl}

SSL-ը նշանակում է **Secure Sockets Layer** (Անվտանգ սոկետային շերտ)։ Սա թվային վկայական է, որը հաստատում է կայքի անհատականությունը և թույլ է տալիս կոդավորված կապ։ Իսկ այսօր այն օգտագործվում է որպես ինտերնետային կապը անվտանգ պահելու ստանդարտ տեխնոլոգիա և ցանկացած զգայուն տվյալների պաշտպանության համար, որոնք փոխանցվում են երկու համակարգերի միջև՝ կանխելով թաքնված մարդկանց տեղեկությունները կարդալը և փոփոխելը, ներառյալ հնարավոր անձնական մանրամասները։ Ժամանակակից բրաուզերները պահանջում են SSL, ինչը դարձնում է այն անհրաժեշտ՝ կայք ստեղծելիս և գործարկելիս։

## Media (Մեդիա) {#media}

Media-ն այն պատկերներն, աուդիոյի, տեսանյութերի և այլ ֆայլերը, որոնք կազմում են կայքը։

Ցանցային կայքերը WordPress Multisite-ում կիսում են մեկ ընդհանուր բազա (database), սակայն նրանք պահպանում են առանձին ուղիներ՝ մեդիա ֆայլերի համար համակարգի ֆայլային համակարգում։

Ստանդարտ WordPress-ի դիրքը (`wp-content/uploads`) մնում է նույնը, սակայն այն փոխվում է՝ արտացոլելով կանցիկ կայքի եզակի ID-ն։ Հետևաբար, ցանցային կայքի մեդիա ֆայլերը հայտնվում են որպես `wp-contents/uploads/site/[id]`։

## Permalinks (Հորիզոնական հղումներ) {#permalinks}

Permalinks-ը ձեր անհատական բլոգային գրառման կամ էջի մշտական URL-ներն են ձեր կայքում։ Permalinks-ը նաև հայտնի է որպես **pretty links** (քարեւոր հղումներ)։ Ըստ սխալի, WordPress-ի URL-ները սովորաբար օգտագործում են query string ձևաչափը, որը տեսքով նման է հետևյալին.

[**http://www.example.com/registration**](http://www.example.com/registration)

## Ultimate Multisite {#ultimate-multisite}

Ultimate Multisite એ WordPress નું એક plugin છે, જે WordPress Multisite ઇન્સ્ટોલેશન માટે બનાવવામાં આવ્યું છે, જે તમારા WordPress ઇન્સ્ટોલેશનને સાઇટ્સના પ્રીમિયમ નેટવર્ક - જેમ કે [WordPress.com](https://WordPress.com) - માં ફેરવે છે – જે ક્લાયન્ટ્સને માસિક, ત્રિમાસિક અથવા વાર્ષિક ફી દ્વારા સાઇટ્સ બનાવવા દે છે (તમે ફ્રી પ્લાન પણ બનાવી શકો છો).

## Checkout Form {#checkout-form}

Checkout Form એ એક સિંગલ અથવા મલ્ટી-સ્ટેપ ઓર્ડર ફોર્મ છે જેમાં Ultimate Multisite રજીસ્ટ્રેશન દ્વારા સબસાઇટ, મેમ્બરશિપ અને યુઝર એકાઉન્ટ બનાવવાનો સમાવેશ થાય છે. તેમાં વિવિધ ફીલ્ડ્સ અને પેમેન્ટ ફોર્મ હોય છે જેનો ઉપયોગ યુઝરે સાઇન-અપ પ્રક્રિયા દરમિયાન કરવો પડે છે.

## Webhook {#webhook}

Webhook (જેને વેબ કોલબેક અથવા HTTP પુશ API પણ કહેવામાં આવે છે) એ એક એવી રીત છે જેના દ્વારા કોઈ એપ અન્ય એપ્લિકેશન્સને રીઅલ-ટાઇમ માહિતી પૂરી પાડી શકે છે. વેબહુબ ડેટાને તે જ સમયે અન્ય એપ્લિકેશન્સને પહોંચાડે છે, એટલે કે તમને તરત જ ડેટા મળે છે.

**Ultimate Multisite webhooks** અનંત શક્યતાઓ ખોલે છે, જે નેટવર્ક એડમિનને તમામ પ્રકારના વિચિત્ર-પણ ઉપયોગી ઇન્ટિગ્રેશન કરવા દે છે, ખાસ કરીને _Zapier અને IFTTT_ જેવા સેવાઓ સાથે જોડીને તેનો ઉપયોગ કરવામાં આવે તો.

## Events {#events}

Event એ યુઝર અથવા અન્ય સ્રોતની ક્રિયા (જેમ કે માઉસ ક્લિક)ના પરિણામે થતી કોઈ ક્રિયા છે. Ultimate Multisite તમારા સમગ્ર નેટવર્કની અંદર થતી તમામ ઇવેન્ટ્સ અને લોગ્સનો રેકોર્ડ રાખે છે. તે તમારી મલ્ટીસાઇટમાં થતી વિવિધ પ્રવૃત્તિઓ, જેમ કે પ્લાન બદલવા જેવી બાબતોને ટ્રેક કરે છે.
