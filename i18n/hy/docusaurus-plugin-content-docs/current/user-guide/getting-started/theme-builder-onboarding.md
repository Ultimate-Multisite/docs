# Theme Builder Onboarding Flow

Superdav AI Agent v1.12.0-ը ներդրում է ուղորդված **Theme Builder onboarding flow**՝ ձեզ օգնելու ստեղծել հատուկ բլոկային թեմա (custom block theme) ձեր սկզբնական կարգավորման ընթացքում։ Սա հին Site Builder մոդիֆիկացիան փոխարինում է ավելի պլյունտ և օգնության մեջ գտնվող մոտեցմամբ։

## Ի՞նչ է Theme Builder Onboarding Flow-ը

Theme Builder onboarding flow-ը ինտերակտիվ կարգավորման ուղորդականացույց է, որը.

- Ձեզ ուղղում է դիզայնի որոշումներով (գույներ, տիպոգրաֆիա, դասավորություն)
- Հավաքում է ձեր կայքի տեսողական անհատականության նախընտրությունները
- Ստեղծում հատուկ բլոկային թեմա՝ հարմարեցված ձեր կարիքներին
- Ավտոմատ կերպով ակտիվացնում է թեման, երբ այն պատրվում է

Այս flow-ը աշխատում է **Setup Assistant agent**-ի միջոցով, որը հարցեր է տալիս հստակեցման համար և ձեր թեման կառուցում է քայլ առ քայլ։

## Theme Builder Onboarding-ի սկիզբը

### Առաջին անգամ աշխատելու կարգավորում (First-Run Setup)

Երբ առաջին անգամ Superdav AI Agent-ը գործարկեք նոր WordPress տեղադրման վրա, դուք կտեսնեք.

```
Welcome to Superdav AI Agent!

What would you like to do?
1. Build a custom theme (Theme Builder)
2. Use an existing theme
3. Skip setup for now
```

Ընտրեք **"Build a custom theme"**՝ Theme Builder onboarding flow-ի մեջ մտնելու համար։

### Ձեռքով ակտիվացում (Manual Activation)

Դուք կարող եք նաև ցանկացած պահի սկսել Theme Builder onboarding-ը՝ խնդրելով.

```
"Start the Theme Builder onboarding"
```

կամ

```
"Help me create a custom theme"
```

## Onboarding քայլերը

### Քայլ 1. Մոդիի ընտրություն (Mode Selection)

Setup Assistant agent-ը հարցնում է ձեր նախընտրության մասին.

```
How would you like to build your theme?
- Guided (I'll ask questions and build it for you)
- Hands-on (I'll show you options and you decide)
```

**Guided mode**-ը խորհուրդ է տրվում մեծամասն օգտվողների համար. agent-ը դիզայնի առաջարկներ կատարում է ձեր ոլորտի և նպատակների հիման վրա։

### Քայլ 2. Կայքի բնութագրում (Site Specification)

Ձեզ կհարցվեն ձեր կայքի մասին.

- **Սайտի նպատակը**: Էլեկտրոնային առևտրի կայք (E-commerce), բլոգ, պորտֆոլիո, SaaS և այլն։
- **Նպատակային լսարան**: Ո՞վ են ձեր այցելուները։
- **Բրենդի գույներ**: Հիմնական և երկրորդական գույներ (կամ «ես չգիտեմ»)։
- **Տոնը**: Պրոֆեսիոնալ, ստեղծագործական, խաղային, մինիմալիստական և այլն։

Այս տեղեկությունները պահվում են ձեր **site_brief** հիշողության մեջ, որը հետագա նիստերում օգտագործում են գործակիցները։

### Քայլ 3. Дизайн համակարգի որոշումներ (Design System Decisions)

Գործակիցը ձեզ ուղղորդում է դիզայնի տոկենների ընտրության մեջ.

- **Տիպոգրաֆիկա**: Ֆոնտային ընտանիք (serif, sans-serif, monospace) և չափերի սանդղակ
- **Գույնի паլիտրա (Color palette)**: Հիմնական, երկրորդական, շեշտադրող և նեյտրալ գույներ
- **Տարածություն (Spacing)**: Կոմպակտ, սովորական կամ լայն տարածություններով դասավորություն
- **Շարժում (Motion)**: Անիմացիաներ և անցումներ (եթե ցանկանում եք)

### Քայլ 4. թեմայի ստեղծում (Theme Generation)

Setup Assistant գործակիցը ձեր կատուտ բլոկային թեմային կառուցում է հետևյալով.

- `theme.json`՝ պարունակող ձեր բոլոր դիզայնի տոկենները
- Հաճախակի դասավորությունների համար բլոկային տեխնիկներ (հիմնական էջ, բլոգ, կոնտակտ)
- Ձեր դիզայն համակարգին համապատասխանող հատուկ բլոկային ոճեր
- Թեմայի 메տատվյալներ և WordPress-ի աջակցության հայտարարություններ

### Քայլ 5. Ակտիվացում և ստուգում (Activation and Verification)

Թեման ավտոմատ կերպով ակտիվանում է, և դուք կտեսնեք հետևյալը.

```
✓ Ձեր կատուտ թեման հիմա գործում է։
  Թեմայի անուն: [Ձեր կայքի անուն] Theme
  Գույներ: [Հիմնական] / [Երկրորդական]
  Տիպոգրաֆիկա: [Ֆոնտային ընտանիք]

  Մուտք գործեք ձեր կայքը՝ նոր դիզայնը տեսնելու համար։
```

Այնուհետև կարող եք այցելել ձեր կայքը՝ համոզվելու համար, որ թեման ճիշտ է ցուցադրվում։

## Սайտի սահմանում և site_brief հիշողություն (Site Specification and site_brief Memory)

Onboarding-ի ընթացքում գործակիցը ձեր կայքի սահմանումները գրանցում է **site_brief** հիշողության կատեգորիայում։ Սա ներառում է.

- Սайտի նպատակը և նպատակները
- Նպատակային լսարանը
- Բրենդի գույները և տոնը
- Դիզայնի նախասիրությունները
- Բովանդակության կառուցվածքը

### Ինչո՞ւ է site_brief-ը կարևոր

Հետագա նիստերում գործակիցները հղվում են ձեր site_brief-ին՝ հետևելու համար.

### Ձեր site\_brief-ը դիտելու համար

Դու կարող ես հարցնել օգտատիրոջին.

```
"Show me my site brief"
```

կամ

```
"What do you know about my site?"
```

Այս գործակիցը կցուցադրի ձեր պահված site-ի նկարագրությունը։

## Onboarding-ից հետո անհատականացում

Theme Builder-ի onboarding-ը ավարտվելուց հետո դու կարող ես.

### Օգտագործել Design System Aesthetics հմտությունը

Խնդրիր դիզայնի մանրամասնումներ.

```
"Refine the typography to be more modern"
```

կամ

```
"Adjust the color palette to be warmer"
```

**Design System Aesthetics** հմտությունը կուղղորդի ձեզ թիրախային դիզայնի թարմացումների ուղղությամբ։

### Խմբագրել theme.json-ը անմիջապես

Առաջադեմ օգտվողների համար, խմբագրեք `/wp-content/themes/[theme-name]/theme.json`՝ կարգավորելու համար.

- Color tokens (Գույնային տոկեններ)
- Typography scales (Տիպոգրաֆիկայի մասշտաբները)
- Spacing values (Հարևանության արժեքները)
- Border and shadow definitions (Կոճղերի և ստվերների սահմանումները)

### Ստեղծել Custom Block Templates

Օգտագործեք WordPress block editor-ը՝ ստեղծելու հատուկ տիպեր.

- Homepage layouts (Հիմնական էջերի դասավորություններ)
- Blog post pages (Բլոգային հոդվածների էջեր)
- Product pages (Ապրանքների էջեր)
- Contact forms (Կոնտակտային ձևեր)

## Համեմատություն. Հին vs. Նոր Onboarding

| Առանձնահատկություն | Site Builder (Հինը) | Theme Builder (Նորը) |
|---|---|---|
| Կազմաձևման մեթոդ | Wizard-based form (Վիզարդային ձև) | Agent-guided conversation (Գործակցով ուղղորդված զրույց) |
| Theme-ի ստեղծում | Limited templates (Սահմանափակ տիպեր) | Custom scaffolding (Մանրամասն կառուցվածք) |
| Design tokens | Manual entry (Ձեռքով մուտքագրում) | Guided decisions (Ուղղորդված որոշումներ) |
| Ճկունություն | Fixed options (Ֆիքսված տարբերակներ) | Customizable (Անհատականացվող) |
| Ապագա թարմացումներ | Not referenced (Հղում չկա) | Stored in site\_brief (Պահվում է site\_brief-ում) |

## Խնդրի լուծումներ (Troubleshooting)

**Onboarding-ը չի ավարտվել**
- Վերսկսեք հոսքը. "Start the Theme Builder onboarding" (Սկսել Theme Builder-ի onboarding-ը)
- Ստուգեք, որ ձեր WordPress-ը թարմացված է։
- Հաստատեք, որ Setup Assistant գործակիցը 켜ված է։

**Իմ site_brief-ը չի օգտագործվում**
- Հաստատեք, որ ասոնտը (agent) հուշում է մուտք գործել հիշողություն (memory)։
- Հարցրեք ասոնտին՝ «հիշեցնել իմ site brief-ը» (recall my site brief)։
- Ստուգեք, թե արդյոք հիշողությունը 켜ված է ձեր կարգավորումներում։

**Ստեղծված թեման համապատասխանում չէ իմ նախընտրություններին**
- Օգտագործեք Design System Aesthetics հմտությունը՝ այն մաքրելու համար (refine it)։
- Հարցրեք ասոնտին՝ «կառուցել թեման [հատուկ փոփոխություններով]» (regenerate the theme with [specific changes])։
- Խմբագրեք theme.json-ը ուղղակիորեն՝ ճշգրիտ վերահսկողության համար։

## Հաջորդ քայլերը

Theme Builder-ի ներառումն ավարտելուց հետո.

1. **Ստուգեք ձեր կայքը**: Բ సందర్శించండి ձեր կայքում՝ նոր թեման տեսնելու համար։
2. **Մաքրեք դիզայնը**: Օգտագործեք Design System Aesthetics հմտությունը կարգավորումների համար։
3. **Ավելացրեք բովանդակություն**: Սկսեք կայքի բովանդակության կառուցումը։
4. **Բացահայտեք հնարավորությունները**: Իմանալ այլ ասոնտների հմտությունների մասին, ինչպիսիք են scaffold-block-theme և activate-theme։
