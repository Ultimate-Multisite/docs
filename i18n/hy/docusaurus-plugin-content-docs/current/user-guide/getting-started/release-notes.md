# Հայտարարության նշումներ (Release Notes) {#release-notes}

## Տարբերակ 2.13.0 — Հրապարակվել է 2026-06-05-ին {#version-2130--released-on-2026-06-05}

- Նոր՝ հաճախորդի հաշվի, գնման, հաշվարկների, կայքերի, ինվոիցների, տեխլատերների փոփոխության և ադմինիստրացիոն գործողությունների համար դոմենային քարտեզագրման (domain mapping) հոսքերի համար սուվերեն-տենանտի (sovereign-tenant) աջակցություն՝ տենանտային ցանցերին թույլ տալով ուղղել հաճախորդներին մայր կայքը՝ կառավարված գործողությունների համար։
- Նոր՝ կրկնվող անդամակցության համար թարմացման (renewal) վերահսկողության ստուգումներ են ավելացվել. այսպիսով, եթե պահված հաշվարկային համաձայնություն, բաժանորդագրություն կամ վաուլտի թոքեն բացակայում է, գեյթવેները կարող են անջատել ավտոմատ թարմացումը։
- Նոր՝ սպասվող կայքերի ստեղծման համար HMAC-ով հաստատված loopback publication (հրապարակում)՝ գնման և կայքի պատրաստման միջև ավելի հուսալի դարձնելու համար այն հոස්թերին, որտեղ հետևի գործողությունները ուշացվում են։
- Նոր՝ SSO URL-ների, գնման ձևաչափերի (checkout-form base domains) և ավտոմատ դոմենային գրանցումների ստեղծման համար մշակողի (developer) հարթակներ (extension points)։
- Ուղղում՝ SSO-ն ավելի հուսալի է քարտեզագրված դոմենների, անանուն բրոքերների այցելությունների, ելքից դուրս գալու և պլայնգինային կախվածությունների հակասությունների միջև։
- Ուղղում՝ սպասվող կայքերի ստեղծման դեպքում հաջորդաբար վերականգնվում է հին publication flag-ներից և չի թողնում հաճախորդները կայքի ստեղծման էջում։
- Ուղղում՝ ընդհանուր գնման ձևաչափերի (shared checkout-form base domains) համար դոմենային գրանցումներ չեն ստեղծվում, և երբ ոչ մի ինտեգրացիա չի ակտիվ է, անօգտագործված հոස්թ-փրովայդերի (host-provider) հետևի գործողությունները բացառվում են։
- Ուղղում՝ գնումների, հաշվարկային հասցեների, գաղտնաբառի վերականգնման, էլ. փոստի հաստատման, տեխլատերի փոփոխության, տուրերի և հաճախորդի դաշտի (customer dashboard) սահմանային դեպքերը չեն արգելում սովորական հաճախորդի հոսքերը։
- Ուղղում՝ բազմապատկերված էլ. փոստերը պահում են ստացողներին գաղտնի, միևնույն ժամանակ խուսափելով SMTP/pluign fatal error-ներից այն դեպքերում, երբ ստացողների ցուցակները կամ փոստային տրանսպորտները ձախողվում են։
- Ուղղում՝ անդամակցության թարմացումները, ժամկետի ցուցադրումը և վճարումների հավաքագրման սահմանային դեպքերը խուսափում են անմիջական ժամկետներից, կrashes-ից կամ անհրաժեշտ վճարումների բացակայությունից։
- Բարելավում՝ WordPress-ի հետ համատեղելիությունը ստուգվում է մինչև 7.0 տարբերակները, արտադրական Vue asset-ները կառուցվում են npm-ից՝ վերաշարադրվելով, և Cypress end-to-end ծածկույթը այժմ ավելի շատ գնման, սահմանման, SSO և գեյթվային հոսքերը ստուգում է։

## 2.12.0-րդ տարբերակը — թողարկված է 2026-05-15-ին {#version-2120--released-on-2026-05-15}

- Նոր: Ավելացվել է Hostinger (hPanel) որպես աջակցող հոස්տիչ պրովայդեր՝ տիրույթների քարտեզագրման (domain mapping) ինտեգրացիայի հետ։
- Նոր: Site Exporter-ը այժմ մշակում է ցանցային ներմուծման փաթեթները՝ ցանցով լայնածավալ կայքերի վերականգնման համար հարթեցնելու նպատակով։
- Ուղղում. BCC բազմակի ուղարկվող էլ․ նամակները այժմ օգտագործում են անհայտ ստացողներ (undisclosed-recipients) հեդեր, որպեսզի կանխենք ստացողների հասցեները բացահայտել։
- Ուղղում. Ամսագրման ժամկետը այլևս չի կորչում՝ անվճար արժեքով պահպանելիս։
- Ուղղում. Stripe-ի անդամակցության թարմացումները հաջողությամբ մաքրում են զեղչերը՝ առանց հնացած deleteDiscount API-ի կանչի։
- Ուղղում. Տիրույթների քարտեզագրված կայքերի SSO վերահղումները սահմանափակվել են՝ անվերջ վերահղման ցիկլերը կանխելու համար։
- Ուղղում. Ստեղծման ու ստացման (Setup wizard) պատկերների ընտրությունը այժմ ճիշտ թարմացնում է հիմնական տվյալների մոդելը։
- Ուղղում. Site Exporter CLI-ն պահպանում է ցանցային կայքի ճիշտ սխալ값을։
- Բարելավում. Պլագինի փաթեթից հեռացվել է wp-cli-ը, ինչը նվազեցնում է պլագինի չափը։

## 2.11.0-րդ տարբերակը — թողարկված է 2026-05-11-ին {#version-2110--released-on-2026-05-11}

- Նոր: Site exports-ը այժմ փաթեթավորում է ինքնաբուժվող `index.php`՝ որպեսզի ZIP-ը կարելի է տեղադրել նոր հոස්տիչի վրա առանց առանձին պլագին տեղադրելու։
- Նոր: Network export-ը թույլ է տալիս ադմիններին Site Export admin էջից արտահանել բոլոր ենթակայքերը մեկ արխիվով։
- Նոր. Site Templates plan toggle-ը հնարավոր է այժմ կիրառվում փոխարեն, որպես հետընթաց շղթա (fallback chain), ճիշտ սահմանափակելով տեխնոլոգիական պլանների համար տեխնոլոգիական տողերի հասանելիությունը։
- Նոր. Checkout form editor-ը զգուշացում է, երբ արտադրանք ավելացվում է՝ առանց անհրաժեշտ դաշտի կոնֆիգուրացիայի։
- Նոր. Import/Export settings տաբը հստակ նկարագրում է իր շրջանակը և ուղղակի հղում է Site Export գործիքին։

## 2.10.0-րդ տարբերակը — թողարկված է 2026-05-05-ին {#version-2100--released-on-2026-05-05}

- Նոր՝ PayPal-ի ուղեցույթով ստուգման գործիքը ձեռքով մուտքագրման համար OAuth թողարկումով, որը հնարավորություն է տալիս անխափան կերպով գեյթվեյի կազմաձևում։
- Նոր՝ Տողերի փոփոխության ժամանակ օգտատիրոջ պանելը նոր դիզայնով է վերաձևավորված՝ ներառելով current-template քարտ, մշտական գրիդ և **Reset current template** կոճակ։
- Ուղղում. Տողերի փոփոխությունը այլևս չի կախում UI-ն AJAX-ի ձախողումից։
- Ուղղում. Տողերի փոփոխության թույլտվությունները պաշտպանված են անավետ մուտքից։
- Ուղղում. Սайտի գերագրման ներմուծումները վավելացվում են ստուգվելուց առաջ։
- Ուղղում. Առանձնահատուկ հասցեի հայտարարությունը հայտնվում է, երբ հասցեն դատարկ է։
- Ուղղում. PHP 8.1-ի null-to-string անվանման նախազգուշացումները լուծված են։
- Ուղղում. လက်ရှိ lazy-loaded տարրերը init hook-ից առաջ դարձվել են՝ ժամանակային խնդիրներից խուսափելու համար։
- Ուղղում. SSO-ի (Single Sign-On) ուղին հարգվում է բոլոր մուտք գործման հոսքերով։
- Ուղղում. Դատարկ սайտի անհատական տարբերակները պահպանվում են խմբագրելիս։

## 2.9.0-րդ տարբերակը — Հայտարարվել է 2026-04-30-ին {#version-290--released-on-2026-04-30}

- Նոր՝ Ավելացվել է մեկ սайտի արտահանման և ներմուծման հնարավորություն **Tools > Export & Import** բաժնում։
- Ուղղում. ZIP ֆայլերը այժմ տրվում են հավաստագրված ներբեռնման կետով (authenticated download endpoint)։
- Ուղղում. SQL injection-ի և հարցուqueries-ի խնդիրները ուղղվել են պահակ արտահանման/ներմուծման հարցումներում։
- Ուղղում. Երբ ադմինը ձեռքով հաստատում է օգտատիրոջ email-ը, չի հրապարակվում պահակ սайտը։
- Ուղղում. Երբ անդամակցությունը բացակայում է, մենագրված pending_site գրանցումները մաքրվում են։
- Ուղղում. Կարգավորումների նավիգացիայի padding-ը և որոնման հարևանական նավիգացիան ուղղվել են։
- Ուղղում. Pending site-երը այժմ առաջինն են All Sites տեսքի մեջ։
- Ուղղում. Էկրանի պատկերների մատակարարը (mShots) User-Agent header-ը հավելված է՝ 403 սխալներից խուսափելու համար։
- Ուղղում. Import cron-ի ժամանակային կախվածությունը լուծվել է։
- Ուղղում. Tour ID-երը նորմալացվել են ներքին կարգավորումների բանալիների մեջ՝ օգտատիրոջ կարգավորումներում սիմվոլներով (underscores)։
- Բարելավում. Ավելի լավ համատեղելիության համար ZipArchive-ը փոխարինվել է Alchemy/Zippy-ի տեղում։
