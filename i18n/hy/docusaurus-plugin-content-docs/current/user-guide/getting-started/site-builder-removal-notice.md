# Կայքի կառուցման ռեժիմի հեռացման մասին հայտարարություն {#site-builder-mode-removal-notice}

**Site Builder mode-ը հեռացվել է Superdav AI Agent v1.12.0-ում։** Եթե դուք օգտագործում էիք Site Builder mode-ը, ապա պետք է անցնեք **Setup Assistant agent-ին**, որը նախատեսված է թեմայի ստեղծման և կայքի կարգավորման համար։

##Ի՞նչ է պատահել։

### Site Builder Mode (Հին տարբերակ) {#what-happened}

Site Builder mode-ը վիզարդային ինտերֆեյս էր հետևյալու համար.

- Տողերով կայքեր ստեղծելու համար
- Հիմնական կարգավորումներ կատարելու համար
- Թեմա ընտրելու համար
- Սկզբնական բովանդակություն սահմանելու համար

### Ի՞նչ է այն փոխարինում։ {#site-builder-mode-legacy}

**Setup Assistant agent-ը** հիմա կառավարում է Site Builder mode-ի բոլոր ֆունկցիոնալությունները հետևյալով.

- Ավելի պարզ և օգտագործողի ուղղված սահմանում
- Ավելի լավ թեմայի անհատականացման տարբերակներ
- Theme Builder-ի ներառելիության հետ համատեղելիություն
- Հաջորդ քայլերի համար կայուն `site_brief` հիշողություն

## Եթե օգտագործում էիք Site Builder Mode-ը {#what-replaced-it}

### Ձեր կայքերը անվտանգ են {#if-you-were-using-site-builder-mode}

- Site Builder mode-ով ստեղծված գոյություն ունեցող կայքերը շարունակում են աշխատել
- Տվյալների կորուստի կամ կայքի խափանում չկա
- Դուք կարող եք շարունակել սովորական պարզապես կառավարել ձեր կայքերը

### Անցեք Setup Assistant Agent-ին {#your-sites-are-safe}

Նոր կայքերի ստեղծման կամ թեմայի փոփոխությունների համար օգտագործեք Setup Assistant agent-ը.

```
"Help me set up a new site"
```

կամ

```
"Start the Theme Builder onboarding"
```

Setup Assistant agent-ը նույն ֆունկցիոնալությունը տրամադրում է, բայց ավելի մեծ ճկունությամբ։

## Համեմատություն. Site Builder vs. Setup Assistant {#migrate-to-setup-assistant-agent}

| Առանձնահատկություն | Site Builder (Հեռացված) | Setup Assistant (Նոր) |
|---|---|---|
| Սահմանման մեթոդ | Վիզարդային ձևաթուղթ | Agent-ի հետ զրույց |
| Թեմայի ընտրություն | Նախապես սահմանված տողեր | Հատուկ գեներացում |
| Անհատականացում | Սահմանափակ տարբերակներ | Ամբողջ դիզայնի համակարգ |
| Site brief | Պահվում չէ | Կայուն հիշողություն |
| Ապագա նիստեր | Կրկնվող սահմանում | Օգտագործել պահված site_brief-ը |
| Ճկունություն | Ֆիքսված ընթացակարգ | Հետադարձ արձագանքող զրույց |

## Անցում Setup Assistant Agent-ին {#comparison-site-builder-vs-setup-assistant}

### Նոր կայքերի համար {#migrating-to-setup-assistant-agent}

Ստեղծող ռեժիմի (Site Builder mode) փոխարեն՝

1. Խնդրում. «Օգնեք ինձ նոր կայք ստեղծել»։
2. Ստեղծման օգնական գործակիցը ձեզ կուղղորդի հետևյալով.
   - Կայքի նպատակ և նպատակներ
   - Թիրախային լսարան
   - Բրենդի ինքնություն
   - Թեմայի ստեղծում
   - Սկզբնական կոնֆիգուրացիա

### Գոյություն ունեցող կայքերի համար {#for-new-sites}

Եթե ունեք գոյություն ունեցող կայք Site Builder mode-ից.

1. Դուք կարող եք շարունակել օգտագործել այն ինչպես կա։
2. Թեմայի թարմացման համար խնդրեք. «Իմ կայքի վերափոխում» (Redesign my site)։
3. Ստեղծման օգնական գործակիցը կօգնի ձեզ նոր թեմա ստեղծել։
4. Ձեր կայքի տվյալները չեն փոխվի։

### Թեմայի փոփոխությունների համար {#for-existing-sites}

Site Builder mode-ի թեմա ընտրության փոխարեն՝

1. Խնդրեք. «Փոխել իմ թեման» (Change my theme)։
2. Ստեղծման օգնական գործակիցը կկատարի հետևյալը.
   - Հարցում է ձեր դիզայնի նախասիրությունների մասին
   - Ստեղծում է ստեղնաշարի (custom) թեմա
   - Ակտիվացնում այն ձեր կայքում

## Հիմնական տարբերություններ {#for-theme-changes}

### Site Builder Mode {#key-differences}

```
1. Ընտրել տեխնոլոգիան (template)
2. Ընտրել թեմա
3. Կոնֆիգուրացնել հիմնական կարգավորումները
4. Ավարտ
```

### Setup Assistant Agent {#site-builder-mode}

```
1. Նկարագրել ձեր կայքի նպատակը
2. Սահմանել ձեր թիրախային լսարանը
3. Ընտրել դիզայնի նախասիրությունները
4. Գործակիցը ստեղծում է ստեղնաշարի (custom) թեմա
5. Գործակիցը ակտիվացնում է թեման
6. Կայքի ամփոփումը պահվում է հետագա նիստերի համար
```

## Setup Assistant Agent-ի առավելությունները {#setup-assistant-agent}

### Ավելի ճկուն {#benefits-of-setup-assistant-agent}

- Նկարագրեք ձեր կայքը բնական լեզվով
- Ստացեք ստեղնաշարի (custom) առաջարկներ
- Հարմարվեք ձեր հատուկ կարիքների հետ

### Ավելի լավ անհատականացում {#more-flexible}

- Ստեղնաշարի (custom) թեմայի ստեղծում
- Դիզայնի համակարգի որոշումներ
- Հաստատուն դիզայնի տոկեններ (persistent design tokens)

### Հաստատուն հիշողություն {#better-customization}

- Ձեր `site_brief`-ը պահվում է
- Առաջին գործակիցները կհասկանան ձեր կայքը
- Չպետք է կրկնել ստեղծման տեղեկատվությունը

### Ինտեգրված ընթացիկա (Integrated Workflow) {#persistent-memory}

- Թեմայի կառուցման ներդրում (Theme Builder onboarding)
- Դիզայնի համակարգի էսթետիկայի հմտություն
- Տեսանելիության վերահսկողության կարողություն (Ability Visibility controls)
- Այն բոլորը միասին աշխատում են անխափան։

## Խնդրի լուծումներ (Troubleshooting) {#integrated-workflow}

### Եթե չեմ գտնում Site Builder mode-ը {#troubleshooting}

Site Builder mode-ը հեռացվել է։ Փոխարեն օգտագործեք Setup Assistant agent-ը.

"Ինձ նոր սայթ կազմակերպելու օգնեք"

### Ցանկանում եմ վերստեղծել սայթը Site Builder-ից {#i-cant-find-site-builder-mode}

Դուք կարող եք այն վերստեղծել Setup Assistant գործակցի միջոցով.

1. Հարցում․ «Ինձ նոր սայթ կազմակերպելու օգնեք»
2. Նկարագրեք ձեր սկզբնական սայթի նպատակը և դիզայնը։
3. Գործակիցը (agent) կստեղծի նման թեմա (theme)։
4. Ձեր `site_brief`-ը կպահվի հետագա հղումների համար։

### Իմ գոյություն ունեցող Site Builder սայթը չի աշխատում {#i-want-to-recreate-a-site-from-site-builder}

Site Builder-ով ստեղծված գոյություն ունեցող սայթերը դեռ աշխատում են։ Եթե խնդիրներ եք հանդիպում.

1. Ստուգեք, թե արդյոք ձեր թեման (theme) դեռ ակտիվ է
2. Հաստատեք, որ ձեր plugin-ները 켜ված են (enabled)
3. Ստուգեք WordPress-ի սխալների գրանցումը (error logs)
4. Եթե խնդիրները շարունակվում են, կապվեք աջակցության հետ

### Արդյոք կարող եմ օգտագործել հին Site Builder տեմplատները {#my-existing-site-builder-site-isnt-working}

Site Builder-ի տեմplատները այլևս հասանելի չեն։ Սակայն.

- Ձեր գոյություն ունեցող սայթերը դեռ աշխատում են
- Դուք կարող եք նման սայթեր վերստեղծել Setup Assistant գործակցի միջոցով
- Setup Assistant գործակիցը առաջարկում է ավելի շատ անհատականացման տարբերություններ

## Հաջորդ քայլերը (Next Steps) {#can-i-still-use-my-old-site-builder-templates}

1. **Նոր սայթերի համար**: Օգտագործեք Setup Assistant գործակցի
2. **Գոյություն ունեցող սայթերի համար**: Շարունակեք օգտագործել դրանք ինչպես կան
3. **Թեմայի փոփոխության համար**: Հարցրեք Setup Assistant գործակցից օգնության մասին
4. **Դիզայնի մշակման համար**: Օգտագործեք Design System Aesthetics հմտությունը (skill)
