# Կայքի Սահմանումը (Site Specification) Գործունեության Հմտություն {#site-specification-skill}

**Site Specification** հմտությունը կառուցվածքային մոտեցում է՝ ձեր կայքի նպատակները, լսարանը և բրենդի ինքնությունը գրանցելու համար։ Այս տեղեկատվությունը պահվում է ձեր **site_brief** հիշողության մեջ, որը օգտագործում են ասոնտները (agents) տարբեր նիստերի ընթացքում համապարփակ և համատեքստով գիտակցված օգնություն տալու համար։

## Ի՞նչ է Site Specification-ը {#what-is-site-specification}

Site Specification-ը այն գործընթացն է, երբ փաստաթղթավորում եք.

- **Կայքի նպատակը (Site purpose)**: Ինչ է անում ձեր կայքը և ինչու է գոյություն ունեն։
- **Թիրախային լսարանը (Target audience)**: Ով է այցելում ձեր կայքը և ինչ կարիք ունեն։
- **Բրենդի ինքնությունը (Brand identity)**: Ձեր գույները, տոնը և տեսողական ոճը։
- **Բիզնեսի նպատակները (Business goals)**: Ի՞նչ է հաջողություն ձեր կայքի համար։
- **Բովանդակության կառուցվածքը (Content structure)**: Ինչպես է կազմակերպված ձեր կայքը։

Այս սահմանումը դառնում է ձեր **site_brief**-ը՝ մշտական հիշողություն, որը օգտագործում են ասոնտները ձեր կայքի համատեքստը հասկանալու համար։

## Ինչո՞ւ օգտագործել Site Specification-ը {#why-use-site-specification}

### Համապատասխանություն նիստերի ընթացքում {#consistency-across-sessions}

site_brief-ի առանց, ձեզ անհրաժեշտ կլիներ ձեր կայքի նպատակը բացատրել ամեն անգամ՝ յուրաքանչյուր նոր նիստ սկսելիս։ Այնով, ասոնտները անմիջապես հասկանում են.

- Ձեր կայքի նպատակները և լսարանը
- Ձեր բրենդի գույները և տոնը
- Ձեր բովանդակության կառուցվածքը
- Ձեր բիզնեսի նպատակները

### Ավելի լավ առաջարկներ {#better-recommendations}

Ասոնտները օգտագործում են ձեր site_brief-ը՝

- Առաջարկելով առանձնահատկություններ, որոնք համապատասխանում են ձեր կայքի նպատակին
- Խորհուրդ տալով բովանդակության կառուցվածքներ, որոնք համապատասխանում են ձեր նպատակներին
- Առաջարկելով դիզայններ, որոնք համապատասխանում են ձեր բրենդին
- Խուսափելով անհամապատասխան հնարավորությունների առաջարկումից

### Արագ ներառում (Faster Onboarding) {#faster-onboarding}

Նոր ասոնտները (կամ նոր նիստերի ասոնտները) կարող են արագ սովորել՝ ձեր site_brief-ը կարդալով, այլ ոչ թե հստակեցնող հարցեր տալով։

## Site Specification-ի Սկզբնարկում {#initiating-site-specification}

### Theme Builder-ի Ներառման Ընդունման Գործընթացում (During Theme Builder Onboarding) {#during-theme-builder-onboarding}

Site Specification հմտությունը ավտոմատ կերպով սկսվում է **Theme Builder onboarding flow**-ի ընթացքում։ Setup Assistant ասոնտը հարցեր է տալիս և կառուցում ձեր site_brief-ը։

### Մանրամասն Սկզբնարկում (Manual Initiation) {#manual-initiation}

Դուք կարող եք ցանկացած պահի սկսել Կայքի բնութագրումը (Site Specification):

```
"Եկեք սահմանեն моյս կայքի բնութագրումը"
```

կամ

```
"Օգնո՛ւմին կայքի հակնարկում գեներացնել"
```

## Կայքի բնութագրման գործընթաց (The Site Specification Process) {#the-site-specification-process}

### Քայլ 1. Կայքի նպատակը (Site Purpose) {#step-1-site-purpose}

Աгентը կհարցնի՝

```
Ի՞նչ է քո կայքի հիմնական նպատակը։
- Էլեկտրոնային առևտրի խանութ (E-commerce store)
- Բլոգ կամ բովանդակության կայք (Blog or content site)
- Պորտֆոլիո կամ ցուցադրություն (Portfolio or showcase)
- SaaS հավելված (SaaS application)
- Համայնք կամ ֆորում (Community or forum)
- Այլ. [აღწერիր]
```

Դու կարող ես ընտրել մի կատեգორიա կամ նկարագրել սեփական նպատակը։

### Քայլ 2. Թիրախային լսարան (Target Audience) {#step-2-target-audience}

```
Ո՞վ է քո հիմնական լսարանը։
- Կonsumerներ / ընդհանուր հանրություն (Consumers / general public)
- Բիզնես մասնագետներ (Business professionals)
- Ծրագրավորողներ / տեխնիկական օգտվողներ (Developers / technical users)
- Անկերտներ / կրթողներ (Students / educators)
- Այլ. [აღწერիր]

Ի՞նչ են նրանց հիմնական կարիքները։
```

### Քայլ 3. Բրենդային ինքնություն (Brand Identity) {#step-3-brand-identity}

```
Ինչպե՞ս են քո բրենդի գույները։
- Հիմնական գույն: [գույնի ընտրիչ կամ հեքսսադեկալ կոդ]
- Երկրորդ գույն: [գույնի ընտրիչ կամ հեքսսադեկալ կոդ]
- Ընդգծող գույն (Accent color): [ընտրովի]

Ինչպե՞ս կուտակես նկարագրել քո բրենդի տոնը։
- Պրոֆեսիոնալ / կորպորատիվ (Professional / corporate)
- Ստեղծագործական / արվեստական (Creative / artistic)
- Խաղային / կասկածելի (Playful / casual)
- Մինիմալ / ժամանակակից (Minimal / modern)
- Հրավառ / ընկերական (Warm / friendly)
```

### Քայլ 4. Բիզնեսի նպատակներ (Business Goals) {#step-4-business-goals}

```
Ի՞նչ է հաջողությունը քո կայքի համար։
- Հետևորդների գեներացում (Generate leads)
- Ապրանքների վաճառք (Sell products)
- Համայնքի կառուցում (Build community)
- Գիտելիքների կիսում (Share knowledge)
- Ավտորիություն հաստատել (Establish authority)
- Այլ. [აღწერիր]

Ի՞նչ է քո հիմնական չափանիշը (metric):
- Շահույթ (Revenue)
- Օգտատերերի ներգրավվածություն (User engagement)
- Բովանդակության հասանելիություն (Content reach)
- Համապատասխանեցումներ (Conversions)
- Այլ
```

### Քայլ 5. Բովանդակության կառուցվածք (Content Structure) {#step-5-content-structure}

```
Ինչպե՞ս է կազմակերպվում քո բովանդակությունը։
- Плоский (Flat) (բոլոր բովանդակությունը նույն մակարդակի վրա)
- Հիերարխիկ (Hierarchical) (կատեգորիաներ և ենթակատեգորիաներ)
- Քրոնոլոգիական (Chronological) (բլոգային ոճով)
- Ապրանքի վրա հիմնված (Product-based) (կատալոգ)
- Այլ. [აღწერիր

## Ձեր site_brief-ի դիտում և թարմացում

### Ձեր site_brief-ը դիտել

Հարցրեք օգտակարին.

```
"Show me my site brief"
```

կամ

```
"What do you know about my site?"
```

Օգտակարը կցուցադրի ձեր պահված նշումները։

### Ձեր site_brief-ը թարմացնել

Եթե ձեր կայքը զարգանում է, կարող եք այն թարմացնել.

```
"Update my site brief: we're now targeting B2B customers"
```

կամ

```
"Refresh my site specification"
```

Սա Site Specification հմտությունը կկատարի նոր անգամ՝ օգտագործելով ձեր առկա տեղեկությունները որպես սկզբնական կետ։

## Ինչպես են օգտակարները օգտագործում site_brief-ը

### Դիզայնի առաջարկներ

Երբ դուք խնդրում եք դիզայնի փոփոխություններ, օգտակարները հղվում են ձեր site_brief-ին.

```
Դուք: "Redesign my homepage"
Օգտակարը: "Ձեր site brief-ի համաձայն, դուք թիրախավորում եք 25-45 տարեկան կանայք՝ տաք և արտեզանական տոնով։ Ես ստեղծեմ դիզայն, որը շեշտադրում է ձեր ձեռագործ զարդերը և կայուն արժեքները։"
```

### Ֆիքնուրների առաջարկներ

Օգտակարները առաջարկում են ֆիքնուրներ, որոնք համապատասխանում են ձեր նպատակներին.

```
Դուք: "What features should I add?"
Օգտակարը: "Հաճախորդների հետ կապված էլ-կոմերսի խանութի համար, ես առաջարկում եմ՝ հավատարիմության ծրագիր, հաճախորդների կարծիքներ, էլ-փոստի նորություններ և արտադրանքի առաջարկներ։"
```

### Բովանդակության կառուցվածք

Օգտակարները առաջարկում են բովանդակության կազմակերպում ձեր կառուցվածքի հիման վրա.

```
Դու. «Ինչպե՞ս պետք է կազմակերպեմ իմ բլոգը»
Աгент. «Քանի որ դու օգտագործում եք արտադրանքի կատալոգ՝ բլոգի հետ, ես առաջարկում եմ հրապարակումները կազմակերպել արտադրանքի կատեգորիաներով և ստեղծել «ստիլինգ ուղեցույթներ» (styling guides), որոնք միաժամանակ ցուցադրում են բազմաթիվ արտադրանքներ։»
```

## Ամենապարզ լավ պրակտիկաները (Best Practices)

### Մի 되՛ր կոնկրետ

«Ընդհանուր լսարանի» փոխարեն նկարագրիր իրական լսարանը.

- ✓ «25-45 տարեկան կանայք, հետաքրքրված կայուն էֆֆորտով հագներով»
- ✗ «Բոլորը»

### Թարմացրու պարբերաբար

Քանի որ ձեր կայքը զարգանում է, թարմացրու `site_brief`-ը.

- Երբ փոխում եք նոր լսարան
- Երբ ավելացնում եք նոր արտադրանքի տեսակներ
- Երբ ձեր բրենդի ինքնությունն է փոխվում
- Երբ ձեր բիզնեսի նպատակները փոխվում են

### Օգտագործիր համապատասխան տերմինաբանություն

Օգտագործիր նույն հասկացություններով բոլոր քայլերում.

- ✓ Միշտ ասա «կայուն զարդեր» (ոչ թե «էկո-բարձրության զարդեր» և «սպիտակ արտադրանքներ»)
- ✓ Համապատասխանաբար հղում կատարիր ձեր լսարանին

### Ավելացրու համատեքստ

Տրամադրիր հետևող տեղեկություն, որը օգնում է ասացչելիությանը մյուսների համար.

- «Մենք թիրախավորում են մասնագետներ, ովքեր գնահատում են որակը՝ գնից ավելի»
- «Մեր լսարանը տեխնոլոգիապես պատրաստ է և սպասում է ժամանակակից դիզայնի»
- «Մենք սկսած փոքր բյուջեով ենք աշխատող ստարտափ, ուստի մեզ անհրաժեշտ են արդյունավետ լուծումներ»

## Հարաբերություն Theme Builder-ի ներառման հետ

Site Specification հմտությունը ինտեգրված է **Theme Builder-ի ներառման ընթացքում**։ Երբ ավարտում եք ներառումը, ձեր `site_brief`-ը ավտոմատ կերպով ստեղծվում է այն տեղեկություններով, որոնք դու մ આપել ես։

Դու կարող ես անկախ գործարկել Site Specification-ը, եթե ցանկանում ես.

- Ձեր նշումները կատարելուց հետո սահմանափակել (Refine)
- Թարմացնել ձեր site brief-ը՝ կայքի զարգացման համաձայն
- Մանրամասն սահմանում ստեղծել Theme Builder-ին սկսելուց առաջ

## Խնդիրների լուծում (Troubleshooting)

**Իմ site_brief-ը չի օգտագործվում**
- Հաստատիր, որ ասենքուն մուտք ունի հիշողության (memory) վրա
- Հարցրու ասենքուն՝ «վերանչել իմ site brief-ը» (recall my site brief)
- Ստուգիր, թե արդյոք հիշողությունը 켜ված է քո կարգավորումներում

**Ես ուզում եմ սկսել նոր site_brief-ով**
- Հարցրեք օգտատիրոջին.՝ "Հեռացրու իմ site brief-ը և սկսիր նորից"
- Այնուհետև կատարեք Site Specification-ը նորից

**Օգտատերը առաջարկում է բաներ, որոնք չեն համապատասխանում իմ site_brief-ին**
- Հարցրեք օգտատիրոջին.՝ "Ստուգիր իմ site brief-ը"
- Թարմացրեք ձեր site_brief-ը, եթե այն հնացած է
- Ավելի տվյալներ տրամադրեք ձեր խնդրանքներում

## Հաջորդ քայլերը

Ձեր site specification-ը սահմանելուց հետո.

1. **Օգտագործեք Theme Builder**: Ստեղծեք կոնկրետ թեմա՝ հիմնվելով ձեր site_brief-ի վրա
2. **Կարգավորեք դիզայնը**: Օգտագործեք Design System Aesthetics հմտությունը մանրամասն դիզայնի համար
3. **Պլանավորեք բովանդակությունը**: Հարցրեք օգտատերին՝ բովանդակության կառուցվածքի առաջարկների համար
4. **Կառուցեք հնարավորությունները (Features)**: Խնդրեք հնարավորություններ, որոնք համապատասխանում են ձեր բիզնեսի նպատակներին
