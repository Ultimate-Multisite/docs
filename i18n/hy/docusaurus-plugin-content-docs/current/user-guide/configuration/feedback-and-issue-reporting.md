# Հաճախորդի հետ կապ և խնդիրների հաղորդում

Gratis AI Agent v1.5.0-ը ներկայացնում է ներկառուցված արձագանքների վերաբերյալ հետադարձ կապի և խնդիրների հաղորդման համակարգ, որը թույլ է ավարտվող օգտատերերին ուղղակի չօգնուող պատասխանները նշելու և խնդիրներ հաղորդելու համար՝ քեյթի մոդալից։ Այս համակարգը ներառում է համաձայնության կառավարում, ավտոմատ հաղորդման հրաման և բэքենդում AI-ի օգնությամբ տրիaje (triage) գործընթաց։

## Ձախ ճակատի կոճակը (Thumbs-Down Button) {#thumbs-down-button}

AI օգնականից ուղարկվող յուրաքանչյուր հաղորդագրության վրա երևում է **ձախ ճակատի** (👎) կոճակ։ Երբ օգտատերը դրան սեղմում է, նա կարող է պատասխանը որպես անօգնական կամ սխալ նշել։

- Կոճակը երևում է մյուս հաղորդագրության կողքին՝ մկանոն շարժման ժամանակ (hover)։
- Այն սեղմելիս բացվում է **Հետադարձ կապի համաձայնության մոդալը** (Feedback Consent Modal)։
- Հետադարձ կապը կապված է զրույցի թրադով և տվյալ հաղորդագրությամբ։

## Հետադարձ կապի համաձայնության մոդալ {#feedback-consent-modal}

Երբ օգտատերը սեղմում է ձախ ճակատի կոճակը, որևէ տվյալներ չուղարկելուց առաջ երևում է համաձայնության մոդալ։ Մոդալը.

- Բացատրում է, թե ինչ տեղեկություն են հավաքվում (զրույցի հատված, բրաուզերի համատեքստ)։
- Հարցնում է օգտվողին հաստատել, որ համաձայնում է այս տվյալները կիսելու համար։
- Տրամադրում է ազատ տեքստային դաշտ՝ օգտատերին նկարագրելու համար, թե ինչ է սխալ։
- Առաջարկում է **Չեղարկել** (Cancel) տարբերակ՝ առանց ներկայացնելու փոխելու համար։

Այնուամենայնիվ հետադարձ կապ չի գրանցվում, մինչև օգտատերը հստակ չհաստատի։

## Ավտո-հրահանման հետադարձ կապի բැනեր (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Զրույցի ավարտին օգնականը կարող է ցուցադրել **ավտո-հրահանման հետադարձ կապի բැනեր**՝ անտեսելի հաղորդագրություն, որը հարցնում է, արդյոք նիստը օգտակար էր։

Այս բැනերը ավտոմատ կերպով երևում են զրույցի երկարության և արդյունքների հիմնավորված մեթոդներով (heuristics)։ Այն կապվում է ձախ ճակատի կոճակի հետ նույն հետադարձ կապի հոսքի հետ։ Օգտատերը կարող է բැනերը փակել՝ առանց հետադարձ կապի տրամադրելու։

## /report-issue Սլաշ Command {#report-issue-slash-command}

Օգտատերը կարող է ուղղակիորեն հաղորդել հետադարձ կապի հոսքը՝ քեյթի մուտքագրման մեջ `/report-issue` տպելով։ Այս հրամանը.

Feedback Consent Modal-ը անմիջապես Բացում է։
Նկարագրության դաշտը ներկայիս զրույցի համատեքստով տեղեկություններով նախապատրաստում է։
Սա թույլ է տալիս օգտվողներին հանձնելու առաջ ուղարկելուց առաջ ավելի մանրամասն տեղեկություն ավելացնել։

`/report-issue` հրամանը հասանելի է բոլոր չաթիչների ռեժիմներում (inline, floating widget, full-screen)։

## AI-ով Տրիaje (Triage) {#ai-assisted-triage}

Առաջարկված հետադարձ կապը ուղղվում է AI տրիaje շերտին, որը.
- Կատեգորիզացնում է հաղորդագրությունը (փաստական սխալ, օգուտ չհասկանալու պատասխան, անհատական բովանդակություն և այլն)։
- Հավելում է համապատասխան զրույցի մետատվյալները։
- Տրիaje ամփոփումը փոխանցում է կոնֆիգուրացված **Feedback Endpoint**-ին (տես [Settings > Advanced](#settings-advanced)՝)։

Սա ձեռքով վերանայելու ժամանակը նվազեցնում է, առաջին հերթին ամենակրիտիկական խնդիրները բացահայտելով։

## Settings > Advanced {#settings-advanced}

Հետադարձ կապի ֆոնդը աշխատեցնելու համար կարգավորեք հետևյալ դաշտերը **Gratis AI Agent → Settings → Advanced**-ում.

| Դաշտ | Նկարագրություն |
|---|---|
| **Feedback Endpoint URL** | URL-ը, որը ստանում է POST հարցումներ՝ հետադարձ կապի բեռերի (JSON) հետ։ |
| **Feedback API Key** | Bearer token, որը ուղարկվում է `Authorization` header-ի մեջ հետադարձ կապի ներկայացման համար։ |

Եթե երկու դաշտերը թողնեք դատարկ՝ հետադարձ կապի հավաքագրումը անջատելու համար։ Անհատական հետադարձ տոճքերը մնում են օգտվողների տեսանելի, բայց ուղարկումները չեն փոխանցվի։

:::tip
Հետադարձ կապի ինդեպոինտը պետք է ընդունի JSON մարմին՝ առնվազն `message_id`, `conversation_id`, `feedback_text` և `triage_category` դաշտերով։ Սպասելի սխեման համար տես ձեր ինդեպոինտի մատակարարի փաստաթղթերը։
:::
