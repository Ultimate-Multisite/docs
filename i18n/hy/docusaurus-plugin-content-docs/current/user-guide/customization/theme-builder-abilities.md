# Theme Builder ունակություններ. Block Themes ստեղծում և ակտիվացում

Superdav AI Agent v1.12.0-ը ներկայացնում է երկու հզոր ունակություն, որոնք թույլ են տալիս ձեզ ուղղակի չատի միջոցով ստեղծել և տեղադրել սահմանված բլոկային թեմաներ։

## Ընդհանուր տեսքը

**scaffold-block-theme** և **activate-theme** ունակությունները թույլ են տալիս ասենք Agent-ներին.
- Ձեր նկարագրությունների հիման վրա ամբողջական, արտադրության պատրաստ բլոկային թեման ստեղծել։
- Անհատական միջամտության առանցքով ձեր կայքում թեման ավտոմատ ակտիվացնել։
- Հաղորդագրված դիզայնի որոշումների միջոցով համահունչ տեսողական անհատություն ստեղծել։

## Block Theme-ի ստեղծում (Scaffold)

**scaffold-block-theme** ունակությունը ստեղծում է WordPress բլոկային թեմա՝ ամբողջական թեման կառուցվածքով, ներառյալ.

- `theme.json` կոնֆիգուրացիան դիզայնի տոկեններով (design tokens)
- ընդհանուր դասավորությունների համար բլոկ ծրագրերի տեխնիկական գ檔案ներ (template files)
- հատուկ բլոկային ոճեր և տարբերակներ (variations)
- թեմայի 메տատվալներ (metadata) և աջակցության հայտարարություններ

### Ինչպես կանչել (Invoke)

Ձեր չատի մեջ Superdav AI Agent-ի հետ կարող եք խնդրել թեման ստեղծել.

```
"Ստեղծիր 'Modern Agency' անվան բլոկային թեմա՝ կապույտ և සුදු գունային սխեման,
sans-serif տիպոգրաֆիայով և պրոֆեսիոնալ դասավորությամբ"
```

Agent-ը կկատարի հետևյալ քայլերը.
1. Հաղորդագրության միջոցով ձեր դիզայնի նախընտրությունները հավաքել։
2. Ամբողջական թեմայի կառուցվածքը ստեղծել։
3. Բոլոր անհրաժեշտ թեմայի ֆայլերը պատրաստել։
4. Թեման ակտիվացման համար պատրաստել։

### Սպասվող արդյունքը (Expected Output)

Երբ ունակությունը հաջողությամբ կատարվի, դուք կտեսնեք.

- Թեման ստեղծված լինելու հաստատում։
- Թեմայի անունը և դիրքը (location)։
- Կիրառված դիզայնի տոկենների ամփոփումը (գույներ, տիպոգրաֆիա, հարևանության տարածություն)։
- Պատրաստ լինելու կարգավիճակ (Ready-to-activate status)։

Օրինակի արդյունք.
```
✓ Թեմա "Modern Agency" հաջողությամբ ստեղծված է
  Դիրք: /wp-content/themes/modern-agency/
  Գույներ: Հիմնական #0066CC, Երկրորդական #FFFFFF
  Տիպոգրաֆիա: Inter (sans-serif)
  Վիճակ: Պատրաստ է ակտիվացման համար
```

## Թեմայի ակտիվացում (Activate Theme)

**activate-theme** հնարավորությունը թույլ է տալիս ձեր կայքը փոխել նոր սկզբից կառուցված կամ արդեն գոյություն ունեցող բլոկային թեմայի։

### Ինչպես օգտագործել

Թե՞ որևէ թեմա սկզբում կառուցելուց հետո կարող եք այն անմիջապես ակտիվացնել.

```
"Activate the Modern Agency theme"
```

Կամ ցանկացած գոյություն ունեցող թեմա ակտիվացնելու համար.

```
"Switch to the Twentytwentyfour theme"
```

### Սպասվող արդյունքը

Երբ ակտիվացումը հաջող է կատարվում.

- Ակտիվ թեմայի հաստատում
- Նախորդ թեմայի անունը (հղման համար)
- Կայքի URL-ը, որտեղ թեման այժմ գործում է
- Թեմային կոնկրետ կարգավորումների նշումներ

Օրինակ՝ արդյունք.
```
✓ Theme activated successfully
  Active theme: Modern Agency
  Previous theme: Twentytwentyfour
  Live at: https://yoursite.com
  Note: Check your homepage to verify the layout
```

## Գործընթաց՝ Սկզբնական կառուցում և Ակտիվացում (Workflow: Scaffold and Activate)

Սովորական գործընթացը համատեղում է երկու հնարավորությունը.

1. **Թեմայի ստեղծման խնդրանք**: "Create a block theme for my SaaS landing page"
2. **Agent-ը թեման կառուցում է (scaffolds)**: Ստեղծում է ֆայլեր և դիզայնի տոկեններ (design tokens)
3. **Գնահատում և մշակում**: Եթե անհրաժեշտ է, քննարկել դիզայնի փոփոխությունները
4. **Ակտիվացում**: "Activate the theme now"
5. **Ստուգում**: Ձեր կայքը այցելեք՝ նոր դիզայնը գործող է արդյոք հաստատելու համար

## Դիզայնի տոկեններ և Կանոնավորում (Design Tokens and Customization)

Սկզբնական կառուցված թեմաները օգտագործում են WordPress-ի դիզայնի տոկենները (`theme.json` միջոցով).

- **Գույներ**: Հիմնական, երկրորդական, շեշտադրող, նեյտրալ паլիտրա (palette)
- **Տիպոգրաֆիկա**: Ֆոնտային ընտ족ры (font families), չափեր, քաշեր (weights), տողերի բարձրություն
- **Հեռավորություններ**: Padding, margin, gap scales
- **Կիթեզներ (Borders)**: Радиус և լայնության տոկեններ
- **Նուշումներ (Shadows)**: Բարձրացման մակարդակներ

Այս տոկենները կենտրոնացված են `theme.json`-ում, ինչը հեշտացնում է ձեր ամբողջ դիզայնի համակարգը կարգավորել մեկ ֆայլից։

## Սահմանափակումներ և Նշումներ (Limitations and Notes)

- Themes are scaffolded in `/wp-content/themes/` և պետք է հետևեն WordPress-ի անվանման կանոններին
- Գործարկելու համար անհրաժեշտ է համապատասխան թույլտվություններ ձեր WordPress сайтի վրա
- Թեմաներում սահմանափակ է հատուկ PHP կոդը. բարդ ֆունկցիոնալությունների համար օգտագործեք plugin-ներ
- Блоկային թեմաները լավագույնս աշխատում են WordPress 5.9 և ավելի տարբերակներով

## Խնդիրների լուծում (Troubleshooting)

**Թեման չի հայտնվում սահմանելուց հետո**
- Ստուգեք, թե արդյոք թեման դիրեկտորիան գոյություն ունի և ճիշտ թույլտվություններ ունի
- Ստուգեք, որ `theme.json`-ը վավեր JSON է
- Համոզվեք, որ թեմայի անունը չի կոնֆլիկտ առաջացնում գոյություն ունեցող թեմաների հետ

**Գործարկումը ձախողվում է**
- Հաստատեք, որ ունեք ադմինիստրատորական (administrator) թույլտվություններ
- Ստուգեք, որ WordPress-ը կարող է կարդալ թեմաների դիրեկտորիան
- Մանրամասների համար ստուգեք WordPress-ի սխալների ֆայլերը (error logs)

**Design tokens-ները չեն կիրառվում**
- Հաստատեք, որ `theme.json`-ի սինտաքսը ճիշտ է
- Մաքրեք ցանկացած 캐싱 plugin
- Ստուգեք, թե արդյոք ձեր WordPress-ի տարբերակը աջակցում է օգտագործվող tokens-ներին

## Հաջորդ քայլերը (Next Steps)

Ձեր թեման գործարկելուց հետո կարող եք.
- Օգտագործել **Design System Aesthetics** հմտությունը՝ տիպոգրաֆիայի, գույների և տարածության վերանայման համար
- Մասնակի բլոկային ոճերը կարգավորել WordPress block editor-ի միջոցով
- Ավելացնել սահմանված CSS թեմաների `style.css` ֆայլում
- Ստեղծել հատուկ բլոկային տիպեր (templates)՝ կոնկրետ էջերի տեսակների համար
