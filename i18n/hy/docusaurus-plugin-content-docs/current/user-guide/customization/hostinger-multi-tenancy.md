# Hostinger Multi-Tenancy

Ultimate Multisite: Multi-Tenancy 1.2.0 Hostinger multi-tenancy (բազմաօգտատերության) հնարավորություն է ավելացրել, որպեսզի հնարավոր լինի հյուրընկալված ադմիններով տիրույթները մասնակցեն սեվրեն (sovereign) տիրույթների ստեղծման գործընթացին՝ համատեղելով արդեն առկա Hostinger-ի դոմենային քարտեզագրման (domain-mapping) ինտեգրացիայի հետ։

Օգտագործեք այս հնարավորությունը, երբ տիրույթների ենթադոմենները և առանձնացված տիրույթի ենթակառուցվածքը կառավարվում են Hostinger hPanel-ով։

## Կài գործարկման նշումներ (Setup notes)

1. Կարգավորեք հիմնական Hostinger ինտեգրացիան **Ultimate Multisite > Settings > Host Integrations**-ում։
2. Հաստատեք, որ Hostinger API token-ը կարող է կառավարել թիրախային դոմենը կամ ենթադոմենը։
3. Թողարկել Multi-Tenancy addon-ը (հնարավորություն)։
4. Կարգավորեք տիրույթի առանձնացման ռազմավարությունը՝ թողարկելուց առաջ։
5. Կատարեք մIGRATION-ի հաստատման գործընթացը, որպեսզի արտադրական տրաֆիկը չուղարկվի տիրույթին։

Hostinger-ի հնարավորությունը օգտագործում է ընդհանուր Hostinger կապը հոստի-կողմ գործողությունների համար։ DNS-ը դեռ պետք է ուղղված լինի ճիշտ Hostinger հաշվին, և hPanel-ի հաշվի սահմանափակումները դեռևս կիրառելի են։

## Հատուկ փոփոխություններով հնարավորությունների մասին (Capability-specific changes)

- Սեվրեն տիրույթներ կարող են ստեղծվել հոստի-հասկանալով դոմենային գործողություններով։
- Միևնույն մեքենայի բազայի հոստի տողերը նորմալացվում են մինչև վավերականության ստուգումը (grant verification)։
- Hostinger-ի կողմից կառավարվող տիրույթները պետք է օգտագործեն hPanel-ում ցուցադրված բազայի հոստի արժեքը, եթե WordPress runtime-ը դա չի պահանջում տեղական գերագրում (local override)։
- SSO այցելությունները կախված են տիրույթի դոմենի Hostinger-ով հյուրընկալվող տիրույթին վերածվելուց։

## Hostinger տիրույթների խնդիրների լուծում (Troubleshooting Hostinger tenants)

- Եթե տիրույթի ինստալացիան ձախողվում է, ստուգեք, թե արդյոք դոմենը արդեն կապված է Hostinger հաշվի հետ։
- Եթե բազայի վավերականությունը ձախողվում է, համեմատեք տիրույթի DB ինստիտուտի (username), բազայի անունը և հոստի կապը hPanel-ի հետ։
- Եթե **Visit (SSO)** ձախողվում է, հաստատեք, որ DNS-ը և SSL-ը ակտիվ են տիրույթի դոմենի համար։
- Եթե teardown-ը թողնում է հոստիային ավելորդ ռեսուրսներ, ջնջեք hPanel-ից մնացած բոլոր բազաները, օգտատերները կամ թղթապանակները՝ արխիվների հաստատումից հետո։
