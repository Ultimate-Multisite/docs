# Արտահանել և ներմուծել

Ultimate Multisite 2.9.0-ը **Tools > Export & Import** (Գործիքներ > Արտահանել և ներմուծել) բաժնի տակ մեկ կայքի համար նախատեսված միայն մեկ կայքի **Export & Import** գործիք է ավելացրել։ Այն օգտագործեք, երբ ձեզ անհրաժեշտ է մեկ WordPress կայքը փաթုပ်ել ZIP ֆայլի տեսքով, այդ ZIP-ը վերականգնել կամ կայքը փոխել համատեղելի Ultimate Multisite և մեկ կայքի WordPress տեղադրումների միջև։

## Անհրաժեշտ թույլտվություններ

Դուք պետք է մուտք գործեք որպես ադմինիստրատոր, ով կարող է հասանելիություն ունենալ կայքում արտահանվող կամ ներմուծվող WordPress **Tools** (Գործիքներ) մենյուին։ Մուլտիսայթ ցանցում, եթե դուք օգտագործում եք ցանցային Ultimate Multisite գործիքների միջոցով ենթակայքերի արտահանում կամ ներմուծումներ, օգտագործեք ցանցային ադմինիստրատորի հաշիվը։

Export ZIP ներբեռնումները մատուցվում են ինտերֆեյսով հաստատված ներբեռնման կետի միջոց, ուստի պահպանեք ադմինիստրատորի սեփականությունը (session) մինչև ներբեռնումը ավարտվի և չբացառեք գեներացված ներբեռնման URL-ները հանրային տեղերում։

## Կայքի արտահանում ZIP-ի մեջ

1. Ձեր պատճենելու կայքում WordPress ադմինիստրատորում գնացեք **Tools > Export & Import** (Գործիքներ > Արտահանել և ներմուծել)։
2. Բացեք արտահանման տարածքը և ընտրեք այն կայքը, որը ցանկանում եք փաթုပ်ել։
3. Երբ այդ տարբերակները հասանելի են, ընտրեք ներառվող լրիվ բովանդակությունը, ինչպիսին կարող է լինել uploads (ներբեռնումներ), plugins (ปลั๊กիններ) և themes (թեմաներ)։
4. Սկսեք արտահանման գործընթացը և սպասեք, մինչև այն ավարտվի։ Մեծ կայքերը կարող են պետք է ավարտվենपृष्ठադրումից առաջ՝ հիմնական գործընթացը պատրաստ լինելու համար։
5. Ներբեռնեք ավարտված ZIP-ը արտահանման ցուցակից։

Պահպանեք ZIP-ը անվտանգ վայրում։ Այն կարող է պարունակել կայքի բովանդակություն, կարգավորումներ, մեդիայի ֆայլեր և ընտրված կոդային ռեսուրսներ։

## Արտահանումը ներառում է ինչը

Արտահանման ZIP-ը կարող է պարունակել.

- Ընտրված կայքի բազայի բովանդակությունը և կոնֆիգուրացիան (configuration)։
- Եթե ներառվում են uploads-ները, ապա նույնպես ներբեռնված մեդիայի ֆայլերը։
- Եթե ընտրված են այդ տարբերակները, ապա plugins և themes-ը։
- Import metadata (ներմուծման մետատվյալներ), որոնք օգտագործվում են Export & Import գործիքը թիրախային տեղադրումը կառուցելու համար։

Ճիշտ ZIP չափը կախված է մեդիայի քանակից, ընտրված plugin-ներից և թեմաներից, ինչպես նաև սайտի ինդեքսների բազայի (database tables) չափից։

## Սайտի ներմուծում ZIP-ից

1. Գնացեք հայտարարվող WordPress վերջնական սайտի **Tools > Export & Import**-ի դեպի։
2. Բացեք ներմուծման տարածքը և վերբեռնեք այն ZIP-ֆայլը, որը ստեղծվել է Export & Import գործիքով։
3. Մուտքագրեք փոխարինող URL (replacement URL), եթե սайտը պետք է օգտագործի նոր հասցե, կամ թողեք դաշտը դատարկ՝ սկզբնական URL-ը պահելու համար։
4. Ընտրեք, արդյոք բեռնված ZIP-ը պետք է ջնջվի ներմուծումը ավարտվելուց հետո։
5. Սեղմեք **Begin Import** (Սկսել ներմուծումը)։
6. Հետևեք սպասվող ներմուծմանը մինչև այն ավարվի։ **Cancel Import**-ը (Չեղարկել ներմուծումը) օգտագործեք միայն այն դեպքում, եթե անհրաժեշտ է գործընթացը կանգնեցնել ամբողջական ավարտից առաջ։
7. Նախքան թույլ տալ սովորական տրաֆիկին կամ հաճախորդներին մուտք, վերանայեք ներմուծված սайտը։

Մեկ-սайտ WordPress ինստալացիայի դեպքում ZIP-ի ներմուծումը ներկայիս սайտը փոխարինում է ներմուծված սайտով։ Նախքան սկսելը, ստեղծեք թիրախային սайտի ամբողջական անվտանգության պատճենը (backup) և խուսափեք նույն սайտի համար միաժամանակ բազմակի ներմուծումներ սկսելուց։

## Սահմանափակումներ և համատեղելիության նշումներ

* Շատ մեծ վերբեռնման թղթապանակներ կամ մեդի գրադարանները կարող են մեծ ZIP ֆայլեր ստեղծել։ Նախքան մեծ կայքերը արտահանելը կամ ներմուծելը, համոզվեք PHP-ի վերբեռնման սահմանափակումների, կատարման սահմանափակումների, սկավառակի տարածության, հիշողության և սերվերի ժամանակային սահմանափակումների կարգավորումների մասին։
* Շատ մեծ մեդի գրադարանները կարող են անհրաժեշտ լինել տեղաշարժվել միջին տրաֆիկի (low-traffic) պահպանման ժամանակ։
* Թիրախային WordPress-ը պետք է աշխատի համատեղելի WordPress, PHP, Ultimate Multisite, plugin և theme տարբերակներով։
* Մեկ կայքից ներմուծումը փոխարինում է թիրախային կայքը։ Այն միացման (merge) գործիք չէ։
* Multisite-to-single-site և single-site-to-multisite տեղաշարժերը աջակցվում են միայն այն դեպքում, երբ թիրախային միջավայրը կարող է աշխատել արտահանված կայքի plugin-ները, theme-երը, URL-ները և Ultimate Multisite-ի անհրաժեշտ բաղադրիչները։
* ZIP-ը պահպանեք մասնավոր։ Այն կարող է պարունակել տվյալների բազայի (database) բովանդակություն, վերբեռնված ֆայլեր և արտահանված կայքի կոնֆիգուրացիոն մանրամասներ։

Հին ցանցային մակարդակի արտահանման գործընթացների համար տես [Site Export](/user-guide/administration/site-export)։
