# Gratis AI Agent Կարգավորումներ

**Settings → Advanced** էջը Gratis AI Agent-ի մեջ թույլ է տալիս վերահսկողի մակարդակի կարգավորումներ կատարել այն բացառիկ համակարգերի համար, որոնք ներդրված են v1.5.0-ով։ Այս էջը փաստագրում է **Feedback Endpoint** դաշտերը և դրանց ակնկալվող ձևաչափը։

## Կարգավորման մուտք

1. WordPress admin-ում գնացեք **Gratis AI Agent → Settings**-ի։
2. Սեղմեք **Advanced** տաբբերին։

## Feedback Endpoint-ի Կազմաձևում

Feedback endpoint-ը ստանում է POST հարցումներ՝ AI agent-ից, երբ օգտվողը հետադարձ կապ (thumbs-down) կոճակով, ավտո-հայտարարության բանալիով կամ `/report-issue` հրամանով։

| Դաշտ | Նկարագրություն |
|---|---|
| **Feedback Endpoint URL** | URL-ը, որը ստանում է հետադարձ կապի ներկայացումները HTTP POST հարցումների տեսքով JSON մարմնով։ |
| **Feedback API Key** | Bearer token, որը ուղարկվում է յուրաքանչյուր feedback հարցման `Authorization` header-ում։ Եթե ձեր endpoint-ին ինտերֆորմացիա չի պահանջում, թողեք այն դատարկ։ |

### Ակնկալվող JSON Payload

Ձեր feedback endpoint-ը պետք է ընդունի JSON մարմին՝ առնվազն հետևյալ դաշտերը ներառելով.

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Payload-ի մեջ կարող են ներառվել լրացուցիչ դաշտեր՝ կախված զրույցի համատեքստից։

### `triage_category` Արժեքները

AI տրիաժի ស្រելքը (triage layer) առաջ ուղարկելուց առաջ `triage_category`-ին վերագրում է հետևյալ արժեքներից մեկը.

| Արժեք | Իշխանություն |
|---|---|
| `factual_error` | Անհատակիցը տրամադրել է սխալ փաստացի տեղեկություն։ |
| `unhelpful_answer` | Պատասխանը տեխնիկապես ճիշտ էր, բայց օգուտ չէր։ |
| `inappropriate_content` | Պատասխանը պարունակում էր այն բովանդակություն, որը չպետք է ցուցադրվի օգտվողներին։ |
| `other` | Հետադարձ կապը չի համապատասխանում հայտնի կատեգորիային։ |

### Իմենտիֆիկացիա (Authentication)

Եթե ձեր endpoint-ը պահանջում է ինտենտիֆիկացիա (authentication), սահմանեք **Feedback API Key** դաշտը ձեր bearer token-ով։ Ագենտը կուղարկի հետևյալը.

```
Authorization: Bearer <your-api-key>
```

Եթե **Feedback API Key** դաշտը դատարկ է, `Authorization` header չի ուղարկվի։

### Feedback Կուլեկցիայի անջատում

Թողեք ինչպես **Feedback Endpoint URL**, այնպես էլ **Feedback API Key** դաշտերը դատարկ։ Հատուկ մանրամասներով (thumbs-down) կոճակը և feedback UI-ն մնում են օգտվողների տեսանելի, բայց ներկայացված հաղորդումները չեն փոխանցվում ոչ մի արտաքին ծառայության։

## Brave Search API Key

Նույնպես **Advanced** տաբ-ում, **Brave Search API Key** դաշտը թույլ է տալիս ակտիվացնել [Internet Search](../configuration/internet-search) հնարավորությունը։

| Field | Description |
|---|---|
| **Brave Search API Key** | Ձեր Brave Search developer dashboard-ից ստացած API key-ը։ Անհրաժեշտ է AI assistant-ում internet search-ի ակտիվացման համար։ |

Դաշտի անունը ներառում է կլիկելի հղում՝ Brave Search API գրանցման էջին։ Դատարկ թողեք, որպեսզի անջատեք internet search-ը։

Այս բանալու վերաբերյալ օգտվողների համար նախատեսված փաստաթուղթում බලන්න [Internet Search](../configuration/internet-search)։

## Feature Flags (Ֆիքչեր Դրոշներ)

Նույնպես ներդրված v1.9.0-ով, **Settings → Feature Flags** տաբը տրամադրում է անընդհատ ֆունկցիոնալության համար toggle switches։ Յուրաքանչյուր flag-ը կամ ակտիվ է կամ անջատված է ամբողջ mạngի համար. այս պահին site-ի առանձնացուցիչ գերակայություն չկա։

### Feature Flags-ի մուտքագրում

1. WordPress admin-ում գնացեք **Gratis AI Agent → Settings**։
2. Կլիկեք **Feature Flags** տաբը։

### Access Control Flags (Մուտքի Կառավարման Դրոշներ)

| 🚩 | Default | Նկարագրություն |
|---|---|---|
| **Միացնել Տրամադրողներին (Administrators)** | Միացված չէ (Off) | Այս տարբերակը 켜ելիս, միայն `administrator` դերը ունեցող օգտվողները կարող են բացել AI Agent-ի քաթի պանելը։ Մնացած բոլոր դերերը կտեսնեն «Կապվեք ձեր վարչական հետ» հաղորդագրությունը։ |
| **Միացնել Ցանցային Վարչականներին (Network Admins)** | Միացված չէ (Off) | Եթե այս տարբերակը միացված է մուլտիսայտ ցանցում, միայն Super Admin-ները կարող են օգտագործել agent-ը։ Անհատական կայքերի ադմինները արգելված են։ Եթե երկուսն էլ 켜ված են, այս տարբերակը գերազանցում է «Միացնել Տրամադրողներին»-ին։ |
| **Թույլատրել Սուբսկրիբտորների Մուտք** | Միացված չէ (Off) | Եթե այս տարբերակը 켜ված է, `subscriber` դերը ունեցող օգտվողները կարող են օգտագործել քաթի ինտերֆեյսը, բայց սահմանափակվում են միայն ընթերցման հնարավորություններով (չկան գրառումներ ստեղծելու կամ կարգավորումներ փոխելու իրավունքներ)։ |
| **Անջատել ՉՄասնակիցների համար** | Միացված չէ (Off) | Ինտեգրվում է Ultimate Multisite-ի անդամության կարգավիճակի հետ։ Եթե այս տարբերակը 켜ված է, քաթը թաքնվում է այն կայքերի համար, որոնք ակտիվ անդամակցություն չունեն։ |

### Բրենդավորման Դրոշեր (Branding Flags)

| 🚩 | Default | Նկարագրություն |
|---|---|---|
| **Թաքցնել «Powered by Gratis AI Agent» Տողը** | Միացված չէ (Off) | Հեռացնում է քաթի վի젯ի ներքևում ցուցադրվող բրենդավորման նշումը։ Խորհուրդ է տրվում սպիտակ թեթև (white-label) տարբերակների համար։ |
| **Անձնական Agent-ի Անուն** | *(пусто)* | Փոխարինում է քաթի վերևի գլխավոր հատվածում և ադմինի մենյուում ստանդարտ «Gratis AI Agent» թեգը ձեր սեփական անվանով։ Դատարկ թողեք՝ ստանդարտը օգտագործելու համար։ |
| **Թաքցնել Agent-ի Ընտրիչը** | Միացված չէ (Off) | Եթե այս տարբերակը 켜ված է, օգտվողները չեն կարող անցնել հինգ ներկառուցված agent-ների միջև։ Ընթացիկ agent-ը կֆիքսված է այնով, ինչ սահմանված է Settings → General-ում որպես լռելյայն (default)։ |
| **Օգտագործել Կայքի Իկոնան որպես Chat Avatar** | Միացված չէ (Off) | Փոխարինում է քաթի վի젯ի գլխավոր հատվածում ստանդարտ AI պատկերակը WordPress-ի կայքի պատկերով (սահմանվում է Appearance → Customize → Site Identity-ում)։ |

### Изменеումների Կիրառում

Ցանկացած դրոշ փոխելուց հետո սեղմեք **Save Settings**-ը։ Փոփոխությունները անմիջապես ուժի մեջ են մտնում՝ անհրաժեշտ չէ կաշկեն (cache flush) կամ պլագինը վերակայելու համար։
