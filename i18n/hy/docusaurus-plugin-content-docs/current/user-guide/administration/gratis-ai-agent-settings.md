# Gratis AI Agent-ի կարգավորումներ

**Settings → Advanced** էկրանը Gratis AI Agent-ում տրամադրում է ադմինիստրատորի մակարդակի կազմաձևում backend ինտեգրումների համար։ Այս էջը փաստաթղթավորում է feedback-ի փոխանցումը, որոնման մատակարարի բանալիները, կառավարվող Superdav ծառայության կարգավորումը, Google Calendar-ի կառավարիչները, TextBee SMS-ի կարգավորումները և ամբողջ ցանցի գործառույթների դրոշակները։

## Կարգավորումներ մուտք գործելը {#accessing-settings}

1. WordPress-ի ադմինիստրատորում գնացեք **Gratis AI Agent → Settings**։
2. Սեղմեք **Advanced** ներդիրը։

## Feedback Endpoint-ի կազմաձևում {#feedback-endpoint-configuration}

Feedback endpoint-ը ստանում է POST հարցումներ AI agent-ից, երբ օգտատերը feedback է ուղարկում thumbs-down կոճակի, auto-prompt բաների կամ `/report-issue` հրամանի միջոցով։

| Դաշտ | Նկարագրություն |
|---|---|
| **Feedback Endpoint URL** | URL-ը, որը ստանում է feedback-ի ներկայացումները որպես HTTP POST հարցումներ՝ JSON մարմնով։ |
| **Feedback API Key** | bearer token, որն ուղարկվում է յուրաքանչյուր feedback հարցման `Authorization` header-ում։ Թողեք դատարկ, եթե ձեր endpoint-ը authentication չի պահանջում։ |

### Սպասվող JSON payload {#expected-json-payload}

Ձեր feedback endpoint-ը պետք է ընդունի JSON մարմին՝ առնվազն հետևյալ դաշտերով.

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Լրացուցիչ դաշտեր կարող են լինել payload-ում՝ կախված խոսակցության համատեքստից։

### `triage_category` արժեքներ {#triagecategory-values}

AI triage շերտը payload-ը փոխանցելուց առաջ `triage_category`-ին վերագրում է հետևյալ արժեքներից մեկը.

| Արժեք | Իմաստ |
|---|---|
| `factual_error` | Օգնականը տրամադրել է սխալ փաստական տեղեկություն։ |
| `unhelpful_answer` | Պատասխանը տեխնիկապես ճիշտ էր, բայց օգտակար չէր։ |
| `inappropriate_content` | Պատասխանը պարունակում էր բովանդակություն, որը չպետք է ցուցադրվի օգտատերերին։ |
| `other` | Feedback-ը չի համապատասխանել հայտնի կատեգորիայի։ |

### Authentication {#authentication}

Եթե ձեր endpoint-ը պահանջում է authentication, **Feedback API Key** դաշտը սահմանեք ձեր bearer token-ի արժեքով։ Agent-ն ուղարկում է.

```
Authorization: Bearer <your-api-key>
```

Եթե **Feedback API Key** դաշտը դատարկ է, `Authorization` header չի ուղարկվում։

### Feedback-ի հավաքագրման անջատում {#disabling-feedback-collection}

Ե՛վ **Feedback Endpoint URL**, և՛ **Feedback API Key** դաշտերը թողեք դատարկ։ Thumbs-down կոճակը և feedback UI-ը մնում են տեսանելի օգտատերերին, բայց ներկայացումները չեն փոխանցվում որևէ արտաքին ծառայության։

## Brave Search API Key {#brave-search-api-key}

Նաև **Advanced** ներդիրում **Brave Search API Key** դաշտը միացնում է [Ինտերնետ որոնում](../configuration/internet-search) հնարավորությունը։

| Դաշտ | Նկարագրություն |
|---|---|
| **Brave Search API Key** | Ձեր API բանալին Brave Search-ի մշակողի Dashboard-ից։ Պահանջվում է AI օգնականում ինտերնետ որոնումը միացնելու համար։ |

Դաշտի պիտակը ներառում է սեղմվող հղում դեպի Brave Search API-ի գրանցման էջ։ Թողեք դատարկ՝ ինտերնետ որոնումն անջատելու համար։

Այս գործառույթի վերջնական օգտատիրոջ փաստաթղթերի համար տես [Ինտերնետ որոնում](../configuration/internet-search)։

## Կառավարվող Superdav ծառայություն {#managed-superdav-service}

Superdav AI Agent v1.18.0-ը ավելացնում է կառավարվող Superdav ծառայության endpoint-ներ և ավտոմատ կապի տրամադրում աջակցվող կայքերի համար։ Օգտագործեք այս կառավարիչները, երբ ձեր կայքը պետք է միանա հյուրընկալվող մատակարարին՝ ձեռքով կազմաձևված ծառայության endpoint-ի փոխարեն։

| Դաշտ | Նկարագրություն |
|---|---|
| **Managed Superdav Service** | Միացնում է հյուրընկալվող Superdav ծառայության կապը աջակցվող կայքերի համար։ |
| **Provision Connection** | Սկսում է endpoint-ի և հավատարմագրերի ավտոմատ տրամադրումը։ Օգտագործեք սա՝ հաստատելուց հետո, որ կայքը պետք է օգտագործի կառավարվող մատակարարին։ |
| **Service Endpoint / Connection Status** | Ցույց է տալիս ընթացիկ endpoint-ը կամ կապի վիճակը տրամադրումից հետո։ |

Տրամադրումից հետո պահպանեք կարգավորումները և ստուգեք կապի վիճակը՝ նախքան կառավարվող ծառայության աշխատանքային հոսքերին ապավինելը։ Եթե տրամադրումը ձախողվի, վերանայեք ցուցադրվող կրկնափորձի ուղեցույցը և հաստատեք, որ կայքն ունի հյուրընկալվող մատակարարն օգտագործելու թույլտվություն։

## Google Calendar-ի կազմաձևում {#google-calendar-configuration}

Երբ Superdav AI Agent v1.18.0-ի օրացույցի գործառույթները միացված են, agent-ը կարող է կարդալ կազմաձևված օրացույցները և միջոցառումների մանրամասները։ Օրացույցի գործիքները կարդալուն ուղղված են և օգտակար են ժամանակացույցին տեղյակ հիշեցումների, մասնակիցների հետ հետագա կապի և կոնտակտների համապատասխանեցման համար։

| Դաշտ | Նկարագրություն |
|---|---|
| **Google Calendar Credentials** | Պահում է օրացույցի տվյալները կարդալու համար անհրաժեշտ հավատարմագրերը կամ token կապը։ |
| **Calendar Selection** | Սահմանափակում է, թե կազմաձևված որ օրացույցները կարող է agent-ը ստուգել։ |
| **Calendar Connection Status** | Հաստատում է՝ արդյոք ընթացիկ հավատարմագրերը կարող են կարդալ օրացույցներ և միջոցառումներ։ |

Օրացույցի հավատարմագրերը սահմանափակեք միայն այն օրացույցներով, որոնք անհրաժեշտ են agent-ին։ Վերամիացրեք կամ փոխարինեք հավատարմագրերը, եթե վիճակը նշում է ժամկետանց token։

## TextBee SMS ծանուցումներ {#textbee-sms-notifications}

Superdav AI Agent v1.18.0-ը ավելացնում է TextBee-ն որպես SMS մատակարար՝ կազմաձևված ծանուցման աշխատանքային հոսքերի համար։ SMS ծանուցումները պետք է զուգակցվեն մարդու հաստատման դարպասների հետ զգայուն կամ օգտատերերին ուղղված հաղորդագրությունների համար։

| Դաշտ | Նկարագրություն |
|---|---|
| **TextBee API Key** | Վավերացնում է հարցումները դեպի TextBee SMS մատակարար։ |
| **TextBee Device / Sender** | Ընտրում է TextBee ուղարկողին կամ սարքը, որն օգտագործվում է ելքային հաղորդագրությունների համար, երբ դա պահանջվում է մատակարարի կողմից։ |
| **SMS Notifications Enabled** | Թույլ է տալիս հաստատված աշխատանքային հոսքերին ուղարկել տեքստային հաղորդագրությունների ծանուցումներ։ Թողեք անջատված՝ SMS ուղարկումները կանխելու համար։ |

Թեստային հաղորդագրություն ուղարկեք միայն ադմինիստրատորին պատկանող համարի, ապա հաստատեք approval-gate-ի վարքագիծը՝ նախքան պլանավորված կամ մասնակիցներին ուղղված հիշեցումները միացնելը։

## Գործառույթների դրոշակներ {#feature-flags}

Նաև ներկայացված v1.9.0-ում՝ **Settings → Feature Flags** ներդիրը տրամադրում է փոխարկիչներ ընտրովի ֆունկցիոնալության համար։ Յուրաքանչյուր դրոշակ կա՛մ միացված է, կա՛մ անջատված ամբողջ ցանցում․ այս պահին առանձին կայքի համար վերագրում չկա։

### Գործառույթների դրոշակներ մուտք գործելը {#accessing-feature-flags}

1. WordPress-ի ադմինիստրատորում գնացեք **Gratis AI Agent → Settings**։
2. Սեղմեք **Feature Flags** ներդիրը։

### Մուտքի կառավարման դրոշակներ {#access-control-flags}

| Դրոշակ | Լռելյայն | Նկարագրություն |
|---|---|---|
| **Սահմանափակել Administrators-ով** | Անջատված | Երբ միացված է, միայն `administrator` դերով օգտվողները կարող են բացել AI Agent զրույցի վահանակը։ Մնացած բոլոր դերերը փոխարենը տեսնում են «Կապվեք ձեր administrator-ի հետ» հաղորդագրությունը։ |
| **Սահմանափակել Network Admins-ով** | Անջատված | Երբ միացված է multisite ցանցում, միայն Super Admins-ը կարող են օգտագործել գործակալը։ Առանձին կայքերի admins-ները արգելափակվում են։ Գերակայում է «Սահմանափակել Administrators-ով»-ի նկատմամբ, եթե երկուսն էլ միացված են։ |
| **Թույլատրել Subscriber-ի մուտքը** | Անջատված | Երբ միացված է, `subscriber` դերով օգտվողները կարող են օգտագործել զրույցի միջերեսը, բայց սահմանափակված են միայն կարդալու հնարավորություններով (առանց գրառումների ստեղծման կամ կարգավորումների փոփոխությունների)։ |
| **Անջատել ոչ անդամների համար** | Անջատված | Ինտեգրվում է Ultimate Multisite անդամակցության կարգավիճակի հետ։ Երբ միացված է, զրույցը թաքցվում է այն կայքերի համար, որոնք չունեն ակտիվ անդամակցություն։ |

### Բրենդավորման դրոշակներ {#branding-flags}

| Դրոշակ | Լռելյայն | Նկարագրություն |
|---|---|---|
| **Թաքցնել «Powered by Gratis AI Agent» ստորագիր** | Անջատված | Հեռացնում է զրույցի վիջեթի ներքևում ցուցադրվող բրենդավորման վերագրման տողը։ Խորհուրդ է տրվում white-label տեղակայումների համար։ |

### Փոփոխությունների կիրառում {#applying-changes}
