# WP Engine Ինտեգրում

## Ընդհանուր տեսքը
WP Engine-ը պրեմիում կառավարվող WordPress հոස්թինգային հարթակ է, որը մատուցում է օպտիմալ արագություն, անվտանգություն և մասշտաբայնություն WordPress սայթերի համար։ Այս ինտեգրումը թույլ է տալիս ավտոմատ դոմենի սինխրոնիզացիա Ultimate Multisite-ի և WP Engine-ի միջև։

## Հատկություններ
- Ավտոմատ դոմենի սինխրոնիզացիա
- Բազմաсайт (multisite) տեղադրումների համար ենթադիր տիրույթների (subdomain) աջակցություն
- WP Engine-ի գոյություն ունեցող համակարգերի հետ անխափան ինտեգրում

## Պահանջներ
Ինտեգրումը ավտոմատորեն հայտնաբերում է, թե արդյոք դուք հոස්թավորում եք WP Engine-ի վրա և օգտագործում է ներկառուցված WP Engine API-ն։ Եթե WP Engine plugin-ը ակտիվ է և պատշաճ կերպով կարգավորված է, լրացուցիչ կոնֆիգուրացիայի անհրաժեշտ չէ։

Սակայն, եթե ձեզ անհրաժեշտ է ինտեգրումը ձեռքով կոնֆիգուրացնել, դուք կարող եք սահմանել հետևյալ հաստատուններից մեկը՝ ձեր `wp-config.php` ֆայլում.

```php
define('WPE_APIKEY', 'your_api_key'); // Ավելի նախընտրելի մեթոդ
// ԿԱՄ
define('WPE_API', 'your_api_key'); // Այլընտրանքային մեթոդ
```

## Կài đặtի ցուցումներ

### 1. Ստուգեք WP Engine Plugin-ը
Եթե դուք հոස්թավորում եք WP Engine-ի վրա, WP Engine plugin-ը արդեն պետք է տեղադրված և ակտիվ լինի։ Ստուգեք հետևյալները.

1. WP Engine plugin-ը ակտիվ է
2. Գոյություն ունի `wp-content/mu-plugins/wpengine-common/class-wpeapi.php` ֆայլը

### 2. Թողեք ինտեգրումը
1. Ձեր WordPress admin-ում գնացեք Ultimate Multisite > Settings (Կարգավորումներ)
2. Գնացեք «Domain Mapping» (Դոմենի քարտեզագրում) տաբին
3. Կանգնեք «Host Integrations» (Հոස්տ ինտեգրումներ) բաժնի մոտ
4. Թողեք WP Engine-ի ինտեգրումը ակտիվ
5. Սեղմեք «Save Changes» (Փոփոխությունները պահել)

## Ինչպես է այն աշխատում

### Դոմենի սինխրոնիզացիա
Երբ Ultimate Multisite-ում դոմենը քարտեզագրվում է.

1. Ինտեգրումը WP Engine API-ն օգտագործում է՝ դոմենը ձեր WP Engine տեղադրման մեջ հավելելու համար
2. WP Engine-ը կառավարում է դոմենի կոնֆիգուրացիան և SSL վկայագրության տրամադրումը
3. Երբ դոմենի քարտեզագրումը հեռացվում է, ինտեգրումը դոմենը WP Engine-ից հեռացնում է

### Սուբդոմենի (Subdomain) աջակցություն

Սա ենթադրական ենթադրություն է՝ հաշվի առնելով տրված ինտերֆեյսը։

Ենթադրային ենթադրությամբ, այստեղ ներկայացվում է WordPress-ի փաստաթղթերի թարգմանությունը.

**Սոբդոմեն (subdomain) մուլտիսայթ ինստալացիաների համար.**

1. Ինտեգրումը WP Engine-ի հետ կապում է յուրաքանչյուր սոբդոմեն նոր սայթ ստեղծելիս։
2. WP Engine-ը մշակում է սոբդոմենի կոնֆիգուրացիան։
3. Երբ սայթը ջնջվում է, ինտեգրումը այն հեռացնում է WP Engine-ից։

## Կարևոր նշումներ

### Wildcard Դոմեններ (Wildcard Domains)

Սոբդոմեն մուլտիսայթ ինստալացիաների համար խորհուրդ է տրվում կապվել WP Engine-ի հետ՝ պահանջելով wildcard domain կոնֆիգուրացիա։ Սա թույլ է տալիս բոլոր սոբդոմենները ավտոմատ աշխատեն առանց յուրաքանչյուրը առանձին ավելացնելու անհրաժեշտության։

### SSL Գրանցումներ (SSL Certificates)

WP Engine-ը ավտոմատ կերպով մշակում է բոլոր այն դոմենների համար SSL վկայագրականի տրամադրումը և թարմացումը, որոնք ավելացվում են այս ինտեգրման միջոց։ Առանձին կոնֆիգուրացիա անհրաժեշտ չէ։

## Խնդիրների լուծում (Troubleshooting)

### API Կապի խնդիրներ
- Ստուգեք, թե արդյոք WP Engine պլագինը ակտիվ է և պատշաճ կերպով կոնֆիգուրացված։
- Եթե դուք ձեռքով սահմանել եք API key-ը, ստուգեք, թե արդյոք այն ճիշտ է։
- Եթե API-ի հետ խնդիրներ ունեք, կապվեք WP Engine-ի հետ։

### Դոմեն չի ավելացվել
- Ստուգեք Ultimate Multisite-ի լոգերը՝ որևէ սխալ հաղորդագրություն առկա արդյոք։
- Հաստատեք, որ դոմենը արդեն WP Engine-ում չի ավելացվել։
- Ապահովեք, որ ձեր WP Engine պլանը աջակցում է այն դոմենների քանակին, որոնք դուք ավելացնում եք։

### Սոբդոմենի խնդիրներ
- Եթե սոբդոմենները չեն աշխատում, կապվեք WP Engine-ի հետ՝ պահանջելով wildcard domain կոնֆիգուրացիա։
- Հաստատեք, որ ձեր DNS կարգավորումները ճիշտ են կազմված հիմնական դոմենի և սոբդոմենների համար։
