# Closte Ինտեգրում

## Ընդհանուր տեսքը {#overview}
Closte-ը Google Cloud ենթաճարպի վրա կառուցված WordPress հոස්ինգի պլատֆորմ է։ Այս ინտեգրումը թույլ է տալիս ավտոմատ դոմենի սինխրոնիզացիա և SSL վկայագրերի կառավարում Ultimate Multisite-ի և Closte-ի միջև։

## Առանձնահատկություններ {#features}
- Ավտոմատ դոմենի սինխրոնիզացիա
- SSL վկայագրերի կառավարում
- Wildcard դոմենների աջակցություն
- Closte-ում գործարկելիս ոչ մի կոնֆիգուրացիայի անհրաժեշտ չէ

## Առանձնահատկություններ {#requirements}
Եթե օգտագործում եք Closte, հետևյալ հաստատունը պետք է սահմանվի ձեր `wp-config.php` ֆայլում.

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Այս հաստատունը, որպես կանոն, արդեն սահմանված է, եթե դուք хоสต์ավորում եք Closte-ում։

## Կài đặtի ցուցումներ {#setup-instructions}

### 1. Ստուգեք ձեր Closte API Key-ը {#1-verify-your-closte-api-key}
Եթե դուք хоสต์ավորում եք Closte-ում, `CLOSTE_CLIENT_API_KEY` հաստատունը արդեն պետք է սահմանված լինի ձեր `wp-config.php` ֆայլում։ Դա կարող եք ստուգել՝ ստուգելով ձեր `wp-config.php` ֆայլը։

### 2. Թողեք ინտեգրումը {#2-enable-the-integration}
1. Ձեր WordPress ադմինից գնացեք Ultimate Multisite > Settings (Կարգավորումներ)
2. Գնացեք "Domain Mapping" (Դոմենի քարտեզագրում) բաժինը
3. Ձգեք ներքև և գտնեք "Host Integrations" (Հոස් ინտեգրումներ)
4. Թողեք Closte-ի ინտեգրումը ակտիվ
5. Սեղմեք "Save Changes" (Փոփոխությունները պահել)

## Ինչպես է այն աշխատում {#how-it-works}

Երբ դոմենը քարտեզագրվում է Ultimate Multisite-ում.

1. Ինտեգրումը հայտի հարցում է Closte-ի API-ին՝ դոմենը ձեր կիրառման մեջ ավելացնելու համար։
2. Closte-ը ավտոմատ կերպով կառավարում է SSL վկայագրերի տրամադրումը (provisioning)։
3. Երբ դոմենի քարտեզագրումը հեռացվում է, ინտեգրումը դոմենը Closte-ից հեռացնում է։

Ինտեգրումը նաև աշխատում է Ultimate Multisite-ի DNS ստուգման ժամանակային միջակայքի կարգավորման հետ, ինչը թույլ է տալիս կոնֆիգուրացնել այն, թե որքան հաճախ համակարգը ստուգում է DNS-ի տարածումը և SSL վկայագրերի տրամադրումը։

## Դոմենի գրանցման գրանցում {#domain-record-creation}

Այս ինտեգրումը երաշխավորում է, որ կայք ստեղծելիս կամ դրա պատճենում (duplication) անմիջապես ստեղծվի տիրույթի գրանցում (domain record)։ Սա հատկապես կարևոր է Closte-ի ինտեգրման համար, քանի որ տիրույթի գրանցման ստեղծումը հրահանգում է Closte API-ին՝ տիրույթը և SSL վկայականը (certificate) ստեղծելու համար։

## Խնդիրների լուծում (Troubleshooting) {#troubleshooting}

### API միացման խնդիրներ {#api-connection-issues}
- Ստուգեք, որ ձեր Closte API բանալին (key) ճիշտ է։
- Համոզվեք, որ ձեր Closte հաշվին անհրաժեշտ թույլտվությունները (permissions) կան։

### SSL վկայականների խնդիրներ {#ssl-certificate-issues}
- Closte-ին կարող է ժամանակ պահանջել մի քանի ժամանակ՝ SSL վկայական ստոխանակելու համար (սովորաբար 5-10 րոպե)։
- Հաստատեք, որ ձեր տիրույթները (domains) ճիշտ են ուղղված Closte-ի սերվերի IP հասցեին։
- Ստուգեք ձեր տիրույթի DNS գրանցումները՝ համոզվելու համար, որ դրանք ճիշտ են կարգավորված։

### Տիրույթը չի ավելացվել {#domain-not-added}
- Ստուգեք Ultimate Multisite-ի logs-ներում (հետևյալ գործողությունների տվյալները) ցանկացած սխալի հաղորդագրության համար։
- Հաստատեք, որ տիրույթը արդեն Closte-ում չի ավելացվել։
- Ապահովեք, որ ձեր տիրույթի DNS գրանցումները ճիշտ են կարգավորված։

### DNS ստուգման միջակայք (DNS Check Interval) {#dns-check-interval}
- Եթե SSL վկայականների ստոխանակումը շատ երկար է տևում, դուք կարող եք փոփոխել Տիրույթի քարտեզագրման (Domain Mapping) կարգավորումներում DNS ստուգման միջակայքը։
- Սխալի արժեքը 300 վայրկյան է (5 րոպե), բայց դուք կարող եք այն սահմանել մինչև 10 վայրկյան՝ թեստավորման ընթացքում ավելի արագ ստուգումների համար։
