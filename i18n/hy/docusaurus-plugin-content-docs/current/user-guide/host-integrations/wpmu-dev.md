# WPMU DEV Ինտեգրում {#wpmu-dev-integration}

## Ընդհանուր տեսքը {#overview}
WPMU DEV-ը WordPress-ի համար հարթակ է, որը առաջարկում է սերվերների հոස්տինգ, plugin-ներ և ծառայություններ։ Այս ინտեգրումը թույլ է տալիս ավտոմատ դոմենի սինխրոնիզացիա և SSL վկայագրերի կառավարում Ultimate Multisite-ի և WPMU DEV հոස්տինգի միջև։

## Առանձնահատկություններ {#features}
- Դոմենի ավտոմատ սինխրոնիզացիա
- SSL վկայագրերի կառավարում
- Լրացուցիչ SSL վկայագրերի ստուգման փորձեր

## Առ պահանջներ {#requirements}
Ինտեգրումը ինքնաբերաբար հայտնաբերում է, թե արդյոք դուք հոස්տավորում եք WPMU DEV-ում և օգտագործում է ներկառուցված API-ն։ Եթե դուք հոස්տավորում եք WPMU DEV-ում, լրացուցիչ կոնֆիգուրացիա անելու կարիք չկա։

Ինտեգրումը ստուգում է `WPMUDEV_HOSTING_SITE_ID` հաստատունի առկայությունը, որը ավտոմատ սահմանվում է WPMU DEV-ում հոස්տավորելիս։

## Կài đặtի ցուցումներ {#setup-instructions}

### 1. Ստուգեք WPMU DEV Հոස්տինգը {#1-verify-wpmu-dev-hosting}
Եթե դուք հոස්տավորում եք WPMU DEV-ում, անհրաժեշտ հաստատունները արդեն սահմանված են։ Ստուգեք հետևյալը.

1. `WPMUDEV_HOSTING_SITE_ID` հաստատունը սահմանված է ձեր միջավայրում (environment)
2. Ձեզ կա WPMU DEV-ի գործող մենքենթ (membership) և API-ի մուտք

### 2. Թողեք ინտեգրումը {#2-enable-the-integration}
1. Ձեր WordPress admin-ում գնացեք Ultimate Multisite > Settings
2. Գնացեք «Domain Mapping» բաժինը
3. Կանգնեք «Host Integrations»-ի տակ և թողեք WPMU DEV ინտեգրումը ակտիվ (Enable)
4. Սեղմեք «Save Changes»

## Ինչպես է այն աշխատում {#how-it-works}

### Դոմենի սինխրոնիզացիա {#domain-syncing}
Երբ Ultimate Multisite-ում դոմենը կապվում է (mapped):

1. Ինտեգրումը WPMU DEV API-ն օգտագործում է՝ դոմենը ձեր հոස්տինգի հաշվին ավելացնելու համար
2. Այն ավտոմատ կերպով ավելացնում է դոմենի www տարբերակը նույնպես
3. WPMU DEV-ը պատասխանատու է դոմենի կոնֆիգուրացիայի և SSL վկայագրերի տրամադրման համար

### SSL վկայագրերի կառավարում {#ssl-certificate-management}

Ինտեգրումը կարգավորված է WPMU DEV հոස්թինգի համար SSL վկայականների ստուգման փորձերի քանակը մեծացնելու համար, քանի որ SSL վկայականները տրամադրելու և տեղադրելու համար կարող է ժամանակ պահանջվել։ Ըստ ստանդարտ 5 փորձերի համեմատ, այն կփորձի մինչև 10 անգամ SSL վկայականների ստուգումը։

## Կարևոր նշումներ {#important-notes}

### Դոմենի հեռացում {#domain-removal}

Ընթացիկ պահին WPMU DEV API-ն չունի դոմեններ հեռացնելու միջոց։ Երբ Ultimate Multisite-ում հեռացվում է դոմենային քարտեզագրումը (domain mapping), դոմենը կմնա ձեր WPMU DEV հոස්թինգի հաշվում։ Անհրաժեշտության դեպքում անհրաժեշտ է այն ձեռքով հեռացնել WPMU DEV հոස්թինգի վահանայից (dashboard)։

### API ինտենտիֆիկացիա (Authentication) {#api-authentication}

Ինտեգրումը օգտագործում է WPMU DEV API key-ը, որը պահվում է ձեր WordPress ցուցակում `wpmudev_apikey` տարբերակի տեսքով։ Սա ավտոմատ կերպով կարգավորվում է՝ ձեր սайտը WPMU DEV-ի հետ կապելիս։

## Խնդրի լուծում (Troubleshooting) {#troubleshooting}

### API միացման խնդիրներ {#api-connection-issues}
- Ստուգեք, որ ձեր սайտը պատշաճ կերպով կապված է WPMU DEV-ի հետ։
- Ստուգեք, թե արդյոք `wpmudev_apikey` տարբերակը կարգավորված է ձեր WordPress ցուցակում։
- Համոզվեք, որ ձեր WPMU DEV անդամակցությունը (membership) ակտիվ է։

### SSL վկայականների խնդիրներ {#ssl-certificate-issues}
- WPMU DEV-ին կարող է ժամանակ պահանջվել SSL վկայականներ տրամադրելու համար (սովորաբար 5-15 րոպե)։
- Ինտեգրումը կարգավորված է SSL վկայականների ստուգման համար մինչև 10 անգամ։
- Եթե բազմակի փորձերից հետո SSL վկայականները դեռ չեն տրամադրվել, կապվեք WPMU DEV-ի հետ աջակցության հետ։

### Դոմենը չի ավելացվել {#domain-not-added}
- Ստուգեք Ultimate Multisite-ի logs-ում (հետևյալների գրանցում) որևէ սխալի հաղորդագրություն։
- Համոզվեք, որ դոմենը արդեն չի ավելացվել WPMU DEV-ում։
- Համոզվեք, որ ձեր WPMU DEV հոස්թինգի պլանը աջակցում է այն թվին, որոնք դուք ավելացնում եք դոմեններ։
