# Ultimate AI Connector՝ համատեղելի վերջնակետերի համար {#ultimate-ai-connector-for-compatible-endpoints}

Ultimate AI Connector՝ համատեղելի վերջնակետերի համար, ինտեգրում է OpenAI-ի հետ համատեղելի AI մատակարարների API-ները ձեր WordPress բազմակայք ցանցի հետ։ Կարգավորեք մի քանի AI մատակարարներ՝ յուրաքանչյուրն իր սեփական վերջնակետի URL-ով, API բանալիով և լռելյայն մոդելով, և ուղղորդեք հարցումները ավտոմատ պահուստային անցումով, երբ մատակարարը անհասանելի է։

## Հիմնական հնարավորություններ {#key-features}

- **Բազմամատակարար աջակցություն**: Ավելացրեք մի քանի AI մատակարարի քարտեր, յուրաքանչյուրն իր սեփական վերջնակետի URL-ով, API բանալիով և լռելյայն մոդելով
- **Քաշելով վերադասավորում**: Դասավորեք մատակարարներին ըստ առաջնահերթության՝ օգտագործելով յուրաքանչյուր քարտի բռնակը․ հարցումները օգտագործում են մատակարարներին հերթականությամբ
- **Ավտոմատ պահուստային ուղղորդում**: Եթե հիմնական մատակարարը անհասանելի է, հարցումները ավտոմատ անցնում են հաջորդ կարգավորված մատակարարին
- **Մատակարարին հատուկ կարգավորում**: Յուրաքանչյուր մատակարար կարգավորվում է անկախ՝ իր սեփական հավատարմագրերով և մոդելի ընտրությամբ
- **Քարտերի վրա հիմնված կարգավորումների UI**: Կառուցված է կայուն WordPress 6.9+ Card բաղադրիչների վրա (CardBody, CardHeader, CardDivider)
- **Ավտոմատ բացում ավելացման ժամանակ**: Նոր մատակարարի քարտերը ավտոմատ բացվում են, որպեսզի կարողանաք անմիջապես լրացնել մանրամասները
- **Սկրիպտի քեշի շրջանցում**: Մատակարարի սկրիպտների քեշը շրջանցվում է plugin-ի թարմացման ժամանակ՝ ապահովելու, որ բեռնվի վերջին տարբերակը

## Պահանջներ {#requirements}

- WordPress 5.3 կամ ավելի բարձր
- PHP 7.4 կամ ավելի բարձր
- Ultimate Multisite plugin (ակտիվ)
- OpenAI-ի հետ համատեղելի API վերջնակետ (ինքնուրույն հոսթավորված կամ երրորդ կողմի)

## Տեղադրում {#installation}

1. Վերբեռնեք հավելման ֆայլերը ձեր `/wp-content/plugins/` գրացուցակ
2. Ակտիվացրեք plugin-ը WordPress-ի «Պլագիններ» ընտրացանկի միջոցով
3. Անցեք **Ultimate Multisite → AI Connector**՝ հավելումը կարգավորելու համար

## Կարգավորում {#configuration}

### Բազմամատակարար կարգավորում {#multi-provider-setup}

Կարգավորման էկրանը օգտագործում է քարտերի վրա հիմնված դասավորություն, որտեղ յուրաքանչյուր քարտ ներկայացնում է մեկ AI մատակարար։ Կարող եք ավելացնել այնքան մատակարար, որքան անհրաժեշտ է։

#### Մատակարարի ավելացում {#adding-a-provider}

1. Գնացեք **Ultimate Multisite → AI Connector** ցանցի ադմինիստրատորի բաժնում
2. Սեղմեք **Ավելացնել մատակարար**
3. Նոր մատակարարի քարտ է հայտնվում և ավտոմատ բացվում
4. Լրացրեք մատակարարի մանրամասները.
   - **Մատակարարի անուն** — պիտակ՝ այս մատակարարին նույնականացնելու համար (օր.՝ "OpenAI", "Mistral", "Local Ollama")
   - **Վերջնակետի URL** — OpenAI-ի հետ համատեղելի API-ի բազային URL-ը (օր.՝ `https://api.openai.com/v1`)
   - **API բանալի** — ձեր API բանալին այս մատակարարի համար
   - **Լռելյայն մոդել** — մոդելի նույնացուցիչը, որը լռելյայն օգտագործվում է այս մատակարարի համար (օր.՝ `gpt-4o`, `mistral-small`)
5. Սեղմեք **Պահպանել կարգավորումները**

#### Մատակարարի հեռացում {#removing-a-provider}

Սեղմեք **Հեռացնել** կոճակը ցանկացած մատակարարի քարտի վրա՝ այն ջնջելու համար։ Պահպանեք կարգավորումները՝ փոփոխությունը կիրառելու համար։

#### Մատակարարների վերադասավորում {#reordering-providers}

Քաշեք ցանկացած մատակարարի քարտի ձախ կողմում գտնվող բռնակի պատկերակը (⣿)՝ այն վերադասավորելու համար։ Հերթականությունը որոշում է պահուստային անցման առաջնահերթությունը. հարցումներն ուղարկվում են առաջին մատակարարին և, եթե հարցումը ձախողվում է, անցնում են հաջորդ մատակարարներին։

### Պահուստային ուղղորդում {#fallback-routing}

Երբ հիմնական մատակարարին ուղարկված հարցումը ձախողվում է՝ ժամանակի սպառման, սահմանաչափի կամ ծառայության խափանման պատճառով, միակցիչը ավտոմատ կրկին փորձում է ցուցակի հաջորդ մատակարարով։ Սա շարունակվում է ցուցակով ներքև, մինչև մատակարարներից մեկը հաջողվի կամ բոլոր մատակարարները փորձարկված լինեն։

| Սցենար | Վարքագիծ |
|----------|-----------|
| Հիմնական մատակարարը պատասխանում է | Հարցումը սպասարկվում է սովորական ձևով |
| Հիմնական մատակարարը ձախողվում է | Հարցումը կրկին փորձվում է հաջորդ մատակարարով |
| Բոլոր մատակարարները ձախողվում են | Սխալը վերադարձվում է կանչողին |

Պահուստային անցումը թափանցիկ է կանչող plugin-ի կամ թեմայի համար. նրանք ստանում են կամ հաջող պատասխան, կամ վերջնական սխալ՝ անկախ նրանից, թե քանի մատակարար է փորձարկվել։

### Մատակարարի կարգավորման տեղեկանք {#provider-configuration-reference}

| Դաշտ | Պարտադիր | Նկարագրություն |
|-------|----------|-------------|
| Մատակարարի անուն | Այո | Ցուցադրվող պիտակ այս մատակարարի համար |
| Վերջնակետի URL | Այո | OpenAI-ի հետ համատեղելի API-ի բազային URL |
| API բանալի | Այո | Վավերացման բանալի այս մատակարարի համար |
| Լռելյայն մոդել | Այո | Մոդելի նույնացուցիչ, որն օգտագործվում է, երբ հարցման մեջ մոդել նշված չէ |

## Խնդիրների լուծում {#troubleshooting}

### Մատակարարներ նշված չեն {#no-providers-are-listed}

Եթե մատակարարների ցուցակը պահպանելուց հետո դատարկ է, հաստատեք, որ՝

- Մատակարարներ ավելացնելուց հետո սեղմել եք **Պահպանել կարգավորումները**
- Ձեր դիտարկիչը չի արգելափակել կարգավորումների էջի պահպանումը (ստուգեք դիտարկիչի վահանակը JavaScript սխալների համար)
- Plugin-ը ակտիվ է, և դուք ցանցի ադմինիստրատորի էկրանին եք

### Հարցումները ձախողվում են բոլոր մատակարարների համար {#requests-are-failing-for-all-providers}

- Ստուգեք, որ յուրաքանչյուր մատակարարի վերջնակետի URL-ը հասանելի է ձեր սերվերից (ոչ միայն ձեր դիտարկիչից)
- Ստուգեք, որ յուրաքանչյուր API բանալի վավեր է և ժամկետանց չէ
- Վերանայեք մատակարարի սահմանաչափերի և քվոտայի կարգավիճակը նրա վահանակում
- Միացրեք WordPress-ի վրիպազերծման մատյանագրումը (`WP_DEBUG_LOG`)՝ միակցիչից մանրամասն սխալի հաղորդագրություններ գրանցելու համար

### Մատակարարը չի օգտագործվում պահուստային անցման համար {#a-provider-is-not-being-used-for-fallback}

Քաշելով վերադասավորումը փոխում է պահուստային անցման հերթականությունը։ Հաստատեք, որ մատակարարը, որից ակնկալում եք ստանալ պահուստային հարցումներ, ճիշտ դիրքում է ցուցակում, և որ նրա վերջնակետի URL-ը և API բանալին ճիշտ են։

### Նոր մատակարարի քարտը ավտոմատ չի բացվում {#new-provider-card-does-not-expand-automatically}

Ավտոմատ բացման համար անհրաժեշտ է, որ JavaScript-ը միացված լինի ձեր դիտարկիչում։ Եթե քարտը սեղմելիս չի բացվում, ստուգեք դիտարկիչի վահանակը սխալների համար։ Քարտի բովանդակությունը միշտ հասանելի է՝ սեղմելով քարտի վերնագիրը, նույնիսկ եթե ավտոմատ բացումը չի գործարկվում։
