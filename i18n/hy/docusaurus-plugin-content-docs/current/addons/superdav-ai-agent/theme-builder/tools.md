# Գործիքներ և կազմաձևում {#tools-and-configuration}

Theme Builder-ը օգտագործում է գործիքների հավաքածու՝ ձեր WordPress կայքը վերլուծելու, նախագծելու և կառուցելու համար։ 1.16.0 տարբերակում **sd-ai-agent/site-scrape-ն այժմ Tier 1 գործիք է**, ինչի շնորհիվ այն լռելյայն հասանելի է Theme Builder-ում։ Superdav AI Agent v1.18.0-ը նաև ներկայացնում է առանձին տարածվող Advanced ուղեկից փաթեթ՝ վստահելի մշակողների աշխատանքային գործընթացների համար․ այդ գործիքները WordPress.org հիմնական փաթեթի մաս չեն և պետք է առանձին տեղադրվեն ու լիազորվեն։

## Հասանելի գործիքներ {#available-tools}

### Tier 1 գործիքներ (միշտ հասանելի) {#tier-1-tools-always-available}

Tier 1 գործիքները լռելյայն հասանելի են Theme Builder-ում՝ առանց լրացուցիչ կազմաձևման։

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**Նպատակ․** Վերլուծել գոյություն ունեցող կայքերը՝ դիզայնի ներշնչանք, բովանդակության կառուցվածք և գործառույթներ դուրս բերելու համար։

**Հնարավորություններ․**

- **Կայքի վերլուծություն** — հավաքել և վերլուծել մրցակից կամ ներշնչանքի կայքերը
- **Դիզայնի դուրսբերում** — նույնականացնել գույները, տառատեսակները և դասավորության օրինաչափությունները
- **Բովանդակության կառուցվածք** — հասկանալ էջերի կազմակերպումն ու հիերարխիան
- **Հատկությունների հայտնաբերում** — նույնականացնել plugin-ները և գործառույթները
- **Արտադրողականության վերլուծություն** — ստուգել էջի արագությունն ու օպտիմալացումը
- **SEO վերլուծություն** — վերանայել meta tag-երը և կառուցվածքային տվյալները

**Օգտագործում․**

```
Վերլուծիր example.com-ի դիզայնը և օգտագործիր այն որպես ներշնչանք իմ կայքի համար։
```

**Ինչ է այն դուրս բերում․**

- Գունապնակ և գույների օգտագործում
- Տպագրություն (տառատեսակներ և չափեր)
- Դասավորության կառուցվածք և միջակայքեր
- Նավիգացիոն օրինաչափություններ
- Բովանդակության կազմակերպում
- Պատկերների և մեդիայի օգտագործում
- Փոխազդեցիկ տարրեր
- Բջջային հարմարվողականություն

**Սահմանափակումներ․**

- Չի կարող հավաքել գաղտնաբառով պաշտպանված կայքեր
- Հարգում է robots.txt-ը և կայքի քաղաքականությունները
- Կարող է չֆիքսել դինամիկ բովանդակությունը
- JavaScript-ով ծանրաբեռնված կայքերի վերլուծությունը կարող է սահմանափակ լինել
- Մեծ կայքերի վերլուծությունը կարող է ավելի երկար տևել

### Tier 2 գործիքներ (ըստ ընտրության) {#tier-2-tools-optional}

Tier 2 գործիքները հասանելի են, երբ բացահայտ միացված են Theme Builder-ի կարգավորումներում։

#### Ընդլայնված վերլուծություններ {#advanced-analytics}

Տրամադրում է մանրամասն արտադրողականության չափումներ․

- Էջի բեռնման ժամանակներ
- Core Web Vitals
- SEO միավոր
- Մատչելիության միավոր
- Լավագույն գործելակերպերի միավոր

#### Բովանդակության օպտիմալացնող {#content-optimizer}

Վերլուծում և առաջարկում է բարելավումներ հետևյալի համար․

- Ընթեռնելիություն
- SEO օպտիմալացում
- Հիմնաբառերի օգտագործում
- Բովանդակության կառուցվածք
- Գործողության կոչի տեղադրում

### Tier 3 գործիքներ (Premium) {#tier-3-tools-premium}

Tier 3 գործիքները պահանջում են լրացուցիչ կազմաձևում կամ premium հատկություններ։

#### AI բովանդակության գեներատոր {#ai-content-generator}

Գեներացնում է բովանդակություն հետևյալի համար․

- Ապրանքի նկարագրություններ
- Ծառայությունների էջեր
- Բլոգի գրառումներ
- Meta նկարագրություններ
- Գործողության կոչի տեքստ

#### Ընդլայնված դիզայնի գործիքներ {#advanced-design-tools}

Տրամադրում է դիզայնի ընդլայնված հնարավորություններ․

- Custom CSS-ի գեներացում
- Անիմացիաների ստեղծում
- Փոխազդեցիկ տարրերի դիզայն
- Գույնի ընդլայնված տեսություն
- Տպագրության օպտիմալացում

### Advanced ուղեկից գործիքներ (առանձին փաթեթ) {#advanced-companion-tools-separate-package}

Advanced ուղեկից փաթեթը տարածվում է առանձին՝ Superdav AI Agent-ի հիմնական թողարկումից։ Այն նախատեսված է վստահելի մշակողների և կայքի սեփականատերերի միջավայրերի համար, որտեղ ադմինիստրատորները բացահայտ թույլ են տալիս գործակալին օգտագործել բարձր ազդեցության գործիքավորում։

Advanced ուղեկից հնարավորությունները կարող են ներառել․

- **Ֆայլային համակարգի գործիքներ** — ստուգել և փոփոխել հաստատված ֆայլերը վստահելի միջավայրերում
- **Տվյալների բազայի գործիքներ** — հարցումներ կատարել կամ թարմացնել կայքի տվյալները, երբ բացահայտ լիազորված է
- **WP-CLI գործիքներ** — գործարկել WordPress-ի սպասարկման և ստուգման հրամաններ
- **REST dispatcher գործիքներ** — կանչել գրանցված REST endpoint-ներ վերահսկվող dispatch աշխատանքային գործընթացների միջոցով
- **Plugin կառուցման գործիքներ** — նախակառուցել և կրկնարկել plugin կոդի վրա
- **Git snapshot գործիքներ** — ստեղծել snapshot-ներ նախքան ռիսկային մշակողական գործողությունները
- **Օգտատերերի կառավարման գործիքներ** — օգնել վերանայել կամ կարգավորել օգտատերերի գրառումները, որտեղ թույլատրված է
- **Benchmark գործիքներ** — գործարկել արտադրողականության կամ կարողությունների benchmark-ներ մշակման վերանայման համար

Մի փաստագրեք դրանք որպես բոլոր տեղադրումների համար ընդհանուր հասանելի Theme Builder գործիքներ։ Դրանք հասանելի են միայն այն դեպքում, երբ Advanced ուղեկից փաթեթը տեղադրված է, միացված և սահմանափակված վստահելի ադմինիստրատորներով։

## Գործիքների կազմաձևում {#tool-configuration}

### Գործիքների միացում {#enabling-tools}

Theme Builder-ում լրացուցիչ գործիքներ միացնելու համար․

1. Անցեք **Gratis AI Agent → Settings**
2. Գնացեք **Theme Builder → Tools**
3. Միացրեք կամ անջատեք գործիքները ըստ անհրաժեշտության
4. Պահպանեք կարգավորումները

### Գործիքների թույլտվություններ {#tool-permissions}

Որոշ գործիքներ պահանջում են թույլտվություններ․

- **Կայքի հավաքում** — պահանջում է ինտերնետ հասանելիություն
- **Վերլուծություններ** — պահանջում է Google Analytics ինտեգրում
- **Բովանդակության գեներացում** — պահանջում է API բանալիներ
- **Ընդլայնված հատկություններ** — կարող են պահանջել premium subscription
- **Advanced ուղեկից գործիքներ** — պահանջում են առանձին տարածվող Advanced փաթեթը և ադմինիստրատորի բացահայտ վստահությունը

### API բանալիներ և հավատարմագրեր {#api-keys-and-credentials}

Կազմաձևեք API բանալիները այն գործիքների համար, որոնք պահանջում են դրանք․

1. Գնացեք **Gratis AI Agent → Settings → API Keys**
2. Մուտքագրեք հավատարմագրերը յուրաքանչյուր գործիքի համար
3. Փորձարկեք կապը
4. Պահպանեք անվտանգ կերպով

### Advanced ուղեկից փաթեթի տեղադրում {#installing-the-advanced-companion-package}

Advanced ուղեկից փաթեթը թողարկվում է առանձին՝ WordPress.org հիմնական ZIP-ից։ Տեղադրեք այն միայն թողարկման պաշտոնական նախագծային տարածման ալիքից, ապա սահմանափակեք հասանելիությունը վստահելի ադմինիստրատորներով։ Վերանայեք մարդու հաստատման դարպասները և git snapshot-ի վարքագիծը՝ նախքան արտադրական կայքում ֆայլային համակարգի, տվյալների բազայի, WP-CLI, REST dispatcher-ի, plugin կառուցման, օգտատերերի կառավարման կամ benchmark գործիքները միացնելը։

## sd-ai-agent/site-scrape-ի օգտագործում {#using-sd-ai-agentsite-scrape}

### Հիմնական օգտագործում {#basic-usage}

Խնդրեք Theme Builder-ին վերլուծել կայք․

```
Վերլուծիր իմ մրցակցի կայքի դիզայնը competitor.com հասցեում
և առաջարկիր դիզայնի բարելավումներ իմ կայքի համար։
```

### Հատուկ վերլուծություն {#specific-analysis}

Հարցրեք վերլուծության հատուկ տեսակներ․

```
Դուրս բեր example.com-ի գունապնակը և օգտագործիր այն որպես ներշնչանք։
```

```
Վերլուծիր example.com-ի նավիգացիոն կառուցվածքը և կիրառիր նմանատիպ
կազմակերպում իմ կայքում։
```

```
Ստուգիր example.com-ի բջջային հարմարվողականությունը և ապահովիր, որ իմ կայքը
նույնքան հարմարվողական լինի։
```

### Համեմատական վերլուծություն {#comparison-analysis}

Համեմատեք մի քանի կայքեր․

```
Համեմատիր site1.com-ի և site2.com-ի դիզայնները և ստեղծիր դիզայն,
որը համադրում է երկուսի լավագույն տարրերը։
```

## Գործիքների սահմանափակումներ և նկատառումներ {#tool-limitations-and-considerations}

### Հարցումների սահմանափակում {#rate-limiting}

- Scraping-ը սահմանափակված է հաճախականությամբ՝ սերվերի գերբեռնվածությունը կանխելու համար
- Նույն կայքին ուղղված բազմակի հարցումները կարող են դանդաղեցվել
- Մեծ կայքերի վերլուծությունը կարող է ավելի երկար տևել

### Բովանդակության ճշգրտություն {#content-accuracy}

- Դինամիկ բովանդակությունը կարող է ամբողջությամբ չֆիքսվել
- JavaScript-ով արտապատկերված բովանդակությունը կարող է թերի լինել
- Իրական ժամանակի տվյալները կարող են չարտացոլվել
- Որոշ բովանդակություն կարող է գտնվել վճարային պատերի հետևում

### Իրավական և էթիկական նկատառումներ {#legal-and-ethical-considerations}

- Հարգեք robots.txt-ը և կայքի քաղաքականությունները
- Մի քերեք հեղինակային իրավունքով պաշտպանված բովանդակությունը վերարտադրության համար
- Օգտագործեք վերլուծությունը ոգեշնչման, ոչ թե պատճենման համար
- Ստուգեք, որ իրավունքներ ունեք օգտագործելու հանված բովանդակությունը
- Հետևեք վերլուծվող կայքերի ծառայության պայմաններին

### Արտադրողականության ազդեցություն {#performance-impact}

- Մեծ կայքերի scraping-ը կարող է ժամանակ պահանջել
- Միաժամանակյա բազմակի scraping-ները կարող են ավելի դանդաղ լինել
- Ցանցային կապը ազդում է արագության վրա
- Մեծ տվյալների հավաքածուները կարող են ավելի շատ մշակում պահանջել

## Լավագույն պրակտիկաներ {#best-practices}

### Կայքի վերլուծության օգտագործում {#using-site-analysis}

1. **Վերլուծեք մի քանի կայք** — հավաքեք ոգեշնչում մի քանի աղբյուրներից
2. **Կենտրոնացեք կառուցվածքի վրա** — սովորեք դասավորության և կազմակերպման օրինաչափությունները
3. **Հանեք գույները** — օգտագործեք գունապնակները որպես մեկնարկային կետեր
4. **Ուսումնասիրեք տպագրությունը** — նույնականացրեք ձեզ դուր եկող տառատեսակների համադրությունները
5. **Վերանայեք նավիգացիան** — հասկացեք ընտրացանկերի կազմակերպումը

### Էթիկական scraping {#ethical-scraping}

1. **Հարգեք robots.txt-ը** — հետևեք կայքի ուղեցույցներին
2. **Մի պատճենեք բովանդակությունը** — օգտագործեք վերլուծությունը միայն ոգեշնչման համար
3. **Ստուգեք իրավունքները** — համոզվեք, որ կարող եք օգտագործել հանված բովանդակությունը
4. **Նշեք աղբյուրները** — հղում տվեք ոգեշնչման աղբյուրներին
5. **Հետևեք պայմաններին** — պահպանեք կայքի ծառայության պայմանները

### Գործիքի արդյունավետության առավելագույնացում {#maximizing-tool-effectiveness}

1. **Եղեք կոնկրետ** — պահանջեք վերլուծության կոնկրետ տեսակներ
2. **Տրամադրեք համատեքստ** — բացատրեք ձեր կայքի նպատակը
3. **Սահմանեք ակնկալիքները** — նկարագրեք ձեր դիզայնի նպատակները
4. **Կրկնեք** — կատարելագործեք արդյունքների հիման վրա
5. **Համադրեք գործիքները** — օգտագործեք մի քանի գործիք համապարփակ վերլուծության համար

## Խնդիրների լուծում {#troubleshooting}

### Կայքը չի scraping-վում {#site-wont-scrape}

- Ստուգեք՝ արդյոք կայքը հանրայնորեն հասանելի է
- Ստուգեք, որ robots.txt-ը թույլ է տալիս scraping
- Փորձեք այլ կայք
- Ստուգեք ինտերնետ կապը
- Կապվեք աջակցության հետ, եթե խնդիրը շարունակվում է

### Թերի վերլուծություն {#incomplete-analysis}

- Կայքը կարող է ունենալ դինամիկ բովանդակություն
- JavaScript-ը կարող է ամբողջությամբ չարտապատկերվել
- Մեծ կայքերում կարող է ժամանակի սպառում լինել
- Փոխարենը փորձեք վերլուծել կոնկրետ էջեր
- Պահանջեք վերլուծության կոնկրետ տեսակներ

### Դանդաղ արտադրողականություն {#slow-performance}

- Մեծ կայքերի վերլուծությունը ավելի երկար է տևում
- Միաժամանակյա բազմակի scraping-ներն ավելի դանդաղ են
- Ցանցային կապը ազդում է արագության վրա
- Փորձեք ոչ պիկ ժամերին
- Սկզբում վերլուծեք ավելի փոքր բաժիններ
