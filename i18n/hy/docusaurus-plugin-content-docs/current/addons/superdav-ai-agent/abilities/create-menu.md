# Ստեղծել մենյու

**Ստեղծել մենյու** կարողությունը ստեղծում է WordPress նավիգացիոն մենյուներ՝ էջերի վերնագրերից առանձին նավիգացիոն պիտակների աջակցությամբ։

## Ընդհանուր ակնարկ

Այս կարողությունն ընդլայնում է ստանդարտ մենյու ստեղծելու գործառույթը՝ հնարավորություն տալով նշել `navigation_label` պարամետրը։ Սա թույլ է տալիս ստեղծել մենյուներ, որտեղ նավիգացիայում ցուցադրվող պիտակը տարբերվում է էջի վերնագրից՝ ապահովելով ավելի մեծ ճկունություն կայքի կառուցվածքի և օգտատիրոջ փորձառության մեջ։

## Պարամետրեր

| Պարամետր | Տեսակ | Պարտադիր | Նկարագրություն |
|-----------|------|----------|-------------|
| `name` | string | Այո | Մենյուի անուն, օրինակ՝ `Primary Navigation` |
| `location` | string | Ոչ | Theme-ի տեղակայման վայր, որին պետք է կցել այս մենյուն, օրինակ՝ `primary` |
| `navigation_label` | string | Ոչ | Նավիգացիայում ցուցադրվող պիտակ (էջի վերնագրից տարբերվող) |

## Վերադարձվող արժեք

```json
{
  "success": true,
  "menu_id": 7,
  "name": "Primary Navigation",
  "navigation_label": "Main Menu"
}
```

## Նավիգացիոն պիտակն ընդդեմ էջի վերնագրի

`navigation_label` պարամետրը թույլ է տալիս առանձնացնել ներքին մենյուի անունը օգտատերերին նավիգացիայում ցուցադրվող պիտակից՝

- **`name`** — WordPress-ի կողմից օգտագործվող ներքին մենյուի նույնացուցիչ (օրինակ՝ «Primary Navigation»)
- **`navigation_label`** — Կայքի այցելուներին նավիգացիայում ցուցադրվող պիտակը (օրինակ՝ «Main Menu»)

Սա օգտակար է, երբ՝
- Ձեր ներքին անվանակոչման պայմանավորվածությունը տարբերվում է օգտատերերին տեսանելի պիտակներից
- Դուք ցանկանում եք նավիգացիայում ունենալ ավելի կարճ պիտակներ, քան admin վահանակում
- Ձեզ անհրաժեշտ է աջակցել մի քանի լեզուների՝ պիտակների տարբեր երկարություններով
- Դուք ստեղծում եք մենյուներ կոնկրետ տարածաշրջանների կամ օգտատերերի խմբերի համար

## Օգտագործման օրինակներ

### Օրինակ 1․ Պարզ մենյու՝ նավիգացիոն պիտակով

**Հուշում՝**
```
Ստեղծեք «Main Navigation» անունով հիմնական նավիգացիոն մենյու՝ «Menu» նավիգացիոն պիտակով։
```

**Արդյունք՝**
```json
{
  "success": true,
  "menu_id": 7,
  "name": "Main Navigation",
  "navigation_label": "Menu"
}
```

### Օրինակ 2․ Մենյու կոնկրետ Theme-ի տեղակայման վայրի համար

**Հուշում՝**
```
Ստեղծեք «Footer Links» անունով footer մենյու՝ «Quick Links» նավիգացիոն պիտակով և կցեք այն footer տեղակայման վայրին։
```

**Արդյունք՝**
```json
{
  "success": true,
  "menu_id": 8,
  "name": "Footer Links",
  "navigation_label": "Quick Links",
  "location": "footer"
}
```

## Ինտեգրում Theme Builder-ի հետ

Theme Builder օգտագործելիս «Ստեղծել մենյու» կարողությունը՝

1. Ավտոմատ հայտնաբերում է հասանելի theme մենյուի տեղակայման վայրերը
2. Ստեղծում է մենյուներ՝ ձեր դիզայնին համապատասխան նավիգացիոն պիտակներով
3. Կցում է մենյուները ճիշտ theme տեղակայման վայրերին
4. Աջակցում է մենյուի տարրերի ավելացմանը ստեղծումից հետո

## Առնչվող կարողություններ

- **`add_menu_item`** — Ավելացնել տարրեր գոյություն ունեցող մենյուում
- **`update_menu`** — Վերանվանել մենյուն կամ վերակցել այն theme-ի տեղակայման վայրին
- **`delete_menu`** — Հեռացնել մենյուն ձեր կայքից

## Լավագույն փորձառություններ

- **Օգտագործեք հստակ նավիգացիոն պիտակներ** — պահեք պիտակները կարճ և նկարագրական օգտատերերի համար
- **Համապատասխանեցրեք theme-ի տեղակայման վայրերը** — կցեք մենյուները ճիշտ theme տեղակայման վայրին՝ պատշաճ ցուցադրման համար
- **Պլանավորեք մենյուի կառուցվածքը** — որոշեք ձեր մենյուի հիերարխիան՝ նախքան տարրեր ստեղծելը
- **Փորձարկեք արձագանքունակությունը** — ստուգեք, որ մենյուները ճիշտ են ցուցադրվում շարժական սարքերում
- **Տեղայնացրեք պիտակները** — օգտագործեք տարբեր նավիգացիոն պիտակներ տարբեր լեզվական տարբերակների համար

## Սահմանափակումներ

- Նավիգացիոն պիտակները միայն ցուցադրման համար են․ ներքին `name`-ը դեռ օգտագործվում է WordPress նույնականացման համար
- Theme-ի աջակցությունը տարբերվում է․ ոչ բոլոր theme-երն են աջակցում մենյուի բոլոր տեղակայման վայրերը
- Մենյուի տարրերը պետք է ավելացվեն առանձին՝ մենյուի ստեղծումից հետո
- Նավիգացիոն պիտակը փոխելու համար անհրաժեշտ է թարմացնել մենյուն
