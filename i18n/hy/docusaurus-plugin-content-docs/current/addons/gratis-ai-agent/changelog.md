# Փոփոխությունների մատյան

## 1.9.0 — Թողարկվել է 2026-04-28-ին

### Նոր

- **`create_contact_form` կարողություն** — ստեղծում է կապի ձև՝ օգտագործելով ակտիվ ձևի plugin-ը (Contact Form 7, WPForms, Fluent Forms կամ Gravity Forms), և վերադարձնում է shortcode, որը պատրաստ է ներդրվելու ցանկացած գրառման կամ էջի մեջ։
- **`set_featured_image` կարողություն** — գրառմանը կամ էջին նշանակում է featured image՝ գոյություն ունեցող Media Library հավելվածի ID-ից կամ հեռակա URL-ից․ URL տրամադրելու դեպքում պատկերը ինքնաբար ներմուծվում է։
- **`batch_create_posts` կարողություն** — ստեղծում է մի քանի գրառում մեկ կարողության կանչով։ Աջակցում է նույն պարամետրերին, ինչ `create_post`-ը, հաղորդում է յուրաքանչյուր գրառման հաջողությունը/ձախողումը և ունի ընտրովի `stop_on_error` ռեժիմ։
- **`page_template` պարամետր** — ավելացվել է `create_post`-ին և `update_post`-ին։ Ստեղծման կամ թարմացման պահին նշանակում է PHP էջի ձևանմուշի ֆայլ (օր.՝ `page-full-width.php`)։ `update_post`-ին փոխանցեք դատարկ տող՝ theme-ի լռելյայնին վերադառնալու համար։
- **Կլիենտային screenshot կարողություններ** — `capture_screenshot`, `compare_screenshots` և `review_page_design`։ Ստացեք կենդանի էջերի ամբողջական կամ viewport screenshot-ներ server-side headless browser-ի միջոցով, համեմատեք երկու screenshot և ստացեք AI-ի ստեղծած դիզայնի վերանայում՝ ընդգրկելով դասավորությունը, տպագրությունը, գույնը և մատչելիությունը։
- **Հինգ ներկառուցված agent** — Content Writer, Site Builder, Design Studio, Plugin Manager և Support Assistant։ Յուրաքանչյուր agent ունի գործիքների հատուկ հավաքածու, հարմարեցված համակարգային հուշում և սկզբնական առաջարկներ։ Փոխարկելի է չատի վերնամասում գտնվող նոր **Agent Picker**-ի միջոցով։ Տես [Ներկառուցված agent-ներ](../../user-guide/configuration/built-in-agents)։
- **Feature flags** — նոր **Settings → Feature Flags** ներդիր՝ հասանելիության վերահսկման փոխարկիչներով (սահմանափակել ադմինիստրատորներով, սահմանափակել ցանցային ադմիններով, բաժանորդի հասանելիություն, անջատել ոչ անդամների համար) և բրենդավորման տարբերակներով (թաքցնել footer attribution-ը, անհատական agent-ի անուն, թաքցնել agent picker-ը, օգտագործել site-ի պատկերակը որպես chat avatar)։ Տես [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings)։
- **Վերականգնել վերջին աշխատաշրջանը** — չատի վահանակն այժմ էջի բեռնման և widget-ի բացման ժամանակ ինքնաբար վերաբեռնում է ամենավերջին խոսակցությունը, որպեսզի համատեքստը երբեք չկորչի էջերի միջև նավարկության ընթացքում։
- **Plugin-ի գործողության հղումներ** — Settings-ի և Abilities Registry-ի արագ հղումներն այժմ հայտնվում են WordPress-ի **Plugins → Installed Plugins** էկրանին՝ plugin-ի նկարագրության ներքևում։

### Բարելավված

- **Պատկերի աղբյուրի կրկնափորձ** — agent-ն այժմ կրկին փորձում է բոլոր կազմաձևված անվճար պատկերի աղբյուրները՝ նախքան ներբեռնման ձախողման դեպքում AI-ի ստեղծած պատկերին անցնելը։
- **Մոդելի տեղեկությունների վահանակ** — միշտ տեսանելի է չատի վերնամասում․ այլևս չի թաքցվում առաջին հաղորդագրությունից հետո։
- **Ինքնաբար ոլորման վարքագիծ** — ինքնաբար ոլորումը դադարեցվում է, երբ օգտատերը վերև է ոլորում կարդալու համար․ հայտնվում է լողացող **Ոլորել դեպի ներքև** կոճակ, որը ինքնաբար անհետանում է, երբ օգտատերը հասնում է վերջին հաղորդագրությանը։
- **Agent Picker UI** — վերադիզայնավորվել է որպես form-table ծածկույթ՝ յուրաքանչյուր agent-ի պատկերակներով, ինչը հեշտացնում է agent-ների նույնականացումն ու փոխարկումը մեկ հայացքով։
- **Lazy-loaded JS chunks** — չատի widget-ի սկզբնական JavaScript փաթեթն այժմ բաժանված է lazy-loaded մասերի՝ նվազեցնելով սկզբնական փաթեթի չափերը 75–90%-ով։
- **Չատի widget-ի վերադիզայն** — միասնական AI պատկերակը փոխարինում է նախորդ անհատական avatar-ին․ համահունչ է ներկառուցված agent համակարգին։
- **URL հղումների ստեղծում** — համակարգային հաղորդագրություններում և սխալի հաղորդագրության պղպջակներում հայտնվող URL-ներն այժմ ցուցադրվում են որպես սեղմվող հղումներ։

### Ուղղված

- **Կարողությունների հայտնաբերելիություն** — ուղղվել են նկարագրությունները, համակարգային հուշման հղումները և namespace-ի համապատասխանությունը, որպեսզի բոլոր կարողությունները հուսալիորեն հայտնվեն agent-ի գործիքների ցանկում։
- **Providers cache** — providers-ն այժմ cache է արվում ամբողջ site-ի համար տարբերակի հաշվիչի միջոցով՝ կանխելով հնացած provider-ների խնդիրները multisite ցանցերում։
- **`ability_invalid_output`** — լուծվել է 12 կարողության handler-ներում․ բոլորը վերադարձնում են պատշաճ կառուցվածքով JSON պատասխաններ։
- **`pending_client_tool_calls` pipeline** — միացվել է սկզբից մինչև վերջ, որպեսզի կլիենտային գործիքների կանչերը ճիշտ ավարտվեն և արդյունքները վերադարձնեն մոդելին։
- **Պատմության drawer** — հետ չվերադարձվող փոփոխությունները բացառված են վերադարձման ցանկից․ **Դիտել ամբողջ պատմությունը** հղումն այժմ ճիշտ է աշխատում։
- **Փոփոխությունների/վերադարձման համակարգ** — հինգ առանձին սխալ ուղղվել և միավորվել են նոր ադմին ինտերֆեյսի ներքո։
- **Save Settings toast** — snackbar ծանուցումն այժմ հուսալիորեն հայտնվում է **Save Settings** սեղմելուց հետո։
- **Trash համատեքստային մենյու** — ավելացվել է **Ջնջել ընդմիշտ** տարբերակը, որպեսզի տարրերը հնարավոր լինի կոշտ ջնջել՝ առանց աղբարկղի տեսքից դուրս գալու։
- **Խմբագրել և կրկին ուղարկել** — խմբագրման պատկերակին սեղմելն այժմ խմբագրման ռեժիմ է մտնում միայն սեղմված հաղորդագրության համար, ոչ թե շղթայի բոլոր հաղորդագրությունների։
- **Չատի դասավորության բարձրություն** — չատի վահանակը հարմարեցնում է իր բարձրությունը, երբ plugin-ի ներարկած բովանդակություն (ադմին ծանուցումներ, բաներներ) հայտնվում է էջի վերևում՝ կանխելով մուտքագրման հատվածի էկրանից դուրս մղվելը։

## 1.4.0 — Թողարկվել է 2026-04-09-ին

### Նոր

- **WP-CLI benchmark command** (`wp gratis-ai-agent benchmark`) — գործարկում է Agent Capabilities v1 benchmark փաթեթը հրամանային տողից՝ CI հոսքագծերի և մոդելի գնահատման աշխատանքային հոսքերի համար։ Աջակցում է առանձին հարցերի գործարկումներ, provider/model վերագրումներ և JSON/CSV ելք։
- **Agent Capabilities v1 benchmark suite** — բարդ, բազմաքայլ հուշումների կառուցվածքային հավաքածու, որը փորձարկում է կարողությունների ամբողջ մակերեսը՝ գնահատման և token/տևողության հաշվետվությամբ։
- **Custom Post Type կարողություններ** — `register_post_type`, `list_post_types`, `delete_post_type`։ Գրանցումները պահպանվում են վերագործարկումների միջև options աղյուսակի միջոցով։
- **Custom Taxonomy կարողություններ** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`։ Աջակցում է հիերարխիկ և հարթ taxonomy-ներ՝ ըստ ցանկության rewrite slug-երով։
- **Design System կարողություններ** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`։ Ներառում է հինգ ընտրված preset՝ `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`։
- **Global Styles կարողություններ** — `get_global_styles`, `set_global_styles`, `reset_global_styles`։ Կարդում և գրում է theme.json արժեքները WordPress Global Styles API-ի միջոցով։
- **Navigation Menu կառավարման կարողություն** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`։ Աջակցում է ներդրված մենյուներ և theme location-ի վերագրում։
- **Options Management կարողություն** — `get_option`, `set_option`, `delete_option`, `list_options`։ Ներառում է ներկառուցված անվտանգության արգելացուցակ, որը պաշտպանում է կարևոր WordPress options-ը փոփոխումից։
- **Installable Abilities Registry** — `list_available_abilities`, `install_ability`, `recommend_plugin`։ Հայտնաբերեք և ակտիվացրեք կարողությունների փաթեթներ, որոնք տարածվում են որպես WordPress plugin-ներ։
- **Site Builder Orchestration v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`։ Բազմափուլ site կառուցման պլաններ՝ plugin հայտնաբերմամբ, քայլերի ելքի հղումներով, առաջընթացի հետևմամբ և սխալների ինքնավար վերականգնմամբ։
- **Ռեստորանի կայքի benchmark հարց** (`q-restaurant-website`) — ամբողջական թեստ, որը ներառում է CPT գրանցում, design system, navigation և plugin հայտնաբերում։
- **AI provider connector plugin-ներ** ավելացվել են WordPress Playground blueprint-ներում՝ տեղային զարգացման ավելի արագ կարգավորման համար։

### Բարելավված
