# Բազմավարձակալություն

Բաշխեք ենթակայքերը սերվերների և տվյալների բազաների միջև՝ արդյունավետության և անվտանգության համար

## 1.2.0 տարբերակի թողարկման հիմնական կետերը {#version-120-release-highlights}

Multi-Tenancy 1.2.0-ը ավելացնում է սուվերեն վարձակալի հիմքը արտադրական մեկուսացման աշխատանքային հոսքերի համար՝

- Վիճակ չպահող վարձակալի ավտոմուտք՝ կարճաժամկետ, կրկնությունից պաշտպանված SSO այցերով։
- Յուրաքանչյուր ենթակայքի համար տվյալների բազայի և ֆայլային համակարգի մեկուսացում սուվերեն վարձակալների համար։
- Hostinger բազմավարձակալության հնարավորություն՝ հոսթը հաշվի առնող վարձակալի տրամադրման համար։
- Սուվերեն միգրացիայի ստուգման դարպասներ՝ հին տվյալների ստուգումների և ասինխրոն հրումային մշակման համար։
- Կարգավորվող նույն մեքենայի տվյալների բազայի հոսթի կապում Bedrock, FrankenPHP և localhost տեղադրումների համար։
- Վարձակալի սկզբնաբեռնումի, տվյալների բազայի գրողի, սխեմայի սկզբնաբեռնումի և օգտատերերի տրամադրման բարելավումներ։
- Ավելի անվտանգ սուվերեն հեռացում, որը մաքրում է վարձակալի հավատարմագրերը։

Տես օգտատիրոջ ուղեցույցները՝ [սուվերեն միգրացիայի աշխատանքային հոսքի](/user-guide/getting-started/sovereign-migration-workflow), [բազմավարձակալության մեկուսացման](/user-guide/configuration/multi-tenancy-isolation) և [սուվերեն վարձակալի գործողությունների](/user-guide/administration/sovereign-tenant-operations) համար։

## Պահանջներ {#requirements}

- WordPress 5.3 կամ ավելի բարձր
- PHP 7.4 կամ ավելի բարձր
- Multisite Ultimate plugin (ակտիվ)

## Տեղադրում {#installation}

1. Վերբեռնեք հավելման ֆայլերը ձեր `/wp-content/plugins/` պանակում
2. Ակտիվացրեք plugin-ը WordPress-ի «Plugins» ընտրացանկի միջոցով
3. Կարգավորեք հավելման կարգավորումները ցանցի ադմինիստրատորի բաժնում

## Մշակում {#development}

### Կարգավորում {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Նախագծի կառուցվածք {#project-structure}

```
ultimate-multisite-addon-template/

│   ├── class-multisite-ultimate-updater.php  # Update handler
│   └── class-multi_tenancy-main.php             # Main functionality

│   ├── class-*-base.php         # Base test class
│   └── class-*-test.php         # Test cases

├── assets/                      # Static assets
│   ├── css/                     # Stylesheets
│   ├── js/                      # JavaScript files

├── lang/                        # Language files

├── .phpcs.xml.dist              # Code standards configuration

```

### Փորձարկում {#testing}

Հավելումը ներառում է համապարփակ փորձարկման շրջանակ՝

- **Միավորային փորձարկումներ**․ PHPUnit-ի վրա հիմնված փորձարկումներ WordPress փորձարկման հավաքածուով
- **Հիմնական փորձարկման դաս**․ տրամադրում է օգնական մեթոդներ ընդհանուր փորձարկման սցենարների համար
- **Փորձարկման ծածկույթ**․ կարգավորված է կոդի ծածկույթը հետևելու համար
- **Multisite փորձարկում**․ փորձարկումները գործարկվում են multisite միջավայրում

### Կոդի չափանիշներ {#code-standards}

- **WordPress կոդավորման չափանիշներ**․ պարտադրվում են PHPCS-ի միջոցով
- **PHP 7.4+ համատեղելիություն**․ աջակցվում են ժամանակակից PHP հնարավորությունները
- **Ստատիկ վերլուծություն**․ PHPStan ինտեգրում՝ տիպերի ստուգման համար
- **Կոդի արդիականացում**․ Rector կանոններ PHP թարմացումների համար

### Կառուցման համակարգ {#build-system}

- **Ռեսուրսների մշակում**․ CSS/JS ֆայլերի փոքրացում
- **Թարգմանություն**․ POT ֆայլի ստեղծում
- **Փաթեթավորում**․ արխիվի ստեղծում տարածման համար
- **Մշակման ռեժիմ**․ չփոքրացված ռեսուրսներ վրիպազերծման համար

## Կարգավորում {#configuration}

Հավելումը աջակցում է տարբեր կարգավորման տարբերակներ, որոնք հասանելի են ցանցի ադմինիստրատորի միջերեսի միջոցով։

### Սուվերեն վարձակալի կարգավորում {#sovereign-tenant-configuration}

Սուվերեն վարձակալների համար կարգավորեք վարձակալի տվյալների բազան, ֆայլային համակարգի արմատը, տվյալների բազայի հոսթի կապումը և հոսթի հնարավորությունը՝ նախքան երթևեկությունը վարձակալին ուղղորդելը։ Գործարկեք միգրացիայի ստուգումը յուրաքանչյուր փոփոխությունից հետո, որը ազդում է տվյալների բազայի հավատարմագրերի, հոսթի կապման, երթուղավորման կամ օգտատերերի տրամադրման վրա։

Օգտագործեք `localhost` միայն այն դեպքում, երբ վարձակալի գործարկման միջավայրը և տվյալների բազայի թույլտվությունները ակնկալում են տեղական սոկետ կամ նույն մեքենայի հասանելիություն։ Օգտագործեք `127.0.0.1` կամ ծառայության հոսթի անուն, երբ դա ձեր մատակարարի կողմից կարգավորված թույլտվության հոսթն է։

### Hostinger հնարավորություն {#hostinger-capability}

Hostinger բազմավարձակալության հնարավորությունն աշխատում է ընդհանուր Hostinger ինտեգրման նշանով։ Հաստատեք, որ տիրույթը առկա է hPanel-ում, DNS-ը մատնանշում է Hostinger Account-ին, և վարձակալի տվյալների բազայի կարգավորումները համապատասխանում են hPanel-ին՝ նախքան ստուգումը։

## Կցիչներ և զտիչներ {#hooks-and-filters}

### Գործողություններ {#actions}

- `ultimate-multisite-addon-template_init` - Գործարկվում է հավելման սկզբնավորումից հետո
- `ultimate-multisite-addon-template_loaded` - Գործարկվում է հավելման բոլոր ֆայլերը բեռնելուց հետո

### Զտիչներ {#filters}

- `ultimate-multisite-addon-template_settings` - Փոփոխել հավելման կարգավորումները
- `ultimate-multisite-addon-template_enabled` - Վերասահմանել հավելման միացման/անջատման կարգավիճակը
