# Tutor Multisite Compatibility

Un plugin WordPress care adaugă soluții de compatibilitate pentru Tutor LMS în mediile multisite.

## Descriere {#description}

Acest plugin oferă soluții esențiale de compatibilitate pentru rularea Tutor LMS pe rețelele WordPress multisite. El gestionează înscrierea utilizatorilor, administrarea instructorilor, limitările de cursuri și alte funcționalități specifice mediilor multisite.

## Funcționalități {#features}

- **Managementul Tabelelor de Bază de Date**: Detectează și creează automat tabelele lipsă ale Tutor LMS care eșuează din cauza restricțiilor de cheie străină (foreign key constraints) specifice mediilor multisite.
- **Înscrierea Utilizatorilor Multisite**: Handler-e AJAX personalizate pentru gestionarea înscrierii utilizatorilor pe site-urile din rețea.
- **Managementul Rolului de Instructor**: Atribuirea și gestionarea automată a rolului de instructor în mediile multisite.
- **Limitări de Cursuri**: Integrare cu Ultimate Multisite pentru gestionarea limitelor de creare a cursurilor și a funnel-urilor, bazate pe planurile de site.
- **Vederi Admin Personalizate**: Interfață de listă și management a instructorilor, conștientă de structura multisite.
- **Instrumente Admin**: Interfață de management al bazei de date pentru administratorii de rețea și de site.

## Cerințe {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Plugin Tutor LMS (aplicat automat prin antetul `Requires Plugins`)
- Instalare WordPress Multisite
- Plugin Ultimate Multisite (pentru funcționalitatea limitărilor de cursuri)

## Instalare {#installation}

1. Urcați fișierele plugin-ului în directorul `/wp-content/plugins/tutor-multisite-compatibility/`
2. Asigurați-vă că Tutor LMS este instalat și activat
3. Activați plugin-ul prin meniul 'Plugins' din WordPress
4. Plugin-ul va integra automat cu Tutor LMS

## Managementul Bazei de Date {#database-management}

### Problema {#the-problem}

În mediile WordPress multisite, crearea tabelelor Tutor LMS eșuează pentru tabelele care au restricții de cheie străină către tabela de utilizatori. Acest lucru se întâmplă deoarece:

- Tabelele specifice site-ului folosesc prefixe precum `wp_2_tutor_carts`
- Cheile străine fac referire la `wp_2_users` (care nu există)
- Tabela reală de utilizatori este `wp_users` (care este partajată pe întreaga rețea)

### Soluția {#the-solution}

Acest plugin face automat următoarele:

1. **Detectează Tabele Lipsă**: Verifică tabelele care nu au reușit să fie create din cauza restricțiilor de cheie străină.
2. **Creează Tabele cu Referințe Corecte**: Utilizează numele corect al tabelei de utilizatori (`wp_users` în multisite).
3. **Gestionează Multiple Scenarii**:
   - Activarea/reactivarea plugin-ului
   - Crearea unui site nou în rețea
   - Verificarea manuală a tabelelor prin interfața admin

### Tabelele Afectate {#affected-tables}

- `tutor_coupon_usages` - Conține o cheie străină către tabela de utilizatori
- `tutor_carts` - Conține o cheie străină către tabela de utilizatori
- `tutor_notification_preferences` - Tabela TutorPro cu o cheie străină către tabela de utilizatori

### Interfața Admin {#admin-interface}

Accesați instrumentele de management al bazei de date la:

- **Admin Rețea**: Settings → Tutor Multisite DB
- **Admin Site**: Tools → Tutor Multisite DB

Interfața vă permite să:
- Vizualizați tabelele lipsă pe toate site-urile
- A activa manual crearea tabelelor
- Verificați configurația bazei de date
- Vizualizați informații detaliate de stare
