# Tenencia múltiple

Distribuye subsitios entre servidores y bases de datos para mejorar el rendimiento y la seguridad

## Aspectos destacados del lanzamiento de la versión 1.2.0

Multi-Tenancy 1.2.0 añade la base de inquilinos soberanos para flujos de trabajo de aislamiento en producción:

- Inicio de sesión automático de inquilinos sin estado con visitas SSO de corta duración y protegidas contra repetición.
- Aislamiento de base de datos y sistema de archivos por subsitio para inquilinos soberanos.
- Capacidad de tenencia múltiple de Hostinger para el aprovisionamiento de inquilinos consciente del host.
- Puertas de verificación de migración soberana para comprobaciones de datos heredados y procesamiento push asíncrono.
- Enlace configurable del host de base de datos en la misma máquina para instalaciones Bedrock, FrankenPHP y localhost.
- Mejoras en el arranque del inquilino, el escritor de base de datos, el arranque del esquema y el aprovisionamiento de usuarios.
- Eliminación soberana más segura que limpia las credenciales del inquilino.

Consulta las guías de usuario para el [flujo de trabajo de migración soberana](/user-guide/getting-started/sovereign-migration-workflow), el [aislamiento de tenencia múltiple](/user-guide/configuration/multi-tenancy-isolation) y las [operaciones de inquilinos soberanos](/user-guide/administration/sovereign-tenant-operations).

## Requisitos

- WordPress 5.3 o superior
- PHP 7.4 o superior
- plugin Multisite Ultimate (activo)

## Instalación

1. Sube los archivos del complemento a tu directorio `/wp-content/plugins/`
2. Activa el plugin a través del menú 'Plugins' en WordPress
3. Configura los ajustes del complemento en la administración de red

### Pruebas

El complemento incluye un marco de pruebas completo:

- **Pruebas unitarias**: Pruebas basadas en PHPUnit con el conjunto de pruebas de WordPress
- **Clase base de prueba**: Proporciona métodos auxiliares para escenarios de prueba comunes
- **Cobertura de pruebas**: Configurada para hacer seguimiento de la cobertura de código
- **Pruebas multisite**: Las pruebas se ejecutan en un entorno multisite

### Estándares de código

- **WordPress Coding Standards**: Aplicados mediante PHPCS
- **Compatibilidad con PHP 7.4+**: Compatible con funciones modernas de PHP
- **Análisis estático**: Integración de PHPStan para comprobación de tipos
- **Modernización de código**: Reglas de Rector para actualizaciones de PHP

### Sistema de compilación

- **Procesamiento de recursos**: Minificación de archivos CSS/JS
- **Traducción**: Generación de archivo POT
- **Empaquetado**: Creación de archivo para distribución
- **Modo de desarrollo**: Recursos sin minificar para depuración

## Configuración

El complemento admite varias opciones de configuración accesibles a través de la interfaz de administración de red.

### Configuración de inquilinos soberanos

Para inquilinos soberanos, configura la base de datos del inquilino, la raíz del sistema de archivos, el enlace del host de base de datos y la capacidad del host antes de enrutar tráfico al inquilino. Ejecuta la verificación de migración después de cada cambio que afecte a las credenciales de la base de datos, el enlace del host, el enrutamiento o el aprovisionamiento de usuarios.

Usa `localhost` solo cuando el entorno de ejecución del inquilino y las concesiones de la base de datos esperen acceso mediante socket local o en la misma máquina. Usa `127.0.0.1` o un nombre de host de servicio cuando ese sea el host de concesión configurado por tu proveedor.

### Capacidad de Hostinger

La capacidad de tenencia múltiple de Hostinger funciona con el token de integración compartido de Hostinger. Confirma que el dominio esté presente en hPanel, que DNS apunte a la cuenta de Hostinger y que los ajustes de la base de datos del inquilino coincidan con hPanel antes de la verificación.

## Hooks y filtros

### Acciones

- `ultimate-multisite-addon-template_init` - Se dispara después de la inicialización del complemento
- `ultimate-multisite-addon-template_loaded` - Se dispara después de que se carguen todos los archivos del complemento

### Filtros

- `ultimate-multisite-addon-template_settings` - Modifica los ajustes del complemento
- `ultimate-multisite-addon-template_enabled` - Sobrescribe el estado de activación/desactivación del complemento
