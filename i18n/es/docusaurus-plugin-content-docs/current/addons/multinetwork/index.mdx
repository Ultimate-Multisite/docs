# Multi-Network

El complemento Multinetwork de Ultimate Multisite amplía tu plataforma Ultimate Multisite con una potente funcionalidad multirred. Este complemento permite a tus clientes comprar y gestionar redes WordPress multisite completas, no solo sitios individuales.

## Características clave {#key-features}

* **Aprovisionamiento de redes**: Crea y configura automáticamente redes WordPress multisite completas para tus clientes
* **Plantillas de red**: Permite que los clientes exploren, previsualicen y elijan una plantilla de red prediseñada antes del registro
* **Gestión de redes**: Capacidades completas de gestión para entornos multirred
* **Redes aisladas**: Cada red de cliente funciona de forma independiente con sus propias tablas de base de datos y configuración
* **SSO entre redes**: Ofrece a los clientes acceso fluido a sus subredes con autenticación mediante enlace mágico
* **Arquitectura escalable**: Gestiona múltiples redes de forma eficiente con asignación de recursos optimizada
* **Integración**: Se integra perfectamente con los sistemas de suscripción y facturación de Ultimate Multisite

## Casos de uso {#use-cases}

* **Alojamiento de marca blanca**: Proporciona a los clientes sus propias redes multisite con marca personalizada
* **Soluciones para agencias**: Permite a las agencias gestionar múltiples redes de clientes desde una sola plataforma
* **Implementaciones empresariales**: Da soporte a grandes organizaciones con requisitos complejos de múltiples redes
* **Plataformas educativas**: Crea redes separadas para diferentes escuelas o departamentos

## Requisitos {#requirements}

* Ultimate Multisite 2.0.0 o superior
* Instalación WordPress Multisite
* PHP 7.4 o superior
* MySQL 5.6 o superior

## Instalación {#installation}

1. Sube los archivos del plugin al directorio `/wp-content/plugins/ultimate-multisite-multinetwork`
2. Activa el plugin mediante la pantalla 'Plugins' en WordPress
3. Configura los ajustes de multirred mediante Ultimate Multisite → Ajustes

## Plantillas de red {#network-templates}

La versión 1.3.0 añade un flujo de descubrimiento y selección de plantillas para los clientes que compran redes nuevas. Usa plantillas de red cuando quieras que cada nueva red de cliente comience a partir de una estructura preparada, un diseño, una biblioteca de contenido o una configuración específica para un sector.

### Previsualizador de plantillas {#template-previewer}

El previsualizador de plantillas de red permite a los clientes explorar las plantillas disponibles antes del checkout. Cada plantilla puede abrirse en un panel de previsualización en vivo, para que los clientes puedan comparar diseños y contenido antes de comprar una red.

Usa el previsualizador para ayudar a los clientes a:

* Revisar el aspecto y la estructura de cada plantilla de red disponible
* Comparar varias plantillas sin salir del flujo de compra
* Elegir la plantilla que mejor se adapte a su caso de uso antes del registro

Mantén los nombres y las descripciones de las plantillas fáciles de entender para los clientes. El previsualizador resulta más útil cuando los clientes pueden comprender el caso de uso previsto, como redes de agencias, redes escolares, redes de franquicias o paquetes de alojamiento de marca blanca.

### Selector de plantillas en el checkout {#template-picker-at-checkout}

El selector de plantillas puede añadirse al flujo de checkout de registro como un campo. Cuando está configurado, los clientes seleccionan durante el checkout la plantilla de red que desean, y la plantilla seleccionada se utiliza cuando se aprovisiona su nueva red.

Para usar el selector en un flujo de checkout:

1. Prepara las redes que quieras ofrecer como plantillas.
2. Activa el campo de selección de plantillas Multinetwork en el formulario de registro o checkout correspondiente.
3. Elige qué plantillas deben estar disponibles para ese flujo de checkout.
4. Publica el flujo de checkout y prueba un registro para confirmar que la plantilla seleccionada se aplica a la nueva red.

Si un flujo de checkout solo ofrece un producto de red, el selector aún puede ser útil como un paso de incorporación guiada. Para embudos con varios planes, muestra solo las plantillas que sean compatibles con el producto o plan seleccionado.

## SSO con enlace mágico entre redes {#cross-network-magic-link-sso}

El SSO con enlace mágico entre redes permite a los clientes moverse entre su área de Account principal y las subredes sin tener que iniciar sesión manualmente de nuevo. Cuando un cliente sigue un enlace de acceso entre redes, Ultimate Multisite genera un enlace mágico con límite de tiempo y autentica al cliente en la red de destino.

Esto mejora la experiencia de los clientes que gestionan más de una red o que necesitan acceder a redes de plantilla recién clonadas inmediatamente después de la compra.

Lista de comprobación del administrador:

* Confirma que el complemento Multinetwork esté activo en toda la instalación WordPress multisite.
* Asegúrate de que los clientes tengan el acceso de red correcto antes de ofrecerles enlaces entre redes.
* Mantén los dominios de red y los dominios mapeados correctamente configurados para que los enlaces mágicos se resuelvan en la red de destino prevista.
* Prueba el dashboard del cliente y cualquier botón de acceso posterior al checkout después de cambiar los ajustes de red, dominio o checkout.

Los enlaces mágicos deben tratarse como enlaces de inicio de sesión. Están diseñados para un acceso fluido del cliente y no deben reenviarse ni reutilizarse fuera de la sesión prevista.

## Preguntas frecuentes {#frequently-asked-questions}

### ¿Qué es multirred? {#what-is-multinetwork}

Multirred es una característica de WordPress que te permite ejecutar varias redes multisite independientes en una sola instalación WordPress.

### ¿En qué se diferencia de multisite normal? {#how-is-this-different-from-regular-multisite}

Multisite normal permite múltiples sitios dentro de una red. Multirred permite múltiples redes, cada una con sus propios sitios, todas dentro de una sola instalación WordPress.

### ¿Requiere esto una configuración especial del servidor? {#does-this-require-special-server-configuration}

Sí, multirred requiere una configuración y preparación específicas del servidor. Consulta la documentación para obtener instrucciones de configuración detalladas.
