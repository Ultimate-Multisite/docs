# Ultimate Multisite Requisitos

La primera cosa que podrías preguntarte antes de comprar Ultimate Multisite es cuáles son los requisitos para ejecutar nuestro plugin. Por suerte, solo hay unos pocos requisitos básicos para ejecutar Ultimate Multisite: un **domain name**, un buen **hosting provider** y una instalación de WordPress **Multisite**. ¡Eso es todo lo que necesitas!

## Nombre de dominio

Primero, hablemos del nombre de dominio.

Al elegir un nombre de dominio, ten en cuenta cuál es tu nicho específico y lo que crearás con Ultimate Multisite: ¿Quieres crear una agencia? ¿Vender sitios web tipo tienda? ¿O tal vez un hosting web de WordPress? Todo es posible con Ultimate Multisite.

Nuestro plugin ofrece servicios de suscripción, recolección de pagos, formularios de pago, cupones de descuento y comunicaciones con clientes. Una gran parte del trabajo necesario para instalar, configurar y mantener un WordPress Multisite es facilitado por Ultimate Multisite.

Cualquier nicho que elijas puede ser gestionado usando nuestro plugin siempre y cuando vendas sitios web como servicio. ¡Asegúrate de elegir un nombre de dominio que coincida con tu plan de negocio!

## Proveedor de hosting

Ahora que tienes un nombre de dominio atractivo que representa tu nicho y negocio, necesitarás elegir una buena compañía de hosting web para alojar tu red con Ultimate Multisite. Hay algunas buenas opciones en el mercado. Algunos proveedores de hosting se integran bien con Ultimate Multisite, lo que podría ser un factor decisivo dependiendo de lo que planeas construir.

Ultimate Multisite debería funcionar en cualquier proveedor de hosting. La única diferencia es que algunas compañías de hosting como _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ y _Runcloud_ tienen una integración más estrecha para el mapeo de dominios. Para otros hosting, es posible que necesites mapear los dominios personalizados manualmente. También es una buena idea obtener un proveedor de hosting que soporte AutoSSL. _Closte_, _Runcloud_ y _cPanel_ lo soportan.

En cuanto a las especificaciones del servidor, no hay una fórmula específica ya que dependerá de las características y funciones que ofrecerás y del tamaño de tu red. Un hosting compartido puede ser un buen comienzo, pero tan pronto como tu red comience a crecer, probablemente necesitarás un VPS dedicado.

## WordPress Multisite

Ahora, hablemos un poco sobre una instalación de WordPress Multisite.

WordPress Multisite te permite tener una red de sitios en una sola instalación. Esta es una función incorporada, pero no está activa por defecto. Necesitas habilitar esta función dentro de tu instalación de WordPress y el panel de control de tu hosting web.

Tener WordPress con Multisite habilitado **es la base para Ultimate Multisite**. Dado que el uso principal de Ultimate Multisite es ayudar a los usuarios a gestionar y vender sitios web como servicio, esto debe hacerse en WordPress con Multisite habilitado.

Tenemos **un artículo en profundidad** que enseña paso a paso lo que se necesita hacer para habilitar la función Multisite en tu instalación de WordPress.

Después de haber habilitado Multisite en WordPress, es hora de instalar Ultimate Multisite. Para instalar correctamente Ultimate Multisite, debes tener:

* WordPress: v5.3+ (Recomendado: Última versión estable)
* PHP: 7.4.x (soporte a > 8.0 próximamente)
* MySQL: v5+ (Recomendado: 5.6, en caso de que no tengas 8.0 disponible en tu proveedor de hosting)

Estos son los únicos requisitos de software para Ultimate Multisite.

Ten en cuenta que Ultimate Multisite puede funcionar con PHP 8.0, pero recomendamos ejecutarlo con PHP 7.4.x.

Además, recomendamos instalarlo en **main domains**, no en subdominios o subdirectorios. Puede funcionar en subdominios/subdirectorios, pero puede presentar algunos errores.
