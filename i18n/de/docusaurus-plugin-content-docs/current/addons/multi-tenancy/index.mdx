# Mandantenfähigkeit

Verteilen Sie Unterseiten auf Server und Datenbanken für Performance und Sicherheit

## Highlights der Version 1.2.0

Multi-Tenancy 1.2.0 fügt die souveräne Mandantenbasis für Workflows zur Produktionsisolierung hinzu:

- Zustandslose Mandanten-Autoanmeldung mit kurzlebigen, gegen Wiederholung geschützten SSO-Besuchen.
- Datenbank- und Dateisystemisolierung pro Unterseite für souveräne Mandanten.
- Hostinger-Fähigkeit für Mandantenfähigkeit zur hostbewussten Mandantenbereitstellung.
- Verifizierungstore für souveräne Migrationen für Prüfungen von Legacy-Daten und asynchrone Push-Verarbeitung.
- Konfigurierbare Datenbank-Host-Bindung auf derselben Maschine für Bedrock-, FrankenPHP- und Localhost-Installationen.
- Verbesserungen an Mandanten-Bootstrap, Datenbank-Writer, Schema-Bootstrap und Benutzerbereitstellung.
- Sichererer souveräner Abbau, der Mandanten-Zugangsdaten bereinigt.

Siehe die Benutzerhandbücher für den [Workflow zur souveränen Migration](/user-guide/getting-started/sovereign-migration-workflow), [Isolierung bei Mandantenfähigkeit](/user-guide/configuration/multi-tenancy-isolation) und [Betrieb souveräner Mandanten](/user-guide/administration/sovereign-tenant-operations).

## Anforderungen

- WordPress 5.3 oder höher
- PHP 7.4 oder höher
- Multisite Ultimate plugin (aktiv)

## Installation

1. Laden Sie die Add-on-Dateien in Ihr Verzeichnis `/wp-content/plugins/` hoch
2. Aktivieren Sie das plugin über das Menü „Plugins“ in WordPress
3. Konfigurieren Sie die Add-on-Einstellungen im Netzwerk-Adminbereich

## Konfiguration

Das Add-on unterstützt verschiedene Konfigurationsoptionen, die über die Netzwerk-Adminoberfläche zugänglich sind.

### Konfiguration souveräner Mandanten

Für souveräne Mandanten konfigurieren Sie die Mandanten-Datenbank, das Dateisystem-Stammverzeichnis, die Datenbank-Host-Bindung und die Host-Fähigkeit, bevor Sie Traffic zum Mandanten routen. Führen Sie nach jeder Änderung, die Datenbank-Zugangsdaten, Host-Bindung, Routing oder Benutzerbereitstellung betrifft, eine Migrationsverifizierung aus.

Verwenden Sie `localhost` nur, wenn die Mandanten-Laufzeitumgebung und die Datenbankberechtigungen lokalen Socket- oder Zugriff auf derselben Maschine erwarten. Verwenden Sie `127.0.0.1` oder einen Service-Hostnamen, wenn dies der Grant-Host ist, der von Ihrem Anbieter konfiguriert wurde.

### Hostinger-Fähigkeit

Die Hostinger-Fähigkeit für Mandantenfähigkeit arbeitet mit dem gemeinsamen Hostinger-Integrationstoken. Bestätigen Sie, dass die Domain in hPanel vorhanden ist, DNS auf den Hostinger Account zeigt und die Mandanten-Datenbankeinstellungen vor der Verifizierung mit hPanel übereinstimmen.

## Hooks und Filter

### Aktionen

- `ultimate-multisite-addon-template_init` - Wird nach der Add-on-Initialisierung ausgelöst
- `ultimate-multisite-addon-template_loaded` - Wird ausgelöst, nachdem alle Add-on-Dateien geladen wurden

### Filter

- `ultimate-multisite-addon-template_settings` - Add-on-Einstellungen ändern
- `ultimate-multisite-addon-template_enabled` - Aktivierungs-/Deaktivierungsstatus des Add-ons überschreiben
