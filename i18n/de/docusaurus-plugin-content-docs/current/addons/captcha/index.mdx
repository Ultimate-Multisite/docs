# Captcha

**Stoppt Bots. Schützt Einnahmen. Kein Konfigurationsaufwand.**

Verwalten Sie ein Multisite-Netzwerk mit benutzerdefinierten Domains? Sie kennen den Schmerz: Google reCAPTCHA und hCaptcha verlangen, dass Sie *jede einzelne Domain* in ihrer Admin-Konsole registrieren. Fügen Sie eine neue Kundenseite hinzu? Aktualisieren Sie die Captcha-Konfiguration. Ordnen Sie eine benutzerdefinierte Domain zu? Aktualisieren Sie erneut. Es hört nie auf.

**Ultimate Multisite: Captcha ändert alles.**

## Cap Captcha: Selbstgehosteter Schutz, der einfach funktioniert

Unser neuer **Cap Captcha**-Anbieter ist ein Game-Changer für Multisite-Netzwerke:

* **Zero Configuration** — Aktivieren Sie das Addon und Sie sind geschützt. Keine API-Schlüssel, kein Google-Konto, kein hCaptcha-Dashboard, keine Domain-Whitelist.
* **Works on Every Domain Automatically** — Subdomains, zugewiesene Domains, Kundendomains – sie sind alle sofort geschützt.
* **Self-Hosted & Private** — Ihre Daten verlassen nie Ihren Server. Vollständig GDPR-konform ohne Drittanbieter-Tracking.
* **Invisible Proof-of-Work** — Benutzer lösen ein rechnerisches Rätsel im Hintergrund. Kein Klicken von Feuerhydranten, keine frustrierenden Bildraster.
* **Lightweight** — Nur 20 KB. Verlangsamt Ihren Checkout nicht.

## WooCommerce Card Testing Protection

Card-Testing-Angriffe kosten Händlern jährlich Milliarden. Betrüger nutzen Bots, um gestohlene Kreditkarten gegen Ihren WooCommerce-Checkout zu validieren – Sie zahlen die Bearbeitungsgebühren, sie erhalten gültige Kartennummern.

**Wir schützen dort, wo es am wichtigsten ist:**

* **Store API Protection** — Blockiert Bot-Angriffe auf `/wp-json/wc/store/v1/checkout`, die herkömmliche Formular-Captchas umgehen
* **PayPal Payments Compatible** — Funktioniert nahtlos mit WooCommerce PayPal Payments
* **Real-Time Statistics** — Sehen Sie genau, wie viele Angriffe Sie blockieren

## Installation

1. Laden Sie `ultimate-multisite-captcha` in Ihr `/wp-content/plugins/`-Verzeichnis hoch
2. Aktivieren Sie das Plugin im Netzwerk
3. **Das war's. Sie sind geschützt.**

Cap Captcha aktiviert sich automatisch – keine Einstellungen zum Konfigurieren, keine API-Schlüssel einzugeben. Jeder unterstützte Endpunkt in Ihrem gesamten Netzwerk ist geschützt, sobald Sie aktivieren.

**Optional:** Um stattdessen Google reCAPTCHA oder hCaptcha zu verwenden, navigieren Sie zu **Ultimate Multisite → Settings → Captcha** und wählen Sie Ihren bevorzugten Anbieter.

## Häufig gestellte Fragen

## Muss ich etwas konfigurieren?

Nein! Cap Captcha wird automatisch aktiviert, wenn Sie das Addon aktivieren. Ihr gesamtes Netzwerk ist sofort geschützt – keine Einstellungen zum Konfigurieren, keine API-Schlüssel einzugeben.

## Benötige ich API-Schlüssel für Cap Captcha?

Nein. Cap Captcha ist vollständig selbstgehostet. Keine externen Konten, keine Domainregistrierung, keine Konfigurationsprobleme.

## Wird dies meinen Checkout verlangsamen?

Cap Captcha ist nur 20 KB und führt Proof-of-Work-Herausforderungen im Hintergrund aus. Ihre Kunden bemerken keine Verzögerung.

## Schützt dies vor Card-Testing-Angriffen?

Ja! Im Gegensatz zu herkömmlichen Captchas, die nur HTML-Formulare schützen, bindet unsere WooCommerce-Integration direkt in die Store API ein, um Bot-Angriffe auf API-Ebene zu blockieren – wo die meisten Card-Testing-Angriffe stattfinden.

## Kann ich unterschiedliche Captcha-Anbieter auf verschiedenen Sites verwenden?

Der Captcha-Anbieter wird netzwerkweit festgelegt, Sie können jedoch die Schwierigkeitsstufen je nach Bedarf anpassen.

## Ist dies GDPR-konform?

Cap Captcha ist zu 100 % selbstgehostet und sendet keine Daten an externe Dienste. Für Google reCAPTCHA und hCaptcha lesen Sie bitte deren jeweiligen Datenschutzrichtlinien.
