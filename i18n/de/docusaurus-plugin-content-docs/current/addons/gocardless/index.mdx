# GoCardless

Multisite Ultimate: GoCardless Zahlung

Erlauben Sie Ihren Kunden, Multisite Ultimate-Mitgliedschaften und Produkte mit Lastschrift über GoCardless zu bezahlen!

## Installation

1. Laden Sie 'ultimate-multisite-gocardless' in das Verzeichnis '/wp-content/plugins/' hoch
2. Aktivieren Sie das Plugin über das Menü 'Plugins' in WordPress
3. Folgen Sie dem Schritt-für-Schritt-Assistenten, um das Plugin einzurichten

## Verwendungsleitfaden

_GoCardless_ ist ein Zahlungs-Gateway, das in **Europa und im Vereinigten Königreich** verfügbar ist und **Lastschrift**-Zahlungen ermöglicht.

Das **Ultimate Multisite: GoCardless Gateway** fügt _GoCardless_-Unterstützung zu Ultimate Multisite hinzu und ermöglicht es Ihnen, Ihre Kunden in dieser Region mit Lastschrift zu belasten.

Dieses Tutorial erklärt, wie Sie _GoCardless_ einrichten, um Zahlungen über dieses Gateway in Ihrem Ultimate Multisite-Netzwerk zu akzeptieren.

_**Hinweis: GoCardless funktioniert nur mit den folgenden Währungen:**_ **EUR, GBP, AUD, SEK und DKK.**

## Installation des Add-ons

Zunächst müssen Sie das GoCardless-Add-on für Ultimate Multisite installieren. Das können Sie tun, indem Sie zu **Ultimate Multisite > Einstellungen > Add-ons (im Seitenmenü)** gehen und das GoCardless-Add-on in dieser Liste auswählen.

Nachdem das Add-on installiert und aktiviert wurde, gehen Sie zu **Ultimate Multisite > Einstellungen > Zahlungen**, schalten die GoCardless-Option ein und speichern.

![Payment gateway settings](/img/config/settings-payment-gateways.png)

## Erhalt eines Zugriffstokens

Erstellen Sie ein Konto bei _GoCardless_ (oder im _GoCardless_-Sandbox, wenn Sie es **zuerst testen** möchten, was empfohlen wird). Danach navigieren Sie nach dem Einloggen zum Menüpunkt **Entwickler**.

<!-- Screenshot unavailable: GoCardless dashboard showing Developers menu position -->

Auf der Seite **Entwickler** klicken Sie auf die Schaltfläche **Erstellen** in der oberen rechten Ecke des Bildschirms und wählen dann die Option **Zugriffstoken**.

<!-- Screenshot unavailable: GoCardless Developers page with Create button and Access Token option -->

Dies öffnet ein neues Modal-Fenster, in dem Sie ein neues Token erstellen können. Geben Sie ihm einen erkennbaren Namen und stellen Sie sicher, dass Sie die Option **Lesen-Schreiben-Zugriff** im Menü **Scope** auswählen.

<!-- Screenshot unavailable: GoCardless modal for creating new access token with Read-write scope -->

Das Zugriffstoken wird auf dem Bildschirm angezeigt. Kopieren Sie es und fügen Sie es in Ihr **Ultimate Multisite > Einstellungen > Zahlungen > GoCardless** Menü ein.

Stellen Sie sicher, dass Sie es im richtigen Feld platzieren. Wenn es ein **live token** **n** ist, schalten Sie den **Sandbox-Modus aus**, bevor Sie das Token einfügen.

![Payment gateway settings with GoCardless](/img/config/settings-payment-gateways.png)

<!-- Screenshot unavailable: GoCardless token fields in Ultimate Multisite payment settings -->

Das war's mit dem ersten Schritt. Als Nächstes müssen Sie einen Webhook-Listener einrichten.

## Einrichten des Webhook-Listeners

Bei Abonnements muss Ultimate Multisite über Änderungen des Abonnementstatus, neue Zahlungen und ähnliche Ereignisse informiert werden.

Die Art und Weise, wie Zahlungs-Gateways Ultimate Multisite über diese Änderungen informieren, ist die Verwendung von **Webhook-Aufrufen**. Aus diesem Grund müssen wir _GoCardless_ mitteilen, welche URL mit allen Informationen aufgerufen werden soll, damit die Integration zu 100 % funktioniert.

Gehen Sie zurück zu Ihrem **GoCardless > Entwickler**-Panel und klicken Sie erneut auf die Schaltfläche Erstellen, wählen Sie diesmal jedoch die Option **Webhook-Endpunkt**.

<!-- Screenshot unavailable: GoCardless Developers page with Webhook endpoint option -->A new window will open with the webhook endpoint options. It asks for the webhook URL, so head back to your **Ultimate Multisite settings > Payments > GoCardless** and copy the webhook URL listed in there:

<!-- Screenshot unavailable: Ultimate Multisite GoCardless settings showing webhook URL -->

Zusätzlich können Sie ein **Webhook-Geheimnis** eingeben, das zum Signieren Ihrer Webhook-Aufrufe verwendet wird. Was immer Sie in dieses Feld eingeben, müssen Sie auch im **Ultimate Multisite Webhook Secret-Feld** eingeben.

<!-- Screenshot unavailable: GoCardless webhook endpoint configuration form with URL and secret fields -->

<!-- Screenshot unavailable: Ultimate Multisite GoCardless webhook secret field -->

Schließen Sie ab, indem Sie den Endpunkt speichern, indem Sie auf die Schaltfläche **Webhook-Endpunkt erstellen** klicken, und anschließend Ihre **Ultimate Multisite-Einstellungen** ebenfalls speichern.

Sie sollten jetzt bereit sein, _GoCardless_ zu akzeptieren!
