# Gratis AI Agent

Gratis AI Agent ist ein autonomer WordPress AI Assistent, der es ihm ermöglicht, komplette WordPress-Websites anhand natürlicher Sprachbefehle zu planen, zu erstellen und zu verwalten. Er stellt einen strukturierten Satz von **Fähigkeiten** (Abilities) bereit – diskrete, testbare Aktionen, die der Agent ausführen kann – und deckt alles von der Content-Verwaltung bis zur Anpassung des Design-Systems ab. Version 1.9.0 fügt Fähigkeiten zur Inhaltserstellung, visuelle Seitenprüfung mittels Client-Side-Screenshots, fünf integrierte Agentenprofile und Feature Flags für Zugriffskontrolle und Branding hinzu.

## Key Features

- **Abilities system** — ein modulares, erweiterbares Verzeichnis von Aktionen, die der Agent auf Ihrer WordPress-Installation ausführen kann
- **Content Management** — Erstellen von Beiträgen und Seiten (einschließlich der Zuweisung von Seitenvorlagen), Stapelverarbeitung mehrerer Beiträge in einem Aufruf, Festlegen von Beidermaßen (Featured Images) und Erstellen von Kontaktformularen
- **Visual Review** — Erfassung von Screenshots von Live-Seiten, Vergleich von Vorher/Nachher-Zuständen und Erstellung eines KI-generierten Design-Reviews, das Layout, Typografie, Farbe und Barrierefreiheit abdeckt
- **Five built-in agents** — Content Writer, Site Builder, Design Studio, Plugin Manager und Support Assistant; über den Agent Picker im Chat-Header umschaltbar
- **Feature flags** — Zugriffskontroll- und Branding-Schalter unter Einstellungen → Feature Flags für Rollenbeschränkungen, White-Label-Benennung und Widget-Anpassungen
- **Custom Post Type management** — Registrieren, Auflisten und Löschen von Custom Post Types mit Persistenz über Neustarts hinweg
- **Custom Taxonomy management** — Registrieren, Auflisten und Löschen von Custom Taxonomies mit Persistenz
- **Design System abilities** — Einfügen benutzerdefinierten CSS, Verwalten kuratierter Block-Muster, Festlegen des Seitenlogos und Anwenden von theme.json Presets
- **Global Styles management** — Lesen und Schreiben von theme.json Werten für Farbschemata, Typografie, Abstände und mehr
- **Navigation Menu management** — Programmatisches Erstellen und Aktualisieren von Navigationsmenüs und deren Elementen
- **Options Management** — Lesen und Schreiben von WordPress-Optionen mit einer integrierten Sicherheits-Blockliste zum Schutz kritischer Einstellungen
- **Installable Abilities Registry** — Entdecken und Aktivieren zusätzlicher Ability-Packs, ohne die Core-Plugin-Dateien zu ändern
- **Site Builder Orchestration v2** — Mehrstufige Site-Build-Pläne mit Plugin-Erkennung, Fortschrittsverfolgung und Fehlerbehebung
- **WP-CLI benchmark command** — Ausführen der Agenten-Fähigkeits-Benchmark-Suite über die Kommandozeile für CI und Modellbewertung

## Requirements

- WordPress 6.0 oder höher
- PHP 8.1 oder höher
- WordPress Multisite (netzwerkaktiviert) oder Single-Site-Installation
- Ein aktives AI Provider Connector Plugin (Anthropic, OpenAI oder kompatibel)

## Installation

1. Laden Sie die Plugin-Dateien in `/wp-content/plugins/gratis-ai-agent` hoch
2. Aktivieren Sie das Plugin über den Bildschirm **Plugins** in WordPress
3. Navigieren Sie zu **Gratis AI Agent → Settings** und verbinden Sie Ihren AI Provider
4. (Optional) Durchsuchen Sie das **Abilities Registry**, um zusätzliche Ability-Packs zu installieren

## Quick Start

Nach der Aktivierung und Verbindung zu einem AI Provider öffnen Sie das **Gratis AI Agent** Panel in der WordPress-Admin-Sidebar. Geben Sie eine Anweisung in natürlicher Sprache ein und der Agent plant und führt die erforderlichen Schritte mithilfe seiner verfügbaren Fähigkeiten aus.

**Beispielanweisungen:**

```
Create a "Portfolio" custom post type with a "Project Category" taxonomy.
```

```
Inject a dark-mode colour palette into the theme and set the primary colour to #1a1a2e.
```

```
Build a restaurant website with an online menu, booking form, and contact page.
```

## Abilities Overview

Abilities sind die atomaren Aktionen, die der Agent ausführen kann. Jede Ability ist eine registrierte PHP-Klasse, die ein Schema (Parameter und Rückgabetypen) bereitstellt, das der Agent zur Laufzeit liest. Die Abilities sind in funktionale Bereiche unterteilt:

| Area | Abilities |
|---|---|
| **Content Management** | `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form` |
| **Visual Review** | `capture_screenshot`, `compare_screenshots`, `review_page_design` |
| **Custom Post Types** | `register_post_type`, `list_post_types`, `delete_post_type` |
| **Custom Taxonomies** | `register_taxonomy`, `list_taxonomies`, `delete_taxonomy` |
| **Design System** | `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset` |
| **Global Styles** | `get_global_styles`, `set_global_styles`, `reset_global_styles` |
| **Navigation Menus** | `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus` |
| **Options Management** | `get_option`, `set_option`, `delete_option`, `list_options` |
| **Installable Abilities** | `list_available_abilities`, `install_ability`, `recommend_plugin` |
| **Site Builder** | `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error` |

Siehe die [Abilities Reference](./abilities) für die vollständige Parameterdokumentation und Anwendungsbeispiele.

## WP-CLI

Gratis AI Agent liefert eine `wp gratis-ai-agent` Befehlsfamilie, um Benchmarks auszuführen und Abilities über die Kommandozeile zu verwalten. Die vollständige Befehlsliste finden Sie in der [WP-CLI Reference](./wp-cli).

```bash
# Run the full benchmark suite
wp gratis-ai-agent benchmark

# List installed abilities
wp gratis-ai-agent abilities list

# Install an ability pack
wp gratis-ai-agent abilities install <slug>
```

## Configuration

Alle Einstellungen finden Sie unter **Gratis AI Agent → Settings** im WordPress-Admin:

| Setting | Description |
|---|---|
| **AI Provider** | Wählen und konfigurieren Sie den aktiven AI Provider Connector |
| **Model** | Wählen Sie das Sprachmodell, das für die Agenten-Sitzungen verwendet werden soll |
| **Max Tokens** | Token-Budget pro Anfrage (Standard: 4096) |
| **Ability Timeout** | Maximale Sekunden, die eine einzelne Ability laufen darf (Standard: 30) |
| **Debug Logging** | Protokolliert Ability-Aufrufe und Agenten-Denkprozesse im WordPress Debug Log |

## Options Management Safety Blocklist

Die Options Management Ability enthält eine integrierte Blockliste, die den Agenten daran hindert, sensible WordPress-Optionen zu lesen oder zu schreiben. Die Standard-Blockliste umfasst:

- `admin_email`, `blogadmin_email`
- `siteurl`, `home` (URL-Optionen — verwenden Sie hierfür ein dediziertes Migrations-Tool)
- `auth_key`, `secure_auth_key`, `logged_in_key`, `nonce_key` (Geheimschlüssel)
- `active_plugins`, `template` (Plugin-/Theme-Aktivierung — verwenden Sie dedizierte Abilities)
- Zahlungs-Gateway-Anmeldeinformationen und API-Schlüssel, die von installierten Plugins registriert wurden

Zusätzliche Einträge können über den Filter `gratis_ai_agent_options_blocklist` hinzugefügt werden.

## Hooks and Filters

### Actions

- `gratis_ai_agent_ability_registered` — wird ausgelöst, nachdem eine Ability registriert wurde, empfängt den Namen der Ability-Klasse
- `gratis_ai_agent_plan_started` — wird ausgelöst, wenn ein Site Builder Plan beginnt mit der Ausführung
- `gratis_ai_agent_plan_completed` — wird ausgelöst, wenn ein Plan erfolgreich abgeschlossen wurde
- `gratis_ai_agent_plan_failed` — wird ausgelöst, wenn ein Plan auf einen nicht wiederherstellbaren Fehler stößt

### Filters

- `gratis_ai_agent_abilities` — modifiziert die registrierte Ability-Liste, bevor der Agent sie lädt
- `gratis_ai_agent_options_blocklist` — erweitert oder reduziert die Options-Sicherheits-Blockliste
- `gratis_ai_agent_benchmark_questions` — fügt benutzerdefinierte Benchmark-Fragen der Bewertungs-Suite hinzu
- `gratis_ai_agent_plan_plugins` — modifiziert die Plugin-Erkennungsliste während der Site-Plan-Generierung
