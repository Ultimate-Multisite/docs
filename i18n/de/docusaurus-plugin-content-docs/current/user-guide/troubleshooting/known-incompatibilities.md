# Bekannte Inkompatibilitäten mit anderen Plugins

Nach der Installation und Aktivierung von Ultimate Multisite können Sie auf Probleme stoßen, die auf nicht behobene Inkompatibilitäten mit anderen Plugins zurückzuführen sind.

Wir arbeiten aktiv daran, diese Liste mit jeder neuen Version zu verkleinern, aber wenn ein Plugin hier aufgeführt ist, müssen Sie es möglicherweise deaktivieren, um ein vollständig funktionsfähiges Ultimate Multisite Network zu erhalten.

Hier ist eine Liste der Plugins und was Sie in diesen Fällen tun sollten:

**Einige WPMU DEV Plugins** Einige der Plugins, die _nicht mehr_ von WPMU DEV gepflegt werden, sind dafür bekannt, dass sie den Ultimate Multisite-Registrierungsfluss im letzten Schritt stoppen (der Benutzer kann kein Konto erstellen). Hier ist eine Liste der Plugins, die diese Probleme verursachen:

  * Signup Language: _Ultimate Multisite hat ein Add-on, das die gleiche Funktionalität bietet, genannt Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin erforderlich._

  * New Blog Templates: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin erforderlich._

  * Set Password on Multisite Blog Creation: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin erforderlich._

  * Signup TOS: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin erforderlich._

  * Anti-Splog: Obwohl weniger leistungsfähig, ist auch _dies in Ultimate Multisite integriert._

**Hinweis** : Alle anderen WPMU Dev Plugins können normal zusammen mit Ultimate Multisite verwendet werden. Beispiele sind _Smush_, _Forminator_, _Defender_, etc.
