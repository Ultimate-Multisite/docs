# Captcha

**Misa Bots. Dzivirira Mari Inopinda. Hapana Kunetsa kwe Configuration.**

Kumhanyisa network ye multisite ine domains dzakagadzirirwa? Unoziva kurwadziwa kwacho: Google reCAPTCHA ne hCaptcha zvinoda kuti unyorese *domain imwe neimwe* mu admin console yavo. Wedzera site yemutengi mutsva? Gadzirisa captcha config. Batanidza domain yakagadzirirwa? Gadzirisa zvakare. Hazvimboperi.

**Ultimate Multisite: Captcha inoshandura zvese.**

## Cap Captcha: Dziviriro Inogamuchirwa pa Server Yako Inongoshanda {#cap-captcha-self-hosted-protection-that-just-works}

Provider wedu mutsva we **Cap Captcha** ishanduko huru kuma network e multisite:

* **Zero Configuration** — Activate addon uye unenge wadzivirirwa. Hapana API keys, hapana Google account, hapana hCaptcha dashboard, hapana domain whitelisting.
* **Inoshanda pa Domain Yese Otomatiki** — Subdomains, domains dzakabatanidzwa, domains dzevatengi—dzese dzinodzivirirwa pakarepo.
* **Inogamuchirwa pa Server Yako & Yakavanzika** — Data rako harimbobvi pa server yako. Inotevedzera GDPR zvizere pasina third-party tracking.
* **Proof-of-Work Isingaonekwi** — Vashandisi vanogadzirisa dambudziko rekuverenga kumashure. Hapana kudzvanya fire hydrants, hapana image grids dzinoshungurudza.
* **Yakareruka** — Ingori 20KB. Haizononotse checkout yako.

## Dziviriro ye WooCommerce Card Testing {#woocommerce-card-testing-protection}

Kurwiswa kwe card testing kunodyira vatengesi mabhiriyoni gore rega rega. Matsotsi anoshandisa bots kusimbisa makadhi ekiredhiti akabiwa pa WooCommerce checkout yako—iwe unobhadhara processing fees, ivo vanowana manhamba emakadhi anoshanda.

**Tinodzivirira panonyanya kukosha:**

* **Dziviriro ye Store API** — Inovhara kurwiswa kwe bots pa `/wp-json/wc/store/v1/checkout` kunopfuura traditional form captchas
* **Inoenderana ne PayPal Payments** — Inoshanda zvakanaka ne WooCommerce PayPal Payments
* **Statistics dze Nguva-Chaiyo** — Ona chaizvo kuti kurwiswa kungani kwauri kuvhara

## Kuiswa {#installation}

1. Upload `ultimate-multisite-captcha` ku directory yako ye `/wp-content/plugins/`
2. Network Activate plugin
3. **Ndizvozvo. Wadzivirirwa.**

Cap Captcha inozvi activate otomatiki—hapana settings dzekugadzirisa, hapana API keys dzekuisa. Endpoint yese inotsigirwa mu network yako yese inodzivirirwa panguva yauno activate.

**Sarudzo:** Kushandisa Google reCAPTCHA kana hCaptcha pachinzvimbo, enda ku **Ultimate Multisite → Settings → Captcha** uye sarudza provider yaunoda.

## Rate Limiting {#rate-limiting}

Ultimate Multisite: Captcha v1.5.0 inounza hard-stop rate limiter inodzivirira network yako kubva ku brute-force uye card-testing attacks.

### Inoshanda Sei {#how-it-works}

Rate limiter inoverenga request yese ye GET ne POST pane nzvimbo dzakadzivirirwa ne captcha:

* **WordPress login endpoints** — wp-login, register, lost-password, comments
* **WooCommerce checkout** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

Kana mushanyi akadarika rate limit, plugin inopindura ne:

* **HTTP 429 (Too Many Requests)** status code
* **Retry-After** header inoratidza nguva yekuedzazve
* **Randomised tarpit sleep** (1–5 seconds by default, hard-capped pa 15 seconds) kunonotsa attackers

### Configuration {#configuration}

#### Tarpit Timing {#tarpit-timing}

Dzora kunonoka kwe randomised kunoiswa kuma requests ane rate-limited:

* **`cap_rate_limit_tarpit_min`** — Minimum sleep duration mumasekondi (default: 1)
* **`cap_rate_limit_tarpit_max`** — Maximum sleep duration mumasekondi (default: 5, hard-capped pa 15)

Settings idzi dziripo mu **Ultimate Multisite → Settings → Captcha**.

### Kuwedzerika {#extensibility}

#### IP Whitelist Filter {#ip-whitelist-filter}

Sunungura IP ranges dzinovimbwa kubva ku rate limiting:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // Whitelist your monitoring service
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### Pre-Block Action {#pre-block-action}

Mhanyisa logic yakagadzirirwa pakarepo rate-limit block isati yatumirwa:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // Log the block, notify admins, or update custom metrics
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## Kuonekwa kwe Client IP & Trusted Proxies {#client-ip-detection--trusted-proxies}

Ultimate Multisite: Captcha v1.5.0 inounza kuonekwa kwe client-IP kunodzivirira spoofing. Method ye `Captcha_Core::get_client_ip()` yava single source of truth yekupa visitor IP attribution mu:

* Makiyi emabhaketi ekudzora mwero
* Mupi weCaptcha `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* Mahash eIP enhamba

### Muenzaniso weKuvimba {#trust-model}

Chawedzero chinosimbisa hurongwa hwakasimba hwekuvimbika kweIP:

1. **REMOTE_ADDR** — Inogara ichivimbwa (IP yemubatanidzwa wepedyo)
2. **CF-Connecting-IP** — Inoremekedzwa chete kana:
   - Mubatanidzwa wepedyo uri mukati merudzi rweIP rweCloudflare rwazvino
   - `cap_trust_cloudflare_headers` yakagoneswa (default: OFF)
3. **X-Forwarded-For** — Inoremekedzwa chete kana:
   - Mubatanidzwa wepedyo uri murondedzero yako yakagadziriswa neadmin yema trusted-proxy
   - Header inoverengwa kubva kurudyi kuenda kuruboshwe, ichisvetuka mahop anovimbwa/Cloudflare

### Kugadzirisa {#configuration-1}

#### Kuvimba neHeader yeCloudflare {#cloudflare-header-trust}

Gonesa kuvimba neheader yeCloudflare `CF-Connecting-IP`:

* **`cap_trust_cloudflare_headers`** — Seta kuON kuti usarudze kuvimba ne`CF-Connecting-IP`
  - Default: OFF (yakadzimwa)
  - Chawedzero chinouya neCloudflare CIDR snapshot yakabatanidzwa
  - Marange eCIDR anovandudzwa vhiki nevhiki kuburikidza ne wp-cron
  - Fallback yakabatanidzwa inoshandiswa kana kuvandudza kukatadza

#### Kugadzirisa Trusted Proxy {#trusted-proxy-configuration}

Tsanangura ma proxy ako epamberi nema load-balancers:

* **`cap_trusted_proxies`** — Nzvimbo yemavara yeCIDRs kana IPs dzisina zvimwe (imwe pamutsetse)
  - Makomendi anotenderwa (mitsetse inotanga ne`#`)
  - Pasina setting iyi, `X-Forwarded-For` inoregeredzwa kunyange kana rate limiter yagoneswa
  - Muenzaniso:
    ```
    # Our load balancer
    192.0.2.0/24

    # Backup proxy
    198.51.100.50
    ```

#### Kuzviona Otomatiki Pakutanga Kugonesa {#first-enable-auto-detection}

Paunotanga kugonesa rate limiter, chawedzero chinoona mamiriro ako angangodaro eCloudflare neproxy uye chinoratidza chiziviso cheadmin chine kudzvanya kamwe chete "Shandisa magadzirirwo aonekwa".

* Chawedzero **hachimbofa chakanyora pamusoro** pemavalue ako akachengetwa
* Kana traffic inotevera ikaratidza kuti config yako haichawirirani nechokwadi (semuenzaniso, Cloudflare yakachinja marange eCIDR), chiziviso chemismatch chisingabvisiki chinoburitsa update yakakurudzirwa

### Nei Izvi Zvichikosha {#why-this-matters}

Muenzaniso wakasimba wekuvimbika kweIP unovhara nzira yakakosha yekunyepedzera: zvikumbiro zvakananga kuorigin-server zvakatakura header ye`CF-Connecting-IP` yakagadzirwa zvaimbogadzwa mubhaketi neIP yakanyepedzerwa panzvimbo pemubatanidzwa chaiwo. Izvi zvakanyanya kukosha kune:

* **Kudzora mwero** — Vanorwisa havagoni kupfuura miganhu nekuzviti vane mamwe maIP
* **Nhamba** — Metrics dzako dzekurwiswa dzinoratidza maIP evashanyi chaiwo, kwete akagadzirwa
* **Kuongorora Captcha** — Vapi vanogamuchira IP chaiyo yemushanyi pakuyera njodzi
