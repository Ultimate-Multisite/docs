# جست‌وجوی دامنه در Header {#header-domain-search}

از این راه‌اندازی زمانی استفاده کنید که یک فرم کوچک در Header می‌خواهید که جست‌وجوی دامنه را شروع کند، سپس به مشتری اجازه دهد از میان دامنه‌های در دسترس داخل پرداخت Ultimate Multisite انتخاب کند.

## الزامات {#requirements}

- Ultimate Multisite به‌صورت network-active.
- Multisite Ultimate Domain Seller به‌صورت network-active.
- حداقل یک محصول فعال ثبت دامنه با:
  - `domain_provider` که روی یک ارائه‌دهنده پیکربندی‌شده تنظیم شده باشد.
  - TLDهای پشتیبانی‌شده پیکربندی شده باشند، برای مثال `com`، `net` و `org`.
- یک فرم پرداخت معتبر که شامل فیلد **انتخاب دامنه** باشد.

## فرم پرداخت {#checkout-form}

1. فرم پرداختی را که توسط صفحه ثبت‌نام استفاده می‌شود ایجاد یا ویرایش کنید.
2. فیلدهای معمول و الزامی پرداخت/Account را اضافه کنید، از جمله **نام کاربری**. فرم پرداختی که فقط شامل فیلد دامنه باشد توسط اعتبارسنجی Ultimate Multisite رد می‌شود.
3. یک فیلد **انتخاب دامنه** اضافه کنید.
4. زمانی که جریان باید به‌جای زیر‌دامنه‌های رایگان یا دامنه‌های موجود بر دامنه‌های ثبت‌شده تمرکز داشته باشد، حالت انتخاب دامنه را روی **فقط ثبت** تنظیم کنید.
5. محصول ثبت دامنه را به آن فیلد اختصاص دهید.

صفحه ثبت‌نام باید فرم پرداخت را نمایش دهد، برای مثال:

```text
[wu_checkout slug="domain-form"]
```

## فرم Header {#header-form}

یک فرم کوچک `GET` در Header سایت اضافه کنید که عبارت جست‌وجوی واردشده را به‌عنوان `domain_name` به صفحه پرداخت ارسال کند:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

در JavaScript سفارشی Header، دامنه‌ای را از پیش انتخاب نکنید. Header باید فقط عبارت جست‌وجو را منتقل کند. اسکریپت پرداخت Domain Seller مقدار `?domain_name=example` را می‌خواند، کادر جست‌وجوی پرداخت را پر می‌کند و جست‌وجوی در دسترس بودن را اجرا می‌کند تا مشتری بتواند از میان دامنه‌های برگشتی انتخاب کند.

## رفتار مورد انتظار {#expected-behaviour}

جست‌وجوی `example` در Header باید این را باز کند:

```text
/register/?domain_name=example
```

سپس پرداخت باید نتایج قابل انتخابی مانند موارد زیر را نمایش دهد:

- `example.com`
- `example.net`
- `example.org` ناموجود
- سایر TLDهای پشتیبانی‌شده توسط ارائه‌دهنده

پس از انتخاب یک نتیجه در دسترس، خلاصه سفارش باید محصول ثبت دامنه و نام دامنه انتخاب‌شده را شامل شود.

## تأیید {#verification}

1. صفحه اصلی را باز کنید.
2. یک نام ساده و بدون پسوند را جست‌وجو کنید، برای مثال `example`.
3. تأیید کنید URL پرداخت شامل `?domain_name=example` است.
4. تأیید کنید فیلد دامنه پرداخت شامل `example` است.
5. تأیید کنید فهرستی از گزینه‌های دامنه ظاهر می‌شود.
6. برای یک دامنه در دسترس روی **انتخاب** کلیک کنید.
7. تأیید کنید خلاصه سفارش شامل `Domain Registration - example.com` یا دامنه انتخاب‌شده است.

## عیب‌یابی {#troubleshooting}

- اگر هیچ فهرستی ظاهر نشد، زبانه شبکه مرورگر را برای `admin-ajax.php?action=wu_domain_search` بررسی کنید و تأیید کنید که `domains` یا `results` غیرخالی برمی‌گرداند.
- اگر فرم پرداخت هنگام ذخیره‌سازی در اعتبارسنجی ناموفق شد، فیلدهای الزامی Account مانند **نام کاربری** را اضافه کنید.
- اگر انتخاب یک دامنه سبد خرید را به‌روزرسانی نمی‌کند، تأیید کنید `window.wu_checkout_form` وجود دارد و دارایی‌های پرداخت Domain Seller در صفحه پرداخت بارگذاری شده‌اند.
