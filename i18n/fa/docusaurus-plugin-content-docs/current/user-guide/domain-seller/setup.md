# Domain Seller: راه‌اندازی و پیکربندی ارائه‌دهنده

افزونه Domain Seller همراه با یک راهنمای راه‌اندازی مرحله‌به‌مرحله ارائه می‌شود که شما را در تمام مراحل لازم همراهی می‌کند. این صفحه جریان راهنما و نحوه پیکربندی یا پیکربندی مجدد ارائه‌دهنده‌ها پس از آن را پوشش می‌دهد.

## الزامات

- **Multisite Ultimate** نسخه 2.4.12 یا بالاتر، فعال‌شده در شبکه
- **PHP** 7.4+
- اطلاعات API برای حداقل یک ثبت‌کننده پشتیبانی‌شده

## راهنمای راه‌اندازی اولین اجرا

راهنمای راه‌اندازی در اولین باری که plugin را در شبکه فعال می‌کنید، به‌صورت خودکار اجرا می‌شود. همچنین در هر زمان از مسیر **Network Admin › Ultimate Multisite › Domain Seller Setup** در دسترس است.

### مرحله ۱ — انتخاب یک ارائه‌دهنده

ثبت‌کننده‌ای را که می‌خواهید متصل کنید انتخاب کنید. گزینه‌های پشتیبانی‌شده:

| ارائه‌دهنده | مدیریت DNS | حریم خصوصی WHOIS |
|---|---|---|
| OpenSRS | بله | بله |
| Namecheap | خیر | بله (WhoisGuard، رایگان) |
| HostAfrica | بله | بله (محافظت از شناسه) |
| Openprovider | بله | بله |
| Hostinger | از طریق نگاشت دامنه اصلی Hostinger برای دامنه‌های میزبانی‌شده | بله |
| GoDaddy | خیر | خیر |
| ResellerClub | بله | خیر |
| NameSilo | خیر | خیر |
| Enom | بله | خیر |

### مرحله ۲ — وارد کردن اطلاعات

هر ارائه‌دهنده فیلدهای اطلاعاتی متفاوتی دارد:

**OpenSRS** — نام کاربری و کلید خصوصی (از OpenSRS Reseller Control Panel)

**Namecheap** — نام کاربری و کلید API (از Account › Tools › API Access)

**HostAfrica** — endpoint مربوط به Domains Reseller API و اطلاعات ورود از ماژول فروشنده HostAfrica. در حال حاضر endpoint جداگانه‌ای برای sandbox مستند نشده است؛ پیش از اجرای ثبت‌های زنده، با بررسی‌های امن فقط‌خواندنی آزمایش کنید.

**Openprovider** — نام کاربری و گذرواژه با دسترسی API فعال. حالت sandbox اختیاری از API sandbox مربوط به Openprovider استفاده می‌کند، و یک شناسه مشتری پیش‌فرض اختیاری می‌تواند برای ثبت‌ها دوباره استفاده شود.

**Hostinger** — توکن API مشترک Hostinger hPanel از یکپارچه‌سازی اصلی Hostinger. همین توکن هم نگاشت دامنه اصلی و هم عملیات ثبت Domain Seller را تأمین می‌کند.

**GoDaddy** — کلید API و رمز (از developer.godaddy.com)

**ResellerClub** — شناسه فروشنده و کلید API (از پنل کنترل ResellerClub)

**NameSilo** — کلید API (از namesilo.com › Account › API Manager)

**Enom** — شناسه Account و توکن API

در صورت وجود، **حالت Sandbox** را فعال کنید تا پیش از رفتن به حالت زنده، در برابر محیط آزمایشی ارائه‌دهنده تست کنید.

### مرحله ۳ — آزمایش اتصال

روی **Test Connection** کلیک کنید. راهنما یک فراخوانی سبک API ارسال می‌کند تا اطلاعات ورود و اتصال را بررسی کند. پیش از ادامه، هر مشکل مربوط به اطلاعات ورود را برطرف کنید.

### مرحله ۴ — وارد کردن TLDها

روی **Import TLDs** کلیک کنید تا همه TLDهای موجود و قیمت‌گذاری عمده‌فروشی از ارائه‌دهنده متصل دریافت شود. این کار فهرست TLD مورد استفاده توسط محصولات دامنه را پر می‌کند. واردسازی ممکن است برای ارائه‌دهنده‌هایی با کاتالوگ‌های بزرگ TLD حدود ۳۰ تا ۶۰ ثانیه طول بکشد.

TLDها همچنین روزی یک‌بار از طریق یک کار cron زمان‌بندی‌شده به‌صورت خودکار دوباره همگام‌سازی می‌شوند.

### مرحله ۵ — ایجاد یک محصول دامنه

راهنما یک محصول دامنه پیش‌فرض همه‌گیر با ۱۰٪ افزایش قیمت ایجاد می‌کند. می‌توانید این محصول را بلافاصله ویرایش کنید یا رد کنید و محصولات را به‌صورت دستی در **Ultimate Multisite › Products** ایجاد کنید.

برای راهنمای کامل پیکربندی محصول، [محصولات دامنه و قیمت‌گذاری](./domain-products) را ببینید.

## پیکربندی مجدد یک ارائه‌دهنده

به **Network Admin › Ultimate Multisite › Settings › Domain Seller** بروید (یا در فهرست plugin روی **Settings** کلیک کنید).

صفحه تنظیمات شامل موارد زیر است:

- **فعال‌سازی فروش دامنه** — روشن/خاموش کردن کل قابلیت
- **ارائه‌دهنده پیش‌فرض** — ارائه‌دهنده‌ای که برای جست‌وجوهای دامنه و محصولات جدید استفاده می‌شود
- **حداکثر TLDها در هر جست‌وجو** — تعداد TLDهایی که هنگام جست‌وجوی مشتری بررسی می‌شوند؛ مقادیر بالاتر گزینه‌های بیشتری نشان می‌دهند اما کندتر هستند
- **مدت‌زمان cache موجودبودن** — مدت‌زمان cache کردن نتایج موجودبودن و قیمت‌گذاری؛ مقادیر پایین‌تر دقیق‌ترند اما فراخوانی‌های API را افزایش می‌دهند
- **مدیریت محصولات دامنه** — پیوند سریع به فهرست Products
- **پیکربندی ارائه‌دهنده‌ها** — Integration Wizard را برای افزودن یا پیکربندی مجدد ارائه‌دهنده‌ها باز می‌کند

### افزودن یک ارائه‌دهنده دوم

روی **Configure providers** کلیک کنید و راهنما را دوباره برای ثبت‌کننده جدید اجرا کنید. می‌توانید چند ارائه‌دهنده را هم‌زمان پیکربندی‌شده داشته باشید. هر محصول دامنه را به یک ارائه‌دهنده مشخص اختصاص دهید، یا آن را روی پیش‌فرض بگذارید.

### همگام‌سازی دستی TLDها

در صفحه تنظیمات، کنار هر ارائه‌دهنده پیکربندی‌شده روی **Sync TLDs** کلیک کنید تا آخرین قیمت‌گذاری دریافت شود. این کار پس از به‌روزرسانی قیمت‌گذاری عمده‌فروشی توسط ارائه‌دهنده یا افزودن TLDهای جدید مفید است.

هر ارائه‌دهنده در کانال گزارش خود می‌نویسد. گزارش‌ها در **Network Admin › Ultimate Multisite › Logs** قابل مشاهده‌اند:

| `domain-seller-registration` | همه تلاش‌های ثبت (موفق و ناموفق) |
| `domain-seller-renewal` | نتایج کار تمدید |
| `domain-seller-opensrs` | فعالیت خام API مربوط به OpenSRS |
| `domain-seller-namecheap` | فعالیت خام API مربوط به Namecheap |
| `domain-seller-hostafrica` | فعالیت خام API مربوط به HostAfrica |
| `domain-seller-openprovider` | فعالیت خام API مربوط به Openprovider |
| `domain-seller-hostinger` | فعالیت خام API مربوط به Hostinger |
| `domain-seller-godaddy` | فعالیت خام API مربوط به GoDaddy |
| `domain-seller-resellerclub` | فعالیت خام API مربوط به ResellerClub |
| `domain-seller-namesilo` | فعالیت خام API مربوط به NameSilo |

## نکات قابلیت‌های ارائه‌دهنده

همه APIهای ثبت‌کننده‌ها عملیات یکسانی را ارائه نمی‌دهند. این افزونه عملیات پشتیبانی‌نشده را با خطاهای واضح برای مدیر نمایش می‌دهد، به‌جای اینکه بی‌سروصدا شکست بخورد.

- **HostAfrica** گسترده‌ترین جریان کاری زندهٔ نمایندگی فروش را پشتیبانی می‌کند، شامل جست‌وجو، همگام‌سازی TLD/قیمت‌گذاری، ثبت، تمدید، انتقال، به‌روزرسانی nameserver، رکوردهای DNS، کدهای EPP، قفل registrar، و محافظت از ID.
- **Openprovider** همگام‌سازی TLD با قیمت نمایندگی فروش، ثبت، تمدید، انتقال‌ها، به‌روزرسانی nameserver، zoneهای DNS، کدهای EPP، قفل registrar، و حریم خصوصی WHOIS را پشتیبانی می‌کند. این سرویس با یک bearer token کوتاه‌مدت احراز هویت می‌کند که addon آن را به‌صورت خودکار تازه‌سازی می‌کند.
- **Hostinger** جست‌وجوی در دسترس بودن، ثبت، جست‌وجوی portfolio، به‌روزرسانی nameserver، قفل registrar، و حریم خصوصی WHOIS را از طریق توکن hPanel API مشترک پشتیبانی می‌کند. Domains API عمومی Hostinger قیمت‌گذاری نمایندگی فروش/عمده، انتقال ورودی، تمدید صریح، یا بازیابی کد EPP را ارائه نمی‌کند؛ تمدیدها فقط به‌صورت تمدید خودکار هستند.
