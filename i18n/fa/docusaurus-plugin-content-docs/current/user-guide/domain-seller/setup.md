# فروش دامنه: راه‌اندازی و پیکربندی ارائه‌دهنده

افزونه Domain Seller با یک ویزارد (Wizard) راهنمای گام به گام عرضه می‌شود که شما را در تمام مراحل مورد نیاز همراهی می‌کند. این صفحه، جریان کار ویزارد و نحوه پیکربندی یا پیکربندی مجدد ارائه‌دهندگان پس از آن را پوشش می‌دهد.

## پیش‌نیازها

- **Ultimate Multisite** نسخه v2.4.12 یا بالاتر، فعال شده در سطح شبکه (network-activated)
- **PHP** 7.4+
- اعتبارنامه‌های API برای حداقل یک رجیسترار پشتیبانی شده

## ویزارد راه‌اندازی اولیه

ویزارد راه‌اندازی، اولین باری که پلاگین را در سطح شبکه فعال می‌کنید، به صورت خودکار اجرا می‌شود. همچنین در هر زمان از مسیر **Network Admin › Ultimate Multisite › Domain Seller Setup** در دسترس است.

### گام ۱ — انتخاب یک ارائه‌دهنده

رجیسترار (Registrar) که می‌خواهید متصل کنید را انتخاب کنید. گزینه‌های پشتیبانی شده عبارتند از:

| Provider | مدیریت DNS | حریم خصوصی WHOIS |
|---|---|---|
| OpenSRS | بله | بله |
| Namecheap | خیر | بله (WhoisGuard، رایگان) |
| GoDaddy | خیر | خیر |
| ResellerClub | بله | خیر |
| NameSilo | خیر | خیر |
| Enom | بله | خیر |

### گام ۲ — وارد کردن اعتبارنامه‌ها

هر ارائه‌دهنده فیلدهای اعتبارنامه‌ای متفاوتی دارد:

**OpenSRS** — نام کاربری و کلید خصوصی (از پنل کنترل فروشنده OpenSRS)

**Namecheap** — نام کاربری و کلید API (از Account › Tools › API Access)

**GoDaddy** — کلید API و Secret (از developer.godaddy.com)

**ResellerClub** — شناسه فروشنده (Reseller ID) و کلید API (از پنل کنترل ResellerClub)

**NameSilo** — کلید API (از namesilo.com › Account › API Manager)

**Enom** — شناسه حساب (Account ID) و توکن API

در صورت وجود، **Sandbox mode** را بررسی کنید تا قبل از استفاده عملی، با محیط تست ارائه‌دهنده کار کنید.

### گام ۳ — تست اتصال

روی **Test Connection** کلیک کنید. ویزارد یک فراخوانی API سبک برای تأیید اعتبارنامه‌ها و اتصال ارسال می‌کند. قبل از ادامه، هرگونه مشکل در اعتبارنامه‌ها را برطرف کنید.

### گام ۴ — وارد کردن TLDها

برای دریافت تمام TLDهای موجود و قیمت عمده‌فروشی از ارائه‌دهنده متصل، روی **Import TLDs** کلیک کنید. این کار لیست TLD مورد استفاده در محصولات دامنه را پر می‌کند. این عملیات ممکن است برای ارائه‌دهندگانی با کاتالوگ TLD بزرگ، ۳۰ تا ۶۰ ثانیه طول بکشد.

TLDها همچنین روزی یک بار به صورت خودکار از طریق یک cron job زمان‌بندی شده، همگام‌سازی (re-sync) می‌شوند.

### گام ۵ — ایجاد یک محصول دامنه

ویزارد یک محصول دامنه پیش‌فرض با حاشیه سود ۱۰٪ ایجاد می‌کند. می‌توانید بلافاصله این محصول را ویرایش کنید یا آن را نادیده بگیرید و محصولات را به صورت دستی در مسیر **Ultimate Multisite › Products** ایجاد نمایید.

برای راهنمای کامل پیکربندی محصولات، به [Domain Products and Pricing](./domain-products) مراجعه کنید.

## پیکربندی مجدد یک ارائه‌دهنده

به مسیر **Network Admin › Ultimate Multisite › Settings › Domain Seller** بروید (یا در لیست پلاگین، روی **Settings** کلیک کنید).

صفحه تنظیمات شامل موارد زیر است:

- **Enable domain selling** — فعال/غیرفعال کردن کل قابلیت
- **Default provider** — ارائه‌دهنده‌ای که برای جستجوی دامنه و محصولات جدید استفاده می‌شود
- **Max TLDs per search** — تعداد TLDهایی که هنگام جستجوی مشتری بررسی می‌شود؛ مقادیر بالاتر گزینه‌های بیشتری نشان می‌دهند اما کندتر هستند
- **Availability cache duration** — مدت زمانی که نتایج موجود بودن و قیمت‌گذاری در حافظه پنهان (cache) ذخیره می‌شود؛ مقادیر پایین‌تر دقیق‌تر هستند اما فراخوانی API را افزایش می‌دهند
- **Manage domain products** — لینک سریع به لیست محصولات
- **Configure providers** — باز کردن ویزارد Integration برای افزودن یا پیکربندی مجدد ارائه‌دهندگان

### افزودن ارائه‌دهنده دوم

روی **Configure providers** کلیک کرده و ویزارد را برای رجیسترار جدید دوباره اجرا کنید. شما می‌توانید چندین ارائه‌دهنده را به طور همزمان پیکربندی کنید. هر محصول دامنه را به یک ارائه‌دهنده خاص اختصاص دهید یا آن را روی پیش‌فرض بگذارید.

### همگام‌سازی دستی TLDها

در صفحه تنظیمات، کنار هر ارائه‌دهنده پیکربندی شده، روی **Sync TLDs** کلیک کنید تا آخرین قیمت‌ها را دریافت نمایید. این کار پس از اینکه یک ارائه‌دهنده قیمت عمده‌فروشی خود را به‌روز کرد یا TLDهای جدیدی اضافه نمود، مفید است.

## لاگ‌ها (Logs)

هر ارائه‌دهنده در کانال لاگ مخصوص به خود می‌نویسد. لاگ‌ها در مسیر **Network Admin › Ultimate Multisite › Logs** قابل مشاهده هستند:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | تمام تلاش‌های ثبت (موفق و ناموفق) |
| `domain-seller-renewal` | نتایج وظیفه تمدید (Renewal job) |
| `domain-seller-opensrs` | فعالیت خام API OpenSRS |
| `domain-seller-namecheap` | فعالیت خام API Namecheap |
| `domain-seller-godaddy` | فعالیت خام API GoDaddy |
| `domain-seller-resellerclub` | فعالیت خام API ResellerClub |
| `domain-seller-namesilo` | فعالیت خام API NameSilo |
| `domain-seller-enom` | فعالیت خام API Enom |
