# مدیریت فیلد تسویه حساب و DNS مشتری

## فیلد انتخاب دامنه

فیلد **انتخاب دامنه** یک عنصر تسویه حساب است که به مشتریان این امکان را می‌دهد که چگونگی دریافت دامنه سایت خود را انتخاب کنند. با افزودن این فیلد به هر فرم تسویه حسابی، می‌توانید فروش دامنه را فعال کنید.

### افزودن فیلد به فرم تسویه حساب

۱. به **Network Admin › Ultimate Multisite › Checkout Forms** بروید.
۲. یک فرم تسویه حساب را باز یا ایجاد کنید.
۳. در ویرایشگر تسویه حساب، روی **Add Field** کلیک کنید.
۴. **Domain Selection** را از لیست فیلدها انتخاب کنید.
۵. گزینه‌های فیلد را پیکربندی کنید (در زیر ببینید).
۶. فرم را ذخیره کنید.

### گزینه‌های فیلد

**حالت‌های دامنه (Domain modes)** — انتخاب کنید که مشتری کدام تب‌ها را مشاهده کند. هر حالت را می‌توان به صورت مستقل فعال یا غیرفعال کرد:

| حالت | عملکرد |
|---|---|
| **Subdomain** | مشتری از یک زیردامنه رایگان در شبکه شما استفاده می‌کند (مثلاً `mysite.yournetwork.com`). نیازی به پرداخت نیست. |
| **Register New Domain** | مشتری یک دامنه جدید جستجو کرده و آن را از طریق ارائه‌دهنده پیکربندی شده شما ثبت می‌کند. برای قیمت‌گذاری، از محصول دامنه مطابقت‌دهنده استفاده می‌شود. |
| **Existing Domain** | مشتری دامنه‌ای را که از قبل مالک آن است، نگاشت (map) می‌کند. هزینه ثبت وجود ندارد. دامنه به طور خودکار به سایت او نگاشت می‌شود. |

**حالت پیش‌فرض (Default mode)** — هنگامی که هر سه حالت فعال باشند، کدام تب ابتدا باز می‌شود. اگر می‌خواهید ثبت دامنه اختیاری بماند، آن را روی **Subdomain** تنظیم کنید، یا اگر می‌خواهید خرید را تشویق کنید، آن را روی **Register New Domain** تنظیم نمایید.

**محصول دامنه (Domain product)** — به صورت اختیاری، این فیلد را به یک محصول دامنه خاص متصل (pin) کنید. اگر تنظیم نشود، افزونه به طور خودکار محصول مطابقت‌دهنده را بر اساس TLD که مشتری جستجو می‌کند، انتخاب می‌کند.

### فیلدهای تماس ثبت‌کننده (Registrant contact fields)

هنگامی که مشتری تب **Register New Domain** را انتخاب می‌کند، فرم تسویه حساب به صورت درون‌خطی فیلدهای تماس ثبت‌کننده را اضافه می‌کند:

- نام کوچک / نام خانوادگی
- آدرس ایمیل
- آدرس (خط ۱، شهر، ایالت/استان، کد پستی، کشور)
- شماره تلفن

این فیلدها توسط همه ثبت‌کنندگان الزامی هستند و قبل از فراخوانی API ثبت، اعتبارسنجی می‌شوند. شماره تلفن‌ها به طور خودکار به فرمت بین‌المللی `+CC.NNN` که توسط ثبت‌کنندگان مورد انتظار است، قالب‌بندی می‌شوند.

### آدرس URL سایت تولید شده خودکار

هنگامی که مشتری دامنه‌ای را ثبت یا نگاشت می‌کند، فیلد آدرس URL سایت به طور خودکار از دامنه انتخاب شده پر می‌شود. مشتریان نیازی به پر کردن فیلد URL جداگانه ندارند.

### رفتار جستجو

- در حین تایپ کردن توسط مشتری، در دسترس بودن دامنه به صورت بلادرنگ با AJAX بررسی می‌شود.
- در صورت عدم در دسترس بودن دامنه ترجیحی، پیشنهادهای TLD جایگزین نمایش داده می‌شود.
- قیمت‌گذاری به صورت زنده دریافت و به وضوح نمایش داده می‌شود (قیمت ثبت، قیمت تمدید، هزینه اختیاری حریم خصوصی WHOIS).
- کوپن‌ها برای محصولات دامنه به همان شیوه‌ای اعمال می‌شوند که برای هر محصول دیگری اعمال می‌شوند.

**تنظیم پاسخگویی جستجو:**

```php
// افزایش تأخیر دباوباک (Debounce delay) (میلی‌ثانیه) برای کاهش فراخوانی‌های API در اتصالات کند
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // پیش‌فرض: 500
});
```

**افزودن فیلدهای سفارشی به فرم جستجوی دامنه:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## مدیریت DNS مشتری

مشتریان می‌توانند رکوردهای DNS دامنه‌های ثبت شده خود را از صفحه **My Account**، زیر ورودی دامنه خود، مدیریت کنند.

### انواع رکورد پشتیبانی شده

| نوع | کاربرد |
|---|---|
| **A** | نگاشت نام میزبان به آدرس IPv4 |
| **AAAA** | نگاشت نام میزبان به آدرس IPv6 |
| **CNAME** | ایجاد یک نام مستعار که به نام میزبان دیگری اشاره می‌کند |
| **MX** | تنظیم سرور مبادله ایمیل |
| **TXT** | افزودن رکوردهای متنی SPF، DMARC، تأیید یا سایر موارد |

### کدام ارائه‌دهندگان مدیریت DNS را پشتیبانی می‌کنند؟

مدیریت DNS (افزودن، ویرایش، حذف رکورد) با **OpenSRS**، **ResellerClub** و **Enom** در دسترس است. دامنه‌های Namecheap، GoDaddy و NameSilo وضعیت و اطلاعات انقضا را نمایش می‌دهند، اما DNS باید مستقیماً در پنل کنترل ثبت‌کننده مدیریت شود.

### رکوردهای DNS پیش‌فرض

می‌توانید رکوردهای DNS پیش‌فرضی را پیکربندی کنید که هنگام ثبت دامنه به طور خودکار اعمال می‌شوند. به **Settings › Domain Seller › Default DNS Records** بروید.

مقادیر رکورد پیش‌فرض از دو توکن پشتیبانی می‌کنند:

| توکن | جایگزین می‌شود با |
|---|---|
| `{DOMAIN}` | نام دامنه ثبت شده (مثلاً `example.com`) |
| `{SITE_URL}` | آدرس URL سایت وردپرس برای سایت مشتری |

**مثال — اشاره کردن دامنه اصلی و www به آدرس IP سرور شما:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### مدیر: مشاهده و ویرایش DNS

مدیران شبکه می‌توانند رکوردهای DNS را برای هر دامنه مشتری از صفحه ویرایش دامنه در **Network Admin › Ultimate Multisite › Domains** مشاهده و ویرایش کنند.
