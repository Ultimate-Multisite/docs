# فیلد پرداخت و مدیریت DNS مشتری

## فیلد پرداخت انتخاب دامنه

فیلد **انتخاب دامنه** یک عنصر پرداخت است که به مشتریان امکان می‌دهد انتخاب کنند دامنهٔ سایتشان را چگونه دریافت کنند. آن را به هر فرم پرداختی اضافه کنید تا فروش دامنه فعال شود.

### افزودن فیلد به فرم پرداخت

1. به **مدیریت شبکه › Ultimate Multisite › فرم‌های پرداخت** بروید
2. یک فرم پرداخت را باز کنید یا بسازید
3. در ویرایشگر پرداخت، روی **افزودن فیلد** کلیک کنید
4. از فهرست فیلدها **انتخاب دامنه** را انتخاب کنید
5. گزینه‌های فیلد را پیکربندی کنید (پایین را ببینید)
6. فرم را ذخیره کنید

### گزینه‌های فیلد

**حالت‌های دامنه** — انتخاب کنید مشتری کدام تب‌ها را ببیند. هر حالت می‌تواند به‌طور مستقل فعال یا غیرفعال شود:

| حالت | چه کاری انجام می‌دهد |
|---|---|
| **زیردامنه** | مشتری از یک زیردامنهٔ رایگان روی شبکهٔ شما استفاده می‌کند (مثلاً `mysite.yournetwork.com`). نیازی به پرداخت نیست. |
| **ثبت دامنهٔ جدید** | مشتری یک دامنهٔ جدید را جست‌وجو می‌کند و آن را از طریق ارائه‌دهندهٔ پیکربندی‌شدهٔ شما ثبت می‌کند. برای قیمت‌گذاری از محصول دامنهٔ منطبق استفاده می‌کند. |
| **دامنهٔ موجود** | مشتری دامنه‌ای را که از قبل مالک آن است نگاشت می‌کند. هزینهٔ ثبتی وجود ندارد. دامنه به‌طور خودکار به سایت او نگاشت می‌شود. |

**حالت پیش‌فرض** — وقتی هر سه حالت فعال هستند، کدام تب ابتدا باز شود. برای اختیاری نگه داشتن ثبت دامنه، روی **زیردامنه** تنظیم کنید، یا برای تشویق خریدها، روی **ثبت دامنهٔ جدید** تنظیم کنید.

**محصول دامنه** — در صورت تمایل این فیلد را به یک محصول دامنهٔ مشخص سنجاق کنید. اگر تنظیم نشود، افزونه به‌طور خودکار بر اساس TLDی که مشتری جست‌وجو می‌کند، محصول منطبق را انتخاب می‌کند.

### فیلدهای تماس ثبت‌کننده

وقتی مشتری تب **ثبت دامنهٔ جدید** را انتخاب می‌کند، فرم پرداخت فیلدهای تماس ثبت‌کننده را به‌صورت درون‌خطی اضافه می‌کند:

- نام / نام خانوادگی
- آدرس ایمیل
- نشانی (خط ۱، شهر، ایالت/استان، کد پستی، کشور)
- شماره تلفن

این موارد توسط همهٔ ثبت‌کننده‌ها الزامی هستند و پیش از انجام فراخوانی API ثبت اعتبارسنجی می‌شوند. شماره‌های تلفن به‌طور خودکار به قالب بین‌المللی `+CC.NNN` که ثبت‌کننده‌ها انتظار دارند، قالب‌بندی می‌شوند.

### URL سایت تولیدشده به‌صورت خودکار

وقتی مشتری دامنه‌ای را ثبت یا نگاشت می‌کند، فیلد URL سایت به‌طور خودکار از دامنهٔ انتخاب‌شده پر می‌شود. مشتریان نیازی ندارند یک فیلد URL جداگانه را پر کنند.

### رفتار جست‌وجو

- در دسترس بودن دامنه هم‌زمان با تایپ مشتری، به‌صورت لحظه‌ای با AJAX بررسی می‌شود
- وقتی دامنهٔ ترجیحی در دسترس نباشد، پیشنهادهای TLD جایگزین نمایش داده می‌شوند
- قیمت‌گذاری به‌صورت زنده دریافت می‌شود و به‌وضوح نمایش داده می‌شود (قیمت ثبت، قیمت تمدید، هزینهٔ اختیاری حریم خصوصی WHOIS)
- کدهای کوپن برای محصولات دامنه درست مانند هر محصول دیگری اعمال می‌شوند

**تنظیم پاسخ‌گویی جست‌وجو:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**افزودن فیلدهای سفارشی به فرم جست‌وجوی دامنه:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## مدیریت DNS مشتری

مشتریان می‌توانند رکوردهای DNS دامنه‌های ثبت‌شدهٔ خود را از صفحهٔ **My Account**، زیر ورودی دامنهٔ خود مدیریت کنند.

### انواع رکورد پشتیبانی‌شده

| نوع | کاربرد |
|---|---|
| **A** | نگاشت نام میزبان به آدرس IPv4 |
| **AAAA** | نگاشت نام میزبان به آدرس IPv6 |
| **CNAME** | ایجاد یک نام مستعار که به نام میزبان دیگری اشاره می‌کند |
| **MX** | تنظیم سرور تبادل ایمیل |
| **TXT** | افزودن SPF، DMARC، تأییدیه یا رکوردهای متنی دیگر |

### کدام ارائه‌دهنده‌ها از مدیریت DNS پشتیبانی می‌کنند؟

مدیریت DNS (افزودن، ویرایش، حذف رکوردها) با **OpenSRS**، **ResellerClub**، **Enom**، **HostAfrica** و **Openprovider** در دسترس است. دامنه‌های **Hostinger** می‌توانند nameserverها را از طریق Domain Seller به‌روزرسانی کنند؛ رکوردهای DNS برای دامنه‌های میزبانی‌شده توسط ادغام اصلی نگاشت دامنهٔ Hostinger مدیریت می‌شوند. دامنه‌های Namecheap، GoDaddy و NameSilo وضعیت و اطلاعات انقضا را نمایش می‌دهند، اما DNS باید مستقیماً در پنل کنترل ثبت‌کننده مدیریت شود.

### رکوردهای DNS پیش‌فرض

می‌توانید رکوردهای DNS پیش‌فرضی را پیکربندی کنید که هنگام ثبت دامنه به‌طور خودکار اعمال شوند. به **تنظیمات › Domain Seller › رکوردهای DNS پیش‌فرض** بروید.

مقادیر رکورد پیش‌فرض از دو توکن پشتیبانی می‌کنند:

| توکن | جایگزین می‌شود با |
|---|---|
| `{DOMAIN}` | نام دامنهٔ ثبت‌شده (مثلاً `example.com`) |
| `{SITE_URL}` | URL سایت WordPress برای سایت مشتری |

**مثال — اشاره دادن دامنهٔ ریشه و www به IP سرور شما:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### مدیر: مشاهده و ویرایش DNS

مدیران شبکه می‌توانند رکوردهای DNS هر دامنهٔ مشتری را از صفحهٔ ویرایش دامنه در **مدیریت شبکه › Ultimate Multisite › دامنه‌ها** مشاهده و ویرایش کنند.
