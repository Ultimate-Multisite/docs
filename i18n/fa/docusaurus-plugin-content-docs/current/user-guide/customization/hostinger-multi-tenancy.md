# قابلیت چند مستأجری در هاستینگر

Ultimate Multisite: Multi-Tenancy 1.2.0 یک قابلیت چند مستأجری هاستینگر را اضافه می‌کند تا دامنه‌های میزبانی بتوانند در کنار ادغام نگاشت دامنه موجود هاستینگر، در فرآیند فراهم کردن مستأجران حاکمیتی (sovereign tenant provisioning) شرکت کنند.

از این قابلیت زمانی استفاده کنید که دامنه‌های مستأجر و زیرساخت‌های مجزای مستأجر از طریق پنل hPanel هاستینگر مدیریت می‌شوند.

## نکات تنظیمات {#setup-notes}

۱. ادغام اصلی هاستینگر را در بخش **Ultimate Multisite > Settings > Host Integrations** پیکربندی کنید.
۲. تأیید کنید که توکن API هاستینگر می‌تواند دامنه یا زیردامنه هدف را مدیریت کند.
۳. افزونه چند مستأجری (Multi-Tenancy addon) را فعال کنید.
۴. استراتژی جداسازی مستأجر را قبل از انتشار مستأجر پیکربندی کنید.
۵. گردش کار تأیید مهاجرت (migration verification workflow) را قبل از ارسال ترافیک تولید به مستأجر اجرا کنید.

قابلیت هاستینگر از اتصال مشترک هاستینگر برای عملیات سمت میزبان استفاده می‌کند. همچنان باید DNS به حساب هاستینگر صحیح اشاره کند و محدودیت‌های حساب hPanel اعمال می‌شود.

## تغییرات خاص قابلیت‌ها {#capability-specific-changes}

- می‌توان مستأجران حاکمیتی را با عملیات دامنه آگاه از میزبان ایجاد کرد.
- رشته‌های میزبان دیتابیس روی یک ماشین، قبل از اعتبارسنجی دسترسی نرمال‌سازی می‌شوند.
- مستأجرانی که توسط هاستینگر مدیریت می‌شوند باید از مقدار میزبان دیتابیس نشان داده شده در hPanel استفاده کنند، مگر اینکه محیط وردپرس نیاز به بازنویسی محلی داشته باشد.
- بازدیدهای SSO بر روی حل شدن دامنه مستأجر به مستأجر میزبانی شده توسط هاستینگر متکی هستند.

## عیب‌یابی مستأجران هاستینگر {#troubleshooting-hostinger-tenants}

- اگر نصب یک مستأجر با شکست مواجه شد، تأیید کنید که دامنه از قبل به حساب هاستینگر متصل شده است.
- اگر اعتبارسنجی دیتابیس با شکست مواجه شد، نام کاربری دیتابیس، نام دیتابیس و اتصال میزبان را با hPanel مقایسه کنید.
- اگر **بازدید (SSO)** با شکست مواجه شد، تأیید کنید که DNS و SSL برای دامنه مستأجر فعال هستند.
- اگر حذف کردن (teardown) منابع میزبان باقی‌مانده‌ای را به جا می‌گذارد، پس از تأیید پشتیبان‌ها، هر دیتابیس، کاربر یا پوشه باقی‌مانده در hPanel را حذف کنید.
