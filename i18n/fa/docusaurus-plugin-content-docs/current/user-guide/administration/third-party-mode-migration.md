# مهاجرت به حالت شخص ثالث {#third-party-mode-migration}

Superdav AI Agent v1.12.0 نحوه مدیریت قابلیت‌های شخص ثالث را تغییر داده است. **حالت شخص ثالث اکنون به طور پیش‌فرض روی خودکار (auto) تنظیم شده است**، که این امر امکان ادغام با Abilities API بومی وردپرس را در وردپرس ۷.۰+ بدون نیاز به تنظیمات دستی فراهم می‌کند.

## چه چیزی تغییر کرده است؟ {#what-changed}

### قبل از v1.12.0 {#before-v1120}

قابلیت‌های شخص ثالث نیاز به تنظیمات دستی داشتند:

- شما باید به صراحت "حالت شخص ثالث" را فعال می‌کردید.
- قابلیت‌ها از یک رجیستری (ثبت‌کننده) سفارشی بارگذاری می‌شدند.
- ادغام با Abilities API وردپرس اختیاری بود.
- حالت قدیمی (Legacy mode) پیش‌فرض بود.

### بعد از v1.12.0 {#after-v1120}

قابلیت‌های شخص ثالث به صورت خودکار کار می‌کنند:

- حالت شخص ثالث به طور پیش‌فرض روی "auto" تنظیم می‌شود.
- قابلیت‌ها به صورت بومی با Abilities API وردپرس ادغام می‌شوند.
- در وردپرس ۷.۰+ نیازی به تنظیمات دستی نیست.
- حالت قدیمی (Legacy mode) همچنان برای نسخه‌های قدیمی‌تر وردپرس در دسترس است.

## چه کسانی تحت تأثیر قرار می‌گیرند؟ {#who-is-affected}

### نصب‌های جدید (وردپرس ۷.۰+) {#new-installations-wordpress-70}

**نیازی به اقدام نیست.** حالت شخص ثالث به طور خودکار روی "auto" تنظیم شده و قابلیت‌ها آماده استفاده هستند.

### نصب‌های موجود {#existing-installations}

**تنظیمات شما حفظ می‌شود.** اگر از موارد زیر استفاده می‌کردید:

- **حالت قدیمی (Legacy mode)**: شما در حالت قدیمی باقی می‌مانید (تغییری ایجاد نمی‌شود)
- **حالت شخص ثالث دستی (Manual third-party mode)**: شما در حالت دستی باقی می‌مانید (تغییری ایجاد نمی‌شود)
- **حالت خودکار (Auto mode)**: شما با حالت خودکار ادامه می‌دهید (تغییری ایجاد نمی‌شود)

### نسخه‌های وردپرس قبل از ۷.۰ {#wordpress-versions-before-70}

**حالت قدیمی (Legacy mode) همچنان در دسترس است.** اگر از وردپرس ۶.x یا نسخه‌های قدیمی‌تر استفاده می‌کنید:

- حالت شخص ثالث به طور پیش‌فرض روی "legacy" تنظیم می‌شود.
- در صورت تمایل می‌توانید حالت شخص ثالث را به صورت دستی فعال کنید.
- برای استفاده از Abilities API بومی، به وردپرس ۷.۰+ ارتقا دهید.

## درک حالت‌ها {#understanding-the-modes}

### حالت خودکار (Auto Mode) (پیش‌فرض جدید) {#auto-mode-new-default}

**حالت خودکار (Auto mode)** از ادغام Abilities API بومی وردپرس استفاده می‌کند:

- قابلیت‌ها از طریق هوک‌های وردپرس (WordPress hooks) ثبت می‌شوند.
- سازگاری کامل با Abilities API وردپرس ۷.۰+
- کشف خودکار قابلیت‌های شخص ثالث
- نیازی به تنظیمات دستی نیست

**چه زمانی استفاده شود**: وردپرس ۷.۰+ با قابلیت‌های شخص ثالث

### حالت دستی (Manual Mode) {#manual-mode}

**حالت دستی (Manual mode)** نیاز به تنظیمات صریح دارد:

- شما مشخص می‌کنید که کدام قابلیت‌های شخص ثالث باید بارگذاری شوند.
- برای تست یا بارگذاری انتخابی قابلیت مفید است.
- نیاز به ویرایش فایل‌های پیکربندی دارد.
- کنترل بیشتری دارد، اما راه‌اندازی بیشتری نیاز دارد.

**چه زمانی استفاده شود**: تست، بارگذاری انتخابی قابلیت، یا پیکربندی‌های سفارشی

### حالت قدیمی (Legacy Mode) {#legacy-mode}

**حالت قدیمی (Legacy mode)** از سیستم قدیمی قابلیت شخص ثالث استفاده می‌کند:

- رجیستری قابلیت سفارشی (نه Abilities API وردپرس)
- سازگار با نسخه‌های قدیمی‌تر وردپرس است.
- ادغام بومی با وردپرس ندارد.
- منسوخ شده اما همچنان پشتیبانی می‌شود.

**چه زمانی استفاده شود**: وردپرس ۶.x یا قدیمی‌تر، یا زمانی که نیاز به سازگاری با حالت قدیمی دارید.

## بررسی حالت فعلی شما {#checking-your-current-mode}

### از طریق پنل مدیریت {#via-admin-panel}

۱. به **پنل مدیریت وردپرس** → **Superdav AI Agent** → **تنظیمات** بروید.
۲. دنبال تنظیم **Third-Party Mode** بگردید.
۳. حالت فعلی و گزینه‌های تغییر آن را مشاهده خواهید کرد.

### از طریق کد {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', یا 'legacy'
```

## تغییر حالت شما {#changing-your-mode}

### تغییر به حالت خودکار (Auto Mode) {#switch-to-auto-mode}

اگر از وردپرس ۷.۰+ استفاده می‌کنید و می‌خواهید از حالت خودکار استفاده کنید:

۱. به **Superdav AI Agent** → **تنظیمات** بروید.
۲. **Third-Party Mode** را پیدا کنید.
۳. **Auto (WordPress Abilities API)** را انتخاب کنید.
۴. روی **ذخیره (Save)** کلیک کنید.

Superdav AI Agent به طور خودکار قابلیت‌های شخص ثالث را کشف و ثبت می‌کند.

### تغییر به حالت دستی (Manual Mode) {#switch-to-manual-mode}

اگر می‌خواهید کنترل دستی داشته باشید که کدام قابلیت‌ها بارگذاری شوند:

۱. به **Superdav AI Agent** → **تنظیمات** بروید.
۲. **Third-Party Mode** را پیدا کنید.
۳. **Manual** را انتخاب کنید.
۴. روی **ذخیره (Save)** کلیک کنید.
۵. فایل پیکربندی خود را ویرایش کنید تا مشخص کنید کدام قابلیت‌ها باید بارگذاری شوند.

### تغییر به حالت قدیمی (Legacy Mode) {#switch-to-legacy-mode}

اگر نیاز به سازگاری با حالت قدیمی دارید:

۱. به **Superdav AI Agent** → **تنظیمات** بروید.
۲. **Third-Party Mode** را پیدا کنید.
۳. **Legacy** را انتخاب کنید.
۴. روی **ذخیره (Save)** کلیک کنید.

## مزایای حالت خودکار (Auto Mode) {#benefits-of-auto-mode}

### کشف خودکار {#automatic-discovery}

قابلیت‌ها به طور خودکار از منابع زیر کشف می‌شوند:

- پلاگین‌های نصب شده
- قالب فعال
- پلاگین‌های ضروری (Must-use plugins)
- پلاگین‌های Drop-in

نیازی به ثبت دستی نیست.

### ادغام بومی (Native Integration) {#native-integration}

قابلیت‌ها با Abilities API وردپرس ادغام می‌شوند:

- سازگار با هسته وردپرس
- با پنل مدیریت وردپرس کار می‌کند
- با سایر پلاگین‌هایی که از Abilities API استفاده می‌کنند، سازگار است.
- با تکامل وردپرس، آینده‌نگر است.

### مدیریت ساده شده {#simplified-management}

- فایل‌های پیکربندی برای ویرایش وجود ندارد.
- نیازی به ثبت دستی قابلیت نیست.
- کنترل‌های دیده‌شدن قابلیت (Ability Visibility) به طور خودکار کار می‌کنند.
- اعلان‌های پنل مدیریت شما را در مورد قابلیت‌های طبقه‌بندی نشده مطلع می‌کنند.

### عملکرد بهتر {#better-performance}

- قابلیت‌ها کش می‌شوند (cached).
- در صورت نیاز بارگذاری تنبل (Lazy-loaded) می‌شوند.
- برای وردپرس ۷.۰+ بهینه شده‌اند.

## مسیر مهاجرت {#migration-path}

### اگر از وردپرس ۶.x استفاده می‌کنید {#if-youre-on-wordpress-6x}

۱. **به وردپرس ۷.۰+ ارتقا دهید** (هنگامی که آماده بودید).
۲. **Superdav AI Agent** را به v1.12.0+ به‌روز کنید.
۳. **حالت شخص ثالث را به Auto تغییر دهید** (اختیاری؛ حالت قدیمی همچنان کار می‌کند).
۴. **دیده‌شدن قابلیت‌ها (ability visibility)** را بررسی کنید تا مطمئن شوید کنترل‌های دسترسی مناسب هستند.

### اگر از وردپرس ۷.۰+ استفاده می‌کنید {#if-youre-on-wordpress-70}

۱. **Superdav AI Agent** را به v1.12.0+ به‌روز کنید.
۲. **تأیید کنید که حالت شخص ثالث روی Auto تنظیم شده است** (باید به طور پیش‌فرض این‌گونه باشد).
۳. **دیده‌شدن قابلیت‌ها (ability visibility)** را بررسی کنید تا مطمئن شوید کنترل‌های دسترسی مناسب هستند.
۴. **قابلیت‌های شخص ثالث** را تست کنید تا مطمئن شوید کار می‌کنند.

## عیب‌یابی {#troubleshooting}

### قابلیت‌ها در حالت خودکار بارگذاری نمی‌شوند {#abilities-arent-loading-in-auto-mode}

- تأیید کنید که از وردپرس ۷.۰+ استفاده می‌کنید.
- بررسی کنید که حالت شخص ثالث روی "Auto" تنظیم شده باشد.
- تأیید کنید که پلاگین ارائه‌دهنده قابلیت فعال است.
- لاگ‌های خطای وردپرس را برای خطاهای ثبت بررسی کنید.

### می‌خواهم حالت قدیمی (Legacy mode) را حفظ کنم {#i-want-to-keep-legacy-mode}

- به **تنظیمات** → **Third-Party Mode** بروید.
- **Legacy** را انتخاب کنید.
- روی **ذخیره (Save)** کلیک کنید.
- حالت قدیمی همچنان کار خواهد کرد.

### قابلیت‌های سفارشی من نمایش داده نمی‌شوند {#my-custom-abilities-arent-showing}

- تأیید کنید که آن‌ها از طریق هوک‌های وردپرس ثبت شده‌اند.
- بررسی کنید که آن‌ها Abilities API را به درستی پیاده‌سازی می‌کنند.
- لاگ‌های خطای وردپرس را بررسی کنید.
- از صفحه مدیریت **Ability Visibility** برای مشاهده تمام قابلیت‌های ثبت شده استفاده کنید.

### پیامی با عنوان "قابلیت طبقه‌بندی نشده" دریافت می‌کنم {#im-getting-unclassified-ability-notices}

- این برای قابلیت‌های شخص ثالث جدید طبیعی است.
- آن‌ها را در اعلان مدیریت بررسی و طبقه‌بندی کنید.
- برای جزئیات طبقه‌بندی به **Ability Visibility** مراجعه کنید.

## سازگاری با حالت قدیمی (Backward Compatibility) {#backward-compatibility}

### پیکربندی‌های موجود {#existing-configurations}

اگر پیکربندی‌های قابلیت شخص ثالث موجود دارید:

- **حالت قدیمی (Legacy mode)**: پیکربندی شما همچنان کار می‌کند.
- **حالت دستی (Manual mode)**: پیکربندی شما همچنان کار می‌کند.
- **حالت خودکار (Auto mode)**: پیکربندی شما نادیده گرفته می‌شود (حالت خودکار کنترل را به دست می‌گیرد).

برای حفظ پیکربندی سفارشی خود، در حالت دستی یا حالت قدیمی بمانید.

### جدول زمانی منسوخ شدن (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: حالت‌های قدیمی و دستی همچنان کاملاً پشتیبانی می‌شوند.
- **v1.13.0+**: ممکن است حالت قدیمی، اعلان‌های منسوخ شدن نشان دهد.
- **v2.0.0**: ممکن است حالت قدیمی حذف شود (به اطلاع ثانوی خواهد رسید).

## بهترین شیوه‌ها {#best-practices}

### برای نصب‌های جدید {#for-new-installations}

- از حالت خودکار (Auto mode) استفاده کنید (پیش‌فرض است).
- اجازه دهید Superdav AI Agent قابلیت‌ها را به طور خودکار کشف کند.
- از Ability Visibility برای کنترل دسترسی استفاده کنید.

### برای نصب‌های موجود {#for-existing-installations}

- در صورت امکان به وردپرس ۷.۰+ ارتقا دهید.
- برای مدیریت ساده‌تر، به حالت خودکار تغییر دهید.
- قابلیت‌ها را با استفاده از Ability Visibility بررسی و طبقه‌بندی کنید.

### برای قابلیت‌های سفارشی {#for-custom-abilities}

- قابلیت‌ها را از طریق هوک‌های وردپرس (Abilities API) ثبت کنید.
- از رجیستری‌های قابلیت سفارشی اجتناب کنید.
- در وردپرس ۷.۰+ با حالت خودکار تست کنید.

## مراحل بعدی {#next-steps}

۱. **نسخه وردپرس خود را بررسی کنید**: تأیید کنید که برای حالت خودکار، از ۷.۰+ استفاده می‌کنید.
۲. **حالت شخص ثالث خود را بررسی کنید**: به تنظیمات بروید و حالت فعلی خود را چک کنید.
۳. **در صورت نیاز به‌روزرسانی کنید**: اگر از وردپرس ۷.۰+ استفاده می‌کنید، به حالت خودکار تغییر دهید.
۴. **قابلیت‌ها را طبقه‌بندی کنید**: هر قابلیت طبقه‌بندی نشده را بررسی و طبقه‌بندی کنید.
۵. **تست کنید**: تأیید کنید که قابلیت‌های شخص ثالث شما به درستی کار می‌کنند.
