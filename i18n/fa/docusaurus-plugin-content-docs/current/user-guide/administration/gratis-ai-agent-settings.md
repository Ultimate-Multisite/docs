# تنظیمات Agent هوش مصنوعی Gratis

صفحه **تنظیمات → پیشرفته (Advanced)** در Gratis AI Agent، تنظیمات سطح مدیر (administrator-level) را برای ادغام‌های بک‌اند (backend integrations) که از نسخه v1.5.0 معرفی شده‌اند، فراهم می‌کند. این صفحه مستندات فیلدهای **نقطه پایانی بازخورد (Feedback Endpoint)** و فرمت مورد انتظار آن‌ها را ارائه می‌دهد.

## دسترسی به تنظیمات

۱. در پنل مدیریت WordPress، به **Gratis AI Agent → تنظیمات (Settings)** بروید.
۲. تب **پیشرفته (Advanced)** را کلیک کنید.

## پیکربندی نقطه پایانی بازخورد (Feedback Endpoint)

نقطه پایانی بازخورد، درخواست‌های POST را از Agent هوش مصنوعی دریافت می‌کند، هر زمان که کاربری از دکمه شست رو به پایین، بنر پرامپت خودکار، یا دستور `/report-issue` بازخورد ارسال کند.

| فیلد | توضیحات |
|---|---|
| **Feedback Endpoint URL** | آدرسی که درخواست‌های بازخورد را به صورت درخواست‌های HTTP POST با بدنه JSON دریافت می‌کند. |
| **Feedback API Key** | یک توکن حامل (bearer token) که در هدر `Authorization` هر درخواست بازخورد ارسال می‌شود. اگر نقطه پایانی شما نیازی به احراز هویت ندارد، این فیلد را خالی بگذارید. |

### محتوای JSON مورد انتظار

نقطه پایانی بازخورد شما باید یک بدنه JSON با حداقل فیلدهای زیر را بپذیرد:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "پاسخ در مورد قیمت‌گذاری اشتباه بود.",
  "triage_category": "factual_error"
}
```

بسته به زمینه مکالمه، ممکن است فیلدهای اضافی در این محتوا وجود داشته باشند.

### مقادیر `triage_category`

لایه مرتب‌سازی (triage layer) هوش مصنوعی، قبل از ارسال محتوا، یکی از مقادیر زیر را به `triage_category` اختصاص می‌دهد:

| مقدار | معنی |
|---|---|
| `factual_error` | دستیار اطلاعات واقعی اشتباهی ارائه کرده است. |
| `unhelpful_answer` | پاسخ از نظر فنی درست بوده، اما مفید نبوده است. |
| `inappropriate_content` | پاسخ حاوی محتوایی بوده که نباید به کاربران نمایش داده شود. |
| `other` | بازخورد با هیچ دسته‌بندی شناخته‌شده‌ای مطابقت نداشته است. |

### احراز هویت (Authentication)

اگر نقطه پایانی شما نیاز به احراز هویت دارد، فیلد **Feedback API Key** را با توکن حامل خود تنظیم کنید. Agent موارد زیر را ارسال می‌کند:

```
Authorization: Bearer <your-api-key>
```

اگر فیلد **Feedback API Key** خالی باشد، هیچ هدر `Authorization` ارسال نخواهد شد.

### غیرفعال‌سازی جمع‌آوری بازخورد

هم فیلد **Feedback Endpoint URL** و هم **Feedback API Key** را خالی بگذارید. دکمه شست رو به پایین و رابط کاربری بازخورد برای کاربران قابل مشاهده باقی می‌مانند، اما هیچ بازخوردی به هیچ سرویس خارجی ارسال نمی‌شود.

## کلید API جستجوی Brave (Brave Search API Key)

همچنین در تب **پیشرفته (Advanced)**، فیلد **Brave Search API Key** قابلیت [جستجوی اینترنتی (Internet Search)](../configuration/internet-search) را فعال می‌کند.

| فیلد | توضیحات |
|---|---|
| **Brave Search API Key** | کلید API شما از داشبورد توسعه‌دهنده Brave Search. برای فعال کردن جستجوی اینترنتی در دستیار هوش مصنوعی مورد نیاز است. |

برچسب این فیلد شامل یک لینک قابل کلیک به صفحه ثبت نام API Brave Search است. برای غیرفعال کردن جستجوی اینترنتی، آن را خالی بگذارید.

برای مستندات کاربر نهایی در مورد این قابلیت، به [جستجوی اینترنتی (Internet Search)](../configuration/internet-search) مراجعه کنید.

## پرچم‌های قابلیت (Feature Flags)

این قابلیت که در v1.9.0 معرفی شد، تب **تنظیمات → پرچم‌های قابلیت (Feature Flags)** را فراهم می‌کند و سوئیچ‌های روشن/خاموش برای عملکردهای اختیاری ارائه می‌دهد. هر پرچم یا در سطح شبکه فعال یا غیرفعال است؛ در حال حاضر هیچ بازنویسی در سطح سایت وجود ندارد.

### دسترسی به پرچم‌های قابلیت

۱. در پنل مدیریت WordPress، به **Gratis AI Agent → تنظیمات (Settings)** بروید.
۲. تب **پرچم‌های قابلیت (Feature Flags)** را کلیک کنید.

### پرچم‌های کنترل دسترسی

| پرچم | پیش‌فرض | توضیحات |
|---|---|---|
| **Restrict to Administrators** | خاموش (Off) | هنگام فعال شدن، فقط کاربرانی با نقش `administrator` می‌توانند پنل چت Agent هوش مصنوعی را باز کنند. سایر نقش‌ها در عوض پیامی با عنوان "لطفاً با مدیر خود تماس بگیرید" مشاهده می‌کنند. |
| **Restrict to Network Admins** | خاموش (Off) | هنگام فعال شدن در یک شبکه Multisite، فقط Super Adminها می‌توانند از Agent استفاده کنند. مدیران سایت‌های فردی مسدود می‌شوند. اگر هر دو فعال باشند، بر "Restrict to Administrators" اولویت دارد. |
| **Allow Subscriber Access** | خاموش (Off) | هنگام فعال شدن، کاربرانی با نقش `subscriber` می‌توانند از رابط چت استفاده کنند، اما محدود به قابلیت‌های فقط خواندنی هستند (نمی‌توانند پست ایجاد یا تنظیمات تغییر دهند). |
| **Disable for Non-Members** | خاموش (Off) | با وضعیت عضویت Ultimate Multisite ادغام می‌شود. هنگام فعال شدن، چت برای سایت‌هایی که عضویت فعال ندارند، پنهان می‌شود. |

### پرچم‌های برندسازی (Branding Flags)

| پرچم | پیش‌فرض | توضیحات |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | خاموش (Off) | خط ذکر برند (branding attribution) که در پایین ویجت چت نمایش داده می‌شود را حذف می‌کند. برای استقرار‌های وایت‌لیبل (white-label) توصیه می‌شود. |
| **Custom Agent Name** | *(خالی)* | برچسب پیش‌فرض "Gratis AI Agent" در هدر چت و منوی مدیریت را با نام محصول خود جایگزین می‌کند. برای استفاده از پیش‌فرض، آن را خالی بگذارید. |
| **Hide Agent Picker** | خاموش (Off) | هنگام فعال شدن، کاربران نمی‌توانند بین پنج Agent داخلی سوئیچ کنند. Agent فعلی روی هر چیزی که در تنظیمات → عمومی (Settings → General) به عنوان پیش‌فرض پیکربندی شده، ثابت می‌ماند. |
| **Use Site Icon as Chat Avatar** | خاموش (Off) | آیکون پیش‌فرض AI در هدر ویجت چت را با آیکون سایت WordPress (که در نمایش → سفارشی‌سازی → هویت سایت تنظیم شده است) جایگزین می‌کند. |

### اعمال تغییرات

پس از تغییر هر پرچم، روی **ذخیره تنظیمات (Save Settings)** کلیک کنید. تغییرات بلافاصله اعمال می‌شوند — نیازی به پاک کردن کش یا فعال‌سازی مجدد پلاگین نیست.
