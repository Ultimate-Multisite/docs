# تنظیمات Gratis AI Agent

صفحهٔ **Settings → Advanced** در Gratis AI Agent پیکربندی در سطح مدیر را برای یکپارچه‌سازی‌های بک‌اند فراهم می‌کند. این صفحه ارسال بازخورد، کلیدهای ارائه‌دهندهٔ جستجو، راه‌اندازی سرویس مدیریت‌شدهٔ Superdav، کنترل‌های Google Calendar، تنظیمات پیامک TextBee، و پرچم‌های قابلیت در سطح شبکه را مستند می‌کند.

## دسترسی به تنظیمات

1. در مدیریت WordPress، به **Gratis AI Agent → Settings** بروید.
2. روی زبانهٔ **Advanced** کلیک کنید.

## پیکربندی نقطهٔ پایانی بازخورد

نقطهٔ پایانی بازخورد هر زمان که کاربر از طریق دکمهٔ شست رو به پایین، بنر اعلان خودکار، یا دستور `/report-issue` بازخوردی ارسال کند، درخواست‌های POST را از عامل هوش مصنوعی دریافت می‌کند.

| فیلد | توضیح |
|---|---|
| **نشانی نقطهٔ پایانی بازخورد** | URLی که ارسال‌های بازخورد را به‌صورت درخواست‌های HTTP POST با بدنهٔ JSON دریافت می‌کند. |
| **Feedback API Key** | یک bearer token که در `Authorization` header هر درخواست بازخورد ارسال می‌شود. اگر نقطهٔ پایانی شما نیاز به احراز هویت ندارد، خالی بگذارید. |

### محتوای JSON مورد انتظار

نقطهٔ پایانی بازخورد شما باید بدنهٔ JSONی را با دست‌کم فیلدهای زیر بپذیرد:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

بسته به زمینهٔ گفتگو، ممکن است فیلدهای دیگری نیز در محتوا وجود داشته باشند.

### مقادیر `triage_category`

لایهٔ تریاژ هوش مصنوعی پیش از ارسال محتوا، یکی از مقادیر زیر را به `triage_category` اختصاص می‌دهد:

| مقدار | معنا |
|---|---|
| `factual_error` | دستیار اطلاعات واقعی نادرستی ارائه کرده است. |
| `unhelpful_answer` | پاسخ از نظر فنی درست بوده اما مفید نبوده است. |
| `inappropriate_content` | پاسخ شامل محتوایی بوده که نباید به کاربران نمایش داده شود. |
| `other` | بازخورد با هیچ دستهٔ شناخته‌شده‌ای مطابقت نداشته است. |

### احراز هویت

اگر نقطهٔ پایانی شما نیاز به احراز هویت دارد، فیلد **Feedback API Key** را روی bearer token خود تنظیم کنید. عامل ارسال می‌کند:

```
Authorization: Bearer <your-api-key>
```

اگر فیلد **Feedback API Key** خالی باشد، هیچ `Authorization` headerی ارسال نمی‌شود.

### غیرفعال‌کردن جمع‌آوری بازخورد

هر دو فیلد **نشانی نقطهٔ پایانی بازخورد** و **Feedback API Key** را خالی بگذارید. دکمهٔ شست رو به پایین و رابط کاربری بازخورد برای کاربران قابل مشاهده می‌مانند، اما ارسال‌ها به هیچ سرویس خارجی‌ای فرستاده نمی‌شوند.

## Brave Search API Key

همچنین در زبانهٔ **Advanced**، فیلد **Brave Search API Key** قابلیت [جستجوی اینترنتی](../configuration/internet-search) را فعال می‌کند.

| فیلد | توضیح |
|---|---|
| **Brave Search API Key** | کلید API شما از داشبورد توسعه‌دهندهٔ Brave Search. برای فعال‌سازی جستجوی اینترنتی در دستیار هوش مصنوعی لازم است. |

برچسب فیلد شامل پیوندی قابل کلیک به صفحهٔ ثبت‌نام Brave Search API است. برای غیرفعال‌کردن جستجوی اینترنتی، خالی بگذارید.

برای مستندات کاربر نهایی دربارهٔ این قابلیت، [جستجوی اینترنتی](../configuration/internet-search) را ببینید.

## سرویس مدیریت‌شدهٔ Superdav

Superdav AI Agent v1.18.0 نقطه‌های پایانی سرویس مدیریت‌شدهٔ Superdav و آماده‌سازی خودکار اتصال را برای سایت‌های پشتیبانی‌شده اضافه می‌کند. زمانی از این کنترل‌ها استفاده کنید که سایت شما باید به‌جای یک نقطهٔ پایانی سرویس که به‌صورت دستی پیکربندی شده، به ارائه‌دهندهٔ میزبانی‌شده متصل شود.

| فیلد | توضیح |
|---|---|
| **سرویس مدیریت‌شدهٔ Superdav** | اتصال سرویس میزبانی‌شدهٔ Superdav را برای سایت‌های پشتیبانی‌شده فعال می‌کند. |
| **آماده‌سازی اتصال** | آماده‌سازی خودکار نقطهٔ پایانی و اعتبارنامه را آغاز می‌کند. پس از تأیید اینکه سایت باید از ارائه‌دهندهٔ مدیریت‌شده استفاده کند، از این گزینه استفاده کنید. |
| **نقطهٔ پایانی سرویس / وضعیت اتصال** | نقطهٔ پایانی یا وضعیت اتصال فعلی را پس از آماده‌سازی نشان می‌دهد. |

پس از آماده‌سازی، تنظیمات را ذخیره کنید و پیش از اتکا به گردش‌کارهای سرویس مدیریت‌شده، وضعیت اتصال را بررسی کنید. اگر آماده‌سازی ناموفق بود، راهنمایی تلاش مجددِ نمایش‌داده‌شده را مرور کنید و تأیید کنید که سایت اجازهٔ استفاده از ارائه‌دهندهٔ میزبانی‌شده را دارد.

## پیکربندی Google Calendar

وقتی قابلیت‌های تقویم Superdav AI Agent v1.18.0 فعال باشند، عامل می‌تواند تقویم‌های پیکربندی‌شده و جزئیات رویدادها را بخواند. ابزارهای تقویم خواندن‌محور هستند و برای یادآوری‌های آگاه از زمان‌بندی، پیگیری شرکت‌کنندگان، و تطبیق مخاطبان مفیدند.

| فیلد | توضیح |
|---|---|
| **اعتبارنامه‌های Google Calendar** | اعتبارنامه‌ها یا اتصال توکن لازم برای خواندن داده‌های تقویم را ذخیره می‌کند. |
| **انتخاب تقویم** | محدود می‌کند که عامل کدام تقویم‌های پیکربندی‌شده را می‌تواند بررسی کند. |
| **وضعیت اتصال تقویم** | تأیید می‌کند که آیا اعتبارنامه‌های فعلی می‌توانند تقویم‌ها و رویدادها را بخوانند یا نه. |

اعتبارنامه‌های تقویم را به تقویم‌هایی محدود نگه دارید که عامل به آن‌ها نیاز دارد. اگر وضعیت نشان‌دهندهٔ توکن منقضی‌شده است، اعتبارنامه‌ها را دوباره متصل یا چرخش دهید.

## اعلان‌های پیامکی TextBee

Superdav AI Agent v1.18.0 سرویس TextBee را به‌عنوان ارائه‌دهندهٔ پیامک برای گردش‌کارهای اعلان پیکربندی‌شده اضافه می‌کند. اعلان‌های پیامکی باید برای پیام‌های حساس یا روبه‌کاربر با دروازه‌های تأیید انسانی همراه شوند.

| فیلد | توضیح |
|---|---|
| **TextBee API Key** | درخواست‌ها به ارائه‌دهندهٔ پیامک TextBee را احراز هویت می‌کند. |
| **دستگاه / فرستندهٔ TextBee** | وقتی ارائه‌دهنده لازم بداند، فرستنده یا دستگاه TextBee مورد استفاده برای پیام‌های خروجی را انتخاب می‌کند. |
| **اعلان‌های پیامکی فعال است** | به گردش‌کارهای تأییدشده اجازه می‌دهد اعلان‌های پیام متنی ارسال کنند. برای جلوگیری از ارسال پیامک، غیرفعال بگذارید. |

یک پیام آزمایشی را فقط به شماره‌ای متعلق به مدیر ارسال کنید، سپس پیش از فعال‌سازی یادآوری‌های زمان‌بندی‌شده یا روبه‌شرکت‌کننده، رفتار دروازهٔ تأیید را تأیید کنید.

## پرچم‌های قابلیت

همچنین در v1.9.0 معرفی شده است؛ زبانهٔ **Settings → Feature Flags** کلیدهای تغییر وضعیت را برای قابلیت‌های اختیاری فراهم می‌کند. هر پرچم در سطح شبکه یا فعال است یا غیرفعال؛ در حال حاضر بازنویسی برای هر سایت به‌صورت جداگانه وجود ندارد.

### دسترسی به پرچم‌های قابلیت

1. در مدیریت WordPress، به **Gratis AI Agent → Settings** بروید.
2. روی زبانهٔ **Feature Flags** کلیک کنید.

### پرچم‌های کنترل دسترسی

| پرچم | پیش‌فرض | توضیحات |
|---|---|---|
| **محدود کردن به مدیران** | غیرفعال | وقتی فعال باشد، فقط کاربرانی با نقش `administrator` می‌توانند پنل گفت‌وگوی AI Agent را باز کنند. سایر نقش‌ها به‌جای آن پیام «با مدیر خود تماس بگیرید» را می‌بینند. |
| **محدود کردن به مدیران شبکه** | غیرفعال | وقتی در یک شبکه چندسایتی فعال باشد، فقط Super Adminها می‌توانند از agent استفاده کنند. مدیران سایت‌های جداگانه مسدود می‌شوند. اگر هر دو فعال باشند، بر «محدود کردن به مدیران» اولویت دارد. |
| **اجازه دسترسی مشترک** | غیرفعال | وقتی فعال باشد، کاربرانی با نقش `subscriber` می‌توانند از رابط گفت‌وگو استفاده کنند اما به قابلیت‌های فقط‌خواندنی محدود هستند (بدون ایجاد نوشته یا تغییر تنظیمات). |
| **غیرفعال کردن برای غیرعضوها** | غیرفعال | با وضعیت عضویت Ultimate Multisite یکپارچه می‌شود. وقتی فعال باشد، گفت‌وگو برای سایت‌هایی که عضویت فعال ندارند پنهان می‌شود. |

### پرچم‌های برندینگ

| پرچم | پیش‌فرض | توضیحات |
|---|---|---|
| **پنهان کردن پاورقی "Powered by Gratis AI Agent"** | غیرفعال | خط انتساب برندینگ را که در پایین ابزارک گفت‌وگو نمایش داده می‌شود حذف می‌کند. برای استقرارهای white-label توصیه می‌شود. |
| **نام سفارشی agent** | *(خالی)* | برچسب پیش‌فرض "Gratis AI Agent" را در سربرگ گفت‌وگو و منوی مدیریت با نام محصول خودتان جایگزین می‌کند. برای استفاده از پیش‌فرض، خالی بگذارید. |
| **پنهان کردن انتخاب‌گر agent** | غیرفعال | وقتی فعال باشد، کاربران نمی‌توانند بین پنج agent داخلی جابه‌جا شوند. agent فعلی به هر چیزی که در تنظیمات ← عمومی به‌عنوان پیش‌فرض پیکربندی شده ثابت می‌ماند. |
| **استفاده از نماد سایت به‌عنوان آواتار گفت‌وگو** | غیرفعال | نماد پیش‌فرض AI را در سربرگ ابزارک گفت‌وگو با نماد سایت WordPress (تنظیم‌شده در نمایش ← سفارشی‌سازی ← هویت سایت) جایگزین می‌کند. |

### پرچم‌های ایمنی خودکارسازی

Superdav AI Agent v1.18.0 دروازه‌های تأیید انسانی و رکوردهای یادآوری را برای اجرای امن‌تر خودکارسازی معرفی می‌کند. این کنترل‌ها بسته به بسته نصب‌شده ممکن است در پرچم‌های قابلیت یا تنظیمات پیشرفته خودکارسازی ظاهر شوند.

| پرچم | پیش‌فرض | توضیحات |
|---|---|---|
| **نیازمند تأیید انسانی** | توصیه‌شده: فعال | خودکارسازی‌های حساس را متوقف می‌کند تا یک کاربر مجاز اقدام پیشنهادی را بررسی و تأیید کند. |
| **حذف تکرار یادآوری** | فعال | یادآوری‌های ارسال‌شده را ثبت می‌کند تا تلاش‌های مجدد یا اجراهای زمان‌بندی‌شده اعلان‌های تکراری ارسال نکنند. |
| **فعال کردن ابزارهای تقویم** | غیرفعال تا زمان پیکربندی | به agent اجازه می‌دهد تقویم‌ها و رویدادهای پیکربندی‌شده Google را بخواند. |
| **فعال کردن اعلان‌های پیامکی** | غیرفعال تا زمان پیکربندی | به جریان‌های کاری تأییدشده اجازه می‌دهد پس از ذخیره شدن اعتبارنامه‌ها، اعلان‌های پیامکی TextBee ارسال کنند. |

### اعمال تغییرات

پس از تغییر هر پرچم، روی **ذخیره تنظیمات** کلیک کنید. تغییرات بلافاصله اعمال می‌شوند — نیازی به پاک‌سازی کش یا فعال‌سازی مجدد plugin نیست.
