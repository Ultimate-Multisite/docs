# Gratis AI Agent {#gratis-ai-agent}

Gratis AI Agent یک دستیار هوش مصنوعی مستقل برای وردپرس است که می‌تواند با دستورات زبان طبیعی، سایت‌های کامل وردپرسی را برنامه‌ریزی، بسازد و مدیریت کند. این دستیار مجموعه‌ای ساختاریافته از **قابلیت‌ها** (abilities) را در اختیار شما قرار می‌دهد؛ این قابلیت‌ها اقدامات مجزایی هستند که دستیار می‌تواند فراخوانی کند و طیف وسیعی از کارهای مدیریت محتوا تا سفارشی‌سازی سیستم طراحی را پوشش می‌دهند. نسخه ۱.۹.۰ قابلیت‌های تولید محتوا، بازبینی بصری صفحات از طریق اسکرین‌شات‌های سمت کلاینت، پنج پروفایل پیش‌فرض دستیار، و فِلیگ‌های ویژگی (feature flags) برای کنترل دسترسی و برندسازی را اضافه کرده است.

## ویژگی‌های کلیدی {#key-features}

- **سیستم قابلیت‌ها (Abilities system)** — یک کاتالوگ ماژولار و قابل توسعه از اقداماتی که دستیار می‌تواند روی نصب وردپرسی شما انجام دهد.
- **مدیریت محتوا (Content Management)** — ایجاد پست و صفحه (از جمله تخصیص قالب صفحه)، ایجاد دسته‌ای از پست‌ها در یک فراخوانی، تنظیم تصاویر شاخص، و ساخت فرم تماس.
- **بازبینی بصری (Visual Review)** — گرفتن اسکرین‌شات از صفحات زنده، مقایسه حالت‌های قبل و بعد، و دریافت یک بازبینی طراحی تولید شده توسط هوش مصنوعی که شامل چیدمان، تایپوگرافی، رنگ و دسترسی‌پذیری است.
- **پنج دستیار پیش‌فرض** — شامل نویسنده محتوا (Content Writer)، سازنده سایت (Site Builder)، استودیوی طراحی (Design Studio)، مدیر افزونه (Plugin Manager)، و دستیار پشتیبانی (Support Assistant)؛ این دستیارها از طریق انتخاب‌کننده دستیار (Agent Picker) در هدر چت قابل تعویض هستند.
- **فِلیگ‌های ویژگی (Feature flags)** — کلیدهای فعال/غیرفعال‌سازی برای کنترل دسترسی و برندسازی در مسیر تنظیمات (Settings) → فِلیگ‌های ویژگی (Feature Flags) برای محدودیت نقش، نام‌گذاری وایت‌لیبل (white-label)، و سفارشی‌سازی ویجت.
- **مدیریت نوع پست سفارشی (Custom Post Type management)** — ثبت، لیست و حذف انواع پست سفارشی با حفظ داده‌ها پس از راه‌اندازی مجدد.
- **مدیریت تاکسونومی سفارشی (Custom Taxonomy management)** — ثبت، لیست و حذف تاکسونومی‌های سفارشی با حفظ داده‌ها.
- **قابلیت‌های سیستم طراحی (Design System abilities)** — تزریق CSS سفارشی، مدیریت الگوهای بلوک (block patterns) منتخب، تنظیم لوگوی سایت، و اعمال پیش‌تنظیمات theme.json.
- **مدیریت استایل‌های سراسری (Global Styles management)** — خواندن و نوشتن مقادیر theme.json برای پالت‌های رنگ، تایپوگرافی، فاصله و موارد دیگر.
- **مدیریت منوی ناوبری (Navigation Menu management)** — ایجاد و به‌روزرسانی منوهای ناوبری و آیتم‌های آن‌ها به صورت برنامه‌نویسی.
- **مدیریت تنظیمات (Options Management)** — خواندن و نوشتن تنظیمات وردپرس با یک لیست سیاه ایمنی داخلی برای محافظت از تنظیمات حیاتی.
- **ثبت قابلیت‌های قابل نصب (Installable Abilities Registry)** — کشف و فعال‌سازی بسته‌های قابلیت اضافی بدون تغییر فایل‌های اصلی افزونه.
- **ارکستراسیون سازنده سایت v2 (Site Builder Orchestration v2)** — برنامه‌های ساخت سایت چند مرحله‌ای با کشف افزونه، ردیابی پیشرفت، و بازیابی خطا.
- **دستور بنچمارک WP-CLI (WP-CLI benchmark command)** — اجرای مجموعه بنچمارک قابلیت دستیار از خط فرمان برای CI و ارزیابی مدل.

## پیش‌نیازها {#requirements}

- وردپرس ۶.۰ یا بالاتر
- PHP ۸.۱ یا بالاتر
- وردپرس چندسایتی (فعال در سطح شبکه) یا نصب تک‌سایتی
- یک افزونه کانکتور هوش مصنوعی فعال (Anthropic، OpenAI، یا سازگار)

## نصب {#installation}

۱. فایل‌های افزونه را در مسیر `/wp-content/plugins/gratis-ai-agent` آپلود کنید.
۲. افزونه را از طریق صفحه **افز افزونه‌ها (Plugins)** در وردپرس فعال کنید.
۳. به **Gratis AI Agent → تنظیمات (Settings)** بروید و ارائه‌دهنده هوش مصنوعی خود را متصل کنید.
۴. (اختیاری) برای نصب بسته‌های قابلیت اضافی، **ثبت قابلیت‌ها (Abilities Registry)** را مرور کنید.

## شروع سریع {#quick-start}

پس از فعال‌سازی و اتصال به یک ارائه‌دهنده هوش مصنوعی، پنل **Gratis AI Agent** را در نوار کناری مدیریت وردپرس باز کنید. یک دستور زبان طبیعی تایپ کنید و دستیار مراحل مورد نیاز را با استفاده از قابلیت‌های موجودش برنامه‌ریزی و اجرا خواهد کرد.

**دستورات نمونه:**

```
Create a "Portfolio" custom post type with a "Project Category" taxonomy.
```

```
Inject a dark-mode colour palette into the theme and set the primary colour to #1a1a2e.
```

```
Build a restaurant website with an online menu, booking form, and contact page.
```

## نمای کلی قابلیت‌ها (Abilities Overview) {#abilities-overview}

قابلیت‌ها، اقدامات اتمی هستند که دستیار می‌تواند انجام دهد. هر قابلیت یک کلاس PHP ثبت شده است که یک شمای (پارامترها و انواع بازگشتی) را در اختیار قرار می‌دهد که دستیار در زمان اجرا آن را می‌خواند. قابلیت‌ها در حوزه‌های عملکردی گروه‌بندی شده‌اند:

| حوزه | قابلیت‌ها |
|---|---|
| **مدیریت محتوا** | `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form` |
| **بازبینی بصری** | `capture_screenshot`, `compare_screenshots`, `review_page_design` |
| **انواع پست سفارشی** | `register_post_type`, `list_post_types`, `delete_post_type` |
| **تاکسونومی‌های سفارشی** | `register_taxonomy`, `list_taxonomies`, `delete_taxonomy` |
| **سیستم طراحی** | `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset` |
| **استایل‌های سراسری** | `get_global_styles`, `set_global_styles`, `reset_global_styles` |
| **منوهای ناوبری** | `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus` |
| **مدیریت تنظیمات** | `get_option`, `set_option`, `delete_option`, `list_options` |
| **قابلیت‌های قابل نصب** | `list_available_abilities`, `install_ability`, `recommend_plugin` |
| **سازنده سایت** | `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error` |

برای مستندات کامل پارامترها و مثال‌های استفاده، به [Abilities Reference](./abilities) مراجعه کنید.

## WP-CLI {#wp-cli}

Gratis AI Agent مجموعه‌ای از دستورات `wp gratis-ai-agent` را برای اجرای بنچمارک‌ها و مدیریت قابلیت‌ها از خط فرمان ارائه می‌دهد. برای لیست کامل دستورات، به [WP-CLI Reference](./wp-cli) مراجعه کنید.

```bash
# Run the full benchmark suite
wp gratis-ai-agent benchmark

# List installed abilities
wp gratis-ai-agent abilities list

# Install an ability pack
wp gratis-ai-agent abilities install <slug>
```

## پیکربندی {#configuration}

تمام تنظیمات در پنل **Gratis AI Agent → تنظیمات (Settings)** در مدیریت وردپرس قرار دارند:

| تنظیمات | توضیحات |
|---|---|
| **AI Provider** | انتخاب و پیکربندی کانکتور ارائه‌دهنده هوش مصنوعی فعال |
| **Model** | انتخاب مدل زبان مورد استفاده برای جلسات دستیار |
| **Max Tokens** | بودجه توکن در هر درخواست (پیش‌فرض ۴۰۹۶) |
| **Ability Timeout** | حداکثر ثانیه‌ای که یک قابلیت می‌تواند اجرا شود (پیش‌فرض ۳۰) |
| **Debug Logging** | ثبت فراخوانی قابلیت‌ها و استدلال دستیار در لاگ اشکال‌زدایی وردپرس |

## لیست سیاه ایمنی مدیریت تنظیمات (Options Management Safety Blocklist) {#options-management-safety-blocklist}

قابلیت مدیریت تنظیمات شامل یک لیست سیاه داخلی است که از خواندن یا نوشتن تنظیمات حساس وردپرس توسط دستیار جلوگیری می‌کند. لیست سیاه پیش‌فرض شامل موارد زیر است:

- `admin_email`, `blogadmin_email`
- `siteurl`, `home` (تنظیمات URL — برای این موارد از ابزارهای مهاجرت اختصاصی استفاده کنید)
- `auth_key`, `secure_auth_key`, `logged_in_key`, `nonce_key` (کلیدهای مخفی)
- `active_plugins`, `template` (فعال‌سازی افزونه/قالب — از قابلیت‌های اختصاصی استفاده کنید)
- اعتبارنامه‌ها و کلیدهای API درگاه پرداخت که توسط افزونه‌های نصب شده ثبت شده‌اند.

می‌توان موارد اضافی را از طریق فیلتر `gratis_ai_agent_options_blocklist` اضافه کرد.

## هوک‌ها و فیلترها (Hooks and Filters) {#hooks-and-filters}

### اکشن‌ها (Actions) {#actions}

- `gratis_ai_agent_ability_registered` — پس از ثبت یک قابلیت فراخوانی می‌شود و نام کلاس قابلیت را دریافت می‌کند.
- `gratis_ai_agent_plan_started` — هنگام شروع اجرای برنامه ساز سایت فراخوانی می‌شود.
- `gratis_ai_agent_plan_completed` — هنگام تکمیل موفقیت‌آمیز یک برنامه فراخوانی می‌شود.
- `gratis_ai_agent_plan_failed` — هنگام مواجه شدن یک برنامه با خطای غیرقابل بازیابی فراخوانی می‌شود.

### فیلترها (Filters) {#filters}

- `gratis_ai_agent_abilities` — لیست قابلیت‌های ثبت شده را قبل از بارگذاری توسط دستیار تغییر می‌دهد.
- `gratis_ai_agent_options_blocklist` — لیست سیاه ایمنی تنظیمات را گسترش یا کاهش می‌دهد.
- `gratis_ai_agent_benchmark_questions` — سؤالات بنچمارک سفارشی را به مجموعه ارزیابی اضافه می‌کند.
- `gratis_ai_agent_plan_plugins` — لیست کشف افزونه‌ها را در طول تولید برنامه سایت تغییر می‌دهد.
