# تغییرات گزارش (Changelog)

## ۱.۹.۰ — منتشر شده در ۲۰۲۶-۰۴-۲۸ {#190--released-on-2026-04-28}

### قابلیت‌های جدید {#new}

- **قابلیت `create_contact_form`**: با استفاده از پلاگین فرم فعال (Contact Form 7، WPForms، Fluent Forms، یا Gravity Forms)، یک فرم تماس ایجاد می‌کند و یک shortcode برمی‌گرداند که آماده قرارگیری در هر پست یا صفحه‌ای است.
- **قابلیت `set_featured_image`**: یک تصویر شاخص (featured image) را از یک شناسه پیوست موجود در Media Library یا یک URL از راه دور، به یک پست یا صفحه اختصاص می‌دهد؛ اگر URL ارائه شود، تصویر را به صورت خودکار وارد (import) می‌کند.
- **قابلیت `batch_create_posts`**: امکان ایجاد چندین پست را تنها با یک فراخوانی (call) فراهم می‌کند. از پارامترهای یکسان `create_post` پشتیبانی می‌کند، موفقیت/شکست را برای هر پست گزارش می‌دهد و دارای حالت اختیاری `stop_on_error` است.
- **پارامتر `page_template`**: به `create_post` و `update_post` اضافه شده است. یک فایل قالب صفحه PHP (مانند `page-full-width.php`) را هنگام ایجاد یا به‌روزرسانی اختصاص می‌دهد. برای بازگشت به پیش‌فرض تم، در `update_post` یک رشته خالی ارسال کنید.
- **قابلیت‌های اسکرین‌شات سمت کلاینت**: شامل `capture_screenshot`، `compare_screenshots` و `review_page_design`. این قابلیت‌ها اسکرین‌شات کامل یا نمای قابل مشاهده (viewport) از صفحات زنده را از طریق یک مرورگر بدون سرور (headless browser) می‌گیرند، دو اسکرین‌شات را مقایسه (diff) می‌کنند و یک بررسی طراحی تولید شده توسط هوش مصنوعی ارائه می‌دهند که شامل چیدمان، تایپوگرافی، رنگ و دسترسی‌پذیری است.
- **پنج عامل داخلی (Built-in Agents)**: شامل Content Writer، Site Builder، Design Studio، Plugin Manager و Support Assistant. هر عامل مجموعه‌ای اختصاصی از ابزارها، یک پرامپت سیستمی سفارشی و پیشنهادهای شروع کار دارد. این عامل‌ها از طریق **Agent Picker** جدید در هدر چت قابل تعویض هستند. برای مشاهده، به [Built-in Agents](../../user-guide/configuration/built-in-agents) مراجعه کنید.
- **Feature flags (پرچم‌های ویژگی)**: تب جدید **Settings → Feature Flags** با کلیدهای کنترل دسترسی (محدود به مدیران، محدود به مدیران شبکه، دسترسی مشترک، غیرفعال برای غیرعضو) و گزینه‌های برندسازی (پنهان کردن منبع فوتر، نام عامل سفارشی، پنهان کردن Agent Picker، استفاده از آیکون سایت به عنوان آواتار چت). برای مشاهده، به [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings) مراجعه کنید.
- **بازیابی آخرین جلسه**: پنل چت اکنون با بارگذاری صفحه و باز شدن ویجت، آخرین مکالمه را به صورت خودکار بارگذاری می‌کند، بنابراین زمینه (context) هرگز در طول ناوبری صفحات از دست نمی‌رود.
- **لینک‌های اکشن پلاگین**: لینک‌های سریع به تنظیمات (Settings) و رجیستری قابلیت‌ها (Abilities Registry) اکنون در صفحه **Plugins → Installed Plugins** وردپرس، زیر توضیحات پلاگین، ظاهر می‌شوند.

### بهبودها {#improved}

- **تلاش مجدد منبع تصویر**: عامل اکنون قبل از اینکه در صورت شکست دانلود، به تصویر تولید شده توسط هوش مصنوعی روی بیاورد، تمام منابع تصویر رایگان پیکربندی شده را دوباره امتحان می‌کند.
- **پنل اطلاعات مدل**: همیشه در هدر چت قابل مشاهده است؛ دیگر پس از پیام اول پنهان نمی‌شود.
- **رفتار اسکرول خودکار**: اسکرول خودکار زمانی که کاربر برای خواندن به بالا اسکرول می‌کند، متوقف می‌شود؛ یک دکمه شناور **Scroll to bottom** ظاهر شده و به محض رسیدن کاربر به آخرین پیام، به طور خودکار محو می‌شود.
- **رابط کاربری Agent Picker**: به صورت یک Overlay جدول-فرم با آیکون‌های اختصاصی برای هر عامل بازطراحی شده است، که شناسایی و تعویض عامل‌ها را در یک نگاه آسان‌تر می‌کند.
- **تکه‌های JS بارگذاری تنبل (Lazy-loaded JS chunks)**: Bundle جاوااسکریپت اولیه ویجت چت اکنون به تکه‌های بارگذاری تنبل تقسیم شده است، که اندازه اولیه Bundle را ۷۵ تا ۹۰ درصد کاهش می‌دهد.
- **بازطراحی ویجت چت**: آیکون یکپارچه هوش مصنوعی جایگزین آواتار سفارشی قبلی شده است؛ این تغییر با سیستم عامل داخلی سازگار است.
- **لینک‌سازی URL**: URLهایی که در پیام‌های سیستمی و حباب‌های پیام خطا ظاهر می‌شوند، اکنون به عنوان لینک‌های قابل کلیک رندر می‌شوند.

### رفع اشکال‌ها {#fixed}

- **قابلیت کشف (Discoverability) قابلیت‌ها**: توضیحات، ارجاعات پرامپت سیستمی و هم‌ترازی فضای نام (namespace) تصحیح شد تا تمام قابلیت‌ها به طور قابل اعتمادی در لیست ابزارهای عامل ظاهر شوند.
- **کش ارائه‌دهندگان (Providers cache)**: ارائه‌دهندگان اکنون در کل سایت از طریق یک شمارنده نسخه (version counter) کش می‌شوند، که از مشکلات مربوط به ارائه‌دهنده منسوخ (stale-provider) در شبکه‌های multisite جلوگیری می‌کند.
- **`ability_invalid_output`**: در ۱۲ هندلر قابلیت رفع شد؛ همه اکنون پاسخ‌های JSON با ساختار صحیح برمی‌گردانند.
- **پایپ‌لاین `pending_client_tool_calls`**: از ابتدا تا انتها سیم‌کشی شد تا فراخوانی‌های ابزار سمت کلاینت به درستی تکمیل شده و نتایج را به مدل برگردانند.
- **کشوی تاریخچه (History drawer)**: تغییرات غیرقابل بازگشت از لیست بازگشت حذف شدند؛ لینک **View full history** اکنون به درستی کار می‌کند.
- **سیستم تغییرات/بازگشت**: پنج باگ جداگانه رفع و تحت رابط کاربری جدید ادغام شدند.
- **Toast ذخیره تنظیمات**: اعلان اسنک‌بار اکنون پس از کلیک بر روی **Save Settings** به طور قابل اعتمادی ظاهر می‌شود.
- **منوی زمینه سطل زباله (Trash context menu)**: گزینه **Delete Permanently** اضافه شد تا آیتم‌ها بتوانند بدون خروج از نمای سطل زباله، حذف سخت شوند.
- **ویرایش و ارسال مجدد**: کلیک بر روی آیکون ویرایش اکنون فقط حالت ویرایش را برای پیام کلیک شده فعال می‌کند، نه تمام پیام‌های موجود در رشته.
- **ارتفاع چیدمان چت**: پنل چت ارتفاع خود را زمانی که محتوای تزریق شده توسط پلاگین (اعلان‌های ادمین، بنرها) بالای صفحه ظاهر می‌شود، تنظیم می‌کند و از بیرون رانده شدن ناحیه ورودی جلوگیری می‌کند.

## ۱.۴.۰ — منتشر شده در ۲۰۲۶-۰۴-۰۹ {#140--released-on-2026-04-09}

### قابلیت‌های جدید {#new-1}

- **دستور معیار WP-CLI** (`wp gratis-ai-agent benchmark`): مجموعه معیار Agent Capabilities v1 را از خط فرمان برای خطوط لوله CI و گردش کار ارزیابی مدل اجرا می‌کند. از اجرای هر سؤال به صورت جداگانه، بازنویسی ارائه‌دهنده/مدل، و خروجی JSON/CSV پشتیبانی می‌کند.
- **مجموعه معیار Agent Capabilities v1**: مجموعه‌ای ساختاریافته از پرامپت‌های پیچیده و چند مرحله‌ای که کل سطح قابلیت‌ها را آزمایش می‌کنند، همراه با امتیازدهی و گزارش توکن/مدت زمان.
- **قابلیت‌های Custom Post Type**: `register_post_type`، `list_post_types`، `delete_post_type`. ثبت‌ها از طریق جدول options در برابر راه‌اندازی مجددها باقی می‌مانند.
- **قابلیت‌های Custom Taxonomy**: `register_taxonomy`، `list_taxonomies`، `delete_taxonomy`. از تاکسونومی‌های سلسله مراتبی و تخت با اسلاگ‌های بازنویسی اختیاری پشتیبانی می‌کند.
- **قابلیت‌های Design System**: `inject_custom_css`، `add_block_pattern`، `list_block_patterns`، `set_site_logo`، `apply_theme_json_preset`. شامل پنج پیش‌تنظیمات منتخب است: `minimal-dark`، `warm-editorial`، `corporate-blue`، `vibrant-startup`، `classic-blog`.
- **قابلیت‌های Global Styles**: `get_global_styles`، `set_global_styles`، `reset_global_styles`. مقادیر theme.json را از طریق API استایل‌های جهانی وردپرس می‌خواند و می‌نویسد.
- **قابلیت مدیریت منوی ناوبری**: `create_menu`، `update_menu`، `add_menu_item`، `remove_menu_item`، `list_menus`. از منوهای تودرتو و تخصیص مکان تم (theme location) پشتیبانی می‌کند.
- **قابلیت مدیریت تنظیمات (Options)**: `get_option`، `set_option`، `delete_option`، `list_options`. شامل یک لیست سیاه ایمنی داخلی است که از تغییر تنظیمات حیاتی وردپرس محافظت می‌کند.
- **رجیستری قابلیت‌های قابل نصب (Installable Abilities Registry)**: `list_available_abilities`، `install_ability`، `recommend_plugin`. بسته‌های قابلیت را که به صورت پلاگین وردپرس توزیع می‌شوند، کشف و فعال کنید.
- **Site Builder Orchestration v2**: `create_site_plan`، `execute_site_plan`، `get_plan_progress`، `handle_plan_error`. برنامه‌های ساخت سایت چند فازی با کشف پلاگین، ارجاعات خروجی مرحله‌ای، ردیابی پیشرفت و بازیابی خودکار خطا.
- **سؤال معیار وب‌سایت رستوران** (`q-restaurant-website`): یک تست سرتاسری که شامل ثبت CPT، سیستم طراحی، ناوبری و کشف پلاگین است.
- **پلاگین‌های کانکتور ارائه‌دهنده هوش مصنوعی**: برای راه‌اندازی محلی سریع‌تر، به Blueprints وردپرس Playground اضافه شدند.

### بهبودها {#improved-1}

- README با مستندات و دستورالعمل‌های راه‌اندازی کانکتور ارائه‌دهنده هوش مصنوعی به‌روز شد.

### رفع اشکال‌ها {#fixed-1}

- ۲۵ شکست تست PHPUnit در شاخه `main` رفع شد.
- فرمت URL انتشار GitHub در `blueprint.json` تصحیح شد.
- شماره‌گذاری Task ID برای جلوگیری از تداخل با شناسه‌های قدیمی.
