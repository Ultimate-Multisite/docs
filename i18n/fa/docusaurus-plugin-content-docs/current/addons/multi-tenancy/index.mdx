# چندمستأجری

زیرسایت‌ها را برای کارایی و امنیت بین سرورها و پایگاه‌های داده توزیع کنید

## نکات برجسته انتشار نسخه 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 بنیان مستأجر مستقل را برای گردش‌کارهای جداسازی در محیط تولید اضافه می‌کند:

- ورود خودکار بدون‌حالت مستأجر با بازدیدهای SSO کوتاه‌عمر و محافظت‌شده در برابر پخش مجدد.
- جداسازی پایگاه داده و فایل‌سیستم برای هر زیرسایت، ویژه مستأجران مستقل.
- قابلیت چندمستأجری Hostinger برای تأمین مستأجر آگاه از میزبان.
- دروازه‌های تأیید مهاجرت مستقل برای بررسی داده‌های قدیمی و پردازش ارسال ناهمگام.
- اتصال قابل‌پیکربندی میزبان پایگاه داده روی همان ماشین برای نصب‌های Bedrock، FrankenPHP و localhost.
- بهبودهای راه‌اندازی اولیه مستأجر، نویسنده پایگاه داده، راه‌اندازی اولیه طرح‌واره، و تأمین کاربر.
- پاک‌سازی امن‌تر مستأجر مستقل که اعتبارنامه‌های مستأجر را پاک می‌کند.

راهنماهای کاربر را برای [گردش‌کار مهاجرت مستقل](/user-guide/getting-started/sovereign-migration-workflow)، [جداسازی چندمستأجری](/user-guide/configuration/multi-tenancy-isolation)، و [عملیات مستأجر مستقل](/user-guide/administration/sovereign-tenant-operations) ببینید.

## الزامات {#requirements}

- WordPress 5.3 یا بالاتر
- PHP 7.4 یا بالاتر
- افزونه Multisite Ultimate (فعال)

## نصب {#installation}

1. فایل‌های افزونه جانبی را در دایرکتوری `/wp-content/plugins/` خود بارگذاری کنید
2. افزونه را از طریق منوی «افزونه‌ها» در WordPress فعال کنید
3. تنظیمات افزونه جانبی را در مدیریت شبکه پیکربندی کنید

## توسعه {#development}

### راه‌اندازی {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### آزمون {#testing}

این افزونه جانبی شامل یک چارچوب آزمون جامع است:

- **آزمون‌های واحد**: آزمون‌های مبتنی بر PHPUnit با مجموعه آزمون WordPress
- **کلاس پایه آزمون**: روش‌های کمکی برای سناریوهای رایج آزمون فراهم می‌کند
- **پوشش آزمون**: برای پیگیری پوشش کد پیکربندی شده است
- **آزمون چندسایتی**: آزمون‌ها در محیط چندسایتی اجرا می‌شوند

### استانداردهای کد {#code-standards}

- **استانداردهای کدنویسی WordPress**: از طریق PHPCS اعمال می‌شوند
- **سازگاری با PHP 7.4+**: قابلیت‌های مدرن PHP پشتیبانی می‌شوند
- **تحلیل ایستا**: یکپارچه‌سازی PHPStan برای بررسی نوع
- **نوسازی کد**: قواعد Rector برای ارتقاهای PHP

### سیستم ساخت {#build-system}

- **پردازش دارایی‌ها**: کوچک‌سازی برای فایل‌های CSS/JS
- **ترجمه**: تولید فایل POT
- **بسته‌بندی**: ایجاد آرشیو برای توزیع
- **حالت توسعه**: دارایی‌های کوچک‌سازی‌نشده برای اشکال‌زدایی

## پیکربندی {#configuration}

این افزونه جانبی از گزینه‌های پیکربندی گوناگونی پشتیبانی می‌کند که از طریق رابط مدیریت شبکه در دسترس هستند.

### پیکربندی مستأجر مستقل {#sovereign-tenant-configuration}

برای مستأجران مستقل، پیش از مسیریابی ترافیک به مستأجر، پایگاه داده مستأجر، ریشه فایل‌سیستم، اتصال میزبان پایگاه داده، و قابلیت میزبان را پیکربندی کنید. پس از هر تغییری که بر اعتبارنامه‌های پایگاه داده، اتصال میزبان، مسیریابی، یا تأمین کاربر اثر می‌گذارد، تأیید مهاجرت را اجرا کنید.

از `localhost` فقط زمانی استفاده کنید که زمان اجرای مستأجر و مجوزهای پایگاه داده انتظار دسترسی از طریق سوکت محلی یا همان ماشین را دارند. زمانی از `127.0.0.1` یا نام میزبان سرویس استفاده کنید که آن میزبان مجوزی است که توسط ارائه‌دهنده شما پیکربندی شده است.

### قابلیت Hostinger {#hostinger-capability}

قابلیت چندمستأجری Hostinger با توکن یکپارچه‌سازی مشترک Hostinger کار می‌کند. پیش از تأیید، مطمئن شوید دامنه در hPanel وجود دارد، DNS به Account در Hostinger اشاره می‌کند، و تنظیمات پایگاه داده مستأجر با hPanel مطابقت دارد.

## هوک‌ها و فیلترها {#hooks-and-filters}

### کنش‌ها {#actions}

- `ultimate-multisite-addon-template_init` - پس از مقداردهی اولیه افزونه جانبی اجرا می‌شود
- `ultimate-multisite-addon-template_loaded` - پس از بارگذاری همه فایل‌های افزونه جانبی اجرا می‌شود

### فیلترها {#filters}

- `ultimate-multisite-addon-template_settings` - تنظیمات افزونه جانبی را تغییر دهید
- `ultimate-multisite-addon-template_enabled` - وضعیت فعال/غیرفعال بودن افزونه جانبی را بازنویسی کنید
