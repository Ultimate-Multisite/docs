# کانکتور هوش مصنوعی Ultimate برای نقاط پایانی سازگار {#ultimate-ai-connector-for-compatible-endpoints}

کانکتور هوش مصنوعی Ultimate برای نقاط پایانی سازگار، APIهای ارائه‌دهندگان هوش مصنوعی سازگار با OpenAI را با شبکه multisite وردپرس شما ادغام می‌کند. شما می‌توانید چندین ارائه‌دهنده هوش مصنوعی را پیکربندی کنید — که هر کدام دارای URL نقطه پایانی، کلید API و مدل پیش‌فرض مخصوص به خود هستند — و درخواست‌ها را با قابلیت بازگشت خودکار (automatic fallback) در صورت عدم دسترسی یک ارائه‌دهنده، مسیریابی کنید.

## ویژگی‌های کلیدی {#key-features}

- **پشتیبانی از چند ارائه‌دهنده**: افزودن کارت‌های متعدد ارائه‌دهنده هوش مصنوعی، که هر کدام دارای URL نقطه پایانی، کلید API و مدل پیش‌فرض خود هستند.
- **مرتب‌سازی با کشیدن (Drag-to-reorder)**: ارائه‌دهنده‌ها را با استفاده از دستگیره (grip handle) روی هر کارت، بر اساس اولویت مرتب کنید؛ درخواست‌ها به ترتیب از ارائه‌دهنده‌ها استفاده می‌کنند.
- **مسیریابی با بازگشت خودکار**: اگر ارائه‌دهنده اصلی در دسترس نباشد، درخواست‌ها به طور خودکار به ارائه‌دهنده پیکربندی شده بعدی منتقل می‌شوند.
- **پیکربندی برای هر ارائه‌دهنده**: هر ارائه‌دهنده به طور مستقل با اعتبارنامه‌ها و انتخاب مدل خود پیکربندی می‌شود.
- **رابط کاربری مبتنی بر کارت**: ساخته شده بر روی کامپوننت‌های کارت پایدار وردپرس 6.9+ (CardBody, CardHeader, CardDivider).
- **گسترش خودکار هنگام افزودن**: کارت‌های ارائه‌دهنده جدید به طور خودکار باز می‌شوند تا بتوانید بلافاصله جزئیات را وارد کنید.
- **رفع کش (Cache busting) اسکریپت**: اسکریپت‌های ارائه‌دهنده هنگام به‌روزرسانی پلاگین، کش‌بست می‌شوند تا اطمینان حاصل شود که آخرین نسخه بارگذاری می‌شود.

## پیش‌نیازها {#requirements}

- وردپرس 5.3 یا بالاتر
- PHP 7.4 یا بالاتر
- پلاگین Ultimate Multisite (فعال)
- یک نقطه پایانی API سازگار با OpenAI (میزبان شده توسط خود شما یا شخص ثالث)

## نصب {#installation}

1. فایل‌های افزونه (addon) را در دایرکتوری `/wp-content/plugins/` آپلود کنید.
2. پلاگین را از طریق منوی 'Plugins' در وردپرس فعال کنید.
3. برای پیکربندی افزونه به **Ultimate Multisite → AI Connector** بروید.

## پیکربندی {#configuration}

### تنظیم چند ارائه‌دهنده {#multi-provider-setup}

صفحه پیکربندی از یک طرح‌بندی مبتنی بر کارت استفاده می‌کند که در آن هر کارت نماینده یک ارائه‌دهنده هوش مصنوعی است. شما می‌توانید به تعداد ارائه‌دهنده‌های مورد نیازتان، آن‌ها را اضافه کنید.

#### افزودن یک ارائه‌دهنده {#adding-a-provider}

1. در بخش مدیریت شبکه، به **Ultimate Multisite → AI Connector** بروید.
2. روی **Add Provider** کلیک کنید.
3. یک کارت ارائه‌دهنده جدید ظاهر شده و به طور خودکار باز می‌شود.
4. جزئیات ارائه‌دهنده را پر کنید:
   - **Provider Name** — برچسبی برای شناسایی این ارائه‌دهنده (مثلاً "OpenAI"، "Mistral"، "Local Ollama").
   - **Endpoint URL** — آدرس پایه API سازگار با OpenAI (مثلاً `https://api.openai.com/v1`).
   - **API Key** — کلید API شما برای این ارائه‌دهنده.
   - **Default Model** — شناسه مدلی که به طور پیش‌فرض برای این ارائه‌دهنده استفاده می‌شود (مثلاً `gpt-4o`، `mistral-small`).
5. روی **Save Settings** کلیک کنید.

#### حذف یک ارائه‌دهنده {#removing-a-provider}

برای حذف یک ارائه‌دهنده، روی دکمه **Remove** در هر کارت ارائه‌دهنده کلیک کنید. برای اعمال تغییر، تنظیمات را ذخیره کنید.

#### مرتب‌سازی مجدد ارائه‌دهنده‌ها {#reordering-providers}

برای مرتب‌سازی مجدد، روی آیکون دستگیره (⣿) در سمت چپ هر کارت ارائه‌دهنده بکشید. این ترتیب، اولویت بازگشت (fallback priority) را تعیین می‌کند: درخواست‌ها ابتدا به اولین ارائه‌دهنده ارسال شده و در صورت شکست درخواست، به ارائه‌دهنده‌های بعدی بازمی‌گردند.

### مسیریابی بازگشت (Fallback Routing) {#fallback-routing}

هنگامی که درخواست به ارائه‌دهنده اصلی شکست بخورد — به دلیل اتمام زمان (timeout)، محدودیت نرخ (rate limit) یا قطعی سرویس — کانکتور به طور خودکار با ارائه‌دهنده بعدی در لیست، تلاش مجدد انجام می‌دهد. این فرآیند تا زمانی که یک ارائه‌دهنده موفق شود یا تمام ارائه‌دهنده‌ها امتحان شوند، ادامه می‌یابد.

| سناریو | رفتار |
|----------|-----------|
| ارائه‌دهنده اصلی پاسخ دهد | درخواست به طور عادی سرو می‌شود |
| ارائه‌دهنده اصلی شکست بخورد | درخواست با ارائه‌دهنده بعدی تلاش مجدد می‌شود |
| تمام ارائه‌دهنده‌ها شکست بخورند | خطا به فراخواننده (caller) بازگردانده می‌شود |

قابلیت بازگشت برای پلاگین یا تم فراخواننده کاملاً شفاف است — آن‌ها صرف نظر از اینکه چند ارائه‌دهنده امتحان شده‌اند، یا یک پاسخ موفق یا یک خطای نهایی دریافت می‌کنند.

### مرجع پیکربندی ارائه‌دهنده {#provider-configuration-reference}

| فیلد | الزامی | توضیحات |
|-------|----------|-------------|
| Provider Name | بله | برچسب نمایش برای این ارائه‌دهنده |
| Endpoint URL | بله | آدرس پایه API سازگار با OpenAI |
| API Key | بله | کلید احراز هویت برای این ارائه‌دهنده |
| Default Model | بله | شناسه مدلی که در صورت عدم تعیین مدل در درخواست، استفاده می‌شود |

## عیب‌یابی {#troubleshooting}

### هیچ ارائه‌دهنده‌ای لیست نشده است {#no-providers-are-listed}

اگر پس از ذخیره، لیست ارائه‌دهنده‌ها خالی بود، لطفاً مطمئن شوید که:

- پس از افزودن ارائه‌دهنده‌ها، روی **Save Settings** کلیک کرده‌اید.
- مرورگر شما مانع ذخیره صفحه تنظیمات نشده باشد (خطاهای جاوااسکریپت را در کنسول مرورگر بررسی کنید).
- پلاگین فعال باشد و شما در صفحه مدیریت شبکه (network admin) باشید.

### درخواست‌ها برای همه ارائه‌دهنده‌ها شکست می‌خورند {#requests-are-failing-for-all-providers}

- تأیید کنید که URL نقطه پایانی هر ارائه‌دهنده از سرور شما قابل دسترسی است (نه فقط از مرورگر شما).
- بررسی کنید که هر کلید API معتبر باشد و منقضی نشده باشد.
- وضعیت محدودیت نرخ و سهمیه ارائه‌دهنده را در داشبورد آن‌ها مرور کنید.
- برای ثبت پیام‌های خطای دقیق از کانکتور، لاگ‌گیری اشکال‌زدایی وردپرس (`WP_DEBUG_LOG`) را فعال کنید.

### یک ارائه‌دهنده برای بازگشت استفاده نمی‌شود {#a-provider-is-not-being-used-for-fallback}

مرتب‌سازی مجدد، ترتیب بازگشت را تغییر می‌دهد. تأیید کنید که ارائه‌دهنده‌ای که انتظار دارید درخواست‌های بازگشت به آن برسد، در لیست به درستی قرار گرفته و URL نقطه پایانی و کلید API آن صحیح است.

### کارت ارائه‌دهنده جدید به طور خودکار باز نمی‌شود {#new-provider-card-does-not-expand-automatically}

قابلیت گسترش خودکار (Auto-expand) نیاز به فعال بودن جاوااسکریپت در مرورگر شما دارد. اگر کارت با کلیک باز نشد، خطاهای کنسول مرورگر را بررسی کنید. محتویات کارت همیشه با کلیک روی سربرگ کارت قابل دسترسی است، حتی اگر گسترش خودکار فعال نشود.
