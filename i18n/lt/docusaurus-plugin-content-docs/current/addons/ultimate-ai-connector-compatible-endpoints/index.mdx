# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints integruoja su OpenAI suderinamų AI teikėjų API su jūsų WordPress multisite tinklu. Sukonfigūruokite kelis AI teikėjus — kiekvieną su savo galinio taško URL, API raktu ir numatytuoju modeliu — ir nukreipkite užklausas su automatiniu atsarginiu perjungimu, kai teikėjas nepasiekiamas.

## Pagrindinės funkcijos

- **Kelių teikėjų palaikymas**: pridėkite kelias AI teikėjų korteles, kiekvieną su savo galinio taško URL, API raktu ir numatytuoju modeliu
- **Vilkimas tvarkai keisti**: išdėstykite teikėjus pagal prioritetą naudodami kiekvienos kortelės suėmimo rankenėlę; užklausos naudoja teikėjus nurodyta tvarka
- **Automatinis atsarginis nukreipimas**: jei pagrindinis teikėjas nepasiekiamas, užklausos automatiškai perjungiamos į kitą sukonfigūruotą teikėją
- **Konfigūracija kiekvienam teikėjui**: kiekvienas teikėjas konfigūruojamas atskirai su savais prisijungimo duomenimis ir modelio pasirinkimu
- **Kortelėmis pagrįsta nustatymų UI**: sukurta naudojant stabilias WordPress 6.9+ Card komponentes (CardBody, CardHeader, CardDivider)
- **Automatinis išskleidimas pridėjus**: naujos teikėjų kortelės automatiškai išskleidžiamos, kad galėtumėte iškart įvesti informaciją
- **Script cache busting**: teikėjų skriptų cache atnaujinamas atnaujinus įskiepį, kad būtų įkelta naujausia versija

## Reikalavimai

- WordPress 5.3 arba naujesnė
- PHP 7.4 arba naujesnė
- Ultimate Multisite įskiepis (aktyvus)
- Su OpenAI suderinamas API galinis taškas (savarankiškai talpinamas arba trečiosios šalies)

## Diegimas

1. Įkelkite priedo failus į savo `/wp-content/plugins/` katalogą
2. Aktyvinkite įskiepį per WordPress meniu „Plugins“
3. Eikite į **Ultimate Multisite → AI Connector**, kad sukonfigūruotumėte priedą

## Konfigūracija

### Kelių teikėjų sąranka

Konfigūracijos ekranas naudoja kortelėmis pagrįstą išdėstymą, kuriame kiekviena kortelė reiškia vieną AI teikėją. Galite pridėti tiek teikėjų, kiek reikia.

#### Teikėjo pridėjimas

1. Eikite į **Ultimate Multisite → AI Connector** tinklo administravime
2. Spustelėkite **Pridėti teikėją**
3. Atsiranda nauja teikėjo kortelė ir automatiškai išsiskleidžia
4. Užpildykite teikėjo informaciją:
   - **Teikėjo pavadinimas** — etiketė šiam teikėjui identifikuoti (pvz., „OpenAI“, „Mistral“, „Local Ollama“)
   - **Galinio taško URL** — su OpenAI suderinamos API bazinis URL (pvz., `https://api.openai.com/v1`)
   - **API raktas** — jūsų šio teikėjo API raktas
   - **Numatytasis modelis** — modelio identifikatorius, kurį pagal numatytuosius nustatymus naudoti šiam teikėjui (pvz., `gpt-4o`, `mistral-small`)
5. Spustelėkite **Išsaugoti nustatymus**

#### Teikėjo pašalinimas

Spustelėkite mygtuką **Pašalinti** bet kurioje teikėjo kortelėje, kad ją ištrintumėte. Išsaugokite nustatymus, kad pritaikytumėte pakeitimą.

#### Teikėjų tvarkos keitimas

Vilkite suėmimo piktogramą (⣿) bet kurios teikėjo kortelės kairėje pusėje, kad pakeistumėte jos vietą. Tvarka nustato atsarginio perjungimo prioritetą: užklausos siunčiamos pirmajam teikėjui ir, jei užklausa nepavyksta, perjungiamos į paskesnius teikėjus.

### Atsarginis nukreipimas

Kai užklausa pagrindiniam teikėjui nepavyksta — dėl laiko limito, greičio apribojimo arba paslaugos sutrikimo — jungtis automatiškai pakartoja bandymą su kitu sąrašo teikėju. Tai tęsiama sąrašu žemyn, kol teikėjas sėkmingai atsako arba išbandomi visi teikėjai.

| Scenarijus | Elgsena |
|----------|-----------|
| Pagrindinis teikėjas atsako | Užklausa aptarnaujama įprastai |
| Pagrindinis teikėjas nepavyksta | Užklausa pakartojama su kitu teikėju |
| Visi teikėjai nepavyksta | Klaida grąžinama kvietėjui |

Atsarginis perjungimas kviečiančiam įskiepiui arba temai yra skaidrus — jie gauna arba sėkmingą atsakymą, arba galutinę klaidą, nepriklausomai nuo to, kiek teikėjų buvo išbandyta.

### Teikėjo konfigūracijos nuoroda

| Laukas | Privaloma | Aprašymas |
|-------|----------|-------------|
| Teikėjo pavadinimas | Taip | Rodoma šio teikėjo etiketė |
| Galinio taško URL | Taip | Su OpenAI suderinamos API bazinis URL |
| API raktas | Taip | Šio teikėjo autentifikavimo raktas |
| Numatytasis modelis | Taip | Modelio identifikatorius, naudojamas, kai užklausoje nenurodytas modelis |

## Trikčių šalinimas

### Nerodomas nė vienas teikėjas

Jei po išsaugojimo teikėjų sąrašas tuščias, patikrinkite, ar:

- Spustelėjote **Išsaugoti nustatymus** po teikėjų pridėjimo
- Jūsų naršyklė neužblokavo nustatymų puslapio išsaugojimo (patikrinkite naršyklės konsolę dėl JavaScript klaidų)
- Įskiepis aktyvus ir esate tinklo administravimo ekrane

### Užklausos nepavyksta visiems teikėjams

- Patikrinkite, ar kiekvieno teikėjo galinio taško URL pasiekiamas iš jūsų serverio (ne tik iš jūsų naršyklės)
- Patikrinkite, ar kiekvienas API raktas galioja ir nėra pasibaigęs
- Peržiūrėkite teikėjo greičio apribojimų ir kvotų būseną jų dashboarde
- Įjunkite WordPress derinimo žurnalavimą (`WP_DEBUG_LOG`), kad užfiksuotumėte išsamius jungties klaidų pranešimus

### Teikėjas nenaudojamas atsarginiam perjungimui

Vilkimas tvarkai keisti pakeičia atsarginio perjungimo tvarką. Patikrinkite, ar teikėjas, kuris turėtų gauti atsargines užklausas, sąraše yra tinkamoje vietoje ir ar jo galinio taško URL bei API raktas yra teisingi.

### Nauja teikėjo kortelė automatiškai neišsiskleidžia

Automatiniam išskleidimui naršyklėje turi būti įjungtas JavaScript. Jei kortelė neišsiskleidžia spustelėjus, patikrinkite naršyklės konsolę dėl klaidų. Kortelės turinys visada pasiekiamas spustelėjus kortelės antraštę, net jei automatinis išskleidimas nesuveikia.
