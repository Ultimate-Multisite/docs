# Multi-Tenancy

Paskirstykite posvetaines tarp serverių ir duomenų bazių našumui ir saugumui užtikrinti

## 1.2.0 versijos leidimo svarbiausi akcentai {#version-120-release-highlights}

Multi-Tenancy 1.2.0 prideda suverenių nuomininkų pagrindą gamybinės aplinkos izoliavimo darbo eigoms:

- Būsenos nesaugantis nuomininko automatinis prisijungimas su trumpalaikiais, nuo pakartojimo apsaugotais SSO apsilankymais.
- Kiekvienai posvetainei skirta duomenų bazės ir failų sistemos izoliacija suvereniems nuomininkams.
- Hostinger kelių nuomininkų galimybė, skirta į prieglobą atsižvelgiančiam nuomininkų parengimui.
- Suverenios migracijos patikros vartai senųjų duomenų patikroms ir asinchroniniam siuntimo apdorojimui.
- Konfigūruojamas tos pačios mašinos duomenų bazės prieglobos susiejimas Bedrock, FrankenPHP ir localhost diegimams.
- Nuomininko paleidimo, duomenų bazės rašymo modulio, schemos paleidimo ir naudotojų parengimo patobulinimai.
- Saugesnis suverenios aplinkos pašalinimas, kuris išvalo nuomininko prisijungimo duomenis.

Žr. naudotojo vadovus: [suverenios migracijos darbo eiga](/user-guide/getting-started/sovereign-migration-workflow), [kelių nuomininkų izoliacija](/user-guide/configuration/multi-tenancy-isolation) ir [suverenių nuomininkų operacijos](/user-guide/administration/sovereign-tenant-operations).

## Reikalavimai {#requirements}

- WordPress 5.3 arba naujesnė
- PHP 7.4 arba naujesnė
- Multisite Ultimate įskiepis (aktyvus)

## Diegimas {#installation}

1. Įkelkite priedo failus į savo `/wp-content/plugins/` katalogą
2. Aktyvuokite įskiepį per WordPress meniu „Plugins“
3. Sukonfigūruokite priedo nustatymus tinklo administravimo sąsajoje

## Kūrimas {#development}

### Sąranka {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Testavimas {#testing}

Priede yra išsami testavimo sistema:

- **Vienetų testai**: PHPUnit pagrindu sukurti testai su WordPress testų rinkiniu
- **Bazinė testų klasė**: Suteikia pagalbinius metodus įprastiems testavimo scenarijams
- **Testų aprėptis**: Sukonfigūruota stebėti kodo aprėptį
- **Multisite testavimas**: Testai vykdomi multisite aplinkoje

### Kodo standartai {#code-standards}

- **WordPress kodo standartai**: Užtikrinami naudojant PHPCS
- **PHP 7.4+ suderinamumas**: Palaikomos modernios PHP funkcijos
- **Statinė analizė**: PHPStan integracija tipų tikrinimui
- **Kodo modernizavimas**: Rector taisyklės PHP atnaujinimams

### Surinkimo sistema {#build-system}

- **Išteklių apdorojimas**: CSS/JS failų sumažinimas
- **Vertimas**: POT failo generavimas
- **Pakavimas**: Archyvo kūrimas platinimui
- **Kūrimo režimas**: Nesumažinti ištekliai derinimui

## Konfigūracija {#configuration}

Priedas palaiko įvairias konfigūracijos parinktis, pasiekiamas per tinklo administravimo sąsają.

### Suverenaus nuomininko konfigūracija {#sovereign-tenant-configuration}

Suvereniems nuomininkams sukonfigūruokite nuomininko duomenų bazę, failų sistemos šaknį, duomenų bazės prieglobos susiejimą ir prieglobos galimybę prieš nukreipdami srautą į nuomininką. Paleiskite migracijos patikrą po kiekvieno pakeitimo, kuris paveikia duomenų bazės prisijungimo duomenis, prieglobos susiejimą, maršrutizavimą arba naudotojų parengimą.

Naudokite `localhost` tik tada, kai nuomininko vykdymo aplinka ir duomenų bazės leidimai tikisi vietinio lizdo arba tos pačios mašinos prieigos. Naudokite `127.0.0.1` arba paslaugos prieglobos vardą, kai tai yra jūsų teikėjo sukonfigūruotas leidimų prieglobos adresas.

### Hostinger galimybė {#hostinger-capability}

Hostinger kelių nuomininkų galimybė veikia su bendrinamu Hostinger integracijos tokenu. Prieš patikrą įsitikinkite, kad domenas yra hPanel, DNS nukreiptas į Hostinger paskyrą, o nuomininko duomenų bazės nustatymai atitinka hPanel.

## Kabliai ir filtrai {#hooks-and-filters}

### Veiksmai {#actions}

- `ultimate-multisite-addon-template_init` - Suaktyvinama po priedo inicializavimo
- `ultimate-multisite-addon-template_loaded` - Suaktyvinama, kai įkeliami visi priedo failai

### Filtrai {#filters}

- `ultimate-multisite-addon-template_settings` - Keisti priedo nustatymus
- `ultimate-multisite-addon-template_enabled` - Perrašyti priedo įjungimo / išjungimo būseną
