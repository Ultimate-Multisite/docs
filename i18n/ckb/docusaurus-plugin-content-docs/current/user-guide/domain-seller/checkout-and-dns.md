# فیلد پرداخت و مدیریت DNS مشتری {#checkout-field-and-customer-dns-management}

## فیلد انتخاب دامنه در صفحه تسویه حساب {#the-domain-selection-checkout-field}

فیلد **Domain Selection** یک المان در صفحه تسویه حساب است که به مشتری اجازه می‌دهد انتخاب کند چگونه دامنه‌ی سایت خود را دریافت کند. آن را به هر فرم تسویه حساب اضافه کنید تا فروش دامنه فعال شود.

### افزودن فیلد به فرم تسویه حساب {#adding-the-field-to-a-checkout-form}

۱. به مسیر **Network Admin › Ultimate Multisite › Checkout Forms** بروید
۲. یک فرم تسویه حساب باز یا ایجاد کنید
۳. در ویرایشگر تسویه حساب، روی **Add Field** کلیک کنید
۴. **Domain Selection** را از لیست فیلدها انتخاب کنید
۵. گزینه‌های فیلد را پیکربندی کنید (در زیر ببینید)
۶. فرم را ذخیره کنید

### گزینه‌های فیلد {#field-options}

**Domain modes** — انتخاب کنید که مشتری کدام تب‌ها را ببیند. هر حالت می‌تواند به صورت مستقل فعال یا غیرفعال شود:

| Mode | کاری که انجام می‌دهد |
|---|---|
| **Subdomain** | مشتری از یک زیردامنه رایگان در شبکه شما استفاده می‌کند (مثلاً `mysite.yournetwork.com`). نیازی به پرداخت نیست. |
| **Register New Domain** | مشتری به دنبال یک دامنه جدید می‌گردد و آن را از طریق ارائه‌دهنده پیکربندی شده شما ثبت می‌کند. محصول دامنه‌ی متناظر برای قیمت‌گذاری استفاده می‌شود. |
| **Existing Domain** | مشتری دامنه‌ای که قبلاً مالک آن است را نگاشت (map) می‌کند. هیچ هزینه ثبت دامنه وجود ندارد. دامنه به طور خودکار به سایت او نگاشت می‌شود. |

**Default mode** — وقتی هر سه حالت فعال هستند، کدام تب اول باز می‌شود؟ برای اینکه ثبت دامنه اختیاری باقی بماند روی **Subdomain** تنظیم کنید، یا برای تشویق خرید روی **Register New Domain** تنظیم کنید.

**Domain product** — در صورت تمایل این فیلد را به یک محصول دامنه‌ی خاص متصل (pin) کنید. اگر تنظیم نشود، افزونه به طور خودکار بر اساس TLD که مشتری جستجو می‌کند، محصول متناظر را انتخاب می‌کند.

### فیلدهای تماس ثبت‌کننده {#registrant-contact-fields}

وقتی مشتری تب **Register New Domain** را انتخاب می‌کند، فرم تسویه حساب فیلدهای تماس ثبت‌کننده را به صورت درون خطی اضافه می‌کند:

- نام و نام خانوادگی
- آدرس (خط ۱، شهر، استان/منطقه، کد پستی، کشور)
- شماره تلفن

ئەم زانیاریان لەلایەن هەموو تۆمارکارانەوە پێویستن و پێش ئەوەی بانکی تۆمارکردنی API بێت، پارچە ژمارەی تەلەفۆنەکە بە شێوازی نێودەوڵەتی `+CC.NNN` دابەش دەکرێت کە لەلایەن تۆمارکارانەوە چاوەڕوان دەکرێت.

### URLی سایتەکە کە خۆی دروست دەبێت (Auto-generated site URL) {#auto-generated-site-url}

کاتێک کڕیارێک تۆمار دەکات یان دامێنی دیاری دەکات، فیلدی URLی سایتەکە بە شێوەیەکی ئۆتۆماتیک لە ناو دامێنی هەڵبژراوەکەدا پڕ دەکرێت. کڕیاران پێویست نییە فیلدێکی جیاواز بۆ URL پڕ بکەنەوە.

### ڕێکارەکانی گەڕان (Search behaviour) {#search-behaviour}

- بە شێوەیەکی ڕاستەوخۆ لەگەڵ تایپکردنی کڕیارەکە، بە AJAX دەبینرێت کە ئامادەیی دامێنی ئەلیکراو دەکرێت.
- کاتێک دامێنی باشتر نییە، پێشنیاری TLD جیاواز دەردەکەون.
- نرخە بە شێوەیەکی ڕاستەوخۆ دەهێنرێت و بە ڕوونی نیشان دەدرێت (نرخی تۆمارکردن، نرخێکی نوێکردنەوە، کارامەی پنهانکردنی WHOIS کە ئاساییە).
- کۆدەکانی کوپن بۆ بەرهەمەکانی دامێنی هەمان شێوەی کار دەکەن و هەر بەرهەمێکی دیکە.

**تکستکردنی وەڵامدانەوەی گەڕان (Tuning search responsiveness):**

```php
// کەمکردنەوەی تاخیرەی debounce (لە میلیچێندەدا) بۆ کەمکردنەوەی بانکی API لەسەر پەیوەندییە خێراکان
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // پێشنیار: 500
});
```

**پێشاندنی فیلدەکانی تایبەت بۆ فۆرم گەڕان دامێنی (Adding custom fields to the domain search form):**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'تێبینیی زیاتر (Additional notes)',
    ];
    return $fields;
});
```

## بەڕێوەبردنی DNS بۆ کڕیارەکان (Customer DNS management) {#customer-dns-management}

کڕیاران دەتوانن تۆمارەکانی DNS بۆ دامێنی تۆمارکراوەکانی لە ڕێگەی لاپەڕەی **My Account**، لە خوارەوەی تۆماری دامێنی خۆیان بەڕێوەبەر بکەن.

### جۆرەکانی تۆمارەکان کە پشتگیریکردنی بەڕێوەبردنی DNS هەیە (Supported record types) {#supported-record-types}

| جۆر (Type) | کارکردن (Use) |
|---|---|
| **A** | ناوی میزبانەکە دەگۆڕێت بۆ پیتێکی IPv4 |
| **AAAA** | ناوی میزبانەکە دەگۆڕێت بۆ پیتێکی IPv6 |
| **CNAME** | پەیوەندییەکی جیاواز دروست دەکات کە ئاماژەیەکی دیکەیە بە پێیەوە دەدات |
| **MX** | خەرجکارەکانی پۆستکردن (mail exchange server) دیاری دەکات |
| **TXT** | تۆمارە دەقدار (SPF، DMARC، ڕێنمایی یان هەر تۆماری دیکە) زیاد دەکات |

### کامان پێشکەشکارانی بەڕێوەبردنی DNS پشتگیریکردنیان هەیە؟ {#which-providers-support-dns-management}

جێبەجێکردنی DNS (زیادکردن، دەستکاری، دیاریکردنەوەی تۆمارەکان) لەگەڵ **OpenSRS**، **ResellerClub**، **Enom**، **HostAfrica** و **Openprovider** بێت. ناوەکانی **Hostinger** دەتوانن ناوی NameServer هاودەگۆڕن لە ڕێگەی Domain Seller؛ تۆمارە DNS بۆ ناوەکانی هاودەگیرکراو بە شێوەیەکی سەرەکی لەلایەن یەکگرتووی Hostingerەوە بەڕێوەدەچێت. ناوەکانی **Namecheap**، **GoDaddy** و **NameSilo** پۆلسی و زانیاریی کاتی کۆتایی دەردەخەن، بەڵام DNS دەبێت ڕاستەوخۆ لە پۆلسی کۆنترۆڵی تۆمارەکەدا بەڕێوەدەچێت.

### تۆمارە DNS سەرەکی (Default DNS records) {#default-dns-records}

تۆ دەتوانیت ئەو تۆمارە DNSی سەرەکییە دیاری بکەیت کە بە شێوەیەکی ئۆتۆماتیک کاتێک ناوێکی دابنێریت، جێبەجێ بکرێن. بگواشبە **Settings › Domain Seller › Default DNS Records**.

قەبارەی بەهاکانی تۆمارە سەرەکی دوو تۆکن (token) پشتگیرانین:

| Token | دەگۆڕدرێت بۆ |
|---|---|
| `{DOMAIN}` | ناوی ناوەکە کە تۆمارکراوە (بۆ نموونە، `example.com`) |
| `{SITE_URL}` | لایەکەی WordPress بۆ شوێنی سایتەکە بەکارهێنەرەکە |

**نموونە — پاشەکەوت و www دەداتە IPی خوادەیەکەت:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ئیدارەکار (Admin): بینین و دەستکاری DNS {#admin-viewing-and-editing-dns}

ئیدارەکانی تۆڕی دەتوانن تۆمارە DNSی هەر ناوێکی بەکارهێنەر لە ڕێگەی لاپەڕەی دەستکاری ناوەکە لە **Network Admin › Ultimate Multisite › Domains** ببینن و دەستکاری بکەن.
