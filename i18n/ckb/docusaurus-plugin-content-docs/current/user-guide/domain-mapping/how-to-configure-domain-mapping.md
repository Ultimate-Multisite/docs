# چۆنی ڕێکخستنی مۆدێلەکانی ناوچە (v2)

_**تێبینی گرنگ: ئەم بابەتە ئاماژەیە بۆ Ultimate Multisite وەرگرتنی 2.x.**_

یەکێک لە بەهێزترین تایبەتمەندییەکان یەکێتی پریمیم تۆڕەکان بریتییە لە توانای پێشکەشکردنی دەرفەتێک بە کلاینتەکان بۆ ئەوەی ناوێکی سەرەکی (top-level domain) بە سایتەکانیان پەیوەست بکەن. لە کۆتاییدا، کامێک زیاتر پڕۆفیشناڵتر دەربێت: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) یان [_**joesbikeshop.com**_](http://joesbikeshop.com)? بۆیە Ultimate Multisite ئەم تایبەتمەندییە بە شێوەیەکی ناوخۆ دەدات، و پێویستی بە بەکارهێنانی پڵگینەکانی سێیەم نییە.

## مۆدێلەکانی ناوچە چیین؟

وەک ناونیشانەکەی ئاماژەیە، مۆدێلەکانی ناوچە توانای Ultimate Multisite دەبێت داواکارییەک بۆ مۆدێلێکی تایبەت وەربگرێت و ئەو داواکارییە بە سایتە گونجاوە لە تۆڕەکەدا پەیوەست بکات کە ئەو مۆدێلەی تایبەتە یەکێکە.

### چۆنی ڕێکخستنی مۆدێلەکانی ناوچە لە تۆڕی Ultimate Multisite دەکەیتەوە

مۆدێلەکانی ناوچە پێویستی بە هەندێک ڕێکخستن لەلایەن خودی ئێوە هەیە بۆ ئەوەی کار بکات. بەڵام خۆشگوزارە، Ultimate Multisite کاری سەخت بۆت ئۆتۆماتیک دەکات بۆ ئەوەی بە ئاسانی پێویستییەکان پڕ بکەیتەوە.

لە کاتی دامەزراندنی Ultimate Multisite، ویزاردەکە بە شێوەیەکی خۆکار کۆپی و جێبەجێکردنی **sunrise.php** لەناو دۆلەی دیاریکراو ئەنجام دەدات. **ویزاردەکە ڕێگە نادات کە بەم هەنگاوە پێشتر بڕۆیت.**

<!-- Screenshot unavailable: Ultimate Multisite installation wizard with sunrise.php step -->

ئەمەش واتای ئەوەیە کە دوای تەواوکردنی دامەزراندنی Ultimate Multisite، دەتوانیت بەردەوام بیت لە ڕێکخستنی مۆدێلێکی تایبەت.

تێبینی بکە کە مۆدێلەکانی ناوچە لە Ultimate Multisite پێویست نییە. تۆ دەتوانیت هەڵبژاردن بکەیت بۆ بەکارهێنانی تایبەتمەندی مۆدێلەکانی ناوچەی خۆی ی WordPress Multisite یان هەر چارەسەرێکی دیکەی مۆدێلەکانی ناوچە.

ئایا پێویستە کە دۆمێین دامێنی Ultimate Multisite-ی دەستکاری بکەیت بۆ ئەوەی ڕێگە بدات بە چارچێوەکانی دیکەی مۆدێل دامێنانی دۆمێین (domain mapping) بکاربهێنرێت، دەتوانیت ئەم تایبەتمەندییە لە خوارەوە دەستکاری بکەیت: **Ultimate Multisite > Settings > Domain Mapping**.

![Domain Mapping settings page showing admin redirect, mapping message and DNS options](/img/config/domain-mapping-settings.png)

لخۆم ئەم ڕێگە، دەتوانیت ل خوارەوە ئەو ڕێگایەیش ببینیت: **Force Admin Redirect** (بڵاوکردنەوەی بەڕێوەبەری سەرەتایی). ئەم ڕێگە وەک کۆنتڕۆڵێک کار دەکات کە دیاری دەکەیت ئایا کڕیارەکان دەتوانن بەرنامەی بەڕێوەبردنیان (admin dashboard) لەسەر دۆمێینی تایبەتی خۆیان و یان زیرەکی ناوخۆیی (subdomain) بگرن یان تەنها لەسەر یەکێکیان.

ئەگەر **Force redirect to mapped domain** (بڵاوکردنەوەی بەڕێوەبەری سەرەتایی بۆ دۆمێنی مۆدێلکراو) هەڵبژێریت، کڕیارەکان تەنها دەتوانن بەرنامەی بەڕێوەبردنیان لەسەر دۆمێنی تایبەتی خۆیان بگرن.

ئەگەر **Force redirect to network domain** (بڵاوکردنەوەی بەڕێوەبەری سەرەتایی بۆ دۆمێنی تۆڕی کار) هەڵبژێریت، ئەمە بە پێچەوانەی کار دەکات - کڕیارەکان تەنها ڕێگە دەدرێت بەرنامەکانیان لەسەر زیرەکی ناوخۆیی بگرن، تەنانەت ئەگەر هەوڵ بدەن خۆیان لەسەر دۆمێنی تایبەتی خۆیان یەکات بکەن.

وەک ئەوەی **Allow access to the admin by both mapped domain domain and network domain** (ڕێگەدان بە دەستگەیشتن بۆ بەڕێوەبەری سەرەتایی لەلایەن دۆمێنی مۆدێلکراو و دۆمێنی تۆڕی کار) ڕێگە بداتان بەرنامەکانیان لەسەر زیرەکی ناوخۆیی و دۆمێنی تایبەتی خۆیان بگرن.

![Admin Redirect dropdown expanded showing the three redirect options](/img/config/domain-mapping-redirect-options.png)

دوو ڕێگای هەیە بۆ مۆدێلکردنی دۆمێینێکی تایبەت (custom domain). یەکەم ئەوەیە کە ناوی دۆمێینەکە لە داش보ار ADMINISTRATORی تۆڕی کار وەک سوپەر ئادمن (super admin) دەستکاری بکەیت، و دووەم بریتییە لە ڕێگای زیرەکی ناوخۆیی (subsite admin dashboard) لە خوارەوە لەو لاپەڕەیەدا.

بەڵام پێش ئەوەی دەست بە مۆدێلکردنی دۆمێینێکی تایبەت بکەیت بۆ یەکێک لە زیرەکی ناوخۆییەکانی تۆڕی کار، دەبێت دڵنیا بیت کە **DNS settings** (سەرکردایەتی DNS) ناوی دۆمێینەکە بە شێوەیەکی دروست ڕێکخراون.

###

### دڵنیابوون لەوەی کە تنظیمات DNS-ی دۆمێین بە شێوەیەکی دروست ڕێکخراون

بۆ ئەوەی مێپینگەکە کار بکات، دەبێت دڵنیابیت کە دامێنی دیاری (domain) کە plan دەکەیت بۆ مێپینگ، ئاماژەیەکی بە پیت (IP address) ی تۆڕەکەت دەکات. ئاگادارم بکە لەوەی پێویستە بزانین IPی تۆڕەکە - ئەو IPیەەی دامێنی جێگیرکراوەت لەسەردا - نەک IPی دامێنی تایبەتی کە دەتەوێت مێپینگ بکەیت. بۆ گەڕان بە IPی دامێنی دیاریکراو، پێشنیار دەکەین ببیتە [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html)، بۆ نموونە.

بۆ مێپینگکردنی دروست دامێنی، دەبێت **A RECORD** لە ڕێکخستنی **DNS** یەکەت زیاد بکەیت کە ئاماژەیەکی بەو **IP address** دەکات. بەڕێوەبردنی DNS زۆر جیاوازە لە نێوان چەندین تۆمارگرنی دامێنی، بەڵام وردەکاری زۆر Tutorials لەسەر ئۆنلاین هەیە ئەگەر بگەڕێیت بۆ " _Creating A Record on XXXX_ " کە XXXX ئەو تۆمارگرنی دامێنییە کە تایبەتتە (نموونە: " _Creating A Record on_ _GoDaddy_ ").

ئەگەر هەست دەکەیت کێشەیەک لە بەدەستهێنانی ئەمە هەیە، **لەگەڵ پشتیوانی تۆمارگرنی دامێنی خۆت پەیوەندی بکە** و ئەوان دەتوانن یارمەتیت بدەن لەم بەشەدا.

ئەگەر plan دەکەیت کە کلاینتاکان ڕێگە بدات دامێنی خۆیان مێپینگ بکەن، ئەوا دەبێت ئەو کارە خۆیان لەسەر ئەم بەشە بکەن. پێدانیان بۆ سیستەمی پشتیوانی تۆمارگرنی دامێنی بڵێوەتەوە ئەگەر ناتوانن A Record دروست بکەن.

### مێپینگکردنی ناوی دامێنی تایبەت وەک Super Admin

کاتێک بە یەکێکی دیکەی Super Admin لە تۆڕەکەت لاگ in، دەتوانیت بە ئاسانی ناوی دامێنی تایبەتی زیاد و بەڕێوەبەر بکەیت لە خوارەوە: **Ultimate Multisite > Domains**.

![Domains list page in Ultimate Multisite](/img/admin/domains-list.png)

پای این صفحه، می توانید روی دکمه **Add Domain** در بالا کلیک کنید و یک پنجره مودال باز می‌شود که در آن می‌توانید **نام دامنه سفارشی**، **زیرسایت** مورد نظر خود را برای اعمال نام دامنه سفارشی تنظیم و پر کنید و تصمیم بگیرید که آیا می‌خواهید آن را به عنوان **دامنه اصلی** تنظیم کنید یا خیر (توجه داشته باشید که شما می‌توانید **چند نام دامنه را به یک زیرسایت نگاشت دهید**).

![Add Domain modal with domain name, site picker and primary domain toggle](/img/admin/domain-add-modal.png)

بعد از وارد کردن تمام اطلاعات، می‌توانید روی دکمه **Add Existing Domain** در پایین کلیک کنید.

این کار فرآیند تأیید و دریافت اطلاعات DNS دامنه سفارشی را آغاز می‌کند. همچنین یک لاگ در پایین صفحه خواهید دید تا بتوانید مراحل انجام این کار را دنبال کنید. ممکن است این فرآیند چند دقیقه طول بکشد تا تمام شود.

Ultimate Multisite v2.13.0 همچنین رکورد دامنه داخلی را به صورت خودکار هنگام ایجاد یک سایت جدید بر روی هاستی که باید به عنوان یک دامنه مخصوص هر سایت در نظر گرفته شود، ایجاد می‌کند. اگر هاست دامنه اصلی شبکه باشد، یا یکی از دامنه‌های پایه فرم پرداخت مشترک پیکربندی شده در فیلد **Site URL** تنظیم شده باشد، رکورد نگاشت‌شده خودکار دامنه نادیده گرفته می‌شود تا دامنه پایه مشترک برای هر سایتی که از آن استفاده می‌کند، در دسترس باقی بماند.

وضعیت **Stage** یا وضعیت باید از **Checking DNS** به **Ready** تغییر کند اگر همه چیز به درستی تنظیم شده باشد.

<!-- Screenshot unavailable: Domain row showing the Checking DNS stage in the domains list -->

<!-- Screenshot unavailable: Domain row showing the Ready stage with the green status indicator -->

اگر روی نام دامنه کلیک کنید، می‌توانید گزینه‌هایی را در داخل آن مشاهده کنید. بیایید نگاهی سریع به آن‌ها بیندازیم:

![Domain detail page with stage, site, active, primary and SSL toggles](/img/admin/domain-edit.png)

**مرحلة (Stage):** ئەمە ئەو مرحلهیەەی کە دۆمینەکە لەسەرە. کاتێک یەکەم جار دۆمینەکە زیاد دەکەیت، ئەوا بە احتمال زیاد لەسەر **پشکنینی DNS** (Checking DNS) دەبێت. پڕۆسەکە لێرەدا چاککردنەوەی تۆمارنامەکانی DNS دەکات و دەبینێت ئایا دروستن و ڕاستن. دوای ئەوە، دۆمینەکە بۆ مرحلهی **پشکنینی SSL** (Checking SSL) دەچێت. Ultimate Multisite دەداتزانێت کە دۆمینەکە SSL هەیە یان نییە و دۆمینەکەت بە **Ready** یان **Ready (بدون SSL)** پۆلێن دەکات.

**سایتی (Site):** ئەو subdomainە کە پەیوەندی بەم دۆمینەوە دەبێت. دۆمینەکە کە مَپکراوە، ناوەڕۆکی ئەم سایتە تایبەت دەردەخات.

**چالاک (Active):** دەتوانیت ئەم کارە لەسەر بکەیت یان نەکات بۆ چالاککردنەوەی یان نەکاتەوەی دۆمینەکە.

**ئایا دۆمین سەرەکییە؟ (Is Primary Domain?):** کڕیارەکان دەتوانن زیاتر لە یەک دۆمین مَپکراو بن بۆ هەر سایتێک. ئەم کارە بەکاربهێنە بۆ هەڵبژاردنی ئەوەی ئایا ئەمە دۆمین سەرەکییە بۆ ئەو سایتە تایبەتە.

**ئایا ئاسایشییە؟ (Is Secure?):** لەگەڵ ئەوەی Ultimate Multisite پێش ئەوەی دەستپێکردن بکات، چاک دەکات کە دۆمینەکە SSL گواستراوە یان نییە، تۆ دەتوانیت بە شێوەیەکی دەستی هەڵبژێریت کە دۆمینەکە لەگەڵ یان بدون SSL بڵاو بکرێتەوە. ئاگاداری ئەوە بکە کە ئەگەر ئەو وێبسایتیش SSLی نەبوو و هەوڵ بدات بە شێوەیەکی زور باجوری دەستپێبکەیت، ڕەنگە هەڵەیەک دروست بکات بۆت.

### مَپکردنی ناوی دۆمین تایبەت وەک یوان سایت (Mapping custom domain name as Subsite user)

بەڕێوەبەری یوان سایتان دەتوانن ناوەکانی دۆمین تایبەتی خۆیان لە داش보ار بەڕێوەبردنی یوان سایتەکەدا مَپ بکەن.

پێشتر، پێویستە دڵنیا بیت کە ئەم کارە لە ژێر ڕێکخستنی **Domain mapping** (مَپکردنی دۆمین) چالاک دەکەیت. وێنەی خوارەوە ببینە.

<!-- Screenshot unavailable: Domain mapping settings allowing subsite users to map domains via Customer DNS Management toggle -->

دەتوانیت ئەم کارەیش لە ژێر **Plan** (خەمانە) یان هەڵبژاردنی بەرهەمەکاندا لەسەر **Ultimate Multisite > Products** دیاری و ڕێک بخەیت.

![Custom Domains section on the product edit page](/img/config/product-custom-domains.png)

پێویە ئەگەر یەکێک لەو ڕێگاکانە چالاک بکرێت و بەکارهێنەرێکی subsite ڕێگەپێدراوە کە ناوی دۆمەینەکانی تایبەت (custom domain names) مَـاپ بکات، ئەو بەکارهێنەرە دەتوانێت یەکەی metaboxێک لەسەر لاپەڕەی **Account** ببینێت بە ناوی **Domains**.

<!-- Screenshot unavailable: metabox-ی Domains لە لاپەڕەی Accountی subsite لەگەڵ دغڕاندنی Domain -->

بۆ ئەوەی بتوانێت، دەتوانرێت بڕێكەری **Add Domain** بکات و پەنجەرەیەکی modal (modal window) دەدات کە چەند ڕێنماییێک لە نێویش هەیە.

<!-- Screenshot unavailable: Modal-ی Add Domain کە ڕێنمایی DNS A-record بۆ بەکارهێنەرانی subsite دەردەخات -->

ئەو کاتدا، دەتوانرێت بڕێكەری **Next Step** بکات و بەرەو زیادکردنی ناوی دۆمەینی تایبەت بگات. هەروەها دەتوانی چۆن دیارییە سەرەکی (primary domain) بێت یان نەبێت هەڵبژاردن بکات.

<!-- Screenshot unavailable: Form-ی Add Domain کە فیلدی ناوی دۆمەینی تایبەت و سوییچکردنی primary domain لەخۆدەگرێت -->

<!-- Screenshot unavailable: پێشوەختەی زیادکردنی Domain کە تێیدا پشکنینی DNS دەست پێدەکات -->

کلیک کردن لەسەر **Add Domain** دەست دەکات بە پڕۆسەی پشکنین و گەڕاندەوەی زانیاری DNS بۆ ناوی دۆمەینی تایبەت.

### دەربارەی Domain Syncing (هەڵگرتنی دۆمەین)

Domain Syncing پرۆسەیەکی ئەوەیە کە Ultimate Multisite ناوی دۆمەینی تایبەت لە هەژماری هاستکردنەکەت وەک دۆمەینی زیادە (add-on domain) زیاد بکات **بۆ ئەوەی مَـاپکردنی دۆمەین کار بکات**.

Domain syncing بە شێوەیەکی ئۆتۆماتیک ڕوودەدات ئەگەر پڕۆڤایدرێکی هاستکردنەکەت لەگەڵ تایبەتمەندی مَـاپکردنی دۆمەینی Ultimate Multisite هەبێت. ئێستا، ئەم پڕۆڤایدرانە نیشاندراون: _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ و _Cpanel._

کاتێک یەکگرتووی پڕۆڤایدر (host-provider integration) چالاک بێت، Ultimate Multisite دەتوانرێت ئەرکی دروستکردنی DNS یان subdomain بۆ ئەو سایتانە نوێکراوەکان لەلایەن پڕۆڤایدرەوە دابنێت. ئەگەر هیچ گرژێکی (integration) بۆ ئەو کارە نییە، کارەکە بە شێوەیەکی پاشبنەمایی (background job) هەڵدەدرێت بۆ ئەوەی ڕێگری لە تۆمارکردنی کارەکانی "no-op queue entries" بکات. پشکنینی DNS و SSL بۆ دۆمەینە مَـاپکراوەکان بەردەوام دەبێت لەسەر پرۆسەی عادي (normal domain-stage process).

توتال مالتسایت تنظیمات تحت تب **Integration** را فعال کنید.

![تب Integrations در تنظیمات Ultimate Multisite که ارائه‌دهندگان هاستینگ را نشان می‌دهد](/img/config/integrations-tab.png)

<!-- Screenshot unavailable: لینک‌های پیکربندی ارائه‌دهنده هاستینگ در تب Integrations settings -->
