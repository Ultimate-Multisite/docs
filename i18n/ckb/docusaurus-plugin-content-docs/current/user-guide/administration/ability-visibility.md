# دۆخی بینراوکردنەوەی تواناکان (Ability Visibility)

Superdav AI Agent v1.12.0 تایبەتمەندی **Ability Visibility** دەهێنێت کە کۆنتڕۆڵ دەکات چ ڕووبەرێک (surface) ئەو تواناکانەی دەردەخات. ئەمە بەڕێوەبەران دەدات دیاری بکەن چ تواناکانی ئەژمارە لە ڕێگەی REST API، MCP servers، WooCommerce integrations و دیکە ئامرازەکاندا بێت.

## Ability Visibility چییە؟ {#what-is-ability-visibility}

Ability Visibility سیستەمی ڕێگەپێدانێکە کە کۆنتڕۆڵ دەکات:

- **چ تواناکان** بۆ ئەژمارە دەبن
- **لە کوێ دەردەخەن** (REST API، MCP، WooCommerce و هتد)
- **کێ دەتوانێت دەستکارییان بکات** (لە ڕێگەی لیستەکانی ڕێگەپێدانەوەی هاوبەشکان - partner allow-lists)
- **چۆن پۆلێن دەکرێن** (ئەو کەیان ناسراوە یان نەکراوە)

هەر توانایەک ئاستێکی بینراوکردنەوەی هەیە کە دیاری دەکات لە چ ڕووبەرێکدا بەردەستە.

## ئاستەکان بۆ بینراوکردنەوە (Visibility Levels) {#visibility-levels}

### جیهانی (Public) {#public}

**تواناکانی جیهانی** لە هەموو شوێنێکدا دەبن:

- REST API endpoints
- MCP servers
- WooCommerce integrations
- Chat interfaces
- هەموو ڕۆڵە بەکارهێنەران (بە ڕێگەپێدانەکانی پێویست)

نموونە: `scaffold-block-theme`، `activate-theme`، `send-email`

### ناوخۆیی (Internal) {#internal}

**تواناکانی ناوخۆیی** تەنها لەناو دامەزراوەی WordPress یەکەت دەبن:

- Chat interfaces
- Admin panels
- بەکارهێنەرانی تۆڕکراو (Logged-in users) تەنها
- نەکراون ڕێگەی REST API یان هاوبەشەکانی دەرەوەیان دەدات

نموونە: `manage-settings`، `view-analytics`، `export-data`

### هاوبەش (Partner) {#partner}

**تواناکانی هاوبەش** تەنها بۆ هاوبەشانی لیستکراوەکان بەردەستن:

- پێویستی بە تۆمارکردنی ڕێگەپێدان لە لیستەکانی هاوبەشکان
- دەکرێت بە REST API ببنەوە لەگەڵ ناسنامەی هاوبەش (partner credentials)
- دەکرێت بە MCP ببنەوە لەگەڵ پەسەندکردنی هاوبەشی هاوبەش
- بۆ بەکارهێنەرانی ئاسایی بەردەست نین

نموونە: `bulk-import-users`، `modify-billing`، `access-analytics`

### ناتوانرێت (Disabled) {#disabled}

**تواناکانی ناتوانرێت** لە هیچ شوێنێکدا بەردەست نین:

- ڕێگەی هیچ ڕووبەرێک دەدات
- لە چاتدا بەردەست نین
- لە Admin panels بەردەست نین
- بۆ تواناکان کە کۆن بوون یان تاقیکردنەوەیان هەیە باشە

## Manage Visibility of Abilities {#managing-ability-visibility}

### Accessing Ability Visibility Settings {#accessing-ability-visibility-settings}

1. ไป بۆ **WordPress Admin** → **Superdav AI Agent** → **Settings**
2. کلیک کن لەسەر تابلۆی **Abilities** (تواناکان)
3. تۆ لیستی هەموو تواناکانی دامەزراوەکان دەبینیت و ئاستی بینینیان

### دۆزینەوەی وردەکاری تواناکان {#viewing-ability-details}

کلیک بکە سەر هەر تواناکێک بۆ ئەوەی ببینیت:

- **Name** (ناوی): ناسنامەی ئەو توانایە
- **Description** (وەرگێڕان): ئەو توانایە چی دەکات
- **Current visibility** (بینینی ئێستای): Public (بۆ هەمووان)، Internal (داخلي)، Partner (هەڵگر)، یان Disabled (وەسڵکراوە)
- **Partner allow-list**: چیا هاوکاران دەتوانن دەسیل بکەنە ناو (ئەمە ئەگەر بینینی Partner بێت)
- **Last updated** (کاتی نوێکردنەوە): کاتێک بینینەکە لە کۆتاییدا گۆڕاوە
- **Status** (دۆخی توانایە): Recognized (پێناسراوە) یان Unclassified (ناشناس)

### گۆڕینی ئاستی بینین {#changing-visibility-levels}

بۆ گۆڕینی بینینی توانایەک:

1. ئەو توانایە لە لیستەکەدا کلیک بکە
2. ئاستێکی نوێی بینین لە دراپداونەکەدا هەڵبژێرە
3. ئەگەر "Partner" هەڵبژارد، ناسنامەی هاوکاران بۆ لیستەکەی زیاد بکە
4. **Save** (سەردانکردن) کلیک بکە

نموونە:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### کارکردنی زۆر (Bulk Operations) {#bulk-operations}

بۆ گۆڕینی بینین بۆ چەند توانایەک:

1. لەسەر ئەو توانایانە پۆپ کردنی (check boxes) کلیک بکە
2. ئاستی بینینی نوێیەکە لە دراپداونەکەدا هەڵبژێرە
3. **Apply** (باشکردنەوە) کلیک بکە

## Partner Allow-List {#partner-allow-list}

**Partner allow-list** دیار دەکات چیا هاوکارانی دەرەکی دەتوانن دەسیل بکەنە تواناکانی ئاستی Partner.

### زیادکردنی هاوکاران {#adding-partners}

1. گەهارت بۆ **Superdav AI Agent** → **Settings** → **Partners**
2. **Add Partner** (هاوکار زیاد بکە) کلیک بکە
3. ناسنامەی هاوکارییەکە بنووسە (بە گشتی API key یان Organization ID)
4. بەختەوەری، ناوی هاوکار و وەسفیەک بۆ زیاد بکە
5. **Save** (سەردانکردن) کلیک بکە

### دیاریکردنی تواناکان بۆ هاوکاران {#assigning-abilities-to-partners}

پاش زیادکردنی هاوکارێک:

١. سەردانی **Abilities** (قابلیت‌ها) تبەکە بکە.
٢. ئامرازێک بەدوای ئەو ئەوانە بدۆزەرەوە کە بینینی هاوبەش (Partner visibility) هەیە.
٣. بۆ دەستکارییەکەی کلیک بکە.
٤. لە بەشی "Partner allow-list" (لیستی ڕێگەپێدان بۆ هاوبەشەکان)، بۆ ئەو هاوبەشانە چۆڵ بکە کە دەبێت دەستکارییان هەبێت.
٥. **Save** (سەردانکردن) کلیک بکە.

### دامەزلکەنی دەستکاریی هاوبەش (Revoking Partner Access) {#revoking-partner-access}

 بۆ ئەوەی دەستکارییەکی هاوبەش لە دەستپێکردنەوە بگرێت:

١. دەستکارییەکە دەستکاری بکە.
٢. چۆڵ بکە لە لیستەکەی هاوبەشەکە.
٣. **Save** (سەردانکردن) کلیک بکە.

هاوڕێیەکە بە خێرایی دەستکاریی ئەو ئامرازە لەدەست دەدات.

## دامەزراوەکانی نۆکلاسیکی (Unclassified Abilities) {#unclassified-abilities}

کاتێک پلاتفۆرمێکی تری (third-party ability) دامەزراو کە Superdav AI Agent بە ناسینەوەیەکش ناچار نییە، وایکردنی **Unclassified** (نۆکلاسیک) دەبێت.

### ئاگاداری ئەدمن بۆ دامەزراوەکانی نۆکلاسیکی {#admin-notice-for-unclassified-abilities}

ئەت دەبینیت ئاگاداری ئەدمنێک:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### داواکردنی دامەزراوەکانی نۆکلاسیکی (Reviewing Unclassified Abilities) {#reviewing-unclassified-abilities}

١. لە ئاگادارییەکەدا، **Review Abilities** (پێشینەوەی دەستکارییەکان) کلیک بکە.
٢. بۆ هەر دامەزراوەکێکی نۆکلاسیکی، ئەم شتان دەبینیت:
   - ناوی دەستکاری و وەسفی وردەکارییەکەی
   - سەرچاوە (ئایا ئەو پڵگین/ئادۆنە بوو کە لەسەر دامەزراندنی کرد)
   - ئاستی بینینی پێشنیارکراو
   - هەڵسارکردنی ئەمنی

٣. ئاستێکی بینین هەڵبژێرە:
   - **Public** (پەیوەست): بەوە بڕیار بدات کە دەستکارییەکە متمانە بکە؛ لە هەموو شوێنێک ڕابنرێتەوە.
   - **Internal** (داخلي): تەنها بۆ بەکارهێنانی ناوخۆیی سنووردار بکە.
   - **Partner** (هاوبەش): سنووردار بکە بۆ هاوبەشانی دیاریکراو.
   - **Disabled** (بەدواداچوون): ئەم دەستکارییە بەکار ناهات.

٤. بۆ شۆڕکردن، **Classify** (پێناساندن) کلیک بکە بۆ پاراستنی بڕیارەکەت.

### بۆچی دامەزراوەکانی نۆکلاسیکی پێویستە پیناسان؟ {#why-classify-unclassified-abilities}

پێناساندنی دەستکارییەکان:

- **ئامانتی زیاتر دەکات**: تۆ بە ڕوونی پەسەند دەکەیت چی دەردەبڕێت.
- **دڵەڕاوەکان ڕێدەگرێت**: دامەزراوەکانی نۆکلاسیکی بە شێوەیەکی پیش‌تەوا پێشنیارکراو بەکارنەهێنرێت.
- **ئامرازەکان دەگشێنێتەوە**: دوای پیناسان، ئەو دەستکارییە دەبێت ئامادە بێت بۆ بەکارهێنان.
- **بڕیارەکان تۆمار دەکات**: هەڵبژاردنەکان بۆ تاقیکردنی کار تۆمار دەکرێن.

## بینایی در سطوح مختلف {#visibility-in-different-surfaces}

### REST API {#rest-api}

**قابلیت‌های عمومی** از طریق endpointهای REST در دسترس هستند:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**قابلیت‌های داخلی و شرکایی** از طریق REST API در دسترس نیستند.

### سرورهای MCP {#mcp-servers}

**قابلیت‌های عمومی** از طریق MCP در دسترس هستند:

MCP Server: Superdav AI Agent
قابلیت‌های موجود:
- scaffold-block-theme
- activate-theme
- send-email

**قابلیت‌های داخلی** از طریق MCP نمایش داده نمی‌شوند.

**قابلیت‌های شرکایی** فقط با اعتبارنامه‌های شرکای در دسترس هستند.

### یکپارچه‌سازی ووکامرس (WooCommerce Integration) {#woocommerce-integration}

**قابلیت‌های عمومی** مربوط به ووکامرس در دسترس هستند:

- مدیریت محصول
- پردازش سفارش
- ارتباط با مشتری

**قابلیت‌های داخلی** به ووکامرس نمایش داده نمی‌شوند.

### رابط چت (Chat Interface) {#chat-interface}

**تمام قابلیت‌ها** (عمومی، داخلی، شرکایی) بسته به مجوزهای کاربر در چت در دسترس هستند:

- مدیران همه قابلیت‌ها را می‌بینند
- کاربران عادی فقط قابلیت‌های عمومی را می‌بینند
- شرکا قابلیت‌های عمومی + شرکایی (اگر لیست سفید شده باشند) را می‌بینند

## بهترین شیوه‌های امنیتی {#security-best-practices}

### اصل حداقل امتیاز (Principle of Least Privilege) {#principle-of-least-privilege}

- قابلیت‌ها را روی محدودترین سطح دید تنظیم کنید که همچنان اجازه استفاده از آن‌ها را بدهد.
- برای عملیات حساس از دید شرکایی استفاده کنید.
- قابلیت‌هایی را غیرفعال کنید که از آن‌ها استفاده نمی‌کنید.

### ممیزی‌های منظم (Regular Audits) {#regular-audits}

- هر ماه دید قابلیت‌ها را بررسی کنید.
- به دنبال قابلیت‌های بدون دسته‌بندی باشید.
- دسترسی برای شرکای غیرفعال شده را حذف کنید.

### لاگ‌برداری و نظارت (Logging and Monitoring) {#logging-and-monitoring}

- پایش کنید کدام قابلیت‌ها بیشتر استفاده می‌شوند.
- الگوهای دسترسی شرکا را ردیابی کنید.
- در مورد استفاده غیرمعمول از قابلیت‌ها هشدار دهید.

### قابلیت‌های شخص ثالث (Third-Party Abilities) {#third-party-abilities}

- قبل از فعال کردن، قابلیت‌های شخص ثالث را بررسی کنید.
- آن‌ها را به صراحت طبقه‌بندی کنید.
- با دید داخلی یا شرکایی شروع کنید.
- فقط پس از تأیید، به عمومی ارتقا دهید

## Photo-darye {#troubleshooting}

**ئەمرکارییەک لە لیستەکەدا نیشانناوە**
- دڵنیابConvertTo the ability is installed and active
- چاک بکە کە پلاتین/ئەدەدۆنەکە چالاکە (enabled) بێت
- پەڕەکە دووبارە بارکەرەوە (Refresh the page)

**ناتوانم نیشانین بە گۆڕینی بینراو (visibility)ی ئەمرکارییەکی دیاریکراو**
- دڵنیابConvertTo you have administrator permissions
- چاک بکە کە ئەمرکارییەکە لەلایەن پلاتینێک یەکلاکەرەوە (locked) نییە
- هەوڵ بدە پلاتینە تێکچووەکان بە ئازادکردنەوە بێدەنگ بن (disabling conflicting plugins)

**هۆمارێک ناتوانێت دەسیل بکات بۆ ئەمرکارییەک**
- دڵنیابConvertTo the partner is in the allow-list
- چاک بکە کە بینراوی ئەمرکاریەکە بە Partner دیاریکراوەتەوە (set to Partner)
- دڵنیابە سەرچاوەکانی پارێزگاریکردنی هاوکارەکە دروستن (credentials are correct)
- لە تۆمارەکانی API بۆ هەڵەی پەسەندکردن (authentication errors) چاک بکە

**ئەمرکارییەک کە پۆلێنکراوە نییە بەردەوام دەبێتەوە**
- ئەوان بەپێی ئاماژەکانی ئەدمن دا وا بکە و پۆلێن بکە (Review and classify them in the admin notice)
- دڵنیابە پلاتینەکە کە ئەم ئەمرکارییە پێدەدات، نوێترین بێت (up to date)

## گواستنەوە لە ڕێگەی Mode کۆن (Legacy Mode Migration) {#migration-from-legacy-mode}

ئەگەر لەو کاتی نوێکردنەوەی Superdav AI Agent دەکەویت:

- هەموو ئەمرکارییە ئێستاکان بە شێوەیەکی ئۆتۆماتیک وەک Public پۆلێن دەکرێن
- بینەرەوە و بینراو (visibility)یەکەت بگۆڕە بە پێویستی
- هیچ کارێک پێویست نییە لە ماوەی ئەوەی کە دەسیاری وردتر بکەیت

بۆ زیاتر زانیاری لەسەر گواستنەوە بۆ یەکگرتووکردنی APIی ئەمرکاریی تایبەت، سەردانی **گواستنەوەی Mode سێیەم (Third-Party Mode Migration)** بکە.

## هەنگاوەکانی دوای ئەمە (Next Steps) {#next-steps}

پاش دیاریکردنی بینراوی ئەمرکاری:

1. **بینەرەوەکانت بەدوอดู**: هەموو ئەمرکارییە دامەزراوەکان چاک بکە
2. **ئەمرکارییەک کە پۆلێنکراوە نییە پۆلێن بکە**: وەڵام بدە لە ئاماژەکانی ئەدمنەوە
3. **هاوکاران SETUP بکە**: هاوکاران زیاد بکە ئەگەر بینراوی Partner بەکاردەهێڵیت
4. **بەکاتەوەی بەکارهێنان**: چۆن تێبینی بکە کە گۆڕانکارییەکان زیاتر بەکاردێت
5. **دۆکیومێنت بکە**: یادداشت بۆ ئەو هۆکاران دەبێت کە بۆ هەر ئاستێکی بینراو یەکێک هەڵبژاردەت
