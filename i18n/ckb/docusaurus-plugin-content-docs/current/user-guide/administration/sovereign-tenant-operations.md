# کارکردن بەکارھاتە سوورین (Sovereign Tenant Operations)

Ultimate Multisite 1.2.0 ئامادەکردنی ئامرازەکانی کار بۆ تەنە سوورینەکان (sovereign tenants): ئەو سایتە ساب-سایتانەی کە لەگەڵ داتابێس، ڕیشەی فایڵسیستم و پۆلێتی ڕێڕەوی خۆیان دەبنەوە، بەڵام لە ڕووی بینین لەلایەن ئیدارەی تۆڕەکەوە دەبینرێت.

ئەم پەڕە بەکاربهێنە کاتێک بەڕێوەدەبات سایتەکانی کڕیارانی جیاواز، گواستنەوەی سایتە دوورەکان (remote-site handoffs)، یان گواستنەوەکان کە سایتێکی سادەی ناو ساب-سایتێکی سوورین دەگواسنە ژێر پێکهاتەی زیرەکی سوورین.

## چ شتێک بۆ ئیدارەکان دەگۆڕێت

- **Autologinی تەنە無شێوەیی تەنە سوورین (Stateless tenant autologin)** — ئیدارەکانی تۆڕ دەتوانن سەردانی تەنە سوورین بکەن بەبێ پشت بە شتێکی کۆنە لەسەر پۆلێتی هاوبەش. توکنەکەی SSO تایبەتمەندە، سەرچاوەی خۆی دیاریکراوە (origin-pinned)، پارێزراوی دۆنگەوەی دووبارەکران (replay-protected) و بۆ ماوەیەکی کورت کۆتایی پێدەهێنێت.
- **ڕێڕەوی دەروستکراو بۆ تەنە سوورین (Sovereign-aware routing)** — تۆڕە کۆنە جیاوازەکان و تەنە سوورینەکان لە ڕێگەی هەمان پۆلێتی سایتەکە چارەسەر دەکەن، ئەمەش جیاوازییە سەرەتاییەکانی دەستپێکردن (bootstrap) لە نۆڵ و دامەزراوەکانی جیاواز کەم دەکاتەوە.
- **دۆخی گواستنەوەی پشکنینکراو (Verified migration state)** — پشکنینی گواستنەوە یەکสันstvenوکردنی بەکارهێنەران، ڕێزماندنی نووسەری داتابێس، دۆخی چڕکردنی قۆناغەکان (queue drain status)، و نەبوونی خشتەی کۆن پێش ئەوەی تەنە سوورینەکە تەواو دەپێناسراوە.
- **تەواوکردنی زیاتر سەلامەتتر (Safer teardown)** — لە ئێستادا، گۆڕینی تەنە سوورینەکە بە شێوەیەکی پاک ئەو ناسنامانەی تەنە سوورینەکە دەداتەوە بۆ ئەوەی تەنەکان کە دیارکراون نەهێنرێت، ناتوانن دەسیگای داتابێسێکی کۆن و پاشانده بکەن.

## سەردانی تەنە سوورینێک

1. **Network Admin > Ultimate Multisite > Sites** بکەرەوە.
2. تەنە سوورینەکە هەڵبژێرە.
3. کاتێک بەکارخستنی **Visit (SSO)** دەتوانیت، لە کۆپی کردن پۆسڵان یان دروستکردنی خشتەی ئیدارەیەتییەکی موقت بەکاربهێنە.

ڕێگای سەردانەکە توکنێکی لاوەکی کورت بۆ ئەو تەنە سوورینە دروست دەکات و ڕووداوەکە لە تۆمار پشکنینی تەنە سوورینەکەدا تۆمار دەکرێت. ئەگەر دۆمبۆنەکە شکستپێکرد، چاک بکە ئایا ناو دامێنی تەنە سوورینەکە بە باشی چارەسەر دەبێت و ئایا تەنە سوورینەکە دەتوانرێت بگاتە کۆتایی پەیج SSO-یە لە لای تۆڕەکە.

## لیستێکی چککردن بۆ کارکردنی سایت دوورەکان (Remote-site operations checklist)

پێش ئەوەی تەنە سوورین یان تەنە دوورەکان دەگۆڕیت، ئەم خاڵانە پشتڕاست بکەرەوە:

دومینی تاننت (tenant domain) به میزبان اشاره می‌کند که مالک فایل سیستم تاننت است.
میزبان پایگاه داده تاننت با اتصال میزبان پیکربندی شده برای آن نصب مطابقت دارد.
دستورهای تأیید مهاجرت (migration verification commands) برای تاننت موفقیت‌آمیز هستند.
صف‌های انتظار مهاجرت ناهمزمان (Async migration queues) قبل از اعمال تغییرات DNS یا مالکیت تخلیه می‌شوند.
کاربر ادمین تاننت در طول مهاجرت فراهم شده و می‌تواند از طریق SSO وارد شود.

## حذف تاننت‌های حاکمیتی (sovereign tenants)

حذف یک تاننت حاکمیتی عملی است و مخرب خواهد بود. ابتدا وضعیت پشتیبان‌گیری و خروجی را تأیید کنید، سپس از صفحه مدیریت سایت آن را حذف نمایید. جریان حذف نسخه ۱.۲.۰ اعتبارنامه‌های پایگاه داده تاننت را به عنوان بخشی از پاکسازی حذف می‌کند، اما مدیران همچنان باید بررسی کنند که آیا کاربران دیتابیس و پوشه‌های سطح میزبان هنگام استفاده از پنل‌های میزبانی خارجی حذف شده‌اند یا خیر.

:::warning
در حالی که تأیید مهاجرت هنوز در حال اجراست یا وظایف ارسال ناهمزمان (async push jobs) در صف هستند، تاننت حاکمیتی را حذف نکنید. منتظر بمانید تا تأیید کامل شود تا فرآیند حذف اعتبارنامه‌های مورد نیاز برای وظایف معلق را از بین نبرد.
:::
