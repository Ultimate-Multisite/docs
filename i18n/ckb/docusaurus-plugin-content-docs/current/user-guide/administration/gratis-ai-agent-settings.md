# تنظیمات رایگان عامل هوش مصنوعی (AI Agent)

صفحه **Settings → Advanced** در Gratis AI Agent امکان پیکربندی سطح مدیر را برای ادغام‌های بک‌اند که در نسخه v1.5.0 معرفی شده‌اند، فراهم می‌کند. این صفحه مستند کردن فیلدهای **Feedback Endpoint** و فرمت مورد انتظار آن‌هاست.

## دسترسی به تنظیمات

۱. در پنل مدیریت وردپرس، به مسیر **Gratis AI Agent → Settings** بروید.
۲. روی تب **Advanced** کلیک کنید.

## پیکربندی نقطه پایان بازخورد (Feedback Endpoint)

نقطه پایان بازخورد درخواست‌های POST را از عامل هوش مصنوعی دریافت می‌کند هر زمان که کاربر از طریق دکمه لایک/دیسلایک، بنر اعلان خودکار یا دستور `/report-issue` بازخورد ارسال کند.

| فیلد | توضیحات |
|---|---|
| **Feedback Endpoint URL** | آدرس (URL) که ارسال‌های بازخورد را به عنوان درخواست‌های HTTP POST با بدنه JSON دریافت می‌کند. |
| **Feedback API Key** | یک توکن برنده (bearer token) است که در هدر `Authorization` هر درخواست بازخورد ارسال می‌شود. اگر نقطه پایانی شما نیازی به احراز هویت ندارد، آن را خالی بگذارید. |

### محموله JSON مورد انتظار (Expected JSON Payload)

نقطه پایان بازخورد شما باید یک بدنه JSON با حداقل فیلدهای زیر را بپذیرد:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

بسته به زمینه مکالمه، ممکن است فیلدهای دیگری نیز در محموله وجود داشته باشند.

### مقادیر `triage_category`

لایه طبقه‌بندی هوش مصنوعی یکی از مقادیر زیر را قبل از ارسال محموله به جلو به `triage_category` اختصاص می‌دهد:

| مقدار | معنی |
|---|---|
| `factual_error` | دستیار اطلاعات واقعی اشتباهی ارائه داده است. |
| `unhelpful_answer` | پاسخ از نظر فنی درست بوده اما مفید نبوده است. |
| `inappropriate_content` | پاسخ حاوی محتوایی بوده که نباید به کاربران نمایش داده شود. |
| `other` | بازخورد با هیچ دسته‌بندی شناخته‌شده‌ای مطابقت نداشت. |

### احراز هویت (Authentication)

ئەگەر کۆتاییی پۆنتەکەت پێویستی بە ڕێگەپێدان (authentication) هەبێت، **Feedback API Key** فیلدی بکە بە توکنەکە (bearer token) یاری خۆت. ئەژمارەکە دەڵێت:

```
Authorization: Bearer <your-api-key>
```

ئەگەر فیلدی **Feedback API Key** خاوەنی بێت، هیچ سەردانیی `Authorization` headerێک نێردراوە.

### ناتواندن کۆکردنەوەی فیدبیک (Disabling Feedback Collection)

هەردوو فیلدی **Feedback Endpoint URL** و **Feedback API Key** بەپێی خۆیان بخاتە خاڵێکی پەت (blank). دۆخی دەستکەوتنی (thumbs-down button) و ڕووی فیدبیک بۆ بەکارهێنەران لەخۆ دەبینراوە، بەڵام داواکارییەکان بۆ هیچ خزمانی دەرەکییەک نەنێردرێت.

## Brave Search API Key

شێوەزادا لە تابلۆی **Advanced** (ناوەندی)دا، فیلدی **Brave Search API Key** دەتوانێت توانای [Internet Search](../configuration/internet-search) ی بەکاربهێنانی کردەوە.

| فیلد | وەسفکردن |
|---|---|
| **Brave Search API Key** | توکن API یەکێتییەکەت لە داشبۆردەوڵگرتارەکانی Brave Search. پێویستە بۆ چالاککردنی گەڕان لە ئینتەرنێت لە یارمەتیی ئەژمارەکەدا (AI assistant). |

ناوەکانی فیلد لینکێک دەبینن کە دەتوانیت بەکاری بهێنیت بۆ پەڕاندنی Brave Search API. بۆ ناتواندنەوەی گەڕان لە ئینتەرنێت، بیکە بە پەت.

بۆ زیاتر زانیارییەکانی بەکارهێنەران دەربارەی ئەم تایبەتمەندییە، سەردانی [Internet Search](../configuration/internet-search) بکە.

## Feature Flags (راگەکانی تایبەت)

وەک پێشتر لە v1.9.0دا نیشانکراوە، تابلۆی **Settings → Feature Flags** دەتوانێت سوییچەکانی چالاککردن بۆ ئەو کارانەی کە ئاسایی نین و بەپێی متمانە (optional functionality). هەر راگێک یان چالاک بکرێت یان ناتوانرێت لە سەرانسەری تۆڕەکەدا؛ لە ئێستادا هیچ ڕێگرییەکی تایبەت بە سایتێک نییە.

### دەستگەیشتن بۆ Feature Flags

1. لە ئامێر ADMINISTRATORی WordPress، بگواشبە **Gratis AI Agent → Settings**.
2. سەری **Feature Flags** بکە.

### Feature Flags ی کۆنترۆڵی دەستگەیشتن (Access Control Flags)

| Flag | Default | Description |
|---|---|---|
| **Restrict to Administrators** | Off | کات کردن برای مدیران فقط زمانی فعال می‌شود که کاربران با نقش `administrator` بتوانند پنل چت عامل هوش مصنوعی را باز کنند. سایر نقش‌ها پیام "با مدیر خود تماس بگیرید" را می‌بینند. |
| **Restrict to Network Admins** | Off | در شبکه‌های چند سایت، وقتی این گزینه فعال است، فقط سوپر ادمین‌ها می‌توانند از عامل استفاده کنند. ادمین‌های سایت‌های منفرد مسدود می‌شوند. اگر هر دو گزینه فعال باشند، این تنظیم اولویت دارد و بر "محدود کردن برای مدیران" غلبه می‌کند. |
| **Allow Subscriber Access** | Off | وقتی فعال است، کاربران با نقش `subscriber` می‌توانند از رابط چت استفاده کنند اما محدود به قابلیت‌های فقط خواندنی هستند (بدون ایجاد پست یا تغییر تنظیمات). |
| **Disable for Non-Members** | Off | این گزینه با وضعیت عضویت Ultimate Multisite یکپارچه می‌شود. وقتی فعال است، چت برای سایت‌هایی که عضویت فعالی ندارند مخفی می‌شود. |

### پرچم‌های برندینگ (Branding Flags)

| Flag | Default | Description |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | این خط نشان‌دهنده برند در پایین ویجت چت را حذف می‌کند. برای استقرار با برچسب سفید (white-label) توصیه می‌شود. |
| **Custom Agent Name** | *(خالی)* | نام پیش‌فرض "Gratis AI Agent" در هدر چت و منوی ادمین را با نام محصول خود جایگزین می‌کند. اگر خالی بگذارید، از مقدار پیش‌فرض استفاده می‌شود. |
| **Hide Agent Picker** | Off | وقتی فعال است، کاربران نمی‌توانند بین پنج عامل داخلی جابجا شوند. عامل فعلی به تنظیمات پیش‌فرض در Settings → General تغییر داده شده و ثابت می‌ماند. |
| **Use Site Icon as Chat Avatar** | Off | آیکون هوش مصنوعی پیش‌فرض در هدر ویجت چت با آیکون سایت وردپرس (که در Appearance → Customize → Site Identity تنظیم می‌شود) جایگزین می‌شود. |

### اعمال تغییرات

پس از تغییر هر پرچمی، روی **Save Settings** کلیک کنید. تغییرات بلافاصله اعمال می‌شوند — نیازی به پاک کردن کش یا فعال‌سازی مجدد پلاگین نیست.
