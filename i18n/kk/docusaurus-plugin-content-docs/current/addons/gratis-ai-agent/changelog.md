# Өзгерістер журналы

## 1.9.0 — 2026-04-28 күні шығарылды {#190--released-on-2026-04-28}

### Жаңа {#new}

- **`create_contact_form` мүмкіндігі** — белсенді форма plugin-ін (Contact Form 7, WPForms, Fluent Forms немесе Gravity Forms) пайдаланып байланыс формасын жасайды және кез келген жазбаға немесе бетке ендіруге дайын shortcode қайтарады.
- **`set_featured_image` мүмкіндігі** — бар Media Library тіркемесінің ID-сінен немесе қашықтағы URL-ден жазбаға немесе бетке таңдаулы сурет тағайындайды; URL берілгенде суретті автоматты түрде импорттайды.
- **`batch_create_posts` мүмкіндігі** — бір мүмкіндік шақыруында бірнеше жазба жасайды. `create_post` сияқты параметрлерді қолдайды, әр жазба бойынша сәттілік/сәтсіздік туралы хабарлайды және қосымша `stop_on_error` режимі бар.
- **`page_template` параметрі** — `create_post` және `update_post` ішіне қосылды. Жасау немесе жаңарту кезінде PHP бет үлгісі файлын (мысалы, `page-full-width.php`) тағайындайды. Тақырыптың әдепкі мәніне қайтару үшін `update_post` ішіне бос жол беріңіз.
- **Клиент жағындағы скриншот мүмкіндіктері** — `capture_screenshot`, `compare_screenshots` және `review_page_design`. Сервер жағындағы headless браузер арқылы тірі беттердің толық немесе viewport скриншоттарын түсіреді, екі скриншотты салыстырады және орналасу, типография, түс және қолжетімділік бойынша AI жасаған дизайн шолуын береді.
- **Бес кіріктірілген агент** — Content Writer, Site Builder, Design Studio, Plugin Manager және Support Assistant. Әр агенттің арнайы құралдар жиыны, бейімделген жүйелік prompt-ы және бастапқы ұсыныстары бар. Чат тақырыбындағы жаңа **Agent Picker** арқылы ауыстыруға болады. [Кіріктірілген агенттер](../../user-guide/configuration/built-in-agents) бөлімін қараңыз.
- **Функция жалаушалары** — қолжетімділікті басқару ауыстырғыштары (әкімшілермен шектеу, желі әкімшілерімен шектеу, жазылушыларға қолжетімділік, мүше еместер үшін өшіру) және брендтеу опциялары (төменгі деректемедегі атрибуцияны жасыру, реттелетін агент атауы, агент таңдағышты жасыру, сайт белгішесін чат аватары ретінде пайдалану) бар жаңа **Settings → Feature Flags** қойындысы. [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings) бөлімін қараңыз.
- **Соңғы сессияны қалпына келтіру** — чат панелі енді бет жүктелгенде және виджет ашылғанда ең соңғы әңгімені автоматты түрде қайта жүктейді, сондықтан беттер арасында өткенде контекст ешқашан жоғалмайды.
- **Plugin әрекет сілтемелері** — Settings және Abilities Registry бөлімдеріне жылдам сілтемелер енді WordPress **Plugins → Installed Plugins** экранында plugin сипаттамасының астында көрсетіледі.

### Жақсартылды {#improved}

- **Сурет көзін қайта көру** — агент енді жүктеп алу сәтсіз болғанда AI жасаған суретке ауыспас бұрын барлық бапталған тегін сурет көздерін қайта байқап көреді.
- **Модель ақпараты панелі** — чат тақырыбында әрдайым көрінеді; енді бірінші хабардан кейін жасырылмайды.
- **Автоматты айналдыру әрекеті** — пайдаланушы оқу үшін жоғары айналдырғанда автоматты айналдыру кідіреді; қалқымалы **Төменге айналдыру** батырмасы пайда болады және пайдаланушы ең соңғы хабарға жеткенде автоматты түрде жабылады.
- **Agent Picker UI** — әр агентке арналған белгішелері бар форма-кесте қабаттамасы ретінде қайта жасалды, бұл агенттерді бір қарағанда анықтап, ауыстыруды жеңілдетеді.
- **Lazy-loaded JS бөліктері** — чат виджетінің бастапқы JavaScript бумасы енді lazy-loaded бөліктерге бөлінді, бұл бастапқы бума өлшемдерін 75–90% азайтады.
- **Чат виджетін қайта жобалау** — бұрынғы реттелетін аватардың орнына бірыңғай AI белгішесі қойылды; кіріктірілген агент жүйесімен үйлесімді.
- **URL сілтемеге айналдыру** — жүйелік хабарларда және қате хабарларының көпіршіктерінде пайда болатын URL-дер енді басылатын сілтемелер ретінде көрсетіледі.

### Түзетілді {#fixed}

- **Мүмкіндіктердің табылуы** — барлық мүмкіндіктер агенттің құралдар тізімінде сенімді түрде көрінуі үшін сипаттамалар, жүйелік prompt сілтемелері және namespace сәйкестігі түзетілді.
- **Провайдерлер кэші** — провайдерлер енді нұсқа санағышы арқылы бүкіл сайт бойынша кэштеледі, бұл multisite желілерінде ескірген провайдер мәселелерін болдырмайды.
- **`ability_invalid_output`** — 12 мүмкіндік өңдегішінде шешілді; барлығы дұрыс құрылымдалған JSON жауаптарын қайтарады.
- **`pending_client_tool_calls` pipeline** — клиент жағындағы құрал шақырулары дұрыс аяқталып, нәтижелерді модельге қайтаруы үшін басынан аяғына дейін жалғанды.
- **Тарих тартпасы** — қайтаруға болмайтын өзгерістер қайтару тізімінен алынып тасталды; **Толық тарихты көру** сілтемесі енді дұрыс жұмыс істейді.
- **Өзгерістер/қайтару жүйесі** — бес бөлек қате түзетіліп, жаңа әкімші интерфейсі астында біріктірілді.
- **Save Settings toast** — **Save Settings** басылғаннан кейін snackbar хабарламасы енді сенімді түрде пайда болады.
- **Себет контекстік мәзірі** — элементтерді себет көрінісінен шықпай-ақ түбегейлі жоюға болатындай **Біржола жою** опциясы қосылды.
- **Өңдеу және қайта жіберу** — өңдеу белгішесін басу енді тредтегі барлық хабарларды емес, тек басылған хабарды ғана өңдеу режиміне енгізеді.
- **Чат орналасуының биіктігі** — plugin арқылы енгізілген мазмұн (әкімші ескертулері, баннерлер) беттің үстінде пайда болғанда чат панелі биіктігін бейімдейді, бұл енгізу аймағының экраннан тыс ығысуын болдырмайды.

## 1.4.0 — 2026-04-09 күні шығарылды {#140--released-on-2026-04-09}

### Жаңа {#new-1}

- **WP-CLI benchmark command** (`wp gratis-ai-agent benchmark`) — CI конвейерлері мен модельді бағалау жұмыс процестері үшін пәрмен жолынан Agent Capabilities v1 эталондық жиынтығын іске қосады. Әр сұрақ бойынша іске қосуды, провайдер/модель қайта анықтауларын және JSON/CSV шығысын қолдайды.
- **Agent Capabilities v1 эталондық жиынтығы** — бағалау және токен/ұзақтық есептері бар толық қабілеттер ауқымын тексеретін күрделі, көпқадамды сұраулардың құрылымдалған жиынтығы.
- **Custom Post Type қабілеттері** — `register_post_type`, `list_post_types`, `delete_post_type`. Тіркеулер қайта іске қосулар арасында options кестесі арқылы сақталады.
- **Custom Taxonomy қабілеттері** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. Қосымша rewrite slug-тары бар иерархиялық және жазық таксономияларды қолдайды.
- **Design System қабілеттері** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. Бес іріктелген preset қамтиды: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **Global Styles қабілеттері** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. WordPress Global Styles API арқылы theme.json мәндерін оқиды және жазады.
- **Навигация мәзірін басқару қабілеті** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. Кірістірілген мәзірлерді және theme орналасуын тағайындауды қолдайды.
- **Options басқару қабілеті** — `get_option`, `set_option`, `delete_option`, `list_options`. Маңызды WordPress options параметрлерін өзгертуден қорғайтын кіріктірілген қауіпсіздік blocklist қамтиды.
- **Орнатылатын қабілеттер тізілімі** — `list_available_abilities`, `install_ability`, `recommend_plugin`. WordPress plugin-дері ретінде таратылатын қабілет бумаларын тауып, белсендіреді.
- **Site Builder Orchestration v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. Plugin табу, қадам шығысы сілтемелері, прогресті бақылау және автономды қате қалпына келтіруі бар көпфазалы сайт құру жоспарлары.
- **Мейрамхана веб-сайты эталондық сұрағы** (`q-restaurant-website`) — CPT тіркеуін, design system-ді, навигацияны және plugin табуды қамтитын толық циклді тест.
- **AI провайдер коннектор plugin-дері** жергілікті әзірлеу орнатымын жылдамдату үшін WordPress Playground blueprints-ке қосылды.

### Жақсартылды {#improved-1}

- README AI провайдер коннекторы құжаттамасымен және орнату нұсқауларымен жаңартылды.

### Түзетілді {#fixed-1}

- `main` тармағындағы 25 PHPUnit тест сәтсіздігі шешілді.
- `blueprint.json` ішіндегі GitHub релиздері URL пішімі түзетілді.
- Ескі ID-лермен қайшылықтарды болдырмау үшін тапсырма ID-лерін қайта нөмірлеу.
