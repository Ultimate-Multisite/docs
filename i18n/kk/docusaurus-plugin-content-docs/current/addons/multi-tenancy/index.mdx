# Көп жалға алушылық

Өнімділік пен қауіпсіздік үшін ішкі сайттарды серверлер мен дерекқорлар бойынша таратыңыз

## 1.2.0 нұсқасының шығарылымдағы басты жаңалықтары {#version-120-release-highlights}

Multi-Tenancy 1.2.0 өндірістік оқшаулау жұмыс процестері үшін дербес жалға алушы негізін қосады:

- Қысқа мерзімді, қайта ойнатудан қорғалған SSO кірулері бар күйсіз жалға алушының автоматты кіруі.
- Дербес жалға алушылар үшін әр ішкі сайтқа арналған дерекқор және файлдық жүйе оқшаулауы.
- Хостты ескеретін жалға алушыны дайындауға арналған Hostinger көп жалға алушылық мүмкіндігі.
- Ескі деректерді тексерулер және асинхронды жіберу өңдеуі үшін дербес көшіруді растау қақпалары.
- Bedrock, FrankenPHP және localhost орнатулары үшін бір машинадағы дерекқор хостын байланыстыруды баптау мүмкіндігі.
- Жалға алушыны бастапқы іске қосу, дерекқор жазушысы, схема бастапқы іске қосуы және пайдаланушыны дайындау жақсартулары.
- Жалға алушы тіркелгі деректерін тазалайтын қауіпсіздеу дербес жою.

Пайдаланушы нұсқаулықтарын қараңыз: [дербес көшіру жұмыс процесі](/user-guide/getting-started/sovereign-migration-workflow), [көп жалға алушылық оқшаулауы](/user-guide/configuration/multi-tenancy-isolation) және [дербес жалға алушы операциялары](/user-guide/administration/sovereign-tenant-operations).

## Талаптар {#requirements}

- WordPress 5.3 немесе одан жоғары
- PHP 7.4 немесе одан жоғары
- Multisite Ultimate плагині (белсенді)

## Орнату {#installation}

1. Қосымша файлдарын `/wp-content/plugins/` каталогыңызға жүктеп салыңыз
2. Плагинді WordPress ішіндегі 'Plugins' мәзірі арқылы белсендіріңіз
3. Қосымша баптауларын желі әкімшісі бөлімінде баптаңыз

## Әзірлеу {#development}

### Баптау {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Тестілеу {#testing}

Қосымша толыққанды тестілеу фреймворкін қамтиды:

- **Модульдік тесттер**: WordPress тест жинағы бар PHPUnit негізіндегі тесттер
- **Негізгі тест класы**: Жиі кездесетін тест сценарийлері үшін көмекші әдістерді ұсынады
- **Тест қамтуы**: Код қамтуын бақылауға бапталған
- **Multisite тестілеуі**: Тесттер multisite ортасында орындалады

### Код стандарттары {#code-standards}

- **WordPress кодтау стандарттары**: PHPCS арқылы міндетті түрде орындалады
- **PHP 7.4+ үйлесімділігі**: Заманауи PHP мүмкіндіктеріне қолдау көрсетіледі
- **Статикалық талдау**: Түрлерді тексеру үшін PHPStan интеграциясы
- **Кодты жаңғырту**: PHP жаңартуларына арналған Rector ережелері

### Құрастыру жүйесі {#build-system}

- **Ресурстарды өңдеу**: CSS/JS файлдарын ықшамдау
- **Аударма**: POT файлын жасау
- **Пакеттеу**: Таратуға арналған архив жасау
- **Әзірлеу режимі**: Қатені түзетуге арналған ықшамдалмаған ресурстар

## Конфигурация {#configuration}

Қосымша желі әкімшісі интерфейсі арқылы қолжетімді әртүрлі конфигурация параметрлерін қолдайды.

### Дербес жалға алушы конфигурациясы {#sovereign-tenant-configuration}

Дербес жалға алушылар үшін трафикті жалға алушыға бағыттамас бұрын жалға алушы дерекқорын, файлдық жүйе түбірін, дерекқор хостын байланыстыруды және хост мүмкіндігін баптаңыз. Дерекқор тіркелгі деректеріне, хост байланыстыруына, бағыттауға немесе пайдаланушыны дайындауға әсер ететін әр өзгерістен кейін көшіруді растауды іске қосыңыз.

`localhost` мәнін жалға алушының орындау ортасы мен дерекқор рұқсаттары жергілікті сокетті немесе бір машинадағы қатынасты күткен кезде ғана пайдаланыңыз. Провайдеріңіз баптаған рұқсат хосты сол болса, `127.0.0.1` немесе қызмет хост атауын пайдаланыңыз.

### Hostinger мүмкіндігі {#hostinger-capability}

Hostinger көп жалға алушылық мүмкіндігі ортақ Hostinger интеграция токенімен жұмыс істейді. Растау алдында доменнің hPanel ішінде бар екенін, DNS Hostinger есептік жазбасына бағытталғанын және жалға алушы дерекқор баптаулары hPanel-ге сәйкес келетінін растаңыз.

## Ілмектер мен сүзгілер {#hooks-and-filters}

### Әрекеттер {#actions}

- `ultimate-multisite-addon-template_init` - Қосымша бастапқы іске қосылғаннан кейін іске қосылады
- `ultimate-multisite-addon-template_loaded` - Барлық қосымша файлдары жүктелгеннен кейін іске қосылады

### Сүзгілер {#filters}

- `ultimate-multisite-addon-template_settings` - Қосымша баптауларын өзгерту
- `ultimate-multisite-addon-template_enabled` - Қосымшаны қосу/өшіру күйін қайта анықтау
