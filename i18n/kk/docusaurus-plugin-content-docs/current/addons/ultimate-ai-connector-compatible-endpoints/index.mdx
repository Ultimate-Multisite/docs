# Үйлесімді соңғы нүктелерге арналған Ultimate AI Connector {#ultimate-ai-connector-for-compatible-endpoints}

Үйлесімді соңғы нүктелерге арналған Ultimate AI Connector OpenAI-мен үйлесімді AI жеткізуші API-ларын WordPress мультисайт желіңізбен біріктіреді. Бірнеше AI жеткізушіні конфигурациялаңыз — әрқайсысының өз соңғы нүкте URL-ы, API кілті және әдепкі моделі болады — және жеткізуші қолжетімсіз болғанда автоматты резервтік ауысумен сұрауларды бағыттаңыз.

## Негізгі мүмкіндіктер {#key-features}

- **Бірнеше жеткізушіні қолдау**: Бірнеше AI жеткізуші картасын қосыңыз, әрқайсысының өз соңғы нүкте URL-ы, API кілті және әдепкі моделі болады
- **Сүйреп қайта реттеу**: Әр картадағы ұстау тұтқасын пайдаланып жеткізушілерді басымдық бойынша орналастырыңыз; сұраулар жеткізушілерді ретімен пайдаланады
- **Автоматты резервтік бағыттау**: Егер негізгі жеткізуші қолжетімсіз болса, сұраулар автоматты түрде келесі конфигурацияланған жеткізушіге ауысады
- **Әр жеткізушіге арналған конфигурация**: Әр жеткізуші өз тіркелгі деректерімен және модель таңдауымен тәуелсіз конфигурацияланады
- **Картаға негізделген баптаулар UI-ы**: Тұрақты WordPress 6.9+ Card компоненттеріне (CardBody, CardHeader, CardDivider) негізделген
- **Қосу кезінде автоматты кеңейту**: Жаңа жеткізуші карталары автоматты түрде кеңейеді, сондықтан мәліметтерді бірден толтыра аласыз
- **Скрипт кэшін жаңарту**: Соңғы нұсқаның жүктелуін қамтамасыз ету үшін plugin жаңартылғанда жеткізуші скрипттерінің кэші жаңартылады

## Талаптар {#requirements}

- WordPress 5.3 немесе жоғары
- PHP 7.4 немесе жоғары
- Ultimate Multisite plugin (белсенді)
- OpenAI-мен үйлесімді API соңғы нүктесі (өз хостингіңізде немесе үшінші тарап)

## Орнату {#installation}

1. Addon файлдарын `/wp-content/plugins/` каталогыңызға жүктеңіз
2. Plugin-ді WordPress ішіндегі 'Plugins' мәзірі арқылы белсендіріңіз
3. Addon-ды конфигурациялау үшін **Ultimate Multisite → AI Connector** бөліміне өтіңіз

## Конфигурация {#configuration}

### Бірнеше жеткізушіні баптау {#multi-provider-setup}

Конфигурация экраны картаға негізделген орналасуды пайдаланады, мұнда әр карта бір AI жеткізушіні білдіреді. Қажет болғанша көп жеткізуші қоса аласыз.

#### Жеткізуші қосу {#adding-a-provider}

1. Желі әкімшісінде **Ultimate Multisite → AI Connector** бөліміне өтіңіз
2. **Жеткізуші қосу** түймесін басыңыз
3. Жаңа жеткізуші картасы пайда болып, автоматты түрде кеңейеді
4. Жеткізуші мәліметтерін толтырыңыз:
   - **Жеткізуші атауы** — осы жеткізушіні анықтауға арналған белгі (мысалы, "OpenAI", "Mistral", "Local Ollama")
   - **Соңғы нүкте URL-ы** — OpenAI-мен үйлесімді API-дың негізгі URL-ы (мысалы, `https://api.openai.com/v1`)
   - **API кілті** — осы жеткізушіге арналған API кілтіңіз
   - **Әдепкі модель** — осы жеткізуші үшін әдепкі бойынша қолданылатын модель идентификаторы (мысалы, `gpt-4o`, `mistral-small`)
5. **Баптауларды сақтау** түймесін басыңыз

#### Жеткізушіні жою {#removing-a-provider}

Оны жою үшін кез келген жеткізуші картасындағы **Жою** түймесін басыңыз. Өзгерісті қолдану үшін баптауларды сақтаңыз.

#### Жеткізушілерді қайта реттеу {#reordering-providers}

Қайта реттеу үшін кез келген жеткізуші картасының сол жағындағы ұстау белгішесін (⣿) сүйреңіз. Рет резервтік ауысу басымдығын анықтайды: сұраулар бірінші жеткізушіге жіберіледі және сұрау сәтсіз болса, кейінгі жеткізушілерге ауысады.

### Резервтік бағыттау {#fallback-routing}

Негізгі жеткізушіге жіберілген сұрау тайм-аут, жылдамдық шектеуі немесе қызмет үзілісі салдарынан сәтсіз болса, connector тізімдегі келесі жеткізушімен автоматты түрде қайта әрекет жасайды. Бұл жеткізуші сәтті жауап бергенше немесе барлық жеткізушілер тексерілгенше тізім бойынша жалғасады.

| Сценарий | Мінез-құлық |
|----------|-----------|
| Негізгі жеткізуші жауап береді | Сұрау қалыпты түрде өңделеді |
| Негізгі жеткізуші сәтсіз болады | Сұрау келесі жеткізушімен қайталанады |
| Барлық жеткізушілер сәтсіз болады | Қате шақырушыға қайтарылады |

Резервтік ауысу шақырушы plugin немесе theme үшін мөлдір — қанша жеткізуші тексерілгеніне қарамастан, олар не сәтті жауап, не соңғы қате алады.

### Жеткізуші конфигурациясының анықтамалығы {#provider-configuration-reference}

| Өріс | Міндетті | Сипаттама |
|-------|----------|-------------|
| Жеткізуші атауы | Иә | Осы жеткізушіге арналған көрсетілетін белгі |
| Соңғы нүкте URL-ы | Иә | OpenAI-мен үйлесімді API-дың негізгі URL-ы |
| API кілті | Иә | Осы жеткізушіге арналған аутентификация кілті |
| Әдепкі модель | Иә | Сұрауда модель көрсетілмегенде қолданылатын модель идентификаторы |

## Ақауларды жою {#troubleshooting}

### Жеткізушілер тізімде жоқ {#no-providers-are-listed}

Егер сақтағаннан кейін жеткізуші тізімі бос болса, мынаны растаңыз:

- Жеткізушілерді қосқаннан кейін **Баптауларды сақтау** түймесін басқаныңыз
- Браузеріңіз баптаулар бетінің сақталуын бұғаттамағаны (JavaScript қателерін браузер консолінен тексеріңіз)
- Plugin белсенді және сіз желі әкімшісі экранындасыз

### Барлық жеткізушілер үшін сұраулар сәтсіз болып жатыр {#requests-are-failing-for-all-providers}

- Әр жеткізушінің соңғы нүкте URL-ына серверіңізден қол жеткізуге болатынын тексеріңіз (тек браузеріңізден емес)
- Әр API кілтінің жарамды екенін және мерзімі өтпегенін тексеріңіз
- Жеткізушінің жылдамдық шектеуі мен квота күйін олардың dashboard-ында қарап шығыңыз
- Connector-дан толық қате хабарларын алу үшін WordPress debug журналын (`WP_DEBUG_LOG`) қосыңыз

### Жеткізуші резервтік ауысу үшін қолданылмай жатыр {#a-provider-is-not-being-used-for-fallback}

Сүйреп қайта реттеу резервтік ауысу ретін өзгертеді. Резервтік сұрауларды алуы тиіс жеткізуші тізімде дұрыс орналасқанын және оның соңғы нүкте URL-ы мен API кілті дұрыс екенін растаңыз.

### Жаңа жеткізуші картасы автоматты түрде кеңеймейді {#new-provider-card-does-not-expand-automatically}

Автоматты кеңейту үшін браузеріңізде JavaScript қосулы болуы керек. Егер карта басқанда кеңеймесе, қателерді браузер консолінен тексеріңіз. Автоматты кеңейту іске қосылмаса да, карта мазмұнына әрқашан карта тақырыбын басу арқылы қол жеткізуге болады.
