# 7-сабақ: Оны өзіңізге бейімдеу

Клиенттеріңіз ешқашан "қандай да бір WordPress plugin-ын" пайдаланып жатқандай сезінбеуі керек. Олар FitSite-ты -- өз саласына арнап жасалған платформаны пайдаланып жатқандай сезінуі керек. Бұл сабақ брендингті, white-label жасауды және платформаны нағыз өнім сияқты сезіндіруді қамтиды.

## Қай жерден тоқтадық {#where-we-left-off}

FitSite-та фитнес-студия иелерін план таңдаудан тірі сайтқа дейін апаратын жұмыс істейтін checkout ағыны бар. Енді бүкіл тәжірибені біртұтас, брендтелген өнім сияқты етеміз.

## Платформа доменіңіз {#your-platform-domain}

Брендіңіздің негізі — доменіңіз. FitSite үшін:

- **Негізгі домен**: `fitsite.com` (маркетинг сайтыңыз және желі түбірі)
- **Клиент сайттары**: `studioname.fitsite.com` (субдомендер)
- **Жеке домендер**: Growth және Pro пландарындағы клиенттер өз доменін байланыстыра алады

### Доменіңізді баптау {#setting-up-your-domain}

1. Платформа доменіңізді тіркеңіз
2. Оны hosting провайдеріңізге бағыттаңыз
3. Клиент субдомендері үшін wildcard DNS (`*.fitsite.com`) баптаңыз
4. Wildcard SSL белсенді екеніне көз жеткізіңіз

Толық нұсқаулық үшін [Доменді байланыстыруды қалай баптау керек](/user-guide/domain-mapping/how-to-configure-domain-mapping) бөлімін қараңыз.

## Admin тәжірибесін white-label ету {#white-labeling-the-admin-experience}

Фитнес-студия иесі өз сайтының Dashboard-ына кіргенде, WordPress немесе Ultimate Multisite брендингін емес, сіздің брендіңізді көруі керек.

### Жеке Login беті {#custom-login-page}

WordPress login бетін мыналарды көрсету үшін бейімдеңіз:

- FitSite логотипіңіз
- Фитнеске лайық фон суреттері
- Бренд түстеріңіз

### Dashboard брендингі {#dashboard-branding}

[Admin Page Creator](/addons/admin-page-creator) addon-ын немесе custom CSS қолданып:

- WordPress логотипін FitSite логотипіңізбен ауыстырыңыз
- Admin түс схемасын брендіңізге сай бейімдеңіз
- Фитнеске арналған жылдам сілтемелер мен көмек ресурстары бар жеке Dashboard виджетін қосыңыз

### Жеке Admin беттері {#custom-admin-pages}

Фитнес-студия иелері үшін ең маңызды әрекеттерді көрсететін жеке admin беттерін жасауды қарастырыңыз:

- "Сабақ кестеңізді өңдеу"
- "Жаттықтырушы профильдерін жаңарту"
- "Сайтыңызды көру"

Бұл нишаға қатысты әрекеттерді стандартты WordPress мәзірінің ішінде жасырып қоймай, алдыңғы қатарға шығару арқылы үйрену қисығын азайтады.

## Коммуникацияларыңызды брендтеу {#branding-your-communications}

Әр email, шот-фактура және хабарландыру брендіңізді күшейтуі керек.

### Жүйелік email-дер {#system-emails}

**Ultimate Multisite > Settings > Emails** бөліміне өтіп, барлық жүйелік email-дерді бейімдеңіз:

- **Жіберуші аты**: FitSite
- **Жіберуші email-і**: hello@fitsite.com
- **Email үлгілері**: Бренд түстеріңіз бен логотипіңізді қолданыңыз
- **Тіл**: Барлық жерде фитнеске бейімделген

Бейімделетін негізгі email-дер:

| Email | Жалпы нұсқа | FitSite нұсқасы |
|-------|----------------|-----------------|
| Қош келдіңіз | "Жаңа сайтыңыз дайын" | "Фитнес-студия веб-сайтыңыз іске қосылды" |
| Төлем түбіртегі | "Төлем қабылданды" | "FitSite жазылым төлемі расталды" |
| Сынақ мерзімі аяқталуы | "Сынақ мерзіміңіз жақында аяқталады" | "FitSite сынақ мерзіміңіз 3 күннен кейін аяқталады -- студия веб-сайтыңызды белсенді ұстаңыз" |

### Шот-фактуралар {#invoices}

Шот-фактура үлгілерін мыналармен бейімдеңіз:

- FitSite логотипіңіз және бренд түстеріңіз
- Бизнес мәліметтеріңіз
- Фитнеске арналған өнім атаулары (жалпы plan ID-лері емес)

## Клиентке көрінетін сайт {#the-customer-facing-site}

Негізгі доменіңізге (`fitsite.com`) платформаны сататын маркетинг сайты қажет. Бұл Ultimate Multisite желі admin-інен бөлек -- ол бизнесіңіздің көпшілікке көрінетін бет-бейнесі.

Негізгі беттер:

- **Басты бет**: Фитнес бизнесіне арналған нақты құндылық ұсынысы
- **Мүмкіндіктер**: FitSite фитнес терминдерімен не істейді
- **Бағалар**: Нишаға тән мүмкіндік салыстырулары бар үш планыңыз
- **Мысалдар**: Платформада жасалған сайттарды көрсету
- **Тіркелу**: Checkout формаңызға сілтемелер

:::tip Маркетинг сайтыңыз желі сайты бола алады
Маркетинг сайтыңызды өз желіңіздің ішіндегі сайт ретінде жасаңыз. Бұл оны сол Dashboard-тан басқаруға мүмкіндік береді және өз платформаңыздың мүмкіндіктерін көрсетеді.
:::

## Клиенттерге арналған жеке домен {#custom-domain-for-customers}

Жеке домендер қамтылған пландардағы клиенттер үшін процесті анық құжаттаңыз:

1. Клиент өз доменін registrar-да тіркейді немесе көшіреді
2. Клиент DNS-ті платформаңызға бағыттау үшін жаңартады (нақты жазбаларды беріңіз)
3. Ultimate Multisite доменді байланыстыруды және SSL-ді басқарады

Бұл процесс үшін техникалық емес фитнес-студия иелеріне арнап арнайы көмек мақаласын немесе білім базасы жазбасын жасаңыз.

## FitSite желісі қазіргі күйінде {#the-fitsite-network-so-far}

```
FitSite желісі
├── WordPress Multisite (субдомен режимі)
├── Ultimate Multisite (бапталған + брендтелген)
├── Платформа домені (fitsite.com + wildcard SSL)
├── Сайт үлгілері (Studio Essential, Gym Pro, Fitness Chain)
├── Өнімдер (Starter, Growth, Pro + Order Bumps)
├── Checkout ағыны (нишаға арналған, тексерілген)
├── Брендинг
│   ├── Жеке login беті
│   ├── Брендтелген admin Dashboard
│   ├── Нишаға арналған жүйелік email-дер
│   ├── Брендтелген шот-фактуралар
│   └── fitsite.com доменіндегі маркетинг сайты
└── Onboarding ағынына дайын (келесі сабақ)
```

## Бұл сабақта не жасадық {#what-we-built-this-lesson}

- **Платформа домені және DNS** брендтелген тәжірибе үшін бапталды
- **White-label admin** барлық жерде FitSite брендингімен жасалды
- **Бейімделген коммуникациялар** -- email-дер, шот-фактуралар және хабарландырулардың бәрі брендке сай
- **Маркетинг сайты** FitSite-ты фитнес-студия иелеріне сатады
- **Жеке домен құжаттамасы** өз доменін қалайтын клиенттер үшін жасалды
