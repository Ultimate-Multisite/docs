# Третья Сторона Режим Миграция

Superdav AI Agent v1.12.0 өшөөлүүдийн (third-party) чадвар ашиглах аргыг өөрчилсөн. **Третья сторона режим одоо автот** гэсэн утгаар анхны тохиргоо хийгдэхгүйгээр WordPress 7.0+ дээр WordPress Abilities API-тэй шууд холбогдох боломжийг олгож байна.

## Юу өөрчлөгдсөн бэ?

### v1.12.0-ээс өмнө

Третья стооны чадварыг гараар тохируулах шаардлага байсан:

- "третья сторона режим"-ыг тодорхой идэвхжүүлэх хэрэгтэй байсан
- Чадварууд тусгай бүртгэлээс (custom registry) ачаалдаг байсан
- WordPress Abilities API-тай холбогдох нь сонголттой байсан
- Хуучин режим (legacy mode) анхны тохиргоо байсан

### v1.12.0-өөс хойш

Третья стооны чадвар автоматаар ажилладаг:

- Третья сторона режим "auto" гэсэн утгаар анхны тохиргоо байна
- Чадварууд WordPress Abilities API-тэй шууд нийцдэг
- WordPress 7.0+ дээр гараар тохируулах шаардлагагүй
- Хуучин хувилбарын WordPress-ийн хувьд хуучин режим (legacy mode) ашиглах боломжтой

## Хэн нөлөөлөгдөж байна вэ?

### Шинэ Угсралт (WordPress 7.0+)

**Ямар ч үйлдэл шаардлагагүй.** Третья сторона режим автоматаар "auto" гэсэн утгаар тохируулагдана, чадварууд шууд ажиллана.

### Одоо байгаа Угсралт

**Таны тохиргоо хадгалагдана.** Хэрэв та дараах аль нэгийг ашиглаж байсан бол:

- **Legacy mode**: Та хуучин режимын үлдэнэ (өөрчлөлтгүй)
- **Manual third-party mode**: Та гараар тохируулсан режимд үлдэнэ (өөрчлөлтгүй)
- **Auto mode**: Та авто режимтай үргэлжлүүлнэ (өөрчлөлтгүй)

### 7.0-ээс өмнөх WordPress хувилбарууд

**Legacy mode ашиглах боломжтой.** Хэрэв та WordPress 6.x эсвэл түүнээс өмнөх хувилбарыг ашиглаж байгаа бол:

- Третья сторона режим "legacy" гэсэн утгаар анхны тохиргоо байна
- Хэрэв хүсвэл третья стооны режимыг гараар идэвхжүүлэх боломжтой
- Шууд Abilities API ашиглахын тулд WordPress 7.0+ руу шилжих шаардлагатай

## Режимуудыг ойлгох

### Auto Mode (Шинэ Анхны Тохиргоо)

**Auto mode** нь WordPress Abilities API-тэй шууд холбогдохыг ашигладаг:

- Чадварууд WordPress hooks-аар бүртгэгддэг
- WordPress 7.0+ Abilities API-тай бүрэн нийцэл
- Третья стооны чадваруудыг автоматаар олж авна
- Гараар тохируулах шаардлагагүй

**Қашан қолдану керек**: WordPress 7.0+ және үшінші тараптың мүмкіндіктері (third-party abilities) бар

### Қолдану режимі (Manual Mode)

**Қолдану режимі** нақты конфигурацияны талап етеді:

- Қай үшінші тарапты мүмкіндіктерді жүктеу керектігін көрсету
- Тестлау немесе белгілі бір мүмкіндіктерді таңдап жүктеу үшін пайдалы
- Конфигурация файлда өзгерту қажет
- Көбірек бақылау, бірақ көбірек орнату қажет

**Қашан қолдану керек**: Тестлау, белгілі бір мүмкіндіктерді таңдап жүктеу немесе арнайы конфигурациялар үшін

### Ескі режим (Legacy Mode)

**Ескі режим** ескі үшінші тарапты мүмкіндіктер жүйесін қолданады:

- Арнайы мүмкіндіктер тізілімі (WordPress Abilities API емес)
- Ескі WordPress нұсқаларымен кебіре бейімделген
- Негізгі WordPress интеграциясы жоқ
- Қолданылмай жатқан, бірақ қолдау көрсетілет

**Қашан қолдану керек**: WordPress 6.x немесе одан ескі нұсқаларында, немесе ескі үйлесімді қажет болса

## Ағымдағы режимін тексеру

### Әкімшілік панелі арқылы (Via Admin Panel)

1. **WordPress Admin** → **Superdav AI Agent** → **Settings**-ке өтіңіз
2. **Third-Party Mode** параметрді іздеңіз
3. Ағымдағы режиміңізді және оны өзгерту мүмкіндіктерін көресіз

### Код арқылы (Via Code)

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual' немесе 'legacy'
```

## Режимін өзгерту

### Авто режимге ауысу (Switch to Auto Mode)

Егер сіз WordPress 7.0+ нұсқасында болса және авто режимді қолданғыңыз келсе:

1. **Superdav AI Agent** → **Settings**-ке өтіңіз
2. **Third-Party Mode**-ды табыңыз
3. **Auto (WordPress Abilities API)**-ны таңдаңыз
4. **Save**-қа басыңыз

Superdav AI Agent үшін үшінші тарапты мүмкіндіктерді автоматты түрде анықтап, тіркеп алады.

### Қолдану режиміне ауысу (Switch to Manual Mode)

Егер сіз қай мүмкіндіктерді жүктеу керектігін қолмен бақылағыңыз келсе:

1. **Superdav AI Agent** → **Settings**-ке өтіңіз
2. **Third-Party Mode**-ды табыңыз
3. **Manual**-ды таңдаңыз
4. **Save**-қа басыңыз
5. Қай мүмкіндіктерді жүктеу керектігін көрсету үшін конфигурация файлын өзгертіңіз

### Ескі режимге ауысу (Switch to Legacy Mode)

Егер сізге ескі үйлесім қажет болса:

1. **Superdav AI Agent**-қа өтіңіз → **Settings** (Жоспарлау)
2. **Third-Party Mode**-ды табыңыз (Үшінші тарап режим)
3. **Legacy**-ны таңдаңыз (Ескі режим)
4. **Save**-қа басыңыз (Сақтау)

## Auto Mode Артықшылықтары

### Автоматты Араптау (Automatic Discovery)

Қажетті мүмкіндіктер келесілерден автоматты түрде анықталады:

- Орнатылған плагиндер
- Белгіленген тема (Active theme)
- Міндетті плагиндер (Must-use plugins)
- Drop-in плагиндер

Қолымен тіркеу қажет емес.

### Табиғи Интеграция (Native Integration)

Мүмкіндіктер WordPress Abilities API-мен интеграцияланады:

- WordPress негізгі функцияларымен үйлесімді
- WordPress администраторымен жұмыс істейді
- Abilities API-ді қолданатын басқа плагиндермен қоса болады
- WordPress дамыған сайын болашаққа дайын (Future-proof)

### Қарапайым Басқару (Simplified Management)

- Өңдеу қажет ететін конфигурация файлдары жоқ
- Қолымен мүмкіндіктерді тіркеу қажет емес
- Ability Visibility бақылаулары автоматты жұмыс істейді
- Администраторлық хабарламалар сипаттамасы жоқ мүмкіндіктер туралы ескертеді

### Жақсы Өнімділік (Better Performance)

- Мүмкіндіктер кешті сақталады (cached)
- Қажет болған кезде баяу жүктеледі (Lazy-loaded on demand)
- WordPress 7.0+ үшін оңтайландырылған

## Миграция Жолы (Migration Path)

### Егер сіз WordPress 6.x-те болсаңыз

1. **WordPress 7.0+ нұсқасына көтеріліңіз** (даяр болған кезде)
2. **Superdav AI Agent-ті v1.12.0+ нұсқасына жаңартыңыз**
3. **third-party mode-ды Auto-ға өзгертіңіз** (опционалды; legacy режим әлі жұмыс істейді)
4. Дұрыс қол жеткізу бақылауларын қамтамасыз ету үшін мүмкіндіктер көрінісін қараңыз

### Егер сіз WordPress 7.0+-те болсаңыз

1. **Superdav AI Agent-ті v1.12.0+ нұсқасына жаңартыңыз**
2. **third-party mode-тың Auto-ға орнатылғанын тексеріңіз** (қалдық режим реттеу бойынша орынды болуы керек)
3. Дұрыс қол жеткізу бақылауларын қамтамасыз ету үшін мүмкіндіктер көрінісін қараңыз
4. Олар жұмыс істейтінін растай отырып, third-party мүмкіндіктерін тестілеңіз

## Түзету (Troubleshooting)

### Auto mode-да мүмкіндіктер жүктелмейді

- WordPress 7.0+ нұсқасында екеніне көріңіз
- third-party mode-тың "Auto" деп орнатылғанын тексеріңіз
- Мүмкіндікті ұсынған плагиндердің белсенді екенін тексеріңіз
- Тіркелу қателері үшін WordPress қате журналдарын қараңыз

### Мен legacy mode-ды сақтап қалуым керек

- **Settings** → **Third-Party Mode**-қа өтіңіз
- **Legacy**-ны таңдаңыз
- **Save**-қа басыңыз
- Legacy режим әлі жұмыс істеуде болады

### Маған арнайы мүмкіндіктерім көрсетілмейді

- Олар WordPress хуктары арқылы тіркелгенін тексеріңіз
- Олар Abilities API-ды дұрыс іске асыратынын тексеретін
- WordPress қате журналдарын қараңыз
- Барлық тіркелген мүмкіндіктеріне көру үшін **Ability Visibility** администратор парағын пайдаланыңыз

### "Классификацияланбаған мүмкіндік" ескертулерін аламын

- Бұл жаңа үшінші тараптың мүмкіндіктері үшін қалыпты жағдай
- Оларды администратордағы ескертуде қарап, классификациялаңыз
- Классификация туралы мәліметтер үшін **Ability Visibility**-ді көріңіз

## Кезең-кезеңмен үйлесімділік (Backward Compatibility)

### Қазіргі конфигурациялар

Егер сізде бар үшінші тараптың мүмкіндіктерінің конфигурациялары болса:

- **Legacy mode (Ескі режим)**: Сіздің конфигурацияңыз жұмыс істеуі жалғасады
- **Manual mode (Қолмен басқару режимі)**: Сіздің конфигурацияңыз жұмыс істеуі жалғасады
- **Auto mode (Авто режим)**: Сіздің конфигурацияңыз елеулі болады (авто режим оның орнына алады)

Арнайы конфигурацияңызды сақтау үшін Қолмен басқару немесе Ескі режимде қалуды ұсынамыз.

### Ескіру кестесі (Deprecation Timeline)

- **v1.12.0**: Legacy және Manual режимдер толық қолдау көрсетіледі
- **v1.13.0+**: Legacy режим ескіру туралы ескертулерді көрсетуі мүмкін
- **v2.0.0**: Legacy режим жойылуы мүмкін (Түсіндірілген)

## Ең жақсы тәжірибелер (Best Practices)

### Жаңа орнатылымдар үшін

- Auto mode-ды қолданыңыз (бұл стандартты режим)
- Superdav AI Agent-ті автоматты түрде мүмкіндіктерін тауып алуға мүмкіндік беріңіз
- Қол жеткізуді басқару үшін Ability Visibility-ді пайдаланыңыз

### Барлық орнатылымдар үшін

- Мүмкін болса WordPress 7.0+ нұсқасына көтеріліңіз
- Жоспарлауды жеңілдету үшін Auto mode-ға ауысыңыз
- Ability Visibility арқылы мүмкіндіктеріне қарап, классификациялаңыз

### Арнайы мүмкіндіктер үшін

- WordPress хуктары арқылы мүмкіндіктері тіркеңіз (Abilities API)
- Қолмен жасалған мүмкіндік тізілімдерін қолданбаңыз
- Auto mode-да WordPress 7.0+ нұсқасында тексеріңіз

## Келесі қадамдар

1. **WordPress нұсқаңызды тексеріңіз**: Авто режим үшін 7.0+ версияда екеніне көріңіз.
2. **Үшінші тарапты режимін қарастырыңыз**: Settings (Жоспарлау) бөліміне өтіп, қазіргі режимін тексеріңіз.
3. **Қажет болса жаңартыңыз**: Егер сіз WordPress 7.0+ нұсқасында болса, Auto режимге ауысыңыз.
4. **Қабілеттерді жіктеңіз**: Жіктелуі жоқ кез келген қабілеттерді қарап, оларды жіктеп алыңыз.
5. **Тест жасаңыз**: Үшінші тарапты қабілеттеріңіздің дұрыс жұмыс істейтінін тексеріңіз.

## Қосымша тақырыптар

- **Қабілеттер көрінуі (Ability Visibility)**: Қай жерде қандай қабілеттер көрсетілетін бақылау
- **WordPress Abilities API**: WordPress-тің өзіндегі қабілеттерді тіркеу туралы біліңіз
- **Үшінші тарапты қабілеттерді дамыту (Third-Party Ability Development)**: Auto режиммен жұмыс істейтін қабілеттерді жасау
