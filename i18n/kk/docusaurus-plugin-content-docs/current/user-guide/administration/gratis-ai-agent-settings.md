# Gratis AI Agent параметрлері {#gratis-ai-agent-settings}

Gratis AI Agent ішіндегі **Settings → Advanced** экраны backend интеграцияларына арналған әкімші деңгейіндегі конфигурацияны ұсынады. Бұл бет feedback қайта жіберуді, іздеу провайдері кілттерін, басқарылатын Superdav қызметін баптауды, Google Calendar басқару элементтерін, TextBee SMS параметрлерін және бүкіл желі бойынша мүмкіндік жалаушаларын құжаттайды.

## Параметрлерге кіру {#accessing-settings}

1. WordPress әкімші бөлімінде **Gratis AI Agent → Settings** бөліміне өтіңіз.
2. **Advanced** қойындысын басыңыз.

## Feedback endpoint конфигурациясы {#feedback-endpoint-configuration}

Feedback endpoint пайдаланушы бас бармақты төмен қарату түймесі, авто-сұрау баннері немесе `/report-issue` командасы арқылы feedback жіберген сайын AI агентінен POST сұрауларын қабылдайды.

| Өріс | Сипаттама |
|---|---|
| **Feedback Endpoint URL** | JSON денесі бар HTTP POST сұраулары ретінде feedback жіберілімдерін қабылдайтын URL. |
| **Feedback API Key** | Әр feedback сұрауының `Authorization` header ішінде жіберілетін bearer token. Егер endpoint аутентификацияны талап етпесе, бос қалдырыңыз. |

### Күтілетін JSON payload {#expected-json-payload}

Сіздің feedback endpoint кемінде келесі өрістері бар JSON денесін қабылдауы керек:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Payload ішінде әңгіме контекстіне байланысты қосымша өрістер болуы мүмкін.

### `triage_category` мәндері {#triagecategory-values}

AI triage қабаты payload қайта жіберілмес бұрын `triage_category` үшін келесі мәндердің бірін тағайындайды:

| Мән | Мағынасы |
|---|---|
| `factual_error` | Көмекші қате нақты ақпарат берді. |
| `unhelpful_answer` | Жауап техникалық тұрғыдан дұрыс болды, бірақ пайдалы емес. |
| `inappropriate_content` | Жауапта пайдаланушыларға көрсетілмеуі тиіс мазмұн болды. |
| `other` | Feedback белгілі санатқа сәйкес келмеді. |

### Аутентификация {#authentication}

Егер endpoint аутентификацияны талап етсе, **Feedback API Key** өрісін bearer token мәніне орнатыңыз. Агент мынаны жібереді:

```
Authorization: Bearer <your-api-key>
```

Егер **Feedback API Key** өрісі бос болса, `Authorization` header жіберілмейді.

### Feedback жинауды өшіру {#disabling-feedback-collection}

**Feedback Endpoint URL** және **Feedback API Key** өрістерінің екеуін де бос қалдырыңыз. Бас бармақты төмен қарату түймесі мен feedback UI пайдаланушыларға көрініп тұрады, бірақ жіберілімдер ешқандай сыртқы қызметке қайта жіберілмейді.

## Brave Search API Key {#brave-search-api-key}

Сондай-ақ **Advanced** қойындысында **Brave Search API Key** өрісі [Интернеттен іздеу](../configuration/internet-search) мүмкіндігін қосады.

| Өріс | Сипаттама |
|---|---|
| **Brave Search API Key** | Brave Search әзірлеуші Dashboard-ынан алынған API кілтіңіз. AI көмекшісінде интернеттен іздеуді қосу үшін қажет. |

Өріс белгісінде Brave Search API тіркелу бетіне апаратын басылатын сілтеме бар. Интернеттен іздеуді өшіру үшін бос қалдырыңыз.

Бұл мүмкіндік бойынша соңғы пайдаланушы құжаттамасын [Интернеттен іздеу](../configuration/internet-search) бөлімінен қараңыз.

## Басқарылатын Superdav қызметі {#managed-superdav-service}

Superdav AI Agent v1.18.0 қолдау көрсетілетін сайттар үшін басқарылатын Superdav қызмет endpoint-терін және автоматты қосылым қамтамасыз етуді қосады. Сайтыңыз қолмен конфигурацияланған қызмет endpoint орнына hosted провайдерге қосылуы керек болғанда осы басқару элементтерін пайдаланыңыз.

| Өріс | Сипаттама |
|---|---|
| **Managed Superdav Service** | Қолдау көрсетілетін сайттар үшін hosted Superdav қызмет қосылымын қосады. |
| **Provision Connection** | Endpoint пен тіркелгі деректерін автоматты қамтамасыз етуді бастайды. Мұны сайт басқарылатын провайдерді пайдалануы керек екенін растағаннан кейін қолданыңыз. |
| **Service Endpoint / Connection Status** | Қамтамасыз етуден кейін ағымдағы endpoint немесе қосылым күйін көрсетеді. |

Қамтамасыз етуден кейін, басқарылатын-қызмет жұмыс процестеріне сүйенбес бұрын параметрлерді сақтап, қосылым күйін тексеріңіз. Егер қамтамасыз ету сәтсіз болса, көрсетілген қайталау нұсқаулығын қарап шығып, сайттың hosted провайдерді пайдалануға рұқсаты бар екенін растаңыз.

## Google Calendar конфигурациясы {#google-calendar-configuration}

Superdav AI Agent v1.18.0 күнтізбе мүмкіндіктері қосылғанда, агент конфигурацияланған күнтізбелерді және оқиға мәліметтерін оқи алады. Күнтізбе құралдары оқуға бағытталған және кестені ескеретін еске салғыштар, қатысушылармен кейінгі байланыс және контакт сәйкестендіру үшін пайдалы.

| Өріс | Сипаттама |
|---|---|
| **Google Calendar Credentials** | Күнтізбе деректерін оқу үшін қажетті тіркелгі деректерін немесе token қосылымын сақтайды. |
| **Calendar Selection** | Агент тексере алатын конфигурацияланған күнтізбелерді шектейді. |
| **Calendar Connection Status** | Ағымдағы тіркелгі деректері күнтізбелер мен оқиғаларды оқи алатынын растайды. |

Күнтізбе тіркелгі деректерін агентке қажет күнтізбелермен ғана шектеңіз. Егер күй мерзімі өткен token бар екенін көрсетсе, тіркелгі деректерін қайта қосыңыз немесе ауыстырыңыз.

## TextBee SMS хабарландырулары {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 конфигурацияланған хабарландыру жұмыс процестері үшін SMS провайдері ретінде TextBee қосады. SMS хабарландырулары сезімтал немесе пайдаланушыға бағытталған хабарлар үшін адам мақұлдауы қақпаларымен бірге қолданылуы керек.

| Өріс | Сипаттама |
|---|---|
| **TextBee API Key** | TextBee SMS провайдеріне жіберілетін сұрауларды аутентификациялайды. |
| **TextBee Device / Sender** | Провайдер талап еткенде, шығыс хабарлар үшін пайдаланылатын TextBee жіберушісін немесе құрылғысын таңдайды. |
| **SMS Notifications Enabled** | Мақұлданған жұмыс процестеріне мәтіндік хабарлама хабарландыруларын жіберуге мүмкіндік береді. SMS жіберуді болдырмау үшін өшірулі қалдырыңыз. |

Сынақ хабарын тек әкімшіге тиесілі нөмірге жіберіңіз, содан кейін жоспарланған немесе қатысушыларға бағытталған еске салғыштарды қоспас бұрын мақұлдау қақпасының әрекетін растаңыз.

## Мүмкіндік жалаушалары {#feature-flags}

Сондай-ақ v1.9.0 нұсқасында енгізілген **Settings → Feature Flags** қойындысы қосымша функционалға арналған ауыстырғыштарды ұсынады. Әр жалауша бүкіл желі бойынша не қосулы, не өшірулі болады; қазіргі уақытта әр сайт бойынша бөлек қайта анықтау жоқ.

### Мүмкіндік жалаушаларына кіру {#accessing-feature-flags}

1. WordPress әкімші бөлімінде **Gratis AI Agent → Settings** бөліміне өтіңіз.
2. **Feature Flags** қойындысын басыңыз.

### Қолжетімділікті басқару жалаушалары {#access-control-flags}

| Жалауша | Әдепкі | Сипаттама |
|---|---|---|
| **Әкімшілермен шектеу** | Өшірулі | Қосылған кезде, AI Agent чат панелін тек `administrator` рөлі бар пайдаланушылар аша алады. Барлық басқа рөлдер оның орнына "Әкімшіңізге хабарласыңыз" хабарын көреді. |
| **Желі әкімшілерімен шектеу** | Өшірулі | Multisite желісінде қосылған кезде, агентті тек Super Admins пайдалана алады. Жеке сайт әкімшілері бұғатталады. Екеуі де қосулы болса, "Әкімшілермен шектеу" параметрінен басым болады. |
| **Жазылушыға кіруге рұқсат беру** | Өшірулі | Қосылған кезде, `subscriber` рөлі бар пайдаланушылар чат интерфейсін пайдалана алады, бірақ тек оқуға арналған мүмкіндіктермен шектеледі (жазба жасау немесе баптауларды өзгерту жоқ). |
| **Мүше еместер үшін өшіру** | Өшірулі | Ultimate Multisite мүшелік мәртебесімен бірігеді. Қосылған кезде, белсенді мүшелігі жоқ сайттар үшін чат жасырын болады. |

### Брендинг жалаушалары {#branding-flags}

| Жалауша | Әдепкі | Сипаттама |
|---|---|---|
| **"Powered by Gratis AI Agent" төменгі жазуын жасыру** | Өшірулі | Чат виджетінің төменгі жағында көрсетілетін брендинг атрибуция жолын алып тастайды. White-label орналастырулар үшін ұсынылады. |
| **Арнаулы агент атауы** | *(бос)* | Чат тақырыбындағы және әкімші мәзіріндегі әдепкі "Gratis AI Agent" белгісін өзіңіздің өнім атауыңызбен ауыстырады. Әдепкіні пайдалану үшін бос қалдырыңыз. |
| **Агент таңдағышын жасыру** | Өшірулі | Қосылған кезде, пайдаланушылар бес кірістірілген агент арасында ауыса алмайды. Ағымдағы агент Settings → General бөлімінде әдепкі ретінде бапталған мәнге бекітіледі. |
| **Сайт белгішесін чат аватары ретінде пайдалану** | Өшірулі | Чат виджеті тақырыбындағы әдепкі AI белгішесін WordPress сайт белгішесімен ауыстырады (Appearance → Customize → Site Identity бөлімінде орнатылады). |

### Автоматтандыру қауіпсіздігі жалаушалары {#automation-safety-flags}

Superdav AI Agent v1.18.0 қауіпсіздеу автоматтандыру іске қосуларын қамтамасыз ету үшін адамның мақұлдау қақпаларын және еске салу жазбаларын енгізеді. Бұл басқару элементтері орнатылған пакетке байланысты мүмкіндік жалаушаларында немесе кеңейтілген автоматтандыру баптауларында пайда болуы мүмкін.

| Жалауша | Әдепкі | Сипаттама |
|---|---|---|
| **Адамның мақұлдауын талап ету** | Қосу ұсынылады | Уәкілетті пайдаланушы ұсынылған әрекетті қарап, растағанға дейін сезімтал автоматтандыруларды кідіртеді. |
| **Еске салуларды қайталауды болдырмау** | Қосулы | Қайталау әрекеттері немесе жоспарланған іске қосулар қайталанатын хабарландырулар жібермеуі үшін жіберілген еске салуларды жазып отырады. |
| **Күнтізбе құралдарын қосу** | Бапталғанға дейін өшірулі | Агентке бапталған Google күнтізбелері мен оқиғаларын оқуға мүмкіндік береді. |
| **SMS хабарландыруларын қосу** | Бапталғанға дейін өшірулі | Деректері сақталғаннан кейін мақұлданған жұмыс процестеріне TextBee SMS хабарландыруларын жіберуге мүмкіндік береді. |

### Өзгерістерді қолдану {#applying-changes}

Кез келген жалаушаны ауыстырғаннан кейін **Баптауларды сақтау** түймесін басыңыз. Өзгерістер бірден күшіне енеді — кэшті тазалау немесе pluginді қайта белсендіру қажет емес.
