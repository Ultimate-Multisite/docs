# Gratis AI Agent Параметрлеу 설정

Gratis AI Agent-тың **Settings → Advanced** беті v1.5.0-де енгізілген бэкэнд интеграциялары үшін администратор деңгейін параметрлеуге мүмкіндік береді. Бұл бетте **Feedback Endpoint** өрістері мен олардың күтілетін форматы туралы ақпарат берілген.

## Параметрлерге Қол ету

1. WordPress администраторында **Gratis AI Agent → Settings**-ке өтіңіз.
2. **Advanced** табына басыңыз.

## Feedback Endpoint Конфигурациясы

Feedback endpoint-і пайдаланушы "thumbs-down" (басқау) баты арқылы, автоматты түрде шыққан баннер арқылы немесе `/report-issue` командасы арқылы кері байланыс жіберген кезде AI агентінен POST сұраныстары қабылдайды.

| Өріс | Сипаттамасы |
|---|---|
| **Feedback Endpoint URL** | Бұл URL, JSON денесімен HTTP POST сұраныстары ретінде кері байланысты қабылдайтын URL-dır. |
| **Feedback API Key** | Әрбір кері байланыс сұранысының `Authorization` басылында жіберілетін bearer токені. Егер сіздің endpoint-іңіз аутентификация талап етпесе, оны бос bırakңыз. |

### Күтілетін JSON Payload

Сіздің feedback endpoint-іңіз келесі өрістерді қамтыған JSON денесін қабылдауы керек:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Сөйлесу контекстіне байланысты payload-та қосымша өрістер болуы мүмкін.

### `triage_category` Мәндері

AI сұрақтарды жіберу қабаты (triage layer) payload-ды алдында келесі мәнтердің бірін `triage_category`-ға тағайындайды:

| Мәндер | Мағынасы |
|---|---|
| `factual_error` | Ассистент дұрыс емес факті ақпарат берді. |
| `unhelpful_answer` | Жауап техникалық тұрғыдан дұрыс болды, бірақ пайдалы емес. |
| `inappropriate_content` | Жауапта пайдаланушыларға көрсетілмеуі керек контент болды. |
| `other` | Кері байланыс белгілі бір категорияға сәйкес келді. |

### Аутентификация

Егер сіздің endpoint-і аутентификацияны талап етсе, **Feedback API Key** өрісіне өздерінің bearer token-ын қойыңыз. Агент келесіні жібереді:

```
Authorization: Bearer <your-api-key>
```

Егер **Feedback API Key** өрісі бос болса, `Authorization` таңбасы жіберіледі.

### Feedback жинауды өшіру

**Feedback Endpoint URL** және **Feedback API Key** өрістерін бос bırakңыз. "Thumbs-down" баты мен кері байланыс интерфейсі пайдаланушылар үшін көрініп тұрады, бірақ жіберілген деректер ешкімнің сыртқы қызметіне жол берілмейді.

## Brave Search API Key

Сонымен қатар **Advanced** табасындағы **Brave Search API Key** өрісі интернет іздеу мүмкіндігін [Internet Search](../configuration/internet-search) арқылы іске қосады.

| Field | Description |
|---|---|
| **Brave Search API Key** | Brave Search әзірлеуші панелінен алынған API кілттеріңіз. AI ассистентте интернет іздеуді іске қосу үшін қажет. |

Өріс таңбасы Brave Search API тіркелу парағына баруға сілтемесі бар. Интернет іздеуді өшіру үшін бос bırakңыз.

Бұл мүмкіндік туралы пайдаланушыларға арналған құжатта [Internet Search](../configuration/internet-search) қараңыз.

## Feature Flags (Функция белгілері)

v1.9.0-де енгізілген **Settings → Feature Flags** табасы опционалды функционалдық мүмкіндіктер үшін өшіру/жағу кілттері береді. Әр флаг барлық желі бойынша іске қосылған немесе өшірілген болады; осы уақытта сайтқа арнайы өзгертулер жоқ.

### Feature Flags-ке қол жеткізу

1. WordPress администраторында **Gratis AI Agent → Settings**-ке өтіңіз.
2. **Feature Flags** табасын басыңыз.

### Access Control Flags (Қолжетімділік белгілері)

| Белгіше | Басталдырылған мән | Сипаттамасы |
|---|---|---|
| **Администраторларға шектеу** | Аяқ (Off) | Бұл опция қосылған кезде, тек `administrator` рөлі бар пайдаланушылар Ынтымақтастық агент чат панелін аша алады. Басқа рөлдерге "Администратормен байланысыңыз" деген хабарлама көрінеді. |
| **Желі администраторларына шектеу** | Аяқ (Off) | Мүлкі желіде қосылған кезде бұл опция қосылса, тек Супер Администраторлар агентті қолдана алады. Жеке сайт администраторлары бұғатталады. Егер екі опция да қосылса, "Администраторларға шектеу" мәніне басымдық беріледі. |
| **Жазылушыларға қол жеткізуге рұқсат беру** | Аяқ (Off) | Бұл опция қосылған кезде, `subscriber` рөлі бар пайдаланушылар чат интерфейсін қолдана алады, бірақ олар тек оқуға ғана мүмкіндіктермен шектеледі (қатанар құру немесе параметрлерді өзгертуге рұқсат жоқ). |
| **Мемлекеттік емес пайдаланушылар үшін өшіру** | Аяқ (Off) | Бұл Ultimate Multisite мүшелік мәртебесімен интеграцияланады. Егер бұл опция қосылса, белсенді мүшелігі жоқ сайттар үшін чат жасырылады. |

### Брендті белгілеулер (Branding Flags)

| Белгіше | Басталдырылған мән | Сипаттамасы |
|---|---|---|
| **"Gratis AI Agent" футерді жабу** | Аяқ (Off) | Чат виджетінің төменгі бөлігінде көрсетілген брендті белгілеу жолын алып тастайды. Ақ-ақ лейблдерингке арналған ұсынылады. |
| **Қосымша агент атауы** | *(бос)* | Чат басындағы және администратормен менюдағы стандартты "Gratis AI Agent" белгісін өз өнім атауыңызбен ауыстырады. Стандартты қолданғыңыз келсе бос bırakсаңыз. |
| **Агент таңдағыш панелін жабу** | Аяқ (Off) | Бұл опция қосылған кезде, пайдаланушылар бес ішіндегі енгізілген агенттер арасында ауыса алмайды. Қазіргі агент Settings → General-де белгіленген мәніне тұрақты түрде бекітіледі. |
| **Сайт иконкасын чат аватары ретінде қолдану** | Аяқ (Off) | Чат виджетінің басындағы стандартты AI иконкасын WordPress сайт иконкасымен ауыстырады (Appearance → Customize → Site Identity-де белгіленеді). |

### Өзгерістерді қолдану

Кез келген белгішелерді өзгертіп алғаннан кейін **Settings Save**-ды басыңыз. Өзгерістер дереу күшіне енеді — кэш тазарту немесе плагинді қайта іске қосу қажет емес.
