# Жаңарту Жоспары (Release Notes)

## 2.13.0 нұсқасы — 2026-06-05 күні шығарылды {#version-2130--released-on-2026-06-05}

Жаңа мүмкіндіктер: Клиенттік аккаунт, чекаут (checkout), төлем, сайт, счёт, шаблон ауыстыру және доменді карталау процестері үшін суверейн-тенанты (sovereign-tenant) қолдау енгізілді, бұл тенанты желілерінің клиенттерді басқарылатын әрекеттерге арналған негізгі сайтқа қайта бағыттауына мүмкіндік береді.
Жаңа мүмкіндіктер: Қайтару кезеңдеріне (renewal) қатысты құжаттарды тексеру енгізілді, бұл егер сақталған төлем келісімшарты, жазылым немесе vault токені болмаса, gateway-лер автоматты қайтаруды өшіруге мүмкіндік береді.
Жаңа мүмкіндіктер: Сайтті енгізу кезеңіндегі күтуші (pending site creation) үшін HMAC-верификациясымен циклдік жариялау енгізілді, бұл артық жұмыстардың кешігуінен басқарылатын хосттарда чекауттан сайтқа қызмет көрсетуді одан да сенімді етеді.
Жаңа мүмкіндіктер: SSO URL-деріне, чекаут-формасының негізгі домендеріне және автоматты домен-рекордтарды жасауға арналған әзірлеушімнің (developer) кеңейту нүктелері енгізілді.
Түзетулер: SSO қатысты домендер бойынша, аноним бұралулар (anonymous broker visits), шығу және плагин-аралық тәуелсіздік конфликттері арқылы SSO одан да сенімді болды.
Түзетулер: Сайтті енгізу кезеңіндегі күтуші сайттар енді ескірген жариялау белгілерінен қалпына келтірілуі және клиенттерді сайтты енгізу экранында тұрақсыз қалдырудан аулақ болады.
Түзетулер: Ортақ чекаут-формасы негізгі домендері үшін домен рекорды жасалмады, сондай-ақ интеграция белсенді емес кезде пайдаланылмайтын хост-провайдер артық жұмыстары өткізіледі.
Түзетулер: Чекаут, төлем мекенжайы, парольді қайталау, электронды тастыру, шаблон ауыстыру, турлар және клиенттік дашборды шешімдемесі нормалық клиенттік ағындарды бөгдемейді.
Түзетулер: Бұйрық хабарламалар (broadcast emails) енді алушыларды жеке сақтай отырып, ал қабылдау тізімдері немесе электронды пошта тасымалы сәтсіз болған кезде SMTP/плагин қатанарлық қателерден аулақ болады.
Түзетулер: Жазылымдардың қайтаруы, мерзімі көрсетілуі және төлем жинау кезеңіндегі шешімдер деректердің дереу мерзімін асырудан, қалпына келмейтін жағдайлардан немесе қажетті төлемдерден өткізбейді.
Жақсартылған: WordPress-ке 7.0 дейін үйлесім тексерілді, өндірістік Vue активтері npm көздерінен қайта құрылды және Cypress ең басты (end-to-end) қамту енді чекаут, орнату, SSO және gateway ағындарын одан да көп сынап көрсетеді.

## 2.12.0 нұсқасы — 2026-05-15 күні шығарылды {#version-2120--released-on-2026-05-15}

- Жаңа: Ultimate Multisite үшін Hostinger (hPanel) хост провайдерін домен картасы интеграциясымен қолдау көрсетуге қостық

- Түзету: BCC хабарламаларында қатысушылардың мекенжайларын көрсетпеу үшін ашылмаған алушылар басылы (undisclosed-recipients) таңбасын қолдана бастады

- Түзету: Stripe мүшелік жаңартулары енді ескірген deleteDiscount API-ды шақырмай, жеңілдіктерді дұрыс тазалайды
- Түзету: домен картасы бойынша SSO қайта бағыттары шектеулі етілді, бұл шексіз қайта бағу циклдерін болдырмау үшін жасалды
- Түзету: Орнату wizard сурет таңдаусы енді негізгі деректер моделін дұрыс жаңартады
- Түзету: Site Exporter CLI енді дұрыс стандартты желі сайтының таңдалуын сақтайды
- Жаңартылған: Плагин пакетінен bundled wp-cli алып тасталды, плагин өлшемін азайтуға мүмкіндік берді

## 2.11.0 нұсқасы — 2026-05-11 күні шығарылды {#version-2110--released-on-2026-05-11}

- Жаңа: Network export администраторлар Site Export admin бетінен барлық субсайттарды бір архивте экспорта алады
- Жаңа: Site Templates жоспар таңдауын енді резервтік тізбек арқылы күшін атқарады, бұл жоспар шектеулері үшін шаблон қолжетімділігін дұрыс шектейді
- Жаңа: Checkout form editor қажетті өріс орнатылмаған кезде продукт қосу туралы ескерту береді
- Жаңа: Import/Export settings табы енді оның қамту аймағын анық сипаттайды және Site Export құралына тікелей сілтеме береді

## 2.10.0 нұсқасы — 2026-05-05 күні шығарылды {#version-2100--released-on-2026-05-05}

Түзету: Шаблон ауыстыру рұқсат күйлері немесе рұқсатсыз қол жеткізуге қарсы қорғалады.

Түзету: Мекен-жай бос болған кезде төлем мекен-жайы сұрағы көрсетіледі.
Түзету: PHP 8.1 null-to-string ескіру хабарламалары шешілді.
Түзету: Timing мәселелерін алдын алу үшін lazy-loaded site init hook-тан бұрын жүктеледі.
Түзету: Бөлім беру (SSO) жолы барлық кіру ағымдарында құрғалады.
Түзету: Сақтау кезінде бос сайт идентификациясы опциялары сақталған.

## 2.8.0 нұсқасы — 2026-04-29 күні шығарылды {#version-280--released-on-2026-04-29}

Жаңа мүмкіндіктер: Басқа Параметрлер (Other Options) параметрлері UI-сында Jumper toggle қосылды.
Жаңа мүмкіндіктер: Checkout формалары тізімдегі Status бағаны қосылды.
Жаңа мүмкіндіктер: Custom MU-plugin sunrise кеңейтулері үшін Addon sunrise файл жүктеушісі қосылды.
Жаңартылған: Параметрлер парағынан қате хабарлауды (error-reporting) қосу мүмкіндігі алынып тасталды.
Түзетілді: Рахмет бетінің сайт картасы — сурет шектеулі етілді және сілтемелер дұрыс стиленді.
Түзетілді: Экран түсіруші (Screenshot) провайдегі thum.io-дан WordPress.com mShots-қа ауыстырылды.
Түзетілді: Регистрацияны және Дефолт Рольді қосу енгізу қазір жаңа орнату кезінде дұрыс дефолттарға орнатылады.
Түзетілді: `get_site_url()` доменде порт (port) болған кезде бос мәнді қайтармайды.
Түзетілді: `copy_media` параметрі бос болған кезде медиа файлдарының көшірілуі дұрыс іске асырылады.
Түзетілді: Сеть-активациядан кейін sitemeta жазылғаннан кейін Object cache дұрыс жойылады (invalidate).
Түзетілді: 3 бөліктен тұратын домендер үшін DNS тексеру кезінде Custom domain автоматты түрде негізгіге (primary) көтеріледі.
Түзетілді: Ескірген төлем тазартылған кезде күтілетін мүшелік (membership) бұзылады.
Түзетілді: Inline кіру сұрағын жоюдан кейін пароль күш тексері қайта іске қосылады.
Түзетілді: Сайт сипатталған кезде шектелген кезеңде (infinite page reload) бұзылу тоқтатылды.
Түзетілді: Плагин белсендірілген және параметрлер сақталған кезде WP core тіркеу опциясы синхроланды.
Түзетілді: PHP 8.4-ке қолайлы болу үшін `calculate_expiration` içinde Null expiration guard қосылды.
Түзетілді: Клиентте белсенді мүшелігі бар клиенттерде қайта тіркелулер бұзылады.
Түзетілді: Checkout процесінде `date_expiration`-ға Null тексеру қосылды.
Түзетілді: Сайт құру кезінде шектеулер, мүшелік сипаттамасы және доменді көтеру (domain promotion) қатаңдылығы жаңартылды.
