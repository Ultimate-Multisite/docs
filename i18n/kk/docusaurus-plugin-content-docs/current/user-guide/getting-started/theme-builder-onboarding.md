# Theme Builder Бағдарламаға Кіру Процесі (Onboarding Flow)

Superdav AI Agent v1.12.0 сізге алғашқы орнату кезінде арнайы блок темасын жасауға көмектесетін бағытталған **Theme Builder onboarding flow**-ды енгізеді. Бұл ескі Site Builder режимін одан да икемді, агентті қолдау алатын тәсілмен ауыстырады.

## Theme Builder Onboarding Flow дегеніміз не? {#what-is-the-theme-builder-onboarding-flow}

Theme Builder onboarding flow – бұл интерактивті орнату Wizard-ы (бағдарламасы), ол:

- Дизайн шешімдеріңізді бағыттайды (реңктер, типография, құрылым)
- Сіздің сайттың визуалды тұтастық қалауларыңызды жинақтайды
- Сіздің қажеттіліктеріңізге сай арнайы блок темасын генерациялайды
- Толық аяқталған кезде темуді автоматты түрде іске қосады

Бұл flow **Setup Assistant agent** арқылы жұмыс істейді, ол нақтылау сұрақтарын қояды және сіздің темуңызды кезең-кезеңмен құрастырады.

## Theme Builder-ды Қайдан Бастау Керек? {#starting-the-theme-builder-onboarding}

### Бірінші рет орнату (First-Run Setup) {#first-run-setup}

Сіз Superdav AI Agent-ты жаңа WordPress орнатуында алғаш рет іске қоссаңыз, келесіні көресіз:

```
Superdav AI Agent-қа қош келдіңіз!

Не істегіңіз келеді?
1. Арнайы тема құру (Theme Builder)
2. Бар түріндегі тему пайдалану
3. Қазір үшін орнатуды өткізу
```

Theme Builder onboarding flow-ына кіру үшін **"Арнайы тема құру"**-ды таңдаңыз.

### Қолданып іске қосу (Manual Activation) {#manual-activation}

Сіз кез келген уақытта келесіні сұрау арқылы Theme Builder onboarding-ты бастай аласыз:

```
"Start the Theme Builder onboarding"
```

немесе

```
"Help me create a custom theme"
```

## Onboarding Кезеңдері {#the-onboarding-steps}

### 1-қадам: Режимді Таңдау (Mode Selection) {#step-1-mode-selection}

Setup Assistant agent сіздің қалауыңызды сұрайды:

```
Темаңызды қандай режимде құруды армандасыз?
- Бағдарланған (Мен сұрақтар қоямын және сіз үшін құрастырамын)
- Өзіңіз істеуші (Мен сізге мүмкіндіктерді көрсеремін, сіз шешесіз)
```

**Бағдарланған режим** көптеген пайдаланушыларға ұсынылады; агент сіздің салаңыз бен мақсаттарыңызға негізделген дизайн ұсыныстарын береді.

### 2-қадам: Сайттың Сипаттамасы (Site Specification) {#step-2-site-specification}

Сізге сайт туралы сұрақтар қойылады:

- **Сайттың мақсаты**: Электрондық сауда (E-commerce), блог, портфолио, SaaS және т.б.
- **Мақсатты аудитория**: Сіздің қонақтарыңыз кімдер?
- **Бренд түстері**: Негізгі және екінші түстер (немесе "білмеймін")
- **Тон**: Кәсіби, шығармашылық, ойын-азат, минималды т.б.

Бұл ақпарат сіздің **site_brief** есте сақтауыңызда сақталады және агенттер осы деректерді келесі сессияларда пайдаланады.

### 3-қадам: Дизайн жүйесін (Design System) шешімдері {#step-3-design-system-decisions}

Агент сізге дизайн токендерін таңдау барысында бағыт береді:

- **Типография**: Шрифт отбасы (serif, sans-serif, monospace) және өлшем шкаласы
- **Түстер палитрасы**: Негізгі, екінші, акцент және нейтралды түстер
- **Аспаптар арасындағы кеңістік (Spacing)**: Қысқартылған, нормалды немесе кең аспаптарды қолдану
- **Қозғалыс (Motion)**: Анимациялар мен ауысулар (қажет болса)

### 4-қадам: Теманы генерациялау {#step-4-theme-generation}

Setup Assistant агент сіздің арнайы блок темаңызды келесімен құрастырады:

- Барлық дизайн токендері бар `theme.json`
- Көрініс ретіндегі (басты бет, блог, байланыс) блок үлгілері
- Сіздің дизайн жүйеңізге сәйкес келетін арнайы блок стилдері
- Тема метадеректер мен WordPress қолдау туралы мәлімдемелер

### 5-қадам: Қолдану және тексеру {#step-5-activation-and-verification}

Тема автоматты түрде іске қосылады, және сіз мыналарды көресіз:

```
✓ Сіздің арнайы темаңыз енді жұмыс істеп тұр!
  Тема атауы: [Сіздің Сайт Атыңыз] Темасы
  Түстер: [Негізгі] / [Екінші]
  Типография: [Шрифт Отбасы]

  Жаңа дизайнды көру үшін сайтқа кіріңіз.
```

Содан кейін, сіз теманың дұрыс көрсетілгенін тексеру үшін сайтыңызға баруға болады.

## Сайт спецификациясы және site_brief есте сақтау {#site-specification-and-sitebrief-memory}

Орналасу кезінде агент сіздің сайттың сипаттамасын **site_brief** есте сақтау категориясына жинақтайды. Бұл:

- Сайттың мақсаты мен міндеттері
- Мақсатты аудитория
- Бренд түстері және тон
- Дизайн қалаулары
- Мазмұн құрылымы

### site_brief неге маңызды? {#why-sitebrief-matters}

Келесі сессияларда агенттер сіздің site_brief деректеріне сүйенеді, бұл:

### Сайт синопсисін (site\_brief) көрсету {#viewing-your-sitebrief}

Сіз агентке мына сұрақтарды қоя аласыз:

```
"Менің сайт синопсисімді көрсет"
```

немесе

```
"Сайтым туралы не білесің?"
```

Агент сақталған сайт спецификацияңызды көрсетеді.

## Бастапқы орналасудан кейінгі реттеулер (Customizing After Onboarding) {#customizing-after-onboarding}

Theme Builder бастапқы орналасу аяқталғаннан кейін, сіз мыналарды істей аласыз:

### Дизайн жүйесі эстетикасы дағдысын қолдану {#use-the-design-system-aesthetics-skill}

Дизайнды жетілдіру сұраңыз:

```
"Типографиканы одан әрі заманауи етіп ретте"
```

немесе

```
"Түс палитрасын жылырақ ретте"
```

**Design System Aesthetics skill** сізге нақты дизайн жаңартуларды бағыттайды.

### theme.json-ды тікелей редакциялау {#edit-themejson-directly}

Жоғары деңгейдегі пайдаланушылар үшін, параметрлерді реттеу үшін `/wp-content/themes/[theme-name]/theme.json` файлына қол жеткізіп, мыналарды өзгертуге болады:

- Color tokens (Түс токендері)
- Typography scales (Типографика шкалалары)
- Spacing values (Аралық мәндері)
- Border and shadow definitions (Шығыс және көлеңке анықтамалары)

### Құпия блок үлгілерін жасау {#create-custom-block-templates}

WordPress блок редаторын пайдаланып, келесілер үшін арнайы үлгілерді құра аласыз:

- Homepage layouts (Басты бет құрылымдары)
- Blog post pages (Блог пост парақтары)
- Product pages (Өнім парақтары)
- Contact forms (Байланыс формалары)

## Салыстыру: Ескі және Жаңа Бастапқы Орналасу {#comparison-old-vs-new-onboarding}

| Ерекшелік | Сайт құрастырушы (Ескі) | Theme Builder (Жаңа) |
|---|---|---|
| Орнату әдісі | Wizard-based form (Визуалды бағдарламалық интерфейс арқылы форма) | Agent-guided conversation (Агентмен бағытталған сұхбат) |
| Тема генерациясы | Limited templates (Шектеулі үлгілер) | Custom scaffolding (Қолданылатын негіз құрылым) |
| Дизайн токендері | Manual entry (Қолайлы енгізу) | Guided decisions (Бағытталған шешімдер) |
| Гибділік | Fixed options (Белгілі нұсқалар) | Customizable (Қолданылатын) |
| Болашақ жаңартулар | Not referenced (Есте сақталмаған) | Stored in site\_brief (site\_brief-те сақталған) |

## Түзету мәселелерін шешу (Troubleshooting) {#troubleshooting}

**Бастапқы орналасу аяқталмады**
- Осы процесті қайта бастаңыз: "Start the Theme Builder onboarding" (Theme Builder бастапқы орналасуын бастау)
- WordPress орнатуың жаңартылғанын тексеріңіз.
- Setup Assistant агенттің қосылғанын растаңыз.

**Менің site_brief-ім қолланылмайды**
- Агенттің жадқа (memory) қол жеткізіп отыр екенін тексеріңіз.
- Агенттен "site brief-імді есте сақтау" деп сұраңыз.
- Жоспарларыңызда жадтың қосылғанын тексеріңіз.

**Генерацияланған тема мен өнімдеріммен сәйкес келмейді**
- Оны жетілдіру үшін Design System Aesthetics skill (құрылымдық эстетика) мүмкіндігін қолданыңыз.
- Агенттен "[нақты өзгерістер] арқылы тему қайта генерациялау" деп сұраңыз.
- Нақты бақылау үшін theme.json файлына тікелей өңдеу жасаңыз.

## Келесі қадамдар {#next-steps}

Theme Builder-ды орнатудан кейін:

1. **Сіздің сайтты тексеріңіз**: Жаңа темуді көру үшін сайтыңызға кіріңіз.
2. **Дизайнді жетілдіріңіз**: Нақты өзгерістер енгізу үшін Design System Aesthetics skill-ды қолданыңыз.
3. **Мазмұнды қосыңыз**: Сайттың мазмұнын құруды бастаңыз.
4. **Қабілеттерді зерттеңіз**: scaffold-block-theme және activate-theme сияқты басқа агент мүмкіндіктері туралы біліңіз.
