# Сайт Сипаттама Маهرة

**Сайт сипаттама маهرة** — бұл сіздің сайтыңыздың мақсаттарын, аудиториясын және бренд тұжырымдамасын ресімдеу тәсілі. Бұл ақпарат **site_brief** есте сақтау орнында сақталады, осылайша агенттер кезектегі сессияларда бірдей, контекстке байланысты көмек көрсету үшін оған сілтеме береді.

## Сайт Сипаттама дегеніміз не? {#what-is-site-specification}

Сайт сипаттама — келесілерді құжаттау процесі:

- **Сайт мақсаты**: Сіздің сайтыңыз не істейтіні және не үшін бар екені
- **Мақсатты аудитория**: Кім сіздің сайтыңызды қамтамасыз етеді және олар не қажет
- **Бренд тұжырымдамасы**: Сіздің түстеріңіз, тоныңыз және визуалды стиліңіз
- **Бизнес мақсаттары**: Сайт үшін табыс не сияқты көрінеді
- **Контент құрылымы**: Сіздің сайтыңыз қалай ұйымдастырылған

Бұл сипаттама сіздің **site_brief**-і болады, бұл агенттер сіздің сайтыңыздың контекстін түсіну үшін қолданатын тұрақты есте сақтау орны.

## Неліктен Сайт Сипаттамасын Қолдану Керек? {#why-use-site-specification}

### Сессиялар Арасында Бірдейлік {#consistency-across-sessions}

site_brief болмаса, әр жаңа сессия басталған кезде сіздің сайтыңыздың мақсатын қайта түсіндіруіңіз керек болады. Олмен агенттер дереу келесілерді түсінеді:

- Сайттың мақсаттары мен аудиториясы
- Бренд түстеріңіз және тоныңыз
- Контент құрылымыңыз
- Бизнес міндеттеріңіз

### Жақсы Ұсыныстар {#better-recommendations}

Агенттер site_brief-ті қолдана отырып:

- Сайттың мақсатына сәйкес келетін мүмкіндіктерді ұсынады
- Мақсаттарыңызға сәйкес келетін контент құрылымдарын ұсынады
- Брендіңізге сәйкес келетін дизайнды ұсынады
- Қайта танылмайтын мүмкіндіктерді ұсынудан аулақ болады

### Жылдам Бастау (Onboarding) {#faster-onboarding}

Жаңа агенттер (немесе жаңа сессиялардағы агенттер) нақты сұрақтар қоюдың орнына site_brief-ті оқып, тез үйрене алады.

## Сайт Сипаттамасын Бастау {#initiating-site-specification}

### Theme Builder Бастау Кезінде {#during-theme-builder-onboarding}

Site Specification маهرةсі **Theme Builder басталу ағымы** кезінде автоматты түрде іске қосылады. Setup Assistant агент сұрақтар қойып, сіздің site_brief-ті құрады.

### Қолдану арқылы Бастау {#manual-initiation}

Сіз кез келген уақытта Сайт специifikasyonын бастай аласыз:

```
"Менің сайтымнің спецификасын анықтайық"
```

немесе

```
"Сайт нотаты жасауға көмектесіңіз"
```

## Сайт спецификациясы процесі {#the-site-specification-process}

### 1-қадам: Сайттың мақсаты {#step-1-site-purpose}

Агент сұрайды:

```
Сіздің сайтыңыздың негізгі мақсаты қандай?
- Электрондық сауда дүкені (E-commerce store)
- Блог немесе контент сайты
- Портфолио немесе көрме
- SaaS қосымшасы
- Қоғамдастық немесе форум
- Басқа: [сипаттаңыз]
```

Сіз категорияны таңдай аласыз немесе өз мақсатыңызды сипаттай аласыз.

### 2-қадам: Мақсатты аудитория {#step-2-target-audience}

```
Сіздің негізгі аудиторияңыз кім?
- Тұтынушылар / жалпы халық
- Бизнес кәсіпкерлері
- Дамытушылар / техникалық пайдаланушылар
- Оқушылар / білім берушілер
- Басқа: [сипаттаңыз]

Олардың негізгі қажеттіліктері қандай?
```

### 3-қадам: Брендтің бейнесі (Brand Identity) {#step-3-brand-identity}

```
Сіздің брендіңіздің түстері қандай?
- Негізгі түс: [түс таңдағыш немесе шек код]
- Екінші түс: [түс таңдағыш немесе шек код]
- Акценттік түс: [опционалды]

Брендіңіздің тоны туралы қалай сипаттасаңыз келеді?
- Кәсіби / корпоративті
- Креативті / өнерпай
- Ойыншық / ыңғайлы
- Минималды / заманауи
- Жылы / достық
```

### 4-қадам: Бизнес мақсаттары {#step-4-business-goals}

```
Сіздің сайтыңыз үшін сәттілік не сипатталады?
- Клиенттерді тарту (leads)
- Өнім сату
- Қоғамдастық құру
- Білім бөлісу
- Авторитет орнату
- Басқа: [сипаттаңыз]

Сіздің негізгі өлшеміңіз не?
- Көлем (Revenue)
- Пайдаланушылардың қатысуы (User engagement)
- Контентті тарату
- Ақшалай әрекеттерге айналуы (Conversions)
- Басқа
```

### 5-қадам: Контент құрылымы {#step-5-content-structure}

```
Контентіңіз қалай ұйымдастырылған?
- Тұтас (барлық контент бір деңгейде)
- Иерархиялық (категориялар мен қосымша категориялар)
- Хронологиялық (блог стилінде)
- Өнімге негізделген (каталог)
- Басқа: [сипаттаңыз]

Қай контент түрлерін қолданасыз?
- Блог посттары
- Өнім беттері
- Кейс зерттеулер (Case studies)
- Документация
- Видеолар
- Басқа
```

## Сіздің site_brief жадыңыз {#your-sitebrief-memory}

Сайт спецификациясын толық аяқтағаннан кейін, сіздің ақпаратыңыз агент жадыңызда **site_brief** ретінде сақталады. Бұл құрылымдалған тіркес келесіні қамтиды:

## Сайтің сипаттамасын қарау және жаңарту {#viewing-and-updating-your-sitebrief}

### Сайт сипаттамаңызды көрсету {#view-your-sitebrief}

Агентке сұраңыз:

```
"Менің сайт сипаттамасымды көрсет"
```

немесе

```
"Сайт туралы не білесің?"
```

Агент сақталған спецификацияңызды көрсетеді.

### Сайт сипаттамасын жаңарту {#update-your-sitebrief}

Егер сіздің сайтыңыз өзгеретін болса, оны жаңартуға болады:

```
"Сайт сипаттамасымды жаңарту: енді біз B2B клиенттерге бағытталғанбыз"
```

немесе

```
"Сайт спецификациясын жаңарту"
```

Бұл сіздің бар ақпаратты бастапқы нүкте ретінде пайдаланып, Сайт Спецификация (Site Specification) құзыреттілігін қайта іске қосады.

## Агенттер сайт сипаттамасын қалай қолданады {#how-agents-use-sitebrief}

### Дизайн ұсыныстары {#design-recommendations}

Дизайн өзгерістерді сұраған кезде агент сіздің сайт сипаттамаңызға сүйенеді:

```
Сіз: "Мен басты бетті қайта дизайнына ауыстыр"
Агент: "Сайт сипаттамаңызға сүйенгенде, сіз 25-45 жастағы әйелдерге жылы, қолдан жасалған стильді аудиторияға бағытталдыңыз. Мен қолдан жасалған сақиналарыңыз бен тұрақты құндылықтарыңызды атап өтетін дизайн құрастырамын."
```

### Функция ұсыныстары {#feature-suggestions}

Агенттер сіздің мақсаттарыңызға сәйкес функцияларды ұсынады:

```
Сіз: "Қай қандай мүмкіндіктерді қосу керек?"
Агент: "Клиенттерге деген адалдыққа бағытталған электрондық сауда дүкені үшін мен келесілерді ұсынамын: адалдық бағдарламасы, клиенттердің пікірлері, электронды хабарламалар (email newsletter) және өнім ұсыныстары."
```

### Мазмұн құрылымы {#content-structure}

Агенттер сіздің құрылымыңызға негізделген мазмұнды жинақтауды ұсынады:

Сіз: "Блогты қалай ұйымдастыру керек?"
Агент: "Сіз блогпен өнім каталогыды қолдансаңыз, посттарды өнім категориялары бойынша ұйымдастыруға және бірнеше өнімді бірге көрсетілетін 'стильдік нұсқаулықтар' жасауға кеңес беремін."

## Ең жақсы тәжірибелер {#best-practices}

### Нақты болыңыз {#be-specific}

"Жалпы аудитория" дегеннен гөрі, нақты аудиторияңызды сипаттаңыз:

- ✓ "25-45 жастағы әйелдер, тұрақты модаға қызығушылық танушылар"
- ✗ "Барлығы"

### Үнемі жаңартыңыз {#update-regularly}

Сіздің сайтыңыз дамыған сайын `site_brief` (сайт сипаттамасы) мәліметтерін жаңартып отырыңыз:

- Жаңа аудиторияға ауыстырған кезде
- Жаңа өнім жиынтықтарын қоссаңыз
- Брендтің тұжырымдамасы өзгергенде
- Бизнес мақсаттары өзгергенде

### Бірдей терминологияны қолданыңыз {#use-consistent-terminology}

Сессиялар бойы бірдей терминдерді қолданыңыз:

- ✓ Әрқашан "тұрақты әшекейлер" деп айтыңыз (мысалы, "эко-досты әшекейлер" немесе "жасыл өнімдер" емес)
- ✓ Аудиторияны үнемі бірдей атаңыз

### Контекстті қосыңыз {#include-context}

Агенттердің шешімдеріңізді түсінуіне көмектесетін фон беріңіз:

- "Біз бағадан гөрі сапаны бағалайтын кәсібилерге бағытталған"
- "Біздің аудиториямы технологиялық білімді және заманауи дизайнды күтеді"
- "Біз бастапқы капиталмен жұмыс істейтін стартапымыз, сондықтан бізге шығын тиімді шешімдер қажет"

## Theme Builder-ге енумен байланысы {#relationship-to-theme-builder-onboarding}

Site Specification skill (Сайт сипаттамасы) функциясы **Theme Builder onboarding flow** (Theme Builder-дің ену процесі) ішіне интеграцияланған. Ену процесін аяқтаған кезде, сіз берген ақпаратпен `site_brief` автоматты түрде жасалады.

Егер сізге келесілер үшін Site Specification-ды тәуелсіз жүргізу қажет болса:

- Бастапқы орнатудан кейін сипаттамаңызды жетілдіру
- Сайт дамыған сайын сайт сипаттамасын жаңарту
- Theme Builder-ді бастамас бұрын егжей-тегжейлі сипаттама жасау

## Түзету мәселелерін шешу {#troubleshooting}

**Менің `site_brief` қолланылмай тұр**
- Агенттің есте сақтауға (memory) қол жеткізіп отырғанын растаңыз
- Агенттен "менің site brief-імді еске түсіру" деп сұраңыз
- Есте сақтаудың сіздің параметрлерінде қосылғанын тексеріңіз

**Мен жаңа site\_brief-пен қайта бастағым келеді**
- Агентке сұраңыз: "Site brief-імді тазалап, жаңадан баста"
- Содан кейін Site Specification-ды қайта орындаңыз

**Агент менің site\_brief-іммен сәйкес келмейтін ұсыныстар береді**
- Агенттен: "Site brief-імді қарашы" деп сұраңыз
- Егер site\_brief ескірсе, оны жаңартыңыз
- Сұрауларыңызға қосымша контекст беріңіз

## Келесі қадамдар {#next-steps}

site specification-ды анықтап алғаннан кейін:

1. **Theme Builder-ді қолданыңыз**: site\_brief-іне негізделген арнайы theme құрыңыз
2. **Дизайнды жетілдіріңіз**: егжей-тегжейлі дизайн жұмыстары үшін Design System Aesthetics skill-ін пайдаланыңыз
3. **Мазмұнды жоспарлаңыз**: мазмұн құрылымы бойынша ұсыныстар алу үшін агенттерден сұраңыз
4. **Функционалдық элементтерді құрыңыз**: бизнес мақсаттарыңызға сай функцияларды сұраңыз
