# Басқа плагиндтермен танымал келіспейділіктер {#known-incompatibilities-with-other-plugins}

Ultimate Multisite-ты орнатып, іске қосығаннан кейін, басқа плагиндтермен шешілмеген келіспейділіктерден dolayı мәселе туындауы мүмкін.

Біз әр жаңа إصلاحпен бұл тізімді азайту бойынша белсенді жұмыс істейміз, бірақ егер плагин осында тізімде болса, толыққанды Ultimate Multisite желісін жұмыс істеу үшін оны өшіру қажет болуы мүмкін.

Міне, олардың тізімі және мұндай жағдайларда не істеу керектігі:

**Бірнеше WPMU DEV плагиндері** WPMU DEV-мен енді қолдамайтын кейбір плагиндер Ultimate Multisite тіркелу процесін соңғы қадамда (көбірек пайдаланушы аккаунт жасай алмайды) тоқтататыны белгілі. Бұл мәселе тудыратын плагиндердің тізімі:

  * Signup Language: _Ultimate Multisite-тың осы функционалдықты ұсынатын қосымшасы бар, оның атауы Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Бұл Ultimate Multisite-қа енгізілген, сондықтан қосымша плагин қажет емес._

  * New Blog Templates: _Бұл Ultimate Multisite-қа енгізілген, сондықтан қосымша плагин қажет емес._

  * Set Password on Multisite Blog Creation: _Бұл Ultimate Multisite-қа енгізілген, сондықтан қосымша плагин қажет емес._

  * Signup TOS: _Бұл Ultimate Multisite-қа енгізілген, сондықтан қосымша плагин қажет емес._

  * Anti-Splog: Бұл әлдеқайда аз күшті болғанымен, бұл да _Ultimate Multisite-қа енгізілген_.

**Ескерту**: Басқа барлық WPMU Dev плагиндерін Ultimate Multisite-пен қатар қалыпты түрде қолдануға болады. Мысалы: _Smush_ , _Forminator_ , _Defender_, және т.б._
