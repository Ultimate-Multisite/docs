# Доменлік продукциялар және бағалау {#domain-products-and-pricing}

Доменлік продукциялар — бұл сіз бағаны, TLD-лерді (топ düzeyін), WHOIS құпиялылығын және провайдерді таңдауды басқаратын құралдар. Әр доменлік продукт стандартты Multisite Ultimate продукту болып табылады және оның қосымша **Domain Settings** вкладкасы бар.

## Доменлік продукт жасау {#creating-a-domain-product}

1. **Network Admin › Ultimate Multisite › Products**-ке өтіңіз
2. **Add New**-ға басу арқылы жаңасын қосыңыз
3. Продукт түрін **Domain** деп белгілеңіз
4. **Domain Settings** вкладкасын (аşağıда қараңыз) реттеңіз
5. Сақтау

Доменлік продукциялар продукт тізімінде көк түсті **Domain** бейнесімен пайда болады және оларды **Domain Products** вкладкасы арқылы сүзгіле аласыз.

## Domain settings tab (Домен параметрлері) {#domain-settings-tab}

### Провайдер {#provider}

Бұл продукт үшін тіркелуді қай ресми тіркеуші басқаратынын таңдаңыз. Бұл глобал **Default provider** (Глобалдық Default провайдер) орнатылған Domain Seller параметрлерінде белгіленген мәнімен анықталады.

### Қолдау көрсетілетін TLD-лер {#supported-tlds}

Басқа продукттармен сәйкес келмейтін барлық TLD-лерге қолданылатын **catch-all product** (барлығына арналған продукт) жасау үшін бос bırakңыз. Белгілі бір топ düzeyдерге ғана қолданылатын **TLD-specific product** (TLD-ге арналған продукт) жасау үшін TLD-лердің келесін көрсетіңіз (мысалы: `.com, .net, .org`).

**Продукт сәйкестендіру жұмысы:** Клиент домен іздеген кезде, қосымша ең нақты сәйкес келетін продуктты таңдайды. TLD тізімінде `.com` бар продукт басқа продуктан тыс catch-all продукттан артық құқыққа ие болады. Егер TLD-ге арналған продукт сәйкес келмесе, catch-all қолданылады. Продукттер болмаса, домен іздеу көрсетілмейді.

### Markup type (Бағалау түрі) {#markup-type}

Сіздің жеткізуші құнынан ретейлді бағаңызды есептеу тәсілін үш режим басқарады:

| Режим | Жұмыс істеуі |
|---|---|
| **Percentage** (Проценттік) | Жеткізуші құнына пайыз қосады. 10 доллар тұратын жеткізуші доменге 20% markup қоссаңыз, бағасы 12 доллар болады. |
| **Fixed markup** (Белгілі markup) | Белгілі бір нақты ақша сомасын қосады. 10 доллар тұратын доменге 5 доллар markup қоссаңыз, бағасы 15 доллар болады. |
| **Fixed price** (Белгілі баға) | Жеткізуші құнына толық мән бермейді. Әрқашан сіз енгізген соманы тұрады. |

### Introductory pricing (Кіріспе бағасы) {#introductory-pricing}

### Скидка на первый год предложить {#whois-privacy}

Первый год для скидочного тарифа установить. Регулярный **Обновляемый тариф** (с второго года и далее) зейнерімен **Вводный тариф** (первого года) і встановіть окремо. Клієнт бачить обидві ціни під час оформлення замовлення, щоб знати, чого очікувати при поновленні.

### Конфіденційність WHOIS {#tld-import-and-sync}

Контролює, чи пропонується захист WHOIS від усіх для доменів, зареєстрованих через цей продукт.

| Налаштування | Поведінка |
|---|---|
| **Вимкнено** | Захист WHOIS ніколи не пропонується або не активується. |
| **Завжди включено** | Захист WHOIS автоматично активується при реєстрації безкоштовно. |
| **Вибір клієнта** | Під час оформлення замовлення з'являється чекбокс. Встановіть **Ціну на конфіденційність** як плату за рік або залиште $0, щоб запропонувати її безкоштовно. |

Для Namecheap захист WHOIS використовує WhoisGuard (завжди безкоштовно). Для OpenSRS він використовує сервіс приватності OpenSRS (може мати вартість при оптових закупівлях).

## Імпорт та синхронізація TLD {#auto-renewal}

Доменні продукти відображають ціни оптового продажу в режимі реального часу, отримані від підключеного провайдера. Для цього необхідно імпортувати TLD (домени верхнього рівня).

- **Ручна синхронізація:** Налаштування › Продавець домену › Синхронізувати TLD (за провайдером)
- **Автоматична синхронізація:** Запускається щодня через заплановане завдання cron на всіх налаштованих провайдерах.

Після синхронізації перейдіть у вкладку Налаштування домену будь-якого доменного продукту та скористайтеся вибором TLD, щоб побачити доступні TLD з їхніми поточними оптовими цінами.

## Автоматичне поновлення {#admin-manual-domain-registration}

Поновлення доменів прив'язане до статусу членства клієнта:

- Коли членство поновлюється і домен прив'язується, поновлення домену автоматично ставиться у чергу
- Спроби поновлення використовують активний платіжний шлюз клієнта
- Неуспішні поновлення повторюються автоматично з експоненційним відступом (exponential backoff)
- Електронні сповіщення надсилаються про успішне поновлення, невдачі та наближення закінчення терміну дії

| Событие | Шаблон ID |
|---|---|
| Домен зарегистрирован | `domain_registered` |
| Домен продлен | `domain_renewed` |
| Продление не удалось | `domain_renewal_failed` |
| Домен скоро истечет | `domain_expiring_soon` |

## Администратор: Ручная регистрация домена

Клиента через процесс оформления заказа без прохождения его через него зарегистрировать домен можно следующим образом:

1. Перейдите в **Network Admin › Ultimate Multisite › Register Domain** (Администрирование сети › Ultimate Multisite › Зарегистрировать домен)
2. Выберите клиента и введите имя домена
3. Заполните контактные данные регистратора (имя, адрес, телефон)
4. Нажмите **Register** (Зарегистрировать)

Запись домена будет создана и привязана к учетной записи клиента.
