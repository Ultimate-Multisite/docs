# Домен Салещик: Настройка и Конфигурация Провайдера

Дополнение Domain Seller поставляется с управляемым мастером настройки, который проведет вас через каждый необходимый шаг. Эта страница описывает процесс работы мастера и то, как настроить или перенастроить провайдеров после этого.

## Требования {#requirements}

- **Multisite Ultimate** v2.4.12 или выше, активированный по сети
- **PHP** 7.4+
- API учетные данные для хотя бы одного поддерживаемого регистратора

## Мастер настройки при первом запуске {#first-run-setup-wizard}

Мастер настройки автоматически запускается в первый раз, когда вы активируете плагин по сети. Он также доступен в любое время через **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Шаг 1 — Выберите провайдера {#step-1--choose-a-provider}

Выберите регистратора, с которым вы хотите подключиться. Поддерживаемые варианты:

| Провайдер | Управление DNS | Конфиденциальность WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Нет | Да (WhoisGuard, бесплатно) |
| HostAfrica | Да | Да (Защита ID) |
| Openprovider | Да | Да |
| Hostinger | Через основное сопоставление доменов Hostinger для хостинговых доменов | Да |
| GoDaddy | Нет | Нет |
| ResellerClub | Да | Нет |
| NameSilo | Нет | Нет |
| Enom | Да | Нет |

### Шаг 2 — Введите учетные данные {#step-2--enter-credentials}

У каждого провайдера поля для ввода учетных данных разные:

**OpenSRS** — Имя пользователя и приватный ключ (из Панели управления реселлером OpenSRS)

**Namecheap** — Имя пользователя и API-ключ (из Account › Tools › API Access)

**HostAfrica** — Конечная точка API для реселлера доменов и учетные данные от модуля реселлера HostAfrica. Отдельная песочница конечная точка в настоящее время не документирована; тестируйте безопасными операциями только на чтение перед запуском реальных регистраций.

**Openprovider** — Имя пользователя и пароль с включенным доступом к API. Опциональный режим песочницы использует sandbox API Openprovider, а опциональное имя клиента по умолчанию может быть повторно использовано для регистраций.

**Hostinger** — Негізгі Hostinger интеграциясынан алынған ортақ Hostinger hPanel API токені. Бұл токен негізгі домен картасын және Домен Сатушыны тіркеу операцияларын жүргізуге арналған.

**GoDaddy** — API кілті және құпия (developer.godaddy.com-дан)

**ResellerClub** — Реселлер ID және API кілті (ResellerClub басқару панелінен)

**NameSilo** — API кілті (namesilo.com › Account › API Manager-дан)

**Enom** — Есептік ID және API токені

Қолжетімді болса, тіркелуден бұрын жеткізуге арналған тестілеу үшін **Sandbox mode**-ды тексеріңіз.

### 3-қадам — Байланысты тексеру {#step-3--test-the-connection}

**Test Connection**-ға басыңыз. Мастер құрал токендер мен қосылысты тексеру үшін жеңіл API шағынын жібереді. Бұрын кез келген токен мәселелерін түзетіп, жалғасуға дайын болыңыз.

### 4-қадам — TLD-ларды импорттау {#step-4--import-tlds}

Қосылған провайдерден барлық қолжетімді TLD-ларды және үлкен жеткізу бағаларын алу үшін **Import TLDs**-ке басыңыз. Бұл домен өнімдерін пайдалануда қолданылатын TLD тізімін толтырады. Үлкен TLD каталогы бар провайдерлер үшін импорт 30–60 секундқа созылуы мүмкін.

TLD-лар күнде бір рет кестеленген cron жұмыс арқылы автоматты түрде қайта синхроланады.

### 5-қадам — Домен өнімін жасау {#step-5--create-a-domain-product}

Мастер құрал 10% маржамен стандартты жалпы домен өнімін жасайды. Сіз осы өнімді дереу редакциялау немесе **Ultimate Multisite › Products** астында қолмен өнімдерді жасап, өткізуіңізді өткізіп жіере аласыз.

Толық өнім конфигурациясын көру үшін [Domain Products and Pricing](./domain-products) сілтемесіне қараңыз.

## Провайдерді қайта конфигуралау {#reconfiguring-a-provider}

**Network Admin › Ultimate Multisite › Settings › Domain Seller** (немесе плагин тізімінен **Settings**-ке басыңыз).

Бұл параметрлер парағында мыналар бар:

- **Домен продау алуды** — бұл мүмкіндікті толығымен қосу/өшіру (toggle) арқылы іске қосу немесе өшіру.
- **Бастапқы провайдер** — домен іздеулер мен жаңа өнімдер үшін қолданылатын провайдер.
- **Іздеуге арналған TLD саны** — клиент іздеген кезде қанша TLD тексеруді көрсету керек; жоғары мәндер көбірек нұсқаларды көрсетеді, бірақ олар баяу жұмыс істейді.
- **Қолжетімділік кештің ұзақтығы** — қолжетімділік және баға нәтижелерін қанша уақыт кешті сақтау керек; төмен мәндер нақтырақ, бірақ API шақыруларын арттырады.
- **Домен өнімдерін басқару** — Өнімдер тізіміне жылдам сілтеме.
- **Провайдерлерді конфигурациялау** — провайдерлерді қосу немесе қайта реттеу үшін Интеграция Wizard (Integration Wizard) ашылады.

### Екінші провайдерді қосу {#adding-a-second-provider}

**Configure providers**-ды басып, жаңа тіркеушіге арналған wizard-ды қайта іске қосаңыз. Сіз бір мезгілде бірнеше провайдерді конфигурациялауға боласыз. Әр домен өнімін нақты провайдерге тағайындаңыз немесе оны стандартты ретінде қалдырыңыз.

### TLD-ларды қолмен синхролау {#syncing-tlds-manually}

Settings парағында, соңғы бағаны алу үшін конфигурацияланған кез келген провайдердің жанында **Sync TLDs**-ді басыңыз. Бұл провайдер үкіметтік сатып алу бағаларын жаңартуы немесе жаңа TLD-ларды қосудан кейін пайдалы болады.

## Журналдар (Logs) {#logs}

Әр провайдер өз журналы арқылы жазып отырады. Журналдар **Network Admin › Ultimate Multisite › Logs** астында қарауға болады:

| Лог канал | Содержимое |
|---|---|
| `domain-seller-registration` | Все попытки регистрации (успешные и неудачные) |
| `domain-seller-renewal` | Результаты задания на продление |
| `domain-seller-opensrs` | Сырая активность API OpenSRS |
| `domain-seller-namecheap` | Сырая активность API Namecheap |
| `domain-seller-hostafrica` | Сырая активность API HostAfrica |
| `domain-seller-openprovider` | Сырая активность API Openprovider |
| `domain-seller-hostinger` | Сырая активность API Hostinger |
| `domain-seller-godaddy` | Сырая активность API GoDaddy |
| `domain-seller-resellerclub` | Сырая активность API ResellerClub |
| `domain-seller-namesilo` | Сырая активность API NameSilo |
| `domain-seller-enom` | Сырая активность API Enom |

## Продвиговых возможностей провайдеров (Provider capability notes) {#provider-capability-notes}

Не все регистраторов API предоставляют одинаковые операции. Дополнение показывает неподдерживаемые операции с понятными ошибками для администратора, вместо того чтобы молча завершаться неудачей.

- **HostAfrica** поддерживает самый широкий рабочий процесс реселлера в реальном времени, включая поиск, синхронизацию TLD/цены, регистрацию, продление, перевод, обновление имен серверов, DNS-записи, коды EPP, блокировку регистратора и защиту ID.
- **Openprovider** поддерживает синхронизацию TLD по цене реселлера, регистрацию, продление, переводы, обновление имен серверов, DNS-зоны, коды EPP, блокировку регистратора и конфиденциальность WHOIS. Он аутентифицируется с кратковременным токеном bearer, который дополнение автоматически обновляет.
- **Hostinger** поддерживает поиск доступности, регистрацию, поиск портфолио, обновление имен серверов, блокировку регистратора и конфиденциальность WHOIS через общий токен API hPanel. Публичный API доменов Hostinger не раскрывает цены реселлера/оптовых цен, входящий перевод, явное продление или получение кодов EPP; продления происходят только автоматически.
