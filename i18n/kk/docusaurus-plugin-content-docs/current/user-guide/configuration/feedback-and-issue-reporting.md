# Клиенттер пікірісі және мәселелерді езеге салу {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 енгізу арқылы пайдаланушылар тікелей чат интерфейсінен көмектеспейтін жауаптарды белгілеуге және мәселені хабарлауға мүмкіндік беретін кіріктірілген кері байланыс және мәселелерді езеге салу жүйесі енгізілді. Бұл жүйе келісімді басқару, автоматтандырылған езеге салу командасы және бэкендте AI көмегімен талдауды қамтиды.

## Басқамайтын (Thumbs-Down) Батырма {#thumbs-down-button}

AI ассистенті арқылы жіберілген әрбір хабарда **басқамайтын** (👎) батырма көрсетіледі. Пайдаланушы оны басып өткенде, жауапты көмектеспейтін немесе дұрыс емес деп белгілей алады.

- Батырма әр ассистент хабарламасының жанында курсормен ұстағанда пайда болады.
- Оны басу арқылы **Кері байланыс келісімді модальы** ашылады.
- Кері байланыс қызметтің тарихына және нақты хабарламаға қосылады.

## Кері байланыс келісімді модальы (Feedback Consent Modal) {#feedback-consent-modal}

Пайдаланушы басқамайтын батырманы басып өткенде, кез келген деректер жіберілуден бұрын келісімді сұрау модалы пайда болады. Модаль:

- Қандай ақпарат жиналатынын түсіндіреді (сөйлесудан үзінді, браузер контексті).
- Пайдаланушыдан осы деректерді бөлісуге келісім беретінін растауды сұрайды.
- Қате немесе қандай қате кеткенін сипаттау үшін пайдаланушыға еркін мәтінді өріс ұсынады.
- Жоспарланбаған жағдайда жібермейді **Қажетсіз (Cancel)** опциясын ұсынады.

Пайдаланушы нақты растама бергенше ешқандай кері байланыс тіркелмейді.

## Автоматтандырылған Кері байланыс баннері (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Сөйлесу аяқталған кезде, ассистент **автоматтандырылған кері байланыс баннерін** көрсетуі мүмкін — бұл сессия пайдалы болды деп сұрауға арналған қысымсыз хабарлама.

Бұл баннер сөйлесудің ұзындығы мен нәтижесінің белгілі алгоритмдеріне негізделген автоматты түрде пайда болады. Ол басқамайтын батырмамен бірдей кері байланыс ағынына сілтеме береді. Пайдаланушы кері байланыс бермей баннерді жоюға болады.

## /report-issue Слэш командасы {#report-issue-slash-command}

Пайдаланушылар чат кірінішіне `/report-issue` деп жазып, кері байланыс ағынын тікелей іске қосуы мүмкін. Бұл команда:

- Пікінінге (Feedback Consent Modal) модалі бірден ашылады.
- Сипаттама өрісін қазіргі әңгіме туралы контекстпен автоматты толтырады.
- Пайдаланушылар жіберу алдында қосымша мәліметтерді қосуға мүмкіндік береді.

`/report-issue` командасы барлық чат режимдерінде (inline, floating widget, full-screen) қолжетімді.

## AI-Assisted Triage (Жасанды интеллект арқылы талдау) {#ai-assisted-triage}

Жіберілген пікірлер жасанды интеллект арқылы талдау қабатына жіберіледі, ол:

- Есепті санаттауды жүзеге асырады (фактық қате, көмексіз жауап, тиіспейтін контент және т.б.).
- Қосымша әңгіме метадеータсын қосады.
- Талдау қорытындысын конфигурацияланған **Feedback Endpoint**-ке (Settings > Advanced [Settings > Advanced] қараңыз) жібереді.

Бұл ең маңызды мәселелерді алдымен көріп, қолмен қарау уақытын қысқартады.

## Settings > Advanced {#settings-advanced} (Реттілік > Қосымша) {#settings--advanced}

Пікірлер артық бэкэндиін іске қосу үшін **Gratis AI Agent → Settings → Advanced** бөлімінде келесі өрістерді реттеңіз:

| Field (Өріс) | Description (Сипаттамасы) |
|---|---|
| **Feedback Endpoint URL** (Пікірлерді жіберетін конекциялық нүкте URL-ы) | Пікір жүктемелерін (JSON) қабылдайтын URL. |
| **Feedback API Key** (Пікірлерді растау үшін `Authorization` басылында жіберілетін Bearer токені) | Пікірлерді жіберген кезде аутентификация үшін қолданылатын токен. |

Екі өрісті бос қалдырып қойсаңыз, пікір жинауын өшіре аласыз. Әрбір пікір батырмалары пайдаланушыларға көрініп тұрады, бірақ жіберілген пікірлер жолға жіберілмейді.

:::tip
Пікір конекциясы кемінде `message_id`, `conversation_id`, `feedback_text` және `triage_category` өрістері бар JSON денесін қабылдауы керек. Күтілетін схема үшін сіздің конекцияңыз жеткізушінің құжаттамасын қараңыз.
:::
