# Панель управления интеграциясын жетілдіру {#enhance-control-panel-integration}

## Жалпы ақпарат {#overview}
Enhance — бұл қуатты хостинг автоматтандыру және басқару мүмкіндіктерін ұсынатын заманауи панель. Бұл интеграция Ultimate Multisite мен Enhance Control Panel арасында домендерді автоматты синхролау және SSL сертификаттарын басқаруға мүмкіндік береді.

**Қосымша талқылау:** Қоғамдастық кеңестер мен қосымша ақпарат алу үшін [GitHub Талқылауы #265](https://github.com/Multisite-Ultimate/ultimate-multisite/discussions/265) қараңыз.

## Ерекшеліктер {#features}
- Ultimate Multisite-те домендер карталалған кезде домендерді автоматты синхролау
- DNS шешімделгенде LetsEncrypt арқылы автоматты SSL сертификатын беру
- Субдомен режимде жұмыс істейтін желілер үшін субдомен қолдауы
- Карталар өшірілген кезде доменді алып тастау
- API құжаттарын тексеру үшін қосылымды тексеру

## Қажеттіліктер {#requirements}

### Жүйелік қажеттіліктер {#system-requirements}
- Enhance Control Panel орнатылған және қолжетімді болуы керек
- WordPress Multisite орнатылған және Enhance серверіне хостингтелген немесе оған қосылған
- Apache веб-сервері (Enhance қазіргі уақытта Apache конфигурацияларын қолдайды; LiteSpeed Enterprise азаматтық бағамен қолжетімді)

### API Қолжетімділігі {#api-access}
API токендерін жасау үшін Enhance Control Panelге администратор ретіндегі қолжетімділігіңіз болуы керек.

## API құжаттарыңызды алу {#getting-your-api-credentials}

### 1. API Токенін жасау {#1-create-an-api-token}

1. Администратор ретінде Enhance Control Panel-ге кіріңіз
2. Навигациялық менюдан **Settings** (Жоспарлау) батырмасын басыңыз
3. **Access Tokens** (Қолжетімділік токендері) бөліміне өтіңіз
4. **Create Token** (Токен жасау) батырмасын басыңыз
5. Токенге сипаттамалы атау беріңіз (мысалы, "Ultimate Multisite Integration")
6. **System Administrator** (Жүйелік администратор) рөлін тағайындаңыз
7. Мерзімді аяқтау үшін:
   - Токен ешқары мерзімі жоқ болуын қаласаңыз бос bırakтыңыз (бос қалдырыңыз)
   - Немесе қауіпсіздік мақсаттары үшін нақты мерзімді белгілеңіз
8. **Create** (Жасау) батырмасын басыңыз

Оларды жасағаннан кейін сізге **Access Token** және **Organization ID** көрсетіледі. Оларды дереу сақтаңыз, өйткені токен бір рет ғана көрсетіледі.

### 2. Сіздің Organization ID-ңізді алу {#2-get-your-organization-id}

Organization ID синяя ақпараттық қораптың "Org ID: {your_id}" деп белгіленген Access Tokens парағында көрсетілген.

Organization ID келесідей UUID форматында болады: `d8554b6d-5d0d-6719-009b-fec1189aa8f3`

Сіз клиенттің Organization ID-сын мына жолмен де таба аласыз:
1. **Customers** парағына өтіңіз
2. Қамқор клиент үшін **Manage customer**-ды басыңыз
3. URL-ді қараңыз - Organization ID `/customers/` артындағы әріптерден тұратын алғашқы нәрсе.

### 3. Сіздің Server ID-ңізді алу {#3-get-your-server-id}

Server ID-ны (домен операциялары үшін қажет) табу үшін:

1. Enhance Control Panel-де **Servers**-ке өтіңіз
2. WordPress орнатылған серверге басыңыз
3. Server ID (UUID форматында) URL-де немесе сервер сипаттамасында көрінеді
4. Немесе, серверлер тізімін алу үшін API-ды қолдануға болады:

```bash
curl -s -X GET https://your-enhance-panel.com/api/servers \
  -H "Accept: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" | jq
```

Server ID келесі UUID форматын ұстанады: `00000000-0000-0000-0000-000000000000`

### 4. Сіздің API URL-іңізді алу {#4-get-your-api-url}

Сіздің API URL-іңіз – Enhance Control Panel URL-ына `/api/`-ды қоса отырып алынады:

```
https://your-enhance-panel.com/api/
```

**Маңызды:** `/api/` жолы міндетті. Көрінетін қателер:
- Тек доменді және `/api/`-ны қолданбау
- Қауіпсіздік үшін HTTPS-ке қарағанда HTTP қолдану (қайтару үшін HTTPS қажет)

## Конфигурация {#configuration}

### Міндетті константалар {#required-constants}

Төмендегі константаларды `wp-config.php` файлыңызға қосыңыз:

// Бұйрық панелі интеграциясын жетілдіру
define('WU_ENHANCE_API_TOKEN', 'your-bearer-token-here');
define('WU_ENHANCE_API_URL', 'https://your-enhance-panel.com/api/');
define('WU_ENHANCE_SERVER_ID', 'your-server-uuid-here');

### Интеграция wizard арқылы орнату {#setup-via-integration-wizard}

1. WordPress администраторында **Ultimate Multisite** > **Settings** (Жоғарғы көпсайт) жолдарына өтіңіз
2. **Integrations** (Интеграциялар) табына барыңыз
3. **Enhance Control Panel Integration**-ды табыңыз және **Configuration** (Конфигурация) батырмасын басыңыз
4. Wizard сізге орнату процесінде бағыт береді:
   - **1-қадам:** Кіріспе және мүмкіндіктерге шолу
   - **2-қадам:** API құжаттарыңызды енгізіңіз (Token, API URL, Server ID)
   - **3-қадам:** Байланысты тексеріңіз
   - **4-қадам:** Қарап-қартып және белсендіріңіз

Сіз келесілерді таңдай аласыз:
- Wizard арқылы константаларды `wp-config.php` файлына автоматты түрде енгізуге
- Константа анықтамаларын көшіріп, қолмен қосуға

## Қосымша WordPress конфигурациясы {#additional-wordpress-configuration}

Қоғамдық пікірлерге сүйене отырып ([Discussion #265](https://github.com/Multisite-Ultimate/ultimate-multisite/discussions/265)), сізге келесі қосымша параметрлерді реттеу қажет болуы мүмкін:

### .htaccess Конфигурациясы {#htaccess-configuration}

Домен картасында мәселе туындаса:
1. Ескі Enhance `.htaccess` файлын өшіріңіз
2. Оны стандартты WordPress Multisite `.htaccess` файлымен ауыстырыңыз

### Cookie константалары {#cookie-constants}

Карталанған домендер бойынша cookie-ларды дұрыс басқару үшін осы константаларды `wp-config.php`-ке қосыңыз:

```php
define('COOKIE_DOMAIN', $_SERVER['HTTP_HOST']);
define('COOKIEPATH', '/');
define('ADMIN_COOKIE_PATH', '/');
```

## Қалай жұмыс істейді {#how-it-works}

### Домен карталанған кезде {#when-a-domain-is-mapped}

1. Пользователь Ultimate Multisite-тегінде кастомный доменді карталайды (немесе поддомен режимінде жаңа сайт жасалады).
2. Интеграция Enhance API-ге POST сұрағысын жібереді: `/servers/{server_id}/domains`
3. Enhance доменді сіздің сервер конфигурацияңызға қосады
4. DNS сіздің серверіңізге бағытталған кезде, Enhance LetsEncrypt арқылы SSL сертификатын автоматты түрде қамтамасыз етеді
5. Домен HTTPS-пен белсенді болады

### Домен алынылған кезде {#when-a-domain-is-removed}

1. Ultimate Multisite-те домен картасы жойылады
2. Интеграция доменнің ID-сін табу үшін Enhance-ке сұрау береді
3. Төмендегіге DELETE сұрағы жіберіледі: `/servers/{server_id}/domains/{domain_id}`
4. Enhance доменді сіздің сервер конфигурацияңыздан алады

### DNS және SSL тексеруі {#dns-and-ssl-checking}

Ultimate Multisite-те енгізілген DNS және SSL тексеру функциялары бар:
- Тексеру аралығын **Domain Mapping Settings** (барыштағы мән: 300 секунд/5 минут) параметрлерінен реттей аласыз
- Доменді белсенді деп белгілеуден бұрын жүйе DNS таралуын тексеріп отырады
- SSL сертификатының жарамдылығы автоматты түрде тексеріледі
- Enhance SSL қамтамасыз етуді автоматты түрде басқарады, сондықтан қолмен SSL конфигурациялау қажет емес

## Орнатуды тексеру {#verifying-setup}

### Байланысты сынау {#test-the-connection}

1. Интеграция Wizard-ында **Test Connection** (Байланысты тексеру) қадамын пайдаланыңыз
2. Плагин сіздің серверіңіздегі домендер тізімін көрсетуді ты ప్రయత్тайды
3. Үлкен мәліметтер:
   - API құжаттары дұрыс
   - API URL қолжетімді
   - Server ID дұрыс
   - Рұқсаттар योग्य орнатылған

### Доменді карталғаннан кейін {#after-mapping-a-domain}

1. Ultimate Multisite-те тест доменді карталаңыз
2. Ultimate Multisite журналында тексеріңіз (**Ultimate Multisite** > **Logs** > **integration-enhance**)
3. Enhance Control Panelде домен қосылғанын растаңыз:
   - **Servers** > **Your Server** > **Domains**-қа өтіңіз
   - Жаңа домен тізімде пайда болуы керек
4. DNS тарағаннан кейін SSL автоматты түрде қамтамасыз етілгенін тексеріңіз

## Оғайындау (Troubleshooting) {#troubleshooting}

### API Байланыс Мәселелері {#api-connection-issues}

**Қате: "Enhance API-ға қосыла алмады"**
- `WU_ENHANCE_API_URL` соңында `/api/` бар екенін тексеріңіз.
- HTTP емес, HTTPS қолданылып жатқанын растаңыз.
- Enhance панелінің WordPress серверінен қолжетімді екенін тексеріңіз.
- Байланысты бұғаттатын (firewall) ешқандай ережелер жоқ екенін тексеріңіз.

**Қате: "Enhance API токені табылмады"**
- `wp-config.php` файлында `WU_ENHANCE_API_TOKEN` анықталғанын растаңыз.
- Токен Enhance-де өшірілмеген немесе мерзімі өткен екенін тексеріңіз.
- Токен мәнінде жазу қатесі бар екенін тексеріңіз.

**Қате: "Сервер ID конфигурацияланбаған"**
- `wp-config.php` файлында `WU_ENHANCE_SERVER_ID` анықталғанын растаңыз.
- Сервер ID дұрыс UUID форматында екенін тексеріңіз.
- Сервердің Enhance панелінде бар екенін растаңыз.

### Домен Қосылмаған {#domain-not-added}

**Журналдарды тексеру:**
1. **Ultimate Multisite** > **Logs** (Журналдар) бөліміне өтіңіз.
2. **integration-enhance** бойынша сүзгілеу жасаңыз.
3. Мәселені білдіретін қате хабарламаларды іздеңіз.

**Көрінетін себептер:**
- Домен атауы дұрыс форматта ма екенін тексеріңіз.
- Домен Enhance-де алдын ала бар болуы мүмкін.
- API рұқсаттары жеткіліксіз (токенде "System Administrator" рөлі бар екенін қамтамасыз етіңіз).
- Сервер ID Enhance-дегі нақты сервермен сәйкес келмейді.

### SSL Sertifikatсы Мәселелері {#ssl-certificate-issues}

**SSL енгізілмеу:**
- DNS-тің сервердің IP-браундысына бағытталғанын тексеріңіз.
- Домен дұрыс шешілетінін тексеріңіз: `nslookup yourdomain.com`
- Enhance SSL енгізуден бұрын DNS-тің шешілуін талап етеді.
- SSL енгізу әдетте DNS таратылғаннан кейін 5-10 минут алады.
- SSL-ге арналған қателер үшін Enhance Control Panel журналына қараңыз.

**Enhance-те қолмен SSL-ді шешу:**
1. **Servers** > **Your Server** (Серверлері) > **Domains** (Домендер) бөліміне өтіңіз.
2. Доменіңізді табыңыз және оның SSL мәртебесін тексеріңіз.
3. Қажет болса, SSL енгізуді қолмен іске қосуға болады.

### DNS Тексеру Кезегі {#dns-check-interval}

Домендер немемелер немемеле сертификаттарды активдеуге көп уақыт кетеді:
1. **Ultimate Multisite** > **Settings** > **Domain Mapping**-ке өтіңіз
2. **DNS Check Interval** параметрлерін табыңыз
3. Оны стандартты 300 секундтан төменгі мәзге (минималды: 10 секунд) өзгертіңіз
4. **Назар аударыңыз:** Төменгі аралықтар дегеніңіз, тексерулер жиірек істейді, бірақ сервердің жүктемесі жоғары болады

### Аутентификация қателері {#authentication-errors}

**HTTP 401/403 қателері:**
- Enhance-те API токенін қайта жасаңыз (Regenerate)
- Токенде **System Administrator** рөлі бар екенін тексеріңіз
- Токен мерзімі өтпегенін тексеріңіз
- Дұрыс ұйым ID-сін қолданып отырғаныңызды қамтамасыз етіңіз (алайда, бұл URL-де әдетте қажет емес)

### Журналды талдау {#log-analysis}

Толық журналды қосыңыз:
```php
// Көсуді арттыру үшін wp-config.php-ке қосыңыз
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

Содан кейін журналдарды келесі жерден тексеріңіз:
- Ultimate Multisite журналы: **Ultimate Multisite** > **Logs**
- WordPress дебаг журналы: `wp-content/debug.log`
- Enhance панелі журналы: Enhance-тің администратор интерфейсінде қолжетімді

## API Сілтемелері {#api-reference}

### Аутентификация {#authentication}
Барлық API сұраныстары Bearer токен аутентификациясын қолданады:
```
Authorization: Bearer YOUR_TOKEN_HERE
```

### Көбінесе қолданылатын эндпоинттар {#common-endpoints-used}

**Серверлер тізімін алу:**
```
GET /servers
```

**Сервердегі домендерді тізімdeleу:**
```
GET /servers/{server_id}/domains
```

**Домен қосу:**
```
POST /servers/{server_id}/domains
Body: {"domain": "example.com"}
```

**Доменді өшіру:**
```
DELETE /servers/{server_id}/domains/{domain_id}
```

### Толық API құжаттамасы {#full-api-documentation}
Толық API құжаттамасы: [https://apidocs.enhance.com](https://apidocs.enhance.com)

## Ең жақсы тәжірибелер {#best-practices}

### Қауіпсіздік {#security}
- **API токендерін ешқашан версионды басқару жүйесіне (version control) коммит етпеңіз**
- Токендерді Git-тен тыс болуы тиіс `wp-config.php` файлына сақтаңыз
- Тиісті рұқсаттармен токендерді қолданыңыз (толық интеграция үшін Система әкімшісін пайдаланыңыз)
- Өндірістік орталар үшін токен мерзімдерін белгілеңіз
- Токендерді мерзімі бойынша ауыстырыңыз

### Өнімділік {#performance}
- Артық API шақыруларды болдырмау үшін стандартты DNS тексеру аралығын (300 секунд) қолданыңыз
- Үлкен масштабтағы домен операцияларын жүргізу кезінде Enhance сервері ресурстарын бақылаңыз
- Көптеген домендерді бір мезгілде картаға алуды қажет ететін болса, домендерді реттеуді кезең-кезеңді қарастырыңыз

### Бақылау {#monitoring}
- Интеграциялық қателер үшін Ultimate Multisite журналына үнемі тексеріңіз
- Домендерді қосудағы сәтсіздіктерге бақылау орнатыңыз
- SSL сертификаттары дұрыс жұмыс істеп тұрғанын тексеріңіз
- Enhance серверінің мүмкіндіктері мен домен шектеулерін бақылаңыз

## Ескертулер {#notes}

Бұл интеграция тек домен псевдонимелерін ғана қамтамасыз етеді; Enhance SSL-ді автоматты түрде басқарады.
Интеграция арнайы домен карталарын және поддоменге негізделген сайттарды қолдайды.
Автоматты www поддомены құру Domain Mapping параметрлерінде реттеуге болады.
Enhance қазіргі уақытта Apache конфигурацияларын (LiteSpeed Enterprise қолжетімді) қолдайды.
Ultimate Multisite-тан доменді алып тастаған кезде, ол Enhance-тан доменді алады, бірақ тиісті SSL сертификаттары дереу өшірілуі мүмкін емес.
