# साइट बिल्डर मोड हटाउने सूचना

**Superdav AI Agent v1.12.0 मा Site Builder mode हटाउन गरिएको छ।** यदि तपाईंले Site Builder mode प्रयोग गर्दै हुनुहुन्थ्यो भने, थीम निर्माण र साइट सेटअपका लागि **Setup Assistant agent** मा माइग्रेट गर्नुपर्नेछ।

## के भयो?

### Site Builder Mode (पुरानो)

Site Builder mode निम्न कुराहरूको लागि एक विजार्ड-आधारित इन्टरफेस थियो:

- टेम्प्लेटबाट साइटहरू बनाउने
- आधारभूत सेटिङहरू कन्फिगर गर्ने
- थीम छान्ने
- प्रारम्भिक सामग्री सेटअप गर्ने

### यसले के प्रतिस्थापन गर्‍यो?

**Setup Assistant agent** ले निम्न कुराहरूसँग सबै Site Builder कार्यक्षमतालाई ह्यान्डल गर्छ:

- अझ बढी लचिलो, एजेन्ट-निर्देशित सेटअप
- राम्रो थीम कस्टमाइजेसन विकल्पहरू
- Theme Builder onboarding सँगको एकीकरण
- भविष्यका सत्रहरूको लागि स्थायी `site_brief` मेमोरी

## यदि तपाईं Site Builder Mode प्रयोग गर्दै हुनुहुन्थ्यो भने

### तपाईंको साइटहरू सुरक्षित छन्

- Site Builder mode ले सिर्जित विद्यमान साइटहरू काम गरिरहन्छन्
- कुनै डेटा हानि वा साइटमा अवरोध हुँदैन
- तपाईंले आफ्नो साइटहरू सामान्य रूपमा व्यवस्थापन जारी राख्न सक्नुहुन्छ

### Setup Assistant Agent मा माइग्रेट गर्नुहोस्

नयाँ साइट सेटअप वा थीम परिवर्तनका लागि, Setup Assistant agent प्रयोग गर्नुहोस्:

```
"Help me set up a new site"
```

वा

```
"Start the Theme Builder onboarding"
```

Setup Assistant agent ले बढी लचिलो तर उस्तै कार्यक्षमता प्रदान गर्दछ।

## तुलना: Site Builder बनाम Setup Assistant

| विशेषता | Site Builder (हटाएको) | Setup Assistant (नयाँ) |
|---------|----------------------|----------------------|
| सेटअप विधि | विजार्ड फारम | एजेन्ट वार्तालाप |
| थीम चयन | पूर्व-परिभाषित टेम्प्लेट | कस्टम निर्माण |
| कस्टमाइजेसन | सीमित विकल्पहरू | पूर्ण डिजाइन प्रणाली |
| साइट ब्रीफ | भण्डारण गरिएको छैन | स्थायी मेमोरी |
| भविष्यका सत्रहरू | दोहोर्याएको सेटअप | भण्डार गरिएको `site_brief` प्रयोग गर्ने |
| लचिलोपन | निश्चित वर्कफ्लो | अनुकूलन हुने वार्तालाप |

## Setup Assistant Agent मा माइग्रेट गर्ने

### नयाँ साइटहरूको लागि

साइट बिल्डर मोडको सट्टा प्रयोग गर्नुहोस्:

१. अनुरोध: "म एउटा नयाँ साइट सेटअप गर्न मद्दत गर्नुहोस्"
२. सेटअप असिस्टेन्ट एजेन्टले तपाईंलाई निम्न कुराहरूमा मार्गदर्शन गर्नेछ:
   - साइटको उद्देश्य र लक्ष्य
   - लक्षित दर्शक (Target audience)
   - ब्रान्ड पहिचान (Brand identity)
   - थीम निर्माण (Theme generation)
   - प्रारम्भिक कन्फिगरेसन

### विद्यमान साइटहरूको लागि

यदि तपाईंसँग साइट बिल्डर मोडबाट एउटा विद्यमान साइट छ भने:

१. तपाईं यसलाई जस्तो छ त्यस्तै प्रयोग गर्न सक्नुहुन्छ
२. थीम अपडेट गर्न, अनुरोध गर्नुहोस्: "मेरो साइटको डिजाइन बदल्नुहोस्"
३. सेटअप असिस्टेन्ट एजेन्टले तपाईंलाई नयाँ थीम बनाउन मद्दत गर्नेछ
४. तपाईंको साइट डेटा अपरिवर्तित रहनेछ

### थीम परिवर्तनका लागि

साइट बिल्डर मोडको थीम चयनको सट्टा:

१. अनुरोध गर्नुहोस्: "मेरो थीम बदल्नुहोस्"
२. सेटअप असिस्टेन्ट एजेन्टले निम्न कार्य गर्दछ:
   - तपाईंको डिजाइन प्राथमिकताहरूको बारेमा सोध्ने
   - कस्टम थीम उत्पन्न गर्ने
   - यसलाई तपाईंको साइटमा सक्रिय गर्ने

## मुख्य भिन्नताहरू

### साइट बिल्डर मोड

```
१. टेम्प्लेट छान्नुहोस्
२. थीम चयन गर्नुहोस्
३. आधारभूत सेटिङहरू कन्फिगर गर्नुहोस्
४. समाप्त
```

### सेटअप असिस्टेन्ट एजेन्ट

```
१. तपाईंको साइटको उद्देश्य वर्णन गर्नुहोस्
२. आफ्नो लक्षित दर्शक परिभाषित गर्नुहोस्
३. डिजाइन प्राथमिकताहरू छान्नुहोस्
४. एजेन्टले कस्टम थीम उत्पन्न गर्दछ
५. एजेन्टले थीम सक्रिय गर्दछ
६. भविष्यका सत्रहरूको लागि साइट संक्षिप्त विवरण (site_brief) भण्डारण गरिन्छ
```

## सेटअप असिस्टेन्ट एजेन्टका फाइदाहरू

### बढी लचिलो (More Flexible)

- प्राकृतिक भाषामा आफ्नो साइटको वर्णन गर्नुहोस्
- कस्टम सिफारिसहरू प्राप्त गर्नुहोस्
- तपाईंको विशिष्ट आवश्यकता अनुसार अनुकूलन गर्न सक्नुहुन्छ

### राम्रो कस्टमाइजेसन (Better Customization)

- कस्टम थीम निर्माण
- डिजाइन प्रणालीका निर्णयहरू
- स्थायी डिजाइन टोकनहरू (Persistent design tokens)

### स्थायी मेमोरी (Persistent Memory)

- तपाईंको `site_brief` भण्डारण गरिन्छ
- भविष्यका एजेन्टहरूले तपाईंको साइट बुझ्छन्
- सेटअप जानकारी दोहोर्याउनु पर्दैन

### एकीकृत कार्यप्रवाह (Integrated Workflow)

- थीम बिल्डर अनबोर्डिङ
- डिजाइन प्रणालीको सौंदर्य कौशल (Design System Aesthetics skill)
- दृश्यता नियन्त्रण गर्ने क्षमता (Ability Visibility controls)
- सबै काम सहज रूपमा एकसाथ चल्छ

## समस्या निवारण (Troubleshooting)

### म साइट बिल्डर मोड फेला पार्न सक्दिनँ

साइट बिल्डर मोड हटाइसकेको छ। यसको सट्टा सेटअप असिस्टेन्ट एजेन्ट प्रयोग गर्नुहोस्:

"म नयाँ साइट सेटअप गर्न मलाई सहयोग गर्नुहोस्"

### म Site Builder बाट एउटा साइट बनाउन चाहन्छु

तपाईं Setup Assistant agent प्रयोग गरेर यसलाई पुनः बनाउन सक्नुहुन्छ:

१. अनुरोध गर्नुहोस्: "म नयाँ साइट सेटअप गर्न सहयोग गर्नुहोस्"
२. आफ्नो पुरानो साइटको उद्देश्य र डिजाइन वर्णन गर्नुहोस्
३. एजेन्टले तपाईंको जस्तै थीम सिर्जना गर्नेछ
४. भविष्यको सन्दर्भका लागि तपाईंको `site_brief` सुरक्षित गरिनेछ

### मेरो विद्यमान Site Builder साइट काम गरिरहेको छैन

Site Builder मोड प्रयोग गरेर पहिले बनाएका साइटहरू अझै पनि काम गर्छन्। यदि तपाईंलाई समस्या भइरहेको छ भने:

१. तपाईंको थीम सक्रिय छ कि छैन जाँच गर्नुहोस्
२. तपाईंको प्लगइनहरू सक्षम छन् कि छैनन् प्रमाणित गर्नुहोस्
३. WordPress एरर लगहरू (error logs) जाँच गर्नुहोस्
४. समस्या अझै रहेमा सपोर्टलाई सम्पर्क गर्नुहोस्

### म आफ्ना पुरानो Site Builder टेम्प्लेट प्रयोग गर्न सक्छु?

Site Builder टेम्प्लेट अब उपलब्ध छैनन्। तर:

- तपाईंको विद्यमान साइटहरू काम गरिरहन्छन्
- Setup Assistant agent प्रयोग गरेर तपाईंले यस्तै जस्तै साइटहरू पुनः बनाउन सक्नुहुन्छ
- Setup Assistant agent ले थप अनुकूलन (customization) विकल्पहरू प्रदान गर्दछ

## अर्को चरणहरू

१. **नयाँ साइटहरूको लागि**: Setup Assistant agent प्रयोग गर्नुहोस्
२. **विद्यमान साइटहरूको लागि**: तिनीहरूलाई जस्ताको तस्तै प्रयोग गरिरहनुहोस्
३. **थीम परिवर्तनका लागि**: Setup Assistant agent बाट सहयोग अनुरोध गर्नुहोस्
४. **डिजाइन परिष्करणका लागि**: Design System Aesthetics skill प्रयोग गर्नुहोस्

## सम्बन्धित विषयहरू

- **Theme Builder Onboarding**: कस्टम थीमहरूको लागि निर्देशित सेटअप
- **Setup Assistant Agent**: एजेन्टद्वारा निर्देशित साइट सेटअप
- **Site Specification Skill**: तपाईंको साइटको लक्ष्य र दर्शकलाई परिभाषित गर्नुहोस्
- **Design System Aesthetics Skill**: तपाईंको साइटको दृश्य पहिचान (visual identity) परिष्कृत गर्नुहोस्
