# थीम बिल्डर ऑनबोर्डिंग फ्लो

Superdav AI Agent v1.12.0 तपाईंलाई सुरुवाती सेटअपको समयमा कस्टम ब्लक थीम बनाउन मद्दत गर्ने एउटा निर्देशित **Theme Builder onboarding flow** प्रस्तुत गर्दछ। यसले पुरानो Site Builder मोडलाई अझ लचिलो, एजेन्ट-सहायता प्राप्त दृष्टिकोणले प्रतिस्थापन गर्दछ।

## Theme Builder Onboarding Flow भनेको के हो?

Theme Builder onboarding flow एक अन्तरक्रियात्मक सेटअप विजार्ड (interactive setup wizard) हो जसले निम्न कुराहरू गर्छ:

- डिजाइन निर्णयहरू (रङ, टाइपोग्राफी, लेआउट) मा तपाईंलाई मार्गदर्शन गर्छ
- तपाईंको साइटको दृश्य पहिचान प्राथमिकताहरू गोल्न गर्छ
- तपाईंको आवश्यकता अनुसार कस्टम ब्लक थीम उत्पन्न गर्छ
- पूरा भएपछि स्वचालित रूपमा थीम सक्रिय गर्छ

यो फ्लो **Setup Assistant agent** द्वारा संचालित हुन्छ, जसले स्पष्टीकरणका प्रश्नहरू सोध्छ र तपाईंको थीमलाई चरणबद्ध रूपमा बनाउँछ।

## Theme Builder Onboarding सुरु गर्ने

### पहिलो पटक सेटअप (First-Run Setup)

जब तपाईं नयाँ WordPress इन्स्टलेसनमा Superdav AI Agent लाई पहिलो पटक चलाउनुहुन्छ, तपाईंले निम्न कुराहरू देख्नुहुनेछ:

```
Superdav AI Agent मा स्वागत छ!

तपाईं के गर्न चाहनुहुन्छ?
1. कस्टम थीम बनाउने (Theme Builder)
2. विद्यमान थीम प्रयोग गर्ने
3. अहिले सेटअप छुटाउने
```

Theme Builder onboarding flow मा प्रवेश गर्न **"Build a custom theme"** चयन गर्नुहोस्।

### म्यानुअल सक्रियता (Manual Activation)

तपाईं कुनै पनि समयमा निम्न अनुरोध गरेर Theme Builder onboarding सुरु गर्न सक्नुहुन्छ:

```
"Start the Theme Builder onboarding"
```

वा

```
"Help me create a custom theme"
```

## ऑनबोर्डिङका चरणहरू

### चरण १: मोड चयन (Mode Selection)

Setup Assistant agent ले तपाईंको प्राथमिकताको बारेमा सोध्छ:

```
तपाईं आफ्नो थीम कसरी बनाउन चाहनुहुन्छ?
- निर्देशित (म प्रश्नहरू सोध्छु र तपाईंको लागि बनाउँछु)
- प्रत्यक्ष काम गर्ने शैली (म तपाईंलाई विकल्प देखाउँछु र तपाईं निर्णय गर्नुहुन्छ)
```

अधिकांश प्रयोगकर्ताहरूको लागि **Guided mode** सिफारिस गरिन्छ; एजेन्टले तपाईंको उद्योग र लक्ष्यहरूको आधारमा डिजाइनका सुझावहरू दिन्छ।

### चरण २: साइट विवरण (Site Specification)

तपाईंले आफ्नो साइटको बारेमा सोध्न आउनुहुनेछ:

- **साइटको उद्देश्य**: ई-कमर्स, ब्लग, पोर्टफोलियो, SaaS आदि।
- **लक्ष्य दर्शक**: तपाईंका आगन्तुकहरू को हुन्?
- **ब्रान्ड रंग**: प्राथमिक र द्वितीयक रंग (वा "मलाई थाहा छैन")
- **स्वर (Tone)**: व्यावसायिक, रचनात्मक, चञ्चल, न्यूनतम, आदि।

यो जानकारी तपाईंको **site_brief** मेमोरीमा भण्डारण गरिएको हुन्छ, जसलाई भविष्यका सत्रहरूमा एजेन्टहरूले सन्दर्भ गर्छन्।

### चरण ३: डिजाइन सिस्टम निर्णयहरू

एजेन्टले तपाईंलाई डिजाइन टोकन (design token) चयन गर्ने प्रक्रियामा मार्गदर्शन गर्दछ:

- **टाइपोग्राफी**: फन्ट परिवार (serif, sans-serif, monospace) र साइजको क्रम
- **रंग योजना**: प्राथमिक, द्वितीयक, उच्चारण (accent), र तटस्थ (neutral) रंगहरू
- **स्पेसिंग (Spacing)**: सानो, सामान्य, वा प्रशस्त लेआउटहरू
- **मोशन (Motion)**: एनिमेसन र ट्रान्जिशनहरू (यदि चाहनुहुन्छ भने)

### चरण ४: थीम निर्माण

सेटअप असिस्टेन्ट एजेन्टले तपाईंको कस्टम ब्लक थीमलाई निम्न कुराहरूसहित ढाँचा (scaffold) गर्छ:

- सबै डिजाइन टोकनहरू समावेश भएको `theme.json`
- सामान्य लेआउटहरूको लागि ब्लक टेम्प्लेट (होमपेज, ब्लग, सम्पर्क)
- तपाईंको डिजाइन सिस्टमसँग मेल खाने कस्टम ब्लक शैलीहरू
- थीम मेटाडेटा र वर्डप्रेस समर्थन घोषणाहरू

### चरण ५: सक्रियता र प्रमाणीकरण (Activation and Verification)

थीम स्वचालित रूपमा सक्रिय हुन्छ, र तपाईंले निम्न कुराहरू देख्नुहुनेछ:

```
✓ तपाईंको कस्टम थीम अब सञ्चालनमा छ!
  Theme name: [Your Site Name] Theme
  Colors: [Primary] / [Secondary]
  Typography: [Font Family]

  नयाँ डिजाइन हेर्न आफ्नो साइटमा जानुहोस्।
```

त्यसपछि, तपाईंले थीम सही तरिकाले प्रदर्शन गरिरहेको छ वा छैन भनेर पुष्टि गर्न आफ्नो साइटमा जान सक्नुहुन्छ।

## साइट विवरण र site_brief मेमोरी

अनबोर्डिङ (onboarding) को समयमा, एजेन्टले तपाईंको साइटको विवरणलाई **site_brief** मेमोरी श्रेणीमा सङ्कलन गर्छ। यसमा निम्न कुराहरू समावेश हुन्छन्:

- साइटको उद्देश्य र लक्ष्यहरू
- लक्षित दर्शक
- ब्रान्ड रंग र स्वर
- डिजाइन प्राथमिकताहरू
- सामग्री संरचना (Content structure)

### site_brief किन महत्त्वपूर्ण छ?

भविष्यका सत्रहरूमा, एजेन्टहरूले तपाईंको site_brief लाई निम्न कुराहरूको लागि सन्दर्भ गर्छन्:

### आफ्नो साइटको संक्षिप्त जानकारी हेर्नुहोस् (Viewing Your site_brief)

तपाईं एजेन्टलाई यसो सोध्न सक्नुहुन्छ:

```
"Show me my site brief"
```

वा

```
"What do you know about my site?"
```

एजेन्टले तपाईंको भण्डार गरिएको साइट स्पेसिफिकेसन देखाउनेछ।

## ऑनबोर्डिङ पछि अनुकूलन गर्ने (Customizing After Onboarding)

Theme Builder को ऑनबोर्डिङ पूरा भएपछि, तपाईं निम्न कुराहरू गर्न सक्नुहुन्छ:

### डिजाइन सिस्टम एस्थेटिक्स स्किल प्रयोग गर्नुहोस् (Use the Design System Aesthetics Skill)

डिजाइन सुधारका लागि अनुरोध गर्नुहोस्:

```
"Refine the typography to be more modern"
```

वा

```
"Adjust the color palette to be warmer"
```

**Design System Aesthetics skill** ले तपाईंलाई लक्षित डिजाइन अपडेटहरूमा मार्गदर्शन गर्छ।

### theme.json मा सिधै सम्पादन गर्नुहोस् (Edit theme.json Directly)

उन्नत प्रयोगकर्ताहरूको लागि, निम्न कुराहरू समायोजन गर्न `/wp-content/themes/[theme-name]/theme.json` सम्पादन गर्नुहोस्:

- कलर टोकनहरू
- टाइपोग्राफी स्केलहरू
- स्पेसइङ मानहरू
- बर्डर र शैडो परिभाषा

### कस्टम ब्लक टेम्प्लेटहरू सिर्जना गर्नुहोस् (Create Custom Block Templates)

WordPress ब्लक एडिटर प्रयोग गरेर निम्नका लागि कस्टम टेम्प्लेटहरू बनाउनुहोस्:

- होमपेज लेआउटहरू
- ब्लग पोस्ट पेजहरू
- उत्पादन पेजहरू
- सम्पर्क फारमहरू

## तुलना: पुरानो बनाम नयाँ ऑनबोर्डिङ (Comparison: Old vs. New Onboarding)

|---|---|---|
| सेटअप विधि | विजार्ड-आधारित फारम | एजेन्ट-निर्देशित कुराकानी |
| थीम निर्माण | सीमित टेम्प्लेटहरू | कस्टम स्क्याफोल्डिङ |

## समस्या निवारण (Troubleshooting)

**ऑनबोर्डिङ प्रवाह पूरा भएन (The onboarding flow didn't complete)**
- प्रवाह पुनः सुरु गर्नुहोस्: "Start the Theme Builder onboarding"
- तपाईंको WordPress इन्स्टलेसन अद्यावधिक भएको छ कि छैन जाँच गर्नुहोस्
- सेटअप असिस्टेन्ट एजेन्ट सक्रिय छ वा छैन पुष्टि गर्नुहोस्

**मेरो साइट_ब्रीफ प्रयोग भइरहेको छैन**
- एजेन्टसँग मेमोरीमा पहुँच छ कि छैन भनेर पुष्टि गर्नुहोस्।
- एजेन्टलाई "मेरो साइट_ब्रीफ सम्झनुहोस्" (recall my site brief) भनेर सोध्नुहोस्।
- तपाईंको सेटिङहरूमा मेमोरी सक्षम गरिएको छ वा छैन भनेर जाँच गर्नुहोस्।

**उत्पन्न भएको थीम मेरो प्राथमिकतासँग मेल खाँदैन**
- यसलाई परिष्कृत गर्न Design System Aesthetics skill प्रयोग गर्नुहोस्।
- एजेन्टलाई "[विशिष्ट परिवर्तनहरू] सहित थीम पुनः उत्पन्न गर्नुहोस्" (regenerate the theme with [specific changes]) भनेर सोध्नुहोस्।
- सटीक नियन्त्रणका लागि theme.json मा सिधै सम्पादन गर्नुहोस्।

## अर्को चरणहरू

Theme Builder को अनबोर्डिङ पूरा गरेपछि:

१. **तपाईंको साइट प्रमाणित गर्नुहोस्**: नयाँ थीम हेर्न आफ्नो साइटमा जानुहोस्।
२. **डिजाइन परिष्कृत गर्नुहोस्**: समायोजनका लागि Design System Aesthetics skill प्रयोग गर्नुहोस्।
३. **सामग्री थप्नुहोस्**: आफ्नो साइटको सामग्री बनाउन सुरु गर्नुहोस्।
४. **क्षमताहरू अन्वेषण गर्नुहोस्**: scaffold-block-theme र activate-theme जस्ता अन्य एजेन्ट क्षमताहरूको बारेमा सिक्नुहोस्।
