# साइट स्पेसिफिकेसन सील

**साइट स्पेसिफिकेसन स्किल** भनेको तपाईंको वेबसाइटको लक्ष्य, दर्शक र ब्रान्ड पहिचानलाई व्यवस्थित तरिकाले रेकर्ड गर्ने एउटा ढाँचा हो। यो जानकारी तपाईंको **site_brief** मेमोरीमा भण्डारण गरिन्छ, जसलाई विभिन्न सत्रहरूमा एजेन्टहरूले सन्दर्भ दिन प्रयोग गर्छन् ताकि लगातार र सन्दर्भ-सचेत सहायता प्रदान गर्न सकियोस्।

## साइट स्पेसिफिकेसन के हो? {#what-is-site-specification}

साइट स्पेसिफिकेसन भनेको निम्न कुराहरूको दस्तावेज गर्ने प्रक्रिया हो:

- **साइटको उद्देश्य**: तपाईंको वेबसाइटले के गर्छ र किन अस्तित्वमा छ
- **लक्ष्य दर्शक**: कसले तपाईंको वेबसाइटमा आउँछ र उनीहरूलाई के चाहिन्छ
- **ब्रान्ड पहिचान**: तपाईंको रंग, टोन (स्वर) र दृश्य शैली
- **व्यावसायिक लक्ष्य**: तपाईंको साइटका लागि सफलता कस्तो देखिन्छ
- **सामग्री संरचना**: तपाईंको साइट कसरी व्यवस्थित छ

यो स्पेसिफिकेसन तपाईंको **site_brief** बन्नेछ, जुन एजेन्टहरूले तपाईंको साइटको सन्दर्भ बुझ्न प्रयोग गर्ने स्थायी मेमोरी हो।

## साइट स्पेसिफिकेसन किन प्रयोग गर्ने? {#why-use-site-specification}

### सत्रहरूमा निरन्तरता (Consistency Across Sessions) {#consistency-across-sessions}

यदि site_brief नभएको भए, हरेक नयाँ सत्र सुरु गर्दा तपाईंले आफ्नो साइटको उद्देश्य फेरि व्याख्या गर्नुपर्थ्यो। यसको साथमा, एजेन्टहरू तुरुन्तै बुझ्छन्:

- तपाईंको साइटका लक्ष्य र दर्शक
- तपाईंको ब्रान्डका रंग र टोन
- तपाईंको सामग्री संरचना
- तपाईंको व्यावसायिक उद्देश्यहरू

### राम्रो सिफारिसहरू (Better Recommendations) {#better-recommendations}

एजेन्टहरूले आफ्नो site_brief प्रयोग गरेर निम्न कुराहरू गर्छन्:

- तपाईंको साइटको उद्देश्यसँग मिल्ने फिचरहरू सुझाव दिन
- तपाईंको लक्ष्यहरूसँग मेल खाने सामग्री संरचनाहरू सिफारिस गर्न
- तपाईंको ब्रान्डसँग सुसंगत डिजाइन प्रस्ताव गर्न
- असंगत फिचरहरू सुझाव दिनबाट बच्न

### छिटो सुरुवात (Faster Onboarding) {#faster-onboarding}

नयाँ एजेन्टहरू (वा नयाँ सत्रहरूमा भएका एजेन्टहरू) स्पष्टीकरणका प्रश्न सोध्नुको सट्टा तपाईंको site_brief पढ्दा चाँडै काम सुरु गर्न सक्छन्।

## साइट स्पेसिफिकेसन कसरी सुरु गर्ने? {#initiating-site-specification}

### थीम बिल्डर ओन्बोर्डिङको समयमा (During Theme Builder Onboarding) {#during-theme-builder-onboarding}

साइट स्पेसिफिकेसन स्किल स्वचालित रूपमा **Theme Builder onboarding flow** को दौरान सक्रिय हुन्छ। सेटअप असिस्टेन्ट एजेन्टले प्रश्न सोध्छ र तपाईंको site_brief बनाउँछ।

### म्यानुअल सुरुवात (Manual Initiation) {#manual-initiation}

तपाईं कुनै पनि समयमा साइट स्पेसिफिकेसन सुरु गर्न सक्नुहुन्छ:

```
"मेरो साइटको स्पेसिफिकेसन परिभाषित गरौं"
```

वा

```
"मलाई एक साइट ब्रीफ बनाउन मद्दत गर्नुहोस्"
```

## साइट स्पेसिफिकेसन प्रक्रिया {#the-site-specification-process}

### चरण १: साइटको उद्देश्य (Site Purpose) {#step-1-site-purpose}

एजेन्टले सोध्छ:

```
तपाईंको साइटको मुख्य उद्देश्य के हो?
- ई-कमर्स स्टोर
- ब्लग वा सामग्री साइट
- पोर्टफोलियो वा प्रदर्शन स्थल
- SaaS एप्लिकेशन
- समुदाय वा फोरम
- अन्य: [वर्णन गर्नुहोस्]
```

तपाईंले कुनै श्रेणी छान्नुहोस् वा आफ्नो उद्देश्य वर्णन गर्न सक्नुहुन्छ।

### चरण २: लक्षित दर्शक (Target Audience) {#step-2-target-audience}

```
तपाईंको मुख्य लक्षित दर्शक को हुन्?
- उपभोक्ता / सामान्य जनता
- व्यवसायका पेशेवरहरू
- डेभलपर / प्राविधिक प्रयोगकर्ताहरू
- विद्यार्थी / शिक्षकहरू
- अन्य: [वर्णन गर्नुहोस्]

उनीहरूको मुख्य आवश्यकता के हो?
```

### चरण ३: ब्रान्ड पहिचान (Brand Identity) {#step-3-brand-identity}

```
तपाईंको ब्रान्डका रंगहरू के हुन्?
- प्राथमिक रंग: [रङ चयनकर्ता वा हेक्स कोड]
- द्वितीयक रंग: [रङ चयनकर्ता वा हेक्स कोड]
- उच्चारण रंग (Accent color): [वैकल्पिक]

तपाईंले आफ्नो ब्रान्डको टोनलाई कसरी वर्णन गर्नुहुन्छ?
- व्यावसायिक / कर्पोरेट
- रचनात्मक / कलात्मक
- चञ्चल / अनौपचारिक
- न्यूनतम / आधुनिक
- न्यानो / मैत्रीपूर्ण
```

### चरण ४: व्यवसायका लक्ष्यहरू (Business Goals) {#step-4-business-goals}

```
तपाईंको साइटको लागि सफलता कस्तो देखिन्छ?
- लीडहरू उत्पन्न गर्ने
- उत्पादन बेच्ने
- समुदाय निर्माण गर्ने
- ज्ञान साझा गर्ने
- अधिकार स्थापित गर्ने
- अन्य: [वर्णन गर्नुहोस्]

तपाईंको मुख्य मापन (metric) के हो?
- राजस्व (Revenue)
- प्रयोगकर्ता संलग्नता (User engagement)
- सामग्रीको पहुँच (Content reach)
- कन्वर्जन्सहरू (Conversions)
- अन्य
```

### चरण ५: सामग्री संरचना (Content Structure) {#step-5-content-structure}

```
तपाईंको सामग्री कसरी व्यवस्थित गरिएको छ?
- सपाट (सबै सामग्री एउटै स्तरमा)
- पदानुक्रमित (श्रेणी र उप-श्रेणीहरू)
- कालानुक्रमिक (ब्लग शैली)
- उत्पादन-आधारित (कैटलग)
- अन्य: [वर्णन गर्नुहोस्]

तपाईंले कुन प्रकारको सामग्री प्रयोग गर्नुहुन्छ?
- ब्लग पोस्टहरू
- उत्पादन पृष्ठहरू
- केस स्टडीहरू
- कागजातहरू (Documentation)
- भिडियोहरू
- अन्य
```

## तपाईंको site_brief मेमोरी {#your-sitebrief-memory}

साइट स्पेसिफिकेसन पूरा गरेपछि, तपाईंको जानकारी **site_brief** को रूपमा तपाईंको एजेन्ट मेमोरीमा सुरक्षित हुन्छ। यो निम्न कुराहरू समावेश भएको एक संरचित रेकर्ड हो:

## आफ्नो साइट ब्रीफ हेर्न र अपडेट गर्ने {#viewing-and-updating-your-sitebrief}

### आफ्नो साइट ब्रीफ हेर्ने {#view-your-sitebrief}

एजेन्टलाई सोध्नुहोस्:

```
"Show me my site brief"
```

वा

```
"What do you know about my site?"
```

एजेन्टले तपाईंको भण्डार गरिएको विवरण देखाउनेछ।

### आफ्नो साइट ब्रीफ अपडेट गर्ने {#update-your-sitebrief}

यदि तपाईंको साइट परिवर्तन हुन्छ भने, तपाईं यसलाई अपडेट गर्न सक्नुहुन्छ:

```
"Update my site brief: we're now targeting B2B customers"
```

वा

```
"Refresh my site specification"
```

यसले तपाईंको पहिले नै भएको जानकारीलाई आधार मानेर Site Specification skill लाई फेरि चलाउनेछ।

## एजेन्टहरूले साइट ब्रीफ कसरी प्रयोग गर्छन् {#how-agents-use-sitebrief}

### डिजाइन सिफारिसहरू {#design-recommendations}

जब तपाईं डिजाइन परिवर्तनका लागि सोध्नुहुन्छ, एजेन्टहरूले तपाईंको साइट ब्रीफको सन्दर्भ लिन्छन्:

```
तपाईं: "Redesign my homepage"
एजेन्ट: "तपाईंको साइट ब्रीफ अनुसार, तपाईं २५-४५ वर्षकी महिलालाई
गर्दै गर्नुभएको तातो, कलात्मक टोनमा लक्षित गरिरहनुभएको छ। म एउटा डिजाइन बनाउनेछु
जसले तपाईंको हस्तनिर्मित गहना र दिगो मूल्यहरूमा जोड दिनेछ।"
```

### फिचर सुझावहरू {#feature-suggestions}

एजेन्टहरूले तपाईंको लक्ष्यहरूसँग मिल्ने फिचरहरू सुझाव दिन्छन्:

```
तपाईं: "What features should I add?"
एजेन्ट: "ग्राहक वफादारीमा केन्द्रित ई-कमर्स स्टोरका लागि,
म निम्न कुराहरू सिफारिस गर्छु: वफादारी कार्यक्रम (loyalty program), ग्राहक समीक्षा (customer reviews), इमेल न्यूजलेटर, र उत्पादन सिफारिसहरू।"
```

### सामग्री संरचना {#content-structure}

एजेन्टहरूले तपाईंको संरचनाको आधारमा सामग्रीको व्यवस्थापन प्रस्ताव गर्छन्:

तपाईं: "म मेरो ब्लग कसरी व्यवस्थित गर्ने?"
एजेन्ट: "तपाईंले प्रोडक्ट प्या catalog प्रयोग गर्नुहुन्छ, त्यसैले म तपाईंलाई पोस्टहरूलाई प्रोडक्ट श्रेणी अनुसार व्यवस्थित गर्न र धेरै उत्पादन एकै ठाउँमा देखाउने 'स्टाइलिङ गाइड' बनाउन सुझाव दिन्छु।"

## उत्कृष्ट अभ्यासहरू (Best Practices) {#best-practices}

### विशिष्ट हुनुहोस् (Be Specific) {#be-specific}

"सामान्य दर्शक" भन्नुभन्दा आफ्नो वास्तविक दर्शकको वर्णन गर्नुहोस्:

- ✓ "२० देखि ४५ वर्षकी महिला, टिकाऊ फ्याशनमा रुचि राख्ने"
- ✗ "सबैजना"

### नियमित रूपमा अपडेट गर्नुहोस् (Update Regularly) {#update-regularly}

तपाईंको साइट विकसित हुँदै जाँदा आफ्नो `site_brief` अपडेट गर्नुहोस्:

- जब तपाईं नयाँ दर्शकमा केन्द्रित हुनुहुन्छ
- जब तपाईं नयाँ उत्पादन लाइनहरू थप्नुहुन्छ
- जब तपाईंको ब्रान्डको पहिचान परिवर्तन हुन्छ
- जब तपाईंको व्यवसायिक लक्ष्यहरू बदलिन्छ

### सुसंगत शब्दावली प्रयोग गर्नुहोस् (Use Consistent Terminology) {#use-consistent-terminology}

हरेक सत्रमा एउटै शब्दहरू प्रयोग गर्नुहोस्:

- ✓ सधैं "टिकाऊ गहना" भन्नुहोस् (न "इकोट-फ्रेन्डली गहना" र "हरियो उत्पादन")
- ✓ आफ्नो दर्शकलाई लगातार एउटै तरिकाले सम्बोधन गर्नुहोस्

### सन्दर्भ समावेश गर्नुहोस् (Include Context) {#include-context}

एजेन्टलाई तपाईंको निर्णयहरू बुझ्न मद्दत गर्ने पृष्ठभूमि प्रदान गर्नुहोस्:

- "हामी गुणस्तरलाई मूल्यभन्दा बढी महत्त्व दिने पेशेवरहरूको लक्षित समूहमा केन्द्रित छौं"
- "हाम्रो दर्शक प्रविधिको जानकार छन् र आधुनिक डिजाइनको अपेक्षा गर्छन्"
- "हामी एक स्टार्टअप भएकाले, हामीलाई लागत-प्रभावी समाधानहरू चाहिन्छ"

## थीम बिल्डर ओन्बोर्डिङसँग सम्बन्ध (Relationship to Theme Builder Onboarding) {#relationship-to-theme-builder-onboarding}

Site Specification स्किल **Theme Builder onboarding flow** मा एकीकृत गरिएको छ। तपाईंले ओन्बोर्डिङ पूरा गर्दा, तपाईंले प्रदान गर्नुभएको जानकारी सहितको `site_brief` स्वचालित रूपमा सिर्जना हुन्छ।

यदि तपाईं निम्न कुराहरू गर्न चाहनुहुन्छ भने पनि तपाईं Site Specification लाई स्वतन्त्र रूपमा चलाउन सक्नुहुन्छ:

- प्रारम्भिक सेटअप पछि आफ्नो स्पेसिफिकेसन परिष्कृत गर्न
- तपाईंको साइट विकसित हुँदै जाँदा आफ्नो साइट ब्रीफ अपडेट गर्न
- Theme Builder सुरु गर्नुअघि विस्तृत स्पेसिफिकेसन बनाउन

## समस्या निवारण (Troubleshooting) {#troubleshooting}

**मेरो site_brief प्रयोग भइरहेको छैन**
- एजेन्टसँग मेमोरीमा पहुँच छ कि छैन भनेर पुष्टि गर्नुहोस्
- एजेन्टलाई "मेरो साइट ब्रीफ सम्झनुहोस्" (recall my site brief) भन्न अनुरोध गर्नुहोस्
- तपाईंको सेटिङहरूमा मेमोरी सक्रिय छ वा छैन भनेर जाँच गर्नुहोस्

**म नया साइट_ब्रीफबाट सुरु गर्न म चाहन्छु**
- एजेन्टलाई सोध्नुहोस्: "मेरो साइट ब्रीफ सफा गर्नुहोस् र नयाँ सुरु गर्नुहोस्"
- त्यसपछि साइट स्पेसिफिकेशन फेरि चलाउनुहोस्

**एजेन्टले मेरो साइट ब्रीफसँग मेल नखाने सुझावहरू दिइरहेको छ**
- एजेन्टलाई सोध्नुहोस्: "मेरो साइट ब्रीफ समीक्षा गर्नुहोस्"
- यदि तपाईंको साइट ब्रीफ पुरानो छ भने त्यसलाई अपडेट गर्नुहोस्
- आफ्नो अनुरोधमा थप सन्दर्भ प्रदान गर्नुहोस्
