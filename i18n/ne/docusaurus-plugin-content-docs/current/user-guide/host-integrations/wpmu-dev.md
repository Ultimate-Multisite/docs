# WPMU DEV एकीकरण (Integration) {#wpmu-dev-integration}

## अवलोकन (Overview) {#overview}
WPMU DEV एक व्यापक वर्डप्रेस प्लेटफर्म हो जहाँ वर्डप्रेस साइटहरूका लागि होस्टिङ, प्लगइन र सेवाहरू उपलब्ध छन्। यो एकीकरणले Ultimate Multisite र WPMU DEV होस्टिङबीच डोमेन सिङ्क्रनाइजेसन र SSL प्रमाणपत्र व्यवस्थापनलाई स्वचालित बनाउँछ।

## सुविधाहरू (Features) {#features}
- स्वचालित डोमेन सिङ्क्रनाइजेसन
- SSL प्रमाणपत्र व्यवस्थापन
- विस्तारित SSL प्रमाणपत्र प्रमाणीकरण प्रयासहरू

## आवश्यकताहरू (Requirements) {#requirements}
यो एकीकरणले तपाईं WPMU DEV मा होस्ट गरिरहनुभएको छ वा छैन भने स्वचालित रूपमा पत्ता लगाउँछ र बिल्ट-इन API प्रयोग गर्दछ। यदि तपाईं WPMU DEV मा होस्ट गरिरहनुभएको छ भने कुनै अतिरिक्त कन्फिगरेसन आवश्यक पर्दैन।

यो एकीकरणले `WPMUDEV_HOSTING_SITE_ID` नामक कन्स्ट्यान्टको उपस्थिति जाँच गर्दछ, जुन WPMU DEV मा होस्ट गर्दा स्वचालित रूपमा परिभाषित हुन्छ।

## सेटअप निर्देशनहरू (Setup Instructions) {#setup-instructions}

### १. WPMU DEV होस्टिङ प्रमाणित गर्नुहोस् (Verify WPMU DEV Hosting) {#1-verify-wpmu-dev-hosting}

यदि तपाईं WPMU DEV मा होस्ट गरिरहनुभएको छ भने, आवश्यक कन्स्ट्यान्टहरू पहिले नै परिभाषित हुनुपर्छ। निम्न कुराहरू जाँच गर्नुहोस्:

1. `WPMUDEV_HOSTING_SITE_ID` कन्स्ट्यान्ट तपाईंको वातावरणमा परिभाषित गरिएको छ।
2. तपाईंसँग API पहुँच सहित सक्रिय WPMU DEV सदस्यता छ।

### २. एकीकरण सक्षम गर्नुहोस् (Enable the Integration) {#2-enable-the-integration}

1. आफ्नो वर्डप्रेस एडमिनमा, Ultimate Multisite > Settings मा जानुहोस्।
2. "Domain Mapping" ट्याबमा जानुहोस्।
3. तल स्क्रोल गरेर "Host Integrations" मा जानुहोस्।
4. WPMU DEV एकीकरण सक्षम गर्नुहोस्।
5. "Save Changes" मा क्लिक गर्नुहोस्।

## यो कसरी काम गर्छ (How It Works) {#how-it-works}

### डोमेन सिङ्क्रनाइजेसन (Domain Syncing) {#domain-syncing}

जब Ultimate Multisite मा कुनै डोमेन नक्सा गरिन्छ:

1. एकीकरणले आफ्नो होस्टिङ खातामा डोमेन थप्नको लागि WPMU DEV API प्रयोग गर्दछ।
2. यसले स्वचालित रूपमा डोमेनको www संस्करण पनि थप्छ।
3. डोमेन कन्फिगरेसन र SSL प्रमाणपत्र जारी गर्ने काम WPMU DEV ले गर्छ।

### SSL प्रमाणपत्र व्यवस्थापन (SSL Certificate Management) {#ssl-certificate-management}

यो इन्टिग्रेशनले SSL प्रमाणपत्र प्रमाणीकरण प्रयासहरूको संख्या बढाउन सेट गरिएको छ, किनभने SSL प्रमाणपत्र जारी गर्न र स्थापना गर्न केही समय लाग्न सक्छ। डिफल্ট रूपमा, यसले मानक ५ प्रयासको तुलनामा SSL प्रमाणपत्र प्रमाणीकरणका लागि १० सम्म प्रयास गर्नेछ।

## महत्त्वपूर्ण नोटहरू {#important-notes}

### डोमेन हटाउने (Domain Removal) {#domain-removal}

हाल, WPMU DEV API ले डोमेन हटाउने कुनै तरिका प्रदान गर्दैन। Ultimate Multisite मा डोमेन मैपिंग हटाउँदा, त्यो डोमेन तपाईंको WPMU DEV होस्टिङ खातामा रहनेछ। आवश्यक भएमा तपाईंले यसलाई WPMU DEV होस्टिङ ड्यासबोर्डबाट आफैं हटाउनुपर्नेछ।

### API प्रमाणीकरण (API Authentication) {#api-authentication}

यो इन्टिग्रेशनले WPMU DEV API key प्रयोग गर्दछ जुन तपाईंको WordPress डेटाबेसमा `wpmudev_apikey` विकल्पको रूपमा भण्डारण गरिएको हुन्छ। आफ्नो साइटलाई WPMU DEV सँग जोड्दा यो स्वचालित रूपमा सेट गरिन्छ।

## समस्या निवारण (Troubleshooting) {#troubleshooting}

### API कनेक्ट हुने समस्याहरू {#api-connection-issues}
- तपाईंको साइट WPMU DEV सँग ठीकसँग जोडिएको छ कि छैन भनेर जाँच गर्नुहोस्।
- तपाईंको WordPress डेटाबेसमा `wpmudev_apikey` विकल्प सेट गरिएको छ कि छैन भनेर जाँच गर्नुहोस्।
- तपाईंको WPMU DEV सदस्यता सक्रिय छ वा छैन भनेर सुनिश्चित गर्नुहोस्।

### SSL प्रमाणपत्र समस्याहरू {#ssl-certificate-issues}
- WPMU DEV ले SSL प्रमाणपत्र जारी गर्न केही समय लाग्न सक्छ (सामान्यतया ५-१५ मिनेट)।
- इन्टिग्रेशनले SSL प्रमाणपत्रहरूको लागि १० सम्म जाँच गर्ने गरी सेट गरिएको छ।
- धेरै प्रयास गरेपछि पनि SSL प्रमाणपत्र जारी नभएमा, WPMU DEV सपोर्टलाई सम्पर्क गर्नुहोस्।

### डोमेन थपिएको छैन (Domain Not Added) {#domain-not-added}
- कुनै त्रुटि सन्देशहरूका लागि Ultimate Multisite को लगहरू (logs) जाँच गर्नुहोस्।
- डोमेन पहिले नै WPMU DEV मा थपिएको छैन भनेर पुष्टि गर्नुहोस्।
- तपाईंले थप्दै गरेको डोमेन्सको संख्यालाई तपाईंको WPMU DEV होस्टिङ योजनाले समर्थन गर्छ वा गर्दैन भनेर सुनिश्चित गर्नुहोस्।
