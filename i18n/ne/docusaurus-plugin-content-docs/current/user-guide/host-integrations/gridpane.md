# GridPane एकीकरण

## अवलोकन
GridPane भनेको गम्भीर वर्डप्रेस प्रफेशनल्सका लागि बनाइएको एउटा विशेष वर्डप्रेस होस्टिङ कन्ट्रोल प्यानल हो। यो एकीकरणले Ultimate Multisite र GridPane बीच डोमेन सिङ्क्रनाइजेसन (automatic domain syncing) र SSL प्रमाणपत्र व्यवस्थापनलाई स्वचालित बनाउँछ।

## सुविधाहरू
- स्वचालित डोमेन सिङ्क्रनाइजेसन
- SSL प्रमाणपत्र व्यवस्थापन
- SUNRISE कन्स्ट्यान्टको स्वचालित कन्फिगरेशन

## आवश्यकताहरू
तपाईंको `wp-config.php` फाइलमा निम्न कन्स्ट्यान्टहरू परिभाषित गर्नुपर्छ:

```php
define('WU_GRIDPANE', true);
define('WU_GRIDPANE_API_KEY', 'your_api_key');
define('WU_GRIDPANE_SERVER_ID', 'your_server_id');
define('WU_GRIDPANE_APP_ID', 'your_app_id');
```

## सेटअप निर्देशनहरू

### १. आफ्नो GridPane API प्रमाणहरू प्राप्त गर्नुहोस्

१. आफ्नो GridPane ड्यासबोर्डमा लगइन गर्नुहोस्।
२. "Settings" > "API" मा जानुहोस्।
३. यदि तपाईंसँग पहिले नै API key छैन भने एक API key उत्पन्न गर्नुहोस्।
४. आफ्नो API key कपी गर्नुहोस्।

### २. आफ्नो सर्भर र साइट ID प्राप्त गर्नुहोस्

१. आफ्नो GridPane ड्यासबोर्डमा, "Servers" मा जानुहोस्।
२. जहाँ तपाईंको वर्डप्रेस मल्टीसाइट होस्ट गरिएको छ त्यो सर्भर चयन गर्नुहोस्।
३. सर्भर ID नोट गर्नुहोस् (URL मा वा सर्भर विवरण पृष्ठमा देखिने)।
४. "Sites" मा जानुहोस् र आफ्नो वर्डप्रेस साइट चयन गर्नुहोस्।
५. साइट ID नोट गर्नुहोस् (URL मा वा साइट विवरण पृष्ठमा देखिने)।

### ३. wp-config.php मा कन्स्ट्यान्टहरू थप्नुहोस्

निम्न कन्स्ट्यान्टहरू आफ्नो `wp-config.php` फाइलमा थप्नुहोस्:

```php
define('WU_GRIDPANE', true);
define('WU_GRIDPANE_API_KEY', 'your_api_key');
define('WU_GRIDPANE_SERVER_ID', 'your_server_id');
define('WU_GRIDPANE_APP_ID', 'your_site_id');
```

### ४. एकीकरण सक्षम गर्नुहोस्

१. आफ्नो वर्डप्रेस एडमिनमा, Ultimate Multisite > Settings मा जानुहोस्।
२. "Domain Mapping" ट्याबमा जानुहोस्।
३. तल स्क्रोल गरेर "Host Integrations" मा जानुहोस्।
४. GridPane एकीकरण (integration) सक्षम गर्नुहोस्।
५. "Save Changes" मा क्लिक गर्नुहोस्।

## यो कसरी काम गर्छ

जब Ultimate Multisite मा कुनै डोमेन नक्सा (map) गरिन्छ:

१. यो इन्टिग्रेशनले तपाईंको साइटमा डोमेन थप्नका लागि GridPane को API मा अनुरोध पठाउँछ।
२. GridPane ले स्वचालित रूपमा SSL प्रमाणपत्र प्रदान गर्ने काम गर्छ।
३. जब कुनै डोमेन मैपिंग हटाउन सकिन्छ, तब त्यो इन्टिग्रेशनले GridPane बाट त्यस डोमेनलाई हटाउनेछ।

यो इन्टिग्रेशनले तपाईंको wp-config.php फाइलमा आवश्यक रहेको SUNRISE constant लाई पनि स्वचालित रूपमा व्यवस्थापन गर्छ, जसले डोमेन मैपिंग सही तरिकाले काम गर्नका लागि आवश्यक छ।

## SUNRISE Constant व्यवस्थापन

GridPane इन्टिग्रेशनको एक विशेष विशेषता भनेको यो हो कि यसले wp-config.php मा भएको SUNRISE constant लाई स्वचालित रूपमा फिर्ता (revert) गर्छ, ताकि GridPane को आफ्नै डोमेन मैपिंग प्रणालीसँग कुनै समस्या नहोस्। यसले दुवै प्रणालीलाई बिना कुनै समस्याका साथ काम गर्न मद्दत गर्छ।

## समस्या समाधान (Troubleshooting)

### API कनेक्टिभिटी समस्याहरू
- तपाईंको API key सही छ कि छैन भने जाँच गर्नुहोस्।
- तपाईंको सर्भर र साइट ID सही छन् कि छैनन् भने जाँच गर्नुहोस्।
- तपाईंको GridPane खातामा आवश्यक अनुमतिहरू छन् कि छैनन् भने सुनिश्चित गर्नुहोस्।

### SSL प्रमाणपत्र समस्याहरू
- GridPane ले SSL प्रमाणपत्र जारी गर्न केही समय लिन सक्छ।
- तपाईंको डोमेनहरू तपाईंको सर्भरको IP ठेगानामा ठीकसँग इंगित (pointing) गरिएका छन् कि छैनन् भने जाँच गर्नुहोस्।
- आफ्नो साइटको लागि GridPane को SSL सेटिङहरू जाँच गर्नुहोस्।

### डोमेन थपिएको छैन
- कुनै पनि त्रुटि सन्देशहरूको लागि Ultimate Multisite लगहरू (logs) जाँच गर्नुहोस्।
- त्यो डोमेन पहिले नै GridPane मा थपिएको छैन भनेर पुष्टि गर्नुहोस्।
- तपाईंको डोमेनका DNS रेकर्डहरू सही रूपमा कन्फिगर गरिएका छन् कि छैनन् भने सुनिश्चित गर्नुहोस्।
