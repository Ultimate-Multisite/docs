# Closte Integration

## अवलोकन {#overview}
Closte भनेको Google Cloud इन्फ्रास्ट्रक्चरमा बनेको एउटा व्यवस्थापन गरिएको वर्डप्रेस होस्टिङ प्लेटफर्म हो। यो इन्टिग्रेशनले Ultimate Multisite र Closte बीच स्वचालित डोमेन सिङ्क्रनाइजेसन र SSL प्रमाणपत्र व्यवस्थापन सक्षम गर्दछ।

## सुविधाहरू {#features}
- स्वचालित डोमेन सिङ्क्रनाइजेसन
- SSL प्रमाणपत्र व्यवस्थापन
- वाइल्डकार्ड डोमेन समर्थन
- Closte मा चलाउँदा कुनै कन्फिगरेसनको आवश्यकता पर्दैन

## आवश्यकताहरू {#requirements}
यदि तपाईं Closte प्रयोग गर्दै हुनुहुन्छ भने, तपाईंको `wp-config.php` फाइलमा निम्न constant परिभाषित गर्नुपर्छ:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Closte मा होस्ट गर्दा यो constant सामान्यतया पहिले नै परिभाषित गरिएको हुन्छ।

## सेटअप निर्देशनहरू {#setup-instructions}

### १. आफ्नो Closte API Key प्रमाणित गर्नुहोस् {#1-verify-your-closte-api-key}

यदि तपाईं Closte मा होस्ट गर्दै हुनुहुन्छ भने, `CLOSTE_CLIENT_API_KEY` constant तपाईंको `wp-config.php` फाइलमा पहिले नै परिभाषित हुनुपर्छ। यसलाई जाँच गर्न तपाईंले आफ्नो `wp-config.php` फाइल हेर्न सक्नुहुन्छ।

### २. इन्टिग्रेशन सक्षम गर्नुहोस् {#2-enable-the-integration}

१. आफ्नो वर्डप्रेस एडमिनमा, Ultimate Multisite > Settings मा जानुहोस्
२. "Domain Mapping" ट्याबमा जानुहोस्
३. तल स्क्रोल गरेर "Host Integrations" मा जानुहोस्
४. Closte इन्टिग्रेशन सक्षम गर्नुहोस्
५. "Save Changes" मा क्लिक गर्नुहोस्

## यो कसरी काम गर्छ {#how-it-works}

Ultimate Multisite मा डोमेन नक्सा (map) गरिएपछि:

१. इन्टिग्रेशनले आफ्नो एपमा डोमेन थप्नको लागि Closte को API मा अनुरोध पठाउँछ
२. Closte ले SSL प्रमाणपत्र प्रदान गर्ने कार्य स्वचालित रूपमा गर्छ
३. जब कुनै डोमेन नक्सा हटाउन सकिन्छ, तब इन्टिग्रेशनले Closte बाट त्यो डोमेन हटाउँछ

इन्टिग्रेशनले Ultimate Multisite मा रहेको DNS चेक इन्टर्भल सेटिङसँग पनि काम गर्दछ, जसले तपाईंलाई प्रणालीले DNS प्रसार (propagation) र SSL प्रमाणपत्र जारी गर्ने बारे कति पटक जाँच गर्छ भनेर कन्फिगर गर्न अनुमति दिन्छ।

## डोमेन रेकर्ड निर्माण {#domain-record-creation}

यो इन्टिग्रेसनले कुनै साइट बनाउँदा वा डुप्लिकेट गर्दा डोमेन रेकर्ड स्वचालित रूपमा सिर्जना हुन्छ भन्ने कुरा सुनिश्चित गर्छ। यो विशेष गरी Closte इन्टिग्रेसनका लागि महत्त्वपूर्ण छ, किनभने डोमेन रेकर्ड सिर्जना गर्दा Closte API लाई डोमेन र SSL प्रमाणपत्र बनाउन ट्रिगर गरिन्छ।

## समस्या समाधान (Troubleshooting) {#troubleshooting}

### API कनेक्ट हुने समस्याहरू {#api-connection-issues}
- तपाईंको Closte API key सही छ कि छैन भनेर जाँच गर्नुहोस्।
- सुनिश्चित गर्नुहोस् कि तपाईंको Closte खातामा आवश्यक अनुमतिहरू छन्।

### SSL प्रमाणपत्रका समस्याहरू {#ssl-certificate-issues}
- Closte ले SSL प्रमाणपत्र जारी गर्न केही समय लिन सक्छ (सामान्यतया ५-१० मिनेट)।
- तपाईंको डोमेनहरू तपाईंको Closte सर्भरको IP ठेगानामा सही तरिकाले इंगित गरिएका छन् कि छैनन् भनेर जाँच गर्नुहोस्।
- तपाईंको डोमेनका DNS रेकर्डहरू सही रूपमा कन्फिगर गरिएका छन् कि छैनन् भनेर जाँच गर्न DNS रेकर्डहरू हेर्नुहोस्।

### डोमेन थपिएको छैन {#domain-not-added}
- कुनै त्रुटि सन्देशहरू (error messages) का लागि Ultimate Multisite लगहरू (logs) जाँच गर्नुहोस्।
- डोमेन पहिले नै Closte मा थपिएको छैन भनेर पुष्टि गर्नुहोस्।
- तपाईंको डोमेनका DNS रेकर्डहरू सही रूपमा कन्फिगर गरिएका छन् कि छैनन् भनेर सुनिश्चित गर्नुहोस्।

### DNS जाँच गर्ने अन्तराल (DNS Check Interval) {#dns-check-interval}
- यदि SSL प्रमाणपत्र जारी गर्न धेरै समय लाग्छ भने, तपाईंले डोमेन मेप्यापिङ सेटिङहरू (Domain Mapping settings) मा DNS जाँच गर्ने अन्तराल समायोजन गर्न सक्नुहुन्छ।
- डिफल্ট अन्तराल ३०० सेकेन्ड (५ मिनेट) हुन्छ, तर परीक्षणको क्रममा छिटो जाँच गर्न यसलाई १० सेकेन्डसम्म पनि सेट गर्न सकिन्छ।
