# अन्य प्लगइन्ससँग ज्ञात असंगतताहरू {#known-incompatibilities-with-other-plugins}

Ultimate Multisite इन्स्टल गरेपछि र सक्रिय (activate) गरेपछि, अन्य प्लगइनहरूसँग समाधान नगरेका असंगतताका कारण तपाईं समस्याहरू सामना गर्न सक्नुहुन्छ।

हामी हरेक नयाँ रिलीजको साथ यस सूचीलाई कम गर्न सक्रिय रूपमा काम गरिरहेका छौं, तर यदि कुनै प्लगइन यहाँ सूचीबद्ध छ भने, पूर्ण रूपमा कार्यशील Ultimate Multisite नेटवर्क राख्न तपाईंले त्यसलाई निष्क्रिय (deactivate) गर्नुपर्ने हुन सक्छ।

यहाँ तिनीहरूको सूची र यस्तो अवस्थामा तपाईंले के गर्नुपर्छ भन्ने बारे जानकारी दिइएको छ:

**केही WPMU DEV प्लगइनहरू** केही प्लगइनहरू जुन अब WPMU DEV द्वारा मर्मत गरिएका छैनन्, ती Ultimate Multisite दर्ता प्रक्रियाको अन्तिम चरणमा (प्रयोगकर्ताले खाता बनाउन सक्दैन) अवरोध ल्याउँछन् भन्ने कुरा ज्ञात छ। यी समस्याहरू निम्त्याउने प्लगइनहरूको सूची यहाँ दिइएको छ:

  * Signup Language: _Ultimate Multisite मा यसको एउटा add-on छ जसले समान कार्यक्षमता प्रदान गर्दछ, जसलाई Ultimate Multisite: Language Switcher भनिन्छ._

  * Set Blog Description on Blog Creation: _यो Ultimate Multisite मा बिल्ट-इन (built-in) गरिएको छ, त्यसैले अतिरिक्त प्लगइनको आवश्यकता छैन._

  * New Blog Templates: _यो Ultimate Multisite मा बिल्ट-इन गरिएको छ, त्यसैले अतिरिक्त प्लगइनको आवश्यकता छैन._

  * Set Password on Multisite Blog Creation: _यो Ultimate Multisite मा बिल्ट-इन गरिएको छ, त्यसैले अतिरिक्त प्लगइनको आवश्यकता छैन._

  * Signup TOS: _यो Ultimate Multisite मा बिल्ट-इन गरिएको छ, त्यसैले अतिरिक्त प्लगइनको आवश्यकता छैन._

  * Anti-Splog: यद्यपि यो कम शक्तिशाली छ, यो पनि _Ultimate Multisite मा बिल्ट-इन गरिएको छ._

**नोट** : अन्य सबै WPMU Dev प्लगइनहरूलाई Ultimate Multisite सँग सामान्य रूपमा प्रयोग गर्न सकिन्छ। उदाहरणहरूमा _Smush_ , _Forminator_ , _Defender,_ आदि समावेश छन्।
