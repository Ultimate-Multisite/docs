# निःशुल्क AI एजेन्ट सेटिङहरू

Gratis AI Agent मा **Settings → Advanced** स्क्रिनले v1.5.0 मा ल्याइएका ब्याकएन्ड इन्टिग्रेशन्सको लागि एडमिन-स्तरको कन्फिगरेसन प्रदान गर्दछ। यो पेजमा **Feedback Endpoint** फिल्ड र त्यसको अपेक्षित ढाँचाको बारेमा जानकारी दिइएको छ।

## सेटिङहरू कसरी खोल्ने

१. वर्डप्रेस एडमिनमा, **Gratis AI Agent → Settings** मा जानुहोस्।
२. **Advanced** ट्याबमा क्लिक गर्नुहोस्।

## Feedback Endpoint कन्फिगरेसन

AI एजेन्टले कुनै पनि प्रयोगकर्ताले थम्ब्स-डाउन बटन मार्फत, ऑटो-प्रम्प्ट ब्यानरमार्फत, वा `/report-issue` कमन्ड मार्फत प्रतिक्रिया (feedback) पठाउँदा feedback endpoint ले POST अनुरोधहरू प्राप्त गर्दछ।

| फिल्ड | विवरण |
|---|---|
| **Feedback Endpoint URL** | यो त्यो URL हो जसले HTTP POST अनुरोधहरूसँग JSON बडीको रूपमा प्रतिक्रियाहरू प्राप्त गर्दछ। |
| **Feedback API Key** | प्रत्येक प्रतिक्रिया अनुरोधको `Authorization` हेडरमा पठाइने bearer token। यदि तपाईंको endpoint ले प्रमाणीकरण (authentication) आवश्यक नगरेको भए खाली छोड्नुहोस्। |

### अपेक्षित JSON पेलोड

तपाईंको feedback endpoint ले कम्तिमा निम्न फिल्डहरू भएको JSON बडी स्वीकार गर्नुपर्छ:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

संवादको सन्दर्भ अनुसार पेलोडमा थप फिल्डहरू हुन सक्छन्।

### `triage_category` मानहरू

AI ट्राइएज लेयरले पेलोड अग्रेषण गर्नुअघि `triage_category` मा निम्न मध्ये एक मान तोक्छ:

| मान | अर्थ |
|---|---|
| `factual_error` | सहायकले गलत तथ्याङ्कीय जानकारी प्रदान गरेको छ। |
| `unhelpful_answer` | जवाफ प्राविधिक रूपमा सही थियो तर उपयोगी थिएन। |
| `inappropriate_content` | जवाफमा प्रयोगकर्ताहरूलाई देखाउनु हुँदैन भन्ने सामग्री समावेश थियो। |
| `other` | प्रतिक्रियाले कुनै ज्ञात श्रेणीसँग मेल खाएको छैन। |

### प्रमाणीकरण (Authentication)

यदि तपाईंको एन्डपॉइन्टले प्रमाणीकरण (authentication) माग्छ भने, **Feedback API Key** फिल्डमा आफ्नो बेयरर टोकन सेट गर्नुहोस्। एजेन्टले निम्न कुरा पठाउँछ:

```
Authorization: Bearer <your-api-key>
```

यदि **Feedback API Key** फिल्ड खाली छ भने, कुनै पनि `Authorization` हेडर पठाइँदैन।

### Feedback सङ्कलन बन्द गर्ने (Disabling Feedback Collection)

दुवै **Feedback Endpoint URL** र **Feedback API Key** फिल्डहरू खाली छोड्नुहोस्। थम्ब्स-डाउन बटन र प्रतिक्रिया UI प्रयोगकर्ताहरूका लागि देखिने रहन्छ, तर कुनै पनि बाह्य सेवामा प्रस्तुतीकरण पठाइँदैन।

## Brave Search API Key

त्यसैगरी **Advanced** ट्याबमा, **Brave Search API Key** फिल्डले [Internet Search](../configuration/internet-search) क्षमतालाई सक्रिय गर्दछ।

| फिल्ड | विवरण |
|---|---|
| **Brave Search API Key** | Brave Search डेभेलपर ड्यासबोर्डबाट तपाईंको एपीआई की (API key)। एआई सहायकमा इन्टरनेट खोज सक्षम गर्न आवश्यक छ। |

फिल्ड लेबलमा Brave Search API साइन-अप पेजको क्लिक गर्न सकिने लिंक समावेश गरिएको हुन्छ। इन्टरनेट खोज बन्द गर्न खाली छोड्नुहोस्।

यस फिचरबारे अन्त्यकर्ताका लागि [Internet Search](../configuration/internet-search) हेर्नुहोस्।

## Feature Flags (फिचर फ्ल्यागहरू)

v1.9.0 मा पनि परिचय दिइएको, **Settings → Feature Flags** ट्याबले वैकल्पिक कार्यक्षमताका लागि टोगल स्विच प्रदान गर्दछ। प्रत्येक फ्ल्याग नेटवर्कभर सक्रिय वा बन्द हुन्छ; अहिले कुनै साइट-विशिष्ट ओभरराइड (override) छैन।

### Feature Flags मा पहुँच गर्ने तरिका

१. वर्डप्रेस एडमिनमा, **Gratis AI Agent → Settings** मा जानुहोस्।
२. **Feature Flags** ट्याबमा क्लिक गर्नुहोस्।

### Access Control Flags

| Flag | Default | विवरण |
|---|---|---|
| **प्रशासकहरूमा मात्र सीमित गर्नुहोस् (Restrict to Administrators)** | Off | यो अनलाइन हुँदा, `administrator` भूमिका भएका प्रयोगकर्ताहरूले मात्र AI Agent च्याट प्यानल खोल्न सक्छन्। अन्य सबै भूमिकाका प्रयोगकर्ताहरूलाई "तपाईंको प्रशासकसँग सम्पर्क गर्नुहोस्" भन्ने सन्देश देखिन्छ। |
| **नेटवर्क एडमिनमा सीमित गर्नुहोस् (Restrict to Network Admins)** | Off | मल्टिसाइट नेटवर्कमा अनलाइन हुँदा, केवल सुपर एडमिनहरूले मात्र एजेन्ट प्रयोग गर्न सक्छन्। व्यक्तिगत साइट एडमिनहरूलाई अवरुद्ध गरिन्छ। यदि दुवै सक्रिय छन् भने "प्रशासकहरूमा मात्र सीमित गर्नुहोस्" भन्दा यो प्राथमिकतामा हुन्छ। |
| **सबस्क्राइबर पहुँच अनुमति दिनुहोस् (Allow Subscriber Access)** | Off | यो अनलाइन हुँदा, `subscriber` भूमिका भएका प्रयोगकर्ताहरूले च्याट इन्टरफेस प्रयोग गर्न सक्छन् तर उनीहरू केवल पढ्न मात्र सक्षम हुन्छन् (पोस्ट बनाउने वा सेटिङ परिवर्तन गर्ने अधिकार छैन)। |
| **गैर-सदस्यहरूको लागि अक्षम गर्नुहोस् (Disable for Non-Members)** | Off | यो Ultimate Multisite सदस्यता स्थितिमा एकीकृत हुन्छ। यो अनलाइन हुँदा, सक्रिय सदस्यता नभएका साइटहरूका लागि च्याट लुकाइन्छ। |

### ब्रान्डिङ फ्ल्यागहरू (Branding Flags)

| Flag | Default | विवरण |
|---|---|---|
| **"Gratis AI Agent" फुटर लुकाउने (Hide "Powered by Gratis AI Agent" Footer)** | Off | च्याट विजेटको तल देखिने ब्रान्डिङ जानकारी लाइन हटाउँछ। व्हाइट-लेबल डिप्लॉयमेन्टहरूको लागि सिफारिस गरिन्छ। |
| **कस्टम एजेन्ट नाम (Custom Agent Name)** | *(खाली)* | च्याट हेडर र एडमिन मेनुमा डिफल्ट "Gratis AI Agent" लेबलको सट्टा आफ्नो उत्पादनको नाम राख्न प्रयोग गर्नुहोस्। डिफल्ट प्रयोग गर्न खाली छोड्नुहोस्। |
| **एजेन्ट पिकर लुकाउने (Hide Agent Picker)** | Off | यो अनलाइन हुँदा, प्रयोगकर्ताहरूले पाँचवटा बिल्ट-इन एजेन्टहरू बीच स्विच गर्न सक्दैनन्। वर्तमान एजेन्ट सेटिङहरू → सामान्य मा कन्फिगर गरिएको डिफल्ट जस्तै राखिएको हुन्छ। |
| **साइट आइकनलाई च्याट अवतारको रूपमा प्रयोग गर्ने (Use Site Icon as Chat Avatar)** | Off | च्याट विजेट हेडरमा डिफल्ट AI आइकनको सट्टा वर्डप्रेस साइट आइकन देखाउँछ (Appearance → Customize → Site Identity मा सेट गरिएको)। |

### परिवर्तनहरू लागू गर्ने तरिका (Applying Changes)

कुनै पनि फ्ल्याग अनलाइन toggling पछि **Save Settings** मा क्लिक गर्नुहोस्। परिवर्तन तुरुन्तै लागू हुन्छ — कुनै क्याच फ्लश वा प्लगइन पुनः सक्रिय पार्नु आवश्यक छैन।
