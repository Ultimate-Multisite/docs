# डोमेन विक्रेता: सेटअप र प्रदायक कन्फिगरेशन

डोमेन सेलर addon मा एउटा गाइड गरिएको सेटअप विजार्ड (wizard) समावेश छ जसले तपाईंलाई आवश्यक हरेक चरणमा मार्गदर्शन गर्छ। यो पेजमा हामी त्यो विजार्डको प्रवाह र त्यसपछि प्रदायकहरू कसरी कन्फिगर वा पुनः कन्फिगर गर्ने भन्ने बारे जानकारी दिइएको छ।

## आवश्यकताहरू

- **Multisite Ultimate** v2.4.12 वा सोभन्दा नयाँ, नेटवर्क सक्रिय गरिएको
- **PHP** 7.4+
- कम्तिमा एउटा समर्थित रजिस्ट्रारका लागि API क्रेडेंशियल्स

## पहिलो पटक सेटअप विजार्ड

जब तपाईंले प्लगइनलाई नेटवर्क सक्रिय गर्नुहुन्छ, तब यो सेटअप विजार्ड स्वचालित रूपमा सुरु हुन्छ। यसको पहुँच कुनै पनि समय **Network Admin › Ultimate Multisite › Domain Seller Setup** बाट पनि गर्न सकिन्छ।

### चरण १ — प्रदायक छान्नुहोस्

तपाईं कुन रजिस्ट्रारसँग जोड्न चाहनुहुन्छ, त्यो चयन गर्नुहोस्। समर्थित विकल्पहरू:

| प्रदायक | DNS व्यवस्थापन | WHOIS गोपनीयता |
|---|---|---|
| OpenSRS | हो | हो |
| Namecheap | होइन | हो (WhoisGuard, निःशुल्क) |
| HostAfrica | हो | हो (ID सुरक्षा) |
| Openprovider | हो | हो |
| Hostinger | होस्ट गरिएका डोमेनहरूको लागि कोर Hostinger डोमेन मैपिंग मार्फत | हो |
| GoDaddy | होइन | होइन |
| ResellerClub | हो | होइन |
| NameSilo | होइन | होइन |
| Enom | हो | होइन |

### चरण २ — क्रेडेंशियल्स प्रविष्ट गर्नुहोस्

प्रत्येक प्रदायकका आफ्नै फरक क्रेडेंशियल फिल्डहरू छन्:

**OpenSRS** — प्रयोगकर्ता नाम र प्राइभेट की (OpenSRS Reseller Control Panel बाट)

**Namecheap** — प्रयोगकर्ता नाम र API की (Account › Tools › API Access बाट)

**HostAfrica** — HostAfrica रिसेलर मोडेलबाट डोमेन रिसेलर API एन्डपॉइन्ट र क्रेडेंशियल्स। अहिले छुट्टै सैंडबॉक्स एन्डपॉइन्टको दस्तावेज छैन; लाइभ दर्ता चलाउनुअघि सुरक्षित पढ्न मात्र प्रयोग हुने जाँच गर्नुहोस्।

**Openprovider** — API पहुँच सक्षम गरिएको प्रयोगकर्ता नाम र पासवर्ड। वैकल्पिक सैंडबॉक्स मोडले Openprovider सैंडबॉक्स API प्रयोग गर्दछ, र दर्ताका लागि वैकल्पिक डिफाल्ट ग्राहक ह्यान्डल पुनः प्रयोग गर्न सकिन्छ।

**Hostinger** — मुख्य Hostinger इन्टिग्रेशनबाट साझा गरिएको hPanel API टोकन। यो नै कोर डोमेन मैपिंग र डोमेन सेलर दर्ताका कार्यहरू चलाउन प्रयोग हुने टोकन हो।

**GoDaddy** — API key र secret (developer.godaddy.com बाट)

**ResellerClub** — Reseller ID र API key (ResellerClub कन्ट्रोल प्यानलबाट)

**NameSilo** — API key (namesilo.com › Account › API Manager बाट)

**Enom** — Account ID र API token

लाइभ सुरु गर्नुअघि प्रदायकको टेस्ट वातावरणसँग परीक्षण गर्नका लागि उपलब्ध भएमा **Sandbox mode** जाँच गर्नुहोस्।

### चरण ३ — कनेक्शन परीक्षण गर्नुहोस्

**Test Connection** मा क्लिक गर्नुहोस्। यो विजार्डले प्रमाणहरू र कनेक्टिविटी प्रमाणित गर्न हल्का API कल पठाउँछ। जारी राख्नुअघि कुनै पनि प्रमाण (credentials) समस्याहरू सुधार्नुहोस्।

### चरण ४ — TLDs आयात गर्नुहोस्

कनेक्ट गरिएको प्रदायकबाट सबै उपलब्ध TLDs र होलसेल मूल्यहरू ल्याउन **Import TLDs** मा क्लिक गर्नुहोस्। यसले डोमेन उत्पादनहरूमा प्रयोग हुने TLD सूची भर्छ। ठूला TLD सूची भएका प्रदायकहरूको लागि आयात गर्न ३०–६० सेकेन्ड लाग्न सक्छ।

TLDs स्वचालित रूपमा दैनिक एक पटक scheduled cron job मार्फत पुनः सिङ्क (re-sync) पनि हुन्छ।

### चरण ५ — डोमेन उत्पादन सिर्जना गर्नुहोस्

विजार्डले १०% मार्कअप सहित डिफल्ट क्याच-अल सबै डोमेन उत्पादन सिर्जना गर्छ। तपाईं यस उत्पादनलाई तुरुन्तै सम्पादन गर्न सक्नुहुन्छ वा **Ultimate Multisite › Products** अन्तर्गत म्यानुअल रूपमा उत्पादनहरू सिर्जना गर्न छाडेर स्किप गर्न सक्नुहुन्छ।

पूर्ण उत्पादन कन्फिगरेसन गाइडको लागि [Domain Products and Pricing](./domain-products) हेर्नुहोस्।

## प्रदायकलाई पुनः कन्फिगर गर्ने (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** मा जानुहोस् (वा प्लगइन सूचीमा **Settings** मा क्लिक गर्नुहोस्)।

सेटिंगहरूमा निम्न कुराहरू समावेश हुन्छन्:

- **डोमेन बेच्न सक्षम गर्नुहोस्** — यो सम्पूर्ण फिचरलाई अन/अफ गर्न टोगल गर्नुहोस्।
- **डिफल्ट प्रदायक** — डोमेन खोज र नयाँ उत्पादनहरूको लागि प्रयोग गरिने प्रदायक।
- **खोजमा अधिकतम TLDs** — ग्राहकले खोज गर्दा कति TLDs जाँच गर्ने; उच्च मानहरूले धेरै विकल्प देखाउँछन् तर ढिलो हुन्छन्।
- **उपलब्धता क्याश अवधि** — उपलब्धता र मूल्य निर्धारण परिणामलाई कति समयसम्म क्याश गर्ने; कम मानहरू बढी सटीक हुन्छन् तर API कलहरू बढ्छन्।
- **डोमेन उत्पादन व्यवस्थापन गर्नुहोस्** — उत्पादनहरूको सूचीमा द्रुत लिंक।
- **प्रदायकहरू कन्फिगर गर्नुहोस्** — प्रदायकहरू थप्न वा पुनः कन्फिगर गर्न इन्टिग्रेशन विजार्ड खोल्नुहोस्।

### दोस्रो प्रदायक थप्ने

**Configure providers** मा क्लिक गर्नुहोस् र नयाँ रजिस्ट्रारको लागि विजार्ड फेरि चलाउनुहोस्। तपाईं एकैसाथ धेरै प्रदायकहरू कन्फिगर गर्न सक्नुहुन्छ। प्रत्येक डोमेन उत्पादनलाई एउटा विशिष्ट प्रदायकसँग जोड्नुहोस्, वा डिफल्टमा छोडिदिनुहोस्।

### TLDs लाई म्यानुअल रूपमा सिंक गर्ने

सेटिंग्स पेजमा, नवीनतम मूल्य निर्धारण ल्याउन कुनै पनि कन्फिगर गरिएको प्रदायकको छेउमा **Sync TLDs** मा क्लिक गर्नुहोस्। यो कुनै प्रदायकले थोक मूल्य अद्यावधिक गर्दा वा नयाँ TLDs थप्दा उपयोगी हुन्छ।

## लगहरू (Logs)

प्रत्येक प्रदायकले आफ्नो छुट्टै लग च्यानलमा लेख्छ। लगहरू **Network Admin › Ultimate Multisite › Logs** अन्तर्गत हेर्न सकिन्छ:

| Log channel | Contents |

| `domain-seller-opensrs` | कच्चा OpenSRS API गतिविधि |
| `domain-seller-namecheap` | कच्चा Namecheap API गतिविधि |

| `domain-seller-hostinger` | कच्चा Hostinger API गतिविधि |
| `domain-seller-godaddy` | कच्चा GoDaddy API गतिविधि |

## प्रदायक क्षमता नोटहरू (Provider capability notes)

हरेक रजिस्ट्रारको API मा एउटै कामहरू उपलब्ध हुँदैनन्। Addon ले समर्थन नगरिएका कामहरूलाई चुपचाप असफल हुने सट्टा स्पष्ट एडमिन-फेस गर्ने त्रुटिहरू देखाउँछ।

- **HostAfrica** सबैभन्दा व्यापक लाइभ रिसेलर वर्कफ्लोलाई समर्थन गर्दछ, जसमा लुकअप (lookup), TLD/मूल्य निर्धारण सिंक, दर्ता, नवीकरण, स्थानान्तरण, नेमसर्भर अपडेट, DNS रेकर्डहरू, EPP कोडहरू, रजिस्ट्रार लक र ID सुरक्षा समावेश छन्।
- **Openprovider** रिसेलर मूल्यमा TLD सिंक, दर्ता, नवीकरण, स्थानान्तरण, नेमसर्भर अपडेट, DNS जोनहरू, EPP कोडहरू, रजिस्ट्रार लक र WHOIS गोपनीयतालाई समर्थन गर्दछ। यसले छोटो समयसम्म चल्ने bearer token प्रयोग गरेर प्रमाणित गर्छ जुन addon ले स्वचालित रूपमा रिफ्रेश गर्छ।
- **Hostinger** साझा hPanel API टोकन मार्फत उपलब्धता खोज (availability search), दर्ता, पोर्टफोलियो लुकअप, नेमसर्भर अपडेट, रजिस्ट्रार लक र WHOIS गोपनीयतालाई समर्थन गर्दछ। Hostinger को सार्वजनिक Domains API ले रिसेलर/ wholesale मूल्य निर्धारण, इनबाउंड स्थानान्तरण, स्पष्ट नवीकरण वा EPP-कोड प्राप्ति प्रदर्शन गर्दैन; नवीकरणहरू स्वचालित रूपमा मात्र नवीकरण हुन्छन्।
