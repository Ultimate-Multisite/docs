# चेकआउट फिल्ड र ग्राहक DNS व्यवस्थापन {#checkout-field-and-customer-dns-management}

## डोमेन चयन चेकआउट फिल्ड {#the-domain-selection-checkout-field}

**डोमेन चयन** फिल्ड एक चेकआउट तत्व हो जसले ग्राहकलाई आफ्नो साइटको डोमेन कसरी प्राप्त गर्ने भन्ने विकल्प दिन्छ। डोमेन बिक्री सक्षम गर्न यसलाई कुनै पनि चेकआउट फारममा थप्नुहोस्।

### चेकआउट फारममा फिल्ड थप्ने {#adding-the-field-to-a-checkout-form}

१. **Network Admin › Ultimate Multisite › Checkout Forms** मा जानुहोस्
२. चेकआउट फारम खोल्नुहोस् वा बनाउनुहोस्
३. चेकआउट एडिटरमा, **Add Field** मा क्लिक गर्नुहोस्
४. फिल्ड सूचीबाट **Domain Selection** छान्नुहोस्
५. फिल्ड विकल्पहरू कन्फिगर गर्नुहोस् (तल हेर्नुहोस्)
६. फारम सेभ गर्नुहोस्

### फिल्ड विकल्पहरू {#field-options}

**डोमेन मोडहरू** — ग्राहकले कुन ट्याब देख्छ भन्ने छान्नुहोस्। प्रत्येक मोडलाई स्वतन्त्र रूपमा सक्षम वा अक्षम गर्न सकिन्छ:

| मोड | यसले के गर्छ |
|---|---|
| **Subdomain** | ग्राहकले आफ्नो नेटवर्कमा निःशुल्क सबडोमेन प्रयोग गर्छ (जस्तै: `mysite.yournetwork.com`)। कुनै भुक्तानी आवश्यक छैन। |
| **Register New Domain** | ग्राहकले नयाँ डोमेन खोज्छ र तपाईंको कन्फिगर गरिएको प्रदायक मार्फत दर्ता गर्छ। मूल्य निर्धारणका लागि मेल खाने डोमेन उत्पादन प्रयोग गरिन्छ। |
| **Existing Domain** | ग्राहकले पहिले नै स्वामित्वमा रहेको डोमेनलाई नक्सा (map) गर्छ। कुनै दर्ता शुल्क छैन। डोमेन स्वचालित रूपमा उनीहरूको साइटसँग जोडिएको हुन्छ। |

**डिफल्ट मोड** — जब सबै तीन मोडहरू सक्षम हुन्छन्, कुन ट्याब पहिले खुल्छ? यदि तपाईंले डोमेन दर्ता वैकल्पिक राख्न **Subdomain** मा सेट गर्नुभयो भने, वा खरिदलाई प्रोत्साहन दिन **Register New Domain** मा सेट गर्नुभयो भने।

**डोमेन उत्पादन** — यो फिल्डलाई कुनै विशिष्ट डोमेन उत्पादनसँग जोड्न (pin) छनौट गर्न सकिन्छ। यदि सेट गरिएको छैन भने, एड-अन स्वचालित रूपमा ग्राहकले खोज्ने TLD को आधारमा मेल खाने उत्पादन चयन गर्छ।

### दर्ताकर्ता सम्पर्क फिल्डहरू {#registrant-contact-fields}

जब ग्राहक **Register New Domain** ट्याब चयन गर्छन्, चेकआउट फारममा दर्ताकर्ताको सम्पर्क विवरणहरू इन्लाइन (inline) मा थपिन्छन्:

- पहिलो नाम / अन्तिम नाम
- इमेल ठेगाना
- ठेगाना (लाइन १, शहर, राज्य/प्रांत, पुष्टिकोड, देश)
- फोन नम्बर

यी सबै रजिस्ट्रारहरूद्वारा आवश्यक हुन्छन् र दर्ता API कल गर्नुअघि प्रमाणित गरिन्छ। फोन नम्बरहरू स्वचालित रूपमा रजिस्ट्रारहरूले अपेक्षा गर्ने `+CC.NNN` अन्तर्राष्ट्रिय ढाँचामा स्वरूपित गरिएका हुन्छन्।

### स्वतः सिर्जित साइट URL (Auto-generated site URL) {#auto-generated-site-url}

जब कुनै ग्राहक दर्ता गर्छ वा डोमेन नक्सा गर्छ, साइट URL फिल्ड स्वचालित रूपमा रोजिएको डोमेनबाट भर्न गरिन्छ। ग्राहकहरूले छुट्टै URL फिल्ड भर्नु पर्दैन।

### खोज व्यवहार (Search behaviour) {#search-behaviour}

- ग्राहक टाइप गर्दा AJAX मार्फत वास्तविक समयमा डोमेन उपलब्धता जाँच गरिन्छ
- यदि मनपर्ने डोमेन उपलब्ध छैन भने वैकल्पिक TLD सुझावहरू देखाउँछ
- मूल्य निर्धारण लाइभ रूपमा प्राप्त गरिन्छ र स्पष्ट रूपमा प्रदर्शन गरिन्छ (दर्ता मूल्य, नवीकरण मूल्य, वैकल्पिक WHOIS गोपनीयता शुल्क)
- कूपन कोडहरू अन्य कुनै पनि उत्पादनको लागि डोमेन उत्पादनहरूमा लागू हुन्छन्

**खोज प्रतिक्रिया मिलाउन ट्यून गर्ने:**

```php
// ढिलो इन्टरनेट जडानमा API कल कम गर्न डिबाउंस ढिलाइ (मिलीसेकेन्ड) बढाउने
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // डिफल्ट: 500
});
```

**डोमेन खोज फारममा कस्टम फिल्डहरू थप्ने:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'अतिरिक्त नोटहरू',
    ];
    return $fields;
});
```

## ग्राहक DNS व्यवस्थापन (Customer DNS management) {#customer-dns-management}

ग्राहकहरूले आफ्नो दर्ता गरिएका डोमेनहरूको लागि **My Account** पृष्ठबाट, आफ्नो डोमेनको प्रविष्टि अन्तर्गत DNS रेकर्डहरू व्यवस्थापन गर्न सक्छन्।

### समर्थित रेकर्ड प्रकारहरू (Supported record types) {#supported-record-types}

| प्रकार | प्रयोग |
|---|---|
| **A** | होस्टनामलाई IPv4 ठेगानामा नक्सा गर्ने |
| **AAAA** | होस्टनामलाई IPv6 ठेगानामा नक्सा गर्ने |
| **CNAME** | अर्को होस्टनाममा इशारा गर्ने एलायस (alias) बनाउने |
| **MX** | मेल इञ्जिन सर्भर सेट गर्ने |
| **TXT** | SPF, DMARC, प्रमाणीकरण वा अन्य टेक्स्ट रेकर्डहरू थप्ने |

### कुन प्रदायकहरूले DNS व्यवस्थापन समर्थन गर्छन्? {#which-providers-support-dns-management}

DNS व्यवस्थापन (रेकर्डहरू थप्ने, सम्पादन गर्ने, मेटाउने) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, र **Openprovider** मा उपलब्ध छ। **Hostinger** डोमेनहरूलाई Domain Seller मार्फत नेमसर्भर अपडेट गर्न सकिन्छ; होस्ट गरिएका डोमेनहरूको DNS रेकर्डहरू कोर Hostinger डोमेन-म्यापिङ इन्टिग्रेशनद्वारा व्यवस्थापन गरिन्छ। Namecheap, GoDaddy, र NameSilo डोमेनहरूले स्थिति र समाप्ति जानकारी देखाउँछन् तर DNS लाई रजिस्ट्रारको कन्ट्रोल प्यानलमा सिधै व्यवस्थापन गर्नुपर्छ।

### डिफाल्ट DNS रेकर्डहरू {#default-dns-records}

कुनै पनि डोमेन दर्ता गर्दा स्वचालित रूपमा लागू हुने डिफाल्ट DNS रेकर्डहरू तपाईंले सेट गर्न सक्नुहुन्छ। **Settings › Domain Seller › Default DNS Records** मा जानुहोस्।

डिफाल्ट रेकर्ड मानहरू दुई टोकनलाई समर्थन गर्दछ:

| Token | यसले बदलिन्छ |
|---|---|
| `{DOMAIN}` | दर्ता गरिएको डोमेन नाम (जस्तै, `example.com`) |
| `{SITE_URL}` | ग्राहकको साइटको लागि वर्डप्रेस साइट URL |

**उदाहरण — एपिक्स डोमेन र www लाई आफ्नो सर्भर IP मा इंगित गर्नुहोस्:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### एडमिन: DNS हेर्ने र सम्पादन गर्ने {#admin-viewing-and-editing-dns}

नेटवर्क एडमिनहरूले **Network Admin › Ultimate Multisite › Domains** मा डोमेनको एडिट पेजबाट कुनै पनि ग्राहक डोमेनका लागि DNS रेकर्डहरू हेर्न र सम्पादन गर्न सक्छन्।
