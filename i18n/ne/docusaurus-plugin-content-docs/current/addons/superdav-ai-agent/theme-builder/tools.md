# उपकरण र कन्फिगरेसन

Theme Builder ले तपाईंको WordPress साइट विश्लेषण, डिजाइन, र निर्माण गर्न उपकरणहरूको सेट प्रयोग गर्छ। संस्करण 1.16.0 मा, **sd-ai-agent/site-scrape अब स्तर 1 उपकरण हो**, जसले यसलाई Theme Builder मा पूर्वनिर्धारित रूपमा उपलब्ध गराउँछ। Superdav AI Agent v1.18.0 ले विश्वसनीय विकासकर्ता कार्यप्रवाहहरूका लागि अलग रूपमा वितरण गरिएको Advanced सहायक प्याकेज पनि परिचय गराउँछ; ती उपकरणहरू WordPress.org कोर प्याकेजको भाग होइनन् र अलग रूपमा स्थापना तथा आधिकारिकरण गर्नुपर्छ।

## उपलब्ध उपकरणहरू

### स्तर 1 उपकरणहरू (सधैं उपलब्ध)

स्तर 1 उपकरणहरू कुनै अतिरिक्त कन्फिगरेसन बिना Theme Builder मा पूर्वनिर्धारित रूपमा उपलब्ध हुन्छन्।

#### sd-ai-agent/site-scrape

**उद्देश्य:** डिजाइन प्रेरणा, सामग्री संरचना, र कार्यक्षमता निकाल्न विद्यमान वेबसाइटहरूको विश्लेषण गर्नु।

**क्षमताहरू:**

- **वेबसाइट विश्लेषण** — प्रतिस्पर्धी वा प्रेरणादायी वेबसाइटहरू स्क्रेप र विश्लेषण गर्नु
- **डिजाइन निकासी** — रङ, फन्ट, र लेआउट ढाँचाहरू पहिचान गर्नु
- **सामग्री संरचना** — पृष्ठ संगठन र पदानुक्रम बुझ्नु
- **सुविधा पत्ता लगाउने** — plugin हरू र कार्यक्षमता पहिचान गर्नु
- **प्रदर्शन विश्लेषण** — पृष्ठ गति र अनुकूलन जाँच गर्नु
- **SEO विश्लेषण** — मेटा ट्याग र संरचित डाटा समीक्षा गर्नु

**प्रयोग:**

```
example.com को डिजाइन विश्लेषण गर्नुहोस् र यसलाई मेरो साइटका लागि प्रेरणाको रूपमा प्रयोग गर्नुहोस्।
```

**यसले के निकाल्छ:**

- रङ प्यालेट र रङ प्रयोग
- टाइपोग्राफी (फन्ट र आकारहरू)
- लेआउट संरचना र स्पेसिङ
- नेभिगेसन ढाँचाहरू
- सामग्री संगठन
- छवि र मिडिया प्रयोग
- अन्तरक्रियात्मक तत्वहरू
- मोबाइल प्रतिक्रियाशीलता

**सीमितताहरू:**

- पासवर्ड-सुरक्षित साइटहरू स्क्रेप गर्न सक्दैन
- robots.txt र साइट नीतिहरूको सम्मान गर्छ
- गतिशील सामग्री क्याप्चर नगर्न सक्छ
- JavaScript-धेरै भएका साइटहरूमा सीमित विश्लेषण हुन सक्छ
- ठूला साइटहरूको विश्लेषण गर्न बढी समय लाग्न सक्छ

### स्तर 2 उपकरणहरू (वैकल्पिक)

स्तर 2 उपकरणहरू Theme Builder सेटिङहरूमा स्पष्ट रूपमा सक्षम गरिँदा उपलब्ध हुन्छन्।

#### उन्नत एनालिटिक्स

विस्तृत प्रदर्शन मेट्रिक्स प्रदान गर्छ:

- पृष्ठ लोड समय
- Core Web Vitals
- SEO स्कोर
- पहुँचयोग्यता स्कोर
- उत्कृष्ट अभ्यास स्कोर

#### सामग्री अनुकूलक

यसका लागि सुधारहरू विश्लेषण र सुझाव गर्छ:

- पठनीयता
- SEO अनुकूलन
- कीवर्ड प्रयोग
- सामग्री संरचना
- Call-to-action स्थान निर्धारण

### स्तर 3 उपकरणहरू (प्रिमियम)

स्तर 3 उपकरणहरूलाई अतिरिक्त कन्फिगरेसन वा प्रिमियम सुविधाहरू आवश्यक पर्छ।

#### AI सामग्री जनरेटर

यसका लागि सामग्री उत्पन्न गर्छ:

- उत्पादन विवरणहरू
- सेवा पृष्ठहरू
- ब्लग पोस्टहरू
- मेटा विवरणहरू
- Call-to-action प्रतिलिपि

#### उन्नत डिजाइन उपकरणहरू

उन्नत डिजाइन क्षमताहरू प्रदान गर्छ:

- कस्टम CSS उत्पादन
- एनिमेसन सिर्जना
- अन्तरक्रियात्मक तत्व डिजाइन
- उन्नत रङ सिद्धान्त
- टाइपोग्राफी अनुकूलन

### उन्नत सहायक उपकरणहरू (अलग प्याकेज)

Advanced सहायक प्याकेज कोर Superdav AI Agent रिलिजबाट अलग रूपमा वितरण गरिन्छ। यो विश्वसनीय विकासकर्ता र साइट-मालिक वातावरणहरूका लागि हो, जहाँ प्रशासकहरूले एजेन्टलाई उच्च-प्रभाव उपकरण प्रयोग गर्न स्पष्ट रूपमा अनुमति दिन्छन्।

Advanced सहायक क्षमताहरूमा समावेश हुन सक्छ:

- **फाइल प्रणाली उपकरणहरू** — विश्वसनीय वातावरणहरूमा स्वीकृत फाइलहरू निरीक्षण र परिमार्जन गर्नु
- **डाटाबेस उपकरणहरू** — स्पष्ट रूपमा आधिकारिकरण गरिँदा साइट डाटा क्वेरी वा अपडेट गर्नु
- **WP-CLI उपकरणहरू** — WordPress मर्मतसम्भार र निरीक्षण आदेशहरू चलाउनु
- **REST डिस्प्याचर उपकरणहरू** — नियन्त्रित डिस्प्याच कार्यप्रवाहहरू मार्फत दर्ता गरिएका REST endpoint हरू कल गर्नु
- **Plugin builder उपकरणहरू** — plugin कोडको संरचना तयार गर्नु र पुनरावृत्ति गर्नु
- **Git स्न्यापसट उपकरणहरू** — जोखिमपूर्ण विकासकर्ता सञ्चालनहरूअघि स्न्यापसटहरू सिर्जना गर्नु
- **प्रयोगकर्ता-व्यवस्थापन उपकरणहरू** — अनुमति भएको ठाउँमा प्रयोगकर्ता अभिलेखहरू समीक्षा वा समायोजन गर्न मद्दत गर्नु
- **बेन्चमार्क उपकरणहरू** — विकास समीक्षाका लागि प्रदर्शन वा क्षमता बेन्चमार्कहरू चलाउनु

यीलाई हरेक स्थापनाका लागि सामान्य रूपमा उपलब्ध Theme Builder उपकरणहरूका रूपमा दस्तावेजीकरण नगर्नुहोस्। यी केवल Advanced सहायक प्याकेज स्थापना, सक्षम, र विश्वसनीय प्रशासकहरूमा सीमित गरिएको अवस्थामा मात्र उपलब्ध हुन्छन्।

## उपकरण कन्फिगरेसन

### उपकरणहरू सक्षम गर्ने

Theme Builder मा अतिरिक्त उपकरणहरू सक्षम गर्न:

1. **Gratis AI Agent → Settings** मा जानुहोस्
2. **Theme Builder → Tools** मा जानुहोस्
3. आवश्यकताअनुसार उपकरणहरू अन/अफ टगल गर्नुहोस्
4. सेटिङहरू सुरक्षित गर्नुहोस्

### उपकरण अनुमतिहरू

केही उपकरणहरूलाई अनुमतिहरू आवश्यक पर्छ:

- **साइट स्क्रेपिङ** — इन्टरनेट पहुँच आवश्यक पर्छ
- **एनालिटिक्स** — Google Analytics एकीकरण आवश्यक पर्छ
- **सामग्री उत्पादन** — API कुञ्जीहरू आवश्यक पर्छ
- **उन्नत सुविधाहरू** — प्रिमियम सदस्यता आवश्यक पर्न सक्छ
- **Advanced सहायक उपकरणहरू** — अलग रूपमा वितरण गरिएको Advanced प्याकेज र स्पष्ट प्रशासक विश्वास आवश्यक पर्छ

### API कुञ्जीहरू र क्रेडेन्सियलहरू

ती आवश्यक पर्ने उपकरणहरूका लागि API कुञ्जीहरू कन्फिगर गर्नुहोस्:

1. **Gratis AI Agent → Settings → API Keys** मा जानुहोस्
2. प्रत्येक उपकरणका लागि क्रेडेन्सियलहरू प्रविष्ट गर्नुहोस्
3. जडान परीक्षण गर्नुहोस्
4. सुरक्षित रूपमा सेभ गर्नुहोस्

### Advanced सहायक प्याकेज स्थापना गर्ने

Advanced सहायक प्याकेज WordPress.org कोर ZIP बाट अलग रूपमा रिलिज गरिन्छ। यसलाई रिलिजका लागि आधिकारिक परियोजना वितरण च्यानलबाट मात्र स्थापना गर्नुहोस्, त्यसपछि पहुँचलाई विश्वसनीय प्रशासकहरूमा सीमित गर्नुहोस्। उत्पादन साइटमा फाइल प्रणाली, डाटाबेस, WP-CLI, REST डिस्प्याचर, plugin builder, प्रयोगकर्ता-व्यवस्थापन, वा बेन्चमार्क उपकरणहरू सक्षम गर्नु अघि मानव स्वीकृति गेटहरू र git स्न्यापसट व्यवहार समीक्षा गर्नुहोस्।

## sd-ai-agent/site-scrape प्रयोग गर्दै

### आधारभूत प्रयोग

Theme Builder लाई वेबसाइट विश्लेषण गर्न भन्नुहोस्:

```
competitor.com मा रहेको मेरो प्रतिस्पर्धीको साइटको डिजाइन विश्लेषण गर्नुहोस्
र मेरो साइटका लागि डिजाइन सुधारहरू सुझाव गर्नुहोस्।
```

### विशिष्ट विश्लेषण

विशिष्ट प्रकारका विश्लेषण अनुरोध गर्नुहोस्:

```
example.com बाट रङ प्यालेट निकाल्नुहोस् र यसलाई प्रेरणाको रूपमा प्रयोग गर्नुहोस्।
```

```
example.com को नेभिगेसन संरचना विश्लेषण गर्नुहोस् र त्यस्तै
संगठन मेरो साइटमा लागू गर्नुहोस्।
```

```
example.com को मोबाइल प्रतिक्रियाशीलता जाँच गर्नुहोस् र मेरो साइट
त्यत्तिकै प्रतिक्रियाशील छ भनी सुनिश्चित गर्नुहोस्।
```

### तुलना विश्लेषण

धेरै साइटहरू तुलना गर्नुहोस्:

```
site1.com र site2.com का डिजाइनहरू तुलना गर्नुहोस् र दुवैका उत्कृष्ट तत्वहरू
मिलाएर एउटा डिजाइन सिर्जना गर्नुहोस्।
```

## उपकरण सीमितताहरू र विचारणीय कुराहरू

### दर सीमितता

- सर्भरमा अत्यधिक भार पर्न नदिन scraping दर-सीमित गरिएको छ
- एउटै site मा गरिएका धेरै अनुरोधहरू throttled हुन सक्छन्
- ठूला sites विश्लेषण गर्न बढी समय लाग्न सक्छ

### सामग्रीको शुद्धता

- गतिशील सामग्री पूर्ण रूपमा समातिएको नहुन सक्छ
- JavaScript-रेन्डर गरिएको सामग्री अपूर्ण हुन सक्छ
- वास्तविक-समय डाटा प्रतिबिम्बित नहुन सक्छ
- केही सामग्री paywalls पछाडि हुन सक्छ

### कानुनी र नैतिक विचारहरू

- robots.txt र site नीतिहरूको सम्मान गर्नुहोस्
- पुनःप्रकाशनका लागि प्रतिलिपि अधिकार भएको सामग्री scrape नगर्नुहोस्
- विश्लेषणलाई प्रेरणाका लागि प्रयोग गर्नुहोस्, प्रतिलिपि गर्न होइन
- निकालेको सामग्री प्रयोग गर्ने अधिकार तपाईंसँग छ भनी प्रमाणित गर्नुहोस्
- विश्लेषण गरिएका sites का सेवा सर्तहरू पालना गर्नुहोस्

### कार्यसम्पादन प्रभाव

- ठूला sites scrape गर्न समय लाग्न सक्छ
- एकै समयमा धेरै scrapes गर्दा ढिलो हुन सक्छ
- नेटवर्क जडानले गतिमा प्रभाव पार्छ
- ठूला datasets लाई थप प्रशोधन आवश्यक पर्न सक्छ

## उत्कृष्ट अभ्यासहरू

### Site विश्लेषण प्रयोग गर्दै

1. **धेरै sites विश्लेषण गर्नुहोस्** — धेरै स्रोतहरूबाट प्रेरणा सङ्कलन गर्नुहोस्
2. **संरचनामा ध्यान दिनुहोस्** — layout र संगठनका ढाँचाहरू सिक्नुहोस्
3. **रङहरू निकाल्नुहोस्** — रङ palettes लाई सुरुवाती बिन्दुका रूपमा प्रयोग गर्नुहोस्
4. **टाइपोग्राफी अध्ययन गर्नुहोस्** — तपाईंलाई मन पर्ने font संयोजनहरू पहिचान गर्नुहोस्
5. **नेभिगेसन समीक्षा गर्नुहोस्** — menu संगठन बुझ्नुहोस्

### नैतिक Scraping

1. **robots.txt को सम्मान गर्नुहोस्** — site दिशानिर्देशहरू पालना गर्नुहोस्
2. **सामग्री प्रतिलिपि नगर्नुहोस्** — विश्लेषणलाई केवल प्रेरणाका लागि प्रयोग गर्नुहोस्
3. **अधिकार प्रमाणित गर्नुहोस्** — निकालेको सामग्री प्रयोग गर्न सक्नुहुन्छ भन्ने सुनिश्चित गर्नुहोस्
4. **स्रोतहरू उल्लेख गर्नुहोस्** — प्रेरणाका स्रोतहरूलाई श्रेय दिनुहोस्
5. **सर्तहरू पालना गर्नुहोस्** — site का सेवा सर्तहरू पालना गर्नुहोस्

### Tool को प्रभावकारिता अधिकतम बनाउने

1. **विशिष्ट हुनुहोस्** — विशिष्ट प्रकारका विश्लेषण अनुरोध गर्नुहोस्
2. **सन्दर्भ प्रदान गर्नुहोस्** — तपाईंको site को उद्देश्य व्याख्या गर्नुहोस्
3. **अपेक्षाहरू निर्धारण गर्नुहोस्** — तपाईंका design लक्ष्यहरू वर्णन गर्नुहोस्
4. **पुनरावृत्ति गर्नुहोस्** — परिणामका आधारमा परिष्कृत गर्नुहोस्
5. **tools संयोजन गर्नुहोस्** — व्यापक विश्लेषणका लागि धेरै tools प्रयोग गर्नुहोस्

## समस्या समाधान

### Site Scrape हुँदैन

- site सार्वजनिक रूपमा पहुँचयोग्य छ कि छैन जाँच गर्नुहोस्
- robots.txt ले scraping अनुमति दिन्छ कि भनेर प्रमाणित गर्नुहोस्
- फरक site प्रयास गर्नुहोस्
- internet connection जाँच गर्नुहोस्
- समस्या जारी रहेमा support लाई सम्पर्क गर्नुहोस्

### अपूर्ण विश्लेषण

- site मा गतिशील सामग्री हुन सक्छ
- JavaScript पूर्ण रूपमा rendered नभएको हुन सक्छ
- ठूला sites timeout हुन सक्छन्
- यसको सट्टा विशिष्ट pages विश्लेषण गर्ने प्रयास गर्नुहोस्
- विशिष्ट विश्लेषण प्रकारहरू अनुरोध गर्नुहोस्

### ढिलो कार्यसम्पादन

- ठूला sites विश्लेषण गर्न बढी समय लाग्छ
- एकै समयमा धेरै scrapes गर्दा ढिलो हुन्छ
- नेटवर्क जडानले गतिमा प्रभाव पार्छ
- कम भीड हुने समयमा प्रयास गर्नुहोस्
- पहिले साना sections विश्लेषण गर्नुहोस्
