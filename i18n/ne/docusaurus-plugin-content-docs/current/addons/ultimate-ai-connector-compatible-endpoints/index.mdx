# मिल्दो अन्त्यबिन्दुहरूका लागि Ultimate AI Connector {#ultimate-ai-connector-for-compatible-endpoints}

मिल्दो अन्त्यबिन्दुहरूका लागि Ultimate AI Connector ले OpenAI-संग मिल्दो AI प्रदायक API हरूलाई तपाईंको WordPress बहुसाइट सञ्जालसँग एकीकृत गर्छ। धेरै AI प्रदायकहरू कन्फिगर गर्नुहोस् — प्रत्येकको आफ्नै अन्त्यबिन्दु URL, API कुञ्जी, र पूर्वनिर्धारित मोडेलसहित — र कुनै प्रदायक उपलब्ध नभएमा स्वचालित फलब्याकसहित अनुरोधहरू रुट गर्नुहोस्।

## मुख्य सुविधाहरू {#key-features}

- **बहु-प्रदायक समर्थन**: धेरै AI प्रदायक कार्डहरू थप्नुहोस्, प्रत्येकको आफ्नै अन्त्यबिन्दु URL, API कुञ्जी, र पूर्वनिर्धारित मोडेलसहित
- **तान्दै-पुनःक्रम मिलाउने**: प्रत्येक कार्डको ग्रिप ह्यान्डल प्रयोग गरेर प्रदायकहरूलाई प्राथमिकताअनुसार मिलाउनुहोस्; अनुरोधहरूले प्रदायकहरूलाई क्रमअनुसार प्रयोग गर्छन्
- **स्वचालित फलब्याक रुटिङ**: प्राथमिक प्रदायक उपलब्ध नभएमा, अनुरोधहरू स्वचालित रूपमा अर्को कन्फिगर गरिएको प्रदायकमा फलब्याक हुन्छन्
- **प्रति-प्रदायक कन्फिगरेसन**: प्रत्येक प्रदायक आफ्नै प्रमाणपत्र र मोडेल चयनसहित स्वतन्त्र रूपमा कन्फिगर गरिन्छ
- **कार्ड-आधारित सेटिङ UI**: स्थिर WordPress 6.9+ Card कम्पोनेन्टहरू (CardBody, CardHeader, CardDivider) मा निर्मित
- **थप्दा स्वतः-विस्तार**: नयाँ प्रदायक कार्डहरू स्वचालित रूपमा विस्तार हुन्छन् ताकि तपाईं तुरुन्तै विवरणहरू भर्न सक्नुहोस्
- **स्क्रिप्ट क्यास बस्टिङ**: सबैभन्दा पछिल्लो संस्करण लोड भएको सुनिश्चित गर्न plugin अद्यावधिक हुँदा प्रदायक स्क्रिप्टहरूको क्यास-बस्ट गरिन्छ

## आवश्यकताहरू {#requirements}

- WordPress 5.3 वा सोभन्दा माथि
- PHP 7.4 वा सोभन्दा माथि
- Ultimate Multisite plugin (सक्रिय)
- OpenAI-संग मिल्दो API अन्त्यबिन्दु (स्व-होस्ट गरिएको वा तेस्रो-पक्ष)

## स्थापना {#installation}

1. addon फाइलहरू तपाईंको `/wp-content/plugins/` डाइरेक्टरीमा अपलोड गर्नुहोस्
2. WordPress मा 'प्लगिनहरू' मेनुबाट plugin सक्रिय गर्नुहोस्
3. addon कन्फिगर गर्न **Ultimate Multisite → AI Connector** मा जानुहोस्

## कन्फिगरेसन {#configuration}

### बहु-प्रदायक सेटअप {#multi-provider-setup}

कन्फिगरेसन स्क्रिनले कार्ड-आधारित लेआउट प्रयोग गर्छ जहाँ प्रत्येक कार्डले एउटा AI प्रदायक प्रतिनिधित्व गर्छ। तपाईंलाई आवश्यक परे जति प्रदायकहरू थप्न सक्नुहुन्छ।

#### प्रदायक थप्ने {#adding-a-provider}

1. सञ्जाल admin मा **Ultimate Multisite → AI Connector** मा जानुहोस्
2. **प्रदायक थप्नुहोस्** क्लिक गर्नुहोस्
3. नयाँ प्रदायक कार्ड देखिन्छ र स्वचालित रूपमा विस्तार हुन्छ
4. प्रदायक विवरणहरू भर्नुहोस्:
   - **प्रदायक नाम** — यो प्रदायक पहिचान गर्न लेबल (जस्तै "OpenAI", "Mistral", "Local Ollama")
   - **अन्त्यबिन्दु URL** — OpenAI-संग मिल्दो API को आधार URL (जस्तै `https://api.openai.com/v1`)
   - **API कुञ्जी** — यस प्रदायकका लागि तपाईंको API कुञ्जी
   - **पूर्वनिर्धारित मोडेल** — यस प्रदायकका लागि पूर्वनिर्धारित रूपमा प्रयोग हुने मोडेल पहिचानकर्ता (जस्तै `gpt-4o`, `mistral-small`)
5. **सेटिङहरू सुरक्षित गर्नुहोस्** क्लिक गर्नुहोस्

#### प्रदायक हटाउने {#removing-a-provider}

कुनै पनि प्रदायक कार्डमा रहेको **हटाउनुहोस्** बटन क्लिक गरेर यसलाई मेटाउनुहोस्। परिवर्तन लागू गर्न सेटिङहरू सुरक्षित गर्नुहोस्।

#### प्रदायकहरूको क्रम मिलाउने {#reordering-providers}

कुनै पनि प्रदायक कार्डको बाँया छेउमा रहेको ग्रिप आइकन (⣿) तानेर यसको क्रम मिलाउनुहोस्। क्रमले फलब्याक प्राथमिकता निर्धारण गर्छ: अनुरोधहरू पहिलो प्रदायकमा पठाइन्छन् र अनुरोध असफल भएमा पछिल्ला प्रदायकहरूमा फलब्याक हुन्छन्।

### फलब्याक रुटिङ {#fallback-routing}

प्राथमिक प्रदायकमा अनुरोध असफल हुँदा — टाइमआउट, दर सीमा, वा सेवा अवरोधका कारण — connector ले सूचीको अर्को प्रदायकसँग स्वचालित रूपमा पुनःप्रयास गर्छ। कुनै प्रदायक सफल नभएसम्म वा सबै प्रदायकहरू प्रयास गरिन नसकेसम्म यो सूचीमा तलतिर जारी रहन्छ।

| परिदृश्य | व्यवहार |
|----------|-----------|
| प्राथमिक प्रदायकले प्रतिक्रिया दिन्छ | अनुरोध सामान्य रूपमा सेवा गरिन्छ |
| प्राथमिक प्रदायक असफल हुन्छ | अनुरोध अर्को प्रदायकसँग पुनःप्रयास गरिन्छ |
| सबै प्रदायकहरू असफल हुन्छन् | कलरलाई त्रुटि फर्काइन्छ |

फलब्याक कल गर्ने plugin वा theme का लागि पारदर्शी हुन्छ — कति प्रदायकहरू प्रयास गरिए पनि, तिनीहरूले सफल प्रतिक्रिया वा अन्तिम त्रुटि प्राप्त गर्छन्।

### प्रदायक कन्फिगरेसन सन्दर्भ {#provider-configuration-reference}

| फिल्ड | आवश्यक | विवरण |
|-------|----------|-------------|
| प्रदायक नाम | हो | यस प्रदायकका लागि देखाइने लेबल |
| अन्त्यबिन्दु URL | हो | OpenAI-संग मिल्दो API को आधार URL |
| API कुञ्जी | हो | यस प्रदायकका लागि प्रमाणीकरण कुञ्जी |
| पूर्वनिर्धारित मोडेल | हो | अनुरोधमा कुनै मोडेल निर्दिष्ट नभएमा प्रयोग हुने मोडेल पहिचानकर्ता |

## समस्या समाधान {#troubleshooting}

### कुनै प्रदायकहरू सूचीकृत छैनन् {#no-providers-are-listed}

सुरक्षित गरेपछि प्रदायक सूची खाली छ भने, पुष्टि गर्नुहोस् कि:

- तपाईंले प्रदायकहरू थपेपछि **सेटिङहरू सुरक्षित गर्नुहोस्** क्लिक गर्नुभएको छ
- तपाईंको ब्राउजरले सेटिङ पृष्ठ सुरक्षित हुनबाट रोकेको छैन (JavaScript त्रुटिहरूका लागि ब्राउजर कन्सोल जाँच गर्नुहोस्)
- plugin सक्रिय छ र तपाईं सञ्जाल admin स्क्रिनमा हुनुहुन्छ

### सबै प्रदायकहरूका लागि अनुरोधहरू असफल भइरहेका छन् {#requests-are-failing-for-all-providers}

- प्रत्येक प्रदायकको अन्त्यबिन्दु URL तपाईंको सर्भरबाट पहुँचयोग्य छ भनी प्रमाणित गर्नुहोस् (तपाईंको ब्राउजरबाट मात्र होइन)
- प्रत्येक API कुञ्जी मान्य छ र म्याद सकिएको छैन भनी जाँच गर्नुहोस्
- तिनीहरूको dashboard मा प्रदायकको दर-सीमा र कोटा स्थिति समीक्षा गर्नुहोस्
- connector बाट विस्तृत त्रुटि सन्देशहरू कैद गर्न WordPress debug logging (`WP_DEBUG_LOG`) सक्षम गर्नुहोस्

### कुनै प्रदायक फलब्याकका लागि प्रयोग भइरहेको छैन {#a-provider-is-not-being-used-for-fallback}

तान्दै-पुनःक्रम मिलाउँदा फलब्याक क्रम परिवर्तन हुन्छ। तपाईंले फलब्याक अनुरोधहरू प्राप्त गर्ने अपेक्षा गरेको प्रदायक सूचीमा सही रूपमा राखिएको छ र यसको अन्त्यबिन्दु URL तथा API कुञ्जी सही छन् भनी पुष्टि गर्नुहोस्।

### नयाँ प्रदायक कार्ड स्वचालित रूपमा विस्तार हुँदैन {#new-provider-card-does-not-expand-automatically}

स्वतः-विस्तारका लागि तपाईंको ब्राउजरमा JavaScript सक्षम हुनुपर्छ। क्लिक गर्दा कार्ड विस्तार हुँदैन भने, त्रुटिहरूका लागि ब्राउजर कन्सोल जाँच गर्नुहोस्। स्वतः-विस्तार ट्रिगर नभए पनि कार्ड हेडर क्लिक गरेर कार्डका सामग्रीहरू सधैं पहुँचयोग्य हुन्छन्।
