# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency ले तपाईंको सञ्जाललाई धेरै मुद्रामा भुक्तानी स्वीकार गर्न सक्षम बनाउँछ। यसले प्रत्यक्ष वा म्यानुअल विनिमय दरहरू प्रयोग गरेर मूल्यहरू स्वचालित रूपमा रूपान्तरण गर्छ, कुनै पनि मुद्राका लागि प्रत्येक उत्पादनमा ठ्याक्कै मूल्य सेट गर्न दिन्छ, र भू-स्थानमार्फत प्रत्येक आगन्तुकको रुचाइएको मुद्रा पत्ता लगाउँछ — यी सबै एउटै प्रशासनिक Dashboard बाट व्यवस्थापन हुन्छन्।

## मुख्य विशेषताहरू {#key-features}

- **स्वचालित विनिमय दर रूपान्तरण** — मूल्यहरू तपाईंले रोजेको प्रदायकबाट प्राप्त दरहरू प्रयोग गरेर तपाईंको आधार मुद्राबाट रूपान्तरण गरिन्छन्
- **प्रत्येक-उत्पादन मूल्य निर्धारण ओभरराइडहरू** — स्वचालित रूपान्तरणलाई बाइपास गर्न कुनै पनि उत्पादनमा ठ्याक्कै मुद्रा-विशिष्ट मूल्यहरू सेट गर्नुहोस्
- **मुद्रा चयनकर्ता साइनअप फिल्ड** — ग्राहकहरूलाई चेकआउट/साइनअपका क्रममा आफ्नो रुचाइएको मुद्रा रोज्न दिन्छ
- **धेरै विनिमय दर प्रदायकहरू** — ECB, Frankfurter, Stripe, वा पूर्ण रूपमा म्यानुअल दरहरू
- **भू-स्थानमा आधारित मुद्रा पहिचान** — आगन्तुकको देशसँग मेल खाने मुद्रा स्वचालित रूपमा पूर्व-चयन गर्छ
- **प्रशासनिक Dashboard** — सक्रिय मुद्राहरू, विनिमय दरहरू, र प्रदायक सेटिङहरू एउटै ठाउँबाट व्यवस्थापन गर्नुहोस्

## आवश्यकताहरू {#requirements}

- Ultimate Multisite 2.0.0 वा उच्च
- WordPress Multisite स्थापना
- PHP 7.4 वा उच्च

## स्थापना {#installation}

1. प्लगइन फाइलहरू `/wp-content/plugins/ultimate-multisite-multi-currency` मा अपलोड गर्नुहोस्
2. WordPress मा **प्लगइनहरू** स्क्रिनमार्फत प्लगइन सक्रिय गर्नुहोस् (सञ्जाल सक्रिय गर्नुहोस्)
3. प्रशासनिक Dashboard खोल्न **Ultimate Multisite → Multi-Currency** मा जानुहोस्
4. तपाईंले स्वीकार गर्न चाहेका मुद्राहरू थप्नुहोस् र आफ्नो विनिमय दर प्रदायक कन्फिगर गर्नुहोस्

## कन्फिगरेसन {#configuration}

### प्रशासनिक Dashboard {#admin-dashboard}

Multi-Currency प्रशासनिक Dashboard (**Ultimate Multisite → Multi-Currency**) निम्नका लागि केन्द्रीय स्थान हो:

- मुद्राहरू थप्ने, सक्षम गर्ने, वा असक्षम गर्ने
- विनिमय दर प्रदायक रोज्ने र आवश्यक पर्दा दरहरू ताजा गर्ने
- सबै सक्रिय मुद्राहरूका हालका दरहरू समीक्षा गर्ने
- स्वचालित प्रदायकहरू उपयुक्त नभएको अवस्थामा म्यानुअल दरहरू सेट गर्ने

### विनिमय दर प्रदायकहरू {#exchange-rate-providers}

| प्रदायक | विवरण |
|---|---|
| **ECB** | European Central Bank — EUR-आधारित सञ्जालहरूका लागि निःशुल्क दैनिक दरहरू |
| **Frankfurter** | खुला-स्रोत ECB मिरर API — निःशुल्क, API key आवश्यक छैन |
| **Stripe** | Stripe का प्रत्यक्ष FX दरहरू प्रयोग गर्छ — सक्रिय Stripe एकीकरण आवश्यक पर्छ |
| **म्यानुअल** | तपाईंले दरहरू प्रत्यक्ष रूपमा प्रविष्ट गर्नुहुन्छ; बाह्य API कलहरू हुँदैनन् |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** अन्तर्गत आफ्नो प्रदायक चयन गर्नुहोस्। दरहरू कुनै पनि समयमा म्यानुअल रूपमा ताजा गर्न सकिन्छ वा स्वचालित रूपमा अद्यावधिक हुन तालिकाबद्ध गर्न सकिन्छ।

### प्रत्येक-उत्पादन मूल्य निर्धारण ओभरराइडहरू {#per-product-pricing-overrides}

जब स्वचालित रूपान्तरण पर्याप्त सटीक हुँदैन, तपाईं कुनै पनि उत्पादनमा कुनै विशिष्ट मुद्राका लागि ठ्याक्कै मूल्य सेट गर्न सक्नुहुन्छ:

1. उत्पादन सम्पादक खोल्नुहोस् (**Ultimate Multisite → Products → Edit**)
2. **Currency Pricing** खण्डसम्म स्क्रोल गर्नुहोस्
3. तपाईंले ओभरराइड गर्न चाहेको प्रत्येक मुद्राका लागि ठ्याक्कै मूल्य प्रविष्ट गर्नुहोस्
4. उत्पादन सुरक्षित गर्नुहोस्

ओभरराइड नभएका उत्पादनहरूले तपाईंको आधार मुद्राबाट रूपान्तरण गरिएको मूल्य प्रयोग गर्छन्।

### मुद्रा चयनकर्ता साइनअप फिल्ड {#currency-selector-signup-field}

मुद्रा चयनकर्ता फिल्ड तपाईंको साइनअप/चेकआउट प्रवाहमा थप्न सकिन्छ, ताकि ग्राहकहरूले दर्ताका बेला आफ्नो रुचाइएको मुद्रा रोज्न सकून्:

1. **Ultimate Multisite → Multi-Currency → Settings** मा जानुहोस्
2. **साइनअपमा मुद्रा चयनकर्ता देखाउनुहोस्** सक्षम गर्नुहोस्
3. वैकल्पिक रूपमा पूर्वनिर्धारित मुद्रा सेट गर्नुहोस् वा भू-स्थानलाई पूर्व-चयन गरिएको मान निर्धारण गर्न दिनुहोस्

चयन गरिएको मुद्रा सदस्यतासँग भण्डारण गरिन्छ र सबै पछिल्ला बिलिङका लागि प्रयोग गरिन्छ।

### भू-स्थानमा आधारित मुद्रा पहिचान {#geolocation-based-currency-detection}

भू-स्थान पहिचान सक्षम हुँदा, एडअनले आगन्तुकको देशलाई मुद्रासँग मिलाउँछ र मुद्रा चयनकर्ता फिल्डमा त्यसलाई पूर्व-चयन गर्छ। सक्षम गर्न:

1. **Ultimate Multisite → Multi-Currency → Settings** मा जानुहोस्
2. **आगन्तुकको स्थानबाट मुद्रा पत्ता लगाउनुहोस्** सक्षम गर्नुहोस्
3. **Geolocation Mapping** तालिकामा देशहरूलाई मुद्राहरूसँग म्याप गर्नुहोस्

यदि कुनै आगन्तुकको देशका लागि म्यापिङ छैन भने, सञ्जालको आधार मुद्रा फलब्याकका रूपमा प्रयोग गरिन्छ।

## बारम्बार सोधिने प्रश्नहरू {#frequently-asked-questions}

### यदि विनिमय दर प्रदायक उपलब्ध छैन भने के हुन्छ? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

एडअन पछिल्लो सफलतापूर्वक प्राप्त गरिएका दरहरूमा फलब्याक गर्छ। यदि कहिल्यै कुनै दर प्राप्त गरिएको छैन भने, यो तपाईंले कन्फिगर गरेका म्यानुअल दरहरूमा फलब्याक गर्छ। सुरक्षा जालका रूपमा म्यानुअल दरहरू कन्फिगर गर्न सिफारिस गरिन्छ।

### के म फरक मुद्राहरूका लागि फरक प्रदायकहरू प्रयोग गर्न सक्छु? {#can-i-use-different-providers-for-different-currencies}

सक्नुहुन्न — सबै मुद्राहरूका लागि एक पटकमा एउटा प्रदायक मात्र सक्रिय हुन्छ। व्यक्तिगत दरहरूमाथि पूर्ण नियन्त्रण चाहिन्छ भने **म्यानुअल** प्रयोग गर्नुहोस्।

### के भू-स्थानका लागि छुट्टै प्लगइन आवश्यक पर्छ? {#does-geolocation-require-a-separate-plugin}

कुनै बाह्य भू-स्थान प्लगइन आवश्यक पर्दैन। एडअनले WordPress को अन्तर्निर्मित IP-देखि-देश पहिचान प्रयोग गर्छ। बढी सटीकताका लागि, प्रदान गरिएका फिल्टर हुकहरूमार्फत समर्पित भू-स्थान सेवा एकीकृत गर्न सकिन्छ।

### प्रत्येक-उत्पादन ओभरराइडहरू करभन्दा अघि लागू हुन्छन् कि पछि? {#are-per-product-overrides-applied-before-or-after-tax}

प्रत्येक-उत्पादन ओभरराइडहरूले लक्षित मुद्रामा कर-पूर्व मूल्य सेट गर्छन्। तपाईंको विद्यमान कर कन्फिगरेसन प्रयोग गरेर ओभरराइड मूल्यको माथि कर गणना गरिन्छ।
