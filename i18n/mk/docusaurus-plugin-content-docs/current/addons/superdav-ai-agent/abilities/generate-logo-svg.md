# Генерирање лого SVG {#generate-logo-svg}

Способноста **Генерирање лого SVG** му овозможува на Theme Builder да создава и вградува приспособени SVG логоа директно во вашиот WordPress сајт со автоматско прочистување безбедно за namespace.

## Преглед {#overview}

Оваа способност генерира скалабилни векторски графики (SVG) за логоа врз основа на насоката за брендирање и дизајнерските претпочитања на вашиот сајт. Генерираните SVG-ја автоматски се прочистуваат за да се осигури дека се безбедни за употреба во WordPress, притоа задржувајќи го визуелниот интегритет.

## Параметри {#parameters}

| Параметар | Тип | Задолжително | Опис |
|-----------|------|----------|-------------|
| `site_name` | string | Да | Името на сајтот за кој треба да се генерира логото |
| `style` | string | Да | Дизајнерскиот стил (на пр., "modern", "classic", "minimalist", "playful") |
| `colors` | array | Не | Низа од hex кодови на бои што ќе се користат во логото (на пр., `["#678233", "#ffffff"]`) |
| `width` | number | Не | Ширина на SVG во пиксели (стандардно: 200) |
| `height` | number | Не | Висина на SVG во пиксели (стандардно: 200) |
| `include_text` | boolean | Не | Дали да се вклучи името на сајтот како текст во логото (стандардно: true) |

## Формат на излез {#output-format}

Способноста враќа SVG низа со следнава структура:

```json
{
  "svg": "<svg xmlns=\"http://www.w3.org/2000/svg\" viewBox=\"0 0 200 200\">...</svg>",
  "width": 200,
  "height": 200,
  "colors_used": ["#678233", "#ffffff"],
  "sanitised": true
}
```

## Однесување при прочистување на SVG {#svg-sanitisation-behaviour}

Генерираните SVG-ја поминуваат низ автоматско прочистување безбедно за namespace за да:

- **Отстранат небезбедни атрибути** — ги отстранува обработувачите на настани, скриптите и потенцијално опасните атрибути
- **Зачуваат namespaces** — ги задржува SVG namespaces (xmlns, xlink) за правилно прикажување
- **Ја потврдат структурата** — осигурува дека SVG е во согласност со W3C стандардите
- **Ги кодираат ентитетите** — правилно ги избегнува специјалните знаци во текстуалната содржина
- **Отстранат надворешни референци** — ги отстранува надворешните stylesheets и референците кон слики

Ова осигурува дека SVG е безбеден за директно вградување во WordPress без потреба од дополнително прочистување.

## Пример за употреба {#usage-example}

**Поттик:**
```
Генерирај модерно лого за мојот технолошки стартап наречен "CloudSync" користејќи сина и бела боја.
```

**Резултат:**
Способноста создава SVG лого што:
- Го вклучува името на сајтот "CloudSync"
- Ја користи наведената сино-бела шема на бои
- Ги следи принципите на модерен дизајн
- Автоматски е прочистено и подготвено за употреба

## Интеграција со Theme Builder {#integration-with-theme-builder}

Кога се користи изборот на дизајнерска насока во Theme Builder, способноста Генерирање лого SVG:

1. Ја анализира вашата дизајнерска насока и палета на бои
2. Генерира приспособено SVG лого што одговара на вашите претпочитања
3. Автоматски го вградува логото во заглавието/областа за брендирање на вашиот сајт
4. Го зачувува SVG како приспособено лого во WordPress media

## Најдобри практики {#best-practices}

- **Обезбедете јасни стилски претпочитања** — опишете го дизајнерскиот стил што го сакате (модерен, класичен, разигран итн.)
- **Наведете бои** — вклучете ги боите на вашиот бренд за доследност
- **Тестирајте го прикажувањето** — проверете дали логото се прикажува правилно на различни големини на екран
- **Дополнително приспособете** — користете ги алатките на WordPress за приспособување лого за да ја прилагодите големината и поставеноста

## Ограничувања {#limitations}

- SVG логоата се генерираат како статички графики (не анимирани)
- Сложените логоа со многу елементи може да бараат рачно доработување
- Приспособени фонтови не се поддржани; текстот користи системски фонтови
- Многу големи или многу мали димензии може да влијаат на квалитетот
