# Ultimate AI Connector за компатибилни крајни точки

Ultimate AI Connector за компатибилни крајни точки ги интегрира API-јата на OpenAI-компатибилни AI провајдери со вашата WordPress multisite мрежа. Конфигурирајте повеќе AI провајдери — секој со сопствен URL на крајна точка, API клуч и стандарден модел — и рутирајте барања со автоматско префрлање на резервен провајдер кога некој провајдер е недостапен.

## Клучни функции

- **Поддршка за повеќе провајдери**: Додајте повеќе картички за AI провајдери, секоја со сопствен URL на крајна точка, API клуч и стандарден модел
- **Влечење за прередување**: Подредете ги провајдерите по приоритет користејќи ја рачката за фаќање на секоја картичка; барањата ги користат провајдерите по редослед
- **Автоматско резервно рутирање**: Ако примарниот провајдер е недостапен, барањата автоматски се префрлаат на следниот конфигуриран провајдер
- **Конфигурација по провајдер**: Секој провајдер се конфигурира независно со сопствени акредитиви и избор на модел
- **Кориснички интерфејс за поставки базиран на картички**: Изграден врз стабилните WordPress 6.9+ Card компоненти (CardBody, CardHeader, CardDivider)
- **Автоматско проширување при додавање**: Новите картички за провајдери автоматски се прошируваат за да можете веднаш да ги внесете деталите
- **Освежување на кешот за скрипти**: Скриптите на провајдерите се освежуваат од кеш при ажурирање на plugin за да се осигури дека е вчитана најновата верзија

## Барања

- WordPress 5.3 или понова верзија
- PHP 7.4 или понова верзија
- Ultimate Multisite plugin (активен)
- OpenAI-компатибилна API крајна точка (самостојно хостирана или од трета страна)

## Инсталација

1. Поставете ги addon датотеките во вашиот `/wp-content/plugins/` директориум
2. Активирајте го plugin преку менито „Plugins“ во WordPress
3. Одете до **Ultimate Multisite → AI Connector** за да го конфигурирате addon

## Конфигурација

### Поставување на повеќе провајдери

Екранот за конфигурација користи распоред базиран на картички, каде што секоја картичка претставува еден AI провајдер. Можете да додадете онолку провајдери колку што ви се потребни.

#### Додавање провајдер

1. Одете на **Ultimate Multisite → AI Connector** во мрежниот админ
2. Кликнете **Add Provider**
3. Се појавува нова картичка за провајдер и автоматски се проширува
4. Пополнете ги деталите за провајдерот:
   - **Име на провајдер** — ознака за идентификација на овој провајдер (на пр. „OpenAI“, „Mistral“, „Local Ollama“)
   - **URL на крајна точка** — основниот URL на OpenAI-компатибилното API (на пр. `https://api.openai.com/v1`)
   - **API клуч** — вашиот API клуч за овој провајдер
   - **Стандарден модел** — идентификаторот на моделот што стандардно ќе се користи за овој провајдер (на пр. `gpt-4o`, `mistral-small`)
5. Кликнете **Save Settings**

#### Отстранување провајдер

Кликнете го копчето **Remove** на која било картичка за провајдер за да ја избришете. Зачувајте ги поставките за да ја примените промената.

#### Прередување провајдери

Влечете ја иконата за фаќање (⣿) на левата страна од која било картичка за провајдер за да ја прередите. Редоследот го одредува приоритетот за резервно префрлање: барањата се испраќаат до првиот провајдер и се префрлаат на следните провајдери ако некое барање не успее.

### Резервно рутирање

Кога барање до примарниот провајдер ќе не успее — поради истекување на време, ограничување на стапката или прекин на услугата — конекторот автоматски повторува со следниот провајдер во списокот. Ова продолжува надолу по списокот додека некој провајдер не успее или додека не се пробаат сите провајдери.

| Сценарио | Однесување |
|----------|-----------|
| Примарниот провајдер одговара | Барањето се опслужува нормално |
| Примарниот провајдер не успева | Барањето повторно се обидува со следниот провајдер |
| Сите провајдери не успеваат | Грешка се враќа до повикувачот |

Резервното префрлање е транспарентно за plugin или theme што повикува — тие добиваат или успешен одговор или конечна грешка, без разлика колку провајдери биле пробани.

### Референца за конфигурација на провајдер

| Поле | Задолжително | Опис |
|-------|----------|-------------|
| Име на провајдер | Да | Приказна ознака за овој провајдер |
| URL на крајна точка | Да | Основен URL на OpenAI-компатибилното API |
| API клуч | Да | Клуч за автентикација за овој провајдер |
| Стандарден модел | Да | Идентификатор на модел што се користи кога во барањето не е наведен модел |

## Решавање проблеми

### Не се прикажани провајдери

Ако списокот со провајдери е празен по зачувување, потврдете дека:

- Кликнавте **Save Settings** по додавањето провајдери
- Вашиот прелистувач не ја блокирал страницата со поставки од зачувување (проверете ја конзолата на прелистувачот за JavaScript грешки)
- Plugin е активен и се наоѓате на екранот за мрежен админ

### Барањата не успеваат за сите провајдери

- Проверете дали URL-от на крајната точка на секој провајдер е достапен од вашиот сервер (не само од вашиот прелистувач)
- Проверете дали секој API клуч е валиден и не е истечен
- Прегледајте го статусот на ограничување на стапката и квотата на провајдерот во нивниот dashboard
- Овозможете WordPress debug евидентирање (`WP_DEBUG_LOG`) за да снимите детални пораки за грешки од конекторот

### Провајдер не се користи за резервно префрлање

Влечењето за прередување го менува редоследот за резервно префрлање. Потврдете дека провајдерот што очекувате да прима резервни барања е правилно поставен во списокот и дека неговиот URL на крајна точка и API клуч се точни.

### Новата картичка за провајдер не се проширува автоматски

Автоматското проширување бара JavaScript да биде овозможен во вашиот прелистувач. Ако картичката не се проширува при клик, проверете ја конзолата на прелистувачот за грешки. Содржината на картичката секогаш е достапна со кликнување на заглавието на картичката, дури и ако автоматското проширување не се активира.
