# Ultimate AI Connector за WebLLM дневник на промени {#ultimate-ai-connector-for-webllm-changelog}

## Верзија 1.1.0 — Објавено на 2026-04-09 {#version-110--released-on-2026-04-09}

* Ново: Пловечки виџет за разговор со индикатор за статус во администраторската лента — секој најавен корисник може да му испрати prompt на LLM од страната на прелистувачот директно од предниот дел.
* Ново: SharedWorker извршување — MLC engine сега работи во SharedWorker, така што повеќе картички споделуваат една GPU сесија наместо да се натпреваруваат за неа.
* Ново: apiFetch middleware пресретнувач — WordPress REST барањата што се совпаѓаат со шемата на AI Client SDK транспарентно се насочуваат кон локалниот WebLLM broker, без потреба од loopback HTTP повратно патување.
* Ново: UI за поставки на виџетот во панелот Connector за вклучување/исклучување на виџетот за разговор и конфигурирање на однесувањето за автоматски prompt.
* Поправка: Присили IndexedDB cache backend за преземањата на тежините на моделот да ги преживеат HuggingFace xet CDN пренасочувањата што ја расипуваат стандардната Cache API патека.
* Поправка: Прескокни го context_window KV-cache override за embedding модели (тие немаат декодер и override предизвикуваше грешка при извршување).
* Поправка: Огласи го cold-start кандидатскиот модел во /webllm/v1/models пред worker картичката да се вчита, така што SDK корисниците веднаш гледаат модел.
* Поправка: Поврзи WebLLM во wpai_preferred_text_models filter, така што функцијата AI Experiments се насочува кон engine на прелистувачот кога е конфигуриран модел.
* Подобрено: Cache-busting, нормализација на содржината и поправки на хардверски референци откриени при end-to-end тестирање.
