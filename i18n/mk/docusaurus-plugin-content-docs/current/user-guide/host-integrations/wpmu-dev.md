# Интеграција со WPMU DEV

## Преглед
WPMU DEV е сеопфатна WordPress платформа која нуди хостинг, плагини и услуги за WordPress сајтови. Оваа интеграција овозможува автоматско синхронизирање на домен и управување со SSL сертификати помеѓу Ultimate Multisite и WPMU DEV хостингот.

## Функции
- Автоматско синхронизирање на domeni
- Управување со SSL сертификати
- Проширени обиди за верификација на SSL сертификатите

## Задолжинија
Интеграцијата автоматски го препознава дали се хостирате на WPMU DEV и користи вградената API. Не е потребно дополнително конфигурирање ако се хостирате на WPMU DEV.

Интеграцијата проверува присуството на константната `WPMUDEV_HOSTING_SITE_ID`, која автоматски се дефинира при хостирање на WPMU DEV.

## Инструкции за поставка

### 1. Проверете хостингот со WPMU DEV

Ако се хостирате на WPMU DEV, неопходните константи веќе треба да бидат дефинирани. Проверете дека:

1. Константната `WPMUDEV_HOSTING_SITE_ID` е дефинирана во вашата средина
2. Имате активна членство на WPMU DEV со пристап до API

### 2. Вклучете интеграцијата

1. Во администрацијата на WordPress, отидете на Ultimate Multisite > Settings (Поставки)
2. Најдете ја табот "Domain Mapping" (Мапирање на domeni)
3. Пролијте надолу до "Host Integrations" (Интеграции со хостинг)
4. Вклучете интеграцијата со WPMU DEV
5. Кликнете на "Save Changes" (Зачувај промени)

## Како работи

### Синхронизирање на domeni

Кога домен е мапиран во Ultimate Multisite:

1. Интеграцијата користи WPMU DEV API за да го додаде дометот во вашата сметка за хостинг
2. Таа автоматски додаде и `www` верзијата на доменот
3. WPMU DEV се грижи за конфигурацијата на доменот и издавањето на SSL сертификати

### Управување со SSL сертификати

Интеграцијата е конфигурирана да зголеми бројот проби на верификација на SSL сертификати за хостирањето на WPMU DEV, бидејќи може време да се издава и инсталираат SSL сертификатите. По дефолт, таа ќе обиде до 10 пати за верификација на SSL сертификат, во место кога стандардно има само 5 проби.

## Важни забелешки

### Прекинување на домен (Domain Removal)

В момента WPMU DEV API не нуди начин да се отстранат domeni. Кога мапирање на домен е прекинуто во Ultimate Multisite, доменот ќе остане во вашата сметка за хостирање на WPMU DEV. Ќе ќе треба да го отстраните рачно од табдашбордот на WPMU DEV хостирање ако е потребно.

### Аутентикација на API (API Authentication)

Интеграцијата користи API клучен WPMU DEV што е зачуван во вашата WordPress база на податоци како опција `wpmudev_apikey`. Ова се поставува автоматски кога ќе поврзете својот сайт со WPMU DEV.

## Решавање проблеми (Troubleshooting)

### Проблеми со врска на API (API Connection Issues)
- Проверете дали вашиот сайт е правилно поврзан со WPMU DEV
- Проверете дали опцијата `wpmudev_apikey` е поставувана во вашата WordPress база на податоци
- Осигурајте се дека вашето членство во WPMU DEV е активно

### Проблеми со SSL сертификати (SSL Certificate Issues)
- WPMU DEV може да им поземе време за издавање SSL сертификати (обично 5-15 минути)
- Интеграцијата е конфигурирана да проверува до 10 пати за SSL сертификати
- Ако SSL сертификатите сè уште не се издаваат по неколку проби, контактирајте ја поддршката на WPMU DEV

### Домен не додаден (Domain Not Added)
- Проверете ги логатите во Ultimate Multisite за било какви пораки за грешки
- Проверете дали доменот веќе не е додаден во WPMU DEV
- Осигурајте се дека вашиот план за хостирање на WPMU DEV поддржува бројот domeni што го додавате
