# Интеграција со Closte {#closte-integration}

## Преглед {#overview}
Closte е управување хостинг платформа за WordPress, изградена на Google Cloud инфраструктурата. Ова интегрира автоматско синхронизирање на домен и управување со SSL сертификати помеѓу Ultimate Multisite и Closte.

## Функции {#features}
- Автоматско синхронизирање на domeni
- Управување со SSL сертификати
- Поддршка за wildcard domeni
- Не е потребно никакое конфигурирање ако работите на Closte

## Задолжинија {#requirements}
Ако користите Closte, оваа констаната мора да биде дефинирана во вашата `wp-config.php` датотека:

```php
define('CLOSTE_CLIENT_API_KEY', 'ваша_api_ключ');
```

Оваа констаната обично веќе е дефинирана ако го хостувате на Closte.

## Инструкции за поставување {#setup-instructions}

### 1. Проверете ја вашата API клучна за Closte {#1-verify-your-closte-api-key}

Ако работите на Closte, констаната `CLOSTE_CLIENT_API_KEY` веќе треба да биде дефинирана во вашата `wp-config.php` датотека. Можете да ја проверите преку проверка на вашата `wp-config.php` датотека.

### 2. Вклучете ја интеграцијата {#2-enable-the-integration}

1. Во административниот панел на WordPress, отидете на Ultimate Multisite > Settings
2. Најдете го табот "Domain Mapping" (Мапирање на domeni)
3. Пролискувајте надолу до "Host Integrations" (Интеграции со хостови)
4. Вклучете ја интеграцијата со Closte
5. Кликнете на "Save Changes" (Зачувај промени)

## Како работи {#how-it-works}

Кога домен се мапира во Ultimate Multisite:

1. Интеграцијата испраќа барање на API-то на Closte за да го додаде доменот во вашата апликација
2. Closte автоматски управува со провизијање на SSL сертификати
3. Кога ќе се откаже мапирање на домен, интеграцијата ќе го откаже доменот од Closte

Интеграцијата работи и со поставката за интервал на проверка на DNS во Ultimate Multisite, што ви овозможува да конфигурирате колку често системот проверува пропирање на DNS и издавање на SSL сертификати.

## Создавање на запис за домен {#domain-record-creation}

Ова интеграција обезбедува дека кога се креира или дуплира сайт, автоматски се создава запис за домен. Ова е особено важно за интеграцијата со Closte, бидејќи создавањето на записот за домен активира API-от на Closte да го креи доминот и SSL сертификатот.

## Решавање проблеми (Troubleshooting) {#troubleshooting}

### Проблеми со врската со API {#api-connection-issues}
- Проверете дали вашата Closte API клучна е правилна
- Осигурајте се дека вашата сметка во Closte има потребните дозволи

### Проблеми со SSL сертификатите {#ssl-certificate-issues}
- Closte може да им затражи неколку време за издавање на SSL сертификати (обично 5-10 минути)
- Проверете дали вашите домени правилно насочуваат кон IP адресата на вашиот Closte сервер
- Проверете ги DNS записите за вашиот домен за да се осигурате дека се правилно конфигурирани

### Доминот не е додаден {#domain-not-added}
- Проверете ги логатите во Ultimate Multisite за било какви пораки за грешки
- Проверете дали доминот веќе не е додаден во Closte
- Осигурајте се дека DNS записите на вашиот домен се правилно конфигурирани

### Интервал за проверка на DNS {#dns-check-interval}
- Ако издавањето на SSL сертификати трае премногу долго, можете да го промените интервалот за проверка на DNS во поставките за мапирање на домените (Domain Mapping settings)
- Дефолтниот интервал е 300 секунди (5 минути), но можете да го поставите најнизок како 10 секунди за побрза проверка за тестови
