# Интеграција со WP Engine {#wp-engine-integration}

## Преглед {#overview}
WP Engine е премиум менаџмент хостинг платформа за WordPress која нуди оптимизирано перформанси, сигурност и скалабилност за вашите WordPress страници. Ова интегрира автоматско синхронизирање на домени помеѓу Ultimate Multisite и WP Engine.

## Функции {#features}
- Автоматско синхронизирање на домените
- Поддршка за субадомени за инсталации со многу страници (multisite)
- Беспрекорен интеграција со постоечките системи на WP Engine

## Задолжинија {#requirements}
Интеграцијата автоматски го препознава дали разместувате на WP Engine и користи вградената WP Engine API. Не е потребно дополнително конфигурирање ако плагинот WP Engine е активен и правилно конфигуриран.

Меѓутоа, ако треба да ја конфигурирате интеграцијата ручно, можете да дефинирате една од овие константи во вашата `wp-config.php` датотека:

```php
define('WPE_APIKEY', 'ваша_api_ключ'); // Префациран метод
// ИЛИ
define('WPE_API', 'ваша_api_ключ'); // Алтернативен метод
```

## Инструкции за поставување {#setup-instructions}

### 1. Проверете плагинот WP Engine {#1-verify-wp-engine-plugin}

Ако разместувате на WP Engine, плагинот WP Engine веќе треба да биде инсталиран и активиран. Проверете дали:

1. Плагинот WP Engine е активен
2. Да постои датоката `wp-content/mu-plugins/wpengine-common/class-wpeapi.php`

### 2. Активирај ја интеграцијата {#2-enable-the-integration}

1. Во административниот панел на WordPress, отидете на Ultimate Multisite > Settings
2. Направите префрлање на табот "Domain Mapping" (Мапирање на домените)
3. Пролискувајте надолу до "Host Integrations" (Интеграции со хостови)
4. Активирајте интеграцијата со WP Engine
5. Кликнете на "Save Changes" (Зачувај промени)

## Како работи {#how-it-works}

### Синхронизирање на домените {#domain-syncing}

Кога домен е мапиран во Ultimate Multisite:

1. Интеграцијата користи WP Engine API за да го додаде доменот во вашата инсталација на WP Engine
2. WP Engine ја управува конфигурацијата на доменот и издавањето на SSL сертификат
3. Кога ќе се откаже мапирање на домен, интеграцијата ќе го откаже доменот од WP Engine

### Поддршка за субадомени {#subdomain-support}

За инсталации на поддомен мултисайт:

1. Интеграцијата го додава секој поддомен во WP Engine кога се создава нов сайт.
2. WP Engine ја управува конфигурацијата на поддомените.
3. Кога ќе се отстрани сайт, интеграцијата ќе го отстрани поддоменот од WP Engine.

## Важни забелешки {#important-notes}

### Дигитални доменни имена (Wildcard Domains) {#wildcard-domains}

За инсталации на мултисайт со поддомени, препорачуваме да контактирате WP Engine поддршка и побарате конфигурација за дигитално доменско име (wildcard domain). Ова овозможува сите поддомени да работат автоматски без потреба од тоа што ќе ги додадете секој по еден.

### SSL Сертификати {#ssl-certificates}

WP Engine автоматски го управува издавањето и обновено на SSL сертификатите за сите домени кои се додадени преку оваа интеграција. Не е потребна дополнителна конфигурација.

## Решавање проблеми (Troubleshooting) {#troubleshooting}

### Проблеми со врската на API {#api-connection-issues}
- Проверете дали WP Engine plugin е активен и правилно конфигуриран.
- Ако сте го дефинирале рачно API клучот, проверете дали е точен.
- Контактирајте ја поддршката на WP Engine ако имате проблеми со API.

### Домен не е додаден {#domain-not-added}
- Проверете ги логатите на Ultimate Multisite за некои пораки за грешки.
- Проверете дали доменet веќе не е додаден во WP Engine.
- Осигурајте се дека вашиот план на WP Engine поддржува бројот на домените што ги додавате.

### Проблеми со поддомените {#subdomain-issues}
- Ако поддомените не работат, контактирајте ја поддршката на WP Engine и побарајте конфигурација за дигитално доменско име (wildcard domain).
- Проверете дали вашите DNS поставки се правилно конфигурирани за главниот домен и поддомените.
