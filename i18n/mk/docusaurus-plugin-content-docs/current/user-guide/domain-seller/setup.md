# Продавач домен: Поставување и конфигурација на провидера

Додатокот Domain Seller испраќа со водич за поставување кој ви ќе ги преведе сите потребни чекори. Оваа страница ја опишува процесот на водичот и како да го конфигурирате или повторно да го конфигурирате провидерите по него.

## Потребијања

- **Multisite Ultimate** v2.4.12 или подоцна, активиран преку мрежа
- **PHP** 7.4+
- API податоци за најмалку еден поддржан регистратор

## Водич за прво користење

Водичот за поставување автоматски се покренува при првиот пат кога ќе активирате плагинот преку мрежа. Тој е достапен и во било кое време од **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Чекор 1 — Изберете провидер

Изберете регистрарот со кој сакате да се поврзете. Поддржани опции:

| Провидер | Управување на DNS | Приватност WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Не | Да (WhoisGuard, безплатно) |
| HostAfrica | Да | Да (Заштита на ID-то) |
| Openprovider | Да | Да |
| Hostinger | Преку мапирање на домените на хостирани домени за основен Hostinger домен | Да |
| GoDaddy | Не | Не |
| ResellerClub | Да | Не |
| NameSilo | Не | Не |
| Enom | Да | Не |

### Чекор 2 — Внесете податоци за пријавување

Секој провидер има различни текстови за пријавување:

**OpenSRS** — Корисничко име и приватна клучна (од OpenSRS Reseller Control Panel)

**Namecheap** — Корисничко име и API клуч (од Account › Tools › API Access)

**HostAfrica** — Крајна точка на API за реселер домените и податоци од модулот за реселер на HostAfrica. В момента нема посебна sandbox крајна точка документирана; тестирајте со безбедни читачки проверки пред да го започнете живи регистрации.

**Openprovider** — Корисничко име и лозинка со активиран API пристап. Опционален режим на sandbox користи Openprovider sandbox API, а опционална внатрешна идентификација на клиентот може да се повторно користи за регистрирањата.

**Hostinger** — Токен на shared Hostinger hPanel API од централната интеграција со Hostinger. Истаот токен го зачувува мапирањето на главни домените и операциите за регистрација на продавачите на домени.

**GoDaddy** — API ключ и тајн (од developer.godaddy.com)

**ResellerClub** — ID на реселер и API ключ (од контролниот панел на ResellerClub)

**NameSilo** — API ключ (од namesilo.com › Account › API Manager)

**Enom** — ID на аколувањето и API токен

Проверете **Sandbox mode** каде што е достапен за тестирање против тестовиот околината на провадирот пред да започнете со реални податоци.

### Чекор 3 — Тестирајте ја поврзаноста

Кликнете на **Test Connection**. Магијата испраќа лек API повик за проверка на податоците и коннективностите. Исправете ги сите проблеми со податоци пред да продолжиме.

### Чекор 4 — Импортирајте TLD-и

Кликнете на **Import TLDs** за да го преземете сите достапни TLD-и и опции за големи купувачи од поврзаниот провадирот. Ова пополнува листата на TLD-и што се користи за домен продуктот. Импортот може да трае 30–60 секунди за провадири со големи каталоги на TLD-и.

TLD-и се автоматски ресинхронизираат и неколку пати во текот на денот преку планиран cron job.

### Чекор 5 — Создадете домен продукт

Магијата создава еден стандарден домен продукт со 10% маржа за продажба (markup). Можете да го уредувате овој продукт веднаш или да пропуштите и да креирате продукти ручно под **Ultimate Multisite › Products**.

Видете [Domain Products and Pricing](./domain-products) за целосен водич за конфигурација на продуктот.

## Реконфигурирање на провадирот

Отидете на **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или кликнете на **Settings** во списокот на плагини).

Страницата со поставки вклучува:

- **Вклучување продажба на домени** — прекинете оваа функција на вкл/и-од.
- **Дефолтен провајдер** — провајдерот што се користи за пребарување домени и нови производи.
- **Максимален број TLD-ови по пребарување** — колку TLD-ови да проверите кога клиент бара; поголемите вредности покажуваат повеќе опции, но се побавни.
- **Траење на кеширање на достапност** — колку време да кеширате резултатите за достапност и цени; пониските вредности се попрецизни, но зголемуваат API повици.
- **Управување со домен производи** — брза линк кон списокот на производите.
- **Конфигурирај провајдери** — отвора Модичен асистент (Integration Wizard) за додадување или преконфигурирање провајдери.

### Додавување на вториот провајдер

Кликнете на **Configure providers** и повторно извршете асистентот за новиот регистратор. Можете да имате повеќе конфигурирани провајдери истовремено. Потчете секој домен продукт со специфичен провајдер, или оставете го на дефолтен.

### Ручно синхронизирање на TLD-ови

На страницата за поставки, кликнете на **Sync TLDs** до било кој конфигуриран провајдер за да повлечете најновите цени. Ова е корисно откако некој провајдер ќе ажурира вкупните цени или додаде нови TLD-ови.

## Логи

Секој провајдер пишува во своја сопствена канален логи. Лога се гледа под **Network Admin › Ultimate Multisite › Logs**:

| Канал за логирање | Содржина |
|---|---|
| `domain-seller-registration` | Сите проби на регистрација (успешни и неуспешни) |
| `domain-seller-renewal` | Резултати од работата за обновено |
| `domain-seller-opensrs` | Сирова активност на OpenSRS API |
| `domain-seller-namecheap` | Сирова активност на Namecheap API |
| `domain-seller-hostafrica` | Сирова активност на HostAfrica API |
| `domain-seller-openprovider` | Сирова активност на Openprovider API |
| `domain-seller-hostinger` | Сирова активност на Hostinger API |
| `domain-seller-godaddy` | Сирова активност на GoDaddy API |
| `domain-seller-resellerclub` | Сирова активност на ResellerClub API |
| `domain-seller-namesilo` | Сирова активност на NameSilo API |
| `domain-seller-enom` | Сирова активност на Enom API |

## Забелешки за капацитетите на провајдерите

Некој регистратор API не ја отстранува истото функционирање. Додатот (addon) прикажува неподржани операции со јасни грешки што се очекуваат од администраторот, наместо да ги прекине тишине.

- **HostAfrica** поддржува најшироко работно процесот за реселџери, вклучувајќи пребарување (lookup), синхронизација на TLD/цените, регистрација, обнова, префрлање, ажурирање на имена на серверите (nameserver updates), DNS записи, EPP кодови, заштита на регистраторски лок и заштита на ИД.
- **Openprovider** поддржува синхронизација на TLD цени за реселџери, регистрација, обнова, префрлања, ажурирање на имена на серверите, DNS зони, EPP кодови, заштита на регистраторски лок и приватност на WHOIS. Тој се автентифицира со краток токен (bearer token) со краткотраен рок што додатот го освежува автоматски.
- **Hostinger** поддржува пребарување на достапност, регистрација, преглед на портфолиото, ажурирање на имена на серверите, заштита на регистраторски лок и приватност на WHOIS преку споделен токен на hPanel API. Публичниот API на Hostinger не ја отстранува реселџерската/wholesale цени, влезни префрлања (inbound transfer), експлицитна обнова или добивање EPP код; обновите се само автоматски обновени.
