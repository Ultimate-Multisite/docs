# Поле за плаќање и управување со DNS на клиентот {#checkout-field-and-customer-dns-management}

## Поле за избор на домен {#the-domain-selection-checkout-field}

Полето **Domain Selection** е елемент во процесот на плаќање кој дава на клиентите избор како да добијат свој домен. Додадете го во било која форма за плаќање за да овозможите продажба на domeni.

### Додавање на полето во форма за плаќање {#adding-the-field-to-a-checkout-form}

1. Одете на **Network Admin › Ultimate Multisite › Checkout Forms**
2. Отворете или создадете форма за плаќање
3. Во редактор за плаќање, кликнете на **Add Field** (Додади поле)
4. Изберете **Domain Selection** од списокот на полета
5. Конфигурирајте ги опциите на полето (прегледајте подолу)
6. Зачувајте ја формата

### Опции за полето {#field-options}

**Domain modes** — Изберете кои табови ќе ги види клиентите. Секој режим може да се активира или деактивира независно:

| Режим | Што прави |
|---|---|
| **Subdomain** (Субдомен) | Клиентот користи безбеслен субдомен на вашата мрежа (на пр. `mysite.yournetwork.com`). Не е потребно плаќање. |
| **Register New Domain** (Регистрира нов домен) | Клиентот бара нов домен и го регистрира преку вашиот конфигуриран провајдер. Користи соодветниот продукт за цени на цена. |
| **Existing Domain** (Существувачки домен) | Клиентот мапи домен што веќе го има. Нема такса за регистрација. Домен е автоматски поврзан со неговиот сайт. |

**Default mode** (Дефолт режим) — Кога си активирани си трите режими, кој таб ќе се отвори прв. Поставете на **Subdomain** за да го оставите регистрирањето на домен опционално, или **Register New Domain** за да ги поттикнете купувањата.

**Domain product** (Продукт за домен) — Опционално, приврзете ова поле на специфичен продукт за домен. Ако не е поставен, аддон ќе го избере автоматски соодветниот продукт врз основа на TLD што го пребарува клиентот.

### Поле за контакт на регистрирањето {#registrant-contact-fields}

Кога клиент ќе избере табот **Register New Domain**, форма за плаќање ќе додаде полета за контакт на регистрирањето во рамките на формата:

- Име / Презиме
- Е-пошта
- Адреса (линија 1, град, држава/провинција, поштаска локација, земја)
- Телефонски број

Ови се бараат од сите регистрирања и валидирани пред да се направи повик за API регистрација. Телефонските броеви автоматски се форматираат во меѓународниот формат `+CC.NNN` што го очекуваат регистрираните.

### Автоматско генериран URL на веб-страницата {#auto-generated-site-url}

Кога корисник се регистрира или мапи домен, полето за URL на веб-страницата автоматски се пополнува со избраниот домен. Корисниците не мора да пополнуваат посебно поле за URL.

### Посебно однесување на пребарување (Search behaviour) {#search-behaviour}

- Достапноста на домените се проверува во реално време со AJAX додека корисникот пишува
- Се прикажуваат алтернативни предлози за TLD кога преферираното домен не е достапно
- Цените се земаат живо и ја прикажуваат јасно (цена за регистрација, цена за обнова, опционална такса за приватност WHOIS)
- Кодови за купони се применуваат на домен продуктот како и на другите продукти

**Нагласување на одговорноста на пребарување:**

```php
// Зголеми ја затезката (debounce delay) во милисекули за намалување на повик на API при бавни поврзувања
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // дефолт: 500
});
```

**Додавување на кастом полета во формот за пребарување на домени:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Дополнителни забелешки',
    ];
    return $fields;
});
```

## Управување на DNS за корисниците {#customer-dns-management}

Корисниците можат да управуваат со записи на DNS за своите регистрирани домени од страницата **Моја сметка**, под влогот на нивниот домен.

### Поддржани типови на записи {#supported-record-types}

| Тип | Користење |
|---|---|
| **A** | Мапи го името на хостот на IPv4 адрес |
| **AAAA** | Мапи го името на хостот на IPv6 адрес |
| **CNAME** | Создава алijas што укажува на друг име на хост |
| **MX** | Поставува сервер за пошта (mail exchange server) |
| **TXT** | Додава SPF, DMARC, верификација или други текстови записи |

### Кои провајдери поддржуваат управување со DNS? {#which-providers-support-dns-management}

Управу на DNS записи (додавање, уредување, брисање) можете извршувајте со **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** и **Openprovider**. Домените од **Hostinger** можат да ги ажурираат имена на DNS серверите преку Domain Seller; DNS записи за хостирани домени се управуваат преку вградената интеграција Hostinger domain-mapping. Домените од Namecheap, GoDaddy и NameSilo прикажуваат статус и информации за истекот, но DNS треба да се управува директно во контролниот панел на регистраторот.

### Дефолтни DNS записи {#default-dns-records}

Можете да конфигурирате дефолтни DNS записи кои автоматски се применуваат кога домен е регистриран. Одете на **Settings › Domain Seller › Default DNS Records**.

Вредностите на дефолтните записи поддржуваат два токена:

| Токен | Заменето со |
|---|---|
| `{DOMAIN}` | Регистрираниот име на доменот (на пр. `example.com`) |
| `{SITE_URL}` | URL-от на WordPress сайтот за клиентот |

**Пример — насочување на apex и www кон IP-то на вашиот сервер:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: гледање и уредување на DNS {#admin-viewing-and-editing-dns}

Мрежни администратори можат да ги видат и да ги уредуваат DNS записи за било кој клиентски домен од страницата за уредување на доменот во **Network Admin › Ultimate Multisite › Domains**.
