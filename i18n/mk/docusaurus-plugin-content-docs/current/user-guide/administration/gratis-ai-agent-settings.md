# Настройки на Безплатен AI Агент

Екрано **Settings → Advanced** во Gratis AI Agent нулира административни конфигурации за интеграциите на бекендот во внатрешното подобружение v1.5.0. Оваа страница документира полетата **Feedback Endpoint** и нивниот очекуван формат.

## Достигнување до Настройките

1. Во администрацијата на WordPress, отидете на **Gratis AI Agent → Settings**.
2. Кликнете на табот **Advanced**.

## Конфигурирање на Feedback Endpoint-от

Feedback endpoint прифаќа POST барања од AI агентот секогаш кога корисникот ќе испрати повратна информација преку копчето со палец надолу (thumbs-down), банерот за автоматски промпт или командата `/report-issue`.

| Поле | Опис |
|---|---|
| **Feedback Endpoint URL** | URL што прифаќа податоци за повратна информација како HTTP POST барања со JSON тело. |
| **Feedback API Key** | Bearer токен што се испраќа во `Authorization` хедерот на секој потреби за повратна информација. Оставете празно ако вашиот endpoint не бара автентикација. |

### Очекуван JSON Payload

Вашиот feedback endpoint мора да прифати JSON тело со најмалку следниве полета:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Одговорот беше неточен во однос на цените.",
  "triage_category": "factual_error"
}
```

Дополнителни полета можат да присутни во payload-от во зависност од контекстот на разговорот.

### Вредности за `triage_category`

Слојот за триаж (triage layer) на AI им доделува една од следниве вредности на `triage_category` пред да го препрати payload-от:

| Вредност | Значење |
|---|---|
| `factual_error` | Асистентот понуди неточни фактички информации. |
| `unhelpful_answer` | Одговорот беше технички точен, но не корисен. |
| `inappropriate_content` | Одговорот содржеше содржини кои не треба да се прикажуваат на корисниците. |
| `other` | Повратната информација не одговара на позната категорија. |

### Автентикација

Ако вашиот endpoint бара автентикација, поставите полето **Feedback API Key** со вашиот bearer токен. Агентот ќе испрати:

```
Authorization: Bearer <your-api-key>
```

Ако полето **Feedback API Key** е празно, нема да се испрати ниту `Authorization` header.

### Деактивирање на собирање обратна врска (Feedback Collection)

Оставите и полето **Feedback Endpoint URL** и **Feedback API Key** празни. Бутон со палец надолу (thumbs-down) и UI за обратна врска остануваат видливи за корисниците, но податоците не се препраќаат ниту ка некоја надворешна услуга.

## Brave Search API Key

Исто така на табот **Advanced**, полето **Brave Search API Key** овозможува можноста за [Internet Search](../configuration/internet-search).

| Поле | Опис |
|---|---|
| **Brave Search API Key** | Вашата API клуч од датотечен панел на развивачите на Brave Search. Неопходно е за да се активира интернет пребарување во AI асистентот. |

Етикета на полето вклучу кликаби линк до страницата за регистрација на Brave Search API. Оставите празно за да го деактивирате интернет пребарување.

За детали за оваа функција, видите [Internet Search](../configuration/internet-search) за документација за крајни корисници.

## Feature Flags (Флаги на функции)

Исто така во внатрешност во v1.9.0, табот **Settings → Feature Flags** нуди прекиSwitch-и за опционална функционалност. Секо флаг е абониран или деактивиран на целата мрежа; во моментот нема надградба по локација (per-site override).

### Достапување до Feature Flags

1. Во администрацијата на WordPress, отидете на **Gratis AI Agent → Settings**.
2. Кликнете на табот **Feature Flags**.

### Флаги за контрола на пристап (Access Control Flags)

| Флаг | Дефолт | Опис |
|---|---|---|
| **Ограничување за администратори** | Итно (Off) | Кога е активиран, само корисниците со ролите `administrator` можат да отворат панелот за чат на AI Agent. Сите останати ролови ќе видат порака „Контактирајте го вашиот администратор“. |
| **Ограничување за мрежни администратори** | Итно (Off) | Кога е активиран во мрежа со многу страници, само Супер Администраторите можат да користат агентот. Индивидуалните администратори на локални страници се блокираат. Ова има приоритет пред „Ограничување за администратори“ ако и двата зборуваат. |
| **Овозможување пристап за претплатители** | Итно (Off) | Кога е активиран, корисниците со ролите `subscriber` можат да користат интерфејсот за чат, но се ограничуваат на читање-скина (без создавање постови или промена на поставки). |
| **Деактивирање за не-придружителни** | Итно (Off) | Интегрира со статусот на членство во Ultimate Multisite. Кога е активиран, чатот ќе биде скриен за страници кои немаат активна претплата. |

### Флаги за брендирање

| Флаг | Дефолт | Опис |
|---|---|---|
| **Скривање на футер „Помогујте со Gratis AI Agent“** | Итно (Off) | Се отстранува линијата за брендирање што се прикажува на подолу на чат виджетот. Препорачливо за деплојменти со бела етикета (white-label). |
| **Приказната на агентот** | *(празен)* | Заместува ја дефолт ознаката „Gratis AI Agent“ во главниот дел на чатот и административниот мени со вашето име на продукт. Оставете празно за да користите дефолтното. |
| **Скривање на изборот на агенти** | Итно (Off) | Кога е активиран, корисниците не можат да префрлат помеѓу пет вградени агенти. Тековниот агент е фиксиран на тоа што е конфигурирано како дефолт во Settings → General. |
| **Користење на иконата на страната како аватар за чат** | Итно (Off) | Заместува ја дефолтна AI икона во главниот дел на чатот со иконата на WordPress страната (поставување под Appearance → Customize → Site Identity). |

### Примена на промени

Кликнете на **Save Settings** по да префрлите било кој флаг. Промените влијаат веднаш — не е потребно да се чисти кешот или да се реактивира плагинот.
