# Поставки за Gratis AI Agent {#gratis-ai-agent-settings}

Екранот **Settings → Advanced** во Gratis AI Agent обезбедува конфигурација на администраторско ниво за backend интеграции. Оваа страница документира препраќање повратни информации, клучеви за даватели на пребарување, поставување на управувана Superdav услуга, контроли за Google Calendar, TextBee SMS поставки и feature flags на ниво на мрежа.

## Пристапување до поставките {#accessing-settings}

1. Во WordPress администрацијата, одете на **Gratis AI Agent → Settings**.
2. Кликнете на табот **Advanced**.

## Конфигурација на endpoint за повратни информации {#feedback-endpoint-configuration}

Endpoint-от за повратни информации прима POST барања од AI agent секогаш кога корисник поднесува повратна информација преку копчето со палец надолу, банерот за автоматски поттик или командата `/report-issue`.

| Поле | Опис |
|---|---|
| **Feedback Endpoint URL** | URL што прима поднесоци со повратни информации како HTTP POST барања со JSON тело. |
| **Feedback API Key** | bearer token испратен во `Authorization` header на секое барање за повратни информации. Оставете празно ако вашиот endpoint не бара автентикација. |

### Очекуван JSON payload {#expected-json-payload}

Вашиот endpoint за повратни информации мора да прифаќа JSON тело со најмалку следниве полиња:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Во payload-от може да бидат присутни дополнителни полиња, во зависност од контекстот на разговорот.

### Вредности за `triage_category` {#triagecategory-values}

AI слојот за тријажа доделува една од следниве вредности на `triage_category` пред да го препрати payload-от:

| Вредност | Значење |
|---|---|
| `factual_error` | Асистентот обезбеди неточни фактички информации. |
| `unhelpful_answer` | Одговорот беше технички точен, но не беше корисен. |
| `inappropriate_content` | Одговорот содржеше содржина што не треба да им се прикажува на корисниците. |
| `other` | Повратната информација не се совпаѓаше со позната категорија. |

### Автентикација {#authentication}

Ако вашиот endpoint бара автентикација, поставете го полето **Feedback API Key** на вашиот bearer token. Агентот испраќа:

```
Authorization: Bearer <your-api-key>
```

Ако полето **Feedback API Key** е празно, не се испраќа `Authorization` header.

### Оневозможување собирање повратни информации {#disabling-feedback-collection}

Оставете ги празни и полето **Feedback Endpoint URL** и полето **Feedback API Key**. Копчето со палец надолу и UI за повратни информации остануваат видливи за корисниците, но поднесоците не се препраќаат до ниту една надворешна услуга.

## Brave Search API Key {#brave-search-api-key}

Исто така на табот **Advanced**, полето **Brave Search API Key** ја овозможува способноста [Интернет-пребарување](../configuration/internet-search).

| Поле | Опис |
|---|---|
| **Brave Search API Key** | Вашиот API клуч од Brave Search developer dashboard. Потребен е за да се овозможи интернет-пребарување во AI асистентот. |

Ознаката на полето вклучува линк на кој може да се кликне до страницата за регистрација за Brave Search API. Оставете празно за да го оневозможите интернет-пребарувањето.

Видете [Интернет-пребарување](../configuration/internet-search) за документација за крајни корисници за оваа функција.

## Управувана Superdav услуга {#managed-superdav-service}

Superdav AI Agent v1.18.0 додава управувани Superdav service endpoints и автоматско обезбедување врска за поддржани сајтови. Користете ги овие контроли кога вашиот сајт треба да се поврзе со хостираниот давател наместо со рачно конфигуриран service endpoint.

| Поле | Опис |
|---|---|
| **Managed Superdav Service** | Ја овозможува хостираната врска со Superdav услуга за поддржани сајтови. |
| **Provision Connection** | Започнува автоматско обезбедување endpoint и акредитиви. Користете го ова откако ќе потврдите дека сајтот треба да го користи управуваниот давател. |
| **Service Endpoint / Connection Status** | Го прикажува тековниот endpoint или состојбата на врската по обезбедувањето. |

По обезбедувањето, зачувајте ги поставките и проверете ја состојбата на врската пред да се потпрете на работни текови со управувана услуга. Ако обезбедувањето не успее, прегледајте ги прикажаните насоки за повторен обид и потврдете дека сајтот има дозвола да го користи хостираниот давател.

## Конфигурација на Google Calendar {#google-calendar-configuration}

Кога календарските функции на Superdav AI Agent v1.18.0 се овозможени, агентот може да чита конфигурирани календари и детали за настани. Календарските алатки се ориентирани кон читање и се корисни за потсетници свесни за распоред, follow-up со учесници и совпаѓање контакти.

| Поле | Опис |
|---|---|
| **Google Calendar Credentials** | Ги складира акредитивите или token врската потребни за читање календарски податоци. |
| **Calendar Selection** | Ограничува кои конфигурирани календари агентот смее да ги прегледува. |
| **Calendar Connection Status** | Потврдува дали тековните акредитиви можат да читаат календари и настани. |

Ограничете ги календарските акредитиви само на календарите што му се потребни на агентот. Повторно поврзете или ротирајте ги акредитивите ако состојбата укажува на истечен token.

## TextBee SMS известувања {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 додава TextBee како SMS давател за конфигурирани работни текови за известувања. SMS известувањата треба да се спарат со контролни точки за човечко одобрување за чувствителни пораки или пораки насочени кон корисници.

| Поле | Опис |
|---|---|
| **TextBee API Key** | Автентицира барања до TextBee SMS давателот. |
| **TextBee Device / Sender** | Го избира TextBee испраќачот или уредот што се користи за излезни пораки, кога тоа го бара давателот. |
| **SMS Notifications Enabled** | Им дозволува на одобрените работни текови да испраќаат известувања со текстуални пораки. Оставете оневозможено за да спречите испраќање SMS. |

Испратете тест-порака само на број во сопственост на администратор, потоа потврдете го однесувањето на контролната точка за одобрување пред да овозможите закажани потсетници или потсетници насочени кон учесници.

## Feature Flags {#feature-flags}

Исто така воведен во v1.9.0, табот **Settings → Feature Flags** обезбедува прекинувачи за вклучување/исклучување на опционална функционалност. Секој flag е или овозможен или оневозможен на ниво на мрежа; во овој момент нема override по сајт.

### Пристапување до Feature Flags {#accessing-feature-flags}

1. Во WordPress администрацијата, одете на **Gratis AI Agent → Settings**.
2. Кликнете на табот **Feature Flags**.

### Flags за контрола на пристап {#access-control-flags}

| Ознака | Стандардно | Опис |
|---|---|---|
| **Ограничи на администратори** | Исклучено | Кога е овозможено, само корисници со улога `administrator` можат да го отворат панелот за разговор со AI Agent. Сите други улоги наместо тоа гледаат порака „Контактирајте го вашиот администратор“. |
| **Ограничи на Network Admins** | Исклучено | Кога е овозможено на multisite мрежа, само Super Admins можат да го користат агентот. Администраторите на поединечни сајтови се блокирани. Има предност пред „Ограничи на администратори“ ако и двете се овозможени. |
| **Дозволи пристап за претплатници** | Исклучено | Кога е овозможено, корисници со улога `subscriber` можат да го користат интерфејсот за разговор, но се ограничени на можности само за читање (без создавање објави или промени на поставки). |
| **Оневозможи за не-членови** | Исклучено | Се интегрира со статусот на членство на Ultimate Multisite. Кога е овозможено, разговорот е скриен за сајтови што немаат активно членство. |

### Ознаки за брендирање {#branding-flags}

| Ознака | Стандардно | Опис |
|---|---|---|
| **Сокриј го подножјето „Powered by Gratis AI Agent“** | Исклучено | Ја отстранува линијата за брендирање прикажана на дното на виџетот за разговор. Се препорачува за white-label имплементации. |
| **Прилагодено име на агентот** | *(празно)* | Ја заменува стандардната ознака „Gratis AI Agent“ во заглавието на разговорот и администраторското мени со вашето сопствено име на производ. Оставете празно за да се користи стандардното. |
| **Сокриј го избирачот на агенти** | Исклучено | Кога е овозможено, корисниците не можат да се префрлаат меѓу петте вградени агенти. Тековниот агент е фиксиран на она што е конфигурирано како стандардно во Settings → General. |
| **Користи ја иконата на сајтот како аватар за разговор** | Исклучено | Ја заменува стандардната AI икона во заглавието на виџетот за разговор со WordPress иконата на сајтот (поставена под Appearance → Customize → Site Identity). |

### Ознаки за безбедност на автоматизацијата {#automation-safety-flags}

Superdav AI Agent v1.18.0 воведува порти за човечко одобрување и записи за потсетници за побезбедни извршувања на автоматизација. Овие контроли може да се појават во ознаките за функции или напредните поставки за автоматизација, во зависност од инсталираниот пакет.

| Ознака | Стандардно | Опис |
|---|---|---|
| **Бара човечко одобрување** | Препорачано вклучено | Ги паузира чувствителните автоматизации додека овластен корисник не ја прегледа и потврди предложената акција. |
| **Дедупликација на потсетници** | Вклучено | Ги евидентира испратените потсетници, така што повторните обиди или закажаните извршувања не испраќаат дупликат известувања. |
| **Овозможи алатки за календар** | Исклучено додека не се конфигурира | Му дозволува на агентот да чита конфигурирани Google календари и настани. |
| **Овозможи SMS известувања** | Исклучено додека не се конфигурира | Им дозволува на одобрените работни текови да испраќаат TextBee SMS известувања откако акредитивите ќе се зачуваат. |

### Применување промени {#applying-changes}

Кликнете **Зачувај поставки** по вклучување или исклучување на која било ознака. Промените стапуваат во сила веднаш — не е потребно празнење на кешот или повторно активирање на plugin.
