# Познати несовместимости со други плагини

По после инсталирање и активирање на Ultimate Multisite, може да имате проблеми поради нерешени несовместимости со други плагини.

Активно работиме на намалување на оваа листа со секој нов издадење, но ако некој плагин е на овој листа, можеби ќе треба да го деактивирате за да имате целосно функционална Ultimate Multisite Мрежа.

Еве ја листата и што треба да направите во такви случаи:

**Некои WPMU DEV плагини** Некои плагини кои повеќе не се поддржат од WPMU DEV се познати дека можат да го застанат процесот на регистрација на Ultimate Multisite на последниот чекор (корисникот не може да создаде акаунт). Еве листа на плагините што предизвикуваат овие проблеми:

  * Signup Language: _Ultimate Multisite има дополнение кое нуди иста функционалност, наречено Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Ова е вградено во Ultimate Multisite, па нема потреба од дополнителен плагин._

  * New Blog Templates: _Ова е вградено во Ultimate Multisite, па нема потреба од дополнителен плагин._

  * Set Password on Multisite Blog Creation: _Ова е вградено во Ultimate Multisite, па нема потреба од дополнителен плагин._

  * Signup TOS: _Ова е вградено во Ultimate Multisite, па нема потреба од дополнителен плагин._

  * Anti-Splog: Иако е помалку моќен, тој е и _вграден во Ultimate Multisite._

**Забелешка**: Сите останати WPMU Dev плагини може да се користат нормално заедно со Ultimate Multisite. Примери вклучуваат _Smush_ , _Forminator_ , _Defender_, итн.
