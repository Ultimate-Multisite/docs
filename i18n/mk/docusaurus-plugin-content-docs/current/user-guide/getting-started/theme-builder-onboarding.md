# Поток за почетно наставување за Theme Builder {#theme-builder-onboarding-flow}

Superdav AI Agent v1.12.0 воведува насочен **поток за почетно наставување за Theme Builder** кој ви помага да креирате кастомна тема со блокови во време на вашата првична поставка. Ова го заменува старата модален режим Site Builder со пофлексибилен пристап под асистент на агентот.

## Што е Потокот за почетно наставување за Theme Builder? {#what-is-the-theme-builder-onboarding-flow}

Потокот за почетно наставување за Theme Builder е интерактивен магичен прибор (wizard) кој:

- Ви води низ одлуките за дизајн (бои, типографија, расположение)
- Се собира вашите преференции за визуелната идентичност на вашиот сайт
- Генерализира кастомна тема со блокови подделена според вашите потреби
- Автоматски активира ја темата кога ќе заврши

Потокот се вклучува со **агентот Setup Assistant**, кој поставува разјасни прашања и гради вашата тема поетапно.

## Започнување на Потокот за почетно наставување за Theme Builder {#starting-the-theme-builder-onboarding}

### Почетна поставка (First-Run Setup) {#first-run-setup}

Кога први пати ќе го запустите Superdav AI Agent на нов WordPress инсталација, ќе видите:

```
Добредојдовте во Superdav AI Agent!

Што би сакале да направите?
1. Создадете кастомна тема (Theme Builder)
2. Користете постоечка тема
3. Прескокајте поставките за време на оваа работа
```

Изберете **"Создадете кастомна тема"** за да влезете во потокот за почетно наставување за Theme Builder.

### Ручна активација (Manual Activation) {#manual-activation}

Можете и да започнете Поток за почетно наставување за Theme Builder било кога со барање:

```
"Започни го Потокот за почетно наставување за Theme Builder"
```

или

```
"Помогнете ми да креирам кастомна тема"
```

## Чекори на Наставувањето (The Onboarding Steps) {#the-onboarding-steps}

### Ст aп 1: Избор на режим (Mode Selection) {#step-1-mode-selection}

Агентот Setup Assistant поставува прашање за вашата преференца:

```
Како би сакале да ја креирате вашата тема?
- Воведен (Ќе ви ќе прашувам и ќе ја градам за вас)
- Практичен (Ќе ви покажам опции а вие ќе решите)
```

**Воведениот режим** се препорачува за повеќето корисници; агентот дава предлози за дизајн врз основа на вашата индустрија и цели.

### Ст aп 2: Спецификација на сайтот (Site Specification) {#step-2-site-specification}

Ќе ви биде прашано за вашиот сайт:

- **Цел на сајтот**: Е-комерцијална платформа, блог, портфолио, SaaS итн.
- **Циљна публика**: Кои се посетителите?
- **Борови бои**: Примарни и вторични бои (или „не знам“)
- **Тон**: Професионален, креативен, играчки, минималистичен итн.

Ова информации се чуваат во вашата меморија под име **site_brief**, на која агентите ќе се одземат во идните сесии.

### Чекор 3: Одлуки за Дизајн Системet (Design System) {#step-3-design-system-decisions}

Агентот ќе ви води низ изборите за дизајн токени:

- **Типографија**: Семејство на шрифтови (serif, sans-serif, monospace) и скала на големини
- **Палета на бои**: Примарни, вторични, акцентни и неутрални бои
- **Простор (Spacing)**: Компактни, нормални или просторни дизајни
- **Крење (Motion)**: Анимации и прелади (ако е потребно)

### Чекор 4: Генерација на Тема (Theme Generation) {#step-4-theme-generation}

Агентот за помош (Setup Assistant agent) ќе ви го построи вашиот кастом блок тема со:

- `theme.json` што содржи сите вашите дизајн токени
- Шаблони за блокови за чести типови (поглавје, блог, контакт)
- Кастом стилови на блокови кои одговараат на вашиот дизајн систем
- Метаданни за темата и декларации за поддршка со WordPress

### Чекор 5: Активација и Проверка {#step-5-activation-and-verification}

Темата се активира автоматски, а ви ќе видите:

```
✓ Вашата кастом тема е активна!
  Име на темата: [Ваше Име на Сајтот] Theme
  Бои: [Примарни] / [Вторични]
  Типографија: [Семејство на шрифтови]

  Посетете го вашиот сайт за да видите новиот дизајн.
```

Потоа можете да посетите својот сайт за да проверите дали темата се прикажува правилно.

## Специфика на Сајтот и Меморијата site_brief {#site-specification-and-sitebrief-memory}

Во процесот на вклучување (onboarding), агентот ќе го добие спецификацијата на вашиот сайт во категорија под име **site_brief** меморија. Ова вклучува:

- Цел и цели на сајтот
- Циљна публика
- Борови бои и тон
- Дизајн преференции
- Структура на содржината

### Зошто site_brief е Важен {#why-sitebrief-matters}

Во идните сесии, агентите ќе го користат вашиот site_brief за:

Зачувај конзистентност на дизајнот при промените
Предложете функции кои се усогласени со целта на вашиот сайт
Давај контекстуални препораки
Избегнувај повторување на прашања за поставување

### Преглед на вашата site_brief {#viewing-your-sitebrief}

Можете да прашате на агентот:

```
"Покажи ми моето site brief"
```

или

```
"Што знаеш за мојот сайт?"
```

Агентот ќе прикаже вашето зачувано спецификација на сайтот.

## Прилагодување по вработување (Onboarding) {#customizing-after-onboarding}

По завршувањето на вработување со Theme Builder, можете да:

### Користи вештината Design System Aesthetics Skill {#use-the-design-system-aesthetics-skill}

Забарајте подобрувања на дизајнот:

```
"Подобри ја типографијата да биде по модерна"
```

или

```
"Прилагоди ја палета на бои да биде топла"
```

**Вештината Design System Aesthetics skill** ќе ви води низ насочени подобрувања на дизајнот.

### Редацирајте директно theme.json {#edit-themejson-directly}

За напредна корисници, редактирајте `/wp-content/themes/[theme-name]/theme.json` за да го промените:
- Color tokens (токен на бои)
- Typography scales (малаци на типографијата)
- Spacing values (вреди на разминување)
- Border and shadow definitions (дефиниции на граници и сенки)

### Создадете кастом шаблони за блокови (Custom Block Templates) {#create-custom-block-templates}

Користите WordPress block editor за да создадете кастом шаблони за:
- Homepage layouts (дизајн на главната страница)
- Blog post pages (страници за блог постови)
- Product pages (страници за производи)
- Contact forms (форми за контакт)

## Сравна анализа: Старо против Нов вработување {#comparison-old-vs-new-onboarding}

| Функција | Site Builder (Старо) | Theme Builder (Нов) |
|---|---|---|
| Метод на поставување | Форма базирана на Wizard | Разговор водичен од агентот |
| Генерација на теми | Ограничени шаблони | Кастом скелет (scaffolding) |
| Дизајн токени | Ручна вметнување | Водички одлуки |
| Еластичност | Фиксирани опции | Прилагодувачки |
| Порака за подобрување | Не се референцира | Зачувана во site_brief |

## Решавање проблеми (Troubleshooting) {#troubleshooting}

**Вработувањето не е завршено**
- Почините го процесот: "Start the Theme Builder onboarding" (Започни со вработување на Theme Builder)
- Проверете дали вашата WordPress инсталација е ажурирана
- Проверете дали агентот Setup Assistant е активиран

**Мој `site_brief` не се користи**
- Провери дали агентот има пристап до меморијата
- Потребаш да побараш од агентот: "повтори мој `site brief`"
- Провери дали е овозможена меморијата во вашите поставки

**Генерираниот тема не совршава моите преференции**
- Користите вештината Design System Aesthetics за фино нагласување
- Потребаш да побараш од агентот: "повтори генерирање на темата со [конкретни промени]"
- Редацирајте `theme.json` директно за прецизно контролирање

## Следен чекори {#next-steps}

По завршувањето на процесот на вградување (onboarding) на Theme Builder:

1. **Провери го вашиот сайт**: Посетите го својот сайт за да видите новата тема
2. **Фино нагласувај го дизајнот**: Користите вештината Design System Aesthetics за корекции
3. **Додадете содржина**: Започнете да градите содржината на вашиот сайт
4. **Истражете способностите**: Научете за други способности на агентот како `scaffold-block-theme` и активирање на темата (`activate-theme`)
