# Навмешност на спецификацијата на сайтот (Site Specification Skill)

Вештината за **Спецификација на сайтот** е структуриран пристап до собирање на целите на вашиот сайт, публиката и идентичноста на брендот. Ова информации се чуваат во вашата меморија `site_brief`, што агентите го користат помеѓу сесиите за да обезбедат конзистентна помош и знаење за контекстот.

## Што е Спецификација на сайтот? {#what-is-site-specification}

Спецификацијата на сайтот е процесот на документирање на:

- **Целта на сайтот**: што прави вашиот сайт и зошто постои
- **Целната публика**: кој го посетува вашиот сайт и што му треба
- **Идентитетот на брендот**: вашите бои, тон и визуелен стил
- **Бизнис целите**: како изгледа успехот за вашиот сайт
- **Структурата на содржината**: како е организиран вашиот сайт

Ова спецификација станува вашата `site_brief`, персистентна меморија што агентите ја користат за да разберат контекстот на вашиот сайт.

## Зошто да користите Спецификација на сайтот? {#why-use-site-specification}

### Конзистентност помеѓу сесиите {#consistency-across-sessions}

Без `site_brief`, ќе морате повторно да објаснувате целта на вашиот сайт секој пат кога почнувате нова сесија. Со него, агентите веднаш разбираат:

- Целите и публиката на вашиот сайт
- Вашите бои и тон на брендот
- Структурата на вашата содржина
- Вашиот бизнис цели

### Подобри препораки {#better-recommendations}

Агентите ја користат вашата `site_brief` за:

- Предлагање функции кои се усогласени со целта на вашиот сайт
- Препорачување структури на содржина што одговара на вашите цели
- Предлагање дизајни кои се конзистентни со вашиот бренд
- Избегнување на предложење несовместими функции

### Побрзо вклучување (Onboarding) {#faster-onboarding}

Новите агенти (или агентите во нови сесии) може брзо да добијат знаење користејќи ја вашата `site_brief` наместо да прашуваат за појаснувачки прашања.

## Почнување на Спецификацијата на сайтот {#initiating-site-specification}

### Во процесот на вклучување (Onboarding) на Theme Builder-от {#during-theme-builder-onboarding}

Вештината за Спецификација на сайтот автоматски се иницира во **процесот на вклучување на Theme Builder**. Агентот Setup Assistant поставува прашања и гради вашата `site_brief`.

### Ручно иницирање {#manual-initiation}

Можете да започнете со спецификација на сайта во било кога:

```
"Да дефинирам мојата спецификација на сайт"
```

или

```
"Помогнете ми да создадам бриф за сайт"
```

## Процес на Спецификација на Сајтот {#the-site-specification-process}

### Чекор 1: Цел на Сајтот {#step-1-site-purpose}

Агентот прашува:

```
Какова е основната цел на вашиот сайт?
- Е-комерски магазин
- Блог или сопствена страница за содржини
- Портфолио или изложување дела
- SaaS апликација
- Заедница или форум
- Друго: [опишете]
```

Можете да изберете категорија или да опишете своја цел.

### Чекор 2: Целна Аудиторија {#step-2-target-audience}

```
Кој е вашиот примарна аудиторија?
- Потребители / генерална публика
- Бизнис професионалци
- Развоивачи / технички корисници
- Студенти / образовани луѓе
- Друго: [опишете]

Кои се нивните главни потреби?
```

### Чекор 3: Брендова Идентитет {#step-3-brand-identity}

```
Како изгледа вашите бои на брендот?
- Примарна боја: [избор на бои или шекс код]
- Вторична боја: [избор на бои или шекс код]
- Акцентна боја: [опционално]

Како би опишале тонот на вашиот бренд?
- Професионален / корпоративен
- Креативен / уметнички
- Игралив / неформален
- Минималистичен / модерен
- Топла / пријателски
```

### Чекор 4: Бизнис Цели {#step-4-business-goals}

```
Како изгледа успехот за вашиот сайт?
- Генерација на лиди (lead generation)
- Продажба на производи
- Градење заедница
- Споделување знаење
- Устаставување авторитет
- Друго: [опишете]

Како е вашата основна метрика?
- Приход
- Ангажман на корисниците
- Достигнување на содржини
- Конверзии
- Друго
```

### Чекор 5: Структура на Содржината {#step-5-content-structure}

```
Како е организирана вашата содржина?
- Плоска (се смината на исто ниво)
- Хиерархична (категории и подкатегории)
- Хронолошка (стил блог)
- Базирана на производите (каталог)
- Друго: [опишете]

Кои типови содржини користите?
- Блогови
- Страници за производи
- Студија случаи (case studies)
- Документација
- Видео
- Друго
```

## Паметта на вашиот site_brief {#your-sitebrief-memory}

По завршувањето на Спецификацијата на Сајтот, вашите информации се чуваат како **site_brief** во вашата агентова памет. Ова е структуриран запис што содржи:

## Преглед и ажуривање на вашата site_brief {#viewing-and-updating-your-sitebrief}

### Прегледај ја вашата site_brief {#view-your-sitebrief}

Прашај на агентот:

```
"Покажи ми мојата site brief"
```

или

```
"Што знаеш за мојата страница?"
```

Агентот ќе прикаже вашето зачувано спецификација.

### Ажурирај ја вашата site_brief {#update-your-sitebrief}

Ако вашата страница се развива, можете да ја ажурирате:

```
"Ажурирај ја мојата site brief: сега целим B2B клиенти"
```

или

```
"Освежи мојата спецификација на страницата"
```

Ова ќе повторно изврши вештината Site Specification со постоечките информации како почеток.

## Како агентите користат site_brief {#how-agents-use-sitebrief}

### Препораки за дизајн {#design-recommendations}

Кога прашувате за промени во дизајнот, агентите се однесуваат на вашата site_brief:

```
Ти: "Редизајни ја мојата главна страница"
Агент: "На база на вашата site brief, целите женци помеѓу 25-45 години со топла, артистична тоналност. Ќе создадам дизајн кој ќе го нагласи вашиот ручно изработен бижуер и вашите вредности поврзани со одржливоста."
```

### Предлози за функции {#feature-suggestions}

Агентите предлагаат функции кои се усогласени со вашите цели:

```
Ти: "Кои функции треба да ги додадам?"
Агент: "За е-комерцијална продавница фокусирана на лојалност на клиентите, препорачувам: програма за лојалност, коментари од клиенти, е-маил бюлетин и предлози за производи."
```

### Структура на содржината {#content-structure}

Агентите предлагаат организација на содржината врз основа на вашата структура:

Как да организирам блога си?
Отговор: Тъй като използвате каталог за продукти с блог, бих препоръчал да организирате публикации по продуктови категории и да създадете „стилистични ръководства“, които показват няколко продукта заедно.

## Най-добри практики {#best-practices}

### Бъдете конкретни {#be-specific}

Вместо "обща публика", описвайте действителната си аудитория:

- ✓ "Жени на 25-45 години, заинтересовани за устойчива мода"
- ✗ "Всико останало"

### Актуализирайте редовно {#update-regularly}

Докато вашият сайт се развива, актуализирайте `site_brief`:

- Когато преминете към нова аудитория
- Когато добавите нови линии от продукти
- Когато промените идентичността на вашия бранд
- Когато се променят бизнес целите ви

### Използвайте последователна терминология {#use-consistent-terminology}

Използвайте същите термини през всички сесии:

- ✓ Винаги кажете "устойчиви бижута" (а не "екологични бижута" и "зелени продукти")
- ✓ Постоянно се обръщайте към вашата аудитория по един начин

### Включете контекст {#include-context}

Дайте информация, която помага на агентите да разберат вашите решения:

- "Целим професионалисти, които ценят качество над цената"
- "Нашата аудитория е технически подготвена и очаква съвременен дизайн"
- "Ние сме стартъп с минимален капитал, така че ни трябват достъпни решения"

## Връзка с onboarding на Theme Builder {#relationship-to-theme-builder-onboarding}

Уменията за Site Specification са интегрирани в **потока за onboarding на Theme Builder**. Когато завършите процеса за onboarding, `site_brief` ви се създава автоматично с информацията, която сте предоставили.

Можете също да стартирате Site Specification независимо, ако искате да:

- Подобрите спецификацията си след първоначална настройка
- Актуализирате сайта си brief, докато сайтът ви се развива
- Създадете детайлна спецификация преди да започнете с Theme Builder

## Отстраняване на проблеми (Troubleshooting) {#troubleshooting}

**Моят site_brief не се използва**
- Потвърдете дали агентите имат достъп до паметта (memory)
- Попитайте агента да "спомни моим site brief"
- Проверете дали паметта е включена в настройките си

**Сака да почнам наново со нов `site_brief`**
- Прашај ја агентот: "Очисти мој `site_brief` и започни све од ново"
- Потоа повтори Site Specification

**Агентот прави препораки кои не се совршта со мојот `site_brief`**
- Прашај на агентот да "прегледа мојот `site_brief`"
- Ажурирај го вашиот `site_brief` ако е непостарено
- Додај повеќе контекст во вашите барања

## Следен чекови {#next-steps}

По дефинирање на вашата Site Specification:

1. **Користи Theme Builder**: Создадете кастом тема базирана на вашиот `site_brief`
2. **Подобрување на дизајнот**: Користите вештината Design System Aesthetics за детализиран работа со дизајн
3. **Планирање на содржината**: Прашајте агенти за препораки за структура на содржината
4. **Изградба на функции**: Барајте функции кои се усогласени со вашите бизнис цели
