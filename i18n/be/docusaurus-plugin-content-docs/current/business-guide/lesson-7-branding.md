# Урок 7: Адаптация пад ваш бренд

Вашы кліенты ніколі не павінны адчуваць, што яны выкарыстоўваюць "каторысьці WordPress plugin". Яны павінны адчуваць, што выкарыстоўваюць FitSite — платформу, створаную для іх індустрыяй. У гэтым уроку мы разглядаем брэндзінг, white-labeling і тое, як змагчы платформу адчуваць, нібы гэта сапраўдны прадукт.

## Адкуль мы пачалі

FitSite мае працуе паведамленне (checkout flow), якое вядзе ўладальнікаў фітнес-студыяў ад выбару плана да працуега сайта. Цяпер мы здэнчым увесь досвед, каб ён выглядаў як адзіны, брэндэваны прадукт.

## Домен вашай платформы

Асновай вашага брэнда з'яўляецца ваш домен. Для FitSite:

- **Гоўны домен**: `fitsite.com` (ваш маркетынгавы сайт і галінная кропка сеткі)
- **Сайты кліентаў**: `studioname.fitsite.com` (субдомены)
- **Карыстальніцкія домены**: Кліенты па планах Growth і Pro могуць падключыць свой уласны домен

### Настройка домену

1. Зарэгістраруйце свой платформенны домен
2. Накіруйце яго на вашага хостынг-провайдэра
3. Настройце wildcard DNS (`*.fitsite.com`) для субдоменных сайтаў кліентаў
4. Убедзіцеся, што актыўны wildcard SSL

Для дэталнай інструкцыі спадывіце [Як настроіць мапінг домена](/user-guide/domain-mapping/how-to-configure-domain-mapping).

## White-Labeling адмін-досвіду

Калі ўладальнік фітнес-студыі ўваходзіць на адмін-панель (dashboard) сваёй сайта, ён павінен бачыць ваш брэнд, а не брэнд WordPress ці Ultimate Multisite.

### Карыстальніцкія ўваходныя страліцы

Настройце WordPress login page, каб яна паказвала:

- Ваш логотип FitSite
- Фон, адпаведны фітнес-сферы
- Кольóры вашага брэнда

### Брэндзінг адмін-панелі

Выкарыстоўвайце аддон [Admin Page Creator](/addons/admin-page-creator) або карыстальніцкі CSS, каб:

- Замяніць логотип WordPress на ваш логотип FitSite
- Настройваць адмін-кольóры, каб яны падытчваліся да вашыга брэнда
- Дадаць карыстальніцкі віджэт на адмін-панель з быстрымі спасылмі і рэсурсамі для фітнес-сферы

### Карыстальніцкія адмін-страніцы

Падумайце пра стварэнне карыстальніцкіх адмін-страніц, якія выяўляюць найбольш рэлевантныя дзеянні для ўладальнікаў фітнес-студыяў:

- "Рэдагаваць графік класаў"
- "Актуалізоваць профілі тренераў"
- "Паглядзець свой сайт"

Гэта зніжае кривую вучэння, пакідаjąчы нішавыя дзеянні на першым месцы, а не забіваючы іх у стандартнае меню WordPress.

## Брэндзінг вашай камунікацыі

Кожны электронны ліст, рахунак і паведамленне павінны пацвярджаць ваш брэнд.

### Сістэмавыя электронныя лісты

Праййдзіце ў **Ultimate Multisite > Settings > Emails** і наладзьце ўсе сістэмавыя электронныя лісты:

- **Ад знаку**: FitSite
- **Ад электроннай адрасы**: hello@fitsite.com
- **Шаблоны лістоў**: Выкарыстоўвайце колеры і логотип вашага брэнда
- **Мова**: Фітнес-спецыфічна ў цэлым

Ключавыя лісты для налады:

| Ліст | Універсальная версія | Версія FitSite |
|-------|----------------|-----------------|
| Вітальны | "Ваш новы сайт гатовы" | "Ваш сайт фітнес-студыі працуе" |
| Квітанцыя за плацёж | "Пацверджаны плацёж" | "Пацверджаны плацёж падпіскі FitSite" |
| Трыал заканчваецца | "Ваш трыал заканчваецца карочэй за часам" | "Ваш трыал FitSite заканчваецца праз 3 дні — пакіньце свой сайт фітнес-студыі працуе" |

### Рахункі

Настройце шаблоны рахункаў, уключая:

- Ваш логотип FitSite і колеры брэнда
- Вашы бізнес-дадзеныя
- Назвы прадуктаў, спецыфічныя для фітнес-сферы (а не ўніверсальныя ID планаў)

## Сайт для кліентаў

Ваш гоўны домен (`fitsite.com`) павінен мець маркетынгавы сайт, які продае платформу. Гэта асобна ад адміні strataтывы сеткі Ultimate Multisite — гэта публічная візітка вашага бізнесу.

Ключавыя страліцы:

- **Гоўны сайт**: Ясная прапаноў каштоўнасці для фітнес-бізнесу
- **Функцыі**: Што робіць FitSite, але ў фітнес-тэрмах
- **Цанаўка**: Вашы тры планы з параўнаннем функцый, спецыфічным для нішы
- **Прыклады**: Паказчык сайтаў, створаных на гэтай платформе
- **Запісацца**: Спасылкі на вашу форму плацёжа

:::tip Ваш маркетынгавы сайт можа быць сеткавым сайтам
Стварыце свой маркетынгавы сайт як сайт унутры вашай сеткі. Гэта дазволіць вам кіраваць ім з адной адмін-панелі і паказаць уласнае камандванне вашай платформы.
:::

## Карыстальніцкі домен для кліентаў

Для кліентаў, якія падключаюць планы, уключаючы карыстальніцкія домены, чэканкаце гэты працэс ясна:

1. Кліент рэгіструе або пераносіць свой домен да рэгістратара
2. Кліент абнаўляе DNS, каб паказаць на вашу платформу (надаць точныя записи)
3. Ultimate Multisite займаецца мапінг-доменам і SSL

Стварыце артыкул дапаможнага матэрыялу або ўпытванне базы ведаў, спецыфікачна для гэтага працэсу, напісаны для неспецыялістаў-ўладальнікаў фітнес-студыяў.

## Сетка FitSite на дадзены момант

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured + branded)
├── Platform Domain (fitsite.com + wildcard SSL)
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow (niche-specific, tested)
├── Branding
│   ├── Custom login page
│   ├── Branded admin dashboard
│   ├── Niche-specific system emails
│   ├── Branded invoices
│   └── Marketing site on fitsite.com
└── Ready for onboarding flow (next lesson)
```

## Што мы змаглі зрабіць у гэтым уроку

- **Платформенны домен і DNS** настроены для брэндэванага досведнічання
- **White-labeled адмін-панель** з брэндзінг па ўсім FitSite
- **Карыстальніцкія камунікацыі** — электронныя лісты, рахункі і паведамленні ў ўсіх выпадах па брэндзе
- **Маркетынгавы сайт**, які продае FitSite ўладальнікам фітнес-студыяў
- **Дакументацыя па карыстальніцкіх доменах** для кліентаў, якія жадаюць свой уласны домен
