# Зварот ад карыстальнікаў і паведамленне пра праблемы

Gratis AI Agent v1.5.0 прадстаўляе ўбудаваную сістэму збору зваротаў і паведамлення пра праблемы, якая дазваляе карыстальнікам адзначаць непрыгажо адказы і паведамляць пра праблемы непасрэдна з чат-інтерфейсу. Гэтая сістэма ўключае кіраванне згодай, аўтаматычнае каманда паведамлення і трыаж, ажыццяўлены на бэкандзе.

## Кнопка «Не пагоджаецца» {#thumbs-down-button}

Каждое паведамленне, адпраўленае AI-асіスタнтам, мае кнопку **«Не пагоджаецца»** (👎). Калі карыстальнік націснуе на яе, ён можа адзначыць адказ як непрыгажо або неверны.

- Кнопка з'яўляецца пры наведванні на наступстве кожнага паведамлення ад асіスタнта.
- Націску на яе адкрывае **Модаль "Згода на зварот"** (Feedback Consent Modal).
- Зварот сувязаваны з тэмай размовы і канкрэтным паведамленнем.

## Модаль "Згода на зварот" {#feedback-consent-modal}

Калі карыстальнік націснуе на кнопку «Не пагоджаецца», папярэдні ў любым адпраўленні даных з'яўляецца модаль, якая патрабуе згоды. У гэтым модале:

- Паясняецца, якія дадзеныя будуць збірацца (усумок размовы, кантэкст браўзера).
- Карыстальніка просяць пацвердзіць, што ён згодны падзяліцца гэтымі дадзенымі.
- Прадугледжваецца поле для тэкста, дзе карыстальнік можа апісаць, што пайшло не так.
- Прапануецца кнопка **«Скасаваць»** (Cancel), каб зачыніць модаль без адпраўкі.

Жо，「зварот не фіктарыраецца, пакуль карыстальнік не пацвердзіць гэта ў адкрыты.

## Баннер звароту па аўтаматычным прампто (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Пасля завяршэння размовы асіスタнт можа адказаць **баннер звароту па аўтаматычным прампто** — недарэўлівы паведамленне, што пытаецца, была літаратура карыснай.

Гэты баннер з'яўляецца аўтаматычна на аснове даўжыні і выніку размовы. Ён звязваны з тым жа парачкам звароту, што і кнопка «Не пагоджаецца». Карыстальнікі могуць зачыніць баннер, не пастаўляючы зварот.

## Каманда /report-issue {#report-issue-slash-command}

Карыстальнікі могуць выклікаць парачак звароту непасрэдна, напісаўшы `/report-issue` у полі ўваходу чату. Гэтая каманда:

- Неадкладна адкрывае Модаль "Згода на зварот".
- Папярэдня заўражывае поле апісання кантэкстам аб бягунчай размове.
- Дазваляе карыстальнікам дадаць падатлівую дэталё.

Каманда `/report-issue` доступна ў ўсіх рэжымах чату (inline, floating widget, full-screen).

## Трыаж з памошчю AI {#ai-assisted-triage}

Адпраўлены зварот пераўтвараецца ў слой трыажу AI, які:

- Катэгорызуе паведамленне (фактычная памылка, непрыгажо адказна, непадходяшы кантэнт і г. д.).
- Дадае адпаведныя метаданыя размовы.
- Пэрасылае рэзюмэ трыажу ў настроены **Feedback Endpoint** (глядзіце [Settings > Advanced](#settings-advanced)).

Гэта зніжае час ручной праверкі, паказваючы самыя крытычныя праблемы ў першую чаргу.

## Settings > Advanced {#settings-advanced}

Каб актываваць бэканд збору зваротаў, наладзьце наступныя поля ў **Gratis AI Agent → Settings → Advanced**:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | URL, які прымае POST-запыты з пагружальнымі дадзенымі звароту (JSON). |
| **Feedback API Key** | Bearer token, адпраўлены ў загалоўку `Authorization` для аўтэнтыфікацыі адпраўкі зваротаў. |

Паставіце ўжодве поля пустымі, каб выключыць збор зваротаў. Асобныя кнопкі звароту застаюцца бачнымі для карыстальнікаў, але адпраўкі не будуць перасыланяцца.

:::tip
Feedback endpoint павінен прымаць JSON-тэла з мінімум полямі `message_id`, `conversation_id`, `feedback_text` і `triage_category`. Для чаканага схемы (schema) глядзіце ў дакументацыю вашага пастаўшчыка endpoint.
:::
