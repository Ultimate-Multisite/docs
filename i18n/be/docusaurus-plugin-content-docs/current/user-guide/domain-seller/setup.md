# Domain Seller: наладжванне і канфігурацыя пастаўшчыка

Дадатак Domain Seller пастаўляецца з майстрам наладжвання з падказкамі, які праводзіць вас праз кожны неабходны крок. Гэтая старонка апісвае ход працы майстра і тое, як наладзіць або пераналадзіць пастаўшчыкоў пазней.

## Патрабаванні {#requirements}

- **Multisite Ultimate** v2.4.12 або вышэй, актываваны ў сетцы
- **PHP** 7.4+
- Уліковыя даныя API як мінімум для аднаго падтрыманага рэгістратара

## Майстар першапачатковага наладжвання {#first-run-setup-wizard}

Майстар наладжвання запускаецца аўтаматычна пры першай сеткавай актывацыі плагіна. Ён таксама даступны ў любы час з **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Крок 1 — выберыце пастаўшчыка {#step-1--choose-a-provider}

Выберыце рэгістратара, якога хочаце падключыць. Падтрыманыя варыянты:

| Пастаўшчык | Кіраванне DNS | Прыватнасць WHOIS |
|---|---|---|
| OpenSRS | Так | Так |
| Namecheap | Не | Так (WhoisGuard, бясплатна) |
| HostAfrica | Так | Так (абарона ID) |
| Openprovider | Так | Так |
| Hostinger | Праз асноўнае супастаўленне даменаў Hostinger для размешчаных даменаў | Так |
| GoDaddy | Не | Не |
| ResellerClub | Так | Не |
| NameSilo | Не | Не |
| Enom | Так | Не |

### Крок 2 — увядзіце ўліковыя даныя {#step-2--enter-credentials}

Кожны пастаўшчык мае розныя палі ўліковых даных:

**OpenSRS** — імя карыстальніка і прыватны ключ (з OpenSRS Reseller Control Panel)

**Namecheap** — імя карыстальніка і ключ API (з Account › Tools › API Access)

**HostAfrica** — endpoint API рэсэлера даменаў і ўліковыя даныя з рэсэлерскага модуля HostAfrica. Асобны sandbox endpoint цяпер не дакументаваны; тэстуйце з бяспечнымі праверкамі толькі для чытання перад запускам жывых рэгістрацый.

**Openprovider** — імя карыстальніка і пароль з уключаным доступам API. Неабавязковы рэжым sandbox выкарыстоўвае sandbox API Openprovider, а неабавязковы стандартны ідэнтыфікатар кліента можна паўторна выкарыстоўваць для рэгістрацый.

**Hostinger** — агульны токен API Hostinger hPanel з асноўнай інтэграцыі Hostinger. Той самы токен забяспечвае асноўнае супастаўленне даменаў і аперацыі рэгістрацыі Domain Seller.

**GoDaddy** — ключ API і сакрэт (з developer.godaddy.com)

**ResellerClub** — ID рэсэлера і ключ API (з панэлі кіравання ResellerClub)

**NameSilo** — ключ API (з namesilo.com › Account › API Manager)

**Enom** — ID Account і токен API

Адзначце **Рэжым sandbox**, дзе даступна, каб пратэставаць у тэставым асяроддзі пастаўшчыка перад пераходам у жывы рэжым.

### Крок 3 — праверце злучэнне {#step-3--test-the-connection}

Націсніце **Праверыць злучэнне**. Майстар адпраўляе лёгкі выклік API, каб праверыць уліковыя даныя і падключэнне. Выпраўце любыя праблемы з уліковымі данымі перад працягам.

### Крок 4 — імпартуйце TLD {#step-4--import-tlds}

Націсніце **Імпартаваць TLD**, каб атрымаць усе даступныя TLD і аптовыя цэны ад падключанага пастаўшчыка. Гэта запаўняе спіс TLD, які выкарыстоўваецца даменнымі прадуктамі. Імпарт можа заняць 30–60 секунд для пастаўшчыкоў з вялікімі каталогамі TLD.

TLD таксама аўтаматычна паўторна сінхранізуюцца раз на дзень праз запланаванае заданне cron.

### Крок 5 — стварыце даменны прадукт {#step-5--create-a-domain-product}

Майстар стварае стандартны ўніверсальны даменны прадукт з нацэнкай 10%. Вы можаце адразу адрэдагаваць гэты прадукт або прапусціць і стварыць прадукты ўручную ў **Ultimate Multisite › Products**.

Глядзіце [Даменныя прадукты і цэнаўтварэнне](./domain-products) для поўнага кіраўніцтва па канфігурацыі прадукту.

## Пераналаджванне пастаўшчыка {#reconfiguring-a-provider}

Перайдзіце ў **Network Admin › Ultimate Multisite › Settings › Domain Seller** (або націсніце **Settings** у спісе плагінаў).

Старонка налад змяшчае:

- **Уключыць продаж даменаў** — уключыць або выключыць усю функцыю
- **Стандартны пастаўшчык** — пастаўшчык, які выкарыстоўваецца для пошуку даменаў і новых прадуктаў
- **Максімум TLD на пошук** — колькі TLD правяраць, калі кліент шукае; большыя значэнні паказваюць больш варыянтаў, але працуюць павольней
- **Працягласць кэша даступнасці** — як доўга кэшаваць вынікі даступнасці і цэнаўтварэння; меншыя значэнні больш дакладныя, але павялічваюць колькасць выклікаў API
- **Кіраваць даменнымі прадуктамі** — хуткая спасылка на спіс Products
- **Наладзіць пастаўшчыкоў** — адкрывае Integration Wizard, каб дадаць або пераналадзіць пастаўшчыкоў

### Даданне другога пастаўшчыка {#adding-a-second-provider}

Націсніце **Наладзіць пастаўшчыкоў** і зноў запусціце майстар для новага рэгістратара. Вы можаце мець некалькі пастаўшчыкоў, наладжаных адначасова. Прызначце кожны даменны прадукт пэўнаму пастаўшчыку або пакіньце яго на стандартным.

### Ручная сінхранізацыя TLD {#syncing-tlds-manually}

На старонцы налад націсніце **Сінхранізаваць TLD** побач з любым наладжаным пастаўшчыком, каб атрымаць апошнія цэны. Гэта карысна пасля таго, як пастаўшчык абнаўляе аптовыя цэны або дадае новыя TLD.

| `domain-seller-renewal` | Вынікі заданняў падаўжэння |
| `domain-seller-opensrs` | Сырая актыўнасць API OpenSRS |
| `domain-seller-namecheap` | Сырая актыўнасць API Namecheap |
| `domain-seller-hostafrica` | Сырая актыўнасць API HostAfrica |
| `domain-seller-openprovider` | Сырая актыўнасць API Openprovider |
| `domain-seller-hostinger` | Сырая актыўнасць API Hostinger |
| `domain-seller-godaddy` | Сырая актыўнасць API GoDaddy |
| `domain-seller-resellerclub` | Сырая актыўнасць API ResellerClub |
| `domain-seller-namesilo` | Сырая актыўнасць API NameSilo |
| `domain-seller-enom` | Сырая актыўнасць API Enom |

## Заўвагі пра магчымасці пастаўшчыкоў {#provider-capability-notes}

Не кожны API рэгістратара адкрывае аднолькавыя аперацыі. Дадатак паказвае непадтрыманыя аперацыі праз зразумелыя памылкі для адміністратара замест таго, каб моўчкі збойваць.

- **HostAfrica** падтрымлівае найшырэйшы працоўны працэс жывога рэсэлера, уключаючы пошук, сінхранізацыю TLD/цэн, рэгістрацыю, падаўжэнне, перанос, абнаўленне nameserver, DNS-запісы, EPP-коды, registrar lock і абарону ID.
- **Openprovider** падтрымлівае сінхранізацыю TLD па рэсэлерскіх цэнах, рэгістрацыю, падаўжэнне, пераносы, абнаўленне nameserver, DNS-зоны, EPP-коды, registrar lock і прыватнасць WHOIS. Ён аўтэнтыфікуецца з кароткатэрміновым bearer token, які addon абнаўляе аўтаматычна.
- **Hostinger** падтрымлівае пошук даступнасці, рэгістрацыю, пошук у партфелі, абнаўленне nameserver, registrar lock і прыватнасць WHOIS праз агульны API-токен hPanel. Публічны Domains API Hostinger не адкрывае рэсэлерскія/аптовыя цэны, уваходны перанос, відавочнае падаўжэнне або атрыманне EPP-кода; падаўжэнні адбываюцца толькі аўтаматычна.
