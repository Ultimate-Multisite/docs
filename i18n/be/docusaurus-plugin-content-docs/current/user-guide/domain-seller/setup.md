# Domain Seller: Nastrój i konfigurцыя paстаўшчыка

Addon Domain Seller пастаўляецца з krok-po-kroku wizardom nastroju, які праводзіць вас праз кожны неабходны крок. Гэтая старонка ахоплівае праход праз wizard i паказвае, як канфігураваць або пераканфігураваць пастаўшчыкоў пасля гэтага.

## Патрабаванні

- **Ultimate Multisite** v2.4.12 або больш высокі, актываваны на сетцы (network-activated)
- **PHP** 7.4+
- API-уданні для адныму з падтрымліваных рэгістрараў

## Wizard nastroju пры першым запуску

Wizard nastroju аўтаматычна запускаецца ў першы раз, калі вы актывуеце plugin на сетцы. Ён таксама даступны ў любы час у **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Крок 1 — Выбар пастаўшчыка

Выберыце рэгістрара, які вы хочаце падключыць. Падтрымліваныя варыянты:

| Пастаўшчык | Кіраванне DNS | Pryватнасць WHOIS |
|---|---|---|
| OpenSRS | Так | Так |
| Namecheap | Nie | Так (WhoisGuard, бясплатна) |
| GoDaddy | Nie | Nie |
| ResellerClub | Так | Nie |
| NameSilo | Nie | Nie |
| Enom | Так | Nie |

### Крок 2 — Увод уданніх

Кажны пастаўшчык мае адрозныя поля для ўводзін уданніх:

**OpenSRS** — Параклінік і прыватны ключ (з OpenSRS Reseller Control Panel)

**Namecheap** — Параклінік і API-ключ (з Account › Tools › API Access)

**GoDaddy** — API-ключ і сакрат (з developer.godaddy.com)

**ResellerClub** — Reseller ID і API-ключ (з адпаведнага кантрольнага панэля ResellerClub)

**NameSilo** — API-ключ (з namesilo.com › Account › API Manager)

**Enom** — ID канту і API-токен

Праверце **Sandbox mode**, калі гэта магчыма, каб пратэставаць у адпавежнай прасторы пастаўшчыка, перш чым запускаць у практыку.

### Крок 3 — Праверка злучэння

Націсніце **Test Connection**. Wizard адпраўляе лёгкі API-запыт для праверы ўданніх і злучэння. Выправіце любыя праблемы з уданнімі, перш чым працягваць.

### Крок 4 — Імпарт TLDs

Націсніце **Import TLDs**, каб загрузіць усе даступныя TLDs і гульдовыя цэны з падключнага пастаўшчыка. Гэта заповняе спіс TLDs, які выкарыстоўваецца для доменных прадуктаў. Імпарт можа заняць ад 30 да 60 секунд для пастаўшчыкоў з вялікім каталізам TLDs.

TLDs таксама аўтаматычна перасінходзяць ад разу ў дзень праз распісваны cron job.

### Крок 5 — Стварыць доменны прадукт

Wizard стварае пачатковы "catch-all" доменны прадукт з 10% націжкам. Вы можаце адрэдагаваць гэты прадукт неадкладна або праскочыць і стварыць прадукты навадоўніка ў **Ultimate Multisite › Products**.

Для паўнавартаснага параўтавання прадуктаў і цэнаў глядзіце [Domain Products and Pricing](./domain-products).

## Пераканфігурацыя пастаўшчыка

Праййміце на **Network Admin › Ultimate Multisite › Settings › Domain Seller** (або націсніце **Settings** у спісе plugin).

На гэтай старонцы знаходзяцца:

- **Enable domain selling** — пераключэнне ўсяй функцыі ў выключную/у практыку
- **Default provider** — пастаўшчык, які выкарыстоўваецца для пошуку доменных імеў і новых прадуктаў
- **Max TLDs per search** — колькі TLDs праверыць, калі кліент шукае; больш высокія значэнні паказваюць больш варыянтаў, але яшчэ паволенней
- **Availability cache duration** — на}|\|колькі даўга кешаваць вынікі даступнасці і цэнаў; больш нізкія значэнні больш правільныя, але павялічваюць выклікі API
- **Manage domain products** — хуткі спасыл на спіс Products
- **Configure providers** — адкрывае Integration Wizard для дадання або пераканфігурацыі пастаўшчыкоў

### Даданне другога пастаўшчыка

Націсніце **Configure providers** і запусціце wizard яшчэ раз для новага рэгістрара. Вы можаце мець канфігураваных некалькіх пастаўшчыкоў адначасова. Пазначайце кожны доменны прадукт канкрэтныму пастаўшчыку або пакідайце яго на пачатковы.

### Ручная синхронізацыя TLDs

На старонцы настроек націсніце **Sync TLDs** побач з любым канфігураваным пастаўшчыкам, каб загрузіць найлепшыя цэны. Гэта карысна пасля таго, калі пастаўшчык абноўіў гульдовыя цэны або дадаў новыя TLDs.

## Журналы (Logs)

Кожны пастаўшчык пішае ў свой уласны канал журналаў. Журналы можна прагледзець у **Network Admin › Ultimate Multisite › Logs**:

| Канал журнала | Змест |
|---|---|
| `domain-seller-registration` | Усе спробы рэгістрацыі (успішныя і неуспішныя) |
| `domain-seller-renewal` | Вынікі працы па пановленні |
| `domain-seller-opensrs` | Сыры API-актыўнасць OpenSRS |
| `domain-seller-namecheap` | Сыры API-актыўнасць Namecheap |
| `domain-seller-godaddy` | Сыры API-актыўнасць GoDaddy |
| `domain-seller-resellerclub` | Сыры API-актыўнасць ResellerClub |
| `domain-seller-namesilo` | Сыры API-актыўнасць NameSilo |
| `domain-seller-enom` | Сыры API-актыўнасць Enom |
