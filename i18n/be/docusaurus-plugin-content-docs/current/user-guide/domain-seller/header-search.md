# Пошук дамена ў загалоўку {#header-domain-search}

Выкарыстоўвайце гэтую наладу, калі патрэбна невялікая форма ў загалоўку, якая запускае пошук дамена, а затым дазваляе кліенту выбраць з даступных даменаў у checkout Ultimate Multisite.

## Патрабаванні {#requirements}

- Ultimate Multisite актыўны ў сетцы.
- Multisite Ultimate Domain Seller актыўны ў сетцы.
- Прынамсі адзін актыўны прадукт рэгістрацыі дамена з:
  - `domain_provider`, зададзеным як наладжаны правайдар.
  - Наладжанымі падтрымліванымі TLD, напрыклад `com`, `net` і `org`.
- Сапраўдная форма checkout, якая змяшчае поле **Выбар дамена**.

## Форма checkout {#checkout-form}

1. Стварыце або адрэдагуйце форму checkout, якая выкарыстоўваецца старонкай рэгістрацыі.
2. Дадайце звычайныя абавязковыя палі checkout/уліковага запісу, уключаючы **Імя карыстальніка**. Форма checkout, якая змяшчае толькі поле дамена, адхіляецца праверкай Ultimate Multisite.
3. Дадайце поле **Выбар дамена**.
4. Усталюйце рэжым Domain Selection на **Толькі рэгістрацыя**, калі працэс павінен засяроджвацца на зарэгістраваных даменах, а не на бясплатных паддаменах або існуючых даменах.
5. Прызначце прадукт рэгістрацыі дамена гэтаму полю.

Старонка рэгістрацыі павінна адлюстроўваць форму checkout, напрыклад:

```text
[wu_checkout slug="domain-form"]
```

## Форма ў загалоўку {#header-form}

Дадайце невялікую форму `GET` у загаловак сайта, якая адпраўляе ўведзены пошукавы тэрмін на старонку checkout як `domain_name`:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

Не выбірайце дамен загадзя ў карыстальніцкім JavaScript загалоўка. Загаловак павінен толькі перадаваць пошукавы тэрмін. Скрыпт checkout Domain Seller чытае `?domain_name=example`, запаўняе поле пошуку ў checkout і запускае пошук даступнасці, каб кліент мог выбраць з вернутых даменаў.

## Чаканыя паводзіны {#expected-behaviour}

Пошук `example` у загалоўку павінен адкрыць:

```text
/register/?domain_name=example
```

Затым checkout павінен паказаць вынікі, якія можна выбраць, напрыклад:

- `example.com`
- `example.net`
- `example.org` недаступны
- іншыя TLD, якія падтрымліваюцца правайдарам

Пасля выбару даступнага выніку зводка заказу павінна ўключаць прадукт рэгістрацыі дамена і выбранае даменнае імя.

## Праверка {#verification}

1. Адкрыйце галоўную старонку.
2. Знайдзіце простае імя, напрыклад `example`.
3. Пацвердзіце, што URL checkout уключае `?domain_name=example`.
4. Пацвердзіце, што поле дамена checkout змяшчае `example`.
5. Пацвердзіце, што з’яўляецца спіс варыянтаў даменаў.
6. Націсніце **Выбраць** для даступнага дамена.
7. Пацвердзіце, што зводка заказу змяшчае `Domain Registration - example.com` або выбраны дамен.

## Вырашэнне праблем {#troubleshooting}

- Калі спіс не з’яўляецца, праверце ўкладку сеткі ў браўзеры для `admin-ajax.php?action=wu_domain_search` і пацвердзіце, што ён вяртае непустыя `domains` або `results`.
- Калі форма checkout не праходзіць праверку пры захаванні, дадайце абавязковыя палі ўліковага запісу, такія як **Імя карыстальніка**.
- Калі выбар дамена не абнаўляе кошык, пацвердзіце, што `window.wu_checkout_form` існуе і што рэсурсы checkout Domain Seller загружаны на старонцы checkout.
