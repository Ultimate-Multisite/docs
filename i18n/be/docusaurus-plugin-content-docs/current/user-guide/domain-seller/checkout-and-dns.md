# Кіраванне полям аплаты і DNS для кліентаў

## Поле выбару домену

Поле **Domain Selection** — гэта элемент аплаты, які дае кліентам выбар, як атрымаць доменнае імя для сваёй сайта. Дадайце яго ў любую форму аплаты, каб актываваць продажу доменных імен.

### Дадаванне поля ў форму аплаты

1. Праййміце ў **Network Admin › Ultimate Multisite › Checkout Forms**
2. Адкраніце або створыце форму аплаты
3. У рэдактары аплаты націсніце **Add Field**
4. Выберыце **Domain Selection** з спісу паля
5. Наладзьце параметры поля (глядзіце ніжэй)
6. Захаваць форму

### Параметры поля

**Domain modes** — Выберыце, якія ўкладкі будуць бачныя для кліента. Кожны рэжым можна актываваць або выключыць незалежна:

| Mode | Што гэта робіць |
|---|---|
| **Subdomain** | Кліент выкарыстоўвае бясплатны субдомен на вашай сетцы (напрыклад, `mysite.yournetwork.com`). Аплата не патрэбна. |
| **Register New Domain** | Кліент шукае і рэгіструе новае доменнае імя праз вашага канфігураванага провайдэра. Для канчатковай цаны выкарыстоўваецца прадукт, які па пабіцці адпавядае домену. |
| **Existing Domain** | Кліент прывяжае домен, які ўжо мае. Не патрабуе платы за рэгістрацыю. Домен аўтаматычна прывяжаецца да яго сайта. |

**Default mode** — Калі ўсе тры рэжымы актываваны, які ўкладка адкрыецца першы. Уставіце на **Subdomain**, каб пакінуць рэгістрацыю домену неабязальнай, або на **Register New Domain**, каб замагчы пакупніцтва.

**Domain product** — Па важнасці, можна зафіксаваць гэтае поле на канкрэтным доменным выprodukце. Калі не ўстаноўлена, аддон аўтаматычна выбірае адпаведны прадукт на аснове TLD, які шукае кліент.

### Паля кантактаў рэгістранта

Калі кліент выбірае ўкладку **Register New Domain**, форма аплаты дадае поля кантактаў рэгістранта ў пачатковым рэжыме:

- Па іме / На іме
- Электронная адрас
- Адрэс (лінія 1, горад, вобласць/провінцыя, паштоўка, краіна)
- Номер тэлефона

Гэтыя паля патрабуюцца ў усіх рэгістратарах і праверка іх адбываецца, перш чым будзе зроблены выклік API рэгістрацыі. Номеры тэлефона аўтаматычна фарматуюцца ў міжнародны формат `+CC.NNN`, які патрабуюць рэгістратары.

### Аўтагенэраваны URL сайта

Калі кліент рэгіструе або прывяжае домен, поле URL сайта аўтаматычна замалвараецца з выбранага домену. Кліентам не робіцца патраба адносна замалвараць аднае поле URL.

### Паводзіны пошуку

- Даступнасць домену праверкаецца ў рэальным часе з выкарыстаннем AJAX, калі кліент пішае
- Альтэрнатыўныя прапановы TLD паказваюцца, калі пераважны домен недаступны
- Цэна пабіраецца ў рэжыме і ясна адлюстроўваецца (цена рэгістрацыі, годовая плата за пановленне, неабязава плата за прыватнасць WHOIS)
- Купонныя коды праходзяць на доменныя выprodukцы так само, як і на любы іншы прадукт

**Налады рэагавання пошуку:**

```php
// Павялічэнне запынення (мілісекунды) для зніжэння выклікаў API на спастых злучэннях
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Дадаванне карысных паля ў форму пошуку домену:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Дадатковыя заўвагі',
    ];
    return $fields;
});
```

## Кіраванне DNS для кліентаў

Кліенты могуць кіраваць DNS-запісамі для рэгістраваных доменных імен з адкання **My Account**, у секцыі, прысвечанай іх домену.

### Падтрымкі варты запісаў

| Type | Выкарыстанне |
|---|---|
| **A** | Павялічэнне імены хоста на IP-адрэс IPv4 |
| **AAAA** | Павялічэнне імены хоста на IP-адрэс IPv6 |
| **CNAME** | Стварыць псеўнімас, які ўказвае на іншае імена хоста |
| **MX** | Устаноўваць сервер абмена паштай |
| **TXT** | Дадаць запісы SPF, DMARC, праверкі або іншы тэкст |

### Якія провайдэры падтрымліваюць кіраванне DNS?

Кіраванне DNS (дадаванне, рэдагаванне, выдаленне запісаў) доступна з **OpenSRS**, **ResellerClub** і **Enom**. Домены Namecheap, GoDaddy і NameSilo паказваюць статус і інфармацыю пра вынік, але DNS трэба кіраваць непасрэдна ў пані Панелі кіравання рэгістратара.

### Пачатковыя DNS-запісы

Вы можаце канфігураваць пачатковыя DNS-запісы, якія аўтаматычна ўжываюцца, калі рэгіструецца домен. Праййміце ў **Settings › Domain Seller › Default DNS Records**.

Значэнні пачатковых запісаў падтрымліваюць два токены:

| Token | Замяняецца на |
|---|---|
| `{DOMAIN}` | Рэгістраванае доменнае імя (напрыклад, `example.com`) |
| `{SITE_URL}` | URL сайта WordPress для сайта кліента |

**Прыклад — ўказванне апікальнага домену і www на IP-адрэс вашага сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Адмін: прагляд і рэдагаванне DNS

Адмін-пакары могуць прагледзець і рэдагаваць DNS-запісы для домену ў любым кліенце з адкання домену ў **Network Admin › Ultimate Multisite › Domains**.
