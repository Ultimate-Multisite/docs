# Кіраванне полем афармлення замовы і DNS кліентаў

## Поле выбару дамена пры афармленні замовы {#the-domain-selection-checkout-field}

Поле **Выбар дамена** — гэта элемент афармлення замовы, які дае кліентам выбар, як атрымаць дамен іх сайта. Дадайце яго ў любую форму афармлення замовы, каб уключыць продаж даменаў.

### Даданне поля ў форму афармлення замовы {#adding-the-field-to-a-checkout-form}

1. Перайдзіце ў **Адміністраванне сеткі › Ultimate Multisite › Формы афармлення замовы**
2. Адкрыйце або стварыце форму афармлення замовы
3. У рэдактары афармлення замовы націсніце **Дадаць поле**
4. Выберыце **Выбар дамена** са спіса палёў
5. Наладзьце параметры поля (гл. ніжэй)
6. Захавайце форму

### Параметры поля {#field-options}

**Рэжымы дамена** — Выберыце, якія ўкладкі бачыць кліент. Кожны рэжым можна ўключыць або адключыць незалежна:

| Рэжым | Што ён робіць |
|---|---|
| **Паддамен** | Кліент выкарыстоўвае бясплатны паддамен у вашай сетцы (напрыклад, `mysite.yournetwork.com`). Аплата не патрэбна. |
| **Зарэгістраваць новы дамен** | Кліент шукае новы дамен і рэгіструе яго праз наладжанага вамі правайдара. Выкарыстоўвае адпаведны даменны прадукт для вызначэння цаны. |
| **Існуючы дамен** | Кліент прывязвае дамен, якім ужо валодае. Плата за рэгістрацыю не спаганяецца. Дамен аўтаматычна прывязваецца да яго сайта. |

**Прадвызначаны рэжым** — Калі ўключаны ўсе тры рэжымы, якая ўкладка адкрываецца першай. Усталюйце **Паддамен**, каб рэгістрацыя дамена заставалася неабавязковай, або **Зарэгістраваць новы дамен**, каб заахвочваць пакупкі.

**Даменны прадукт** — Пры жаданні прывяжыце гэта поле да канкрэтнага даменнага прадукту. Калі не зададзена, дадатак аўтаматычна выбірае адпаведны прадукт на аснове TLD, які шукае кліент.

### Палі кантактных даных рэгістранта {#registrant-contact-fields}

Калі кліент выбірае ўкладку **Зарэгістраваць новы дамен**, форма афармлення замовы дадае ўбудаваныя палі кантактных даных рэгістранта:

- Імя / Прозвішча
- Адрас электроннай пошты
- Адрас (радок 1, горад, штат/вобласць, паштовы індэкс, краіна)
- Нумар тэлефона

Яны патрабуюцца ўсімі рэгістратарамі і правяраюцца перад выкананнем выкліку API рэгістрацыі. Нумары тэлефонаў аўтаматычна фарматуюцца ў міжнародны фармат `+CC.NNN`, якога чакаюць рэгістратары.

### Аўтаматычна створаны URL сайта {#auto-generated-site-url}

Калі кліент рэгіструе або прывязвае дамен, поле URL сайта аўтаматычна запаўняецца з выбранага дамена. Кліентам не трэба запаўняць асобнае поле URL.

### Паводзіны пошуку {#search-behaviour}

- Даступнасць дамена правяраецца ў рэальным часе з дапамогай AJAX, пакуль кліент уводзіць тэкст
- Альтэрнатыўныя прапановы TLD паказваюцца, калі жаданы дамен недаступны
- Цэны атрымліваюцца ўжывую і выразна адлюстроўваюцца (цана рэгістрацыі, цана падаўжэння, неабавязковая плата за прыватнасць WHOIS)
- Коды купонаў прымяняюцца да даменных прадуктаў гэтак жа, як і да любога іншага прадукту

**Наладжванне хуткасці водгуку пошуку:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Даданне карыстальніцкіх палёў у форму пошуку дамена:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Кіраванне DNS кліентаў {#customer-dns-management}

Кліенты могуць кіраваць DNS-запісамі для сваіх зарэгістраваных даменаў са старонкі **Мой Account**, у запісе адпаведнага дамена.

### Падтрымліваныя тыпы запісаў {#supported-record-types}

| Тып | Выкарыстанне |
|---|---|
| **A** | Супаставіць імя хоста з IPv4-адрасам |
| **AAAA** | Супаставіць імя хоста з IPv6-адрасам |
| **CNAME** | Стварыць псеўданім, які паказвае на іншае імя хоста |
| **MX** | Задаць сервер абмену поштай |
| **TXT** | Дадаць SPF, DMARC, праверачныя або іншыя тэкставыя запісы |

### Якія правайдары падтрымліваюць кіраванне DNS? {#which-providers-support-dns-management}

Кіраванне DNS (даданне, рэдагаванне, выдаленне запісаў) даступнае з **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** і **Openprovider**. Дамены **Hostinger** могуць абнаўляць серверы імёнаў праз Domain Seller; DNS-запісы для размешчаных даменаў кіруюцца асноўнай інтэграцыяй прывязкі даменаў Hostinger. Дамены Namecheap, GoDaddy і NameSilo паказваюць статус і інфармацыю пра заканчэнне тэрміну дзеяння, але DNS трэба кіраваць непасрэдна ў панэлі кіравання рэгістратара.

### Прадвызначаныя DNS-запісы {#default-dns-records}

Вы можаце наладзіць прадвызначаныя DNS-запісы, якія аўтаматычна прымяняюцца пры рэгістрацыі дамена. Перайдзіце ў **Налады › Domain Seller › Прадвызначаныя DNS-запісы**.

Прадвызначаныя значэнні запісаў падтрымліваюць два токены:

| Токен | Замяняецца на |
|---|---|
| `{DOMAIN}` | Зарэгістраванае даменнае імя (напрыклад, `example.com`) |
| `{SITE_URL}` | URL сайта WordPress для сайта кліента |

**Прыклад — накіраваць каранёвы дамен і www на IP вашага сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Адміністратар: прагляд і рэдагаванне DNS {#admin-viewing-and-editing-dns}

Адміністратары сеткі могуць праглядаць і рэдагаваць DNS-запісы для любога дамена кліента са старонкі рэдагавання дамена ў **Адміністраванне сеткі › Ultimate Multisite › Дамены**.
