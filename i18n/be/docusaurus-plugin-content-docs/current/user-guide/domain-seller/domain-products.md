# Прадукты домейнаў і цэнаўтварэнне

Прадукты домейнаў дазваляюць вам кантраляваць цэнаўтварэнне, TLD-рас extension, прыватнасць WHOIS і выбар провайдэра. Кожны прадукт домейнаў — гэта стандартны прадукт Ultimate Multisite з дадатковай вкладкай **Налады домейнаў** (Domain Settings).

## Стваৰэнне прадукту домейнаў

1. Займіцеся **Network Admin › Ultimate Multisite › Products**
2. Націсніце **Add New**
3. Уста摆це тып прадукту на **Domain**
4. Наладзьце вкладку **Налады домейнаў** (глядзіце ніжэй)
5. Захаваць

Прадукты домейнаў з'яўляюцца з фіялетавым значком **Domain** у спісе прадуктаў і іх можна адфільтраваць, выкарыстоўваючы вкладку **Domain Products**.

## Вкладка налад домайнаў

### Провайдэр

Выберыце рэгістрара, які адказна за рэгістрацыю для гэтага прадукту. Па спілку ўстаноўлены **Default provider** у наладках Domain Seller.

### Падтрымкаваныя TLDs

Паставіце пустым, каб стварыць **універсальны прадукт** (catch-all product), які будзе працаваць для ўсіх TLD, якія не збіліся з іншым прадуктам.

Увярце спіс TLD, раздзялены кама (напрыклад, `.com, .net, .org`), каб стварыць **прадукт, спецыфічны для TLD**, які будзе працаваць толькі для гэтых extension.

**Як працуе збіў прадуктаў:** Калі кліент шукае домейн, аддон выбірае найбольш спецыфічны прадукт, які збіўся. Прадукт з `.com` у сваім спісе TLD мае прыярытэт над універсальным прадуктам. Калі не знаходзіцца жодны спецыфічны прадукт TLD, выкарыстоўваецца універсальны. Калі прадукты не існуюць, пошук домейна не адлюстраваецца.

### Тып націнкі

Тры рэжымы кантралююць, як ваш рознічны прайс вызначаецца з пакупніцкіх выдаткаў:

| Рэжым | Як гэта працуе |
|---|---|
| **Percentage** | Дадае адсоток да пакупніцкіх выдаткаў. Націнка 20% на домейн пакупніцкай вартысці $10 дае $12. |
| **Fixed markup** | Дадае фіксаваную суму ў доларах. Націнка $5 на домейн пакупніцкай вартысці $10 дае $15. |
| **Fixed price** | Паўгоджае пакупніцкія выдаткі. Заўсёды спалічвае суму, якую вы ўводзіце. |

### Пачатковае цэнаўтварэнне

Актываваць, каб прапанаваць паніжаны прайс на першы год. Уста摆це асобны **Introductory price** (прайс на год-1) побач з звычайным **Renewal price** (прайс на год 2+). Кліент бачыць абодва прайсы падчас аплатам, каб ведаць, чаго адчакаваць пры пановленні.

### Pryватнасць WHOIS

Кантралюе, ці прапануецца пратэктарызацыя прыватнасці WHOIS для домейнаў, рэгістраваных праз гэты прадукт.

| Налады | Паводзіны |
|---|---|
| **Disabled** | Pryватнасць WHOIS ніколі не прапануецца або не актывуецца. |
| **Always Included** | Pryватнасць WHOIS аўтаматычна актывуецца пры рэгістрацыі без платы. |
| **Customer Choice** | Паяўляецца праверка падчас аплатам. Уста摆це **Privacy price**, каб спалічваць за год, або паставіце $0, каб прапанаваць гэта бясплатна. |

Для Namecheap, WHOIS privacy выкарыстоўвае WhoisGuard (заўсёды бясплатна). Для OpenSRS, яно выкарыстоўвае OpenSRS privacy service (можа мець вартісць на пакупніцкім узроўні).

## Імпарт і сінкаванне TLD

Прадукты домейнаў адлюстраванне прайсы ў рэальным часе, атрыманыя ад падключанага провайдэра. Для працы гэтага патрабуецца імпартаваць TLD.

- **Ручны сінк:** Settings › Domain Seller › Sync TLDs (для кожнага провайдэра)
- **Аўтаматычны сінк:** Праходзіць штодня праз распарадкаваны cron job для ўсіх наладжаных провайдэраў

Пасля сінкавання зайдзіце на любую вкладку налад домейнаў прадукту і выкарыстоўвайце TLD picker, каб пабачыць даступныя TLD з іх бягучымі пакупніцкімі прайсамі.

## Аўтапановленне

Пановленне домейнаў звязана з статусам канчаткі былінга кліента:

- Калі былінга пановляецца і домейн звязаны, пановленне домейна аўтаматычна ставіцца ў чергу
- Папыткі пановлення выкарыстоўваюць актыўны шлюз аплатам кліента
- Неўдакладныя пановленні аўтаматычна параўноўваюцца з экспоненційным адставаннем (exponential backoff)
- Паведамленні па электроннай пошце адпраўляюцца ў выпадку паспяховага пановлення, неўдакладлення і набліжання да даты экспірацыі

ID шаблонаў электроннай пошты для падзей жыцкавага цыклу домейна:

| Падбыт | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## Адмін: Ручная рэгістрацыя домейна

Каб рэгістраваць домейн ад імя кліента, не праходзячы праз аплатам:

1. Займіцеся **Network Admin › Ultimate Multisite › Register Domain**
2. Выберыце кліента і ўвярце імя домейна
3. Зафармуйце дэталі кантактаў рэгістравантажа (імя, адрас, тэлефон)
4. Націсніце **Register**

Запіс домейна створаецца і звязваецца з кантактам кліента.
