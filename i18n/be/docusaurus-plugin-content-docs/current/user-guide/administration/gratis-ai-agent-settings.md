# Налады Gratis AI Agent

Экран **Settings → Advanced** у Gratis AI Agent дае канфігурацыю ўзроўню адміністратара для backend-інтэграцый. На гэтай старонцы дакументуюцца перанакіраванне водгукаў, ключы пастаўшчыкоў пошуку, наладжванне кіраванага сэрвісу Superdav, элементы кіравання Google Calendar, налады TextBee SMS і сеткавыя сцягі функцый.

## Доступ да налад

1. У адміністраванні WordPress перайдзіце ў **Gratis AI Agent → Settings**.
2. Націсніце ўкладку **Advanced**.

## Канфігурацыя endpoint для водгукаў

Endpoint для водгукаў атрымлівае POST-запыты ад AI агента кожны раз, калі карыстальнік адпраўляе водгук праз кнопку з вялікім пальцам уніз, банер аўтазапыту або каманду `/report-issue`.

| Поле | Апісанне |
|---|---|
| **Feedback Endpoint URL** | URL, які атрымлівае адпраўленыя водгукі як HTTP POST-запыты з JSON-целам. |
| **Feedback API Key** | bearer token, які адпраўляецца ў `Authorization` header кожнага запыту водгуку. Пакіньце пустым, калі ваш endpoint не патрабуе аўтэнтыфікацыі. |

### Чаканая JSON-нагрузка

Ваш endpoint для водгукаў павінен прымаць JSON-цела як мінімум з наступнымі палямі:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

У нагрузцы могуць прысутнічаць дадатковыя палі ў залежнасці ад кантэксту размовы.

### Значэнні `triage_category`

Слой AI-сартавання прызначае адно з наступных значэнняў для `triage_category` перад перанакіраваннем нагрузкі:

| Значэнне | Значэнне |
|---|---|
| `factual_error` | Асістэнт падаў няправільную фактычную інфармацыю. |
| `unhelpful_answer` | Адказ быў тэхнічна правільны, але не карысны. |
| `inappropriate_content` | Адказ утрымліваў змесціва, якое не варта паказваць карыстальнікам. |
| `other` | Водгук не адпавядаў вядомай катэгорыі. |

### Аўтэнтыфікацыя

Калі ваш endpoint патрабуе аўтэнтыфікацыі, задайце ў поле **Feedback API Key** ваш bearer token. Агент адпраўляе:

```
Authorization: Bearer <your-api-key>
```

Калі поле **Feedback API Key** пустое, `Authorization` header не адпраўляецца.

### Адключэнне збору водгукаў

Пакіньце палі **Feedback Endpoint URL** і **Feedback API Key** пустымі. Кнопка з вялікім пальцам уніз і UI для водгукаў застаюцца бачнымі карыстальнікам, але адпраўкі не перанакіроўваюцца ні ў які знешні сэрвіс.

## Brave Search API Key

Таксама на ўкладцы **Advanced** поле **Brave Search API Key** уключае магчымасць [Інтэрнэт-пошук](../configuration/internet-search).

| Поле | Апісанне |
|---|---|
| **Brave Search API Key** | Ваш API-ключ з панэлі распрацоўшчыка Brave Search. Патрабуецца для ўключэння інтэрнэт-пошуку ў AI асістэнце. |

Метка поля ўключае клікабельную спасылку на старонку рэгістрацыі Brave Search API. Пакіньце пустым, каб адключыць інтэрнэт-пошук.

Глядзіце [Інтэрнэт-пошук](../configuration/internet-search) для дакументацыі для канчатковых карыстальнікаў па гэтай функцыі.

## Кіраваны сэрвіс Superdav

Superdav AI Agent v1.18.0 дадае endpoint-ы кіраванага сэрвісу Superdav і аўтаматычнае падрыхтаванне падключэнняў для падтрымліваемых сайтаў. Выкарыстоўвайце гэтыя элементы кіравання, калі ваш сайт павінен падключацца да размешчанага пастаўшчыка замест уручную сканфігураванага endpoint сэрвісу.

| Поле | Апісанне |
|---|---|
| **Managed Superdav Service** | Уключае падключэнне да размешчанага сэрвісу Superdav для падтрымліваемых сайтаў. |
| **Provision Connection** | Запускае аўтаматычнае падрыхтаванне endpoint і ўліковых даных. Выкарыстоўвайце гэта пасля пацвярджэння, што сайт павінен выкарыстоўваць кіраванага пастаўшчыка. |
| **Service Endpoint / Connection Status** | Паказвае бягучы endpoint або стан падключэння пасля падрыхтавання. |

Пасля падрыхтавання захавайце налады і праверце стан падключэння, перш чым спадзявацца на працоўныя працэсы кіраванага сэрвісу. Калі падрыхтаванне не ўдалося, праглядзіце паказаныя парады па паўторнай спробе і пацвердзіце, што сайт мае дазвол выкарыстоўваць размешчанага пастаўшчыка.

## Канфігурацыя Google Calendar

Калі функцыі календара Superdav AI Agent v1.18.0 уключаны, агент можа чытаць сканфігураваныя календары і дэталі падзей. Інструменты календара арыентаваны на чытанне і карысныя для напамінаў з улікам раскладу, наступных дзеянняў з удзельнікамі і супастаўлення кантактаў.

| Поле | Апісанне |
|---|---|
| **Google Calendar Credentials** | Захоўвае ўліковыя даныя або token-падключэнне, неабходнае для чытання даных календара. |
| **Calendar Selection** | Абмяжоўвае, якія сканфігураваныя календары агент можа праглядаць. |
| **Calendar Connection Status** | Пацвярджае, ці могуць бягучыя ўліковыя даныя чытаць календары і падзеі. |

Абмяжоўвайце ўліковыя даныя календара толькі тымі календарамі, якія патрэбныя агенту. Перападключыце або змяніце ўліковыя даныя, калі стан паказвае на пратэрмінаваны token.

## TextBee SMS-апавяшчэнні

Superdav AI Agent v1.18.0 дадае TextBee як SMS-пастаўшчыка для сканфігураваных працоўных працэсаў апавяшчэнняў. SMS-апавяшчэнні варта спалучаць з этапамі зацвярджэння чалавекам для адчувальных паведамленняў або паведамленняў, звернутых да карыстальнікаў.

| Поле | Апісанне |
|---|---|
| **TextBee API Key** | Аўтэнтыфікуе запыты да SMS-пастаўшчыка TextBee. |
| **TextBee Device / Sender** | Выбірае адпраўніка або прыладу TextBee, што выкарыстоўваецца для выходных паведамленняў, калі гэта патрабуецца пастаўшчыком. |
| **SMS Notifications Enabled** | Дазваляе зацверджаным працоўным працэсам адпраўляць апавяшчэнні тэкставымі паведамленнямі. Пакіньце адключаным, каб прадухіліць адпраўку SMS. |

Адпраўце тэставае паведамленне толькі на нумар, які належыць адміністратару, затым пацвердзіце паводзіны этапаў зацвярджэння перад уключэннем запланаваных напамінаў або напамінаў, звернутых да ўдзельнікаў.

## Сцягі функцый

Таксама ўведзеная ў v1.9.0, укладка **Settings → Feature Flags** дае пераключальнікі для неабавязковай функцыянальнасці. Кожны сцяг або ўключаны, або адключаны ва ўсёй сетцы; на гэты момант няма пераазначэння для асобнага сайта.

### Доступ да сцягоў функцый

1. У адміністраванні WordPress перайдзіце ў **Gratis AI Agent → Settings**.
2. Націсніце ўкладку **Feature Flags**.

### Сцягі кантролю доступу

| Сцяг | Прадвызначана | Апісанне |
|---|---|---|
| **Абмежаваць для адміністратараў** | Off | Калі ўключана, толькі карыстальнікі з роляй `administrator` могуць адкрыць панэль чата AI Agent. Усе іншыя ролі замест гэтага бачаць паведамленне "Звяжыцеся са сваім адміністратарам". |
| **Абмежаваць для адміністратараў сеткі** | Off | Калі ўключана ў multisite-сетцы, толькі Super Admins могуць карыстацца агентам. Адміністратары асобных сайтаў блакіруюцца. Мае прыярытэт над "Абмежаваць для адміністратараў", калі ўключаны абодва параметры. |
| **Дазволіць доступ падпісчыкам** | Off | Калі ўключана, карыстальнікі з роляй `subscriber` могуць карыстацца інтэрфейсам чата, але абмежаваныя магчымасцямі толькі для чытання (без стварэння запісаў або змянення налад). |
| **Адключыць для не-ўдзельнікаў** | Off | Інтэгруецца са статусам членства Ultimate Multisite. Калі ўключана, чат схаваны для сайтаў, якія не маюць актыўнага членства. |

### Сцягі брэндынгу

| Сцяг | Прадвызначана | Апісанне |
|---|---|---|
| **Схаваць ніжні радок "Powered by Gratis AI Agent"** | Off | Выдаляе радок атрыбуцыі брэндынгу, які паказваецца ўнізе віджэта чата. Рэкамендуецца для разгортванняў з белай этыкеткай. |
| **Карыстальніцкая назва агента** | *(пусты)* | Замяняе стандартную метку "Gratis AI Agent" у загалоўку чата і меню адміністратара на назву вашага ўласнага прадукту. Пакіньце пустым, каб выкарыстоўваць стандартнае значэнне. |
| **Схаваць выбар агента** | Off | Калі ўключана, карыстальнікі не могуць пераключацца паміж пяццю ўбудаванымі агентамі. Бягучы агент фіксуецца на тым, што наладжана як стандартнае ў Налады → Агульныя. |
| **Выкарыстоўваць значок сайта як аватар чата** | Off | Замяняе стандартны значок AI у загалоўку віджэта чата значком сайта WordPress (задаецца ў Вонкавы выгляд → Наладзіць → Ідэнтычнасць сайта). |

### Сцягі бяспекі аўтаматызацыі

Superdav AI Agent v1.18.0 уводзіць шлюзы адабрэння чалавекам і запісы напамінаў для больш бяспечных запускаў аўтаматызацыі. Гэтыя элементы кіравання могуць з'яўляцца ў сцягах функцый або пашыраных наладах аўтаматызацыі, у залежнасці ад усталяванага пакета.

| Сцяг | Прадвызначана | Апісанне |
|---|---|---|
| **Патрабаваць адабрэнне чалавекам** | Рэкамендуецца ўключыць | Прыпыняе адчувальныя аўтаматызацыі, пакуль аўтарызаваны карыстальнік не прагледзіць і не пацвердзіць прапанаванае дзеянне. |
| **Дэдуплікацыя напамінаў** | On | Запісвае адпраўленыя напаміны, каб паўторныя спробы або запланаваныя запускі не адпраўлялі дублікаты апавяшчэнняў. |
| **Уключыць інструменты календара** | Off, пакуль не наладжана | Дазваляе агенту чытаць наладжаныя календары і падзеі Google. |
| **Уключыць SMS-апавяшчэнні** | Off, пакуль не наладжана | Дазваляе зацверджаным працоўным працэсам адпраўляць SMS-апавяшчэнні TextBee пасля захавання ўліковых даных. |

### Прымяненне змен

Націсніце **Захаваць налады** пасля пераключэння любога сцяга. Змены ўступаюць у сілу неадкладна — не патрабуецца ачыстка кэша або паўторная актывацыя плагіна.
