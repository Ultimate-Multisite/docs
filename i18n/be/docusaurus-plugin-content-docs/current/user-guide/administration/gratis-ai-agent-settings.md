# Налады Gratis AI Agent

Экран **Налады → Advanced** (Налады → Падзейні) у Gratis AI Agent прадугледжвае канфігурацыю ўзроўню адміністратара для бэкэндавых інтэграцый, прадстаўленых у версіі v1.5.0. Гэтая старонка апісвае поля **Feedback Endpoint** і іх чакаваны фарматы.

## Доступ да Наладаў

1. У адмініструватары WordPress зайдзіце ў **Gratis AI Agent → Settings**.
2. Націсніце на тэab **Advanced**.

## Канфігурацыя Feedback Endpoint

Feedback endpoint атрымлівае POST-запыты ад AI агента ў кожны момант, калі карыстальнік адпраўляе водгук праз кнопку "палец-ніж" (thumbs-down), баннер аўтаматычнага прампту або камандрай `/report-issue`.

| Поле | Апісан |
|---|---|
| **Feedback Endpoint URL** | URL, які атрымлівае адпраўленне водгукаў як HTTP POST-запыты з JSON-тэлам. |
| **Feedback API Key** | Bearer-токен, адпраўляецца ў заголовок `Authorization` кожнага запыту водгуку. Пакіньце пустым, калі ваш endpoint не патрабуе аўтэнтыкацыі. |

### Чакаванае JSON-тэла (Payload)

Ваш feedback endpoint павінен прымаць JSON-тэла з мінімум наступнымі полямі:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Адказ быў неверным адносна цаны.",
  "triage_category": "factual_error"
}
```

Додатковыя поля могуць быць прысутнічаць у тэле ў залежнасці ад кантексту размовы.

### Значэнні `triage_category`

Сцэнрыванне AI (AI triage layer) прызначае адно з наступных значэнняў для `triage_category` перад перадачай тэла:

| Значэнне | Значэнне |
|---|---|
| `factual_error` | Асістэнт прадастаўляў неверную фактычную інфармацыю. |
| `unhelpful_answer` | Адказ быў тэхнічна правільным, але не карысным. |
| `inappropriate_content` | Адказ містшы кантэнт, які не павінен быць паказаны карыстальнікам. |
| `other` | Водгук не падыходзіў да вядомай катэгорыі. |

### Аўтэнтыкацыя

Калі ваш endpoint патрабуе аўтэнтыкацыі, ўставіце ў поле **Feedback API Key** свой bearer-токен. Агент адпраўляе:

```
Authorization: Bearer <your-api-key>
```

Калі поле **Feedback API Key** пусты, заголовок `Authorization` не адпраўляецца.

### Выключэнне збору водгукаў

Пакіньце пустымі поля **Feedback Endpoint URL** і **Feedback API Key**. Кнопка "палец-ніж" і UI водгукаў застаюцца бачным для карыстальнікаў, але адпраўленне не перадаецца на канчатковы (external) сэрвіс.

## Brave Search API Key

Таж на тэab **Advanced** поле **Brave Search API Key** дазваляе функцыянальнасць [Internet Search](../configuration/internet-search).

| Поле | Апісан |
|---|---|
| **Brave Search API Key** | Ваш API-ключ з дашборда Brave Search developer. Неабходны для ўключэння пошуку ў Інтэрнэце ў AI асістэнта. |

Маркіроўка поля ўключае клікабельнае спасылкі на старонку рэгістрацыі Brave Search API. Пакіньце пустым, каб выключыць пошук ў Інтэрнэце.

Для дадатковай інфармацыі для карыстальнікаў пра гэтую функцыю спадырайце [Internet Search](../configuration/internet-search).

## Feature Flags

Таж прадстаўлены ў v1.9.0, тэab **Settings → Feature Flags** прадугледжвае пераключальнікі для неабязавальнай функцыянальнасці. Кожны флаг можа быць уключаны або выключэн на ўсім сетцы; на дадзены момант няма перавышэння на ўзроўні сайта.

### Доступ да Feature Flags

1. У адмініструватары WordPress зайдзіце ў **Gratis AI Agent → Settings**.
2. Націсніце на тэab **Feature Flags**.

### Флагі кантролю доступу (Access Control Flags)

| Flag | Default | Description |
|---|---|---|
| **Restrict to Administrators** | Off | Калі ўключаны, толькі карыстальнікі з роллёй `administrator` могуць адкрыць чат-панель AI Agent. Усе іншыя ролі бачаць паведамленне "Свяжыцеся з вашым адміністратарам". |
| **Restrict to Network Admins** | Off | Калі ўключаны на мультысайтавай сетцы, толькі Super Admins могуць выкарыстоўваць агента. Адміністратары індывідуальных сайтаў забанены. Мае прыярытэт над "Restrict to Administrators", калі ўключаны абодва. |
| **Allow Subscriber Access** | Off | Калі ўключаны, карыстальнікі з роллёй `subscriber` могуць выкарыстоўваць чат-інтерфейс, але іх функцыянальнасць абмежавана толькі чытаннем (нельга ствараць пасты або змяняць налады). |
| **Disable for Non-Members** | Off | Інтыгрэецца з статусам عзноўніка Ultimate Multisite. Калі ўключаны, чат скрываецца для сайтаў, якія не маюць актыўнага ўзносу. |

### Флагі брэнда (Branding Flags)

| Flag | Default | Description |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | Адмяняе паказванне маркіроўкі ў падокэ ў нізку чат-віджэта. Рэкамендуецца для рэжымаў "біялаўка" (white-label deployments). |
| **Custom Agent Name** | *(blank)* | Замяняе пачатковую маркіроўку "Gratis AI Agent" у заголоўкі чату і адмін-меню на назву вашага прадукту. Пакіньце пустым, каб выкарыстоўваць пачатковую. |
| **Hide Agent Picker** | Off | Калі ўключаны, карыстальнікі не могуць пераключацца паміж п'яць ўбудаванымі агентамі. Актыўны агент фіксуецца на тым, што ўстаноўлена як пачатковы ў Settings → General. |
| **Use Site Icon as Chat Avatar** | Off | Замяняе пачатковы значок AI у заголоўкі чат-віджэта на значок сайта WordPress (устаноўлены ў Appearance → Customize → Site Identity). |

### Застоўванне Зменыў

Націсніце **Save Settings** пасля пераключэння знаёў. Змены ўступаюць у сілу неадкладна — не патрабуецца очишчаць кэш або актываваць плагін.
