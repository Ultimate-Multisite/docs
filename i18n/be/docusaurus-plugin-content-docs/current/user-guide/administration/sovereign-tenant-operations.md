# Права на суверенных орендараў (Sovereign Tenant Operations) {#sovereign-tenant-operations}

Ultimate Multisite версія 1.2.0 дадала інструменты для працы з сувереннымі орендарамі: подсайтami, якія паработаюць са сваёй базай даных, коронкай файловай сістэмы і контекставання маршрутывання, але застануцца бачнымі для адміна сеткі.

Выкарыстоўвайце гэтую старонку пры кіраванні асобнымі кліентскімі сайтам, перадачах удаленых сайтаў або міграцыях, якія пераводзяць звычайны подсайт у суверенную інфраструктуру.

## Што змяняецца для адмінараў {#what-changes-for-administrators}

- **Безстатэйнае аторызавана ўход (Stateless tenant autologin)** — Адмінам сеткі можна паглядзець на суверенный орендаль без залежнасці ад доўгацілую спяваючу статычную сістэму сеансу. Токен SSO мае вызначаны сфера, зафіксаваны прычынамі (origin-pinned), абаротаецца ад працягвання і маецца кароткі час валідацыі.
- **Сувереннае маршрутыванне (Sovereign-aware routing)** — Старыя асобныя сеткі і суверенные орендаль разгружаюцца праз той жа шлях маршруtau сайта, што зніжае розніцы пры запуску паміж старымі і новымі асобнымі усталяванняў.
- **Праверка стану міграцыі (Verified migration state)** — Праверка міграцыі праверка мае нават у hoạchванне карыстальнікаў, права напісання базы даных, статус выпраўлення пашчялін (queue drain status) і адсутнасць старых табліц перад тым, як орендаль будзе прыняты за акончаны.
- **Безпечнейе зняття (Safer teardown)** — Зняцце суверенного орендаля давайце павядавіць токен орендаля чыстым спосабам, каб удаленыя орендалі не пакідалі старыя прыклады базавых даных.

## Пагляд на суверенный орендаль (Visiting a sovereign tenant) {#visiting-a-sovereign-tenant}

1. Отчынэце **Network Admin > Ultimate Multisite > Sites**.
2. Выберыце суверенного орендаля.
3. Выкарыстоўвайце **Visit (SSO)**, калі гэта даступна, замінюючы копіраванне пароляў або стварэнне тэмпых адмінацкіх кантэктаў.

Прылад вылажыць шматкі пакоротга жывёлы токен ухаванню для гэтага орендаля і зафіксуе собыці SSO ў аудит-трасе орендаля. Калі кнопка не працуе, правердзіце, што домен орендаля разгружаецца дазваленым усталяванню і што орендаль можа дасягнуць кінцовай пункту SSO на боку сеткі.

## Паняць нейкіраваных тэнантів (Deleting sovereign tenants) {#deleting-sovereign-tenants}

Далі паняць нейкіраваны тэнант — гэта руйнівны працэс. Перш за ўсё, павярхіце становы бэкапаў і выгружкі, а потым дадай далёкі з экрана кіравання сайтам. У процесе дэмантавання версіі 1.2.0 удаляе учетвароўкі базы даных тэнанта як частку чысці, але адміністраторы павінны ўсё ж праверыць, што хост-уровень базавых даных і палотнакі знікалі, калі вы карыстаецеся далёкімі панелямі для хостингу.

:::warning
Не удаляй нейкіраваны тэнант, пакуль вірыкаецца праверка міграцыі або пакуль у шматі queued запытаў на асінхронны пуш (async push jobs). Пачакай, пакуль праверка не завершыцца, каб дэмантаванне не удаляла учетвароўкі, якія патрэбныя для паколькі прыкіраваных запытаў.
:::
