# Інтэграцыя з WP Engine {#wp-engine-integration}

## Агляд {#overview}
WP Engine — гэта прэміум-хостынг-платформа WordPress з кіраванным рэжымам, якая прадугледжвае аптымізаваную прадукцыйнасць, бяспеку і магчымасць масштабавання для сайтаў WordPress. Гэтая інтэграцыя дазваляе аўтаматычна выпраўляць домены паміж Ultimate Multisite і WP Engine.

## Функцыі {#features}
- Аўтаматычнае выпраўленне доменаў
- Падтрымка субдоменаў для інтэраліцыйных устаноў
- Бесшоўная інтэграцыя з існуючымі сістэмамі WP Engine

## Патрабаванні {#requirements}
Інтэграцыя аўтаматычна выяўляе, ці хостінг выконваецца на WP Engine, і выкарыстоўвае ўбудаваны API WP Engine. Не патрабуецца дадатковай канфігурацыі, калі плагін WP Engine актыўны і належным чынам настроены.

Аднак, калі вам трэба канфігураваць інтэграцыю ўручную, вы можаце вызначыць адзін з гэтых канстант у файле `wp-config.php`:

```php
define('WPE_APIKEY', 'your_api_key'); // Пільгачны метад
// АБО
define('WPE_API', 'your_api_key'); // Альтэрнатыўны метад
```

## Інструкцыі па ўстаноўцы {#setup-instructions}

### 1. Праверка плагіна WP Engine {#1-verify-wp-engine-plugin}

Калі вы хостінг на WP Engine, плагін WP Engine павінен быць ужо ўсталяваны і актываваны. Праверце, што:

1. Плагін WP Engine актыўны
2. Існуе файл `wp-content/mu-plugins/wpengine-common/class-wpeapi.php`

### 2. Актывацыя інтэграцыі {#2-enable-the-integration}

1. У адмініструванні WordPress заййміцеся ў Ultimate Multisite > Settings
2. Пайміце на тэчку "Domain Mapping"
3. Праскользьце да "Host Integrations"
4. Актывуйце інтэграцыю WP Engine
5. Націсніце "Save Changes"

## Як гэта працуе {#how-it-works}

### Выпраўленне доменаў {#domain-syncing}

Калі домен прывязаны ў Ultimate Multisite:

1. Інтэграцыя выкарыстоўвае API WP Engine, каб дадаць домен да вашай устаноўкі WP Engine
2. WP Engine адпраўляе канфігурацыю домена і выкадруванне SSL-сертыфіката
3. Калі прывязка домена адпраўляецца, інтэграцыя адключыць домен ад WP Engine

### Падтрымка субдоменаў {#subdomain-support}

Для інтэраліцыйных устаноў субдоменаў:

1. Інтэграцыя дадае кожны субдомен у WP Engine, калі ствараецца новы сайт
2. WP Engine адпраўляе канфігурацыю субдомена
3. Калі сайт адпраўляецца, інтэграцыя адключыць субдомен ад WP Engine

## Важнае паведамленне {#important-notes}

### Домены-вайлдкарды (Wildcard Domains) {#wildcard-domains}

Для інтэраліцыйных устаноў субдоменаў рэкамендуецца звярнуцца ў падтрымку WP Engine, каб пажадаць канфігурацыю домена-вайлдкарда. Гэта дазваляе ўсім субдоменам працаваць аўтаматычна без неабходнасці дадаваць кожны з іх адным-адным.

### SSL-сертыфікаты {#ssl-certificates}

WP Engine аўтаматычна адпраўляе і пановіць SSL-сертыфікаты для ўсіх доменаў, дададзеных праз гэтую інтэграцыю. Дадатковая канфігурацыя не патрабуецца.

## Выяўленне праблем {#troubleshooting}

### Праблемы з падключэннем API {#api-connection-issues}
- Праверце, што плагін WP Engine актыўны і належным чынам настроены
- Калі вы ўручную вызначылі API key, праверце, што ён правільны
- Звярніцеся ў падтрымку WP Engine, калі ў вас возникнуць цяжкасці з API

### Домен не дададзены {#domain-not-added}
- Праверце журналы Ultimate Multisite на наяўнасць памылак
- Праверце, што домен не дададзены ўжо ў WP Engine
- Убедзіцеся, што ваш план WP Engine падтрымлівае колькасць доменаў, якія вы дадаеце

### Праблемы з субдоменамі {#subdomain-issues}
- Калі субдомены не працуюць, звярніцеся ў падтрымку WP Engine, каб пажадаць канфігурацыю домена-вайлдкарда
- Праверце, што вашы DNS-настройкі правільна канфігураваны для галоўнага домена і субдоменаў
