# Інтеграцыя Closte

## Агляд
Closte — гэта платформа для хостыంగ్‌у WordPress з кіраваннем, якая пабудаваная на інфраструктуры Google Cloud. Гэтая интеграцыя дазваляе аўтаматычна выпраўляць домены і кіраваць сертыфікатам SSL паміж Ultimate Multisite і Closte.

## Функцыі
- Аўтаматычнае выпраўленне доменаў
- Кіраванне сертыфікатам SSL
- Падтрымка доменаў-вайлдкардаў
- Не патрабуе канфігурацыі, калі працуе на Closte

## Патрабаванні
Калі вы выкарыстоўваеце Closte, наступная канстанта павінна быць вызначана ў файле `wp-config.php`:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Звычайна гэтая канстанта ўжо вызначана, калі вы хостуеце на Closte.

## Інструкцыі па наладзе

### 1. Праверце ваш API-ключ Closte

Калі вы хостуеце на Closte, канстанта `CLOSTE_CLIENT_API_KEY` павінна ўжо быць вызначана ў вашым файле `wp-config.php`. Вы можаце гэта праверыць, прагледжаўшы свой файл `wp-config.php`.

### 2. Уключце интеграцыю

1. У адміні stratum WordPress заййміцеся ў Ultimate Multisite > Settings
2. Пайдзіце ў тэчку "Domain Mapping"
3. Зробіце скрол да "Host Integrations"
4. Уключыце интеграцыю Closte
5. Націсніце "Save Changes"

## Як гэта працуе

Калі домен выпраўляецца ў Ultimate Multisite:

1. Інтеграцыя адпраўляе запыт да API Closte, каб дадаць домен да вашай праграмы.
2. Closte аўтаматычна займаецца прадастаўленнем сертыфікатаў SSL.
3. Калі выматкаванне домен выпраўляецца, интеграцыя выдаліць домен з Closte.

Інтеграцыя таксама працуе з настаўленнем паслядоўнасці праверкі DNS ў Ultimate Multisite, што дазваляе вам наладзіць, як часта сістэма праверыць распыленне DNS і выдачу сертыфікатаў SSL.

## Стваৰэнне рэкаordu домена

Гэтая интеграцыя гарантуе, што калі ствараецца або дубліраваецца сайт, аўтаматычна ствараецца рэкакорд домена. Гэта асабліва важна для интеграцыі Closte, бо стварэнне рэкаordu домена запускае API Closte для стварэння домена і сертыфікату SSL.

## Выяўленне праблем

### Праблемы з падключэннем API
- Праверце, ці правільны ваш API-ключ Closte
- Упэўніцеся, што ваш кант Closte мае неабходныя права

### Праблемы з сертыфікатам SSL
- Closte можа займець трохі часу для выдачы сертыфікатаў SSL (завычайна 5-10 хвілін)
- Праверце, што вашы домены правільна паказваюцца на IP-адрэс вашага сервера Closte
- Прагледзьце рэкаорды DNS для вашага домена, каб пабедацца, што яны правільна наладжаны

### Домен не дададзены
- Прагледзьце журналы Ultimate Multisite на наяўнасць памылак
- Упэўніцеся, што домен яшчэ не дададзены ў Closte
- Упэўніцеся, што рэкаорды DNS вашага домена правільна наладжаны

### Паслядоўнасць праверкі DNS
- Калі сертыфікаты SSL займаюць занадта шмат часу для выдачы, вы можаце адрэгуляваць паслядоўнасць праверкі DNS ў настаўленнях "Domain Mapping"
- Пачатковая паслядоўнасць — 300 секунд (5 хвілін), але вы можаце ўсталяваць яе на мінімум 10 секунд для больш хуткага праверлення падчас тэставання
