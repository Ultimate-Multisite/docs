# Інструменты і канфігурацыя

Theme Builder выкарыстоўвае набор інструментаў для аналізу, праектавання і стварэння вашага WordPress сайта. У версіі 1.16.0 **sd-ai-agent/site-scrape цяпер з'яўляецца інструментам Узроўню 1**, што робіць яго даступным па змаўчанні ў Theme Builder. Superdav AI Agent v1.18.0 таксама прадстаўляе асобна распаўсюджваны пашыраны спадарожны пакет для давераных працоўных працэсаў распрацоўшчыкаў; гэтыя інструменты не ўваходзяць у асноўны пакет WordPress.org і павінны быць усталяваны і аўтарызаваны асобна.

## Даступныя інструменты

### Інструменты Узроўню 1 (заўсёды даступныя)

Інструменты Узроўню 1 даступныя па змаўчанні ў Theme Builder без дадатковай канфігурацыі.

#### sd-ai-agent/site-scrape

**Мэта:** Аналізаваць існыя вэб-сайты, каб здабываць натхненне для дызайну, структуру кантэнту і функцыянальнасць.

**Магчымасці:**

- **Аналіз вэб-сайта** — збіраць і аналізаваць сайты канкурэнтаў або сайты для натхнення
- **Вылучэнне дызайну** — вызначаць колеры, шрыфты і шаблоны макета
- **Структура кантэнту** — разумець арганізацыю старонак і іерархію
- **Выяўленне функцый** — вызначаць plugins і функцыянальнасць
- **Аналіз прадукцыйнасці** — правяраць хуткасць старонак і аптымізацыю
- **SEO-аналіз** — правяраць метатэгі і структураваныя даныя

**Выкарыстанне:**

```
Прааналізуй дызайн example.com і выкарыстай яго як натхненне для майго сайта.
```

**Што ён здабывае:**

- Каляровая палітра і выкарыстанне колераў
- Тыпаграфіка (шрыфты і памеры)
- Структура макета і інтэрвалы
- Шаблоны навігацыі
- Арганізацыя кантэнту
- Выкарыстанне выяў і медыя
- Інтэрактыўныя элементы
- Адаптыўнасць для мабільных прылад

**Абмежаванні:**

- Не можа збіраць даныя з сайтаў, абароненых паролем
- Улічвае robots.txt і палітыкі сайта
- Можа не захапіць дынамічны кантэнт
- Сайты з вялікай колькасцю JavaScript могуць мець абмежаваны аналіз
- Аналіз вялікіх сайтаў можа заняць больш часу

### Інструменты Узроўню 2 (неабавязковыя)

Інструменты Узроўню 2 даступныя, калі яны яўна ўключаны ў наладах Theme Builder.

#### Пашыраная аналітыка

Прадастаўляе падрабязныя метрыкі прадукцыйнасці:

- Час загрузкі старонкі
- Core Web Vitals
- SEO-ацэнка
- Ацэнка даступнасці
- Ацэнка найлепшых практык

#### Аптымізатар кантэнту

Аналізуе і прапануе паляпшэнні для:

- Чытэльнасці
- SEO-аптымізацыі
- Выкарыстання ключавых слоў
- Структуры кантэнту
- Размяшчэння заклікаў да дзеяння

### Інструменты Узроўню 3 (прэміум)

Інструменты Узроўню 3 патрабуюць дадатковай канфігурацыі або прэміум-функцый.

#### Генератар кантэнту AI

Генеруе кантэнт для:

- Апісанняў прадуктаў
- Старонак паслуг
- Запісаў блога
- Метаапісанняў
- Тэкстаў заклікаў да дзеяння

#### Пашыраныя інструменты дызайну

Прадастаўляе пашыраныя магчымасці дызайну:

- Генерацыя карыстальніцкага CSS
- Стварэнне анімацый
- Дызайн інтэрактыўных элементаў
- Пашыраная тэорыя колеру
- Аптымізацыя тыпаграфікі

### Пашыраныя спадарожныя інструменты (асобны пакет)

Пашыраны спадарожны пакет распаўсюджваецца асобна ад асноўнага выпуску Superdav AI Agent. Ён прызначаны для давераных асяроддзяў распрацоўшчыкаў і ўладальнікаў сайтаў, дзе адміністратары яўна дазваляюць агенту выкарыстоўваць інструменты з высокім уздзеяннем.

Магчымасці пашыранага спадарожнага пакета могуць уключаць:

- **Інструменты файлавай сістэмы** — праглядаць і змяняць ухваленыя файлы ў давераных асяроддзях
- **Інструменты базы даных** — запытваць або абнаўляць даныя сайта пры яўнай аўтарызацыі
- **Інструменты WP-CLI** — запускаць каманды абслугоўвання і праверкі WordPress
- **Інструменты дыспетчара REST** — выклікаць зарэгістраваныя REST endpoints праз кантраляваныя працоўныя працэсы дыспетчарызацыі
- **Інструменты канструктара plugin** — ствараць каркас і ітэраваць код plugin
- **Інструменты здымкаў Git** — ствараць здымкі перад рызыкоўнымі аперацыямі распрацоўшчыка
- **Інструменты кіравання карыстальнікамі** — дапамагаць праглядаць або карэктаваць запісы карыстальнікаў, дзе гэта дазволена
- **Інструменты бенчмаркінгу** — запускаць бенчмаркі прадукцыйнасці або магчымасцей для агляду распрацоўкі

Не дакументуйце іх як агульнадаступныя інструменты Theme Builder для кожнай усталёўкі. Яны даступныя толькі калі пашыраны спадарожны пакет усталяваны, уключаны і абмежаваны даверанымі адміністратарамі.

## Канфігурацыя інструментаў

### Уключэнне інструментаў

Каб уключыць дадатковыя інструменты ў Theme Builder:

1. Перайдзіце ў **Gratis AI Agent → Налады**
2. Перайдзіце ў **Theme Builder → Інструменты**
3. Уключайце або выключайце інструменты паводле патрэбы
4. Захавайце налады

### Дазволы інструментаў

Некаторыя інструменты патрабуюць дазволаў:

- **Збор даных сайта** — патрабуе доступу да інтэрнэту
- **Аналітыка** — патрабуе інтэграцыі з Google Analytics
- **Генерацыя кантэнту** — патрабуе API-ключоў
- **Пашыраныя функцыі** — могуць патрабаваць прэміум-падпіскі
- **Пашыраныя спадарожныя інструменты** — патрабуюць асобна распаўсюджванага пашыранага пакета і яўнага даверу адміністратара

### API-ключы і ўліковыя даныя

Наладзьце API-ключы для інструментаў, якім яны патрэбныя:

1. Перайдзіце ў **Gratis AI Agent → Налады → API-ключы**
2. Увядзіце ўліковыя даныя для кожнага інструмента
3. Праверце злучэнне
4. Захавайце бяспечна

### Усталёўка пашыранага спадарожнага пакета

Пашыраны спадарожны пакет выпускаецца асобна ад асноўнага ZIP WordPress.org. Усталёўвайце яго толькі з афіцыйнага канала распаўсюджвання праекта для адпаведнага выпуску, затым абмяжуйце доступ даверанымі адміністратарамі. Праверце шлюзы чалавечага ўхвалення і паводзіны здымкаў git перад уключэннем інструментаў файлавай сістэмы, базы даных, WP-CLI, дыспетчара REST, канструктара plugin, кіравання карыстальнікамі або бенчмаркінгу на вытворчым сайце.

## Выкарыстанне sd-ai-agent/site-scrape

### Базавае выкарыстанне

Папрасіце Theme Builder прааналізаваць вэб-сайт:

```
Прааналізуй дызайн сайта майго канкурэнта на competitor.com
і прапануй паляпшэнні дызайну для майго сайта.
```

### Канкрэтны аналіз

Запытайце канкрэтныя тыпы аналізу:

```
Вылучы каляровую палітру з example.com і выкарыстай яе як натхненне.
```

```
Прааналізуй структуру навігацыі example.com і прымяні падобную
арганізацыю да майго сайта.
```

```
Правер мабільную адаптыўнасць example.com і пераканайся, што мой сайт
такі ж адаптыўны.
```

### Параўнальны аналіз

Параўнайце некалькі сайтаў:

```
Параўнай дызайны site1.com і site2.com і ствары дызайн,
які аб'ядноўвае найлепшыя элементы абодвух.
```

## Абмежаванні і меркаванні адносна інструментаў

### Абмежаванне хуткасці

- Scraping абмежаваны па частаце, каб прадухіліць перагрузку сервера
- Некалькі запытаў да аднаго і таго ж сайта могуць абмяжоўвацца
- Аналіз вялікіх сайтаў можа заняць больш часу

### Дакладнасць змесціва

- Дынамічнае змесціва можа быць захоплена не цалкам
- Змесціва, адлюстраванае праз JavaScript, можа быць няпоўным
- Даныя ў рэальным часе могуць не адлюстроўвацца
- Частка змесціва можа знаходзіцца за платным доступам

### Юрыдычныя і этычныя меркаванні

- Паважайце robots.txt і палітыкі сайта
- Не Scraping абароненае аўтарскім правам змесціва для ўзнаўлення
- Выкарыстоўвайце аналіз для натхнення, а не для капіравання
- Пераканайцеся, што ў вас ёсць правы на выкарыстанне вынятага змесціва
- Выконвайце ўмовы абслугоўвання прааналізаваных сайтаў

### Уплыў на прадукцыйнасць

- Scraping вялікіх сайтаў можа заняць час
- Некалькі адначасовых Scraping могуць быць павольнейшымі
- Сеткавае падключэнне ўплывае на хуткасць
- Вялікія наборы даных могуць патрабаваць больш апрацоўкі

## Лепшыя практыкі

### Выкарыстанне аналізу сайта

1. **Аналізуйце некалькі сайтаў** — збірайце натхненне з некалькіх крыніц
2. **Засяродзьцеся на структуры** — вывучайце шаблоны макета і арганізацыі
3. **Вынімайце колеры** — выкарыстоўвайце каляровыя палітры як адпраўныя пункты
4. **Вывучайце тыпаграфіку** — вызначайце спалучэнні шрыфтоў, якія вам падабаюцца
5. **Праглядзіце навігацыю** — зразумейце арганізацыю меню

### Этычны Scraping

1. **Паважайце robots.txt** — прытрымлівайцеся рэкамендацый сайта
2. **Не капіруйце змесціва** — выкарыстоўвайце аналіз толькі для натхнення
3. **Правярайце правы** — пераканайцеся, што можаце выкарыстоўваць вынятае змесціва
4. **Указвайце крыніцы** — пазначайце крыніцы натхнення
5. **Выконвайце ўмовы** — выконвайце ўмовы абслугоўвання сайта

### Максімізацыя эфектыўнасці інструмента

1. **Будзьце канкрэтнымі** — запытвайце канкрэтныя тыпы аналізу
2. **Дадавайце кантэкст** — растлумачце мэту вашага сайта
3. **Задавайце чаканні** — апішыце свае дызайнерскія мэты
4. **Ітэруйце** — удакладняйце на аснове вынікаў
5. **Камбінуйце інструменты** — выкарыстоўвайце некалькі інструментаў для комплекснага аналізу

## Вырашэнне праблем

### Сайт не паддаецца Scraping

- Праверце, ці сайт публічна даступны
- Пераканайцеся, што robots.txt дазваляе Scraping
- Паспрабуйце іншы сайт
- Праверце інтэрнэт-злучэнне
- Звярніцеся ў падтрымку, калі праблема захоўваецца

### Няпоўны аналіз

- Сайт можа мець дынамічнае змесціва
- JavaScript можа быць адлюстраваны не цалкам
- Вялікія сайты могуць перавысіць час чакання
- Паспрабуйце замест гэтага аналізаваць канкрэтныя старонкі
- Запытайце канкрэтныя тыпы аналізу

### Павольная прадукцыйнасць

- Аналіз вялікіх сайтаў займае больш часу
- Некалькі адначасовых Scraping павольнейшыя
- Сеткавае падключэнне ўплывае на хуткасць
- Паспрабуйце ў гадзіны найменшай нагрузкі
- Спачатку аналізуйце меншыя раздзелы
