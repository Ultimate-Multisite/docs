# Інструменты і канфігурацыя

Theme Builder выкарыстоўвае набор інструментаў для аналізу, расправы і стварэння вашага сайта ў WordPress. У версіі 1.16.0 **sd-ai-agent/site-scrape стаў інструментам Tier 1**, што робіць яго даступным па спецыяталежнасці ў Theme Builder.

## Даступныя інструменты

### Інструменты Tier 1 (Заўсёды даступныя)

Інструменты Tier 1 даступныя па спецыяталежнасці ў Theme Builder без неабходнай дадатковай канфігурацыі.

#### sd-ai-agent/site-scrape

**Мэта:** Аналіз існуючых сайтаў для выяўлення ідэй для дызайну, структуры кантэнту і функцыянальнасці.

**Магчымасці:**

- **Аналіз сайта:** Сканіраванне і аналіз сайтаў канкурэнтаў або ўдохнення
- **Выяўленне дызайну:** Выяўленне колераў, шрыфтаў і паслядоўных паслядоўванняў раскладу
- **Структура кантэнту:** Паазнаванне арганізацыі і іерархіі старонкі
- **Выяўленне функцый:** Паазнаванне плагінаў і функцыянальнасці
- **Аналіз прадуккаласці:** Прыверка хуткасці загрузкі старонкі і аптымізацыі
- **SEO аналіз:** Пагледжаньне метатэгаў і структураваных дадзеных

**Выкарыстанне:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**Што ён выяўляе:**

- Палітра колераў і выкарыстанне колераў
- Типографія (шрыфты і памеры)
- Структура раскладу і паміжкрэневыя прасторы
- Паслядоўныя навігацыі
- Арганізацыя кантэнту
- Выкарыстанне малюнкаў і медыя
- Інтэрактыўныя элементы
- Мабільная адаптыўнасць

**Обмежаванні:**

- Не можа сканіраваць сайты, якія абаронены паролем
- Паважае robots.txt і палітыку сайта
- Магчыма, што не ўлоўіць дынамічны кантэнт
- Сайты з вялікім выкарыстаннем JavaScript могуць мець абмежаваны аналіз
- Вялікія сайты могуць патрабаваць больш часу для аналізу

### Інструменты Tier 2 (Па жаданні)

Інструменты Tier 2 даступныя, калі яны вызначана актываваны ў напрамленні канфігурацыі Theme Builder.

#### Advanced Analytics

Прадастаўляе падрабязныя метрыкі прадуккаласці:

- Час загрузкі старонкі
- Core Web Vitals
- SEO score
- Accessibility score
- Best practices score

#### Content Optimizer

Аналізуе і прапануе паляпменне для:

- Чытальнасці
- SEO аптымізацыі
- Выкарыстання ключавых словаў
- Структуры кантэнту
- Размяшчэння заклічэнняў да дзеяння (Call-to-action)

### Інструменты Tier 3 (Прэміум)

Інструменты Tier 3 патрабуюць дадатковай канфігурацыі або прэміум-функцый.

#### AI Content Generator

Генеруе кантэнт для:

- Апісаў прадуктаў
- Стравак паслуг
- Публікацый блога
- Метаапісанняў
- Тэксту заклічэнняў да дзеяння

#### Advanced Design Tools

Прадастаўляе пашыраваныя магчымасці дызайну:

- Генерацыя кастомнага CSS
- Стваৰэнне анімацыі
- Дызайн інтэрактыўных элементаў
- Пашыравастая тэорыя колераў
- Аптымізацыя тыпографіі

## Канфігурацыя інструментаў

### Актывацыя інструментаў

Каб актываваць дадатковыя інструменты ў Theme Builder:

1. Адыйце ў **Gratis AI Agent → Settings**
2. Адыйце ў **Theme Builder → Tools**
3. Пераключайце інструменты ўключ/выключ па патрабаванні
4. Захаваць напрамленне

### Паўлетці інструментаў

Некаторыя інструменты патрабуюць паўлеткі:

- **Site scraping** — патрабуе доступу да Інтэрнэту
- **Analytics** — патрабуе інтэграцыі Google Analytics
- **Content generation** — патрабуе API ключаў
- **Advanced features** — можа патрабаваць прэміум-арыентацыі

### API ключы і ўваходныя даведы

Настройце API ключы для інструментаў, якія іх патрабуюць:

1. Адыйце ў **Gratis AI Agent → Settings → API Keys**
2. Увярце ўваходныя даведы для кожнага інструмента
3. Праверыць злучэнне
4. Пахаваць бяспечна

## Выкарыстанне sd-ai-agent/site-scrape

### Базавае выкарыстанне

Папросіце Theme Builder прааналізаваць сайт:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### Спецыфічны аналіз

Пажадаць спецыфічныя тыпы аналізу:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### Параўнанне аналізу

Параўняць некалькі сайтаў:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## Обмежаванні і падыгледжанне інструментаў

### Ліміт сканіравання (Rate Limiting)

- Сканіраванне лімітавана, каб zapрэчыць перагрузванню сервера
- Некалькі запытаў на адзін сайт могуць быць абмежаваны (throttled)
- Вялікія сайты могуць патрабаваць больш часу для аналізу

### Дакладнасць кантэнту

- Дынамічны кантэнт можа не быць у цэлым ўлоўлены
- Кантэнт, сфармаваны JavaScript, можа быць недастатковым
- Рэальны час дадзеныя можа не адлюстравацца
- Некаторыя кантэнт можа быць за платных сцяжках (paywalls)

### Паканаўныя і этычныя падыгледжанні

- Паважаць robots.txt і палітыку сайта
- Не сканіраваць кантэнт, які абаронены аўтарскім царкоўем, для рэпрадукцыі
- Выкарыстоўваць аналіз для ўдохнення, а не для капіявання
- Праверыць, што ў вас ёсць права на выкарыстанне выяўленага кантэнту
- Выконваць умоваў паслугаў аналізаваных сайтаў

### Уплыў на прадуккаласць

- Сканіраванне вялікіх сайтаў можа займець час
- Некалькі адначасовых сканіраванняў могуць быць больш павольнымі
- Злучэнне ў сетцы ўплывае на хуткасць
- Вялікія наборы дадзеных могуць патрабаваць больш прапрацоўкі

## Найлепшыя практыкі

### Выкарыстанне аналізу сайта

1. **Аналізуйце некалькі сайты** — збірайце ўдохновение з некалькіх крыніў
2. **Фарказуйцеся на структуры** — вывучце паслядоўныя паслядоўванні раскладу і арганізацыі
3. **Выяўляйце колеры** — выкарыстоўвайце палітравыя колеры як пачатковыя пункты
4. **Вывучце тыпографіку** — паазначце камабінацыі шрыфтаў, якія вам падыходзяць
5. **Пагледзьце навігацыю** — разуменне арганізацыі меню

### Этычнае сканіраванне

1. **Паважайце robots.txt** — выконвайце паказаныя на сайце правілы
2. **Не капіядзіце кантэнт** — выкарыстоўвайце аналіз толькі для ўдохнення
3. **Праверыце права** — пераканайцеся, што вы можаце выкарыстоўваць выяўлены кантэнт
4. **Укабляйце крыніцы** — адзначайце крыніцы ўдохнення
5. **Выконвайце ўмовы** — выконвайце ўмовы паслугаў сайтаў

### Максімальнае выкарыстанне інструментаў

1. **Буйце спецыфічнымі** — папрасіце спецыфічныя тыпы аналізу
2. **Падавайце кантэкст** — паясніце мэту вашага сайта
3. **Вызначайце чакаванні** — апішыце свае мэты дызайну
4. **Ітэрацыя** — рэагуйце на вынікі
5. **Камабінаванне інструментаў** — выкарыстоўвайце некалькі інструментаў для паўнавартаснага аналізу

## Выяўленне праблем

### Сайт не сканіраваецца

- Праверце, ці адкрыты сайт для публічнага доступу
- Праверце, ці дазваляе robots.txt сканіраванне
- Паспрабуйце іншы сайт
- Праверце злучэнне ў Інтэрнэт
- Звярніцеся на падтрымку, калі праблема працягвае

### Некамплектыўны аналіз

- Сайт можа мець дынамічны кантэнт
- JavaScript можа не быць у цэлым сфармаваны
- Вялікія сайты могуць прайсці па выніку (timeout)
- Паспрабуйце аналізаваць спецыфічныя старонкі
- Пажадайце спецыфічныя тыпы аналізу

### Павольная прадуккаласць

- Вялікія сайты патрабуюць больш часу для аналізу
- Некалькі адначасовых сканіраванняў павольнішыя
- Злучэнне ў сетцы ўплывае на хуткасць
- Паспрабуйце ў непйкаштоўны час
- Сперш аналізуйце меншыя секцыі
