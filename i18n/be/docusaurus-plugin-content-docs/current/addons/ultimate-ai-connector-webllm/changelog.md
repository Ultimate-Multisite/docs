# Журнал змены Ultimate AI Connector для WebLLM {#ultimate-ai-connector-for-webllm-changelog}

## Версія 1.1.0 — Выпушчана 2026-04-09 {#version-110--released-on-2026-04-09}

* Ноўнасць: Плаваючы віджэт чату з паказальнікам стану ў адмін-бар — любы заведваны карыстальнік можа даць запыт LLM на босмарзе непасрэдна з фронтэнда.
* Ноўнасць: Папраўнік SharedWorker — мадыфікатар MLC цяпер працуе ў SharedWorker, таму некалькі ўкладанняў (табаў) выкарыстоўваюць адну сесію GPU, а не "бараць яго паміж сабой".
* Ноўнасць: Інтерцэптар middleware apiFetch — Запыты WordPress REST, якія падыходзяць да пасылак AI Client SDK, праходзяць празрысты маршрут на мясцовы брокер WebLLM, не патрабуючы HTTP-цыклу паўроту.
* Ноўнасць: UI ў наступствах віджэта ў панялічцы Connector для пераключэння віджэта чату і канфігурацыі паводзінаў аўтаматычнага запыту.
* Выпраўка: Вымушаны бэкэнд кэша IndexedDB, каб загрузка вагтаў мадэлі праходзіла праз рэдырэкты HuggingFace xet CDN, якія pháюць пачатковы шлях Cache API.
* Выпраўка: Паўзець перавышэнне KV-кэша для context_window для мадэляў замацавання (у іх няма дэкадэра, і гэтае перавышэнне выклікае памылку працу).
* Выпраўка: Адэ কৰি кандзідата мадэлі для халаднага запуску ў /webllm/v1/models да таго, як загрузіцца ўкладка папраўніка, каб карыстальнікі SDK бачылі мадэль неадкладна.
* Выпраўка: Падключэнне WebLLM да фільтра wpai_preferred_text_models, каб функцыя AI Experiments маршрутызавалася на босмарзны мадыфікатар, калі канфігуруецца мадэль.
* Уляпнена: Выпраўленыя праблемы з кэшаванне (cache-busting), нормалізацыяй кантэнту і адпраўленнем рэцэнтнага абору апаратнага забеспячэння, якія вынік тэставання "з канца да канца".
