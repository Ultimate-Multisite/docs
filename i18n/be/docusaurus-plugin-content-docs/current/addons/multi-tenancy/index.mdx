# Мультытэнантнасць {#multi-tenancy}

Размяркоўвайце падсайты па серверах і базах даных для прадукцыйнасці і бяспекі

## Асноўныя моманты выпуску версіі 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 дадае аснову суверэннага арандатара для вытворчых працоўных працэсаў ізаляцыі:

- Бязстанавы аўтаўваход арандатара з кароткачасовымі, абароненымі ад паўторнага прайгравання наведваннямі SSO.
- Ізаляцыя базы даных і файлавай сістэмы для кожнага падсайта для суверэнных арандатараў.
- Магчымасць мультытэнантнасці Hostinger для стварэння арандатараў з улікам хоста.
- Кантрольныя шлюзы праверкі суверэннай міграцыі для праверак састарэлых даных і асінхроннай апрацоўкі адпраўкі.
- Наладжвальная прывязка хоста базы даных на той жа машыне для ўсталяванняў Bedrock, FrankenPHP і localhost.
- Палепшанні пачатковай загрузкі арандатара, запісвальніка базы даных, пачатковай загрузкі схемы і стварэння карыстальнікаў.
- Больш бяспечнае выдаленне суверэннага арандатара, якое ачышчае ўліковыя даныя арандатара.

Глядзіце карыстальніцкія кіраўніцтвы па [працоўным працэсе суверэннай міграцыі](/user-guide/getting-started/sovereign-migration-workflow), [ізаляцыі мультытэнантнасці](/user-guide/configuration/multi-tenancy-isolation) і [аперацыях з суверэннымі арандатарамі](/user-guide/administration/sovereign-tenant-operations).

## Патрабаванні {#requirements}

- WordPress 5.3 або вышэй
- PHP 7.4 або вышэй
- Плагін Multisite Ultimate (актыўны)

## Усталяванне {#installation}

1. Запампуйце файлы дадатка ў ваш каталог `/wp-content/plugins/`
2. Актывуйце плагін праз меню 'Plugins' у WordPress
3. Наладзьце параметры дадатка ў сеткавым адміністраванні

## Распрацоўка {#development}

### Наладжванне {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Тэставанне {#testing}

Дадатак уключае комплексную сістэму тэставання:

- **Модульныя тэсты**: Тэсты на базе PHPUnit з наборам тэстаў WordPress
- **Базавы клас тэстаў**: Забяспечвае дапаможныя метады для распаўсюджаных тэставых сцэнарыяў
- **Пакрыццё тэстамі**: Наладжана для адсочвання пакрыцця кода
- **Тэставанне Multisite**: Тэсты запускаюцца ў асяроддзі multisite

### Стандарты кода {#code-standards}

- **Стандарты кадавання WordPress**: Забяспечваюцца праз PHPCS
- **Сумяшчальнасць з PHP 7.4+**: Падтрымліваюцца сучасныя магчымасці PHP
- **Статычны аналіз**: Інтэграцыя PHPStan для праверкі тыпаў
- **Мадэрнізацыя кода**: Правілы Rector для абнаўленняў PHP

### Сістэма зборкі {#build-system}

- **Апрацоўка рэсурсаў**: Мінімізацыя файлаў CSS/JS
- **Пераклад**: Генерацыя файла POT
- **Упакоўка**: Стварэнне архіва для распаўсюджвання
- **Рэжым распрацоўкі**: Немінімізаваныя рэсурсы для адладкі

## Канфігурацыя {#configuration}

Дадатак падтрымлівае розныя параметры канфігурацыі, даступныя праз інтэрфейс сеткавага адміністравання.

### Канфігурацыя суверэннага арандатара {#sovereign-tenant-configuration}

Для суверэнных арандатараў наладзьце базу даных арандатара, каранёвы каталог файлавай сістэмы, прывязку хоста базы даных і магчымасць хоста перад маршрутызацыяй трафіку да арандатара. Запускайце праверку міграцыі пасля кожнай змены, якая ўплывае на ўліковыя даныя базы даных, прывязку хоста, маршрутызацыю або стварэнне карыстальнікаў.

Выкарыстоўвайце `localhost` толькі тады, калі асяроддзе выканання арандатара і дазволы базы даных чакаюць лакальны сокет або доступ на той жа машыне. Выкарыстоўвайце `127.0.0.1` або імя хоста сэрвісу, калі гэта хост дазволу, наладжаны вашым пастаўшчыком.

### Магчымасць Hostinger {#hostinger-capability}

Магчымасць мультытэнантнасці Hostinger працуе з агульным інтэграцыйным токенам Hostinger. Пераканайцеся, што дамен прысутнічае ў hPanel, DNS указвае на Account Hostinger, а параметры базы даных арандатара адпавядаюць hPanel перад праверкай.

## Хукі і фільтры {#hooks-and-filters}

### Дзеянні {#actions}

- `ultimate-multisite-addon-template_init` - Спрацоўвае пасля ініцыялізацыі дадатка
- `ultimate-multisite-addon-template_loaded` - Спрацоўвае пасля загрузкі ўсіх файлаў дадатка

### Фільтры {#filters}

- `ultimate-multisite-addon-template_settings` - Змяніць параметры дадатка
- `ultimate-multisite-addon-template_enabled` - Перазапісаць стан уключэння/выключэння дадатка
