# Журнал змены {#changelog}

## 1.9.0 — Выпушчаны 2026-04-28 {#190--released-on-2026-04-28}

### Новыя {#new}

- **Способнасць `create_contact_form`** — стварае формы кантактаў, выкарыстоўваючы актыўны plugin для форм (Contact Form 7, WPForms, Fluent Forms або Gravity Forms), і вяртае shortcode, які гатова для ўстаўлення ў любы паст або старонку.
- **Способнасць `set_featured_image`** — пазначае апіляванне (featured image) для паста або старонкі па існуючым ID прыдатка з Media Library або па аддаленным URL; аўтаматычна імпортуе імадж, калі прадстаўлены URL.
- **Способнасць `batch_create_posts`** — стварае некалькі пастаў па выкліку адной способнасці. Падтрымлівае тыя ж параметры, што і `create_post`, паведамляе пра поспех/неўдачу для кожнага паста і мае неабавязвы рэжым `stop_on_error`.
- **Параметр `page_template`** — дададзены ў `create_post` і `update_post`. Пазначае файл PHP-шаблона (напрыклад, `page-full-width.php`) падчас стварэння або рэдагавання. Для `update_post` перадаць пусты стрык, каб вяртацца да ўстаноўчага шаблону тэмы.
- **Client-side screenshot abilities** — `capture_screenshot`, `compare_screenshots` і `review_page_design`. Спраймаюць поўныя або viewport-скрыны жывых старонкі праз server-side headless browser, выяўляюць адрозненні паміж двума скрынамі і прадастаўляюць агляд дизайна, згенераваны AI, які ахоплівае раскладку, тыпографіку, колер і даступнасць.
- **Пяць ўбудаваных агентаў** — Content Writer, Site Builder, Design Studio, Plugin Manager і Support Assistant. Кожны агент мае выклікае набор інструментаў, спецыяльна наладжаваны system prompt і пачатковыя прапановы. Перайключаць можна праз новы **Agent Picker** у загалоўным затыллю чату. Сведыце ў [Built-in Agents](../../user-guide/configuration/built-in-agents).
- **Feature flags** — новая вкладка **Settings → Feature Flags** з пераключателями кантролю доступу (обмежаванне для адміністратараў, абмежаванне для адміністратараў сеткі, доступ для subscripters, выключэнне для нечлебаў) і варыянем брэндынг (скрыць апісаванне ў падвале, карыстальнік кастомнага імя агента, схаваць Agent Picker, выкарыстоўваць іконку сайта як аватар чату). Сведыце ў [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings).
- **Адновленне апошняй сесіі** — паніжжае чат-панель аўтаматычна перазагружае найбольш нядаўнюю размову пры загрузцы старонкі і пры адкрыцці віджета, таму кантэкст ніколі не губляецца падчас навігацыі па старонках.
- **Паўيينгавыя спасылкі на дзеянні** — хуткія спасылкі на Settings і Abilities Registry цяпер з'яўляюцца на экране WordPress **Plugins → Installed Plugins** пад апісанням plugin.

### Уляпнены {#improved}

- **Папыт адноўлення крыніцы імадж** — агент цяпер папытваецца ўсі ўказаныя ў вольным доступе крыніцы імадж, перш чым пераходзіць да імаджа, згенераванага AI, у выпадку неўдачы загрузкі.
- **Панель інфармацыі пра Model** — заўсёды бачная ў загалоўным затыллю чату; больш не схаваецца пасля першага паведамлення.
- **Паводзіны аўтаматычнага скролінга** — аўтаматычны скролінг пакідае прабел, калі карыстальнік скроліць ўверх, каб прачытаць; з'яўляецца і аўтаматычна знікае плывучы кнопка **Scroll to bottom**, калі карыстальнік дасягае найнядаўнейшага паведамлення.
- **UI Agent Picker** — перапрацаваны ў накладку ў фарма-табліцы з іконкамі для кожнага агента, што паляпшае ідэнтыфікацыю і пераключэнне агентаў ў адзіны погляд.
- **Lazy-loaded JS chunks** — пачатковы JavaScript-бандл чат-віджета цяпер падзелены на lazy-loaded chunks, што зніжае пачатковы памер бандла на 75–90%.
- **Перапрацоўка чат-віджета** — адзіны AI-іконка замяняе ранейшую кастомную аватарку; адпавядае ўбудаванай сістэме агентаў.
- **URL linkification** — URL-адрэсы, якія з'яўляюцца ў сістэмных паведамленнях і бабліках паведамленняў пра памылкі, цяпер адбываюцца як клікабельныя спасылкі.

### Выправлены {#fixed}

- **Выяўленне Abilities** — выпраўлены апісанне, адпраўкі system prompt і вырівнанне namespace, таму што ўсе abilities надзейна з'яўляюцца ў спісе інструментаў агента.
- **Кэш Providers** — providers цяпер кэшуюцца на ўсім сайце праз лічыльнік версій, zapixваючы праблемы зісталеных providers ў сетках multisite.
- **`ability_invalid_output`** — выпраўлена ў 12 обработчыках abilities; усе вяртаюць належна структураваныя адказы JSON.
- **Pipeline `pending_client_tool_calls`** — звязаны ад пачатку да канца, таму што выклікі інструмента на client-side заканчваюцца правільна і вяртаюць вынікі ў model.
- **History drawer** — неадворотныя змены выключэнні з спісу аднаўлення; спасылка **View full history** цяпер працуе правільна.
- **Сістэма змены/аднаўлення** — выпраўлены пяць асобных памылак і зроблены адзінства ў новым адмінскім інтэрфейсе.
- **Toast для захавання Settings** — snackbar-паведамленне цяпер надзейна з'яўляецца пасля кліку на **Save Settings**.
- **Контэкстнае меню Trash** — дададзена варыясць **Delete Permanently**, таму што элементы можна хард-вынасці без пакідання рэжыму Trash.
- **Рэдагаванне і адпраўка** — клік на іконку рэдагавання цяпер уводзіць рэжым рэдагавання толькі для кнопкага паведамлення, а не для ўсіх паведамленняў у тэрадзе.
- **Высота раскладу чату** — чат-панель адаптуе сваю вышыню, калі ўстаўленыя plugin кантэнт (адмінскія паведамленні, баннеры) з'яўляюцца над старонкай, zapixваючы вобласць уводзін ад таго, што яна не з'еўецца з экрана.

## 1.4.0 — Выпушчаны 2026-04-09 {#140--released-on-2026-04-09}

### Новыя {#new-1}

- **Команда бенчмарка WP-CLI** (`wp gratis-ai-agent benchmark`) — прагоняе набор бенчмаркаў Agent Capabilities v1 з каманднай крокі для CI pipelines і працэсаў ацэны model. Падтрымлівае прагон па кожным пытанні, перавызначэнне providers/model і выхад JSON/CSV.

- **Способнасці для Custom Taxonomy** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. Падтрымлівае іерархічныя і плоскія taxonomie з неабавязўнымі rewrite slugs.
