# ריבוי דיירים

הפץ תתי-אתרים בין שרתים ומסדי נתונים לשיפור ביצועים ואבטחה

## נקודות עיקריות במהדורת גרסה 1.2.0

ריבוי דיירים 1.2.0 מוסיף את תשתית הדייר הריבוני עבור תהליכי עבודה של בידוד בסביבת ייצור:

- התחברות אוטומטית ללא מצב לדייר עם ביקורי SSO קצרי-חיים ומוגנים מפני שידור חוזר.
- בידוד מסד נתונים ומערכת קבצים לכל תת-אתר עבור דיירים ריבוניים.
- יכולת ריבוי דיירים של Hostinger להקצאת דיירים מודעת-מארח.
- שערי אימות הגירה ריבונית לבדיקות נתונים ישנים ולעיבוד דחיפה אסינכרוני.
- קישור מארח מסד נתונים הניתן להגדרה באותה מכונה עבור התקנות Bedrock, FrankenPHP ו-localhost.
- שיפורים באתחול דייר, כותב מסד נתונים, אתחול סכימה והקצאת משתמשים.
- פירוק ריבוני בטוח יותר שמנקה פרטי גישה של דיירים.

ראה את מדריכי המשתמש עבור [תהליך העבודה של הגירה ריבונית](/user-guide/getting-started/sovereign-migration-workflow), [בידוד ריבוי דיירים](/user-guide/configuration/multi-tenancy-isolation), ו[תפעול דיירים ריבוניים](/user-guide/administration/sovereign-tenant-operations).

## דרישות

- WordPress 5.3 ומעלה
- PHP 7.4 ומעלה
- תוסף Multisite Ultimate (פעיל)

## התקנה

1. העלה את קובצי תוסף ההרחבה לתיקיית `/wp-content/plugins/` שלך
2. הפעל את התוסף דרך תפריט 'תוספים' ב-WordPress
3. הגדר את הגדרות תוסף ההרחבה בממשק ניהול הרשת

## פיתוח

### הגדרה

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### בדיקות

תוסף ההרחבה כולל מסגרת בדיקות מקיפה:

- **בדיקות יחידה**: בדיקות מבוססות PHPUnit עם חבילת הבדיקות של WordPress
- **מחלקת בדיקה בסיסית**: מספקת מתודות עזר לתרחישי בדיקה נפוצים
- **כיסוי בדיקות**: מוגדר למעקב אחר כיסוי קוד
- **בדיקות Multisite**: הבדיקות רצות בסביבת multisite

### תקני קוד

- **תקני הקידוד של WordPress**: נאכפים באמצעות PHPCS
- **תאימות PHP 7.4+**: נתמכות יכולות PHP מודרניות
- **ניתוח סטטי**: שילוב PHPStan לבדיקת טיפוסים
- **מודרניזציית קוד**: כללי Rector לשדרוגי PHP

### מערכת בנייה

- **עיבוד נכסים**: הקטנה עבור קובצי CSS/JS
- **תרגום**: יצירת קובץ POT
- **אריזה**: יצירת ארכיון להפצה
- **מצב פיתוח**: נכסים לא מוקטנים לצורך ניפוי שגיאות

## תצורה

תוסף ההרחבה תומך במגוון אפשרויות תצורה הנגישות דרך ממשק ניהול הרשת.

### תצורת דייר ריבוני

עבור דיירים ריבוניים, הגדר את מסד הנתונים של הדייר, שורש מערכת הקבצים, קישור מארח מסד הנתונים ויכולת המארח לפני ניתוב תעבורה לדייר. הרץ אימות הגירה לאחר כל שינוי שמשפיע על פרטי הגישה למסד הנתונים, קישור המארח, הניתוב או הקצאת המשתמשים.

השתמש ב-`localhost` רק כאשר סביבת הריצה של הדייר והרשאות מסד הנתונים מצפות לגישה דרך socket מקומי או מאותה מכונה. השתמש ב-`127.0.0.1` או בשם מארח של שירות כאשר זהו מארח ההרשאה שהוגדר על ידי הספק שלך.

### יכולת Hostinger

יכולת ריבוי הדיירים של Hostinger עובדת עם אסימון השילוב המשותף של Hostinger. ודא שהדומיין מופיע ב-hPanel, ש-DNS מצביע אל חשבון Hostinger, ושהגדרות מסד הנתונים של הדייר תואמות ל-hPanel לפני האימות.

## Hooks ומסננים

### פעולות

- `ultimate-multisite-addon-template_init` - מופעל לאחר אתחול תוסף ההרחבה
- `ultimate-multisite-addon-template_loaded` - מופעל לאחר שכל קובצי תוסף ההרחבה נטענו

### מסננים

- `ultimate-multisite-addon-template_settings` - שנה את הגדרות תוסף ההרחבה
- `ultimate-multisite-addon-template_enabled` - עקוף את מצב ההפעלה/השבתה של תוסף ההרחבה
