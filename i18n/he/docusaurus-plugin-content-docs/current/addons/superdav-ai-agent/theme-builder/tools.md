# כלים ותצורה

Theme Builder משתמש בערכת כלים כדי לנתח, לעצב ולבנות את אתר ה-WordPress שלך. בגרסה 1.16.0, **sd-ai-agent/site-scrape הוא כעת כלי Tier 1**, מה שהופך אותו לזמין כברירת מחדל ב-Theme Builder. Superdav AI Agent v1.18.0 מציג גם חבילת ליווי Advanced המופצת בנפרד עבור תהליכי עבודה של מפתחים מהימנים; כלים אלה אינם חלק מחבילת הליבה של WordPress.org ויש להתקין ולאשר אותם בנפרד.

## כלים זמינים {#available-tools}

### כלי Tier 1 (זמינים תמיד) {#tier-1-tools-always-available}

כלי Tier 1 זמינים כברירת מחדל ב-Theme Builder ללא כל תצורה נוספת.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**מטרה:** לנתח אתרים קיימים כדי לחלץ השראה עיצובית, מבנה תוכן ופונקציונליות.

**יכולות:**

- **ניתוח אתרים** — סריקה וניתוח של אתרי מתחרים או אתרי השראה
- **חילוץ עיצוב** — זיהוי צבעים, גופנים ותבניות פריסה
- **מבנה תוכן** — הבנת ארגון הדפים וההיררכיה
- **זיהוי תכונות** — זיהוי plugins ופונקציונליות
- **ניתוח ביצועים** — בדיקת מהירות דפים ואופטימיזציה
- **ניתוח SEO** — סקירת תגיות מטא ונתונים מובנים

**שימוש:**

```
נתח את העיצוב של example.com והשתמש בו כהשראה לאתר שלי.
```

**מה הוא מחלץ:**

- פלטת צבעים ושימוש בצבעים
- טיפוגרפיה (גופנים וגדלים)
- מבנה פריסה ומרווחים
- תבניות ניווט
- ארגון תוכן
- שימוש בתמונות ובמדיה
- רכיבים אינטראקטיביים
- התאמה למובייל

**מגבלות:**

- לא יכול לסרוק אתרים המוגנים בסיסמה
- מכבד את robots.txt ואת מדיניות האתר
- ייתכן שלא ילכוד תוכן דינמי
- אתרים עתירי JavaScript עשויים לקבל ניתוח מוגבל
- ניתוח של אתרים גדולים עשוי להימשך זמן רב יותר

### כלי Tier 2 (אופציונליים) {#tier-2-tools-optional}

כלי Tier 2 זמינים כאשר הם מופעלים במפורש בהגדרות Theme Builder.

#### ניתוח מתקדם {#advanced-analytics}

מספק מדדי ביצועים מפורטים:

- זמני טעינת דפים
- Core Web Vitals
- ציון SEO
- ציון נגישות
- ציון שיטות מיטביות

#### ממטב תוכן {#content-optimizer}

מנתח ומציע שיפורים עבור:

- קריאות
- אופטימיזציית SEO
- שימוש במילות מפתח
- מבנה תוכן
- מיקום קריאות לפעולה

### כלי Tier 3 (Premium) {#tier-3-tools-premium}

כלי Tier 3 דורשים תצורה נוספת או תכונות Premium.

#### מחולל תוכן AI {#ai-content-generator}

מייצר תוכן עבור:

- תיאורי מוצרים
- דפי שירותים
- פוסטים בבלוג
- תיאורי מטא
- נוסח קריאות לפעולה

#### כלי עיצוב מתקדמים {#advanced-design-tools}

מספק יכולות עיצוב מתקדמות:

- יצירת CSS מותאם אישית
- יצירת אנימציות
- עיצוב רכיבים אינטראקטיביים
- תורת צבעים מתקדמת
- אופטימיזציית טיפוגרפיה

### כלי ליווי Advanced (חבילה נפרדת) {#advanced-companion-tools-separate-package}

חבילת הליווי Advanced מופצת בנפרד ממהדורת הליבה של Superdav AI Agent. היא מיועדת לסביבות של מפתחים ובעלי אתרים מהימנים שבהן מנהלים מאפשרים במפורש לסוכן להשתמש בכלים בעלי השפעה גבוהה.

יכולות הליווי Advanced יכולות לכלול:

- **כלי מערכת קבצים** — בדיקה ושינוי של קבצים מאושרים בסביבות מהימנות
- **כלי מסד נתונים** — שאילתות או עדכון נתוני אתר כאשר הדבר מאושר במפורש
- **כלי WP-CLI** — הרצת פקודות תחזוקה ובדיקה של WordPress
- **כלי משגר REST** — קריאה ל-REST endpoints רשומים דרך תהליכי שילוח מבוקרים
- **כלי בונה plugin** — יצירת שלד ופיתוח איטרטיבי של קוד plugin
- **כלי תמונות מצב של Git** — יצירת תמונות מצב לפני פעולות מפתחים מסוכנות
- **כלי ניהול משתמשים** — סיוע בסקירה או התאמה של רשומות משתמשים היכן שמותר
- **כלי Benchmark** — הרצת מדדי ביצועים או יכולות לצורך סקירת פיתוח

אין לתעד כלים אלה ככלי Theme Builder הזמינים באופן כללי לכל התקנה. הם זמינים רק כאשר חבילת הליווי Advanced מותקנת, מופעלת ומוגבלת למנהלים מהימנים.

## תצורת כלים {#tool-configuration}

### הפעלת כלים {#enabling-tools}

כדי להפעיל כלים נוספים ב-Theme Builder:

1. נווט אל **Gratis AI Agent → Settings**
2. עבור אל **Theme Builder → Tools**
3. הפעל או כבה כלים לפי הצורך
4. שמור הגדרות

### הרשאות כלים {#tool-permissions}

חלק מהכלים דורשים הרשאות:

- **סריקת אתר** — דורשת גישה לאינטרנט
- **Analytics** — דורש אינטגרציה עם Google Analytics
- **יצירת תוכן** — דורשת API keys
- **תכונות מתקדמות** — עשויות לדרוש subscription Premium
- **כלי ליווי Advanced** — דורשים את חבילת Advanced המופצת בנפרד ואמון מנהל מפורש

### API Keys ופרטי גישה {#api-keys-and-credentials}

הגדר API keys עבור כלים שדורשים אותם:

1. עבור אל **Gratis AI Agent → Settings → API Keys**
2. הזן פרטי גישה עבור כל כלי
3. בדוק חיבור
4. שמור בצורה מאובטחת

### התקנת חבילת הליווי Advanced {#installing-the-advanced-companion-package}

חבילת הליווי Advanced מופצת בנפרד מקובץ ה-ZIP של ליבת WordPress.org. התקן אותה רק מערוץ ההפצה הרשמי של הפרויקט עבור המהדורה, ולאחר מכן הגבל גישה למנהלים מהימנים. סקור שערי אישור אנושיים והתנהגות תמונות מצב של git לפני הפעלת כלי מערכת קבצים, מסד נתונים, WP-CLI, משגר REST, בונה plugin, ניהול משתמשים או benchmark באתר ייצור.

## שימוש ב-sd-ai-agent/site-scrape {#using-sd-ai-agentsite-scrape}

### שימוש בסיסי {#basic-usage}

בקש מ-Theme Builder לנתח אתר:

```
נתח את עיצוב האתר של המתחרה שלי ב-competitor.com
והצע שיפורי עיצוב לאתר שלי.
```

### ניתוח ספציפי {#specific-analysis}

בקש סוגי ניתוח ספציפיים:

```
חלץ את פלטת הצבעים מ-example.com והשתמש בה כהשראה.
```

```
נתח את מבנה הניווט של example.com והחל ארגון
דומה על האתר שלי.
```

```
בדוק את ההתאמה למובייל של example.com וודא שהאתר שלי
רספונסיבי באותה מידה.
```

### ניתוח השוואתי {#comparison-analysis}

השווה כמה אתרים:

```
השווה את העיצובים של site1.com ושל site2.com וצור עיצוב
שמשלב את הרכיבים הטובים ביותר של שניהם.
```

## מגבלות ושיקולים של כלים {#tool-limitations-and-considerations}

### הגבלת קצב {#rate-limiting}

- Scraping מוגבל בקצב כדי למנוע עומס יתר על השרת
- בקשות מרובות לאותו אתר עשויות להיות מואטות
- אתרים גדולים עשויים להימשך זמן רב יותר לניתוח

### דיוק התוכן {#content-accuracy}

- ייתכן שתוכן דינמי לא ייקלט במלואו
- תוכן שמרונדר באמצעות JavaScript עשוי להיות חלקי
- ייתכן שנתונים בזמן אמת לא ישתקפו
- חלק מהתוכן עשוי להיות מאחורי חומות תשלום

### שיקולים משפטיים ואתיים {#legal-and-ethical-considerations}

- כבדו את robots.txt ואת מדיניות האתר
- אל תבצעו Scraping לתוכן המוגן בזכויות יוצרים לצורך שכפול
- השתמשו בניתוח להשראה, לא להעתקה
- ודאו שיש לכם זכויות להשתמש בתוכן שחולץ
- פעלו לפי תנאי השירות של האתרים המנותחים

### השפעה על ביצועים {#performance-impact}

- Scraping של אתרים גדולים עשוי לקחת זמן
- כמה פעולות Scraping בו־זמנית עשויות להיות איטיות יותר
- קישוריות רשת משפיעה על המהירות
- מערכי נתונים גדולים עשויים לדרוש עיבוד נוסף

## שיטות עבודה מומלצות {#best-practices}

### שימוש בניתוח אתר {#using-site-analysis}

1. **נתחו כמה אתרים** — אספו השראה מכמה מקורות
2. **התמקדו במבנה** — למדו דפוסי פריסה וארגון
3. **חלצו צבעים** — השתמשו בפלטות צבעים כנקודות התחלה
4. **למדו טיפוגרפיה** — זהו שילובי גופנים שאתם אוהבים
5. **סקירת ניווט** — הבינו את ארגון התפריט

### Scraping אתי {#ethical-scraping}

1. **כבדו את robots.txt** — פעלו לפי הנחיות האתר
2. **אל תעתיקו תוכן** — השתמשו בניתוח להשראה בלבד
3. **אמתו זכויות** — ודאו שאתם יכולים להשתמש בתוכן שחולץ
4. **ייחסו מקורות** — תנו קרדיט למקורות ההשראה
5. **פעלו לפי התנאים** — עמדו בתנאי השירות של האתר

### מיקסום יעילות הכלי {#maximizing-tool-effectiveness}

1. **היו ספציפיים** — בקשו סוגי ניתוח ספציפיים
2. **ספקו הקשר** — הסבירו את מטרת האתר שלכם
3. **הגדירו ציפיות** — תארו את יעדי העיצוב שלכם
4. **חזרו ושפרו** — לטשו על בסיס התוצאות
5. **שלבו כלים** — השתמשו בכמה כלים לניתוח מקיף

## פתרון בעיות {#troubleshooting}

### האתר לא ניתן ל-Scraping {#site-wont-scrape}

- בדקו אם האתר נגיש לציבור
- ודאו ש-robots.txt מאפשר Scraping
- נסו אתר אחר
- בדקו את חיבור האינטרנט
- פנו לתמיכה אם הבעיה נמשכת

### ניתוח חלקי {#incomplete-analysis}

- ייתכן שיש באתר תוכן דינמי
- ייתכן ש-JavaScript לא מרונדר במלואו
- אתרים גדולים עשויים להגיע לפסק זמן
- נסו לנתח עמודים ספציפיים במקום זאת
- בקשו סוגי ניתוח ספציפיים

### ביצועים איטיים {#slow-performance}

- ניתוח אתרים גדולים נמשך זמן רב יותר
- כמה פעולות Scraping בו־זמנית איטיות יותר
- קישוריות רשת משפיעה על המהירות
- נסו בשעות שאינן שעות עומס
- נתחו תחילה מקטעים קטנים יותר
