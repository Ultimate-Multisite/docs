# כלים והגדרות

Theme Builder משתמש במערכת כלים כדי לנתח, לעצב ולבנות את אתר ה-WordPress שלכם. בגרסה 1.16.0, **sd-ai-agent/site-scrape הוא כעת כלי Tier 1**, מה שהופך אותו לזמין כברירת מחדל ב-Theme Builder.

## הכלים הזמינים

### כלים מסוג Tier 1 (זמינים תמיד)

כלים מסוג Tier 1 זמינים כברירת מחדל ב-Theme Builder ללא צורך בהגדרות נוספות.

#### sd-ai-agent/site-scrape

**מטרה:** ניתוח אתרים קיימים כדי לחלץ השראה עיצובית, מבנה תוכן ותכונות פונקציונליות.

**יכולות:**

- **ניתוח אתר** — סריקה וניתוח של אתרי מתחרים או אתרי השראה
- **חילוץ עיצוב** — זיהוי צבעים, גופנים ודפוסי פריסה
- **מבנה תוכן** — הבנת ארגון והיררכיה של דפים
- **זיהוי תכונות** — זיהוי פלאגינים ותכונות פונקציונליות
- **ניתוח ביצועים** — בדיקת מהירות דף ואופטימיזציה
- **ניתוח SEO** — סקירה של מטא תגיות ונתונים מובנים

**שימוש:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**מה הוא מחלץ:**

- פלטת צבעים ושימוש בצבעים
- טיפוגרפיה (גופנים וגדלים)
- מבנה פריסה ורווחים
- דפוסי ניווט
- ארגון תוכן
- שימוש בתמונות ומדיה
- אלמנטים אינטראקטיביים
- תגובתיות למובייל

**מגבלות:**

- אינו יכול לסרוק אתרים מוגנים בסיסמה
- מכבד את קובץ robots.txt ומדיניות האתר
- ייתכן שלא יצליח ללכוד תוכן דינמי
- לאתרים כבדי JavaScript עשוי להיות ניתוח מוגבל
- לאתרים גדולים יותר עשוי לקחת יותר זמן לניתוח

### כלים מסוג Tier 2 (אופציונלי)

כלים מסוג Tier 2 זמינים כאשר מופעלים במפורש בהגדרות Theme Builder.

#### Advanced Analytics

מספק מדדי ביצועים מפורטים:

- זמני טעינת דף
- Core Web Vitals
- ציון SEO
- ציון נגישות
- ציון שיטות עבודה מומלצות

#### Content Optimizer

מנתח ומציע שיפורים עבור:

- קריאות
- אופטימיזציה ל-SEO
- שימוש במילות מפתח
- מבנה תוכן
- מיקום קריאה לפעולה (Call-to-action)

### כלים מסוג Tier 3 (פרימיום)

כלים מסוג Tier 3 דורשים הגדרות נוספות או תכונות פרימיום.

#### AI Content Generator

מייצר תוכן עבור:

- תיאורי מוצרים
- דפי שירות
- פוסטים לבלוג
- מטא תיאורים
- טקסט קריאה לפעולה

#### Advanced Design Tools

מספק יכולות עיצוב מתקדמות:

- יצירת CSS מותאם אישית
- יצירת אנימציות
- עיצוב אלמנטים אינטראקטיביים
- תורת צבעים מתקדמת
- אופטימיזציה טיפוגרפית

## הגדרת כלים

### הפעלת כלים

כדי להפעיל כלים נוספים ב-Theme Builder:

1. נווט אל **Gratis AI Agent → Settings**
2. עבור אל **Theme Builder → Tools**
3. הפעל/כבה כלים לפי הצורך
4. שמור את ההגדרות

### הרשאות כלים

כלים מסוימים דורשים הרשאות:

- **Site scraping** — דורש גישה לאינטרנט
- **Analytics** — דורש אינטגרציה עם Google Analytics
- **Content generation** — דורש מפתחות API
- **Advanced features** — עשוי לדרוש מנוי פרימיום

### מפתחות API ופרטי התחברות

הגדר מפתחות API עבור כלים הדורשים זאת:

1. עבור אל **Gratis AI Agent → Settings → API Keys**
2. הזן את פרטי ההתחברות עבור כל כלי
3. בדוק את החיבור
4. שמור בצורה מאובטחת

## שימוש ב-sd-ai-agent/site-scrape

### שימוש בסיסי

בקש מ-Theme Builder לנתח אתר:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### ניתוח ספציפי

בקש סוגי ניתוח ספציפיים:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### ניתוח השוואתי

השווה מספר אתרים:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## מגבלות ושיקולים של הכלים

### הגבלת קצב (Rate Limiting)

- הסריקה מוגבלת בקצב כדי למנוע עומס על השרת
- בקשות מרובות לאתר אחד עלולות להיות מוגבלות (throttled)
- לאתרים גדולים יותר עשוי לקחת יותר זמן לניתוח

### דיוק התוכן

- תוכן דינמי עלול שלא להילקח במלואו
- תוכן שנוצר באמצעות JavaScript עלול להיות לא שלם
- נתונים בזמן אמת עלולים שלא להיות משתקפים
- חלק מהתוכן עשוי להיות מאחורי חומות תשלום

### שיקולים משפטיים ואתיים

- לכבד את robots.txt ומדיניות האתר
- לא לסרוק תוכן המוגן בזכויות יוצרים לצורך שכפול
- להשתמש בניתוח להשראה, ולא להעתקה
- לוודא שיש לכם זכויות להשתמש בתוכן שחולץ
- לציית לתנאי השירות של האתרים שנותחו

### השפעת ביצועים

- סריקת אתרים גדולים עלולה לקחת זמן
- מספר סריקות בו-זמניות עלול להיות איטי יותר
- חיבור רשת משפיע על המהירות
- מערכי נתונים גדולים עשויים לדרוש יותר עיבוד

## שיטות עבודה מומלצות

### שימוש בניתוח אתרים

1. **נתח מספר אתרים** — אסוף השראה ממקורות שונים
2. **התמקד במבנה** — למד דפוסי פריסה וארגון
3. **חלץ צבעים** — השתמש בפלטות צבעים כנקודת התחלה
4. **למד על טיפוגרפיה** — זהה שילוב גופנים שאתה אוהב
5. **סקור ניווט** — הבן את ארגון התפריט

### סריקה אתית

1. **כבד את robots.txt** — ציית להנחיות האתר
2. **אל תעתיק תוכן** — השתמש בניתוח להשראה בלבד
3. **אמת זכויות** — ודא שאתה יכול להשתמש בתוכן שחולץ
4. **ציין מקורות** — תן קרדיט למקורות ההשראה
5. **ציית לתנאים** — הקפד על תנאי השירות של האתר

### מקסום יעילות הכלים

1. **היה ספציפי** — בקש סוגי ניתוח ספציפיים
2. **ספק הקשר** — הסבר מהי מטרת האתר שלך
3. **קבע ציפיות** — תאר את מטרות העיצוב שלך
4. **בצע איטרציות** — שפר בהתבסס על התוצאות
5. **שלב כלים** — השתמש במספר כלים לניתוח מקיף

## פתרון תקלות

### האתר לא נסרק

- בדוק אם האתר נגיש לציבור
- ודא ש-robots.txt מאפשר סריקה
- נסה אתר אחר
- בדוק את חיבור האינטרנט
- צור קשר עם התמיכה אם הבעיה נמשכת

### ניתוח לא שלם

- ייתכן שהאתר מכיל תוכן דינמי
- ייתכן ש-JavaScript לא נפרש במלואו
- לאתרים גדולים עשוי להיות Timeout
- נסה לנתח דפים ספציפיים במקום זאת
- בקש סוגי ניתוח ספציפיים

### ביצועים איטיים

- אתרים גדולים לוקחים יותר זמן לניתוח
- מספר סריקות בו-זמניות איטי יותר
- חיבור רשת משפיע על המהירות
- נסה בזמנים שאינם שעות שיא
- נתח קטעים קטנים יותר תחילה
