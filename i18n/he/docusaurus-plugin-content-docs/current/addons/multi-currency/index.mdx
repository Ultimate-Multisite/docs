# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency מאפשר לרשת שלך לקבל תשלומים במספר מטבעות. הוא מבצע המרה אוטומטית של מחירים באמצעות ש汇ת מטבע חיה או ידנית, מאפשר לך להגדיר מחירים מדויקים למוצר ספציפי בכל מטבע, וזהה את המטבע המועדף של כל מבקר באמצעות מיקום גיאוגרפי — כל זה מנוהל מתוך לוח מחוונים מרכזי אחד.

## תכונות עיקריות {#key-features}

- **המרה אוטומטית של ש汇ת מטבע** — המחירים מומרים מהמטבע הבסיסי שלך באמצעות ש汇ת מטבע שנמשכת מהספק שבחרת
- **דריסות תמחור למוצר** — הגדר מחירים מדויקים המותאמים למטבע ספציפי על כל מוצר כדי לעקוף המרה אוטומטית
- **שדה בחירת מטבע בהרשמה** — מאפשר ללקוחות לבחור את המטבע המועדף עליהם במהלך תהליך התשלום/הרשמה
- **ספקי ש汇ת מטבע מרובים** — ECB, Frankfurter, Stripe, או ש汇ת מטבע ידנית מלאה
- **זיהוי מטבע מבוסס מיקום גיאוגרפי** — בוחר מראש אוטומטית את המטבע התואם למדינה של המבקר
- **לוח מחוונים למנהל** — נהל מטבעות פעילים, ש汇ת מטבע וגדרות ספקים ממקום אחד

## דרישות קדם {#requirements}

- Ultimate Multisite 2.0.0 ומעלה
- התקנת WordPress Multisite
- PHP 7.4 ומעלה

## התקנה {#installation}

1. העלה את קבצי התוסף למיקום `/wp-content/plugins/ultimate-multisite-multi-currency`
2. הפעל את התוסף דרך מסך **Plugins** ב-WordPress (הפעלת רשת)
3. נווט אל **Ultimate Multisite → Multi-Currency** כדי לפתוח את לוח המחוונים
4. הוסף את המטבעות שברצונך לקבל ולהגדר את ספק ש汇ת המטבע שלך

## הגדרות {#configuration}

### לוח המחוונים למנהל {#admin-dashboard}

לוח המחוונים של Multi-Currency (**Ultimate Multisite → Multi-Currency**) הוא המקום המרכזי עבור:

- הוספה, הפעלה או השבתה של מטבעות
- בחירת ספק ש汇ת מטבע ורענון ש汇ת המטבע לפי דרישה
- סקירה של ש汇ת המטבע הנוכחית עבור כל המטבעות הפעילים
- הגדרת ש汇ת מטבע ידנית כאשר ספקים אוטומטיים אינם מתאימים

### ספקי ש汇ת מטבע {#exchange-rate-providers}

| Provider | תיאור |
|---|---|
| **ECB** | הבנק המרכזי האירופי — ש汇ת מטבע יומית חינם עבור רשתות מבוססות EUR |
| **Frankfurter** | ממשק API מראה (Mirror) קוד פתוח של ECB — חינם, לא נדרש מפתח API |
| **Stripe** | משתמש בש汇ת המטבע החיה של Stripe — דורש אינטגרציה פעילה עם Stripe |
| **Manual** | אתה מזין את הש汇ת המטבע ישירות; ללא קריאות API חיצוניות |

בחר את הספק שלך תחת **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider**. ניתן לרענן את הש汇ת המטבע ידנית בכל עת או לתזמן עדכון אוטומטי.

### דריסות תמחור למוצר {#per-product-pricing-overrides}

כאשר ההמרה האוטומטית אינה מדויקת מספיק, תוכל להגדיר מחיר מדויק למטבע ספציפי על כל מוצר:

1. פתח את עורך המוצר (**Ultimate Multisite → Products → Edit**)
2. גלול אל הקטע **Currency Pricing**
3. הזן את המחיר המדויק עבור כל מטבע שברצונך לדרוס
4. שמור את המוצר

מוצרים ללא דריסה משתמשים במחיר המומר מהמטבע הבסיסי שלך.

### שדה בחירת מטבע בהרשמה {#currency-selector-signup-field}

ניתן להוסיף את שדה בחירת המטבע לתהליך ההרשמה/תשלום שלך, כך שהלקוחות יבחרו את המטבע המועדף עליהם כבר בהרשמה:

1. עבור אל **Ultimate Multisite → Multi-Currency → Settings**
2. הפעל את **Show currency selector on signup**
3. ניתן להגדיר אופציונלית מטבע ברירת מחדל או לתת למיקום גיאוגרפי לקבוע את הערך שנבחר מראש

המטבע שנבחר נשמר עם המנוי ומשמש לכל חיוב עתידי.

### זיהוי מטבע מבוסס מיקום גיאוגרפי {#geolocation-based-currency-detection}

כאשר זיהוי מיקום גיאוגרפי מופעל, התוסף משייך את המדינה של המבקר למטבע ובוחר אותו מראש בשדה בחירת המטבע. כדי להפעיל:

1. עבור אל **Ultimate Multisite → Multi-Currency → Settings**
2. הפעל את **Detect currency from visitor location**
3. הירש את המדינות למטבעות בטבלת **Geolocation Mapping**

אם אין מיפוי למדינת המבקר, משתמשים במטבע הבסיסי של הרשת כברירת מחדל.

## שאלות נפוצות {#frequently-asked-questions}

### מה קורה אם ספק ש汇ת המטבע אינו זמין? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

התוסף חוזר לש汇ת המטבע האחרונה שנמשכה בהצלחה. אם מעולם לא נמשכו ש汇ת מטבע, הוא חוזר לש汇ת המטבע הידנית שהגדרת. מומלץ להגדיר ש汇ת מטבע ידנית כרשת ביטחון.

### האם אוכל להשתמש בספקים שונים עבור מטבעות שונים? {#can-i-use-different-providers-for-different-currencies}

לא — ספק אחד פעיל בכל רגע נתון עבור כל המטבעות. השתמש ב-**Manual** אם אתה זקוק לשליטה מלאה בש汇ת המטבע של כל מטבע בנפרד.

### האם זיהוי מיקום גיאוגרפי דורש תוסף נפרד? {#does-geolocation-require-a-separate-plugin}

אין צורך בתוסף מיקום גיאוגרפי חיצוני. התוסף משתמש בזיהוי IP-למדינה המובנה של WordPress. לדיוק גבוה יותר, ניתן לשלב שירות מיקום גיאוגרפי ייעודי דרך ה-hooks שסופקו.

### האם דריסות למוצר מיושמות לפני או אחרי המסה? {#are-per-product-overrides-applied-before-or-after-tax}

דריסות למוצר קובעות את המחיר לפני המסה במטבע היעד. המסה מחושבת מעל מחיר הדריסה באמצעות הגדרת המסה הקיימת שלך.
