# מחבר ה-AI האולטימטיבי עבור נקודות קצה תואמות

מחבר ה-AI האולטימטיבי עבור נקודות קצה תואמות משלב ממשקי API של ספקי AI התואמים ל-OpenAI עם רשת ה-WordPress Multisite שלך. הגדר מספר ספקי AI — כל אחד עם כתובת URL של נקודת קצה, מפתח API ומודל ברירת מחדל משלו — ונתב בקשות עם מנגנון נסיגה אוטומטי (fallback) כאשר ספק אינו זמין.

## תכונות עיקריות

- **תמיכה במספר ספקים**: הוסף כרטיסיות של ספקי AI מרובים, שלכל אחת מהן כתובת URL של נקודת קצה, מפתח API ומודל ברירת מחדל משלו.
- **סידור מחדש באמצעות גרירה**: סדר את הספקים לפי סדר עדיפות באמצעות ידית האחיזה בכל כרטיס; הבקשות משתמשות בספקים לפי הסדר.
- **ניתוב נסיגה אוטומטי**: אם הספק הראשי אינו זמין, הבקשות עוברות אוטומטית לספק המוגדר הבא.
- **הגדרות לכל ספק**: כל ספק מוגדר באופן עצמאי עם פרטי ההתחברות והבחירה במודל שלו.
- **ממשק משתמש מבוסס כרטיסיות**: בנוי על רכיבי ה-Card היציבים של WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **הרחבה אוטומטית בעת הוספה**: כרטיסיות ספקים חדשים מתרחבות אוטומטית כדי שתוכל למלא את הפרטים מיד.
- **מניעת מטמון סקריפטים (Cache busting)**: סקריפטי הספקים מנקים מטמון בעדכון הפלאגין כדי להבטיח שהגרסה האחרונה נטענת.

## דרישות קדם

- WordPress 5.3 ומעלה
- PHP 7.4 ומעלה
- פלאגין Ultimate Multisite (פעיל)
- נקודת קצה API תואמת ל-OpenAI (מאוחסנת עצמית או צד שלישי)

## התקנה

1. העלה את קבצי ה-addon לתיקייה `/wp-content/plugins/` שלך.
2. הפעל את הפלאגין דרך תפריט 'Plugins' ב-WordPress.
3. נווט אל **Ultimate Multisite → AI Connector** כדי להגדיר את ה-addon.

## הגדרות

### הגדרת מספר ספקים

מסך ההגדרות משתמש במערך מבוסס כרטיסיות, כאשר כל כרטיס מייצג ספק AI אחד. תוכל להוסיף כמה ספקים שתצטרך.

#### הוספת ספק

1. עבור אל **Ultimate Multisite → AI Connector** במנהל הרשת.
2. לחץ על **Add Provider**.
3. כרטיס הספק החדש מופיע ומתרחב אוטומטית.
4. מלא את פרטי הספק:
   - **Provider Name** — תווית לזיהוי הספק הזה (לדוגמה: "OpenAI", "Mistral", "Local Ollama").
   - **Endpoint URL** — כתובת ה-URL הבסיסית של ה-API התואם ל-OpenAI (לדוגמה: `https://api.openai.com/v1`).
   - **API Key** — מפתח ה-API שלך עבור ספק זה.
   - **Default Model** — מזהה המודל שמשמש כברירת מחדל עבור ספק זה (לדוגמה: `gpt-4o`, `mistral-small`).
5. לחץ על **Save Settings**.

#### הסרת ספק

לחץ על כפתור **Remove** בכל כרטיס ספק כדי למחוק אותו. שמור את ההגדרות כדי להחיל את השינוי.

#### סידור מחדש של ספקים

גרור את אייקון האחיזה (⣿) בצד שמאל של כל כרטיס ספק כדי לסדר אותו מחדש. הסדר קובע את סדר הנסיגה: הבקשות נשלחות לספק הראשון ונסוגות לספקים הבאים אם בקשה נכשלת.

### ניתוב נסיגה (Fallback Routing)

כאשר בקשה לספק הראשי נכשלת — עקב תפוגת זמן (timeout), מגבלת קצב (rate limit) או הפסקת שירות — המחבר מנסה אוטומטית עם הספק הבא ברשימה. תהליך זה ממשיך למטה ברשימה עד שספק מצליח או שכל הספקים נבדקו.

| תרחיש | התנהגות |
|----------|-----------|
| הספק הראשי מגיב | הבקשה מוגשת כרגיל |
| הספק הראשי נכשל | הבקשה מנסה מחדש עם הספק הבא |
| כל הספקים נכשלים | שגיאה מוחזרת לקורא |

הנסיגה שקופה עבור הפלאגין או התבנית הקוראים — הם מקבלים תגובה מוצלחת או שגיאה סופית, ללא קשר למספר הספקים שנבדקו.

### מילון הגדרות ספק

| שדה | נדרש | תיאור |
|-------|----------|-------------|
| Provider Name | כן | תווית תצוגה עבור ספק זה |
| Endpoint URL | כן | כתובת ה-URL הבסיסית של ה-API התואם ל-OpenAI |
| API Key | כן | מפתח האימות עבור ספק זה |
| Default Model | כן | מזהה המודל המשמש כאשר לא צוין מודל בבקשה |

## פתרון תקלות

### לא מופיעים ספקים

אם רשימת הספקים ריקה לאחר השמירה, ודא כי:

- לחצת על **Save Settings** לאחר הוספת הספקים.
- הדפדפן שלך לא חסם את שמירת דף ההגדרות (בדוק את קונסולת המפתחים לאיתור שגיאות JavaScript).
- הפלאגין פעיל ואתה נמצא במסך מנהל הרשת.

### הבקשות נכשלות עבור כל הספקים

- ודא שכתובת ה-URL של נקודת הקצה של כל ספק נגישה מהשרת שלך (ולא רק מהדפדפן שלך).
- בדוק שכל מפתח ה-API תקין ולא פג תוקפו.
- עיין במצב מגבלת הקצב והקווטה של הספק בלוח המחוונים שלו.
- הפעל את רישום ניפוי השגוי של WordPress (`WP_DEBUG_LOG`) כדי ללכוד הודעות שגיאה מפורטות מהמחבר.

### ספק אינו משמש לנסיגה

סידור מחדש באמצעות גרירה משנה את סדר הנסיגה. ודא שהספק שאתה מצפה שיקבל בקשות נסיגה ממוקם נכון ברשימה וכי כתובת ה-URL של נקודת הקצה ומפתח ה-API שלו נכונים.

### כרטיס ספק חדש אינו מתרחב אוטומטית

הרחבה אוטומטית דורשת ש-JavaScript יהיה מופעל בדפדפן שלך. אם הכרטיס אינו מתרחב בלחיצה, בדוק את קונסולת המפתחים לאיתור שגיאות. תוכן הכרטיס תמיד נגיש בלחיצה על כותרת הכרטיס גם אם ההרחבה האוטומטית לא מופעלת.
