# תהליך מיגרציה ריבוני

Ultimate Multisite: Multi-Tenancy 1.2.0 מוסיפה שערי אימות מיגרציה להעברת סובסייט סטנדרטי לתשתית של שוכר ריבוני (sovereign tenant).

## לפני שאתה מתחיל

ודא שיש לך:

- גיבוי עדכני של אתר המקור.
- תוסף Multi-Tenancy פעיל.
- מסד הנתונים, מערכת הקבצים הראשית והדומיין של השוכר היעד מוכנים.
- קישור מארח מסד נתונים התואם לסביבת היעד.
- גישה להרצת פקודות WP-CLI לרשת.

## תהליך מומלץ

1. הכן את מסד הנתונים ומערכת הקבצים של השוכר היעד.
2. רשום או עדכן את הגדרות בידוד השוכרים (tenant isolation settings).
3. הרץ את מיגרציית השוכר.
4. ספק או אשר משתמשי שוכר.
5. ריק (Drain) משימות מיגרציה א-סינכרוניות.
6. הרץ אימות מיגרציה ריבוני.
7. גש לשוכר באמצעות SSO.
8. שנה DNS או ניתוב רק לאחר שהאימות עבר בהצלחה.

## שערי אימות (Verification gates)

תהליך האימות בודק את המיגרציה מכמה זוויות:

- סכמת השוכר קיימת וניתן לכתוב אותה על ידי כותב מסד הנתונים של השוכר.
- מארח מסד הנתונים שהוגדר כולל הרשאות תקפות.
- משתמשי השוכר קיימים ותואמים את מספר המשתמשים הצפוי של ההתקנה הריבונית.
- ניתן לרוק (drain) בהצלחה את תור הדחיפה הא-סינכרוני.
- נתיבי נתונים ישנים בצד הרשת אינם נדרשים יותר לשוכר הריבוני.

התייחס לאיזורים של אימות כחסימה לפני ההשקה. תקן את בעיית מסד הנתונים, המשתמשים, התור או הניתוח שדווחו, ואז הרץ את האימות שוב לפני חשיפת השוכר ללקוחות.

## ביקור ראשון בייצור (First production visit)

לאחר שהאימות עבר בהצלחה, השתמש ב**Visit (SSO)** מ מסך ניהול האתר עבור הביקור הניהולי הראשון. זה מאשר את ניתוב השוכר, טיפול בטוקן SSO, קיבוע המקור (origin pinning) והפרוויזיית משתמשי צד השוכר בשלב אחד מבוקר.
