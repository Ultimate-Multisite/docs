# בונה תוספים ו-Sandbox {#plugin-builder--sandbox}

Gratis AI Agent v1.5.0 מציג את **מערכת בונה התוספים ו-Sandbox**, המאפשרת למساعد ה-AI ליצור, להפעיל ולנהל תוספי WordPress ברשת שלך — הכל דרך סביבת Sandbox מבודדת ובטוחה.

## סקירה כללית {#overview}

ה-Plugin Builder מאפשר למساعد ה-AI לכתוב תוספי WordPress מותאמים אישית בתגובה לבקשות בשפה טבעית. התוספים שנוצרים מאומתים, מאוחסנים ומופעלים בתוך שכבת Sandbox לפני שהם משפיעים על תפקוד האתר החי.

מקרי שימוש כוללים:

- יצירת תוספי שירות קלים ללא מעורבות מפתח.
- יצירת אב-טיפוס לתכונות הדורשות hooks של WordPress או סוגי פוסט מותאמים אישית.
- יצירת סקריפטים אוטומציה קצרים למבצעים באצווה.

## יצירת תוסף באמצעות AI {#generating-a-plugin-via-ai}

כדי ליצור תוסף, פתח את ממשק הצ'אט של Gratis AI Agent ותאר מה אתה צריך. לדוגמה:

> "צור תוסף שמוסיף הודעת מנהל מותאמת אישית על ה-dashboard."

ה-AI י:

1. ייצור את קוד ה-PHP של התוסף באמצעות יצירת קוד מובנה.
2. יאמת את הפלט לגבי שגיאות תחביר ודפוסים לא בטוחים.
3. ישמור את התוסף שנוצר במחסן ה-Sandbox.
4. יחזיר אישור עם ה-slug של התוסף וכפתור **Activate in Sandbox**.

תוכל לשפר את התוצאה על ידי המשך שיחה באותו שרשור לפני ההפעלה.

## הפעלת Sandbox {#sandbox-activation}

הפעלת תוסף שנוצר ב-Sandbox שונה מהפעלתו ברשת החיה. ה-Sandbox:

- מריץ את התוסף בסביבת WordPress מבודדת (wp-env).
- לוכד כל שגיאת PHP, אזהרה או קונפליקט hook.
- מדווח על תוצאת ההפעלה בחזרה בממשק הצ'אט.

כדי להפעיל תוסף ב-Sandbox, לחץ על הכפתור **Activate in Sandbox** בתגובת ה-AI בצ'אט, או השתמש בפקודת ה-slash:

```
/activate-plugin <plugin-slug>
```

הודעת סטטוס מאשרת אם ההפעלה הצליחה או נכשלה. במקרה של כישלון, יומן השגיאות מוצג בשרשור הצ'אט.

## ניהול תוספים שנוצרו {#managing-generated-plugins}

תוספים שנוצרו מופיעים בכתובת **Gratis AI Agent → Plugin Builder → Manage Plugins**. ממסך זה אתה יכול:

| פעולה | תיאור |
|---|---|
| **View source** | סקירה של קוד ה-PHP המלא של התוסף. |
| **Re-activate in sandbox** | הרצת בדיקת ההפעלה ב-Sandbox מחדש. |
| **Install on network** | פריסה של התוסף לרשת החיה (דורש אישור ידני). |
| **Update** | מתן גרסה חדשה דרך ה-AI, והחלפת הקוד הקיים. |
| **Delete** | הסרת התוסף ממחסן ה-Sandbox. מפעיל אותו מחדש מכל האתרים קודם. |

:::warning
**Install on network** פורס את התוסף שנוצר ל-WordPress Multisite החי שלך. סקור את קוד התוסף לפני ההמשך. Gratis AI Agent יבקש אישור לפני השלמת התקנה חיה.
:::

## התקנת תוסף שנוצר ברשת {#installing-a-generated-plugin-on-the-network}

כאשר אתה מרוצה מתוסף שעבר בדיקת Sandbox, אתה יכול להתקין אותו ברשת החיה:

1. עבור אל **Gratis AI Agent → Plugin Builder → Manage Plugins**.
2. לחץ על **Install on Network** ליד התוסף שברצונך לפרוס.
3. אשר את הדיאלוג. התוסף מותקן ל-`wp-content/plugins/` ומופעל ברמת הרשת.

לחלופין, השתמש בפקודת ה-slash בממשק הצ'אט:

```
/install-plugin <plugin-slug>
```

## עדכוני תוספים {#plugin-updates}

כדי לעדכן תוסף שנוצר, תאר את השינוי למساعد ה-AI בשיחה חדשה:

> "עדכן את תוסף dashboard-notice כך שיציג את ההודעה רק למנהלים."

ה-AI ייצור גרסה חדשה, אשר תופיע ב-Sandbox לצד הגרסה הנוכחית. אתה סוקר את ההבדלים ומאשר לפני שהעדכון מיושם.

## אינטגרציית HookScanner {#hookscanner-integration}

ה-Plugin Builder משתמש ב-**HookScanner** משולב כדי לנתח את ה-hooks והפילטרים שנרשמו על ידי כל תוסף שנוצר. הפלט של HookScanner מוצג בתגובת הצ'אט וכולל:

- hooks פעולה שנרשמו (`add_action` calls).
- hooks פילטר שנרשמו (`add_filter` calls).
- כל hooks שנמצאו בתלות של התוסף (מדלג על התיקיות `vendor/` ו-`node_modules/`).

זה עוזר לך להבין את התנהגות התוסף לפני הפעלתו.

## שיקולי אבטחה {#security-considerations}

- תוספים שנוצרו מאוחסנים בנפרד מהתוספים שהותקנו ידנית ואינם נגישים דרך מסך ניהול התוספים הסטנדרטי של WordPress עד שתתקין אותם במפורש ברשת.
- ה-Sandbox משתמש באימות נתיב (path validation) כדי למנוע מעבר תיקיות בעת כתיבת קבצי תוסף.
- תוספים עם קריאות פונקציה מסוכנות (כגון `eval`, `exec`, `system`) מסומנים במהלך האימות ולא יופעלו.
