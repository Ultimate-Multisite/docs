# משוב ודיווח על תקלות {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 מציג מערכת מובנית למשוב ודיווח על תקלות, המאפשרת למשתמשים הסופיים לסמן תגובות שאינן מועילות ולדווח על בעיות ישירות מממשק הצ'אט. מערכת זו כוללת ניהול הסכמה, פקודת דיווח אוטומטית, ותהליך מיון (Triage) מבוסס AI בצד השרת.

## כפתור האגודל למטה {#thumbs-down-button}

כל הודעה שנשלחת על ידי עוזר ה-AI מציגה כפתור **אגודל למטה** (👎). כאשר משתמש לוחץ עליו, הוא יכול לסמן תגובה כלא מועילה או שגויה.

*   הכפתור מופיע במעבר העכבר ליד כל הודעה של העוזר.
*   לחיצה עליו פותחת את **מודל הסכמת המשוב** (Feedback Consent Modal).
*   המשוב מקושר לשרשור השיחה ולמסמך הספציפי.

## מודל הסכמת המשוב {#feedback-consent-modal}

כאשר משתמש לוחץ על כפתור האגודל למטה, מופיע מודל הסכמה לפני שכל נתון נשלח. המודל:

*   מסביר אילו מידע ייאסף (קטע מהשיחה, הקשר הדפדפן).
*   מבקש מהמשתמש לאשר שהוא מסכים לשיתוף נתונים אלה.
*   מספק שדה טקסט חופשי שבו המשתמש יכול לתאר מה הלך לא בסדר.
*   מציע אפשרות **ביטול** כדי לסגור את המודל מבלי לשלוח.

לא מתועד משוב עד שהמשתמש מאשר במפורש.

## באנר משוב מבוסס פרומפט אוטומטי {#auto-prompt-feedback-banner}

בסוף שיחה, העוזר עשוי להציג **באנר משוב מבוסס פרומפט אוטומטי** – הודעה לא פולשנית ששואלת אם המפגש היה מועיל.

באנר זה מופיע אוטומטית בהתבסס על אורך השיחה וההיוריסטיקות של התוצאה. הוא מקשר לתהליך המשוב הזהה לכפתור האגודל למטה. משתמשים יכולים לסגור את הבאנר מבלי לספק משוב.

## פקודת סלאש /report-issue {#report-issue-slash-command}

משתמשים יכולים להפעיל את תהליך המשוב ישירות על ידי הקלדת `/report-issue` בשדה הקלט של הצ'אט. פקודה זו:

*   פותחת את מודל הסכמת המשוב באופן מיידי.
*   ממלאה מראש את שדה התיאור בהקשר לגבי השיחה הנוכחית.
*   מאפשרת למשתמשים להוסיף פרטים נוספים לפני השליחה.

פקודת `/report-issue` זמינה בכל מצבי הצ'אט (inline, floating widget, full-screen).

## מיון מבוסס AI {#ai-assisted-triage}

המשוב שנשלח מועבר לשכבת מיון (Triage) מבוססת AI, אשר:

*   ממיין את הדיווח (שגיאה עובדתית, תשובה לא מועילה, תוכן לא הולם וכו').
*   מצרף מטא-דאטה רלוונטי של השיחה.
*   מעביר את סיכום המיון ל**נקודת הקצה למשוב** (Feedback Endpoint) המוגדרת (ראה [Settings > Advanced](#settings-advanced)).

זה מקטין את זמן המراجعة הידנית על ידי הצגת הבעיות הקריטיות ביותר תחילה.

## Settings > Advanced {#settings-advanced}

כדי להפעיל את צד השרת של המשוב, הגדר את השדות הבאים תחת **Gratis AI Agent → Settings → Advanced**:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | כתובת ה-URL שמקבלת בקשות POST עם נתוני משוב (JSON). |
| **Feedback API Key** | Bearer token שנשלח בכותרת `Authorization` לאימות שליחת המשוב. |

השאר את שני השדות ריקים כדי לבטל איסוף משוב. כפתורי המשוב האישיים נשארים גלויים למשתמשים, אך ההגשות לא יועברו.

:::tip
נקודת הקצה למשוב חייבת לקבל גוף JSON עם לפחות השדות `message_id`, `conversation_id`, `feedback_text`, ו-`triage_category`. ראה את התיעוד של ספק נקודת הקצה שלך עבור הסכמה הצפויה.
:::
