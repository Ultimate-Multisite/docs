# ניהול שדה התשלום ו-DNS של לקוחות

## שדה התשלום לבחירת דומיין

השדה **בחירת דומיין** הוא רכיב תשלום שמאפשר ללקוחות לבחור כיצד לקבל את הדומיין של האתר שלהם. הוסיפו אותו לכל טופס תשלום כדי לאפשר מכירת דומיינים.

### הוספת השדה לטופס תשלום

1. עברו אל **Network Admin › Ultimate Multisite › טופסי תשלום**
2. פתחו או צרו טופס תשלום
3. בעורך התשלום, לחצו על **הוספת שדה**
4. בחרו **בחירת דומיין** מרשימת השדות
5. הגדירו את אפשרויות השדה (ראו להלן)
6. שמרו את הטופס

### אפשרויות השדה

**מצבי דומיין** — בחרו אילו לשוניות הלקוח יראה. ניתן להפעיל או להשבית כל מצב בנפרד:

| מצב | מה הוא עושה |
|---|---|
| **תת-דומיין** | הלקוח משתמש בתת-דומיין חינמי ברשת שלכם (לדוגמה, `mysite.yournetwork.com`). אין צורך בתשלום. |
| **רישום דומיין חדש** | הלקוח מחפש דומיין חדש ורושם אותו דרך הספק שהגדרתם. משתמש במוצר הדומיין התואם לצורך התמחור. |
| **דומיין קיים** | הלקוח ממפה דומיין שכבר בבעלותו. אין דמי רישום. הדומיין ממופה אוטומטית לאתר שלו. |

**מצב ברירת מחדל** — כאשר כל שלושת המצבים מופעלים, איזו לשונית נפתחת ראשונה. הגדירו ל-**תת-דומיין** כדי להשאיר את רישום הדומיין כאופציונלי, או ל-**רישום דומיין חדש** כדי לעודד רכישות.

**מוצר דומיין** — ניתן להצמיד שדה זה למוצר דומיין מסוים. אם לא הוגדר, התוסף בוחר אוטומטית את המוצר התואם לפי ה-TLD שהלקוח מחפש.

### שדות איש קשר של הנרשם

כאשר לקוח בוחר בלשונית **רישום דומיין חדש**, טופס התשלום מוסיף שדות איש קשר של הנרשם בתוך הטופס:

- שם פרטי / שם משפחה
- כתובת דוא"ל
- כתובת (שורה 1, עיר, מדינה/מחוז, מיקוד, ארץ)
- מספר טלפון

שדות אלה נדרשים על ידי כל הרשמים ומאומתים לפני שמתבצעת קריאת API לרישום. מספרי טלפון מעוצבים אוטומטית לפורמט הבינלאומי `+CC.NNN` שהרשמים מצפים לו.

### כתובת אתר שנוצרת אוטומטית

כאשר לקוח רושם או ממפה דומיין, שדה כתובת האתר מתמלא אוטומטית מתוך הדומיין שנבחר. הלקוחות אינם צריכים למלא שדה כתובת נפרד.

### התנהגות חיפוש

- זמינות הדומיין נבדקת בזמן אמת באמצעות AJAX בזמן שהלקוח מקליד
- הצעות TLD חלופיות מוצגות כאשר הדומיין המועדף אינו זמין
- התמחור נשלף בזמן אמת ומוצג בבירור (מחיר רישום, מחיר חידוש, עמלת פרטיות WHOIS אופציונלית)
- קודי קופון חלים על מוצרי דומיין כמו על כל מוצר אחר

**כוונון תגובתיות החיפוש:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**הוספת שדות מותאמים אישית לטופס חיפוש הדומיין:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ניהול DNS של לקוחות

לקוחות יכולים לנהל רשומות DNS עבור הדומיינים הרשומים שלהם מתוך עמוד **My Account**, תחת הרשומה של הדומיין שלהם.

### סוגי רשומות נתמכים

| סוג | שימוש |
|---|---|
| **A** | מיפוי שם מארח לכתובת IPv4 |
| **AAAA** | מיפוי שם מארח לכתובת IPv6 |
| **CNAME** | יצירת כינוי שמצביע לשם מארח אחר |
| **MX** | הגדרת שרת החלפת דואר |
| **TXT** | הוספת SPF, DMARC, אימות, או רשומות טקסט אחרות |

### אילו ספקים תומכים בניהול DNS?

ניהול DNS (הוספה, עריכה, מחיקת רשומות) זמין עם **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, ו-**Openprovider**. דומיינים של **Hostinger** יכולים לעדכן שרתי שמות דרך Domain Seller; רשומות DNS לדומיינים מאוחסנים מנוהלות על ידי אינטגרציית מיפוי הדומיינים המרכזית של Hostinger. דומיינים של Namecheap, GoDaddy, ו-NameSilo מציגים פרטי סטטוס ותפוגה, אך יש לנהל DNS ישירות בלוח הבקרה של הרשם.

### רשומות DNS כברירת מחדל

ניתן להגדיר רשומות DNS כברירת מחדל שיוחלו אוטומטית כאשר דומיין נרשם. עברו אל **הגדרות › Domain Seller › רשומות DNS כברירת מחדל**.

ערכי רשומות ברירת המחדל תומכים בשני אסימונים:

| אסימון | מוחלף ב- |
|---|---|
| `{DOMAIN}` | שם הדומיין הרשום (לדוגמה, `example.com`) |
| `{SITE_URL}` | כתובת אתר ה-WordPress עבור אתר הלקוח |

**דוגמה — הפניית דומיין apex ו-www לכתובת ה-IP של השרת שלכם:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### מנהל: צפייה ועריכה של DNS

מנהלי רשת יכולים לצפות ולערוך רשומות DNS עבור כל דומיין של לקוח מעמוד העריכה של הדומיין ב-**Network Admin › Ultimate Multisite › דומיינים**.
