# ניהול שדה דומיין ו-DNS של הלקוח

## שדה בחירת הדומיין בתשלום

שדה **Domain Selection** הוא אלמנט תשלום המאפשר ללקוחות לבחור כיצד להשיג את דומיין האתר שלהם. הוסף אותו לכל טופס תשלום כדי לאפשר מכירת דומיינים.

### הוספת השדה לטופס תשלום

1. עבור אל **Network Admin › Ultimate Multisite › Checkout Forms**
2. פתח או צור טופס תשלום
3. בעורך התשלום, לחץ על **Add Field**
4. בחר **Domain Selection** מרשימת השדות
5. הגדר את אפשרויות השדה (ראה בהמשך)
6. שמור את הטופס

### אפשרויות השדה

**Domain modes** — בחר אילו לשוניות הלקוח יראה. ניתן להפעיל או להשבית כל מצב באופן עצמאי:

| Mode | מה זה עושה |
|---|---|
| **Subdomain** | הלקוח משתמש בדומיין משנה חינמי ברשת שלך (לדוגמה: `mysite.yournetwork.com`). אין צורך בתשלום. |
| **Register New Domain** | הלקוח מחפש דומיין חדש ורושם אותו דרך הספק שהוגדר אצלך. משתמש במוצר הדומיין התואם לצורך תמחור. |
| **Existing Domain** | הלקוח מקשר דומיין שכבר יש לו. אין דמי רישום. הדומיין מקושר אוטומטית לאתר שלו. |

**Default mode** — כאשר כל שלושת המצבים מופעלים, איזו לשונית נפתחת ראשונה. הגדר ל-**Subdomain** כדי לשמור על רישום הדומיין אופציונלי, או ל-**Register New Domain** כדי לעודד רכישות.

**Domain product** — ניתן לקבע שדה זה באופן אופציונלי למוצר דומיין ספציפי. אם לא הוגדר, ה-addon בוחר אוטומטית את המוצר התואם בהתבסס על TLD שהלקוח מחפש.

### שדות יצירת קשר של הרישום (Registrant contact fields)

כאשר הלקוח בוחר בלשונית **Register New Domain**, טופס התשלום מוסיף שדות יצירת קשר של הרישום באופן מקוון:

- שם פרטי / שם משפחה
- כתובת אימייל
- כתובת (שורה 1, עיר, מחוז/מדינה, מיקוד, מדינה)
- מספר טלפון

שדות אלה נדרשים על ידי כל רשם (registrar) ומאומתים לפני שמתבצע קריאת ה-API של הרישום. מספרי טלפון מעוצבים אוטומטית לפורמט הבינלאומי `+CC.NNN` הצפוי על ידי רשמים.

### כתובת ה-URL של האתר שנוצרה אוטומטית

כאשר הלקוח רושם או מקשר דומיין, שדה כתובת ה-URL של האתר מתמלא אוטומטית מהדומיין שנבחר. הלקוחות אינם צריכים למלא שדה URL נפרד.

### התנהגות החיפוש

- זמינות הדומיין נבדקת בזמן אמת באמצעות AJAX בזמן שהלקוח מקליד
- הצעות TLD חלופיות מוצגות כאשר הדומיין המועדף אינו זמין
- התמחור נמשך חי ומוצג בבירור (מחיר רישום, מחיר חידוש, דמי פרטיות WHOIS אופציונליים)
- קודי קופון יחולו על מוצרי דומיין באותה מידה כמו על כל מוצר אחר

**כוונון תגובת החיפוש:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**הוספת שדות מותאמים אישית לטופס חיפוש הדומיין:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ניהול ה-DNS של הלקוח

לקוחות יכולים לנהל רשומות DNS עבור הדומיינים שהם רשמו מדף **My Account**, תחת הרשומה של הדומיין שלהם.

### סוגי רשומות נתמכים

| Type | שימוש |
|---|---|
| **A** | מיפוי שם מארח (hostname) לכתובת IPv4 |
| **AAAA** | מיפוי שם מארח (hostname) לכתובת IPv6 |
| **CNAME** | יצירת כינוי (alias) המצביע לשם מארח אחר |
| **MX** | הגדרת שרת החלפת המייל (mail exchange server) |
| **TXT** | הוספת רשומות טקסט כמו SPF, DMARC, אימות או רשומות טקסט אחרות |

### אילו ספקיות תומכות בניהול DNS?

ניהול DNS (הוספה, עריכה, מחיקה של רשומות) זמין עם **OpenSRS**, **ResellerClub** ו-**Enom**. דומיינים של Namecheap, GoDaddy ו-NameSilo מציגים מידע על סטטוס ופקיעה, אך ה-DNS חייב להיות מנוהל ישירות בלוח הבקרה של הרשם.

### רשומות DNS ברירת מחדל

ניתן להגדיר רשומות DNS ברירת מחדל שיופעלו אוטומטית כאשר דומיין נרשם. עבור אל **Settings › Domain Seller › Default DNS Records**.

ערכי רשומות ברירת המחדל תומכים בשני טוקנים:

| Token | מוחלף ב- |
|---|---|
| `{DOMAIN}` | שם הדומיין הרישום (לדוגמה: `example.com`) |
| `{SITE_URL}` | כתובת האתר של וורדפרס של הלקוח |

**דוגמה — הצבעת דומיין האפיקס ו-www לכתובת ה-IP של השרת שלך:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### מנהל: צפייה ועריכת DNS

מנהלי רשת יכולים לצפות ולערוך רשומות DNS עבור כל דומיין של לקוח מדף העריכה של הדומיין ב-**Network Admin › Ultimate Multisite › Domains**.
