# Domain Seller: הגדרה ותצורת ספק

התוסף Domain Seller מגיע עם אשף הגדרה מודרך שמוביל אותך דרך כל שלב נדרש. עמוד זה מכסה את זרימת האשף וכיצד להגדיר או להגדיר מחדש ספקים לאחר מכן.

## דרישות {#requirements}

- **Multisite Ultimate** v2.4.12 ומעלה, מופעל ברשת
- **PHP** 7.4+
- פרטי גישה ל-API עבור רשם נתמך אחד לפחות

## אשף הגדרה בהרצה ראשונה {#first-run-setup-wizard}

אשף ההגדרה מופעל אוטומטית בפעם הראשונה שבה מפעילים את התוסף ברשת. הוא זמין גם בכל עת מתוך **מנהל רשת › Ultimate Multisite › הגדרת Domain Seller**.

### שלב 1 — בחירת ספק {#step-1--choose-a-provider}

בחר את הרשם שברצונך לחבר. אפשרויות נתמכות:

| ספק | ניהול DNS | פרטיות WHOIS |
|---|---|---|
| OpenSRS | כן | כן |
| Namecheap | לא | כן (WhoisGuard, חינם) |
| HostAfrica | כן | כן (הגנת מזהה) |
| Openprovider | כן | כן |
| Hostinger | דרך מיפוי דומיינים של ליבת Hostinger עבור דומיינים מאוחסנים | כן |
| GoDaddy | לא | לא |
| ResellerClub | כן | לא |
| NameSilo | לא | לא |
| Enom | כן | לא |

### שלב 2 — הזנת פרטי גישה {#step-2--enter-credentials}

לכל ספק יש שדות פרטי גישה שונים:

**OpenSRS** — שם משתמש ומפתח פרטי (מתוך OpenSRS Reseller Control Panel)

**Namecheap** — שם משתמש ומפתח API (מתוך Account › כלים › גישת API)

**HostAfrica** — נקודת קצה של Domains Reseller API ופרטי גישה ממודול המשווקים של HostAfrica. כרגע לא מתועדת נקודת קצה נפרדת ל-Sandbox; בדוק באמצעות בדיקות בטוחות לקריאה בלבד לפני הפעלת רישומים חיים.

**Openprovider** — שם משתמש וסיסמה עם גישת API מופעלת. מצב Sandbox אופציונלי משתמש ב-Openprovider sandbox API, וניתן לעשות שימוש חוזר במזהה לקוח ברירת מחדל אופציונלי עבור רישומים.

**Hostinger** — אסימון ה-API המשותף של Hostinger hPanel מתוך שילוב Hostinger בליבה. אותו אסימון מפעיל את מיפוי הדומיינים בליבה ואת פעולות הרישום של Domain Seller.

**GoDaddy** — מפתח API וסוד (מתוך developer.godaddy.com)

**ResellerClub** — מזהה משווק ומפתח API (מתוך לוח הבקרה של ResellerClub)

**NameSilo** — מפתח API (מתוך namesilo.com › Account › מנהל API)

**Enom** — מזהה Account ואסימון API

סמן **מצב Sandbox** כאשר הוא זמין כדי לבדוק מול סביבת הבדיקה של הספק לפני עלייה לאוויר.

### שלב 3 — בדיקת החיבור {#step-3--test-the-connection}

לחץ על **בדיקת חיבור**. האשף שולח קריאת API קלה כדי לאמת את פרטי הגישה ואת הקישוריות. תקן כל בעיית פרטי גישה לפני שתמשיך.

### שלב 4 — ייבוא TLDs {#step-4--import-tlds}

לחץ על **ייבוא TLDs** כדי למשוך את כל ה-TLDs הזמינים ואת תמחור הסיטונאות מהספק המחובר. פעולה זו ממלאת את רשימת ה-TLD המשמשת מוצרי דומיין. הייבוא עשוי להימשך 30–60 שניות עבור ספקים עם קטלוגי TLD גדולים.

TLDs מסונכרנים מחדש גם אוטומטית פעם ביום באמצעות משימת cron מתוזמנת.

### שלב 5 — יצירת מוצר דומיין {#step-5--create-a-domain-product}

האשף יוצר מוצר דומיין ברירת מחדל כולל עם תוספת מחיר של 10%. ניתן לערוך מוצר זה מיד או לדלג וליצור מוצרים ידנית תחת **Ultimate Multisite › מוצרים**.

ראה [מוצרי דומיין ותמחור](./domain-products) למדריך תצורת המוצר המלא.

## הגדרה מחדש של ספק {#reconfiguring-a-provider}

עבור אל **מנהל רשת › Ultimate Multisite › הגדרות › Domain Seller** (או לחץ על **הגדרות** ברשימת התוספים).

עמוד ההגדרות מכיל:

- **הפעלת מכירת דומיינים** — החלפת המאפיין כולו בין פעיל/כבוי
- **ספק ברירת מחדל** — הספק המשמש לחיפושי דומיינים ולמוצרים חדשים
- **מספר TLDs מרבי לכל חיפוש** — כמה TLDs לבדוק כאשר לקוח מחפש; ערכים גבוהים יותר מציגים יותר אפשרויות אך איטיים יותר
- **משך מטמון זמינות** — למשך כמה זמן לשמור במטמון תוצאות זמינות ותמחור; ערכים נמוכים יותר מדויקים יותר אך מגדילים קריאות API
- **ניהול מוצרי דומיין** — קישור מהיר לרשימת המוצרים
- **הגדרת ספקים** — פותח את אשף האינטגרציה כדי להוסיף או להגדיר מחדש ספקים

### הוספת ספק שני {#adding-a-second-provider}

לחץ על **הגדרת ספקים** והפעל שוב את האשף עבור הרשם החדש. ניתן להגדיר כמה ספקים בו-זמנית. שייך כל מוצר דומיין לספק מסוים, או השאר אותו על ברירת המחדל.

### סנכרון TLDs ידני {#syncing-tlds-manually}

בעמוד ההגדרות, לחץ על **סנכרון TLDs** לצד כל ספק מוגדר כדי למשוך את התמחור העדכני ביותר. הדבר שימושי לאחר שספק מעדכן תמחור סיטונאי או מוסיף TLDs חדשים.

## הערות יכולות ספק {#provider-capability-notes}

לא כל API של רשם חושף את אותן פעולות. התוסף מציג פעולות שאינן נתמכות באמצעות שגיאות ברורות המיועדות למנהל, במקום להיכשל בשקט.

- **HostAfrica** תומך בזרימת עבודה חיה הרחבה ביותר למשווקים, כולל חיפוש, סנכרון TLD/תמחור, רישום, חידוש, העברה, עדכוני nameserver, רשומות DNS, קודי EPP, נעילת רשם, והגנת מזהה.
- **Openprovider** תומך בסנכרון TLD במחירי משווק, רישום, חידוש, העברות, עדכוני nameserver, אזורי DNS, קודי EPP, נעילת רשם, ופרטיות WHOIS. הוא מאמת באמצעות bearer token קצר-חיים שהתוסף מרענן אוטומטית.
- **Hostinger** תומך בחיפוש זמינות, רישום, חיפוש בפורטפוליו, עדכוני nameserver, נעילת רשם, ופרטיות WHOIS דרך טוקן API hPanel המשותף. Domains API הציבורי של Hostinger אינו חושף תמחור משווק/סיטונאי, העברה נכנסת, חידוש מפורש, או שליפת קוד EPP; חידושים הם חידוש אוטומטי בלבד.
