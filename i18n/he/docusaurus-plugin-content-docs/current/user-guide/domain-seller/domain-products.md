# מוצרי דומיין ותמחור

מוצרי דומיין הם הדרך שלך לשלוט בתמחור, TLDs, פרטיות WHOIS ובבחירת ספק. כל מוצר דומיין הוא מוצר Multisite Ultimate רגיל עם לשונית נוספת בשם **Domain Settings**.

## יצירת מוצר דומיין

1. עבור אל **Network Admin › Ultimate Multisite › Products**
2. לחץ על **Add New**
3. הגדר את סוג המוצר ל-**Domain**
4. הגדר את לשונית **Domain Settings** (ראה בהמשך)
5. שמור

מוצרי דומיין מופיעים עם תגית סגולה של **Domain** ברשימת המוצרים ואפשר לסנן אותם באמצעות לשונית **Domain Products**.

## לשונית הגדרות דומיין (Domain settings tab)

### Provider

בחר איזה רגיסטר (registrar) מטפל ברישום עבור מוצר זה. ברירת המחדל היא **Default provider** הגלובלי המוגדר בהגדרות Domain Seller.

### Supported TLDs

השאר ריק כדי ליצור **מוצר Catch-all** שמתאים לכל TLD שאינו תואם על ידי מוצר אחר.

הזן רשימה מופרדת בפסיקים של TLDs (לדוגמה: `.com, .net, .org`) כדי ליצור **מוצר ספציפי ל-TLD** שמתאים רק לסיומות הללו.

**איך עובדת התאמת המוצר:** כאשר לקוח מחפש דומיין, התוסף בוחר את המוצר התואם והספציפי ביותר. מוצר עם `.com` ברשימת ה-TLDs שלו מקבל עדיפות על פני מוצר Catch-all. אם אף מוצר ספציפי ל-TLD לא תואם, משתמשים ב-Catch-all. אם לא קיימים מוצרים, חיפוש הדומיין לא יוצג.

### Markup type

שלושה מצבים שולטים כיצד מחושב מחיר הקמעונאות שלך בהתבסס על עלות הסיטונאות:

| Mode | How it works |
|---|---|
| **Percentage** | מוסיף אחוז מסוים מעל עלות הסיטונאות. תוספת של 20% על דומיין סיטונאי בשווי 10$ נותנת 12$. |
| **Fixed markup** | מוסיף סכום דולרי קבוע. תוספת של 5$ על דומיין בשווי 10$ נותנת 15$. |
| **Fixed price** | מתעלם לחלוטין מעלות הסיטונאות. גובה תמיד את הסכום שהזנת. |

### Introductory pricing

הפעלה כדי להציע מחיר מוזל לשנה הראשונה. הגדר **Introductory price** נפרד (מחיר השנה הראשונה) לצד **Renewal price** הרגיל (שנה 2 ואילך). הלקוח רואה את שני המחירים המוצגים במהלך התשלום, כדי שידע למה לצפות בהתחדשות.

### WHOIS privacy

שולט אם מוגנת פרטיות WHOIS עבור דומיינים שנרשמים דרך מוצר זה.

| Setting | Behaviour |
|---|---|
| **Disabled** | פרטיות WHOIS לעולם אינה מוצעת או מופעלת. |
| **Always Included** | פרטיות WHOIS מופעלת אוטומטית ברישום ללא עלות. |
| **Customer Choice** | מופיע תיבת סימון במהלך התשלום. הגדר את **Privacy price** כדי לגבות עבורו כל שנה, או השאר אותו ב-0$ כדי להציע אותו בחינם. |

עבור Namecheap, פרטיות WHOIS משתמשת ב-WhoisGuard (חינם תמיד). עבור OpenSRS, היא משתמשת בשירות הפרטיות של OpenSRS (עלול להיות בעלות סיטונאית).

## ייבוא וסנכרון TLD

מוצרי דומיין מציגים תמחור סיטונאי בזמן אמת שנמשך מהספק המחובר. כדי שזה יעבוד, יש לייבא את ה-TLDs.

- **סנכרון ידני:** Settings › Domain Seller › Sync TLDs (לפי ספק)
- **סנכרון אוטומטי:** רץ מדי יום באמצעות משימת cron מתוזמנת עבור כל הספקים המוגדרים.

לאחר סנכרון, עבור ללשונית הגדרות הדומיין של כל מוצר דומיין והשתמש במארז ה-TLD כדי לראות את ה-TLDs הזמינים יחד עם מחירי הסיטונאות הנוכחיים שלהם.

## חידוש אוטומטי (Auto-renewal)

חידושי דומיין קשורים למעמד המנוי של הלקוח:

- כאשר המנוי מתחדש ודומיין מקושר, חידוש הדומיין נכנס לתור אוטומטית
- ניסיונות חידוש משתמשים בשער התשלום הפעיל של הלקוח
- חידושים שנכשלו מנסים שוב אוטומטית עם Backoff אקספוננציאלי
- מוקדות אימייל נשלחות עבור חידושים מוצלחים, כשלונות ופקיעות קרובות

מזהי תבניות אימייל לאירועי מחזור חיי הדומיין:

| Event | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## מנהל: רישום דומיין ידני

כדי לרשום דומיין מטעם לקוח מבלי שהוא יעבור תהליך תשלום:

1. עבור אל **Network Admin › Ultimate Multisite › Register Domain**
2. בחר את הלקוח והזן את שם הדומיין
3. מלא את פרטי הקשר של הרישום (שם, כתובת, טלפון)
4. לחץ על **Register**

רשומת הדומיין נוצרת וקושרת לחשבון הלקוח.
