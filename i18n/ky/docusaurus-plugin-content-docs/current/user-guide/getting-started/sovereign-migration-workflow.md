# Суверенитеттегирүү Workflow

Ultimate Multisite: Multi-Tenancy 1.2.0 стандарт подсайтты суверенитет теңдеме инфраструктурасына которуу үчүн миграцияны текшерүү барьерлерин кошот.

## Баштапкы иштерден мурун {#before-you-start}

Төмөнкүлөрдү такталыңыз:

- Эски сайттын учурдагы резерв копии бар экенин.
- Multi-Tenancy addon активдүү экенин.
- Баа объекти (destination tenant) базасы, файл системасынын түпкүлү (filesystem root) жана домен даяр экенин.
- База хостун багыты (database host binding) баа объекти чөйрөсү менен дал келгенин.
- Тамырдык тармак үчүн WP-CLI буйругуна кирүү мүмкүнчүлүгү бар экенин.

## Сунушталган workflow {#recommended-workflow}

1. Баа объекти базасын жана файл системасын даярдаңыз.
2. Баа объекттеринин изоляция (isolation) жөндөмдөрүн каттоодон же жаңыртыңыз.
3. Баа миграциясын аткарыңыз.
4. Баачууларды (tenant users) колдонуңуз же текшериңиз.
5. Асинхрон миграция жумуштарын (async migration jobs) тазалаңыз.
6. Суверенитет миграциясын текшерүүнү аткарыңыз.
7. SSO менен баа объектине кириңиз.
8. Текшерүү өтсө, DNS же маршрутизацияны алмаштырууну гана андан кийин жасаңыз.

## Текшерүү барьерлери (Verification gates) {#verification-gates}

Текшерүү workflow миграцияны бир нече бурчтан текшерет:

- Баа объекти схемасы бар жана аны баа базасынын жазуучусу аркылуу жазып жибере алат.
- Конфигурацияланган база хостунда күчтүү укуктар (grants) бар экени.
- Баачуулар бар жана күтүлгөн суверенитет орнотуучу колдонуучу санына дал келиши.
- Асинхрон жөнөтүү kuyрусу (async push queue) ийгиликтүү тазаланышы мүмкүн.
- Суверенитет баа объекти үчүн эски тармак тарабындагы маалыматтык жолдор дагы талап кылынбайт.

Текшерүүнүн каталарын иш баштоодон мурун токтотуучу фактор катары караңыз. Баяндалган база, колдонуучу, kuyру же маршрутизация маселесин оңдонуп, андан кийин баа объектине кардаркыларга көрсөтүүдөн мурун текшерүүнү кайра аткарыңыз.

## Биринчи продукциялык кирилиш (First production visit) {#first-production-visit}

Текшерүү өтсө, биринчи административ кирилиш үчүн сайттын башкаруу экранынан **Visit (SSO)** функциясын колдонуңуз. Бул баа объектинин маршрутизациясын, SSO токенине мамилесин, булак белгиленишин жана баачуу тарабындагы колдонуучуларды даярдоо процессин бир контролдогон кадамда текшерет.
