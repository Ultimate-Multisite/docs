# Домен картированиен кантип конфигурациялоо (v2)

_**МААЛЫМДАР ҮЧҮН ЭСКАРТЫК ТУУЛУУ: Бул макала Ultimate Multisite 2.x версиясына тиешелүү.**_

Премиум сетьтин эң күчтүү функцияларынын бири — кардарларыңызга сайттарына топ-уровень домен (top-level domain) тиркеп берүү мүмкүнчүлүгүн сунуштоо. Анткени, кайсысы көбүрөөк профессионалдуу көрүнөт: [_**joesbikeshop.yournetwork.com**_ же [_**joesbikeshop.com**_? Ошондуктан Ultimate Multisite бул функцияны үчүнчү тарап плагинин колдонуусуз даяр кылган, алдында да用意лган.

## Домен картирование деген эмне?

Атынан айткандай, домен картирование — Ultimate Multisite тарабынан каалаган домендү сурап алуу жана ошол суранычты сетьтеги тиешелүү сайтта аталган домен менен картага салуу мүмкүнчүлүгү.

### Ultimate Multisite Сетиңизде домен картирование кантип орнотуу керек

Домен картирование иштөө үчүн сиздин тарабынан бир нече жөнгө салуулар талап кылынат. Баарын айтсак, Ultimate Multisite бул оор ишти автоматташтырып берет, ошондуктан сиз талаптарга оңой жооп бере аласыз.

Ultimate Multisite орнотууда, wizards (жумушчу) **sunrise.php** файлын белгилүү папкага автоматтык түрдө көчүрүп, орнотуп берет. **Бул кадам аткарылганча wizards сизге улантууга жол бербейт**.

<!-- Screenshot unavailable: Ultimate Multisite installation wizard with sunrise.php step -->

Бул дегенди, Ultimate Multisite орнотувчу wizards сетьиңизди даярдап бүтүрүп жатканда, сиз дароо каалаган домендү картилоого баштасаңыз болот.

Ultimate Multisiteде домен картирование милдеттүү эмес экенин белгилей туруңуз. Сиз WordPress Multisite-тин өзүнүн домен картирование функциясын же башка домен картирование чечисин колдонууну тандап койсоңуз болот.

Эгер сизге Ultimate Multisite домен маппингди башка домен маппинг чечимдерине жол берүү үчүн өчүрүү керек болсо, сиз бул функцияны **Ultimate Multisite > Settings > Domain Mapping** (Домен Маппинг) аркылуу өчүрө аласыз.

![Domain Mapping settings page showing admin redirect, mapping message and DNS options](/img/config/domain-mapping-settings.png)

Бул опциянын астында сиз **Force Admin Redirect** (Админди күчөтүүдөргө багыттоо) опциясын да көрө аласыз. Бул опция сизге мурунку милдеттенмелердин алдында кардарлар өзүнүн кастом домени жана поддоменү аркылуу административдик панелдөрүн колдонуп кире аларын контроллоого мүмкүндүк берет.

Эгер сиз **Force redirect to mapped domain** (Мапталган доменгө күчөтүүдөргө багытуу) опциясын тандайсыз, кардарлар административдик панелдөрүн гана кастом домени аркылуу колдонуп кире алышат.

**Network domain** (Сеть доменине күчөтүүдөргө багытуу) опциясы эس сапатта келет – кардарлар кастом домени аркылуу кирүүгө аракетсе да, административдик панелдөрүн гана поддоменү аркылуу колдонууга уруксат берилет.

Ал эми **Allow access to the admin by both mapped domain domain and network domain** (Мапталган домен жана сеть домени аркылуу администраторго кирүүгө уруксат берүү) опциясы аларды административдик панелдөрүн поддомен жана кастом домен боюнча колдонууга мүмкүнчүлүк берет.

![Admin Redirect dropdown expanded showing the three redirect options](/img/config/domain-mapping-redirect-options.png)

Кастом домени маппулоо үчүн эки жол бар. Биринчиси – сеть администратордук панелинде домен атынан супер администратор катары маппута кылуу, экинчиси – аккаунт баракчасы астындагы подсайт администратордук панел аркылуу.

Бирок кастом доменди сетьдеги бир подсайтта маппулоого баштоодон мурун, домен атынын **DNS жөндөмдөрүн** туура конфигурациялашыңыз керек.

###

### Домен DNS жөндөмдөрүн туура конфигурациялоону текшерүү

Карта иштөө бирге иштеши үчүн, сиз мап кылууну пландап жаткан домендин сиздин тармактын IP дарегине багытталганын камсыз кылышыңыз керек. Эмне үчүн, Ultimate Multisite орнотулган доменин IP дарегин — мап кылууну каалаган өзүнчө домендүн IP дарегин эмес — тармактын IP дарегин билүүңүз керек. Белгиделген домендин IP дарегин издөө үчүн, мисалы, [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) сыяктуу жерге барууну сунуштайбыз.

Домени туура мап кылуу үчүн, сиз **DNS** конфигурацияңызга ошол **IP дарегине** багытталган **A RECORD** кошууңуз керек. DNSны башкаруу ар кандай домен катчылары арасында бир топ өзгөрүлүшү мүмкүн, бирок " _XXXX-те A Record кантип түзүү_ " деп издөсөңүз (мысалы: " _GoDaddy-де A Record кантип түзүү_"), ага багытталган көптөгөн окутуулар интернетте бар.

Бул иштебеге туш келсеңиз, **домен катчыңыздын колдоо кызматына** кайрылыңыз, алар бул бөлүктө сизге жардам бере алышат.

Эгер сиз кардарыңызга өз домендөрүн мап кылууну уруксат берүүнү пландасаңыз, алар бул бөлүктө өз алдынча иштеши керек болот. Аларга A Recordду түзө албаса, катчылардын колдоо системасына багыт бериңиз.

### Супер администратор катары өзүнчө домен мүнөзүн мап кылуу

Тармагыңызда супер администратор катары кирип киргенде, сиз **Ultimate Multisite > Domains** деген жерден өзүнчө домен мүнөзүн оңой кошо аласыз жана башкара аласыз.

![Domains list page in Ultimate Multisite](/img/admin/domains-list.png)

Бул бетте үстүрдөгү **Add Domain** баскычына басып көрсеңиз, модал (modal) терезе ачылат жана ал жерден сиз **custom domain name** (конкреттүү домен аты), бул домен атына колдонулган **subsite** (подсайт) сунуштарын коюп киргизе аласыз. Ошондой эле, аны **primary domain** (негизги домен) катары коюп койсоңуз же койбосоңузду чечип көрөсүз (эскертүү: бир подсайтка бир нече домен атыны маппоо сунуштарын коюуга болот).

![Домен аты, сайт тандагыч жана негизги домен туташтыруучу модал терезе](/img/admin/domain-add-modal.png)

Бардык маалыматтарды киргизеп алгандан кийин, төмөндөгү **Add Existing Domain** баскычына басыңыз.

Бул бул конкреттүү домен атынын DNS маалыматтарын текшерип алуу жана алуу процессин баштайт. Процесди кандайча жүргүзгөнүн көрүү үчүн беттин төмөнкү бөлүгүндө журнал (log) да көрө аласыз. Бул процесс бүтүүгө бир нече мүнөт жетиши мүмкүн.

Ultimate Multisite v2.13.0 жаңы сайт жаралган учурда, ал хосттун сайына тиешелүү домен катары каралышы керек болгондо, ички домен кат-номаны автоматтык түрдө түзөт. Эгерде хост тармактын негизги домени же **Site URL** талаасында конфигурацияланган бөлүштүрүлгөн checkout-form домендөрүнүн бири болсо, автоматтык түрдө маппоо домен кат-номасы өткөрүлбөйт, ошондо андан колдонулган бөлүштүрүлгөн негизги домен ар бир сайт үчүн жеткиликтүү калат.

**Stage** (Баскыч) же статус **Checking DNS** (DNS-ти текшерүүдө) абалынан бардык нерсе туура коюлганда **Ready** (Даяр) абалына өзгөрөт.

<!-- Screenshot unavailable: Домен тизмесинде Checking DNS баскычын көрсөткөн Домен сабы -->

<!-- Screenshot unavailable: Жашыл статус индикатору менен Ready баскычын көрсөткөн Домен сабы -->

Домен атына басып көрсеңиз, аны ичиндеги бир нече опцияларды көрө аласыз. Аларды кыскача карап көрөлү:

![Баскыч, сайт, активдүү жана негизги домен туташтыруучу опциялар менен Домен деталдык баракча](/img/admin/domain-edit.png)

**Stage:** Бултуп (Domain) кайсы стадияда экенин көрсөтөт. Эгер сиз доменди биринчи жолу кошосоңуз, ал адатта **DNS текшерүү** стадиясында болот. Процесс DNS киргизүүлөрүн текшерип, алар туурабы аныктайт. Андан кийин, домен **SSL текшерүү** стадиясына коюлат. Ultimate Multisite доменде SSL бар же жокпусун текшерип, сиздин домениңизди **Даяр** же **SSLсиз Даяр** катары классификациялайт.

**Site:** Бул доменге тиешелүү поддомен (subdomain). Мапталган домен бул атайын сайттын мазмунун көрсөтөт.

**Active:** Домени активде кылуу же өчүрүү үчүн бул опцияны күйүп же өчүрө аласыз.

**Is Primary Domain?:** Сиздин кармалары ар бир сайт үчүн бир нече мапталган доменге ээ болушу мүмкүн. Бул атайын сайттын негизги домени экенин тандоо үчүн бул опцияны колдонуңуз.

**Is Secure?:** Ultimate Multisite доменде SSL сертификаты бар же жокпусун текшерип, аны иштетүүгө жол бербесе да, сиз доменди SSL сертификаты менен жесиз жүктөө үчүн колдонууну колдоно аласыз. Эскертүү: эгер веб-сайтта SSL сертификаты болбосо жана ага күчтөй SSL менен жүктөөнү аракет кылсаңыз, ал сизге каталар береbilir.

### Подсайт колдонуучусу катары колдонулган домен аты (custom domain name) маппалоо

Подсайт администраторлору да өзүнүн подсайт администратордук панелде колдонулган колдонулган домен аттарын маппалашы мүмкүн.

Биринчиден, сиз бул опцияны **Domain mapping** жөндөөлөрүндө күйгүзүңүз керек экенин текшериңиз. Төмөндөгү скриншоту көрүңүз.

<!-- Screenshot unavailable: Domain mapping settings allowing subsite users to map domains via Customer DNS Management toggle -->

Ошондой эле, бул опцияны **Ultimate Multisite > Products** даражасында же продукт опцияларында коюп же конфигурациялооңуз мүмкүн.

![Custom Domains section on the product edit page](/img/config/product-custom-domains.png)

Эгер сиз бул опциялардын бирин иштетсеңиз жана субсайт колдонуучусу кастом домендөрдү мапполоого уруксат берилсе, субсайт колдонуучусу **Account** (Аккаунт) баатында **Domains** (Домендер) деп аталган метабоксду көрөт.

<!-- Screenshot unavailable: Subsite Account page-де Domains метабокси -->

Колдонуучу **Add Domain** (Доменди кошуу) баскычын басып, бир нече көрсөтмөлөр менен модал виндoу көрөт.

<!-- Screenshot unavailable: Subsite колдонуучулар үчүн DNS A-рекорду көрсөткөн Add Domain модалы -->

Андан кийин колдонуучу **Next Step** (Кийинки кадам) басып, кастом домен мүнөтүн кошуу процессине өтө алат. Ошондой эле бул негизги домен болобу же болбошунсун тандаса болот.

<!-- Screenshot unavailable: Кастом домен мүнөтү талаасы жана негизги домен туташтыруучу баскыч менен Доменди кошуу формасы -->

<!-- Screenshot unavailable: DNS верификацию иштеткен Доменди кошуу ырастоо кадамы -->

**Add Domain** (Доменди кошуу) басып, кастом домендин DNS маалыматтарын текшерип алуу жана алейкит алуу процесси башталган болот.

### Домен синхролоо жөнүндө

Домен синхролоо — бул Ultimate Multisite'тин кастом домен мүнөтүн иштетүү үчүн хостинг эсебиңиздеги кошумча домен катары кошуу процессиdir.

Эгерде сиздин хостинг провайдери Ultimate Multisite домен мапполоо функциясы менен интеграциялангыса, домен синхролоо автоматтык түрдө иштеп чыгат. Учурда бул хостинг провайдерлери _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways_ жана _Cpanel._

Хостинг провайдери интеграциясы активдүү болгондо, Ultimate Multisite жаңы түзүлгөн сайттар үчүн провайдер тарабындагы DNS же поддоменди түзүү тапшырмасын да саптөй алат. Эч кандай интеграция ошол тапшырманы угуп турбаса, фоноча иштөө операциясы (background job) жоюлуп калат, бул "no-op queue entries" (ишсиз kuyк жазулары) болушунан сактанат. Мапполоо домендер үчүн DNS жана SSL текшерүүлөрү нормалдуу домен этап процесси аркылуу уланып жүрөт.

Бул интеграцияны Ultimate Multisite жөндөөлүү (settings) дагы **Integration** таамында активдештирүүңүз керек.

![Ultimate Multisite жөндөөлүүдөгү Integration таамырынын сүрөтү хостинг провайдерлерин көрсөтүп турат](/img/config/integrations-tab.png)

_Эскертүү: эгер сиздин хостинг провайдери жогору аталган провайдерлордун бири болбосо, **сиз өзүңүз домен аты менен синхрондоо же кошууңуз керек**._
