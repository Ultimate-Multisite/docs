# Планын төмөндөтүү (v2)

_**МААЛЫМДАР КӨРӨКҮН ЭСУЛТАР: Бул макала Ultimate Multisite 2.x версиясына тиешелүү._**_

Пландын жетишсиз бюджет бар же алар субсайттарын иштетүү үчүн көп ресурска муктаж эмес деп чечсе, кардарлары бул планды төмөндөтүп коюшу мүмкүн.

## Планды кантип төмөндөтүү керек

Кардарлары өзүнүн субсайт администратордук панелинде кирип, аккаунт саадалыгы (account page) астындагы **Change** баскычын басарып, планды төмөндөтө алышат.

![Сиздин мүбүр картаңыз жана Өзгөртүү баскычы бар кардардын аккаунт саадалыгы](/img/account-page/membership-change-button.png)

**Change** баскычын басандан кийин, колдонуучу/кардар өзүнүн жазылуусун өзгөртүүгө каалаган планды тандаса болот. Бул аларды төлөм саадалыгы (checkout page) чекитүүчү сапка жөнөтөт.

![Кардар тараптан планды төмөндөтүү опциялары саадалыгы](/img/account-page/downgrade-picker.png)

Бул мисалда биз планды **Premium**дан **Free**га төмөндөтүп жатабыз.

Процесди аткаруу үчүн колдонуучу жөн гана **Complete Checkout** баскычын басуу керек. Андан кийин ал кардардын мүбүрүк боюнча өзгөрүү жөнүндө билдирүү менен аккаунт саадалыгына кайтарылат. Өзгөртүүлөр кардардын **кийинки төлөм циклинде** күч алсын.

![Мүбүрүк өзгөрүшүнүн күтүлгөн баннери бар аккаунт саадалыгы](/img/account-page/pending-change.png)

### Колдонуучу планды төмөндөтсө эмне болот

Планды төмөндөтүү кардардын субсайтындагы бардык конфигурацияны өзгөртпөйт дегенди эске алуу маанилүү.

Субсайттын шаблонду өзгөртүү автоматтык түрдө планды өзгөртп жибербейт, анткени сайттын шаблонду өзгөртүү субсайтынын толугу менен жок кылып, баштап коюуну талап кылат. Бул керексиз маалымат жоголуусун алдын алуу үчүн жасалган. Ошондуктан, посттордан башка диск убактысы, темалар, плагиндер ж.б. сакталат.

Биз сиздин негизги маселеңиз ар бир пландын астында белгилген чектөөлөр жана квоталар жөнүндө экенин түштенбиз, бирок эгерде биз кандайдыр бир конфигурациясын өчүрүп же өзгөртсөк, бул колдонуучунун подсайттарына келтирүүчү зыян алып келиши мүмкүн экенин элестетишибиз керек.

Пландын белгилеген чектөөлөрүн ашып кеткен жазбалар үчүн сизде 3 түрк опция бар: **жазбаларды ошол күйүп калуу** *,* **жазбаларды коргоого (trash) которуу** *,* же **жазбаларды долбоор (draft) кылуу** *.* Булди Ultimate Multisite жөндөөлөрүнүн параметрлеринен конфигурацияласаңыз болот.

![Network Admin Settings Sites page showing post-limit-exceeded behaviour options](/img/account-page/settings-sites-post-limit.png)

### Төлөм кандай болот

2.0 версиясында төлөмдө бөлүштүрүү (proration) боюнча эч кандай реттиштер талап edilбейт.

Бул анткени система мурунку мүшелиги **анын төлөм циклин бүткүнү күтөт, андан кийин** жаңы план/мүшелик күч албайт. Жаңы мүшелик үчүн жаңы төлөм мөөнөтү автоматтык түрдө келечектеги төлөм циклинде колдонулуп алынып, алынат.
