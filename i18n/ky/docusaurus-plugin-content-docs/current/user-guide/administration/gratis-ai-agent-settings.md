# Gratis AI Agent жөндөөлөрү

Gratis AI Agent ичиндеги **Settings → Advanced** экраны backend интеграциялары үчүн администратор деңгээлиндеги конфигурацияны берет. Бул баракта пикирди жөнөтүү, издөө провайдеринин ачкычтары, башкарылуучу Superdav кызматынын орнотуусу, Google Calendar башкаруулары, TextBee SMS жөндөөлөрү жана тармак боюнча функция желектери документтештирилет.

## Жөндөөлөргө кирүү {#accessing-settings}

1. WordPress админ бөлүгүндө **Gratis AI Agent → Settings** бөлүмүнө өтүңүз.
2. **Advanced** өтмөгүн басыңыз.

## Пикир endpoint конфигурациясы {#feedback-endpoint-configuration}

Пикир endpoint AI agent thumbs-down баскычы, auto-prompt баннери же `/report-issue` буйругу аркылуу колдонуучу пикир жөнөткөн сайын POST сурамдарын кабыл алат.

| Талаа | Сүрөттөмө |
|---|---|
| **Feedback Endpoint URL** | JSON body менен HTTP POST сурамдары катары пикир жөнөтүүлөрүн кабыл алган URL. |
| **Feedback API Key** | Ар бир пикир сурамынын `Authorization` header ичинде жөнөтүлгөн bearer token. Эгер endpoint аутентификация талап кылбаса, бош калтырыңыз. |

### Күтүлгөн JSON payload {#expected-json-payload}

Пикир endpoint кеминде төмөнкү талаалары бар JSON body кабыл алышы керек:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Сүйлөшүү контекстине жараша payload ичинде кошумча талаалар болушу мүмкүн.

### `triage_category` маанилери {#triagecategory-values}

AI triage катмары payload жөнөтүүдөн мурун `triage_category` үчүн төмөнкү маанилердин бирин дайындайт:

| Маани | Мааниси |
|---|---|
| `factual_error` | Жардамчы туура эмес фактылык маалымат берди. |
| `unhelpful_answer` | Жооп техникалык жактан туура болгон, бирок пайдалуу эмес. |
| `inappropriate_content` | Жоопто колдонуучуларга көрсөтүлбөшү керек болгон мазмун бар эле. |
| `other` | Пикир белгилүү категорияга туура келген жок. |

### Аутентификация {#authentication}

Эгер endpoint аутентификация талап кылса, **Feedback API Key** талаасын bearer token катары коюңуз. Agent төмөнкүнү жөнөтөт:

```
Authorization: Bearer <your-api-key>
```

Эгер **Feedback API Key** талаасы бош болсо, `Authorization` header жөнөтүлбөйт.

### Пикир чогултууну өчүрүү {#disabling-feedback-collection}

**Feedback Endpoint URL** жана **Feedback API Key** талааларынын экөөнү тең бош калтырыңыз. Thumbs-down баскычы жана пикир UI колдонуучуларга көрүнүп тура берет, бирок жөнөтүүлөр эч кандай тышкы кызматка жөнөтүлбөйт.

## Brave Search API Key {#brave-search-api-key}

Ошондой эле **Advanced** өтмөгүндө **Brave Search API Key** талаасы [Интернет издөө](../configuration/internet-search) мүмкүнчүлүгүн иштетет.

| Талаа | Сүрөттөмө |
|---|---|
| **Brave Search API Key** | Brave Search иштеп чыгуучу dashboard ичинен алынган API ачкычыңыз. AI жардамчыда интернет издөөнү иштетүү үчүн талап кылынат. |

Талаа энбелгисинде Brave Search API катталуу барагына басылуучу шилтеме бар. Интернет издөөнү өчүрүү үчүн бош калтырыңыз.

Бул функция боюнча акыркы колдонуучу документтери үчүн [Интернет издөө](../configuration/internet-search) караңыз.

## Башкарылуучу Superdav кызматы {#managed-superdav-service}

Superdav AI Agent v1.18.0 колдоого алынган сайттар үчүн башкарылуучу Superdav кызмат endpointтерин жана автоматтык туташуу даярдоону кошот. Сайтыңыз кол менен конфигурацияланган кызмат endpoint ордуна жайгаштырылган провайдерге туташышы керек болгондо бул башкарууларды колдонуңуз.

| Талаа | Сүрөттөмө |
|---|---|
| **Managed Superdav Service** | Колдоого алынган сайттар үчүн жайгаштырылган Superdav кызматынын туташуусун иштетет. |
| **Provision Connection** | Автоматтык endpoint жана credential даярдоону баштайт. Сайт башкарылуучу провайдерди колдонушу керек экенин ырастагандан кийин муну колдонуңуз. |
| **Service Endpoint / Connection Status** | Даярдоодон кийин учурдагы endpoint же туташуу абалын көрсөтөт. |

Даярдоодон кийин, жөндөөлөрдү сактап, башкарылуучу кызмат иш агымдарына таянуудан мурун туташуу абалын текшериңиз. Эгер даярдоо ишке ашпаса, көрсөтүлгөн кайра аракет кылуу боюнча көрсөтмөнү карап чыгып, сайттын жайгаштырылган провайдерди колдонууга уруксаты бар экенин ырастаңыз.

## Google Calendar конфигурациясы {#google-calendar-configuration}

Superdav AI Agent v1.18.0 календарь функциялары иштетилгенде, agent конфигурацияланган календарларды жана окуянын чоо-жайын окуй алат. Календарь куралдары окууга багытталган жана графикти эске алган эскертмелер, катышуучуларды кийинки көзөмөлдөө жана байланыштарды дал келтирүү үчүн пайдалуу.

| Талаа | Сүрөттөмө |
|---|---|
| **Google Calendar Credentials** | Календарь маалыматтарын окуу үчүн талап кылынган credentialдерди же token туташуусун сактайт. |
| **Calendar Selection** | Agent текшере ала турган конфигурацияланган календарларды чектейт. |
| **Calendar Connection Status** | Учурдагы credentialдер календарларды жана окуяларды окуй алар-албасын ырастайт. |

Календарь credentialдерин agentке керектүү календарлар менен гана чектеңиз. Эгер абал мөөнөтү бүткөн token көрсөтсө, credentialдерди кайра туташтырыңыз же алмаштырыңыз.

## TextBee SMS билдирүүлөрү {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 конфигурацияланган билдирүү иш агымдары үчүн SMS провайдери катары TextBee кошот. SMS билдирүүлөрү сезимтал же колдонуучуга багытталган билдирүүлөр үчүн адамдын жактыруу дарбазалары менен жупталышы керек.

| Талаа | Сүрөттөмө |
|---|---|
| **TextBee API Key** | TextBee SMS провайдерине сурамдарды аутентификациялайт. |
| **TextBee Device / Sender** | Провайдер талап кылганда, чыгуучу билдирүүлөр үчүн колдонулган TextBee жөнөтүүчүсүн же түзмөгүн тандайт. |
| **SMS Notifications Enabled** | Жактырылган иш агымдарына текст билдирүү билдирүүлөрүн жөнөтүүгө уруксат берет. SMS жөнөтүүлөрүн алдын алуу үчүн өчүрүлгөн бойдон калтырыңыз. |

Сыноо билдирүүсүн администраторго таандык номерге гана жөнөтүңүз, андан кийин пландалган же катышуучуларга багытталган эскертмелерди иштетүүдөн мурун жактыруу дарбазасынын жүрүм-турумун ырастаңыз.

## Функция желектери {#feature-flags}

Ошондой эле v1.9.0 ичинде киргизилген **Settings → Feature Flags** өтмөгү кошумча функционалдуулук үчүн которгучтарды берет. Ар бир желек тармак боюнча иштетилген же өчүрүлгөн болот; учурда ар бир сайт үчүн өзүнчө override жок.

### Функция желектерине кирүү {#accessing-feature-flags}

1. WordPress админ бөлүгүндө **Gratis AI Agent → Settings** бөлүмүнө өтүңүз.
2. **Feature Flags** өтмөгүн басыңыз.

### Кирүүнү башкаруу желектери {#access-control-flags}

| Белги | Баштапкы абалы | Сүрөттөмө |
|---|---|---|
| **Администраторлор менен чектөө** | Өчүк | Иштетилгенде, AI Agent чат панелин `administrator` ролу бар колдонуучулар гана ача алат. Башка бардык ролдор анын ордуна "Администраторуңузга кайрылыңыз" билдирүүсүн көрүшөт. |
| **Network Admins менен чектөө** | Өчүк | Multisite тармагында иштетилгенде, агентти Super Admins гана колдоно алат. Жеке сайт админдери бөгөттөлөт. Экөө тең иштетилсе, "Администраторлор менен чектөө" параметринен артыкчылыкка ээ болот. |
| **Subscriber кирүүсүнө уруксат берүү** | Өчүк | Иштетилгенде, `subscriber` ролу бар колдонуучулар чат интерфейсин колдоно алат, бирок окуу үчүн гана мүмкүнчүлүктөр менен чектелет (жазма түзүү же жөндөөлөрдү өзгөртүү жок). |
| **Мүчө эместер үчүн өчүрүү** | Өчүк | Ultimate Multisite мүчөлүк абалы менен интеграцияланат. Иштетилгенде, активдүү мүчөлүгү жок сайттар үчүн чат жашырылат. |

### Брендинг белгилери {#branding-flags}

| Белги | Баштапкы абалы | Сүрөттөмө |
|---|---|---|
| **"Powered by Gratis AI Agent" футерин жашыруу** | Өчүк | Чат виджетинин ылдый жагында көрсөтүлгөн брендинг атрибуция сабын алып салат. White-label жайылтуулар үчүн сунушталат. |
| **Ыңгайлаштырылган агент аты** | *(бош)* | Чат башындагы жана админ менюсундагы демейки "Gratis AI Agent" энбелгисин өзүңүздүн продукт атыңыз менен алмаштырат. Демейкини колдонуу үчүн бош калтырыңыз. |
| **Агент тандагычты жашыруу** | Өчүк | Иштетилгенде, колдонуучулар беш камтылган агенттин ортосунда которула албайт. Учурдагы агент Settings → General бөлүмүндө демейки катары кандай конфигурацияланган болсо, ошого бекитилет. |
| **Сайт иконун чат аватары катары колдонуу** | Өчүк | Чат виджетинин башындагы демейки AI иконун WordPress сайт иконu менен алмаштырат (Appearance → Customize → Site Identity астында коюлат). |

### Автоматташтыруу коопсуздук белгилери {#automation-safety-flags}

Superdav AI Agent v1.18.0 коопсузураак автоматташтыруу иштетүүлөрү үчүн адамдын жактыруу дарбазаларын жана эскертме жазууларын киргизет. Бул башкаруулар орнотулган пакетке жараша функция белгилеринде же кеңейтилген автоматташтыруу жөндөөлөрүндө пайда болушу мүмкүн.

| Белги | Баштапкы абалы | Сүрөттөмө |
|---|---|---|
| **Адамдын жактыруусун талап кылуу** | Сунушталат: күйүк | Ыйгарым укуктуу колдонуучу сунушталган аракетти карап чыгып тастыктаганга чейин сезимтал автоматташтырууларды тындырууда. |
| **Эскертмелерди кайталабоо** | Күйүк | Кайра аракеттер же пландалган иштетүүлөр кайталанган билдирмелерди жөнөтпөшү үчүн жөнөтүлгөн эскертмелерди жазып турат. |
| **Календар куралдарын иштетүү** | Конфигурацияланганга чейин өчүк | Агентке конфигурацияланган Google календарларын жана окуяларын окууга уруксат берет. |
| **SMS билдирмелерин иштетүү** | Конфигурацияланганга чейин өчүк | Каттоо маалыматтары сакталгандан кийин жактырылган иш агымдарына TextBee SMS билдирмелерин жөнөтүүгө уруксат берет. |

### Өзгөртүүлөрдү колдонуу {#applying-changes}

Каалаган белгини которгондон кийин **Save Settings** басыңыз. Өзгөртүүлөр дароо күчүнө кирет — кэшти тазалоо же pluginди кайра активдештирүү талап кылынбайт.
