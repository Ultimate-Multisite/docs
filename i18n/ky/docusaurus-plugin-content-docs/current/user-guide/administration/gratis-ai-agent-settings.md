# Бесплатные настройки AI Агента

В экране **Настройки → Расширенные** в Gratis AI Agent вы можете настроить интеграции на уровне администратора, которые были добавлены в версии v1.5.0. Эта страница описывает поля **Feedback Endpoint** и их ожидаемый формат.

## Доступ к настройкам

1. В админке WordPress перейдите в **Gratis AI Agent → Настройки**.
2. Нажмите на вкладку **Расширенные (Advanced)**.

## Настройка Feedback Endpoint

Endpoint для обратной связи принимает POST-запросы от AI агента всякий раз, когда пользователь отправляет отзыв через кнопку "палец вниз", баннер с автоподсказкой или команду `/report-issue`.

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** | URL, который принимает отзывы в виде HTTP POST запросов с JSON телом. |
| **Feedback API Key** | Токен типа bearer, отправляемый в заголовке `Authorization` каждого запроса на отзыв. Оставьте пустым, если ваш endpoint не требует аутентификации. |

### Ожидаемая JSON-нагрузка (Payload)

Ваш endpoint для обратной связи должен принимать JSON тело с как минимум следующими полями:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Ответ был неверным относительно цен.",
  "triage_category": "factual_error"
}
```

В зависимости от контекста диалога в нагрузку могут присутствовать дополнительные поля.

### Значения `triage_category`

Слой сортировки (triage layer) AI присваивает одно из следующих значений полю `triage_category` перед пересылкой нагрузки:

| Значение | Смысл |
|---|---|
| `factual_error` | Ассистент предоставил неверную фактическую информацию. |
| `unhelpful_answer` | Ответ был технически верным, но бесполезным. |
| `inappropriate_content` | Ответ содержал контент, который не должен показываться пользователям. |
| `other` | Отзыв не соответствует известной категории. |

### Аутентификация

Эгер сиздин конекторы аутентификация талап кылса, **Feedback API Key** өрісүнө өзүңүздүн bearer токенинди коюңуз. Агент төмөнкүлөрдү жөнөтөт:

```
Authorization: Bearer <your-api-key>
```

Эгерде **Feedback API Key** өріси бош болсо, `Authorization` башети жөнөтүлбөйт.

### Feedback Коллекциясын Өчүрүү

**Feedback Endpoint URL** жана **Feedback API Key** өрісүн тең бош калтырыңыз. "Thumbs-down" баскычы жана обратная связь интерфейси колдонуучуларга көрүнүп турат, бирок киргизүүлөр эч кандай тышкы кызматка жөнөтүлбөйт.

## Brave Search API Key

Ошондой эле **Advanced** табасында **Brave Search API Key** өріси интернет издөө мүмкүнчүлүгүн [Internet Search](../configuration/internet-search) аркылуу иштетүүгө мүмкүндүк берет.

| Өріс | Сүрөттөмө |
|---|---|
| **Brave Search API Key** | Brave Search өнүктүрүүчү панелиндеги API ключуңуз. AI ассистентте интернет издөөгө активдештирүү үчүн керек. |

Өріс аталышында Brave Search API каттоо баракчасына басып кире турган шилтеме камтылган. Интернет издөөнү өчүрүү үчүн бош калтырыңыз.

Бул функция жөнүндө акыркы колдонуучу документациясы үчүн [Internet Search](../configuration/internet-search) сыяктуу жерден көрө аласыз.

## Feature Flags (Функция белгилери)

v1.9.0-дан бери киргизилген **Settings → Feature Flags** табасы милдеттүү эмес функциялар үчүн туташтыруучу туташтыруучуларды (toggle switches) камтыйт. Ар бир флаг толук активдүү же өчүрүлгөн болот; учурда сайт боюнча өзүнчө өзгөртүү мүмкүнчүлүгү жок.

### Feature Flags-ка Кирүү

1. WordPress администраторду ачыңыз, андан кийин **Gratis AI Agent → Settings**-ке барыңыз.
2. **Feature Flags** табасына басыңыз.

### Access Control Flags (Кирүү Контрол Флагдары)

| Flag | Default | Сүрөттөмө |
|---|---|---|
| **Администраторлорго чектеу** | Апсырылган эмес (Off) | Бул опция күчтөлсө, `administrator` ролу бар колдонуучулар гана AI Agent чат панелин ача ала алат. Башка ролдор "администраторуңуз менен байланышыңыз" деген билдирүүнү көрөт. |
| **Сеть администраторлоруна чектеу** | Апсырылган эмес (Off) | Мультисайт сетке күчтөлсө, бул опция активдүү болсо, гана Супер Администраторлор агентти колдоно алат. Ар бир сайттын администраторлору багынган болот. Эгерде эки опция да күчтөлсө, бул "Администраторлорго чектеу" дегенден алдында иштейт. |
| **Чоңдордун кирүүсүн мүмкүн кылуу** | Апсырылган эмес (Off) | Бул опция активдүү болсо, `subscriber` ролу бар колдонуучулар чат интерфейсин колдоно алат, бирок алар окуучулук режимге чектелет (пост жазуу же параметрлерди өзгөртүү мүмкүнчүлүгү жок). |
| **Мемлекеттик эмес адамдар үчүн өчүрүү** | Апсырылган эмес (Off) | Бул опция Ultimate Multisite мүчөлүк статусуна интеграцияланат. Активдүү мүчөттөр жок сайттар үчүн чат жашырылат. |

### Бренддинг флагы

| Flag | Default | Сүрөттөмө |
|---|---|---|
| **"Gratis AI Agent" футерди жабуу** | Апсырылган эмес (Off) | Чат виджетинин төмөнкү бөлүгүндө көрсөтүлгөн бренддинг белгисин алып салат. Билдинг-лейблдери үчүн сунушталган. |
| **Сүрөттүк агент аты** | *(бошотулган)* | Чат башталышындагы жана администратор менюсундагы стандарттык "Gratis AI Agent" белгиси ордуна өзүнүн продукцияңыздын атын коюңуз. Бошотуу үчүн бош калтырыңыз. |
| **Агент тандагычты жабуу** | Апсырылган эмес (Off) | Бул опция активдүү болсо, колдонуучулар беш киргизүү агентлеринин ортосунда алмаша албайт. Азыркы агент Settings → General дайда белгиленген дефолт катары коюлган агент менен бекитилген болот. |
| **Сайт иконкасын чат аватар катары колдонуу** | Апсырылган эмес (Off) | Чат виджетинин башталышындагы стандарттык AI иконкасын WordPress сайтынын иконкасы менен алмаштырат (Appearance → Customize → Site Identity да белгиленет). |

### Өзгөртүүлөрдү колдонуу

Кайсы флактарды өзгөртсө, **Save Settings**-ти басыңыз. Өзгөртүүлөр дароо күч алган — кэш тазалоо же плагинини кайра иштетүү зарыл эмес.
