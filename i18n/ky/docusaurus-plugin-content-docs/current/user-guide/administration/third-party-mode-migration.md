# Үчтүүчү режимге өтүү (Third-Party Mode Migration) {#third-party-mode-migration}

Superdav AI Agent v1.12.0 үчүн үчүнчү тараптык жөндөмөлөр кантип иштешин өзгөрттү. **Үчтүүчү режим азыр автоматтык түрдө иштешет**, бул WordPress 7.0+ версияларында колдонуучунун колунан конфигурациялоосуз, натив WordPress Abilities API интеграциясын ишке киргизет.

## Эмне өзгөрдү? {#what-changed}

### v1.12.0ден мурун {#before-v1120}

Үчтүүчү жөндөмөлөр мануалдык конфигурация талап кылган:

- "Third-party mode" (Үчтүүчү режимди) ачык көрсөтүү керек эле
- Жөндөмөлөр кастомдык реестрден жүктөлгөнү
- WordPress Abilities API менен интеграциялоо милдеттүү эмес эле
- Эски режим (Legacy mode) дефолт катары болгон

### v1.12.0дон кийин {#after-v1120}

Үчтүүчү жөндөмөлөр автоматтык түрдө иштешет:

- Үчтүүчү режим "auto" (автоматтык) деп дефолтталган
- Жөндөмөлөр WordPress Abilities API менен натив интеграцияланат
- WordPress 7.0+ версияларында мануал конфигурациялоого болбойт
- Эски режим мурунку WordPress версиялары үчүн да колдонууга мүмкүн

## Кимге таасир болот? {#who-is-affected}

### Жаңы орнотуулар (WordPress 7.0+) {#new-installations-wordpress-70}

**Эч нерсе кылууга болбойт.** Үчтүүчү режим автоматтык түрдө "auto" деп коюлат, жана жөндөмөлөр даяр hali менен иштеп начать болот.

### Маанилүү орнотуулар (Existing Installations) {#existing-installations}

**Сиздин 설정уларыңыз сакталат.** Эгер сиз төмөнкүлөрдү колдонуп жатсаңыз:

- **Legacy mode (Эски режим)**: Сиз эски режимде каласыз (эч нерсе өзгөрбөйт)
- **Manual third-party mode (Мануалдык үчүнчү тараптык режим)**: Сиз мануалдык режимде каласыз (эч нерсе өзгөрбөйт)
- **Auto mode (Автоматтык режим)**: Сиз автоматтык режимде улантасыз (эч нерсе өзгөрбөйт)

### 7.0ден мурунку WordPress версиялары {#wordpress-versions-before-70}

**Эски режим колдонууга мүмкүн.** Эгер сиз WordPress 6.x же андан эски версиясында болсоңуз:

- Үчтүүчү режим "legacy" (эски) деп дефолтталган
- Кааласаңыз, үчүнчү тараптык режимин мануалдык түрдө иштетирсеңиз болот
- Натив Abilities API-ден пайдалануу үчүн WordPress 7.0+ версиясына 업그레이д кылыңыз

## Режимдерди түшүнүү {#understanding-the-modes}

### Auto Mode (Жаңы дефолт) {#auto-mode-new-default}

**Auto mode** натив WordPress Abilities API интеграциясын колдонот:

- Жөндөмөлөр WordPress hooks аркылуу катталат
- WordPress 7.0+ Abilities API менен толук дал келүү
- Үчтүүчү жөндөмөлөрдү автоматтык түрдө табуу
- Мануал конфигурациялоого болбойт

**Качан колдонуу керек**: WordPress 7.0+ жана үчүнчү тараптын мүмкүнчүлүктөрүн колдонуу

### Руналдуу режим (Manual Mode) {#manual-mode}

**Руналдуу режимди** ачык конфигурациялоо талап кылат:

- Кайсы үчүнчү тараптын мүмкүнчүлүктөрүн жүктөө керектигин көрсөтүңүз
- Тест жүргүзүү же белгилүү бир мүмкүнчүлүктөрдү тандап жүктөө үчүн пайдалуу
- Конфигурация файлда редакциялоо талап кылынат
- Көбүрөөк контроль, бирок көбүрөөк даярдоо керек

**Качан колдонуу керек**: Тест жүргүзүү, белгилүү бир мүмкүнчүлүктөрдү тандап жүктөө же ылайыкташтыруулар үчүн

### Эски режим (Legacy Mode) {#legacy-mode}

**Эски режим** эски үчүнчү тараптын мүмкүнчүлүк системасын колдонуur:

- Өздей катышуучулук реестри (WordPress Abilities API эмес)
- Эски WordPress версиялары менен так туура иштейт
- Негизги WordPress интеграциясы жок
- Учурдулган, бирок азыр да колдонулуп жатат

**Качан колдонуу керек**: WordPress 6.x же андан мурунку версияларда, же эски тууралуу ылайыкташтыруу керек болгон учурда

## Азыркы режимин текшерүү {#checking-your-current-mode}

### Администратор панелин аркылуу {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings**-ке кириңиз
2. **Third-Party Mode** (Третья сторона режими) орнотусун карап көрүңүз
3. Азыркы режимин жана аны өзгөртүү үчүн опцияларды көрөсүз

### Код аркылуу {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', же 'legacy'
```

## Режимин өзгөртүү {#changing-your-mode}

### Авто режимге өтүү (Switch to Auto Mode) {#switch-to-auto-mode}

Эгер сиз WordPress 7.0+ версиясында болсоңуз жана авто режимин колдонууну кааласаңыз:

1. **Superdav AI Agent** → **Settings**-ке кириңиз
2. **Third-Party Mode**-ду табасыз
3. **Auto (WordPress Abilities API)**-ды тандаңыз
4. **Save**-га басыңыз

Superdav AI Agent үчүнчү тараптын мүмкүнчүлүктөрүн автоматтык түрдө ачып, каттоодон өтүрөт.

### Руналдуу режимге өтүү (Switch to Manual Mode) {#switch-to-manual-mode}

Эгер сиз кайсы мүмкүнчүлүктөрдү жүктөө керектигин колдону менен башкарууну кааласаңыз:

1. **Superdav AI Agent** → **Settings**-ке кириңиз
2. **Third-Party Mode**-ду табасыз
3. **Manual**-ды тандаңыз
4. **Save**-га басыңыз
5. Кайсы мүмкүнчүлүктөрдү жүктөө керектигин көрсөтүү үчүн конфигурация файлын редакциялаңыз

### Эски режимге өтүү (Switch to Legacy Mode) {#switch-to-legacy-mode}

Эгер сиз эски тууралуу ылайыкташтырууну талап кылсаңыз:

1. **Superdav AI Agent**-ке барыңыз → **Settings** (Жаңырылуулар)
2. **Third-Party Mode** (Үч-үчүнчү режимди) тандаңыз
3. **Legacy** (Эски режимди) тандаңыз
4. **Save** (Сақтау) баскычын басыңыз

## Auto Mode-дун артыкчылыктары {#benefits-of-auto-mode}

### Автоматтык ачылуу {#automatic-discovery}

Иш-аракеттер автоматтык түрдө төмөнкүлөрдөн табылат:

- Орнолгон плагиндер
- Иш жүрүп жаткан тема (theme)
- Милдеттүү плагиндер
- Drop-in плагиндер

Кол менен каттоодон өтүүнүн керекси жок.

### Биринчи интеграция {#native-integration}

Иш-аракеттер WordPress Abilities API менен интеграцияланат:

- WordPress негизги функцияларына ылайыктуу
- WordPress администратору менен иштөөгө мүмкүнчүлүк берет
- Abilities API колдонуп башка плагиндер менен дал келет
- WordPress өнүгүп жаткан учурда келечектине даяр

### Жөнөкөйлөтүлгөн башкаруу {#simplified-management}

- Редакциялоо үчүн конфигурация файлдары жок
- Кол менен иш-аракеттерди каттоодон өтүүнүн керекси жок
- Ability Visibility (Иш-аракеттин көрүнүшү) контролдору автоматтык түрдө иштешет
- Администратордун эскертүүлөрү сизге классификацияланбаган иш-аракеттер жөнүндө маалымат берет

### Жакшыраак аткарылыш {#better-performance}

- Иш-аракеттер 캐шталган (cached) болот
- Маанилүү болгондо кечиктирилген түрүндө жүгүртүлөт (Lazy-loaded on demand)
- WordPress 7.0+ үчүн оптималдаштырылган

## Миграция жолу {#migration-path}

### Эгер сиз WordPress 6.x де болсоңуз {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ версиясына көтөрүү** (даяр болгондо)
2. **Superdav AI Agent-ти v1.12.0+ версиясына жаңыртуу**
3. **Үч-үчүнчү режимди Auto'го өзгөртүү** (опционалдуу; эски режим да иштеп турат)
4. Туура кирүү контролдорун камсыз кылуу үчүн **ability visibility** (иш-аракеттин көрүнүшүн) карап чыгыңыз

### Эгер сиз WordPress 7.0+ де болсоңуз {#if-youre-on-wordpress-70}

1. **Superdav AI Agent-ти v1.12.0+ версиясына жаңыртуу**
2. **Үч-үчүнчү режимдин Auto'го коюлганын текшериңиз** (дефолт катары ошондой болушу керек)
3. Туура кирүү контролдорун камсыз кылуу үчүн **ability visibility** (иш-аракеттин көрүнүшүн) карап чыгыңыз
4. Алар иштеп жатканын тастыктоо үчүн **third-party abilities** (үч-үчүнчү иш-аракеттерди) сынап көргөңүз

## Ар кандай маселелерди чечүү {#troubleshooting}

### Auto mode'до аткарылыш жүгүртүлбөйт {#abilities-arent-loading-in-auto-mode}

- Сиз WordPress 7.0+ де болгонуңузду текшериңиз
- Үч-үчүнчү режимдин "Auto" деп коюлганын текшериңиз
- Иш-аракетти берген плагиндер активдүү экенин текшериңиз
- Каттоо каталары үчүн WordPress ката журналдарын (error logs) текшериңиз

### Эски режимди сактап калгым келет {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** (Жаңырылуулар) барыңыз
- **Legacy** (Эски режимди) тандаңыз
- **Save** (Сақтау) баскычын басыңыз
- Эски режим да иштеп турубу.

### Маганы кастомдык жөндөмөлөрүм көрсөтүлбөйт элек {#my-custom-abilities-arent-showing}

- Алар WordPress хуктары аркылуу катталганын текшериңиз
- Алар Abilities API'sini туура ишке ашырганын текшериңиз
- WordPress ката журналдарын карап чыгыңыз
- Бардык катталган жөндөмөлөрдү көрүү үчүн **Ability Visibility** административ саадалыгын колдонуңуз

### "Классификациясыз жөндөмөлөр" жөнүндө маалыматтар келет {#im-getting-unclassified-ability-notices}

- Бул жаңы үчүнчү тараптын жөндөмөлөрү үчүн нормалдуу нерсе
- Аларды административ нотада карап чыгып, классификациялаңыз
- Классификация боюнча толук маалымат алуу үчүн **Ability Visibility** саадалыгын көрүңүз

## Артылып келүүгө ыңгайлуулук (Backward Compatibility) {#backward-compatibility}

### Маанилүү конфигурациялар {#existing-configurations}

Эгер сизде бар жөндөмөлөрүнүн конфигурациялары болсо:

- **Legacy mode (Эски режим)**: Сиздин конфигурацияңыз иштөө дагы уланууда
- **Manual mode (Кол менен режим)**: Сиздин конфигурацияңыз иштөө дагы уланууда
- **Auto mode (Авто режим)**: Сиздин конфигурацияңыз унутулат (авто режим алдында ага өтөт)

Кастомдык конфигурацияңызды сактап калуу үчүн, Manual же Legacy модада болуңуз.

### Жалгашуу графиги (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: Legacy жана Manual режим толук колдонулуп жатат
- **v1.13.0+**: Legacy режим убадалуу маалымдамаларды көрсөтүшү мүмкүн
- **v2.0.0**: Legacy режим алынышы мүмкүн (Текшерүүдө)

## Ички иш-чаралар (Best Practices) {#best-practices}

### Жаңы орнотуулар үчүн {#for-new-installations}

- Auto mode'ду колдонуңуз (бул стандарттык режим)
- Superdav AI Agent'тин жөндөмөлөрдү автоматтык түрдө ачып алганга уруксат бериңиз
- Кирүү мүмкүнчүлүктөрүн көзөмөлдөө үчүн Ability Visibility'ди колдонуңуз

### Учурда бар орнотуулар үчүн {#for-existing-installations}

- Мүмкүн болсо WordPress 7.0+ версиясына 업그레이д кылыңыз
- Жөнгө салууну жөнөкөйлөтүү үчүн Auto mode'го өтүңүз
- Ability Visibility аркылуу жөндөмөлөрдү карап чыгып, классификациялаңыз

### Кастомдык жөндөмөлөр үчүн {#for-custom-abilities}

- WordPress хуктары аркылуу жөндөмөлөрдү каттаңыз (Abilities API)
- Кастомдык жөндөмөлөр реестрлерин колдонуудан сактаңыз
- Auto mode менен WordPress 7.0+ версиясында тест кылгыла

1. **WordPress версияңызды текшериңиз**: Авто режим үчүн 7.0+ версиюду колдонуп жатканыңызды текшериңиз.
2. **Үч-үч-бучу режимин карап чыгыңыз**: Settings (Жөөлөр) бөлүмүнө кирип, учурдагы режимин текшериңиз.
3. **Зарыл болсо жаңыртыңыз**: Эгер сиз WordPress 7.0+ колдонсоңуз, Auto режимге өтүңүз.
4. **Иш-аракеттерди классификациялаңыз**: Классификациясы жок мүмкүнчүлүктөрдү карап чыгып, аларды классификациялаңыз.
5. **Тестирлөө**: Үч-үч-бучу (third-party) мүмкүнчүлүктөрү туура иштеп жатканын текшериңиз.

## Байланыш темалар {#next-steps}

- **Иш-аракеттердин көрүнүшү (Ability Visibility)**: Кайсы иш-аракеттерди кайсы жерде көрсөтүү керектигин контролдоо
- **WordPress Abilities API**: WordPressтин өзүнүн мүмкүнчүлүктөрүн кантип каттоо жөнүндө билгиле
- **Үч-үч-бучу мүмкүнчүлүктөрдү иштеп чыгуу (Third-Party Ability Development)**: Auto режим менен иштөөчү мүмкүнчүлүктөрдү түзүү
