# Кардар маалымат жана көйгөйлөрдү билдирүү

Gratis AI Agent v1.5.0 жаңысы колдонуучуларга чат интерфейси аркылуу жардам бербеген жоопторду белгилөө жана көйгөйлөрдү түздөн-түз билдирүүгө мүмкүндүк берген киргизилген кайталоо системасын сунуштайт. Бул системада макулдукты башкаруу, автоматтык отчет берүү буйругу жана артында AI жардамы менен тартипке келтирүү (triage) камтылган.

## Баа жок баскыч (Thumbs-Down Button) {#thumbs-down-button}

AI ассистент тарабынан жөнөтүлгөн ар бир билдирүү **баа жок** (👎) баскычын көрсөтөт. Колдонуучу аны бастырса, жоопту жардам бербеген же ката деп белгилөөгө болот.

- Баскыч ар бир ассистент билдиришинин жанында курсор коюлганда пайда болот.
- Аны бастыруу **Байланыш макулдугу модалына** (Feedback Consent Modal) алып барат.
- Билимдер диалог тармагына жана атайын билдирүүгө тиешелүү болот.

## Байланыш макулдугу модалы (Feedback Consent Modal) {#feedback-consent-modal}

Колдонуучу баа жок баскычын бастырса, кандайдыр бир маалымат жөнөтүлбөй алдында макулдук модалы пайда болот. Модал:

- Кайсы маалыматтар чогулуп алынарын түшүндүрөт (диалогтун үзүндүсү, браузер контексти).
- Колдонуучудан бул маалыматтарды бөлүшүүгө макул экенин тастыктоо сурайт.
- Колдонуучунун ката эмне болгонун сүрөттөө үчүн сырткы текст талаасын берет.
- Жөнөтбөй жабуу үчүн **Жабуу** (Cancel) опциясын сунуштайт.

Колдонуучу ачык тастыктаганча эч кандай маалымат катталбайт.

## Автоматтык билдирүү баннери (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Диалогдун аягында ассистент **автоматтык билдирүү баннерин** көрсөтүшү мүмкүн — бул сессия жардам берди деп сураган кыймылсыз билдирүү.

Бул баннер диалогтун узундугуна жана натыйжасына карата алгоритмдердин негизинде автоматтык түрдө пайда болот. Ал баа жок баскыч менен бирдей макулдук агымына шилтеме берет. Колдонуучу билдирүү бербестен баннери жабуп коюшу мүмкүн.

## /report-issue Слэш буйругу (Slash Command) {#report-issue-slash-command}

Колдонуучу чат киргизүүгө `/report-issue` деп жазып, кайталоо агымын түз эле ишке киргизе алат. Буйрук:

- Каналды дароогуп "Feedback Consent Modal" (Маалымдашуу модали) ачылатпай.
- Сүрөттөмөнү учурдагы сүйлөшүүлүш контексти менен толуктап коюуну сунуштайт.
- Аталыштыруудан мурун колдонуучуларга кошумча деталдарды киргизүүгө мүмкүндүк берет.

`/report-issue` буйругу бардык чат режимдеринде (inline, floating widget, full-screen) колдонууга мүмкүн.

## AI-Assisted Triage (AI аркылуу тартиптөө) {#ai-assisted-triage}

Берилген пикирлер AI тартиптөө каты (triage layer) аркылуу өтүп барат жана ал:

- Кызматчылыкты (факты ката, жардам бербеген жооп, тиешелүү эмес мазмун ж.б.) аныктайт.
- Тиешелүү сүйлөшүү метадеталарын (metadata) кошуп берет.
- Тартиптөө сунуштамасын конфигурацияланган **Feedback Endpoint** (Settings > Advanced бөлүмүнө караңыз) аркылуу жөнөтөт.

Бул колдонуучулардын маанилүү көйгөйлөрүн биринчиден көрүү аркылуу колдоо үчүн колдонулган колдонуунун убактысын азайтат.

## Settings > Advanced {#settings-advanced} (Сотустар > Артыраак) {#settings--advanced}

Пикирлер артындагы бэкендик (feedback backend) иштетүү үчүн, **Gratis AI Agent → Settings → Advanced** бөлүмүндө төмөнкү талааларды конфигурациялаңыз:

| Тала | Сүрөттөмөсү |
|---|---|
| **Feedback Endpoint URL** | Пикир мазмундук (JSON) POST сурагын алган URL. |
| **Feedback API Key** | Пикирди жөнөтүүдө идентификациялоо үчүн `Authorization` башталышында жөнөтүлгөн Bearer токени. |

Пикир чогултууну өчүрүү үчүн эки талааны да бош калтырыңыз. Индивидуалдык пикир баскычтары колдонуучуларга көрүнүп турушса да, алар жөнөтүлбөйт.

:::tip
Пикир конекти (endpoint) `message_id`, `conversation_id`, `feedback_text` жана `triage_category` талаалары бар JSON денесин кабыл алышы керек. Күтүлгөн схема үчүн конектөөлөргө карата документтерди издеңиз.
:::
