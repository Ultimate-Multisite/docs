# Hostinger Multi-Tenancy {#hostinger-multi-tenancy}

Ultimate Multisite: Multi-Tenancy 1.2.0 дода калтыр (Multi-Tenancy) функциясын кошуп турат, бул хостун домендерди бардык өзгөчөлүү арендачылар (tenant) менен бирге башкарууга мүмкүндүк берет, ошондой эле мурунку Hostinger домен картасы интеграциясы менен бирге.

Бул мүмкүнчүлүктү Hostinger hPanel аркылуу арендачы домендөрү жана атайын изоляцияланган арендачы инфраструктурасы башкарылганда колдонуңуз.

## Орнотуу боюнча эскертүүлөр {#setup-notes}

1. Негизги Hostinger интеграциясын **Ultimate Multisite > Settings > Host Integrations** аркылуу конфигурациялаңыз.
2. Hostinger API токенине максат домендүн же поддомендин башкарыла ала турганын текшериңиз.
3. Multi-Tenancy addonду активдештириңиз.
4. Арендачыны жарыялогондон мурун арендачы изоляция стратегиясын конфигурациялаңыз.
5. Продуктивдик трафикти арендачыга жөнөтүүдөн мурун миграцияны текшерүү workflow'уну аткарыңыз.

Hostinger мүмкүнчүлүгү хосту тарап операциялары үчүн бөлүшүлгөн Hostinger байланышын колдонот. DNS дагы туура Hostinger эсебине багытталышы керек, ошондой эле hPanel эсеби чектөөлөрү колдонулат.

## Мүмкүнөйдө келип чыккан көйгөйлөрдү чечүү (Troubleshooting) {#capability-specific-changes}

- Эгер арендачын орнотуу ишке ашпаса, домендүн Hostinger эсебине алдын ала тиешелүү экенин текшериңиз.
- Эгерде базалык маалыматты текшерүү ишке ашпаса, арендачынын DB колдонуучусу, базасынын аты жана хост байланышын hPanel менен салыштырыңыз.
- Эгер **Visit (SSO)** ишке ашпаса, домендүн DNS жана SSL активдүү экенин текшериңиз.
- Эгерде деструкциядан кийин хосту ресурстары калат калса, резервдик көчүрмөлөрдү текшергенден кийин hPanelден калган бардык базаларды, колдонуучуларды же папкаларды алып салыңыз.
