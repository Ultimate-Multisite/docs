# Домен Салду: Настройка и Конфигурация Провайдера

Дополнение Domain Seller поставляется с управляющим мастером настройки, который проведет вас через каждый необходимый шаг. Эта страница описывает процесс работы мастера и то, как настроить или перенастроить провайдеров после этого.

## Требования

- **Multisite Ultimate** v2.4.12 или выше, активированный по сети
- **PHP** 7.4+
- API учетные данные для хотя бы одного поддерживаемого регистратора

## Мастер настройки при первом запуске

Мастер настройки автоматически запускается в первый раз, когда вы активируете плагин по сети. Он также доступен в любое время через **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Шаг 1 — Выберите провайдера

Выберите регистратора, с которым вы хотите подключиться. Поддерживаемые варианты:

| Провайдер | Управление DNS | Конфиденциальность WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Нет | Да (WhoisGuard, бесплатно) |
| HostAfrica | Да | Да (Защита ID) |
| Openprovider | Да | Да |
| Hostinger | Через основное сопоставление домена Hostinger для хостинговых доменов | Да |
| GoDaddy | Нет | Нет |
| ResellerClub | Да | Нет |
| NameSilo | Нет | Нет |
| Enom | Да | Нет |

### Шаг 2 — Введите учетные данные

У каждого провайдера есть свои поля для ввода данных:

**OpenSRS** — Имя пользователя и приватный ключ (из Панели управления реселлером OpenSRS)

**Namecheap** — Имя пользователя и API-ключ (из Account › Tools › API Access)

**HostAfrica** — Конечная точка API реселлера доменов и учетные данные от модуля реселлера HostAfrica. Отдельная песочница пока не документирована; тестируйте безопасными операциями только на чтение перед запуском реальных регистраций.

**Openprovider** — Имя пользователя и пароль с включенным доступом к API. Опциональный режим песочницы использует sandbox API Openprovider, а опциональное имя клиента по умолчанию можно использовать повторно для регистраций.

**Hostinger** — Негизиられた Hostinger hPanel API токени, бул негизги Hostinger интеграциясынан алынган. Ошол токен негизги домен картасы жана Домен Саткийи каттоо операциялары үчүн колдонулат.

**GoDaddy** — API ключi жана сыр (developer.godaddy.com-дан)

**ResellerClub** — Reseller ID жана API ключу (ResellerClub контрол панеленден алынган)

**NameSilo** — API ключу (namesilo.com › Account › API Manager-дан)

**Enom** — Аккаунт ID жана API токени

Жазылууга чейин, мүмкүн болгон учурда **Sandbox mode**-ду текшериңиз, бул сизди живы режимге өтө алдында жеткирүүчүлөр тестотолук чөйрөсүнө карап сынап көрүүгө мүмкүндүк берет.

### 3-Степ — Байланышты текшерүү

**Test Connection**-ды басыңыз. Уюлдук (wizard) сертификаттарды жана байланышты текшерүү үчүн жеңил API чакырыгын жөнөтөт. Ушундан мурун бардык сертификат маселелерин чечип алыңыз.

### 4-Степ — TLD'лерди импорттоо

Байланышты болгон живыдан бардык колдонулуучу TLD'лерди жана топторунун баасын импорттоо үчүн **Import TLDs**-ды басыңыз. Бул домендук продукттарда колдонулган TLD тизмесин толтурат. Ички импорт 30–60 секундга чейин созулушу мүмкүн, чоң TLD каталогу бар живылар үчүн.

TLD'лер күнүнө бир жолу белгиленген cron жумушу аркылуу автоматтык түрдө кайрасинхрондалат.

### 5-Степ — Домен продуктусун түзүү

Уюлдук 10% маржа менен стандарттык "catch-all" домен продуктусун түзөт. Сиз бул продуктту дароо редакцияласаңыз болот же **Ultimate Multisite › Products** астында колдонулган продукттарды колдонуу аркылуу өтүп, мануалдуу түрдө продукт түзө аласыз.

Толук продукт конфигурациясы боюнча колдонмону [Domain Products and Pricing](./domain-products) караңыз.

## Живыны кайра конфигуралоо

**Network Admin › Ultimate Multisite › Settings › Domain Seller** (же плагин тизмесинде **Settings**-ке басыңыз).

Бул параметрлерди толтурат:

- **Домен сатууну активдештирүү** — бул функцияны толугу менен өчүрүү/баштап берүү (on/off) ретчеңиз

- **Стандарттык жеткирүүчү (Default provider)** — домен издөөлөр жана жаңы продуктулар үчүн колдонулган жеткирүүчү.

- **Ирделген TLD саны** — кардар издөсө канча TLD'ди текшеребилет; жогорку баалуу көрсөткүчтөр көбүрөөк варианттарды көрсөтөт, бирок алар аздап баяжууlu болот.

- **Кайрылуучу 캐ш (Availability cache) узактыгы** — кайрылуу жана баа көрсөткүчтөрүн канчалык узакта сакташы керек; төмөнкү баалуу көрсөткүчтөр тактырар, бирок API чакыруулары көбөйөт.

- **Домен продуктуларын башкаруу** — Продуктар тизмесине тез кирип келүүчү шилтеме.

- **Жеткирүүчүлөрдү конфигурациялоо** — Жеткирүүчүлөрдү кошуу же аларды кайра конфигурациялоо үчүн Интеграция wizards'ын ачат.

### Экинчи жеткирүүчүн кошуу

**Configure providers**-ды басып, жаңы реестр үчүн wizard'ды кайра иштетиңиз. Силер бир эле учурда бир нече жеткирүүчүлөрдү конфигурациялашыңыз мүмкүн. Ар бир домен продуктуну белгилүү бир жеткирүүчүнө тандаңыз же стандарттык катары калтырыңыз.

### TLD'дерин колдонуп синхролоо

Жеткирүүчүлөрдүн ар бири өзүнүн лог каналына жазып турат. Логдорду **Network Admin › Ultimate Multisite › Logs** (Сеть администратору › Ultimate Multisite › Логдор) астында көрө аласыз:

| Лог канал | Содержимое |
|---|---|
| `domain-seller-registration` | Все попытки регистрации (успешные и неудачные) |
| `domain-seller-renewal` | Результаты задания на продление |
| `domain-seller-opensrs` | Сырая активность API OpenSRS |
| `domain-seller-namecheap` | Сырая активность API Namecheap |
| `domain-seller-hostafrica` | Сырая активность API HostAfrica |
| `domain-seller-openprovider` | Сырая активность API Openprovider |
| `domain-seller-hostinger` | Сырая активность API Hostinger |
| `domain-seller-godaddy` | Сырая активность API GoDaddy |
| `domain-seller-resellerclub` | Сырая активность API ResellerClub |
| `domain-seller-namesilo` | Сырая активность API NameSilo |
| `domain-seller-enom` | Сырая активность API Enom |

## Продвигоочулдугунун мүмкүнчүлүктөрү жөнүндө маалымдамалар

Ар бир регистратордун API'си бирдей операцияларды көрсөтпөйт. Аддон (addon) колдоого болбогон операцияларды тияксыз иштетпей, алардын ордуна так администраторго көрүнүшү мүмкүн болгон каталар менен көрсөтөт.

- **HostAfrica** кеңири жигердүү кайра сатуу (reseller) иш алдынгусун колдойт: издөө, TLD/баасы синхролоо, каттоо, продление, которуу, имендер серверлерине маселелерди берүү, DNS каттары, EPP коддору, регистратордун бөгөттөлүшү жана ID коргоо.
- **Openprovider** кайра сатуу баасындагы TLD синхролоо, каттоо, продление, которуулар, имендер серверлерине маселелерди берүү, DNS зоналары, EPP коддору, регистратордун бөгөттөлүшү жана WHOIS жашыруундугун колдойт. Ал кыска мөөнөтү өтүп кетирген bearer токени менен иштешет, аны аддон автоматтык түрдө жаңылайт.
- **Hostinger** бирге бөлүштүрүлгөн hPanel API токени аркылуу жеткиликтүүлүк издөө, каттоо, портфолио издөө, имендер серверлерине маселелерди берүү, регистратордун бөгөттөлүшү жана WHOIS жашыруундугун колдойт. Hostinger'дин публик Домендер API-си кайра сатуучу/опто баасын, киргизүү которуусун, ачык продлениесин же EPP коду алууну көрсөтпөйт; кайралар автоматтык түрдө жаңыланышы мүмкүн.
