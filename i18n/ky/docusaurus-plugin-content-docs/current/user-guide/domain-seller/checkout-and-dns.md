# Checkout Field and Customer DNS Менемени

## Домен тандалы селекциясы checkout талаасы {#the-domain-selection-checkout-field}

**Domain Selection** (Домен тандалышы) талаасы — бул а客様ларга сайттын домендүн кантип алынары жөнүндө тандоо мүмкүнчүлүгю берет. Домен сатууну ишке киргизүү үчүн аны кез келген checkout формасына кошуңуз.

### Талааны checkout формасына кошуу {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › Checkout Forms** (Сеть администратору › Ultimate Multisite › Checkout Формалары) бөлүмүнө кириңиз
2. Checkout формасын ачыңыз же жаратыңыз
3. Checkout редакторунда **Add Field** (Талааны кошуу) баскыңыз
4. талаалар тизмесинен **Domain Selection** (Домен тандалышы) дегенди тандаңыз
5. Талаандын опцияларын конфигурациялаңыз (астында караңыз)
6. Форманы сактаңыз

### Талаандын опциялары {#field-options}

**Domain modes** (Домен режимдери) — а客様 кандай вкладкаларды көрөөрүн тандайт. Ар бир режим өз алдынча күйгөн же өчүрүлө алат:

| Mode | Эмне кылат |
|---|---|
| **Subdomain** (Субдомен) | А客様 ваша сетьте акысыз субдоменди колдонот (мисалы, `mysite.yournetwork.com`). Төлөм керек эмес. |
| **Register New Domain** (Жаңы доменди каттоо) | А客様 жаңы домен издөө жана аны сиздин конфигурациялаган провайдер аркылуу каттоодон өткөрөт. Баалоо үчүн дал келген домен продукту колдонулат. |
| **Existing Domain** (Учурдагы домен) | А客様 өзүнө кирип жаткан доменди мапする (багыттайт). Каттоо акысы жок. Домен автоматтык түрдө алардын сайтына мап кылынат. |

**Default mode** (Стандарттык режим) — ар бир үч режим күйгөлгөндө, кайсы вкладка биринчи ачылат? Доменди каттоо опционалдуу калтыруу үчүн **Subdomain** деп коюңуз же сатып алууларды ынаштыруу үчүн **Register New Domain** деп коюңуз.

**Domain product** (Домен продукту) — бул талааны белгилүү бир домен продуктуна максаттуу кылууну мүмкүн болот. Эгер аны белгилебесе, анда addon автоматтык түрдө а客様 издөөчү TLDга (домен тобу) негизделген дал келген продукту тандайт.

### Каттоо контактдери талаалары {#registrant-contact-fields}

А客様 **Register New Domain** (Жаңы доменди каттоо) вкладкасын тандаганда, checkout формасы каттаруу контактдери талааларын ошол жерде кошот:

- Аты / Аты-жөнү
- Электрондук почта дареги
- Дарек (1-жайыры, шаар, аймак/область, почта индекси, өлкө)
- Телефон номери

Все это требуется всеми регистраторами и проверяется перед тем, как будет сделан вызов API регистрации. Номера телефонов автоматически форматируются в международный формат `+CC.NNN`, который ожидают регистраторы.

### Автоматически сгенерируемый URL сайта {#auto-generated-site-url}

Когда клиент регистрируется или привязывает домен, поле URL сайта автоматически заполняется выбранным доменом. Клиентам не нужно заполнять отдельное поле для URL.

### Поведение поиска {#search-behaviour}

- Доступность домена проверяется в реальном времени через AJAX по мере ввода клиента текста
- Предлагаются альтернативные TLD (домены верхнего уровня), когда предпочтительный домен недоступен
- Цены подгружаются в режиме реального времени и отображаются четко (цена регистрации, цена продления, необязательная плата за конфиденциальность WHOIS)
- Коды купонов применяются к продуктам домена так же, как и к любому другому продукту

**Настройка отзывчивости поиска:**

```php
// Увеличить задержку дебаунса (в миллисекундах), чтобы уменьшить количество вызовов API при медленных соединениях
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // по умолчанию: 500
});
```

**Добавление пользовательских полей в форму поиска домена:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Дополнительные примечания',
    ];
    return $fields;
});
```

## Управление DNS клиентами {#customer-dns-management}

Клиенты могут управлять записями DNS для своих зарегистрированных доменов на странице **Мой аккаунт**, в разделе, относящемся к их домену.

### Поддерживаемые типы записей {#supported-record-types}

| Тип | Использование |
|---|---|
| **A** | Сопоставление имени хоста с IPv4-адресом |
| **AAAA** | Сопоставление имени хоста с IPv6-адресом |
| **CNAME** | Создание псевдонима, указывающего на другое имя хоста |
| **MX** | Установка сервера почтового обмена |
| **TXT** | Добавление записей SPF, DMARC, верификации или других текстовых записей |

### Какие провайдеры поддерживают управление DNS? {#which-providers-support-dns-management}

DNS-тиркеме (жазыктарды кошуу, өзгөртүү, жоюу) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** жана **Openprovider** аркылуу жеткиликтүү. **Hostinger** домендерин Доменди сатуучу (Domain Seller) аркылуу сервердик аты-жөнүн өзгөртө аласыз; хостинг кылган домендердин DNS тиркемелери Hostinger'дин негизги домен картасы интеграциясы тарабынан башкарылат. Namecheap, GoDaddy жана NameSilo домендери статус жана акыркы күндү көрсөтөт, бирок DNS-тиркемени регистрдин өзүнүн контрол панелинде түз башкаруу керек.

### Стандарттык DNS тиркемелери {#default-dns-records}

Домен катталганда автоматтык түрдө колдонулган стандарттык DNS тиркемелерин конфигурацияласаңыз болот. **Settings › Domain Seller › Default DNS Records**-ка барыңыз.

Стандарттык тиркеме маанилери эки токенди колдойт:

| Токен | Оңдолгон мааниси |
|---|---|
| `{DOMAIN}` | Катталган домен аты (мисалы, `example.com`) |
| `{SITE_URL}` | Клинттин сайты үчүн WordPress сайтынын URL-ы |

**Мисал — apex доменди жана www'ну сервердин IP дарегине багыттоо:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: DNS-тиркемени көрүү жана өзгөртүү {#admin-viewing-and-editing-dns}

Сеть администраторлору домендин редакторду **Network Admin › Ultimate Multisite › Domains**-дан көрсөтүп, DNS тиркемелерин көрүп, өзгөртө алышат.
