# Көп ижарачылык интеграциясы

Ultimate Multisite: Multi-Tenancy 1.2.0 суверендүү ижарачылар, миграцияны текшерүү жана ижарачынын жашоо циклин автоматташтыруу үчүн бир нече интеграция түйүндөрүн өзгөртөт.

## Ижарачыны баштапкы ишке киргизүү агымы

Ижарачыларды түзгөн же өзгөрткөн интеграциялар төмөнкү тартипти сакташы керек:

1. Ижарачы реестринин жазуусун жана изоляция моделин аныктаңыз.
2. Ижарачынын маалымат базасынын жазгычын түзүңүз же текшериңиз.
3. Ижарачынын схемасын баштапкы ишке киргизиңиз.
4. Ижарачы колдонуучуларын камсыздаңыз.
5. Ижарачынын багыттоосун жана файл тутумунун жолдорун каттаңыз.
6. Ижарачыны жеткиликтүү кылуудан мурун миграцияны текшерүүнү иштетиңиз.

Суверендүү ижарачы тармактык маалымат базасынын туташуусун кайра колдоно алат деп болжобоңуз. Кошумча берген ижарачы реестрин жана жазгыч абстракцияларын колдонуңуз.

## SSO жана REST илгичтери

Абалды сактабаган ижарачынын автоматтык кирүүсү максат билдирүүсү, JTI кайра ойнотуудан коргоо, мөөнөт чеги жана келип чыгышты бекитүү менен кыска мөөнөттүү токендерди колдонот. Кирүү баскычтарын же алыстан башкаруу шилтемелерин кошкон интеграциялар ижарачынын кирүү URL даректерин түздөн-түз түзүүнүн ордуна колдоого алынган SSO агымы аркылуу ижарачы сапарларын жаратышы керек.

Тармак тарабындагы API аудит окуялары жана күнүмдүк жыйынтыктар суверендүү ижарачы шлюздары үчүн жеткиликтүү. Ижарачынын жашоо циклинин endpoint'терин чакырган тышкы тутумдарды мүчүлүштүктөрдөн арылтууда ошол журналдарды колдонуңуз.

## Суверендүү кардардын аракет URL даректери

Ultimate Multisite v2.13.0 суверендүү ижарачынын кардар аракеттерин Account, төлөмдү бүтүрүү, эсеп-кысап, эсеп-фактура, сайт, шаблонду алмаштыруу жана доменди байланыштырууу агымдары үчүн негизги сайтка кайра багыттайт. Ижарачы тарабындагы башкаруу шилтемелерин көрсөтүүчү интеграциялар бул аракеттерди негизги сайттын кардар панелине багыттап, колдонуучу аракетти аяктагандан кийин ижарачыга кайра кайта алышы керек болгондо текшерилген кайтуу максатын камтышы керек.

Домендер аралык башкаруу шилтемелери үчүн негизги SSO оромун колдонуңуз:

```php
$url = wu_with_sso($main_site_customer_url);
```

Түзүлгөн URL `wu_sso_url` аркылуу чыпкаланууга болот; ал SSO URL'ин, учурдагы колдонуучуну, максаттуу сайт ID'син жана кайра багыттоо контекстин алат. Кошумчалар бул чыпканы провайдерге мүнөздүү контекстти кошуу же Ultimate Multisite'тын токен текшерүүсүн сактоо менен брокер URL'ин алмаштыруу үчүн колдоно алат.

Суверендүү ижарачынын ичинде мүчөлүк, эсеп-фактура, эсеп-кысап дареги, шаблон же доменди башкаруу абалын кайталап сактабаңыз. Ижарачы Dashboard'ун ишке киргизгич, ал эми негизги сайттын кардар панелин башкарылган аракеттер үчүн чындык булагы катары караңыз.

## Миграцияны текшерүү

Миграция же жашоо цикли интеграциясы ижарачы маалыматтарын өзгөрткөндөн кийин, текшерүү дарбазаларын иштетиңиз:

- `wp tenant verify-no-legacy --site=<site-id>` ижарачы мындан ары эски тармак тарабындагы жолдорго көз каранды эмес экенин ырастайт.
- `wp tenant verify-sovereign-push --site=<site-id>` суверендүү түртүү тапшырмалары иштетилип, кезектен чыгарыларын ырастайт.

Интеграциялар текшерүү ийгиликсиз болсо, аны жайылтууга тоскоолдук катары карап, ката чечилмейинче ижарачыны жандуу деп белгилөөдөн качышы керек.

## Ижарачыны өчүрүү

Өчүрүү агымдары ижарачынын маалымат базасынын эсептик дайындары тазаланышы үчүн кошумчанын teardown жолун чакырышы керек. Тышкы интеграциялар teardown ийгиликтүү аяктагандан кийин провайдер ресурстарын алып салышы мүмкүн, бирок текшерүү же асинхрондук түртүү тапшырмалары дагы эле иштеп жатканда хост маалымат базаларын же папкаларды өчүрбөшү керек.

## Эскирген маалымат базасынын роутери

Эски `Database_Router` эскирүү stub'у менен алмаштырылды. Жаңы интеграциялар эски роутер классына көз каранды болуунун ордуна, ижарачыларды учурдагы сайт роутери жана ижарачы реестри API'лери аркылуу аныкташы керек.
