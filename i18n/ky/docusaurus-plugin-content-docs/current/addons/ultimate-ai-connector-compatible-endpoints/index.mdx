# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints OpenAI менен шайкеш AI провайдер API'лерин WordPress мультисайт тармагыңыз менен интеграциялайт. Бир нече AI провайдерди — ар бири өзүнүн акыркы чекит URL'и, API ачкычы жана демейки модели менен — конфигурациялап, провайдер жеткиликсиз болгондо сурамдарды автоматтык резервдик багыттоо менен жөнөтүңүз.

## Негизги мүмкүнчүлүктөр

- **Көп провайдерди колдоо**: Бир нече AI провайдер картасын кошуңуз, ар бири өзүнүн акыркы чекит URL'и, API ачкычы жана демейки модели менен
- **Сүйрөп кайра иреттөө**: Ар бир картадагы кармагычты колдонуп, провайдерлерди артыкчылык боюнча жайгаштырыңыз; сурамдар провайдерлерди ирети менен колдонот
- **Автоматтык резервдик багыттоо**: Негизги провайдер жеткиликсиз болсо, сурамдар автоматтык түрдө кийинки конфигурацияланган провайдерге өтөт
- **Ар бир провайдер үчүн конфигурация**: Ар бир провайдер өзүнүн эсептик дайындары жана модель тандоосу менен өз алдынча конфигурацияланат
- **Картага негизделген жөндөөлөр UI'и**: Туруктуу WordPress 6.9+ Card компоненттеринин (CardBody, CardHeader, CardDivider) негизинде курулган
- **Кошкондо автоматтык кеңейтүү**: Жаңы провайдер карталары автоматтык түрдө кеңейет, ошондуктан деталдарды дароо толтура аласыз
- **Скрипт кэшин жаңыртуу**: Провайдер скрипттери plugin жаңыртылганда кэштен тазаланып, эң акыркы версия жүктөлүшүн камсыздайт

## Талаптар

- WordPress 5.3 же андан жогору
- PHP 7.4 же андан жогору
- Ultimate Multisite plugin'и (активдүү)
- OpenAI менен шайкеш API акыркы чекити (өзүңүздө жайгаштырылган же үчүнчү тараптыкы)

## Орнотуу

1. Addon файлдарын `/wp-content/plugins/` директорияңызга жүктөңүз
2. Plugin'ди WordPress ичиндеги 'Plugins' менюсу аркылуу активдештириңиз
3. Addon'ду конфигурациялоо үчүн **Ultimate Multisite → AI Connector** бөлүмүнө өтүңүз

## Конфигурация

### Көп провайдерди жөндөө

Конфигурация экраны картага негизделген жайгашууну колдонот, анда ар бир карта бир AI провайдерди билдирет. Керегиңизче көп провайдер кошо аласыз.

#### Провайдер кошуу

1. Тармак админ бөлүмүндө **Ultimate Multisite → AI Connector** бөлүмүнө өтүңүз
2. **Провайдер кошуу** баскычын басыңыз
3. Жаңы провайдер картасы пайда болуп, автоматтык түрдө кеңейет
4. Провайдер деталдарын толтуруңуз:
   - **Провайдердин аты** — бул провайдерди аныктоо үчүн энбелги (мис., "OpenAI", "Mistral", "Local Ollama")
   - **Акыркы чекит URL'и** — OpenAI менен шайкеш API'нин негизги URL'и (мис., `https://api.openai.com/v1`)
   - **API ачкычы** — бул провайдер үчүн API ачкычыңыз
   - **Демейки модель** — бул провайдер үчүн демейки колдонулуучу модель идентификатору (мис., `gpt-4o`, `mistral-small`)
5. **Жөндөөлөрдү сактоо** баскычын басыңыз

#### Провайдерди алып салуу

Каалаган провайдер картасындагы **Алып салуу** баскычын басып, аны өчүрүңүз. Өзгөртүүнү колдонуу үчүн жөндөөлөрдү сактаңыз.

#### Провайдерлерди кайра иреттөө

Каалаган провайдер картасынын сол тарабындагы кармагыч сөлөкөтүн (⣿) сүйрөп, аны кайра иреттеңиз. Ирет резервдик өтүү артыкчылыгын аныктайт: сурамдар биринчи провайдерге жөнөтүлөт жана сурам ишке ашпай калса, кийинки провайдерлерге өтөт.

### Резервдик багыттоо

Негизги провайдерге сурам ишке ашпай калганда — таймаут, ылдамдык чектөөсү же кызматтын үзгүлтүгү себептүү — connector тизмедеги кийинки провайдер менен автоматтык түрдө кайра аракет кылат. Бул провайдер ийгиликтүү жооп бергенге же бардык провайдерлер текшерилгенге чейин тизмени ылдый карай улантат.

| Сценарий | Жүрүм-турум |
|----------|-----------|
| Негизги провайдер жооп берет | Сурам кадимкидей тейленет |
| Негизги провайдер ишке ашпай калды | Сурам кийинки провайдер менен кайра аракет кылынат |
| Бардык провайдерлер ишке ашпай калды | Чакыруучуга ката кайтарылат |

Резервдик өтүү чакырган plugin же тема үчүн ачык байкалбайт — канча провайдер колдонулганынын айырмасы жок, алар ийгиликтүү жоопту же акыркы катаны алышат.

### Провайдер конфигурациясынын маалымдамасы

| Талаа | Талап кылынат | Сүрөттөмө |
|-------|----------|-------------|
| Провайдердин аты | Ооба | Бул провайдер үчүн көрсөтүлүүчү энбелги |
| Акыркы чекит URL'и | Ооба | OpenAI менен шайкеш API'нин негизги URL'и |
| API ачкычы | Ооба | Бул провайдер үчүн аутентификация ачкычы |
| Демейки модель | Ооба | Сурамда модель көрсөтүлбөгөндө колдонулуучу модель идентификатору |

## Көйгөйлөрдү чечүү

### Эч бир провайдер тизмеде жок

Сактагандан кийин провайдер тизмеси бош болсо, төмөнкүлөрдү ырастаңыз:

- Провайдерлерди кошкондон кийин **Жөндөөлөрдү сактоо** баскычын басканыңызды
- Браузериңиз жөндөөлөр барагынын сактоосуна бөгөт койбогонун (JavaScript каталары үчүн браузер консолун текшериңиз)
- Plugin активдүү экенин жана сиз тармак админ экранында экениңизди

### Бардык провайдерлер үчүн сурамдар ишке ашпай жатат

- Ар бир провайдердин акыркы чекит URL'и сервериңизден жеткиликтүү экенин текшериңиз (браузериңизден гана эмес)
- Ар бир API ачкычы жарактуу экенин жана мөөнөтү бүтпөгөнүн текшериңиз
- Провайдердин ылдамдык чектөөсү жана квота абалын алардын башкаруу тактасынан карап чыгыңыз
- Connector'дон келген деталдуу ката билдирүүлөрүн кармоо үчүн WordPress мүчүлүштүктөрдү оңдоо журналын (`WP_DEBUG_LOG`) иштетиңиз

### Провайдер резервдик өтүү үчүн колдонулбай жатат

Сүйрөп кайра иреттөө резервдик өтүү иретин өзгөртөт. Резервдик сурамдарды алышын күткөн провайдер тизмеде туура жайгашканын жана анын акыркы чекит URL'и менен API ачкычы туура экенин ырастаңыз.

### Жаңы провайдер картасы автоматтык түрдө кеңейбейт

Автоматтык кеңейүү үчүн браузериңизде JavaScript иштетилген болушу керек. Эгер карта басканда кеңейбесе, каталар үчүн браузер консолун текшериңиз. Автоматтык кеңейүү иштебесе да, картанын башын басуу менен карта мазмунуна дайыма жетүүгө болот.
