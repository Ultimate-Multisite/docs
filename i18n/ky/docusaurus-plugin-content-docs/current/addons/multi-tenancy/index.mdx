# Көп ижарачылык {#multi-tenancy}

Өндүрүмдүүлүк жана коопсуздук үчүн ички сайттарды серверлер жана маалымат базалары боюнча бөлүштүрүңүз

## 1.2.0 версиясынын чыгарылышындагы негизги жаңылыктар {#version-120-release-highlights}

Multi-Tenancy 1.2.0 өндүрүштүк обочолонтуу иш агымдары үчүн эгемендүү tenant негизин кошот:

- Кыска мөөнөттүү, кайра ойнотуудан корголгон SSO кирүүлөрү менен абалсыз tenant autologin.
- Эгемендүү tenant-тар үчүн ар бир ички сайт боюнча маалымат базасын жана файл системасын обочолонтуу.
- Хостту эске алган tenant даярдоо үчүн Hostinger multi-tenancy мүмкүнчүлүгү.
- Мурунку маалыматтарды текшерүү жана асинхрондук түртүү иштетүүсү үчүн эгемендүү көчүрүү текшерүү дарбазалары.
- Bedrock, FrankenPHP жана localhost орнотуулары үчүн конфигурациялануучу ошол эле машинадагы маалымат базасынын хостун байлоо.
- Tenant bootstrap, маалымат базасынын жазгычы, схема bootstrap жана колдонуучуну даярдоо жакшыртуулары.
- Tenant эсептик дайындарын тазалаган коопсузураак эгемендүү жок кылуу.

Колдонуучу колдонмолорун караңыз: [эгемендүү көчүрүү иш агымы](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy обочолонтуусу](/user-guide/configuration/multi-tenancy-isolation) жана [эгемендүү tenant операциялары](/user-guide/administration/sovereign-tenant-operations).

## Талаптар {#requirements}

- WordPress 5.3 же андан жогору
- PHP 7.4 же андан жогору
- Multisite Ultimate plugin (активдүү)

## Орнотуу {#installation}

1. Addon файлдарын `/wp-content/plugins/` каталогуңузга жүктөңүз
2. WordPress ичиндеги 'Plugins' менюсу аркылуу plugin-ди активдештириңиз
3. Тармак администраторунда addon жөндөөлөрүн конфигурациялаңыз

## Иштеп чыгуу {#development}

### Жөндөө {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Тестирлөө {#testing}

Addon кеңири тестирлөө алкагын камтыйт:

- **Модулдук тесттер**: WordPress тест топтому менен PHPUnit негизиндеги тесттер
- **Негизги тест классы**: Жалпы тест сценарийлери үчүн жардамчы ыкмаларды берет
- **Тест камтуусу**: Код камтуусун көзөмөлдөөгө конфигурацияланган
- **Multisite тестирлөө**: Тесттер multisite чөйрөсүндө иштейт

### Код стандарттары {#code-standards}

- **WordPress коддоо стандарттары**: PHPCS аркылуу мажбурлап колдонулат
- **PHP 7.4+ шайкештиги**: Заманбап PHP функциялары колдоого алынат
- **Статикалык талдоо**: Түрдү текшерүү үчүн PHPStan интеграциясы
- **Кодду заманбапташтыруу**: PHP жаңыртуулары үчүн Rector эрежелери

### Куруу системасы {#build-system}

- **Активдерди иштетүү**: CSS/JS файлдары үчүн кичирейтүү
- **Котормо**: POT файлдарын түзүү
- **Таңгактоо**: Таркатуу үчүн архив түзүү
- **Иштеп чыгуу режими**: Мүчүлүштүктөрдү оңдоо үчүн кичирейтилбеген активдер

## Конфигурация {#configuration}

Addon тармак администратор интерфейси аркылуу жеткиликтүү болгон ар кандай конфигурация параметрлерин колдойт.

### Эгемендүү tenant конфигурациясы {#sovereign-tenant-configuration}

Эгемендүү tenant-тар үчүн трафикти tenant-ка багыттоодон мурун tenant маалымат базасын, файл системасынын түп каталогун, маалымат базасынын хост байлоосун жана хост мүмкүнчүлүгүн конфигурациялаңыз. Маалымат базасынын эсептик дайындарына, хост байлоосуна, багыттоого же колдонуучуну даярдоого таасир эткен ар бир өзгөрүүдөн кийин көчүрүү текшерүүсүн иштетиңиз.

`localhost` tenant аткаруу чөйрөсү жана маалымат базасынын уруксаттары жергиликтүү сокетти же ошол эле машинадагы жеткиликтүүлүктү күткөндө гана колдонуңуз. Провайдериңиз конфигурациялаган уруксат хосту ошол болсо, `127.0.0.1` же кызмат хост атын колдонуңуз.

### Hostinger мүмкүнчүлүгү {#hostinger-capability}

Hostinger multi-tenancy мүмкүнчүлүгү жалпы Hostinger интеграция токени менен иштейт. Текшерүүдөн мурун домен hPanel ичинде бар экенин, DNS Hostinger аккаунтуна багытталганын жана tenant маалымат базасынын жөндөөлөрү hPanel менен дал келерин ырастаңыз.

## Hooks жана Filters {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - Addon башталгандан кийин иштетилет
- `ultimate-multisite-addon-template_loaded` - Бардык addon файлдары жүктөлгөндөн кийин иштетилет

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - Addon жөндөөлөрүн өзгөртүү
- `ultimate-multisite-addon-template_enabled` - Addon иштетүү/өчүрүү абалын кайра аныктоо
