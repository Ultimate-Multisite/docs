## Domain mapping ជាអ្វី? {#whats-domain-mapping}

ដូចឈ្មោះបានបង្ហាញ Domain mapping គឺជាសមត្ថភាពដែល Ultimate Multisite ផ្តល់ឱ្យ ដើម្បីទទួលសំណើសម្រាប់ domain ផ្ទាល់ខ្លួន ហើយភ្ជាប់សំណើនោះទៅគេហទំព័រដែលត្រូវគ្នាក្នុងបណ្ដាញ ដែលមាន domain នោះភ្ជាប់ជាមួយ។

### របៀបរៀបចំ domain mapping នៅលើបណ្ដាញ Ultimate Multisite របស់អ្នក {#how-to-setup-domain-mapping-on-your-ultimate-multisite-network}

Domain mapping ត្រូវការការរៀបចំខ្លះពីអ្នកដើម្បីដំណើរការ។ សំណាងល្អ Ultimate Multisite ធ្វើស្វ័យប្រវត្តិកម្មការងារលំបាកសម្រាប់អ្នក ដូច្នេះអ្នកអាចបំពេញលក្ខខណ្ឌបានយ៉ាងងាយស្រួល។

ក្នុងអំឡុងការដំឡើង Ultimate Multisite, wizard នឹងចម្លង និងដំឡើង **sunrise.php** ទៅថតដែលបានកំណត់ដោយស្វ័យប្រវត្តិ។ **wizard នឹងមិនអនុញ្ញាតឱ្យអ្នកបន្តទេ រហូតដល់ជំហាននេះបានបញ្ចប់**។

<!-- Screenshot unavailable: Ultimate Multisite installation wizard with sunrise.php step -->

នេះមានន័យថា ពេល wizard ដំឡើង Ultimate Multisite បានបញ្ចប់ការរៀបចំបណ្ដាញរបស់អ្នក អ្នកអាចចាប់ផ្តើមភ្ជាប់ domain ផ្ទាល់ខ្លួនបានភ្លាមៗ។

សូមចំណាំថា domain mapping ក្នុង Ultimate Multisite មិនមែនជាការចាំបាច់ទេ។ អ្នកមានជម្រើសប្រើមុខងារ domain mapping ដើមរបស់ WordPress Multisite ឬដំណោះស្រាយ domain mapping ផ្សេងទៀត។

ប្រសិនបើអ្នកត្រូវការបិទ domain mapping របស់ Ultimate Multisite ដើម្បីឱ្យដំណោះស្រាយ domain mapping ផ្សេងទៀតដំណើរការ អ្នកអាចបិទមុខងារនេះនៅក្រោម **Ultimate Multisite > ការកំណត់ > Domain Mapping**។

![ទំព័រការកំណត់ Domain Mapping ដែលបង្ហាញការបញ្ជូនបន្តរបស់អ្នកគ្រប់គ្រង សារ mapping និងជម្រើស DNS](/img/config/domain-mapping-settings.png)

នៅខាងក្រោមជម្រើសនេះ អ្នកក៏អាចឃើញជម្រើស **បង្ខំការបញ្ជូនបន្តរបស់អ្នកគ្រប់គ្រង** ផងដែរ។ ជម្រើសនេះអនុញ្ញាតឱ្យអ្នកគ្រប់គ្រងថា អតិថិជនរបស់អ្នកអាចចូលទៅ admin dashboard របស់ពួកគេបានទាំងនៅលើ domain ផ្ទាល់ខ្លួន និង subdomain ឬបានត្រឹមតែមួយក្នុងចំណោមទាំងពីរ។

ប្រសិនបើអ្នកជ្រើស **បង្ខំឱ្យបញ្ជូនបន្តទៅ domain ដែលបានភ្ជាប់** អតិថិជនរបស់អ្នកនឹងអាចចូលទៅ admin dashboard របស់ពួកគេបានតែនៅលើ domain ផ្ទាល់ខ្លួនរបស់ពួកគេប៉ុណ្ណោះ។

ជម្រើស **បង្ខំឱ្យបញ្ជូនបន្តទៅ** **domain បណ្ដាញ** នឹងធ្វើផ្ទុយទាំងស្រុង ពោលគឺអតិថិជនរបស់អ្នកនឹងត្រូវបានអនុញ្ញាតឱ្យចូលទៅ Dashboard របស់ពួកគេតែនៅលើ subdomain របស់ពួកគេប៉ុណ្ណោះ ទោះបីព្យាយាមចូលនៅលើ domain ផ្ទាល់ខ្លួនរបស់ពួកគេក៏ដោយ។

ហើយជម្រើស **អនុញ្ញាតឱ្យចូលទៅផ្នែកអ្នកគ្រប់គ្រងបានទាំង domain ដែលបានភ្ជាប់ និង domain បណ្ដាញ** អនុញ្ញាតឱ្យពួកគេចូលទៅ admin dashboard របស់ពួកគេបានទាំងនៅលើ subdomain និង domain ផ្ទាល់ខ្លួន។

មានវិធីពីរយ៉ាងដើម្បីភ្ជាប់ domain ផ្ទាល់ខ្លួន។ វិធីទីមួយគឺភ្ជាប់ឈ្មោះ domain ពី admin dashboard របស់បណ្ដាញអ្នក ក្នុងនាមជាអ្នកគ្រប់គ្រងជាន់ខ្ពស់។ វិធីទីពីរគឺតាមរយៈ admin dashboard របស់គេហទំព័ររង នៅក្រោមទំព័រ Account។

ប៉ុន្តែមុនពេលអ្នកចាប់ផ្តើមភ្ជាប់ domain ផ្ទាល់ខ្លួនទៅគេហទំព័ររងមួយក្នុងបណ្ដាញរបស់អ្នក អ្នកត្រូវប្រាកដថា **ការកំណត់ DNS** របស់ឈ្មោះ domain ត្រូវបានកំណត់រចនាសម្ព័ន្ធត្រឹមត្រូវ។

### ធានាថាការកំណត់ DNS របស់ domain ត្រូវបានកំណត់រចនាសម្ព័ន្ធត្រឹមត្រូវ {#making-sure-the-domain-dns-settings-are-properly-configured}

ដើម្បីឱ្យការភ្ជាប់ដំណើរការ អ្នកត្រូវប្រាកដថា domain ដែលអ្នកគ្រោងនឹងភ្ជាប់ កំពុងចង្អុលទៅអាសយដ្ឋាន IP របស់បណ្ដាញអ្នក។ សូមចំណាំថា អ្នកត្រូវការអាសយដ្ឋាន IP របស់បណ្ដាញ គឺអាសយដ្ឋាន IP របស់ domain ដែល Ultimate Multisite ត្រូវបានដំឡើង មិនមែនអាសយដ្ឋាន IP របស់ domain ផ្ទាល់ខ្លួនដែលអ្នកចង់ភ្ជាប់ទេ។ ដើម្បីស្វែងរកអាសយដ្ឋាន IP របស់ domain ជាក់លាក់ យើងស្នើឱ្យចូលទៅ [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) ជាឧទាហរណ៍។

ដើម្បីភ្ជាប់ domain ឱ្យបានត្រឹមត្រូវ អ្នកត្រូវបន្ថែម **A RECORD** មួយនៅក្នុងការកំណត់រចនាសម្ព័ន្ធ **DNS** របស់អ្នក ដែលចង្អុលទៅ **អាសយដ្ឋាន IP** នោះ។ ការគ្រប់គ្រង DNS ខុសគ្នាខ្លាំងរវាងក្រុមហ៊ុនចុះបញ្ជី domain ផ្សេងៗ ប៉ុន្តែមានមេរៀនណែនាំជាច្រើននៅលើអ៊ីនធឺណិតអំពីរឿងនេះ ប្រសិនបើអ្នកស្វែងរក " _ការបង្កើត A Record នៅលើ XXXX_ " ដែល XXXX គឺជាក្រុមហ៊ុនចុះបញ្ជី domain របស់អ្នក (ឧទា៖ " _ការបង្កើត A Record នៅលើ_ _GoDaddy_ ")។

ប្រសិនបើអ្នកជួបបញ្ហាក្នុងការធ្វើឱ្យវាដំណើរការ **សូមទាក់ទងផ្នែកគាំទ្ររបស់ក្រុមហ៊ុនចុះបញ្ជី domain របស់អ្នក** ហើយពួកគេនឹងអាចជួយអ្នកក្នុងផ្នែកនេះបាន។

ប្រសិនបើអ្នកគ្រោងអនុញ្ញាតឱ្យអតិថិជនរបស់អ្នកភ្ជាប់ domain របស់ពួកគេផ្ទាល់ ពួកគេនឹងត្រូវធ្វើការងារផ្នែកនេះដោយខ្លួនឯង។ បើពួកគេមិនអាចបង្កើត A Record បាន សូមណែនាំពួកគេទៅកាន់ប្រព័ន្ធគាំទ្ររបស់ក្រុមហ៊ុនចុះបញ្ជីរបស់ពួកគេ។

### ភ្ជាប់ឈ្មោះ domain ផ្ទាល់ខ្លួនក្នុងនាមជាអ្នកគ្រប់គ្រងជាន់ខ្ពស់ {#mapping-custom-domain-name-as-super-admin}

នៅពេលអ្នកបានចូលជាអ្នកគ្រប់គ្រងជាន់ខ្ពស់នៅលើបណ្ដាញរបស់អ្នក អ្នកអាចបន្ថែម និងគ្រប់គ្រងឈ្មោះ domain ផ្ទាល់ខ្លួនបានយ៉ាងងាយ ដោយចូលទៅក្រោម **Ultimate Multisite > Domain**។

![ទំព័របញ្ជី Domain ក្នុង Ultimate Multisite](/img/admin/domains-list.png)

នៅក្រោមទំព័រនេះ អ្នកអាចចុចប៊ូតុង **បន្ថែម Domain** នៅផ្នែកខាងលើ ហើយវានឹងបង្ហាញផ្ទាំង modal មួយ ដែលអ្នកអាចកំណត់ និងបំពេញ **ឈ្មោះ domain ផ្ទាល់ខ្លួន**, **គេហទំព័ររង** ដែលអ្នកចង់អនុវត្តឈ្មោះ domain ផ្ទាល់ខ្លួនទៅ និងសម្រេចថាតើអ្នកចង់កំណត់វាជា **ឈ្មោះ domain ចម្បង** ឬអត់ (សូមចំណាំថា អ្នកអាចភ្ជាប់ **ឈ្មោះ domain ច្រើនទៅគេហទំព័ររងមួយ** បាន)។

នៅពេលការរួមបញ្ចូលជាមួយអ្នកផ្តល់ host សកម្ម Ultimate Multisite ក៏អាចដាក់បញ្ចូលភារកិច្ចបង្កើត DNS ឬដែនរងនៅខាងអ្នកផ្តល់សេវា សម្រាប់វេបសាយដែលទើបបង្កើតថ្មី។ ប្រសិនបើគ្មានការរួមបញ្ចូលណាមួយកំពុងស្តាប់ភារកិច្ចនោះ ការងារផ្ទៃខាងក្រោយនឹងត្រូវរំលង ដើម្បីជៀសវាងធាតុជួរដែលគ្មានប្រតិបត្តិការ។ ការពិនិត្យ DNS និង SSL សម្រាប់ដែនដែលបានភ្ជាប់នៅតែបន្តដំណើរការតាមដំណើរការដំណាក់កាលដែនធម្មតា។
