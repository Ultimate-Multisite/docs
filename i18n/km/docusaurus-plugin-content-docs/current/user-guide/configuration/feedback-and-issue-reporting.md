# ការផ្តល់មតិយោបល់ និងការរាយការណ៍បញ្ហា

Gratis AI Agent v1.5.0 បានណែនាំប្រព័ន្ធផ្តល់មតិយោបល់ និងការរាយការណ៍បញ្ហាដែលភ្ជាប់មកជាមួយ ដែលអនុញ្ញាតឱ្យអ្នកប្រើប្រាស់ចុងក្រោយអាចសម្គាល់ការឆ្លើយតបដែលមិនមានប្រយោជន៍ និងរាយការណ៍ពីបញ្ហាដោយផ្ទាល់ពីចំណុចប្រទាក់ឆាត។ ប្រព័ន្ធនេះរួមបញ្ចូលទាំងការគ្រប់គ្រងការយល់ព្រម (consent management) បញ្ជាការរាយការណ៍ដោយស្វ័យប្រវត្តិ និងការបែងចែកអាទិភាពដោយ AI នៅផ្នែកខាងក្រោយ (backend)។

## ប៊ូតុង Thumbs-Down {#thumbs-down-button}

សារនីមួយៗដែលបញ្ជូនដោយជំនួយការ AI នឹងបង្ហាញប៊ូតុង **thumbs-down** (👎)។ នៅពេលអ្នកប្រើប្រាស់ចុចលើវា ពួកគេអាចសម្គាល់ការឆ្លើយតបនោះថាគ្មានប្រយោជន៍ ឬមិនត្រឹមត្រូវ។

- ប៊ូតុងនេះនឹងលេចឡើងនៅពេលដាក់ម៉ៅ (hover) បន្ទាប់ពីសារជំនួយការនីមួយៗ។
- ការចុចលើវាបើក **Feedback Consent Modal**។
- មតិយោបល់នោះនឹងត្រូវបានភ្ជាប់ទៅជាមួយខ្សែសង្វាក់នៃការសន្ទនា និងសារជាក់លាក់នោះ។

## Feedback Consent Modal {#feedback-consent-modal}

នៅពេលអ្នកប្រើប្រាស់ចុចលើប៊ូតុង thumbs-down វានឹងបង្ហាញផ្ទាំងការយល់ព្រម (consent modal) មុនពេលទិន្នន័យណាមួយត្រូវបានបញ្ជូន។ ផ្ទាំងនេះ៖

- ពន្យល់ថាព័ត៌មានអ្វីខ្លះនឹងត្រូវបានប្រមូល (ផ្នែកខ្លីនៃការសន្ទនា, បរិបទរបស់ browser)។
- សួរអ្នកប្រើប្រាស់ឱ្យបញ្ជាក់ថាពួកគេយល់ព្រមចែករំលែកទិន្នន័យនេះ។
- ផ្តល់នូវវាលសម្រាប់វាយអត្ថបទដោយសេរី ដើម្បីឱ្យអ្នកប្រើប្រាស់អាចពិពណ៌នាអំពីអ្វីដែលបានកើតឡើងខុស។
- ផ្តល់ជម្រើស **Cancel** ដើម្បីបិទវាដោយមិនបានដាក់បញ្ជូន។

នឹងមិនមានការកត់ត្រាមតិយោបល់ទេ រហូតទាល់តែអ្នកប្រើប្រាស់បញ្ជាក់យ៉ាងច្បាស់។

## Auto-Prompt Feedback Banner {#auto-prompt-feedback-banner}

នៅចុងបញ្ចប់នៃការសន្ទនា ជំនួយការអាចបង្ហាញ **auto-prompt feedback banner** — ដែលជាសារដែលមិនរំខាន សួរថាតើវគ្គនោះមានប្រយោជន៍ឬទេ។

បැනរបែបនេះនឹងលេចឡើងដោយស្វ័យប្រវត្តិផ្អែកលើប្រវែងនៃការសន្ទនា និងលទ្ធផលជាក់ស្តែង។ វាភ្ជាប់ទៅនឹងដំណើរការផ្តល់មតិយោបល់ដូចគ្នានឹងប៊ូតុង thumbs-down។ អ្នកប្រើប្រាស់អាចបិទបැනរបែបនេះដោយមិនចាំបាច់ផ្តល់មតិយោបល់ទេ។

## /report-issue Slash Command {#report-issue-slash-command}

អ្នកប្រើប្រាស់អាចបង្កឱ្យដំណើរការនៃវគ្គផ្តល់មតិយោបល់ដោយផ្ទាល់តាមរយៈការវាយ `/report-issue` នៅក្នុងช่องបញ្ចូលឆាត។ បញ្ជាការនេះ៖

- បើក Modal ទទួលការយល់ព្រម (Feedback Consent Modal) ជាបន្ទាន់។
- បំពេញព័ត៌មានក្នុងប្រអប់ពិពណ៌នាដោយផ្ដល់បរិបទអំពីការសន្ទនាក្នុងពេលបច្ចុប្បន្ន។
- អនុញ្ញាតឱ្យអ្នកប្រើប្រាស់បន្ថែមព័ត៌មានលម្អិតមុនពេលដាក់ស្នើ។

ពាក្យបញ្ជា `/report-issue` មាននៅក្នុងគ្រប់របៀបឆាតទាំងអស់ (inline, floating widget, full-screen)។

## AI-Assisted Triage (ការបែងចែកដោយជំនួយពី AI) {#ai-assisted-triage}

មតិយោបល់ដែលបានដាក់ស្នើនឹងត្រូវបានបញ្ជូនទៅស្រទាប់បែងចែកដោយ AI ដែល៖

- ចែកប្រភេទរបាយការណ៍ (កំហុសការពិត, ចម្លើយមិនមានប្រយោជន៍, មាតិកាមិនសមរម្យ ជាដើម)។
- បន្ថែមទិន្នន័យមេ (metadata) ពាក់ព័ន្ធនៃការសន្ទនា។
- បញ្ជូនសេចក្តីសង្ខេបនៃការបែងចែកទៅកាន់ **Feedback Endpoint** ដែលបានកំណត់រចនាសម្ព័ន្ធ (សូមមើល [Settings > Advanced](#settings-advanced))។

នេះជួយកាត់បន្ថយពេលវេលាចំណាយក្នុងការពិនិត្យដោយដៃ ដោយបង្ហាញបញ្ហាដ៏សំខាន់បំផុតជាមុន។

## Settings > Advanced {#settings-advanced} (ការកំណត់ > កម្រិតខ្ពស់) {#settings--advanced}

ដើម្បីបើកដំណើរការ backend ទទួលមតិយោបល់ សូមកំណត់វាលខាងក្រោមនៅក្រោម **Gratis AI Agent → Settings → Advanced**៖

| Field (វាល) | Description (ការពិពណ៌នា) |
|---|---|
| **Feedback Endpoint URL** | URL ដែលទទួលសំណើ POST រួមជាមួយ payload នៃមតិយោបល់ (JSON)។ |
| **Feedback API Key** | Bearer token ដែលផ្ញើនៅក្នុង header `Authorization` ដើម្បីផ្ទៀងផ្ទាត់ការដាក់ស្នើសំណើមតិយោបល់។ |

ទុកវាលទាំងពីរឱ្យទទេដើម្បីបិទការប្រមូលមតិយោបល់។ ប៊ូតុងមតិយោបល់នីមួយៗនៅតែអាចមើលឃើញសម្រាប់អ្នកប្រើប្រាស់ ប៉ុន្តែវានឹងមិនត្រូវបានបញ្ជូនទៅមុខទេ។

:::tip
Feedback endpoint ត្រូវតែទទួល body ជា JSON ដែលមានយ៉ាងហោចណាស់វាល `message_id`, `conversation_id`, `feedback_text` និង `triage_category`។ សូមមើលឯកសារអ្នកផ្តល់ endpoint របស់អ្នកសម្រាប់ស្គ្រៀន (schema) ដែលរំពឹងទុក។
:::
