# ការកែសម្រួលទម្រង់ចុះឈ្មោះរបស់អ្នក

ដើម្បីធ្វើឱ្យបណ្តាញរបស់អ្នកមើលទៅមានលក្ខណៈពិសេសខុសពី SaaS ផ្សេងទៀតដែលបង្កើតនៅលើវេទិកា WordPress, Ultimate Multisite អនុញ្ញាតឱ្យអ្នកកែសម្រួលទំព័រចុះឈ្មោះ និងចូលប្រើប្រាស់ (login pages) របស់អ្នកជាមួយនឹងមុខងារ **Checkout Forms** របស់យើង។

ទោះបីជាវាជាវិធីងាយស្រួល និងបត់បែនក្នុងការសាកល្បងវិធីសាស្រ្តផ្សេងៗនៅពេលព្យាយាមបំប្លែងអតិថិជនថ្មីក៏ដោយ ប៉ុន្តែវាក៏ត្រូវបានប្រើប្រាស់ភាគច្រើនដើម្បីបង្កើតទម្រង់ចុះឈ្មោះផ្ទាល់ខ្លួនផងដែរ។ អត្ថបទនេះមានគោលបំណងបង្ហាញអ្នកពីរបៀបដែលអ្នកអាចធ្វើវាបាន។

## ទំព័រចូលប្រើប្រាស់ និងចុះឈ្មោះ (Login and registration pages):

បន្ទាប់ពីដំឡើង Ultimate Multisite, វាបង្កើតទំព័រចូលប្រើប្រាស់ និងចុះឈ្មោះផ្ទាល់ខ្លួនដោយស្វ័យប្រវត្តិនៅលើគេហទំព័រចម្បងរបស់អ្នក។ អ្នកអាចផ្លាស់ប្តូរទំព័រលំនាំដើមទាំងនេះបានគ្រប់ពេល ដោយចូលទៅកាន់ទំព័រ **Ultimate Multisite > Settings > Login & Registration** របស់អ្នក។

![Login and Registration settings page](/img/config/settings-general.png)

នេះគឺជាទិដ្ឋភាពពេញលេញនៃទំព័រការកំណត់រចនាសម្ព័ន្ធការចូលប្រើប្រាស់ និងចុះឈ្មោះ៖

![Login and Registration settings full page](/img/config/settings-login-registration-full.png)

សូមមកមើលជម្រើសនីមួយៗដែលអ្នកអាចកែសម្រួលនៅលើទំព័រ **Login & Registration**៖

  * **Enable registration (បើកការចុះឈ្មោះ):** ជម្រើសនេះនឹងបើក ឬបិទការចុះឈ្មោះនៅលើបណ្តាញរបស់អ្នក។ ប្រសិនបើវាត្រូវបានបិទ អ្នកប្រើប្រាស់របស់អ្នកនឹងមិនអាចចុះឈ្មោះ និងជាវផលិតផលរបស់អ្នកបានទេ។

  * **Enable email verification (បើកការផ្ទៀងផ្ទាត់អ៊ីមែល):** ប្រសិនបើជម្រើសនេះត្រូវបានបើក អតិថិជនដែលជាវផែនការឥតគិតថ្លៃ ឬផែនការបង់ប្រាក់ដែលមានរយៈពេលសាកល្បងនឹងទទួលបានអ៊ីមែលផ្ទៀងផ្ទាត់ ហើយពួកគេត្រូវចុចលើតំណផ្ទៀងផ្ទាត់ដើម្បីបង្កើតគេហទំព័ររបស់ពួកគេ។

* **หน้าลงทะเบียนเริ่มต้น (Default registration page):** นี่คือหน้าเริ่มต้นสำหรับการลงทะเบียนครับ หน้าหน้านี้จำเป็นต้องเผยแพร่บนเว็บไซต์ของคุณและมีฟอร์มการลงทะเบียน (หรือที่เรียกว่า checkout form) ซึ่งลูกค้าของคุณจะใช้สมัครรับผลิตภัณฑ์ของคุณ คุณสามารถสร้างหน้าลงทะเบียนและฟอร์ม checkout ได้ตามต้องการ เพียงแต่จำไว้ว่าคุณต้องใส่ shortcode ของฟอร์ม checkout ลงในหน้าลงทะเบียน มิฉะนั้นมันจะไม่แสดงผล

  * **ใช้หน้าเข้าสู่ระบบที่กำหนดเอง (Use custom login page):** ตัวเลือกนี้ช่วยให้คุณสามารถใช้หน้าเข้าสู่ระบบที่ปรับแต่งเองได้ แทนที่จะเป็นหน้า wp-login.php เริ่มต้น หากเปิดตัวเลือกนี้ คุณสามารถเลือกได้ว่าหน้าใดจะถูกใช้สำหรับการเข้าสู่ระบบในตัวเลือก **Default login page** (อยู่ด้านล่าง)

  * **ซ่อน URL การเข้าสู่ระบบเดิม (Obfuscate the original login url (wp-login.php))**: หากคุณต้องการซ่อน URL การเข้าสู่ระบบเดิม คุณสามารถเปิดตัวเลือกนี้ได้ ซึ่งมีประโยชน์เพื่อป้องกันการโจมตีแบบ brute-force หากเปิดใช้งานตัวเลือกนี้ Ultimate Multisite จะแสดงข้อผิดพลาด 404 เมื่อผู้ใช้พยายามเข้าถึงลิงก์ wp-login.php เดิม

  * **บังคับให้เผยแพร่เว็บไซต์พร้อมกัน (Force synchronous site publication):** หลังจากลูกค้าสมัครผลิตภัณฑ์บนเครือข่าย เว็บไซต์ที่รอการเผยแพร่ใหม่จะต้องถูกแปลงเป็นเว็บไซต์เครือข่ายจริง กระบวนการเผยแพร่จะเกิดขึ้นผ่าน Job Queue แบบไม่พร้อมกัน หากเปิดตัวเลือกนี้ จะเป็นการบังคับให้การเผยแพร่เกิดขึ้นในคำขอ (request) เดียวกับการสมัครสมาชิก

ตอนนี้ เรามาดูตัวเลือกอื่นๆ ที่ยังคงเกี่ยวข้องกับกระบวนการเข้าสู่ระบบและการลงทะเบียน ซึ่งอยู่ด้านล่าง **Other options** บนหน้า Login & registration เดียวกัน:

  * **บทบาทเริ่มต้น (Default role):** นี่คือบทบาทที่ลูกค้าของคุณจะมีบนเว็บไซต์หลังจากกระบวนการสมัครเสร็จสิ้น

* **បើក Jumper:** បើកជម្រើស Jumper shortcut នៅក្នុងតំបន់ admin។ Jumper អនុញ្ញាតឱ្យអ្នកគ្រប់គ្រងអាចរុញទៅកាន់អេក្រង់ Ultimate Multisite, network objects និងគោលដៅផ្សេងទៀតដែលគាំទ្របានយ៉ាងឆាប់រហ័ស ដោយមិនចាំបាច់រុករកតាមម៉ឺនុយទាំងអស់។ បិទវាចោល ប្រសិនបើអ្នកចូលចិត្តលាក់ឧបករណ៍นำทางលឿននោះចេញពីចំណុចប្រទាក់ admin។

* **បន្ថែមអ្នកប្រើប្រាស់ទៅក្នុងគេហទំព័រ chính ផងដែរ:** ការបើកជម្រើសនេះនឹងបន្ថែមអ្នកប្រើប្រាស់ទៅក្នុងគេហទំព័រ chính នៃ network របស់អ្នកបន្ទាប់ពីដំណើរការចុះឈ្មោះរួច។ ប្រសិនបើអ្នកបើកជម្រើសនេះ ជម្រើសមួយដើម្បីកំណត់ **default role** របស់អ្នកលើគេហទំព័ររបស់អ្នកក៏នឹងលេចឡើងនៅខាងក្រោមភ្លាមៗដែរ។

* **បើកគណនីច្រើន:** អនុញ្ញាតឱ្យអ្នកប្រើប្រាស់មានគណនីនៅក្នុងគេហទំព័រផ្សេងៗនៃ network របស់អ្នកដោយប្រើអាសយដ្ឋានអ៊ីមែលដូចគ្នា។ ប្រសិនបើជម្រើសនេះត្រូវបានបិទ អ្នកអតិថិជនរបស់អ្នកនឹងមិនអាចបង្កើតគណនីនៅលើគេហទំព័រផ្សេងទៀតដែលដំណើរការលើ network របស់អ្នកដោយប្រើអាសយដ្ឋានអ៊ីមែលដូចគ្នានោះទេ។

ហើយនោះគឺជាជម្រើសទាំងអស់ដែលទាក់ទងនឹងការចូល និងចុះឈ្មោះដែលអ្នកអាចកំណត់បាន! កុំភ្លេចរក្សាទុកការកំណត់របស់អ្នកបន្ទាប់ពីបញ្ចប់ការកែសម្រួល។

## ការប្រើប្រាស់ទម្រង់ចុះឈ្មោះច្រើន៖

Ultimate Multisite 2.0 មានកម្មវិធីកែសម្រួលទម្រង់ checkout ដែលអនុញ្ញាតឱ្យអ្នកបង្កើតទម្រង់បានច្រើនតាមដែលអ្នកចង់បាន ដោយមានវាលផ្សេងៗគ្នា, ផលិតផលដែលផ្តល់ជូនជាដើម។

ទាំងទំព័រ login និង registration ត្រូវបានបញ្ចូលជាមួយ shortcodes៖ **[wu_login_form]** នៅលើទំព័រ login និង **[wu_checkout]** សម្រាប់ទំព័រ registration។ អ្នកអាចកែសម្រួលទំព័រ registration បន្ថែមទៀតដោយការបង្កើត ឬបង្កើត checkout forms។

ដើម្បីចូលប្រើមុខងារនេះ សូមទៅកាន់ម៉ឺនុយ **Checkout Forms** នៅលើរបារចំហៀងខាងឆ្វេង។

![Checkout Forms menu in the sidebar](/img/config/checkout-forms-list.png)

នៅលើទំព័រនេះ អ្នកអាចមើលឃើញ checkout forms ទាំងអស់ដែលអ្នកមាន។

ប្រសិនបើអ្នកចង់បង្កើតថ្មី សូមចុចលើ **Add Checkout Form** នៅផ្នែកខាងលើនៃទំព័រ។

អ្នកអាចជ្រើសរើសជម្រើសមួយក្នុងចំណោមបីនេះជាចំណុចចាប់ផ្តើមរបស់អ្នក៖ single step (ជំហានតែមួយ), multi-step (ច្រើនជំហាន) ឬ blank (ទទេ)។ បន្ទាប់មក ចុចដើម្បី **Go to the Editor** (ទៅកាន់កម្មវិធីនិពន្ធ)។

![Add Checkout Form with single step, multi-step, or blank options](/img/config/checkout-forms-list.png)

ឬ អ្នកក៏អាចកែសម្រួល ឬចម្លងទម្រង់ដែលអ្នកមានរួចហើយបានដោយចុចលើជម្រើសខាងក្រោមឈ្មោះរបស់វា។ នៅទីនោះ អ្នកនឹងឃើញជម្រើសសម្រាប់ចម្លង shortcode នៃទម្រង់នោះ ឬដើម្បីលុបទម្រង់នោះ។

![Checkout form hover actions with edit, duplicate, and delete](/img/config/checkout-form-hover-actions.png)

ប្រសិនបើអ្នកជ្រើសរើស single step (ជំហានតែមួយ) ឬ multi-step (ច្រើនជំហាន) ទម្រង់ checkout នឹងត្រូវបានបំពេញរួចជាស្រេចជាមួយនឹងជំហានមូលដ្ឋានដើម្បីឱ្យវាដំណើរការ។ បន្ទាប់មក ប្រសិនបើអ្នកចង់ អ្នកអាចបន្ថែមជំហានបន្ថែមទៅលើវាបាន។

### ការកែសម្រួល Checkout Form៖

ដូចដែលយើងបានរៀបរាប់ពីមុន អ្នកអាចបង្កើត checkout forms សម្រាប់គោលបំណងផ្សេងៗគ្នា។ ក្នុងឧទាហរណ៍នេះ យើងនឹងធ្វើការជាមួយ registration form (ទម្រង់ចុះឈ្មោះ)។

បន្ទាប់ពីចូលទៅកាន់ checkout form editor សូមដាក់ឈ្មោះឱ្យវា (ដែលនឹងត្រូវបានប្រើសម្រាប់ការយោងផ្ទៃក្នុងតែប៉ុណ្ណោះ) និង slug (ដែលប្រើដើម្បីបង្កើត shortcakes ជាដើម)។

![Checkout form editor with name and slug fields](/img/config/checkout-form-editor.png)

នៅលើแท็บទីមួយនៃ modal window សូមបំពេញខ្លឹមសារនៃជំហានរបស់ទម្រង់របស់អ្នក។ ផ្តល់ ID, name និង description ដល់វា។ អង្គភាពទាំងនេះភាគច្រើនត្រូវបានប្រើសម្រាប់ការយោងផ្ទៃក្នុងប៉ុណ្ណោះ។

![Checkout step content tab with ID, name, and description](/img/config/checkout-form-step-content.png)

បន្ទាប់មក កំណត់ការមើលឃើញ (visibility) នៃជំហាននោះ។ អ្នកអាចជ្រើសរើសរវាង **Always show** (បង្ហាញជានិច្ច), **Only show for logged in users** (បង្ហាញសម្រាប់អ្នកប្រើប្រាស់ដែលបានចូលគណនីតែប៉ុណ្ណោះ) ឬ **Only show for guests** (បង្ហាញសម្រាប់ភ្ញៀវតែប៉ុណ្ណោះ)។

![Checkout step visibility options](/img/config/checkout-form-step-visibility.png)

ទីបំផុត សូមកំណត់ស្ទីលនៃជំហាននីមួយៗ។ នេះគឺជាវាលដែលមិនតម្រូវឱ្យមានការបំពេញ (optional fields)។

![Checkout step style configuration](/img/config/checkout-form-step-style.png)

ឥឡូវនេះដល់ពេលបន្ថែមវាលទៅក្នុងជំហានដំបូងរបស់យើងហើយ។ អ្នកគ្រាន់តែចុចលើ **Add New Field** ដើម្បីបន្ថែមវា ហើយជ្រើសរើសប្រភេទផ្នែក (section type) ដែលអ្នកចង់បាន។

វាលនីមួយៗមានប៉ារ៉ាម៉ែត្រខុសៗគ្នាដែលត្រូវបំពេញ។ សម្រាប់ការចូលប្រើដំបូងនេះ យើងនឹងជ្រើសរើសវាល **Username** (ឈ្មោះអ្នកប្រើប្រាស់)។

អ្នកអាចបន្ថែមជំហាន និងវាលបានច្រើនតាមតម្រូវការរបស់អ្នក។ ដើម្បីបង្ហាញផលិតផលរបស់អ្នកឱ្យអតិថិជនជ្រើសរើសមួយ សូមប្រើវាល **Pricing Table** (តារាងតម្លៃ)។ ប្រសិនបើអ្នកចង់ឱ្យអតិថិជនជ្រើសរើស قالب (template) អ្នកអាចបន្ថែមវាល **Template Selection** (ការជ្រើសរើស قالب)។ ហើយដូច្នេះទៅ។

_**ចំណាំ:** ប្រសិនបើអ្នកបង្កើតផលិតផលបន្ទាប់ពីបានបង្កើតទម្រង់ checkout របស់អ្នករួច អ្នកនឹងត្រូវបន្ថែមផលិតផលនោះនៅក្នុងផ្នែក Pricing table។ ប្រសិនបើអ្នកមិនបានបន្ថែមទេ ផលិតផលនោះនឹងមិនបង្ហាញដល់អតិថិជននៅលើទំព័រចុះឈ្មោះទេ។

_**ចំណាំ ២:** username, email, password, site title, site URL, order summary, payment, និង submit button គឺជាវាលដែលត្រូវបំពេញជាកាតព្វកិច្ចដើម្បីបង្កើតទម្រង់ checkout។

ខណៈពេលដែលអ្នកកំពុងធ្វើការលើទម្រង់ checkout របស់អ្នក អ្នកអាចប្រើប៊ូតុង **Preview** ដើម្បីមើលថាតើអតិថិជននឹងឃើញទម្រង់នោះយ៉ាងដូចម្តេច។ អ្នកក៏អាចប្តូររវាងការមើលជាអ្នកប្រើប្រាស់ដែលមានស្រាប់ (existing user) ឬអ្នកចូលមើល (visitor) បានដែរ។

## การเลือกผลิตภัณฑ์และเทมเพลตล่วงหน้าผ่าน URL parameters:

ถ้าคุณต้องการสร้างตารางราคาที่ปรับแต่งเองสำหรับผลิตภัณฑ์ของคุณ และเลือกผลิตภัณฑ์หรือเทมเพลตที่จะให้ลูกค้าเลือกจากหน้าตารางราคาหรือหน้าเทมเพลตของคุณในฟอร์มชำระเงิน คุณสามารถใช้ URL parameters สำหรับเรื่องนี้ได้

### **สำหรับแผน (Plans):**

ไปที่ **Ultimate Multisite > Products > Select a plan** คุณจะเห็นปุ่ม **Click to copy Shareable Link** ที่ด้านบนของหน้า นี่คือลิงก์ที่คุณสามารถใช้เพื่อเลือกแผนเฉพาะเจาะจงนี้ล่วงหน้าในฟอร์มชำระเงินของคุณได้เลย

![หน้าผลิตภัณฑ์พร้อมปุ่มลิงก์ที่แชร์ได้](/img/config/products-list.png)

ចំណាំថា តំណចែករំលែកនេះមានសុពលភាពសម្រាប់តែ **Plans (ផែនការ)** ប៉ុណ្ណោះ។ អ្នកមិនអាចប្រើតំណចែករំលែកសម្រាប់ package ឬ service បានទេ។

### សម្រាប់ template (គំរូ):

ប្រសិនបើអ្នកចង់ជ្រើសរើស site templates ជាមុននៅលើ form checkout របស់អ្នក អ្នកអាចប្រើ parameter៖ **?template_id=X** នៅលើ URL នៃ trang registration របស់អ្នក។ "X" ត្រូវតែត្រូវបានជំនួសដោយ **លេខ site template ID**។ ដើម្បីទទួលបានលេខនេះ សូមចូលទៅកាន់ **Ultimate Multisite > Sites**។

ចុចលើ **Manage** ដែលនៅខាងក្រោម site template ដែលអ្នកចង់ប្រើ។ អ្នកនឹងឃើញលេខ SITE ID។ គ្រាន់តែប្រើលេខនេះសម្រាប់ site template ជាក់លាក់នេះដើម្បីជ្រើសរើសជាមុននៅលើ form checkout របស់អ្នក។ ក្នុងករណីរបស់យើង URL parameter នឹងមើលទៅដូចជា **?template_id=2**។

![Sites list showing site template ID](/img/config/site-templates-list.png)
