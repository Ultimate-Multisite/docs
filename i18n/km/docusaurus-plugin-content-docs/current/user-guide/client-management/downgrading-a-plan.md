# ការបျាករណ៍ផែនការ (v2)

_**ចំណាំសំខាន់៖ អត្ថបទនេះសំដៅលើ Ultimate Multisite version 2.x។**_

ការបျាករណ៍ផែនការ ឬការជាវឡើងវិញគឺជាអ្វីដែលអតិថិជនរបស់អ្នកអាចធ្វើបានទូទៅ ប្រសិនបើពួកគេមានថវិកាមានកំណត់ ឬសម្រេចចិត្តថាពួកគេមិនត្រូវការធនធានច្រើនដើម្បីដំណើរការ subsite របស់ពួកគេទេ។

## របៀបបျាករណ៍ផែនការ

អតិថិជនរបស់អ្នកអាចបျាករណ៍ផែនការបានគ្រប់ពេល ដោយគ្រាន់តែចូលទៅក្នុង admin dashboard របស់ subsite របស់ពួកគេ ហើយចុច **Change** នៅក្រោមទំព័រគណនី (account page)។

![Customer account page with Your Membership card and Change button](/img/account-page/membership-change-button.png)

បន្ទាប់ពីចុចប៊ូតុង **Change** អ្នកប្រើប្រាស់/អតិថិជននឹងត្រូវបានបញ្ជូនទៅកាន់ទំព័រ checkout ដែលពួកគេអាចជ្រើសរើសផែនការដែលពួកគេចង់ផ្លាស់ប្តូរការជាវរបស់ពួកគេ។

![Downgrade plan options page on the customer side](/img/account-page/downgrade-picker.png)

ក្នុងឧទាហរណ៍នេះ យើងកំពុងបျាករណ៍ផែនការពី **Premium** ទៅ **Free**។

ដើម្បីបន្តដំណើរការ អ្នកប្រើប្រាស់គ្រាន់តែចុចប៊ូតុង **Complete Checkout**។ បន្ទាប់មកវានឹងនាំពួកគេត្រលប់ទៅកាន់ទំព័រគណនីដែលបង្ហាញសារអំពីការផ្លាស់ប្តូរដែលកំពុងរង់ចាំសម្រាប់ការជាវនោះ។ ការផ្លាស់ប្តូរនឹងចូលជាធរ Bautistaលើ **វដ្ដនៃការចេញវិក្កយបត្របន្ទាប់** របស់អតិថិជន។

![Account page showing pending membership change banner](/img/account-page/pending-change.png)

### តើមានអ្វីកើតឡើងនៅពេលអ្នកប្រើប្រាស់បျាករណ៍ផែនការរបស់ពួកគេ?

វាសំខាន់ណាស់ដែលត្រូវចំណាំថា ការបျាករណ៍ផែនការមិនបានផ្លាស់ប្តូរការកំណត់ដែលមានស្រាប់នៅក្នុង subsite របស់អ្នកប្រើប្រាស់នោះទេ។

វាមិនបានផ្លាស់ប្តូរ template របស់ site ដោយស្វ័យប្រវត្តិទេ ព្រោះការផ្លាស់ប្តូរ template នឹងលុប និងកំណត់ឡើងវិញនូវ subsite ទាំងមូល។ នេះគឺដើម្បីជៀសវាងការបាត់បង់ទិន្នន័យដែលមិនចាំបាច់។ ដូច្នេះ ទំហំផ្ទុកឌីស (disk space) ប្រធានបទ (themes) ប្លុកបន្ថែម (plugins) ជាដើមនឹងនៅតែរក្សាទុក លើកលែងតែ posts (អត្ថបទ)។

យើងយល់ថា កង្វល់ចម្បងរបស់អ្នកគឺដែនកំណត់ និងកូតាដែលអ្នកបានកំណត់ក្រោមផែនការនីមួយៗ ប៉ុន្តែយើងត្រូវពិចារណាពីការខូចខាតដែលអាចកើតមានដល់ subsites របស់អ្នក ប្រសិនបើយើងលុប ឬផ្លាស់ប្តូរការកំណត់រចនាសម្ព័ន្ធណាមួយរបស់វា។

សម្រាប់ posts ដែលលើសពីដែនកំណត់ដែលបានកំណត់ក្នុងផែនការ អ្នកមានជម្រើស ៣ យ៉ាង៖ **រក្សាទុក posts ដូចដើម** *,* **ផ្ទេរ posts ទៅ trash** *,* ឬ **ផ្ទេរ posts ទៅ draft** *. អ្នកអាចកំណត់រចនាសម្ព័ន្ធនេះនៅក្រោម Ultimate Multisite settings។

![Network Admin Settings Sites page showing post-limit-exceeded behaviour options](/img/account-page/settings-sites-post-limit.png)

### តើការបង់ប្រាក់មានអ្វីកើតឡើង

នៅក្នុងកំណែ 2.0 វាលែងតម្រូវឱ្យមានការកែសម្រួលណាមួយលើការបែងចែក (proration) នៃការបង់ប្រាក់ទៀតហើយ។

នេះគឺដោយសារប្រព័ន្ធនឹងរង់ចាំឱ្យសមាជិកដែលមានស្រាប់ **បញ្ចប់វដ្ដការគិតថ្លៃរបស់ពួកគេ មុនពេល** ផែនការ/សមាជិកថ្មីចូលជាធរ Bautista។ ចំនួនទឹកប្រាក់បង់ប្រាក់ថ្មីសម្រាប់សមាជិកថ្មីនឹងត្រូវបានអនុវត្ត និងគិតថ្លៃដោយស្វ័យប្រវត្តិនៅក្នុងវដ្ដការគិតថ្លៃបន្ទាប់។
