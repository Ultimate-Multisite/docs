# ការគ្រប់គ្រងវាល Checkout និង DNS របស់អតិថិជន

## វាលជ្រើសរើសដែន (Domain Selection field) {#the-domain-selection-checkout-field}

វាល **Domain Selection** គឺជាធាតុមួយនៅក្នុងដំណើរការ checkout ដែលផ្តល់ជម្រើសដល់អតិថិជនអំពីរបៀបដែលពួកគេនឹងទទួលបានដែនសម្រាប់គេហទំព័ររបស់ពួកគេ។ បន្ថែមវាទៅក្នុងទម្រង់ checkout ណាមួយដើម្បីបើកដំណើរការការលក់ដែន។

### ការបន្ថែមវាលទៅក្នុងទម្រង់ checkout {#adding-the-field-to-a-checkout-form}

1. ចូលទៅកាន់ **Network Admin › Ultimate Multisite › Checkout Forms**
2. បើក ឬបង្កើតទម្រង់ checkout
3. នៅក្នុងកម្មវិធីនិពន្ធ (editor) checkout សូមចុច **Add Field**
4. ជ្រើសរើស **Domain Selection** ពីបញ្ជីវាល
5. កំណត់ជម្រើសនៃវាល (សូមមើលខាងក្រោម)
6. រក្សាទុកទម្រង់

### ជម្រើសនៃវាល (Field options) {#field-options}

**Domain modes** — ជ្រើសរើសផ្ទាំងណាដែលអតិថិជននឹងឃើញ។ របៀបនីមួយៗអាចបើកដំណើរការ ឬបិទបានដោយឯករាជ្យ៖

| Mode | វាធ្វើអ្វីខ្លះ |
|---|---|
| **Subdomain** | អតិថិជនប្រើ subdomain ឥតគិតថ្លៃនៅលើបណ្តាញរបស់អ្នក (ឧទាហរណ៍៖ `mysite.yournetwork.com`)។ មិនត្រូវការការបង់ប្រាក់ទេ។ |
| **Register New Domain** | អតិថិជនស្វែងរកដែនថ្មី ហើយចុះឈ្មោះវាតាមរយៈអ្នកផ្តល់សេវាកម្មដែលបានកំណត់របស់អ្នក។ វាប្រើផលិតផលដែនដែលត្រូវគ្នាសម្រាប់ការកំណត់តម្លៃ។ |
| **Existing Domain** | អតិថិជនភ្ជាប់ដែនដែលពួកគេមានរួចហើយ។ មិនមានថ្លៃចុះឈ្មោះទេ។ ដែននឹងត្រូវបានភ្ជាប់ដោយស្វ័យប្រវត្តិទៅកាន់គេហទំព័ររបស់ពួកគេ។ |

**Default mode** — នៅពេលដែលរបៀបទាំងបីត្រូវបានបើកដំណើរការ តើផ្ទាំងមួយណាដែលបើកមុន? កំណត់វាជា **Subdomain** ដើម្បីរក្សាការចុះឈ្មោះដែនឱ្យនៅតែជាជម្រើស ឬ **Register New Domain** ដើម្បីលើកទឹកចិត្តដល់ការទិញ។

**Domain product** — អាចកំណត់វាលនេះទៅផលិតផលដែនជាក់លាក់មួយបាន។ ប្រសិនបើមិនបានកំណត់ទេ Addon នឹងជ្រើសរើសផលិតផលដែលត្រូវគ្នាដោយស្វ័យប្រវត្តិផ្អែកលើ TLD ដែលអតិថិជនស្វែងរក។

### វាលទំនាក់ទំនងអ្នកចុះឈ្មោះ (Registrant contact fields) {#registrant-contact-fields}

នៅពេលដែលអតិថិជនជ្រើសរើសផ្ទាំង **Register New Domain** ទម្រង់ checkout នឹងបន្ថែមវាលទំនាក់ទំនងអ្នកចុះឈ្មោះចូលទៅក្នុងទម្រង់ដោយផ្ទាល់៖

- ឈ្មោះទីមួយ / នាមត្រកូល
- ប្រវត្តិរូបអ៊ីមែល (Email address)
- អាសយដ្ឋាន (បន្ទាត់ទី ១, ទីក្រុង, រដ្ឋ/ខេត្ត, លេខទូរស័ព្ទ, ប្រទេស)
- លេខទូរស័ព្ទ

តម្រូវការទាំងនេះគឺចាំបាច់សម្រាប់អ្នកចុះឈ្មោះទាំងអស់ ហើយត្រូវបានផ្ទៀងផ្ទាត់មុនពេលហៅ API សម្រាប់ការចុះឈ្មោះ។ លេខទូរស័ព្ទនឹងត្រូវបានដាក់ទ្រង់ទ្រាយអន្តរជាតិ `+CC.NNN` ដែលអ្នកចុះឈ្មោះទាមទារដោយស្វ័យប្រវត្តិ។

### URL គេហទំព័រដែលបង្កើតដោយស្វ័យប្រវត្តិ (Auto-generated site URL) {#auto-generated-site-url}

នៅពេលដែលអតិថិជនចុះឈ្មោះ ឬកំណត់តំបន់ដែន (domain mapping) បន្ទាត់ site URL នឹងត្រូវបានបំពេញដោយស្វ័យប្រវត្តិពីដែនដែលបានជ្រើសរើស។ អតិថិជនមិនចាំបាច់បំពេញវាល URL ផ្សេងទៀតទេ។

### អាកប្បកិរិយានៃការស្វែងរក (Search behaviour) {#search-behaviour}

- ការត្រួតពិនិត្យភាពទំនេរនៃដែនត្រូវបានធ្វើឡើងក្នុងពេលវេលាជាក់ស្តែងជាមួយ AJAX នៅពេលអ្នកប្រើប្រាស់វាយអត្ថបទ
- ការផ្តល់ជម្រើស TLD ផ្សេងទៀតនឹងត្រូវបានបង្ហាញនៅពេលដែលដែនដែលពេញចិត្តមិនអាចរកបាន។
- តម្លៃត្រូវបានទាញយកផ្ទាល់ និងបង្ហាញយ៉ាងច្បាស់ (តម្លៃចុះឈ្មោះ, តម្លៃបន្តការប្រើប្រាស់, ថ្លៃ WHOIS privacy ដែលជាជម្រើស)។
- កូដបញ្ចុះតម្លៃនឹងអនុវត្តចំពោះផលិតផលដែនដូចទៅនឹងផលិតផលផ្សេងទៀតដែរ។

**ការកំណត់រចនាសម្ព័ន្ធប្រតិកម្មនៃការស្វែងរក (Tuning search responsiveness):**

```php
// បង្កើនការពន្យារពេល debounce (មីលីវិន) ដើម្បីកាត់បន្ថយការហៅ API នៅលើការតភ្ជាប់ដែលយឺត
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // តាមលំនាំដើម: 500
});
```

**ការបន្ថែមវាលផ្ទាល់ខ្លួនទៅក្នុងទម្រង់ស្វែងរកដែន (Adding custom fields to the domain search form):**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'ចំណាំបន្ថែម',
    ];
    return $fields;
});
```

## ការគ្រប់គ្រង DNS របស់អតិថិជន (Customer DNS management) {#customer-dns-management}

អតិថិជនអាចគ្រប់គ្រងកំណត់ត្រា DNS សម្រាប់ដែនដែលបានចុះឈ្មោះរបស់ពួកគេពីទំព័រ **My Account** ក្រោមធាតុនៃដែននោះ។

### ប្រភេទកំណត់ត្រាដែលគាំទ្រ (Supported record types) {#supported-record-types}

| ប្រភេទ | ការប្រើប្រាស់ |
|---|---|
| **A** | បញ្ជាក់ឈ្មោះម៉ាស៊ីន (hostname) ទៅកាន់អាសយដ្ឋាន IPv4 |
| **AAAA** | បញ្ជាក់ឈ្មោះម៉ាស៊ីន (hostname) ទៅកាន់អាសយដ្ឋាន IPv6 |
| **CNAME** | បង្កើតឈ្មោះជំនួស (alias) ដែលចង្អុលទៅឈ្មោះម៉ាស៊ីនផ្សេងទៀត |
| **MX** | កំណត់ម៉ាស៊ីនប្តូរអ៊ីមែល (mail exchange server) |
| **TXT** | បន្ថែមកំណត់ត្រា SPF, DMARC, ការផ្ទៀងផ្ទាត់ ឬកំណត់ត្រាអត្ថបទផ្សេងទៀត |

### តើអ្នកផ្តល់សេវាណាខ្លះដែលគាំទ្រការគ្រប់គ្រង DNS? {#which-providers-support-dns-management}

ការគ្រប់គ្រង DNS (បន្ថែម, កែសម្រួល, លុបកំណត់ត្រា) អាចធ្វើបានជាមួយ **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, និង **Openprovider**។ ដែនដីរបស់ **Hostinger** អាចធ្វើបច្ចុប្បន្នភាពឈ្មោះ Name Servers តាមរយៈ Domain Seller។ កំណត់ត្រា DNS សម្រាប់ដែនដែលបានបង្ហោះ (hosted domains) ត្រូវបានគ្រប់គ្រងដោយការរួមបញ្ចូលកម្មវិធី core Hostinger domain-mapping។ ដែនដីរបស់ **Namecheap**, **GoDaddy**, និង **NameSilo** បង្ហាញស្ថានភាព និងព័ត៌មានថ្ងៃផុតកំណត់ ប៉ុន្តែ DNS ត្រូវតែត្រូវបានគ្រប់គ្រងដោយផ្ទាល់នៅក្នុង control panel របស់ registrar (អ្នកចុះបញ្ជី)។

### កំណត់ត្រា DNS តាមលំនាំដើម (Default DNS records) {#default-dns-records}

អ្នកអាចកំណត់រចនាសម្ព័ន្ធកំណត់ត្រា DNS តាមលំនាំដើម ដែលនឹងត្រូវបានអនុវត្តដោយស្វ័យប្រវត្តិនៅពេលដែនដីមួយត្រូវបានចុះបញ្ជី។ សូមចូលទៅកាន់ **Settings › Domain Seller › Default DNS Records**។

តម្លៃកំណត់ត្រាដែលបានកំណត់តាមលំនាំដើមគាំទ្រ token ពីរ៖

| Token | ជំនួសដោយ |
|---|---|
| `{DOMAIN}` | ឈ្មោះដែនដីដែលបានចុះបញ្ជី (ឧទាហរណ៍៖ `example.com`) |
| `{SITE_URL}` | URL របស់ WordPress site សម្រាប់គេហទំព័ររបស់អតិថិជន |

**ឧទាហរណ៍ — បញ្ជាក់ apex domain និង www ទៅកាន់ IP server របស់អ្នក:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Admin: ការមើល និងការកែសម្រួល DNS {#admin-viewing-and-editing-dns}

អ្នកគ្រប់គ្រងបណ្តាញ (Network admins) អាចមើល និងកែសម្រួលកំណត់ត្រា DNS សម្រាប់ដែនដីរបស់អតិថិជនណាមួយពីទំព័ររៀងរបស់ដែនដីនៅក្នុង **Network Admin › Ultimate Multisite › Domains**។
