# ផលិតផលដែន និងការកំណត់តម្លៃ (Domain Products and Pricing)

ផលិតផលដែនគឺជាវិធីដែលអ្នកគ្រប់គ្រងការកំណត់តម្លៃ၊ TLDs (Top-Level Domains) ការការពារ WHOIS privacy និងការជ្រើសរើសអ្នកផ្តល់សេវា។ ផលិតផលដែននីមួយៗគឺជាផលិតផលស្តង់ដាររបស់ Ultimate Multisite ដែលមានផ្ទាំង **Domain Settings** បន្ថែម។

## ការបង្កើតផលិតផលដែន (Creating a domain product) {#creating-a-domain-product}

១. ចូលទៅកាន់ **Network Admin › Ultimate Multisite › Products**
២. ចុចលើ **Add New**
៣. កំណត់ប្រភេទផលិតផលជា **Domain**
៤. រៀបចំផ្ទាំង **Domain Settings** (សូមមើលខាងក្រោម)
៥. សន្សំ (Save)

ផលិតផលដែននឹងបង្ហាញជាមួយស្លាកពណ៌ស្វាយ **Domain** នៅក្នុងបញ្ជីផលិតផល ហើយអាចច្រោះដោយប្រើផ្ទាំង **Domain Products**។

## ផ្ទាំងការកំណត់ដែន (Domain settings tab) {#domain-settings-tab}

### អ្នកផ្តល់សេវា (Provider) {#provider}

ជ្រើសរើសអ្នកចុះឈ្មោះដែលនឹងគ្រប់គ្រងសម្រាប់ផលិតផលនេះ។ ជាលំនាំដើមគឺ **Default provider** ទូទាំងគេ ដែលបានកំណត់នៅក្នុងការកំណត់ Domain Seller។

### TLDs ដែលគាំទ្រ (Supported TLDs) {#supported-tlds}

ទុកវាឱ្យទទេដើម្បីបង្កើត **catch-all product** ដែលអនុវត្តចំពោះ TLDs ទាំងអស់ដែលមិនត្រូវបានផ្គូផ្គងដោយផលិតផលផ្សេងទៀត។

បញ្ចូលបញ្ជី TLDs ដែលបំបែកដោយសញ្ញាក្បៀស (ឧទាហរណ៍៖ `.com, .net, .org`) ដើម្បីបង្កើត **TLD-specific product** ដែលអនុវត្តតែលើផ្នែកបន្ថែមទាំងនោះប៉ុណ្ណោះ។

**របៀបដែលការផ្គូផ្គងផលិតផលដំណើរការ:** នៅពេលអតិថិជនស្វែងរកដែនមួយ ផ្នែកបន្ថែមនឹងជ្រើសរើសផលិតផលដែលត្រូវគ្នាបំផុត។ ផលិតផលដែលមាន `.com` នៅក្នុងបញ្ជី TLD របស់វា នឹងមានអាទិភាពជាងផលិតផល catch-all។ ប្រសិនបើគ្មានផលិតផលជាក់លាក់សម្រាប់ TLD ណាមួយផ្គូផ្គងទេ នោះផលិតផល catch-all នឹងត្រូវបានប្រើ។ ប្រសិនបើគ្មានផលិតផលណាដែលមានទេ ការស្វែងរកដែននឹងមិនត្រូវបានបង្ហាញទេ។

### ប្រភេទ Markup (Markup type) {#markup-type}

មានបីរបៀបដែលគ្រប់គ្រងពីរបៀបគណនាថ្លៃលក់រាយរបស់អ្នកពីតម្លៃដើម (wholesale cost)៖

| របៀប (Mode) | របៀបដំណើរការ (How it works) |
|---|---|
| **Percentage** | បន្ថែមភាគរយលើតម្លៃដើម។ ការបន្ថែម Markup ២០% លើដែនដែលមានតម្លៃដើម ១០ ដុល្លារ នឹងទទួលបាន ១២ ដុល្លារ។ |
| **Fixed markup** | បន្ថែមចំនួនទឹកប្រាក់ថេរមួយ។ ការបន្ថែម Markup ៥ ដុល្លារ លើដែនដែលមានតម្លៃដើម ១០ ដុល្លារ នឹងទទួលបាន ១៥ ដុល្លារ។ |
| **Fixed price** | មិនគិតពីតម្លៃដើម (wholesale cost) ទាំងស្រុងទេ។ បញ្ charge ចំនួនដែលអ្នកបញ្ចូលជានិច្ច។ |

### ការកំណត់តម្លៃណែនាំ (Introductory pricing) {#introductory-pricing}

### ការផ្តល់ជូនតម្លៃពិសេសសម្រាប់ឆ្នាំដំបូង {#whois-privacy}

បើកដំណើរការដើម្បីផ្តល់ជូនតម្លៃពិសេសសម្រាប់ឆ្នាំដំបូង។ កំណត់ **Introductory price** (តម្លៃឆ្នាំទី១) ឱ្យខុសពី **Renewal price** (តម្លៃបន្តសម្រាប់ឆ្នាំទី២ និងលើៗ) ដែលជាតម្លៃធម្មតា។ អតិថិជននឹងឃើញតម្លៃទាំងពីរបង្ហាញក្នុងពេលទូទាត់ប្រាក់ ដើម្បីដឹងថាត្រូវរំពឹងអ្វីនៅពេលបន្តការបង់ប្រាក់។

### ការការពារ WHOIS privacy {#tld-import-and-sync}

គ្រប់គ្រងថាតើផ្តល់ការការពារ WHOIS privacy សម្រាប់ដែនដែលបានចុះឈ្មោះតាមរយៈផលិតផលនេះឬអត់។

| ការកំណត់ | អាកប្បកិរិយា |
|---|---|
| **Disabled** (បិទ) | មិនមានការផ្តល់ ឬបើកដំណើរការ WHOIS privacy ទេ។ |
| **Always Included** (រួមបញ្ចូលជានិច្ច) | WHOIS privacy នឹងត្រូវបានបើកដោយស្វ័យប្រវត្តិនៅពេលចុះឈ្មោះដោយគ្មានការគិតថ្លៃ។ |
| **Customer Choice** (ជម្រើសអតិថិជន) | ប្រអប់ធីកមួយនឹងលេចឡើងក្នុងអំឡុងពេលទូទាត់ប្រាក់។ កំណត់ **Privacy price** ឱ្យគិតប្រចាំឆ្នាំ ឬទុកវាជា $0 ដើម្បីផ្តល់ជូនដោយឥតគិតថ្លៃ។ |

សម្រាប់ Namecheap, WHOIS privacy ប្រើ WhoisGuard (ឥតគិតថ្លៃជានិច្ច)។ សម្រាប់ OpenSRS វាប្រើសេវាកម្មความเป็นส่วนตัวរបស់ OpenSRS (អាចមានតម្លៃនៅពេលលក់ដុំ)។

## ការនាំចូល និងការធ្វើសមកាលកម្ម TLD {#auto-renewal}

ផលិតផលដែនបង្ហាញតម្លៃលក់ដុំពេលវេលាជាក់ស្តែងដែលទាញយកពីអ្នកផ្តល់សេវាដែលបានភ្ជាប់។ ដើម្បីឱ្យវាក្នុងការដំណើរការបាន តត្រូវតែនាំចូល TLDs មុន។

- **Manual sync** (ធ្វើសមកាលកម្មដោយដៃ): Settings › Domain Seller › Sync TLDs (តាមអ្នកផ្តល់សេវា)
- **Automatic sync** (ធ្វើសមកាលកម្មដោយស្វ័យប្រវត្តិ): ដំណើរការរៀងរាល់ថ្ងៃតាមរយៈ scheduled cron job នៅលើអ្នកផ្តល់សេវាកម្មទាំងអស់ដែលបានកំណត់។

បន្ទាប់ពីធ្វើសមកាលកម្មរួច សូមចូលទៅកាន់แท็บ Domain Settings នៃផលិតផលដែនណាមួយ ហើយប្រើ TLD picker ដើម្បីមើល TLDs ដែលមានជាមួយនឹងតម្លៃលក់ដុំបច្ចុប្បន្នរបស់វា។

## ការបន្តដោយស្វ័យប្រវត្តិ (Auto-renewal) {#admin-manual-domain-registration}

ការបន្តដែនត្រូវបានភ្ជាប់ទៅនឹងស្ថានភាពសមាជិករបស់អតិថិជន៖

- នៅពេលដែលសមាជិកបន្ត ហើយដែនត្រូវបានភ្ជាប់ ការបន្តដែននឹងត្រូវបានដាក់ក្នុងបញ្ជីដោយស្វ័យប្រវត្តិ។
- ការព្យាយាមបន្តប្រើ gateway ការទូទាត់ដែលមានសកម្មភាពរបស់អតិថិជន។
- ការបន្តដែលបរាជ័យនឹងត្រូវបានព្យាយាមម្តងទៀតដោយស្វ័យប្រវត្តិជាមួយនឹង exponential backoff (ការបង្កើនពេលវេលាក្នុងការប៉ុនប៉ងម្តងៗ)។
- ការជូនដំណឹងតាមអ៊ីមែលនឹងត្រូវបានផ្ញើសម្រាប់ការបន្តដោយជោគជ័យ ការបរាជ័យ និងពេលដែលត្រូវផុតកំណត់ជិតមកដល់។

Email template IDs សម្រាប់ព្រឹត្តិការណ៍នៃវដ្ដជីវិតដែន៖

| Event | Template ID |
|---|---|
| ឈ្មោះដែនបានចុះបញ្ជី | `domain_registered` |
| ការច புதுப்பிக்கដែន | `domain_renewed` |
| ការ갱신បរាជ័យ | `domain_renewal_failed` |
| ដែននឹងផុតកំណត់ឆាប់ៗនេះ | `domain_expiring_soon` |

## Admin: ការចុះបញ្ជីដែនដោយដៃ (Manual domain registration)

ដើម្បីចុះបញ្ជីដែនជំនួសអតិថិជន ដោយមិនឱ្យពួកគេឆ្លងកាត់ដំណើរការទូទាត់ (checkout):

1. ចូលទៅកាន់ **Network Admin › Ultimate Multisite › Register Domain**
2. ជ្រើសរើសអតិថិជន ហើយបញ្ចូលឈ្មោះដែន
3. បំពេញព័ត៌មានទំនាក់ទំនងរបស់អ្នកចុះបញ្ជី (ឈ្មោះ, អាសយដ្ឋាន, ទូរស័ព្ទ)
4. ចុចលើ **Register**

កំណត់ត្រាដែននឹងត្រូវបានបង្កើតឡើង និងភ្ជាប់ទៅជាមួយគណនីរបស់អតិថិជន។
