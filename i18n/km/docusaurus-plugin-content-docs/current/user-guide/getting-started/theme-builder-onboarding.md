# Theme Builder Onboarding Flow

Superdav AI Agent v1.12.0 បង្ហាញពី **Theme Builder onboarding flow** ដែលជាការណែនាំប្រព័ន្ធមួយជួយអ្នកបង្កើត custom block theme ក្នុងអំឡុងពេលตั้งค่าដំបូង។ វាជំនួសរបៀប Site Builder ចាស់ដោយវិធីសាស្រ្តដែលអាចបត់បែនបាន និងមានជំនួយពី Agent។

## Theme Builder Onboarding Flow ជាអ្វី?

Theme Builder onboarding flow គឺជាកម្មវិធីជួយអ្នកកំណត់រចនាសម្ព័ន្ធបែបអន្តរកម្ម ដែល៖

- បង្ហាញអ្នកតាមរយៈការសម្រេចចិត្តផ្នែករចនា (ពណ៌, អក្សរ, ការរៀបចំ)
- ចងចាំចំណូលចិត្តនៃរូបភាពស្បែករបស់អ្នក
- បង្កើត custom block theme ដែលសមស្របនឹងតម្រូវការរបស់អ្នក
- ដំណើរការស្បែកដោយស្វ័យប្រវត្តិនៅពេលបញ្ចប់

Flow នេះត្រូវបានជំរុញដោយ **Setup Assistant agent** ដែលសួរនូវសំណួរដើម្បីបំភ្លឺ និងបង្កើត theme របស់អ្នកទៅជាបន្តបន្ទាប់។

## ការចាប់ផ្តើម Theme Builder Onboarding

### ការตั้งค่าលើកដំបូង (First-Run Setup)

នៅពេលអ្នកបើក Superdav AI Agent ជាលើកដំបូងនៅលើការដំឡើង WordPress ថ្មី អ្នកនឹងឃើញ៖

```
សូមស្វាគមន៍មកកាន់ Superdav AI Agent!

តើអ្នកចង់ធ្វើអ្វី?
1. បង្កើត theme ផ្ទាល់ខ្លួន (Theme Builder)
2. ប្រើ theme ដែលមានរួចហើយ
3. លុបចោលការตั้งค่าសម្រាប់ពេលនេះ
```

ជ្រើសរើស **"Build a custom theme"** ដើម្បីចូលទៅក្នុង Theme Builder onboarding flow។

### ការបើកដោយដៃ (Manual Activation)

អ្នកក៏អាចចាប់ផ្តើម Theme Builder onboarding បានគ្រប់ពេល ដោយស្នើសុំ៖

```
"Start the Theme Builder onboarding"
```

ឬ

```
"Help me create a custom theme"
```

## ជំហាននៃការណែនាំ (The Onboarding Steps)

### ជំហានទី ១: ការជ្រើសរើសរបៀប (Mode Selection)

Setup Assistant agent សួរអំពីចំណូលចិត្តរបស់អ្នក៖

```
តើអ្នកចង់បង្កើត theme របស់អ្នកដោយរបៀបណា?
- មានការណែនាំ (ខ្ញុំនឹងសួរសំណួរ និងបង្កើតវាឱ្យអ្នក)
- ធ្វើដោយខ្លួនឯង (ខ្ញុំនឹងបង្ហាញជម្រើស ហើយអ្នកសម្រេចចិត្ត)
```

**Guided mode** ត្រូវបានណែនាំសម្រាប់អ្នកប្រើប្រាស់ភាគច្រើន។ Agent នឹងផ្តល់អនុសាសន៍ផ្នែករចនាផ្អែកលើឧស្សាហកម្ម និងគោលដៅរបស់អ្នក។

### ជំហានទី ២: ការបញ្ជាក់អំពីគេហទំព័រ (Site Specification)

អ្នកនឹងត្រូវបានសួរអំពីគេហទំព័ររបស់អ្នក៖

- **គោលបំណងនៃគេហទំព័រ**: E-commerce (ពាណិជ្ជកម្មអេឡិចត្រូនិច), ប្លុក (blog), ផលប័ត្រ (portfolio), SaaS ជាដើម។
- **ទស្សនិកជនគោលដៅ**: អ្នកចូលមើលរបស់អ្នកជាអ្នកណា?
- **ពណ៌ម៉ាកយីហោ**: ពណ៌ចម្បង និងពណ៌បន្ទាប់បន្សំ (ឬ "ខ្ញុំមិនប្រាកដទេ")
- **សំនៀង (Tone)**: រហ័សរហួន, បែបច្នៃប្រឌិត, លេងសើច, សាមញ្ញបំផុត ជាដើម។

ព័ត៌មាននេះត្រូវបានរក្សាទុកនៅក្នុងអង្គចងចាំ **site_brief** របស់អ្នក ដែលអន្តរកម្មផ្សេងៗនឹងយោងទៅលើក្នុងវគ្គនាពេលអនាគត។

### ជំហានទី ៣៖ ការសម្រេចចិត្តនៃប្រព័ន្ធរចនា (Design System Decisions)

អន្តរកម្មនឹងណែនាំអ្នកអំពីការជ្រើសរើស design token ជាច្រើនដូចខាងក្រោម៖

- **ពុម្ពអក្សរ (Typography)**: ប្រភេទអក្សរ (serif, sans-serif, monospace) និងលំដាប់នៃទំហំ
- **ពណ៌ (Color palette)**: ពណ៌ចម្បង, ពណ៌បន្ទាប់បន្សំ, ពណ៌បញ្ជាក់ (accent), និងពណ៌ស្រាល (neutral colors)
- **ចន្លោះទំនេរ (Spacing)**: ការរៀបចំ Layout ដែលតូចច្រឡឹង, បែបធម្មតា, ឬបែបធំទូលាយ
- **ចលនា (Motion)**: អានិមន៍ និងការផ្លាស់ប្តូរ (ប្រសិនបើអ្នកចង់បាន)

### ជំហានទី ៤៖ ការបង្កើត Theme

អន្តរកម្ម Setup Assistant នឹងជួយអ្នកបង្កើត custom block theme ដោយមាន៖

- `theme.json` ដែលផ្ទុក design tokens ទាំងអស់របស់អ្នក
- Block templates សម្រាប់ Layout ទូទៅ (ទំព័រដើម, ប្លុក, ទំព័រទំនាក់ទំនង)
- រចនាប័ទ្ម block ដែលត្រូវនឹង design system របស់អ្នក
- Metadata នៃ Theme និងការប្រកាសគាំទ្រ WordPress

### ជំហានទី ៥៖ ការបើកដំណើរការ និងការផ្ទៀងផ្ទាត់ (Activation and Verification)

Theme នឹងត្រូវបានបើកដំណើរការដោយស្វ័យប្រវត្តិ ហើយអ្នកនឹងឃើញ៖

```
✓ Theme custom របស់អ្នកបានដំណើរការហើយ!
  ឈ្មោះ Theme: [ឈ្មោះគេហទំព័ររបស់អ្នក] Theme
  ពណ៌: [Primary] / [Secondary]
  ពុម្ពអក្សរ: [Font Family]

  ចូលទៅកាន់គេហទំព័ររបស់អ្នកដើម្បីមើលការរចនាថ្មី។
```

បន្ទាប់មក អ្នកអាចចូលទៅកាន់គេហទំព័ររបស់អ្នកដើម្បីផ្ទៀងផ្ទាត់ថា theme ដំណើរការត្រឹមត្រូវហើយឬនៅ។

## ការបញ្ជាក់លក្ខណៈពិសេសនៃគេហទំព័រ និងអង្គចងចាំ site_brief (Site Specification and site_brief Memory)

ក្នុងអំឡុងពេលចាប់ផ្តើមប្រើប្រាស់ (onboarding) អន្តរកម្មនឹងចាប់យកលក្ខណៈពិសេសនៃគេហទំព័ររបស់អ្នកនៅក្នុងប្រភេទអង្គចងចាំ **site_brief**។ នេះរួមបញ្ចូលទាំង៖

- គោលបំណង និងគោលដៅនៃគេហទំព័រ
- ទស្សនិកជនគោលដៅ
- ពណ៌ម៉ាកយីហោ និងសំនៀង (tone)
- ការពេញចិត្តលើការរចនា
- រចនាសម្ព័ន្ធខ្លឹមសារ

### ហេតុអ្វីបានជា site_brief មានសារៈសំខាន់

ក្នុងវគ្គនាពេលអនាគត អន្តរកម្មនឹងយោងទៅលើ site_brief របស់អ្នកដើម្បី៖

### ការរក្សាភាពស៊ីសង្វាក់នៃរចនាប័ទ្មក្នុងអំឡុងពេលផ្លាស់ប្តូរ

*   រក្សាភាពស៊ីសង្វាក់នៃរចនាប័ទ្មនៅពេលមានការផ្លាស់ប្តូរ។
*   ផ្តល់យោបល់មុខងារដែលស្របនឹងគោលបំណងនៃគេហទំព័ររបស់អ្នក។
*   ផ្តល់អនុសាសន៍ដែលយល់ពីបរិបទជាក់ស្តែង។
*   ជៀសវាងការសួរអំពីការរៀបចំឡើងវិញម្តងហើយម្តងទៀត។

### ការមើលសេចក្តីសង្ខេបគេហទំព័ររបស់អ្នក (Viewing Your site_brief)

អ្នកអាចសួរ Agent ដូចខាងក្រោម៖

```
"Show me my site brief"
```

ឬ

```
"What do you know about my site?"
```

Agent នឹងបង្ហាញព័ត៌មានលម្អិតអំពីការកំណត់រចនាសម្ព័ន្ធគេហទំព័រដែលអ្នកបានរក្សាទុក។

## ការកែសម្រួលបន្ទាប់ពីការចាប់ផ្តើម (Customizing After Onboarding)

បន្ទាប់ពីការបណ្តុះបណ្តាល Theme Builder ត្រូវបានបញ្ចប់ អ្នកអាច៖

### ប្រើជំនាញរចនាប័ទ្មប្រព័ន្ធរចនា (Use the Design System Aesthetics Skill)

ស្នើសុំការកែលម្អផ្នែករចនា៖

```
"Refine the typography to be more modern"
```

ឬ

```
"Adjust the color palette to be warmer"
```

**ជំនាញ Design System Aesthetics** នឹងណែនាំអ្នកឆ្លងកាត់ការធ្វើបច្ចុប្បន្នភាពផ្នែករចនាដែលផ្តោតជាក់លាក់។

### កែសម្រួល theme.json ដោយផ្ទាល់ (Edit theme.json Directly)

សម្រាប់អ្នកប្រើប្រាស់ขั้นสูง សូមកែសម្រួលឯកសារ `/wp-content/themes/[theme-name]/theme.json` ដើម្បីកែសម្រួល៖

*   color tokens
*   typography scales
*   spacing values
*   border and shadow definitions

### បង្កើត قالب Block ផ្ទាល់ខ្លួន (Create Custom Block Templates)

ប្រើ WordPress block editor ដើម្បីបង្កើត قالبផ្ទាល់ខ្លួនសម្រាប់៖

*   homepage layouts
*   blog post pages
*   product pages
*   contact forms

## ការប្រៀបធៀប: កាលពីមុន vs. ថ្មីនៃការចាប់ផ្តើម (Comparison: Old vs. New Onboarding)

| លក្ខណៈពិសេស | Site Builder (ចាស់) | Theme Builder (ថ្មី) |
| :--- | :--- | :--- |
| វិធីសាស្ត្ររៀបចំ | ទម្រង់បែបបទផ្អែកលើ Wizard | ការសន្ទនាដែលដឹកនាំដោយ Agent |
| ការបង្កើត Theme | Template មានកំណត់ | Scaffolding ដែលអាចប្ដូរបានតាមតម្រូវការ |
| Design tokens | បញ្ចូលដោយដៃ | ការសម្រេចចិត្តដែលត្រូវបានណែនាំ |
| ភាពបត់បែន | ជម្រើសថេរ | អាចកែសម្រួលបាន |
| ការធ្វើបច្ចុប្បន្នភាពនាពេលអនាគត | មិនត្រូវបានយោងលើទេ | រក្សាទុកក្នុង site_brief |

## ការដោះស្រាយបញ្ហា (Troubleshooting)

**ដំណើរការចាប់ផ្តើមមិនបានបញ្ចប់ (The onboarding flow didn't complete)**
*   ចាប់ផ្តើមដំណើរការឡើងវិញ៖ "Start the Theme Builder onboarding"
*   ពិនិត្យមើលថាការដំឡើង WordPress របស់អ្នកទាន់សម័យឬអត់។
*   ផ្ទៀងផ្ទាត់ថា Setup Assistant agent ត្រូវបានបើកដំណើរការហើយ។

**site_brief របស់ខ្ញុំមិនត្រូវបានប្រើទេ**
- បញ្ជាក់ថា Agent មានការចូលប្រើប្រាស់ memory (ការចងចាំ) ឬអត់។
- សួរ Agent ឱ្យ "recall my site brief" (ទាញយកសេចក្តីសង្ខេបគេហទំព័ររបស់ខ្ញុំមកវិញ)។
- စពិនិត្យមើលថាតើ memory ត្រូវបានបើកនៅក្នុងការកំណត់របស់អ្នកឬអត់។

**Theme ដែលបង្កើតឡើងមិនត្រូវនឹងចំណូលចិត្តរបស់ខ្ញុំទេ**
- ប្រើជំនាញ Design System Aesthetics ដើម្បីកែលម្អវា។
- សួរ Agent ឱ្យ "regenerate the theme with [specific changes]" (បង្កើត theme ឡើងវិញជាមួយនឹង [ការផ្លាស់ប្តូរជាក់លាក់])។
- កែសម្រួល file `theme.json` ដោយផ្ទាល់ដើម្បីគ្រប់គ្រងបានត្រឹមត្រូវ។

## ជំហានបន្ទាប់

បន្ទាប់ពីបញ្ចប់ការណែនាំ Theme Builder:

1. **ផ្ទៀងផ្ទាត់គេហទំព័ររបស់អ្នក**: ចូលទៅកាន់គេហទំព័ររបស់អ្នកដើម្បីមើល theme ថ្មី។
2. **កែលម្អការរចនា**: ប្រើជំនាញ Design System Aesthetics ដើម្បីធ្វើការកែតម្រូវ។
3. **បន្ថែមខ្លឹមសារ**: ចាប់ផ្តើមបង្កើតខ្លឹមសារសម្រាប់គេហទំព័ររបស់អ្នក។
4. **ស្វែងយល់ពីសមត្ថភាព**: រៀនអំពីសមត្ថភាពផ្សេងទៀតរបស់ Agent ដូចជា `scaffold-block-theme` និង `activate-theme`។
