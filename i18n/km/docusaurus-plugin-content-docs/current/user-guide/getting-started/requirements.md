# តម្រូវការសម្រាប់ Ultimate Multisite

មុននឹងទិញ Ultimate Multisite អ្នកប្រហែលជាចង់ដឹងថា តើមានតម្រូវការអ្វីខ្លះដើម្បីដំណើរការ plugin របស់យើង។ ជាសំណាងល្អ មានតម្រូវការមូលដ្ឋានតែប៉ុណ្ណោះសម្រាប់ការដំណើរការ Ultimate Multisite៖ **ឈ្មោះដែន (domain name)**, **អ្នកផ្តល់សេវា hosting** ដែលល្អ និងការដំឡើង WordPress **Multisite**។ នោះគឺគ្រប់យ៉ាងដែលអ្នកត្រូវការ!

## ឈ្មោះដែន (Domain Name)

ទីមួយ សូមនិយាយអំពីឈ្មោះដែន។

នៅពេលជ្រើសរើសឈ្មោះដែន សូមគិតពីផ្នែកជាក់លាក់របស់អ្នក (niche) និងអ្វីដែលអ្នកនឹងបង្កើតជាមួយ Ultimate Multisite៖ តើអ្នកចង់បង្កើតភ្នាក់ងារ (agency) បង្កើតគេហទំព័រដូចហាងលក់ដូរ? ឬប្រហែលជាការផ្តល់សេវា hosting សម្រាប់ WordPress? អ្វីៗគឺអាចធ្វើទៅបានទាំងអស់ជាមួយ Ultimate Multisite។

Plugin របស់យើងផ្តល់នូវសេវាកម្មជាការជាវ (subscription services), ការប្រមូលប្រាក់, ទម្រង់ checkout, ប័ណ្ណបញ្ចុះតម្លៃ និងការទំនាក់ទំនងជាមួយអតិថិជន។ ការងារច្រើនដែលទាមទារសម្រាប់ការដំឡើង កំណត់ការកំណត់រចនាសម្ព័ន្ធ និងការថែរក្សា WordPress Multisite ត្រូវបានសម្រួលដោយ Ultimate Multisite។

ផ្នែកណាក៏ដោយដែលអ្នកជ្រើសរើស អាចត្រូវបានគ្រប់គ្រងដោយប្រើ plugin របស់យើង ដរាបណាអ្នកលក់គេហទំព័រជាសេវា! សូមប្រាកដថាអ្នកជ្រើសរើសឈ្មោះដែនដែលត្រូវនឹងផែនការអាជីវកម្មរបស់អ្នក។

## អ្នកផ្តល់សេវា hosting (Hosting Provider)

ឥឡូវនេះ នៅពេលអ្នកមានឈ្មោះដែនដែលទាក់ទាញដែលតំណាងឱ្យផ្នែក និងអាជីវកម្មរបស់អ្នកហើយ អ្នកនឹងត្រូវជ្រើសរើសក្រុមហ៊ុន web hosting ល្អដើម្បីបង្ហោះបណ្តាញរបស់អ្នកជាមួយ Ultimate Multisite។ មានជម្រើសល្អៗនៅលើទីផ្សារមួយចំនួន។ ក្រុមហ៊ុន hosting ខ្លះអាចរួមបញ្ចូលគ្នាបានយ៉ាងល្អជាមួយ Ultimate Multisite ដែលនេះអាចជាកត្តាកំណត់អាស្រ័យលើអ្វីដែលអ្នកគ្រោងនឹងបង្កើត។

Ultimate Multisite អាចដំណើរការជាមួយผู้ให้บริการโฮสติ้งทุกรายได้ครับ ความแตกต่างเพียงอย่างเดียวคือ ผู้ให้บริการโฮสติ้งบางราย เช่น _WPMU DEV Hosting_ , _WP Engine_ , _Closte_ , _Cloudways_ , _Gridpane_ และ _Runcloud_ จะมีการเชื่อมต่อที่แน่นหนากว่าสำหรับการแมปโดเมน สำหรับผู้ให้บริการโฮสติ้งรายอื่น คุณอาจจะต้องทำการแมปโดเมนแบบกำหนดเองด้วยตนเอง นอกจากนี้ การเลือกผู้ให้บริการโฮสติ้งที่รองรับ AutoSSL ก็เป็นความคิดที่ดีครับ _Closte_ , _Runcloud_ และ _cPanel_ รองรับฟีเจอร์นี้

สำหรับข้อกำหนดของเซิร์ฟเวอร์ ไม่มีสูตรตายตัว เพราะมันจะขึ้นอยู่กับฟีเจอร์และฟังก์ชันที่คุณจะนำเสนอ รวมถึงขนาดเครือข่ายของคุณ การใช้ Shared hosting อาจเป็นจุดเริ่มต้นที่ดี แต่เมื่อเครือข่ายของคุณเริ่มเติบโต คุณน่าจะต้องเปลี่ยนไปใช้ Dedicated VPS ครับ

## WordPress Multisite

ทีนี้ เรามาพูดถึงการติดตั้ง WordPress Multisite กันสักหน่อยครับ

WordPress Multisite ช่วยให้คุณสามารถมีเว็บไซต์หลายๆ เว็บไซต์อยู่ในการติดตั้งเดียวได้ ฟีเจอร์นี้มีมาให้ในตัว แต่โดยปกติแล้วมันจะยังไม่ได้เปิดใช้งาน คุณจำเป็นต้องเปิดใช้งานฟีเจอร์นี้ทั้งในส่วนของ WordPress และใน control panel ของเว็บโฮสติ้งของคุณ

การมี WordPress ที่เปิดใช้งาน Multisite **คือพื้นฐานสำหรับ Ultimate Multisite** เนื่องจากวัตถุประสงค์หลักของ Ultimate Multisite คือการช่วยให้ผู้ใช้จัดการและขายเว็บไซต์เป็นบริการ (as a service) ดังนั้นสิ่งนี้จะต้องทำบน WordPress ที่เปิดใช้งาน Multisite แล้วครับ

เรามี **บทความเชิงลึก** ที่สอนทีละขั้นตอนว่าต้องทำอะไรบ้างเพื่อเปิดใช้งานฟีเจอร์ Multisite บนการติดตั้ง WordPress ของคุณ

หลังจากที่คุณเปิดใช้งาน Multisite บน WordPress แล้ว ก็ถึงเวลาที่จะติดตั้ง Ultimate Multisite เพื่อให้ติดตั้งได้อย่างถูกต้อง คุณต้องมีสิ่งเหล่านี้:

  * WordPress: v5.3+ (แนะนำ: เวอร์ชันที่เสถียรล่าสุด)

  * PHP: 7.4.x (รองรับเวอร์ชัน > 8.0 กำลังจะมาเร็วๆ นี้)

  * MySQL: v5+ (แนะนำ: 5.6 ในกรณีที่คุณไม่มี 8.0 ให้ใช้งานบนผู้ให้บริการโฮสติ้งของคุณ)

តម្រូវការផ្នែកទន់ (software requirements) ទាំងនេះគឺជាតម្រូវការតែមួយគត់សម្រាប់ Ultimate Multisite។

សូមចងចាំថា Ultimate Multisite អាចដំណើរការជាមួយ PHP 8.0 ប៉ុន្តែយើងសូមណែនាំឱ្យប្រើប្រាស់វាជាមួយ PHP 7.4.x។

លើសពីនេះ យើងសូមណែនាំឱ្យដំឡើងវាលើ **domain សំខាន់ៗ (main domains)** ជាជាង subdomain ឬ subdirectory។ វាអាចដំណើរការលើ subdomain/subdirectory បានដែរ ប៉ុន្តែវាអាចនឹងបង្ហាញកំហុសខ្លះ។

សម្រាប់មគ្គុទ្ទេសក៍លម្អិតអំពីរបៀបដំឡើង Ultimate Multisite សូម **ចុចទីនេះ**។
