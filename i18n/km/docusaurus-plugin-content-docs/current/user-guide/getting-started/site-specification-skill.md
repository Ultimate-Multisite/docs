# ทักษะการกำหนดสเปคเว็บไซต์ (Site Specification Skill)

**ทักษะการกำหนดสเปคเว็บไซต์ (Site Specification skill)** คือแนวทางที่เป็นระบบในการเก็บข้อมูลเกี่ยวกับเป้าหมาย กลุ่มเป้าหมาย และเอกลักษณ์ของแบรนด์เว็บไซต์ของคุณ ข้อมูลนี้จะถูกเก็บไว้ในหน่วยความจำที่ชื่อว่า **site_brief** ซึ่งเอเจนต์ต่างๆ จะอ้างถึงข้ามเซสชันเพื่อมอบความช่วยเหลือที่สอดคล้องกันและเข้าใจบริบท

## Site Specification คืออะไร?

Site Specification คือกระบวนการบันทึกข้อมูลดังต่อไปนี้:

- **วัตถุประสงค์ของเว็บไซต์ (Site purpose)**: เว็บไซต์ของคุณทำอะไร และมีอยู่เพื่ออะไร
- **กลุ่มเป้าหมาย (Target audience)**: ใครเข้ามาเยี่ยมชมเว็บไซต์ของคุณ และพวกเขาต้องการอะไร
- **เอกลักษณ์ของแบรนด์ (Brand identity)**: สี โทนเสียง และสไตล์ภาพของคุณ
- **เป้าหมายทางธุรกิจ (Business goals)**: ความสำเร็จสำหรับเว็บไซต์ของคุณมีหน้าตาเป็นอย่างไร
- **โครงสร้างเนื้อหา (Content structure)**: เว็บไซต์ของคุณถูกจัดระเบียบไว้อย่างไร

ข้อมูลนี้จะกลายเป็น **site_brief** ของคุณ ซึ่งเป็นหน่วยความจำที่คงอยู่ ที่เอเจนต์ใช้เพื่อทำความเข้าใจบริบทของเว็บไซต์ของคุณ

## ทำไมต้องใช้ Site Specification?

### ความสม่ำเสมอในทุกเซสชัน (Consistency Across Sessions)

หากไม่มี site_brief คุณจะต้องอธิบายวัตถุประสงค์ของเว็บไซต์ซ้ำทุกครั้งที่คุณเริ่มเซสชันใหม่ แต่เมื่อมีมัน เอเจนต์จะเข้าใจทันทีว่า:

- เป้าหมายและกลุ่มเป้าหมายของเว็บไซต์คุณคืออะไร
- สีและโทนเสียงของแบรนด์คุณเป็นอย่างไร
- โครงสร้างเนื้อหาของคุณเป็นแบบไหน
- วัตถุประสงค์ทางธุรกิจของคุณคืออะไร

### คำแนะนำที่ดีขึ้น (Better Recommendations)

เอเจนต์จะใช้ site_brief ของคุณเพื่อ:

- แนะนำฟีเจอร์ที่สอดคล้องกับวัตถุประสงค์ของเว็บไซต์
- แนะนำโครงสร้างเนื้อหาที่ตรงกับเป้าหมายของคุณ
- เสนอการออกแบบที่สอดคล้องกับแบรนด์ของคุณ
- หลีกเลี่ยงการแนะนำฟีเจอร์ที่ไม่เข้ากัน

### การเริ่มต้นใช้งานที่รวดเร็ว (Faster Onboarding)

เอเจนต์ใหม่ (หรือเอเจนต์ในเซสชันใหม่) สามารถเรียนรู้ได้อย่างรวดเร็วโดยการอ่าน site_brief ของคุณ แทนที่จะต้องถามคำถามเพื่อชี้แจงรายละเอียด

## การเริ่มใช้ Site Specification

### ระหว่างการเริ่มต้นใช้งาน Theme Builder (During Theme Builder Onboarding)

ทักษะ Site Specification จะถูกเริ่มขึ้นโดยอัตโนมัติในระหว่าง **กระบวนการเริ่มต้นใช้งาน Theme Builder** เอเจนต์ Setup Assistant จะถามคำถามและสร้าง site_brief ของคุณให้เสร็จสมบูรณ์

អ្នកអាចចាប់ផ្តើមការបញ្ជាក់លក្ខណៈពិសេសនៃគេហទំព័រ (Site Specification) គ្រប់ពេលដែលអ្នកចង់៖

```
"តោះ! សូមកំណត់លក្ខណៈពិសេសនៃគេហទំព័ររបស់ខ្ញុំ"
```

ឬ

```
"ជួយខ្ញុំបង្កើតសេចក្តីសង្ខេបគេហទំព័រ (site brief)"
```

## ដំណើរការនៃការបញ្ជាក់លក្ខណៈពិសេសនៃគេហទំព័រ (The Site Specification Process)

### ជំហានទី ១៖ គោលបំណងនៃគេហទំព័រ (Site Purpose)

Agent នឹងសួរថា៖

```
តើគោលបំណងចម្បងនៃគេហទំព័ររបស់អ្នកគឺជាអ្វី?
- butik ទិញលក់អនឡាញ (E-commerce store)
- ប្លុក ឬគេហទំព័រមាតិកា (Blog or content site)
- ប្រវត្តិរូប ឬការបង្ហាញស្នាដៃ (Portfolio or showcase)
- កម្មវិធី SaaS (SaaS application)
- សហគមន៍ ឬវេទិកាពិភាក្សា (Community or forum)
- ផ្សេងទៀត៖ [ពណ៌នា]
```

អ្នកអាចជ្រើសរើសប្រភេទ ឬពណ៌នាគោលបំណងរបស់អ្នកដោយខ្លួនឯង។

### ជំហានទី ២៖ អ្នកប្រើប្រាស់គោលដៅ (Target Audience)

```
តើនរណាជាអ្នកប្រើប្រាស់ចម្បងរបស់អ្នក?
- អ្នកប្រើប្រាស់ទូទៅ / ប្រជាជនទូទៅ (Consumers / general public)
- អ្នកជំនាញអាជីវកម្ម (Business professionals)
- អ្នកអភិវឌ្ឍន៍ / អ្នកប្រើប្រាស់បច្ចេកទេស (Developers / technical users)
- សិស្ស / អ្នកអប់រំ (Students / educators)
- ផ្សេងទៀត៖ [ពណ៌នា]

តម្រូវការសំខាន់ៗរបស់ពួកគេគឺជាអ្វី?
```

### ជំហានទី ៣៖ អត្តសញ្ញាណម៉ាកយីហោ (Brand Identity)

```
ពណ៌ម៉ាករបស់អ្នកគឺជាអ្វីខ្លះ?
- ពណ៌ចម្បង៖ [ជ្រើសរើសពណ៌ ឬលេខកូដ Hex]
- ពណ៌បន្ទាប់បន្សំ៖ [ជ្រើសរើសពណ៌ ឬលេខកូដ Hex]
- ពណ៌បញ្ជាក់ (Accent color)៖ [ជម្រើស]

តើអ្នកនឹងពិពណ៌នាអំពីសម្លេងម៉ាករបស់អ្នកដូចម្តេច?
- រួមសម្បត្តិ / ក្រុមហ៊ុន (Professional / corporate)
- បែបច្នៃប្រឌិត / សិល្បៈ (Creative / artistic)
- លេងសើច / ទូទៅ (Playful / casual)
- រចនាបថតិចតួច / ទំនើប (Minimal / modern)
- កក់ក្តៅ / មិត្តភាព (Warm / friendly)
```

### ជំហានទី ៤៖ គោលដៅអាជីវកម្ម (Business Goals)

```
ភាពជោគជ័យសម្រាប់គេហទំព័ររបស់អ្នកមើលទៅដូចម្តេច?
- បង្កើត leads (Generate leads)
- លក់ផលិតផល (Sell products)
- កសាងសហគមន៍ (Build community)
- ចែករំលែកចំណេះដឹង (Share knowledge)
- បង្កើតភាពជឿជាក់ (Establish authority)
- ផ្សេងទៀត៖ [ពណ៌នា]

តើអ្វីជាសូចនាករចម្បងរបស់អ្នក?
- ប្រាក់ចំណូល (Revenue)
- ការចូលរួមពីអ្នកប្រើប្រាស់ (User engagement)
- ការផ្សព្វផ្សាយមាតិកា (Content reach)
- ការបំប្លែងការទិញ (Conversions)
- ផ្សេងទៀត
```

### ជំហានទី ៥៖ រចនាសម្ព័ន្ធមាតិកា (Content Structure)

```
មាតិការបស់អ្នកត្រូវបានរៀបចំយ៉ាងដូចម្តេច?
- បែបរាបស្មើ (Flat - មាតិកាទាំងអស់ស្ថិតក្នុងកម្រិតតែមួយ)
- បែបឋានានុក្រម (Hierarchical - ប្រភេទ និងអនុប្រភេទ)
- បែបតាមពេលវេលា (Chronological - ដូចជាប្លុក)
- បែបផ្អែកលើផលិតផល (Product-based - កាតាឡុក)
- ផ្សេងទៀត៖ [ពណ៌នា]

តើអ្នកប្រើប្រាស់ប្រភេទមាតិកាអ្វីខ្លះ?
- ប្លុក (Blog posts)
- ទំព័រផលិតផល (Product pages)
- សិក្ខាសាលាករណីសិក្សា (Case studies)
- ការបណ្តុះបញ្ហោ (Documentation)
- វីដេអូ (Videos)
- ផ្សេងទៀត
```

## ការចងចាំ site_brief របស់អ្នក (Your site_brief Memory)

បន្ទាប់ពីបំពេញការបញ្ជាក់លក្ខណៈពិសេសនៃគេហទំព័ររួច រាល់ព័ត៌មានរបស់អ្នកនឹងត្រូវបានរក្សាទុកជា **site_brief** នៅក្នុងអង្គចងចាំ (memory) របស់ agent។ វាគឺជា

## ការមើល និងធ្វើបច្ចុប្បន្នភាព site_brief របស់អ្នក

### មើល site_brief របស់អ្នក

```
"Show me my site brief"
```

ឬ

```

```

### ធ្វើបច្ចុប្បន្នភាព site_brief របស់អ្នក

ប្រសិនបើគេហទំព័ររបស់អ្នកផ្លាស់ប្តូរ អ្នកអាចធ្វើបច្ចុប្បន្នភាពវាបាន៖

```
"Update my site brief: we're now targeting B2B customers"
```

ឬ

```
"Refresh my site specification"
```

នេះនឹងដំណើរការជំនួស Site Specification skill ឡើងវិញ ដោយប្រើព័ត៌មានដែលមានស្រាប់របស់អ្នកជាចំណុចចាប់ផ្តើម។

## តើ Agent ប្រើ site_brief យ៉ាងដូចម្តេច?

### ការណែនាំផ្នែករចនា (Design Recommendations)

នៅពេលអ្នកសុំការផ្លាស់ប្តូរផ្នែករចនាសម្ព័ន្ធ អ្នកនឹងឃើញថា Agent យោងទៅលើ site_brief របស់អ្នក៖

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### ការផ្តល់យោបល់មុខងារ (Feature Suggestions)

Agent នឹងណែនាំមុខងារដែលស្របទៅនឹងគោលដៅរបស់អ្នក៖

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### រចនាសម្ព័ន្ធខ្លឹមសារ (Content Structure)

Agent នឹងស្នើការរៀបចំខ្លឹមសារដោយផ្អែកលើរចនាសម្ព័ន្ធរបស់អ្នក៖

អ្នក: "ខ្ញុំគួររៀបចំបញ្ជីប្លុករបស់ខ្ញុំដូចម្តេច?"
ជំនួយការ: "ដោយសារអ្នកប្រើផលិតផលកាតាឡុកដែលមានប្លុក (product catalog with blog) ខ្ញុំសូមណែនាំឱ្យរៀបចំការបង្ហោះតាមប្រភេទផលិតផល ហើយបង្កើត 'คู่មគ្គបង្ហាញពីរបៀបរចនា' (styling guides) ដែលបង្ហាញផលិតផលច្រើនក្នុងពេលតែមួយ។"

## ការអនុវត្តល្អបំផុត (Best Practices)

### កំណត់ឲ្យជាក់លាក់ (Be Specific)

ជំនួសឱ្យការនិយាយថា "អ្នកទស្សនាទូទៅ" សូមពិពណ៌នាអំពីអ្នកទស្សនាពិតប្រាកដរបស់អ្នក៖

- ✓ "ស្ត្រីអាយុ ២៥-៤៥ ឆ្នាំ ដែលចាប់អារម្មណ៍លើម៉ូដប្រកបដោយនិរន្តរភាព (sustainable fashion)"
- ✗ "គ្រប់គ្នា"

### អាប់เดតជាទៀងទាត់ (Update Regularly)

នៅពេលដែលគេហទំព័ររបស់អ្នកវិវត្តន៍ សូមធ្វើបច្ចុប្បន្នភាព site_brief របស់អ្នក៖

- នៅពេលអ្នកផ្លាស់ប្តូរទៅរកក្រុមអ្នកទស្សនាថ្មី
- នៅពេលអ្នកបន្ថែមខ្សែផលិតផលថ្មីៗ
- នៅពេលអត្តសញ្ញាណម៉ាកយីហោរបស់អ្នកផ្លាស់ប្តូរ
- នៅពេលគោលដៅអាជីវកម្មរបស់អ្នកផ្លាស់ប្តូរ

### ប្រើពាក្យពេចន៍ដូចគ្នា (Use Consistent Terminology)

ប្រើពាក្យដូចគ្នានៅក្នុងគ្រប់វគ្គ៖

- ✓ តែងតែនិយាយថា "គ្រឿងអលង្ការនិរន្តរភាព" (មិនមែន "គ្រឿងអលង្ការដែលជាមិត្តនឹងបរិស្ថាន" និង "ផលិតផលបៃតង")
- ✓ តែងតែសំដៅទៅលើអ្នកទស្សនាដោយប្រើពាក្យដូចគ្នា

### រួមបញ្ចូលបរិបទ (Include Context)

ផ្តល់ព័ត៌មានផ្ទៃខាងក្រោយដែលជួយឱ្យជំនួយការយល់ពីការសម្រេចចិត្តរបស់អ្នក៖

- "យើងកំពុងកំណត់គោលដៅអ្នកជំនាញដែលឲ្យតម្លៃលើគុណភាពជាងតម្លៃ"
- "អ្នកទស្សនារបស់យើងពូកែបច្ចេកវិទ្យា និងរំពឹងលើការរចនាបែបទំនើប"
- "យើងជាក្រុមហ៊ុនថ្មីដែលចាប់ផ្តើមដោយដើមទុនតិចតួច (bootstrapped startup) ដូច្នេះយើងត្រូវការដំណោះស្រាយដែលមានប្រសិទ្ធភាពចំណាយ"

## សមស្របនឹងការណែនាំ Theme Builder (Relationship to Theme Builder Onboarding)

ជំនាញ Site Specification ត្រូវបានបញ្ចូលទៅក្នុង **ដំណើរការណែនាំ Theme Builder**។ នៅពេលអ្នកបញ្ចប់ការណែនាំ ប្លុក site_brief របស់អ្នកនឹងត្រូវបានបង្កើតដោយស្វ័យប្រវត្តិជាមួយនឹងព័ត៌មានដែលអ្នកបានផ្តល់ឱ្យ។

អ្នកក៏អាចដំណើរការ Site Specification ដោយឯករាជ្យ ប្រសិនបើអ្នកចង់៖

- កែលម្អការកំណត់ជាក់លាក់របស់អ្នកបន្ទាប់ពីការរៀបចំដំបូង
- អាប់เดត site brief របស់អ្នកនៅពេលគេហទំព័ររបស់អ្នកវិវត្តន៍
- បង្កើតការកំណត់ជាក់លាក់លម្អិតមុននឹងចាប់ផ្តើម Theme Builder

## ការដោះស្រាយបញ្ហា (Troubleshooting)

**site_brief របស់ខ្ញុំមិនត្រូវបានប្រើទេ**
- ផ្ទៀងផ្ទាត់ថាជំនួយការមានសិទ្ធិចូលប្រើប្រាស់ memory (ការចងចាំ) ឬទេ?
- សួរជំនួយការឱ្យ "រំលឹក site brief របស់ខ្ញុំ" (recall my site brief)។
- စពិនិត្យមើលថាតើ memory ត្រូវបានបើកនៅក្នុងការកំណត់របស់អ្នកឬអត់។

**ខ្ញុំចង់ចាប់ផ្តើមថ្មីជាមួយ site\_brief ថ្មី**
- សួរ Agent៖ "Clear my site brief and start fresh" (លុប site brief របស់ខ្ញុំចេញ ហើយចាប់ផ្តើមថ្មី)
- បន្ទាប់មកដំណើរការ Site Specification ម្តងទៀត

**Agent កំពុងផ្តល់អនុសាសន៍ដែលមិនត្រូវនឹង site\_brief របស់ខ្ញុំទេ**
- សួរ Agent ឱ្យ "review my site brief" (ពិនិត្យមើល site brief របស់ខ្ញុំ)
- អាប់ដេត site\_brief របស់អ្នក ប្រសិនបើវាលែងទាន់សម័យ
- ផ្តល់បរិបទបន្ថែមនៅក្នុងសំណើរបស់អ្នក

## ជំហានបន្ទាប់

បន្ទាប់ពីកំណត់ការបញ្ជាក់ Site Specification របស់អ្នករួច៖

1. **ប្រើ Theme Builder**: បង្កើត theme ដែលប្ដូរតាម site\_brief របស់អ្នក
2. **កែលម្អការរចនា (Refine Design)**: ប្រើជំនាញ Design System Aesthetics ដើម្បីធ្វើការរចនាលម្អិត
3. **គ្រោងខ្លឹមសារ (Plan Content)**: សួរ Agent សម្រាប់អនុសាសន៍អំពីរចនាសម្ព័ន្ធខ្លឹមសារ
4. **បង្កើតមុខងារ (Build Features)**: ស្នើសុំមុខងារដែលស្របនឹងគោលដៅអាជីវកម្មរបស់អ្នក
