# Gratis AI Agent ការកំណត់ {#gratis-ai-agent-settings}

អេក្រង់ **Settings → Advanced** នៅក្នុង Gratis AI Agent ផ្តល់ការកំណត់កម្រិតអ្នកគ្រប់គ្រង សម្រាប់ការរួមបញ្ចូល backend។ ទំព័រនេះពន្យល់អំពីការបញ្ជូន feedback បន្ត, key របស់អ្នកផ្តល់សេវាស្វែងរក, ការរៀបចំសេវា Superdav ដែលគ្រប់គ្រង, ការគ្រប់គ្រង Google Calendar, ការកំណត់ TextBee SMS និង feature flags ទូទាំងបណ្តាញ។

## ការចូលទៅកាន់ការកំណត់ {#accessing-settings}

1. នៅក្នុង WordPress admin សូមចូលទៅ **Gratis AI Agent → Settings**។
2. ចុចផ្ទាំង **Advanced**។

## ការកំណត់ Feedback Endpoint {#feedback-endpoint-configuration}

feedback endpoint ទទួលសំណើ POST ពី AI agent នៅពេលណាដែលអ្នកប្រើដាក់ feedback តាមប៊ូតុងមេដៃចុះក្រោម, បដា auto-prompt ឬពាក្យបញ្ជា `/report-issue`។

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | URL ដែលទទួលការដាក់ feedback ជាសំណើ HTTP POST ជាមួយ JSON body។ |
| **Feedback API Key** | bearer token ដែលផ្ញើក្នុង `Authorization` header នៃសំណើ feedback នីមួយៗ។ ទុកទទេ ប្រសិនបើ endpoint របស់អ្នកមិនត្រូវការការផ្ទៀងផ្ទាត់។ |

### JSON Payload ដែលរំពឹងទុក {#expected-json-payload}

feedback endpoint របស់អ្នកត្រូវតែទទួល JSON body ដែលមានយ៉ាងហោចណាស់វាលខាងក្រោម៖

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

អាចមានវាលបន្ថែមក្នុង payload អាស្រ័យលើបរិបទនៃការសន្ទនា។

### តម្លៃ `triage_category` {#triagecategory-values}

ស្រទាប់ triage របស់ AI កំណត់តម្លៃមួយក្នុងចំណោមតម្លៃខាងក្រោមទៅ `triage_category` មុនពេលបញ្ជូន payload បន្ត៖

| Value | Meaning |
|---|---|
| `factual_error` | ជំនួយការបានផ្តល់ព័ត៌មានពិតដែលមិនត្រឹមត្រូវ។ |
| `unhelpful_answer` | ចម្លើយត្រឹមត្រូវផ្នែកបច្ចេកទេស ប៉ុន្តែមិនមានប្រយោជន៍។ |
| `inappropriate_content` | ចម្លើយមានមាតិកាដែលមិនគួរបង្ហាញដល់អ្នកប្រើ។ |
| `other` | feedback មិនត្រូវនឹងប្រភេទដែលស្គាល់។ |

### ការផ្ទៀងផ្ទាត់ {#authentication}

ប្រសិនបើ endpoint របស់អ្នកត្រូវការការផ្ទៀងផ្ទាត់ សូមកំណត់វាល **Feedback API Key** ជា bearer token របស់អ្នក។ agent ផ្ញើ៖

```
Authorization: Bearer <your-api-key>
```

ប្រសិនបើវាល **Feedback API Key** ទទេ នោះនឹងមិនផ្ញើ `Authorization` header ទេ។

### ការបិទការប្រមូល Feedback {#disabling-feedback-collection}

ទុកវាល **Feedback Endpoint URL** និង **Feedback API Key** ទទេ។ ប៊ូតុងមេដៃចុះក្រោម និង feedback UI នៅតែបង្ហាញដល់អ្នកប្រើ ប៉ុន្តែការដាក់ស្នើនឹងមិនត្រូវបានបញ្ជូនបន្តទៅសេវាខាងក្រៅណាមួយទេ។

## Brave Search API Key {#brave-search-api-key}

នៅលើផ្ទាំង **Advanced** ផងដែរ វាល **Brave Search API Key** បើកសមត្ថភាព [ការស្វែងរកលើអ៊ីនធឺណិត](../configuration/internet-search)។

| Field | Description |
|---|---|
| **Brave Search API Key** | API key របស់អ្នកពី Brave Search developer dashboard។ ត្រូវការដើម្បីបើកការស្វែងរកលើអ៊ីនធឺណិតក្នុង AI assistant។ |

ស្លាកវាលមានតំណដែលអាចចុចបានទៅកាន់ទំព័រចុះឈ្មោះ Brave Search API។ ទុកទទេ ដើម្បីបិទការស្វែងរកលើអ៊ីនធឺណិត។

សូមមើល [ការស្វែងរកលើអ៊ីនធឺណិត](../configuration/internet-search) សម្រាប់ឯកសារអ្នកប្រើចុងក្រោយអំពីមុខងារនេះ។

## សេវា Superdav ដែលគ្រប់គ្រង {#managed-superdav-service}

Superdav AI Agent v1.18.0 បន្ថែម endpoints សម្រាប់សេវា Superdav ដែលគ្រប់គ្រង និងការផ្តល់ការតភ្ជាប់ដោយស្វ័យប្រវត្តិ សម្រាប់គេហទំព័រដែលគាំទ្រ។ ប្រើការគ្រប់គ្រងទាំងនេះ នៅពេលគេហទំព័ររបស់អ្នកគួរតភ្ជាប់ទៅអ្នកផ្តល់សេវាដែលបានបង្ហោះ ជំនួសឱ្យ service endpoint ដែលកំណត់ដោយដៃ។

| Field | Description |
|---|---|
| **Managed Superdav Service** | បើកការតភ្ជាប់ទៅសេវា Superdav ដែលបានបង្ហោះ សម្រាប់គេហទំព័រដែលគាំទ្រ។ |
| **Provision Connection** | ចាប់ផ្តើមការផ្តល់ endpoint និងព័ត៌មានសម្ងាត់ដោយស្វ័យប្រវត្តិ។ ប្រើវាបន្ទាប់ពីបញ្ជាក់ថាគេហទំព័រគួរប្រើអ្នកផ្តល់សេវាដែលគ្រប់គ្រង។ |
| **Service Endpoint / Connection Status** | បង្ហាញ endpoint បច្ចុប្បន្ន ឬស្ថានភាពការតភ្ជាប់បន្ទាប់ពីការផ្តល់។ |

បន្ទាប់ពីការផ្តល់ សូមរក្សាទុកការកំណត់ ហើយផ្ទៀងផ្ទាត់ស្ថានភាពការតភ្ជាប់ មុនពេលពឹងផ្អែកលើលំហូរការងារសេវាដែលគ្រប់គ្រង។ ប្រសិនបើការផ្តល់បរាជ័យ សូមពិនិត្យការណែនាំសាកល្បងម្ដងទៀតដែលបានបង្ហាញ ហើយបញ្ជាក់ថាគេហទំព័រមានសិទ្ធិប្រើអ្នកផ្តល់សេវាដែលបានបង្ហោះ។

## ការកំណត់ Google Calendar {#google-calendar-configuration}

នៅពេលមុខងារ calendar របស់ Superdav AI Agent v1.18.0 ត្រូវបានបើក agent អាចអាន calendars និងព័ត៌មានលម្អិតព្រឹត្តិការណ៍ដែលបានកំណត់។ ឧបករណ៍ calendar ផ្តោតលើការអាន និងមានប្រយោជន៍សម្រាប់ការរំលឹកដែលដឹងអំពីកាលវិភាគ, ការតាមដានអ្នកចូលរួម និងការផ្គូផ្គងទំនាក់ទំនង។

| Field | Description |
|---|---|
| **Google Calendar Credentials** | រក្សាទុកព័ត៌មានសម្ងាត់ ឬការតភ្ជាប់ token ដែលត្រូវការ ដើម្បីអានទិន្នន័យ calendar។ |
| **Calendar Selection** | កំណត់ថា agent អាចពិនិត្យ calendars ដែលបានកំណត់ណាខ្លះ។ |
| **Calendar Connection Status** | បញ្ជាក់ថាព័ត៌មានសម្ងាត់បច្ចុប្បន្នអាចអាន calendars និងព្រឹត្តិការណ៍បានឬអត់។ |

រក្សាព័ត៌មានសម្ងាត់ calendar ឱ្យកំណត់ត្រឹម calendars ដែល agent ត្រូវការ។ តភ្ជាប់ឡើងវិញ ឬបង្វិលព័ត៌មានសម្ងាត់ ប្រសិនបើស្ថានភាពបង្ហាញថា token ផុតកំណត់។

## ការជូនដំណឹង TextBee SMS {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 បន្ថែម TextBee ជាអ្នកផ្តល់សេវា SMS សម្រាប់លំហូរការងារជូនដំណឹងដែលបានកំណត់។ ការជូនដំណឹង SMS គួរត្រូវបានភ្ជាប់ជាមួយច្រកអនុម័តដោយមនុស្ស សម្រាប់សារដែលរសើប ឬបង្ហាញដល់អ្នកប្រើ។

| Field | Description |
|---|---|
| **TextBee API Key** | ផ្ទៀងផ្ទាត់សំណើទៅអ្នកផ្តល់សេវា TextBee SMS។ |
| **TextBee Device / Sender** | ជ្រើស sender ឬឧបករណ៍ TextBee ដែលប្រើសម្រាប់សារចេញ នៅពេលអ្នកផ្តល់សេវាត្រូវការ។ |
| **SMS Notifications Enabled** | អនុញ្ញាតឱ្យលំហូរការងារដែលបានអនុម័តផ្ញើការជូនដំណឹងជាសារអក្សរ។ ទុកឱ្យបិទ ដើម្បីរារាំងការផ្ញើ SMS។ |

ផ្ញើសារសាកល្បងតែទៅលេខដែលជាកម្មសិទ្ធិរបស់អ្នកគ្រប់គ្រង បន្ទាប់មកបញ្ជាក់ឥរិយាបថច្រកអនុម័ត មុនពេលបើកការរំលឹកតាមកាលវិភាគ ឬការរំលឹកដែលបង្ហាញដល់អ្នកចូលរួម។

## Feature Flags {#feature-flags}

បានណែនាំផងដែរក្នុង v1.9.0 ផ្ទាំង **Settings → Feature Flags** ផ្តល់កុងតាក់បិទបើកសម្រាប់មុខងារជាជម្រើស។ flag នីមួយៗគឺបើក ឬបិទទូទាំងបណ្តាញ។ ពេលនេះមិនមានការកំណត់ជំនួសតាមគេហទំព័រនីមួយៗទេ។

### ការចូលទៅកាន់ Feature Flags {#accessing-feature-flags}

1. នៅក្នុង WordPress admin សូមចូលទៅ **Gratis AI Agent → Settings**។
2. ចុចផ្ទាំង **Feature Flags**។

### Access Control Flags {#access-control-flags}

| សញ្ញាកំណត់ | លំនាំដើម | ការពិពណ៌នា |
|---|---|---|
