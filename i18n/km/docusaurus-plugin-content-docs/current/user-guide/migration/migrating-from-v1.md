# ការផ្លាស់ប្តូរពី V1

## Ultimate Multisite បានផ្លាស់ប្តូរពីជំនាន់នៃការចេញផ្សាយ 1.x ដំបូងរបស់ខ្លួន មកជាជំនាន់ 2.x។

Ultimate Multisite 2.0 និងលើជាងនេះគឺជាការសរសេរឡើងវិញទាំងស្រុងនៃ codebase ដែលមានន័យថា មានការចែករំលែកតិចតួចណាស់រវាងកំណែចាស់ និងថ្មី។ ដោយសារតែហេតុផលនោះ នៅពេលអ្នកធ្វើបច្ចុប្បន្នភាពពី 1.x ទៅ 2.x ទិន្នន័យរបស់អ្នកនឹងត្រូវផ្លាស់ប្តូរទៅជាទ្រង់ទ្រាយដែលកំណែថ្មីអាចយល់បាន។

សំណាងល្អ Ultimate Multisite 2.0+ **ភ្ជាប់មកជាមួយ migrator** ដែលត្រូវបានបង្កើតឡើងនៅក្នុង core ដែលមានសមត្ថភាពក្នុងការរកឃើញទិន្នន័យពីកំណែចាស់ និងបំប្លែងវាទៅជាទ្រង់ទ្រាយថ្មី។ ការផ្លាស់ប្តូរនេះនឹងកើតឡើងក្នុងអំឡុងពេល **Setup Wizard** នៃកំណែ 2.0+។

មេរៀននេះនឹងពន្យល់អំពីរបៀបដែល migrator ដំណើរការ អ្វីដែលត្រូវធ្វើក្នុងករណីបរាជ័យ និងរបៀបដោះស្រាយបញ្ហាដែលអាចកើតឡើងក្នុងដំណើរការនេះ។

_**សំខាន់៖ មុនពេលអ្នកចាប់ផ្តើមធ្វើបច្ចុប្បន្នភាពពីកំណែ 1.x ទៅកំណែ 2.0 សូមប្រាកដថាអ្នកបានបង្កើត backup នៃ database របស់ site អ្នក**_

## ជំហានដំបូង

ជំហានដំបូងគឺការទាញយក file .zip របស់ plugin ហើយដំឡើងកំណែ 2.0 នៅលើ network admin dashboard របស់អ្នក។

បន្ទាប់ពីអ្នក [install and activate version 2.0](../getting-started/installing-ultimate-multisite.md) ប្រព័ន្ធនឹងរកឃើញដោយស្វ័យប្រវត្តិថា Multisite របស់អ្នកកំពុងដំណើរការលើកំណែចាស់ ហើយអ្នកនឹងឃើញសារនេះនៅផ្នែកខាងលើនៃ trang plugin។

_**ចំណាំ៖ ប្រសិនបើអ្នកបានដំឡើង Ultimate Multisite 1.x នៅលើ Multisite របស់អ្នក អ្នកនឹងមានជម្រើសក្នុងការជំនួស plugin ដោយកំណែដែលអ្នកទើបតែទាញយក។ សូមបន្តចុចដើម្បី **Replace current with uploaded**។**_

បន្ទាប់ពីធ្វើការ activate license រួច អ្នកអាចចាប់ផ្តើមការដំឡើងពិតប្រាកដដោយចុច **Install** នៅលើទំព័របន្ទាប់។ ការធ្វើបែបនេះនឹងបង្កើតឯកសារ និង database ដែលចាំបាច់សម្រាប់កំណែ 2.0 ដើម្បីដំណើរការដោយស្វ័យប្រវត្តិ។

##ឥឡូវនេះ ដំណើរការផ្លាស់ប្តូរ (Migration)

កម្មវិធីធ្វើការផ្លាស់ប្តូរ (migrator) មានមុខងារសុវត្ថិភាពភ្ជាប់មកជាមួយ ដែលវាត្រួតពិនិត្យទិន្នន័យទាំងអស់នៅក្នុង multisite របស់អ្នក ដើម្បីធានាថា ទិន្នន័យ Ultimate Multisite របស់អ្នកអាចត្រូវបានផ្លាស់ប្តូរដោយគ្មានបញ្ហា។ ចុចលើប៊ូតុង **Run Check** ដើម្បីចាប់ផ្តើមដំណើរការនេះ។

បន្ទាប់ពីដំណើរការត្រួតពិនិត្យរួច អ្នកនឹងមានជម្រើសពីរ៖ លទ្ធផលអាចនឹងមកជា **ជាមួយ** កំហុស ឬ **គ្មាន** កំហុស។

### ជាមួយកំហុស (With Error)

ប្រសិនបើអ្នកទទួលបានសារកំហុស អ្នកត្រូវទាក់ទងក្រុមគាំទ្ររបស់យើង ដើម្បីឱ្យពួកគេជួយអ្នកក្នុងការដោះស្រាយបញ្ហានោះ។ សូមប្រាកដថាអ្នក **ផ្តល់កំណត់ហេតុនៃកំហុស (error log)** នៅពេលអ្នកបង្កើត ticket។ អ្នកអាចទាញយក log ឬចុចលើតំណដែលនិយាយថា "contact our support team"។ វានឹងបើក widget ជំនួយនៅខាងស្តាំនៃទំព័ររបស់អ្នក ដែលមានប្រអប់សម្រាប់បំពេញព័ត៌មានរួចហើយ រួមទាំង error logs នៅក្រោមផ្នែក description (ការពិពណ៌នា)។

_**ដោយសារប្រព័ន្ធបានរកឃើញកំហុស អ្នកនឹងមិនអាចបន្តផ្លាស់ប្តូរទៅកំណែ 2.0 បានទេ។ អ្នកអាចត្រឡប់ទៅកំណែ 1.x ដើម្បីដំណើរការបណ្តាញរបស់អ្នកបន្ត រហូតទាល់តែបញ្ហានោះត្រូវបានជួសជុល។**_

### គ្មានកំហុស (Without Error)

ប្រសិនបើប្រព័ន្ធមិនរកឃើញកំហុសណាមួយទេ អ្នកនឹងឃើញសារជោគជ័យ និងប៊ូតុង **Migrate** នៅខាងក្រោម ដែលអនុញ្ញាតឱ្យអ្នកបន្តដំណើរការផ្លាស់ប្តូរនេះ។ នៅលើទំព័រនេះ អ្នកនឹងត្រូវបានរំលឹកឱ្យបង្កើត backup នៃ database របស់អ្នកមុនពេលបន្ត ដែលយើងសូមណែនាំយ៉ាងខ្លាំង។ ចុច **Migrate** ប្រសិនបើអ្នកមាន backup រួចហើយ។

និងអស់ហើយ!

អ្នកអាចបន្តដំណើរការកម្មវិធីជំនួយ (Wizard) ដើម្បីធ្វើបច្ចុប្បន្នភាពឡូហ្គោ និងអ្វីផ្សេងទៀតនៅលើបណ្តាញរបស់អ្នក ឬចាប់ផ្តើមស្វែងយល់ពីម៉ឺនុយ និងចំណុចប្រទាក់ថ្មីនៃ Ultimate Multisite version 2.0។ សូមរីករាយជាមួយវាទៅ!
