# ការរួមបញ្ចូលជាមួយ WP Engine

## ទិដ្ឋភាពទូទៅ
WP Engine គឺជាវេទិកាបង្ហោះ WordPress ដែលគ្រប់គ្រងលំដាប់ខ្ពស់ ដែលផ្តល់នូវដំណើរការល្អបំផុត សុវត្ថិភាព និងអាចពង្រីកបានសម្រាប់គេហទំព័រ WordPress។ ការរួមបញ្ចូលនេះអនុញ្ញាតឱ្យមានការធ្វើសមកាលកម្មដែនដោយស្វ័យប្រវត្តិរវាង Ultimate Multisite និង WP Engine។

## លក្ខណៈពិសេស
- ការធ្វើសមកាលកម្មដែនដោយស្វ័យប្រវត្តិ
- ការគាំទ្រ subdomain សម្រាប់ការដំឡើង multisite
- ការរួមបញ្ចូលគ្នាដោយរលូនជាមួយប្រព័ន្ធដែលមានស្រាប់របស់ WP Engine

## តម្រូវការ
ការរួមបញ្ចូលនេះនឹងរកឃើញដោយស្វ័យប្រវត្តិថាតើអ្នកកំពុងបង្ហោះនៅលើ WP Engine ហើយប្រើប្រាស់ API របស់ WP Engine ដែលមានស្រាប់។ មិនចាំបាច់មានការកំណត់បន្ថែមទេ ប្រសិនបើ plugin WP Engine ត្រូវបានបើក និងកំណត់ត្រឹមត្រូវ។

ទោះជាយ៉ាងណាក៏ដោយ ប្រសិនបើអ្នកត្រូវការកំណត់រចនាសម្ព័ន្ធការរួមបញ្ចូលដោយដៃ អ្នកអាចកំណត់មួយក្នុងចំណោមថេរទាំងនេះនៅក្នុងឯកសារ `wp-config.php` របស់អ្នក៖

```php
define('WPE_APIKEY', 'your_api_key'); // វិធីសាស្រ្តដែលពេញចិត្តជាងគេ
// ឬ
define('WPE_API', 'your_api_key'); // វិធីសាស្រ្តជំនួស
```

## ការណែនាំអំពីការរៀបចំ

### ១. ផ្ទៀងផ្ទាត់ plugin WP Engine
ប្រសិនបើអ្នកកំពុងបង្ហោះនៅលើ WP Engine plugin WP Engine គួរតែត្រូវបានដំឡើង និងបើកដំណើរការរួចហើយ។ សូមពិនិត្យមើលថា៖

1. plugin WP Engine ត្រូវបានបើកដំណើរការ
2. ឯកសារ `wp-content/mu-plugins/wpengine-common/class-wpeapi.php` មានវត្តមាន

### ២. បើកដំណើរការការរួមបញ្ចូល
1.នៅក្នុង admin របស់ WordPress របស់អ្នក សូមចូលទៅកាន់ Ultimate Multisite > Settings
2. ចូលទៅកាន់ tab "Domain Mapping" (ការបែងចែកដែន)
3. រំលងចុះក្រោមទៅ "Host Integrations" (ការរួមបញ្ចូលម្ចាស់ផ្ទះ)
4. បើកដំណើរការការរួមបញ្ចូល WP Engine
5. ចុចលើ "Save Changes" (រក្សាទុកការផ្លាស់ប្តូរ)

## វាដំណើរការដូចម្តេច

### ការធ្វើសមកាលកម្មដែន
នៅពេលដែលដែនត្រូវបានកំណត់នៅក្នុង Ultimate Multisite៖

1. ការរួមបញ្ចូលប្រើប្រាស់ WP Engine API ដើម្បីបន្ថែមដែនទៅក្នុងការដំឡើង WP Engine របស់អ្នក
2. WP Engine នឹងគ្រប់គ្រងការកំណត់រចនាសម្ព័ន្ធដែន និងការចេញវិញ្ញាបនបត្រ SSL
3. នៅពេលដែលការបែងចែកដែនត្រូវបានលុប ចំណែកការរួមបញ្ចូលនឹងលុបដែននោះចេញពី WP Engine

### ការគាំទ្រ subdomain

សម្រាប់ការដំឡើង subdomain multisite៖

១. ការរួមបញ្ចូលនេះនឹងបន្ថែម subdomain នីមួយៗទៅកាន់ WP Engine នៅពេលបង្កើតគេហទំព័រថ្មី។
២. WP Engine នឹងគ្រប់គ្រងការកំណត់រចនាសម្ព័ន្ធ subdomain។
៣. នៅពេលលុបគេហទំព័រចេញ ការរួមបញ្ចូលនេះនឹងដក subdomain ចេញពី WP Engine។

## កំណត់សម្គាល់សំខាន់ៗ

### Wildcard Domains (ឈ្មោះដែនពហុ)

សម្រាប់ការដំឡើង subdomain multisite វាត្រូវបានណែនាំឱ្យទាក់ទងទៅកាន់ការគាំទ្ររបស់ WP Engine ដើម្បីស្នើសុំការកំណត់រចនាសម្ព័ន្ធ wildcard domain។ នេះអនុញ្ញាតឱ្យ subdomain ទាំងអស់ដំណើរការដោយស្វ័យប្រវត្តិ ដោយមិនចាំបាច់បន្ថែមនីមួយៗដោយឡែកទេ។

### SSL Certificates (វិញ្ញាបនបត្រ SSL)

WP Engine នឹងគ្រប់គ្រងការចេញ និងការបន្តសុពលភាពនៃវិញ្ញាបនបត្រ SSL ដោយស្វ័យប្រវត្តិសម្រាប់ដែនទាំងអស់ដែលបានបន្ថែមតាមរយៈការរួមបញ្ចូលនេះ។ មិនត្រូវការការកំណត់រចនាសម្ព័ន្ធបន្ថែមណាមួយឡើយ។

## ការដោះស្រាយបញ្ហា (Troubleshooting)

### បញ្ហានៃការតភ្ជាប់ API
- សូមពិនិត្យមើលថាតើ plugin WP Engine ដំណើរការ និងត្រូវបានកំណត់រចនាសម្ព័ន្ធត្រឹមត្រូវឬអត់។
- ប្រសិនបើអ្នកបានកំណត់ API key ដោយដៃ សូមពិនិត្យមើលថាតើវាត្រឹមត្រូវដែរឬទេ។
- ទាក់ទងការគាំទ្ររបស់ WP Engine ប្រសិនបើអ្នកជួបបញ្ហាជាមួយ API។

### ដែនមិនត្រូវបានបន្ថែម
- សូមពិនិត្យមើល logs របស់ Ultimate Multisite ដើម្បីរកសារកំហុសណាមួយ។
- សូមផ្ទៀងផ្ទាត់ថាតើដែននោះមិនទាន់ត្រូវបានបន្ថែមទៅ WP Engine រួចហើយឬនៅ។
- សូមប្រាកដថាផែនការ WP Engine របស់អ្នកគាំទ្រចំនួនដែនដែលអ្នកកំពុងបន្ថែម។

### បញ្ហានៃ subdomain
- ប្រសិនបើ subdomain មិនដំណើរការ សូមទាក់ទងការគាំទ្ររបស់ WP Engine ដើម្បីស្នើសុំការកំណត់រចនាសម្ព័ន្ធ wildcard domain។
- សូមផ្ទៀងផ្ទាត់ថាតើការកំណត់ DNS របស់អ្នកត្រូវបានកំណត់ត្រឹមត្រូវសម្រាប់ដែនសំខាន់ និង subdomain។
