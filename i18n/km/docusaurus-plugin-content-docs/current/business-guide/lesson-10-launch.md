# មេរៀនទី 10៖ ថ្ងៃបើកដំណើរការ {#lesson-10-launch-day}

អ្វីៗត្រូវបានរៀបចំរួចហើយ។ មុនពេលអ្នកបើកឱ្យប្រើប្រាស់ មេរៀននេះនឹងនាំអ្នកពិនិត្យបញ្ជីត្រៀមមុនបើកដំណើរការ ដើម្បីប្រាកដថាគ្មានអ្វីខូច បាត់ ឬធ្វើឱ្យអាម៉ាស់។

## កន្លែងដែលយើងបានបញ្ចប់ {#where-we-left-off}

FitSite មានគំរូ គម្រោង ការបង់ប្រាក់ អត្តសញ្ញាណម៉ាក ការណែនាំចាប់ផ្តើម និងតម្លៃ ដែលបានកំណត់រួចរាល់។ ឥឡូវយើងត្រូវផ្ទៀងផ្ទាត់ថាអ្វីៗដំណើរការ ហើយបើកឱ្យប្រើប្រាស់ជាផ្លូវការ។

## បញ្ជីត្រៀមមុនបើកដំណើរការ {#pre-launch-checklist}

ពិនិត្យរាល់ចំណុចទាំងអស់។ កុំរំលងចំណុចណាមួយ។

### ហេដ្ឋារចនាសម្ព័ន្ធវេទិកា {#platform-infrastructure}

- [ ] សេវាបង្ហោះមានស្ថិរភាព និងដំណើរការល្អនៅពេលមានបន្ទុក
- [ ] Wildcard SSL សកម្ម ហើយ subdomain ទាំងអស់បង្ហាញតាម HTTPS
- [ ] ការភ្ជាប់ដែនដំណើរការ -- សាកល្បងបង្កើតវេបសាយមួយ ហើយភ្ជាប់ដែនផ្ទាល់ខ្លួន
- [ ] ការបម្រុងទុកត្រូវបានកំណត់ និងសាកល្បងរួច (ស្ដារឡើងវិញយ៉ាងហោចណាស់មួយ ដើម្បីផ្ទៀងផ្ទាត់)
- [ ] មានការត្រួតពិនិត្យស្ថានភាពរួចហើយ -- អ្នកនឹងដឹង ប្រសិនបើវេទិកាធ្លាក់ដំណើរការ

### គំរូ {#templates}

- [ ] គំរូទាំងបីបង្ហាញត្រឹមត្រូវលើវេបសាយថ្មី
- [ ] ខ្លឹមសារជំនួសមានប្រយោជន៍ និងគ្មានកំហុសអក្ខរាវិរុទ្ធ
- [ ] រូបភាពទាំងអស់មានអាជ្ញាបណ្ណត្រឹមត្រូវ (គ្មាន watermark របស់រូបថត stock)
- [ ] ការឆ្លើយតបលើទូរស័ព្ទដំណើរការលើគ្រប់ទំព័រគំរូ
- [ ] ល្បឿនផ្ទុកទំព័រអាចទទួលយកបាន (សាកល្បងជាមួយឧបករណ៍ដូចជា GTmetrix ឬ PageSpeed Insights)
- [ ] គ្មានតំណភ្ជាប់ខូច ឬឯកសារប្រភពបាត់លើគំរូណាមួយ

### គម្រោង និងផលិតផល {#plans-and-products}

- [ ] គម្រោងទាំងបីសកម្ម និងអាចមើលឃើញ
- [ ] ការពិពណ៌នាគម្រោងត្រឹមត្រូវ និងសមស្របនឹងទីផ្សារជាក់លាក់
- [ ] តម្លៃត្រឹមត្រូវ (ប្រចាំខែ និងប្រចាំឆ្នាំ)
- [ ] ថ្លៃរៀបចំត្រូវបានកំណត់លើគម្រោងត្រឹមត្រូវ
- [ ] រយៈពេលសាកល្បងដំណើរការលើគម្រោង Starter
- [ ] ការផ្តល់ជម្រើសបន្ថែមបង្ហាញត្រឹមត្រូវក្នុងពេលបង់ប្រាក់
- [ ] ការកំណត់កម្រិត plugin និង theme ត្រូវបានអនុវត្តត្រឹមត្រូវតាមគម្រោងនីមួយៗ

### លំហូរការបង់ប្រាក់ {#checkout-flow}

- [ ] បំពេញការចុះឈ្មោះសាកល្បងពេញលេញលើគម្រោងនីមួយៗ (ប្រើរបៀបទូទាត់សាកល្បង)
- [ ] ការជ្រើសរើសគំរូបង្ហាញគំរូត្រឹមត្រូវតាមគម្រោង
- [ ] ការទូទាត់ដំណើរការជោគជ័យ
- [ ] អតិថិជនទទួលបានអ៊ីមែលស្វាគមន៍បន្ទាប់ពីចុះឈ្មោះ
- [ ] វេបសាយថ្មីត្រូវបានបង្កើតជាមួយគំរូត្រឹមត្រូវ
- [ ] អតិថិជនអាចចូលទៅវេបសាយថ្មីរបស់ពួកគេភ្លាមៗ
- [ ] កូដបញ្ចុះតម្លៃដំណើរការត្រឹមត្រូវ

### អត្តសញ្ញាណម៉ាក {#branding}

- [ ] ទំព័រចូលប្រើបង្ហាញអត្តសញ្ញាណម៉ាក FitSite
- [ ] Admin Dashboard បង្ហាញអត្តសញ្ញាណម៉ាក FitSite
- [ ] អ៊ីមែលប្រព័ន្ធទាំងអស់ប្រើអត្តសញ្ញាណម៉ាក FitSite និងភាសាផ្តោតលើវិស័យហាត់ប្រាណ
- [ ] វិក្កយបត្របង្ហាញត្រឹមត្រូវជាមួយព័ត៌មានអាជីវកម្មរបស់អ្នក
- [ ] វេបសាយទីផ្សារបើកដំណើរការ ហើយភ្ជាប់ទៅទម្រង់បង់ប្រាក់

### ការណែនាំចាប់ផ្តើម {#onboarding}

- [ ] Quick Start widget បង្ហាញលើ Dashboard របស់អតិថិជនថ្មី
- [ ] តំណ Quick Start ទាំងអស់នាំទៅទំព័រត្រឹមត្រូវ
- [ ] លំដាប់អ៊ីមែលស្វាគមន៍ត្រូវបានកំណត់ និងសាកល្បងរួច
- [ ] អត្ថបទមូលដ្ឋានចំណេះដឹងត្រូវបានបោះផ្សាយ និងអាចចូលមើលបាន
- [ ] ទំព័រ Account បង្ហាញព័ត៌មានគម្រោងត្រឹមត្រូវ និងជម្រើសដំឡើងកម្រិត

### ផ្នែកច្បាប់ និងអាជីវកម្ម {#legal-and-business}

- [ ] លក្ខខណ្ឌសេវាកម្មត្រូវបានបោះផ្សាយ និងភ្ជាប់ពីការបង់ប្រាក់
- [ ] គោលការណ៍ឯកជនភាពត្រូវបានបោះផ្សាយ និងអាចចូលមើលបាន
- [ ] គោលការណ៍សងប្រាក់វិញត្រូវបានកំណត់ និងកត់ត្រាជាឯកសារ
- [ ] នីតិបុគ្គលអាជីវកម្មត្រូវបានរៀបចំសម្រាប់ទទួលការទូទាត់
- [ ] ច្រកទូទាត់ស្ថិតក្នុងរបៀបផ្លូវការ (មិនមែនរបៀបសាកល្បង)
- [ ] ការកំណត់ពន្ធត្រឹមត្រូវសម្រាប់តំបន់អំណាចរបស់អ្នក

## បើកដំណើរការសាកល្បង ទល់នឹង បើកដំណើរការពេញលេញ {#soft-launch-vs-hard-launch}

សូមពិចារណាការបើកដំណើរការជា 2 ដំណាក់កាល៖

### ដំណាក់កាលទី 1៖ បើកដំណើរការសាកល្បង {#phase-1-soft-launch}

អញ្ជើញម្ចាស់ស្ទូឌីយោហាត់ប្រាណ 5-10 នាក់ឱ្យចុះឈ្មោះ មុនពេលបើកជាសាធារណៈ។ ពួកគេគឺជាអតិថិជន beta របស់អ្នក។ ផ្តល់ការបញ្ចុះតម្លៃយ៉ាងច្រើនឱ្យពួកគេ (បញ្ចុះ 50% អស់មួយជីវិត ឬឥតគិតថ្លៃ 3 ខែ) ជាថ្នូរនឹង៖

- មតិយោបល់ស្មោះត្រង់អំពីបទពិសោធន៍ចុះឈ្មោះ និងការណែនាំចាប់ផ្តើម
- ការអនុញ្ញាតឱ្យប្រើវេបសាយរបស់ពួកគេជាឧទាហរណ៍បង្ហាញ
- ការរាយការណ៍កំហុស ឬបញ្ហាណាមួយដែលពួកគេជួបប្រទះ

វិធីនេះផ្តល់ឱ្យអ្នកនូវមតិយោបល់ពិតពីអតិថិជន និងវេបសាយកំពុងដំណើរការដើម្បីបង្ហាញ មុនពេលអ្នកបើកជាសាធារណៈ។

### ដំណាក់កាលទី 2៖ បើកដំណើរការជាសាធារណៈ {#phase-2-public-launch}

បន្ទាប់ពីបញ្ចូលមតិយោបល់ពីការបើកសាកល្បងរួច៖

- ប្តូរច្រកទូទាត់ទៅរបៀបផ្លូវការ
- បោះផ្សាយវេបសាយទីផ្សាររបស់អ្នក
- ចាប់ផ្តើមស្វែងរកអតិថិជន (មេរៀនទី 11)
- ប្រកាសលើបណ្តាញពាក់ព័ន្ធក្នុងឧស្សាហកម្មហាត់ប្រាណ

## សកម្មភាពនៅថ្ងៃបើកដំណើរការ {#launch-day-actions}

នៅថ្ងៃដែលអ្នកបើកជាសាធារណៈ៖

1. **ប្តូរទៅការទូទាត់ផ្លូវការ** -- បិទរបៀបសាកល្បងលើច្រកទូទាត់របស់អ្នក
2. **ផ្ទៀងផ្ទាត់ម្តងទៀត** -- ធ្វើការចុះឈ្មោះសាកល្បងពេញលេញជាមួយការទូទាត់ពិត (សងប្រាក់វិញឱ្យខ្លួនឯងបន្ទាប់មក)
3. **តាមដានយ៉ាងជិតស្និទ្ធ** -- មើលកំហុស ការចុះឈ្មោះបរាជ័យ ឬបញ្ហាទូទាត់
4. **ត្រៀមខ្លួនជួយ** -- អតិថិជនពិតដំបូងរបស់អ្នកអាចត្រូវការជំនួយ ហើយការឆ្លើយតបលឿនបង្កើនទំនុកចិត្ត
5. **អបអរខ្លីៗ** -- បន្ទាប់មកត្រឡប់ទៅធ្វើការវិញ

## អ្វីដែលអាចកើតបញ្ហា {#what-can-go-wrong}

ត្រៀមខ្លួនសម្រាប់៖

- **បញ្ហាច្រកទូទាត់**៖ ត្រៀមព័ត៌មានទំនាក់ទំនងជំនួយរបស់អ្នកផ្តល់ច្រកទូទាត់
- **បញ្ហាវិញ្ញាបនបត្រ SSL**៖ ដឹងពីរបៀបពិនិត្យ និងបន្តវិញ្ញាបនបត្រ
- **ការបញ្ជូនអ៊ីមែលបរាជ័យ**៖ សាកល្បងថាអ៊ីមែលពិតជាមកដល់ (ពិនិត្យថត spam)
- **ដំណើរការនៅពេលមានបន្ទុក**៖ ប្រសិនបើមានចរាចរណ៍កើនឡើងភ្លាមៗ ត្រូវដឹងពីរបៀបពង្រីកសេវាបង្ហោះរបស់អ្នក
- **អតិថិជនច្រឡំ**៖ ត្រៀមមូលដ្ឋានចំណេះដឹង និងបណ្តាញជំនួយរបស់អ្នក
