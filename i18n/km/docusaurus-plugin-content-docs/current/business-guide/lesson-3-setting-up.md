# មេរៀនទី ៣៖ រៀបចំបណ្តាញរបស់អ្នក

ដល់ពេលសាងសង់ហើយ។ ក្នុងមេរៀននេះ អ្នកនឹងដំឡើង Ultimate Multisite ហើយកំណត់រចនាសម្ព័ន្ធគ្រឹះនៃបណ្តាញ FitSite។ ការសម្រេចចិត្តទាំងអស់នៅទីនេះ ត្រូវបានធ្វើឡើងដោយគិតពីទីផ្សារជំនាញហាត់ប្រាណ។

## យើងបានឈប់ត្រឹមណា {#where-we-left-off}

យើងបានជ្រើសរើសស្ទូឌីយោហាត់ប្រាណជាទីផ្សារជំនាញរបស់យើង ហើយបានបញ្ជាក់ថាឱកាសនេះមានតម្លៃ។ ឥឡូវនេះ យើងនឹងបម្លែងគំនិតនោះទៅជាវេទិកាដែលដំណើរការបាន។

## ជ្រើសរើស Hosting របស់អ្នក {#choosing-your-hosting}

ជម្រើស hosting របស់អ្នកសំខាន់សម្រាប់វេទិកាទីផ្សារជំនាញ ជាងសម្រាប់គេហទំព័រតែមួយ។ អ្នកមិនមែនកំពុងផ្ទុក site តែមួយទេ -- អ្នកកំពុងផ្ទុកបណ្តាញមួយដែលនឹងរីកចម្រើនទៅជារាប់សិប ឬរាប់រយ site។

### អ្វីដែលត្រូវរកមើល {#what-to-look-for}

- **ការគាំទ្រ WordPress Multisite**៖ មិនមែន host ទាំងអស់គ្រប់គ្រង multisite បានល្អទេ
- **Wildcard SSL**៖ ចាំបាច់សម្រាប់បណ្តាញដែលផ្អែកលើ subdomain
- **ធនធានដែលពង្រីកបាន**៖ អ្នកត្រូវការកន្លែងសម្រាប់រីកចម្រើន ដោយមិនចាំបាច់ផ្លាស់ទី
- **ការរួមបញ្ចូល Ultimate Multisite**៖ ការផ្គូផ្គងដែន និង SSL ស្វ័យប្រវត្តិ ជួយសន្សំកម្លាំងប្រតិបត្តិការបានច្រើន

### វិធីសាស្ត្រដែលបានណែនាំ {#recommended-approach}

ជ្រើសរើស host ពីបញ្ជី [អ្នកផ្តល់សេវាដែលឆបគ្នា](/user-guide/host-integrations/closte)។ សេវាទាំងនេះត្រូវបានសាកល្បងជាមួយ Ultimate Multisite ហើយផ្តល់ការរួមបញ្ចូលដែលអ្នកត្រូវការ សម្រាប់ការផ្គូផ្គងដែន និងស្វ័យប្រវត្តិកម្ម SSL។

សម្រាប់ FitSite យើងនឹងប្រើរចនាសម្ព័ន្ធ subdomain។ នេះមានន័យថា site របស់អតិថិជនដំបូងនឹងបង្ហាញជា `studioname.fitsite.com` មុនពេលពួកគេជ្រើសផ្គូផ្គងដែនផ្ទាល់ខ្លួនរបស់ពួកគេ។

## ដំឡើង WordPress Multisite {#installing-wordpress-multisite}

ប្រសិនបើអ្នកមិនទាន់មានការដំឡើង WordPress Multisite ទេ៖

1. ដំឡើង WordPress លើអ្នកផ្តល់សេវា hosting របស់អ្នក
2. អនុវត្តតាមមគ្គុទ្ទេសក៍ [របៀបដំឡើង WordPress Multisite](/user-guide/getting-started/how-to-install-wordpress-multisite)
3. ជ្រើសរើសរចនាសម្ព័ន្ធ **subdomain** នៅពេលត្រូវបានសួរ

:::tip ហេតុអ្វីជ្រើស Subdomains?
Subdomains ផ្តល់ឱ្យ site របស់អតិថិជននីមួយៗនូវអាសយដ្ឋានដាច់ដោយឡែកផ្ទាល់ខ្លួន (`studio.fitsite.com`) ជំនួសឱ្យ path (`fitsite.com/studio`)។ វាមើលទៅមានវិជ្ជាជីវៈជាងសម្រាប់អតិថិជនរបស់អ្នក ហើយជៀសវាងការប៉ះទង្គិច permalink។ សូមមើល [Ultimate Multisite 101](/user-guide/getting-started/ultimate-multisite-101) សម្រាប់ការប្រៀបធៀបលម្អិត។
:::

## ដំឡើង Ultimate Multisite {#installing-ultimate-multisite}

អនុវត្តតាមមគ្គុទ្ទេសក៍ [ដំឡើង Ultimate Multisite](/user-guide/getting-started/installing-ultimate-multisite) ដើម្បី៖

1. បង្ហោះ និងធ្វើឱ្យ plugin សកម្មទូទាំងបណ្តាញ
2. ដំណើរការ [អ្នកជួយរៀបចំ](/user-guide/getting-started/multisite-setup-wizard)

ក្នុងអំឡុងពេលប្រើអ្នកជួយរៀបចំ សូមចងចាំទីផ្សារជំនាញ FitSite៖

- **រូបិយប័ណ្ណ**៖ កំណត់ទៅជារូបិយប័ណ្ណដែលអតិថិជនស្ទូឌីយោហាត់ប្រាណរបស់អ្នកប្រើ
- **ឈ្មោះក្រុមហ៊ុន**៖ "FitSite" (ឬឈ្មោះម៉ាកដែលអ្នកបានជ្រើស)
- **ឡូហ្គោក្រុមហ៊ុន**៖ បង្ហោះឡូហ្គោម៉ាករបស់អ្នក -- វានឹងបង្ហាញលើវិក្កយបត្រ និងអ៊ីមែល

## កំណត់រចនាសម្ព័ន្ធសម្រាប់ទីផ្សារជំនាញហាត់ប្រាណ {#configuring-for-the-fitness-niche}

នៅពេលបានដំឡើង Ultimate Multisite រួច សូមធ្វើជម្រើសកំណត់រចនាសម្ព័ន្ធជាក់លាក់សម្រាប់ទីផ្សារជំនាញទាំងនេះ៖

### ការកំណត់ទូទៅ {#general-settings}

ចូលទៅកាន់ **Ultimate Multisite > ការកំណត់** ហើយកំណត់រចនាសម្ព័ន្ធ៖

- **ឈ្មោះ Site**៖ FitSite
- **តួនាទីលំនាំដើម**៖ អ្នកគ្រប់គ្រង -- ម្ចាស់ស្ទូឌីយោហាត់ប្រាណត្រូវការការគ្រប់គ្រងពេញលេញលើមាតិកា site របស់ពួកគេ
- **ការចុះឈ្មោះ**៖ បើកការចុះឈ្មោះអ្នកប្រើ ដើម្បីឱ្យម្ចាស់ស្ទូឌីយោអាចចុះឈ្មោះដោយខ្លួនឯង

### ការកំណត់រចនាសម្ព័ន្ធអ៊ីមែល {#email-configuration}

អ៊ីមែលប្រព័ន្ធរបស់អ្នកគួរតែនិយាយតាមភាសានៃទីផ្សារជំនាញរបស់អ្នក។ ចូលទៅកាន់ **Ultimate Multisite > ការកំណត់ > អ៊ីមែល** ហើយកែតម្រូវ៖

- ជំនួសពាក្យទូទៅដូចជា "site ថ្មីរបស់អ្នក" ដោយសារដែលពាក់ព័ន្ធនឹងហាត់ប្រាណ
- ឧទាហរណ៍ប្រធានបទស្វាគមន៍៖ "គេហទំព័រស្ទូឌីយោហាត់ប្រាណរបស់អ្នករួចរាល់ហើយ"
- ឧទាហរណ៍ខ្លឹមសារស្វាគមន៍៖ យោងដល់ស្ទូឌីយោ ថ្នាក់រៀន និងការចាប់ផ្តើមប្រើ site ហាត់ប្រាណរបស់ពួកគេ

យើងនឹងកែលម្អចំណុចទាំងនេះបន្ថែមទៀតនៅមេរៀនទី ៨ (ការណែនាំអតិថិជនឱ្យចាប់ផ្តើមប្រើប្រាស់) ប៉ុន្តែការកំណត់សម្លេងឥឡូវនេះ ធ្វើឱ្យសូម្បីតែការចុះឈ្មោះសាកល្បងដំបូងក៏មានអារម្មណ៍ស្របតាមទីផ្សារជំនាញដែរ។

### ការកំណត់រចនាសម្ព័ន្ធដែន {#domain-configuration}

ប្រសិនបើប្រើអ្នកផ្តល់សេវា hosting ដែលឆបគ្នា សូមកំណត់រចនាសម្ព័ន្ធការផ្គូផ្គងដែនឥឡូវនេះ៖

1. ចូលទៅកាន់ **Ultimate Multisite > ការកំណត់ > ការផ្គូផ្គងដែន**
2. អនុវត្តតាមមគ្គុទ្ទេសក៍រួមបញ្ចូលសម្រាប់ host ជាក់លាក់របស់អ្នក
3. សាកល្បងថា subsites ថ្មីទទួលបាន SSL ដោយស្វ័យប្រវត្តិ

វាធានាថា នៅពេលយើងចាប់ផ្តើមបង្កើតគំរូ និង site សាកល្បងនៅមេរៀនបន្ទាប់ អ្វីៗដំណើរការពេញលេញពីដើមដល់ចប់។

## បណ្តាញ FitSite មកដល់ពេលនេះ {#the-fitsite-network-so-far}

នៅចុងមេរៀននេះ នេះជាអ្វីដែលអ្នកមាន៖

```
បណ្តាញ FitSite
├── WordPress Multisite (របៀប subdomain)
├── Ultimate Multisite (បានដំឡើង និងកំណត់រចនាសម្ព័ន្ធ)
├── Hosting ជាមួយ wildcard SSL
├── បានកំណត់រចនាសម្ព័ន្ធការផ្គូផ្គងដែន
├── គំរូអ៊ីមែលជាក់លាក់តាមទីផ្សារជំនាញ (ដំបូង)
└── រួចរាល់សម្រាប់គំរូ site (មេរៀនបន្ទាប់)
```

## អ្វីដែលយើងបានសាងសង់ក្នុងមេរៀននេះ {#what-we-built-this-lesson}

- ការដំឡើង **WordPress Multisite ដែលដំណើរការ** ក្នុងរបៀប subdomain
- **បានដំឡើង Ultimate Multisite** និងកំណត់រចនាសម្ព័ន្ធជាមួយម៉ាក FitSite
- **Hosting និង SSL** ត្រូវបានរៀបចំសម្រាប់បណ្តាញដែលកំពុងរីកចម្រើន
- **ការផ្គូផ្គងដែន** ត្រូវបានកំណត់រចនាសម្ព័ន្ធសម្រាប់អ្នកផ្តល់សេវា hosting របស់អ្នក
- **សម្លេងអ៊ីមែលជាក់លាក់តាមទីផ្សារជំនាញ** ត្រូវបានបង្កើតតាំងពីថ្ងៃដំបូង
