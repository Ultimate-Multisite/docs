# Tutor Multisite Compatibility

កម្មវិធីបន្ថែម WordPress ដែលបន្ថែមការកែសម្រួលភាពឆបគ្នាសម្រាប់ Tutor LMS នៅក្នុងបរិស្ថាន multisite។

## ការពិពណ៌នា {#description}

កម្មវិធីបន្ថែមនេះផ្តល់ការកែសម្រួលភាពឆបគ្នាសំខាន់ៗសម្រាប់ដំណើរការ Tutor LMS លើបណ្តាញ WordPress multisite។ វាគ្រប់គ្រងការចុះឈ្មោះអ្នកប្រើ ការគ្រប់គ្រងគ្រូបង្រៀន ការកំណត់វគ្គសិក្សា និងមុខងារផ្សេងៗដែលពាក់ព័ន្ធនឹង multisite។

## លក្ខណៈពិសេស {#features}

- **ការគ្រប់គ្រងតារាងមូលដ្ឋានទិន្នន័យ**: រកឃើញ និងបង្កើតតារាង Tutor LMS ដែលបាត់ដោយស្វ័យប្រវត្តិ នៅពេលបរាជ័យដោយសារការកំណត់ foreign key ក្នុង multisite
- **ការចុះឈ្មោះអ្នកប្រើក្នុង Multisite**: កម្មវិធីគ្រប់គ្រង AJAX ផ្ទាល់ខ្លួនសម្រាប់គ្រប់គ្រងការចុះឈ្មោះអ្នកប្រើនៅទូទាំង site ក្នុងបណ្តាញ
- **ការគ្រប់គ្រងតួនាទីគ្រូបង្រៀន**: ការផ្តល់ និងគ្រប់គ្រងតួនាទីគ្រូបង្រៀនដោយស្វ័យប្រវត្តិនៅក្នុងបរិស្ថាន multisite
- **ការកំណត់វគ្គសិក្សា**: ការរួមបញ្ចូលជាមួយ Ultimate Multisite សម្រាប់គ្រប់គ្រងកំណត់ការបង្កើតវគ្គសិក្សា និង funnel ផ្អែកលើគម្រោង site
- **ទិដ្ឋភាពគ្រប់គ្រងផ្ទាល់ខ្លួន**: បញ្ជីគ្រូបង្រៀន និងចំណុចប្រទាក់គ្រប់គ្រងដែលយល់ដឹងពី multisite
- **ឧបករណ៍គ្រប់គ្រង**: ចំណុចប្រទាក់គ្រប់គ្រងមូលដ្ឋានទិន្នន័យសម្រាប់អ្នកគ្រប់គ្រងបណ្តាញ និង site

## តម្រូវការ {#requirements}

- WordPress 5.0+
- PHP 7.4+
- កម្មវិធីបន្ថែម Tutor LMS (អនុវត្តដោយស្វ័យប្រវត្តិតាម header `Requires Plugins`)
- ការដំឡើង WordPress Multisite
- កម្មវិធីបន្ថែម Ultimate Multisite (សម្រាប់មុខងារកំណត់វគ្គសិក្សា)

## ការដំឡើង {#installation}

1. ផ្ទុកឡើងឯកសារកម្មវិធីបន្ថែមទៅ `/wp-content/plugins/tutor-multisite-compatibility/`
2. ត្រូវប្រាកដថា Tutor LMS ត្រូវបានដំឡើង និងធ្វើឱ្យសកម្ម
3. ធ្វើឱ្យកម្មវិធីបន្ថែមសកម្មតាមម៉ឺនុយ 'កម្មវិធីបន្ថែម' ក្នុង WordPress
4. កម្មវិធីបន្ថែមនឹងរួមបញ្ចូលជាមួយ Tutor LMS ដោយស្វ័យប្រវត្តិ

## មុខងារដែលបានផ្លាស់ទីពី MU plugin {#functionality-moved-from-mu-plugin}

កម្មវិធីបន្ថែមនេះមានមុខងារដែលពីមុនស្ថិតនៅក្នុង `wp-content/mu-plugins/daves.php`៖

- ការកំណត់ការផ្លាស់ប្តូរស្ថានភាពវគ្គសិក្សា
- ការកំណត់ការបង្កើតវគ្គសិក្សា
- ការរួមបញ្ចូល និងការកំណត់ CartFlows
- ការគ្រប់គ្រងតួនាទីគ្រូបង្រៀន Tutor
- ការគ្រប់គ្រងការចុះឈ្មោះអ្នកប្រើ multisite
- ការបង្ហាញ ID ផលិតផលវគ្គសិក្សា

## ការគ្រប់គ្រងមូលដ្ឋានទិន្នន័យ {#database-management}

### បញ្ហា {#the-problem}

នៅក្នុងបរិស្ថាន WordPress multisite ការបង្កើតតារាង Tutor LMS បរាជ័យសម្រាប់តារាងដែលមានការកំណត់ foreign key ទៅតារាងអ្នកប្រើ។ វាកើតឡើងដោយសារ៖

- តារាងជាក់លាក់តាម site ប្រើ prefix ដូចជា `wp_2_tutor_carts`
- Foreign key យោងទៅ `wp_2_users` (ដែលមិនមាន)
- តារាងអ្នកប្រើពិតប្រាកដគឺ `wp_users` (ចែករំលែកទូទាំងបណ្តាញ)

### ដំណោះស្រាយ {#the-solution}

កម្មវិធីបន្ថែមនេះធ្វើដោយស្វ័យប្រវត្តិ៖

1. **រកឃើញតារាងដែលបាត់**: ពិនិត្យតារាងដែលបរាជ័យក្នុងការបង្កើតដោយសារការកំណត់ foreign key
2. **បង្កើតតារាងជាមួយការយោងត្រឹមត្រូវ**: ប្រើឈ្មោះតារាងអ្នកប្រើត្រឹមត្រូវ (`wp_users` ក្នុង multisite)
3. **គ្រប់គ្រងស្ថានការណ៍ច្រើន**:
   - ការធ្វើឱ្យកម្មវិធីបន្ថែមសកម្ម/សកម្មឡើងវិញ
   - ការបង្កើត site ថ្មីក្នុងបណ្តាញ
   - ការពិនិត្យតារាងដោយដៃតាមចំណុចប្រទាក់គ្រប់គ្រង

### តារាងដែលរងផលប៉ះពាល់ {#affected-tables}

- `tutor_coupon_usages` - មាន foreign key ទៅតារាងអ្នកប្រើ
- `tutor_carts` - មាន foreign key ទៅតារាងអ្នកប្រើ
- `tutor_notification_preferences` - តារាង TutorPro ដែលមាន foreign key ទៅតារាងអ្នកប្រើ

### ចំណុចប្រទាក់គ្រប់គ្រង {#admin-interface}

ចូលប្រើឧបករណ៍គ្រប់គ្រងមូលដ្ឋានទិន្នន័យនៅ៖

- **អ្នកគ្រប់គ្រងបណ្តាញ**: ការកំណត់ → Tutor Multisite DB
- **អ្នកគ្រប់គ្រង Site**: ឧបករណ៍ → Tutor Multisite DB

ចំណុចប្រទាក់អនុញ្ញាតឱ្យអ្នក៖
- មើលតារាងដែលបាត់នៅទូទាំង site ទាំងអស់
- ចាប់ផ្តើមការបង្កើតតារាងដោយដៃ
- ពិនិត្យការកំណត់រចនាសម្ព័ន្ធមូលដ្ឋានទិន្នន័យ
- មើលព័ត៌មានស្ថានភាពលម្អិត

## ការគាំទ្រ {#support}

កម្មវិធីបន្ថែមនេះត្រូវបានរចនាឡើងជាពិសេសសម្រាប់វេទិកា KursoPro។ សម្រាប់ការគាំទ្របច្ចេកទេស សូមទាក់ទងក្រុមអភិវឌ្ឍន៍។
