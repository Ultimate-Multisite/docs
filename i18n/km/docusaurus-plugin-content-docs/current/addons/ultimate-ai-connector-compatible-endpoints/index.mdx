# Ultimate AI Connector សម្រាប់ចំណុចបញ្ចប់ដែលឆបគ្នា

Ultimate AI Connector សម្រាប់ចំណុចបញ្ចប់ដែលឆបគ្នា ភ្ជាប់ APIs របស់អ្នកផ្តល់សេវា AI ដែលឆបគ្នាជាមួយ OpenAI ទៅនឹងបណ្ដាញ WordPress multisite របស់អ្នក។ កំណត់រចនាសម្ព័ន្ធអ្នកផ្តល់សេវា AI ច្រើន — ម្នាក់ៗមាន endpoint URL, API key និងម៉ូដែលលំនាំដើមផ្ទាល់ខ្លួន — ហើយបញ្ជូនសំណើជាមួយការប្តូរទៅជម្រើសបន្ទាប់ដោយស្វ័យប្រវត្តិ នៅពេលអ្នកផ្តល់សេវាម្នាក់មិនអាចប្រើបាន។

## មុខងារសំខាន់ៗ {#key-features}

- **គាំទ្រអ្នកផ្តល់សេវាច្រើន**: បន្ថែមកាតអ្នកផ្តល់សេវា AI ច្រើន ម្នាក់ៗមាន endpoint URL, API key និងម៉ូដែលលំនាំដើមផ្ទាល់ខ្លួន
- **អូសដើម្បីរៀបលំដាប់ឡើងវិញ**: រៀបចំអ្នកផ្តល់សេវាតាមអាទិភាព ដោយប្រើដៃចាប់នៅលើកាតនីមួយៗ; សំណើនឹងប្រើអ្នកផ្តល់សេវាតាមលំដាប់
- **ការបញ្ជូនប្តូរទៅជម្រើសបន្ទាប់ដោយស្វ័យប្រវត្តិ**: ប្រសិនបើអ្នកផ្តល់សេវាចម្បងមិនអាចប្រើបាន សំណើនឹងប្តូរទៅអ្នកផ្តល់សេវាដែលបានកំណត់រចនាសម្ព័ន្ធបន្ទាប់ដោយស្វ័យប្រវត្តិ
- **ការកំណត់រចនាសម្ព័ន្ធតាមអ្នកផ្តល់សេវា**: អ្នកផ្តល់សេវានីមួយៗត្រូវបានកំណត់រចនាសម្ព័ន្ធដោយឯករាជ្យ ជាមួយព័ត៌មានសម្គាល់ និងជម្រើសម៉ូដែលផ្ទាល់ខ្លួន
- **UI ការកំណត់ផ្អែកលើកាត**: បង្កើតលើសមាសភាគ Card របស់ WordPress 6.9+ ដែលមានស្ថេរភាព (CardBody, CardHeader, CardDivider)
- **ពង្រីកដោយស្វ័យប្រវត្តិពេលបន្ថែម**: កាតអ្នកផ្តល់សេវាថ្មីពង្រីកដោយស្វ័យប្រវត្តិ ដើម្បីឱ្យអ្នកអាចបំពេញព័ត៌មានលម្អិតភ្លាមៗ
- **ការបំបាត់ឃ្លាំងសម្ងាត់ស្គ្រីប**: ស្គ្រីបរបស់អ្នកផ្តល់សេវាត្រូវបានបំបាត់ឃ្លាំងសម្ងាត់នៅពេលធ្វើបច្ចុប្បន្នភាពកម្មវិធីបន្ថែម ដើម្បីធានាថាកំណែថ្មីបំផុតត្រូវបានផ្ទុក

## តម្រូវការ {#requirements}

- WordPress 5.3 ឬខ្ពស់ជាងនេះ
- PHP 7.4 ឬខ្ពស់ជាងនេះ
- កម្មវិធីបន្ថែម Ultimate Multisite (សកម្ម)
- ចំណុចបញ្ចប់ API ដែលឆបគ្នាជាមួយ OpenAI (បង្ហោះដោយខ្លួនឯង ឬដោយភាគីទីបី)

## ការដំឡើង {#installation}

1. ផ្ទុកឡើងឯកសារកម្មវិធីបន្ថែមទៅថត `/wp-content/plugins/` របស់អ្នក
2. បើកដំណើរការកម្មវិធីបន្ថែមតាមម៉ឺនុយ 'កម្មវិធីបន្ថែម' ក្នុង WordPress
3. ចូលទៅកាន់ **Ultimate Multisite → AI Connector** ដើម្បីកំណត់រចនាសម្ព័ន្ធកម្មវិធីបន្ថែម

## ការកំណត់រចនាសម្ព័ន្ធ {#configuration}

### ការរៀបចំអ្នកផ្តល់សេវាច្រើន {#multi-provider-setup}

អេក្រង់កំណត់រចនាសម្ព័ន្ធប្រើប្លង់ផ្អែកលើកាត ដែលកាតនីមួយៗតំណាងឱ្យអ្នកផ្តល់សេវា AI ម្នាក់។ អ្នកអាចបន្ថែមអ្នកផ្តល់សេវាបានច្រើនតាមតម្រូវការ។

#### ការបន្ថែមអ្នកផ្តល់សេវា {#adding-a-provider}

1. ចូលទៅ **Ultimate Multisite → AI Connector** ក្នុងផ្នែកគ្រប់គ្រងបណ្ដាញ
2. ចុច **បន្ថែមអ្នកផ្តល់សេវា**
3. កាតអ្នកផ្តល់សេវាថ្មីនឹងបង្ហាញ ហើយពង្រីកដោយស្វ័យប្រវត្តិ
4. បំពេញព័ត៌មានលម្អិតរបស់អ្នកផ្តល់សេវា:
   - **ឈ្មោះអ្នកផ្តល់សេវា** — ស្លាកសម្រាប់សម្គាល់អ្នកផ្តល់សេវានេះ (ឧ. "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL** — URL មូលដ្ឋាននៃ API ដែលឆបគ្នាជាមួយ OpenAI (ឧ. `https://api.openai.com/v1`)
   - **API Key** — API key របស់អ្នកសម្រាប់អ្នកផ្តល់សេវានេះ
   - **ម៉ូដែលលំនាំដើម** — អត្តសញ្ញាណម៉ូដែលដែលត្រូវប្រើជាលំនាំដើមសម្រាប់អ្នកផ្តល់សេវានេះ (ឧ. `gpt-4o`, `mistral-small`)
5. ចុច **រក្សាទុកការកំណត់**

#### ការលុបអ្នកផ្តល់សេវា {#removing-a-provider}

ចុចប៊ូតុង **លុប** នៅលើកាតអ្នកផ្តល់សេវាណាមួយ ដើម្បីលុបវា។ រក្សាទុកការកំណត់ ដើម្បីអនុវត្តការផ្លាស់ប្តូរ។

#### ការរៀបលំដាប់អ្នកផ្តល់សេវាឡើងវិញ {#reordering-providers}

អូសរូបតំណាងដៃចាប់ (⣿) នៅផ្នែកខាងឆ្វេងនៃកាតអ្នកផ្តល់សេវាណាមួយ ដើម្បីរៀបលំដាប់វាឡើងវិញ។ លំដាប់កំណត់អាទិភាពនៃការប្តូរទៅជម្រើសបន្ទាប់: សំណើត្រូវបានផ្ញើទៅអ្នកផ្តល់សេវាដំបូង ហើយប្តូរទៅអ្នកផ្តល់សេវាបន្ទាប់ ប្រសិនបើសំណើបរាជ័យ។

### ការបញ្ជូនប្តូរទៅជម្រើសបន្ទាប់ {#fallback-routing}

នៅពេលសំណើទៅអ្នកផ្តល់សេវាចម្បងបរាជ័យ — ដោយសារអស់ពេលកំណត់ ការកំណត់អត្រា ឬសេវាដាច់ — ឧបករណ៍ភ្ជាប់នឹងសាកល្បងម្ដងទៀតជាមួយអ្នកផ្តល់សេវាបន្ទាប់ក្នុងបញ្ជីដោយស្វ័យប្រវត្តិ។ វានឹងបន្តចុះតាមបញ្ជី រហូតដល់អ្នកផ្តល់សេវាម្នាក់ទទួលបានជោគជ័យ ឬអ្នកផ្តល់សេវាទាំងអស់ត្រូវបានសាកល្បងរួច។

| ស្ថានភាព | របៀបដំណើរការ |
|----------|-----------|
| អ្នកផ្តល់សេវាចម្បងឆ្លើយតប | សំណើត្រូវបានបម្រើជាធម្មតា |
| អ្នកផ្តល់សេវាចម្បងបរាជ័យ | សំណើត្រូវបានសាកល្បងម្ដងទៀតជាមួយអ្នកផ្តល់សេវាបន្ទាប់ |
| អ្នកផ្តល់សេវាទាំងអស់បរាជ័យ | កំហុសត្រូវបានត្រឡប់ទៅអ្នកហៅ |

ការប្តូរទៅជម្រើសបន្ទាប់មានភាពថ្លាចំពោះកម្មវិធីបន្ថែម ឬរូបរាងដែលហៅវា — ពួកវានឹងទទួលបានការឆ្លើយតបជោគជ័យ ឬកំហុសចុងក្រោយ មិនថាបានសាកល្បងអ្នកផ្តល់សេវាប៉ុន្មាននាក់ក៏ដោយ។

### ឯកសារយោងការកំណត់រចនាសម្ព័ន្ធអ្នកផ្តល់សេវា {#provider-configuration-reference}

| វាល | ចាំបាច់ | ការពិពណ៌នា |
|-------|----------|-------------|
| ឈ្មោះអ្នកផ្តល់សេវា | បាទ/ចាស | ស្លាកបង្ហាញសម្រាប់អ្នកផ្តល់សេវានេះ |
| Endpoint URL | បាទ/ចាស | URL មូលដ្ឋាននៃ API ដែលឆបគ្នាជាមួយ OpenAI |
| API Key | បាទ/ចាស | សោផ្ទៀងផ្ទាត់សម្រាប់អ្នកផ្តល់សេវានេះ |
| ម៉ូដែលលំនាំដើម | បាទ/ចាស | អត្តសញ្ញាណម៉ូដែលដែលប្រើ នៅពេលមិនមានម៉ូដែលត្រូវបានបញ្ជាក់ក្នុងសំណើ |

## ការដោះស្រាយបញ្ហា {#troubleshooting}

### មិនមានអ្នកផ្តល់សេវាត្រូវបានរាយបញ្ជី {#no-providers-are-listed}

ប្រសិនបើបញ្ជីអ្នកផ្តល់សេវាទទេបន្ទាប់ពីរក្សាទុក សូមបញ្ជាក់ថា:

- អ្នកបានចុច **រក្សាទុកការកំណត់** បន្ទាប់ពីបន្ថែមអ្នកផ្តល់សេវា
- កម្មវិធីរុករករបស់អ្នកមិនបានទប់ស្កាត់ទំព័រការកំណត់មិនឱ្យរក្សាទុកទេ (ពិនិត្យ console របស់កម្មវិធីរុករកសម្រាប់កំហុស JavaScript)
- កម្មវិធីបន្ថែមសកម្ម ហើយអ្នកស្ថិតនៅលើអេក្រង់គ្រប់គ្រងបណ្ដាញ

### សំណើបរាជ័យសម្រាប់អ្នកផ្តល់សេវាទាំងអស់ {#requests-are-failing-for-all-providers}

- ផ្ទៀងផ្ទាត់ថា endpoint URL របស់អ្នកផ្តល់សេវានីមួយៗអាចចូលដំណើរការបានពីម៉ាស៊ីនបម្រើរបស់អ្នក (មិនមែនតែពីកម្មវិធីរុករករបស់អ្នកទេ)
- ពិនិត្យថា API key នីមួយៗត្រឹមត្រូវ ហើយមិនទាន់ផុតកំណត់
- ពិនិត្យស្ថានភាពកំណត់អត្រា និងកូតារបស់អ្នកផ្តល់សេវានៅក្នុងផ្ទាំងគ្រប់គ្រងរបស់ពួកគេ
- បើក WordPress debug logging (`WP_DEBUG_LOG`) ដើម្បីចាប់យកសារកំហុសលម្អិតពីឧបករណ៍ភ្ជាប់

### អ្នកផ្តល់សេវាម្នាក់មិនត្រូវបានប្រើសម្រាប់ការប្តូរទៅជម្រើសបន្ទាប់ {#a-provider-is-not-being-used-for-fallback}

ការអូសដើម្បីរៀបលំដាប់ឡើងវិញផ្លាស់ប្តូរលំដាប់ប្តូរទៅជម្រើសបន្ទាប់។ បញ្ជាក់ថាអ្នកផ្តល់សេវាដែលអ្នករំពឹងឱ្យទទួលសំណើប្តូរទៅជម្រើសបន្ទាប់ ស្ថិតនៅទីតាំងត្រឹមត្រូវក្នុងបញ្ជី ហើយ endpoint URL និង API key របស់វាត្រឹមត្រូវ។

### កាតអ្នកផ្តល់សេវាថ្មីមិនពង្រីកដោយស្វ័យប្រវត្តិ {#new-provider-card-does-not-expand-automatically}

ការពង្រីកដោយស្វ័យប្រវត្តិត្រូវការ JavaScript ត្រូវបានបើកក្នុងកម្មវិធីរុករករបស់អ្នក។ ប្រសិនបើកាតមិនពង្រីកពេលចុច សូមពិនិត្យ console របស់កម្មវិធីរុករកសម្រាប់កំហុស។ មាតិកាកាតអាចចូលប្រើបានជានិច្ច ដោយចុចក្បាលកាត ទោះបីការពង្រីកដោយស្វ័យប្រវត្តិមិនដំណើរការក៏ដោយ។
