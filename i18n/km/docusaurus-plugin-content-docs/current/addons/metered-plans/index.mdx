# គម្រោងគិតតាមការប្រើប្រាស់

addon គម្រោងគិតតាមការប្រើប្រាស់ របស់ Ultimate Multisite បម្លែងរបៀបដែលអ្នកគិតថ្លៃពីអតិថិជន ដោយបន្ថែមសមត្ថភាពគិតថ្លៃតាមការប្រើប្រាស់ដែលបត់បែនបាន។ ជំនួសឱ្យកំណត់រឹងដែលរារាំងអ្នកប្រើពេលប្រើលើស អ្នកអាចបើកការគិតថ្លៃលើសកំណត់ ដែលគិតថ្លៃអតិថិជនដោយស្វ័យប្រវត្តិសម្រាប់ការប្រើប្រាស់បន្ថែម។

## លក្ខណៈសំខាន់ៗ {#key-features}

* **ការគិតថ្លៃតាមការប្រើប្រាស់**៖ គិតថ្លៃអតិថិជនផ្អែកលើការប្រើប្រាស់ពិតប្រាកដ ជំនួសឱ្យកម្រិតតម្លៃថេរ
* **ការគ្រប់គ្រងការប្រើលើសកំណត់**៖ គិតថ្លៃដោយស្វ័យប្រវត្តិសម្រាប់ការប្រើប្រាស់ដែលលើសកំណត់គម្រោង
* **ការតាមដានការប្រើប្រាស់**៖ ត្រួតពិនិត្យការប្រើប្រាស់ធនធានរបស់អតិថិជនក្នុងពេលវេលាពិត
* **ការកំណត់តម្លៃបត់បែន**៖ កំណត់អត្រាផ្ទាល់ខ្លួនសម្រាប់ធនធានផ្សេងៗ ដូចជា ការផ្ទុក ទទឹងបញ្ជូន ទីតាំង ជាដើម
* **ការគិតថ្លៃ AI Token**៖ តាមដាន និងគិតថ្លៃ token បញ្ចូល និងលទ្ធផល សម្រាប់ការប្រើប្រាស់ AI នៅលើទីតាំងរងរបស់អតិថិជន
* **ការចេញវិក្កយបត្រដោយស្វ័យប្រវត្តិ**៖ បង្កើតវិក្កយបត្រដោយស្វ័យប្រវត្តិ នៅពេលការប្រើប្រាស់លើសកម្រិតដែលបានកំណត់
* **របាយការណ៍ការប្រើប្រាស់**៖ ការវិភាគ និងរបាយការណ៍ការប្រើប្រាស់លម្អិតសម្រាប់អតិថិជន និងអ្នកគ្រប់គ្រង

## ស័ក្តិសមសម្រាប់ {#perfect-for}

* **អ្នកផ្តល់សេវាបង្ហោះ**៖ ផ្តល់គម្រោងដែលពង្រីកបានតាមតម្រូវការអតិថិជន
* **វេទិកា SaaS**៖ គិតថ្លៃផ្អែកលើការប្រើប្រាស់ធនធានពិតប្រាកដ
* **បណ្តាញភ្នាក់ងារ**៖ ម៉ូដែលកំណត់តម្លៃបត់បែនសម្រាប់គម្រោងអតិថិជន
* **ការបង្ហោះលក់បន្ត**៖ អនុញ្ញាតឱ្យដៃគូផ្តល់តម្លៃផ្អែកលើការប្រើប្រាស់

## របៀបដំណើរការ {#how-it-works}

1. កំណត់រចនាសម្ព័ន្ធធនធានដែលគិតតាមការប្រើប្រាស់ ដូចជា ទំហំថាស ទទឹងបញ្ជូន ចំនួនទីតាំង ជាដើម
2. កំណត់ដែនកំណត់គម្រោងមូលដ្ឋាន និងអត្រាប្រើលើសកំណត់
3. ប្រព័ន្ធតាមដានការប្រើប្រាស់នៅទូទាំងបណ្តាញរបស់អ្នកដោយស្វ័យប្រវត្តិ
4. នៅពេលអតិថិជនលើសដែនកំណត់ ការគិតថ្លៃលើសកំណត់ត្រូវបានគណនា
5. វិក្កយបត្រត្រូវបានបង្កើតដោយស្វ័យប្រវត្តិផ្អែកលើវដ្តគិតថ្លៃ
6. អតិថិជនអាចមើលការប្រើប្រាស់ និងការគិតថ្លៃខាងមុខរបស់ពួកគេក្នុងពេលវេលាពិត

## ការគិតថ្លៃ AI Token {#ai-token-billing}

កំណែ 1.1.0 បន្ថែមការគិតថ្លៃ AI token សម្រាប់បណ្តាញដែលផ្តល់លក្ខណៈ AI តាមរយៈ WordPress AI Client SDK។ គម្រោងគិតតាមការប្រើប្រាស់ស្តាប់សំណើបង្កើត AI ដែលបានបញ្ចប់ កត់ត្រាចំនួន token បញ្ចូល និងលទ្ធផលសម្រាប់ទីតាំងរងបច្ចុប្បន្ន ហើយបម្លែង token ទាំងនោះទៅជាចំណាយ AI ដែលអាចគិតថ្លៃបាន សម្រាប់រយៈពេលគិតថ្លៃបច្ចុប្បន្នរបស់អតិថិជន។

ប្រើការគិតថ្លៃ AI token នៅពេលអ្នកចង់ផ្តល់ឧបករណ៍ AI លើទីតាំងអតិថិជន ដោយមិនទទួលយកថ្លៃម៉ូដែលដែលពិបាកទាយទុកជាមុន។ សំណើនីមួយៗត្រូវបានភ្ជាប់ទៅនឹងទីតាំងរង អតិថិជន សមាជិកភាព អ្នកផ្តល់សេវា និងម៉ូដែល ដូច្នេះអ្នកគ្រប់គ្រងអាចពិនិត្យការប្រើប្រាស់ ខណៈអតិថិជនអាចមើលថាការប្រើប្រាស់ AI របស់ពួកគេប៉ះពាល់ដល់គណនីរបស់ពួកគេយ៉ាងដូចម្តេច។

### កំណត់រចនាសម្ព័ន្ធអត្រាក្នុងមួយ Token {#configure-per-token-rates}

អត្រា AI ត្រូវបានកំណត់តាមម៉ូដែល ជាអត្រាបញ្ចូល និងអត្រាលទ្ធផលដាច់ដោយឡែក។ អត្រាត្រូវបានគណនាក្នុងមួយ 1 លាន token ដែលត្រូវនឹងទំព័រតម្លៃរបស់អ្នកផ្តល់សេវា AI ទូទៅ។

1. នៅក្នុងអ្នកគ្រប់គ្រងបណ្តាញ បើក **Multisite Ultimate → ការកំណត់ → គម្រោងគិតតាមការប្រើប្រាស់**។
2. បើកការវាស់ចំណាយ AI សម្រាប់ផលិតផល ឬសមាជិកភាពដែលត្រូវគិតថ្លៃការប្រើប្រាស់ AI។
3. បន្ថែមធាតុអត្រាម៉ូដែល ដោយប្រើលេខសម្គាល់ម៉ូដែលដែល connector របស់អ្នកបញ្ជូនត្រឡប់មក ដូចជា `gpt-4o`, `gpt-4o-mini` ឬលេខសម្គាល់ម៉ូដែលដែលឆបគ្នាជាមួយ OpenAI ផ្សេងទៀត។
4. បញ្ចូលតម្លៃ **អត្រាបញ្ចូល** និង **អត្រាលទ្ធផល** ដាច់ដោយឡែក ជារូបិយប័ណ្ណទីតាំងរបស់អ្នក ក្នុងមួយ 1 លាន token។
5. កំណត់ប្រាក់ឧបត្ថម្ភចំណាយ AI ដែលបានរួមបញ្ចូលសម្រាប់អតិថិជន និងឥរិយាបថពេលលើសកំណត់សម្រាប់គម្រោង។

នៅពេលលេខសម្គាល់ម៉ូដែលពិតប្រាកដមិនត្រូវបានកំណត់រចនាសម្ព័ន្ធ គម្រោងគិតតាមការប្រើប្រាស់អាចផ្គូផ្គងតាមបុព្វបទម៉ូដែល។ ឧទាហរណ៍ អត្រាដែលបានកំណត់សម្រាប់ `claude-sonnet-4` អាចអនុវត្តទៅលើលេខសម្គាល់ម៉ូដែលដែលមានត្រាពេលវេលា ហើយចាប់ផ្តើមដោយបុព្វបទនោះ។ អ្នកអភិវឌ្ឍន៍ក៏អាចកែតារាងអត្រាចុងក្រោយដោយប្រើ filter [`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates) បានផងដែរ។

### AI Connectors ដែលគាំទ្រ {#supported-ai-connectors}

ការគិតថ្លៃ AI token ដំណើរការជាមួយ connector ដែលបញ្ចេញព្រឹត្តិការណ៍បង្កើតរបស់ WordPress AI Client SDK និងរួមបញ្ចូលការប្រើប្រាស់ token នៅក្នុង metadata នៃចម្លើយតប។ វារួមបញ្ចូលគ្រួសារ Ultimate Multisite AI connector ដែលបានចងក្រងឯកសារនៅទីនេះ៖

* [Ultimate AI Connector សម្រាប់ Endpoint ដែលឆបគ្នា](../ultimate-ai-connector-compatible-endpoints/) សម្រាប់ API ដែលឆបគ្នាជាមួយ OpenAI រួមមាន OpenAI, Mistral, ច្រកផ្លូវដែលឆបគ្នាជាមួយ Ollama និងអ្នកផ្តល់សេវាផ្សេងទៀតដែលបង្ហាញ metadata ការប្រើប្រាស់ token ដែលឆបគ្នា
* [Ultimate AI Connector សម្រាប់ WebLLM](../ultimate-ai-connector-webllm/) សម្រាប់ការប្រើប្រាស់ WebLLM ដើមនៅក្នុងកម្មវិធីរុករក នៅពេលសំណើត្រូវបានបញ្ជូនតាមលំហូរព្រឹត្តិការណ៍ AI Client SDK
* connector របស់អ្នកផ្តល់សេវា AI ដែលប្រើដោយ [Gratis AI Agent](../gratis-ai-agent/) នៅពេលអ្នកផ្តល់សេវាដែលបានជ្រើសរាយការណ៍ចំនួន token របស់ prompt និង completion តាមរយៈ SDK

connector ផ្ទាល់ខ្លួនអាចចូលរួមបាន ដោយបាញ់ព្រឹត្តិការណ៍ស្តង់ដារ `wp_ai_client_before_generate_result` និង `wp_ai_client_after_generate_result` ជាមួយ metadata ម៉ូដែល អ្នកផ្តល់សេវា និងការប្រើប្រាស់ token។ ការកំណត់ connector ត្រូវបានអនុវត្តពីទីតាំងមេទៅទីតាំងរង ដើម្បីឱ្យទីតាំងអតិថិជនប្រើអ្នកផ្តល់សេវា និងដែនកំណត់ដែលបណ្តាញបានអនុម័ត។

អតិថិជនមើលឃើញការប្រើប្រាស់ AI ជាមួយធនធានគិតតាមការប្រើប្រាស់ផ្សេងទៀតនៅក្នុងតំបន់គណនីរបស់ពួកគេ។ ជួរចំណាយ AI បង្ហាញសរុបនៃរយៈពេលគិតថ្លៃបច្ចុប្បន្ន ដែលបានគណនាពី token prompt និង completion ដែលបានកត់ត្រា អត្រាម៉ូដែលដែលបានកំណត់ និងប្រាក់ឧបត្ថម្ភណាមួយដែលបានរួមបញ្ចូលក្នុងគម្រោង។

ប្រសិនបើអតិថិជនមានវិធីបង់ប្រាក់នៅក្នុងឯកសារ ការប្រើប្រាស់លើសប្រាក់ឧបត្ថម្ភដែលបានរួមបញ្ចូល ត្រូវបានចាត់ទុកជាការប្រើលើសកំណត់ដែលអាចគិតថ្លៃបាន។ ប្រសិនបើគម្រោងត្រូវបានកំណត់សម្រាប់ការកាត់ផ្តាច់រឹង ហើយអតិថិជនមិនមានវិធីបង់ប្រាក់ដែលអាចប្រើបាន សំណើ AI អាចត្រូវបានរារាំងនៅពេលឈានដល់ប្រាក់ឧបត្ថម្ភ ហើយអតិថិជនត្រូវបានស្នើឱ្យបន្ថែមព័ត៌មានបង់ប្រាក់ មុនបន្ត។

## តម្រូវការ {#requirements}

* Multisite Ultimate 2.0.0 ឬខ្ពស់ជាងនេះ
* ការដំឡើង WordPress Multisite
* PHP 7.4 ឬខ្ពស់ជាងនេះ

## ការដំឡើង {#installation}

1. ផ្ទុកឡើងឯកសារ plugin ទៅ `/wp-content/plugins/ultimate-multisite-metered-plans`
2. ធ្វើឱ្យ plugin សកម្មតាមរយៈអេក្រង់ 'Plugins' ក្នុង WordPress
3. ចូលទៅកាន់ Multisite Ultimate → ការកំណត់ → គម្រោងគិតតាមការប្រើប្រាស់
4. កំណត់រចនាសម្ព័ន្ធធនធានគិតតាមការប្រើប្រាស់ និងតម្លៃរបស់អ្នក
5. បើកការគិតថ្លៃតាមការប្រើប្រាស់សម្រាប់ផលិតផលក្នុងកម្មវិធីកែសម្រួលផលិតផល
