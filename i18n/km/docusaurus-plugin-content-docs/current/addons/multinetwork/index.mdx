# បណ្តាញច្រើន {#multi-network}

កម្មវិធីបន្ថែមបណ្តាញច្រើនរបស់ Ultimate Multisite ពង្រីកវេទិកា Ultimate Multisite របស់អ្នកជាមួយមុខងារបណ្តាញច្រើនដ៏មានថាមពល។ កម្មវិធីបន្ថែមនេះអនុញ្ញាតឱ្យអតិថិជនរបស់អ្នកទិញ និងគ្រប់គ្រងបណ្តាញ WordPress multisite ទាំងមូល មិនត្រឹមតែគេហទំព័រដាច់ដោយឡែកប៉ុណ្ណោះទេ។

## មុខងារសំខាន់ៗ {#key-features}

* **ការផ្តល់បណ្តាញ**: បង្កើត និងកំណត់រចនាសម្ព័ន្ធបណ្តាញ WordPress multisite ពេញលេញដោយស្វ័យប្រវត្តិសម្រាប់អតិថិជនរបស់អ្នក
* **គំរូបណ្តាញ**: អនុញ្ញាតឱ្យអតិថិជនរកមើល មើលជាមុន និងជ្រើសគំរូបណ្តាញដែលបានរៀបចំជាមុន មុនពេលចុះឈ្មោះ
* **ការគ្រប់គ្រងបណ្តាញ**: សមត្ថភាពគ្រប់គ្រងពេញលេញសម្រាប់បរិស្ថានបណ្តាញច្រើន
* **បណ្តាញដាច់ដោយឡែក**: បណ្តាញនីមួយៗរបស់អតិថិជនដំណើរការដោយឯករាជ្យ ជាមួយតារាងមូលដ្ឋានទិន្នន័យ និងការកំណត់រចនាសម្ព័ន្ធផ្ទាល់ខ្លួន
* **SSO ឆ្លងបណ្តាញ**: ផ្តល់ឱ្យអតិថិជននូវការចូលប្រើបណ្តាញរងរបស់ពួកគេយ៉ាងរលូន ដោយប្រើការផ្ទៀងផ្ទាត់តាមតំណ magic
* **ស្ថាបត្យកម្មអាចពង្រីកបាន**: គ្រប់គ្រងបណ្តាញច្រើនបានយ៉ាងមានប្រសិទ្ធភាព ជាមួយការបែងចែកធនធានដែលបានបង្កើនប្រសិទ្ធភាព
* **ការរួមបញ្ចូល**: រួមបញ្ចូលយ៉ាងរលូនជាមួយប្រព័ន្ធជាវ និងវិក្កយបត្ររបស់ Ultimate Multisite

## ករណីប្រើប្រាស់ {#use-cases}

* **ការបង្ហោះ White Label**: ផ្តល់ឱ្យអតិថិជននូវបណ្តាញ multisite ដែលមានម៉ាកផ្ទាល់ខ្លួន
* **ដំណោះស្រាយសម្រាប់ភ្នាក់ងារ**: អនុញ្ញាតឱ្យភ្នាក់ងារគ្រប់គ្រងបណ្តាញអតិថិជនច្រើនពីវេទិកាតែមួយ
* **ការដាក់ឱ្យប្រើសម្រាប់សហគ្រាស**: គាំទ្រអង្គការធំៗដែលមានតម្រូវការបណ្តាញច្រើនស្មុគស្មាញ
* **វេទិកាអប់រំ**: បង្កើតបណ្តាញដាច់ដោយឡែកសម្រាប់សាលា ឬនាយកដ្ឋានផ្សេងៗ

## តម្រូវការ {#requirements}

* Ultimate Multisite 2.0.0 ឬខ្ពស់ជាងនេះ
* ការដំឡើង WordPress Multisite
* PHP 7.4 ឬខ្ពស់ជាងនេះ
* MySQL 5.6 ឬខ្ពស់ជាងនេះ

## ការដំឡើង {#installation}

1. ផ្ទុកឡើងឯកសារកម្មវិធីបន្ថែមទៅថត `/wp-content/plugins/ultimate-multisite-multinetwork`
2. ធ្វើឱ្យកម្មវិធីបន្ថែមសកម្មតាមរយៈអេក្រង់កម្មវិធីបន្ថែមក្នុង WordPress
3. កំណត់ការកំណត់បណ្តាញច្រើនតាមរយៈ Ultimate Multisite → ការកំណត់

## គំរូបណ្តាញ {#network-templates}

កំណែ 1.3.0 បន្ថែមលំហូររកឃើញ និងជ្រើសគំរូសម្រាប់អតិថិជនដែលទិញបណ្តាញថ្មី។ ប្រើគំរូបណ្តាញនៅពេលអ្នកចង់ឱ្យបណ្តាញអតិថិជនថ្មីនីមួយៗចាប់ផ្តើមពីរចនាសម្ព័ន្ធ ការរចនា បណ្ណាល័យមាតិកា ឬការរៀបចំតាមវិស័យជាក់លាក់ដែលបានរៀបចំរួច។

### ឧបករណ៍មើលគំរូជាមុន {#template-previewer}

ឧបករណ៍មើលគំរូបណ្តាញជាមុនអនុញ្ញាតឱ្យអតិថិជនរកមើលគំរូដែលមាន មុនពេលបង់ប្រាក់។ គំរូនីមួយៗអាចបើកក្នុងផ្ទាំងមើលជាមុនផ្ទាល់ ដូច្នេះអតិថិជនអាចប្រៀបធៀបប្លង់ និងមាតិកាមុនពេលពួកគេទិញបណ្តាញ។

ប្រើឧបករណ៍មើលជាមុនដើម្បីជួយអតិថិជន៖

* ពិនិត្យមើលរូបរាង និងរចនាសម្ព័ន្ធរបស់គំរូបណ្តាញនីមួយៗដែលមាន
* ប្រៀបធៀបគំរូច្រើនដោយមិនចាកចេញពីលំហូរទិញ
* ជ្រើសគំរូដែលស័ក្តិសមបំផុតនឹងករណីប្រើប្រាស់របស់ពួកគេ មុនពេលចុះឈ្មោះ

រក្សាឈ្មោះ និងការពិពណ៌នាគំរូឱ្យងាយយល់សម្រាប់អតិថិជន។ ឧបករណ៍មើលជាមុនមានប្រយោជន៍បំផុតនៅពេលអតិថិជនអាចយល់ពីករណីប្រើប្រាស់ដែលបានគ្រោងទុក ដូចជាបណ្តាញភ្នាក់ងារ បណ្តាញសាលា បណ្តាញសាខាសិទ្ធិពាណិជ្ជកម្ម ឬកញ្ចប់បង្ហោះ white-label។

### ឧបករណ៍ជ្រើសគំរូពេលបង់ប្រាក់ {#template-picker-at-checkout}

ឧបករណ៍ជ្រើសគំរូអាចបន្ថែមទៅលំហូរបង់ប្រាក់ចុះឈ្មោះជាវាលមួយ។ ពេលបានកំណត់រួច អតិថិជនជ្រើសគំរូបណ្តាញដែលពួកគេចង់បានក្នុងអំឡុងពេលបង់ប្រាក់ ហើយគំរូដែលបានជ្រើសនឹងត្រូវប្រើនៅពេលបណ្តាញថ្មីរបស់ពួកគេត្រូវបានផ្តល់។

ដើម្បីប្រើឧបករណ៍ជ្រើសក្នុងលំហូរបង់ប្រាក់៖

1. រៀបចំបណ្តាញដែលអ្នកចង់ផ្តល់ជាគំរូ។
2. បើកវាលជ្រើសគំរូបណ្តាញច្រើនក្នុងទម្រង់ចុះឈ្មោះ ឬបង់ប្រាក់ដែលពាក់ព័ន្ធ។
3. ជ្រើសគំរូណាខ្លះដែលគួរមានសម្រាប់លំហូរបង់ប្រាក់នោះ។
4. បោះពុម្ពផ្សាយលំហូរបង់ប្រាក់ ហើយសាកល្បងការចុះឈ្មោះ ដើម្បីបញ្ជាក់ថាគំរូដែលបានជ្រើសត្រូវបានអនុវត្តទៅបណ្តាញថ្មី។

បើលំហូរបង់ប្រាក់មួយផ្តល់ផលិតផលបណ្តាញតែមួយ ឧបករណ៍ជ្រើសនៅតែអាចមានប្រយោជន៍ជាជំហានណែនាំចាប់ផ្តើមប្រើ។ សម្រាប់លំហូរលក់ដែលមានគម្រោងច្រើន សូមបង្ហាញតែគំរូដែលឆបគ្នាជាមួយផលិតផល ឬគម្រោងដែលបានជ្រើស។

## SSO តាមតំណ Magic ឆ្លងបណ្តាញ {#cross-network-magic-link-sso}

SSO តាមតំណ magic ឆ្លងបណ្តាញអនុញ្ញាតឱ្យអតិថិជនផ្លាស់ទីរវាងតំបន់គណនីចម្បងរបស់ពួកគេ និងបណ្តាញរង ដោយមិនចាំបាច់ចូលដោយដៃម្តងទៀត។ នៅពេលអតិថិជនតាមតំណចូលប្រើឆ្លងបណ្តាញ Ultimate Multisite នឹងបង្កើតតំណ magic ដែលមានកំណត់ពេលវេលា ហើយផ្ទៀងផ្ទាត់អតិថិជនចូលទៅបណ្តាញគោលដៅ។

វាធ្វើឱ្យបទពិសោធន៍ប្រសើរឡើងសម្រាប់អតិថិជនដែលគ្រប់គ្រងបណ្តាញច្រើនជាងមួយ ឬត្រូវការចូលប្រើបណ្តាញគំរូដែលបានចម្លងថ្មីភ្លាមៗបន្ទាប់ពីទិញ។

បញ្ជីពិនិត្យសម្រាប់អ្នកគ្រប់គ្រង៖

* បញ្ជាក់ថាកម្មវិធីបន្ថែមបណ្តាញច្រើនសកម្មទូទាំងការដំឡើង WordPress multisite។
* ប្រាកដថាអតិថិជនមានសិទ្ធិចូលប្រើបណ្តាញត្រឹមត្រូវ មុនពេលផ្តល់តំណឆ្លងបណ្តាញឱ្យពួកគេ។
* រក្សាដែនបណ្តាញ និងដែនដែលបានផ្គូផ្គងឱ្យបានកំណត់ត្រឹមត្រូវ ដើម្បីឱ្យតំណ magic ដោះស្រាយទៅបណ្តាញគោលដៅដែលបានគ្រោងទុក។
* សាកល្បងផ្ទាំងគ្រប់គ្រងអតិថិជន និងប៊ូតុងចូលប្រើក្រោយបង់ប្រាក់ណាមួយ បន្ទាប់ពីផ្លាស់ប្តូរការកំណត់បណ្តាញ ដែន ឬការបង់ប្រាក់។

តំណ magic គួរត្រូវបានចាត់ទុកជាតំណចូល។ វាត្រូវបានរចនាសម្រាប់ការចូលប្រើរបស់អតិថិជនឱ្យរលូន ហើយមិនគួរបញ្ជូនបន្ត ឬប្រើឡើងវិញក្រៅសម័យដែលបានគ្រោងទុកទេ។

## សំណួរដែលសួរញឹកញាប់ {#frequently-asked-questions}

### តើបណ្តាញច្រើនគឺជាអ្វី? {#what-is-multinetwork}

បណ្តាញច្រើនគឺជាមុខងារ WordPress ដែលអនុញ្ញាតឱ្យអ្នកដំណើរការបណ្តាញ multisite ឯករាជ្យច្រើនលើការដំឡើង WordPress តែមួយ។

### តើវាខុសពី multisite ធម្មតាយ៉ាងដូចម្តេច? {#how-is-this-different-from-regular-multisite}

Multisite ធម្មតាអនុញ្ញាតឱ្យមានគេហទំព័រច្រើនក្នុងបណ្តាញមួយ។ បណ្តាញច្រើនអនុញ្ញាតឱ្យមានបណ្តាញច្រើន ដែលនីមួយៗមានគេហទំព័រផ្ទាល់ខ្លួន ទាំងអស់នៅក្នុងការដំឡើង WordPress តែមួយ។

### តើវាត្រូវការការកំណត់រចនាសម្ព័ន្ធម៉ាស៊ីនបម្រើពិសេសទេ? {#does-this-require-special-server-configuration}

បាទ/ចាស បណ្តាញច្រើនត្រូវការការរៀបចំ និងការកំណត់រចនាសម្ព័ន្ធម៉ាស៊ីនបម្រើជាក់លាក់។ សូមពិគ្រោះឯកសារណែនាំសម្រាប់សេចក្តីណែនាំលម្អិតអំពីការរៀបចំ។
