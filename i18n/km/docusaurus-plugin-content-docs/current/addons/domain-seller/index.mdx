# Domain Seller

**Multisite Ultimate: Domain Seller** បម្លែងបណ្តាញ WordPress Multisite របស់អ្នកឱ្យក្លាយជាវេទិកាលក់ដូមេនដ៏មានអានុភាព។ អនុញ្ញាតឱ្យអតិថិជនចុះបញ្ជីដូមេនផ្ទាល់ខ្លួនដោយផ្ទាល់នៅពេលចុះឈ្មោះ ឬពី Dashboard របស់ពួកគេ ដោយមានការពិនិត្យភាពអាចប្រើបានភ្លាមៗ និងតម្លៃច្បាស់លាស់។

### ហេតុអ្វីត្រូវប្រើ Domain Seller? {#why-domain-seller}

ការចុះបញ្ជីដូមេនគឺជាការលក់បន្ថែមមួយដែលត្រូវបានស្នើសុំច្រើនបំផុតសម្រាប់វេទិកា WaaS (គេហទំព័រជាសេវាកម្ម) ណាមួយ។ ជំនួសឱ្យការបញ្ជូនអតិថិជនរបស់អ្នកទៅអ្នកចុះបញ្ជីភាគីទីបី អ្នកអាចផ្តល់ដូមេនដោយផ្ទាល់បាន — រក្សាចំណូលបន្ថែមនៅក្នុងបណ្តាញរបស់អ្នក ខណៈពេលផ្តល់បទពិសោធន៍រលូនដល់អតិថិជន។

### លក្ខណៈពិសេសសំខាន់ៗ {#key-features}

#### គាំទ្រអ្នកចុះបញ្ជីដូមេនច្រើន {#multiple-domain-registrar-support}
* **OpenSRS** — ការចុះបញ្ជី ការបន្ត និងការគ្រប់គ្រងពេញលេញ
* **Namecheap** — ការរួមបញ្ចូលពេញលេញជាមួយ WhoisGuard និងការសាកល្បង sandbox
* **HostAfrica** — ការស្វែងរកដូមេន តម្លៃ ការចុះបញ្ជី ការផ្ទេរ DNS និងការការពារ ID តាមរយៈ Domains Reseller API
* **Openprovider** — តម្លៃសម្រាប់អ្នកលក់បន្ត ការចុះបញ្ជី ការបន្ត ការផ្ទេរ DNS ឯកជនភាព WHOIS និងការធ្វើសមកាលកម្ម TLD
* **Hostinger** — ការស្វែងរក និងចុះបញ្ជីដូមេនដោយប្រើ Hostinger integration token រួម
* **ស្ថាបត្យកម្មអាចពង្រីកបាន** — សាងសង់លើលំនាំ Integration Registry ធ្វើឱ្យងាយស្រួលក្នុងការបន្ថែមអ្នកផ្តល់សេវាច្រើនទៀត
* **ការធ្វើតេស្តការតភ្ជាប់មានស្រាប់** — ផ្ទៀងផ្ទាត់ព័ត៌មានសម្ងាត់ និងការតភ្ជាប់របស់អ្នកផ្តល់សេវាមុនពេលដាក់ឱ្យប្រើពិត

#### អ្នកជំនួយការរៀបចំមានការណែនាំ {#guided-setup-wizard}
* **ការចាប់ផ្តើមប្រើដោយចុចមួយដង** — នាំអ្នកឆ្លងកាត់ការកំណត់រចនាសម្ព័ន្ធអ្នកផ្តល់សេវា ការនាំចូល TLD និងការបង្កើតផលិតផលដូមេន
* **ការធ្វើសមកាលកម្ម TLD ដោយស្វ័យប្រវត្តិ** — នាំចូល TLD ទាំងអស់ដែលមាន និងតម្លៃលក់ដុំពីអ្នកផ្តល់សេវាដែលបានភ្ជាប់
* **ការបង្កើតផលិតផលលំនាំដើម** — បង្កើតផលិតផលដូមេនដែលរួចរាល់សម្រាប់ប្រើ ជាមួយតម្លៃបន្ថែមលំនាំដើមសមហេតុផល

#### តម្លៃដូមេនបត់បែនបាន {#flexible-domain-pricing}
* **របៀបបន្ថែមតម្លៃបីប្រភេទ** — បន្ថែមតម្លៃជាភាគរយ បន្ថែមតម្លៃថេរ ឬតម្លៃថេរក្នុងមួយ TLD
* **ផលិតផលតាម TLD ជាក់លាក់** — បង្កើតផលិតផលដូមេនផ្សេងៗសម្រាប់ក្រុម TLD ផ្សេងៗ (.com/.net, .io, កូដប្រទេស ជាដើម)
* **តម្លៃណែនាំដំបូង** — ផ្តល់តម្លៃបញ្ចុះសម្រាប់ឆ្នាំដំបូង ដើម្បីទាក់ទាញការចុះបញ្ជីថ្មី
* **តម្លៃភ្លាមៗ** — តម្លៃលក់ដុំបន្តផ្ទាល់ពីអ្នកផ្តល់សេវា ជាមួយការរក្សាទុកឆ្លាតវៃ
* **គាំទ្រគូប៉ុង** — សមស្របពេញលេញជាមួយកូដបញ្ចុះតម្លៃ Multisite Ultimate

#### ការការពារឯកជនភាព WHOIS {#whois-privacy-protection}
* **ការកំណត់រចនាសម្ព័ន្ធតាមផលិតផល** — កំណត់ឯកជនភាព WHOIS ជា បិទ រួមបញ្ចូលជានិច្ច ឬជម្រើសអតិថិជន
* **ការជ្រើសរើសចូលរបស់អតិថិជន** — នៅពេលកំណត់ជា ជម្រើសអតិថិជន ប្រអប់ធីកនៅពេលបង់ប្រាក់អនុញ្ញាតឱ្យអតិថិជនបន្ថែមការការពារឯកជនភាព
* **តម្លៃអាចកំណត់រចនាសម្ព័ន្ធបាន** — កំណត់តម្លៃក្នុងមួយឆ្នាំសម្រាប់កម្មវិធីបន្ថែមឯកជនភាព ឬផ្តល់ឱ្យដោយឥតគិតថ្លៃ
* **ការរួមបញ្ចូលជាមួយអ្នកចុះបញ្ជី** — បើក WhoisGuard (Namecheap) ឬឯកជនភាព WHOIS (OpenSRS) ដោយស្វ័យប្រវត្តិនៅពេលចុះបញ្ជី

#### បទពិសោធន៍បង់ប្រាក់រួម {#unified-checkout-experience}
* **របៀបដូមេនបីប្រភេទ** — អតិថិជនជ្រើសរើសរវាងដូមេនរង (ឥតគិតថ្លៃ) ការចុះបញ្ជីដូមេនថ្មី ឬការភ្ជាប់ដូមេនដែលមានស្រាប់ដែលពួកគេកាន់កាប់រួចហើយ
* **ការស្វែងរកដូមេនភ្លាមៗ** — ពិនិត្យភាពអាចប្រើបានដោយប្រើ AJAX ដោយមិនចាំបាច់ផ្ទុកទំព័រឡើងវិញ
* **ការណែនាំឆ្លាតវៃ** — ណែនាំ TLD ជំនួសនៅពេលដូមេនដែលចង់បានត្រូវបានគេយកហើយ
* **តម្លៃច្បាស់លាស់** — បង្ហាញតម្លៃចុះបញ្ជី តម្លៃបន្ត និងថ្លៃឯកជនភាព WHOIS ជាជម្រើសយ៉ាងច្បាស់
* **វាលទំនាក់ទំនងអ្នកចុះបញ្ជី** — ប្រមូលព័ត៌មានអ្នកចុះបញ្ជីដែលត្រូវការ (ឈ្មោះ អាសយដ្ឋាន ទូរស័ព្ទ) នៅក្នុងដំណើរការបង់ប្រាក់
* **URL គេហទំព័របង្កើតដោយស្វ័យប្រវត្តិ** — អាសយដ្ឋានគេហទំព័រត្រូវបានកំណត់ដោយស្វ័យប្រវត្តិពីដូមេនដែលបានជ្រើស

#### ប្រព័ន្ធបន្តដោយស្វ័យប្រវត្តិ {#auto-renewal-system}
* **ការបន្តភ្ជាប់នឹងសមាជិកភាព** — ដូមេនបន្តដោយស្វ័យប្រវត្តិខណៈពេលសមាជិកភាពរបស់អតិថិជននៅសកម្ម
* **ការកំណត់ពេលឆ្លាតវៃ** — ការងារបន្តត្រូវបានដាក់ជាជួរដោយផ្អែកលើកាលបរិច្ឆេទផុតកំណត់របស់ដូមេន
* **តក្កវិជ្ជាព្យាយាមឡើងវិញ** — ព្យាយាមឡើងវិញដោយឆ្លាតវៃសម្រាប់ការប៉ុនប៉ងបន្តដែលបរាជ័យ
* **ការជូនដំណឹងតាមអ៊ីមែល** — ការជូនដំណឹងអាចកំណត់រចនាសម្ព័ន្ធបានសម្រាប់ការបន្តជិតមកដល់ ការបន្តជោគជ័យ ការបរាជ័យ និងការផុតកំណត់

#### ការគ្រប់គ្រង DNS របស់អតិថិជន {#customer-dns-management}
* **DNS សេវាខ្លួនឯង** — អតិថិជនអាចបន្ថែម កែសម្រួល និងលុបកំណត់ត្រា DNS (A, AAAA, CNAME, MX, TXT) ពី Dashboard របស់ពួកគេ
* **ស្ថានភាពដូមេន** — ស្ថានភាពចុះបញ្ជីភ្លាមៗ កាលបរិច្ឆេទផុតកំណត់ និងព័ត៌មាន nameserver
* **ការភ្ជាប់ដូមេនដោយស្វ័យប្រវត្តិ** — ដូមេនដែលបានចុះបញ្ជី ឬដូមេនដែលមានស្រាប់ ត្រូវបានភ្ជាប់ទៅគេហទំព័ររបស់អតិថិជនដោយស្វ័យប្រវត្តិ

#### ឧបករណ៍អ្នកគ្រប់គ្រង {#admin-tools}
* **ការចុះបញ្ជីដូមេនដោយដៃ** — ចុះបញ្ជីដូមេនជំនួសអតិថិជនពីផ្ទាំងអ្នកគ្រប់គ្រង
* **ការតាមដានដូមេន** — មើលដូមេន ស្ថានភាព និងកាលបរិច្ឆេទផុតកំណត់របស់អតិថិជនទាំងអស់ពីទំព័រតែមួយ
* **ការកត់ត្រាតាមអ្នកផ្តល់សេវាជាក់លាក់** — បណ្តាញកំណត់ហេតុដាច់ដោយឡែកសម្រាប់អ្នកចុះបញ្ជីនីមួយៗ (ឧ. domain-seller-namecheap.log)
* **ព័ត៌មានលម្អិតការចុះបញ្ជី** — មើលព័ត៌មានអ្នកចុះបញ្ជី និងទិន្នន័យចុះបញ្ជីនៅលើទំព័រកែសម្រួលដូមេនស្នូល

### អ្នកផ្តល់សេវាដែលគាំទ្រ {#supported-providers}

* **OpenSRS** — ការរួមបញ្ចូលពេញលេញជាមួយការចុះបញ្ជី ការបន្ត ការគ្រប់គ្រង DNS និងឯកជនភាព WHOIS
* **Namecheap** — ការរួមបញ្ចូលពេញលេញជាមួយការចុះបញ្ជី ការបន្ត ការពិនិត្យភាពអាចប្រើបានជាក្រុម និង WhoisGuard ឥតគិតថ្លៃ
* **HostAfrica** — ការចុះបញ្ជី ការបន្ត ការផ្ទេរ ការគ្រប់គ្រង DNS ការទាញយកកូដ EPP ការចាក់សោរបស់អ្នកចុះបញ្ជី និងការការពារ ID
* **Openprovider** — ការចុះបញ្ជី ការបន្ត ការផ្ទេរ ការគ្រប់គ្រង DNS ការទាញយកកូដ EPP ការចាក់សោរបស់អ្នកចុះបញ្ជី ឯកជនភាព WHOIS និងការធ្វើសមកាលកម្ម TLD សម្រាប់អ្នកលក់បន្ត
* **Hostinger** — ការចុះបញ្ជី ការរកមើលដូមេន ការធ្វើបច្ចុប្បន្នភាព nameserver ការចាក់សោរបស់អ្នកចុះបញ្ជី និងឯកជនភាព WHOIS; ការបន្តត្រូវបានគ្រប់គ្រងដោយ Hostinger auto-renew
* **GoDaddy** — ការចុះបញ្ជី និងការបន្ត
* **ResellerClub** — ការចុះបញ្ជី ការបន្ត និងការគ្រប់គ្រង DNS
* **NameSilo** — ការចុះបញ្ជី និងការបន្ត
* **Enom** — ការចុះបញ្ជី ការបន្ត និងការគ្រប់គ្រង DNS

### ព័ត៌មានលម្អិតបច្ចេកទេស {#technical-details}

* **តម្រូវឱ្យមាន៖** PHP 7.4+, Multisite Ultimate 2.4.12+
* **ស្ថាបត្យកម្ម៖** លំនាំ Integration Registry ជាមួយ Domain_Selling_Capability interface
* **ការផ្ទុកដោយស្វ័យប្រវត្តិ៖** Jetpack autoloader ជាមួយការផ្គូផ្គង class តាម PSR-4
* **ស្តង់ដារ៖** ស្តង់ដារសរសេរកូដ WordPress, strict typing, prepared statements

## ការដំឡើង {#installation}

1. ត្រូវប្រាកដថា **Multisite Ultimate v2.4.12+** ត្រូវបានដំឡើង និងបើកដំណើរការលើបណ្តាញ
2. ផ្ទុក plugin ឡើងទៅ `/wp-content/plugins/ultimate-multisite-domain-seller/`
3. បើកដំណើរការ plugin លើបណ្តាញពី **Plugin** នៅក្នុង WordPress ផ្នែកគ្រប់គ្រងបណ្តាញ
4. **អ្នកជំនួយការដំឡើង** នឹងចាប់ផ្តើមដោយស្វ័យប្រវត្តិនៅពេលបើកដំណើរការលើកដំបូង — អនុវត្តតាមជំហានដើម្បី៖
   * ភ្ជាប់ registrar ដែនរបស់អ្នក
   * សាកល្បងការតភ្ជាប់
   * នាំចូល TLDs ដែលមាន និងតម្លៃលក់ដុំ
   * បង្កើតផលិតផលដែនដំបូងរបស់អ្នកជាមួយការបន្ថែមតម្លៃ
5. បន្ថែម​វាល **ការជ្រើសរើសដែន** ទៅក្នុងទម្រង់ checkout របស់អ្នកនៅក្នុង **Ultimate Multisite > ទម្រង់ Checkout**

## សំណួរដែលសួរញឹកញាប់ {#frequently-asked-questions}

### ត្រូវការ Multisite Ultimate កំណែណា? {#what-version-of-multisite-ultimate-is-required}

កំណែ 2.4.12 ឬខ្ពស់ជាងនេះ។ addon នេះមិនឆបគ្នាជាមួយ WP Ultimo v1.x ឬកំណែ Ultimate Multisite ទាបជាង 2.4.12 ទេ។

### តើគាំទ្រ provider ដែនណាខ្លះ? {#which-domain-providers-are-supported}

មាន registrar ប្រាំបួនដែលត្រូវបានគាំទ្រ៖ **OpenSRS**, **Namecheap**, **HostAfrica**, **Openprovider**, **Hostinger**, **GoDaddy**, **ResellerClub**, **NameSilo**, និង **Enom**។ សមត្ថភាពខុសគ្នាតាម provider ព្រោះ API របស់ registrar បង្ហាញប្រតិបត្តិការខុសៗគ្នា។ សូមមើលមគ្គុទ្ទេសក៍ដំឡើងសម្រាប់ទិដ្ឋភាពទូទៅតាម provider នីមួយៗ។ ស្ថាបត្យកម្ម Integration Registry របស់ addon ធ្វើឱ្យការបន្ថែម provider ផ្ទាល់ខ្លួនមានភាពងាយស្រួល — សូមមើល [មគ្គុទ្ទេសក៍អ្នកអភិវឌ្ឍន៍](/developer/integration-guide/custom-registrar)។

### តើខ្ញុំអាចផ្តល់ការការពារឯកជនភាព WHOIS បានទេ? {#can-i-offer-whois-privacy-protection}

បាន។ ផលិតផលដែននីមួយៗមានការកំណត់ឯកជនភាព WHOIS ជាមួយរបៀបបី៖ បិទ, រួមបញ្ចូលជានិច្ច, ឬអតិថិជនជ្រើសរើស។ ពេលកំណត់ជា អតិថិជនជ្រើសរើស អតិថិជននឹងឃើញប្រអប់ធីកនៅពេល checkout ជាមួយតម្លៃ (ឬ "ឥតគិតថ្លៃ" សម្រាប់ WhoisGuard របស់ Namecheap)។ សញ្ញាសម្គាល់ឯកជនភាពត្រូវបានផ្ញើទៅ registrar ដោយស្វ័យប្រវត្តិនៅពេលចុះឈ្មោះ។

### តើខ្ញុំអាចកំណត់តម្លៃខុសៗគ្នាសម្រាប់ TLDs ផ្សេងៗបានទេ? {#can-i-set-different-pricing-for-different-tlds}

បាន។ បង្កើតផលិតផលដែនដាច់ដោយឡែកសម្រាប់ក្រុម TLD ផ្សេងៗ។ ផលិតផលនីមួយៗអាចមានប្រភេទការបន្ថែមតម្លៃផ្ទាល់ខ្លួន (ភាគរយ, ការបន្ថែមតម្លៃថេរ, ឬតម្លៃថេរ), TLDs ដែលគាំទ្រ, និងតម្លៃណែនាំដំបូង។

### តើអតិថិជនឃើញជម្រើស checkout អ្វីខ្លះ? {#what-checkout-options-do-customers-see}

វាលការជ្រើសរើសដែនផ្តល់ផ្ទាំងបីដល់អតិថិជន៖ **ដែនរង** (ប្រើដែនរងឥតគិតថ្លៃលើបណ្តាញរបស់អ្នក), **ចុះឈ្មោះដែនថ្មី** (ស្វែងរក និងចុះឈ្មោះដែនថ្មី), និង **ដែនដែលមានស្រាប់** (ភ្ជាប់ដែនដែលពួកគេកាន់កាប់រួចហើយ)។ មានតែរបៀបដែលអ្នកបើកប៉ុណ្ណោះនឹងបង្ហាញ។

### តើវាគាំទ្រការបន្តស្វ័យប្រវត្តិទេ? {#does-it-support-automatic-renewals}

បាទ/ចាស។ ដែនអាចត្រូវបានកំណត់ឱ្យបន្តដោយស្វ័យប្រវត្តិ ហើយត្រូវបានភ្ជាប់ទៅស្ថានភាព membership របស់អតិថិជន។ ការទូទាត់បន្តត្រូវបានដំណើរការតាមរយៈ payment gateway ដែលមានស្រាប់របស់អតិថិជន។

### តើអតិថិជនអាចគ្រប់គ្រងកំណត់ត្រា DNS របស់ពួកគេបានទេ? {#can-customers-manage-their-dns-records}

បាន។ អតិថិជនទទួលបានចំណុចប្រទាក់គ្រប់គ្រង DNS នៅក្នុង dashboard របស់ពួកគេ ដើម្បីបន្ថែម កែសម្រួល និងលុបកំណត់ត្រា A, AAAA, CNAME, MX, និង TXT។ ការគ្រប់គ្រង DNS មានជាមួយ OpenSRS, ResellerClub, Enom, HostAfrica, និង Openprovider។ DNS របស់ Hostinger សម្រាប់ដែនដែលបានបង្ហោះត្រូវបានគ្រប់គ្រងដោយ integration ភ្ជាប់ដែនស្នូលរបស់ Hostinger។

### តើមានអ្នកជំនួយការដំឡើងទេ? {#is-there-a-setup-wizard}

មាន។ អ្នកជំនួយការដំឡើងដំណើរការដោយស្វ័យប្រវត្តិនៅពេលបើកដំណើរការលើកដំបូង ហើយណែនាំអ្នកឱ្យភ្ជាប់ registrar, នាំចូល TLDs, និងបង្កើតផលិតផលដែនដំបូងរបស់អ្នក។ អ្នកក៏អាចដំណើរការអ្នកជំនួយការឡើងវិញពីការកំណត់ addon បានផងដែរ។
