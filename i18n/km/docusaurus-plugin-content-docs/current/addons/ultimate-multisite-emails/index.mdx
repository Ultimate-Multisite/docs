# អ៊ីមែល

កម្មវិធីបន្ថែមអ៊ីមែលរបស់ Ultimate Multisite អនុញ្ញាតឱ្យម្ចាស់បណ្តាញរៀបចំ និងគ្រប់គ្រងគណនីអ៊ីមែលអតិថិជន ជាមួយនឹងគេហទំព័រ និងសមាជិកភាព។ កំណែ 1.1.0 បន្ថែមការគ្រប់គ្រងគណនីអ៊ីមែលសម្រាប់អតិថិជន ការចូលតែម្តងទៅវេបមែល ការគាំទ្រ forwarder របស់ CyberPanel និងការកែលម្អការរៀបចំពេលបង់ប្រាក់។

## មុខងារសំខាន់ៗ {#key-features}

* **វីជេតគណនីអ៊ីមែលអតិថិជន**: អតិថិជនអាចបង្កើត និងគ្រប់គ្រងប្រអប់សំបុត្រពីទំព័រ Account របស់ពួកគេ បន្ទាប់ពីពួកគេមានសមាជិកភាពអ៊ីមែលសកម្ម។
* **Webmail SSO និងតំណភ្ជាប់វេទមន្ត**: អតិថិជនអាចបើកវេបមែលដោយផ្ទាល់ពីទំព័រ Account ដោយផ្ទៀងផ្ទាត់តាមចុចមួយដង នៅពេលអ្នកផ្តល់សេវាគាំទ្រវា។
* **ការផ្លាស់ប្តូរពាក្យសម្ងាត់**: អតិថិជន និងអ្នកគ្រប់គ្រងអាចធ្វើបច្ចុប្បន្នភាពពាក្យសម្ងាត់ប្រអប់សំបុត្រពីវីជេតគ្រប់គ្រងអ៊ីមែល។
* **ការរៀបចំសិទ្ធិពេលបង់ប្រាក់**: ការបង់ប្រាក់បង្កើតសមាជិកភាពអ៊ីមែល ភ្ជាប់វាទៅដែនគេហទំព័ររបស់អតិថិជន និងធ្វើឱ្យឧបករណ៍ប្រអប់សំបុត្រលើទំព័រ Account អាចប្រើបាន។
* **Forwarder របស់ CyberPanel**: បណ្តាញដែលប្រើ CyberPanel អាចបង្កើត និងគ្រប់គ្រង email forwarder ជាផ្នែកនៃលំហូរសេវាអ៊ីមែល។
* **ការកំណត់ដែលស្គាល់អ្នកផ្តល់សេវា**: ទំព័រការកំណត់ត្រងអ្នកផ្តល់សេវាទៅជាសមាហរណកម្មដែលសកម្ម និងបានកំណត់រចនាសម្ព័ន្ធ ដូច្នេះអ្នកគ្រប់គ្រងឃើញតែជម្រើសដែលអាចប្រើបាន។

## តម្រូវការ {#requirements}

* Ultimate Multisite 2.0.0 ឬខ្ពស់ជាងនេះ
* ការដំឡើង WordPress Multisite
* យ៉ាងហោចណាស់មានសមាហរណកម្មអ្នកផ្តល់សេវាអ៊ីមែលមួយដែលបានកំណត់រចនាសម្ព័ន្ធ
* ព័ត៌មានសម្ងាត់ API របស់អ្នកផ្តល់សេវា ដែលមានសិទ្ធិបង្កើតប្រអប់សំបុត្រ ផ្លាស់ប្តូរពាក្យសម្ងាត់ និងចេញតំណចូលវេបមែល នៅកន្លែងដែលគាំទ្រ

## ការដំឡើង {#installation}

1. ផ្ទុកឡើងឯកសារ plugin ទៅ `/wp-content/plugins/ultimate-multisite-emails`។
2. បើកដំណើរការកម្មវិធីបន្ថែមទូទាំងបណ្តាញពី **Plugins** នៅក្នុងផ្នែកគ្រប់គ្រងបណ្តាញ WordPress។
3. ទៅកាន់ **Ultimate Multisite** > **ការកំណត់** ហើយបើកការកំណត់អ៊ីមែលរបស់កម្មវិធីបន្ថែម។
4. បើកសមាហរណកម្មអ្នកផ្តល់សេវាអ៊ីមែលដែលអ្នកចង់ឱ្យអតិថិជនប្រើ។
5. រក្សាទុកការកំណត់ បន្ទាប់មកសាកល្បងការបង្កើតគណនីជាមួយអតិថិជនដែលមិនមែនផលិតកម្ម ឬគេហទំព័រសាកល្បង។

## របៀបដែលលំហូរលក់អ៊ីមែលដំណើរការ {#how-the-email-seller-flow-works}

ការលក់អ៊ីមែលមានពីរស្រទាប់ដាច់ដោយឡែក៖

1. **សិទ្ធិប្រើប្រាស់** — ផលិតផល/សមាជិកភាពអ៊ីមែល Ultimate Multisite ដែលបញ្ជាក់ថាអតិថិជនត្រូវបានអនុញ្ញាតឱ្យប្រើអ៊ីមែលសម្រាប់ដែនគេហទំព័រជាក់លាក់ អ្នកផ្តល់សេវា កូតា និងកម្រិតគណនី។
2. **ការបង្កើតប្រអប់សំបុត្រ** — ប្រអប់សំបុត្រពិតរបស់អ្នកផ្តល់សេវា ដូចជា `info@example.com` ដែលត្រូវបានបង្កើតនៅពេលអតិថិជន ឬអ្នកគ្រប់គ្រងបណ្តាញជ្រើសរើសផ្នែកមូលដ្ឋាន និងពាក្យសម្ងាត់។

ក្នុងអំឡុងពេលបង់ប្រាក់ កម្មវិធីបន្ថែមបង្កើត ឬបើកដំណើរការ **សមាជិកភាពអ៊ីមែល** ហើយកត់ត្រាដែនគេហទំព័រ និងអ្នកផ្តល់សេវារបស់អតិថិជនលើសមាជិកភាពនោះ។ វាមិនទាយឈ្មោះប្រអប់សំបុត្រ ឬពាក្យសម្ងាត់ទេ។ បន្ទាប់ពីការទិញបានបញ្ចប់ ហើយមានបរិបទគេហទំព័រ/ដែន អតិថិជនអាចបើកទំព័រ Account ហើយបង្កើតឈ្មោះប្រអប់សំបុត្រដែលពួកគេចង់បាន។

នេះមានន័យថាលំហូរអតិថិជនធម្មតាគឺ៖

1. អតិថិជនទិញគម្រោងគេហទំព័រ ហើយបន្ថែមផលិតផលអ៊ីមែលក្នុងអំឡុងពេលបង់ប្រាក់។
2. Ultimate Multisite បង្កើតសមាជិកភាពគេហទំព័រ និងសមាជិកភាពអ៊ីមែលបន្ទាប់បន្សំ។
3. កម្មវិធីបន្ថែម Emails ភ្ជាប់សមាជិកភាពអ៊ីមែលទៅដែនគេហទំព័រ និងអ្នកផ្តល់សេវារបស់អតិថិជន។
4. អតិថិជនបើក **Account** នៅក្នុង Dashboard របស់ពួកគេ។
5. ផ្ទាំង **គណនីអ៊ីមែល** បង្ហាញឡើងសម្រាប់សមាជិកភាពអ៊ីមែលសកម្មនោះ។
6. អតិថិជនចុច **បង្កើត Account** បញ្ចូលផ្នែកមូលដ្ឋានដូចជា `info` កំណត់ពាក្យសម្ងាត់ ហើយកម្មវិធីបន្ថែមបង្កើត `info@their-domain.com` នៅអ្នកផ្តល់សេវាដែលបានកំណត់រចនាសម្ព័ន្ធ។

## កំណត់រចនាសម្ព័ន្ធផលិតផលអ៊ីមែល {#configure-an-email-product}

បង្កើតផលិតផលអ៊ីមែលមួយសម្រាប់កញ្ចប់អ៊ីមែលនីមួយៗដែលអ្នកចង់លក់ ឬផ្តល់ឱ្យដោយឥតគិតថ្លៃ។ ប្រសិនបើអ្នកថ្មីចំពោះការរៀបចំផលិតផល សូមចាប់ផ្តើមពី [ការបង្កើតផលិតផលជាវដំបូងរបស់អ្នក](../../user-guide/configuration/creating-your-first-subscription-product) ហើយបន្ទាប់មកត្រឡប់មកទីនេះសម្រាប់ការកំណត់ជាក់លាក់សម្រាប់អ៊ីមែល។

1. ទៅកាន់ **ការគ្រប់គ្រងបណ្តាញ > Ultimate Multisite > ផលិតផល > បន្ថែមផលិតផល**។
2. កំណត់ **ប្រភេទផលិតផល** ទៅជា **អ៊ីមែល**។
3. កំណត់តម្លៃ។ ប្រើតម្លៃកើតឡើងវិញដែលត្រូវបង់សម្រាប់គម្រោងលក់ ឬកំណត់ផលិតផលឱ្យឥតគិតថ្លៃ/តម្លៃសូន្យ ប្រសិនបើវាគួរជាគម្រោងអ៊ីមែលបន្ថែមជូន។
4. បើកផ្នែក **ការកំណត់អ៊ីមែល**។
5. ជ្រើសរើស **អ្នកផ្តល់សេវាអ៊ីមែល** ដែលបានកំណត់រចនាសម្ព័ន្ធ។
6. កំណត់ **ចំនួនគណនី**, **កូតាផ្ទុក (MB)**, កម្រិត alias, forwarder និង autoresponder តាមតម្រូវការ។
7. រក្សាទុកផលិតផល។

មានតែអ្នកផ្តល់សេវាដែលបានបើក និងបានកំណត់រចនាសម្ព័ន្ធពេញលេញលើទំព័រ **សមាហរណកម្ម** ប៉ុណ្ណោះដែលបង្ហាញក្នុងឧបករណ៍ជ្រើសរើសអ្នកផ្តល់សេវា។ ប្រសិនបើឧបករណ៍ជ្រើសរើសទទេ សូមកំណត់រចនាសម្ព័ន្ធ និងសាកល្បងអ្នកផ្តល់សេវាជាមុន។

## បន្ថែមអ៊ីមែលទៅការបង់ប្រាក់ {#add-email-to-checkout}

ផលិតផលអ៊ីមែលជាទូទៅត្រូវបានផ្តល់ជាការបន្ថែមទៅគម្រោងគេហទំព័រ។ សូមមើល [ទម្រង់បង់ប្រាក់](../../user-guide/configuration/checkout-forms) សម្រាប់លំហូរការងារទូទៅរបស់កម្មវិធីកែសម្រួលការបង់ប្រាក់។

1. បង្កើត ឬកែសម្រួលគម្រោងគេហទំព័រដែលអតិថិជនទិញជាមុន។
2. បន្ថែមផលិតផលអ៊ីមែលជាការបន្ថែម/ការផ្តល់ជូនបន្ថែមដែលមានសម្រាប់លំហូរបង់ប្រាក់នោះ។
3. បញ្ជាក់ថាទម្រង់បង់ប្រាក់នៅតែរួមបញ្ចូលវាលគេហទំព័រចាំបាច់ធម្មតា ជាពិសេសវាល URL/ដែនគេហទំព័រ។
4. ដំណើរការការបង់ប្រាក់សាកល្បងជាមួយអតិថិជនសាកល្បង។

## ផ្ដល់គម្រោងអ៊ីមែលឥតគិតថ្លៃឱ្យអតិថិជន {#giving-a-customer-a-free-email-plan}

ដើម្បីផ្ដល់អ៊ីមែលឱ្យអតិថិជនដោយមិនគិតថ្លៃ សូមផ្ដល់ **សិទ្ធិប្រើអ៊ីមែល** ជាមុន បន្ទាប់មកបង្កើត ឬអនុញ្ញាតឱ្យអតិថិជនបង្កើតប្រអប់សំបុត្រ។

ជម្រើសដែលបានណែនាំ៖
