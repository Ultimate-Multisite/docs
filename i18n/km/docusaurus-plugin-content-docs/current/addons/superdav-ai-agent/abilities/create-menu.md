# បង្កើត​ម៉ឺនុយ {#create-menu}

សមត្ថភាព **បង្កើត​ម៉ឺនុយ** បង្កើត​ម៉ឺនុយ​រុករក​របស់ WordPress ដោយ​គាំទ្រ​ស្លាក​រុករក​ដាច់ដោយឡែក​ពី​ចំណងជើង​ទំព័រ។

## ទិដ្ឋភាពទូទៅ {#overview}

សមត្ថភាព​នេះ​ពង្រីក​មុខងារ​បង្កើត​ម៉ឺនុយ​ស្តង់ដារ ដោយ​មាន​សមត្ថភាព​បញ្ជាក់​ប៉ារ៉ាម៉ែត្រ `navigation_label`។ វា​អនុញ្ញាត​ឱ្យ​អ្នក​បង្កើត​ម៉ឺនុយ ដែល​ស្លាក​បង្ហាញ​ក្នុង​ការ​រុករក​ខុស​ពី​ចំណងជើង​ទំព័រ ដើម្បី​ផ្តល់​ភាព​បត់បែន​កាន់តែ​ច្រើន​ក្នុង​រចនាសម្ព័ន្ធ​គេហទំព័រ និង​បទពិសោធន៍​អ្នកប្រើ។

## ប៉ារ៉ាម៉ែត្រ {#parameters}

| ប៉ារ៉ាម៉ែត្រ | ប្រភេទ | ត្រូវការ | ការពិពណ៌នា |
|-----------|------|----------|-------------|
| `name` | string | បាទ/ចាស | ឈ្មោះ​ម៉ឺនុយ ឧទាហរណ៍ `Primary Navigation` |
| `location` | string | ទេ | ទីតាំង theme ដើម្បី​កំណត់​ម៉ឺនុយ​នេះ​ទៅ ឧទាហរណ៍ `primary` |
| `navigation_label` | string | ទេ | ស្លាក​សម្រាប់​បង្ហាញ​ក្នុង​ការ​រុករក (ខុស​ពី​ចំណងជើង​ទំព័រ) |

## តម្លៃ​ត្រឡប់ {#return-value}

```json
{
  "success": true,
  "menu_id": 7,
  "name": "Primary Navigation",
  "navigation_label": "Main Menu"
}
```

## ស្លាក​រុករក ប្រៀបធៀប​នឹង​ចំណងជើង​ទំព័រ {#navigation-label-vs-page-title}

ប៉ារ៉ាម៉ែត្រ `navigation_label` អនុញ្ញាត​ឱ្យ​អ្នក​បំបែក​ឈ្មោះ​ម៉ឺនុយ​ខាងក្នុង​ពី​ស្លាក​ដែល​បង្ហាញ​ដល់​អ្នកប្រើ៖

- **`name`** — អត្តសញ្ញាណ​ម៉ឺនុយ​ខាងក្នុង​ដែល WordPress ប្រើ (ឧទាហរណ៍ "Primary Navigation")
- **`navigation_label`** — ស្លាក​ដែល​បង្ហាញ​ដល់​អ្នកចូលមើល​គេហទំព័រ​ក្នុង​ការ​រុករក (ឧទាហរណ៍ "Main Menu")

វា​មាន​ប្រយោជន៍​នៅពេល៖
- របៀប​ដាក់​ឈ្មោះ​ខាងក្នុង​របស់​អ្នក​ខុស​ពី​ស្លាក​ដែល​អ្នកប្រើ​ឃើញ
- អ្នក​ចង់​បាន​ស្លាក​ខ្លី​ជាង​ក្នុង​ការ​រុករក បើ​ធៀប​នឹង​ក្នុង​ផ្ទាំង​គ្រប់គ្រង
- អ្នក​ត្រូវ​គាំទ្រ​ភាសា​ច្រើន​ដែល​មាន​ប្រវែង​ស្លាក​ខុសៗ​គ្នា
- អ្នក​កំពុង​បង្កើត​ម៉ឺនុយ​សម្រាប់​តំបន់ ឬ​ក្រុម​អ្នកប្រើ​ជាក់លាក់

## ឧទាហរណ៍​ការប្រើប្រាស់ {#usage-examples}

### ឧទាហរណ៍ 1៖ ម៉ឺនុយ​សាមញ្ញ​ដែល​មាន​ស្លាក​រុករក {#example-1-simple-menu-with-navigation-label}

**សំណើ៖**
```
បង្កើត​ម៉ឺនុយ​រុករក​ចម្បង​ឈ្មោះ "Main Navigation" ដែល​មាន​ស្លាក​រុករក "Menu"។
```

**លទ្ធផល៖**
```json
{
  "success": true,
  "menu_id": 7,
  "name": "Main Navigation",
  "navigation_label": "Menu"
}
```

### ឧទាហរណ៍ 2៖ ម៉ឺនុយ​សម្រាប់​ទីតាំង Theme ជាក់លាក់ {#example-2-menu-for-specific-theme-location}

**សំណើ៖**
```
បង្កើត​ម៉ឺនុយ​ជើង​ទំព័រ​ឈ្មោះ "Footer Links" ដែល​មាន​ស្លាក​រុករក "Quick Links" ហើយ​កំណត់​វា​ទៅ​ទីតាំង​ជើង​ទំព័រ។
```

**លទ្ធផល៖**
```json
{
  "success": true,
  "menu_id": 8,
  "name": "Footer Links",
  "navigation_label": "Quick Links",
  "location": "footer"
}
```

## ការរួមបញ្ចូល​ជាមួយ Theme Builder {#integration-with-theme-builder}

នៅពេល​ប្រើ Theme Builder សមត្ថភាព​បង្កើត​ម៉ឺនុយ​នឹង៖

1. រកឃើញ​ទីតាំង​ម៉ឺនុយ theme ដែល​មាន​ដោយ​ស្វ័យប្រវត្តិ
2. បង្កើត​ម៉ឺនុយ​ដែល​មាន​ស្លាក​រុករក​សមស្រប​សម្រាប់​រចនា​របស់​អ្នក
3. កំណត់​ម៉ឺនុយ​ទៅ​ទីតាំង theme ត្រឹមត្រូវ
4. គាំទ្រ​ការ​បន្ថែម​ធាតុ​ម៉ឺនុយ​បន្ទាប់​ពី​បង្កើត​រួច

## សមត្ថភាព​ពាក់ព័ន្ធ {#related-abilities}

- **`add_menu_item`** — បន្ថែម​ធាតុ​ទៅ​ម៉ឺនុយ​ដែល​មាន​ស្រាប់
- **`update_menu`** — ប្ដូរ​ឈ្មោះ​ម៉ឺនុយ ឬ​កំណត់​វា​ឡើងវិញ​ទៅ​ទីតាំង theme
- **`delete_menu`** — លុប​ម៉ឺនុយ​ចេញ​ពី​គេហទំព័រ​របស់​អ្នក

## អនុវត្តន៍​ល្អបំផុត {#best-practices}

- **ប្រើ​ស្លាក​រុករក​ច្បាស់លាស់** — រក្សា​ស្លាក​ឱ្យ​ខ្លី និង​ពិពណ៌នា​បាន​ច្បាស់​សម្រាប់​អ្នកប្រើ
- **ផ្គូផ្គង​ទីតាំង theme** — កំណត់​ម៉ឺនុយ​ទៅ​ទីតាំង theme ត្រឹមត្រូវ ដើម្បី​បង្ហាញ​បាន​ល្អ
- **រៀបចំ​ផែនការ​រចនាសម្ព័ន្ធ​ម៉ឺនុយ** — សម្រេច​លើ​ឋានានុក្រម​ម៉ឺនុយ​របស់​អ្នក មុន​ពេល​បង្កើត​ធាតុ
- **សាកល្បង​ភាព​ឆ្លើយតប** — ផ្ទៀងផ្ទាត់​ថា​ម៉ឺនុយ​បង្ហាញ​បាន​ត្រឹមត្រូវ​លើ​ឧបករណ៍​ចល័ត
- **ធ្វើ​មូលដ្ឋានីយកម្ម​ស្លាក** — ប្រើ​ស្លាក​រុករក​ខុសៗ​គ្នា​សម្រាប់​កំណែ​ភាសា​ផ្សេងៗ

## ដែនកំណត់ {#limitations}

- ស្លាក​រុករក​គឺ​សម្រាប់​បង្ហាញ​តែ​ប៉ុណ្ណោះ; `name` ខាងក្នុង​នៅតែ​ត្រូវ​បាន​ប្រើ​សម្រាប់​ការ​កំណត់​អត្តសញ្ញាណ​របស់ WordPress
- ការ​គាំទ្រ theme ខុសគ្នា​ទៅតាម theme; មិនមែន​គ្រប់ theme ទាំងអស់​គាំទ្រ​គ្រប់​ទីតាំង​ម៉ឺនុយ​ទេ
- ធាតុ​ម៉ឺនុយ​ត្រូវ​បន្ថែម​ដោយឡែក បន្ទាប់​ពី​បង្កើត​ម៉ឺនុយ​រួច
- ការ​ប្ដូរ​ស្លាក​រុករក​ត្រូវការ​ធ្វើ​បច្ចុប្បន្នភាព​ម៉ឺនុយ
