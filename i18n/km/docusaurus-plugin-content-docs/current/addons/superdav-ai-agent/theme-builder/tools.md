# ឧបករណ៍ និងការកំណត់រចនាសម្ព័ន្ធ

Theme Builder ប្រើឧបករណ៍មួយចំនួន ដើម្បីវិភាគ រចនា និងបង្កើតគេហទំព័រ WordPress របស់អ្នក។ ក្នុងកំណែ 1.16.0, **sd-ai-agent/site-scrape ឥឡូវជា​ឧបករណ៍ Tier 1** ដែលធ្វើឱ្យវាមានស្រាប់តាមលំនាំដើមក្នុង Theme Builder។ Superdav AI Agent v1.18.0 ក៏ណែនាំកញ្ចប់ Advanced រួមដំណើរដែលចែកចាយដាច់ដោយឡែក សម្រាប់លំហូរការងាររបស់អ្នកអភិវឌ្ឍន៍ដែលជឿទុកចិត្តបានផងដែរ។ ឧបករណ៍ទាំងនោះមិនមែនជាផ្នែកនៃកញ្ចប់ស្នូល WordPress.org ទេ ហើយត្រូវដំឡើង និងអនុញ្ញាតដាច់ដោយឡែក។

## ឧបករណ៍ដែលមាន {#available-tools}

### ឧបករណ៍ Tier 1 (មានជានិច្ច) {#tier-1-tools-always-available}

ឧបករណ៍ Tier 1 មានស្រាប់តាមលំនាំដើមក្នុង Theme Builder ដោយមិនត្រូវការការកំណត់រចនាសម្ព័ន្ធបន្ថែមទេ។

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**គោលបំណង៖** វិភាគគេហទំព័រដែលមានស្រាប់ ដើម្បីដកស្រង់គំនិតរចនា រចនាសម្ព័ន្ធមាតិកា និងមុខងារ។

**សមត្ថភាព៖**

- **ការវិភាគគេហទំព័រ** — ប្រមូល និងវិភាគគេហទំព័ររបស់គូប្រជែង ឬគេហទំព័រជាគំរូ
- **ការដកស្រង់ការរចនា** — កំណត់ពណ៌ ពុម្ពអក្សរ និងលំនាំប្លង់
- **រចនាសម្ព័ន្ធមាតិកា** — យល់ពីការរៀបចំទំព័រ និងលំដាប់ថ្នាក់
- **ការរកឃើញមុខងារ** — កំណត់ plugin និងមុខងារ
- **ការវិភាគប្រសិទ្ធភាព** — ពិនិត្យល្បឿនទំព័រ និងការធ្វើឱ្យប្រសើរ
- **ការវិភាគ SEO** — ពិនិត្យស្លាក meta និងទិន្នន័យមានរចនាសម្ព័ន្ធ

**ការប្រើប្រាស់៖**

```
វិភាគការរចនារបស់ example.com ហើយប្រើវាជាគំនិតសម្រាប់គេហទំព័ររបស់ខ្ញុំ។
```

**អ្វីដែលវាដកស្រង់៖**

- ក្ដារពណ៌ និងការប្រើពណ៌
- អក្សររចនា (ពុម្ពអក្សរ និងទំហំ)
- រចនាសម្ព័ន្ធប្លង់ និងគម្លាត
- លំនាំការរុករក
- ការរៀបចំមាតិកា
- ការប្រើរូបភាព និងមេឌៀ
- ធាតុអន្តរកម្ម
- ភាពឆ្លើយតបលើទូរសព្ទ

**ដែនកំណត់៖**

- មិនអាចប្រមូលទិន្នន័យពីគេហទំព័រដែលការពារដោយពាក្យសម្ងាត់បានទេ
- គោរព robots.txt និងគោលការណ៍គេហទំព័រ
- អាចមិនចាប់យកមាតិកាដែលផ្លាស់ប្តូរដោយស្វ័យប្រវត្តិបានទេ
- គេហទំព័រដែលពឹងផ្អែកលើ JavaScript ច្រើន អាចមានការវិភាគកម្រិតកំណត់
- គេហទំព័រធំៗអាចចំណាយពេលយូរជាងក្នុងការវិភាគ

### ឧបករណ៍ Tier 2 (ស្រេចចិត្ត) {#tier-2-tools-optional}

ឧបករណ៍ Tier 2 អាចប្រើបាន នៅពេលបើកជាក់លាក់ក្នុងការកំណត់ Theme Builder។

វិភាគ និងស្នើការកែលម្អសម្រាប់៖

- ការធ្វើឱ្យ SEO ប្រសើរ

### ឧបករណ៍រួមដំណើរ Advanced (កញ្ចប់ដាច់ដោយឡែក) {#advanced-companion-tools-separate-package}

កញ្ចប់រួមដំណើរ Advanced ត្រូវបានចែកចាយដាច់ដោយឡែកពីការចេញផ្សាយស្នូល Superdav AI Agent។ វាត្រូវបានបង្កើតសម្រាប់បរិស្ថានអ្នកអភិវឌ្ឍន៍ និងម្ចាស់គេហទំព័រដែលជឿទុកចិត្តបាន ដែលអ្នកគ្រប់គ្រងអនុញ្ញាតជាក់លាក់ឱ្យ agent ប្រើឧបករណ៍ដែលមានឥទ្ធិពលខ្ពស់។

សមត្ថភាពរួមដំណើរ Advanced អាចរួមមាន៖

- **ឧបករណ៍ប្រព័ន្ធឯកសារ** — ពិនិត្យ និងកែប្រែឯកសារដែលបានអនុម័តក្នុងបរិស្ថានដែលជឿទុកចិត្តបាន
- **ឧបករណ៍មូលដ្ឋានទិន្នន័យ** — សួរ ឬធ្វើបច្ចុប្បន្នភាពទិន្នន័យគេហទំព័រ នៅពេលបានអនុញ្ញាតជាក់លាក់
- **ឧបករណ៍ WP-CLI** — ដំណើរការពាក្យបញ្ជាថែទាំ និងពិនិត្យ WordPress
- **ឧបករណ៍បញ្ជូន REST** — ហៅ REST endpoint ដែលបានចុះបញ្ជី តាមលំហូរការងារបញ្ជូនដែលគ្រប់គ្រង
- **ឧបករណ៍បង្កើត plugin** — បង្កើតគ្រោង និងកែសម្រួលកូដ plugin ជាបន្តបន្ទាប់
- **ឧបករណ៍ snapshot Git** — បង្កើត snapshot មុនប្រតិបត្តិការអភិវឌ្ឍន៍ដែលមានហានិភ័យ
- **ឧបករណ៍គ្រប់គ្រងអ្នកប្រើ** — ជួយពិនិត្យ ឬកែតម្រូវកំណត់ត្រាអ្នកប្រើ នៅកន្លែងដែលបានអនុញ្ញាត
- **ឧបករណ៍ benchmark** — ដំណើរការ benchmark ប្រសិទ្ធភាព ឬសមត្ថភាព សម្រាប់ការពិនិត្យអភិវឌ្ឍន៍

កុំកត់ត្រាឧបករណ៍ទាំងនេះថាជាឧបករណ៍ Theme Builder ដែលមានសម្រាប់ការដំឡើងទាំងអស់។ វាមានតែពេលកញ្ចប់រួមដំណើរ Advanced ត្រូវបានដំឡើង បើក និងកំណត់ឱ្យប្រើដោយអ្នកគ្រប់គ្រងដែលជឿទុកចិត្តបានប៉ុណ្ណោះ។

## ការកំណត់រចនាសម្ព័ន្ធឧបករណ៍ {#tool-configuration}

### ការបើកឧបករណ៍ {#enabling-tools}

ដើម្បីបើកឧបករណ៍បន្ថែមក្នុង Theme Builder៖

1. ទៅកាន់ **Gratis AI Agent → ការកំណត់**
2. ទៅកាន់ **Theme Builder → ឧបករណ៍**
3. បើក/បិទឧបករណ៍តាមតម្រូវការ
4. រក្សាទុកការកំណត់

### សិទ្ធិឧបករណ៍ {#tool-permissions}

ឧបករណ៍ខ្លះត្រូវការសិទ្ធិ៖

- **ការប្រមូលទិន្នន័យគេហទំព័រ** — ត្រូវការការចូលប្រើអ៊ីនធឺណិត
- **ការវិភាគ** — ត្រូវការការភ្ជាប់ Google Analytics
- **ការបង្កើតមាតិកា** — ត្រូវការ API keys
- **មុខងារកម្រិតខ្ពស់** — អាចត្រូវការ subscription premium
- **ឧបករណ៍រួមដំណើរ Advanced** — ត្រូវការកញ្ចប់ Advanced ដែលចែកចាយដាច់ដោយឡែក និងការជឿទុកចិត្តជាក់លាក់ពីអ្នកគ្រប់គ្រង

### API Keys និងព័ត៌មានសម្គាល់ {#api-keys-and-credentials}

កំណត់រចនាសម្ព័ន្ធ API keys សម្រាប់ឧបករណ៍ដែលត្រូវការវា៖

1. ទៅកាន់ **Gratis AI Agent → ការកំណត់ → API Keys**
2. បញ្ចូលព័ត៌មានសម្គាល់សម្រាប់ឧបករណ៍នីមួយៗ
3. សាកល្បងការតភ្ជាប់
4. រក្សាទុកដោយសុវត្ថិភាព

## ដែនកំណត់ និងការពិចារណាអំពីឧបករណ៍ {#tool-limitations-and-considerations}

### ការកំណត់អត្រា {#rate-limiting}

- ការទាញយកទិន្នន័យត្រូវបានកំណត់អត្រា ដើម្បីការពារកុំឱ្យម៉ាស៊ីនមេផ្ទុកលើស
- សំណើច្រើនទៅកាន់គេហទំព័រដូចគ្នា អាចត្រូវបានបន្ថយល្បឿន
- គេហទំព័រធំៗអាចចំណាយពេលយូរជាងមុនក្នុងការវិភាគ

### ភាពត្រឹមត្រូវនៃមាតិកា {#content-accuracy}

- មាតិកាដែលផ្លាស់ប្តូរតាមពេល អាចមិនត្រូវបានចាប់យកពេញលេញ
- មាតិកាដែលបង្ហាញដោយ JavaScript អាចមិនពេញលេញ
- ទិន្នន័យពេលវេលាពិត អាចមិនត្រូវបានបង្ហាញ
- មាតិកាខ្លះអាចស្ថិតនៅក្រោយជញ្ជាំងបង់ប្រាក់

### ការពិចារណាផ្នែកច្បាប់ និងសីលធម៌ {#legal-and-ethical-considerations}

- គោរព robots.txt និងគោលការណ៍គេហទំព័រ
- កុំទាញយកមាតិកាដែលមានសិទ្ធិអ្នកនិពន្ធ ដើម្បីចម្លងផ្សាយឡើងវិញ
- ប្រើការវិភាគសម្រាប់ជាការបំផុសគំនិត មិនមែនសម្រាប់ចម្លង
- ផ្ទៀងផ្ទាត់ថាអ្នកមានសិទ្ធិប្រើមាតិកាដែលបានស្រង់ចេញ
- អនុវត្តតាមលក្ខខណ្ឌសេវាកម្មរបស់គេហទំព័រដែលបានវិភាគ

### ផលប៉ះពាល់លើដំណើរការ {#performance-impact}

- ការទាញយកទិន្នន័យពីគេហទំព័រធំៗអាចចំណាយពេល
- ការទាញយកទិន្នន័យច្រើនក្នុងពេលតែមួយ អាចយឺតជាងមុន
- ការតភ្ជាប់បណ្តាញប៉ះពាល់ដល់ល្បឿន
- សំណុំទិន្នន័យធំៗអាចត្រូវការការដំណើរការបន្ថែម

## វិធីអនុវត្តល្អបំផុត {#best-practices}

### ការប្រើការវិភាគគេហទំព័រ {#using-site-analysis}

1. **វិភាគគេហទំព័រច្រើន** — ប្រមូលការបំផុសគំនិតពីប្រភពជាច្រើន
2. **ផ្តោតលើរចនាសម្ព័ន្ធ** — សិក្សាលំនាំប្លង់ និងការរៀបចំ
3. **ស្រង់ពណ៌** — ប្រើបន្ទះពណ៌ជាចំណុចចាប់ផ្តើម
4. **សិក្សាអក្សររចនា** — កំណត់បន្សំពុម្ពអក្សរដែលអ្នកចូលចិត្ត
5. **ពិនិត្យការរុករក** — យល់ពីការរៀបចំម៉ឺនុយ

### ការទាញយកទិន្នន័យប្រកបដោយសីលធម៌ {#ethical-scraping}

1. **គោរព robots.txt** — អនុវត្តតាមគោលការណ៍ណែនាំរបស់គេហទំព័រ
2. **កុំចម្លងមាតិកា** — ប្រើការវិភាគសម្រាប់ការបំផុសគំនិតប៉ុណ្ណោះ
3. **ផ្ទៀងផ្ទាត់សិទ្ធិ** — ធានាថាអ្នកអាចប្រើមាតិកាដែលបានស្រង់ចេញ
4. **បញ្ជាក់ប្រភព** — ផ្តល់កិត្តិយសដល់ប្រភពបំផុសគំនិត
5. **អនុវត្តតាមលក្ខខណ្ឌ** — គោរពលក្ខខណ្ឌសេវាកម្មរបស់គេហទំព័រ

### ការបង្កើនប្រសិទ្ធភាពឧបករណ៍ {#maximizing-tool-effectiveness}

1. **បញ្ជាក់ឱ្យច្បាស់** — ស្នើប្រភេទការវិភាគជាក់លាក់
2. **ផ្តល់បរិបទ** — ពន្យល់គោលបំណងគេហទំព័ររបស់អ្នក
3. **កំណត់ការរំពឹងទុក** — ពិពណ៌នាគោលដៅរចនារបស់អ្នក
4. **កែលម្អជាបន្តបន្ទាប់** — កែសម្រួលផ្អែកលើលទ្ធផល
5. **បន្សំឧបករណ៍** — ប្រើឧបករណ៍ច្រើនសម្រាប់ការវិភាគគ្រប់ជ្រុងជ្រោយ

## ការដោះស្រាយបញ្ហា {#troubleshooting}

### គេហទំព័រមិនអាចទាញយកទិន្នន័យបាន {#site-wont-scrape}

- ពិនិត្យថាគេហទំព័រអាចចូលប្រើជាសាធារណៈបានឬអត់
- ផ្ទៀងផ្ទាត់ថា robots.txt អនុញ្ញាតឱ្យទាញយកទិន្នន័យ
- សាកល្បងគេហទំព័រផ្សេង
- ពិនិត្យការតភ្ជាប់អ៊ីនធឺណិត
- ទាក់ទងផ្នែកជំនួយ ប្រសិនបើបញ្ហានៅតែបន្ត

### ការវិភាគមិនពេញលេញ {#incomplete-analysis}

- គេហទំព័រអាចមានមាតិកាដែលផ្លាស់ប្តូរតាមពេល
- JavaScript អាចមិនត្រូវបានបង្ហាញពេញលេញ
- គេហទំព័រធំៗអាចអស់ពេលកំណត់
- សាកល្បងវិភាគទំព័រជាក់លាក់ជំនួសវិញ
- ស្នើប្រភេទការវិភាគជាក់លាក់

### ដំណើរការយឺត {#slow-performance}

- គេហទំព័រធំៗចំណាយពេលយូរជាងមុនក្នុងការវិភាគ
- ការទាញយកទិន្នន័យច្រើនក្នុងពេលតែមួយយឺតជាងមុន
- ការតភ្ជាប់បណ្តាញប៉ះពាល់ដល់ល្បឿន
- សាកល្បងនៅម៉ោងដែលមិនសូវមមាញឹក
- វិភាគផ្នែកតូចៗជាមុន
