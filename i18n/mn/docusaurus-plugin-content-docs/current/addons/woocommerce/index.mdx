# WooCommerce интеграц

WooCommerce интеграц нь манай Manual Gateway ажилладаг аргатай маш төстэй ажилладаг. Энэ нь таны хэрэглэгчдэд WooCommerce суулгац дээр байгаа төлбөрийн аль ч маягтыг сонгох боломж олгож, маш их уян хатан байдлыг санал болгодог. Мөн энгийн: хэрэглэгчийн төлбөр тооцооны хугацаа дуусмагц хэрэглэгчийн захиалгыг түр зогсооно. Үүний зэрэгцээ сүлжээний үндсэн сайтын WooCommerce суулгац дээр захиалга үүснэ. Дараа нь хэрэглэгч төлбөр хийх холбоостой нэхэмжлэхийг имэйлээр хүлээн авна. Төлбөр хүлээн авмагц захиалга шинэчлэгдэнэ.

## Онцлогууд {#features}

- Ultimate Multisite болон WooCommerce хоорондын хялбар интеграц
- Хэд хэдэн төлбөрийн арга боломжтой
- Багцад орсон захиалгын үндсэн хэсгийг ашигласан автомат давтамжит төлбөр тооцоо, нэмэлт WooCommerce Subscriptions plugin сайжруулалттай
- Checkout-ийн нийт дүн нь Ultimate Multisite татваруудыг тусдаа, татвар ногдохгүй WooCommerce шимтгэлийн мөрүүд болгон оруулдаг тул үйлчлүүлэгчид төлбөр хийхээс өмнө татварын дүнг шалгах боломжтой

## Баримтжуулалт {#documentation}

**WooCommerce интеграцыг тохируулах**

**Ultimate Multisite-аар төлбөрийн сонголтоо өргөжүүлэх: WooCommerce интеграц**

## Суулгалт {#installation}

1. 'ultimate-multisite-woocommerce'-ийг '/wp-content/plugins/' санд байршуулна уу
2. WordPress дахь 'Plugins' цэсээр plugin-ийг идэвхжүүлнэ үү
3. plugin-ийг тохируулахын тулд алхам алхмаар Wizard-ийг дагана уу

## Ашиглах заавар {#usage-guide}

_**ЧУХАЛ ТЭМДЭГЛЭЛ: Энэ нийтлэл нь Ultimate Multisite 2.x хувилбарт хамаарна.**_

_**АНХААР:** Ultimate Multisite: WooCommerce Integration нь WooCommerce-ийг ядаж таны үндсэн site дээр идэвхжүүлсэн байхыг шаарддаг._

_Stripe_ болон _PayPal_ зарим улс оронд боломжгүй байдаг нь Ultimate Multisite хэрэглэгчдийг манай plugin-ийг үр дүнтэй ашиглахад хязгаарлаж эсвэл саад болдог гэдгийг бид ойлгож байна. Тиймээс бид **_WooCommerce_-тэй интеграц хийх нэмэлт** бүтээсэн.

_WooCommerce_ бол маш алдартай eCommerce plugin юм. Дэлхий даяарх хөгжүүлэгчид өөр өөр төлбөрийн гарцуудыг түүнтэй интеграц хийх нэмэлтүүдийг бүтээсэн. Бид үүнийг ашиглан Ultimate Multisite төлбөр тооцооны системд ашиглаж болох төлбөрийн гарцуудыг өргөжүүлсэн.

## Үүнийг тохируулах {#setting-it-up}

Эхлээд та **лицензтэй хэрэглэгчдэд үнэгүй** WooCommerce Integration нэмэлтийг суулгах хэрэгтэй. Үүнийг суулгахын тулд super admin dashboard дээрээ Ultimate Multisite дээр дарж **Add-ons**-ийг сонгоно уу.

![Add-ons хуудас](/img/admin/addons-page.png)

Эндээс та бүх Ultimate Multisite нэмэлтүүдийг олох боломжтой. **Ultimate Multisite: WooCommerce Integration** нэмэлт дээр дарна уу.

![WooCommerce Integration-ийг харуулсан Add-ons хуудас](/img/admin/addons-page.png)

Нэмэлтийн дэлгэрэнгүй мэдээлэлтэй цонх гарч ирнэ. Зүгээр л **Install Now** дээр дарна уу *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

Суулгалт дууссаны дараа та plugins хуудас руу шилжүүлэгдэнэ. Энд зүгээр л **Network Activate** дээр дарснаар _WooCommerce_ нэмэлт таны сүлжээнд идэвхжинэ.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

Үүнийг идэвхжүүлсний дараа, хэрэв таны website дээр WooCommerce plugin суулгаж идэвхжүүлээгүй хэвээр байвал **та сануулга хүлээн авна**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

Суулгасны дараа Ultimate Multisite төлбөрийн тохиргооны доор нэмэлт сонголт харагдана. Тэндээс та _WooCommerce_ сонголтыг **идэвхжүүлж** болно.

![WooCommerce идэвхжүүлсэн Ultimate Multisite төлбөрийн гарцын тохиргоо](/img/addons/woocommerce-payment-gateway.png)

Та урд талд харагдах дэлгэцийн нэр, хэрэглэгчдээ checkout урсгалаар чиглүүлэх заавар зэрэг зарим үндсэн тохиргоог тохируулж болно. Дараа нь ашиглахыг хүссэн төлбөрийн гарцаа идэвхжүүлж тохируулахын тулд **Үндсэн site-ийн _WooCommerce_ тохиргоо** руу орж болно.

### Checkout татварын шимтгэлийн мөрүүд {#checkout-tax-fee-lines}

Ultimate Multisite checkout татвар агуулсан үед интеграц нь Ultimate Multisite татварын дүн бүрийг WooCommerce checkout-д тусдаа, татвар ногдохгүй шимтгэлийн мөр болгон нэмдэг. Энэ нь төлбөр хийхээс өмнө татварын нийт дүнг харагдахуйц байлгахын зэрэгцээ WooCommerce тухайн татварын шимтгэлээс дахин татвар авахыг сэргийлнэ.

## Төлбөрийг хэрхэн удирдах вэ {#how-to-manage-the-payment}

WooCommerce Integration нь манай **Manual Gateway** ажилладаг аргатай маш төстэй ажилладаг. Сул тал нь төлбөрийн хугацааны төгсгөлд төлбөр бүрийг гараар төлөх ёстой. Урсгал хэрхэн ажилладгийг энд харуулав:

  * Хэрэглэгчийн төлбөрийн мөчлөг дуусмагц гишүүнчлэл **түр зогсоно**. Үүний зэрэгцээ таны сүлжээний үндсэн site дээрх _WooCommerce_ суулгац дээр захиалга үүснэ.

  * Дараа нь хэрэглэгчид төлбөр хийх холбоостой нэхэмжлэхийг имэйлээр хүлээн авна (холбоос нь мөн тэдний subsite dashboard дахь төлбөрийн түүх дээр харагдана). Тэр холбоос хэрэглэгчийг _**WooCommerce**_ **checkout** руу чиглүүлэх бөгөөд хэрэглэгчид төлбөрөө боловсруулахын тулд **боломжтой _WooCommerce_ гарцуудын** нэгийг сонгох боломжтой болно.

  * Төлбөр хүлээн авмагц гишүүнчлэл **шинэчлэгдэж** , төлбөр Ultimate Multisite дээр бүртгэгдэж, захиалгыг дууссан төлөвт оруулна.

### Зарим чухал тэмдэглэл: {#some-important-notes}

  * Ultimate Multisite-ийн анхдагч ажиллагаа энэ интеграцын хэлбэртэй мөн ажилласаар байна. Жишээлбэл, **Subscription Management** дэлгэцээс **буцаан олголт гаргах** үед сонгосон төлбөрийн маягтыг ашиглан буцаан олголт хийхийн тулд WooCommerce-тэй холбогдоно.

  * Ultimate Multisite: WooCommerce Integration нь ажиллахын тулд WP-Cron-д тулгуурладаг. Ихэнх үед энэ нь маш сайн ажилладаг боловч site бага хандалттай бол cron нь төлөвлөсөн үйл явдлуудыг шалгахаар өдөөгдөхгүй, үйл явдал алдагддаг бөгөөд энэ нь гүйцэтгэлийн саатал үүсгэж болно.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

Сар бүр төлбөрийг гараар эхлүүлэх нь эцсийн хэрэглэгчдэд тохиромжтой биш, учир нь захиалгууд төлбөр тооцооны мөчлөг бүрд автоматаар давтагдах ёстой гэж хүлээгддэг. WooCommerce Integration add-on нь одоо үндсэн захиалгын кодыг багтаадаг тул давтагдах бүтээгдэхүүнүүд давтагдах төлбөр тооцоонд үргэлж захиалгыг ашиглана.

Та энэ интеграцийн тусламжтайгаар давтагдах бүтээгдэхүүн зарахын тулд тусдаа [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) худалдан авах эсвэл суулгах **шаардлагагүй**. Хэрэв тэр plugin худалдан авагдаж, суулгагдаж, таны үндсэн сайт дээр боломжтой байвал Ultimate Multisite үүнийг ашиглах бөгөөд WooCommerce нэмэлт захиалгын удирдлагын боломжуудыг харуулна. Хэрэв боломжгүй бол багцлагдсан захиалгын үндсэн хэсэг давтагдах бүтээгдэхүүнүүдийг мөн адил зохицуулна.

![WooCommerce захиалгын интеграцийн тохиргоо](/img/addons/woocommerce-subscription-integration.png)

Ингээд л боллоо. Таны эцсийн хэрэглэгчид давтагдах бүтээгдэхүүний төлбөр тооцооны мөчлөгийн төгсгөлд төлбөрийг гараар хийх шаардлагагүй болно.

Таны үндсэн сайт дээр _WooCommerce_ доор захиалга үүсэхээс гадна Account бүрд зориулж захиалга мөн үүсгэнэ.

### Амжилтгүй сунгалтын түдгэлзүүлэх үйлдэл {#failed-renewal-suspension-behavior}

Анхдагчаар, WooCommerce Subscriptions амжилтгүй төлбөрийн дахин оролдлогын хуваариа боловсруулах үед membership-үүд идэвхтэй хэвээр үлдэнэ. Энэ нь ердийн WooCommerce Subscriptions-ийн хүлээлгийн цонхыг хадгалдаг тул сэргээгдсэн сунгалт нь subscription болон membership-ийг хандалтыг хугацаанаас нь өмнө тасалдуулахгүйгээр дахин синк хийх боломжтой.

Илүү хатуу хандалтын хяналт хүсэж буй administrators нь WooCommerce Integration тохиргоонд **Амжилтгүй сунгалтын үед Membership-үүдийг нэн даруй түдгэлзүүлэх**-ийг идэвхжүүлж болно. Энэ сонгон идэвхжүүлэх тохиргоо идэвхтэй үед Ultimate Multisite нь WooCommerce Subscriptions-ийн дахин оролдлогын боловсруулалт дуусахыг хүлээлгүйгээр сунгалтын төлбөр амжилтгүй болсон даруйд холбогдох membership-ийг түдгэлзүүлнэ.

Developers нь ижил шийдвэрийг `wu_woo_suspend_on_payment_failure` filter-ээр өөрчилж болно. Hook лавлах хуудсууд add-on эх сурвалжаас үүсгэгддэг тул энэ тойм нь зөвхөн тусгай түдгэлзүүлэх бодлогын filter нэрийг онцолж байна.
