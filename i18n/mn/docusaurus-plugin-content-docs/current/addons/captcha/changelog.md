# Captcha өөрчлөлтийн бүртгэл

Хувилбар: 1.5.0 - 2026-05-22-нд гарсан
* Шинэ: Хатуу зогсоолтой давтамж хязгаарлагч — captcha-аар хамгаалагдсан талбарууд дээрх GET болон POST бүрийг тоолно (wp-login / register / lost-password / comments, WooCommerce my-account / checkout / pay-for-order, Ultimate Multisite checkout / inline-login) бөгөөд HTTP 429, `Retry-After` header, мөн санамсаргүй tarpit хүлээлтээр (1–5 секунд, хатуу дээд хязгаар 15 секунд) хариулна.
* Шинэ: Tarpit цонхыг тохируулах `cap_rate_limit_tarpit_min` / `cap_rate_limit_tarpit_max` тохиргоонууд.
* Шинэ: Итгэмжлэгдсэн IP мужуудыг чөлөөлөх `wu_cap_rate_limit_whitelist_ip` filter.
* Шинэ: Хатуу зогсоох хариу илгээгдэхийн яг өмнө ажиллах `wu_cap_rate_limit_will_block` action.
* Шинэ: Хуурамчаар дуурайлгахад тэсвэртэй client-IP илрүүлэлт. `Captcha_Core::get_client_ip()` (давтамж хязгаарлалтын bucket түлхүүрүүд, captcha siteverify `remoteip`, статистикийн IP hash-уудын үнэн эх сурвалж) одоо хатуу итгэлцлийн загварыг мөрдүүлнэ: REMOTE_ADDR нь суурь, `CF-Connecting-IP` нь зөвхөн шууд холбогч тал одоогийн Cloudflare IP муж дотор байх үед хүндэтгэгдэнэ, `X-Forwarded-For` нь зөвхөн шууд холбогч тал админ тохируулсан итгэмжлэгдсэн proxy жагсаалтад байх үед хүндэтгэгдэнэ; зочны IP дээр тогтохоос өмнө итгэмжлэгдсэн/CF дамжуулалтуудыг алгасах баруунаас зүүн тийш шалгалттай.
* Шинэ: `cap_trust_cloudflare_headers` тохиргоо (анхдагчаар OFF) — Cloudflare-ийн ард байх үед `CF-Connecting-IP` итгэлцлийг сонгон идэвхжүүлнэ. plugin нь багцалсан Cloudflare CIDR snapshot-тай ирдэг бөгөөд шинэчлэлт амжилтгүй болбол багцалсан нөөц хувилбараар wp-cron-оор долоо хоног бүр шинэчилнэ.
* Шинэ: `cap_trusted_proxies` тохиргоо — таны өөрийн урд шугамын proxy / load-balancer-уудыг жагсаах CIDR эсвэл дан IP-уудын textarea (мөр бүрт нэг, `#` тайлбар зөвшөөрнө). Үүнгүйгээр давтамж хязгаарлагч идэвхтэй байсан ч `X-Forwarded-For` үл тоогдоно.
* Шинэ: Анх идэвхжүүлэхэд магадлалтай Cloudflare / proxy төлөвийг автоматаар илрүүлж, нэг товшилтын "Илрүүлсэн тохиргоог хэрэглэх" админ мэдэгдэл харуулна. plugin таны хадгалсан утгуудыг хэзээ ч дарж бичихгүй; хэрэв дараагийн урсгал таны тохиргоо бодит байдалтай нийцэхээ больсныг илтгэвэл (жишээ нь Cloudflare CIDR мужуудаа өөрчилж, таны proxy CIDR одоо хуучирсан бол) зөвлөсөн шинэчлэлтийг харуулсан хаах боломжгүй зөрүүгийн мэдэгдэл гарна.
* Зассан: Үл үзэгдэх горим `cap_security_level`-ийг FAST руу чимээгүй бууруулахаа больсон — админы тохируулсан түвшин хүндэтгэгдэнэ. Өөрийн тусгай логик хүссэн сайтуудад шинэ `wu_cap_server_security_level` filter боломжтой.
* Зассан: Статистикийн `rate_limits_triggered` тоолуур одоо зөвхөн ховор post-success backstop зам дээр бус, блок бүр дээр нэмэгдэнэ.
* Зассан: `Captcha_Core::get_client_ip()` нь одоо давтамж хязгаарлагч, captcha provider-ууд (reCAPTCHA + hCaptcha `siteverify`), статистик даяарх зочны IP оноолтын цорын ганц үнэн эх сурвалж болсон — ингэснээр хуурамч `CF-Connecting-IP` header агуулсан origin-server рүү шууд ирсэн хүсэлтүүд бодит шууд холбогчийн оронд хуурамч IP-ээр bucket-лагдах байсан хуурамчлах векторыг хаалаа.
* Зассан: WooCommerce classic checkout давтамж-хязгаарлалтын хаалга одоо `woocommerce_before_checkout_form`-ын оронд `template_redirect` (priority 1) дээр ажиллана. Сагс хоосон үед form түвшний hook огт ажилладаггүй тул product огт нэмдэггүй flood урсгал хязгаарлагчийг бүхэлд нь тойрч гарч байсан.
* Зассан: WooCommerce pay-for-order давтамж-хязгаарлалтын хаалга одоо `woocommerce_before_pay_action`-ын оронд `template_redirect` дээр ажиллана. Сүүлийнх нь зөвхөн `wp_verify_nonce('woocommerce-pay')` амжилттай болсны дараа ажилладаг бөгөөд энэ нь unauth халдагчид (жинхэнэ threat model) хязгаарлагчийг огт өдөөдөггүй гэсэн үг.
* Зассан: WooCommerce Store API (blocks) checkout давтамж-хязгаарлалтын хаалга одоо `woocommerce_store_api_checkout_update_order_from_request`-ын оронд `rest_pre_dispatch` дээр ажиллана. Сүүлийнх нь Store API сагс болон billing талбаруудыг баталгаажуулсны дараа л ажилладаг тул unauth bot-ууд validator-аас 400 авч, хязгаарлагчийг огт өдөөдөггүй байсан.
* Зассан: Ultimate Multisite inline-login давтамж-хязгаарлалтын хаалга одоо `wu_before_inline_login`-ын оронд `wu_ajax_nopriv_wu_inline_login` priority 1 дээр (мөн logged-in толь хувилбар дээр) ажиллана. Сүүлийнх нь зөвхөн `check_ajax_referer('wu_checkout')` амжилттай болсны дараа ажилладаг тул хүчинтэй wu_checkout nonce-гүй unauth bot-ууд 403 авч, хязгаарлагчийг огт өдөөдөггүй байсан.
* Зассан: `Rate_Limiter::enforce()` одоо `surface|ip`-ээр түлхүүрлэсэн хүсэлт бүрт нэг удаагийн хамгаалалт хэрэглэдэг тул render бүрт хоёр удаа ажилладаг upstream hook-ууд (ялангуяа Ultimate Multisite дахь `wu_setup_checkout`) үр дүнтэй давтамж-хязгаарлалтын босгыг хоёр дахин багасгахаа больсон.
* Зассан: Давтамж-хязгаарлалтын талбарын хаалгууд `Captcha_Core::is_whitelisted()` (`wu_captcha_whitelisted` filter)-ийг шалгахаа больсон. Тэр filter нь "captcha-г өөр талбар аль хэдийн зохицуулсан" гэдгийг илтгэдэг бөгөөд flood хамгаалалтаас тусдаа — WooCommerce integration нь Woo nonce байгаа үед WordPress login captcha-г алгасуулахын тулд үүнийг hook хийж байсан нь давтамж тоололт руу нэвчин, Woo POST-уудыг хязгаарлагчаас зайлсхийх боломжтой болгож байсан. Одоо үйлчлэх цорын ганц тойрох арга нь давтамж-хязгаарлалтад зориулсан `wu_cap_rate_limit_whitelist_ip` filter юм.
