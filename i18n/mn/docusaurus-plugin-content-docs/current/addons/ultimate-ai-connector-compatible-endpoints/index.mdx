# Compatible Endpoints-д зориулсан Ultimate AI Connector

Compatible Endpoints-д зориулсан Ultimate AI Connector нь OpenAI-тэй нийцтэй AI нийлүүлэгчийн API-уудыг таны WordPress multisite сүлжээтэй нэгтгэдэг. Олон AI нийлүүлэгчийг — тус бүр өөрийн төгсгөлийн цэгийн URL, API түлхүүр, өгөгдмөл загвартайгаар — тохируулж, нийлүүлэгч боломжгүй үед автомат нөөц чиглүүлэлтээр хүсэлтүүдийг дамжуулна.

## Гол боломжууд

- **Олон нийлүүлэгчийн дэмжлэг**: Олон AI нийлүүлэгчийн картыг нэмж, тус бүрд нь өөрийн төгсгөлийн цэгийн URL, API түлхүүр, өгөгдмөл загварыг тохируулна
- **Чирж-дахин эрэмбэлэх**: Карт бүрийн барих бариулыг ашиглан нийлүүлэгчдийг давуу эрхийн дарааллаар байрлуулна; хүсэлтүүд нийлүүлэгчдийг дарааллаар нь ашиглана
- **Автомат нөөц чиглүүлэлт**: Үндсэн нийлүүлэгч боломжгүй бол хүсэлтүүд дараагийн тохируулсан нийлүүлэгч рүү автоматаар шилжинэ
- **Нийлүүлэгч тус бүрийн тохиргоо**: Нийлүүлэгч бүр өөрийн нэвтрэх мэдээлэл болон загварын сонголтоор тусдаа тохируулагдана
- **Карт дээр суурилсан тохиргооны UI**: Тогтвортой WordPress 6.9+ Card бүрэлдэхүүнүүд (CardBody, CardHeader, CardDivider) дээр бүтээгдсэн
- **Нэмэхэд автоматаар дэлгэх**: Шинэ нийлүүлэгчийн картууд автоматаар дэлгэгдэж, та дэлгэрэнгүй мэдээллийг шууд бөглөх боломжтой
- **Скриптийн кэш хүчингүй болгох**: Нэмэлтийг шинэчлэх үед нийлүүлэгчийн скриптүүдийн кэшийг хүчингүй болгож, хамгийн сүүлийн хувилбарыг ачаална

## Шаардлагууд

- WordPress 5.3 эсвэл түүнээс дээш
- PHP 7.4 эсвэл түүнээс дээш
- Ultimate Multisite plugin (идэвхтэй)
- OpenAI-тэй нийцтэй API төгсгөлийн цэг (өөрөө байршуулсан эсвэл гуравдагч талын)

## Суулгалт

1. Нэмэлтийн файлуудыг өөрийн `/wp-content/plugins/` санд байршуулна
2. WordPress дахь 'Plugins' цэсээр plugin-ийг идэвхжүүлнэ
3. Нэмэлтийг тохируулахын тулд **Ultimate Multisite → AI Connector** руу очно

## Тохиргоо

### Олон нийлүүлэгчийн тохируулга

Тохиргооны дэлгэц нь карт дээр суурилсан зохион байгуулалт ашигладаг бөгөөд карт бүр нэг AI нийлүүлэгчийг илэрхийлнэ. Та хэрэгтэй хэмжээгээрээ олон нийлүүлэгч нэмж болно.

#### Нийлүүлэгч нэмэх

1. Сүлжээний удирдлагад **Ultimate Multisite → AI Connector** руу очно
2. **Нийлүүлэгч нэмэх** дээр дарна
3. Шинэ нийлүүлэгчийн карт гарч ирээд автоматаар дэлгэгдэнэ
4. Нийлүүлэгчийн дэлгэрэнгүй мэдээллийг бөглөнө:
   - **Нийлүүлэгчийн нэр** — энэ нийлүүлэгчийг таних шошго (жишээ нь "OpenAI", "Mistral", "Local Ollama")
   - **Төгсгөлийн цэгийн URL** — OpenAI-тэй нийцтэй API-ийн үндсэн URL (жишээ нь `https://api.openai.com/v1`)
   - **API түлхүүр** — энэ нийлүүлэгчид зориулсан таны API түлхүүр
   - **Өгөгдмөл загвар** — энэ нийлүүлэгчид өгөгдмөлөөр ашиглах загварын танигч (жишээ нь `gpt-4o`, `mistral-small`)
5. **Тохиргоог хадгалах** дээр дарна

#### Нийлүүлэгч устгах

Аливаа нийлүүлэгчийн карт дээрх **Устгах** товчийг дарж устгана. Өөрчлөлтийг хэрэгжүүлэхийн тулд тохиргоог хадгална.

#### Нийлүүлэгчдийг дахин эрэмбэлэх

Аливаа нийлүүлэгчийн картын зүүн талд байгаа барих дүрсийг (⣿) чирж дахин эрэмбэлнэ. Дараалал нь нөөц шилжилтийн давуу эрхийг тодорхойлно: хүсэлтүүд эхний нийлүүлэгч рүү илгээгдэж, хүсэлт амжилтгүй бол дараагийн нийлүүлэгчид рүү шилжинэ.

### Нөөц чиглүүлэлт

Үндсэн нийлүүлэгч рүү илгээсэн хүсэлт — хугацаа хэтрэх, хурдны хязгаарлалт, эсвэл үйлчилгээний тасалдлаас болж — амжилтгүй болсон үед connector нь жагсаалтын дараагийн нийлүүлэгчээр автоматаар дахин оролддог. Энэ нь нэг нийлүүлэгч амжилттай болох эсвэл бүх нийлүүлэгчийг туршиж дуустал жагсаалтаар үргэлжилнэ.

| Нөхцөл байдал | Үйлдэл |
|----------|-----------|
| Үндсэн нийлүүлэгч хариу өгнө | Хүсэлт хэвийн үйлчлэгдэнэ |
| Үндсэн нийлүүлэгч амжилтгүй болно | Хүсэлтийг дараагийн нийлүүлэгчээр дахин оролдоно |
| Бүх нийлүүлэгч амжилтгүй болно | Дуудагчид алдаа буцаана |

Нөөц шилжилт нь дуудаж буй plugin эсвэл theme-д ил тод байдаг — хэдэн нийлүүлэгчийг туршсанаас үл хамааран тэд амжилттай хариу эсвэл эцсийн алдаа хүлээн авна.

### Нийлүүлэгчийн тохиргооны лавлах

| Талбар | Шаардлагатай | Тайлбар |
|-------|----------|-------------|
| Нийлүүлэгчийн нэр | Тийм | Энэ нийлүүлэгчийн харуулах шошго |
| Төгсгөлийн цэгийн URL | Тийм | OpenAI-тэй нийцтэй API-ийн үндсэн URL |
| API түлхүүр | Тийм | Энэ нийлүүлэгчийн баталгаажуулалтын түлхүүр |
| Өгөгдмөл загвар | Тийм | Хүсэлтэд загвар заагаагүй үед ашиглах загварын танигч |

## Алдаа засварлах

### Нийлүүлэгч харагдахгүй байна

Хадгалсны дараа нийлүүлэгчийн жагсаалт хоосон байвал дараахыг баталгаажуулна уу:

- Нийлүүлэгч нэмсний дараа та **Тохиргоог хадгалах** дээр дарсан
- Таны хөтөч тохиргооны хуудсыг хадгалахыг хориглоогүй (JavaScript алдааг хөтчийн console-оос шалгана уу)
- Plugin идэвхтэй бөгөөд та сүлжээний удирдлагын дэлгэц дээр байгаа

### Бүх нийлүүлэгчийн хүсэлт амжилтгүй болж байна

- Нийлүүлэгч бүрийн төгсгөлийн цэгийн URL таны серверээс хүрч болох эсэхийг шалгана (зөвхөн таны хөтчөөс биш)
- API түлхүүр бүр хүчинтэй бөгөөд хугацаа нь дуусаагүй эсэхийг шалгана
- Нийлүүлэгчийн хурдны хязгаарлалт болон квотын төлөвийг тэдний dashboard дээр хянана
- Connector-оос ирэх дэлгэрэнгүй алдааны мессежүүдийг бүртгэхийн тулд WordPress debug logging (`WP_DEBUG_LOG`)-ийг идэвхжүүлнэ

### Нийлүүлэгч нөөц шилжилтэд ашиглагдахгүй байна

Чирж-дахин эрэмбэлэх нь нөөц шилжилтийн дарааллыг өөрчилдөг. Нөөц хүсэлт хүлээн авна гэж хүлээж буй нийлүүлэгч тань жагсаалтад зөв байрласан бөгөөд түүний төгсгөлийн цэгийн URL болон API түлхүүр зөв эсэхийг баталгаажуулна уу.

### Шинэ нийлүүлэгчийн карт автоматаар дэлгэгдэхгүй байна

Автоматаар дэлгэхийн тулд таны хөтөч дээр JavaScript идэвхтэй байх шаардлагатай. Хэрэв карт дарсны дараа дэлгэгдэхгүй бол хөтчийн console-оос алдаа шалгана уу. Автоматаар дэлгэх ажиллахгүй байсан ч картын толгой дээр дарснаар картын агуулгад үргэлж хандах боломжтой.
