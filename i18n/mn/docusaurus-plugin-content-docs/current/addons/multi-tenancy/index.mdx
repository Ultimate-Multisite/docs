# Олон түрээслэгчийн орчин {#multi-tenancy}

Гүйцэтгэл ба аюулгүй байдлын төлөө дэд сайтуудыг серверүүд болон өгөгдлийн сангуудад хуваарилах

## 1.2.0 хувилбарын онцлох шинэчлэлтүүд {#version-120-release-highlights}

Multi-Tenancy 1.2.0 нь үйлдвэрлэлийн тусгаарлалтын ажлын урсгалд зориулсан бүрэн эрхт түрээслэгчийн суурийг нэмсэн:

- Богино настай, давтан ашиглалтаас хамгаалсан SSO зочлолтой төлөвгүй түрээслэгчийн автомат нэвтрэлт.
- Бүрэн эрхт түрээслэгчдэд зориулсан дэд сайт бүрийн өгөгдлийн сан ба файлын системийн тусгаарлалт.
- Хостыг мэдэрдэг түрээслэгч үүсгэлтэд зориулсан Hostinger олон түрээслэгчийн боломж.
- Хуучин өгөгдөл шалгах болон асинхрон түлхэх боловсруулалтад зориулсан бүрэн эрхт шилжилтийн баталгаажуулалтын хаалтууд.
- Bedrock, FrankenPHP болон localhost суулгалтуудад зориулсан нэг машин дээрх өгөгдлийн сангийн хост холболтыг тохируулах боломж.
- Түрээслэгчийн эхлүүлэлт, өгөгдлийн сан бичигч, схемийн эхлүүлэлт болон хэрэглэгч үүсгэлтийн сайжруулалтууд.
- Түрээслэгчийн итгэмжлэлүүдийг цэвэрлэдэг илүү аюулгүй бүрэн эрхт устгал.

[Бүрэн эрхт шилжилтийн ажлын урсгал](/user-guide/getting-started/sovereign-migration-workflow), [олон түрээслэгчийн тусгаарлалт](/user-guide/configuration/multi-tenancy-isolation), болон [бүрэн эрхт түрээслэгчийн ажиллагаа](/user-guide/administration/sovereign-tenant-operations)-ны хэрэглэгчийн гарын авлагуудыг үзнэ үү.

## Шаардлагууд {#requirements}

- WordPress 5.3 эсвэл түүнээс дээш
- PHP 7.4 эсвэл түүнээс дээш
- Multisite Ultimate залгаас (идэвхтэй)

## Суулгалт {#installation}

1. Нэмэлтийн файлуудыг `/wp-content/plugins/` санд байршуулна
2. WordPress доторх 'Plugins' цэсээр залгаасыг идэвхжүүлнэ
3. Сүлжээний админ хэсэгт нэмэлтийн тохиргоог тохируулна

## Хөгжүүлэлт {#development}

### Тохируулах {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Төслийн бүтэц {#project-structure}

```
ultimate-multisite-addon-template/
├── inc/                          # PHP классууд болон оруулгууд
│   ├── class-multisite-ultimate-updater.php  # Шинэчлэлтийн боловсруулагч
│   └── class-multi_tenancy-main.php             # Үндсэн ажиллагаа
├── tests/                        # Нэгж тестүүд
│   ├── bootstrap.php            # Тестийн эхлүүлэлт
│   ├── class-*-base.php         # Суурь тестийн класс
│   └── class-*-test.php         # Тестийн тохиолдлууд
├── views/                       # Загвар файлууд
├── assets/                      # Статик нөөцүүд
│   ├── css/                     # Загварын хүснэгтүүд
│   ├── js/                      # JavaScript файлууд
│   └── images/                  # Зургууд
├── lang/                        # Хэлний файлууд
├── composer.json                # PHP хамаарлууд
├── package.json                 # Node.js хамаарлууд
├── phpunit.xml.dist             # PHPUnit тохиргоо
├── .phpcs.xml.dist              # Кодын стандартын тохиргоо
└── rector.php                   # Кодыг орчин үеийн болгох дүрмүүд
```

### Тестлэх {#testing}

Нэмэлт нь иж бүрэн тестийн хүрээг агуулна:

- **Нэгж тестүүд**: WordPress тестийн багцтай PHPUnit-д суурилсан тестүүд
- **Суурь тестийн класс**: Нийтлэг тестийн нөхцөлүүдэд зориулсан туслах аргуудыг өгнө
- **Тестийн хамрах хүрээ**: Кодын хамрах хүрээг хянахаар тохируулагдсан
- **Олон сайтын тестлэлт**: Тестүүд олон сайтын орчинд ажиллана

### Кодын стандартууд {#code-standards}

- **WordPress кодчиллын стандартууд**: PHPCS-ээр хэрэгжүүлнэ
- **PHP 7.4+ нийцтэй байдал**: Орчин үеийн PHP боломжуудыг дэмжинэ
- **Статик шинжилгээ**: Төрөл шалгалтад зориулсан PHPStan интеграц
- **Кодыг орчин үеийн болгох**: PHP шинэчлэлтэд зориулсан Rector дүрмүүд

### Бүтээх систем {#build-system}

- **Нөөц боловсруулалт**: CSS/JS файлуудын багасгалт
- **Орчуулга**: POT файл үүсгэлт
- **Багцлалт**: Түгээлтэд зориулсан архив үүсгэлт
- **Хөгжүүлэлтийн горим**: Алдаа засахад зориулсан багасгаагүй нөөцүүд

## Тохиргоо {#configuration}

Нэмэлт нь сүлжээний админ интерфейсээр хандах боломжтой олон төрлийн тохиргооны сонголтуудыг дэмждэг.

### Бүрэн эрхт түрээслэгчийн тохиргоо {#sovereign-tenant-configuration}

Бүрэн эрхт түрээслэгчдийн хувьд түрээслэгчийн өгөгдлийн сан, файлын системийн үндэс, өгөгдлийн сангийн хост холболт болон хостын боломжийг түрээслэгч рүү урсгал чиглүүлэхээс өмнө тохируулна. Өгөгдлийн сангийн итгэмжлэл, хост холболт, чиглүүлэлт эсвэл хэрэглэгч үүсгэлтэд нөлөөлөх өөрчлөлт бүрийн дараа шилжилтийн баталгаажуулалтыг ажиллуулна уу.

`localhost`-ыг зөвхөн түрээслэгчийн ажиллах орчин болон өгөгдлийн сангийн эрхүүд локал сокет эсвэл нэг машин дээрх хандалтыг шаардсан үед ашиглана. Таны үйлчилгээ үзүүлэгчийн тохируулсан эрхийн хост нь тэр бол `127.0.0.1` эсвэл үйлчилгээний хостын нэрийг ашиглана.

### Hostinger боломж {#hostinger-capability}

Hostinger олон түрээслэгчийн боломж нь хуваалцсан Hostinger интеграцын токентой ажиллана. Баталгаажуулалтын өмнө домэйн hPanel-д байгаа, DNS Hostinger Account руу зааж байгаа, мөн түрээслэгчийн өгөгдлийн сангийн тохиргоо hPanel-тэй таарч байгаа эсэхийг баталгаажуулна уу.

## Hook-ууд ба шүүлтүүрүүд {#hooks-and-filters}

### Үйлдлүүд {#actions}

- `ultimate-multisite-addon-template_init` - Нэмэлт эхлүүлсний дараа өдөөгдөнө
- `ultimate-multisite-addon-template_loaded` - Нэмэлтийн бүх файл ачаалсны дараа өдөөгдөнө

### Шүүлтүүрүүд {#filters}

- `ultimate-multisite-addon-template_settings` - Нэмэлтийн тохиргоог өөрчлөх
- `ultimate-multisite-addon-template_enabled` - Нэмэлтийг идэвхжүүлэх/идэвхгүй болгох төлөвийг дарж тохируулах
