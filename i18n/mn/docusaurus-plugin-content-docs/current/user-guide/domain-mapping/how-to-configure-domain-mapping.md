# Домен маппинг-ийг тохируулах (v2)

_**МЭЦ: Энэ нийтлэл Ultimate Multisite 2.x хувилбарыг хэрэглэдэг.**_

Премиум сүлжээний хамгийн хүчирхэг онцлог бол бидний үйлчлүүлэгчдэд өөрсдийн сайтуудад хамгийн дээд түвшний домен нэмэх боломжийг олгох явдал юм. Яагаад гэвэл, аль нь илүү мэргэжлийн харагдана: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) эсвэл [_**joesbikeshop.com**_](http://joesbikeshop.com)? Тиймээс Ultimate Multisite энэ онцлогийг гуравдагч талын plugin ашиглах шаардлагагүйгээр сууринд нь багтаасан байдлаар санал болгодог.

## Домен маппинг гэж юу вэ?

Нэрээс харахад, домен маппинг гэдэг нь Ultimate Multisite-ийн өөрийнхөө хүссэн домены хүсэлтийг авч, түүнийг тухайн домен нэмэгдсэн сүлжээний харгалзах сайтуудад холбох боломжийг хэлнэ.

### Өөрийн Ultimate Multisite Сүлжээнд Домен Маппинг-ийг Тохируулах

Домен маппинг ажиллахын тулд таны талын зарим тохиргоо хийх шаардлагатай. Гэхдээ баярлалаар, Ultimate Multisite таны хувьд хүнд ажилыг автоматаар хийж өгөх тул та шаардлагыг хялбархан хангаж чадна.

Ultimate Multisite-ийг суулгах үед wizard (туслах програм) нь **sunrise.php** файлыг заасан фолдерт автоматаар хуулж, суулгана. Энэ алхам дуусах хүртэл wizard таныг цааш явахыг зөвшөөрөхгүй.

<!-- Screenshot unavailable: Ultimate Multisite installation wizard with sunrise.php step -->

Энэ нь Ultimate Multisite-ийн суулгах wizard сүлжээний тохиргоо дууссаны дараа та шууд домен маппинг хийж эхлэхийг мэдэх ёстой гэсэн үг юм.

Ultimate Multisite-д домен маппинг зайлшгүй шаардлагагүй гэдгийг анхаарах хэрэгтэй. Та WordPress Multisite-ийн үндсэн домен маппинг функц эсвэл бусад ямар нэгэн домен маппинг шийдлийг ашиглах сонголттой.

Хэрэв та Ultimate Multisite-ийн домен маппингийг устгах шаардлагатай бол бусад домен маппинг шийдлүүдэд өгөх боломжтой. Үүнийг **Ultimate Multisite > Settings > Domain Mapping** дотор устгах функцээр идэвхгүй болгож болно.

![Domain Mapping settings page showing admin redirect, mapping message and DNS options](/img/config/domain-mapping-settings.png)

Энэ сонголтын доор та **Force Admin Redirect** гэсэн сонголтыг мөн харах боломжтой. Энэ сонголт нь таны хэрэглэгчид өөрийн тохируулсан домен болон дэддомен дээрх администратор дашбордонд нэвтрэх боломжийг ямар байлгахыг зохицуулах боломжийг олгоно.

Хэрвээ та **Force redirect to mapped domain**-ийг сонговол, таны хэрэглэгчид зөвхөн өөрийн тохируулсан домен дээрх администратор дашбордонд нэвтрэх боломжтой болно.

**Force redirect to network domain** гэсэн сонголт нь эсрэг үйлдэл хийе - таны хэрэглэгчид өөрийн тохируулсан домен дээр орохоос хамаагүй, зөвхөн дэддомен дээрх дашбордонд нэвтрэх боломжтой болно.

Мөн **Allow access to the admin by both mapped domain domain and network domain** гэсэн сонголт нь тэднийг både дэддомен болон тохируулсан домен дээрх администратор дашбордонд нэвтрэх боломжийг олгоно.

![Admin Redirect dropdown expanded showing the three redirect options](/img/config/domain-mapping-redirect-options.png)

Тохируулсан домен маппинг хийх хоёр арга байдаг. Эхний нь таны сүлжээний администратор дашбордоос супер администратор (super admin) болгон домен нэрийг мапперч авах бөгөөд хоёр дахь нь хаягийн хуудасны дор байгаа дэдсайт (subsite) администраторын дашбордыг ашиглах явдал юм.

Гэхдээ та тохируулсан доменээ сүлжээний аль нэг дэдсайтад мапперч эхлэхээс өмнө, домен нэрийн **DNS settings** зөв тохирсон байгаа эсэхийг шалгах хэрэгтэй.

### Домен DNS тохиргоо зөв байгааг шалгах

Маппинг ажиллахын тулд таны мапуулахыг төлөвлөж буй домен нь таны Сүлжээний IP хаягаар заагдаж байгаа эсэхийг шалгах хэрэгтэй. Анхаарах зүйл бол танд Сүлжээний IP хаяг - Ultimate Multisite суулгасан доменийн IP хаяг - та мапуулахыг хүссэн өөрийн домен доменийн IP хаяг биш гэдгийг анхаарна уу. Тодорхой доменийн IP хаягийг олохын тулд бид [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html)-т орохыг санал болгож байна, жишээлбэл.

Домен дээр зөв мапуулахын тулд та **DNS** тохиргоонд тэр **IP хаягаар заасан A RECORD** нэмэх хэрэгтэй. DNS-ийн удирдлага домен бүрийн бүрт ялгаатай байдаг ч, " _Creating A Record on XXXX_ "-ээр хайвал (XXX нь таны домен регистратор гэсэн утгаар) олон туслах зааврыг онлайн олж болно (жишээлбэл: " _Creating A Record on_ _GoDaddy_ ").

Хэрэв энэ ажиллагаа хийхэд хүндрэлтэй байвал **домен регистрарын дэмжлэгт хандана уу**, тэд танд энэ хэсгийг туслах боломжтой.

Хэрэв та өөрийн үйлчлүүлэгчдэд өөрсдийн домениасаа мапуулдаг байхыг зөвшөөрөх төлөвлөгөөтэй бол, тэд энэ хэсгийг өөрсдөө хийх шаардлагатай. Хэрэв тэд A Record-ийг үүсгэх боломжгүй бол тэдний домен регистрарын дэмжлэгийн систем рүү чиглүүлнэ үү.

### Супер Администратор гэж өөрийн домениасаа мапуулах

Та таны сүлжээнд супер администратор (super admin) гэж нэвтэрсэн бол **Ultimate Multisite > Domains**-д орж, өөрийн домениасаа мапускайг хялбархан нэмээд удирдах боломжтой.

![Domains list page in Ultimate Multisite](/img/admin/domains-list.png)

Энэ хуудасны доор талтсан дээрх **Add Domain** (Домен нэмэх) товчийг дарвал таны өмнө нь **custom domain name** (онцгой домен нэр), таны ашиглах **subsite** (дотоод сайт) болон үүнийг **primary domain** (үндсэн домен) гэж тохируулах эсэхийг сонгох боломжтой модал цонх гарч ирнэ. (Тайлбар: нэг subsite-д олон домен нэр холбож болно).

![Домен нэмэх модал цонх нь домен нэр, сайт сонголт ба үндсэн домены товгт байгаа](https://img/admin/domain-add-modal.png)

Бүх мэдээллийг оруулсны дараа та доорх **Add Existing Domain** (Одоо байгаа домен нэмэх) товчийг дарж болно.

Энэ нь custom domain-ийн DNS мэдээллийг шалгаж, авч авах процессыг эхлүүлнэ. Процессыг яаж явж байгааг таны дагахаар хуудасны доор лог (log) харагдана. Энэ процесс дуусах хэдэн минут шаардах боломжтой.

Ultimate Multisite v2.13.0 нь хост дээр шинэ сайт үүсэх үед, түүнийг site-д тусгайлан авч үзэх домен гэж тооцох ёстой бол дотоод домен бүртгэлийг автоматаар үүсгэнэ. Хэрэв хост нь сүлжээний үндсэн домен эсвэл **Site URL** талбарт тохируулсан shared checkout-form base domain-удын аль нэг бол, автоматаар холбогдсон домен бүртгэл алгашигдаж, тухайн домен дээр ашигладаг бүх сайтнуудад нийт үндсэн домен хялбар байдлаар хангагдана.

Хэрэв бүх зүйл зөв тохируулсан бол **Stage** (Үе шат) нь **Checking DNS** (DNS-ийг шалгаж байна) -аас **Ready** (Бэлэн) руу өөрчлөгднө.

![Домен жагсаалтад Checking DNS үе шатыг харуулсан домен мөр](https://img/admin/domain-stage-checking.png)

![Green status indicator-тай Ready үе шатыг харуулсан домен мөр](https://img/admin/domain-stage-ready.png)

Та домен нэрийг дарвал түүний доторх зарим сонголтуудыг харж болно. Тэдгээрийг хурдан үзээд үзье:

![Үе шат, сайт, идэвхтэй (active), үндсэн (primary) ба SSL-ийн товчлуур байгаа домен дэлгэцийн хуудас](https://img/admin/domain-edit.png)

**Статус (Stage):** Энэ нь таны домен ямар үе байгааг харуулдаг. Та домен анх нэмэхэд, энэ нь ихэвчлэн **DNS шалгах (Checking DNS)** үе байдаг. Энэ процесс DNS оролтуудыг шалгаж, тэдгээрийг зөв эсэхийг баталгаажуулна. Дараа нь домен **SSL шалгах (Checking SSL)** үетэй болно. Ultimate Multisite домен дээр SSL байгаа эсэхийг шалгаж, таны доменыг **Бэлэн (Ready)** эсвэл **SSLгүй бэлэн (Ready (without SSL))** гэсэн ангилалд оруулна.

**Сайт (Site):** Энэхүү доментэй холбоотой дэддомен юм. Хэрэв та доменыг тохируулбал, тухайн сайтны агуулгыг харуулдаг.

**Идэвхтэй (Active):** Та энэ сонголтыг асааж эсвэл унтрааж болно. Энэ нь домены идэвхтэй эсвэл идэвхгүй байхад ашиглагдана.

**Үндсэн домен юу вэ (Is Primary Domain?):** Таны хэрэглэгчид сайт бүрийн хувьд нэгээс дээш тохируулсан доментой байж болно. Энэ сонголтыг ашиглан энэ нь тухайн сайтны үндсэн домен болохыг сонгох боломжтой.

**Аюулгүй юу (Is Secure?):** Ultimate Multisite домен дээр SSL сертификат байгаа эсэхийг шалгадаг ч, та өөрөө SSL-тэй эсвэл үгүй байхаас хамааруулан доменыг ачаалах сонголтыг хийж болно. Анхаарах зүйл бол хэрэв вэбсайт дээр SSL сертификат байхгүй бөгөөд та түүнийг SSL-тэй гэж хүчээр ачаалгаар орохоор хичээвэл алдаа гарч болзошгүй.

### Хэрэглэгчийн доментоо дэдсайт (Subsite) хэлбэрээр тохируулах

Дэдсайтын администратор нар өөрийн дэдсайтын администратор тахианы бүртгэлээс гадна өөрсдийн доментоо дэдсайт хэлбэрийг тохируулж болно.

Эхлээд, **Domain mapping (Домен тохируулалт)** тохиргоо доор энэ сонголтыг идэвхтэй байлгасан эсэхийг шалгах хэрэгтэй. Доорх зурагт харна уу.

<!-- Screenshot unavailable: Domain mapping settings allowing subsite users to map domains via Customer DNS Management toggle -->

Та мөн **Ultimate Multisite > Products** доторх **Plan (Зарлалт)** түвшний эсвэл бүтээгдэхүүний тохиргоо доор энэ сонголтыг тохируулж, тохируулах боломжтой.

![Custom Domains section on the product edit page](/img/config/product-custom-domains.png)

Хэрэв тухайн сонголтуудын аль нэгийг идэвхжүүлсэн бөгөөд дэдсайт (subsite) хэрэглэгч нь өөрийн тохируулсан домен нэрийг (custom domain names) хайрцагт оруулах боломжтой бол, тухайн дэдсайтын хэрэглэгч **Account** (Тохиргоо) хуудсанд **Domains** (Домен) гэсэн метабокс (metabox)-ыг харах ёстой.

<!-- Screenshot unavailable: Domains metabox on the subsite Account page with Add Domain button -->

Хэрэглэгч **Add Domain** (Домен нэмэх) товчийг дарж, зааврын хэсгүүдтэй хамт модаль цонх гарч ирнэ.

<!-- Screenshot unavailable: Add Domain modal showing DNS A-record instructions for subsite users -->

Дараа нь хэрэглэгч **Next Step** (Дараагийн алхам)-ыг дарж, өөрийн тохируулсан домен нэрийг нэмэх үйл явцыг үргэлжлүүлнэ. Түүнчлэн энэ домен нь үндсэн домен болох эсэхийг сонгох боломжтой.

<!-- Screenshot unavailable: Add Domain form with custom domain name field and primary domain toggle -->

<!-- Screenshot unavailable: Add Domain confirmation step that triggers DNS verification -->

**Add Domain**-ыг дарвал тухайн домен нэрийг баталгаажуулж, DNS мэдээллийг авч авах үйл явц эхэлнэ.

### About Domain Syncing (Домен синхронизуулалт тухай)

Domain Syncing гэдэг нь Ultimate Multisite-ийн домен тохируулгын ажиллахын тулд таны хостинг аккаунтод өөрийн домен нэрийг "add-on domain" (нэмэлт домен) болгон нэмэх үйл явц юм.

Хэрэв таны хостинг провайдер Ultimate Multisite-ийн домен тохируулгын функцтэй холбоотой байвал, Domain syncing нь автоматаар явагддаг. Одоогоор ийм холбоотой хостинг провайдерудын дунд _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ болон _Cpanel_ байна.

Хостинг провайдерийн интеграци (integration) идэвхтэй байвал, Ultimate Multisite нь шинээр үүссэн сайтуудад тухайн провайдер талын DNS эсвэл дэддомен (subdomain) үүсгэх ажлыг дараалан хийж болно. Хэрэв ямар нэгэн ажил гүйцэтгэлийг сонсохгүй байвал, "no-op queue entries" (ямар ч үйлдэл хийхгүй багцын оролт) -ыг зайлуулахын тулд ардны ажлыг алгасна. Тохируулсан доменүүдийн DNS болон SSL шалгалт нь ердийн домен-стаг (domain-stage) үйл явцаар үргэлжлүүлэн ажиллана.

Энэ интеграцийг Ultimate Multisite тохиргоо доторх **Integration** таб дээр идэвхжүүлэх шаардлагатай.

![Ultimate Multisite тохиргооны Integration табны зураг, хостинг үзүүлэгчдийг харуулсан](/img/config/integrations-tab.png)
