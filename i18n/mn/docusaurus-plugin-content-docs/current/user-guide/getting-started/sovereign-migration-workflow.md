# Өөрийн шиг шилжих ажлын урсгал

Ultimate Multisite: Multi-Tenancy 1.2.0 нь ердийн дэдсайт-ыг өөрийн төрийн (sovereign) багцын дэдсайт руу шилжүүлэх үед шалгах цэгүүдийг нэмсэн.

## Эхлэхээс өмнө хийх зүйлс

Та дараах зүйлсийг бэлэн байгаа эсэхийг баталгаажуулна уу:

- Эх сурвалж сайтын одоогийн бэкап байх.
- Multi-Tenancy addon ажиллаж байгаа байх.
- Зорилтот багцын (tenant) өгөгдлийн сан, файлын систем үндэс (filesystem root), домен бэлэн байх.
- Зорилтот орчны хослол (database host binding) зохицсон байх.
- Сүлжээнд WP-CLI команд ажиллуулах эрхтэй байх.

## Зөвлөдөг ажлын урсгал

1. Зорилтот багцын өгөгдлийн сан болон файлын системийг бэлтгэх.
2. Багцын тусгайлан хязгаарлалт (tenant isolation) тохиргоог бүртгэх эсвэл шинэчлэх.
3. Багцын шилжилтийг ажиллуулах.
4. Багцын хэрэглэгчийг үүсгэх эсвэл шалгах.
5. Асинх шилжилт хийж буй ажлын дарааллыг (async migration jobs) устгах.
6. Өөрийн шиг шилжих баталгаажуулалт (sovereign migration verification)-ыг ажиллуулах.
7. SSO-тойгоор багцын руу орох.
8. Баталгаажуулалт амжилтад хүрсний дараа DNS эсвэл маршрутизацийг өөрчлөх.

## Шалгах цэгүүд (Verification gates)

Шалгах ажлын урсгал шилжилтийг хэд хэдэн талаас шалгадаг:

- Багцын схемийн бүтэц байгаа бөгөөд багцын өгөгдлийн сан бичдэг байх.
- Тохируулсан өгөгдлийн сан хост нь хүчинтэй эрхий (grants)тэй байх.
- Багцын хэрэглэгчид байгаа бөгөөд өөрийн шиг тавигдаж буй багцын хэрэглэгчдийн тоотой таарч байх.
- Асинх push queue амжилтаар устгагдах боломжтой байх.
- Өөрийн шиг шилжих багцад өмнөх сүлжээний (legacy network-side) өгөгдлийн замаар шаардлагагүй болсон байх.

Шалгах үед алдаа гарвал эхлэхээс өмнөх саад гэж үзнэ. Өгөгдлийн сан, хэрэглэгч, queue эсвэл маршрутизацийн асуудлыг засаад дахин шалгалт хийгээд багцыг хэрэглэгчдэд харуулахаас өмнө дахин шалгах хэрэгтэй.

## Эхний үйлдвэрлэлийн оролцоо (First production visit)

Шалгалт амжилтад хүрсний дараа, анхны администратор руу орохын тулд сайт удирдах дэлгэцээс **Visit (SSO)**-г ашиглана. Энэ нь багцын маршрутлалт, SSO токен боловсруулалт, эх үүсвэрийн тогтоолт, мөн багцын талын хэрэглэгчийг нэг зохицуулсан алхамтайга шалгадаг.
