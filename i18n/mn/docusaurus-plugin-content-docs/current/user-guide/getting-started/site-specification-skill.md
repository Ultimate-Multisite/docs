# Сайтын Тодорхойлолтын Чадвар

**Сайт тодорхойлох (Site Specification) чадвар** гэдэг нь таны сайтын зорилго, үзэгчдийн бүтэц болон брэндийн өвөрмөц байдлыг ойлгох, түүнийг бичих бүтцийн арга юм. Энэ мэдээлэл нь **site_brief** гэх санах ойд хадгалагдаж, агент нар хэрэглэх бөгөөд энэ нь өөр өөр сессионд ч тогтвортой, контексттэй тусламж үзүүлэхэд ашиглагддаг.

## Сайт тодорхойлох гэдэг юу вэ?

Сайт тодорхойлох гэдэг нь дараах зүйлсийг баримтжуулах үйл явц юм:

- **Сайтын зорилго**: Таны сайт юу хийдэг бөгөөд яагаад байдаг вэ.
- **Зорилтот үзэгч**: Хэн таны сайтуудыг үзэж байгаа бөгөөд тэдэнд юу хэрэгтэй вэ.
- **Брэндийн өвөрмөц байдал**: Таны өнгө, дуу хоолой (tone) болон харагдах хэв маяг.
- **Бизнесийн зорилго**: Таны сайт амжилттай байхын тулд юу хийхийг хүсдэг вэ.
- **Агуулгын бүтэц**: Таны сайт ямар байдлаар зохион байгуулагдсан бэ.

Энэ тодорхойлолт нь таны **site_brief** болж, агент нар таны сайтын контекст ойлгоход ашигладаг тогтвортой санах ойд хөрвөнө.

## Яагаад Сайт тодорхойлох чадварыг ашиглах вэ?

### Сессион бүрт тогтвортой байдал

site_brief байхгүй бол та шинэ сессион эхлэх бүрт сайтын зорилгыг дахин тайлбарлах шаардлагатай болно. Энэ нь байгаа үед агент нар шууд ойлгодог:

- Таны сайтын зорилго ба үзэгчдийн төрөл
- Таны брэндийн өнгө, дуу хоолой
- Таны агуулгын бүтэц
- Таны бизнесийн зорилго

### Сайжруулсан зөвлөмжүүд

Агент нар site_brief-ийг ашиглан дараах үйлдлүүдийг хийдэг:

- Таны сайтын зорилгод нийцсэн функцуудыг санал болгох
- Таны зорилгод тохирох агуулгын бүтцийг зөвлөх
- Таны брэндийн дагуу дизайны саналыг өргөн барих
- Тохирохгүй функцуудыг санал болгохоос зайлсхийх

### Хурдан эхлүүлэх (Onboarding)

Шинэ агент нар (эсвэл шинэ сессионд орсон агент нар) тодруулсан асуулт асуухын оронд site_brief-ийг уншихаар хурдан ойлгодог.

## Сайт тодорхойлохыг эхлүүлэх

### Theme Builder Onboarding үед

Site Specification чадвар нь **Theme Builder onboarding flow** (Сайт бүтээгч орнол) үеэр автоматаар эхлүүлдэг. Setup Assistant агент асуулт тавьж, site_brief-ийг бүрдүүлдэг.

### Гаралт өгөхөөр эхлүүлэх (Manual Initiation)

Таны сайтын тодорхойлолтыг ямар ч арай эхлэх боломжтой:

```
"Миний сайтын тодорхойлолтыг тодорхойлъё"
```

эсвэл

```
"Сайтны товч тайлбар бичихэд туслаач"
```

## Сайтын тодорхойлолтын үйл явц

### Алхам 1: Сайтын зорилго

Агент дараах асуултыг задална:

```
Таны сайтын гол зорилго юу вэ?
- Электрон худалдааны дэлгүүр (E-commerce store)
- Блогоос эсвэл контент сайт
- Портфолио эсвэл үзүүлбэр
- SaaS аппликейшн
- Нийгэмлэг эсвэл форум
- Бусад: [тодорхойл]
```

Та ангилалыг сонгож болно эсвэл өөрийн зорилгыг тодорхойлж болно.

### Алхам 2: Зорилтот үзэгч

```
Таны гол зорилтот үзэгч хэн бэ?
- Хэрэглэгчид / ерөнхий нийт хүмүүс
- Бизнесийн мэргэдлийн хүмүүс
- Хөгжүүлэгчид / техникийн хэрэглэгчид
- Оюутнууд / боловсролын байгууллагууд
- Бусад: [тодорхойл]

Тэдний гол хэрэгцээ юу вэ?
```

### Алхам 3: Брэндийн өвөрмөц байдал (Brand Identity)

```
Таны брэндийн өнгөнүүд ямар байна вэ?
- Үндсэн өнгө: [өнгө сонгогч эсвэл hex код]
- Хоёр дахь өнгө: [өнгө сонгогч эсвэл hex код]
- Анхаарах өнгө (Accent color): [боломжтой]

Таны брэндийн аялга хэрхэн тодорхойлдог вэ?
- Профессионал / корпоратив
- Сүт цэцэрлэг / урлагийн
- Тоглоом шинжтэй / ердийн
- Минималист / орчин үеийн
- Дулаан / найрсаг
```

### Алхам 4: Бизнесийн зорилго

```
Таны сайтын амжилт ямар харагдах вэ?
- Мэдээлэл олох (leads)
- Бүтээгдэхүүн худалдах
- Нийгэмлэгийг бий болгох
- Оролцоог хуваалцах
- Үүрэг байрлалыг тогтоох
- Бусад: [тодорхойл]

Таны гол үзүүлэлт (metric) юу вэ?
- Орлого
- Хэрэглэгчийн оролцоо
- Контентын хүрэх хэмжээ
- Худалдан авалт хийх тоо (Conversions)
- Бусад
```

### Алхам 5: Агуулгын бүтэц

```
Таны контент ямар байдлаар зохион байгуулагддаг вэ?
- Тез (бүх контент ижил түвшинд)
- Хавтанчилсан (категори, дэдкатегори)
- Цаг хугацааны дараалсан (блогоор)
- Бүтээгдэхүүний үндэслэсэн (каталог)
- Бусад: [тодорхойл]

Та ямар төрлийн контент ашигладаг вэ?
- Блогоны нийтлэл
- Бүтээгдэхүүний хуудас
- Үндсэн судалгаа (Case studies)
- Хөтөлбөр/Тайлбар (Documentation)
- Видео
- Бусад
```

## Таны site_brief санах ой

Сайтын тодорхойлолтыг дуусгасны дараа, таны мэдээлэл **site_brief** гэсэн хэсэгт агентны санах ойд хадгалагдана. Энэ нь дараах зүйлсийг агуулсан бүтэцтэй бүртгэл юм:

## site_brief-ийн харалт ба шинэчлэх

### Site brief-ийг үзэх

Агентэд дараах зөв асуулт асууж болно:

```
"Миний site brief-ийг харуулна уу"
```

эсвэл

```
"Миний сайт тухай юу мэдэх вэ?"
```

Агент таны хадгалагдсан тодорхойлолтыг харуулна.

### Site brief-ийг шинэчлэх

Хэрэв таны сайт өөрчлөгдсөн бол та үүнийг шинэчлэж болно:

```
"Миний site brief-ийг шинэчилнэ: бид одоо B2B хэрэглэгчдэд чиглэнэ"
```

эсвэл

```
"Миний сайт тухай тодорхойлолтыг шинэчилнэ"
```

Энэ нь таны байгаа мэдээллийг эхлэлийн цэг болгон ашиглан Site Specification чадварыг дахин гүйцэтгэнэ.

## Агентууд site_brief-ийг хэрхэн ашигладаг вэ

### Дизайны зөвлөмжүүд

Та дизайн өөрчлөлтийн тухай асуувал, агент таны site brief-ийг дурдах болно:

```
Та: "Миний үндсэн хуудсыг дахин загварч үү"
Агент: "Таны site brief-д 25-45 настай эмэгтэйчүүдийг хамарсан, дулаахан, урлагийн (artisanal) өнгө аястай гэж заасан байна. Би таны гарын авлагатай болон тогтвортой хэрэглээний утгыг онцолсон дизайн бүтээе."
```

### Онцлог шинжилгээ хийх санал

Агентууд таны зорилгод нийцсэн онцлог шинжилгээ хийх саналыг өгнө:

```
Та: "Би юу нэмэх ёстой вэ?"
Агент: "Хэрэв хэрэглэгчийн үнэнч байдлыг чухалчилсан э-коммерц сайтын хувьд, би дараах зүйлсийг санал болгоно: үнэнч байдлын програм (loyalty program), хэрэглэгчийн үнэлгээ, цахим шуудангийн мэдэгдэл (email newsletter) болон бүтээгдэхүүний санал."
```

### Агуулгын бүтэц

Агентууд таны бүтэцэд үндэслэн агуулгын зохион байгуулалтыг санаачилдаг:

Таны блог хэрхэн зохион байгуулах вэ?
Би таны бүтээгдэхүүний каталогтой хамт блог ашиглаж байгаа тул, би танд нийтлэг сэдвээр пост хийхээс илүү бүтээгдэхүүний ангилалаар зохион байгуулах, мөн олон бүтээгдлээр нэгтгэсэн "стилийн гарын авлага" (styling guides) үүсгэхэд зөвлөж байна.

## Хамгийн сайн практик

### Тодорхой байх
"Ерөнхий үзэгчдийн" оронд өөрийн бодит үзэгчдийг тодорхой тайлбарлаарай:

- ✓ "25-45 настай, байнга тогтвортой модыг сонирхож буй эмэгтэйчүүд"
- ✗ "Хүн бүр"

### Үргэлж шинэчлэх
Таны сайт өөрчлөгдөхөд `site_brief`-ийг дахин шинэчилнэ үү:

- Шинэ үзэгчдэд чиглэх үед
- Шинэ бүтээгдэхүүний мөр нэмэх үед
- Брэндийн өвөрмөц байдал өөрчлөгдөх үед
- Бизнесийн зорилго өөрчлөгдөх үед

### Ижил терминологийг ашиглах
Хэрэглээний хугацаанд ижил нэр томьёог ашиглана уу:

- ✓ Үргэлж "байнга тогтвортой бодитой зүйл" гэж хэлнэ (бодлогын үед "эко-хэрэглээний зүйл" эсвэл "ногоон бүтээгдэхүүн" гэж хэлэхгүй)
- ✓ Үзэгчдийг ижил байдлаар дурдах

### Контекст өгөх
Агент таны шийдвэрийг ойлгоход туслах ардчилсан мэдээллийг өгөөрэй:

- "Бид үнэ보다 чанараас илүүтэй мэдэгдэж буй мэргэжилтнүүдийг зорино"
- "Манай үзэгчид технологийн мэдлэгтэй бөгөөд орчин үеийн дизайныг хүлээж байна"
- "Бид өөрсдийн хүчээр эхэлсэн компани тул бидэнд зардлаар боломжит шийдлүүд хэрэгтэй"

## Theme Builder-т холбоотой байдал

Site Specification чадвар нь **Theme Builder onboarding flow**-д нэгтгэсэн байна. Та onboarding дуусгасны дараа таны `site_brief` өгсөн мэдээллээр автоматаар үүснэ.

Хэрэв та дараах зүйлсийг хийхийг хүсвэл Site Specification-ийг бие даан ажиллуулах боломжтой:

- Эхний тохиргоо хийсний дараа өөрийн тодорхойлолтыг сайжруулах
- Сайт өөрчлөгдөхөд `site_brief`-ийг шинэчлэх
- Theme Builder эхлэхээс өмнө дэлгэрэнгүй тодорхойлолт хийх

## Асуудал шийдэх (Troubleshooting)

**Миний site_brief ашиглагдаагүй байна**
- Агентд memory (санах ой)-т хандах эрх байгаа эсэхийг баталгаажуулна уу.
- Агентэд "миний site_brief-ийг дур" гэж асуугаарай.
- Таны тохиргооноос memory-г идэвхтэй (enabled) байгаа эсэхийг шалгаарай.

**Би шинэ site\_brief-ээр эхлэхийг хүсэж байна**
- Агентэд: "Миний site brief-ийг цэвэрлэж, шинээр эхлэх" гэж асуух.
- Дараа нь Site Specification-ыг дахин гүйцэтгэх.

**Агент минь миний site\_brief-тэй таарахгүй зөвлөмж өгч байна**
- Агентэд: "Миний site brief-ийг хянана уу" гэж асуух.
- Хэрэв site\_brief хуучирсан бол үүнийг шинэчлэх.
- Өөрийн хүсэлтүүдэд нэмэлт контекст өгөх.

## Дараагийн алхамууд

Site specification-ыг тодорхойлсны дараа:

1. **Theme Builder ашиглах**: Таны site\_brief-ээ үндэслэн өөрийн загвар (theme) үүсгэх
2. **Дизайныг сайжруулах**: Дэлгэрэнгүй дизайн хийхийн тулд Design System Aesthetics чадварыг ашиглах
3. **Контентыг төлөвлөх**: Контентын бүтэц хэрхэн байх талаар агентуудаас зөвлөмж асуух
4. **Онцлогийг үүсгэх**: Бизнесийн зорилготой нийцсэн онцлог (features) хүсэлт гаргах
