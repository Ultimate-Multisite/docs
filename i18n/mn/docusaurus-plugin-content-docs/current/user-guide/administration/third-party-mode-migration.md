# Гуравдагч талын режимээс шилжих

Superdav AI Agent v1.12.0 нь гуравдагч талын чадваруудыг хэрхэн зохицуулахыг өөрчилсөн. **Гуравдагч талын режим одоо авто (auto) гэсэн утгаар анхны тохиргоотой болж байгаа тул WordPress 7.0+ дээр гараар тохируулдаггүйгээр үндсэн WordPress Abilities API-тэй шууд холбогдож чадна.**

## Юу өөрчлөгдсөн бэ?

### v1.12.0-ээс өмнө

Гуравдагч талын чадваруудыг гараар тохируулдаг байсан:

- "third-party mode"-ийг тодорхой идэвхжүүлэх шаардлагатай байсан
- Чадваруудыг тусгай бүртгэлээс ачаалдаг байсан
- WordPress Abilities API-тэй холбогдох нь сонголттой байсан
- Хуучин режим (Legacy mode) анхны тохиргоо байсан

### v1.12.0-өөс хойш

Гуравдагч талын чадварууд автоматаар ажиллана:

- Гуравдагч талын режим "auto" гэсэн утгаар анхны тохиргоотой
- Чадварууд WordPress Abilities API-тэй шууд нийцнэ
- WordPress 7.0+ дээр гараар тохиргоо хийх шаардлагагүй
- Хуучин режим нь өмнөх хувилбаруудад ашиглахаар үлдэнэ

## Хэн нөлөөлөгдөж байна вэ?

### Шинээр суулгасан (WordPress 7.0+)

**Ямар ч үйлдлээ хийх шаардлагагүй.** Гуравдагч талын режим автоматаар "auto" гэсэн утгаар тохиргоолагдаж, чадварууд шууд ажиллана.

### Одоо байгаа суулгасан системүүд

**Таны тохиргоо хадгалагдсан.** Хэрэв та дараах аль нэгийг ашиглаж байсан бол:

- **Legacy mode**: Та хуучин режимынхаа үлдэнэ (өөрчлөлтгүй)
- **Manual third-party mode**: Та гараар тохируулсан режимынхаа үлдэнэ (өөрчлөлтгүй)
- **Auto mode**: Та авто режимтай үргэлжлүүлэн ажиллана (өөрчлөлтгүй)

### 7.0-ээс өмнөх WordPress хувилбарууд

**Legacy mode ашиглах боломжтой.** Хэрэв та WordPress 6.x эсвэл түүнээс өмнөх хувилбарыг ашиглаж байгаа бол:

- Гуравдагч талын режим "legacy" гэсэн утгаар анхны тохиргоотой
- Хэрэв хүсвэл гуравдагч талын режимыг гараар идэвхжүүлэх боломжтой
- Үндсэн Abilities API-ийг ашиглахын тулд WordPress 7.0+ руу шилжих шаардлагатай

## Режимуудыг ойлгох

### Auto Mode (Шинэ анхны тохиргоо)

**Auto mode** нь үндсэн WordPress Abilities API-тэй шууд холбогдоно:

- Чадварууд WordPress hooks-аар бүртгэгддэг
- WordPress 7.0+ Abilities API-тай бүрэн нийцэлтэй
- Гуравдагч талын чадваруудыг автоматаар олж авна
- Гараар тохиргоо хийх шаардлагагүй

**Хэзээ ашиглах**: WordPress 7.0+ болон гуравдагч талын чадвар (third-party abilities) ашиглах үед

### Manual Mode (Гаралттай режим)

**Manual mode** нь тодорхой тохиргоо хийхийг шаарддаг:

- Та аль гуравдагч талын чадваруудыг ачаахыг зааж өгнө
- Тест хийх эсвэл сонгодоор чадвар ачаах үед хэрэгтэй
- Тохиргооны файлыг өөрчлөх шаарддаг
- Хяналт илүү их, гэхдээ тохиргоо хийх нь илүү олон

**Хэзээ ашиглах**: Тест хийх, сонгодоор чадвар ачаах, эсвэл өөрийн онцгой тохиргоо хийх үед

### Legacy Mode (Уламжлалт режим)

**Legacy mode** нь хуучин гуравдагч талын чадварын системээ ашигладаг:

- Өөрийн чадварын бүртгэл (WordPress Abilities API биш)
- Хуучин WordPress хувилбаруудад нийцсэн
- WordPress-тэй шууд холбоогүй
- Хуучирсан ч дэмждэг

**Хэзээ ашиглах**: WordPress 6.x эсвэл түүнээс өмнөх хувилбар, эсвэл уламжлалт нийцлийг шаарддаг үед

## Одоогийн режимыг шалгах

### Admin Panel-ээр (Удирдлагын хэсгээс)

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** руу орно
2. **Third-Party Mode** тохиргоог хайна уу
3. Одоогийн режим болон өөрчлөх сонголтуудыг харж болно

### Кодоор (Code)

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', эсвэл 'legacy'
```

## Режимыг өөрчлөх

### Auto Mode руу шилжих (Авто режим)

Хэрэв та WordPress 7.0+ дээр байгаа бөгөөд авто режимыг ашиглахыг хүсвэл:

1. **Superdav AI Agent** → **Settings** руу орно
2. **Third-Party Mode**-ийг олно
3. **Auto (WordPress Abilities API)**-г сонгоно
4. **Save**-д дарна

Superdav AI Agent нь гуравдагч талын чадваруудыг автоматаар олж бүртгэж өгнө.

### Manual Mode руу шилжих (Гаралттай режим)

Хэрэв аль чадвар ачаахыг гараар хянахыг хүсвэл:

1. **Superdav AI Agent** → **Settings** руу орно
2. **Third-Party Mode**-ийг олно
3. **Manual**-г сонгоно
4. **Save**-д дарна
5. Аль чадварыг ачаахыг тодорхойлохын тулд тохиргооны файлыг өөрчилнө

### Legacy Mode руу шилжих (Уламжлалт режим)

Хэрэв та уламжлалт нийцлийг шаарддаг бол:

1. **Superdav AI Agent** рууц рууц (Settings) рүү орно
2. **Third-Party Mode**-г олно
3. **Legacy**-г сонгоно
4. **Save**-д дарна

## Auto Mode-ийн давуу талууд

### Автомат олж авах (Automatic Discovery)

Боломжууд дараах зүйлсээс автоматаар олддог:

- Угтаа байсан plugin-үүд
- Идэвхтэй theme
- Заавал хэрэглэх ёстой plugin-үүд
- Drop-in plugin-үүд

Хувийн бүртгэл хийх шаардлагагүй.

### Эхний системтэй нийцэл (Native Integration)

Боломжууд WordPress Abilities API-тай холбогдоно:

- WordPress-ийн үндсэн хэсэгтэй ижил байна
- WordPress admin-тэй ажилладаг
- Abilities API-г ашигладаг бусад plugin-үүдтэй нийцдэг
- WordPress шинэчлэгдэх тусам ирээдүйд бэлэн байдаг

### Хялбар удирдах (Simplified Management)

- Засах шаардлагатай configuration file байхгүй
- Хувийн ability бүртгэл хийх шаардлагагүй
- Ability Visibility тохиргоо автоматаар ажиллана
- Admin notices танд ангилагдаагүй abilities-ийг мэдэгдүүлнэ

### Сайхан гүйцэтгэл (Better Performance)

- Abilities cache хийгддэг
- Хэрэгцээтэй үед lazy-load хийгддэг
- WordPress 7.0+ дээр оновчтой ажилладаг

## Шилжилт хийх зам (Migration Path)

### Хэрэв та WordPress 6.x дээр байгаа бол

1. **WordPress 7.0+ руу шинэчил** (бэлэн бол)
2. **Superdav AI Agent-ийг v1.12.0+-д шинэчлэх**
3. **third-party mode-г Auto руу өөрчлөх** (боломжтой; legacy mode ажиллана)
4. Зөв нэвтрэх хяналтыг баталгаажуулахын тулд ability visibility-г шалгах

### Хэрэв та WordPress 7.0+ дээр байгаа бол

1. **Superdav AI Agent-ийг v1.12.0+-д шинэчлэх**
2. **third-party mode Auto-т тохирсон эсэхийг шалгах** (default-оор ирнэ)
3. Зөв нэвтрэх хяналтыг баталгаажуулахын тулд ability visibility-г шалгах
4. Ажиллаж байгаа эсэхийг баталгаажуулахын тулд third-party abilities-ийг тест хийх

## Асуудал шийдэх (Troubleshooting)

### Auto mode-д abilities ачаагүй бол

- Та WordPress 7.0+ дээр байгаа эсэхийг шалгана
- third-party mode "Auto"-т тохирсон эсэхийг шалгана
- Ability-г өгч буй plugin идэвхтэй байгаа эсэхийг шалгана
- Бүртгэл алдаатай байгаа эсэхийг WordPress error logs-оос шалгана

### Би legacy mode-ийг хадгалахыг хүсэж байна

- **Settings** → **Third-Party Mode** руу орно
- **Legacy**-г сонгоно
- **Save**-д дарна
- Legacy mode үргэлж ажиллана

### Миний өгөгдсөн чадварууд харагдахгүй байна

- Тэд WordPress hook-оор бүртгэгдсэн эсэхийг шалгана уу
- Тэд Abilities API-г зөв хэрэгжүүлж байгаа эсэхийг шалгана уу
- WordPress error logs-ыг үзнэ үү
- Бүх бүртгэгдсэн чадваруудыг **Ability Visibility** администратор хуудсаар харна уу

### "анклассификейд чадвар" (unclassified ability) мэдэгдэл гарч байна

- Энэ нь шинэ гуравдагч талын чадваруудтай үед ердийн зүйл юм
- Администратор дээрх мэдэгдлээр тэдгээрийг үзэж, ангилаалах хэрэгтэй
- Ангилаллын талаар **Ability Visibility**-г харна уу

## Урьдчилсан нийцэл (Backward Compatibility)

### Одоо байгаа тохиргоонууд

Хэрэв таны одоо байгаа гуравдагч талын чадварын тохиргоо байвал:

- **Legacy mode** (Уламжлалт режим): Таны тохиргоо ажилласаар байх үлдэнэ
- **Manual mode** (Хүчээр удирдах режим): Таны тохиргоо ажилласаар байх үлдэнэ
- **Auto mode** (Авто режим): Таны тохиргоо үл тоомсорлдог (авто режим давамгайлж байна)

Өөрийн өгөгдсөн тохиргоог хадгалахын тулд Manual эсвэл Legacy модонд байх хэрэгтэй.

### Хуучирсан хугацааны төлөв (Deprecation Timeline)

- **v1.12.0**: Legacy болон Manual режим бүрэн дэмждэг
- **v1.13.0+**: Legacy mode-д хуучирсан мэдэгдэл гарч болно
- **v2.0.0**: Legacy mode арилгах боломжтой (тодорхойгүй)

## Хамгийн сайн практик (Best Practices)

### Шинэ суулгалт хийх үед

- Auto mode-ийг ашиглана уу (энэ нь анхны тохиргоо)
- Superdav AI Agent-д чадваруудыг автоматаар олж үзүүлэв
- Хүснэгтээ хянахад **Ability Visibility**-г ашиглана уу

### Одоо байгаа суулгалт хийх үед

- Боломжтой бол WordPress 7.0+ руу шилжинэ үү
- Удирдан чиглэлийг хялбарчлахын тулд Auto mode руу шилжих нь зөв
- **Ability Visibility** ашиглан чадваруудыг үзэж, ангилаалах

### Өгөгдсөн чадварууд (Custom Abilities)

- WordPress hook-оор бүртгэнэ үү (Abilities API)
- өөрийн чадварын бүртгэлээс зайлсхийх хэрэгтэй
- Auto mode-той хамт WordPress 7.0+ дээр туршиж үзнэ үү

## Дараагийн алхамууд (Next Steps)

1. **WordPress хувилбарыг шалгах**: Авто режим ашиглахад тааруулахын тулд та 7.0+ хувилбар дээр байгаа эсэхийг баталгаажуулаарай.
2. **Түүхий (third-party) режийг хянах**: Settings руу орж одоогийн режимээ шалгана уу.
3. **Шаардлагатай бол шинэчлэх**: Хэрэв та WordPress 7.0+ дээр байгаа бол Auto mode руу шилжих нь зүйтэй.
4. **Чадварлагыг ангилах**: Ангилагдаагүй байдаг чадварлагуудыг хяниж, ангилан үзээрэй.
5. **Тест хийх**: Таны түүхий (third-party) чадварлагууд зөв ажиллаж байгаа эсэхийг шалгана уу.

## Холбоотой сэдвүүд

- **Чадварлагын харагдахуйц байдал**: Ямар чадварлагууд хаана илэрхийлэгдэхээ хэрхэн зохицуулахыг мэдэх
- **WordPress Abilities API**: WordPress-ийн үндсэн чадварлагыг бүртгэх тухай суралцах
- **Түүхий (Third-Party) Чадварлагыг хөгжүүлэх**: Auto mode-той ажилладаг чадварлагууд үүсгэх
