# Gratis AI Agent Тохируулгууд (Settings)

Gratis AI Agent-ийн **Settings → Advanced** дэлгэц нь v1.5.0-оос эхлээд нэвтрүүлсэн систем дээрх хойд талын холболтуудыг удирдахад зориулсан администратор түвшний тохиргоог харуулдаг. Энэ хуудас **Feedback Endpoint** хэсэг болон түүний шаардлагатай форматаас тусгай мэдээлэл өгнө.

## Тохиргоо руу орох

1. WordPress-ийн администратор дотор, **Gratis AI Agent → Settings** руу орно.
2. **Advanced** (Дээд түвшний) таб дээр дарна.

## Feedback Endpoint тохиргоо

Feedback endpoint нь хэрэв хэрэглэгч "баасан доод" (thumbs-down) товч, авто-prompt banner, эсвэл `/report-issue` командыг ашиглан санал өгөх үед AI агентээс POST хүсэлт авдаг.

| Хэсэг | Тодорхойлолт |
|---|---|
| **Feedback Endpoint URL** | HTTP POST хүсэлтийн JSON бие байна дахь feedback-ийг хүлээн авах URL. |
| **Feedback API Key** | Нэг бүрийн feedback хүсэлтэд `Authorization` header дотор илгээгдэх bearer token. Хэрэв таны endpoint-т баталгаажуулалт шаардлагагүй бол хоосон үлдээгээрэй. |

### Хүлээгдэж буй JSON Payload

Таны feedback endpoint нь дараах хамгийн бага хэсгүүдийг агуулсан JSON биеийг хүлээн авах ёстой:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Хэлцээс хамааруулан payload-д нэмэлт хэсгүүд байж болно.

### `triage_category` Үнээрх утгууд

AI triage давхарга нь payload-г урьдчилан шилжүүлэхийн өмнө `triage_category`-д дараах утгуудын аль нэгийг онооно:

| Утга | Мэдэгдэл |
|---|---|
| `factual_error` | Туслах нь буруу мэдээллийг асуудалтай өгсөн. |
| `unhelpful_answer` | Хариулт техникийн хувьд зөв боловч хэрэггүй байсан. |
| `inappropriate_content` | Хариултад хэрэглэгчдэд харуулах ёсгүй агуулга багтсан. |
| `other` | Feedback нь мэдэгдсэн ангилалд таарахгүй байна. |

### Баталгаажуулалт (Authentication)

Хэрэв таны endpoint-д баталгаажуулалт шаардлагатай бол **Feedback API Key** талбарыг өөрийн bearer token-ээр тохируулна уу. Агент дараах байдлаар илгээдэг:

```
Authorization: Bearer <your-api-key>
```

Хэрэв **Feedback API Key** талбар хоосон бол `Authorization` header ямар ч илгээгдэхгүй.

### Feedback цуглуулахыг унтраах

**Feedback Endpoint URL** болон **Feedback API Key** талбаруудыг хоёуланг нь хоосон үлдээгээрэй. Дундаж хурууны доод товч (thumbs-down button) болон feedback UI хэрэглэгчдэд харагдах байх боловч, илгээмж ямар нэгэн гадаад үйлчилгээнд дамжуулагдахгүй.

## Brave Search API Key

Мөн **Advanced** таб дээрх **Brave Search API Key** талбар нь [Internet Search](../configuration/internet-search) чадварыг идэвхжүүлдэг.

| Талбар | Тодорхойлолт |
|---|---|
| **Brave Search API Key** | Brave Search хөгжүүлэгчийн dashboard-оос авсан API key. AI ассистентт интернэт хайрцагтыг идэвхжүүлэхийн тулд шаардлагатай. |

Талбарын гарчиг нь Brave Search API-ийн бүртгэл хийх хуудас руу хүргэх товчлуур агуулдаг. Интернэт хайрцагтыг унтраахын тулд хоосон үлдээгээрэй.

Энэ функц тухай эцсийн хэрэглэгчийн баримт бичлагыг [Internet Search](../configuration/internet-search) -д үзнэ үү.

## Feature Flags (Онцлох тэмдэглэгээ)

v1.9.0-оос нэвтрүүлсэн **Settings → Feature Flags** таб нь сонголттой функцуудын хувьд унтраах/төлөвлөх товчлуур (toggle switches) өгдөг. Нэг flag нь бүх сүлжээнд идэвхтэй эсвэл унтраагдсан байдаг; одоогийн байдлаар сайт тус бүрт ямар нэгэн давталт хийх боломжгүй.

### Feature Flags-ыг ашиглах

1. WordPress admin-д орж, **Gratis AI Agent → Settings** руу очино.
2. **Feature Flags** таб дээр дарна.

### Access Control Flags (Хүцвэрлэл батлах тэмдэглэгээ)

| Flag | Default | Угвар |
|---|---|---|
| **Зарлаг администраторам** | Выкл | У включенным состоянием только пользователи с ролью `administrator` могут открывать панель чата AI Agent. Все остальные роли увидят сообщение "Свяжитесь с вашим администратором". |
| **Ограничить сетевыми администраторами** | Выкл | При включении в сети мультисайтов только Суперадминистраторы могут использовать агента. Администраторы отдельных сайтов заблокированы. Если оба параметра включены, это имеет приоритет над "Зарлаг администраторам". |
| **Разрешить доступ подписчиков** | Выкл | При включенном состоянии пользователи с ролью `subscriber` могут использовать интерфейс чата, но их возможности ограничены только чтением (нет создания постов или изменения настроек). |
| **Отключить для не-участников** | Выкл | Интегрируется со статусом членства Ultimate Multisite. При включенном состоянии чат скрывается для сайтов, у которых нет активного членства. |

### Флаги брендинга (Branding Flags)

| Flag | Default | Угвар |
|---|---|---|
| **Скрыть подвал "Powered by Gratis AI Agent"** | Выкл | Удаляет строку с указанием бренда внизу виджета чата. Рекомендуется для развертываний с белым брендингом (white-label). |
| **Название агента по умолчанию** | *(пусто)* | Заменяет стандартную метку "Gratis AI Agent" в заголовке чата и меню администратора на ваше собственное название продукта. Оставьте пустым, чтобы использовать значение по умолчанию. |
| **Скрыть селектор агентов** | Выкл | При включенном состоянии пользователи не могут переключаться между пятью встроенными агентами. Текущий агент фиксируется на том, который настроен как значение по умолчанию в Настройках → Общие (Settings → General). |
| **Использовать иконку сайта как аватар чата** | Выкл | Заменяет стандартную иконку AI в заголовке виджета чата на иконку вашего WordPress-сайта (настраивается в Внешний вид → Настроить → Идентичность сайта). |

### Применение изменений

Нажмите **Сохранить настройки** после переключения любого флага. Изменения вступают в силу немедленно — перезагрузка кэша или повторная активация плагина не требуются.
