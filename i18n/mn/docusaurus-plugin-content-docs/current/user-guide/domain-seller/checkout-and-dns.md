# Checkout Field байнга болон Хэрэглэгчийн DNS Удирдлага

## Домен Сонгох checkout хэсэг

**Domain Selection** (Домен Сонголт)-ийн талбар гэдэг нь үйлчлүүлэгчдэд өөрийн сайтын доменыг яаж авах талаар сонголт олгодог checkout элемент юм. Домен борлуулахыг хийхийн тулд ямар ч checkout form-д үүнийг нэмээрэй.

### Checkout form-д талбары нэмэх

1. **Network Admin › Ultimate Multisite › Checkout Forms** руу орно.
2. Checkout form-ийг нээ거나 шинээр үүсгэ.
3. Checkout editor дээр **Add Field** (Талбар нэмэх)-ыг дарна.
4. Талбарын жагсаалтаас **Domain Selection**-ыг сонгоно.
5. Талбарын тохиргоог хийнэ (доор харагдах).
6. Формыг хадгална.

### Талбарын тохиргоо

**Domain modes** (Доменийн горим) — Хэрэглэгч ямар табнуудыг харж байгааг сонгоно. Нэг горимыг бие даан идэвхтэй эсвэл идэвхгүй болгож болно:

| Mode | Яаж ажилладаг вэ |
|---|---|
| **Subdomain** (Поддомен) | Хэрэглэгч танай сүлжээнд үнэгүй поддомен ашиглана (жишээлбэл, `mysite.yournetwork.com`). Төлбөр шаардлагагүй. |
| **Register New Domain** (Шинэ домен бүртгэх) | Хэрэглэгч шинэ домен хайж, танай тохируулсан провайдерийн дамжуулан бүртгэнэ. Үнийн тооцоонд тохирох доменийн бүтээгийг ашиглана. |
| **Existing Domain** (Одоо байгаа домен) | Хэрэглэгч өөрийн аль хэдийн эзэмшсэн доменыг холбоно. Бүртгэлийн төлбөр байхгүй. Домен нь тэдний сайтай автоматаар холбогдоно. |

**Default mode** (Анхаарах горим) — Бүх гурван горим идэвхтэй үед аль таб эхлээд нээгдэнэ. Домен бүртгэлийг сонголтгүй байлгахын тулд **Subdomain**-ээр тохируулж болно, эсвэл худалдан авалтыг уриалсан байхаар **Register New Domain**-аар тохируулах боломжтой.

**Domain product** (Доменийн бүтээг) — Энэ талбарыг тодорхой доменийн бүтээгтэй холбохыг сонголт хийх боломжтой. Хэрэв тохируулбал, хэрэглэгч хайсан TLD-д үндэслэн addon нь тохирох бүтээгийг автоматаар сонгоно.

### Бүртгүүлэгчийн холбоо барих талбарууд

Хэрэглэгч **Register New Domain** (Шинэ домен бүртгэх) табыг сонговол, checkout form нь дараах бүртгүүлэгчийн холбооны талбаруудыг шууд нэмнэ:

- Эхний нэр / Аавгийн нэр
- И-мэйл хаяг
- Хаяг (1-р мөр, хот, мужийн/тэй байршил, postal code, улс)
- Утасны дугаар

Бүх бүртгэгчдэд шаардлагатай бөгөөд бүртгэл API дуудлага хийгдэхээс өмнө баталгаажуулдаг зүйлс юм. Утасны дугаар нь бүртгэгчдийн хүлээж буй `+CC.NNN` олон улсын форматад автоматаар форматлагдана.

### Автомат үүсгэсэн сайтын URL

Хэрэв хүн бүртгэл хийх эсвэл домен тохируулах үед, сайт URL талбарыг сонгосон доменээс автоматаар дүүргэнэ. Хэрэглэгч өөр URL талбар бөглөх шаардлагагүй.

### Хайлт хийх зан төлөв

- Доменийн боломжийг AJAX ашиглан хэрэглэгч бичих үед бодит цагийн хугацаанд шалгадаг.
- Хэрэв сонгосон домен байхгүй бол өөр TLD (Top-Level Domain) санал харуулна.
- Үнэ нь шууд авч, тодорхой харуулдаг (бүртгүүлэх үнэ, шинэчлэх үнэ, сонголттой WHOIS нууцлалтын төлбөр).
- Купон код нь бусад аливаа бүтээгдэхтэй ижил домен бүтээгдэхүүнд хэрэглэгддэг.

**Хайлтад хариу үйлдлийг тохируулах:**

```php
// Удаан холболттой үед API дуудлагыг бууруулахын тулд debounce delay-г (millisecond) нэмэгдүүлэх
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // анхны утга: 500
});
```

**Домен хайлт хийх маягт өөр талбарууд нэмэх:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Нэмэлт тэмдэглэл',
    ];
    return $fields;
});
```

## Хэрэглэгчийн DNS удирдлага

Хэрэглэгч нь **My Account** хуудсаас өөрийн бүртгэсэн доменүүдийн DNS бүртгэлийг, тухайн доменийн оролцоондээ удирдах боломжтой.

### Дэмжих бүртгэлийн төрлүүд

| Төрөл | Хэрэглэх зорилго |
|---|---|
| **A** | Хост нэрийг IPv4 хаягаар тохируулах |
| **AAAA** | Хост нэрийг IPv6 хаягаар тохируулах |
| **CNAME** | Өөр хост нэр рүү чиглэсэн аливаа алдааны (alias) холбоо үүсгэх |
| **MX** | Ме일 солилцох серверийн тохиргоо хийх |
| **TXT** | SPF, DMARC, баталгаажуулалт эсвэл бусад текст бүртгэлийг нэмэх |

### DNS удирдлагыг дэмждэг үйлчилгээч нар хэд вэ?

DNS-ийн удирдлага (рекордүүдийг нэмэх, өөрчлөх, устгах) нь **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** болон **Openprovider**-т боломжтой. **Hostinger** доменүүдийг Домен Борлуулагч (Domain Seller) ашиглан нэршил өөрчлөх боломжтой. Хүртээмэл доменүүдийн DNS-ийн бүртгэлийг үндсэн Hostinger домен-маппинг интеграци хийгээр удирддаг. Namecheap, GoDaddy, ба NameSilo доменүүд төлөв болон хугацаа дуусах мэдээллийг харуулдаг ч DNS-ийг бүртгэгчийн хяналтын самбарт шууд удирдах шаардлагатай.

### Анхны DNS бүртгэл (Default DNS records)

Домен бүртгэгдэх үед автоматаар ашиглагдах анхны DNS бүртгэлийг тохируулах боломжтой. **Settings › Domain Seller › Default DNS Records** руу орно.

Анхны бүртгэлийн утгууд хоёр төрлийн токенд тулгуурладаг:

| Токен | Орлуулдаг зүйл |
|---|---|
| `{DOMAIN}` | Бүртгэгдсэн домен нэр (жишээ нь `example.com`) |
| `{SITE_URL}` | Хэрэглэгчийн сайтын WordPress-ийн URL |

**Жишээ — apex домен болон www-г таны сервер IP руу чиглүүлэх:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: DNS-ийг харах ба өөрчлөх

Сүлжээний администраторууд домен бүрийн DNS бүртгэлийг **Network Admin › Ultimate Multisite › Domains** -ын домен өөрчлөх хуудаснаас хараад, өөрчилж болно.
