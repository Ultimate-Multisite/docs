# Домен бүтээгдэхүүн ба үнийн төлөвлөгөө

Домен бүтээгдэхүүн гэдэг нь таны үнэ, TLD (домен нэрлэх тэмдэг), WHOIS нууцлалт, боомт сонгох зэрэгт хэрхэн хяналтаа гаргадаг вэ гэдгийг тодорхойлдог. Нэг домен бүтээгдэхүүн нь стандарт Multisite Ultimate бүтээгдэхүүн бөгөөд **Domain Settings** (Доменин тохиргоо)-ийн нэмэлт табтай байдаг.

## Домен бүтээгдэхүүн үүсгэх {#creating-a-domain-product}

1. **Network Admin › Ultimate Multisite › Products** руу орно.
2. **Add New**-г дарна.
3. Бүтээгдэхүүний төрлийг **Domain** (Домен) гэж тохируулна.
4. **Domain Settings** таб (доор харагдана) -ийг тохируулна.
5. Хадгална.

Домен бүтээгдэхүүн нь бүтээгдэхүүний жагсаалтад улаан өнгөтэй **Domain** (Домен)-ын багцтай гарч ирнэ бөгөөд **Domain Products** таб ашиглан шүүлтүүр хийж болно.

## Domain settings таб {#domain-settings-tab}

### Provider (Боомт) {#provider}

Энэ бүтээгдэхүүнээр бүртгэлийг хэн зохицуулахыг сонгоно. Энэ нь Domain Seller-ийн тохиргооноос тогтоосон нийт **Default provider** (Анхаарах боомт)-той ижил байна.

### Supported TLDs (Далгаж буй TLD-үүд) {#supported-tlds}

Бусад бүтээгдэхүүнээр таарахгүй бүх TLD-д хамаарах **catch-all product** (бүхэлдээ хамардаг бүтээгдэхүүн)-ийг үүсгэхийн тулд хоосон үлдээ. Хэрвээ `.com, .net, .org` зэрэг TLD-үүдийн саналтай жагсаалтыг оруулбал, зөвхөн тэдгээрт хамаарах **TLD-specific product** (тухайн TLD-д зориулсан бүтээгдэхүүн)-ийг үүсгэж болно.

**Бүтээгдэхүүний таарах зарчим:** Хэрэв хэрэглэгч домен хайвал, addon хамгийн тодорхой таарч буй бүтээгдэхүүнийг сонгодог. Түүний TLD-ийн жагсаалтад `.com` байгаа бүтээгдэхүүн нь catch-all бүтээгдэхүүнийсээс илүү өндөр приоритеттэй байдаг. Хэрэв ямар ч TLD-specific бүтээгдэхүүн таарахгүй бол catch-all ашиглагдана. Ямар ч бүтээгдэхүүн байхгүй бол домен хайлт харагдахгүй.

### Markup type (Үнийн тооцооны төрөл) {#markup-type}

Таны өртөгөөс худалдааны үнэ хэрхэн тооцогдохыг гурван режим удирддаг:

| Mode (Режим) | Хэрхэн ажилладаг вэ |
|---|---|
| **Percentage** (Процент) | Худалдан авалтын өртөг дээр хувиар нэмнэ. Жишээлбэл, 10$ үнэтэй доменд 20% markup хийвэл 12$ болно. |
| **Fixed markup** (Тодорхой хувь) | Тодорхой мөнгөн дүнг нэмнэ. Жишээлбэл, 10$ доменд 5$ markup хийвэл 15$ болно. |
| **Fixed price** (Тодорхой үнэ) | Худалдан авалтын өртгийг бүрэн үл тоомсорлоно. Та оруулсан мөн дүнг үргэлж төлнө. |

### Introductory pricing (Оршил үнийн төлөвлөгөө) {#introductory-pricing}

Эхний жилээ хямд үнээр санал болгох боломжийг олгох. Ерөнхий **Сэргээх үнийн** (2-р жил ба түүнээс хойших) үнийн хажууд тусгай **Оролтын үнэ** (1-р жилийн үнэ) тохируулна. Хэрэглэгч худалдан авалт хийх явцад хоёр үнийг харж байгаа тул сэргээх үед юу хүлээж байгаагаа мэдэх боломжтой.

### WHOIS нууцлалт (privacy) {#whois-privacy}

Энэ нь энэхүү бүтээгдэхүүнээр бүртгэгдсэн доменүүдэд WHOIS нууцлал хамгаалалтыг олгох эсэхийг тохируулдаг.

| Тохиргоо | Үйлдэл |
|---|---|
| **Хийхгүй** (Disabled) | WHOIS нууцлалт хэрэгждэггүй, үргэлж идэвхгүй байдаг. |
| **Үргэлж багтсан** (Always Included) | Бүртгэлийн үед төлбөргүй автоматаар WHOIS нууцлалт идэвхтэй болдог. |
| **Хэрэглэгчийн сонголт** (Customer Choice) | Худалдан авалтын явцад тасхирч гардаг. **Нууцлалтын үнийг** жил бүрийн төлбөрт тохируулж болно, эсвэл үүнийг 0$ гэж үлдээх нь үнэгүй олгоход хүргэнэ. |

Namecheap-ийн хувьд WHOIS нууцлалт WhoisGuard (заавал үнэгүй) ашигладаг. OpenSRS-ийн хувьд тэдний OpenSRS нууцлалтын үйлчилгээг ашигладаг (бүхэл бүтэн худалдааны үед зардлыг байж болно).

## TLD импорт ба синхрончлол {#tld-import-and-sync}

Домен бүтээгдэхүүнүүд холбогдсон провайдероос авсан бодит цагийн wholesale үнийг харуулдаг. Энэ ажиллахын тулд TLD-удыг импорт хийх шаардлагатай.

- **Хүчээр синхрончлох** (Manual sync): Settings › Domain Seller › Sync TLDs (per provider)
- **Автомат синхрончлол** (Automatic sync): Бүх тохируулсан провайдерудын хувьд долоо хоног бүр сэргийлэсэн cron job-ээр ажилладаг.

Синхрончилсны дараа, ямар нэгэн домен бүтээгдэхүүний Domain Settings таб руу орж, TLD сонгогч (TLD picker) ашиглан одоо байгаа wholesale үнэтэй боломжит TLD-удыг харах боломжтой.

## Автомат сэргээлт (Auto-renewal) {#auto-renewal}

Домен сэргээлтийн хувьд хэрэглэгчийн гишүүнчлэлийн статустай холбоотой:

- Гишүүнчлэл сэргэх үед домен холбогдсон бол, доменийн сэргээлт автоматаар дараалалд ордог.
- Сэргээлтийн оролдлого нь хэрэглэгчийн идэвхтэй төлбөрийн хаалттай (payment gateway) ашиглагддаг.
- Амжилтгүй сэргээлтийн оролдлогуудыг экспоненциал буурах аргаар автоматаар дахин оролдоно.
- Амжилттай сэргээлт, амжилтгүй байдал болон хугацаа дуусах тухай мэдэгдлийг и-мэйлээр илгээдэг.

Доменийн амьдралын үе шатуудын (domain lifecycle events) и-мэйлийн загварын ID:

| Үйл явдал | Тэмдэглэгээ ID |
|---|---|
| Домен бүртгэл хийгдсэн | `domain_registered` |
| Домен шинэчлэгдсэн | `domain_renewed` |
| Шинэчлэл амжилтгүй болсон | `domain_renewal_failed` |
| Домен ойртож дуусах гэж байна | `domain_expiring_soon` |

## Администратор: Гаралттай домен бүртгэл хийх {#admin-manual-domain-registration}

Хэрэв хэрэглэгч шолле (checkout) хийхгүйгээр тэдний нэрээр домен бүртгэл хийхийг хүсвэл дараах алхмуудыг дагах хэрэгтэй:

1. **Network Admin › Ultimate Multisite › Register Domain** руу орно.
2. Хэрэглэгчийг сонгоод доменийн нэрийг оруулна.
3. Бүртгүүлэгчийн холбоо барих мэдээллийг (нэр, хаяг, утас) заавал бөглөнө.
4. **Register**-ийг дарна.

Доменийн бүртгэлийн бүртгэл үүснэ бөгөөд энэ нь тухайн хэрэглэгчийн аккаунтай холбогдоно.
