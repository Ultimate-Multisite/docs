# Домен Стейлер: Настройка и Конфигурация Провайдера

Додаток Domain Seller поставляется с пошаговым мастером настройки, который проведет вас через каждый необходимый шаг. Эта страница описывает процесс работы мастера и то, как настроить или перенастроить провайдеров после этого.

## Требования

- **Multisite Ultimate** v2.4.12 или выше, активированный по сети
- **PHP** 7.4+
- API учетные данные для хотя бы одного поддерживаемого регистратора

## Мастер настройки при первом запуске

Мастер настройки автоматически запускается в первый раз, когда вы активируете плагин по сети. Он также доступен в любое время через **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Шаг 1 — Выберите провайдера

Выберите регистратора, с которым вы хотите подключиться. Поддерживаемые варианты:

| Провайдер | Управление DNS | Конфиденциальность WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Нет | Да (WhoisGuard, бесплатно) |
| HostAfrica | Да | Да (Защита ID) |
| Openprovider | Да | Да |
| Hostinger | Через основное сопоставление домена Hostinger для хостинговых доменов | Да |
| GoDaddy | Нет | Нет |
| ResellerClub | Да | Нет |
| NameSilo | Нет | Нет |
| Enom | Да | Нет |

### Шаг 2 — Введите учетные данные

У каждого провайдера разные поля для ввода данных:

**OpenSRS** — Имя пользователя и приватный ключ (из панели управления реселлером OpenSRS)

**Namecheap** — Имя пользователя и API ключ (из Account › Tools › API Access)

**HostAfrica** — Конечная точка API реселлера доменов и учетные данные от модуля реселлера HostAfrica. Отдельная песочница пока не документирована; тестируйте безопасными операциями только для чтения перед запуском реальных регистраций.

**Openprovider** — Имя пользователя и пароль с включенным API-доступом. Опциональный режим песочницы использует sandbox API Openprovider, а опциональное имя клиента по умолчанию можно использовать повторно для регистраций.

**Hostinger** — Үндсэн Hostinger integration-аса ирж буй хослуулсан Hostinger hPanel API токен. Энэхүү токен нь үндсэн домен зураглалт болон Домен Борлуулагчийн бүртгэлийн үйлдлүүдэд ашиглагддаг.

**GoDaddy** — API key ба нууц (developer.godaddy.com-оос)

**ResellerClub** — Reseller ID ба API key (ResellerClub удирдлагын панельээс)

**NameSilo** — API key (namesilo.com › Account › API Manager-аас)

**Enom** — Акаунт ID ба API токен

Хэрэв боломжтой бол **Sandbox mode**-ыг шалгана уу. Энэ нь үйлчилгээ үзүүлэгчийн туршилтын орчинд ашиглахаас өмнө тест хийхэд тусална.

### 3-р алхам — холболтыг шалгах

**Test Connection**-ыг дарна уу. Энэ wizard нь зөв баримт бичлүүдийг баталгаажуулж, холболтын чадварыг шалгахаар хурдан API дуудлага хийнэ. Үргэлжлэхийн өмнө ямар нэгэн баримт бичлийн асуудал байвал засах хэрэгтэй.

### 4-р алхам — TLD-үүдийг импортлох

**Import TLDs**-ийг дарж, холбогдсон үйлчилгээгээс боломжит бүх TLD (Top-Level Domain) болон эрдэнэ үнийн дугаарлалтыг татаж авна. Энэ нь домен бүтээгдэхүүнд ашиглагдах TLD-ийн жагсаалтыг дүүргэнэ. Хэрэв үйлчилгээ үзүүлэгчийн TLD-ийн каталог том бол импорт хийхэд 30–60 секунд шаардаж болно.

TLD-үүд өдөрт нэг удаа хуваарьт ажилладаг cron job-ээр автоматаар дахин синхрожигддаг.

### 5-р алхам — домен бүтээгдэхүүн үүсгэх

Wizard нь 10% markup (захиалгын хувь) -тай анхны бүх төрлийн домен бүтээгдэхүүнийг үүсгэнэ. Та энэ бүтээгдэхүүнийг шууд өөрчилж болно, эсвэл **Ultimate Multisite › Products**-ийн доор гараар бүтээгдэхүүн үүсгэхээ алгасаж болно.

Бүх бүтээгдэхүүний тохиргооны тухай бүрэн зааврыг [Domain Products and Pricing](./domain-products)-д үзнэ үү.

## Үйлчилгээ үзүүлэгчийг дахин тохируулах

**Network Admin › Ultimate Multisite › Settings › Domain Seller** руу орно (эсвэл plugin list-ээс **Settings**-ийг дарна уу).

Тохиргооны хуудас дараах зүйлсийг агуулдаг:

- **Домен худалдаа авах** — энэ функцийг асаах эсвэл унтраах тохиргоо хийнэ.
- **Анхны ханган татвар (Default provider)** — домен хайлт хийх болон шинэ бүтээгдэхүүний хувьд ашиглагдах татварыг заадаг.
- **Хайлтад хамгийн их TLD** — хэрэв хэрэглэгч хайлт хийх үед хэдэн TLD-ийг шалгахыг тодорхойлно; өндөр утгууд илүү олон сонголтыг харуулдаг боловч арай удаан байдаг.
- **Хүртээмэлийн кэшийн хугацаа (Availability cache duration)** — хүртээмж ба үнийн үр дүнг хэр их хугацаанд кэшлэхийг тодорхойлно; бага утгууд илүү нарийвчлалтай байдаг боловч API дуудлага нэмэгддэг.
- **Домен бүтээгдэхүүний удирдлага (Manage domain products)** — Бүтээгдэхүүний жагсаалтад шууд хүрэх хурдан холбоо.
- **Татваруудыг тохируулах (Configure providers)** — Татвар нэмэх эсвэл дахин тохируулахын тулд Integration Wizard-ийг нээнэ.

### Хоёр дахь татварыг нэмэх

**Configure providers**-д дарж шинэ регистр дээрх татвараар wizard-ыг дахин ажиллуулна уу. Та зэрэг олон татварыг тохируулах боломжтой. Домен бүтээгдэхүүн бүрийг тусгай татварт хамаарах байдлаар зааж өгөх эсвэл анхны утгаар үлдээж болно.

### TLD-ийг гараар синхрох

Тохиргооны хуудсан дээр, тохируулсан аль нэг татварын хажууд **Sync TLDs**-д дарж хамгийн сүүлийн үеийн үнийн мэдээллийг татаж авна. Энэ нь татвар wholesale үнийг шинэчилсэн эсвэл шинэ TLD-уудыг нэмсэн дараа ашигтай байдаг.

## Лог (Logs)

Татвар бүр өөрийн логич шугаманд бичиж байна. Логуудыг **Network Admin › Ultimate Multisite › Logs** дотор харах боломжтой:

| Лог канал | Содержимое |
|---|---|
| `domain-seller-registration` | Все попытки регистрации (успешные и неудачные) |
| `domain-seller-renewal` | Результаты задач по продлению домена |
| `domain-seller-opensrs` | Сырая активность API OpenSRS |
| `domain-seller-namecheap` | Сырая активность API Namecheap |
| `domain-seller-hostafrica` | Сырая активность API HostAfrica |
| `domain-seller-openprovider` | Сырая активность API Openprovider |
| `domain-seller-hostinger` | Сырая активность API Hostinger |
| `domain-seller-godaddy` | Сырая активность API GoDaddy |
| `domain-seller-resellerclub` | Сырая активность API ResellerClub |
| `domain-seller-namesilo` | Сырая активность API NameSilo |
| `domain-seller-enom` | Сырая активность API Enom |

## Примечания о возможностях провайдеров

Не все регистраторы предоставляют одинаковые операции через свой API. Дополнение (addon) показывает неподдерживаемые операции с понятными ошибками для администратора, вместо того чтобы просто молча завершаться неудачей.

- **HostAfrica** поддерживает самый широкий рабочий процесс для реселлеров в реальном времени, включая поиск, синхронизацию TLD/цены, регистрацию, продление, перенос, обновление имен серверов, DNS-записи, коды EPP, блокировку регистратора и защиту ID.
- **Openprovider** поддерживает синхронизацию TLD по цене реселлера, регистрацию, продление, переносы, обновление имен серверов, DNS-зоны, коды EPP, блокировку регистратора и конфиденциальность WHOIS. Он аутентифицируется с помощью краткосрочного токена bearer, который дополнение автоматически обновляет.
- **Hostinger** поддерживает поиск доступности, регистрацию, поиск портфолио, обновление имен серверов, блокировку регистратора и конфиденциальность WHOIS через общий токен API hPanel. Публичный API доменов Hostinger не предоставляет информацию о ценах для реселлеров/оптовых покупателей, входящие переводы, явное продление или получение кодов EPP; продления происходят только автоматически.
