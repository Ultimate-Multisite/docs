# Closte Интеграци {#closte-integration}

## Ерөнхий өгүүлэл {#overview}
Closte бол Google Cloud дээр баригдсан WordPress хостинг платформ юм. Энэхүү интеграци нь Ultimate Multisite болон Closte-ийн хоорондох талбайн синхронизулалт, SSL сертификатын удирдлагыг автоматаар хийгдэх боломжийг олгодог.

## Онцлог {#features}
- Талбайн автоматаар синхронируулдаг
- SSL сертификатыг удирдах
- Wildcard талбаны домен дэмждэг
- Closte дээр ажиллаж байгаа бол ямар ч тохиргоо хийх шаардлагагүй

## Шаардлага {#requirements}
Хэрэв та Closte дээр хостинг хийж байгаа бол `wp-config.php` файлд дараах тогтмол (constant)-ыг тодорхойлох хэрэгтэй:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Энэ тогтмолыг Closte дээр хостинг хийж байгаа бол ихэвчлэн аль хэдийн тодорхойлогдсон байдаг.

## Тохируулах заавар {#setup-instructions}

### 1. Таны Closte API Key-г шалгах {#1-verify-your-closte-api-key}
Хэрэв та Closte дээр хостинг хийж байгаа бол `CLOSTE_CLIENT_API_KEY` тогтмол нь `wp-config.php` файлд аль хэдийн тодорхойлогдсон байх ёстой. Үүнийг `wp-config.php` файлыг шалгаснаар баталгаажуулах боломжтой.

### 2. Интеграцийг идэвшүүлэх {#2-enable-the-integration}
1. WordPress администратор руу орж, Ultimate Multisite > Settings руу очино.
2. "Domain Mapping" (Талбайн тохиргоо) таб руу шилжинэ.
3. "Host Integrations" (Хост интеграциуд) хэсэг рүү урагш скролл хийгээд.
4. Closte-ийн интеграцийг идэвшүүлнэ.
5. "Save Changes" (Өөрчлөлтийг хадгалах) товчийг дарна.

## Энэ яаж ажилладаг вэ? {#how-it-works}

Ultimate Multisite дээр талбайн тохиргоо хийгдэх үед:

1. Интеграци нь Closte-ийн API руу хүсэлт илгээж, таны аппликейшн дотор талбаны нэрийг нэмнэ.
2. Closte SSL сертификатын асуудлыг автоматаар шийддэг.
3. Талбайн тохиргоог устгахад, интеграци нь тухайн талбаны нэрийг Closte-ээс хасна.

Мөн энэ интеграци Ultimate Multisite дэх DNS шалгах хугацааны тохиргоотой хамт ажилладаг бөгөөд систем ямар давтамгаар DNS-ийн тархалт болон SSL сертификат олголтыг шалгахыг тохируулах боломжийг олгодог.

## Талбайн бүртгэлийн бичилт (Domain Record Creation) {#domain-record-creation}

Энэхүү холболт нь сайт үүсгэх эсвэл дуудах үед домен рүү бүртгэлийг автоматаар үүсгэж өгдөг. Энэ нь Closte-ийн интеграцийн хувьд ялангуяа чухал бөгөөд, домен рүү бүртгэлийн бичлэг үүсэхэд Closte API домены болон SSL сертификатыг үүсгэх үйлдлийг идэвхжүүлдэг.

## Асуудал шийдэх {#troubleshooting}

### API холболтын асуудлууд {#api-connection-issues}
- Таны Closte API key зөв байгаа эсэхийг шалгана уу.
- Таны Closte аккаунтад шаардлагатай эрхүүд байгаа эсэхийг баталгаажуулна уу.

### SSL сертификатын асуудлууд {#ssl-certificate-issues}
- Closte нь SSL сертификат олгоход зарим цаг хугацаа шаардах боломжтой (ихэвчлэн 5-10 минут).
- Таны доменүүд Closte серверийн IP хаяг руу зөв чиглэгдэж байгаа эсэхийг шалгана уу.
- Доменийн DNS бичлэгийг шалгаж, зөв тохирсон эсэхийг баталгаажуулна уу.

### Домен нэмэгдээгүй байх {#domain-not-added}
- Ultimate Multisite-ийн логуудаас ямар нэгэн алдаа мессеж байгаа эсэхийг шалгана уу.
- Таны домен Closte-т аль хэдийн нэмэгдсэн эсэхийг баталгаажуулна уу.
- Доменийн DNS бичлэг зөв тохирсон эсэхийг шалгана уу.

### DNS шалгах хугацаа {#dns-check-interval}
- Хэрэв SSL сертификат олгоход хэт их цаг хугацаа шаардвал, Domain Mapping тохиргооны дотор DNS шалгах хугацааг өөрчилж болно.
- Анхны хугацаа нь 300 секунд (5 минут) байна, гэхдээ туршилт хийх үед илүү хурдан шалгахын тулд та үүнийг 10 секунд хүртэл тохируулах боломжтой.
