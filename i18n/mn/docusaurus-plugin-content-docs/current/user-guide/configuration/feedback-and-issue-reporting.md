# Хэрэглэгчийн санал бодол ба асуудал мэдэгдэх {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 нь төгсгөлийн хэрэглэгчдийн хариултыг үр дүнгүй гэж тэмдэглэх, чат интерфейсээс шууд асуудлыг мэдээлэх боломжийг олгодог дотоод санал бодол ба асуудал мэдэгдэх системийг нэвтрүүлсэн. Энэ систем нь зөвшөөрөл удирдлага, автоматаар явагдах мэдэгдлийн команд, мөн ард талын AI-аас туслах ангилалтыг багтаасан байдаг.

## Дуугүй гэсэн товч (Thumbs-Down Button) {#thumbs-down-button}

AI ассистентээс илгээгдсэн бүх мессеж дээр **дуугүй гэсэн** (👎) товч гарч ирнэ. Хэрэглэгч үүнийг дарвал хариултыг туслахгүй эсвэл буруу гэж тэмдэглэж болно.

- Товч нь ассистент мессежийн дагуу хөдөлгөсөн үед гарч ирнэ.
- Үүнийг дарвал **Санал зөвшөөрлийн модаль** (Feedback Consent Modal) нээгдэнэ.
- Санал бодол нь тухайн ярианы мөр ба тодорхой мессежтэй холбоотой байдаг.

## Санал зөвшөөрлийн модаль (Feedback Consent Modal) {#feedback-consent-modal}

Хэрэглэгч дуугүй гэсэн товчийг дарсан үед, ямар ч өгөгдөл илгээхээс өмнө зөвшөөрлийн модаль гарч ирнэ. Энэхүү модаль:

- Ямар мэдээлэл цуглуулж байгааг (ярианы хэсэг, браузерын контекст) тайлбарладаг.
- Хэрэглэгч энэхүү өгөгдлийг хуваалцах зөвшөөрөхөө баталгаажуулахыг асууна.
- Асуудал юу болсныг тодорхойлохын тулд хэрэглэгчийн бичих талбай (free-text field) өгдөг.
- Идэвхгүй байдлаар хаах **Унтраах** (Cancel) сонголтыг санал өгөхгүйгээр гарах боломжийг олгодог.

Хэрэглэгч тодорхой баталгаажуулбал, ямар ч санал бодол бүртгэгддэггүй.

## Автомат мэдэгдлийн баннер (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Яриа дууссаны дараа ассистент нь **автомат мэдэгдлийн баннер** - энэ нь тухайн сесс шилдэг байсан эсэхийг асуусан, санамсаргүй мессеж юм.

Энэхүү баннер нь ярианы урт ба үр дүнгийн тодорхойлолт (heuristics) дээр үндэслэн автоматаар гарч ирдэг. Энэ нь дуугүй гэсэн товчны хамт адил санал өгөх замыг харуулдаг. Хэрэглэгч санал өгөхгүйгээр баннерыг хааж болно.

## /report-issue Слэш команд (Slash Command) {#report-issue-slash-command}

Хэрэглэгчид чат оролтод `/report-issue` гэж бичих замаар шууд санал бодол өгөх замыг эхлүүлж болно. Энэ команд:

- Мэдэгдсэн хүсэлтийн зөвшөөрөл (Feedback Consent Modal)-ийг шууд нээх.
- Үндсэн тайлбарын талбарыг одоогийн ярианы контексттэй мэдээллээр урьсан байдлаар дүүргэх.
- Хэрэглэгч илгээхээс өмнө нэмэлт дэлгэрэнгүй мэдээллийг нэмж болно.

`/report-issue` командыг бүх чат модонд (inline, floating widget, full-screen) ашиглаж болно.

## AI-Assisted Triage (AI-аар туслах ангилалт) {#ai-assisted-triage}

Илгээсэн санал нь AI ангилалтын давхаргын (triage layer) дагуу явна бөгөөд энэ нь:

- Саналтай асуудлыг ангилна (баримт буруу, туслахгүй хариулт, зохистой биш агуулга гэх мэт).
- Хамаарах ярианы метаданс (metadata) -ыг холбоно.
- Ангилалтын хураангуйг тохируулсан **Feedback Endpoint** руу илгээхэд (Settings > Advanced-д харна уу).

Энэ нь хамгийн чухал асуудлуудыг эхлээд гаргаж ирүүлэх замаар гараар шалгах хугацааг багасгадаг.

## Settings > Advanced {#settings-advanced} (Тохиргоо > Гүнзгийрүүлсэн) {#settings--advanced}

Feedback backend-ийг ажиллуулахын тулд **Gratis AI Agent → Settings → Advanced** доторх дараах талбаруудыг тохируулна уу:

| Талбар | Тодорхойлолт |
|---|---|
| **Feedback Endpoint URL** | Feedback payload (JSON) -ийг хүлээн авдаг POST хүсэлтүүдийн URL. |
| **Feedback API Key** | Санал илгээхэд баталгаажуулахын тулд `Authorization` header-д илгээгдэх Bearer token. |

Хэрэв хоёр талбарыг хоосон үлдээвэл feedback цуглуулалт идэвхгүй болно. Гэхдээ хэрэглэгчдэд тусгайлан зориулсан feedback товчлуур нь харагдаж байх бөгөөд илгээсэн санал руу дамжуулнагүй.

:::tip
Feedback endpoint нь `message_id`, `conversation_id`, `feedback_text`, болон `triage_category` талбаруудыг агуулсан JSON body-г хүлээн авах ёстой. Хүлээгдэж буй схем (schema)-ын талаар endpoint-ийн өгөгдсөн баримт бичгийг үзнэ үү.
:::
