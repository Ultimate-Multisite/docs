# Tutor Multisite Compatibility {#tutor-multisite-compatibility}

Ett WordPress-plugin som lägger till kompatibilitetsfixar för Tutor LMS i multisite-miljöer.

## Beskrivning {#description}

Detta plugin tillhandahåller nödvändiga kompatibilitetsfixar för att köra Tutor LMS på WordPress multisite-nätverk. Det hanterar användarinskrivningar, handledarhantering, kursbegränsningar och annan funktionalitet som är specifik för multisite.

## Funktioner {#features}

- **Hantera datatabeller**: Upptäcker och skapar automatiskt saknade Tutor LMS-tabeller som misslyckas på grund av multisite-främmande nyckelbegränsningar
- **Multisite-användarinskrivning**: Anpassade AJAX-hanterare för att hantera användarinskrivningar över nätverksplatser
- **Hantera handledarroller**: Automatisk tilldelning och hantering av handledarroller i multisite-miljöer
- **Kursbegränsningar**: Integration med Ultimate Multisite för att hantera gränser för kurs- och funnelskapande baserat på platsplaner
- **Anpassade adminvyer**: Multisite-medveten lista och gränssnitt för handledare
- **Adminverktyg**: Gränssnitt för databasadministration för nätverks- och platsadministratörer

## Krav {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Tutor LMS plugin (tvingas automatiskt via `Requires Plugins`-rubriken)
- WordPress Multisite-installation
- Ultimate Multisite plugin (för funktionaliteten med kursbegränsningar)

## Installation {#installation}

1. Ladda upp pluginfilerna till `/wp-content/plugins/tutor-multisite-compatibility/`
2. Se till att Tutor LMS är installerat och aktiverat
3. Aktivera pluginet via menyn 'Plugins' i WordPress
4. Pluginet integreras automatiskt med Tutor LMS

## Databasadministration {#database-management}

### Problemet {#the-problem}

I WordPress multisite-miljöer misslyckas skapandet av Tutor LMS-tabeller för tabeller med främmande nyckelbegränsningar till användartabellen. Detta händer eftersom:

- Plats-specifika tabeller använder prefix som `wp_2_tutor_carts`
- Främmande nycklar refererar till `wp_2_users` (vilket inte existerar)
- Den faktiska användartabellen är `wp_users` (delad över hela nätverket)

### Lösningen {#the-solution}

Detta plugin gör automatiskt följande:

1. **Upptäcker saknade tabeller**: Kontrollerar efter tabeller som misslyckades med att skapas på grund av främmande nyckelbegränsningar
2. **Skapar tabeller med korrekta referenser**: Använder det korrekta namnet på användartabellen (`wp_users` i multisite)
3. **Hantera flera scenarier**:
   - Pluginaktivering/reaktivering
   - Skapande av ny plats i nätverket
   - Manuell tabellkontroll via admin-gränssnittet

### Berörda tabeller {#affected-tables}

- `tutor_coupon_usages` - Innehåller främmande nyckel till användartabellen
- `tutor_carts` - Innehåller främmande nyckel till användartabellen
- `tutor_notification_preferences` - TutorPro-tabell med främmande nyckel till användartabellen

### Admin-gränssnitt {#admin-interface}

Få tillgång till verktygen för databasadministration på:

- **Nätverksadmin**: Inställningar → Tutor Multisite DB
- **Platsadmin**: Verktyg → Tutor Multisite DB

Gränssnittet låter dig:
- Visa saknade tabeller över alla platser
- Manuell trigga tabellskapande
- Kontrollera databaskonfiguration
- Visa detaljerad statusinformation
