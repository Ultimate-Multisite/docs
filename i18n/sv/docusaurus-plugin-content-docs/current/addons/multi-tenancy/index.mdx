# Multi-Tenancy

Distribuera underwebbplatser över servrar och databaser för prestanda och säkerhet

## Höjdpunkter i Version 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 lägger till den suveräna klientgrunden för isoleringsarbetsflöden i produktion:

- Tillståndslös automatisk inloggning för klienter med kortlivade, uppspelningsskyddade SSO-besök.
- Databas- och filsystemsisolering per underwebbplats för suveräna klienter.
- Hostinger multi-tenancy-kapacitet för värdmedveten klientprovisionering.
- Verifieringsgrindar för suverän migrering för kontroller av äldre data och asynkron push-bearbetning.
- Konfigurerbar databasvärdsbindning på samma maskin för Bedrock, FrankenPHP och localhost-installationer.
- Förbättringar av klientbootstrap, databasskrivare, schemabootstrap och användarprovisionering.
- Säkrare suverän nedmontering som rensar klientuppgifter.

Se användarguiderna för [arbetsflöde för suverän migrering](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy-isolering](/user-guide/configuration/multi-tenancy-isolation) och [åtgärder för suveräna klienter](/user-guide/administration/sovereign-tenant-operations).

## Krav {#requirements}

- WordPress 5.3 eller högre
- PHP 7.4 eller högre
- Multisite Ultimate-tillägg (aktivt)

## Installation {#installation}

1. Ladda upp tilläggsfilerna till din `/wp-content/plugins/`-katalog
2. Aktivera tillägget via menyn 'Tillägg' i WordPress
3. Konfigurera tilläggsinställningarna i nätverksadministrationen

### Testning {#testing}

Tillägget inkluderar ett omfattande testramverk:

- **Enhetstester**: PHPUnit-baserade tester med WordPress-testsvit
- **Bastestklass**: Tillhandahåller hjälpmetoder för vanliga testscenarier
- **Testtäckning**: Konfigurerad för att spåra kodtäckning
- **Multisite-testning**: Tester körs i multisite-miljö

### Kodstandarder {#code-standards}

- **WordPress Coding Standards**: Upprätthålls via PHPCS
- **PHP 7.4+-kompatibilitet**: Moderna PHP-funktioner stöds

- **Kodmodernisering**: Rector-regler för PHP-uppgraderingar

### Byggsystem {#build-system}

- **Bearbetning av tillgångar**: Minifiering för CSS/JS-filer
- **Översättning**: Generering av POT-fil

- **Utvecklingsläge**: Ominifierade tillgångar för felsökning

## Konfiguration {#configuration}

Tillägget stöder olika konfigurationsalternativ som är åtkomliga via nätverksadministrationsgränssnittet.

### Konfiguration av suverän klient {#sovereign-tenant-configuration}

För suveräna klienter, konfigurera klientdatabasen, filsystemets rot, databasvärdsbindning och värdkapacitet innan trafik dirigeras till klienten. Kör migreringsverifiering efter varje ändring som påverkar databasuppgifter, värdbindning, dirigering eller användarprovisionering.

Använd `localhost` endast när klientens körningsmiljö och databasbehörigheter förväntar sig lokal socket eller åtkomst på samma maskin. Använd `127.0.0.1` eller ett tjänstvärdnamn när det är behörighetsvärden som konfigurerats av din leverantör.

### Hostinger-kapacitet {#hostinger-capability}

Hostinger multi-tenancy-kapaciteten fungerar med den delade Hostinger-integrationstokenen. Bekräfta att domänen finns i hPanel, att DNS pekar till Hostinger-kontot och att klientens databasinställningar matchar hPanel före verifiering.

## Hooks och filter {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - Utlöses efter initiering av tillägget
- `ultimate-multisite-addon-template_loaded` - Utlöses efter att alla tilläggsfiler har laddats

### Filter {#filters}

- `ultimate-multisite-addon-template_settings` - Ändra tilläggsinställningar
- `ultimate-multisite-addon-template_enabled` - Åsidosätt aktivera/inaktivera-status för tillägget
