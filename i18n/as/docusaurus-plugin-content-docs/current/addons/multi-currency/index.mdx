# বহু-মুদ্রা (Multi-Currency)

Ultimate Multisite: Multi-Currency ব্যৱস্থাই আপোনাৰ নেটৱৰ্কক বিভিন্ন মুদ্রাত পৰিশোধ গ্ৰহ কৰাত সহায় কৰে। ই স্বয়ংক্রিয়ভাৱে লাইভ বা হাতেৰে দিয়া বিনিময় হাৰ ব্যৱহাৰ কৰি মূল্যসমূহ ৰূপান্তৰ কৰে, আপোনাক যিকোনো মুদ্রাৰ বাবে প্ৰোডাক্ট-ভিত্তিক সঠিক মূল্য নিৰ্ধাৰণ কৰিবলৈ দিয়ে, আৰু ভৌগোলিক স্থান (geolocation) অনুসৰি প্ৰতিজন ভিজিটৰৰ পছন্দৰ মুদ্রা চিনাক্ত কৰে — সকলো একমাত্ৰ admin dashboardৰ পৰা ব্যৱস্থাপনা কৰিব পাৰে।

## মূল বৈশিষ্ট্যসমূহ (Key Features)

- **স্বয়ংক্রিয় বিনিময় হাৰ ৰূপান্তৰণ (Automatic exchange rate conversion)** — আপোনাৰ মূল মুদ্রাৰ পৰা মূল্যসমূহ আপুনি বাছনি কৰা প্ৰোভাইডাৰৰ পৰা লোৱা হাৰ ব্যৱহাৰ কৰি ৰূপান্তৰ কৰা হয়।
- **প্ৰোডাক্ট-ভিত্তিক মূল্যৰ ওপৰত দিয়া বিকল্প (Per-product pricing overrides)** — স্বয়ংক্রিয় ৰূপান্তৰক এৰাই চলি যিকোনো প্ৰোডাক্টত কোনো বিশেষ মুদ্রাৰ বাবে সঠিক মূল্য নিৰ্ধাৰণ কৰিব পাৰে।
- **মুদ্রা নির্বাচক নাম লেখোৱা ফিল্ড (Currency selector signup field)** — এই বৈশিষ্ট্যটোৱে কাস্টমৰক চেকআউট/নাম লেখোৱাৰ সময়ত তেওঁলোকৰ পছন্দৰ মুদ্রা বাছি ল'বলৈ দিয়ে।
- **বহু বিনিময় হাৰ প্ৰোভাইডাৰ (Multiple exchange rate providers)** — ECB, Frankfurter, Stripe, বা সম্পূৰ্ণ হাতেৰে দিয়া হাৰ ব্যৱহাৰ কৰিব পাৰি।
- **ভৌগোলিক স্থান-ভিত্তিক মুদ্রা চিনাক্তকৰণ (Geolocation-based currency detection)** — ভিজিটৰৰ দেশৰ সৈতে মিল থকা মুদ্রাটো স্বয়ংক্রিয়ভাৱে আগতে বাছনি কৰে।
- **admin dashboard** — এক স্থানৰ পৰা কাৰ্য্যশীল মুদ্রা, বিনিময় হাৰ আৰু প্ৰোভাইডাৰৰ সেটিংছ ব্যৱস্থাপনা কৰিব পাৰে।

## প্ৰয়োজনীয়তা (Requirements)

- Ultimate Multisite 2.0.0 বা তাতকৈ অধিক
- WordPress Multisite installaion
- PHP 7.4 বা তাতকৈ অধিক

## স্থাপন (Installation)

1. প্লাগইন ফাইলসমূহক `/wp-content/plugins/ultimate-multisite-multi-currency` লৈ আপলোড কৰক।
2. WordPress-ৰ **Plugins** স্ক্ৰীণৰ মাজেৰে প্লাগইনটো অ্যাক্টিভেট কৰক (Network Activate)।
3. admin dashboard খুলিবলৈ **Ultimate Multisite → Multi-Currency** লৈ যাওক।
4. আপুনি গ্ৰহ কৰিব বিচৰা মুদ্রা যোগ কৰক আৰু আপোনাৰ বিনিময় হাৰ প্ৰোভাইডাৰ কনফিগাৰ কৰক।

## কনফিগাৰেশ্যন (Configuration)

### Admin Dashboard

Multi-Currency admin dashboard (**Ultimate Multisite → Multi-Currency**) হৈছে এই কামসমূহৰ কেন্দ্ৰীয় স্থান:

- মুদ্রা যোগ কৰা, সক্ষম কৰা বা নিষ্ক্ৰম কৰা।
- এটা বিনিময় হাৰ প্ৰোভাইডাৰ বাছনি কৰা আৰু চাহিদা অনুসৰি হাৰ Refresh কৰা।
- সকলো কাৰ্য্যশীল মুদ্রাৰ বৰ্তমান হাৰ পৰীক্ষা কৰা।
- স্বয়ংক্রিয় প্ৰোভাইডাৰ উপযুক্ত নহ'লে হাতেৰে হাৰ নিৰ্ধাৰণ কৰা।

### Exchange Rate Providers

| Provider | Description |
|---|---|
| **ECB** | European Central Bank — EUR-ভিত্তিক নেটৱৰ্কৰ বাবে বিনামূলীয়া দৈনিক হাৰ। |
| **Frankfurter** | Open-source ECB mirror API — বিনামূলীয়া, কোনো API keyৰ প্ৰয়োজন নাই। |
| **Stripe** | Stripe-ৰ লাইভ FX হাৰ ব্যৱহাৰ কৰে — এটা কাৰ্য্যশীল Stripe integrationৰ প্ৰয়োজন। |
| **Manual** | আপুনি নিজে হাৰ প্ৰৱেশ কৰে; বাহ্যিক API কলৰ প্ৰয়োজন নহয়। |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** ত আপোনাৰ প্ৰোভাইডাৰ বাছনি কৰক। হাৰসমূহ যিকোনো সময়ত হাতেৰে Refresh কৰিব পাৰি বা স্বয়ংক্রিয়ভাৱে আপডেট কৰিবলৈ সময়সূচী কৰিব পাৰি।

### Per-Product Pricing Overrides

যেতিয়া স্বয়ংক্রিয় ৰূপান্তৰ যথেষ্ট সঠিক নহয়, তেতিয়া আপুনি যিকোনো প্ৰোডাক্টত কোনো বিশেষ মুদ্রাৰ বাবে এটা সঠিক মূল্য নিৰ্ধাৰণ কৰিব পাৰে:

1. প্ৰোডাক্ট এডিটেৰ খুলক (**Ultimate Multisite → Products → Edit**)।
2. **Currency Pricing** অংশলৈ নামি যাওক।
3. আপুনি কিমানটা মুদ্রাৰ বাবে বিকল্প দিব বিচাৰে, তাৰ বাবে সঠিক মূল্য প্ৰৱেশ কৰক।
4. প্ৰোডাক্টটো সেভ কৰক।

যি প্ৰোডাক্টত কোনো বিকল্প দিয়া নহয়, সেইবোৰ মূল মুদ্রাৰ পৰা ৰূপান্তৰ কৰা মূল্য ব্যৱহাৰ কৰে।

### Currency Selector Signup Field

এই মুদ্রা নির্বাচক ফিল্ডটো আপোনাৰ নাম লেখোৱা/চেকআউট প্ৰবাহত যোগ কৰিব পাৰি, যাতে কাস্টমৰসকলে নাম লেখোৱাৰ সময়ত তেওঁলোকৰ পছন্দৰ মুদ্রা বাছি ল'ব পাৰে:

1. **Ultimate Multisite → Multi-Currency → Settings** লৈ যাওক।
2. **Show currency selector on signup** সক্ষম কৰক।
3. ঐচ্ছিকভাৱে এটা ডিফল্ট মুদ্রা নিৰ্ধাৰণ কৰক বা ভৌগোলিক স্থানৰ দ্বাৰা আগতে বাছনি কৰা মূল্যটো নিৰ্ধাৰণ কৰিবলৈ দিয়ক।

বাছাই কৰা মুদ্রাটো subscription লগত সংৰক্ষিত হয় আৰু তাৰ পিছৰ সকলো বিলিংৰ বাবে ব্যৱহাৰ কৰা হয়।

### Geolocation-Based Currency Detection

যেতিয়া ভৌগোলিক স্থান চিনাক্তকৰণ সক্ষম কৰা হয়, তেতিয়া এই addons-এ ভিজিটৰৰ দেশক এটা মুদ্রাৰ সৈতে ম্যাপ কৰে আৰু মুদ্রা নির্বাচক ফিল্ডত ইয়াক আগতে বাছনি কৰে। সক্ষম কৰিবলৈ:

1. **Ultimate Multisite → Multi-Currency → Settings** লৈ যাওক।
2. **Detect currency from visitor location** সক্ষম কৰক।
3. **Geolocation Mapping** টেবুলত দেশসমূহক মুদ্রাৰ সৈতে ম্যাপ কৰক।

যদি কোনো ভিজিটৰৰ দেশৰ বাবে কোনো ম্যপিং নাই, তেন্তে নেটৱৰ্কৰ মূল মুদ্রাটো বিকল্প হিচাপে ব্যৱহাৰ কৰা হয়।

## সঘনে কৰা প্ৰশ্নসমূহ (Frequently Asked Questions)

### যদি বিনিময় হাৰ প্ৰোভাইডাৰ উপলব্ধ নহয় তেন্তে কি হ'ব?

এই addons-এ শেষ সফলভাৱে লোৱা হাৰসমূহলৈ উভতি যায়। যদি কেতিয়াও কোনো হাৰ লোৱা নহ'ল, তেন্তে ই আপুনি কনফিগাৰ কৰা হাতেৰে দিয়া হাৰলৈ উভতি যায়। এটা সেফটি নেট হিচাপে হাতেৰে দিয়া হাৰ কনফিগাৰ কৰাটো বাঞ্ছনীয়।

### মই কি বিভিন্ন মুদ্রাৰ বাবে বিভিন্ন প্ৰোভাইডাৰ ব্যৱহাৰ কৰিব পাৰোঁ?

না — সকলো মুদ্রাৰ বাবে এটা প্ৰোভাইডাৰ একে সময়ত কাৰ্য্যশীল থাকে। যদি আপোনাৰ ব্যক্তিগত হাৰৰ ওপৰত সম্পূৰ্ণ নিয়ন্ত্ৰণৰ প্ৰয়োজন হয়, তেন্তে **Manual** ব্যৱহাৰ কৰক।

### ভৌগোলিক স্থানৰ বাবে কিবা এটা পৃথক প্লাগইনৰ প্ৰয়োজন নে?

বাহ্যিক কোনো ভৌগোলিক স্থান প্লাগইনৰ প্ৰয়োজন নাই। এই addons-এ WordPress-ৰ বিল্ট-ইন IP-to-country detection ব্যৱহাৰ কৰে। অধিক সঠিকতাৰ বাবে, দিয়া ফিল্টাৰ হুকসমূহৰ মাজেৰে এটা ডেডিকেটেড ভৌগোলিক স্থান সেৱাক সংহত কৰিব পাৰি।

### প্ৰোডাক্ট-ভিত্তিক বিকল্পসমূহ ট্ৰাকৰ আগতে নে পাছতে প্ৰয়োগ কৰা হয়?

প্ৰোডাক্ট-ভিত্তিক বিকল্পসমূহে লক্ষ্য মুদ্রাত ট্ৰাকৰ আগৰ মূল্য নিৰ্ধাৰণ কৰে। আপোনাৰ বিদ্যমান ট্ৰাক কনফিগাৰেশ্যন ব্যৱহাৰ কৰি বিকল্প মূল্যৰ ওপৰত ট্ৰাক গণনা কৰা হয়।
