# ডোমেইন প্ৰডাক্ট আৰু মূল্যনিৰণয়ন

ডোমেইন প্ৰডাক্ট হৈছে এনে এক পদ্ধতি যাৰ জৰিয়তে আপুনি মূল্যনিৰণয়ন, TLDs, WHOIS privacy, আৰু প্ৰৱাইডৰ নিৰ্বাচন নিয়ন্ত্ৰণ কৰিব পাৰে। প্ৰতিটো ডোমেইন প্ৰডাক্ট হৈছে এটা সাধাৰণ Multisite Ultimate প্ৰডাক্ট, য'ত এটা অতিৰিক্ত **Domain Settings** ট্ৰেব থাকে।

## এটা ডোমেইন প্ৰডাক্ট সৃষ্টি কৰা

১. **Network Admin › Ultimate Multisite › Products** লৈ যাওক
২. **Add New** ক্লিক কৰক
৩. প্ৰডাক্টৰ ধৰণ (product type) **Domain** সেট কৰক
৪. **Domain Settings** ট্ৰেবটো কনফিগাৰ কৰক (তলত চাওক)
৫. সেভ কৰক

ডোমেইন প্ৰডাক্টবোৰ প্ৰডাক্ট তালিকাখনত বেঙুনীয়া **Domain** ব্যাজৰ সৈতে দেখুৱдено হয় আৰু **Domain Products** ট্ৰেব ব্যৱহাৰ কৰি ফিল্টাৰ কৰিব পাৰি।

## Domain settings ট্ৰেব

### Provider

এই প্ৰডাক্টটোৰ বাবে কোনটো ৰেজিষ্ট্ৰাৰ (registrar) ডোমেইন ৰেজিষ্ট্ৰেচন ব্যৱহ কৰিব, সেয়া ইয়াত নিৰ্বাচন কৰক। ই ডিফল্টভাৱে Domain Seller settings ত থকা গ্লোবেল **Default provider** লয়।

### Supported TLDs

যদি আপুনি ইয়াত কোনো কিছু লিখা নহয়, তেন্তে এটা **catch-all product** সৃষ্টি হ'ব যি আন কোনো প্ৰডাক্টৰ দ্বাৰা মিল নোহোৱা সকলো TLD লৈ প্ৰযোজ্য হ'ব।

এনে TLDবোৰৰ কমা-বিভাজিত তালিকা (যেনে, `.com, .net, .org`) লিখক এটা **TLD-specific product** সৃষ্টি কৰিবলৈ যি কেৱল সেই এক্সটেনচনবোৰলৈহে প্ৰযোজ্য হ'ব।

**প্ৰডাক্ট মেচিং কেনেকৈ কৰে:** কোনো ग्राहकৰ দ্বাৰা ডোমেইন বিচাৰিলে, এই addons-এ আটাইতকৈ গুৰুত্বপূৰ্ণ মিল থকা প্ৰডাক্টটো লয়। যাৰ TLD তালিকাখনত `.com` আছে, সেই প্ৰডাক্টটোৱে catch-all প্ৰডাক্টটোৰ ওপৰত গুৰুত্ব দিয়ে। যদি কোনো TLD-specific প্ৰডাক্ট মিল নহয়, তেন্তে catch-all ব্যৱহাৰ কৰা হয়। যদি কোনো প্ৰডাক্ট নাই, তেন্তে ডোমেইন অনুসন্ধান দেখুৱাই নযায়।

### Markup type

আপোনাৰ খুচুৰা মূল্য (retail price) কেনেকৈ গ্ৰོ་ছ মূল্য (wholesale cost) ৰ পৰা গণনা কৰা হয়, তাৰ বাবে তিনিটা মোড নিয়ন্ত্ৰণ কৰে:

| Mode | কেনেকৈ কাম কৰে |
|---|---|
| **Percentage** | গ্ৰོ་ছ মূল্যৰ ওপৰত এটা শতাংশ যোগ কৰে। $১0 গ্ৰོ་ছ ডোমেইনৰ ওপৰত ২০% markup দিলে $১২ হয়। |
| **Fixed markup** | এটা নিৰ্দিষ্ট ডলৰ পৰিমাণ যোগ কৰে। $১0 ডোমেইনৰ ওপৰত $৫ markup দিলে $১৫ হয়। |
| **Fixed price** | গ্ৰོ་ছ মূল্যক সম্পূৰ্ণৰূপে আওকাণ কৰে। সদায় আপুনি লিখা পৰিমাণটোক চার্জ কৰে। |

### Introductory pricing

এটা ছাঁতাকৰতকৈ কম মূল্যৰ বাবে অফাৰ দিবলৈ ইয়াক সক্ষম কৰক। এটা পৃথক **Introductory price** (১ম বছৰৰ মূল্য) সাধাৰণ **Renewal price** (২য় বছৰ আৰু তাৰ পাছৰ মূল্য) ৰ কাষত সেট কৰক। ग्राहकজনে চেকআউটৰ সময়ত দুয়োটা মূল্য দেখিবলৈ পায়, যাতে তেওঁলোকে পুনৰনিৰ্মাণৰ সময়ত কি আশা কৰিব লাগে সেয়া জনা যায়।

### WHOIS privacy

এই প্ৰডাক্টৰ জৰিয়তে ৰেজিষ্ট্ৰেচন কৰা ডোমেইনৰ বাবে WHOIS privacy protection অফাৰ কৰা হয় নে নাই, তাক নিয়ন্ত্ৰণ কৰে।

| Setting | Behaviour |
|---|---|
| **Disabled** | WHOIS privacy কেতিয়াও অফাৰ বা সক্ষম কৰা নহয়। |
| **Always Included** | WHOIS privacy কোনো মূল্য নোহোৱাকৈ ৰেজিষ্ট্ৰেচনৰ সময়ত স্বয়ংক্রিয়ভাৱে সক্ষম কৰা হয়। |
| **Customer Choice** | চেকআউটৰ সময়ত এটা চেকবক্স দেখুৱায়। **Privacy price** সেট কৰি প্ৰতি বছৰ চার্জ কৰিব পাৰে, অথবা ইয়াক $0 ৰাখি ইয়াক বিনামূলীয়াকৈ অফাৰ কৰিব পাৰে। |

Namecheap-ৰ বাবে, WHOIS privacy-এ WhoisGuard ব্যৱহাৰ কৰে (সদায় বিনামূলীয়া)। OpenSRS-ৰ বাবে, ই OpenSRS privacy service ব্যৱহাৰ কৰে (গ্ৰོ་ছ স্তৰত মূল্য হ'ব পাৰে)।

## TLD import আৰু sync

ডোমেইন প্ৰডাক্টবোৰে সংযোগ কৰা প্ৰৱাইডৰৰ পৰা লৈ অহা real-time গ্ৰོ་ছ মূল্য দেখুৱায়। ইয়াৰ বাবে, TLDবোৰ ইমপৰ্ট কৰা প্ৰয়োজন।

- **Manual sync:** Settings › Domain Seller › Sync TLDs (প্ৰৱাইডৰ অনুসৰি)
- **Automatic sync:** সকলো কনফিগাৰেট কৰা প্ৰৱাইডৰৰ ওপৰত এটা scheduled cron job জৰিয়তে দৈনিক চলি থাকে

sync কৰাৰ পিছত, যিকোনো ডোমেইন প্ৰডাক্টৰ Domain Settings ট্ৰেবলৈ যাওক আৰু উপলব্ধ TLDবোৰ আৰু তেওঁলোকৰ বৰ্তমানৰ গ্ৰོ་ছ মূল্য দেখাৰ বাবে TLD picker ব্যৱহাৰ কৰক।

## Auto-renewal

ডোমেইন পুনৰনিৰ্মাণ (renewal) হৈছে ग्राहकৰ সদস্যপদৰ স্থিতিৰ সৈতে জড়িত:

- যেতিয়া এটা সদস্যপদ পুনৰনিৰ্মাণ হয় আৰু এটা ডোমেইন সংযোগ কৰা হয়, তেতিয়া ডোমেইন পুনৰনিৰ্মাণটো স্বয়ংক্রিয়ভাৱে কিউ কৰা হয়
- পুনৰনিৰ্মাণৰ প্ৰয়াসবোৰ ग्राहकৰ active payment gateway ব্যৱহাৰ কৰে
- বিফল হোৱা পুনৰনিৰ্মাণবোৰ exponential backoff লৈ স্বয়ংক্রিয়ভাৱে পুনৰ চেষ্টা কৰা হয়
- সফল পুনৰনিৰ্মাণ, বিফলতা, আৰু অহা মেয়াদোত্তকৰ বাবে ইমেইল notificatioon প্ৰেণ কৰা হয়

ডোমেইন জীৱনচক্ৰ ইভেন্টৰ বাবে ইমেইল টেমপ্লেট ID:

| Event | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## Admin: Manual domain registration

যদি আপুনি কোনো ग्राहकৰ হৈ তেওঁলোকক চেকআউটৰ জৰিয়তে নযাওঁ, তেন্তে ডোমেইন ৰেজিষ্ট্ৰেট কৰিবলগীয়া হয়:

১. **Network Admin › Ultimate Multisite › Register Domain** লৈ যাওক
২. ग्राहकক নিৰ্বাচন কৰক আৰু ডোমেইন নামটো লিখক
৩. ৰেজিষ্ট্ৰেণ্টৰ যোগাযোগৰ বিৱৰণ (নাম, ঠিকনা, ফোন) भৰক
৪. **Register** ক্লিক কৰক

ডোমেইন ৰেকৰ্ডটো সৃষ্টি হয় আৰু ग्राहकৰ একাউণ্টৰ সৈতে সংযোগ কৰা হয়।
