# চেকআউট ক্ষেত্ৰ আৰু গ্ৰাহকৰ DNS ব্যৱস্থাপনা

## ড’মেইন বাছনি চেকআউট ক্ষেত্ৰ

**ড’মেইন বাছনি** ক্ষেত্ৰটো এটা চেকআউট উপাদান, যিয়ে গ্ৰাহকক তেওঁলোকৰ ছাইটৰ ড’মেইন কেনেকৈ পাব সেইটো বাছনি কৰাৰ সুযোগ দিয়ে। ড’মেইন বিক্ৰী সক্ষম কৰিবলৈ ইয়াক যিকোনো চেকআউট ফৰ্মত যোগ কৰক।

### চেকআউট ফৰ্মত ক্ষেত্ৰটো যোগ কৰা

1. **নেটৱৰ্ক এডমিন › Ultimate Multisite › চেকআউট ফৰ্মসমূহ**লৈ যাওক
2. এটা চেকআউট ফৰ্ম খোলক বা সৃষ্টি কৰক
3. চেকআউট সম্পাদকত, **ক্ষেত্ৰ যোগ কৰক** ক্লিক কৰক
4. ক্ষেত্ৰৰ তালিকাৰ পৰা **ড’মেইন বাছনি** বাছক
5. ক্ষেত্ৰৰ বিকল্পসমূহ কনফিগাৰ কৰক (তলত চাওক)
6. ফৰ্মটো সংৰক্ষণ কৰক

### ক্ষেত্ৰৰ বিকল্পসমূহ

**ড’মেইন মোডসমূহ** — গ্ৰাহকে কোনবোৰ টেব দেখিব বাছক। প্ৰতিটো মোড স্বতন্ত্ৰভাৱে সক্ষম বা অক্ষম কৰিব পাৰি:

| মোড | ই কি কৰে |
|---|---|
| **Subdomain** | গ্ৰাহকে আপোনাৰ নেটৱৰ্কত এটা বিনামূলীয়া subdomain ব্যৱহাৰ কৰে (যেনে, `mysite.yournetwork.com`)। কোনো পেমেন্টৰ প্ৰয়োজন নাই। |
| **নতুন ড’মেইন পঞ্জীয়ন কৰক** | গ্ৰাহকে এটা নতুন ড’মেইন সন্ধান কৰে আৰু আপোনাৰ কনফিগাৰ কৰা প্ৰদানকাৰীৰ জৰিয়তে ইয়াক পঞ্জীয়ন কৰে। মূল্য নিৰ্ধাৰণৰ বাবে মিল থকা ড’মেইন প্ৰডাক্ট ব্যৱহাৰ কৰে। |
| **বিদ্যমান ড’মেইন** | গ্ৰাহকে ইতিমধ্যে নিজৰ থকা এটা ড’মেইন ম্যাপ কৰে। কোনো পঞ্জীয়ন মাচুল নাই। ড’মেইনটো স্বয়ংক্ৰিয়ভাৱে তেওঁলোকৰ ছাইটলৈ ম্যাপ কৰা হয়। |

**ডিফল্ট মোড** — তিনিওটা মোড সক্ষম থাকিলে, কোনটো টেব প্ৰথমে খোল খাব। ড’মেইন পঞ্জীয়ন ঐচ্ছিক ৰাখিবলৈ **Subdomain** হিচাপে ছেট কৰক, বা ক্ৰয় উৎসাহিত কৰিবলৈ **নতুন ড’মেইন পঞ্জীয়ন কৰক** হিচাপে ছেট কৰক।

**ড’মেইন প্ৰডাক্ট** — ঐচ্ছিকভাৱে এই ক্ষেত্ৰটো এটা নিৰ্দিষ্ট ড’মেইন প্ৰডাক্টলৈ পিন কৰক। ছেট নকৰিলে, এডঅনে গ্ৰাহকে সন্ধান কৰা TLD-ৰ ওপৰত ভিত্তি কৰি মিল থকা প্ৰডাক্ট স্বয়ংক্ৰিয়ভাৱে বাছনি কৰে।

### পঞ্জীয়নকাৰীৰ যোগাযোগ ক্ষেত্ৰসমূহ

যেতিয়া গ্ৰাহকে **নতুন ড’মেইন পঞ্জীয়ন কৰক** টেব বাছনি কৰে, চেকআউট ফৰ্মে পঞ্জীয়নকাৰীৰ যোগাযোগ ক্ষেত্ৰসমূহ ইনলাইনভাৱে যোগ কৰে:

- নাম / উপাধি
- ইমেইল ঠিকনা
- ঠিকনা (লাইন ১, চহৰ, ৰাজ্য/প্ৰদেশ, ডাক ক’ড, দেশ)
- ফোন নম্বৰ

এইবোৰ সকলো পঞ্জীয়কৰ বাবে আৱশ্যক আৰু পঞ্জীয়ন API কল কৰাৰ আগতে যাচাই কৰা হয়। ফোন নম্বৰসমূহ পঞ্জীয়কসকলে আশা কৰা `+CC.NNN` আন্তঃৰাষ্ট্ৰীয় বিন্যাসলৈ স্বয়ংক্ৰিয়ভাৱে ফৰ্মেট কৰা হয়।

### স্বয়ংক্ৰিয়ভাৱে সৃষ্টি হোৱা ছাইট URL

যেতিয়া গ্ৰাহকে এটা ড’মেইন পঞ্জীয়ন বা ম্যাপ কৰে, ছাইট URL ক্ষেত্ৰটো বাছনি কৰা ড’মেইনৰ পৰা স্বয়ংক্ৰিয়ভাৱে পূৰণ কৰা হয়। গ্ৰাহকে পৃথক URL ক্ষেত্ৰ পূৰণ কৰাৰ প্ৰয়োজন নাই।

### সন্ধানৰ আচৰণ

- গ্ৰাহকে টাইপ কৰাৰ লগে লগে AJAX-ৰ সহায়ত ড’মেইনৰ উপলব্ধতা বাস্তৱ-সময়ত পৰীক্ষা কৰা হয়
- পছন্দৰ ড’মেইন উপলব্ধ নাথাকিলে বিকল্প TLD পৰামৰ্শসমূহ দেখুওৱা হয়
- মূল্য লাইভভাৱে আনা হয় আৰু স্পষ্টভাৱে দেখুওৱা হয় (পঞ্জীয়ন মূল্য, নবীকৰণ মূল্য, ঐচ্ছিক WHOIS গোপনীয়তা মাচুল)
- কুপন ক’ডসমূহ ড’মেইন প্ৰডাক্টত আন যিকোনো প্ৰডাক্টৰ দৰে প্ৰযোজ্য হয়

**সন্ধানৰ সঁহাৰিৰ গতি টিউন কৰা:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ড’মেইন সন্ধান ফৰ্মত কাষ্টম ক্ষেত্ৰ যোগ কৰা:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## গ্ৰাহকৰ DNS ব্যৱস্থাপনা

গ্ৰাহকে তেওঁলোকৰ পঞ্জীয়ন কৰা ড’মেইনৰ DNS ৰেকৰ্ডসমূহ **মোৰ Account** পৃষ্ঠাৰ পৰা, তেওঁলোকৰ ড’মেইনৰ এণ্ট্ৰিৰ অধীনত ব্যৱস্থাপনা কৰিব পাৰে।

### সমৰ্থিত ৰেকৰ্ড প্ৰকাৰসমূহ

| প্ৰকাৰ | ব্যৱহাৰ |
|---|---|
| **A** | হ’ষ্টনেমক IPv4 ঠিকনালৈ ম্যাপ কৰক |
| **AAAA** | হ’ষ্টনেমক IPv6 ঠিকনালৈ ম্যাপ কৰক |
| **CNAME** | আন এটা হ’ষ্টনেমলৈ আঙুলিওৱা এটা উপনাম সৃষ্টি কৰক |
| **MX** | মেইল এক্সচেঞ্জ চাৰ্ভাৰ ছেট কৰক |
| **TXT** | SPF, DMARC, যাচাইকৰণ, বা আন টেক্সট ৰেকৰ্ড যোগ কৰক |

### কোনবোৰ প্ৰদানকাৰীয়ে DNS ব্যৱস্থাপনা সমৰ্থন কৰে?

DNS ব্যৱস্থাপনা (ৰেকৰ্ড যোগ কৰা, সম্পাদনা কৰা, মচি পেলোৱা) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, আৰু **Openprovider**-ৰ সৈতে উপলব্ধ। **Hostinger** ড’মেইনসমূহে Domain Seller-ৰ জৰিয়তে nameserver আপডেট কৰিব পাৰে; হ’ষ্ট কৰা ড’মেইনৰ DNS ৰেকৰ্ডসমূহ মূল Hostinger ড’মেইন-ম্যাপিং সংযোজনে ব্যৱস্থাপনা কৰে। Namecheap, GoDaddy, আৰু NameSilo ড’মেইনসমূহে অৱস্থা আৰু মেয়াদ শেষৰ তথ্য দেখুৱায়, কিন্তু DNS পোনপটীয়াকৈ পঞ্জীয়কৰ নিয়ন্ত্ৰণ পেনেলত ব্যৱস্থাপনা কৰিব লাগিব।

### ডিফল্ট DNS ৰেকৰ্ডসমূহ

ড’মেইন পঞ্জীয়ন হ’লে স্বয়ংক্ৰিয়ভাৱে প্ৰয়োগ হোৱা ডিফল্ট DNS ৰেকৰ্ডসমূহ আপুনি কনফিগাৰ কৰিব পাৰে। **ছেটিংছ › Domain Seller › ডিফল্ট DNS ৰেকৰ্ডসমূহ**লৈ যাওক।

ডিফল্ট ৰেকৰ্ড মানসমূহে দুটা টোকেন সমৰ্থন কৰে:

| টোকেন | যাৰ সৈতে সলনি কৰা হয় |
|---|---|
| `{DOMAIN}` | পঞ্জীয়ন কৰা ড’মেইন নাম (যেনে, `example.com`) |
| `{SITE_URL}` | গ্ৰাহকৰ ছাইটৰ বাবে WordPress ছাইট URL |

**উদাহৰণ — apex ড’মেইন আৰু www-কে আপোনাৰ চাৰ্ভাৰ IP-লৈ আঙুলিয়াওক:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### এডমিন: DNS চোৱা আৰু সম্পাদনা কৰা

নেটৱৰ্ক এডমিনসকলে যিকোনো গ্ৰাহকৰ ড’মেইনৰ DNS ৰেকৰ্ডসমূহ **নেটৱৰ্ক এডমিন › Ultimate Multisite › ড’মেইনসমূহ**ত থকা ড’মেইনৰ সম্পাদনা পৃষ্ঠাৰ পৰা চাব আৰু সম্পাদনা কৰিব পাৰে।
