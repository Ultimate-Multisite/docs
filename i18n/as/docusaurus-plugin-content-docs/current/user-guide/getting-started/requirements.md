# Ultimate Multisite-ৰ প্ৰয়োজনীয়তা

Ultimate Multisite কিনি ল'বৰ আগতে আপুনি নিজকে প্ৰশ্ন কৰিব পাৰে যে এই plugin টো চলিবলৈ কি কি প্ৰয়োজনীয়তা আছে। ভাল খবৰ হ'ল যে Ultimate Multisite চলোৱাৰ বাবে কেৱল কিছুমান মৌলিক প্ৰয়োজনীয়তা আছে: এটা **domain name**, এটা ভাল **hosting provider**, আৰু এটা WordPress **Multisite** installation। ইমানহে লাগে!

## Domain Name

প্ৰথমে, আমি domain name-ৰ বিষয়ে কথা পাতিওঁ।

এটা domain name বাছি ল'বৰ সময়ত, আপোনাৰ নিৰ্দিষ্ট niche কি আৰু Ultimate Multisite ব্যৱহাৰ কৰি আপুনি কি কি বনোৱা গ্ৰহণ কৰিব, সেয়া মনত ৰাখিব: আপুনি কিবা এটা এজেঞ্চি (agency) বনাওঁ নেকি? দোকানৰ দৰে website বিক্ৰী কৰিব নেকি? নহ'লে হয়তো এটা WordPress web hosting? Ultimate Multisite লগত সকলো সম্ভৱ।

আমাৰ plugin টোৱে subscription services, payment collection, checkout forms, discount vouchers আৰু customer communications আদি প্ৰদান কৰে। এটা WordPress Multisite install কৰিবলৈ, configure কৰিবলৈ আৰু maintain কৰিবলৈ যিমানবোৰ কাম লাগে, তাৰ এটা বৃহৎ অংশ Ultimate Multisite-ৰ দ্বাৰা সহজ হৈ পৰে।

আপুনি যি কোনো niche বাছি লয়, যদি আপুনি website-ক এটা সেৱা হিচাপে বিক্ৰী কৰে, তেন্তে আমাৰ plugin ব্যৱহাৰ কৰি তাক ব্যৱস্থাপনা কৰিব পাৰিব! নিশ্চিত কৰক যে আপোনাৰ domain name টো আপোনাৰ business planৰ সৈতে মিলিব।

## Hosting Provider

এতিয়া যে আপোনাৰ niche আৰু businessক প্ৰতিনিধিত্ব কৰা এটা আকৰ্ষণীয় domain name আছে, Ultimate Multisite লগত আপোনাৰ network host কৰিবলৈ আপুনি এটা ভাল web hosting company বাছি ল'ব লাগিব। বজাৰত কিছুমান ভাল বিকল্প আছে। কিছুমান hosting provider Ultimate Multisite লগত ভালদৰে integrate কৰে, যি আপুনি কি বনাওঁ পৰিকল্পনা কৰিছে তাৰ ওপৰত নিৰ্ণায়ক কাৰক হ'ব পাৰে।

Ultimate Multisite যিকোনো hosting provider ত কাম কৰিব লাগে। এটামাত্ৰ পাৰ্থক্য হ'ল যে _WPMU DEV Hosting_ , _WP Engine_ , _Closte_ , _Cloudways_ , _Gridpane_ আৰু _Runcloud_ আদি কিছুমান hosting company-ৰ domain mapping-ৰ বাবে অধিক নিবিড় integration আছে। আন hosting-ৰ বাবে, আপুনি custom domains ম্যানুৱেলি map কৰিব লাগিব পাৰে। AutoSSL সমৰ্থন কৰা এটা hosting provider ল'বওঁ এটা ভাল ধাৰণা। _Closte_ , _Runcloud_ আৰু _cPanel_ ই ইয়াৰ সমৰ্থন কৰে।

server specifications-ৰ কথা ক'বলৈ গ'লে, কোনো নিৰ্দিষ্ট সূত্ৰ নাই, কাৰণ ই আপুনি কি কি features আৰু functions আগবঢ়াব আৰু আপোনাৰ network-ৰ আকাৰৰ ওপৰত নিৰ্ভৰ কৰিব। এটা shared hosting এটা ভাল আৰম্ভণি হ'ব পাৰে, কিন্তু যেতিয়া আপোনাৰ network ডাঙৰ হ'ব ধৰে, তেতিয়া আপোনাক হয়তো এটা dedicated VPSৰ প্ৰয়োজন হ'ব।

## WordPress Multisite

এতিয়া, আমি WordPress Multisite installation-ৰ বিষয়ে অলপ কথা পাতিওঁ।

WordPress Multisite-এ আপোনাক এটা একক installation ত বহুতো site-ৰ network ৰখাৰ সুবিধা দিয়ে। এইটো এটা built-in feature, কিন্তু ই ডিফল্টভাৱে active নহয়। আপুনি এই feature টো আপোনাৰ WordPress installation আৰু web hosting control panel দুয়োটাৰ ভিতৰত enable কৰিব লাগিব।

WordPress-ত Multisite enable কৰা **Ultimate Multisite-ৰ ভেটি**। কাৰণ Ultimate Multisite-ৰ মূল ব্যৱহাৰটো হ'ল user-সকলক website-ক এটা সেৱা হিচাপে manage আৰু বিক্ৰী কৰাত সহায় কৰা, সেয়েহে এইটো Multisite enable কৰা WordPress ত কৰিব লাগিব।

আমি এটা **গভীৰ article** ৰাখিছো, য'ত আপোনাৰ WordPress installation ত Multisite feature enable কৰিবলৈ কি কি কৰিব লাগিব, তাৰ পদক্ষেপ-প্ৰত্যেকক শিকোৱা হৈছে।

আপুনি WordPress ত Multisite enable কৰাৰ পিছত, Ultimate Multisite install কৰাৰ সময় আহিছে। Ultimate Multisite সঠিকভাৱে install কৰিবলৈ, আপোনাৰ তলত দিয়া উপাদানবোৰ থাকিব লাগিব:

  * WordPress: v5.3+ (Recommended: Last stable version)

  * PHP: 7.4.x (support to > 8.0 coming soon)

  * MySQL: v5+ (Recommended: 5.6, in case you don't have 8.0 available on your hosting provider)

এইবোৰ Ultimate Multisite-ৰ একমাত্ৰ software প্ৰয়োজনীয়তা।

মনত ৰাখিব যে Ultimate Multisite PHP 8.0 লগত কাম কৰিব পাৰে, কিন্তু আমি PHP 7.4.x ব্যৱহাৰ কৰিবলৈ পৰামৰ্শ দিওঁ।

লগতে, আমি ইয়াক **main domains** ত install কৰিবলৈ পৰামৰ্শ দিওঁ, subdomains বা subdirectories ত নহয়। ই subdomains/subdirectories ত কাম কৰিব পাৰে, কিন্তু ই কিছুমান error দেখুৱাব পাৰে।

Ultimate Multisite কেনেকৈ install কৰিব লাগে তাৰ বাবে এটা বিশদ গাইডৰ বাবে, **ইয়াত ক্লিক কৰক**।
