# হোস্টিংগার মাল্টি-টেন্যান্সি

Ultimate Multisite: Multi-Tenancy 1.2.0-এ একটি হোস্টিংগার মাল্টি-টেন্যান্সি ক্ষমতা যুক্ত করা হয়েছে, যার ফলে বিদ্যমান ডোমেইন ম্যাপিং ইন্টিগ্রেশনের পাশাপাশি হোস্ট করা ডোমেইনগুলি সার্বভৌম টেন্যান্ট প্রোভিশনিং-এর অংশ হতে পারে।

যখন টেন্যান্ট ডোমেইন এবং বিচ্ছিন্ন টেন্যান্ট অবকাঠামো Hostinger hPanel এর মাধ্যমে পরিচালিত হয়, তখন এই সুবিধাটি ব্যবহার করুন।

## সেটআপ নোটস

১. **Ultimate Multisite > Settings > Host Integrations** এর অধীনে মূল হোস্টিংগার ইন্টিগ্রেশন কনফিগার করুন।
২. নিশ্চিত করুন যে Hostinger API টোকেন লক্ষ্য করা ডোমেইন বা সাবডোমেইন পরিচালনা করতে পারে।
৩. মাল্টি-টেন্যান্সি অ্যাড-অনটি সক্ষম করুন।
৪. টেন্যান্ট প্রকাশ করার আগে টেন্যান্ট আইসোলেশন কৌশল কনফিগার করুন।
৫. টেন্যান্টে প্রোডাকশন ট্র্যাফিক পাঠানোর আগে মাইগ্রেশন ভেরিফিকেশন ওয়ার্কফ্লো চালান।

হোস্টিংগার ক্ষমতা হোস্ট-সাইড অপারেশনের জন্য শেয়ার করা হোস্টিংগার সংযোগ ব্যবহার করে। ডোমেইনটি এখনও সঠিক হোস্টিংগার অ্যাকাউন্টে নির্দেশিত হতে হবে, এবং hPanel অ্যাকাউন্টের সীমাগুলি এখনও প্রযোজ্য থাকবে।

## ক্ষমতা-নির্দিষ্ট পরিবর্তনসমূহ

- হোস্ট-সচেতন ডোমেইন অপারেশনের মাধ্যমে সার্বভৌম টেন্যান্ট তৈরি করা যেতে পারে।
- যাচাইকরণের আগে একই মেশিনের ডেটাবেস হোস্ট স্ট্রিংগুলিকে স্বাভাবিক (normalize) করা হয়।
- WordPress রানটাইম যদি স্থানীয় ওভাররাইড চায়, তবে Hostinger-পরিচালিত টেন্যান্টগুলি hPanel এ দেখানো ডেটাবেস হোস্ট মান ব্যবহার করবে।
- SSO ভিজিট নির্ভর করে টেন্যান্ট ডোমেইনটি হোস্টিংগার দ্বারা হোস্ট করা টেন্যান্টে রেজোলিউশন করার উপর।

## হোস্টিংগার টেন্যান্ট ট্রাবলশুটিং

- যদি কোনো টেন্যান্ট ইনস্টলেশন ব্যর্থ হয়, তবে যাচাই করুন যে ডোমেইনটি ইতিমধ্যেই হোস্টিংগার অ্যাকাউন্টে সংযুক্ত আছে কিনা।
- যদি ডেটাবেস ভেরিফিকেশন ব্যর্থ হয়, তবে hPanel এর সাথে টেন্যান্টের DB ইউজারনেম, ডেটাবেস নাম এবং হোস্ট বাইন্ডিং তুলনা করুন।
- যদি **Visit (SSO)** ব্যর্থ হয়, তবে টেন্যান্ট ডোমেইনের জন্য DNS এবং SSL সক্রিয় আছে কিনা তা নিশ্চিত করুন।
- যদি টরাউন করার পরে হোস্ট রিসোর্স পিছনে পড়ে থাকে, তবে ব্যাকআপ নিশ্চিত করার পরে hPanel থেকে যেকোনো অবশিষ্ট ডেটাবেস, ইউজার বা ফোল্ডার সরিয়ে ফেলুন।
