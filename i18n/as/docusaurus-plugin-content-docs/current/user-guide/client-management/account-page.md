# আপোনাৰ Client-ৰ Account Page (v2)

_**গুৰুত্বপূৰ্ণ টোকা: এই article টি Ultimate Multisiteৰ version 2.x লৈ বৰ্ণনা কৰে।**_

যেতিয়া কোনো ग्राहक আপোনাৰ network ত এটা plan লয়, তেওঁক এটা website আৰু তাৰ dashboard-লৈ প্ৰৱেশ কৰাৰ সুবিধা হয়। এই dashboard ত তেওঁলোকৰ পেমেন্ট, মেম্বৰশ্বিপ, ডোমেইন, planৰ সীমা আদিৰ বিষয়ে গুৰুত্বপূৰ্ণ তথ্য থাকে।

এই tutorial ত, আমি আপোনাক ग्राहकৰ account page লৈ গাইড কৰিম, আৰু আপুনি দেখিব যে আপোনাৰ ग्राहकলোকে ইয়াৰ ভিতৰত কি কি দেখা আৰু কৰিব পাৰিব।

## Account Page

আপোনাৰ client-ৰ dashboardৰ ভিতৰত **Account** ক্লিক কৰি এই account page লৈ যাব পাৰে।

![Account menu in the customer dashboard](/img/account-page/account-menu.png)

![Customer account page overview](/img/account-page/overview.png)

এটা ग्राहकয়ে ইয়াত ক্লিক কৰাৰ পিছত, তেওঁলোকে তেওঁলোকৰ মেম্বৰশ্বিপ, বিলিং এড্ৰես, ইনভয়েছ, ডোমেইন, siteৰ সীমা আদিৰ এটা সামৰণ (overview) দেখিব পাৰিব। ইয়াৰ উপৰিও, তেওঁলোকে **Site Template** (যদি আপোনাৰ network ত ই অনুমতি দিয়া হয়) সলনি কৰিব পাৰিব।

তেওঁলোকে মেম্বৰশ্বিপ আন এটা plan লৈ সলনি কৰিব পাৰে, অথবা আপুনি যি আন কোনো package বা service আগবঢ়ায়, সেয়া ক্ৰয় কৰিব পাৰে। আহক, আমি প্ৰতিটো অংশ পৃথকভাৱে চাওঁ।

### Your Membership Overview:

আপোনাৰ client-ৰ website নামৰ ঠিক তলৰ প্ৰথম ব্লকটোৱে তেওঁলোকৰ বৰ্তমান plan আৰু ইয়াৰ লগত ক্ৰয় কৰা service/package-ৰ এটা সামৰণ দেখুৱায়। এই ব্লকটোৱে মেম্বৰশ্বিপ নম্বৰ, ইয়াৰ বাবে দিয়া প্ৰথম ধনৰ পৰিমাণ, plan আৰু যিকোনো service/package-ৰ মূল্য, আৰু এই মেম্বৰশ্বিপৰ বাবে ক'তবাৰকৈ বিল কৰা হৈছে, সেইবোৰো দেখুৱায়। তেওঁলোকে এই মেম্বৰশ্বিপ **Active** (সক্ৰিয়), **Expired** (শেষ হোৱা) নে **Canceled** (বাতিল) হৈছে, সেয়াও দেখিব পাৰে।

![Membership overview showing plan, amount, and billing details](/img/account-page/membership-card.png)

এই ব্লকটোৰ ঠিক তলত, আপোনাৰ client-সকলে **About This Site** আৰু **Site Limits** ব্লকবোৰ দেখিব পাৰিব। এই ব্লকবোৰে তেওঁলোকৰ planৰ সকলো সীমা দেখুৱায়: disk space, posts, pages, visits, আদি... এই সীমাবোৰ **Ultimate Multisite > Products** ত প্ৰতিটো plan page ত কনফিগাৰ কৰিব পাৰি।

![About This Site and Site Limits blocks showing plan limitations](/img/account-page/site-limits.png)

**Your Membership** ৰ সোঁফাণত, client-সকলে **Change** ক্লিক কৰিব পাৰে। ইয়াৰ দ্বাৰা উপলব্ধ সকলো plan আৰু package/service দেখুৱাব। যদি তেওঁলোকে আন এটা plan বাছি লয়, তেন্তে planৰ সীমা বৰ্তমান মেম্বৰশ্বিপৰ সীমাৰ সলনি হ'ব—সেয়া তেওঁলোকে plan ডাউনগ্ৰেড কৰিছে নে আপগ্ৰেড কৰিছে, তাৰ কোনো গুৰুত্ব নাই।

এতিয়া, যদি আপোনাৰ client-সকলে এই বৰ্তমান মেম্বৰশ্বিপৰ বাবে package বা service ক্ৰয় কৰিবলৈ বাছি লয়—যেনে অধিক disk space বা visits—তেন্তে বৰ্তমান মেম্বৰশ্বিপটো সলনি নহ'ব, কেৱল নতুন package-বোৰ ইয়াত যোগ কৰা হ'ব।

মনত ৰাখিব যে coupon code এই মেম্বৰশ্বিপ সলনি কৰা page ত যোগ কৰিব নোৱাৰি। যদি client-এ প্ৰথম মেম্বৰশ্বিপ ক্ৰয় কৰাৰ সময়ত কোনো coupon code ব্যৱহাৰ কৰিছিল, তেন্তে সেই কোডটো নতুন মেম্বৰশ্বিপলৈও প্ৰযোজ্য হ'ব।

### Updating the Billing Address:

account page ত, আপোনাৰ client-সকলে তেওঁলোকৰ বিলিং এড্ৰեսও আপডেট কৰিব পাৰে। তেওঁলোকক কেৱল _Billing Address_ৰ কাষত থকা **Update** ক্লিক কৰিব লাগিব।

![Billing Address section with Update button](/img/account-page/billing-address.png)

আপোনাৰ client-ৰ বাবে এটা নতুন window খুলিব। তেওঁক কেৱল নতুন এড্ৰեսটো ভৰাই _Save Changes_ ক্লিক কৰিব লাগিব।

![Update billing address form](/img/account-page/billing-address-form.png)

### Changing the Site Template:

আপোনাৰ client-সকলক তেওঁলোকৰ site template সলনি কৰিবলৈ অনুমতি দিবলৈ, আপুনি **Ultimate Multisite > Settings > Sites** লৈ গৈ **Allow Template Switching** অপচনটো চালু (toggle on) কৰিব লাগিব।

ইয়াৰ উপৰিও, **Ultimate Multisite > Products** ত, আপোনাৰ planবোৰ বাছি লওক আৰু **Site Templates** ট্ৰি-লৈ যাওক। নিশ্চিত কৰক যে **Allow Site Templates** অপচনটো চালু (toggled on) কৰা হৈছে আৰু **Site Template Selection Mode** ত, **Choose Available Site Templates** অপচনটো বাছনি কৰা হৈছে।

![Product site templates tab with template selection mode](/img/config/product-site-templates.png)

আপুনি আপোনাৰ website ত উপলব্ধ সকলো site template দেখিবলৈ পাব। আপুনি কোনবোৰ উপলব্ধ কৰিব বিচাৰে আৰু কোনবোৰ এই planৰ অধীনত সংস্ক্রামিত client-সকলৰ বাবে উপলব্ধ নহ'ব, তাৰ বাবে বাছি লওক। মনত ৰাখিব যে এই অপচনটোৱে checkout form-কও প্ৰভাৱিত কৰে, গতিকে যি কোনো template যাক **Not Available** বুলি বাছি লোৱা হ'ব, সেয়া এই planৰ বাবে ৰেজিষ্ট্ৰেচন page ত দেখিব নোৱাৰি।

এতিয়া আপোনাৰ client-সকলে তেওঁলোকৰ account pageৰ ভিতৰত **Change Site Template** ক্লিক কৰিব পাৰে।

![Change Site Template button on account page](/img/account-page/change-template-button.png)

Ultimate Multisite 2.10.0 ত এটা নতুন ডিজাইন কৰা template switch panel আছে। এই panel টোপনি এটা **current-template card** লৈ আৰম্ভ হয়, যাতে client-সকলে কোনো বিকল্প বাছনি কৰাৰ আগতে কোনটো template active আছে, সেয়া দেখিব পাৰে।

উপলব্ধ site template-ৰ এটা স্থায়ী grid client-সকলে তেওঁলোকৰ বিকল্পসমূহ পৰীক্ষা কৰোঁতে দৃশ্যমান থাকে। ইয়াৰ দ্বাৰা তেওঁলোকে বৰ্তমানৰ বাছনিৰ পৰিহ্ৰম নোহোৱাকৈ তেওঁলোকৰ planৰ বাবে উপলব্ধ template-বোৰৰ তুলনা কৰাত সহায় পায়।

![Available site templates list for the plan](/img/config/site-templates-list.png)

যিটো template সলনি কৰিব বিচাৰে, সেয়া বাছনি কৰাৰ পিছত, তেওঁক সলনি কৰাৰ বাবে নিশ্চিত কৰিবলৈ কোৱা হ'ব।

![Site template switch confirmation dialog](/img/account-page/template-switch-confirm.png)

নিশ্চিতকৰণটো চালু (toggling on) কৰি আৰু **Process Switch** ক্লিক কৰাৰ পিছত, নতুন site template টো আপোনাৰ client-ৰ website ত ব্যৱহাৰ হ'ব।

client-সকলে এই panelৰ পৰা **Reset current template** ব্যৱহাৰ কৰিব পাৰে, যেতিয়া তেওঁলোকে site টো পুনৰ বৰ্তমান দিয়া template লৈ যাবলগীয়া হয়। আন এটা template লৈ সলনি কৰাৰ দৰেই, template reset কৰিলে site content মাৰি যাব পাৰে, গতিকে client-সকলে কেৱল তেওঁলোকে reset কৰাৰ কাৰ্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য্য
