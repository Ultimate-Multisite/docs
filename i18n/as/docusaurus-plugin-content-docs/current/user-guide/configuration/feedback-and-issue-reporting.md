# ग्राहकৰ মতামত আৰু সমস্যা প্ৰতিবেদন

Gratis AI Agent v1.5.0-এ এটা অন্তৰ্নিহিত (built-in) মতামত আৰু সমস্যা প্ৰতিবেদন ব্যৱস্থাৰে (feedback and issue reporting system) সাজু কৰি তুলিছে। এই ব্যৱস্থাই শেষ ব্যৱহাৰকাৰীসকলক চ্যাট interfase बाटै সহায় নোহোৱা উত্তৰ চিহ্নিত কৰিবলৈ আৰু সমস্যা প্ৰতিবেদন কৰিবলৈ অনুমতি দিয়ে। এই ব্যৱস্থাত সন্মতি ব্যৱস্থাপনা (consent management), এটা স্বয়ংক্রিয় প্ৰতিবেদন কাৰ্য্য (automated reporting command), আৰু ব্যাকএণ্ডে AI-সহায়িত প্ৰথম স্তৰৰ পৰীক্ষা (AI-assisted triage) অন্তৰ্ভুক্ত।

## থুম্বছ-ডাউন বাটন (Thumbs-Down Button) {#thumbs-down-button}

AI সহায়কৰ দ্বাৰা প্ৰেৰণ কৰা প্ৰতিটো বাৰ্তাৰ সৈতে এটা **থুম্বছ-ডাউন** (👎) বাটন দেখুৱাই। ব্যৱহাৰকাৰীয়ে ইয়াক ক্লিক কৰিলে, তেওঁ এটা উত্তৰক সহায় নোহোৱা বা ভুল বুলি চিহ্নিত কৰিব পাৰে।

- এই বাটনটো সহায়কৰ বাৰ্তাৰ কাষত মাউছ লৈ গ'লে (on hover) দেখুৱাই।
- ইয়াক ক্লিক কৰিলে **Feedback Consent Modal** খুলি যায়।
- এই মতামতটো কথোপকথনৰ থ্ৰেড আৰু নিৰ্দিষ্ট বাৰ্তাৰ সৈতে জড়িত থাকে।

## Feedback Consent Modal {#feedback-consent-modal}

ব্যৱহাৰকাৰীয়ে থুম্বছ-ডাউন বাটনটো ক্লিক কৰিলে, কোনো তথ্য প্ৰেৰণ কৰাৰ আগতে এটা সন্মতি Modal দেখুৱাই। এই Modal-এ:

- কি কি তথ্য সংগ্ৰহ কৰা হ'ব (কথোপকথনৰ অংশ, ব্ৰাউজাৰৰ পৰিৱেশ) সেয়া ব্যাখ্যা কৰে।
- ব্যৱহাৰকাৰীক এই তথ্য ভাগ কৰাত তেওঁৰ সন্মতি দিয়াৰ বাবে নিশ্চিত কৰিবলৈ কয়।
- ব্যৱহাৰকাৰীয়ে কি ভুল হৈছে বুলি বৰ্ণনা কৰিবলৈ এটা মুক্ত-পাঠ্য (free-text) স্থান দিয়ে।
- প্ৰেৰণ নকৰাকৈ বন্ধ কৰিবলৈ এটা **Cancel** বিকল্প দিয়ে।

ব্যৱহাৰকাৰীয়ে স্পষ্টকৈ সন্মতি নিদিবলৈ কোনো মতামত সংগ্ৰহ নহয়।

## Auto-Prompt Feedback Banner {#auto-prompt-feedback-banner}

এটা কথোপকথনৰ শেষত, সহায়কজনে এটা **auto-prompt feedback banner** দেখুৱাব পাৰে — এটা কম বাধা সৃষ্টিকাৰী (non-intrusive) বাৰ্তা যিয়ে সেশনটো সহায়ক আছিল নে নাই বুলি সোধে।

এই ব্যৱস্থাটো স্বয়ংক্রিয়ভাৱে কথোপকথনৰ দৈৰ্ঘ্য আৰু ফলাফলৰ ওপৰত ভিত্তি কৰি (heuristics) দেখুৱাই। ই থুম্বছ-ডাউন বাটনৰ দৰেই একেটা মতামত প্ৰবাহলৈ (feedback flow) সংযোগ কৰে। ব্যৱহাৰকাৰীসকলে মতামত নিদিওঁতে এই ব্যৱস্থাটো বন্ধ কৰিব পাৰে।

## /report-issue Slash Command {#report-issue-slash-command}

ব্যৱহাৰকাৰীসকলে চ্যাট ইনপুটত `/report-issue` টাইপ কৰি পোনপটীয়াকৈ মতামত প্ৰবাহটো আৰম্ভ কৰিব পাৰে। এই কাৰ্য্যটোৱে:

- তৎক্ষণাত Feedback Consent Modal খুলি দিয়ে।
- বৰ্তমানৰ কথোপকথনৰ বিষয়ে প্ৰতিবেদন স্থানত (description field) স্বয়ংক্রিয়ভাৱে তথ্য যোগ কৰে।
- প্ৰেৰণ কৰাৰ আগতে ব্যৱহাৰকাৰীক অধিক বিৱৰণ যোগ কৰিবলৈ দিয়ে।

`/report-issue` কাৰ্য্যটো সকলো চ্যাট মোডত (inline, floating widget, full-screen) উপলব্ধ।

## AI-Assisted Triage {#ai-assisted-triage}

প্ৰেৰণ কৰা মতামতটো এটা AI triage layer লৈ যায় যিয়ে:

- প্ৰতিবেদনক এটা শ্ৰেণীত বিভক্ত কৰে (তথ্যগত ভুল, সহায় নোহোৱা উত্তৰ, অনুপযুক্ত বিষয়বস্তু, ইত্যাদি)।
- প্ৰাসংগিক কথোপকথনৰ মেটাডেটা (metadata) যোগ কৰে।
- triage summary টো কনফিগাৰেট কৰা **Feedback Endpoint** লৈ প্ৰেৰণ কৰে (দেখুৱাওক [Settings > Advanced](#settings-advanced))।

এনে কৰাৰ ফলত হাতেৰে পৰীক্ষা কৰাৰ সময়ৰ অপচয় কম হয়, কাৰণ ই আটাইতকৈ গুৰুত্বপূৰ্ণ সমস্যাবোৰক প্ৰথমে ওপৰত লৈ আহে।

## Settings > Advanced {#settings-advanced}

মতামত ব্যাকএণ্ড সক্ষম কৰিবলৈ, **Gratis AI Agent → Settings → Advanced** তলত দিয়া স্থানসমূহ কনফিগাৰ কৰক:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | এই URL টোৱে মতামতৰ পেলোএড (payloads) লৈ POST request গ্ৰহণ কৰে (JSON)। |
| **Feedback API Key** | মতামত প্ৰেৰণ কৰাৰ বাবে প্ৰমাণীকৰণ কৰিবলৈ `Authorization` header ত পঠোৱা Bearer token। |

মতামত সংগ্ৰহ বন্ধ কৰিবলৈ দুয়োটা স্থান খালি ৰাখক। ব্যৱহাৰকাৰীসকলৰ বাবে ব্যক্তিগত মতামত বাটনবোৰ দৃশ্যমান থাকে, কিন্তু প্ৰেৰণ কৰা নহ'ব।

:::tip
মতামত endpoint টোৱে কম বেছি `message_id`, `conversation_id`, `feedback_text`, আৰু `triage_category` ফিল্ড থকা এটা JSON body গ্ৰহণ কৰিব লাগিব। আশা কৰা স্কিমাটোৰ বাবে আপোনাৰ endpoint প্ৰোভাইডেৰৰ ডকুমেন্টেশ্যন চাওক।
:::
