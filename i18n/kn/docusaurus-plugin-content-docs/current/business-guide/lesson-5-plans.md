# ಪಾಠ 5: ನಿಮ್ಮ ಯೋಜನೆಗಳನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು

ನಿಮ್ಮ ಯೋಜನೆಗಳ ಟಿಯರ್‌ಗಳು ಕೇವಲ ಬೆಲೆ ಮಟ್ಟಗಳು ಮಾತ್ರವಲ್ಲ — ಅವು ನಿಮ್ಮ ನಿರ್ದಿಷ್ಟ ಗ್ರಾಹಕರು (niche customers) ನಿಜವಾಗಿ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಾರೆ ಎಂಬುದನ್ನು ಪ್ರತಿಬಿಂಬಿಸುತ್ತವೆ. ಈ ಪಾಠದಲ್ಲಿ, ವಿವಿಧ ಹಂತಗಳಲ್ಲಿರುವ ಫಿಟ್‌ನೆಸ್ ವ್ಯವಹಾರಗಳ ನಿಜವಾದ ಅಗತ್ಯಗಳಿಗೆ ಹೊಂದಿಕೆಯಾಗುವ ಉತ್ಪನ್ನ ಟಿಯರ್‌ಗಳನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುತ್ತೀರಿ.

## ನಾವು ಎಲ್ಲಿ ನಿಲ್ಲಿಸಿದ್ದೆವು {#where-we-left-off}

FitSite ನಲ್ಲಿ ಮೂರು ಟೆಂಪ್ಲೇಟ್‌ಗಳು ಸಿದ್ಧವಾಗಿವೆ (Studio Essential, Gym Pro, Fitness Chain). ಈಗ, ಗ್ರಾಹಕರು ಏನನ್ನು ಪಡೆಯುತ್ತಾರೆ ಮತ್ತು ಅವರು ಎಷ್ಟು ಪಾವತಿಸುತ್ತಾರೆ ಎಂಬುದನ್ನು ನಿರ್ಧರಿಸುವ ಯೋಜನೆಗಳನ್ನು ನಾವು ರಚಿಸಬೇಕು.

## ಗ್ರಾಹಕ ವಿಭಾಗಗಳ ಆಧಾರದ ಮೇಲೆ ಯೋಚಿಸುವುದು {#thinking-in-customer-segments}

ಹೆಚ್ಚಿನ ಜನರು ಮಾಡುವ ತಪ್ಪು ಎಂದರೆ, ಯೋಜನೆಗಳನ್ನು ತಾಂತ್ರಿಕ ವೈಶಿಷ್ಟ್ಯಗಳ (storage, bandwidth, pages ನಂಬರ್) ಆಧಾರದ ಮೇಲೆ ವಿನ್ಯಾಸಗೊಳಿಸುವುದು. ನಿಮ್ಮ ನಿರ್ದಿಷ್ಟ ಗ್ರಾಹಕರು ಆ ಪದಗಳಲ್ಲಿ ಯೋಚಿಸುವುದಿಲ್ಲ. ಒಬ್ಬ ಜಿಮ್ ಮಾಲೀಕ ತನ್ನ ವ್ಯವಹಾರವನ್ನು ನಡೆಸಲು ಏನು ಬೇಕು ಎಂದು ಯೋಚಿಸುತ್ತಾನೆ.

ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋಗಳಿಗೆ, ಮೂರು ನೈಸರ್ಗಿಕ ವಿಭಾಗಗಳು ಅಸ್ತಿತ್ವದಲ್ಲಿವೆ:

| Segment | ಯಾರು ಅವರು | ಅವರಿಗೆ ಏನು ಬೇಕು |
|---------|-------------|----------------|
| **Solo trainers / small studios** | 1-3 ಸಿಬ್ಬಂದಿ, ಒಂದೇ ಸ್ಥಳ, ಪ್ರಾರಂಭಿಸುತ್ತಿರುವವರು | ವೃತ್ತಿಪರವಾದ ವೆಬ್‌ಸೈಟ್, class ಮಾಹಿತಿ, ಸಂಪರ್ಕ ಫಾರ್ಮ್ |
| **Established gyms** | 5-20 ಸಿಬ್ಬಂದಿ, ಒಂದೇ ಸ್ಥಳ, ಬೆಳೆಯುತ್ತಿರುವವರು | ಮೇಲಿನ ಎಲ್ಲದರ ಜೊತೆಗೆ ಬುಕಿಂಗ್, ಬ್ಲಾಗ್, ಗ್ಯಾಲರಿ, ಕಸ್ಟಮ್ ಡೊಮೇನ್ |
| **Fitness chains** | ಅನೇಕ ಸ್ಥಳಗಳು, ಸ್ಥಾಪಿತ ಬ್ರ್ಯಾಂಡ್ | ಮೇಲಿನ ಎಲ್ಲದರ ಜೊತೆಗೆ multi-site, ಸ್ಥಳಗಳ ಪುಟಗಳು, ಸಿಬ್ಬಂದಿ ಡೈರೆಕ್ಟರಿ |

ನಿಮ್ಮ ಯೋಜನೆಗಳು ಈ ವಿಭಾಗಗಳಿಗೆ ಹೊಂದಿಕೆಯಾಗಬೇಕು, ಕೇವಲ ಯಾದೃಚ್ಛಿಕ ವೈಶಿಷ್ಟ್ಯಗಳ ಗುಚ್ಛಗಳಿಗೆ ಅಲ್ಲ.

## Plan Group ಅನ್ನು ಸೆಟಪ್ ಮಾಡುವುದು {#setting-up-the-plan-group}

ಈ plan group ಅನ್ನು ಬಳಸುವುದರಿಂದ ಗ್ರಾಹಕರು FitSite plan ಕುಟುಂಬದೊಳಗೆ ಮಾತ್ರ ಅಪ್‌ಗ್ರೇಡ್ ಅಥವಾ ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡಬಹುದು. ಪ್ರತಿ ಯೋಜನೆಯ **Up & Downgrades** tab ನಲ್ಲಿ:

1. ಎಲ್ಲಾ ಮೂರು ಯೋಜನೆಗಳಿಗೆ **Plan Group** ಅನ್ನು "FitSite Plans" ಎಂದು ಹೊಂದಿಸಿ
2. **Product Order** ಅನ್ನು 1 (Starter), 2 (Growth), 3 (Pro) ಎಂದು ಹೊಂದಿಸಿ

ಇದು ಸ್ಪಷ್ಟವಾದ ಅಪ್‌ಗ್ರೇಡ್ ಮಾರ್ಗವನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ: Starter → Growth → Pro.

## Order Bumps ಸೇರಿಸುವುದು {#adding-order-bumps}

Order bumps ಎಂದರೆ ಚೆಕ್‌ಔಟ್ ಸಮಯದಲ್ಲಿ ನೀಡುವ ಹೆಚ್ಚುವರಿ ಉತ್ಪನ್ನಗಳು. FitSite ಗಾಗಿ, ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಪರಿಗಣಿಸಿ:

- **Extra Storage Pack** ($19/month) -- ಹೆಚ್ಚುವರಿ 5 GB disk space
- **Priority Support** ($29/month) -- ವೇಗದ ಪ್ರತಿಕ್ರಿಯೆಯ ಸಮಯ
- **Additional Site** ($39/month) -- ತಮ್ಮ ಯೋಜನೆಯಲ್ಲಿ ಅನುಮತಿಸಿದಕ್ಕಿಂತ ಹೆಚ್ಚಿನ ಸ್ಥಳಗಳು ಬೇಕಾದ ಗ್ರಾಹಕರಿಗೆ

ಇವುಗಳನ್ನು Ultimate Multisite ನಲ್ಲಿ **Package** ರೀತಿಯ ಉತ್ಪನ್ನಗಳಾಗಿ ರಚಿಸಿ ಮತ್ತು ಅವುಗಳನ್ನು ಸಂಬಂಧಿತ ಯೋಜನೆಗಳಿಗೆ ಜೋಡಿಸಿ.

## ಈ ರಚನೆ ಏಕೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ {#why-this-structure-works}

- **Starter** ಪ್ರವೇಶದ ಅಡೆತಡೆಗಳನ್ನು ನಿವಾರಿಸುತ್ತದೆ -- ಕಡಿಮೆ ಬೆಲೆ, ಸರಳ ಕೊಡುಗೆ, ತರಬೇತುದಾರರಿಗೆ ವೇಗವಾಗಿ ಆನ್‌ಲೈನ್ ಆಗಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ
- **Growth** ಜಿಮ್‌ಗಳು ನಿಜವಾಗಿಯೂ ಕೇಳುವ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಸೇರಿಸುತ್ತದೆ -- ಬುಕಿಂಗ್, ಗ್ಯಾಲರಿಗಳು, ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳು
- **Pro** ಬಹು-ಸ್ಥಳ ಬೆಂಬಲದ ಅಗತ್ಯವಿರುವ ಹೆಚ್ಚಿನ ಮೌಲ್ಯದ ವಿಭಾಗಕ್ಕೆ ಸೇವೆ ಸಲ್ಲಿಸುತ್ತದೆ
- **Order bumps** ಗ್ರಾಹಕರು ಮೂಲ ಯೋಜನೆಗಳನ್ನು ಸಂಕೀರ್ಣಗೊಳಿಸದೆ ಕಸ್ಟಮೈಸ್ ಮಾಡಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ
- **ಸ್ಪಷ್ಟ ಅಪ್‌ಗ್ರೇಡ್ ಮಾರ್ಗ** ಎಂದರೆ ಗ್ರಾಹಕರು ನಿಮ್ಮೊಂದಿಗೆ ಬೆಳೆಯುತ್ತಾರೆ, ಹೊರಹೋಗುವುದಿಲ್ಲ

## ಇಲ್ಲಿಯವರೆಗೆ FitSite Network {#the-fitsite-network-so-far}

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products
│   ├── FitSite Starter ($49/mo) → Studio Essential template
│   ├── FitSite Growth ($99/mo) → Studio Essential + Gym Pro templates
│   ├── FitSite Pro ($199/mo) → All templates
│   └── Order Bumps (Extra Storage, Priority Support, Additional Site)
└── Ready for checkout configuration (next lesson)
```

## ಈ ಪಾಠದಲ್ಲಿ ನಾವು ಏನು ನಿರ್ಮಿಸಿದೆವು {#what-we-built-this-lesson}

- ನಿಜವಾದ ಫಿಟ್‌ನೆಸ್ ವ್ಯವಹಾರ ವಿಭಾಗಗಳಿಗೆ ಹೊಂದಿಕೆಯಾದ **ಮೂರು ಯೋಜನೆಗಳ ಟಿಯರ್‌ಗಳು**
- Ultimate Multisite ನ plugin ಮತ್ತು template ನಿಯಂತ್ರಣಗಳನ್ನು ಬಳಸಿ **ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ನಿಯಂತ್ರಿಸುವುದು (Feature gating)**
- ಸ್ಪಷ್ಟ ಅಪ್‌ಗ್ರೇಡ್ ಮಾರ್ಗ ಹೊಂದಿರುವ **ಒಂದು plan group**
- ಹೆಚ್ಚುವರಿ ಆದಾಯಕ್ಕಾಗಿ **Order bump ಉತ್ಪನ್ನಗಳು**
- ತಾಂತ್ರಿಕ ವಿವರಗಳ ಆಧಾರದ ಮೇಲೆ ಅಲ್ಲ, ಗ್ರಾಹಕರ ಮೌಲ್ಯದ ಆಧಾರದ ಮೇಲೆ **ಬೆಲೆ ನಿಗದಿಪಡಿಸುವ ರಚನೆ**
