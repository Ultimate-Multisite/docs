# ಪಾಠ 6: ನೋಂದಣಿ ಅನುಭವ (The Signup Experience)

ಚೆಕ್ಅೌಟ್ ಫ್ಲೋ ಎಂದರೆ ಆಸಕ್ತಿಯನ್ನು ಆದಾಯವಾಗಿ ಪರಿವರ್ತಿಸುವ ಸ್ಥಳ. ಗೊಂದಲಮಯ ಅಥವಾ ಸಾಮಾನ್ಯ ನೋಂದಣಿ ಪ್ರಕ್ರಿಯೆಯು ಗ್ರಾಹಕರನ್ನು ಕಳೆದುಕೊಳ್ಳುತ್ತದೆ. ಅವರ ಭಾಷೆಯನ್ನು ಮಾತನಾಡುವ ಮತ್ತು ಸುಲಭವೆಂದು ಅನಿಸುವ ಒಂದು ನಿರ್ದಿಷ್ಟ ವಲಯಕ್ಕೆ ಸಂಬಂಧಿಸಿದ (niche-specific) ಫ್ಲೋ ಅವರನ್ನು ಪರಿವರ್ತಿಸುತ್ತದೆ.

## ನಾವು ಎಲ್ಲಿ ನಿಲ್ಲಿಸಿದ್ದೆವು

FitSite ಗೆ ಟೆಂಪ್ಲೇಟ್‌ಗಳು ಮತ್ತು ಯೋಜನೆಗಳನ್ನು (plans) ಕಾನ್ಫಿಗರ್ ಮಾಡಲಾಗಿದೆ. ಈಗ, ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋ ಮಾಲೀಕರನ್ನು ಪಾವತಿಸುವ ಗ್ರಾಹಕರನ್ನಾಗಿ ಪರಿವರ್ತಿಸುವ ನೋಂದಣಿ ಮತ್ತು ಚೆಕ್ಅೌಟ್ ಅನುಭವವನ್ನು ನಿರ್ಮಿಸೋಣ.

## ನೋಂದಣಿ ಫ್ಲೋ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

Ultimate Multisite ನ ಚೆಕ್ಅೌಟ್ ಫಾರ್ಮ್‌ಗಳು ಸಂಪೂರ್ಣವಾಗಿ ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದಾದ multi-step ಫಾರ್ಮ್‌ಗಳಾಗಿವೆ. FitSite ಗಾಗಿ, ನಮಗೆ ಬೇಕಾದ ಫ್ಲೋ ಹೀಗಿರಬೇಕು:

1. ಫಿಟ್‌ನೆಸ್ ವ್ಯವಹಾರಗಳಿಗೆ ನಿರ್ಮಿಸಲಾಗಿದೆ ಎಂಬ ಅನುಭವ ನೀಡಬೇಕು
2. ಅಗತ್ಯವಿರುವ ಮಾಹಿತಿಯನ್ನು ಮಾತ್ರ ಸಂಗ್ರಹಿಸಬೇಕು
3. ಗ್ರಾಹಕನನ್ನು ಸಾಧ್ಯವಾದಷ್ಟು ಬೇಗ ಕಾರ್ಯನಿರ್ವಹಿಸುವ (working) ಸೈಟ್‌ಗೆ ಕರೆದೊಯ್ಯಬೇಕು

ಪೂರ್ಣ ತಾಂತ್ರಿಕ ಮಾಹಿತಿಗಾಗಿ [The Registration Flow](/user-guide/configuration/the-registration-flow) ನೋಡಿ.

## FitSite ಚೆಕ್ಅೌಟ್ ವಿನ್ಯಾಸಗೊಳಿಸುವುದು

**Ultimate Multisite > Checkout Forms** ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ ಮತ್ತು ಹೊಸ ಫಾರ್ಮ್ ಅನ್ನು ರಚಿಸಿ.

### ಹಂತ 1: ಯೋಜನೆ ಆಯ್ಕೆ (Plan Selection)

ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋ ಮಾಲೀಕರು ಮೊದಲು ನೋಡಬೇಕಾದ ವಿಷಯವೆಂದರೆ ಯೋಜನೆಗಳು (plans). ಅವುಗಳನ್ನು ಅವರು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ಪದಗಳಲ್ಲಿ ಪ್ರಸ್ತುತಪಡಿಸಬೇಕು.

- **Pricing Table** ಫೀಲ್ಡ್ ಅನ್ನು ಸೇರಿಸಿ
- ಇದು ಮೂರು FitSite ಯೋಜನೆಗಳನ್ನು ತೋರಿಸಲು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ
- ಪಾಠ 5 ರಲ್ಲಿ ನೀವು ಬರೆದ ಯೋಜನೆ ವಿವರಣೆಗಳು ಇಲ್ಲಿ ಕಾಣಿಸುತ್ತವೆ -- ಅವು ತಾಂತ್ರಿಕ ವೈಶಿಷ್ಟ್ಯಗಳ ಬಗ್ಗೆ ಅಲ್ಲ, ಫಿಟ್‌ನೆಸ್ ವ್ಯವಹಾರದ ಅಗತ್ಯಗಳ ಬಗ್ಗೆ ಮಾತನಾಡುತ್ತವೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.

:::tip ನಿರ್ದಿಷ್ಟ ವಲಯದ ಭಾಷೆ ಮುಖ್ಯ
"1 GB storage" ಎಂಬುದು ಜಿಮ್ ಮಾಲೀಕನಿಗೆ ಏನೂ ಅರ್ಥ ನೀಡುವುದಿಲ್ಲ. "ವೃತ್ತಿಪರ ಸ್ಟುಡಿಯೋ ವೆಬ್‌ಸೈಟ್‌ಗೆ ನಿಮಗೆ ಬೇಕಾದ ಎಲ್ಲವೂ" ಎಂಬುದು ಎಲ್ಲದಕ್ಕೂ ಅರ್ಥ. ನಿಮ್ಮ ಗ್ರಾಹಕರ ಭಾಷೆಯಲ್ಲಿ ಯೋಜನೆ ವಿವರಣೆಗಳನ್ನು ಬರೆಯಿರಿ.
:::

### ಹಂತ 2: ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆ (Template Selection)

ಯೋಜನೆಯನ್ನು ಆರಿಸಿದ ನಂತರ, ಗ್ರಾಹಕರು ತಮ್ಮ ಆರಂಭಿಕ ಟೆಂಪ್ಲೇಟ್ ಅನ್ನು ಆಯ್ಕೆ ಮಾಡುತ್ತಾರೆ.

- **Template Selection** ಫೀಲ್ಡ್ ಅನ್ನು ಸೇರಿಸಿ
- ಲಭ್ಯವಿರುವ ಟೆಂಪ್ಲೇಟ್‌ಗಳನ್ನು ಅವರು ಆರಿಸಿದ ಯೋಜನೆಯ ಮೂಲಕ ಫಿಲ್ಟರ್ ಮಾಡಲಾಗುತ್ತದೆ (ಪಾಠ 5 ರಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಲಾಗಿದೆ)
- ಪ್ರತಿ ಟೆಂಪ್ಲೇಟ್‌ಗೂ ಫಿಟ್‌ನೆಸ್-ನಿರ್ದಿಷ್ಟ ವಿನ್ಯಾಸವನ್ನು ತೋರಿಸುವ ಪ್ರಿವ್ಯೂ ಚಿತ್ರ ಇರಬೇಕು

### ಹಂತ 3: ಖಾತೆ ರಚನೆ (Account Creation)

ಇದನ್ನು ಕನಿಷ್ಠವಾಗಿ ಇರಿಸಿ. ಕೇವಲ ಇವುಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ:

- ಇಮೇಲ್ ವಿಳಾಸ (Email address)
- ಪಾಸ್‌ವರ್ಡ್ (Password)
- ಸ್ಟುಡಿಯೋ/ವ್ಯವಹಾರದ ಹೆಸರು (Studio/business name) (ಇದು ಅವರ ಸೈಟ್ ಹೆಸರಾಗುತ್ತದೆ)

ನೋಂದಣಿಯ ಸಮಯದಲ್ಲಿ ನಿಮಗೆ ಅಗತ್ಯವಿಲ್ಲದ ಮಾಹಿತಿಯನ್ನು ಕೇಳಬೇಡಿ. ಪ್ರತಿ ಹೆಚ್ಚುವರಿ ಫೀಲ್ಡ್ ಮಾರಾಟದ ಸಾಧ್ಯತೆಯನ್ನು (conversions) ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.

### ಹಂತ 4: ಸೈಟ್ ಸೆಟಪ್ (Site Setup)

- **Site title**: ಹಂತ 3 ರಲ್ಲಿ ನಮೂದಿಸಿದ ಸ್ಟುಡಿಯೋ ಹೆಸರಿನಿಂದ ಪೂರ್ವ-ಭರ್ತಿ (Pre-fill) ಮಾಡಿ
- **Site URL**: ಸ್ಟುಡಿಯೋ ಹೆಸರಿನಿಂದ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಿ (ಉದಾಹರಣೆಗೆ, `ironworks.fitsite.com`)

### ಹಂತ 5: ಪಾವತಿ (Payment)

- **Payment** ಫೀಲ್ಡ್ ಅನ್ನು ಸೇರಿಸಿ
- ನಿಮ್ಮ ಪಾವತಿ ಗೇಟ್‌ವೇ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ ([Stripe](/user-guide/payment-gateways/stripe) ಚಂದಾ ಶುಲ್ಕದ ಬಿಲ್ಲಿಂಗ್‌ಗೆ ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)
- ನೀವು ಪಾಠ 5 ರಲ್ಲಿ ಆರ್ಡರ್ ಬಂಪ್‌ಗಳನ್ನು (order bumps) ರಚಿಸಿದ್ದರೆ, ಪಾವತಿ ಹಂತದ ಮೊದಲು **Order Bump** ಫೀಲ್ಡ್ ಅನ್ನು ಸೇರಿಸಿ

### ಹಂತ 6: confirmation (ಖಚಿತಪಡಿಸುವಿಕೆ)

- ಫಿಟ್‌ನೆಸ್-ನಿರ್ದಿಷ್ಟ ಭಾಷೆಯೊಂದಿಗೆ confirmation ಸಂದೇಶವನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡಿ
- ಉದಾಹರಣೆ: "ನಿಮ್ಮ ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋ ವೆಬ್‌ಸೈಟ್ ಅನ್ನು ರಚಿಸಲಾಗುತ್ತಿದೆ. ಕೆಲವು ಸೆಕೆಂಡುಗಳಲ್ಲಿ ನಿಮ್ಮ ಹೊಸ ಸೈಟ್‌ಗೆ ನಿಮ್ಮನ್ನು ಮರುನಿರ್ದೇಶಿಸಲಾಗುತ್ತದೆ."

## périod ಆಯ್ಕೆ ಟಾಗ್ಲ್ ಸೇರಿಸುವುದು

ನೀವು ನಿಮ್ಮ ಯೋಜನೆಗಳಲ್ಲಿ ಬೆಲೆ ವ್ಯತ್ಯಾಸಗಳನ್ನು (monthly vs. annual) ಹೊಂದಿದ್ದರೆ, ಗ್ರಾಹಕರು ಬಿಲ್ಲಿಂಗ್ ಅವಧಿಗಳ ನಡುವೆ ಟಾ글 ಮಾಡಲು ಚೆಕ್ಅೌಟ್ ಫಾರ್ಮ್‌ಗೆ **Period Selection** ಫೀಲ್ಡ್ ಅನ್ನು ಸೇರಿಸಿ. ಸೂಚನೆಗಳಿಗಾಗಿ [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle) ನೋಡಿ.

## ಪಾವತಿ ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು

ನೀವು ಇನ್ನೂ ಪಾವತಿ ಗೇಟ್‌ವೇ ಅನ್ನು ಸೆಟಪ್ ಮಾಡದಿದ್ದರೆ:

1. **Ultimate Multisite > Settings > Payment Gateways** ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ
2. [Stripe setup guide](/user-guide/payment-gateways/stripe) ಅಥವಾ ನಿಮ್ಮ ಆದ್ಯತೆಯ ಗೇಟ್‌ವೇ ಅನ್ನು ಅನುಸರಿಸಿ
3. ಟೆಸ್ಟ್ ಪಾವತಿಯೊಂದಿಗೆ ಸಂಪೂರ್ಣ ಚೆಕ್ಅೌಟ್ ಫ್ಲೋ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ

ಪಾವತಿಗಳು ನಿಮ್ಮ ಖಾತೆಗೆ ಹೇಗೆ ಹರಿಯುತ್ತವೆ ಎಂಬುದರ ವಿವರಗಳಿಗಾಗಿ [Getting Paid](/user-guide/payment-gateways/getting-paid) ನೋಡಿ.

## ಫ್ಲೋ ಅನ್ನು ಪರೀಕ್ಷಿಸುವುದು (Testing the Flow)

ಮುಂದಕ್ಕೆ ಹೋಗುವ ಮೊದಲು, ಸಂಪೂರ್ಣ ಟೆಸ್ಟ್ ನೋಂದಣಿಯನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ:

1. ಇನ್ಕಾಗ್ನito/ಪ್ರೈವೇಟ್ ಬ್ರೌಸರ್ ವಿಂಡೋದಲ್ಲಿ ಚೆಕ್ಅೌಟ್ ಫಾರ್ಮ್ ತೆರೆಯಿರಿ
2. ಒಂದು ಯೋಜನೆಯನ್ನು ಆರಿಸಿ
3. ಒಂದು ಟೆಂಪ್ಲೇಟ್ ಆರಿಸಿ
4. ಖಾತೆಯನ್ನು ರಚಿಸಿ
5. ಪಾವತಿಯನ್ನು ಪೂರ್ಣಗೊಳಿಸಿ (ಟೆಸ್ಟ್ ಮೋಡ್ ಬಳಸಿ)
6. ಸೈಟ್ ಸರಿಯಾದ ಟೆಂಪ್ಲೇಟ್‌ನೊಂದಿಗೆ ರಚನೆಯಾಗಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಿ

ಇದನ್ನು ಪರಿಶೀಲಿಸಿ:

- [ ] ಯೋಜನೆ ವಿವರಣೆಗಳು ಸ್ಪಷ್ಟವಾಗಿ ಮತ್ತು ನಿರ್ದಿಷ್ಟ ವಲಯಕ್ಕೆ ಸಂಬಂಧಿಸಿವೆಯೇ
- [ ] ಟೆಂಪ್ಲೇಟ್ ಪ್ರಿವ್ಯೂಗಳು ಫಿಟ್‌ನೆಸ್‌ಗೆ ಸೂಕ್ತವಾದ ವಿನ್ಯಾಸಗಳನ್ನು ತೋರಿಸುತ್ತವೆಯೇ
- [ ] ಸೈಟ್ URL ಸ್ಟುಡಿಯೋ ಹೆಸರಿನಿಂದ ಸರಿಯಾಗಿ ರಚನೆಯಾಗುತ್ತದೆಯೇ
- [ ] ಪಾವತಿ ಯಶಸ್ವಿಯಾಗಿ ಪ್ರಕ್ರಿಯೆಗೊಳ್ಳುತ್ತದೆಯೇ
- [ ] ಗ್ರಾಹಕರು ಆರಿಸಿದ ಟೆಂಪ್ಲೇಟ್‌ನೊಂದಿಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಸೈಟ್‌ಗೆ ತಲುಪುತ್ತಾರೆಯೇ
- [ ] confirmation ಇಮೇಲ್‌ಗಳು ಫಿಟ್‌ನೆಸ್-ನಿರ್ದಿಷ್ಟ ಭಾಷೆಯನ್ನು ಬಳಸುತ್ತವೆಯೇ

## FitSite ನೆಟ್‌ವರ್ಕ್ ಇಲ್ಲಿಯವರೆಗೆ

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow
│   ├── Plan selection with niche-specific descriptions
│   ├── Template selection with fitness previews
│   ├── Minimal account creation
│   ├── Payment via Stripe
│   └── Fitness-specific confirmation
└── Ready for branding (next lesson)
```

## ಈ ಪಾಠದಲ್ಲಿ ನಾವು ಏನು ನಿರ್ಮಿಸಿದೆವು

- ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋ ಮಾಲೀಕರಿಗೆ ಸೂಕ್ತವಾದ **multi-step ಚೆಕ್ಅೌಟ್ ಫಾರ್ಮ್**
- ನೋಂದಣಿ ಫ್ಲೋಾದ್ಯಂತ **ನಿರ್ದಿಷ್ಟ ವಲಯದ ಭಾಷೆ**
- **ಕನಿಷ್ಠ ಘರ್ಷಣೆ** -- ಕೇವಲ ಅಗತ್ಯ ಫೀಲ್ಡ್‌ಗಳು, ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಸೈಟ್‌ಗೆ ವೇಗದ ಮಾರ್ಗ
- ಟೆಸ್ಟ್ ಪರಿಶೀಲನೆಯೊಂದಿಗೆ **ಪಾವತಿ ಏಕೀಕರಣ**
- ಯೋಜನೆ ಆಯ್ಕೆಯಿಂದ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಸೈಟ್‌ವರೆಗಿನ **ಪರೀಕ್ಷಿತ end-to-end ಫ್ಲೋ**
