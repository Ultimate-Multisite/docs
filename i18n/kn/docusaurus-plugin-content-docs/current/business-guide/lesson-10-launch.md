# ಪಾಠ 10: ಲಾಂಚ್ ದಿನ

ಎಲ್ಲವೂ ನಿರ್ಮಿತವಾಗಿದೆ. ನೀವು ಬಾಗಿಲುಗಳನ್ನು ತೆರೆಯುವ ಮೊದಲು, ಈ ಪಾಠವು ಪ್ರೀ-ಲಾಂಚ್ ಚೆಕ್‌ಲಿಸ್ಟ್ ಮೂಲಕ ಹೋಗುತ್ತದೆ, ಇದರಿಂದ ಏನೂ ಮುರಿದಿಲ್ಲ, ಏನೂ ಕಾಣೆಯಾಗಿಲ್ಲ ಅಥವಾ ಏನೂ ಮುಜುಗರದಾಯಕವಾಗಿಲ್ಲ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬಹುದು.

## ನಾವು ಎಲ್ಲಿ ನಿಲ್ಲಿಸಿದ್ದೆವು {#where-we-left-off}

FitSite ನಲ್ಲಿ ಟೆಂಪ್ಲೇಟ್‌ಗಳು, ಯೋಜನೆಗಳು (plans), ಚೆಕ್‌ಔಟ್, ಬ್ರ್ಯಾಂಡಿಂಗ್, ಆನ್‌ಬೋರ್ಡಿಂಗ್ ಮತ್ತು ಬೆಲೆ ನಿಗದಿಪಡಿಸುವಿಕೆ (pricing) ಎಲ್ಲವೂ ಕಾನ್ಫಿಗರ್ ಆಗಿವೆ. ಈಗ ನಾವು ಎಲ್ಲವೂ ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ ಲೈವ್ ಆಗುತ್ತೇವೆ.

## ಪ್ರೀ-ಲಾಂಚ್ ಚೆಕ್‌ಲಿಸ್ಟ್ {#pre-launch-checklist}

ಪ್ರತಿಯೊಂದು ಅಂಶದ ಮೂಲಕ ಹೋಗಿ. ಯಾವುದನ್ನೂ ಬಿಡಬೇಡಿ.

### ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಮೂಲಸೌಕರ್ಯ (Platform Infrastructure) {#platform-infrastructure}

- [ ] ಹೋಸ್ಟಿಂಗ್ ಸ್ಥಿರವಾಗಿದೆಯೇ ಮತ್ತು ಭಾರ ಹೊತ್ತಾಗಲೂ ಉತ್ತಮವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆಯೇ
- [ ] ವೈಲ್ಡ್‌ಕಾರ್ಡ್ SSL ಸಕ್ರಿಯವಾಗಿದೆಯೇ ಮತ್ತು ಎಲ್ಲಾ ಸಬ್‌ಡೊಮೇನ್‌ಗಳು HTTPS ಮೂಲಕ ಸೇವೆ ಸಲ್ಲಿಸುತ್ತವೆಯೇ
- [ ] ಡೊಮೇನ್ ಮ್ಯಾಪಿಂಗ್ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೇ -- ಒಂದು ಸೈಟ್ ರಚಿಸಿ ಮತ್ತು ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಮ್ಯಾಪ್ ಮಾಡಲು ಪರೀಕ್ಷಿಸಿ
- [ ] ಬ್ಯಾಕಪ್‌ಗಳು ಕಾನ್ಫಿಗರ್ ಆಗಿವೆ ಮತ್ತು ಪರೀಕ್ಷಿಸಲಾಗಿದೆ (ಕನಿಷ್ಠ ಒಂದನ್ನು ಮರುಸ್ಥಾಪಿಸಿ ಪರಿಶೀಲಿಸಲು)
- [ ] ಮೇಲ್ವಿಚಾರಣೆ (Monitoring) ವ್ಯವಸ್ಥೆ ಇದೆ -- ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಕೆಳಗೆ ಬಿದ್ದರೆ ನಿಮಗೆ ತಿಳಿಯುತ್ತದೆ

### ಟೆಂಪ್ಲೇಟ್‌ಗಳು (Templates) {#templates}

- [ ] ಎಲ್ಲಾ ಮೂರು ಟೆಂಪ್ಲೇಟ್‌ಗಳು ಹೊಸ ಸೈಟ್‌ಗಳಲ್ಲಿ ಸರಿಯಾಗಿ ಲೋಡ್ ಆಗುತ್ತವೆಯೇ
- [ ] ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್ ಕಂಟೆಂಟ್ ಸಹಾಯಕವಾಗಿದೆಯೇ ಮತ್ತು ಟೈಪೋಗಳಿಂದ ಮುಕ್ತವಾಗಿದೆಯೇ
- [ ] ಎಲ್ಲಾ ಚಿತ್ರಗಳು ಸರಿಯಾಗಿ ಪರವಾನಗಿ ಪಡೆದಿವೆಯೇ (ಸ್ಟಾಕ್ ಫೋಟೋ ವಾಟರ್‌ಮಾರ್ಕ್‌ಗಳಿಲ್ಲ)
- [ ] ಪ್ರತಿ ಟೆಂಪ್ಲೇಟ್ ಪುಟದಲ್ಲಿ ಮೊಬೈಲ್ ರೆಸ್ಪಾನ್ಸಿವ್‌ನೆಸ್ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೇ
- [ ] ಪುಟ ಲೋಡ್ ವೇಗ ಸ್ವೀಕಾರಾರ್ಹವೇ (GTmetrix ಅಥವಾ PageSpeed Insights ನಂತಹ ಟೂಲ್ ಬಳಸಿ ಪರೀಕ್ಷಿಸಿ)
- [ ] ಯಾವುದೇ ಟೆಂಪ್ಲೇಟ್‌ನಲ್ಲಿ ಒಡೆದ ಲಿಂಕ್‌ಗಳು ಅಥವಾ ಕಾಣೆಯಾದ ಆಸ್ತಿಗಳು ಇಲ್ಲವೇ

### ಯೋಜನೆಗಳು ಮತ್ತು ಉತ್ಪನ್ನಗಳು (Plans and Products) {#plans-and-products}

- [ ] ಎಲ್ಲಾ ಮೂರು ಯೋಜನೆಗಳು ಸಕ್ರಿಯವಾಗಿ ಮತ್ತು ಗೋಚರಿಸುತ್ತವೆಯೇ
- [ ] ಯೋಜನೆ ವಿವರಣೆಗಳು ನಿಖರವಾಗಿದೆಯೇ ಮತ್ತು ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರಕ್ಕೆ ಸಂಬಂಧಿಸಿವೆಯೇ
- [ ] ಬೆಲೆ ನಿಗದಿಪಡಿಸುವಿಕೆ ಸರಿಯೇ (ಮಾಸಿಕ ಮತ್ತು ವಾರ್ಷಿಕ)
- [ ] ಸೆಟಪ್ ಶುಲ್ಕಗಳು ಸರಿಯಾದ ಯೋಜನೆಗಳಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಆಗಿವೆಯೇ
- [ ] ಸ್ಟಾರ್ಟರ್ ಪ್ಲಾನ್‌ನಲ್ಲಿ ಟ್ರಯಲ್ ಅವಧಿ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೇ
- [ ] ಚೆಕ್‌ಔಟ್ ಸಮಯದಲ್ಲಿ ಆರ್ಡರ್ ಬಂಪ್‌ಗಳು ಸರಿಯಾಗಿ ಕಾಣಿಸುತ್ತವೆಯೇ
- [ ] ಯೋಜನೆಗಳ ಪ್ರಕಾರ ಪ್ಲಗ್ಇನ್ ಮತ್ತು ಥೀಮ್ ಮಿತಿಗಳನ್ನು ಸರಿಯಾಗಿ ಜಾರಿಗೊಳಿಸಲಾಗಿದೆಯೇ

### ಚೆಕ್‌ಔಟ್ ಫ್ಲೋ (Checkout Flow) {#checkout-flow}

- [ ] ಪ್ರತಿ ಯೋಜನೆಯ ಮೇಲೆ ಸಂಪೂರ್ಣ ಪರೀಕ್ಷಾ ನೋಂದಣಿ ಮಾಡಿ (ಪರೀಕ್ಷಾ ಪಾವತಿ ಮೋಡ್ ಬಳಸಿ)
- [ ] ಟೆಂಪ್ಲೇಟ್ ಆಯ್ಕೆಯು ಯೋಜನೆಯ ಪ್ರಕಾರ ಸರಿಯಾದ ಟೆಂಪ್ಲೇಟ್‌ಗಳನ್ನು ತೋರಿಸುತ್ತದೆಯೇ
- [ ] ಪಾವತಿ ಪ್ರಕ್ರಿಯೆ ಯಶಸ್ವಿಯಾಗಿ ನಡೆಯುತ್ತದೆಯೇ
- [ ] ನೋಂದಣಿಯ ನಂತರ ಗ್ರಾಹಕರಿಗೆ ಸ್ವಾಗತ ಇಮೇಲ್ ಸಿಗುತ್ತದೆಯೇ
- [ ] ಸರಿಯಾದ ಟೆಂಪ್ಲೇಟ್‌ನೊಂದಿಗೆ ಹೊಸ ಸೈಟ್ ರಚನೆಯಾಗುತ್ತದೆಯೇ
- [ ] ಗ್ರಾಹಕರು ತಕ್ಷಣವೇ ತಮ್ಮ ಹೊಸ ಸೈಟ್‌ಗೆ ಲಾಗ್ ಇನ್ ಆಗಬಹುದೇ
- [ ] ರಿಯಾಯಿತಿ ಕೋಡ್‌ಗಳು ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತವೆಯೇ

### ಬ್ರ್ಯಾಂಡಿಂಗ್ (Branding) {#branding}

- [ ] ಲಾಗಿನ್ ಪುಟದಲ್ಲಿ FitSite ಬ್ರ್ಯಾಂಡಿಂಗ್ ಕಾಣಿಸುತ್ತದೆಯೇ
- [ ] ಅಡ್ಮಿನ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ನಲ್ಲಿ FitSite ಬ್ರ್ಯಾಂಡಿಂಗ್ ಕಾಣಿಸುತ್ತದೆಯೇ
- [ ] ಎಲ್ಲಾ ಸಿಸ್ಟಮ್ ಇಮೇಲ್‌ಗಳು FitSite ಬ್ರ್ಯಾಂಡಿಂಗ್ ಮತ್ತು ಫಿಟ್‌ನೆಸ್-ನಿರ್ದಿಷ್ಟ ಭಾಷೆಯನ್ನು ಬಳಸುತ್ತವೆಯೇ
- [ ] ಇನ್‌ವಾಯ್ಸ್‌ಗಳು ನಿಮ್ಮ ವ್ಯಾಪಾರ ವಿವರಗಳೊಂದಿಗೆ ಸರಿಯಾಗಿ ಪ್ರದರ್ಶನಗೊಳ್ಳುತ್ತವೆಯೇ
- [ ] ಮಾರ್ಕೆಟಿಂಗ್ ಸೈಟ್ ಲೈವ್ ಆಗಿದೆ ಮತ್ತು ಚೆಕ್‌ಔಟ್ ಫಾರ್ಮ್‌ಗೆ ಲಿಂಕ್ ಮಾಡಿದೆ

### ಆನ್‌ಬೋರ್ಡಿಂಗ್ (Onboarding) {#onboarding}

- [ ] ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ವಿಜೆಟ್ ಹೊಸ ಗ್ರಾಹಕರ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ಗಳಲ್ಲಿ ಕಾಣಿಸುತ್ತದೆಯೇ
- [ ] ಎಲ್ಲಾ ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ಲಿಂಕ್‌ಗಳು ಸರಿಯಾದ ಪುಟಗಳಿಗೆ ಕೊಂಡೊಯ್ಯುತ್ತವೆಯೇ
- [ ] ಸ್ವಾಗತ ಇಮೇಲ್ ಸೀಕ್ವೆನ್ಸ್ ಕಾನ್ಫಿಗರ್ ಮತ್ತು ಪರೀಕ್ಷಿಸಲಾಗಿದೆಯೇ
- [ ] ಜ್ಞಾನದ ಮೂಲದ ಲೇಖನಗಳು ಪ್ರಕಟವಾಗಿ ಮತ್ತು ಲಭ್ಯವಾಯಿತೇ
- [ ] ಖಾತೆ ಪುಟವು ಸರಿಯಾದ ಯೋಜನೆ ಮಾಹಿತಿ ಮತ್ತು ಅಪ್‌ಗ್ರೇಡ್ ಆಯ್ಕೆಗಳನ್ನು ತೋರಿಸುತ್ತದೆಯೇ

### ಕಾನೂನು ಮತ್ತು ವ್ಯಾಪಾರ (Legal and Business) {#legal-and-business}

- [ ] ಸೇವಾ ನಿಯಮಗಳು (Terms of service) ಪ್ರಕಟವಾಗಿ ಮತ್ತು ಚೆಕ್‌ಔಟ್‌ನಿಂದ ಲಿಂಕ್ ಆಗಿವೆಯೇ
- [ ] ಗೌಪ್ಯತಾ ನೀತಿಯು (Privacy policy) ಪ್ರಕಟವಾಗಿ ಮತ್ತು ಲಭ್ಯವಾಯಿತೇ
- [ ] ಮರುಪಾವತಿ ನೀತಿಯು (Refund policy) ವ್ಯಾಖ್ಯಾನಿಸಿ ಮತ್ತು ದಾಖಲಿಸಲಾಗಿದೆಯೇ
- [ ] ಪಾವತಿಗಳನ್ನು ಸ್ವೀಕರಿಸಲು ವ್ಯಾಪಾರ ಘಟಕವನ್ನು (Business entity) ಹೊಂದಿಸಲಾಗಿದೆಯೇ
- [ ] ಪಾವತಿ ಗೇಟ್‌ವೇ ಲೈವ್ ಮೋಡ್‌ನಲ್ಲಿ ಇದೆ (ಪರೀಕ್ಷಾ ಮೋಡ್‌ನಲ್ಲಿಲ್ಲ)
- [ ] ನಿಮ್ಮ ನ್ಯಾಯವ್ಯಾಪ್ತಿಗಾಗಿ ತೆರಿಗೆ ಕಾನ್ಫಿಗರೇಶನ್ ಸರಿಯಾಗಿದೆಯೇ

## ಸಾಫ್ಟ್ ಲಾಂಚ್ vs. ಹಾರ್ಡ್ ಲಾಂಚ್ {#soft-launch-vs-hard-launch}

ಎರಡು ಹಂತಗಳ ಲಾಂಚ್ ಅನ್ನು ಪರಿಗಣಿಸಿ:

### ಹಂತ 1: ಸಾಫ್ಟ್ ಲಾಂಚ್ (Soft Launch) {#phase-1-soft-launch}

ಸಾರ್ವಜನಿಕ ಲಾಂಚ್‌ಗೂ ಮುನ್ನ 5-10 ಫಿಟ್‌ನೆಸ್ ಸ್ಟುಡಿಯೋ ಮಾಲೀಕರಿಗೆ ನೋಂದಾಯಿಸಲು ಆಹ್ವಾನಿಸಿ. ಇವರು ನಿಮ್ಮ பீಟಾ ಗ್ರಾಹಕರು. ಅವರಿಗೆ ಈ ಬದಲಾವಣೆಯ ಬದಲಾಗಿ ಗಮನಾರ್ಹ ರಿಯಾಯಿತಿ (ಜೀವಮಾನಕ್ಕೆ 50% ರಿಯಾಯಿತಿ, ಅಥವಾ 3 ತಿಂಗಳುಗಳು ಉಚಿತ) ನೀಡಿ:

- ನೋಂದಣಿ ಮತ್ತು ಆನ್‌ಬೋರ್ಡಿಂಗ್ ಅನುಭವದ ಕುರಿತು ಪ್ರಾಮಾಣಿಕ ಪ್ರತಿಕ್ರಿಯೆ
- ಅವರ ಸೈಟ್ ಅನ್ನು ಪ್ರದರ್ಶನ ಉದಾಹರಣೆಯಾಗಿ ಬಳಸಲು ಅನುಮತಿ
- ಅವರು ಎದುರಿಸುವ ಯಾವುದೇ ಬಗ್‌ಗಳು ಅಥವಾ ಸಮಸ್ಯೆಗಳನ್ನು ವರದಿ ಮಾಡುವುದು

ಇದು ನಿಮಗೆ ನಿಜವಾದ ಗ್ರಾಹಕರ ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಸಾರ್ವಜನಿಕರ ಮುಂದೆ ಪ್ರದರ್ಶಿಸಲು ಲೈವ್ ಸೈಟ್‌ಗಳನ್ನು ನೀಡುತ್ತದೆ.

### ಹಂತ 2: ಸಾರ್ವಜನಿಕ ಲಾಂಚ್ (Public Launch) {#phase-2-public-launch}

ಸಾಫ್ಟ್ ಲಾಂಚ್ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಅಳವಡಿಸಿಕೊಂಡ ನಂತರ:

- ಪಾವತಿ ಗೇಟ್‌ವೇ ಅನ್ನು ಲೈವ್ ಮೋಡ್‌ಗೆ ಬದಲಾಯಿಸಿ
- ನಿಮ್ಮ ಮಾರ್ಕೆಟಿಂಗ್ ಸೈಟ್ ಅನ್ನು ಪ್ರಕಟಿಸಿ
- ಗ್ರಾಹಕ ಸ್ವಾಧೀನವನ್ನು ಪ್ರಾರಂಭಿಸಿ (ಪಾಠ 11)
- ಸಂಬಂಧಿತ ಫಿಟ್‌ನೆಸ್ ಉದ್ಯಮ ಚಾನೆಲ್‌ಗಳಲ್ಲಿ ಘೋಷಿಸಿ

## ಲಾಂಚ್ ದಿನದ ಕ್ರಿಯೆಗಳು {#launch-day-actions}

ನೀವು ಸಾರ್ವಜನಿಕರಾಗುವ ದಿನದಂದು:

1. **ಲೈವ್ ಪಾವತಿಗಳಿಗೆ ಬದಲಾಯಿಸಿ** -- ನಿಮ್ಮ ಪಾವತಿ ಗೇಟ್‌ವೇ ಮೇಲೆ ಪರೀಕ್ಷಾ ಮೋಡ್ ಅನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಿ
2. **ಮತ್ತೊಮ್ಮೆ ಪರಿಶೀಲಿಸಿ** -- ನಿಜವಾದ ಪಾವತಿಯೊಂದಿಗೆ ಸಂಪೂರ್ಣ ಪರೀಕ್ಷಾ ನೋಂದಣಿ ಮಾಡಿ (ನಂತರ ನಿಮಗೆ ಮರುಪಾವತಿ ಮಾಡಿ)
3. **ಗಮನಿಸುತ್ತಿರಿ** -- ದೋಷಗಳು, ವಿಫಲ ನೋಂದಣಿಗಳು ಅಥವಾ ಪಾವತಿ ಸಮಸ್ಯೆಗಳಿಗಾಗಿ ಗಮನಿಸಿ
4. **ಲಭ್ಯವಿರಿ** -- ನಿಮ್ಮ ಮೊದಲ ನಿಜವಾದ ಗ್ರಾಹಕರಿಗೆ ಸಹಾಯ ಬೇಕಾಗಬಹುದು, ಮತ್ತು ವೇಗದ ಪ್ರತಿಕ್ರಿಯೆಯು ವಿಶ್ವಾಸವನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ
5. **ಸಣ್ಣದಾಗಿ ಆಚರಿಸಿ** -- ನಂತರ ಕೆಲಸಕ್ಕೆ ಮರಳಿಸಿ

## ಏನಾದರೂ ತಪ್ಪಾಗಬಹುದೇ? {#what-can-go-wrong}

ಇದಕ್ಕೆ ಸಿದ್ಧರಾಗಿರಿ:

- **ಪಾವತಿ ಗೇಟ್‌ವೇ ಸಮಸ್ಯೆಗಳು**: ನಿಮ್ಮ ಗೇಟ್‌ವೇ ಪೂರೈಕೆದಾರರ ಬೆಂಬಲ ಸಂಪರ್ಕವನ್ನು ಸಿದ್ಧವಾಗಿಡಿ
- **SSL ಪ್ರಮಾಣಪತ್ರ ಸಮಸ್ಯೆಗಳು**: ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಹೇಗೆ ಪರಿಶೀಲಿಸುವುದು ಮತ್ತು ನವೀಕರಿಸುವುದು ಎಂದು ತಿಳಿಯಿರಿ
- **ಇಮೇಲ್ ವಿತರಣಾ ವೈಫಲ್ಯಗಳು**: ಇಮೇಲ್‌ಗಳು ನಿಜವಾಗಿಯೂ ಬರುತ್ತವೆಯೇ ಎಂದು ಪರೀಕ್ಷಿಸಿ (ಸ್ಪ್ಯಾಮ್ ಫೋಲ್ಡರ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ)
- **ಭಾರದ ಅಡಿಯಲ್ಲಿ ಕಾರ್ಯಕ್ಷಮತೆ**: ನಿಮಗೆ ಟ್ರಾಫಿಕ್ ಏರಿಕೆ ಬಂದರೆ, ನಿಮ್ಮ ಹೋಸ್ಟಿಂಗ್ ಅನ್ನು ಹೇಗೆ ವಿಸ್ತರಿಸುವುದು ಎಂದು ತಿಳಿಯಿರಿ
- **ಗ್ರಾಹಕರ ಗೊಂದಲ**: ನಿಮ್ಮ ಜ್ಞಾನದ ಮೂಲ ಮತ್ತು ಬೆಂಬಲ ಚಾನೆಲ್‌ಗಳನ್ನು ಸಿದ್ಧವಾಗಿಡಿ
