# ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ (Ability Visibility)

Superdav AI Agent v1.12.0 ರಲ್ಲಿ **ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ (Ability Visibility)** ನಿಯಂತ್ರಣಗಳನ್ನು ಪರಿಚಯಿಸಲಾಗಿದೆ. ಇದು ಯಾವ ವೇದಿಕೆಗಳು (surfaces) ಯಾವ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತವೆ ಎಂಬುದನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ. ಇದರ ಮೂಲಕ ನಿರ್ವಾಹಕರು ಯಾವ ಏಜೆಂಟ್ ಸಾಮರ್ಥ್ಯಗಳು REST APIs, MCP ಸರ್ವರ್‌ಗಳು, WooCommerce ಇಂಟಿಗ್ರೇಷನ್‌ಗಳು ಮತ್ತು ಇತರ ಅಂತರ್ಫೇಸ್‌ಗಳ ಮೂಲಕ ಲಭ್ಯವಿರಬೇಕು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ಹೊಂದಿಸಬಹುದು.

## ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ ಎಂದರೇನು? {#what-is-ability-visibility}

ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ (Ability Visibility) ಎಂಬುದು ಒಂದು ಅನುಮತಿ ವ್ಯವಸ್ಥೆಯಾಗಿದ್ದು, ಇದು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ:

- ఏ ఏ ಸಾಮರ್ಥ್ಯಗಳು ಏಜೆಂಟ್‌ಗಳಿಗೆ ಲಭ್ಯವಿವೆ
- ಅವು ಎಲ್ಲಿ ಪ್ರದರ್ಶಿತವಾಗಿವೆ (REST API, MCP, WooCommerce, ಇತ್ಯಾದಿ)
- ಯಾರು ಅವುಗಳನ್ನು ಪ್ರವೇಶಿಸಬಹುದು (ಪartner allow-list ಮೂಲಕ)
- ಅವುಗಳನ್ನು ಹೇಗೆ ವರ್ಗೀಕರಿಸಲಾಗಿದೆ (recognised vs. unclassified)

ಪ್ರತಿಯೊಂದು ಸಾಮರ್ಥ್ಯವೂ ಒಂದು ಗೋಚರತಾ ಮಟ್ಟವನ್ನು ಹೊಂದಿರುತ್ತದೆ, ಇದು ವಿವಿಧ ವೇದಿಕೆಗಳಲ್ಲಿ ಅದರ ಲಭ್ಯತೆಯನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ.

## ಗೋಚರತಾ ಮಟ್ಟಗಳು (Visibility Levels) {#visibility-levels}

### ಸಾರ್ವಜನಿಕ (Public) {#public}

**ಸಾರ್ವಜನಿಕ ಸಾಮರ್ಥ್ಯಗಳು** ಎಲ್ಲೆಡೆ ಲಭ್ಯವಿರುತ್ತವೆ:

- REST API ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳು
- MCP ಸರ್ವರ್‌ಗಳು
- WooCommerce ಇಂಟಿಗ್ರೇಷನ್‌ಗಳು
- ಚಾಟ್ ಅಂತರ್ಫೇಸ್‌ಗಳು
- ಎಲ್ಲಾ ಬಳಕೆದಾರ ಪಾತ್ರಗಳು (ಸೂಕ್ತ ಅನುಮತಿಗಳೊಂದಿಗೆ)

ಉದಾಹರಣೆ: `scaffold-block-theme`, `activate-theme`, `send-email`

### ಆಂತರಿಕ (Internal) {#internal}

**ಆಂತರಿಕ ಸಾಮರ್ಥ್ಯಗಳು** ನಿಮ್ಮ WordPress installation ಒಳಗೆ ಮಾತ್ರ ಲಭ್ಯವಿರುತ್ತವೆ:

- ಚಾಟ್ ಅಂತರ್ಫೇಸ್‌ಗಳು
- ಆಡ್ಮಿನ್ ಪ್ಯಾನೆಲ್‌ಗಳು
- ಲಾಗ್ ಇನ್ ಆಗಿರುವ ಬಳಕೆದಾರರಿಗೆ ಮಾತ್ರ
- REST API ಅಥವಾ ಬಾಹ್ಯ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳ ಮೂಲಕ ಪ್ರದರ್ಶಿತವಾಗಿರುವುದಿಲ್ಲ

ಉದಾಹರಣೆ: `manage-settings`, `view-analytics`, `export-data`

### ಪಾಲುದಾರ (Partner) {#partner}

**ಪಾಲುದಾರ ಸಾಮರ್ಥ್ಯಗಳು** ಅನುಮತಿಸಿದ (whitelisted) ಪಾಲುದಾರರಿಗೆ ಮಾತ್ರ ಲಭ್ಯವಿರುತ್ತವೆ:

- ಸ್ಪಷ್ಟ ಪಾಲುದಾರ ಅನುಮತಿ ಪಟ್ಟಿಯಲ್ಲಿ (allow-list) ಸೇರ್ಪಡೆ ಅಗತ್ಯ
- ಪಾಲುದಾರರ credentials ಬಳಸಿ REST API ಮೂಲಕ ಲಭ್ಯ
- ಪಾಲುದಾರರ authentication ಬಳಸಿ MCP ಮೂಲಕ ಲಭ್ಯ
- ಸಾಮಾನ್ಯ ಬಳಕೆದಾರರಿಗೆ ಲಭ್ಯವಿಲ್ಲ

ಉದಾಹರಣೆ: `bulk-import-users`, `modify-billing`, `access-analytics`

### ನಿಷ್ಕ್ರಿಯ (Disabled) {#disabled}

**ನಿಷ್ಕ್ರಿಯ ಸಾಮರ್ಥ್ಯಗಳು** ಎಲ್ಲಿಯೂ ಲಭ್ಯವಿರುವುದಿಲ್ಲ:

- ಯಾವುದೇ ವೇದಿಕೆಯ ಮೂಲಕ ಪ್ರದರ್ಶಿತವಾಗಿಲ್ಲ
- ಚಾಟ್‌ನಲ್ಲಿ ಲಭ್ಯವಿಲ್ಲ
- ಆಡ್ಮಿನ್ ಪ್ಯಾನೆಲ್‌ಗಳಲ್ಲಿ ಲಭ್ಯವಿಲ್ಲ
- ಹಳೆಯ ಅಥವಾ ಪ್ರಾಯೋಗಿಕ ಸಾಮರ್ಥ್ಯಗಳಿಗೆ ಉಪಯುಕ್ತ

## ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆಯನ್ನು ನಿರ್ವಹಿಸುವುದು {#managing-ability-visibility}

### ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು அணுகುವುದು {#accessing-ability-visibility-settings}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** ಗೆ ಹೋಗಿ
2. **Abilities** ಟ್ಯಾಬ್‌ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
3. ಇಲ್ಲಿ ನೀವು ಎಲ್ಲಾ ಸ್ಥಾಪಿಸಲಾದ ಸಾಮರ್ಥ್ಯಗಳು ಮತ್ತು ಅವುಗಳ ಗೋಚರತಾ ಮಟ್ಟಗಳ ಪಟ್ಟಿಯನ್ನು ನೋಡುತ್ತೀರಿ

### ಸಾಮರ್ಥ್ಯದ ವಿವರಗಳನ್ನು ನೋಡುವುದು {#viewing-ability-details}

ಯಾವುದೇ ಸಾಮರ್ಥ್ಯದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಈ ಕೆಳಗಿನವುಗಳನ್ನು ನೋಡಿ:

- **Name**: ಸಾಮರ್ಥ್ಯದ ಗುರುತಿನ ಸಂಖ್ಯೆ (identifier)
- **Description**: ಸಾಮರ್ಥ್ಯ ಏನು ಮಾಡುತ್ತದೆ
- **Current visibility**: ಸಾರ್ವಜನಿಕ (Public), ಆಂತರಿಕ (Internal), ಪಾಲುದಾರ (Partner), ಅಥವಾ ನಿಷ್ಕ್ರಿಯ (Disabled)
- **Partner allow-list**: ಯಾವ ಪಾಲುದಾರರು ಪ್ರವೇಶಿಸಬಹುದು (ಪಾಲುದಾರ ಗೋಚರತೆಯ ಸಂದರ್ಭದಲ್ಲಿ)
- **Last updated**: ಗೋಚರತೆಯನ್ನು ಕೊನೆಯದಾಗಿ ಯಾವಾಗ ಬದಲಾಯಿಸಲಾಯಿತು
- **Status**: ಗುರುತಿಸಲಾಗಿದೆ (Recognized) ಅಥವಾ ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲ (Unclassified)

### ಗೋಚರತಾ ಮಟ್ಟಗಳನ್ನು ಬದಲಾಯಿಸುವುದು {#changing-visibility-levels}

ಯಾವುದೇ ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆಯನ್ನು ಬದಲಾಯಿಸಲು:

1. ಪಟ್ಟಿಯಲ್ಲಿರುವ ಸಾಮರ್ಥ್ಯದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
2. ಡ್ರಾಪ್‌ಡೌನ್‌ನಿಂದ ಹೊಸ ಗೋಚರತಾ ಮಟ್ಟವನ್ನು ಆರಿಸಿ
3. "Partner" ಆಯ್ಕೆ ಮಾಡಿದರೆ, ಅನುಮತಿ ಪಟ್ಟಿಗೆ (allow-list) ಪಾಲುದಾರರ ಗುರುತಿನ ಸಂಖ್ಯೆಗಳನ್ನು ಸೇರಿಸಿ
4. **Save** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

ಉದಾಹರಣೆ:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### ಬ್ಯಾಚ್ ಕಾರ್ಯಾಚರಣೆಗಳು (Bulk Operations) {#bulk-operations}

ಹಲವಾರು ಸಾಮರ್ಥ್ಯಗಳ ಗೋಚರತೆಯನ್ನು ಬದಲಾಯಿಸಲು:

1. ಸಾಮರ್ಥ್ಯಗಳ ಪಕ್ಕದಲ್ಲಿರುವ ಬಾಕ್ಸ್‌ಗಳನ್ನು ಟಿಕ್ ಮಾಡಿ
2. ಬ್ಯಾಚ್ ಆಕ್ಷನ್ ಡ್ರಾಪ್‌ಡೌನ್‌ನಿಂದ ಹೊಸ ಗೋಚರತಾ ಮಟ್ಟವನ್ನು ಆರಿಸಿ
3. **Apply** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

## ಪಾಲುದಾರ ಅನುಮತಿ ಪಟ್ಟಿ (Partner Allow-List) {#partner-allow-list}

**ಪಾಲುದಾರ ಅನುಮತಿ ಪಟ್ಟಿ (partner allow-list)** ಎಂಬುದು ಯಾವ ಬಾಹ್ಯ ಪಾಲುದಾರರು ಪಾಲುದಾರ-ಮಟ್ಟದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪ್ರವೇಶಿಸಬಹುದು ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ.

### ಪಾಲುದಾರರನ್ನು ಸೇರಿಸುವುದು {#adding-partners}

1. **Superdav AI Agent** → **Settings** → **Partners** ಗೆ ಹೋಗಿ
2. **Add Partner** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
3. ಪಾಲುದಾರರ ಗುರುತಿನ ಸಂಖ್ಯೆಯನ್ನು ನಮೂದಿಸಿ (ಸಾಮಾನ್ಯವಾಗಿ API key ಅಥವಾ organization ID)
4. ಐಚ್ಛಿಕವಾಗಿ ಪಾಲುದಾರರ ಹೆಸರು ಮತ್ತು ವಿವರಣೆಯನ್ನು ಸೇರಿಸಿ
5. **Save** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

### ಪಾಲುದಾರರಿಗೆ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನಿಯೋಜಿಸುವುದು {#assigning-abilities-to-partners}

ಒಬ್ಬ ಪಾಲುದಾರರನ್ನು ಸೇರಿಸಿದ ನಂತರ:

1. **Abilities** ಟ್ಯಾಬ್‌ಗೆ ಹೋಗಿ
2. ಪಾಲುದಾರ ಗೋಚರತೆಯನ್ನು ಹೊಂದಿರುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಹುಡುಕಿ
3. ಸಂಪಾದಿಸಲು ಕ್ಲಿಕ್ ಮಾಡಿ
4. "Partner allow-list" ವಿಭಾಗದಲ್ಲಿ, ಪ್ರವೇಶ ಹೊಂದಬೇಕಾದ ಪಾಲುದಾರರ ಬಾಕ್ಸ್‌ಗಳನ್ನು ಟಿಕ್ ಮಾಡಿ
5. **Save** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

### ಪಾಲುದಾರ ಪ್ರವೇಶವನ್ನು ಹಿಂತೆಗೆದುಕೊಳ್ಳುವುದು (Revoking Partner Access) {#revoking-partner-access}

ಒಬ್ಬ ಪಾಲುದಾರರ ಪ್ರವೇಶವನ್ನು ತೆಗೆದುಹಾಕಲು:

1. ಸಾಮರ್ಥ್ಯವನ್ನು ಸಂಪಾದಿಸಿ
2. ಅನುಮತಿ ಪಟ್ಟಿಯಲ್ಲಿ (allow-list) ಪಾಲುದಾರರ ಬಾಕ್ಸ್ ಅನ್ನು ಅನ್‌ಟಿಕ್ ಮಾಡಿ
3. **Save** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

ಪಾಲುದಾರರು ತಕ್ಷಣವೇ ಆ ಸಾಮರ್ಥ್ಯಕ್ಕೆ ಪ್ರವೇಶವನ್ನು ಕಳೆದುಕೊಳ್ಳುತ್ತಾರೆ.

## ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳು (Unclassified Abilities) {#unclassified-abilities}

ನೀವು Superdav AI Agent ಗುರುತಿಸದ ಮೂರನೇ ವ್ಯಕ್ತಿಯ ಸಾಮರ್ಥ್ಯವನ್ನು (third-party ability) ಸ್ಥಾಪಿಸಿದಾಗ, ಅದನ್ನು **ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲ (Unclassified)** ಎಂದು ಗುರುತಿಸಲಾಗುತ್ತದೆ.

### ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳಿಗೆ ಆಡ್ಮಿನ್ ನೋಟೀಸ್ {#admin-notice-for-unclassified-abilities}

ನೀವು ಈ ಆಡ್ಮಿನ್ ನೋಟೀಸ್ ಅನ್ನು ನೋಡುತ್ತೀರಿ:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದು {#reviewing-unclassified-abilities}

1. ನೋಟೀಸ್‌ನಲ್ಲಿ **Review Abilities** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ
2. ಪ್ರತಿ ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಕ್ಕಾಗಿ, ನೀವು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ನೋಡುತ್ತೀರಿ:
   - ಸಾಮರ್ಥ್ಯದ ಹೆಸರು ಮತ್ತು ವಿವರಣೆ
   - ಮೂಲ (Source) (ಯಾವ plugin/addon ಇದನ್ನು ಸ್ಥಾಪಿಸಿದೆ)
   - ಸೂಚಿಸಿದ ಗೋಚರತಾ ಮಟ್ಟ
   - ಭದ್ರತಾ ಮೌಲ್ಯಮಾಪನ (Security assessment)

3. ಗೋಚರತಾ ಮಟ್ಟವನ್ನು ಆರಿಸಿ:
   - **Public**: ಸಾಮರ್ಥ್ಯದ ಮೇಲೆ ನಂಬಿಕೆ ಇಡಿ; ಎಲ್ಲೆಡೆ ಪ್ರದರ್ಶಿಸಿ
   - **Internal**: ಆಂತರಿಕ ಬಳಕೆಗೆ ಮಾತ್ರ ಮಿತಿಗೊಳಿಸಿ
   - **Partner**: ನಿರ್ದಿಷ್ಟ ಪಾಲುದಾರರಿಗೆ ಮಿತಿಗೊಳಿಸಿ
   - **Disabled**: ಈ ಸಾಮರ್ಥ್ಯವನ್ನು ಬಳಸಬೇಡಿ

4. ನಿಮ್ಮ ನಿರ್ಧಾರವನ್ನು ಉಳಿಸಲು **Classify** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ

### ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಏಕೆ ವರ್ಗೀಕರಿಸಬೇಕು? {#why-classify-unclassified-abilities}

ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವರ್ಗೀಕರಿಸುವುದರಿಂದ:

- **ಭದ್ರತೆ ಸುಧಾರಿಸುತ್ತದೆ**: ನೀವು ಏನು ಪ್ರದರ್ಶಿಸಲಾಗಬೇಕು ಎಂಬುದನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ಅನುಮೋದಿಸುತ್ತೀರಿ
- **ಅಪಘಾತಗಳನ್ನು ತಡೆಯುತ್ತದೆ**: ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳು ಡೀಫಾಲ್ಟ್ ಆಗಿ ನಿಷ್ಕ್ರಿಯಗೊಂಡಿರುತ್ತವೆ
- **ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ**: ವರ್ಗೀಕರಿಸಿದ ನಂತರ, ಸಾಮರ್ಥ್ಯವು ಲಭ್ಯವಾಗುತ್ತದೆ
- **ನಿರ್ಧಾರಗಳನ್ನು ದಾಖಲಿಸುತ್ತದೆ**: ನಿಮ್ಮ ಆಯ್ಕೆಗಳನ್ನು ಆಡಿಟ್ ಉದ್ದೇಶಗಳಿಗಾಗಿ ದಾಖಲಿಸಲಾಗುತ್ತದೆ

## ವಿವಿಧ ವೇದಿಕೆಗಳಲ್ಲಿ ಗೋಚರತೆ (Visibility in Different Surfaces) {#visibility-in-different-surfaces}

### REST API {#rest-api}

**ಸಾರ್ವಜನಿಕ ಸಾಮರ್ಥ್ಯಗಳು** REST ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳ ಮೂಲಕ ಲಭ್ಯವಿರುತ್ತವೆ:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**ಆಂತರಿಕ ಮತ್ತು ಪಾಲುದಾರ ಸಾಮರ್ಥ್ಯಗಳು** REST API ಮೂಲಕ ಲಭ್ಯವಿರುವುದಿಲ್ಲ.

### MCP ಸರ್ವರ್‌ಗಳು {#mcp-servers}

**ಸಾರ್ವಜನಿಕ ಸಾಮರ್ಥ್ಯಗಳು** MCP ಮೂಲಕ ಲಭ್ಯವಿರುತ್ತವೆ:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**ಆಂತರಿಕ ಸಾಮರ್ಥ್ಯಗಳು** MCP ಮೂಲಕ ಪ್ರದರ್ಶಿತವಾಗಿರುವುದಿಲ್ಲ.

**ಪಾಲುದಾರ ಸಾಮರ್ಥ್ಯಗಳು** ಪಾಲುದಾರರ credentials ಯೊಂದಿಗೆ ಮಾತ್ರ ಲಭ್ಯ.

### WooCommerce ಇಂಟಿಗ್ರೇಷನ್ {#woocommerce-integration}

WooCommerce ಗೆ ಸಂಬಂಧಿಸಿದ **ಸಾರ್ವಜನಿಕ ಸಾಮರ್ಥ್ಯಗಳು** ಲಭ್ಯವಿರುತ್ತವೆ:

- ಉತ್ಪನ್ನ ನಿರ್ವಹಣೆ (Product management)
- ಆರ್ಡರ್ ಪ್ರಕ್ರಿಯೆ (Order processing)
- ಗ್ರಾಹಕ ಸಂವಹನ (Customer communication)

**ಆಂತರಿಕ ಸಾಮರ್ಥ್ಯಗಳು** WooCommerce ಗೆ ಪ್ರದರ್ಶಿತವಾಗಿರುವುದಿಲ್ಲ.

### ಚಾಟ್ ಅಂತರ್ಫೇಸ್ (Chat Interface) {#chat-interface}

**ಎಲ್ಲಾ ಸಾಮರ್ಥ್ಯಗಳು** (ಸಾರ್ವಜನಿಕ, ಆಂತರಿಕ, ಪಾಲುದಾರ) ಬಳಕೆದಾರರ ಅನುಮತಿಗಳ ಆಧಾರದ ಮೇಲೆ ಚಾಟ್‌ನಲ್ಲಿ ಲಭ್ಯವಿರುತ್ತವೆ:

- ನಿರ್ವಾಹಕರು ಎಲ್ಲಾ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನೋಡುತ್ತಾರೆ
- ಸಾಮಾನ್ಯ ಬಳಕೆದಾರರು ಸಾರ್ವಜನಿಕ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಮಾತ್ರ ನೋಡುತ್ತಾರೆ
- ಪಾಲುದಾರರು ಸಾರ್ವಜನಿಕ + ಪಾಲುದಾರ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನೋಡುತ್ತಾರೆ (whitelisted ಆಗಿದ್ದರೆ)

## ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು (Security Best Practices) {#security-best-practices}

### ಕನಿಷ್ಠ ಸೌಲಭ್ಯದ ತತ್ವ (Principle of Least Privilege) {#principle-of-least-privilege}

- ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಅವುಗಳ ಬಳಕೆಗೆ ಇನ್ನೂ ಅನುಮತಿಸುವ ಅತ್ಯಂತ ನಿರ್ಬಂಧಿತ ಗೋಚರತೆಯನ್ನು ಹೊಂದಿಸಿ
- ಸೂಕ್ಷ್ಮ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಪಾಲುದಾರ ಗೋಚರತೆಯನ್ನು ಬಳಸಿ
- ನೀವು ಬಳಸದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಿ

### ನಿಯಮಿತ ಆಡಿಟ್‌ಗಳು (Regular Audits) {#regular-audits}

- ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆಯನ್ನು ಮಾಸಿಕವಾಗಿ ಪರಿಶೀಲಿಸಿ
- ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳಿಗಾಗಿ ಪರಿಶೀಲಿಸಿ
- ಬಳಸದ ಪಾಲುದಾರರಿಗೆ ಪ್ರವೇಶವನ್ನು ತೆಗೆದುಹಾಕಿ

### ಲಾಗಿಂಗ್ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ (Logging and Monitoring) {#logging-and-monitoring}

- ಯಾವ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೆಚ್ಚು ಬಳಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ
- ಪಾಲುದಾರರ ಪ್ರವೇಶದ ಮಾದರಿಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ
- ಅಸಾಮಾನ್ಯ ಸಾಮರ್ಥ್ಯದ ಬಳಕೆಗೆ ಎಚ್ಚರಿಕೆ ನೀಡಿ

### ಮೂರನೇ ವ್ಯಕ್ತಿಯ ಸಾಮರ್ಥ್ಯಗಳು (Third-Party Abilities) {#third-party-abilities}

- ಸಕ್ರಿಯಗೊಳಿಸುವ ಮೊದಲು ಮೂರನೇ ವ್ಯಕ್ತಿಯ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
- ಅವುಗಳನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ವರ್ಗೀಕರಿಸಿ
- ಆಂತರಿಕ ಅಥವಾ ಪಾಲುದಾರ ಗೋಚರತೆಯಿಂದ ಪ್ರಾರಂಭಿಸಿ
- ಪರಿಶೀಲನೆಯ ನಂತರ ಮಾತ್ರ ಸಾರ್ವಜನಿಕಕ್ಕೆ ಪ್ರಚಾರ ಮಾಡಿ

## ಸಮಸ್ಯೆ ಪರಿಹಾರ (Troubleshooting) {#troubleshooting}

**ಸಾಮರ್ಥ್ಯ ಪಟ್ಟಿಯಲ್ಲಿ ಕಾಣಿಸುತ್ತಿಲ್ಲ**
- ಸಾಮರ್ಥ್ಯವನ್ನು ಸ್ಥಾಪಿಸಲಾಗಿದೆ ಮತ್ತು ಸಕ್ರಿಯವಾಗಿದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
- plugin/addon ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
- ಪುಟವನ್ನು ರಿಫ್ರೆಶ್ ಮಾಡಿ

**ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆಯನ್ನು ಬದಲಾಯಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತಿಲ್ಲ**
- ನಿಮಗೆ ನಿರ್ವಾಹಕ ಅನುಮತಿಗಳು (administrator permissions) ഉണ്ടെന്ന് ಪರಿಶೀಲಿಸಿ
- ಸಾಮರ್ಥ್ಯವನ್ನು plugin ನಿಂದ ಲಾಕ್ ಮಾಡಲಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ವಿರೋಧಾಭಾಸದ plugin ಗಳನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲು ಪ್ರಯತ್ನಿಸಿ

**ಪಾಲುದಾರನಿಗೆ ಸಾಮರ್ಥ್ಯವನ್ನು ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತಿಲ್ಲ**
- ಪಾಲುದಾರರು ಅನುಮತಿ ಪಟ್ಟಿಯಲ್ಲಿ (allow-list) ಇದ್ದಾರೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆ ಪಾಲುದಾರ (Partner) ಎಂದು ಹೊಂದಿಸಲಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಪಾಲುದಾರರ credentials ಸರಿಯಾಗಿವೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
- authentication ದೋಷಗಳಿಗಾಗಿ API logs ಅನ್ನು ಪರಿಶೀಲಿಸಿ

**ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳು ಕಾಣಿಸುತ್ತಲೇ ಇವೆ**
- ಆಡ್ಮಿನ್ ನೋಟೀಸ್‌ನಲ್ಲಿ ಅವುಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ವರ್ಗೀಕರಿಸಿ
- ನಿಮ್ಮ ವರ್ಗೀಕರಣವನ್ನು ಉಳಿಸಲಾಗಿದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಸಾಮರ್ಥ್ಯವನ್ನು ಒದಗಿಸುವ plugin ನ ಅಪ್‌ಡೇಟ್ ಆಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

## 레거ಸಿ ಮೋಡ್‌ನಿಂದ ವಲಸೆ (Migration from Legacy Mode) {#migration-from-legacy-mode}

ನೀವು Superdav AI Agent ನ ಹಳೆಯ ಆವೃತ್ತಿಯಿಂದ ಅಪ್‌เกรೇಡ್ ಮಾಡುತ್ತಿದ್ದರೆ:

- ಎಲ್ಲಾ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸಾರ್ವಜನಿಕ (Public) ಎಂದು ವರ್ಗೀಕರಿಸಲಾಗುತ್ತದೆ
- ಅಗತ್ಯವಿದ್ದರೆ ಗೋಚರತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಹೊಂದಿಸಿ
- ಪ್ರವೇಶವನ್ನು ನಿರ್ಬಂಧಿಸಲು ಬಯದಿದ್ದರೆ ಹೊರತುಪಡಿಸಿ ಯಾವುದೇ ಕ್ರಿಯೆ ಅಗತ್ಯವಿಲ್ಲ

Native Abilities API ಇಂಟಿಗ್ರೇಷನ್‌ಗೆ ಪರಿವರ್ತನೆ ಕುರಿತು ಹೆಚ್ಚಿನ ವಿವರಗಳಿಗಾಗಿ **Third-Party Mode Migration** ನೋಡಿ.

## ಮುಂದಿನ ಹಂತಗಳು (Next Steps) {#next-steps}

ಸಾಮರ್ಥ್ಯದ ಗೋಚರತೆಯನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ನಂತರ:

1. **ನಿಮ್ಮ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪರಿಶೀಲಿಸಿ**: ಸ್ಥಾಪಿಸಲಾದ ಎಲ್ಲಾ ಸಾಮರ್ಥ್ಯಗಳ ಆಡಿಟ್ ಮಾಡಿ
2. **ವರ್ಗೀಕರಿಸಲಾಗಿಲ್ಲದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವರ್ಗೀಕರಿಸಿ**: ಆಡ್ಮಿನ್ ನೋಟೀಸ್‌ಗಳಿಗೆ ಪ್ರತಿಕ್ರಿಯಿಸಿ
3. **ಪಾಲುದಾರರನ್ನು ಹೊಂದಿಸಿ**: ಪಾಲುದಾರ ಗೋಚರತೆಯನ್ನು ಬಳಸುತ್ತಿದ್ದರೆ ಪಾಲುದಾರರನ್ನು ಸೇರಿಸಿ
4. **ಬಳಕೆಯನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ**: ಯಾವ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೆಚ್ಚು ಬಳಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ
5. **ನಿರ್ಧಾರಗಳನ್ನು ದಾಖಲಿಸಿ**: ಪ್ರತಿ ಗೋಚರತಾ ಮಟ್ಟವನ್ನು ಏಕೆ ಆರಿಸಿದೆ ಎಂಬುದರ ಕುರಿತು ಟಿಪ್ಪಣಿಗಳನ್ನು ಇಟ್ಟುಕೊಳ್ಳಿ
