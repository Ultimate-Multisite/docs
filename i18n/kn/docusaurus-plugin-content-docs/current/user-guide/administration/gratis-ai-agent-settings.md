# Gratis AI Agent ಸೆಟ್ಟಿಂಗ್ಸ್

Gratis AI Agent ನಲ್ಲಿರುವ **Settings → Advanced** ಪರದೆಯು v1.5.0 ರಲ್ಲಿ ಪರಿಚಯಿಸಲಾದ ಬ್ಯಾಕ-ಎಂಡ್ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳಿಗೆ ಅಡ್ಮಿನಿಸ್ಟ್ರೇಟರ್ ಮಟ್ಟದ ಕಾನ್ಫಿಗರೇಶನ್ ಅನ್ನು ಒದಗಿಸುತ್ತದೆ. ಈ ಪುಟವು **Feedback Endpoint** ಫೀಲ್ಡ್‌ಗಳು ಮತ್ತು ಅವುಗಳ ನಿರೀಕ್ಷಿತ ಸ್ವರೂಪವನ್ನು ದಾಖಲಿಸುತ್ತದೆ.

## ಸೆಟ್ಟಿಂಗ್ಸ್‌ಗೆ ಪ್ರವೇಶಿಸುವುದು

1. WordPress ಅಡ್ಮಿನ್‌ನಲ್ಲಿ, **Gratis AI Agent → Settings** ಗೆ ಹೋಗಿ.
2. **Advanced** ಟ್ಯಾಬ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ.

## ಫೀಡ್‌ಬ್ಯಾಕ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಕಾನ್ಫಿಗರೇಶನ್

ಯೂಸರ್ಗಳು ಥಂಬ್ಸ್-ಡೌನ್ ಬಟನ್, ಆಟೋ-ಪ್ರಾಂಪ್ಟ್ 배ನರ್, ಅಥವಾ `/report-issue` ಕಮಾಂಡ್ ಮೂಲಕ ಫೀಡ್‌ಬ್ಯಾಕ್ ಸಲ್ಲಿಸಿದಾಗ, ಫೀಡ್‌ಬ್ಯಾಕ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ (feedback endpoint) ನಿಂದ POST ರಿಕ್ವೆಸ್ಟ್‌ಗಳನ್ನು ಸ್ವೀಕರಿಸಲಾಗುತ್ತದೆ.

| ಫೀಲ್ಡ್ | ವಿವರಣೆ |
|---|---|
| **Feedback Endpoint URL** | JSON ಬಾಡಿ ಜೊತೆಗೆ HTTP POST ರಿಕ್ವೆಸ್ಟ್‌ಗಳ ರೂಪದಲ್ಲಿ ಫೀಡ್‌ಬ್ಯಾಕ್ ಸಲ್ಲಿಸುವ URL. |
| **Feedback API Key** | ಪ್ರತಿ ಫೀಡ್‌ಬ್ಯಾಕ್ ರಿಕ್ವೆಸ್ಟ್‌ನ `Authorization` ಹೆಡರ್‌ನಲ್ಲಿ ಕಳುಹಿಸಲಾದ ಬೇರಿئر ಟೋಕನ್ (bearer token). ನಿಮ್ಮ ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ ಆಥೆಂಟಿಕೇಶನ್ ಅಗತ್ಯವಿಲ್ಲದಿದ್ದರೆ ಇದನ್ನು ಖಾಲಿ ಬಿಡಿ. |

### ನಿರೀಕ್ಷಿತ JSON ಪೇಲೋಡ್

ನಿಮ್ಮ ಫೀಡ್‌ಬ್ಯಾಕ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಕನಿಷ್ಠ ಈ ಕೆಳಗಿನ ಫೀಲ್ಡ್‌ಗಳನ್ನು ಹೊಂದಿರುವ JSON ಬಾಡಿಯನ್ನು ಸ್ವೀಕರಿಸಬೇಕು:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

ಸಂಭಾಷಣೆಯ ಸಂದರ್ಭವನ್ನು ಅವಲಂಬಿಸಿ ಪೇಲೋಡ್‌ನಲ್ಲಿ ಹೆಚ್ಚುವರಿ ಫೀಲ್ಡ್‌ಗಳು ಇರಬಹುದು.

### `triage_category` ಮೌಲ್ಯಗಳು

ಪೇಲೋಡ್ ಅನ್ನು ಫಾರ್ವರ್ಡ್ ಮಾಡುವ ಮೊದಲು, AI ಟ್ರೈಯಾಜ್ ಲೇಯರ್ (triage layer) ಈ ಕೆಳಗಿನ ಮೌಲ್ಯಗಳಲ್ಲಿ ಒಂದನ್ನು `triage_category` ಗೆ ನಿಯೋಜಿಸುತ್ತದೆ:

| ಮೌಲ್ಯ | ಅರ್ಥ |
|---|---|
| `factual_error` | ಅಸಿಸ್ಟೆಂಟ್ ತಪ್ಪಾದ ವಾಸ್ತವ ಮಾಹಿತಿಯನ್ನು ನೀಡಿದೆ. |
| `unhelpful_answer` | ಪ್ರತಿಕ್ರಿಯೆಯು ತಾಂತ್ರಿಕವಾಗಿ ಸರಿಯಾಗಿದ್ದರೂ ಉಪಯುಕ್ತವಾಗಿಲ್ಲ. |
| `inappropriate_content` | ಪ್ರತಿಕ್ರಿಯೆಯಲ್ಲಿ ಬಳಕೆದಾರರಿಗೆ ತೋರಿಸಬಾರದ ವಿಷಯವಿತ್ತು. |
| `other` | ಫೀಡ್‌ಬ್ಯಾಕ್ ತಿಳಿದಿರುವ ವರ್ಗಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗಿಲ್ಲ. |

### ಆಥೆಂಟಿಕೇಶನ್ (Authentication)

ನಿಮ್ಮ ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ ಆಥೆಂಟಿಕೇಶನ್ ಅಗತ್ಯವಿದ್ದರೆ, **Feedback API Key** ಫೀಲ್ಡ್‌ನಲ್ಲಿ ನಿಮ್ಮ ಬೇರಿئر ಟೋಕನ್ ಅನ್ನು ಹೊಂದಿಸಿ. ಏಜೆಂಟ್ ಈ ರೀತಿ ಕಳಿಸುತ್ತದೆ:

```
Authorization: Bearer <your-api-key>
```

**Feedback API Key** ಫೀಲ್ಡ್ ಖಾಲಿಯಿದ್ದರೆ, ಯಾವುದೇ `Authorization` ಹೆಡರ್ ಅನ್ನು ಕಳುಹಿಸಲಾಗುವುದಿಲ್ಲ.

### ಫೀಡ್‌ಬ್ಯಾಕ್ ಸಂಗ್ರಹಣೆಯನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸುವುದು

**Feedback Endpoint URL** ಮತ್ತು **Feedback API Key** ಎರಡೂ ಫೀಲ್ಡ್‌ಗಳನ್ನು ಖಾಲಿ ಬಿಡಿ. ಥಂಬ್ಸ್-ಡೌನ್ ಬಟನ್ ಮತ್ತು ಫೀಡ್‌ಬ್ಯಾಕ್ UI ಬಳಕೆದಾರರಿಗೆ ಗೋಚರವಾಗಿರುತ್ತದೆ, ಆದರೆ ಸಲ್ಲಿಸಿದ ಫೀಡ್‌ಬ್ಯಾಕ್ ಯಾವುದೇ ಬಾಹ್ಯ ಸೇವೆಗೆ ಫಾರ್ವರ್ಡ್ ಆಗುವುದಿಲ್ಲ.

## Brave Search API Key

**Advanced** ಟ್ಯಾಬ್‌ನಲ್ಲಿರುವ **Brave Search API Key** ಫೀಲ್ಡ್ [Internet Search](../configuration/internet-search) ಸಾಮರ್ಥ್ಯವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ.

| ಫೀಲ್ಡ್ | ವಿವರಣೆ |
|---|---|
| **Brave Search API Key** | Brave Search developer ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ನಿಂದ ನಿಮ್ಮ API ಕೀ. AI ಅಸಿಸ್ಟೆಂಟ್‌‌ನಲ್ಲಿ ಇಂಟರ್ನೆಟ್ ಸರ್ಚ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ಇದು ಅಗತ್ಯ. |

ಈ ಫೀಲ್ಡ್ ಲೇಬಲ್‌ನಲ್ಲಿ Brave Search APIಗೆサイン-ಅಪ್ ಪುಟಕ್ಕೆ ಕ್ಲಿಕ್ ಮಾಡಬಹುದಾದ ಲಿಂಕ್ ಇದೆ. ಇಂಟರ್ನೆಟ್ ಸರ್ಚ್ ಅನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲು ಇದನ್ನು ಖಾಲಿ ಬಿಡಿ.

ಈ ವೈಶಿಷ್ಟ್ಯದ ಕುರಿತು ಅಂತಿಮ-ಬಳಕೆದಾರರ ದಾಖಲೆಗಾಗಿ [Internet Search](../configuration/internet-search) ನೋಡಿ.

## ಫೀಚರ್ ಫ್ಲ್ಯಾಗ್‌ಗಳು (Feature Flags)

v1.9.0 ರಲ್ಲಿ ಪರಿಚಯಿಸಲಾದ **Settings → Feature Flags** ಟ್ಯಾಬ್‌ವು ಐಚ್ಛಿಕ ಕಾರ್ಯಕ್ಷಮತೆಗಳಿಗಾಗಿ ಟಾগল ಸ್ವಿಚ್‌ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. ಪ್ರತಿಯೊಂದು ಫ್ಲ್ಯಾಗ್ ಅನ್ನು ನೆಟ್‌ವರ್ಕ್-ವ್ಯಾಪಿವಾಗಿ ಸಕ್ರಿಯಗೊಳಿಸಲಾಗಿದೆ ಅಥವಾ ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲಾಗಿದೆ; ಪ್ರಸ್ತುತ ಇದು ಯಾವುದೇ ನಿರ್ದಿಷ್ಟ ಸೈಟ್‌ಗೆ ಅನ್ವಯಿಸುವ ಆಯ್ಕೆಯಲ್ಲ.

### ಫೀಚರ್ ಫ್ಲ್ಯಾಗ್‌ಗಳಿಗೆ ಪ್ರವೇಶಿಸುವುದು

1. WordPress ಅಡ್ಮಿನ್‌ನಲ್ಲಿ, **Gratis AI Agent → Settings** ಗೆ ಹೋಗಿ.
2. **Feature Flags** ಟ್ಯಾಬ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ.

### ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಫ್ಲ್ಯಾಗ್‌ಗಳು (Access Control Flags)

| ಫ್ಲ್ಯಾಗ್ | ಡಿಫಾಲ್ಟ್ | ವಿವರಣೆ |
|---|---|---|
| **Restrict to Administrators** | Off | ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದರೆ, `administrator` ಪಾತ್ರ ಹೊಂದಿರುವ ಬಳಕೆದಾರರು ಮಾತ್ರ AI Agent chat panel ಅನ್ನು ತೆರೆಯಬಹುದು. ಇತರ ಎಲ್ಲಾ ಪಾತ್ರಗಳು ಬದಲಾಗಿ "Contact your administrator" ಸಂದೇಶವನ್ನು ನೋಡುತ್ತವೆ. |
| **Restrict to Network Admins** | Off | ಮಲ್ಟಿಸಿಟ್ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದರೆ, ಸೂಪರ್ ಅಡ್ಮին್‌ಗಳು ಮಾತ್ರ ಏಜೆಂಟ್ ಅನ್ನು ಬಳಸಬಹುದು. 개개 ಸೈಟ್ ಅಡ್ಮին್‌ಗಳು ಬ್ಲಾಕ್ ಆಗುತ್ತಾರೆ. ಎರಡೂ ಸಕ್ರಿಯಗೊಂಡಿದ್ದರೆ ಇದು "Restrict to Administrators" ಗಿಂತ ಹೆಚ್ಚಿನ ಪ್ರಾಮುಖ್ಯತೆಯನ್ನು ಪಡೆಯುತ್ತದೆ. |
| **Allow Subscriber Access** | Off | ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದರೆ, `subscriber` ಪಾತ್ರ ಹೊಂದಿರುವ ಬಳಕೆದಾರರು ಚಾಟ್ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಬಳಸಬಹುದು, ಆದರೆ ಅವರು ಓದಲು ಮಾತ್ರ (read-only) ಸಾಮರ್ಥ್ಯಗಳಿಗೆ ಸೀಮಿತರಾಗಿರುತ್ತಾರೆ (ಪೋಸ್ಟ್ ರಚನೆ ಅಥವಾ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಬದಲಾಯಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ). |
| **Disable for Non-Members** | Off | Ultimate Multisite ಸದಸ್ಯತ್ವದ ಸ್ಥಿತಿಯೊಂದಿಗೆ ಸಂಯೋಜಿಸುತ್ತದೆ. ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದರೆ, ಸಕ್ರಿಯ ಸದಸ್ಯತ್ವ ಹೊಂದಿಲ್ಲದ ಸೈಟ್‌ಗಳಿಗೆ ಚಾಟ್ ಅನ್ನು ಮರೆಮಾಚಲಾಗುತ್ತದೆ. |

### ಬ್ರ್ಯಾಂಡಿಂಗ್ ಫ್ಲ್ಯಾಗ್‌ಗಳು (Branding Flags)

| ಫ್ಲ್ಯಾಗ್ | ಡಿಫಾಲ್ಟ್ | ವಿವರಣೆ |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | ಚಾಟ್ ವಿಜೆಟ್‌ನ ಕೆಳಭಾಗದಲ್ಲಿ ತೋರಿಸುವ ಬ್ರ್ಯಾಂಡಿಂಗ್ ಆಟ್ರಿಬ್ಯೂಷನ್ ಲೈನ್ ಅನ್ನು ತೆಗೆದುಹಾಕುತ್ತದೆ. ವೈಟ್-ಲೇಬಲ್ ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್‌ಗಳಿಗೆ ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ. |
| **Custom Agent Name** | *(blank)* | ಚಾಟ್ ಹೆಡರ್ ಮತ್ತು ಅಡ್ಮಿನ್ ಮೆನುವಿನಲ್ಲಿರುವ डिಫಾಲ್ಟ್ "Gratis AI Agent" ಲೇಬಲ್ ಅನ್ನು ನಿಮ್ಮದೇ ಉತ್ಪನ್ನದ ಹೆಸರಿನಿಂದ ಬದಲಾಯಿಸುತ್ತದೆ. ಡಿಫಾಲ್ಟ್ ಬಳಸಲು ಖಾಲಿ ಬಿಡಿ. |
| **Hide Agent Picker** | Off | ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದರೆ, ಬಳಕೆದಾರರು ಐದು ಅಂತರ್ನಿರ್ಮಿತ ಏಜೆಂಟ್‌ಗಳ ನಡುವೆ ಬದಲಾಯಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ. ಪ್ರಸ್ತುತ ಏಜೆಂಟ್ ಅನ್ನು Settings → General ನಲ್ಲಿ ಡಿಫಾಲ್ಟ್ ಆಗಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಿರುವದಕ್ಕೆ ಸೀಮಿತಗೊಳಿಸಲಾಗುತ್ತದೆ. |
| **Use Site Icon as Chat Avatar** | Off | ಚಾಟ್ ವಿಜೆಟ್ ಹೆಡರ್‌ನಲ್ಲಿರುವ डिಫಾಲ್ಟ್ AI ಐಕಾನ್ ಅನ್ನು WordPress ಸೈಟ್ ಐಕಾನ್‌ನೊಂದಿಗೆ (Appearance → Customize → Site Identity ನಲ್ಲಿ ಹೊಂದಿಸಲಾಗಿದೆ) ಬದಲಾಯಿಸುತ್ತದೆ. |

### ಬದಲಾವಣೆಗಳನ್ನು ಅನ್ವಯಿಸುವುದು

ಯಾವುದೇ ಫ್ಲ್ಯಾಗ್ ಅನ್ನು ಟಾগল ಮಾಡಿದ ನಂತರ **Save Settings** ಕ್ಲಿಕ್ ಮಾಡಿ. ಬದಲಾವಣೆಗಳು ತಕ್ಷಣವೇ ಜಾರಿಗೆ ಬರುತ್ತವೆ — ಯಾವುದೇ ಕ್ಯಾಶ್ ಫ್ಲಶ್ ಅಥವಾ ಪ್ಲಗ್ಇನ್ ರೀಕ್ಟಿವೇಷನ್ ಅಗತ್ಯವಿಲ್ಲ.
