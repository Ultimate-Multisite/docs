# ಸಾರ್ವಭೌಮ ಬಾಡಿಗೆದಾರರ ಕಾರ್ಯನಿರ್ವಹಣೆ (Sovereign Tenant Operations) {#sovereign-tenant-operations}

Ultimate Multisite: Multi-Tenancy 1.2.0 ನಲ್ಲಿ ಸಾರ್ವಭೌಮ ಬಾಡಿಗೆದಾರರಿಗೆ (sovereign tenants) ಆಪರೇಟಿಂಗ್ ಟೂಲಿಂಗ್ ಅನ್ನು ಸೇರಿಸಲಾಗಿದೆ: ಇವು ಉಪಸೈಟ್‌ಗಳಾಗಿವೆ, ಇವು ತಮ್ಮದೇ ಆದ ಡೇಟಾಬೇಸ್, ಫೈಲ್सिस्टम ರೂಟ್ ಮತ್ತು ರೌಟಿಂಗ್ ಸಂದರ್ಭಗಳಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ, ಆದರೆ ನೆಟ್‌ವರ್ಕ್ ადಮಿನಿಂದ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತವೆ.

ಪ್ರಮುಖವಾದ ಗ್ರಾಹಕರ ಸೈಟ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುವಾಗ, ದೂರದ ಸೈಟ್‌ಗಳ ಹಸ್ತಾಂತರಗಳು (remote-site handoffs) ಅಥವಾ ಒಂದು ಪ್ರಮಾಣಿತ ಉಪಸೈಟ್ ಅನ್ನು ಸಾರ್ವಭೌಮ ಮೂಲಸೌಕರ್ಯಕ್ಕೆ ವರ್ಗಾಯಿಸುವಂತಹ മൈಗ್ರೇಷನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುವಾಗ ಈ ಪುಟವನ್ನು ಬಳಸಿ.

## ადಮಿನರಿಗಳಿಗೆ ಬದಲಾವಣೆಗಳು {#what-changes-for-administrators}

- **ಸ್ಟೇಟ್‌ಲೆಸ್ ಟೆನೆಂಟ್ ಆಟೋಲೊぎನ್ (Stateless tenant autologin)** — ನೆಟ್‌ವರ್ಕ್ ადಮಿನರು ದೀರ್ಘಕಾಲ ಉಳಿಯುವ condiv ಸೆಷನ್ ಸ್ಥಿತಿಯನ್ನು ಅವಲಂಬಿಸದೆ ಸಾರ್ವವಭೌಮ ಟೆನೆಂಟ್‌ಗೆ ಭೇಟಿ ನೀಡಬಹುದು. SSO ಟೋಕನ್ ಉದ್ದೇಶಕ್ಕೆ ನಿರ್ದಿಷ್ಟವಾಗಿರುತ್ತದೆ, ಮೂಲವನ್ನು (origin) ಫಿಕ್ಸ್ ಮಾಡಲಾಗುತ್ತದೆ, ಪುನರಾವರ್ತನೆಯನ್ನು (replay) ತಡೆಯುತ್ತದೆ ಮತ್ತು ಕಡಿಮೆ ಅವಧಿಯ ಮುಕ್ತಾಯದ ವಿಂಡೋಗೆ ಮಿತಿಗೊಳಿಸಲಾಗುತ್ತದೆ.
- **ಸಾರ್ವಭೌಮ-ಚಾಹುವಾದ ರೌಟಿಂಗ್ (Sovereign-aware routing)** — ಹಳೆಯ ಪ್ರತ್ಯೇಕ ನೆಟ್‌ವರ್ಕ್‌ಗಳು ಮತ್ತು ಸಾರ್ವಭೌಮ ಟೆನೆಂಟ್‌ಗಳು ಒಂದೇ ಸೈಟ್ ರೂಟರ್ ಮಾರ್ಗವನ್ನು ಬಳಸುತ್ತವೆ, ಇದು ಹಳೆಯ ಮತ್ತು ಹೊಸ ಪ್ರತ್ಯೇಕ ಇನ್‌ಸ್ಟಾಲ್‌ಗಳ ನಡುವಿನ ಬೂಟ್ಸ್ಟ್ರ್ಯಾಪ್ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.
- **ಪರಿಶೀಲಿಸಿದ മൈಗ್ರೇಷನ್ ಸ್ಥಿತಿ (Verified migration state)** — ಒಂದು ಟೆನೆಂಟ್ ಪೂರ್ಣಗೊಂಡ ನಂತರ ಅದನ್ನು ಪರಿಗಣಿಸುವ ಮೊದಲು, ಈ ಪರಿಶೀಲನೆಯು ಬಳಕೆದಾರ ಪ್ರವೇಶದ ವ್ಯವಸ್ಥೆ, ಡೇಟಾಬೇಸ್ ರೈಟರ್ ಅನುಮತಿಗಳು, ಕ್ಯೂ ಡ್ರೇನ್ ಸ್ಥಿತಿ ಮತ್ತು ಹಳೆಯ ಟೇಬಲ್‌ನ ಅನುಪಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ.
- **ಸುರಕ್ಷಿತ ವಿಘಟನೆ (Safer teardown)** — ಸಾರ್ವವಭೌಮ ವಿಘಟನೆಯು ಈಗ ಟೆನೆಂಟ್ ಕ್ರೆಡenciaisಗಳನ್ನು ಸ್ವಚ್ಛವಾಗಿ ತೆಗೆದುಹಾಕುತ್ತದೆ, ಇದರಿಂದ ಅಳಿಸಿದ ಟೆನೆಂಟ್‌ಗಳು ಹಳೆಯ ಡೇಟಾಬೇಸ್ ಪ್ರವೇಶವನ್ನು ಉಳಿಸುವುದಿಲ್ಲ.

## ಸಾರ್ವವಭೌಮ ಟೆನೆಂಟ್‌ಗೆ ಭೇಟಿ ನೀಡುವುದು {#visiting-a-sovereign-tenant}

1. **Network Admin > Ultimate Multisite > Sites** ತೆರೆಯಿರಿ.
2. ಸಾರ್ವವಭೌಮ ಟೆನೆಂಟ್ ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿ.
3. ಸಾಧ್ಯವಾದಾಗ ಪಾಸ್‌ವರ್ಡ್‌ಗಳನ್ನು ನಕಲಿಸುವುದನ್ನು ಅಥವಾ ತಾತ್ಕಾಲಿಕ ადಮನ್ ಖಾತೆಗಳನ್ನು ರಚಿಸುವುದನ್ನು ಬಿಟ್ಟು **Visit (SSO)** ಬಳಸಿ.

ಈ ಭೇಟಿ ಪ್ರಕ್ರಿಯೆಯು ಆ ಟೆನೆಂಟ್‌ಗಾಗಿ ಒಂದು ಸಣ್ಣ ಅವಧಿಯ ಲಾಗಿನ್ ಟೋಕನ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ ಮತ್ತು ಟೆನೆಂಟ್ ಆಡಿಟ್ ಟ್ರೇಲ್‌ನಲ್ಲಿ SSO ಘಟನೆಯನ್ನು ದಾಖಲಿಸುತ್ತದೆ. ಬಟನ್ ವಿಫಲವಾದರೆ, ಆ ಟೆನೆಂಟ್ ಡೊಮೇನ್ ನಿರೀಕ್ಷಿತ ಇನ್‌ಸ್ಟಾಲ್‌ಗೆ ಪರಿವರ್ತಿತವಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ ಮತ್ತು ಟೆನೆಂಟ್ ನೆಟ್‌ವರ್ಕ್-ಪಕ್ಕದ SSO ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ ತಲುಪುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ.

## ದೂರದ ಸೈಟ್ ಕಾರ್ಯನಿರ್ವಹಣೆಯ ಚೆಕ್‌ಲಿಸ್ಟ್ (Remote-site operations checklist) {#remote-site-operations-checklist}

ಸಾರ್ವವಭೌಮ ಅಥವಾ ದೂರದ ಟೆನೆಂಟ್ ಅನ್ನು ಬದಲಾಯಿಸುವ ಮೊದಲು, ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

## ಸಾರ್ವಭೌಮ (sovereign) ተುತ್ತುದಾರರನ್ನು ಅಳಿಸುವುದು {#deleting-sovereign-tenants}

ಸಾರ್ವಭೌಮ ተುತ್ತುದಾರರನ್ನು ಅಳಿಸುವುದು ಡೇಟಾವನ್ನು ಶಾಶ್ವತವಾಗಿ ನಾಶಮಾಡುವ ಪ್ರಕ್ರಿಯೆಯಾಗಿದೆ. ಮೊದಲು ಬ್ಯಾಕಪ್ ಮತ್ತು ಎಕ್ಸ್‌ಪೋರ್ಟ್ ಸ್ಥಿತಿಯನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ, ನಂತರ ಸೈಟ್ ಮ್ಯಾನೇಜ್‌ಮೆಂಟ್ ಪರದೆಯಿಂದ ಅದನ್ನು ಅಳಿಸಿ. 1.2.0 ಟಿಯರ್‌ಡೌನ್ ಫ್ಲೋ (teardown flow) ಕ್ಲೀನಿಂಗ್‌ನ ಭಾಗವಾಗಿ ተುತ್ತುದಾರರ ಡೇಟಾಬೇಸ್ ಕ್ರೆಡಿenciaisಗಳನ್ನು ತೆಗೆದುಹಾಕುತ್ತದೆ, ಆದರೆ ಬಾಹ್ಯ ಹೋಸ್ಟಿಂಗ್ ಪ್ಯಾನೆಲ್‌ಗಳನ್ನು ಬಳಸುವಾಗ ಹೋಸ್ಟ್-ಲೆವೆಲ್ ಡೇಟಾಬೇಸ್ ಬಳಕೆದಾರರು ಮತ್ತು ಫೋಲ್ಡರ್‌ಗಳು ಹೋಗಿವೆ ಎಂಬುದನ್ನು ადಮಿನಿಸ್ಟ್ರೇಟರ್ ಪರಿಶೀಲಿಸುವುದು ಉತ್ತಮ.

:::warning
ಮಿಗ್ರೇಷನ್ ವೆರಿಫಿಕೇಶನ್ (migration verification) ನಡೆಯುತ್ತಿರುವಾಗ ಅಥವಾ ಅಸಿಂಕ್ ಪುಶ್ ಜಾಬ್‌ಗಳು (async push jobs) ಕ್ಯೂನಲ್ಲಿರುವಾಗ ಸಾರ್ವವಭೌಮ ተುತ್ತುದಾರರನ್ನು ಅಳಬೇಡಿ. ಟಿಯರ್‌ಡೌನ್ ಪ್ರಕ್ರಿಯೆಯು ಬಾಕಿ ಇರುವ ಜಾಬ್‌ಗಳಿಗೆ ಅಗತ್ಯವಿರುವ ಕ್ರೆಡಿenciaisಗಳನ್ನು ತೆಗೆದುಹಾಕುವುದನ್ನು ತಪ್ಪಿಸಲು ವೆರಿಫಿಕೇಶನ್ ಪೂರ್ಣಗೊಳ್ಳುವವರೆಗೆ ಕಾಯಿರಿ.
:::
