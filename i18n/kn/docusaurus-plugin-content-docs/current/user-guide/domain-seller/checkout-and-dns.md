# ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ಕ್ಷೇತ್ರ ಮತ್ತು ಗ್ರಾಹಕರ DNS ನಿರ್ವಹಣೆ {#checkout-field-and-customer-dns-management}

## ಡೊಮೇನ್ ಆಯ್ಕೆ ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ಕ್ಷೇತ್ರ {#the-domain-selection-checkout-field}

**ಡೊಮೇನ್ ಆಯ್ಕೆ** ಕ್ಷೇತ್ರವು ಗ್ರಾಹಕರಿಗೆ ತಮ್ಮ ಜಾಲತಾಣದ ಡೊಮೇನ್ ಪಡೆಯುವ ವಿಧಾನವನ್ನು ಆಯ್ಕೆಮಾಡಲು ಅವಕಾಶ ನೀಡುವ ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ಅಂಶವಾಗಿದೆ. ಡೊಮೇನ್ ಮಾರಾಟವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ಇದನ್ನು ಯಾವುದೇ ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ನಮೂನೆಗೆ ಸೇರಿಸಿ.

### ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ನಮೂನೆಗೆ ಕ್ಷೇತ್ರವನ್ನು ಸೇರಿಸುವುದು {#adding-the-field-to-a-checkout-form}

1. **ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಹಣೆ › Ultimate Multisite › ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ನಮೂನೆಗಳು** ಗೆ ಹೋಗಿ
2. ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ನಮೂನೆಯನ್ನು ತೆರೆಯಿರಿ ಅಥವಾ ರಚಿಸಿ
3. ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ಸಂಪಾದಕದಲ್ಲಿ, **ಕ್ಷೇತ್ರ ಸೇರಿಸಿ** ಕ್ಲಿಕ್ ಮಾಡಿ
4. ಕ್ಷೇತ್ರ ಪಟ್ಟಿಯಿಂದ **ಡೊಮೇನ್ ಆಯ್ಕೆ** ಆಯ್ಕೆಮಾಡಿ
5. ಕ್ಷೇತ್ರ ಆಯ್ಕೆಗಳನ್ನು ಸಂರಚಿಸಿ (ಕೆಳಗೆ ನೋಡಿ)
6. ನಮೂನೆಯನ್ನು ಉಳಿಸಿ

### ಕ್ಷೇತ್ರ ಆಯ್ಕೆಗಳು {#field-options}

**ಡೊಮೇನ್ ವಿಧಾನಗಳು** — ಗ್ರಾಹಕರು ಯಾವ ಟ್ಯಾಬ್‌ಗಳನ್ನು ನೋಡುತ್ತಾರೆ ಎಂಬುದನ್ನು ಆಯ್ಕೆಮಾಡಿ. ಪ್ರತಿ ವಿಧಾನವನ್ನು ಪ್ರತ್ಯೇಕವಾಗಿ ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು ಅಥವಾ ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಬಹುದು:

| ವಿಧಾನ | ಇದು ಏನು ಮಾಡುತ್ತದೆ |
|---|---|
| **ಉಪಡೊಮೇನ್** | ಗ್ರಾಹಕರು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಉಚಿತ ಉಪಡೊಮೇನ್ ಬಳಸುತ್ತಾರೆ (ಉದಾ., `mysite.yournetwork.com`). ಪಾವತಿ ಅಗತ್ಯವಿಲ್ಲ. |
| **ಹೊಸ ಡೊಮೇನ್ ನೋಂದಾಯಿಸಿ** | ಗ್ರಾಹಕರು ಹೊಸ ಡೊಮೇನ್ ಹುಡುಕಿ, ನೀವು ಸಂರಚಿಸಿದ ಪೂರೈಕೆದಾರರ ಮೂಲಕ ಅದನ್ನು ನೋಂದಾಯಿಸುತ್ತಾರೆ. ಬೆಲೆಗೆ ಹೊಂದಾಣಿಕೆಯ ಡೊಮೇನ್ product ಬಳಸುತ್ತದೆ. |
| **ಈಗಿರುವ ಡೊಮೇನ್** | ಗ್ರಾಹಕರು ಈಗಾಗಲೇ ತಮ್ಮದಾಗಿರುವ ಡೊಮೇನ್ ಅನ್ನು ಮ್ಯಾಪ್ ಮಾಡುತ್ತಾರೆ. ನೋಂದಣಿ ಶುಲ್ಕವಿಲ್ಲ. ಡೊಮೇನ್ ಅನ್ನು ಅವರ ಜಾಲತಾಣಕ್ಕೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಮ್ಯಾಪ್ ಮಾಡಲಾಗುತ್ತದೆ. |

**ಡೀಫಾಲ್ಟ್ ವಿಧಾನ** — ಮೂರೂ ವಿಧಾನಗಳು ಸಕ್ರಿಯವಾಗಿರುವಾಗ ಯಾವ ಟ್ಯಾಬ್ ಮೊದಲು ತೆರೆಯಬೇಕು. ಡೊಮೇನ್ ನೋಂದಣಿಯನ್ನು ಐಚ್ಛಿಕವಾಗಿಡಲು **ಉಪಡೊಮೇನ್** ಗೆ ಹೊಂದಿಸಿ, ಅಥವಾ ಖರೀದಿಗಳನ್ನು ಉತ್ತೇಜಿಸಲು **ಹೊಸ ಡೊಮೇನ್ ನೋಂದಾಯಿಸಿ** ಗೆ ಹೊಂದಿಸಿ.

**ಡೊಮೇನ್ product** — ಐಚ್ಛಿಕವಾಗಿ ಈ ಕ್ಷೇತ್ರವನ್ನು ನಿರ್ದಿಷ್ಟ ಡೊಮೇನ್ product ಗೆ ಪಿನ್ ಮಾಡಿ. ಹೊಂದಿಸದಿದ್ದರೆ, ಗ್ರಾಹಕರು ಹುಡುಕುವ TLD ಆಧರಿಸಿ addon ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೊಂದಾಣಿಕೆಯ product ಆಯ್ಕೆಮಾಡುತ್ತದೆ.

### ನೋಂದಣಿದಾರ ಸಂಪರ್ಕ ಕ್ಷೇತ್ರಗಳು {#registrant-contact-fields}

ಗ್ರಾಹಕರು **ಹೊಸ ಡೊಮೇನ್ ನೋಂದಾಯಿಸಿ** ಟ್ಯಾಬ್ ಆಯ್ಕೆಮಾಡಿದಾಗ, ಪಾವತಿ ಪೂರ್ಣಗೊಳಿಸುವ ನಮೂನೆ ನೋಂದಣಿದಾರ ಸಂಪರ್ಕ ಕ್ಷೇತ್ರಗಳನ್ನು ಅದೇ ಸಾಲಿನಲ್ಲಿ ಸೇರಿಸುತ್ತದೆ:

- ಮೊದಲ ಹೆಸರು / ಕೊನೆಯ ಹೆಸರು
- ಇಮೇಲ್ ವಿಳಾಸ
- ವಿಳಾಸ (ಸಾಲು 1, ನಗರ, ರಾಜ್ಯ/ಪ್ರಾಂತ್ಯ, ಅಂಚೆ ಕೋಡ್, ದೇಶ)
- ದೂರವಾಣಿ ಸಂಖ್ಯೆ

ಇವುಗಳನ್ನು ಎಲ್ಲಾ ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಳು ಅಗತ್ಯಪಡಿಸುತ್ತವೆ ಮತ್ತು ನೋಂದಣಿ API ಕರೆ ಮಾಡುವ ಮೊದಲು ಮಾನ್ಯಗೊಳಿಸಲಾಗುತ್ತದೆ. ದೂರವಾಣಿ ಸಂಖ್ಯೆಗಳು ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಳು ನಿರೀಕ್ಷಿಸುವ `+CC.NNN` ಅಂತರರಾಷ್ಟ್ರೀಯ ಸ್ವರೂಪಕ್ಕೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಫಾರ್ಮ್ಯಾಟ್ ಆಗುತ್ತವೆ.

### ಸ್ವಯಂ ರಚಿಸಲಾದ ಜಾಲತಾಣ URL {#auto-generated-site-url}

ಗ್ರಾಹಕರು ಡೊಮೇನ್ ನೋಂದಾಯಿಸಿದಾಗ ಅಥವಾ ಮ್ಯಾಪ್ ಮಾಡಿದಾಗ, ಜಾಲತಾಣ URL ಕ್ಷೇತ್ರವು ಆಯ್ಕೆಮಾಡಿದ ಡೊಮೇನ್‌ನಿಂದ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ತುಂಬುತ್ತದೆ. ಗ್ರಾಹಕರು ಪ್ರತ್ಯೇಕ URL ಕ್ಷೇತ್ರವನ್ನು ತುಂಬುವ ಅಗತ್ಯವಿಲ್ಲ.

### ಹುಡುಕಾಟ ವರ್ತನೆ {#search-behaviour}

- ಗ್ರಾಹಕರು ಟೈಪ್ ಮಾಡುವಾಗ ಡೊಮೇನ್ ಲಭ್ಯತೆಯನ್ನು AJAX ಮೂಲಕ ನೈಜ ಸಮಯದಲ್ಲಿ ಪರಿಶೀಲಿಸಲಾಗುತ್ತದೆ
- ಆದ್ಯತೆಯ ಡೊಮೇನ್ ಲಭ್ಯವಿಲ್ಲದಿದ್ದಾಗ ಪರ್ಯಾಯ TLD ಸಲಹೆಗಳನ್ನು ತೋರಿಸಲಾಗುತ್ತದೆ
- ಬೆಲೆಯನ್ನು ನೇರವಾಗಿ ಪಡೆಯಲಾಗುತ್ತದೆ ಮತ್ತು ಸ್ಪಷ್ಟವಾಗಿ ತೋರಿಸಲಾಗುತ್ತದೆ (ನೋಂದಣಿ ಬೆಲೆ, ನವೀಕರಣ ಬೆಲೆ, ಐಚ್ಛಿಕ WHOIS ಗೌಪ್ಯತಾ ಶುಲ್ಕ)
- ಕೂಪನ್ ಕೋಡ್‌ಗಳು ಬೇರೆ ಯಾವುದೇ product ನಂತೆಯೇ ಡೊಮೇನ್ product ಗಳಿಗೂ ಅನ್ವಯಿಸುತ್ತವೆ

**ಹುಡುಕಾಟ ಪ್ರತಿಕ್ರಿಯಾಶೀಲತೆಯನ್ನು ಸರಿಹೊಂದಿಸುವುದು:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ಡೊಮೇನ್ ಹುಡುಕಾಟ ನಮೂನೆಗೆ ಕಸ್ಟಮ್ ಕ್ಷೇತ್ರಗಳನ್ನು ಸೇರಿಸುವುದು:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ಗ್ರಾಹಕರ DNS ನಿರ್ವಹಣೆ {#customer-dns-management}

ಗ್ರಾಹಕರು ತಮ್ಮ ನೋಂದಾಯಿತ ಡೊಮೇನ್‌ಗಳ DNS ದಾಖಲೆಗಳನ್ನು **My Account** ಪುಟದಿಂದ, ತಮ್ಮ ಡೊಮೇನ್‌ನ ನಮೂದಿನಡಿ ನಿರ್ವಹಿಸಬಹುದು.

### ಬೆಂಬಲಿತ ದಾಖಲೆ ಪ್ರಕಾರಗಳು {#supported-record-types}

| ಪ್ರಕಾರ | ಬಳಕೆ |
|---|---|
| **A** | ಹೋಸ್ಟ್‌ನೇಮ್ ಅನ್ನು IPv4 ವಿಳಾಸಕ್ಕೆ ಮ್ಯಾಪ್ ಮಾಡಿ |
| **AAAA** | ಹೋಸ್ಟ್‌ನೇಮ್ ಅನ್ನು IPv6 ವಿಳಾಸಕ್ಕೆ ಮ್ಯಾಪ್ ಮಾಡಿ |
| **CNAME** | ಮತ್ತೊಂದು ಹೋಸ್ಟ್‌ನೇಮ್‌ಗೆ ಸೂಚಿಸುವ ಅಲಿಯಾಸ್ ರಚಿಸಿ |
| **MX** | ಮೇಲ್ ಎಕ್ಸ್‌ಚೇಂಜ್ ಸರ್ವರ್ ಹೊಂದಿಸಿ |
| **TXT** | SPF, DMARC, ಪರಿಶೀಲನೆ, ಅಥವಾ ಇತರ ಪಠ್ಯ ದಾಖಲೆಗಳನ್ನು ಸೇರಿಸಿ |

### ಯಾವ ಪೂರೈಕೆದಾರರು DNS ನಿರ್ವಹಣೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತಾರೆ? {#which-providers-support-dns-management}

DNS ನಿರ್ವಹಣೆ (ದಾಖಲೆಗಳನ್ನು ಸೇರಿಸುವುದು, ಸಂಪಾದಿಸುವುದು, ಅಳಿಸುವುದು) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, ಮತ್ತು **Openprovider** ಜೊತೆ ಲಭ್ಯವಿದೆ. **Hostinger** ಡೊಮೇನ್‌ಗಳು Domain Seller ಮೂಲಕ ನೆಮ್‌ಸರ್ವರ್‌ಗಳನ್ನು ನವೀಕರಿಸಬಹುದು; ಹೋಸ್ಟ್ ಮಾಡಿದ ಡೊಮೇನ್‌ಗಳ DNS ದಾಖಲೆಗಳನ್ನು ಮೂಲ Hostinger ಡೊಮೇನ್-ಮ್ಯಾಪಿಂಗ್ ಸಂಯೋಜನೆ ನಿರ್ವಹಿಸುತ್ತದೆ. Namecheap, GoDaddy, ಮತ್ತು NameSilo ಡೊಮೇನ್‌ಗಳು ಸ್ಥಿತಿ ಮತ್ತು ಅವಧಿ ಮುಗಿಯುವ ಮಾಹಿತಿಯನ್ನು ತೋರಿಸುತ್ತವೆ, ಆದರೆ DNS ಅನ್ನು ನೇರವಾಗಿ ರಿಜಿಸ್ಟ್ರಾರ್‌ನ ನಿಯಂತ್ರಣ ಫಲಕದಲ್ಲಿ ನಿರ್ವಹಿಸಬೇಕು.

### ಡೀಫಾಲ್ಟ್ DNS ದಾಖಲೆಗಳು {#default-dns-records}

ಡೊಮೇನ್ ನೋಂದಾಯಿಸಿದಾಗ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನ್ವಯಿಸುವ ಡೀಫಾಲ್ಟ್ DNS ದಾಖಲೆಗಳನ್ನು ನೀವು ಸಂರಚಿಸಬಹುದು. **Settings › Domain Seller › Default DNS Records** ಗೆ ಹೋಗಿ.

ಡೀಫಾಲ್ಟ್ ದಾಖಲೆ ಮೌಲ್ಯಗಳು ಎರಡು ಟೋಕನ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆ:

| ಟೋಕನ್ | ಇದರೊಂದಿಗೆ ಬದಲಾಯಿಸಲಾಗುತ್ತದೆ |
|---|---|
| `{DOMAIN}` | ನೋಂದಾಯಿತ ಡೊಮೇನ್ ಹೆಸರು (ಉದಾ., `example.com`) |
| `{SITE_URL}` | ಗ್ರಾಹಕರ ಜಾಲತಾಣದ WordPress ಜಾಲತಾಣ URL |

**ಉದಾಹರಣೆ — apex ಡೊಮೇನ್ ಮತ್ತು www ಅನ್ನು ನಿಮ್ಮ ಸರ್ವರ್ IP ಗೆ ಸೂಚಿಸಿ:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ನಿರ್ವಾಹಕರು: DNS ನೋಡುವುದು ಮತ್ತು ಸಂಪಾದಿಸುವುದು {#admin-viewing-and-editing-dns}

ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕರು ಯಾವುದೇ ಗ್ರಾಹಕ ಡೊಮೇನ್‌ನ DNS ದಾಖಲೆಗಳನ್ನು ಆ ಡೊಮೇನ್‌ನ ಸಂಪಾದನಾ ಪುಟದಿಂದ **ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಹಣೆ › Ultimate Multisite › ಡೊಮೇನ್‌ಗಳು** ನಲ್ಲಿ ನೋಡಬಹುದು ಮತ್ತು ಸಂಪಾದಿಸಬಹುದು.
