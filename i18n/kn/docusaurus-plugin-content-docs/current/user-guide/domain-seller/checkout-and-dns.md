# చెక్‌అవుట్ ఫీల్డ్ ಮತ್ತು కస్టಮರ್ DNS ನಿರ್ವಹಣೆ

## ಡೊಮೇನ್ ಆಯ್ಕೆ చెక్‌అవుట్ ಫೀಲ್ಡ್

**Domain Selection** ಫೀಲ್ಡ್ ಎಂಬುದು ಒಂದು చెక్‌అవుಟ್ ಅಂಶವಾಗಿದೆ. ಇದು ಗ್ರಾಹಕರಿಗೆ ತಮ್ಮ ಸೈಟ್‌ನ ಡೊಮೇನ್ ಅನ್ನು ಹೇಗೆ ಪಡೆಯಬೇಕು ಎಂಬುದರ ಆಯ್ಕೆಯನ್ನು ನೀಡುತ್ತದೆ. ಡೊಮೇನ್ ಮಾರಾಟವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ಇದನ್ನು ಯಾವುದೇ చెక్‌అవుಟ್ ಫಾರ್ಮ್‌ಗೆ ಸೇರಿಸಿ.

### చెక్‌అవుಟ್ ಫಾರ್ಮ್‌ಗೆ ಫೀಲ್ಡ್ ಸೇರಿಸುವುದು

1. **Network Admin › Ultimate Multisite › Checkout Forms** ಗೆ ಹೋಗಿ
2. ಒಂದು చెక్‌అవుಟ್ ಫಾರ್ಮ್ ಅನ್ನು ತೆರೆಯಿರಿ ಅಥವಾ ರಚಿಸಿ
3. చెక్‌అవుట్ ಎಡಿಟರ್‌ನಲ್ಲಿ, **Add Field** ಕ್ಲಿಕ್ ಮಾಡಿ
4. ಫೀಲ್ಡ್ ಲಿಸ್ಟ್‌ನಿಂದ **Domain Selection** ಅನ್ನು ಆರಿಸಿ
5. ಫೀಲ್ಡ್ ಆಯ್ಕೆಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ (ಕೆಳಗೆ ನೋಡಿ)
6. ಫಾರ್ಮ್ ಅನ್ನು ಸೇವ್ ಮಾಡಿ

### ಫೀಲ್ಡ್ ಆಯ್ಕೆಗಳು

**Domain modes** — ಗ್ರಾಹಕರು ಯಾವ ಟ್ಯಾಬ್‌ಗಳನ್ನು ನೋಡುತ್ತಾರೆ ಎಂಬುದನ್ನು ಆರಿಸಿ. ಪ್ರತಿಯೊಂದು ಮೋಡ್ ಅನ್ನು ಸ್ವತಂತ್ರವಾಗಿ ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು ಅಥವಾ ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಬಹುದು:

| Mode | ಇದು ಏನು ಮಾಡುತ್ತದೆ |
|---|---|
| **Subdomain** | ಗ್ರಾಹಕರು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಉಚಿತ ಸಬ್‌ಡೊಮೇನ್ ಅನ್ನು ಬಳಸುತ್ತಾರೆ (ಉದಾಹರಣೆಗೆ, `mysite.yournetwork.com`). ಪಾವತಿ ಅಗತ್ಯವಿಲ್ಲ. |
| **Register New Domain** | ಗ್ರಾಹಕರು ಹೊಸ ಡೊಮೇನ್ ಅನ್ನು ಹುಡುಕಿ, ಅದನ್ನು ನಿಮ್ಮ ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಪೂರೈಕೆದಾರರ ಮೂಲಕ ನೋಂದಾಯಿಸುತ್ತಾರೆ. ಬೆಲೆ ನಿಗದಿಪಡಿಸಲು ಹೊಂದಿಕೆಯಾಗುವ ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ಬಳಸುತ್ತದೆ. |
| **Existing Domain** | ಗ್ರಾಹಕರು ತಾವೇ ಹೊಂದಿರುವ ಡೊಮೇನ್ ಅನ್ನು ಮ್ಯಾಪ್ ಮಾಡುತ್ತಾರೆ. ನೋಂದಣಿ ಶುಲ್ಕವಿಲ್ಲ. ಡೊಮೇನ್ ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅವರ ಸೈಟ್‌ಗೆ ಮ್ಯಾಪ್ ಮಾಡಲಾಗುತ್ತದೆ. |

**Default mode** — ಮೂರು ಮೋಡ್‌ಗಳು ಸಹ ಸಕ್ರಿಯಗೊಂಡಾಗ, ಯಾವ ಟ್ಯಾಬ್ ಮೊದಲು ತೆರೆಯುತ್ತದೆ. ಡೊಮೇನ್ ನೋಂದಣಿಯನ್ನು ಐಚ್ಛಿಕವಾಗಿ ಉಳಿಸಲು ಇದನ್ನು **Subdomain** ಗೆ, ಅಥವಾ ಖರೀದಿಗಳನ್ನು ಪ್ರೋತ್ಸಾಹಿಸಲು **Register New Domain** ಗೆ ಹೊಂದಿಸಿ.

**Domain product** — ಐಚ್ಛಿಕವಾಗಿ ಈ ಫೀಲ್ಡ್ ಅನ್ನು ನಿರ್ದಿಷ್ಟ ಡೊಮೇನ್ ಉತ್ಪನ್ನಕ್ಕೆ ಪಿನ್ ಮಾಡಬಹುದು. ಇದನ್ನು ಹೊಂದಿಸದಿದ್ದರೆ, ಆಡ್-ಆನ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಗ್ರಾಹಕರು ಹುಡುಕುವ TLD ಆಧಾರದ ಮೇಲೆ ಹೊಂದಿಕೆಯಾಗುವ ಉತ್ಪನ್ನವನ್ನು ಆಯ್ಕೆ ಮಾಡುತ್ತದೆ.

### Registrant contact fields

ಗ್ರಾಹಕರು **Register New Domain** ಟ್ಯಾಬ್ ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿದಾಗ, చెక్‌అవుట్ ಫಾರ್ಮ್‌ನಲ್ಲಿ ನೋಂದಣಾ ಸಂಪರ್ಕ ಫೀಲ್ಡ್‌ಗಳು ಲೈನ್‌‌ನಲ್ಲಿ ಸೇರಿಸಲ್ಪಡುತ್ತವೆ:

- ಮೊದಲ ಹೆಸರು / ಕೊನೆಯ ಹೆಸರು
- ಇಮೇಲ್ ವಿಳಾಸ
- ವಿಳಾಸ (ಲೈನ್ 1, ನಗರ, ರಾಜ್ಯ/ಪ್ರಾಂವನ್ಸ್, ಪಿನ್ ಕೋಡ್, ದೇಶ)
- ಫೋನ್ ಸಂಖ್ಯೆ

ಇವು ಎಲ್ಲಾ ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಳಿಂದ ಅಗತ್ಯವಿರುವ ಅಂಶಗಳು ಮತ್ತು ನೋಂದಣಿ API ಕರೆ ಮಾಡಿಸುವ ಮೊದಲು ವ್ಯಾಲಿಡೇಟ್ (validates) ಆಗುತ್ತವೆ. ಫೋನ್ ಸಂಖ್ಯೆಗಳನ್ನು ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಳು ನಿರೀಕ್ಷಿಸುವ `+CC.NNN` ಅಂತರರಾಷ್ಟ್ರೀಯ ಸ್ವರೂಪಕ್ಕೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಫಾರ್ಮ್ಯಾಟ್ ಮಾಡಲಾಗುತ್ತದೆ.

### Auto-generated site URL

ಗ್ರಾಹಕರು ಡೊಮೇನ್ ಅನ್ನು ನೋಂದಾಯಿಸಿದಾಗ ಅಥವಾ ಮ್ಯಾಪ್ ಮಾಡಿದಾಗ, ಸೈಟ್ URL ಫೀಲ್ಡ್ ಅನ್ನು ಆರಿಸಲಾದ ಡೊಮೇನ್‌ನಿಂದ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ತುಂಬಲಾಗುತ್ತದೆ. ಗ್ರಾಹಕರು ಪ್ರತ್ಯೇಕ URL ಫೀಲ್ಡ್ ಅನ್ನು ಭರ್ತಿ ಮಾಡುವ ಅಗತ್ಯವಿಲ್ಲ.

### Search behaviour

- ಗ್ರಾಹಕರು ಟೈಪ್ ಮಾಡುವಾಗ ಡೊಮೇನ್ ಲಭ್ಯತೆಯನ್ನು AJAX ಬಳಸಿ ನೈಜ-ಸಮಯದಲ್ಲಿ ಪರಿಶೀಲಿಸಲಾಗುತ್ತದೆ
- ಆದ್ಯತೆಯ ಡೊಮೇನ್ ಲಭ್ಯವಿಲ್ಲದಿದ್ದರೆ ಪರ್ಯಾಯ TLD ಸಲಹೆಗಳನ್ನು ತೋರಿಸಲಾಗುತ್ತದೆ
- ಬೆಲೆಗಳನ್ನು ಲೈವ್ ಆಗಿ ಪಡೆದು ಸ್ಪಷ್ಟವಾಗಿ ಪ್ರದರ್ಶಿಸಲಾಗುತ್ತದೆ (ನೋಂದಣಿ ಬೆಲೆ, ನವೀಕರಣ ಬೆಲೆ, ಐಚ್ಛಿಕ WHOIS ಗೌಪ್ಯತಾ ಶುಲ್ಕ)
- ಕೂಪನ್ ಕೋಡ್‌ಗಳು ಯಾವುದೇ ಇತರ ಉತ್ಪನ್ನದಂತೆ ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತವೆ

**Search responsiveness ಟ್ಯೂನಿಂಗ್:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ಡೊಮೇನ್ ಸರ್ಚ್ ಫಾರ್ಮ್‌ಗೆ ಕಸ್ಟಮ್ ಫೀಲ್ಡ್‌ಗಳನ್ನು ಸೇರಿಸುವುದು:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ಗ್ರಾಹಕ DNS ನಿರ್ವಹಣೆ

ಗ್ರಾಹಕರು ತಮ್ಮ ನೋಂದಾಯಿತ ಡೊಮೇನ್‌ಗಳ DNS ರೆಕಾರ್ಡ್‌ಗಳನ್ನು **My Account** ಪುಟದಿಂದ, ತಮ್ಮ ಡೊಮೇನ್‌ನ ಎಂಟ್ರಿ ಅಡಿಯಲ್ಲಿ ನಿರ್ವಹಿಸಬಹುದು.

### ಬೆಂಬಲಿತ ರೆಕಾರ್ಡ್ ಪ್ರಕಾರಗಳು

| Type | ಬಳಕೆ |
|---|---|
| **A** | ಹೋಸ್ಟ್ neame ಅನ್ನು IPv4 ವಿಳಾಸಕ್ಕೆ ಮ್ಯಾಪ್ ಮಾಡುತ್ತದೆ |
| **AAAA** | ಹೋಸ್ಟ್ neame ಅನ್ನು IPv6 ವಿಳಾಸಕ್ಕೆ ಮ್ಯಾಪ್ ಮಾಡುತ್ತದೆ |
| **CNAME** | ಮತ್ತೊಂದು ಹೋಸ್ಟ್ neameಗೆ ಬಾಯ್‌ಸ್ (alias) ಅನ್ನು ರಚಿಸುತ್ತದೆ |
| **MX** | ಇಮೇಲ್ ಎಕ್ಸ್ಚೇಂಜ್ ಸರ್ವರ್ ಅನ್ನು ಹೊಂದಿಸುತ್ತದೆ |
| **TXT** | SPF, DMARC, verification, ಅಥವಾ ಇತರ ಪಠ್ಯ ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ಸೇರಿಸುತ್ತದೆ |

### ಯಾವ ಪೂರೈಕೆದಾರರು DNS ನಿರ್ವಹಣೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತಾರೆ?

DNS ನಿರ್ವಹಣೆ (ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ಸೇರಿಸುವುದು, ಸಂಪಾದಿಸುವುದು, ಅಳಿಸುವುದು) **OpenSRS**, **ResellerClub**, ಮತ್ತು **Enom** ಜೊತೆಗೆ ಲಭ್ಯವಿದೆ. Namecheap, GoDaddy, ಮತ್ತು NameSilo ಡೊಮೇನ್‌ಗಳು ಸ್ಥಿತಿ ಮತ್ತು ಅವಧಿ ಮಾಹಿತಿಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತವೆ, ಆದರೆ DNS ಅನ್ನು ರಿಜಿಸ್ಟ್ರಾರ್‌ನ ಕಂಟ್ರೋಲ್ ಪ್ಯಾನೆಲ್‌ನಲ್ಲಿ ನೇರವಾಗಿ ನಿರ್ವಹಿಸಬೇಕು.

### Default DNS ರೆಕಾರ್ಡ್‌ಗಳು

ಡೊಮೇನ್ ಅನ್ನು ನೋಂದಾಯಿಸಿದಾಗ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನ್ವಯಿಸುವ default DNS ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ನೀವು ಕಾನ್ಫಿಗರ್ ಮಾಡಬಹುದು. **Settings › Domain Seller › Default DNS Records** ಗೆ ಹೋಗಿ.

Default ರೆಕಾರ್ಡ್ ಮೌಲ್ಯಗಳು ಎರಡು ಟೋಕನ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆ:

| Token | ಯಾವುದರಿಂದ ಬದಲಾಯಿಸಲಾಗುತ್ತದೆ |
|---|---|
| `{DOMAIN}` | ನೋಂದಾಯಿತ ಡೊಮೇನ್ ಹೆಸರು (ಉದಾಹರಣೆಗೆ, `example.com`) |
| `{SITE_URL}` | ಗ್ರಾಹಕರ ಸೈಟ್‌ಗಾಗಿ WordPress ಸೈಟ್ URL |

**ಉದಾಹರಣೆ — ಅಪೆಕ್ಸ್ ಡೊಮೇನ್ ಮತ್ತು www ಅನ್ನು ನಿಮ್ಮ ಸರ್ವರ್ IP ಗೆ ಪಾಯಿಂಟ್ ಮಾಡುವುದು:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ಆಡ್ಮಿನ್: DNS ವೀಕ್ಷಣೆ ಮತ್ತು ಸಂಪಾದನೆ

Network адmins ಅವರು **Network Admin › Ultimate Multisite › Domains** ನಲ್ಲಿರುವ ಡೊಮೇನ್‌ನ ಎಡಿಟ್ ಪುಟದಿಂದ ಯಾವುದೇ ಗ್ರಾಹಕರ ಡೊಮೇನ್‌ಗಳ DNS ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಬಹುದು ಮತ್ತು ಸಂಪಾದಿಸಬಹುದು.
