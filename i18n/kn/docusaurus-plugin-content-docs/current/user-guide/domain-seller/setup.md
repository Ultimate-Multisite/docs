# Domain Seller: ಸೆಟಪ್ ಮತ್ತು ಪ್ರೊವೈಡರ್ ಕಾನ್ಫಿಗರೇಶನ್

Domain Seller addon ಜೊತೆಗೆ ಒಂದು ಮಾರ್ಗದರ್ಶಿ ಸೆಟಪ್ ವಿಝಾರ್ಡ್ ಬರುತ್ತದೆ. ಇದು ನಿಮಗೆ ಅಗತ್ಯವಿರುವ ಪ್ರತಿ ಹಂತದ ಮೂಲಕ 안내 ಮಾಡುತ್ತದೆ. ಈ ಪುಟವು ವಿಝಾರ್ಡ್‌ನ ಹರಿವು (flow) ಮತ್ತು ನಂತರ ಪ್ರೊವೈಡರ್‌ಗಳನ್ನು ಹೇಗೆ ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು ಅಥವಾ ಮರು-ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು ಎಂಬುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಅಗತ್ಯತೆಗಳು (Requirements)

- **Multisite Ultimate** v2.4.12 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ, ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಸಕ್ರಿಯಗೊಳಿಸಿರಬೇಕು (network-activated)
- **PHP** 7.4+
- ಕನಿಷ್ಠ ಒಂದು ಬೆಂಬಲಿತ ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಾಗಿ API ಕ್ರೆಡೆನ್ಶಿಯಲ್ಸ್

## ಮೊದಲ ಬಳಕೆಯ ಸೆಟಪ್ ವಿಝಾರ್ಡ್

ನೀವು ಮೊದಲ ಬಾರಿಗೆ ಪ್ಲಗ್ಇನ್ ಅನ್ನು ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಸಕ್ರಿಯಗೊಳಿಸಿದಾಗ ಸೆಟಪ್ ವಿಝಾರ್ಡ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ. ಇದು ಯಾವಾಗ ಬೇಕಾದರೂ **Network Admin › Ultimate Multisite › Domain Seller Setup** ನಿಂದ ಲಭ್ಯವಿದೆ.

### ಹಂತ 1 — ಪ್ರೊವೈಡರ್ ಆಯ್ಕೆಮಾಡಿ

ನೀವು ಸಂಪರ್ಕಿಸಲು ಬಯಸುವ ರಿಜಿಸ್ಟ್ರಾರ್ ಅನ್ನು ಆಯ್ಕೆಮಾಡಿ. ಬೆಂಬಲಿತ ಆಯ್ಕೆಗಳು:

| Provider | DNS management | WHOIS privacy |
|---|---|---|
| OpenSRS | Yes | Yes |
| Namecheap | No | Yes (WhoisGuard, free) |
| GoDaddy | No | No |
| ResellerClub | Yes | No |
| NameSilo | No | No |
| Enom | Yes | No |

### ಹಂತ 2 — ಕ್ರೆಡೆನ್ಶಿಯಲ್ಸ್ ನಮೂದಿಸಿ

ಪ್ರತಿಯೊಂದು ಪ್ರೊವೈಡರ್‌ಗೆ ವಿಭಿನ್ನ ಕ್ರೆಡೆನ್ಶಿಯಲ್ ಫೀಲ್ಡ್‌ಗಳಿವೆ:

**OpenSRS** — ಬಳಕೆದಾರಹೆಸರು (Username) ಮತ್ತು ಪ್ರೈವೇಟ್ ಕೀ (OpenSRS Reseller Control Panel ನಿಂದ)

**Namecheap** — ಬಳಕೆದಾರಹೆಸರು (Username) ಮತ್ತು API ಕೀ (Account › Tools › API Access ನಿಂದ)

**GoDaddy** — API ಕೀ ಮತ್ತು secret (developer.godaddy.com ನಿಂದ)

**ResellerClub** — ರೆಸೆಲ್ಲರ್ ID ಮತ್ತು API ಕೀ (ResellerClub control panel ನಿಂದ)

**NameSilo** — API ಕೀ (namesilo.com › Account › API Manager ನಿಂದ)

**Enom** — account ID ಮತ್ತು API token

ಲಭ್ಯವಿದ್ದರೆ **Sandbox mode** ಅನ್ನು ಪರಿಶೀಲಿಸಿ, ಇದರಿಂದ ಲೈವ್ ಆಗುವ ಮೊದಲು ಪ್ರೊವೈಡರ್‌ನ ಟೆಸ್ಟ್ ಎನ್ವಿರಾನ್‌ಮೆಂಟ್‌ಗೆ ಪರೀಕ್ಷಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

### ಹಂತ 3 — ಸಂಪರ್ಕವನ್ನು ಪರೀಕ್ಷಿಸಿ

**Test Connection** ಕ್ಲಿಕ್ ಮಾಡಿ. ವಿಝಾರ್ಡ್ ಕ್ರೆಡೆನ್ಶಿಯಲ್ಸ್ ಮತ್ತು ಸಂಪರ್ಕತೆಯನ್ನು ಪರಿಶೀಲಿಸಲು ಒಂದು ಲಘು API ಕಳವಳವನ್ನು (lightweight API call) ಕಳುಹಿಸುತ್ತದೆ. ಮುಂದುವರಿಯುವ ಮೊದಲು ಯಾವುದೇ ಕ್ರೆಡೆನ್ಶಿಯಲ್ ಸಮಸ್ಯೆಗಳನ್ನು ಸರಿಪಡಿಸಿ.

### ಹಂತ 4 — TLD ಗಳನ್ನು ಆಮದಮಾಡಿ (Import TLDs)

ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ TLD ಗಳು ಮತ್ತು ಹಾಲ್‌ಸೇಲ್ ಬೆಲೆಗಳನ್ನು (wholesale pricing) ಪಡೆಯಲು **Import TLDs** ಕ್ಲಿಕ್ ಮಾಡಿ. ಇದು ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳನ್ನು ಬಳಸುವ TLD ಪಟ್ಟಿಯನ್ನು ತುಂಬಿಸುತ್ತದೆ. ದೊಡ್ಡ TLD ಕ್ಯಾಟಲಾಗ್‌ಗಳನ್ನು ಹೊಂದಿರುವ ಪ್ರೊವೈಡರ್‌ಗಳಿಗೆ ಈ ಆಮದಮಾಡುವ ಪ್ರಕ್ರಿಯೆಗೆ 30–60 ಸೆಕೆಂಡುಗಳು ಬೇಕಾಗಬಹುದು.

TLD ಗಳನ್ನು ಪ್ರತಿದಿನ ಒಂದು ಬಾರಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ (automatically) ಸಿಂಕ್ (re-synced) ಮಾಡಲಾಗುತ್ತದೆ.

### ಹಂತ 5 — ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸಿ

ವಿಝಾರ್ಡ್ 10% ಲಾಭಾಂಶದೊಂದಿಗೆ (markup) ಒಂದು ಡೀಫಾಲ್ಟ್ ಕ್ಯಾಚ್-ಆಲ್ ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸುತ್ತದೆ. ನೀವು ಈ ಉತ್ಪನ್ನವನ್ನು ತಕ್ಷಣವೇ ಸಂಪಾದಿಸಬಹುದು ಅಥವಾ ಬಿಟ್ಟು, **Ultimate Multisite › Products** ಅಡಿಯಲ್ಲಿ ಉತ್ಪನ್ನಗಳನ್ನು ಕೈಯಾರೆ ರಚಿಸಬಹುದು.

ಪೂರ್ಣ ಉತ್ಪನ್ನ ಕಾನ್ಫಿಗರೇಶನ್ ಗೈಡ್‌ಗಾಗಿ [Domain Products and Pricing](./domain-products) ನೋಡಿ.

## ಪ್ರೊವೈಡರ್ ಅನ್ನು ಮರು-ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** ಗೆ ಹೋಗಿ (ಅಥವಾ ಪ್ಲಗ್ಇನ್ ಪಟ್ಟಿಯಲ್ಲಿ **Settings** ಕ್ಲಿಕ್ ಮಾಡಿ).

ಸೆಟ್ಟಿಂಗ್ಸ್ ಪುಟದಲ್ಲಿ ಈ ಕೆಳಗಿನವುಗಳು ಇರುತ್ತವೆ:

- **Enable domain selling** — ಸಂಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಆನ್/ಆಫ್ ಮಾಡಲು ಟগল ಮಾಡಿ
- **Default provider** — ಡೊಮೇನ್ ಹುಡುಕಾಟಗಳು ಮತ್ತು ಹೊಸ ಉತ್ಪನ್ನಗಳಿಗೆ ಬಳಸುವ ಪ್ರೊವೈಡರ್
- **Max TLDs per search** — ಗ್ರಾಹಕರು ಹುಡುಕುವಾಗ ಎಷ್ಟು TLD ಗಳನ್ನು ಪರಿಶೀಲಿಸಬೇಕು; ಹೆಚ್ಚಿನ ಮೌಲ್ಯಗಳು ಹೆಚ್ಚು ಆಯ್ಕೆಗಳನ್ನು ತೋರಿಸುತ್ತವೆ ಆದರೆ ನಿಧಾನವಾಗಿರುತ್ತವೆ
- **Availability cache duration** — ಲಭ್ಯತೆ ಮತ್ತು ಬೆಲೆ ಫಲಿತಾಂಶಗಳನ್ನು ಎಷ್ಟು ಸಮಯದವರೆಗೆ ಕ್ಯಾಶ್ (cache) ಮಾಡಬೇಕು; ಕಡಿಮೆ ಮೌಲ್ಯಗಳು ಹೆಚ್ಚು ನಿಖರವಾಗುತ್ತವೆ ಆದರೆ API ಕರೆಗಳನ್ನು ಹೆಚ್ಚಿಸುತ್ತವೆ
- **Manage domain products** — ಉತ್ಪನ್ನಗಳ ಪಟ್ಟಿಗೆ ತ್ವರಿತ ಲಿಂಕ್
- **Configure providers** — ಪ್ರೊವೈಡರ್‌ಗಳನ್ನು ಸೇರಿಸಲು ಅಥವಾ ಮರು-ಕಾನ್ಫಿಗರ್ ಮಾಡಲು ಇಂಟಿಗ್ರೇಷನ್ ವಿಝಾರ್ಡ್ ಅನ್ನು ತೆರೆಯುತ್ತದೆ

### ಎರಡನೇ ಪ್ರೊವೈಡರ್ ಸೇರಿಸುವುದು

**Configure providers** ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಹೊಸ ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಾಗಿ ವಿಝಾರ್ಡ್ ಅನ್ನು ಮತ್ತೊಮ್ಮೆ ಚಲಾಯಿಸಿ. ನೀವು ಏಕಕಾಲದಲ್ಲಿ ಅನೇಕ ಪ್ರೊವೈಡರ್‌ಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಬಹುದು. ಪ್ರತಿ ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ನಿರ್ದಿಷ್ಟ ಪ್ರೊವೈಡರ್‌ಗೆ ನಿಯೋಜಿಸಿ, ಅಥವಾ ಅದನ್ನು ಡೀಫಾಲ್ಟ್‌ನಲ್ಲಿ ಬಿಡಿ.

### TLD ಗಳನ್ನು ಕೈಯಾರೆ ಸಿಂಕ್ ಮಾಡುವುದು (Syncing TLDs manually)

ಸೆಟ್ಟಿಂಗ್ಸ್ ಪುಟದಲ್ಲಿ, ಯಾವುದೇ ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಪ್ರೊವೈಡರ್ ಪಕ್ಕದಲ್ಲಿ **Sync TLDs** ಕ್ಲಿಕ್ ಮಾಡಿ, ಇದರಿಂದ ಇತ್ತೀಚಿನ ಬೆಲೆಗಳನ್ನು ಪಡೆಯಬಹುದು. ಪ್ರೊವೈಡರ್ ಹಾಲ್‌ಸೇಲ್ ಬೆಲೆಗಳನ್ನು ಅಪ್ಡೇಟ್ ಮಾಡಿದಾಗ ಅಥವಾ ಹೊಸ TLD ಗಳನ್ನು ಸೇರಿಸಿದಾಗ ಇದು ಉಪಯುಕ್ತವಾಗಿದೆ.

## ಲಾಗ್‌ಗಳು (Logs)

ಪ್ರತಿಯೊಂದು ಪ್ರೊವೈಡರ್ ತನ್ನದೇ ಆದ ಲಾಗ್ ಚಾನೆಲ್‌ಗೆ ಬರೆಯುತ್ತದೆ. ಲಾಗ್‌ಗಳನ್ನು **Network Admin › Ultimate Multisite › Logs** ಅಡಿಯಲ್ಲಿ ನೋಡಬಹುದು:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | ಎಲ್ಲಾ ನೋಂದಣಿ ಪ್ರಯತ್ನಗಳು ( ಯಶಸ್ಸು ಮತ್ತು ವೈಫಲ್ಯ) |
| `domain-seller-renewal` | ನವೀಕರಣದ ಕೆಲಸದ ಫಲಿತಾಂಶಗಳು |
| `domain-seller-opensrs` | ಕಚ್ಚಾ OpenSRS API ಚಟುವಟಿಕೆ |
| `domain-seller-namecheap` | ಕಚ್ಚಾ Namecheap API ಚಟುವಟಿಕೆ |
| `domain-seller-godaddy` | ಕಚ್ಚಾ GoDaddy API ಚಟುವಟಿಕೆ |
| `domain-seller-resellerclub` | ಕಚ್ಚಾ ResellerClub API ಚಟುವಟಿಕೆ |
| `domain-seller-namesilo` | ಕಚ್ಚಾ NameSilo API ಚಟುವಟಿಕೆ |
| `domain-seller-enom` | ಕಚ್ಚಾ Enom API ಚಟುವಟಿಕೆ |
