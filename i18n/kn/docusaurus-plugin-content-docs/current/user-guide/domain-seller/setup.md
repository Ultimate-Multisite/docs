# Domain Seller: ಸೆಟಪ್ ಮತ್ತು ಪೂರೈಕೆದಾರ ಸಂರಚನೆ

Domain Seller addon ಅಗತ್ಯವಿರುವ ಪ್ರತಿಯೊಂದು ಹಂತದ ಮೂಲಕ ನಿಮಗೆ ಮಾರ್ಗದರ್ಶನ ನೀಡುವ ಸೆಟಪ್ ವಿಜಾರ್ಡ್‌ನೊಂದಿಗೆ ಬರುತ್ತದೆ. ಈ ಪುಟವು ವಿಜಾರ್ಡ್ ಹರಿವು ಮತ್ತು ನಂತರ ಪೂರೈಕೆದಾರರನ್ನು ಹೇಗೆ ಸಂರಚಿಸಬೇಕು ಅಥವಾ ಮರುಸಂರಚಿಸಬೇಕು ಎಂಬುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಅವಶ್ಯಕತೆಗಳು {#requirements}

- **Multisite Ultimate** v2.4.12 ಅಥವಾ ಹೆಚ್ಚಿನದು, ನೆಟ್‌ವರ್ಕ್-ಸಕ್ರಿಯಗೊಳಿಸಿದ
- **PHP** 7.4+
- ಬೆಂಬಲಿತ ಕನಿಷ್ಠ ಒಂದು ನೋಂದಣಾಧಿಕಾರಿಗಾಗಿ API ರುಜುವಾತುಗಳು

## ಮೊದಲ-ಚಾಲನೆ ಸೆಟಪ್ ವಿಜಾರ್ಡ್ {#first-run-setup-wizard}

ನೀವು ಮೊದಲ ಬಾರಿ ಪ್ಲಗಿನ್ ಅನ್ನು ನೆಟ್‌ವರ್ಕ್-ಸಕ್ರಿಯಗೊಳಿಸಿದಾಗ ಸೆಟಪ್ ವಿಜಾರ್ಡ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ. ಇದು ಯಾವಾಗ ಬೇಕಾದರೂ **ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕ › Ultimate Multisite › Domain Seller ಸೆಟಪ್** ನಿಂದಲೂ ಲಭ್ಯವಿದೆ.

### ಹಂತ 1 — ಪೂರೈಕೆದಾರರನ್ನು ಆಯ್ಕೆಮಾಡಿ {#step-1--choose-a-provider}

ನೀವು ಸಂಪರ್ಕಿಸಲು ಬಯಸುವ ನೋಂದಣಾಧಿಕಾರಿಯನ್ನು ಆಯ್ಕೆಮಾಡಿ. ಬೆಂಬಲಿತ ಆಯ್ಕೆಗಳು:

| ಪೂರೈಕೆದಾರ | DNS ನಿರ್ವಹಣೆ | WHOIS ಗೌಪ್ಯತೆ |
|---|---|---|
| OpenSRS | ಹೌದು | ಹೌದು |
| Namecheap | ಇಲ್ಲ | ಹೌದು (WhoisGuard, ಉಚಿತ) |
| HostAfrica | ಹೌದು | ಹೌದು (ID ರಕ್ಷಣೆ) |
| Openprovider | ಹೌದು | ಹೌದು |
| Hostinger | ಹೋಸ್ಟ್ ಮಾಡಿದ ಡೊಮೇನ್‌ಗಳಿಗೆ ಮೂಲ Hostinger ಡೊಮೇನ್ ಮ್ಯಾಪಿಂಗ್ ಮೂಲಕ | ಹೌದು |
| GoDaddy | ಇಲ್ಲ | ಇಲ್ಲ |
| ResellerClub | ಹೌದು | ಇಲ್ಲ |
| NameSilo | ಇಲ್ಲ | ಇಲ್ಲ |
| Enom | ಹೌದು | ಇಲ್ಲ |

### ಹಂತ 2 — ರುಜುವಾತುಗಳನ್ನು ನಮೂದಿಸಿ {#step-2--enter-credentials}

ಪ್ರತಿಯೊಂದು ಪೂರೈಕೆದಾರರಿಗೂ ವಿಭಿನ್ನ ರುಜುವಾತು ಕ್ಷೇತ್ರಗಳಿವೆ:

**OpenSRS** — ಬಳಕೆದಾರಹೆಸರು ಮತ್ತು ಖಾಸಗಿ ಕೀ (OpenSRS Reseller Control Panel ನಿಂದ)

**Namecheap** — ಬಳಕೆದಾರಹೆಸರು ಮತ್ತು API ಕೀ (Account › Tools › API Access ನಿಂದ)

**HostAfrica** — HostAfrica ಮರುಮಾರಾಟಗಾರ ಮಾಡ್ಯೂಲ್‌ನಿಂದ Domains Reseller API endpoint ಮತ್ತು ರುಜುವಾತುಗಳು. ಪ್ರತ್ಯೇಕ sandbox endpoint ಪ್ರಸ್ತುತ ದಾಖಲಾಗಿಲ್ಲ; ಲೈವ್ ನೋಂದಣಿಗಳನ್ನು ನಡೆಸುವ ಮೊದಲು ಸುರಕ್ಷಿತ ಓದಲು-ಮಾತ್ರ ಪರಿಶೀಲನೆಗಳೊಂದಿಗೆ ಪರೀಕ್ಷಿಸಿ.

**Openprovider** — API ಪ್ರವೇಶ ಸಕ್ರಿಯಗೊಂಡ ಬಳಕೆದಾರಹೆಸರು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್. ಐಚ್ಛಿಕ sandbox ಮೋಡ್ Openprovider sandbox API ಅನ್ನು ಬಳಸುತ್ತದೆ, ಮತ್ತು ಐಚ್ಛಿಕ ಡೀಫಾಲ್ಟ್ ಗ್ರಾಹಕ ಹ್ಯಾಂಡಲ್ ಅನ್ನು ನೋಂದಣಿಗಳಿಗೆ ಮರುಬಳಸಬಹುದು.

**Hostinger** — ಮೂಲ Hostinger ಏಕೀಕರಣದಿಂದ ಹಂಚಿಕೊಂಡ Hostinger hPanel API token. ಅದೇ token ಮೂಲ ಡೊಮೇನ್ ಮ್ಯಾಪಿಂಗ್ ಮತ್ತು Domain Seller ನೋಂದಣಿ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಶಕ್ತಿ ನೀಡುತ್ತದೆ.

**GoDaddy** — API ಕೀ ಮತ್ತು ರಹಸ್ಯ (developer.godaddy.com ನಿಂದ)

**ResellerClub** — ಮರುಮಾರಾಟಗಾರ ID ಮತ್ತು API ಕೀ (ResellerClub ನಿಯಂತ್ರಣ ಫಲಕದಿಂದ)

**NameSilo** — API ಕೀ (namesilo.com › Account › API Manager ನಿಂದ)

**Enom** — Account ID ಮತ್ತು API token

ಲೈವ್‌ಗೆ ಹೋಗುವ ಮೊದಲು ಪೂರೈಕೆದಾರರ ಪರೀಕ್ಷಾ ಪರಿಸರದ ವಿರುದ್ಧ ಪರೀಕ್ಷಿಸಲು ಲಭ್ಯವಿದ್ದಲ್ಲಿ **Sandbox mode** ಪರಿಶೀಲಿಸಿ.

### ಹಂತ 3 — ಸಂಪರ್ಕವನ್ನು ಪರೀಕ್ಷಿಸಿ {#step-3--test-the-connection}

**ಸಂಪರ್ಕವನ್ನು ಪರೀಕ್ಷಿಸಿ** ಕ್ಲಿಕ್ ಮಾಡಿ. ರುಜುವಾತುಗಳು ಮತ್ತು ಸಂಪರ್ಕವನ್ನು ಪರಿಶೀಲಿಸಲು ವಿಜಾರ್ಡ್ ಹಗುರವಾದ API ಕರೆ ಕಳುಹಿಸುತ್ತದೆ. ಮುಂದುವರಿಯುವ ಮೊದಲು ಯಾವುದೇ ರುಜುವಾತು ಸಮಸ್ಯೆಗಳನ್ನು ಸರಿಪಡಿಸಿ.

### ಹಂತ 4 — TLDಗಳನ್ನು ಆಮದುಮಾಡಿ {#step-4--import-tlds}

ಸಂಪರ್ಕಿತ ಪೂರೈಕೆದಾರರಿಂದ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ TLDಗಳು ಮತ್ತು ಸಗಟು ಬೆಲೆಗಳನ್ನು ಪಡೆಯಲು **TLDಗಳನ್ನು ಆಮದುಮಾಡಿ** ಕ್ಲಿಕ್ ಮಾಡಿ. ಇದು ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಬಳಸುವ TLD ಪಟ್ಟಿಯನ್ನು ಭರ್ತಿ ಮಾಡುತ್ತದೆ. ದೊಡ್ಡ TLD ಕ್ಯಾಟಲಾಗ್‌ಗಳನ್ನು ಹೊಂದಿರುವ ಪೂರೈಕೆದಾರರಿಗೆ ಆಮದು 30–60 ಸೆಕೆಂಡುಗಳನ್ನು ತೆಗೆದುಕೊಳ್ಳಬಹುದು.

ನಿಗದಿತ cron job ಮೂಲಕ TLDಗಳನ್ನು ದಿನಕ್ಕೆ ಒಮ್ಮೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿಯೂ ಮರು-ಸಿಂಕ್ ಮಾಡಲಾಗುತ್ತದೆ.

### ಹಂತ 5 — ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸಿ {#step-5--create-a-domain-product}

ವಿಜಾರ್ಡ್ 10% ಮಾರ್ಕಪ್‌ನೊಂದಿಗೆ ಡೀಫಾಲ್ಟ್ ಕ್ಯಾಚ್-ಆಲ್ ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸುತ್ತದೆ. ನೀವು ಈ ಉತ್ಪನ್ನವನ್ನು ತಕ್ಷಣ ಸಂಪಾದಿಸಬಹುದು ಅಥವಾ ಬಿಟ್ಟುಬಿಟ್ಟು **Ultimate Multisite › ಉತ್ಪನ್ನಗಳು** ಅಡಿಯಲ್ಲಿ ಉತ್ಪನ್ನಗಳನ್ನು ಕೈಯಾರೆ ರಚಿಸಬಹುದು.

ಪೂರ್ಣ ಉತ್ಪನ್ನ ಸಂರಚನಾ ಮಾರ್ಗದರ್ಶಿಗಾಗಿ [ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಬೆಲೆನಿಗದಿ](./domain-products) ನೋಡಿ.

## ಪೂರೈಕೆದಾರರನ್ನು ಮರುಸಂರಚಿಸುವುದು {#reconfiguring-a-provider}

**ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕ › Ultimate Multisite › ಸೆಟ್ಟಿಂಗ್‌ಗಳು › Domain Seller** ಗೆ ಹೋಗಿ (ಅಥವಾ ಪ್ಲಗಿನ್ ಪಟ್ಟಿಯಲ್ಲಿ **ಸೆಟ್ಟಿಂಗ್‌ಗಳು** ಕ್ಲಿಕ್ ಮಾಡಿ).

ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಪುಟದಲ್ಲಿ ಇವುಗಳಿವೆ:

- **ಡೊಮೇನ್ ಮಾರಾಟವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ** — ಸಂಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಆನ್/ಆಫ್ ಮಾಡಲು ಟಾಗಲ್
- **ಡೀಫಾಲ್ಟ್ ಪೂರೈಕೆದಾರ** — ಡೊಮೇನ್ ಹುಡುಕಾಟಗಳು ಮತ್ತು ಹೊಸ ಉತ್ಪನ್ನಗಳಿಗೆ ಬಳಸುವ ಪೂರೈಕೆದಾರ
- **ಪ್ರತಿ ಹುಡುಕಾಟಕ್ಕೆ ಗರಿಷ್ಠ TLDಗಳು** — ಗ್ರಾಹಕರು ಹುಡುಕಿದಾಗ ಎಷ್ಟು TLDಗಳನ್ನು ಪರಿಶೀಲಿಸಬೇಕು; ಹೆಚ್ಚಿನ ಮೌಲ್ಯಗಳು ಹೆಚ್ಚು ಆಯ್ಕೆಗಳನ್ನು ತೋರಿಸುತ್ತವೆ ಆದರೆ ನಿಧಾನವಾಗಿರುತ್ತವೆ
- **ಲಭ್ಯತೆ ಕ್ಯಾಶ್ ಅವಧಿ** — ಲಭ್ಯತೆ ಮತ್ತು ಬೆಲೆನಿಗದಿ ಫಲಿತಾಂಶಗಳನ್ನು ಎಷ್ಟು ಕಾಲ ಕ್ಯಾಶ್ ಮಾಡಬೇಕು; ಕಡಿಮೆ ಮೌಲ್ಯಗಳು ಹೆಚ್ಚು ನಿಖರವಾಗಿರುತ್ತವೆ ಆದರೆ API ಕರೆಗಳನ್ನು ಹೆಚ್ಚಿಸುತ್ತವೆ
- **ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳನ್ನು ನಿರ್ವಹಿಸಿ** — ಉತ್ಪನ್ನಗಳ ಪಟ್ಟಿಗೆ ತ್ವರಿತ ಕೊಂಡಿ
- **ಪೂರೈಕೆದಾರರನ್ನು ಸಂರಚಿಸಿ** — ಪೂರೈಕೆದಾರರನ್ನು ಸೇರಿಸಲು ಅಥವಾ ಮರುಸಂರಚಿಸಲು ಏಕೀಕರಣ ವಿಜಾರ್ಡ್ ತೆರೆಯುತ್ತದೆ

### ಎರಡನೇ ಪೂರೈಕೆದಾರರನ್ನು ಸೇರಿಸುವುದು {#adding-a-second-provider}

**ಪೂರೈಕೆದಾರರನ್ನು ಸಂರಚಿಸಿ** ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಹೊಸ ನೋಂದಣಾಧಿಕಾರಿಗಾಗಿ ವಿಜಾರ್ಡ್ ಅನ್ನು ಮತ್ತೆ ಚಲಾಯಿಸಿ. ನೀವು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಅನೇಕ ಪೂರೈಕೆದಾರರನ್ನು ಸಂರಚಿಸಿರಬಹುದು. ಪ್ರತಿಯೊಂದು ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ನಿರ್ದಿಷ್ಟ ಪೂರೈಕೆದಾರರಿಗೆ ನಿಯೋಜಿಸಿ, ಅಥವಾ ಅದನ್ನು ಡೀಫಾಲ್ಟ್‌ನಲ್ಲೇ ಬಿಡಿ.

### TLDಗಳನ್ನು ಕೈಯಾರೆ ಸಿಂಕ್ ಮಾಡುವುದು {#syncing-tlds-manually}

ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಪುಟದಲ್ಲಿ, ಇತ್ತೀಚಿನ ಬೆಲೆನಿಗದಿಯನ್ನು ಪಡೆಯಲು ಯಾವುದೇ ಸಂರಚಿತ ಪೂರೈಕೆದಾರರ ಪಕ್ಕದಲ್ಲಿರುವ **TLDಗಳನ್ನು ಸಿಂಕ್ ಮಾಡಿ** ಕ್ಲಿಕ್ ಮಾಡಿ. ಪೂರೈಕೆದಾರರು ಸಗಟು ಬೆಲೆನಿಗದಿಯನ್ನು ನವೀಕರಿಸಿದ ನಂತರ ಅಥವಾ ಹೊಸ TLDಗಳನ್ನು ಸೇರಿಸಿದ ನಂತರ ಇದು ಉಪಯುಕ್ತ.

| `domain-seller-hostinger` | ಕಚ್ಚಾ Hostinger API ಚಟುವಟಿಕೆ |
| `domain-seller-godaddy` | ಕಚ್ಚಾ GoDaddy API ಚಟುವಟಿಕೆ |

## ಪೂರೈಕೆದಾರ ಸಾಮರ್ಥ್ಯ ಟಿಪ್ಪಣಿಗಳು {#provider-capability-notes}

ಪ್ರತಿಯೊಂದು ನೋಂದಣಾಧಿಕಾರಿ API ಒಂದೇ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸುವುದಿಲ್ಲ. ಮೌನವಾಗಿ ವಿಫಲಗೊಳ್ಳುವ ಬದಲು, ಬೆಂಬಲಿಸದ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು addon ಸ್ಪಷ್ಟ ನಿರ್ವಾಹಕ-ಮುಖದ ದೋಷಗಳೊಂದಿಗೆ ತೋರಿಸುತ್ತದೆ.

- **HostAfrica** lookup, TLD/pricing sync, registration, renewal, transfer, nameserver updates, DNS records, EPP codes, registrar lock, ಮತ್ತು ID protection ಒಳಗೊಂಡಂತೆ ಅತ್ಯಂತ ವ್ಯಾಪಕವಾದ ಲೈವ್ reseller ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.
- **Openprovider** reseller ಬೆಲೆಯ TLD sync, registration, renewal, transfers, nameserver updates, DNS zones, EPP codes, registrar lock, ಮತ್ತು WHOIS privacy ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಇದು ಅಲ್ಪಾವಧಿಯ bearer token ಮೂಲಕ ಪ್ರಮಾಣೀಕರಿಸುತ್ತದೆ; ಅದನ್ನು addon ಸ್ವಯಂಚಾಲಿತವಾಗಿ refresh ಮಾಡುತ್ತದೆ.
- **Hostinger** ಹಂಚಿಕೊಂಡ hPanel API token ಮೂಲಕ availability search, registration, portfolio lookup, nameserver updates, registrar lock, ಮತ್ತು WHOIS privacy ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. Hostinger‌ನ ಸಾರ್ವಜನಿಕ Domains API reseller/wholesale pricing, inbound transfer, explicit renewal, ಅಥವಾ EPP-code retrieval ಅನ್ನು ಬಹಿರಂಗಪಡಿಸುವುದಿಲ್ಲ; renewals auto-renew ಮಾತ್ರ.
