# ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಬೆಲೆ ನಿಗದಿ (Domain Products and Pricing)

ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಎಂದರೆ ನೀವು ಬೆಲೆ ನಿಗದಿ, TLDs, WHOIS privacy ಮತ್ತು ಪೂರೈಕೆದಾರರ ಆಯ್ಕೆಯನ್ನು ನಿಯಂತ್ರಿಸುವ ವಿಧಾನ. ಪ್ರತಿ ಡೊಮೇನ್ ಉತ್ಪನ್ನವು ಒಂದು standard Multisite Ultimate ಉತ್ಪನ್ನವಾಗಿದ್ದು, ಅದಕ್ಕೆ ಹೆಚ್ಚುವರಿ **Domain Settings** ಟ್ಯಾಬ್ ಇದೆ.

## ಡೊಮೇನ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸುವುದು (Creating a domain product) {#creating-a-domain-product}

1. **Network Admin › Ultimate Multisite › Products** ಗೆ ಹೋಗಿ
2. **Add New** ಕ್ಲಿಕ್ ಮಾಡಿ
3. ಉತ್ಪನ್ನದ ಪ್ರಕಾರವನ್ನು **Domain** ಎಂದು ಹೊಂದಿಸಿ
4. **Domain Settings** ಟ್ಯಾಬ್ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ (ಕೆಳಗೆ ನೋಡಿ)
5. ಸೇವ್ ಮಾಡಿ

ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಉತ್ಪನ್ನ ಪಟ್ಟಿಯಲ್ಲಿ ನೇರವಾದ ಬಣ್ಣದ **Domain** ಬ್ಯಾಡ್ಜ್‌ನೊಂದಿಗೆ ಕಾಣಿಸುತ್ತವೆ ಮತ್ತು **Domain Products** ಟ್ಯಾಬ್ ಬಳಸಿ ಫಿಲ್ಟರ್ ಮಾಡಬಹುದು.

## ಡೊಮೇನ್ ಸೆಟ್ಟಿಂಗ್ಸ್ ಟ್ಯಾಬ್ (Domain settings tab) {#domain-settings-tab}

### ಪೂರೈಕೆದಾರ (Provider) {#provider}

ಈ ಉತ್ಪನ್ನಕ್ಕಾಗಿ ನೋಂದಣಿಯನ್ನು ಯಾವ registrar ನಿರ್ವಹಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಆಯ್ಕೆಮಾಡಿ. ಇದು ಡೊಮೇನ್ ಸೆಲ್ಲರ್ ಸೆಟ್ಟಿಂಗ್ಸ್‌‌ನಲ್ಲಿ ಹೊಂದಿಸಲಾದ ಜಾಗತಿಕ **Default provider** ಗೆ ಡೀಫಾಲ್ಟ್ ಆಗಿರುತ್ತದೆ.

### ಬೆಂಬಲಿತ TLDs (Supported TLDs) {#supported-tlds}

ಎಲ್ಲಾ TLDಗಳಿಗೆ ಅನ್ವಯಿಸುವ **catch-all product** ಅನ್ನು ರಚಿಸಲು ಖಾಲಿ ಬಿಡಿ, ಇದು ಬೇರೆ ಯಾವುದೇ ಉತ್ಪನ್ನದಿಂದ ಹೊಂದಿಕೆಯಾಗದ ಎಲ್ಲಾ TLDಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ.

**TLD-ನಿರ್ದಿಷ್ಟ ಉತ್ಪನ್ನ** ಅನ್ನು ರಚಿಸಲು, ಕಮಾ-ಪ್ರತ್ಯೇಕ ಪಟ್ಟಿಯ TLDಗಳನ್ನು ನಮೂದಿಸಿ (ಉದಾಹರಣೆಗೆ, `.com, .net, .org`) ಇದು ಆ ವಿಸ್ತರಣೆಗಳಿಗೆ ಮಾತ್ರ ಅನ್ವಯಿಸುತ್ತದೆ.

**ಉತ್ಪನ್ನ ಹೊಂದಿಕೆಯ ವಿಧಾನ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ:** ಗ್ರಾಹಕರು ಡೊಮೇನ್ ಹುಡುಕಿದಾಗ, ಆಡ್-ಆನ್ ಅತ್ಯಂತ ನಿರ್ದಿಷ್ಟವಾಗಿ ಹೊಂದಿಕೆಯಾಗುವ ಉತ್ಪನ್ನವನ್ನು ಆರಿಸುತ್ತದೆ. ಅದರ TLD ಪಟ್ಟಿಯಲ್ಲಿ `.com` ಇರುವ ಉತ್ಪನ್ನವು catch-all ಉತ್ಪನ್ನಕ್ಕಿಂತ ಹೆಚ್ಚಿನ ಪ್ರಾಮುಖ್ಯತೆಯನ್ನು ಪಡೆಯುತ್ತದೆ. ಯಾವುದೇ TLD-ನಿರ್ದಿಷ್ಟ ಉತ್ಪನ್ನ ಹೊಂದಿಕೆಯಾಗದಿದ್ದರೆ, catch-all ಅನ್ನು ಬಳಸಲಾಗುತ್ತದೆ. ಯಾವುದೇ ಉತ್ಪನ್ನಗಳು ಇಲ್ಲದಿದ್ದರೆ, ಡೊಮೇನ್ ಹುಡುಕಾಟವನ್ನು ತೋರಿಸುವುದಿಲ್ಲ.

### ಮಾರ್ಕ್ಅಪ್ ಪ್ರಕಾರ (Markup type) {#markup-type}

ನಿಮ್ಮ ಚಿಲ್ಲರೆ ಬೆಲೆ (retail price) ಅನ್ನು ಬಿಳಿಕೆ ವೆಚ್ಚದಿಂದ (wholesale cost) ಹೇಗೆ ಲೆಕ್ಕಾಚಾರ ಮಾಡಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ಮೂರು ಮೋಡ್‌ಗಳು ನಿಯಂತ್ರಿಸುತ್ತವೆ:

| ಮೋಡ್ (Mode) | ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ (How it works) |
|---|---|
| **Percentage** | ಬಿಳಿಕೆ ವೆಚ್ಚದ ಮೇಲೆ ಒಂದು ಶೇಕಡಾವಾರು ಪ್ರಮಾಣವನ್ನು ಸೇರಿಸುತ್ತದೆ. $10 ಬಿಳಿಕೆ ಡೊಮೇನ್‌ಗೆ 20% ಮಾರ್ಕ್ಅಪ್ $12 ನೀಡುತ್ತದೆ. |
| **Fixed markup** | ಒಂದು ನಿಗದಿತ ಡಾಲರ್ ಮೊತ್ತವನ್ನು ಸೇರಿಸುತ್ತದೆ. $10 ಡೊಮೇನ್‌ಗೆ $5 ಮಾರ್ಕ್ಅಪ್ $15 ನೀಡುತ್ತದೆ. |
| **Fixed price** | ಬಿಳಿಕೆ ವೆಚ್ಚವನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ನಿರ್ಲಕ್ಷಿಸುತ್ತದೆ. ನೀವು ನಮೂದಿಸಿದ ಮೊತ್ತವನ್ನು ಯಾವಾಗಲೂ ವಿಧಿಸುತ್ತದೆ. |

### ಪರಿಚಯಾತ್ಮಕ ಬೆಲೆ ನಿಗದಿ (Introductory pricing) {#introductory-pricing}

ರಿಯಾಯಿತಿ ಪಡೆದ ಮೊದಲ ವರ್ಷದ ಬೆಲೆಯನ್ನು ನೀಡಲು ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ. ಸಾಮಾನ್ಯ **Renewal price** (ವರ್ಷ 2+) ಜೊತೆಗೆ ಪ್ರತ್ಯೇಕ **Introductory price** (ವರ್ಷ-1 ಬೆಲೆ) ಅನ್ನು ಹೊಂದಿಸಿ. ಗ್ರಾಹಕರು ಚೆಕ್‌ಔಟ್ ಸಮಯದಲ್ಲಿ ಎರಡೂ ಬೆಲೆಗಳನ್ನು ನೋಡುತ್ತಾರೆ, ಇದರಿಂದ ಅವರು ನವೀಕರಣದ ಸಮಯದಲ್ಲಿ ಏನನ್ನು ನಿರೀಕ್ಷಿಸಬಹುದು ಎಂದು ತಿಳಿಯುತ್ತದೆ.

### WHOIS privacy {#whois-privacy}

ಈ ಉತ್ಪನ್ನದ ಮೂಲಕ ನೋಂದಾಯಿಸಲಾದ ಡೊಮೇನ್‌ಗಳಿಗೆ WHOIS privacy ರಕ್ಷಣೆ ನೀಡಲಾಗುತ್ತದೆಯೇ ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ.

| ಸೆಟ್ಟಿಂಗ್ (Setting) | ವರ್ತನೆ (Behaviour) |
|---|---|
| **Disabled** | WHOIS privacy ಅನ್ನು ಎಂದಿಗೂ ನೀಡಲಾಗುವುದಿಲ್ಲ ಅಥವಾ ಸಕ್ರಿಯಗೊಳಿಸಲಾಗುವುದಿಲ್ಲ. |
| **Always Included** | WHOIS privacy ಅನ್ನು ನೋಂದಣಿಯ ಸಮಯದಲ್ಲಿ ಯಾವುದೇ ಶುಲ್ಕವಿಲ್ಲದೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸಕ್ರಿಯಗೊಳಿಸಲಾಗುತ್ತದೆ. |
| **Customer Choice** | ಚೆಕ್‌ಔಟ್ ಸಮಯದಲ್ಲಿ ಒಂದು ಚೆಕ್‌ಬಾಕ್ಸ್ ಕಾಣಿಸುತ್ತದೆ. ಪ್ರತಿ ವರ್ಷದ ಆಧಾರದ ಮೇಲೆ ಶುಲ್ಕ ವಿಧಿಸಲು **Privacy price** ಅನ್ನು ಹೊಂದಿಸಿ, ಅಥವಾ ಉಚಿತವಾಗಿ ನೀಡಲು ಅದನ್ನು $0 ನಲ್ಲಿ ಬಿಡಿ. |

Namecheap-ನ ಸಂದರ್ಭದಲ್ಲಿ, WHOIS privacy ಗೆ WhoisGuard (ಎಲ್ಲಾ ಸಮಯದಲ್ಲೂ ಉಚಿತ) ಬಳಸಲಾಗುತ್ತದೆ. OpenSRS-ನ ಸಂದರ್ಭದಲ್ಲಿ, ಇದು OpenSRS privacy ಸೇವೆಯನ್ನು ಬಳಸುತ್ತದೆ (ಬಿಳಿಕೆ ವೆಚ್ಚದಲ್ಲಿ ಶುಲ್ಕವಿರಬಹುದು).

## TLD ಆಮದು ಮತ್ತು ಸಿಂಕ್ (TLD import and sync) {#tld-import-and-sync}

ಡೊಮೇನ್ ಉತ್ಪನ್ನಗಳು ಸಂಪರ್ಕಗೊಂಡ ಪೂರೈಕೆದಾರರಿಂದ ಪಡೆದ ನೈಜ-ಸಮಯದ ಬಿಳಿಕೆ ಬೆಲೆಗಳನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತವೆ. ಇದು ಕೆಲಸ ಮಾಡಲು, TLDಗಳನ್ನು ಆಮದು ಮಾಡಿಕೊಳ್ಳಬೇಕು.

- **Manual sync:** Settings › Domain Seller › Sync TLDs (per provider)
- **Automatic sync:** ಎಲ್ಲಾ ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಪೂರೈಕೆದಾರರಾದ್ಯಂತ ವೇಳಾಪಟ್ಟಿಯ cron job ಮೂಲಕ ಪ್ರತಿದಿನ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ

ಸಿಂಕ್ ಮಾಡಿದ ನಂತರ, ಯಾವುದೇ ಡೊಮೇನ್ ಉತ್ಪನ್ನದ Domain Settings ಟ್ಯಾಬ್‌ಗೆ ಹೋಗಿ, ಲಭ್ಯವಿರುವ TLDಗಳನ್ನು ಅವುಗಳ ಪ್ರಸ್ತುತ ಬಿಳಿಕೆ ಬೆಲೆಗಳೊಂದಿಗೆ ನೋಡಲು TLD ಪಿಕರ್ ಅನ್ನು ಬಳಸಿ.

## ಸ್ವಯಂಚಾಲಿತ ನವೀಕರಣ (Auto-renewal) {#auto-renewal}

ಡೊಮೇನ್ ನವೀಕರಣಗಳು ಗ್ರಾಹಕರ ಸದಸ್ಯತ್ವದ ಸ್ಥಿತಿಗೆ ಜೋಡಿಸಲ್ಪಟ್ಟಿರುತ್ತವೆ:

- ಸದಸ್ಯತ್ವ ನವೀಕರಿಸಲ್ಪಟ್ಟಾಗ ಮತ್ತು ಡೊಮೇನ್ ಲಿಂಕ್ ಆಗಿದ್ದರೆ, ಡೊಮೇನ್ ನವೀಕರಣವು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಕ್ಯೂ (queued) ಆಗುತ್ತದೆ
- ನವೀಕರಣ ಪ್ರಯತ್ನಗಳು ಗ್ರಾಹಕರ ಸಕ್ರಿಯ ಪಾವತಿ 게ವೇ (payment gateway) ಅನ್ನು ಬಳಸುತ್ತವೆ
- ವಿಫಲವಾದ ನವೀಕರಣಗಳನ್ನು ಎಕ್ಸ್‌ಪೋನೆನ್ಷಿಯಲ್ ಬ್ಯಾಕ್‌ಆಫ್ (exponential backoff) ಜೊತೆಗೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪುನರಾವರ್ತಿಸಲಾಗುತ್ತದೆ
- ಯಶಸ್ವಿ ನವೀಕರಣಗಳು, ವೈಫಲ್ಯಗಳು ಮತ್ತು ಮುಂಬರುವ ಅವಧಿ ಮುಗಿಯುವಿಕೆಗಳಿಗಾಗಿ ಇಮೇಲ್ ಅಧಿಸೂಚನೆಗಳನ್ನು ಕಳುಹಿಸಲಾಗುತ್ತದೆ

ಡೊಮೇನ್ life cycle ಘಟನೆಗಳಿಗಾಗಿ ಇಮೇಲ್ ಟೆಂಪ್ಲೇಟ್ IDಗಳು:

| ಘಟನೆ (Event) | ಟೆಂಪ್ಲೇಟ್ ID (Template ID) |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## ಆಡ್ಮಿನ್: ಕೈಯಾರೆ ಡೊಮೇನ್ ನೋಂದಣಿ (Admin: Manual domain registration) {#admin-manual-domain-registration}

ಗ್ರಾಹಕರು ಚೆಕ್‌ಔಟ್ ಮೂಲಕ ಹೋಗದೆ ಅವರ ಪರವಾಗಿ ಡೊಮೇನ್ ಅನ್ನು ನೋಂದಾಯಿಸಲು:

1. **Network Admin › Ultimate Multisite › Register Domain** ಗೆ ಹೋಗಿ
2. ಗ್ರಾಹಕರನ್ನು ಆಯ್ಕೆಮಾಡಿ ಮತ್ತು ಡೊಮೇನ್ ಹೆಸರನ್ನು ನಮೂದಿಸಿ
3. ನೋಂದಣಾ ಸಂಪರ್ಕ ವಿವರಗಳನ್ನು ಭರ್ಪಡಿಸಿ (ಹೆಸರು, ವಿಳಾಸ, ಫೋನ್)
4. **Register** ಕ್ಲಿಕ್ ಮಾಡಿ

ಡೊಮೇನ್ ದಾಖಲೆಯನ್ನು ರಚಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಗ್ರಾಹಕರ ಖಾತೆಗೆ ಲಿಂಕ್ ಮಾಡಲಾಗುತ್ತದೆ.
