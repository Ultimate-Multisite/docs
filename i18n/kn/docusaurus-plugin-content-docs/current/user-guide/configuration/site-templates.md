# ಸೈಟ್ ಟೆಂಪ್ಲೇಟ್‌ಗಳು (v2)

_**ಗಮನಿಸಿ: ಈ ಲೇಖನವು Ultimate Multisite ಆವೃತ್ತಿ 2.x ಗೆ ಸಂಬಂಧಿಸಿದೆ. ನೀವು ಆವೃತ್ತಿ 1.x ಬಳಸುತ್ತಿದ್ದರೆ,**_ **ಈ ಲೇಖನವನ್ನು ನೋಡಿ** _**.**_

Ultimate Multisite ಜೊತೆ ಪ್ರೀಮಿಯಂ ನೆಟ್‌ವರ್ಕ್ ರಚಿಸುವಾಗ ನಮ್ಮ ಗುರಿ ಏನೆಂದರೆ, ಸಾಧ್ಯವಾದಷ್ಟು ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು ಮತ್ತು ಗ್ರಾಹಕರು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ರಚಿಸುವಾಗ ಹೊಂದಿಕೊಳ್ಳುವಿಕೆ ಮತ್ತು ವಿವಿಧ ಆಯ್ಕೆಗಳನ್ನು ನೀಡುವುದು. ಈ ಸಮತೋಲನ ಸಾಧಿಸಲು Ultimate Multisite Site Templates ವೈಶಿಷ್ಟ್ಯವನ್ನು ಬಳಸುವುದು ಒಂದು ಸುಲಭ ಮಾರ್ಗ.

## Site Template ಎಂದರೇನು?

ಹೆಸರೇ ಸೂಚಿಸುವಂತೆ, Site Template ಎಂಬುದು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಹೊಸ ಸೈಟ್‌ಗಳನ್ನು ರಚಿಸುವಾಗ ಆಧಾರವಾಗಿ ಬಳಸಬಹುದಾದ ಮೂಲ ಸೈಟ್ ಆಗಿದೆ.

ಇದರರ್ಥ ನೀವು ಒಂದು ಮೂಲ ಸೈಟ್ ರಚಿಸಬಹುದು, ವಿವಿಧ plugins ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು, ಸಕ್ರಿಯ theme ಹೊಂದಿಸಬಹುದು ಮತ್ತು ನಿಮಗೆ ಬೇಕಾದ ರೀತಿಯಲ್ಲಿ ಅದನ್ನು ಕಸ್ಟಮೈಜ್ ಮಾಡಬಹುದು. ನಂತರ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ಹೊಸ ಖಾತೆ ರಚಿಸಿದಾಗ, ಅರ್ಥಪೂರ್ಣ ವಿಷಯವಿಲ್ಲದ ಡೀಫಾಲ್ಟ್ WordPress ಸೈಟ್ ಪಡೆಯುವ ಬದಲು, ಎಲ್ಲಾ ಕಸ್ಟಮೈಸೇಷನ್‌ಗಳು ಮತ್ತು ವಿಷಯಗಳು ಈಗಾಗಲೇ ಇರುವ ನಿಮ್ಮ ಮೂಲ ಸೈಟ್‌ನ ನಕಲು ಅವರಿಗೆ ಸಿಗುತ್ತದೆ.

ಇದು ಅದ್ಭುತವಾಗಿದೆ, ಆದರೆ ಹೊಸ site template ಹೇಗೆ ರಚಿಸುವುದು? ಇದು ಸಾಧ್ಯವಾದಷ್ಟು ಸರಳವಾಗಿದೆ.

## ಹೊಸ Site Template ರಚಿಸುವುದು ಮತ್ತು ಸಂಪಾದಿಸುವುದು

Site Templates ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿರುವ ಸಾಮಾನ್ಯ ಸೈಟ್‌ಗಳೇ ಆಗಿವೆ. ಹೊಸ template ರಚಿಸಲು ನೀವು ಸರಳವಾಗಿ **Network Admin > Ultimate Multisite > Sites > Add Site** ಗೆ ಹೋಗಬಹುದು.

**![Sites ಪಟ್ಟಿ ಪುಟದಲ್ಲಿ Add site ಬಟನ್](/img/config/site-templates-list.png)**

ಇದು ಒಂದು modal ವಿಂಡೋ ತೆರೆಯುತ್ತದೆ, ಅಲ್ಲಿ **Site title, Site Domain/path,** ಮತ್ತು **Site type** ಕೇಳಲಾಗುತ್ತದೆ. **Site Type** ಡ್ರಾಪ್-ಡೌನ್ ಕ್ಷೇತ್ರದಲ್ಲಿ ನೀವು **Site Template** ಆಯ್ಕೆ ಮಾಡಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ*.*

_![Site type dropdown ಇರುವ Add site template modal](/img/config/site-templates-list.png)_

ಫಾರ್ಮ್‌ನ ಕೆಳಭಾಗದಲ್ಲಿ, ನೀವು **Copy Site** ಟಾಗಲ್ ಸ್ವಿಚ್ ಗಮನಿಸಬಹುದು. ಮೊದಲಿನಿಂದ site template ರಚಿಸುವ ಬದಲು ಸಮಯ ಉಳಿಸಲು ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ site template ಅನ್ನು ಆರಂಭಿಕ ಹಂತವಾಗಿ ಬಳಸಿ ಹೊಸ site template ರಚಿಸಲು ಇದು ಅನುಮತಿಸುತ್ತದೆ.

![Copy site toggle ಇರುವ Add site template modal](/img/config/site-templates-list.png)

### Site Template ನ ವಿಷಯಗಳನ್ನು ಕಸ್ಟಮೈಜ್ ಮಾಡುವುದು

ನಿಮ್ಮ site template ಕಸ್ಟಮೈಜ್ ಮಾಡಲು, ಅದರ dashboard panel ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ ಮತ್ತು ಬೇಕಾದ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿ. ನೀವು ಹೊಸ posts, pages ರಚಿಸಬಹುದು, plugins ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು ಮತ್ತು ಸಕ್ರಿಯ theme ಬದಲಾಯಿಸಬಹುದು. Customizer ಗೆ ಹೋಗಿ ಎಲ್ಲಾ ರೀತಿಯ ಕಸ್ಟಮೈಸೇಷನ್ ಆಯ್ಕೆಗಳನ್ನು ಸಹ ಬದಲಾಯಿಸಬಹುದು.

ಗ್ರಾಹಕರು ಆ Site Template ಆಧಾರದ ಮೇಲೆ ಹೊಸ ಸೈಟ್ ರಚಿಸಿದಾಗ ಆ ಎಲ್ಲಾ ಡೇಟಾ ನಕಲಾಗುತ್ತದೆ.

### ಸುಧಾರಿತ ಆಯ್ಕೆಗಳು

ನೀವು ಕಸ್ಟಮ್ ಕೋಡಿಂಗ್ ಬಗ್ಗೆ ತಿಳಿದಿದ್ದರೆ, ಹೊಸ ಸೈಟ್ ರಚನೆಯಾದ ನಂತರ ಮಾಹಿತಿಯನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಾಯಿಸಲು ನಮ್ಮ Search and Replace API ಬಳಸಬಹುದು. About ಪುಟದಲ್ಲಿ ಕಂಪನಿ ಹೆಸರುಗಳನ್ನು ಬದಲಾಯಿಸುವುದು, Contact ಪುಟದಲ್ಲಿ ಸಂಪರ್ಕ ಇಮೇಲ್ ಬದಲಾಯಿಸುವುದು ಮುಂತಾದವುಗಳಿಗೆ ಇದು ಉಪಯುಕ್ತ.

### Site Templates ಬಳಸುವುದು

ಸರಿ, ನೀವು ವಿವಿಧ ವಿನ್ಯಾಸಗಳು, themes ಮತ್ತು ಸೆಟ್ಟಿಂಗ್‌ಗಳೊಂದಿಗೆ ಹಲವಾರು Site Templates ರಚಿಸಿದ್ದೀರಿ. ಈಗ ಅವುಗಳನ್ನು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಉಪಯುಕ್ತವಾಗಿ ಹೇಗೆ ಮಾಡುವುದು?

ಮೂಲಭೂತವಾಗಿ, ನೀವು ಈಗ ಬಳಸಬಹುದಾದ ಎರಡು ವಿಧಾನಗಳಿವೆ (ಏಕಕಾಲದಲ್ಲಿ ಅಲ್ಲ):

  * ನಿಮ್ಮ ಪ್ರತಿ Plan ಗೆ ಒಂದು Site Template ಜೋಡಿಸುವುದು

**ಅಥವಾ**

  * ಸೈನ್-ಅಪ್ ಸಮಯದಲ್ಲಿ ನಿಮ್ಮ ಗ್ರಾಹಕರು site templates ಆಯ್ಕೆ ಮಾಡಲು ಅನುಮತಿಸುವುದು.

#### ಮೋಡ್ 1: Site Template ನಿಯೋಜಿಸಿ

ಈ ಮೋಡ್‌ನಲ್ಲಿ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ಖಾತೆ ರಚಿಸುವಾಗ template ಆಯ್ಕೆ ಮಾಡಲು ಸಾಧ್ಯವಾಗುವುದಿಲ್ಲ, ಬದಲಿಗೆ ನಿಮ್ಮ ಪ್ರತಿ Plan ಗೆ ಯಾವ template ಬಳಸಬೇಕೆಂದು ನೀವು ನಿರ್ಧರಿಸುತ್ತೀರಿ.

ಇದನ್ನು ಮಾಡಲು, ನೀವು **Ultimate Multisite > Products > Edit** ಗೆ ಹೋಗಬೇಕು.

![Site template ನಿಯೋಜಿಸಲು Edit product](/img/config/product-site-templates.png)

ಇದು ನಿಮ್ಮನ್ನು **Edit Product** ಪುಟಕ್ಕೆ ಕರೆದೊಯ್ಯುತ್ತದೆ. **Product Options** ವಿಭಾಗದಲ್ಲಿ, **Site template** ಟ್ಯಾಬ್ ಹುಡುಕಿ ಮತ್ತು ಡ್ರಾಪ್-ಡೌನ್ ಕ್ಷೇತ್ರದಿಂದ **Assign Site Template** ಆಯ್ಕೆ ಆರಿಸಿ. ಇದು ಲಭ್ಯವಿರುವ site templates ಪಟ್ಟಿ ತೋರಿಸುತ್ತದೆ ಮತ್ತು product ಗೆ ಸಮರ್ಪಿತವಾದ ಒಂದು site template ಮಾತ್ರ ಆಯ್ಕೆ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.

![Assign site template ಆಯ್ಕೆ ಇರುವ Product site template ಟ್ಯಾಬ್](/img/config/product-site-templates.png)

#### ಮೋಡ್ 2: ಲಭ್ಯವಿರುವ Site Template ಆಯ್ಕೆ ಮಾಡಿ

ಈ ಮೋಡ್‌ನಲ್ಲಿ, ಸೈನ್-ಅಪ್ ಪ್ರಕ್ರಿಯೆಯ ಸಮಯದಲ್ಲಿ ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಆಯ್ಕೆ ನೀಡುತ್ತೀರಿ. product ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಅಡಿಯಲ್ಲಿ ನೀವು ನಿರ್ಧರಿಸುವ ವಿವಿಧ site templates ಗಳಿಂದ ಅವರು ಆಯ್ಕೆ ಮಾಡಬಹುದು. ಆಯ್ಕೆ ಮಾಡಿದ product ಅಡಿಯಲ್ಲಿ ಅವರು ಆಯ್ಕೆ ಮಾಡಬಹುದಾದ site template ಅನ್ನು ಮಿತಿಗೊಳಿಸುವ ಆಯ್ಕೆ ನಿಮಗಿದೆ. ಹೆಚ್ಚಿನ ಬೆಲೆಯ product ಗೆ ವಿಭಿನ್ನ ಕಾರ್ಯಗಳು ಮತ್ತು ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಹೈಲೈಟ್ ಮಾಡಲು ಪ್ರತಿ product ಅಡಿಯಲ್ಲಿ ವಿಭಿನ್ನ site templates ಸೆಟ್‌ಗಳನ್ನು ಹೊಂದಲು ಇದು ಸೂಕ್ತ.

**Edit Product** ಪುಟದಲ್ಲಿ, **Product Options** ವಿಭಾಗದಲ್ಲಿ, **Site template** ಟ್ಯಾಬ್ ಹುಡುಕಿ ಮತ್ತು ಡ್ರಾಪ್-ಡೌನ್ ಕ್ಷೇತ್ರದಿಂದ **Choose Available Site Template** ಆಯ್ಕೆ ಆರಿಸಿ. ಇದು ಲಭ್ಯವಿರುವ site templates ಪಟ್ಟಿ ತೋರಿಸುತ್ತದೆ ಮತ್ತು ಲಭ್ಯವಾಗಬೇಕಾದ site template ಆಯ್ಕೆ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ. ಅದರ Behavior ಆಯ್ಕೆ ಮಾಡುವ ಮೂಲಕ ಇದನ್ನು ಮಾಡಬಹುದು: ಪಟ್ಟಿಯಲ್ಲಿ site template ಸೇರಿಸಬೇಕಾದರೆ **Available**. ಆಯ್ಕೆಯಾಗಿ ತೋರಿಸಬಾರದೆಂದರೆ _**Not Available**_. ಮತ್ತು ಪಟ್ಟಿಯಲ್ಲಿರುವ site templates ಗಳಲ್ಲಿ ಒಂದನ್ನು ಡೀಫಾಲ್ಟ್ ಆಯ್ಕೆಯಾಗಿ ಬೇಕಾದರೆ **Pre-selected**.

![Behavior ಆಯ್ಕೆಗಳೊಂದಿಗೆ Choose available site templates](/img/config/product-site-templates.png)

### ಡೀಫಾಲ್ಟ್ ಮೋಡ್: Checkout form ನಲ್ಲಿ Site template ಆಯ್ಕೆ

ನೋಂದಣಿ ಸಮಯದಲ್ಲಿ ನಿಮ್ಮ ಎಲ್ಲಾ site templates ಲಭ್ಯವಾಗಬೇಕೆಂದು ಬಯಸಿದರೆ, ಅಥವಾ ನೀವು ರಚಿಸುವ ಪ್ರತಿ product ಅಡಿಯಲ್ಲಿ site templates ನಿಯೋಜಿಸುವ ಅಥವಾ ನಿರ್ದಿಷ್ಟಪಡಿಸುವ ಹೆಚ್ಚುವರಿ ಕೆಲಸ ಮಾಡಲು ಬಯಸದಿದ್ದರೆ. ನಿಮ್ಮ **Checkout Form** ಅಡಿಯಲ್ಲಿ site template ಆಯ್ಕೆಯನ್ನು ಸರಳವಾಗಿ ಹೊಂದಿಸಬಹುದು. ಇದನ್ನು ಮಾಡಲು, **Ultimate Multisite > Checkout Forms** ಗೆ ಹೋಗಿ. ನಂತರ ಕಾನ್ಫಿಗರ್ ಮಾಡಲು ಬಯಸುವ form ಅಡಿಯಲ್ಲಿ **Edit** ಕ್ಲಿಕ್ ಮಾಡಿ.

![Checkout forms ಪಟ್ಟಿ ಪುಟ](/img/config/checkout-forms-list.png)

ಇದು **Edit Checkout Form** ಪುಟ ತೆರೆಯುತ್ತದೆ. **Template Selection** ಕ್ಷೇತ್ರ ಹುಡುಕಿ ಮತ್ತು ಅದರ ಅಡಿಯಲ್ಲಿ **Edit** ಕ್ಲಿಕ್ ಮಾಡಿ.

![Template selection field ಇರುವ Checkout form editor](/img/config/checkout-form-editor.png)

ಒಂದು modal ವಿಂಡೋ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತದೆ. **Template Sites** ಕ್ಷೇತ್ರದಲ್ಲಿ ನೋಂದಣಿ ಸಮಯದಲ್ಲಿ ಲಭ್ಯವಾಗಬೇಕಾದ ಎಲ್ಲಾ site templates ಆಯ್ಕೆ ಮಾಡಿ ಪಟ್ಟಿ ಮಾಡಬಹುದು. ಇಲ್ಲಿ ನೀವು ನಿರ್ದಿಷ್ಟಪಡಿಸುವ site templates ಬಳಕೆದಾರರು ಯಾವುದೇ product ಆಯ್ಕೆ ಮಾಡಿದರೂ ಲಭ್ಯವಾಗುತ್ತವೆ.

![Checkout form editor ನಲ್ಲಿ Template sites ಕ್ಷೇತ್ರ](/img/config/checkout-form-step.png)

### Site Template ಆಯ್ಕೆಗಳು

Ultimate Multisite ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಅಡಿಯಲ್ಲಿ ಆನ್ ಅಥವಾ ಆಫ್ ಮಾಡಬಹುದಾದ ಇತರ site templates ಕಾರ್ಯಗಳಿವೆ.

![Ultimate Multisite ಸೆಟ್ಟಿಂಗ್‌ಗಳಲ್ಲಿ Site template ಆಯ್ಕೆಗಳು](/img/config/settings-sites.png)

#### Template ಬದಲಾಯಿಸಲು ಅನುಮತಿಸಿ

ಈ ಆಯ್ಕೆಯನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುವುದರಿಂದ ಖಾತೆ ಮತ್ತು ಸೈಟ್ ರಚನೆಯಾದ ನಂತರ ಸೈನ್-ಅಪ್ ಪ್ರಕ್ರಿಯೆಯಲ್ಲಿ ಆಯ್ಕೆ ಮಾಡಿದ template ಅನ್ನು ನಿಮ್ಮ ಗ್ರಾಹಕರು ಬದಲಾಯಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ತಮ್ಮ ಮೂಲ ಆಯ್ಕೆ ತಮ್ಮ ನಿರ್ದಿಷ್ಟ ಅಗತ್ಯಗಳಿಗೆ ಉತ್ತಮವಾಗಿಲ್ಲ ಎಂದು ನಂತರ ಕಂಡುಕೊಂಡರೆ template ಮರುಆಯ್ಕೆ ಮಾಡಲು ಅನುಮತಿಸುವುದರಿಂದ ಇದು ಗ್ರಾಹಕರ ದೃಷ್ಟಿಕೋನದಿಂದ ಉಪಯುಕ್ತ.

#### ಬಳಕೆದಾರರು ತಮ್ಮ ಸೈಟ್ ಅನ್ನು templates ಆಗಿ ಬಳಸಲು ಅನುಮತಿಸಿ

subsite ಬಳಕೆದಾರರು ತಮ್ಮ ಸ್ವಂತ ಸೈಟ್ ನಿರ್ಮಿಸಲು ಮತ್ತು ವಿನ್ಯಾಸಗೊಳಿಸಲು ಸಮಯ ಕಳೆದಿರುವುದರಿಂದ, ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಇನ್ನೊಂದು subsite ರಚಿಸುವಾಗ ಲಭ್ಯವಿರುವ site templates ಗಳಲ್ಲಿ ಒಂದಾಗಿ ಅದನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ ಬಳಸಲು ಬಯಸಬಹುದು. ಈ ಆಯ್ಕೆ ಅವರಿಗೆ ಅದನ್ನು ಸಾಧಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.

#### Template ನಕಲು ಮಾಡುವಾಗ ಮೀಡಿಯಾ ನಕಲಿಸಿ

ಈ ಆಯ್ಕೆಯನ್ನು ಟಿಕ್ ಮಾಡುವುದರಿಂದ template ಸೈಟ್‌ನಲ್ಲಿ ಅಪ್‌ಲೋಡ್ ಮಾಡಿದ ಮೀಡಿಯಾವನ್ನು ಹೊಸದಾಗಿ ರಚಿಸಿದ ಸೈಟ್‌ಗೆ ನಕಲಿಸುತ್ತದೆ. ಪ್ರತಿ plan ನಲ್ಲಿ ಇದನ್ನು ಬದಲಾಯಿಸಬಹುದು.

#### **Search Engines Site Templates ಅನ್ನು index ಮಾಡುವುದನ್ನು ತಡೆಯಿರಿ**

ಈ ಲೇಖನದಲ್ಲಿ ಚರ್ಚಿಸಿದಂತೆ Site templates ಮೂಲ ಸೈಟ್‌ಗಳಾಗಿದ್ದರೂ ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನ ಭಾಗವಾಗಿವೆ, ಅಂದರೆ search engines ಕಂಡುಹಿಡಿಯಲು ಇನ್ನೂ ಲಭ್ಯವಿವೆ. ಈ ಆಯ್ಕೆ search engines index ಮಾಡದಂತೆ site templates ಅನ್ನು ಮರೆಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.

## ಸ್ವಯಂ search-and-replace ನೊಂದಿಗೆ Site Templates ಅನ್ನು ಪೂರ್ವ-ಭರ್ತಿ ಮಾಡುವುದು

Ultimate Multisite ನ ಅತ್ಯಂತ ಶಕ್ತಿಶಾಲಿ ವೈಶಿಷ್ಟ್ಯಗಳಲ್ಲಿ ಒಂದು ನೋಂದಣಿ ಫಾರ್ಮ್‌ಗೆ ಅನಿಯಮಿತ ಟೆಕ್ಸ್ಟ್, ಬಣ್ಣ ಮತ್ತು ಆಯ್ಕೆ ಕ್ಷೇತ್ರಗಳನ್ನು ಸೇರಿಸುವ ಸಾಮರ್ಥ್ಯ. ಆ ಡೇಟಾ ಸೆರೆಹಿಡಿದ ನಂತರ, ಆಯ್ಕೆ ಮಾಡಿದ site template ನ ನಿರ್ದಿಷ್ಟ ಭಾಗಗಳಲ್ಲಿ ವಿಷಯವನ್ನು ಪೂರ್ವ-ಭರ್ತಿ ಮಾಡಲು ಬಳಸಬಹುದು. ನಂತರ, ಹೊಸ ಸೈಟ್ ಪ್ರಕಟವಾಗುವಾಗ, Ultimate Multisite ನೋಂದಣಿ ಸಮಯದಲ್ಲಿ ನಮೂದಿಸಿದ ನಿಜವಾದ ಮಾಹಿತಿಯೊಂದಿಗೆ placeholders ಅನ್ನು ಬದಲಾಯಿಸುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ನೋಂದಣಿ ಸಮಯದಲ್ಲಿ ನಿಮ್ಮ ಅಂತಿಮ-ಬಳಕೆದಾರರ ಕಂಪನಿ ಹೆಸರು ಪಡೆಯಲು ಮತ್ತು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೋಮ್ ಪೇಜ್‌ನಲ್ಲಿ ಕಂಪನಿ ಹೆಸರು ಹಾಕಲು ಬಯಸಿದರೆ. ನಿಮ್ಮ template ಸೈಟ್ ಹೋಮ್ ಪೇಜ್‌ನಲ್ಲಿ ಕೆಳಗಿನ ಚಿತ್ರದಲ್ಲಿರುವಂತೆ placeholders ಸೇರಿಸಬೇಕು (placeholders ಅನ್ನು ಡಬಲ್ ಕರ್ಲಿ ಬ್ರೇಸ್‌ಗಳಿಂದ ಸುತ್ತುವರಿಯಬೇಕು - {{placeholder_name}}).

### "placeholders ತುಂಬಿದ template" ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸುವುದು

ಇದೆಲ್ಲಾ ಉತ್ತಮ, ಆದರೆ ನಾವು ಒಂದು ಕೆಟ್ಟ ಸಮಸ್ಯೆಗೆ ಬರುತ್ತೇವೆ: ಈಗ ನಮ್ಮ site templates - ನಮ್ಮ ಗ್ರಾಹಕರು ಭೇಟಿ ನೀಡಬಹುದಾದವು - ಹೆಚ್ಚು ಹೇಳದ ಕೆಟ್ಟ placeholders ತುಂಬಿವೆ.

ಇದನ್ನು ಪರಿಹರಿಸಲು, placeholders ಗೆ ನಕಲಿ ಮೌಲ್ಯಗಳನ್ನು ಹೊಂದಿಸುವ ಆಯ್ಕೆ ನೀಡುತ್ತೇವೆ, ಮತ್ತು ನಿಮ್ಮ ಗ್ರಾಹಕರು ಭೇಟಿ ನೀಡುವಾಗ template ಸೈಟ್‌ಗಳಲ್ಲಿ ಅವುಗಳ ವಿಷಯಗಳನ್ನು search ಮತ್ತು replace ಮಾಡಲು ಆ ಮೌಲ್ಯಗಳನ್ನು ಬಳಸುತ್ತೇವೆ.

**Ultimate Multisite > Settings > Sites** ಗೆ ಹೋಗಿ, ನಂತರ sidebar ನಲ್ಲಿ **Edit Placeholders** ಲಿಂಕ್ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ template placeholders editor ಗೆ ಪ್ರವೇಶ ಪಡೆಯಬಹುದು.
