# ಗ್ರಾಹಕರ ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಸಮಸ್ಯೆ ವರದಿ (Customer Feedback & Issue Reporting) {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 ಒಂದು ಅಂತರ್ನಿರ್ಮಿತ (built-in) ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಸಮಸ್ಯೆ ವರದಿ ವ್ಯವಸ್ಥೆಯನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ. ಇದು ಅಂತಿಮ ಬಳಕೆದಾರರು (end users) ಸಹಾಯ ಮಾಡದ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಗುರುತಿಸಲು ಮತ್ತು ಚಾಟ್ ಇಂಟರ್ಫೇಸ್‌ನಿಂದ ನೇರವಾಗಿ ಸಮಸ್ಯೆಗಳನ್ನು ವರದಿ ಮಾಡಲು ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ. ಈ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ಸಮ್ಮತಿ ನಿರ್ವಹಣೆ (consent management), ಸ್ವಯಂಚಾಲಿತ ವರದಿ ಮಾಡುವ ಕಮಾಂಡ್ ಮತ್ತು ಬ್ಯಾಕ-ಎಂಡ್‌‌ನಲ್ಲಿ AI-ಸಹಾಯದ ಟ್ರೈಯಾಜ್ (triage) ಸೇರಿವೆ.

## ಥಂಬ್ಸ್-ಡೌನ್ ಬಟನ್ (Thumbs-Down Button) {#thumbs-down-button}

AI ಸಹಾಯಕದಿಂದ ಕಳುಹಿಸಲಾದ ಪ್ರತಿಯೊಂದು ಸಂದೇಶದೊಂದಿಗೆ ಒಂದು **ಥಂಬ್ಸ್-ಡೌನ್** (👎) ಬಟನ್ ಕಾಣಿಸುತ್ತದೆ. ಬಳಕೆದಾರರು ಇದನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ, ಅವರು ಒಂದು ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಸಹಾಯ ಮಾಡದ ಅಥವಾ ತಪ್ಪಾದವೆಂದು ಗುರುತಿಸಬಹುದು.

- ಈ ಬಟನ್ ಪ್ರತಿ ಸಹಾಯಕ ಸಂದೇಶದ ಪಕ್ಕದಲ್ಲಿ ಮೌಸ್ ಹಾವರ್ (hover) ಮಾಡಿದಾಗ ಕಾಣಿಸುತ್ತದೆ.
- ಇದನ್ನು ಕ್ಲಿಕ್ ಮಾಡುವುದರಿಂದ **Feedback Consent Modal** ತೆರೆದುಕೊಳ್ಳುತ್ತದೆ.
- ಈ ಪ್ರತಿಕ್ರಿಯೆಯು ಸಂಭಾಷಣೆಯ ಥ್ರೆಡ್ ಮತ್ತು ನಿರ್ದಿಷ್ಟ ಸಂದೇಶದೊಂದಿಗೆ ಜೋಡಿಸಲಾಗುತ್ತದೆ.

## Feedback Consent Modal {#feedback-consent-modal}

ಬಳಕೆದಾರರು ಥಂಬ್ಸ್-ಡೌನ್ ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದಾಗ, ಯಾವುದೇ ಡೇಟಾವನ್ನು ಕಳುಹಿಸುವ ಮೊದಲು ಒಂದು ಸಮ್ಮತಿ ಮೋಡಲ್ (consent modal) ಕಾಣಿಸುತ್ತದೆ. ಈ ಮೋಡಲ್:

- ಯಾವ ಮಾಹಿತಿಯನ್ನು ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ವಿವರಿಸುತ್ತದೆ (ಸಂಭಾಷಣೆಯ ತುಣುಕು, ಬ್ರೌಸರ್ ಸಂದರ್ಭ).
- ಈ ಡೇಟಾವನ್ನು ಹಂಚಿಕೊಳ್ಳಲು ಬಳಕೆದಾರರು ಸಮ್ಮತಿಸುತ್ತಾರೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು ಕೇಳುತ್ತದೆ.
- ಬಳಕೆದಾರರು ಏನೋ ತಪ್ಪಾಗಿದೆ ಎಂದು ವಿವರಿಸಲು ಒಂದು ಫ್ರೀ-ಟೆಕ್ಸ್ಟ್ ಫೀಲ್ಡ್ ಅನ್ನು ಒದಗಿಸುತ್ತದೆ.
- ಸಲ್ಲಿಸದೆ ಮುಚ್ಚಲು **Cancel** ಆಯ್ಕೆಯನ್ನು ನೀಡುತ್ತದೆ.

ಬಳಕೆದಾರರು ಸ್ಪಷ್ಟವಾಗಿ ಖಚಿತಪಡಿಸುವವರೆಗೆ ಯಾವುದೇ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ದಾಖಲಿಸಲಾಗುವುದಿಲ್ಲ.

## Auto-Prompt Feedback Banner {#auto-prompt-feedback-banner}

ಸಂಭಾಷಣೆಯ ಕೊನೆಯಲ್ಲಿ, ಸಹಾಯಕನು ಒಂದು **auto-prompt feedback banner** ಅನ್ನು ಪ್ರದರ್ಶಿಸಬಹುದು — ಇದು ಸೆಷನ್ ಉಪಯುಕ್ತವಾಗಿದೆಯೇ ಎಂದು ಕೇಳುವ ಅಡ್ಡಿಪಡಿಸದ ಸಂದೇಶವಾಗಿದೆ.

ಈ ಬ್ಯಾನರ್ ಸಂಭಾಷಣೆಯ ಉದ್ದ ಮತ್ತು ಫಲಿತಾಂಶದ ಆಧಾರದ ಮೇಲೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಕಾಣಿಸುತ್ತದೆ. ಇದು ಥಂಬ್ಸ್-ಡೌನ್ ಬಟನ್‌ನಂತೆಯೇ ಅದೇ ಪ್ರತಿಕ್ರಿಯೆ ಹರಿವಕ್ಕೆ (feedback flow) ಲಿಂಕ್ ಮಾಡುತ್ತದೆ. ಬಳಕೆದಾರರು ಪ್ರತಿಕ್ರಿಯೆ ನೀಡದೆ ಈ ಬ್ಯಾನರ್ ಅನ್ನು ಮುಚ್ಚಬಹುದು.

## /report-issue ಸ್ಲಾಶ್ ಕಮಾಂಡ್ (/report-issue Slash Command) {#report-issue-slash-command}

ಬಳಕೆದಾರರು ಚಾಟ್ ಇನ್‌ಪುಟ್‌ನಲ್ಲಿ `/report-issue` ಎಂದು ಟೈಪ್ ಮಾಡುವ ಮೂಲಕ ನೇರವಾಗಿ ಪ್ರತಿಕ್ರಿಯೆ ಹರಿವನ್ನು déclencher (trigger) ಮಾಡಬಹುದು. ಈ ಕಮಾಂಡ್:

- ತಕ್ಷಣವೇ Feedback Consent Modal ಅನ್ನು ತೆರೆಯುತ್ತದೆ.
- ಪ್ರಸ್ತುತ ಸಂಭಾಷಣೆಯ ಕುರಿತು ಸಂದರ್ಭದೊಂದಿಗೆ (context) ವಿವರಣೆ ಫೀಲ್ಡ್ ಅನ್ನು ಪೂರ್ವ-ಭರ್ತಿ (pre-populates) ಮಾಡುತ್ತದೆ.
- ಸಲ್ಲಿಸುವ ಮೊದಲು ಬಳಕೆದಾರರು ಹೆಚ್ಚುವರಿ ವಿವರಗಳನ್ನು ಸೇರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.

`/report-issue` ಕಮಾಂಡ್ ಎಲ್ಲಾ ಚಾಟ್ ಮೋಡ್‌ಗಳಲ್ಲಿ (inline, floating widget, full-screen) ಲಭ್ಯವಿದೆ.

## AI-Assisted Triage {#ai-assisted-triage}

ಸಲ್ಲಿಸಲಾದ ಪ್ರತಿಕ್ರಿಯೆಯು ಒಂದು AI ಟ್ರೈಯಾಜ್ ಲೇಯರ್‌ಗೆ (triage layer) ರವಾನೆಯಾಗುತ್ತದೆ, ಅದು:

- ವರದಿಯನ್ನು ವರ್ಗೀಕರಿಸುತ್ತದೆ (ವಾಸ್ತವದ ದೋಷ, ಸಹಾಯ ಮಾಡದ ಉತ್ತರ, ಅಸ適切 ವಿಷಯ, ಇತ್ಯಾದಿ).
- ಸಂಬಂಧಿತ ಸಂಭಾಷಣೆ ಮೆಟಾಡೇಟಾವನ್ನು (metadata) ಲಗತ್ತಿಸುತ್ತದೆ.
- ಟ್ರೈಯಾಜ್ ಸಾರಾಂಶವನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ **Feedback Endpoint** ಗೆ ಫಾರ್ವರ್ಡ್ ಮಾಡುತ್ತದೆ (voir [Settings > Advanced](#settings-advanced)).

ಇದು ಅತ್ಯಂತ ನಿರ್ಣಾಯಕ ಸಮಸ್ಯೆಗಳನ್ನು ಮೊದಲು ಮೇಲಕ್ಕೆ ತರುವುದರಿಂದ (surfacing) ಕೈಯಾರೆ ಪರಿಶೀಲಿಸುವ ಸಮಯವನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.

## Settings > Advanced {#settings-advanced}

ಪ್ರತಿಕ್ರಿಯೆ ಬ್ಯಾಕ-ಎಂಡ್‌ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು, **Gratis AI Agent → Settings → Advanced** ಅಡಿಯಲ್ಲಿ ಈ ಕೆಳಗಿನ ಫೀಲ್ಡ್‌ಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | ಪ್ರತಿಕ್ರಿಯೆ ಪೇಲೋಡ್‌ಗಳೊಂದಿಗೆ (feedback payloads) POST ವಿನಂತಿಗಳನ್ನು ಸ್ವೀಕರಿಸುವ URL (JSON). |
| **Feedback API Key** | ಪ್ರತಿಕ್ರಿಯೆ ಸಲ್ಲಿಸುವಿಕೆಗಳನ್ನು (submissions) ಅಧಿಕೃತಗೊಳಿಸಲು `Authorization` ಹೆಡರ್‌ನಲ್ಲಿ ಕಳುಹಿಸುವ Bearer token. |

ಪ್ರತಿಕ್ರಿಯೆ ಸಂಗ್ರಹಣೆಯನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲು ಎರಡೂ ಫೀಲ್ಡ್‌ಗಳನ್ನು ಖಾಲಿ ಬಿಡಿ. ವೈಯಕ್ತಿಕ ಪ್ರತಿಕ್ರಿಯೆ ಬಟನ್‌ಗಳು ಬಳಕೆದಾರರಿಗೆ ಗೋಚರಿಸುತ್ತಲೇ ಇರುತ್ತವೆ, ಆದರೆ ಸಲ್ಲಿಸುವಿಕೆಗಳನ್ನು ಫಾರ್ವರ್ಡ್ ಮಾಡಲಾಗುವುದಿಲ್ಲ.

:::tip
ಪ್ರತಿಕ್ರಿಯೆ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಕನಿಷ್ಠ `message_id`, `conversation_id`, `feedback_text`, ಮತ್ತು `triage_category` ಫೀಲ್ಡ್‌ಗಳೊಂದಿಗೆ JSON ಬಾಡಿ (JSON body) ಅನ್ನು ಸ್ವೀಕರಿಸಬೇಕು. ನಿರೀಕ್ಷಿತ ಸ್ಕೀಮಾ (expected schema) ಗಾಗಿ ನಿಮ್ಮ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಪровайಡರ್‌ನ ದಾಖಲೆಗಳನ್ನು ನೋಡಿ.
:::
