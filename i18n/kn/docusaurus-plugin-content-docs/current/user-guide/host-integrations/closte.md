# Closte ಏಕೀಕರಣ

## ಅವಲೋಕನ
Closte ಎಂಬುದು Google Cloud ಮೂಲಸೌಕರ್ಯದ ಮೇಲೆ ನಿರ್ಮಿಸಲಾದ ನಿರ್ವಹಿತ WordPress ಹೋಸ್ಟಿಂಗ್ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಆಗಿದೆ. ಈ ಏಕೀಕರಣವು Ultimate Multisite ಮತ್ತು Closte ನಡುವೆ ಸ್ವಯಂಚಾಲಿತ ಡೊಮೇನ್ ಸಿಂಕ್ ಮತ್ತು SSL ಪ್ರಮಾಣಪತ್ರ ನಿರ್ವಹಣೆಯನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ.

## ವೈಶಿಷ್ಟ್ಯಗಳು
- ಸ್ವಯಂಚಾಲಿತ ಡೊಮೇನ್ ಸಿಂಕಿಂಗ್
- SSL ಪ್ರಮಾಣಪತ್ರ ನಿರ್ವಹಣೆ
- ವೈಲ್ಡ್‌ಕಾರ್ಡ್ ಡೊಮೇನ್ ಬೆಂಬಲ
- Closte ನಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿದ್ದರೆ ಯಾವುದೇ ಕಾನ್ಫಿಗರೇಶನ್ ಅಗತ್ಯವಿಲ್ಲ

## ಅವಶ್ಯಕತೆಗಳು
ನೀವು Closte ಬಳಸುತ್ತಿದ್ದರೆ, ನಿಮ್ಮ `wp-config.php` ಫೈಲ್‌ನಲ್ಲಿ ಈ ಕೆಳಗಿನ constant ಅನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಬೇಕು:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

ನೀವು Closte ನಲ್ಲಿ ಹೋಸ್ಟ್ ಮಾಡುತ್ತಿದ್ದರೆ ಈ constant ಸಾಮಾನ್ಯವಾಗಿ ಈಗಾಗಲೇ ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿರುತ್ತದೆ.

## ಸೆಟಪ್ ಸೂಚನೆಗಳು

### 1. ನಿಮ್ಮ Closte API ಕೀ ಅನ್ನು ಪರಿಶೀಲಿಸಿ

ನೀವು Closte ನಲ್ಲಿ ಹೋಸ್ಟ್ ಮಾಡುತ್ತಿದ್ದರೆ, `CLOSTE_CLIENT_API_KEY` constant ನಿಮ್ಮ `wp-config.php` ಫೈಲ್‌ನಲ್ಲಿ ಈಗಾಗಲೇ ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿರಬೇಕು. ನಿಮ್ಮ `wp-config.php` ಫೈಲ್ ಅನ್ನು ಪರಿಶೀಲಿಸುವ ಮೂಲಕ ಇದನ್ನು ದೃಢೀಕರಿಸಬಹುದು.

### 2. ಏಕೀಕರಣವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ

1. ನಿಮ್ಮ WordPress admin ನಲ್ಲಿ, Ultimate Multisite > Settings ಗೆ ಹೋಗಿ
2. "Domain Mapping" ಟ್ಯಾಬ್‌ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ
3. "Host Integrations" ಗೆ ಕೆಳಗೆ ಸ್ಕ್ರಾಲ್ ಮಾಡಿ
4. Closte ಏಕೀಕರಣವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
5. "Save Changes" ಕ್ಲಿಕ್ ಮಾಡಿ

## ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ

Ultimate Multisite ನಲ್ಲಿ ಡೊಮೇನ್ ಮ್ಯಾಪ್ ಮಾಡಿದಾಗ:

1. ಏಕೀಕರಣವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಡೊಮೇನ್ ಸೇರಿಸಲು Closte ನ API ಗೆ ವಿನಂತಿಯನ್ನು ಕಳುಹಿಸುತ್ತದೆ
2. Closte ಸ್ವಯಂಚಾಲಿತವಾಗಿ SSL ಪ್ರಮಾಣಪತ್ರ ಒದಗಿಸುವಿಕೆಯನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ
3. ಡೊಮೇನ್ ಮ್ಯಾಪಿಂಗ್ ತೆಗೆದುಹಾಕಿದಾಗ, ಏಕೀಕರಣವು Closte ನಿಂದ ಡೊಮೇನ್ ಅನ್ನು ತೆಗೆದುಹಾಕುತ್ತದೆ

ಈ ಏಕೀಕರಣವು Ultimate Multisite ನಲ್ಲಿನ DNS ಪರಿಶೀಲನಾ ಮಧ್ಯಂತರ ಸೆಟ್ಟಿಂಗ್‌ನೊಂದಿಗೆ ಸಹ ಕೆಲಸ ಮಾಡುತ್ತದೆ, DNS ಪ್ರಸರಣ ಮತ್ತು SSL ಪ್ರಮಾಣಪತ್ರ ವಿತರಣೆಗಾಗಿ ಸಿಸ್ಟಮ್ ಎಷ್ಟು ಬಾರಿ ಪರಿಶೀಲಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಲು ನಿಮಗೆ ಅನುಮತಿಸುತ್ತದೆ.

## ಡೊಮೇನ್ ರೆಕಾರ್ಡ್ ರಚನೆ

ಸೈಟ್ ರಚಿಸಿದಾಗ ಅಥವಾ ನಕಲು ಮಾಡಿದಾಗ ಡೊಮೇನ್ ರೆಕಾರ್ಡ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಲ್ಪಡುತ್ತದೆ ಎಂದು ಈ ಏಕೀಕರಣವು ಖಚಿತಪಡಿಸುತ್ತದೆ. Closte ಏಕೀಕರಣಕ್ಕೆ ಇದು ವಿಶೇಷವಾಗಿ ಮುಖ್ಯವಾಗಿದೆ, ಏಕೆಂದರೆ ಡೊಮೇನ್ ರೆಕಾರ್ಡ್ ರಚನೆಯು ಡೊಮೇನ್ ಮತ್ತು SSL ಪ್ರಮಾಣಪತ್ರವನ್ನು ರಚಿಸಲು Closte API ಅನ್ನು ಪ್ರಚೋದಿಸುತ್ತದೆ.

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### API ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳು
- ನಿಮ್ಮ Closte API ಕೀ ಸರಿಯಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ನಿಮ್ಮ Closte ಖಾತೆಯು ಅಗತ್ಯ ಅನುಮತಿಗಳನ್ನು ಹೊಂದಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

### SSL ಪ್ರಮಾಣಪತ್ರ ಸಮಸ್ಯೆಗಳು
- SSL ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ವಿತರಿಸಲು Closte ಗೆ ಸ್ವಲ್ಪ ಸಮಯ ಬೇಕಾಗಬಹುದು (ಸಾಮಾನ್ಯವಾಗಿ 5-10 ನಿಮಿಷಗಳು)
- ನಿಮ್ಮ ಡೊಮೇನ್‌ಗಳು ನಿಮ್ಮ Closte ಸರ್ವರ್‌ನ IP ವಿಳಾಸಕ್ಕೆ ಸರಿಯಾಗಿ ಪಾಯಿಂಟ್ ಆಗಿವೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ನಿಮ್ಮ ಡೊಮೇನ್‌ನ DNS ರೆಕಾರ್ಡ್‌ಗಳು ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಆಗಿವೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ

### ಡೊಮೇನ್ ಸೇರಿಸಲಾಗಿಲ್ಲ
- ಯಾವುದೇ ದೋಷ ಸಂದೇಶಗಳಿಗಾಗಿ Ultimate Multisite ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
- ಡೊಮೇನ್ ಈಗಾಗಲೇ Closte ಗೆ ಸೇರಿಸಲಾಗಿಲ್ಲ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ನಿಮ್ಮ ಡೊಮೇನ್‌ನ DNS ರೆಕಾರ್ಡ್‌ಗಳು ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಆಗಿವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

### DNS ಪರಿಶೀಲನಾ ಮಧ್ಯಂತರ
- SSL ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ವಿತರಿಸಲು ತುಂಬಾ ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತಿದ್ದರೆ, Domain Mapping ಸೆಟ್ಟಿಂಗ್‌ಗಳಲ್ಲಿ DNS ಪರಿಶೀಲನಾ ಮಧ್ಯಂತರವನ್ನು ಹೊಂದಿಸಬಹುದು
- ಡೀಫಾಲ್ಟ್ ಮಧ್ಯಂತರ 300 ಸೆಕೆಂಡುಗಳು (5 ನಿಮಿಷಗಳು), ಆದರೆ ಪರೀಕ್ಷೆಯ ಸಮಯದಲ್ಲಿ ವೇಗವಾಗಿ ಪರಿಶೀಲಿಸಲು ಇದನ್ನು 10 ಸೆಕೆಂಡುಗಳಿಗೆ ಕಡಿಮೆ ಮಾಡಬಹುದು
