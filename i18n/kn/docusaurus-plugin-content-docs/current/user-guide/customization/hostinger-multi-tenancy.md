#โฮสติ้ง มัลติเทนันซี

Ultimate Multisite: Multi-Tenancy 1.2.0 ได้เพิ่มความสามารถแบบ Hostinger multi-tenancy เข้ามา ทำให้โดเมนที่โฮสต์อยู่สามารถเข้าร่วมในการจัดเตรียมผู้เช่าที่เป็นอิสระ (sovereign tenant provisioning) ควบคู่ไปกับการผสานรวมการแมปโดเมนของ Hostinger ที่มีอยู่

ให้ใช้ความสามารถนี้เมื่อคุณจัดการโดเมนของผู้เช่าและโครงสร้างพื้นฐานของผู้เช่าที่แยกออกจากกันผ่าน Hostinger hPanel

## หมายเหตุในการตั้งค่า

1. กำหนดค่าการผสานรวมหลักของ Hostinger ภายใต้ **Ultimate Multisite > Settings > Host Integrations**
2. ยืนยันว่าโทเค็น API ของ Hostinger สามารถจัดการโดเมนหรือโดเมนย่อยเป้าหมายได้
3. เปิดใช้งาน Addon Multi-Tenancy
4. กำหนดกลยุทธ์การแยกผู้เช่าก่อนที่จะเผยแพร่ผู้เช่า
5. รันเวิร์กโฟลว์การตรวจสอบการย้ายข้อมูล (migration verification workflow) ก่อนที่จะส่งทราฟฟิกไปยังผู้เช่าในระบบจริง

ความสามารถของ Hostinger ใช้การเชื่อมต่อ Hostinger ที่ใช้ร่วมกันสำหรับการดำเนินการฝั่งโฮสต์ โดเมนยังคงต้องชี้ไปยังบัญชี Hostinger ที่ถูกต้อง และข้อจำกัดของบัญชี hPanel ยังคงมีผลบังคับใช้

## การเปลี่ยนแปลงเฉพาะความสามารถ

- ผู้เช่าที่เป็นอิสระ (Sovereign tenants) สามารถสร้างขึ้นได้ด้วยการดำเนินการโดเมนที่รับรู้ถึงโฮสต์
- สตริงโฮสต์ฐานข้อมูลบนเครื่องเดียวกันจะถูกปรับให้เป็นมาตรฐานก่อนการตรวจสอบสิทธิ์
- ผู้เช่าที่จัดการโดย Hostinger ควรใช้ค่าโฮสต์ฐานข้อมูลที่แสดงใน hPanel เว้นแต่รันไทม์ของ WordPress จะต้องการการแทนที่แบบท้องถิ่น (local override)
- การเข้าชม SSO อาศัยโดเมนของผู้เช่าในการแก้ไขให้ชี้ไปยังผู้เช่าที่โฮสต์โดย Hostinger

## การแก้ไขปัญหาสำหรับผู้เช่า Hostinger

- หากการติดตั้งผู้เช่าล้มเหลว ให้ตรวจสอบว่าโดเมนได้ถูกผูกกับบัญชี Hostinger แล้วหรือไม่
- หากการตรวจสอบฐานข้อมูลล้มเหลว ให้เปรียบเทียบชื่อผู้ใช้ฐานข้อมูล (DB username), ชื่อฐานข้อมูล (database name) และการผูกโฮสต์ (host binding) ของผู้เช่ากับ hPanel
- หาก **Visit (SSO)** ล้มเหลว ให้ยืนยันว่า DNS และ SSL เปิดใช้งานสำหรับโดเมนของผู้เช่าแล้ว
- หากการยกเลิกการตั้งค่าทิ้งทรัพยากรโฮสต์ไว้ ให้ลบฐานข้อมูล ผู้ใช้ หรือโฟลเดอร์ที่เหลือทั้งหมดออกจาก hPanel หลังจากยืนยันการสำรองข้อมูล
