# ನಿಮ್ಮ ಗ್ರಾಹಕರ ಖಾತೆ ಪುಟ (v2)

_**ಮುಖ್ಯ ಟಿಪ್ಪಣಿ: ಈ ಲೇಖನವು Ultimate Multisite ಆವೃತ್ತಿ 2.x ಅನ್ನು ಉಲ್ಲೇಖಿಸುತ್ತದೆ.**_

ಗ್ರಾಹಕರು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಒಂದು ಪ್ಲಾನ್‌ಗೆ ಚಂದಾದಾರರಾದಾಗ, ಅವರಿಗೆ ವೆಬ್‌ಸೈಟ್ ಮತ್ತು ಅದರ dashboard ಗೆ ಪ್ರವೇಶ ಸಿಗುತ್ತದೆ. ಇದರಲ್ಲಿ ಅವರ ಪಾವತಿಗಳು, ಸದಸ್ಯತ್ವಗಳು, ಡೊಮೇನ್‌ಗಳು, ಪ್ಲಾನ್ ಮಿತಿಗಳು ಮುಂತಾದ ಮಹತ್ವದ ಮಾಹಿತಿಗಳಿರುತ್ತವೆ...

ಈ ಟ್ಯುಟೋರಿಯಲ್‌ನಲ್ಲಿ, ನಾವು ಗ್ರಾಹಕರ ಖಾತೆ ಪುಟದ ಬಗ್ಗೆ ಮಾರ್ಗದರ್ಶನ ನೀಡುತ್ತೇವೆ ಮತ್ತು ನಿಮ್ಮ ಗ್ರಾಹಕರು ಅದರೊಳಗೆ ಏನನ್ನು ನೋಡಬಹುದು ಹಾಗೂ ಮಾಡಬಹುದು ಎಂಬುದನ್ನು ತಿಳಿಸುತ್ತೇವೆ.

## ಖಾತೆ ಪುಟ

ನಿಮ್ಮ ಗ್ರಾಹಕರ dashboard ನಲ್ಲಿ **Account** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಖಾತೆ ಪುಟವನ್ನು ಪ್ರವೇಶಿಸಬಹುದು.

![ಗ್ರಾಹಕ dashboard ನಲ್ಲಿ Account ಮೆನು](/img/admin/dashboard.png)

![ಗ್ರಾಹಕ ಖಾತೆ ಪುಟದ ಅವಲೋಕನ](/img/admin/dashboard.png)

ಗ್ರಾಹಕರು ಅದರ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ಅವರಿಗೆ ಸದಸ್ಯತ್ವ, ಬಿಲ್ಲಿಂಗ್ ವಿಳಾಸ, ಇನ್‌ವಾಯ್ಸ್‌ಗಳು, ಡೊಮೇನ್‌ಗಳು, ಸೈಟ್ ಮಿತಿಗಳ ಅವಲೋಕನ ಕಾಣುತ್ತದೆ ಮತ್ತು **Site Template** ಅನ್ನು ಬದಲಾಯಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ (ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಅನುಮತಿಸಿದ್ದರೆ).

ಅವರು ಸದಸ್ಯತ್ವವನ್ನು ಬೇರೊಂದು ಪ್ಲಾನ್‌ಗೆ ಬದಲಾಯಿಸಬಹುದು ಅಥವಾ ನೀವು ನೀಡುವ ಬೇರೊಂದು ಪ್ಯಾಕೇಜ್ ಅಥವಾ ಸೇವೆಯನ್ನು ಖರೀದಿಸಬಹುದು. ಪ್ರತಿಯೊಂದು ವಿಭಾಗವನ್ನು ಪ್ರತ್ಯೇಕವಾಗಿ ನೋಡೋಣ.

### ನಿಮ್ಮ ಸದಸ್ಯತ್ವ ಅವಲೋಕನ:

ನಿಮ್ಮ ಗ್ರಾಹಕರ ವೆಬ್‌ಸೈಟ್ ಹೆಸರಿನ ಕೆಳಗಿರುವ ಮೊದಲ ಬ್ಲಾಕ್ ಅವರ ಪ್ರಸ್ತುತ ಪ್ಲಾನ್ ಮತ್ತು ಅದರೊಂದಿಗೆ ಖರೀದಿಸಿದ ಸೇವೆಗಳು/ಪ್ಯಾಕೇಜ್‌ಗಳ ಅವಲೋಕನವನ್ನು ತೋರಿಸುತ್ತದೆ. ಈ ಬ್ಲಾಕ್ ಸದಸ್ಯತ್ವ ಸಂಖ್ಯೆ, ಆರಂಭದಲ್ಲಿ ಪಾವತಿಸಿದ ಮೊತ್ತ, ಪ್ಲಾನ್ ಮತ್ತು ಯಾವುದೇ ಸೇವೆ/ಪ್ಯಾಕೇಜ್‌ನ ಬೆಲೆ ಮತ್ತು ಈ ಸದಸ್ಯತ್ವಕ್ಕೆ ಎಷ್ಟು ಬಾರಿ ಬಿಲ್ ಮಾಡಲಾಗಿದೆ ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತದೆ. ಸದಸ್ಯತ್ವವು **Active** , **Expired** ಅಥವಾ **Canceled** ಆಗಿದೆಯೇ ಎಂಬುದನ್ನೂ ಅವರು ನೋಡಬಹುದು.

![ಪ್ಲಾನ್, ಮೊತ್ತ ಮತ್ತು ಬಿಲ್ಲಿಂಗ್ ವಿವರಗಳನ್ನು ತೋರಿಸುವ ಸದಸ್ಯತ್ವ ಅವಲೋಕನ](/img/admin/memberships-list.png)

ಈ ಬ್ಲಾಕ್‌ನ ಕೆಳಗೆ, ನಿಮ್ಮ ಗ್ರಾಹಕರು **About This Site** ಮತ್ತು **Site Limits** ಬ್ಲಾಕ್‌ಗಳನ್ನು ನೋಡಬಹುದು. ಈ ಬ್ಲಾಕ್‌ಗಳು ಅವರ ಪ್ಲಾನ್‌ಗೆ ಇರುವ ಎಲ್ಲಾ ಮಿತಿಗಳನ್ನು ತೋರಿಸುತ್ತವೆ: ಡಿಸ್ಕ್ ಸ್ಪೇಸ್, ಪೋಸ್ಟ್‌ಗಳು, ಪೇಜ್‌ಗಳು, ಭೇಟಿಗಳು, ಇತ್ಯಾದಿ... ಈ ಮಿತಿಗಳನ್ನು **Ultimate Multisite > Products** ನಲ್ಲಿ ಪ್ರತಿ ಪ್ಲಾನ್ ಪುಟದಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಬಹುದು.

![ಪ್ಲಾನ್ ಮಿತಿಗಳನ್ನು ತೋರಿಸುವ About This Site ಮತ್ತು Site Limits ಬ್ಲಾಕ್‌ಗಳು](/img/admin/dashboard.png)

**Your Membership** ಬಲಭಾಗದಲ್ಲಿ, ಗ್ರಾಹಕರು **Change** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಬಹುದು. ಇದು ಅವರಿಗೆ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ ಪ್ಲಾನ್‌ಗಳು ಮತ್ತು ಪ್ಯಾಕೇಜ್‌ಗಳು/ಸೇವೆಗಳನ್ನು ತೋರಿಸುತ್ತದೆ. ಅವರು ಬೇರೊಂದು ಪ್ಲಾನ್ ಆರಿಸಿದರೆ, ಆ ಪ್ಲಾನ್‌ನ ಮಿತಿಗಳು ಪ್ರಸ್ತುತ ಸದಸ್ಯತ್ವದ ಮಿತಿಗಳ ಬದಲಿಗೆ ಅನ್ವಯವಾಗುತ್ತವೆ - ಅವರು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುತ್ತಿದ್ದರೂ ಅಥವಾ ಅಪ್‌ಗ್ರೇಡ್ ಮಾಡುತ್ತಿದ್ದರೂ ಪರವಾಗಿಲ್ಲ.

ಈಗ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ಈ ಪ್ರಸ್ತುತ ಸದಸ್ಯತ್ವಕ್ಕೆ ಪ್ಯಾಕೇಜ್‌ಗಳು ಅಥವಾ ಸೇವೆಗಳನ್ನು ಖರೀದಿಸಲು ಆರಿಸಿದರೆ - ಹೆಚ್ಚುವರಿ ಡಿಸ್ಕ್ ಸ್ಪೇಸ್ ಅಥವಾ ಭೇಟಿಗಳಂತಹ - ಪ್ರಸ್ತುತ ಸದಸ್ಯತ್ವ ಬದಲಾಗುವುದಿಲ್ಲ, ಬದಲಿಗೆ ಹೊಸ ಪ್ಯಾಕೇಜ್‌ಗಳು ಮಾತ್ರ ಸೇರಿಸಲ್ಪಡುತ್ತವೆ.

ಈ ಸದಸ್ಯತ್ವ ಬದಲಾವಣೆ ಪುಟದಲ್ಲಿ ಕೂಪನ್ ಕೋಡ್‌ಗಳನ್ನು ಸೇರಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ ಎಂಬುದನ್ನು ಗಮನಿಸಿ. ಗ್ರಾಹಕರು ಮೊದಲ ಸದಸ್ಯತ್ವ ಖರೀದಿಯಲ್ಲಿ ಕೂಪನ್ ಕೋಡ್ ಬಳಸಿದ್ದರೆ, ಆ ಕೋಡ್ ಈ ಹೊಸ ಸದಸ್ಯತ್ವಕ್ಕೂ ಅನ್ವಯವಾಗುತ್ತದೆ.

### ಬಿಲ್ಲಿಂಗ್ ವಿಳಾಸ ನವೀಕರಿಸುವುದು:

ಖಾತೆ ಪುಟದಲ್ಲಿ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಬಿಲ್ಲಿಂಗ್ ವಿಳಾಸವನ್ನು ನವೀಕರಿಸಬಹುದು. ಅವರು _Billing Address_ ಪಕ್ಕದಲ್ಲಿರುವ **Update** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಬೇಕು.

![Update ಬಟನ್ ಇರುವ Billing Address ವಿಭಾಗ](/img/admin/customers-list.png)

ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಹೊಸ ವಿಂಡೋ ಕಾಣಿಸುತ್ತದೆ. ಅವರು ಹೊಸ ವಿಳಾಸವನ್ನು ಭರ್ತಿ ಮಾಡಿ _Save Changes_ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಬೇಕು.

![ಬಿಲ್ಲಿಂಗ್ ವಿಳಾಸ ನವೀಕರಿಸುವ ಫಾರ್ಮ್](/img/admin/customers-list.png)

### Site Template ಬದಲಾಯಿಸುವುದು:

ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಅವರ site template ಬದಲಾಯಿಸಲು ಅನುಮತಿಸಲು, ನೀವು **Ultimate Multisite > Settings > Sites** ಗೆ ಹೋಗಿ **Allow Template Switching** ಆಯ್ಕೆಯನ್ನು ಆನ್ ಮಾಡಬೇಕು.

ಹಾಗೆಯೇ, **Ultimate Multisite > Products** ನಲ್ಲಿ, ನಿಮ್ಮ ಪ್ಲಾನ್‌ಗಳನ್ನು ಆರಿಸಿ **Site Templates** ಟ್ಯಾಬ್‌ಗೆ ಹೋಗಿ. **Allow Site Templates** ಆಯ್ಕೆ ಆನ್ ಆಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ ಮತ್ತು **Site Template Selection Mode** ನಲ್ಲಿ **Choose Available Site Templates** ಆಯ್ಕೆಯನ್ನು ಆರಿಸಿ.

![Template selection mode ಇರುವ Product site templates ಟ್ಯಾಬ್](/img/config/product-site-templates.png)

ನಿಮ್ಮ ವೆಬ್‌ಸೈಟ್‌ನಲ್ಲಿ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ site template ಗಳನ್ನು ನೀವು ನೋಡಬಹುದು. ಯಾವುದನ್ನು ಲಭ್ಯವಾಗಿಸಬೇಕು ಮತ್ತು ಯಾವುದನ್ನು ಈ ಪ್ಲಾನ್‌ನಡಿ ಚಂದಾದಾರರಾಗಿರುವ ಗ್ರಾಹಕರಿಗೆ ಲಭ್ಯವಾಗಿಸಬಾರದು ಎಂಬುದನ್ನು ಆರಿಸಿ. ಈ ಆಯ್ಕೆಗಳು checkout ಫಾರ್ಮ್‌ಗೂ ಅನ್ವಯವಾಗುತ್ತವೆ ಎಂಬುದನ್ನು ಗಮನಿಸಿ, ಆದ್ದರಿಂದ **Not Available** ಎಂದು ಆರಿಸಿದ ಯಾವುದೇ template ಈ ಪ್ಲಾನ್‌ನ ನೋಂದಣಿ ಪುಟದಲ್ಲಿ ಕಾಣಿಸುವುದಿಲ್ಲ.

ಈಗ ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಖಾತೆ ಪುಟದಲ್ಲಿ **Change Site Template** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಬಹುದು.

![ಖಾತೆ ಪುಟದಲ್ಲಿ Change Site Template ಬಟನ್](/img/admin/dashboard.png)  
ಈ ಪ್ಲಾನ್‌ಗೆ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ Site Template ಗಳ ಪಟ್ಟಿ ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಕಾಣಿಸುತ್ತದೆ.

![ಪ್ಲಾನ್‌ಗೆ ಲಭ್ಯವಿರುವ site template ಗಳ ಪಟ್ಟಿ](/img/config/site-templates-list.png)

ಅವರು ಬದಲಾಯಿಸಲು ಬಯಸುವುದನ್ನು ಆರಿಸಿದ ನಂತರ, ಬದಲಾವಣೆಯನ್ನು ದೃಢೀಕರಿಸಲು ಕೇಳಲಾಗುತ್ತದೆ.

![Site template ಬದಲಾವಣೆ ದೃಢೀಕರಣ ಡೈಲಾಗ್](/img/admin/dashboard.png)

ದೃಢೀಕರಣವನ್ನು ಆನ್ ಮಾಡಿ **Process Switch** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ಹೊಸ site template ನಿಮ್ಮ ಗ್ರಾಹಕರ ವೆಬ್‌ಸೈಟ್‌ನಲ್ಲಿ ಬಳಕೆಯಾಗುತ್ತದೆ.

### ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಸೇರಿಸುವುದು:

ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಖಾತೆ ಪುಟದಲ್ಲಿ ಈ ಪ್ಲಾನ್‌ಗೆ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಸೇರಿಸುವ ಆಯ್ಕೆಯನ್ನೂ ಹೊಂದಿರುತ್ತಾರೆ. ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಬಳಸಲು ಅನುಮತಿಸಲು, **Ultimate Multisite > Settings >** **Domain Mapping** ಗೆ ಹೋಗಿ.

**Enable Domain Mapping** ಆಯ್ಕೆಯನ್ನು ಆನ್ ಮಾಡಿ. ಇದು ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ನೆಟ್‌ವರ್ಕ್ ಮಟ್ಟದಲ್ಲಿ ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಬಳಸಲು ಅನುಮತಿಸುತ್ತದೆ.

product ಮಟ್ಟದಲ್ಲಿ domain mapping ಸಕ್ರಿಯವಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಲು ಮರೆಯಬೇಡಿ - ಏಕೆಂದರೆ ನೀವು ಒಂದು product ಗೆ ಗ್ರಾಹಕರು ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಬಳಸದಂತೆ ಮಿತಿಗೊಳಿಸಬಹುದು.

**Ultimate Multisite > Products** ಗೆ ಹೋಗಿ. ನಿಮ್ಮ ಆಯ್ಕೆಯ ಪ್ಲಾನ್ ಆರಿಸಿ **Custom Domains** ಟ್ಯಾಬ್‌ಗೆ ಹೋಗಿ. **Allow Custom Domains** ಆಯ್ಕೆಯನ್ನು ಆನ್ ಮಾಡಿ.

![Allow Custom Domains toggle ಇರುವ Custom Domains ಟ್ಯಾಬ್](/img/config/product-custom-domains.png)

ಇದು ಈ ನಿರ್ದಿಷ್ಟ ಪ್ಲಾನ್‌ಗೆ ಚಂದಾದಾರರಾಗಿರುವ ಎಲ್ಲಾ ಗ್ರಾಹಕರಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಬಳಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಈಗ, ಖಾತೆ ಪುಟದಲ್ಲಿ, ನಿಮ್ಮ ಗ್ರಾಹಕರು **Add Domain** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಸೇರಿಸಬಹುದು.

![ಖಾತೆ ಪುಟದಲ್ಲಿ Add Domain ಬಟನ್](/img/admin/domains-list.png)

ತೆರೆಯುವ ಮೊದಲ ವಿಂಡೋ ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಈ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಕೆಲಸ ಮಾಡಲು ಅವರ DNS ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ಹೇಗೆ ನವೀಕರಿಸಬೇಕು ಎಂಬ ಸೂಚನೆಗಳ ಸಂದೇಶವನ್ನು ತೋರಿಸುತ್ತದೆ.

![ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಸೇರಿಸುವಾಗ ತೋರಿಸುವ DNS ಸೂಚನೆಗಳು](/img/admin/domains-list.png)

ಈ ಸಂದೇಶವನ್ನು (ನಿಮ್ಮಿಂದ) **Ultimate Multisite > Settings > Domain Mapping > Add New Domain Instructions** ನಲ್ಲಿ ಸಂಪಾದಿಸಬಹುದು.

![Domain Mapping ನಲ್ಲಿ Add New Domain Instructions ಸೆಟ್ಟಿಂಗ್](/img/config/settings-domain-mapping.png)

**Next Step** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಹೆಸರನ್ನು ಸೇರಿಸಬಹುದು ಮತ್ತು ಈ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಪ್ರಾಥಮಿಕವಾಗಿರಬೇಕೆ ಎಂದು ಆರಿಸಬಹುದು. ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್‌ಗಳಿಗೆ ಒಂದಕ್ಕಿಂತ ಹೆಚ್ಚು ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ಗಳನ್ನು ಬಳಸಬಹುದು ಎಂಬುದನ್ನು ಗಮನಿಸಿ, ಆದ್ದರಿಂದ ಯಾವುದು ಪ್ರಾಥಮಿಕವಾಗಿರಬೇಕು ಎಂದು ಅವರು ಆರಿಸಬಹುದು.

![ಪ್ರಾಥಮಿಕ ಡೊಮೇನ್ ಆಯ್ಕೆಯೊಂದಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೇನ್ ಹೆಸರು ನಮೂದು](/img/admin/domains-list.png)

**Add Domain** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ಡೊಮೇನ್ ನಿಮ್ಮ ಗ್ರಾಹಕರ ಖಾತೆಗೆ ಸೇರಿಸಲ್ಪಡುತ್ತದೆ. ಈಗ ಅವರು ಮಾಡಬೇಕಾಗಿರುವುದು ಈ ಕಸ್ಟಮ್ ಡೊಮೇನ್‌ನ DNS ರೆಕಾರ್ಡ್‌ಗಳನ್ನು ಅವರ domain registrar ನಲ್ಲಿ ಬದಲಾಯಿಸುವುದು.

### ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾಯಿಸುವುದು:

ಖಾತೆ dashboard ಒಳಗೆ, ನಿಮ್ಮ ಗ್ರಾಹಕರು **Change Password** ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ತಮ್ಮ ಪಾಸ್‌ವರ್ಡ್ ಅನ್ನು ಬದಲಾಯಿಸಬಹುದು.

![ಖಾತೆ ಪುಟದಲ್ಲಿ Change Password ಬಟನ್](/img/admin/dashboard.png)

ಇದು ಹೊಸ ವಿಂಡೋ ತೋರಿಸುತ್ತದೆ, ಅಲ್ಲಿ ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಪ್ರಸ್ತುತ ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಿ ನಂತರ ಬಳಸಲು ಬಯಸುವ ಹೊಸ ಪಾಸ್‌ವರ್ಡ್ ನಮೂದಿಸಬೇಕು.

![ಪ್ರಸ್ತುತ ಮತ್ತು ಹೊಸ ಪಾಸ್‌ವರ್ಡ್ ಕ್ಷೇತ್ರಗಳಿರುವ ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾಯಿಸುವ ಫಾರ್ಮ್](/img/admin/dashboard.png)

### ಅಪಾಯ ವಲಯ:

ನಮ್ಮಲ್ಲಿ **Danger Zone** ಭಾಗದಲ್ಲಿ ಎರಡು ಆಯ್ಕೆಗಳಿವೆ: **Delete Site** ಮತ್ತು **Delete Account**. ಇವೆರಡೂ Danger Zone ಭಾಗದಲ್ಲಿ ಇವೆ ಏಕೆಂದರೆ ಈ ಎರಡು ಕ್ರಿಯೆಗಳನ್ನು ಹಿಂತೆಗೆದುಕೊಳ್ಳಲು ಸಾಧ್ಯವಿಲ್ಲ. ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್ ಅಥವಾ ಖಾತೆಯನ್ನು ಅಳಿಸಿದರೆ, ಅವುಗಳನ್ನು ಮರಳಿ ಪಡೆಯಲು ಸಾಧ್ಯವಿಲ್ಲ.

![Delete Site ಮತ್ತು Delete Account ಆಯ್ಕೆಗಳಿರುವ Danger Zone](/img/admin/dashboard.png)

ನಿಮ್ಮ ಗ್ರಾಹಕರು ಈ ಎರಡು ಆಯ್ಕೆಗಳಲ್ಲಿ ಯಾವುದಾದರೂ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದರೆ, ಅವರಿಗೆ ವೆಬ್‌ಸೈಟ್ ಅಥವಾ ಖಾತೆಯನ್ನು ತೆಗೆದುಹಾಕುವ ಆಯ್ಕೆಯನ್ನು ಆನ್ ಮಾಡಬೇಕಾದ ವಿಂಡೋ ತೋರಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಈ ಕ್ರಿಯೆಯನ್ನು ರದ್ದುಗೊಳಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ ಎಂದು ಎಚ್ಚರಿಸಲಾಗುತ್ತದೆ.

![Site ಅಳಿಸುವ ದೃಢೀಕರಣ ಡೈಲಾಗ್](/img/admin/dashboard.png)

![Account ಅಳಿಸುವ ದೃಢೀಕರಣ ಡೈಲಾಗ್](/img/admin/dashboard.png)

ಅವರು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್ ಅಳಿಸಿದರೆ, ಅವರ ಖಾತೆ ಮತ್ತು ಸದಸ್ಯತ್ವ ಬದಲಾಗದೆ ಉಳಿಯುತ್ತದೆ. ಅವರು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್‌ನಲ್ಲಿರುವ ಎಲ್ಲಾ ವಿಷಯವನ್ನು ಮಾತ್ರ ಕಳೆದುಕೊಳ್ಳುತ್ತಾರೆ. ಅವರು ತಮ್ಮ ಖಾತೆಯನ್ನು ಅಳಿಸಿದರೆ, ಎಲ್ಲಾ ವೆಬ್‌ಸೈಟ್‌ಗಳು, ಸದಸ್ಯತ್ವಗಳು ಮತ್ತು ಈ ಖಾತೆಗೆ ಸಂಬಂಧಿಸಿದ ಮಾಹಿತಿ ಕಳೆದುಹೋಗುತ್ತದೆ.
