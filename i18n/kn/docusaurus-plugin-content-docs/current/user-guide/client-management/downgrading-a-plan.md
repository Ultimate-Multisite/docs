# ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದು (v2)

_**ಮುಖ್ಯ ಸೂಚನೆ: ಈ ಲೇಖನವು Ultimate Multisite ಆವೃತ್ತಿ 2.x ಅನ್ನು ಉಲ್ಲೇಖಿಸುತ್ತದೆ.**_

ಯೋಜನೆ ಅಥವಾ ಸಬ್‌ಸ್ಕ್ರಿಪ್ಷನ್ ಅನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದು ನಿಮ್ಮ ಗ್ರಾಹಕರು ಸಾಮಾನ್ಯವಾಗಿ ಮಾಡುವ ಕ್ರಿಯೆಯಾಗಿದೆ - ಅವರ ಬಜೆಟ್ ಸೀಮಿತವಾಗಿದ್ದರೆ ಅಥವಾ ತಮ್ಮ ಸಬ್‌ಸೈಟ್ ನಡೆಸಲು ಹೆಚ್ಚಿನ ಸಂಪನ್ಮೂಲಗಳ ಅಗತ್ಯವಿಲ್ಲ ಎಂದು ನಿರ್ಧರಿಸಿದರೆ.

## ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವ ವಿಧಾನ

ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಸಬ್‌ಸೈಟ್ ಅಡ್ಮಿನ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ಗೆ ಲಾಗಿನ್ ಆಗಿ, ಅವರ ಖಾತೆ ಪುಟದಲ್ಲಿ **Change** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಯಾವಾಗ ಬೇಕಾದರೂ ತಮ್ಮ ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡಬಹುದು.

![ಸದಸ್ಯತ್ವದ ಕೆಳಗೆ Change ಬಟನ್ ಇರುವ ಖಾತೆ ಪುಟ](/img/admin/memberships-list.png)

**Change** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ಬಳಕೆದಾರ/ಗ್ರಾಹಕರನ್ನು ಚೆಕ್‌ಔಟ್ ಪುಟಕ್ಕೆ ಕಳುಹಿಸಲಾಗುತ್ತದೆ, ಅಲ್ಲಿ ಅವರು ತಮ್ಮ ಸಬ್‌ಸ್ಕ್ರಿಪ್ಷನ್ ಅನ್ನು ಬದಲಾಯಿಸಲು ಬಯಸುವ ಯೋಜನೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು.

![ಡೌನ್‌ಗ್ರೇಡ್‌ಗಾಗಿ ಯೋಜನೆ ಆಯ್ಕೆಗಳನ್ನು ತೋರಿಸುವ ಚೆಕ್‌ಔಟ್ ಪುಟ](/img/admin/memberships-list.png)

ಈ ಉದಾಹರಣೆಯಲ್ಲಿ, ನಾವು **Premium** ಯೋಜನೆಯಿಂದ **Free** ಯೋಜನೆಗೆ ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುತ್ತಿದ್ದೇವೆ.

ಮುಂದುವರಿಯಲು ಬಳಕೆದಾರರು **Complete Checkout** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಬೇಕು. ಅದು ಅವರನ್ನು ಖಾತೆ ಪುಟಕ್ಕೆ ಮರಳಿ ಕರೆದೊಯ್ಯುತ್ತದೆ, ಅಲ್ಲಿ ಸದಸ್ಯತ್ವದ ಬಾಕಿ ಇರುವ ಬದಲಾವಣೆಯ ಸಂದೇಶವನ್ನು ತೋರಿಸಲಾಗುತ್ತದೆ. ಬದಲಾವಣೆಗಳು ಗ್ರಾಹಕರ **ಮುಂದಿನ ಬಿಲ್ಲಿಂಗ್ ಸೈಕಲ್**ನಲ್ಲಿ ಜಾರಿಗೆ ಬರುತ್ತವೆ.

![ಬಾಕಿ ಇರುವ ಸದಸ್ಯತ್ವ ಬದಲಾವಣೆ ಸಂದೇಶವನ್ನು ತೋರಿಸುವ ಖಾತೆ ಪುಟ](/img/admin/memberships-list.png)

### ಬಳಕೆದಾರರು ತಮ್ಮ ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡಿದಾಗ ಏನಾಗುತ್ತದೆ

ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದರಿಂದ ಬಳಕೆದಾರರ ಸಬ್‌ಸೈಟ್‌ನಲ್ಲಿರುವ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಕಾನ್ಫಿಗರೇಶನ್ ಬದಲಾಗುವುದಿಲ್ಲ ಎಂಬುದನ್ನು ಗಮನಿಸುವುದು ಮುಖ್ಯ.

ಇದು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸೈಟ್ ಟೆಂಪ್ಲೇಟ್ ಅನ್ನು ಬದಲಾಯಿಸುವುದಿಲ್ಲ ಏಕೆಂದರೆ ಸೈಟ್ ಟೆಂಪ್ಲೇಟ್ ಬದಲಾಯಿಸುವುದರಿಂದ ಸಬ್‌ಸೈಟ್ ಸಂಪೂರ್ಣವಾಗಿ ಅಳಿಸಲ್ಪಡುತ್ತದೆ ಮತ್ತು ರೀಸೆಟ್ ಆಗುತ್ತದೆ. ಅನಗತ್ಯ ಡೇಟಾ ನಷ್ಟವನ್ನು ತಪ್ಪಿಸಲು ಇದನ್ನು ಮಾಡಲಾಗುತ್ತದೆ. ಆದ್ದರಿಂದ ಡಿಸ್ಕ್ ಸ್ಪೇಸ್, ಥೀಮ್‌ಗಳು, plugins ಇತ್ಯಾದಿಗಳು ಪೋಸ್ಟ್‌ಗಳನ್ನು ಹೊರತುಪಡಿಸಿ ಹಾಗೆಯೇ ಉಳಿಯುತ್ತವೆ.

ಪ್ರತಿ ಯೋಜನೆಯ ಅಡಿಯಲ್ಲಿ ನೀವು ಹೊಂದಿಸಿದ ಮಿತಿಗಳು ಮತ್ತು ಕೋಟಾಗಳು ನಿಮ್ಮ ಮುಖ್ಯ ಕಾಳಜಿಯಾಗಿರುತ್ತದೆ ಎಂದು ನಮಗೆ ಅರ್ಥವಾಗುತ್ತದೆ, ಆದರೆ ಬಳಕೆದಾರರ ಸಬ್‌ಸೈಟ್‌ನ ಯಾವುದೇ ಕಾನ್ಫಿಗರೇಶನ್‌ಗಳನ್ನು ಅಳಿಸಿದರೆ ಅಥವಾ ಬದಲಾಯಿಸಿದರೆ ಅದರಿಂದ ಆಗುವ ಹಾನಿಯನ್ನು ನಾವು ಪರಿಗಣಿಸಬೇಕಾಗುತ್ತದೆ.

ಯೋಜನೆಯಲ್ಲಿ ಹೊಂದಿಸಿದ ಮಿತಿಯನ್ನು ಮೀರಿದ ಪೋಸ್ಟ್‌ಗಳಿಗೆ, ನಿಮಗೆ 3 ವಿಭಿನ್ನ ಆಯ್ಕೆಗಳಿವೆ: **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಹಾಗೆಯೇ ಇಡಿ**, **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಟ್ರ್ಯಾಶ್‌ಗೆ ಸರಿಸಿ**, ಅಥವಾ **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಡ್ರಾಫ್ಟ್‌ಗೆ ಸರಿಸಿ**. ನೀವು ಇದನ್ನು Ultimate Multisite ಸೆಟ್ಟಿಂಗ್ಸ್‌ನಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಬಹುದು.

![Ultimate Multisite ಸೆಟ್ಟಿಂಗ್ಸ್‌ನಲ್ಲಿ ಪೋಸ್ಟ್ ಮಿತಿ ಮೀರಿದ ಆಯ್ಕೆಗಳು](/img/config/settings-sites.png)

### ಪಾವತಿಗೆ ಏನಾಗುತ್ತದೆ

ಆವೃತ್ತಿ 2.0 ನಲ್ಲಿ, proration ವಿಷಯದಲ್ಲಿ ಪಾವತಿಯ ಮೇಲೆ ಯಾವುದೇ ಹೊಂದಾಣಿಕೆಗಳ ಅಗತ್ಯವಿಲ್ಲ.

ಇದು ಏಕೆಂದರೆ ಹೊಸ ಯೋಜನೆ/ಸದಸ್ಯತ್ವವು ಜಾರಿಗೆ ಬರುವ ಮೊದಲು ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಸದಸ್ಯತ್ವವು **ತನ್ನ ಬಿಲ್ಲಿಂಗ್ ಸೈಕಲ್ ಪೂರ್ಣಗೊಳಿಸುವವರೆಗೆ** ಸಿಸ್ಟಂ ಕಾಯುತ್ತದೆ. ಹೊಸ ಸದಸ್ಯತ್ವಕ್ಕೆ ಹೊಸ ಬಿಲ್ಲಿಂಗ್ ಮೊತ್ತವು ಮುಂದಿನ ಬಿಲ್ಲಿಂಗ್ ಸೈಕಲ್‌ನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನ್ವಯಿಸಲ್ಪಡುತ್ತದೆ ಮತ್ತು ಚಾರ್ಜ್ ಮಾಡಲಾಗುತ್ತದೆ.
