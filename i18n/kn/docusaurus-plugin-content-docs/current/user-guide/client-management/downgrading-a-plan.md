# ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದು (v2)

_**ಪ್ರಮುಖ ಸೂಚನೆ: ಈ ಲೇಖನವು Ultimate Multisite ಆವೃತ್ತಿ 2.x ಅನ್ನು ಉಲ್ಲೇಖಿಸುತ್ತದೆ.**_

ನಿಮ್ಮ ಕ್ಲೈಂಟ್‌ಗಳಿಗೆ ಸೀಮಿತ ಬಜೆಟ್ ಇದ್ದರೆ ಅಥವಾ ತಮ್ಮ ಉಪಸೈಟ್ ನಡೆಸಲು ಹೆಚ್ಚು ಸಂಪನ್ಮೂಲಗಳು ಬೇಕಾಗುವುದಿಲ್ಲ ಎಂದು ನಿರ್ಧರಿಸಿದರೆ, ಯೋಜನೆ ಅಥವಾ ಚಂದಾದಾರಿಕೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದು ಅವರು ಸಾಮಾನ್ಯವಾಗಿ ಮಾಡುವ ಕ್ರಿಯೆಯಾಗಿದೆ.

## ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದು ಹೇಗೆ {#how-to-downgrade-a-plan}

ನಿಮ್ಮ ಕ್ಲೈಂಟ್‌ಗಳು ತಮ್ಮ ಉಪಸೈಟ್ ನಿರ್ವಾಹಕ Dashboardಗೆ ಲಾಗಿನ್ ಮಾಡಿ, ತಮ್ಮ Account ಪುಟದ ಅಡಿಯಲ್ಲಿ **ಬದಲಿಸಿ** ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಯಾವುದೇ ಸಮಯದಲ್ಲಿ ತಮ್ಮ ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡಬಹುದು.

![Your Membership ಕಾರ್ಡ್ ಮತ್ತು ಬದಲಿಸಿ ಬಟನ್ ಇರುವ ಗ್ರಾಹಕರ Account ಪುಟ](/img/account-page/membership-change-button.png)

**ಬದಲಿಸಿ** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದ ನಂತರ, ಬಳಕೆದಾರ/ಕ್ಲೈಂಟ್ ಅವರನ್ನು ಚೆಕ್‌ಔಟ್ ಪುಟಕ್ಕೆ ಮರುನಿರ್ದೇಶಿಸಲಾಗುತ್ತದೆ. ಅಲ್ಲಿ ಅವರು ತಮ್ಮ ಚಂದಾದಾರಿಕೆಯನ್ನು ಬದಲಿಸಲು ಬೇಕಾದ ಯೋಜನೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು.

![ಗ್ರಾಹಕ ಬದಿಯಲ್ಲಿನ ಡೌನ್‌ಗ್ರೇಡ್ ಯೋಜನೆ ಆಯ್ಕೆಗಳ ಪುಟ](/img/account-page/downgrade-picker.png)

ಈ ಉದಾಹರಣೆಯಲ್ಲಿ, ನಾವು ಯೋಜನೆಯನ್ನು **Premium** ಇಂದ **Free** ಗೆ ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುತ್ತಿದ್ದೇವೆ.

ಮುಂದುವರಿಯಲು ಬಳಕೆದಾರರು **ಚೆಕ್‌ಔಟ್ ಪೂರ್ಣಗೊಳಿಸಿ** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿದರೆ ಸಾಕು. ನಂತರ ಅದು ಅವರನ್ನು Account ಪುಟಕ್ಕೆ ಮರಳಿ ಕರೆದೊಯ್ಯುತ್ತದೆ ಮತ್ತು ಸದಸ್ಯತ್ವದ ಬಾಕಿ ಇರುವ ಬದಲಾವಣೆಯ ಕುರಿತು ಸಂದೇಶವನ್ನು ತೋರಿಸುತ್ತದೆ. ಬದಲಾವಣೆಗಳು ಗ್ರಾಹಕರ **ಮುಂದಿನ ಬಿಲ್ಲಿಂಗ್ ಚಕ್ರದಲ್ಲಿ** ಜಾರಿಗೆ ಬರುತ್ತವೆ.

![ಬಾಕಿ ಇರುವ ಸದಸ್ಯತ್ವ ಬದಲಾವಣೆ ಬ್ಯಾನರ್ ತೋರಿಸುವ Account ಪುಟ](/img/account-page/pending-change.png)

### ಬಳಕೆದಾರರು ತಮ್ಮ ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡಿದಾಗ ಏನಾಗುತ್ತದೆ {#what-happens-when-a-user-downgrades-their-plan}

ಯೋಜನೆಯನ್ನು ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುವುದರಿಂದ ಬಳಕೆದಾರರ ಉಪಸೈಟ್‌ನಲ್ಲಿರುವ ಈಗಿನ ಸಂರಚನೆ ಬದಲಾಗುವುದಿಲ್ಲ ಎಂಬುದನ್ನು ಗಮನಿಸುವುದು ಮುಖ್ಯ.

ಇದು ಸೈಟ್ ಟೆಂಪ್ಲೇಟ್ ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬದಲಿಸುವುದಿಲ್ಲ, ಏಕೆಂದರೆ ಸೈಟ್ ಟೆಂಪ್ಲೇಟ್ ಬದಲಿಸಿದರೆ ಉಪಸೈಟ್ ಸಂಪೂರ್ಣವಾಗಿ ಅಳಿಸಲ್ಪಟ್ಟು ಮರುಹೊಂದಿಸಲಾಗುತ್ತದೆ. ಅನಗತ್ಯ ಡೇಟಾ ನಷ್ಟವನ್ನು ತಪ್ಪಿಸಲು ಹೀಗೆ ಮಾಡಲಾಗಿದೆ. ಆದ್ದರಿಂದ ಪೋಸ್ಟ್‌ಗಳನ್ನು ಹೊರತುಪಡಿಸಿ, ಡಿಸ್ಕ್ ಸ್ಥಳ, ಥೀಮ್‌ಗಳು, ಪ್ಲಗಿನ್‌ಗಳು ಇತ್ಯಾದಿ ಹಾಗೆಯೇ ಉಳಿಯುತ್ತವೆ.

ಪ್ರತಿ ಯೋಜನೆಯ ಅಡಿಯಲ್ಲಿ ನೀವು ನಿಗದಿಪಡಿಸಿರುವ ಮಿತಿಗಳು ಮತ್ತು ಕೋಟಾಗಳೇ ನಿಮ್ಮ ಮುಖ್ಯ ಕಾಳಜಿಯಾಗಿರಬಹುದು ಎಂದು ನಮಗೆ ತಿಳಿದಿದೆ. ಆದರೆ ಬಳಕೆದಾರರ ಉಪಸೈಟ್‌ನ ಯಾವುದೇ ಸಂರಚನೆಗಳನ್ನು ಅಳಿಸಿದರೆ ಅಥವಾ ಬದಲಿಸಿದರೆ ಉಂಟಾಗುವ ಹಾನಿಯನ್ನೂ ನಾವು ಪರಿಗಣಿಸಬೇಕು.

ಯೋಜನೆಯಲ್ಲಿ ನಿಗದಿಪಡಿಸಿದ ಮಿತಿಯನ್ನು ಮೀರಿದ ಪೋಸ್ಟ್‌ಗಳಿಗೆ, ನಿಮಗೆ 3 ಬೇರೆ ಆಯ್ಕೆಗಳು ಇವೆ: **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಹಾಗೆಯೇ ಉಳಿಸಿ** *,* **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಕಸದ ಬುಟ್ಟಿಗೆ ಸರಿಸಿ** *,* ಅಥವಾ **ಪೋಸ್ಟ್‌ಗಳನ್ನು ಕರಡುಗೆ ಸರಿಸಿ** *.* ಇದನ್ನು Ultimate Multisite ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಅಡಿಯಲ್ಲಿ ಸಂರಚಿಸಬಹುದು.

![ಪೋಸ್ಟ್ ಮಿತಿ ಮೀರಿದಾಗ ನಡೆದುಕೊಳ್ಳುವ ಆಯ್ಕೆಗಳು ತೋರಿಸುವ Network Admin ಸೆಟ್ಟಿಂಗ್‌ಗಳ Sites ಪುಟ](/img/account-page/settings-sites-post-limit.png)

### ಪಾವತಿಗೆ ಏನಾಗುತ್ತದೆ {#what-happens-to-the-payment}

ಆವೃತ್ತಿ 2.0 ನಲ್ಲಿ, ಪ್ರೊರೇಶನ್ ದೃಷ್ಟಿಯಿಂದ ಪಾವತಿಯಲ್ಲಿ ಇನ್ನು ಯಾವುದೇ ಹೊಂದಾಣಿಕೆಗಳ ಅಗತ್ಯವಿಲ್ಲ.

ಇದಕ್ಕೆ ಕಾರಣ, ಹೊಸ ಯೋಜನೆ/ಸದಸ್ಯತ್ವ ಜಾರಿಗೆ ಬರುವ ಮೊದಲು ವ್ಯವಸ್ಥೆಯು ಈಗಿನ ಸದಸ್ಯತ್ವವು **ತನ್ನ ಬಿಲ್ಲಿಂಗ್ ಚಕ್ರವನ್ನು ಪೂರ್ಣಗೊಳಿಸುವವರೆಗೆ** ಕಾಯುತ್ತದೆ. ಹೊಸ ಸದಸ್ಯತ್ವಕ್ಕೆ ಹೊಸ ಬಿಲ್ಲಿಂಗ್ ಮೊತ್ತವು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನ್ವಯಿಸಿ ಮುಂದಿನ ಬಿಲ್ಲಿಂಗ್ ಚಕ್ರದಲ್ಲಿ ವಿಧಿಸಲಾಗುತ್ತದೆ.
