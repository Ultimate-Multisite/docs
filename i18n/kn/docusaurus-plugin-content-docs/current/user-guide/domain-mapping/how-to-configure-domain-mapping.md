# ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಅನ್ನು ಹೇಗೆ ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು (v2)

_**ಮುಖ್ಯ ಸೂಚನೆ: ಈ ಲೇಖನವು Ultimate Multisite ಆವೃತ್ತಿ 2.x ಗೆ ಸಂಬಂಧಿಸಿದೆ.**_

ಪ್ರೀಮಿಯಂ ನೆಟ್‌ವರ್ಕ್‌ನ ಅತ್ಯಂತ ಶಕ್ತಿಶಾಲಿ ವೈಶಿಷ್ಟ್ಯಗಳಲ್ಲಿ ಒಂದು ಎಂದರೆ ನಮ್ಮ ಗ್ರಾಹಕರಿಗೆ ಅವರ ಸೈಟ್‌ಗಳಿಗೆ ಟಾಪ್-ಲೆವೆಲ್ ಡೊಮೈನ್ ಜೋಡಿಸುವ ಅವಕಾಶ ನೀಡುವುದು. ಯೋಚಿಸಿ ನೋಡಿ, ಯಾವುದು ಹೆಚ್ಚು ವೃತ್ತಿಪರವಾಗಿ ಕಾಣುತ್ತದೆ: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) ಅಥವಾ [_**joesbikeshop.com**_](http://joesbikeshop.com)? ಅದಕ್ಕಾಗಿಯೇ Ultimate Multisite ಈ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಅಂತರ್ನಿರ್ಮಿತವಾಗಿ ನೀಡುತ್ತದೆ, ಥರ್ಡ್-ಪಾರ್ಟಿ plugin ಗಳನ್ನು ಬಳಸುವ ಅಗತ್ಯವಿಲ್ಲದೆ.

## ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಎಂದರೇನು?

ಹೆಸರೇ ಸೂಚಿಸುವಂತೆ, ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಎಂದರೆ Ultimate Multisite ನೀಡುವ ಒಂದು ಸೌಲಭ್ಯ - ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ಗಾಗಿ ಬರುವ ವಿನಂತಿಯನ್ನು ಸ್ವೀಕರಿಸಿ, ಆ ನಿರ್ದಿಷ್ಟ ಡೊಮೈನ್ ಜೋಡಿಸಲಾದ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿರುವ ಸಂಬಂಧಿತ ಸೈಟ್‌ಗೆ ಆ ವಿನಂತಿಯನ್ನು ಮ್ಯಾಪ್ ಮಾಡುವುದು.

### ನಿಮ್ಮ Ultimate Multisite ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಅನ್ನು ಹೇಗೆ ಸೆಟಪ್ ಮಾಡುವುದು

ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಕೆಲಸ ಮಾಡಲು ನಿಮ್ಮ ಕಡೆಯಿಂದ ಸ್ವಲ್ಪ ಸೆಟಪ್ ಅಗತ್ಯವಿದೆ. ಒಳ್ಳೆಯ ಸುದ್ದಿ ಎಂದರೆ, Ultimate Multisite ಕಠಿಣ ಕೆಲಸವನ್ನು ನಿಮಗಾಗಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಮಾಡುತ್ತದೆ, ಆದ್ದರಿಂದ ನೀವು ಅವಶ್ಯಕತೆಗಳನ್ನು ಸುಲಭವಾಗಿ ಪೂರೈಸಬಹುದು.

Ultimate Multisite ಇನ್‌ಸ್ಟಾಲೇಶನ್ ಸಮಯದಲ್ಲಿ, ವಿಝಾರ್ಡ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ **sunrise.php** ಅನ್ನು ನಿಗದಿತ ಫೋಲ್ಡರ್‌ಗೆ ನಕಲಿಸಿ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುತ್ತದೆ. **ಈ ಹಂತ ಪೂರ್ಣಗೊಳ್ಳುವವರೆಗೆ ವಿಝಾರ್ಡ್ ನಿಮ್ಮನ್ನು ಮುಂದುವರಿಯಲು ಬಿಡುವುದಿಲ್ಲ**.

![sunrise.php ಹಂತದೊಂದಿಗೆ Ultimate Multisite ಇನ್‌ಸ್ಟಾಲೇಶನ್ ವಿಝಾರ್ಡ್](/img/config/settings-domain-mapping.png)

ಇದರರ್ಥ Ultimate Multisite ಇನ್‌ಸ್ಟಾಲೇಶನ್ ವಿಝಾರ್ಡ್ ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್ ಸೆಟಪ್ ಮಾಡುವುದನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನೀವು ತಕ್ಷಣವೇ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಮ್ಯಾಪ್ ಮಾಡಲು ಪ್ರಾರಂಭಿಸಬಹುದು.

Ultimate Multisite ನಲ್ಲಿ ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಕಡ್ಡಾಯವಲ್ಲ ಎಂಬುದನ್ನು ಗಮನಿಸಿ. WordPress Multisite ನ ಸ್ಥಳೀಯ ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಕಾರ್ಯ ಅಥವಾ ಬೇರೆ ಯಾವುದೇ ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಪರಿಹಾರವನ್ನು ಬಳಸುವ ಆಯ್ಕೆ ನಿಮಗಿದೆ.

ಇತರ ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಪರಿಹಾರಗಳಿಗೆ ದಾರಿ ಮಾಡಿಕೊಡಲು Ultimate Multisite ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಅನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಬೇಕಾದರೆ, **Ultimate Multisite > Settings > Domain Mapping** ಅಡಿಯಲ್ಲಿ ಈ ವೈಶಿಷ್ಟ್ಯವನ್ನು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಬಹುದು.

![ಎನೇಬಲ್ ಟಾಗಲ್‌ನೊಂದಿಗೆ Domain Mapping ಸೆಟ್ಟಿಂಗ್ಸ್](/img/config/settings-domain-mapping.png)

ಈ ಆಯ್ಕೆಯ ಕೆಳಗೆ, **Force Admin Redirect** ಆಯ್ಕೆಯನ್ನು ಸಹ ನೀವು ನೋಡಬಹುದು. ಈ ಆಯ್ಕೆಯು ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಮತ್ತು ಸಬ್‌ಡೊಮೈನ್ ಎರಡರಲ್ಲೂ ತಮ್ಮ admin dashboard ಅನ್ನು ಪ್ರವೇಶಿಸಬಹುದೇ ಅಥವಾ ಕೇವಲ ಒಂದರಲ್ಲಿ ಮಾತ್ರ ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.

ನೀವು **Force redirect to mapped domain** ಆಯ್ಕೆ ಮಾಡಿದರೆ, ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ಗಳಲ್ಲಿ ಮಾತ್ರ ತಮ್ಮ admin dashboard ಅನ್ನು ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

**Force redirect to network domain** ಆಯ್ಕೆಯು ನಿಖರವಾಗಿ ವಿರುದ್ಧವಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ - ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ಗಳಲ್ಲಿ ಸೈನ್ ಇನ್ ಮಾಡಲು ಪ್ರಯತ್ನಿಸಿದರೂ, ಅವರು ತಮ್ಮ ಸಬ್‌ಡೊಮೈನ್‌ನಲ್ಲಿ ಮಾತ್ರ ತಮ್ಮ dashboard ಗಳನ್ನು ಪ್ರವೇಶಿಸಲು ಅನುಮತಿಸಲಾಗುತ್ತದೆ.

ಮತ್ತು **Allow access to the admin by both mapped domain and network domain** ಆಯ್ಕೆಯು ಅವರಿಗೆ ಸಬ್‌ಡೊಮೈನ್ ಮತ್ತು ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಎರಡರಲ್ಲೂ ತಮ್ಮ admin dashboard ಗಳನ್ನು ಪ್ರವೇಶಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.

![ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್‌ಗಾಗಿ Force Admin Redirect ಆಯ್ಕೆಗಳು](/img/config/settings-domain-mapping.png)

ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಅನ್ನು ಮ್ಯಾಪ್ ಮಾಡಲು ಎರಡು ವಿಧಾನಗಳಿವೆ. ಮೊದಲನೆಯದು ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್ admin dashboard ನಿಂದ super admin ಆಗಿ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಮ್ಯಾಪ್ ಮಾಡುವುದು ಮತ್ತು ಎರಡನೆಯದು ಖಾತೆ ಪುಟದ ಅಡಿಯಲ್ಲಿ subsite admin dashboard ಮೂಲಕ.

ಆದರೆ ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿರುವ ಯಾವುದೇ subsite ಗಳಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಅನ್ನು ಮ್ಯಾಪ್ ಮಾಡಲು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ಡೊಮೈನ್ ಹೆಸರಿನ **DNS ಸೆಟ್ಟಿಂಗ್ಸ್** ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಆಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬೇಕು.

### 

### ಡೊಮೈನ್ DNS ಸೆಟ್ಟಿಂಗ್ಸ್ ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಆಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳುವುದು

ಮ್ಯಾಪಿಂಗ್ ಕೆಲಸ ಮಾಡಲು, ನೀವು ಮ್ಯಾಪ್ ಮಾಡಲು ಯೋಜಿಸುತ್ತಿರುವ ಡೊಮೈನ್ ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನ IP ವಿಳಾಸಕ್ಕೆ ಪಾಯಿಂಟ್ ಆಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬೇಕು. ನಿಮಗೆ ನೆಟ್‌ವರ್ಕ್ IP ವಿಳಾಸ ಬೇಕು - Ultimate Multisite ಇನ್‌ಸ್ಟಾಲ್ ಆಗಿರುವ ಡೊಮೈನ್‌ನ IP ವಿಳಾಸ - ನೀವು ಮ್ಯಾಪ್ ಮಾಡಲು ಬಯಸುವ ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ನ IP ವಿಳಾಸವಲ್ಲ ಎಂಬುದನ್ನು ಗಮನಿಸಿ. ನಿರ್ದಿಷ್ಟ ಡೊಮೈನ್‌ನ IP ವಿಳಾಸವನ್ನು ಹುಡುಕಲು, ಉದಾಹರಣೆಗೆ [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) ಗೆ ಹೋಗುವುದನ್ನು ನಾವು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ.

ಡೊಮೈನ್ ಅನ್ನು ಸರಿಯಾಗಿ ಮ್ಯಾಪ್ ಮಾಡಲು, ನಿಮ್ಮ **DNS** ಕಾನ್ಫಿಗರೇಶನ್‌ನಲ್ಲಿ ಆ **IP ವಿಳಾಸ** ಕ್ಕೆ ಪಾಯಿಂಟ್ ಆಗುವಂತೆ **A RECORD** ಸೇರಿಸಬೇಕು. DNS ನಿರ್ವಹಣೆಯು ವಿವಿಧ ಡೊಮೈನ್ ರಿಜಿಸ್ಟ್ರಾರ್‌ಗಳ ನಡುವೆ ಬಹಳ ಭಿನ್ನವಾಗಿರುತ್ತದೆ, ಆದರೆ "_Creating A Record on XXXX_" ಎಂದು ಹುಡುಕಿದರೆ ಆನ್‌ಲೈನ್‌ನಲ್ಲಿ ಬಹಳಷ್ಟು ಟ್ಯುಟೋರಿಯಲ್‌ಗಳು ಸಿಗುತ್ತವೆ, ಇಲ್ಲಿ XXXX ನಿಮ್ಮ ಡೊಮೈನ್ ರಿಜಿಸ್ಟ್ರಾರ್ (ಉದಾ.: "_Creating A Record on GoDaddy_").

ಇದನ್ನು ಕೆಲಸ ಮಾಡಲು ನಿಮಗೆ ತೊಂದರೆಯಾದರೆ, **ನಿಮ್ಮ ಡೊಮೈನ್ ರಿಜಿಸ್ಟ್ರಾರ್ ಬೆಂಬಲವನ್ನು ಸಂಪರ್ಕಿಸಿ** ಮತ್ತು ಅವರು ಈ ಭಾಗದಲ್ಲಿ ನಿಮಗೆ ಸಹಾಯ ಮಾಡಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

ನಿಮ್ಮ ಗ್ರಾಹಕರು ತಮ್ಮ ಸ್ವಂತ ಡೊಮೈನ್‌ಗಳನ್ನು ಮ್ಯಾಪ್ ಮಾಡಲು ಅನುಮತಿಸಲು ನೀವು ಯೋಜಿಸಿದರೆ, ಅವರು ಈ ಭಾಗದ ಕೆಲಸವನ್ನು ತಾವೇ ಮಾಡಬೇಕಾಗುತ್ತದೆ. A Record ರಚಿಸಲು ಅವರಿಗೆ ಸಾಧ್ಯವಾಗದಿದ್ದರೆ ಅವರ ರಿಜಿಸ್ಟ್ರಾರ್ ಬೆಂಬಲ ವ್ಯವಸ್ಥೆಯ ಕಡೆಗೆ ಅವರನ್ನು ಮಾರ್ಗದರ್ಶಿಸಿ.

### Super Admin ಆಗಿ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಮ್ಯಾಪ್ ಮಾಡುವುದು

ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನಲ್ಲಿ super admin ಆಗಿ ಲಾಗ್ ಇನ್ ಆಗಿರುವಾಗ, **Ultimate Multisite > Domains** ಅಡಿಯಲ್ಲಿ ಹೋಗುವ ಮೂಲಕ ನೀವು ಸುಲಭವಾಗಿ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರುಗಳನ್ನು ಸೇರಿಸಬಹುದು ಮತ್ತು ನಿರ್ವಹಿಸಬಹುದು.

![Ultimate Multisite ನಲ್ಲಿ Domains ಪಟ್ಟಿ ಪುಟ](/img/admin/domains-list.png)

ಈ ಪುಟದ ಅಡಿಯಲ್ಲಿ, ಮೇಲ್ಭಾಗದಲ್ಲಿರುವ **Add Domain** ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಬಹುದು ಮತ್ತು ಇದು ಒಂದು modal ವಿಂಡೋವನ್ನು ತರುತ್ತದೆ, ಅಲ್ಲಿ ನೀವು **ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರು**, ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಅನ್ವಯಿಸಲು ಬಯಸುವ **subsite**, ಮತ್ತು ಇದನ್ನು **ಪ್ರಾಥಮಿಕ ಡೊಮೈನ್** ಹೆಸರಾಗಿ ಹೊಂದಿಸಬೇಕೇ ಎಂದು ನಿರ್ಧರಿಸಬಹುದು (ನೀವು **ಒಂದು subsite ಗೆ ಬಹು ಡೊಮೈನ್ ಹೆಸರುಗಳನ್ನು ಮ್ಯಾಪ್ ಮಾಡಬಹುದು** ಎಂಬುದನ್ನು ಗಮನಿಸಿ).

![ಡೊಮೈನ್ ಹೆಸರು ಮತ್ತು site ಫೀಲ್ಡ್‌ಗಳೊಂದಿಗೆ Add Domain modal](/img/admin/domains-list.png)

ಎಲ್ಲಾ ಮಾಹಿತಿಯನ್ನು ನಮೂದಿಸಿದ ನಂತರ, ಕೆಳಭಾಗದಲ್ಲಿರುವ **Add Existing Domain** ಬಟನ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಬಹುದು.

ಇದು ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ನ DNS ಮಾಹಿತಿಯನ್ನು ಪರಿಶೀಲಿಸುವ ಮತ್ತು ಪಡೆಯುವ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ. ಪುಟದ ಕೆಳಭಾಗದಲ್ಲಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅನುಸರಿಸಲು ಒಂದು ಲಾಗ್ ಸಹ ಕಾಣಿಸುತ್ತದೆ. ಈ ಪ್ರಕ್ರಿಯೆ ಪೂರ್ಣಗೊಳ್ಳಲು ಕೆಲವು ನಿಮಿಷಗಳು ತೆಗೆದುಕೊಳ್ಳಬಹುದು.

ಎಲ್ಲವೂ ಸರಿಯಾಗಿ ಸೆಟಪ್ ಆಗಿದ್ದರೆ **Stage** ಅಥವಾ ಸ್ಥಿತಿಯು **Checking DNS** ನಿಂದ **Ready** ಗೆ ಬದಲಾಗಬೇಕು.

![Checking DNS ನಿಂದ Ready ಗೆ ಬದಲಾಗುತ್ತಿರುವ Domain stage](/img/admin/domains-list.png)

![domains ಪಟ್ಟಿಯಲ್ಲಿ Ready ಸ್ಥಿತಿಯಲ್ಲಿರುವ Domain](/img/admin/domains-list.png)

ಡೊಮೈನ್ ಹೆಸರಿನ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿದರೆ, ಅದರೊಳಗೆ ಕೆಲವು ಆಯ್ಕೆಗಳನ್ನು ನೋಡಬಹುದು. ಅವುಗಳನ್ನು ಸಂಕ್ಷಿಪ್ತವಾಗಿ ನೋಡೋಣ:

![Stage, site, active, ಮತ್ತು SSL ಆಯ್ಕೆಗಳೊಂದಿಗೆ Domain ವಿವರ ಪುಟ](/img/admin/domains-list.png)

**Stage:** ಡೊಮೈನ್ ಇರುವ ಹಂತ ಇದು. ನೀವು ಮೊದಲು ಡೊಮೈನ್ ಸೇರಿಸಿದಾಗ, ಇದು ಬಹುಶಃ **Checking DNS** ಹಂತದಲ್ಲಿ ಇರುತ್ತದೆ. ಪ್ರಕ್ರಿಯೆಯು DNS ಎಂಟ್ರಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಅವು ಸರಿಯಾಗಿವೆ ಎಂದು ದೃಢೀಕರಿಸುತ್ತದೆ. ನಂತರ, ಡೊಮೈನ್ ಅನ್ನು **Checking SSL** ಹಂತದಲ್ಲಿ ಇಡಲಾಗುತ್ತದೆ. Ultimate Multisite ಡೊಮೈನ್‌ಗೆ SSL ಇದೆಯೇ ಇಲ್ಲವೇ ಎಂದು ಪರಿಶೀಲಿಸುತ್ತದೆ ಮತ್ತು ನಿಮ್ಮ ಡೊಮೈನ್ ಅನ್ನು **Ready** ಅಥವಾ **Ready (without SSL)** ಎಂದು ವರ್ಗೀಕರಿಸುತ್ತದೆ.

**Site:** ಈ ಡೊಮೈನ್‌ನೊಂದಿಗೆ ಸಂಯೋಜಿತವಾಗಿರುವ subdomain. ಮ್ಯಾಪ್ ಮಾಡಿದ ಡೊಮೈನ್ ಈ ನಿರ್ದಿಷ್ಟ ಸೈಟ್‌ನ ವಿಷಯವನ್ನು ತೋರಿಸುತ್ತದೆ.

**Active:** ಡೊಮೈನ್ ಅನ್ನು ಸಕ್ರಿಯ ಅಥವಾ ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲು ಈ ಆಯ್ಕೆಯನ್ನು ಆನ್ ಅಥವಾ ಆಫ್ ಮಾಡಬಹುದು.

**Is Primary Domain?:** ನಿಮ್ಮ ಗ್ರಾಹಕರು ಪ್ರತಿ ಸೈಟ್‌ಗೆ ಒಂದಕ್ಕಿಂತ ಹೆಚ್ಚು ಮ್ಯಾಪ್ ಮಾಡಿದ ಡೊಮೈನ್ ಹೊಂದಬಹುದು. ನಿರ್ದಿಷ್ಟ ಸೈಟ್‌ಗೆ ಇದು ಪ್ರಾಥಮಿಕ ಡೊಮೈನ್ ಆಗಿದೆಯೇ ಎಂದು ಆಯ್ಕೆ ಮಾಡಲು ಈ ಆಯ್ಕೆಯನ್ನು ಬಳಸಿ.

**Is Secure?:** Ultimate Multisite ಡೊಮೈನ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುವ ಮೊದಲು SSL ಪ್ರಮಾಣಪತ್ರ ಇದೆಯೇ ಇಲ್ಲವೇ ಎಂದು ಪರಿಶೀಲಿಸಿದರೂ, SSL ಪ್ರಮಾಣಪತ್ರದೊಂದಿಗೆ ಅಥವಾ ಇಲ್ಲದೆ ಡೊಮೈನ್ ಅನ್ನು ಲೋಡ್ ಮಾಡಲು ನೀವು ಹಸ್ತಚಾಲಿತವಾಗಿ ಆಯ್ಕೆ ಮಾಡಬಹುದು. ವೆಬ್‌ಸೈಟ್‌ಗೆ SSL ಪ್ರಮಾಣಪತ್ರ ಇಲ್ಲದಿದ್ದರೆ ಮತ್ತು ನೀವು SSL ನೊಂದಿಗೆ ಬಲವಂತವಾಗಿ ಲೋಡ್ ಮಾಡಲು ಪ್ರಯತ್ನಿಸಿದರೆ, ಅದು ದೋಷಗಳನ್ನು ನೀಡಬಹುದು ಎಂಬುದನ್ನು ಗಮನಿಸಿ.

### Subsite ಬಳಕೆದಾರರಾಗಿ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಮ್ಯಾಪ್ ಮಾಡುವುದು

Subsite ನಿರ್ವಾಹಕರು ತಮ್ಮ subsite admin dashboard ನಿಂದ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರುಗಳನ್ನು ಸಹ ಮ್ಯಾಪ್ ಮಾಡಬಹುದು.

ಮೊದಲು, **Domain mapping** ಸೆಟ್ಟಿಂಗ್ಸ್ ಅಡಿಯಲ್ಲಿ ಈ ಆಯ್ಕೆಯನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬೇಕು. ಕೆಳಗಿನ ಸ್ಕ್ರೀನ್‌ಶಾಟ್ ನೋಡಿ.

![subsite ಬಳಕೆದಾರರಿಗೆ ಡೊಮೈನ್ ಮ್ಯಾಪ್ ಮಾಡಲು ಅನುಮತಿಸುವ Domain mapping ಸೆಟ್ಟಿಂಗ್ಸ್](/img/config/settings-domain-mapping.png)

**Ultimate Multisite > Products** ನಲ್ಲಿ **Plan** ಮಟ್ಟದಲ್ಲಿ ಅಥವಾ ಉತ್ಪನ್ನ ಆಯ್ಕೆಗಳ ಅಡಿಯಲ್ಲಿ ಈ ಆಯ್ಕೆಯನ್ನು ಸೆಟ್ ಅಥವಾ ಕಾನ್ಫಿಗರ್ ಮಾಡಬಹುದು.

![ಉತ್ಪನ್ನ ಸೆಟ್ಟಿಂಗ್ಸ್‌ನಲ್ಲಿ Product custom domains ಆಯ್ಕೆ](/img/config/settings-domain-mapping.png)

ಆ ಯಾವುದೇ ಆಯ್ಕೆಗಳು ಸಕ್ರಿಯಗೊಳಿಸಲ್ಪಟ್ಟಾಗ ಮತ್ತು subsite ಬಳಕೆದಾರರಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರುಗಳನ್ನು ಮ್ಯಾಪ್ ಮಾಡಲು ಅನುಮತಿಸಿದಾಗ, subsite ಬಳಕೆದಾರರಿಗೆ **Account** ಪುಟದ ಅಡಿಯಲ್ಲಿ **Domains** ಎಂಬ metabox ಕಾಣಿಸಬೇಕು.

![subsite account ಪುಟದಲ್ಲಿ Domains metabox](/img/admin/domains-list.png)

ಬಳಕೆದಾರರು **Add Domain** ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಬಹುದು ಮತ್ತು ಅದು ಕೆಲವು ಸೂಚನೆಗಳೊಂದಿಗೆ modal ವಿಂಡೋ ತರುತ್ತದೆ.

![subsite ಬಳಕೆದಾರರಿಗೆ DNS ಸೂಚನೆಗಳೊಂದಿಗೆ Add Domain modal](/img/admin/domains-list.png)

ಬಳಕೆದಾರರು ನಂತರ **Next Step** ಕ್ಲಿಕ್ ಮಾಡಿ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಸೇರಿಸಲು ಮುಂದುವರಿಯಬಹುದು. ಇದು ಪ್ರಾಥಮಿಕ ಡೊಮೈನ್ ಆಗಿರಬೇಕೇ ಎಂದು ಸಹ ಆಯ್ಕೆ ಮಾಡಬಹುದು.

![primary domain ಆಯ್ಕೆಯೊಂದಿಗೆ ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರು ಫೀಲ್ಡ್ ಸೇರಿಸಿ](/img/admin/domains-list.png)

![DNS ಪರಿಶೀಲನೆ ಪ್ರಾರಂಭಿಸಲು Add Domain ಕ್ಲಿಕ್ ಮಾಡಿ](/img/admin/domains-list.png)**Add Domain** ಕ್ಲಿಕ್ ಮಾಡುವುದರಿಂದ ಕಸ್ಟಮ್ ಡೊಮೈನ್‌ನ DNS ಮಾಹಿತಿಯನ್ನು ಪರಿಶೀಲಿಸುವ ಮತ್ತು ಪಡೆಯುವ ಪ್ರಕ್ರಿಯೆ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ.

### Domain Syncing ಬಗ್ಗೆ

Domain Syncing ಎಂದರೆ **ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ಕೆಲಸ ಮಾಡಲು** Ultimate Multisite ಕಸ್ಟಮ್ ಡೊಮೈನ್ ಹೆಸರನ್ನು ನಿಮ್ಮ ಹೋಸ್ಟಿಂಗ್ ಖಾತೆಗೆ add-on domain ಆಗಿ ಸೇರಿಸುವ ಪ್ರಕ್ರಿಯೆ.

ನಿಮ್ಮ ಹೋಸ್ಟಿಂಗ್ ಪ್ರೊವೈಡರ್ Ultimate Multisite ಡೊಮೈನ್ ಮ್ಯಾಪಿಂಗ್ ವೈಶಿಷ್ಟ್ಯದೊಂದಿಗೆ ಇಂಟಿಗ್ರೇಶನ್ ಹೊಂದಿದ್ದರೆ Domain syncing ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಡೆಯುತ್ತದೆ. ಪ್ರಸ್ತುತ, ಈ ಹೋಸ್ಟಿಂಗ್ ಪ್ರೊವೈಡರ್‌ಗಳು _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ ಮತ್ತು _Cpanel._

**Integration** ಟ್ಯಾಬ್ ಅಡಿಯಲ್ಲಿ Ultimate Multisite ಸೆಟ್ಟಿಂಗ್ಸ್‌ನಲ್ಲಿ ಈ ಇಂಟಿಗ್ರೇಶನ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಬೇಕು.

![Ultimate Multisite ಸೆಟ್ಟಿಂಗ್ಸ್‌ನಲ್ಲಿ Integration ಟ್ಯಾಬ್](/img/config/settings-domain-mapping.png)

![domain syncing ಗಾಗಿ ಹೋಸ್ಟಿಂಗ್ ಪ್ರೊವೈಡರ್ integration ಸೆಟ್ಟಿಂಗ್ಸ್](/img/config/settings-domain-mapping.png)

_ನಿಮ್ಮ ಹೋಸ್ಟಿಂಗ್ ಪ್ರೊವೈಡರ್ ಮೇಲೆ ಹೇಳಿದ ಪ್ರೊವೈಡರ್‌ಗಳಲ್ಲಿ ಒಂದಾಗಿಲ್ಲದಿದ್ದರೆ, **ನಿಮ್ಮ ಹೋಸ್ಟಿಂಗ್ ಖಾತೆಗೆ ಡೊಮೈನ್ ಹೆಸರನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ sync ಅಥವಾ ಸೇರಿಸಬೇಕಾಗುತ್ತದೆ** ಎಂಬುದನ್ನು ಗಮನಿಸಿ._
