# Ultimate Multisite ಅವಶ್ಯಕತೆಗಳು

Ultimate Multisite ಖರೀದಿಸುವ ಮೊದಲು ನೀವು ಕೇಳಿಕೊಳ್ಳಬಹುದಾದ ಮೊದಲ ಪ್ರಶ್ನೆಯೆಂದರೆ ನಮ್ಮ plugin ಚಲಾಯಿಸಲು ಏನು ಅವಶ್ಯಕತೆಗಳಿವೆ ಎಂದು. ಅದೃಷ್ಟವಶಾತ್, Ultimate Multisite ಚಲಾಯಿಸಲು ಕೆಲವು ಮೂಲಭೂತ ಅವಶ್ಯಕತೆಗಳು ಮಾತ್ರ ಬೇಕಾಗುತ್ತವೆ: ಒಂದು **domain name**, ಉತ್ತಮ **hosting provider** ಮತ್ತು WordPress **Multisite** installation. ಇಷ್ಟೇ ನಿಮಗೆ ಬೇಕಾಗಿರುವುದು!

## Domain Name

ಮೊದಲು, domain name ಬಗ್ಗೆ ಮಾತನಾಡೋಣ.

Domain name ಆಯ್ಕೆ ಮಾಡುವಾಗ, ನಿಮ್ಮ ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರ ಯಾವುದು ಮತ್ತು Ultimate Multisite ಬಳಸಿ ನೀವು ಏನು ರಚಿಸಲು ಹೊರಟಿದ್ದೀರಿ ಎಂಬುದನ್ನು ಮನಸ್ಸಿನಲ್ಲಿಟ್ಟುಕೊಳ್ಳಿ: ನೀವು agency ರಚಿಸಲು ಬಯಸುತ್ತೀರಾ? ಅಂಗಡಿ ಮಾದರಿಯ ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ಮಾರಾಟ ಮಾಡಬೇಕಾ? ಅಥವಾ WordPress web hosting? Ultimate Multisite ನೊಂದಿಗೆ ಎಲ್ಲವೂ ಸಾಧ್ಯ.

ನಮ್ಮ plugin subscription ಸೇವೆಗಳು, ಪಾವತಿ ಸಂಗ್ರಹಣೆ, checkout forms, discount vouchers ಮತ್ತು ಗ್ರಾಹಕ ಸಂವಹನಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. WordPress Multisite ಅನ್ನು install ಮಾಡಲು, configure ಮಾಡಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಅಗತ್ಯವಾದ ಹೆಚ್ಚಿನ ಕೆಲಸವನ್ನು Ultimate Multisite ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

ನೀವು ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ಸೇವೆಯಾಗಿ ಮಾರಾಟ ಮಾಡುವವರೆಗೆ, ನೀವು ಆಯ್ಕೆ ಮಾಡಿಕೊಳ್ಳುವ ಯಾವುದೇ ಕ್ಷೇತ್ರವನ್ನು ನಮ್ಮ plugin ಬಳಸಿ ನಿರ್ವಹಿಸಬಹುದು! ನಿಮ್ಮ ವ್ಯಾಪಾರ ಯೋಜನೆಗೆ ಹೊಂದಿಕೊಳ್ಳುವ domain name ಆಯ್ಕೆ ಮಾಡಿಕೊಳ್ಳಿ.

## Hosting Provider

ಈಗ ನಿಮ್ಮ ಕ್ಷೇತ್ರ ಮತ್ತು ವ್ಯಾಪಾರವನ್ನು ಪ್ರತಿನಿಧಿಸುವ ಆಕರ್ಷಕ domain name ಇದೆ, Ultimate Multisite ನೊಂದಿಗೆ ನಿಮ್ಮ network ಅನ್ನು host ಮಾಡಲು ಉತ್ತಮ web hosting ಕಂಪನಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡಬೇಕಾಗುತ್ತದೆ. ಮಾರುಕಟ್ಟೆಯಲ್ಲಿ ಕೆಲವು ಉತ್ತಮ ಆಯ್ಕೆಗಳಿವೆ. ಕೆಲವು hosting providers Ultimate Multisite ನೊಂದಿಗೆ ಚೆನ್ನಾಗಿ ಸಂಯೋಜಿಸುತ್ತವೆ, ನೀವು ಏನು ನಿರ್ಮಿಸಲು ಯೋಜಿಸುತ್ತಿದ್ದೀರಿ ಎಂಬುದರ ಆಧಾರದ ಮೇಲೆ ಇದು ನಿರ್ಣಾಯಕ ಅಂಶವಾಗಬಹುದು.

Ultimate Multisite ಯಾವುದೇ hosting provider ನಲ್ಲಿ ಕೆಲಸ ಮಾಡಬೇಕು. ಒಂದೇ ವ್ಯತ್ಯಾಸವೆಂದರೆ _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ ಮತ್ತು _Runcloud_ ನಂತಹ ಕೆಲವು hosting ಕಂಪನಿಗಳು domain mapping ಗೆ ಹೆಚ್ಚು ಬಿಗಿಯಾದ integration ಹೊಂದಿವೆ. ಇತರ hosting ಗಳಿಗೆ, ನೀವು custom domains ಅನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ map ಮಾಡಬೇಕಾಗಬಹುದು. AutoSSL ಬೆಂಬಲಿಸುವ hosting provider ಪಡೆಯುವುದು ಸಹ ಒಳ್ಳೆಯ ಯೋಚನೆ. _Closte_, _Runcloud_ ಮತ್ತು _cPanel_ ಇದನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆ.

Server specifications ವಿಷಯಕ್ಕೆ ಬಂದಾಗ, ನೀವು ಒದಗಿಸುವ features ಮತ್ತು functions ಹಾಗೂ ನಿಮ್ಮ network ನ ಗಾತ್ರವನ್ನು ಅವಲಂಬಿಸಿರುವುದರಿಂದ ನಿರ್ದಿಷ್ಟ ಸೂತ್ರವಿಲ್ಲ. Shared hosting ಉತ್ತಮ ಆರಂಭವಾಗಬಹುದು, ಆದರೆ ನಿಮ್ಮ network ಬೆಳೆಯಲು ಪ್ರಾರಂಭಿಸಿದ ತಕ್ಷಣ, ನಿಮಗೆ ಬಹುಶಃ ಮೀಸಲಾದ VPS ಬೇಕಾಗುತ್ತದೆ.

## WordPress Multisite

ಈಗ, WordPress Multisite installation ಬಗ್ಗೆ ಸ್ವಲ್ಪ ಮಾತನಾಡೋಣ.

WordPress Multisite ಒಂದೇ installation ನಲ್ಲಿ ಸೈಟ್‌ಗಳ network ಹೊಂದಲು ನಿಮಗೆ ಅನುಮತಿಸುತ್ತದೆ. ಇದು built-in feature ಆಗಿದೆ, ಆದರೆ ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸಕ್ರಿಯವಾಗಿರುವುದಿಲ್ಲ. ನಿಮ್ಮ WordPress installation ಮತ್ತು web hosting control panel ಒಳಗೆ ಈ feature ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಬೇಕು.

Multisite ಸಕ್ರಿಯಗೊಳಿಸಿದ WordPress **Ultimate Multisite ಗೆ ಆಧಾರವಾಗಿದೆ**. Ultimate Multisite ನ ಮುಖ್ಯ ಬಳಕೆಯು ಬಳಕೆದಾರರಿಗೆ ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ಸೇವೆಯಾಗಿ ನಿರ್ವಹಿಸಲು ಮತ್ತು ಮಾರಾಟ ಮಾಡಲು ಸಹಾಯ ಮಾಡುವುದಾಗಿರುವುದರಿಂದ, ಇದನ್ನು Multisite ಸಕ್ರಿಯಗೊಳಿಸಿದ WordPress ನಲ್ಲಿ ಮಾಡಬೇಕಾಗುತ್ತದೆ.

ನಿಮ್ಮ WordPress installation ನಲ್ಲಿ Multisite feature ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ಏನು ಮಾಡಬೇಕು ಎಂಬುದನ್ನು ಹಂತ ಹಂತವಾಗಿ ಕಲಿಸುವ **ವಿಸ್ತೃತ ಲೇಖನ** ನಮ್ಮಲ್ಲಿದೆ.

WordPress ನಲ್ಲಿ Multisite ಸಕ್ರಿಯಗೊಳಿಸಿದ ನಂತರ, Ultimate Multisite install ಮಾಡುವ ಸಮಯ ಬಂದಿದೆ. Ultimate Multisite ಅನ್ನು ಸರಿಯಾಗಿ install ಮಾಡಲು, ನಿಮ್ಮಲ್ಲಿ ಇರಬೇಕು:

  * WordPress: v5.3+ (ಶಿಫಾರಸು: ಕೊನೆಯ ಸ್ಥಿರ ಆವೃತ್ತಿ)

  * PHP: 7.4.x (> 8.0 ಬೆಂಬಲ ಶೀಘ್ರದಲ್ಲಿ ಬರಲಿದೆ)

  * MySQL: v5+ (ಶಿಫಾರಸು: 5.6, ನಿಮ್ಮ hosting provider ನಲ್ಲಿ 8.0 ಲಭ್ಯವಿಲ್ಲದಿದ್ದರೆ)

Ultimate Multisite ಗೆ ಇವು ಮಾತ್ರ software ಅವಶ್ಯಕತೆಗಳು.

Ultimate Multisite PHP 8.0 ನೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಬಹುದು ಆದರೆ PHP 7.4.x ನೊಂದಿಗೆ ಚಲಾಯಿಸಲು ನಾವು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ ಎಂಬುದನ್ನು ಮನಸ್ಸಿನಲ್ಲಿಟ್ಟುಕೊಳ್ಳಿ.

ಅಲ್ಲದೆ, ಇದನ್ನು **ಮುಖ್ಯ domains** ನಲ್ಲಿ install ಮಾಡಲು ನಾವು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ, subdomains ಅಥವಾ subdirectories ನಲ್ಲಿ ಅಲ್ಲ. ಇದು subdomains/subdirectories ನಲ್ಲಿ ಕೆಲಸ ಮಾಡಬಹುದು, ಆದರೆ ಕೆಲವು ದೋಷಗಳನ್ನು ತೋರಿಸಬಹುದು.

Ultimate Multisite ಅನ್ನು install ಮಾಡುವ ವಿಸ್ತೃತ ಮಾರ್ಗದರ್ಶಿಗಾಗಿ, **ಇಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡಿ**.
