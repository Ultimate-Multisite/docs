# ಸಾರ್ವಭೌಮ ವಲಿಕಾ ಕಾರ್ಯವಿಧಾನ (Sovereign Migration Workflow)

Ultimate Multisite: Multi-Tenancy 1.2.0 ನಲ್ಲಿ ಸ್ಟ್ಯಾಂಡರ್ಡ್サブಸೈಟ್ ಅನ್ನು ಸಾರ್ವಭೌಮ ಟೆನೆಂಟ್ ಮೂಲಸೌಕರ್ಯಕ್ಕೆ ವರ್ಗಾಯಿಸುವಾಗ ಮೈಗ್ರೇಷನ್ ಪರಿಶೀಲನಾ ಗೇಟ್‌ಗಳನ್ನು (migration verification gates) ಸೇರಿಸಲಾಗಿದೆ.

## ನೀವು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು {#before-you-start}

ನಿಮ್ಮ ಬಳಿ ಈ ಕೆಳಗಿನವು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

- ಮೂಲ ಸೈಟ್‌ನ ಪ್ರಸ್ತುತ ಬ್ಯಾಕಪ್.
- Multi-Tenancy add-on ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿದೆ.
- ಗಮ್ಯdestino ಟೆನೆಂಟ್ ಡೇಟಾಬೇಸ್, ಫೈಲ್ಸಿಸ್ಟಮ್ ರೂಟ್ ಮತ್ತು ಡೊಮೇನ್ ಸಿದ್ಧವಾಗಿದೆ.
- ಗಮ್ಯdestino ಪರಿಸರಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗುವ ಡೇಟಾಬೇಸ್ ಹೋಸ್ಟ್ ಬೈಂಡಿಂಗ್ (database host binding).
- ನೆಟ್‌ವರ್ಕ್‌ನ WP-CLI ಕಮಾಂಡ್‌ಗಳನ್ನು ಚಲಾಯಿಸಲು ಪ್ರವೇಶ.

## ಶಿಫಾರಸು ಮಾಡಿದ ಕಾರ್ಯವಿಧಾನ {#recommended-workflow}

1. ಗಮ್ಯdestino ಟೆನೆಂಟ್ ಡೇಟಾಬೇಸ್ ಮತ್ತು ಫೈಲ್ಸಿಸ್ಟಮ್ ಅನ್ನು ಸಿದ್ಧಪಡಿಸಿ.
2. ಟೆನೆಂಟ್ ಐಸலேஷன் ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ನೋಂದಾಯಿಸಿ ಅಥವಾ ಅಪ್ಡೇಟ್ ಮಾಡಿ.
3. ಟೆನೆಂಟ್ ಮೈಗ್ರೇಷನ್ (tenant migration) ಅನ್ನು இயக்கவும்.
4. ಟೆನೆಂಟ್ ಬಳಕೆದಾರರನ್ನು ಪ್ರವೇಶಿಸಿ ಅಥವಾ ಪರಿಶೀಲಿಸಿ.
5. ಅಸಿನ್ಕ್ರೋನಸ್ ಮೈಗ್ರೇಷನ್ ಜಾಬ್‌ಗಳನ್ನು ಖಾಲಿ ಮಾಡಿ (drain).
6. ಸಾರ್ವಭೌಮ ಮೈಗ್ರೇಷನ್ ಪರಿಶೀಲನೆಯನ್ನು இயக்கவும்.
7. SSO (Single Sign-On) ಬಳಸಿ ಟೆನೆಂಟ್ ಅನ್ನು ಭೇಟಿ ಮಾಡಿ.
8. ಪರಿಶೀಲನೆ ಯಶಸ್ವಿಯಾದ ನಂತರ ಮಾತ್ರ DNS ಅಥವಾ ರೌಟಿಂಗ್ ಅನ್ನು ಬದಲಾಯಿಸಿ.

## ಪರಿಶೀಲನಾ ಗೇಟ್‌ಗಳು (Verification gates) {#verification-gates}

ಪರಿಶೀಲನಾ ಕಾರ್ಯವಿಧಾನವು ಹಲವಾರು ಕೋನಗಳಿಂದ ಮೈಗ್ರೇಷನ್ ಅನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ:

- ಟೆನೆಂಟ್ ಸ್ಕೀಮಾ (schema) ಅಸ್ತಿತ್ವದಲ್ಲಿರುತ್ತದೆ ಮತ್ತು ಟೆನೆಂಟ್ ಡೇಟಾಬೇಸ್ ರೈಟರ್‌ನಿಂದ ಬರೆಯಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.
- ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಡೇಟಾಬೇಸ್ ಹೋಸ್ಟ್‌ನಲ್ಲಿ ಮಾನ್ಯವಾದ ಅನುಮತಿಗಳು (grants) ಇರುತ್ತವೆ.
- ಟೆನೆಂಟ್ ಬಳಕೆದಾರರು ಪ್ರಸ್ತುತವಿರುತ್ತಾರೆ ಮತ್ತು ನಿರೀಕ್ಷಿತ ಸಾರ್ವಭೌಮ ಇನ್‌ಸ್ಟಾಲ್ ಬಳಸುವವರ ಸಂಖ್ಯೆಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತವೆ.
- ಅಸಿನ್ಕ್ರೋನಸ್ ಪುಶ್ ಕ್ಯೂ (async push queue) ಯಶಸ್ವಿಯಾಗಿ ಖಾಲಿಯಾಗಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.
- ಸಾರ್ವಭೌಮ ಟೆನೆಂಟ್‌ಗೆ ಹಳೆಯ ನೆಟ್‌ವರ್ಕ್-ಪಕ್ಕದ ಡೇಟಾ ಪಥಗಳು (Legacy network-side data paths) ಅಗತ್ಯವಿಲ್ಲ.

ಪರಿಶೀಲನಾ ವೈಫಲ್ಯಗಳನ್ನು ಪ್ರಾರಂಭಕ್ಕೆ ತಡೆಯುವ ಅಂಶಗಳಾಗಿ ಪರಿಗಣಿಸಿ. ವರದಿಯಾದ ಡೇಟಾಬೇಸ್, ಬಳಕೆದಾರ, ಕ್ಯೂ ಅಥವಾ ರೌಟಿಂಗ್ ಸಮಸ್ಯೆಯನ್ನು ಸರಿಪಡಿಸಿದ ನಂತರ, ಟೆನೆಂಟ್ ಅನ್ನು ಗ್ರಾಹಕರಿಗೆ ಪ್ರದರ್ಶಿಸುವ ಮೊದಲು ಮತ್ತೊಮ್ಮೆ ಪರಿಶೀಲನೆಯನ್ನು இயக்கவும்.

## ಮೊದಲ ಉತ್ಪಾದನಾ ಭೇಟಿ (First production visit) {#first-production-visit}

ಪರಿಶೀಲನೆ ಯಶಸ್ವಿಯಾದ ನಂತರ, ಮೊದಲ ადಮಿನ ಭೇಟಿಗೆ ಸೈಟ್ ಮ್ಯಾನೇಜ್‌ಮೆಂಟ್ ಪರದೆಯಿಂದ **Visit (SSO)** ಅನ್ನು ಬಳಸಿ. ಇದು ಟೆನೆಂಟ್ ರೌಟಿಂಗ್, SSO ಟೋಕನ್ ನಿರ್ವಹಣೆ, ಮೂಲ ಪ pinpointing ಮತ್ತು ಟೆನೆಂಟ್-ಪಕ್ಕದ ಬಳಕೆದಾರ ಪ್ರವೇಶವನ್ನು ಒಂದು ನಿಯಂತ್ರಿತ ಹಂತದಲ್ಲಿ ಖಚಿತಪಡಿಸುತ್ತದೆ.
