# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ಗೆ ಹಲವು ಕರೆನ್ಸಿಗಳಲ್ಲಿ ಪಾವತಿಗಳನ್ನು ಸ್ವೀಕರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಇದು ಲೈವ್ ಅಥವಾ ಕೈಯಾರೆ ವಿನಿಮಯ ದರಗಳನ್ನು ಬಳಸಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬೆಲೆಗಳನ್ನು ಪರಿವರ್ತಿಸುತ್ತದೆ, ಯಾವುದೇ ಕರೆನ್ಸಿಗಾಗಿ ಉತ್ಪನ್ನದ ನಿರ್ದಿಷ್ಟ ಬೆಲೆಗಳನ್ನು ನಿಗದಿಪಡಿಸಲು ಅನುಮತಿಸುತ್ತದೆ ಮತ್ತು ಜಿಯೋಲೋಕೇಶನ್ ಮೂಲಕ ಪ್ರತಿಯೊಬ್ಬ ಸಂದರ್ಶಕನ ಆದ್ಯತೆಯ ಕರೆನ್ಸಿಯನ್ನು ಪತ್ತೆಹಚ್ಚುತ್ತದೆ — ಇವೆಲ್ಲವನ್ನೂ ಒಂದೇ ಆಡಳಿತಾತ್ಮಕ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ನಿಂದ ನಿರ್ವಹಿಸಬಹುದು.

## ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು (Key Features) {#key-features}

- **ಸ್ವಯಂಚಾಲಿತ ವಿನಿಮಯ ದರ ಪರಿವರ್ತನೆ** — ನಿಮ್ಮ ಮೂಲ ಕರೆನ್ಸಿ (base currency) ಯಿಂದ ಬೆಲೆಗಳನ್ನು ನಿಮ್ಮ ಆರಿಸಿದ ಪೂರೈಕೆದಾರರಿಂದ (provider) ಪಡೆದ ದರಗಳನ್ನು ಬಳಸಿ ಪರಿವರ್ತಿಸಲಾಗುತ್ತದೆ.
- **ಉತ್ಪನ್ನದ ಆಧಾರದ ಬೆಲೆ ಮರುನಿರ್ಧಾರಣೆ (Per-product pricing overrides)** — ಸ್ವಯಂಚಾಲಿತ ಪರಿವರ್ತನೆಯನ್ನು ತಪ್ಪಿಸಲು ಯಾವುದೇ ಉತ್ಪನ್ನದ ಮೇಲೆ ಕರೆನ್ಸಿ-ನಿರ್ದಿಷ್ಟ ಬೆಲೆಗಳನ್ನು ನಿಗದಿಪಡಿಸಿ.
- **ಕರೆನ್ಸಿ ಆಯ್ಕೆದಾರರ ನೋಂದಣಿ ಫೀಲ್ಡ್ (Currency selector signup field)** — ಚೆಕ್‌ಔಟ್/ನೋಂದಣಿಯ ಸಮಯದಲ್ಲಿ ಗ್ರಾಹಕರು ತಮ್ಮ ಆದ್ಯತೆಯ ಕರೆನ್ಸಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಇದು ಅನುಮತಿಸುತ್ತದೆ.
- **ಬಹು ವಿನಿಮಯ ದರ ಪೂರೈಕೆದಾರರು** — ECB, Frankfurter, Stripe, ಅಥವಾ ಸಂಪೂರ್ಣವಾಗಿ ಕೈಯಾರೆ ದರಗಳು.
- **ಜಿಯೋಲೋಕೇಶನ್ ಆಧಾರಿತ ಕರೆನ್ಸಿ ಪತ್ತೆಹಚ್ಚುವಿಕೆ** — ಸಂದರ್ಶಕನ ದೇಶಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗುವ ಕರೆನ್ಸಿಯನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪೂರ್ವ-ಆಯ್ಕೆ ಮಾಡುತ್ತದೆ.
- **ಆಡಳಿತಾತ್ಮಕ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ (Admin dashboard)** — ಸಕ್ರಿಯ ಕರೆನ್ಸಿಗಳು, ವಿನಿಮಯ ದರಗಳು ಮತ್ತು ಪೂರೈಕೆದಾರರ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಒಂದೇ ಸ್ಥಳದಿಂದ ನಿರ್ವಹಿಸಿ.

## ಅವಶ್ಯಕತೆಗಳು (Requirements) {#requirements}

- Ultimate Multisite 2.0.0 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದು
- WordPress Multisite installation
- PHP 7.4 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದು

## ಅಳವಡಿಕೆ (Installation) {#installation}

1. ಪ್ಲಗ್ಇನ್ ಫೈಲ್‌ಗಳನ್ನು `/wp-content/plugins/ultimate-multisite-multi-currency` ಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ.
2. WordPress ನಲ್ಲಿರುವ **Plugins** ಪರದೆಯ ಮೂಲಕ ಪ್ಲಗ್ಇನ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ (Network Activate).
3. ಆಡಳಿತಾತ್ಮಕ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ತೆರೆಯಲು **Ultimate Multisite → Multi-Currency** ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ.
4. ನೀವು ಸ್ವೀಕರಿಸಲು ಬಯಸುವ ಕರೆನ್ಸಿಗಳನ್ನು ಸೇರಿಸಿ ಮತ್ತು ನಿಮ್ಮ ವಿನಿಮಯ ದರ ಪೂರೈಕೆದಾರರನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ.

## ಕಾನ್ಫಿಗರೇಶನ್ (Configuration) {#configuration}

### ಆಡಳಿತಾತ್ಮಕ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ (Admin Dashboard) {#admin-dashboard}

Multi-Currency ಆಡಳಿತಾತ್ಮಕ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ (**Ultimate Multisite → Multi-Currency**) ಈ ಕೆಳಗಿನ ಕೆಲಸಗಳಿಗೆ ಕೇಂದ್ರ ಸ್ಥಳವಾಗಿದೆ:

- ಕರೆನ್ಸಿಗಳನ್ನು ಸೇರಿಸಲು, ಸಕ್ರಿಯಗೊಳಿಸಲು ಅಥವಾ ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಲು.
- ವಿನಿಮಯ ದರ ಪೂರೈಕೆದಾರರನ್ನು ಆಯ್ಕೆಮಾಡಿ ಮತ್ತು ಬೇಡಿಕೆಯ ಮೇರೆಗೆ ದರಗಳನ್ನು ರಿಫ್ರೆಶ್ ಮಾಡಲು.
- ಎಲ್ಲಾ ಸಕ್ರಿಯ ಕರೆನ್ಸಿಗಳ ಪ್ರಸ್ತುತ ದರಗಳನ್ನು ಪರಿಶೀಲಿಸಲು.
- ಸ್ವಯಂಚಾಲಿತ ಪೂರೈಕೆದಾರರು ಸೂಕ್ತವಾಗಿಲ್ಲದ ಸಂದರ್ಭದಲ್ಲಿ ಕೈಯಾರೆ ದರಗಳನ್ನು ನಿಗದಿಪಡಿಸಲು.

### ವಿನಿಮಯ ದರ ಪೂರೈಕೆದಾರರು (Exchange Rate Providers) {#exchange-rate-providers}

| Provider | Description |
|---|---|
| **ECB** | European Central Bank — EUR ಆಧಾರಿತ ನೆಟ್‌ವರ್ಕ್‌ಗಳಿಗೆ ಉಚಿತ ದೈನಂದಿನ ದರಗಳು |
| **Frankfurter** | Open-source ECB mirror API — ಉಚಿತ, API ಕೀ ಅಗತ್ಯವಿಲ್ಲ |
| **Stripe** | Stripe ನ ಲೈವ್ FX ದರಗಳನ್ನು ಬಳಸುತ್ತದೆ — ಸಕ್ರಿಯ Stripe integration ಅಗತ್ಯವಿದೆ |
| **Manual** | ನೀವು ನೇರವಾಗಿ ದರಗಳನ್ನು ನಮೂದಿಸುತ್ತೀರಿ; ಯಾವುದೇ ಬಾಹ್ಯ API ಕರೆಗಳು ಅಗತ್ಯವಿಲ್ಲ |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** ಅಡಿಯಲ್ಲಿ ನಿಮ್ಮ ಪೂರೈಕೆದಾರರನ್ನು ಆಯ್ಕೆಮಾಡಿ. ದರಗಳನ್ನು ಯಾವುದೇ ಸಮಯದಲ್ಲಿ ಕೈಯಾರೆ ರಿಫ್ರೆಶ್ ಮಾಡಬಹುದು ಅಥವಾ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅಪ್‌ಡೇಟ್ ಮಾಡಲು ವೇಳಾಪಟ್ಟಿ ಮಾಡಬಹುದು.

### ಉತ್ಪನ್ನದ ಆಧಾರದ ಬೆಲೆ ಮರುನಿರ್ಧಾರಣೆ (Per-Product Pricing Overrides) {#per-product-pricing-overrides}

ಸ್ವಯಂಚಾಲಿತ ಪರಿವರ್ತನೆಯು ಸಾಕಷ್ಟು ನಿಖರವಾಗದಿದ್ದಾಗ, ನೀವು ಯಾವುದೇ ಉತ್ಪನ್ನದ ಮೇಲೆ ನಿರ್ದಿಷ್ಟ ಕರೆನ್ಸಿಯ ನಿಖರ ಬೆಲೆಯನ್ನು ನಿಗದಿಪಡಿಸಬಹುದು:

1. ಉತ್ಪನ್ನ ಸಂಪಾದಕವನ್ನು ತೆರೆಯಿರಿ (**Ultimate Multisite → Products → Edit**)
2. **Currency Pricing** ವಿಭಾಗಕ್ಕೆ ಸ್ಕ್ರಾಲ್ ಮಾಡಿ.
3. ನೀವು ಮರುನಿರ್ಧಾರಿಸಲು ಬಯಸುವ ಪ್ರತಿ ಕರೆನ್ಸಿಗಾಗಿ ನಿಖರ ಬೆಲೆಯನ್ನು ನಮೂದಿಸಿ.
4. ಉತ್ಪನ್ನವನ್ನು ಉಳಿಸಿ (Save).

ಮರುನಿರ್ಧಾರಣೆ ಇಲ್ಲದ ಉತ್ಪನ್ನಗಳು ನಿಮ್ಮ ಮೂಲ ಕರೆನ್ಸಿ (base currency) ಯಿಂದ ಪರಿವರ್ತಿಸಲಾದ ಬೆಲೆಯನ್ನು ಬಳಸುತ್ತವೆ.

### ಕರೆನ್ಸಿ ಆಯ್ಕೆದಾರರ ನೋಂದಣಿ ಫೀಲ್ಡ್ (Currency Selector Signup Field) {#currency-selector-signup-field}

ಗ್ರಾಹಕರು ನೋಂದಣಿಯ ಸಮಯದಲ್ಲಿ ತಮ್ಮ ಆದ್ಯತೆಯ ಕರೆನ್ಸಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಕರೆನ್ಸಿ ಆಯ್ಕೆದಾರರ ಫೀಲ್ಡ್ ಅನ್ನು ನಿಮ್ಮ ನೋಂದಣಿ/ಚೆಕ್‌ಔಟ್ ಫ್ಲೋಗೆ ಸೇರಿಸಬಹುದು:

1. **Ultimate Multisite → Multi-Currency → Settings** ಗೆ ಹೋಗಿ.
2. **Show currency selector on signup** ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ.
3. ಐಚ್ಛಿಕವಾಗಿ ಒಂದು ಡೀಫಾಲ್ಟ್ ಕರೆನ್ಸಿಯನ್ನು ಹೊಂದಿಸಿ ಅಥವಾ ಜಿಯೋಲೋಕೇಶನ್ ಅನ್ನು ಪೂರ್ವ-ಆಯ್ಕೆ ಮೌಲ್ಯವನ್ನು ನಿರ್ಧರಿಸಲು ಬಿಡಿ.

ಆಯ್ಕೆ ಮಾಡಿದ ಕರೆನ್ಸಿಯನ್ನು ಚಂದಾದಾರಿಕೆಯೊಂದಿಗೆ (subscription) ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಎಲ್ಲಾ ನಂತರದ ಬಿಲ್ಲಿಂಗ್‌ಗೆ ಬಳಸಲಾಗುತ್ತದೆ.

### ಜಿಯೋಲೋಕೇಶನ್ ಆಧಾರಿತ ಕರೆನ್ಸಿ ಪತ್ತೆಹಚ್ಚುವಿಕೆ (Geolocation-Based Currency Detection) {#geolocation-based-currency-detection}

ಜಿಯೋಲೋಕೇಶನ್ ಪತ್ತೆಹಚ್ಚುವಿಕೆ ಸಕ್ರಿಯಗೊಂಡಾಗ, ಈ add-on ಸಂದರ್ಶಕನ ದೇಶವನ್ನು ಕರೆನ್ಸಿಗೆ ಮ್ಯಾಪ್ ಮಾಡಿ ಮತ್ತು ಕರೆನ್ಸಿ ಆಯ್ಕೆದಾರರ ಫೀಲ್ಡ್‌ನಲ್ಲಿ ಅದನ್ನು ಪೂರ್ವ-ಆಯ್ಕೆ ಮಾಡುತ್ತದೆ. ಇದನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು:

1. **Ultimate Multisite → Multi-Currency → Settings** ಗೆ ಹೋಗಿ.
2. **Detect currency from visitor location** ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ.
3. **Geolocation Mapping** ಟೇಬಲ್‌ನಲ್ಲಿ ದೇಶಗಳನ್ನು ಕರೆನ್ಸಿಗಳಿಗೆ ಮ್ಯಾಪ್ ಮಾಡಿ.

ಸಂದರ್ಶಕನ ದೇಶಕ್ಕೆ ಯಾವುದೇ ಮ್ಯಾಪಿಂಗ್ ಇಲ್ಲದಿದ್ದರೆ, ನೆಟ್‌ವರ್ಕ್‌ನ ಮೂಲ ಕರೆನ್ಸಿಯನ್ನು ಬದಲಿ (fallback) ಯಾಗಿ ಬಳಸಲಾಗುತ್ತದೆ.

## ಆಗಾಗ್ಗೆ ಕೇಳಲಾಗುವ ಪ್ರಶ್ನೆಗಳು (Frequently Asked Questions) {#frequently-asked-questions}

### ವಿನಿಮಯ ದರ ಪೂರೈಕೆದಾರ ಲಭ್ಯವಿಲ್ಲದಿದ್ದರೆ ಏನಾಗುತ್ತದೆ? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

add-on ಕೊನೆಯದಾಗಿ ಯಶಸ್ವಿಯಾಗಿ ಪಡೆದ ದರಗಳಿಗೆ ಬದಲಿ (fallback) ನೀಡುತ್ತದೆ. ಯಾವುದೇ ದರಗಳನ್ನು ಎಂದಿಗೂ ಪಡೆಯದಿದ್ದರೆ, ಇದು ನೀವು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಕೈಯಾರೆ ದರಗಳಿಗೆ ಬದಲಿ ನೀಡುತ್ತದೆ. ಸುರಕ್ಷತಾ ಜಾಲವಾಗಿ ಕೈಯಾರೆ ದರಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡುವುದು ಸೂಕ್ತವಾಗಿದೆ.

### ನಾನು ವಿಭಿನ್ನ ಕರೆನ್ಸಿಗಳಿಗೆ ವಿಭಿನ್ನ ಪೂರೈಕೆದಾರರನ್ನು ಬಳಸಬಹುದೇ? {#can-i-use-different-providers-for-different-currencies}

ಇಲ್ಲ — ಎಲ್ಲಾ ಕರೆನ್ಸಿಗಳಿಗೆ ಒಂದೇ ಪೂರೈಕೆದಾರರು ಸಕ್ರಿಯರಾಗಿರುತ್ತಾರೆ. 개개 ವರಗಳ ಮೇಲೆ ಸಂಪೂರ್ಣ ನಿಯಂತ್ರಣ ಬೇಕಾದರೆ **Manual** ಬಳಸಿ.

### ಜಿಯೋಲೋಕೇಶನ್‌ಗೆ ಪ್ರತ್ಯೇಕ ಪ್ಲಗ್ಇನ್ ಬೇಕೇ? {#does-geolocation-require-a-separate-plugin}

ಬಾಹ್ಯ ಜಿಯೋಲೋಕೇಶನ್ ಪ್ಲಗ್ಇನ್ ಅಗತ್ಯವಿಲ್ಲ. add-on WordPress ನ ಅಂತರ್ನಿರ್ಮಿತ IP-to-country ಪತ್ತೆಹಚ್ಚುವಿಕೆಯನ್ನು ಬಳಸುತ್ತದೆ. ಹೆಚ್ಚಿನ ನಿಖರತೆಗಾಗಿ, ಒದಗಿಸಲಾದ ಫಿಲ್ಟರ್ ಹುಕ್‌ಗಳ ಮೂಲಕ ಮೀಸಲಾದ ಜಿಯೋಲೋಕೇಶನ್ ಸೇವೆಯನ್ನು ಸಂಯೋಜಿಸಬಹುದು.

### ಉತ್ಪನ್ನದ ಮರುನಿರ್ಧಾರಣೆಗಳು ತೆರಿಗೆಗೆ ಮೊದಲು ಅಥವಾ ನಂತರ ಅನ್ವಯಿಸುತ್ತವೆಯೇ? {#are-per-product-overrides-applied-before-or-after-tax}

ಉತ್ಪನ್ನದ ಮರುನಿರ್ಧಾರಣೆಗಳು ಗುರಿ ಕರೆನ್ಸಿಯಲ್ಲಿ ತೆರಿಗೆಗೆ ಮೊದಲು ಬೆಲೆಯನ್ನು ನಿಗದಿಪಡಿಸುತ್ತವೆ. ನಿಮ್ಮ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ತೆರಿಗೆ ಕಾನ್ಫಿಗರೇಶನ್ ಬಳಸಿ ಮರುನಿರ್ಧಾರಿತ ಬೆಲೆಯ ಮೇಲೆ ತೆರಿಗೆಯನ್ನು ಲೆಕ್ಕಹಾಕಲಾಗುತ್ತದೆ.
