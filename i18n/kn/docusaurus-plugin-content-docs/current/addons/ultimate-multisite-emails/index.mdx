# ಇಮೇಲ್‌ಗಳು

Ultimate Multisite: Emails addon ನೆಟ್‌ವರ್ಕ್ ಮಾಲೀಕರಿಗೆ ಸೈಟ್‌ಗಳು ಮತ್ತು ಸದಸ್ಯತ್ವಗಳ ಜೊತೆಗೆ ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ಒದಗಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಆವೃತ್ತಿ 1.1.0 ಗ್ರಾಹಕರಿಗಾಗಿ ಇಮೇಲ್ ಖಾತೆ ನಿರ್ವಹಣೆ, ವೆಬ್‌ಮೇಲ್ ಸಿಂಗಲ್ ಸೈನ್-ಆನ್, CyberPanel ಫಾರ್ವರ್ಡರ್ ಬೆಂಬಲ, ಮತ್ತು checkout ಸಮಯದ ಒದಗಿಸುವಿಕೆ ಸುಧಾರಣೆಗಳನ್ನು ಸೇರಿಸುತ್ತದೆ.

## ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು

* **ಗ್ರಾಹಕ ಇಮೇಲ್ ಖಾತೆ ವಿಜೆಟ್**: ಸಕ್ರಿಯ ಇಮೇಲ್ ಸದಸ್ಯತ್ವ ಹೊಂದಿದ ನಂತರ ಗ್ರಾಹಕರು ತಮ್ಮ ಖಾತೆ ಪುಟದಿಂದ ಮೇಲ್‌ಬಾಕ್ಸ್‌ಗಳನ್ನು ರಚಿಸಬಹುದು ಮತ್ತು ನಿರ್ವಹಿಸಬಹುದು.
* **ವೆಬ್‌ಮೇಲ್ SSO ಮತ್ತು ಮ್ಯಾಜಿಕ್ ಲಿಂಕ್‌ಗಳು**: ಪೂರೈಕೆದಾರರು ಬೆಂಬಲಿಸಿದಾಗ, ಗ್ರಾಹಕರು ಒಂದು-ಕ್ಲಿಕ್ ದೃಢೀಕರಣದೊಂದಿಗೆ ಖಾತೆ ಪುಟದಿಂದ ನೇರವಾಗಿ ವೆಬ್‌ಮೇಲ್ ತೆರೆಯಬಹುದು.
* **ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾವಣೆಗಳು**: ಗ್ರಾಹಕರು ಮತ್ತು ನಿರ್ವಾಹಕರು ಇಮೇಲ್ ನಿರ್ವಹಣೆ ವಿಜೆಟ್‌ನಿಂದ ಮೇಲ್‌ಬಾಕ್ಸ್ ಪಾಸ್‌ವರ್ಡ್‌ಗಳನ್ನು ನವೀಕರಿಸಬಹುದು.
* **Checkout ಅರ್ಹತೆ ಒದಗಿಸುವಿಕೆ**: Checkout ಇಮೇಲ್ ಸದಸ್ಯತ್ವವನ್ನು ರಚಿಸುತ್ತದೆ, ಅದನ್ನು ಗ್ರಾಹಕರ ಸೈಟ್ ಡೊಮೇನ್‌ಗೆ ಜೋಡಿಸುತ್ತದೆ, ಮತ್ತು ಖಾತೆ-ಪುಟದ ಮೇಲ್‌ಬಾಕ್ಸ್ ಸಾಧನಗಳನ್ನು ಲಭ್ಯವಾಗಿಸುತ್ತದೆ.
* **CyberPanel ಫಾರ್ವರ್ಡರ್‌ಗಳು**: CyberPanel ಆಧಾರಿತ ನೆಟ್‌ವರ್ಕ್‌ಗಳು ಇಮೇಲ್ ಸೇವಾ ಹರಿವಿನ ಭಾಗವಾಗಿ ಇಮೇಲ್ ಫಾರ್ವರ್ಡರ್‌ಗಳನ್ನು ರಚಿಸಿ ನಿರ್ವಹಿಸಬಹುದು.
* **ಪೂರೈಕೆದಾರ-ಅರಿವುಳ್ಳ ಸೆಟ್ಟಿಂಗ್‌ಗಳು**: ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಪುಟವು ಪೂರೈಕೆದಾರರನ್ನು ಸಕ್ರಿಯ, ಸಂರಚಿಸಲಾದ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳಿಗೆ ಫಿಲ್ಟರ್ ಮಾಡುತ್ತದೆ. ಹೀಗಾಗಿ ನಿರ್ವಾಹಕರು ಬಳಸಬಹುದಾದ ಆಯ್ಕೆಗಳನ್ನು ಮಾತ್ರ ನೋಡುತ್ತಾರೆ.

## ಅಗತ್ಯಗಳು

* Ultimate Multisite 2.0.0 ಅಥವಾ ಹೆಚ್ಚಿನದು
* WordPress Multisite ಸ್ಥಾಪನೆ
* ಕನಿಷ್ಠ ಒಂದು ಸಂರಚಿಸಲಾದ ಇಮೇಲ್ ಪೂರೈಕೆದಾರ ಇಂಟಿಗ್ರೇಷನ್
* ಮೇಲ್‌ಬಾಕ್ಸ್‌ಗಳನ್ನು ರಚಿಸಲು, ಪಾಸ್‌ವರ್ಡ್‌ಗಳನ್ನು ಬದಲಿಸಲು, ಮತ್ತು ಬೆಂಬಲಿತಲ್ಲಿ ವೆಬ್‌ಮೇಲ್ ಪ್ರವೇಶ ಲಿಂಕ್‌ಗಳನ್ನು ನೀಡಲು ಅನುಮತಿಗಳಿರುವ ಪೂರೈಕೆದಾರ API ರುಜುವಾತುಗಳು

## ಸ್ಥಾಪನೆ

1. plugin ಫೈಲ್‌ಗಳನ್ನು `/wp-content/plugins/ultimate-multisite-emails` ಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ.
2. WordPress ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕದಲ್ಲಿನ **Plugins** ನಿಂದ addon ಅನ್ನು ನೆಟ್‌ವರ್ಕ್ ಸಕ್ರಿಯಗೊಳಿಸಿ.
3. **Ultimate Multisite** > **Settings** ಗೆ ಹೋಗಿ addon ಇಮೇಲ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ತೆರೆಯಿರಿ.
4. ಗ್ರಾಹಕರು ಬಳಸಬೇಕೆಂದು ನೀವು ಬಯಸುವ ಇಮೇಲ್ ಪೂರೈಕೆದಾರ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ.
5. ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಉಳಿಸಿ, ನಂತರ ಉತ್ಪಾದನಾ ಬಳಕೆಯಲ್ಲದ ಗ್ರಾಹಕ ಅಥವಾ staging ಸೈಟ್‌ನೊಂದಿಗೆ ಖಾತೆ ರಚನೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ.

## ಇಮೇಲ್ ಮಾರಾಟಗಾರರ ಹರಿವು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ

ಇಮೇಲ್ ಮಾರಾಟದಲ್ಲಿ ಎರಡು ಪ್ರತ್ಯೇಕ ಹಂತಗಳಿವೆ:

1. **ಅರ್ಹತೆ** — Ultimate Multisite ಇಮೇಲ್ ಉತ್ಪನ್ನ/ಸದಸ್ಯತ್ವ. ಇದು ಗ್ರಾಹಕರಿಗೆ ನಿರ್ದಿಷ್ಟ ಸೈಟ್ ಡೊಮೇನ್, ಪೂರೈಕೆದಾರ, ಕೋಟಾ, ಮತ್ತು ಖಾತೆ ಮಿತಿಗಾಗಿ ಇಮೇಲ್ ಬಳಸಲು ಅನುಮತಿಯಿದೆ ಎಂದು ಸೂಚಿಸುತ್ತದೆ.
2. **ಮೇಲ್‌ಬಾಕ್ಸ್ ರಚನೆ** — ಗ್ರಾಹಕ ಅಥವಾ ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕರು ಸ್ಥಳೀಯ ಭಾಗ ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಆಯ್ಕೆ ಮಾಡಿದಾಗ ರಚಿಸಲ್ಪಡುವ `info@example.com` ಮುಂತಾದ ನಿಜವಾದ ಪೂರೈಕೆದಾರ ಮೇಲ್‌ಬಾಕ್ಸ್.

Checkout ಸಮಯದಲ್ಲಿ, addon **ಇಮೇಲ್ ಸದಸ್ಯತ್ವ**ವನ್ನು ರಚಿಸುತ್ತದೆ ಅಥವಾ ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ, ಮತ್ತು ಆ ಸದಸ್ಯತ್ವದಲ್ಲಿ ಗ್ರಾಹಕರ ಸೈಟ್ ಡೊಮೇನ್ ಹಾಗೂ ಪೂರೈಕೆದಾರರನ್ನು ದಾಖಲಿಸುತ್ತದೆ. ಇದು ಮೇಲ್‌ಬಾಕ್ಸ್ ಹೆಸರು ಅಥವಾ ಪಾಸ್‌ವರ್ಡ್ ಅನ್ನು ಊಹಿಸುವುದಿಲ್ಲ. ಖರೀದಿ ಪೂರ್ಣಗೊಂಡು ಸೈಟ್/ಡೊಮೇನ್ ಸಂದರ್ಭ ಲಭ್ಯವಾದ ನಂತರ, ಗ್ರಾಹಕರು ಖಾತೆ ಪುಟವನ್ನು ತೆರೆದು ತಮಗೆ ಬೇಕಾದ ಮೇಲ್‌ಬಾಕ್ಸ್ ಹೆಸರುಗಳನ್ನು ರಚಿಸಬಹುದು.

ಅಂದರೆ ಸಾಮಾನ್ಯ ಗ್ರಾಹಕ ಹರಿವು ಹೀಗಿರುತ್ತದೆ:

1. ಗ್ರಾಹಕರು ಸೈಟ್ plan ಖರೀದಿಸಿ checkout ಸಮಯದಲ್ಲಿ ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ಸೇರಿಸುತ್ತಾರೆ.
2. Ultimate Multisite ಸೈಟ್ ಸದಸ್ಯತ್ವ ಮತ್ತು ದ್ವಿತೀಯ ಇಮೇಲ್ ಸದಸ್ಯತ್ವವನ್ನು ರಚಿಸುತ್ತದೆ.
3. Emails addon ಇಮೇಲ್ ಸದಸ್ಯತ್ವವನ್ನು ಗ್ರಾಹಕರ ಸೈಟ್ ಡೊಮೇನ್ ಮತ್ತು ಪೂರೈಕೆದಾರರಿಗೆ ಜೋಡಿಸುತ್ತದೆ.
4. ಗ್ರಾಹಕರು ತಮ್ಮ dashboard ನಲ್ಲಿ **Account** ತೆರೆಯುತ್ತಾರೆ.
5. ಆ ಸಕ್ರಿಯ ಇಮೇಲ್ ಸದಸ್ಯತ್ವಕ್ಕಾಗಿ **ಇಮೇಲ್ ಖಾತೆಗಳು** ಪ್ಯಾನೆಲ್ ಕಾಣಿಸುತ್ತದೆ.
6. ಗ್ರಾಹಕರು **ಖಾತೆ ರಚಿಸಿ** ಕ್ಲಿಕ್ ಮಾಡಿ, `info` ಮುಂತಾದ ಸ್ಥಳೀಯ ಭಾಗವನ್ನು ನಮೂದಿಸಿ, ಪಾಸ್‌ವರ್ಡ್ ನಿಗದಿಪಡಿಸುತ್ತಾರೆ. ನಂತರ addon ಸಂರಚಿಸಲಾದ ಪೂರೈಕೆದಾರನಲ್ಲಿ `info@their-domain.com` ರಚಿಸುತ್ತದೆ.

## ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ಸಂರಚಿಸಿ

ನೀವು ಮಾರಾಟ ಮಾಡಲು ಅಥವಾ ಉಚಿತವಾಗಿ ನೀಡಲು ಬಯಸುವ ಪ್ರತಿಯೊಂದು ಇಮೇಲ್ ಪ್ಯಾಕೇಜ್‌ಗೆ ಒಂದು ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ರಚಿಸಿ. ಉತ್ಪನ್ನ ಸೆಟಪ್‌ಗೆ ಹೊಸವರಾದರೆ, [ನಿಮ್ಮ ಮೊದಲ Subscription ಉತ್ಪನ್ನವನ್ನು ರಚಿಸುವುದು](../../user-guide/configuration/creating-your-first-subscription-product) ಇಂದ ಪ್ರಾರಂಭಿಸಿ, ನಂತರ ಇಮೇಲ್-ನಿರ್ದಿಷ್ಟ ಸೆಟ್ಟಿಂಗ್‌ಗಳಿಗಾಗಿ ಇಲ್ಲಿ ಮರಳಿ ಬನ್ನಿ.

1. **Network Admin > Ultimate Multisite > Products > Add Product** ಗೆ ಹೋಗಿ.
2. **Product Type** ಅನ್ನು **ಇಮೇಲ್** ಗೆ ಹೊಂದಿಸಿ.
3. ಬೆಲೆ ನಿಗದಿಯನ್ನು ಸಂರಚಿಸಿ. ಮಾರಾಟದ plan ಗಾಗಿ ಪಾವತಿಸಬೇಕಾದ ಮರುಕಳಿಸುವ ಬೆಲೆಯನ್ನು ಬಳಸಿ, ಅಥವಾ ಅದು ಪೂರಕ ಇಮೇಲ್ plan ಆಗಿರಬೇಕಾದರೆ ಉತ್ಪನ್ನವನ್ನು ಉಚಿತ/ಶೂನ್ಯ-ಬೆಲೆಗೆ ಹೊಂದಿಸಿ.
4. **ಇಮೇಲ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳು** ವಿಭಾಗವನ್ನು ತೆರೆಯಿರಿ.
5. ಸಂರಚಿಸಲಾದ **ಇಮೇಲ್ ಪೂರೈಕೆದಾರ** ಅನ್ನು ಆಯ್ಕೆಮಾಡಿ.
6. ಅಗತ್ಯವಿದ್ದಂತೆ **ಖಾತೆಗಳ ಸಂಖ್ಯೆ**, **ಸಂಗ್ರಹ ಕೋಟಾ (MB)**, alias ಮಿತಿಗಳು, ಫಾರ್ವರ್ಡರ್‌ಗಳು, ಮತ್ತು autoresponder‌ಗಳನ್ನು ಹೊಂದಿಸಿ.
7. ಉತ್ಪನ್ನವನ್ನು ಉಳಿಸಿ.

**Integrations** ಪುಟದಲ್ಲಿ ಸಕ್ರಿಯಗೊಳಿಸಲ್ಪಟ್ಟ ಮತ್ತು ಸಂಪೂರ್ಣವಾಗಿ ಸಂರಚಿಸಲ್ಪಟ್ಟ ಪೂರೈಕೆದಾರರು ಮಾತ್ರ ಪೂರೈಕೆದಾರ ಆಯ್ಕೆಮಾಡುವಿಕೆಯಲ್ಲಿ ಕಾಣಿಸುತ್ತಾರೆ. ಆಯ್ಕೆಮಾಡುವಿಕೆ ಖಾಲಿಯಾಗಿದ್ದರೆ, ಮೊದಲು ಪೂರೈಕೆದಾರರನ್ನು ಸಂರಚಿಸಿ ಮತ್ತು ಪರೀಕ್ಷಿಸಿ.

## Checkout ಗೆ ಇಮೇಲ್ ಸೇರಿಸಿ

ಇಮೇಲ್ ಉತ್ಪನ್ನಗಳನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಸೈಟ್ plan ಗೆ add-on ಗಳಾಗಿ ನೀಡಲಾಗುತ್ತದೆ. ಸಾಮಾನ್ಯ checkout ಸಂಪಾದಕ ಕಾರ್ಯಪ್ರವಾಹಕ್ಕಾಗಿ [Checkout ಫಾರ್ಮ್‌ಗಳು](../../user-guide/configuration/checkout-forms) ನೋಡಿ.

1. ಗ್ರಾಹಕರು ಮೊದಲು ಖರೀದಿಸುವ ಸೈಟ್ plan ರಚಿಸಿ ಅಥವಾ ಸಂಪಾದಿಸಿ.
2. ಆ checkout ಹರಿವಿಗೆ ಲಭ್ಯವಿರುವ add-on/order bump ಆಗಿ ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ಸೇರಿಸಿ.
3. checkout ಫಾರ್ಮ್‌ನಲ್ಲಿ ಇನ್ನೂ ಸಾಮಾನ್ಯ ಅಗತ್ಯ ಸೈಟ್ ಕ್ಷೇತ್ರಗಳು, ವಿಶೇಷವಾಗಿ ಸೈಟ್ URL/ಡೊಮೇನ್ ಕ್ಷೇತ್ರ, ಸೇರಿವೆ ಎಂಬುದನ್ನು ಖಚಿತಪಡಿಸಿ.
4. staging ಗ್ರಾಹಕರೊಂದಿಗೆ ಪರೀಕ್ಷಾ checkout ನಡೆಸಿ.
5. checkout ನಂತರ, ಗ್ರಾಹಕರ [ಖಾತೆ ಪುಟ](../../user-guide/client-management/account-page) ತೆರೆಯಿರಿ ಮತ್ತು **ಇಮೇಲ್ ಖಾತೆಗಳು** ಪ್ಯಾನೆಲ್ ಕಾಣಿಸುತ್ತಿದೆ ಎಂಬುದನ್ನು ಖಚಿತಪಡಿಸಿ.

ಇಮೇಲ್ ಡೊಮೇನ್ ಅನ್ನು ಗ್ರಾಹಕರ ಸೈಟ್ ಡೊಮೇನ್‌ನಿಂದ ತೆಗೆದುಕೊಳ್ಳಲಾಗುತ್ತದೆ. ಇಮೇಲ್ ಉತ್ಪನ್ನ ಹಂತದಲ್ಲಿ ಗ್ರಾಹಕರು ಇಮೇಲ್ ಡೊಮೇನ್ ನಮೂದಿಸುವುದಿಲ್ಲ.

## ಗ್ರಾಹಕ ಖಾತೆ ಇಮೇಲ್ ನಿರ್ವಹಣೆ

ಆವೃತ್ತಿ 1.1.0 ಗ್ರಾಹಕ ಖಾತೆ ಪುಟಕ್ಕೆ ಇಮೇಲ್ ನಿರ್ವಹಣೆ ವಿಜೆಟ್ ಅನ್ನು ಸೇರಿಸುತ್ತದೆ. ಗ್ರಾಹಕರು ತಮ್ಮದೇ ಸೈಟ್‌ಗಳಿಗೆ ಮೇಲ್‌ಬಾಕ್ಸ್ ರಚನೆ ಮತ್ತು ನಿರ್ವಹಣೆಯನ್ನು ಸ್ವಯಂ-ಸೇವೆಯಾಗಿ ಮಾಡಬೇಕಾದಾಗ ಇದನ್ನು ಬಳಸಿ.

ಖಾತೆ ಪುಟದಿಂದ, ಗ್ರಾಹಕರು ಇವುಗಳನ್ನು ಮಾಡಬಹುದು:

1. ತಮ್ಮ ಪ್ರಸ್ತುತ ಇಮೇಲ್ ಸದಸ್ಯತ್ವ/ಸೈಟ್ ಡೊಮೇನ್‌ಗೆ ಸಂಪರ್ಕಿತ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ವೀಕ್ಷಿಸಬಹುದು.
2. ಅವರ ಇಮೇಲ್ ಉತ್ಪನ್ನವು ಹೆಚ್ಚುವರಿ ಖಾತೆಗಳನ್ನು ಅನುಮತಿಸಿದಾಗ ಹೊಸ ಮೇಲ್‌ಬಾಕ್ಸ್ ರಚಿಸಬಹುದು.
3. ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಮೇಲ್‌ಬಾಕ್ಸ್ ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾಯಿಸಬಹುದು.
4. ಬೆಂಬಲಿತ ವೆಬ್‌ಮೇಲ್ ಪೂರೈಕೆದಾರರನ್ನು SSO ಅಥವಾ ಮ್ಯಾಜಿಕ್-ಲಿಂಕ್ ಲಾಗಿನ್ ಮೂಲಕ ತೆರೆಯಬಹುದು.

ವಿಜೆಟ್ Ultimate Multisite ಖಾತೆ-ಪುಟದ ಪ್ರಸ್ತುತಗಳ ವ್ಯವಸ್ಥೆಯನ್ನು ಬಳಸುತ್ತದೆ. ಆದ್ದರಿಂದ ಕ್ರಿಯೆಗಳು ಗ್ರಾಹಕರು ವೀಕ್ಷಿಸುತ್ತಿರುವ ಖಾತೆ ಸಂದರ್ಭಕ್ಕೆ ಸೀಮಿತವಾಗಿರುತ್ತವೆ. ಇದರಿಂದ ಗ್ರಾಹಕರು ಮತ್ತೊಂದು ಸದಸ್ಯತ್ವ ಅಥವಾ ಸೈಟ್‌ಗೆ ಸೇರಿದ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ನಿರ್ವಹಿಸುವುದನ್ನು ತಡೆಯಲಾಗುತ್ತದೆ.

ಗ್ರಾಹಕರು ಇಮೇಲ್ ಫಲಕವನ್ನು ನೋಡದಿದ್ದರೆ, ಅವರಿಗೆ **ಸಕ್ರಿಯ ಇಮೇಲ್ ಉತ್ಪನ್ನ ಸದಸ್ಯತ್ವ** ಇದೆಯೇ, ಆ ಸದಸ್ಯತ್ವವನ್ನು ಸೈಟ್ ಡೊಮೇನ್‌ಗೆ ಜೋಡಿಸಲಾಗಿದೆಯೇ, ಮತ್ತು ಉತ್ಪನ್ನಕ್ಕೆ ಸಂರಚಿಸಿದ provider ಇದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ.

## ಗ್ರಾಹಕರಿಗೆ ಉಚಿತ ಇಮೇಲ್ ಯೋಜನೆ ನೀಡುವುದು

ಗ್ರಾಹಕರಿಗೆ ಶುಲ್ಕ ವಿಧಿಸದೆ ಇಮೇಲ್ ನೀಡಲು, ಮೊದಲು **ಇಮೇಲ್ ಹಕ್ಕು** ನೀಡಿ. ನಂತರ mailbox ರಚಿಸಿ ಅಥವಾ ಗ್ರಾಹಕರಿಂದ ರಚಿಸಿಕೊಳ್ಳಲು ಬಿಡಿ.

ಶಿಫಾರಸು ಮಾಡಿದ ಆಯ್ಕೆಗಳು:

* **ಉಚಿತ ಇಮೇಲ್ ಉತ್ಪನ್ನ ರಚಿಸಿ**: ಇಮೇಲ್ ಉತ್ಪನ್ನದ ಬೆಲೆಯನ್ನು ಉಚಿತ/ಶೂನ್ಯಕ್ಕೆ ಹೊಂದಿಸಿ. ನಂತರ **Ultimate Multisite > ಸದಸ್ಯತ್ವಗಳು > ಸದಸ್ಯತ್ವ ಸೇರಿಸಿ** ಇಂದ ಆ ಗ್ರಾಹಕರಿಗೆ ಸಕ್ರಿಯ ಸದಸ್ಯತ್ವ ಸೇರಿಸಿ. ಇಮೇಲ್ ಉತ್ಪನ್ನ ಮತ್ತು ಗ್ರಾಹಕರನ್ನು ಆಯ್ಕೆಮಾಡಿ. ಉಚಿತ ಸದಸ್ಯತ್ವಗಳು ಬದಲಾಯಿಸುವವರೆಗೆ ಅಥವಾ ರದ್ದುಗೊಳಿಸುವವರೆಗೆ ಜೀವಿತಾವಧಿಯದ್ದಾಗಿರುತ್ತವೆ. ಇಮೇಲ್ ಸದಸ್ಯತ್ವವು ಈಗಿರುವ ಸೈಟ್ ಅನ್ನು ಅನುಸರಿಸಬೇಕಿದ್ದರೆ, ಅದನ್ನು ಆ ಸೈಟ್/ಡೊಮೇನ್ ಸಂದರ್ಭಕ್ಕೆ ಜೋಡಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿ.
* **100% ರಿಯಾಯಿತಿ ಅಥವಾ ಉಚಿತ checkout ಲಿಂಕ್ ಬಳಸಿ**: ಗ್ರಾಹಕರು checkout ಮೂಲಕ ಹೋಗಲು ಬಿಡಿ. ಆಗ Ultimate Multisite ಇಮೇಲ್ ಸದಸ್ಯತ್ವ ಮತ್ತು ಪಾವತಿ ದಾಖಲೆಯನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸುತ್ತದೆ.
* **ನಿರ್ವಾಹಕರಾಗಿ mailbox ರಚಿಸಿ**: ಗ್ರಾಹಕರಿಗಾಗಿ provider ನಲ್ಲಿ ನಿರ್ದಿಷ್ಟ mailbox ರಚಿಸಲು **Ultimate Multisite > ಇಮೇಲ್ Accountಗಳು > ಇಮೇಲ್ Account ಸೇರಿಸಿ** ಗೆ ಹೋಗಿ. ಇದು ಬೆಂಬಲಕ್ಕೆ ಉಪಯುಕ್ತ, ಆದರೆ ಗ್ರಾಹಕರ ಸ್ವಯಂ-ಸೇವೆ Account-ಪುಟ ಸಾಧನಗಳನ್ನು ನಡೆಸುವ ಇಮೇಲ್ ಉತ್ಪನ್ನ ಸದಸ್ಯತ್ವಕ್ಕೆ ಇದು ಬದಲಿಯಾಗುವುದಿಲ್ಲ.

ಗ್ರಾಹಕರು Account ಪುಟದಿಂದ mailbox ಅನ್ನು ತಾವೇ ನಿರ್ವಹಿಸಬೇಕೆಂದಿದ್ದರೆ, ಆ ಡೊಮೇನ್‌ಗೆ ಸಕ್ರಿಯ ಇಮೇಲ್ ಸದಸ್ಯತ್ವವಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿ. ಆ ಸದಸ್ಯತ್ವವಿಲ್ಲದೆ ಕೈಯಾರೆ ರಚಿಸಿದ provider mailbox ಇರಬಹುದು, ಆದರೆ ಗ್ರಾಹಕರು ಅದರ ಸ್ವಯಂ-ಸೇವೆ ಇಮೇಲ್ ಫಲಕವನ್ನು ನೋಡದೇ ಇರಬಹುದು.

## ನಿರ್ವಾಹಕರು ರಚಿಸಿದ Accountಗಳು ಮತ್ತು "ಆರ್ಡರ್ ಸಂಖ್ಯೆ ಇಲ್ಲ"

**ಇಮೇಲ್ Accountಗಳು** ನಿರ್ವಾಹಕ ಪುಟವು provider mailbox ಅನ್ನು ರಚಿಸುತ್ತದೆ ಅಥವಾ ಸಂಪಾದಿಸುತ್ತದೆ. ಇಮೇಲ್ ಯೋಜನೆಯನ್ನು ಮಾರಾಟ ಮಾಡಲು ಅಥವಾ ನೀಡಲು ಇದು ಮೊದಲ ಹಂತವಲ್ಲ.

ಇಮೇಲ್ Account ಸಂಪಾದಿಸುವಾಗ, **ಸದಸ್ಯತ್ವ** ಕ್ಷೇತ್ರವು ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ Ultimate Multisite ಸದಸ್ಯತ್ವಗಳನ್ನು ಅವುಗಳ ಉಲ್ಲೇಖ/ಆರ್ಡರ್ ಸಂಖ್ಯೆಯಿಂದ ಹುಡುಕುತ್ತದೆ. ಗ್ರಾಹಕರಿಗೆ ಇಮೇಲ್ ಸದಸ್ಯತ್ವ ಇರುವ ಮೊದಲು ನೀವು Account ನಿಯೋಜಿಸಲು ಪ್ರಯತ್ನಿಸಿದರೆ, ಆಯ್ಕೆಮಾಡಲು ಆರ್ಡರ್ ಸಂಖ್ಯೆ ಇರದೇ ಇರಬಹುದು.

ಬದಲಾಗಿ ಈ ಕ್ರಮವನ್ನು ಬಳಸಿ:

1. ಗ್ರಾಹಕರ ಇಮೇಲ್ ಸದಸ್ಯತ್ವವನ್ನು ರಚಿಸಿ ಅಥವಾ ಹುಡುಕಿ. ಖರೀದಿಗಳಿಗಾಗಿ, checkout ಇದನ್ನು ಮಾಡುತ್ತದೆ. ಪೂರಕ ಯೋಜನೆಗಾಗಿ, [ಸದಸ್ಯತ್ವಗಳು](../../user-guide/administration/managing-memberships) ಇಂದ ಸದಸ್ಯತ್ವವನ್ನು ಕೈಯಾರೆ ರಚಿಸಿ ಅಥವಾ ಗ್ರಾಹಕರನ್ನು ಉಚಿತ checkout ಮೂಲಕ ಕಳುಹಿಸಿ.
2. ಸದಸ್ಯತ್ವ ಸಕ್ರಿಯವಾಗಿದೆ ಮತ್ತು ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ಬಳಸುತ್ತದೆ ಎಂದು ದೃಢಪಡಿಸಿ.
3. ಇಮೇಲ್ ಸದಸ್ಯತ್ವವು ಸೈಟ್ ಡೊಮೇನ್/provider ಗೆ ಜೋಡಿಸಲಾಗಿದೆ ಎಂದು ದೃಢಪಡಿಸಿ.
4. ಗ್ರಾಹಕ Account ಪುಟದಿಂದ ಅಥವಾ **Ultimate Multisite > ಇಮೇಲ್ Accountಗಳು** ಇಂದ mailbox ರಚಿಸಿ.
5. ಅಗತ್ಯವಿದ್ದರೆ, mailbox ದಾಖಲೆಯನ್ನು ಸಂಪಾದಿಸಿ ಮತ್ತು ಗ್ರಾಹಕ, ಸದಸ್ಯತ್ವ, ಹಾಗೂ ಸೈಟ್ ಅನ್ನು ನಿಯೋಜಿಸಿ.

ನಿಮಗೆ ಗ್ರಾಹಕರಿಗಾಗಿ provider mailbox ರಚಿಸುವುದಷ್ಟೇ ಬೇಕಿದ್ದರೆ ಮತ್ತು ಸ್ವಯಂ-ಸೇವೆ Account-ಪುಟ ನಿರ್ವಹಣೆ ಅಗತ್ಯವಿಲ್ಲದಿದ್ದರೆ, ನಿರ್ವಾಹಕರು ರಚಿಸಿದ Account ಸಾಕಾಗಬಹುದು. ಗ್ರಾಹಕರು ಅದನ್ನು Ultimate Multisite ನಲ್ಲಿ ನೋಡಿ ನಿರ್ವಹಿಸಬೇಕೆಂದಿದ್ದರೆ, ಇಮೇಲ್ ಸದಸ್ಯತ್ವವನ್ನೂ ರಚಿಸಿ.

## Webmail SSO ಮತ್ತು Magic-Link ಲಾಗಿನ್

Webmail SSO ಗ್ರಾಹಕರಿಗೆ Account ಪುಟದಿಂದ ನೇರ **webmail ತೆರೆಯಿರಿ** ಕ್ರಿಯೆಯನ್ನು ನೀಡುತ್ತದೆ. ಸಕ್ರಿಯ provider ಬೆಂಬಲಿಸಿದಾಗ, addon ಅಲ್ಪಾವಧಿಯ access URL ಅನ್ನು ಕೋರುತ್ತದೆ ಮತ್ತು ಗ್ರಾಹಕರು mailbox credentials ನಕಲಿಸಬೇಕಾಗದಂತೆ ಅವರನ್ನು webmail ಗೆ ಮರುನಿರ್ದೇಶಿಸುತ್ತದೆ.

Provider ವರ್ತನೆ ಬದಲಾಗಬಹುದು:

* native SSO APIಗಳಿರುವ providers ನೇರ authenticated webmail session ರಚಿಸಬಹುದು.
* magic-link ಬೆಂಬಲವಿರುವ providers ಸಮಯ-ಮಿತಿಯ login link ರಚಿಸಬಹುದು.
* SSO ಅಥವಾ magic-link ಬೆಂಬಲವಿಲ್ಲದ providers mailbox ವಿವರಗಳು ಮತ್ತು password-management ಕ್ರಿಯೆಗಳನ್ನು ತೋರಿಸಬೇಕು, ಆದರೆ one-click webmail ಬಟನ್ ಇರಬಾರದು.

ಅಗತ್ಯ API credentials ಸಂರಚಿಸಿರುವ providers ಗಾಗಿ ಮಾತ್ರ webmail ಪ್ರವೇಶವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ. provider ಗಾಗಿ ಕ್ರಿಯೆ ಕಾಣಿಸದಿದ್ದರೆ, ಮೊದಲು provider ಸಕ್ರಿಯವಾಗಿದೆ, ಸಂರಚಿಸಲಾಗಿದೆ ಮತ್ತು webmail login links ಬೆಂಬಲಿಸುತ್ತದೆ ಎಂದು ದೃಢಪಡಿಸಿ.

## Checkout Provisioning

addon checkout ವೇಳೆ ಇಮೇಲ್ **ಸದಸ್ಯತ್ವ ಸಂದರ್ಭ**ವನ್ನು provision ಮಾಡುತ್ತದೆ. ಇದು provider, Account ಮಿತಿ, quota ಮತ್ತು ಸೈಟ್ ಡೊಮೇನ್ ಅನ್ನು ದಾಖಲಿಸುತ್ತದೆ. ಖರೀದಿಯ ನಂತರ ಗ್ರಾಹಕರು Account ಪುಟದಿಂದ ತಮ್ಮ mailbox ಹೆಸರುಗಳನ್ನು ರಚಿಸಬಹುದು.

ಶಿಫಾರಸು ಮಾಡಿದ rollout:

1. ಇಮೇಲ್ ಉತ್ಪನ್ನವನ್ನು ನೀಡುವ ಮೊದಲು provider ಮತ್ತು ಉತ್ಪನ್ನ ಮಿತಿಗಳನ್ನು ಸಂರಚಿಸಿ.
2. staging ಉತ್ಪನ್ನದೊಂದಿಗೆ ಖರೀದಿಯನ್ನು ಪರೀಕ್ಷಿಸಿ ಮತ್ತು ಇಮೇಲ್ ಸದಸ್ಯತ್ವ ರಚಿಸಲಾಗಿದೆ ಎಂದು ದೃಢಪಡಿಸಿ.
3. ಡೊಮೇನ್‌ಗಾಗಿ **ಇಮೇಲ್ Accountಗಳು** widget ಕಾಣುತ್ತದೆ ಎಂದು ದೃಢಪಡಿಸಲು ಗ್ರಾಹಕ Account ಪುಟವನ್ನು ಪರಿಶೀಲಿಸಿ.
4. widget ಇಂದ ಪರೀಕ್ಷಾ mailbox ರಚಿಸಿ.
5. production ಉತ್ಪನ್ನಗಳಿಗಾಗಿ flow ಸಕ್ರಿಯಗೊಳಿಸುವ ಮೊದಲು password-change ಮತ್ತು webmail ಕ್ರಿಯೆಗಳನ್ನು ದೃಢಪಡಿಸಿ.

Provisioning ವಿಫಲವಾದರೆ, ಇಮೇಲ್ ಸದಸ್ಯತ್ವ ಅಥವಾ provider mailbox ಗೆ ನಿರ್ವಾಹಕರ ಮುಂದಿನ ಕ್ರಮ ಅಗತ್ಯವಿದ್ದರೂ ಗ್ರಾಹಕ Account ಮತ್ತು ಸದಸ್ಯತ್ವ ಇರಬಹುದು. ಮರುಪ್ರಯತ್ನಿಸುವ ಮೊದಲು Ultimate Multisite activity logs ಮತ್ತು provider logs ಪರಿಶೀಲಿಸಿ.

## CyberPanel ಇಮೇಲ್ Forwarders

CyberPanel-backed networks ಗಾಗಿ, version 1.1.0 ಇಮೇಲ್ forwarder ಬೆಂಬಲವನ್ನು ಸೇರಿಸುತ್ತದೆ. ಗ್ರಾಹಕರು ಪೂರ್ಣ mailbox ರಚಿಸದೆ `info@customer-domain.test` ಮುಂತಾದ ವಿಳಾಸಗಳಿಂದ ಮೇಲ್ ಅನ್ನು ಮತ್ತೊಂದು inbox ಗೆ ಮರುನಿರ್ದೇಶಿಸಬೇಕಾದಾಗ forwarders ಬಳಸಿ.

Forwarders ನೀಡುವ ಮೊದಲು, CyberPanel integration ಸಂರಚಿಸಲಾಗಿದೆ ಮತ್ತು ತಲುಪಬಹುದಾಗಿದೆ ಎಂದು ದೃಢಪಡಿಸಿ. ಅಗತ್ಯ host, username, password ಮತ್ತು package constants ಗಾಗಿ [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) ಮಾರ್ಗದರ್ಶಿ ನೋಡಿ.

## ಬದಲಾವಣೆಗಳ ಪಟ್ಟಿ

### Version 1.1.0 - 2026-05-05 ರಂದು ಬಿಡುಗಡೆ ಮಾಡಲಾಗಿದೆ

* ಹೊಸದು: Webmail SSO ಮತ್ತು magic-link login — ಗ್ರಾಹಕರು one-click authentication ಮೂಲಕ ತಮ್ಮ Account‌ನಿಂದ ನೇರವಾಗಿ webmail ಪ್ರವೇಶಿಸಬಹುದು.
* ಹೊಸದು: ಇಮೇಲ್ Account ನಿರ್ವಹಣೆಯ widget‌ಗೆ password ಬದಲಿಸುವ ಕಾರ್ಯಕ್ಷಮತೆ ಸೇರಿಸಲಾಗಿದೆ.
* ಹೊಸದು: CyberPanel ಇಮೇಲ್ forwarder ಬೆಂಬಲ.
* ಹೊಸದು: ಸ್ವಯಂ-ಸೇವೆಯ Account ರಚನೆ ಮತ್ತು ನಿರ್ವಹಣೆಗೆ ಗ್ರಾಹಕರ Account ಪುಟದಲ್ಲಿ ಇಮೇಲ್ ನಿರ್ವಹಣಾ widget.
* ಹೊಸದು: ಗ್ರಾಹಕರ ಇಮೇಲ್ Account entitlement provisioning ಅನ್ನು checkout ಪ್ರವಾಹಕ್ಕೆ ಸಂಯೋಜಿಸಲಾಗಿದೆ.
* ಸರಿಪಡಿಸಲಾಗಿದೆ: ಸರಿಯಾದ UMS ನೋಂದಣಿ pattern ಬಳಸಲು Account ಪುಟದ widget hook ನವೀಕರಿಸಲಾಗಿದೆ.
* ಸರಿಪಡಿಸಲಾಗಿದೆ: Account ರಚಿಸುವ form ಅನ್ನು vanilla JS ಮತ್ತು ಸರಿಯಾದ ಪೂರ್ಣ HTML document output‌ನಿಂದ ಬದಲಾಯಿಸಲಾಗಿದೆ.
* ಸರಿಪಡಿಸಲಾಗಿದೆ: ಇಮೇಲ್ ನಿರ್ವಹಣಾ widget ಈಗ admin Account ಪುಟಗಳಿಗೆ UMS currents system ಬಳಸುತ್ತದೆ.
* ಸರಿಪಡಿಸಲಾಗಿದೆ: Admin ಇಮೇಲ್ Account ರಚನಾ form password field ಒಳಗೊಂಡಿದೆ ಮತ್ತು Email_Account_Manager ಬಳಸುತ್ತದೆ.
* ಸರಿಪಡಿಸಲಾಗಿದೆ: Email_Account_Manager initialization‌ನಿಂದ ಪ್ರತಿ-request log spam ತೆಗೆದುಹಾಕಲಾಗಿದೆ.
* ಸುಧಾರಿಸಲಾಗಿದೆ: Settings ಪುಟವು ಈಗ ಸಕ್ರಿಯ ಮತ್ತು configure ಮಾಡಲಾದ ಇಮೇಲ್ providers ಮಾತ್ರ ತೋರಿಸಲು filter ಮಾಡುತ್ತದೆ.

## ಪದೇ ಪದೇ ಕೇಳಲಾಗುವ ಪ್ರಶ್ನೆಗಳು

### ಗ್ರಾಹಕರು ತಮ್ಮದೇ ಇಮೇಲ್ Accounts ರಚಿಸಬಹುದೇ?

ಹೌದು. ಗ್ರಾಹಕರಿಗೆ domain‌ಗೆ ಸಕ್ರಿಯ ಇಮೇಲ್ membership ಇದ್ದು product ಹೆಚ್ಚುವರಿ accounts ಅನುಮತಿಸಿದರೆ, ಗ್ರಾಹಕರು Account ಪುಟದಿಂದ ತಮ್ಮದೇ mailboxes ರಚಿಸಿ ನಿರ್ವಹಿಸಬಹುದು.

### ಪ್ರತಿಯೊಂದು provider Webmail SSO ಬೆಂಬಲಿಸುತ್ತದೆಯೇ?

ಇಲ್ಲ. SSO ಮತ್ತು magic-link ಬೆಂಬಲವು ಸಕ್ರಿಯ ಇಮೇಲ್ provider ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿದೆ. access-link API ಇಲ್ಲದ providers mailbox ರಚನೆ ಮತ್ತು password ಬದಲಾವಣೆಗಳನ್ನು ಇನ್ನೂ ಬೆಂಬಲಿಸಬಹುದು.

### Administrators ಗ್ರಾಹಕರಿಗಾಗಿ ಇಮೇಲ್ Accounts ರಚಿಸಬಹುದೇ?

ಹೌದು. Administrators admin interface‌ನಿಂದ ಇಮೇಲ್ Accounts ರಚಿಸಬಹುದು, Account ರಚಿಸುವಾಗ ಆರಂಭಿಕ password ನಿಗದಿಪಡಿಸುವುದೂ ಸೇರಿದೆ. ಗ್ರಾಹಕರ self-service ನಿರ್ವಹಣೆಗೆ, ಗ್ರಾಹಕರಿಗೆ domain‌ಗೆ ಸಕ್ರಿಯ ಇಮೇಲ್ membership ಇದೆ ಎಂಬುದನ್ನೂ ಖಚಿತಪಡಿಸಿ.

### ನಾನು ಇಮೇಲ್ Account ಅನ್ನು ಉಚಿತವಾಗಿ ನೀಡಬಹುದೇ?

ಹೌದು. ಉಚಿತ/ಶೂನ್ಯ-ಬೆಲೆಯ ಇಮೇಲ್ product ರಚಿಸಿ ಅದನ್ನು ಗ್ರಾಹಕರಿಗೆ ಸಕ್ರಿಯ membership ಆಗಿ ಸೇರಿಸಿ, ಅಥವಾ ಗ್ರಾಹಕರು 100% discount ಜೊತೆಗೆ checkout ಪೂರ್ಣಗೊಳಿಸಲು ಬಿಡಿ. ಇಮೇಲ್ membership ಇದ್ದ ನಂತರ, ಗ್ರಾಹಕ ಅಥವಾ administrator ನಿಜವಾದ mailbox ರಚಿಸಬಹುದು.

### ಇಮೇಲ್ Account ನಿಯೋಜಿಸುವಾಗ order number ಅನ್ನು ಏಕೆ ಕೇಳುತ್ತದೆ?

Membership field ಅದರ reference/order number ಮೂಲಕ ಇರುವ Ultimate Multisite membership ಅನ್ನು ಹುಡುಕುತ್ತದೆ. ಗ್ರಾಹಕರು ಇಮೇಲ್ product ಖರೀದಿಸದಿದ್ದರೆ ಮತ್ತು ನೀವು ಕೈಯಾರೆ ಇಮೇಲ್ membership ರಚಿಸಿಲ್ಲದಿದ್ದರೆ, ನಿಯೋಜಿಸಲು ಇಮೇಲ್ membership/order ಇರುವುದಿಲ್ಲ. ಮೊದಲು ಇಮೇಲ್ membership ರಚಿಸಿ, ನಂತರ mailbox ರಚಿಸಿ ಅಥವಾ ನಿಯೋಜಿಸಿ.

### Forwarders mailboxes ಒಂದೇ ಆಗಿವೆಯೇ?

ಇಲ್ಲ. mailbox mail ಅನ್ನು ಸಂಗ್ರಹಿಸುತ್ತದೆ ಮತ್ತು login credentials ಹೊಂದಿರುತ್ತದೆ. forwarder mail ಅನ್ನು ಮತ್ತೊಂದು address‌ಗೆ redirect ಮಾಡುತ್ತದೆ ಮತ್ತು ಪ್ರತ್ಯೇಕ inbox ಒದಗಿಸುವುದಿಲ್ಲ.
