# ಇಮೇಲ್‌ಗಳು (Emails)

Ultimate Multisite: Emails addon ಅನ್ನು ನೆಟ್‌ವರ್ಕ್ ಮಾಲೀಕರು (network owners) ಸೈಟ್‌ಗಳು ಮತ್ತು ಸದಸ್ಯತ್ವಗಳ ಜೊತೆಗೆ ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ಸಜ್ಜುಗೊಳಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. Version 1.1.0 ರಲ್ಲಿ ಗ್ರಾಹಕರಿಗೆ ನೇರವಾಗಿ ಬಳಸಬಹುದಾದ ಇಮೇಲ್ ಖಾತೆ ನಿರ್ವಹಣೆ, webmail single sign-on, CyberPanel forwarder ಬೆಂಬಲ ಮತ್ತು checkout ಸಮಯದಲ್ಲಿ ಸಜ್ಜುಗೊಳಿಸುವಿಕೆ ಸುಧಾರಣೆಗಳನ್ನು ಸೇರಿಸಲಾಗಿದೆ.

## ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು (Key Features)

* **ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆ ವಿಡ್ಜೆಟ್ (Customer email account widget)**: ಗ್ರಾಹಕರು ಯಾವುದೇ support ticket ತೆರೆಯದೆ ತಮ್ಮ account page ನಿಂದ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ರಚಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.
* **Webmail SSO ಮತ್ತು magic links**: ಒದಗಿಸುವವರು (provider) ಬೆಂಬಲ ನೀಡಿದರೆ, ಗ್ರಾಹಕರು account page ನಿಂದ ನೇರವಾಗಿ webmail ಅನ್ನು ಒಂದು ಕ್ಲಿಕ್ ಮೂಲಕ authentication ಮೂಲಕ ತೆರೆಯಬಹುದು.
* **ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾವಣೆಗಳು (Password changes)**: ಗ್ರಾಹಕರು ಮತ್ತು ಆಡಳಿತಾಧಿಕಾರಿಗಳು ಇಮೇಲ್ ನಿರ್ವಹಣಾ ವಿಡ್ಜೆಟ್‌ನಿಂದ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ಪಾಸ್‌ವರ್ಡ್‌ಗಳನ್ನು ಅಪ್ಡೇಟ್ ಮಾಡಬಹುದು.
* **Checkout ಸಜ್ಜುಗೊಳಿಸುವಿಕೆ (Checkout provisioning)**: ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆ ರಚನೆಯನ್ನು checkout ಸಮಯದಲ್ಲಿ ನಡೆಸಬಹುದು, ಇದರಿಂದ purchase flow ಪೂರ್ಣಗೊಂಡ ನಂತರ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ಸಿದ್ಧವಾಗಿರುತ್ತದೆ.
* **CyberPanel forwarders**: CyberPanel-ಆಧಾರಿತ ನೆಟ್‌ವರ್ಕ್‌ಗಳು ಇಮೇಲ್ ಸೇವೆಗಳ ಪ್ರಕ್ರಿಯೆಯ ಭಾಗವಾಗಿ ಇಮೇಲ್ ಫಾರ್ವರ್ಡರ್‌ಗಳನ್ನು ರಚಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.
* **Provider-aware ಸೆಟ್ಟಿಂಗ್‌ಗಳು**: ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಪುಟವು ಲಭ್ಯವಿರುವ, ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳಿಗೆ ಮಾತ್ರ ಫಿಲ್ಟರ್ ಮಾಡುತ್ತದೆ, ಇದರಿಂದ ಆಡಳಿತಾಧಿಕಾರಿಗಳು ಬಳಸಬಹುದಾದ ಆಯ್ಕೆಗಳನ್ನು ಮಾತ್ರ ನೋಡುತ್ತಾರೆ.

## ಅವಶ್ಯಕತೆಗಳು (Requirements)

* Ultimate Multisite 2.0.0 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ
* WordPress Multisite installation
* ಕನಿಷ್ಠ ಒಂದು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಇಮೇಲ್ ಒದಗಿಸುವವರ ಇಂಟಿಗ್ರೇಷನ್
* ಮೇಲ್ ಪೆಟ್ಟಿಗೆಗಳನ್ನು ರಚಿಸಲು, ಪಾಸ್‌ವರ್ಡ್‌ಗಳನ್ನು ಬದಲಾಯಿಸಲು ಮತ್ತು webmail access links ನೀಡಲು ಅನುಮತಿ ಹೊಂದಿರುವ Provider API credentials

## ಅಳವಡಿಕೆ (Installation)

1. ಪ್ಲಗ್ಇನ್ ಫೈಲ್‌ಗಳನ್ನು `/wp-content/plugins/ultimate-multisite-emails` ಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ.
2. WordPress network admin ನಲ್ಲಿ **Plugins** ನಿಂದ addon ಅನ್ನು ನೆಟ್‌ವರ್ಕ್ ಆಕ್ಟಿವೇಟ್ ಮಾಡಿ.
3. **Ultimate Multisite** > **Settings** ಗೆ ಹೋಗಿ ಮತ್ತು addon ಇಮೇಲ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ತೆರೆಯಿರಿ.
4. ಗ್ರಾಹಕರು ಬಳಸಬೇಕೆಂದು ನೀವು ಬಯಸುವ ಇಮೇಲ್ ಒದಗಿಸುವವರ ಇಂಟಿಗ್ರೇಷನ್‌ಗಳನ್ನು ಎನಬಲ್ ಮಾಡಿ.
5. ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸೇವ್ ಮಾಡಿ, ನಂತರ non-production ಗ್ರಾಹಕ ಅಥವಾ staging site ಬಳಸಿ account ರಚನೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ.

## ಗ್ರಾಹಕರ account ಇಮೇಲ್ ನಿರ್ವಹಣೆ (Customer Account Email Management)

Version 1.1.0 ಗ್ರಾಹಕರ account page ಗೆ ಇಮೇಲ್ ನಿರ್ವಹಣಾ ವಿಡ್ಜೆಟ್ ಅನ್ನು ಸೇರಿಸುತ್ತದೆ. ಗ್ರಾಹಕರು ತಮ್ಮದೇ ಸೈಟ್‌ಗಳಿಗಾಗಿ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ರಚನೆ ಮತ್ತು ನಿರ್ವಹಣೆಯನ್ನು ಸ್ವಯಂ-ಸೇವೆ (self-service) ಮಾಡಬೇಕಾದ ಸಂದರ್ಭದಲ್ಲಿ ಇದನ್ನು ಬಳಸಿ.

account page ನಿಂದ, ಗ್ರಾಹಕರು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡಬಹುದು:

1. ತಮ್ಮ ಪ್ರಸ್ತುತ ಸದಸ್ಯತ್ವ ಅಥವಾ ಸೈಟ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿರುವ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ವೀಕ್ಷಿಸಬಹುದು.
2. ಅವರ ಯೋಜನೆಯು ಹೆಚ್ಚುವರಿ ಖಾತೆಗಳನ್ನು ಅನುಮತಿಸಿದರೆ, ಹೊಸ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ರಚಿಸಬಹುದು.
3. ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾಯಿಸಬಹುದು.
4. ಬೆಂಬಲಿತ webmail ಒದಗಿಸುವವರ ಮೂಲಕ SSO ಅಥವಾ magic-link ಲಾಗಿನ್ ಮೂಲಕ ಪ್ರವೇಶಿಸಬಹುದು.

ಈ ವಿಡ್ಜೆಟ್ Ultimate Multisite account-page currents system ಅನ್ನು ಬಳಸುತ್ತದೆ, ಆದ್ದರಿಂದ ಕ್ರಿಯೆಗಳು ಗ್ರಾಹಕರು ನೋಡುತ್ತಿರುವ account ಸಂದರ್ಭಕ್ಕೆ (account context) ಸೀಮಿತವಾಗಿರುತ್ತವೆ. ಇದು ಗ್ರಾಹಕರು ಬೇರೆ ಸದಸ್ಯತ್ವ ಅಥವಾ ಸೈಟ್‌ಗೆ ಸೇರಿದ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ನಿರ್ವಹಿಸುವುದನ್ನು ತಡೆಯುತ್ತದೆ.

## Webmail SSO ಮತ್ತು Magic-Link ಲಾಗಿನ್ (Webmail SSO and Magic-Link Login)

Webmail SSO ಗ್ರಾಹಕರಿಗೆ account page ನಿಂದ ನೇರವಾದ **Open webmail** ಕ್ರಿಯೆಯನ್ನು ನೀಡುತ್ತದೆ. ಸಕ್ರಿಯ ಒದಗಿಸುವವರಿಂದ (active provider) ಇದು ಬೆಂಬಲಿತವಾದರೆ, addon ಒಂದು short-lived access URL ಅನ್ನು ವಿನಂತಿಸುತ್ತದೆ ಮತ್ತು ಗ್ರಾಹಕರಿಗೆ webmail ಗೆ ಮರುನಿರ್ದೇಶಿಸುತ್ತದೆ, ಇದರಿಂದ ಅವರು ಮೇಲ್ ಪೆಟ್ಟಿಗೆ credentials ಅನ್ನು ಕಾಪಿ ಮಾಡಲು ಕೇಳಬೇಕಾಗಿ ಬರುವುದಿಲ್ಲ.

ಒದಗಿಸುವವರ ನಡವಳಿಕೆಯು ಬದಲಾಗಬಹುದು:

* ನೈಸರ್ಗಿಕ SSO API ಹೊಂದಿರುವ ಒದಗಿಸುವವರು ನೇರವಾದ authentication webmail session ಅನ್ನು ರಚಿಸಬಹುದು.
* magic-link ಬೆಂಬಲ ಹೊಂದಿರುವ ಒದಗಿಸುವವರು ಸಮಯ-ಮಿತಿಯ ಲಾಗಿನ್ ಲಿಂಕ್ ಅನ್ನು ರಚಿಸಬಹುದು.
* SSO ಅಥವಾ magic-link ಬೆಂಬಲವಿಲ್ಲದ ಒದಗಿಸುವವರು ಇನ್ನೂ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ವಿವರಗಳು ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ನಿರ್ವಹಣಾ ಕ್ರಿಯೆಗಳನ್ನು ತೋರಿಸಬೇಕು, ಆದರೆ ಒಂದು-ಕ್ಲಿಕ್ webmail ಬಟನ್ ಇರುವುದಿಲ್ಲ.

ನೀವು ಅಗತ್ಯವಾದ API credentials ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿರುವ ಒದಗಿಸುವವರಿಗಾಗಿ ಮಾತ್ರ webmail access ಅನ್ನು ಎನಬಲ್ ಮಾಡಿ. ಒಂದು ಒದಗಿಸುವವರಿಂದ ಕ್ರಿಯೆ ಕಾಣೆಯಾಗಿದ್ದರೆ, ಮೊದಲು ಆ ಒದಗಿಸುವವರು ಸಕ್ರಿಯವಾಗಿ, ಕಾನ್ಫಿಗರ್ ಆಗಿದ್ದು, webmail ಲಾಗಿನ್ ಲಿಂಕ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.

## Checkout ಸಜ್ಜುಗೊಳಿಸುವಿಕೆ (Checkout Provisioning)

addon ಅನ್ನು checkout ಭಾಗವಾಗಿ ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ರಚಿಸಲು ಬಳಸಬಹುದು. ಯೋಜನೆಯು ಡೀಫಾಲ್ಟ್ ಆಗಿ ಮೇಲ್ ಪೆಟ್ಟಿಗೆಯನ್ನು ಒಳಗೊಂಡಿದ್ದರೆ ಅಥವಾ checkout ಫಾರ್ಮ್ ಇಮೇಲ್-ಖಾತೆ ವಿವರಗಳನ್ನು ಸಂಗ್ರಹಿಸಿದರೆ ಇದು ಉಪಯುಕ್ತವಾಗಿದೆ.

ಶಿಫಾರಸು ಮಾಡಲಾದ ವಿಧಾನ:

1. checkout provisioning ಅನ್ನು ಎನಬಲ್ ಮಾಡುವ ಮೊದಲು ಒದಗಿಸುವವರು ಮತ್ತು ಯೋಜನೆಯ ಮಿತಿಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ.
2. staging product ಬಳಸಿ purchase ಅನ್ನು ಪರೀಕ್ಷಿಸಿ ಮತ್ತು checkout ನಂತರ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ರಚನೆಯಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.
3. ಹೊಸ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ಇಮೇಲ್ ವಿಡ್ಜೆಟ್‌ನಲ್ಲಿ ಕಾಣಿಸುತ್ತಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು ಗ್ರಾಹಕರ account page ಅನ್ನು ಪರಿಶೀಲಿಸಿ.
4. production products ಗಾಗಿ flow ಅನ್ನು ಎನಬಲ್ ಮಾಡುವ ಮೊದಲು ಪಾಸ್‌ವರ್ಡ್-ಬದಲಾವಣೆ ಮತ್ತು webmail ಕ್ರಿಯೆಗಳನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.

ಸಜ್ಜುಗೊಳಿಸುವಿಕೆ ವಿಫಲವಾದರೆ, ಮೇಲ್ ಪೆಟ್ಟಿಗೆಗೆ ಆಡಳಿತಾಧಿಕಾರಿಗಳ ಅನುಸರಣೆ ಅಗತ್ಯವಿದ್ದರೂ ಗ್ರಾಹಕರ account ಮತ್ತು ಸದಸ್ಯತ್ವ ಇನ್ನೂ ಅಸ್ತಿತ್ವದಲ್ಲಿರಬಹುದು. ಪುನರಾವರ್ತಿಸುವ ಮೊದಲು Ultimate Multisite activity logs ಮತ್ತು ಒದಗಿಸುವವರ logs ಅನ್ನು ಪರಿಶೀಲಿಸಿ.

## CyberPanel ಇಮೇಲ್ ಫಾರ್ವರ್ಡರ್‌ಗಳು (CyberPanel Email Forwarders)

CyberPanel-ಆಧಾರಿತ ನೆಟ್‌ವರ್ಕ್‌ಗಳಿಗಾಗಿ, version 1.1.0 ಇಮೇಲ್ ಫಾರ್ವರ್ಡರ್ ಬೆಂಬಲವನ್ನು ಸೇರಿಸುತ್ತದೆ. ಗ್ರಾಹಕರಿಗೆ `info@customer-domain.test` ನಂತಹ ವಿಳಾಸಗಳು ಸಂಪೂರ್ಣ ಮೇಲ್ ಪೆಟ್ಟಿಗೆಯನ್ನು ರಚಿಸದೆ ಇಮೇಲ್ ಅನ್ನು ಇನ್ನೊಂದು ಇನ್‌ಬಾಕ್ಸ್‌ಗೆ ಮರುನಿರ್ದೇಶಿಸಲು ಬೇಕಾದಾಗ ಫಾರ್ವರ್ಡರ್‌ಗಳನ್ನು ಬಳಸಿ.

ಫಾರ್ವರ್ಡರ್‌ಗಳನ್ನು ನೀಡುವ ಮೊದಲು, CyberPanel ಇಂಟಿಗ್ರೇಷನ್ ಕಾನ್ಫಿಗರ್ ಆಗಿದ್ದು ಮತ್ತು ಲಭ್ಯವಾಯಿತೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಅಗತ್ಯವಿರುವ host, username, password ಮತ್ತು package constants ಗಾಗಿ [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) ಮಾರ್ಗದರ್ಶಿಯನ್ನು ನೋಡಿ.

* ಹೊಸ: ಸ್ವಯಂ-ಸೇವೆ ಖಾತೆ ರಚನೆ ಮತ್ತು ನಿರ್ವಹಣೆಗಾಗಿ ಗ್ರಾಹಕರ account page ನಲ್ಲಿ ಇಮೇಲ್ ನಿರ್ವಹಣಾ ವಿಡ್ಜೆಟ್.
* ಹೊಸ: ಸ್ವಯಂಚಾಲಿತ ಸಜ್ಜುಗೊಳಿಸುವಿಕೆಗಾಗಿ checkout flow ಗೆ ಗ್ರಾಹಕರ ಇಮೇಲ್ ಖಾತೆ ರಚನೆಯನ್ನು ಸಂಯೋಜಿಸಲಾಗಿದೆ.
* ಫಿಕ್ಸ್: account page widget hook ಅನ್ನು ಸರಿಯಾದ UMS registration pattern ಬಳಸುವಂತೆ ಅಪ್ಡೇಟ್ ಮಾಡಲಾಗಿದೆ.
* ಫಿಕ್ಸ್: Create Account ಫಾರ್ಮ್ ಅನ್ನು vanilla JS ಮತ್ತು ಸರಿಯಾದ full HTML document output ಯೊಂದಿಗೆ ಬದಲಾಯಿಸಲಾಗಿದೆ.
* ಫಿಕ್ಸ್: ಇಮೇಲ್ ನಿರ್ವಹಣಾ ವಿಡ್ಜೆಟ್ ಈಗ admin account pages ಗಾಗಿ UMS currents system ಅನ್ನು ಬಳಸುತ್ತದೆ.
* ಫಿಕ್ಸ್: Admin ಇಮೇಲ್ ಖಾತೆ ರಚನೆಯ ಫಾರ್ಮ್ ಪಾಸ್‌ವರ್ಡ್ ಫೀಲ್ಡ್ ಅನ್ನು ಒಳಗೊಂಡಿದೆ ಮತ್ತು Email_Account_Manager ಅನ್ನು ಬಳಸುತ್ತದೆ.
* ಫಿಕ್ಸ್: Email_Account_Manager initialization ನಿಂದ ಪ್ರತಿ-요청 log spam ಅನ್ನು ತೆಗೆದುಹಾಕಲಾಗಿದೆ.
* ಸುಧಾರಣೆ: ಸೆಟ್ಟಿಂಗ್‌ಗಳ ಪುಟವು ಈಗ ಸಕ್ರಿಯ ಮತ್ತು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದ ಇಮೇಲ್ ಒದಗಿಸುವವರಿಗೆ ಮಾತ್ರ ಫಿಲ್ಟರ್ ಮಾಡಲು ತೋರಿಸುತ್ತದೆ.

## ಆಗಾಗ್ಗೆ ಕೇಳಲಾಗುವ ಪ್ರಶ್ನೆಗಳು (Frequently Asked Questions)

## ಗ್ರಾಹಕರು ತಮ್ಮದೇ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ರಚಿಸಬಹುದೇ?

ಹೌದು. ಗ್ರಾಹಕರ account widget ಎನಬಲ್ ಆಗಿದ್ದರೆ ಮತ್ತು ಯೋಜನೆಯು ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ಅನುಮತಿಸಿದರೆ, ಗ್ರಾಹಕರು account page ನಿಂದ ತಮ್ಮದೇ ಮೇಲ್ ಪೆಟ್ಟಿಗೆಗಳನ್ನು ರಚಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

## ಪ್ರತಿ ಒದಗಿಸುವವವೂ webmail SSO ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆಯೇ?

ಇಲ್ಲ. SSO ಮತ್ತು magic-link ಬೆಂಬಲವು ಸಕ್ರಿಯ ಇಮೇಲ್ ಒದಗಿಸುವವरावर ಅವಲಂಬಿತವಾಗಿರುತ್ತದೆ. access-link API ಇಲ್ಲದ ಒದಗಿಸುವವರು ಇನ್ನೂ ಮೇಲ್ ಪೆಟ್ಟಿಗೆ ರಚನೆ ಮತ್ತು ಪಾಸ್‌ವರ್ಡ್ ಬದಲಾವಣೆಗಳನ್ನು ಬೆಂಬಲಿಸಬಹುದು.

## ಆಡಳಿತಾಧಿಕಾರಿಗಳು ಗ್ರಾಹಕರಿಗಾಗಿ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ರಚಿಸಬಹುದೇ?

ಹೌದು. ಆಡಳಿತಾಧಿಕಾರಿಗಳು ಆಡಳಿತಾಧಿಕಾರಿ ಇಂಟರ್ಫೇಸ್‌ನಿಂದ ಇಮೇಲ್ ಖಾತೆಗಳನ್ನು ರಚಿಸಬಹುದು, ಖಾತೆಯನ್ನು ರಚಿಸುವಾಗ ಆರಂಭಿಕ ಪಾಸ್‌ವರ್ಡ್ ಅನ್ನು ಹೊಂದಿಸುವುದು ಸೇರಿದಂತೆ.

## ಫಾರ್ವರ್ಡರ್‌ಗಳು ಮೇಲ್ ಪೆಟ್ಟಿಗೆಗಳಂತೆಯೇವೇ?

ಇಲ್ಲ. ಮೇಲ್ ಪೆಟ್ಟಿಗೆಯು ಮೇಲ್ ಅನ್ನು ಸಂಗ್ರಹಿಸುತ್ತದೆ ಮತ್ತು ಲಾಗಿನ್ credentials ಹೊಂದಿರುತ್ತದೆ. ಫಾರ್ವರ್ಡರ್ ಒಂದು ವಿಳಾಸವನ್ನು ಇನ್ನೊಂದು ವಿಳಾಸಕ್ಕೆ ಮರುನಿರ್ದೇಶಿಸುತ್ತದೆ ಮತ್ತು ಪ್ರತ್ಯೇಕ ಇನ್‌ಬಾಕ್ಸ್ ನೀಡುವುದಿಲ್ಲ.
