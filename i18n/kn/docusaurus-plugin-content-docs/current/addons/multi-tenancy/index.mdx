# Multi-Tenancy

ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ಭದ್ರತೆಗಾಗಿ ಉಪಸೈಟ್‌ಗಳನ್ನು ಸರ್ವರ್‌ಗಳು ಮತ್ತು ಡೇಟಾಬೇಸ್‌ಗಳಾದ್ಯಂತ ಹಂಚಿ

## Version 1.2.0 ಬಿಡುಗಡೆ ಮುಖ್ಯಾಂಶಗಳು

Multi-Tenancy 1.2.0 ಉತ್ಪಾದನಾ ಪ್ರತ್ಯೇಕತಾ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ sovereign tenant ಅಡಿಪಾಯವನ್ನು ಸೇರಿಸುತ್ತದೆ:

- ಅಲ್ಪಾವಧಿಯ, ಮರುಪ್ರಯೋಗ-ರಕ್ಷಿತ SSO ಭೇಟಿಗಳೊಂದಿಗೆ ಸ್ಥಿತಿರಹಿತ tenant autologin.
- sovereign tenants ಗಾಗಿ ಪ್ರತಿ-ಉಪಸೈಟ್ ಡೇಟಾಬೇಸ್ ಮತ್ತು ಫೈಲ್‌ಸಿಸ್ಟಮ್ ಪ್ರತ್ಯೇಕತೆ.
- host-ಅರಿವುಳ್ಳ tenant provisioning ಗಾಗಿ Hostinger multi-tenancy ಸಾಮರ್ಥ್ಯ.
- ಹಳೆಯ ಡೇಟಾ ಪರಿಶೀಲನೆಗಳು ಮತ್ತು async push ಪ್ರಕ್ರಿಯೆಗಾಗಿ sovereign migration ಪರಿಶೀಲನಾ ಗೇಟ್‌ಗಳು.
- Bedrock, FrankenPHP, ಮತ್ತು localhost ಸ್ಥಾಪನೆಗಳಿಗಾಗಿ ಸಂರಚಿಸಬಹುದಾದ ಅದೇ-ಯಂತ್ರದ ಡೇಟಾಬೇಸ್ host binding.
- Tenant bootstrap, ಡೇಟಾಬೇಸ್ writer, schema bootstrap, ಮತ್ತು ಬಳಕೆದಾರ provisioning ಸುಧಾರಣೆಗಳು.
- tenant credentials ಅನ್ನು ಸ್ವಚ್ಛಗೊಳಿಸುವ ಹೆಚ್ಚು ಸುರಕ್ಷಿತ sovereign teardown.

[sovereign migration workflow](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy isolation](/user-guide/configuration/multi-tenancy-isolation), ಮತ್ತು [sovereign tenant operations](/user-guide/administration/sovereign-tenant-operations) ಬಳಕೆದಾರ ಮಾರ್ಗದರ್ಶಿಗಳನ್ನು ನೋಡಿ.

## ಅವಶ್ಯಕತೆಗಳು

- WordPress 5.3 ಅಥವಾ ಹೆಚ್ಚಿನದು
- PHP 7.4 ಅಥವಾ ಹೆಚ್ಚಿನದು
- Multisite Ultimate ಪ್ಲಗಿನ್ (ಸಕ್ರಿಯ)

## ಸ್ಥಾಪನೆ

1. ಆಡ್‌ಆನ್ ಫೈಲ್‌ಗಳನ್ನು ನಿಮ್ಮ `/wp-content/plugins/` ಡೈರೆಕ್ಟರಿಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ
2. WordPress ನಲ್ಲಿ 'Plugins' ಮೆನು ಮೂಲಕ ಪ್ಲಗಿನ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
3. ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕದಲ್ಲಿ ಆಡ್‌ಆನ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸಂರಚಿಸಿ

### ಪರೀಕ್ಷೆ

ಆಡ್‌ಆನ್ ಸಮಗ್ರ ಪರೀಕ್ಷಾ ಚೌಕಟ್ಟನ್ನು ಒಳಗೊಂಡಿದೆ:

- **Unit Tests**: WordPress test suite ಜೊತೆಗೆ PHPUnit-ಆಧಾರಿತ tests
- **ಮೂಲ Test ವರ್ಗ**: ಸಾಮಾನ್ಯ test ಸಂದರ್ಭಗಳಿಗಾಗಿ ಸಹಾಯಕ ವಿಧಾನಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ
- **Test Coverage**: code coverage ಅನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಲು ಸಂರಚಿಸಲಾಗಿದೆ
- **Multisite Testing**: tests multisite ಪರಿಸರದಲ್ಲಿ ಚಲಿಸುತ್ತವೆ

### Code Standards

- **WordPress Coding Standards**: PHPCS ಮೂಲಕ ಜಾರಿಗೊಳಿಸಲಾಗಿದೆ
- **PHP 7.4+ ಹೊಂದಾಣಿಕೆ**: ಆಧುನಿಕ PHP ವೈಶಿಷ್ಟ್ಯಗಳಿಗೆ ಬೆಂಬಲವಿದೆ
- **Static Analysis**: type checking ಗಾಗಿ PHPStan ಏಕೀಕರಣ
- **Code Modernization**: PHP ಅಪ್‌ಗ್ರೇಡ್‌ಗಳಿಗಾಗಿ Rector ನಿಯಮಗಳು

### Build System

- **Asset Processing**: CSS/JS ಫೈಲ್‌ಗಳಿಗೆ minification
- **ಅನುವಾದ**: POT ಫೈಲ್ ರಚನೆ
- **Packaging**: ವಿತರಣೆಗಾಗಿ archive ರಚನೆ
- **Development Mode**: debugging ಗಾಗಿ unminified assets

## ಸಂರಚನೆ

ಆಡ್‌ಆನ್ ನೆಟ್‌ವರ್ಕ್ ನಿರ್ವಾಹಕ ಇಂಟರ್‌ಫೇಸ್ ಮೂಲಕ ಪ್ರವೇಶಿಸಬಹುದಾದ ವಿವಿಧ ಸಂರಚನಾ ಆಯ್ಕೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.

### Sovereign tenant ಸಂರಚನೆ

sovereign tenants ಗಾಗಿ, tenant ಗೆ ಟ್ರಾಫಿಕ್ route ಮಾಡುವ ಮೊದಲು tenant ಡೇಟಾಬೇಸ್, ಫೈಲ್‌ಸಿಸ್ಟಮ್ root, database host binding, ಮತ್ತು host ಸಾಮರ್ಥ್ಯವನ್ನು ಸಂರಚಿಸಿ. database credentials, host binding, routing, ಅಥವಾ user provisioning ಮೇಲೆ ಪರಿಣಾಮ ಬೀರುವ ಪ್ರತಿಯೊಂದು ಬದಲಾವಣೆಯ ನಂತರ migration verification ಅನ್ನು ಚಲಾಯಿಸಿ.

tenant runtime ಮತ್ತು database grants ಸ್ಥಳೀಯ socket ಅಥವಾ same-machine ಪ್ರವೇಶವನ್ನು ನಿರೀಕ್ಷಿಸಿದಾಗ ಮಾತ್ರ `localhost` ಬಳಸಿ. ನಿಮ್ಮ provider ಸಂರಚಿಸಿರುವ grant host ಅದೇ ಆಗಿದ್ದರೆ `127.0.0.1` ಅಥವಾ service hostname ಬಳಸಿ.

### Hostinger ಸಾಮರ್ಥ್ಯ

Hostinger multi-tenancy ಸಾಮರ್ಥ್ಯವು ಹಂಚಿಕೆಯ Hostinger integration token ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಪರಿಶೀಲನೆಗೂ ಮೊದಲು domain hPanel ನಲ್ಲಿ ಇದೆ, DNS Hostinger account ಕಡೆ ಸೂಚಿಸುತ್ತದೆ, ಮತ್ತು tenant database settings hPanel ಗೆ ಹೊಂದಿಕೆಯಾಗಿವೆ ಎಂಬುದನ್ನು ದೃಢಪಡಿಸಿ.

## Hooks ಮತ್ತು Filters

### Actions

- `ultimate-multisite-addon-template_init` - ಆಡ್‌ಆನ್ ಆರಂಭೀಕರಣದ ನಂತರ fire ಆಗುತ್ತದೆ
- `ultimate-multisite-addon-template_loaded` - ಎಲ್ಲಾ ಆಡ್‌ಆನ್ ಫೈಲ್‌ಗಳು ಲೋಡ್ ಆದ ನಂತರ fire ಆಗುತ್ತದೆ

### Filters

- `ultimate-multisite-addon-template_settings` - ಆಡ್‌ಆನ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಬದಲಿಸಿ
- `ultimate-multisite-addon-template_enabled` - ಆಡ್‌ಆನ್ enable/disable ಸ್ಥಿತಿಯನ್ನು override ಮಾಡಿ
