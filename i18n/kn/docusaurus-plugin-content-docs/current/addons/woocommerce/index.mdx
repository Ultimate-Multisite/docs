# WooCommerce ಸಂಯೋಜನೆ {#woocommerce-integration}

WooCommerce ಸಂಯೋಜನೆ ನಮ್ಮ ಕೈಯಾರೆ ಗೇಟ್‌ವೇ ಕೆಲಸ ಮಾಡುವ ರೀತಿಗೆ ಬಹಳ ಹೋಲಿಕೆಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಇದು ನಿಮ್ಮ ಬಳಕೆದಾರರಿಗೆ ಹೆಚ್ಚಿನ ಲವಚಿಕತೆಯನ್ನು ನೀಡುತ್ತದೆ. ನಿಮ್ಮ WooCommerce ಸ್ಥಾಪನೆಯಲ್ಲಿ ಲಭ್ಯವಿರುವ ಯಾವುದೇ ಪಾವತಿ ಫಾರ್ಮ್‌ಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಅವರಿಗೆ ಅವಕಾಶ ಕೊಡುತ್ತದೆ. ಮತ್ತು ಇದು ಸರಳ: ಬಳಕೆದಾರರ ಬಿಲ್ಲಿಂಗ್ ಅವಧಿ ಮುಗಿದ ನಂತರ, ಬಳಕೆದಾರರ ಚಂದಾದಾರಿಕೆಯನ್ನು ತಡೆಹಿಡಿಯಲಾಗುತ್ತದೆ. ಅದೇ ಸಮಯದಲ್ಲಿ, ನೆಟ್‌ವರ್ಕ್‌ನ ಮುಖ್ಯ ಸೈಟ್‌ನ WooCommerce ಸ್ಥಾಪನೆಯಲ್ಲಿ ಒಂದು ಆರ್ಡರ್ ರಚಿಸಲಾಗುತ್ತದೆ. ನಂತರ ಬಳಕೆದಾರರು ತಮ್ಮ ಇಮೇಲ್‌ನಲ್ಲಿ ಪಾವತಿ ಮಾಡಲು ಲಿಂಕ್ ಇರುವ ಇನ್‌ವಾಯ್ಸ್ ಪಡೆಯುತ್ತಾರೆ. ಪಾವತಿ ಸ್ವೀಕರಿಸಿದ ನಂತರ, ಚಂದಾದಾರಿಕೆಯನ್ನು ನವೀಕರಿಸಲಾಗುತ್ತದೆ.

## ವೈಶಿಷ್ಟ್ಯಗಳು {#features}

- Ultimate Multisite ಮತ್ತು WooCommerce ನಡುವೆ ಸುಲಭ ಸಂಯೋಜನೆ
- ಹಲವು ಪಾವತಿ ವಿಧಾನಗಳು ಲಭ್ಯ
- ಸೇರಿರುವ ಚಂದಾದಾರಿಕೆಗಳ ಕೋರ್ ಬಳಸಿ ಸ್ವಯಂಚಾಲಿತ ಮರುಕಳಿಸುವ ಬಿಲ್ಲಿಂಗ್, ಐಚ್ಛಿಕ WooCommerce Subscriptions ಪ್ಲಗಿನ್ ಸುಧಾರಣೆಗಳೊಂದಿಗೆ
- Checkout ಒಟ್ಟುಗಳು Ultimate Multisite ತೆರಿಗೆಗಳನ್ನು ಪ್ರತ್ಯೇಕ, ತೆರಿಗೆ ವಿಧಿಸದ WooCommerce ಶುಲ್ಕ ಸಾಲುಗಳಾಗಿ ಒಳಗೊಂಡಿರುತ್ತವೆ. ಹೀಗಾಗಿ ಗ್ರಾಹಕರು ಪಾವತಿಗೆ ಮೊದಲು ತೆರಿಗೆ ಮೊತ್ತಗಳನ್ನು ಪರಿಶೀಲಿಸಬಹುದು

## ಸ್ಥಾಪನೆ {#installation}

1. 'ultimate-multisite-woocommerce' ಅನ್ನು '/wp-content/plugins/' ಡೈರೆಕ್ಟರಿಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ
2. WordPress‌ನ 'Plugins' ಮೆನು ಮೂಲಕ ಪ್ಲಗಿನ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
3. ಪ್ಲಗಿನ್ ಸೆಟ್ ಅಪ್ ಮಾಡಲು ಹಂತ ಹಂತದ ವಿಝಾರ್ಡ್ ಅನುಸರಿಸಿ

## ಬಳಕೆ ಮಾರ್ಗದರ್ಶಿ {#usage-guide}

_**ಪ್ರಮುಖ ಟಿಪ್ಪಣಿ: ಈ ಲೇಖನ Ultimate Multisite ಆವೃತ್ತಿ 2.x ಅನ್ನು ಉಲ್ಲೇಖಿಸುತ್ತದೆ.**_

_**ಗಮನಿಸಿ:** Ultimate Multisite: WooCommerce Integration ಗೆ WooCommerce ಕನಿಷ್ಠ ನಿಮ್ಮ ಮುಖ್ಯ ಸೈಟ್‌ನಲ್ಲಿ ಸಕ್ರಿಯವಾಗಿರಬೇಕು._

ಕೆಲವು ದೇಶಗಳಲ್ಲಿ _Stripe_ ಮತ್ತು _PayPal_ ಲಭ್ಯವಿಲ್ಲ ಎಂಬುದನ್ನು ನಾವು ಅರ್ಥಮಾಡಿಕೊಂಡಿದ್ದೇವೆ. ಇದರಿಂದ Ultimate Multisite ಬಳಕೆದಾರರು ನಮ್ಮ ಪ್ಲಗಿನ್ ಅನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಬಳಸುವುದು ಸೀಮಿತವಾಗಬಹುದು ಅಥವಾ ತಡೆಯಬಹುದು. ಆದ್ದರಿಂದ ನಾವು _WooCommerce_ ಅನ್ನು ಸಂಯೋಜಿಸಲು ಒಂದು **ಆಡ್-ಆನ್ ರಚಿಸಿದ್ದೇವೆ**.

_WooCommerce_ ಬಹಳ ಜನಪ್ರಿಯ ಇ-ಕಾಮರ್ಸ್ ಪ್ಲಗಿನ್ ಆಗಿದೆ. ಜಗತ್ತಿನಾದ್ಯಂತ ಡೆವಲಪರ್‌ಗಳು ವಿವಿಧ ಪಾವತಿ ಗೇಟ್‌ವೇಗಳನ್ನು ಇದಕ್ಕೆ ಸಂಯೋಜಿಸಲು ಆಡ್-ಆನ್‌ಗಳನ್ನು ರಚಿಸಿದ್ದಾರೆ. Ultimate Multisite ಬಿಲ್ಲಿಂಗ್ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ನೀವು ಬಳಸಬಹುದಾದ ಪಾವತಿ ಗೇಟ್‌ವೇಗಳನ್ನು ವಿಸ್ತರಿಸಲು ನಾವು ಇದನ್ನು ಉಪಯೋಗಿಸಿಕೊಂಡಿದ್ದೇವೆ.

### Checkout ತೆರಿಗೆ ಶುಲ್ಕ ಸಾಲುಗಳು {#checkout-tax-fee-lines}

Ultimate Multisite Checkout ತೆರಿಗೆಗಳನ್ನು ಒಳಗೊಂಡಿದ್ದಾಗ, ಸಂಯೋಜನೆಯು ಪ್ರತಿ Ultimate Multisite ತೆರಿಗೆ ಮೊತ್ತವನ್ನು WooCommerce Checkout‌ಗೆ ಪ್ರತ್ಯೇಕ, ತೆರಿಗೆ ವಿಧಿಸದ ಶುಲ್ಕ ಸಾಲಾಗಿ ಸೇರಿಸುತ್ತದೆ. ಇದು ತೆರಿಗೆ ಒಟ್ಟು ಮೊತ್ತವನ್ನು ಪಾವತಿಗೆ ಮೊದಲು ಗೋಚರವಾಗಿರಿಸುತ್ತದೆ ಮತ್ತು WooCommerce ಮತ್ತೆ ಆ ತೆರಿಗೆ ಶುಲ್ಕದ ಮೇಲೆ ತೆರಿಗೆ ವಿಧಿಸುವುದನ್ನು ತಡೆಯುತ್ತದೆ.

## ಪಾವತಿಯನ್ನು ಹೇಗೆ ನಿರ್ವಹಿಸುವುದು {#how-to-manage-the-payment}

WooCommerce ಸಂಯೋಜನೆ ನಮ್ಮ **ಕೈಯಾರೆ ಗೇಟ್‌ವೇ** ಕೆಲಸ ಮಾಡುವ ರೀತಿಗೆ ಬಹಳ ಹೋಲಿಕೆಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ನ್ಯೂನತೆ ಎಂದರೆ ಪ್ರತಿ ಪಾವತಿಯನ್ನು ಬಿಲ್ಲಿಂಗ್ ಅವಧಿಯ ಕೊನೆಯಲ್ಲಿ ಕೈಯಾರೆ ಪಾವತಿಸಬೇಕು. ಹರಿವು ಹೀಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ:

  * ಬಳಕೆದಾರರ ಬಿಲ್ಲಿಂಗ್ ಚಕ್ರ ಮುಗಿದ ನಂತರ, ಸದಸ್ಯತ್ವವನ್ನು **ತಡೆಹಿಡಿಯಲಾಗುತ್ತದೆ**. ಅದೇ ಸಮಯದಲ್ಲಿ, ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್‌ನ ಮುಖ್ಯ ಸೈಟ್‌ನ _WooCommerce_ ಸ್ಥಾಪನೆಯಲ್ಲಿ ಒಂದು ಆರ್ಡರ್ ರಚಿಸಲಾಗುತ್ತದೆ.

  * ನಂತರ ಬಳಕೆದಾರರು ತಮ್ಮ ಇಮೇಲ್‌ನಲ್ಲಿ ಪಾವತಿ ಮಾಡಲು ಲಿಂಕ್ ಇರುವ ಇನ್‌ವಾಯ್ಸ್ ಪಡೆಯುತ್ತಾರೆ. ಆ ಲಿಂಕ್ ಅವರ ಉಪಸೈಟ್ Dashboard‌ನ ಬಿಲ್ಲಿಂಗ್ ಇತಿಹಾಸದಲ್ಲಿಯೂ ಕಾಣಿಸುತ್ತದೆ. ಆ ಲಿಂಕ್ ಬಳಕೆದಾರರನ್ನು _**WooCommerce**_ **Checkout** ಗೆ ಮರುನಿರ್ದೇಶಿಸುತ್ತದೆ. ಅಲ್ಲಿ ಬಳಕೆದಾರರು ತಮ್ಮ ಪಾವತಿಯನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಲು **ಲಭ್ಯವಿರುವ _WooCommerce_ ಗೇಟ್‌ವೇಗಳಲ್ಲಿ** ಒಂದನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು.

  * ಪಾವತಿ ಸ್ವೀಕರಿಸಿದ ನಂತರ, ಸದಸ್ಯತ್ವವನ್ನು **ನವೀಕರಿಸಲಾಗುತ್ತದೆ**, ಪಾವತಿ Ultimate Multisite ನಲ್ಲಿ ದಾಖಲಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಆರ್ಡರ್ ಅನ್ನು ಪೂರ್ಣಗೊಂಡಂತಾಗಿಸಲಾಗುತ್ತದೆ.

### ಕೆಲವು ಪ್ರಮುಖ ಟಿಪ್ಪಣಿಗಳು: {#some-important-notes}

  * Ultimate Multisite‌ನ ಡಿಫಾಲ್ಟ್ ಕಾರ್ಯಕ್ಷಮತೆ ಈ ಸಂಯೋಜನೆ ರೂಪದೊಂದಿಗೆ ಇನ್ನೂ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಉದಾಹರಣೆಗೆ, **ಚಂದಾದಾರಿಕೆ ನಿರ್ವಹಣೆ** ಪರದೆಯಿಂದ **ಮರುಪಾವತಿಗಳನ್ನು ನೀಡುವುದು** ಆಯ್ಕೆಮಾಡಿದ ಪಾವತಿ ಫಾರ್ಮ್ ಬಳಸಿ ಮರುಪಾವತಿ ನೀಡಲು WooCommerce ಜೊತೆ ಸಂವಹನ ಮಾಡುತ್ತದೆ.

  * Ultimate Multisite: WooCommerce Integration ಕೆಲಸ ಮಾಡಲು WP-Cron ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿದೆ. ಇದು ಹೆಚ್ಚಿನ ಬಾರಿ ಚೆನ್ನಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಆದರೆ ಸೈಟ್‌ನಲ್ಲಿ ಕಡಿಮೆ ಟ್ರಾಫಿಕ್ ಇದ್ದರೆ, ನಿಯೋಜಿತ ಈವೆಂಟ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಲು ಕ್ರಾನ್ ಪ್ರೇರಿತವಾಗುವುದಿಲ್ಲ ಮತ್ತು ಈವೆಂಟ್ ತಪ್ಪಿಹೋಗುತ್ತದೆ. ಇದರಿಂದ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆಯಲ್ಲಿ ವಿಳಂಬಗಳು ಉಂಟಾಗಬಹುದು.

## WooCommerce Subscriptions ಪ್ಲಗಿನ್ {#woocommerce-subscriptions-plugin}

ಪ್ರತಿ ತಿಂಗಳು ಪಾವತಿಯನ್ನು ಕೈಯಾರೆ ಪ್ರಾರಂಭಿಸುವುದು ಅಂತಿಮ ಬಳಕೆದಾರರಿಗೆ ಸೂಕ್ತವಲ್ಲ, ಏಕೆಂದರೆ subscriptionಗಳು ಪ್ರತಿ billing cycleನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಮರುಕಳಿಸುವುದಾಗಿ ನಿರೀಕ್ಷಿಸಲಾಗುತ್ತದೆ. WooCommerce Integration add-on ಈಗ core subscriptions code ಅನ್ನು ಒಳಗೊಂಡಿದೆ, ಆದ್ದರಿಂದ recurring products ಗಳು recurring billing ಗಾಗಿ ಯಾವಾಗಲೂ subscriptions ಅನ್ನು ಬಳಸುತ್ತವೆ.

ಈ integration ಮೂಲಕ recurring products ಮಾರಾಟ ಮಾಡಲು ಪ್ರತ್ಯೇಕ [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) ಅನ್ನು ಖರೀದಿಸುವ ಅಥವಾ install ಮಾಡುವ ಅಗತ್ಯ **ಇಲ್ಲ**. ಆ plugin ಅನ್ನು ಖರೀದಿಸಿ, install ಮಾಡಿ, ನಿಮ್ಮ ಮುಖ್ಯ siteನಲ್ಲಿ ಲಭ್ಯವಿದ್ದರೆ, Ultimate Multisite ಅದನ್ನು ಬಳಸುತ್ತದೆ ಮತ್ತು WooCommerce ಹೆಚ್ಚುವರಿ subscription management features ಅನ್ನು ತೋರಿಸುತ್ತದೆ. ಅದು ಲಭ್ಯವಿರದಾಗ, ಒಳಗೊಂಡಿರುವ subscriptions core recurring products ಅನ್ನು ಇನ್ನೂ ನಿರ್ವಹಿಸುತ್ತದೆ.

![WooCommerce subscription integration settings](/img/addons/woocommerce-subscription-integration.png)

ಅಷ್ಟೇ. ನಿಮ್ಮ ಅಂತಿಮ ಬಳಕೆದಾರರು recurring products ಗಾಗಿ ತಮ್ಮ billing cycle ಅಂತ್ಯದಲ್ಲಿ ಇನ್ನು ಮುಂದೆ ಕೈಯಾರೆ ಪಾವತಿ ಮಾಡುವ ಅಗತ್ಯವಿರುವುದಿಲ್ಲ.

ನಿಮ್ಮ ಮುಖ್ಯ siteನಲ್ಲಿ _WooCommerce_ ಅಡಿಯಲ್ಲಿ order ರಚನೆಯಾಗುವುದರ ಜೊತೆಗೆ, ಪ್ರತಿಯೊಂದು Account ಗೆ subscription ಕೂಡ ರಚಿಸಲಾಗುತ್ತದೆ.

### ವಿಫಲ renewal suspension ವರ್ತನೆ {#failed-renewal-suspension-behavior}

ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ, WooCommerce Subscriptions ತನ್ನ failed-payment retry schedule ಅನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸುವಾಗ memberships ಸಕ್ರಿಯವಾಗಿಯೇ ಇರುತ್ತವೆ. ಇದರಿಂದ ಸಾಮಾನ್ಯ WooCommerce Subscriptions grace window ಉಳಿಯುತ್ತದೆ, ಆದ್ದರಿಂದ ಮರಳಿ ಯಶಸ್ವಿಯಾದ renewal subscription ಮತ್ತು membership ಅನ್ನು ಪ್ರವೇಶವನ್ನು ಮುಂಚಿತವಾಗಿ ಅಡ್ಡಿಪಡಿಸದೆ ಮತ್ತೆ sync ಗೆ ತರಬಹುದು.

ಹೆಚ್ಚು ಕಟ್ಟುನಿಟ್ಟಿನ access control ಬಯಸುವ administrators ಗಳು WooCommerce Integration settings ನಲ್ಲಿ **Suspend Memberships Immediately on Failed Renewals** ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಬಹುದು. ಈ opt-in setting ಸಕ್ರಿಯವಾಗಿರುವಾಗ, WooCommerce Subscriptions retry handling ಮುಗಿಯುವವರೆಗೆ ಕಾಯುವುದಕ್ಕಿಂತ, renewal payment ವಿಫಲವಾದ ತಕ್ಷಣ Ultimate Multisite ಸಂಬಂಧಿತ membership ಅನ್ನು suspend ಮಾಡುತ್ತದೆ.

Developers ಗಳು ಇದೇ ನಿರ್ಧಾರವನ್ನು `wu_woo_suspend_on_payment_failure` filter ಮೂಲಕ override ಮಾಡಬಹುದು. Hook reference pages add-on source ನಿಂದ ರಚಿಸಲ್ಪಡುತ್ತವೆ, ಆದ್ದರಿಂದ ಈ overview custom suspension policies ಗಾಗಿ filter ಹೆಸರನ್ನು ಮಾತ್ರ ಸೂಚಿಸುತ್ತದೆ.
