# ಲೋಗೋ SVG ರಚಿಸಿ (Generate Logo SVG)

**Generate Logo SVG** ಸಾಮರ್ಥ್ಯವು (ability) ಥೀಮ್ ಬಿಲ್ಡರ್‌ಗೆ ನಿಮ್ಮ WordPress ಸೈಟ್‌ಗೆ ಕಸ್ಟಮ್ ಲೋಗೋ SVG ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತ namespace-safe ಶುದ್ಧೀಕರಣದೊಂದಿಗೆ (sanitisation) ರಚಿಸಲು ಮತ್ತು ಅಳವಡಿಸಲು ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ.

## ಅವಲೋಕನ (Overview)

ಈ ಸಾಮರ್ಥ್ಯವು ನಿಮ್ಮ ಸೈಟ್‌ನ ಬ್ರ್ಯಾಂಡಿಂಗ್ ದಿಕ್ಕು ಮತ್ತು ವಿನ್ಯಾಸ ಆದ್ಯತೆಗಳ ಆಧಾರದ ಮೇಲೆ ವಿಸ್ತರಿಸಬಹುದಾದ ವೆಕ್ಟರ್ ಗ್ರಾಫಿಕ್ಸ್ (SVG) ಲೋಗೋಗಳನ್ನು ರಚಿಸುತ್ತದೆ. ರಚಿಸಲಾದ SVG ಗಳನ್ನು WordPress ನಲ್ಲಿ ಬಳಸಲು ಸುರಕ್ಷಿತವಾಗಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು, ದೃಶ್ಯ ಸಮಗ್ರತೆಯನ್ನು (visual integrity) ಕಾಪಾಡಿಕೊಳ್ಳುವ ಮೂಲಕ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಶುದ್ಧೀಕರಿಸಲಾಗುತ್ತದೆ.

## ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು (Parameters)

| ಪ್ಯಾರಾಮೀಟರ್ (Parameter) | ವಿಧ (Type) | ಅಗತ್ಯವೇ (Required) | ವಿವರಣೆ (Description) |
|-----------|------|----------|-------------|
| `site_name` | string | Yes | ಲೋಗೋವನ್ನು ರಚಿಸಲು ಬೇಕಾದ ಸೈಟ್‌ನ ಹೆಸರು |
| `style` | string | Yes | ವಿನ್ಯಾಸ ಶೈಲಿ (ಉದಾಹರಣೆಗೆ, "modern", "classic", "minimalist", "playful") |
| `colors` | array | No | ಲೋಗೋದಲ್ಲಿ ಬಳಸಬೇಕಾದ ಆಕ್ಸಾಂಕ (hex) ಬಣ್ಣ ಕೋಡ್‌ಗಳ ಅರೇ (ಉದಾ: `["#678233", "#ffffff"]`) |
| `width` | number | No | ಪಿಕ್ಸೆಲ್‌ಗಳಲ್ಲಿ SVG ಅಗಲ (default: 200) |
| `height` | number | No | ಪಿಕ್ಸೆಲ್‌ಗಳಲ್ಲಿ SVG ಎತ್ತರ (default: 200) |
| `include_text` | boolean | No | ಲೋಗೋದಲ್ಲಿ ಸೈಟ್ ಹೆಸರನ್ನು ಪಠ್ಯವಾಗಿ ಸೇರಿಸಬೇಕೇ (default: true) |

## ಔಟ್‌ಪುಟ್ ಫಾರ್ಮ್ಯಾಟ್ (Output Format)

ಈ ಸಾಮರ್ಥ್ಯವು ಈ ಕೆಳಗಿನ ರಚನೆಯೊಂದಿಗೆ SVG ಸ್ಟ್ರಿಂಗ್ ಅನ್ನು ಮರಳಿ ನೀಡುತ್ತದೆ:

```json
{
  "svg": "<svg xmlns=\"http://www.w3.org/2000/svg\" viewBox=\"0 0 200 200\">...</svg>",
  "width": 200,
  "height": 200,
  "colors_used": ["#678233", "#ffffff"],
  "sanitised": true
}
```

## SVG ಶುದ್ಧೀಕರಣದ ವರ್ತನೆ (SVG Sanitisation Behaviour)

ರಚಿಸಲಾದ SVG ಗಳು ಸ್ವಯಂಚಾಲಿತ namespace-safe ಶುದ್ಧೀಕರಣಕ್ಕೆ ಒಳಗಾಗುತ್ತವೆ. ಇದರ ಉದ್ದೇಶಗಳು:

- **ಅಸುರಕ್ಷಿತ ಗುಣಲಕ್ಷಣಗಳನ್ನು ತೆಗೆದುಹಾಕುವುದು (Remove unsafe attributes)** — ಇವೆಂಟ್ ಹ್ಯಾಂಡಲರ್‌ಗಳು, ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು ಮತ್ತು ಸಂಭಾವ್ಯವಾಗಿ ಅಪಾಯಕಾರಿ ಗುಣಲಕ್ಷಣಗಳನ್ನು ತೆಗೆದುಹಾಕುತ್ತದೆ.
- **namespace ಗಳನ್ನು ಉಳಿಸುವುದು (Preserve namespaces)** — ಸರಿಯಾದ ರೆಂಡರಿಂಗ್‌ಗಾಗಿ SVG namespace ಗಳನ್ನು (xmlns, xlink) ಕಾಪಾಡುತ್ತದೆ.
- **ರಚನೆಯನ್ನು ಪರಿಶೀಲಿಸುವುದು (Validate structure)** — SVG သည် W3C ಮಾನದಂಡಗಳಿಗೆ ಅನುಗುಣವಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ.
- **ಎಂಟಿಟೀಸ್เข้ารหัส ಮಾಡುವುದು (Encode entities)** — ಪಠ್ಯ ವಿಷಯದಲ್ಲಿನ ವಿಶೇಷ ಅಕ್ಷರಗಳನ್ನು ಸರಿಯಾಗಿ ತಪ್ಪಿಸುತ್ತದೆ (escape).
- **ಬಾಹ್ಯ ಉಲ್ಲೇಖಗಳನ್ನು ತೆಗೆದುಹಾಕುವುದು (Remove external references)** — ಬಾಹ್ಯ ಶೈಲಿ ಹಾಳೆಗಳು (stylesheets) ಮತ್ತು ಚಿತ್ರ ಉಲ್ಲೇಖಗಳನ್ನು ತೆಗೆದುಹಾಕುತ್ತದೆ.

ಇದರಿಂದ SVG ಅನ್ನು ಹೆಚ್ಚುವರಿ ಶುದ್ಧೀಕರಣದ ಅಗತ್ಯವಿಲ್ಲದೆ WordPress ನಲ್ಲಿ ನೇರವಾಗಿ ಅಳವಡಿಸಲು ಸುರಕ್ಷಿತವಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ.

## ಬಳಕೆಯ ಉದಾಹರಣೆ (Usage Example)

**ಪ್ರಾಂಪ್ಟ್ (Prompt):**
```
Generate a modern logo for my tech startup called "CloudSync" using blue and white colors.
```

**ಫಲಿತಾಂಶ (Result):**
ಈ ಸಾಮರ್ಥ್ಯವು ಒಂದು SVG ಲೋಗೋವನ್ನು ರಚಿಸುತ್ತದೆ, ಅದು:
- "CloudSync" ಎಂಬ ಸೈಟ್ ಹೆಸರನ್ನು ಒಳಗೊಂಡಿದೆ
- ನಿರ್ದಿಷ್ಟಪಡಿಸಿದ ನೀಲಿ ಮತ್ತು ಬಿಳಿ ಬಣ್ಣದ ಯೋಜನೆಯನ್ನು ಬಳಸುತ್ತದೆ
- ಆಧುನಿಕ ವಿನ್ಯಾಸ ತತ್ವಗಳನ್ನು ಅನುಸರಿಸುತ್ತದೆ
- ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಶುದ್ಧೀಕರಿಸಲ್ಪಟ್ಟಿದ್ದು, ಬಳಸಲು ಸಿದ್ಧವಾಗಿದೆ

## ಥೀಮ್ ಬಿಲ್ಡರ್‌ನೊಂದಿಗೆ ಏಕೀಕರಣ (Integration with Theme Builder)

ಥೀಮ್ ಬಿಲ್ಡರ್‌ನ ವಿನ್ಯಾಸ-ದಿಕ್ಕು ಆಯ್ಕೆಯನ್ನು ಬಳಸುವಾಗ, Generate Logo SVG ಸಾಮರ್ಥ್ಯವು:

1. ನಿಮ್ಮ ವಿನ್ಯಾಸ ದಿಕ್ಕು ಮತ್ತು ಬಣ್ಣದ ಪ್ಯಾಲೆಟ್ ಅನ್ನು ವಿಶ್ಲೇಷಿಸುತ್ತದೆ
2. ನಿಮ್ಮ ಆದ್ಯತೆಗಳಿಗೆ ಹೊಂದಿಕೆಯಾಗುವ ಕಸ್ಟಮ್ SVG ಲೋಗೋವನ್ನು ರಚಿಸುತ್ತದೆ
3. ಲೋಗೋವನ್ನು ನಿಮ್ಮ ಸೈಟ್‌ನ ಹೆಡರ್/ಬ್ರ್ಯಾಂಡಿಂಗ್ ಪ್ರದೇಶದಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅಳವಡಿಸುತ್ತದೆ
4. SVG ಅನ್ನು WordPress മീഡിയದಲ್ಲಿ ಕಸ್ಟಮ್ ಲೋಗೋವಾಗಿ ಸಂಗ್ರಹಿಸುತ್ತದೆ

## ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು (Best Practices)

- **ಸ್ಪಷ್ಟ ಶೈಲಿ ಆದ್ಯತೆಗಳನ್ನು ನೀಡಿ** — ನೀವು ಬಯಸುವ ವಿನ್ಯಾಸ ಶೈಲಿಯನ್ನು ವಿವರಿಸಿ (ಆಧುನಿಕ, ಶಾಸ್ತ್ರೀಯ, ಆಟದಂತಹ ಇತ್ಯಾದಿ).
- **ಬಣ್ಣಗಳನ್ನು ನಿರ್ದಿಷ್ಟಪಡಿಸಿ** — ಸ್ಥಿರತೆಗಾಗಿ ನಿಮ್ಮ ಬ್ರ್ಯಾಂಡ್ ಬಣ್ಣಗಳನ್ನು ಸೇರಿಸಿ.
- **ರೆಂಡರಿಂಗ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ** — ವಿಭಿನ್ನ ಪರದೆಯ ಗಾತ್ರಗಳಲ್ಲಿ ಲೋಗೋ ಸರಿಯಾಗಿ ಪ್ರದರ್ಶನಗೊಳ್ಳುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ.
- **ಹೆಚ್ಚು ಕಸ್ಟಮೈಸ್ ಮಾಡಿ** — ಗಾತ್ರ ಮತ್ತು placement ಅನ್ನು ಹೊಂದಿಸಲು WordPress ನ ಲೋಗೋ ಕಸ್ಟಮೈಸೇಶನ್ ಟೂಲ್‌ಗಳನ್ನು ಬಳಸಿ.

## ಮಿತಿಗಳು (Limitations)

- SVG ಲೋಗೋಗಳನ್ನು ಸ್ಥಿರ ಗ್ರಾಫಿಕ್ಸ್ (animated ಅಲ್ಲ) ಆಗಿ ರಚಿಸಲಾಗುತ್ತದೆ.
- ಅನೇಕ ಅಂಶಗಳನ್ನು ಹೊಂದಿರುವ ಸಂಕೀರ್ಣ ಲೋಗೋಗಳಿಗೆ ಕೈಯಾರೆ ಪರಿಷ್ಕರಣೆ ಅಗತ್ಯವಾಗಬಹುದು.
- ಕಸ್ಟಮ್ ಫಾಂಟ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸಲಾಗುವುದಿಲ್ಲ; ಪಠ್ಯವು ಸಿಸ್ಟಮ್ ಫಾಂಟ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ.
- ತುಂಬಾ ದೊಡ್ಡ ಅಥವಾ ತುಂಬಾ ಚಿಕ್ಕ dimensions ಗುಣಮಟ್ಟದ ಮೇಲೆ ಪರಿಣಾಮ ಬೀರಬಹುದು.
