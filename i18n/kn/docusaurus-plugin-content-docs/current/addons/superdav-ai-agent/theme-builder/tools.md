# ಟೂಲ್‌ಗಳು ಮತ್ತು ಕಾನ್ಫಿಗರೇಶನ್

Theme Builder ನಿಮ್ಮ WordPress ಸೈಟ್ ಅನ್ನು ವಿಶ್ಲೇಷಿಸಲು, ವಿನ್ಯಾಸಗೊಳಿಸಲು ಮತ್ತು ನಿರ್ಮಿಸಲು ಒಂದು ಸೆಟ್ ಟೂಲ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ. version 1.16.0 ರಲ್ಲಿ, **sd-ai-agent/site-scrape ಈಗ Tier 1 ಟೂಲ್ ಆಗಿದೆ**, ಇದು Theme Builder ನಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಆಗಿ ಲಭ್ಯವಿದೆ.

## ಲಭ್ಯವಿರುವ ಟೂಲ್‌ಗಳು

### Tier 1 ಟೂಲ್‌ಗಳು (ಎಲ್ಲಾ ಸಮಯದಲ್ಲೂ ಲಭ್ಯ)

Tier 1 ಟೂಲ್‌ಗಳು ಯಾವುದೇ ಹೆಚ್ಚುವರಿ ಕಾನ್ಫಿಗರೇಶನ್ ಇಲ್ಲದೆ Theme Builder ನಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಆಗಿ ಲಭ್ಯವಿವೆ.

#### sd-ai-agent/site-scrape

**ಉದ್ದೇಶ:** ವಿನ್ಯಾಸದ ಸ್ಫೂರ್ತಿ, ವಿಷಯದ ರಚನೆ ಮತ್ತು ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಹೊರತೆಗೆಯಲು (extract) ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ವಿಶ್ಲೇಷಿಸುತ್ತದೆ.

**ಸಾಮರ್ಥ್ಯಗಳು:**

- **ವೆಬ್‌ಸೈಟ್ ವಿಶ್ಲೇಷಣೆ** — ಸ್ಪರ್ಧಾ ಅಥವಾ ಸ್ಫೂರ್ತಿ ವೆಬ್‌ಸೈಟ್‌ಗಳಿಂದ ಡೇಟಾ ಸಂಗ್ರಹಿಸಿ ವಿಶ್ಲೇಷಣೆ ಮಾಡುವುದು
- **ವಿನ್ಯಾಸ ಹೊರತೆಗೆಯುವಿಕೆ** — ಬಣ್ಣಗಳು, ಫಾಂಟ್‌ಗಳು ಮತ್ತು ಲೇಔಟ್ ಮಾದರಿಗಳನ್ನು ಗುರುತಿಸುತ್ತದೆ
- **ವಿಷಯದ ರಚನೆ** — ಪುಟದ ಆಯೋಜನೆ ಮತ್ತು ಶ್ರೇಣೀಕರಣವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತದೆ
- **ఫీಚರ್ ಪತ್ತೆ** — ಪ್ಲಗ್ಇನ್‌ಗಳು ಮತ್ತು ಕಾರ್ಯಕ್ಷಮತೆಗಳನ್ನು ಗುರುತಿಸುತ್ತದೆ
- **ಕಾರ್ಯಕ್ಷಮತೆ ವಿಶ್ಲೇಷಣೆ** — ಪುಟದ ವೇಗ ಮತ್ತು ಆಪ್ಟಿಮೈಸೇಶನ್ ಅನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ
- **SEO ವಿಶ್ಲೇಷಣೆ** — ಮೆಟಾ ಟ್ಯಾಗ್‌ಗಳು ಮತ್ತು ರಚನಾತ್ಮಕ ಡೇಟಾವನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ

**ಬಳಕೆ:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**ಇದು ಏನು ಹೊರತೆಗೆಯುತ್ತದೆ:**

- ಬಣ್ಣದ ಪ್ಯಾಲೆಟ್ ಮತ್ತು ಬಣ್ಣದ ಬಳಕೆ
- ಟೈಪೋಗ್ರಫಿ (ಫಾಂಟ್‌ಗಳು ಮತ್ತು ಗಾತ್ರಗಳು)
- ಲೇಔಟ್ ರಚನೆ ಮತ್ತು இடைவெளி (spacing)
- ನ್ಯಾವಿಗೇಷನ್ ಮಾದರಿಗಳು
- ವಿಷಯದ ಆಯೋಜನೆ
- ಚಿತ್ರಗಳು ಮತ್ತು ಮಾಧ್ಯಮದ ಬಳಕೆ
- ಸಂವಾದಾತ್ಮಕ ಅಂಶಗಳು (Interactive elements)
- ಮೊಬೈಲ್ ಪ್ರತಿಕ್ರಿಯೆ (Mobile responsiveness)

**ಮಿತಿಗಳು:**

- ಪಾಸ್‌ವರ್ಡ್-ರಕ್ಷಿತ ಸೈಟ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ
- robots.txt ಮತ್ತು ಸೈಟ್ ನೀತಿಗಳನ್ನು ಗೌರವಿಸುತ್ತದೆ
- ಡೈನಾಮಿಕ್ ವಿಷಯವನ್ನು ಸೆರೆಹಿಡಿಯದಿರಬಹುದು
- JavaScript-ಭಾರವಾದ ಸೈಟ್‌ಗಳಲ್ಲಿ ವಿಶ್ಲೇಷಣೆ ಸೀಮಿತವಾಗಿರಬಹುದು
- ದೊಡ್ಡ ಸೈಟ್‌ಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಲು ಹೆಚ್ಚು ಸಮಯ ಬೇಕಾಗಬಹುದು

### Tier 2 ಟೂಲ್‌ಗಳು (ಐಚ್ಛಿಕ)

Tier 2 ಟೂಲ್‌ಗಳು Theme Builder ಸೆಟ್ಟಿಂಗ್‌ಗಳಲ್ಲಿ ಸ್ಪಷ್ಟವಾಗಿ ಎನಬಿಲ್ ಮಾಡಿದಾಗ ಲಭ್ಯವಿವೆ.

#### Advanced Analytics

ವಿವರವಾದ ಕಾರ್ಯಕ್ಷಮತೆ ಮೆಟ್ರಿಕ್‌ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- ಪುಟದ ಲೋಡ್ ಸಮಯಗಳು
- Core Web Vitals
- SEO ಸ್ಕೋರ್
- Accessibility ಸ್ಕೋರ್
- Best practices ಸ್ಕೋರ್

#### Content Optimizer

ಇದಕ್ಕಾಗಿ ವಿಶ್ಲೇಷಿಸುತ್ತದೆ ಮತ್ತು ಸುಧಾರಣೆಗಳನ್ನು ಸೂಚಿಸುತ್ತದೆ:

- ಓದಲು ಸುಲಭವಾಗುವಿಕೆ (Readability)
- SEO ಆಪ್ಟಿಮೈಸೇಶನ್
- ಕೀವರ್ಡ್ ಬಳಕೆ
- ವಿಷಯದ ರಚನೆ
- ಕರೆ-ಟು-ಆಕ್ಷನ್ (Call-to-action) placement

### Tier 3 ಟೂಲ್‌ಗಳು (ಪ್ರೀಮಿಯಂ)

Tier 3 ಟೂಲ್‌ಗಳಿಗೆ ಹೆಚ್ಚುವರಿ ಕಾನ್ಫಿಗರೇಶನ್ ಅಥವಾ ಪ್ರೀಮಿಯಂ ಫೀಚರ್‌ಗಳು ಬೇಕಾಗುತ್ತವೆ.

#### AI Content Generator

ಇದಕ್ಕಾಗಿ ವಿಷಯವನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ:

- ಉತ್ಪನ್ನ ವಿವರಣೆಗಳು
- ಸೇವಾ ಪುಟಗಳು
- ಬ್ಲಾಗ್ ಪೋಸ್ಟ್‌ಗಳು
- ಮೆಟಾ ವಿವರಣೆಗಳು
- ಕರೆ-ಟು-ಆಕ್ಷನ್ ಕಾಪಿ

#### Advanced Design Tools

ಹೆಚ್ಚುವರಿ ವಿನ್ಯಾಸ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- ಕಸ್ಟಮ್ CSS generation
- ಅನಿಮೇಷನ್ ರಚನೆ
- ಸಂವಾದಾತ್ಮಕ ಅಂಶ ವಿನ್ಯಾಸ
- ಸುಧಾರಿತ ಬಣ್ಣದ ಸಿದ್ಧಾಂತ (Advanced color theory)
- ಟೈಪೋಗ್ರಫಿ ಆಪ್ಟಿಮೈಸೇಶನ್

## ಟೂಲ್ ಕಾನ್ಫಿಗರೇಶನ್

### ಟೂಲ್‌ಗಳನ್ನು ಎನಬಲ್ ಮಾಡುವುದು

Theme Builder ನಲ್ಲಿ ಹೆಚ್ಚುವರಿ ಟೂಲ್‌ಗಳನ್ನು ಎನಬಲ್ ಮಾಡಲು:

1. **Gratis AI Agent → Settings** ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ
2. **Theme Builder → Tools** ಗೆ ಹೋಗಿ
3. ಅಗತ್ಯಕ್ಕೆ ತಕ್ಕಂತೆ ಟೂಲ್‌ಗಳನ್ನು ಆನ್/ಆಫ್ ಮಾಡಿ
4. ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಸೇವ್ ಮಾಡಿ

### ಟೂಲ್ ಅನುಮತಿಗಳು (Permissions)

ಕೆಲವು ಟೂಲ್‌ಗಳಿಗೆ ಅನುಮತಿಗಳು ಬೇಕಾಗುತ್ತವೆ:

- **Site scraping** — ಇಂಟರ್ನೆಟ್ ಪ್ರವೇಶದ ಅಗತ್ಯವಿದೆ
- **Analytics** — Google Analytics integration ಅಗತ್ಯವಿದೆ
- **Content generation** — API keys ಅಗತ್ಯವಿದೆ
- **Advanced features** — ಪ್ರೀಮಿಯಂ subscription ಅಗತ್ಯವಾಗಬಹುದು

### API Keys ಮತ್ತು Credentials

ಅದರ ಅಗತ್ಯವಿರುವ ಟೂಲ್‌ಗಳಿಗೆ API keys ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ:

1. **Gratis AI Agent → Settings → API Keys** ಗೆ ಹೋಗಿ
2. ಪ್ರತಿ ಟೂಲ್‌ಗೆ credentials ನಮೂದಿಸಿ
3. ಸಂಪರ್ಕವನ್ನು ಪರೀಕ್ಷಿಸಿ (Test connection)
4. ಸುರಕ್ಷಿತವಾಗಿ ಸೇವ್ ಮಾಡಿ

## sd-ai-agent/site-scrape ಬಳಸುವುದು

### ಮೂಲ ಬಳಕೆ (Basic Usage)

Theme Builder ಗೆ ಒಂದು ವೆಬ್‌ಸೈಟ್ ಅನ್ನು ವಿಶ್ಲೇಷಿಸಲು ಕೇಳಿ:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### ನಿರ್ದಿಷ್ಟ ವಿಶ್ಲೇಷಣೆ (Specific Analysis)

ನಿರ್ದಿಷ್ಟ ರೀತಿಯ ವಿಶ್ಲೇಷಣೆಯನ್ನು ವಿನಂತಿಸಿ:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### ಹೋಲಿಕೆ ವಿಶ್ಲೇಷಣೆ (Comparison Analysis)

複数の ಸೈಟ್‌ಗಳನ್ನು ಹೋಲಿಕೆ ಮಾಡಿ:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## ಟೂಲ್ ಮಿತಿಗಳು ಮತ್ತು ಪರಿಗಣನೆಗಳು

### Rate Limiting

- ಸರ್ವರ್ ಓವರ್‌ಲೋಡ್ ತಡೆಯಲು ಸ್ಕ್ರಾಪಿಂಗ್ ಅನ್ನು rate-limit ಮಾಡಲಾಗುತ್ತದೆ
- ಒಂದೇ ಸೈಟ್‌ಗೆ ಅನೇಕ ವಿನಂತಿಗಳನ್ನು (requests) ನಿಧಾನಗೊಳಿಸಬಹುದು (throttled)
- ದೊಡ್ಡ ಸೈಟ್‌ಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಲು ಹೆಚ್ಚು ಸಮಯ ಬೇಕಾಗಬಹುದು

### ವಿಷಯದ ನಿಖರತೆ (Content Accuracy)

- ಡೈನಾಮಿಕ್ ವಿಷಯವನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಸೆರೆಹಿಡಿಯದಿರಬಹುದು
- JavaScript-rendered ವಿಷಯ ಅಪೂರ್ಣವಾಗಿರಬಹುದು
- ನೈಜ-ಸಮಯದ ಡೇಟಾವನ್ನು ಪ್ರತಿಬಿಂಬಿಸದಿರಬಹುದು
- ಕೆಲವು ವಿಷಯಗಳು paywalls ಹಿಂದೆ ಇರಬಹುದು

### ಕಾನೂನು ಮತ್ತು ನೈತಿಕ ಪರಿಗಣನೆಗಳು (Legal and Ethical Considerations)

- robots.txt ಮತ್ತು ಸೈಟ್ ನೀತಿಗಳನ್ನು ಗೌರವಿಸಿ
- ಪುನರುತ್ಪಾದನೆಗಾಗಿ कॉपीराइट ಮಾಡಿದ ವಿಷಯವನ್ನು ಸ್ಕ್ರಾಪ್ ಮಾಡಬೇಡಿ
- ವಿಶ್ಲೇಷಣೆಯನ್ನು ಸ್ಫೂರ್ತಿಗಾಗಿ ಬಳಸಿ, ನಕಲು ಮಾಡಲು ಅಲ್ಲ
- ಹೊರತೆಗೆದ ವಿಷಯವನ್ನು ಬಳಸಲು ನಿಮಗೆ ಹಕ್ಕುಗಳು ಇವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ವಿಶ್ಲೇಷಿಸಿದ ಸೈಟ್‌ಗಳ ಸೇವಾ ನಿಯಮಗಳನ್ನು (terms of service) ಪಾಲಿಸಿ

### ಕಾರ್ಯಕ್ಷಮತೆಯ ಪರಿಣಾಮ (Performance Impact)

- ದೊಡ್ಡ ಸೈಟ್‌ಗಳನ್ನು ಸ್ಕ್ರಾಪ್ ಮಾಡಲು ಸಮಯ ಬೇಕಾಗಬಹುದು
- ಅನೇಕ ಏಕಕಾಲೀನ ಸ್ಕ್ರಾಪ್‌ಗಳು ನಿಧಾನವಾಗಿರುತ್ತವೆ
- ನೆಟ್‌ವರ್ಕ್ ಸಂಪರ್ಕ ವೇಗವನ್ನು ಪ್ರಭಾವಿಸುತ್ತದೆ
- ದೊಡ್ಡ ಡೇಟಾಸೆಟ್‌ಗಳಿಗೆ ಹೆಚ್ಚು ಪ್ರೊಸೆಸಿಂಗ್ ಬೇಕಾಗಬಹುದು

## ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು (Best Practices)

### ಸೈಟ್ ವಿಶ್ಲೇಷಣೆಯನ್ನು ಬಳಸುವುದು

1. **ಅನೇಕ ಸೈಟ್‌ಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಿ** — ಹಲವಾರು ಮೂಲಗಳಿಂದ ಸ್ಫೂರ್ತಿ ಸಂಗ್ರಹಿಸಿ
2. **ರಚನೆಯ ಮೇಲೆ ಗಮನ ನೀಡಿ** — ಲೇಔಟ್ ಮತ್ತು ಆಯೋಜನಾ ಮಾದರಿಗಳನ್ನು ಕಲಿಯಿರಿ
3. **ಬಣ್ಣಗಳನ್ನು ಹೊರತೆಗೆಯಿರಿ** — ಬಣ್ಣದ ಪ್ಯಾಲೆಟ್‌ಗಳನ್ನು ಆರಂಭಿಕ ಬಿಂದುಗಳಾಗಿ ಬಳಸಿ
4. **ಟೈಪೋಗ್ರಫಿಯನ್ನು ಅಧ್ಯಯನ ಮಾಡಿ** — ನಿಮಗೆ ಇಷ್ಟವಾದ ಫಾಂಟ್ ಸಂಯೋಜನೆಗಳನ್ನು ಗುರುತಿಸಿ
5. **ನ್ಯಾವಿಗೇಷನ್ ಪರಿಶೀಲಿಸಿ** — ಮೆನು ಆಯೋಜನೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

### ನೈತಿಕ ಸ್ಕ್ರಾಪಿಂಗ್

1. **robots.txt ಗೌರವಿಸಿ** — ಸೈಟ್ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಪಾಲಿಸಿ
2. **ವಿಷಯವನ್ನು ನಕಲು ಮಾಡಬೇಡಿ** — ಸ್ಫೂರ್ತಿಗಾಗಿ ವಿಶ್ಲೇಷಣೆಯನ್ನು ಬಳಸಿ ಮಾತ್ರ
3. **ಹಕ್ಕುಗಳನ್ನು ಪರಿಶೀಲಿಸಿ** — ಹೊರತೆಗೆದ ವಿಷಯವನ್ನು ಬಳಸಲು ನಿಮಗೆ ಸಾಧ್ಯ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
4. **ಮೂಲಗಳನ್ನು ನಮೂದಿಸಿ** — ಸ್ಫೂರ್ತಿ ಮೂಲಗಳಿಗೆ ಕ್ರೆಡಿಟ್ ನೀಡಿ
5. **ನಿಯಮಗಳನ್ನು ಪಾಲಿಸಿ** — ಸೈಟ್ ಸೇವಾ ನಿಯಮಗಳಿಗೆ ಬದ್ಧರಾಗಿರಿ

### ಟೂಲ್ ಪರಿಣಾಮಕಾರಿತ್ವವನ್ನು ಹೆಚ್ಚಿಸುವುದು

1. **ನಿರ್ದಿಷ್ಟವಾಗಿರಿ** — ನಿರ್ದಿಷ್ಟ ರೀತಿಯ ವಿಶ್ಲೇಷಣೆಯನ್ನು ವಿನಂತಿಸಿ
2. **ಸಂದರ್ಭ ನೀಡಿ** — ನಿಮ್ಮ ಸೈಟ್‌ನ ಉದ್ದೇಶವನ್ನು ವಿವರಿಸಿ
3. **ನಿರೀಕ್ಷೆಗಳನ್ನು ನಿಗದಿಪಡಿಸಿ** — ನಿಮ್ಮ ವಿನ್ಯಾಸ ಗುರಿಗಳನ್ನು ವಿವರಿಸಿ
4. **ಪುನರಾವರ್ತಿಸಿ (Iterate)** — ಫಲಿತಾಂಶಗಳ ಆಧಾರದ ಮೇಲೆ ಪರಿಷ್ಕರಿಸಿ
5. **ಟೂಲ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸಿ** — ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆಗಾಗಿ ಅನೇಕ ಟೂಲ್‌ಗಳನ್ನು ಬಳಸಿ

## ಟ್ರಬಲ್ಶೂಟಿಂಗ್ (Troubleshooting)

### ಸೈಟ್ ಸ್ಕ್ರಾಪ್ ಆಗುತ್ತಿಲ್ಲ

- ಸೈಟ್ ಸಾರ್ವಜನಿಕವಾಗಿ ಲಭ್ಯವಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಸ್ಕ್ರಾಪಿಂಗ್ ಮಾಡಲು robots.txt ಅನುಮತಿಸುತ್ತದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- ಬೇರೆ ಸೈಟ್ ಅನ್ನು ಪ್ರಯತ್ನಿಸಿ
- ಇಂಟರ್ನೆಟ್ ಸಂಪರ್ಕವನ್ನು ಪರಿಶೀಲಿಸಿ
- ಸಮಸ್ಯೆ ಮುಂದುವರಿದರೆ サಪೋರ್ಟ್‌ಗೆ ಸಂಪರ್ಕಿಸಿ

### ಅಪೂರ್ಣ ವಿಶ್ಲೇಷಣೆ (Incomplete Analysis)

- ಸೈಟ್‌ನಲ್ಲಿ ಡೈನಾಮಿಕ್ ವಿಷಯವಿರಬಹುದು
- JavaScript ಸಂಪೂರ್ಣವಾಗಿ ರೆಂಡರ್ ಆಗದಿರಬಹುದು
- ದೊಡ್ಡ ಸೈಟ್‌ಗಳು ಟೈಮೌಟ್ ಆಗಬಹುದು
- ಬದಲಾಗಿ ನಿರ್ದಿಷ್ಟ ಪುಟಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಲು ಪ್ರಯತ್ನಿಸಿ
- ನಿರ್ದಿಷ್ಟ ವಿಶ್ಲೇಷಣೆ ಪ್ರಕಾರಗಳನ್ನು ವಿನಂತಿಸಿ

### ಕಡಿಮೆ ಕಾರ್ಯಕ್ಷಮತೆ (Slow Performance)

- ದೊಡ್ಡ ಸೈಟ್‌ಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಲು ಹೆಚ್ಚು ಸಮಯ ಬೇಕಾಗುತ್ತದೆ
- ಅನೇಕ ಏಕಕಾಲೀನ ಸ್ಕ್ರಾಪ್‌ಗಳು ನಿಧಾನವಾಗಿರುತ್ತವೆ
- ನೆಟ್‌ವರ್ಕ್ ಸಂಪರ್ಕ ವೇಗವನ್ನು ಪ್ರಭಾವಿಸುತ್ತದೆ
- ಕಡಿಮೆ ಬಿಡುವಿನ ಸಮಯದಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ
- ಮೊದಲು ಚಿಕ್ಕ ವಿಭಾಗಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಿ
