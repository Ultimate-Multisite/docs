# Multi-Network

Ultimate Multisite: Multinetwork ಆಡ್-ಆನ್ ನಿಮ್ಮ Ultimate Multisite ವೇದಿಕೆಯನ್ನು ಶಕ್ತಿಯುತವಾದ multinetwork ಕಾರ್ಯಕ್ಷಮತೆಯೊಂದಿಗೆ ವಿಸ್ತರಿಸುತ್ತದೆ. ಈ ಆಡ್-ಆನ್ ನಿಮ್ಮ ಗ್ರಾಹಕರು 개개 sites ಗಳನ್ನು ಮಾತ್ರವಲ್ಲದೆ, ಸಂಪೂರ್ಣ WordPress multisite networks ಗಳನ್ನು ಖರೀದಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ.

## ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು (Key Features) {#key-features}

* **Network Provisioning**: ನಿಮ್ಮ ಗ್ರಾಹಕರಿಗಾಗಿ ಸಂಪೂರ್ಣ WordPress multisite networks ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಿ ಮತ್ತು ಕಾನ್ಫಿಗರ್ ಮಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
* **Network Templates**: ಗ್ರಾಹಕರು ನೋಂದಾಯಿಸುವ ಮೊದಲು, ಲಭ್ಯವಿರುವ prebuilt network template ಗಳನ್ನು ब्राउಜ್ ಮಾಡಲು, ಪೂರ್ವವೀಕ್ಷಣೆ ಮಾಡಲು ಮತ್ತು ಆಯ್ಕೆ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.
* **Network Management**: multinetwork ಪರಿಸರಗಳಿಗೆ ಸಂಪೂರ್ಣ ನಿರ್ವಹಣಾ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.
* **Isolated Networks**: ಪ್ರತಿ ಗ್ರಾಹಕರ network ಸ್ವತಂತ್ರವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ, ಅದು ತನ್ನದೇ ಆದ database tables ಮತ್ತು configuration ಅನ್ನು ಹೊಂದಿರುತ್ತದೆ.
* **Cross-Network SSO**: ಮ್ಯಾಜಿಕ್-ಲಿಂಕ್ authentication ಮೂಲಕ ಗ್ರಾಹಕರಿಗೆ ಅವರ sub-networks ಗಳಿಗೆ ಸುಗಮ ಪ್ರವೇಶವನ್ನು ನೀಡುತ್ತದೆ.
* **Scalable Architecture**: ಆಪ್ಟಿಮೈಸ್ಡ್ resource allocation ಮೂಲಕ ಅನೇಕ networks ಗಳನ್ನು ಸಮರ್ಥವಾಗಿ ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
* **Integration**: Ultimate Multisite ನ subscription ಮತ್ತು billing systems ಜೊತೆಗೆ ಸುಗಮವಾಗಿ ಸಂಯೋಜಿಸುತ್ತದೆ.

## ಬಳಕೆಯ ಸಂದರ್ಭಗಳು (Use Cases) {#use-cases}

* **White Label Hosting**: ಗ್ರಾಹಕರಿಗೆ ತಮ್ಮದೇ ಆದ ಬ್ರ್ಯಾಂಡೆಡ್ multisite networks ಗಳನ್ನು ಒದಗಿಸಲು.
* **Agency Solutions**: ಏಜೆನ್ಸಿಗಳಿಗೆ ಒಂದೇ ವೇದಿಕೆಯಿಂದ ಅನೇಕ client networks ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ.
* **Enterprise Deployments**: ಸಂಕೀರ್ಣ multi-network ಅಗತ್ಯತೆಗಳನ್ನು ಹೊಂದಿರುವ ದೊಡ್ಡ ಸಂಸ್ಥೆಗಳಿಗೆ ಬೆಂಬಲ ನೀಡುತ್ತದೆ.
* **Education Platforms**: ವಿವಿಧ ಶಾಲೆಗಳು ಅಥವಾ ವಿಭಾಗಗಳಿಗೆ ಪ್ರತ್ಯೇಕ networks ಗಳನ್ನು ರಚಿಸಲು.

## ಅವಶ್ಯಕತೆಗಳು (Requirements) {#requirements}

* Ultimate Multisite 2.0.0 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದನ್ನು
* WordPress Multisite installation
* PHP 7.4 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದನ್ನು
* MySQL 5.6 ಅಥವಾ ಅದಕ್ಕಿಂತ ಹೆಚ್ಚಿನದನ್ನು

## ಅಳವಡಿಕೆ (Installation) {#installation}

1. plugin files ಗಳನ್ನು `/wp-content/plugins/ultimate-multisite-multinetwork` ಡೈರೆಕ್ಟರಿಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ.
2. WordPress ನಲ್ಲಿರುವ 'Plugins' ಸ್ಕ್ರೀನ್ ಮೂಲಕ plugin ಅನ್ನು ಆಕ್ಟಿವೇಟ್ ಮಾಡಿ.
3. Ultimate Multisite → Settings ಮೂಲಕ multinetwork ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ.

## Network Templates {#network-templates}

Version 1.3.0 ಹೊಸ networks ಗಳನ್ನು ಖರೀದಿಸುವ ಗ್ರಾಹಕರಿಗಾಗಿ template discovery ಮತ್ತು selection flow ಅನ್ನು ಸೇರಿಸುತ್ತದೆ. ಪ್ರತಿ ಹೊಸ ಗ್ರಾಹಕರ network ಅನ್ನು ಸಿದ್ಧಪಡಿಸಿದ structure, design, content library, ಅಥವಾ vertical-specific setup ನಿಂದ ಪ್ರಾರಂಭಿಸಲು ನೀವು ಬಯಸಿದಾಗ network templates ಗಳನ್ನು ಬಳಸಿ.

### Template previewer {#template-previewer}

network template previewer ಗ್ರಾಹಕರು checkout ಮಾಡುವ ಮೊದಲು ಲಭ್ಯವಿರುವ templates ಗಳನ್ನು ब्राउಜ್ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ. ಪ್ರತಿ template ಅನ್ನು ಲೈವ್ preview panel ನಲ್ಲಿ ತೆರೆಯಬಹುದು, ಇದರಿಂದ ಗ್ರಾಹಕರು network ಅನ್ನು ಖರೀದಿಸುವ ಮೊದಲು లేಔಟ್‌ಗಳು ಮತ್ತು content ಗಳನ್ನು ಹೋಲಿಕೆ ಮಾಡಿಕೊಳ್ಳಬಹುದು.

ಗ್ರಾಹಕರಿಗೆ ಸಹಾಯ ಮಾಡಲು previewer ಅನ್ನು ಬಳಸಿ:

* ಲಭ್ಯವಿರುವ ಪ್ರತಿ network template ನ ನೋಟ ಮತ್ತು structure ಅನ್ನು ಪರಿಶೀಲಿಸಲು
* purchase flow ಅನ್ನು ಬಿಡದೆ ಅನೇಕ templates ಗಳನ್ನು ಹೋಲಿಕೆ ಮಾಡಲು
* ನೋಂದಾಯಿಸುವ ಮೊದಲು ತಮ್ಮ ಬಳಕೆಯ ಸಂದರ್ಭಕ್ಕೆ ಉತ್ತಮವಾಗಿ ಹೊಂದಿಕೆಯಾಗುವ template ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಲು

template ಹೆಸರುಗಳು ಮತ್ತು ವಿವರಣೆಗಳನ್ನು ಗ್ರಾಹಕ ಸ್ನೇಹಿಯಾಗಿ ಇರಿಸಿ. ಏಜೆನ್ಸಿ networks, ಶಾಲಾ networks, ಫ್ರಾಂಚೈಸ್ networks, ಅಥವಾ white-label hosting bundles ನಂತಹ ಉದ್ದೇಶಿತ ಬಳಕೆಯ ಸಂದರ್ಭವನ್ನು ಗ್ರಾಹಕರು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಾಧ್ಯವಾದಾಗ previewer ಹೆಚ್ಚು ಉಪಯುಕ್ತವಾಗಿರುತ್ತದೆ.

### Template picker at checkout {#template-picker-at-checkout}

template picker ಅನ್ನು sign-up checkout flow ನಲ್ಲಿ ಒಂದು field ಆಗಿ ಸೇರಿಸಬಹುದು. ಇದನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿದಾಗ, ಗ್ರಾಹಕರು checkout ಮಾಡುವ ಸಮಯದಲ್ಲಿ ಅವರು ಬಯಸುವ network template ಅನ್ನು ಆಯ್ಕೆ ಮಾಡುತ್ತಾರೆ, ಮತ್ತು ಆರಿಸಲಾದ template ಅನ್ನು ಅವರ ಹೊಸ network ಅನ್ನು provision ಮಾಡುವಾಗ ಬಳಸಲಾಗುತ್ತದೆ.

checkout flow ನಲ್ಲಿ picker ಅನ್ನು ಬಳಸಲು:

1. ನೀವು template ಆಗಿ ನೀಡಲು ಬಯಸುವ networks ಗಳನ್ನು ಸಿದ್ಧಪಡಿಸಿ.
2. ಸಂಬಂಧಿತ sign-up ಅಥವಾ checkout form ನಲ್ಲಿ Multinetwork template selection field ಅನ್ನು ಆನ್ ಮಾಡಿ.
3. ಯಾವ templates ಗಳು ಆ checkout flow ಗಾಗಿ ಲಭ್ಯವಿರಬೇಕು ಎಂಬುದನ್ನು ಆರಿಸಿ.
4. checkout flow ಅನ್ನು ಪಬ್ಲಿಷ್ ಮಾಡಿ ಮತ್ತು ಆರಿಸಲಾದ template ಅನ್ನು ಹೊಸ network ಗೆ ಅನ್ವಯಿಸಲಾಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು sign-up ಅನ್ನು ಪರೀಕ್ಷಿಸಿ.

ಒಂದು checkout flow ಕೇವಲ ಒಂದು network product ಅನ್ನು ನೀಡಿದರೆ, picker ಇನ್ನೂ ಒಂದು ಮಾರ್ಗದರ್ಶಿ ಆನ್‌ಬೋರ್ಡಿಂಗ್ ಹಂತವಾಗಿ ಉಪಯುಕ್ತವಾಗಬಹುದು. multi-plan funnels ಗಾಗಿ, ಆರಿಸಲಾದ product ಅಥವಾ plan ನೊಂದಿಗೆ ಹೊಂದಿಕೆಯಾಗುವ templates ಗಳನ್ನು ಮಾತ್ರ ಪ್ರದರ್ಶಿಸಿ.

## Cross-Network Magic Link SSO {#cross-network-magic-link-sso}

Cross-network magic link SSO ಗ್ರಾಹಕರು ತಮ್ಮ ಪ್ರಾಥಮಿಕ account area ಮತ್ತು sub-networks ನಡುವೆ ಕೈಯಾರೆ sign in ಮಾಡದೆ ಚಲಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಗ್ರಾಹಕರು cross-network access link ಅನ್ನು ಅನುಸರಿಸಿದಾಗ, Ultimate Multisite ಒಂದು ಸಮಯ-ಮಿತಿಯ ಮ್ಯಾಜಿಕ್-ಲಿಂಕ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ ಮತ್ತು ಗ್ರಾಹಕನನ್ನು ಗಮ್ಯಸ್ಥಾನ network ನಲ್ಲಿ ಅಧಿಕೃತಗೊಳಿಸುತ್ತದೆ.

ಇದು ಒಂದಕ್ಕಿಂತ ಹೆಚ್ಚು networks ಅನ್ನು ನಿರ್ವಹಿಸುವ ಅಥವಾ purchase ಮಾಡಿದ ತಕ್ಷಣ ಹೊಸದಾಗಿ clone ಮಾಡಿದ template networks ಗಳಿಗೆ ಪ್ರವೇಶಿಸುವ ಗ್ರಾಹಕರಿಗೆ ಅನುಭವವನ್ನು ಸುಧಾರಿಸುತ್ತದೆ.

Admin checklist:

* WordPress multisite installation ನಲ್ಲಿ Multinetwork addon ಆಕ್ಟಿವ್ ಆಗಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.
* ಗ್ರಾಹಕರಿಗೆ cross-network links ನೀಡುವ ಮೊದಲು ಸರಿಯಾದ network access ಇದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.
* ಮ್ಯಾಜಿಕ್ ಲಿಂಕ್‌ಗಳು ಉದ್ದೇಶಿತ ಗಮ್ಯಸ್ಥಾನ network ಗೆ resolve ಆಗುವಂತೆ network domains ಮತ್ತು mapped domains ಗಳನ್ನು ಸರಿಯಾಗಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಿ.
* network, domain, ಅಥವಾ checkout ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಬದಲಾಯಿಸಿದ ನಂತರ customer dashboard ಮತ್ತು ಯಾವುದೇ post-checkout access buttons ಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ.

ಮ್ಯಾಜಿಕ್ ಲಿಂಕ್‌ಗಳನ್ನು login links ವಾಗಿ ಪರಿಗಣಿಸಬೇಕು. ಅವು ಸುಗಮ ಗ್ರಾಹಕ ಪ್ರವೇಶಕ್ಕಾಗಿ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ ಮತ್ತು ಉದ್ದೇಶಿತ session ಹೊರಗೆ ಫಾರ್ವರ್ಡ್ ಅಥವಾ ಮರುಬಳಕೆ ಮಾಡಬಾರದು.

## ಆಗಾಗ್ಗೆ ಕೇಳಲಾಗುವ ಪ್ರಶ್ನೆಗಳು (Frequently Asked Questions) {#frequently-asked-questions}

### multinetwork ಎಂದರೇನು? {#what-is-multinetwork}

Multinetwork ಎಂಬುದು WordPress ವೈಶಿಷ್ಟ್ಯವಾಗಿದ್ದು, ಇದು ಒಂದೇ WordPress installation ನಲ್ಲಿ ಅನೇಕ ಸ್ವತಂತ್ರ multisite networks ಗಳನ್ನು ನಡೆಸಲು ನಿಮಗೆ ಅನುಮತಿಸುತ್ತದೆ.

### ಇದು ಸಾಮಾನ್ಯ multisite ನಿಂದ ಹೇಗೆ ಭಿನ್ನವಾಗಿದೆ? {#how-is-this-different-from-regular-multisite}

ಸಾಮಾನ್ಯ multisite ಒಂದು network ನಲ್ಲಿ ಅನೇಕ sites ಗಳನ್ನು ಅನುಮತಿಸುತ್ತದೆ. multinetwork ಒಂದು WordPress installation ನಲ್ಲಿ, ತಮ್ಮದೇ ಆದ sites ಗಳನ್ನು ಹೊಂದಿರುವ ಅನೇಕ networks ಗಳನ್ನು ಅನುಮತಿಸುತ್ತದೆ.

### ಇದಕ್ಕೆ ವಿಶೇಷ ಸರ್ವರ್ ಕಾನ್ಫಿಗರೇಶನ್ ಬೇಕೇ? {#does-this-require-special-server-configuration}

ಹೌದು, multinetwork ಗೆ ನಿರ್ದಿಷ್ಟ ಸರ್ವರ್ ಸೆಟಪ್ ಮತ್ತು ಕಾನ್ಫಿಗರೇಶನ್ ಅಗತ್ಯವಿದೆ. ವಿವರವಾದ ಸೆಟಪ್ ಸೂಚನೆಗಳಿಗಾಗಿ ದಯವಿಟ್ಟು documentation ಅನ್ನು ಸಂಪರ್ಕಿಸಿ.
