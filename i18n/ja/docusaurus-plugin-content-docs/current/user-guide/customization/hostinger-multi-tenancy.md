# ホスティングマルチテナンシー

Ultimate Multisite: Multi-Tenancy 1.2.0 では、ホストされたドメインが既存の Hostinger ドメインマッピング統合と並行して主権的なテナントプロビジョニングに参加できるように、Hostinger のマルチテナンシー機能を追加しました。

この機能は、テナントドメインと隔離されたテナントインフラストラクチャを Hostinger hPanel を介して管理する場合に使用してください。

## セットアップ時の注意点

1. **Ultimate Multisite > Settings > Host Integrations** でコアの Hostinger 統合を設定します。
2. Hostinger API トークンが対象ドメインまたはサブドメインを管理できることを確認します。
3. マルチテナンシーアドオンを有効にします。
4. テナントを公開する前に、テナントの分離戦略を設定します。
5. 本番トラフィックをテナントに送信する前に、移行検証ワークフローを実行します。

Hostinger 機能は、ホスト側での操作のために共有された Hostinger 接続を使用します。DNS は引き続き正しい Hostinger アカウントを指している必要があり、hPanel のアカウント制限も適用されます。

## 機能固有の変更点

- 主権テナントは、ホスト認識型のドメイン操作で作成できます。
- 同じマシン上のデータベースホスト文字列は、検証前に正規化されます。
- Hostinger によって管理されるテナントは、WordPress ランタイムがローカルの上書きを必要としない限り、hPanel に表示されているデータベースホスト値を使用する必要があります。
- SSO 訪問は、テナントドメインが Hostinger がホストするテナントに解決されることに依存します。

## Hostinger テナントのトラブルシューティング

- テナントのインストールが失敗した場合、そのドメインがすでに Hostinger アカウントに紐づいていることを確認してください。
- データベース検証が失敗した場合は、テナントの DB ユーザー名、データベース名、ホストバインディングを hPanel と比較してください。
- **Visit (SSO)** が失敗した場合は、テナントドメインの DNS および SSL が有効になっていることを確認してください。
- テアダウン後にホストリソースが残っている場合は、バックアップを確認した後、hPanel から残っているデータベース、ユーザー、またはフォルダを削除してください。
