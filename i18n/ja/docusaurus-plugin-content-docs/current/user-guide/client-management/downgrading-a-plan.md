# プランのダウングレード（v2）

_**重要な注記：この記事は Ultimate Multisite バージョン 2.x を対象としています。**_

プランやサブスクリプションのダウングレードは、予算が限られている場合や、サブサイトの運用に多くのリソースは不要だと判断した場合に、クライアントが行うことの多い操作です。

## プランをダウングレードする方法 {#how-to-downgrade-a-plan}

クライアントは、サブサイトの管理Dashboardにログインし、Accountページで**変更**をクリックすることで、いつでもプランをダウングレードできます。

![Your Membershipカードと変更ボタンがある顧客のAccountページ](/img/account-page/membership-change-button.png)

**変更**ボタンをクリックすると、ユーザー／クライアントは決済ページにリダイレクトされ、サブスクリプションの変更先となるプランを選択できます。

![顧客側のプランダウングレード選択ページ](/img/account-page/downgrade-picker.png)

この例では、プランを**Premium**から**Free**へダウングレードします。

続行するには、ユーザーは**決済を完了**ボタンをクリックするだけです。その後、Accountページに戻り、メンバーシップの保留中の変更に関するメッセージが表示されます。変更は顧客の**次回請求サイクル**で有効になります。

![保留中のメンバーシップ変更バナーを表示しているAccountページ](/img/account-page/pending-change.png)

### ユーザーがプランをダウングレードするとどうなるか {#what-happens-when-a-user-downgrades-their-plan}

プランをダウングレードしても、ユーザーのサブサイトにある既存の設定は変更されない点に注意してください。

サイトテンプレートの変更はサブサイトを完全に消去してリセットするため、自動的には変更されません。これは不要なデータ損失を避けるためです。そのため、投稿を除き、ディスク容量、テーマ、プラグインなどはそのまま維持されます。

各プランで設定した制限や上限が主な懸念点であることは理解していますが、設定を削除または変更した場合にユーザーのサブサイトへ与える影響も考慮する必要があります。

プランで設定された上限を超える投稿については、3つのオプションがあります：**投稿をそのまま保持** *,* **投稿をゴミ箱へ移動** *,* または **投稿を下書きへ移動** *.* これは Ultimate Multisite の設定で構成できます。

![投稿上限超過時の動作オプションを表示しているNetwork Admin設定のサイトページ](/img/account-page/settings-sites-post-limit.png)

### 支払いはどうなるか {#what-happens-to-the-payment}

バージョン 2.0 では、日割り計算に関して支払いの調整は不要になりました。

これは、新しいプラン／メンバーシップが有効になる前に、既存のメンバーシップが**請求サイクルを完了するまで**システムが待機するためです。新しいメンバーシップの新しい請求金額は、次回請求サイクルで自動的に適用され、請求されます。
