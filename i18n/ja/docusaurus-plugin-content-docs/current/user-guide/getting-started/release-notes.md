# リリースノート

## Version 2.12.0 — 2026-05-15 リリース

*   **新機能:** ホスティングプロバイダーとして Hostinger (hPanel) がサポートされ、ドメインマッピング連携が追加されました。
*   **新機能:** Site Exporter がネットワークインポートバンドルを処理するようになり、ネットワーク全体でのサイト復元がよりスムーズになりました。
*   **修正:** BCCブロードキャストメールが、受信者アドレスの漏洩を防ぐために、非公開の `undisclosed-recipients` ヘッダーを使用するようになりました。
*   **修正:** メンバーシップの有効期限が、日付ではない値で保存された場合でも破損しなくなりました。
*   **修正:** Stripeのメンバーシップ更新が、非推奨となった `deleteDiscount` APIを呼び出さずに、割引を正しくクリアするようになりました。
*   **修正:** ドメインマッピングされたサイトでのSSOリダイレクトが、無限リダイレクトループを防ぐために制限されるようになりました。
*   **修正:** セットアップウィザードの画像ピッカーの選択が、基となるデータモデルを正しく更新するようになりました。
*   **修正:** Site Exporter CLI が、正しいデフォルトのネットワークサイト選択を保持するようになりました。
*   **改善:** プラグインのサイズを削減するため、バンドルされていた wp-cli がプラグインパッケージから削除されました。

## Version 2.11.0 — 2026-05-11 リリース

*   **新機能:** Siteエクスポートが、自己起動可能な `index.php` をバンドルするようになり、ZIPファイルを別のプラグインインストールなしで新しいホストにインストールできるようになりました。
*   **新機能:** ネットワークエクスポートにより、管理者はSite Exportの管理画面からすべてのサブサイトを単一のアーカイブとしてエクスポートできるようになりました。
*   **新機能:** Site Templatesのプラン切り替えのトグルが、フォールバックチェーンを通じて強制されるようになり、プラン制限に応じたテンプレートの利用可能性が正しく制限されるようになりました。
*   **新機能:** チェックアウトフォームエディタが、必須フィールドが設定されていない製品が追加された場合に警告を表示するようになりました。
*   **新機能:** Import/Export設定タブが、そのスコープを明確に説明し、Site Exportツールに直接リンクするようになりました。

## Version 2.10.0 — 2026-05-05 リリース

*   **新機能:** PayPalのガイド付きセットアップウィザードが、手動での認証情報入力に対応し、OAuthフラグゲートによりシームレスなゲートウェイ設定が可能になりました。
*   **新機能:** Template Switchの顧客パネルが、現在のテンプレートカード、永続的なグリッド、および**現在のテンプレートをリセット**ボタンを備えて再設計されました。
*   **修正:** Template switchingが、AJAXの失敗時にUIがフリーズしなくなりました。
*   **修正:** Template switchingの権限状態が、不正なアクセスから保護されるようになりました。
*   **修正:** Site overrideの入力が、保存前に検証されるようになりました。
*   **修正:** 請求先住所のプロンプトが、住所が空の場合に表示されるようになりました。
*   **修正:** PHP 8.1のnull-to-stringの非推奨通知が解決されました。
*   **修正:** Currentsが、タイミングの問題を防ぐため、initフックの前に遅延ロードされるようになりました。
*   **修正:** フィルタリングされたSSOパスが、すべてのログインフローで尊重されるようになりました。
*   **修正:** 空のサイトの識別オプションが、保存時に保持されるようになりました。

## Version 2.9.0 — 2026-04-30 リリース

*   **新機能:** **ツール > エクスポート＆インポート** の下に、単一サイトのエクスポートとインポートが追加されました。
*   **修正:** エクスポートZIPファイルが、認証されたダウンロードエンドポイントを通じて提供されるようになりました。
*   **修正:** 保留中のエクスポート/インポートクエリにおけるSQLインジェクションのリスクとクエリの問題が修正されました。
*   **修正:** 管理者が顧客のメールアドレスを手動で検証しても、保留中のサイトが公開されない問題が修正されました。
*   **修正:** メンバーシップが欠落している場合に、孤立した `pending_site` レコードがクリーンアップされるようになりました。
*   **修正:** 設定ナビゲーションのパディングと検索アンカーナビゲーションが修正されました。
*   **修正:** 保留中のサイトが、すべてのサイトビューで最初に表示されるようになりました。
*   **修正:** スクリーンショットプロバイダー (mShots) のUser-Agentヘッダーが追加され、403エラーを防ぐようになりました。
*   **修正:** インポートのcronスケジュールの循環依存が解決されました。
*   **修正:** ユーザー設定キーのTour IDがアンダースコアに正規化されるようになりました。
*   **改善:** 互換性を高めるため、ZipArchiveがAlchemy/Zippyの代わりに使用されるようになりました。

## Version 2.8.0 — 2026-04-29 リリース

*   **新機能:** Other Options設定UIにJumperのトグルが追加されました。
*   **新機能:** チェックアウトフォームのリストテーブルにステータス列が追加されました。
*   **新機能:** カスタムMU-plugin sunrise拡張機能のためのアドオンsunriseファイルローダーが追加されました。
*   **改善:** 設定ページからエラー報告のオプトイン設定が削除されました。
*   **修正:** サンキューページサイトカードの画像が制約され、リンクが正しくスタイル付けされるようになりました。
*   **修正:** スクリーンショットプロバイダーが thum.io から WordPress.com mShots に切り替わりました。
*   **修正:** Enable RegistrationとDefault Roleが、新規インストール時に正しいデフォルト値に設定されるようになりました。
*   **修正:** `get_site_url()` が、ドメインにポートが含まれていても空を返さなくなりました。
*   **修正:** `copy_media` 設定が空の場合でも、メディアファイルが正しくコピーされるようになりました。
*   **修正:** ネットワークアクティベートのsitemeta書き込み後、オブジェクトキャッシュが正しく無効化されるようになりました。
*   **修正:** 3部構成のドメインの場合、カスタムドメインがDNS検証時にプライマリに自動昇格するようになりました。
*   **修正:** 有効期限が切れた支払い情報がクリーンアップされる際に、保留中のメンバーシップがキャンセルされるようになりました。
*   **修正:** パスワード強度チェッカーが、インラインログインプロンプトが閉じた後に再バインドされるようになりました。
*   **修正:** サイトがすでに作成されている場合、サンキューページでの無限ページリロードが停止するようになりました。
*   **修正:** WP coreの登録オプションが、プラグインの有効化時と設定保存時に同期されるようになりました。
*   **修正:** PHP 8.4の互換性のために、`calculate_expiration` にNullの有効期限ガードが追加されました。
*   **修正:** 顧客がすでにアクティブなメンバーシップを持っている場合、重複サインアップがブロックされるようになりました。
*   **修正:** チェックアウトで `date_expiration` のNullチェックが追加されました。
*   **修正:** サイトプロビジョニングが強化されました — 制限、メンバーシップの推論、ドメインの昇格。
*   **修正:** チェックが失敗した場合、事前インストールチェックの状態ラベルが「NOT Activated」に修正されました。
*   **修正:** チェックアウトドメインがメール検証URLに使用されるようになりました。
*   **修正:** パスワードフィールドがない場合、チェックアウト後の自動ログインが修正されました。
*   **修正:** 無料メンバーシップが期限切れにならなくなり、永続的なものとして扱われるようになりました。
*   **修正:** メール検証ゲートが、顧客がメールを検証するまでサイトの公開を保留するようになりました。
*   **修正:** SES v2 APIエンドポイントのベースパスとアイデンティティルートが修正されました。
*   **修正:** `wu_inline_login_error` フックが、pre-submitキャッチブロック内で発信されるようになりました。
