# リリースノート {#release-notes}

## Version 2.13.0 — 2026-06-05 リリース {#version-2130--released-on-2026-06-05}

- 新規: 顧客 Account、購入手続き、請求、サイト、請求書、テンプレート切り替え、ドメインマッピングのフローに sovereign-tenant サポートを追加し、テナントネットワークが管理アクションのために顧客をメインサイトへ戻せるようにしました。
- 新規: 定期メンバーシップ向けに更新用認証情報チェックを追加し、保存済みの請求契約、サブスクリプション、または vault token がない場合にゲートウェイが自動更新を無効化できるようにしました。
- 新規: 保留中のサイト作成に HMAC 検証済みのループバック公開を追加し、バックグラウンドジョブが遅延するホストで購入手続きからサイト提供までの信頼性を高めました。
- 新規: SSO URL、購入手続きフォームのベースドメイン、自動ドメインレコード作成向けの開発者用拡張ポイントを追加しました。
- 修正: マッピング済みドメイン、匿名ブローカー訪問、ログアウト、複数 plugin 間の依存関係競合における SSO の信頼性を高めました。
- 修正: 保留中のサイト作成が古い公開フラグから復旧し、顧客がサイト作成画面で止まらないようにしました。
- 修正: 共有の購入手続きフォーム用ベースドメインではドメインレコードを作成しなくなり、連携が有効でない場合は未使用のホストプロバイダーのバックグラウンドジョブをスキップするようにしました。
- 修正: 購入手続き、請求先住所、パスワードリセット、メール認証、テンプレート切り替え、ツアー、顧客 Dashboard のエッジケースが通常の顧客フローを妨げないようにしました。
- 修正: 一斉送信メールで受信者の非公開を維持しつつ、受信者リストまたはメール送信処理が失敗した場合の SMTP/plugin の致命的エラーを回避するようにしました。
- 修正: メンバーシップ更新、有効期限表示、支払い回収のエッジケースで、即時失効、クラッシュ、必要な支払い漏れを回避するようにしました。
- 改善: WordPress 互換性を 7.0 までテストし、本番用 Vue アセットを npm ソースから再ビルドし、Cypress のエンドツーエンドカバレッジでより多くの購入手続き、セットアップ、SSO、ゲートウェイフローを検証するようにしました。

## Version 2.12.0 — 2026-05-15 リリース {#version-2120--released-on-2026-05-15}

- 新規: ドメインマッピング連携に対応したホストプロバイダーとして Hostinger (hPanel) を追加しました
- 新規: Site Exporter がネットワークインポートバンドルを処理し、ネットワーク全体のサイト復元を効率化できるようになりました
- 修正: BCC 一斉送信メールで undisclosed-recipients ヘッダーを使用し、受信者アドレスの露出を防ぐようにしました
- 修正: 日付以外の値で保存したときに、メンバーシップの有効期限日が破損しなくなりました
- 修正: Stripe メンバーシップ更新で、非推奨の deleteDiscount API を呼び出さずに割引を正しくクリアするようにしました
- 修正: ドメインマッピング済みサイトでの SSO リダイレクトに上限を設け、無限リダイレクトループを防ぐようにしました
- 修正: セットアップウィザードの画像ピッカー選択が基盤のデータモデルを正しく更新するようにしました
- 修正: Site Exporter CLI が正しいデフォルトネットワークサイト選択を保持するようにしました
- 改善: plugin パッケージから同梱の wp-cli を削除し、plugin サイズを削減しました

## Version 2.11.0 — 2026-05-11 リリース {#version-2110--released-on-2026-05-11}

- 新規: サイトエクスポートに自己起動型の `index.php` を同梱し、別途 plugin をインストールしなくても ZIP を新しいホストにインストールできるようにしました。
- 新規: ネットワークエクスポートにより、管理者が Site Export 管理ページからすべてのサブサイトを単一アーカイブとしてエクスポートできるようにしました。
- 新規: Allow Site Templates プラン切り替えをフォールバックチェーン経由で適用し、プラン制限に応じてテンプレートの利用可否を正しく制限するようにしました。
- 新規: 購入手続きフォームエディターが、必須フィールドの設定なしに商品が追加された場合に警告するようにしました。
- 新規: Import/Export 設定タブで対象範囲を明確に説明し、Site Export ツールへ直接リンクするようにしました。

## Version 2.10.0 — 2026-05-05 リリース {#version-2100--released-on-2026-05-05}

- 新規: 手動認証情報入力向けの PayPal ガイド付きセットアップウィザードを追加し、シームレスなゲートウェイ設定のための OAuth フラグゲートに対応しました。
- 新規: テンプレート切り替えの顧客パネルを、現在のテンプレートカード、永続グリッド、**現在のテンプレートをリセット**ボタンで再設計しました。
- 修正: AJAX 失敗時にテンプレート切り替えで UI が停止しなくなりました。
- 修正: テンプレート切り替えの権限状態を、不正アクセスから保護しました。
- 修正: サイト上書き入力を保存前に検証するようにしました。
- 修正: 住所が空の場合に請求先住所の入力促進を表示するようにしました。
- 修正: PHP 8.1 の null から文字列への非推奨通知を解消しました。
- 修正: タイミング問題を防ぐため、init hook の前に Currents を遅延読み込みするようにしました。
- 修正: フィルター済みの SSO パスをすべてのログインフローで尊重するようにしました。
- 修正: 空のサイト識別情報オプションを保存時に保持するようにしました。

## Version 2.9.0 — 2026-04-30 リリース {#version-290--released-on-2026-04-30}

- 新規: 単一サイトのエクスポートとインポートを **ツール > エクスポートとインポート** に追加しました。
- 修正: エクスポート ZIP ファイルを認証済みダウンロード endpoint 経由で提供するようにしました。
- 修正: 保留中のエクスポート/インポートクエリにおける SQL インジェクションリスクとクエリ問題を修正しました。
- 修正: 管理者が顧客メールを手動認証したときに、保留中のサイトが公開されない問題を修正しました。
- 修正: メンバーシップがない場合に、孤立した pending_site レコードをクリーンアップするようにしました。
- 修正: 設定ナビゲーションの余白と検索アンカー移動を修正しました。
- 修正: All Sites ビューで保留中のサイトを先に表示するようにしました。
- 修正: 403 エラーを防ぐため、スクリーンショットプロバイダー (mShots) の User-Agent header を追加しました。
- 修正: インポート cron スケジュールの循環依存を解消しました。
- 修正: ツアー ID をユーザー設定キー内でアンダースコアに正規化しました。
- 改善: 互換性向上のため、Alchemy/Zippy の代わりに ZipArchive を使用するようにしました。
