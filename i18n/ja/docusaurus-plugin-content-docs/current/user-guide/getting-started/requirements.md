# Ultimate Multisite 要件

Ultimate Multisite を購入する前に自分自身に尋ねるかもしれない最初のことは、私たちのプラグインを実行するための要件は何かです。幸いなことに、Ultimate Multisite を実行するための基本的な要件はわずか数個だけです：**ドメイン名**、良い **ホスティングプロバイダー**、そして WordPress **Multisite** のインストールです。これだけで十分です！

## ドメイン名

まず、ドメイン名について話しましょう。

ドメイン名を選ぶ際は、あなたの特定のニッチと Ultimate Multisite で何を作るかを考慮してください：エージェンシーを作りたいですか？ショップのようなウェブサイトを販売したいですか？それとも WordPress のウェブホスティングを提供したいですか？Ultimate Multisite ならすべてが可能です。

当プラグインはサブスクリプションサービス、支払い収集、チェックアウトフォーム、割引クーポン、顧客コミュニケーションを提供します。WordPress Multisite のインストール、設定、保守に必要な作業の大部分は Ultimate Multisite によって容易になります。

サービスとしてウェブサイトを販売する限り、選んだどんなニッチでも当プラグインで管理できます！ビジネスプランに合ったドメイン名を選ぶようにしてください。

## ホスティングプロバイダー

ニッチとビジネスを代表する魅力的なドメイン名を手に入れたら、Ultimate Multisite でネットワークをホストするために良いウェブホスティング会社を選ぶ必要があります。市場にはいくつかの良いオプションがあります。いくつかのホスティングプロバイダーは Ultimate Multisite とよく統合されており、構築予定に応じて決定的な要因になるかもしれません。

Ultimate Multisite はどのホスティングプロバイダーでも動作するはずです。唯一の違いは、_WPMU DEV Hosting_、_WP Engine_、_Closte_、_Cloudways_、_Gridpane_、_Runcloud_ のような一部のホスティング会社がドメインマッピングに対してより緊密な統合を持っていることです。他のホスティングでは、カスタムドメインを手動でマッピングする必要があるかもしれません。また、AutoSSL をサポートするホスティングプロバイダーを選ぶのも良いアイデアです。_Closte_、_Runcloud_、_cPanel_ がそれをサポートしています。

サーバー仕様に関しては、提供する機能やネットワークの規模に応じて異なるため、特定の公式はありません。共有ホスティングは良いスタートになるかもしれませんが、ネットワークが成長し始めたら、専用 VPS が必要になるでしょう。

## WordPress Multisite

さて、WordPress Multisite のインストールについて少し話しましょう。

WordPress Multisite は、単一のインストールでサイトのネットワークを持つことを可能にします。これは組み込み機能ですが、デフォルトでは有効になっていません。WordPress のインストールとウェブホスティングのコントロールパネル内でこの機能を有効にする必要があります。

WordPress で Multisite を有効にしていることは **Ultimate Multisite の基盤です**。Ultimate Multisite の主な用途は、ユーザーがサービスとしてウェブサイトを管理・販売できるようにすることなので、WordPress で Multisite を有効にしておく必要があります。

WordPress のインストールで Multisite 機能を有効にするために必要な手順を段階的に教える **詳細な記事** を用意しています。

WordPress で Multisite を有効にしたら、Ultimate Multisite をインストールする時です。Ultimate Multisite を正しくインストールするには、以下が必要です：

* WordPress: v5.3+ (推奨: 最新安定版)
* PHP: 7.4.x (8.0 以上へのサポートは近日公開予定)
* MySQL: v5+ (推奨: 5.6、ホスティングプロバイダーで 8.0 が利用できない場合)

これらが Ultimate Multisite の唯一のソフトウェア要件です。

Ultimate Multisite は PHP 8.0 でも動作する可能性がありますが、PHP 7.4.x で実行することを推奨します。

また、**メインドメイン** にインストールすることを推奨します。サブドメインやサブディレクトリでは動作する可能性がありますが、エラーが発生することがあります。

Ultimate Multisite のインストール方法の詳細ガイドについては、**ここをクリック**してください。
