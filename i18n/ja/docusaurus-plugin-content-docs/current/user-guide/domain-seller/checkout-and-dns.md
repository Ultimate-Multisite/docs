# チェックアウトフィールドと顧客のDNS管理

## ドメイン選択チェックアウトフィールド

**ドメイン選択**フィールドは、顧客にサイトのドメインの取得方法の選択肢を提供するチェックアウト要素です。これをチェックアウトフォームに追加することで、ドメインの販売を可能にします。

### チェックアウトフォームへのフィールドの追加

1. **ネットワーク管理 › Ultimate Multisite › チェックアウトフォーム** に移動します。
2. チェックアウトフォームを開くか、作成します。
3. チェックアウトエディターで、**フィールドを追加**をクリックします。
4. フィールドリストから **ドメイン選択** を選びます。
5. フィールドオプションを設定します（下記参照）。
6. フォームを保存します。

### フィールドオプション

**ドメインモード** — 顧客に表示するタブを選択します。各モードは個別に有効または無効にできます。

| モード | 機能 |
|---|---|
| **サブドメイン** | 顧客がネットワーク内の無料のサブドメインを使用します（例：`mysite.yournetwork.com`）。支払い不要です。 |
| **新規ドメインの登録** | 顧客が新しいドメインを検索し、設定されたプロバイダーを通じて登録します。価格設定には、一致するドメイン商品を使用します。 |
| **既存ドメイン** | 顧客がすでに所有しているドメインをマッピングします。登録料はかかりません。ドメインは自動的にそのサイトにマッピングされます。 |

**デフォルトモード** — 3つのモードすべてが有効な場合、最初に開くタブを決定します。ドメイン登録を任意に保ちたい場合は **サブドメイン** に、購入を促したい場合は **新規ドメインの登録** に設定します。

**ドメイン商品** — オプションで、このフィールドを特定のドメイン商品に固定できます。設定しない場合、アドオンは顧客が検索したTLDに基づいて、一致する商品を自動的に選択します。

### 登録者連絡先フィールド

顧客が **新規ドメインの登録** タブを選択すると、チェックアウトフォームに登録者連絡先フィールドがインラインで追加されます。

- 名 / 姓
- メールアドレス
- 住所（番地1、市、州/都道府県、郵便番号、国）
- 電話番号

これらはすべてのレジストラによって必須とされており、登録APIコールが実行される前に検証されます。電話番号は、レジストラが期待する国際形式の `+CC.NNN` に自動的にフォーマットされます。

### 自動生成されるサイトURL

顧客がドメインを登録またはマッピングすると、サイトURLフィールドは選択されたドメインから自動的に入力されます。顧客は別途URLフィールドに記入する必要はありません。

### 検索の動作

- 顧客が入力するたびに、AJAXを使用してドメインの空き状況がリアルタイムでチェックされます。
- 希望のドメインが利用できない場合、代替のTLD候補が表示されます。
- 価格はライブで取得され、明確に表示されます（登録価格、更新価格、オプションのWHOISプライバシー料金）。
- クーポンコードは、他の商品と同様にドメイン商品にも適用されます。

**検索の応答性を調整する:**

```php
// 遅い接続でのAPIコールを減らすために、デバウンス遅延（ミリ秒）を増やします
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ドメイン検索フォームにカスタムフィールドを追加する:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => '追加のメモ',
    ];
    return $fields;
});
```

## 顧客のDNS管理

顧客は、**マイアカウント** ページから、自身のドメインのエントリの下で、DNSレコードを管理できます。

### サポートされているレコードタイプ

| タイプ | 用途 |
|---|---|
| **A** | ホスト名をIPv4アドレスにマッピングします |
| **AAAA** | ホスト名をIPv6アドレスにマッピングします |
| **CNAME** | 別のホストを指すエイリアスを作成します |
| **MX** | メール交換サーバーを設定します |
| **TXT** | SPF、DMARC、検証、またはその他のテキストレコードを追加します |

### DNS管理をサポートするプロバイダーは？

DNS管理（レコードの追加、編集、削除）は、**OpenSRS**、**ResellerClub**、および **Enom** で利用可能です。Namecheap、GoDaddy、NameSiloのドメインはステータスと有効期限の情報は表示しますが、DNSはレジストラのコントロールパネルで直接管理する必要があります。

### デフォルトのDNSレコード

ドメインが登録される際に自動的に適用されるデフォルトのDNSレコードを設定できます。**設定 › ドメインセラー › デフォルトのDNSレコード** に移動してください。

デフォルトのレコード値は、2つのトークンをサポートしています。

| トークン | 置換されるもの |
|---|---|
| `{DOMAIN}` | 登録されたドメイン名（例：`example.com`） |
| `{SITE_URL}` | 顧客のサイトのWordPressサイトURL |

**例 — エイペックスドメインとwwwをサーバーIPに向ける:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### 管理者：DNSの表示と編集

ネットワーク管理者は、**ネットワーク管理 › Ultimate Multisite › ドメイン** のドメイン編集ページから、任意の顧客ドメインのDNSレコードを表示および編集できます。
