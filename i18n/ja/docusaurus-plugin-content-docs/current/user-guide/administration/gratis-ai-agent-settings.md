# Gratis AI Agent の設定

Gratis AI Agent の **設定 → 詳細** 画面では、v1.5.0 で導入されたバックエンド連携に関する管理者レベルの設定が可能です。このページでは、**フィードバックエンドポイント**のフィールドとその期待される形式について説明します。

## 設定へのアクセス方法

1. WordPress の管理画面から、**Gratis AI Agent → 設定** に移動します。
2. **詳細** タブをクリックします。

## フィードバックエンドポイントの設定

フィードバックエンドポイントは、ユーザーが「いいね」ボタン（親指を下ろすボタン）や自動プロンプトバナー、または `/report-issue` コマンドを通じてフィードバックを送信するたびに、AI エージェントから POST リクエストを受け取ります。

| フィールド | 説明 |
|---|---|
| **フィードバックエンドポイント URL** | JSON ボディを持つ HTTP POST リクエストとしてフィードバックの送信を受け取る URL です。 |
| **フィードバック API キー** | 各フィードバックリクエストの `Authorization` ヘッダーに含まれるベアラートークンです。エンドポイントが認証を必要としない場合は、空欄のままにしてください。 |

### 期待される JSON ペイロード

フィードバックエンドポイントは、少なくとも以下のフィールドを含む JSON ボディを受け入れる必要があります。

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "価格に関する回答が間違っていました。",
  "triage_category": "factual_error"
}
```

ペイロードには、会話のコンテキストに応じて追加のフィールドが含まれる場合があります。

### `triage_category` の値

AI のトリアージ層は、ペイロードを転送する前に、`triage_category` に以下のいずれかの値を割り当てます。

| 値 | 意味 |
|---|---|
| `factual_error` | アシスタントが誤った事実情報を提供しました。 |
| `unhelpful_answer` | 回答は技術的には正しいものの、役に立ちませんでした。 |
| `inappropriate_content` | 回答にユーザーに表示すべきでないコンテンツが含まれていました。 |
| `other` | フィードバックが既知のカテゴリに一致しませんでした。 |

### 認証

エンドポイントが認証を必要とする場合は、**フィードバック API キー** フィールドにベアラートークンを設定してください。エージェントは以下を送信します。

```
Authorization: Bearer <your-api-key>
```

**フィードバック API キー** フィールドが空の場合、`Authorization` ヘッダーは送信されません。

### フィードバック収集の無効化

**フィードバックエンドポイント URL** と **フィードバック API キー** の両方を空欄にしてください。親指を下ろすボタンとフィードバック UI はユーザーに表示されたままですが、送信データは外部サービスには転送されません。

## Brave Search API キー

**詳細** タブにもある **Brave Search API キー** フィールドを使用すると、[インターネット検索](../configuration/internet-search) 機能が有効になります。

| フィールド | 説明 |
|---|---|
| **Brave Search API キー** | Brave Search の開発者ダッシュボードから取得した API キーです。AI アシスタントでインターネット検索を有効にするために必要です。 |

このフィールドラベルには、Brave Search API のサインアップページへのクリック可能なリンクが含まれています。インターネット検索を無効にする場合は、空欄のままにしてください。

エンドユーザー向けのドキュメントについては、[インターネット検索](../configuration/internet-search) を参照してください。

## 機能フラグ (Feature Flags)

v1.9.0 で導入された **設定 → 機能フラグ** タブでは、オプションの機能に対するトグルスイッチが提供されます。各フラグはネットワーク全体で有効または無効に設定され、現時点ではサイトごとの上書き設定はできません。

### 機能フラグへのアクセス方法

1. WordPress の管理画面から、**Gratis AI Agent → 設定** に移動します。
2. **機能フラグ** タブをクリックします。

### アクセス制御フラグ

| フラグ | デフォルト | 説明 |
|---|---|---|
| **管理者のみに制限** | オフ | 有効にすると、`administrator` ロールを持つユーザーのみが AI Agent チャットパネルを開けるようになります。その他のロールのユーザーは「管理者に連絡してください」というメッセージが表示されます。 |
| **ネットワーク管理者のみに制限** | オフ | マルチサイトネットワークで有効にすると、スーパーアドミンのみがエージェントを使用できます。個別のサイト管理者はブロックされます。両方が有効な場合、「管理者のみに制限」よりも優先されます。 |
| **購読者へのアクセスを許可** | オフ | 有効にすると、`subscriber` ロールを持つユーザーがチャットインターフェースを使用できますが、読み取り専用の機能に制限されます（投稿の作成や設定の変更はできません）。 |
| **非メンバーの無効化** | オフ | Ultimate Multisite のメンバーシップステータスと連携します。有効にすると、アクティブなメンバーシップを持っていないサイトではチャットが非表示になります。 |

### ブランディングフラグ

| フラグ | デフォルト | 説明 |
|---|---|---|
| **「Powered by Gratis AI Agent」フッターを非表示** | オフ | チャットウィジェットの下部に表示されるブランドの帰属表示行を削除します。ホワイトラベル展開に推奨されます。 |
| **カスタムエージェント名** | *(空欄)* | チャットヘッダーと管理メニューのデフォルトの「Gratis AI Agent」ラベルを、独自の製品名に置き換えます。デフォルトを使用する場合は空欄のままにしてください。 |
| **エージェントピッカーを非表示** | オフ | 有効にすると、ユーザーは組み込みの 5 つのエージェント間を切り替えることができなくなります。現在のエージェントは、設定 → 一般 でデフォルトとして構成されたものに固定されます。 |
| **サイトアイコンをチャットアバターとして使用** | オフ | チャットウィジェットヘッダーのデフォルトの AI アイコンを、WordPress のサイトアイコン（外観 → カスタマイズ → サイトのアイデンティティで設定）に置き換えます。 |

### 変更の適用

フラグを切り替えた後は、**設定を保存** をクリックしてください。変更は即座に適用されます。キャッシュのクリアやプラグインの再アクティベーションは必要ありません。
