# サードパーティモードの移行 {#third-party-mode-migration}

Superdav AI Agent v1.12.0では、サードパーティの機能（アビリティ）の取り扱い方法が変更されました。**サードパーティモードはデフォルトで「自動（auto）」になりました**。これにより、手動での設定なしに、WordPress 7.0以降でネイティブなWordPress Abilities APIの統合が可能になりました。

## 変更点は何ですか？ {#what-changed}

### v1.12.0より前 {#before-v1120}

サードパーティの機能を利用するには、手動での設定が必要でした。

*   「サードパーティモード」を明示的に有効にする必要がありました。
*   機能はカスタムレジストリから読み込まれていました。
*   WordPress Abilities APIとの統合はオプションでした。
*   デフォルトはレガシーモードでした。

### v1.12.0以降 {#after-v1120}

サードパーティの機能は自動で動作します。

*   サードパーティモードのデフォルトが「自動（auto）」になりました。
*   機能がWordPress Abilities APIにネイティブに統合されます。
*   WordPress 7.0以降では手動設定は不要です。
*   古いバージョンのWordPress向けにレガシーモードは引き続き利用可能です。

## 影響を受けるのは誰ですか？ {#who-is-affected}

### 新規インストール（WordPress 7.0以降） {#new-installations-wordpress-70}

**対応は不要です。** サードパーティモードは自動的に「自動（auto）」に設定され、特別な設定なしで機能します。

### 既存のインストール {#existing-installations}

**設定は保持されます。** もし、以前以下を使用していた場合：

*   **レガシーモード**: レガシーモードのままです（変更なし）。
*   **手動サードパーティモード**: 手動モードのままです（変更なし）。
*   **自動モード**: 自動モードのままです（変更なし）。

### WordPress 7.0より前のバージョン {#wordpress-versions-before-70}

**レガシーモードは引き続き利用可能です。** WordPress 6.x以前をご利用の場合：

*   サードパーティモードのデフォルトは「レガシー（legacy）」です。
*   必要に応じてサードパーティモードを手動で有効にできます。
*   ネイティブなAbilities APIを使用するには、WordPress 7.0以降にアップグレードしてください。

## モードの理解 {#understanding-the-modes}

### 自動モード（新しいデフォルト） {#auto-mode-new-default}

**自動モード**は、ネイティブなWordPress Abilities APIの統合を使用します。

*   機能はWordPressのフックを通じて登録されます。
*   WordPress 7.0以降のAbilities APIと完全な互換性があります。
*   サードパーティの機能が自動的に検出されます。
*   手動設定は不要です。

**使用するタイミング**: WordPress 7.0以降でサードパーティの機能を利用する場合

### 手動モード {#manual-mode}

**手動モード**では、明示的な設定が必要です。

*   どのサードパーティの機能を読み込むかを指定します。
*   テストや特定の機能のみを読み込ませたい場合に便利です。
*   設定ファイルの編集が必要です。
*   制御性は高まりますが、設定の手間も増えます。

**使用するタイミング**: テスト、特定の機能の選択的な読み込み、またはカスタム設定を行う場合

### レガシーモード {#legacy-mode}

**レガシーモード**は、古いサードパーティの機能システムを使用します。

*   カスタムの機能レジストリ（WordPress Abilities APIではない）を使用します。
*   古いバージョンのWordPressとの後方互換性があります。
*   ネイティブなWordPress統合はありません。
*   非推奨ですが、引き続きサポートされます。

**使用するタイミング**: WordPress 6.x以前、またはレガシーな互換性が必要な場合

## 現在のモードの確認方法 {#checking-your-current-mode}

### 管理画面から {#via-admin-panel}

1.  **WordPress管理画面** → **Superdav AI Agent** → **設定** に移動します。
2.  **サードパーティモード** の設定を探します。
3.  現在のモードと変更オプションが表示されます。

### コードから {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', または 'legacy'
```

## モードの変更方法 {#changing-your-mode}

### 自動モードへの切り替え {#switch-to-auto-mode}

WordPress 7.0以降で自動モードを使用したい場合：

1.  **Superdav AI Agent** → **設定** に移動します。
2.  **サードパーティモード** を探します。
3.  **Auto (WordPress Abilities API)** を選択します。
4.  **保存** をクリックします。

Superdav AI Agentがサードパーティの機能を自動的に検出して登録します。

### 手動モードへの切り替え {#switch-to-manual-mode}

どの機能を読み込むかを手動で制御したい場合：

1.  **Superdav AI Agent** → **設定** に移動します。
2.  **サードパーティモード** を探します。
3.  **Manual** を選択します。
4.  **保存** をクリックします。
5.  設定ファイルを編集し、読み込ませたい機能を指定します。

### レガシーモードへの切り替え {#switch-to-legacy-mode}

レガシーな互換性が必要な場合：

1.  **Superdav AI Agent** → **設定** に移動します。
2.  **サードパーティモード** を探します。
3.  **Legacy** を選択します。
4.  **保存** をクリックします。

## 自動モードのメリット {#benefits-of-auto-mode}

### 自動検出 {#automatic-discovery}

機能は以下から自動的に検出されます。

*   インストールされているプラグイン
*   アクティブなテーマ
*   must-useプラグイン
*   ドロップインプラグイン

手動での登録は不要です。

### ネイティブ統合 {#native-integration}

機能がWordPress Abilities APIと統合されます。

*   WordPressコアと一貫しています。
*   WordPress管理画面で動作します。
*   Abilities APIを使用する他のプラグインと互換性があります。
*   WordPressの進化に合わせて将来性があります。

### 管理の簡素化 {#simplified-management}

*   編集する設定ファイルがありません。
*   手動での機能登録が不要です。
*   機能の可視性（Visibility）コントロールが自動で機能します。
*   管理通知で分類されていない機能について警告が表示されます。

### パフォーマンスの向上 {#better-performance}

*   機能がキャッシュされます。
*   必要に応じて遅延読み込み（Lazy-loaded）されます。
*   WordPress 7.0以降に最適化されています。

## 移行手順 {#migration-path}

### WordPress 6.xの場合 {#if-youre-on-wordpress-6x}

1.  **WordPress 7.0以降にアップグレード** します（準備ができたら）。
2.  **Superdav AI Agentをv1.12.0以降に更新** します。
3.  **サードパーティモードを自動（Auto）に変更** します（必須ではありません。レガシーモードも引き続き動作します）。
4.  **機能の可視性** を確認し、適切なアクセス制御がされていることを確認します。

### WordPress 7.0以降の場合 {#if-youre-on-wordpress-70}

1.  **Superdav AI Agentをv1.12.0以降に更新** します。
2.  **サードパーティモードが自動（Auto）に設定されているか確認** します（デフォルトでそうなっているはずです）。
3.  **機能の可視性** を確認し、適切なアクセス制御がされていることを確認します。
4.  **サードパーティの機能をテスト** し、正しく動作することを確認します。

## トラブルシューティング {#troubleshooting}

### 自動モードで機能が読み込まれない {#abilities-arent-loading-in-auto-mode}

*   WordPress 7.0以降であることを確認してください。
*   サードパーティモードが「自動（Auto）」に設定されているか確認してください。
*   機能を提供するプラグインがアクティブであることを確認してください。
*   WordPressのエラーログに登録エラーがないか確認してください。

### レガシーモードを維持したい {#i-want-to-keep-legacy-mode}

*   **設定** → **サードパーティモード** に移動します。
*   **Legacy** を選択します。
*   **保存** をクリックします。
*   レガシーモードは引き続き動作します。

### カスタム機能が表示されない {#my-custom-abilities-arent-showing}

*   WordPressのフックを通じて登録されているか確認してください。
*   Abilities APIを正しく実装しているか確認してください。
*   WordPressのエラーログを確認してください。
*   **機能の可視性** 管理画面を使用して、登録されているすべての機能を確認してください。

### 「分類されていない機能」の通知が表示される {#im-getting-unclassified-ability-notices}

*   これは新しいサードパーティの機能では正常な動作です。
*   管理通知で確認し、分類を行ってください。
*   分類の詳細については、**機能の可視性** を参照してください。

## 後方互換性 {#backward-compatibility}

### 既存の設定 {#existing-configurations}

既存のサードパーティの機能設定がある場合：

*   **レガシーモード**: 設定は引き続き動作します。
*   **手動モード**: 設定は引き続き動作します。
*   **自動モード**: 設定は無視されます（自動モードが優先されます）。

カスタム設定を維持したい場合は、手動モードまたはレガシーモードのままでください。

### 非推奨化のタイムライン {#deprecation-timeline}

*   **v1.12.0**: レガシーモードと手動モードは引き続き完全にサポートされます。
*   **v1.13.0以降**: レガシーモードで非推奨通知が表示される場合があります。
*   **v2.0.0**: レガシーモードが削除される場合があります（未定）。

## ベストプラクティス {#best-practices}

### 新規インストールの場合 {#for-new-installations}

*   自動モードを使用してください（デフォルトです）。
*   Superdav AI Agentに機能の自動検出を任せてください。
*   機能の可視性を使用してアクセスを制御してください。

### 既存のインストールの場合 {#for-existing-installations}

*   可能な限りWordPress 7.0以降にアップグレードしてください。
*   管理を簡素化するために自動モードに切り替えてください。
*   機能の可視性を使用して機能を確認し、分類してください。

### カスタム機能の場合 {#for-custom-abilities}

*   WordPressのフック（Abilities API）を通じて機能を登録してください。
*   カスタムの機能レジストリの使用は避けてください。
*   WordPress 7.0以降の自動モードでテストを行ってください。

## 次のステップ {#next-steps}

1.  **WordPressのバージョンを確認**: 自動モードを使用するには7.0以降であることを確認してください。
2.  **サードパーティモードを確認**: 設定に移動し、現在のモードを確認してください。
3.  **必要に応じて更新**: WordPress 7.0以降の場合は自動モードに切り替えてください。
4.  **機能を分類**: 分類されていない機能がないか確認し、分類してください。
5.  **テスト**: サードパーティの機能が正しく動作するか確認してください。
