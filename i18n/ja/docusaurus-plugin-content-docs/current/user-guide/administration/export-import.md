# エクスポートとインポート

Ultimate Multisite 2.9.0では、**ツール > エクスポートとインポート** の場所に、単一サイト用の**エクスポートとインポート**ツールが追加されました。このツールは、WordPressサイト全体をZIPファイルとしてパッケージ化したり、そのZIPファイルを復元したり、互換性のあるUltimate Multisiteと単一サイトのWordPressインストール間でサイトを移行する必要がある場合に使用します。

## 必要な権限 {#required-permissions}

エクスポートまたはインポートを行うサイトのWordPressの**ツール**メニューにアクセスできる管理者としてサインインする必要があります。マルチサイトネットワークの場合、ネットワークレベルのUltimate Multisiteツールからサブサイトをエクスポートまたはインポートする場合は、ネットワーク管理者のアカウントを使用してください。

エクスポートされたZIPファイルのダウンロードは、認証されたダウンロードエンドポイントを通じて提供されるため、ダウンロードが完了するまで管理者セッションをアクティブに保ち、生成されたダウンロードURLを公開共有しないでください。

## サイトをZIPとしてエクスポートする {#exporting-a-site-to-a-zip}

1. コピーしたいサイトのWordPress管理画面に移動し、**ツール > エクスポートとインポート** に進みます。
2. エクスポートエリアを開き、パッケージ化したいサイトを選択します。
3. 利用可能なオプションがある場合は、アップロード、プラグイン、テーマなど、含めたいオプションコンテンツを選択します。
4. エクスポートを開始し、プロセスが完了するまで待ちます。大規模なサイトの場合、ZIPが準備できるまでにバックグラウンドで処理が完了する場合があります。
5. エクスポートリストから完了したZIPファイルをダウンロードします。

ZIPファイルは安全な場所に保管してください。サイトのコンテンツ、設定、メディアファイル、選択されたコードアセットが含まれている可能性があります。

## エクスポートに含まれるもの {#what-the-export-includes}

エクスポートZIPには、以下のものが含まれる場合があります。

*   選択したサイトのデータベースコンテンツと設定。
*   アップロードが選択されている場合のアップロードされたメディアファイル。
*   プラグインやテーマが選択されている場合のプラグインとテーマ。
*   ターゲットのインストールでサイトを再構築するために、エクスポートとインポートツールが使用するインポートメタデータ。

正確なZIPサイズは、メディアの量、選択されたプラグインとテーマ、サイトのデータベーステーブルのサイズによって異なります。

## ZIPからサイトをインポートする {#importing-a-site-from-a-zip}

1. 移行先のWordPressサイトで **ツール > エクスポートとインポート** に移動します。
2. インポートエリアを開き、エクスポートとインポートツールによって作成されたZIPファイルをアップロードします。
3. サイトに新しいアドレスを使用させたい場合は、置換URLを入力するか、元のURLを保持するためにフィールドを空欄のままにします。
4. インポート完了後にアップロードしたZIPファイルを削除するかどうかを選択します。
5. **インポート開始** をクリックします。
6. インポートが完了するまで、保留中のインポートを監視します。プロセスを途中で停止する必要がある場合にのみ **インポートをキャンセル** を使用してください。
7. 通常のトラフィックや顧客アクセスを許可する前に、インポートされたサイトを確認してください。

単一サイトのWordPressインストールの場合、ZIPをインポートすると、現在のサイトがインポートされたサイトに置き換わります。作業を始める前に、必ずターゲットサイトの完全なバックアップを作成し、同じサイトに対して複数のインポートを同時に開始することは避けてください。

## 制限事項と互換性に関する注意点 {#limitations-and-compatibility-notes}

*   非常に大きなアップロードディレクトリやメディアライブラリは、大きなZIPファイルを生成する可能性があります。大規模なサイトをエクスポートまたはインポートする前に、PHPのアップロード制限、実行制限、ディスク容量、メモリ、サーバーのタイムアウト設定を確認してください。
*   非常に大きなメディアライブラリは、トラフィックの少ないメンテナンス時間帯に移動する必要がある場合があります。
*   ターゲットのWordPressインストールは、互換性のあるWordPress、PHP、Ultimate Multisite、プラグイン、テーマのバージョンで動作する必要があります。
*   単一サイトへのインポートは、ターゲットサイトを置き換えます。これはマージツールではありません。
*   マルチサイトから単一サイトへの移行、および単一サイトからマルチサイトへの移行は、ターゲット環境がエクスポートされたサイトのプラグイン、テーマ、URL、および必要なUltimate Multisiteコンポーネントを実行できる場合にのみサポートされます。
*   ZIPファイルは非公開に保ってください。エクスポートされたサイトのデータベースコンテンツ、アップロードファイル、および設定詳細が含まれている可能性があります。

古いネットワークレベルのエクスポートワークフローについては、[Site Export](/user-guide/administration/site-export) を参照してください。
