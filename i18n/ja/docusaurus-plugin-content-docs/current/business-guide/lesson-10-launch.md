# Lesson 10: 立ち上げ日

すべてが構築されました。しかし、実際にオープンする前に、何も壊れていないか、欠けていないか、恥ずかしい思いをしないかをチェックするための、本レッスンでのプレローンチチェックリストを見ていきましょう。

## ここまでの進捗

FitSiteには、テンプレート、プラン、checkout、ブランディング、オンボーディング、価格設定まですべての設定が完了しています。ここからは、すべてが機能しているかを確認し、いよいよ公開（ライブ）へ移行します。

## プレローンチチェックリスト

すべての項目を一つずつ確認してください。飛ばしてはいけません。

### プラットフォームのインフラストラクチャ

- [ ] ホスティングが安定しており、負荷がかかっても良好に動作するか
- [ ] ワイルドカードSSLが有効で、すべてのサブドメインがHTTPS経由でサービスを提供しているか
- [ ] ドメインマッピングが機能するか — サイトの作成とカスタムドメインのマッピングを試す
- [ ] バックアップが設定され、テストされているか（復元を少なくとも1回行い、動作確認をする）
- [ ] モニタリングが導入されているか — プラットフォームがダウンした場合に気づけるか

### テンプレート

- [ ] 3つのすべてのテンプレートが新しいサイトで正しく読み込まれるか
- [ ] ダミーコンテンツが役立ち、誤字脱字がないか
- [ ] すべての画像が適切にライセンスされているか（ストックフォトの透かしが入っていないか）
- [ ] すべてのテンプレートページでモバイルレスポンシブが機能するか
- [ ] ページ読み込み速度が許容範囲か（GTmetrixやPageSpeed Insightsなどのツールでテストする）
- [ ] どのテンプレートにも壊れたリンクや欠落したアセットがないか

### プランと商品

- [ ] 3つのすべてのプランがアクティブで表示されるか
- [ ] プランの説明が正確で、ニッチに特化しているか
- [ ] 価格設定が正しいか（月額と年額）
- [ ] セットアップ料金が適切なプランに設定されているか
- [ ] Starterプランのトライアル期間が機能するか
- [ ] checkout中にOrder bumpsが正しく表示されるか
- [ ] プランごとに、PluginやThemeの制限が正しく適用されているか

### Checkoutフロー

- [ ] 各プランで完全なテストサインアップを行う（テスト支払いモードを使用する）
- [ ] テンプレート選択が、プランごとに正しいテンプレートを表示するか
- [ ] 支払いが正常に処理されるか
- [ ] サインアップ後、顧客がウェルカムメールを受け取るか
- [ ] 新しいサイトが正しいテンプレートで作成されるか
- [ ] 顧客がすぐに新しいサイトにログインできるか
- [ ] ディスカウントコードが正しく機能するか

### ブランディング

- [ ] ログインページにFitSiteのブランディングが表示されるか
- [ ] Admin dashboardにFitSiteのブランディングが表示されるか
- [ ] すべてのシステムメールにFitSiteのブランディングとフィットネス特有の言葉遣いが使われているか
- [ ] 請求書にあなたのビジネス詳細が正しく表示されるか
- [ ] マーケティングサイトが公開され、checkoutフォームにリンクしているか

### オンボーディング

- [ ] Quick Startウィジェットが新しい顧客のダッシュボードに表示されるか
- [ ] すべてのQuick Startリンクが正しいページを指しているか
- [ ] ウェルカムメールシーケンスが設定され、テストされているか
- [ ] ナレッジベースの記事が公開され、アクセス可能か
- [ ] アカウントページに正しいプラン情報とアップグレードオプションが表示されるか

### 法的およびビジネス

- [ ] 利用規約が公開され、checkoutからリンクされているか
- [ ] プライバシーポリシーが公開され、アクセス可能か
- [ ] 返金ポリシーが定義され、文書化されているか
- [ ] 支払いを受け取るためのビジネスエンティティが設定されているか
- [ ] Payment gatewayがライブモード（テストモードではない）であるか
- [ ] 税金の設定があなたの管轄区域に対して正しいか

## ソフトローンチ vs. ハードローンチ

2段階のローンチを検討しましょう。

### フェーズ1：ソフトローンチ

一般公開の前に、5〜10のフィットネススタジオオーナーを招待してサインアップしてもらいます。彼らがベータ顧客です。彼らから以下のものと引き換えに、大幅な割引（生涯50%オフ、または3ヶ月無料）を提供します。

- サインアップおよびオンボーディング体験に関する正直なフィードバック
- 彼らのサイトをショーケースの例として使用する許可
- 彼らが遭遇したバグや問題の報告

これにより、一般公開する前に、実際の顧客からのフィードバックとショーケース用のライブサイトを得ることができます。

### フェーズ2：パブリックローンチ

ソフトローンチのフィードバックを組み込んだ後：

- Payment gatewayをライブモードに切り替える
- マーケティングサイトを公開する
- 顧客獲得を開始する（Lesson 11）
- 関連するフィットネス業界のチャネルで告知する

## 立ち上げ日に行うこと

一般公開する日には：

1. **ライブ支払いへの切り替え** — Payment gatewayのテストモードを無効にする
2. **再確認** — 実際の支払いを使って完全なテストサインアップを行う（後で自分に返金する）
3. **綿密な監視** — エラー、サインアップの失敗、支払い問題がないか注意深く見る
4. **対応可能であること** — 初めての実際の顧客はサポートが必要な場合があり、迅速な対応が信頼を築く
5. **短時間の祝賀** — その後、仕事に戻る

## 何が問題になるか

以下の点に備えてください。

- **Payment gatewayの問題**: ゲートウェイプロバイダーのサポート窓口の連絡先を用意しておく
- **SSL証明書の問題**: 証明書の確認方法と更新方法を知っておく
- **メール配信の失敗**: メールが実際に届くかテストする（迷惑メールフォルダを確認する）
- **負荷時のパフォーマンス**: トラフィックが急増した場合に、ホスティングをスケールする方法を知っておく
- **顧客の混乱**: ナレッジベースとサポートチャネルを用意しておく

## 本レッスンで学んだこと

- インフラストラクチャ、コンテンツ、支払い、法務を網羅した包括的なプレローンチチェックリスト
- 一般公開前に実際のフィードバックを得るためのソフトローンチ戦略
- 自信を持ってライブ移行するための立ち上げ日の手順
- 一般的な立ち上げ日の問題に対するコンティンジェンシープラン
