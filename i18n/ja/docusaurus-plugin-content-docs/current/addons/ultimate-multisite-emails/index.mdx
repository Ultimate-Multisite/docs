# Emails

Ultimate Multisite: Emails アドオンを使用すると、ネットワークオーナーはサイトやメンバーシップと並行して、顧客のメールアカウントをプロビジョニングおよび管理できます。バージョン 1.1.0 では、顧客向けのメールアカウント管理、Webメールのシングルサインオン（SSO）、CyberPanelフォワーダーのサポート、およびチェックアウト時のプロビジョニング機能が強化されました。

## 主な機能

* **顧客メールアカウントウィジェット**: 顧客は、サポートチケットを開くことなく、アカウントページからメールアカウントを作成・管理できます。
* **WebメールSSOとマジックリンク**: プロバイダーが対応している場合、アカウントページからワンクリック認証でWebメールに直接アクセスできます。
* **パスワード変更**: 顧客と管理者の両方が、メール管理ウィジェットからメールボックスのパスワードを更新できます。
* **チェックアウト時のプロビジョニング**: チェックアウト中に顧客のメールアカウント作成を実行できるため、購入フロー完了後にメールボックスが利用可能になります。
* **CyberPanelフォワーダー**: CyberPanelを利用しているネットワークでは、メールサービスフローの一部としてメールフォワーダーを作成・管理できます。
* **プロバイダー対応の設定**: 設定ページが、現在アクティブで設定されているプロバイダーのみにフィルタリングするため、管理者は利用可能なオプションのみを確認できます。

## 要件

* Ultimate Multisite 2.0.0 以上
* WordPress Multisite のインストール
* 設定されたメールプロバイダー統合が少なくとも1つ
* メールボックスの作成、パスワードの変更、およびWebメールアクセスリンクの発行（サポートされている場合）の権限を持つプロバイダーAPI認証情報

## インストール

1. プラグインファイルを `/wp-content/plugins/ultimate-multisite-emails` にアップロードします。
2. WordPressのネットワーク管理画面の**プラグイン**から、アドオンをネットワーク全体で有効化します。
3. **Ultimate Multisite** > **設定** に移動し、アドオンのメール設定を開きます。
4. 顧客に利用してほしいメールプロバイダー統合を有効化します。
5. 設定を保存した後、本番環境ではない顧客またはステージングサイトでアカウント作成をテストしてください。

## 顧客アカウントのメール管理

バージョン 1.1.0 では、顧客アカウントページにメール管理ウィジェットが追加されました。顧客自身が自身のサイトのメールボックスの作成と維持をセルフサービスで行ってほしい場合に利用してください。

アカウントページから、顧客は以下のことができます。

1. 現在のメンバーシップまたはサイトに接続されているメールアカウントの確認。
2. プランが追加アカウントを許可している場合、新しいメールボックスの作成。
3. 既存のメールボックスのパスワードの変更。
4. サポートされているWebメールプロバイダーをSSOまたはマジックリンクログイン経由で開く。

このウィジェットは Ultimate Multisite のアカウントページ現在のシステムを使用するため、アクションは顧客が閲覧しているアカウントのコンテキストに限定されます。これにより、顧客が別のメンバーシップやサイトに属するメールアカウントを管理してしまうことを防ぎます。

## WebメールSSOとマジックリンクログイン

Webメール SSO は、アカウントページから顧客に直接「Webメールを開く」アクションを提供します。アクティブなプロバイダーによってサポートされている場合、アドオンは短期間のアクセスURLを要求し、顧客にメールボックスの認証情報をコピーさせることなくWebメールにリダイレクトします。

プロバイダーの動作は異なる場合があります。

* ネイティブなSSO APIを持つプロバイダーは、直接認証されたWebメールセッションを作成できます。
* マジックリンクのサポートを持つプロバイダーは、時間制限のあるログインリンクを生成できます。
* SSOまたはマジックリンクのサポートがないプロバイダーでも、メールボックスの詳細とパスワード管理のアクションは表示されますが、ワンクリックのWebメールボタンは表示されません。

Webメールアクセスは、必要なAPI認証情報を設定したプロバイダーでのみ有効にしてください。アクションがプロバイダーに存在しない場合は、まずそのプロバイダーがアクティブで、設定されており、Webメールログインリンクをサポートしていることを確認してください。

## チェックアウト時のプロビジョニング

アドオンは、チェックアウトの一部として顧客のメールアカウントを作成できます。これは、プランにデフォルトでメールボックスが含まれている場合や、チェックアウトフォームでメールアカウントの詳細を収集する場合に便利です。

推奨される導入手順:

1. チェックアウト時のプロビジョニングを有効にする前に、プロバイダーとプランの制限を設定してください。
2. ステージング製品での購入をテストし、チェックアウト後にメールボックスが作成されることを確認してください。
3. 顧客アカウントページを確認し、新しいメールボックスがメールウィジェットに表示されていることを確認してください。
4. 本番製品でフローを有効にする前に、パスワード変更とWebメールのアクションを確認してください。

プロビジョニングが失敗した場合、アカウントとメンバーシップは存在し続ける可能性がありますが、メールボックスには管理者のフォローアップが必要です。再試行する前に、Ultimate Multisite のアクティビティログとプロバイダーのログを確認してください。

## CyberPanelメールフォワーダー

CyberPanelを利用しているネットワークの場合、バージョン 1.1.0 でメールフォワーダーのサポートが追加されました。フォワーダーは、完全なメールボックスを作成することなく、`info@customer-domain.test` のようなアドレスが別の受信トレイにメールを転送する必要がある場合に役立ちます。

フォワーダーを提供する前に、CyberPanel統合が設定され、到達可能であることを確認してください。必要なホスト、ユーザー名、パスワード、およびパッケージ定数については、[CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) ガイドを参照してください。

## よくある質問

## 顧客は自分でメールアカウントを作成できますか？

はい。顧客アカウントウィジェットが有効で、プランがメールアカウントを許可している場合、顧客はアカウントページから独自のメールボックスを作成・管理できます。

## すべてのプロバイダーがWebメールSSOをサポートしていますか？

いいえ。SSOおよびマジックリンクのサポートは、アクティブなメールプロバイダーに依存します。アクセスリンクAPIを持たないプロバイダーでも、メールボックスの作成とパスワードの変更をサポートできます。

## 管理者は顧客のメールアカウントを作成できますか？

はい。管理者は、アカウント作成時に初期パスワードを設定するなどして、管理画面からメールアカウントを作成できます。

## フォワーダーはメールボックスと同じですか？

いいえ。メールボックスはメールを保存し、ログイン認証情報を持っています。フォワーダーはメールを別の宛先に転送するだけであり、個別の受信トレイは提供しません。
