# GoCardless

Multisite Ultimate: GoCardless 決済

GoCardless を使ったダイレクトデビットで、顧客が Multisite Ultimate のメンバーシップや商品を支払えるようにしましょう！

## インストール

1. 'ultimate-multisite-gocardless' を '/wp-content/plugins/' ディレクトリにアップロードします
2. WordPress の 'プラグイン' メニューから plugin を有効化します
3. ステップごとのウィザードに従って plugin を設定します

## 使い方ガイド

_GoCardless_ は、**ヨーロッパと英国**で利用できる決済ゲートウェイで、**ダイレクトデビット**決済に対応しています。

**Ultimate Multisite: GoCardless Gateway** は Ultimate Multisite に _GoCardless_ 対応を追加し、その地域の顧客にダイレクトデビットで請求できるようにします。

このチュートリアルでは、Ultimate Multisite ネットワークで _GoCardless_ を通じた支払い受付を開始するための設定方法を説明します。

_**注: GoCardless は次の通貨でのみ動作します:**_ **EUR、GBP、AUD、SEK、DKK。**

## アドオンのインストール

まず、Ultimate Multisite 用の GoCardless アドオンをインストールする必要があります。**Ultimate Multisite > 設定 > アドオン（サイドバー）**に移動し、その一覧から GoCardless アドオンを選択します。

アドオンをインストールして有効化したら、**Ultimate Multisite > 設定 > 決済**に移動し、GoCardless オプションを切り替えて保存します。

![決済ゲートウェイ設定](/img/config/settings-payment-gateways.png)

## Access Token の取得

_GoCardless_ で Account を作成します（**まずテストする**場合は _GoCardless_ サンドボックスで作成します。推奨です）。その後、ログインして **Developers** メニュー項目に移動します。

<!-- スクリーンショットなし: GoCardless Dashboard で Developers メニューの位置を表示 -->

**Developers** ページで、画面右上の **Create** ボタンをクリックし、**Access Token** オプションをクリックします。

<!-- スクリーンショットなし: GoCardless Developers ページの Create ボタンと Access Token オプション -->

新しいモーダルウィンドウが開き、新しいトークンを作成できます。識別しやすい名前を付け、**Scope** メニューで **Read-write access** オプションを選択していることを確認します。

<!-- スクリーンショットなし: Read-write スコープ付きの新しい Access Token を作成する GoCardless モーダル -->

Access Token が画面に表示されます。それをコピーして、**Ultimate Multisite > 設定 > 決済 > GoCardless** メニューに貼り付けます。

必ず正しいフィールドに入力してください。それが**本番用トーク** **ン**の場合は、トークンを貼り付ける前に **sandbox mode off** に切り替えます。

![GoCardless 付き決済ゲートウェイ設定](/img/config/settings-payment-gateways.png)

<!-- スクリーンショットなし: Ultimate Multisite 決済設定内の GoCardless トークンフィールド -->

最初の手順はこれで完了です。次に、webhook リスナーを設定する必要があります。

## Webhook リスナーの設定

サブスクリプションを扱う場合、Ultimate Multisite はサブスクリプション状態の変更、新しい支払いの実行、その他同様の情報を受け取る必要があります。

決済ゲートウェイが Ultimate Multisite にそれらの変更を通知する方法は、**webhook 呼び出し**を使うことです。そのため、連携を完全に機能させるには、そのすべての情報をどの URL に送信するかを _GoCardless_ に伝える必要があります。

**GoCardless > Developers** パネルに戻り、もう一度 Create ボタンをクリックします。ただし今回は **Webhook endpoint** オプションを選択します。

<!-- スクリーンショットなし: Webhook endpoint オプションがある GoCardless Developers ページ -->webhook endpoint オプションの新しいウィンドウが開きます。webhook URL を求められるため、**Ultimate Multisite 設定 > 決済 > GoCardless** に戻り、そこに表示されている webhook URL をコピーします。

<!-- スクリーンショットなし: webhook URL を表示している Ultimate Multisite GoCardless 設定 -->

さらに、webhook 呼び出しの署名に使用する **webhook secret** を入力できます。そのフィールドに入力した内容は、**Ultimate Multisite Webhook Secret フィールド**にも入力する必要があります。

<!-- スクリーンショットなし: URL とシークレットフィールドを含む GoCardless webhook endpoint 設定フォーム -->

<!-- スクリーンショットなし: Ultimate Multisite GoCardless webhook secret フィールド -->

最後に、**Create webhook endpoint** ボタンをクリックして endpoint を保存し、**Ultimate Multisite 設定も保存**します。

これで _GoCardless_ を受け付ける準備が整いました！
