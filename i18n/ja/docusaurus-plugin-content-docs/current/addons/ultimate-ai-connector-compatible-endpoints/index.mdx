# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints は、OpenAI互換のAIプロバイダーAPIをWordPressマルチサイトネットワークに統合します。複数のAIプロバイダー（それぞれに独自のエンドポイントURL、APIキー、デフォルトモデルを設定可能）を構成し、プロバイダーが利用できない場合に自動的にリクエストを切り替える（フォールバック）ことができます。

## 主な機能

- **マルチプロバイダー対応**: 複数のAIプロバイダーカードを追加でき、それぞれに独自のエンドポイントURL、APIキー、デフォルトモデルを設定できます。
- **ドラッグ＆ドロップでの並び替え**: 各カードのグリップハンドルを使ってプロバイダーの優先順位を調整できます。リクエストは設定された順にプロバイダーを使用します。
- **自動フォールバックルーティング**: プライマリプロバイダーが利用できない場合、リクエストは自動的に次に設定されたプロバイダーに切り替わります。
- **プロバイダーごとの設定**: 各プロバイダーは、独自の認証情報とモデル選択に基づいて個別に設定できます。
- **カードベースのUI**: 安定したWordPress 6.9+のCardコンポーネント（CardBody, CardHeader, CardDivider）を基に構築されています。
- **追加時の自動展開**: 新しいプロバイダーカードを追加すると自動的に展開し、すぐに詳細を入力できます。
- **スクリプトのキャッシュバスター**: プラグイン更新時にプロバイダーのスクリプトがキャッシュバスターされ、最新バージョンが確実に読み込まれます。

## 要件

- WordPress 5.3 以上
- PHP 7.4 以上
- Ultimate Multisite プラグイン（有効化）
- OpenAI互換のAPIエンドポイント（セルフホストまたはサードパーティ）

## インストール

1. アドオンファイルを `/wp-content/plugins/` ディレクトリにアップロードします。
2. WordPressの「プラグイン」メニューからプラグインを有効化します。
3. **Ultimate Multisite → AI Connector** に移動してアドオンを設定します。

## 設定

### マルチプロバイダーのセットアップ

設定画面はカードベースのレイアウトになっており、各カードが1つのAIプロバイダーを表します。必要なだけプロバイダーを追加できます。

#### プロバイダーの追加

1. ネットワーク管理画面で **Ultimate Multisite → AI Connector** に移動します。
2. **Add Provider** をクリックします。
3. 新しいプロバイダーカードが表示され、自動的に展開します。
4. プロバイダーの詳細を入力します。
   - **Provider Name** — このプロバイダーを識別するためのラベル（例：「OpenAI」、「Mistral」、「ローカルOllama」）
   - **Endpoint URL** — OpenAI互換APIのベースURL（例：`https://api.openai.com/v1`）
   - **API Key** — このプロバイダー用のAPIキー
   - **Default Model** — このプロバイダーのデフォルトで使用するモデル識別子（例：`gpt-4o`、`mistral-small`）
5. **Save Settings** をクリックします。

#### プロバイダーの削除

プロバイダーカードの上の **Remove** ボタンをクリックすると削除できます。設定を保存して変更を適用してください。

#### プロバイダーの並び替え

プロバイダーカードの左側にあるグリップアイコン（⣿）をドラッグして並び替えができます。この順序がフォールバックの優先順位を決定します。リクエストは最初のプロバイダーに送信され、リクエストが失敗した場合、次のプロバイダーへとフォールバックします。

### フォールバックルーティング

プライマリプロバイダーへのリクエストがタイムアウト、レート制限、サービス停止などの理由で失敗した場合、コネクタは自動的にリスト内の次のプロバイダーで再試行します。これは、プロバイダーが成功するか、すべてのプロバイダーを試すまでリストを順に続きます。

| シナリオ | 動作 |
|----------|-----------|
| プライマリプロバイダーが応答した場合 | リクエストは通常通り処理されます |
| プライマリプロバイダーが失敗した場合 | 次のプロバイダーでリクエストが再試行されます |
| すべてのプロバイダーが失敗した場合 | エラーが呼び出し元に返されます |

フォールバックは、呼び出し元のプラグインやテーマからは透過的です。試行されたプロバイダーの数に関わらず、成功した応答、または最終的なエラーのいずれかを受け取ります。

### プロバイダー設定リファレンス

| フィールド | 必須 | 説明 |
|-------|----------|-------------|
| Provider Name | はい | このプロバイダーの表示ラベル |
| Endpoint URL | はい | OpenAI互換APIのベースURL |
| API Key | はい | このプロバイダーの認証キー |
| Default Model | はい | リクエストでモデルが指定されていない場合に使用されるモデル識別子 |

## トラブルシューティング

### プロバイダーがリストに表示されない

保存後もプロバイダーリストが空の場合は、以下の点を確認してください。

- プロバイダーを追加した後、**Save Settings** をクリックしたか
- ブラウザが設定ページの保存をブロックしていないか（ブラウザのコンソールでJavaScriptエラーを確認してください）
- プラグインが有効化されており、ネットワーク管理画面にいるか

### すべてのプロバイダーでリクエストが失敗する

- 各プロバイダーのエンドポイントURLが、サーバーから到達可能か（ブラウザからだけではなく）確認してください。
- 各APIキーが有効であり、期限切れになっていないか確認してください。
- プロバイダーのダッシュボードで、レート制限とクォータの状態を確認してください。
- 詳細なエラーメッセージをキャプチャするために、WordPressのデバッグロギング（`WP_DEBUG_LOG`）を有効にしてください。

### フォールバックに使用されるプロバイダーが異なる

ドラッグ＆ドロップでの並び替えはフォールバックの順序を変更します。フォールバックリクエストを受け取るはずのプロバイダーがリスト内で正しい位置に配置されているか、また、そのエンドポイントURLとAPIキーが正しいかを確認してください。

### 新しいプロバイダーカードが自動展開しない

自動展開機能は、ブラウザでJavaScriptが有効になっている必要があります。カードがクリックしても展開しない場合は、ブラウザのコンソールでエラーを確認してください。自動展開がトリガーされない場合でも、カードヘッダーをクリックすることでカードの内容は常にアクセス可能です。
