# ツールと設定

Theme Builder は、一連のツールを使って WordPress サイトを分析、設計、構築します。バージョン 1.16.0 では、**sd-ai-agent/site-scrape が Tier 1 ツールになり**、Theme Builder で標準利用できるようになりました。Superdav AI Agent v1.18.0 では、信頼できる開発者ワークフロー向けに別配布の Advanced companion package も導入されています。これらのツールは WordPress.org のコアパッケージには含まれず、別途インストールして承認する必要があります。

## 利用可能なツール

### Tier 1 ツール（常に利用可能）

Tier 1 ツールは、追加設定なしで Theme Builder から標準で利用できます。

#### sd-ai-agent/site-scrape

**目的:** 既存のウェブサイトを分析し、デザインの着想、コンテンツ構造、機能を抽出します。

**機能:**

- **ウェブサイト分析** — 競合サイトや参考サイトをスクレイピングして分析
- **デザイン抽出** — 色、フォント、レイアウトパターンを特定
- **コンテンツ構造** — ページ構成と階層を把握
- **機能検出** — plugin と機能を特定
- **パフォーマンス分析** — ページ速度と最適化を確認
- **SEO 分析** — メタタグと構造化データを確認

**使い方:**

```
example.com のデザインを分析し、私のサイトの着想として使ってください。
```

**抽出される内容:**

- カラーパレットと色の使い方
- タイポグラフィ（フォントとサイズ）
- レイアウト構造と余白
- ナビゲーションパターン
- コンテンツ構成
- 画像とメディアの使い方
- インタラクティブ要素
- モバイル対応

**制限事項:**

- パスワード保護されたサイトはスクレイピングできません
- robots.txt とサイトポリシーを尊重します
- 動的コンテンツを取得できない場合があります
- JavaScript の多いサイトでは分析が限定される場合があります
- 大規模サイトは分析に時間がかかる場合があります

### Tier 2 ツール（任意）

Tier 2 ツールは、Theme Builder 設定で明示的に有効化すると利用できます。

#### 高度な分析

詳細なパフォーマンス指標を提供します:

- ページ読み込み時間
- Core Web Vitals
- SEO スコア
- アクセシビリティスコア
- ベストプラクティススコア

#### コンテンツ最適化

以下を分析し、改善を提案します:

- 読みやすさ
- SEO 最適化
- キーワードの使用
- コンテンツ構造
- 行動喚起の配置

### Tier 3 ツール（Premium）

Tier 3 ツールには、追加設定または premium 機能が必要です。

#### AI コンテンツ生成

以下のコンテンツを生成します:

- product 説明
- サービスページ
- ブログ記事
- メタ説明
- 行動喚起コピー

#### 高度なデザインツール

高度なデザイン機能を提供します:

- カスタム CSS 生成
- アニメーション作成
- インタラクティブ要素のデザイン
- 高度な色彩理論
- タイポグラフィ最適化

### Advanced Companion Tools（別パッケージ）

Advanced companion package は、Superdav AI Agent のコアリリースとは別に配布されます。管理者が、影響の大きいツールの使用を agent に明示的に許可する、信頼できる開発者および site 所有者環境を対象としています。

Advanced companion の機能には、以下が含まれる場合があります:

- **ファイルシステムツール** — 信頼できる環境で承認済みファイルを確認、変更
- **データベースツール** — 明示的に承認された場合に site データを照会または更新
- **WP-CLI ツール** — WordPress の保守および検査コマンドを実行
- **REST ディスパッチャーツール** — 制御されたディスパッチワークフローを通じて登録済み REST endpoint を呼び出し
- **Plugin ビルダーツール** — plugin コードをひな形作成し、反復改善
- **Git スナップショットツール** — リスクのある開発操作の前にスナップショットを作成
- **ユーザー管理ツール** — 許可された範囲でユーザー記録の確認や調整を支援
- **ベンチマークツール** — 開発レビュー向けにパフォーマンスまたは機能ベンチマークを実行

これらを、すべてのインストールで一般利用できる Theme Builder ツールとして記載しないでください。Advanced companion package がインストール、有効化され、信頼できる管理者に制限されている場合にのみ利用できます。

## ツール設定

### ツールの有効化

Theme Builder で追加ツールを有効にするには:

1. **Gratis AI Agent → 設定** に移動します
2. **Theme Builder → ツール** に進みます
3. 必要に応じてツールのオン/オフを切り替えます
4. 設定を保存します

### ツール権限

一部のツールには権限が必要です:

- **サイトスクレイピング** — インターネットアクセスが必要です
- **分析** — Google Analytics 連携が必要です
- **コンテンツ生成** — API キーが必要です
- **高度な機能** — premium subscription が必要な場合があります
- **Advanced companion tools** — 別配布の Advanced package と管理者による明示的な信頼が必要です

### API キーと認証情報

必要なツールの API キーを設定します:

1. **Gratis AI Agent → 設定 → API キー** に移動します
2. 各ツールの認証情報を入力します
3. 接続をテストします
4. 安全に保存します

### Advanced Companion Package のインストール

Advanced companion package は、WordPress.org コア ZIP とは別にリリースされます。対象リリースの公式プロジェクト配布チャネルからのみインストールし、その後、信頼できる管理者にアクセスを制限してください。本番 site でファイルシステム、データベース、WP-CLI、REST ディスパッチャー、plugin ビルダー、ユーザー管理、またはベンチマークツールを有効にする前に、人による承認ゲートと git スナップショットの動作を確認してください。

## sd-ai-agent/site-scrape の使用

### 基本的な使い方

Theme Builder にウェブサイトの分析を依頼します:

```
competitor.com にある競合サイトのデザインを分析し、
私のサイトのデザイン改善を提案してください。
```

### 特定の分析

特定の種類の分析を依頼します:

```
example.com からカラーパレットを抽出し、着想として使ってください。
```

```
example.com のナビゲーション構造を分析し、私のサイトに同様の
構成を適用してください。
```

```
example.com のモバイル対応を確認し、私のサイトも
同等にレスポンシブであることを確保してください。
```

### 比較分析

複数のサイトを比較します:

```
site1.com と site2.com のデザインを比較し、両方の優れた要素を
組み合わせたデザインを作成してください。
```

## ツールの制限事項と考慮点

### レート制限

- スクレイピングはサーバー過負荷を防ぐためにレート制限されます
- 同じサイトへの複数のリクエストは制限される場合があります
- 大規模なサイトは分析に時間がかかる場合があります

### コンテンツの正確性

- 動的コンテンツは完全に取得されない場合があります
- JavaScript でレンダリングされるコンテンツは不完全な場合があります
- リアルタイムデータは反映されない場合があります
- 一部のコンテンツはペイウォールの内側にある場合があります

### 法的および倫理的な考慮事項

- robots.txt とサイトのポリシーを尊重してください
- 著作権で保護されたコンテンツを複製目的でスクレイピングしないでください
- コピーではなく、着想を得るために分析を使用してください
- 抽出したコンテンツを使用する権利があることを確認してください
- 分析対象サイトの利用規約に従ってください

### パフォーマンスへの影響

- 大規模なサイトのスクレイピングには時間がかかる場合があります
- 複数の同時スクレイピングは遅くなる場合があります
- ネットワーク接続が速度に影響します
- 大きなデータセットはより多くの処理が必要な場合があります

## ベストプラクティス

### サイト分析の使用

1. **複数のサイトを分析する** — 複数の情報源から着想を集めます
2. **構造に注目する** — レイアウトと構成のパターンを学びます
3. **色を抽出する** — カラーパレットを出発点として使用します
4. **タイポグラフィを調べる** — 好みのフォントの組み合わせを特定します
5. **ナビゲーションを確認する** — メニュー構成を理解します

### 倫理的なスクレイピング

1. **robots.txt を尊重する** — サイトのガイドラインに従います
2. **コンテンツをコピーしない** — 分析は着想を得るためだけに使用します
3. **権利を確認する** — 抽出したコンテンツを使用できることを確認します
4. **出典を明記する** — 着想元をクレジットします
5. **規約に従う** — サイトの利用規約を遵守します

### ツールの効果を最大化する

1. **具体的にする** — 具体的な分析タイプを依頼します
2. **文脈を提供する** — 自分のサイトの目的を説明します
3. **期待値を設定する** — デザイン目標を説明します
4. **反復する** — 結果に基づいて改善します
5. **ツールを組み合わせる** — 包括的な分析のために複数のツールを使用します

## トラブルシューティング

### サイトをスクレイピングできない

- サイトが一般公開されているか確認してください
- robots.txt がスクレイピングを許可しているか確認してください
- 別のサイトを試してください
- インターネット接続を確認してください
- 問題が続く場合はサポートに連絡してください

### 不完全な分析

- サイトに動的コンテンツが含まれている場合があります
- JavaScript が完全にレンダリングされない場合があります
- 大規模なサイトではタイムアウトする場合があります
- 代わりに特定のページを分析してみてください
- 特定の分析タイプを依頼してください

### パフォーマンスが遅い

- 大規模なサイトは分析に時間がかかります
- 複数の同時スクレイピングは遅くなります
- ネットワーク接続が速度に影響します
- 混雑していない時間帯に試してください
- まず小さなセクションを分析してください
