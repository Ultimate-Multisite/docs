# Multi-Network

Ultimate Multisite: Multinetwork アドオンは、強力なマルチネットワーク機能で Ultimate Multisite プラットフォームを拡張します。このアドオンにより、顧客は個別のサイトだけでなく、WordPress マルチサイトネットワーク全体を購入して管理できます。

## 主な機能 {#key-features}

* **ネットワークのプロビジョニング**: 顧客向けに完全な WordPress マルチサイトネットワークを自動で作成し、設定します
* **ネットワークテンプレート**: 登録前に、顧客が事前構築済みのネットワークテンプレートを閲覧、プレビュー、選択できます
* **ネットワーク管理**: マルチネットワーク環境向けの包括的な管理機能
* **分離されたネットワーク**: 各顧客ネットワークは、独自のデータベーステーブルと設定で独立して動作します
* **ネットワーク間 SSO**: マジックリンク認証により、顧客がサブネットワークへシームレスにアクセスできます
* **スケーラブルなアーキテクチャ**: 最適化されたリソース割り当てにより、複数のネットワークを効率的に処理します
* **連携**: Ultimate Multisite のサブスクリプションおよび請求システムとシームレスに連携します

## ユースケース {#use-cases}

* **ホワイトラベルホスティング**: 顧客に独自ブランドのマルチサイトネットワークを提供します
* **代理店向けソリューション**: 代理店が単一のプラットフォームから複数のクライアントネットワークを管理できます
* **エンタープライズ展開**: 複雑なマルチネットワーク要件を持つ大規模組織をサポートします
* **教育プラットフォーム**: 学校や部門ごとに個別のネットワークを作成します

## 要件 {#requirements}

* Ultimate Multisite 2.0.0 以上
* WordPress Multisite インストール
* PHP 7.4 以上
* MySQL 5.6 以上

## インストール {#installation}

1. プラグインファイルを `/wp-content/plugins/ultimate-multisite-multinetwork` ディレクトリにアップロードします
2. WordPress の「プラグイン」画面からプラグインを有効化します
3. Ultimate Multisite → 設定 からマルチネットワーク設定を構成します

## ネットワークテンプレート {#network-templates}

バージョン 1.3.0 では、新しいネットワークを購入する顧客向けに、テンプレートの検出と選択のフローが追加されます。新しい顧客ネットワークを、準備済みの構造、デザイン、コンテンツライブラリ、または業種特化の設定から開始したい場合は、ネットワークテンプレートを使用します。

### テンプレートプレビュー {#template-previewer}

ネットワークテンプレートプレビューにより、顧客は購入手続き前に利用可能なテンプレートを閲覧できます。各テンプレートはライブプレビューパネルで開けるため、顧客はネットワークを購入する前にレイアウトやコンテンツを比較できます。

プレビューを使うと、顧客は次のことができます。

* 利用可能な各ネットワークテンプレートの見た目と構造を確認する
* 購入フローを離れずに複数のテンプレートを比較する
* 登録前に、自分の用途に最も合うテンプレートを選択する

テンプレート名と説明は、顧客に分かりやすいものにしてください。代理店ネットワーク、学校ネットワーク、フランチャイズネットワーク、ホワイトラベルホスティングバンドルなど、想定用途を顧客が理解できる場合に、プレビューは最も役立ちます。

### 購入手続きでのテンプレート選択 {#template-picker-at-checkout}

テンプレート選択機能は、登録時の購入手続きフローにフィールドとして追加できます。設定すると、顧客は購入手続き中に希望するネットワークテンプレートを選択し、選択されたテンプレートが新しいネットワークのプロビジョニング時に使用されます。

購入手続きフローで選択機能を使うには、次の手順を行います。

1. テンプレートとして提供したいネットワークを準備します。
2. 関連する登録フォームまたは購入手続きフォームで、Multinetwork テンプレート選択フィールドを有効化します。
3. その購入手続きフローで利用可能にするテンプレートを選択します。
4. 購入手続きフローを公開し、登録をテストして、選択したテンプレートが新しいネットワークに適用されることを確認します。

購入手続きフローで提供するネットワーク商品が 1 つだけの場合でも、選択機能は案内付きオンボーディング手順として役立ちます。複数プランの導線では、選択された商品またはプランと互換性のあるテンプレートだけを表示してください。

## ネットワーク間マジックリンク SSO {#cross-network-magic-link-sso}

ネットワーク間マジックリンク SSO により、顧客は再度手動でサインインすることなく、主要な Account 領域とサブネットワークの間を移動できます。顧客がネットワーク間アクセスリンクをたどると、Ultimate Multisite は有効期限付きのマジックリンクを生成し、移動先ネットワークで顧客を認証します。

これにより、複数のネットワークを管理する顧客や、購入直後に新しく複製されたテンプレートネットワークへすぐアクセスする必要がある顧客の体験が向上します。

管理者チェックリスト:

* Multinetwork アドオンが WordPress マルチサイトインストール全体で有効になっていることを確認します。
* 顧客にネットワーク間リンクを表示する前に、正しいネットワークアクセス権があることを確認します。
* マジックリンクが意図した移動先ネットワークに解決されるよう、ネットワークドメインとマッピング済みドメインを正しく設定しておきます。
* ネットワーク、ドメイン、または購入手続き設定を変更した後は、顧客 Dashboard と購入手続き後のアクセスボタンをテストします。

マジックリンクはログインリンクとして扱う必要があります。これは顧客のシームレスなアクセス用に設計されており、意図したセッション外で転送または再利用すべきではありません。

## よくある質問 {#frequently-asked-questions}

### マルチネットワークとは何ですか？ {#what-is-multinetwork}

マルチネットワークは、単一の WordPress インストール上で複数の独立したマルチサイトネットワークを実行できる WordPress の機能です。

### 通常のマルチサイトとの違いは何ですか？ {#how-is-this-different-from-regular-multisite}

通常のマルチサイトでは、1 つのネットワーク内に複数のサイトを持つことができます。マルチネットワークでは、複数のネットワークを持つことができ、それぞれが独自のサイトを持ち、すべてが 1 つの WordPress インストール内で動作します。

### 特別なサーバー設定は必要ですか？ {#does-this-require-special-server-configuration}

はい、マルチネットワークには特定のサーバーセットアップと設定が必要です。詳しいセットアップ手順については、ドキュメントを参照してください。
