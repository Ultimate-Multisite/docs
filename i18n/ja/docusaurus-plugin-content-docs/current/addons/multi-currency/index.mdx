# Multi-Currency

Ultimate Multisite: Multi-Currencyを使用すると、ネットワーク全体で複数の通貨での支払いを受け付けることができます。本アドオンは、ライブまたは手動の交換レートを使用して価格を自動的に換算し、任意の通貨で商品ごとの正確な価格を設定できるようにし、さらにジオロケーション（地理的位置情報）を通じて訪問者が好む通貨を自動で検出します。これらすべてを単一の管理ダッシュボードから管理できます。

## 主な機能 {#key-features}

- **自動為替レート換算** — 選択したプロバイダーから取得したレートを使用して、基本通貨から価格を換算します。
- **商品ごとの価格上書き** — 自動換算をバイパスし、任意の商品に通貨ごとの正確な価格を設定できます。
- **通貨セレクターのサインアップフィールド** — チェックアウト時やサインアップ時に、顧客が希望の通貨を選択できます。
- **複数の為替レートプロバイダー** — ECB、Frankfurter、Stripe、または完全に手動でのレート設定が可能です。
- **ジオロケーションに基づく通貨検出** — 訪問者の国に一致する通貨を自動で事前選択します。
- **管理ダッシュボード** — アクティブな通貨、為替レート、プロバイダーの設定をすべて一箇所で管理できます。

## 要件 {#requirements}

- Ultimate Multisite 2.0.0以上
- WordPress Multisiteのインストール
- PHP 7.4以上

## インストール {#installation}

1. プラグインファイルを `/wp-content/plugins/ultimate-multisite-multi-currency` にアップロードします。
2. WordPressの**プラグイン**画面からプラグインを有効化します（ネットワーク有効化）。
3. **Ultimate Multisite → Multi-Currency** に移動し、管理ダッシュボードを開きます。
4. 受け付けたい通貨を追加し、為替レートプロバイダーを設定します。

## 設定 {#configuration}

### 管理ダッシュボード {#admin-dashboard}

Multi-Currencyの管理ダッシュボード（**Ultimate Multisite → Multi-Currency**）は、以下の作業を行うための中心的な場所です。

- 通貨の追加、有効化、または無効化
- 為替レートプロバイダーの選択と、オンデマンドでのレート更新
- すべてのアクティブな通貨の現在のレートの確認
- 自動プロバイダーが適切でない場合の、手動レートの設定

### 為替レートプロバイダー {#exchange-rate-providers}

| Provider | Description |
|---|---|
| **ECB** | 欧州中央銀行 — EURベースのネットワーク向けの無料の日次レート |
| **Frankfurter** | オープンソースのECBミラーAPI — 無料でAPIキーは不要 |
| **Stripe** | StripeのライブFXレートを使用 — アクティブなStripe連携が必要です |
| **Manual** | レートを直接入力します。外部API呼び出しは不要です。 |

プロバイダーは、**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** から選択します。レートはいつでも手動で更新するか、自動で更新するようスケジュール設定が可能です。

### 商品ごとの価格上書き {#per-product-pricing-overrides}

自動換算では精度が不十分な場合、任意の商品に対して特定の通貨の正確な価格を設定できます。

1. 商品エディターを開きます（**Ultimate Multisite → Products → Edit**）。
2. **Currency Pricing** セクションまでスクロールします。
3. 上書きしたい各通貨の正確な価格を入力します。
4. 商品を保存します。

上書き設定のない商品は、基本通貨から換算された価格が使用されます。

### 通貨セレクターのサインアップフィールド {#currency-selector-signup-field}

この通貨セレクターフィールドをサインアップ/チェックアウトフローに追加することで、顧客が登録時に希望の通貨を選択できるようになります。

1. **Ultimate Multisite → Multi-Currency → Settings** に移動します。
2. **Show currency selector on signup** を有効にします。
3. オプションでデフォルトの通貨を設定するか、ジオロケーションに事前選択値を決定させることができます。

選択された通貨はサブスクリプションに保存され、以降のすべての請求に使用されます。

### ジオロケーションに基づく通貨検出 {#geolocation-based-currency-detection}

ジオロケーション検出を有効にすると、アドオンが訪問者の国を通貨にマッピングし、通貨セレクターフィールドでそれを事前選択します。有効にするには：

1. **Ultimate Multisite → Multi-Currency → Settings** に移動します。
2. **Detect currency from visitor location** を有効にします。
3. **Geolocation Mapping** テーブルで国を通貨にマッピングします。

訪問者の国にマッピングがない場合、ネットワークの基本通貨がフォールバックとして使用されます。

## よくある質問 {#frequently-asked-questions}

### 為替レートプロバイダーが利用できない場合はどうなりますか？ {#what-happens-if-an-exchange-rate-provider-is-unavailable}

アドオンは、最後に正常に取得されたレートにフォールバックします。一度もレートが取得されていない場合は、設定した手動レートにフォールバックします。安全策として、手動レートを設定しておくことをお勧めします。

### 異なる通貨に異なるプロバイダーを使用できますか？ {#can-i-use-different-providers-for-different-currencies}

いいえ。すべての通貨に対して、一度に1つのプロバイダーのみがアクティブになります。個々のレートを完全に制御したい場合は、**Manual** を使用してください。

### ジオロケーションは別のプラグインが必要ですか？ {#does-geolocation-require-a-separate-plugin}

外部のジオロケーションプラグインは必要ありません。アドオンはWordPressに組み込まれているIPから国への検出機能を使用します。より高い精度が必要な場合は、提供されているフィルターフック経由で専用のジオロケーションサービスを統合できます。

### 商品ごとの上書き価格は、税金の前ですか、後ですか？ {#are-per-product-overrides-applied-before-or-after-tax}

商品ごとの上書き価格は、ターゲット通貨での税抜き価格を設定します。税金は、既存の税金設定を使用して、上書き価格に追加で計算されます。
