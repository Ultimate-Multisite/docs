# Ultimate AI Connector for WebLLM 変更履歴

## Version 1.1.0 — 2026-04-09 リリース {#version-110--released-on-2026-04-09}

* **新機能:** 管理バーのステータスインジケーター付きフローティングチャットウィジェット — ログインしているユーザーであれば、フロントエンドからブラウザ側のLLMに直接プロンプトを送信できるようになりました。
* **新機能:** SharedWorkerランタイム — MLCエンジンがSharedWorker内で動作するようになったため、複数のタブがGPUセッションを奪い合うことなく、1つのセッションを共有できます。
* **新機能:** apiFetchミドルウェアインターセプター — AI Client SDKのパターンに一致するWordPress RESTリクエストが、ローカルのWebLLMブローカーに透過的にルーティングされるようになりました。これにより、ループバックのHTTPラウンドトリップが不要になりました。
* **新機能:** コネクタパネルにウィジェット設定UIを追加 — チャットウィジェットのオン/オフ切り替えや、自動プロンプトの動作設定が可能になりました。
* **修正:** IndexedDBキャッシュバックエンドを強制適用 — デフォルトのCache APIパスが壊れるHuggingFace xet CDNのリダイレクトがあっても、モデルの重みダウンロードが維持されるようになりました。
* **修正:** 埋め込みモデルのcontext_window KV-キャッシュの上書きをスキップ — 埋め込みモデルにはデコーダーがないため、この上書きが実行時にエラーを引き起こしていました。
* **修正:** ワーカータブがロードされる前に、/webllm/v1/modelsでコールドスタート候補モデルを公開 — これにより、SDKの利用者がモデルをすぐに確認できるようになりました。
* **修正:** WebLLMをwpai_preferred_text_modelsフィルターにフック — これにより、AI Experiments機能がモデルを設定している場合にブラウザエンジンにルーティングされるようになりました。
* **改善:** エンドツーエンドのテスト中に、キャッシュバスター、コンテンツ正規化、ハードウェア参照に関する修正が適用されました。
