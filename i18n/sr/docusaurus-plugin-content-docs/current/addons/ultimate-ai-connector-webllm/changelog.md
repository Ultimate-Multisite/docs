# Ultimate AI Connector за WebLLM дневник измена

## Верзија 1.1.0 — Објављено 2026-04-09 {#version-110--released-on-2026-04-09}

* Ново: Плутајући виџет за ћаскање са индикатором статуса у администраторској траци — сваки пријављени корисник може да зада упит LLM-у на страни прегледача директно са предњег дела.
* Ново: SharedWorker runtime — MLC мотор сада ради у SharedWorker-у, тако да више картица дели једну GPU сесију уместо да се надмећу око ње.
* Ново: apiFetch middleware interceptor — WordPress REST захтеви који се поклапају са AI Client SDK обрасцем транспарентно се усмеравају ка локалном WebLLM посреднику, без потребе за loopback HTTP повратним путовањем.
* Ново: Кориснички интерфејс подешавања виџета у Connector панелу за укључивање/искључивање виџета за ћаскање и конфигурисање понашања аутоматског упита.
* Исправка: Присилно коришћење IndexedDB cache backend-а како би преузимања тежина модела преживела HuggingFace xet CDN преусмеравања која кваре подразумевану Cache API путању.
* Исправка: Прескакање context_window KV-cache override-а за моделе угнежђивања (немају декодер и override је изазивао runtime грешку).
* Исправка: Оглашавање кандидатског модела за хладни старт у /webllm/v1/models пре него што се картица радника учита, тако да SDK потрошачи одмах виде модел.
* Исправка: Повезивање WebLLM-а са wpai_preferred_text_models филтером, тако да се функција AI Experiments усмерава ка мотору у прегледачу када је модел конфигурисан.
* Побољшано: Исправке за cache-busting, нормализацију садржаја и хардверске референце које су се појавиле током тестирања од почетка до краја.
