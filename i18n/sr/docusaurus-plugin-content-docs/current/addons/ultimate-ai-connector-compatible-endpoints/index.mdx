# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints интегрише API-је AI пружалаца компатибилне са OpenAI са вашом WordPress multisite мрежом. Конфигуришите више AI пружалаца — сваки са сопственим URL-ом крајње тачке, API кључем и подразумеваним моделом — и усмеравајте захтеве уз аутоматски fallback када пружалац није доступан.

## Кључне функције {#key-features}

- **Подршка за више пружалаца**: Додајте више картица AI пружалаца, сваку са сопственим URL-ом крајње тачке, API кључем и подразумеваним моделом
- **Превлачење за промену редоследа**: Распоредите пружаоце по приоритету помоћу дршке за хватање на свакој картици; захтеви користе пружаоце редом
- **Аутоматско fallback усмеравање**: Ако примарни пружалац није доступан, захтеви аутоматски прелазе на следећег конфигурисаног пружаоца
- **Конфигурација по пружаоцу**: Сваки пружалац се конфигурише независно, са сопственим акредитивима и избором модела
- **Кориснички интерфејс подешавања заснован на картицама**: Изграђен на стабилним WordPress 6.9+ Card компонентама (CardBody, CardHeader, CardDivider)
- **Аутоматско проширивање при додавању**: Нове картице пружалаца се аутоматски проширују како бисте одмах могли да унесете детаље
- **Разбијање кеша скрипти**: Скрипте пружалаца добијају нову кеш верзију при ажурирању plugin-а како би се обезбедило учитавање најновије верзије

## Захтеви {#requirements}

- WordPress 5.3 или новији
- PHP 7.4 или новији
- Ultimate Multisite plugin (активан)
- API крајња тачка компатибилна са OpenAI (самостално хостована или од треће стране)

## Инсталација {#installation}

1. Отпремите датотеке додатка у ваш `/wp-content/plugins/` директоријум
2. Активирајте plugin преко менија „Додаци” у WordPress-у
3. Идите на **Ultimate Multisite → AI Connector** да конфигуришете додатак

## Конфигурација {#configuration}

### Подешавање више пружалаца {#multi-provider-setup}

Екран за конфигурацију користи распоред заснован на картицама, где свака картица представља једног AI пружаоца. Можете додати онолико пружалаца колико вам је потребно.

#### Додавање пружаоца {#adding-a-provider}

1. Идите на **Ultimate Multisite → AI Connector** у мрежној администрацији
2. Кликните на **Додај пружаоца**
3. Појављује се нова картица пружаоца и аутоматски се проширује
4. Унесите детаље о пружаоцу:
   - **Назив пружаоца** — ознака за идентификацију овог пружаоца (нпр. "OpenAI", "Mistral", "Local Ollama")
   - **URL крајње тачке** — основни URL API-ја компатибилног са OpenAI (нпр. `https://api.openai.com/v1`)
   - **API кључ** — ваш API кључ за овог пружаоца
   - **Подразумевани модел** — идентификатор модела који се подразумевано користи за овог пружаоца (нпр. `gpt-4o`, `mistral-small`)
5. Кликните на **Сачувај подешавања**

#### Уклањање пружаоца {#removing-a-provider}

Кликните на дугме **Уклони** на било којој картици пружаоца да га обришете. Сачувајте подешавања да примените измену.

#### Промена редоследа пружалаца {#reordering-providers}

Превуците икону дршке (⣿) на левој страни било које картице пружаоца да промените њен редослед. Редослед одређује fallback приоритет: захтеви се шаљу првом пружаоцу и прелазе на наредне пружаоце ако захтев не успе.

### Fallback усмеравање {#fallback-routing}

Када захтев ка примарном пружаоцу не успе — због истека времена, ограничења стопе или прекида услуге — конектор аутоматски поново покушава са следећим пружаоцем на листи. Ово се наставља низ листу док неки пружалац не успе или док сви пружаоци не буду испробани.

| Сценарио | Понашање |
|----------|-----------|
| Примарни пружалац одговара | Захтев се нормално обрађује |
| Примарни пружалац не успева | Захтев се поново покушава са следећим пружаоцем |
| Сви пружаоци не успевају | Грешка се враћа позиваоцу |

Fallback је транспарентан за plugin или тему који позивају — они добијају или успешан одговор или коначну грешку, без обзира на то колико је пружалаца испробано.

### Референца конфигурације пружаоца {#provider-configuration-reference}

| Поље | Обавезно | Опис |
|-------|----------|-------------|
| Назив пружаоца | Да | Ознака за приказ за овог пружаоца |
| URL крајње тачке | Да | Основни URL API-ја компатибилног са OpenAI |
| API кључ | Да | Кључ за аутентификацију за овог пружаоца |
| Подразумевани модел | Да | Идентификатор модела који се користи када у захтеву није наведен модел |

## Решавање проблема {#troubleshooting}

### Ниједан пружалац није наведен {#no-providers-are-listed}

Ако је листа пружалаца празна након чувања, потврдите да:

- Сте кликнули на **Сачувај подешавања** након додавања пружалаца
- Ваш прегледач није блокирао чување странице подешавања (проверите конзолу прегледача за JavaScript грешке)
- Plugin је активан и налазите се на екрану мрежне администрације

### Захтеви не успевају за све пружаоце {#requests-are-failing-for-all-providers}

- Проверите да ли је URL крајње тачке сваког пружаоца доступан са вашег сервера (не само из вашег прегледача)
- Проверите да ли је сваки API кључ важећи и да није истекао
- Прегледајте статус ограничења стопе и квоте пружаоца у њиховом dashboard-у
- Омогућите WordPress евидентирање за отклањање грешака (`WP_DEBUG_LOG`) да бисте забележили детаљне поруке о грешкама из конектора

### Пружалац се не користи за fallback {#a-provider-is-not-being-used-for-fallback}

Превлачење за промену редоследа мења fallback редослед. Потврдите да је пружалац за кога очекујете да прима fallback захтеве правилно позициониран на листи и да су његов URL крајње тачке и API кључ исправни.

### Нова картица пружаоца се не проширује аутоматски {#new-provider-card-does-not-expand-automatically}

Аутоматско проширивање захтева да JavaScript буде омогућен у вашем прегледачу. Ако се картица не прошири на клик, проверите конзолу прегледача за грешке. Садржај картице је увек доступан кликом на заглавље картице, чак и ако се аутоматско проширивање не покрене.
