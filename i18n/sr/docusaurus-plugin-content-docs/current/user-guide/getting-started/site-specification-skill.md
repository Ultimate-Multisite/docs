# Вештина спецификације сајта

**Вештина спецификације сајта** је структуриран приступ бележењу циљева, публике и идентитета бренда вашег сајта. Ове информације се чувају у вашој **site_brief** меморији, на коју се агенти позивају током сесија како би пружали доследну помоћ свесну контекста.

## Шта је спецификација сајта?

Спецификација сајта је процес документовања:

- **Сврха сајта**: Шта ваш сајт ради и зашто постоји
- **Циљна публика**: Ко посећује ваш сајт и шта им је потребно
- **Идентитет бренда**: Ваше боје, тон и визуелни стил
- **Пословни циљеви**: Како изгледа успех за ваш сајт
- **Структура садржаја**: Како је ваш сајт организован

Ова спецификација постаје ваш **site_brief**, трајна меморија коју агенти користе да разумеју контекст вашег сајта.

## Зашто користити спецификацију сајта?

### Доследност током сесија

Без site_brief, морали бисте поново да објашњавате сврху свог сајта сваки пут када започнете нову сесију. Са њим, агенти одмах разумеју:

- Циљеве и публику вашег сајта
- Боје и тон вашег бренда
- Структуру вашег садржаја
- Ваше пословне циљеве

### Боље препоруке

Агенти користе ваш site_brief да:

- Предложе функције усклађене са сврхом вашег сајта
- Препоруче структуре садржаја које одговарају вашим циљевима
- Предложе дизајне доследне вашем бренду
- Избегну предлагање некомпатибилних функција

### Брже увођење

Нови агенти (или агенти у новим сесијама) могу брзо да се упознају са ситуацијом читањем вашег site_brief уместо постављања питања за појашњење.

## Покретање спецификације сајта

### Током Theme Builder увођења

Вештина спецификације сајта се аутоматски покреће током **Theme Builder тока увођења**. Setup Assistant agent поставља питања и гради ваш site_brief.

### Ручно покретање

Можете започети спецификацију сајта у било ком тренутку:

```
„Хајде да дефинишемо спецификацију мог сајта“
```

или

```
„Помози ми да направим кратак опис сајта“
```

## Процес спецификације сајта

### Корак 1: Сврха сајта

Агент пита:

```
Која је примарна сврха вашег сајта?
- Продавница електронске трговине
- Блог или сајт са садржајем
- Портфолио или изложбени приказ
- SaaS апликација
- Заједница или форум
- Друго: [опишите]
```

Можете изабрати категорију или описати сопствену сврху.

### Корак 2: Циљна публика

```
Ко је ваша примарна публика?
- Потрошачи / широка јавност
- Пословни професионалци
- Програмери / технички корисници
- Студенти / наставници
- Друго: [опишите]

Које су њихове главне потребе?
```

### Корак 3: Идентитет бренда

```
Које су боје вашег бренда?
- Примарна боја: [бирач боје или хексадецимални код]
- Секундарна боја: [бирач боје или хексадецимални код]
- Акцентна боја: [опционо]

Како бисте описали тон свог бренда?
- Професионалан / корпоративан
- Креативан / уметнички
- Разигран / лежеран
- Минималан / модеран
- Топао / пријатељски
```

### Корак 4: Пословни циљеви

```
Како изгледа успех за ваш сајт?
- Генерисање потенцијалних клијената
- Продаја производа
- Изградња заједнице
- Дељење знања
- Успостављање ауторитета
- Друго: [опишите]

Која је ваша примарна метрика?
- Приход
- Ангажовање корисника
- Досег садржаја
- Конверзије
- Друго
```

### Корак 5: Структура садржаја

```
Како је ваш садржај организован?
- Равно (сав садржај је на истом нивоу)
- Хијерархијски (категорије и поткатегорије)
- Хронолошки (у стилу блога)
- Засновано на производима (каталог)
- Друго: [опишите]

Које типове садржаја користите?
- Објаве на блогу
- Странице производа
- Студије случаја
- Документација
- Видео-снимци
- Друго
```

## Ваша site_brief меморија

Након довршавања спецификације сајта, ваше информације се чувају као **site_brief** у меморији вашег агента. Ово је структуриран запис који садржи:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## Преглед и ажурирање вашег site_brief

### Прикажите свој site_brief

Питајте агента:

```
„Прикажи ми мој кратак опис сајта“
```

или

```
„Шта знаш о мом сајту?“
```

Агент ће приказати вашу сачувану спецификацију.

### Ажурирајте свој site_brief

Ако се ваш сајт развија, можете га ажурирати:

```
„Ажурирај мој кратак опис сајта: сада циљамо B2B купце“
```

или

```
„Освежи спецификацију мог сајта“
```

Ово ће поново покренути вештину спецификације сајта користећи ваше постојеће информације као полазну тачку.

## Како агенти користе site_brief

### Препоруке за дизајн

Када затражите измене дизајна, агенти се позивају на ваш site_brief:

```
Ви: „Редизајнирај моју почетну страницу“
Агент: „На основу вашег кратког описа сајта, циљате жене од 25 до 45 година
са топлим, занатским тоном. Направићу дизајн који наглашава
ваш ручно рађени накит и одрживе вредности.“
```

### Предлози функција

Агенти предлажу функције усклађене са вашим циљевима:

```
Ви: „Које функције треба да додам?“
Агент: „За продавницу електронске трговине усмерену на лојалност купаца,
препоручио бих: програм лојалности, рецензије купаца, имејл билтен
и препоруке производа.“
```

### Структура садржаја

Агенти предлажу организацију садржаја на основу ваше структуре:

```
Ви: „Како да организујем свој блог?“
Агент: „Пошто користите каталог производа са блогом, предложио бих
организовање објава по категоријама производа и креирање „водича за стилизовање“
који заједно приказују више производа.“
```

## Најбоље праксе

### Будите конкретни

Уместо „општа публика“, опишите своју стварну публику:

- ✓ „Жене од 25 до 45 година, заинтересоване за одрживу моду“
- ✗ „Сви“

### Редовно ажурирајте

Како се ваш сајт развија, ажурирајте свој site_brief:

- Када се преусмерите на нову публику
- Када додате нове линије производа
- Када се идентитет вашег бренда промени
- Када се ваши пословни циљеви промене

### Користите доследну терминологију

Користите исте термине током сесија:

- ✓ Увек реците "sustainable jewelry" (не "eco-friendly jewelry" и "green products")
- ✓ Доследно се обраћајте својој публици на исти начин

### Укључите контекст

Пружите позадину која помаже агентима да разумеју ваше одлуке:

- "Циљамо професионалце који више цене квалитет него цену"
- "Наша публика је технолошки поткована и очекује модеран дизајн"
- "Ми смо bootstrapped стартап, па су нам потребна исплатива решења"

## Однос са Theme Builder увођењем

Вештина Спецификација сајта интегрисана је у **Theme Builder ток увођења**. Када завршите увођење, ваш site_brief се аутоматски креира са информацијама које сте навели.

Спецификацију сајта можете покренути и самостално ако желите да:

- Прецизирате своју спецификацију након почетног подешавања
- Ажурирате свој опис сајта како се ваш сајт развија
- Креирате детаљну спецификацију пре почетка рада са Theme Builder

## Решавање проблема

**Мој site_brief се не користи**
- Потврдите да агент има приступ меморији
- Замолите агента да "се присети мог описа сајта"
- Проверите да ли је меморија омогућена у вашим подешавањима

**Желим да почнем испочетка са новим site_brief**
- Замолите агента: "Обриши мој опис сајта и почни испочетка"
- Затим поново покрените Спецификацију сајта

**Агент даје препоруке које се не поклапају са мојим site_brief**
- Замолите агента да "прегледа мој опис сајта"
- Ажурирајте свој site_brief ако је застарео
- Наведите додатни контекст у својим захтевима

## Следећи кораци

Након дефинисања спецификације сајта:

1. **Користите Theme Builder**: Креирајте прилагођену тему на основу свог site_brief
2. **Прецизирајте дизајн**: Користите вештину Естетика дизајн система за детаљан рад на дизајну
3. **Планирајте садржај**: Затражите од агената препоруке за структуру садржаја
4. **Изградите функционалности**: Затражите функционалности усклађене са вашим пословним циљевима
