# Gratis AI Agent подешавања

Екран **Settings → Advanced** у Gratis AI Agent пружа конфигурацију администраторског нивоа за backend интеграције. Ова страница документује прослеђивање повратних информација, кључеве провајдера претраге, подешавање управљане Superdav услуге, контроле за Google Calendar, TextBee SMS подешавања и ознаке функционалности на нивоу мреже.

## Приступ подешавањима {#accessing-settings}

1. У WordPress администрацији идите на **Gratis AI Agent → Settings**.
2. Кликните на картицу **Advanced**.

## Конфигурација endpoint-а за повратне информације {#feedback-endpoint-configuration}

Endpoint за повратне информације прима POST захтеве од AI агента кад год корисник пошаље повратну информацију преко дугмета са палцем надоле, банера са аутоматским упитом или команде `/report-issue`.

| Поље | Опис |
|---|---|
| **Feedback Endpoint URL** | URL који прима поднеске повратних информација као HTTP POST захтеве са JSON телом. |
| **Feedback API Key** | bearer token послат у `Authorization` header-у сваког захтева за повратне информације. Оставите празно ако ваш endpoint не захтева аутентификацију. |

### Очекивани JSON payload {#expected-json-payload}

Ваш endpoint за повратне информације мора да прихвати JSON тело са најмање следећим пољима:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

Додатна поља могу бити присутна у payload-у у зависности од контекста разговора.

### Вредности `triage_category` {#triagecategory-values}

AI слој за тријажу додељује једну од следећих вредности за `triage_category` пре прослеђивања payload-а:

| Вредност | Значење |
|---|---|
| `factual_error` | Асистент је пружио нетачне чињеничне информације. |
| `unhelpful_answer` | Одговор је био технички тачан, али није био користан. |
| `inappropriate_content` | Одговор је садржао садржај који не треба приказивати корисницима. |
| `other` | Повратна информација се није подударала са познатом категоријом. |

### Аутентификација {#authentication}

Ако ваш endpoint захтева аутентификацију, поставите поље **Feedback API Key** на ваш bearer token. Агент шаље:

```
Authorization: Bearer <your-api-key>
```

Ако је поље **Feedback API Key** празно, `Authorization` header се не шаље.

### Онемогућавање прикупљања повратних информација {#disabling-feedback-collection}

Оставите оба поља **Feedback Endpoint URL** и **Feedback API Key** празна. Дугме са палцем надоле и UI за повратне информације остају видљиви корисницима, али се поднесци не прослеђују ниједној спољној услузи.

## Brave Search API Key {#brave-search-api-key}

Такође на картици **Advanced**, поље **Brave Search API Key** омогућава могућност [Интернет претрага](../configuration/internet-search).

| Поље | Опис |
|---|---|
| **Brave Search API Key** | Ваш API кључ из Brave Search developer dashboard-а. Обавезан је за омогућавање интернет претраге у AI асистенту. |

Ознака поља укључује везу на коју се може кликнути до странице за регистрацију за Brave Search API. Оставите празно да бисте онемогућили интернет претрагу.

Погледајте [Интернет претрага](../configuration/internet-search) за документацију за крајње кориснике о овој функцији.

## Управљана Superdav услуга {#managed-superdav-service}

Superdav AI Agent v1.18.0 додаје управљане Superdav service endpoint-е и аутоматско обезбеђивање везе за подржане сајтове. Користите ове контроле када ваш сајт треба да се повеже са хостованим провајдером уместо са ручно конфигурисаним service endpoint-ом.

| Поље | Опис |
|---|---|
| **Managed Superdav Service** | Омогућава везу са хостованом Superdav услугом за подржане сајтове. |
| **Provision Connection** | Покреће аутоматско обезбеђивање endpoint-а и акредитива. Користите ово након потврде да сајт треба да користи управљаног провајдера. |
| **Service Endpoint / Connection Status** | Приказује тренутни endpoint или стање везе након обезбеђивања. |

Након обезбеђивања, сачувајте подешавања и проверите статус везе пре него што се ослоните на токове рада управљане услуге. Ако обезбеђивање не успе, прегледајте приказана упутства за поновни покушај и потврдите да сајт има дозволу да користи хостованог провајдера.

## Google Calendar конфигурација {#google-calendar-configuration}

Када су омогућене календарске функције Superdav AI Agent v1.18.0, агент може да чита конфигурисане календаре и детаље догађаја. Календарски алати су оријентисани на читање и корисни су за подсетнике свесне распореда, праћење учесника и подударање контаката.

| Поље | Опис |
|---|---|
| **Google Calendar Credentials** | Чува акредитиве или token везу потребну за читање календарских података. |
| **Calendar Selection** | Ограничава које конфигурисане календаре агент сме да прегледа. |
| **Calendar Connection Status** | Потврђује да ли тренутни акредитиви могу да читају календаре и догађаје. |

Ограничите календарске акредитиве на календаре који су агенту потребни. Поново повежите или ротирајте акредитиве ако статус указује на истекао token.

## TextBee SMS обавештења {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 додаје TextBee као SMS провајдера за конфигурисане токове рада обавештења. SMS обавештења треба упарити са капијама људског одобрења за осетљиве поруке или поруке намењене корисницима.

| Поље | Опис |
|---|---|
| **TextBee API Key** | Аутентификује захтеве ка TextBee SMS провајдеру. |
| **TextBee Device / Sender** | Бира TextBee пошиљаоца или уређај који се користи за одлазне поруке, када то провајдер захтева. |
| **SMS Notifications Enabled** | Дозвољава одобреним токовима рада да шаљу обавештења текстуалним порукама. Оставите онемогућено да бисте спречили SMS слања. |

Пошаљите тест поруку само на број у власништву администратора, затим потврдите понашање капије одобрења пре него што омогућите заказане подсетнике или подсетнике намењене учесницима.

## Ознаке функционалности {#feature-flags}

Такође уведена у v1.9.0, картица **Settings → Feature Flags** пружа прекидаче за опционалну функционалност. Свака ознака је или омогућена или онемогућена на нивоу целе мреже; тренутно не постоји замена по сајту.

### Приступ ознакама функционалности {#accessing-feature-flags}

1. У WordPress администрацији идите на **Gratis AI Agent → Settings**.
2. Кликните на картицу **Feature Flags**.

### Ознаке контроле приступа {#access-control-flags}

| Ознака | Подразумевано | Опис |
|---|---|---|
| **Ограничи на администраторе** | Искључено | Када је омогућено, само корисници са улогом `administrator` могу да отворе AI Agent панел за ћаскање. Све остале улоге уместо тога виде поруку „Контактирајте свог администратора“. |
| **Ограничи на мрежне администраторе** | Искључено | Када је омогућено на multisite мрежи, само Super Admins могу да користе агента. Администратори појединачних сајтова су блокирани. Има предност над „Ограничи на администраторе“ ако су обе опције омогућене. |
| **Дозволи приступ претплатницима** | Искључено | Када је омогућено, корисници са улогом `subscriber` могу да користе интерфејс за ћаскање, али су ограничени на могућности само за читање (без креирања објава или промена подешавања). |
| **Онемогући за нечланове** | Искључено | Интегрише се са статусом чланства Ultimate Multisite. Када је омогућено, ћаскање је сакривено за сајтове који немају активно чланство. |

### Ознаке брендирања {#branding-flags}

| Ознака | Подразумевано | Опис |
|---|---|---|
| **Сакриј подножје „Powered by Gratis AI Agent“** | Искључено | Уклања ред са брендираним приписивањем приказан на дну виџета за ћаскање. Препоручује се за white-label имплементације. |
| **Прилагођено име агента** | *(празно)* | Замењује подразумевану ознаку „Gratis AI Agent“ у заглављу ћаскања и администраторском менију вашим сопственим именом производа. Оставите празно да бисте користили подразумевано. |
| **Сакриј избор агента** | Искључено | Када је омогућено, корисници не могу да прелазе између пет уграђених агената. Тренутни агент је фиксиран на оно што је подешено као подразумевано у Settings → General. |
| **Користи икону сајта као аватар ћаскања** | Искључено | Замењује подразумевану AI икону у заглављу виџета за ћаскање WordPress иконом сајта (подешеном у Appearance → Customize → Site Identity). |

### Ознаке безбедности аутоматизације {#automation-safety-flags}

Superdav AI Agent v1.18.0 уводи контролне тачке људског одобрења и записе подсетника за безбеднија покретања аутоматизације. Ове контроле се могу појавити у ознакама функција или напредним подешавањима аутоматизације, у зависности од инсталираног пакета.

| Ознака | Подразумевано | Опис |
|---|---|---|
| **Захтевај људско одобрење** | Препоручено укључено | Паузира осетљиве аутоматизације док овлашћени корисник не прегледа и потврди предложену радњу. |
| **Дедупликација подсетника** | Укључено | Бележи послате подсетнике тако да поновни покушаји или заказана покретања не шаљу дуплирана обавештења. |
| **Омогући алате календара** | Искључено док се не подеси | Омогућава агенту да чита подешене Google календаре и догађаје. |
| **Омогући SMS обавештења** | Искључено док се не подеси | Омогућава одобреним радним токовима да шаљу TextBee SMS обавештења након што се акредитиви сачувају. |

### Примена промена {#applying-changes}

Кликните на **Сачувај подешавања** након укључивања или искључивања било које ознаке. Промене ступају на снагу одмах — није потребно пражњење кеша нити поновна активација plugin-а.
