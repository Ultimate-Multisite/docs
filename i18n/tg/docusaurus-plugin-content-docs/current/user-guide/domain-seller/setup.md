# Донишҷот: Насоз ва Таъсилоти Провайдерҳо {#domain-seller-setup-and-provider-configuration}

Донишҷот (Domain Seller) addon бо ройани он ба шумо як дастурро барои гузариш аз ҳама коркардаи зарурӣ пешниҳод мекунад. Ин саҳифа дар бораи ҷараёни ин дастур ва тарзи тартиб додани ё тасдиқ кардани (reconfigure) провайдерҳо пас аз он меомӯзад.

## Талавоти система {#requirements}

- **Multisite Ultimate** версияи 2.4.12 ё баландтар, ки дар шабака фаъол шудааст
- **PHP** 7.4+
- Маълумоти API барои ҳад аз як регистратор дастгирӯшшуда

## Дастури таҷҳизони аввал (First-run setup wizard) {#first-run-setup-wizard}

Дастури таҷҳизони аввал ҳарзамон ба кор оғоз мешавад, вақте ки шумо дигар бор ин плагинро дар шабака фаъол мекунед. Он ҳамчун **Network Admin › Ultimate Multisite › Domain Seller Setup** ҳамеша мавҷуд аст.

### Қадами 1 — Интелектуал (Choose a provider) {#step-1--choose-a-provider}

Регистраториро интихоб кунед, ки мехоҳед бо он пайваст шавед. Вариятҳои дастгирӯшшуда:

| Провайдер | Идораи DNS | Ҳифзи маълумоти WHOIS |
|---|---|---|
| OpenSRS | Ҳа | Ҳа |
| Namecheap | На | Ҳа (WhoisGuard, озод) |
| HostAfrica | Ҳа | Ҳа (Ҳифзи ID) |
| Openprovider | Ҳа | Ҳа |
| Hostinger | Тавассути маппинг домени асосии Hostinger барои доменҳои хостидашуда | Ҳа |
| GoDaddy | На | На |
| ResellerClub | Ҳа | На |
| NameSilo | На | На |
| Enom | Ҳа | На |

### Қадами 2 — Маълумоти ворид кардан (Enter credentials) {#step-2--enter-credentials}

Ҳар як провайдери оғоз кардани маълумотҳои гуноки дорад:

**OpenSRS** — Номи корбар ва калиди махфи (аз панели Reseller Control Panel-и OpenSRS)

**Namecheap** — Номи корбар ва API key (аз Account › Tools › API Access)

**HostAfrica** — Endpoint-и API-и Reseller Domain ва маълумоти тасдиқ аз модули reseller-и HostAfrica. Ҳолатан, endpoint-и sandbox муайяне ҳуҷҷат додашуда нест; пеш аз оғоз кардани қайдҳо, бо текшишҳои амн (read-only) тест кунед.

**Openprovider** — Номи корбар ва пароль бо иҷозати API фаъол. Режам андозаи sandbox (sandbox mode) аз API-и Openprovider истифода мебарад, ва ҳатто тасдиқи воридшавандаи муқаррарӣ барои қайдҳо метавонад барои интихоб иловагӣ бошад.

**Hostinger** — Токен hPanel хостинг общим от основной интеграции Hostinger. Этот же токен используется для сопоставления основных доменов и операций регистрации продавцов доменов.

**GoDaddy** — API ключ и секретный ключ (с developer.godaddy.com)

**ResellerClub** — ID реселлера и API ключ (из панели управления ResellerClub)

**NameSilo** — API ключ (с namesilo.com › Account › API Manager)

**Enom** — Идентификатор учетной записи и токен API

Проверьте **режим Sandbox**, если он доступен, чтобы протестировать соединение в тестовой среде провайдера перед запуском в реальный режим.

### Шаг 3 — Проверить соединение {#step-3--test-the-connection}

Нажмите **Test Connection** (Проверить соединение). Мастер отправит легкий API-запрос для проверки учетных данных и связи. Исправьте любые проблемы с учетными данными, прежде чем продолжить.

### Шаг 4 — Импорт TLD {#step-4--import-tlds}

Нажмите **Import TLDs** (Импортировать TLD), чтобы загрузить все доступные TLD и оптовые цены от подключенного провайдера. Это заполнит список TLD, используемый для доменных продуктов. Импорт может занять от 30 до 60 секунд для провайдеров с большими каталогами TLD.

TLD также автоматически синхронизируются один раз в день через запланированную задачу cron.

### Шаг 5 — Создать продукт домена {#step-5--create-a-domain-product}

Мастер создает продукт по умолчанию, который является "catch-all" (общий) для доменов с наценкой 10%. Вы можете отредактировать этот продукт немедленно или пропустить его и создать продукты вручную в разделе **Ultimate Multisite › Products** (Ultimate Multisite › Продукты).

См. [Domain Products and Pricing](./domain-products) для полного руководства по настройке продуктов.

## Перенастройка провайдера {#reconfiguring-a-provider}

Перейдите в **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или нажмите **Settings** в списке плагинов).

На странице настроек содержится:

- **Ҳифзи фурӯши домен** — интихоб кардани корӣ (on/off) барои ҳамаи усулҳо

- **Провайдери восита** — провайдери аз он истифода мешавад барои ҷустуҷӯи доменҳо ва маҳсулоти нав

- **Ҳамаҷонибаҳои максималии TLD** — шумо чӣ қадар TLD (доминий нишонаҳо) -ро ҳангори ҷустуҷӯи муштариён санҷед; арзишҳои баландтар имкониятҳои бештарро нишон медиҳанд, аммо суръат аз он мепайвашад

- **Давра корӣ барои кэши мавҷудияти домен** — шумо чӣ қадар натиҷаҳои мавҷудияти ва нархи ройхат карда мехсонед; арзишҳои пасттар аниқтар хоҳанд буд, аммо ба таври назаррасои API (API calls) афзоиш мекунад

- **Идоракунии маҳсулоти домен** — пайваши суръатбаранг ба рӯйхати Products

- **Ташкили провайдерҳо** — кушодани Integration Wizard барои илова кардан ё танзим кардани мувофиқӣ (reconfigure) провайдерҳо

### Илова кардани провайдери дуюм {#adding-a-second-provider}

Ба **Configure providers** клинанд ва ин wizard-ро боз барои ройхатдори нав иҷро кунед. Шумо метавонед бисёр провайдерро ҳамзамон танзим намоед. Ҳар як маҳсулоти доменро ба провайдери мушаххас тақсим кунед, ё онро дар ҳолати восита (default) гузоред.

### Синхронизатсияи TLD-ҳо бо интихобӣ {#syncing-tlds-manually}

Дар саҳифаи تنظیمات, паси клинанд **Sync TLDs** дар паҳлӯи ҳар як провайдери танзимшударо барои гирифтани арзиши навтарин. Ин пас аз он ки як провайдер нархи боқӣ (wholesale pricing) -ро тағйир дода ё TLDҳои нави илова кардааст, муфид аст.

## Нотаотҳои қобилияти пешгир ванда (Provider capability notes) {#provider-capability-notes}

Ҳамаи API-ҳои регистратор ҳамон амалиётро нишон намедиҳанд. Модулҳо амалиёти дастгирӣ намудашударо бо хатоҳои равшан ба администратор нишон медиҳанд, ба ҷо ин ки онҳо бехабар ба таври пасттар ба саркарда мераванд.

- **HostAfrica** ҳамаи корҳои мувофиқи реселлерро дастгирӣ мекунад, аз ҷумла талош (lookup), синхронизатсияи TLD/нарх, қайд кардан, таҷдид кардан, гузариш, тағйири сохтори номбар (nameserver updates), намудаҳои DNS, кодеҳои EPP, қатъи регистратор ва ҳифзи ID.
- **Openprovider** синхронизатсияи TLD бо нархи реселлерро, қайд кардан, таҷдид кардан, гузаришҳо, тағйири сохтори номбар (nameserver updates), зонаҳои DNS, кодеҳои EPP, қатъи регистратор ва махфият WHOIS-и онро дастгирӣ мекунад. Он бо токенҳои bearer муқарраршуда (short-lived) тасдиқ мешавад, ки модул худаш автоматик танзим мекунад.
- **Hostinger** аз ҳисоби токени API-и hPanel, ҷустуви мавҷудияти вебсайт, қайд кардан, талоши портфолио (portfolio lookup), тағйири сохтори номбар, қатъи регистратор ва махфият WHOIS-ро дастгирӣ мекунад. API-и умумии Hostinger барои нархи реселлер/богонӣ, гузаришҳои воридӣ (inbound transfer), таҷдиди явӣ (explicit renewal) ё гирифтани кодеҳои EPP нишон намедиҳад; танҳо таҷдид автоматик аст.
