# Маҳорати Спецификацияи Сайт

Маҳорати **Site Specification** (Спецификацияи Сайт) як усули надорамонӣ барои ба қайд гирифтани ҳадафи, аудитория ва мазмуни бренд-и сайт-и шумост. Ин иттилоот дар хотира **site_brief**-и шумо сақла мешавад, ки агентҳо ҳангори сессияҳои гуногун аз он истифода мебаранд, то кӯмаки мунтазам ва боэътимоди контекстуалӣ пешниҳоди андозаванд.

## Site Specification чист?

Site Specification ин равиш аст, ки:

- **Мақсади сайт**-ро ҳуҷҷатнасонед: сайт-и шумо чӣ кор мекунад ва чаро вуҷуд дорад
- **Аудиторияи мақсадӣ**: ки кӯшиш мекунанд сайт-и шумо ва онҳо чӣ лозим аст
- **Ҳадоригии бренд (Brand identity)**: ранги, паҳнӣ ва услуби назаррасои шуморо тасвир кунед
- **Ҳадафҳои тиҷоратӣ**: муваффақият барои сайт-и шумо чӣ гуна аст
- **Мақоми мундариҷа (Content structure)**: сайт-и шумо чӣ гуна ташкил ёфтааст

Ин спецификация ба **site_brief**-и шумо мегаршад, хотирае ки доимӣ аст ва агентҳо аз он барои фаҳмидани контексти сайт-и шумо истифода мекунанд.

## Чаро Site Specification-ро истифода баранд?

### Мунтазамӣ дар сессияҳои гуногун

Агар **site_brief** нагашт, ҳарзат аст, ки ҳарвақте сессион нав оғоз мекунед, мақсади сайт-и худ-ро дигар бор кунед. Бо он, агентҳо фавран мефаҳманд:

- Ҳадафҳои ва аудиторияи сайт-и шумо
- Ранги ва паҳнии бренди шумо
- Мақоми мундариҷаи шумо
- Ҳадафҳои тиҷоратӣ

### Тавсияҳои беҳтар

Агентҳо аз **site_brief**-и шумо барои:

- Хусусиятҳое, ки ба мақсади сайт-и шумо мувофиқанд, тавсия диҳанд
- Мақоми мундариҷаро пешниҳод кунанд, ки ба ҳадафҳои шумо мувофиқат мекунад
- Нақшаҳоро пешниҳод намоянд, ки ба бренди шумо мувофиқат мекунанд
- Тавсияи хусусиятҳое, ки ҳамгирӣ намекунанд, аз он сарчаш кунед

### Оғози кор суръатбахштар

Агентҳои нав (ё агентҳо дар сессияҳои нав) бо хондан аз **site_brief**-и шумо ба зуд фаҳмида мешаванд ва набояд пурсиҳои равшанро бихонанд.

## Оғоз кардани Site Specification

### Дар интиқоли Theme Builder

Маҳорати Site Specification ҳангори равиши оғози кор бо **Theme Builder** автоматик оғоз мешавад. Агенти Setup Assistant саволҳоро пурсед ва site_brief-и шуморо сохта тавонад.

### Оғози мустақил

Шумои муайянкунии сайтро ҳаръаш оғоз кардан метавонед:

```
"Биёни муайянкунии сайтиро таҳия кунем"
```

ё

```
"Ба ман дар таҳияи мазмуни сайт кӯмак кунед"
```

## Раванди Муайянкунии Сайт

### Қадами 1: Мақсади Сайти

Агент пурсед:

```
Мақсади асосии сайти шумо чист?
- Донишгоҳи электронӣ (E-commerce store)
- блог ё сайти мазмунӣ
- Портфолио ё нишон додани корҳо
- Иловаи SaaS
- Муштарият ё форум
- Дигар: [тавсиф кунед]
```

Шумо метавонед як маҷмааро интихоб кунед ё мақсади худ тавсиф намоед.

### Қадами 2: Маълумияти Ҳамкориён (Target Audience)

```
Маълумияти асосии шумо кист?
- Мажореён / ҷамъият дарUmum
- Мутахассисони тиҷоратӣ
- Муаллимон / истифодабандҳои техникӣ
- Олиёҳо / таълимгоҳҳо
- Дигар: [тавсиф кунед]

Ҳажда мехоҳанд чист?
```

### Қадами 3: Ҳудентии Бренд (Brand Identity)

```
Раангҳои бренди шумо чист?
- Ранги асосӣ: [интерфейси интихоб ё коди хекс]
- Ранги донишӣ: [интерфейси интихоб ё коди хекс]
- Ранги таъкид (Accent color): [икораш]

Шумо рангҳои бренди худро чӣ тавсиф мекунед?
- Мутахассисоӣ / корпоратив
- Ясоварӣ / санъатӣ
- Бодиқӣ / расмӣ
- Минимал / муосир
- Гарм / дӯстона
```

### Қадами 4: Ҳадафҳои Бизнес (Business Goals)

```
Муваффақият барои сайти шумо чӣ гуна аст?
- Маълумот ҷамъ овардан (leads generate)
- Маҳсулот фурӯш кардан
- Муштарият сохтани муштарият (community build)
- Донишро паҳн кардан
- Мавқеи касбии худро таъсис додан
- Дигар: [тавсиф кунед]

Нардонии асосии шумо чист?
- Даромад (Revenue)
- Ҳамwerking бо истифодабандагон (User engagement)
- Паҳншавии муалво (Content reach)
- Тағйирот ба сарфаи корӣ (Conversions)
- Дигар
```

### Қадами 5: Назари Мазмун (Content Structure)

```
Мазмуни шумо чӣ гуна ташкил ёфтааст?
- Плат (ҳамаи мазмун дар як сатҳе ҳастанд)
- Ҳиерархикӣ (маҷмӯаҳо ва категорияҳои иловагӣ)
- Хронологикӣ (ба таври блогӣ)
- Асосии маҳсулот (каталог)
- Дигар: [тавсиф кунед]

Шумо аз кадом намудҳои мазмун истифода мекунед?
- Постҳои блог
- Сахтҳои маҳсулот
- Касҳои таҷриба (Case studies)
- Документатсия
- Видеоҳо
- Дигар
```

## Хотираи site_brief-и шумо

Баъди пур кардани Муайянкунии Сайт, маълумоти шумо ҳамчун **site_brief** дар хотираи агент сақла мешавад. Ин як рӯйхати сохторӣ аст, ки ин корҳоро дар бар мегирад:

## Навігляд та оновлення вашого site_brief

### Переглянути ваш site_brief

Запитайте агента:

```
"Покажи мені мій site brief"
```

або

```
"Що ти знаєш про мій сайт?"
```

Агент покаже вашу збережену специфікацію.

### Оновити ваш site_brief

Якщо ваш сайт змінюється, ви можете його оновити:

```
"Оновити мій site brief: тепер ми націлюємося на B2B клієнтів"
```

або

```
"Оновити мову специфікації сайту"
```

Це перезапустить навичку Site Specification з вашою існуючою інформацією як початковою точкою.

## Як агенти використовують site_brief

### Рекомендації щодо дизайну

Коли ви просите зміни в дизайні, агенти посилаються на ваш site_brief:

```
Ви: "Переробити мій головний екран"
Агент: "На основі вашого site brief, ви орієнтуєтеся на жінок у віці 25-45 років із теплим, майстерним тоном. Я створю дизайн, який підкреслить ваше виготовлене вручну біжу та принципи сталості."
```

### Пропозиції щодо функціоналу

Агенти пропонують функції, які відповідають вашим цілям:

```
Ви: "Які функції мені варто додати?"
Агент: "Для інтернет-магазину, орієнтованого на лояльність клієнтів, я б рекомендував: програму лояльності, відгуки клієнтів, електронну розсилку та рекомендації товарів."
```

### Структура контенту

Агенти пропонують організацію контенту на основі вашої структури:

Шумо: «Как мне свой блог организовать?»
Агент: «Поскольку вы используете каталог продуктов с блогом, я бы посоветовал организовывать посты по категориям продуктов и создавать 'стилистические руководства', которые демонстрируют несколько продуктов вместе».

## Лучшие практики

### Будьте конкретны

Вместо "широкая аудитория" опишите свою реальную аудиторию:

- ✓ «Женщины в возрасте 25-45 лет, интересующиеся устойчивой модой»
- ✗ «Все»

### Регулярно обновляйте информацию

По мере развития вашего сайта обновляйте `site_brief`:

- Когда вы меняете целевую аудиторию
- Когда вы добавляете новые линейки продуктов
- Когда меняется ваша идентичность бренда
- Когда меняются ваши бизнес-цели

### Используйте единую терминологию

Используйте одни и те же термины в течение всех сессий:

- ✓ Всегда говорите "устойчивые украшения" (а не "экологичные украшения" и "зеленые продукты")
- ✓ Постоянно называйте свою аудиторию одним и тем же способом

### Предоставляйте контекст

Давайте информацию, которая поможет агенту понять ваши решения:

- «Мы нацелены на профессионалов, которые ценят качество выше цены»
- «Наша аудитория разбирается в технологиях и ожидает современного дизайна»
- «Мы — стартап с минимальным финансированием, поэтому нам нужны экономичные решения»

## Связь с настройкой Theme Builder

Навык Site Specification интегрирован в **процесс настройки Theme Builder**. Когда вы завершаете настройку, ваш `site_brief` автоматически создается с той информацией, которую вы предоставили.

Вы также можете запускать Site Specification независимо, если хотите:

- Уточнить свою спецификацию после первоначальной настройки
- Обновить свой site brief по мере развития вашего сайта
- Создать подробную спецификацию перед началом работы с Theme Builder

## Устранение неполадок

**Мой site_brief не используется**
- Убедитесь, что у агента есть доступ к памяти
- Попросите агента «вспомнить мой site brief» (recall my site brief)
- Проверьте, включена ли память в ваших настройках

**Ман себя с чистого листа начать с новым site\_brief**
- Спросите агента: "Очистить мой site brief и начать заново"
- Затем снова запустите Site Specification

**Агент делает рекомендации, которые не соответствуют моему site\_brief**
- Попросите агента: "Проверить мой site brief"
- Обновите свой site\_brief, если он устарел
- Предоставьте дополнительный контекст в своих запросах

## Следующие шаги

После того как вы определили спецификацию вашего сайта:

1. **Используйте Theme Builder**: Создайте кастомную тему на основе вашего site\_brief
2. **Уточните дизайн**: Используйте навык Design System Aesthetics для детальной работы по дизайну
3. **Планируйте контент**: Попросите агентов о рекомендациях по структуре контента
4. **Создавайте функции**: Запросите функции, соответствующие вашим бизнес-целям
