# Асосути асосӣ

Барои корбарони нав дар WordPress Multisite ва ин ба таври аввал оғоз мекунанд, метавонад бисёр калима ва ибораҳои нав барои омӯхтани онҳо дошта бошед. Омӯзидани онҳо аҳамияти муҳим аст, зеро шумо бояд платформа ва тарзи коркарди онро ба таври умумӣ фаҳмеян.

Дар ин мақола, мо кӯшиш мекунем ки баъзе аз мафҳумҳои асосии WordPress-ро таъриф ва шарҳ диҳем. Баъзе аз онҳо барои корбарон, баъзе барои рушдамон ва баъзе ҳамчун ҳар дуярдӯст мувофиқтар мебошанд.

## WordPress Multisite

**Multisite** (Ташкилоти бисёр сайт) дар WordPress намуди инсталлят аст, ки иҷозат медиҳад шумо як шабакаи веб-сайтҳоро аз як дашбор (dashboard) -и WordPress сохта ва идора кунед. Шумо метавонед ҳама чизҳоро идора кунед, аз ҷумла шумораи сайтҳо, хусусиятҳо, темаҳо ва намудаҳои корбарон. Имкон аст сотсорон ва ҳазорҳо сайтҳоро идора кунед.

## Шабака (Network)

Дар мафҳуми WordPress, шабакаи Multisite он ҷо мебошад, ки шумо метавонед шумораи **subsites**-ро аз як дашбор идора кунед. Гarchi сохтани шабакаи Multisite ба таври пайвастӣ аз ҷониби пешгирвандагони хостинг фарқ мекунад, натиҷаи он одатан фақат каме дастурҳо дар файли **wp-config.php** мебошад, ки ба WordPress ишора медиҳанд, ки он дар ин режим муайян кор мекунад.

Ин тафовутҳои гуногуни бисёфта байни шабакаи Multisite ва инсталлятҳои WordPress-и алоҳида вуҷуд дорад, ки мо онҳоро ба таври қисқа шарҳ медиҳем.

## Database (База маълумот)

**Database** (База маълумот) як маҷмӯи маълумоти сохторӣ ва надорамест аст. Дар истилоҳи компютер, база маълумот ба барномаҳо ишора менамояд, ки барои нигоҳ доштани маълумот ва тартиб додани он истифода мешаванд. Онро ҳамчун шкафаи файли навиданӣ намоясонед, ки дар он ҷо маълумот дар қисмҳои гуногун бо номи **tables** (ҷадвалҳо) нигоҳ дошта мешавад.

WordPress Multisite аз як база маълумот истифода мебарад ва ҳар як subsite худи ҷадвалҳоро ба даст мегирад, ки ID-и блог дар пефикс (prefix) он аст. Аз рӯи ин, яъне пас аз он ки шумо инсталлят шабакаро иҷро кунед ва як subsite созед, шумо бояд ин ҷадвалҳоро дошта бошед:

_wp_1_options_ \- ҷадвали опшнҳо барои подсайти аввал

_wp_2_options_ \- ҷадвали опшнҳо барои подсайти дуюм

## Провайдер хостинг (Hosting provider)

Провайдер хостинг корхона ё шахсони ааммуро имкон медиҳад, ки вебсайти худро тавассути Интернет омода созанд. Услубҳои пешниҳоди провайдеров хостинг метавонанд гуногун бошанд, аммо одатан аз лоиҳаи вебсайт, макони сарчашма (storage space) дар хост ва пайвастагии ба Интернет иборат мебошанд.

## Домен (Domain)

Номи доменӣ адреси аст, ки одамон барои расонидани сайт-и шумо истифода мебаранд. Он ба браузер вебсайт мегӯяд ки кӯшиш кунад дар кадом ҷо ба ҷойгир кардани сайти шумо. Ҳамчун адрес кори, доменӣ он аст, ки одамон онлайн ба сайт-и шумо расо меёванд. Вали аз он чӣ гуна бо истифода аз вебсайти мо мехоҳед, бояд дар кадом ҷо адреси вебсайтро дар адрес браузер интихоб кунед, ки он [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) аст, ки [**ultimatemultisite.com**](http://ultimatemultisite.com) номи доменӣ аст.

## Поддомен (Subdomain)

Поддомен як намуди ҳиерархияи вебсайт дар пайолери домени асосӣ аст, аммо ба ҷои истифода аз папкаҳо барои ташкили мундариҷа дар вебсайт, он чунин сайт-и худро месозад. Он ҳамчун [**https://site1.domain.com/**](https://site1.domain.com/) пешниҳод мешавад, ки дар он _site1_ номи поддомен ва [_domain.com_](http://domain.com) домени асосӣ аст.

## Подкаталог (Subdirectory)

Подкаталог як намуди ҳиерархияи вебсайт дар пайолери домени асосӣ аст, ки аз папкаҳо барои ташкили мундариҷа дар вебсайт истифода мебарад. Подкаталог ба папка ҳам мебошад ва номҳо метавонанд якхела истифода шаванд. Он ҳамчун [**https://domain.com/site1**](https://domain.com/site1) пешниҳод мешавад, ки дар он _site1_ номи подкаталог ва [_domain.com_](http://domain.com) домени асосӣ аст.

## Подсайт (Subsite)

Подсайт — это дочерний сайт, который вы создаете в сети Multisite. Он может быть либо **поддоменом**, либо **подкаталогом**, в зависимости от того, как настроена ваша установка WordPress Multisite.

## Супер администратор (Super Admin)

Супер администратор WordPress — это роль пользователя с полными возможностями для управления всеми подсайтами в сети Multisite. Для пользователей Multisite это **самый высокий уровень доступа**, который вы можете предоставить вашей установке WordPress.

## Плагин (Plugin)

В целом, плагин — это набор кода, который добавляет дополнительную функциональность вашему сайту WordPress. Это может быть что угодно: от простого изменения логотипа входа до сложной электронной коммерции. _Woocommerce и Contact Form_ — примеры плагинов.

На WordPress Multisite плагины можно устанавливать только из административной панели сети под управлением Супер администратора. Администраторы подсайтов могут активировать и деактивировать плагины только внутри своего подсайта.

## Темы (Themes)

Тема WordPress — это группа файлов (_графика, стилей и кода_), которая определяет общий вид сайта. Она предоставляет все фронтенд-стили, такие как оформление шрифтов, макет страницы, цвета и т. д.

Как и плагины, темы в WordPress Multisite можно устанавливать только Супер администратором, а активировать на уровне подсайта — администраторами этих подсайтов.

## Шаблон сайта (Site Template)

**Шаблон сайта** — это шаблонный сайт, который можно использовать как основу при создании новых сайтов в вашей сети.

Это означает, что вы можете создать базовый сайт, активировать разные плагины, установить активную тему и настроить его так, как вам нужно. Затем, когда ваш клиент создает новую учетную запись, вместо стандартного сайта WordPress без какого-либо значимого контента, он получит копию вашего базового сайта со всеми настройками и содержимым уже готовыми.

## Сопоставление доменов (Domain Mapping)

**Domain mapping** боWordPress — это способ перенаправлять пользователей на нужный хостинг через адрес сайта. В WordPress Multisite подсайты создаются либо с помощью поддиректории, либо с использованием домена-поддомена (subdomain). Что же такое domain mapping? Это позволяет пользователям подсайтов использовать верхнеуровневый домен, например [**joesbikeshop.com**](http://joesbikeshop.com), чтобы адрес их сайта выглядел более профессионально.

## SSL

SSL означает **Secure Sockets Layer**. Это цифровой сертификат, который подтверждает личность веб-сайта и обеспечивает зашифрованное соединение. В наши дни он используется как стандартная технология для обеспечения безопасности интернет-соединения и защиты любых конфиденциальных данных, передаваемых между двумя системами, предотвращая чтение и изменение любой информации, включая потенциальные личные данные, злоумышленниками. Современные браузеры требуют SSL, что делает его необходимым при создании и запуске веб-сайта.

## Media (Медиафайлы)

Медиа — это изображения, аудио, видео и другие файлы, которые делают веб-сайт.

Сеть сайтов использует одну общую базу данных в WordPress Multisite, но поддерживает отдельные пути на файловой системе для медиафайлов.

Стандартное расположение WordPress (wp-content/uploads) остается прежним; однако его путь изменяется, чтобы отражать уникальный ID сетевого сайта. В результате файлы медиа для сетевого сайта появляются по адресу wp-contents/uploads/site/[id].

## Permalinks (Постоянные ссылки)

Permalinks — это постоянные URL ваших отдельных записей или страниц на сайте. Permalinks также называют **pretty links** (красивые ссылки). По умолчанию URL WordPress используют формат строки запроса, который выглядит примерно так:

[**http://www.example.com/registration**](http://www.example.com/registration)

## Ultimate Multisite

Ultimate Multisite это плагин для WordPress, созданный для установок WordPress Multisite, который превращает вашу установку WordPress в премиальную сеть сайтов — как [WordPress.com] — позволяя клиентам создавать сайты по ежемесячной, квартальной или годовой подписке (вы также можете создавать бесплатные тарифы).

## Checkout Form (Форма оформления заказа)

Checkout Form — это форма заказа, которая может быть одношаговой или многошаговой и включает создание субсайта, членства и учетных записей пользователей через регистрацию Ultimate Multisite. Она состоит из различных полей и форм оплаты, которые пользователь должен заполнить в процессе регистрации.

## Webhook (Вебхук)

Webhook (также называемый веб-обратным вызовом или HTTP push API) — это способ для приложения предоставлять другим приложениям информацию в режиме реального времени. Вебхук доставляет данные другим приложениям по мере их возникновения, что означает, что вы получаете данные немедленно.

**Ultimate Multisite webhooks** открывают бесконечные возможности, позволяя администраторам сети выполнять всевозможные безумные, но полезные интеграции, особенно если использовать их в сочетании с такими сервисами, как _Zapier и IFTTT_.

## Events (События)

Event (событие) — это действие, которое происходит в результате действия пользователя или другого источника, например, клика мыши. Ultimate Multisite ведет учет всех событий и журналов, происходящих во всей вашей сети. Он отслеживает различные активности, происходящие на вашем мультисайте, такие как изменения планов.
