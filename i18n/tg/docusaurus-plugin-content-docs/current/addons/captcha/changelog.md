# Журнали тағйироти Captcha {#captcha-changelog}

Version: 1.5.0 - Нашр шуд дар 2026-05-22
* Нав: Маҳдудкунандаи суръати hard-stop — ҳар GET ва POST-ро дар сатҳҳои бо captcha ҳифзшуда (wp-login / register / lost-password / comments, WooCommerce my-account / checkout / pay-for-order, Ultimate Multisite checkout / inline-login) мешуморад ва бо HTTP 429, як `Retry-After` header ва хоби тасодуфии tarpit (1–5с, бо ҳадди сахт 15с) ҷавоб медиҳад.
* Нав: Танзимоти `cap_rate_limit_tarpit_min` / `cap_rate_limit_tarpit_max` барои танзими равзанаи tarpit.
* Нав: Филтри `wu_cap_rate_limit_whitelist_ip` барои озод кардани диапазонҳои IP-и боэътимод.
* Нав: Амали `wu_cap_rate_limit_will_block`, ки фавран пеш аз фиристода шудани ҷавоби hard-stop иҷро мешавад.
* Нав: Муайянкунии IP-и client-и тобовар ба қалбакӣ. `Captcha_Core::get_client_ip()` (манбаи ҳақиқат барои калидҳои сатили маҳдудияти суръат, captcha siteverify `remoteip` ва hash-ҳои IP-и омор) акнун модели сахти эътимодро маҷбур мекунад: REMOTE_ADDR поя аст, `CF-Connecting-IP` танҳо вақте эътироф мешавад, ки ҳамтои бевосита дар дохили диапазони ҷории IP-и Cloudflare бошад, ва `X-Forwarded-For` танҳо вақте эътироф мешавад, ки ҳамтои бевосита дар рӯйхати trusted-proxy-и дар admin танзимшуда бошад, бо ҳаракати аз рост ба чап, ки пеш аз қарор гирифтан бар IP-и меҳмон hop-ҳои боэътимод/CF-ро мегузарад.
* Нав: Танзими `cap_trust_cloudflare_headers` (пешфарз OFF) — ҳангоми пушти Cloudflare будан ба эътимоди `CF-Connecting-IP` розигӣ диҳед. plugin бо snapshot-и Cloudflare CIDR-и дарунсохт меояд ва онро ҳар ҳафта тавассути wp-cron нав мекунад, бо fallback-и дарунсохт агар навсозӣ ноком шавад.
* Нав: Танзими `cap_trusted_proxies` — textarea-и CIDR-ҳо ё IP-ҳои холис (якто дар ҳар сатр, шарҳҳои `#` иҷозатанд), ки proxy-ҳо / load-balancer-ҳои хатти пеши худатонро номбар мекунад. Бе ин, `X-Forwarded-For` ҳатто вақте ки маҳдудкунандаи суръат фаъол аст, нодида гирифта мешавад.
* Нав: Ҳангоми бори аввал фаъол кардан, муайянкунии худкори эҳтимолии ҳолати Cloudflare / proxy бо огоҳиномаи admin-и яккликаи "Танзимоти муайяншударо татбиқ кунед". plugin ҳеҷ гоҳ арзишҳои захирашудаи шуморо рӯйнавис намекунад; агар трафики баъдӣ нишон диҳад, ки config-и шумо дигар ба воқеият мувофиқат намекунад (масалан, Cloudflare диапазонҳои CIDR-ро иваз кард ва proxy CIDR-и шумо акнун кӯҳна аст), огоҳиномаи номувофиқатии ғайриқобили бастан навсозии тавсияшударо нишон медиҳад.
* Ислоҳ шуд: Ҳолати ноаён дигар `cap_security_level`-ро бесадо ба FAST паст намекунад — сатҳи танзимкардаи admin риоя мешавад. Филтри нави `wu_cap_server_security_level` барои сайтҳое дастрас аст, ки мантиқи фармоишӣ мехоҳанд.
* Ислоҳ шуд: Ҳисобкунаки омории `rate_limits_triggered` акнун дар ҳар блок зиёд мешавад, на танҳо дар роҳи нодири backstop пас аз муваффақият.
* Ислоҳ шуд: `Captcha_Core::get_client_ip()` акнун манбаи ягонаи ҳақиқат барои нисбатдиҳии IP-и меҳмон дар саросари маҳдудкунандаи суръат, провайдерҳои captcha (reCAPTCHA + hCaptcha `siteverify`) ва омор аст — пӯшидани вектори қалбакӣ, ки дар он дархостҳои мустақим ба origin-server бо `CF-Connecting-IP` header-и сохта аз рӯи IP-и қалбакӣ ба ҷои ҳамтои воқеӣ bucket мешуданд.
* Ислоҳ шуд: Дарвозаи маҳдудияти суръати WooCommerce classic checkout акнун дар `template_redirect` (афзалият 1) ба ҷои `woocommerce_before_checkout_form` иҷро мешавад. Hook-и сатҳи form ҳангоми холӣ будани сабад ҳеҷ гоҳ иҷро намешавад, бинобар ин трафики flood, ки ҳеҷ гоҳ product илова намекунад, маҳдудкунандаро пурра давр мезад.
* Ислоҳ шуд: Дарвозаи маҳдудияти суръати WooCommerce pay-for-order акнун дар `template_redirect` ба ҷои `woocommerce_before_pay_action` иҷро мешавад. Охирин танҳо баъд аз муваффақ шудани `wp_verify_nonce('woocommerce-pay')` иҷро мешавад, яъне ҳамлагарони unauth (модели воқеии таҳдид) ҳеҷ гоҳ маҳдудкунандаро ба кор намеандохтанд.
* Ислоҳ шуд: Дарвозаи маҳдудияти суръати WooCommerce Store API (blocks) checkout акнун дар `rest_pre_dispatch` ба ҷои `woocommerce_store_api_checkout_update_order_from_request` иҷро мешавад. Охирин танҳо баъд аз он иҷро мешавад, ки Store API сабад ва field-ҳои ҳисоббарориро тасдиқ мекунад, бинобар ин bot-ҳои unauth аз тасдиқкунанда 400 мегирифтанд ва ҳеҷ гоҳ маҳдудкунандаро ба кор намеандохтанд.
* Ислоҳ шуд: Дарвозаи маҳдудияти суръати Ultimate Multisite inline-login акнун дар `wu_ajax_nopriv_wu_inline_login` бо афзалияти 1 (ва оинаи logged-in) ба ҷои `wu_before_inline_login` иҷро мешавад. Охирин танҳо баъд аз муваффақ шудани `check_ajax_referer('wu_checkout')` иҷро мешавад, бинобар ин bot-ҳои unauth бе nonce-и дурусти wu_checkout 403 мегирифтанд ва ҳеҷ гоҳ маҳдудкунандаро ба кор намеандохтанд.
* Ислоҳ шуд: `Rate_Limiter::enforce()` акнун муҳофизи якбор-дар-ҳар-дархостро, ки бо `surface|ip` калид шудааст, татбиқ мекунад, бинобар ин hook-ҳои болооб, ки дар ҳар render ду бор иҷро мешаванд (махсусан `wu_setup_checkout` дар Ultimate Multisite), дигар остонаи муассири маҳдудияти суръатро ду баробар кам намекунанд.
* Ислоҳ шуд: Дарвозаҳои сатҳи маҳдудияти суръат дигар ба `Captcha_Core::is_whitelisted()` (филтри `wu_captcha_whitelisted`) муроҷиат намекунанд. Он филтр нишон медиҳад, ки "captcha аллакай аз ҷониби сатҳи дигар коркард шудааст" ва ба ҳифзи flood иртиботи мустақим надорад — интегратсияи WooCommerce онро hook мекард, то captcha-и вуруди WordPress-ро ҳангоми мавҷуд будани Woo nonce гузарад, ки ба ҳисобкунии суръат сироят кард ва ба Woo POST-ҳо иҷозат дод, ки аз маҳдудкунанда канорагирӣ кунанд. Филтри махсуси маҳдудияти суръат `wu_cap_rate_limit_whitelist_ip` ягона bypass аст, ки акнун татбиқ мешавад.
