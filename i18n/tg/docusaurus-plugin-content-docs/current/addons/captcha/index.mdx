# Captcha

**Ботҳоро боздоред. Даромадро муҳофизат кунед. Бе дарди сари танзимот.**

Шумо шабакаи multisite-ро бо доменҳои фармоишӣ идора мекунед? Шумо ин дардро медонед: Google reCAPTCHA ва hCaptcha аз шумо талаб мекунанд, ки *ҳар як доменро* дар консоли маъмурии онҳо сабт кунед. Сайти нави муштарӣ илова мекунед? Танзимоти captcha-ро нав кунед. Домени фармоиширо пайваст мекунед? Боз нав кунед. Ин ҳеҷ гоҳ тамом намешавад.

**Ultimate Multisite: Captcha ҳама чизро дигар мекунад.**

## Cap Captcha: Муҳофизати худмизбон, ки танҳо кор мекунад {#cap-captcha-self-hosted-protection-that-just-works}

Провайдери нави мо **Cap Captcha** барои шабакаҳои multisite тағйирдиҳандаи бозӣ аст:

* **Бе танзимот** — Addon-ро фаъол кунед ва шумо муҳофизат мешавед. Бе API key, бе Google account, бе hCaptcha dashboard, бе рӯйхати сафеди доменҳо.
* **Дар ҳар домен ба таври худкор кор мекунад** — Зердоменҳо, доменҳои пайвастшуда, доменҳои муштариён—ҳамаи онҳо фавран муҳофизат мешаванд.
* **Худмизбон ва хусусӣ** — Маълумоти шумо ҳеҷ гоҳ сервери шуморо тарк намекунад. Комилан мутобиқ ба GDPR, бе пайгирии тарафи сеюм.
* **Proof-of-Work-и ноаён** — Корбарон дар пасзамина муаммои ҳисоббарориро ҳал мекунанд. Бе пахш кардани гидрантҳои оташнишонӣ, бе шабакаҳои тасвирии асабонӣкунанда.
* **Сабук** — Ҳамагӣ 20KB. Checkout-и шуморо суст намекунад.

## Муҳофизат аз Card Testing дар WooCommerce {#woocommerce-card-testing-protection}

Ҳамлаҳои card testing ҳар сол ба тоҷирон миллиардҳо зарар мерасонанд. Қаллобон ботҳоро барои санҷидани кортҳои кредитии дуздидашуда дар checkout-и WooCommerce-и шумо истифода мебаранд—шумо ҳаққи коркардро пардохт мекунед, онҳо рақамҳои кортҳои эътиборнок мегиранд.

**Мо он ҷо муҳофизат мекунем, ки аз ҳама муҳим аст:**

* **Муҳофизати Store API** — Ҳамлаҳои ботро ба `/wp-json/wc/store/v1/checkout`, ки captcha-ҳои анъанавии формаҳоро давр мезананд, масдуд мекунад
* **Бо PayPal Payments мувофиқ** — Бо WooCommerce PayPal Payments бенуқсон кор мекунад
* **Омор дар вақти воқеӣ** — Дақиқан бинед, ки чанд ҳамларо масдуд мекунед

## Ҳанӯз Google reCAPTCHA ё hCaptcha лозим аст? {#still-need-google-recaptcha-or-hcaptcha}

Мо шуморо таъмин кардаем. Дастгирии пурра барои:

* Google reCAPTCHA v2 (Checkbox)
* Google reCAPTCHA v2 (Invisible)
* Google reCAPTCHA v3 (Score-based)
* hCaptcha (Standard)
* hCaptcha (Invisible)

Ҳар вақт аз танзимоти шабакаи худ байни провайдерҳо гузаред.

## Барои миқёспазирӣ сохта шудааст {#built-for-scale}

* Танзимоти сатҳи шабака бо бекоркуниҳои барои ҳар сайт
* Dashboard-и омор барои назорати муҳофизат
* Сатҳҳои танзимшавандаи амният (Fast, Medium, Maximum)
* Барои таҳиягарон қулай, бо меъмории провайдери васеъшаванда

## Насб {#installation}

1. `ultimate-multisite-captcha`-ро ба директорияи `/wp-content/plugins/`-и худ бор кунед
2. Plugin-ро дар сатҳи шабака фаъол кунед
3. **Ҳамин аст. Шумо муҳофизат шудаед.**

Cap Captcha ба таври худкор фаъол мешавад—ҳеҷ танзимоте барои танзим кардан нест, ҳеҷ API key барои ворид кардан нест. Ҳар endpoint-и дастгиришаванда дар тамоми шабакаи шумо аз лаҳзаи фаъолсозӣ муҳофизат мешавад.

**Ихтиёрӣ:** Барои истифодаи Google reCAPTCHA ё hCaptcha ба ҷои он, ба **Ultimate Multisite → Settings → Captcha** гузаред ва провайдери дилхоҳи худро интихоб кунед.

## Саволҳои зуд-зуд пурсидашаванда {#frequently-asked-questions}

## Оё ман бояд чизеро танзим кунам? {#do-i-need-to-configure-anything}

Не! Вақте ки addon-ро фаъол мекунед, Cap Captcha ба таври худкор фаъол мешавад. Тамоми шабакаи шумо фавран муҳофизат мешавад—ҳеҷ танзимоте барои танзим кардан нест, ҳеҷ API key барои ворид кардан нест.

## Оё барои Cap Captcha ба API key-ҳо ниёз дорам? {#do-i-need-api-keys-for-cap-captcha}

Не. Cap Captcha комилан худмизбон аст. Бе account-ҳои беруна, бе сабти домен, бе дарди сари танзимот.

## Оё ин checkout-и маро суст мекунад? {#will-this-slow-down-my-checkout}

Cap Captcha ҳамагӣ 20KB аст ва чолишҳои proof-of-work-ро дар пасзамина иҷро мекунад. Муштариёни шумо ҳеҷ таъхиреро ҳис намекунанд.

## Оё ин аз ҳамлаҳои card testing муҳофизат мекунад? {#does-this-protect-against-card-testing-attacks}

Бале! Бар хилофи captcha-ҳои анъанавӣ, ки танҳо формаҳои HTML-ро муҳофизат мекунанд, интегратсияи WooCommerce-и мо мустақиман ба Store API пайваст мешавад, то ҳамлаҳои ботро дар сатҳи API масдуд кунад—ҳамон ҷое, ки аксари ҳамлаҳои card testing рух медиҳанд.

## Оё ман метавонам дар сайтҳои гуногун провайдерҳои гуногуни captcha-ро истифода барам? {#can-i-use-different-captcha-providers-on-different-sites}

Провайдери captcha дар сатҳи шабака муқаррар мешавад, аммо шумо метавонед танзимоти душвориро вобаста ба ниёзҳои худ тағйир диҳед.

## Оё ин ба GDPR мутобиқ аст? {#is-this-gdpr-compliant}

Cap Captcha 100% худмизбон аст ва ҳеҷ маълумот ба хизматрасониҳои беруна фиристода намешавад. Барои Google reCAPTCHA ва hCaptcha, лутфан сиёсатҳои дахлдори махфияти онҳоро баррасӣ кунед.

## Маҳдудсозии суръат {#rate-limiting}

Ultimate Multisite: Captcha v1.5.0 як маҳдудкунандаи суръати қатъиро ҷорӣ мекунад, ки шабакаи шуморо аз ҳамлаҳои brute-force ва card-testing муҳофизат мекунад.

### Чӣ тавр кор мекунад {#how-it-works}

Маҳдудкунандаи суръат ҳар дархости GET ва POST-ро дар сатҳҳои бо captcha муҳофизатшуда ҳисоб мекунад:

* **Endpoint-ҳои воридшавии WordPress** — wp-login, register, lost-password, comments
* **Checkout-и WooCommerce** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

Вақте ки боздидкунанда аз rate limit мегузарад, plugin бо инҳо ҷавоб медиҳад:

* **HTTP 429 (Too Many Requests)** status code
* **Retry-After** header, ки нишон медиҳад кай дубора кӯшиш кардан лозим аст
* **Хоби тасодуфии tarpit** (ба таври пешфарз 1–5 сония, бо ҳадди сахт то 15 сония) барои суст кардани ҳамлагарон

### Танзимот {#configuration}

#### Вақтбандии Tarpit {#tarpit-timing}

Таъхири тасодуфиро, ки ба дархостҳои rate-limited татбиқ мешавад, идора кунед:

* **`cap_rate_limit_tarpit_min`** — Давомнокии ҳадди ақали хоб дар сонияҳо (пешфарз: 1)
* **`cap_rate_limit_tarpit_max`** — Давомнокии ҳадди аксари хоб дар сонияҳо (пешфарз: 5, бо ҳадди сахт то 15)

Ин танзимот дар **Ultimate Multisite → Settings → Captcha** дастрасанд.

### Васеъшавандагӣ {#extensibility}

#### Филтри рӯйхати сафеди IP {#ip-whitelist-filter}

Диапазонҳои боэътимоди IP-ро аз маҳдудсозии суръат озод кунед:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // Whitelist your monitoring service
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### Амали пеш аз масдудкунӣ {#pre-block-action}

Мантиқи фармоиширо фавран пеш аз фиристода шудани масдудкунии rate-limit иҷро кунед:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // Log the block, notify admins, or update custom metrics
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## Муайянкунии IP-и муштарӣ ва проксиҳои боэътимод {#client-ip-detection--trusted-proxies}

Ultimate Multisite: Captcha v1.5.0 муайянкунии IP-и муштариро ҷорӣ мекунад, ки ба сохтакорӣ тобовар аст. Усули `Captcha_Core::get_client_ip()` ҳоло манбаи ягонаи ҳақиқат барои нисбатдиҳии IP-и боздидкунанда дар саросари зерин аст:

* Калидҳои сатили маҳдудсозии суръат
* Captcha provider `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* Хэшҳои IP-и омор

### Модели эътимод {#trust-model}

Плагин иерархияи қатъии эътимоди IP-ро татбиқ мекунад:

1. **REMOTE_ADDR** — Ҳамеша боэътимод аст (IP-и ҳамтои бевосита)
2. **CF-Connecting-IP** — Танҳо вақте ба назар гирифта мешавад, ки:
   - Ҳамтои бевосита дар дохили диапазони ҷории IP-и Cloudflare бошад
   - `cap_trust_cloudflare_headers` фаъол бошад (пешфарз: OFF)
3. **X-Forwarded-For** — Танҳо вақте ба назар гирифта мешавад, ки:
   - Ҳамтои бевосита дар рӯйхати проксиҳои боэътимоди танзимшудаи админи шумо бошад
   - Header аз рост ба чап таҳлил шуда, гузаришҳои боэътимод/Cloudflare-ро нодида гирад

### Танзимот {#configuration-1}

#### Эътимод ба Header-и Cloudflare {#cloudflare-header-trust}

Эътимод ба header-и `CF-Connecting-IP`-и Cloudflare-ро фаъол кунед:

* **`cap_trust_cloudflare_headers`** — Ба ON гузоред, то ба эътимоди `CF-Connecting-IP` ҳамроҳ шавед
  - Пешфарз: OFF (ғайрифаъол)
  - Плагин бо snapshot-и дарунсохти Cloudflare CIDR меояд
  - Диапазонҳои CIDR ҳар ҳафта тавассути wp-cron нав карда мешаванд
  - Агар навсозӣ ноком шавад, fallback-и дарунсохт истифода мешавад

#### Танзимоти проксии боэътимод {#trusted-proxy-configuration}

Проксиҳо ва load-balancer-ҳои хатти пеши худро муайян кунед:

* **`cap_trusted_proxies`** — Textarea аз CIDR-ҳо ё IP-ҳои холис (якто дар ҳар сатр)
  - Шарҳҳо иҷозат дода мешаванд (сатрҳое, ки бо `#` оғоз мешаванд)
  - Бе ин танзим, `X-Forwarded-For` ҳатто вақте ки маҳдудкунандаи суръат фаъол аст, нодида гирифта мешавад
  - Мисол:
    ```
    # Our load balancer
    192.0.2.0/24

    # Backup proxy
    198.51.100.50
    ```

#### Муайянкунии худкор ҳангоми фаъолсозии аввал {#first-enable-auto-detection}

Вақте шумо маҳдудкунандаи суръатро бори аввал фаъол мекунед, плагин ҳолати эҳтимолии Cloudflare ва проксии шуморо муайян мекунад ва огоҳиномаи админ бо як клик "Apply detected settings" нишон медиҳад.

* Плагин арзишҳои захирашудаи шуморо **ҳеҷ гоҳ рӯйнавис намекунад**
* Агар трафики минбаъда нишон диҳад, ки config-и шумо дигар ба воқеият мувофиқат намекунад (масалан, Cloudflare диапазонҳои CIDR-ро иваз кард), огоҳиномаи номувофиқатии пинҳоннашаванда навсозии тавсияшударо нишон медиҳад

### Чаро ин муҳим аст {#why-this-matters}

Модели қатъии эътимоди IP як вектори муҳими сохтакориро мебандад: дархостҳои мустақим ба origin-server бо header-и сохтаи `CF-Connecting-IP` қаблан ба ҷои IP-и ҳамтои воқеӣ аз рӯи IP-и сохташуда гурӯҳбандӣ мешуданд. Ин махсусан барои инҳо муҳим аст:

* **Маҳдудсозии суръат** — Ҳамлагарон наметавонанд бо сохтакории IP аз маҳдудиятҳо гузаранд
* **Омор** — Метрикаҳои ҳамлаи шумо IP-ҳои воқеии боздидкунандагонро инъикос мекунанд, на IP-ҳои сохташударо
* **Санҷиши Captcha** — Провайдерҳо барои арзёбии хавф IP-и дурусти боздидкунандаро мегиранд
