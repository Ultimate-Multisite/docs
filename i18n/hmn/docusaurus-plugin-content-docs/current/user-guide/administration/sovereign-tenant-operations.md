# Sovereign Tenant Operations

Ultimate Multisite: Multi-Tenancy 1.2.0 нь sovereign tenants (өөрийн өгөгдлийн сан, файлын систем үндсэн хэсэг болон маршрутийн контексттэй ажилладаг подсайт) -ийн хувьд ажиллагааны хэрэгслүүдийг нэмж орууллаа: сүлжээний администратор харахад харагдах ч өөрийн өгөгдлийн сан, файлын систем үндсэн хэсэг болон маршрутийн контексттэй ажилладаг подсайт.

Энэ хуудсыг таны тусгайлан тусгаарлагдсан хэрэглэгчийн сайтнуудыг удирдах үед, холбоогүй сайтуудын хоорондох шилжилт хийх үед эсвэл стандарт подсайтыг sovereign infrastructure руу шилжүүлэх үед ашиглана.

## Администратор нарт ямар өөрчлөлт орсон бэ?

- **Stateless tenant autologin** — Сүлжээний администратор нь урт хугацааны хамтарсан session state-д тулгуурлахгүйгээр sovereign tenant руу орох боломжтой. SSO token нь зорилгоор тодорхойлогдсон, эх үүсвэрийг тогтоосон, давталтыг хамгаалсан бөгөөд богино хугацааны төгсгөлтэй байдаг.
- **Sovereign-aware routing** — Хуучин тусгаарлагдсан сүлжээ болон sovereign tenants нь ижил сайт маршрутын зам ашигладаг тул хуучин ба шинэ тусгаарлагдсан устат байгуулалт хоорондын эхлэлийн ялгаа багасна.
- **Verified migration state** — Шилжилт хийх үед хэрэглэгчийн тохиргоо, өгөгдлийн сангийн бичвэрийн эрх, queue drain статусы, болон хуучин хүснэгтийн байхгүй байгаа эсэхийг шалгаж, tenant-ийг бүрэн дууссан гэж үзэхээс өмнө баталгаажуулдаг.
- **Safer teardown** — Sovereign-ийг устгах үед одоо tenant-ийн мэдээллийг цэвэрхэн арилгаж байгаа тул, устгасан tenant нь хуучин өгөгдлийн сангийн хандалт үлдээхгүй.

## Sovereign tenant руу орох

1. **Network Admin > Ultimate Multisite > Sites**-ийг нээ.
2. Sovereign tenant-ийг сонго.
3. Боломжтой бол нууц үг хуулбарлах эсвэл түр администратор бүрт оруулахын оронд **Visit (SSO)**-г ашигла.

Орох үйл явц нь тухайн tenant-д богино хугацааны login token үүсгэж, SSO-ийн үйл явдлыг tenant-ийн audit trail-д бүртгэнэ. Хэрэв товчлуур ажиллахгүй бол tenant-ийн домен нь хүлээгдэж буй устат байгуулалтад ямар хамааралтай байгаа эс, мөн tenant нь сүлжээний талын SSO endpoint руу хүрэх боломжтой эсэхийг шалгана уу.

## Remote-site operations checklist

Sovereign эсвэл remote tenant-ийг өөрчлөхөөс өмнө дараах зүйлсийг баталгаажуулна:

- Domain ຂອງ tenant (tenant)  apunt (points) ไปที่ host ที่เป็นเจ้าของ filesystem ของ tenant นั้นๆ
- Host ฐานข้อมูลของ tenant ตรงกับ host binding ที่ตั้งค่าไว้สำหรับการติดตั้งนั้น

## Deleting sovereign tenants

ການລຶບ tenant ທີ່ເປັນ sovereign ນັ້ນແມ່ນການເຮັດໃຫ້ຂໍ້ມູນຫາຍໄປ. ກວດສອບສະຖານະ backup ແລະ export ກ່ອນ, ແລ້ວຈຶ່ງລຶບຈາກໜ້າ site management. ຂັ້ນຕອນ teardown 1.2.0 ຈະເອົາ database credentials ຂອງ tenant ອອກເປັນສ່ວນໜຶ່ງຂອງການເຮັດຄວາມສະອາດ, ແຕ່ຜູ້ບໍລິຫານຄວນກວດສອບອີກວ່າ user ແລະ folder ຂອງ database level ຢູ່ host ນັ້ນໆ ໄດ້ຖືກລຶບໄປແລ້ວເມື່ອໃຊ້ external hosting panels.

:::warning
ຢ່າລຶບ tenant sovereign ໃນຂະນະທີ່ການກວດສອບ migration ກໍາລັງດໍາເນີນຢູ່ ຫຼື ໃນຂະນະທີ່ async push jobs ຍັງຢູ່ໃນຄອຍ (queued). ລໍຖ້າໃຫ້ການກວດສອບສໍາເລັດເພື່ອບໍ່ໃຫ້ teardown ເອົາ credentials ທີ່ຈໍາເປັນຂອງ jobs ທີ່ລໍຖ້າມາອອກໄປ.
:::
