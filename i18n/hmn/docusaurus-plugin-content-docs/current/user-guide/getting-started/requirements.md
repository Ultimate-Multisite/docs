# ຂໍ້ກຳນົດຂອງ Ultimate Multisite {#ultimate-multisite-requirements}

ສິ່ງທຳອິດທີ່ເຈົ້າອາດຈະຖາມຕົວເອງກ່ອນທີ່ຈະຊື້ Ultimate Multisite ແມ່ນວ່າຕ້ອງມີຫຍັງແດ່ເພື່ອໃຫ້ plugin ນີ້ເຮັດວຽກໄດ້. ເບິ່ງດີ, ໂຊກດີທີ່ມັນມີຂໍ້ກຳນົດພື້ນຖານບໍ່ຫຼາຍປານ: **ຊື່ໂດເມນ (domain name)**, **ຜູ້ໃຫ້ບໍລິການໂຮດຈິງ (hosting provider)** ທີ່ດີ ແລະ ການຕິດຕັ້ງ WordPress **Multisite**. ນັ້ນແມ່ນທັງໝົດທີ່ເຈົ້າຕ້ອງການ!

## ຊື່ໂດເມນ (Domain Name) {#domain-name}

ກ່ອນອື່ນ, ເຊິ່ງເຮົາຂໍເວົ້າກ່ຽວກັບຊື່ໂດເມນ.

ເວລາເລືອກຊື່ໂດເມນ, ໃຫ້ຈື່ໄວ້ວ່າ niche (ຄວາມຊ່ຽວຊານສະເພາະຂອງເຈົ້າ) ຂອງເຈົ້າແມ່ນຫຍັງ ແລະ ເຈົ້າຈະສ້າງຫຍັງດ້ວຍ Ultimate Multisite: ເຈົ້າຢາກສ້າງອົງການຈັດຕັ້ງ (agency)? ຂາຍເວັບໄຊແບບຮ້ານຄ້າ? ຫຼື ອາດຈະເປັນການໃຫ້ບໍລິການໂຮດຈິງ WordPress? ທຸກຢ່າງເຮັດໄດ້ກັບ Ultimate Multisite.

Plugin ຂອງພວກເຮົາສະໜອງບໍລິການສະໝັກໃຊ້, ການເກັບຄ່າທຳນຽມ, ແບບຟອມ checkout, ວາວສ່ວນຫຼຸດ ແລະ ການສື່ສານກັບລູກຄ້າ. ເຮືອນວຽກທີ່ຕ້ອງເຮັດຫຼາຍເພື່ອຕິດຕັ້ງ, ຕັ້ງຄ່າ ແລະ ຮັກສາ WordPress Multisite ແມ່ນຖືກຊ່ວຍໂດຍ Ultimate Multisite.

ບໍ່ວ່າເຈົ້າຈະເລືອກ niche ອັນໃດ, ສາມາດຈັດການໄດ້ໂດຍໃຊ້ plugin ຂອງພວກເຮົາໄດ້ຕະຫຼອດ ເມື່ອເຈົ້າຂາຍເວັບໄຊເປັນບໍລິການ! ໃຫ້ແນ່ໃຈວ່າເຈົ້າເລືອກຊື່ໂດເມນທີ່ເຂົ້າກັບແຜນທຸລະກິດຂອງເຈົ້າ.

## ຜູ້ໃຫ້ບໍລິການໂຮດຈິງ (Hosting Provider) {#hosting-provider}

ເມື່ອເຈົ້າມີຊື່ໂດເມນທີ່ໜ້າສົນໃຈທີ່ເປັນຕົວແທນຂອງ niche ແລະ ທຸລະກິດຂອງເຈົ້າແລ້ວ, ເຈົ້າຈະຕ້ອງເລືອກບໍລິສັດໂຮດຈິງເວັບໄຊທີ່ດີເພື່ອເກັບຮັກສາເຄືອຂ່າຍຂອງເຈົ້າດ້ວຍ Ultimate Multisite. ມີທາງເລືອກທີ່ດີໃນຕະຫຼາດນີ້. ຜູ້ໃຫ້ບໍລິການໂຮດຈິງບາງແຫ່ງສາມາດເຮັດວຽກເຂົ້າກັນໄດ້ດີກັບ Ultimate Multisite, ເຊິ່ງອາດເປັນປັດໄຈຕັດສິດໃຈຂຶ້ນ ຂຶ້ນຢູ່ກັບວ່າເຈົ້າກຳລັງຈະສ້າງຫຍັງ.

Ultimate Multisite нь ямар ч хостинг провайдери дээр ажиллах боломжтой. Зөвхөн зарим хостинг компаниуд (жишээлбэл _WPMU DEV Hosting_ , _WP Engine_ , _Closte_ , _Cloudways_ , _Gridpane_ ба _Runcloud_) домен мапуулдаг талаар илүү нягт интеграцтэй байдаг. Бусад хостинг компаниудтай ажиллах үед та өөрийн хэрэглээний дагуу домен мапуултыг гараар хийх шаардлагатай байж болно. Мөн AutoSSL-ийг дэмжих хостинг провайдерийг сонгох нь сайн санаа юм. _Closte_ , _Runcloud_ ба _cPanel_ үүнийг дэмжинэ.

Сервер спецификацийн талаар хэлэхэд, энэ нь та ямар функц болон үйлчилгээ үзүүлэхийг, мөн таны сүлжээний хэмжээнд хамаарах тул тодорхой томьёо байхгүй. Хэрэв та эхлээд shared hosting ашиглавал сайн эхлэл болно, гэхдээ таны сүлжээ өсөхөд dedicated VPS-д шилжих шаардлагатай болох юм.

## WordPress Multisite {#wordpress-multisite}

Одоо бид WordPress Multisite-ийн талаар бага зэрэг ярилцая.

WordPress Multisite нь нэг талбарт олон вэбсайтын сүлжээг үүсгэх боломжийг олгодог. Энэ нь системд байгаа функц бөгөөд анх стандарт байдаггүй. Та үүнийг өөрийн WordPress-ийн суулгалт болон веб хостинг хяналтын самбарт (control panel) идэвхжүүлэх шаардлагатай.

WordPress-ийг Multisite-аар идэвхжүүлсэн байх нь **Ultimate Multisite-ийн үндэс** юм. Ultimate Multisite-ийн гол зорилго бол хэрэглэгчдэд вэбсайтуудыг үйлчилгээний хувьд удирдах, борлуулах тул энэ нь Multisite идэвхтэй байсан WordPress дээр хийгдэнэ.

Бид таны WordPress-ийн Multisite функцийг идэвхжүүлэх шаардлагатай алхамуудыг алхам алхмаар заасан **гүнзгий нийтлэл** (in-depth article) бэлдсэн байна.

WordPress дээр Multisite-ийг идэвхжүүлсний дараа, Ultimate Multisite-ийг суулгах цаг боллоо. Ultimate Multisite-ийг зөв суулгахын тулд та дараах зүйлсийг агуулсан байх ёстой:

  * WordPress: v5.3+ (Зөвлөмж: Хамгийн сүүлийн тогтвортой хувилбар)

  * PHP: 7.4.x (8.0-ээс дээш дэмжлэг удахгүй ирнэ)

  * MySQL: v5+ (Зөвлөмж: Хэрэв таны хостинг провайдерд 8.0 байхгүй бол 5.6-г сонгоно)

Le kuv cai yog yam xav txhua software requirements rau Ultimate Multisite.

Tsis txhob hlub ti Ultimate Multisite yuav raug tau peev xwb PHP 8.0, tab sis peem tsis txhob ua haujl siab (run) nws neeg PHP 7.4.x.

Thaum ntawd, peem tsis txhob ua haujl siab nws rau **main domains** (domain lwm), tsis txhob ua rau subdomain lossis subdirectory. Nws yuav raug tau peev xwb rau subdomain/subdirectory, tab sis nws muaj khujivuau tam sim noj.

Thaum ntawd, rau kev pabcuam zoo tshaj plaws hauv kev sib sau (install) Ultimate Multisite, **thov tsis txhob hlub ntawm no**.
