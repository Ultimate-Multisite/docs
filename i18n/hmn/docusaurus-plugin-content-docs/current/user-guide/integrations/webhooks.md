# ការ មើល ជាលើកដំបូងអំពី Webhooks (v2)

_**ចំណាំសំខាន់: សូមចំណាំថា មុខងារ ឬអត្ថបទនេះគឺសម្រាប់អ្នកប្រើប្រាស់ដែលមានជំនាញខ្ពស់។**_

**Webhook** គឺជាវិធីមួយដែលកម្មវិធី ឬ software ដូចជា Ultimate Multisite អាចផ្ដល់ព័ត៌មានពេលវេលាជាក់ស្ដែងទៅឱ្យកម្មវិធីផ្សេងទៀតបាន។ Webhook នឹងបញ្ជូនទិន្នន័យ ឬ payload ទៅកាន់កម្មវិធីផ្សេងៗភ្លាមៗ នៅពេលដែលវាកើតឡើង ដែលមានន័យថាអ្នក **ទទួលបានទិន្នន័យភ្លាមៗ**។

នេះមានប្រយោជន៍ខ្លាំងណាស់ ប្រសិនបើអ្នកត្រូវការភ្ជាប់ (integrate) ឬផ្ទេរទិន្នន័យជាក់លាក់ពី Ultimate Multisite ទៅកាន់ CRM ឬប្រព័ន្ធផ្សេងទៀត រាល់ពេលដែលមានព្រឹត្តិការណ៍មួយត្រូវបានបង្កឡើង។ ឧទាហរណ៍ អ្នកត្រូវផ្ញើនាម និងអាសយដ្ឋានអ៊ីមែលរបស់អ្នកទៅបញ្ជីសារ (mailing list) រាល់ពេលដែលគណនីអ្នកប្រើប្រាស់ថ្មីត្រូវបានបង្កើតឡើង។

## របៀបបង្កើត Webhook {#how-to-create-a-webhook}

ដើម្បីបង្កើត webhook សូមចូលទៅកាន់ network admin dashboard របស់អ្នក។ ចុចលើ **Ultimate Multisite > Webhooks > Add New Webhook**។

![Empty Webhooks list page with Add New Webhook button](/img/admin/webhooks-list-empty.png)

បន្ទាប់មក អ្នកអាចកែសម្រួលការកំណត់រចនាសម្ព័ន្ធ webhook បាន៖

![Add New Webhook form with Name, Event, and URL fields](/img/admin/webhook-add-modal.png)

នៅពេលបង្កើត webhook ថ្មី អ្នកនឹងត្រូវបានសួរព័ត៌មានដូចជា **Name (ឈ្មោះ), URL,** និង **Event (ព្រឹត្តិការណ៍)**។ អ្នកអាចប្រើឈ្មោះអ្វីក៏បានសម្រាប់ webhook របស់អ្នក។ កន្លែងសំខាន់បំផុតគឺ URL និង Event។

![Webhook edit interface showing the URL field and payload preview](/img/admin/webhook-url-field.png)

URL គឺជា **endpoint ឬគោលដៅ** ដែល Ultimate Multisite នឹងផ្ញើ **payload ឬទិន្នន័យ** ទៅដល់។ វាគឺជាកម្មវិធីដែលនឹងទទួលបានទិន្នន័យនោះ។

Zapier គឺជាដំណោះស្រាយដែលគេប្រើច្រើនជាងគេដើម្បីធ្វើឱ្យការភ្ជាប់ជាមួយកម្មវិធីភាគីទីបីកាន់តែងាយស្រួល។ បើគ្មាន platform ដូចជា Zapierទេ អ្នកនឹងត្រូវបង្កើត custom function ដោយដៃ ដែលនឹងចាប់យកទិន្នន័យនោះ ហើយដំណើរការវា។ សូមមើលអត្ថបទនេះអំពី **របៀបប្រើ Ultimate Multisite webhook ជាមួយ Zapier**។
