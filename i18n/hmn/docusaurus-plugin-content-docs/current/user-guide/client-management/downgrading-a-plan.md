# Downgrading a plan (v2)

_**NOTE สำคัญ: ບົດຄວາມນີ້ເວົ້າເຖິງ Ultimate Multisite version 2.x.**_

ການປ່ຽນແປງແຜນງານ ຫຼື ການສະໝັກໃຊ້ໃໝ່ ເປັນສິ່ງທີ່ລູກຄ້າຂອງເຈົ້າອາດເຮັດໄດ້ເລື້ອຍໆ ເມື່ອເຂົາເຈົ້າມີງົບປະມານຈຳກັດ ຫຼື ຕັດສິນໃຈວ່າບໍ່ຕ້ອງການຊັບພະຍາກອນຫຼາຍເພື່ອໃຫ້ subsite ຂອງເຂົາເຈົ້າເຮັດວຽກ.

## ວິທີປ່ຽນແປງແຜນງານ (downgrade)

ລູກຄ້າສາມາດປ່ຽນແປງແຜນງານໄດ້ທຸກເວລາ ເມື່ອເຂົ້າສູ່ລະບົບ admin dashboard ຂອງ subsite ຂອງເຂົາເຈົ້າ ແລ້ວຄລິກທີ່ **Change** (ປ່ຽນແປງ) ຢູ່ໃຕ້ໜ້າ account ຂອງເຂົາເຈົ້າ.

![Customer account page with Your Membership card and Change button](/img/account-page/membership-change-button.png)

ຫຼັງຈາກທີ່ຄລິກປຸ່ມ **Change**, ເພື່ອນຳໃຊ້ ຫຼື ລູກຄ້າຈະຖືກສົ່ງໄປໜ້າ checkout ບ່ອນທີ່ເຂົາເຈົ້າສາມາດເລືອກແຜນງານທີ່ຕ້ອງການປ່ຽນການສະໝັກໃຊ້ຂອງຕົນ.

![Downgrade plan options page on the customer side](/img/account-page/downgrade-picker.png)

ໃນຕົວຢ່າງນີ້, ເຮົາກຳລັງປ່ຽນແປງແຜນງານຈາກ **Premium** ໄປເປັນ **Free**.

ເພື່ອດຳເນີນການຕໍ່, ຜູ້ໃຊ້ພຽງແຕ່ຕ້ອງຄລິກປຸ່ມ **Complete Checkout** (ສຳເລັດການຊຳລະເງິນ). ມັນຈະພາເຂົາເຈົ້າກັບໄປທີ່ໜ້າ account ທີ່ສະແດງຂໍ້ຄວາມກ່ຽວກັບການປ່ຽນແປງທີ່ກຳລັງລໍຖ້າສຳລັບ membership. ການປ່ຽນແປງເຫຼົ່ານີ້ຈະມີຜົນໃນ **ຮອບການເກັບຄ່າທຳນຽມ (billing cycle) ຕໍ່ໄປ** ຂອງລູກຄ້າ.

![Account page showing pending membership change banner](/img/account-page/pending-change.png)

### ເຫດການຫຍັງເກີດຂຶ້ນເມື່ອຜູ້ໃຊ້ປ່ຽນແປງແຜນງານ

ເປັນສິ່ງທີ່ຄວນຈື່ໄວ້ວ່າ ການປ່ຽນແປງແຜນງານບໍ່ໄດ້ປ່ຽນແປງການຕັ້ງຄ່າທີ່ມີຢູ່ແລ້ວໃນ subsite ຂອງຜູ້ໃຊ້.

ມັນຈະບໍ່ປ່ຽນ template ຂອງ site ອັດຕະໂນມັດ ເພາະການປ່ຽນ template site ຈະເຮັດໃຫ້ subsite ນັ້ນຖືກລຶບແລະ reset ທັງໝົດ. ເຮັດແບບນີ້ເພື່ອຫຼີກລ່ຽງການສູນເສຍຂໍ້ມູນທີ່ບໍ່ຈຳເປັນ. ສະນັ້ນ, disk space, themes, plugins ແລະ ອື່ນໆ ຈະຍັງຄົງຢູ່ດີ ເວັ້ນເສຍແຕ່ posts (ບົດຄວາມ).

Бид тань манай гол санаа бол таны төлөвлөгөө бүрт тавьсан хязгаар ба хуваарилалт (limits and quotas) байх ёстой гэдгийг ойлгож байгаа ч, бид аль нэг тохиргоог устгах эсвэл өөрчлөх үед тухайн хэрэглэгчийн дэдсайт руу ямар хохирол учруулах болохыг анхаарах ёстой.

Төлөвлөгөөний хязгаарыг давсан нийтлэлүүдийн хувьд танд 3 янзын сонголт байна: **Нийтлэлүүдийг байгаа байдлаар үлдээх** *,* **Нийтлэлүүдийг хаягдалтай болгох (trash)** *,* эсвэл **Нийтлэлүүдийг төлөвлөгөөнд шилжүүлэх (draft)** *.* Та үүнийг Ultimate Multisite-ийн тохиргоо дотор тохируулах боломжтой.

![Network Admin Settings Sites page showing post-limit-exceeded behaviour options](/img/account-page/settings-sites-post-limit.png)

### Төлбөр юу болно

2.0 хувилбарт, төлбөрийн хэсэгт тохируулга хийх шаардлагагүй болсон (proration).

Учир нь систем одоо байгаа гишүүнчлэлийг **түүний төлбөрийн дурын цикл дуусах хүртэл** шинэ төлөвлөгөө/гишүүнчлэл үйлчилж эхлэхгүй. Шинэ гишүүнчлэлийн төлбөр нь дараагийн төлбөрийн циклд автоматаар хэрэглэгдэн, төлөгдөнө.
