# Ultimate Multisite 需求

在购买 Ultimate Multisite 之前，你可能会先问自己：运行我们的插件需要什么条件？幸运的是，运行 Ultimate Multisite 只需要几个基本条件：一个 **域名**、一个好的 **托管服务商** 和一个 WordPress **Multisite** 安装。就这么简单！

## 域名

首先，让我们谈谈域名。

在选择域名时，请记住你的具体细分市场以及你将使用 Ultimate Multisite 创建什么：你想创建一家代理机构？销售类似商店的网站？还是可能是 WordPress 网站托管？Ultimate Multisite 可以实现一切。

我们的插件提供订阅服务、支付收集、结账表单、折扣券和客户沟通。Ultimate Multisite 大大简化了安装、配置和维护 WordPress Multisite 所需的大量工作。

只要你以服务形式销售网站，任何细分市场都可以使用我们的插件进行管理！请确保选择与业务计划相匹配的域名。

## 托管服务商

现在你已经拥有一个能代表你的细分市场和业务的吸引人域名，你需要选择一家优秀的网络托管公司来托管你的网络与 Ultimate Multisite。市场上有一些不错的选择。一些托管服务商与 Ultimate Multisite 集成良好，这可能是决定性因素，取决于你计划构建的内容。

Ultimate Multisite 应该可以在任何托管服务商上运行。唯一的区别是，一些托管公司如 _WPMU DEV Hosting_、_WP Engine_、_Closte_、_Cloudways_、_Gridpane_ 和 _Runcloud_ 对域名映射有更紧密的集成。对于其他托管，你可能需要手动映射自定义域名。获取支持 AutoSSL 的托管服务商也是一个好主意。_Closte_、_Runcloud_ 和 _cPanel_ 支持它。

至于服务器规格，没有固定公式，因为它取决于你将提供的功能和服务以及网络规模。共享主机可能是一个不错的起点，但一旦你的网络开始增长，你可能需要一个专用 VPS。

## WordPress Multisite

现在，让我们稍微谈谈 WordPress Multisite 的安装。

WordPress Multisite 允许你在单一安装中拥有一个网站网络。这是一个内置功能，但默认情况下并未启用。你需要在 WordPress 安装和网络托管控制面板中启用此功能。

在 WordPress 上启用 Multisite **是 Ultimate Multisite 的基础**。由于 Ultimate Multisite 的主要用途是帮助用户管理和以服务形式销售网站，这需要在启用 Multisite 的 WordPress 上完成。

我们有一篇 **深入文章**，逐步教你在 WordPress 安装中启用 Multisite 功能所需的步骤。

在 WordPress 上启用 Multisite 后，就该安装 Ultimate Multisite 了。要正确安装 Ultimate Multisite，你必须满足以下条件：

- WordPress：v5.3+（推荐：最新稳定版本）
- PHP：7.4.x（>8.0 的支持即将到来）
- MySQL：v5+（推荐：5.6，若托管服务商不支持 8.0）

这些是 Ultimate Multisite 唯一的软件要求。

请记住，Ultimate Multisite 可能支持 PHP 8.0，但我们建议使用 PHP 7.4.x。

另外，我们建议将其安装在 **主域名** 上，而不是子域名或子目录。它可能在子域名/子目录上工作，但可能会出现一些错误。
