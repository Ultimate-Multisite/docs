# 发布说明

## Version 2.12.0 — 发布于 2026-05-15

- 新增：支持 Hostinger (hPanel) 作为受支持的主机提供商，并集成了域名映射功能
- 新增：Site Exporter 现在可以处理网络导入包，从而简化网络范围内的站点恢复流程
- 修复：BCC 广播邮件现在使用未公开收件人头 (undisclosed-recipients header)，防止泄露收件人地址
- 修复：当使用非日期值保存时，会员资格过期日期不再损坏
- 修复：Stripe 会员更新现在可以在不调用已弃用的 deleteDiscount API 的情况下，正确清除折扣
- 修复：在域名映射的站点上，SSO 重定向现在有上限，防止无限重定向循环
- 修复：设置向导的图片选择现在能正确更新底层的数据模型
- 修复：Site Exporter CLI 现在能保留正确的默认网络站点选择
- 改进：移除了插件包中捆绑的 wp-cli，减小了插件体积

## Version 2.11.0 — 发布于 2026-05-11

- 新增：站点导出现在会捆绑一个自启动的 `index.php`，这样即使没有单独安装插件，也可以在全新的主机上安装 ZIP 文件。
- 新增：网络导出允许管理员从 Site Export 管理页面将所有子站点打包到一个存档中。
- 新增：现在通过备用链强制执行“允许 Site Templates”计划开关，正确限制了模板的可用性以符合计划限制。
- 新增：结账表单编辑器会在添加了未配置必填字段的产品时发出警告。
- 新增：导入/导出设置标签页现在清晰地描述了其范围，并直接链接到 Site Export 工具。

## Version 2.10.0 — 发布于 2026-05-05

- 新增：为手动凭证输入提供了 PayPal 指导设置向导，并带有 OAuth 标志门控，实现无缝网关配置。
- 新增：模板切换客户面板进行了重新设计，增加了当前模板卡片、持久化网格和 **重置当前模板** 按钮。
- 修复：模板切换不再在 AJAX 失败时导致用户界面卡死。
- 修复：模板切换的权限状态已加固，防止未经授权的访问。
- 修复：站点覆盖输入在保存前进行了验证。
- 修复：现在在地址为空时会显示账单地址提示。
- 修复：解决了 PHP 8.1 null 到字符串的弃用通知问题。
- 修复：Currents 在 init 钩子之前延迟加载，以防止时序问题。
- 修复：跨所有登录流程都尊重过滤后的 SSO 路径。
- 修复：空白站点身份选项在保存时得以保留。

## Version 2.9.0 — 发布于 2026-04-30

- 新增：在 **工具 > 导出和导入** 下增加了单站点导出和导入功能。
- 修复：导出 ZIP 文件现在通过经过身份验证的下载端点提供服务。
- 修复：修正了待处理导出/导入查询中的 SQL 注入风险和查询问题。
- 修复：当管理员手动验证客户电子邮件时，待处理站点不会被发布。
- 修复：当缺少会员资格时，清理孤立的 pending_site 记录。
- 修复：修正了设置导航的内边距和搜索锚点导航问题。
- 修复：待处理站点现在在“所有站点”视图中显示为第一项。
- 修复：为截图提供商 (mShots) 添加了 User-Agent 头，以防止 403 错误。
- 修复：解决了导入定时任务的循环依赖问题。
- 修复：用户设置键中的 Tour ID 已标准化为下划线。
- 改进：现在改用 ZipArchive 代替 Alchemy/Zippy，以获得更好的兼容性。

## Version 2.8.0 — 发布于 2026-04-29

- 新增：在其他选项设置 UI 中增加了启用 Jumper 开关。
- 新增：在结账表单列表表中增加了状态列。
- 新增：为自定义 MU-plugin sunrise 扩展添加了 Addon sunrise 文件加载器。
- 改进：从设置页面移除了错误报告 opt-in 设置。
- 修复：感谢页面站点卡片——图片现在受到限制，并且链接样式正确。
- 修复：截图提供商已从 thum.io 切换到 WordPress.com mShots。
- 修复：启用注册和默认角色现在在全新安装时设置了正确的默认值。
- 修复：`get_site_url()` 不再在域名包含端口时返回空值。
- 修复：当 `copy_media` 设置为空时，克隆媒体文件现在能正确复制。
- 修复：对象缓存对象在网络激活 sitemeta 写入后能正确失效。
- 修复：对于三段式域名，自定义域名现在在 DNS 验证后自动提升为主要域名。
- 修复：当清理过期付款时，待处理会员资格会被取消。
- 修复：密码强度检查器在内联登录提示被关闭后能重新绑定。
- 修复：当站点已创建时，感谢页面不会无限刷新。
- 修复：WP core 注册选项在插件激活和设置保存时同步。
- 修复：在 `calculate_expiration` 中增加了 null 过期保护，以兼容 PHP 8.4。
- 修复：当客户已拥有活跃会员资格时，阻止重复注册。
- 修复：在结账流程中为 `date_expiration` 添加了空值检查。
- 修复：站点配置流程加强——限制、会员资格推断、域名提升。
- 修复：预安装检查状态标签在检查失败时修正为 NOT Activated。
- 修复：结账域名用于电子邮件验证 URL。
- 修复：当没有密码字段时，结账后自动登录。
- 修复：免费会员资格不再过期——被视为终身会员。
- 修复：电子邮件验证门控会阻止站点发布，直到客户验证电子邮件。
- 修复：SES v2 API 端点基础路径和身份路由已修正。
- 修复：`wu_inline_login_error` 钩子在 pre-submit 捕获块中发出。
