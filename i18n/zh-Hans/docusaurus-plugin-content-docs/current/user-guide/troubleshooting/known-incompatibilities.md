# 与其他插件的已知不兼容性

安装并激活 Ultimate Multisite 后，您可能会遇到由于与其他插件存在未解决的不兼容性而导致的问题。

我们正积极努力在每一次新版本发布时减少此列表，但如果插件在此列出，您可能需要停用它，以获得完全功能的 Ultimate Multisite 网络。

以下是它们的列表以及您在这些情况下应采取的措施：

**一些 WPMU DEV 插件** WPMU DEV 维护的插件中，有些 _不再_ 维护的插件已知会导致 Ultimate Multisite 注册流程在最后一步停止（用户无法创建账户）。以下是导致这些问题的插件列表：

  * Signup Language: _Ultimate Multisite 有一个附加组件提供相同的功能，名为 Ultimate Multisite: Language Switcher._
  * Set Blog Description on Blog Creation: _这已内置于 Ultimate Multisite，无需额外插件._
  * New Blog Templates: _这已内置于 Ultimate Multisite，无需额外插件._
  * Set Password on Multisite Blog Creation: _这已内置于 Ultimate Multisite，无需额外插件._
  * Signup TOS: _这已内置于 Ultimate Multisite，无需额外插件._
  * Anti-Splog: Although less powerful, this is also _已内置于 Ultimate Multisite._

**Note** : 所有其他 WPMU Dev 插件可以正常与 Ultimate Multisite 一起使用。示例包括 _Smush_ , _Forminator_ , _Defender,_ 等。
