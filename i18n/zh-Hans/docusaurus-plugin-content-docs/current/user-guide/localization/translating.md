# 翻译 Ultimate Multisite

Ultimate Multisite 是用英文开发的，但插件完全可翻译，我们积极欢迎翻译贡献。

## 核心插件翻译

核心 Ultimate Multisite 插件通过官方 **WordPress.org 翻译平台** 进行翻译：

**[translate.wordpress.org/projects/wp-plugins/ultimate-multisite/](https://translate.wordpress.org/projects/wp-plugins/ultimate-multisite/)**

这是标准的 WordPress 社区翻译系统。当翻译达到足够的批准阈值时，它们会通过 WordPress 插件更新自动分发给所有 Ultimate Multisite 用户。

**我们需要你的帮助！** 如果你说的不是英语，请考虑贡献翻译。即使只翻译少量字符串也有帮助。开始步骤如下：

1. 访问 [translate.wordpress.org](https://translate.wordpress.org/projects/wp-plugins/ultimate-multisite/)
2. 选择你的语言
3. 点击 `"Contribute Translation"`
4. 你需要一个免费的 WordPress.org 账号

如果你想成为你所在语言的翻译编辑（能够批准翻译），请通过 [GitHub Discussions](https://github.com/Ultimate-Multisite/ultimate-multisite/discussions) 联系我们。

## 附加插件翻译

附加插件在以下位置单独翻译：

**[translate.ultimatemultisite.com](https://translate.ultimatemultisite.com/)**

附加插件翻译目前是 AI 生成的起点。它们涵盖所有支持的语言，但可能包含生硬的措辞或不准确之处。我们欢迎人工译者审阅并建议改进，使其更自然、更准确。

## 本地翻译

如果你更愿意在本地为自己的项目创建翻译，而不是向上游贡献，有两种选择：

### 使用 Poedit

你可以使用桌面应用 **[Poedit](https://poedit.net)** 来翻译 `.pot` 文件：

1. 从 [poedit.net](https://poedit.net) 下载 Poedit
2. 打开 `wp-content/plugins/ultimate-multisite/lang/` 中的 `.pot` 文件
3. 在 Poedit 界面中翻译字符串
4. 保存 -- Poedit 会生成 `.po` 和 `.mo` 文件
5. 将输出文件放入 `lang` 文件夹

### 使用 Loco Translate

**[Loco Translate](https://wordpress.org/plugins/loco-translate/)** WordPress 插件让你可以直接在管理面板中翻译，无需任何桌面工具。它会为你创建并编译翻译文件。
