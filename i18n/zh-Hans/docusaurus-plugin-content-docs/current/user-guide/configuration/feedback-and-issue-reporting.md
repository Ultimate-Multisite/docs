# 用户反馈与问题报告

Gratis AI Agent v1.5.0 引入了内置的反馈和问题报告系统，允许最终用户在聊天界面直接标记无用的回复或报告问题。该系统包括同意管理、自动报告命令和后端 AI 辅助分诊。

## 👎 差评按钮

AI 助手发送的每条消息都会显示一个 **差评** (👎) 按钮。用户点击它，可以将回复标记为无用或不正确。

- 该按钮会在鼠标悬停在每个助手消息旁边时显示。
- 点击它会弹出 **反馈同意模态框**。
- 反馈与整个对话线程和特定的消息相关联。

## 反馈同意模态框

当用户点击差评按钮时，在发送任何数据之前会弹出一个同意模态框。该模态框：

- 解释将收集哪些信息（对话摘录、浏览器上下文）。
- 要求用户确认同意分享这些数据。
- 提供一个自由文本字段，供用户描述哪里出了问题。
- 提供一个 **取消** 选项，用于不提交而关闭模态框。

在用户明确确认之前，不会记录任何反馈。

## 自动提示反馈横幅

在对话结束时，助手可能会显示一个 **自动提示反馈横幅**——这是一个不具侵入性的消息，询问用户本次会话是否有帮助。

该横幅根据对话长度和结果启发式规则自动显示。它链接到与差评按钮相同的反馈流程。用户可以在不提供反馈的情况下关闭该横幅。

## /report-issue 斜杠命令

用户可以通过在聊天输入框中输入 `/report-issue` 来直接触发反馈流程。此命令：

- 立即打开反馈同意模态框。
- 使用当前对话的上下文预填充描述字段。
- 允许用户在提交前添加更多细节。

`/report-issue` 命令在所有聊天模式（行内、浮动小部件、全屏）中均可用。

## AI 辅助分诊

提交的反馈会路由到一个 AI 分诊层，该层会：

- 对报告进行分类（事实错误、无用答案、不当内容等）。
- 附加相关的对话元数据。
- 将分诊摘要转发到配置的 **反馈端点**（参见 [设置 > 高级](#settings-advanced)）。

这通过优先展示最关键的问题，减少了人工审核时间。

## 设置 > 高级 {#settings-advanced}

要启用反馈后端，请在 **Gratis AI Agent → 设置 → 高级** 下配置以下字段：

| 字段 | 描述 |
|---|---|
| **Feedback Endpoint URL** | 接收带有反馈负载（JSON）POST 请求的 URL。 |
| **Feedback API Key** | 在 `Authorization` 头部发送的 Bearer token，用于认证反馈提交。 |

如果需要禁用反馈收集，请将两个字段留空。单个反馈按钮仍对用户可见，但提交的反馈将不会被转发。

:::tip
反馈端点必须接受一个包含至少 `message_id`、`conversation_id`、`feedback_text` 和 `triage_category` 字段的 JSON body。请参阅您的端点提供商的文档以了解预期的模式。
:::
