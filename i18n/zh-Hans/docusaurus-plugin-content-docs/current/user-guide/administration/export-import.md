# 导出与导入

Ultimate Multisite 2.9.0 在 **工具 > 导出与导入** 下增加了一个单站点**导出与导入**工具。当您需要将一个 WordPress 站点打包成 ZIP 文件、恢复该 ZIP 文件，或在兼容的 Ultimate Multisite 和单站点 WordPress 安装之间迁移站点时，请使用此工具。

## 所需权限

您必须以管理员身份登录，并且该管理员对正在导出或导入的站点拥有访问 WordPress **工具**菜单的权限。在多站点网络中，当从网络级别的 Ultimate Multisite 工具导出或导入子站点时，请使用网络管理员账户。

导出 ZIP 文件是通过经过身份验证的下载端点提供的，因此请保持管理员会话处于活动状态，直到下载完成，并且不要公开分享生成的下载 URL。

## 将站点导出为 ZIP

1. 在您想要复制的站点的 WordPress 管理后台，前往 **工具 > 导出与导入**。
2. 打开导出区域，并选择您想要打包的站点。
3. 如果有可用选项，请选择包含的可选内容，例如上传文件、插件和主题。
4. 启动导出，并等待过程完成。大型站点可能需要在后台完成处理，然后 ZIP 文件才会准备就绪。
5. 从导出列表中下载完成的 ZIP 文件。

请将 ZIP 文件保存在安全的位置。它可能包含站点内容、设置、媒体文件和选定的代码资源。

## 导出包含的内容

导出的 ZIP 文件可以包含：

- 所选站点的数据库内容和配置。
- 如果包含上传文件，则包含的上传媒体文件。
- 如果选择了相应的选项，则包含的插件和主题。
- 导出与导入工具用于在目标安装上重建站点所需的导入元数据。

确切的 ZIP 大小取决于媒体文件的数量、选择的插件和主题，以及站点数据库表的大小。

## 从 ZIP 导入站点

1. 在目标 WordPress 站点上，前往 **工具 > 导出与导入**。
2. 打开导入区域，并上传由导出与导入工具创建的 ZIP 文件。
3. 如果站点需要使用新的地址，请输入替换 URL；如果想保留原始 URL，请将此字段留空。
4. 选择导入完成后是否删除上传的 ZIP 文件。
5. 点击 **开始导入**。
6. 监控待处理的导入过程，直到它完成。只有当您需要在完成前停止流程时，才使用 **取消导入**。
7. 在允许正常流量或客户访问之前，请仔细检查导入后的站点。

在单站点 WordPress 安装中，导入 ZIP 文件会用导入的站点替换当前的站点。在开始之前，请为目标站点创建完整的备份，并避免同时为同一站点启动多次导入。

## 限制和兼容性注意事项

- 非常大的上传目录或媒体库可能会生成很大的 ZIP 文件。在导出或导入大型站点之前，请确认 PHP 上传限制、执行限制、磁盘空间、内存和服务器超时设置。
- 非常大的媒体库可能需要在流量较低的维护窗口期间移动。
- 目标 WordPress 安装应运行兼容的 WordPress、PHP、Ultimate Multisite、插件和主题版本。
- 单站点导入会替换目标站点。它不是一个合并工具。
- 仅当目标环境能够运行导出站点所需的插件、主题、URL 和必要的 Ultimate Multisite 组件时，才支持多站点到单站点和单站点到多站点的迁移。
- 请将 ZIP 文件保密。它可能包含来自导出站点的数据库内容、上传文件和配置详细信息。

有关较旧的网络级别导出工作流程，请参阅 [Site Export](/user-guide/administration/site-export)。
