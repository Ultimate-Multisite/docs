# 升级方案 (v2)

_**重要说明：本文指的是 Ultimate Multisite 2.x 版本。**_

您的客户可以随时升级他们的方案。他们可以升级到另一个方案，或购买您在网络上提供的任何附加服务或套餐。

在本教程中，我们将介绍他们如何升级方案，以及升级流程之后会发生什么。

要升级方案，您的客户应访问他们的 Dashboard，并前往 **Account** 页面。

![客户子站点 Dashboard，其中 Account 菜单链接可见](/img/account-page/account-menu.png)

在 Account 页面上，他们会看到当前的会员资格以及与之关联的方案。要升级到另一个方案，他们必须点击 **您的会员资格** 部分右上角的 **更改**。

![Account 页面上的“您的会员资格”卡片，其中有“更改”按钮](/img/account-page/membership-change-button.png)

他们将被重定向到一个结账表单，其中会显示所有可用方案。

他们还可以看到 **适用于当前方案的服务和套餐**，以防他们只是想购买特定服务或套餐（例如我们这里示例中的无限访问量或磁盘空间），而不是升级方案。

![升级选择器，在客户侧显示可用方案和套餐](/img/account-page/upgrade-picker.png)

在他们选择想要购买的产品后，他们将看到现在需要支付的金额（不包括任何现有余额），以及下一个账单日期将被收取的金额。

通常，如果该产品是另一个方案，并且付款将在一次会员资格扣费之间进行，他们会收到第一项方案已支付金额的抵扣额度。

![升级付款摘要，显示已应用的抵扣额度和下一次账单金额](/img/account-page/upgrade-summary.png)

如果他们选择的方案或套餐不会对当前订阅产生任何变化，他们会看到一条解释说明。

![所选方案不会更改订阅时的通知](/img/account-page/upgrade-no-change.png)

结账完成后，新产品将被添加到您客户的账户中，并且新产品的所有限制或功能都会立即添加到其中：访问量、磁盘空间、文章等……

## 升级和降级路径

在您的每个产品上，都会有一个 **升级与降级** 选项卡。该选项卡上的第一个选项是一个名为 **方案组** 的字段。

**方案组** 允许您让 Ultimate Multisite 知道某些方案属于同一个“系列”，因此应当用于构建升级/降级路径选项。

![产品编辑页面的“升级与降级”选项卡，其中有“方案组”字段](/img/config/product-upgrades-plan-group.png)

例如，您有一个 **免费方案**、一个 **基础方案** 和一个 **高级方案** 可用。您希望订阅 **免费方案** 的用户只能升级到 **高级方案**，并且不希望他们看到“基础方案”作为升级选项。您只需要为免费方案和高级方案分配相同的方案组名称，如下面的截图所示。

![免费方案产品页面，已分配高端方案组](/img/config/product-upgrades-free.png)

![高级方案产品页面，已分配高端方案组](/img/config/product-upgrades-premium.png)

这应该会告诉 Ultimate Multisite，网络中有一个名为 **高端** 的方案“系列”。在提供升级或降级时，只有来自同一系列的方案才会作为选项呈现给用户。
