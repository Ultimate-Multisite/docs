# 降级套餐（v2）

_**重要说明：本文适用于 Ultimate Multisite 2.x 版本。**_

降级套餐或订阅是你的客户可能会执行的常见操作，例如他们预算有限，或者他们认为运行其子站点不需要太多资源。

## 如何降级套餐

你的客户可以随时登录其子站点管理员 Dashboard，并在其 Account 页面下点击 **更改** 来降级套餐。

![带有“你的会员资格”卡片和“更改”按钮的客户 Account 页面](/img/account-page/membership-change-button.png)

点击 **更改** 按钮后，用户/客户将被重定向到结账页面，在那里他们可以选择要将订阅更改为的套餐。

![客户侧的降级套餐选项页面](/img/account-page/downgrade-picker.png)

在此示例中，我们将套餐从 **高级** 降级到 **免费**。

要继续，用户只需点击 **完成结账** 按钮。随后系统会将他们带回 Account 页面，并显示一条关于会员资格待处理更改的消息。这些更改将在客户的**下一个计费周期**生效。

![显示待处理会员资格更改横幅的 Account 页面](/img/account-page/pending-change.png)

### 用户降级套餐时会发生什么

需要注意的是，降级套餐不会更改用户子站点中的现有配置。

它不会自动更改站点模板，因为更改站点模板会完全清除并重置子站点。这样做是为了避免不必要的数据丢失。因此，除文章外，磁盘空间、主题、插件等都将保持不变。

我们理解你主要关心的是你在每个套餐下设置的限制和配额，但我们必须考虑如果删除或更改用户子站点的任何配置会对其造成的损害。

对于超过套餐设定限制的文章，你有 3 个不同选项：**按原样保留文章** *，* **将文章移至回收站** *，* 或 **将文章移至草稿** *。* 你可以在 Ultimate Multisite 设置中配置此项。

![Network Admin 设置的站点页面，显示超出文章限制时的行为选项](/img/account-page/settings-sites-post-limit.png)

### 付款会发生什么

在 2.0 版本中，不再需要对付款进行任何按比例分摊方面的调整。

这是因为系统会等待现有会员资格**完成其计费周期后**，新套餐/会员资格才会生效。新会员资格的新计费金额将自动应用，并在下一个计费周期收取。
