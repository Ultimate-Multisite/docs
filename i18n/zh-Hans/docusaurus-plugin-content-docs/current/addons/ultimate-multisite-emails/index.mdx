# 电子邮件

Ultimate Multisite: Emails 插件让网络所有者可以在站点和会员资格旁边预配并管理客户电子邮件账户。版本 1.1.0 增加了面向客户的电子邮件账户管理、网页邮箱单点登录、CyberPanel 转发器支持，以及结账时预配改进。

## 主要功能

* **客户电子邮件账户小组件**：客户在拥有有效的电子邮件会员资格后，可以从自己的账户页面创建和管理邮箱。
* **网页邮箱 SSO 和魔法链接**：当提供商支持时，客户可以通过一键认证直接从账户页面打开网页邮箱。
* **密码更改**：客户和管理员可以从电子邮件管理小组件更新邮箱密码。
* **结账权益预配**：结账会创建电子邮件会员资格，将其链接到客户的站点域名，并使账户页面邮箱工具可用。
* **CyberPanel 转发器**：由 CyberPanel 支持的网络可以将创建和管理电子邮件转发器作为电子邮件服务流程的一部分。
* **感知提供商的设置**：设置页面会将提供商筛选为已启用且已配置的集成，因此管理员只会看到可用选项。

## 要求

* Ultimate Multisite 2.0.0 或更高版本
* WordPress Multisite 安装
* 至少一个已配置的电子邮件提供商集成
* 具有创建邮箱、更改密码，以及在支持时签发网页邮箱访问链接权限的提供商 API 凭据

## 安装

1. 将插件文件上传到 `/wp-content/plugins/ultimate-multisite-emails`。
2. 从 WordPress 网络管理后台的 **插件** 中在网络范围启用该插件。
3. 前往 **Ultimate Multisite** > **设置**，并打开插件电子邮件设置。
4. 启用你希望客户使用的电子邮件提供商集成。
5. 保存设置，然后使用非生产客户或预发布站点测试账户创建。

## 电子邮件销售流程的工作方式

电子邮件销售有两个独立层级：

1. **权益** — 一个 Ultimate Multisite 电子邮件产品/会员资格，表示客户被允许为特定站点域名、提供商、配额和账户限制使用电子邮件。
2. **邮箱创建** — 实际的提供商邮箱，例如 `info@example.com`，在客户或网络管理员选择本地部分和密码时创建。

结账期间，该插件会创建或启用**电子邮件会员资格**，并在该会员资格上记录客户的站点域名和提供商。它不会猜测邮箱名称或密码。购买完成且站点/域名上下文存在后，客户可以打开账户页面并创建他们想要的邮箱名称。

这意味着正常的客户流程是：

1. 客户购买站点套餐，并在结账时添加电子邮件产品。
2. Ultimate Multisite 创建站点会员资格和次级电子邮件会员资格。
3. Emails 插件将电子邮件会员资格链接到客户的站点域名和提供商。
4. 客户在其 Dashboard 中打开 **Account**。
5. **电子邮件账户**面板会针对该有效的电子邮件会员资格显示。
6. 客户点击 **创建账户**，输入本地部分（例如 `info`），设置密码，然后该插件会在已配置的提供商处创建 `info@their-domain.com`。

## 配置电子邮件产品

为你想要销售或赠送的每个电子邮件套餐创建一个电子邮件产品。如果你刚开始进行产品设置，请从[创建你的第一个订阅产品](../../user-guide/configuration/creating-your-first-subscription-product)开始，然后返回这里进行电子邮件专用设置。

1. 前往 **网络管理 > Ultimate Multisite > 产品 > 添加产品**。
2. 将 **产品类型** 设置为 **电子邮件**。
3. 配置定价。对于销售的套餐，使用付费循环价格；如果它应作为赠送的电子邮件套餐，则将产品设置为免费/零价格。
4. 打开 **电子邮件设置**部分。
5. 选择已配置的 **电子邮件提供商**。
6. 根据需要设置 **账户数量**、**存储配额（MB）**、别名限制、转发器和自动回复器。
7. 保存产品。

只有在 **集成**页面上已启用且已完整配置的提供商才会出现在提供商选择器中。如果选择器为空，请先配置并测试提供商。

## 将电子邮件添加到结账

电子邮件产品通常作为站点套餐的附加项提供。有关通用结账编辑器工作流程，请参阅[结账表单](../../user-guide/configuration/checkout-forms)。

1. 创建或编辑客户首先购买的站点套餐。
2. 将电子邮件产品添加为该结账流程的可用附加项/订单加购。
3. 确认结账表单仍包含正常必填的站点字段，尤其是站点 URL/域名字段。
4. 使用预发布客户运行一次测试结账。
5. 结账后，打开客户的[账户页面](../../user-guide/client-management/account-page)，并确认**电子邮件账户**面板出现。

电子邮件域名取自客户的站点域名。客户不会在电子邮件产品步骤中输入电子邮件域名。

## 客户账户电子邮件管理

版本 1.1.0 在客户账户页面添加了电子邮件管理小组件。当客户应自助为自己的站点创建和维护邮箱时，请使用它。

在账户页面中，客户可以：

1. 查看连接到其当前电子邮件会员资格/站点域名的电子邮件账户。
2. 当其电子邮件产品允许额外账户时，创建新邮箱。
3. 更改现有邮箱密码。
4. 通过 SSO 或魔法链接登录打开受支持的网页邮箱提供商。

该小组件使用 Ultimate Multisite 账户页面当前项系统，因此操作会限定在客户正在查看的账户上下文中。这可以防止客户管理属于其他会员资格或站点的电子邮件账户。

如果客户看不到电子邮件面板，请检查他们是否拥有**有效的电子邮件产品会员资格**，该会员资格是否已链接到站点域名，并且该产品是否已配置提供商。

## 给客户免费电子邮件方案

要免费向客户提供电子邮件，请先授予**电子邮件权益**，然后创建邮箱，或让客户自行创建邮箱。

推荐选项：

* **创建免费电子邮件产品**：将电子邮件产品价格设为免费/零，然后从 **Ultimate Multisite > 会员资格 > 添加会员资格** 为该客户添加一个有效会员资格。选择电子邮件产品和客户。免费会员资格在更改或取消之前为终身有效。如果该电子邮件会员资格需要跟随现有站点，请确保它已链接到该站点/域名上下文。
* **使用 100% 折扣或免费结账链接**：让客户完成结账，这样 Ultimate Multisite 会自动创建电子邮件会员资格和付款记录。
* **管理员创建邮箱**：前往 **Ultimate Multisite > 电子邮件账户 > 添加电子邮件账户**，为客户在提供商处创建特定邮箱。这对支持很有用，但它不能替代用于驱动客户自助 Account 页面工具的电子邮件产品会员资格。

如果你希望客户从 Account 页面自行管理邮箱，请确保他们拥有该域名的有效电子邮件会员资格。没有该会员资格的手动创建的提供商邮箱可以存在，但客户可能看不到它的自助电子邮件面板。

## 管理员创建的账户和“没有订单号”

**电子邮件账户**管理页面用于创建或编辑提供商邮箱。它不是销售或授予电子邮件方案的第一步。

编辑电子邮件账户时，**会员资格**字段会按引用号/订单号搜索现有 Ultimate Multisite 会员资格。如果你在客户拥有电子邮件会员资格之前尝试分配账户，可能没有可选择的订单号。

请改用以下操作顺序：

1. 创建或找到客户的电子邮件会员资格。对于购买，结账会完成此操作。对于赠送方案，请从[会员资格](../../user-guide/administration/managing-memberships)手动创建会员资格，或让客户通过免费结账。
2. 确认会员资格处于有效状态，并使用电子邮件产品。
3. 确认电子邮件会员资格已链接到站点域名/提供商。
4. 从客户 Account 页面或从 **Ultimate Multisite > 电子邮件账户** 创建邮箱。
5. 如有需要，编辑邮箱记录并分配客户、会员资格和站点。

如果你只需要为客户创建提供商邮箱，并且不需要 Account 页面自助管理，那么管理员创建的账户可能已经足够。如果你希望客户在 Ultimate Multisite 中看到并管理它，也请创建电子邮件会员资格。

## Webmail SSO 和 Magic-Link 登录

Webmail SSO 让客户可以从 Account 页面直接执行**打开 Webmail**操作。当有效提供商支持时，该 addon 会请求一个短期有效的访问 URL，并将客户重定向到 webmail，而无需让他们复制邮箱凭据。

提供商行为可能有所不同：

* 具有原生 SSO API 的提供商可以创建直接认证的 webmail 会话。
* 支持 magic-link 的提供商可以生成有时限的登录链接。
* 不支持 SSO 或 magic-link 的提供商仍应显示邮箱详情和密码管理操作，但不会显示一键 webmail 按钮。

仅为已配置所需 API 凭据的提供商启用 webmail 访问。如果某个提供商缺少该操作，请先确认该提供商处于有效状态、已配置，并且支持 webmail 登录链接。

## 结账预配

该 addon 会在结账期间预配电子邮件**会员资格上下文**。它会记录提供商、账户限制、配额和站点域名，以便客户在购买后可以从 Account 页面创建自己的邮箱名称。

推荐发布流程：

1. 在提供电子邮件产品之前，配置提供商和产品限制。
2. 使用预发布产品测试购买，并确认已创建电子邮件会员资格。
3. 查看客户 Account 页面，确认该域名显示**电子邮件账户**小工具。
4. 从该小工具创建测试邮箱。
5. 在为生产产品启用该流程之前，确认密码更改和 webmail 操作。

如果预配失败，客户 Account 和会员资格仍可能存在，但电子邮件会员资格或提供商邮箱需要管理员跟进。重试前，请检查 Ultimate Multisite 活动日志和提供商日志。

## CyberPanel 电子邮件转发器

对于由 CyberPanel 支持的网络，版本 1.1.0 添加了电子邮件转发器支持。当客户需要将 `info@customer-domain.test` 等地址重定向到另一个收件箱，而不创建完整邮箱时，请使用转发器。

在提供转发器之前，请确认 CyberPanel 集成已配置且可访问。有关所需的主机、用户名、密码和包常量，请参阅 [CyberPanel 集成](../../user-guide/administration/cyberpanel-integration)指南。

## 常见问题

### 客户可以创建自己的电子邮件账户吗？

可以。当客户拥有某个域名的有效电子邮件会员资格，并且产品允许额外账户时，客户可以从 Account 页面创建和管理自己的邮箱。

### 每个提供商都支持 webmail SSO 吗？

不支持。SSO 和 magic-link 支持取决于启用的电子邮件提供商。没有访问链接 API 的提供商仍可支持邮箱创建和密码更改。

### 管理员可以为客户创建电子邮件账户吗？

可以。管理员可以从管理界面创建电子邮件账户，包括在创建账户时设置初始密码。对于客户自助管理，还请确保客户拥有该域名的有效电子邮件会员资格。

### 我可以免费赠送一个电子邮件账户吗？

可以。创建一个免费/零价格的电子邮件产品，并将其作为客户的有效会员资格添加，或让客户使用 100% 折扣完成结账。电子邮件会员资格存在后，客户或管理员即可创建实际邮箱。

### 为什么分配电子邮件账户时会要求提供订单号？

Membership 字段会通过其引用/订单号查找现有的 Ultimate Multisite 会员资格。如果客户没有购买电子邮件产品，并且你没有手动创建电子邮件会员资格，就没有可分配的电子邮件会员资格/订单。请先创建电子邮件会员资格，然后再创建或分配邮箱。

### 转发器和邮箱一样吗？

不一样。邮箱会存储邮件并具有登录凭据。转发器会将邮件重定向到另一个地址，并且不提供单独的收件箱。
