# 工具和配置

Theme Builder 使用一组工具来分析、设计并构建你的 WordPress 站点。在 1.16.0 版本中，**sd-ai-agent/site-scrape 现在是 Tier 1 工具**，因此默认可在 Theme Builder 中使用。Superdav AI Agent v1.18.0 还引入了一个单独分发的 Advanced 配套包，用于受信任的开发者工作流；这些工具不属于 WordPress.org 核心包，必须单独安装并授权。

## 可用工具

### Tier 1 工具（始终可用）

Tier 1 工具默认可在 Theme Builder 中使用，无需任何额外配置。

#### sd-ai-agent/site-scrape

**用途：** 分析现有网站，以提取设计灵感、内容结构和功能。

**能力：**

- **网站分析** — 抓取并分析竞争对手或灵感网站
- **设计提取** — 识别颜色、字体和布局模式
- **内容结构** — 理解页面组织和层级
- **功能检测** — 识别 plugin 和功能
- **性能分析** — 检查页面速度和优化情况
- **SEO 分析** — 审查元标签和结构化数据

**用法：**

```
分析 example.com 的设计，并将其作为我站点的灵感。
```

**它会提取的内容：**

- 调色板和颜色使用方式
- 排版（字体和大小）
- 布局结构和间距
- 导航模式
- 内容组织
- 图片和媒体使用方式
- 交互元素
- 移动端响应能力

**限制：**

- 无法抓取受密码保护的站点
- 遵守 robots.txt 和站点策略
- 可能无法捕获动态内容
- 大量依赖 JavaScript 的站点可能只能进行有限分析
- 大型站点可能需要更长时间分析

### Tier 2 工具（可选）

Tier 2 工具在 Theme Builder 设置中显式启用后可用。

#### 高级分析

提供详细的性能指标：

- 页面加载时间
- Core Web Vitals
- SEO 分数
- 可访问性分数
- 最佳实践分数

#### 内容优化器

分析并建议以下方面的改进：

- 可读性
- SEO 优化
- 关键词使用
- 内容结构
- 行动号召位置

### Tier 3 工具（高级）

Tier 3 工具需要额外配置或高级功能。

#### AI 内容生成器

为以下内容生成文案：

- product 描述
- 服务页面
- 博客文章
- 元描述
- 行动号召文案

#### 高级设计工具

提供高级设计能力：

- 自定义 CSS 生成
- 动画创建
- 交互元素设计
- 高级色彩理论
- 排版优化

### Advanced 配套工具（单独包）

Advanced 配套包与核心 Superdav AI Agent 版本分开分发。它适用于受信任的开发者和站点所有者环境，在这些环境中，管理员明确允许 agent 使用高影响力工具。

Advanced 配套能力可包括：

- **文件系统工具** — 在受信任环境中检查和修改已批准的文件
- **数据库工具** — 在明确授权时查询或更新站点数据
- **WP-CLI 工具** — 运行 WordPress 维护和检查命令
- **REST 调度器工具** — 通过受控调度工作流调用已注册的 REST endpoint
- **Plugin 构建器工具** — 搭建并迭代 plugin 代码
- **Git 快照工具** — 在有风险的开发者操作之前创建快照
- **用户管理工具** — 在允许的情况下帮助审查或调整用户记录
- **基准测试工具** — 运行性能或能力基准测试以供开发审查

不要将这些记录为每次安装都普遍可用的 Theme Builder 工具。它们仅在 Advanced 配套包已安装、已启用并限制给受信任管理员时可用。

## 工具配置

### 启用工具

要在 Theme Builder 中启用额外工具：

1. 导航到 **Gratis AI Agent → Settings**
2. 前往 **Theme Builder → Tools**
3. 按需开启/关闭工具
4. 保存设置

### 工具权限

某些工具需要权限：

- **站点抓取** — 需要互联网访问
- **分析** — 需要 Google Analytics 集成
- **内容生成** — 需要 API 密钥
- **高级功能** — 可能需要高级订阅
- **Advanced 配套工具** — 需要单独分发的 Advanced 包以及明确的管理员信任

### API 密钥和凭据

为需要它们的工具配置 API 密钥：

1. 前往 **Gratis AI Agent → Settings → API Keys**
2. 输入每个工具的凭据
3. 测试连接
4. 安全保存

### 安装 Advanced 配套包

Advanced 配套包与 WordPress.org 核心 ZIP 分开发布。仅从该版本的官方项目分发渠道安装它，然后将访问权限限制给受信任的管理员。在生产站点上启用文件系统、数据库、WP-CLI、REST 调度器、plugin 构建器、用户管理或基准测试工具之前，请审查人工批准关卡和 git 快照行为。

## 使用 sd-ai-agent/site-scrape

### 基本用法

请求 Theme Builder 分析一个网站：

```
分析我竞争对手在 competitor.com 的站点设计，
并为我的站点建议设计改进。
```

### 特定分析

请求特定类型的分析：

```
从 example.com 提取调色板，并将其作为灵感使用。
```

```
分析 example.com 的导航结构，并将类似的
组织方式应用到我的站点。
```

```
检查 example.com 的移动端响应能力，并确保我的站点
同样具备响应能力。
```

### 对比分析

比较多个站点：

```
比较 site1.com 和 site2.com 的设计，并创建一个
结合两者最佳元素的设计。
```

## 工具限制和注意事项

### 速率限制

- 抓取会受到速率限制，以防止服务器过载
- 对同一站点的多次请求可能会被限流
- 大型站点可能需要更长时间来分析

### 内容准确性

- 动态内容可能无法被完整捕获
- JavaScript 渲染的内容可能不完整
- 实时数据可能无法反映
- 某些内容可能位于付费墙之后

### 法律和伦理考虑

- 尊重 robots.txt 和站点政策
- 不要抓取受版权保护的内容用于复制
- 将分析用于获取灵感，而不是复制
- 确认你有权使用提取的内容
- 遵循被分析站点的服务条款

### 性能影响

- 抓取大型站点可能需要时间
- 多个同时进行的抓取可能更慢
- 网络连接会影响速度
- 大型数据集可能需要更多处理

## 最佳实践

### 使用站点分析

1. **分析多个站点** — 从多个来源收集灵感
2. **关注结构** — 学习布局和组织模式
3. **提取颜色** — 使用调色板作为起点
4. **研究排版** — 找出你喜欢的字体组合
5. **查看导航** — 理解菜单组织方式

### 伦理抓取

1. **尊重 robots.txt** — 遵循站点指南
2. **不要复制内容** — 仅将分析用于获取灵感
3. **验证权利** — 确保你可以使用提取的内容
4. **注明来源** — 为灵感来源署名
5. **遵循条款** — 遵守站点服务条款

### 最大化工具效果

1. **具体明确** — 请求特定类型的分析
2. **提供背景** — 说明你的站点用途
3. **设定期望** — 描述你的设计目标
4. **迭代** — 根据结果进行优化
5. **组合工具** — 使用多个工具进行全面分析

## 故障排除

### 站点无法抓取

- 检查站点是否可公开访问
- 确认 robots.txt 允许抓取
- 尝试其他站点
- 检查互联网连接
- 如果问题仍然存在，请联系支持

### 分析不完整

- 站点可能包含动态内容
- JavaScript 可能未完全渲染
- 大型站点可能超时
- 尝试改为分析特定页面
- 请求特定分析类型

### 性能缓慢

- 大型站点需要更长时间分析
- 多个同时进行的抓取会更慢
- 网络连接会影响速度
- 尝试在非高峰时段进行
- 先分析较小的部分
