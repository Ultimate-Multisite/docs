# Superdav AI Agent

Superdav AI Agent 是一款先进的 WordPress AI 助手，它在 Gratis AI Agent 的基础上增加了主题感知能力、增强的互联网搜索功能以及改进的网站构建器功能。版本 1.16.0 新增了自定义 Logo 生成、调色板对比度验证、发现访谈中的照片上传、结构化酒店菜单生成，以及主题构建器中的桌面/移动端预览渲染功能。

## 主要功能

- **主题感知能力 (Theme-aware skills)** — 根据您当前激活的 WordPress 主题和已安装的插件，自动调整指导和能力。
- **多互联网搜索提供商** — 可在 Brave Search 和 Tavily 之间选择，以获得更丰富的搜索结果和高级研究能力。
- **网站构建器联系表单** — 使用您已安装的表单插件，直接通过聊天界面创建功能完善的联系表单。
- **增强的网站构建器** — 改进了多阶段网站构建计划，具备更好的插件发现和错误恢复能力。
- **内置主题技能** — 为 Block Themes、Classic Themes、Kadence Blocks 和 Kadence Theme 提供专业指导。
- **改进的 WooCommerce 集成** — 重构为使用原生的 WooCommerce API，提高了可靠性和兼容性。
- **自动提供商检测** — 当插件或主题被激活或停用时，自动检测并刷新可用的插件和主题列表。
- **自定义 Logo 生成** — 生成并嵌入自定义 SVG Logo，并自动进行命名空间安全的清理。
- **调色板对比度验证** — 在将颜色应用到主题之前，检查颜色组合是否符合 WCAG 可访问性标准。
- **照片增强的发现过程** — 在主题构建器发现访谈中上传照片，以影响设计方向。
- **酒店菜单** — 为餐厅、咖啡馆和酒店等行业生成结构化的食品和饮料菜单页面。
- **桌面和移动预览** — 在选择设计方向时，可以在桌面和移动设备上预览您的设计。
- **一级网站抓取 (Tier 1 site scraping)** — sd-ai-agent/site-scrape 默认可用，用于分析竞争对手和灵感网站。

## 要求

- WordPress 6.0 或更高版本
- PHP 8.1 或更高版本
- WordPress Multisite（网络激活）或单站点安装
- 一个活动的 AI 提供商连接插件（Anthropic、OpenAI 或兼容插件）
- Gratis AI Agent v1.9.0 或更高版本（Superdav AI Agent 扩展了 Gratis AI Agent）

## 安装

1. 将插件文件上传到 `/wp-content/plugins/superdav-ai-agent`
2. 在 WordPress 的 **插件 (Plugins)** 界面激活插件
3. 确保 Gratis AI Agent 也已激活
4. 导航到 **Gratis AI Agent → 设置 (Settings)** 并配置您的互联网搜索提供商（可选）

## 快速上手

激活后，Superdav AI Agent 会自动使用主题感知能力和额外的功能增强您的 Gratis AI Agent 安装。打开 WordPress 管理侧边栏中的 **Gratis AI Agent** 面板，即可开始使用新功能。

**示例指令：**

```
在联系页面添加一个联系表单。
```

```
搜索网络上最新的 WordPress 安全最佳实践。
```

```
使用当前主题的区块模式创建一个英雄区域。
```

## 主题感知能力 (Theme-Aware Skills)

Superdav AI Agent 会自动检测您当前的主题和已安装的插件，并将相应的主题感知技能加载到助手的上下文环境中。可用的技能包括：

| 技能 | 适用对象 |
|---|---|
| **Block Themes** | 使用 WordPress 区块式（全站编辑）架构的主题 |
| **Classic Themes** | 使用 PHP 模板和经典编辑器的传统 WordPress 主题 |
| **Kadence Blocks** | 激活了 Kadence Blocks 插件的网站 |
| **Kadence Theme** | 使用 Kadence 主题作为当前主题的网站 |

有关每项技能的详细指导，请参阅 [主题感知能力](../../user-guide/configuration/theme-aware-skills)。

## 互联网搜索提供商

Superdav AI Agent v1.10.0 增加了 **Tavily** 作为 Brave Search 的替代品，用于互联网搜索查询。这两个提供商都提供实时搜索结果，并且可以在 **Gratis AI Agent → 设置 (Settings) → 高级 (Advanced)** 中进行配置。

| 提供商 | 功能 | 设置 |
|---|---|---|
| **Brave Search** | 快速、注重隐私的搜索结果 | [Brave Search API](https://brave.com/search/api/) |
| **Tavily** | 更丰富的搜索结果，高级研究能力 | [Tavily API](https://tavily.com/) |

有关配置说明，请参阅 [互联网搜索](../../user-guide/configuration/internet-search)。

## 网站构建器联系表单

网站构建器（Site Builder）代理现在可以直接从聊天界面创建联系表单。只需要求网站构建器创建一个联系表单，它就会自动检测您已安装的表单插件（Contact Form 7、WPForms、Fluent Forms 或 Gravity Forms），并生成一个可立即使用的表单。

有关详细指导，请参阅 [网站构建器联系表单](../../user-guide/administration/site-builder-contact-forms)。

## 主题构建器功能

Superdav AI Agent v1.16.0 的主题构建器包含强大的新功能：

### 带照片的发现访谈

主题构建器的发现访谈现在包含照片上传步骤，您可以在此提供视觉灵感。上传您品牌、竞争对手网站或设计灵感的照片，以帮助主题构建器了解您的设计方向。

有关详细指导，请参阅 [发现访谈](./theme-builder/discovery-interview.md)。

### 带有桌面和移动预览的设计方向

在主题构建器生成完整主题之前，可以在桌面和移动设备上查看您的设计。可以在多个响应式断点查看实时预览。

有关详细指导，请参阅 [设计方向](./theme-builder/design-direction.md)。

### 酒店菜单

主题构建器现在可以为餐厅、咖啡馆、酒吧和其他酒店业务生成结构化的食品和饮料菜单页面。菜单包括定价、描述、饮食信息和过敏原警告。

有关详细指导，请参阅 [酒店菜单](./theme-builder/hospitality-menus.md)。

### 新增能力

- **[生成 Logo SVG](./abilities/generate-logo-svg.md)** — 创建自定义 SVG Logo，并自动进行清理
- **[验证调色板对比度](./abilities/validate-palette-contrast.md)** — 检查颜色组合是否符合 WCAG 标准
- **[创建菜单](./abilities/create-menu.md)** — 创建带有不同导航标签的导航菜单

有关可用工具和一级网站抓取工具的信息，请参阅 [工具和配置](./theme-builder/tools.md)。

## 配置

所有设置都在 WordPress 管理后台的 **Gratis AI Agent → 设置 (Settings)** 下。Superdav AI Agent 添加了以下配置选项：

| 设置 | 描述 |
|---|---|
| **互联网搜索提供商** | 在 Brave Search 和 Tavily 之间选择（可选） |
| **Tavily API Key** | Tavily 互联网搜索的 API 密钥（如果选择了 Tavily） |

## v1.16.0 改进

- **AI 客户端缓存** — 现在由瞬态 (transients) 支持，实现跨请求持久化，防止长时间运行的代理任务丢失数据。
- **一级网站抓取工具** — sd-ai-agent/site-scrape 现在在主题构建器中默认可用，用于分析竞争对手和灵感网站。
- **插件行操作链接** — 已修正并重命名以提高清晰度。

## v1.10.0 改进

- **WooCommerce 集成重构** — 现在使用原生的 WooCommerce API，提高了可靠性和兼容性。
- **提供商列表自动刷新** — 自动检测插件是否被激活或停用，并更新可用提供商列表。
- **增强的网站构建器** — 改进了多阶段网站构建计划，具备更好的错误恢复能力。
- **主题感知能力** — 四项新的内置技能，可自动适应您的当前主题。

## v1.10.0 Bug 修复

- 修复了 `navigate-to` 能力在某些管理页面上导致无限重新加载循环的问题。
- 修复了 `list-posts` 能力未能正确将分类和标签名称解析为 slug 的问题。
- 修复了 WP-CLI 命令在上次重构后缺少命名空间别名的问题。
- 修复了事件自动化在尚未创建自动化表的情况下也能优雅处理网站的问题。
- 修复了 `memory-save` 能力在系统指令构建器中使用正确的命名空间前缀的问题。
- 修复了标量工具结果在返回给 AI 之前被正确包裹的问题。
- 修复了使用统计数据在从旧版本升级时，能正确处理遗留能力键格式的问题。
