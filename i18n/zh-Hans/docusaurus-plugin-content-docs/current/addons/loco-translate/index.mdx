# Loco Translate

Ultimate Multisite：Loco Translate 集成插件为您的 WordPress 多站点网络提供了站点特定的翻译功能。如果您曾经需要网络中的不同站点对同一插件或主题使用不同的翻译，那么此插件是完美的解决方案。

**What This Addon Does**

此插件无缝集成了 Ultimate Multisite 与流行的 Loco Translate 插件，使网络中的每个子站点都能维护自己的自定义翻译。无论您是在运营多地区平台、服务不同语言社区，还是仅仅需要在整个网络中实现翻译灵活性，使用此插件都能轻松完成。

**Perfect For:**

* 需要不同本地化需求的多地区企业
* 服务多元社区的教育机构
* 提供多语言支持的 SaaS 平台
* 管理多家客户站点的 Web 机构
* 需要翻译灵活性的任何多站点网络

## 安装

1. 确保已安装并网络激活 Ultimate Multisite
2. 安装并激活 Loco Translate 插件
3. 将此插件上传至您的插件目录
4. 在网络层激活 Loco Translate Integration 插件
5. 开始在任何子站点通过 Loco Translate 创建站点特定翻译

就这样！集成会自动工作，无需任何额外配置。

## Frequently Asked Questions

## 安装后需要配置吗？

不需要！集成是自动的。只要插件和 Loco Translate 都已激活，子站点即可立即开始创建自己的自定义翻译。

## 如果子站点不创建自定义翻译会怎样？

该站点将自动使用网络范围内的翻译作为回退。不会出现任何错误，一切都能无缝工作。

## 子站点能访问网络范围内的翻译吗？

普通站点管理员只能查看和编辑自己站点的翻译。超级管理员可以根据正在查看的站点，访问网络范围内的翻译和站点特定翻译。

## 这适用于插件和主题吗？

是的！站点特定翻译适用于 WordPress 插件和主题。

## 这会降低我的多站点网络性能吗？

不会。该插件使用高效的文件加载，仅在需要时检查站点特定翻译，不会产生明显的性能影响。

## 我可以将网络翻译复制到特定站点作为起点吗？

可以！在 Loco Translate 中编辑翻译时，您可以轻松以网络翻译为起点，并为每个站点进行自定义。

## 这适用于 WooCommerce 和其他插件吗？

绝对可以！任何使用标准 WordPress 翻译函数的插件或主题都能与站点特定翻译一起使用。
