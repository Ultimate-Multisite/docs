# 多币种支持

Ultimate Multisite: Multi-Currency 插件可让您的网络接受多种货币的支付。它能自动使用实时或手动汇率转换价格，让您为任何产品设置特定货币的精确价格，并通过地理位置检测每个访问者偏好的货币——所有这些功能都可以在一个管理后台中完成。

## 主要功能 {#key-features}

- **自动汇率转换** — 价格将使用从您选择的提供商获取的汇率，从您的基础货币进行转换。
- **按产品定价覆盖** — 您可以在任何产品上设置特定货币的精确价格，以绕过自动转换。
- **货币选择器注册字段** — 允许客户在结账/注册时选择他们偏好的货币。
- **多种汇率提供商** — 包括 ECB、Frankfurter、Stripe 或完全手动设置的汇率。
- **基于地理位置的货币检测** — 自动预选与访问者国家匹配的货币。
- **管理后台** — 在一个地方管理活动的货币、汇率和提供商设置。

## 要求 {#requirements}

- Ultimate Multisite 2.0.0 或更高版本
- WordPress Multisite 安装
- PHP 7.4 或更高版本

## 安装 {#installation}

1. 将插件文件上传到 `/wp-content/plugins/ultimate-multisite-multi-currency`
2. 通过 WordPress 的 **插件** 界面激活插件（网络激活）
3. 导航到 **Ultimate Multisite → Multi-Currency** 以打开管理后台
4. 添加您希望接受的货币，并配置您的汇率提供商

## 配置 {#configuration}

### 管理后台 {#admin-dashboard}

Multi-Currency 管理后台（**Ultimate Multisite → Multi-Currency**）是以下操作的中心位置：

- 添加、启用或禁用货币
- 选择汇率提供商并在需要时刷新汇率
- 查看所有活动货币的当前汇率
- 当自动提供商不适用时，设置手动汇率

### 汇率提供商 {#exchange-rate-providers}

| 提供商 | 描述 |
|---|---|
| **ECB** | 欧洲中央银行 — 适用于以欧元为基础的网络，提供免费的每日汇率。 |
| **Frankfurter** | 开源的 ECB 镜像 API — 免费，无需 API 密钥。 |
| **Stripe** | 使用 Stripe 的实时外汇汇率 — 需要激活 Stripe 集成。 |
| **手动** | 您直接输入汇率；无需外部 API 调用。 |

请在 **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** 下选择您的提供商。汇率可以随时手动刷新，或设置为自动更新。

### 按产品定价覆盖 {#per-product-pricing-overrides}

当自动转换不够精确时，您可以为任何产品设置特定货币的精确价格：

1. 打开产品编辑器（**Ultimate Multisite → Products → Edit**）
2. 滚动到 **Currency Pricing**（货币定价）部分
3. 为您希望覆盖的每种货币输入精确价格
4. 保存产品

没有设置覆盖价格的产品将使用从您的基础货币转换而来的价格。

### 货币选择器注册字段 {#currency-selector-signup-field}

可以将货币选择器字段添加到您的注册/结账流程中，让客户在注册时选择他们偏好的货币：

1. 前往 **Ultimate Multisite → Multi-Currency → Settings**
2. 启用 **Show currency selector on signup**（在注册时显示货币选择器）
3. 可选地设置默认货币，或让地理位置决定预选值

选定的货币将与订阅一起存储，并用于所有后续的计费。

### 基于地理位置的货币检测 {#geolocation-based-currency-detection}

当启用地理位置检测时，该插件会将访问者的国家映射到一种货币，并在货币选择器字段中预选它。要启用此功能：

1. 前往 **Ultimate Multisite → Multi-Currency → Settings**
2. 启用 **Detect currency from visitor location**（从访问者位置检测货币）
3. 在 **Geolocation Mapping**（地理位置映射）表中将国家映射到货币

如果访问者的国家没有映射，则使用网络的基础货币作为备用选项。

## 常见问题解答 {#frequently-asked-questions}

### 如果汇率提供商不可用怎么办？ {#what-happens-if-an-exchange-rate-provider-is-unavailable}

该插件将回退到上次成功获取的汇率。如果从未获取过汇率，则回退到您配置的手动汇率。建议配置手动汇率作为安全网。

### 我可以为不同的货币使用不同的提供商吗？ {#can-i-use-different-providers-for-different-currencies}

不可以——所有货币一次只激活一个提供商。如果您需要完全控制单个汇率，请使用 **Manual**（手动）。

### 地理位置检测是否需要单独的插件？ {#does-geolocation-require-a-separate-plugin}

不需要外部的地理位置插件。该插件使用 WordPress 内置的 IP 到国家检测功能。如果需要更高的准确性，可以通过提供的 filter hooks 集成专用的地理位置服务。

### 按产品覆盖的价格是在税前还是税后？ {#are-per-product-overrides-applied-before-or-after-tax}

按产品覆盖的价格设置的是目标货币的税前价格。税款将使用您现有的税收配置，在覆盖价格之上计算。
