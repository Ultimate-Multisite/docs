# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints 将 OpenAI 兼容的 AI 提供商 API 与您的 WordPress 多站点网络集成。您可以配置多个 AI 提供商——每个提供商都有其独立的端点 URL、API 密钥和默认模型——并实现请求路由，当某个提供商不可用时会自动回退（fallback）。

## Key Features

- **多提供商支持**: 可以添加多个 AI 提供商卡片，每个卡片都有独立的端点 URL、API 密钥和默认模型
- **拖拽重新排序**: 使用每个卡片上的握柄（grip handle）按优先级排列提供商；请求将按照顺序使用这些提供商
- **自动回退路由**: 如果主要提供商不可用，请求会自动回退到下一个配置的提供商
- **按提供商配置**: 每个提供商都独立配置其凭证和模型选择
- **卡片式设置 UI**: 基于稳定的 WordPress 6.9+ Card 组件（CardBody, CardHeader, CardDivider）构建
- **添加时自动展开**: 新的提供商卡片会自动展开，让您可以立即填写详细信息
- **脚本缓存清除**: 在插件更新时，提供商脚本会进行缓存清除（cache-busting），确保加载的是最新版本

## Requirements

- WordPress 5.3 或更高版本
- PHP 7.4 或更高版本
- Ultimate Multisite 插件（已激活）
- 一个 OpenAI 兼容的 API 端点（自托管或第三方）

## Installation

1. 将插件文件上传到您的 `/wp-content/plugins/` 目录
2. 通过 WordPress 的“插件”菜单激活插件
3. 导航到 **Ultimate Multisite → AI Connector** 来配置插件

## Configuration

### Multi-Provider Setup

配置界面采用卡片式布局，每个卡片代表一个 AI 提供商。您可以添加所需的任何数量的提供商。

#### Adding a Provider

1. 在网络管理员界面，转到 **Ultimate Multisite → AI Connector**
2. 点击 **Add Provider**
3. 会出现一个新的提供商卡片并自动展开
4. 填写提供商详细信息：
   - **Provider Name** — 用于识别此提供商的标签（例如：“OpenAI”、“Mistral”、“本地 Ollama”）
   - **Endpoint URL** — OpenAI 兼容 API 的基础 URL（例如：`https://api.openai.com/v1`）
   - **API Key** — 此提供商的 API 密钥
   - **Default Model** — 默认使用的模型标识符（例如：`gpt-4o`、`mistral-small`）
5. 点击 **Save Settings**

#### Removing a Provider

点击任何提供商卡片上的 **Remove** 按钮即可删除它。请保存设置以应用更改。

#### Reordering Providers

拖动任何提供商卡片左侧的握柄图标（⣿）即可重新排序。顺序决定了回退的优先级：请求会发送给第一个提供商，如果请求失败，则会回退到后续的提供商。

### Fallback Routing

当向主要提供商的请求失败时——可能是由于超时、速率限制或服务中断——连接器会自动使用列表中的下一个提供商重试。这个过程会持续进行，直到某个提供商成功，或者所有提供商都已尝试完毕。

| Scenario | Behaviour |
|----------|-----------|
| Primary provider responds | Request is served normally |
| Primary provider fails | Request retried with next provider |
| All providers fail | Error returned to the caller |

回退机制对调用插件或主题是透明的——无论尝试了多少提供商，它们最终只会收到一个成功的响应或一个最终的错误。

### Provider Configuration Reference

| Field | Required | Description |
|-------|----------|-------------|
| Provider Name | Yes | 此提供商的显示标签 |
| Endpoint URL | Yes | OpenAI 兼容 API 的基础 URL |
| API Key | Yes | 此提供商的认证密钥 |
| Default Model | Yes | 当请求中未指定模型时使用的模型标识符 |

## Troubleshooting

### No providers are listed

如果保存后提供商列表为空，请确认以下几点：

- 您是否在添加提供商后点击了 **Save Settings**
- 您的浏览器是否阻止了设置页面的保存（请检查浏览器控制台是否有 JavaScript 错误）
- 插件是否已激活，并且您是否位于网络管理员界面

### Requests are failing for all providers

- 验证每个提供商的端点 URL 是否可以从您的服务器访问（而不仅仅是从您的浏览器访问）
- 检查每个 API 密钥是否有效且未过期
- 在提供商的控制台中查看其速率限制和配额状态
- 启用 WordPress 调试日志 (`WP_DEBUG_LOG`) 以捕获连接器详细的错误消息

### A provider is not being used for fallback

拖拽重新排序会改变回退的顺序。请确认您期望接收回退请求的提供商是否已正确放置在列表中，并且其端点 URL 和 API 密钥是否正确。

### New provider card does not expand automatically

自动展开功能要求浏览器启用 JavaScript。如果卡片没有在点击时展开，请检查浏览器控制台是否有错误。即使自动展开没有触发，您仍然可以通过点击卡片头部访问卡片内容。
