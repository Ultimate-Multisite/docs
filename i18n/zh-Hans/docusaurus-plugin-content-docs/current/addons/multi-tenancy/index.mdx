# 多租户 {#multi-tenancy}

将子站点分布到多个服务器和数据库，以提升性能与安全性

## Version 1.2.0 发布亮点 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 为生产隔离工作流添加了主权租户基础：

- 通过短生命周期、防重放保护的 SSO 访问实现无状态租户自动登录。
- 为主权租户提供按子站点划分的数据库和文件系统隔离。
- 面向感知主机的租户配置提供 Hostinger 多租户能力。
- 用于旧数据检查和异步推送处理的主权迁移验证门控。
- 可配置的同机数据库主机绑定，适用于 Bedrock、FrankenPHP 和 localhost 安装。
- 租户引导、数据库写入器、架构引导和用户配置改进。
- 更安全的主权拆除流程，会清理租户凭据。

请参阅用户指南：[主权迁移工作流](/user-guide/getting-started/sovereign-migration-workflow)、[多租户隔离](/user-guide/configuration/multi-tenancy-isolation)和[主权租户操作](/user-guide/administration/sovereign-tenant-operations)。

## 要求 {#requirements}

- WordPress 5.3 或更高版本
- PHP 7.4 或更高版本
- Multisite Ultimate plugin（已启用）

## 安装 {#installation}

1. 将 addon 文件上传到你的 `/wp-content/plugins/` 目录
2. 通过 WordPress 中的“Plugins”菜单启用 plugin
3. 在网络管理员中配置 addon 设置

## 配置 {#configuration}

该 addon 支持多种配置选项，可通过网络管理员界面访问。

### 主权租户配置 {#sovereign-tenant-configuration}

对于主权租户，在将流量路由到租户之前，请配置租户数据库、文件系统根目录、数据库主机绑定和主机能力。每次更改影响数据库凭据、主机绑定、路由或用户配置时，都要运行迁移验证。

仅当租户运行时和数据库授权预期使用本地套接字或同机访问时，才使用 `localhost`。当你的提供商配置的授权主机是 `127.0.0.1` 或某个服务主机名时，请使用它们。

### Hostinger 能力 {#hostinger-capability}

Hostinger 多租户能力使用共享的 Hostinger 集成令牌。验证前，请确认该域名存在于 hPanel 中，DNS 指向 Hostinger 账户，并且租户数据库设置与 hPanel 匹配。

## Hooks 和 Filters {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - 在 addon 初始化后触发
- `ultimate-multisite-addon-template_loaded` - 在所有 addon 文件加载后触发

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - 修改 addon 设置
- `ultimate-multisite-addon-template_enabled` - 覆盖 addon 启用/禁用状态
