# Multi-Tenancy

Distribusikan subsitus di berbagai server dan database untuk performa dan keamanan

## Sorotan rilis versi 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 menambahkan fondasi tenant berdaulat untuk alur kerja isolasi produksi:

- Autologin tenant stateless dengan kunjungan SSO berumur pendek dan terlindungi dari replay.
- Isolasi database dan filesystem per-subsitus untuk tenant berdaulat.
- Kemampuan multi-tenancy Hostinger untuk penyediaan tenant yang sadar host.
- Gerbang verifikasi migrasi berdaulat untuk pemeriksaan data lama dan pemrosesan push asinkron.
- Pengikatan host database pada mesin yang sama yang dapat dikonfigurasi untuk instalasi Bedrock, FrankenPHP, dan localhost.
- Peningkatan bootstrap tenant, penulis database, bootstrap skema, dan penyediaan pengguna.
- Teardown berdaulat yang lebih aman yang membersihkan kredensial tenant.

Lihat panduan pengguna untuk [alur kerja migrasi berdaulat](/user-guide/getting-started/sovereign-migration-workflow), [isolasi multi-tenancy](/user-guide/configuration/multi-tenancy-isolation), dan [operasi tenant berdaulat](/user-guide/administration/sovereign-tenant-operations).

## Persyaratan {#requirements}

- WordPress 5.3 atau lebih tinggi
- PHP 7.4 atau lebih tinggi
- plugin Multisite Ultimate (aktif)

## Instalasi {#installation}

1. Unggah file addon ke direktori `/wp-content/plugins/` Anda
2. Aktifkan plugin melalui menu 'Plugins' di WordPress
3. Konfigurasikan pengaturan addon di admin jaringan

## Konfigurasi {#configuration}

Addon ini mendukung berbagai opsi konfigurasi yang dapat diakses melalui antarmuka admin jaringan.

### Konfigurasi tenant berdaulat {#sovereign-tenant-configuration}

Untuk tenant berdaulat, konfigurasikan database tenant, root filesystem, pengikatan host database, dan kemampuan host sebelum merutekan lalu lintas ke tenant. Jalankan verifikasi migrasi setelah setiap perubahan yang memengaruhi kredensial database, pengikatan host, routing, atau penyediaan pengguna.

Gunakan `localhost` hanya ketika runtime tenant dan grant database mengharapkan soket lokal atau akses pada mesin yang sama. Gunakan `127.0.0.1` atau hostname layanan ketika itulah host grant yang dikonfigurasi oleh penyedia Anda.

### Kemampuan Hostinger {#hostinger-capability}

Kemampuan multi-tenancy Hostinger bekerja dengan token integrasi Hostinger bersama. Pastikan domain ada di hPanel, DNS mengarah ke akun Hostinger, dan pengaturan database tenant cocok dengan hPanel sebelum verifikasi.

### Action {#actions}

- `ultimate-multisite-addon-template_init` - Dipicu setelah inisialisasi addon

- `ultimate-multisite-addon-template_enabled` - Timpa status aktif/nonaktif addon
