# Ultimate AI Connector for Compatible Endpoints

Ultimate AI Connector for Compatible Endpoints mengintegrasikan API penyedia AI yang kompatibel dengan OpenAI ke jaringan multisite WordPress Anda. Konfigurasikan beberapa penyedia AI — masing-masing dengan URL endpoint, API key, dan model default sendiri — dan alihkan permintaan dengan *automatic fallback* (pengalihan otomatis) ketika salah satu penyedia tidak tersedia.

## Key Features

- **Dukungan multi-penyedia**: Tambahkan beberapa kartu penyedia AI, masing-masing dengan URL endpoint, API key, dan model default sendiri
- **Urut ulang dengan *drag***: Atur penyedia berdasarkan prioritas menggunakan pegangan (*grip handle*) pada setiap kartu; permintaan akan menggunakan penyedia secara berurutan
- **Pengalihan *automatic fallback***: Jika penyedia utama tidak tersedia, permintaan akan otomatis beralih ke penyedia yang dikonfigurasi berikutnya
- **Konfigurasi per-penyedia**: Setiap penyedia dikonfigurasi secara independen dengan kredensial dan pilihan modelnya sendiri
- **UI pengaturan berbasis kartu**: Dibangun di atas komponen Kartu WordPress 6.9+ yang stabil (CardBody, CardHeader, CardDivider)
- **Auto-expand saat tambah**: Kartu penyedia baru akan otomatis diperluas sehingga Anda dapat mengisi detailnya dengan segera
- ***Script cache busting***: Skrip penyedia akan di-*cache-bust* saat pembaruan plugin untuk memastikan versi terbaru dimuat

## Requirements

- WordPress 5.3 atau lebih tinggi
- PHP 7.4 atau lebih tinggi
- Plugin Ultimate Multisite (aktif)
- Endpoint API yang kompatibel dengan OpenAI (self-hosted atau pihak ketiga)

## Installation

1. Unggah file *addon* ke direktori `/wp-content/plugins/` Anda
2. Aktifkan plugin melalui menu 'Plugins' di WordPress
3. Arahkan ke **Ultimate Multisite → AI Connector** untuk mengonfigurasi *addon*

## Configuration

### Multi-Provider Setup

Layar konfigurasi menggunakan tata letak berbasis kartu di mana setiap kartu mewakili satu penyedia AI. Anda dapat menambahkan sebanyak penyedia yang Anda butuhkan.

#### Adding a Provider

1. Buka **Ultimate Multisite → AI Connector** di *network admin*
2. Klik **Add Provider**
3. Kartu penyedia baru akan muncul dan otomatis diperluas
4. Isi detail penyedia:
   - **Provider Name** — label untuk mengidentifikasi penyedia ini (misalnya, "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL** — URL dasar dari API yang kompatibel dengan OpenAI (misalnya, `https://api.openai.com/v1`)
   - **API Key** — API key Anda untuk penyedia ini
   - **Default Model** — pengenal model yang akan digunakan secara default untuk penyedia ini (misalnya, `gpt-4o`, `mistral-small`)
5. Klik **Save Settings**

#### Removing a Provider

Klik tombol **Remove** pada kartu penyedia mana pun untuk menghapusnya. Simpan pengaturan untuk menerapkan perubahan.

#### Reordering Providers

Tarik ikon pegangan (⣿) di sisi kiri kartu penyedia mana pun untuk mengurutkannya kembali. Urutan menentukan prioritas *fallback*: permintaan dikirim ke penyedia pertama dan beralih ke penyedia berikutnya jika permintaan gagal.

### Fallback Routing

Ketika permintaan ke penyedia utama gagal — karena *timeout*, *rate limit*, atau pemadaman layanan — *connector* akan otomatis mencoba kembali dengan penyedia berikutnya dalam daftar. Ini berlanjut ke bawah daftar sampai penyedia berhasil atau semua penyedia telah dicoba.

| Scenario | Behaviour |
|----------|-----------|
| Primary provider responds | Request is served normally |
| Primary provider fails | Request retried with next provider |
| All providers fail | Error returned to the caller |

*Fallback* bersifat transparan bagi *plugin* atau tema pemanggil — mereka akan menerima respons sukses atau *error* akhir, terlepas dari berapa banyak penyedia yang telah dicoba.

### Provider Configuration Reference

| Field | Required | Description |
|-------|----------|-------------|
| Provider Name | Yes | Label tampilan untuk penyedia ini |
| Endpoint URL | Yes | URL dasar dari API yang kompatibel dengan OpenAI |
| API Key | Yes | Kunci otentikasi untuk penyedia ini |
| Default Model | Yes | Pengenal model yang digunakan ketika tidak ada model yang ditentukan dalam permintaan |

## Troubleshooting

### No providers are listed

Jika daftar penyedia kosong setelah menyimpan, pastikan bahwa:

- Anda telah mengklik **Save Settings** setelah menambahkan penyedia
- Browser Anda tidak memblokir halaman pengaturan dari penyimpanan (periksa konsol browser untuk *error* JavaScript)
- Plugin aktif dan Anda berada di layar *network admin*

### Requests are failing for all providers

- Verifikasi bahwa endpoint URL setiap penyedia dapat dijangkau dari server Anda (tidak hanya dari browser Anda)
- Periksa bahwa setiap API key valid dan belum kedaluwarsa
- Tinjau status *rate-limit* dan kuota penyedia di *dashboard* mereka
- Aktifkan *debug logging* WordPress (`WP_DEBUG_LOG`) untuk menangkap pesan *error* terperinci dari *connector*

### A provider is not being used for fallback

*Drag-to-reorder* mengubah urutan *fallback*. Pastikan penyedia yang Anda harapkan menerima permintaan *fallback* berada di posisi yang benar dalam daftar dan bahwa endpoint URL serta API key-nya sudah benar.

### New provider card does not expand automatically

*Auto-expand* memerlukan JavaScript diaktifkan di browser Anda. Jika kartu tidak diperluas saat diklik, periksa konsol browser untuk *error*. Isi kartu selalu dapat diakses dengan mengklik *card header* meskipun *auto-expand* tidak terpicu.
