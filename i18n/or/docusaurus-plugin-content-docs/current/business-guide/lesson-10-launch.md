# ପାଠ ୧୦: ଲଞ୍ଚ ଦିନ {#lesson-10-launch-day}

ସବୁକିଛି ତିଆରି ହୋଇଗଲା। ଦ୍ୱାର ଖୋଲିବା ପୂର୍ବରୁ, କିଛି ଭାଙ୍ଗିନାହିଁ, ଛାଡ଼ିଯାଇନାହିଁ, କିମ୍ବା ଅସୁବିଧାଜନକ ନୁହେଁ ବୋଲି ନିଶ୍ଚିତ କରିବା ପାଇଁ ଏହି ପାଠ ଆପଣଙ୍କୁ ପ୍ରାକ୍-ଲଞ୍ଚ ଯାଞ୍ଚସୂଚୀ ମାଧ୍ୟମରେ ନେଇଯାଏ।

## ଆମେ କେଉଁଠାରେ ଛାଡ଼ିଥିଲୁ {#where-we-left-off}

FitSiteରେ ନମୁନା, ପ୍ଲାନ୍, କ୍ରୟ-ପ୍ରକ୍ରିୟା, ବ୍ରାଣ୍ଡିଂ, ଆରମ୍ଭିକ ମାର୍ଗଦର୍ଶନ, ଏବଂ ମୂଲ୍ୟନିର୍ଣ୍ଣୟ ସବୁ ବିନ୍ୟାସ ହୋଇଛି। ଏବେ ଆମେ ସବୁକିଛି କାମ କରୁଛି କି ନାହିଁ ଯାଞ୍ଚ କରି ଲାଇଭ୍ କରିବୁ।

## ପ୍ରାକ୍-ଲଞ୍ଚ ଯାଞ୍ଚସୂଚୀ {#pre-launch-checklist}

ପ୍ରତ୍ୟେକ ଆଇଟମ୍ ଉପରେ କାମ କରନ୍ତୁ। କୌଣସିଟି ଛାଡ଼ନ୍ତୁ ନାହିଁ।

### ପ୍ଲାଟଫର୍ମ ଅବକାଠାମୋ {#platform-infrastructure}

- [ ] ହୋଷ୍ଟିଂ ସ୍ଥିର ଅଛି ଏବଂ ଲୋଡ୍ ଅଧୀନରେ ଭଲ କାମ କରୁଛି
- [ ] ୱାଇଲ୍ଡକାର୍ଡ SSL ସକ୍ରିୟ ଅଛି ଏବଂ ସମସ୍ତ ସବ୍ଡୋମେନ୍ HTTPS ଦ୍ୱାରା ସର୍ଭ୍ ହେଉଛି
- [ ] ଡୋମେନ୍ ମ୍ୟାପିଂ କାମ କରୁଛି -- ଏକ ସାଇଟ୍ ସୃଷ୍ଟି କରି ଏବଂ କଷ୍ଟମ୍ ଡୋମେନ୍ ମ୍ୟାପ୍ କରି ପରୀକ୍ଷା କରନ୍ତୁ
- [ ] ବ୍ୟାକଅପ୍ ବିନ୍ୟାସିତ ଏବଂ ପରୀକ୍ଷିତ ହୋଇଛି (ଯାଞ୍ଚ କରିବାକୁ କମ୍ ସେ କମ୍ ଗୋଟିଏ ପୁନଃସ୍ଥାପନ କରନ୍ତୁ)
- [ ] ମନିଟରିଂ ରହିଛି -- ପ୍ଲାଟଫର୍ମ ବନ୍ଦ ହେଲେ ଆପଣ ଜାଣିପାରିବେ

### ନମୁନା {#templates}

- [ ] ନୂତନ ସାଇଟ୍‌ରେ ତିନୋଟି ନମୁନା ସଠିକ୍ ଭାବେ ଲୋଡ୍ ହେଉଛି
- [ ] ପ୍ଲେସହୋଲ୍ଡର୍ ବିଷୟବସ୍ତୁ ଉପଯୋଗୀ ଏବଂ ଟାଇପ୍ ଭୁଲ୍ ବିହୀନ
- [ ] ସମସ୍ତ ଛବି ଠିକ୍ ଭାବେ ଲାଇସେନ୍ସ୍ ହୋଇଛି (ଷ୍ଟକ୍ ଫଟୋ ୱାଟରମାର୍କ ନାହିଁ)
- [ ] ପ୍ରତ୍ୟେକ ନମୁନା ପୃଷ୍ଠାରେ ମୋବାଇଲ୍ ପ୍ରତିକ୍ରିୟାଶୀଳତା କାମ କରୁଛି
- [ ] ପୃଷ୍ଠା ଲୋଡ୍ ଗତି ଗ୍ରହଣଯୋଗ୍ୟ (GTmetrix କିମ୍ବା PageSpeed Insights ଭଳି ଟୁଲ୍ ସହିତ ପରୀକ୍ଷା କରନ୍ତୁ)
- [ ] କୌଣସି ନମୁନାରେ ଭଙ୍ଗା ଲିଙ୍କ୍ କିମ୍ବା ଅନୁପସ୍ଥିତ ସମ୍ପଦ ନାହିଁ

### ପ୍ଲାନ୍ ଏବଂ ପ୍ରଡକ୍ଟ {#plans-and-products}

- [ ] ତିନୋଟି ପ୍ଲାନ୍ ସକ୍ରିୟ ଏବଂ ଦୃଶ୍ୟମାନ
- [ ] ପ୍ଲାନ୍ ବର୍ଣ୍ଣନା ସଠିକ୍ ଏବଂ ବିଶେଷ କ୍ଷେତ୍ର-ନିର୍ଦ୍ଦିଷ୍ଟ
- [ ] ମୂଲ୍ୟ ସଠିକ୍ (ମାସିକ ଏବଂ ବାର୍ଷିକ)
- [ ] ସେଟଅପ୍ ଶୁଳ୍କ ଠିକ୍ ପ୍ଲାନ୍‌ରେ ବିନ୍ୟାସିତ
- [ ] Starter ପ୍ଲାନ୍‌ରେ ପରୀକ୍ଷା ଅବଧି କାମ କରୁଛି
- [ ] କ୍ରୟ-ପ୍ରକ୍ରିୟା ସମୟରେ ଅର୍ଡର୍ ବମ୍ପ୍ସ ସଠିକ୍ ଭାବେ ଦେଖାଯାଉଛି
- [ ] ପ୍ରତ୍ୟେକ ପ୍ଲାନ୍ ଅନୁଯାୟୀ ପ୍ଲଗଇନ୍ ଏବଂ ଥିମ୍ ସୀମାବଦ୍ଧତା ସଠିକ୍ ଭାବେ ଲାଗୁ ହେଉଛି

### କ୍ରୟ-ପ୍ରବାହ {#checkout-flow}

- [ ] ପ୍ରତ୍ୟେକ ପ୍ଲାନ୍‌ରେ ସମ୍ପୂର୍ଣ୍ଣ ପରୀକ୍ଷା ସାଇନଅପ୍ କରନ୍ତୁ (ପରୀକ୍ଷା ପେମେଣ୍ଟ ମୋଡ୍ ବ୍ୟବହାର କରନ୍ତୁ)
- [ ] ନମୁନା ଚୟନ ପ୍ରତ୍ୟେକ ପ୍ଲାନ୍ ପାଇଁ ସଠିକ୍ ନମୁନା ଦେଖାଉଛି
- [ ] ପେମେଣ୍ଟ ସଫଳତାର ସହିତ ପ୍ରକ୍ରିୟା ହେଉଛି
- [ ] ସାଇନଅପ୍ ପରେ ଗ୍ରାହକ ସ୍ୱାଗତ ଇମେଲ୍ ପାଉଛନ୍ତି
- [ ] ସଠିକ୍ ନମୁନା ସହିତ ନୂତନ ସାଇଟ୍ ସୃଷ୍ଟି ହେଉଛି
- [ ] ଗ୍ରାହକ ତୁରନ୍ତ ତାଙ୍କ ନୂତନ ସାଇଟ୍‌କୁ ଲଗଇନ୍ କରିପାରୁଛନ୍ତି
- [ ] ଡିସ୍କାଉଣ୍ଟ କୋଡ୍ ସଠିକ୍ ଭାବେ କାମ କରୁଛି

### ବ୍ରାଣ୍ଡିଂ {#branding}

- [ ] ଲଗଇନ୍ ପୃଷ୍ଠାରେ FitSite ବ୍ରାଣ୍ଡିଂ ଦେଖାଯାଉଛି
- [ ] ଆଡମିନ୍ Dashboardରେ FitSite ବ୍ରାଣ୍ଡିଂ ଦେଖାଯାଉଛି
- [ ] ସମସ୍ତ ସିଷ୍ଟମ୍ ଇମେଲ୍ FitSite ବ୍ରାଣ୍ଡିଂ ଏବଂ ଫିଟନେସ୍-ନିର୍ଦ୍ଦିଷ୍ଟ ଭାଷା ବ୍ୟବହାର କରୁଛି
- [ ] ଆପଣଙ୍କ ବ୍ୟବସାୟ ବିବରଣୀ ସହ ଇନଭଏସ୍ ସଠିକ୍ ଭାବେ ଦେଖାଯାଉଛି
- [ ] ମାର୍କେଟିଂ ସାଇଟ୍ ଲାଇଭ୍ ଅଛି ଏବଂ କ୍ରୟ-ଫର୍ମକୁ ଲିଙ୍କ୍ କରୁଛି

### ଆରମ୍ଭିକ ମାର୍ଗଦର୍ଶନ {#onboarding}

- [ ] ନୂତନ ଗ୍ରାହକ Dashboardରେ Quick Start ୱିଜେଟ୍ ଦେଖାଯାଉଛି
- [ ] ସମସ୍ତ Quick Start ଲିଙ୍କ୍ ସଠିକ୍ ପୃଷ୍ଠାକୁ ଯାଉଛି
- [ ] ସ୍ୱାଗତ ଇମେଲ୍ କ୍ରମ ବିନ୍ୟାସିତ ଏବଂ ପରୀକ୍ଷିତ
- [ ] ଜ୍ଞାନଭଣ୍ଡାର ଲେଖାଗୁଡ଼ିକ ପ୍ରକାଶିତ ଏବଂ ପହଞ୍ଚଯୋଗ୍ୟ
- [ ] Account ପୃଷ୍ଠା ସଠିକ୍ ପ୍ଲାନ୍ ସୂଚନା ଏବଂ ଅପଗ୍ରେଡ୍ ବିକଳ୍ପ ଦେଖାଉଛି

### ଆଇନଗତ ଏବଂ ବ୍ୟବସାୟ {#legal-and-business}

- [ ] ସେବା ସର୍ତ୍ତ ପ୍ରକାଶିତ ହୋଇଛି ଏବଂ କ୍ରୟ-ପ୍ରକ୍ରିୟାରୁ ଲିଙ୍କ୍ ହୋଇଛି
- [ ] ଗୋପନୀୟତା ନୀତି ପ୍ରକାଶିତ ଏବଂ ପହଞ୍ଚଯୋଗ୍ୟ
- [ ] ରିଫଣ୍ଡ ନୀତି ନିର୍ଦ୍ଧାରିତ ଏବଂ ଲିପିବଦ୍ଧ
- [ ] ପେମେଣ୍ଟ ଗ୍ରହଣ ପାଇଁ ବ୍ୟବସାୟ ଏକକ ସେଟଅପ୍ ହୋଇଛି
- [ ] ପେମେଣ୍ଟ ଗେଟୱେ ଲାଇଭ୍ ମୋଡ୍‌ରେ ଅଛି (ପରୀକ୍ଷା ମୋଡ୍ ନୁହେଁ)
- [ ] ଆପଣଙ୍କ ଅଞ୍ଚଳାଧିକାର ପାଇଁ କର ବିନ୍ୟାସ ସଠିକ୍

## ସଫ୍ଟ ଲଞ୍ଚ ବନାମ ହାର୍ଡ ଲଞ୍ଚ {#soft-launch-vs-hard-launch}

ଦୁଇ-ପର୍ଯ୍ୟାୟ ଲଞ୍ଚ ବିଚାର କରନ୍ତୁ:

### ପର୍ଯ୍ୟାୟ ୧: ସଫ୍ଟ ଲଞ୍ଚ {#phase-1-soft-launch}

ସାର୍ବଜନୀନ ଲଞ୍ଚ ପୂର୍ବରୁ ୫-୧୦ ଜଣ ଫିଟନେସ୍ ଷ୍ଟୁଡିଓ ମାଲିକଙ୍କୁ ସାଇନଅପ୍ କରିବାକୁ ଆମନ୍ତ୍ରଣ କରନ୍ତୁ। ସେମାନେ ଆପଣଙ୍କ ବେଟା ଗ୍ରାହକ। ଏହା ପାଇଁ ବଦଳରେ ସେମାନଙ୍କୁ ଲକ୍ଷଣୀୟ ଡିସ୍କାଉଣ୍ଟ ଦିଅନ୍ତୁ (ଜୀବନଭରି ୫୦% ଛାଡ଼, କିମ୍ବା ୩ ମାସ ମାଗଣା):

- ସାଇନଅପ୍ ଏବଂ ଆରମ୍ଭିକ ମାର୍ଗଦର୍ଶନ ଅନୁଭବ ଉପରେ ସତ୍ୟନିଷ୍ଠ ପ୍ରତିକ୍ରିୟା
- ସେମାନଙ୍କ ସାଇଟ୍‌କୁ ପ୍ରଦର୍ଶନୀ ଉଦାହରଣ ଭାବେ ବ୍ୟବହାର କରିବାର ଅନୁମତି
- ସେମାନେ ଦେଖିଥିବା କୌଣସି ବଗ୍ କିମ୍ବା ସମସ୍ୟା ରିପୋର୍ଟ କରିବା

ଏହା ଆପଣଙ୍କୁ ସାର୍ବଜନୀନ ଭାବେ ଖୋଲିବା ପୂର୍ବରୁ ସତ୍ୟ ଗ୍ରାହକ ପ୍ରତିକ୍ରିୟା ଏବଂ ପ୍ରଦର୍ଶନ ପାଇଁ ଲାଇଭ୍ ସାଇଟ୍ ଦେଇଥାଏ।

### ପର୍ଯ୍ୟାୟ ୨: ସାର୍ବଜନୀନ ଲଞ୍ଚ {#phase-2-public-launch}

ସଫ୍ଟ ଲଞ୍ଚ ପ୍ରତିକ୍ରିୟା ଅନ୍ତର୍ଭୁକ୍ତ କରିସାରିଲେ:

- ପେମେଣ୍ଟ ଗେଟୱେକୁ ଲାଇଭ୍ ମୋଡ୍‌କୁ ସ୍ୱିଚ୍ କରନ୍ତୁ
- ଆପଣଙ୍କ ମାର୍କେଟିଂ ସାଇଟ୍ ପ୍ରକାଶ କରନ୍ତୁ
- ଗ୍ରାହକ ଅଧିଗ୍ରହଣ ଆରମ୍ଭ କରନ୍ତୁ (ପାଠ ୧୧)
- ସମ୍ପର୍କିତ ଫିଟନେସ୍ ଶିଳ୍ପ ଚ୍ୟାନେଲ୍‌ରେ ଘୋଷଣା କରନ୍ତୁ

## ଲଞ୍ଚ ଦିନର କାର୍ଯ୍ୟ {#launch-day-actions}

ଯେଦିନ ଆପଣ ସାର୍ବଜନୀନ ହେବେ:

1. **ଲାଇଭ୍ ପେମେଣ୍ଟକୁ ସ୍ୱିଚ୍ କରନ୍ତୁ** -- ଆପଣଙ୍କ ପେମେଣ୍ଟ ଗେଟୱେରେ ପରୀକ୍ଷା ମୋଡ୍ ବନ୍ଦ କରନ୍ତୁ
2. **ଆଉ ଥରେ ଯାଞ୍ଚ କରନ୍ତୁ** -- ପ୍ରକୃତ ପେମେଣ୍ଟ ସହିତ ସମ୍ପୂର୍ଣ୍ଣ ପରୀକ୍ଷା ସାଇନଅପ୍ କରନ୍ତୁ (ପରେ ନିଜକୁ ରିଫଣ୍ଡ କରନ୍ତୁ)
3. **ନିକଟରୁ ମନିଟର୍ କରନ୍ତୁ** -- ତ୍ରୁଟି, ବିଫଳ ସାଇନଅପ୍, କିମ୍ବା ପେମେଣ୍ଟ ସମସ୍ୟା ଉପରେ ନଜର ରଖନ୍ତୁ
4. **ଉପଲବ୍ଧ ରୁହନ୍ତୁ** -- ଆପଣଙ୍କ ପ୍ରଥମ ପ୍ରକୃତ ଗ୍ରାହକମାନେ ସହାୟତା ଚାହିଁପାରନ୍ତି, ଏବଂ ଦ୍ରୁତ ପ୍ରତିକ୍ରିୟା ବିଶ୍ୱାସ ଗଢ଼େ
5. **ସଂକ୍ଷିପ୍ତ ଭାବେ ଉତ୍ସବ ପାଳନ କରନ୍ତୁ** -- ତାପରେ ପୁନି କାମକୁ ଫେରନ୍ତୁ

## କଣ ଭୁଲ୍ ହୋଇପାରେ {#what-can-go-wrong}

ଏଗୁଡ଼ିକ ପାଇଁ ପ୍ରସ୍ତୁତ ରୁହନ୍ତୁ:

- **ପେମେଣ୍ଟ ଗେଟୱେ ସମସ୍ୟା**: ଆପଣଙ୍କ ଗେଟୱେ ପ୍ରଦାନକାରୀଙ୍କ ସହାୟତା ସମ୍ପର୍କ ପ୍ରସ୍ତୁତ ରଖନ୍ତୁ
- **SSL ସର୍ଟିଫିକେଟ୍ ସମସ୍ୟା**: ସର୍ଟିଫିକେଟ୍ କିପରି ଯାଞ୍ଚ ଏବଂ ନବୀକରଣ କରିବେ ଜାଣିରଖନ୍ତୁ
- **ଇମେଲ୍ ଡେଲିଭରି ବିଫଳତା**: ଇମେଲ୍ ବାସ୍ତବରେ ପହଞ୍ଚୁଛି କି ନାହିଁ ପରୀକ୍ଷା କରନ୍ତୁ (ସ୍ପାମ୍ ଫୋଲ୍ଡର୍ ଯାଞ୍ଚ କରନ୍ତୁ)
- **ଲୋଡ୍ ଅଧୀନରେ କାର୍ଯ୍ୟଦକ୍ଷତା**: ଯଦି ଟ୍ରାଫିକ୍ ହଠାତ୍ ବଢ଼େ, ଆପଣଙ୍କ ହୋଷ୍ଟିଂ କିପରି ସ୍କେଲ୍ କରିବେ ଜାଣନ୍ତୁ
- **ଗ୍ରାହକ ଦ୍ୱନ୍ଦ୍ୱ**: ଆପଣଙ୍କ ଜ୍ଞାନଭଣ୍ଡାର ଏବଂ ସହାୟତା ଚ୍ୟାନେଲ୍ ପ୍ରସ୍ତୁତ ରଖନ୍ତୁ
