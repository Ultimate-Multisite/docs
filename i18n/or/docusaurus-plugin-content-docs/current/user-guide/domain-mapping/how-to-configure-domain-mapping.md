# ଡୋମେନ ମ୍ୟାପିଂ କିପରି ବିନ୍ୟାସ କରିବେ (v2)

_**ଗୁରୁତ୍ୱପୂର୍ଣ୍ଣ ଟିପ୍ପଣୀ: ଏହି ଲେଖା Ultimate Multisite version 2.x ବିଷୟରେ ଅଟେ।**_

ଏକ ପ୍ରିମିୟମ ନେଟୱର୍କର ସବୁଠାରୁ ଶକ୍ତିଶାଳୀ ବୈଶିଷ୍ଟ୍ୟଗୁଡ଼ିକ ମଧ୍ୟରୁ ଗୋଟିଏ ହେଉଛି ଆମ କ୍ଲାୟେଣ୍ଟମାନଙ୍କୁ ସେମାନଙ୍କ ସାଇଟରେ ଏକ ଟପ୍-ଲେଭେଲ ଡୋମେନ ଯୋଡ଼ିବାର ସୁଯୋଗ ଦେବା। ଶେଷରେ, କେଉଁଟି ଅଧିକ ପେଶାଦାର ଦେଖାଯାଏ: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) କିମ୍ବା [_**joesbikeshop.com**_](http://joesbikeshop.com)? ସେହିକାରଣରୁ Ultimate Multisite ତୃତୀୟ-ପକ୍ଷ plugin ବ୍ୟବହାର କରିବାର ଆବଶ୍ୟକତା ବିନା ଏହି ବୈଶିଷ୍ଟ୍ୟକୁ ଅନ୍ତର୍ନିହିତ ଭାବେ ଦେଇଥାଏ।

## ଡୋମେନ ମ୍ୟାପିଂ କ’ଣ? {#whats-domain-mapping}

ନାମରୁ ବୁଝାଯାଏ, ଡୋମେନ ମ୍ୟାପିଂ ହେଉଛି Ultimate Multisite ଦ୍ୱାରା ଦିଆଯାଇଥିବା କ୍ଷମତା, ଯାହା ଏକ କଷ୍ଟମ ଡୋମେନ ପାଇଁ ଅନୁରୋଧ ଗ୍ରହଣ କରି ସେହି ନିର୍ଦ୍ଦିଷ୍ଟ ଡୋମେନ ଯୋଡ଼ାଯାଇଥିବା ନେଟୱର୍କର ସମ୍ବନ୍ଧିତ ସାଇଟ ସହିତ ସେହି ଅନୁରୋଧକୁ ମ୍ୟାପ କରେ।

### ଆପଣଙ୍କ Ultimate Multisite Network ରେ ଡୋମେନ ମ୍ୟାପିଂ କିପରି ସେଟ୍ ଅପ୍ କରିବେ {#how-to-setup-domain-mapping-on-your-ultimate-multisite-network}

ଡୋମେନ ମ୍ୟାପିଂ କାମ କରିବା ପାଇଁ ଆପଣଙ୍କ ପକ୍ଷରୁ କିଛି ସେଟ୍ ଅପ୍ ଆବଶ୍ୟକ। ଧନ୍ୟବାଦ, Ultimate Multisite କଷ୍ଟକର କାମକୁ ଆପଣଙ୍କ ପାଇଁ ସ୍ୱୟଂଚାଳିତ କରେ, ଯାହାଫଳରେ ଆପଣ ସହଜରେ ଆବଶ୍ୟକତାଗୁଡ଼ିକ ପୂରଣ କରିପାରିବେ।

Ultimate Multisite ସ୍ଥାପନ ସମୟରେ, ୱିଜାର୍ଡ **sunrise.php** କୁ ନିର୍ଦ୍ଦିଷ୍ଟ ଫୋଲ୍ଡରକୁ ସ୍ୱୟଂଚାଳିତ ଭାବେ କପି କରି ସ୍ଥାପନ କରିବ। **ଏହି ପଦକ୍ଷେପ ସମ୍ପୂର୍ଣ୍ଣ ନହେଉପର୍ଯ୍ୟନ୍ତ ୱିଜାର୍ଡ ଆପଣଙ୍କୁ ଆଗକୁ ବଢ଼ିବାକୁ ଦେବ ନାହିଁ**।

<!-- Screenshot unavailable: Ultimate Multisite installation wizard with sunrise.php step -->

ଏହାର ଅର୍ଥ, Ultimate Multisite ସ୍ଥାପନ ୱିଜାର୍ଡ ଆପଣଙ୍କ ନେଟୱର୍କ ସେଟ୍ ଅପ୍ କରିବା ସମ୍ପୂର୍ଣ୍ଣ କରିଦେଲେ, ଆପଣ ସତେଜ କଷ୍ଟମ ଡୋମେନ ମ୍ୟାପ କରିବା ଆରମ୍ଭ କରିପାରିବେ।

ଧ୍ୟାନ ଦିଅନ୍ତୁ, Ultimate Multisite ରେ ଡୋମେନ ମ୍ୟାପିଂ ବାଧ୍ୟତାମୂଳକ ନୁହେଁ। ଆପଣଙ୍କ ପାଖରେ WordPress Multisite ର ନିଜସ୍ୱ ଡୋମେନ ମ୍ୟାପିଂ ଫଙ୍କସନ୍ କିମ୍ବା ଅନ୍ୟ କୌଣସି ଡୋମେନ ମ୍ୟାପିଂ ସମାଧାନ ବ୍ୟବହାର କରିବାର ବିକଳ୍ପ ଅଛି।

ଅନ୍ୟ ଡୋମେନ ମ୍ୟାପିଂ ସମାଧାନକୁ ସ୍ଥାନ ଦେବା ପାଇଁ ଯଦି ଆପଣ Ultimate Multisite ଡୋମେନ ମ୍ୟାପିଂକୁ ଅକ୍ଷମ କରିବାକୁ ଚାହାନ୍ତି, ତେବେ **Ultimate Multisite > ସେଟିଂସ୍ > ଡୋମେନ ମ୍ୟାପିଂ** ଅଧୀନରେ ଏହି ବୈଶିଷ୍ଟ୍ୟକୁ ଅକ୍ଷମ କରିପାରିବେ।

![ଆଡମିନ ରିଡାଇରେକ୍ଟ, ମ୍ୟାପିଂ ସନ୍ଦେଶ ଏବଂ DNS ବିକଳ୍ପ ଦେଖାଉଥିବା ଡୋମେନ ମ୍ୟାପିଂ ସେଟିଂସ୍ ପୃଷ୍ଠା](/img/config/domain-mapping-settings.png)

ଏହି ବିକଳ୍ପର ଠିକ୍ ତଳେ, ଆପଣ **Force Admin Redirect** ବିକଳ୍ପଟି ମଧ୍ୟ ଦେଖିପାରିବେ। ଏହି ବିକଳ୍ପ ଆପଣଙ୍କ ଗ୍ରାହକମାନେ ସେମାନଙ୍କ କଷ୍ଟମ ଡୋମେନ ଏବଂ ସବଡୋମେନ, ଉଭୟରେ ଆଡମିନ Dashboard ପ୍ରବେଶ କରିପାରିବେ କି ନାହିଁ, କିମ୍ବା କେବଳ ଗୋଟିଏରେ ପ୍ରବେଶ କରିପାରିବେ, ତାହା ନିୟନ୍ତ୍ରଣ କରିବାକୁ ଦିଏ।

ଯଦି ଆପଣ **Force redirect to mapped domain** ବାଛନ୍ତି, ଆପଣଙ୍କ ଗ୍ରାହକମାନେ କେବଳ ସେମାନଙ୍କ କଷ୍ଟମ ଡୋମେନରେ ଆଡମିନ Dashboard ପ୍ରବେଶ କରିପାରିବେ।

**Force redirect to** **network domain** ବିକଳ୍ପ ଠିକ୍ ବିପରୀତ କାମ କରିବ - ଆପଣଙ୍କ ଗ୍ରାହକମାନେ ସେମାନଙ୍କ କଷ୍ଟମ ଡୋମେନରେ ସାଇନ୍ ଇନ୍ କରିବାକୁ ଚେଷ୍ଟା କଲେ ମଧ୍ୟ, କେବଳ ସବଡୋମେନରେ ସେମାନଙ୍କ Dashboard ପ୍ରବେଶ କରିବାକୁ ଅନୁମତି ପାଇବେ।

ଏବଂ **Allow access to the admin by both mapped domain domain and network domain** ବିକଳ୍ପ ସେମାନଙ୍କୁ ସବଡୋମେନ ଏବଂ କଷ୍ଟମ ଡୋମେନ, ଉଭୟରେ ଆଡମିନ Dashboard ପ୍ରବେଶ କରିବାକୁ ଦିଏ।

![ତିନୋଟି ରିଡାଇରେକ୍ଟ ବିକଳ୍ପ ଦେଖାଉଥିବା ବିସ୍ତାରିତ ଆଡମିନ ରିଡାଇରେକ୍ଟ ଡ୍ରପଡାଉନ](/img/config/domain-mapping-redirect-options.png)

କଷ୍ଟମ ଡୋମେନ ମ୍ୟାପ କରିବାର ଦୁଇଟି ଉପାୟ ଅଛି। ପ୍ରଥମଟି ହେଉଛି ସୁପର ଆଡମିନ ଭାବେ ଆପଣଙ୍କ ନେଟୱର୍କ ଆଡମିନ Dashboard ରୁ ଡୋମେନ ନାମ ମ୍ୟାପ କରିବା, ଏବଂ ଦ୍ୱିତୀୟଟି Account ପୃଷ୍ଠା ଅଧୀନରେ ସବସାଇଟ ଆଡମିନ Dashboard ମାଧ୍ୟମରେ।

କିନ୍ତୁ ଆପଣଙ୍କ ନେଟୱର୍କର କୌଣସି ଗୋଟିଏ ସବସାଇଟକୁ କଷ୍ଟମ ଡୋମେନ ମ୍ୟାପ କରିବା ଆରମ୍ଭ କରିବା ପୂର୍ବରୁ, ଡୋମେନ ନାମର **DNS settings** ଠିକ୍ ଭାବରେ ବିନ୍ୟାସ ହୋଇଛି ବୋଲି ନିଶ୍ଚିତ କରିବାକୁ ପଡ଼ିବ।

###

### ଡୋମେନ DNS ସେଟିଂସ୍ ଠିକ୍ ଭାବରେ ବିନ୍ୟାସ ହୋଇଛି ବୋଲି ନିଶ୍ଚିତ କରିବା {#making-sure-the-domain-dns-settings-are-properly-configured}

ମ୍ୟାପିଂ କାମ କରିବା ପାଇଁ, ଆପଣ ମ୍ୟାପ କରିବାକୁ ଯୋଜନା କରୁଥିବା ଡୋମେନ ଆପଣଙ୍କ Network ର IP address କୁ ସୂଚିତ କରୁଛି ବୋଲି ନିଶ୍ଚିତ କରିବା ଆବଶ୍ୟକ। ଧ୍ୟାନ ଦିଅନ୍ତୁ, ଆପଣଙ୍କୁ Network IP address ଆବଶ୍ୟକ - ଯେଉଁ ଡୋମେନରେ Ultimate Multisite ସ୍ଥାପିତ ଅଛି ସେହି ଡୋମେନର IP address - ଆପଣ ମ୍ୟାପ କରିବାକୁ ଚାହୁଁଥିବା କଷ୍ଟମ ଡୋମେନର IP address ନୁହେଁ। ନିର୍ଦ୍ଦିଷ୍ଟ ଡୋମେନର IP address ଖୋଜିବା ପାଇଁ, ଉଦାହରଣ ସ୍ୱରୂପ ଆମେ [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) କୁ ଯିବାକୁ ପରାମର୍ଶ ଦେଉଛୁ।

ଡୋମେନକୁ ଠିକ୍ ଭାବରେ ମ୍ୟାପ କରିବା ପାଇଁ, ଆପଣଙ୍କ **DNS** ବିନ୍ୟାସରେ ସେହି **IP address** କୁ ସୂଚିତ କରୁଥିବା ଏକ **A RECORD** ଯୋଡ଼ିବା ଆବଶ୍ୟକ। ଭିନ୍ନ ଡୋମେନ ରେଜିଷ୍ଟ୍ରାରମାନଙ୍କ ମଧ୍ୟରେ DNS ପରିଚାଳନା ବହୁତ ଭିନ୍ନ ହୁଏ, କିନ୍ତୁ ଆପଣ " _Creating A Record on XXXX_ " ଖୋଜିଲେ, ଯେଉଁଠାରେ XXXX ହେଉଛି ଆପଣଙ୍କ ଡୋମେନ ରେଜିଷ୍ଟ୍ରାର (ଉଦାହରଣ: " _Creating A Record on_ _GoDaddy_ "), ଏହାକୁ ନେଇ ଅନଲାଇନରେ ଅନେକ ଟ୍ୟୁଟୋରିଆଲ ମିଳିବ।

ଯଦି ଏହାକୁ କାମ କରାଇବାରେ ଆପଣଙ୍କୁ ସମସ୍ୟା ହେଉଛି, **ଆପଣଙ୍କ ଡୋମେନ ରେଜିଷ୍ଟ୍ରାର ସପୋର୍ଟ ସହିତ ଯୋଗାଯୋଗ କରନ୍ତୁ** ଏବଂ ସେମାନେ ଏହି ଅଂଶରେ ଆପଣଙ୍କୁ ସହାୟତା କରିପାରିବେ।

ଯଦି ଆପଣ ଆପଣଙ୍କ କ୍ଲାୟେଣ୍ଟମାନଙ୍କୁ ସେମାନଙ୍କ ନିଜ ଡୋମେନ ମ୍ୟାପ କରିବାକୁ ଅନୁମତି ଦେବାକୁ ଯୋଜନା କରୁଛନ୍ତି, ତେବେ ଏହି ଅଂଶର କାମ ସେମାନଙ୍କୁ ନିଜେ କରିବାକୁ ପଡ଼ିବ। ଯଦି ସେମାନେ A Record ତିଆରି କରିପାରୁନାହାନ୍ତି, ତେବେ ସେମାନଙ୍କୁ ନିଜ ରେଜିଷ୍ଟ୍ରାର ସପୋର୍ଟ ସିଷ୍ଟମ୍ ଦିଗରେ ଦେଖାନ୍ତୁ।

### ସୁପର ଆଡମିନ ଭାବେ କଷ୍ଟମ ଡୋମେନ ନାମ ମ୍ୟାପ କରିବା {#mapping-custom-domain-name-as-super-admin}

ଆପଣଙ୍କ ନେଟୱର୍କରେ ସୁପର ଆଡମିନ ଭାବେ ଲଗ୍ ଇନ୍ କରିଥିବାବେଳେ, **Ultimate Multisite > ଡୋମେନଗୁଡ଼ିକ** ଅଧୀନକୁ ଯାଇ ଆପଣ ସହଜରେ କଷ୍ଟମ ଡୋମେନ ନାମ ଯୋଡ଼ି ଏବଂ ପରିଚାଳନା କରିପାରିବେ।

![Ultimate Multisite ରେ ଡୋମେନ ତାଲିକା ପୃଷ୍ଠା](/img/admin/domains-list.png)

ଏହି ପୃଷ୍ଠାରେ, ଆପଣ ଉପରେ ଥିବା **Add Domain** ବଟନ୍ କ୍ଲିକ୍ କରିପାରିବେ ଏବଂ ଏଥିରୁ ଏକ ମୋଡାଲ ୱିଣ୍ଡୋ ଖୋଲିବ, ଯେଉଁଠାରେ ଆପଣ **custom domain name** , ଆପଣ କଷ୍ଟମ ଡୋମେନ ନାମ ଲାଗୁ କରିବାକୁ ଚାହୁଁଥିବା **the subsite** ସେଟ୍ କରି ଭରିପାରିବେ, ଏବଂ ଏହାକୁ **primary domain** ନାମ ଭାବେ ସେଟ୍ କରିବେ କି ନାହିଁ ନିଷ୍ପତ୍ତି କରିପାରିବେ (ଧ୍ୟାନ ଦିଅନ୍ତୁ, ଆପଣ **multiple domain names to one subsite** ମ୍ୟାପ କରିପାରିବେ)।

![ଡୋମେନ ନାମ, ସାଇଟ ବାଛନ୍ତା ଏବଂ ପ୍ରାଥମିକ ଡୋମେନ ଟଗଲ୍ ସହିତ Add Domain ମୋଡାଲ](/img/admin/domain-add-modal.png)

ସମସ୍ତ ସୂଚନା ଭରିବା ପରେ, ଆପଣ ତଳେ ଥିବା **Add Existing Domain** ବଟନ୍ କ୍ଲିକ୍ କରିପାରିବେ।

ଏହା କଷ୍ଟମ ଡୋମେନର DNS ସୂଚନା ଯାଞ୍ଚ କରିବା ଏବଂ ଆଣିବା ପ୍ରକ୍ରିୟା ଆରମ୍ଭ କରିବ। ପ୍ରକ୍ରିୟା କେଉଁ ଧାପରେ ଅଛି ତାହା ଅନୁସରଣ କରିବା ପାଇଁ ଆପଣ ପୃଷ୍ଠାର ତଳେ ଏକ ଲଗ୍ ମଧ୍ୟ ଦେଖିବେ। ଏହି ପ୍ରକ୍ରିୟା ସମ୍ପୂର୍ଣ୍ଣ ହେବାକୁ କିଛି ମିନିଟ୍ ଲାଗିପାରେ।

Ultimate Multisite v2.13.0 ମଧ୍ୟ ନୂଆ ସାଇଟ୍ ତିଆରି ହେଲେ, ଯେଉଁ ହୋଷ୍ଟକୁ ପ୍ରତି-ସାଇଟ୍ ଡୋମେନ ଭାବେ ଗଣନା କରିବା ଉଚିତ, ସେଠାରେ ଭିତରୀଣ ଡୋମେନ ରେକର୍ଡ ସ୍ୱୟଂଚାଳିତ ଭାବେ ତିଆରି କରେ। ଯଦି ହୋଷ୍ଟଟି ନେଟୱର୍କର ପ୍ରଧାନ ଡୋମେନ, କିମ୍ବା **Site URL** ଫିଲ୍ଡରେ ସେଟ୍ କରାଯାଇଥିବା ସେୟାର୍ଡ ଚେକଆଉଟ୍-ଫର୍ମ ବେସ୍ ଡୋମେନମାନଙ୍କ ମଧ୍ୟରୁ ଗୋଟିଏ ହୁଏ, ତେବେ ସ୍ୱୟଂଚାଳିତ ମ୍ୟାପ୍ଡ-ଡୋମେନ ରେକର୍ଡକୁ ଛାଡ଼ି ଦିଆଯାଏ, ଯାହାଫଳରେ ସେହି ସେୟାର୍ଡ ବେସ୍ ଡୋମେନ ଏହା ବ୍ୟବହାର କରୁଥିବା ପ୍ରତ୍ୟେକ ସାଇଟ୍ ପାଇଁ ଉପଲବ୍ଧ ରହେ।

ଯେତେବେଳେ ଗ୍ରାହକ Domain Seller v1.3.0 କିମ୍ବା ତାଠାରୁ ନୂଆ ସଂସ୍କରଣ ମାଧ୍ୟମରେ ନୂଆ ଡୋମେନ ନିବନ୍ଧନ କରନ୍ତି, Ultimate Multisite ଡିଫଲ୍ଟ ଭାବେ ନିବନ୍ଧିତ ଡୋମେନକୁ ଗ୍ରାହକଙ୍କ ନେଟୱର୍କ ସାଇଟ୍ ସହିତ ସ୍ୱୟଂଚାଳିତ ଭାବେ ମ୍ୟାପ୍ କରେ। ସଫଳ ନିବନ୍ଧନ ପରେ ପ୍ରଶାସକମାନଙ୍କୁ ଆଉ ଅଲଗା ମ୍ୟାପ୍ଡ-ଡୋମେନ ରେକର୍ଡ ଯୋଡ଼ିବା ଦରକାର ନାହିଁ, ଯଦିନା ସେମାନେ ପ୍ରଧାନ-ଡୋମେନ ଫ୍ଲାଗ୍, ସକ୍ରିୟତା ଅବସ୍ଥା, କିମ୍ବା SSL ପରିଚାଳନା ଭଳି ବିକଳ୍ପ ସମଯୋଜନ କରିବାକୁ ଚାହାନ୍ତି।

ସବୁକିଛି ଠିକ୍ ଭାବେ ସେଟ୍ ଅପ୍ ହୋଇଥିଲେ **ପର୍ଯ୍ୟାୟ** କିମ୍ବା ଅବସ୍ଥା **DNS ଯାଞ୍ଚ ହେଉଛି** ରୁ **ପ୍ରସ୍ତୁତ** କୁ ବଦଳିବା ଉଚିତ।

<!-- ସ୍କ୍ରିନସଟ୍ ଉପଲବ୍ଧ ନାହିଁ: ସବୁଜ ଅବସ୍ଥା ସୂଚକ ସହିତ ପ୍ରସ୍ତୁତ ପର୍ଯ୍ୟାୟ ଦେଖାଉଥିବା ଡୋମେନ ଧାଡ଼ି -->

ଆପଣ ଡୋମେନ ନାମରେ କ୍ଲିକ୍ କଲେ, ତାହାର ଭିତରେ କିଛି ବିକଳ୍ପ ଦେଖିପାରିବେ। ସେଗୁଡ଼ିକୁ ଶୀଘ୍ର ଦେଖିନେବା:

![ପର୍ଯ୍ୟାୟ, ସାଇଟ୍, ସକ୍ରିୟ, ପ୍ରଧାନ ଏବଂ SSL ଟଗଲ୍ ସହିତ ଡୋମେନ ବିବରଣୀ ପୃଷ୍ଠା](/img/admin/domain-edit.png)

**ପର୍ଯ୍ୟାୟ:** ଏହା ହେଉଛି ଡୋମେନ ଥିବା ପର୍ଯ୍ୟାୟ। ଆପଣ ପ୍ରଥମେ ଡୋମେନ ଯୋଡ଼ିଲେ, ସମ୍ଭବତଃ ଏହା **DNS ଯାଞ୍ଚ ହେଉଛି** ପର୍ଯ୍ୟାୟରେ ରହିବ। ପ୍ରକ୍ରିୟାଟି DNS ଏଣ୍ଟ୍ରିଗୁଡ଼ିକୁ ଯାଞ୍ଚ କରି ସେଗୁଡ଼ିକ ଠିକ୍ ଅଛି ବୋଲି ନିଶ୍ଚିତ କରିବ। ପରେ ଡୋମେନକୁ **SSL ଯାଞ୍ଚ ହେଉଛି** ପର୍ଯ୍ୟାୟରେ ରଖାଯିବ। Ultimate Multisite ଡୋମେନରେ SSL ଅଛି କି ନାହିଁ ଯାଞ୍ଚ କରିବ ଏବଂ ଆପଣଙ୍କ ଡୋମେନକୁ **ପ୍ରସ୍ତୁତ** କିମ୍ବା **ପ୍ରସ୍ତୁତ (SSL ବିନା)** ଭାବେ ବର୍ଗୀକୃତ କରିବ।

**ସାଇଟ୍:** ଏହି ଡୋମେନ ସହିତ ସଂପୃକ୍ତ ସବଡୋମେନ। ମ୍ୟାପ୍ଡ ଡୋମେନ ଏହି ନିର୍ଦ୍ଦିଷ୍ଟ ସାଇଟ୍‌ର ବିଷୟବସ୍ତୁ ଦେଖାଇବ।

**ସକ୍ରିୟ:** ଡୋମେନକୁ ସକ୍ରିୟ କିମ୍ବା ନିଷ୍କ୍ରିୟ କରିବାକୁ ଆପଣ ଏହି ବିକଳ୍ପକୁ ଅନ୍ କିମ୍ବା ଅଫ୍ କରିପାରିବେ।

**ଏହା ପ୍ରଧାନ ଡୋମେନ କି?:** ଆପଣଙ୍କ ଗ୍ରାହକମାନେ ପ୍ରତ୍ୟେକ ସାଇଟ୍ ପାଇଁ ଏକାଧିକ ମ୍ୟାପ୍ଡ ଡୋମେନ ରଖିପାରନ୍ତି। ନିର୍ଦ୍ଦିଷ୍ଟ ସାଇଟ୍ ପାଇଁ ଏହା ପ୍ରଧାନ ଡୋମେନ କି ନୁହେଁ ଚୟନ କରିବାକୁ ଏହି ବିକଳ୍ପ ବ୍ୟବହାର କରନ୍ତୁ।

**ଏହା ସୁରକ୍ଷିତ କି?:** Ultimate Multisite ଡୋମେନକୁ ସକ୍ରିୟ କରିବା ପୂର୍ବରୁ ତାହାର SSL ସର୍ଟିଫିକେଟ୍ ଅଛି କି ନାହିଁ ଯାଞ୍ଚ କରିଥିଲେ ମଧ୍ୟ, ଆପଣ ହାତେହାତେ SSL ସର୍ଟିଫିକେଟ୍ ସହିତ କିମ୍ବା ବିନା ଡୋମେନ ଲୋଡ୍ କରିବାକୁ ଚୟନ କରିପାରିବେ। ଧ୍ୟାନ ଦିଅନ୍ତୁ, ୱେବସାଇଟ୍‌ରେ SSL ସର୍ଟିଫିକେଟ୍ ନଥିଲେ ଏବଂ ଆପଣ ଏହାକୁ SSL ସହିତ ବଳପୂର୍ବକ ଲୋଡ୍ କରିବାକୁ ଚେଷ୍ଟା କଲେ, ଏହା ଆପଣଙ୍କୁ ତ୍ରୁଟି ଦେଇପାରେ।

### ସବସାଇଟ୍ ବ୍ୟବହାରକାରୀ ଭାବେ କଷ୍ଟମ୍ ଡୋମେନ ନାମ ମ୍ୟାପ୍ କରିବା {#mapping-custom-domain-name-as-subsite-user}

ସବସାଇଟ୍ ପ୍ରଶାସକମାନେ ମଧ୍ୟ ସେମାନଙ୍କ ସବସାଇଟ୍ ପ୍ରଶାସନ Dashboard ରୁ କଷ୍ଟମ୍ ଡୋମେନ ନାମ ମ୍ୟାପ୍ କରିପାରିବେ।

ପ୍ରଥମେ, ଆପଣଙ୍କୁ **ଡୋମେନ ମ୍ୟାପିଂ** ସେଟିଂସ୍‌ରେ ଏହି ବିକଳ୍ପ ସକ୍ରିୟ କରିଛନ୍ତି ବୋଲି ନିଶ୍ଚିତ କରିବା ଦରକାର। ତଳେ ଥିବା ସ୍କ୍ରିନସଟ୍ ଦେଖନ୍ତୁ।

<!-- ସ୍କ୍ରିନସଟ୍ ଉପଲବ୍ଧ ନାହିଁ: Customer DNS Management ଟଗଲ୍ ମାଧ୍ୟମରେ ସବସାଇଟ୍ ବ୍ୟବହାରକାରୀଙ୍କୁ ଡୋମେନ ମ୍ୟାପ୍ କରିବାକୁ ଅନୁମତି ଦେଉଥିବା ଡୋମେନ ମ୍ୟାପିଂ ସେଟିଂସ୍ -->

ଆପଣ **Ultimate Multisite > ଉତ୍ପାଦଗୁଡ଼ିକ** ରେ **ଯୋଜନା** ସ୍ତର କିମ୍ବା ଉତ୍ପାଦ ବିକଳ୍ପମାନଙ୍କ ଅଧୀନରେ ମଧ୍ୟ ଏହି ବିକଳ୍ପ ସେଟ୍ କିମ୍ବା କନଫିଗର୍ କରିପାରିବେ।

![ଉତ୍ପାଦ ସମ୍ପାଦନ ପୃଷ୍ଠାରେ କଷ୍ଟମ୍ ଡୋମେନ ବିଭାଗ](/img/config/product-custom-domains.png)

ସେହି ବିକଳ୍ପମାନଙ୍କ ମଧ୍ୟରୁ କୌଣସିଟି ସକ୍ରିୟ ହେଲେ ଏବଂ ସବସାଇଟ୍ ବ୍ୟବହାରକାରୀଙ୍କୁ କଷ୍ଟମ୍ ଡୋମେନ ନାମ ମ୍ୟାପ୍ କରିବାକୁ ଅନୁମତି ଦିଆଗଲେ, ସବସାଇଟ୍ ବ୍ୟବହାରକାରୀ **Account** ପୃଷ୍ଠାରେ **ଡୋମେନଗୁଡ଼ିକ** ନାମର ଏକ ମେଟାବକ୍ସ ଦେଖିବା ଉଚିତ।

ବ୍ୟବହାରକାରୀ **ଡୋମେନ ଯୋଡ଼ନ୍ତୁ** ବଟନ୍ କ୍ଲିକ୍ କରିପାରିବେ ଏବଂ ଏହା କିଛି ନିର୍ଦ୍ଦେଶ ସହିତ ଏକ ମୋଡାଲ୍ ୱିଣ୍ଡୋ ଖୋଲିବ।

ତାପରେ ବ୍ୟବହାରକାରୀ **ପରବର୍ତ୍ତୀ ପଦକ୍ଷେପ** କ୍ଲିକ୍ କରି କଷ୍ଟମ୍ ଡୋମେନ ନାମ ଯୋଡ଼ିବାକୁ ଆଗକୁ ବଢ଼ିପାରିବେ। ସେମାନେ ଏହା ପ୍ରଧାନ ଡୋମେନ ହେବ କି ନାହିଁ ମଧ୍ୟ ଚୟନ କରିପାରିବେ।

**ଡୋମେନ ଯୋଡ଼ନ୍ତୁ** କ୍ଲିକ୍ କଲେ କଷ୍ଟମ୍ ଡୋମେନର DNS ସୂଚନା ସତ୍ୟାପନ ଏବଂ ଆଣିବା ପ୍ରକ୍ରିୟା ଆରମ୍ଭ ହେବ।

### ଡୋମେନ ସିଙ୍କିଂ ବିଷୟରେ {#about-domain-syncing}

ଡୋମେନ ସିଙ୍କିଂ ହେଉଛି ଏକ ପ୍ରକ୍ରିୟା, ଯେଉଁଠାରେ Ultimate Multisite ଆପଣଙ୍କ ହୋଷ୍ଟିଂ ଆକାଉଣ୍ଟରେ କଷ୍ଟମ୍ ଡୋମେନ ନାମକୁ ଏକ ଆଡ୍-ଅନ୍ ଡୋମେନ ଭାବେ ଯୋଡ଼େ, **ଯାହାଦ୍ୱାରା ଡୋମେନ ମ୍ୟାପିଂ କାମ କରିପାରେ**।

ଆପଣଙ୍କ ହୋଷ୍ଟିଂ ପ୍ରଦାନକାରୀଙ୍କ Ultimate Multisite ଡୋମେନ ମ୍ୟାପିଂ ବୈଶିଷ୍ଟ୍ୟ ସହିତ ଇଣ୍ଟିଗ୍ରେସନ୍ ଥିଲେ ଡୋମେନ ସିଙ୍କିଂ ସ୍ୱୟଂଚାଳିତ ଭାବେ ହୁଏ। ବର୍ତ୍ତମାନ, ଏହି ହୋଷ୍ଟିଂ ପ୍ରଦାନକାରୀମାନେ ହେଲେ _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ ଏବଂ _Cpanel._

ହୋଷ୍ଟ-ପ୍ରଦାନକାରୀ ଇଣ୍ଟିଗ୍ରେସନ୍ ସକ୍ରିୟ ଥିଲେ, Ultimate Multisite ନୂଆ ତିଆରି ସାଇଟ୍‌ଗୁଡ଼ିକ ପାଇଁ ପ୍ରଦାନକାରୀ-ପକ୍ଷ DNS କିମ୍ବା ସବଡୋମେନ ତିଆରି କାମକୁ ମଧ୍ୟ କ୍ୟୁରେ ଯୋଡ଼ିପାରେ। ସେହି କାମ ପାଇଁ କୌଣସି ଇଣ୍ଟିଗ୍ରେସନ୍ ଶୁଣୁନଥିଲେ, ଅକାର୍ଯ୍ୟ କ୍ୟୁ ଏଣ୍ଟ୍ରି ଏଡ଼ାଇବାକୁ ବ୍ୟାକଗ୍ରାଉଣ୍ଡ କାମକୁ ଛାଡ଼ି ଦିଆଯାଏ। ମ୍ୟାପ୍ଡ ଡୋମେନ ପାଇଁ DNS ଏବଂ SSL ଯାଞ୍ଚ ସାଧାରଣ ଡୋମେନ-ପର୍ଯ୍ୟାୟ ପ୍ରକ୍ରିୟା ମାଧ୍ୟମରେ ଚାଲୁ ରହେ।

ଆପଣଙ୍କୁ Ultimate Multisite ସେଟିଂସ୍‌ରେ **ଇଣ୍ଟିଗ୍ରେସନ୍** ଟ୍ୟାବ୍‌ର ଅଧୀନରେ ଏହି ଇଣ୍ଟିଗ୍ରେସନ୍ ସକ୍ରିୟ କରିବାକୁ ପଡ଼ିବ।
