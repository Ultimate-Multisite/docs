# ચેકઆઉટ ફિલ્ડ અને ગ્રાહક DNS મેનેજમેન્ટ {#checkout-field-and-customer-dns-management}

## ડોમેન સિલેક્શન ચેકઆઉટ ફિલ્ડ {#the-domain-selection-checkout-field}

**Domain Selection** ફિલ્ડ એ એક ચેકઆઉટ એલિમેન્ટ છે જે ગ્રાહકોને તેમની સાઇટ માટે ડોમેન મેળવવાની રીત પસંદ કરવાની તક આપે છે. ડોમેન વેચવાનું સક્ષમ કરવા માટે તેને કોઈપણ ચેકઆઉટ ફોર્મમાં ઉમેરો.

### ચેકઆઉટ ફોર્મમાં ફિલ્ડ ઉમેરવી {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › Checkout Forms** પર જાઓ
2. એક ચેકઆઉટ ફોર્મ ખોલો અથવા બનાવો
3. ચેકઆઉટ એડિટર (editor) માં, **Add Field** પર ક્લિક કરો
4. ફિલ્ડ લિસ્ટમાંથી **Domain Selection** પસંદ કરો
5. ફિલ્ડના વિકલ્પો સેટ કરો (નીચે જુઓ)
6. ફોર્મ સેવ કરો

### ફિલ્ડના વિકલ્પો {#field-options}

**Domain modes** — ગ્રાહક કયા ટેબ જોશે તે પસંદ કરો. દરેક મોડને સ્વતંત્ર રીતે ચાલુ અથવા બંધ કરી શકાય છે:

| Mode | શું કરે છે |
|---|---|
| **Subdomain** | ગ્રાહક તમારા નેટવર્ક પર એક ફ્રી સબડોમેનનો ઉપયોગ કરે છે (દા.ત., `mysite.yournetwork.com`). કોઈ ચુકવણીની જરૂર નથી. |
| **Register New Domain** | ગ્રાહક એક નવું ડોમેન શોધે છે અને તેને તમારા konfigured provider દ્વારા રજિસ્ટર કરે છે. કિંમત નક્કી કરવા માટે મેચિંગ ડોમેન પ્રોડક્ટનો ઉપયોગ થાય છે. |
| **Existing Domain** | ગ્રાહક પહેલેથી જ માલિકીનું ડોમેન મેપ કરે છે. કોઈ નોંધણી ફી નથી. ડોમેન આપોઆપ તેમની સાઇટ સાથે મેપ થઈ જાય છે. |

**Default mode** — જ્યારે ત્રણેય મોડ ચાલુ હોય, ત્યારે કયું ટેબ પહેલા ખુલશે તે સેટ કરો. ડોમેન રજિસ્ટ્રેશનને વૈકલ્પિક રાખવા માટે તેને **Subdomain** પર સેટ કરો, અથવા ખરીદીઓને પ્રોત્સાહિત કરવા માટે **Register New Domain** પર સેટ કરો.

**Domain product** — આ ફિલ્ડને ચોક્કસ ડોમેન પ્રોડક્ટ સાથે જોડી શકાય છે. જો સેટ ન હોય, તો એડ-ઓન આપોઆપ ગ્રાહક જે TLD (Top Level Domain) શોધે તેના આધારે મેચિંગ પ્રોડક્ટ પસંદ કરશે.

### રજિસ્ટ્રન્ટ સંપર્ક ફિલ્ડ્સ {#registrant-contact-fields}

જ્યારે કોઈ ગ્રાહક **Register New Domain** ટેબ પસંદ કરે છે, ત્યારે ચેકઆઉટ ફોર્મમાં નીચેની રજિસ્ટ્રન્ટ સંપર્ક ફિલ્ડ્સ ઓનલાઈન ઉમેરાય છે:

- પ્રથમ નામ / અટક
- ઈમેલ એડ્રેસ
- સરનામું (લાઇન ૧, શહેર, રાજ્ય/પ્રાંત, પિન કોડ, દેશ)
- ફોન નંબર

ସବୁ ରେଜିଷ୍ଟ୍ରାରଙ୍କ ପାଇଁ ଏହା ଆବଶ୍ୟକ ଏବଂ ରେଜିଷ୍ଟ୍ରେସନ୍ API କଲ୍ ମଞ୍ଜୁର କରିବା ପୂର୍ବ ଭାଟି ଯାଞ୍ଚ କରାଯାଏ। ଫୋନ୍ ନମ୍ବରଗୁଡ଼ିକ ସ୍ୱୟଂଚିତ ଭାବରେ ରେଜିଷ୍ଟ୍ରାରମାନଙ୍କ ଦ୍ୱାରା ଅପେକ୍ଷା କରାଯାଉଥିବା `+CC.NNN` ଆନ୍ତର୍ଜାତିକ ଫର୍ମାଟ୍‌କୁ ନିୟୋଜିତ ହୋଇଥାଏ।

### ସ୍ୱୟଂଚିତ साईट URL (Auto-generated site URL) {#auto-generated-site-url}

ଯେତେବେଳେ କୌଣସି ଗ୍ରାହକ ରେଜିଷ୍ଟର କରନ୍ତି ବା ଏକ ଡୋମେନ୍ ମ୍ୟାପ୍ କରନ୍ତି, ସାଇଟ୍ URL କ୍ଷେତ୍ରଟି ଚୟନ କରାଯାଇଥିବା ଡୋମେନ୍ ଆଧାରରେ ସ୍ୱୟଂଚିତ ଭାବରେ ପୂରଣ ହୁଏ। ଗ୍ରାହକମାନଙ୍କୁ ଏକ ଅଲଗା URL କ୍ଷେତ୍ର ଭରିବାର ଆବଶ୍ୟକ ନାହିଁ।

### ଖୋଜିବା ବିଭାଗ (Search behaviour) {#search-behaviour}

- ଗ୍ରାହକ ଯେପରି টাইପ୍ କରନ୍ତି, ସେଥିରେ AJAX ମାଧ୍ୟମରେ ଡୋମେନ୍ లభ్యତା ରିଅଲ୍-ଟାଇମ୍ ରେ ଯାଞ୍ଚ କରାଯାଏ।
- ଯଦି ପ୍ରିୟ ଡୋମେନ୍ ଅବଲୈଭୀ ହୁଏ, ଉପସ୍ଥିତ ବିକଳ୍ପ TLD (Top-Level Domain) ସୂଚନା ଦିଆଯାଏ।
- ମୂଲ୍ୟ ଲାଇଭ୍‌ରେ ଆଣାଯାଏ ଏବଂ ସ୍ପଷ୍ଟ ଭାବରେ ପ୍ରଦର୍ଶିତ ହୁଏ (ରେଜିଷ୍ଟ୍ରେସନ୍ ମୂଲ୍ୟ, ନବୀକରଣ ମୂଲ୍ୟ, ଅପ୍ସନାଲ WHOIS ପ୍ରାଇଭେସି ଫିଅ)।
- କଉପନ୍ କୋଡ୍ଗୁଡ଼ିକ ଅନ୍ୟ କୌଣସି ଉତ୍ପାଦ ପରି ଡୋମେନ୍ ଉତ୍ପାଦలపై ସେଇଠାରୁ ଆବଲମ୍ବିତ ହୁଏ।

**ଖୋଜିବା ପ୍ରତିକ୍ରିୟାକୁ ଟ୍ୟୁନିଂ କରିବା (Tuning search responsiveness):**

```php
// କ୍ଷମତାକ୍ଷମ ଅସୁବିଧା ସହିତ API କଲ୍ ବହୁତ ହ୍ରାସ କରିବା ପାଇଁ debouncing ବିଳୟ (milliseconds) ବୃଦ୍ଧି କରନ୍ତୁ
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ଡୋମେନ୍ ସର୍ଚ୍ଚ ଫର୍ମରେ କଷ୍ଟମ୍ କ୍ଷେତ୍ର ଯୋଗ କରିବା (Adding custom fields to the domain search form):**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'ଅତିରିକ୍ତ ନୋଟ୍ସ (Additional notes)',
    ];
    return $fields;
});
```

## ଗ୍ରାହକ DNS ପରିଚାଳନା (Customer DNS management) {#customer-dns-management}

ଗ୍ରାହକମାନେ ସେମାନଙ୍କ ରେଜିଷ୍ଟ୍ରେଡ୍ ଡୋମେନ୍‌ଗୁଡ଼ିକର DNS ରେକର୍ଡଗୁଡ଼ିକୁ **My Account** ପୃଷ୍ଠାରୁ, ସେମାନଙ୍କ ଡୋମେନ୍ ପ୍ରବେଶ ଅଧୀନରେ ପରିଚାଳନା କରିପାରନ୍ତି।

### ସମର୍ଥନୀୟ ରେକର୍ଡ ପ୍ରକାର (Supported record types) {#supported-record-types}

| ପ୍ରକାର | ବ୍ୟବହାର |
|---|---|
| **A** | ନାମକୁ IPv4 ଆଡ୍ରେସ ସହିତ ମ୍ୟାପ୍ କରିବା |
| **AAAA** | ନାମକୁ IPv6 ଆଡ୍ରେସ ସହିତ ମ୍ୟାପ୍ କରିବା |
| **CNAME** | ଅନ୍ୟ ଏକ ନାମକୁ ପଏଣ୍ଟ କରୁଥିବା ଉପସ୍ଥିତି (alias) ସୃଷ୍ଟି କରିବା |
| **MX** | ମେଲ୍ ଏକ୍ସଚେଞ୍ଜ ସର୍ଭର ସେଟ୍ କରିବା |
| **TXT** | SPF, DMARC, ଯାଞ୍ଚ କିମ୍ବା ଅନ୍ୟ କୌଣସି ଲେଖାକାରୀ (text) ରେକର୍ଡ ଯୋଗ

DNS ମ୍ୟାନେଜମେଣ୍ଟ (ରେକର୍ଡ୍ ଯୋଗել, ସଂଶୋଧନ କରିବା, ମିୂଳିବା) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, ଏବଂ **Openprovider** ସହିତ ଉପଲବ୍ଧ। **Hostinger** ডମେନ୍‌କୁ Domain Seller ମାଧ୍ୟମରେ ନେମ୍ସର୍ଭର ଅପଡେଟ୍ କରିପାରିବେ; ହୋଷ୍ଟ୍ କରାଯାଇଥିବା ଡମେନ୍ ପାଇଁ DNS ରେକର୍ଡ୍ଗୁଡ଼ିକ କୋର୍ (core) Hostinger ଡମେନ୍-ମ୍ୟାପିଂ ଇଣ୍ଟିଗ୍ରେସନ ଦ୍ୱାରା ମ୍ୟାନେଜ୍ ହୁଏ। Namecheap, GoDaddy, ଏବଂ NameSilo ଡମେନ୍ ସ୍ଥିତି ଏବଂ ମେକ୍ଷିକା (expiry) ସୂଚନା ଦର୍ଶାନ୍ତି କିନ୍ତୁ DNSକୁ ପ୍ରୋଗ୍ରାମର ନିୟନ୍ତ୍ରଣ ପ୍ୟାନେଲରେ ସିଧାସଳଖ ମ୍ୟାନେଜ କରିବାକୁ ପଡ଼ିବ।

### ଡିଫାଉଟ୍ DNS ରେକର୍ଡ୍ {#which-providers-support-dns-management}

ଯେତେବେଳେ କୌଣସି ଡମେନ୍ రిజిଷ୍ଟ୍ରେସ୍ ହୁଏ, ସେତେବେଳେ ଆଧାରୀ (default) DNS ରେକର୍ଡ୍ଗୁଡ଼ିକ ସେବା ଦେଉଥିବା ଅଟୋମାଟିକ୍ ଭାବରେ ପ୍ରୟୋଗ କରାଯାଇପାରିବ। **Settings › Domain Seller › Default DNS Records** ରେ ଯାଆନ୍ତୁ।

ଡିଫାଉଟ୍ ରେକର୍ଡ୍ ମୂଲ୍ୟ ଦୁଇଟି ଟୋକନ୍ ସମର୍ଥନ କରେ:

| Token | ବଦଳାଇବା ପାଇଁ |
|---|---|
| `{DOMAIN}` | రిజిଷ୍ଟ୍ରେସ୍ ହୋଇଥିବା ଡମେନ୍ ନାମ (ଉଦାହରଣ: `example.com`) |
| `{SITE_URL}` | ଗ୍ରାହକଙ୍କ ସାଇଟ୍ ପାଇଁ WordPress ସାଇଟ୍ URL |

**ଉଦାହରଣ — ଆପେକ୍ସ ଡମେନ୍ ଏବଂ wwwକୁ ଆପଣଙ୍କ ସର୍ଭର IP କୁ ଦିଆଯିବା:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ପ୍ରଶାସକ (Admin): DNS ଦେଖିବା ଏବଂ ସଂଶୋଧନ କରିବା {#default-dns-records}

నెట్‌ୱାର୍କ ଡାମିନ୍ ପାଇଁ କୌଣସି ଗ୍ରାହକ ଡମେନ୍ ପାଇଁ DNS ରେକର୍ଡ୍ ଦେଖିବା ଏବଂ ସଂଶୋଧନ କରିବା **Network Admin › Ultimate Multisite › Domains** ରୁ ଡମେନ୍‌ର ଏଡିଟ୍ ପେଜ୍ ମାଧ୍ୟମରେ କରାଯାଇପାରିବ।
