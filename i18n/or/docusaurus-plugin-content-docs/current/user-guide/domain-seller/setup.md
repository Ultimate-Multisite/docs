## প্রয়োজনীয়তা (Requirements)

- **Multisite Ultimate** v2.4.12 বা তার বেশি সংস্করণ, নেটওয়ার্ক অ্যাক্টিভেট করা থাকতে হবে।
- **PHP** 7.4 বা তার উপরে থাকতে হবে।
- অন্তত একটি সমর্থিত রেজিস্ট্রারের জন্য API ক্রেডেনশিয়াল প্রয়োজন।

## প্রথমবার সেটআপ উইজার্ড (First-run setup wizard)

আপনি যখন প্রথমবার প্লাগইনটি নেটওয়ার্ক অ্যাক্টিভেট করবেন, তখন সেটআপ উইজার্ডটি স্বয়ংক্রিয়ভাবে চালু হয়ে যাবে। আপনি যেকোনো সময় **Network Admin › Ultimate Multisite › Domain Seller Setup** থেকে এটি ব্যবহার করতে পারবেন।

### ধাপ ২ — ক্রেডেনশিয়াল প্রবেশ করান (Enter credentials)

প্রতিটি প্রোভাইডারের নিজস্ব ক্রেডেনশিয়াল ফিল্ড রয়েছে:

**OpenSRS** — ইউজারনেম এবং প্রাইভেট কী (OpenSRS রিসেলার কন্ট্রোল প্যানেল থেকে)।

**Namecheap** — ইউজারনেম এবং API কী (Account › Tools › API Access থেকে)।

**HostAfrica** — HostAfrica রিসেলার মডিউল থেকে ডোমেইনস রিসেলার API এন্ডপয়েন্ট এবং ক্রেডেনশিয়াল। বর্তমানে আলাদা কোনো স্যান্ডবক্স এন্ডপয়েন্ট নথিভুক্ত করা নেই; লাইভ রেজিস্ট্রেশন চালানোর আগে নিরাপদ রিড-অনলি চেক দিয়ে পরীক্ষা করুন।

**Openprovider** — ইউজারনেম এবং পাসওয়ার্ড, যেখানে API অ্যাক্সেস চালু থাকতে হবে। ঐচ্ছিক স্যান্ডবক্স মোডে Openprovider স্যান্ডবক্স API ব্যবহার করা হয়, এবং রেজিস্ট্রেশনের জন্য একটি ঐচ্ছিক ডিফল্ট কাস্টমার হ্যান্ডেল পুনরায় ব্যবহার করা যেতে পারে।

**Hostinger** — કોર Hostinger ઇન્ટિગ્રેશનમાંથી શેર કરેલી hPanel API ટોકન. આ જ ટોકન કોર ડોમેન મેપિંગ અને ડોમેન સેલર રજીસ્ટ્રેશન ઓપરેશન્સ માટે વપરાય છે.

**GoDaddy** — API કી અને સીક્રેટ (developer.godaddy.com માંથી)

**ResellerClub** — રિસેલર ID અને API કી (ResellerClub કંટ્રોલ પેનલમાંથી)

**NameSilo** — API કી (namesilo.com › Account › API Manager માંથી)

**Enom** — એકાઉન્ટ ID અને API ટોકન

જો ઉપલબ્ધ હોય તો **Sandbox mode** તપાસો જેથી લાઇવમાં જ যাওয়ার પહેલા પ્રદાતાના ટેસ્ટ એન્વાયર્નમેન્ટ સામે ચકાસણી કરી શકાય.

### પગલું 3 — કનેક્શનનું પરીક્ષણ કરો

**Test Connection** પર ક્લિક કરો. આ વિઝાર્ડ ઓળખપત્રો અને જોડાણની ચકાસણી કરવા માટે એક હળવી API કોલ મોકલે છે. આગળ વધતા પહેલા કોઈપણ ઓળખપત્રની સમસ્યાઓને સુધારો.

### પગલું 4 — TLDs ઇમ્પોર્ટ કરો

**Import TLDs** પર ક્લિક કરો જેથી જોડાયેલા પ્રદાતા પાસેથી બધા ઉપલબ્ધ TLDs અને હોલસેલ પ્રાઇસિંગ ખેંચી શકાય. આ ડોમેન પ્રોડક્ટ્સમાં વપરાતી TLD યાદીને ભરે છે. મોટા TLD કેટલોગ ધરાવતા પ્રદાતાઓ માટે ઇમ્પોર્ટમાં 30–60 સેકન્ડ લાગી શકે છે.

TLDs દરરોજ એક શેડ્યૂલ્ડ cron job દ્વારા આપોઆપ ફરીથી સિંક થાય છે.

### પગલું 5 — ડોમેન પ્રોડક્ટ બનાવો

વિઝાર્ડ 10% માર્કઅપ સાથે ડિફોલ્ટ કેચ-ઓલ પ્રોડક્ટ બનાવે છે. તમે આ પ્રોડક્ટને તરત જ એડિટ કરી શકો છો અથવા **Ultimate Multisite › Products** હેઠળ મેન્યુઅલી પ્રોડક્ટ્સ બનાવીને અવગણી શકો છો.

સંપૂર્ણ પ્રોડક્ટ રૂપરેખાંકન માર્ગદર્શિકા માટે [Domain Products and Pricing](./domain-products) જુઓ.

## પ્રદાતાને ફરીથી રૂપરેખાંકિત કરવું (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** પર જાઓ (અથવા પ્લગિન લિસ્ટમાં **Settings** પર ક્લિક કરો).

સેટિંગ્સ પેજમાં આ વસ્તુઓ છે:

- **ডোমেইন বিক্রি চালু করা** — এই পুরো ফিচারটি অন/অফ করার জন্য টগল করুন।
- **ডিফল্ট প্রোভাইডার** — ডোমেইন খোঁজা এবং নতুন প্রোডাক্টের জন্য যে প্রোভাইডার ব্যবহার করা হবে।
- **প্রতি অনুসন্ধানে সর্বোচ্চ TLD** — গ্রাহক যখন সার্চ করেন তখন কতগুলো TLD চেক করা হবে; উচ্চ মান বেশি অপশন দেখাবে কিন্তু ধীরগতির হয়।
- **অ্যাভেইলেবিলিটি ক্যাশ ডিউরেশন** — অ্যাভেইলেবিলিটি এবং প্রাইসিং রেজাল্ট কতক্ষণ ক্যাশ করা থাকবে; কম মানগুলি বেশি নির্ভুল তবে API কল বৃদ্ধি করে।
- **ডোমেইন প্রোডাক্ট পরিচালনা করুন** — প্রোডাক্ট লিস্টে দ্রুত যাওয়ার জন্য একটি শর্টকাট লিঙ্ক।
- **প্রোভাইডার কনফিগার করুন** — প্রোভাইডার যোগ বা পুনরায় কনফিগার করার জন্য ইন্টিগ্রেশন উইজার্ড খুলুন।

### দ্বিতীয় প্রোভাইডার যুক্ত করা

**Configure providers** এ ক্লিক করুন এবং নতুন রেজিস্ট্রারের জন্য উইজার্ডটি আবার চালান। আপনি একই সাথে একাধিক প্রোভাইডার কনফিগার রাখতে পারেন। প্রতিটি ডোমেইন প্রোডাক্টকে একটি নির্দিষ্ট প্রোভাইডারের সাথে বরাদ্দ করুন, অথবা ডিফল্ট সেটি রেখে দিতে পারেন।

### TLD ম্যানুয়ালি সিঙ্ক করা

সেটিংস পেজে, সর্বশেষ প্রাইসিং আনার জন্য যেকোনো কনফিগার করা প্রোভাইডারের পাশে **Sync TLDs** এ ক্লিক করুন। যখন কোনো প্রোভাইডার পাইকারি মূল্য আপডেট করে বা নতুন TLD যোগ করে তখন এটি খুব কাজে লাগে।

## ପ୍ରୋଭାଇଡର କ୍ଷମତା ନୋଟ୍ସ (Provider capability notes)

ପ୍ରତିଟି ରେଜିଷ୍ଟ୍ରାର API ସେମାନଙ୍କଠାରୁ ସମାନ କାର୍ଯ୍ୟକଳାପ ଦେଖାଏ ନାହିଁ। ଆଡଉନ୍ (addon) ସେହି ଅନୁପଯୁକ୍ତ କାର୍ଯ୍ୟକଳାପଗୁଡ଼ିକୁ ଚୁପ୍ ଭାଙ୍ଗିବା ପରିବର୍ତ୍ତେ, ସ୍ପଷ୍ଟ एडମିନ୍-ଭେସିଂ ତ୍ରୁଟି ଦେଖାଇଥାଏ।

- **HostAfrica** ସବୁଠାରୁ ବ୍ୟାପକ ଲାଇଭ୍ ରେସେଲର (reseller) କାର୍ଯ୍ୟକ୍ରମକୁ ସମର୍ଥନ କରେ, ଯେଉଁଥିରେ ଲୁକ୍ଅପ୍ (lookup), TLD/ପ୍ରସିଙ୍ଗ (pricing) ସିଙ୍କ୍, ରେଜିଷ୍ଟ୍ରେସନ୍, ରନୁଅଲ୍, ଟ୍ରାନ୍ସଫର, ନେମ୍ସର୍ଭର ଅପଡେଟ୍, DNS ରେକର୍ଡ୍, EPP କୋଡ୍, ରେଜିଷ୍ଟ୍ରାର ଲକ୍ (lock), ଏବଂ ID ପ୍ରତିରକ୍ଷା ସମେତ ସମସ୍ତ କାର୍ଯ୍ୟକୁ ସମର୍ଥନ କରେ।
- **Openprovider** ରେସେଲର-ପ୍ରାଇସ୍ TLD ସିଙ୍କ୍, ରେଜିଷ୍ଟ୍ରେସନ୍, ରନୁଅଲ୍, ଟ୍ରାନ୍ସଫର, ନେମ୍ସର୍ଭର ଅପଡେଟ୍, DNS ଜୋନ୍, EPP କୋଡ୍, ରେଜିଷ୍ଟ୍ରାର ଲକ୍, ଏବଂ WHOIS ପ୍ରାଇଭେସି (privacy) ସମର୍ଥନ କରେ। ଏହା ଏକ ସଂକ୍ଷିପ୍ତ-ଅବଧିବାହୀ ବେୟରନ୍ ଟୋକେନ୍‌ରେ ଅଧିକାର ପ୍ରାପ୍ତ କରେ, ଯାହାକୁ ଆଡଉନ୍ ନିଜେ ଆତ୍ମେଶିତ ଭାବରେ ମନରୁ ସର୍ବଦା ଅପଡେଟ୍ କରିଥାଏ।
- **Hostinger** ସାଧାରଣ hPanel API ଟୋକେନ୍ ମାଧ୍ୟମରେ ଆବଲିଲିଟି ସର୍ଚ୍ଚ, ରେଜିଷ୍ଟ୍ରେସନ୍, ପୋର୍ଟଫୋଲିଓ ଲୁକ୍ଅପ୍, ନେମ୍ସର୍ଭର ଅପଡେଟ୍, ରେଜିଷ୍ଟ୍ରାର ଲକ୍, ଏବଂ WHOIS ପ୍ରାଇଭେସି ସମର୍ଥନ କରେ। Hostinger ର ପବ୍ଲିକ୍ ডୋमेन API ରେସେଲର/ହୋଲ୍‌ସେଲ୍ ପ୍ରାଇସିଙ୍ଗ, ଆଇନ୍‌ବ୍ୟୁଡ୍ ଟ୍ରାନ୍ସଫର, ଏକ୍ସପ리ଟ୍ ରନୁଅଲ୍, କିମ୍ବା EPP-କୋଡ୍ retrieval ଦେଖାଏ ନାହିଁ; ରନୁଅଲ୍ ସ୍ୱୟଂଚିତ ଭାବରେ ଅତୀତ (auto-renew) ହୁଏ।
