# ମୌଳିକ ଧାରଣାଗୁଡ଼ିକ (Basic Concepts)

ନୂଆ WordPress Multisite ବ୍ୟବହାରକାରୀ ଏବଂ Ultimate Multisite ଆରମ୍ଭ କରୁଥିବା କାହା ପାଇଁ ପ୍ରଥମେ ଅନେକ ନୂଆ ଶବ୍ଦ ଓ ଭାବନା ମନେ କରିବା ସମୟରେ ଥିବ। ଏଗୁଡ଼ିକୁ ଶିଖିବା ଗୋଟିଏ ଗୁରୁତ୍ୱପୂର୍ଣ୍ଣ କାର୍ଯ୍ୟ, କାରଣ ଆପଣଙ୍କୁ ପ୍ଲାଟଫର୍ମ ଏବଂ ଏହା ମୋଟ ଭାବେ କିପରି କାମ କରେ ବୁଝିବାକୁ ପଡ଼ିବ।

ଏହି ଲେଖାରେ, ଆମେ WordPress ର କିଛି ମୁଖ୍ୟ ଧାରଣାକୁ ବ୍ୟାଖ୍ୟା କରିବା ଏବଂ ସ୍ଥିର କରିବାକୁ ଚେଷ୍ଟା କରିବା। କିଛି ଅଧିକ ବିଷୟ ବ୍ୟବହାରକାରୀଙ୍କ ପାଇଁ, କିଛି ଡେଭଲପର୍ମେଣ୍ଟର ପାଇଁ, ଏବଂ କିଛି ଦୁଇଜ ପାଇଁ ପ୍ରାପ୍ତ ହେବ।

## WordPress Multisite {#wordpress-multisite}

WordPress **Multisite** ହେଉଛି ଏକ ପ୍ରକାରର WordPress ଇନଷ୍ଟଲେସନ୍ ଯାହା ଆପଣଙ୍କୁ ଏକ ମୋଟ WordPress dashboard ରୁ ଅନେକ ୱେବସାଇଟ୍‌ର ଏକ ନେଟୱାର୍କ ସୃଷ୍ଟି କରିବା ଏବଂ ପରିଚାଳନା କରିବାକୁ ଅନୁମତି ଦିଏ। ଆପଣ ସାଇଟ୍ ସଂଖ୍ୟା, ଫିଚର୍, থିମ୍ (themes) ଏବଂ ୟୁଜର ରୋଲ୍‌ଗୁଡ଼ିକ ବ୍ୟବହାର କରି ମଧ୍ୟ ସବୁ କାମ ପରିଚାଳନା କରିପାରିବେ। ଆପଣ ଶତାବ୍ଦୀ ଏବଂ ହଜାର ହଜାର ସାଇଟ୍ ପର୍ଯ୍ୟନ୍ତ ପରିଚାଳନା କରିପାରିବେ।

## ନେଟୱାର୍କ (Network) {#network}

WordPress ଦୃଷ୍ଟିରୁ, ଏକ multisite network ହେଉଛି ଏଠାରେ ଅନେକ **subsites**କୁ ଏକ ମୋଟ dashboard ରୁ ପରିଚାଳନା କରାଯାଇପାରିବ। ଭୋସ୍ଟିଂ ପ୍ରଦୋଷକଙ୍କ ମଧ୍ୟରେ multisite network ସୃଷ୍ଟି କରିବା ଉପରେ ବିଭିନ୍ନତା ରହିଥାଏ, କିନ୍ତୁ ଏହାର ଅନ୍ତିମ ଫଳାଫଳ ହେଉଛି WordPress କୁ ଜଣାଇବା ପାଇଁ **wp-config.php** ଫାଇଲରେ କିଛି ଅତିରିକ୍ତ ନିର୍ଦ୍ଦେଶ (directives) ରହିଥାଏ ଯାହା ଏହା ଏହି ନିର୍ଦ୍ଦିଷ୍ଟ ମୋଡରେ କାର୍ଯ୍ୟ କରୁଛି ବୋଲି ଜଣାଇ ଦିଏ।

ଏକ stand-alone WordPress ଇନଷ୍ଟଲେସନ୍ ଏବଂ multisite network ମଧ୍ୟରେ ଅନେକ ଭିନ୍ନତା ରହିଛି, ଯାହା ଆମେ ସଂକ୍ଷିପ୍ତ ଭାବରେ ଆଲୋଚନା କରିବା।

## ଡାଟାବେସ୍ (Database) {#database}

ଏକ database ହେଉଛି ଏକ ସୁବିଧାର ସମ୍ପୂର୍ଣ୍ଣ, ବ୍ୟବସ୍ଥିତ ଡାଟାଗୁଡ଼ିକର ଏକ ସମୂହ। କମ୍ପ୍ୟୁଟିଂ ଶବ୍ଦରେ, database ଅର୍ଥ ହେଉଛି ଡାଟାକୁ ସଞ୍ଚୟ କରିବା ଏବଂ ବ୍ୟବସ୍ଥା କରିବା ପାଇଁ ବ୍ୟବହୃତ ସଫ୍ଟୱେର୍। ଏହାକୁ ଏକ ଫାଇଲ କାबिनेट ଭାବରେ ଭାବନ୍ତୁ, ଯେଉଁଠାରେ ଆପଣ ଡାଟାକୁ 'टेबल्स' (tables) ନାମରେ ବିଭିନ୍ନ ଅଂଶରେ ସଞ୍ଚୟ କରିଥାନ୍ତି।

WordPress Multisite ଗୋଟିଏ database ବ୍ୟବହାର କରେ ଏବଂ ପ୍ରତ୍ୟେକ subsite ପାଇଁ ତାହାର ନିଜର tables ରହିଥାଏ, ଯେଉଁଥିରେ blog id ପ୍ରିଫିକ୍ସ ଭାବରେ ଅଛି। ତେଣୁ, ଆପଣ ଏକ network installation ସ୍ଥାପନ କଲେ, ଆପଣଙ୍କର database ଏବଂ ଏକ subsite ସୃଷ୍ଟି ହେବା ମାତ୍ର, ଆପଣଙ୍କ ପାଖରେ ଏହି tables ଗୁଡ଼ିକ ରହିବା ଉଚିତ୍:

_wp_1_options_ - ପ୍ରଥମ ସବସାଇଟ୍ ପାଇଁ ଅପସନ୍ସ ଟେବଲ

_wp_2_options_ - ଦ୍ୱିତୀୟ ସବସାଇଟ୍ ପାଇଁ ଅପସନ୍ସ ଟେବଲ

## ହୋଷ୍ଟିଂ ପ୍ରଦୋତ୍ତା (Hosting provider) {#hosting-provider}

ହୋଷ୍ଟିଂ ପ୍ରଦୋତ୍ତା ହେଉଛି ଏକ କମ୍ପାନୀ ଯାହା ବ୍ୟବସାୟ ଏବଂ ବ୍ୟକ୍ତିମାନଙ୍କୁ ସେମାନଙ୍କ ୱେବସାଇଟ୍‌କୁ ବିଶ୍ୱବିՃବାନ ୱେବ (World Wide Web) ମାଧ୍ୟମରେ ଉପଲବ୍ଧ କରିବା ପାଇଁ ସହାୟକ ହୁଏ। ୱେବ ହୋଷ୍ଟିଂ ପ୍ରଦୋତ୍ତାମାନେ ଯେଉଁ ସେବାଗୁଡ଼ିକ ପ୍ରଦାନ କରନ୍ତି, ତାହା ବହୁତ ଭିନ୍ନ ହୋଇପାରେ, କିନ୍ତୁ ଏଥିରେ ସାଧାରଣତଃ ୱେବସାଇଟ୍ ଡିଜାଇନ୍, ହୋଷ୍ଟରେ ସ୍ଥାନ (storage space), ଏବଂ ఇంటర్నెట్‌తో ଯୋଗାଯୋଗ (connectivity) ଅନ୍ତର୍ଭୁକ୍ତ ହୁଏ।

## ଡୋमेन (Domain) {#domain}

ଡୋमेन ନାମ ହେଉଛି ଏକ ଆଇଡି (address) ଯାହାଲୋକମାନେ ଆପଣଙ୍କ ସାଇଟ୍‌କୁ ଦେଖିବା ପାଇଁ ବ୍ୟବହାର କରନ୍ତି। ଏହା ୱେବ ବ୍ରାଉଜର୍‌କୁ ଆପଣଙ୍କ ସାଇଟ୍ କୁ କେଉଁଠି ଦେଖିବାକୁ ଦେବ, ତାହା କହିଥାଏ। ଏକ ଗୋଲଗା (street address) ପରି, ଡୋमेन ହେଉଛି ୱେବରେ ਲੋਕମାନେ ଆପଣଙ୍କ ସାଇଟ୍‌କୁ ଭେଟିବା ପାଇଁ ବ୍ୟବହାର କରୁଥିବା ଉପାୟ। ଏହା ଆପଣଙ୍କ ଦୋକାନ ମୁଗ୍ଧରେ ଏକ ସାଇନ୍ ରଖିଥିବା ପରି। ଯଦି ଆପଣ ଆମ ୱେବସାଇଟ୍‌କୁ ଦେଖିବାକୁ ଚାହୁଁଛନ୍ତି, ତେବେ ଆପଣ ଆପଣଙ୍କ ବ୍ରାଉଜର୍‌ର ଅଡ୍ରେସ୍‌ରେ ଆମର ୱେବ ଠିକଣାଟି ଲେଖିବାକୁ ପଡ଼ିବ ଯାହା ହେଉଛି [_www.ultimatemultisite.com_], ଯେଉଁଠାରେ [**ultimatemultisite.com**](http://ultimatemultisite.com) ହେଉଛି ଡୋमेन ନାମ।

## ସବସାଇଟ୍‌ଡୋमेन (Subdomain) {#subdomain}

ସବସାଇଟ୍‌ଡୋमेन ହେଉଛି ମୁଖ୍ୟ ଡୋमेन (main domain) ଅଧୀନରେ ଏକ ପ୍ରକାରର ୱେବସାଇଟ୍ ହିରାକି, କିନ୍ତୁ ସାଇଟ୍‌କୁ ବ୍ୟବସାୟ କରିବା ପାଇଁ ଫୋଲ୍ଡର୍ (folders) ବ୍ୟବହାର କରିବା ପରିବର୍ତ୍ତେ ଏହା ନିଜର ଏକ ସାଇଟ୍ ଭଳି ହୋଇଥାଏ। ଏହା [**https://site1.domain.com/**](https://site1.domain.com/) ଭଳି ଦେଖାଯାଏ, ଯେଉଁଠାରେ _site1_ ହେଉଛି ସବସାଇଟ୍‌ଡୋमेनର ନାମ ଏବଂ [_domain.com_] ହେଉଛି ମୁଖ୍ୟ ଡୋमेन।

## ସବସାଇଟ୍‌ଡିରେଟୋರಿ (Subdirectory) {#subdirectory}

ସବସାଇଟ୍‌ଡିରେଟୋରି ହେଉଛି ଏକ ମୂଳ ଡୋमेन (root domain) ଅଧୀନରେ ଏକ ପ୍ରକାରର ୱେବସାଇଟ୍ ହିରାକି, ଯାହା ସାଇଟ୍‌କୁ ବ୍ୟବହାର କରିବା ପାଇଁ ଫୋଲ୍ଡର୍ ବ୍ୟବହାର କରେ। ଏହା ଏକ ସବସଫୋଲ୍ଡର୍ (subfolder) ପରି, ଏବଂ ନାମଗୁଡ଼ିକର ବ୍ୟବହାର ଅନେକ ସମୟରେ ଏକ ହୋଇପାରନ୍ତି। ଏହା [**https://domain.com/site1**](https://domain.com/site1) ଭଳି ଦେଖାଯାଏ, ଯେଉଁଠାରେ _site1_ ହେଉଛି ସବସାଇଟ୍‌ଡିରେଟୋରୀର ନାମ ଏବଂ [_domain.com_] ହେଉଛି ମୁଖ୍ୟ ଡୋमेन।

## ସବସାଇଟ୍ (Subsite) {#subsite}

ସବସାଇଟ୍ (Subsite) ହେଉଛି ଏକ ଚାଇଲ୍ଡ ସାଇଟ୍ ଯାହା ଆପଣଙ୍କର ମల్ଟିସାଇଟ୍ ନେଟୱାର୍କରେ ତିଆରି କରନ୍ତି। ଆପଣଙ୍କର WordPress Multisite ଇନଷ୍ଟଲେସନ୍ କିପରି କନଫିଗରେ ଅଛି, ତା' ଉପରେ ଏହା **ସବଡୋମେନ୍ (subdomain)** ବା **ସବଡ଼ଡିରେଟ୍ରି (subdirectory)** ମଧ୍ୟ ହୋଇପାରେ।

## ସୁପର ଆଡମିନ୍ (Super Admin) {#super-admin}

WordPress Super Admin ହେଉଛି ଏକ ୟୁଜର୍ ରୋଲ୍, ଯାହାକୁ Multisite ନେଟୱାର୍କର ସମସ୍ତ ସବସାଇଟ୍‌କୁ ମ୍ୟାନେଜ କରିବା ପାଇଁ ସମ୍ପୂର୍ଣ୍ଣ କ୍ଷମତା ଦିଆଯାଇଥାଏ। Multisite ୟୁଜର୍ ପାଇଁ, ଏହା ଆପଣଙ୍କ WordPress ଇନଷ୍ଟଲେସନ୍‌କୁ ଯେଉଁ **ବିଶେଷ ସବୁଠାରୁ ଉଚ୍ଚ ସ୍ତରରେ ପ୍ରବେଶ (highest level of access)** ଦେଇପାରନ୍ତି।

## ପ୍ଲଗିନ୍ (Plugin) {#plugin}

ସାଧାରଣତଃ, ଏକ ପ୍ଲଗିନ୍ ହେଉଛି କୋଡ୍‌ର ଏକ ସମୂହ ଯାହା ଆପଣଙ୍କ WordPress ସାଇଟ୍‌କୁ ଅତିରିକ୍ତ କାର୍ଯ୍ୟଦକ୍ଷତା ଯୋଗାଏ। ଏହା ଗୋଟିଏ ସବୁଠାରୁ ସରଳ, ଯେପରି ଲଗଇନ୍ ଲୋଗୋ ବଦଳାଇବା ପର୍ଯ୍ୟନ୍ତ ହୋଇପାରେ, କିମ୍ବା ଏକ ଆଇ-କମର୍ସ (e-commerce) କାର୍ଯ୍ୟଦକ୍ଷତା ଯୋଗାଣ କରିବା ଭଳି ଜଟିଳ ମଧ୍ୟ ହୋଇପାରେ। _Woocommerce and Contact Form_ ଏଥିରେ ଉଦାହରଣ।

WordPress Multisite ରେ, ପ୍ଲଗିନ୍ ଗୁଡ଼ିକ ସବୁଠାରୁ ପ୍ରଥମେ ନେଟୱାର୍କ ଆଡମିନ୍ ଦ୍ୱାରା Super Admin ଭାବରେ ଇନଷ୍ଟଲେ କରାଯାଇପାରନ୍ତି। ସବସାଇଟ୍ ଆଡମିନ୍ମାନେ ନିଜର ସବସାଇଟ୍ ମଧ୍ୟରେ ପ୍ଲଗିନ୍ ଅକ୍ଟିଭେଟ ଏବଂ ଡିଅକ୍ଟିଭେଟ କରିପାରନ୍ତି।

## ଥିମ୍ (Themes) {#themes}

WordPress ଥିମ୍ ହେଉଛି କିଛି ଫାଇଲ୍‌ର ଏକ ଗୋଷ୍ଠୀ (_ଗ୍ରାଫିକ୍ସ, ଷ୍ଟାଇଲ୍ ଶୀଟ୍ ଏବଂ କୋଡ୍_) ଯାହା ସାଇଟ୍‌ର ମୁଖ୍ୟ ଦୃଶ୍ୟକୁ ନିର୍ଦ୍ଧାରଣ କରେ। ଏହା ଫଣ୍ଟ୍ ଷ୍ଟାଇଲିଙ୍ଗ, ପେଜ୍ ଲେଆଉଟ, ରଙ୍ଗ ଆଦି ସମସ୍ତ ପ୍ରକାରର ଫ୍ରଣ୍ଟ-ଏଣ୍ଡ ଷ୍ଟାଇଲିଂ ଯୋଗାଏ।

ପ୍ଲଗିନ୍‌ ପରି, WordPress Multisite ରେ ଥିମ୍ ମଧ୍ୟ Super Admin ଦ୍ୱାରା ଇନଷ୍ଟଲେ କରାଯାଇପାରନ୍ତି ଏବଂ ସବସାଇଟ୍ ଆଡମିନ୍ମାନେ ନିଜର ସବସାଇଟ୍ ରେ ଅକ୍ଟିଭେଟ କରିପାରନ୍ତି।

## ସାଇଟ୍ ଟେମ୍ପ୍ଲେଟ୍ (Site Template) {#site-template}

**ସାଇଟ୍ ଟେମ୍ପ୍ଲେଟ୍** ହେଉଛି ଏକ ବୋୟରଲେଟ୍ ସାଇଟ୍, ଯାହାକୁ ଆପଣଙ୍କ ନେଟୱାର୍କରେ ନୂଆ ସାଇଟ୍ ତିଆରି କରିବା ପାଇଁ ମୂଳ ଭାବରେ ବ୍ୟବହାର କରାଯାଇପାରିବ।

ଏହାର ଅର୍ଥ ହେଉଛି, ଆପଣ ଏକ ମୂଳ ସାଇଟ୍ ତିଆରି କରିପାରିବେ, ବିଭିନ୍ନ ପ୍ଲଗିନ୍ ଅକ୍ଟିଭେଟ କରିପାରିବେ, ଏକ କାର୍ଯ୍ୟକାରୀ ଥିମ୍ ବାନ୍ କରିପାରିବେ, ଏବଂ ଆପଣ ଯେପରି ଚାହାଁନ୍ତି ସେହି ଅନୁଯାୟୀ ଏହାକୁ କଷ୍ଟମାଇଜ୍ (customize) କରିପାରିବେ। ତା' ପରେ, ଯେତେବେଳେ ଆପଣଙ୍କ ଗ୍ରାହକ ନୂଆ ଆକାଉଣ୍ଟ ତିଆରି କରିବେ, ସେ ସାଧାରଣ WordPress ସାଇଟ୍ ମାନ ବଦଳରେ ଆପଣଙ୍କର ଏହି ମୂଳ ସାଇଟ୍‌ର ଏକ କପି ପାଇବେ, ଯେଉଁଥିରେ ସମସ୍ତ କଷ୍ଟମାଇଜେସନ୍ ଏବଂ ବିଷୟବସ୍ତୁ (contents) ପ୍ରାଥମିକ ଭାବରେ ଥିବ।

## ଡୋमेन ମ୍ୟପିଙ୍ଗ (Domain {#domain-mapping}

વર્ડપ્રેસ સાથે ડોમેન મેપિંગ એ યુઝર્સને વેબસાઇટના સરનામા દ્વારા સાચા હોસ્ટ પર રીડાયરેક્ટ કરવાની એક રીત છે. વર્ડપ્રેસ મલ્ટીસાઇટમાં, સબ-સાઇટ્સ બનાવવા માટે કાં તો કોઈ સબ-ડ direક્ટરી અથવા સબડોમેનનો ઉપયોગ થાય છે. ડોમેન મેપિંગ એ એવી વસ્તુ છે જે સબ-સાઇટ યુઝર્સને [**joesbikeshop.com**](http://joesbikeshop.com) જેવા ટોપ-લેવલ ડોમેનનો ઉપયોગ કરવાની મંજૂરી આપે છે જેથી તેમની સાઇટ એડ્રેસ વધુ પ્રોફેશનલ લાગે.

## SSL {#ssl}

SSL નો અર્થ **Secure Sockets Layer** થાય છે. તે એક ડિજિટલ સર્ટિફિકેટ છે જે વેબસાઇટની ઓળખને પ્રમાણિત કરે છે અને એન્ક્રિપ્ટેડ કનેક્શનને સક્ષમ કરે છે. આજકાલ તેનો ઉપયોગ ઇન્ટરનેટ કનેક્શન સુરક્ષિત રાખવા અને બે સિસ્ટમો વચ્ચે મોકલાતા કોઈપણ સંવેદનશીલ ડેટાની સુરક્ષા માટેના સ્ટાન્ડર્ડ ટેકનોલોજી તરીકે થાય છે, જે ગુનેગારોને ટ્રાન્સફર થયેલી કોઈપણ માહિતી વાંચવા અને તેમાં ફેરફાર કરવાથી અટકાવે છે, જેમાં સંભવિત વ્યક્તિગત વિગતો પણ સામેલ છે. આધુનિક બ્રાઉઝર્સ SSL ની માંગ કરે છે, તેથી વેબસાઇટ બનાવતી વખતે અને ચલાવતી વખતે તે આવશ્યક છે.

## મીડિયા (Media) {#media}

મીડિયા એ ઈમેજ, ઓડિયો, વિડિયો અને અન્ય ફાઈલો છે જે વેબસાઇટ બનાવે છે.

નેટવર્ક સાઇટ્સ વર્ડપ્રેસ મલ્ટીસાઇટમાં એક જ ડેટાબેઝ શેર કરે છે, તેઓ મીડિયા ફાઈલો માટે ફાઇલ સિસ્ટમ પર અલગ પાથ (paths) જાળવી રાખે છે.

સ્ટાન્ડર્ડ વર્ડપ્રેસ સ્થાન (wp-content/uploads) યથાવત રહે છે; જોકે, તેનો પાથ નેટવર્ક સાઇટની અનન્ય ID દર્શાવવા માટે બદલવામાં આવે છે. પરિણામે, નેટવર્ક સાઇટ માટે મીડિયા ફાઈલો wp-contents/uploads/site/[id] તરીકે દેખાય છે.

## પરમાલિન્ક્સ (Permalinks) {#permalinks}

પરમાલિન્ક્સ તમારી સાઇટમાં તમારા વ્યક્તિગત બ્લોગ પોસ્ટ અથવા પેજના કાયમી URL હોય છે. પરમાલિન્ક્સને **પ્રીટી લિંક્સ** પણ કહેવામાં આવે છે. ડિફોલ્ટ રૂપે, વર્ડપ્રેસ URL ક્વેરી સ્ટ્રિંગ ફોર્મેટનો ઉપયોગ કરે છે જે કંઈક આ પ્રમાણે દેખાય છે:

[**http://www.example.com/registration**](http://www.example.com/registration)

## અલ્ટીમેટ મલ્ટીસાઇટ (Ultimate Multisite) {#ultimate-multisite}

Ultimate Multisite ହେଉଛି WordPress ରେ ଏକ প্লাগଇନ୍, ଯାହା WordPress Multisite ইনস্টলেସ ପାଇଁ ਬਣਿਆ ਹੈ। ਇਹ ਤੁਹਾਡੇ WordPress ইনস্টлеਸ਼ਨ ਨੂੰ ਇੱਕ ਪ੍ਰੀਮੀਅਮ ਸਾਈਟ ਨੈੱਟਵਰਕ ਵਿੱਚ ਬਦਲ ਦਿੰਦਾ ਹੈ – ਜਿਵੇਂ [WordPress.com](https://WordPress.com) - ਜਿਸ ਨਾਲ ਕਲਾਇੰਟਾਂ ਮਹੀਨਾਵਾਂ, ਤਿ纏 ਜਾਂ ਸਾਲਾਨਾ ਫੀਸ ਰਾਹੀਂ ਸਾਈਟ ਬਣਾ ਸਕਦੇ ਹਨ (ਤੁਸੀਂ ਮੁਫਤ ਪਲਾਨ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ)।

## Checkout Form {#checkout-form}

Checkout Form ਇੱਕ ਸਿੰਗਲ ਜਾਂ ਮਲਟੀ-ਸਟੈਪ ਆਰਡਰ ਫਾਰਮ ਹੁੰਦਾ ਹੈ ਜਿਸ ਵਿੱਚ Ultimate Multisite ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਰਾਹੀਂ ਸਬਸਾਈਟ, ਮੈਂਬਰਸ਼ਿਪ ਅਤੇ ਉਪਭੋਗਤਾ ਖਾਤੇ ਬਣਾਉਣਾ ਸ਼ਾਮਲ ਹੁੰਦਾ ਹੈ। ਇਸ ਵਿੱਚ ਵੱਖ-ਵੱਖ ਫੀਲਡਾਂ ਅਤੇ ਭੁਗਤਾਨ ਫਾਰਮ ਹੁੰਦੇ ਹਨ ਜੋ ਕਿਸੇ ਵੀ ਉਪਭੋਗਤਾ ਨੂੰ ਸਾਈਨ-ਅੱਪ ਪ੍ਰਕਿਰਿਆ ਦੌਰਾਨ ਜਮ੍ਹਾਂ ਕਰਵਾਉਣਾ ਪੈਂਦਾ ਹੈ।

## Webhook {#webhook}

ਇੱਕ web hook (ਜਿਸਨੂੰ web callback ਜਾਂ HTTP push API ਵੀ ਕਿਹਾ ਜਾਂਦਾ ਹੈ) ਇੱਕ ਐਪ ਨੂੰ ਹੋਰ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਰੀਅਲ-ਟਾਈਮ ਜਾਣਕਾਰੀ ਪ੍ਰਦਾਨ ਕਰਨ ਦਾ ਤਰੀਕਾ ਹੈ। ਇੱਕ web hook ਕਿਸੇ ਡਾਟਾ ਨੂੰ ਦੂਸਰੀਆਂ ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਉਸ ਸਮੇਂ ਹੀ ਡਾਟਾ ਪਹੁੰਚਾਉਂਦਾ ਹੈ, ਜਿਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਤੁਹਾਨੂੰ ਡਾਟਾ ਤੁਰੰਤ ਮਿਲ ਜਾਂਦਾ ਹੈ।

**Ultimate Multisite webhooks** ਅਸੀਮਤ ਸੰਭਾਵਨਾਵਾਂ ਖੋਲ੍ਹਦੇ ਹਨ, ਜੋ ਨੈੱਟਵਰਕ ਐਡਮਿਨਾਂ ਨੂੰ ਬਹੁਤ ਸਾਰੇ ਗੁੰਝਲਦਾਰ ਪਰ ਲਾਭਦਾਇਕ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ, ਖਾਸ ਕਰਕੇ ਜੇਕਰ ਇਸਦੀ ਵਰਤੋਂ _Zapier ਅਤੇ IFTTT_ ਵਰਗੀਆਂ ਸੇਵਾਵਾਂ ਦੇ ਨਾਲ ਕੀਤੀ ਜਾਵੇ।

## Events {#events}

ਇੱਕ Event ਇੱਕ ਕਾਰਵਾਈ ਹੁੰਦੀ ਹੈ ਜੋ ਕਿਸੇ ਉਪਭੋਗਤਾ ਜਾਂ ਕਿਸੇ ਹੋਰ ਸਰੋਤ ਦੀ ਕਾਰਵਾਈ, ਜਿਵੇਂ ਕਿ ਮਾਊਸ ਕਲਿੱਕ ਕਰਨ ਦਾ ਨਤੀਜਾ ਹੁੰਦੀ ਹੈ। Ultimate Multisite ਤੁਹਾਡੇ ਪੂਰੇ ਨੈੱਟਵਰਕ ਵਿੱਚ ਹੋ ਰਹੀਆਂ ਸਾਰੀਆਂ Events ਅਤੇ ਲੌਗਾਂ ਦਾ ਰਿਕਾਰਡ ਰੱਖਦਾ ਹੈ। ਇਹ ਤੁਹਾਡੇ multisite ਵਿੱਚ ਹੋਣ ਵਾਲੀ ਵੱਖ-ਵੱਖ ਗਤੀਵਿਧੀਆਂ, ਜਿਵੇਂ ਕਿ ਪਲਾਨ ਬਦਲਾਅ, ਨੂੰ ਟ੍ਰੈਕ ਕਰਦਾ ਹੈ।
