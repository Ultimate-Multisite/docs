# সাইট স্পেসিফিকেশন স্কিল

**সাইট স্পেসিফিকেশন স্কিল** হলো আপনার সাইটের লক্ষ্য, দর্শক এবং ব্র্যান্ডের পরিচয় নথিভুক্ত করার একটি সুসংগঠিত পদ্ধতি। এই তথ্যগুলো আপনার **site_brief** মেমোরিতে সংরক্ষণ করা থাকে, যা এজেন্টরা বিভিন্ন সেশনের মধ্যে ধারাবাহিক এবং প্রসঙ্গ-সচেতন সহায়তা দেওয়ার জন্য রেফার করে।

## সাইট স্পেসিফিকেশন কী?

সাইট স্পেসিফিকেশন হলো নিম্নলিখিত বিষয়গুলো নথিভুক্ত করার প্রক্রিয়া:

- **সাইটের উদ্দেশ্য**: আপনার সাইট কী করে এবং কেন এটি বিদ্যমান
- **লক্ষ্য দর্শক**: কারা আপনার সাইটে আসে এবং তাদের কী প্রয়োজন
- **ব্র্যান্ড পরিচিতি**: আপনার রঙ, সুর (tone) এবং ভিজ্যুয়াল স্টাইল
- **ব্যবসায়িক লক্ষ্য**: আপনার সাইটের জন্য সাফল্য দেখতে কেমন হবে
- **কন্টেন্ট কাঠামো**: আপনার সাইটটি কীভাবে সাজানো আছে

এই স্পেসিফিকেশনটি আপনার **site_brief** হয়ে যায়, যা একটি স্থায়ী মেমরি যেখান থেকে এজেন্টরা আপনার সাইটের প্রেক্ষাপট বুঝতে ব্যবহার করে।

## সাইট স্পেসিফিকেশন কেন ব্যবহার করবেন?

### সেশনের মধ্যে ধারাবাহিকতা

site_brief না থাকলে, প্রতিবার নতুন সেশন শুরু করার সময় আপনাকে আপনার সাইটের উদ্দেশ্য বারবার ব্যাখ্যা করতে হতো। এটি থাকলে, এজেন্টরা তাৎক্ষণিকভাবে বুঝতে পারে:

- আপনার সাইটের লক্ষ্য এবং দর্শক
- আপনার ব্র্যান্ডের রঙ এবং সুর
- আপনার কন্টেন্ট কাঠামো
- আপনার ব্যবসায়িক উদ্দেশ্যগুলো

### আরও ভালো সুপারিশ

এজেন্টরা আপনার site_brief ব্যবহার করে:

- আপনার সাইটের উদ্দেশ্যের সাথে সামঞ্জস্যপূর্ণ ফিচারগুলো সাজেস্ট করতে পারে
- আপনার লক্ষ্যের সাথে মিলে যাওয়া কন্টেন্ট স্ট্রাকচার সুপারিশ করতে পারে
- আপনার ব্র্যান্ডের সাথে সামঞ্জস্যপূর্ণ ডিজাইন প্রস্তাব করতে পারে
- অসামঞ্জস্যপূর্ণ ফিচার সাজেস্ট করা এড়াতে পারে

### দ্রুত অনবোর্ডিং

নতুন এজেন্টরা (বা নতুন সেশনে এজেন্টরা) স্পষ্টীকরণ প্রশ্ন করার পরিবর্তে আপনার site_brief পড়ে দ্রুত কাজ শুরু করতে পারে।

## সাইট স্পেসিফিকেশন শুরু করা

### থিম বিল্ডার অনবোর্ডিং চলাকালীন

**Theme Builder onboarding flow** চলাকালীন Site Specification স্কিলটি স্বয়ংক্রিয়ভাবে শুরু হয়। সেটআপ অ্যাসিস্ট্যান্ট এজেন্ট প্রশ্ন জিজ্ঞাসা করে এবং আপনার site_brief তৈরি করে।

### ম্যানুয়াল সূচনা

તમે કોઈપણ સમયે સાઇટ સ્પેશિફિકેશન શરૂ કરી શકો છો:

```
"ચાલો મારી સાઇટની વિશિષ્ટતાઓ વ્યાખ્યાયિત કરીએ"
```

અથવા

```
"મને એક સાઇટ બ્રીફ બનાવવામાં મદદ કરો"
```

## સાઇટ સ્પેશિફિકેશન પ્રક્રિયા

### પગલું ૧: સાઇટનો હેતુ (Site Purpose)

એજન્ટ પૂછશે:

```
તમારી સાઇટનો મુખ્ય હેતુ શું છે?
- ઈ-કોમર્સ સ્ટોર
- બ્લોગ અથવા કન્ટેન્ટ સાઇટ
- પોર્ટફોલિયો અથવા શોકેસ
- SaaS એપ્લિકેશન
- સમુદાય અથવા ફોરમ
- અન્ય: [વર્ણન કરો]
```

તમે કોઈ કેટેગરી પસંદ કરી શકો છો અથવા પોતાનો હેતુ વર્ણવી શકો છો.

### પગલું ૨: લક્ષિત પ્રેક્ષકો (Target Audience)

```
તમારા મુખ્ય પ્રેક્ષકો કોણ છે?
- ગ્રાહકો / સામાન્ય લોકો
- બિઝનેસ પ્રોફેશનલ્સ
- ડેવલપર્સ / ટેકનિકલ યુઝર્સ
- વિદ્યાર્થીઓ / શિક્ષકો
- અન્ય: [વર્ણન કરો]

તેમની મુખ્ય જરૂરિયાતો શું છે?
```

### પગલું ૩: બ્રાન્ડ ઓળખ (Brand Identity)

```
તમારા બ્રાન્ડના રંગો કયા છે?
- પ્રાથમિક રંગ: [કલર પિકર અથવા હેક્સ કોડ]
- ગૌણ રંગ: [કલર પિકર અથવા હેક્સ કોડ]
- એક્સેન્ટ રંગ: [વૈકલ્પિક]

તમે તમારા બ્રાન્ડના સ્વર (tone) નું વર્ણન કેવી રીતે કરશો?
- પ્રોફેશનલ / કોર્પોરેટ
- ક્રિએટીવ / કલાત્મક
- મનોરંજક / કેઝ્યુઅલ
- મિનિમલ / આધુનિક
- ગરમ / મૈત્રીપૂર્ણ
```

### પગલું ૪: વ્યવસાયિક લક્ષ્યો (Business Goals)

```
તમારી સાઇટ માટે સફળતા કેવી દેખાય છે?
- લીડ્સ જનરેટ કરવી
- ઉત્પાદનો વેચવા
- સમુદાય બનાવવો
- જ્ઞાન વહેંચવું
- સત્તા સ્થાપિત કરવી
- અન્ય: [વર્ણન કરો]

તરીકે તમારું મુખ્ય મેટ્રિક શું છે?
- આવક (Revenue)
- યુઝર એન્ગેજમેન્ટ
- કન્ટેન્ટ પહોંચ (Content reach)
- konversions
- અન્ય
```

### પગલું ૫: સામગ્રીની રચના (Content Structure)

```
તમારી સામગ્રી કેવી રીતે ગોઠવાયેલી છે?
- ફ્લેટ (બધી સામગ્રી એક જ સ્તર પર)
- પદાનુક્રમિક (કેટેગરીઓ અને પેટાકેટેગરીઓ)
- સમય આધારિત (બ્લોગ-શૈલી)
- ઉત્પાદન-આધારિત (કેटलॉग)
- અન્ય: [વર્ણન કરો]

તમે કયા પ્રકારની સામગ્રીનો ઉપયોગ કરો છો?
- બ્લોગ પોસ્ટ્સ
- પ્રોડક્ટ પેજીસ
- કેસ સ્ટડીઝ
- દસ્તાવેજીકરણ (Documentation)
- વિડિયો
- અન્ય
```

## તમારી site_brief મેમરી

સાઇટ સ્પેશિફિકેશન પૂર્ણ કર્યા પછી, તમારી માહિતી તમારા એજન્ટની મેમરીમાં **site_brief** તરીકે સંગ્રહિત થાય છે. આ એક માળખાગત રેકોર્ડ છે જેમાં નીચેની બાબતો શામેલ હોય છે:

## ଆପଣଙ୍କ ସାଇଟ୍ ବ୍ରିଫ୍ ଦେଖିବା ଏବଂ ଅପଡେଟ କରିବା

### ଆପଣଙ୍କର ସାଇଟ୍ ବ୍ରିଫ୍ ଦେଖନ୍ତୁ

ଆପଣ ଏହି ପ୍ରଶ୍ନଗୁଡ଼ିକ ପଚାରିପାରିବେ:

```
"ମୋର ସାଇଟ୍ ବ୍ରିଫ୍ ଦେଖାଅ"
```

କିମ୍ବା

```
"ମୋ ସାଇଟ୍ ବିଷୟରେ ଆପଣ କ'ଣ ଜାଣନ୍ତି?"
```

ଏହି ଠିକଣାଟି ଆପଣଙ୍କର ସଞ୍ଚୟ କରାଯାଇଥିବା ନିର୍ଦ୍ଦେଶ (specification) ପ୍ରଦର୍ଶନ କରିବ।

### ଆପଣଙ୍କର ସାଇଟ୍ ବ୍ରିଫ୍ ଅପଡେଟ କରନ୍ତୁ

ଆପଣଙ୍କ ସାଇଟ୍ ଯଦି ବିକାଶ ହୋଇଥାଏ, ତେବେ ଆପଣ ଏହାକୁ ଅପଡେଟ କରିପାରିବେ:

```
"ମୋର ସାଇଟ୍ ବ୍ରିଫ୍ ଅପଡେଟ କରନ୍ତୁ: ଆମେ ବର୍ତ୍ତମାନ B2B ଗ୍ରାହକଙ୍କ ଉପରେ ଲକ୍ଷ୍ୟ ରଖୁଛୁ"
```

କିମ୍ବା

```
"ମୋ ସାଇଟ୍ ବିଶେଷଣୀ (specification)କୁ ମଜବୁତ କରନ୍ତୁ"
```

ଏହା ଆପଣଙ୍କର ଅସ୍ଥିତ ସୂଚନାକୁ ଆରମ୍ଭ ଭାବରେ ନେଇ Site Specification କୌଶଳକୁ ପୁଣି ଚଲାଇ ଦେବ।

## ଏଜେଣ୍ଟ୍ସ ସାଇଟ୍ ବ୍ରିଫ୍‌କୁ କିପରି ବ୍ୟବହାର କରନ୍ତି

### ଡିଜାଇନ୍ ସୁପାରିଶ (Design Recommendations)

ଯେତେବେଳେ ଆପଣ ଡିଜାଇନରେ ପରିବର୍ତ୍ତନ ପାଇଁ ଅନୁରୋଧ କରନ୍ତି, ଏଜେଣ୍ଟ୍ସ ଆପଣଙ୍କ ସାଇଟ୍ ବ୍ରିଫ୍‌କୁ ଉଲ୍ଲେଖ କରିବେ:

```
ଆପଣ: "ମୋ ହୋମ୍ପେଜ୍ (homepage)କୁ ପୁଣି ଡିଜାଇନ୍ କରନ୍ତୁ"
ଏଜେଣ୍ଟ: "ଆପଣଙ୍କ ସାଇଟ୍ ବ୍ରିଫ୍ ଅନୁଯାୟୀ, ଆପଣ ୨୫-୪୫ ବର୍ଷର ମହିଳାମାନଙ୍କ ଉପରେ ଲକ୍ଷ୍ୟ ରଖିଛନ୍ତି ଏବଂ ଆପଣଙ୍କର ଢଙ୍ଗ ହେଉଛି ଗରମ, କାରିଗରି (artisanal) ଏବଂ ସ୍ଥାୟୀ। ମୁଁ ଏକ ଡିଜାଇନ୍ ପ୍ରସ୍ତୁତ କରିବି ଯାହା ଆପଣଙ୍କ ହାତରେ ତିଆରି ଜୋହଳ ଓ ସ୍ଥାୟୀ ମୂଲ୍ୟବୋଧକୁ ଗୁରୁତ୍ୱ ଦେବ।"
```

### ଅବଶ୍ୟକତା ପାଇଁ ପ୍ରସ୍ତାବ (Feature Suggestions)

ଏଜେଣ୍ଟ୍ସ ଆପଣଙ୍କ ଲକ୍ଷ୍ୟ ସହିତ ସମନ୍ୱୟ ହେଉଥିବା ବିଶେଷତାଗୁଡ଼ିକର ପ୍ରସ୍ତାବ କରେ:

```
ଆପଣ: "ମୋର କିପରି ଅଧିକ ବିଶେଷତା ଯୋଗ କରିପାରିବ?"
ଏଜେଣ୍ଟ: "କଷ୍ଟମର୍ ଲୟାଲ୍ଟି (customer loyalty) ଉପରେ ହିଁ ଆଧାରିତ ଏକ e-commerce ସାଇଟ୍ ପାଇଁ, ମୁଁ ନିମ୍ନଲିଖିତଗୁଡ଼ିକର ପ୍ରସ୍ତାବ କରିବି: ଲୟାଲ୍ଟି ପ୍ରୋଗ୍ରାମ, କଷ୍ଟମର୍ ରିଭିଉ (customer reviews), ଇମେଲ୍ ନିଉଜଲ୍ଟର ଏବଂ ଉତ୍ପାଦ ସୁପାରିଶ।"
```

### ବିଷୟବସ୍ତୁ ଢାଞ୍ଚା (Content Structure)

ଆପଣଙ୍କ ଢାଞ୍ଚା ଆଧାରରେ, ଏଜେଣ୍ଟ୍ସ ବିଷୟବସ୍ତୁ ବ୍ୟବହାରିକ ସମନ୍ୱୟ ପାଇଁ ପ୍ରସ୍ତାବ କରନ୍ତି:

ଆପଣଙ୍କ ବ୍ଲଗକୁ କିପରି ସଜାଡ଼ିବେ?
ଉତ୍ତର: ଆପଣ ଏକ  prodक्ट କଟାଲଗ୍ (product catalog) ସହିତ ବ୍ଲଗ୍ ବ୍ୟବହାର କରୁଛନ୍ତି, ତେଣୁ ମୋର ପରାମର୍ଶ ହେଉଛି ପ୍ରତିଟି ପୋଷ୍ଟକୁ prodक्ट କଟାଗୋରୀ ଅନୁଯାୟୀ ସଜାଡ଼ିବା ଏବଂ ଅନେକ prodक्टକୁ ଏକାଠି କରି ଦର୍ଶାଇବା ପାଇଁ 'ଷ୍ଟାଇଲିଂ ଗାଇଡ୍' (styling guides) ତିଆରି କରିବା।

## ଶ୍ରେଷ୍ଠ ଅଭ୍ୟାସ (Best Practices)

### ସ୍ପଷ୍ଟ ରୁହନ୍ତୁ (Be Specific)

"ସାଧାରଣ ପାଠକମାନଙ୍କ ପାଇଁ" ବୋଲି କହିବା ପରିବର୍ତ୍ତେ, ଆପଣଙ୍କ ପ୍ରକୃତ ପାଠକଙ୍କ ବିଷୟରେ ବର୍ଣ୍ଣନା କରନ୍ତୁ:

- ✓ "୨୫-୪୫ ବର୍ଷର ମହିଳାମାନେ, ଯେଉଁମାନେ ସ୍ଥାୟୀ ଫ୍ୟାଶନରେ ଆଗ୍ରହୀ"
- ✗ "ସବୁକୁ"

### ନିୟମିତ ଅପଡେଟ୍ କରନ୍ତୁ (Update Regularly)

ଆପଣଙ୍କ ସାଇଟ୍ ବିକାଶ ହେଉଥିବା ବେଳେ, ଆପଣଙ୍କ `site_brief` ମଧ୍ୟ ଅପଡେଟ୍ କରନ୍ତୁ:

- ଯେତେବେଳେ ଆପଣ ନୂଆ ପାଠକଙ୍କ ଆଡ଼କୁ ପରିବର୍ତ୍ତନ କରନ୍ତି
- ଯେତେବେଳେ ଆପଣ ନୂଆ prodक्ट ଲାଇନ୍ ଯୋଗ କରନ୍ତି
- ଯେତେବେଳେ ଆପଣଙ୍କ ବ୍ରାଣ୍ଡ ଚିହ୍ନଟ (brand identity) ବଦଳିଯାଏ
- ଯେତେବେଳେ ଆପଣଙ୍କ ବ୍ୟବସାୟିକ ଲକ୍ଷ୍ୟ ପରିବର୍ତ୍ତନ ହୁଏ

### ସମନ୍ୱିତ ଶବ୍ଦ ବ୍ୟବହାର କରନ୍ତୁ (Use Consistent Terminology)

ସେହି ସମୟରେ ଏକାଠି ରହିବା ପାଇଁ ସମାନ ଶବ୍ଦ ବ୍ୟବହାର କରନ୍ତୁ:

- ✓ ସର୍ବଦା "ସ୍ଥାୟୀ ଜୋଇଲରୀ" (sustainable jewelry) କୁ କୁହନ୍ତୁ (ଯେଉଁଥିରେ "ଇକୋ-ଫ୍ରେଣ୍ଡ୍ ଜୋଇଲରୀ" ବା "ଗ୍ରୀନ୍ prodक्ट" ବୋଲି କହିବେ ନାହିଁ)
- ✓ ଆପଣଙ୍କ ପାଠକମାନଙ୍କୁ ସର୍ବଦା ଏକାସାଙ୍ଗରେ ସେହି ଢଙ୍ଗରେ ଅଭିଶ୍ୱରଣ କରନ୍ତୁ

### ପ୍ରସଙ୍ଗ ଯୋଡ଼ନ୍ତୁ (Include Context)

ଆପଣଙ୍କ ନିଷ୍ପତ୍ତି ବୁଝିବା ପାଇଁ ପଛର ପରିବେଶ ଦିଅନ୍ତୁ:

- "ଆମେ ଏଭଳି ମାନଦାନ୍ୟକତାକୁ ଉପରେ ଗୁଣକୁ ପ୍ରାଧାନ୍ୟ ଦେଉଥିବା ବୃତ୍ତିଗତ ବ୍ୟକ୍ତିମାନଙ୍କୁ ଲକ୍ଷ୍ୟ କରୁଛୁ"
- "ଆମ ପାଠକମାନେ ପ୍ରଯୁଜ୍ୟ (tech-savvy) ଏବଂ ଆଧୁନିକ ဒီဇိုင်းକୁ ଅପେକ୍ଷା କରନ୍ତି"
- "ଆମେ ଏକ ବଟ୍‌ଷ୍ଟ୍ରପ୍ଡ୍ (bootstrapped) ଷ୍ଟାର୍ଟଅପ୍, ତେଣୁ ଆମକୁ ମୂଲ୍ୟ-ସମ୍ପାଦନୀୟ ଉପାୟ ଦରକାର"

## Theme Builder ଅନ୍ବୋର୍ଡିଂ ସହିତ ସମ୍ପର୍କ (Relationship to Theme Builder Onboarding)

Site Specification କୌଶଳଟି **Theme Builder ଅନ୍ବୋର୍ଡିଂ ପ୍ରବାହରେ** ସମାଗ୍ରିକ ଭାବରେ ସମାଗ୍ରିକ ହୋଇଛି। ଯେତେବେଳେ ଆପଣ ଅନ୍ବୋର୍ଡିଂ ସମ୍ପୂର୍ଣ୍ଣ କରିବେ, ଆପଣ ଯେଉଁ ସୂଚନା ଦେଇଛନ୍ତି ତାହା ସହିତ ଆପଣଙ୍କ `site_brief` ଆଧାରରେ ଆଧାରିତ ଭାବରେ ସ୍ଥାପନ କରାଯିବ।

ଆପଣ ନିମ୍ନଲିଖିତ କାର୍ଯ୍ୟ ପାଇଁ Site Specificationକୁ ଏକାନ୍ତ ଭାବେ ଚଲାଇପାରିବେ:

- ପ୍ରାରମ୍ଭିକ ସେଟଅପ୍ ପରେ ଆପଣଙ୍କ ବିଶ୍ଳେଷଣ (specification)କୁ ଠିକ୍ କରିବା ପାଇଁ
- ଆପଣଙ୍କ ସାଇଟ୍ ବିକାଶ ହେଉଥିବା ଅନୁଯାୟୀ ଆପଣଙ୍କ `site_brief` ମ

**ମୁଁ ଏକ ନୂଆ site\_brief ସହିତ ଆରମ୍ଭ କରିବାକୁ ଚାହେଁ**
- ଇଞ୍ଜିନକୁ ପଚାରନ୍ତୁ: "ମୋର site brief ମୁକ୍ତ କରନ୍ତୁ ଏବଂ ନୂଆ ଆରମ୍ଭ କରନ୍ତୁ"
- ତା'ପରେ Site Specification ପୁଣିଥରେ ଚଲାଇଲେ

**ଇଞ୍ଜିନଟି ମୋର site\_brief ସହିତ ମେଳ ନଉଥିବା ପରାମର୍ଶ ଦେଉଛି**
- ଇଞ୍ଜିନକୁ କୁହନ୍ତୁ: "ମୋର site brief ଯାଞ୍ଚ କରନ୍ତୁ"
- ଯଦି ଆପଣଙ୍କର site\_brief ଅପତଙ୍ଗ (outdated) ହୋଇଯାଇଛି, ତେବେ ତାହାକୁ ଅପଡେଟ୍ କରନ୍ତୁ
- ଆପଣଙ୍କ ଅନୁରୋଧରେ ଅତିରିକ୍ତ ସୂଚନା ପ୍ରଦାନ କରନ୍ତୁ

## ପରବର୍ତ୍ତୀ ପଦକ୍ଷେପ (Next Steps)

ଆପଣ ଆପଣଙ୍କର site specification ଠିକ୍ କରିବା ପରେ:

1. **Theme Builder ବ୍ୟବହାର କରନ୍ତୁ**: ଆପଣଙ୍କ site\_brief ଆଧାରରେ ଏକ কাস্টମ ଥିମ୍ (custom theme) ତିଆରି କରନ୍ତୁ
2. **ଡିଜାଇନ୍ ସୁଧାରନ୍ତୁ**: ବିସ୍ତୃତ ଡିଜାଇନ୍ କାମ ପାଇଁ Design System Aesthetics skill ବ୍ୟବହାର କରନ୍ତୁ
3. **କଣ୍ଟେଣ୍ଟ ଯୋଜନା କରନ୍ତୁ**: କଣ୍ଟେଣ୍ଟ ଢାଞ୍ଚା (content structure) ସମ୍ପର୍କରେ ଆଇଜେଣ୍ଟମାନଙ୍କଠାରୁ ପରାମର୍ଶ ମାଗନ୍ତୁ
4. **ଫିଚର୍ସ ବନେୟନ୍ତୁ**: ଆପଣଙ୍କ ବ୍ୟବସାୟିକ ଲକ୍ଷ୍ୟ ସହିତ ମେଳ କରୁଥିବା ଫିଚର୍ସ ଅନୁରୋଧ କରନ୍ତୁ
