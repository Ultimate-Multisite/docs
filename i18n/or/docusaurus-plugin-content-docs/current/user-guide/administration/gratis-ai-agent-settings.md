# ମୁକ୍ତ AI ଇଜେଣ୍ଟ ସେଟିଙ୍ଗସ

Gratis AI Agent ର **Settings → Advanced** స్క్రీన్ରେ v1.5.0 ଠାରୁ ଆରମ୍ଭ ହୋଇଥିବା ବ୍ୟାକ୍‌ଏଣ୍ଡ ఇంటిଗ୍ରେସନ୍ ପାଇଁ ପ୍ରଶାସକ-ସ୍ତରର କନଫିଗୁରେସନ୍ ଦେଖାଯାଏ। ଏହି ପୃଷ୍ଠାଟି **Feedback Endpoint** ଫିଲ୍ଡ ଏବଂ ଏହାର ଅପେକ୍ଷା କରାଯାଉଥିବା ଫର୍ମାଟ ବିଷୟରେ ବର୍ଣ୍ଣନା କରେ।

## ସେଟିଙ୍ଗକୁ ପ୍ରବେଶ କରିବା

୧. WordPress admin ମଧ୍ୟରେ, **Gratis AI Agent → Settings** କୁ ଯାଆନ୍ତୁ।
୨. **Advanced** ଟ୍ୟାବ୍‌ଟି ଉପରେ କ୍ଲିକ୍ କରନ୍ତୁ।

## Feedback Endpoint କନଫିଗୁରେସନ୍

AI agent ଦ୍ୱାରା ଯେତେବେଳେ କୌଣସି ବ୍ୟବହାରୀ ଥମ୍ବ୍-ଡାଉନ୍ ବଟନ୍ ମାଧ୍ୟମରେ, ଅଟୋ-ପ୍ରମୋଟ ବ୍ୟାନର (auto-prompt banner) ମାଧ୍ୟମରେ, କିମ୍ବା `/report-issue` କମାଣ୍ଡ ମାଧ୍ୟମରେ ଫିଡବକ୍ ଦେଉଥିବାରେ feedback endpoint ରେ POST requests ଆସିଥାଏ।

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | ଏହି URL ଟି HTTP POST request ସଦୃଶ JSON ବଡ଼ି (body) ସହିତ feedback submission ଗ୍ରହଣ କରେ। |
| **Feedback API Key** | ପ୍ରତ୍ୟେକ feedback request ର `Authorization` header ମଧ୍ୟରେ ଏହା ଏକ bearer token ଭାବରେ ପଠାଯାଏ। ଯଦି ଆପଣଙ୍କ endpoint ରେ ଅଥେଣ୍ଟିଫିକେସନ୍ (authentication) ଆବଶ୍ୟକ ନୁହେଁ, ତେବେ ଏହାକୁ ଖାଲି ଛାଡ଼ନ୍ତୁ। |

### ଅପେକ୍ଷା କରାଯାଉଥିବା JSON Payload

ଆପଣଙ୍କ feedback endpoint ରେ ନିମ୍ନଲିଖିତ ଫିଲ୍ଡଗୁଡ଼ିକ ସହିତ କମିତି କମିତ JSON body ଗ୍ରହଣ କରିବା ଆବଶ୍ୟକ:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

କଥାବାର୍ତ୍ତା (conversation) ସନ୍ଦର୍ଭ ଅନୁଯାୟୀ payload ରେ ଅଧିକ ଫିଲ୍ଡ ମଧ୍ୟ ଉପସ୍ଥିତ ହୋଇପାରେ।

### `triage_category` ମୂଲ୍ୟଗୁଡ଼ିକ

AI triage layer ଦ୍ୱାରା payload ପଠାଇବା ପୂର୍ବରୁ `triage_category` କୁ ନିମ୍ନଲିଖିତ ମଧ୍ୟରୁ ଗୋଟିଏ ମୂଲ୍ୟ ଦିଆଯାଏ:

| Value | Meaning |
|---|---|
| `factual_error` | ଅସହାୟ ସାହାଯ୍ୟକାରୀ ଠିକ ଭାବରେ ତଥ୍ୟ ପ୍ରଦାନ କରିଛି। |
| `unhelpful_answer` | ଉତ୍ତରଟି ଟେକ୍ନିକାଲି ଭାବରେ ଠିକ୍, କିନ୍ତୁ ବ୍ୟବହାରିକ ନଥିଲା। |
| `inappropriate_content` | ଉତ୍ତରରେ ଏପରି ବିଷୟବସ୍ତୁ ଥିଲା ଯାହାକୁ ବ୍ୟବହାରୀଙ୍କ ଦେଖାଇବା ଉଚିତ ନୁହେଁ। |
| `other` | feedback କୌଣସି ଜ୍ଞାନୀ କ୍ଷେତ୍ର ସହିତ ମେଳ ହୋଇନାହିଁ। |

### ଅଥେଣ୍ଟିଫିକେସନ୍

ଯଦି ଆପଣଙ୍କ ଇଣ୍ଟରଫେସ୍ (endpoint) କୌଣସି ପ୍ରକାର ଅଥେନ୍ଟିଫିକେସନ (authentication) ଆବଶ୍ୟକ କରେ, ତେବେ **Feedback API Key** ଫିଲ୍ଡକୁ ଆପଣଙ୍କ ବ୍ରିୟର ଟୋକେନ୍ (bearer token) ସେଟ୍ କରନ୍ତୁ। ଏଗ୍ରେଣ୍ଟ୍ ଏହା ପଠାଇଥାଏ:

```
Authorization: Bearer <your-api-key>
```

ଯଦି **Feedback API Key** ଫିଲ୍ଡଟି ଖାଲି ଥାଏ, ତେବେ କୌଣସି `Authorization` ହେଡର ପଠାଯିବ ନାହିଁ।

### Feedback ସଂଗ୍ରହ ବନ୍ଦ କରିବା (Disabling Feedback Collection)

**Feedback Endpoint URL** ଏବଂ **Feedback API Key** ଦୁଇଟି ଫିଲ୍ଡକୁ ଖାଲି ଛାଡ଼ନ୍ତୁ। ଥମ୍ସ-ଡାଉନ୍ ବଟନ୍ ଏବଂ ഫିଡବ୍ୟାକ୍ UI ବିଶେଷଜ୍ଞମାନଙ୍କ ପାଇଁ ଦେଖାଯିବ, କିନ୍ତୁ ସେଠାରୁ କୌଣସି ବାହ୍ୟ ସେବାକୁ ରିପୋର୍ଟ ଜମା ହେବ ନାହିଁ।

## Brave Search API Key

**Advanced** ଟ୍ୟାବରେ ମଧ୍ୟ, **Brave Search API Key** ଫିଲ୍ଡଟି [Internet Search](../configuration/internet-search) କ୍ଷମତାକୁ ସକାରଣ କରିଥାଏ।

| ଫିଲ୍ଡ | ବର୍ଣ୍ଣନା |
|---|---|
| **Brave Search API Key** | Brave Search ଡେଭଲପର ଡ୍ୟାଶବୋର୍ଡରୁ ଆପଣଙ୍କର API key। AI ଅସିଷ୍ଟେଣ୍ଟରେ ఇంటర్నెట్ ସାର୍ଚକୁ ଏନେବେଲ କରିବା ପାଇଁ ଏହା ଆବଶ୍ୟକ। |

ଫିଲ୍ଡର ନାମରେ Brave Search API ସାଇନ୍-ଅପ୍ ପେଜକୁ ଯିବା ପାଇଁ ଏକ କ୍ଲିକ କରିହଉଥିବା ଲିଙ୍କ ଅଛି। ఇంటర్నెట్ ସାର୍ଚ୍ ବିଷୟରେ ବ୍ୟବହାରକାରୀ ଦୃଷ୍ଟିକୋଣର ଦସ୍ତାବିତ ପାଇଁ [Internet Search](../configuration/internet-search) ଦେଖନ୍ତୁ।

## Feature Flags (ଫିଚର୍ ଫ୍ଲାଗ୍ସ)

v1.9.0 ମଧ୍ୟ ପ୍ରବେଶ କରାଯାଇଥିବା **Settings → Feature Flags** ଟ୍ୟାବରେ ଆପଣ ଅତିରିକ୍ତ କାର୍ଯ୍ୟକଳାପ ପାଇଁ toggles (on/off switches) ଦେଖିପାରିବେ। ପ୍ରତ୍ୟେକ ଫ୍ଲାଗ୍ ବ୍ୟାପକ ଭାବରେ ଏନେବଲ୍ଡ୍ ହୋଇଥାଏ କିମ୍ବା ଅନେବଲ୍ଡ୍, ଏଥିରେ ସମୟରେ କୌଣସି ସାଇଟ୍-ସ୍ପେସिफିକ୍ ଉପରେ ଅତିରିକ୍ତ ନିର୍ଦ୍ଦେଶ ଦେବା ସୁଯୋଗ ନାହିଁ।

### Feature Flagsକୁ ପ୍ରବେଶ କରିବା (Accessing Feature Flags)

1. WordPress admin ମଧ୍ୟରେ, **Gratis AI Agent → Settings** ରେ ଯାଆନ୍ତୁ।
2. **Feature Flags** ଟ୍ୟାବ୍‌ଟି ଉପରେ କ୍ଲିକ୍ କରନ୍ତୁ।

### Access Control Flags

| Flag | Default | ವಿವರಣೆ |
|---|---|---|
| **Administrator-ಗಳಿಗೆ મર્યાદિત કરો** | બંધ (Off) | જ્યારે આ ચાલુ કરવામાં આવે છે, ત્યારે ફક્ત `administrator` ભૂમિકા ધરાવતા વપરાશકર્તાઓ જ AI એજન્ટ ચેટ પેનલ ખોલી શકે છે. અન્ય તમામ ભૂમિકાઓને "તમારા એડમિન સાથે સંપર્ક કરો" નો મેસેજ દેખાશે. |
| **નેટવર્ક એડમિન-ಗಳಿಗೆ મર્યાદિત કરો** | બંધ (Off) | જ્યારે મલ્ટીસાઇટ નેટવર્કમાં ચાલુ કરવામાં આવે છે, ત્યારે ફક્ત સુપર એડમિન જ એજન્ટનો ઉપયોગ કરી શકે છે. વ્યક્તિગત સાઇટ એડમિનને અવરોધવામાં આવશે. જો બંને ચાલુ હોય તો આ ફ્લેગ "Administrator-ಗಳಿಗೆ મર્યાદિત કરો" કરતાં વધુ મહત્વ ધરાવે છે. |
| **સબ્સ્ક્રાઇબરને ઍક્સેસ આપો** | બંધ (Off) | જ્યારે આ ચાલુ કરવામાં આવે છે, ત્યારે `subscriber` ભૂમિકા ધરાવતા વપરાશકર્તાઓ ચેટ ઇન્ટરફેસનો ઉપયોગ કરી શકે છે પરંતુ તેઓ ફક્ત વાંચવા-ફક્ત ક્ષમતાઓ સુધી મર્યાદિત હોય છે (પોસ્ટ બનાવવી અથવા સેટિંગ્સ બદલવાનો કોઈ વિકલ્પ નહીં). |
| **બિન-સભ્ય માટે અક્ષમ કરો** | બંધ (Off) | આ Ultimate Multisite સભ્યપદની સ્થિતિ સાથે જોડાયેલું છે. જ્યારે ચાલુ કરવામાં આવે છે, ત્યારે જે સાઇટ્સ પાસે સક્રિય સભ્યપદ નથી તેના માટે ચેટ છુપાયેલી રહેશે. |

| Flag | Default | ವಿವರಣೆ |
|---|---|---|
| **"Gratis AI Agent દ્વારા સંચાલિત" ફૂટર છુપાવો** | બંધ (Off) | ચેટ વિજેટની નીચે દેખાતી બ્રાન્ડિંગ માહિતીને દૂર કરે છે. વ્હાઇટ-લેબલ ડિપ્લોયમેન્ટ માટે આ ભલામણ કરવામાં આવે છે. |

| **સાઇટ આઇકનનો ચેટ અવતાર તરીકે ઉપયોગ કરો** | બંધ (Off) | ચેટ વિજેટ હેડરમાં ડિફોલ્ટ AI આઇકનને વર્ડપ્રેસ સાઇટ આઇકન સાથે બદલે છે (Appearance → Customize → Site Identity માં સેટ કરેલું). |

### ફેરફારો લાગુ કરવા

કોઈપણ ફ્લેગને ટૉગલ કર્યા પછી **Save Settings** પર ક્લિક કરો. ફેરફારો તરત જ અમલમાં આવે છે — કેશ ફ્લશ અથવા પ્લગિન ફરીથી એક્ટિવેટ કરવાની જરૂર નથી.
