# କ୍ଷମତା ଦୃଶ୍ୟମାନତା (Ability Visibility)

Superdav AI Agent v1.12.0 ରେ **କ୍ଷମତା ଦୃଶ୍ୟମାନତା** ନାମକ ନିୟନ୍ତ୍ରଣଗୁଡ଼ିକ ଆପଣଙ୍କୁ ସେହି ସର୍ଫେସ୍‌କୁ କେଉଁ କ୍ଷମତା ପ୍ରଦାନ କରିବେ, ତାହା ବାହାର କରିବା ପାଇଁ ଦିଏ। ଏହା ଅଡମିନିଷ୍ଟ୍ରେଟର୍ମାନଙ୍କୁ REST API, MCP సర్ဗର, WooCommerce ସଂଯୋଗ ଏବଂ ଅନ୍ୟାନ୍ୟ ଆଇଣ୍ଟରଫେସ୍ ମାଧ୍ୟମରେ କେଉଁ ରିପୋଜିଟୋର କ୍ଷମତାଗୁଡ଼ିକ లେବେ, ତାହାକୁ ନିର୍ଦ୍ଧାରଣ କରିବା ପାଇଁ ସାହାଯ୍ୟ କରେ।

## କ୍ଷମତା ଦୃଶ୍ୟମାନତା କ'ଣ?

କ୍ଷମତା ଦୃଶ୍ୟମାନତା ହେଉଛି ଏକ ଅନୁମତି ବ୍ୟବସ୍ଥା ଯାହା ନିମ୍ନଲିଖିତ ବିଷୟଗୁଡ଼ିକ ନିୟନ୍ତ୍ରଣ କରେ:

- **କେଉଁ କ୍ଷମତାଗୁଡ଼ିକ** ଇଜେଣ୍ଟମାନଙ୍କ ପାଇଁ లେବେ
- **ସେଗୁଡ଼ିକ କେଉଁଠି ପ୍ରଦାନ ହେବ** (REST API, MCP, WooCommerce, ଆଦି)
- **କେଉଁମାନେ ଏହାକୁ ପ୍ରବେଶ କରିପାରିବେ** (ପାର୍ଶ୍ୱ ସୂଚୀ ମାଧ୍ୟମରେ)
- **ସେଗୁଡ଼ିକ କିପରି ବର୍ଗୀକୃତ ହୋଇଛି** (ଅନୁଲିକ୍ କରାଯାଇଛି ବା ଅନୁଲିକ୍ ନକରାଯାଇଛି)

ପ୍ରତିଟି କ୍ଷମତା ଏକ ଦୃଶ୍ୟମାନତା ସ୍ତର ବହନ କରେ ଯାହା ବିଭିନ୍ନ ସର୍ଫେସ୍ ଉପରେ ଏହାର లభ్యତାକୁ ନିର୍ଦ୍ଧାରଣ କରେ।

## ଦୃଶ୍ୟମାନତା ସ୍ତର (Visibility Levels)

### ଲୋକପ୍ରିୟ (Public)

**ଲୋକପ୍ରିୟ କ୍ଷମତାଗୁଡ଼ିକ** ସବୁଠାରେ లଭେ:

- REST API ଏଣ୍ଡପଏଣ୍ଟ
- MCP సర్ဗର
- WooCommerce ସଂଯୋଗ
- ଚ୍ୟାଟ ଇଣ୍ଟରଫେସ୍
- ସମସ୍ତ ୟୁଜର୍ ଭୂମିକା (ଉପଯୁକ୍ତ ଅନୁମତି ସହିତ)

ଉଦାହରଣ: `scaffold-block-theme`, `activate-theme`, `send-email`

### ଆଭ୍ୟନ୍ତରୀଣ (Internal)

**ଆଭ୍ୟନ୍ତରୀଣ କ୍ଷମତାଗୁଡ଼ିକ** କେବଳ ଆପଣଙ୍କ WordPress ଇନଷ୍ଟଲେସନ୍ ମଧ୍ୟରେ లେବେ:

- ଚ୍ୟାଟ ଇଣ୍ଟରଫେସ୍
- ଅଡମିନ୍ ପ୍ୟାନେଲ୍
- କେବଳ ଲଗ୍ଡ୍-ଇନ୍ ୟୁଜର୍ମାନେ
- REST API ବା ବାହ୍ୟ ସଂଯୋଗ ମାଧ୍ୟମରେ ପ୍ରଦାନ ନକରାଯାଇଛି

ଉଦାହରଣ: `manage-settings`, `view-analytics`, `export-data`

### ପାର୍ଶ୍ୱ (Partner)

**ପାର୍ଶ୍ୱ କ୍ଷମତାଗୁଡ଼ିକ** କେବଳ ଲିଷ୍ଟ୍ କରିଥିବା ପାର୍ଶ୍ୱ ସହଯୋଗୀଙ୍କ ପାଇଁ లେବେ:

- ଏକ ନିର୍ଦ୍ଧାରିତ ପାର୍ଶ୍ୱ ସୂଚୀରେ ପ୍ରବେଶ ଆବଶ୍ୟକ
- ପାର୍ଶ୍ୱ କ्रेडेंশিଆଲ୍ ମାଧ୍ୟମରେ REST API ମାଧ୍ୟମରେ లଭେ
- ପାର୍ଶ୍ୱ ଅଥେଣ୍ଟିଫିକେସନ୍ ମାଧ୍ୟମରେ MCP ମାଧ୍ୟମରେ లଭେ
- ସାଧାରଣ ୟୁଜର୍ମାନଙ୍କ ପାଇଁ ಲଭେ ନାହିଁ

ଉଦାହରଣ: `bulk-import-users`, `modify-billing`, `access-analytics`

### ବାର୍ଷିକ (Disabled)

**ବାର୍ଷିକ କ୍ଷମତାଗୁଡ଼ିକ** କୌଣସି ସର୍ଫେସ୍ ଉପରେ లଭେ ନାହିଁ:

- କୌଣସି ସର୍ଫେସ୍ ମାଧ୍ୟମରେ ପ୍ରଦାନ ନକରାଯାଇଛି
- ଚ୍ୟାଟରେ ಲଭେ ନାହିଁ
- ଅଡମିନ୍ ପ୍ୟାନେଲ୍‌ରେ ಲଭେ ନାହିଁ
- ବର୍ତ୍ତମାନ ବ୍ୟବହାର କରାଯାଉଥିବା କିମ୍ବା ପ୍ରୋତ୍ସାହନପୂର୍ଣ୍ଣ କ୍ଷମତାଗୁ

## କ୍ଷମତା ଦୃଶ୍ୟ (Ability Visibility) ପରିଚାଳନା କରିବା

### କ୍ଷମତା ଦୃଶ୍ୟ ସେଟିଙ୍ଗକୁ ପ୍ରବେଶ କରିବା

୧. **WordPress Admin** → **Superdav AI Agent** → **Settings** ଯିବେ
୨. **Abilities** ଟ୍ୟାବରେ (tab) କ୍ଲିକ୍ କରନ୍ତୁ
୩. ଆପଣ ଦେଖିବେ ସମସ୍ତ ଇନଷ୍ଟଲ୍ ହୋଇଥିବା କ୍ଷମତା ଏବଂ ସେଗୁଡ଼ିକର ଦୃଶ୍ୟ ସ୍ତର (visibility levels)

### କ୍ଷମତା ବିବରଣୀ ଦେଖିବା

କୌଣସି କ୍ଷମତା ଉପରେ କ୍ଲିକ୍ କରି ଦେଲେ ଆପଣ ଏହା ଜାଣିପାରିବେ:

- **Name**: ସେହି କ୍ଷମତାର ପରିଚୟ (identifier)
- **Description**: ଏହି କ୍ଷମତାଟି କ'ଣ କରୁଛି
- **Current visibility**: ଉପଲବ୍ଧ (Public), ଆନ୍ତର୍ଜାତିକ (Internal), ଭାଗିଦାର (Partner), ବା ଅବଶୀଦ (Disabled)
- **Partner allow-list**: କେଉଁ ଭାଗିଦାରମାନେ ପ୍ରବେଶ କରିପାରିବେ (ଯଦି Partner visibility ଥାଏ)
- **Last updated**: ଦୃଶ୍ୟ ଶେଷ କେବେ ବଦଳା ଯାଇଛି
- **Status**: ଚିହ୍ନଟ କରାଯାଇଛି (Recognized) ବା ଅବଶୀଦ (Unclassified)

### ଦୃଶ୍ୟ ସ୍ତର ବଦଳାଇବା

କ୍ଷମତା ଏବଂ ଏହାର ଦୃଶ୍ୟ ସ୍ତର ବଦଳାଇବା ପାଇଁ:

୧. ତା list ମଧ୍ୟରେ କ୍ଷମତା ଉପରେ କ୍ଲିକ୍ କରନ୍ତୁ
୨. డ୍ରପଡାଉନ୍ (dropdown) ରୁ ନୂଆ ଦୃଶ୍ୟ ସ୍ତର ବାଛନ୍ତୁ
୩. ଯଦି "Partner" ବାଛିଲେ, ଆପଣ allow-list ମଧ୍ୟରେ ଭାଗିଦାର ID ଲେଖିପାରିବେ
୪. **Save** କ୍ଲିକ୍ କରନ୍ତୁ

ଉଦାହରଣ:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### ବ୍ୟବଧାନୀୟ କାର୍ଯ୍ୟ (Bulk Operations)

ଅନେକ କ୍ଷମତାର ଦୃଶ୍ୟ ସ୍ତର ବଦଳାଇବା ପାଇଁ:

୧. କ୍ଷମତାଗୁଡ଼ିକ ପାଖରେ ଥିବା ବକ୍ସ ମାର୍କ କରିବନ୍ତୁ
୨. ବ୍ୟବଧାନୀୟ କାର୍ଯ୍ୟ (bulk action) డ୍ରପଡାଉନ୍ ରୁ ନୂଆ ଦୃଶ୍ୟ ସ୍ତର ବାଛନ୍ତୁ
୩. **Apply** କ୍ଲିକ୍ କରନ୍ତୁ

## ଭାଗିଦାର ଇଜ୍ୱଳସ (Partner Allow-List)

**partner allow-list** ହେଉଛି ଏହା ଯେଉଁ ଅପେକ୍ଷୀ ଭାଗିଦାରମାନେ Partner-level କ୍ଷମତାକୁ ପ୍ରବେଶ କରିପାରିବେ, ତାହା ନିର୍ଦ୍ଧାରଣ କରେ।

### ଭାଗିଦାରମାନଙ୍କୁ ଯୋଡ଼ିବା (Adding Partners)

୧. **Superdav AI Agent** → **Settings** → **Partners** ଯିବେ
୨. **Add Partner** କ୍ଲିକ୍ କରନ୍ତୁ
୩. ଭାଗିଦାର ID (ଯାହା ସାଧାରଣତଃ ଏକ API key ବା ଅର୍ଗଣାଇଜେସନ୍ ID) ପ୍ରବେଶ କରନ୍ତୁ
୪. ଆପ୍ସନସିଆର, ଏକ ଭାଗିଦାର ନାମ ଏବଂ ବିବରଣୀ ଯୋଡ଼ିପାରନ୍ତି
୫. **Save** କ୍ଲିକ୍ କରନ୍ତୁ

### ଭାଗିଦାରଙ୍କ ପାଇଁ କ୍ଷମତା ମୁଖ୍ୟିକରଣ କରିବା (Assigning Abilities to Partners)

ଭାଗିଦାର ଯୋଡ଼ିବା ପରେ:

১. **Abilities** ট্যাবে যান
২. পার্টনার দৃশ্যমানতা (Partner visibility) সহ একটি অ্যাবিলিটি খুঁজুন
৩. সম্পাদনা করার জন্য ক্লিক করুন
৪. "Partner allow-list" অংশে, যে পার্টনারদের অ্যাক্সেস থাকা উচিত তাদের বক্সে টিক দিন
৫. **Save** এ ক্লিক করুন

### পার্টনার অ্যাক্সেস বাতিল করা (Revoking Partner Access)

একটি পার্টনারের অ্যাক্সেস সরানোর জন্য:

১. অ্যাবিলিটিটি সম্পাদনা করুন
২. অনুমোদিত তালিকায় (allow-list) পার্টনারের বক্সটি আনচেক করুন
৩. **Save** এ ক্লিক করুন

পার্টনারটি অবিলম্বে সেই অ্যাবিলিটির অ্যাক্সেস হারাতে পারবে।

## অশ্রেণীবদ্ধ অ্যাবিলিটি (Unclassified Abilities)

যখন আপনি একটি থার্ড-পার্টি অ্যাবিলিটি ইনস্টল করেন যা Superdav AI Agent চিনতে পারে না, তখন এটিকে **Unclassified** হিসাবে চিহ্নিত করা হয়।

### অশ্রেণীবদ্ধ অ্যাবিলিটির জন্য অ্যাডমিন নোটিশ (Admin Notice for Unclassified Abilities)

আপনি একটি অ্যাডমিন নোটিশ দেখতে পাবেন:

```
⚠️ Unclassified Abilities Detected

নিম্নলিখিত অ্যাবিলিটিগুলি ইনস্টল করা হয়েছে কিন্তু স্বীকৃত নয়:
- custom-import-tool
- external-api-wrapper

এই অ্যাবিলিটিগুলির নিরাপত্তা বা সামঞ্জস্যের ঝুঁকি থাকতে পারে।
অনুগ্রহ করে পর্যালোচনা করুন এবং শ্রেণীবদ্ধ করুন।

[Review Abilities] [Dismiss]
```

### অশ্রেণীবদ্ধ অ্যাবিলিটি পর্যালোচনা করা (Reviewing Unclassified Abilities)

১. নোটিশে **Review Abilities** এ ক্লিক করুন
২. প্রতিটি অশ্রেণীবদ্ধ অ্যাবিলিটির জন্য, আপনি দেখতে পাবেন:
   - অ্যাবিলিটির নাম এবং বর্ণনা
   - উৎস (কোন প্লাগইন/অ্যাডon এটি ইনস্টল করেছে)
   - প্রস্তাবিত দৃশ্যমানতার স্তর (Suggested visibility level)
   - নিরাপত্তা মূল্যায়ন (Security assessment)

৩. একটি দৃশ্যমানতার স্তর বেছে নিন:
   - **Public**: অ্যাবিলিটিকে বিশ্বাস করুন; সব জায়গায় উন্মুক্ত রাখুন
   - **Internal**: শুধুমাত্র অভ্যন্তরীণ ব্যবহারের জন্য সীমাবদ্ধ করুন
   - **Partner**: নির্দিষ্ট পার্টনারদের জন্য সীমাবদ্ধ করুন
   - **Disabled**: এই অ্যাবিলিটি ব্যবহার করবেন না

৪. আপনার সিদ্ধান্ত সেভ করতে **Classify** এ ক্লিক করুন

### অশ্রেণীবদ্ধ অ্যাবিলিটি কেন শ্রেণীবদ্ধ করা উচিত? (Why Classify Unclassified Abilities?)

অ্যাবিলিটিগুলিকে শ্রেণীবদ্ধ করার কারণ:

- **নিরাপত্তা উন্নত করে**: আপনি স্পষ্টভাবে কী উন্মুক্ত করছেন তা অনুমোদন করেন
- **দুর্ঘটনা প্রতিরোধ করে**: অশ্রেণীবদ্ধ অ্যাবিলিটিগুলি ডিফল্টভাবে নিষ্ক্রিয় থাকে
- **বৈশিষ্ট্য সক্ষম করে**: একবার শ্রেণীবদ্ধ হলে, অ্যাবিলিটিটি ব্যবহারের জন্য উপলব্ধ হয়ে যায়
- **সিদ্ধান্ত নথিভুক্ত করে**: নিরীক্ষার (audit) উদ্দেশ্যে আপনার পছন্দগুলি লগ করা হয়

## ବିଭିନ୍ନ ସର୍ଫେସ୍ ରେ ଦୃଶ୍ୟମାନତା (Visibility in Different Surfaces)

### REST API

**ସାଧାରଣ ଅନୁପଯୋଗୀ କ୍ଷମତାଗୁଡ଼ିକ** REST endpoints ମାଧ୍ୟମରେ ಲହୀନ ହୋଇଥାଏ:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**ଆନ୍ତର୍ଜନୀକର ଏବଂ ପାର୍ଟନର୍ କ୍ଷମତାଗୁଡ଼ିକ** REST API ମାଧ୍ୟମରେ ಲହୀନ ନୁହେଁ।

### MCP Servers

**ସାଧାରଣ ଅନୁପଯୋଗୀ କ୍ଷମତାଗୁଡ଼ିକ** MCP ମାଧ୍ୟମରେ లହୀନ ହୋଇଥାଏ:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**ଆନ୍ତର୍ଜନୀକର କ୍ଷମତାଗୁଡ଼ିକ** MCP ମାଧ୍ୟମରେ ପ୍ରଦର୍ଶିତ ହୁଏ ନାହିଁ।

**ପାର୍ଟନର୍ କ୍ଷମତାଗୁଡ଼ିକ** କେବଳ ପାର୍ଟନର୍ କ୍ରେଡେନସିଆଲ୍ ସହିତ ಲହୀନ ହୋଇଥାଏ।

### WooCommerce Integration

**WooCommerce ସମ୍ବନ୍ଧୀୟ ସାଧାରଣ ଅନୁପଯୋଗୀ କ୍ଷମତାଗୁଡ଼ିକ** లହୀନ ହୋଇଥାଏ:

- ଉତ୍ପାଦ ପରିଚାଳନା (Product management)
- ଆରେଡର ପ୍ରକ୍ରିୟା (Order processing)
- ଗ୍ରାହକ ସମ୍ପର୍କ (Customer communication)

**ଆନ୍ତର୍ଜନୀକର କ୍ଷମତାଗୁଡ଼ିକ** WooCommerce ମଧ୍ୟରେ ପ୍ରଦର୍ଶିତ ହୁଏ ନାହିଁ।

### ଚ୍ୟାଟ ଇଣ୍ଟରଫେସ୍

**ସମସ୍ତ କ୍ଷମତାଗୁଡ଼ିକ** (ସାଧାରଣ, ଆନ୍ତର୍ଜନୀକର, ପାର୍ଟନର୍) ଚ୍ୟାଟରେ ಲହୀନ ହୋଇଥାଏ, ଯାହା ବ୍ୟବହାରକାରୀ ଅନୁମତି ଉପରେ ନିର୍ଭର କରେ:

- ପ୍ରଶାସକମାନେ ସମସ୍ତ କ୍ଷମତା ଦେଖନ୍ତି
- ସାଧାରଣ ବ୍ୟବହାରକାରୀମାନେ କେବଳ ସାଧାରଣ କ୍ଷମତା ଦେଖନ୍ତି
- ପାର୍ଟନର୍ମାନେ ସାଧାରଣ + ପାର୍ଟନର୍ କ୍ଷମତା (ଯଦି ୱାଇଟିଷ୍ଟ୍ ହୋଇଥାଆନ୍ତି) ଦେଖନ୍ତି

## ସୁରକ୍ଷା ନିୟମବିଧାନ (Security Best Practices)

### ଅପମାନୀର ସବୁଠାରୁ କମ୍ ଅଧିକାର ପ୍ରଣାଳୀ (Principle of Least Privilege)

- କ୍ଷମତାଗୁଡ଼ିକୁ ସେହି ସବୁଠାରୁ କଠୋର ଦୃଶ୍ୟମାନତା ରେ ସେଟ କରନ୍ତୁ ଯାହା ଆପଣଙ୍କୁ ବ୍ୟବହାର କରିବା ପାଇଁ ଅନୁମତି ଦେଉଛି।
- ସୂକ୍ଷ୍ମ କାର୍ଯ୍ୟକଳୟଗୁଡ଼ିକ ପାଇଁ ପାର୍ଟନର୍ ଦୃଶ୍ୟମାନତା ବ୍ୟବହାର କରନ୍ତୁ।
- ଆପଣ ବ୍ୟବହାର କରୁଥିବା କ୍ଷମତାଗୁଡ଼ିକୁ ବନ୍ଦ କରନ୍ତୁ ନାହିଁ ଯାହା ଆପଣ ବ୍ୟବହାର କରୁନାହାନ୍ତି।

### ନିୟମିତ ଅଡିଟ୍ (Regular Audits)

- ମାସିକ ଭାବରେ କ୍ଷମତା ଦୃଶ୍ୟମାନତାକୁ ଯାଞ୍ଚ କରନ୍ତୁ
- ଅବଗତ ବିହୀନ କ୍ଷମତାଗୁଡ଼ିକ ଥିବା ପାଇଁ ଯାଞ୍ଚ କରନ୍ତୁ
- ବ୍ୟବହାର ନକରିଥିବା ପାର୍ଟନର୍ମାନଙ୍କ ପାଇଁ ପ୍ରବେଶ ବନ୍ଦ କରନ୍ତୁ

### ଲଗ୍ ଆଣ୍ଡ ମନିଟରିଂ (Logging and Monitoring)

- କେଉଁ କ୍ଷମତାଗୁଡ଼ିକ ସବୁଠାରୁ ଅଧିକ ବ୍ୟବହୃତ ହେଉଛି, ତାହା ମାନିଟର କରନ୍ତୁ
- ପାର୍ଟନର୍ ପ୍ରବେଶ ଚାପକୁ ଟ୍ରାକ୍ କରନ୍ତୁ
- ଅସାଧାରଣ କ୍ଷମତା ବ୍ୟବହାର ପାଇଁ ଆଲର୍ଟ ଦିଅନ୍ତୁ

## ట్రబుల్షూటింగ్ (Troubleshooting)

**లిస్ట్‌లో ఒక అబిలిటీ కనిపించడం లేదు**
- ఆ అబిలిటీ ఇన్‌స్టాల్ చేయబడి, యాక్టివ్‌గా ఉందో లేదో తనిఖీ చేయండి.
- ప్లగిన్/అడ్డాన్ ఎనేబుల్ (enabled) అయిందో లేదో చూడండి.
- పేజీని రిఫ్రెష్ చేయండి.

**నేను ఒక అబిలిటీ యొక్క వీక్షణను మార్చలేను**
- మీకు అడ్మినిస్ట్రేటర్ అనుమతులు ఉన్నాయో లేదో తనిఖీ చేయండి.
- ప్లగిన్ ద్వారా ఆ అబిలిటీ లాక్ చేయబడిందో లేదో చూడండి.
- విభేదించే (conflicting) ప్లగిన్‌లను డిసేబుల్ చేయడానికి ప్రయత్నించండి.

**ఒక భాగస్వామికి ఒక అబిలిటీ యాక్సెస్ అవ్వడం లేదు**
- భాగస్వామి ఆ ఆలు (allow-list) లో ఉందో లేదో తనిఖీ చేయండి.
- ఆ అబిలిటీ వీక్షణ 'Partner' గా సెట్ చేయబడిందో లేదో చూడండి.
- భాగస్వామి వివరాలు సరిగ్గా ఉన్నాయో లేదో నిర్ధారించుకోండి.
- ప్రామాణీకరణ (authentication) లోపాలను తనిఖీ చేయడానికి API లాగ్‌లను చూడండి.

**వర్గీకరించబడని అబిలిటీలు కనిపిస్తూనే ఉన్నాయి**
- అడ్మిన్ నోటిఫికేషన్‌లో వాటిని సమీక్షించి, వర్గీకరించండి.
- మీరు చేసిన వర్గీకరణ సేవ్ అయ్యిందో లేదో తనిఖీ చేయండి.
- ఆ అబిలిటీని అందించే ప్లగిన్ తాజా వెర్షన్‌లో ఉందో లేదో నిర్ధారించుకోండి.

## లెగసీ మోడ్ నుండి మైగ్రేషన్ (Migration from Legacy Mode)

మీరు సూపర్డేవ్ AI ఏజెంట్ (Superdav AI Agent) యొక్క పాత వెర్షన్ నుండి అప్‌గ్రేడ్ చేస్తుంటే:

- అన్ని ప్రస్తుత అబిలిటీలు ఆటోమేటిక్‌గా 'Public' గా వర్గీకరించబడతాయి.
- అవసరమైన విధంగా వీక్షణను సమీక్షించి, సర్దుబాటు చేయండి.
- మీరు యాక్సెస్ పరిమితం చేయాలనుకుంటే తప్ప ఎలాంటి చర్యలు తీసుకోనవసరం లేదు.

నేటివే అబిలిటీస్ API ఇంటిగ్రేషన్‌కు మారడం గురించి మరిన్ని వివరాల కోసం **థర్డ్-పార్టీ మోడ్ మైగ్రేషన్** చూడండి.

## తదుపరి దశలు (Next Steps)

అబిలిటీ వీక్షణను కాన్ఫిగర్ చేసిన తర్వాత:

1. **మీ అబిలిటీలను సమీక్షించండి**: ఇన్‌స్టాల్ చేయబడిన అన్ని అబిలిటీలను ఆడిట్ చేయండి.
2. **వర్గీకరించబడని అబిలిటీలను వర్గీకరించండి**: అడ్మిన్ నోటిఫికేషన్‌లకు స్పందించండి.
3. ** భాగస్వాములను సెటప్ చేయండి**: మీరు పార్టనర్ వీక్షణను ఉపయోగిస్తుంటే భాగస్వాములను జోడించండి.
4. **వాడుకను పర్యవేక్షించండి**: ఏ అబిలిటీలు ఎక్కువగా వాడుతున్నాయో ట్రాక్ చేయండి.
5. **నిర్ణయాలను డాక్యుమెంట్ చేయండి**: మీరు ప్రతి వీక్షణ స్థాయికి ఎందుకు ఎంచుకున్నారో నోట్స్ ఉంచుకోండి.
