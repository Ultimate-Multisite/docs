# ସୁସଙ୍ଗତ ଅନ୍ତ୍ୟବିନ୍ଦୁ ପାଇଁ Ultimate AI Connector {#ultimate-ai-connector-for-compatible-endpoints}

ସୁସଙ୍ଗତ ଅନ୍ତ୍ୟବିନ୍ଦୁ ପାଇଁ Ultimate AI Connector, OpenAI-ସୁସଙ୍ଗତ AI ପ୍ରଦାତା APIଗୁଡ଼ିକୁ ଆପଣଙ୍କ WordPress ମଲ୍ଟିସାଇଟ୍ ନେଟୱର୍କ ସହିତ ଏକତ୍ର କରେ। ଏକାଧିକ AI ପ୍ରଦାତା ବିନ୍ୟାସ କରନ୍ତୁ — ପ୍ରତ୍ୟେକର ନିଜସ୍ୱ ଅନ୍ତ୍ୟବିନ୍ଦୁ URL, API କୀ, ଏବଂ ପୂର୍ବନିର୍ଦ୍ଧାରିତ ମଡେଲ୍ ସହିତ — ଏବଂ କୌଣସି ପ୍ରଦାତା ଉପଲବ୍ଧ ନଥିଲେ ସ୍ୱୟଂଚାଳିତ ବିକଳ୍ପ ସହିତ ଅନୁରୋଧଗୁଡ଼ିକୁ ମାର୍ଗଦର୍ଶନ କରନ୍ତୁ।

## ମୁଖ୍ୟ ବୈଶିଷ୍ଟ୍ୟ {#key-features}

- **ଏକାଧିକ ପ୍ରଦାତା ସମର୍ଥନ**: ଏକାଧିକ AI ପ୍ରଦାତା କାର୍ଡ ଯୋଡନ୍ତୁ, ପ୍ରତ୍ୟେକର ନିଜସ୍ୱ ଅନ୍ତ୍ୟବିନ୍ଦୁ URL, API କୀ, ଏବଂ ପୂର୍ବନିର୍ଦ୍ଧାରିତ ମଡେଲ୍ ସହିତ
- **ଟାଣି ପୁନଃକ୍ରମ କରନ୍ତୁ**: ପ୍ରତ୍ୟେକ କାର୍ଡର ଗ୍ରିପ୍ ହ୍ୟାଣ୍ଡଲ୍ ବ୍ୟବହାର କରି ପ୍ରାଥମିକତା ଅନୁଯାୟୀ ପ୍ରଦାତାମାନଙ୍କୁ ସଜାନ୍ତୁ; ଅନୁରୋଧଗୁଡ଼ିକ କ୍ରମ ଅନୁଯାୟୀ ପ୍ରଦାତାମାନଙ୍କୁ ବ୍ୟବହାର କରେ
- **ସ୍ୱୟଂଚାଳିତ ବିକଳ୍ପ ମାର୍ଗଦର୍ଶନ**: ପ୍ରାଥମିକ ପ୍ରଦାତା ଉପଲବ୍ଧ ନଥିଲେ, ଅନୁରୋଧଗୁଡ଼ିକ ସ୍ୱୟଂଚାଳିତ ଭାବେ ପରବର୍ତ୍ତୀ ବିନ୍ୟାସିତ ପ୍ରଦାତାକୁ ଯାଏ
- **ପ୍ରତି ପ୍ରଦାତା ବିନ୍ୟାସ**: ପ୍ରତ୍ୟେକ ପ୍ରଦାତା ନିଜସ୍ୱ ପରିଚୟପତ୍ର ଏବଂ ମଡେଲ୍ ଚୟନ ସହିତ ସ୍ୱାଧୀନ ଭାବେ ବିନ୍ୟାସିତ ହୁଏ
- **କାର୍ଡ-ଆଧାରିତ ସେଟିଙ୍ଗ୍ UI**: ସ୍ଥିର WordPress 6.9+ Card ଉପାଦାନଗୁଡ଼ିକ (CardBody, CardHeader, CardDivider) ଉପରେ ନିର୍ମିତ
- **ଯୋଡାଗଲେ ସ୍ୱୟଂ-ବିସ୍ତାର**: ନୂଆ ପ୍ରଦାତା କାର୍ଡଗୁଡ଼ିକ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବିସ୍ତାରିତ ହୁଏ, ଯାହାରେ ଆପଣ ତୁରନ୍ତ ବିବରଣୀ ପୂରଣ କରିପାରିବେ
- **ସ୍କ୍ରିପ୍ଟ କ୍ୟାଶ୍ ବସ୍ଟିଂ**: ସବୁଠାରୁ ନୂତନ ସଂସ୍କରଣ ଲୋଡ୍ ହେବା ନିଶ୍ଚିତ କରିବାକୁ plugin ଅଦ୍ୟତନରେ ପ୍ରଦାତା ସ୍କ୍ରିପ୍ଟଗୁଡ଼ିକର କ୍ୟାଶ୍ ଭଙ୍ଗ କରାଯାଏ

## ଆବଶ୍ୟକତା {#requirements}

- WordPress 5.3 କିମ୍ବା ତାଠାରୁ ଉଚ୍ଚ
- PHP 7.4 କିମ୍ବା ତାଠାରୁ ଉଚ୍ଚ
- Ultimate Multisite plugin (ସକ୍ରିୟ)
- ଏକ OpenAI-ସୁସଙ୍ଗତ API ଅନ୍ତ୍ୟବିନ୍ଦୁ (ନିଜେ ହୋଷ୍ଟ କରାଯାଇଥିବା କିମ୍ବା ତୃତୀୟ-ପକ୍ଷ)

## ସ୍ଥାପନ {#installation}

1. ଆଡଅନ୍ ଫାଇଲଗୁଡ଼ିକୁ ଆପଣଙ୍କ `/wp-content/plugins/` ଡିରେକ୍ଟରୀକୁ ଅପଲୋଡ୍ କରନ୍ତୁ
2. WordPress ର 'ପ୍ଲଗଇନ୍‌ଗୁଡ଼ିକ' ମେନୁ ମାଧ୍ୟମରେ plugin ସକ୍ରିୟ କରନ୍ତୁ
3. ଆଡଅନ୍ ବିନ୍ୟାସ କରିବାକୁ **Ultimate Multisite → AI Connector** କୁ ଯାଆନ୍ତୁ

## ବିନ୍ୟାସ {#configuration}

### ଏକାଧିକ ପ୍ରଦାତା ସେଟଅପ୍ {#multi-provider-setup}

ବିନ୍ୟାସ ପରଦା ଏକ କାର୍ଡ-ଆଧାରିତ ଲେଆଉଟ୍ ବ୍ୟବହାର କରେ, ଯେଉଁଠାରେ ପ୍ରତ୍ୟେକ କାର୍ଡ ଗୋଟିଏ AI ପ୍ରଦାତାକୁ ଦର୍ଶାଏ। ଆପଣ ଆବଶ୍ୟକ ଅନୁଯାୟୀ ଯେତେ ପ୍ରଦାତା ଚାହୁଁଛନ୍ତି ଯୋଡିପାରିବେ।

#### ପ୍ରଦାତା ଯୋଡିବା {#adding-a-provider}

1. ନେଟୱର୍କ ପ୍ରଶାସନରେ **Ultimate Multisite → AI Connector** କୁ ଯାଆନ୍ତୁ
2. **ପ୍ରଦାତା ଯୋଡନ୍ତୁ** କ୍ଲିକ୍ କରନ୍ତୁ
3. ଏକ ନୂଆ ପ୍ରଦାତା କାର୍ଡ ଦେଖାଯାଏ ଏବଂ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବିସ୍ତାରିତ ହୁଏ
4. ପ୍ରଦାତା ବିବରଣୀ ପୂରଣ କରନ୍ତୁ:
   - **ପ୍ରଦାତା ନାମ** — ଏହି ପ୍ରଦାତାକୁ ଚିହ୍ନଟ କରିବା ପାଇଁ ଏକ ଲେବେଲ୍ (ଯଥା "OpenAI", "Mistral", "Local Ollama")
   - **ଅନ୍ତ୍ୟବିନ୍ଦୁ URL** — OpenAI-ସୁସଙ୍ଗତ API ର ଆଧାର URL (ଯଥା `https://api.openai.com/v1`)
   - **API କୀ** — ଏହି ପ୍ରଦାତା ପାଇଁ ଆପଣଙ୍କ API କୀ
   - **ପୂର୍ବନିର୍ଦ୍ଧାରିତ ମଡେଲ୍** — ଏହି ପ୍ରଦାତା ପାଇଁ ପୂର୍ବନିର୍ଦ୍ଧାରିତ ଭାବେ ବ୍ୟବହାର କରିବାକୁ ମଡେଲ୍ ପରିଚୟକ (ଯଥା `gpt-4o`, `mistral-small`)
5. **ସେଟିଙ୍ଗ୍ ସଞ୍ଚୟ କରନ୍ତୁ** କ୍ଲିକ୍ କରନ୍ତୁ

#### ପ୍ରଦାତା ହଟାଇବା {#removing-a-provider}

କୌଣସି ପ୍ରଦାତା କାର୍ଡରେ ଥିବା **ହଟାନ୍ତୁ** ବଟନ୍ କ୍ଲିକ୍ କରି ଏହାକୁ ଡିଲିଟ୍ କରନ୍ତୁ। ପରିବର୍ତ୍ତନ ଲାଗୁ କରିବାକୁ ସେଟିଙ୍ଗ୍ ସଞ୍ଚୟ କରନ୍ତୁ।

#### ପ୍ରଦାତାମାନଙ୍କୁ ପୁନଃକ୍ରମ କରିବା {#reordering-providers}

କୌଣସି ପ୍ରଦାତା କାର୍ଡର ବାମ ପାର୍ଶ୍ୱରେ ଥିବା ଗ୍ରିପ୍ ଆଇକନ୍ (⣿) ଟାଣି ଏହାକୁ ପୁନଃକ୍ରମ କରନ୍ତୁ। କ୍ରମ ବିକଳ୍ପ ପ୍ରାଥମିକତା ନିର୍ଦ୍ଧାରଣ କରେ: ଅନୁରୋଧଗୁଡ଼ିକ ପ୍ରଥମ ପ୍ରଦାତାକୁ ପଠାଯାଏ ଏବଂ ଅନୁରୋଧ ବିଫଳ ହେଲେ ପରବର୍ତ୍ତୀ ପ୍ରଦାତାମାନଙ୍କୁ ଯାଏ।

### ବିକଳ୍ପ ମାର୍ଗଦର୍ଶନ {#fallback-routing}

ସମୟସୀମା ଶେଷ, ହାର ସୀମା, କିମ୍ବା ସେବା ବନ୍ଦ ଥିବା କାରଣରୁ ପ୍ରାଥମିକ ପ୍ରଦାତାକୁ ଅନୁରୋଧ ବିଫଳ ହେଲେ, କନେକ୍ଟର୍ ସୂଚୀର ପରବର୍ତ୍ତୀ ପ୍ରଦାତା ସହିତ ସ୍ୱୟଂଚାଳିତ ଭାବେ ପୁନଃଚେଷ୍ଟା କରେ। କୌଣସି ପ୍ରଦାତା ସଫଳ ହେବା କିମ୍ବା ସମସ୍ତ ପ୍ରଦାତାଙ୍କୁ ଚେଷ୍ଟା କରାଯିବା ପର୍ଯ୍ୟନ୍ତ ଏହା ସୂଚୀ ଅନୁଯାୟୀ ଚାଲିଥାଏ।

| ପରିସ୍ଥିତି | ବ୍ୟବହାର |
|----------|-----------|
| ପ୍ରାଥମିକ ପ୍ରଦାତା ପ୍ରତିକ୍ରିୟା ଦେଏ | ଅନୁରୋଧ ସାଧାରଣ ଭାବେ ସେବା ଦିଆଯାଏ |
| ପ୍ରାଥମିକ ପ୍ରଦାତା ବିଫଳ ହୁଏ | ପରବର୍ତ୍ତୀ ପ୍ରଦାତା ସହିତ ଅନୁରୋଧ ପୁନଃଚେଷ୍ଟା କରାଯାଏ |
| ସମସ୍ତ ପ୍ରଦାତା ବିଫଳ ହୁଅନ୍ତି | କଲ୍ କରୁଥିବା ବ୍ୟକ୍ତିଙ୍କୁ ତ୍ରୁଟି ଫେରାଯାଏ |

ବିକଳ୍ପ ବ୍ୟବସ୍ଥା କଲ୍ କରୁଥିବା plugin କିମ୍ବା theme ପାଇଁ ପାରଦର୍ଶୀ — କେତେ ପ୍ରଦାତାଙ୍କୁ ଚେଷ୍ଟା କରାଯାଇଥିଲା ତାହା ଯାହାହେଉ, ସେମାନେ ସଫଳ ପ୍ରତିକ୍ରିୟା କିମ୍ବା ଶେଷ ତ୍ରୁଟି ପାଆନ୍ତି।

## ସମସ୍ୟା ସମାଧାନ {#troubleshooting}

### କୌଣସି ପ୍ରଦାତା ତାଲିକାଭୁକ୍ତ ନାହାନ୍ତି {#no-providers-are-listed}

ସଞ୍ଚୟ କରିବା ପରେ ପ୍ରଦାତା ସୂଚୀ ଖାଲି ଥିଲେ, ନିଶ୍ଚିତ କରନ୍ତୁ ଯେ:

- ପ୍ରଦାତା ଯୋଡିବା ପରେ ଆପଣ **ସେଟିଙ୍ଗ୍ ସଞ୍ଚୟ କରନ୍ତୁ** କ୍ଲିକ୍ କରିଥିଲେ
- ଆପଣଙ୍କ ବ୍ରାଉଜର୍ ସେଟିଙ୍ଗ୍ ପୃଷ୍ଠାକୁ ସଞ୍ଚୟ କରିବାରୁ ଅବରୋଧ କରିନଥିଲା (JavaScript ତ୍ରୁଟି ପାଇଁ ବ୍ରାଉଜର୍ କନସୋଲ୍ ଯାଞ୍ଚ କରନ୍ତୁ)
- plugin ସକ୍ରିୟ ଅଛି ଏବଂ ଆପଣ ନେଟୱର୍କ ପ୍ରଶାସନ ପରଦାରେ ଅଛନ୍ତି

### ସମସ୍ତ ପ୍ରଦାତା ପାଇଁ ଅନୁରୋଧ ବିଫଳ ହେଉଛି {#requests-are-failing-for-all-providers}

- ପ୍ରତ୍ୟେକ ପ୍ରଦାତାର ଅନ୍ତ୍ୟବିନ୍ଦୁ URL ଆପଣଙ୍କ ସର୍ଭରରୁ ପହଞ୍ଚିବାଯୋଗ୍ୟ କି ନାହିଁ ଯାଞ୍ଚ କରନ୍ତୁ (କେବଳ ଆପଣଙ୍କ ବ୍ରାଉଜରରୁ ନୁହେଁ)
- ପ୍ରତ୍ୟେକ API କୀ ବୈଧ ଏବଂ ମେୟାଦ ଶେଷ ହୋଇନାହିଁ କି ନାହିଁ ଯାଞ୍ଚ କରନ୍ତୁ
- ସେମାନଙ୍କ Dashboard ରେ ପ୍ରଦାତାର ହାର-ସୀମା ଏବଂ କୋଟା ସ୍ଥିତି ସମୀକ୍ଷା କରନ୍ତୁ
- କନେକ୍ଟର୍ ଠାରୁ ବିସ୍ତୃତ ତ୍ରୁଟି ସନ୍ଦେଶ ଧରିବାକୁ WordPress ଡିବଗ୍ ଲଗିଂ (`WP_DEBUG_LOG`) ସକ୍ରିୟ କରନ୍ତୁ

### ବିକଳ୍ପ ପାଇଁ ଏକ ପ୍ରଦାତା ବ୍ୟବହୃତ ହେଉନାହିଁ {#a-provider-is-not-being-used-for-fallback}

ଟାଣି ପୁନଃକ୍ରମ କରିବା ବିକଳ୍ପ କ୍ରମକୁ ପରିବର୍ତ୍ତନ କରେ। ଆପଣ ଯେ ପ୍ରଦାତାକୁ ବିକଳ୍ପ ଅନୁରୋଧ ପାଇବାକୁ ଆଶା କରୁଛନ୍ତି, ସେହି ପ୍ରଦାତା ସୂଚୀରେ ଠିକ୍ ସ୍ଥାନରେ ଅଛି ଏବଂ ତାହାର ଅନ୍ତ୍ୟବିନ୍ଦୁ URL ଓ API କୀ ଠିକ୍ ଅଛି କି ନାହିଁ ନିଶ୍ଚିତ କରନ୍ତୁ।

### ନୂଆ ପ୍ରଦାତା କାର୍ଡ ସ୍ୱୟଂଚାଳିତ ଭାବେ ବିସ୍ତାରିତ ହୁଏ ନାହିଁ {#new-provider-card-does-not-expand-automatically}

ସ୍ୱୟଂ-ବିସ୍ତାର ପାଇଁ ଆପଣଙ୍କ ବ୍ରାଉଜରରେ JavaScript ସକ୍ରିୟ ଥିବା ଆବଶ୍ୟକ। କ୍ଲିକ୍ କଲେ କାର୍ଡ ବିସ୍ତାରିତ ନହେଲେ, ତ୍ରୁଟି ପାଇଁ ବ୍ରାଉଜର୍ କନସୋଲ୍ ଯାଞ୍ଚ କରନ୍ତୁ। ସ୍ୱୟଂ-ବିସ୍ତାର ଚାଲୁ ନହେଲେ ମଧ୍ୟ କାର୍ଡ ମୁଣ୍ଡିକାକୁ କ୍ଲିକ୍ କରି କାର୍ଡର ବିଷୟବସ୍ତୁ ସବୁବେଳେ ପହଞ୍ଚିବାଯୋଗ୍ୟ ଅଟେ।
