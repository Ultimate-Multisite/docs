# Captcha

**Sispann Bot yo. Pwoteje Revni. Zewo Traka Konfigirasyon.**

W ap jere yon rezo plizyè sit ak domèn pèsonalize? Ou konnen doulè a: Google reCAPTCHA ak hCaptcha mande pou ou anrejistre *chak grenn domèn* nan konsòl administrasyon yo. Ajoute yon nouvo sit kliyan? Mete konfigirasyon captcha a ajou. Map yon domèn pèsonalize? Mete ajou ankò. Sa pa janm fini.

**Ultimate Multisite: Captcha chanje tout bagay.**

## Cap Captcha: Pwoteksyon Self-Hosted Ki Jis Mache {#cap-captcha-self-hosted-protection-that-just-works}

Nouvo founisè **Cap Captcha** nou an se yon gwo chanjman pou rezo plizyè sit yo:

* **Zewo Konfigirasyon** — Aktive addon an epi ou pwoteje. Pa gen API keys, pa gen Google account, pa gen hCaptcha dashboard, pa gen lis blan pou domèn.
* **Mache Sou Chak Domèn Otomatikman** — Soudomèn, domèn map, domèn kliyan—yo tout pwoteje touswit.
* **Self-Hosted & Prive** — Done ou pa janm kite sèvè ou. Konplètman konfòm ak GDPR san swiv twazyèm pati.
* **Prèv Travay Envizib** — Itilizatè yo rezoud yon devinèt enfòmatik an aryè-plan. Pa gen klike sou bouch ensandi, pa gen griy imaj ki fristre moun.
* **Lejè** — Sèlman 20KB. Li p ap ralanti checkout ou.

## Pwoteksyon Kont Tès Kat WooCommerce {#woocommerce-card-testing-protection}

Atak tès kat koute machann yo plizyè milya chak ane. Fwodè yo itilize bot pou valide kat kredi yo vòlè sou checkout WooCommerce ou—ou peye frè pwosesis yo, yo jwenn nimewo kat ki valid.

**Nou pwoteje kote sa pi enpòtan:**

* **Pwoteksyon Store API** — Bloke atak bot sou `/wp-json/wc/store/v1/checkout` ki kontoune captcha fòm tradisyonèl yo
* **Konpatib ak PayPal Payments** — Mache san pwoblèm ak WooCommerce PayPal Payments
* **Estatistik An Tan Reyèl** — Wè egzakteman konbyen atak w ap bloke

## Ou Toujou Bezwen Google reCAPTCHA oswa hCaptcha? {#still-need-google-recaptcha-or-hcaptcha}

Nou gen sa ou bezwen. Sipò konplè pou:

* Google reCAPTCHA v2 (Bwat pou Tcheke)
* Google reCAPTCHA v2 (Envizib)
* Google reCAPTCHA v3 (Baze sou Nòt)
* hCaptcha (Estanda)
* hCaptcha (Envizib)

Chanje ant founisè yo nenpòt lè nan paramèt rezo ou.

## Bati pou Echèl {#built-for-scale}

* Paramèt pou tout rezo a ak ranplasman pou chak sit
* Dashboard estatistik pou siveye pwoteksyon
* Nivo sekirite ajistab (Rapid, Mwayen, Maksimòm)
* Zanmitay pou devlopè ak achitekti founisè ki ka elaji

## Enstalasyon {#installation}

1. Telechaje `ultimate-multisite-captcha` nan repètwa `/wp-content/plugins/` ou
2. Aktive plugin nan sou rezo a
3. **Se tout. Ou pwoteje.**

Cap Captcha aktive otomatikman—pa gen paramèt pou konfigire, pa gen API keys pou antre. Chak endpoint ki sipòte atravè tout rezo ou a pwoteje depi moman ou aktive li.

**Opsyonèl:** Pou itilize Google reCAPTCHA oswa hCaptcha pito, ale nan **Ultimate Multisite → Settings → Captcha** epi chwazi founisè ou prefere a.

## Kesyon Yo Poze Souvan {#frequently-asked-questions}

## Èske sa ap ralanti checkout mwen? {#will-this-slow-down-my-checkout}

Cap Captcha gen sèlman 20KB epi li lanse defi prèv travay an aryè-plan. Kliyan ou yo p ap remake okenn reta.

## Èske sa pwoteje kont atak tès kat? {#does-this-protect-against-card-testing-attacks}

Wi! Kontrèman ak captcha tradisyonèl ki sèlman pwoteje fòm HTML, entegrasyon WooCommerce nou an konekte dirèkteman nan Store API pou bloke atak bot nan nivo API a—kote pifò atak tès kat rive.

## Limitasyon To {#rate-limiting}

Ultimate Multisite: Captcha v1.5.0 prezante yon limiteur to ki kanpe nèt pou pwoteje rezo ou kont atak brute-force ak card-testing.

### Kijan Li Mache {#how-it-works}

Limiteur to a konte chak demann GET ak POST sou sifas ki pwoteje ak captcha:

* **Endpoint koneksyon WordPress** — wp-login, register, lost-password, comments
* **Checkout WooCommerce** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

Lè yon vizitè depase limit to a, plugin nan reponn ak:

* Kòd estati **HTTP 429 (Twòp Demann)**
* Header **Retry-After** ki endike kilè pou eseye ankò
* **Dòmi tarpit owaza** (1–5 segonn pa defo, ak limit di nan 15 segonn) pou ralanti atakan yo

### Konfigirasyon {#configuration}

#### Distribisyon Tarpit {#tarpit-timing}

Kontwole reta owaza ki aplike sou demann ki limite pa to:

* **`cap_rate_limit_tarpit_min`** — Dire minimòm dòmi an segonn (defo: 1)
* **`cap_rate_limit_tarpit_max`** — Dire maksimòm dòmi an segonn (defo: 5, limit di nan 15)

Paramèt sa yo disponib nan **Ultimate Multisite → Settings → Captcha**.

### Kapasite pou Elaji {#extensibility}

#### Filtè Lis Blan IP {#ip-whitelist-filter}

Egzante seri IP ou fè konfyans yo de limitasyon to:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // Whitelist your monitoring service
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### Aksyon Anvan Blokaj {#pre-block-action}

Egzekite lojik pèsonalize imedyatman anvan yo voye yon blokaj limit to:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // Log the block, notify admins, or update custom metrics
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## Deteksyon IP Client & Proxy Ou Fè Konfyans {#client-ip-detection--trusted-proxies}

Ultimate Multisite: Captcha v1.5.0 prezante deteksyon IP client ki rezistan kont fo idantite. Metòd `Captcha_Core::get_client_ip()` la kounye a se sèl sous verite pou atribisyon IP vizitè atravè:

* Kle bokit limit pousantaj
* Founisè Captcha `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* Ach IP estatistik yo

### Modèl Konfyans {#trust-model}

plugin nan aplike yon yerachi konfyans IP ki strik:

1. **REMOTE_ADDR** — Toujou fè konfyans (IP kanmarad imedya a)
2. **CF-Connecting-IP** — Respekte sèlman lè:
   - Kanmarad imedya a anndan yon seri IP Cloudflare aktyèl
   - `cap_trust_cloudflare_headers` aktive (default: OFF)
3. **X-Forwarded-For** — Respekte sèlman lè:
   - Kanmarad imedya a nan lis proxy ou fè konfyans ou te konfigire nan admin nan
   - Header la analize soti adwat pou ale agoch, pandan li sote etap proxy/Cloudflare ki fè konfyans yo

### Konfigirasyon {#configuration-1}

#### Konfyans Header Cloudflare {#cloudflare-header-trust}

Aktive konfyans pou header `CF-Connecting-IP` Cloudflare la:

* **`cap_trust_cloudflare_headers`** — Mete sou ON pou chwazi fè `CF-Connecting-IP` konfyans
  - Default: OFF (dezaktive)
  - plugin nan vini ak yon snapshot CIDR Cloudflare ki enkli
  - Seri CIDR yo rafrechi chak semèn atravè wp-cron
  - Yo itilize fallback ki enkli a si rafrechisman an echwe

#### Konfigirasyon Proxy ki Fè Konfyans {#trusted-proxy-configuration}

Defini proxy premye liy ou yo ak load-balancers ou yo:

* **`cap_trusted_proxies`** — Zòn tèks pou CIDR oswa IP senp (youn pa liy)
  - Kòmantè yo otorize (liy ki kòmanse ak `#`)
  - San paramèt sa a, yo inyore `X-Forwarded-For` menm lè limitè pousantaj la aktive
  - Egzanp:
    ```
    # Our load balancer
    192.0.2.0/24

    # Backup proxy
    198.51.100.50
    ```

#### Deteksyon Otomatik Lè Aktive Premye Fwa {#first-enable-auto-detection}

Lè ou aktive limitè pousantaj la pou premye fwa, plugin nan detekte pozisyon Cloudflare ak proxy ou gen plis chans genyen an epi li montre yon avi admin "Aplike paramèt yo detekte yo" ak yon sèl klik.

* plugin nan **pa janm ranplase** valè ou te sove yo
* Si trafik ki vin apre montre config ou a pa koresponn ak reyalite ankò (pa egzanp, Cloudflare te chanje seri CIDR yo), yon avi dezakò ki pa ka retire ap parèt ak mizajou yo rekòmande a

### Poukisa Sa Enpòtan {#why-this-matters}

Yon modèl konfyans IP ki strik fèmen yon vektè fo idantite kritik: demann dirèk sou sèvè orijin ki pote yon header `CF-Connecting-IP` fo ta deja klase nan bokit dapre IP fo a olye de kanmarad reyèl la. Sa espesyalman enpòtan pou:

* **Limit pousantaj** — Atakè yo pa ka kontoune limit yo lè yo fè IP fo
* **Estatistik** — Mezi atak ou yo reflete IP vizitè reyèl yo, pa IP fo yo
* **Verifikasyon Captcha** — Founisè yo resevwa IP vizitè ki kòrèk la pou evalyasyon risk
