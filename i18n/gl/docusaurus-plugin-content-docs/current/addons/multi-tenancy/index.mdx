# Multitenencia {#multi-tenancy}

Distribúe subsitios entre servidores e bases de datos para mellorar o rendemento e a seguridade

## Aspectos destacados da versión 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 engade a base de inquilino soberano para fluxos de traballo de illamento en produción:

- Autoinicio de sesión de inquilino sen estado con visitas SSO de curta duración e protexidas contra repetición.
- Illamento de base de datos e sistema de ficheiros por subsitio para inquilinos soberanos.
- Capacidade de multitenencia de Hostinger para aprovisionamento de inquilinos consciente do host.
- Portas de verificación de migración soberana para comprobacións de datos herdados e procesamento push asíncrono.
- Vinculación configurable do host de base de datos na mesma máquina para instalacións Bedrock, FrankenPHP e localhost.
- Melloras no arranque do inquilino, escritor de base de datos, arranque do esquema e aprovisionamento de usuarios.
- Desmantelamento soberano máis seguro que limpa as credenciais do inquilino.

Consulta as guías de usuario para o [fluxo de traballo de migración soberana](/user-guide/getting-started/sovereign-migration-workflow), o [illamento de multitenencia](/user-guide/configuration/multi-tenancy-isolation) e as [operacións de inquilino soberano](/user-guide/administration/sovereign-tenant-operations).

## Requisitos {#requirements}

- WordPress 5.3 ou superior
- PHP 7.4 ou superior
- plugin Multisite Ultimate (activo)

## Instalación {#installation}

1. Sobe os ficheiros do addon ao teu directorio `/wp-content/plugins/`
2. Activa o plugin a través do menú 'Plugins' en WordPress
3. Configura os axustes do addon na administración de rede

### Probas {#testing}

O addon inclúe un marco de probas completo:

- **Probas unitarias**: probas baseadas en PHPUnit coa suite de probas de WordPress
- **Clase base de proba**: proporciona métodos auxiliares para escenarios de proba comúns
- **Cobertura de probas**: configurada para rastrexar a cobertura do código
- **Probas Multisite**: as probas execútanse nun contorno multisite

### Estándares de código {#code-standards}

- **WordPress Coding Standards**: aplicados mediante PHPCS
- **Compatibilidade con PHP 7.4+**: admite funcionalidades modernas de PHP
- **Análise estática**: integración de PHPStan para comprobación de tipos
- **Modernización de código**: regras de Rector para actualizacións de PHP

### Sistema de compilación {#build-system}

- **Procesamento de recursos**: minificación de ficheiros CSS/JS
- **Tradución**: xeración de ficheiro POT
- **Empaquetado**: creación de arquivo para distribución
- **Modo de desenvolvemento**: recursos sen minificar para depuración

## Configuración {#configuration}

O addon admite varias opcións de configuración accesibles a través da interface de administración de rede.

### Configuración de inquilino soberano {#sovereign-tenant-configuration}

Para inquilinos soberanos, configura a base de datos do inquilino, a raíz do sistema de ficheiros, a vinculación do host da base de datos e a capacidade do host antes de enrutar tráfico ao inquilino. Executa a verificación de migración despois de cada cambio que afecte ás credenciais da base de datos, á vinculación do host, ao enrutamento ou ao aprovisionamento de usuarios.

Usa `localhost` só cando o tempo de execución do inquilino e as concesións da base de datos esperen acceso mediante socket local ou na mesma máquina. Usa `127.0.0.1` ou un nome de host de servizo cando ese sexa o host de concesión configurado polo teu provedor.

### Capacidade de Hostinger {#hostinger-capability}

A capacidade de multitenencia de Hostinger funciona co token de integración compartido de Hostinger. Confirma que o dominio está presente en hPanel, que DNS apunta á Account de Hostinger e que os axustes da base de datos do inquilino coinciden con hPanel antes da verificación.

## Hooks e filtros {#hooks-and-filters}

### Accións {#actions}

- `ultimate-multisite-addon-template_init` - Disparado despois da inicialización do addon
- `ultimate-multisite-addon-template_loaded` - Disparado despois de que se carguen todos os ficheiros do addon

### Filtros {#filters}

- `ultimate-multisite-addon-template_settings` - Modificar os axustes do addon
- `ultimate-multisite-addon-template_enabled` - Sobrescribir o estado de activación/desactivación do addon
