# Multi-Currency

Ultimate Multisite: Multi-Currency permite que a túa rede acepte pagos en múltiples moedas. Converte automaticamente os prezos usando tipos de cambio en directo ou manuais, permíteche definir prezos exactos por produto para calquera moeda e detecta a moeda preferida de cada visitante mediante xeolocalización — todo xestionado desde un único Dashboard de administración.

## Características principais

- **Conversión automática de tipos de cambio** — os prezos convértense desde a túa moeda base usando tipos obtidos do provedor que escollas
- **Substitucións de prezos por produto** — define prezos exactos específicos por moeda en calquera produto para evitar a conversión automática
- **Campo de rexistro selector de moeda** — permite que os clientes escollan a súa moeda preferida durante o checkout/rexistro
- **Múltiples provedores de tipos de cambio** — ECB, Frankfurter, Stripe ou tipos completamente manuais
- **Detección de moeda baseada en xeolocalización** — preselecciona automaticamente a moeda que coincide co país do visitante
- **Dashboard de administración** — xestiona moedas activas, tipos de cambio e axustes do provedor desde un único lugar

## Requisitos

- Ultimate Multisite 2.0.0 ou superior
- Instalación de WordPress Multisite
- PHP 7.4 ou superior

## Instalación

1. Sube os ficheiros do plugin a `/wp-content/plugins/ultimate-multisite-multi-currency`
2. Activa o plugin a través da pantalla **Plugins** en WordPress (Activar na rede)
3. Vai a **Ultimate Multisite → Multi-Currency** para abrir o Dashboard de administración
4. Engade as moedas que queres aceptar e configura o teu provedor de tipos de cambio

## Configuración

### Dashboard de administración

O Dashboard de administración de Multi-Currency (**Ultimate Multisite → Multi-Currency**) é o lugar central para:

- Engadir, activar ou desactivar moedas
- Escoller un provedor de tipos de cambio e actualizar os tipos baixo demanda
- Revisar os tipos actuais de todas as moedas activas
- Definir tipos manuais cando os provedores automáticos non sexan axeitados

### Provedores de tipos de cambio

| Provedor | Descrición |
|---|---|
| **ECB** | Banco Central Europeo — tipos diarios gratuítos para redes baseadas en EUR |
| **Frankfurter** | API espello de ECB de código aberto — gratuíta, non require clave de API |
| **Stripe** | Usa os tipos FX en directo de Stripe — require unha integración de Stripe activa |
| **Manual** | Introduces os tipos directamente; sen chamadas a API externas |

Selecciona o teu provedor en **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider**. Os tipos poden actualizarse manualmente en calquera momento ou programarse para actualizarse automaticamente.

### Substitucións de prezos por produto

Cando a conversión automática non é o suficientemente precisa, podes definir un prezo exacto para unha moeda específica en calquera produto:

1. Abre o editor de produto (**Ultimate Multisite → Products → Edit**)
2. Desprázate ata a sección **Currency Pricing**
3. Introduce o prezo exacto para cada moeda que queiras substituír
4. Garda o produto

Os produtos sen unha substitución usan o prezo convertido desde a túa moeda base.

### Campo de rexistro selector de moeda

O campo selector de moeda pode engadirse ao teu fluxo de rexistro/checkout para que os clientes escollan a súa moeda preferida no rexistro:

1. Vai a **Ultimate Multisite → Multi-Currency → Settings**
2. Activa **Show currency selector on signup**
3. Opcionalmente, define unha moeda predeterminada ou deixa que a xeolocalización determine o valor preseleccionado

A moeda seleccionada gárdase coa subscrición e úsase para toda a facturación posterior.

### Detección de moeda baseada en xeolocalización

Cando a detección por xeolocalización está activada, o addon asigna o país do visitante a unha moeda e preselecciónaa no campo selector de moeda. Para activala:

1. Vai a **Ultimate Multisite → Multi-Currency → Settings**
2. Activa **Detect currency from visitor location**
3. Asigna países a moedas na táboa **Geolocation Mapping**

Se non existe ningunha asignación para o país dun visitante, úsase a moeda base da rede como alternativa.

## Preguntas frecuentes

### Que acontece se un provedor de tipos de cambio non está dispoñible?

O addon recorre aos últimos tipos obtidos correctamente. Se nunca se obtiveron tipos, recorre aos tipos manuais que configuraches. Recoméndase configurar tipos manuais como rede de seguridade.

### Podo usar provedores diferentes para moedas diferentes?

Non — hai un único provedor activo á vez para todas as moedas. Usa **Manual** se necesitas control total sobre os tipos individuais.

### A xeolocalización require un plugin separado?

Non se require ningún plugin externo de xeolocalización. O addon usa a detección de IP a país integrada de WordPress. Para unha maior precisión, pódese integrar un servizo de xeolocalización dedicado mediante os filter hooks proporcionados.

### As substitucións por produto aplícanse antes ou despois dos impostos?

As substitucións por produto definen o prezo antes de impostos na moeda de destino. O imposto calcúlase sobre o prezo de substitución usando a túa configuración fiscal existente.
