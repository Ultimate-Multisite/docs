# ငှားရမ်းသူအများစနစ်

စွမ်းဆောင်ရည်နှင့် လုံခြုံရေးအတွက် subsite များကို ဆာဗာများနှင့် ဒေတာဘေ့စ်များအနှံ့ ဖြန့်ဝေပါ

## Version 1.2.0 ထုတ်ပြန်မှု အဓိကချက်များ {#version-120-release-highlights}

Multi-Tenancy 1.2.0 သည် ထုတ်လုပ်ရေး ပတ်ဝန်းကျင် သီးခြားထားမှု လုပ်ငန်းစဉ်များအတွက် အချုပ်အခြာပိုင် tenant အခြေခံကို ထည့်သွင်းထားသည်-

- သက်တမ်းတို၊ ပြန်သုံးတိုက်ခိုက်မှုကာကွယ်ထားသော SSO ဝင်ရောက်မှုများဖြင့် state မသိမ်းသော tenant အလိုအလျောက်ဝင်ရောက်မှု။
- အချုပ်အခြာပိုင် tenant များအတွက် subsite တစ်ခုချင်းစီအလိုက် ဒေတာဘေ့စ်နှင့် ဖိုင်စနစ် သီးခြားထားမှု။
- host ကိုသိရှိသော tenant ပြင်ဆင်ပေးမှုအတွက် Hostinger multi-tenancy စွမ်းရည်။
- legacy ဒေတာစစ်ဆေးမှုများနှင့် async push လုပ်ဆောင်မှုအတွက် အချုပ်အခြာပိုင် ပြောင်းရွှေ့မှု အတည်ပြု gate များ။
- Bedrock၊ FrankenPHP နှင့် localhost တပ်ဆင်မှုများအတွက် တူညီစက်ပေါ်ရှိ ဒေတာဘေ့စ် host ချိတ်ဆက်မှုကို ပြင်ဆင်သတ်မှတ်နိုင်ခြင်း။
- tenant bootstrap၊ ဒေတာဘေ့စ် writer၊ schema bootstrap နှင့် အသုံးပြုသူ ပြင်ဆင်ပေးမှု တိုးတက်မှုများ။
- tenant အထောက်အထားများကို ရှင်းလင်းပေးသော ပိုမိုလုံခြုံသည့် အချုပ်အခြာပိုင် ဖြုတ်ချမှု။

[အချုပ်အခြာပိုင် ပြောင်းရွှေ့မှု လုပ်ငန်းစဉ်](/user-guide/getting-started/sovereign-migration-workflow)၊ [multi-tenancy သီးခြားထားမှု](/user-guide/configuration/multi-tenancy-isolation) နှင့် [အချုပ်အခြာပိုင် tenant လုပ်ဆောင်မှုများ](/user-guide/administration/sovereign-tenant-operations) အတွက် အသုံးပြုသူလမ်းညွှန်များကို ကြည့်ပါ။

## လိုအပ်ချက်များ {#requirements}

- WordPress 5.3 သို့မဟုတ် အထက်
- PHP 7.4 သို့မဟုတ် အထက်
- Multisite Ultimate plugin (အသက်သွင်းထားပြီး)

## တပ်ဆင်ခြင်း {#installation}

1. addon ဖိုင်များကို သင့် `/wp-content/plugins/` directory သို့ အပ်လုဒ်တင်ပါ
2. WordPress ရှိ 'Plugins' မီနူးမှတစ်ဆင့် plugin ကို အသက်သွင်းပါ
3. network admin တွင် addon ဆက်တင်များကို ပြင်ဆင်သတ်မှတ်ပါ

### စမ်းသပ်ခြင်း {#testing}

ဤ addon တွင် ပြည့်စုံသော စမ်းသပ်မှု framework ပါဝင်သည်-

- **Unit Tests**: WordPress test suite ဖြင့် PHPUnit အခြေပြု test များ
- **အခြေခံ Test Class**: ပုံမှန် test အခြေအနေများအတွက် အကူအညီ method များ ပေးသည်
- **Test Coverage**: code coverage ကို ခြေရာခံရန် ပြင်ဆင်သတ်မှတ်ထားသည်
- **Multisite Testing**: test များကို multisite ပတ်ဝန်းကျင်တွင် လုပ်ဆောင်သည်

### Code စံနှုန်းများ {#code-standards}

- **WordPress Coding Standards**: PHPCS မှတစ်ဆင့် လိုက်နာစေသည်
- **PHP 7.4+ Compatibility**: ခေတ်မီ PHP လုပ်ဆောင်ချက်များကို ပံ့ပိုးထားသည်
- **Static Analysis**: type စစ်ဆေးရန် PHPStan ပေါင်းစည်းမှု
- **Code Modernization**: PHP upgrade များအတွက် Rector စည်းမျဉ်းများ

### Build စနစ် {#build-system}

- **Asset Processing**: CSS/JS ဖိုင်များအတွက် minification
- **ဘာသာပြန်ခြင်း**: POT ဖိုင် ထုတ်လုပ်ခြင်း
- **ထုပ်ပိုးခြင်း**: ဖြန့်ချိရန် archive ဖန်တီးခြင်း
- **ဖွံ့ဖြိုးရေး Mode**: debugging အတွက် မချုံ့ထားသော asset များ

## ပြင်ဆင်သတ်မှတ်ခြင်း {#configuration}

ဤ addon သည် network admin interface မှတစ်ဆင့် အသုံးပြုနိုင်သော configuration ရွေးချယ်စရာများစွာကို ပံ့ပိုးထားသည်။

### အချုပ်အခြာပိုင် tenant configuration {#sovereign-tenant-configuration}

အချုပ်အခြာပိုင် tenant များအတွက် traffic ကို tenant သို့ route မလုပ်မီ tenant ဒေတာဘေ့စ်၊ ဖိုင်စနစ် root၊ ဒေတာဘေ့စ် host ချိတ်ဆက်မှုနှင့် host စွမ်းရည်ကို ပြင်ဆင်သတ်မှတ်ပါ။ ဒေတာဘေ့စ် အထောက်အထားများ၊ host ချိတ်ဆက်မှု၊ routing သို့မဟုတ် အသုံးပြုသူ ပြင်ဆင်ပေးမှုကို သက်ရောက်သော ပြောင်းလဲမှုတိုင်းပြီးနောက် ပြောင်းရွှေ့မှု အတည်ပြုစစ်ဆေးမှုကို လုပ်ဆောင်ပါ။

tenant runtime နှင့် ဒေတာဘေ့စ် grant များက local socket သို့မဟုတ် တူညီစက် access ကို မျှော်လင့်သည့်အခါတွင်သာ `localhost` ကို သုံးပါ။ သင့် provider မှ ပြင်ဆင်ထားသော grant host ဖြစ်ပါက `127.0.0.1` သို့မဟုတ် service hostname ကို သုံးပါ။

### Hostinger စွမ်းရည် {#hostinger-capability}

Hostinger multi-tenancy စွမ်းရည်သည် မျှဝေထားသော Hostinger integration token နှင့် အလုပ်လုပ်သည်။ အတည်ပြုခြင်းမပြုမီ domain သည် hPanel တွင် ရှိကြောင်း၊ DNS သည် Hostinger account သို့ ညွှန်ပြကြောင်းနှင့် tenant ဒေတာဘေ့စ် ဆက်တင်များသည် hPanel နှင့် ကိုက်ညီကြောင်း အတည်ပြုပါ။

## Hook များနှင့် Filter များ {#hooks-and-filters}

### Action များ {#actions}

- `ultimate-multisite-addon-template_init` - addon စတင်ပြီးနောက် ဖြစ်ပေါ်သည်
- `ultimate-multisite-addon-template_loaded` - addon ဖိုင်အားလုံး တင်ပြီးနောက် ဖြစ်ပေါ်သည်

### Filter များ {#filters}

- `ultimate-multisite-addon-template_settings` - addon ဆက်တင်များကို ပြင်ဆင်ပါ
- `ultimate-multisite-addon-template_enabled` - addon ဖွင့်/ပိတ် အခြေအနေကို override လုပ်ပါ
