# Multi-Network {#multi-network}

Ultimate Multisite: Multinetwork addon သည် သင့်ရဲ့ Ultimate Multisite platform ကို စွမ်းအားမြင့် multinetwork လုပ်ဆောင်ချက်များဖြင့် တိုးချဲ့ပေးထားပါတယ်။ ဒီ Addon ကို အသုံးပြုခြင်းအားဖြင့် သင့်ရဲ့ ဝယ်ယူသူများဟာ တစ်ဦးချင်း ဝက်ဘ်ဆိုက်များသာမက၊ တစ်ခုလုံးသော WordPress multisite network များကို ဝယ်ယူပြီး စီမံခန့်ခွဲနိုင်မှာ ဖြစ်ပါတယ်။

## အဓိက လုပ်ဆောင်ချက်များ (Key Features) {#key-features}

* **Network Provisioning**: သင့်ဝယ်ယူသူများအတွက် စုံလင်ပြည့်စုံတဲ့ WordPress multisite network များကို အလိုအလျောက် ဖန်တီးပေးခြင်းနှင့် စနစ်တကျ စီစဉ်ပေးခြင်း။
* **Network Templates**: ဝယ်ယူသူများအနေနဲ့ စာရင်းသွင်းခြင်းမပြုမီ ကြိုတင်တည်ဆောက်ထားတဲ့ network template များကို ကြည့်ရှုလေ့လာ၊ ကြိုတင်ကြည့်ရှုပြီး ရွေးချယ်နိုင်စေခြင်း။
* **Network Management**: multinetwork ပတ်ဝန်းကျင်များကို အပြည့်အဝ စီမံခန့်ခွဲနိုင်ခြင်း။
* **Isolated Networks**: ဝယ်ယူသူ တစ်ဦးချင်းစီရဲ့ network များဟာ သူတို့ရဲ့ database tables နဲ့ configuration တွေနဲ့ အုပ်စုလိုက် ခွဲခြားထားတဲ့အတွက် သီးခြားစီ လည်ပတ်နိုင်ခြင်း။
* **Cross-Network SSO**: ဝယ်ယူသူများအနေနဲ့ magic-link authentication ဖြင့် သူတို့ရဲ့ sub-network များဆီကို အဆင်ပြေစွာ ဝင်ရောက်နိုင်စေခြင်း။
* **Scalable Architecture**: အရင်းအမြစ်များကို အကောင်းဆုံး ခွဲဝေပေးခြင်းဖြင့် network များစွာကို ထိရောက်စွာ ကိုင်တွယ်နိုင်ခြင်း။
* **Integration**: Ultimate Multisite ရဲ့ subscription နဲ့ billing စနစ်တွေနဲ့ ချောမွေ့စွာ ပေါင်းစပ်အသုံးပြုနိုင်ခြင်း။

## အသုံးပြုနိုင်သည့် နေရာများ (Use Cases) {#use-cases}

* **White Label Hosting**: ဝယ်ယူသူများအတွက် သူတို့ကိုယ်ပိုင် brand ပါတဲ့ multisite network များကို ပံ့ပိုးပေးခြင်း။
* **Agency Solutions**: Agency များအနေနဲ့ client network များစွာကို platform တစ်ခုတည်းကနေ စီမံခန့်ခွဲနိုင်စေခြင်း။
* **Enterprise Deployments**: ရှုပ်ထွေးတဲ့ multi-network လိုအပ်ချက်တွေရှိတဲ့ ကြီးမားတဲ့ အဖွဲ့အစည်းတွေကို ပံ့ပိုးပေးခြင်း။
* **Education Platforms**: ကျောင်းအသစ်များ သို့မဟုတ် ဌာနအသစ်များအတွက် သီးခြား network များကို ဖန်တီးပေးခြင်း။

## လိုအပ်ချက်များ (Requirements) {#requirements}

* Ultimate Multisite 2.0.0 သို့မဟုတ် အထက်
* WordPress Multisite တပ်ဆင်ထားမှု
* PHP 7.4 သို့မဟုတ် အထက်
* MySQL 5.6 သို့မဟုတ် အထက်

## တပ်ဆင်ခြင်း (Installation) {#installation}

1. plugin file များကို `/wp-content/plugins/ultimate-multisite-multinetwork` directory ထဲသို့ တင်ပါ။
2. WordPress ရဲ့ 'Plugins' screen မှတစ်ဆင့် plugin ကို activate လုပ်ပါ။
3. Ultimate Multisite → Settings မှတစ်ဆင့် multinetwork setting များကို စီစဉ်ပေးပါ။

## Network Templates {#network-templates}

Version 1.3.0 သည် network အသစ်ဝယ်ယူသူများအတွက် template ရှာဖွေမှုနှင့် ရွေးချယ်မှု လုပ်ငန်းစဉ် (flow) ကို ထပ်ပေါင်းထည့်ထားပါတယ်။ ဝယ်ယူသူ အသစ် network တစ်ခုချင်းစီဟာ ကြိုတင်ပြင်ဆင်ထားတဲ့ structure၊ design၊ content library သို့မဟုတ် vertical-specific setup တစ်ခုကနေ စတင်စေချင်တဲ့အခါ network templates များကို အသုံးပြုသင့်ပါတယ်။

### Template previewer {#template-previewer}

Network template previewer သည် ဝယ်ယူသူများအား checkout မလုပ်မီ ရရှိနိုင်သော template များကို ကြည့်ရှုလေ့လာခွင့် ပေးပါတယ်။ template တစ်ခုချင်းစီကို live preview panel တွင် ဖွင့်ကြည့်နိုင်တာကြောင့် ဝယ်ယူသူများသည် network တစ်ခုကို ဝယ်ယူခြင်းမပြုမီ layout များနှင့် content များကို နှိုင်းယှဉ်ကြည့်ရှုနိုင်ပါတယ်။

ဝယ်ယူသူများအား အောက်ပါအချက်များအတွက် အကူအညီပေးရန် previewer ကို အသုံးပြုနိုင်သည်-

* ရရှိနိုင်သော network template တစ်ခုချင်းစီ၏ ပုံစံနှင့် structure ကို ပြန်လည်စစ်ဆေးခြင်း။
* ဝယ်ယူမှု လုပ်ငန်းစဉ်မှ မခွာဘဲ template များစွာကို နှိုင်းယှဉ်ကြည့်ရှုခြင်း။
* စာရင်းသွင်းခြင်းမပြုမီ သူတို့ရဲ့ အသုံးပြုမှုအခြေအနေနဲ့ အသင့်တော်ဆုံး template ကို ရွေးချယ်ခြင်း။

template အမည်များနှင့် ဖော်ပြချက်များကို ဝယ်ယူသူများ အလွယ်တကူ နားလည်နိုင်အောင် ရေးသားထားပါ။ agency network များ၊ ကျောင်း network များ၊ franchise network များ သို့မဟုတ် white-label hosting bundle များကဲ့သို့သော အသုံးပြုမှုအခြေအနေကို ဝယ်ယူသူများ နားလည်နိုင်မှ previewer သည် အကျိုးအရှိဆုံး ဖြစ်ပါသည်။

### Template picker at checkout {#template-picker-at-checkout}

Template picker ကို signup checkout flow မှာ field တစ်ခုအနေနဲ့ ထည့်သွင်းနိုင်ပါတယ်။ ၎င်းကို စနစ်တကျ စီစဉ်ထားပါက ဝယ်ယူသူများသည် checkout လုပ်နေစဉ်မှာ သူတို့လိုချင်တဲ့ network template ကို ရွေးချယ်နိုင်ပြီး၊ ရွေးချယ်ထားတဲ့ template ကို သူတို့ရဲ့ network အသစ်ကို provision လုပ်တဲ့အခါ အသုံးပြုပေးမှာ ဖြစ်ပါတယ်။

checkout flow မှာ picker ကို အသုံးပြုရန်-

1. template အဖြစ် ပေးလိုတဲ့ network များကို ပြင်ဆင်ထားပါ။
2. သက်ဆိုင်ရာ signup သို့မဟုတ် checkout form မှာ Multinetwork template selection field ကို enable လုပ်ပါ။
3. ဘယ် template တွေက ဒီ checkout flow အတွက် ရရှိနိုင်သင့်သလဲဆိုတာ ရွေးချယ်ပါ။
4. checkout flow ကို publish လုပ်ပြီး၊ ရွေးချယ်ထားတဲ့ template ဟာ network အသစ်မှာ အသုံးပြုခံရရဲ့လားဆိုတာ စစ်ဆေးဖို့ signup တစ်ခုကို စမ်းသပ်ပါ။

checkout flow တစ်ခုက network product တစ်ခုတည်းကိုသာ ပေးတယ်ဆိုရင်တောင် picker ဟာ လမ်းညွှန်ပေးတဲ့ onboarding အဆင့်အနေနဲ့ အသုံးဝင်နေပါသေးတယ်။ multi-plan funnels များအတွက်၊ ရွေးချယ်ထားတဲ့ product သို့မဟုတ် plan နဲ့ ကိုက်ညီတဲ့ template များကိုသာ ပြသသင့်ပါတယ်။

## Cross-Network Magic Link SSO {#cross-network-magic-link-sso}

Cross-network magic link SSO သည် ဝယ်ယူသူများအား သူတို့ရဲ့ primary account area နဲ့ sub-network များကြားမှာ ကိုယ်တိုင် login ပြန်လုပ်စရာမလိုဘဲ ရွေ့လျားနိုင်စေပါတယ်။ ဝယ်ယူသူတစ်ဦးက cross-network access link ကို ဝင်ရောက်လိုက်တဲ့အခါ Ultimate Multisite က အချိန်ကန့်သတ်ထားတဲ့ magic link တစ်ခုကို ဖန်တီးပေးပြီး၊ ဝယ်ယူသူကို ပစ်မှတ် network ထဲကို အထောက်အထားစစ်ဆေးပေးပါတယ်။

ဒါဟာ network တစ်ခုထက်ပိုတဲ့ network တွေကို စီမံခန့်ခွဲရတဲ့ ဝယ်ယူသူတွေ ဒါမှမဟုတ် ဝယ်ယူပြီးပြီးချင်း အသစ် clone လုပ်ထားတဲ့ template network တွေကို ချက်ချင်းဝင်ရောက်ဖို့ လိုအပ်တဲ့ ဝယ်ယူသူတွေအတွက် အတွေ့အကြုံကို ပိုမိုကောင်းမွန်စေပါတယ်။

Admin စစ်ဆေးရမည့်အချက်များ (Admin checklist):

* WordPress multisite တပ်ဆင်မှုတစ်ခုလုံးမှာ Multinetwork addon က active ဖြစ်မဖြစ် စစ်ဆေးပါ။
* ဝယ်ယူသူတွေကို cross-network link တွေ ပေးအပ်တဲ့အခါ သူတို့မှာ မှန်ကန်တဲ့ network access ရှိမရှိ သေချာပါစေ။
* magic link တွေဟာ ရည်ရွယ်တဲ့ destination network ကို ညွှန်ပြနိုင်ဖို့ network domain များနှင့် mapped domain များကို မှန်ကန်စွာ စီစဉ်ထားပါ။
* network၊ domain သို့မဟုတ် checkout setting များကို ပြောင်းလဲပြီးနောက် customer dashboard နဲ့ post-checkout access button များကို စမ်းသပ်ပါ။

Magic link များကို login link များအဖြစ် သဘောထားသင့်ပါတယ်။ ၎င်းတို့ကို ချောမွေ့တဲ့ customer access အတွက် ဒီဇိုင်းထုတ်ထားတာဖြစ်ပြီး၊ ရည်ရွယ်တဲ့ session အပြင်ဘက်မှာ forward လုပ်တာ ဒါမှမဟုတ် ပြန်အသုံးပြုတာမျိုး မလုပ်သင့်ပါဘူး။

## မေးလေ့ရှိသော မေးခွန်းများ (Frequently Asked Questions) {#frequently-asked-questions}

### multinetwork ဆိုတာ ဘာလဲ။ {#what-is-multinetwork}

Multinetwork ဆိုတာ WordPress ရဲ့ လုပ်ဆောင်ချက်တစ်ခုဖြစ်ပြီး၊ single WordPress တပ်ဆင်မှုတစ်ခုပေါ်မှာ independent multisite network များစွာကို လည်ပတ်နိုင်စေပါတယ်။

### ဒါက ပုံမှန် multisite နဲ့ ဘယ်လိုကွာလဲ။ {#how-is-this-different-from-regular-multisite}

ပုံမှန် multisite ဟာ network တစ်ခုအတွင်းမှာ ဝက်ဘ်ဆိုက်များစွာကို ခွင့်ပြုပါတယ်။ Multinetwork ကတော့ network များစွာကို ခွင့်ပြုပြီး၊ အဲဒီ network တစ်ခုချင်းစီမှာ သူတို့ရဲ့ ဝက်ဘ်ဆိုက်တွေနဲ့ အုပ်စုဖွဲ့ထားတာတွေဟာ single WordPress တပ်ဆင်မှုတစ်ခုထဲမှာ ရှိနေတာ ဖြစ်ပါတယ်။

### ဒါက server မှာ အထူးဖွဲ့စည်းမှု လိုအပ်လား။ {#does-this-require-special-server-configuration}

ဟုတ်ပါတယ်၊ multinetwork အတွက် သီးသန့် server setup နဲ့ configuration လိုအပ်ပါတယ်။ အသေးစိတ် setup ညွှန်ကြားချက်များအတွက် documentation ကို ကြည့်ရှုပေးပါ။
