# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency ကို အသုံးပြုခြင်းဖြင့် သင့် network မှာ ငွေကြေးအမျိုးမျိုးနဲ့ ငွေပေးချေမှုတွေကို လက်ခံနိုင်မှာ ဖြစ်ပါတယ်။ ဒါက ဈေးနှုန်းတွေကို လက်ရှိ (live) ဒါမှမဟုတ် ကိုယ်တိုင်သတ်မှတ်ထားတဲ့ ငွေလဲနှုန်းတွေနဲ့ အလိုအလျောက် ပြောင်းလဲပေးနိုင်ပါတယ်။ ထုတ်ကုန်တစ်ခုချင်းစီအတွက် ဘယ်ငွေကြေးမှာပဲဖြစ်ဖြစ် ဈေးနှုန်းကို အတိအကျ သတ်မှတ်နိုင်ပြီး၊ သုံးစွဲသူရဲ့ တည်နေရာ (geolocation) အပေါ် အခြေခံပြီး သူနှစ်သက်တဲ့ ငွေကြေးကိုပါ သိရှိပေးနိုင်ပါတယ်။ ဒီအရာအားလုံးကို စီမံခန့်ခွဲမှုဘုတ်ဝိုင်း (admin dashboard) တစ်ခုတည်းကနေ လုပ်ဆောင်နိုင်ပါတယ်။

## အဓိက လုပ်ဆောင်ချက်များ (Key Features) {#key-features}

- **အလိုအလျောက် ငွေလဲနှုန်း ပြောင်းလဲပေးခြင်း** — သတ်မှတ်ထားတဲ့ ပံ့ပိုးပေးသူ (provider) ဆီကနေ ရယူတဲ့ နှုန်းထားတွေနဲ့ အခြေခံငွေကြေးကနေ ဈေးနှုန်းတွေကို ပြောင်းလဲပေးပါတယ်။
- **ထုတ်ကုန်တစ်ခုချင်းစီအတွက် ဈေးနှုန်းကို ကိုယ်တိုင်သတ်မှတ်ခြင်း** — အလိုအလျောက် ပြောင်းလဲမှုတွေကို ကျော်လွန်ပြီး ထုတ်ကုန်တစ်ခုမှာ ငွေကြေးအလိုက် ဈေးနှုန်းကို အတိအကျ သတ်မှတ်နိုင်ပါတယ်။
- **ငွေကြေးရွေးချယ်စရာ စာရင်းသွင်းကွက်** — ဝယ်ယူမှု (checkout) လုပ်တဲ့အခါ ဒါမှမဟုတ် စာရင်းသွင်းတဲ့အခါမှာ ဖောက်သည်တွေက သူတို့နှစ်သက်တဲ့ ငွေကြေးကို ရွေးချယ်ခွင့်ပေးပါတယ်။
- **ငွေလဲနှုန်း ပံ့ပိုးပေးသူများစွာ** — ECB, Frankfurter, Stripe တို့ကို အသုံးပြုနိုင်သလို၊ ကိုယ်တိုင်လက်နဲ့ နှုန်းထားတွေ သတ်မှတ်နိုင်ပါတယ်။
- **တည်နေရာအပေါ် အခြေခံပြီး ငွေကြေးကို သိရှိခြင်း** — သုံးစွဲသူရဲ့ နိုင်ငံကို အလိုအလျောက် သိရှိပြီး၊ အဲဒီနိုင်ငံနဲ့ ကိုက်ညီတဲ့ ငွေကြေးကို ကြိုတင်ရွေးပေးပါတယ်။
- **အုပ်ချုပ်မှုဘုတ်ဝိုင်း (Admin dashboard)** — အသုံးပြုနေတဲ့ ငွေကြေးတွေ၊ ငွေလဲနှုန်းတွေနဲ့ ပံ့ပိုးပေးသူရဲ့ စနစ်တွေကို နေရာတစ်ခုတည်းကနေ စီမံခန့်ခွဲနိုင်ပါတယ်။

## လိုအပ်ချက်များ (Requirements) {#requirements}

- Ultimate Multisite 2.0.0 သို့မဟုတ် အထက်
- WordPress Multisite တပ်ဆင်ထားရမည်
- PHP 7.4 သို့မဟုတ် အထက်

## တပ်ဆင်ခြင်း (Installation) {#installation}

1. plugin ဖိုင်တွေကို `/wp-content/plugins/ultimate-multisite-multi-currency` မှာ တင်ပေးပါ။
2. WordPress ရဲ့ **Plugins** စခရင်ကနေ plugin ကို အသက်သွင်းပေးပါ (Network Activate လုပ်ရပါမယ်)။
3. အုပ်ချုပ်မှုဘုတ်ဝိုင်းကို ဖွင့်ဖို့ **Ultimate Multisite → Multi-Currency** ကို သွားပါ။
4. လက်ခံလိုတဲ့ ငွေကြေးတွေကို ထည့်သွင်းပြီး ငွေလဲနှုန်း ပံ့ပိုးပေးသူကို စနစ်တကျ ဖွဲ့စည်းပေးပါ။

## စနစ်ဖွဲ့စည်းခြင်း (Configuration) {#configuration}

### အုပ်ချုပ်မှုဘုတ်ဝိုင်း (Admin Dashboard) {#admin-dashboard}

Multi-Currency အုပ်ချုပ်မှုဘုတ်ဝိုင်း (**Ultimate Multisite → Multi-Currency**) ဟာ အောက်ပါအရာတွေကို လုပ်ဆောင်ဖို့ အဓိကနေရာ ဖြစ်ပါတယ်-

- ငွေကြေးအသစ်တွေ ထည့်ခြင်း၊ အသက်သွင်းခြင်း ဒါမှမဟုတ် ပိတ်ခြင်း
- ငွေလဲနှုန်း ပံ့ပိုးပေးသူကို ရွေးချယ်ခြင်းနဲ့ နှုန်းထားတွေကို လိုအပ်တဲ့အချိန်မှာ ပြန်လည်ရယူခြင်း
- အသုံးပြုနေတဲ့ ငွေကြေးအားလုံးရဲ့ လက်ရှိနှုန်းထားတွေကို ပြန်လည်စစ်ဆေးခြင်း
- အလိုအလျောက် ပံ့ပိုးပေးသူတွေ မသင့်တော်တဲ့အခါ ကိုယ်တိုင်နှုန်းထားတွေ သတ်မှတ်ခြင်း

### ငွေလဲနှုန်း ပံ့ပိုးပေးသူများ (Exchange Rate Providers) {#exchange-rate-providers}

| ပံ့ပိုးပေးသူ (Provider) | ဖော်ပြချက် (Description) |
|---|---|
| **ECB** | ဥရောပ ဗဟိုဘဏ် — EUR အခြေခံ network များအတွက် အခမဲ့ နေ့စဉ်နှုန်းထားများ |
| **Frankfurter** | Open-source ECB mirror API — အခမဲ့၊ API key မလိုအပ်ပါ |
| **Stripe** | Stripe ရဲ့ လက်ရှိ FX နှုန်းထားတွေကို အသုံးပြုသည် — အသက်ဝင်တဲ့ Stripe integration လိုအပ်သည် |
| **Manual** | နှုန်းထားတွေကို သင်ကိုယ်တိုင် တိုက်ရိုက်ထည့်သွင်းရသည်; ပြင်ပ API ခေါ်ဆိုမှု မရှိပါ |

သင့်ရဲ့ ပံ့ပိုးပေးသူကို **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** အောက်မှာ ရွေးချယ်ပါ။ နှုန်းထားတွေကို ဘယ်အချိန်မဆို ကိုယ်တိုင်ပြန်လည်ရယူနိုင်သလို၊ အလိုအလျောက် အပ်ဒိတ်လုပ်ဖို့လည်း စီစဉ်ပေးနိုင်ပါတယ်။

### ထုတ်ကုန်တစ်ခုချင်းစီအတွက် ဈေးနှုန်းကို ကိုယ်တိုင်သတ်မှတ်ခြင်း (Per-Product Pricing Overrides) {#per-product-pricing-overrides}

အလိုအလျောက် ပြောင်းလဲမှုက လုံလောက်တဲ့ အတိအကျမှု မရှိဘူးဆိုရင်၊ ထုတ်ကုန်တစ်ခုမှာ ငွေကြေးတစ်ခုချင်းစီအတွက် ဈေးနှုန်းကို အတိအကျ သတ်မှတ်ပေးနိုင်ပါတယ်-

1. ထုတ်ကုန်ကို ဖွင့်တဲ့ editor မှာ ဝင်ပါ (**Ultimate Multisite → Products → Edit**)
2. **Currency Pricing** အပိုင်းကို အောက်ကို ဆင်းပါ။
3. အစားထိုးလိုတဲ့ ငွေကြေးတစ်ခုချင်းစီအတွက် ဈေးနှုန်းကို အတိအကျ ထည့်သွင်းပါ။
4. ထုတ်ကုန်ကို သိမ်းဆည်းပါ။

အကယ်၍ ဈေးနှုန်းကို ကိုယ်တိုင်သတ်မှတ်ခြင်း မရှိတဲ့ ထုတ်ကုန်တွေဟာ အခြေခံငွေကြေးကနေ ပြောင်းလဲထားတဲ့ ဈေးနှုန်းကို အသုံးပြုပါလိမ့်မယ်။

### ငွေကြေးရွေးချယ်စရာ စာရင်းသွင်းကွက် (Currency Selector Signup Field) {#currency-selector-signup-field}

ဒီငွေကြေးရွေးချယ်စရာ ကွက်ကို သင့်ရဲ့ စာရင်းသွင်း/ဝယ်ယူမှု လုပ်ငန်းစဉ်မှာ ထည့်သွင်းနိုင်ပါတယ်။ ဒါမှ ဖောက်သည်တွေက စာရင်းသွင်းတဲ့အချိန်မှာ သူတို့နှစ်သက်တဲ့ ငွေကြေးကို ရွေးချယ်နိုင်မှာ ဖြစ်ပါတယ်-

1. **Ultimate Multisite → Multi-Currency → Settings** ကို သွားပါ။
2. **Show currency selector on signup** ကို အသက်သွင်းပါ။
3. ရွေးချယ်နိုင်ရင် default ငွေကြေးတစ်ခု သတ်မှတ်နိုင်ပါတယ်၊ ဒါမှမဟုတ် တည်နေရာအပေါ် အခြေခံပြီး ကြိုတင်ရွေးပေးခိုင်းနိုင်ပါတယ်။

ရွေးချယ်ထားတဲ့ ငွေကြေးကို စာရင်းသွင်းမှုနဲ့အတူ သိမ်းဆည်းထားပြီး နောက်ဆက်တွဲ ဘေလ်ပေးမှုအားလုံးမှာ အသုံးပြုမှာ ဖြစ်ပါတယ်။

### တည်နေရာအပေါ် အခြေခံပြီး ငွေကြေးကို သိရှိခြင်း (Geolocation-Based Currency Detection) {#geolocation-based-currency-detection}

တည်နေရာသိရှိမှု (geolocation detection) ကို အသက်သွင်းထားတဲ့အခါ၊ ဒီ addon က သုံးစွဲသူရဲ့ နိုင်ငံကို ငွေကြေးတစ်ခုနဲ့ ချိတ်ဆက်ပြီး ငွေကြေးရွေးချယ်စရာ ကွက်မှာ ကြိုတင်ရွေးပေးပါတယ်။ အသက်သွင်းဖို့အတွက်-

1. **Ultimate Multisite → Multi-Currency → Settings** ကို သွားပါ။
2. **Detect currency from visitor location** ကို အသက်သွင်းပါ။
3. **Geolocation Mapping** စားပွဲမှာ နိုင်ငံတွေကို ငွေကြေးတွေနဲ့ ချိတ်ဆက်ပေးပါ။

သုံးစွဲသူရဲ့ နိုင်ငံအတွက် ချိတ်ဆက်မှု မရှိဘူးဆိုရင်၊ network ရဲ့ အခြေခံငွေကြေးကို အစားထိုးအဖြစ် အသုံးပြုမှာ ဖြစ်ပါတယ်။

## မကြာခဏ မေးလေ့ရှိသော မေးခွန်းများ (Frequently Asked Questions) {#frequently-asked-questions}

### ငွေလဲနှုန်း ပံ့ပိုးပေးသူ မရရှိနိုင်ရင် ဘာဖြစ်မလဲ။ {#what-happens-if-an-exchange-rate-provider-is-unavailable}

ဒီ addon က နောက်ဆုံး အောင်မြင်စွာ ရယူခဲ့တဲ့ နှုန်းထားတွေကို ပြန်သုံးပါလိမ့်မယ်။ ဘယ်တုန်းကမှ နှုန်းထား မရယူခဲ့ရဘူးဆိုရင်၊ သင်ကိုယ်တိုင် စနစ်ဖွဲ့စည်းထားတဲ့ manual နှုန်းထားတွေကို ပြန်သုံးပါလိမ့်မယ်။ ဘေးအန္တရာယ် ကင်းစေဖို့ manual နှုန်းထားတွေကို စနစ်ဖွဲ့စည်းထားဖို့ အကြံပြုလိုပါတယ်။

### ငွေကြေးအလိုက် ပံ့ပိုးပေးသူ မတူတာကို သုံးလို့ရလား။ {#can-i-use-different-providers-for-different-currencies}

မရပါဘူး — ငွေကြေးအားလုံးအတွက် ပံ့ပိုးပေးသူ တစ်ခုပဲ အချိန်မရွေး အသက်ဝင်နေမှာပါ။ တစ်ခုချင်းစီရဲ့ နှုန်းထားတွေအပေါ် အပြည့်အဝ ထိန်းချုပ်ချင်တယ်ဆိုရင် **Manual** ကို အသုံးပြုပါ။

### တည်နေရာသိရှိမှုအတွက် သီးသန့် plugin လိုအပ်ပါသလား။ {#does-geolocation-require-a-separate-plugin}

ပြင်ပတည်နေရာသိရှိမှု plugin ဘာတစ်ခုမှ မလိုအပ်ပါဘူး။ ဒီ addon က WordPress ရဲ့ built-in IP-to-country detection ကို အသုံးပြုပါတယ်။ ပိုမိုတိကျတဲ့ အဖြေရဖို့အတွက်၊ ပံ့ပိုးပေးထားတဲ့ filter hooks တွေကနေ သီးသန့် geolocation service တစ်ခုကို ပေါင်းစပ်နိုင်ပါတယ်။

### ထုတ်ကုန်တစ်ခုချင်းစီရဲ့ ဈေးနှုန်းကို အခွန်မဆောင်ခင်လား၊ အခွန်ဆောင်ပြီးမှလား သတ်မှတ်မှာလဲ။ {#are-per-product-overrides-applied-before-or-after-tax}

ထုတ်ကုန်တစ်ခုချင်းစီရဲ့ ဈေးနှုန်းကို ပစ်မှတ်ငွေကြေးမှာ အခွန်မဆောင်ခင် ဈေးနှုန်းကို သတ်မှတ်ပေးပါတယ်။ အခွန်ကိုတော့ သင့်ရဲ့ လက်ရှိ အခွန်ဖွဲ့စည်းမှုအတိုင်း အပိုထပ်ရတဲ့ ဈေးနှုန်းအပေါ်မှာ တွက်ချက်ပေးပါလိမ့်မယ်။
