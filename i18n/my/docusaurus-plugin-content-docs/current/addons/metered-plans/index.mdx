# အသုံးပြုမှုအပေါ် အခြေခံ စီမံခန့်ခွဲမှု အစီအစဉ်များ (Metered Plans) {#metered-plans}

Ultimate Multisite: Metered Plans addon သည် သင့်ဖောက်သည်များကို ဘောင်ချာထုတ်သည့် နည်းလမ်းကို ပြောင်းလဲပေးပြီး၊ ပိုမိုပြောင်းလွယ်ပြင်လွယ်ရှိသော အသုံးပြုမှုအပေါ် အခြေခံ ငွေတောင်းခံမှု စွမ်းရည်များကို မိတ်ဆက်ပေးပါသည်။ အသုံးပြုမှု ပမာဏကို ကျော်လွန်ပါက အသုံးပြုသူများကို ပိတ်ဆို့သည့် တင်းကျပ်သော အကန့်အသတ်များအစား၊ အသုံးပြုမှု ပိုလျှံမှုများအတွက် ဖောက်သည်များကို အလိုအလျောက် ဘောင်ချာထုတ်ပေးသည့် စနစ်ကို အသုံးပြုနိုင်ပါသည်။

## အဓိက အင်္ဂါရပ်များ (Key Features) {#key-features}

* **အသုံးပြုမှုအပေါ် အခြေခံ ငွေတောင်းခံခြင်း (Metered Billing)**: သတ်မှတ်ထားသော အဆင့်များအပေါ် မူတည်မဘဲ၊ အသုံးပြုမှု အမှန်အပေါ် အခြေခံ၍ ဖောက်သည်များကို ငွေတောင်းခံခြင်း။
* **ပိုလျှံမှု စီမံခန့်ခွဲမှု (Overage Management)**: အစီအစဉ် ကန့်သတ်ချက်များကို ကျော်လွန်သည့် အသုံးပြုမှုအတွက် အလိုအလျောက် ဘောင်ချာထုတ်ပေးခြင်း။
* **အသုံးပြုမှု စောင့်ကြည့်ခြင်း (Usage Tracking)**: ဖောက်သည်များ၏ အရင်းအမြစ် အသုံးပြုမှုကို အချိန်နှင့်တပြေးညီ စောင့်ကြည့်နိုင်ခြင်း။
* **ပြောင်းလွယ်ပြင်လွယ်ရှိသော နှုန်းထားများ (Flexible Pricing)**: မတူညီသော အရင်းအမြစ်များ (သိုလှောင်မှု၊ bandwidth၊ ဝက်ဘ်ဆိုဒ်များ စသည်) အတွက် စိတ်ကြိုက်နှုန်းထားများ သတ်မှတ်နိုင်ခြင်း။
* **AI Token ဖြင့် ငွေတောင်းခံခြင်း (AI Token Billing)**: ဖောက်သည်များ၏ subsite များတစ်လျှောက် AI အသုံးပြုမှုအတွက် input နှင့် output token များကို စောင့်ကြည့်ပြီး ဘောင်ချာထုတ်ပေးနိုင်ခြင်း။
* **အလိုအလျောက် ဘောင်ချာထုတ်ခြင်း (Automatic Invoicing)**: အသုံးပြုမှု အကန့်အသတ်များကို ကျော်လွန်သည့်အခါ ဘောင်ချာများကို အလိုအလျောက် ထုတ်ပေးခြင်း။
* **အသုံးပြုမှု အစီရင်ခံစာများ (Usage Reports)**: ဖောက်သည်များနှင့် စီမံခန့်ခွဲသူများအတွက် အသုံးပြုမှု အသေးစိတ် ခွဲခြမ်းစိတ်ဖြာမှုနှင့် အစီရင်ခံစာများ။

## မည်သူများအတွက် အသင့်တော်ဆုံးလဲ (Perfect For) {#perfect-for}

* **Hosting ဝန်ဆောင်မှုပေးသူများ (Hosting Providers)**: ဖောက်သည်များ၏ လိုအပ်ချက်နှင့်အတူ ကြီးထွားလာနိုင်သော အစီအစဉ်များကို ပေးအပ်လိုသူများ။
* **SaaS Platform များ (SaaS Platforms)**: အမှန်တကယ် အရင်းအမြစ် အသုံးပြုမှုအပေါ် အခြေခံ၍ ငွေတောင်းခံလိုသူများ။
* **Agency Network များ (Agency Networks)**: Client ပရောဂျက်များအတွက် ပြောင်းလွယ်ပြင်လွယ်ရှိသော နှုန်းထားပုံစံများ လိုအပ်သူများ။
* **Reseller Hosting များ (Reseller Hosting)**: အသုံးပြုမှုအပေါ် အခြေခံသည့် နှုန်းထားများကို ပေးနိုင်ရန် မိတ်ဖက်များကို စွမ်းဆောင်ပေးလိုသူများ။

## မည်သို့ အလုပ်လုပ်သလဲ (How It Works) {#how-it-works}

1. အသုံးပြုနိုင်သော အရင်းအမြစ်များကို စီမံခန့်ခွဲပါ (ဥပမာ- disk space၊ bandwidth၊ ဝက်ဘ်ဆိုဒ်အရေအတွက် စသည်)။
2. အခြေခံ အစီအစဉ် ကန့်သတ်ချက်များနှင့် ပိုလျှံမှု နှုန်းထားများကို သတ်မှတ်ပါ။
3. စနစ်သည် သင့် network တစ်ခုလုံးရှိ အသုံးပြုမှုများကို အလိုအလျောက် စောင့်ကြည့်ပေးပါသည်။
4. ဖောက်သည်များက ကန့်သတ်ချက်များကို ကျော်လွန်ပါက၊ ပိုလျှံမှု အခကြေးငွေများကို တွက်ချက်ပေးပါသည်။
5. ဘောင်ချာထုတ်သည့် စက်ဝန်းအပေါ် အခြေခံ၍ ဘောင်ချာများကို အလိုအလျောက် ထုတ်ပေးပါသည်။
6. ဖောက်သည်များသည် ၎င်းတို့၏ အသုံးပြုမှုနှင့် လာမည့် အခကြေးငွေများကို အချိန်နှင့်တပြေးညီ ကြည့်ရှုနိုင်ပါသည်။

## AI Token ဖြင့် ငွေတောင်းခံခြင်း (AI Token Billing) {#ai-token-billing}

Version 1.1.0 တွင် WordPress AI Client SDK မှတစ်ဆင့် AI အင်္ဂါရပ်များကို ပေးအပ်သည့် network များအတွက် AI token ဖြင့် ငွေတောင်းခံခြင်းကို ထည့်သွင်းထားပါသည်။ Metered Plans သည် ပြီးစီးသွားသော AI ထုတ်လုပ်မှု တောင်းဆိုမှုများကို စောင့်ကြည့်လေ့လာပြီး၊ လက်ရှိ subsite အတွက် input နှင့် output token အရေအတွက်များကို မှတ်တမ်းတင်ကာ ထို token များကို ဖောက်သည်၏ လက်ရှိ ဘောင်ချာထုတ်သည့် ကာလအတွက် ဘောင်ချာထုတ်နိုင်သော AI အသုံးပြုမှုအဖြစ် ပြောင်းလဲပေးပါသည်။

ခန့်မှန်းရခက်သော model ကုန်ကျစရိတ်များကို ကိုယ်တိုင် Bear လုပ်စရာမလိုဘဲ ဖောက်သည်များ၏ site များတွင် AI ကိရိယာများကို ပေးအပ်လိုပါက AI token ဖြင့် ငွေတောင်းခံခြင်းကို အသုံးပြုသင့်ပါသည်။ တောင်းဆိုမှုတစ်ခုစီတိုင်းကို subsite၊ ဖောက်သည်၊ အဖွဲ့ဝင်၊ ပေးသူ (provider) နှင့် model တို့နှင့် ချိတ်ဆက်ထားသောကြောင့် စီမံခန့်ခွဲသူများသည် အသုံးပြုမှုကို စစ်ဆေးနိုင်ပြီး ဖောက်သည်များကလည်း ၎င်းတို့၏ AI အသုံးပြုမှုက ၎င်းတို့၏ အကောင့်ကို မည်သို့သက်ရောက်မှုရှိသည်ကို မြင်တွေ့နိုင်ပါသည်။

### Token အလိုက် နှုန်းထားများ စီမံခန့်ခွဲခြင်း (Configure Per-Token Rates) {#configure-per-token-rates}

AI နှုန်းထားများကို model တစ်ခုချင်းစီအတွက် သီးခြား input နှင့် output နှုန်းထားများအဖြစ် စီမံခန့်ခွဲရပါသည်။ နှုန်းထားများကို 1 million tokens အလိုက် တွက်ချက်ပြီး၊ ၎င်းသည် အများစု AI ပေးသူများ၏ နှုန်းထားစာမျက်နှာများနှင့် ကိုက်ညီပါသည်။

1. Network admin တွင် **Multisite Ultimate → Settings → Metered Plans** ကို ဖွင့်ပါ။
2. AI အသုံးပြုမှုကို ဘောင်ချာထုတ်သင့်သည့် product သို့မဟုတ် membership အတွက် AI spend metering ကို ဖွင့်ပါ။
3. သင့် connector မှ ပြန်ပေးသည့် model ID များ (ဥပမာ- `gpt-4o`, `gpt-4o-mini` သို့မဟုတ် အခြား OpenAI-compatible model identifier) ကို အသုံးပြု၍ model rate entries များကို ထည့်ပါ။
4. 1 million tokens အတွက် သင့် site currency ဖြင့် သီးခြား **Input Rate** နှင့် **Output Rate** တန်ဖိုးများကို ရိုက်ထည့်ပါ။
5. အစီအစဉ်အတွက် ဖောက်သည်၏ ပါဝင်သည့် AI spend allowance နှင့် ပိုလျှံမှု အပြုအမူကို သတ်မှတ်ပါ။

တိကျသော model ID ကို မသတ်မှတ်ထားပါက၊ Metered Plans သည် model prefix ဖြင့် ကိုက်ညီအောင် လုပ်ဆောင်ပေးနိုင်ပါသည်။ ဥပမာအားဖြင့်၊ `claude-sonnet-4` အတွက် သတ်မှတ်ထားသော နှုန်းထားသည် ထို prefix ဖြင့် စတင်သည့် timestamped model ID များအတွက် အကျုံးဝင်နိုင်သည်။ Developer များသည် [`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates) filter ကို အသုံးပြု၍ နောက်ဆုံး နှုန်းထားဇယားကိုလည်း ချိန်ညှိနိုင်ပါသည်။

### ပံ့ပိုးထားသော AI Connectors (Supported AI Connectors) {#supported-ai-connectors}

AI token ဖြင့် ငွေတောင်းခံခြင်းသည် WordPress AI Client SDK ထုတ်လုပ်မှု အဖြစ်အပျက်များကို ထုတ်လွှတ်ပြီး response metadata တွင် token အသုံးပြုမှုကို ထည့်သွင်းသည့် connectors များနှင့် အလုပ်လုပ်ပါသည်။ ၎င်းတွင် ဤနေရာတွင် စာရင်းပြုစုထားသော Ultimate Multisite AI connector family ပါဝင်သည်-

* [Ultimate AI Connector for Compatible Endpoints](../ultimate-ai-connector-compatible-endpoints/) သည် OpenAI, Mistral, Ollama-compatible gateways နှင့် ၎င်းနှင့် ကိုက်ညီသော token usage metadata ကို ထုတ်ပြသည့် အခြားပေးသူများ အပါအဝင် OpenAI-compatible API များအတွက်။
* [Ultimate AI Connector for WebLLM](../ultimate-ai-connector-webllm/) သည် တောင်းဆိုမှုများကို AI Client SDK event flow မှတစ်ဆင့် လမ်းကြောင်းပေးသည့်အခါ browser-native WebLLM အသုံးပြုမှုအတွက်။
* [Gratis AI Agent](../gratis-ai-agent/) မှ အသုံးပြုသော AI ပေးသူ connectors များသည် ရွေးချယ်ထားသော ပေးသူက prompt နှင့် completion token အရေအတွက်များကို SDK မှတစ်ဆင့် အစီရင်ခံသည့်အခါ။

Custom connectors များသည် model၊ provider နှင့် token usage metadata တို့ဖြင့် standard `wp_ai_client_before_generate_result` နှင့် `wp_ai_client_after_generate_result` events များကို ပစ်လွှတ်ခြင်းဖြင့် ပါဝင်နိုင်ပါသည်။ Connector settings များကို main site မှ subsite များသို့ အကောင်အထည်ဖော်ထားခြင်းကြောင့် ဖောက်သည် site များသည် network မှ အတည်ပြုထားသော ပေးသူများနှင့် ကန့်သတ်ချက်များကို အသုံးပြုရမည်ဖြစ်သည်။

### ဖောက်သည် အကောင့် အသုံးပြုမှု (Customer Account Usage) {#customer-account-usage}

ဖောက်သည်များသည် ၎င်းတို့၏ အကောင့်ဧရိယာတွင် ၎င်းတို့၏ အခြားသော အသုံးပြုမှုအပေါ် အခြေခံ အရင်းအမြစ်များနှင့်အတူ AI အသုံးပြုမှုကို မြင်တွေ့ရမည်။ AI spend row သည် မှတ်တမ်းတင်ထားသော prompt နှင့် completion token များ၊ သတ်မှတ်ထားသော model နှုန်းထားများ၊ နှင့် အစီအစဉ်ရှိ ပါဝင်သည့် allowance တို့မှ ရရှိလာသော လက်ရှိ ဘောင်ချာထုတ်သည့် ကာလ စုစုပေါင်းကို ပြသသည်။

ဖောက်သည်တစ်ဦးတွင် ငွေပေးချေမှု နည်းလမ်းတစ်ခု ရှိပါက၊ ပါဝင်သည့် allowance ကို ကျော်လွန်သည့် အသုံးပြုမှုသည် ဘောင်ချာထုတ်နိုင်သော ပိုလျှံမှုအဖြစ် သတ်မှတ်သည်။ အကယ်၍ အစီအစဉ်သည် ခက်ခဲသော ကန့်သတ်ချက် (hard cutoff) အတွက် စီမံခန့်ခွဲထားပြီး ဖောက်သည်တွင် ငွေပေးချေမှု နည်းလမ်း မရှိပါက၊ allowance ရောက်ရှိပြီးနောက် AI တောင်းဆိုမှုများကို ပိတ်ဆို့နိုင်ပြီး၊ ဆက်လက်လုပ်ဆောင်ရန် ဖောက်သည်အား ငွေပေးချေမှု အသေးစိတ်အချက်အလက်များ ထည့်ရန် အကြောင်းကြားနိုင်သည်။

## ပံ့ပိုးထားသော အရင်းအမြစ်များ (Supported Resources) {#supported-resources}

* Disk storage (သိုလှောင်မှု)
* Bandwidth/Data transfer (bandwidth/ဒေတာလွှဲပြောင်းမှု)
* Number of sites (ဝက်ဘ်ဆိုဒ်အရေအတွက်)
* Number of users (အသုံးပြုသူအရေအတွက်)
* AI token usage and AI spend (AI token အသုံးပြုမှုနှင့် AI ကုန်ကျစရိတ်)
* Custom resource types (စိတ်ကြိုက် အရင်းအမြစ်အမျိုးအစားများ)

## လိုအပ်ချက်များ (Requirements) {#requirements}

* Multisite Ultimate 2.0.0 သို့မဟုတ် အထက်
* WordPress Multisite တပ်ဆင်မှု
* PHP 7.4 သို့မဟုတ် အထက်

## တပ်ဆင်ခြင်း (Installation) {#installation}

1. plugin files များကို `/wp-content/plugins/ultimate-multisite-metered-plans` သို့ upload လုပ်ပါ။
2. WordPress ၏ 'Plugins' screen မှတစ်ဆင့် plugin ကို activate လုပ်ပါ။
3. Multisite Ultimate → Settings → Metered Plans သို့ သွားပါ။
4. သင့်ရဲ့ metered resources များနှင့် နှုန်းထားများကို စီမံခန့်ခွဲပါ။
5. product editor တွင် product များအတွက် metered billing ကို ဖွင့်ပါ။

## မကြာခဏ မေးလေ့ရှိသော မေးခွန်းများ (Frequently Asked Questions) {#frequently-asked-questions}

## ဖောက်သည်တစ်ဦးသည် ၎င်းတို့၏ ကန့်သတ်ချက်ကို ကျော်လွန်ပါက ဘာဖြစ်မလဲ။ {#what-happens-when-a-customer-exceeds-their-limit}

ဝင်ရောက်ခွင့်ကို ပိတ်ဆို့မည့်အစား၊ စနစ်သည် ပိုလျှံမှုကို စောင့်ကြည့်ပြီး သင်သတ်မှတ်ထားသော နှုန်းထားများအပေါ် အခြေခံ၍ ၎င်းတို့၏ နောက်ဘောင်ချာတွင် ထည့်ပေးပါသည်။

## မတူညီသော product များအတွက် မတူညီသော ပိုလျှံမှု နှုန်းထားများကို သတ်မှတ်လို့ရပါသလား။ {#can-i-set-different-overage-rates-for-different-products}

ဟုတ်ကဲ့၊ product တစ်ခုချင်းစီတွင် ထူးခြားသော metered resources နှင့် ပိုလျှံမှု နှုန်းထားများ ရှိနိုင်ပါသည်။

## ဖောက်သည်များကို အသုံးပြုမှုအကြောင်း ဘယ်လို အကြောင်းကြားပါသလဲ။ {#how-are-customers-notified-about-usage}

ဖောက်သည်များသည် ၎င်းတို့၏ dashboard တွင် လက်ရှိ အသုံးပြုမှုကို ကြည့်ရှုနိုင်ပြီး၊ ကန့်သတ်ချက်နီးလာသည် သို့မဟုတ် ကျော်လွန်ခါနီးတွင် ရွေးချယ်နိုင်သော အကြောင်းကြားချက်များကိုလည်း ရရှိနိုင်ပါသည်။

## AI token အသုံးပြုမှုကို ဘယ်လို နှုန်းထားနဲ့ တွက်ချက်ပါသလဲ။ {#how-is-ai-token-usage-priced}

AI token အသုံးပြုမှုကို model ၏ သတ်မှတ်ထားသော input နှင့် output နှုန်းထားများဖြင့် 1 million tokens အလိုက် နှုန်းထားသတ်မှတ်ထားပါသည်။ Metered Plans သည် တောင်းဆိုမှုတစ်ခုစီ၏ prompt နှင့် completion token အရေအတွက်များကို ထိုနှုန်းထားများဖြင့် မြှောက်ကာ၊ ရလဒ်ကို ဖောက်သည်၏ လက်ရှိ ဘောင်ချာထုတ်သည့် ကာလ AI spend တွင် ပေါင်းထည့်ပြီး၊ အစီအစဉ်၏ allowance သို့မဟုတ် overage စည်းမျဉ်းများကို အသုံးပြုပါသည်။

## ခက်ခဲသော ကန့်သတ်ချက်များကို ဆက်လက် အသုံးပြုလို့ရပါသလား။ {#can-i-still-use-hard-limits}

ဟုတ်ကဲ့၊ metered billing သည် product တစ်ခုချင်းစီအတွက် ရွေးချယ်နိုင်ပါသည်။ သမားရိုးကျ hard-limit product များနှင့် metered product များကို ရောနှောအသုံးပြုနိုင်ပါသည်။
