# VAT ပြောင်းလဲမှုမှတ်တမ်း {#vat-changelog}

Version 1.0.7 - 2026-02-03 တွင် ထုတ်ဝေခဲ့သည်

* ပြင်ဆင်မှု (Fix): VIES SOAP service MS_MAX_CONCURRENT_REQ အမှားများကြောင့် German (DE) VAT နံပါတ်များ အတည်ပြုမှု အမှားဖြစ်ခဲ့ခြင်းကို ပြင်ဆင်ခဲ့ပါသည်။ ယခင် SOAP API မှ ယုံကြည်စိတ်ချရမှု ပိုမိုမြင့်မားသည့် တရားဝင် EU VIES REST API သို့ ပြောင်းလဲအသုံးပြုခဲ့ပါသည်။
* ပြင်ဆင်မှု (Fix): VIES ဝန်ဆောင်မှု ခေတ္တမရရှိနိုင်သည့်အခါ VAT နံပါတ်များကို မှားယွင်းစွာ ပယ်ချခြင်းမရှိဘဲ လက်ခံနိုင်ပြီဖြစ်သည်။ နောက်ပိုင်းတွင် ပြန်လည်စစ်ဆေးရန်အတွက် မှတ်တမ်း (log entry) တစ်ခုကို ဖန်တီးပေးထားပါသည်။
* ပြင်ဆင်မှု (Fix): VAT/Tax ID field ကို EU အဖွဲ့ဝင်နိုင်ငံများအတွက်သာမက နိုင်ငံအားလုံးအတွက် မြင်ရပြီဖြစ်သည်။ Non-EU tax ID များ (ဥပမာ- Swiss CHE နံပါတ်များ) ကို VIES အတည်ပြုမှုမရှိဘဲ ငွေတောင်းခံလွှာ (invoice) တွင် ပြသရန် သိမ်းဆည်းထားပါသည်။
* ပြင်ဆင်မှု (Fix): Greece (GR), Monaco (MC) နှင့် Isle of Man (IM) တို့အတွက် VAT prefix ရှာဖွေမှုသည် မှားယွင်းသော array lookup logic ကြောင့် ပျက်ယွင်းခဲ့ခြင်းကို ပြင်ဆင်ခဲ့ပါသည်။
* တိုးတက်မှု (Improvement): Site Exporter addon နှင့် အတူအသုံးပြုသည့်အခါ အပြင်းအထန် အမှားများဖြစ်စေနိုင်သည့် မလိုအပ်သော wp-cli-bundle dev dependency ကို ဖယ်ရှားခဲ့ပါသည်။
* တိုးတက်မှု (Improvement): အစုံအလင် ယူနစ် စမ်းသပ်မှု အစုံအလင် (53 tests) ကို ထည့်သွင်းခဲ့ပါသည်။

Version 1.0.6 - 2026-01-25 တွင် ထုတ်ဝေခဲ့သည်

* ပြင်ဆင်မှု (Fix): ပြည်တွင်း B2B အရောင်းအဝယ်များတွင် VAT ကို မှန်ကန်စွာ ကောက်ခံနိုင်ပြီဖြစ်သည်။ EU VAT စည်းမျဉ်းများအရ နယ်စပ်ဖြတ်ကျော် B2B အရောင်းအဝယ်များတွင်သာ reverse charge ကို အသုံးပြုရမည်ဖြစ်ပြီး၊ ဝယ်သူနိုင်ငံနှင့် ကုမ္ပဏီနိုင်ငံ တူညီသည့်အခါတွင် မဟုတ်တော့ပါ။
