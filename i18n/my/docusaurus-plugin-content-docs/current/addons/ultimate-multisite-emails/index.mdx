# အီးမေးလ်များ (Emails)

Ultimate Multisite: Emails addon ကို အသုံးပြုခြင်းဖြင့် network ပိုင်ရှင်များသည် ဝက်ဘ်ဆိုဒ်များနှင့် အဖွဲ့ဝင်အကောင့်များအပြင်၊ ဖောက်သည်များ၏ အီးမေးလ်အကောင့်များကိုပါ စီမံခန့်ခွဲပေးနိုင်ပါသည်။ Version 1.1.0 တွင် ဖောက်သည်များကိုယ်တိုင် အီးမေးလ်အကောင့်များကို စီမံခန့်ခွဲနိုင်ခြင်း၊ ဝဘ်မေးလ် Single Sign-On (SSO)၊ CyberPanel forwarder ပံ့ပိုးမှုနှင့် ငွေရှင်းချိန်တွင် အကောင့်ဖွင့်ပေးမှု စသည့် အဆင့်မြှင့်တင်မှုများ ပါဝင်ပါသည်။

## အဓိက လုပ်ဆောင်ချက်များ (Key Features)

* **ဖောက်သည် အီးမေးလ်အကောင့် ဝစ်ဂျက် (Customer email account widget)**: ဖောက်သည်များသည် Support Ticket တစ်ခုဖွင့်စရာမလိုဘဲ ၎င်းတို့၏ အကောင့်စာမျက်နှာမှ အီးမေးလ်အကောင့်များကို ဖန်တီးကာ စီမံခန့်ခွဲနိုင်ပါသည်။
* **Webmail SSO နှင့် magic links**: ပံ့ပိုးပေးသည့် ဝန်ဆောင်မှုများရှိပါက ဖောက်သည်များသည် အကောင့်စာမျက်နှာမှ တစ်ချက်နှိပ်ရုံဖြင့် ဝဘ်မေးလ်ကို တိုက်ရိုက်ဝင်ရောက်ကြည့်ရှုနိုင်ပါသည်။
* **စကားဝှက် ပြောင်းလဲခြင်း (Password changes)**: ဖောက်သည်များနှင့် စီမံခန့်ခွဲသူများ (administrators) သည် အီးမေးလ် စီမံခန့်ခွဲမှု ဝစ်ဂျက်မှတစ်ဆင့် မေးလ်ဘောက်စ် စကားဝှက်များကို အပ်ဒိတ်လုပ်နိုင်ပါသည်။
* **ငွေရှင်းချိန်တွင် အကောင့်ဖွင့်ပေးခြင်း (Checkout provisioning)**: ဖောက်သည်၏ အီးမေးလ်အကောင့်အသစ်ကို ငွေရှင်းနေစဉ်အတွင်းမှာပင် ဖန်တီးပေးနိုင်သောကြောင့် ဝယ်ယူမှု လုပ်ငန်းစဉ် ပြီးဆုံးသည်နှင့် မေးလ်ဘောက်စ် အသင့်ဖြစ်နေမည်ဖြစ်သည်။
* **CyberPanel forwarders**: CyberPanel ကို အသုံးပြုထားသည့် network များသည် အီးမေးလ် ဝန်ဆောင်မှု လုပ်ငန်းစဉ်၏ တစ်စိတ်တစ်ပိုင်းအဖြစ် အီးမေးလ် forwarder များကို ဖန်တီးကာ စီမံခန့်ခွဲနိုင်ပါသည်။
* **ပံ့ပိုးပေးသူအလိုက် စက်တင်များ (Provider-aware settings)**: ဤ Settings စာမျက်နှာသည် ပံ့ပိုးပေးသူများကို အသုံးပြုနိုင်ပြီး၊ စနစ်တကျ ဖွဲ့စည်းထားသည့် ပေါင်းစပ်မှုများ (integrations) သာကို စစ်ထုတ်ပြသပေးသောကြောင့် စီမံခန့်ခွဲသူများသည် အသုံးပြုနိုင်သည့် ရွေးချယ်စရာများကိုသာ မြင်ရမည်ဖြစ်သည်။

## လိုအပ်ချက်များ (Requirements)

* Ultimate Multisite 2.0.0 သို့မဟုတ် အထက်
* WordPress Multisite တပ်ဆင်ထားရမည်
* ဖွဲ့စည်းထားသော အီးမေးလ် ပံ့ပိုးပေးသူ ပေါင်းစပ်မှု (provider integration) အနည်းဆုံး တစ်ခု
* မေးလ်ဘောက်စ်များ ဖန်တီးခြင်း၊ စကားဝှက်များ ပြောင်းလဲခြင်းနှင့် ဝဘ်မေးလ် ဝင်ရောက်ခွင့် link များ ထုတ်ပေးခြင်းတို့ကို ခွင့်ပြုသည့် Provider API credentials များ

## တပ်ဆင်ခြင်း (Installation)

1. ပလပ်အင်ဖိုင်များကို `/wp-content/plugins/ultimate-multisite-emails` နေရာသို့ တင်ပါ။
2. WordPress network admin မှ **Plugins** နေရာမှ addon ကို Network အဆင့်မှာ Activate လုပ်ပါ။
3. **Ultimate Multisite** > **Settings** သို့ သွားပြီး addon ၏ အီးမေးလ် စက်တင်များကို ဖွင့်ပါ။
4. ဖောက်သည်များ အသုံးပြုစေလိုသည့် အီးမေးလ် ပံ့ပိုးပေးသူ ပေါင်းစပ်မှုများကို Enable လုပ်ပါ။
5. စက်တင်များကို Save လုပ်ပြီးနောက်၊ စမ်းသပ်ထုတ်လုပ်မှု (production) မဟုတ်သည့် ဖောက်သည် သို့မဟုတ် staging site ဖြင့် အကောင့်ဖွင့်ခြင်းကို စမ်းသပ်ကြည့်ပါ။

## ဖောက်သည် အကောင့် အီးမေးလ် စီမံခန့်ခွဲမှု (Customer Account Email Management)

Version 1.1.0 တွင် ဖောက်သည် အကောင့်စာမျက်နှာ၌ အီးမေးလ် စီမံခန့်ခွဲမှု ဝစ်ဂျက်ကို ထပ်ပေါင်းထည့်ထားပါသည်။ ဖောက်သည်များက ၎င်းတို့ကိုယ်တိုင် ဝစ်ဂျက်ကို အသုံးပြု၍ မေးလ်ဘောက်စ်များ ဖန်တီးထိန်းသိမ်းရန် လိုအပ်သည့်အခါ ဤအရာကို အသုံးပြုပါ။

အကောင့်စာမျက်နှာမှ ဖောက်သည်များသည် အောက်ပါအရာများကို လုပ်ဆောင်နိုင်သည်-

1. ၎င်းတို့၏ လက်ရှိ အဖွဲ့ဝင်အကောင့် သို့မဟုတ် ဝက်ဘ်ဆိုဒ်နှင့် ချိတ်ဆက်ထားသော အီးမေးလ်အကောင့်များကို ကြည့်ရှုနိုင်ပါသည်။
2. ၎င်းတို့၏ အစီအစဉ် (plan) က အကောင့်အပိုများ ခွင့်ပြုပါက မေးလ်ဘောက်စ်အသစ်တစ်ခု ဖန်တီးနိုင်ပါသည်။
3. ရှိပြီးသား မေးလ်ဘောက်စ် စကားဝှက်ကို ပြောင်းလဲနိုင်ပါသည်။
4. ပံ့ပိုးပေးထားသည့် ဝဘ်မေးလ် ဝန်ဆောင်မှုများကို SSO သို့မဟုတ် magic-link ဖြင့် ဝင်ရောက်နိုင်ပါသည်။

ဤဝစ်ဂျက်သည် Ultimate Multisite ၏ account-page currents system ကို အသုံးပြုထားသောကြောင့် လုပ်ဆောင်မှုများသည် ဖောက်သည် ကြည့်ရှုနေသည့် အကောင့်အကြောင်းအရာ (account context) အတွင်း၌သာ ကန့်သတ်ထားပါသည်။ ၎င်းသည် ဖောက်သည်များက အခြားအဖွဲ့ဝင်အကောင့် သို့မဟုတ် ဝက်ဘ်ဆိုဒ်နှင့် ပိုင်ဆိုင်သည့် အီးမေးလ်အကောင့်များကို စီမံခန့်ခွဲခြင်းမှ ကာကွယ်ပေးပါသည်။

## Webmail SSO နှင့် Magic-Link Login

Webmail SSO သည် ဖောက်သည်များအား အကောင့်စာမျက်နှာမှတစ်ဆင့် တိုက်ရိုက် **Open webmail** လုပ်ဆောင်ချက်ကို ပေးပါသည်။ အသုံးပြုနေသည့် ပံ့ပိုးပေးသူက ပံ့ပိုးပေးပါက၊ addon သည် အချိန်တိုအတွင်း အသုံးပြုခွင့် URL တစ်ခုကို တောင်းယူပြီး ဖောက်သည်အား မေးလ်ဘောက်စ် credentials များကို ကူးယူရန် မလိုဘဲ ဝဘ်မေးလ်ထဲသို့ ပြန်လည်ညွှန်ပြပေးပါသည်။

ပံ့ပိုးပေးသူများ၏ လုပ်ဆောင်ပုံသည် ကွဲပြားနိုင်သည်-

* Native SSO API များရှိသော ပံ့ပိုးပေးသူများသည် တိုက်ရိုက် အထောက်အထားစစ်ဆေးထားသည့် ဝဘ်မေးလ် session တစ်ခုကို ဖန်တီးနိုင်သည်။
* Magic-link ပံ့ပိုးမှုရှိသော ပံ့ပိုးပေးသူများသည် အချိန်ကန့်သတ်ထားသည့် login link တစ်ခုကို ထုတ်ပေးနိုင်သည်။
* SSO သို့မဟုတ် magic-link ပံ့ပိုးမှုမရှိသော ပံ့ပိုးပေးသူများသည် မေးလ်ဘောက်စ် အသေးစိတ်အချက်အလက်များနှင့် စကားဝှက် စီမံခန့်ခွဲမှု လုပ်ဆောင်ချက်များကို ပြသရမည်ဖြစ်သော်လည်း တစ်ချက်နှိပ်ရုံဖြင့် ဝဘ်မေးလ် ဝင်ရောက်ခွင့် ခလုတ်မှာ မရှိပါ။

လိုအပ်သော API credentials များကို ဖွဲ့စည်းထားသည့် ပံ့ပိုးပေးသူများအတွက်သာ Webmail ဝင်ရောက်ခွင့်ကို Enable လုပ်ပါ။ လုပ်ဆောင်ချက်တစ်ခုခု ပံ့ပိုးပေးသူအတွက် ပျောက်နေပါက၊ ပထမဦးစွာ ထိုပံ့ပိုးပေးသူသည် အသက်ဝင်ပြီး၊ ဖွဲ့စည်းထားပြီး၊ webmail login link များကို ပံ့ပိုးပေးခြင်းရှိမရှိကို အတည်ပြုပါ။

## ငွေရှင်းချိန်တွင် အကောင့်ဖွင့်ပေးခြင်း (Checkout Provisioning)

ဤ addon သည် ငွေရှင်းခြင်း၏ တစ်စိတ်တစ်ပိုင်းအဖြစ် ဖောက်သည် အီးမေးလ်အကောင့်များကို ဖန်တီးပေးနိုင်သည်။ ဤအရာသည် အစီအစဉ် (plan) တွင် မေးလ်ဘောက်စ်တစ်ခု ပါဝင်သည့်အခါ သို့မဟုတ် ငွေရှင်းပုံစံက အီးမေးလ်အကောင့် အသေးစိတ်အချက်အလက်များကို စုဆောင်းသည့်အခါ အသုံးဝင်ပါသည်။

အကြံပြုထားသော အဆင့်ဆင့်လုပ်ဆောင်မှု (Recommended rollout):

1. ငွေရှင်းချိန်တွင် အကောင့်ဖွင့်ပေးခြင်းကို Enable မလုပ်မီ ပံ့ပိုးပေးသူနှင့် အစီအစဉ် ကန့်သတ်ချက်များကို ဖွဲ့စည်းပါ။
2. staging product ဖြင့် ဝယ်ယူမှုတစ်ခုကို စမ်းသပ်ပြီး ငွေရှင်းပြီးနောက် မေးလ်ဘောက်စ်ကို ဖန်တီးပေးခြင်း ရှိ၊ မရှိ အတည်ပြုပါ။
3. ဖောက်သည် အကောင့်စာမျက်နှာကို ပြန်လည်သုံးသပ်ပြီး မေးလ်ဘောက်စ်အသစ်သည် အီးမေးလ် ဝစ်ဂျက်တွင် ပေါ်လာခြင်း ရှိ၊ မရှိ အတည်ပြုပါ။
4. ထုတ်လုပ်မှု (production) အတွက် လုပ်ငန်းစဉ်ကို Enable မလုပ်မီ စကားဝှက်ပြောင်းလဲခြင်းနှင့် webmail လုပ်ဆောင်ချက်များကို အတည်ပြုပါ။

အကယ်၍ အကောင့်ဖွင့်ပေးခြင်း ပျက်ကွက်ပါက၊ မေးလ်ဘောက်စ်အတွက် စီမံခန့်ခွဲသူ၏ နောက်ဆက်တွဲလုပ်ဆောင်မှု လိုအပ်နေသော်လည်း ဖောက်သည်အကောင့်နှင့် အဖွဲ့ဝင်အကောင့်သည် ရှိနေနိုင်ပါသည်။ ပြန်လည်ကြိုးစားမလုပ်မီ Ultimate Multisite activity logs နှင့် ပံ့ပိုးပေးသူ logs များကို စစ်ဆေးပါ။

## CyberPanel အီးမေးလ် Forwarders

CyberPanel ကို အသုံးပြုထားသည့် network များအတွက် Version 1.1.0 တွင် အီးမေးလ် forwarder ပံ့ပိုးမှုကို ထပ်ပေါင်းထည့်ထားပါသည်။ ဖောက်သည်များက `info@customer-domain.test` ကဲ့သို့သော လိပ်စာများကို အပြည့်အစုံ မေးလ်ဘောက်စ် ဖန်တီးခြင်းမရှိဘဲ အခြား inbox သို့ လမ်းကြောင်းပြောင်းပေးရန် လိုအပ်သည့်အခါ forwarder များကို အသုံးပြုပါ။

forwarder များကို ပေးအပ်ခြင်းမပြုမီ CyberPanel integration သည် ဖွဲ့စည်းထားပြီး၊ အသုံးပြုနိုင်သည်ကို အတည်ပြုပါ။ လိုအပ်သော host၊ username၊ password နှင့် package constants များအတွက် [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) လမ်းညွှန်ကို ကြည့်ပါ။

## မေးလေ့ရှိသော မေးခွန်းများ (Frequently Asked Questions)

## ဖောက်သည်များက ၎င်းတို့ကိုယ်ပိုင် အီးမေးလ်အကောင့်များ ဖန်တီးနိုင်ပါသလား။

ဟုတ်ကဲ့။ ဖောက်သည် အကောင့် ဝစ်ဂျက်ကို Enable လုပ်ထားပြီး အစီအစဉ်က အီးမေးလ်အကောင့်များကို ခွင့်ပြုပါက ဖောက်သည်များသည် အကောင့်စာမျက်နှာမှ ၎င်းတို့၏ မေးလ်ဘောက်စ်များကို ဖန်တီးကာ စီမံခန့်ခွဲနိုင်ပါသည်။

## ပံ့ပိုးပေးသူတိုင်းက webmail SSO ကို ပံ့ပိုးပေးပါသလား။

မဟုတ်ပါ။ SSO နှင့် magic-link ပံ့ပိုးမှုသည် အသုံးပြုနေသည့် အီးမေးလ် ပံ့ပိုးပေးသူအပေါ် မူတည်ပါသည်။ access-link API မရှိသော ပံ့ပိုးပေးသူများသည် မေးလ်ဘောက်စ် ဖန်တီးခြင်းနှင့် စကားဝှက်များ ပြောင်းလဲခြင်းများကို ဆက်လက် ပံ့ပိုးပေးနိုင်ပါသည်။

## စီမံခန့်ခွဲသူများက ဖောက်သည်များအတွက် အီးမေးလ်အကောင့်များ ဖန်တီးပေးနိုင်ပါသလား။

ဟုတ်ကဲ့။ စီမံခန့်ခွဲသူများသည် အုပ်ချုပ်မှု အင်တာဖေ့စ် (admin interface) မှတစ်ဆင့် အကောင့်အသစ်ဖန်တီးစဉ် အစပိုင်း စကားဝှက်ကို သတ်မှတ်ခြင်းအပါအဝင် အီးမေးလ်အကောင့်များကို ဖန်တီးပေးနိုင်ပါသည်။

## forwarder တွေက mailbox တွေနဲ့ တူပါသလား။

မဟုတ်ပါ။ မေးလ်ဘောက်စ်တစ်ခုသည် မေးလ်များကို သိမ်းဆည်းထားပြီး login credentials များ ပါရှိသည်။ forwarder တစ်ခုသည် မေးလ်များကို အခြားလိပ်စာတစ်ခုသို့ လမ်းကြောင်းပြောင်းပေးရုံသာဖြစ်ပြီး သီးခြား inbox တစ်ခုကို မပေးပါ။
