# ကိရိယာများနှင့် ပြင်ဆင်သတ်မှတ်မှု

Theme Builder သည် သင့် WordPress site ကို ခွဲခြမ်းစိတ်ဖြာရန်၊ ဒီဇိုင်းရေးဆွဲရန်နှင့် တည်ဆောက်ရန် ကိရိယာအစုတစ်ခုကို အသုံးပြုသည်။ ဗားရှင်း 1.16.0 တွင် **sd-ai-agent/site-scrape သည် ယခု Tier 1 ကိရိယာ ဖြစ်လာပြီး** Theme Builder တွင် မူလအတိုင်း အသုံးပြုနိုင်သည်။ Superdav AI Agent v1.18.0 သည် ယုံကြည်ရသော developer လုပ်ငန်းစဉ်များအတွက် သီးခြားဖြန့်ဝေထားသော Advanced companion package ကိုလည်း မိတ်ဆက်ထားသည်။ ထိုကိရိယာများသည် WordPress.org core package ၏ အစိတ်အပိုင်းမဟုတ်ဘဲ သီးခြားထည့်သွင်းပြီး ခွင့်ပြုချက်ပေးရမည်။

## အသုံးပြုနိုင်သော ကိရိယာများ

### Tier 1 ကိရိယာများ (အမြဲ အသုံးပြုနိုင်)

Tier 1 ကိရိယာများကို ထပ်ဆောင်းပြင်ဆင်သတ်မှတ်မှု မလိုဘဲ Theme Builder တွင် မူလအတိုင်း အသုံးပြုနိုင်သည်။

#### sd-ai-agent/site-scrape

**ရည်ရွယ်ချက်:** ရှိပြီးသား ဝဘ်ဆိုက်များကို ခွဲခြမ်းစိတ်ဖြာပြီး ဒီဇိုင်းလှုံ့ဆော်မှု၊ အကြောင်းအရာဖွဲ့စည်းပုံနှင့် လုပ်ဆောင်နိုင်စွမ်းများကို ထုတ်ယူရန်။

**လုပ်ဆောင်နိုင်စွမ်းများ:**

- **ဝဘ်ဆိုက် ခွဲခြမ်းစိတ်ဖြာမှု** — ပြိုင်ဘက် သို့မဟုတ် လှုံ့ဆော်မှုရယူလိုသော ဝဘ်ဆိုက်များကို scrape လုပ်ပြီး ခွဲခြမ်းစိတ်ဖြာသည်
- **ဒီဇိုင်း ထုတ်ယူမှု** — အရောင်များ၊ ဖောင့်များနှင့် layout ပုံစံများကို ခွဲခြားသတ်မှတ်သည်
- **အကြောင်းအရာ ဖွဲ့စည်းပုံ** — စာမျက်နှာ အစီအစဉ်နှင့် အဆင့်ဆင့်ဖွဲ့စည်းမှုကို နားလည်သည်
- **အင်္ဂါရပ် ရှာဖွေမှု** — plugins များနှင့် လုပ်ဆောင်နိုင်စွမ်းများကို ခွဲခြားသတ်မှတ်သည်
- **စွမ်းဆောင်ရည် ခွဲခြမ်းစိတ်ဖြာမှု** — စာမျက်နှာ မြန်နှုန်းနှင့် အကောင်းဆုံးပြုလုပ်မှုကို စစ်ဆေးသည်
- **SEO ခွဲခြမ်းစိတ်ဖြာမှု** — meta tags များနှင့် structured data ကို သုံးသပ်သည်

**အသုံးပြုမှု:**

```
example.com ၏ ဒီဇိုင်းကို ခွဲခြမ်းစိတ်ဖြာပြီး ကျွန်ုပ်၏ site အတွက် လှုံ့ဆော်မှုအဖြစ် အသုံးပြုပါ။
```

**ထုတ်ယူသည့်အရာများ:**

- အရောင် palette နှင့် အရောင်အသုံးပြုပုံ
- စာလုံးပုံစံ (ဖောင့်များနှင့် အရွယ်အစားများ)
- Layout ဖွဲ့စည်းပုံနှင့် အကွာအဝေး
- Navigation ပုံစံများ
- အကြောင်းအရာ အစီအစဉ်
- ရုပ်ပုံများနှင့် media အသုံးပြုပုံ
- အပြန်အလှန်လုပ်ဆောင်နိုင်သော element များ
- မိုဘိုင်းတုံ့ပြန်နိုင်မှု

**ကန့်သတ်ချက်များ:**

- စကားဝှက်ကာကွယ်ထားသော site များကို scrape လုပ်၍ မရပါ
- robots.txt နှင့် site မူဝါဒများကို လိုက်နာသည်
- Dynamic content ကို ဖမ်းယူနိုင်မည် မဟုတ်နိုင်ပါ
- JavaScript များသော site များတွင် ခွဲခြမ်းစိတ်ဖြာမှု ကန့်သတ်နိုင်သည်
- ကြီးမားသော site များကို ခွဲခြမ်းစိတ်ဖြာရန် ပိုကြာနိုင်သည်

### Tier 2 ကိရိယာများ (ရွေးချယ်နိုင်)

Tier 2 ကိရိယာများကို Theme Builder settings တွင် ရှင်းလင်းစွာ ဖွင့်ထားသောအခါ အသုံးပြုနိုင်သည်။

#### အဆင့်မြင့် Analytics

အသေးစိတ် စွမ်းဆောင်ရည် မက်ထရစ်များကို ပေးသည်-

- စာမျက်နှာတင်ချိန်များ
- Core Web Vitals
- SEO အမှတ်
- Accessibility အမှတ်
- အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းများ အမှတ်

#### အကြောင်းအရာ အကောင်းဆုံးပြုလုပ်ကိရိယာ

အောက်ပါအရာများအတွက် ခွဲခြမ်းစိတ်ဖြာပြီး တိုးတက်မှုများကို အကြံပြုသည်-

- ဖတ်ရှုလွယ်ကူမှု
- SEO အကောင်းဆုံးပြုလုပ်မှု
- Keyword အသုံးပြုပုံ
- အကြောင်းအရာ ဖွဲ့စည်းပုံ
- Call-to-action ထားရှိမှု

### Tier 3 ကိရိယာများ (Premium)

Tier 3 ကိရိယာများသည် ထပ်ဆောင်းပြင်ဆင်သတ်မှတ်မှု သို့မဟုတ် premium အင်္ဂါရပ်များ လိုအပ်သည်။

#### AI အကြောင်းအရာ ထုတ်လုပ်ကိရိယာ

အောက်ပါအရာများအတွက် အကြောင်းအရာ ထုတ်လုပ်သည်-

- Product ဖော်ပြချက်များ
- ဝန်ဆောင်မှု စာမျက်နှာများ
- Blog ပို့စ်များ
- Meta ဖော်ပြချက်များ
- Call-to-action စာသား

#### အဆင့်မြင့် ဒီဇိုင်းကိရိယာများ

အဆင့်မြင့် ဒီဇိုင်းလုပ်ဆောင်နိုင်စွမ်းများကို ပေးသည်-

- Custom CSS ထုတ်လုပ်မှု
- Animation ဖန်တီးမှု
- အပြန်အလှန်လုပ်ဆောင်နိုင်သော element ဒီဇိုင်း
- အဆင့်မြင့် အရောင်သီအိုရီ
- စာလုံးပုံစံ အကောင်းဆုံးပြုလုပ်မှု

### Advanced Companion ကိရိယာများ (သီးခြား Package)

Advanced companion package ကို core Superdav AI Agent release မှ သီးခြားဖြန့်ဝေထားသည်။ ၎င်းသည် agent ကို သက်ရောက်မှုမြင့်မားသော ကိရိယာများ အသုံးပြုခွင့်ပေးရန် administrators များက ရှင်းလင်းစွာ ခွင့်ပြုထားသည့် ယုံကြည်ရသော developer နှင့် site-owner ပတ်ဝန်းကျင်များအတွက် ရည်ရွယ်ထားသည်။

Advanced companion လုပ်ဆောင်နိုင်စွမ်းများတွင် ပါဝင်နိုင်သည်-

- **Filesystem ကိရိယာများ** — ယုံကြည်ရသော ပတ်ဝန်းကျင်များတွင် ခွင့်ပြုထားသော ဖိုင်များကို စစ်ဆေးပြီး ပြင်ဆင်သည်
- **Database ကိရိယာများ** — ရှင်းလင်းစွာ ခွင့်ပြုထားသောအခါ site data ကို query လုပ်ခြင်း သို့မဟုတ် update လုပ်ခြင်း
- **WP-CLI ကိရိယာများ** — WordPress ထိန်းသိမ်းမှုနှင့် စစ်ဆေးမှု command များကို run လုပ်သည်
- **REST dispatcher ကိရိယာများ** — ထိန်းချုပ်ထားသော dispatch လုပ်ငန်းစဉ်များမှတစ်ဆင့် မှတ်ပုံတင်ထားသော REST endpoints များကို ခေါ်သည်
- **Plugin builder ကိရိယာများ** — plugin code ကို scaffold လုပ်ပြီး အဆင့်ဆင့်ပြင်ဆင်တိုးတက်စေသည်
- **Git snapshot ကိရိယာများ** — အန္တရာယ်ရှိနိုင်သော developer လုပ်ဆောင်မှုများမတိုင်မီ snapshots များ ဖန်တီးသည်
- **အသုံးပြုသူစီမံခန့်ခွဲမှု ကိရိယာများ** — ခွင့်ပြုထားသောနေရာများတွင် အသုံးပြုသူ မှတ်တမ်းများကို သုံးသပ်ရန် သို့မဟုတ် ချိန်ညှိရန် ကူညီသည်
- **Benchmark ကိရိယာများ** — ဖွံ့ဖြိုးရေးသုံးသပ်မှုအတွက် စွမ်းဆောင်ရည် သို့မဟုတ် လုပ်ဆောင်နိုင်စွမ်း benchmark များကို run လုပ်သည်

၎င်းတို့ကို တပ်ဆင်မှုတိုင်းအတွက် ယေဘုယျအသုံးပြုနိုင်သော Theme Builder ကိရိယာများအဖြစ် မှတ်တမ်းမတင်ပါနှင့်။ Advanced companion package ကို ထည့်သွင်း၊ ဖွင့်ထားပြီး ယုံကြည်ရသော administrators များအတွက် ကန့်သတ်ထားသည့်အခါမှသာ ၎င်းတို့ကို အသုံးပြုနိုင်သည်။

## ကိရိယာ ပြင်ဆင်သတ်မှတ်မှု

### ကိရိယာများ ဖွင့်ခြင်း

Theme Builder တွင် ထပ်ဆောင်းကိရိယာများ ဖွင့်ရန်-

1. **Gratis AI Agent → Settings** သို့ သွားပါ
2. **Theme Builder → Tools** သို့ သွားပါ
3. လိုအပ်သလို ကိရိယာများကို ဖွင့်/ပိတ် ပြောင်းပါ
4. Settings ကို သိမ်းပါ

### ကိရိယာ ခွင့်ပြုချက်များ

အချို့ကိရိယာများသည် ခွင့်ပြုချက်များ လိုအပ်သည်-

- **Site scraping** — အင်တာနက် အသုံးပြုခွင့် လိုအပ်သည်
- **Analytics** — Google Analytics integration လိုအပ်သည်
- **အကြောင်းအရာ ထုတ်လုပ်မှု** — API keys များ လိုအပ်သည်
- **အဆင့်မြင့် အင်္ဂါရပ်များ** — premium subscription လိုအပ်နိုင်သည်
- **Advanced companion ကိရိယာများ** — သီးခြားဖြန့်ဝေထားသော Advanced package နှင့် administrator ၏ ရှင်းလင်းသော ယုံကြည်ခွင့်ပြုမှု လိုအပ်သည်

### API Keys နှင့် Credentials

လိုအပ်သော ကိရိယာများအတွက် API keys များကို ပြင်ဆင်သတ်မှတ်ပါ-

1. **Gratis AI Agent → Settings → API Keys** သို့ သွားပါ
2. ကိရိယာတစ်ခုချင်းစီအတွက် credentials ထည့်ပါ
3. Connection ကို စမ်းသပ်ပါ
4. လုံခြုံစွာ သိမ်းပါ

### Advanced Companion Package ထည့်သွင်းခြင်း

Advanced companion package ကို WordPress.org core ZIP မှ သီးခြား release လုပ်ထားသည်။ ၎င်းကို release အတွက် တရားဝင် project ဖြန့်ဝေရာ channel မှသာ ထည့်သွင်းပါ။ ထို့နောက် အသုံးပြုခွင့်ကို ယုံကြည်ရသော administrators များအတွက်သာ ကန့်သတ်ပါ။ Production site တွင် filesystem, database, WP-CLI, REST dispatcher, plugin builder, user-management သို့မဟုတ် benchmark ကိရိယာများကို မဖွင့်မီ လူ့အတည်ပြုချက် gate များနှင့် git snapshot အပြုအမူကို သုံးသပ်ပါ။

## sd-ai-agent/site-scrape အသုံးပြုခြင်း

### အခြေခံအသုံးပြုမှု

ဝဘ်ဆိုက်တစ်ခုကို ခွဲခြမ်းစိတ်ဖြာရန် Theme Builder ကို မေးပါ-

```
competitor.com ရှိ ကျွန်ုပ်၏ ပြိုင်ဘက် site ၏ ဒီဇိုင်းကို ခွဲခြမ်းစိတ်ဖြာပြီး
ကျွန်ုပ်၏ site အတွက် ဒီဇိုင်းတိုးတက်မှုများကို အကြံပြုပါ။
```

### သီးခြား ခွဲခြမ်းစိတ်ဖြာမှု

သီးခြား ခွဲခြမ်းစိတ်ဖြာမှု အမျိုးအစားများကို တောင်းဆိုပါ-

```
example.com မှ အရောင် palette ကို ထုတ်ယူပြီး ၎င်းကို လှုံ့ဆော်မှုအဖြစ် အသုံးပြုပါ။
```

```
example.com ၏ navigation ဖွဲ့စည်းပုံကို ခွဲခြမ်းစိတ်ဖြာပြီး ဆင်တူသော
အစီအစဉ်ကို ကျွန်ုပ်၏ site တွင် အသုံးပြုပါ။
```

```
example.com ၏ မိုဘိုင်းတုံ့ပြန်နိုင်မှုကို စစ်ဆေးပြီး ကျွန်ုပ်၏ site သည်
အလားတူ တုံ့ပြန်နိုင်ကြောင်း သေချာစေပါ။
```

### နှိုင်းယှဉ် ခွဲခြမ်းစိတ်ဖြာမှု

Site အများအပြားကို နှိုင်းယှဉ်ပါ-

```
site1.com နှင့် site2.com တို့၏ ဒီဇိုင်းများကို နှိုင်းယှဉ်ပြီး နှစ်ခုလုံး၏
အကောင်းဆုံး element များကို ပေါင်းစပ်ထားသော ဒီဇိုင်းတစ်ခု ဖန်တီးပါ။
```

## ကိရိယာ ကန့်သတ်ချက်များနှင့် ထည့်သွင်းစဉ်းစားချက်များ

### Rate Limiting

- server အလွန်ဝန်ပိမှုကို ကာကွယ်ရန် scraping ကိုနှုန်းကန့်သတ်ထားသည်
- တူညီသော site သို့ တောင်းဆိုမှုများစွာ ပြုလုပ်ပါက အမြန်နှုန်းကန့်သတ်ခံရနိုင်သည်
- site ကြီးများကို ခွဲခြမ်းစိတ်ဖြာရန် ပိုကြာနိုင်သည်

### အကြောင်းအရာ တိကျမှန်ကန်မှု

- ပြောင်းလဲနေသော အကြောင်းအရာများကို အပြည့်အဝ ဖမ်းယူနိုင်မည်မဟုတ်ပါ
- JavaScript ဖြင့် ပြသသော အကြောင်းအရာများ မပြည့်စုံနိုင်ပါ
- အချိန်နှင့်တစ်ပြေးညီ ဒေတာများ မပါဝင်နိုင်ပါ
- အချို့အကြောင်းအရာများသည် အခပေးနံရံနောက်တွင် ရှိနိုင်သည်

### ဥပဒေရေးရာနှင့် ကျင့်ဝတ်ဆိုင်ရာ စဉ်းစားချက်များ

- robots.txt နှင့် site မူဝါဒများကို လေးစားပါ
- မူပိုင်ခွင့်ရှိသော အကြောင်းအရာများကို ပြန်လည်ထုတ်ဝေရန် မ scrape ပါနှင့်
- ကူးယူရန်မဟုတ်ဘဲ အကြံဉာဏ်ရရန် ခွဲခြမ်းစိတ်ဖြာမှုကို အသုံးပြုပါ
- ထုတ်ယူထားသော အကြောင်းအရာကို အသုံးပြုခွင့်ရှိကြောင်း စစ်ဆေးပါ
- ခွဲခြမ်းစိတ်ဖြာထားသော site များ၏ ဝန်ဆောင်မှုစည်းမျဉ်းများကို လိုက်နာပါ

### လုပ်ဆောင်မှုအပေါ် သက်ရောက်မှု

- site ကြီးများကို scrape လုပ်ရန် အချိန်ကြာနိုင်သည်
- တစ်ပြိုင်နက် scrape လုပ်မှုများစွာ ပြုလုပ်ပါက ပိုနှေးနိုင်သည်
- ကွန်ရက်ချိတ်ဆက်မှုသည် အမြန်နှုန်းကို သက်ရောက်စေသည်
- ဒေတာအစုကြီးများအတွက် ပိုမိုလုပ်ဆောင်ရန် လိုအပ်နိုင်သည်

## အကောင်းဆုံး လုပ်နည်းများ

### Site ခွဲခြမ်းစိတ်ဖြာမှု အသုံးပြုခြင်း

1. **site များစွာကို ခွဲခြမ်းစိတ်ဖြာပါ** — အရင်းအမြစ်များစွာမှ အကြံဉာဏ်များ စုဆောင်းပါ
2. **ဖွဲ့စည်းပုံကို အာရုံစိုက်ပါ** — layout နှင့် စီစဉ်ပုံပုံစံများကို လေ့လာပါ
3. **အရောင်များကို ထုတ်ယူပါ** — အရောင်အစုံများကို အစပြုချက်အဖြစ် အသုံးပြုပါ
4. **စာလုံးပုံစံကို လေ့လာပါ** — သင်နှစ်သက်သော font ပေါင်းစပ်မှုများကို ရှာဖွေပါ
5. **လမ်းညွှန်မှုကို ပြန်လည်သုံးသပ်ပါ** — menu စီစဉ်ပုံကို နားလည်ပါ

### ကျင့်ဝတ်နှင့်ညီသော Scraping

1. **robots.txt ကို လေးစားပါ** — site လမ်းညွှန်ချက်များကို လိုက်နာပါ
2. **အကြောင်းအရာကို မကူးယူပါနှင့်** — ခွဲခြမ်းစိတ်ဖြာမှုကို အကြံဉာဏ်ရရန်သာ အသုံးပြုပါ
3. **အခွင့်အရေးများကို စစ်ဆေးပါ** — ထုတ်ယူထားသော အကြောင်းအရာကို အသုံးပြုနိုင်ကြောင်း သေချာပါစေ
4. **အရင်းအမြစ်များကို ဖော်ပြပါ** — အကြံဉာဏ်ရရှိသည့် အရင်းအမြစ်များကို အသိအမှတ်ပြုပါ
5. **စည်းမျဉ်းများကို လိုက်နာပါ** — site ၏ ဝန်ဆောင်မှုစည်းမျဉ်းများကို လိုက်နာပါ

### ကိရိယာ ထိရောက်မှုကို အမြင့်ဆုံးမြှင့်တင်ခြင်း

1. **တိကျစွာ တောင်းဆိုပါ** — ခွဲခြမ်းစိတ်ဖြာမှုအမျိုးအစားများကို တိတိကျကျ တောင်းဆိုပါ
2. **အခြေအနေအကြောင်း ရှင်းပြပါ** — သင့် site ၏ ရည်ရွယ်ချက်ကို ရှင်းပြပါ
3. **မျှော်မှန်းချက်များ သတ်မှတ်ပါ** — သင့်ဒီဇိုင်းရည်မှန်းချက်များကို ဖော်ပြပါ
4. **ထပ်ခါတလဲလဲ ပြုပြင်ပါ** — ရလဒ်များအပေါ် မူတည်၍ ပိုကောင်းအောင် ပြင်ဆင်ပါ
5. **ကိရိယာများ ပေါင်းစပ်အသုံးပြုပါ** — စုံလင်သော ခွဲခြမ်းစိတ်ဖြာမှုအတွက် ကိရိယာများစွာကို အသုံးပြုပါ

## ပြဿနာဖြေရှင်းခြင်း

### Site ကို Scrape လုပ်၍ မရပါ

- site ကို အများပြည်သူ ဝင်ရောက်နိုင်မနိုင် စစ်ဆေးပါ
- robots.txt က scraping ကို ခွင့်ပြုထားကြောင်း စစ်ဆေးပါ
- အခြား site တစ်ခုကို စမ်းကြည့်ပါ
- အင်တာနက်ချိတ်ဆက်မှုကို စစ်ဆေးပါ
- ပြဿနာ ဆက်ရှိနေပါက support ကို ဆက်သွယ်ပါ

### မပြည့်စုံသော ခွဲခြမ်းစိတ်ဖြာမှု

- site တွင် ပြောင်းလဲနေသော အကြောင်းအရာများ ရှိနိုင်သည်
- JavaScript ကို အပြည့်အဝ render မလုပ်နိုင်ခြင်း ဖြစ်နိုင်သည်
- site ကြီးများသည် အချိန်ကုန်ဆုံးသွားနိုင်သည်
- အစားထိုးအနေဖြင့် သီးခြားစာမျက်နှာများကို ခွဲခြမ်းစိတ်ဖြာကြည့်ပါ
- သီးခြား ခွဲခြမ်းစိတ်ဖြာမှုအမျိုးအစားများကို တောင်းဆိုပါ

### နှေးကွေးသော လုပ်ဆောင်မှု

- site ကြီးများကို ခွဲခြမ်းစိတ်ဖြာရန် ပိုကြာသည်
- တစ်ပြိုင်နက် scrape လုပ်မှုများစွာသည် ပိုနှေးသည်
- ကွန်ရက်ချိတ်ဆက်မှုသည် အမြန်နှုန်းကို သက်ရောက်စေသည်
- အသုံးပြုမှုနည်းသော အချိန်များတွင် စမ်းကြည့်ပါ
- အပိုင်းငယ်များကို ဦးစွာ ခွဲခြမ်းစိတ်ဖြာပါ
