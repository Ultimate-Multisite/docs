# ဖောက်သည်များ၏ မှတ်ချက်ပေးခြင်းနှင့် ပြဿနာတင်ပြခြင်း

Gratis AI Agent v1.5.0 တွင် အသုံးပြုသူများက အကူအညီမပေးနိုင်သော အကြောင်းပြန်မှုများကို မှတ်ချက်ပေးနိုင်ပြီး၊ ချတ် (chat) အင်တာဖေ့စ်မှ တိုက်ရိုက် ပြဿနာများ တင်ပြနိုင်သည့် built-in feedback နှင့် issue reporting စနစ်ကို ထည့်သွင်းပေးထားပါသည်။ ဤစနစ်တွင် သဘောတူညီမှု စီမံခန့်ခွဲမှု (consent management)၊ အလိုအလျောက် တင်ပြသည့် command နှင့် backend တွင် AI အကူအညီဖြင့် အမျိုးအစားခွဲခြင်း (AI-assisted triage) တို့ ပါဝင်သည်။

## လက်ညှိုးထိုးချသည့် ခလုတ် (Thumbs-Down Button) {#thumbs-down-button}

AI assistant မှ ပေးပို့သည့် မက်ဆေ့ခ်ျတိုင်းတွင် **လက်ညှိုးထိုးချသည့် ခလုတ်** (👎) တစ်ခု ပါရှိပါသည်။ အသုံးပြုသူက ၎င်းကို နှိပ်လိုက်ပါက အကြောင်းပြန်မှုသည် အကူအညီမပေးနိုင်ခြင်း သို့မဟုတ် မှားယွင်းခြင်းအဖြစ် မှတ်ချက်ပေးနိုင်ပါသည်။

- ဤခလုတ်သည် assistant မက်ဆေ့ခ်ျတစ်ခုစီဘေးတွင် hover လုပ်သည့်အခါ ပေါ်လာမည်။
- ၎င်းကို နှိပ်ခြင်းသည် **Feedback Consent Modal** ကို ဖွင့်ပေးမည်။
- ဤမှတ်ချက်သည် စကားပြောဆိုမှု thread နှင့် သီးသန့် မက်ဆေ့ခ်ျနှင့် ချိတ်ဆက်ထားပါသည်။

## Feedback Consent Modal {#feedback-consent-modal}

အသုံးပြုသူက လက်ညှိုးထိုးချသည့် ခလုတ်ကို နှိပ်လိုက်သည့်အခါ၊ အချက်အလက် မပေးပို့မီ သဘောတူညီမှု Modal တစ်ခု ပေါ်လာမည်ဖြစ်သည်။ ဤ Modal သည်-

- မည်သည့်အချက်အလက်များကို စုဆောင်းမည်ကို ရှင်းပြပေးသည် (စကားပြောဆိုမှု အပိုင်းအစ၊ browser context)။
- ဤအချက်အလက်များကို မျှဝေရန် သဘောတူကြောင်း အသုံးပြုသူအား အတည်ပြုခိုင်းသည်။
- အသုံးပြုသူက မည်သည့်အရာ မှားယွင်းခဲ့သည်ကို ဖော်ပြရန် free-text field တစ်ခု ပေးထားသည်။
- မတင်ပြဘဲ ပယ်ဖျက်ရန် **Cancel** ရွေးချယ်မှုကို ပေးထားသည်။

အသုံးပြုသူက ရှင်းလင်းစွာ အတည်ပြုသည်အထိ မှတ်ချက်ပေးမှု တစ်စုံတစ်ရာ မှတ်တမ်းတင်မည် မဟုတ်ပါ။

## Auto-Prompt Feedback Banner {#auto-prompt-feedback-banner}

စကားပြောဆိုမှု အဆုံးတွင်၊ assistant သည် **auto-prompt feedback banner** တစ်ခုကို ပြသနိုင်သည် — ၎င်းသည် session သည် အထောက်အကူဖြစ်ခဲ့ခြင်း ရှိ၊ မရှိ မေးမြန်းသည့် အနှောင့်အယှက်မဖြစ်စေသော မက်ဆေ့ခ်ျတစ်ခု ဖြစ်သည်။

ဤ banner သည် စကားပြောဆိုမှု အရှည်နှင့် ရလဒ်ဆိုင်ရာ heuristics များအပေါ် အခြေခံ၍ အလိုအလျောက် ပေါ်လာသည်။ ၎င်းသည် လက်ညှိုးထိုးချသည့် ခလုတ်ကဲ့သို့ တူညီသော feedback flow သို့ ချိတ်ဆက်ထားသည်။ အသုံးပြုသူများသည် မှတ်ချက်ပေးခြင်းမရှိဘဲ ဤ banner ကို ပယ်ဖျက်နိုင်သည်။

## /report-issue Slash Command {#report-issue-slash-command}

အသုံးပြုသူများသည် chat input တွင် `/report-issue` ဟု ရိုက်ထည့်ခြင်းဖြင့် feedback flow ကို တိုက်ရိုက် စတင်နိုင်သည်။ ဤ command သည်-

- Feedback Consent Modal ကို ချက်ချင်း ဖွင့်ပေးသည်။
- လက်ရှိ စကားပြောဆိုမှုနှင့် ပတ်သက်သည့် context ဖြင့် description field ကို ကြိုတင်ဖြည့်ပေးသည်။
- တင်ပြခြင်းမပြုမီ အသုံးပြုသူများက အသေးစိတ် အချက်အလက်များ ထပ်ဖြည့်နိုင်စေသည်။

`/report-issue` command ကို chat mode အားလုံး (inline, floating widget, full-screen) တွင် အသုံးပြုနိုင်သည်။

## AI-Assisted Triage {#ai-assisted-triage}

တင်ပြထားသော feedback များသည် AI triage layer သို့ ပို့ဆောင်ခံရပြီး ၎င်းသည်-

- အစီရင်ခံစာကို အမျိုးအစားခွဲပေးသည် (အချက်အလက်မှားယွင်းမှု၊ အကူအညီမပေးနိုင်သော အဖြေ၊ မသင့်လျော်သော အကြောင်းအရာ စသည်)။
- သက်ဆိုင်ရာ စကားပြောဆိုမှု metadata များကို ပူးတွဲပေးသည်။
- triage summary ကို သတ်မှတ်ထားသော **Feedback Endpoint** သို့ ပို့ဆောင်ပေးသည် (အသေးစိတ် သိရန် [Settings > Advanced](#settings-advanced) ကို ကြည့်ပါ)။

၎င်းသည် အရေးကြီးဆုံး ပြဿနာများကို ဦးစွာပေါ်ယ်ဖော်ပေးခြင်းဖြင့် manual review အချိန်ကို လျှော့ချပေးသည်။

## Settings > Advanced {#settings-advanced}

Feedback backend ကို ဖွင့်ရန်၊ **Gratis AI Agent → Settings → Advanced** အောက်တွင် အောက်ပါ field များကို configure လုပ်ပေးရပါမည်။

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | feedback payload များ (JSON) ဖြင့် POST request များ လက်ခံရရှိမည့် URL။ |
| **Feedback API Key** | feedback တင်ပြမှုများကို အတည်ပြုရန် `Authorization` header တွင် ပေးပို့ရမည့် Bearer token။ |

feedback စုဆောင်းခြင်းကို ပိတ်ရန် field နှစ်ခုလုံးကို ဗလာထားခဲ့ပါ။ သို့သော်လည်း၊ feedback ခလုတ်များသည် အသုံးပြုသူများအတွက် မြင်နေရမည်ဖြစ်ပြီး၊ တင်ပြမှုများမှာ ပို့ဆောင်မည်မဟုတ်ပါ။

:::tip
Feedback endpoint သည် အနည်းဆုံး `message_id`၊ `conversation_id`၊ `feedback_text` နှင့် `triage_category` field များပါဝင်သည့် JSON body ကို လက်ခံနိုင်ရပါမည်။ မျှော်လင့်ထားသည့် schema အတွက် သင်၏ endpoint provider ၏ documentation ကို ကြည့်ရှုပါ။
:::
