# သင်၏ ပထမဆုံး Subscription Product ဖန်တီးခြင်း (v2)

_**အရေးကြီးသော မှတ်ချက်- ဤဆောင်းပါးသည် Ultimate Multisite version 2.x အသုံးပြုသူများအတွက်သာ ဖြစ်သည်။ version 1.x အသုံးပြုနေပါက၊**_ **_**ဤဆောင်းပါးကို ကြည့်ပါ**_**။

သင့် network ကို စတင်လည်ပတ်ပြီး အလားအလာရှိသော အသုံးပြုသူများထံ သင့်ဝန်ဆောင်မှုများကို ရောင်းချလိုပါက၊ မတူညီသော subscription ရွေးချယ်စရာများ လိုအပ်ပါသည်။ ဤ products များကို မည်သို့ဖန်တီးမည်နည်း။ မည်သည့် product အမျိုးအစားများကို ကမ်းလှမ်းနိုင်သနည်း။ ဤဆောင်းပါးတွင် products များနှင့် ပတ်သက်၍ သိထားသင့်သမျှကို ဖော်ပြပေးမည်ဖြစ်သည်။

## Product အမျိုးအစား

Ultimate Multisite ဖြင့် သင့်ဖောက်သည်များအတွက် product အမျိုးအစား နှစ်မျိုးကမ်းလှမ်းနိုင်သည်- **plans** နှင့် **add-ons** **(Order Bump)**။ Add-ons များကို အမျိုးအစား နှစ်ခုခွဲခြားနိုင်သည်- **packages** နှင့် **services**။ ၎င်းတို့၏ ကွာခြားချက်များနှင့် ထူးခြားချက်များကို အောက်တွင် ဖော်ပြပေးပါမည်။

  * **Plans** - Ultimate Multisite ၏ အခြေခံ product ဖြစ်သည်။ သင့်ဖောက်သည်သည် plan တစ်ခုနှင့် ချိတ်ဆက်မှသာ membership ရနိုင်သည်။ Plan တစ်ခုသည် သင့်ဖောက်သည်များအား site တစ်ခု သို့မဟုတ် တစ်ခုထက်ပိုသော sites များကို (သင့် plan ၏ ပြင်ဆင်သတ်မှတ်ချက်များပေါ် မူတည်၍) သင့် product တည်းဖြတ်စာမျက်နှာတွင် သတ်မှတ်ထားသော ကန့်သတ်ချက်များဖြင့် ပေးဆောင်သည်။

  * **Packages** - Ultimate Multisite plans များ၏ လုပ်ဆောင်ချက်များကို တိုက်ရိုက်သက်ရောက်စေသော add-ons များဖြစ်သည်။ ၎င်းတို့သည် သင့်ဖောက်သည် ဝယ်ယူခဲ့သော မူလ plan တွင် ကန့်သတ်ချက်များကို ပြောင်းလဲခြင်း သို့မဟုတ် resources အသစ်များ၊ plugins သို့မဟုတ် themes များ ထပ်ထည့်ပေးခြင်းများ ပြုလုပ်နိုင်သည်။ ဥပမာအားဖြင့်၊ basic plan တစ်ခုသည် တစ်လလျှင် visits ၁,၀၀၀ သာ ခွင့်ပြုနိုင်ပြီး ဤအရေအတွက်ကို ၁၀,၀၀၀ အထိ တိုးချဲ့ပေးသော package တစ်ခုကို ရရှိနိုင်အောင် ပြုလုပ်နိုင်သည်။

  * **Services:** Ultimate Multisite ၏ လုပ်ဆောင်ချက်များကို မပြောင်းလဲစေသော add-ons များဖြစ်သည်။ ၎င်းတို့သည် သင့်ဖောက်သည် ဝယ်ယူခဲ့သော plan အပြင် သင်ပြုလုပ်ပေးမည့် အလုပ်များဖြစ်သည်။ ဥပမာအားဖြင့်၊ သင့်ဖောက်သည်သည် site တစ်ခုတည်းသာ ခွင့်ပြုသော plan တစ်ခုကို ဝယ်ယူပြီး ထို site ၏ ဒီဇိုင်းကို ပြုလုပ်ပေးမည့် ဝန်ဆောင်မှုအပိုအတွက်လည်း ငွေပေးချေနိုင်သည်။

## Products များ စီမံခန့်ခွဲခြင်း

လူအများစုအတွက် Ultimate Multisite ရှိ **Products** tab **(Ultimate Multisite > Products)** ကို ရိုးရာ hosting ပတ်ဝန်းကျင်ရှိ plans များနှင့် တူညီသည်ဟု ယူဆနိုင်သည်။

Ultimate Multisite အတွင်းတွင် Products tab သည် သီးခြား product သို့မဟုတ် service တစ်ခုအတွက် သက်ဆိုင်သော ဖွဲ့စည်းပုံနှင့် ကန့်သတ်ချက်များကို သတ်မှတ်သည်။ ထိုဖွဲ့စည်းပုံများသည် product သို့မဟုတ် service ဖော်ပြချက်၊ စျေးနှုန်း၊ အခွန်များနှင့် ခွင့်ပြုချက်များအထိ ပါဝင်သည်။

ဤအပိုင်းသည် Ultimate Multisite ၏ ဤအရေးကြီးသော အခြေခံအုတ်မြစ်ကို နားလည်စေရန် လမ်းညွှန်ပေးမည်ဖြစ်သည်။

![Products list page](/img/config/products-list.png)

## Products များ ထည့်သွင်းခြင်း

Plan၊ package သို့မဟုတ် service ဖြစ်စေ အသစ်တစ်ခုကို သတ်မှတ်ရန် စတင်ရမည့်နေရာမှာ **Ultimate Multisite > Products > Add Product** ဖြစ်သည်။

![Add Product button](/img/config/product-add-button.png)

Interface တွင် အဓိက အပိုင်းနှစ်ခု ပါဝင်သည်။ ဘယ်ဘက်တွင် product ကို သတ်မှတ်ရာ၌ အထောက်အကူပြုသော tabs များစွာရှိပြီး ညာဘက်တွင် product ၏ အခြေခံစျေးနှုန်း၊ active အခြေအနေနှင့် product ပုံကို သတ်မှတ်ရန် အပိုင်းအနည်းငယ်ရှိသည်။

![Product edit page overview](/img/config/product-edit-full.png)

### ဖော်ပြချက်

အခြေခံ product အချက်အလက်ကို product အမည်နှင့် ဖော်ပြချက်ထည့်သွင်းခြင်းဖြင့် သတ်မှတ်နိုင်သည်။ ဤခွဲခြားသတ်မှတ်ချက်များသည် plan နှင့် စျေးနှုန်းရွေးချယ်ခြင်း၊ invoices များ၊ upgrades များနှင့် အလားတူနေရာများတွင် product အချက်အလက် လိုအပ်သည့်နေရာတိုင်း၌ ပြသသည်။

![Product description section](/img/config/product-description.png)

### စျေးနှုန်း အမျိုးအစား

Interface ၏ ညာဘက်ခြမ်းတွင် အခြေခံစျေးနှုန်းကို သတ်မှတ်နိုင်သည်။

![Pricing and save section](/img/config/product-pricing-save.png)

Ultimate Multisite သည် မတူညီသော စျေးနှုန်းအမျိုးအစား သုံးခုကို ပံ့ပိုးသည်။ **paid** ရွေးချယ်မှုသည် network administrator ကို product ၏ စျေးနှုန်းနှင့် billing အကြိမ်ရေနှင့် ပတ်သက်သော အချက်အလက်များ ဖြည့်သွင်းရန် တောင်းဆိုသည်။

### စျေးနှုန်း

Price component သည် အခြေခံ product စျေးနှုန်းနှင့် billing interval ကို သတ်မှတ်သည်။

![Pricing and save section](/img/config/product-pricing-save.png)

ဥပမာအားဖြင့် $29.99 စျေးနှုန်းဖြင့် 1 month setting သည် လစဉ် $29.99 ကောက်ခံမည်ဖြစ်သည်။ အလားတူပင် $89.97 စျေးနှုန်းဖြင့် 3 months setting သည် သုံးလတစ်ကြိမ် ထိုပမာဏကို ကောက်ခံမည်ဖြစ်သည်။

### Billing Cycles

Billing cycles အပိုင်းသည် အထက်ဖော်ပြပါ billing interval ၏ အကြိမ်ရေကို သတ်မှတ်ပြီး ယေဘုယျအားဖြင့် contracts သို့မဟုတ် ကာလသတ်မှတ်ချက်များ၏ အခြေအနေတွင် နားလည်သည်။

![Pricing and save section](/img/config/product-pricing-save.png)

ဥပမာအားဖြင့်၊ $29.99 product စျေးနှုန်းဖြင့် 1 month interval နှင့် 12 billing cycles သည် လာမည့် ၁၂ လအတွင်း product အတွက် လစဉ် $29.99 ကောက်ခံမည်ဖြစ်သည်။ တစ်နည်းအားဖြင့်၊ ထိုသို့သော setting သည် ၁၂ လအတွက် လစဉ် $29.99 ကာလသတ်မှတ်ထားသော စျေးနှုန်းကို တည်ဆောက်ပြီး ထို့နောက် billing ရပ်တန့်မည်ဖြစ်သည်။

### Trial ကာလ

Offer trial toggle ကို ဖွင့်ခြင်းဖြင့် network administrator အား product အတွက် trial ကာလကို သတ်မှတ်ခွင့်ပြုသည်။

![Pricing and save section](/img/config/product-pricing-save.png)

Trial ကာလအတွင်း ဖောက်သည်များသည် product ကို အခမဲ့အသုံးပြုနိုင်ပြီး trial ကာလ ကုန်ဆုံးသည်အထိ ငွေကောက်ခံမည် မဟုတ်ပါ။

### Setup Fee

သင့် plan တွင် setup fee လည်း ထည့်သွင်းနိုင်သည်။

![Pricing and save section](/img/config/product-pricing-save.png)

ဆိုလိုသည်မှာ သင့်ဖောက်သည်သည် ပထမဆုံး ငွေကောက်ခံချိန်တွင် (plan စျေးနှုန်းအပြင်) ဤအပိုင်းတွင် သင်သတ်မှတ်ထားသော fee နှင့်ညီမျှသော ပမာဏပိုကို ပေးချေရမည်ဖြစ်သည်။

### Active

Active toggle သည် ဖောက်သည်အသစ်များ စာရင်းသွင်းရန်အတွက် product ရရှိနိုင်မှုကို အထိရောက်ဆုံး သတ်မှတ်သည်။

![Active toggle](/img/config/product-active.png)

ဤ plan တွင် ရှိပြီးသား ဖောက်သည်များရှိနေချိန် toggle ကို disabled အခြေအနေသို့ သတ်မှတ်ခြင်းသည် plan ကို အနာဂတ် စာရင်းသွင်းမှုများမှ ဖယ်ရှားပြီး grandfathers အဖြစ် ထားရှိသည်။ **plan ပေါ်ရှိ ရှိပြီးသား ဖောက်သည်များသည် ငွေကောက်ခံခြင်း ဆက်လက်ခံရမည်** ဖြစ်ပြီး plan အသစ်သို့ ပြောင်းရွှေ့သည်အထိ သို့မဟုတ် plan မှ ဖယ်ရှားသည်အထိ ဖြစ်သည်။

### Product ပုံ

**Upload Image** ခလုတ်သည် network administrator အား product ပုံတစ်ခုကို ရွေးချယ်ရန် သို့မဟုတ် upload လုပ်ရန် media library ကို အသုံးပြုခွင့်ပြုသည်။

![Product image section](/img/config/product-image.png)

### ဖျက်ခြင်း

**Delete Product** ခလုတ်သည် product ကို system မှ ဖျက်ပစ်သည်။ Product ကို publish လုပ်ပြီးသည်နှင့် ၎င်းပေါ်လာသည်။

![Delete product section](/img/config/product-delete.png)

အခြား ဖျက်ခြင်းများနှင့် မတူဘဲ product ကို trash အခြေအနေတွင် မထားရှိပါ။ ထို့ကြောင့် ဖျက်ပစ်လိုက်သည်နှင့် လုပ်ဆောင်ချက်ကို ပြန်လည်ရယူ၍ မရနိုင်ပါ။

### Product Options

အခြေခံအဆင့် product အချက်အလက်များ သတ်မှတ်ပြီးသည်နှင့် product options များသည် network administrator အား product ၏ သီးခြားလက္ခဏာများကို ထပ်မံသတ်မှတ်ရာတွင် အထောက်အကူပြုသည်။

#### General

**General** tab သည် အခြား product-specific tabs များတွင် မသက်ဆိုင်သော product ၏ ယေဘုယျ လက္ခဏာများကို သတ်မှတ်သည်။

![General tab](/img/config/product-general-tab.png)

**product slug** သည် URLs နှင့် Ultimate Multisite ၏ အခြားနေရာများတွင် product ကို ခွဲခြားသတ်မှတ်သည့် slug ကို သတ်မှတ်သည်၊ ၎င်း၏ အဓိပ္ပာယ်မှာ ရှင်းလင်းပါသည်။

Ultimate Multisite သည် Plan၊ Package နှင့် Service ဟူ၍ product အမျိုးအစားများစွာကို ပံ့ပိုးသည်။ **Product Options** tabs များသည် သတ်မှတ်ထားသော product အမျိုးအစားပေါ် မူတည်၍ dynamically ချိန်ညှိသည်။

**Customer Role** သည် site ဖန်တီးသည့်အခါ ဖောက်သည်ကို သတ်မှတ်ပေးသော role ကို သတ်မှတ်သည်။ များသောအားဖြင့် network administrators အများစုအတွက် ၎င်းသည် Ultimate Multisite default သို့မဟုတ် Administrator ဖြစ်လိမ့်မည်။ Ultimate Multisite default role ကို **Ultimate Multisite > Settings > Login & Registration** တွင် သတ်မှတ်နိုင်သည်။

![Customer role settings](/img/config/product-customer-role-settings.png)

#### Up & Downgrades

ဤ tab သည် ဖောက်သည်တစ်ဦး၏ သီးခြား tier အတွင်း ရရှိနိုင်သော upgrade နှင့် downgrade လမ်းကြောင်းများကို သတ်မှတ်သည်။

ဤသဘောတရားကို နားလည်ရန် Ultimate Multisite installation တစ်ခုသည် ၎င်း၏ ဖောက်သည်များအား learning management solutions များ ပေးဆောင်သည့် ဥပမာတစ်ခုကို စဉ်းစားကြည့်ပါ။ ၎င်းကို အောင်မြင်စေရန် plans သုံးခု (Basic၊ Plus နှင့် Premium) ကို သတ်မှတ်ပြီး plan တစ်ခုစီအတွက် သီးခြား plugins များကို activate လုပ်သည် (plugins များ activate လုပ်နည်းအတွက် ဤအပိုင်းတွင် နောက်ပိုင်း ညွှန်ကြားချက်များ ကြည့်ပါ)။

Ultimate Multisite installation သည် business websites သို့မဟုတ် eCommerce websites များကိုလည်း ဝန်ဆောင်မှုပေးပါက ထို plans များသည် မတူညီသော plugins များ install နှင့် activate လုပ်ရန် လိုအပ်နိုင်သည်။

ဤအတိုင်းအတာအထိ eLearning ဖောက်သည်များကို eCommerce plans များသို့ ပြောင်းခွင့်ပြုခြင်းသည် မလိုလားအပ်ပြီး ပြဿနာရှိနိုင်သည်၊ အဘယ်ကြောင့်ဆိုသော် ဤ plans များ၊ စျေးနှုန်းများနှင့် ကန့်သတ်ချက်များသည် သင့်လျော်မှု မရှိနိုင်သောကြောင့် ဖြစ်သည်။

ထို့ကြောင့် ဖောက်သည်၏ လမ်းကြောင်းကို ကန့်သတ်ရန်နှင့် ပြဿနာများ မဖြစ်ပေါ်စေရန် network administrator သည် plan group တစ်ခုကို သတ်မှတ်နိုင်ပြီး ထို group အတွင်း ဖောက်သည် ပြောင်းရွှေ့နိုင်သော plans များကို သတ်မှတ်နိုင်သည်။

![Up and Downgrades tab](/img/config/product-upgrades.png)

Plan group တစ်ခုကို သတ်မှတ်ရန် **plan group** list အတွင်း compatible plans များကို သတ်မှတ်ပါ။ **product order** သည် plans များကို အနိမ့်ဆုံးမှ အမြင့်ဆုံးအထိ မည်သို့ စီစဉ်ပြီး ပြသမည်ကို သတ်မှတ်သည်။

Ultimate Multisite တွင် **order bump** feature လည်း ပါဝင်ပြီး သင့်လျော်သော add-on products နှင့် services များကို plans များတွင် ထည့်နိုင်သည်။ ၎င်းတို့ကို checkout တွင် သို့မဟုတ် upgrade ပြုလုပ်စဉ် plans များတွင် ထည့်နိုင်သော အပိုပစ္စည်းများအဖြစ် ဖောက်သည်ထံ ကမ်းလှမ်းသည်။

#### Price Variations

Price variations သည် network administrator အား ကာလပေါ် မူတည်၍ အခြားရွေးချယ်နိုင်သော စျေးနှုန်း tiers များ သတ်မှတ်ခွင့်ပြုသည်။ ဤ setting သည် 3 months၊ 6 months သို့မဟုတ် annual pricing tiers များ သတ်မှတ်ခြင်း သို့မဟုတ် use case ပေါ် မူတည်၍ အခြားမည်သည့် ကာလနှင့် အကြိမ်ရေမဆို သတ်မှတ်နိုင်စေသည်။

![Price Variations tab](/img/config/product-price-variations.png)

Price variations များ တည်ဆောက်ရန် **enable price variations** toggle ကို active သို့ သတ်မှတ်ပြီး **Add new Price Variation** ခလုတ်ကို နှိပ်ပါ။

Variation တစ်ခုထည့်ရန် variation ၏ duration၊ period နှင့် price ကို သတ်မှတ်ပါ။ ခလုတ်ကို ထပ်နှိပ်ခြင်းဖြင့် နောက်ထပ် variations များ ထည့်နိုင်သည်။

#### Taxes

![Taxes tab](/img/config/product-taxes.png)

#### Site Templates

အခြေခံအားဖြင့် site templates များသည် ဖောက်သည်၏ subscription စတင်သည့်အခါ ၎င်းတို့၏ site သို့ clone လုပ်သော ပြည့်စုံသည့် WordPress websites များဖြစ်သည်။

![Site Templates tab](/img/config/product-site-templates.png)

Network administrator သည် template site ကို activate နှင့် configure လုပ်ထားသော themes၊ plugins နှင့် content များဖြင့် ပုံမှန် WordPress site တစ်ခုအဖြစ် ဖန်တီးပြီး configure လုပ်သည်။ Template site ကို ဖောက်သည်ထံ အတိအကျ clone လုပ်သည်။

ဤ tab သည် network administrator အား subscription အသစ်တစ်ခုတွင် site templates များ၏ အပြုအမူကို သတ်မှတ်ခွင့်ပြုသည်။ Site templates များ အသုံးပြုရန် **allow site templates** toggle ကို active အခြေအနေသို့ သတ်မှတ်ပါ။

**site template selection mode** သည် subscription လုပ်ငန်းစဉ်အတွင်း site templates များ၏ အပြုအမူကို သတ်မှတ်သည်။

**Default** setting သည် checkout form ရှိ အဆင့်များကို လိုက်နာသည်။ Network administrator သည် checkout process တွင် template selection step ကို သတ်မှတ်ထားပြီး ထို step ကို templates များဖြင့် သတ်မှတ်ထားပါက ဤ setting သည် checkout step တွင် တည်ဆောက်ထားသော ညွှန်ကြားချက်များကို လေးစားလိုက်နာမည်ဖြစ်သည်။

**Assign Site Template** သတ်မှတ်ခြင်းသည် သတ်မှတ်ထားသော template ၏ ရွေးချယ်မှုကို အတင်းအကျပ် ပြုလုပ်သည်။ ထို့ကြောင့် checkout process ရှိ မည်သည့် template selection steps မဆို ဖယ်ရှားခံရသည်။

နောက်ဆုံးအနေဖြင့် **Choose Available Site Templates** သည် checkout step တွင် သတ်မှတ်ထားသော templates များကို ဤ setting တွင် ရွေးချယ်ထားသော templates များဖြင့် အစားထိုးသည်။ ဖောက်သည်ရွေးချယ်ရာတွင် အထောက်အကူပြုရန် pre-selected template တစ်ခုလည်း သတ်မှတ်နိုင်သည်။

နောက်ဆုံးအနေဖြင့် network administrator သည် checkout steps များတွင် template selection ဖြစ်စေလိုပါက '_default_' setting သည် လုံလောက်ပါမည်။ တစ်နည်းအားဖြင့် template selection ကို ဖယ်ရှားပြီး lock ချကာ ရွေးချယ်မှုကို plan settings သို့ လွှဲအပ်လိုပါက '_assign new template_' သို့မဟုတ် '_choose available site templates_' ရွေးချယ်မှုများသည် သင့်လျော်နိုင်သည်။

#### Sites

**Sites** tab သည် Ultimate Multisite ၏ limitations functionality ၏ အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။

![Sites tab](/img/config/product-sites.png)

ဤ setting သည် ဖောက်သည်တစ်ဦး ၎င်းတို့၏ membership အောက်တွင် ဖန်တီးနိုင်သော sites အများဆုံး အရေအတွက်ကို သတ်မှတ်သည်။

Limitation ကို ဖွင့်ရန် **limit sites** toggle ကို active အခြေအနေသို့ သတ်မှတ်ပြီး **site allowance** field တွင် sites အများဆုံး အရေအတွက်ကို သတ်မှတ်ပါ။

#### Visits

**Visits** tab သည် Ultimate Multisite ၏ limitations system ၏ နောက်ထပ်အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။ ဤ setting သည် ဖောက်သည်၏ site သို့ unique visitors များကို စာရင်းပြုစုခြင်းနှင့် ထို့နောက် throttling ပြုလုပ်ခြင်းကို ခွင့်ပြုသည်။

![Visits tab](/img/config/product-visits.png)

Marketing ရှုထောင့်မှ network administrators များသည် ကန့်သတ်ချက်များ ပြည့်သွားသည့်အခါ ဖောက်သည်များကို ၎င်းတို့၏ plan ကို upgrade လုပ်ရန် အားပေးသည့် နည်းလမ်းအဖြစ် ဤ setting ကို အသုံးပြုနိုင်သည်။ ဤ setting သည် system resources များ ထိန်းသိမ်းရန် sites များသို့ လွန်ကဲသော traffic ကို ထိန်းချုပ်ပြီး ကာကွယ်ရာတွင်လည်း network administrator ကို အထောက်အကူပြုနိုင်သည်။

ဤ feature ကို အသုံးပြုရန် **limit unique visits** toggle ကို active အခြေအနေသို့ သတ်မှတ်ပြီး **unique visits quota** field တွင် unique visitors အများဆုံး အရေအတွက်ကို သတ်မှတ်ပါ။

ဤကန့်သတ်ချက် ပြည့်သွားသည်နှင့် Ultimate Multisite သည် ကန့်သတ်ချက်များ ကျော်လွန်သွားကြောင်း ညွှန်ပြသော message တစ်ခု ပြသမည်ဖြစ်ပြီး ဖောက်သည်၏ site ကို ဆက်လက် serve လုပ်တော့မည် မဟုတ်ပါ။

#### Users

Ultimate Multisite ၏ 'Users' limitations သည် network administrator အား ဖန်တီးနိုင်ပြီး roles များသို့ assign လုပ်နိုင်သော users အရေအတွက်ကို ကန့်သတ်ခွင့်ပြုသည်။

![Users tab](/img/config/product-users.png)

Limitations feature ကို ဖွင့်ရန် **limit user** toggle ကို ညာဘက်သို့ ပွတ်ဆွဲ၍ active အခြေအနေသို့ သတ်မှတ်ပါ။

ထို့နောက် ကန့်သတ်လိုသော role တစ်ခုစီအတွက် ၎င်း၏ဘေးရှိ toggle ကို active အခြေအနေသို့ သတ်မှတ်ပြီး သင့်လျော်သော field တွင် အများဆုံး upper limit ကို သတ်မှတ်ပါ။

**Post Types** tab သည် network administrator အား WordPress အတွင်းရှိ post types များ၏ ကျယ်ပြန့်သော array တွင် granular limits များ သတ်မှတ်ခွင့်ပြုသည်။

WordPress ၏ ဖွဲ့စည်းပုံကြောင့် posts နှင့် post types များသည် ၎င်း၏ core functionality ၏ အရေးပါသော အစိတ်အပိုင်းတစ်ခုဖြစ်ပြီး Ultimate Multisite ၏ limitations system သည် network administrator အား limits များ တည်ဆောက်ပြီး ထိန်းသိမ်းရာတွင် အထောက်အကူပြုရန် ဒီဇိုင်းထုတ်ထားသည်။

ထို့နောက် ကန့်သတ်လိုသော post type တစ်ခုစီအတွက် ညာဘက်သို့ ပွတ်ဆွဲ၍ toggle ဖွင့်ပြီး သင့်လျော်သော field တွင် အများဆုံး upper limit ကို သတ်မှတ်ပါ။

#### Disk Space

**Disk Space** tab သည် network administrators များအား ဖောက်သည်များ သုံးစွဲသော space ကို ကန့်သတ်ခွင့်ပြုသည်။

![Disk Space tab](/img/config/product-disk-space.png)

ပုံမှန်အားဖြင့် WordPress multisite တွင် core files များကို sites အားလုံးကြား မျှဝေပြီး media files နှင့် uploads များအတွက် သီးခြား directories များ ဖန်တီးထားကာ ဤ settings နှင့် limitations များသည် ၎င်းတို့အတွက် သက်ဆိုင်သည်။

Disk usage limitation ကို ဖွင့်ရန် **limit disk size per site** toggle ကို ညာဘက်သို့ ပွတ်ဆွဲ၍ active အခြေအနေသို့ သတ်မှတ်ပါ။

ထို့နောက် **disk space allowance** field တွင် megabytes ဖြင့် အများဆုံး upper limit ကို သတ်မှတ်ပါ။

#### Custom Domain

ဤ option ကို toggle ဖွင့်ခြင်းဖြင့် ဤ plan တွင် သီးခြားအနေဖြင့် custom domains များ ခွင့်ပြုနိုင်သည်။

![Custom Domains tab](/img/config/product-custom-domains.png)

#### Themes

Product options အတွင်းရှိ **Themes** tab သည် network administrator အား ဖောက်သည်များ ရွေးချယ်ရန် themes များကို ရရှိနိုင်စေခြင်းနှင့် theme ၏ state ကို ရွေးချယ်အရ အတင်းအကျပ် သတ်မှတ်ခြင်းကို ခွင့်ပြုသည်။

![Themes tab](/img/config/product-themes.png)

_**မှတ်ချက်- Themes များကို ဖောက်သည်များ ရရှိနိုင်စေရန် ၎င်းတို့ကို network administrator က network enabled ပြုလုပ်ထားရမည်။**_

![Network themes page](/img/config/product-themes-network-enabled.png)

**visibility** option သည် ဤ theme ကို ဖောက်သည် ၎င်းတို့၏ site အတွင်း **Appearance > Themes** tab ကို ကြည့်သည့်အခါ မြင်ရမည်/မမြင်ရမည်ကို သတ်မှတ်သည်။ ဤ option ကို **Hidden** သို့ သတ်မှတ်ခြင်းသည် theme ကို view မှ ဖယ်ရှားပြီး ရွေးချယ်ပြီး activate လုပ်နိုင်စွမ်းကို ကန့်သတ်သည်။

![Themes tab](/img/config/product-themes.png)

**behavior** selection သည် network administrator အား ဖောက်သည် site ဖန်တီးသည့်အခါ theme ၏ state ကို သတ်မှတ်ခွင့်ပြုသည်။

**Available** state တွင် theme ကို ဖောက်သည် ကိုယ်တိုင် activate လုပ်ရန် ရရှိနိုင်သည်။ ဆန့်ကျင်ဘက်အနေဖြင့် **Not Available** state သည် ဖောက်သည်ထံမှ theme ကို activate လုပ်နိုင်စွမ်းကို ဖယ်ရှားသည်။ နောက်ဆုံးအနေဖြင့် **Force Activate** option သည် theme ၏ ရွေးချယ်မှုနှင့် activation ကို အတင်းအကျပ်ပြုလုပ်ပြီး site ဖန်တီးသည့်အခါ default အဖြစ် သတ်မှတ်သည်။

#### Plugins

Themes tab နှင့် အလားတူပင် Ultimate Multisite သည် network administrator အား ဖောက်သည်များအတွက် plugins များ၏ visibility နှင့် site အသစ်ဖန်တီးသည့်အခါ ၎င်းတို့၏ state ကို သတ်မှတ်ခွင့်ပြုသည်။

![Plugins tab](/img/config/product-plugins.png)

**visibility** drop-down သည် plugin ကို ဖောက်သည် ၎င်းတို့၏ site တွင် Plugins menu option မှတစ်ဆင့် ကြည့်သည့်အခါ မြင်ရမည် သို့မဟုတ် hidden ဖြစ်မည်ကို ခွင့်ပြုသည်။

Network administrator သည် behavior drop-down ရှိ options များကို အသုံးပြု၍ plugins များ၏ အပြုအမူကို ထပ်မံ manipulate လုပ်နိုင်သည်။

![Plugins tab](/img/config/product-plugins.png)

**Default** selection သည် ဖောက်သည်ရွေးချယ်သော site template တွင် သတ်မှတ်ထားသော plugin state ကို လေးစားလိုက်နာသည်။ ထို့ကြောင့် template အတွင်း activate လုပ်ထားသော plugins များသည် template ကို ဖောက်သည်၏ site သို့ clone လုပ်သည့်အခါ activated အဖြစ် ဆက်ရှိမည်ဖြစ်သည်။

**Force Activate** သည် site ဖန်တီးသည့်အခါ plugin ကို active state တွင် ထားရှိပြီး ဆန့်ကျင်ဘက်အနေဖြင့် **Force Inactivate** သည် site ဖန်တီးသည့်အခါ plugin ကို deactivate လုပ်သည်။ ဤအခြေအနေ နှစ်ခုလုံးတွင် plugin ၏ state ကို ဖောက်သည်က ၎င်းတို့၏ WordPress Plugins menu မှတစ်ဆင့် ကိုယ်တိုင် ပြောင်းလဲနိုင်သည်။

**Force Activate & Lock** setting သည် အလားတူ အလုပ်လုပ်သော်လည်း plugin state ကို ဖောက်သည်က ပြောင်းလဲခြင်း မပြုနိုင်အောင် ကာကွယ်သည်။ ထို့ကြောင့် Force Activate and Lock setting သည် plugin ကို active state သို့ အတင်းအကျပ်သတ်မှတ်ပြီး ဖောက်သည်က deactivate လုပ်ခြင်းမှ ကာကွယ်သည်။ အလားတူပင် **Force Inactivate & Lock** setting သည် plugin ကို inactive state သို့ အတင်းအကျပ်သတ်မှတ်ပြီး user က plugin ကို activate လုပ်ခြင်းမှ ကာကွယ်သည်။

Network administrator သည် Force Activate & Lock နှင့် Force Inactivate & Lock settings များကို site templates များနှင့် တွဲဖက်၍ စဉ်းစားလိုနိုင်သည်၊ အဘယ်ကြောင့်ဆိုသော် templates အတွင်းရှိ plugins နှင့် plugin states များသည် ဤ settings များ ရွေးချယ်ပါက သက်ရောက်မှုခံရနိုင်သောကြောင့် ဖြစ်သည်။

#### Reset Limitations

**Reset Limitations** tab သည် product တွင် သတ်မှတ်ထားသော custom limits အားလုံးကို reset ပြုလုပ်သည်။ Limitations များ reset လုပ်ရန် **reset limitations** ခလုတ်ကို နှိပ်ပါ။

![Reset Limitations tab](/img/config/product-reset-limitations.png)

Action ကို အတည်ပြုရန် **confirm reset** toggle ကို ညာဘက်သို့ active အခြေအနေသို့ ပွတ်ဆွဲပြီး **reset limitations** ခလုတ်ကို နှိပ်ပါ။

![Reset Limitations tab](/img/config/product-reset-limitations.png)
