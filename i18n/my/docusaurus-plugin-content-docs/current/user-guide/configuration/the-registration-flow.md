# မှတ်ပုံတင်ခြင်း အဆင့်ဆင့် (v2)

_**အရေးကြီးသော မှတ်ချက်- ဤဆောင်းပါးသည် Ultimate Multisite ဗားရှင်း 2.x ကို ရည်ညွှန်းပါသည်။**_

သုံးစွဲသူများသည် သင့် network တွင် နည်းလမ်းအမျိုးမျိုးဖြင့် မှတ်ပုံတင်နိုင်ပါသည်။ သင့်မှတ်ပုံတင်ဖောင် သို့မဟုတ် ကြိုတင်ရွေးချယ်ထားသော plan သို့ ပေးပို့နိုင်သည့် link ကို အသုံးပြုနိုင်ပါသည်။ ဤနေရာတွင် သင့်ဖောက်သည်များသည် ရရှိနိုင်သော လမ်းကြောင်းများကို အသုံးပြု၍ သင့် network တွင် မည်သို့မှတ်ပုံတင်နိုင်သည်နှင့် မှတ်ပုံတင်ပြီးနောက် ဘာဖြစ်သည်ကို ပြသပေးပါမည်။

## မှတ်ပုံတင်ဖောင် အသုံးပြုခြင်း-

ဤသည်မှာ ပုံမှန်မှတ်ပုံတင်ခြင်း လုပ်ငန်းစဉ်ဖြစ်သည်။ **checkout form** ပါရှိသော မှတ်ပုံတင်စာမျက်နှာကို ဖန်တီးပြီး သင့်ဖောက်သည်များသည် ဤနေရာသို့ သွား၍ သင့် network တွင် မှတ်ပုံတင်ကာ plan တစ်ခု စာရင်းသွင်းနိုင်ပါသည်။ မှတ်ပုံတင်စာမျက်နှာ အများအပြားထားရှိနိုင်ပြီး လိုအပ်ပါက စာမျက်နှာတစ်ခုစီတွင် မတူညီသော မှတ်ပုံတင်ဖောင်များ ထည့်သွင်းနိုင်ပါသည်။

မှတ်ပုံတင်ခြင်းအတွက် မူလစာမျက်နှာမှာ [_**yourdomain.com/register**_](http://yourdomain.com/register) ဖြစ်သော်လည်း **Ultimate Multisite > Settings > Login & Registration > Default Registration Page** တွင် အချိန်မရွေး ပြောင်းလဲနိုင်ပါသည်။

သုံးစွဲသူတစ်ဦးသည် သင့်မှတ်ပုံတင်စာမျက်နှာသို့ ရောက်ရှိသောအခါ (များသောအားဖြင့် **Sign in** သို့မဟုတ် **Buy now** ခလုတ်ကို နှိပ်၍)၊ သင့်မှတ်ပုံတင်ဖောင်ကို မြင်ရပါလိမ့်မည်။

![မှတ်ပုံတင်စာမျက်နှာတွင် ပြသထားသော မှတ်ပုံတင်ဖောင်](/img/config/checkout-forms-list.png)

သူတို့လုပ်ဆောင်ရမည်မှာ မဖြစ်မနေ ဖြည့်သွင်းရမည့် အကွက်များအားလုံး - email၊ username၊ password စသည်တို့ - ကို ဖြည့်ပြီး plan အတွက် ငွေပေးချေရန် သို့မဟုတ် အခမဲ့ plan သို့မဟုတ် ငွေပေးချေမှု အချက်အလက်မလိုအပ်သော trial period ပါရှိသော အခပေး plan အတွက် မှတ်ပုံတင်ပါက သူတို့၏ email လိပ်စာကို အတည်ပြုရန်သာ ဖြစ်ပါသည်။

"Thank you" စာမျက်နှာတွင်၊ သူတို့၏ email လိပ်စာကို အတည်ပြုရန် လိုအပ်သည် သို့မဟုတ် သူတို့၏ website အသက်ဝင်ပြီး စတင်အသုံးပြုနိုင်ပြီဟု ပြောပြသည့် မက်ဆေ့ချ်ကို မြင်ရပါလိမ့်မည်။

![မှတ်ပုံတင်ပြီးနောက် Thank You စာမျက်နှာ](/img/config/checkout-form-editor.png)

email လိပ်စာ အတည်ပြုချက် လိုအပ်ပါက၊ သူတို့၏ email inbox သို့သွား၍ အတည်ပြုခြင်း link ကို နှိပ်ရပါမည်။ သူတို့၏ email လိပ်စာကို မအတည်ပြုပါက website အသက်မဝင်ပါ။

အခပေး plan တွင် မှတ်ပုံတင်ထားပါက သို့မဟုတ် သင့် network တွင် email အတည်ပြုခြင်း မဖြစ်မနေ မလိုအပ်ပါက၊ checkout ပြီးပြီးချင်း သူတို့၏ website အသက်ဝင်သွားပြီး dashboard သို့ sign in ရန် link ကို ပြသပါလိမ့်မည်။

![Site အသက်ဝင်ပြီး dashboard သို့ sign in ရန် link](/img/config/checkout-form-editor.png)

## မျှဝေနိုင်သော Link အသုံးပြုခြင်း-

မျှဝေနိုင်သော link အသုံးပြု၍ မှတ်ပုံတင်ခြင်း လုပ်ငန်းစဉ်သည် မှတ်ပုံတင်ဖောင်နှင့် အခြေခံအားဖြင့် တူညီပါသည်။ ကွာခြားချက်မှာ မျှဝေနိုင်သော link ကို အသုံးပြုခြင်းဖြင့် သင့်ဖောက်သည်များသည် checkout form တွင် product သို့မဟုတ် website template ကြိုတင်ရွေးချယ်ထားနိုင်သည် (Pre-selecting products and templates via URL parameters အပိုင်းကို ကြည့်ပါ) သို့မဟုတ် coupon code ထည့်သွင်းထားနိုင်သည် (Using URL Parameters အပိုင်းကို ကြည့်ပါ)။

မှတ်ပုံတင်ခြင်း လုပ်ငန်းစဉ်သည် အတူတူပင်ဖြစ်သည်- သူတို့၏ အမည်၊ username၊ email လိပ်စာ၊ website အမည်နှင့် ခေါင်းစဉ် စသည်တို့ကို ဖြည့်ရပါမည်... သို့သော် plan သို့မဟုတ် site template ကို သူတို့အတွက် ကြိုတင်ရွေးချယ်ပေးထားပြီး ဖြစ်ပါလိမ့်မည်။

### Manual Payments အသုံးပြု၍ မှတ်ပုံတင်ခြင်း-

PayPal၊ Stripe သို့မဟုတ် Ultimate Multisite နှင့် ၎င်း၏ add-on integrations များမှ ပေးသော အခြား payment gateway များကို အသုံးပြုလိုခြင်း မရှိပါက၊ သင့်ဖောက်သည်များအတွက် manual payments ကို အသုံးပြုနိုင်ပါသည်။ ဤနည်းဖြင့် သူတို့ သင့် network တွင် မှတ်ပုံတင်ပြီးနောက် သင်နှစ်သက်ရာ payment processor တွင် ပေးချေရန် invoice ထုတ်ပေးနိုင်ပါသည်။

မှတ်ပုံတင်ခြင်း လုပ်ငန်းစဉ်သည် အထက်ပါအတိုင်း အတူတူပင်ဖြစ်သော်လည်း မှတ်ပုံတင်စာမျက်နှာတွင် သင့်ဖောက်သည်များသည် ငွေပေးချေမှု ပြီးမြောက်ရန် နောက်ထပ် ညွှန်ကြားချက်များပါသော email တစ်စောင် ရရှိမည်ဟု ဖော်ပြသည့် မက်ဆေ့ချ်ကို မြင်ရပါလိမ့်မည်။

![မှတ်ပုံတင်စဉ် manual payment မက်ဆေ့ချ်](/img/config/settings-payment-gateways.png)

မှတ်ပုံတင်မှု ပြီးမြောက်သွားသောအခါ၊ သင်သတ်မှတ်ထားသော ငွေပေးချေမှု ညွှန်ကြားချက်များကို မြင်ရပါလိမ့်မည် (သူတို့၏ email တွင်လည်း ရရှိပါမည်)။

![မှတ်ပုံတင်ပြီးနောက် ပြသသော ငွေပေးချေမှု ညွှန်ကြားချက်များ](/img/config/settings-payment-gateways.png)

ငွေပေးချေမှု ညွှန်ကြားချက်များကို **Manual** payment option ကို toggle ဖွင့်ပြီးနောက် **Ultimate Multisite > Settings > Payments** တွင် ပြောင်းလဲနိုင်ပါသည်-

![Manual payment toggle နှင့် payment instructions field](/img/config/settings-payment-gateways.png)

သင့်ဖောက်သည်များ manual payment ပြီးမြောက်ပြီး အတည်ပြုချက် ပေးပို့သောအခါ၊ ဖောက်သည် membership နှင့် website ကို အသက်ဝင်စေရန် **ငွေပေးချေမှုကို ကိုယ်တိုင် အတည်ပြုရပါမည်**။

ဤသို့လုပ်ဆောင်ရန် **Ultimate Multisite > Payments** သို့သွား၍ ဖောက်သည် payment ကို ရှာပါ။ ၎င်းသည် **Pending** status ဖြင့် ဆက်လက်ပြသနေသင့်ပါသည်။

![Pending manual payment ပါရှိသော Payments စာရင်း](/img/admin/payments-list.png)

payment နံပါတ်ကို နှိပ်ပါ၊ ထို့နောက် ၎င်း၏ status ကို **Completed** သို့ ပြောင်းလဲနိုင်ပါလိမ့်မည်။

![Payment အသေးစိတ် စာမျက်နှာ](/img/admin/payments-list.png)

![Payment status ကို Completed သို့ ပြောင်းလဲခြင်း](/img/admin/payments-list.png)

၎င်း၏ status ကို **Completed** သို့ ပြောင်းလဲပြီးနောက်၊ **Activate membership** မက်ဆေ့ချ်ကို မြင်ရပါလိမ့်မည်။ ဤဖောက်သည်နှင့် ဆက်စပ်သော membership နှင့် website ကို အသက်ဝင်စေရန် ဤ option ကို **ဖွင့်ပါ**။ ထို့နောက် **Save Payment** ကို နှိပ်ပါ။

![Activate membership toggle နှင့် Save Payment ခလုတ်](/img/admin/payments-list.png)

သင့်ဖောက်သည်သည် ယခု dashboard နှင့် သူတို့ စာရင်းသွင်းထားသော feature များအားလုံးကို ဝင်ရောက်အသုံးပြုနိုင်ပါပြီ။
