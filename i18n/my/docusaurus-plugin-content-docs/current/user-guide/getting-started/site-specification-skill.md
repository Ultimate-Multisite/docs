# Site Specification Skill

**Site Specification skill** ဆိုတာကတော့ သင့်ဝက်ဘ်ဆိုဒ်ရဲ့ ရည်မှန်းချက်တွေ၊ ပစ်မှတ်ပရိသတ်တွေနဲ့ အမှတ်တံဆိပ် အထောက်အထားတွေကို စနစ်တကျ စုစည်းဖော်ပြနိုင်တဲ့ နည်းလမ်းတစ်ခုပါ။ ဒီအချက်အလက်တွေကို **site_brief** ဆိုတဲ့ မှတ်ဉာဏ်ထဲမှာ သိမ်းဆည်းထားပြီး၊ ဒီမှတ်ဉာဏ်ကို agents တွေက session အသစ်တွေမှာပါ အသုံးပြုကာ အမြဲတမ်း တသမတ်တည်းဖြစ်ပြီး အခြေအနေကို သိနားလည်တဲ့ အကူအညီတွေကို ပေးနိုင်ပါတယ်။

## Site Specification ဆိုတာ ဘာလဲ။

Site Specification ဆိုတာ အောက်ပါအချက်တွေကို စာရွက်စာတမ်းအဖြစ် မှတ်တမ်းတင်ခြင်း ဖြစ်ပါတယ်-

- **Site purpose**: သင့်ဝက်ဘ်ဆိုဒ်က ဘာလုပ်လဲ၊ ဘာကြောင့် ရှိရတာလဲ
- **Target audience**: ဘယ်သူတွေက သင့်ဝက်ဘ်ဆိုဒ်ကို လာကြည့်လဲ၊ သူတို့ ဘာတွေ လိုအပ်လဲ
- **Brand identity**: သင့်ရဲ့ အရောင်အသွေး၊ ပြောဆိုတဲ့ အသံ (tone) နဲ့ မြင်ရတဲ့ ပုံစံ (visual style)
- **Business goals**: သင့်ဝက်ဘ်ဆိုဒ်အတွက် အောင်မြင်မှုဆိုတာ ဘယ်လိုပုံစံမျိုးလဲ
- **Content structure**: သင့်ဝက်ဘ်ဆိုဒ်ကို ဘယ်လို စနစ်တကျ စီစဉ်ထားလဲ

ဒီသတ်မှတ်ချက်တွေဟာ **site_brief** အဖြစ် သင့်ရဲ့ အမြဲတမ်း မှတ်ဉာဏ်တစ်ခု ဖြစ်လာပြီး၊ agents တွေက သင့်ဝက်ဘ်ဆိုဒ်ရဲ့ အခြေအနေကို နားလည်ဖို့ အသုံးပြုကြပါတယ်။

## Site Specification ကို ဘာကြောင့် သုံးသင့်သလဲ။

### Session အသစ်တွေမှာ တသမတ်တည်းဖြစ်မှု

site_brief မရှိရင်၊ session အသစ်တစ်ခု စတိုင်း သင့်ဝက်ဘ်ဆိုဒ်ရဲ့ ရည်ရွယ်ချက်ကို အကြိမ်တိုင်း ပြန်ရှင်းပြနေရပါလိမ့်မယ်။ ဒါပေမဲ့ site_brief ရှိရင်တော့ agents တွေက အောက်ပါအချက်တွေကို ချက်ချင်း နားလည်သွားပါလိမ့်မယ်-

- သင့်ဝက်ဘ်ဆိုဒ်ရဲ့ ရည်မှန်းချက်တွေနဲ့ ပရိသတ်
- သင့်ရဲ့ အမှတ်တံဆိပ် အရောင်တွေနဲ့ အသံ (tone)
- သင့်ရဲ့ အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံ
- သင့်ရဲ့ လုပ်ငန်းဆိုင်ရာ ရည်မှန်းချက်များ

### ပိုမိုကောင်းမွန်တဲ့ အကြံပြုချက်များ

agents တွေက သင့်ရဲ့ site_brief ကို အသုံးပြုပြီး-

- သင့်ဝက်ဘ်ဆိုဒ် ရည်ရွယ်ချက်နဲ့ ကိုက်ညီတဲ့ အင်္ဂါရပ်တွေကို အကြံပြုပေးနိုင်တယ်
- သင့်ရဲ့ ရည်မှန်းချက်တွေနဲ့ ကိုက်ညီတဲ့ အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံတွေကို အကြံပေးနိုင်တယ်
- သင့်ရဲ့ အမှတ်တံဆိပ်နဲ့ ကိုက်ညီတဲ့ ဒီဇိုင်းတွေကို အဆိုပြုနိုင်တယ်
- မကိုက်ညီတဲ့ အင်္ဂါရပ်တွေကို အကြံပြုတာမျိုး ရှောင်ရှားပေးနိုင်တယ်

### အသုံးပြုရလွယ်ကူခြင်း (Faster Onboarding)

အစပြုတဲ့ agents အသစ်တွေ (သို့မဟုတ် session အသစ်မှာရှိတဲ့ agents တွေ) ဟာ ရှင်းလင်းတဲ့ မေးခွန်းတွေ မမေးဘဲ သင့်ရဲ့ site_brief ကို ဖတ်ရုံနဲ့ အမြန်ဆုံး အသုံးပြုနိုင်ပါတယ်။

## Site Specification ကို စတင်ခြင်း

### Theme Builder Onboarding လုပ်နေစဉ်

Site Specification skill ကို **Theme Builder onboarding flow** အတွင်းမှာ အလိုအလျောက် စတင်ပေးပါတယ်။ Setup Assistant agent က မေးခွန်းတွေ မေးပြီး သင့်ရဲ့ site_brief ကို တည်ဆောက်ပေးမှာ ဖြစ်ပါတယ်။

### ကိုယ်တိုင် စတင်ခြင်း (Manual Initiation)

ဘယ်အချိန်မဆို Site Specification ကို စတင်နိုင်ပါတယ်-

```
"Let's define my site specification"
```

ဒါမှမဟုတ်

```
"Help me create a site brief"
```

## Site Specification လုပ်ငန်းစဉ်

### အဆင့် ၁: Site Purpose (ဝက်ဘ်ဆိုဒ် ရည်ရွယ်ချက်)

agent က မေးပါလိမ့်မယ်-

```
What is your site's primary purpose?
- E-commerce store
- Blog or content site
- Portfolio or showcase
- SaaS application
- Community or forum
- Other: [describe]
```

သင်ဟာ အမျိုးအစားတစ်ခုကို ရွေးချယ်နိုင်သလို၊ ကိုယ်ပိုင်ရည်ရွယ်ချက်ကိုလည်း ဖော်ပြနိုင်ပါတယ်။

### အဆင့် ၂: Target Audience (ပစ်မှတ်ပရိသတ်)

```
Who is your primary audience?
- Consumers / general public
- Business professionals
- Developers / technical users
- Students / educators
- Other: [describe]

What are their main needs?
```

### အဆင့် ၃: Brand Identity (အမှတ်တံဆိပ် အထောက်အထား)

```
What are your brand colors?
- Primary color: [color picker or hex code]
- Secondary color: [color picker or hex code]
- Accent color: [optional]

How would you describe your brand tone?
- Professional / corporate
- Creative / artistic
- Playful / casual
- Minimal / modern
- Warm / friendly
```

### အဆင့် ၄: Business Goals (လုပ်ငန်း ရည်မှန်းချက်များ)

```
What does success look like for your site?
- Generate leads
- Sell products
- Build community
- Share knowledge
- Establish authority
- Other: [describe]

What's your primary metric?
- Revenue
- User engagement
- Content reach
- Conversions
- Other
```

### အဆင့် ၅: Content Structure (အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံ)

```
How is your content organized?
- Flat (all content at same level)
- Hierarchical (categories and subcategories)
- Chronological (blog-style)
- Product-based (catalog)
- Other: [describe]

What content types do you use?
- Blog posts
- Product pages
- Case studies
- Documentation
- Videos
- Other
```

## သင့်ရဲ့ site_brief မှတ်ဉာဏ်

Site Specification ကို ပြီးမြောက်ပြီးတဲ့အခါ၊ သင့်ရဲ့ အချက်အလက်တွေဟာ agent ရဲ့ မှတ်ဉာဏ်ထဲမှာ **site_brief** အဖြစ် သိမ်းဆည်းထားပါတယ်။ ဒီမှာက အောက်ပါအချက်တွေ ပါဝင်တဲ့ စနစ်ကျတဲ့ မှတ်တမ်းတစ်ခု ဖြစ်ပါတယ်-

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## သင့်ရဲ့ site_brief ကို ကြည့်ရှုခြင်းနှင့် အပ်ဒိတ်လုပ်ခြင်း

### သင့် site_brief ကို ကြည့်ရန်

agent ကို မေးပါ-

```
"Show me my site brief"
```

ဒါမှမဟုတ်

```
"What do you know about my site?"
```

agent က သင့်ရဲ့ သိမ်းဆည်းထားတဲ့ သတ်မှတ်ချက်ကို ပြသပေးပါလိမ့်မယ်။

### သင့် site_brief ကို အပ်ဒိတ်လုပ်ရန်

သင့်ဝက်ဘ်ဆိုဒ် ပြောင်းလဲလာရင်၊ သင် အပ်ဒိတ်လုပ်နိုင်ပါတယ်-

```
"Update my site brief: we're now targeting B2B customers"
```

ဒါမှမဟုတ်

```
"Refresh my site specification"
```

ဒါက သင့်ရဲ့ လက်ရှိအချက်အလက်တွေကို အစမှတ်အဖြစ် အသုံးပြုပြီး Site Specification skill ကို ပြန်လည် run ပေးမှာ ဖြစ်ပါတယ်။

## Agents များက site_brief ကို ဘယ်လို အသုံးပြုသလဲ

### ဒီဇိုင်း အကြံပြုချက်များ

ဒီဇိုင်း ပြောင်းလဲမှုတွေ တောင်းဆိုတဲ့အခါ၊ agents တွေက သင့်ရဲ့ site_brief ကို ကိုးကားအသုံးပြုပါတယ်-

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### အင်္ဂါရပ် အကြံပြုချက်များ

agents တွေက သင့်ရဲ့ ရည်မှန်းချက်တွေနဲ့ ကိုက်ညီတဲ့ အင်္ဂါရပ်တွေကို အကြံပြုပေးပါတယ်-

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံ

agents တွေက သင့်ရဲ့ ဖွဲ့စည်းပုံအပေါ် အခြေခံပြီး အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံကို အဆိုပြုပါတယ်-

```
You: "How should I organize my blog?"
Agent: "Since you use a product catalog with blog, I'd suggest
organizing posts by product category and creating 'styling guides'
that showcase multiple products together."
```

## အကောင်းဆုံး လုပ်ဆောင်မှုများ (Best Practices)

### အတိအကျ ဖြစ်ပါစေ

"general audience" လို့ ပြောမယ့်အစား၊ သင့်ရဲ့ တကယ့်ပရိသတ်ကို ဖော်ပြပါ-

- ✓ "Women aged 25-45, interested in sustainable fashion"
- ✗ "Everyone"

### ပုံမှန် အပ်ဒိတ်လုပ်ပါ

သင့်ဝက်ဘ်ဆိုဒ် ပြောင်းလဲလာတာနဲ့အမျှ၊ သင့်ရဲ့ site_brief ကို အပ်ဒိတ်လုပ်ပေးပါ-

- ပရိသတ်အသစ်ဆီ ဦးတည်တဲ့အခါ
- ထုတ်ကုန်လိုင်းအသစ်တွေ ထပ်ထည့်တဲ့အခါ
- သင့်ရဲ့ အမှတ်တံဆိပ် အထောက်အထား ပြောင်းလဲတဲ့အခါ
- သင့်ရဲ့ လုပ်ငန်းရည်မှန်းချက်တွေ ပြောင်းလဲတဲ့အခါ

### တသမတ်ကျတဲ့ ဝေါဟာရကို အသုံးပြုပါ

session အသစ်တွေမှာပါ တူညီတဲ့ ဝေါဟာရတွေကို အသုံးပြုပါ-

- ✓ အမြဲတမ်း "sustainable jewelry" လို့ ပြောပါ ( "eco-friendly jewelry" နဲ့ "green products" မပြောပါနဲ့)
- ✓ သင့်ပရိသတ်ကို အမြဲတမ်း တူညီတဲ့ ပုံစံနဲ့ ရည်ညွှန်းပါ

### အခြေအနေကို ထည့်သွင်းပေးပါ

agents တွေက သင့်ရဲ့ ဆုံးဖြတ်ချက်တွေကို နားလည်ဖို့ ကူညီပေးမယ့် နောက်ခံအကြောင်းအရာတွေကို ပေးပါ-

- "We're targeting professionals who value quality over price" (ဈေးနှုန်းထက် အရည်အသွေးကို တန်ဖိုးထားတဲ့ ပညာရှင်တွေကို ကျွန်တော်တို့ ပစ်မှတ်ထားပါတယ်)
- "Our audience is tech-savvy and expects modern design" (ကျွန်တော်တို့ ပရိသတ်က နည်းပညာကို ကျွမ်းကျင်ပြီး ခေတ်မီဒီဇိုင်းကို မျှော်လင့်ပါတယ်)
- "We're a bootstrapped startup, so we need cost-effective solutions" (ကျွန်တော်တို့က ကိုယ်ပိုင်ငွေနဲ့ စတင်တဲ့ startup ဖြစ်လို့ ကုန်ကျစရိတ်သက်သာတဲ့ ဖြေရှင်းနည်းတွေ လိုပါတယ်)

## Theme Builder Onboarding နဲ့ ဆက်စပ်မှု

Site Specification skill ကို **Theme Builder onboarding flow** ထဲမှာ ပေါင်းစပ်ထားပါတယ်။ Onboarding ကို ပြီးမြောက်တဲ့အခါ၊ သင်ပေးခဲ့တဲ့ အချက်အလက်တွေနဲ့ site_brief ကို အလိုအလျောက် ဖန်တီးပေးမှာ ဖြစ်ပါတယ်။

လိုချင်ရင် Site Specification ကိုလည်း ကိုယ်ပိုင်လုပ်ဆောင်နိုင်ပါတယ်-

- အစပိုင်း setup လုပ်ပြီးနောက် သင့်ရဲ့ သတ်မှတ်ချက်ကို ပိုမိုကောင်းမွန်အောင် ပြုပြင်လိုတဲ့အခါ
- သင့်ဝက်ဘ်ဆိုဒ် ပြောင်းလဲလာတာနဲ့အမျှ site brief ကို အပ်ဒိတ်လုပ်လိုတဲ့အခါ
- Theme Builder ကို စတင်မယ့် အရင် အချိန်မှာ အသေးစိတ် သတ်မှတ်ချက်တစ်ခု ဖန်တီးလိုတဲ့အခါ

## ပြဿနာဖြေရှင်းခြင်း (Troubleshooting)

**ကျွန်တော့် site_brief ကို မသုံးဘူး**
- agent မှာ memory ကို ဝင်ခွင့်ရှိမရှိ စစ်ဆေးပါ
- agent ကို "recall my site brief" လို့ မေးခိုင်းပါ
- သင့်ရဲ့ settings မှာ memory ကို ဖွင့်ထားခြင်း ရှိ၊ မရှိ စစ်ဆေးပါ

**site_brief အသစ်နဲ့ စတင်ချင်တယ်**
- agent ကို "Clear my site brief and start fresh" လို့ မေးပါ
- ပြီးရင် Site Specification ကို ပြန် run ပါ

**agent က ကျွန်တော့် site_brief နဲ့ မကိုက်ညီတဲ့ အကြံပြုချက်တွေ ပေးနေတယ်**
- agent ကို "review my site brief" လို့ မေးခိုင်းပါ
- သင့်ရဲ့ site_brief ဟာ ခေတ်နောက်ကျနေရင် အပ်ဒိတ်လုပ်ပါ
- သင့်ရဲ့ တောင်းဆိုမှုတွေမှာ အပိုအခြေအနေ (context) အချက်အလက်တွေ ပေးပါ

## နောက်အဆင့်များ (Next Steps)

သင့်ရဲ့ site specification ကို သတ်မှတ်ပြီးတဲ့အခါ-

1. **Use Theme Builder**: သင့်ရဲ့ site_brief အပေါ် အခြေခံပြီး custom theme တစ်ခု ဖန်တီးပါ
2. **Refine Design**: အသေးစိတ် ဒီဇိုင်းအလုပ်တွေအတွက် Design System Aesthetics skill ကို အသုံးပြုပါ
3. **Plan Content**: အကြောင်းအရာ စီစဉ်ဖွဲ့စည်းပုံ အကြံပြုချက်တွေအတွက် agents တွေကို မေးမြန်းပါ
4. **Build Features**: သင့်ရဲ့ လုပ်ငန်းရည်မှန်းချက်တွေနဲ့ ကိုက်ညီတဲ့ အင်္ဂါရပ်တွေကို တောင်းဆိုပါ
