# Ultimate Multisite လိုအပ်ချက်များ

Ultimate Multisite ကို မဝယ်ခင် ပထမဆုံး မေးလောက်တဲ့ မေးခွန်းက ကျွန်တော်တို့ plugin ကို အသုံးပြုဖို့ ဘာတွေ လိုအပ်လဲဆိုတာပါ။ ကံကောင်းတာက Ultimate Multisite ကို အသုံးပြုဖို့ အခြေခံ လိုအပ်ချက် အနည်းငယ်ပဲ ရှိပါတယ်: **domain name** တစ်ခု၊ ကောင်းမွန်တဲ့ **hosting provider** တစ်ခုနဲ့ WordPress **Multisite** installation တစ်ခုပါ။ ဒါပဲ လိုပါတယ်!

## Domain Name

ပထမဆုံး domain name အကြောင်း ပြောကြရအောင်။

Domain name ရွေးတဲ့အခါ သင့်ရဲ့ အထူးပြု niche က ဘာလဲ၊ Ultimate Multisite နဲ့ ဘာတွေ ဖန်တီးမလဲဆိုတာ စဉ်းစားပါ: Agency တစ်ခု တည်ဆောက်ချင်လား? ဆိုင်ပုံစံ ဝဘ်ဆိုက်တွေ ရောင်းချင်လား? ဒါမှမဟုတ် WordPress web hosting လုပ်ချင်လား? Ultimate Multisite နဲ့ အကုန်လုံး ဖြစ်နိုင်ပါတယ်။

ကျွန်တော်တို့ plugin က subscription ဝန်ဆောင်မှုများ၊ ငွေပေးချေမှု စုဆောင်းခြင်း၊ checkout forms၊ discount vouchers နဲ့ customer ဆက်သွယ်ရေးတွေ ပံ့ပိုးပေးပါတယ်။ WordPress Multisite ကို install လုပ်ခြင်း၊ configure လုပ်ခြင်းနဲ့ ထိန်းသိမ်းခြင်း လုပ်ငန်းအများစုကို Ultimate Multisite က လွယ်ကူအောင် လုပ်ဆောင်ပေးပါတယ်။

ဝဘ်ဆိုက်တွေကို ဝန်ဆောင်မှုအဖြစ် ရောင်းချရင် မည်သည့် niche ကိုမဆို ကျွန်တော်တို့ plugin နဲ့ စီမံခန့်ခွဲနိုင်ပါတယ်! သင့် လုပ်ငန်းအစီအစဉ်နဲ့ ကိုက်ညီတဲ့ domain name ကို ရွေးချယ်ပါ။

## Hosting Provider

သင့် niche နဲ့ လုပ်ငန်းကို ကိုယ်စားပြုတဲ့ ဆွဲဆောင်မှုရှိတဲ့ domain name ရပြီဆိုရင်၊ Ultimate Multisite နဲ့ သင့် network ကို host လုပ်ဖို့ ကောင်းမွန်တဲ့ web hosting ကုမ္ပဏီတစ်ခု ရွေးချယ်ရပါမယ်။ စျေးကွက်ထဲမှာ ကောင်းမွန်တဲ့ ရွေးချယ်စရာတွေ ရှိပါတယ်။ အချို့ hosting providers တွေက Ultimate Multisite နဲ့ ကောင်းကောင်း ပေါင်းစပ်အလုပ်လုပ်တာကြောင့် သင် ဘာတည်ဆောက်မလဲပေါ် မူတည်ပြီး အဆုံးအဖြတ်ပေး အချက်တစ်ခု ဖြစ်နိုင်ပါတယ်။

Ultimate Multisite က မည်သည့် hosting provider မှာမဆို အလုပ်လုပ်သင့်ပါတယ်။ ကွာခြားချက်က _WPMU DEV Hosting_၊ _WP Engine_၊ _Closte_၊ _Cloudways_၊ _Gridpane_ နဲ့ _Runcloud_ လို hosting ကုမ္ပဏီအချို့က domain mapping အတွက် ပိုမိုကျစ်လစ်တဲ့ integration ရှိပါတယ်။ အခြား hosting တွေအတွက် custom domains တွေကို ကိုယ်တိုင် map လုပ်ရနိုင်ပါတယ်။ AutoSSL ကို ပံ့ပိုးတဲ့ hosting provider ရွေးတာလည်း စိတ်ကူးကောင်းပါတယ်။ _Closte_၊ _Runcloud_ နဲ့ _cPanel_ က ပံ့ပိုးပါတယ်။

Server specifications အတွက် တိကျတဲ့ ဖော်မြူလာ မရှိပါဘူး၊ သင် ပေးမယ့် features နဲ့ functions တွေ၊ သင့် network ရဲ့ အရွယ်အစားပေါ် မူတည်ပါလိမ့်မယ်။ Shared hosting က စကောင်းသော အစ ဖြစ်နိုင်ပေမယ့်၊ သင့် network ကြီးထွားလာတာနဲ့အမျှ dedicated VPS တစ်ခု လိုလာနိုင်ပါတယ်။

## WordPress Multisite

အခု WordPress Multisite installation အကြောင်း နည်းနည်း ပြောကြရအောင်။

WordPress Multisite က single installation တစ်ခုမှာ site များပါဝင်တဲ့ network တစ်ခု ရှိနိုင်စေပါတယ်။ ဒါက built-in feature တစ်ခုဖြစ်ပေမယ့် default အားဖြင့် active မဖြစ်ပါဘူး။ သင့် WordPress installation နဲ့ web hosting control panel ထဲမှာ ဒီ feature ကို enable လုပ်ရပါမယ်။

Multisite enabled လုပ်ထားတဲ့ WordPress ရှိခြင်းက **Ultimate Multisite အတွက် အခြေခံ ဖြစ်ပါတယ်**။ Ultimate Multisite ရဲ့ အဓိက အသုံးပြုမှုက users တွေကို ဝဘ်ဆိုက်များကို ဝန်ဆောင်မှုအဖြစ် စီမံခန့်ခွဲရောင်းချရာမှာ ကူညီဖို့ ဖြစ်တာကြောင့်၊ ဒါကို Multisite enabled လုပ်ထားတဲ့ WordPress မှာ လုပ်ရပါမယ်။

သင့် WordPress installation မှာ Multisite feature ကို enable လုပ်ဖို့ ဘာတွေ လုပ်ရမလဲဆိုတာ အဆင့်ဆင့် သင်ပေးတဲ့ **အသေးစိတ် ဆောင်းပါး** ရှိပါတယ်။

WordPress မှာ Multisite enable လုပ်ပြီးတဲ့နောက်၊ Ultimate Multisite ကို install လုပ်ဖို့ အချိန်ရောက်ပါပြီ။ Ultimate Multisite ကို မှန်ကန်စွာ install လုပ်ဖို့ ဒါတွေ ရှိရပါမယ်:

  * WordPress: v5.3+ (အကြံပြုချက်: နောက်ဆုံး stable version)

  * PHP: 7.4.x (> 8.0 ပံ့ပိုးမှု မကြာမီ လာပါမယ်)

  * MySQL: v5+ (အကြံပြုချက်: 5.6၊ သင့် hosting provider မှာ 8.0 မရရှိနိုင်ရင်)

ဒါတွေက Ultimate Multisite အတွက် တစ်ခုတည်းသော software လိုအပ်ချက်များ ဖြစ်ပါတယ်။

Ultimate Multisite က PHP 8.0 နဲ့ အလုပ်လုပ်နိုင်ပေမယ့် PHP 7.4.x နဲ့ run ဖို့ အကြံပြုပါတယ်။

ထို့အပြင်၊ subdomains သို့မဟုတ် subdirectories မဟုတ်ဘဲ **main domains** မှာ install လုပ်ဖို့ အကြံပြုပါတယ်။ Subdomains/subdirectories မှာ အလုပ်လုပ်နိုင်ပေမယ့် အမှားအချို့ ပြနိုင်ပါတယ်။

Ultimate Multisite ကို install လုပ်နည်း အသေးစိတ် လမ်းညွှန်အတွက် **ဒီမှာ နှိပ်ပါ**။
