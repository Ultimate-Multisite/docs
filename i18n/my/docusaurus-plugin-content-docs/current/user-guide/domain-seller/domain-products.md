# ဒိုမိန်ထုတ်ကုန်များနှင့် ဈေးနှုန်းသတ်မှတ်ခြင်း

ဒိုမိန်ထုတ်ကုန်များဆိုသည်မှာ ဈေးနှုန်းသတ်မှတ်ခြင်း၊ TLD များ၊ WHOIS ကိုယ်ရေးအချက်အလက် လျှို့ဝှက်ခြင်းနှင့် ပေးသွင်းသူ ရွေးချယ်မှုတို့ကို ထိန်းချုပ်နိုင်သည့် နည်းလမ်းများ ဖြစ်ပါတယ်။ ဒိုမိန်ထုတ်ကုန်တစ်ခုစီတိုင်းသည် **Domain Settings** tab အပိုတစ်ခုပါဝင်သည့် စံ Multisite Ultimate ထုတ်ကုန်တစ်ခု ဖြစ်ပါတယ်။

## ဒိုမိန်ထုတ်ကုန်တစ်ခု ဖန်တီးခြင်း {#creating-a-domain-product}

၁။ **Network Admin › Ultimate Multisite › Products** သို့ သွားပါ။
၂။ **Add New** ကို နှိပ်ပါ။
၃။ ထုတ်ကုန်အမျိုးအစားကို **Domain** ဟု သတ်မှတ်ပါ။
၄။ **Domain Settings** tab ကို စနစ်တကျ ဖွဲ့စည်းပါ။ (အောက်ပါကို ကြည့်ပါ)
၅။ သိမ်းဆည်းပါ။

ဒိုမိန်ထုတ်ကုန်များသည် ထုတ်ကုန်စာရင်းတွင် ခရမ်းရောင် **Domain** တံဆိပ်ဖြင့် ပေါ်လာမည်ဖြစ်ပြီး **Domain Products** tab ကို အသုံးပြု၍ စစ်ထုတ်နိုင်ပါသည်။

## Domain settings tab {#domain-settings-tab}

### Provider {#provider}

ဤထုတ်ကုန်အတွက် မှတ်ပုံတင်ခြင်းကို မည်သည့် registrar က ကိုင်တွယ်မည်ကို ရွေးချယ်ပါ။ Domain Seller settings တွင် သတ်မှတ်ထားသည့် ကမ္ဘာလုံးဆိုင်ရာ **Default provider** ကို အလိုအလျောက် သတ်မှတ်ထားပါသည်။

### Supported TLDs {#supported-tlds}

အားလုံးအတွက် အသုံးပြုနိုင်သော **catch-all product** တစ်ခု ဖန်တီးရန် နေရာလွတ်ထားပါ။ ၎င်းသည် အခြားထုတ်ကုန်တစ်ခုမှ မကိုက်ညီသည့် TLD အားလုံးအတွက် အကျုံးဝင်ပါသည်။

**TLD-specific product** တစ်ခု ဖန်တီးရန် TLD များကို ကော်မာ (comma) ဖြင့် ခွဲခြားထားသော စာရင်းကို ထည့်သွင်းပါ (ဥပမာ- `.com, .net, .org`)။ ၎င်းသည် ထိုအကန့်အသတ်များအတွက်သာ အကျုံးဝင်ပါသည်။

**ထုတ်ကုန်ကို မည်သို့ကိုက်ညီစေသည်:** ဖောက်သည်တစ်ဦးက ဒိုမိန်တစ်ခုကို ရှာဖွေသည့်အခါ၊ အပိုဆောင်း (addon) က အသင့်တော်ဆုံး ကိုက်ညီသည့် ထုတ်ကုန်ကို ရွေးချယ်ပေးပါသည်။ TLD စာရင်းတွင် `.com` ပါဝင်သည့် ထုတ်ကုန်သည် catch-all ထုတ်ကုန်ထက် ဦးစားပေး အသုံးပြုမည်ဖြစ်သည်။ TLD-specific ထုတ်ကုန် မရှိပါက၊ catch-all ကို အသုံးပြုမည်။ ထုတ်ကုန် မရှိပါက၊ ဒိုမိန်ရှာဖွေမှုကို ပြသမည်မဟုတ်ပါ။

### Markup type {#markup-type}

လက်ကားကုန်ကျစရိတ်မှ သင့်လက်လီဈေးနှုန်းကို မည်သို့တွက်ချက်မည်ကို ဤပုံစံသုံးမျိုးက ထိန်းချုပ်ထားသည်-

| Mode | အလုပ်လုပ်ပုံ |
|---|---|
| **Percentage** | လက်ကားကုန်ကျစရိတ်အပေါ် ရာခိုင်နှုန်းတစ်ခု ထပ်ပေါင်းပေးသည်။ $10 လက်ကားဒိုမိန်အပေါ် ၂၀% markup လုပ်ပါက $12 ရရှိမည်။ |
| **Fixed markup** | သတ်မှတ်ထားသော ဒေါ်လာပမာဏတစ်ခု ထပ်ပေါင်းပေးသည်။ $10 ဒိုမိန်အပေါ် $5 markup လုပ်ပါက $15 ရရှိမည်။ |
| **Fixed price** | လက်ကားကုန်ကျစရိတ်ကို လုံးဝလျစ်လျူရှုသည်။ သင်ထည့်သွင်းသည့် ပမာဏကို အမြဲတမ်း ကောက်ခံမည်။ |

### Introductory pricing {#introductory-pricing}

ပထမနှစ်အတွက် လျှော့စျေးပေးလိုပါက ဤအင်္ဂါရပ်ကို ဖွင့်ပါ။ ပုံမှန် **Renewal price** (နှစ် ၂ နှင့်အထက်) နှင့်အတူ သီးခြား **Introductory price** (နှစ် ၁ အတွက် ဈေးနှုန်း) ကို သတ်မှတ်ပါ။ ဖောက်သည်သည် checkout လုပ်နေစဉ် ဈေးနှုန်းနှစ်ခုလုံးကို မြင်ရမည်ဖြစ်ပြီး သက်တမ်းတိုးချိန်တွင် မည်သို့မျှော်လင့်ရမည်ကို သိရှိစေရန် ဖြစ်သည်။

### WHOIS privacy {#whois-privacy}

ဤထုတ်ကုန်မှတစ်ဆင့် မှတ်ပုံတင်သည့် ဒိုမိန်များအတွက် WHOIS ကိုယ်ရေးအချက်အလက် လျှို့ဝှက်မှုကို ပေးအပ်မည်လားကို ထိန်းချုပ်သည်။

| Setting | Behaviour |
|---|---|
| **Disabled** | WHOIS ကိုယ်ရေးအချက်အလက် လျှို့ဝှက်မှုကို ဘယ်တော့မှ ပေးအပ်ခြင်း သို့မဟုတ် ဖွင့်လှစ်ခြင်း မရှိပါ။ |
| **Always Included** | WHOIS ကိုယ်ရေးအချက်အလက် လျှို့ဝှက်မှုကို မှတ်ပုံတင်ချိန်တွင် အခမဲ့ အလိုအလျောက် ဖွင့်ပေးပါသည်။ |
| **Customer Choice** | checkout လုပ်နေစဉ် checkbox တစ်ခု ပေါ်လာမည်။ **Privacy price** ကို နှစ်အလိုက် ကောက်ခံရန် သတ်မှတ်ပါ၊ သို့မဟုတ် အခမဲ့ပေးလိုပါက $0 တွင် ထားရှိပါ။ |

Namecheap အတွက်၊ WHOIS ကိုယ်ရေးအချက်အလက် လျှို့ဝှက်မှုသည် WhoisGuard (အမြဲအခမဲ့) ကို အသုံးပြုသည်။ OpenSRS အတွက်၊ ၎င်းသည် OpenSRS privacy service (လက်ကားဈေးနှုန်းတွင် ကုန်ကျစရိတ်ရှိနိုင်သည်) ကို အသုံးပြုသည်။

## TLD တင်သွင်းခြင်းနှင့် ပေါင်းစပ်ခြင်း (TLD import and sync) {#tld-import-and-sync}

ဒိုမိန်ထုတ်ကုန်များသည် ချိတ်ဆက်ထားသည့် provider မှ ရယူထားသော လက်ရှိအချိန် ဈေးနှုန်းများကို ပြသပေးပါသည်။ ၎င်းအလုပ်လုပ်ရန်အတွက် TLD များကို တင်သွင်းထားရပါမည်။

- **Manual sync:** Settings › Domain Seller › Sync TLDs (ပေးသွင်းသူအလိုက်)
- **Automatic sync:** စနစ်တကျ အချိန်ဇယားဖြင့် (scheduled cron job) နေ့စဉ် လုပ်ဆောင်ပေးပါသည်။

sync လုပ်ပြီးနောက်၊ မည်သည့်ဒိုမိန်ထုတ်ကုန်၏ Domain Settings tab သို့သွား၍ TLD picker ကို အသုံးပြုကာ လက်ရှိလက်ကားဈေးနှုန်းများနှင့်အတူ ရရှိနိုင်သော TLD များကို စစ်ဆေးကြည့်နိုင်ပါသည်။

## အလိုအလျောက် သက်တမ်းတိုးခြင်း (Auto-renewal) {#auto-renewal}

ဒိုမိန်သက်တမ်းတိုးခြင်းများသည် ဖောက်သည်၏ အဖွဲ့ဝင်အဆင့်အတန်းနှင့် ချိတ်ဆက်ထားသည်-

- အဖွဲ့ဝင်အဆင့်အတန်း သက်တမ်းတိုးပြီး ဒိုမိန်တစ်ခု ချိတ်ဆက်ပါက၊ ဒိုမိန်သက်တမ်းတိုးခြင်းကို အလိုအလျောက် စီစဉ်ပေးမည်။
- သက်တမ်းတိုးရန် ကြိုးပမ်းမှုများသည် ဖောက်သည်၏ တက်ကြွသော payment gateway ကို အသုံးပြုသည်။
- သက်တမ်းတိုးမှု မအောင်မြင်ပါက၊ အဆအမြုပ်ကျဆင်းမှု (exponential backoff) ဖြင့် အလိုအလျောက် ပြန်လည်ကြိုးစားပေးမည်။
- အောင်မြင်သော သက်တမ်းတိုးမှုများ၊ မအောင်မြင်မှုများနှင့် မကြာမီ ကုန်ဆုံးတော့မည့် အကြောင်းအရာများအတွက် အီးမေးလ် အကြောင်းကြားချက်များ ပေးပို့မည်။

ဒိုမိန် ဘဝသံသရာ အဖြစ်အပျက်များအတွက် Email template ID များ-

| Event | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## Admin: လက်ဖြင့် ဒိုမိန်မှတ်ပုံတင်ခြင်း {#admin-manual-domain-registration}

ဖောက်သည်တစ်ဦးကို checkout လုပ်ခိုင်းခြင်းမရှိဘဲ၊ ၎င်းတို့ကိုယ်စား ဒိုမိန်တစ်ခုကို မှတ်ပုံတင်လိုပါက-

၁။ **Network Admin › Ultimate Multisite › Register Domain** သို့ သွားပါ။
၂။ ဖောက်သည်ကို ရွေးချယ်ပြီး ဒိုမိန်အမည်ကို ထည့်သွင်းပါ။
၃။ မှတ်ပုံတင်သူ၏ ဆက်သွယ်ရန် အချက်အလက်များကို ဖြည့်စွက်ပါ (အမည်၊ လိပ်စာ၊ ဖုန်း)။
၄။ **Register** ကို နှိပ်ပါ။

ဒိုမိန်မှတ်တမ်းကို ဖန်တီးပြီး ဖောက်သည်၏ အကောင့်နှင့် ချိတ်ဆက်ပေးမည်ဖြစ်သည်။
