# checkout အကွက်နှင့် သုံးစွဲသူ DNS စီမံခန့်ခွဲမှု

## ဒိုမိန်း ရွေးချယ်မှု checkout အကွက် {#the-domain-selection-checkout-field}

**ဒိုမိန်း ရွေးချယ်မှု** အကွက်သည် သုံးစွဲသူများအား ၎င်းတို့၏ ဝဘ်ဆိုက်ဒိုမိန်းကို မည်သို့ရယူမည်ကို ရွေးချယ်ခွင့်ပေးသော checkout အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။ ဒိုမိန်းရောင်းချမှု ဖွင့်ရန် မည်သည့် checkout ဖောင်တွင်မဆို ထည့်ပါ။

### checkout ဖောင်တွင် အကွက်ထည့်ခြင်း {#adding-the-field-to-a-checkout-form}

1. **ကွန်ရက် စီမံခန့်ခွဲသူ › Ultimate Multisite › Checkout ဖောင်များ** သို့ သွားပါ
2. checkout ဖောင်တစ်ခုကို ဖွင့်ပါ သို့မဟုတ် ဖန်တီးပါ
3. checkout တည်းဖြတ်ကိရိယာတွင် **အကွက် ထည့်ရန်** ကို နှိပ်ပါ
4. အကွက်စာရင်းမှ **ဒိုမိန်း ရွေးချယ်မှု** ကို ရွေးပါ
5. အကွက်ရွေးချယ်စရာများကို သတ်မှတ်ပါ (အောက်တွင် ကြည့်ပါ)
6. ဖောင်ကို သိမ်းပါ

### အကွက်ရွေးချယ်စရာများ {#field-options}

**ဒိုမိန်း မုဒ်များ** — သုံးစွဲသူ မြင်ရမည့် tab များကို ရွေးပါ။ မုဒ်တစ်ခုချင်းစီကို သီးခြား ဖွင့်နိုင် သို့မဟုတ် ပိတ်နိုင်သည်။

| မုဒ် | လုပ်ဆောင်ချက် |
|---|---|
| **ဆပ်ဒိုမိန်း** | သုံးစွဲသူသည် သင့်ကွန်ရက်ပေါ်ရှိ အခမဲ့ ဆပ်ဒိုမိန်းကို အသုံးပြုသည် (ဥပမာ `mysite.yournetwork.com`)။ ငွေပေးချေရန် မလိုပါ။ |
| **ဒိုမိန်းအသစ် မှတ်ပုံတင်ရန်** | သုံးစွဲသူသည် ဒိုမိန်းအသစ်တစ်ခုကို ရှာဖွေပြီး သင်သတ်မှတ်ထားသော ဝန်ဆောင်မှုပေးသူမှတစ်ဆင့် မှတ်ပုံတင်သည်။ ဈေးနှုန်းအတွက် ကိုက်ညီသော ဒိုမိန်းထုတ်ကုန်ကို အသုံးပြုသည်။ |
| **ရှိပြီးသား ဒိုမိန်း** | သုံးစွဲသူသည် မိမိပိုင်ပြီးသား ဒိုမိန်းတစ်ခုကို map လုပ်သည်။ မှတ်ပုံတင်ကြေး မရှိပါ။ ဒိုမိန်းကို ၎င်းတို့၏ ဝဘ်ဆိုက်သို့ အလိုအလျောက် map လုပ်ပေးသည်။ |

**မူရင်း မုဒ်** — မုဒ်သုံးခုလုံး ဖွင့်ထားသောအခါ မည်သည့် tab ကို ပထမဆုံး ဖွင့်မည်ကို သတ်မှတ်သည်။ ဒိုမိန်းမှတ်ပုံတင်ခြင်းကို ရွေးချယ်စရာအဖြစ် ထားလိုပါက **ဆပ်ဒိုမိန်း** သို့ သတ်မှတ်ပါ၊ ဝယ်ယူမှုကို အားပေးလိုပါက **ဒိုမိန်းအသစ် မှတ်ပုံတင်ရန်** သို့ သတ်မှတ်ပါ။

**ဒိုမိန်းထုတ်ကုန်** — လိုအပ်ပါက ဤအကွက်ကို သတ်မှတ်ထားသော ဒိုမိန်းထုတ်ကုန်တစ်ခုနှင့် ချိတ်ထားနိုင်သည်။ မသတ်မှတ်ထားပါက အပိုထည့်သွင်းမှုသည် သုံးစွဲသူ ရှာဖွေသော TLD အပေါ်မူတည်၍ ကိုက်ညီသော ထုတ်ကုန်ကို အလိုအလျောက် ရွေးချယ်သည်။

### မှတ်ပုံတင်သူ ဆက်သွယ်ရန် အကွက်များ {#registrant-contact-fields}

သုံးစွဲသူက **ဒိုမိန်းအသစ် မှတ်ပုံတင်ရန်** tab ကို ရွေးသောအခါ checkout ဖောင်တွင် မှတ်ပုံတင်သူ ဆက်သွယ်ရန် အကွက်များကို တစ်ကြောင်းတည်းအတွင်း ထည့်ပေးသည်။

- ပထမအမည် / နောက်ဆုံးအမည်
- အီးမေးလ်လိပ်စာ
- လိပ်စာ (လိုင်း ၁၊ မြို့၊ ပြည်နယ်/တိုင်းဒေသကြီး၊ စာပို့ကုဒ်၊ နိုင်ငံ)
- ဖုန်းနံပါတ်

ဤအချက်များကို မှတ်ပုံတင်အဖွဲ့များအားလုံးက လိုအပ်ပြီး မှတ်ပုံတင် API ခေါ်ဆိုမှု မပြုလုပ်မီ စစ်ဆေးအတည်ပြုသည်။ ဖုန်းနံပါတ်များကို မှတ်ပုံတင်အဖွဲ့များ မျှော်လင့်သော `+CC.NNN` နိုင်ငံတကာပုံစံသို့ အလိုအလျောက် ပြောင်းပေးသည်။

### အလိုအလျောက် ဖန်တီးထားသော ဝဘ်ဆိုက် URL {#auto-generated-site-url}

သုံးစွဲသူက ဒိုမိန်းတစ်ခုကို မှတ်ပုံတင် သို့မဟုတ် map လုပ်သောအခါ ဝဘ်ဆိုက် URL အကွက်ကို ရွေးထားသော ဒိုမိန်းမှ အလိုအလျောက် ဖြည့်ပေးသည်။ သုံးစွဲသူများသည် သီးခြား URL အကွက်ကို ဖြည့်ရန် မလိုပါ။

### ရှာဖွေမှု အပြုအမူ {#search-behaviour}

- သုံးစွဲသူ စာရိုက်နေစဉ် ဒိုမိန်းရရှိနိုင်မှုကို AJAX ဖြင့် အချိန်နှင့်တပြေးညီ စစ်ဆေးသည်
- နှစ်သက်သော ဒိုမိန်း မရရှိနိုင်ပါက အစားထိုး TLD အကြံပြုချက်များကို ပြသသည်
- ဈေးနှုန်းကို တိုက်ရိုက် ရယူပြီး ရှင်းလင်းစွာ ပြသသည် (မှတ်ပုံတင်ဈေးနှုန်း၊ သက်တမ်းတိုးဈေးနှုန်း၊ ရွေးချယ်နိုင်သော WHOIS ကိုယ်ရေးလုံခြုံမှုကြေး)
- ကူပွန်ကုဒ်များသည် အခြားထုတ်ကုန်များကဲ့သို့ပင် ဒိုမိန်းထုတ်ကုန်များတွင်လည်း သက်ရောက်သည်

**ရှာဖွေမှုတုံ့ပြန်မှုမြန်နှုန်း ချိန်ညှိခြင်း:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ဒိုမိန်းရှာဖွေရေးဖောင်သို့ စိတ်ကြိုက်အကွက်များ ထည့်ခြင်း:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## သုံးစွဲသူ DNS စီမံခန့်ခွဲမှု {#customer-dns-management}

သုံးစွဲသူများသည် **ကျွန်ုပ်၏ Account** စာမျက်နှာရှိ ၎င်းတို့၏ ဒိုမိန်းအချက်အလက်အောက်တွင် မိမိတို့ မှတ်ပုံတင်ထားသော ဒိုမိန်းများအတွက် DNS မှတ်တမ်းများကို စီမံနိုင်သည်။

### ပံ့ပိုးထားသော မှတ်တမ်းအမျိုးအစားများ {#supported-record-types}

| အမျိုးအစား | အသုံးပြုမှု |
|---|---|
| **A** | hostname ကို IPv4 လိပ်စာသို့ map လုပ်ရန် |
| **AAAA** | hostname ကို IPv6 လိပ်စာသို့ map လုပ်ရန် |
| **CNAME** | အခြား hostname တစ်ခုသို့ ညွှန်သော alias တစ်ခု ဖန်တီးရန် |
| **MX** | mail exchange server သတ်မှတ်ရန် |
| **TXT** | SPF၊ DMARC၊ အတည်ပြုခြင်း သို့မဟုတ် အခြား စာသားမှတ်တမ်းများ ထည့်ရန် |

### မည်သည့် ဝန်ဆောင်မှုပေးသူများက DNS စီမံခန့်ခွဲမှုကို ပံ့ပိုးသနည်း။ {#which-providers-support-dns-management}

DNS စီမံခန့်ခွဲမှု (မှတ်တမ်းများ ထည့်ခြင်း၊ တည်းဖြတ်ခြင်း၊ ဖျက်ခြင်း) ကို **OpenSRS**၊ **ResellerClub**၊ **Enom**၊ **HostAfrica** နှင့် **Openprovider** တို့တွင် အသုံးပြုနိုင်သည်။ **Hostinger** ဒိုမိန်းများသည် Domain Seller မှတစ်ဆင့် nameserver များကို အပ်ဒိတ်လုပ်နိုင်သည်။ hosted ဒိုမိန်းများအတွက် DNS မှတ်တမ်းများကို မူလ Hostinger ဒိုမိန်း map လုပ်သည့် ပေါင်းစည်းမှုက စီမံသည်။ Namecheap၊ GoDaddy နှင့် NameSilo ဒိုမိန်းများသည် အခြေအနေနှင့် သက်တမ်းကုန်ဆုံးရက် အချက်အလက်များကို ပြသသော်လည်း DNS ကို မှတ်ပုံတင်အဖွဲ့၏ control panel တွင် တိုက်ရိုက် စီမံရမည်။

### မူရင်း DNS မှတ်တမ်းများ {#default-dns-records}

ဒိုမိန်းတစ်ခု မှတ်ပုံတင်သောအခါ အလိုအလျောက် အသုံးပြုမည့် မူရင်း DNS မှတ်တမ်းများကို သင် သတ်မှတ်နိုင်သည်။ **ဆက်တင်များ › Domain Seller › မူရင်း DNS မှတ်တမ်းများ** သို့ သွားပါ။

မူရင်း မှတ်တမ်းတန်ဖိုးများသည် token နှစ်ခုကို ပံ့ပိုးသည်။

| Token | အစားထိုးမည့်အရာ |
|---|---|
| `{DOMAIN}` | မှတ်ပုံတင်ထားသော ဒိုမိန်းအမည် (ဥပမာ `example.com`) |
| `{SITE_URL}` | သုံးစွဲသူ၏ ဝဘ်ဆိုက်အတွက် WordPress ဝဘ်ဆိုက် URL |

**ဥပမာ — apex ဒိုမိန်းနှင့် www ကို သင့် server IP သို့ ညွှန်ရန်:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### စီမံခန့်ခွဲသူ: DNS ကြည့်ရှုခြင်းနှင့် တည်းဖြတ်ခြင်း {#admin-viewing-and-editing-dns}
