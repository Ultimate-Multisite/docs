# ငွေရှင်းကဏ္ဍအတွက် ဒိုမိန်ရွေးချယ်မှုနှင့် Customer DNS စီမံခန့်ခွဲမှု

## Domain Selection ငွေရှင်းကဏ္ဍ

**Domain Selection** field ဆိုတာ ဝယ်ယူသူတွေ သူတို့ရဲ့ site domain ကို ဘယ်လိုနည်းနဲ့ ရွေးချယ်နိုင်မလဲဆိုတာကို ရွေးချယ်ခွင့်ပေးတဲ့ ငွေရှင်းကဏ္ဍတစ်ခု ဖြစ်ပါတယ်။ ဒီ field ကို ငွေရှင်းပုံစံ (checkout form) ဘယ်နေရာမှာမဆို ထည့်သွင်းနိုင်ပြီး၊ domain ရောင်းချနိုင်ဖို့ စတင်အသုံးပြုနိုင်ပါတယ်။

### ငွေရှင်းပုံစံမှာ field ထည့်သွင်းခြင်း

၁။ **Network Admin › Ultimate Multisite › Checkout Forms** ကို သွားပါ။
၂။ ငွေရှင်းပုံစံတစ်ခုကို ဖွင့်ပါ သို့မဟုတ် အသစ်တစ်ခု ဖန်တီးပါ။
၃။ ငွေရှင်းပုံစံ တည်းဖြတ်သူ (editor) မှာ **Add Field** ကို နှိပ်ပါ။
၄။ field စာရင်းထဲကနေ **Domain Selection** ကို ရွေးချယ်ပါ။
၅။ field ရွေးချယ်မှုများကို စီစဉ်ဖွဲ့စည်းပါ။ (အောက်မှာ ကြည့်ပါ)
၆။ ပုံစံကို သိမ်းဆည်းပါ။

### Field ရွေးချယ်မှုများ

**Domain modes** — ဝယ်ယူသူ မြင်ရမယ့် tab တွေကို ရွေးချယ်ပါ။ mode တစ်ခုချင်းစီကို သီးခြားစီ ဖွင့်နိုင်သည် သို့မဟုတ် ပိတ်နိုင်ပါတယ်။

| Mode | ဘာလုပ်သလဲ |
|---|---|
| **Subdomain** | ဝယ်ယူသူက သင့် network ပေါ်က အခမဲ့ subdomain ကို အသုံးပြုပါလိမ့်မယ် (ဥပမာ- `mysite.yournetwork.com`)။ ငွေပေးစရာ မလိုပါ။ |
| **Register New Domain** | ဝယ်ယူသူက domain အသစ်တစ်ခုကို ရှာဖွေပြီး သင့်ရဲ့ သတ်မှတ်ထားတဲ့ provider ကနေ စာရင်းသွင်းပါလိမ့်မယ်။ ဈေးနှုန်းသတ်မှတ်ရာမှာ ကိုက်ညီတဲ့ domain product ကို အသုံးပြုပါတယ်။ |
| **Existing Domain** | ဝယ်ယူသူက သူကိုယ်တိုင် ပိုင်ဆိုင်ပြီးသား domain တစ်ခုကို ချိတ်ဆက်ပေးပါတယ်။ စာရင်းသွင်းခ မရှိပါဘူး။ domain ကို သူတို့ရဲ့ site နဲ့ အလိုအလျောက် ချိတ်ဆက်ပေးပါတယ်။ |

**Default mode** — mode သုံးခုလုံးကို ဖွင့်ထားတဲ့အခါ ဘယ် tab က အရင်ဖွင့်မလဲ။ domain စာရင်းသွင်းတာကို ရွေးချယ်စရာအဖြစ်ထားချင်ရင် **Subdomain** ကို သတ်မှတ်ပါ။ ဒါမှမဟုတ် ဝယ်ယူမှုမြှင့်တင်ချင်ရင် **Register New Domain** ကို သတ်မှတ်ပါ။

**Domain product** — ဒီ field ကို သီးသန့် domain product တစ်ခုနဲ့ ချိတ်ဆက်လိုပါက ရွေးချယ်နိုင်ပါတယ်။ သတ်မှတ်မထားရင်လည်း addon က ဝယ်ယူသူ ရှာဖွေတဲ့ TLD အပေါ်မူတည်ပြီး ကိုက်ညီတဲ့ product ကို အလိုအလျောက် ရွေးပေးမှာပါ။

### စာရင်းသွင်းသူ အဆက်အသွယ် အချက်အလက်များ

ဝယ်ယူသူက **Register New Domain** tab ကို ရွေးချယ်တဲ့အခါ၊ ငွေရှင်းပုံစံက စာရင်းသွင်းသူ အဆက်အသွယ် အချက်အလက် field တွေကို အောက်မှာ အလိုအလျောက် ထည့်ပေးမှာ ဖြစ်ပါတယ်-

- First name / Last name
- Email address
- Address (line 1, city, state/province, postal code, country)
- Phone number

ဒီအချက်အလက်တွေဟာ စာရင်းသွင်းသူအားလုံးက မဖြစ်မနေ လိုအပ်တာတွေဖြစ်ပြီး၊ စာရင်းသွင်း API ခေါ်ဆိုမှု မလုပ်ခင်မှာ စစ်ဆေးအတည်ပြုမှု လုပ်ပေးပါတယ်။ ဖုန်းနံပါတ်တွေကို စာရင်းသွင်းသူတွေက မျှော်လင့်ထားတဲ့ `+CC.NNN` နိုင်ငံတကာ ပုံစံအတိုင်း အလိုအလျောက် ပြောင်းလဲပေးပါတယ်။

### အလိုအလျောက် ထုတ်ပေးသော site URL

ဝယ်ယူသူက domain ကို စာရင်းသွင်းသည်ဖြစ်စေ၊ ချိတ်ဆက်သည်ဖြစ်စေ၊ site URL field ကို ရွေးချယ်ထားတဲ့ domain ကနေ အလိုအလျောက် ဖြည့်ပေးမှာ ဖြစ်ပါတယ်။ ဝယ်ယူသူတွေက URL field အသစ်တစ်ခုကို ကိုယ်တိုင် ဖြည့်စရာ မလိုပါဘူး။

### ရှာဖွေမှု အပြုအမူ (Search behaviour)

- ဝယ်ယူသူက စာရိုက်နေချိန်မှာ domain ရရှိနိုင်မှု အခြေအနေကို AJAX နဲ့ အချိန်နဲ့တပြေးညီ စစ်ဆေးပေးပါတယ်။
- အကြိုက်ဆုံး domain က မရရှိနိုင်တဲ့အခါ အစားထိုး TLD အကြံပြုချက်တွေကို ပြသပေးပါတယ်။
- ဈေးနှုန်းတွေကို အချိန်နဲ့တပြေးညီ ယူဆောင်လာပြီး ရှင်းရှင်းလင်းလင်း ပြသပေးပါတယ် (စာရင်းသွင်းခ၊ သက်တမ်းတိုးခ၊ ရွေးချယ်နိုင်တဲ့ WHOIS privacy fee)။
- Coupon code တွေဟာ တခြား product တွေလိုပဲ domain product တွေမှာလည်း အကျုံးဝင်ပါတယ်။

**ရှာဖွေမှု တုံ့ပြန်မှု မြန်ဆန်စေရန် စီမံခန့်ခွဲခြင်း:**

```php
// API ခေါ်ဆိုမှုတွေကို လျှော့ချဖို့ debounce delay ကို တိုးမြှင့်ပါ (milliseconds)
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**domain ရှာဖွေမှု ပုံစံမှာ custom field များ ထည့်သွင်းခြင်း:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Customer DNS စီမံခန့်ခွဲမှု

ဝယ်ယူသူတွေဟာ သူတို့ရဲ့ စာရင်းသွင်းထားတဲ့ domain တွေအတွက် DNS မှတ်တမ်းတွေကို **My Account** စာမျက်နှာမှာ၊ သူတို့ရဲ့ domain ရဲ့ အပိုင်းအောက်မှာ စီမံခန့်ခွဲနိုင်ပါတယ်။

### ပံ့ပိုးထားသော မှတ်တမ်းအမျိုးအစားများ

| Type | အသုံးပြုပုံ |
|---|---|
| **A** | Hostname ကို IPv4 address နဲ့ ချိတ်ဆက်ပေးခြင်း |
| **AAAA** | Hostname ကို IPv6 address နဲ့ ချိတ်ဆက်ပေးခြင်း |
| **CNAME** | အမည်ပြောင်း (alias) တစ်ခု ဖန်တီးပြီး တခြား hostname ကို ညွှန်ပြခြင်း |
| **MX** | အီးမေးလ်လဲလှယ်ဆာဗာ (mail exchange server) ကို သတ်မှတ်ခြင်း |
| **TXT** | SPF, DMARC, အတည်ပြုချက် (verification) သို့မဟုတ် အခြား text မှတ်တမ်းများ ထည့်ခြင်း |

### DNS စီမံခန့်ခွဲမှုကို ဘယ် provider တွေက ပံ့ပိုးပေးသလဲ။

DNS စီမံခန့်ခွဲမှု (မှတ်တမ်းထည့်ခြင်း၊ တည်းဖြတ်ခြင်း၊ ဖျက်ခြင်း) ကို **OpenSRS**, **ResellerClub**, နဲ့ **Enom** တို့မှာ ရရှိပါတယ်။ Namecheap, GoDaddy, နဲ့ NameSilo domain တွေက အခြေအနေနဲ့ သက်တမ်းကုန်ဆုံးမှု အချက်အလက်တွေကို ပြသပေးပေမယ့် DNS ကိုတော့ registrar ရဲ့ control panel ထဲမှာ တိုက်ရိုက် စီမံခန့်ခွဲရမှာ ဖြစ်ပါတယ်။

### မူလ DNS မှတ်တမ်းများ

domain တစ်ခု စာရင်းသွင်းတဲ့အခါ အလိုအလျောက် အသုံးပြုပေးမယ့် default DNS မှတ်တမ်းတွေကို စီစဉ်ဖွဲ့စည်းနိုင်ပါတယ်။ **Settings › Domain Seller › Default DNS Records** ကို သွားပါ။

Default record value တွေဟာ token နှစ်ခုကို ပံ့ပိုးပေးပါတယ်-

| Token | ဘာနဲ့ အစားထိုးမလဲ |
|---|---|
| `{DOMAIN}` | စာရင်းသွင်းထားတဲ့ domain အမည် (ဥပမာ- `example.com`) |
| `{SITE_URL}` | ဝယ်ယူသူရဲ့ site အတွက် WordPress site URL |

**ဥပမာ — apex domain နဲ့ www ကို သင့် server IP ကို ညွှန်ပြခြင်း:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Admin: DNS ကြည့်ရှုခြင်းနှင့် တည်းဖြတ်ခြင်း

Network admin တွေဟာ **Network Admin › Ultimate Multisite › Domains** မှာရှိတဲ့ domain ရဲ့ edit page ကနေ ဝယ်ယူသူ ဘယ် domain ရဲ့ DNS မှတ်တမ်းတွေပဲဖြစ်ဖြစ် ကြည့်ရှုပြီး တည်းဖြတ်နိုင်ပါတယ်။
