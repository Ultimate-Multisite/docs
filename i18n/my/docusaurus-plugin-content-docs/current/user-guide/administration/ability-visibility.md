# စွမ်းရည် မြင်နိုင်မှု (Ability Visibility)

Superdav AI Agent v1.12.0 တွင် **စွမ်းရည် မြင်နိုင်မှု (Ability Visibility)** ထိန်းချုပ်မှုများကို မိတ်ဆက်ပေးထားပြီး၊ ဤစွမ်းရည်များကို မည်သည့် နေရာများတွင် ဖော်ပြရမည်ကို စီမံခန့်ခွဲပေးပါသည်။ ဤအရာက စီမံခန့်ခွဲသူများအား REST APIs၊ MCP servers၊ WooCommerce integrations နှင့် အခြား interface များမှတစ်ဆင့် မည်သည့် agent စွမ်းရည်များကို အသုံးပြုနိုင်သည်ကို သေချာချိန်ညှိနိုင်စေပါသည်။

## စွမ်းရည် မြင်နိုင်မှုဆိုသည်မှာ ဘာလဲ။ {#what-is-ability-visibility}

စွမ်းရည် မြင်နိုင်မှု (Ability Visibility) ဆိုသည်မှာ အောက်ပါတို့ကို ထိန်းချုပ်ပေးသည့် ခွင့်ပြုချက်စနစ်တစ်ခု ဖြစ်သည်-

- **မည်သည့် စွမ်းရည်များ** ကို agent များ အသုံးပြုနိုင်သလဲ
- **မည်သည့်နေရာများတွင် ဖော်ပြမည်** (REST API၊ MCP၊ WooCommerce စသည်)
- **မည်သူများ ဝင်ရောက်ခွင့်ရနိုင်သလဲ** (အဖော်အပယ် ခွင့်ပြုစာရင်း (partner allow-lists) မှတစ်ဆင့်)
- **မည်သို့ အမျိုးအစားခွဲခြားထားသလဲ** (အသိအမှတ်ပြုသည် (recognized) သို့မဟုတ် အသိအမှတ်မပြုသည် (unclassified))

စွမ်းရည်တစ်ခုစီတိုင်းတွင် ၎င်း၏ ရရှိနိုင်မှုအပေါ် အကျိုးသက်ရောက်သည့် မြင်နိုင်မှုအဆင့် (visibility level) တစ်ခု ပါရှိပါသည်။

## မြင်နိုင်မှုအဆင့်များ (Visibility Levels) {#visibility-levels}

### Public (အများသုံး) {#public}

**Public စွမ်းရည်များ** ကို နေရာတိုင်းတွင် အသုံးပြုနိုင်သည်-

- REST API endpoints များ
- MCP servers များ
- WooCommerce integrations များ
- Chat interfaces များ
- အသုံးပြုသူ အခန်းကဏ္ဍအားလုံး (သင့်လျော်သော ခွင့်ပြုချက်များဖြင့်)

ဥပမာ- `scaffold-block-theme`, `activate-theme`, `send-email`

### Internal (အတွင်းပိုင်း) {#internal}

**Internal စွမ်းရည်များ** ကို သင့် WordPress တပ်ဆင်မှုအတွင်း၌သာ အသုံးပြုနိုင်သည်-

- Chat interfaces များ
- Admin panels များ
- အကောင့်ဝင်ထားသော အသုံးပြုသူများသာ
- REST API သို့မဟုတ် ပြင်ပ integrations များမှတစ်ဆင့် ဖော်ပြမည်မဟုတ်

ဥပမာ- `manage-settings`, `view-analytics`, `export-data`

### Partner (အဖော်အပယ်) {#partner}

**Partner စွမ်းရည်များ** ကို ခွင့်ပြုထားသော အဖော်အပယ် (whitelisted) များအတွက်သာ အသုံးပြုနိုင်သည်-

- ရှင်းလင်းစွာ ခွင့်ပြုထားသော အဖော်အပယ်စာရင်းတွင် ပါဝင်ရန် လိုအပ်သည်
- အဖော်အပယ် credentials များဖြင့် REST API မှတစ်ဆင့် ရရှိနိုင်သည်
- အဖော်အပယ် authentication ဖြင့် MCP မှတစ်ဆင့် ရရှိနိုင်သည်
- ပုံမှန်အသုံးပြုသူများအတွက် မရရှိနိုင်ပါ

ဥပမာ- `bulk-import-users`, `modify-billing`, `access-analytics`

### Disabled (ပိတ်ထားသည်) {#disabled}

**Disabled စွမ်းရည်များ** ကို မည်သည့်နေရာတွင်မှ အသုံးပြုနိုင်မည်မဟုတ်ပါ-

- မည်သည့် နေရာမှတစ်ဆင့်မျှ ဖော်ပြမည်မဟုတ်
- Chat တွင် မရရှိနိုင်
- Admin panels တွင် မရရှိနိုင်
- ခေတ်ကုန်ပြီဖြစ်သော သို့မဟုတ် စမ်းသပ်အသုံးပြုနေသော စွမ်းရည်များအတွက် အသုံးဝင်သည်

## စွမ်းရည် မြင်နိုင်မှုကို စီမံခန့်ခွဲခြင်း {#managing-ability-visibility}

### စွမ်းရည် မြင်နိုင်မှု Setting များသို့ ဝင်ရောက်ခြင်း {#accessing-ability-visibility-settings}

၁။ **WordPress Admin** → **Superdav AI Agent** → **Settings** သို့ သွားပါ။
၂။ **Abilities** tab ကို နှိပ်ပါ။
၃။ တပ်ဆင်ထားသော စွမ်းရည်အားလုံးနှင့် ၎င်းတို့၏ မြင်နိုင်မှုအဆင့်များကို စာရင်းအဖြစ် မြင်ရပါမည်။

### စွမ်းရည် အသေးစိတ်ကို ကြည့်ရှုခြင်း {#viewing-ability-details}

စွမ်းရည်တစ်ခုခုကို နှိပ်ခြင်းဖြင့် အောက်ပါအချက်များကို ကြည့်ရှုနိုင်သည်-

- **Name**: စွမ်းရည်၏ အမှတ်အသား (identifier)
- **Description**: စွမ်းရည်က ဘာလုပ်ဆောင်သလဲ
- **Current visibility**: Public, Internal, Partner, သို့မဟုတ် Disabled
- **Partner allow-list**: မည်သည့် အဖော်အပယ်များ ဝင်ရောက်နိုင်သလဲ (Partner မြင်နိုင်မှုဖြစ်ပါက)
- **Last updated**: မြင်နိုင်မှုကို နောက်ဆုံးအကြိမ် ပြောင်းလဲခဲ့သည့်အချိန်
- **Status**: Recognized (အသိအမှတ်ပြုသည်) သို့မဟုတ် Unclassified (အသိအမှတ်မပြုသည်)

### မြင်နိုင်မှုအဆင့်များကို ပြောင်းလဲခြင်း {#changing-visibility-levels}

စွမ်းရည်တစ်ခု၏ မြင်နိုင်မှုကို ပြောင်းလဲရန်-

၁။ စာရင်းရှိ စွမ်းရည်ကို နှိပ်ပါ။
၂။ dropdown မှ မြင်နိုင်မှုအဆင့်အသစ်ကို ရွေးချယ်ပါ။
၃။ "Partner" ကို ရွေးချယ်ပါက၊ ခွင့်ပြုစာရင်း (allow-list) တွင် အဖော်အပယ် အမှတ်အသားများကို ထည့်ပါ။
၄။ **Save** ကို နှိပ်ပါ။

ဥပမာ-

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### အစုလိုက် လုပ်ဆောင်မှုများ (Bulk Operations) {#bulk-operations}

စွမ်းရည်များစွာ၏ မြင်နိုင်မှုကို ပြောင်းလဲရန်-

၁။ စွမ်းရည်များဘေးရှိ ခလုတ်များကို အမှန်ခြစ်ပါ။
၂။ အစုလိုက် လုပ်ဆောင်မှု dropdown မှ မြင်နိုင်မှုအဆင့်အသစ်ကို ရွေးချယ်ပါ။
၃။ **Apply** ကို နှိပ်ပါ။

## အဖော်အပယ် ခွင့်ပြုစာရင်း (Partner Allow-List) {#partner-allow-list}

**အဖော်အပယ် ခွင့်ပြုစာရင်း (partner allow-list)** သည် မည်သည့် ပြင်ပအဖော်အပယ်များက Partner-level စွမ်းရည်များကို ဝင်ရောက်ခွင့်ရနိုင်သည်ကို ထိန်းချုပ်ပေးပါသည်။

### အဖော်အပယ်များ ထည့်ခြင်း {#adding-partners}

၁။ **Superdav AI Agent** → **Settings** → **Partners** သို့ သွားပါ။
၂။ **Add Partner** ကို နှိပ်ပါ။
၃။ အဖော်အပယ် အမှတ်အသား (ပုံမှန်အားဖြင့် API key သို့မဟုတ် organization ID) ကို ထည့်ပါ။
၄။ ရွေးချယ်ပါက အဖော်အပယ်အမည်နှင့် ဖော်ပြချက်ကို ထည့်ပါ။
၅။ **Save** ကို နှိပ်ပါ။

### အဖော်အပယ်များအား စွမ်းရည်များ ခွဲဝေပေးခြင်း {#assigning-abilities-to-partners}

အဖော်အပယ်အသစ် ထည့်ပြီးနောက်-

၁။ **Abilities** tab သို့ သွားပါ။
၂။ Partner မြင်နိုင်မှုရှိသော စွမ်းရည်တစ်ခုကို ရှာပါ။
၃။ တည်းဖြတ်ရန် နှိပ်ပါ။
၄။ "Partner allow-list" ကဏ္ဍတွင် ဝင်ရောက်ခွင့်ရသင့်သော အဖော်အပယ်များ၏ ခလုတ်များကို အမှန်ခြစ်ပါ။
၅။ **Save** ကို နှိပ်ပါ။

### အဖော်အပယ် ဝင်ရောက်ခွင့် ရုပ်သိမ်းခြင်း {#revoking-partner-access}

အဖော်အပယ်တစ်ဦး၏ ဝင်ရောက်ခွင့်ကို ဖယ်ရှားရန်-

၁။ စွမ်းရည်ကို တည်းဖြတ်ပါ။
၂။ ခွင့်ပြုစာရင်းရှိ အဖော်အပယ်၏ ခလုတ်ကို အမှန်ခြစ်ဖြုတ်ပါ။
၃။ **Save** ကို နှိပ်ပါ။

ထိုအဖော်အပယ်သည် ထိုစွမ်းရည်ကို ချက်ချင်း အသုံးပြုခွင့် ဆုံးရှုံးသွားမည်ဖြစ်သည်။

## အသိအမှတ်မပြု စွမ်းရည်များ (Unclassified Abilities) {#unclassified-abilities}

Superdav AI Agent က အသိအမှတ်မပြုသော ပြင်ပအဖော်အပယ် စွမ်းရည်တစ်ခုကို တပ်ဆင်ပါက၊ ၎င်းကို **Unclassified** အဖြစ် အမှတ်အသားပြုထားမည်ဖြစ်သည်။

### Unclassified စွမ်းရည်များအတွက် Admin အသိပေးချက် {#admin-notice-for-unclassified-abilities}

အောက်ပါ အသိပေးချက်ကို မြင်ရမည်-

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### Unclassified စွမ်းရည်များကို ပြန်လည်သုံးသပ်ခြင်း {#reviewing-unclassified-abilities}

၁။ အသိပေးချက်ရှိ **Review Abilities** ကို နှိပ်ပါ။
၂။ Unclassified စွမ်းရည်တစ်ခုစီအတွက် အောက်ပါအချက်များကို မြင်ရမည်-
   - စွမ်းရည်အမည်နှင့် ဖော်ပြချက်
   - အရင်းအမြစ် (မည်သည့် plugin/addon က တပ်ဆင်ထားသည်)
   - အကြံပြုထားသော မြင်နိုင်မှုအဆင့်
   - လုံခြုံရေး အကဲဖြတ်မှု

၃။ မြင်နိုင်မှုအဆင့်ကို ရွေးချယ်ပါ-
   - **Public**: စွမ်းရည်ကို ယုံကြည်သည်၊ နေရာတိုင်းတွင် ဖော်ပြမည်
   - **Internal**: အတွင်းပိုင်း အသုံးပြုမှုအတွက်သာ ကန့်သတ်မည်
   - **Partner**: သတ်မှတ်ထားသော အဖော်အပယ်များအတွက် ကန့်သတ်မည်
   - **Disabled**: ဤစွမ်းရည်ကို အသုံးမပြုပါနှင့်

၄။ သင့်ဆုံးဖြတ်ချက်ကို သိမ်းဆည်းရန် **Classify** ကို နှိပ်ပါ။

### Unclassified စွမ်းရည်များကို ဘာကြောင့် အမျိုးအစားခွဲခြားသင့်သလဲ။ {#why-classify-unclassified-abilities}

စွမ်းရည်များကို အမျိုးအစားခွဲခြားခြင်းသည်-

- **လုံခြုံရေးကို မြှင့်တင်ပေးသည်**: မည်သည့်အရာကို ဖော်ပြမည်ကို သင်ကိုယ်တိုင် အတည်ပြုပေးခြင်းဖြစ်သည်
- **မတော်တဆမှုများကို ကာကွယ်သည်**: Unclassified စွမ်းရည်များသည် မူလအကျိုးအမြတ် ပိတ်ထားသည်
- **အင်္ဂါရပ်များကို အသက်သွင်းသည်**: အမျိုးအစားခွဲခြားပြီးသည်နှင့် စွမ်းရည်သည် အသုံးပြုနိုင်သည်
- **ဆုံးဖြတ်ချက်များကို မှတ်တမ်းတင်သည်**: သင်၏ရွေးချယ်မှုများကို စစ်ဆေးရန်အတွက် မှတ်တမ်းတင်ထားသည်

## မတူညီသော နေရာများရှိ မြင်နိုင်မှု {#visibility-in-different-surfaces}

### REST API {#rest-api}

**Public စွမ်းရည်များ** ကို REST endpoints များမှတစ်ဆင့် ရရှိနိုင်သည်-

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**Internal နှင့် Partner စွမ်းရည်များ** ကို REST API မှတစ်ဆင့် ရရှိနိုင်မည်မဟုတ်ပါ။

### MCP Servers {#mcp-servers}

**Public စွမ်းရည်များ** ကို MCP မှတစ်ဆင့် ရရှိနိုင်သည်-

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**Internal စွမ်းရည်များ** ကို MCP မှတစ်ဆင့် ဖော်ပြမည်မဟုတ်ပါ။

**Partner စွမ်းရည်များ** ကို အဖော်အပယ် credentials များဖြင့်သာ ရရှိနိုင်ပါသည်။

### WooCommerce Integration {#woocommerce-integration}

WooCommerce နှင့် သက်ဆိုင်သော **Public စွမ်းရည်များ** ကို ရရှိနိုင်သည်-

- ကုန်ပစ္စည်း စီမံခန့်ခွဲမှု
- အော်ဒါ လုပ်ဆောင်မှု
- ဖောက်သည် ဆက်သွယ်မှု

**Internal စွမ်းရည်များ** ကို WooCommerce သို့ ဖော်ပြမည်မဟုတ်ပါ။

### Chat Interface {#chat-interface}

**စွမ်းရည်အားလုံး** (Public, Internal, Partner) ကို အသုံးပြုသူ ခွင့်ပြုချက်ပေါ် မူတည်၍ Chat တွင် ရရှိနိုင်သည်-

- စီမံခန့်ခွဲသူများသည် စွမ်းရည်အားလုံးကို မြင်ရသည်
- ပုံမှန်အသုံးပြုသူများသည် Public စွမ်းရည်များကိုသာ မြင်ရသည်
- အဖော်အပယ်များ (whitelisted ဖြစ်ပါက) သည် Public + Partner စွမ်းရည်များကို မြင်ရသည်

## လုံခြုံရေး အကောင်းဆုံး လုပ်ဆောင်မှုများ (Security Best Practices) {#security-best-practices}

### အခွင့်အရေး အနည်းဆုံး မူသဘောတရား (Principle of Least Privilege) {#principle-of-least-privilege}

- အသုံးပြုနိုင်မှုရှိစေရန် အခွင့်အရေးအနည်းဆုံး မြင်နိုင်မှုအဆင့်ကို သတ်မှတ်ပါ။
- အထိခိုက်ဆုံး လုပ်ဆောင်မှုများအတွက် Partner မြင်နိုင်မှုကို အသုံးပြုပါ။
- မအသုံးပြုသော စွမ်းရည်များကို ပိတ်ထားပါ။

### ပုံမှန် စစ်ဆေးမှုများ (Regular Audits) {#regular-audits}

- စွမ်းရည် မြင်နိုင်မှုကို လစဉ် ပြန်လည်သုံးသပ်ပါ။
- Unclassified စွမ်းရည်များ ရှိမရှိ စစ်ဆေးပါ။
- မအသုံးပြုသော အဖော်အပယ်များ၏ ဝင်ရောက်ခွင့်ကို ဖယ်ရှားပါ။

### မှတ်တမ်းတင်ခြင်းနှင့် စောင့်ကြည့်ခြင်း (Logging and Monitoring) {#logging-and-monitoring}

- မည်သည့် စွမ်းရည်များကို အများဆုံး အသုံးပြုသည်ကို စောင့်ကြည့်ပါ။
- အဖော်အပယ် ဝင်ရောက်မှု ပုံစံများကို စောင့်ကြည့်ပါ။
- ပုံမှန်မဟုတ်သော စွမ်းရည် အသုံးပြုမှုများအတွက် သတိပေးချက်ပေးပါ။

### ပြင်ပအဖော်အပယ် စွမ်းရည်များ (Third-Party Abilities) {#third-party-abilities}

- အသုံးပြုခွင့်ပေးခြင်းမပြုမီ ပြင်ပအဖော်အပယ် စွမ်းရည်များကို ပြန်လည်သုံးသပ်ပါ။
- ၎င်းတို့ကို ရှင်းလင်းစွာ အမျိုးအစားခွဲခြားပါ။
- Internal သို့မဟုတ် Partner မြင်နိုင်မှုဖြင့် စတင်ပါ။
- စစ်ဆေးပြီးမှသာ Public အဖြစ် မြှင့်တင်ပါ။

## ပြဿနာဖြေရှင်းခြင်း (Troubleshooting) {#troubleshooting}

**စွမ်းရည်တစ်ခု စာရင်းထဲ မပေါ်ခြင်း**
- စွမ်းရည်ကို တပ်ဆင်ပြီး အသက်ဝင်ကြောင်း စစ်ဆေးပါ။
- plugin/addon ကို ဖွင့်ထားကြောင်း စစ်ဆေးပါ။
- စာမျက်နှာကို Refresh လုပ်ပါ။

**စွမ်းရည်တစ်ခု၏ မြင်နိုင်မှုကို ပြောင်းလဲ၍ မရခြင်း**
- စီမံခန့်ခွဲသူ ခွင့်ပြုချက်ရှိကြောင်း စစ်ဆေးပါ။
- စွမ်းရည်ကို plugin တစ်ခုက ပိတ်ထားခြင်း ရှိမရှိ စစ်ဆေးပါ။
- ပဋိပက္ခဖြစ်စေနိုင်သော plugin များကို ပိတ်ကြည့်ပါ။

**အဖော်အပယ်တစ်ဦး စွမ်းရည်တစ်ခုကို ဝင်ရောက်ခွင့် မရခြင်း**
- အဖော်အပယ်သည် ခွင့်ပြုစာရင်းတွင် ပါဝင်ကြောင်း စစ်ဆေးပါ။
- စွမ်းရည်၏ မြင်နိုင်မှုသည် Partner အဖြစ် သတ်မှတ်ထားကြောင်း စစ်ဆေးပါ။
- အဖော်အပယ်၏ credentials များ မှန်ကန်ကြောင်း အတည်ပြုပါ။
- authentication အမှားများအတွက် API logs များကို စစ်ဆေးပါ။

**Unclassified စွမ်းရည်များ ဆက်လက် ပေါ်နေခြင်း**
- Admin အသိပေးချက်တွင် ပြန်လည်သုံးသပ်ပြီး အမျိုးအစားခွဲခြားပါ။
- သင့်အမျိုးအစားခွဲခြားမှု သိမ်းဆည်းပြီးကြောင်း စစ်ဆေးပါ။
- စွမ်းရည်ကို ပံ့ပိုးပေးသည့် plugin သည် နောက်ဆုံးပေါ်ဖြစ်ကြောင်း စစ်ဆေးပါ။

## Legacy Mode မှ ပြောင်းရွှေ့ခြင်း (Migration from Legacy Mode) {#migration-from-legacy-mode}

Superdav AI Agent ၏ ပိုမိုအစောပိုင်းဗားရှင်းမှ အဆင့်မြှင့်နေပါက-

- ရှိပြီးသား စွမ်းရည်အားလုံးကို အလိုအလျောက် Public အဖြစ် အမျိုးအစားခွဲခြားပေးထားသည်
- လိုအပ်သလို ပြန်လည်သုံးသပ်ပြီး ချိန်ညှိပါ။
- ဝင်ရောက်ခွင့်ကို ကန့်သတ်လိုခြင်း မရှိပါက ဘာမှလုပ်ရန်မလိုပါ။

Native Abilities API integration သို့ ပြောင်းလဲခြင်းဆိုင်ရာ အသေးစိတ်အချက်များအတွက် **Third-Party Mode Migration** ကို ကြည့်ရှုပါ။

## နောက်အဆင့်များ (Next Steps) {#next-steps}

စွမ်းရည် မြင်နိုင်မှုကို စီမံခန့်ခွဲပြီးနောက်-

၁။ **သင့်စွမ်းရည်များကို ပြန်လည်သုံးသပ်ပါ**: တပ်ဆင်ထားသော စွမ်းရည်အားလုံးကို စစ်ဆေးပါ။
၂။ **Unclassified စွမ်းရည်များကို အမျိုးအစားခွဲခြားပါ**: Admin အသိပေးချက်များကို တုံ့ပြန်ပါ။
၃။ **အဖော်အပယ်များကို စနစ်ထည့်ပါ**: Partner မြင်နိုင်မှု အသုံးပြုပါက အဖော်အပယ်များ ထည့်ပါ။
၄။ **အသုံးပြုမှုကို စောင့်ကြည့်ပါ**: မည်သည့် စွမ်းရည်များကို အများဆုံး အသုံးပြုသည်ကို စောင့်ကြည့်ပါ။
၅။ **ဆုံးဖြတ်ချက်များကို မှတ်တမ်းတင်ပါ**: မြင်နိုင်မှုအဆင့်တစ်ခုစီကို အဘယ်ကြောင့် ရွေးချယ်ခဲ့သည်ကို မှတ်စုထားပါ။
