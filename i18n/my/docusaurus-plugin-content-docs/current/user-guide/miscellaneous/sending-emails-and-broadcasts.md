# အီးမေးလ်များနှင့် Broadcast များ ပေးပို့ခြင်း (v2)

_**အရေးကြီးသော မှတ်ချက်: ဤဆောင်းပါးသည် Ultimate Multisite version 2.x ကို ရည်ညွှန်းပါသည်။**_

Ultimate Multisite တွင် သင့်ဖောက်သည်များနှင့် ဆက်သွယ်နိုင်စေသော feature တစ်ခုပါဝင်ပြီး၊ သတ်မှတ်ထားသော user တစ်ဦး သို့မဟုတ် user အုပ်စုတစ်ခုထံ အီးမေးလ်ပေးပို့ခြင်းအပြင် ၎င်းတို့၏ admin dashboard တွင် ကြေညာချက်များ broadcast လုပ်ခြင်းတို့ကို ပြုလုပ်နိုင်ပါသည်။

## Broadcast များဖြင့် သင့်ဖောက်သည်များ၏ dashboard တွင် admin notice များ ထည့်သွင်းခြင်း

Ultimate Multisite broadcast feature ကို အသုံးပြုပြီး သင့် user များ၏ subsite admin dashboard တွင် **admin notice** များ ထည့်သွင်းနိုင်ပါသည်။

စနစ်ပြုပြင်ထိန်းသိမ်းခြင်း သို့မဟုတ် လက်ရှိ user များကို ထုတ်ကုန်အသစ်များ၊ ဝန်ဆောင်မှုအသစ်များ ကမ်းလှမ်းခြင်းကဲ့သို့သော ကြေညာချက်များ ပြုလုပ်လိုသည့်အခါ ဤအရာသည် အလွန်အသုံးဝင်ပါသည်။ Admin notice သည် သင့် user ၏ dashboard တွင် ဤကဲ့သို့ ပေါ်လာမည်ဖြစ်သည်။

![ဖောက်သည် dashboard တွင် ပြသထားသော Admin notice broadcast](/img/admin/broadcasts-list.png)

Admin notice စတင်ရန် သင်၏ network admin dashboard သို့သွားပြီး **Ultimate Multisite** menu အောက်တွင် **Broadcasts** option ကို တွေ့ရှိနိုင်ပါသည်။

![Ultimate Multisite admin ရှိ Broadcasts menu](/img/admin/broadcasts-list.png)

ဤစာမျက်နှာမှ အပေါ်ရှိ **Add Broadcast** ခလုတ်ကို နှိပ်ပါ။

ထိုအခါ Add broadcast modal window ပေါ်လာပြီး မည်သည့်အမျိုးအစား broadcast ပေးပို့လိုသည်ကို ရွေးချယ်နိုင်ပါသည်။

**Message** ကို ရွေးချယ်ပြီး **Next Step** ခလုတ်ကို နှိပ်ပါ။

![Message အမျိုးအစား ရွေးချယ်ထားသော Add broadcast modal](/img/admin/broadcasts-list.png)

နောက် window တွင် **Target customer** သို့မဟုတ် **Target product** ကို မေးမြန်းပါလိမ့်မည်။ User တစ်ဦးထက်ပို၍ သို့မဟုတ် product တစ်ခုထက်ပို၍ ရွေးချယ်နိုင်သည်ကို သတိပြုပါ။

User account သို့မဟုတ် product ကို ရှာဖွေရန် field အတွင်း keyword စရိုက်ရန် လိုအပ်ပါသည်။

**Message type** field အောက်တွင် notice ၏အရောင်ကို ရွေးချယ်နိုင်ပါသည်။ ဤအရာသည် သင့်မက်ဆေ့ချ်၏ အရေးပေါ်အနေအထားကို ပေါ်လွင်စေမည်ဖြစ်သည်။

ထို့နောက် **Next Step** ကို နှိပ်နိုင်ပါသည်။

![Broadcast အတွက် Target customer နှင့် product ရွေးချယ်ခြင်း](/img/admin/broadcasts-list.png)

နောက် window တွင် subject နှင့် user များထံ broadcast လုပ်လိုသော content/message ကို ရိုက်ထည့်ပြီး သင့်မက်ဆေ့ချ်ကို စတင်ရေးသားနိုင်ပါသည်။

![Broadcast message subject နှင့် content editor](/img/admin/broadcasts-list.png)

သင့်မက်ဆေ့ချ်ကို ဖန်တီးပြီးနောက် **Send** ခလုတ်ကို နှိပ်နိုင်ပါသည်။

ဒါပဲဖြစ်ပါတယ်။ Admin notice သည် သင့် user ၏ dashboard တွင် ချက်ချင်းပေါ်လာမည်ဖြစ်သည်။

## သင့်ဖောက်သည်များထံ အီးမေးလ်များ ပေးပို့ခြင်း

Ultimate Multisite broadcast feature ကို အသုံးပြုပြီး သင့် user များထံ အီးမေးလ်ပေးပို့နိုင်ပါသည်။ သတ်မှတ်ထားသော user များကိုသာ အီးမေးလ်ပေးပို့ခြင်း သို့မဟုတ် ၎င်းတို့ subscribe လုပ်ထားသော product သို့မဟုတ် plan အပေါ်အခြေခံ၍ သတ်မှတ်ထားသော user အုပ်စုကို target လုပ်ခြင်း option များရှိပါသည်။

Email broadcast စတင်ရန် သင်၏ network admin dashboard သို့သွားပြီး Ultimate Multisite menu အောက်တွင် Broadcast option ကို တွေ့ရှိနိုင်ပါသည်။

![Ultimate Multisite admin ရှိ Broadcasts page](/img/admin/broadcasts-list.png)

ဤစာမျက်နှာမှ အပေါ်ရှိ **Add broadcast** ခလုတ်ကို နှိပ်ပါ။

ထိုအခါ Add broadcast modal window ပေါ်လာပြီး မည်သည့်အမျိုးအစား broadcast ပေးပို့လိုသည်ကို ရွေးချယ်နိုင်ပါသည်။ **Email** ကို ရွေးချယ်ပြီး **Next Step** ခလုတ်ကို နှိပ်ပါ။

![Email အမျိုးအစား ရွေးချယ်ထားသော Add broadcast modal](/img/admin/broadcasts-list.png)

နောက် window တွင် **Target customer** သို့မဟုတ် **Target product** ကို မေးမြန်းပါလိမ့်မည်။ User တစ်ဦးထက်ပို၍ သို့မဟုတ် product တစ်ခုထက်ပို၍ ရွေးချယ်နိုင်သည်ကို သတိပြုပါ။

User account သို့မဟုတ် product ကို ရှာဖွေရန် field အတွင်း keyword စရိုက်ရန် လိုအပ်ပါသည်။

သင့် target audience ကို ရွေးချယ်ပြီးသည်နှင့် **Next Step** ကို နှိပ်နိုင်ပါသည်။

![Email broadcast အတွက် Target customer နှင့် product ရွေးချယ်ခြင်း](/img/admin/broadcasts-list.png)

နောက် window တွင် subject နှင့် user များထံ ပေးပို့လိုသော content/message ကို ရိုက်ထည့်ပြီး သင့်အီးမေးလ်ကို စတင်ရေးသားနိုင်ပါသည်။

![Email broadcast subject နှင့် content editor](/img/admin/broadcasts-list.png)

သင့်မက်ဆေ့ချ်ကို ဖန်တီးပြီးနောက် **Send** ခလုတ်ကို နှိပ်နိုင်ပါသည်။

Broadcast feature ကို အသုံးပြု၍ သင့် end-user များထံ အီးမေးလ်ပေးပို့ရန် ဤမျှလွယ်ကူပါသည်။

## System emails

Ultimate Multisite ရှိ System emails များသည် registration၊ payment၊ domain mapping စသည်တို့ကဲ့သို့သော လုပ်ဆောင်ချက်များပြီးနောက် စနစ်မှ အလိုအလျောက်ပေးပို့သော **automatic notification** များဖြစ်ပါသည်။ ဤအီးမေးလ်များကို Ultimate Multisite settings မှ တည်းဖြတ်ခြင်း သို့မဟုတ် ပြုပြင်ခြင်းများ ပြုလုပ်နိုင်ပါသည်။ အခြား Ultimate Multisite installation တစ်ခုမှ ရှိပြီးသား settings များကို reset လုပ်ခြင်းနှင့် import လုပ်ခြင်းတို့ကို ခွင့်ပြုသည့် feature လည်း ပါဝင်ပါသည်။

### Reset လုပ်ခြင်းနှင့် Import လုပ်ခြင်း

Ultimate Multisite version အသစ်များနှင့် add-on များသည် အချိန်နှင့်အမျှ အီးမေးလ်အသစ်များ register လုပ်နိုင်ပြီး လုပ်ဆောင်မည်ဖြစ်သည်။

ပဋိပက္ခများနှင့် အခြားပြဿနာများကို ကာကွယ်ရန် **သတ်မှတ်ထားသော feature တစ်ခု မှန်ကန်စွာအလုပ်လုပ်ရန် အရေးကြီးမှသာလွဲ၍ အီးမေးလ် template အသစ်များကို သင့် install တွင် System Emails အဖြစ် အလိုအလျောက်ထည့်သွင်းမည်မဟုတ်ပါ**။

သို့သော် super admin များနှင့် agent များသည် importer tool မှတစ်ဆင့် ဤ register လုပ်ထားသော အီးမေးလ်အသစ်များကို import လုပ်နိုင်ပါသည်။ ထိုလုပ်ငန်းစဉ်သည် အီးမေးလ် template အသစ်၏ content နှင့် configuration ဖြင့် system email အသစ်တစ်ခု ဖန်တီးပေးမည်ဖြစ်ပြီး၊ super admin သည် လိုအပ်သလို ပြုပြင်မွမ်းမံခြင်း သို့မဟုတ် မူလအတိုင်း ထားရှိခြင်းတို့ ပြုလုပ်နိုင်ပါသည်။

#### System emails များ import လုပ်နည်း

သင့် Ultimate Multisite Settings page သို့သွားပြီး **Emails** tab သို့ သွားပါ။

![Ultimate Multisite settings ရှိ Emails tab](/img/config/settings-emails.png)

ထို့နောက် sidebar တွင် **Customize System Emails** ခလုတ်ကို နှိပ်ပါ။

![Sidebar ရှိ Customize System Emails ခလုတ်](/img/config/settings-emails.png)

System Emails page တွင် အပေါ်ဘက်၌ **Reset & Import** action ခလုတ်ကို တွေ့ရမည်ဖြစ်သည်။ ထိုခလုတ်ကို နှိပ်လျှင် import နှင့် reset modal window ပွင့်လာမည်ဖြစ်သည်။

![System Emails page ရှိ Reset and Import action ခလုတ်](/img/config/settings-emails.png)

ထို့နောက် Import Emails options ကို toggle လုပ်၍ import လုပ်နိုင်သော system emails များကို ကြည့်ရှုနိုင်ပါသည်။

![ရရှိနိုင်သော system emails များ ပြသထားသည့် Import Emails options](/img/config/settings-emails.png)

#### System Emails များ Reset လုပ်ခြင်း

တစ်ခါတစ်ရံတွင် သင်သတ်မှတ်ထားသော email template တစ်ခုအတွက် ပြုလုပ်ခဲ့သော ပြောင်းလဲမှုများသည် သင့်အတွက် အဆင်မပြေတော့သည်ကို သိရှိလာပြီး ၎င်းကို **မူလအခြေအနေ** သို့ reset လုပ်လိုပါလိမ့်မည်။

ထိုသို့သောအခြေအနေတွင် ရွေးချယ်စရာနှစ်ခုရှိပါသည်: system email ကို ရိုးရိုးဖျက်ပစ်ပြီး ပြန်လည် import လုပ်နိုင်ပါသည် (အထက်ပါ ညွှန်ကြားချက်များကို အသုံးပြု၍) - သို့သော် ဤနည်းလမ်းသည် send metrics နှင့် အခြားအရာများကို ဖျက်ပစ်မည်ဖြစ်သောကြောင့် အနည်းဆုံး နှစ်သက်ဖွယ်ဖြစ်ပါသည်။

သို့မဟုတ် **Reset & Import tool** ကို အသုံးပြုပြီး ထို email template ကို reset လုပ်နိုင်ပါသည်။

Email template တစ်ခုကို reset လုပ်ရန် Reset & Import tool သို့ရောက်သည်အထိ အထက်ပါအဆင့်များကို လိုက်နာပြီး **Reset** option ကို toggle လုပ်ကာ မူလ content သို့ ပြန်လည် reset လုပ်လိုသော emails များကို ရွေးချယ်နိုင်ပါသည်။

![Email templates များကို မူလအခြေအနေသို့ ပြန်လည်ရောက်ရှိစေရန် Reset option](/img/config/settings-emails.png)
