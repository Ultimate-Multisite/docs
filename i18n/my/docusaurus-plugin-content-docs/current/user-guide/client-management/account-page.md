# သင့်ဖောက်သည်၏ Account စာမျက်နှာ (v2)

_**အရေးကြီးသော မှတ်ချက်- ဤဆောင်းပါးသည် Ultimate Multisite ဗားရှင်း 2.x ကို ရည်ညွှန်းပါသည်။**_

ဖောက်သည်များသည် သင့်ကွန်ရက်ပေါ်တွင် plan တစ်ခုကို subscribe လုပ်သောအခါ၊ ၎င်းတို့သည် ဝဘ်ဆိုက်တစ်ခုနှင့် ၎င်း၏ dashboard ကို ရရှိပြီး ငွေပေးချေမှုများ၊ membership များ၊ domain များ၊ plan ကန့်သတ်ချက်များ စသည်တို့နှင့်ပတ်သက်သော အရေးကြီးအချက်အလက်များကို ကြည့်ရှုနိုင်ပါသည်။

ဤသင်ခန်းစာတွင်၊ ဖောက်သည်၏ account စာမျက်နှာကို လမ်းညွှန်ပြသပေးမည်ဖြစ်ပြီး သင့်ဖောက်သည်များသည် ၎င်းအတွင်းတွင် ဘာများကို ကြည့်ရှုနိုင်ပြီး ဘာများလုပ်ဆောင်နိုင်သည်ကို သင်တွေ့မြင်ရမည်ဖြစ်သည်။

## Account စာမျက်နှာ

Account စာမျက်နှာကို သင့်ဖောက်သည်၏ dashboard အတွင်းရှိ **Account** ကို နှိပ်၍ ဝင်ရောက်နိုင်ပါသည်။

![ဖောက်သည် dashboard အတွင်းရှိ Account menu](/img/admin/dashboard.png)

![ဖောက်သည် account စာမျက်နှာ အကျဉ်းချုပ်](/img/admin/dashboard.png)

ဖောက်သည်တစ်ဦး နှိပ်လိုက်သောအခါ၊ ၎င်းတို့၏ membership၊ ငွေတောင်းခံလိပ်စာ၊ ပြေစာများ၊ domain များ၊ ဆိုက်ကန့်သတ်ချက်များ အကျဉ်းချုပ်ကို မြင်ရမည်ဖြစ်ပြီး **Site Template** ကိုလည်း ပြောင်းလဲနိုင်မည်ဖြစ်သည် **(သင့်ကွန်ရက်တွင် ခွင့်ပြုထားပါက)**။

၎င်းတို့သည် membership ကို အခြား plan တစ်ခုသို့ ပြောင်းလဲနိုင်သည်၊ သို့မဟုတ် သင်ပေးနေသော အခြား package သို့မဟုတ် service တစ်ခုကို ဝယ်ယူနိုင်ပါသည်။ အပိုင်းတစ်ခုချင်းစီကို သီးခြားကြည့်ကြပါစို့။

### သင့် Membership အကျဉ်းချုပ်-

သင့်ဖောက်သည်များ၏ ဝဘ်ဆိုက်အမည် အောက်ရှိ ပထမဆုံး block သည် ၎င်းတို့၏ လက်ရှိ plan နှင့် ၎င်းနှင့်အတူ ဝယ်ယူထားသော service များ/package များ၏ အကျဉ်းချုပ်ကို ပြသပါသည်။ ဤ block တွင် membership နံပါတ်၊ ၎င်းအတွက် ကနဦးပေးချေသောပမာဏ၊ plan နှင့် service/package တစ်ခုစီ၏ တန်ဖိုးနှင့် ဤ membership အတွက် မည်မျှ ငွေတောင်းခံခဲ့သည်ကို ပြသပါသည်။ membership သည် **Active**၊ **Expired** သို့မဟုတ် **Canceled** ဖြစ်သည်ကိုလည်း ၎င်းတို့ မြင်နိုင်ပါသည်။

![Plan, ပမာဏနှင့် ငွေတောင်းခံအသေးစိတ်များပြသသော Membership အကျဉ်းချုပ်](/img/admin/memberships-list.png)

ဤ block အောက်တွင်၊ သင့်ဖောက်သည်များသည် **About This Site** နှင့် **Site Limits** block များကို မြင်ရနိုင်ပါသည်။ ဤ block များသည် ၎င်းတို့၏ plan နှင့်ပါလာသော ကန့်သတ်ချက်အားလုံးကို ပြသပါသည်- disk space၊ post များ၊ page များ၊ visit များ စသည်တို့ဖြစ်သည်။ ဤကန့်သတ်ချက်များကို **Ultimate Multisite > Products** ရှိ plan စာမျက်နှာတစ်ခုစီတွင် ပြင်ဆင်သတ်မှတ်နိုင်ပါသည်။

![Plan ကန့်သတ်ချက်များပြသသော About This Site နှင့် Site Limits block များ](/img/admin/dashboard.png)

**Your Membership** ၏ ညာဘက်တွင်၊ ဖောက်သည်များသည် **Change** ကို နှိပ်နိုင်ပါသည်။ ၎င်းသည် ရရှိနိုင်သော plan များနှင့် package များ/service များအားလုံးကို ပြသမည်ဖြစ်သည်။ ၎င်းတို့သည် အခြား plan တစ်ခုကို ရွေးချယ်ပါက၊ ထို plan ၏ ကန့်သတ်ချက်များသည် membership ၏ လက်ရှိကန့်သတ်ချက်များကို အစားထိုးမည်ဖြစ်သည် - downgrade သို့မဟုတ် upgrade ဖြစ်စေ အကျုံးဝင်ပါသည်။

ယခု၊ သင့်ဖောက်သည်များသည် ဤလက်ရှိ membership အတွက် package များ သို့မဟုတ် service များ ဝယ်ယူရန် ရွေးချယ်ပါက - ထပ်တိုး disk space သို့မဟုတ် visit များကဲ့သို့ - လက်ရှိ membership သည် မပြောင်းလဲဘဲ package အသစ်များသာ ၎င်းတွင် ထည့်သွင်းမည်ဖြစ်သည်။

ဤ membership ပြောင်းလဲရေးစာမျက်နှာတွင် coupon code များ ထည့်သွင်း၍မရကြောင်း သတိပြုပါ။ ဖောက်သည်သည် ပထမ membership ဝယ်ယူစဉ်တွင် coupon code သုံးခဲ့ပါက၊ ထို code သည် ဤ membership အသစ်တွင်လည်း အကျုံးဝင်မည်ဖြစ်သည်။

### ငွေတောင်းခံလိပ်စာ အပ်ဒိတ်လုပ်ခြင်း-

Account စာမျက်နှာတွင်၊ သင့်ဖောက်သည်များသည် ၎င်းတို့၏ ငွေတောင်းခံလိပ်စာကိုလည်း အပ်ဒိတ်လုပ်နိုင်ပါသည်။ _Billing Address_ ဘေးရှိ **Update** ကို နှိပ်ရုံသာဖြစ်သည်။

![Update ခလုတ်ပါသော Billing Address အပိုင်း](/img/admin/customers-list.png)

သင့်ဖောက်သည်ထံ ဝင်းဒိုးအသစ်တစ်ခု ပေါ်လာမည်ဖြစ်သည်။ လိပ်စာအသစ်ဖြည့်ပြီး _Save Changes_ ကို နှိပ်ရုံသာဖြစ်သည်။

![ငွေတောင်းခံလိပ်စာ အပ်ဒိတ်ဖောင်](/img/admin/customers-list.png)

### Site Template ပြောင်းလဲခြင်း-

သင့်ဖောက်သည်များအား site template များ ပြောင်းလဲခွင့်ပြုရန်၊ **Ultimate Multisite > Settings > Sites** သို့သွားပြီး **Allow Template Switching** ရွေးချယ်မှုကို ဖွင့်ပါ။

ထို့အပြင်၊ **Ultimate Multisite > Products** တွင် သင့် plan များကို ရွေးချယ်ပြီး **Site Templates** tab သို့ သွားပါ။ **Allow Site Templates** ရွေးချယ်မှုကို ဖွင့်ထားကြောင်းနှင့် **Site Template Selection Mode** တွင် **Choose Available Site Templates** ရွေးချယ်မှုကို ရွေးချယ်ထားကြောင်း သေချာပါစေ။

![Template selection mode ပါသော Product site templates tab](/img/config/product-site-templates.png)

သင့်ဝဘ်ဆိုက်ပေါ်ရှိ ရရှိနိုင်သော site template များအားလုံးကို မြင်ရနိုင်မည်ဖြစ်သည်။ ဤ plan အောက်တွင် subscribe လုပ်ထားသော သင့်ဖောက်သည်များအတွက် မည်သည့်အရာများကို ရရှိစေလိုပြီး မည်သည့်အရာများကို မရရှိစေလိုကြောင်း ရွေးချယ်ပါ။ ဤရွေးချယ်မှုများသည် checkout form ကိုလည်း သက်ရောက်မှုရှိသောကြောင့်၊ **Not Available** အဖြစ် ရွေးချယ်ထားသော template မည်သည်မဆို ဤ plan အတွက် မှတ်ပုံတင်စာမျက်နှာတွင် ပေါ်မည်မဟုတ်ကြောင်း သတိပြုပါ။

ယခု သင့်ဖောက်သည်များသည် ၎င်းတို့၏ account စာမျက်နှာအတွင်း **Change Site Template** ကို နှိပ်နိုင်ပါပြီ။

![Account စာမျက်နှာပေါ်ရှိ Change Site Template ခလုတ်](/img/admin/dashboard.png)  
ဤ plan အတွက် ရရှိနိုင်သော Site Template များစာရင်း သင့်ဖောက်သည်ထံ ပေါ်လာမည်ဖြစ်သည်။

![Plan အတွက် ရရှိနိုင်သော site template များစာရင်း](/img/config/site-templates-list.png)

ပြောင်းလဲလိုသည့်အရာကို ရွေးချယ်ပြီးနောက်၊ ပြောင်းလဲမှုကို အတည်ပြုရန် တောင်းဆိုခံရမည်ဖြစ်သည်။

![Site template ပြောင်းလဲမှု အတည်ပြုချက် dialog](/img/admin/dashboard.png)

အတည်ပြုချက်ကို ဖွင့်ပြီး **Process Switch** ကို နှိပ်ပြီးနောက်၊ site template အသစ်ကို သင့်ဖောက်သည်၏ ဝဘ်ဆိုက်တွင် အသုံးပြုမည်ဖြစ်သည်။

### Custom Domain များ ထည့်သွင်းခြင်း-

သင့်ဖောက်သည်များသည် ၎င်းတို့၏ account စာမျက်နှာတွင် ဤ plan အတွက် custom domain တစ်ခု ထည့်သွင်းနိုင်မည်ဖြစ်သည်။ သင့်ဖောက်သည်များအား custom domain များ သုံးခွင့်ပြုရန်၊ **Ultimate Multisite > Settings > Domain Mapping** သို့ သွားပါ။

**Enable Domain Mapping** ရွေးချယ်မှုကို ဖွင့်ပါ။ ၎င်းသည် သင့်ဖောက်သည်များအား ကွန်ရက်အဆင့်တွင် custom domain များ သုံးခွင့်ပြုမည်ဖြစ်သည်။

product အလိုက် domain mapping ကို ဖွင့်ထားခြင်း ရှိ/မရှိ စစ်ဆေးရန် မမေ့ပါနှင့် - အဘယ်ကြောင့်ဆိုသော် product တစ်ခုကို သင့်ဖောက်သည်များ custom domain များ သုံးခွင့်မပြုရန် ကန့်သတ်နိုင်သောကြောင့်ဖြစ်သည်။

**Ultimate Multisite > Products** သို့ သွားပါ။ သင်ရွေးချယ်လိုသော plan ကို ရွေးချယ်ပြီး **Custom Domains** tab သို့ သွားပါ။ **Allow Custom Domains** ရွေးချယ်မှုကို ဖွင့်ပါ။

![Allow Custom Domains toggle ပါသော Custom Domains tab](/img/config/product-custom-domains.png)

၎င်းသည် ဤ plan သီးသန့်ကို subscribe လုပ်ထားသော ဖောက်သည်အားလုံးအား custom domain များ သုံးခွင့်ပြုမည်ဖြစ်သည်။ ယခု Account စာမျက်နှာတွင်၊ သင့်ဖောက်သည်များသည် **Add Domain** ကို နှိပ်၍ custom domain တစ်ခု ထည့်သွင်းနိုင်ပါပြီ။

![Account စာမျက်နှာပေါ်ရှိ Add Domain ခလုတ်](/img/admin/domains-list.png)

ဖွင့်လာသော ပထမဆုံးဝင်းဒိုးသည် သင့်ဖောက်သည်များအား ဤ custom domain သည် သင့်ကွန်ရက်ပေါ်တွင် အလုပ်လုပ်စေရန် ၎င်းတို့၏ DNS record များကို မည်သို့ အပ်ဒိတ်လုပ်ရမည်ကို ညွှန်ကြားသော မက်ဆေ့ချ်တစ်ခု ပြသမည်ဖြစ်သည်။

![Custom domain ထည့်သွင်းသောအခါ ပြသသော DNS ညွှန်ကြားချက်များ](/img/admin/domains-list.png)

ဤမက်ဆေ့ချ်ကို (သင်က) **Ultimate Multisite > Settings > Domain Mapping > Add New Domain Instructions** တွင် တည်းဖြတ်နိုင်ပါသည်။

![Domain Mapping ရှိ Add New Domain Instructions setting](/img/config/settings-domain-mapping.png)

**Next Step** ကို နှိပ်ပြီးနောက်၊ သင့်ဖောက်သည်များသည် ၎င်းတို့၏ custom domain အမည်ကို ထည့်သွင်းနိုင်ပြီး ဤ custom domain သည် primary ဖြစ်မဖြစ် ရွေးချယ်နိုင်ပါသည်။ သင့်ဖောက်သည်များသည် ၎င်းတို့၏ ဝဘ်ဆိုက်များအတွက် custom domain တစ်ခုထက်ပိုပြီး သုံးနိုင်သောကြောင့်၊ မည်သည့်အရာကို primary ဖြစ်စေလိုကြောင်း ရွေးချယ်နိုင်ကြောင်း သတိပြုပါ။

![Primary domain ရွေးချယ်မှုပါသော Custom domain အမည် ထည့်သွင်းခြင်း](/img/admin/domains-list.png)

**Add Domain** ကို နှိပ်ပြီးနောက်၊ domain ကို သင့်ဖောက်သည်၏ account တွင် ထည့်သွင်းမည်ဖြစ်သည်။ ယခု ၎င်းတို့ လုပ်ဆောင်ရမည့်အရာမှာ ၎င်းတို့၏ domain registrar တွင် ဤ custom domain ၏ DNS record များကို ပြောင်းလဲရန်သာဖြစ်သည်။

### Password ပြောင်းလဲခြင်း-

Account dashboard အတွင်း၊ သင့်ဖောက်သည်များသည် **Change Password** ကို နှိပ်၍ ၎င်းတို့၏ password ကိုလည်း ပြောင်းလဲနိုင်ပါသည်။

![Account စာမျက်နှာပေါ်ရှိ Change Password ခလုတ်](/img/admin/dashboard.png)

၎င်းသည် သင့်ဖောက်သည်များ ၎င်းတို့၏ လက်ရှိ password ကို ဖြည့်ပြီးနောက် သုံးလိုသော password အသစ်ကို ဖြည့်ရမည့် ဝင်းဒိုးအသစ်တစ်ခု ပြသမည်ဖြစ်သည်။

![လက်ရှိနှင့် password အသစ် field များပါသော Password ပြောင်းလဲခြင်း ဖောင်](/img/admin/dashboard.png)

### Danger Zone-

**Danger Zone** အပိုင်းတွင် ရွေးချယ်မှုနှစ်ခုလည်း ရှိပါသည်- **Delete Site** နှင့် **Delete Account**။ ဤလုပ်ဆောင်ချက်နှစ်ခုသည် ပြန်လည်ပြင်ဆင်၍မရသောကြောင့် Danger Zone အပိုင်းတွင် ထားရှိခြင်းဖြစ်သည်။ သင့်ဖောက်သည်များသည် ၎င်းတို့၏ ဝဘ်ဆိုက် သို့မဟုတ် account ကို ဖျက်လိုက်ပါက၊ ၎င်းတို့ကို ပြန်လည်ရယူ၍မရပါ။

![Delete Site နှင့် Delete Account ရွေးချယ်မှုများပါသော Danger Zone](/img/admin/dashboard.png)

သင့်ဖောက်သည်များသည် ဤရွေးချယ်မှုနှစ်ခုအနက် တစ်ခုခုကို နှိပ်ပါက၊ ဝဘ်ဆိုက် သို့မဟုတ် account ကို ဖယ်ရှားရန် ရွေးချယ်မှုကို ဖွင့်ရမည့် ဝင်းဒိုးတစ်ခု ပြသမည်ဖြစ်ပြီး ဤလုပ်ဆောင်ချက်ကို ပြန်ဖျက်၍မရကြောင်း သတိပေးခံရမည်ဖြစ်သည်။

![Site ဖျက်ခြင်း အတည်ပြုချက် dialog](/img/admin/dashboard.png)

![Account ဖျက်ခြင်း အတည်ပြုချက် dialog](/img/admin/dashboard.png)

၎င်းတို့သည် ဝဘ်ဆိုက်ကို ဖျက်ပါက၊ ၎င်းတို့၏ account နှင့် membership သည် မထိခိုက်ဘဲ ရှိနေမည်ဖြစ်သည်။ ၎င်းတို့၏ ဝဘ်ဆိုက်ပေါ်ရှိ အကြောင်းအရာအားလုံးကိုသာ ဆုံးရှုံးမည်ဖြစ်သည်။ ၎င်းတို့သည် account ကို ဖျက်ပါက၊ ဤ account နှင့်ပတ်သက်သော ဝဘ်ဆိုက်များ၊ membership များနှင့် အချက်အလက်အားလုံး ဆုံးရှုံးသွားမည်ဖြစ်သည်။
