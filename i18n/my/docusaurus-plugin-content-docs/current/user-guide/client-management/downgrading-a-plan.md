# အစီအစဉ်ကို အဆင့်လျှော့ခြင်း (v2) {#downgrading-a-plan-v2}

_**အရေးကြီး မှတ်ချက်: ဤဆောင်းပါးသည် Ultimate Multisite ဗားရှင်း 2.x ကို ရည်ညွှန်းထားသည်။**_

အစီအစဉ် သို့မဟုတ် စာရင်းသွင်းမှုကို အဆင့်လျှော့ခြင်းသည် သင့်ဖောက်သည်များ၏ ဘတ်ဂျက်ကန့်သတ်ချက်ရှိသည့်အခါ၊ သို့မဟုတ် ၎င်းတို့၏ ဆိုက်ခွဲကို လည်ပတ်ရန် အရင်းအမြစ်များစွာ မလိုအပ်တော့ဟု ဆုံးဖြတ်သည့်အခါ ပြုလုပ်နိုင်သော ပုံမှန်လုပ်ဆောင်ချက်တစ်ခုဖြစ်သည်။

## အစီအစဉ်ကို အဆင့်လျှော့နည်း {#how-to-downgrade-a-plan}

သင့်ဖောက်သည်များသည် ၎င်းတို့၏ ဆိုက်ခွဲ စီမံခန့်ခွဲရေး Dashboard သို့ ဝင်ရောက်ပြီး Account စာမျက်နှာအောက်ရှိ **ပြောင်းလဲရန်** ကို နှိပ်ခြင်းဖြင့် မည်သည့်အချိန်မဆို ၎င်းတို့၏ အစီအစဉ်ကို အဆင့်လျှော့နိုင်သည်။

![သင့်အသင်းဝင်မှု ကတ်နှင့် ပြောင်းလဲရန် ခလုတ်ပါသော ဖောက်သည် Account စာမျက်နှာ](/img/account-page/membership-change-button.png)

**ပြောင်းလဲရန်** ခလုတ်ကို နှိပ်ပြီးနောက် အသုံးပြုသူ/ဖောက်သည်ကို ၎င်းတို့၏ စာရင်းသွင်းမှုကို ပြောင်းလိုသည့် အစီအစဉ် ရွေးချယ်နိုင်သည့် ငွေရှင်းစာမျက်နှာသို့ ပြန်ညွှန်းပေးမည်ဖြစ်သည်။

![ဖောက်သည်ဘက်ရှိ အစီအစဉ် အဆင့်လျှော့ ရွေးချယ်စရာ စာမျက်နှာ](/img/account-page/downgrade-picker.png)

ဤဥပမာတွင် အစီအစဉ်ကို **Premium** မှ **Free** သို့ အဆင့်လျှော့နေပါသည်။

ဆက်လုပ်ရန် အသုံးပြုသူသည် **ငွေရှင်းခြင်း ပြီးမြောက်ရန်** ခလုတ်ကို နှိပ်ရန်သာ လိုအပ်သည်။ ထို့နောက် အသင်းဝင်မှုအတွက် ဆိုင်းငံ့ထားသည့် ပြောင်းလဲမှုအကြောင်း စာတန်းပြသထားသော Account စာမျက်နှာသို့ ပြန်ခေါ်သွားမည်ဖြစ်သည်။ ပြောင်းလဲမှုများသည် ဖောက်သည်၏ **နောက်ထပ် ဘီလ်ကောက်ခံမှု စက်ဝန်း** တွင် သက်ရောက်မည်ဖြစ်သည်။

![ဆိုင်းငံ့ထားသည့် အသင်းဝင်မှု ပြောင်းလဲမှု စာတန်းကို ပြသထားသော Account စာမျက်နှာ](/img/account-page/pending-change.png)

### အသုံးပြုသူတစ်ဦး အစီအစဉ်ကို အဆင့်လျှော့သည့်အခါ ဖြစ်ပျက်မည့်အရာများ {#what-happens-when-a-user-downgrades-their-plan}

အစီအစဉ်ကို အဆင့်လျှော့ခြင်းသည် အသုံးပြုသူ၏ ဆိုက်ခွဲရှိ လက်ရှိဖွဲ့စည်းမှုကို မပြောင်းလဲကြောင်း မှတ်သားထားရန် အရေးကြီးသည်။

ဆိုက်ပုံစံခွက်ကို ပြောင်းလဲခြင်းသည် ဆိုက်ခွဲကို လုံးဝဖျက်ပြီး ပြန်လည်သတ်မှတ်မည်ဖြစ်သောကြောင့် ဆိုက်ပုံစံခွက်ကို အလိုအလျောက် မပြောင်းလဲပါ။ မလိုအပ်သော ဒေတာဆုံးရှုံးမှုကို ရှောင်ရှားရန် ဖြစ်သည်။ ထို့ကြောင့် ပို့စ်များမှလွဲ၍ disk space၊ theme များ၊ plugin များ စသည်တို့သည် မပျက်မစီး ရှိနေမည်ဖြစ်သည်။

အစီအစဉ်တစ်ခုချင်းစီအောက်တွင် သင်သတ်မှတ်ထားသော ကန့်သတ်ချက်များနှင့် ခွဲတမ်းများသည် သင့်အဓိကစိုးရိမ်ချက် ဖြစ်မည်ကို ကျွန်ုပ်တို့ နားလည်ပါသည်။ သို့သော် ၎င်း၏ဖွဲ့စည်းမှုများထဲမှ တစ်ခုခုကို ဖျက်ခြင်း သို့မဟုတ် ပြောင်းလဲခြင်း ပြုလုပ်ပါက အသုံးပြုသူ၏ ဆိုက်ခွဲအပေါ် ဖြစ်စေနိုင်သည့် ထိခိုက်မှုကိုလည်း ထည့်သွင်းစဉ်းစားရပါမည်။

အစီအစဉ်တွင် သတ်မှတ်ထားသော ကန့်သတ်ချက်ကို ကျော်လွန်နေသည့် ပို့စ်များအတွက် သင့်တွင် ရွေးချယ်စရာ ၃ မျိုးရှိသည်: **ပို့စ်များကို ရှိသည့်အတိုင်း ထားရန်** *,* **ပို့စ်များကို အမှိုက်ပုံးသို့ ရွှေ့ရန်** *,* သို့မဟုတ် **ပို့စ်များကို မူကြမ်းသို့ ရွှေ့ရန်** *.* ၎င်းကို Ultimate Multisite ဆက်တင်များအောက်တွင် သင် ပြင်ဆင်သတ်မှတ်နိုင်သည်။

![ပို့စ်ကန့်သတ်ချက်ကျော်လွန်သည့်အခါ ပြုမူပုံ ရွေးချယ်စရာများကို ပြသထားသော Network Admin ဆက်တင်များ ဆိုက်များ စာမျက်နှာ](/img/account-page/settings-sites-post-limit.png)

### ငွေပေးချေမှုတွင် ဖြစ်ပျက်မည့်အရာများ {#what-happens-to-the-payment}

ဗားရှင်း 2.0 တွင် အချိုးကျတွက်ချက်မှုနှင့် ပတ်သက်၍ ငွေပေးချေမှုအပေါ် ပြင်ဆင်ချိန်ညှိမှုများ မလိုအပ်တော့ပါ။

အကြောင်းမှာ စနစ်သည် အစီအစဉ်/အသင်းဝင်မှုအသစ် သက်ရောက်မည်မတိုင်မီ လက်ရှိအသင်းဝင်မှုကို **၎င်း၏ ဘီလ်ကောက်ခံမှု စက်ဝန်း ပြီးဆုံးသည်အထိ** စောင့်မည်ဖြစ်သောကြောင့် ဖြစ်သည်။ အသင်းဝင်မှုအသစ်အတွက် ဘီလ်ပမာဏအသစ်ကို နောက်ထပ် ဘီလ်ကောက်ခံမှု စက်ဝန်းတွင် အလိုအလျောက် အသုံးပြုပြီး ကောက်ခံမည်ဖြစ်သည်။
