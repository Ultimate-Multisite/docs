# အစီအစဉ်ကို အဆင့်နှိမ့်ချခြင်း (v2)

_**အရေးကြီးသော မှတ်ချက်: ဤဆောင်းပါးသည် Ultimate Multisite ဗားရှင်း 2.x အတွက်ဖြစ်ပါသည်။**_

အစီအစဉ် သို့မဟုတ် subscription ကို အဆင့်နှိမ့်ချခြင်းသည် သင့်ဖောက်သည်များ ဘတ်ဂျက်ကန့်သတ်ချက်ရှိသောအခါ သို့မဟုတ် သူတို့၏ subsite ကို လည်ပတ်ရန် resource များ အများကြီးမလိုအပ်တော့ဟု ဆုံးဖြတ်သောအခါ လုပ်လေ့ရှိသော လုပ်ဆောင်ချက်တစ်ခုဖြစ်ပါသည်။

## အစီအစဉ်ကို မည်သို့အဆင့်နှိမ့်ချမည်နည်း

သင့်ဖောက်သည်များသည် သူတို့၏ subsite admin dashboard ထဲသို့ ဝင်ရောက်ပြီး account စာမျက်နှာအောက်ရှိ **Change** ကို နှိပ်ခြင်းဖြင့် မည်သည့်အချိန်မဆို သူတို့၏အစီအစဉ်ကို အဆင့်နှိမ့်ချနိုင်ပါသည်။

![membership အောက်ရှိ Change ခလုတ်ပါသော Account စာမျက်နှာ](/img/admin/memberships-list.png)

**Change** ခလုတ်ကို နှိပ်လိုက်သောအခါ၊ အသုံးပြုသူ/ဖောက်သည်သည် သူတို့၏ subscription ကို ပြောင်းလဲလိုသော အစီအစဉ်ကို ရွေးချယ်နိုင်သည့် checkout စာမျက်နှာသို့ ပြန်လည်ညွှန်းပေးပါလိမ့်မည်။

![အဆင့်နှိမ့်ချရန် အစီအစဉ်ရွေးချယ်မှုများပြသည့် Checkout စာမျက်နှာ](/img/admin/memberships-list.png)

ဤဥပမာတွင်၊ အစီအစဉ်ကို **Premium** မှ **Free** သို့ အဆင့်နှိမ့်ချနေပါသည်။

ဆက်လက်ဆောင်ရွက်ရန် အသုံးပြုသူသည် **Complete Checkout** ခလုတ်ကို နှိပ်ရန်သာလိုပါသည်။ ထို့နောက် membership အတွက် ပြောင်းလဲမှုဆိုင်ရာ စောင့်ဆိုင်းနေသည့် မက်ဆေ့ချ်ပြသည့် account စာမျက်နှာသို့ ပြန်ခေါ်သွားပါလိမ့်မည်။ ပြောင်းလဲမှုများသည် ဖောက်သည်၏ **နောက်ငွေတောင်းခံသည့်အချိန်** တွင် အကျိုးသက်ရောက်ပါလိမ့်မည်။

![membership ပြောင်းလဲမှု စောင့်ဆိုင်းနေသည့် မက်ဆေ့ချ်ပြသည့် Account စာမျက်နှာ](/img/admin/memberships-list.png)

### အသုံးပြုသူ အစီအစဉ်ကို အဆင့်နှိမ့်ချသောအခါ ဘာဖြစ်မည်နည်း

အစီအစဉ်ကို အဆင့်နှိမ့်ချခြင်းသည် အသုံးပြုသူ၏ subsite ရှိ လက်ရှိ configuration ကို ပြောင်းလဲခြင်းမရှိကြောင်း မှတ်သားထားရန် အရေးကြီးပါသည်။

site template ကို အလိုအလျောက် ပြောင်းလဲခြင်းမရှိပါ၊ အကြောင်းမှာ site template ကို ပြောင်းလဲခြင်းသည် subsite ကို လုံးဝဖျက်ပြီး ပြန်လည်သတ်မှတ်ပါလိမ့်မည်။ ဤသည်မှာ မလိုအပ်သော data ဆုံးရှုံးမှုကို ရှောင်ရှားရန်ဖြစ်ပါသည်။ ထို့ကြောင့် disk space၊ themes၊ plugins စသည်တို့သည် posts မှလွဲ၍ မပြောင်းလဲဘဲ ရှိနေပါလိမ့်မည်။

အစီအစဉ်တစ်ခုစီအောက်တွင် သင်သတ်မှတ်ထားသော ကန့်သတ်ချက်များနှင့် quota များသည် သင်၏အဓိက စိုးရိမ်မှုဖြစ်ကြောင်း ကျွန်ုပ်တို့ နားလည်ပါသည်၊ သို့သော် configuration တစ်ခုခုကို ဖျက်ပစ်ခြင်း သို့မဟုတ် ပြောင်းလဲခြင်းသည် အသုံးပြုသူ၏ subsite ကို ထိခိုက်စေနိုင်သည့် ပျက်စီးမှုကို ကျွန်ုပ်တို့ စဉ်းစားရပါသည်။

အစီအစဉ်တွင် သတ်မှတ်ထားသော ကန့်သတ်ချက်ထက် ကျော်လွန်သော posts များအတွက်၊ သင့်တွင် ရွေးချယ်စရာ ၃ ခုရှိပါသည်: **Posts များကို ဒီအတိုင်းထားရှိခြင်း**၊ **Posts များကို အမှိုက်ပုံးသို့ ရွှေ့ခြင်း**၊ သို့မဟုတ် **Posts များကို draft သို့ ရွှေ့ခြင်း**။ ဤအရာကို Ultimate Multisite settings တွင် configure လုပ်နိုင်ပါသည်။

![Ultimate Multisite settings ရှိ Post ကန့်သတ်ချက်ကျော်လွန်မှု ရွေးချယ်စရာများ](/img/config/settings-sites.png)

### ငွေပေးချေမှုအတွက် ဘာဖြစ်မည်နည်း

ဗားရှင်း 2.0 တွင်၊ proration အရ ငွေပေးချေမှုတွင် ပြင်ဆင်မှုများ မလိုအပ်တော့ပါ။

ဤသည်မှာ စနစ်သည် အစီအစဉ်အသစ်/membership အသစ် အကျိုးသက်ရောက်မည့်အချိန်မတိုင်မီ လက်ရှိ membership ၏ **ငွေတောင်းခံသည့်အချိန် ပြီးဆုံးရန်** စောင့်ဆိုင်းသောကြောင့်ဖြစ်ပါသည်။ membership အသစ်အတွက် ငွေတောင်းခံမှုပမာဏအသစ်သည် နောက်ငွေတောင်းခံသည့်အချိန်တွင် အလိုအလျောက် အသုံးပြုပြီး ကောက်ခံပါလိမ့်မည်။
