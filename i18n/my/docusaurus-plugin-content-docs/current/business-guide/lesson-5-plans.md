# သင်ခန်းစာ ၅- သင်၏ အစီအစဉ်များကို ဒီဇိုင်းဆွဲခြင်း

သင်၏ အစီအစဉ်အဆင့်များသည် ဈေးနှုန်းသတ်မှတ်မှု အဆင့်များသက်သက် မဟုတ်ပါ— ၎င်းတို့သည် သင့်၏ အထူးကဏ္ဍရှိ ဖောက်သည်များ မည်သို့ အမှန်တကယ် လုပ်ဆောင်နေသည်ကို ထင်ဟပ်ပြသခြင်း ဖြစ်ပါသည်။ ဤသင်ခန်းစာတွင် သင်သည် ကစားသမားများ၏ အမှန်တကယ် လိုအပ်ချက်များနှင့် ကိုက်ညီသော ထုတ်ကုန်အဆင့်များကို ဒီဇိုင်းဆွဲသွားမှာ ဖြစ်ပါတယ်။

## ကျွန်ုပ်တို့ ဘယ်နေရာမှာ ရပ်ခဲ့သလဲ {#where-we-left-off}

FitSite တွင် template သုံးမျိုး (Studio Essential, Gym Pro, Fitness Chain) အသင့်ရှိနေပါပြီ။ ယခုအခါတွင် ဖောက်သည်များ မည်သည့်အရာများကို အသုံးပြုခွင့်ရမည်၊ မည်သည့်အရာများကို ပေးဆောင်ရမည်ကို ဆုံးဖြတ်ပေးမည့် အစီအစဉ်များကို ဖန်တီးရမှာ ဖြစ်ပါတယ်။

## ဖောက်သည် အုပ်စုများအလိုက် စဉ်းစားခြင်း {#thinking-in-customer-segments}

လူအများစု လုပ်မိတဲ့ အမှားကတော့ အစီအစဉ်တွေကို နည်းပညာဆိုင်ရာ အင်္ဂါရပ်တွေ (storage၊ bandwidth၊ စာမျက်နှာအရေအတွက်) ကို အခြေခံပြီး ဒီဇိုင်းဆွဲတာပါပဲ။ သင့်ရဲ့ အထူးကဏ္ဍရှိ ဖောက်သည်တွေက အဲဒီလို စကားလုံးတွေနဲ့ မစဉ်းစားကြပါဘူး။ အားကစားရုံပိုင်ရှင်တစ်ယောက်က သူတို့ရဲ့ လုပ်ငန်းကို လည်ပတ်ဖို့ ဘာတွေ လိုအပ်လဲဆိုတာကို စဉ်းစားကြတာပါ။

အားကစားစတူဒီယိုများအတွက် သဘာဝကျတဲ့ အုပ်စုသုံးခု ရှိပါတယ်-

| အုပ်စု | ဘယ်သူတွေလဲ | ဘာတွေ လိုအပ်လဲ |
|---------|-------------|----------------|
| **တစ်ဦးတည်း လေ့ကျင့်ပေးသူများ / စတူဒီယိုငယ်များ** | ဝန်ထမ်း ၁-၃ ဦး၊ နေရာတစ်ခုတည်း၊ စတင်လုပ်ကိုင်သူ | ကျွမ်းကျင်တဲ့ website တစ်ခု၊ class အချက်အလက်၊ ဆက်သွယ်ရန် form |
| **တည်ထောင်ပြီးသား အားကစားရုံများ** | ဝန်ထမ်း ၅-၂၀ ဦး၊ နေရာတစ်ခုတည်း၊ ကြီးထွားနေသူ | အပေါ်က အရာအားလုံးအပြင် booking၊ blog၊ gallery၊ custom domain |
| **အားကစားကွင်းဆက်များ** | နေရာများစွာ၊ တည်ငြိမ်တဲ့ brand ရှိသူ | အပေါ်က အရာအားလုံးအပြင် multi-site၊ နေရာစာမျက်နှာများ၊ ဝန်ထမ်းစာရင်း |

သင့်ရဲ့ အစီအစဉ်တွေဟာ စိတ်ကြိုက် အင်္ဂါရပ်အစုအဝေးတွေနဲ့ မကိုက်ဘဲ ဒီအုပ်စုတွေနဲ့ ကိုက်ညီရပါမယ်။

## Plan Group ကို စနစ်ထည့်ခြင်း {#setting-up-the-plan-group}

Plan group ကို အသုံးပြုခြင်းအားဖြင့် ဖောက်သည်များသည် FitSite plan မိသားစုအတွင်းမှာသာ upgrade သို့မဟုတ် downgrade လုပ်နိုင်မှာ ဖြစ်ပါတယ်။ အစီအစဉ်တစ်ခုစီရဲ့ **Up & Downgrades** tab မှာ-

၁။ **Plan Group** ကို အစီအစဉ် သုံးခုလုံးအတွက် "FitSite Plans" ဟု သတ်မှတ်ပါ။
၂။ **Product Order** ကို ၁ (Starter)၊ ၂ (Growth)၊ ၃ (Pro) ဟု သတ်မှတ်ပါ။

ဒါက ရှင်းလင်းတဲ့ upgrade လမ်းကြောင်းတစ်ခုကို ဖန်တီးပေးပါတယ်- Starter → Growth → Pro။

## Order Bumps များ ထည့်ခြင်း {#adding-order-bumps}

Order bumps ဆိုတာ checkout လုပ်တဲ့အချိန်မှာ ပေးရတဲ့ အပိုထပ်ပေါင်းရောင်းကုန်တွေ ဖြစ်ပါတယ်။ FitSite အတွက် အောက်ပါအရာတွေကို စဉ်းစားနိုင်ပါတယ်-

- **Extra Storage Pack** ($19/လ) -- အပို disk space ၅ GB
- **Priority Support** ($29/လ) -- ပိုမြန်တဲ့ တုံ့ပြန်မှုအချိန်များ
- **Additional Site** ($39/လ) -- သူတို့ရဲ့ အစီအစဉ်က ခွင့်ပြုတဲ့ထက် နေရာပိုလိုအပ်တဲ့ ဖောက်သည်များအတွက်

ဒီအရာတွေကို Ultimate Multisite မှာ **Package** အမျိုးအစား ထုတ်ကုန်တွေအဖြစ် ဖန်တီးပြီး သက်ဆိုင်ရာ အစီအစဉ်တွေနဲ့ ချိတ်ဆက်ပေးပါ။

## ဒီဖွဲ့စည်းပုံက ဘာကြောင့် အလုပ်ဖြစ်တာလဲ {#why-this-structure-works}

- **Starter** က စတင်ဝင်ရောက်မှု အတားအဆီးတွေကို ဖယ်ရှားပေးတယ် -- ဈေးနှုန်းသက်သာခြင်း၊ ရိုးရှင်းတဲ့ ပေးကမ်းမှု၊ လေ့ကျင့်ပေးသူတွေကို အမြန် online ဖြစ်စေတယ်
- **Growth** က အားကစားရုံတွေ တကယ်တောင်းဆိုတဲ့ အင်္ဂါရပ်တွေကို ထပ်ပေါင်းပေးတယ် -- booking၊ gallery၊ custom domains
- **Pro** က နေရာများစွာ ပံ့ပိုးမှု လိုအပ်တဲ့ အမြင့်တန်ဖိုး အုပ်စုကို ဝန်ဆောင်မှုပေးတယ်
- **Order bumps** က ဖောက်သည်တွေ အဓိက အစီအစဉ်တွေကို ရှုပ်ထွေးစေဘဲ စိတ်ကြိုက်ပြင်ဆင်ခွင့် ပေးတယ်
- **ရှင်းလင်းတဲ့ upgrade လမ်းကြောင်း** ဆိုတာက ဖောက်သည်တွေ ထွက်ခွာသွားမယ့်အစား သင့်နဲ့အတူ ကြီးထွားလာစေတယ်

## FitSite Network အခုချိန်အထိ {#the-fitsite-network-so-far}

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products
│   ├── FitSite Starter ($49/mo) → Studio Essential template
│   ├── FitSite Growth ($99/mo) → Studio Essential + Gym Pro templates
│   ├── FitSite Pro ($199/mo) → All templates
│   └── Order Bumps (Extra Storage, Priority Support, Additional Site)
└── Ready for checkout configuration (next lesson)
```

## ဒီသင်ခန်းစာမှာ ဘာတွေ တည်ဆောက်ခဲ့သလဲ {#what-we-built-this-lesson}

- အမှန်တကယ် fitness လုပ်ငန်း အုပ်စုတွေနဲ့ ကိုက်ညီတဲ့ **အစီအစဉ်အဆင့် သုံးခု**
- Ultimate Multisite ရဲ့ plugin နဲ့ template ထိန်းချုပ်မှုတွေကို အသုံးပြုတဲ့ **Feature gating**
- ရှင်းလင်းတဲ့ upgrade လမ်းကြောင်းပါတဲ့ **Plan group** တစ်ခု
- အပိုဝင်ငွေအတွက် **Order bump ထုတ်ကုန်များ**
- နည်းပညာဆိုင်ရာ အချက်အလက်တွေအပေါ် မူတည်ခြင်းမရှိဘဲ ဖောက်သည်ရဲ့ တန်ဖိုးအပေါ် အခြေခံတဲ့ **ဈေးနှုန်းဖွဲ့စည်းပုံ**
