# သင်ခန်းစာ ၆- အကောင့်ဖွင့်ခြင်း အတွေ့အကြုံ

ဝယ်ယူမှုလုပ်ငန်းစဉ် (checkout flow) သည် စိတ်ဝင်စားမှုကို ငွေကြေးအဖြစ် ပြောင်းလဲပေးသည့် နေရာဖြစ်သည်။ ရှုပ်ထွေးသည် သို့မဟုတ် ယေဘူယျကျသော အကောင့်ဖွင့်မှု လုပ်ငန်းစဉ်များသည် ဖောက်သည်များကို ဆုံးရှုံးစေသည်။ ၎င်းတို့၏ ဘာသာစကားကို ပြောဆိုနိုင်ပြီး အလွယ်တကူ ခံစားရစေသည့် အထူးနယ်ပယ် (niche-specific) အလိုက် လုပ်ဆောင်သည့် လုပ်ငန်းစဉ်ကသာ ဖောက်သည်များကို အောင်မြင်စွာ ပြောင်းလဲပေးနိုင်မည်။

## ယခင်က ဘယ်နေရာမှာ ရပ်ခဲ့လဲ

FitSite တွင် template များနှင့် plan များကို စီစဉ်ထားပြီးဖြစ်သည်။ ယခုအခါတွင်မူ၊ အားကစားစတူဒီယိုပိုင်ရှင်များကို ငွေပေးချေသည့် ဖောက်သည်များအဖြစ် ပြောင်းလဲပေးမည့် အကောင့်မှတ်ပုံတင်ခြင်းနှင့် ဝယ်ယူမှု အတွေ့အကြုံကို တည်ဆောက်ရပါမည်။

## အကောင့်မှတ်ပုံတင်ခြင်း လုပ်ငန်းစဉ်ကို နားလည်ခြင်း

Ultimate Multisite ၏ checkout form များသည် လုံးဝစိတ်ကြိုက်ပြင်ဆင်နိုင်သော အဆင့်များစွာပါဝင်သည့် form များဖြစ်သည်။ FitSite အတွက် ကျွန်ုပ်တို့ လိုချင်သည်မှာ အောက်ပါအချက်များ ပါဝင်သည့် လုပ်ငန်းစဉ်တစ်ခု ဖြစ်သည်-

1. အားကစားလုပ်ငန်းများအတွက် တည်ဆောက်ထားသည်ဟု ခံစားရစေရမည်။
2. လိုအပ်သည်များကိုသာ စုဆောင်းရမည်။
3. ဖောက်သည်ကို အလုပ်လုပ်နိုင်သည့် website တစ်ခုဆီ အမြန်ဆုံး ရောက်ရှိစေရမည်။

အပြည့်အစုံ နည်းပညာဆိုင်ရာ ကိုးကားချက်အတွက် [The Registration Flow](/user-guide/configuration/the-registration-flow) ကို ကြည့်ပါ။

## FitSite Checkout ကို ဒီဇိုင်းဆွဲခြင်း

**Ultimate Multisite > Checkout Forms** သို့ သွားပြီး form အသစ်တစ်ခုကို ဖန်တီးပါ။

### အဆင့် ၁- Plan ရွေးချယ်ခြင်း

အားကစားစတူဒီယိုပိုင်ရှင်တစ်ဦး အရင်ဆုံးမြင်ရမှာက ၎င်းတို့ နားလည်တဲ့ စကားလုံးတွေနဲ့ တင်ပြထားတဲ့ plan တွေ ဖြစ်ရပါမယ်။

- **Pricing Table** field တစ်ခု ထည့်ပါ။
- FitSite plan သုံးခုလုံးကို ပြသရန် စီစဉ်ပါ။
- သင်သည် သင်ခန်းစာ ၅ တွင် ရေးသားခဲ့သော plan ဖော်ပြချက်များသည် ဤနေရာတွင် ပေါ်လာပါမည် — ၎င်းတို့သည် နည်းပညာဆိုင်ရာ အင်္ဂါရပ်များထက် အားကစားလုပ်ငန်းလိုအပ်ချက်များကို ပြောဆိုနေကြောင်း သေချာပါစေ။

:::tip Niche Language Matters
"၁ GB storage" ဆိုတာ အားကစားရုံပိုင်ရှင်တစ်ယောက်အတွက် ဘာမှမသိတဲ့အရာပါ။ "ပရော်ဖက်ရှင်နယ် စတူဒီယို website အတွက် လိုအပ်တဲ့ အရာအားလုံး" ဆိုတာက အရာအားလုံးကို ဆိုလိုပါတယ်။ Plan ဖော်ပြချက်တွေကို သင့်ဖောက်သည်ရဲ့ ဘာသာစကားနဲ့ ရေးပါ။
:::

### အဆင့် ၂- Template ရွေးချယ်ခြင်း

plan တစ်ခု ရွေးချယ်ပြီးနောက် ဖောက်သည်က ၎င်းတို့ စတင်မည့် template ကို ရွေးချယ်ရမည်။

- **Template Selection** field တစ်ခု ထည့်ပါ။
- ရရှိနိုင်သော template များကို ၎င်းတို့ ရွေးချယ်ထားသည့် plan (သင်ခန်းစာ ၅ တွင် စီစဉ်ထားသည်) ဖြင့် စစ်ထုတ်ပေးရမည်။
- template တစ်ခုစီတိုင်းတွင် အားကစားဆိုင်ရာ ဒီဇိုင်းကို ပြသသည့် ကြိုတင်ကြည့်ရှုနိုင်သော ပုံ (preview image) ပါရမည်။

### အဆင့် ၃- အကောင့်ဖန်တီးခြင်း

၎င်းကို အနည်းဆုံးဖြစ်အောင် ထားပါ။ အောက်ပါအချက်များကိုသာ စုဆောင်းပါ။

- Email address
- Password
- Studio/business name (၎င်းသည် ၎င်းတို့၏ site name ဖြစ်လာမည်)

အကောင့်ဖွင့်ချိန်တွင် မလိုအပ်သည့် အချက်အလက်များကို မမေးပါနှင့်။ အပို field တစ်ခုစီတိုင်းသည် conversion နှုန်းကို လျော့ကျစေသည်။

### အဆင့် ၄- Site စီစဉ်ခြင်း

- **Site title**: အဆင့် ၃ တွင် ထည့်သွင်းထားသော studio name မှ ကြိုတင်ဖြည့်ပါ။
- **Site URL**: studio name မှ အလိုအလျောက်ထုတ်ပေးပါ (ဥပမာ- `ironworks.fitsite.com`)

### အဆင့် ၅- ငွေပေးချေခြင်း

- **Payment** field ကို ထည့်ပါ။
- သင့်ငွေပေးချေမှု gateway ကို စီစဉ်ပါ ([Stripe](/user-guide/payment-gateways/stripe) သည် subscription billing အတွက် အကြံပြုပါသည်။)
- သင်သည် သင်ခန်းစာ ၅ တွင် order bumps များ ဖန်တီးခဲ့ပါက၊ payment အဆင့်မတိုင်မီ **Order Bump** field ကို ထည့်ပါ။

### အဆင့် ၆- အတည်ပြုခြင်း

- အားကစားလုပ်ငန်းဆိုင်ရာ ဘာသာစကားဖြင့် အတည်ပြုသည့် မက်ဆေ့ခ်ျကို စိတ်ကြိုက်ပြင်ဆင်ပါ။
- ဥပမာ- "သင့်အားကစားစတူဒီယို website ကို ဖန်တီးနေပါသည်။ စက္ကန့်အနည်းငယ်အတွင်း သင့် website အသစ်သို့ ပြန်လည်ညွှန်ပြပေးပါမည်။"

## Period Selection Toggle ထည့်ခြင်း

သင့် plan များတွင် ဈေးနှုန်းကွာခြားမှုများ (လစဉ်နှင့် နှစ်အလိုက်) ကို စီစဉ်ထားပါက၊ ဖောက်သည်များက ငွေပေးချေမှု ကာလများကြား ခလုတ်နှိပ်၍ ပြောင်းလဲနိုင်ရန် checkout form တွင် **Period Selection** field ကို ထည့်ပါ။ ညွှန်ကြားချက်များအတွက် [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle) ကို ကြည့်ပါ။

## ငွေပေးချေမှု စီစဉ်ခြင်း

သင်သည် ငွေပေးချေမှု gateway ကို မစတင်ဖွဲ့စည်းရသေးပါက-

1. **Ultimate Multisite > Settings > Payment Gateways** သို့ သွားပါ။
2. [Stripe setup guide](/user-guide/payment-gateways/stripe) သို့မဟုတ် သင်နှစ်သက်သည့် gateway ကို လိုက်နာပါ။
3. စမ်းသပ်ငွေပေးချေမှုဖြင့် အပြည့်အဝ checkout လုပ်ငန်းစဉ်ကို စမ်းသပ်ပါ။

ငွေပေးချေမှုများ သင့်အကောင့်သို့ မည်သို့စီးဆင်းသည်ကို အသေးစိတ်သိရှိရန် [Getting Paid](/user-guide/payment-gateways/getting-paid) ကို ကြည့်ပါ။

## လုပ်ငန်းစဉ်ကို စမ်းသပ်ခြင်း

ဆက်လက်မသွားမီ၊ အပြည့်အဝ စမ်းသပ် အကောင့်ဖွင့်မှုတစ်ခုကို လုပ်ဆောင်ပါ။

1. incognito/private browser window တွင် checkout form ကို ဖွင့်ပါ။
2. plan တစ်ခု ရွေးချယ်ပါ။
3. template တစ်ခု ရွေးချယ်ပါ။
4. အကောင့်တစ်ခု ဖန်တီးပါ။
5. ငွေပေးချေမှု ပြီးမြောက်အောင် လုပ်ဆောင်ပါ (test mode ကို အသုံးပြုပါ)။
6. site သည် မှန်ကန်သော template ဖြင့် ဖန်တီးထားကြောင်း စစ်ဆေးပါ။

အောက်ပါအချက်များကို စစ်ဆေးပါ-

- [ ] Plan ဖော်ပြချက်များသည် ရှင်းလင်းပြီး အထူးနယ်ပယ်နှင့် ကိုက်ညီမှုရှိခြင်း
- [ ] Template ကြိုတင်ကြည့်ရှုမှုများသည် အားကစားနှင့် သင့်လျော်သော ဒီဇိုင်းများကို ပြသခြင်း
- [ ] Site URL သည် studio name မှ မှန်ကန်စွာ ထုတ်ပေးခြင်း
- [ ] ငွေပေးချေမှုသည် အောင်မြင်စွာ လုပ်ဆောင်ခြင်း
- [ ] ဖောက်သည်သည် ရွေးချယ်ထားသော template ဖြင့် အလုပ်လုပ်နိုင်သည့် site ပေါ်သို့ ရောက်ရှိခြင်း
- [ ] အတည်ပြုသည့် email များသည် အားကစားလုပ်ငန်းဆိုင်ရာ ဘာသာစကားကို အသုံးပြုခြင်း

## FitSite Network အခုအချိန်အထိ

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow
│   ├── Plan selection with niche-specific descriptions
│   ├── Template selection with fitness previews
│   ├── Minimal account creation
│   ├── Payment via Stripe
│   └── Fitness-specific confirmation
└── Ready for branding (next lesson)
```

## ယခုသင်ခန်းစာတွင် ကျွန်ုပ်တို့ တည်ဆောက်ခဲ့သည်များ

- အားကစားစတူဒီယိုပိုင်ရှင်များအတွက် အထူးပြုလုပ်ထားသော **အဆင့်များစွာပါဝင်သည့် checkout form**
- အကောင့်ဖွင့်မှု လုပ်ငန်းစဉ်တစ်လျှောက်လုံး **အထူးနယ်ပယ်နှင့် ကိုက်ညီသော ဘာသာစကား**
- **အနည်းဆုံး အခက်အခဲ** — မရှိမဖြစ်လိုအပ်သော field များသာ၊ အလုပ်လုပ်နိုင်သည့် site သို့ အမြန်လမ်းကြောင်း
- စမ်းသပ်စစ်ဆေးမှုပါရှိသော **Payment integration**
- plan ရွေးချယ်ခြင်းမှ အလုပ်လုပ်နိုင်သည့် site အထိ **စမ်းသပ်ထားသော end-to-end flow**
