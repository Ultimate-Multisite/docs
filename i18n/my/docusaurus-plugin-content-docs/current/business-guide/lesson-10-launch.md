# သင်ခန်းစာ ၁၀- စတင်ဖွင့်လှစ်မည့်နေ့ {#lesson-10-launch-day}

အရာအားလုံးကို တည်ဆောက်ပြီးစီးပါပြီ။ တံခါးဖွင့်မည့် အချိန်မတိုင်မီ၊ ဤသင်ခန်းစာသည် မည်သည့်အရာမျှ ပျက်စီးခြင်း၊ ပျောက်ဆုံးခြင်း သို့မဟုတ် ရှက်စရာဖြစ်စေမည့်အရာများ မရှိစေရန် စစ်ဆေးရမည့် အဆင့်ဆင့် စာရင်းကို လမ်းညွှန်ပေးသွားမှာ ဖြစ်ပါတယ်။

## ကျွန်တော်တို့ ဘယ်နေရာမှာ ရပ်ခဲ့သလဲ {#where-we-left-off}

FitSite တွင် template များ၊ အစီအစဉ်များ၊ ငွေရှင်းမှု၊ brand အမှတ်အသားများ၊ အသုံးပြုသူ စတင်လေ့ကျင့်မှု (onboarding) နှင့် ဈေးနှုန်းသတ်မှတ်မှုများ အားလုံးကို စနစ်တကျ ဖွဲ့စည်းထားပြီးသား ဖြစ်ပါတယ်။ ယခုအခါ အရာအားလုံး အလုပ်လုပ်မှုရှိမရှိ စစ်ဆေးပြီး စတင်အသုံးပြုနိုင်ပြီ ဖြစ်ပါတယ်။

## စတင်မိတ်ဆက်မီ စစ်ဆေးရမည့် အချက်များ (Pre-Launch Checklist) {#pre-launch-checklist}

အချက်တိုင်းကို စစ်ဆေးပါ။ ဘာတစ်ခုမှ ကျော်လွန်မသွားပါနဲ့။

### Platform အခြေခံအဆောက်အအုံ (Platform Infrastructure) {#platform-infrastructure}

- [ ] Hosting သည် တည်ငြိမ်ပြီး ဝန်ပိုံများအောက်တွင် ကောင်းမွန်စွာ လုပ်ဆောင်နိုင်မှု ရှိ/မရှိ
- [ ] Wildcard SSL သည် အသက်ဝင်ပြီး Subdomain အားလုံးသည် HTTPS မှတစ်ဆင့် ဝန်ဆောင်မှုပေးနေမှု ရှိ/မရှိ
- [ ] Domain mapping သည် အလုပ်လုပ်မှုရှိခြင်း ရှိ/မရှိ -- စမ်းသပ်၍ site တစ်ခု ဖန်တီးပြီး custom domain ချိတ်ဆက်မှု လုပ်ကြည့်ပါ
- [ ] Backup များကို ဖွဲ့စည်းထားပြီး စမ်းသပ်မှု လုပ်ထားခြင်း ရှိ/မရှိ (အနည်းဆုံး တစ်ခုကို ပြန်လည်ရယူ၍ စစ်ဆေးပါ)
- [ ] စောင့်ကြည့်လေ့လာမှု (Monitoring) စနစ်ကို တည်ဆောက်ထားခြင်း ရှိ/မရှိ -- platform ကျသွားရင် သိနိုင်ရမည်

### Templates များ {#templates}

- [ ] Template သုံးမျိုးစလုံးသည် site အသစ်များတွင် မှန်ကန်စွာ တင်ပြမှု ရှိ/မရှိ
- [ ] နေရာယူထားသည့် အကြောင်းအရာများ (Placeholder content) သည် အထောက်အကူဖြစ်ပြီး စာလုံးပေါင်းအမှား မရှိခြင်း ရှိ/မရှိ
- [ ] ပုံများအားလုံးသည် မှန်ကန်စွာ ခွင့်ပြုချက်ရရှိထားခြင်း ရှိ/မရှိ (stock photo ၏ ရေအမှတ်အသားများ မပါရ)
- [ ] မိုဘိုင်းဖုန်းများတွင် template စာမျက်နှာတိုင်း၌ အသင့်တော်ဆုံးဖြစ်မှု ရှိ/မရှိ
- [ ] စာမျက်နှာတင်ချိန်မြန်နှုန်းသည် လက်ခံနိုင်ဖွယ် ရှိ/မရှိ (GTmetrix သို့မဟုတ် PageSpeed Insights ကဲ့သို့သော tool ဖြင့် စမ်းသပ်ပါ)
- [ ] Template တစ်ခုခုတွင် ချိုးကျနေသော link များ သို့မဟုတ် ပျောက်ဆုံးနေသော asset များ မရှိခြင်း ရှိ/မရှိ

### အစီအစဉ်များနှင့် ထုတ်ကုန်များ (Plans and Products) {#plans-and-products}

- [ ] အစီအစဉ် သုံးမျိုးစလုံးသည် အသက်ဝင်ပြီး မြင်ရမှု ရှိ/မရှိ
- [ ] အစီအစဉ် ဖော်ပြချက်များသည် မှန်ကန်ပြီး niche အလိုက် သီးသန့်ဖြစ်မှု ရှိ/မရှိ
- [ ] ဈေးနှုန်းသတ်မှတ်မှုသည် မှန်ကန်မှု ရှိ/မရှိ (လစဉ်နှင့် နှစ်အလိုက်)
- [ ] Setup fee များသည် မှန်ကန်သော အစီအစဉ်များတွင် ဖွဲ့စည်းထားမှု ရှိ/မရှိ
- [ ] Starter plan တွင် စမ်းသပ်ကာလ (Trial period) သည် အလုပ်လုပ်မှုရှိခြင်း ရှိ/မရှိ
- [ ] Order bumps များသည် ငွေရှင်းမှု လုပ်ငန်းစဉ်အတွင်း မှန်ကန်စွာ ပေါ်လာမှု ရှိ/မရှိ
- [ ] Plugin နှင့် theme ကန့်သတ်ချက်များသည် အစီအစဉ်အလိုက် မှန်ကန်စွာ အကောင်အထည်ဖော်မှု ရှိ/မရှိ

### ငွေရှင်းမှု လုပ်ငန်းစဉ် (Checkout Flow) {#checkout-flow}

- [ ] အစီအစဉ်တစ်ခုစီတိုင်းတွင် အပြည့်အဝ စမ်းသပ် စာရင်းသွင်းမှု လုပ်ကြည့်ပါ (စမ်းသပ်ငွေပေးချေမှု mode ကို အသုံးပြုပါ)
- [ ] Template ရွေးချယ်မှုသည် အစီအစဉ်အလိုက် မှန်ကန်သော template များကို ပြသမှု ရှိ/မရှိ
- [ ] ငွေပေးချေမှု လုပ်ငန်းစဉ်သည် အောင်မြင်စွာ လုပ်ဆောင်မှု ရှိ/မရှိ
- [ ] စာရင်းသွင်းပြီးနောက် Customer သည် ကြိုဆိုသည့် email ကို လက်ခံရရှိမှု ရှိ/မရှိ
- [ ] Site အသစ်သည် မှန်ကန်သော template ဖြင့် ဖန်တီးပေးမှု ရှိ/မရှိ
- [ ] Customer သည် ၎င်းတို့၏ site အသစ်သို့ ချက်ချင်း ဝင်ရောက်နိုင်မှု ရှိ/မရှိ
- [ ] Discount code များသည် မှန်ကန်စွာ အလုပ်လုပ်မှု ရှိ/မရှိ

### Brand အမှတ်အသား (Branding) {#branding}

- [ ] Login page သည် FitSite brand အမှတ်အသားကို ပြသမှု ရှိ/မရှိ
- [ ] Admin dashboard သည် FitSite brand အမှတ်အသားကို ပြသမှု ရှိ/မရှိ
- [ ] စနစ်၏ email အားလုံးသည် FitSite brand အမှတ်အသားနှင့် fitness သီးသန့် ဘာသာစကားကို အသုံးပြုမှု ရှိ/မရှိ
- [ ] Invoice များသည် သင့်လုပ်ငန်းအသေးစိတ်အချက်အလက်များနှင့်အတူ မှန်ကန်စွာ ပြသမှု ရှိ/မရှိ
- [ ] Marketing site သည် အသက်ဝင်ပြီး checkout form သို့ ချိတ်ဆက်မှု ရှိ/မရှိ

### အသုံးပြုသူ စတင်လေ့ကျင့်မှု (Onboarding) {#onboarding}

- [ ] Quick Start widget သည် Customer အသစ်များ၏ dashboard တွင် ပေါ်လာမှု ရှိ/မရှိ
- [ ] Quick Start link အားလုံးသည် မှန်ကန်သော စာမျက်နှာများသို့ ညွှန်ပြမှု ရှိ/မရှိ
- [ ] ကြိုဆိုသည့် email sequence သည် ဖွဲ့စည်းပြီး စမ်းသပ်မှု လုပ်ထားမှု ရှိ/မရှိ
- [ ] Knowledge base article များသည် ထုတ်ဝေပြီး ဝင်ရောက်နိုင်မှု ရှိ/မရှိ
- [ ] Account page သည် မှန်ကန်သော အစီအစဉ်အချက်အလက်နှင့် upgrade ရွေးချယ်မှုများကို ပြသမှု ရှိ/မရှိ

- [ ] Terms of service ကို ထုတ်ဝေပြီး checkout မှတစ်ဆင့် ချိတ်ဆက်မှု ရှိ/မရှိ
- [ ] Privacy policy ကို ထုတ်ဝေပြီး ဝင်ရောက်နိုင်မှု ရှိ/မရှိ
- [ ] Refund policy ကို သတ်မှတ်ပြီး စာရွက်စာတမ်းအဖြစ် မှတ်တမ်းတင်မှု ရှိ/မရှိ
- [ ] ငွေပေးချေမှုများ လက်ခံရန် လုပ်ငန်းအဖွဲ့အစည်း (Business entity) ကို ဖွဲ့စည်းထားမှု ရှိ/မရှိ
- [ ] Payment gateway သည် live mode တွင် ရှိခြင်း (test mode မဟုတ်) ရှိ/မရှိ
- [ ] Tax configuration သည် သင့်နယ်မြေအတွက် မှန်ကန်မှု ရှိ/မရှိ

## Soft Launch နှင့် Hard Launch {#soft-launch-vs-hard-launch}

အဆင့်နှစ်ဆင့်ဖြင့် စတင်ဖွင့်လှစ်ခြင်းကို စဉ်းစားပါ။

### အဆင့် ၁- Soft Launch {#phase-1-soft-launch}

အများပြည်သူထံ မမိတ်ဆက်မီ fitness studio ပိုင်ရှင် ၅-၁၀ ဦးကို ဖိတ်ခေါ်၍ စာရင်းသွင်းခိုင်းပါ။ ၎င်းတို့သည် သင့်၏ beta customer များ ဖြစ်ကြသည်။ ၎င်းတို့ထံမှ အောက်ပါအချက်များအတွက် အရေးပါသော လျှော့စျေး (ဘဝတစ်လျှောက် ၅၀% လျှော့၊ သို့မဟုတ် ၃ လ အခမဲ့) ကို ပေးပါ။

- စာရင်းသွင်းမှုနှင့် onboarding အတွေ့အကြုံအပေါ် ရိုးသားသော feedback များ
- ၎င်းတို့၏ site ကို ပြသမှုဥပမာအဖြစ် အသုံးပြုခွင့် ခွင့်ပြုချက်
- ၎င်းတို့ကြုံတွေ့ရသည့် bug များ သို့မဟုတ် ပြဿနာများကို အစီရင်ခံတင်ပြခြင်း

၎င်းသည် အများပြည်သူထံ မဖွင့်မီ အမှန်တကယ် customer feedback နှင့် ပြသရန် live site များကို ရရှိစေပါသည်။

### အဆင့် ၂- Public Launch {#phase-2-public-launch}

Soft launch feedback များကို ပေါင်းစပ်ပြီးပါက:

- Payment gateway ကို live mode သို့ ပြောင်းလဲပါ
- သင့်၏ marketing site ကို ထုတ်ဝေပါ
- Customer များ စတင်ရယူမှု (Lesson 11) ကို စတင်ပါ
- သက်ဆိုင်ရာ fitness လုပ်ငန်းနယ်ပယ် ချန်နယ်များတွင် ကြေညာပါ

## စတင်ဖွင့်လှစ်မည့်နေ့ လုပ်ဆောင်ရမည့်အရာများ {#launch-day-actions}

အများပြည်သူထံ ပွင့်လင်းသည့်နေ့တွင်:

၁။ **Live ငွေပေးချေမှုများသို့ ပြောင်းလဲပါ** -- သင့် payment gateway ၏ test mode ကို ပိတ်ပါ
၂။ **တစ်ကြိမ်ထပ်မံ စစ်ဆေးပါ** -- အမှန်တကယ် ငွေပေးချေမှုဖြင့် အပြည့်အဝ စမ်းသပ် စာရင်းသွင်းမှု လုပ်ပါ (နောက်မှ ကိုယ့်ကိုယ်ကိုယ် ပြန်လည်ငွေပေးချေပါ)
၃။ **နီးနီးကပ်ကပ် စောင့်ကြည့်ပါ** -- အမှားများ၊ စာရင်းသွင်းမှု ပျက်ကွက်မှုများ သို့မဟုတ် ငွေပေးချေမှု ပြဿနာများကို စောင့်ကြည့်ပါ
၄။ **အကူအညီပေးရန် အသင့်ရှိပါ** -- သင့်၏ ပထမဆုံး အမှန်တကယ် customer များ အကူအညီလိုနိုင်ပြီး၊ အမြန်တုံ့ပြန်မှုက ယုံကြည်မှုကို တည်ဆောက်ပေးသည်
၅။ **ခဏတာ ဂုဏ်ပြုပါ** -- ထို့နောက် အလုပ်ကို ပြန်လုပ်ပါ

## ဘာတွေ မှားယွင်းနိုင်သလဲ {#what-can-go-wrong}

အောက်ပါအရာများအတွက် ပြင်ဆင်ထားပါ-

- **Payment gateway ပြဿနာများ**: သင့် gateway ပံ့ပိုးသူ၏ ဆက်သွယ်ရန် အကောင့်ကို အသင့်ထားပါ
- **SSL certificate ပြဿနာများ**: certificate များကို မည်သို့စစ်ဆေးရမည်နှင့် ပြန်လည်ရယူရမည်ကို သိထားပါ
- **Email ပို့ဆောင်မှု ပျက်ကွက်မှုများ**: email များ အမှန်တကယ် ရောက်ရှိမှု ရှိ/မရှိ စမ်းသပ်ပါ (spam folder များကို စစ်ဆေးပါ)
- **ဝန်ပိုံအောက်တွင် စွမ်းဆောင်ရည်ကျဆင်းမှု**: Traffic တိုးလာပါက၊ သင့် hosting ကို မည်သို့ချဲ့ထွင်ရမည်ကို သိထားပါ
- **Customer များ ရှုပ်ထွေးမှု**: သင့်၏ knowledge base နှင့် support channel များကို အသင့်ထားပါ

## ဤသင်ခန်းစာတွင် ကျွန်တော်တို့ တည်ဆောက်ခဲ့သည်များ {#what-we-built-this-lesson}

- အခြေခံအဆောက်အအုံ၊ အကြောင်းအရာ၊ ငွေပေးချေမှုနှင့် ဥပဒေတို့ကို အကျုံးဝင်သည့် အပြည့်အဝ စတင်မိတ်ဆက်မီ စစ်ဆေးရမည့် အချက်များ
- အများပြည်သူထံ မဖွင့်မီ အမှန်တကယ် feedback ရရှိစေရန် soft launch strategy
- ယုံကြည်မှုရှိစွာ live ဖြစ်စေရန် စတင်ဖွင့်လှစ်မည့်နေ့ လုပ်ထုံးလုပ်နည်းများ
- မကြာခဏ ကြုံရနိုင်သော launch day ပြဿနာများအတွက် အရေးပေါ်အစီအစဉ်
