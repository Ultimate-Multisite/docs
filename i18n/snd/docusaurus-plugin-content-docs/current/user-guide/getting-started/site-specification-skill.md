# साइट स्पेसिफिकेसन سکل سکل {#site-specification-skill}

**سائٹ سپیسیفیکیشن سکل (Site Specification skill)** آپ کی سائٹ کے اہداف، سامعین اور برانڈ کی شناخت کو ریکارڈ کرنے کا ایک منظم طریقہ ہے۔ یہ معلومات آپ کی **site_brief** میموری میں محفوظ ہوتی ہے، جس کا استعمال ایجنٹس مختلف سیشنز میں مستقل اور سیاق و سباق سے آگاہی فراہم کرنے کے لیے کرتے ہیں۔

## سائٹ سپیسیفیکیشن کیا ہے؟ {#what-is-site-specification}

سائٹ سپیسیفیکیشن وہ عمل ہے جس میں درج ذیل چیزوں کو دستاویز کیا جاتا ہے:

- **سائٹ کا مقصد (Site purpose)**: آپ کی سائٹ کیا کرتی ہے اور یہ کیوں موجود ہے۔
- **مستقل سامعین (Target audience)**: کون آپ کی سائٹ پر آتا ہے اور انہیں کیا ضرورت ہوتی ہے۔
- **برانڈ کی شناخت (Brand identity)**: آپ کے رنگ، لہجہ (tone)، اور بصری انداز۔
- **کاروباری اہداف (Business goals)**: آپ کی سائٹ کے لیے کامیابی کی شکل کیسی نظر آتی ہے۔
- **مواد کا ڈھانچہ (Content structure)**: آپ کی سائٹ کیسے منظم ہے۔

یہ سپیسیفیکیشن آپ کی **site_brief** بن جاتی ہے، ایک مستقل یادداشت جسے ایجنٹس آپ کی سائٹ کے سیاق و سباق کو سمجھنے کے لیے استعمال کرتے ہیں۔

## سائٹ سپیسیفیکیشن کیوں استعمال کریں؟ {#why-use-site-specification}

### سیشنز میں مستقل مزاجی (Consistency Across Sessions) {#consistency-across-sessions}

اگر site_brief نہ ہو تو ہر بار نیا سیشن شروع کرنے پر آپ کو اپنی سائٹ کے مقصد کی دوبارہ وضاحت کرنی پڑے گی۔ اس کے ساتھ، ایجنٹس فوری طور پر سمجھ جاتے ہیں:

- آپ کی سائٹ کے اہداف اور سامعین
- آپ کے برانڈ کے رنگ اور لہجہ
- آپ کا مواد کا ڈھانچہ
- آپ کے کاروباری مقاصد

### بہتر سفارشات (Better Recommendations) {#better-recommendations}

ایجنٹس اپنے site_brief کا استعمال یہ کرنے کے لیے کرتے ہیں:

- آپ کی سائٹ کے مقصد سے متعلق فیچرز تجویز کرنا۔
- ایسے مواد کے ڈھانچے تجویز کرنا جو آپ کے اہداف سے میل کھاتے ہیں۔
- اپنے برانڈ کے مطابق ڈیزائن تجویز کرنا۔
- غیر مطابقت والے فیچرز تجویز کرنے سے بچنا۔

### تیز آن بورڈنگ (Faster Onboarding) {#faster-onboarding}

نئے ایجنٹس (یا نئے سیشنز میں ایجنٹس) آپ کی سائٹ کے بارے میں وضاحت طلب سوالات پوچھنے کے بجائے آپ کی site_brief پڑھ کر جلدی سمجھ سکتے ہیں۔

## سائٹ سپیسیفیکیشن شروع کرنا {#initiating-site-specification}

### تھیم بلڈر آن بورڈنگ کے دوران (During Theme Builder Onboarding) {#during-theme-builder-onboarding}

سائٹ سپیسیفیکیشن سکل خود بخود **Theme Builder onboarding flow** کے دوران شروع ہو جاتا ہے۔ سیٹ اپ اسسٹنٹ ایجنٹ سوالات پوچھتا ہے اور آپ کی site_brief بناتا ہے۔

### دستی آغاز (Manual Initiation) {#manual-initiation}

توهان جي سائي اسڪيپيشن شروع ڪرڻ وقت ڪا به وقت آهي:

```
"آءُ منهنجي سائي اسڪيپيشن جو بيان ڪريون ٿو"
```

يا

```
"مونکي هڪ سائي بريف پيدا ڪرڻ ۾ مدد ڪريو"
```

## سائي اسڪيپيشن جي عمل {#the-site-specification-process}

### قدم 1: سائي جو مقصد (Site Purpose) {#step-1-site-purpose}

ايجنت پوءِ پڇندو آهي:

```
توهان جي سائي جو بنيادي مقصد के آهي؟
- اي-ڪمرس اسٽور
- بلاگ يا مواد جي سائي
- پورٽ فوليو يا ڏيکاري
- سي اي اس اي (SaaS) اپلويشن
- ڪميونٽي يا فورم
- ٻيو: [وضاحت ڪريو]
```

توهان هڪ ڪنهن همجي کي چونڊي سگهو ٿا يا پنهنجي مقصد جو بيان ڪري سگهو ٿا.

### قدم 2: حتمي سامع (Target Audience) {#step-2-target-audience}

```
توهان جو بنيادي سامع کون آهي؟
- صارف / عام عوام
- ڪاروباري پروفيسنل
- ڊولپرز / تقني استعمال ڪندڙ
- شاگرد / تعليمي عملي
- ٻيو: [وضاحت ڪريو]

انهن جي اهم ضرورتون ڇا آهن؟
```

### قدم 3: برانڊ جي شناخت (Brand Identity) {#step-3-brand-identity}

```
توهان جا برانڊ جا رنگ के آهن؟
- بنيادي رنگ: [رنگ چوني يا هڪس ڪوڊ]
- ثانوي رنگ: [رنگ چوني يا هڪس ڪوڊ]
- Accent رنگ: [اختياري]

توهان پنهنجي برانڊ جي لهجي کي ڪيئن بيان ڪندا؟
- پروفیشنل / ڪارپوري
- تخليقي / فن پارے
- کيڏيوز / عام
- مينيمل / ماڊرن
- گرم / دوستانه
```

### قدم 4: ڪاروباري مقصد (Business Goals) {#step-4-business-goals}

```
توهان جي سائي لاءِ ڪاميابي ڪهڙي ڏس ٿي؟
- لئڊز پيدا ڪرڻ
- مصنوعات بڪري
- ڪميونٽي جو بنانا
- علم بانيو ڪرڻ
- حڪمت واري مقام قائم ڪرڻ
- ٻيو: [وضاحت ڪريو]

توهان کا بنيادي ميٽر ڇا آهي؟
- آمدني (Revenue)
- صارف جي شموليت (User engagement)
- مواد جي رسائي (Content reach)
- تبادلي عمل (Conversions)
- ٻيو
```

### قدم 5: مواد جو ڈھانجو (Content Structure) {#step-5-content-structure}

```
توهان جو مواد ڪيئن منظم آهي؟
- فلٽ (سڀ مواد هڪ ئي سطح تي)
- هيرارڪيڪل (ڪٽريگيز ۽ ساب ڪٽريگيز)
- وقت جي لحاظ کان (بلاگ طرز)
- مصنوعات جي بنياد تي (ڪاٽالوگ)
- ٻيو: [وضاحت ڪريو]

توهان ڪهڙا مواد جا قسم استعمال ڪندا آهيو؟
- بلاگ پوسٽس
- مصنوعات جا صفحا
- ڪيس اسٽڊيز
- دستاويز (Documentation)
- وڊيوز
- ٻيو
```

## توهان جي site_brief منهنجي {#your-sitebrief-memory}

سائي اسڪيپيشن کي ختم ڪرڻ کان پوءِ، توهان جي معلومات **site_brief** طور توهان جي ايجنت جي هميشي ۾ محفوظ ٿيندي آهي. هي هڪ منظم رڪارڊ آهي جنهن ۾ شامل آهن:

## तपाईंको साइट ब्रीफ हेर्ने र अपडेट गर्ने तरिका {#viewing-and-updating-your-sitebrief}

### आफ्नो साइट ब्रीफ हेर्नुहोस् {#view-your-sitebrief}

एजेन्टलाई सोध्नुहोस्:

```
"Show me my site brief"
```

वा

```
"What do you know about my site?"
```

एजेन्टले तपाईंको सेभ गरिएको विवरण देखाउनेछ।

### आफ्नो साइट ब्रीफ अपडेट गर्नुहोस् {#update-your-sitebrief}

यदि तपाईंको साइट परिवर्तन हुन्छ भने, तपाईं यसलाई अपडेट गर्न सक्नुहुन्छ:

```
"Update my site brief: we're now targeting B2B customers"
```

वा

```
"Refresh my site specification"
```

यसले तपाईंको पुरानो जानकारीलाई आधार मानेर Site Specification skill फेरि चलाउनेछ।

## एजेन्टहरूले साइट ब्रीफ कसरी प्रयोग गर्छन् {#how-agents-use-sitebrief}

### डिजाइन सिफारिसहरू {#design-recommendations}

जब तपाईं डिजाइन परिवर्तनका लागि सोध्नुहुन्छ, एजेन्टहरूले तपाईंको साइट ब्रीफको सन्दर्भ लिन्छन्:

```
तपाईं: "Redesign my homepage"
एजेन्ट: "Your site brief अनुसार, तपाईं २५-४५ वर्षकी महिलालाई लक्षित गर्दै हुनुहुन्छ जसको टोन न्यानो र कलात्मक छ। म एउटा डिजाइन बनाउनेछु जसले तपाईंको हस्तनिर्मित गहना र दिगो मूल्यहरूमा जोड दिन्छ।"
```

### फिचर सुझावहरू {#feature-suggestions}

एजेन्टहरूले तपाईंको लक्ष्यहरूसँग मिल्ने फिचरहरू सुझाव दिन्छन्:

```
तपाईं: "What features should I add?"
एजेन्ट: "ग्राहक निष्ठामा केन्द्रित ई-कमर्स स्टोरका लागि, म निम्न कुराहरू सिफारिस गर्छु: лоयल्टी प्रोग्राम (loyalty program), ग्राहक समीक्षा (customer reviews), इमेल न्यूजलेटर (email newsletter), र उत्पादन सिफारिसहरू (product recommendations)।"
```

### सामग्री संरचना {#content-structure}

एजेन्टहरूले तपाईंको संरचनाको आधारमा सामग्रीको व्यवस्थापन प्रस्ताव गर्छन्:

توهان: "من کي پنهنجي بلاگ کي ڪيئن منظم ڪرڻ گهرجي؟"
جوابدار: "چونکہ توهان وٽ پروڪٽ ڪٽالو (product catalog) سان لاڳاپيل بلاگ آهي، مان مشورو ڏيان ٿو ته پوسٽن کي پروڪٽ جي ڪٽي (category) جي لحاظ کان منظم ڪريو ۽ 'اسٽائلنگ گائڊز' (styling guides) ٺاهيو جيڪي متعدد پروڪٽن کي هڪ ئي ڏيکاريون."

## بهترين عمليون (Best Practices) {#best-practices}

### خاص رهيو (Be Specific) {#be-specific}

"عام سامعن" جي جاءِ تي، پنهنجي اصل سامعن جو بيان ڪريو:

- ✓ "25-45 سالن ونيتيا، جيڪي ماحول دوست فیشن ۾ دلچسپي رکندا"
- ✗ "هر ڪنه"

### باقاعدي اپڊيٽ ڪريو (Update Regularly) {#update-regularly}

جڏهن توهان جي سائيٽ ترقي ڪندي رهندو آهي، ته پنهنجي `site_brief` کي اپڊيٽ ڪريو:

- جڏهن توهان نئين سامعن تي منتقل ٿيندا آهيو
- جڏهن توهان نيني پروڪٽ لائنز شامل ڪندا آهيو
- جڏهن توهان جي برانڊ جي شناخت ۾ تبديلي اچي
- جڏهن توهان جي ڪاروباري مقصد بدلجي

### هڪجهڙا اصطلاحات استعمال ڪريو (Use Consistent Terminology) {#use-consistent-terminology}

هيٺ ڏنل سيسشنن ۾ هڪ ئي اصطلاحات استعمال ڪريو:

- ✓ هميشه "sustainable jewelry" (جنهن کان سواءِ "eco-friendly jewelry" ۽ "green products") چيو
- ✓ پنهنجي سامعن کي مستقل هيٺين طرح refer ڪريو

### سياق شامل ڪريو (Include Context) {#include-context}

هي معلومات فراهم ڪريو جيڪا ايجنت کي توهان جي فيصلي سمجھڻ ۾ مدد ڪندي:

- "اسان انهن پروفیشنلن کي نشانو بنايو ٿا جيڪي قيمت کان وڌيڪ गुण واجبات مان ڳولين ٿا"
- "هڪڙي سامعن جو علم آهي ۽ انهن کي ماڊرن ډيزائن جي توقع آهي"
- "اسان هڪ بوٽسترپڊ اسٽارٽ اپ هم، تنهن ڪري اسان کي لاڳو ٿيندڙ حل گهرجن آهن"

## ٿيم بلڊر انبورڊنگ سان تعلق (Relationship to Theme Builder Onboarding) {#relationship-to-theme-builder-onboarding}

Site Specification سڪل مشين **Theme Builder onboarding flow** ۾ شامل آهي. جڏهن توهان ان بورڊنگ پورو ڪري ويندا آهيو، ته توهان جي `site_brief` خود بخود ان معلومات سان ٺهي ويندي آهي جيڪا توهان فراهم ڪئي هلي.

توهان Site Specification کي آزادانه چالو به ڪري سگهو ٿا جيڪڏهن توهان چاهيو:

- شروعاتي ترتياب کان پوءِ پنهنجي اسپيسيفڪيشن (specification) کي بهتر ڪريو
- جڏهن توهان جي سائيٽ ترقي ڪندي، ته پنهنجي site brief کي اپڊيٽ ڪريو
- Theme Builder شروع ڪرڻ کان اڳ هڪ تفصيلي اسپيسيفڪيشن ٺاهيو

## گهڻي حل (Troubleshooting) {#troubleshooting}

**منهنجي site_brief استعمال نه ٿي رهي آهي**
- تصديق ڪريو ته ايجنت کي memory تک رسائي آهي
- ايجنت کان "my site brief recall" ڪرڻ جو پڇو
- چیک ڪريو ته توهان جي settings ۾ memory فعال آهي

**مونکي هڪ نئين site\_brief سان شروع ڪرڻ چاهي ٿو**
- ايڪيئنٽ کي پڇو: "Clear my site brief and start fresh" (منهنجي سائيٽ بريف صاف ڪري ڇڏيو ۽ نقي طور تي شروع ڪريو)
- پوءِ Site Specification کي ٻيهر چلاؤ

**ايڪيئنٽ جو مشورن منهنجي site\_brief سان مطابقت نه رکين ٿا**
- ايڪيئنٽ کي پڇو: "review my site brief" (منهنجي سائيٽ بريف جي جائزو وٺو)
- جيڪڏهن توهان جي site\_brief پراني آهي ته ان کي اپڊيٽ ڪريو
- پنهنجي درخواستن ۾ اضافي معلومات فراهم ڪريو

## اڳتي قدم (Next Steps) {#next-steps}

پنهنجي site specification جو بيان ڪرڻ کان پوءِ:

1. **Theme Builder استعمال ڪريو**: پنهنجي site\_brief جي بنياد تي هڪ ڪسوتي theme banao
2. **Design کي بهتر ڪريو**: تفصيلي تصميم لاءِ Design System Aesthetics skill جو استعمال ڪريو
3. **Content پلان ڪريو**: content structure جي سفارشن لاءِ ايڪيئنٽ کان پڇو
4. **Features banao**: پنهنجي business goals سان ملند features جو درخواست ڪريو
