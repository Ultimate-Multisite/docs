# Ultimate Multisite کی ضروریات

Ultimate Multisite خریدنے سے پہلے آپ خود سے سب سے پہلے جو سوال پوچھ سکتے ہیں وہ یہ ہے کہ ہمارے اس پگ ان (plugin) کو چلانے کے لیے کیا ضروریات ہیں۔ خوش قسمتی سے، Ultimate Multisite چلانے کے لیے صرف چند بنیادی ضروریات ہیں: ایک **ڈومین نیم**، ایک اچھا **ہوسٹنگ پرووائڈر** اور ایک ورڈپریس **Multisite** انسٹالیشن۔ بس اتنا ہی آپ کو چاہیے!

## ڈومین نیم

سب سے پہلے، آئیے ڈومین نیم کے بارے میں بات کرتے ہیں۔

جب آپ ڈومین نیم کا انتخاب کر رہے ہوں تو یہ ذہن میں رکھیں کہ آپ کی خاص مہارت (niche) کیا ہے اور Ultimate Multisite کے ساتھ آپ کیا بنانے والے ہیں: کیا آپ ایک ایجنسی بنانا چاہتے ہیں؟ دکانوں جیسے ویب سائٹس بیچنا چاہتے ہیں؟ یا شاید ورڈپریس ویب ہوسٹنگ؟ Ultimate Multisite کے ساتھ ہر چیز ممکن ہے۔

ہمارا پگ ان سبسکرپشن سروسز، ادائیگی وصول کرنے، چیک آؤٹ فارمز، رعایت کے وؤچرز اور کسٹمر کمیونیکیشن فراہم کرتا ہے۔ ایک ورڈپریس Multisite کو انسٹال کرنے، ترتیب دینے اور برقرار رکھنے کے لیے بہت زیادہ کام Ultimate Multisite کی مدد سے آسان ہو جاتا ہے۔

آپ جو بھی نیش (niche) منتخب کریں اسے ہمارے پگ ان کا استعمال کرتے ہوئے سنبھالا جا سکتا ہے بشرطیکہ آپ ویب سائٹس کو سروس کے طور پر بیچ رہے ہوں! ایسا ڈومین نیم منتخب کریں جو آپ کے بزنس پلان سے میل کھاتا ہو۔

## ہوسٹنگ پرووائڈر

اب جب آپ نے اپنے نیش اور کاروبار کی نمائندگی کرنے والا ایک اچھا ڈومین نیم حاصل کر لیا ہے، تو آپ کو Ultimate Multisite کے ساتھ اپنا نیٹ ورک ہوسٹ کرنے کے لیے ایک اچھی ویب ہوسٹنگ کمپنی کا انتخاب کرنا ہوگا۔ مارکیٹ میں کچھ اچھے آپشنز ہیں۔ کچھ ہوسٹنگ پرووائڈرز Ultimate Multisite کے ساتھ اچھی طرح سے جڑتے ہیں، جو اس بات پر منحصر ہو سکتا ہے کہ آپ کیا بنانے کی منصوبہ بندی کر رہے ہیں۔

Ultimate Multisite ڪنهن بهيخوٽنگ پروائيڊر تي ڪم ڪري سگهي ٿو. اڪثر فرق صرف اهو آهي ته ڪجهه هوستنگ কোম্পانين جهڙوڪ _WPMU DEV Hosting_، _WP Engine_، _Closte_، _Cloudways_، _Gridpane_ ۽ _Runcloud_ ۾ ڊومين ميپنگ لاءِ وڌيڪ مضبوط انٽيغري آهي. ٻين هوستنگن لاءِ، توهان کي ڪسٽم ڊومينيئل (custom domains) کي ذاتي طور تي ميپ ڪرڻ جي ضرورت پوي ٿي. اهو به هڪ بهترين ڳالهه آهي ته توهان هڪ اهڙو هوستنگ پروائيڊر حاصل ڪريو جو AutoSSL کي سپورٽ ڪري. _Closte_، _Runcloud_ ۽ _cPanel_ ان کي سپورٽ ڪن ٿا.

سرور جي خاص صفاتي (server specifications) جي باري ۾، ڪو مخصوص فارمولا نه آهي ڇاڪا ته اهو توهان جيڪي ڪهڙا فيچرز ۽ ڪم ڪري رهيا آهيو ۽ توهان جي نڪر (network) جو سائز ان تي منحصر ٿيندو. شيئرنگ هوستنگ هڪ بهترين شروعات ٿي سگهي ٿي، پر جڏهن به توهان جو نڪر وڌڻ شروع ڪري، توهان کي غالباً هڪ ڊيڪيٽڊ VPS جي ضرورت پوندي.

## WordPress Multisite

ان هتي، آءُ WordPress Multisite ان اسٽالشن بابت ٿورو گهڻو ٻڌايان.

WordPress Multisite جو مطلب آهي ته توهان هڪ ئي اسٽالشن تي واري سائيٽن جو هڪ نڪر (network) بنائ سگهو ٿا. اهو هڪ بيلٽ-ين ان فيچر آهي، پر اهو डिफल्ट طور تي فعال نه هوندو. توهان کي اهو فيچر پنهنجي WordPress اسٽالشن ۽ ويب هوستنگ ڪنٽرول پینل اندر فعال ڪرڻ جي ضرورت آهي.

WordPress ۾ Multisite کي فعال رکڻ **Ultimate Multisite جو بنياد آهي**. چيو ته Ultimate Multisite جو بنيادي استعمال صارف کي سائيٽن کي مانيج ڪرڻ ۽ سروس جي طور تي بڪري ڪرڻ ۾ مدد ڪرڻ آهي، ان لاءِ اهو WordPress تي Multisite فعال رکڻ سان ڪيلجي پوندا.

توهان لاءِ **هڪ گهري مضمون** موجود آهي جيڪو قدم بہ قدم ٻڌائي ٿو ته توهان جي WordPress اسٽالشن تي Multisite فيچر کي فعال ڪرڻ لاءِ ڇا ڪجهه ڪرڻو آهي.

جڏهن توهان WordPress تي Multisite فعال ڪري ورتو، تڏهن Ultimate Multisite انٽال ڪرڻ جو وقت اچي ويو آهي. Ultimate Multisite کي صحيح طور تي انٽال ڪرڻ لاءِ، توهان جي پاسي هيٺيون شروعات hon ٿيندي:

  * WordPress: v5.3+ (تجاويز: آخري مستحڪم نسخه)

  * PHP: 7.4.x (8.0 کان وڌيڪ سپورٽ قريب آهي)

  * MySQL: v5+ (تجاويز: 5.6، هي چاهي ته توهان جي هوستنگ پروائيڊر تي 8.0 موجود نه هجي)

Ultimate Multisite لاءِ هي سڀئي جوفاري (software requirements) آهن.

اها رکھیو ته Ultimate Multisite PHP 8.0 سان ڪم ڪري سگهي ٿو، پر اسان گهڻو سفارش ڪنداسين ته ان کي PHP 7.4.x سان چلائڻ پسند ڪيو وڃي.

هميشه هيءَ جي سفارش آهي ته ان کي **main domains** تي نصب ڪيو وڃي، نه subcontracted domains يا subdirectories تي نه. اهو subcontracted domains/subdirectories تي ڪم ڪري سگهي ٿو، پر هن ۾ ڪجهه غلطيون (errors) اچڻ جو امکان آهي.

Ultimate Multisite کي ڪيئن نصب ڪرڻ جي تفصيلي گائیڊ لاءِ، **هتي ڇڪيو**.
