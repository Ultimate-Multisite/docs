# ای میلز ۽ برڊڪاسٽس موڪلين (v2)

_**IMPORTANT NOTE: هي مضمون Ultimate Multisite جو 2.x ورژن رفرنس ڪري ٿو.**_

Ultimate Multisite ۾ هڪ خاص فيچر آهي جيڪو توهان کي پنهنجي ڪارنٽ ڪارن سان رابطو ڪرڻ جي اجازت ڏئي ٿو، جڏهن ته توهان مقصد وارو صارف يا صارف جي گروپ کي اي ميل موڪلين ۽ ان جي ڊاش بورڊ تي اعلان کڻڻ لاءِ نوٽيس بھیج سکد ٿا.

## برڊڪاسٽس سان پنهنجي ڪارنٽ ڪارن جا ڊاش بورڊ ۾ اڪمن نوٽيس شامل ڪريو

Ultimate Multisite جو برڊڪاسٽس فيچر استعمال ڪري، توهان پنهنجي صارف جي سبيسٽ ڊاش بورڊ تي **ادمن نوٽيس** شامل ڪري سگهو ٿا.

هي ان لاءِ به ڏاڍو مددگار آهي جڏهن توهان کي ڪنه اعلان ڪرڻ جي ضرورت هجي جهڙوڪ نظام جي مرمت (system maintenance) يا پنهنجي موجودہ ڪارنٽ ڪارن کي نئين مصنوعات يا خدمتون جو يقين ڏيارڻ. اهو ئي آهي ته توهان جي صارف جي ڊاش بورڊ تي ادمن نوٽيس ڪيئن نظر اچي سگهن ٿا.

<!-- Screenshot unavailable: Admin notice broadcast shown on a customer's subsite dashboard -->

ادمن نوٽيس شروع ڪرڻ لاءِ، پنهنجي نڪر (network) ڊاش بورڊ تي وڃو ۽ **Ultimate Multisite** مينيئل underside، توهان کي **Broadcasts** جو آپشن ملندو.

![Broadcasts list page in Ultimate Multisite admin](/img/admin/broadcasts-list.png)

توهان موجودہ برڊڪاسٽس کي سدير ڪري سگهو ٿا:

![Broadcast edit interface](/img/admin/broadcast-edit.png)

هي پيچ کي وٺي، اوپري تي **Add Broadcast** بٽڻ ڇڪيو.

هي Add broadcast modal window کي ظاهر ڪندو آهي جتي توهان چن سکد ته توهان ڪهڙي قسم جي برڊڪاسٽ موڪلين.

اڳتي وڌو ۽ **Message** چونڊيو پنهنجي عمل ڪريو ۽ **Next Step** بٽڻ ڇڪيو.

![Add broadcast modal with the Message option selected](/img/admin/broadcast-add-message.png)

نڪر جو هيٺ ڏنل ويندو ته توهان کي يا ته **Target customer** يا **Target product** جي ضرورت پوندي. اهو نوٽس ڪريو ته توهان هڪ کان وڌيڪ صارف يا ڪجهه کان وڌيڪ مصنوعات چونڊي سگهو ٿا.

کوئی صارف اکاؤنٹ یا پروڈکٹ تلاش کرنے کے لیے آپ کو فیلڈ میں کی ورڈ ٹائپ کرنا شروع کرنا ہوگا۔

**Message type** فیلڈ کے نیچے، آپ نوٹس کا رنگ منتخب کر سکتے ہیں۔ یہ آپ کے پیغام کی جلدی کو نمایاں کرے گا۔

اس کے بعد آپ **Next Step** پر کلک کر سکتے ہیں۔

![Target customers, target product and message type fields for a Message broadcast](/img/admin/broadcast-message-targets.png)

اگلی ونڈو وہ جگہ ہے جہاں آپ عنوان (subject) اور ان مواد/پیغام (content/message) کو لکھنا شروع کر سکتے ہیں جسے آپ صارفین کو بھیجنا چاہتے ہیں۔

![Broadcast message subject and content editor on the compose step](/img/admin/broadcast-edit.png)

اپنا پیغام بنانے کے بعد، آپ **Send** بٹن دبا سکتے ہیں۔

اور بس۔ ایڈمن کا نوٹس فوری طور پر آپ کے صارف کے ڈیش بورڈ پر نظر آنا چاہیے۔

## اپنے کسٹمرز کو ای میل بھیجیں

Ultimate Multisite بر broadast فیچر کا استعمال کرتے ہوئے، آپ اپنے صارفین کو ایک ای میل بھیج سکتے ہیں۔ آپ کے پاس یہ اختیار ہے کہ ای میل صرف مخصوص صارفین کو بھیجیں یا کسی خاص صارف گروپ کو پروڈکٹ یا پلان کی بنیاد پر نشانہ بنائیں۔

ای میل برbroadast شروع کرنے کے لیے، اپنے نیٹ ورک ایڈمن ڈیش بورڈ پر جائیں اور Ultimate Multisite مینو کے نیچے، آپ کو Broadcast کا آپشن ملے گا۔

![Broadcasts list page used as the starting point for an email broadcast](/img/admin/broadcasts-list.png)

اس پیج سے، اوپر **Add broadcast** بٹن پر کلک کریں۔

یہ Add broadcast modal ونڈو کھولے گا جہاں آپ یہ منتخب کر سکتے ہیں کہ آپ کس قسم کی broadast بھیجنا چاہتے ہیں۔ آگے بڑھیں اور **Email** منتخب کریں پھر **Next Step** بٹن پر کلک کریں۔

![Add broadcast modal with the Email option selected](/img/admin/broadcast-add-email.png)

اگلی ونڈو آپ سے یا تو **Target customer** یا **Target product** کے بارے میں پوچھے گی۔ نوٹ کریں کہ آپ ایک سے زیادہ صارفین یا ایک سے زیادہ پروڈکٹ منتخب کر سکتے ہیں۔

کوئی صارف اکاؤنٹ یا پروڈکٹ تلاش کرنے کے لیے آپ کو فیلڈ میں کی ورڈ ٹائپ کرنا شروع کرنا ہوگا۔

ایک بار جب آپ اپنا ہدف آڈینس منتخب کر لیں، تو آپ **Next Step** پر کلک کر سکتے ہیں۔

![Email broadcast targets and product selection for an Email broadcast](/img/admin/broadcast-email-targets.png)

اگلی ونڈو وہ جگہ ہے جہاں آپ عنوان (subject) اور ان مواد/پیغام (content/message) کو لکھنا شروع کر سکتے ہیں جو آپ صارفین کو بھیجنا چاہتے ہیں۔

<!-- Screenshot unavailable: Email broadcast subject and content editor on the compose step -->

اپنا پیغام بنانے کے بعد، آپ **Send** بٹن پر کلک کر سکتے ہیں۔

اور یہ دیکھ لیں کہ بر broadast فیچر کا استعمال کرتے ہوئے اپنے اینڈ یوزرز کو ای میل بھیجنا کتنا آسان ہے۔

## سسٹم ای میلز (System emails)

Ultimate Multisite میں سسٹم ای میلز وہ **خودکار نوٹیفیکیشنز** ہوتی ہیں جو کسی مخصوص کام جیسے رجسٹریشن، ادائیگی، ڈومین میپنگ وغیرہ کے بعد سسٹم کی طرف سے بھیجی جاتی ہیں۔ ان ای میلز کو Ultimate Multisite سیٹنگز سے ایڈٹ یا تبدیل کیا جا سکتا ہے۔ اس کے ساتھ ایک فیچر بھی آتا ہے جو آپ کو دوسرے Ultimate Multisite انسٹالیشن سے موجود سیٹنگز کو ری سیٹ کرنے اور امپورٹ کرنے کی اجازت دیتا ہے۔

### ری سیٹ کرنا اور امپورٹ کرنا (Resetting & Importing)

نئے Ultimate Multisite ورژن، نیز ایڈ-آؤنز، وقت کے ساتھ نئے ای میلز رجسٹر کرتے رہیں گے۔

تصادم اور دیگر مسائل سے بچنے کے لیے، **ہم آپ کے انسٹالیشن پر نئے ای میل ٹیمپلیٹس کو خود بخود سسٹم ای میلز کے طور پر شامل نہیں کریں گے**، جب تک کہ وہ کسی مخصوص فیچر کے صحیح کام کرنے کے لیے بہت ضروری نہ ہوں۔

تاہم، سپر ایڈمنز اور ایجنٹس امپورٹر ٹول کے ذریعے ان نئے رجسٹرڈ ای میلز کو امپورٹ کر سکتے ہیں۔ اس عمل سے ایک نیا سسٹم ای میل بن جائے گا جس میں نئے ای میل ٹیمپلیٹ کا مواد اور کنفیگریشن ہوگی، جس سے سپر ایڈمن کسی بھی تبدیلی کو کر سکتا ہے یا انہیں ویسے ہی رکھ سکتا ہے۔

#### سسٹم ای میلز کیسے امپورٹ کریں (How to import system emails)

پنهنجي Ultimate Multisite جي Settings page تي وڃو ۽ **Emails** ٽاب تي وڃو.

![Emails tab in Ultimate Multisite settings showing the System Emails section](/img/config/settings-emails-tab.png)

پھر، sidebar تي، **Customize System Emails** بوتن تي ڪلڪ ڪريو.

<!-- Screenshot unavailable: Customize System Emails button on the System Emails sidebar panel -->

System Emails page تي، توهان اوپر **Reset & Import** اڪشن بوتن ڏسڻ جڳهه ڏسندا رهندا. ان بوتن تي ڪلڪ ڪرڻ سان import ۽ reset modal window کليل ٿيندي.

![Reset or Import action button on the System Emails admin page](/img/admin/system-emails-reset-import.png)

پھر، توهان Import Emails جي خيارن کي toggle ڪري سگهو ٿا ته ڏسو ته ڪهڙيون system emails import ڪرڻ لاءِ موجود آهن.

<!-- Screenshot unavailable: Reset and Import modal with the Import Emails options expanded -->

#### System Emails کي Reset ڪرڻ

ڪيڏانهن وقت، توهان محسوس ڪندا ته جيڪي توهان هڪ خاص email template تي ڪيل تبديلين توهان لاءِ ڪم نه ڪري رهيون آهن ۽ توهان ان کي ان جي **default state** تي reset ڪرڻ چاهيو ٿا.

اها حالتن ۾، توهان جي دو خيار آهن: توهان سڌو system email delete ڪري سگهو ٿا ۽ ان کي واپس import ڪري سگهو ٿا (اوپر ڏنل تعليمات استعمال ڪندي) - جنهن سان send metrics ۽ ٻيا ڳالهيون مڪمل طور تي ختم ٿي ويندا، جيڪو هن طريقو کي هيٺين ترجيح ڏئي ٿو.

يا توهان ان email template کي reset ڪرڻ لاءِ **Reset & Import tool** جو استعمال ڪري سگهو ٿا.

ان email template کي reset ڪرڻ لاءِ، توهان اوپر وارن قدمن جي takip ڪري سگهو ٿا تائين جتي توهان Reset & Import tool تائين پهچندا، ۽ پوءِ، **Reset** خيار کي toggle ڪريو ۽ ان emails کي select ڪريو جن کي توهان انهن جي default content تي واپس reset ڪرڻ چاهيو.
