# Gratis AI Agent جون سيٽنگون

Gratis AI Agent ۾ **سيٽنگون → ترقي يافته** اسڪرين پٺئين حصي جي انضمامن لاءِ منتظم-سطح جي ترتيب فراهم ڪري ٿي. هي صفحو فيڊبيڪ اڳتي موڪلڻ، ڳولا فراهم ڪندڙن جون چاٻيون، منظم Superdav خدمت جي سيٽ اپ، Google Calendar ڪنٽرولز، TextBee SMS سيٽنگون، ۽ سڄي نيٽ ورڪ لاءِ فيچر فليگس کي دستاويز ڪري ٿو.

## سيٽنگن تائين رسائي {#accessing-settings}

1. WordPress منتظم ۾، **Gratis AI Agent → سيٽنگون** ڏانهن وڃو.
2. **ترقي يافته** ٽيب تي ڪلڪ ڪريو.

## فيڊبيڪ آخري نقطي جي ترتيب {#feedback-endpoint-configuration}

فيڊبيڪ آخري نقطو AI agent کان POST درخواستون وصول ڪري ٿو، جڏهن به ڪو استعمال ڪندڙ انگٺو-هيٺ بٽڻ، خودڪار اشارو بينر، يا `/report-issue` حڪم ذريعي فيڊبيڪ جمع ڪرائي ٿو.

| فيلڊ | وضاحت |
|---|---|
| **فيڊبيڪ آخري نقطي جو URL** | اهو URL جيڪو JSON جسم سان HTTP POST درخواستن طور فيڊبيڪ جمع ڪرائڻ وصول ڪري ٿو. |
| **Feedback API Key** | هر فيڊبيڪ درخواست جي `Authorization` header ۾ موڪليل bearer token. جيڪڏهن توهان جي آخري نقطي کي تصديق جي ضرورت نه آهي ته خالي ڇڏي ڏيو. |

### متوقع JSON مواد {#expected-json-payload}

توهان جي فيڊبيڪ آخري نقطي کي گهٽ ۾ گهٽ هيٺين فيلڊن سان JSON جسم قبول ڪرڻ لازمي آهي:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

ڳالهه ٻولهه جي پسمنظر تي دارومدار رکندي، مواد ۾ اضافي فيلڊون موجود ٿي سگهن ٿيون.

### `triage_category` جون قيمتون {#triagecategory-values}

AI ٽرائياج پرت مواد اڳتي موڪلڻ کان اڳ `triage_category` کي هيٺين قيمتن مان هڪ تفويض ڪري ٿي:

| قيمت | مطلب |
|---|---|
| `factual_error` | اسسٽنٽ غلط حقيقي معلومات فراهم ڪئي. |
| `unhelpful_answer` | جواب فني طور درست هو پر ڪارائتو نه هو. |
| `inappropriate_content` | جواب ۾ اهڙو مواد شامل هو جيڪو استعمال ڪندڙن کي نه ڏيکارڻ گهرجي. |
| `other` | فيڊبيڪ ڪنهن ڄاتل درجي سان نه مليو. |

### تصديق {#authentication}

جيڪڏهن توهان جي آخري نقطي کي تصديق جي ضرورت آهي، ته **Feedback API Key** فيلڊ کي پنهنجي bearer token تي سيٽ ڪريو. ايجنٽ موڪلي ٿو:

```
Authorization: Bearer <your-api-key>
```

جيڪڏهن **Feedback API Key** فيلڊ خالي آهي، ته ڪو به `Authorization` header نه موڪليو ويندو.

### فيڊبيڪ گڏ ڪرڻ کي بند ڪرڻ {#disabling-feedback-collection}

**فيڊبيڪ آخري نقطي جو URL** ۽ **Feedback API Key** ٻئي فيلڊ خالي ڇڏي ڏيو. انگٺو-هيٺ بٽڻ ۽ فيڊبيڪ UI استعمال ڪندڙن کي نظر ايندا رهندا، پر جمع ڪرايل شيون ڪنهن به ٻاهرين خدمت ڏانهن اڳتي نه موڪليون وينديون.

## Brave Search API Key {#brave-search-api-key}

**ترقي يافته** ٽيب تي پڻ، **Brave Search API Key** فيلڊ [انٽرنيٽ ڳولا](../configuration/internet-search) جي صلاحيت کي فعال ڪري ٿو.

| فيلڊ | وضاحت |
|---|---|
| **Brave Search API Key** | Brave Search ڊولپر Dashboard مان توهان جي API چاٻي. AI اسسٽنٽ ۾ انٽرنيٽ ڳولا فعال ڪرڻ لاءِ گهربل. |

فيلڊ ليبل ۾ Brave Search API سائن اپ صفحي ڏانهن ڪلڪ لائق لنڪ شامل آهي. انٽرنيٽ ڳولا بند ڪرڻ لاءِ خالي ڇڏي ڏيو.

هن فيچر بابت آخري استعمال ڪندڙ جي دستاويزن لاءِ [انٽرنيٽ ڳولا](../configuration/internet-search) ڏسو.

## منظم Superdav خدمت {#managed-superdav-service}

Superdav AI Agent v1.18.0 سهڪار ڪيل سائيٽن لاءِ منظم Superdav خدمت جا آخري نقطا ۽ خودڪار ڪنيڪشن فراهمي شامل ڪري ٿو. اهي ڪنٽرول تڏهن استعمال ڪريو جڏهن توهان جي سائيٽ کي هٿ سان ترتيب ڏنل خدمت جي آخري نقطي بدران ميزبان فراهم ڪندڙ سان ڳنڍجڻ گهرجي.

| فيلڊ | وضاحت |
|---|---|
| **منظم Superdav خدمت** | سهڪار ڪيل سائيٽن لاءِ ميزبان Superdav خدمت جو ڪنيڪشن فعال ڪري ٿو. |
| **ڪنيڪشن فراهم ڪريو** | خودڪار آخري نقطي ۽ اسناد جي فراهمي شروع ڪري ٿو. سائيٽ کي منظم فراهم ڪندڙ استعمال ڪرڻ گهرجي، ان جي تصديق ڪرڻ کان پوءِ هي استعمال ڪريو. |
| **خدمت جو آخري نقطو / ڪنيڪشن جي حالت** | فراهمي کان پوءِ موجوده آخري نقطو يا ڪنيڪشن جي حالت ڏيکاري ٿو. |

فراهمي کان پوءِ، سيٽنگون محفوظ ڪريو ۽ منظم-خدمت جي ڪم جي وهڪرن تي ڀروسو ڪرڻ کان اڳ ڪنيڪشن جي حالت جي تصديق ڪريو. جيڪڏهن فراهمي ناڪام ٿئي، ڏيکاريل ٻيهر ڪوشش واري هدايت جو جائزو وٺو ۽ تصديق ڪريو ته سائيٽ کي ميزبان فراهم ڪندڙ استعمال ڪرڻ جي اجازت آهي.

## Google Calendar جي ترتيب {#google-calendar-configuration}

جڏهن Superdav AI Agent v1.18.0 ڪئلينڊر فيچر فعال هوندا آهن، ايجنٽ ترتيب ڏنل ڪئلينڊر ۽ واقعي جا تفصيل پڙهي سگهي ٿو. ڪئلينڊر اوزار پڙهڻ-مرڪوز آهن ۽ شيڊول-آگاهه ياد ڏيارڻ، شرڪت ڪندڙن جي فالو اپ، ۽ رابطا ملائڻ لاءِ ڪارائتا آهن.

| فيلڊ | وضاحت |
|---|---|
| **Google Calendar اسناد** | ڪئلينڊر ڊيٽا پڙهڻ لاءِ گهربل اسناد يا token ڪنيڪشن محفوظ ڪري ٿو. |
| **ڪئلينڊر چونڊ** | محدود ڪري ٿو ته ايجنٽ ڪهڙن ترتيب ڏنل ڪئلينڊرن جو معائنو ڪري سگهي ٿو. |
| **ڪئلينڊر ڪنيڪشن جي حالت** | تصديق ڪري ٿو ته موجوده اسناد ڪئلينڊر ۽ واقعا پڙهي سگهن ٿا يا نه. |

ڪئلينڊر اسناد کي انهن ڪئلينڊرن تائين محدود رکو جن جي ايجنٽ کي ضرورت آهي. جيڪڏهن حالت مدي خارج token ڏيکاري، ته اسناد ٻيهر ڳنڍيو يا تبديل ڪريو.

## TextBee SMS اطلاع {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 ترتيب ڏنل اطلاع جي ڪم جي وهڪرن لاءِ TextBee کي SMS فراهم ڪندڙ طور شامل ڪري ٿو. حساس يا استعمال ڪندڙ-سامهون پيغامن لاءِ SMS اطلاعن کي انساني منظوري جي دروازن سان گڏ رکڻ گهرجي.

| فيلڊ | وضاحت |
|---|---|
| **TextBee API Key** | TextBee SMS فراهم ڪندڙ ڏانهن درخواستن جي تصديق ڪري ٿو. |
| **TextBee ڊوائيس / موڪليندڙ** | جڏهن فراهم ڪندڙ کي گهربل هجي، ٻاهر ويندڙ پيغامن لاءِ استعمال ٿيندڙ TextBee موڪليندڙ يا ڊوائيس چونڊي ٿو. |
| **SMS اطلاع فعال** | منظور ٿيل ڪم جي وهڪرن کي ٽيڪسٽ-پيغام اطلاع موڪلڻ جي اجازت ڏئي ٿو. SMS موڪلڻ کان بچڻ لاءِ غير فعال ڇڏي ڏيو. |

ٽيسٽ پيغام رڳو منتظم جي ملڪيت واري نمبر ڏانهن موڪليو، پوءِ شيڊول ڪيل يا شرڪت ڪندڙ-سامهون ياد ڏياريندڙ فعال ڪرڻ کان اڳ منظوري-دروازي جي روَيي جي تصديق ڪريو.

## فيچر فليگس {#feature-flags}

v1.9.0 ۾ پڻ متعارف ڪرايل، **سيٽنگون → فيچر فليگس** ٽيب اختياري ڪارڪردگي لاءِ ٽوگل سوئچ فراهم ڪري ٿو. هر فليگ سڄي نيٽ ورڪ ۾ يا ته فعال يا غير فعال هوندو آهي؛ هن وقت هر-سائيٽ لاءِ الڳ اوور رائيڊ موجود ناهي.

### فيچر فليگس تائين رسائي {#accessing-feature-flags}

1. WordPress منتظم ۾، **Gratis AI Agent → سيٽنگون** ڏانهن وڃو.
2. **فيچر فليگس** ٽيب تي ڪلڪ ڪريو.

### رسائي ڪنٽرول فليگس {#access-control-flags}

| جهنڊو | ڊفالٽ | وضاحت |
|---|---|---|
| **ايڊمنسٽريٽرن تائين محدود ڪريو** | بند | جڏهن فعال ڪيو وڃي، صرف `administrator` ڪردار وارا استعمال ڪندڙ AI Agent چيٽ پينل کولي سگهن ٿا. ٻيا سڀ ڪردار ان جي بدران "پنهنجي ايڊمنسٽريٽر سان رابطو ڪريو" پيغام ڏسندا. |
| **Network Admins تائين محدود ڪريو** | بند | جڏهن multisite نيٽ ورڪ تي فعال ڪيو وڃي، صرف Super Admins ايجنٽ استعمال ڪري سگهن ٿا. انفرادي سائيٽ ايڊمن روڪيا ويندا آهن. جيڪڏهن ٻئي فعال هجن ته "ايڊمنسٽريٽرن تائين محدود ڪريو" تي فوقيت رکي ٿو. |
| **Subscriber رسائي جي اجازت ڏيو** | بند | جڏهن فعال ڪيو وڃي، `subscriber` ڪردار وارا استعمال ڪندڙ چيٽ انٽرفيس استعمال ڪري سگهن ٿا پر صرف پڙهڻ وارين صلاحيتن تائين محدود هوندا آهن (ڪا پوسٽ ٺاهڻ يا سيٽنگن ۾ تبديليون نه). |
| **غير ميمبرن لاءِ غير فعال ڪريو** | بند | Ultimate Multisite رڪنيت جي حالت سان ضم ٿئي ٿو. جڏهن فعال ڪيو وڃي، چيٽ انهن سائيٽن لاءِ لڪايل هوندي آهي جن وٽ فعال رڪنيت نه هجي. |

### برانڊنگ جهنڊا {#branding-flags}

### آٽوميشن حفاظت جهنڊا {#automation-safety-flags}

Superdav AI Agent v1.18.0 وڌيڪ محفوظ آٽوميشن هلائن لاءِ انساني منظوري جا گيٽ ۽ ياد ڏياريندڙ رڪارڊ متعارف ڪرائي ٿو. اهي ڪنٽرول فيچر جهنڊن يا ترقي يافته آٽوميشن سيٽنگن ۾ ظاهر ٿي سگهن ٿا، انسٽال ٿيل پيڪيج تي دارومدار رکي ٿو.

| جهنڊو | ڊفالٽ | وضاحت |
|---|---|---|
| **انساني منظوري گهربل** | سفارش ڪيل آن | حساس آٽوميشنز کي روڪي ٿو جيستائين هڪ بااختيار استعمال ڪندڙ تجويز ڪيل عمل جو جائزو وٺي ۽ تصديق نه ڪري. |
| **ياد ڏياريندڙ جي نقل روڪڻ** | آن | موڪليل ياد ڏياريندڙ رڪارڊ ڪري ٿو ته جيئن ٻيهر ڪوششون يا شيڊول ٿيل هلائون نقل نوٽيفڪيشن نه موڪلين. |
| **ڪئلينڊر اوزار فعال ڪريو** | ترتيب ٿيڻ تائين بند | ايجنٽ کي ترتيب ڏنل Google ڪئلينڊرن ۽ واقعن کي پڙهڻ جي اجازت ڏئي ٿو. |
| **SMS نوٽيفڪيشن فعال ڪريو** | ترتيب ٿيڻ تائين بند | منظور ٿيل ڪم جي وهڪرن کي اجازت ڏئي ٿو ته سندون محفوظ ٿيڻ کان پوءِ TextBee SMS نوٽيفڪيشن موڪلين. |

### تبديليون لاڳو ڪرڻ {#applying-changes}

ڪنهن به جهنڊي کي تبديل ڪرڻ کان پوءِ **سيٽنگون محفوظ ڪريو** تي ڪلڪ ڪريو. تبديليون فوري طور اثر وٺن ٿيون — ڪا cache صاف ڪرڻ يا plugin ٻيهر فعال ڪرڻ جي ضرورت ناهي.
