# ٻين پيگمنٽس سان جاني ٿيل غير مطابقت (Known incompatibilities with other Plugins) {#known-incompatibilities-with-other-plugins}

Ultimate Multisite کي انسٹال ۽ فعال ڪرڻ کان پوءِ، توهان کي ٻين پيگمنٽس سان ڪجهه مسئلا اچڻ ۾ ٿي سگهن ٿا جيڪي حل نه ڪيا ويا آهن.

اسان هر نئين ريلس (release) سان اسان هن لست کي گهٽائي لاءِ ڪم ڪري رهيا آهيون، پر جيڪڏهن ڪو پيگمنٽ ان لست ۾ شامل آهي، ته توهان کي سڄي ڪم ڪندڙ Ultimate Multisite Network حاصل ڪرڻ لاءِ ان کي غير فعال (deactivate) ڪرڻ جي ضرورت پئي ٿي.

هي ان جي لست ۽ ان صورتحال ۾ توهان کي ڇا ڪرڻ گهرجي انهن جو ڏيڪر آهي:

**ڪجهه WPMU DEV پيگمنٽس** ڪجهه پيگمنٽس جيڪي WPMU DEV طرفان برقرار نه ڪيا ويندا آهن، انهن جي ڪري Ultimate Multisite جي ريجيسٽري (registration) فلو آخري قدم تي رکجي ٿي (ڪيئن صارف ماڻه جو ڪيو حساب نٿو ڪري سگهي). ان قسم جا پيگمنٽس جيڪي اهڙا مسئلا پيدا ڪن انهن جي لست هيٺ ڏنل آهي:

  * Signup Language: _Ultimate Multisite ۾ هڪ add-on آهي جيڪو هيٺ ڏنل ڪم جو يقيني بڻائي ٿو، جنهن جو نالو Ultimate Multisite: Language Switcher آهي._

  * Set Blog Description on Blog Creation: _هي Ultimate Multisite ۾ خود شامل آهي، تنهن ڪري اضافي پيگمنٽ جي ضرورت نه آهي._

  * New Blog Templates: _هي Ultimate Multisite ۾ خود شامل آهي، تنهن ڪري اضافي پيگمنٽ جي ضرورت نه آهي._

  * Set Password on Multisite Blog Creation: _هي Ultimate Multisite ۾ خود شامل آهي، تنهن ڪري اضافي پيگمنٽ جي ضرورت نه آهي._

  * Signup TOS: _هي Ultimate Multisite ۾ خود شامل آهي، تنهن ڪري اضافي پيگمنٽ جي ضرورت نه آهي._

  * Anti-Splog: جيتوڻيڪ اهو ٿورو کمزور آهي، پر هي به _Ultimate Multisite ۾ خود شامل آهي._

**نوٽس:** ٻيا سڀ WPMU Dev پيگمنٽس Ultimate Multisite سان عام طور تي استعمال ڪري سگهجن ٿا. مثالن ۾ _Smush_ , _Forminator_ , _Defender,_ تنهنجي ڪم ڪرڻ گهرجي.
