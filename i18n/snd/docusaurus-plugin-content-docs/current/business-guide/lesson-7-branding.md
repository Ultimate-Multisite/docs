# سبق 7: ان کي پنهنجو بڻائڻ

توهان جي گراهڪن کي ڪڏهن به ائين محسوس نه ٿيڻ گهرجي ته اهي "ڪو WordPress پلگ اِن" استعمال ڪري رهيا آهن. انهن کي ائين محسوس ٿيڻ گهرجي ته اهي FitSite استعمال ڪري رهيا آهن -- هڪ اهڙو پليٽفارم جيڪو انهن جي صنعت لاءِ ٺاهيو ويو آهي. هي سبق برانڊنگ، وائيٽ-ليبلنگ، ۽ پليٽفارم کي هڪ پراڊڪٽ وانگر محسوس ڪرائڻ بابت آهي.

## اسان ڪٿي ڇڏيو هو {#where-we-left-off}

FitSite وٽ هاڻي هڪ ڪم ڪندڙ checkout وهڪرو آهي جيڪو فٽنيس اسٽوڊيو مالڪن کي پلان جي چونڊ کان لائيو سائيٽ تائين وٺي وڃي ٿو. هاڻي اسان سڄي تجربي کي هڪ مربوط، برانڊ ٿيل پراڊڪٽ وانگر محسوس ڪرائينداسين.

## توهان جو پليٽفارم ڊومين {#your-platform-domain}

توهان جي برانڊ جو بنياد توهان جو ڊومين آهي. FitSite لاءِ:

- **مکيه ڊومين**: `fitsite.com` (توهان جي مارڪيٽنگ سائيٽ ۽ نيٽ ورڪ روٽ)
- **گراهڪ سائيٽون**: `studioname.fitsite.com` (سب ڊومين)
- **ڪسٽم ڊومين**: Growth ۽ Pro پلانن وارا گراهڪ پنهنجو ڊومين ميپ ڪري سگهن ٿا

### پنهنجو ڊومين سيٽ اپ ڪرڻ {#setting-up-your-domain}

1. پنهنجو پليٽفارم ڊومين رجسٽر ڪريو
2. ان کي پنهنجي هوسٽنگ فراهم ڪندڙ ڏانهن پوائنٽ ڪريو
3. گراهڪن جي سب ڊومينن لاءِ وائلڊ ڪارڊ DNS (`*.fitsite.com`) ترتيب ڏيو
4. پڪ ڪريو ته وائلڊ ڪارڊ SSL فعال آهي

تفصيلي هدايتن لاءِ [ڊومين ميپنگ ڪيئن ترتيب ڏجي](/user-guide/domain-mapping/how-to-configure-domain-mapping) ڏسو.

## Admin تجربي جي وائيٽ-ليبلنگ {#white-labeling-the-admin-experience}

جڏهن ڪو فٽنيس اسٽوڊيو مالڪ پنهنجي سائيٽ Dashboard ۾ لاگ اِن ٿئي، ته هن کي توهان جي برانڊ نظر اچڻ گهرجي، WordPress يا Ultimate Multisite برانڊنگ نه.

### ڪسٽم لاگ اِن صفحو {#custom-login-page}

WordPress لاگ اِن صفحي کي ترتيب ڏيو ته جيئن هي ڏيکاري:

- توهان جو FitSite لوگو
- فٽنيس لاءِ مناسب پسمنظر جون تصويرون
- توهان جا برانڊ رنگ

### Dashboard برانڊنگ {#dashboard-branding}

[Admin Page Creator](/addons/admin-page-creator) addon يا ڪسٽم CSS استعمال ڪري:

- WordPress لوگو کي پنهنجي FitSite لوگو سان مٽايو
- Admin رنگن جي اسڪيم کي پنهنجي برانڊ سان ملائڻ لاءِ ترتيب ڏيو
- فٽنيس سان لاڳاپيل تڪڙين لنڪن ۽ مدد جي وسيلن سان هڪ ڪسٽم Dashboard وجيٽ شامل ڪريو

### ڪسٽم Admin صفحا {#custom-admin-pages}

اهڙا ڪسٽم Admin صفحا ٺاهڻ تي غور ڪريو جيڪي فٽنيس اسٽوڊيو مالڪن لاءِ سڀ کان لاڳاپيل عمل سامهون آڻين:

- "پنهنجو ڪلاس شيڊيول ايڊٽ ڪريو"
- "ٽرينر پروفائلز اپڊيٽ ڪريو"
- "پنهنجي سائيٽ ڏسو"

هي سکڻ جي ڏکيائي کي گهٽائي ٿو، ڇو ته اهو معياري WordPress مينيو ۾ انهن کي لڪائڻ بدران، نِچ سان لاڳاپيل عملن کي سامهون ۽ مرڪز ۾ رکي ٿو.

## پنهنجي رابطن جي برانڊنگ {#branding-your-communications}

هر اي ميل، انوائس، ۽ نوٽيفڪيشن توهان جي برانڊ کي مضبوط ڪرڻ گهرجي.

### سسٽم اي ميلون {#system-emails}

**Ultimate Multisite > Settings > Emails** ڏانهن وڃو ۽ سڀ سسٽم اي ميلون ڪسٽمائيز ڪريو:

- **موڪليندڙ جو نالو**: FitSite
- **موڪليندڙ جي اي ميل**: hello@fitsite.com
- **اي ميل ٽيمپليٽس**: پنهنجي برانڊ رنگن ۽ لوگو استعمال ڪريو
- **ٻولي**: هر هنڌ فٽنيس سان مخصوص

ڪسٽمائيز ڪرڻ لاءِ اهم اي ميلون:

| اي ميل | عام ورزن | FitSite ورزن |
|-------|----------------|-----------------|
| ڀليڪار | "توهان جي نئين سائيٽ تيار آهي" | "توهان جي فٽنيس اسٽوڊيو ويب سائيٽ لائيو آهي" |
| ادائيگي جي رسيد | "ادائيگي ملي وئي" | "FitSite سبسڪرپشن جي ادائيگي تصديق ٿي وئي" |
| ٽرائل ختم ٿيڻ | "توهان جو ٽرائل جلد ختم ٿي رهيو آهي" | "توهان جو FitSite ٽرائل 3 ڏينهن ۾ ختم ٿيندو -- پنهنجي اسٽوڊيو ويب سائيٽ لائيو رکو" |

### انوائسز {#invoices}

انوائس ٽيمپليٽس کي هنن سان ڪسٽمائيز ڪريو:

- توهان جو FitSite لوگو ۽ برانڊ رنگ
- توهان جي ڪاروبار جا تفصيل
- فٽنيس سان مخصوص پراڊڪٽ نالا (عام پلان IDs نه)

## گراهڪ-سامهون سائيٽ {#the-customer-facing-site}

توهان جي مکيه ڊومين (`fitsite.com`) کي هڪ مارڪيٽنگ سائيٽ جي ضرورت آهي جيڪا پليٽفارم وڪڻي. هي Ultimate Multisite نيٽ ورڪ Admin کان الڳ آهي -- هي توهان جي ڪاروبار جو عوامي چهرو آهي.

اهم صفحا:

- **هوم پيج**: فٽنيس ڪاروبارن لاءِ صاف قدر جي تجويز
- **خاصيتون**: FitSite ڇا ڪري ٿو، فٽنيس جي اصطلاحن ۾
- **قيمتون**: توهان جا ٽي پلان، نِچ سان مخصوص خصوصيتن جي ڀيٽ سان
- **مثال**: پليٽفارم تي ٺهيل سائيٽون ڏيکاريو
- **سائن اپ**: توهان جي checkout فارم ڏانهن لنڪون

:::tip توهان جي مارڪيٽنگ سائيٽ نيٽ ورڪ سائيٽ ٿي سگهي ٿي
پنهنجي مارڪيٽنگ سائيٽ کي پنهنجي ئي نيٽ ورڪ اندر هڪ سائيٽ طور ٺاهيو. هي توهان کي ان کي ساڳئي Dashboard مان منظم ڪرڻ ڏئي ٿو ۽ توهان جي پنهنجي پليٽفارم جي صلاحيتن کي ظاهر ڪري ٿو.
:::

## گراهڪن لاءِ ڪسٽم ڊومين {#custom-domain-for-customers}

انهن پلانن وارن گراهڪن لاءِ جن ۾ ڪسٽم ڊومين شامل آهن، عمل کي صاف نموني دستاويز ڪريو:

1. گراهڪ پنهنجو ڊومين ڪنهن رجسٽرار وٽ رجسٽر يا ٽرانسفر ڪري ٿو
2. گراهڪ DNS اپڊيٽ ڪري توهان جي پليٽفارم ڏانهن پوائنٽ ڪري ٿو (بلڪل صحيح رڪارڊ فراهم ڪريو)
3. Ultimate Multisite ڊومين ميپنگ ۽ SSL سنڀالي ٿو

هن عمل لاءِ خاص طور هڪ مدد وارو مضمون يا ڄاڻ بيس داخلا ٺاهيو، جيڪا غير ٽيڪنيڪل فٽنيس اسٽوڊيو مالڪن لاءِ لکيل هجي.

## هن وقت تائين FitSite نيٽ ورڪ {#the-fitsite-network-so-far}

```
FitSite نيٽ ورڪ
├── WordPress Multisite (سب ڊومين موڊ)
├── Ultimate Multisite (ترتيب ڏنل + برانڊ ٿيل)
├── پليٽفارم ڊومين (fitsite.com + وائلڊ ڪارڊ SSL)
├── سائيٽ ٽيمپليٽس (Studio Essential, Gym Pro, Fitness Chain)
├── پراڊڪٽس (Starter, Growth, Pro + Order Bumps)
├── Checkout وهڪرو (نِچ سان مخصوص، آزمائشي)
├── برانڊنگ
│   ├── ڪسٽم لاگ اِن صفحو
│   ├── برانڊ ٿيل Admin Dashboard
│   ├── نِچ سان مخصوص سسٽم اي ميلون
│   ├── برانڊ ٿيل انوائسز
│   └── fitsite.com تي مارڪيٽنگ سائيٽ
└── onboarding وهڪري لاءِ تيار (اڳيون سبق)
```

## هن سبق ۾ اسان ڇا ٺاهيو {#what-we-built-this-lesson}

- **پليٽفارم ڊومين ۽ DNS** برانڊ ٿيل تجربي لاءِ ترتيب ڏنل
- **وائيٽ-ليبل ٿيل Admin** سڄي FitSite برانڊنگ سان
- **ڪسٽمائيز ڪيل رابطا** -- اي ميلون، انوائسز، ۽ نوٽيفڪيشن سڀ برانڊ مطابق
- **هڪ مارڪيٽنگ سائيٽ** جيڪا FitSite کي فٽنيس اسٽوڊيو مالڪن کي وڪڻي ٿي
- **ڪسٽم ڊومين دستاويز** انهن گراهڪن لاءِ جيڪي پنهنجو ڊومين چاهين ٿا
