# مطابقت رکندڙ آخري نقطن لاءِ Ultimate AI Connector

مطابقت رکندڙ آخري نقطن لاءِ Ultimate AI Connector، OpenAI سان مطابقت رکندڙ AI فراهم ڪندڙ APIs کي توهان جي WordPress multisite نيٽ ورڪ سان ضم ڪري ٿو. ڪيترائي AI فراهم ڪندڙ ترتيب ڏيو — هر هڪ پنهنجي آخري نقطي URL، API ڪنجي، ۽ ڊفالٽ ماڊل سان — ۽ درخواستن کي پاڻمرادو fallback سان رستو ڏيو جڏهن ڪو فراهم ڪندڙ دستياب نه هجي.

## اهم خاصيتون

- **گهڻن فراهم ڪندڙن جي مدد**: ڪيترائي AI فراهم ڪندڙ ڪارڊ شامل ڪريو، هر هڪ پنهنجي آخري نقطي URL، API ڪنجي، ۽ ڊفالٽ ماڊل سان
- **ڇڪي ٻيهر ترتيب ڏيڻ**: هر ڪارڊ تي grip handle استعمال ڪندي فراهم ڪندڙن کي ترجيح موجب ترتيب ڏيو؛ درخواستون فراهم ڪندڙن کي ترتيب سان استعمال ڪنديون آهن
- **پاڻمرادو fallback رستو ڏيڻ**: جيڪڏهن بنيادي فراهم ڪندڙ دستياب نه هجي، درخواستون پاڻمرادو ايندڙ ترتيب ڏنل فراهم ڪندڙ ڏانهن fallback ٿينديون آهن
- **هر فراهم ڪندڙ لاءِ الڳ ترتيب**: هر فراهم ڪندڙ پنهنجي سڃاڻپ وارن تفصيلن ۽ ماڊل چونڊ سان الڳ ترتيب ڏنل هوندو آهي
- **ڪارڊ تي ٻڌل سيٽنگون UI**: مستحڪم WordPress 6.9+ Card جزن (CardBody, CardHeader, CardDivider) تي ٺهيل
- **شامل ڪرڻ تي پاڻمرادو کلي وڃڻ**: نوان فراهم ڪندڙ ڪارڊ پاڻمرادو کلي وڃن ٿا ته جيئن توهان فوري طور تفصيل ڀري سگهو
- **اسڪرپٽ ڪيش ختم ڪرڻ**: پلگ ان اپڊيٽ تي فراهم ڪندڙ اسڪرپٽس جو ڪيش ختم ڪيو ويندو آهي ته جيئن جديد نسخو لوڊ ٿيڻ کي يقيني بڻائجي

## گهرجون

- WordPress 5.3 يا وڌيڪ
- PHP 7.4 يا وڌيڪ
- Ultimate Multisite پلگ ان (فعال)
- OpenAI سان مطابقت رکندڙ API آخري نقطو (پاڻ ميزباني ڪيل يا ٽئين ڌر جو)

## انسٽاليشن

1. addon فائلون پنهنجي `/wp-content/plugins/` ڊائريڪٽري ۾ اپلوڊ ڪريو
2. WordPress ۾ 'Plugins' مينيو ذريعي پلگ ان فعال ڪريو
3. addon ترتيب ڏيڻ لاءِ **Ultimate Multisite → AI Connector** ڏانهن وڃو

## ترتيب

### گهڻن فراهم ڪندڙن جي سيٽ اپ

ترتيب واري اسڪرين ڪارڊ تي ٻڌل لي آئوٽ استعمال ڪري ٿي، جتي هر ڪارڊ هڪ AI فراهم ڪندڙ جي نمائندگي ڪري ٿو. توهان پنهنجي ضرورت موجب جيترا فراهم ڪندڙ چاهيو شامل ڪري سگهو ٿا.

#### فراهم ڪندڙ شامل ڪرڻ

1. نيٽ ورڪ ايڊمن ۾ **Ultimate Multisite → AI Connector** ڏانهن وڃو
2. **فراهم ڪندڙ شامل ڪريو** تي ڪلڪ ڪريو
3. نئون فراهم ڪندڙ ڪارڊ ظاهر ٿيندو ۽ پاڻمرادو کلي ويندو
4. فراهم ڪندڙ جا تفصيل ڀريو:
   - **فراهم ڪندڙ جو نالو** — هن فراهم ڪندڙ کي سڃاڻڻ لاءِ هڪ ليبل (مثال طور "OpenAI"، "Mistral"، "Local Ollama")
   - **آخري نقطي URL** — OpenAI سان مطابقت رکندڙ API جو بنيادي URL (مثال طور `https://api.openai.com/v1`)
   - **API ڪنجي** — هن فراهم ڪندڙ لاءِ توهان جي API ڪنجي
   - **ڊفالٽ ماڊل** — هن فراهم ڪندڙ لاءِ ڊفالٽ طور استعمال ٿيڻ وارو ماڊل سڃاڻپ ڪندڙ (مثال طور `gpt-4o`، `mistral-small`)
5. **سيٽنگون محفوظ ڪريو** تي ڪلڪ ڪريو

#### فراهم ڪندڙ هٽائڻ

ڪنهن به فراهم ڪندڙ ڪارڊ تي **هٽايو** بٽڻ تي ڪلڪ ڪري ان کي حذف ڪريو. تبديلي لاڳو ڪرڻ لاءِ سيٽنگون محفوظ ڪريو.

#### فراهم ڪندڙن کي ٻيهر ترتيب ڏيڻ

ڪنهن به فراهم ڪندڙ ڪارڊ جي کاٻي پاسي grip icon (⣿) کي ڇڪي ان کي ٻيهر ترتيب ڏيو. ترتيب fallback ترجيح طئي ڪري ٿي: درخواستون پهرين فراهم ڪندڙ ڏانهن موڪليون وينديون آهن ۽ جيڪڏهن درخواست ناڪام ٿئي ته ايندڙ فراهم ڪندڙن ڏانهن fallback ٿينديون آهن.

### Fallback رستو ڏيڻ

جڏهن بنيادي فراهم ڪندڙ ڏانهن درخواست ناڪام ٿئي — timeout، rate limit، يا خدمت جي بندش سبب — connector پاڻمرادو فهرست ۾ ايندڙ فراهم ڪندڙ سان ٻيهر ڪوشش ڪري ٿو. اهو فهرست ۾ هيٺ تائين جاري رهندو آهي جيستائين ڪو فراهم ڪندڙ ڪامياب ٿئي يا سڀ فراهم ڪندڙ آزمائجي وڃن.

| صورتحال | رويو |
|----------|-----------|
| بنيادي فراهم ڪندڙ جواب ڏئي ٿو | درخواست عام طور تي پوري ڪئي وڃي ٿي |
| بنيادي فراهم ڪندڙ ناڪام ٿئي ٿو | درخواست ايندڙ فراهم ڪندڙ سان ٻيهر آزمائي وڃي ٿي |
| سڀ فراهم ڪندڙ ناڪام ٿين ٿا | سڏيندڙ ڏانهن غلطي واپس ڪئي وڃي ٿي |

Fallback سڏيندڙ پلگ ان يا theme لاءِ شفاف هوندو آهي — انهن کي يا ته ڪامياب جواب ملي ٿو يا آخري غلطي، ڀلي ڪيترا به فراهم ڪندڙ آزمايا ويا هجن.

### فراهم ڪندڙ جي ترتيب جو حوالو

| فيلڊ | گهربل | وضاحت |
|-------|----------|-------------|
| فراهم ڪندڙ جو نالو | ها | هن فراهم ڪندڙ لاءِ ڏيکاريل ليبل |
| آخري نقطي URL | ها | OpenAI سان مطابقت رکندڙ API جو بنيادي URL |
| API ڪنجي | ها | هن فراهم ڪندڙ لاءِ تصديق جي ڪنجي |
| ڊفالٽ ماڊل | ها | جڏهن درخواست ۾ ڪو ماڊل بيان نه ڪيو وڃي ته استعمال ٿيندڙ ماڊل سڃاڻپ ڪندڙ |

## مسئلا حل ڪرڻ

### ڪو به فراهم ڪندڙ فهرست ۾ ناهي

جيڪڏهن محفوظ ڪرڻ کان پوءِ فراهم ڪندڙن جي فهرست خالي هجي، تصديق ڪريو ته:

- توهان فراهم ڪندڙ شامل ڪرڻ کان پوءِ **سيٽنگون محفوظ ڪريو** تي ڪلڪ ڪيو
- توهان جي browser سيٽنگون صفحي کي محفوظ ٿيڻ کان نه روڪيو (JavaScript غلطين لاءِ browser console چيڪ ڪريو)
- پلگ ان فعال آهي ۽ توهان نيٽ ورڪ ايڊمن اسڪرين تي آهيو

### سڀني فراهم ڪندڙن لاءِ درخواستون ناڪام ٿي رهيون آهن

- تصديق ڪريو ته هر فراهم ڪندڙ جو آخري نقطي URL توهان جي سرور تان پهچ ۾ آهي (صرف توهان جي browser تان نه)
- چيڪ ڪريو ته هر API ڪنجي صحيح آهي ۽ ختم نه ٿي آهي
- سندن dashboard ۾ فراهم ڪندڙ جي rate-limit ۽ quota حالت جو جائزو وٺو
- connector مان تفصيلي غلطي پيغام پڪڙڻ لاءِ WordPress debug logging (`WP_DEBUG_LOG`) فعال ڪريو

### ڪو فراهم ڪندڙ fallback لاءِ استعمال نه ٿي رهيو آهي

Drag-to-reorder fallback ترتيب تبديل ڪري ٿو. تصديق ڪريو ته جنهن فراهم ڪندڙ کان توهان fallback درخواستون وٺڻ جي اميد رکو ٿا، اهو فهرست ۾ صحيح جاءِ تي رکيل آهي ۽ ان جو آخري نقطي URL ۽ API ڪنجي صحيح آهن.

### نئون فراهم ڪندڙ ڪارڊ پاڻمرادو نٿو کلي

Auto-expand لاءِ توهان جي browser ۾ JavaScript فعال هجڻ ضروري آهي. جيڪڏهن ڪارڊ ڪلڪ ڪرڻ تي نه کلي، browser console ۾ غلطيون چيڪ ڪريو. ڪارڊ جو مواد هميشه ڪارڊ header تي ڪلڪ ڪري رسائي لائق آهي، ڀلي auto-expand trigger نه ٿئي.
