# گهڻ-ڪرنسي

Ultimate Multisite: گهڻ-ڪرنسي توهان جي نيٽورڪ کي گهڻين ڪرنسين ۾ ادائگيون قبول ڪرڻ جي قابل بڻائي ٿو. اهو خودڪار طور تي زنده يا هٿرادو مٽا سٽا جي شرحن سان قيمتون تبديل ڪري ٿو، توهان کي ڪنهن به ڪرنسي لاءِ في-پراڊڪٽ ٺيڪ قيمتون مقرر ڪرڻ ڏئي ٿو، ۽ جاگرافيائي هنڌ ذريعي هر گهمندڙ جي ترجيحي ڪرنسي سڃاڻي ٿو — سڀ ڪجهه هڪ ئي ايڊمن Dashboard مان منظم ٿئي ٿو.

## اهم خاصيتون {#key-features}

- **خودڪار مٽا سٽا شرح تبديلي** — قيمتون توهان جي بنيادي ڪرنسي مان توهان جي چونڊيل فراهم ڪندڙ کان ورتل شرحن سان تبديل ٿين ٿيون
- **في-پراڊڪٽ قيمت اووررائيڊس** — خودڪار تبديلي کي پاسو ڪرڻ لاءِ ڪنهن به پراڊڪٽ تي ٺيڪ ڪرنسي-مخصوص قيمتون مقرر ڪريو
- **ڪرنسي چونڊيندڙ سائن اپ فيلڊ** — گراهڪن کي چيڪ آئوٽ/سائن اپ دوران پنهنجي ترجيحي ڪرنسي چونڊڻ ڏئي ٿو
- **گهڻا مٽا سٽا شرح فراهم ڪندڙ** — ECB، Frankfurter، Stripe، يا مڪمل طور هٿرادو شرحون
- **جاگرافيائي هنڌ تي ٻڌل ڪرنسي سڃاڻپ** — گهمندڙ جي ملڪ سان ملندڙ ڪرنسي پاڻمرادو اڳواٽ چونڊي ٿو
- **ايڊمن Dashboard** — فعال ڪرنسين، مٽا سٽا شرحن، ۽ فراهم ڪندڙ سيٽنگن کي هڪ هنڌان منظم ڪريو

## گهرجون {#requirements}

- Ultimate Multisite 2.0.0 يا وڌيڪ
- WordPress Multisite انسٽاليشن
- PHP 7.4 يا وڌيڪ

## انسٽاليشن {#installation}

1. plugin فائلون `/wp-content/plugins/ultimate-multisite-multi-currency` تي اپلوڊ ڪريو
2. WordPress ۾ **پلگ انز** اسڪرين ذريعي plugin کي فعال ڪريو (نيٽورڪ تي فعال ڪريو)
3. ايڊمن Dashboard کولڻ لاءِ **Ultimate Multisite → گهڻ-ڪرنسي** ڏانهن وڃو
4. اهي ڪرنسيون شامل ڪريو جيڪي توهان قبول ڪرڻ چاهيو ٿا ۽ پنهنجو مٽا سٽا شرح فراهم ڪندڙ ترتيب ڏيو

## ترتيب {#configuration}

### ايڊمن Dashboard {#admin-dashboard}

گهڻ-ڪرنسي ايڊمن Dashboard (**Ultimate Multisite → گهڻ-ڪرنسي**) مرڪزي هنڌ آهي جتي توهان:

- ڪرنسيون شامل، فعال، يا غير فعال ڪري سگهو ٿا
- مٽا سٽا شرح فراهم ڪندڙ چونڊي سگهو ٿا ۽ گهرج تي شرحون تازيون ڪري سگهو ٿا
- سڀني فعال ڪرنسين لاءِ موجوده شرحون ڏسي سگهو ٿا
- جڏهن خودڪار فراهم ڪندڙ مناسب نه هجن تڏهن هٿرادو شرحون مقرر ڪري سگهو ٿا

### مٽا سٽا شرح فراهم ڪندڙ {#exchange-rate-providers}

| فراهم ڪندڙ | وضاحت |
|---|---|
| **ECB** | يورپي مرڪزي بئنڪ — EUR-تي ٻڌل نيٽورڪن لاءِ مفت روزاني شرحون |
| **Frankfurter** | اوپن-سورس ECB mirror API — مفت، API key گهربل ناهي |
| **Stripe** | Stripe جون زنده FX شرحون استعمال ڪري ٿو — فعال Stripe انٽيگريشن گهربل آهي |
| **هٿرادو** | توهان شرحون سڌو داخل ڪريو ٿا؛ ڪا ٻاهرين API ڪال نه |

پنهنجو فراهم ڪندڙ **Ultimate Multisite → گهڻ-ڪرنسي → سيٽنگون → مٽا سٽا شرح فراهم ڪندڙ** هيٺ چونڊيو. شرحون ڪنهن به وقت هٿرادو تازيون ڪري سگهجن ٿيون يا خودڪار طور اپڊيٽ ٿيڻ لاءِ شيڊيول ڪري سگهجن ٿيون.

### في-پراڊڪٽ قيمت اووررائيڊس {#per-product-pricing-overrides}

جڏهن خودڪار تبديلي ڪافي درست نه هجي، ته توهان ڪنهن به پراڊڪٽ تي مخصوص ڪرنسي لاءِ ٺيڪ قيمت مقرر ڪري سگهو ٿا:

1. پراڊڪٽ ايڊيٽر کوليو (**Ultimate Multisite → پراڊڪٽس → ترميم**)
2. **ڪرنسي قيمت بندي** سيڪشن تائين اسڪرول ڪريو
3. هر ان ڪرنسي لاءِ ٺيڪ قيمت داخل ڪريو جنهن کي توهان اووررائيڊ ڪرڻ چاهيو ٿا
4. پراڊڪٽ محفوظ ڪريو

اووررائيڊ کان سواءِ پراڊڪٽس توهان جي بنيادي ڪرنسي مان تبديل ٿيل قيمت استعمال ڪن ٿا.

### ڪرنسي چونڊيندڙ سائن اپ فيلڊ {#currency-selector-signup-field}

ڪرنسي چونڊيندڙ فيلڊ توهان جي سائن اپ/چيڪ آئوٽ وهڪري ۾ شامل ڪري سگهجي ٿو ته جيئن گراهڪ رجسٽريشن وقت پنهنجي ترجيحي ڪرنسي چونڊين:

1. **Ultimate Multisite → گهڻ-ڪرنسي → سيٽنگون** ڏانهن وڃو
2. **سائن اپ تي ڪرنسي چونڊيندڙ ڏيکاريو** فعال ڪريو
3. اختياري طور تي ڊفالٽ ڪرنسي مقرر ڪريو يا جاگرافيائي هنڌ کي اڳواٽ چونڊيل قدر طئي ڪرڻ ڏيو

چونڊيل ڪرنسي سبسڪرپشن سان گڏ محفوظ ٿئي ٿي ۽ سڀني ايندڙ بلنگ لاءِ استعمال ٿئي ٿي.

### جاگرافيائي هنڌ تي ٻڌل ڪرنسي سڃاڻپ {#geolocation-based-currency-detection}

جڏهن جاگرافيائي هنڌ سڃاڻپ فعال هجي، addon گهمندڙ جي ملڪ کي ڪرنسي سان ملائي ٿو ۽ ان کي ڪرنسي چونڊيندڙ فيلڊ ۾ اڳواٽ چونڊي ٿو. فعال ڪرڻ لاءِ:

1. **Ultimate Multisite → گهڻ-ڪرنسي → سيٽنگون** ڏانهن وڃو
2. **گهمندڙ جي هنڌ مان ڪرنسي سڃاڻو** فعال ڪريو
3. **جاگرافيائي هنڌ ميپنگ** ٽيبل ۾ ملڪن کي ڪرنسين سان ميپ ڪريو

جيڪڏهن گهمندڙ جي ملڪ لاءِ ڪا ميپنگ موجود نه هجي، ته نيٽورڪ جي بنيادي ڪرنسي fallback طور استعمال ٿئي ٿي.

## اڪثر پڇيا ويندڙ سوال {#frequently-asked-questions}

### جيڪڏهن مٽا سٽا شرح فراهم ڪندڙ دستياب نه هجي ته ڇا ٿيندو؟ {#what-happens-if-an-exchange-rate-provider-is-unavailable}

addon آخري ڀيرو ڪاميابي سان ورتل شرحن ڏانهن fallback ڪري ٿو. جيڪڏهن ڪڏهن به ڪا شرح نه ورتي وئي هجي، ته اهو توهان جي ترتيب ڏنل هٿرادو شرحن ڏانهن fallback ڪري ٿو. حفاظتي جال طور هٿرادو شرحون ترتيب ڏيڻ جي سفارش ڪئي وڃي ٿي.

### ڇا مان مختلف ڪرنسين لاءِ مختلف فراهم ڪندڙ استعمال ڪري سگهان ٿو؟ {#can-i-use-different-providers-for-different-currencies}

نه — سڀني ڪرنسين لاءِ هڪ وقت ۾ هڪ فراهم ڪندڙ فعال هوندو آهي. جيڪڏهن توهان کي انفرادي شرحن تي مڪمل ڪنٽرول گهرجي ته **هٿرادو** استعمال ڪريو.

### ڇا جاگرافيائي هنڌ لاءِ الڳ plugin گهربل آهي؟ {#does-geolocation-require-a-separate-plugin}

ڪا ٻاهرين جاگرافيائي هنڌ plugin گهربل ناهي. addon WordPress جي ٺهيل IP-کان-ملڪ سڃاڻپ استعمال ڪري ٿو. وڌيڪ درستگي لاءِ، مهيا ڪيل filter hooks ذريعي وقف ٿيل جاگرافيائي هنڌ خدمت ضم ڪري سگهجي ٿي.

### ڇا في-پراڊڪٽ اووررائيڊس ٽيڪس کان اڳ لاڳو ٿين ٿا يا پوءِ؟ {#are-per-product-overrides-applied-before-or-after-tax}

في-پراڊڪٽ اووررائيڊس هدف ڪرنسي ۾ ٽيڪس کان اڳ قيمت مقرر ڪن ٿا. ٽيڪس توهان جي موجوده ٽيڪس ترتيب استعمال ڪندي اووررائيڊ قيمت جي مٿان حساب ڪيو وڃي ٿو.
