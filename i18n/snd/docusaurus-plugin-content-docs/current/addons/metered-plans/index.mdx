# ميٽرڊ منصوبا {#metered-plans}

Ultimate Multisite: ميٽرڊ منصوبا addon لچڪدار ميٽرڊ بلنگ صلاحيتون متعارف ڪرائي توهان جي گراهڪن کي بل ڪرڻ جو طريقو بدلائي ٿو. سخت مقرر حدن بدران، جيڪي حد کان وڌڻ تي استعمال ڪندڙن کي روڪين ٿيون، اضافي استعمال لاءِ overage چارجز فعال ڪريو جيڪي پاڻمرادو گراهڪن کي بل ڪن ٿا.

## اهم خاصيتون {#key-features}

* **ميٽرڊ بلنگ**: مقرر درجن بدران حقيقي استعمال جي بنياد تي گراهڪن کان چارج وٺو
* **Overage انتظام**: منصوبي جي حدن کان وڌيڪ استعمال لاءِ پاڻمرادو بل ڪريو
* **استعمال جي ٽريڪنگ**: گراهڪ جي وسيلن جي استعمال کي حقيقي وقت ۾ مانيٽر ڪريو
* **لچڪدار قيمت سازي**: مختلف وسيلن لاءِ ڪسٽم شرحون مقرر ڪريو (اسٽوريج، بينڊوڊٿ، سائيٽون، وغيره)
* **AI ٽوڪن بلنگ**: گراهڪ جي سب سائيٽن تي AI استعمال لاءِ input ۽ output ٽوڪن ٽريڪ ۽ بل ڪريو
* **پاڻمرادو انوائسنگ**: استعمال جون حدون وڌڻ تي پاڻمرادو انوائسز ٺاهيو
* **استعمال جون رپورٽون**: گراهڪن ۽ منتظمين لاءِ تفصيلي استعمال تجزيا ۽ رپورٽنگ

## هنن لاءِ بهترين {#perfect-for}

* **هاسٽنگ فراهم ڪندڙ**: اهڙا scalable منصوبا پيش ڪريو جيڪي گراهڪ جي ضرورتن سان وڌن
* **SaaS پليٽفارمز**: حقيقي وسيلن جي استعمال جي بنياد تي بل ڪريو
* **ايجنسي نيٽ ورڪ**: ڪلائنٽ منصوبن لاءِ لچڪدار قيمت سازي جا ماڊل
* **Reseller Hosting**: ڀائيوارن کي استعمال جي بنياد تي قيمت سازي پيش ڪرڻ جي اجازت ڏيو

## اهو ڪيئن ڪم ڪري ٿو {#how-it-works}

1. ميٽرڊ وسيلا ترتيب ڏيو (ڊسڪ اسپيس، بينڊوڊٿ، سائيٽن جو تعداد، وغيره)
2. بنيادي منصوبي جون حدون ۽ overage شرحون مقرر ڪريو
3. نظام پاڻمرادو توهان جي نيٽ ورڪ ۾ استعمال ٽريڪ ڪري ٿو
4. جڏهن گراهڪ حدن کان وڌن ٿا، overage چارجز حساب ڪيا وڃن ٿا
5. بلنگ چڪر جي بنياد تي انوائسز پاڻمرادو ٺاهيا وڃن ٿا
6. گراهڪ پنهنجي استعمال ۽ ايندڙ چارجز کي حقيقي وقت ۾ ڏسي سگهن ٿا

## AI ٽوڪن بلنگ {#ai-token-billing}

نسخو 1.1.0 انهن نيٽ ورڪن لاءِ AI ٽوڪن بلنگ شامل ڪري ٿو جيڪي WordPress AI Client SDK ذريعي AI خاصيتون پيش ڪن ٿا. ميٽرڊ منصوبا مڪمل ٿيل AI جنريشن درخواستن کي ٻڌن ٿا، موجوده سب سائيٽ لاءِ input ۽ output ٽوڪن ڳڻپون رڪارڊ ڪن ٿا، ۽ انهن ٽوڪنز کي گراهڪ جي موجوده بلنگ مدت لاءِ بل لائق AI خرچ ۾ تبديل ڪن ٿا.

AI ٽوڪن بلنگ استعمال ڪريو جڏهن توهان گراهڪ جي سائيٽن تي AI اوزار پيش ڪرڻ چاهيو ٿا، بغير اڻٿيڻي ماڊل خرچ پاڻ ڀرڻ جي. هر درخواست کي سب سائيٽ، گراهڪ، ميمبرشپ، فراهم ڪندڙ، ۽ ماڊل سان منسوب ڪيو وڃي ٿو ته جيئن منتظمين استعمال جي آڊٽ ڪري سگهن، جڏهن ته گراهڪ ڏسي سگهن ته سندن AI استعمال سندن اڪائونٽ کي ڪيئن متاثر ڪري ٿو.

### في ٽوڪن شرحون ترتيب ڏيو {#configure-per-token-rates}

AI شرحون هر ماڊل لاءِ الڳ input ۽ output شرحن طور ترتيب ڏنل آهن. شرحون في 1 ملين ٽوڪن حساب ڪيون وڃن ٿيون، جيڪو عام AI فراهم ڪندڙن جي قيمت وارن صفحن سان ملندڙ آهي.

1. نيٽ ورڪ ايڊمن ۾، **Multisite Ultimate → Settings → Metered Plans** کوليو.
2. ان پيداوار يا ميمبرشپ لاءِ AI خرچ ميٽرنگ فعال ڪريو جنهن کي AI استعمال بل ڪرڻ گهرجي.
3. پنهنجي connector پاران موٽايل ماڊل IDs استعمال ڪندي ماڊل شرح entries شامل ڪريو، جهڙوڪ `gpt-4o`، `gpt-4o-mini`، يا ٻيو OpenAI-compatible ماڊل identifier.
4. پنهنجي سائيٽ جي ڪرنسي ۾ في 1 ملين ٽوڪن لاءِ الڳ **Input Rate** ۽ **Output Rate** قدر داخل ڪريو.
5. گراهڪ جي شامل AI خرچ allowance ۽ منصوبي لاءِ overage رويو مقرر ڪريو.

جڏهن صحيح ماڊل ID ترتيب ڏنل نه هجي، ميٽرڊ منصوبا ماڊل prefix سان match ڪري سگهن ٿا. مثال طور، `claude-sonnet-4` لاءِ ترتيب ڏنل شرح انهن timestamped ماڊل IDs تي لاڳو ٿي سگهي ٿي جيڪي انهي prefix سان شروع ٿين ٿا. ڊولپرز آخري rate table کي [`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates) filter سان پڻ adjust ڪري سگهن ٿا.

### سپورٽ ٿيل AI Connectors {#supported-ai-connectors}

AI ٽوڪن بلنگ انهن connectors سان ڪم ڪري ٿي جيڪي WordPress AI Client SDK generation events جاري ڪن ٿا ۽ response metadata ۾ token usage شامل ڪن ٿا. هن ۾ هتي documented Ultimate Multisite AI connector خاندان شامل آهي:

* [Compatible Endpoints لاءِ Ultimate AI Connector](../ultimate-ai-connector-compatible-endpoints/) OpenAI-compatible APIs لاءِ، جنهن ۾ OpenAI، Mistral، Ollama-compatible gateways، ۽ ٻيا فراهم ڪندڙ شامل آهن جيڪي compatible token usage metadata ظاهر ڪن ٿا
* [WebLLM لاءِ Ultimate AI Connector](../ultimate-ai-connector-webllm/) browser-native WebLLM استعمال لاءِ جڏهن درخواستون AI Client SDK event flow ذريعي route ڪيون وڃن
* [Gratis AI Agent](../gratis-ai-agent/) پاران استعمال ٿيندڙ AI provider connectors جڏهن چونڊيل فراهم ڪندڙ SDK ذريعي prompt ۽ completion token counts رپورٽ ڪري ٿو

ڪسٽم connectors معياري `wp_ai_client_before_generate_result` ۽ `wp_ai_client_after_generate_result` events کي model، provider، ۽ token usage metadata سان fire ڪري حصو وٺي سگهن ٿا. Connector settings مکيه سائيٽ کان سب سائيٽن تائين لاڳو ڪيون وڃن ٿيون ته جيئن گراهڪ جون سائيٽون نيٽ ورڪ منظور ٿيل فراهم ڪندڙ ۽ حدون استعمال ڪن.

### گراهڪ Account استعمال {#customer-account-usage}

گراهڪ پنهنجي Account علائقي ۾ AI استعمال کي پنهنجي ٻين ميٽرڊ وسيلن سان گڏ ڏسن ٿا. AI خرچ واري قطار رڪارڊ ٿيل prompt ۽ completion tokens، ترتيب ڏنل model rates، ۽ منصوبي تي ڪنهن به شامل allowance مان نڪتل موجوده billing-period ڪل ڏيکاري ٿي.

جيڪڏهن گراهڪ وٽ فائل تي payment method آهي، شامل allowance کان مٿي استعمال کي بل لائق overage طور ورتو وڃي ٿو. جيڪڏهن منصوبو hard cutoff لاءِ ترتيب ڏنل آهي ۽ گراهڪ وٽ ڪو payment method موجود ناهي، ته allowance تي پهچڻ کان پوءِ AI درخواستون روڪي سگهجن ٿيون ۽ گراهڪ کي جاري رکڻ کان اڳ payment details شامل ڪرڻ لاءِ چيو وڃي ٿو.

## سپورٽ ٿيل وسيلا {#supported-resources}

* ڊسڪ اسٽوريج
* بينڊوڊٿ/ڊيٽا منتقلي
* سائيٽن جو تعداد
* استعمال ڪندڙن جو تعداد
* AI ٽوڪن استعمال ۽ AI خرچ
* ڪسٽم وسيلن جا قسم

## گهرجون {#requirements}

* Multisite Ultimate 2.0.0 يا وڌيڪ
* WordPress Multisite انسٽاليشن
* PHP 7.4 يا وڌيڪ

## انسٽاليشن {#installation}

1. plugin فائلون `/wp-content/plugins/ultimate-multisite-metered-plans` تي upload ڪريو
2. WordPress ۾ 'Plugins' screen ذريعي plugin کي activate ڪريو
3. Multisite Ultimate → Settings → Metered Plans ڏانهن وڃو
4. پنهنجا ميٽرڊ وسيلا ۽ قيمت سازي ترتيب ڏيو
5. product editor ۾ products لاءِ ميٽرڊ بلنگ enable ڪريو

## اڪثر پڇيا ويندڙ سوال {#frequently-asked-questions}

## جڏهن گراهڪ پنهنجي حد کان وڌي وڃي ته ڇا ٿيندو؟ {#what-happens-when-a-customer-exceeds-their-limit}

رسائي کي روڪڻ بدران، نظام اضافي استعمال کي ٽريڪ ڪري ٿو ۽ توهان جي ترتيب ڏنل شرحَن جي بنياد تي ان کي سندن ايندڙ انوائس ۾ شامل ڪري ٿو.

## ڇا مان مختلف مصنوعات لاءِ مختلف اضافي استعمال جون شرحون مقرر ڪري سگهان ٿو؟ {#can-i-set-different-overage-rates-for-different-products}

ها، هر پراڊڪٽ وٽ منفرد ميٽرڊ وسيلا ۽ اضافي استعمال جي قيمت ٿي سگهي ٿي.

## گراهڪن کي استعمال بابت ڪيئن اطلاع ڏنو ويندو آهي؟ {#how-are-customers-notified-about-usage}

گراهڪ پنهنجي موجوده استعمال کي پنهنجي Dashboard ۾ ڏسي سگهن ٿا ۽ حدن جي ويجهو پهچڻ يا انهن کان وڌڻ وقت اختياري اطلاع حاصل ڪري سگهن ٿا.

## AI ٽوڪن جي استعمال جي قيمت ڪيئن لڳائي ويندي آهي؟ {#how-is-ai-token-usage-priced}

AI ٽوڪن جي استعمال جي قيمت ماڊل جي ترتيب ڏنل ان پٽ ۽ آئوٽ پٽ شرحَن مان، هر 1 ملين ٽوڪن تي، لڳائي ويندي آهي. ميٽرڊ منصوبا هر درخواست جي پرامپٽ ۽ مڪمل ٿيڻ واري ٽوڪن ڳڻپ کي انهن شرحَن سان ضرب ڏين ٿا، نتيجي کي گراهڪ جي موجوده بلنگ-مدت واري AI خرچ ۾ شامل ڪن ٿا، ۽ منصوبي جي اجازت يا اضافي استعمال جا قاعدا لاڳو ڪن ٿا.

## ڇا مان اڃا به سخت حدون استعمال ڪري سگهان ٿو؟ {#can-i-still-use-hard-limits}

ها، ميٽرڊ بلنگ هر پراڊڪٽ لاءِ اختياري آهي. توهان روايتي سخت-حد وارن پراڊڪٽس کي ميٽرڊ پراڊڪٽس سان گڏ استعمال ڪري سگهو ٿا.
