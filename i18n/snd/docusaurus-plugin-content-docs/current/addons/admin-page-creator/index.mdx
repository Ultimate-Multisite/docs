# ايڊمن صفحو ٺاهيندڙ

## مواد
ايڊمن صفحو ٺاهيندڙ توهان کي WordPress ايڊمن Dashboard اندر نوان ايڊمن صفحا ۽ ذيلي صفحا آساني سان شامل ڪرڻ جي اجازت ڏيندو. اهو هڪ ڀليڪار صفحو يا سبق وارو صفحو ٺاهڻ لاءِ استعمال ڪري سگهجي ٿو، جيڪو نون استعمال ڪندڙن لاءِ لينڊنگ صفحي طور ڪم ڪري سگهي ٿو. توهان Beaver Builder، Elementor، Brizy، ۽ Oxygen جهڙا صفحو ٺاهيندڙ استعمال ڪري صفحن کي بنا تڪليف ڊزائن ڪري سگهو ٿا.

اهو ڪسٽم HTML، CSS ۽ JavaScript سپورٽ پڻ پيش ڪري ٿو، گڏوگڏ plan ۽ استعمال ڪندڙ جي ڪردار جي بنياد تي رسائي ڪنٽرول پڻ!

## خاصيتون
* ڪسٽم HTML ۽ CSS جي سپورٽ
* JavaScript سپورٽ
* PHP مواد موڊ
* استعمال ڪندڙ جي ڪردار جي بنياد تي اجازت
* صفحو ٺاهيندڙن لاءِ سپورٽ

## استعمال جي رهنمائي

جڏهن ته WordPress سائيٽن جي اڳئين حصي ۾ مواد شامل ڪرڻ کي واقعي آسان بڻائي ٿو، ايڊمن پاسي صفحا ۽ ذيلي صفحا ٺاهڻ لاءِ ڪجهه بنيادي ڪوڊنگ مهارتن جي ضرورت پوي ٿي. Ultimate Multisite: ايڊمن صفحو ٺاهيندڙ جو مقصد توهان جي نيٽ ورڪ سائيٽن جي پٺئين حصي ۾ اڳئين حصي واري ساڳي استعمال جي آساني آڻڻ آهي.

## ايڊمن صفحو ٺاهيندڙ ڇو استعمال ڪجي؟

شايد توهان پنهنجي clients جي ايڊمن پينل تي سبقن سان گڏ هڪ ڪسٽم صفحو پيش ڪرڻ چاهيو ٿا، شايد توهان FAQs ۽ رابطو widget سان گڏ هڪ ڪسٽم صفحو شامل ڪرڻ چاهيو ٿا. ايڊمن صفحو ٺاهيندڙ سان امڪان بي انتها آهن، ۽ اهو ئي آهي جيڪو اسان اڄ جي سبق ۾ ڳولينداسين.

## مينيو اختيار

پاسي تي، عام اختيارن واري meta-box ۾، توهان مينيو جو قسم، ليبل، ۽ گهڻو ڪجهه ترتيب ڏئي سگهندا.

### مينيو قسم

توهان جي نئين ڪسٽم ايڊمن صفحي جا ٻه مختلف مينيو قسم ٿي سگهن ٿا: مٿين سطح جو مينيو ۽ ذيلي مينيو.

مٿين سطح جو مينيو صفحو، جيئن نالي مان ظاهر آهي، توهان جو نئون ٺاهيل صفحو ڊفالٽ WordPress صفحن سان گڏ ايڊمن مينيو ۾ شامل ڪندو.

جيڪڏهن توهان مٿين سطح وارو اختيار چونڊيو ٿا، ته توهان مينيو ترتيب ۽ مينيو آئڪن پڻ چونڊي سگهندا.

جيڪڏهن توهان هن صفحي کي ذيلي مينيو صفحو بڻائڻ چاهيو ٿا، ته توهان هڪ والدين صفحو چونڊي سگهندا جنهن سان هن صفحي کي ڳنڍيو وڃي. توهان پنهنجو ڪسٽم صفحو ڊفالٽ WordPress صفحن ۾ يا پنهنجي مٿين سطح وارن ڪسٽم ايڊمن صفحن ۾ شامل ڪري سگهو ٿا.

## ترقي يافته اختيار

توهان پنهنجي ڪسٽم ايڊمن صفحن ۾ هر قسم جو مواد شامل ڪرڻ جي قابل هئڻ گهرجي. ان ۾ CSS قاعدا ۽ فائلون، JavaScript، ۽ ٻاهريون libraries پڻ شامل آهن. توهان اهو Advanced Options Tab استعمال ڪندي ڪري سگهو ٿا.

### ڪسٽم CSS

CSS tab تي، توهان پنهنجو ڪسٽم CSS code شامل ڪري سگهو ٿا، گڏوگڏ ڏورانهين سائيٽن کان CSS فائلون import به ڪري سگهو ٿا (جهڙوڪ CDNs، Google Fonts، وغيره).

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ ڪسٽم CSS ايڊيٽر ٻاهرئين stylesheet import سان -->

_ڪسٽم CSS جي پڻ سپورٽ آهي. توهان ٻاهريون style فائلون پڻ شامل ڪري سگهو ٿا!_

### ڪسٽم JavaScript

ساڳيو JavaScript لاءِ به صحيح آهي. توهان پنهنجي ڪسٽم JavaScript code ۾ استعمال ڪرڻ لاءِ ٻاهريون libraries شامل ڪري سگهو ٿا.

## اجازت ناما

Advanced Options ۾ پڻ، توهان چونڊي سگهندا ته ڪهڙين ذيلي سائيٽن کي هن صفحي تائين رسائي هوندي. اهو مفيد آهي جيڪڏهن توهان مختلف plan سطحن کي مختلف خاص مواد مهيا ڪرڻ جو ارادو رکو ٿا، مثال طور.

<!-- اسڪرين شاٽ دستياب ناهي: ايڊمن صفحو ٺاهيندڙ اجازت سيٽنگون plan ۽ ڪردار جي بنياد واري رسائي ڪنٽرول لاءِ -->

_توهان وٽ مڪمل ڪنٽرول آهي ته هي صفحو ڪٿي ظاهر ٿيڻ وارو آهي_

**اهم** : صفحو ڏيکارڻ لاءِ ٻئي شرطون پوريون ٿيڻ ضروري آهن. ان جو مطلب آهي ته جيڪڏهن استعمال ڪندڙ ڪنهن ڏنل منصوبي جو ڪلائنٽ آهي، پر چونڊيل ڪردارن مان ڪو هڪ به نٿو رکي (يا ان جي ابتڙ) ته صفحو سندن پينل تي ظاهر نه ٿيندو.

## فعال

جيڪڏهن توهان ڪنهن ڪسٽم ايڊمن صفحي کي حذف ڪرڻ يا اجازتن سان ڇيڙڇاڙ ڪرڻ کانسواءِ غيرفعال ڪرڻ چاهيو ٿا، ته بس فعال اختيار تان نشان هٽايو.

مٿئين سطح جي صفحي کي غيرفعال ڪرڻ سان ان والد صفحي سان ڳنڍيل سڀئي ذيلي صفحا پاڻمرادو غيرفعال ٿي ويندا.
