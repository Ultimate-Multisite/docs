# گهڻ-ڪرائيداري

ڪارڪردگي ۽ سلامتي لاءِ ذيلي سائيٽن کي سرورن ۽ ڊيٽابيسن ۾ ورهايو

## نسخو 1.2.0 جاري ٿيڻ جون نمايان ڳالهيون

Multi-Tenancy 1.2.0 پيداوار واري الڳاوَ وارن عملن لاءِ خودمختيار ڪرائيدار جو بنياد شامل ڪري ٿو:

- مختصر مدي وارن، ٻيهر-استعمال کان محفوظ SSO دورن سان اسٽيٽليس ڪرائيدار خودڪار لاگ اِن.
- خودمختيار ڪرائيدارن لاءِ هر ذيلي سائيٽ جي ڊيٽابيس ۽ فائيل سسٽم جو الڳاوَ.
- ميزبان-آگاهه ڪرائيدار مهيا ڪرڻ لاءِ Hostinger گهڻ-ڪرائيداري صلاحيت.
- پراڻي ڊيٽا جي چڪاسن ۽ async push پروسيسنگ لاءِ خودمختيار منتقلي تصديق جا دروازا.
- Bedrock، FrankenPHP، ۽ localhost انسٽالن لاءِ ترتيب ڏئي سگهڻ وارو ساڳي-مشين ڊيٽابيس ميزبان بائنڊنگ.
- ڪرائيدار بوٽ اسٽريپ، ڊيٽابيس رائيٽر، اسڪيمه بوٽ اسٽريپ، ۽ صارف مهيا ڪرڻ ۾ بهتريون.
- وڌيڪ محفوظ خودمختيار ختم ڪرڻ، جيڪو ڪرائيدار جا اسناد صاف ڪري ٿو.

[خودمختيار منتقلي عمل](/user-guide/getting-started/sovereign-migration-workflow)، [گهڻ-ڪرائيداري الڳاوَ](/user-guide/configuration/multi-tenancy-isolation)، ۽ [خودمختيار ڪرائيدار آپريشنز](/user-guide/administration/sovereign-tenant-operations) لاءِ صارف رهنما ڏسو.

## گهرجون

- WordPress 5.3 يا ان کان مٿي
- PHP 7.4 يا ان کان مٿي
- Multisite Ultimate پلگ ان (فعال)

## انسٽاليشن

1. addon فائيلون پنهنجي `/wp-content/plugins/` ڊائريڪٽري ۾ اپلوڊ ڪريو
2. WordPress ۾ 'Plugins' مينيو ذريعي پلگ ان فعال ڪريو
3. نيٽ ورڪ ايڊمن ۾ addon سيٽنگون ترتيب ڏيو

### ٽيسٽنگ

addon ۾ جامع ٽيسٽنگ فريم ورڪ شامل آهي:

- **يونٽ ٽيسٽ**: WordPress ٽيسٽ سوٽ سان PHPUnit-بنياد ٽيسٽ
- **بنيادي ٽيسٽ ڪلاس**: عام ٽيسٽ منظرنامن لاءِ مددگار طريقا مهيا ڪري ٿو
- **ٽيسٽ ڪوريج**: ڪوڊ ڪوريج ٽريڪ ڪرڻ لاءِ ترتيب ڏنل
- **Multisite ٽيسٽنگ**: ٽيسٽون multisite ماحول ۾ هلن ٿيون

### ڪوڊ معيار

- **WordPress ڪوڊنگ معيار**: PHPCS ذريعي لاڳو ڪيل
- **PHP 7.4+ مطابقت**: جديد PHP خاصيتن جي حمايت
- **جامد تجزيو**: قسم چڪاس لاءِ PHPStan انضمام
- **ڪوڊ جديدڪاري**: PHP اپگريڊن لاءِ Rector قاعدا

### بلڊ سسٽم

- **اثاثن جي پروسيسنگ**: CSS/JS فائيلن لاءِ منيفڪيشن
- **ترجمو**: POT فائيل ٺاهڻ
- **پيڪيجنگ**: ورڇ لاءِ آرڪائيو ٺاهڻ
- **ترقي موڊ**: ڊيبگنگ لاءِ غير منيفاءِ ٿيل اثاثا

## ترتيب

addon مختلف ترتيب اختيارن جي حمايت ڪري ٿو، جيڪي نيٽ ورڪ ايڊمن انٽرفيس ذريعي دستياب آهن.

### خودمختيار ڪرائيدار ترتيب

خودمختيار ڪرائيدارن لاءِ، ڪرائيدار ڊيٽابيس، فائيل سسٽم روٽ، ڊيٽابيس ميزبان بائنڊنگ، ۽ ميزبان صلاحيت ترتيب ڏيو، ان کان اڳ جو ٽريفڪ کي ڪرائيدار ڏانهن روٽ ڪيو وڃي. هر اهڙي تبديلي کان پوءِ منتقلي تصديق هلائو جيڪا ڊيٽابيس اسناد، ميزبان بائنڊنگ، روٽنگ، يا صارف مهيا ڪرڻ تي اثرانداز ٿئي.

`localhost` صرف تڏهن استعمال ڪريو جڏهن ڪرائيدار رن ٽائيم ۽ ڊيٽابيس گرانٽس مقامي ساڪيٽ يا ساڳي-مشين رسائي جي اميد رکن. `127.0.0.1` يا سروس ميزبان نالو استعمال ڪريو جڏهن اهو توهان جي فراهم ڪندڙ پاران ترتيب ڏنل گرانٽ ميزبان هجي.

### Hostinger صلاحيت

Hostinger گهڻ-ڪرائيداري صلاحيت گڏيل Hostinger انضمام ٽوڪن سان ڪم ڪري ٿي. تصديق کان اڳ پڪ ڪريو ته ڊومين hPanel ۾ موجود آهي، DNS Hostinger اڪائونٽ ڏانهن اشارو ڪري ٿو، ۽ ڪرائيدار ڊيٽابيس سيٽنگون hPanel سان ملن ٿيون.

## هوڪس ۽ فلٽر

### ايڪشنز

- `ultimate-multisite-addon-template_init` - addon شروعات کان پوءِ هلائي وئي
- `ultimate-multisite-addon-template_loaded` - سڀ addon فائيلون لوڊ ٿيڻ کان پوءِ هلائي وئي

### فلٽر

- `ultimate-multisite-addon-template_settings` - addon سيٽنگون تبديل ڪريو
- `ultimate-multisite-addon-template_enabled` - addon فعال/غير فعال حيثيت کي اوور رائيڊ ڪريو
