# Tutor Multisite Compatibility {#tutor-multisite-compatibility}

هڪ WordPress پلگ اِن جيڪو multisite ماحولن ۾ Tutor LMS لاءِ compatibility درستگيون شامل ڪري ٿو.

## وضاحت {#description}

هي پلگ اِن WordPress multisite نيٽ ورڪن تي Tutor LMS هلائڻ لاءِ ضروري compatibility درستگيون فراهم ڪري ٿو. اهو صارف enrollment، instructor انتظام، ڪورس حدون، ۽ ٻيون multisite-مخصوص functionality سنڀالي ٿو.

## خاصيتون {#features}

- **Database ٽيبل انتظام**: پاڻمرادو گم ٿيل Tutor LMS ٽيبلز کي سڃاڻي ۽ ٺاهي ٿو جيڪي multisite foreign key پابندين سبب ناڪام ٿين ٿيون
- **Multisite صارف Enrollment**: نيٽ ورڪ سائيٽن ۾ صارف enrollment سنڀالڻ لاءِ custom AJAX handlers
- **Instructor ڪردار انتظام**: multisite ماحولن ۾ پاڻمرادو instructor ڪردار assignment ۽ انتظام
- **ڪورس حدون**: سائيٽ plans جي بنياد تي ڪورس ۽ funnel ٺاهڻ جون حدون سنڀالڻ لاءِ Ultimate Multisite سان integration
- **Custom ايڊمن Views**: multisite-aware instructor فهرست ۽ انتظام interface
- **ايڊمن اوزار**: نيٽ ورڪ ۽ سائيٽ administrators لاءِ database انتظام interface

## گهرجون {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Tutor LMS پلگ اِن (`Requires Plugins` header ذريعي پاڻمرادو enforced)
- WordPress Multisite installation
- Ultimate Multisite پلگ اِن (ڪورس حدن جي functionality لاءِ)

## انسٽاليشن {#installation}

1. پلگ اِن فائلون `/wp-content/plugins/tutor-multisite-compatibility/` تي upload ڪريو
2. پڪ ڪريو ته Tutor LMS installed ۽ activated آهي
3. WordPress ۾ 'Plugins' menu ذريعي پلگ اِن activate ڪريو
4. پلگ اِن پاڻمرادو Tutor LMS سان integrate ٿيندو

## MU plugin مان منتقل ڪيل Functionality {#functionality-moved-from-mu-plugin}

هي پلگ اِن اها functionality رکي ٿو جيڪا اڳ ۾ `wp-content/mu-plugins/daves.php` ۾ هئي:

- ڪورس status change جون حدون
- ڪورس ٺاهڻ جون حدون
- CartFlows integration ۽ حدون
- Tutor instructor ڪردار انتظام
- Multisite صارف enrollment سنڀالڻ
- ڪورس product ID display

## Database انتظام {#database-management}

### مسئلو {#the-problem}

WordPress multisite ماحولن ۾، users ٽيبل ڏانهن foreign key پابندين وارن ٽيبلز لاءِ Tutor LMS ٽيبل ٺاهڻ ناڪام ٿئي ٿو. اهو ان ڪري ٿئي ٿو ڇاڪاڻ ته:

- سائيٽ-مخصوص ٽيبلز `wp_2_tutor_carts` جهڙا prefixes استعمال ڪن ٿا
- Foreign keys `wp_2_users` ڏانهن reference ڪن ٿيون (جيڪو موجود ناهي)
- اصل users ٽيبل `wp_users` آهي (نيٽ ورڪ ۾ shared)

### حل {#the-solution}

هي پلگ اِن پاڻمرادو:

1. **گم ٿيل ٽيبلز سڃاڻي ٿو**: انهن ٽيبلز لاءِ check ڪري ٿو جيڪي foreign key پابندين سبب create ٿيڻ ۾ ناڪام ٿيا
2. **صحيح References سان ٽيبلز ٺاهي ٿو**: مناسب user ٽيبل نالو استعمال ڪري ٿو (multisite ۾ `wp_users`)
3. **ڪيترائي Scenarios سنڀالي ٿو**:
   - پلگ اِن activation/reactivation
   - نيٽ ورڪ ۾ نئين سائيٽ ٺاهڻ
   - ايڊمن interface ذريعي manual ٽيبل checking

### متاثر ٿيل ٽيبلز {#affected-tables}

- `tutor_coupon_usages` - users ٽيبل ڏانهن foreign key رکي ٿو
- `tutor_carts` - users ٽيبل ڏانهن foreign key رکي ٿو
- `tutor_notification_preferences` - users ٽيبل ڏانهن foreign key سان TutorPro ٽيبل

### ايڊمن Interface {#admin-interface}

Database انتظام اوزارن تائين رسائي هتي حاصل ڪريو:

- **Network Admin**: Settings → Tutor Multisite DB
- **Site Admin**: Tools → Tutor Multisite DB

Interface توهان کي اجازت ڏئي ٿو ته:
- سڀني سائيٽن ۾ گم ٿيل ٽيبلز ڏسو
- ٽيبل ٺاهڻ manually trigger ڪريو
- database configuration check ڪريو
- تفصيلي status معلومات ڏسو
