# Bekannte Inkompatibilitäten mit anderen Plugins {#known-incompatibilities-with-other-plugins}

Nachdem Sie Ultimate Multisite installiert und aktiviert haben, können Sie Probleme bekommen wegen ungelöster Inkompatibilitäten mit anderen Plugins.

Mir wird aktiv daran gearbeitet, die Liste mit jeder neuen Version zu verkürzen, aber wenn ein Plugin hier gelistet ist, müssen Sie es möglicherweise deaktivieren, damit Ihr Ultimate Multisite Network voll funktionsfähig ist.

Hier ist eine Liste davon und was Sie in solchen Fällen tun sollten:

**Einige WPMU DEV Plugins** Einige der Plugins, die von WPMU DEV nicht mehr gewartet werden, verursachen bekannt dafür, dass der Registrierungsprozess von Ultimate Multisite beim letzten Schritt stoppt (der Benutzer kann kein Konto erstellen). Hier ist eine Liste der Plugins, die diese Probleme verursachen:

  * Signup Language: _Ultimate Multisite hat ein Add-on mit derselben Funktionalität, genannt Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin nötig._

  * New Blog Templates: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin nötig._

  * Set Password on Multisite Blog Creation: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin nötig._

  * Signup TOS: _Dies ist in Ultimate Multisite integriert, daher ist kein zusätzliches Plugin nötig._

  * Anti-Splog: Obwohl es weniger leistungsfähig ist, ist dies ebenfalls _in Ultimate Multisite integriert._

**Hinweis**: Alle anderen WPMU Dev Plugins können normal zusammen mit Ultimate Multisite verwendet werden. Beispiele hierfür sind _Smush_, _Forminator_, _Defender_ usw.
