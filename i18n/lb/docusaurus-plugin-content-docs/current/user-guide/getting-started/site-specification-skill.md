# Site Specification Fähigkeit

D'**Site Specification Fähigkeit** isch en strukturierte Wäg, um d'Zil vo Ihrer Site, d'Zielgruppe und d'Markenidentität z'erfange. Dës Information wird im Ihr **site_brief** Gedächtnis gspeicheret, wo Agenten über Sitzige hinweg bruche, um konsistenti, kontextbewussti Hilf z'biete.

## Was isch Site Specification? {#what-is-site-specification}

Site Specification isch de Prozess, d'Folgende z'dokumentiere:

- **Site Zweck**: Was Ihre Site macht und warum sie existiert
- **Zielgruppe**: Wer Ihre Site bsuecht und was sie bruucht
- **Markenidentität**: Iri Farbe, Ton und visuelle Stil
- **Geschäftszil**: Wie Erfolg für Ihre Site usgseht
- **Inhaltstruktur**: Wie Ihre Site organisiert isch

Dës Specification wird Ihr **site_brief**, es persistents Gedächtnis, wo Agenten bruche, um de Kontext vo Ihrer Site z'verstah.

## Warum Site Specification bruche? {#why-use-site-specification}

### Konsistenz über Sitzige hinweg {#consistency-across-sessions}

Ohni site_brief müesste Sie de Zweck vo Ihrer Site jedes Mal neu erkläre, wenn Sie en neui Sitzig starte. Mit ihm verstah Agenten sofort:

- D'Zil und d'Zielgruppe vo Ihrer Site
- Iri Markenfarbe und Ton
- Iri Inhaltstruktur
- Iri Geschäftszil

### Bessere Empfehlige {#better-recommendations}

Agenten bruche Ihr site_brief, um:

- Features vorzschlage, wo mit em Zweck vo Ihrer Site übereinstimmed
- Inhaltsstrukture z'empfehle, wo zu Ihre Ziel passe
- Designs vorzschlage, wo mit Iri Marke übereinstimme
- D'Vorschläg von inkompatible Features z'vermeide

### Schneller Start {#faster-onboarding}

Neui Agenten (oder Agenten in neie Sitzige) chönd schnell uf d'Führer cho, indem sie Ihr site_brief läse, statt klärendi Froge z'stelle.

## Site Specification starte {#initiating-site-specification}

### Während Theme Builder Onboarding {#during-theme-builder-onboarding}

D'Site Specification Fähigkeit wird automatisch während em **Theme Builder onboarding flow** igleitet. De Setup Assistant Agent stellt Froge und baut Ihr site_brief uf.

### Manuell starte {#manual-initiation}

Meint du d'Site-Spezifikation zu irgendeinem Zitpunkt starten?

```
"Lass uns mini Sitespezifikation definieren"
```

oder

```
"Hilf mir, eine Site-Briefing zu erstellen"
```

## Der Prozess der Sitespezifikation {#the-site-specification-process}

### Schritt 1: Zweck der Seite (Site Purpose) {#step-1-site-purpose}

De Agent fragt:

```
Was ist der Hauptzweck deiner Seite?
- E-Commerce-Shop
- Blog oder Content-Seite
- Portfolio oder Showcase
- SaaS-Anwendung
- Community oder Forum
- Sonstiges: [beschreiben]
```

Du kannst eine Kategorie auswählen oder deinen eigenen Zweck beschreiben.

### Schritt 2: Zielgruppe (Target Audience) {#step-2-target-audience}

```
Wer ist deine Hauptzielgruppe?
- Verbraucher / allgemeine Öffentlichkeit
- Geschäftsprofis
- Entwickler / technische Nutzer
- Studenten / Pädagogen
- Sonstiges: [beschreiben]

Was sind ihre Hauptbedürfnisse?
```

### Schritt 3: Markenidentität (Brand Identity) {#step-3-brand-identity}

```
Was sind deine Markenfarben?
- Primärfarbe: [Farbwähler oder Hex-Code]
- Sekundärfarbe: [Farbwähler oder Hex-Code]
- Akzentfarbe: [optional]

Wie würdest du deinen Markenton beschreiben?
- Professionell / korporativ
- Kreativ / künstlerisch
- Verspielt / lässig
- Minimal / modern
- Warm / freundlich
```

### Schritt 4: Geschäftsziele (Business Goals) {#step-4-business-goals}

```
Wie sieht Erfolg für deine Seite aus?
- Leads generieren
- Produkte verkaufen
- Community aufbauen
- Wissen teilen
- Autorität etablieren
- Sonstiges: [beschreiben]

Was ist dein wichtigster Metrik?
- Umsatz
- Nutzerengagement
- Content-Reichweite
- Conversions (Umwandlungen)
- Sonstiges
```

### Schritt 5: Inhaltsstruktur (Content Structure) {#step-5-content-structure}

```
Wie ist dein Inhalt organisiert?
- Flach (aller Inhalt auf derselben Ebene)
- Hierarchisch (Kategorien und Unterkategorien)
- Chronologisch (blog-Stil)
- Produktbasiert (Katalog)
- Sonstiges: [beschreiben]

Welche Inhaltstypen verwendest du?
- Blogbeiträge
- Produktseiten
- Fallstudien
- Dokumentation
- Videos
- Sonstiges
```

## Dein site_brief Gedächtnis (Your site_brief Memory) {#your-sitebrief-memory}

Nachdem die Sitespezifikation abgeschlossen ist, wird deine Information als **site_brief** im Gedächtnis deines Agents gespeichert. Dies ist ein strukturiertes Protokoll, das Folgendes enthält:

## Viewing und Aktualisiere Din Site-Brief {#viewing-and-updating-your-sitebrief}

### Zeige Deinen Site-Brief an {#view-your-sitebrief}

Frage de Agent:

```
"Show me my site brief"
```

oder

```
"What do you know about my site?"
```

De Agent wird dini gspeicherte Spezifikation aazeige.

### Aktualisiere Deinen Site-brief {#update-your-sitebrief}

Wenn din Site sich verändert, chasch ihn aktualisiere:

```
"Update my site brief: we're now targeting B2B customers"
```

oder

```
"Refresh my site specification"
```

Das wird d Skill Site Specification neu laufe lah mit de vorhandene Information als Startpunkt.

## Wie Agenten den site_brief bruche {#how-agents-use-sitebrief}

### Design-Empfehlige {#design-recommendations}

Wenn du nach Designänderige fragst, verwiise Agenten uf din site_brief:

```
Du: "Redesign my homepage"
Agent: "Basierend uf dim Site-Brief richtet sich dini Zielgruppe an Frauä im Alter vo 25-45 Johr mit eme warme, handwerklicher Ton. Ich werde es Design erstelle, wo dini handgmachte Schmuck und nachhaltigi Wert betont."
```

### Funktionsvorschläg {#feature-suggestions}

Agenten schlage Funktionen vor, wo mit dine Zielen übereinstimme:

```
Du: "What features should I add?"
Agent: "Für en E-Commerce-Shop, wo uf Kundeloyalität fokussiert isch, würd ich folgendes empfehle: Loyalitätsprogramm, Kundebewertige, E-Mail-Newsletter und Produktempfehlige."
```

### Inhaltsstruktur {#content-structure}

Agenten schlage d Inhaltsorganisation basierend uf din Struktur vor:

Sie: "Wie soll ich meinen Blog organisieren?"
Agent: "Da Sie en Produktkatalog mit Blog verwenden, würd ich vorschlage, die Beiträge nach Produktkategorie zu organisieren und 'Styling Guides' zu erstellen, die mehrere Produkte zusammen zeigen."

## Best Practices {#best-practices}

### Seii spezifisch {#be-specific}

Statt "allgemeine Zielgruppe" beschribt Ihre tatsächliche Zielgruppe:

- ✓ "Frauen im Alter von 25-45 Jahren, interessiert an nachhaltiger Mode"
- ✗ "Jeder"

### Regelmässig aktualisiere {#update-regularly}

Wenn sich Ihre Seite entwickelt, aktualisieren Sie Ihren site_brief:

- Wenn Sie auf eine neue Zielgruppe umstellen
- Wenn Sie neue Produktlinien hinzufügen
- Wenn sich Ihre Markenidentität ändert
- Wenn sich Ihre Geschäftsziele verschieben

### Konsistente Terminologie verwenden {#use-consistent-terminology}

Verwenden Sie dieselben Begriffe über alle Sitzungen hinweg:

- ✓ Sagen Sie immer "nachhaltige Schmuckstücke" (nicht "umweltfreundliche Schmuckstücke" und "grüne Produkte")
- ✓ Beziehen Sie sich konsistent auf Ihre Zielgruppe auf die gleiche Weise

### Kontext einbeziehen {#include-context}

Geben Sie Hintergrundinformationen, die den Agenten helfen, Ihre Entscheidungen zu verstehen:

- "Wir zielen auf Berufstätige ab, die Qualität über den Preis stellen"
- "Unsere Zielgruppe ist technikaffin und erwartet modernes Design"
- "Wir sind ein Bootstrapped Startup, daher brauchen wir kosteneffiziente Lösungen"

## Zusammenhang mit dem Theme Builder Onboarding {#relationship-to-theme-builder-onboarding}

Die Site Specification Fähigkeit ist in den **Theme Builder Onboarding Flow** integriert. Wenn Sie das Onboarding abgeschlossen haben, wird Ihr site_brief automatisch mit den von Ihnen bereitgestellten Informationen erstellt.

Sie können die Site Specification auch unabhängig ausführen, wenn Sie möchten:

- Ihre Spezifikation nach der ersten Einrichtung verfeinern
- Ihren site brief aktualisieren, während sich Ihre Seite weiterentwickelt
- Eine detaillierte Spezifikation erstellen, bevor Sie mit dem Theme Builder beginnen

## Fehlerbehebung {#troubleshooting}

**Mein site_brief wird nicht verwendet**
- Bestätigen Sie, dass der Agent Zugriff auf das Gedächtnis hat
- Bitten Sie den Agenten, "meinen site brief abzurufen" (recall my site brief)
- Überprüfen Sie, ob das Gedächtnis in Ihren Einstellungen aktiviert ist

**Ich wott mit eme nöie site\_brief afange**
- Frog de Agent: "Clear my site brief and start fresh" (Lösch min site brief und fang neu a)
- Denn laufe Site Specification no eimal us

**De Agent macht Empfehlige, wo nöd zu mim site\_brief passe**
- Frog de Agent, dass er "review my site brief" (überprüef min site brief)
- Aktualisier din site\_brief, wenn er veraltet isch
- Gib meh Kontext in dine Aafrog
