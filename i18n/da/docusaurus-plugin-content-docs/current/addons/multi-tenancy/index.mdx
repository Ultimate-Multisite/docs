# Multi-tenancy

Fordel undersider på tværs af servere og databaser for ydeevne og sikkerhed

## Højdepunkter i version 1.2.0-udgivelsen

Multi-Tenancy 1.2.0 tilføjer det suveræne lejerfundament til produktionsisolerings-workflows:

- Stateless lejer-autologin med kortlivede, replay-beskyttede SSO-besøg.
- Database- og filsystemisolering pr. underside for suveræne lejere.
- Hostinger multi-tenancy-kapacitet til host-bevidst lejerprovisionering.
- Verifikationsporte for suveræn migrering til tjek af ældre data og asynkron push-behandling.
- Konfigurerbar binding af databasehost på samme maskine til Bedrock-, FrankenPHP- og localhost-installationer.
- Forbedringer af lejer-bootstrap, databaseskriver, skema-bootstrap og brugerprovisionering.
- Sikrere suveræn nedtagning, der rydder op i lejeroplysninger.

Se brugervejledningerne for [workflow til suveræn migrering](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy-isolering](/user-guide/configuration/multi-tenancy-isolation) og [operationer for suveræne lejere](/user-guide/administration/sovereign-tenant-operations).

## Krav

- WordPress 5.3 eller nyere
- PHP 7.4 eller nyere
- Multisite Ultimate plugin (aktiv)

## Installation

1. Upload addon-filerne til din `/wp-content/plugins/`-mappe
2. Aktivér plugin'et via menuen 'Plugins' i WordPress
3. Konfigurer addon-indstillingerne i netværksadministrationen

## Konfiguration

Addon'et understøtter forskellige konfigurationsmuligheder, der er tilgængelige via netværksadministrationsgrænsefladen.

### Konfiguration af suveræn lejer

For suveræne lejere skal du konfigurere lejerdatabasen, filsystemroden, binding af databasehost og host-kapacitet, før trafik routes til lejeren. Kør migreringsverifikation efter hver ændring, der påvirker databaseoplysninger, host-binding, routing eller brugerprovisionering.

Brug kun `localhost`, når lejerens runtime og database grants forventer adgang via lokal socket eller samme maskine. Brug `127.0.0.1` eller et service-hostnavn, når det er den grant-host, der er konfigureret af din udbyder.

### Hostinger-kapacitet

Hostinger multi-tenancy-kapaciteten fungerer med det delte Hostinger-integrationstoken. Bekræft, at domænet findes i hPanel, at DNS peger på Hostinger-kontoen, og at lejerdatabaseindstillingerne matcher hPanel før verifikation.

## Hooks og filtre

### Handlinger

- `ultimate-multisite-addon-template_init` - Udløses efter initialisering af addon'et
- `ultimate-multisite-addon-template_loaded` - Udløses efter, at alle addon-filer er indlæst

### Filtre

- `ultimate-multisite-addon-template_settings` - Rediger addon-indstillinger
- `ultimate-multisite-addon-template_enabled` - Tilsidesæt addon'ets aktiver/deaktiver-status
