# Färðingaskiljan (Site Specification Skill)

**Färðingaskiljan** er byggt nálgun til að fanga ábyrgðar þín á síðunni, viðskiptavini og merki fyrirtækisins. Þessi upplýsingar eru geymd í minnisnúmeri **site_brief**, sem hjálpar myndum til að hafa saman á milli sessjuna til að veita eins konar, samhengi-ánhaugandi hjálp.

## Hvað er Färðingaskiljan?

Färðingaskiljan er ferli að skrifa niður:

- **Lögging síðunnar**: Hvað síðan er og hvernig það finnst
- **Viðskiptavinnu** (Target audience): Færðir fólk yfir síðuna og hvað þeir þurfa
- **Merki fyrirtækisins**: Lækur, tónn og viskugildi
- **Viðskiptaforritan**: Hvað áttar sýnni lyktin fyrir síðuna þinni
- **Innhaldsbygging**: Hvernig síðan er organiserð

Þessi skiljanir verða þinn **site_brief**, sem er stöðugt minnisnúmer sem myndir nota til að skilja samhengi síðunnar þín.

## Hva er áhugaverð í Färðingaskiljan?

### Samræmi á sessjum

Á meðan þú hefur ekki site_brief, þurftu að útskýra löggingu síðunnar þína í hvert sem ný sessjun byrjar. Þegar þú hefur það, skilja myndir strax:

- Áttir síðunnar og viðskiptavini
- Lækur og tónn fyrirtækisins
- Innhaldsbygging
- Viðskiptaforritan

### Betri tillaga

Myndir nota site_brief þinn til að:

- Tala um eiginleika sem passar við löggingu síðunnar
- Tala um innhaldsbygging sem passar við forritið þitt
- Tala um hlutverk sem er samræmt merki fyrirtækisins
- Fyrirfara um tala um eiginleika sem ekki passar

### Hringandi uppsetning (Onboarding)

Ný myndir (eða myndir í ný sessjum) geta raskt að fari á framkvæmdarstigi með því að lesa site_brief þinn í stað þess að spyrja klárleggjandi spurningar.

## Samkvæði Färðingaskiljan

### Áhugaverð í uppsetningu Theme Builder-sins

Färðingaskiljan er sjálfskönnun á byrjun **Theme Builder onboarding flow**. Hjálparmyndin Setup Assistant spyrir spurningar og byggir þinn site_brief.

### Handhönnuð uppsetning

Þú getur byrja við Site Specification nánast hvaða tíma sem þér passar:

```
"Látum okkur skilja samkvæði mitt fyrir vísðu"
```

eða

```
"Hjálpa mér að skapa samkvæði fyrir vísðu"
```

## Ferli Site Specification

### Skref 1: Ábyrgðir á vísðunni (Site Purpose)

Agentur spyr:

```
Hva er fyrsta hlutinn á vísðunni þinni?
- E-commerce store
- Blogg eða innhaldsvefsítandi
- Portfolio eða sýningarsíti
- SaaS application
- Samfélag eða forum
- Ör: [skjaldi]
```

Þú getur valdið hluta eða skjaldið eigin ábyrgð.

### Skref 2: Stöðugæði (Target Audience)

```
Hva er fyrsta hlutinn á stöðugæðum þínum?
- Notendur / almenn almennleiki
- Fyrirtækja fagfólk
- Uppbyggjendur / tæknilegu notendur
- Studendur / menntamenn
- Ör: [skjaldi]

Hva eru aðalþróunir þeirra?
```

### Skref 3: Merki (Brand Identity)

```
Hva eru lögnum þínum merki?
- Primærlög: [valinn lög eða hex kóði]
- Sekundærlög: [valinn lög eða hex kóði]
- Akcentslög: [valinn, valfræð]

Hvernig byrjarð þínum merki?
- Profesjurl / fyrirtækjaleiki
- Kreativur / listræður
- Leikandi / afslappað
- Minimalist / nútíma
- Warm / venjulegur
```

### Skref 4: Stöðugæði viðskipta (Business Goals)

```
Hva er áttumfurinn fyrir vísðuna þinni?
- Generera leiðandi notendur (leads)
- Sälja vörur
- Byrja samfélag
- Deila þekkingu
- Skapa aukna orðstæði
- Ör: [skjaldi]

Hva er aðalmæli þitt?
- Aðlar
- Notenduráting (User engagement)
- Mikilvægi innhalds
- Breytingar í kaupum (Conversions)
- Ör
```

### Skref 5: Innhaldsbygging (Content Structure)

```
Hvernig er innhaldið byggt upp?
- Flatt (all innhald á sama orðstæði)
- Hierárískt (hlutblönd og undirhlutblönd)
- Tímskeiðsleg (blogg-stíl)
- Vörf-byggð (katalóg)
- Ör: [skjaldi]

Hva er innhaldstípar sem þú notar?
- Bloggfærslur
- Vörufarir
- Tilfelli (Case studies)
- Skjaldir
- Viðdeildar
- Ör
```

## Minni fyrir site_brief þinn

Eftir að fullfærð Site Specification er upplýsingum þínum lagt í **site_brief** í minni agentarinnar. Þetta er byggð skrá sem hefur eftirfarandi:

## Sjá sjáð og uppfærðu síðustu skýrslu um síðuna þína (site_brief)

### Sjá síðustu skýrslu um síðuna þína

Spyr við hjálparann:

```
"Visu mér síðustu skýrslu um síðuna mína"
```

eða

```
"Hvað veitir þú um síðuna mína?"
```

Hjálparinn mun sýna upplýsingar sem er geymd í skýrslunni þinni.

### Uppfærðu síðustu skýrslu um síðuna þína

Ef síðan þín þróast, geturðu uppfært hana:

```
"Uppfærðu síðustu skýrslu um síðuna mína: við reynjum nú að hafa B2B viðskiptavini sem markmið."
```

eða

```
"Frísúðu síðustu upplýsingum um síðuna mína"
```

Þetta mun beita aftur skilytgandi ferli (Site Specification skill) með því að nota þann sama upplýsinguna sem byrjunspunkt.

## Hvernig hjálparmenn nota síðustu skýrslu um síðuna

### Design ábyrgðarstefnur

Þegar þú spyr eftir designbreytingum, notar hjálparmenn síðustu skýrsluna um síðuna:

```
Þú: "Redesign my homepage" (Breytinga á hönnunshjónustu mína)
Hjálparmaður: "Á byggð þín á síðustu skýrslunni um síðuna, markmiðir þín eru konur í 25-45 fyrnar með varma og handiverkandi tóni. Ég mun skapa hönnun sem leggur áherslu á handiverkandi smjóar og sjálfbæði."
```

### Föfnustefnur

Hjálparmenn förenstefnur sem eru tengdar markmiðum þínum:

```
Þú: "What features should I add?" (Vilja ég bæta við hvaða eiginleika?)
Hjálparmaður: "Fyrir netkaupustöð sem er að falla á markaðinn fyrir viðskiptavina, mikið áherslu á truflun viðskiptavina, tölvupóstur-njósl og vöruráðstefnu."
```

### Uppbygging innihaldsins

Hjálparmenn förenst uppbyggingu innihaldsins eftir því sem þú hefur sett:

Hvernig ætla ég að organiserja blogginn mitt?
Svara: "Eftir að þú notar vörukatalog með blogg, mikið er hátt að ég sýna að nota vörukategoríur til að organiserja fræðingum og skapa 'stíllleiðbeiningar' sem sýna fleiri vörur saman."

## Bestu háttir að gera þetta

### Verðu sérstök
Í stað "almennni lesendur", skrifa um hvaða lesendur þín eru:

- ✓ "Konurnir í 25-45, sem er áhugaðar í haldverðum kleidan"
- ✗ "Allir"

### Uppfærðu reglulega
Þegar sítið þitt þróast, uppfærðu `site_brief`-inn þinn:

- Þegar þú skiptir á nýja lesendur
- Þegar þú aðskiltir nýjar vörulínur
- Þegar merkið þitt breytist
- Þegar viðskiptah tagdir þín breytist

### Notkun eins og sama hugtaka
Notu sama hugtaki í allar samtali:

- ✓ Segðu alltid "haldverðum smjóur" (ekki "miljanleg smjóur" og "grænar vörur")
- ✓ Neita á samháldinu þínu eins og sama hátt

### Skildu samhengi
Gjöf undirtekjur sem hjálpa hugbúnaðinu að skilja ákvörðunum þín:

- "Við sýjum umfalið fólki sem hefur framtíðarverð fyrir verð"
- "Lesendur okkar eru tækni-áhuga og bíða nútíma hlutverks"
- "Við erum nýr fyrirtæki sem byrjar, svo þurfum við lausnir sem eru kostnaðsvenjur"

## Tenging til Theme Builder onboarding

Skiljun `Site Specification`-skilygrar er í tengslum við **Theme Builder onboarding flow**. Þegar þú hefur fullfærð onboarding-ferlið, blir `site_brief`-inn þinn sjálf án þess að þú sért það.

Þú getur einnig haft Site Specification sem sjálfstæð ef þú vilt:

- Finna nákvæmari skilygrar eftir fyrsta uppsetningu
- Uppfærja `site_brief`-inn þinn þegar sítið þitt þróast
- Skapa nákvæma skilygrar áður en þú byrjar með Theme Builder

## Feilanaskynning

**Ég noti ekki `site_brief`-inn mitt**
- Staða að hugbúnaðinu hefur aðgang til minnis
- Spyr hugbúnaðinu "hugta minn site brief"
- Skildu að minnis sé áberandi í innstillingum þínum

**Ég vil byrja nýtt með nýtt site_brief**
- Spyr á hjálparfólki: "Láttu mér hreinsa site brief og byrja nýtt"
- Þar eftir reknu Site Specification aftur

**Hjálparfólkinn gerir tillögur sem ekki passar við site_brief mína**
- Spyr á hjálparfólki: "Færðu ferð til að skoða site brief mína"
- Uppfærðu site_brief þinn ef hann er orðinn orðinn ofinnréttur
- Gefðu viðbótandi samhengi í fyrirspurnunum þínum

## Næstu skref

Eftir að hafa teikna fram site specification þitt:

1. **Brugi Theme Builder**: Skapa sérstakt theme byggt á site_brief mínum
2. **Fínstilaðu hlutina**: Notaðu Design System Aesthetics skill fyrir nánast designarferli
3. **Planu innihaldið**: Spyr á hjálparfólk um tillögur um hlutfall og uppbyggingu innihaldsins
4. **Byggja eiginleika**: Færðu eftir eiginleikum sem eru í samræmi við fyrirætlanir þín
