# Multi-Network

የUltimate Multisite: Multinetwork addon የእርስዎን Ultimate Multisite platform በኃይለኛ multinetwork አቅም ያሰፋዋል። ይህ addon ደንበኞችዎ ግለሰብ ድረ-ገጾችን ብቻ ሳይሆን፣ ሙሉ የWordPress multisite networks መግዛት እና ማስተዳደር እንዲችሉ ያስችላል።

## ቁልፍ ባህሪያት (Key Features)

* **Network Provisioning (የኔትወርክ አቅዶ ማዘጋጀት)**: ለደንበኞችዎ ሙሉ የWordPress multisite networks በራስ-ሰር መፍጠር እና ማዋቀር።
* **Network Templates (የኔትወርክ አብነቶች)**: ደንበኞች ከመመዝገባቸው በፊት የተዘጋጀ የኔትወርክ አብነት እንዲመለከቱ፣ እንዲገመግሙ እና እንዲመርጡ ያስችላል።
* **Network Management (የኔትወርክ አስተዳደር)**: ለmultinetwork አካባቢዎች ሙሉ የአስተዳደር አቅም።
* **Isolated Networks (የተነጠሉ ኔትወርኮች)**: እያንዳንዱ የደንበኛ ኔትወርክ በራሱ የዳታቤዝ ሰንጠረዦች እና ውቅረት ላይ በነጻነት ይሰራል።
* **Cross-Network SSO (በኔትወርክ መካከል የSSO)**: ደንበኞችዎ በmagic-link authentication አማካኝነት ወደ ንዑስ ኔትወርኮቻቸው ያለችግር እንዲደርሱ ያደርጋል።
* **Scalable Architecture (የሚሰፋ አርክቴክቸር)**: የተመቻቸ የሀብት ምደባን በመጠቀም በርካታ ኔትወርኮችን በአቅም መያዝ።
* **Integration (ማዋሃድ)**: ከUltimate Multisite የደንበኝነት ምዝገባ እና የሂሳብ ክፍያ ሥርዓቶች ጋር ያለችግር ይዋሃዳል።

## የአጠቃቀም ሁኔታዎች (Use Cases)

* **White Label Hosting**: ደንበኞችዎ የራሳቸውን ብራንድ ያላቸው multisite networks እንዲይዙ ያግዛል።
* **Agency Solutions**: ኤጀንሲዎች በርካታ የደንበኛ ኔትወርኮችን ከአንድ መድረክ እንዲያስተዳድሩ ያስችላል።
* **Enterprise Deployments**: ውስብስብ የብዙ-ኔትወርክ መስፈርቶች ባላቸው ትልልቅ ድርጅቶች ድጋፍ።
* **Education Platforms**: ለተለያዩ ትምህርት ቤቶች ወይም ክፍሎች የተለየ ኔትወርኮችን መፍጠር።

## መስፈርቶች (Requirements)

* Ultimate Multisite 2.0.0 ወይም ከዚያ በላይ
* WordPress Multisite installation
* PHP 7.4 ወይም ከዚያ በላይ
* MySQL 5.6 ወይም ከዚያ በላይ

## መጫን (Installation)

1. የፕልጋይኑ ፋይሎችን ወደ `/wp-content/plugins/ultimate-multisite-multinetwork` ዳይሬክቶሪ ይጫኑ።
2. በWordPress ውስጥ ባለው 'Plugins' ስክሪን አማካኝነት ፕልጋይኑን ያንቁ።
3. የmultinetwork ቅንብሮችን በUltimate Multisite → Settings አማካኝነት ያዋቅሩ።

## Network Templates (የኔትወርክ አብነቶች)

Version 1.3.0 ለ አዳዲስ ኔትወርኮች የሚገዙ ደንበኞች አብነትን መፈለግ እና መምረጥ የሚያስችል ፍሰት (flow) አክሏል። እያንዳንዱ አዲስ የደንበኛ ኔትወርክ ከተዘጋጀ መዋቅር፣ ዲዛይን፣ የይዘት ቤተመጽሐፍት ወይም ለተወሰነ ዘርፍ የተዘጋጀ አቀማመጥ እንዲጀምር ሲፈልጉ የኔትወርክ አብነቶችን ይጠቀሙ።

### Template previewer (የአብነት ቅድመ-እይታ)

የኔትወርክ አብነት ቅድመ-እይታ ደንበኞች ከመግዛትዎ በፊት ያሉትን አብነቶች እንዲመለከቱ ያስችላል። እያንዳንዱ አብነት በቀጥታ የሚሰራ ቅድመ-እይታ ፓነል ውስጥ ሊከፈት ስለሚችል፣ ደንበኞች ኔትወርክ ከመግዛታቸው በፊት አቀማመጥዎችን እና ይዘቶችን ሊያነፃፅሩ ይችላሉ።

ደንበኞች እንዲረዱ የሚረዱበትን ሁኔታ ለመርዳት ቅድመ-እይታውን ይጠቀሙ:

* እያንዳንዱ የሚገኝ የኔትወርክ አብነት መልክ እና መዋቅር መገምገም
* ያለ የመግዣ ፍሰት ሳይወጡ በርካታ አብነቶችን ማነፃፀር
* ከመመዝገባቸው በፊት ከየእነርሱ የአጠቃቀም ሁኔታ ጋር የሚስማማውን አብነት መምረጥ

የአብነት ስሞችን እና መግለጫዎችን ለደንበኞች ተስማሚ ያድርጉ። ቅድመ-እይታው ደንበኞች የትኛው የአጠቃቀም ሁኔታ እንደሆነ መረዳት ሲችሉ በጣም ጠቃሚ ነው፣ ለምሳሌ የኤጀንሲ ኔትወርኮች፣ የትምህርት ቤት ኔትወርኮች፣ የፍራንቺዝ ኔትወርኮች ወይም የwhite-label hosting ጥቅል ስብስቦች።

### Template picker at checkout (በመመዝገቢያ ክፍያ ላይ የአብነት መምረጥ)

የአብነት መምረጥ (Template picker) እንደ መስክ በመመዝገቢያ ክፍያ ፍሰት (checkout flow) ላይ ሊጨመር ይችላል። ሲዘጋጅ፣ ደንበኞች በሚከፍሉበት ጊዜ የፈለጉትን የኔትወርክ አብነት ይመርጣሉ፣ እና የተመረጠው አብነት አዲሱ ኔትወርክ ሲዘጋጅ ጥቅም ላይ ይውላል።

በመመዝገቢያ ክፍያ ፍሰት ውስጥ መምረጥን ለመጠቀም:

1. እንደ አብነቶች ሊያቀርቡልዎ የሚፈልጓቸውን ኔትወርኮች ያዘጋጁ።
2. ተዛማጅ የቅዳሜና የሰኞ መመዝገቢያ ወይም የመመዝገቢያ ቅጽ ላይ Multinetwork template selection field ን ያንቁ።
3. የትኛው አብነቶች ለዚያ የመመዝገቢያ ክፍያ ፍሰት መገኘት አለባቸው ይምረጡ።
4. የመመዝገቢያ ክፍያ ፍሰቱን ያትሙ እና የተመረጠው አብነት አዲሱ ኔትወርክ ላይ መተግበሩን ለማረጋገጥ የመመዝገቢያ ሙከራ ያድርጉ።

የመመዝገቢያ ክፍያ ፍሰት አንድ የኔትወርክ ምርት ብቻ የሚያቀርብ ከሆነ፣ መምረጥ አሁንም እንደ መመሪያ የጀማሪ ደረጃ (guided onboarding step) ጠቃሚ ሊሆን ይችላል። ለብዙ ዕቅዶች (multi-plan funnels)፣ ከየተመረጠው ምርት ወይም ዕቅድ ጋር የሚጣጣሙ አብነቶችን ብቻ ያሳዩ።

## Cross-Network Magic Link SSO (በኔትወርክ መካከል የMagic Link SSO)

Cross-network magic link SSO ደንበኞችዎ በዋና የሂሳብ አካባቢቸው እና በንዑስ ኔትወርኮቻቸው መካከል ያለማቋረጥ መዘዋወር እንዲችሉ ያስችላል። ደንበኛው የcross-network access linkን ሲከተል፣ Ultimate Multisite ጊዜ የተገደበ magic link ይፈጥራል እና ደንበኛውን ወደ መድረሻው ኔትወርክ ያረጋግጣል።

ይህ ደንበኞች ከአንድ በላይ ኔትወርክ የሚያስተዳድሩ ወይም አዲስ የተገለበጡ የtemplate ኔትወርኮችን ከገዙ በኋላ ወዲያውኑ መድረስ ያለባቸው ከሆነ ልምዳቸውን ያሻሽላል።

የአስተዳሚ የቼክሊስት ዝርዝር:

* Multinetwork addon በWordPress multisite installation ላይ ንቁ መሆኑን ያረጋግጡ።
* ደንበኞች cross-network links ከመሰጣቸው በፊት ትክክለኛ የኔትወርክ መዳረሻ እንዳላቸው ያረጋግጡ።
* magic links ወደሚፈለገው የመድረሻ ኔትወርክ እንዲመሩ የኔትወርክ ዶሜን እና የተመዘገቡ ዶሜኖች በትክክል ተዋቅረዋል መሆኑን ይጠብቁ።
* የኔትወርክ፣ የዶሜን ወይም የመመዝገቢያ ክፍያ ቅንብሮችን ከቀየሩ በኋላ የደንበኛ ዳሽቦርድ እና ማንኛውም የpost-checkout access buttons መፈተሽ።

Magic links እንደ የመግቢያ ሊንኮች (login links) መታየት አለባቸው። ለቀላል የደንበኛ መዳረሻ የተነደፉ በመሆናቸው፣ ከታሰበው ክፍለ ጊዜ ውጭ መላክ ወይም እንደገና መጠቀም የለባቸውም።

## በተደጋጋሚ የሚጠየቁ ጥያቄዎች (Frequently Asked Questions)

### Multinetwork ምንድን ነው?

Multinetwork አንድ የWordPress installation ላይ በርካታ ተነጥል የሆኑ multisite networks እንዲሰሩ የሚያስችል የWordPress ባህሪ ነው።

### ከወትሮው multisite እንዴት ይለያል?

ወትሮው multisite በአንድ ኔትወርክ ውስጥ ብዙ ድረ-ገጾችን (sites) ያስችላል። Multinetwork ደግሞ በሙሉ በአንድ የWordPress installation ውስጥ፣ እያንዳንዳቸው የራሳቸው ድረ-ገጾች ባሉ ብዙ ኔትወርኮችን (networks) ያስችላል።

### ልዩ የሰርቨር ቅንብር ይፈልጋል?
