# Admin Page Creator {#admin-page-creator}

## Content {#content}
Admin Page Creator በWordPress admin dashboard ውስጥ አዳዲስ የአስተዳደር ገጾችን (admin pages) እና ንዑስ ገጾችን (sub-pages) በቀላሉ እንዲጨምሩ ያስችልዎታል። አዲስ ተጠቃሚዎችን ለመቀበል የሚያገለግል የእንኳን ደህና መጣችሁ ገጽ (welcome page) ወይም የማስተማሪያ ገጽ (tutorial page) ለመፍጠር ሊውል ይችላል። ገጾቹን ያለ ችግር ዲዛይን ለማድረግ Beaver Builder, Elementor, Brizy, እና Oxygen የመሳሰሉ page builders መጠቀም ይችላሉ።

እንዲሁም፣ የራሳችሁን HTML, CSS, እና JavaScript ማስገቢያ (support) እንዲሁም በፕላን እና በተጠቃሚ ሚና (user role) ላይ የተመሰረተ የመዳረሻ ቁጥጥር (access control) ይሰጣል።

## Features {#features}
* የራሳችሁን HTML እና CSS መደገፍ
* JavaScript ድጋፍ
* PHP Content Mode
* በተጠቃሚ ሚና ላይ የተመሰረተ ፈቃድ (Permission)
* page builders መደገፍ

## Installation {#installation}

1. **ultimate-multisite-admin-page-creator** የሚለውን ወደ **/wp-content/plugins/** ዳይሬክቶሪ ይጫኑ።
2. በWordPress Network Admin ውስጥ ባለው **Plugins** ምናሌ ውስጥ ፕልጊኑን ያንቁ (Activate)።

## Usage Guide {#usage-guide}

WordPress በሳይቶች የፊት ክፍል (front end) ላይ ይዘት መጨመር በጣም ቀላል ቢሆንም፣ በአስተዳደር ክፍል (admin side) ገጾችን እና ንዑስ ገጾችን መፍጠር መሰረታዊ የኮዲንግ ክህሎት ይጠይቃል። Ultimate Multisite: Admin Page Creator የዚህን የፊት ክፍል ቀላልነት ወደ የኔትወርክ ሳይቶችዎ የኋላ ክፍል (back end) ለማምጣት ያለመ ነው።

## Why use Admin Page Creator? {#why-use-admin-page-creator}

ምናልባት ለደንበኞችዎ የአስተዳደር ፓነል ላይ የማስተማሪያ ገጽ (tutorials) ሊሰጡበት ይፈልጉ ይሆናል፣ ወይም የጥያቄና መልስ (FAQs) እና የእውቂያ widget ያለው የራሳችሁን ገጽ መጨመር ሊፈልጉ ይችላሉ። በAdmin Page Creator አማካኝነት እድሎቹ ማለቂያ የሌላቸው ሲሆኑ፣ ይህም በዛሬው ትምህርት የምንመረምረው ነው።

## Creating your first Custom Admin Page {#creating-your-first-custom-admin-page}

Ultimate Multisite: Admin Page Creator ከጫኑት እና ካንቀሳቅሰዋታችሁ በኋላ፣ ወደ network admin ምናሌ በመሄድ Admin Pages ላይ ይሂዱ።

![Custom Admin Pages list on Network Admin](/img/admin/admin-pages-list.png)

_በNetwork Admin ላይ የራሳችሁን የአስተዳደር ገጾች ዝርዝር_

እስካሁን የፈጠራችሁን ሁሉንም የአስተዳደር ገጾች ዝርዝር ታያላችሁ (ይህ የመጀመሪያ ጉብኝታችሁ ስለሆነ ዝርዝሩ ባዶ ይሆናል።) ወደ **Add new Admin Page** በመሄድ ይጫኑ። ወደ Admin Page editor ይተላለፋሉ።

## The elements of a Custom Admin Page {#the-elements-of-a-custom-admin-page}

በAdmin Page editor ውስጥ፣ ገጽዎን ለማበጀት የተለያዩ አማራጮች ይኖራሉ።

ይህ የአስተዳደር ምናሌ ስም (admin menu label) ሳይሆን፣ በገጹ 顶部 ላይ የሚታየው ርዕስ ነው።

## Menu Options {#menu-options}

በጎን በኩል፣ በGeneral Options meta-box ላይ፣ የምናሌውን አይነት (menu type)፣ ስም (label) እና ብዙ ተጨማሪ ነገሮችን ማዋቀር ይችላሉ።

### Menu Types {#menu-types}

የራሳችሁ አዲስ የአስተዳደር ገጽ ሁለት የተለያዩ የምናሌ አይነቶችን ሊኖረው ይችላል፡ የላቀ ደረጃ ምናሌ (top-level menu) እና የንዑስ ምናሌ (submenu)።

የላቀ ደረጃ ምናሌ ገጽ፣ ስሙ እንደሚያመለክተው፣ አዲሱን የፈጠራችሁን ገጽ ከነባሪው የWordPress ገጾች ጋር በአስተዳደር ምናሌ ውስጥ ይጨምረዋል።

የላቀ ደረጃ አማራጩን ከመረጣችሁ፣ የምናሌውን ቅደም ተከተል (menu order) እና የምናሌ አዶ (menu icon) መምረጥ ትችላላችሁ።

ይህንን ገጽ የንዑስ ምናሌ ገጽ (sub-menu page) ማድረግ ከፈለጋችሁ፣ ይህንን ገጽ የሚያያይዝ የወላጅ ገጽ (parent page) መምረጥ ትችላላችሁ። የራሳችሁን ገጽ ወደ መደበኛ የWordPress ገጾች ወይም ወደ የላቀ ደረጃ የራሳችሁን የአስተዳደር ገጾች መጨመር ትችላላችሁ።

## Advanced Options {#advanced-options}

በየራሳችሁ የአስተዳደር ገጾች ላይ ሁሉንም ዓይነት ይዘት መጨመር መቻል አለባችሁ። ይህም የCSS ህጎችን እና ፋይሎችን፣ JavaScriptን፣ እና የውጭ ቤተመጽሐፍቶችን (external libraries) ያካትታል። ይህንን በAdvanced Options Tab በመጠቀም ማድረግ ይችላሉ።

### Custom CSS {#custom-css}

በCSS ታብ ላይ፣ የራሳችሁን የCSS ኮድ ማስገቢያ (add)፣ እንዲሁም ከየርቀት ድረ-ገጾች (እንደ CDNs, Google Fonts ወዘተ) የCSS ፋይሎችን ማስመጣት ይችላሉ።

## Permissions {#permissions}

በAdvanced Options ውስጥም፣ የትኞቹ ንዑስ ሳይቶች (sub-sites) ይህንን ገጽ መድረስ እንደሚችሉ መምረጥ ይችላሉ። ለምሳሌ፣ ለተለያዩ የፕላን ደረጃዎች የተለያየ ልዩ ይዘት ለማቅረብ እያቀዱ ከሆነ ይህ ጠቃሚ ነው።

<!-- Screenshot unavailable: Admin Page Creator permissions settings for plan and role-based access control -->

_ይህ ገጽ የት እንደሚታይ ሙሉ ቁጥጥር አለዎት_

**አስፈላጊ** : ገጹ እንዲታይ ሁለቱ ሁኔታዎችም መሟላት አለባቸው። ይህ ማለት ተጠቃሚው የተሰጠውን ፕላን ደንበኛ ቢሆንም፣ ከተመረጡት ሚናዎች አንዱ ባይኖረው (ወይም በተቃራኒው) ገጹ በፓነሉ ላይ አይታይም።

## Active {#active}

የራሳችሁን የአስተዳደር ገጽ ሳይሰርዙት ወይም ፈቃዶቹን ሳይረብሹ ለማጥፋት (disable) ከፈለጋችሁ፣ Active የሚለውን አማራጭ ብቻ ባዶ ያድርጉት (uncheck)።

የላቀ ደረጃ ገጽን ማጥፋት (Deactivating) በዚያ ወላጅ ገጽ ላይ የተያያዙትን ሁሉንም ንዑስ ገጾች በራስ-ሰር ያጥፋዋል።
