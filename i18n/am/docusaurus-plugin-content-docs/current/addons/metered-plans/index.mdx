# በUsage ላይ የተመሰረቱ ዕቅዶች (Metered Plans) {#metered-plans}

የUltimate Multisite: Metered Plans ተጨማሪ ሞጁል (addon) ደንበኞችን ክፍያ የሚጠይቁበትን መንገድ በመቀየር፣ ተለዋዋጭ የUsage ክፍያ አቅሞችን በማስተዋወቅ ይለውጣል። ተጠቃሚዎችን በሚበልጥበት ጊዜ የሚያቆሙ ጥብቅ ገደቦች ከመጠቀም ይልቅ፣ ተጨማሪ አጠቃቀምን በራስ-ሰር ለደንበኞች ክፍያ የሚጠይቁ ክፍያዎችን (overage charges) ያስችላል።

## ዋና ዋና ባህሪያት (Key Features) {#key-features}

* **በUsage ላይ የተመሰረተ ክፍያ (Metered Billing)**: ደንበኞችን ክፍያ በቋሚ ደረጃዎች (fixed tiers) ሳይሆን በእውነተኛ አጠቃቀማቸው ላይ ተመስርተው እንዲከፍሉ ያደርጋል።
* **የከመጠን በላይ አጠቃቀም አስተዳደር (Overage Management)**: ከዕቅዱ ገደብ በላይ ለሚደረግ አጠቃቀም በራስ-ሰር ክፍያ ይጠይቃል።
* **የUsage መከታተል (Usage Tracking)**: የደንበኞችን የሀብት አጠቃቀም በቅጽበት (real-time) ይከታተላል።
* **ተለዋዋጭ ዋጋ አወጣጥ (Flexible Pricing)**: ለተለያዩ ምንጮች (storage, bandwidth, sites ወዘተ) የራሳቸውን ዋጋ እንዲያዘጋጁ ያስችላል።
* **የAI Token ክፍያ (AI Token Billing)**: በደንበኞች ንዑስ ድረ-ገጾች (subsites) ላይ ለሚደረገው AI አጠቃቀም የግብዓት (input) እና የውፅዓት (output) ቶከኖችን ይከታተላል እና ክፍያ ይጠይቃል።
* **በራስ-ሰር የክፍያ መጠየቂያ (Automatic Invoicing)**: የአጠቃቀም ገደቦች ሲበልጡ የክፍያ መጠየቂያዎችን በራስ-ሰር ይፈጥራል።
* **የUsage ሪፖርቶች (Usage Reports)**: ለደንበኞች እና ለአስተዳዳሪዎች ዝርዝር የUsage ትንታኔ እና ሪፖርት ያቀርባል።

## ለማን ተስማሚ ነው (Perfect For) {#perfect-for}

* **የሆስቲንግ አቅራቢዎች (Hosting Providers)**: ከየደንበኞች ፍላጎት ጋር የሚያድጉ ተለዋዋጭ ዕቅዶችን ማቅረብ።
* **SaaS መድረኮች (SaaS Platforms)**: ክፍያ በደረሰ የሀብት አጠቃቀም ላይ መመስረት።
* **የኤጀንሲ ኔትወርኮች (Agency Networks)**: ለደንበኛ ፕሮጀክቶች ተለዋዋጭ የዋጋ ሞዴሎች።
* **የሪሴለር ሆስቲንግ (Reseller Hosting)**: አጋሮች በUsage ላይ የተመሰረተ ዋጋ እንዲያቀርቡ ማስቻል።

## እንዴት ይሰራል (How It Works) {#how-it-works}

1. የUsage ምንጮችን (disk space, bandwidth, number of sites ወዘተ) ያዘጋጁ።
2. የመነሻ ዕቅድ ገደቦችን እና የከመጠን በላይ ክፍያ መጠን (overage rates) ያዘጋጃሉ።
3. ስርዓቱ በአጠቃላይ በኔትዎርክዎ ውስጥ ያለውን Usage በራስ-ሰር ይከታተላል።
4. ደንበኞች ገደብ ሲበልጡ፣ የከመጠን በላይ ክፍያዎች ይሰላሉ።
5. የክፍያ መጠየቂያዎች በክፍያ ዑደት (billing cycle) መሰረት በራስ-ሰር ይፈጠራሉ።
6. ደንበኞች ያለውን Usage እና የሚመጣውን ክፍያ በቅጽበት ማየት ይችላሉ።

## የAI Token ክፍያ (AI Token Billing) {#ai-token-billing}

በVersion 1.1.0፣ በWordPress AI Client SDK አማካኝነት AI ባህሪያትን የሚያቀርቡ ኔትወርኮች ላይ AI token ክፍያን ይጨምራል። Metered Plans የተጠናቀቁ የAI ትውልድ ጥያቄዎችን (generation requests) ይከታተላል፣ ለወቅቱ ንዑስ ድረ-ገጽ የግብዓት እና የውፅዓት ቶከን ብዛት ይመዘግባል፣ እና እነዚህን ቶከኖች ለደንበኛው ወቅታዊ የክፍያ ጊዜ የሚከፈሉ AI ወጪዎችగా ይለውጣል።

የለምን AI መሳሪያዎችን በደንበኞች ድረ-ገጾች ላይ ማቅረብ ሲፈልጉ እና ያልተጠበቀ የሞዴል ወጪዎችን መሸፈን ባይፈልጉ የAI token ክፍያን መጠቀም ይችላሉ። እያንዳንዱ ጥያቄ ለንዑስ ድረ-ገጹ፣ ለደንበኛው፣ ለአባልነት፣ ለአቅራቢው እና ለሞዴሉ ተመድቦ ስለሚመጣ፣ አስተዳዳሪዎች Usageን እንዲገመግሙ ሲችሉ ደንበኞች የAI አጠቃቀማቸው በሂሳባቸው ላይ ምን ተጽዕኖ እንደሚያሳድር ማየት ይችላሉ።

### በየቶከኑ መጠን ማዘጋጀት (Configure Per-Token Rates) {#configure-per-token-rates}

የAI ዋጋዎች ለእያንዳንዱ ሞዴል እንደ የተለያየ የግብዓት እና የውፅዓት መጠን ይዘጋጃሉ። ዋጋዎች በ1 ሚሊዮን ቶከኖች (1 million tokens) ላይ ተመዝግበዋል፣ ይህም ከብዙ የAI አቅራቢዎች የዋጋ ገጾች ጋር ይዛመዳል።

1. በኔትወርክ አድሚን ውስጥ፣ **Multisite Ultimate → Settings → Metered Plans** ይክፈቱ።
2. AI Usage ክፍያ የሚጠይቀው ምርት ወይም አባልነት ከሆነ፣ AI spend metering ን ያንቁ።
3. በኮንክተርዎ (connector) የሚመለስባቸውን ሞዴል ID በመጠቀም የሞዴል መጠን መረጃዎችን ይጨምሩ፣ ለምሳሌ `gpt-4o`, `gpt-4o-mini`, ወይም ሌላ OpenAI-compatible model identifier።
4. በየ1 ሚሊዮን ቶከኖች ላይ ለየየው **የግብዓት መጠን (Input Rate)** እና **የውፅዓት መጠን (Output Rate)** እሴቶችን በየየው የገንዘብ ምንዛሪዎ ያስገቡ።
5. ለዕቅዱ የደንበኛው የተካተተ AI spend allowance እና የከመጠን በላይ አጠቃቀም ባህሪይ (overage behaviour) ያዘጋጁ።

ትክክለኛ የሞዴል ID ካልተዘጋጀ፣ Metered Plans በሞዴል ቅድመ-ጽሑፍ (model prefix) ሊመሳሰል ይችላል። ለምሳሌ፣ ለ`claude-sonnet-4` የተዘጋጀ ዋጋ በዚያ ቅድመ-ጽሑፍ በሚጀምሩ የtimestamped የሞዴል ID ላይ ሊተገበር ይችላል። ገንቢዎች የየመጨረሻውን የዋጋ ሰንጠረዥ በ[`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates) filter በመጠቀም ማስተካከያ ማድረግ ይችላሉ።

### የሚደገፉ የAI ኮንክተሮች (Supported AI Connectors) {#supported-ai-connectors}

የAI token ክፍያ የሚሰራው የWordPress AI Client SDK ትውልድ ክስተቶችን (generation events) የሚያልፍ እና የቶከን አጠቃቀምን በመልስ ሜታዳታ (response metadata) የሚያካትቱ ኮንክተሮችን ነው። ይህ የሚከተሉትን ያካትታል፡

* [Ultimate AI Connector for Compatible Endpoints](../ultimate-ai-connector-compatible-endpoints/) ለOpenAI-compatible APIs፣ OpenAI፣ Mistral፣ Ollama-compatible gateways እና ተኳሃኝ የቶከን አጠቃቀም ሜታዳታ የሚያሳዩ ሌሎች አቅራቢዎችን ጨምሮ።
* [Ultimate AI Connector for WebLLM](../ultimate-ai-connector-webllm/) ለብራውዘር ተወላጅ WebLLM አጠቃቀም፣ ጥያቄዎች በAI Client SDK event flow ውስጥ ሲላኩ።
* በ[Gratis AI Agent](../gratis-ai-agent/) ጥቅም ላይ የሚውሉ የAI አቅራቢ ኮንክተሮች፣ በተመረጠው አቅራቢ የprompt እና የcompletion ቶከን ብዛትን በSDK በኩል ሲዘግም።

የተራቀቁ ኮንክተሮች (Custom connectors) በሞዴል፣ አቅራቢ እና የቶከን አጠቃቀም ሜታዳታ በመጠቀም መደበኛ የ`wp_ai_client_before_generate_result` እና `wp_ai_client_after_generate_result` ክስተቶችን በመላክ ተሳታፊ ሊሆኑ ይችላሉ። የኮንክተር ቅንብሮች ከዋናው ድረ-ገጽ ወደ ንዑስ ድረ-ገጾች ስለሚተገበሩ፣ የደንበኞች ድረ-ገጾች በኔትወርክ-የተፈቀዱ አቅራቢዎች እና ገደቦች እንዲጠቀሙ ያደርጋል።

### የደንበኛ አካውንት Usage (Customer Account Usage) {#customer-account-usage}

ደንበኞች የAI Usageን ከሌሎች የUsage ምንጮቻቸው ጋር በየአካባቢያቸው ያያሉ። የAI spend መስመር (row) የሚታየው በየክፍያ ጊዜው የተሰበሰበውን የprompt እና የcompletion ቶከን ብዛት፣ የተዘጋጁ የሞዴል ዋጋዎች እና በዕቅዱ ውስጥ ያለውን ማንኛውንም የተካተተ allowance ነው።

ደንበኛው የክፍያ ዘዴ ካስቀመጠ፣ ከየተካተተው allowance በላይ የሆነ Usage እንደ ክፍያ የሚጠይቅ ከመጠን በላይ አጠቃቀም (billable overage) ይቆጠራል። ዕቅዱ ጥብቅ መቋረጥ (hard cutoff) እንዲሆን ከተዘጋጀ እና ደንበኛው የክፍያ ዘዴ ባይኖረው፣ allowance እስኪደርስ እና ደንበኛው ክፍያ ዝርዝሮችን እንዲጨምር ሲጠየቅ የAI ጥያቄዎች ሊቋረጡ ይችላሉ።

* የዲስክ ማከማቻ (Disk storage)
* ባንድዊድዝ/የዳታ ማስተላለፍ (Bandwidth/Data transfer)
* የድረ-ገጽ ብዛት (Number of sites)
* የተጠቃሚ ብዛት (Number of users)
* የAI ቶከን Usage እና AI spend
* የተራቀቁ የምንጭ አይነቶች (Custom resource types)

## መስፈርቶች (Requirements) {#requirements}

* Multisite Ultimate 2.0.0 ወይም ከዚያ በላይ
* WordPress Multisite installation
* PHP 7.4 ወይም ከዚያ በላይ

## መጫን (Installation) {#installation}

1. የፕልጋይኑ ፋይሎችን ወደ `/wp-content/plugins/ultimate-multisite-metered-plans` ይጫኑ።
2. በWordPress ውስጥ ባለው 'Plugins' ስክሪን አማካኝነት ፕልጋይኑን ያንቁ (Activate)።
3. ወደ Multisite Ultimate → Settings → Metered Plans ይሂዱ።
4. የUsage ምንጮច እና ዋጋ አወጣጥዎን ያዘጋጁ።
5. በምርት አርታኢ (product editor) ውስጥ ለምርቶች የUsage ክፍያን ያንቁ።

## በተደጋጋሚ የሚጠየቁ ጥያቄዎች (Frequently Asked Questions) {#frequently-asked-questions}

## ደንበኛው ገደቡን ሲበልጥ ምን ይሆናል? {#what-happens-when-a-customer-exceeds-their-limit}

መዳረግ (blocking) ከመስጠት ይልቅ፣ ስርዓቱ ከመጠን በላይ አጠቃቀምን ይከታተላል እና በርስዎ ባዘጋጁት መጠን መሰረት ወደ ቀጣዩ የክፍያ መጠየቂያ ይጨምረዋል።

## ለተለያዩ ምርቶች የተለያዩ የከመጠን በላይ ክፍያ መጠኖችን ማዘጋጀት እችላለሁ? {#can-i-set-different-overage-rates-for-different-products}

አዎ፣ እያንዳንዱ ምርት ልዩ የUsage ምንጮች እና የከመጠን በላይ ክፍያ ዋጋ ሊኖረው ይችላል።

## ደንበኞች ስለ Usage እንዴት ይነገራሉ? {#how-are-customers-notified-about-usage}

ደንበኞች የአሁኑ Usage ስራዎቻቸው (dashboard) ላይ ማየት እና ገደብ ሲቃረብ ወይም ሲበልጥ አማራጭ ማሳወቂያዎችን መቀበል ይችላሉ።

## የAI token Usage ዋጋ እንዴት ነው የሚሰላው? {#how-is-ai-token-usage-priced}

የAI token Usage ዋጋ የሚሰላው ከሞዴሉ በተዘጋጁ የግብዓት እና የውፅዓት መጠን በየ1 ሚሊዮን ቶከኖች ነው። Metered Plans እያንዳንዱን ጥያቄ የprompt እና የcompletion ቶከን ብዛትን በእነዚህ መጠን ይባዛል፣ ውጤቱን ወደ ደንበኛው ወቅታዊ የክፍያ ጊዜ AI spend ይጨምራል፣ እና የዕቅዱን allowance ወይም የከመጠን በላይ አጠቃቀም ህጎችን ይተገብራል።

## ጥብቅ ገደቦችን (hard limits) መጠቀም እችላለሁ? {#can-i-still-use-hard-limits}

አዎ፣ የUsage ክፍያ (metered billing) ለእያንዳንዱ ምርት አማራጭ ነው። ባህላዊ የጥብቅ ገደብ ምርቶችን ከUsage ክፍያ ምርቶች ጋር መቀላቀል ይችላሉ።
