# Domain Seller

**Multisite Ultimate: Domain Seller** የWordPress Multisite ኔትወርክዎን ወደ ኃይለኛ የዶሜን መሸጫ መድረክ ይለውጠዋል። ደንበኞች በመመዝገቢያ ጊዜ ወይም ከዳሽቦርዳቸው በቀጥታ የራሳቸውን ዶሜን እንዲመዘግቡ ያስችላል። ይህ ሲሆን የዶሜን ተገኝነትን በቅጽበት ይፈትናል እንዲሁም ግልጽ የዋጋ መረጃ ይሰጣል።

### ለምን Domain Seller?

የዶሜን ምዝገባ ለሁሉም WaaS (Website as a Service) መድረክ በጣም የሚፈለግ ተጨማሪ አገልግሎት ነው። ደንበኞችን ወደ ውጭ ተቋራጭ (third-party) ሪጅስትራ ከመላክ ይልቅ፣ ዶሜኖችን በቀጥታ በማቅረብ፣ በኔትወርክዎ ውስጥ ብዙ ገቢ እንዲቆይ እና ደንበኞችም እንከን የለሽ ተሞክሮ እንዲያገኙ ማድረግ ይችላሉ።

### ቁልፍ ባህሪያት

#### በርካታ የዶሜን ሪጅስትራ ድጋፍ
* **OpenSRS** — ሙሉ ምዝገባ፣ እድሳት እና አስተዳደር
* **Namecheap** — ከWhoisGuard እና sandbox testing ጋር ሙሉ ውህደት
* **የሚሰፋ አርክቴክቸር (Extensible Architecture)** — በIntegration Registry pattern ላይ የተገነባ በመሆኑ፣ ተጨማሪ አቅራቢዎችን መጨመር ቀላል ያደርገዋል።
* **በውስጥ የተካተተ የ연결ነት ምርመራ (Built-in Connection Testing)** — አገልግሎት ከመስራቱ በፊት የአቅራቢን መለያዎች እና ግንኙነት መፈተሽ።

#### መመሪያ ሰጪ የቅንብር አስማሚ (Guided Setup Wizard)
* **በአንድ ጠቅታ መጀመር (One-click onboarding)** — የአቅራቢ ቅንብር፣ TLD ማስመጣት እና የዶሜን ምርት መፍጠርን ደረጃ በደረጃ ያስተምራዎታል።
* **ራስ-ሰር TLD መመሳሰል (Automatic TLD sync)** — ሁሉንም የሚገኙ TLDs እና የጅምላ ዋጋዎችን (wholesale pricing) ከየተገናኙ አቅራቢዎችዎ ያስገባል።
* **የመነሻ ምርት መፍጠር (Default product creation)** — ተገቢ የዋጋ ጭማሪ (markup) መነሻዎች ባሉበት፣ ለመጠቀም ዝግጁ የሆነ የዶሜን ምርት ይፈጥራል።

#### ተጣጣፊ የዶሜን ዋጋ አወጣጥ
* **ሶስት የዋጋ ጭማሪ ሁነታዎች (Three markup modes)** — በመቶኛ ጭማሪ፣ የተወሰነ ጭማሪ፣ ወይም ለእያንዳንዱ TLD የተወሰነ ዋጋ።
* **በ-TLD የተለዩ ምርቶች** — ለተለያዩ የTLD ቡድኖች (.com/.net, .io, የሀገር ኮዶች ወዘተ) የተለየ የዶሜን ምርት ይፍጠሩ።
* **የመጀመሪያ ቅናሽ ዋጋ (Introductory pricing)** — አዲስ ምዝገባዎችን ለመሳብ የመጀመሪያ ዓመት ቅናሽ ዋጋ ያቅርቡ።
* **በቅጽበት ዋጋ (Real-time pricing)** — ከአቅራቢዎች የሚመጣ የቀጥታ የጅምላ ዋጋ፣ ብልህ ካሽንግ (caching) ጋር።
* **የኩፖን ድጋፍ (Coupon support)** — ከMultisite Ultimate ቅናሽ ኮዶች ጋር ሙሉ ተኳሃኝነት።

#### WHOIS ግላዊነት ጥበቃ
* **በምርት ደረጃ ቅንብር (Per-product configuration)** — WHOIS ግላዊነትን እንደ ተሰናባሪ (Disabled)፣ ሁልጊዜ ተካቷል (Always Included)፣ ወይም የደንበኛ ምርጫ (Customer Choice) መቼት።
* **የደንበኛ ተሳትፎ (Customer opt-in)** — ለደንበኛ ምርጫ ሲቀመጥ፣ የክፍያ ገጹ ላይ የጥበቃ ምርጫ ሳጥን (checkbox) በመስጠት ደንበኞች ግላዊነት ጥበቃ እንዲጨምሩ ያስችላል።
* **ሊቀየሩ የሚችሉ ዋጋዎች (Configurable pricing)** — ለግላዊነት ተጨማሪው ዓመት ዋጋ ማስቀመጥ፣ ወይም በነጻ ማቅረብ።
* **የሪጅስትራ ውህደት (Registrar integration)** — በምዝገባ ጊዜ WhoisGuard (Namecheap) ወይም WHOIS ግላዊነት (OpenSRS) በራስ-ሰር ያን성화ዋል።

#### አንድ ወጥ የክፍያ ተሞክሮ (Unified Checkout Experience)
* **ሶስት የዶሜን ሁነታዎች** — ደንበኞች በንዑስ ዶሜን (በነጻ)፣ አዲስ ዶሜን መመዝገብ፣ ወይም የራሳቸው ቀድሞ ያለ ዶሜን መመደብ (mapping) ይችላሉ።
* **በቅጽበት የዶሜን ፍለጋ (Real-time domain search)** — ገጽ ሳይታደስ (no page reloads) የሚሰራ፣ በAJAX የሚንቀሳቀስ የቅጽበት ተገኝነት ፍተሻ።
* **ብልህ ምክሮች (Smart suggestions)** — የተመረጠው ዶሜን ከተያዘ፣ አማራጭ TLDs ምክሮችን ይሰጣል።
* **ግልጽ ዋጋ አወጣጥ (Transparent pricing)** — የምዝገባ ዋጋን፣ የ እድሳት ዋጋን እና አማራጭ የWHOIS ግላዊነት ክፍያ በግልጽ ያሳያል።
* **የመመዝጋቢ የእውቂያ መስኮች (Registrant contact fields)** — የሚያስፈልጉትን የመመዝጋቢ መረጃ (ስም፣ አድራሻ፣ ስልክ) በክፍያ ገጹ ላይ በቦታው መሰብሰብ።
* **በራስ-ሰር የተፈጠረ የድረ-ገጽ URL (Auto-generated site URL)** — የድረ-ገጹ አድራሻ ከመረጡት ዶሜን በራስ-ሰር ይወሰናል።

#### የራስ-ሰር እድሳት ሥርዓት (Auto-Renewal System)
* **በአባልነት የተያያዘ እድሳት (Membership-tied renewals)** — የደንበኛው የአባልነት ሁኔታ እስካለ ድረስ ዶሜኖቹ በራስ-ሰር ይታደሳሉ።
* **ብልህ መርሐግብር (Smart scheduling)** — የእድሳት ስራዎች በዶሜን የመልቀቅ ቀናት መሠረት ይደረደራሉ።
* **የእንደገና ሙከራ አመክንዮ (Retry logic)** — ለተሳናቸው የእድሳት ሙከራዎች ብልህ የሆነ እንደገና መሞከር።
* **የኢሜይል ማሳወቂያዎች (Email notifications)** — ለሚመጡ እድሳት፣ ለስኬታማ እድሳት፣ ለውድቀቶች እና ለማጥፋት ሊቀየሩ የሚችሉ ማስጠንቀቂያዎች።

#### የደንበኛ DNS አስተዳደር
* **በራስ-ሰር የDNS (Self-service DNS)** — ደንበኞች በዳሽቦርዳቸው A, AAAA, CNAME, MX, እና TXT ሪከርዶችን መጨመር፣ ማርትዕ እና መሰረዝ ይችላሉ።
* **የዶሜን ሁኔታ (Domain status)** — የቅጽበት ምዝገባ ሁኔታ፣ የመልቀቅ ቀናት እና የnameserver መረጃ።
* **በራስ-ሰር የዶሜን መመደብ (Automatic domain mapping)** — የተመዘገቡ ወይም ቀድሞ ያሉ ዶሜኖች በራስ-ሰር ለደንበኛው ድረ-ገጽ ይተዳደራሉ።

#### የአስተዳዳሪ መሳሪያዎች (Admin Tools)
* **በእጅ የዶሜን ምዝገባ (Manual domain registration)** — ዶሜኖችን በደንበኞች ስም ከአስተዳዳሪ ፓነል መመዝገብ።
* **የዶሜን ክትትል (Domain monitoring)** — ሁሉንም የደንበኞች ዶሜኖች፣ ሁኔታዎች እና የመልቀቅ ቀናት በአንድ ገጽ መመልከት።
* **በአቅራቢው የተለየ መመዝገቢያ (Provider-specific logging)** — ለእያንዳንዱ ሪጅስትራ የተለየ የሎግ ሰርጥ (ለምሳሌ: domain-seller-namecheap.log)።
* **የምዝገባ ዝርዝሮች (Registration details)** — የመመዝጋቢ መረጃ እና የምዝገባ መረጃን በዋናው የዶሜን ማርትዕ ገጽ ላይ መመልከት።

### የሚደገፉ አቅራቢዎች (Supported Providers)

* **OpenSRS** — ከምዝገባ፣ እድሳት፣ DNS አስተዳደር እና WHOIS ግላዊነት ጋር ሙሉ ውህደት።
* **Namecheap** — ከምዝገባ፣ እድሳት፣ የጅምላ ተገኝነት ፍተሻ እና ነጻ WhoisGuard ጋር ሙሉ ውህደት።
* **GoDaddy** — ምዝገባ እና እድሳት።
* **ResellerClub** — ምዝገባ፣ እድሳት እና DNS አስተዳደር።
* **NameSilo** — ምዝገባ እና እድሳት።
* **Enom** — ምዝገባ፣ እድሳት እና DNS አስተዳደር።

### ቴክኒካዊ ዝርዝሮች (Technical Details)

* **ይፈልጋል (Requires):** PHP 7.4+, Multisite Ultimate 2.4.12+
* **አርክቴክቸር (Architecture):** Integration Registry pattern with Domain_Selling_Capability interface
* **Autoloading:** Jetpack autoloader with PSR-4 class mapping
* **Standards:** WordPress coding standards, strict typing, prepared statements

## መጫን (Installation)

1. **Multisite Ultimate v2.4.12+** መጫን እና በኔትወርክ ደረጃ መ활성화 መሆኑን ያረጋግጡ።
2. ፕልጊኑን ወደ `/wp-content/plugins/ultimate-multisite-domain-seller/` ይስቀሉ።
3. በWordPress Network Admin ውስጥ ከ**Plugins** ጽሑፍ ውስጥ ፕልጊኑን በኔትወርክ ደረጃ ያን성화ሉ።
4. **Setup Wizard** በመጀመሪያ ጊዜ ሲን성화 በራስ-ሰር ይከፈታል — የሚከተሉትን ለማድረግ ደረጃዎችን ይከተሉ፦
   * የዶሜን ሪጅስትራዎን ያገናኙ (OpenSRS ወይም Namecheap)
   * ግንኙነቱን ይፈትሹ
   * የሚገኙ TLDs እና የጅምላ ዋጋዎችን ያስመጡ
   * የዋጋ ጭማሪ ባለው የመጀመሪያ የዶሜን ምርትዎን ይፍጠሩ
5. **Domain Selection** መስኩን በ**Ultimate Multisite > Checkout Forms** ውስጥ ባለው የክፍያ ቅጽዎ ላይ ይጨምሩ።

## በተደጋጋሚ የሚጠየቁ ጥያቄዎች (Frequently Asked Questions)

### የMultisite Ultimate ምን ስሪት ያስፈልጋል?

ስሪት 2.4.12 ወይም ከዚያ በላይ። ይህ ተጨማሪ አገልግሎት (addon) ከWP Ultimo v1.x ወይም ከ2.4.12 በታች ከሚገኙ Ultimate Multisite ስሪቶች ጋር አይሰራም።

### የትኞቹ የዶሜን አቅራቢዎች ይደገፋሉ?

ስድስት ሪጅስትራዎች ሙሉ በሙሉ ይደገፋሉ፦ **OpenSRS**, **Namecheap**, **GoDaddy**, **ResellerClub**, **NameSilo**, እና **Enom**። የaddon's Integration Registry architecture አዳዲስ አቅራቢዎችን መጨመር ቀላል ያደርገዋል — [developer guide](/developer/integration-guide/custom-registrar) ላይ ይመልከቱ።

### የWHOIS ግላዊነት ጥበቃ ማቅረብ እችላለሁ?

አዎ። እያንዳንዱ የዶሜን ምርት በሶስት ሁነታዎች የWHOIS ግላዊነት ቅንብር አለው፦ ተሰናባሪ (Disabled)፣ ሁልጊዜ ተካቷል (Always Included)፣ ወይም የደንበኛ ምርጫ (Customer Choice)። ለደንበኛ ምርጫ ሲቀመጥ፣ ደንበኞች በክፍያ ገጹ ላይ የዋጋ ሳጥን (ወይም ለNamecheap WhoisGuard "ነጻ") ያያሉ። የግላዊነት ባንዲራው በምዝገባ ጊዜ በሪጅስትራው ውስጥ በራስ-ሰር ይተላለፋል።

### ለተለያዩ TLDs የተለያየ ዋጋ ማስቀመጥ እችላለሁ?

አዎ። ለተለያዩ የTLD ቡድኖች የተለየ የዶሜን ምርት ይፍጠሩ። እያንዳንዱ ምርት የራሱ የዋጋ ጭማሪ አይነት (በመቶኛ፣ የተወሰነ ጭማሪ፣ ወይም የተወሰነ ዋጋ)፣ የሚደገፉ TLDs፣ እና የመነሻ ዋጋ ሊኖረው ይችላል።

### ደንበኞች ምን ዓይነት የክፍያ አማራጮችን ያያሉ?

የDomain Selection መስኩ ለደንበኞች ሶስት ትሮች ይሰጣል፦ **Subdomain** (በኔትወርክዎ ላይ ነጻ ንዑስ ዶሜን መጠቀም)፣ **Register New Domain** (አዲስ ዶሜን መፈለግ እና መመዝገብ)፣ እና **Existing Domain** (የራሳቸው ቀድሞ ያለ ዶሜን መመደብ)። እርስዎ የሚያንጽፉት ሁነታዎች ብቻ ይታያሉ።

### በራስ-ሰር እድሳት ይደግፋል?

አዎ። ዶሜኖች በራስ-ሰር እንዲታደሱ ሊቀመጡ ይችላሉ እና ከደንበኛው የአባልነት ሁኔታ ጋር የተያያዙ ናቸው። የእድሳት ክፍያዎች በደንበኛው ቀድሞ ባለው የክፍያ መግቢያ (payment gateway) በኩል ይከናወናሉ።

### ደንበኞች የDNS ሪከርዶቻቸውን ማስተዳደር ይችላሉ?

አዎ። ደንበኞች A, AAAA, CNAME, MX, እና TXT ሪከርዶችን ለመጨመር፣ ለማርትዕ እና ለማጥፋት በዳሽቦርዳቸው የDNS አስተዳደር መገናኛ ያገኛሉ። የDNS አስተዳደር በOpenSRS, ResellerClub, እና Enom ይገኛል።

### የቅንብር አስማሚ አለ?

አዎ። የቅንብር አስማሚው በመጀመሪያ ጊዜ ሲን성화 በራስ-ሰር ይሰራል እና ሪጅስትራን ማገናኘት፣ TLDs ማስመጣት እና የመጀመሪያ የዶሜን ምርትዎን መፍጠር ያስተምራዎታል። አስማሚውን ከaddon ቅንብሮች থেকেও እንደገና ማስኬድ ይችላሉ።
