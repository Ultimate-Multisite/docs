# Domain Seller

**Multisite Ultimate: Domain Seller** የእርስዎን WordPress Multisite አውታረ መረብ ወደ ኃይለኛ የዶሜይን መሸጫ መድረክ ይቀይራል። ደንበኞች በመመዝገብ ጊዜ ወይም ከdashboard-ቸው በቀጥታ ብጁ ዶሜይኖችን እንዲመዘግቡ ያስችላል፣ በቅጽበታዊ የመገኘት ምርመራ እና ግልጽ ዋጋ አሰጣጥ።

### ለምን Domain Seller?

የዶሜይን ምዝገባ ለማንኛውም WaaS (ድረገጽ እንደ አገልግሎት) መድረክ በጣም ከሚጠየቁ ተጨማሪ ሽያጮች አንዱ ነው። ደንበኞችዎን ወደ የሶስተኛ ወገን መዝጋቢ ከመላክ ይልቅ፣ ዶሜይኖችን በቀጥታ ማቅረብ ይችላሉ — ደንበኞችን እንከን የሌለው ተሞክሮ ሲሰጡ ተጨማሪ ገቢን በአውታረ መረብዎ ውስጥ ያስቀራሉ።

### ቁልፍ ባህሪያት

#### የብዙ ዶሜይን መዝጋቢዎች ድጋፍ
* **OpenSRS** — ሙሉ ምዝገባ፣ እድሳት እና አስተዳደር
* **Namecheap** — ከWhoisGuard እና የsandbox ሙከራ ጋር ሙሉ ውህደት
* **HostAfrica** — በDomains Reseller API በኩል የዶሜይን ፍለጋ፣ ዋጋ አሰጣጥ፣ ምዝገባ፣ ማስተላለፍ፣ DNS እና የመታወቂያ ጥበቃ
* **Openprovider** — የreseller ዋጋ አሰጣጥ፣ ምዝገባ፣ እድሳት፣ ማስተላለፎች፣ DNS፣ WHOIS ግላዊነት እና የTLD ማመሳሰል
* **Hostinger** — የተጋራውን Hostinger ውህደት token በመጠቀም የዶሜይን ፍለጋ እና ምዝገባ
* **ሊሰፋ የሚችል አርክቴክቸር** — በIntegration Registry ንድፍ ላይ የተገነባ፣ ተጨማሪ አቅራቢዎችን ማከል ቀላል ያደርገዋል
* **አብሮገነብ የግንኙነት ሙከራ** — በቀጥታ ከመስራት በፊት የአቅራቢ ማረጋገጫ መረጃዎችን እና ግንኙነትን ያረጋግጡ

#### የሚመራ የማዋቀር መምሪያ
* **በአንድ ጠቅታ ማስጀመር** — በአቅራቢ ውቅር፣ TLD ማስመጣት እና የዶሜይን ምርት ፈጠራ ውስጥ ይመራዎታል
* **ራስ-ሰር የTLD ማመሳሰል** — ከተገናኙ አቅራቢዎችዎ ሁሉንም ያሉ  TLDዎች እና የጅምላ ዋጋ አሰጣጥ ያስመጣል
* **ነባሪ የምርት ፈጠራ** — ምክንያታዊ ነባሪ የmarkup ቅንብሮች ያለው ለመጠቀም ዝግጁ የዶሜይን ምርት ይፈጥራል

#### ተለዋዋጭ የዶሜይን ዋጋ አሰጣጥ
* **ሶስት የmarkup ሁነቶች** — የመቶኛ markup፣ ቋሚ markup፣ ወይም በእያንዳንዱ TLD ቋሚ ዋጋ
* **TLD-ተኮር ምርቶች** — ለተለያዩ የTLD ቡድኖች (.com/.net፣ .io፣ የአገር ኮዶች፣ ወዘተ) የተለዩ የዶሜይን ምርቶችን ይፍጠሩ
* **የመግቢያ ዋጋ አሰጣጥ** — አዲስ ምዝገባዎችን ለመሳብ ቅናሽ ያለው የመጀመሪያ ዓመት ዋጋ ያቅርቡ
* **ቅጽበታዊ ዋጋ አሰጣጥ** — ከአቅራቢዎች የቀጥታ የጅምላ ዋጋ አሰጣጥ ከብልህ caching ጋር
* **የcoupon ድጋፍ** — ከMultisite Ultimate የቅናሽ ኮዶች ጋር ሙሉ ተኳኋኝነት

#### የWHOIS ግላዊነት ጥበቃ
* **በእያንዳንዱ ምርት ውቅር** — የWHOIS ግላዊነትን Disabled፣ Always Included፣ ወይም Customer Choice ያድርጉ
* **የደንበኛ opt-in** — Customer Choice ሲደረግ፣ በcheckout ላይ ያለ checkbox ደንበኞች የግላዊነት ጥበቃን እንዲጨምሩ ያስችላቸዋል
* **ሊዋቀር የሚችል ዋጋ አሰጣጥ** — ለግላዊነት ተጨማሪ አገልግሎት በዓመት ዋጋ ያዘጋጁ፣ ወይም በነጻ ያቅርቡት
* **የመዝጋቢ ውህደት** — በምዝገባ ጊዜ WhoisGuard (Namecheap) ወይም WHOIS ግላዊነት (OpenSRS) ራስ-ሰር ያነቃል

#### የተዋሃደ Checkout ተሞክሮ
* **ሶስት የዶሜይን ሁነቶች** — ደንበኞች በsubdomain (ነጻ)፣ አዲስ ዶሜይን መመዝገብ፣ ወይም ካስቀድሞ ያላቸው ነባር ዶሜይን መስበክ መካከል ይመርጣሉ
* **ቅጽበታዊ የዶሜይን ፍለጋ** — በAJAX የሚሰራ የመገኘት ምርመራ ያለ ገጽ ዳግም መጫን
* **ብልህ ጥቆማዎች** — የተመረጠው ዶሜይን ተይዞ ሲሆን አማራጭ TLD ጥቆማዎች
* **ግልጽ ዋጋ አሰጣጥ** — የምዝገባ ዋጋ፣ የእድሳት ዋጋ እና አማራጭ የWHOIS ግላዊነት ክፍያ ግልጽ ማሳያ
* **የመዝጋቢ የእውቂያ መስኮች** — በcheckout ጊዜ የሚያስፈልግ የመዝጋቢ መረጃ (ስም፣ አድራሻ፣ ስልክ) በመስመር ላይ ይሰበስባል
* **ራስ-ሰር የተፈጠረ የጣቢያ URL** — የጣቢያ አድራሻ ከተመረጠው ዶሜይን ራስ-ሰር ይወጣል

#### የራስ-ሰር እድሳት ስርዓት
* **ከአባልነት ጋር የተያያዙ እድሳቶች** — የደንበኛው አባልነት ንቁ ሆኖ እያለ ዶሜይኖች ራስ-ሰር ይታደሳሉ
* **ብልህ መርሐግብር ማስያዝ** — የእድሳት ስራዎች በዶሜይን ማብቂያ ቀናት መሰረት በተራ ይደረደራሉ
* **የዳግም ሙከራ ሎጂክ** — ያልተሳኩ የእድሳት ሙከራዎች ላይ ብልህ ዳግም ሙከራዎች
* **የኢሜይል ማሳወቂያዎች** — ለሚመጡ እድሳቶች፣ ለተሳኩ እድሳቶች፣ ለውድቀቶች እና ለማብቂያዎች ሊዋቀሩ የሚችሉ ማንቂያዎች

#### የደንበኛ DNS አስተዳደር
* **የራስ-አገልግሎት DNS** — ደንበኞች ከdashboard-ቸው የDNS መዝገቦችን (A፣ AAAA፣ CNAME፣ MX፣ TXT) ማከል፣ ማርትዕ እና መሰረዝ ይችላሉ
* **የዶሜይን ሁኔታ** — ቅጽበታዊ የምዝገባ ሁኔታ፣ የማብቂያ ቀናት እና የnameserver መረጃ
* **ራስ-ሰር የዶሜይን መስበክ** — የተመዘገቡ ወይም ነባር ዶሜይኖች ወደ ደንበኛው ጣቢያ ራስ-ሰር ይመሰብካሉ

#### የAdmin መሳሪያዎች
* **በእጅ የዶሜይን ምዝገባ** — ከadmin panel በመነሳት በደንበኞች ስም ዶሜይኖችን ይመዝግቡ
* **የዶሜይን ክትትል** — ሁሉንም የደንበኛ ዶሜይኖች፣ ሁኔታዎች እና የማብቂያ ቀናት ከአንድ ገጽ ይመልከቱ
* **አቅራቢ-ተኮር መዝገብ ማስቀመጥ** — ለእያንዳንዱ መዝጋቢ የተለዩ የlog ቻናሎች (ለምሳሌ፣ domain-seller-namecheap.log)
* **የምዝገባ ዝርዝሮች** — የመዝጋቢ መረጃ እና የምዝገባ ውሂብ በዋናው የዶሜይን ማርትያ ገጽ ላይ ይመልከቱ

### የሚደገፉ አቅራቢዎች

* **OpenSRS** — ከምዝገባ፣ እድሳት፣ DNS አስተዳደር እና WHOIS ግላዊነት ጋር ሙሉ ውህደት
* **Namecheap** — ከምዝገባ፣ እድሳት፣ የቡድን የመገኘት ምርመራ እና ነጻ WhoisGuard ጋር ሙሉ ውህደት
* **HostAfrica** — ምዝገባ፣ እድሳት፣ ማስተላለፎች፣ DNS አስተዳደር፣ የEPP ኮድ ማግኘት፣ የመዝጋቢ መቆለፊያ እና የመታወቂያ ጥበቃ
* **Openprovider** — ምዝገባ፣ እድሳት፣ ማስተላለፎች፣ DNS አስተዳደር፣ የEPP ኮድ ማግኘት፣ የመዝጋቢ መቆለፊያ፣ WHOIS ግላዊነት እና የreseller TLD ማመሳሰል
* **Hostinger** — ምዝገባ፣ የዶሜይን ፍለጋ፣ የnameserver ዝመናዎች፣ የመዝጋቢ መቆለፊያ እና WHOIS ግላዊነት፤ እድሳቶች በHostinger ራስ-ሰር እድሳት ይከናወናሉ
* **GoDaddy** — ምዝገባ እና እድሳት
* **ResellerClub** — ምዝገባ፣ እድሳት እና DNS አስተዳደር
* **NameSilo** — ምዝገባ እና እድሳት
* **Enom** — ምዝገባ፣ እድሳት እና DNS አስተዳደር

### ቴክኒካዊ ዝርዝሮች

* **ያስፈልጋል:** PHP 7.4+፣ Multisite Ultimate 2.4.12+
* **አርክቴክቸር:** የIntegration Registry ንድፍ ከDomain_Selling_Capability interface ጋር
* **Autoloading:** Jetpack autoloader ከPSR-4 class mapping ጋር
* **መመዘኛዎች:** WordPress የcoding standards፣ strict typing፣ prepared statements

## መጫን

1. **Multisite Ultimate v2.4.12+** መጫኑን እና በኔትወርክ ደረጃ መንቃቱን ያረጋግጡ
2. pluginን ወደ `/wp-content/plugins/ultimate-multisite-domain-seller/` ይስቀሉ
3. pluginን ከWordPress Network Admin ውስጥ ከ**Plugins** በኔትወርክ ደረጃ ያንቁ
4. **Setup Wizard** በመጀመሪያ ማንቃት ጊዜ በራሱ ይጀምራል — ደረጃዎቹን ይከተሉ ለ፦
   * የዶሜን ሬጅስትራርዎን ማገናኘት
   * ግንኙነቱን መፈተሽ
   * ያሉ TLDዎችን እና የጅምላ ዋጋ መረጃን ማስገባት
   * የመጀመሪያውን የዶሜን ምርትዎን ከዋጋ ጭማሪ ጋር መፍጠር
5. የ**Domain Selection** መስክን በ**Ultimate Multisite > Checkout Forms** ውስጥ ወዳለው የcheckout ቅጽዎ ያክሉ

## ብዙ ጊዜ የሚጠየቁ ጥያቄዎች

### የትኛው የMultisite Ultimate ስሪት ያስፈልጋል?

ስሪት 2.4.12 ወይም ከዚያ በላይ። addon ከWP Ultimo v1.x ወይም ከ2.4.12 በታች ካሉ Ultimate Multisite ስሪቶች ጋር ተኳሃኝ አይደለም።

### የትኞቹ የዶሜን አቅራቢዎች ይደገፋሉ?

ዘጠኝ ሬጅስትራሮች ይደገፋሉ፦ **OpenSRS**, **Namecheap**, **HostAfrica**, **Openprovider**, **Hostinger**, **GoDaddy**, **ResellerClub**, **NameSilo**, እና **Enom**። የሬጅስትራር APIዎች የተለያዩ ክዋኔዎችን ስለሚያቀርቡ ችሎታዎች እንደ አቅራቢው ይለያያሉ፤ ለእያንዳንዱ አቅራቢ አጠቃላይ እይታ የማዋቀሪያ መመሪያውን ይመልከቱ። የaddon Integration Registry አርክቴክቸር ብጁ አቅራቢዎችን መጨመር ቀላል ያደርገዋል — [የገንቢ መመሪያ](/developer/integration-guide/custom-registrar)ን ይመልከቱ።

### የWHOIS ግላዊነት ጥበቃ ማቅረብ እችላለሁ?

አዎ። እያንዳንዱ የዶሜን ምርት ሶስት ሁነቶች ያሉት የWHOIS ግላዊነት ቅንብር አለው፦ ተሰናክሏል፣ ሁልጊዜ የተካተተ፣ ወይም የደንበኛ ምርጫ። ወደ የደንበኛ ምርጫ ሲቀናበር፣ ደንበኞች በcheckout ጊዜ ከዋጋው ጋር (ወይም ለNamecheap WhoisGuard "Free") የምልክት ሳጥን ያያሉ። የግላዊነት ምልክቱ በምዝገባ ጊዜ በራሱ ወደ ሬጅስትራሩ ይላካል።

### ለተለያዩ TLDዎች የተለያየ ዋጋ ማዘጋጀት እችላለሁ?

አዎ። ለተለያዩ የTLD ቡድኖች የተለያዩ የዶሜን ምርቶችን ይፍጠሩ። እያንዳንዱ ምርት የራሱ የጭማሪ አይነት (መቶኛ፣ ቋሚ ጭማሪ፣ ወይም ቋሚ ዋጋ)፣ የሚደገፉ TLDዎች፣ እና የመግቢያ ዋጋ ሊኖረው ይችላል።

### ደንበኞች ምን የcheckout አማራጮችን ያያሉ?

የDomain Selection መስክ ለደንበኞች ሶስት ትሮችን ይሰጣል፦ **Subdomain** (በኔትወርክዎ ላይ ነፃ ንዑስ ዶሜን ይጠቀሙ)፣ **Register New Domain** (አዲስ ዶሜን ይፈልጉ እና ይመዝግቡ)፣ እና **Existing Domain** (ቀድሞ ያላቸውን ዶሜን ያዛምዱ)። ያነቃችኋቸው ሁነቶች ብቻ ይታያሉ።

### ራስ-ሰር እድሳቶችን ይደግፋል?

አዎ። ዶሜኖች ራስ-ሰር እንዲታደሱ ሊቀናበሩ ይችላሉ እና ከደንበኛው የmembership ሁኔታ ጋር የተያያዙ ናቸው። የእድሳት ክፍያዎች በደንበኛው ነባር የክፍያ መግቢያ በኩል ይካሄዳሉ።

### ደንበኞች የDNS መዝገቦቻቸውን ማስተዳደር ይችላሉ?

አዎ። ደንበኞች A, AAAA, CNAME, MX, እና TXT መዝገቦችን ለመጨመር፣ ለማርትዕ፣ እና ለመሰረዝ በdashboardዎቻቸው ውስጥ የDNS አስተዳደር በይነገጽ ያገኛሉ። የDNS አስተዳደር ከOpenSRS, ResellerClub, Enom, HostAfrica, እና Openprovider ጋር ይገኛል። Hostinger DNS ለተስተናገዱ ዶሜኖች በዋናው Hostinger የዶሜን-ማዛመድ ውህደት ይከናወናል።

### የማዋቀሪያ አዋቂ አለ?

አዎ። የማዋቀሪያ አዋቂው በመጀመሪያ ማንቃት ጊዜ በራሱ ይሮጣል እና ሬጅስትራር ማገናኘት፣ TLDዎችን ማስገባት፣ እና የመጀመሪያውን የዶሜን ምርትዎን መፍጠር ያስተምራችኋል። እንዲሁም አዋቂውን ከaddon ቅንብሮች እንደገና ማስኬድ ይችላሉ።
