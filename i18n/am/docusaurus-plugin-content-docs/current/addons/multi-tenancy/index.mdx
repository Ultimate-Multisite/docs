# ባለብዙ-ተከራይነት {#multi-tenancy}

ለአፈጻጸም እና ለደህንነት ንዑስ ጣቢያዎችን በአገልጋዮች እና በውሂብ ጎታዎች ላይ ያሰራጩ

## Version 1.2.0 የልቀት ዋና ዋና ነጥቦች {#version-120-release-highlights}

Multi-Tenancy 1.2.0 ለምርት መነጠል የስራ ፍሰቶች የሉዓላዊ ተከራይ መሠረትን ያክላል፦

- ሁኔታ-አልባ የተከራይ ራስ-ሰር መግቢያ ከአጭር ዕድሜ እና ዳግም-መጫወት-የተጠበቁ SSO ጉብኝቶች ጋር።
- ለሉዓላዊ ተከራዮች በእያንዳንዱ ንዑስ ጣቢያ የውሂብ ጎታ እና የፋይል ስርዓት መነጠል።
- ለአስተናጋጅ-አዋቂ የተከራይ ማዘጋጀት የHostinger ባለብዙ-ተከራይነት ችሎታ።
- ለቆዩ ውሂብ ምርመራዎች እና async ግፊት ማስኬድ የሉዓላዊ ማዛወር ማረጋገጫ በሮች።
- ለBedrock፣ FrankenPHP እና localhost ጭነቶች የሚዋቀር በተመሳሳይ-ማሽን የውሂብ ጎታ አስተናጋጅ ማሰር።
- የተከራይ መነሻ ማስጀመር፣ የውሂብ ጎታ ጸሐፊ፣ የschema መነሻ ማስጀመር እና የተጠቃሚ ማዘጋጀት ማሻሻያዎች።
- የተከራይ ምስክርነቶችን የሚያጸዳ ይበልጥ ደህንነቱ የተጠበቀ ሉዓላዊ ማፍረስ።

ለ[የሉዓላዊ ማዛወር የስራ ፍሰት](/user-guide/getting-started/sovereign-migration-workflow)፣ [የባለብዙ-ተከራይነት መነጠል](/user-guide/configuration/multi-tenancy-isolation) እና [የሉዓላዊ ተከራይ ክወናዎች](/user-guide/administration/sovereign-tenant-operations) የተጠቃሚ መመሪያዎችን ይመልከቱ።

## መስፈርቶች {#requirements}

- WordPress 5.3 ወይም ከዚያ በላይ
- PHP 7.4 ወይም ከዚያ በላይ
- Multisite Ultimate plugin (ንቁ)

## መጫን {#installation}

1. የaddon ፋይሎችን ወደ `/wp-content/plugins/` ማውጫዎ ይስቀሉ
2. በWordPress ውስጥ በ'Plugins' ምናሌ በኩል pluginን ያንቁ
3. በኔትወርክ አስተዳዳሪ ውስጥ የaddon ቅንብሮችን ያዋቅሩ

## ልማት {#development}

### ማዋቀር {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### የፕሮጀክት መዋቅር {#project-structure}

```
ultimate-multisite-addon-template/
```

### መሞከር {#testing}

addon ሁሉን አቀፍ የሙከራ መዋቅር ያካትታል፦

- **ዩኒት ሙከራዎች**፦ በPHPUnit ላይ የተመሠረቱ ሙከራዎች ከWordPress የሙከራ ስብስብ ጋር
- **መሠረታዊ የሙከራ ክፍል**፦ ለተለመዱ የሙከራ ሁኔታዎች አጋዥ ዘዴዎችን ይሰጣል
- **የሙከራ ሽፋን**፦ የኮድ ሽፋንን ለመከታተል ተዋቅሯል
- **Multisite መሞከር**፦ ሙከራዎች በmultisite አካባቢ ውስጥ ይሰራሉ

### የኮድ መስፈርቶች {#code-standards}

- **WordPress የኮድ መስፈርቶች**፦ በPHPCS በኩል ይተገበራል
- **PHP 7.4+ ተኳኋኝነት**፦ ዘመናዊ የPHP ባህሪያት ይደገፋሉ
- **Static Analysis**፦ ለአይነት ምርመራ የPHPStan ውህደት
- **የኮድ ዘመናዊकरण**፦ ለPHP ማሻሻያዎች የRector ደንቦች

### የግንባታ ስርዓት {#build-system}

- **የንብረት ማስኬድ**፦ ለCSS/JS ፋይሎች ማነስ
- **ትርጉም**፦ የPOT ፋይል ማመንጨት
- **ማሸግ**፦ ለስርጭት የarchive መፍጠር
- **የልማት ሁነታ**፦ ለdebugging ያልተቀነሱ ንብረቶች

## ውቅር {#configuration}

addon በኔትወርክ አስተዳዳሪ በይነገጽ በኩል የሚደረሱ የተለያዩ የውቅር አማራጮችን ይደግፋል።

### የሉዓላዊ ተከራይ ውቅር {#sovereign-tenant-configuration}

ለሉዓላዊ ተከራዮች፣ ትራፊክን ወደ ተከራዩ ከማዞርዎ በፊት የተከራይ ውሂብ ጎታ፣ የፋይል ስርዓት ስር፣ የውሂብ ጎታ አስተናጋጅ ማሰር እና የአስተናጋጅ ችሎታን ያዋቅሩ። የውሂብ ጎታ ምስክርነቶች፣ የአስተናጋጅ ማሰር፣ ማዞር ወይም የተጠቃሚ ማዘጋጀትን የሚነካ እያንዳንዱ ለውጥ በኋላ የማዛወር ማረጋገጫ ያስኪዱ።

የተከራይ runtime እና የውሂብ ጎታ grants አካባቢያዊ socket ወይም ተመሳሳይ-ማሽን መዳረሻን ሲጠብቁ ብቻ `localhost` ይጠቀሙ። በአቅራቢዎ የተዋቀረው grant host ያ ከሆነ `127.0.0.1` ወይም የአገልግሎት አስተናጋጅ ስም ይጠቀሙ።

### Hostinger ችሎታ {#hostinger-capability}

የHostinger ባለብዙ-ተከራይነት ችሎታ ከተጋራው የHostinger ውህደት token ጋር ይሰራል። ከማረጋገጫ በፊት domainው በhPanel ውስጥ መኖሩን፣ DNS ወደ Hostinger account መጠቆሙን እና የተከራይ ውሂብ ጎታ ቅንብሮች ከhPanel ጋር መዛመዳቸውን ያረጋግጡ።

## Hooks እና Filters {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - ከaddon መነሻ ማስጀመር በኋላ ይተኮሳል
- `ultimate-multisite-addon-template_loaded` - ሁሉም የaddon ፋይሎች ከተጫኑ በኋላ ይተኮሳል

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - የaddon ቅንብሮችን ያሻሽሉ
- `ultimate-multisite-addon-template_enabled` - የaddon ማንቃት/ማሰናከል ሁኔታን ይተኩ
