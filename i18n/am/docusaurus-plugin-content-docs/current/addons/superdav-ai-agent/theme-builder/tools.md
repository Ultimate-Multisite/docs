# መሳሪያዎች እና ውቅር

Theme Builder የWordPress ጣቢያዎን ለመተንተን፣ ለመንደፍ እና ለመገንባት የመሳሪያዎች ስብስብ ይጠቀማል። በስሪት 1.16.0፣ **sd-ai-agent/site-scrape አሁን Tier 1 መሳሪያ ነው**፣ ይህም በTheme Builder ውስጥ በነባሪነት እንዲገኝ ያደርገዋል። Superdav AI Agent v1.18.0 ለታመኑ የገንቢ የስራ ፍሰቶች በተለይ የሚሰራጭ Advanced ተጓዳኝ ጥቅልም ያስተዋውቃል፤ እነዚያ መሳሪያዎች የWordPress.org ዋና ጥቅል ክፍል አይደሉም እና በተለይ መጫን እና መፈቀድ አለባቸው።

## የሚገኙ መሳሪያዎች {#available-tools}

### Tier 1 መሳሪያዎች (ሁልጊዜ የሚገኙ) {#tier-1-tools-always-available}

Tier 1 መሳሪያዎች በTheme Builder ውስጥ ምንም ተጨማሪ ውቅር ሳያስፈልግ በነባሪነት ይገኛሉ።

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**ዓላማ:** የንድፍ መነሳሻ፣ የይዘት መዋቅር እና ተግባራዊነትን ለማውጣት ነባር ድረ ገጾችን መተንተን።

**ችሎታዎች:**

- **የድረ ገጽ ትንተና** — የተወዳዳሪ ወይም የመነሳሻ ድረ ገጾችን መቧጨር እና መተንተን
- **የንድፍ ማውጣት** — ቀለሞችን፣ ፊደል ቅርጾችን እና የአቀማመጥ ንድፎችን መለየት
- **የይዘት መዋቅር** — የገጽ አደረጃጀትን እና ደረጃ መዋቅርን መረዳት
- **የባህሪ መለየት** — plugins እና ተግባራዊነትን መለየት
- **የአፈጻጸም ትንተና** — የገጽ ፍጥነትን እና ማሻሻያን መፈተሽ
- **የSEO ትንተና** — የmeta መለያዎችን እና የተዋቀረ ውሂብን መገምገም

**አጠቃቀም:**

```
የexample.com ንድፍን ተንትነህ ለጣቢያዬ እንደ መነሳሻ ተጠቀምበት።
```

**የሚያወጣው:**

- የቀለም ስብስብ እና የቀለም አጠቃቀም
- ታይፖግራፊ (ፊደል ቅርጾች እና መጠኖች)
- የአቀማመጥ መዋቅር እና ክፍተት
- የአሰሳ ንድፎች
- የይዘት አደረጃጀት
- ምስሎች እና የሚዲያ አጠቃቀም
- መስተጋብራዊ ክፍሎች
- የሞባይል ምላሽ ሰጪነት

**ገደቦች:**

- በይለፍ ቃል የተጠበቁ ጣቢያዎችን መቧጨር አይችልም
- robots.txt እና የጣቢያ ፖሊሲዎችን ያከብራል
- ተለዋዋጭ ይዘትን ላይይዝ ይችላል
- JavaScript በብዛት የሚጠቀሙ ጣቢያዎች ውስን ትንተና ሊኖራቸው ይችላል
- ትላልቅ ጣቢያዎች ለመተንተን ተጨማሪ ጊዜ ሊወስዱ ይችላሉ

### Tier 2 መሳሪያዎች (አማራጭ) {#tier-2-tools-optional}

Tier 2 መሳሪያዎች በTheme Builder ቅንብሮች ውስጥ በግልጽ ሲነቁ ይገኛሉ።

#### የላቀ ትንታኔ {#advanced-analytics}

- የገጽ ጭነት ጊዜዎች
- Core Web Vitals
- የSEO ነጥብ
- የተደራሽነት ነጥብ
- የምርጥ አሰራሮች ነጥብ

#### የይዘት አመቻች {#content-optimizer}

ለሚከተሉት ማሻሻያዎችን ይተነትናል እና ይጠቁማል፦

- የማንበብ ቀላልነት
- የSEO ማመቻቸት
- የቁልፍ ቃል አጠቃቀም
- የይዘት መዋቅር
- የድርጊት ጥሪ አቀማመጥ

### Tier 3 መሳሪያዎች (Premium) {#tier-3-tools-premium}

Tier 3 መሳሪያዎች ተጨማሪ ውቅር ወይም premium ባህሪዎችን ይፈልጋሉ።

#### AI ይዘት አመንጪ {#ai-content-generator}

ለሚከተሉት ይዘት ያመነጫል፦

- የምርት መግለጫዎች
- የአገልግሎት ገጾች
- የብሎግ ልጥፎች
- Meta መግለጫዎች
- የድርጊት ጥሪ ጽሑፍ

#### የላቁ የንድፍ መሳሪያዎች {#advanced-design-tools}

የላቁ የንድፍ ችሎታዎችን ይሰጣል፦

- ብጁ CSS ማመንጨት
- አኒሜሽን መፍጠር
- መስተጋብራዊ ክፍል ንድፍ
- የላቀ የቀለም ንድፈ ሃሳብ
- የታይፖግራፊ ማመቻቸት

### የላቁ ተጓዳኝ መሳሪያዎች (የተለየ ጥቅል) {#advanced-companion-tools-separate-package}

Advanced ተጓዳኝ ጥቅሉ ከዋናው Superdav AI Agent ልቀት ተለይቶ ይሰራጫል። አስተዳዳሪዎች ወኪሉ ከፍተኛ ተጽዕኖ ያላቸውን መሳሪያዎች እንዲጠቀም በግልጽ በሚፈቅዱባቸው ታማኝ የገንቢ እና የጣቢያ ባለቤት አካባቢዎች ውስጥ ለመጠቀም የታሰበ ነው።

የAdvanced ተጓዳኝ ችሎታዎች ሊያካትቱ የሚችሉት፦

- **የፋይል ስርዓት መሳሪያዎች** — በታማኝ አካባቢዎች ውስጥ የተፈቀዱ ፋይሎችን መመርመር እና ማሻሻል
- **የውሂብ ጎታ መሳሪያዎች** — በግልጽ ሲፈቀድ የጣቢያ ውሂብን መጠየቅ ወይም ማዘመን
- **WP-CLI መሳሪያዎች** — የWordPress ጥገና እና ምርመራ ትዕዛዞችን ማስኬድ
- **REST dispatcher መሳሪያዎች** — በተቆጣጠሩ የማስተላለፊያ የስራ ፍሰቶች በኩል የተመዘገቡ REST መጨረሻ ነጥቦችን መጥራት
- **Plugin ገንቢ መሳሪያዎች** — የplugin ኮድን መሠረት መፍጠር እና በየዙሩ ማሻሻል
- **Git ቅጽበታዊ ቅጂ መሳሪያዎች** — አደገኛ የገንቢ ክዋኔዎች ከመፈጸማቸው በፊት ቅጽበታዊ ቅጂዎችን መፍጠር
- **የተጠቃሚ አስተዳደር መሳሪያዎች** — በተፈቀደባቸው ቦታዎች የተጠቃሚ መዝገቦችን ለመገምገም ወይም ለማስተካከል መርዳት
- **የBenchmark መሳሪያዎች** — ለልማት ግምገማ የአፈጻጸም ወይም የችሎታ benchmark መለኪያዎችን ማስኬድ

እነዚህን ለእያንዳንዱ ጭነት በአጠቃላይ የሚገኙ Theme Builder መሳሪያዎች እንደሆኑ አትመዝግቧቸው። የሚገኙት Advanced ተጓዳኝ ጥቅል ሲጫን፣ ሲነቃ እና ለታመኑ አስተዳዳሪዎች ሲገደብ ብቻ ነው።

## የመሳሪያ ውቅር {#tool-configuration}

### መሳሪያዎችን ማንቃት {#enabling-tools}

በTheme Builder ውስጥ ተጨማሪ መሳሪያዎችን ለማንቃት፦

1. ወደ **Gratis AI Agent → Settings** ይሂዱ
2. ወደ **Theme Builder → Tools** ይሂዱ
3. መሳሪያዎችን እንደ አስፈላጊነቱ አብሩ/አጥፉ
4. ቅንብሮችን ያስቀምጡ

### የመሳሪያ ፈቃዶች {#tool-permissions}

አንዳንድ መሳሪያዎች ፈቃዶችን ይፈልጋሉ፦

- **የጣቢያ መቧጨር** — የበይነመረብ መዳረሻ ያስፈልጋል
- **ትንታኔ** — የGoogle Analytics ውህደት ያስፈልጋል
- **ይዘት ማመንጨት** — API ቁልፎች ያስፈልጋሉ
- **የላቁ ባህሪዎች** — premium ምዝገባ ሊያስፈልግ ይችላል
- **የላቁ ተጓዳኝ መሳሪያዎች** — በተለይ የሚሰራጨውን Advanced ጥቅል እና ግልጽ የአስተዳዳሪ እምነት ይፈልጋሉ

### API ቁልፎች እና ምስክርነቶች {#api-keys-and-credentials}

ለሚፈልጉት መሳሪያዎች API ቁልፎችን ያዋቅሩ፦

1. ወደ **Gratis AI Agent → Settings → API Keys** ይሂዱ
2. ለእያንዳንዱ መሳሪያ ምስክርነቶችን ያስገቡ
3. ግንኙነትን ይፈትሹ
4. በደህንነት ያስቀምጡ

### Advanced ተጓዳኝ ጥቅልን መጫን {#installing-the-advanced-companion-package}

Advanced ተጓዳኝ ጥቅሉ ከWordPress.org ዋና ZIP ተለይቶ ይለቀቃል። ለልቀቱ ከይፋዊው የፕሮጀክት ስርጭት ቻናል ብቻ ይጫኑት፣ ከዚያም መዳረሻን ለታመኑ አስተዳዳሪዎች ይገድቡ። በምርት ጣቢያ ላይ የፋይል ስርዓት፣ የውሂብ ጎታ፣ WP-CLI፣ REST dispatcher፣ plugin ገንቢ፣ የተጠቃሚ አስተዳደር ወይም benchmark መሳሪያዎችን ከማንቃትዎ በፊት የሰው ማጽደቂያ መግቢያዎችን እና የgit ቅጽበታዊ ቅጂ ባህሪን ይገምግሙ።

## sd-ai-agent/site-scrape መጠቀም {#using-sd-ai-agentsite-scrape}

### መሠረታዊ አጠቃቀም {#basic-usage}

Theme Builder ድረ ገጽን እንዲተነትን ይጠይቁ፦

```
በcompetitor.com ያለውን የተወዳዳሪዬን ጣቢያ ንድፍ ተንትነህ
ለጣቢያዬ የንድፍ ማሻሻያዎችን ጠቁም።
```

### የተወሰነ ትንተና {#specific-analysis}

የተወሰኑ የትንተና አይነቶችን ይጠይቁ፦

```
ከexample.com የቀለም ስብስቡን አውጣ እና እንደ መነሳሻ ተጠቀምበት።
```

```
የexample.com የአሰሳ መዋቅርን ተንትነህ ተመሳሳይ
አደረጃጀትን በጣቢያዬ ላይ ተግብር።
```

```
የexample.com የሞባይል ምላሽ ሰጪነትን ፈትሽ እና ጣቢያዬ
በእኩል ደረጃ ምላሽ ሰጪ መሆኑን አረጋግጥ።
```

### የንጽጽር ትንተና {#comparison-analysis}

ብዙ ጣቢያዎችን ያነጻጽሩ፦

```
የsite1.com እና site2.com ንድፎችን አነጻጽር እና የሁለቱንም ምርጥ
ክፍሎች የሚያጣምር ንድፍ ፍጠር።
```

## የመሳሪያ ገደቦች እና ግምቶች {#tool-limitations-and-considerations}

### የመጠን ገደብ {#rate-limiting}

- የአገልጋይ ጫናን ለመከላከል መረጃ መቧጨር በመጠን ይገደባል
- ወደ ተመሳሳይ ጣቢያ ብዙ ጥያቄዎች ሊቀነሱ ይችላሉ
- ትልቅ ጣቢያዎችን ለመተንተን የበለጠ ጊዜ ሊወስድ ይችላል

### የይዘት ትክክለኛነት {#content-accuracy}

- ተለዋዋጭ ይዘት ሙሉ በሙሉ ላይያዝ ይችላል
- በJavaScript የሚታይ ይዘት ያልተሟላ ሊሆን ይችላል
- የቀጥታ ጊዜ መረጃ ላይንጸባረቅ ይችላል
- አንዳንድ ይዘት ከክፍያ ግድግዳዎች በስተጀርባ ሊሆን ይችላል

### ሕጋዊ እና ሥነ ምግባራዊ ግምቶች {#legal-and-ethical-considerations}

- robots.txt እና የጣቢያ ፖሊሲዎችን ያክብሩ
- በቅጂ መብት የተጠበቀ ይዘትን ለመባዛት አትቧጭሩ
- ትንተናን ለመነሳሳት ይጠቀሙ፣ ለመቅዳት አይደለም
- የተወጣውን ይዘት ለመጠቀም መብት እንዳለዎት ያረጋግጡ
- የተተነተኑ ጣቢያዎችን የአገልግሎት ውሎች ይከተሉ

### በአፈጻጸም ላይ ያለ ተጽዕኖ {#performance-impact}

- ትልቅ ጣቢያዎችን መቧጨር ጊዜ ሊወስድ ይችላል
- በአንድ ጊዜ ብዙ መቧጨሮች ከተካሄዱ ዝግተኛ ሊሆኑ ይችላሉ
- የአውታረ መረብ ግንኙነት ፍጥነትን ይነካል
- ትልቅ የመረጃ ስብስቦች ተጨማሪ ሂደት ሊፈልጉ ይችላሉ

## ምርጥ ልምዶች {#best-practices}

### የጣቢያ ትንተናን መጠቀም {#using-site-analysis}

1. **ብዙ ጣቢያዎችን ይተንትኑ** — ከብዙ ምንጮች መነሳሳትን ሰብስቡ
2. **በመዋቅር ላይ ያተኩሩ** — የአቀማመጥ እና የድርጅት ንድፎችን ይማሩ
3. **ቀለሞችን ያውጡ** — የቀለም ስብስቦችን እንደ መነሻ ነጥቦች ይጠቀሙ
4. **ታይፖግራፊን ያጥኑ** — የሚወዷቸውን የፊደል ጥምረቶች ይለዩ
5. **አሰሳን ይገምግሙ** — የምናሌ ድርጅትን ይረዱ

### ሥነ ምግባራዊ መቧጨር {#ethical-scraping}

1. **robots.txt ያክብሩ** — የጣቢያ መመሪያዎችን ይከተሉ
2. **ይዘትን አትቅዱ** — ትንተናን ለመነሳሳት ብቻ ይጠቀሙ
3. **መብቶችን ያረጋግጡ** — የተወጣውን ይዘት መጠቀም እንደሚችሉ ያረጋግጡ
4. **ምንጮችን ይጥቀሱ** — የመነሳሳት ምንጮችን እውቅና ይስጡ
5. **ውሎችን ይከተሉ** — የጣቢያ የአገልግሎት ውሎችን ያክብሩ

### የመሳሪያ ውጤታማነትን ከፍ ማድረግ {#maximizing-tool-effectiveness}

1. **ተለይተው ይግለጹ** — የተወሰኑ የትንተና ዓይነቶችን ይጠይቁ
2. **አውድ ያቅርቡ** — የጣቢያዎን ዓላማ ያብራሩ
3. **የሚጠበቀውን ያስቀምጡ** — የንድፍ ግቦችዎን ይግለጹ
4. **ይድገሙ** — በውጤቶች መሠረት ያሻሽሉ
5. **መሳሪያዎችን ያጣምሩ** — ለሁሉንም ያካተተ ትንተና ብዙ መሳሪያዎችን ይጠቀሙ

## ችግር መፍታት {#troubleshooting}

### ጣቢያው አይቧጨርም {#site-wont-scrape}

- ጣቢያው ለሕዝብ ተደራሽ መሆኑን ያረጋግጡ
- robots.txt መቧጨርን እንደሚፈቅድ ያረጋግጡ
- ሌላ ጣቢያ ይሞክሩ
- የኢንተርኔት ግንኙነትን ያረጋግጡ
- ችግሩ ከቀጠለ ድጋፍን ያግኙ

### ያልተሟላ ትንተና {#incomplete-analysis}

- ጣቢያው ተለዋዋጭ ይዘት ሊኖረው ይችላል
- JavaScript ሙሉ በሙሉ ላይታይ ይችላል
- ትልቅ ጣቢያዎች ጊዜ ሊያልቅባቸው ይችላል
- በምትኩ የተወሰኑ ገጾችን መተንተን ይሞክሩ
- የተወሰኑ የትንተና ዓይነቶችን ይጠይቁ

### ዝግተኛ አፈጻጸም {#slow-performance}

- ትልቅ ጣቢያዎችን ለመተንተን የበለጠ ጊዜ ይወስዳል
- በአንድ ጊዜ ብዙ መቧጨሮች ከተካሄዱ ዝግተኛ ናቸው
- የአውታረ መረብ ግንኙነት ፍጥነትን ይነካል
- ከተጨናነቁ ሰዓታት ውጭ ይሞክሩ
- መጀመሪያ ትናንሽ ክፍሎችን ይተንትኑ
