# የደንበኛ ግብረመልስ እና ችግር ሪፖርት ማድረግ

Gratis AI Agent v1.5.0 በአጠቃቀም ላይ ያሉ ተጠቃሚዎች ያልተጠቅመው ምላሽ እንዲያሳውቁ እና ችግሮችን በቀጥታ ከቻት በይነገጽ እንዲያሳውቁ የሚያስችል የተገጠመለት የግብረመልስ እና የችግር ሪፖርት የማድረግ ሥርዓት አቅርቧል። ይህ ሥርዓት የፈቃድ አስተዳደርን (consent management)፣ በራስ-ሰር የሪፖርት አሰጣጥ ትዕዛዝን (automated reporting command)፣ እና በበስተጀርባ የ AI-የተረዳ የመመደብ ሂደትን (AI-assisted triage) ያካትታል።

## የሰዋሰዋዊ ምልክት (Thumbs-Down Button)

በ AI ረዳት የሚላከው እያንዳንዱ መልዕክት **የሰዋሰዋዊ ምልክት (👎)** ቁልፍ አለው። ተጠቃሚው ሲጫን፣ ምላሹ እንደ አልተጠቅመው ወይም እንደተሳሳተ ምልክት ማድረግ ይችላል።

- ቁልፉ በአሲስታንት መልዕክት አጠገብ ሲያልፍ ይታያል።
- ሲጫን ደግሞ **የግብረመልስ ፈቃድ ሞዳል (Feedback Consent Modal)** ይከፈታል።
- የተሰበሰበው ግብረመልስ ከውይይቱ ዙር እና ከተወሰነው መልዕክት ጋር ይያያዛል።

## የግብረመልስ ፈቃድ ሞዳል (Feedback Consent Modal)

ተጠቃሚው የሰዋሰዋዊ ምልክት ቁልፉን ሲጫን፣ ምንም መረጃ ከመላኩ በፊት የፈቃድ ሞዳል (consent modal) ይታያል። ይህ ሞዳል፡

- ምን ዓይነት መረጃ እንደሚሰበሰብ ያብራራል (የውይይት ክፍል፣ የብራውዘር ሁኔታ)።
- ተጠቃሚው ይህንን መረጃ ለማጋራት ፈቃድ እንደሚሰጥ እንዲያረጋግጥ ይጠይቃል።
- ተጠቃሚው ምን ስህተት እንደተፈጠረ ሊገልጽበት የሚችል የጽሑፍ መስክ ይሰጣል።
- ሳይላኩ ለማስወገድ **취소 (Cancel)** አማራጭ ይሰጣል።

ተጠቃሚው በግልጽ እስኪፈቅድ ድረስ ምንም ግብረመልስ አይመዘገብም።

## በራስ-ሰር የሚላክ የግብረመልስ ባነር (Auto-Prompt Feedback Banner)

ውይይቱ ሲጠናቀቅ፣ ረዳቱ **በራስ-ሰር የሚላክ የግብረመልስ ባነር** ሊያሳይ ይችላል — ይህም ክፍሉ ጠቃሚ እንደነበር የሚጠይቅ የማይረብሽ መልዕክት ነው።

ይህ ባነር በውይይቱ ርዝመት እና በውጤት መመዘኛዎች (heuristics) ላይ ተመስርቶ በራስ-ሰር ይታያል። ከሰዋሰዋዊ ምልክት ቁልፍ ጋር ተመሳሳይ የግብረመልስ ፍሰት (feedback flow) ጋር ይገናኛል። ተጠቃሚዎች ግብረመልስ ሳይሰጡ ባነሩን ማስወገድ ይችላሉ።

## /report-issue ስላሽ ትዕዛዝ (Slash Command)

ተጠቃሚዎች በቻት ግብዓት ውስጥ `/report-issue` ብለው በመተየብ የግብረመልስ ፍሰቱን በቀጥታ ማስጀመር ይችላሉ። ይህ ትዕዛዝ፡

- ወዲያውኑ የግብረመልስ ፈቃድ ሞዳልን ይከፍታል።
- የጽሑፍ መግለጫ መስኩን በአሁኑ ውይይት ሁኔታ መረጃ ይሞላል።
- ከመላኩ በፊት ተጨማሪ ዝርዝሮችን እንዲጨምር ያስችላል።

የ`/report-issue` ትዕዛዝ በሁሉም የቻት ሁነታዎች (inline, floating widget, full-screen) ይገኛል።

## በ AI የተረዳ የመመደብ ሂደት (AI-Assisted Triage)

የተሰበሰበው ግብረመልስ ወደ AI የመመደብ ደረጃ (triage layer) ይላካል፣ ይህም፡

- ሪፖርቱን ይመድባል (የእውነታ ስህተት፣ አልተጠቅመው መልስ፣ ተገቢ ያልሆነ ይዘት፣ ወዘተ)።
- ተዛማጅ የውይይት መረጃ (metadata) ይያያዛል።
- የመመደብ ማጠቃለያውን ወደ የተዋቀረው **የግብረመልስ መድረሻ (Feedback Endpoint)** ይልካል። (በ[Settings > Advanced](#settings-advanced) ይመልከቱ)

ይህም በጣም ወሳኝ የሆኑ ችግሮችን በመጀመሪያ በማቅረብ በእጅ የሚደረገውን የপর্যালোচনা ጊዜ ይቀንሳል።

## Settings > Advanced {#settings-advanced}

የግብረመልስ የጀርባ ሥርዓት (feedback backend)ን ለማንቃት፣ በ**Gratis AI Agent → Settings → Advanced** ስር የሚከተሉትን መስኮች ያዋቅሩ፦

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | የግብረመልስ ጭነት (payloads) በ POST ጥያቄዎች የሚቀበለው URL። |
| **Feedback API Key** | የግብረመልስ ማስረከቢያዎችን ለማረጋገጥ በ`Authorization` header ውስጥ የሚላከ Bearer token። |

ሁለቱንም መስኮች ባዶ በመተው የግብረመልስ ስብስብን ማጥፋት ይችላሉ። የግል የግብረመልስ ቁልፎች ለተጠቃሚዎች ይታያሉ፣ ነገር ግን ማስረከቢያዎች ወደ ፊት አይላኩም።

:::tip
የግብረመልስ መድረሻው (feedback endpoint) ቢያንስ `message_id`፣ `conversation_id`፣ `feedback_text` እና `triage_category` መስኮችን ባካተተ JSON body መቀበል አለበት። የሚጠበቀውን ስኪማ (schema) የendpoint ሰጪዎ ሰነድ ይመልከቱ።
:::
