# የክፍያ መውጫ መስክ እና የደንበኛ DNS አስተዳደር

## የዶሜይን ምርጫ የክፍያ መውጫ መስክ

የ**ዶሜይን ምርጫ** መስክ ደንበኞች የጣቢያቸውን ዶሜይን እንዴት እንደሚያገኙ ምርጫ የሚሰጥ የክፍያ መውጫ ንጥረ ነገር ነው። ዶሜይን መሸጥን ለማንቃት ወደ ማንኛውም የክፍያ መውጫ ቅጽ ያክሉት።

### መስኩን ወደ ክፍያ መውጫ ቅጽ ማከል

1. ወደ **የኔትወርክ አስተዳዳሪ › Ultimate Multisite › የክፍያ መውጫ ቅጾች** ይሂዱ
2. የክፍያ መውጫ ቅጽ ይክፈቱ ወይም ይፍጠሩ
3. በክፍያ መውጫ አርታዒው ውስጥ **መስክ አክል** የሚለውን ጠቅ ያድርጉ
4. ከመስኮች ዝርዝር **የዶሜይን ምርጫ** ይምረጡ
5. የመስኩን አማራጮች ያዋቅሩ (ከታች ይመልከቱ)
6. ቅጹን ያስቀምጡ

### የመስክ አማራጮች

**የዶሜይን ሁነቶች** — ደንበኛው የሚያያቸውን ትሮች ይምረጡ። እያንዳንዱ ሁነት በተናጠል ሊነቃ ወይም ሊሰናከል ይችላል፦

| ሁነት | ምን ያደርጋል |
|---|---|
| **ንዑስ ዶሜይን** | ደንበኛው በኔትወርክዎ ላይ ነፃ ንዑስ ዶሜይን ይጠቀማል (ለምሳሌ፣ `mysite.yournetwork.com`)። ክፍያ አያስፈልግም። |
| **አዲስ ዶሜይን መመዝገብ** | ደንበኛው አዲስ ዶሜይን ይፈልጋል እና ባዋቀሩት አቅራቢ በኩል ይመዘግበዋል። ለዋጋ አወጣጥ ተዛማጅ የዶሜይን ምርትን ይጠቀማል። |
| **ነባር ዶሜይን** | ደንበኛው ቀድሞ ያለውን ዶሜይን ያዛምዳል። የምዝገባ ክፍያ የለም። ዶሜይኑ በራስ-ሰር ከጣቢያቸው ጋር ይዛመዳል። |

**ነባሪ ሁነት** — ሦስቱም ሁነቶች ሲነቁ፣ የትኛው ትር መጀመሪያ እንደሚከፈት። የዶሜይን ምዝገባን አማራጭ ለማድረግ ወደ **ንዑስ ዶሜይን** ያቀናብሩ፣ ወይም ግዢዎችን ለማበረታታት ወደ **አዲስ ዶሜይን መመዝገብ** ያቀናብሩ።

**የዶሜይን ምርት** — እንደ አማራጭ ይህን መስክ ከተወሰነ የዶሜይን ምርት ጋር ያያይዙ። ካልተቀናበረ፣ ተጨማሪው ደንበኛው በሚፈልገው TLD መሠረት ተዛማጅ ምርቱን በራስ-ሰር ይመርጣል።

### የተመዝጋቢ የመገኛ መስኮች

ደንበኛ የ**አዲስ ዶሜይን መመዝገብ** ትርን ሲመርጥ፣ የክፍያ መውጫ ቅጹ የተመዝጋቢ የመገኛ መስኮችን በቅጹ ውስጥ ያክላል፦

- ስም / የአባት ስም
- የኢሜይል አድራሻ
- አድራሻ (መስመር 1፣ ከተማ፣ ክልል/ግዛት፣ የፖስታ ኮድ፣ አገር)
- የስልክ ቁጥር

እነዚህ በሁሉም የዶሜይን መዝጋቢዎች ይፈለጋሉ እና የምዝገባ API ጥሪ ከመደረጉ በፊት ይረጋገጣሉ። የስልክ ቁጥሮች በራስ-ሰር የዶሜይን መዝጋቢዎች ወደሚጠብቁት `+CC.NNN` ዓለም አቀፍ ቅርጸት ይቀረጻሉ።

### በራስ-ሰር የሚፈጠር የጣቢያ URL

ደንበኛ ዶሜይን ሲመዘግብ ወይም ሲያዛምድ፣ የጣቢያ URL መስክ ከተመረጠው ዶሜይን በራስ-ሰር ይሞላል። ደንበኞች የተለየ URL መስክ መሙላት አያስፈልጋቸውም።

### የፍለጋ ባህሪ

- ደንበኛው ሲተይብ የዶሜይን መገኘት በAJAX በቅጽበት ይፈተሻል
- የተመረጠው ዶሜይን ካልተገኘ አማራጭ TLD ጥቆማዎች ይታያሉ
- ዋጋ አወጣጥ በቀጥታ ይመጣል እና በግልጽ ይታያል (የምዝገባ ዋጋ፣ የእድሳት ዋጋ፣ አማራጭ የWHOIS ግላዊነት ክፍያ)
- የኩፖን ኮዶች ለዶሜይን ምርቶች እንደ ሌሎች ምርቶች ሁሉ ይተገበራሉ

**የፍለጋ ምላሽ ፍጥነትን ማስተካከል፦**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ወደ ዶሜይን ፍለጋ ቅጽ ብጁ መስኮችን ማከል፦**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## የደንበኛ DNS አስተዳደር

ደንበኞች ለተመዘገቡ ዶሜይኖቻቸው የDNS መዝገቦችን ከ**My Account** ገጽ ላይ፣ በዶሜይናቸው ግቤት ስር ማስተዳደር ይችላሉ።

### የሚደገፉ የመዝገብ ዓይነቶች

| ዓይነት | ጥቅም |
|---|---|
| **A** | የአስተናጋጅ ስምን ከIPv4 አድራሻ ጋር ያዛምዱ |
| **AAAA** | የአስተናጋጅ ስምን ከIPv6 አድራሻ ጋር ያዛምዱ |
| **CNAME** | ወደ ሌላ የአስተናጋጅ ስም የሚያመለክት ተለዋጭ ስም ይፍጠሩ |
| **MX** | የመልዕክት ልውውጥ አገልጋይ ያቀናብሩ |
| **TXT** | SPF፣ DMARC፣ ማረጋገጫ፣ ወይም ሌሎች የጽሑፍ መዝገቦችን ያክሉ |

### የትኞቹ አቅራቢዎች የDNS አስተዳደርን ይደግፋሉ?

የDNS አስተዳደር (መዝገቦችን ማከል፣ ማርትዕ፣ መሰረዝ) በ**OpenSRS**፣ **ResellerClub**፣ **Enom**፣ **HostAfrica** እና **Openprovider** ጋር ይገኛል። የ**Hostinger** ዶሜይኖች በDomain Seller በኩል የስም አገልጋዮችን ማዘመን ይችላሉ፤ ለተስተናገዱ ዶሜይኖች የDNS መዝገቦች በዋናው Hostinger የዶሜይን-ማዛመጃ ውህደት ይተዳደራሉ። የNamecheap፣ GoDaddy እና NameSilo ዶሜይኖች ሁኔታን እና የማብቂያ መረጃን ያሳያሉ፣ ነገር ግን DNS በቀጥታ በዶሜይን መዝጋቢው የመቆጣጠሪያ ፓነል ውስጥ መተዳደር አለበት።

### ነባሪ የDNS መዝገቦች

ዶሜይን ሲመዘገብ በራስ-ሰር የሚተገበሩ ነባሪ የDNS መዝገቦችን ማዋቀር ይችላሉ። ወደ **ቅንብሮች › Domain Seller › ነባሪ የDNS መዝገቦች** ይሂዱ።

ነባሪ የመዝገብ እሴቶች ሁለት ቶከኖችን ይደግፋሉ፦

| ቶከን | በምን ይተካል |
|---|---|
| `{DOMAIN}` | የተመዘገበው የዶሜይን ስም (ለምሳሌ፣ `example.com`) |
| `{SITE_URL}` | ለደንበኛው ጣቢያ የWordPress ጣቢያ URL |

**ምሳሌ — ዋናውን ዶሜይን እና wwwን ወደ አገልጋይዎ IP ያመልክቱ፦**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### አስተዳዳሪ፦ DNSን መመልከት እና ማርትዕ

የኔትወርክ አስተዳዳሪዎች ለማንኛውም የደንበኛ ዶሜይን የDNS መዝገቦችን ከዶሜይኑ የማርትያ ገጽ ላይ በ**የኔትወርክ አስተዳዳሪ › Ultimate Multisite › ዶሜይኖች** ውስጥ ማየት እና ማርትዕ ይችላሉ።
