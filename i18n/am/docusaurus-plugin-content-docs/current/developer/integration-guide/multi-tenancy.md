# Multi-Tenancy ውህደት

Ultimate Multisite: Multi-Tenancy 1.2.0 ለሉዓላዊ ተከራዮች፣ ለማዘዋወር ማረጋገጫ፣ እና ለተከራይ የሕይወት-ዑደት አውቶሜሽን በርካታ የውህደት መነካካት ነጥቦችን ይቀይራል።

## የተከራይ ማስጀመሪያ ፍሰት {#tenant-bootstrap-flow}

ተከራዮችን የሚፈጥሩ ወይም የሚያሻሽሉ ውህደቶች ይህን ቅደም ተከተል መከተል አለባቸው፦

1. የተከራይ መዝገብ መረጃን እና የመለያየት ሞዴሉን ይፍቱ።
2. የተከራይ የውሂብ ጎታ ጸሐፊን ይፍጠሩ ወይም ያረጋግጡ።
3. የተከራይ ንድፍን ያስጀምሩ።
4. የተከራይ ተጠቃሚዎችን ያቅርቡ።
5. የተከራይ ማዘዋወሪያን እና የፋይል ስርዓት መንገዶችን ይመዝግቡ።
6. ተከራዩን ከማጋለጥዎ በፊት የማዘዋወር ማረጋገጫን ያስኪዱ።

ሉዓላዊ ተከራይ የአውታረ መረብ የውሂብ ጎታ ግንኙነትን እንደገና መጠቀም ይችላል ብለው አይገምቱ። በተጨማሪው የቀረቡትን የተከራይ መዝገብ እና የጸሐፊ አብስትራክሽኖች ይጠቀሙ።

## SSO እና REST ማገናኛዎች {#sso-and-rest-hooks}

ሁኔታ-አልባ የተከራይ ራስ-መግቢያ አጭር-ዕድሜ ያላቸውን ቶከኖች ከዓላማ ክሌም፣ JTI የድጋሚ ማጫወት ጥበቃ፣ የማብቂያ ገደብ፣ እና የመነሻ መቆለፊያ ጋር ይጠቀማል። የመግቢያ አዝራሮችን ወይም የርቀት አስተዳደር አገናኞችን የሚጨምሩ ውህደቶች የተከራይ የመግቢያ URL-ዎችን በቀጥታ ከመገንባት ይልቅ በተደገፈው SSO ፍሰት በኩል የተከራይ ጉብኝቶችን ማመንጨት አለባቸው።

የአውታረ መረብ-ወገን API የኦዲት ክስተቶች እና ዕለታዊ ማጠቃለያዎች ለሉዓላዊ ተከራይ መግቢያ በሮች ይገኛሉ። የተከራይ የሕይወት-ዑደት endpoint-ዎችን የሚጠሩ ውጫዊ ስርዓቶችን ሲያርሙ እነዚያን መዝገቦች ይጠቀሙ።

## የሉዓላዊ ደንበኛ እርምጃ URL-ዎች {#sovereign-customer-action-urls}

Ultimate Multisite v2.13.0 የሉዓላዊ-ተከራይ ደንበኛ እርምጃዎችን ለAccount፣ ለcheckout፣ ለbilling፣ ለinvoice፣ ለsite፣ ለtemplate-switching፣ እና ለdomain-mapping ፍሰቶች ወደ ዋናው ጣቢያ ይመልሳል። በተከራይ-ወገን የአስተዳደር አገናኞችን የሚያቀርቡ ውህደቶች እነዚያን እርምጃዎች ወደ ዋና-ጣቢያ የደንበኛ ፓነል መጠቆም አለባቸው፣ እና ተጠቃሚው እርምጃውን ከጨረሰ በኋላ ወደ ተከራዩ መመለስ መቻል ሲኖርበት የተረጋገጠ የመመለሻ ዒላማ ማካተት አለባቸው።

ለክሮስ-ዶሜይን የአስተዳደር አገናኞች ዋናውን SSO መጠቅለያ ይጠቀሙ፦

```php
$url = wu_with_sso($main_site_customer_url);
```

የተመነጨው URL አሁንም በ`wu_sso_url` በኩል ሊጣራ ይችላል፤ ይህም SSO URL-ን፣ የአሁኑን ተጠቃሚ፣ የዒላማ ጣቢያ ID-ን፣ እና የredirect አውድን ይቀበላል። ተጨማሪዎች የአቅራቢ-ልዩ አውድ ለመጨመር ወይም የUltimate Multisite ቶከን ማረጋገጫን በመጠበቅ የbroker URL-ን ለመተካት ያንን ማጣሪያ መጠቀም ይችላሉ።

membership፣ invoice፣ የbilling-address፣ template፣ ወይም የdomain-management ሁኔታን በሉዓላዊ ተከራይ ውስጥ አታባዙ። የተከራይ Dashboard-ን እንደ ማስጀመሪያ፣ የዋና-ጣቢያ ደንበኛ ፓነልን ደግሞ ለተቀናበሩ እርምጃዎች የመዝገብ ስርዓት አድርገው ይቁጠሩ።

## የማዘዋወር ማረጋገጫ {#migration-verification}

አንድ ማዘዋወር ወይም የሕይወት-ዑደት ውህደት የተከራይ ውሂብን ከቀየረ በኋላ፣ የማረጋገጫ በሮቹን ያስኪዱ፦

- `wp tenant verify-no-legacy --site=<site-id>` ተከራዩ ከእንግዲህ በቅርስ የአውታረ መረብ-ወገን መንገዶች ላይ እንደማይመሰረት ያረጋግጣል።
- `wp tenant verify-sovereign-push --site=<site-id>` ሉዓላዊ የግፊት ሥራዎች ማቀናበር እና ማፍሰስ እንደሚችሉ ያረጋግጣል።

ውህደቶች ያልተሳካ ማረጋገጫን እንደ የስምሪት አጋጅ መቁጠር አለባቸው፣ እና ውድቀቱ እስኪፈታ ድረስ ተከራይን ቀጥታ እንደሆነ ከማመልከት መቆጠብ አለባቸው።

## የተከራይ ስረዛ {#tenant-deletion}

የስረዛ ፍሰቶች የተከራይ የውሂብ ጎታ ምስክርነቶች እንዲጸዱ የተጨማሪውን የteardown መንገድ መጥራት አለባቸው። ውጫዊ ውህደቶች teardown ከተሳካ በኋላ የአቅራቢ ሀብቶችን ሊያስወግዱ ይችላሉ፣ ግን ማረጋገጫ ወይም async የግፊት ሥራዎች እየሄዱ እያሉ የhost ውሂብ ጎታዎችን ወይም አቃፊዎችን መሰረዝ የለባቸውም።

## ጊዜው ያለፈበት የውሂብ ጎታ ራውተር {#deprecated-database-router}

የቅርስ `Database_Router` በdeprecation stub ተተክቷል። አዲስ ውህደቶች በአሮጌው የራውተር ክፍል ላይ ከመመሰረት ይልቅ ተከራዮችን በአሁኑ የጣቢያ ራውተር እና የተከራይ መዝገብ API-ዎች በኩል መፍታት አለባቸው።
