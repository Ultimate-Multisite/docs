# WooCommerce সমন্বয়

WooCommerce সমন্বয় আমাদের Manual Gateway যেভাবে কাজ করে, তার খুব কাছাকাছি পদ্ধতিতে কাজ করে। এটি আপনার ব্যবহারকারীদের অনেক নমনীয়তা দেয়, যাতে তারা আপনার WooCommerce ইনস্টলে থাকা যেকোনো পেমেন্ট ফর্ম বেছে নিতে পারে। আর এটি সহজ: ব্যবহারকারীর বিলিং মেয়াদ শেষ হলে, ব্যবহারকারীর সাবস্ক্রিপশন স্থগিত রাখা হয়। একই সময়ে, নেটওয়ার্কের মূল সাইটের WooCommerce ইনস্টলে একটি অর্ডার তৈরি হবে। এরপর ব্যবহারকারী ইমেইলে একটি ইনভয়েস পায়, যেখানে পেমেন্ট করার লিংক থাকে। পেমেন্ট পাওয়া গেলে সাবস্ক্রিপশন নবায়ন করা হয়।

## বৈশিষ্ট্য

- Ultimate Multisite এবং WooCommerce-এর মধ্যে সহজ সমন্বয়
- একাধিক পেমেন্ট পদ্ধতি উপলভ্য
- অন্তর্ভুক্ত সাবস্ক্রিপশন কোর ব্যবহার করে স্বয়ংক্রিয় পুনরাবৃত্ত বিলিং, ঐচ্ছিক WooCommerce Subscriptions plugin উন্নতিসহ
- Checkout মোটে Ultimate Multisite করগুলো আলাদা, করমুক্ত WooCommerce ফি লাইন হিসেবে থাকে, যাতে গ্রাহকেরা পেমেন্টের আগে করের পরিমাণ পর্যালোচনা করতে পারেন

## ডকুমেন্টেশন

**WooCommerce সমন্বয় সেট আপ করা**

**Ultimate Multisite দিয়ে আপনার পেমেন্ট বিকল্প বাড়ানো: WooCommerce সমন্বয়**

## ইনস্টলেশন

1. 'ultimate-multisite-woocommerce' '/wp-content/plugins/' ডিরেক্টরিতে আপলোড করুন
2. WordPress-এ 'Plugins' মেনু থেকে plugin সক্রিয় করুন
3. ধাপে ধাপে Wizard অনুসরণ করে plugin সেট আপ করুন

## ব্যবহার নির্দেশিকা

_**গুরুত্বপূর্ণ নোট: এই নিবন্ধটি Ultimate Multisite সংস্করণ 2.x-এর জন্য।**_

_**সতর্কতা:** Ultimate Multisite: WooCommerce Integration ব্যবহার করতে হলে অন্তত আপনার মূল সাইটে WooCommerce সক্রিয় থাকতে হবে।_

আমরা বুঝি যে কিছু দেশে _Stripe_ এবং _PayPal_ উপলভ্য নয়, যা Ultimate Multisite ব্যবহারকারীদের আমাদের plugin কার্যকরভাবে ব্যবহার করতে সীমিত বা বাধাগ্রস্ত করে। তাই আমরা **_WooCommerce_ সমন্বয়ের জন্য একটি add-on তৈরি করেছি**।

_WooCommerce_ একটি খুব জনপ্রিয় ই-কমার্স plugin। বিশ্বজুড়ে ডেভেলপাররা এতে বিভিন্ন পেমেন্ট গেটওয়ে যুক্ত করার জন্য add-on তৈরি করেছেন। Ultimate Multisite বিলিং সিস্টেমে আপনি যে পেমেন্ট গেটওয়েগুলো ব্যবহার করতে পারেন, তা বাড়াতে আমরা এর সুবিধা নিয়েছি।

## এটি সেট আপ করা

প্রথমে, আপনাকে WooCommerce Integration add-on ইনস্টল করতে হবে, যা **লাইসেন্সধারী ব্যবহারকারীদের জন্য বিনামূল্যে**। এটি ইনস্টল করতে, আপনার super admin dashboard-এ Ultimate Multisite-এ ক্লিক করুন এবং **Add-ons** নির্বাচন করুন।

![Add-ons পৃষ্ঠা](/img/admin/addons-page.png)

এখানে আপনি সব Ultimate Multisite add-on খুঁজে পাবেন। **Ultimate Multisite: WooCommerce Integration** add-on-এ ক্লিক করুন।

![WooCommerce Integration দেখানো Add-ons পৃষ্ঠা](/img/admin/addons-page.png)

add-on-এর বিস্তারিতসহ একটি উইন্ডো পপ আপ হবে। শুধু **Install Now** *.*-এ ক্লিক করুন

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

ইনস্টলেশন শেষ হলে, আপনাকে plugins পৃষ্ঠায় নিয়ে যাওয়া হবে। এখানে শুধু **Network Activate**-এ ক্লিক করুন এবং _WooCommerce_ add-on আপনার নেটওয়ার্কে সক্রিয় হবে।

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

এটি সক্রিয় করার পরও যদি আপনার ওয়েবসাইটে WooCommerce plugin ইনস্টল ও সক্রিয় না থাকে, **আপনি একটি স্মারক বার্তা পাবেন**।

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

ইনস্টল হয়ে গেলে, Ultimate Multisite পেমেন্ট সেটিংসের অধীনে একটি অতিরিক্ত বিকল্প দেখবেন। এরপর সেখান থেকে _WooCommerce_ বিকল্পটি **সক্রিয়** করতে পারবেন।

![WooCommerce সক্রিয় থাকা Ultimate Multisite পেমেন্ট গেটওয়ে সেটিংস](/img/addons/woocommerce-payment-gateway.png)

আপনি কিছু মৌলিক সেটিংস কনফিগার করতে পারেন, যেমন সামনে দেখানো প্রদর্শন নাম এবং checkout প্রবাহে আপনার ব্যবহারকারীদের নির্দেশনা দেওয়ার জন্য নির্দেশাবলি। এরপর আপনি ব্যবহার করতে চান এমন পেমেন্ট গেটওয়ে সক্রিয় ও কনফিগার করতে আপনার **মূল সাইটের _WooCommerce_ সেটিংসে** যেতে পারেন।

### Checkout কর ফি লাইন

যখন কোনো Ultimate Multisite checkout-এ কর থাকে, সমন্বয়টি প্রতিটি Ultimate Multisite করের পরিমাণ WooCommerce checkout-এ আলাদা, করমুক্ত ফি লাইন হিসেবে যোগ করে। এতে পেমেন্টের আগে করের মোট পরিমাণ দৃশ্যমান থাকে এবং WooCommerce যাতে আবার কর ফিতে কর না বসায় তা নিশ্চিত হয়।

## পেমেন্ট কীভাবে পরিচালনা করবেন

WooCommerce Integration আমাদের **Manual Gateway** যেভাবে কাজ করে, তার খুব কাছাকাছি পদ্ধতিতে কাজ করে। অসুবিধা হলো, প্রতিটি পেমেন্ট বিলিং মেয়াদের শেষে ম্যানুয়ালি পরিশোধ করতে হয়। প্রবাহটি এভাবে কাজ করে:

  * ব্যবহারকারীর বিলিং চক্র শেষ হলে, membership **স্থগিত** রাখা হয়। একই সময়ে, আপনার নেটওয়ার্কের মূল সাইটের _WooCommerce_ ইনস্টলে একটি অর্ডার তৈরি হবে।

  * এরপর ব্যবহারকারীরা ইমেইলে একটি ইনভয়েস পায়, যেখানে পেমেন্ট করার লিংক থাকে (লিংকটি তাদের subsite dashboard-এর বিলিং ইতিহাসেও দেখা যায়)। সেই লিংক ব্যবহারকারীকে _**WooCommerce**_ **checkout**-এ নিয়ে যায়, যেখানে ব্যবহারকারীরা পেমেন্ট প্রক্রিয়া করার জন্য **উপলভ্য _WooCommerce_ গেটওয়েগুলোর** একটি নির্বাচন করতে পারবেন।

  * পেমেন্ট পাওয়া গেলে, membership **নবায়ন** করা হয়, পেমেন্টটি Ultimate Multisite-এ লগ হয় এবং অর্ডারটি সম্পন্ন হিসেবে সেট করা হয়।

### কিছু গুরুত্বপূর্ণ নোট:

  * Ultimate Multisite-এর ডিফল্ট কার্যকারিতা এই সমন্বয় পদ্ধতির সঙ্গেও কাজ করে। উদাহরণস্বরূপ, **Subscription Management** স্ক্রিন থেকে **রিফান্ড ইস্যু** করলে নির্বাচিত পেমেন্ট ফর্ম ব্যবহার করে রিফান্ড ইস্যু করতে WooCommerce-এর সঙ্গে যোগাযোগ করা হবে।

  * Ultimate Multisite: WooCommerce Integration কাজ করার জন্য WP-Cron-এর ওপর নির্ভর করে। বেশিরভাগ সময় এটি খুব ভালো কাজ করে, কিন্তু সাইটে ট্রাফিক কম হলে নির্ধারিত ইভেন্ট পরীক্ষা করার জন্য cron চালু হয় না এবং ইভেন্টটি মিস হয়, যার ফলে কার্যকর হতে বিলম্ব হতে পারে।

## WooCommerce Subscriptions Plugin

প্রতি মাসে হাতে করে পেমেন্ট শুরু করা শেষ ব্যবহারকারীদের জন্য আদর্শ নয়, কারণ সাবস্ক্রিপশনগুলো প্রতি বিলিং চক্রে স্বয়ংক্রিয়ভাবে পুনরাবৃত্ত হওয়ার কথা। WooCommerce Integration add-on এখন মূল সাবস্ক্রিপশন কোডসহ আসে, তাই পুনরাবৃত্ত পণ্যগুলো পুনরাবৃত্ত বিলিংয়ের জন্য সবসময় সাবস্ক্রিপশন ব্যবহার করে।

এই integration-এর মাধ্যমে পুনরাবৃত্ত পণ্য বিক্রি করতে আলাদা [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) কেনা বা ইনস্টল করার **প্রয়োজন নেই**। যদি ওই plugin কেনা, ইনস্টল করা এবং আপনার প্রধান সাইটে উপলভ্য থাকে, Ultimate Multisite সেটি ব্যবহার করবে এবং WooCommerce অতিরিক্ত সাবস্ক্রিপশন ব্যবস্থাপনা ফিচার দেখাবে। এটি উপলভ্য না থাকলে, সঙ্গে থাকা সাবস্ক্রিপশন core তবুও পুনরাবৃত্ত পণ্যগুলো পরিচালনা করবে।

![WooCommerce সাবস্ক্রিপশন integration সেটিংস](/img/addons/woocommerce-subscription-integration.png)

এতটুকুই। পুনরাবৃত্ত পণ্যের জন্য আপনার শেষ ব্যবহারকারীদের আর তাদের বিলিং চক্রের শেষে হাতে করে পেমেন্ট করতে হবে না।

আপনার প্রধান সাইটে _WooCommerce_-এর অধীনে একটি অর্ডার তৈরি হওয়ার পাশাপাশি, প্রতিটি Account-এর জন্য একটি সাবস্ক্রিপশনও তৈরি হবে।

### ব্যর্থ নবায়নে স্থগিত করার আচরণ

ডিফল্টভাবে, WooCommerce Subscriptions ব্যর্থ-পেমেন্ট পুনরায় চেষ্টা করার সময়সূচি প্রক্রিয়া করার সময় সদস্যতাগুলো সক্রিয় থাকে। এতে স্বাভাবিক WooCommerce Subscriptions গ্রেস উইন্ডো বজায় থাকে, যাতে কোনো নবায়ন পুনরুদ্ধার হলে সাবস্ক্রিপশন ও সদস্যতা আগেভাগে অ্যাক্সেস ব্যাহত না করেই আবার সিঙ্কে ফিরতে পারে।

যেসব অ্যাডমিনিস্ট্রেটর আরও কঠোর অ্যাক্সেস নিয়ন্ত্রণ চান, তারা WooCommerce Integration সেটিংসে **ব্যর্থ নবায়নে সঙ্গে সঙ্গে সদস্যতা স্থগিত করুন** চালু করতে পারেন। এই opt-in সেটিং চালু থাকলে, নবায়ন পেমেন্ট ব্যর্থ হওয়ার সঙ্গে সঙ্গে Ultimate Multisite সংশ্লিষ্ট সদস্যতা স্থগিত করে; WooCommerce Subscriptions-এর পুনরায় চেষ্টা প্রক্রিয়া শেষ হওয়া পর্যন্ত অপেক্ষা করে না।

ডেভেলপাররা একই সিদ্ধান্ত `wu_woo_suspend_on_payment_failure` ফিল্টার দিয়ে override করতে পারেন। Hook reference পৃষ্ঠাগুলো add-on সোর্স থেকে তৈরি হয়, তাই এই overview-এ custom স্থগিত নীতির জন্য শুধু filter নামটি উল্লেখ করা হয়েছে।
