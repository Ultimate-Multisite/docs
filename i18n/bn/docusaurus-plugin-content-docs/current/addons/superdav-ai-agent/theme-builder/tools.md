# টুল ও কনফিগারেশন {#tools-and-configuration}

Theme Builder আপনার WordPress সাইট বিশ্লেষণ, ডিজাইন এবং তৈরি করতে একগুচ্ছ টুল ব্যবহার করে। সংস্করণ 1.16.0-এ, **sd-ai-agent/site-scrape এখন একটি Tier 1 টুল**, তাই এটি Theme Builder-এ ডিফল্টভাবেই পাওয়া যায়। Superdav AI Agent v1.18.0 বিশ্বস্ত ডেভেলপার কর্মপ্রবাহের জন্য আলাদাভাবে বিতরণ করা একটি Advanced সহায়ক প্যাকেজও চালু করেছে; এসব টুল WordPress.org কোর প্যাকেজের অংশ নয় এবং আলাদাভাবে ইনস্টল ও অনুমোদন করতে হবে।

## উপলভ্য টুল {#available-tools}

### Tier 1 টুল (সবসময় উপলভ্য) {#tier-1-tools-always-available}

Tier 1 টুল কোনো অতিরিক্ত কনফিগারেশন ছাড়াই Theme Builder-এ ডিফল্টভাবে পাওয়া যায়।

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**উদ্দেশ্য:** ডিজাইন অনুপ্রেরণা, কনটেন্ট কাঠামো এবং কার্যকারিতা বের করতে বিদ্যমান ওয়েবসাইট বিশ্লেষণ করা।

**সক্ষমতা:**

- **ওয়েবসাইট বিশ্লেষণ** — প্রতিদ্বন্দ্বী বা অনুপ্রেরণামূলক ওয়েবসাইট স্ক্র্যাপ ও বিশ্লেষণ করা
- **ডিজাইন আহরণ** — রং, ফন্ট এবং লেআউট প্যাটার্ন শনাক্ত করা
- **কনটেন্ট কাঠামো** — পেজের বিন্যাস ও স্তরবিন্যাস বোঝা
- **ফিচার শনাক্তকরণ** — plugin এবং কার্যকারিতা শনাক্ত করা
- **পারফরম্যান্স বিশ্লেষণ** — পেজের গতি ও অপ্টিমাইজেশন পরীক্ষা করা
- **SEO বিশ্লেষণ** — মেটা ট্যাগ ও স্ট্রাকচার্ড ডেটা পর্যালোচনা করা

**ব্যবহার:**

```
example.com-এর ডিজাইন বিশ্লেষণ করুন এবং আমার সাইটের অনুপ্রেরণা হিসেবে ব্যবহার করুন।
```

**এটি যা বের করে:**

- রঙের প্যালেট এবং রঙের ব্যবহার
- টাইপোগ্রাফি (ফন্ট ও আকার)
- লেআউট কাঠামো এবং ফাঁকফোকর
- নেভিগেশন প্যাটার্ন
- কনটেন্ট বিন্যাস
- ছবি ও মিডিয়া ব্যবহার
- ইন্টার‌্যাকটিভ উপাদান
- মোবাইল রেসপনসিভনেস

**সীমাবদ্ধতা:**

- পাসওয়ার্ড-সুরক্ষিত সাইট স্ক্র্যাপ করতে পারে না
- robots.txt এবং সাইট নীতিমালা মেনে চলে
- ডাইনামিক কনটেন্ট ধরতে নাও পারে
- JavaScript-নির্ভর সাইটের বিশ্লেষণ সীমিত হতে পারে
- বড় সাইট বিশ্লেষণে বেশি সময় লাগতে পারে

### Tier 2 টুল (ঐচ্ছিক) {#tier-2-tools-optional}

Theme Builder সেটিংসে স্পষ্টভাবে চালু করলে Tier 2 টুল পাওয়া যায়।

#### উন্নত অ্যানালিটিক্স {#advanced-analytics}

বিস্তারিত পারফরম্যান্স মেট্রিক দেয়:

- পেজ লোডের সময়
- Core Web Vitals
- SEO স্কোর
- অ্যাক্সেসিবিলিটি স্কোর
- সেরা অনুশীলনের স্কোর

#### কনটেন্ট অপ্টিমাইজার {#content-optimizer}

বিশ্লেষণ করে এবং উন্নতির পরামর্শ দেয়:

- পাঠযোগ্যতা
- SEO অপ্টিমাইজেশন
- কীওয়ার্ড ব্যবহার
- কনটেন্ট কাঠামো
- Call-to-action বসানো

### Tier 3 টুল (Premium) {#tier-3-tools-premium}

Tier 3 টুলের জন্য অতিরিক্ত কনফিগারেশন বা premium ফিচার প্রয়োজন।

#### AI কনটেন্ট জেনারেটর {#ai-content-generator}

এর জন্য কনটেন্ট তৈরি করে:

- Product বিবরণ
- সার্ভিস পেজ
- ব্লগ পোস্ট
- মেটা বিবরণ
- Call-to-action কপি

#### উন্নত ডিজাইন টুল {#advanced-design-tools}

উন্নত ডিজাইন সক্ষমতা দেয়:

- কাস্টম CSS তৈরি
- অ্যানিমেশন তৈরি
- ইন্টার‌্যাকটিভ উপাদান ডিজাইন
- উন্নত রঙতত্ত্ব
- টাইপোগ্রাফি অপ্টিমাইজেশন

### Advanced সহায়ক টুল (আলাদা প্যাকেজ) {#advanced-companion-tools-separate-package}

Advanced সহায়ক প্যাকেজটি মূল Superdav AI Agent রিলিজ থেকে আলাদাভাবে বিতরণ করা হয়। এটি বিশ্বস্ত ডেভেলপার ও সাইট-মালিক পরিবেশের জন্য তৈরি, যেখানে প্রশাসকেরা স্পষ্টভাবে agent-কে উচ্চ-প্রভাবের টুলিং ব্যবহার করতে অনুমতি দেন।

Advanced সহায়ক সক্ষমতার মধ্যে থাকতে পারে:

- **ফাইলসিস্টেম টুল** — বিশ্বস্ত পরিবেশে অনুমোদিত ফাইল পরীক্ষা ও পরিবর্তন করা
- **ডেটাবেস টুল** — স্পষ্টভাবে অনুমোদিত হলে সাইটের ডেটা query বা update করা
- **WP-CLI টুল** — WordPress রক্ষণাবেক্ষণ ও পরিদর্শন কমান্ড চালানো
- **REST dispatcher টুল** — নিয়ন্ত্রিত dispatch কর্মপ্রবাহের মাধ্যমে নিবন্ধিত REST এন্ডপয়েন্ট কল করা
- **Plugin builder টুল** — plugin code-এর কাঠামো তৈরি এবং ধাপে ধাপে উন্নয়ন করা
- **Git snapshot টুল** — ঝুঁকিপূর্ণ ডেভেলপার অপারেশনের আগে snapshot তৈরি করা
- **User-management টুল** — অনুমতি থাকলে ব্যবহারকারী রেকর্ড পর্যালোচনা বা সমন্বয়ে সহায়তা করা
- **Benchmark টুল** — ডেভেলপমেন্ট পর্যালোচনার জন্য পারফরম্যান্স বা সক্ষমতার benchmark চালানো

প্রতিটি ইনস্টলেশনের জন্য এগুলোকে সাধারণভাবে উপলভ্য Theme Builder টুল হিসেবে নথিভুক্ত করবেন না। এগুলো কেবল তখনই উপলভ্য, যখন Advanced সহায়ক প্যাকেজ ইনস্টল, চালু এবং বিশ্বস্ত প্রশাসকদের মধ্যে সীমিত করা থাকে।

## টুল কনফিগারেশন {#tool-configuration}

### টুল চালু করা {#enabling-tools}

Theme Builder-এ অতিরিক্ত টুল চালু করতে:

1. **Gratis AI Agent → Settings**-এ যান
2. **Theme Builder → Tools**-এ যান
3. প্রয়োজন অনুযায়ী টুল চালু/বন্ধ করুন
4. সেটিংস সংরক্ষণ করুন

### টুল অনুমতি {#tool-permissions}

কিছু টুলের জন্য অনুমতি প্রয়োজন:

- **সাইট স্ক্র্যাপিং** — ইন্টারনেট অ্যাক্সেস প্রয়োজন
- **অ্যানালিটিক্স** — Google Analytics ইন্টিগ্রেশন প্রয়োজন
- **কনটেন্ট তৈরি** — API key প্রয়োজন
- **উন্নত ফিচার** — premium subscription প্রয়োজন হতে পারে
- **Advanced সহায়ক টুল** — আলাদাভাবে বিতরণ করা Advanced প্যাকেজ এবং প্রশাসকের স্পষ্ট আস্থা প্রয়োজন

### API key এবং ক্রেডেনশিয়াল {#api-keys-and-credentials}

যেসব টুলের API key প্রয়োজন, সেগুলো কনফিগার করুন:

1. **Gratis AI Agent → Settings → API Keys**-এ যান
2. প্রতিটি টুলের জন্য ক্রেডেনশিয়াল লিখুন
3. সংযোগ পরীক্ষা করুন
4. নিরাপদভাবে সংরক্ষণ করুন

### Advanced সহায়ক প্যাকেজ ইনস্টল করা {#installing-the-advanced-companion-package}

Advanced সহায়ক প্যাকেজটি WordPress.org কোর ZIP থেকে আলাদাভাবে প্রকাশিত হয়। রিলিজের জন্য শুধুমাত্র অফিসিয়াল প্রকল্প বিতরণ চ্যানেল থেকে এটি ইনস্টল করুন, তারপর বিশ্বস্ত প্রশাসকদের মধ্যে অ্যাক্সেস সীমিত করুন। প্রোডাকশন সাইটে ফাইলসিস্টেম, ডেটাবেস, WP-CLI, REST dispatcher, plugin builder, user-management বা benchmark টুল চালু করার আগে মানব অনুমোদন গেট এবং git snapshot আচরণ পর্যালোচনা করুন।

## sd-ai-agent/site-scrape ব্যবহার করা {#using-sd-ai-agentsite-scrape}

### মৌলিক ব্যবহার {#basic-usage}

Theme Builder-কে একটি ওয়েবসাইট বিশ্লেষণ করতে বলুন:

```
competitor.com-এ থাকা আমার প্রতিদ্বন্দ্বীর সাইটের ডিজাইন বিশ্লেষণ করুন
এবং আমার সাইটের জন্য ডিজাইন উন্নতির পরামর্শ দিন।
```

### নির্দিষ্ট বিশ্লেষণ {#specific-analysis}

নির্দিষ্ট ধরনের বিশ্লেষণ অনুরোধ করুন:

```
example.com থেকে রঙের প্যালেট বের করুন এবং অনুপ্রেরণা হিসেবে ব্যবহার করুন।
```

```
example.com-এর নেভিগেশন কাঠামো বিশ্লেষণ করুন এবং একই ধরনের
বিন্যাস আমার সাইটে প্রয়োগ করুন।
```

```
example.com-এর মোবাইল রেসপনসিভনেস পরীক্ষা করুন এবং নিশ্চিত করুন আমার সাইট
সমানভাবে রেসপনসিভ।
```

### তুলনামূলক বিশ্লেষণ {#comparison-analysis}

একাধিক সাইট তুলনা করুন:

```
site1.com এবং site2.com-এর ডিজাইন তুলনা করুন এবং এমন একটি ডিজাইন তৈরি করুন
যা দুটির সেরা উপাদান একত্র করে।
```

## টুলের সীমাবদ্ধতা ও বিবেচনা {#tool-limitations-and-considerations}

### রেট সীমাবদ্ধতা {#rate-limiting}

- সার্ভারের অতিরিক্ত চাপ ঠেকাতে scraping-এ rate limit করা হয়
- একই site-এ একাধিক অনুরোধ throttled হতে পারে
- বড় site বিশ্লেষণ করতে বেশি সময় লাগতে পারে

### কনটেন্টের নির্ভুলতা {#content-accuracy}

- ডাইনামিক কনটেন্ট পুরোপুরি ধরা নাও পড়তে পারে
- JavaScript-rendered কনটেন্ট অসম্পূর্ণ হতে পারে
- রিয়েল-টাইম ডেটা প্রতিফলিত নাও হতে পারে
- কিছু কনটেন্ট paywall-এর পেছনে থাকতে পারে

### আইনি ও নৈতিক বিবেচনা {#legal-and-ethical-considerations}

- robots.txt এবং site নীতি মেনে চলুন
- পুনঃপ্রকাশের জন্য কপিরাইটযুক্ত কনটেন্ট scrape করবেন না
- বিশ্লেষণ অনুপ্রেরণার জন্য ব্যবহার করুন, কপি করার জন্য নয়
- বের করা কনটেন্ট ব্যবহারের অধিকার আপনার আছে কি না যাচাই করুন
- বিশ্লেষণ করা site-এর terms of service অনুসরণ করুন

### পারফরম্যান্সের প্রভাব {#performance-impact}

- বড় site scrape করতে সময় লাগতে পারে
- একসঙ্গে একাধিক scrape ধীর হতে পারে
- নেটওয়ার্ক সংযোগ গতি প্রভাবিত করে
- বড় datasets-এর জন্য বেশি processing প্রয়োজন হতে পারে

## সেরা চর্চা {#best-practices}

### Site বিশ্লেষণ ব্যবহার করা {#using-site-analysis}

1. **একাধিক site বিশ্লেষণ করুন** — কয়েকটি উৎস থেকে অনুপ্রেরণা সংগ্রহ করুন
2. **কাঠামোর দিকে মনোযোগ দিন** — layout এবং সংগঠনের pattern শিখুন
3. **রং বের করুন** — শুরু করার ভিত্তি হিসেবে color palette ব্যবহার করুন
4. **টাইপোগ্রাফি দেখুন** — আপনার পছন্দের font combination শনাক্ত করুন
5. **নেভিগেশন পর্যালোচনা করুন** — menu সংগঠন বুঝুন

### নৈতিক Scraping {#ethical-scraping}

1. **robots.txt সম্মান করুন** — site নির্দেশিকা অনুসরণ করুন
2. **কনটেন্ট কপি করবেন না** — বিশ্লেষণ শুধু অনুপ্রেরণার জন্য ব্যবহার করুন
3. **অধিকার যাচাই করুন** — বের করা কনটেন্ট ব্যবহার করতে পারবেন কি না নিশ্চিত করুন
4. **উৎসের কৃতিত্ব দিন** — অনুপ্রেরণার উৎস উল্লেখ করুন
5. **শর্ত মেনে চলুন** — site-এর terms of service অনুসরণ করুন

### Tool-এর কার্যকারিতা সর্বোচ্চ করা {#maximizing-tool-effectiveness}

1. **নির্দিষ্ট হোন** — নির্দিষ্ট ধরনের বিশ্লেষণ চান
2. **প্রসঙ্গ দিন** — আপনার site-এর উদ্দেশ্য ব্যাখ্যা করুন
3. **প্রত্যাশা নির্ধারণ করুন** — আপনার design লক্ষ্য বর্ণনা করুন
4. **পুনরাবৃত্তি করুন** — ফলাফলের ভিত্তিতে পরিমার্জন করুন
5. **Tool মিলিয়ে ব্যবহার করুন** — পূর্ণাঙ্গ বিশ্লেষণের জন্য একাধিক tool ব্যবহার করুন

## সমস্যা সমাধান {#troubleshooting}

### Site Scrape হচ্ছে না {#site-wont-scrape}

- siteটি সবার জন্য accessible কি না পরীক্ষা করুন
- robots.txt scraping অনুমতি দেয় কি না যাচাই করুন
- অন্য site চেষ্টা করুন
- ইন্টারনেট সংযোগ পরীক্ষা করুন
- সমস্যা চলতে থাকলে support-এর সঙ্গে যোগাযোগ করুন

### অসম্পূর্ণ বিশ্লেষণ {#incomplete-analysis}

- site-এ ডাইনামিক কনটেন্ট থাকতে পারে
- JavaScript পুরোপুরি rendered নাও হতে পারে
- বড় site timeout হতে পারে
- এর বদলে নির্দিষ্ট page বিশ্লেষণ করে দেখুন
- নির্দিষ্ট বিশ্লেষণের ধরন অনুরোধ করুন

### ধীর পারফরম্যান্স {#slow-performance}

- বড় site বিশ্লেষণ করতে বেশি সময় লাগে
- একসঙ্গে একাধিক scrape ধীর হয়
- নেটওয়ার্ক সংযোগ গতি প্রভাবিত করে
- off-peak সময়ে চেষ্টা করুন
- আগে ছোট section বিশ্লেষণ করুন
