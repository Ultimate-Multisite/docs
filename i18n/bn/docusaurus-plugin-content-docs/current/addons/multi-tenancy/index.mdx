# Multi-Tenancy {#multi-tenancy}

পারফরম্যান্স ও নিরাপত্তার জন্য সাবসাইটগুলো সার্ভার ও ডেটাবেসজুড়ে বিতরণ করুন

## Version 1.2.0 রিলিজের প্রধান বিষয় {#version-120-release-highlights}

Multi-Tenancy 1.2.0 প্রোডাকশন আইসোলেশন ওয়ার্কফ্লোর জন্য sovereign tenant ভিত্তি যোগ করে:

- স্বল্পস্থায়ী, রিপ্লে-সুরক্ষিত SSO ভিজিটসহ স্টেটলেস tenant autologin।
- sovereign tenants-এর জন্য প্রতি-সাবসাইট ডেটাবেস ও ফাইলসিস্টেম আইসোলেশন।
- host-সচেতন tenant provisioning-এর জন্য Hostinger multi-tenancy সক্ষমতা।
- legacy ডেটা পরীক্ষা ও async push প্রক্রিয়াকরণের জন্য sovereign migration যাচাইকরণ গেট।
- Bedrock, FrankenPHP, এবং localhost ইনস্টলের জন্য কনফিগারযোগ্য একই-মেশিন ডেটাবেস host binding।
- tenant bootstrap, ডেটাবেস writer, schema bootstrap, এবং user provisioning উন্নতি।
- আরও নিরাপদ sovereign teardown, যা tenant credentials পরিষ্কার করে।

[sovereign migration workflow](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy isolation](/user-guide/configuration/multi-tenancy-isolation), এবং [sovereign tenant operations](/user-guide/administration/sovereign-tenant-operations)-এর জন্য user guide দেখুন।

## প্রয়োজনীয়তা {#requirements}

- WordPress 5.3 বা তার বেশি
- PHP 7.4 বা তার বেশি
- Multisite Ultimate plugin (সক্রিয়)

## ইনস্টলেশন {#installation}

1. addon ফাইলগুলো আপনার `/wp-content/plugins/` ডিরেক্টরিতে আপলোড করুন
2. WordPress-এর 'Plugins' মেনু থেকে plugin সক্রিয় করুন
3. network admin-এ addon সেটিংস কনফিগার করুন

## কনফিগারেশন {#configuration}

addon বিভিন্ন কনফিগারেশন অপশন সমর্থন করে, যা network admin interface থেকে পাওয়া যায়।

### Sovereign tenant configuration {#sovereign-tenant-configuration}

sovereign tenants-এর জন্য, tenant-এ traffic routing করার আগে tenant ডেটাবেস, ফাইলসিস্টেম root, database host binding, এবং host capability কনফিগার করুন। ডেটাবেস credentials, host binding, routing, বা user provisioning-কে প্রভাবিত করে এমন প্রতিটি পরিবর্তনের পর migration verification চালান।

tenant runtime এবং ডেটাবেস grant যদি local socket বা একই-মেশিন access আশা করে, শুধু তখনই `localhost` ব্যবহার করুন। আপনার provider যে grant host কনফিগার করেছে সেটি হলে `127.0.0.1` বা service hostname ব্যবহার করুন।

### Hostinger capability {#hostinger-capability}

Hostinger multi-tenancy capability শেয়ার করা Hostinger integration token-এর সঙ্গে কাজ করে। যাচাইকরণের আগে নিশ্চিত করুন domain hPanel-এ আছে, DNS Hostinger account-এ point করছে, এবং tenant ডেটাবেস সেটিংস hPanel-এর সঙ্গে মিলে।

## Hook ও Filter {#hooks-and-filters}

### Action {#actions}

- `ultimate-multisite-addon-template_init` - addon initialization-এর পরে চালু হয়
- `ultimate-multisite-addon-template_loaded` - সব addon ফাইল লোড হওয়ার পরে চালু হয়

### Filter {#filters}

- `ultimate-multisite-addon-template_settings` - addon সেটিংস পরিবর্তন করুন
- `ultimate-multisite-addon-template_enabled` - addon enable/disable status override করুন
