# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency আপনার নেটওয়ার্ককে একাধিক মুদ্রায় পেমেন্ট গ্রহণ করতে সক্ষম করে। এটি স্বয়ংক্রিয়ভাবে লাইভ বা ম্যানুয়াল এক্সচেঞ্জ রেট ব্যবহার করে দাম রূপান্তর করে, আপনাকে যেকোনো মুদ্রার জন্য নির্দিষ্ট পণ্য-ভিত্তিক দাম সেট করতে দেয়, এবং জিওলোকেশন (geolocation) এর মাধ্যমে প্রতিটি ভিজিটরের পছন্দের মুদ্রা সনাক্ত করে — সব কিছু একটি একক অ্যাডমিন ড্যাশবোর্ড থেকে পরিচালনা করা যায়।

## মূল বৈশিষ্ট্য (Key Features) {#key-features}

- **স্বয়ংক্রিয় বিনিময় হার রূপান্তর (Automatic exchange rate conversion)** — আপনার মূল মুদ্রা (base currency) থেকে রেট ব্যবহার করে দাম রূপান্তর করা হয়।
- **পণ্য-ভিত্তিক মূল্য ওভাররাইড (Per-product pricing overrides)** — স্বয়ংক্রিয় রূপান্তর এড়িয়ে যেকোনো পণ্যের জন্য নির্দিষ্ট মুদ্রা-ভিত্তিক দাম সেট করুন।
- **মুদ্রা নির্বাচক সাইনআপ ফিল্ড (Currency selector signup field)** — চেকআউট/সাইনআপ করার সময় গ্রাহকদের তাদের পছন্দের মুদ্রা বেছে নিতে দেয়।
- **একাধিক এক্সচেঞ্জ রেট প্রোভাইডার (Multiple exchange rate providers)** — ECB, Frankfurter, Stripe, অথবা সম্পূর্ণ ম্যানুয়াল রেট।
- **জিওলোকেশন-ভিত্তিক মুদ্রা সনাক্তকরণ (Geolocation-based currency detection)** — স্বয়ংক্রিয়ভাবে ভিজিটরের দেশের সাথে মিলে যায় এমন মুদ্রাটি প্রি-সিলেক্ট করে।
- **অ্যাডমিন ড্যাশবোর্ড (Admin dashboard)** — এক জায়গা থেকে সক্রিয় মুদ্রা, এক্সচেঞ্জ রেট এবং প্রোভাইডার সেটিংস পরিচালনা করুন।

## প্রয়োজনীয়তা (Requirements) {#requirements}

- Ultimate Multisite 2.0.0 বা তার বেশি
- WordPress Multisite ইনস্টলেশন
- PHP 7.4 বা তার বেশি

## ইনস্টলেশন (Installation) {#installation}

1. প্লাগইন ফাইলগুলি `/wp-content/plugins/ultimate-multisite-multi-currency` এ আপলোড করুন।
2. ওয়ার্ডপ্রেসের **Plugins** স্ক্রিনের মাধ্যমে প্লাগইনটি অ্যাক্টিভেট করুন (Network Activate)।
3. অ্যাডমিন ড্যাশবোর্ড খোলার জন্য **Ultimate Multisite → Multi-Currency** এ যান।
4. আপনি যে মুদ্রাগুলি গ্রহণ করতে চান তা যোগ করুন এবং আপনার এক্সচেঞ্জ রেট প্রোভাইডার কনফিগার করুন।

## কনফিগারেশন (Configuration) {#configuration}

### অ্যাডমিন ড্যাশবোর্ড (Admin Dashboard) {#admin-dashboard}

Multi-Currency অ্যাডমিন ড্যাশবোর্ড (**Ultimate Multisite → Multi-Currency**) হলো কেন্দ্রীয় স্থান যেখানে আপনি:

- মুদ্রা যোগ, সক্রিয় বা নিষ্ক্রিয় করতে পারেন।
- একটি এক্সচেঞ্জ রেট প্রোভাইডার বেছে নিতে এবং প্রয়োজন অনুসারে রেট রিফ্রেশ করতে পারেন।
- সমস্ত সক্রিয় মুদ্রার বর্তমান রেট পর্যালোচনা করতে পারেন।
- যখন স্বয়ংক্রিয় প্রোভাইডার উপযুক্ত নয়, তখন ম্যানুয়াল রেট সেট করতে পারেন।

### এক্সচেঞ্জ রেট প্রোভাইডার (Exchange Rate Providers) {#exchange-rate-providers}

| প্রোভাইডার (Provider) | বিবরণ (Description) |
|---|---|
| **ECB** | ইউরোপীয় সেন্ট্রাল ব্যাংক — EUR-ভিত্তিক নেটওয়ার্কের জন্য বিনামূল্যে দৈনিক রেট। |
| **Frankfurter** | ওপেন-সোর্স ECB মিরর API — বিনামূল্যে, কোনো API কী প্রয়োজন নেই। |
| **Stripe** | Stripe-এর লাইভ FX রেট ব্যবহার করে — সক্রিয় Stripe ইন্টিগ্রেশন প্রয়োজন। |
| **Manual** | আপনি সরাসরি রেট প্রবেশ করান; কোনো বাহ্যিক API কল প্রয়োজন নেই। |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** এর অধীনে আপনার প্রোভাইডারটি নির্বাচন করুন। রেটগুলি যেকোনো সময় ম্যানুয়ালি রিফ্রেশ করা যেতে পারে বা স্বয়ংক্রিয়ভাবে আপডেট করার জন্য সময়সূচী সেট করা যেতে পারে।

### পণ্য-ভিত্তিক মূল্য ওভাররাইড (Per-Product Pricing Overrides) {#per-product-pricing-overrides}

যখন স্বয়ংক্রিয় রূপান্তর যথেষ্ট নির্ভুল হয় না, তখন আপনি যেকোনো পণ্যের জন্য একটি নির্দিষ্ট মুদ্রায় সঠিক দাম সেট করতে পারেন:

1. পণ্য এডিটর খুলুন (**Ultimate Multisite → Products → Edit**)।
2. **Currency Pricing** সেকশনে স্ক্রল করুন।
3. আপনি যে মুদ্রাগুলিতে ওভাররাইড করতে চান তার জন্য সঠিক দামটি প্রবেশ করান।
4. পণ্যটি সেভ করুন।

ওভাররাইড ছাড়া পণ্যগুলি আপনার মূল মুদ্রা থেকে রূপান্তরিত দাম ব্যবহার করে।

### মুদ্রা নির্বাচক সাইনআপ ফিল্ড (Currency Selector Signup Field) {#currency-selector-signup-field}

এই মুদ্রা নির্বাচক ফিল্ডটি আপনার সাইনআপ/চেকআউট ফ্লোতে যোগ করা যেতে পারে যাতে গ্রাহকরা রেজিস্ট্রেশনের সময় তাদের পছন্দের মুদ্রা বেছে নিতে পারে:

1. **Ultimate Multisite → Multi-Currency → Settings** এ যান।
2. **Show currency selector on signup** সক্ষম করুন।
3. ঐচ্ছিকভাবে একটি ডিফল্ট মুদ্রা সেট করুন বা জিওলোকেশনকে প্রি-সিলেক্ট করা মান নির্ধারণ করতে দিন।

নির্বাচিত মুদ্রাটি সাবস্ক্রিপশনের সাথে সংরক্ষণ করা হয় এবং পরবর্তী সমস্ত বিলিংয়ের জন্য ব্যবহৃত হয়।

### জিওলোকেশন-ভিত্তিক মুদ্রা সনাক্তকরণ (Geolocation-Based Currency Detection) {#geolocation-based-currency-detection}

যখন জিওলোকেশন ডিটেকশন সক্ষম করা হয়, তখন অ্যাডনটি ভিজিটরের দেশের সাথে একটি মুদ্রাকে ম্যাপ করে এবং মুদ্রা নির্বাচক ফিল্ডে এটি প্রি-সিলেক্ট করে। সক্ষম করতে:

1. **Ultimate Multisite → Multi-Currency → Settings** এ যান।
2. **Detect currency from visitor location** সক্ষম করুন।
3. **Geolocation Mapping** টেবিলে দেশগুলিকে মুদ্রার সাথে ম্যাপ করুন।

যদি কোনো ভিজিটরের দেশের জন্য কোনো ম্যাপিং না থাকে, তবে নেটওয়ার্কের মূল মুদ্রাটি ফলব্যাক হিসেবে ব্যবহৃত হয়।

## প্রায়শই জিজ্ঞাসিত প্রশ্নাবলী (Frequently Asked Questions) {#frequently-asked-questions}

### এক্সচেঞ্জ রেট প্রোভাইডার অনুপলব্ধ হলে কী হবে? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

অ্যাডনটি শেষ সফলভাবে প্রাপ্ত রেটগুলিতে ফিরে যায়। যদি কখনও কোনো রেট না পাওয়া যায়, তবে এটি আপনার কনফিগার করা ম্যানুয়াল রেটগুলিতে ফিরে যায়। একটি সেফটি নেট হিসেবে ম্যানুয়াল রেট কনফিগার করার পরামর্শ দেওয়া হয়।

### আমি কি বিভিন্ন মুদ্রার জন্য বিভিন্ন প্রোভাইডার ব্যবহার করতে পারি? {#can-i-use-different-providers-for-different-currencies}

না — সমস্ত মুদ্রার জন্য একবারে একটি প্রোভাইডার সক্রিয় থাকে। যদি আপনার ব্যক্তিগত রেটগুলির উপর সম্পূর্ণ নিয়ন্ত্রণ প্রয়োজন হয়, তবে **Manual** ব্যবহার করুন।

### জিওলোকেশন কি একটি আলাদা প্লাগইন দাবি করে? {#does-geolocation-require-a-separate-plugin}

কোনো বাহ্যিক জিওলোকেশন প্লাগইন প্রয়োজন নেই। অ্যাডনটি ওয়ার্ডপ্রেসের বিল্ট-ইন IP-টু-কান্ট্রি ডিটেকশন ব্যবহার করে। উচ্চতর নির্ভুলতার জন্য, প্রদত্ত ফিল্টার হুকগুলির মাধ্যমে একটি ডেডিকেটেড জিওলোকেশন সার্ভিস ইন্টিগ্রেট করা যেতে পারে।

### পণ্য-ভিত্তিক ওভাররাইড কি ট্যাক্সের আগে নাকি পরে প্রয়োগ করা হয়? {#are-per-product-overrides-applied-before-or-after-tax}

পণ্য-ভিত্তিক ওভাররাইড টার্গেট মুদ্রায় প্রি-ট্যাক্স দাম সেট করে। আপনার বিদ্যমান ট্যাক্স কনফিগারেশন ব্যবহার করে ওভাররাইড দামের উপর ট্যাক্স গণনা করা হয়।
