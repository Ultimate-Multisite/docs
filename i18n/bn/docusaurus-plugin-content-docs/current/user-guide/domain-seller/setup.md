# ডোমেইন সেলার: সেটআপ এবং প্রোভাইডার কনফিগারেশন

ডোমেইন সেলার অ্যাডনটি একটি গাইড করা সেটআপ উইজার্ড সহ আসে যা আপনাকে প্রতিটি প্রয়োজনীয় ধাপের মধ্য দিয়ে নিয়ে যায়। এই পাতাটি উইজার্ডের প্রবাহ এবং পরে কীভাবে প্রোভাইডার কনফিগার বা পুনরায় কনফিগার করা যায় তা নিয়ে আলোচনা করে।

## প্রয়োজনীয়তা

- **Multisite Ultimate** v2.4.12 বা তার বেশি, নেটওয়ার্ক-অ্যাক্টিভেটেড
- **PHP** 7.4+
- কমপক্ষে একটি সমর্থিত রেজিস্ট্রার-এর জন্য API ক্রেডেনশিয়াল

## প্রথমবার সেটআপ উইজার্ড

প্লাগইনটি প্রথমবার নেটওয়ার্ক-অ্যাক্টিভেট করার সময় সেটআপ উইজার্ডটি স্বয়ংক্রিয়ভাবে চালু হয়। এটি যেকোনো সময় **Network Admin › Ultimate Multisite › Domain Seller Setup** থেকে পাওয়া যায়।

### ধাপ ১ — একটি প্রোভাইডার নির্বাচন করুন

আপনি কোন রেজিস্ট্রার কানেক্ট করতে চান তা নির্বাচন করুন। সমর্থিত অপশনগুলি:

| Provider | DNS management | WHOIS privacy |
|---|---|---|
| OpenSRS | Yes | Yes |
| Namecheap | No | Yes (WhoisGuard, free) |
| GoDaddy | No | No |
| ResellerClub | Yes | No |
| NameSilo | No | No |
| Enom | Yes | No |

### ধাপ ২ — ক্রেডেনশিয়াল লিখুন

প্রতিটি প্রোভাইডারের জন্য ভিন্ন ভিন্ন ক্রেডেনশিয়াল ফিল্ড রয়েছে:

**OpenSRS** — ইউজারনেম এবং প্রাইভেট কী (OpenSRS Reseller Control Panel থেকে)

**Namecheap** — ইউজারনেম এবং API কী (Account › Tools › API Access থেকে)

**GoDaddy** — API কী এবং সিক্রেট (developer.godaddy.com থেকে)

**ResellerClub** — রিসেলার আইডি এবং API কী (ResellerClub control panel থেকে)

**NameSilo** — API কী (namesilo.com › Account › API Manager থেকে)

**Enom** — অ্যাকাউন্ট আইডি এবং API টোকেন

লাইভ হওয়ার আগে প্রোভাইডারের টেস্ট এনভায়রনস্টিকের বিরুদ্ধে পরীক্ষা করার জন্য যেখানে উপলব্ধ সেখানে **Sandbox mode** চেক করুন।

### ধাপ ৩ — সংযোগ পরীক্ষা করুন

**Test Connection** এ ক্লিক করুন। উইজার্ডটি ক্রেডেনশিয়াল এবং সংযোগ যাচাই করার জন্য একটি হালকা API কল পাঠায়। এগিয়ে যাওয়ার আগে যেকোনো ক্রেডেনশিয়াল সমস্যা ঠিক করুন।

### ধাপ ৪ — TLD আমদানি করুন

সংযুক্ত প্রোভাইডার থেকে সমস্ত উপলব্ধ TLD এবং পাইকারি মূল্য (wholesale pricing) টেনে আনার জন্য **Import TLDs** এ ক্লিক করুন। এটি ডোমেইন পণ্যগুলিতে ব্যবহৃত TLD তালিকাটি পূরণ করে। TLD ক্যাটালগ বড় প্রোভাইডারদের জন্য এই আমদানি করতে ৩০-৬০ সেকেন্ড সময় লাগতে পারে।

TLDগুলি একটি নির্ধারিত ক্রন জব (scheduled cron job) এর মাধ্যমে প্রতিদিন একবার স্বয়ংক্রিয়ভাবে পুনরায় সিঙ্ক (re-synced) হয়।

### ধাপ ৫ — একটি ডোমেইন পণ্য তৈরি করুন

উইজার্ডটি একটি ডিফল্ট ক্যাচ-অল ডোমেইন পণ্য তৈরি করে যেখানে ১০% মার্কআপ থাকে। আপনি অবিলম্বে এই পণ্যটি সম্পাদনা করতে পারেন অথবা স্কিপ করে **Ultimate Multisite › Products** এর অধীনে ম্যানুয়ালি পণ্য তৈরি করতে পারেন।

সম্পূর্ণ পণ্য কনফিগারেশন গাইড দেখতে [Domain Products and Pricing](./domain-products)।

## প্রোভাইডার পুনরায় কনফিগার করা

**Network Admin › Ultimate Multisite › Settings › Domain Seller** এ যান (অথবা প্লাগইন তালিকায় **Settings** এ ক্লিক করুন)।

সেটিংস পেজে রয়েছে:

- **Enable domain selling** — পুরো ফিচারটি চালু/বন্ধ করার জন্য টগল করুন
- **Default provider** — ডোমেইন অনুসন্ধান এবং নতুন পণ্যগুলির জন্য ব্যবহৃত প্রোভাইডার
- **Max TLDs per search** — গ্রাহক যখন অনুসন্ধান করেন তখন কতগুলি TLD চেক করা হবে; উচ্চ মানগুলি আরও বেশি অপশন দেখায় তবে ধীর হয়
- **Availability cache duration** — প্রাপ্যতা এবং মূল্যের ফলাফল কতক্ষণ ক্যাশ করা হবে; কম মানগুলি আরও সঠিক তবে API কল বাড়ায়
- **Manage domain products** — প্রোডাক্ট তালিকায় দ্রুত লিঙ্ক
- **Configure providers** — প্রোভাইডার যোগ বা পুনরায় কনফিগার করতে ইন্টিগ্রেশন উইজার্ডটি খোলে

### দ্বিতীয় প্রোভাইডার যোগ করা

**Configure providers** এ ক্লিক করুন এবং নতুন রেজিস্ট্রারের জন্য উইজার্ডটি আবার চালান। আপনি একই সাথে একাধিক প্রোভাইডার কনফিগার করতে পারেন। প্রতিটি ডোমেইন পণ্যকে একটি নির্দিষ্ট প্রোভাইডারের সাথে যুক্ত করুন, অথবা এটিকে ডিফল্ট হিসেবে রাখুন।

### ম্যানুয়ালি TLD সিঙ্ক করা

সেটিংস পেজে, কোনো কনফিগার করা প্রোভাইডারের পাশে **Sync TLDs** এ ক্লিক করে সর্বশেষ মূল্যগুলি টেনে আনুন। কোনো প্রোভাইডার যখন পাইকারি মূল্য আপডেট করে বা নতুন TLD যোগ করে, তখন এটি কার্যকর।

## লগ

প্রতিটি প্রোভাইডার তার নিজস্ব লগ চ্যানেলে ডেটা লেখে। লগগুলি **Network Admin › Ultimate Multisite › Logs** এর অধীনে দেখা যায়:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | সমস্ত রেজিস্ট্রেশন প্রচেষ্টা (সফল এবং ব্যর্থ) |
| `domain-seller-renewal` | রিনিউয়াল জবের ফলাফল |
| `domain-seller-opensrs` | কাঁচা OpenSRS API কার্যকলাপ |
| `domain-seller-namecheap` | কাঁচা Namecheap API কার্যকলাপ |
| `domain-seller-godaddy` | কাঁচা GoDaddy API কার্যকলাপ |
| `domain-seller-resellerclub` | কাঁচা ResellerClub API কার্যকলাপ |
| `domain-seller-namesilo` | কাঁচা NameSilo API কার্যকলাপ |
| `domain-seller-enom` | কাঁচা Enom API কার্যকলাপ |
