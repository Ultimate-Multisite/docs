# হেডার ডোমেইন অনুসন্ধান

যখন আপনি একটি ছোট হেডার ফর্ম চান, যা ডোমেইন অনুসন্ধান শুরু করবে এবং এরপর গ্রাহককে Ultimate Multisite চেকআউটের মধ্যে উপলভ্য ডোমেইন থেকে বেছে নিতে দেবে, তখন এই সেটআপ ব্যবহার করুন।

## প্রয়োজনীয়তা {#requirements}

- Ultimate Multisite নেটওয়ার্কে সক্রিয়।
- Multisite Ultimate Domain Seller নেটওয়ার্কে সক্রিয়।
- অন্তত একটি সক্রিয় ডোমেইন নিবন্ধন পণ্য, যাতে আছে:
  - `domain_provider` একটি কনফিগার করা provider-এ সেট করা।
  - সমর্থিত TLD কনফিগার করা, যেমন `com`, `net`, এবং `org`।
- **ডোমেইন নির্বাচন** ফিল্ডসহ একটি বৈধ চেকআউট ফর্ম।

## চেকআউট ফর্ম {#checkout-form}

1. নিবন্ধন পেজে ব্যবহৃত চেকআউট ফর্ম তৈরি বা সম্পাদনা করুন।
2. **ব্যবহারকারীর নাম**সহ সাধারণ প্রয়োজনীয় চেকআউট/Account ফিল্ড যোগ করুন। শুধু ডোমেইন ফিল্ড থাকা চেকআউট ফর্ম Ultimate Multisite যাচাইকরণে বাতিল হয়।
3. একটি **ডোমেইন নির্বাচন** ফিল্ড যোগ করুন।
4. ফ্লোটি ফ্রি সাবডোমেইন বা বিদ্যমান ডোমেইনের বদলে নিবন্ধিত ডোমেইনে কেন্দ্রীভূত হওয়া উচিত হলে ডোমেইন নির্বাচন মোড **শুধু নিবন্ধন**-এ সেট করুন।
5. সেই ফিল্ডে ডোমেইন নিবন্ধন পণ্য বরাদ্দ করুন।

নিবন্ধন পেজে চেকআউট ফর্ম রেন্ডার হওয়া উচিত, যেমন:

```text
[wu_checkout slug="domain-form"]
```

## হেডার ফর্ম {#header-form}

সাইটের হেডারে একটি ছোট `GET` ফর্ম যোগ করুন, যা দেওয়া অনুসন্ধান শব্দটি `domain_name` হিসেবে চেকআউট পেজে পাঠায়:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

কাস্টম হেডার JavaScript-এ আগে থেকে কোনো ডোমেইন নির্বাচন করবেন না। হেডার শুধু অনুসন্ধান শব্দটি পাঠাবে। Domain Seller চেকআউট স্ক্রিপ্ট `?domain_name=example` পড়ে, চেকআউট অনুসন্ধান বক্স পূরণ করে, এবং উপলভ্যতা অনুসন্ধান চালায়, যাতে গ্রাহক ফেরত আসা ডোমেইনগুলো থেকে বেছে নিতে পারেন।

## প্রত্যাশিত আচরণ {#expected-behaviour}

হেডারে `example` অনুসন্ধান করলে এটি খোলা উচিত:

```text
/register/?domain_name=example
```

এরপর চেকআউটে নির্বাচনযোগ্য ফলাফল দেখানো উচিত, যেমন:

- `example.com`
- `example.net`
- `example.org` উপলভ্য নয়
- provider-সমর্থিত অন্যান্য TLD

উপলভ্য ফলাফল নির্বাচন করার পর, অর্ডার সারাংশে ডোমেইন নিবন্ধন পণ্য এবং নির্বাচিত ডোমেইন নাম থাকা উচিত।

## যাচাই {#verification}

1. হোম পেজ খুলুন।
2. একটি খালি নাম অনুসন্ধান করুন, যেমন `example`।
3. চেকআউট URL-এ `?domain_name=example` আছে নিশ্চিত করুন।
4. চেকআউট ডোমেইন ফিল্ডে `example` আছে নিশ্চিত করুন।
5. ডোমেইন পছন্দের একটি তালিকা দেখা যাচ্ছে নিশ্চিত করুন।
6. উপলভ্য ডোমেইনের জন্য **নির্বাচন করুন** ক্লিক করুন।
7. অর্ডার সারাংশে `Domain Registration - example.com` বা নির্বাচিত ডোমেইন আছে নিশ্চিত করুন।

## সমস্যা সমাধান {#troubleshooting}

- কোনো তালিকা না দেখালে, browser network ট্যাবে `admin-ajax.php?action=wu_domain_search` পরীক্ষা করুন এবং নিশ্চিত করুন এটি খালি নয় এমন `domains` বা `results` ফেরত দিচ্ছে।
- সংরক্ষণের সময় চেকআউট ফর্ম যাচাইকরণে ব্যর্থ হলে, **ব্যবহারকারীর নাম**-এর মতো প্রয়োজনীয় Account ফিল্ড যোগ করুন।
- ডোমেইন নির্বাচন করলে কার্ট আপডেট না হলে, `window.wu_checkout_form` আছে এবং Domain Seller চেকআউট সম্পদগুলো চেকআউট পেজে লোড হয়েছে নিশ্চিত করুন।
