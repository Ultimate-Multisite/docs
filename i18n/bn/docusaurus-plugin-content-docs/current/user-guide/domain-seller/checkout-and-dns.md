# চেকআউট ফিল্ড এবং কাস্টমার DNS ম্যানেজমেন্ট

## ডোমেইন সিলেকশন চেকআউট ফিল্ড

**Domain Selection** ফিল্ডটি একটি চেকআউট উপাদান যা গ্রাহককে তাদের সাইটের ডোমেইন কীভাবে পেতে চান, সেই বিষয়ে বিভিন্ন বিকল্প দেয়। ডোমেইন বিক্রি সক্ষম করতে এটি যেকোনো চেকআউট ফর্মে যোগ করুন।

### চেকআউট ফর্মে ফিল্ড যোগ করা

১. **Network Admin › Ultimate Multisite › Checkout Forms** এ যান।
২. একটি চেকআউট ফর্ম খুলুন বা তৈরি করুন।
৩. চেকআউট এডিটরে, **Add Field** এ ক্লিক করুন।
৪. ফিল্ড তালিকা থেকে **Domain Selection** বেছে নিন।
৫. ফিল্ড অপশনগুলো কনফিগার করুন (নিচে দেখুন)।
৬. ফর্মটি সেভ করুন।

### ফিল্ড অপশনসমূহ

**Domain modes** — গ্রাহক কোন ট্যাবগুলো দেখতে পাবেন, তা এখানে বেছে নিন। প্রতিটি মোড স্বাধীনভাবে চালু বা বন্ধ করা যেতে পারে:

| Mode | এটি কী করে |
|---|---|
| **Subdomain** | গ্রাহক আপনার নেটওয়ার্কে একটি ফ্রি সাবডোমেইন ব্যবহার করেন (যেমন, `mysite.yournetwork.com`)। কোনো পেমেন্টের প্রয়োজন নেই। |
| **Register New Domain** | গ্রাহক একটি নতুন ডোমেইন খুঁজে বের করে এবং আপনার কনফিগার করা প্রোভাইডারের মাধ্যমে তা রেজিস্টার করেন। মূল্য নির্ধারণের জন্য ম্যাচিং ডোমেইন প্রোডাক্ট ব্যবহার করা হয়। |
| **Existing Domain** | গ্রাহক তাদের নিজস্ব একটি ডোমেইন ম্যাপ করেন। কোনো রেজিস্ট্রেশন ফি লাগে না। ডোমেইনটি স্বয়ংক্রিয়ভাবে তাদের সাইটের সাথে ম্যাপ হয়ে যায়। |

**Default mode** — যখন তিনটি মোডই চালু থাকে, তখন কোন ট্যাবটি প্রথমে খুলবে। ডোমেইন রেজিস্ট্রেশনকে ঐচ্ছিক রাখতে **Subdomain** সেট করুন, অথবা কেনাকাটা উৎসাহিত করতে **Register New Domain** সেট করুন।

**Domain product** — ঐচ্ছিকভাবে এই ফিল্ডটিকে একটি নির্দিষ্ট ডোমেইন প্রোডাক্টের সাথে যুক্ত করুন। যদি এটি সেট করা না থাকে, তবে অ্যাডঅনটি স্বয়ংক্রিয়ভাবে গ্রাহক যে TLD টি সার্চ করছেন তার ভিত্তিতে ম্যাচিং প্রোডাক্টটি নির্বাচন করে।

### Registrant contact fields

যখন কোনো গ্রাহক **Register New Domain** ট্যাবটি নির্বাচন করেন, তখন চেকআউট ফর্মটি ইনলাইন হিসেবে রেজিস্ট্র্যান্ট কন্ট্যাক্ট ফিল্ডগুলো যোগ করে:

- First name / Last name
- Email address
- Address (line 1, city, state/province, postal code, country)
- Phone number

এগুলো সমস্ত রেজিস্ট্রার দ্বারা বাধ্যতামূলক এবং রেজিস্ট্রেশন API কল করার আগে এগুলো যাচাই করা হয়। ফোন নম্বরগুলো স্বয়ংক্রিয়ভাবে রেজিস্ট্রারদের প্রত্যাশিত `+CC.NNN` আন্তর্জাতিক ফরম্যাটে সাজানো হয়।

### Auto-generated site URL

যখন কোনো গ্রাহক ডোমেইন রেজিস্টার করেন বা ম্যাপ করেন, তখন নির্বাচিত ডোমেইন থেকে সাইট URL ফিল্ডটি স্বয়ংক্রিয়ভাবে পূরণ হয়ে যায়। গ্রাহকদের আলাদাভাবে কোনো URL ফিল্ড পূরণ করার প্রয়োজন হয় না।

### Search behaviour

- গ্রাহক টাইপ করার সাথে সাথে AJAX ব্যবহার করে রিয়েল-টাইমে ডোমেইন উপলব্ধতা পরীক্ষা করা হয়।
- পছন্দের ডোমেইনটি অনুপলব্ধ হলে বিকল্প TLD সাজেশন দেখানো হয়।
- মূল্য লাইভ ফেচ করা হয় এবং স্পষ্টভাবে দেখানো হয় (রেজিস্ট্রেশন মূল্য, রিনিউয়াল মূল্য, ঐচ্ছিক WHOIS প্রাইভেসি ফি)।
- কুপন কোডগুলো অন্য যেকোনো প্রোডাক্টের মতোই ডোমেইন প্রোডাক্টগুলোতে প্রযোজ্য হয়।

**সার্চ রেসপনসিভনেস টিউন করা:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ডোমেইন সার্চ ফর্মে কাস্টম ফিল্ড যোগ করা:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## কাস্টমার DNS ম্যানেজমেন্ট

গ্রাহকরা **My Account** পেজ থেকে, তাদের ডোমেইনের এন্ট্রির অধীনে, তাদের রেজিস্টার্ড ডোমেইনগুলোর জন্য DNS রেকর্ডগুলো ম্যানেজ করতে পারেন।

### সমর্থিত রেকর্ড প্রকারভেদ

| Type | ব্যবহার |
|---|---|
| **A** | হোস্টনেমকে IPv4 অ্যাড্রেসের সাথে ম্যাপ করা |
| **AAAA** | হোস্টনেমকে IPv6 অ্যাড্রেসের সাথে ম্যাপ করা |
| **CNAME** | অন্য একটি হোস্টনেমের দিকে নির্দেশ করে একটি Alias তৈরি করা |
| **MX** | মেইল এক্সচেঞ্জ সার্ভার সেট করা |
| **TXT** | SPF, DMARC, ভেরিফিকেশন বা অন্যান্য টেক্সট রেকর্ড যোগ করা |

### কোন প্রোভাইডাররা DNS ম্যানেজমেন্ট সমর্থন করে?

DNS ম্যানেজমেন্ট (রেকর্ড যোগ, সম্পাদনা, মুছে ফেলা) **OpenSRS**, **ResellerClub**, এবং **Enom** এর সাথে উপলব্ধ। Namecheap, GoDaddy, এবং NameSilo ডোমেইনগুলো স্ট্যাটাস এবং মেয়াদ শেষের তথ্য দেখায়, তবে DNS অবশ্যই রেজিস্ট্রারর কন্ট্রোল প্যানেলে সরাসরি ম্যানেজ করতে হবে।

### ডিফল্ট DNS রেকর্ড

আপনি ডিফল্ট DNS রেকর্ড কনফিগার করতে পারেন যা ডোমেইন রেজিস্টার করার সময় স্বয়ংক্রিয়ভাবে প্রয়োগ হয়। **Settings › Domain Seller › Default DNS Records** এ যান।

ডিফল্ট রেকর্ড ভ্যালুগুলো দুটি টোকেন সমর্থন করে:

| Token | কিসের সাথে প্রতিস্থাপিত হবে |
|---|---|
| `{DOMAIN}` | রেজিস্টার্ড ডোমেইন নামটি (যেমন, `example.com`) |
| `{SITE_URL}` | গ্রাহকের সাইটের জন্য WordPress সাইট URL |

**উদাহরণ — অ্যাপেক্স ডোমেইন এবং www কে আপনার সার্ভার IP এর সাথে পয়েন্ট করা:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### অ্যাডমিন: DNS দেখা এবং সম্পাদনা করা

Network অ্যাডমিনরা **Network Admin › Ultimate Multisite › Domains** এ ডোমেইনটির এডিট পেজ থেকে যেকোনো গ্রাহকের ডোমেইনের DNS রেকর্ড দেখতে এবং সম্পাদনা করতে পারেন।
