# চেকআউট ফিল্ড এবং গ্রাহকের DNS ব্যবস্থাপনা

## ডোমেইন নির্বাচন চেকআউট ফিল্ড

**ডোমেইন নির্বাচন** ফিল্ড হলো একটি চেকআউট উপাদান, যা গ্রাহককে তার সাইটের ডোমেইন পাওয়ার পদ্ধতি বেছে নিতে দেয়। ডোমেইন বিক্রি চালু করতে এটি যেকোনো চেকআউট ফর্মে যোগ করুন।

### চেকআউট ফর্মে ফিল্ড যোগ করা

1. **নেটওয়ার্ক অ্যাডমিন › Ultimate Multisite › চেকআউট ফর্ম**-এ যান
2. একটি চেকআউট ফর্ম খুলুন বা তৈরি করুন
3. চেকআউট এডিটরে **ফিল্ড যোগ করুন** ক্লিক করুন
4. ফিল্ড তালিকা থেকে **ডোমেইন নির্বাচন** বেছে নিন
5. ফিল্ডের বিকল্পগুলো কনফিগার করুন (নিচে দেখুন)
6. ফর্ম সংরক্ষণ করুন

### ফিল্ডের বিকল্প

**ডোমেইন মোড** — গ্রাহক কোন ট্যাবগুলো দেখবে তা বেছে নিন। প্রতিটি মোড আলাদাভাবে চালু বা বন্ধ করা যায়:

| মোড | এটি কী করে |
|---|---|
| **সাবডোমেইন** | গ্রাহক আপনার নেটওয়ার্কে একটি বিনামূল্যের সাবডোমেইন ব্যবহার করে (যেমন, `mysite.yournetwork.com`)। কোনো পেমেন্ট দরকার নেই। |
| **নতুন ডোমেইন নিবন্ধন** | গ্রাহক একটি নতুন ডোমেইন খোঁজে এবং আপনার কনফিগার করা প্রোভাইডারের মাধ্যমে সেটি নিবন্ধন করে। মূল্যের জন্য মিলে যাওয়া ডোমেইন পণ্য ব্যবহার করা হয়। |
| **বিদ্যমান ডোমেইন** | গ্রাহক তার আগে থেকেই মালিকানাধীন একটি ডোমেইন ম্যাপ করে। কোনো নিবন্ধন ফি নেই। ডোমেইনটি স্বয়ংক্রিয়ভাবে তার সাইটে ম্যাপ হয়। |

**ডিফল্ট মোড** — তিনটি মোডই চালু থাকলে কোন ট্যাবটি আগে খুলবে। ডোমেইন নিবন্ধন ঐচ্ছিক রাখতে **সাবডোমেইন** সেট করুন, অথবা ক্রয় উৎসাহিত করতে **নতুন ডোমেইন নিবন্ধন** সেট করুন।

**ডোমেইন পণ্য** — চাইলে এই ফিল্ডটিকে নির্দিষ্ট একটি ডোমেইন পণ্যে পিন করুন। সেট না করা থাকলে, গ্রাহক যে TLD খোঁজে তার ভিত্তিতে addon স্বয়ংক্রিয়ভাবে মিলে যাওয়া পণ্য নির্বাচন করে।

### নিবন্ধনকারীর যোগাযোগ ফিল্ড

গ্রাহক **নতুন ডোমেইন নিবন্ধন** ট্যাব নির্বাচন করলে, চেকআউট ফর্মে ইনলাইনে নিবন্ধনকারীর যোগাযোগ ফিল্ড যোগ হয়:

- নাম / পদবি
- ইমেইল ঠিকানা
- ঠিকানা (লাইন ১, শহর, রাজ্য/প্রদেশ, পোস্টাল কোড, দেশ)
- ফোন নম্বর

সব নিবন্ধকের জন্য এগুলো আবশ্যক এবং নিবন্ধন API কল করার আগে যাচাই করা হয়। ফোন নম্বর স্বয়ংক্রিয়ভাবে নিবন্ধকদের প্রত্যাশিত `+CC.NNN` আন্তর্জাতিক ফরম্যাটে সাজানো হয়।

### স্বয়ংক্রিয়ভাবে তৈরি সাইট URL

গ্রাহক কোনো ডোমেইন নিবন্ধন বা ম্যাপ করলে, সাইট URL ফিল্ডটি নির্বাচিত ডোমেইন থেকে স্বয়ংক্রিয়ভাবে পূরণ হয়। গ্রাহককে আলাদা URL ফিল্ড পূরণ করতে হয় না।

### অনুসন্ধানের আচরণ

- গ্রাহক টাইপ করার সময় AJAX দিয়ে রিয়েল-টাইমে ডোমেইনের প্রাপ্যতা পরীক্ষা করা হয়
- পছন্দের ডোমেইন পাওয়া না গেলে বিকল্প TLD পরামর্শ দেখানো হয়
- মূল্য লাইভ আনা হয় এবং স্পষ্টভাবে দেখানো হয় (নিবন্ধন মূল্য, নবায়ন মূল্য, ঐচ্ছিক WHOIS গোপনীয়তা ফি)
- কুপন কোড অন্য যেকোনো পণ্যের মতোই ডোমেইন পণ্যে প্রযোজ্য হয়

**অনুসন্ধানের সাড়া দেওয়ার গতি টিউন করা:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ডোমেইন অনুসন্ধান ফর্মে কাস্টম ফিল্ড যোগ করা:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## গ্রাহকের DNS ব্যবস্থাপনা

গ্রাহকেরা **আমার Account** পেজ থেকে, তাদের ডোমেইনের এন্ট্রির অধীনে নিবন্ধিত ডোমেইনের DNS রেকর্ড পরিচালনা করতে পারেন।

### সমর্থিত রেকর্ডের ধরন

| ধরন | ব্যবহার |
|---|---|
| **A** | হোস্টনেমকে IPv4 ঠিকানায় ম্যাপ করুন |
| **AAAA** | হোস্টনেমকে IPv6 ঠিকানায় ম্যাপ করুন |
| **CNAME** | অন্য হোস্টনেমে নির্দেশ করা একটি এলিয়াস তৈরি করুন |
| **MX** | মেইল এক্সচেঞ্জ সার্ভার সেট করুন |
| **TXT** | SPF, DMARC, যাচাইকরণ, বা অন্যান্য টেক্সট রেকর্ড যোগ করুন |

### কোন প্রোভাইডারগুলো DNS ব্যবস্থাপনা সমর্থন করে?

DNS ব্যবস্থাপনা (রেকর্ড যোগ, সম্পাদনা, মুছে ফেলা) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, এবং **Openprovider**-এর সঙ্গে পাওয়া যায়। **Hostinger** ডোমেইনগুলো Domain Seller-এর মাধ্যমে নেমসার্ভার আপডেট করতে পারে; হোস্ট করা ডোমেইনের DNS রেকর্ড মূল Hostinger ডোমেইন-ম্যাপিং ইন্টিগ্রেশন দিয়ে পরিচালিত হয়। Namecheap, GoDaddy, এবং NameSilo ডোমেইন স্ট্যাটাস ও মেয়াদোত্তীর্ণের তথ্য দেখায়, কিন্তু DNS সরাসরি নিবন্ধকের কন্ট্রোল প্যানেলে পরিচালনা করতে হবে।

### ডিফল্ট DNS রেকর্ড

ডোমেইন নিবন্ধনের সময় স্বয়ংক্রিয়ভাবে প্রয়োগ হওয়া ডিফল্ট DNS রেকর্ড কনফিগার করতে পারেন। **সেটিংস › Domain Seller › ডিফল্ট DNS রেকর্ড**-এ যান।

ডিফল্ট রেকর্ডের মান দুটি টোকেন সমর্থন করে:

| টোকেন | যা দিয়ে প্রতিস্থাপিত হবে |
|---|---|
| `{DOMAIN}` | নিবন্ধিত ডোমেইন নাম (যেমন, `example.com`) |
| `{SITE_URL}` | গ্রাহকের সাইটের জন্য WordPress সাইট URL |

**উদাহরণ — অ্যাপেক্স ডোমেইন এবং www আপনার সার্ভার IP-তে নির্দেশ করুন:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### অ্যাডমিন: DNS দেখা এবং সম্পাদনা

নেটওয়ার্ক অ্যাডমিনরা **নেটওয়ার্ক অ্যাডমিন › Ultimate Multisite › ডোমেইন**-এ ডোমেইনের সম্পাদনা পেজ থেকে যেকোনো গ্রাহক ডোমেইনের DNS রেকর্ড দেখতে ও সম্পাদনা করতে পারেন।
