# Third-Party Mode Migration

Superdav AI Agent v1.12.0 পরিবর্তন এনেছে যে থার্ড-পার্টি (third-party) ক্ষমতাগুলো (abilities) কীভাবে পরিচালনা করা হবে। **থার্ড-পার্টি মোড এখন ডিফল্টভাবে auto-তে সেট করা আছে**, যা ম্যানুয়াল কনফিগারেশন ছাড়াই WordPress 7.0+ এ নেটিভ WordPress Abilities API ইন্টিগ্রেশন সক্ষম করে।

## কী পরিবর্তন হয়েছে?

### v1.12.0 এর আগে

থার্ড-পার্টি ক্ষমতাগুলোর জন্য ম্যানুয়াল কনফিগারেশন প্রয়োজন ছিল:

- আপনাকে স্পষ্টভাবে "third-party mode" সক্ষম করতে হতো
- ক্ষমতাগুলো একটি কাস্টম রেজিস্ট্রি থেকে লোড করা হতো
- WordPress Abilities API-এর সাথে ইন্টিগ্রেশন ঐচ্ছিক ছিল
- লেগ্যাসি মোড (Legacy mode) ছিল ডিফল্ট

### v1.12.0 এর পরে

থার্ড-পার্টি ক্ষমতাগুলো স্বয়ংক্রিয়ভাবে কাজ করে:

- থার্ড-পার্টি মোড ডিফল্টভাবে "auto"-তে সেট করা আছে
- ক্ষমতাগুলো WordPress Abilities API-এর সাথে নেটিভভাবে ইন্টিগ্রেট করে
- WordPress 7.0+ এ কোনো ম্যানুয়াল কনফিগারেশন প্রয়োজন নেই
- পুরোনো WordPress সংস্করণের জন্য লেগ্যাসি মোড এখনও উপলব্ধ

## কার উপর প্রভাব পড়বে?

### নতুন ইনস্টলেশন (WordPress 7.0+)

**কোনো পদক্ষেপ নেওয়ার প্রয়োজন নেই।** থার্ড-পার্টি মোড স্বয়ংক্রিয়ভাবে "auto"-তে সেট হয়ে যায় এবং ক্ষমতাগুলো সরাসরি কাজ করে।

### বিদ্যমান ইনস্টলেশন

**আপনার সেটিংস সংরক্ষিত থাকবে।** আপনি যদি ব্যবহার করে থাকেন:

- **Legacy mode**: আপনি লেগ্যাসি মোডেই থাকবেন (কোনো পরিবর্তন নেই)
- **Manual third-party mode**: আপনি ম্যানুয়াল মোডেই থাকবেন (কোনো পরিবর্তন নেই)
- **Auto mode**: আপনি auto মোডেই চালিয়ে যাবেন (কোনো পরিবর্তন নেই)

### 7.0 এর আগের WordPress সংস্করণ

**লেগ্যাসি মোড এখনও উপলব্ধ।** আপনি যদি WordPress 6.x বা তার আগের সংস্করণে থাকেন:

- থার্ড-পার্টি মোড ডিফল্টভাবে "legacy"-তে সেট হয়
- আপনি চাইলে ম্যানুয়ালি থার্ড-পার্টি মোড সক্ষম করতে পারেন
- নেটিভ Abilities API ব্যবহার করতে WordPress 7.0+ এ আপগ্রেড করুন

## মোডগুলো বোঝা

### Auto Mode (নতুন ডিফল্ট)

**Auto mode** নেটিভ WordPress Abilities API ইন্টিগ্রেশন ব্যবহার করে:

- ক্ষমতাগুলো WordPress hooks এর মাধ্যমে রেজিস্টার হয়
- WordPress 7.0+ Abilities API এর সাথে সম্পূর্ণ সামঞ্জস্যতা
- থার্ড-পার্টি ক্ষমতাগুলোর স্বয়ংক্রিয় আবিষ্কার (Automatic discovery)
- কোনো ম্যানুয়াল কনফিগারেশন প্রয়োজন নেই

**কখন ব্যবহার করবেন**: WordPress 7.0+ এবং থার্ড-পার্টি ক্ষমতা থাকলে

### Manual Mode

**Manual mode** এর জন্য সুস্পষ্ট কনফিগারেশন প্রয়োজন:

- আপনি নির্দিষ্ট করেন কোন থার্ড-পার্টি ক্ষমতাগুলো লোড করতে হবে
- টেস্টিং বা নির্দিষ্ট ক্ষমতা লোড করার জন্য উপযোগী
- কনফিগারেশন ফাইল এডিট করার প্রয়োজন
- বেশি নিয়ন্ত্রণ, তবে বেশি সেটআপ প্রয়োজন

**কখন ব্যবহার করবেন**: টেস্টিং, নির্দিষ্ট ক্ষমতা লোড করা, বা কাস্টম কনফিগারেশন

### Legacy Mode

**Legacy mode** পুরোনো থার্ড-পার্টি ক্ষমতা সিস্টেম ব্যবহার করে:

- কাস্টম ক্ষমতা রেজিস্ট্রি (WordPress Abilities API নয়)
- পুরোনো WordPress সংস্করণের সাথে সামঞ্জস্যপূর্ণ
- কোনো নেটিভ WordPress ইন্টিগ্রেশন নেই
- বাতিল (Deprecated) হলেও এখনও সমর্থিত

**কখন ব্যবহার করবেন**: WordPress 6.x বা তার আগের সংস্করণ, অথবা যখন আপনার লেগ্যাসি সামঞ্জস্যতা প্রয়োজন

## আপনার বর্তমান মোড পরীক্ষা করা

### অ্যাডমিন প্যানেলের মাধ্যমে

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** এ যান
2. **Third-Party Mode** সেটিংসটি খুঁজুন
3. আপনি আপনার বর্তমান মোড এবং এটি পরিবর্তন করার অপশন দেখতে পাবেন

### কোডের মাধ্যমে

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', বা 'legacy'
```

## আপনার মোড পরিবর্তন করা

### Auto Mode-এ পরিবর্তন করুন

আপনি যদি WordPress 7.0+ এ থাকেন এবং auto মোড ব্যবহার করতে চান:

1. **Superdav AI Agent** → **Settings** এ যান
2. **Third-Party Mode** খুঁজুন
3. **Auto (WordPress Abilities API)** নির্বাচন করুন
4. **Save** এ ক্লিক করুন

Superdav AI Agent স্বয়ংক্রিয়ভাবে থার্ড-পার্টি ক্ষমতাগুলো আবিষ্কার এবং রেজিস্টার করবে।

### Manual Mode-এ পরিবর্তন করুন

আপনি যদি ম্যানুয়ালি নিয়ন্ত্রণ করতে চান যে কোন ক্ষমতাগুলো লোড হবে:

1. **Superdav AI Agent** → **Settings** এ যান
2. **Third-Party Mode** খুঁজুন
3. **Manual** নির্বাচন করুন
4. **Save** এ ক্লিক করুন
5. কোন ক্ষমতাগুলো লোড করতে হবে তা নির্দিষ্ট করতে আপনার কনফিগারেশন ফাইলটি এডিট করুন

### Legacy Mode-এ পরিবর্তন করুন

যদি আপনার লেগ্যাসি সামঞ্জস্যতা প্রয়োজন হয়:

1. **Superdav AI Agent** → **Settings** এ যান
2. **Third-Party Mode** খুঁজুন
3. **Legacy** নির্বাচন করুন
4. **Save** এ ক্লিক করুন

## Auto Mode-এর সুবিধা

### স্বয়ংক্রিয় আবিষ্কার (Automatic Discovery)

ক্ষমতাগুলো স্বয়ংক্রিয়ভাবে নিম্নলিখিত উৎসগুলো থেকে আবিষ্কার হয়:

- ইনস্টল করা প্লাগইন
- সক্রিয় থিম
- must-use প্লাগইন
- drop-in প্লাগইন

ম্যানুয়াল রেজিস্ট্রেশনের প্রয়োজন নেই।

### নেটিভ ইন্টিগ্রেশন (Native Integration)

ক্ষমতাগুলো WordPress Abilities API এর সাথে ইন্টিগ্রেট করে:

- WordPress core এর সাথে সামঞ্জস্যপূর্ণ
- WordPress admin এর সাথে কাজ করে
- Abilities API ব্যবহার করে অন্যান্য প্লাগইনের সাথে সামঞ্জস্যপূর্ণ
- WordPress বিকশিত হওয়ার সাথে সাথে ভবিষ্যতের জন্য প্রস্তুত

### সরল ব্যবস্থাপনা (Simplified Management)

- এডিট করার জন্য কোনো কনফিগারেশন ফাইল নেই
- কোনো ম্যানুয়াল ক্ষমতা রেজিস্ট্রেশন নেই
- Ability Visibility নিয়ন্ত্রণ স্বয়ংক্রিয়ভাবে কাজ করে
- অ্যাডমিন নোটিফিকেশন আপনাকে শ্রেণীবদ্ধ না হওয়া ক্ষমতা সম্পর্কে সতর্ক করে

### উন্নত পারফরম্যান্স (Better Performance)

- ক্ষমতাগুলো ক্যাশ করা হয়
- প্রয়োজন অনুযায়ী lazy-load হয়
- WordPress 7.0+ এর জন্য অপটিমাইজ করা হয়েছে

## মাইগ্রেশন পাথ

### আপনি যদি WordPress 6.x এ থাকেন

1. **WordPress 7.0+ এ আপগ্রেড করুন** (যখন প্রস্তুত হবেন)
2. **Superdav AI Agent** কে v1.12.0+ এ আপডেট করুন
3. **থার্ড-পার্টি মোড Auto-তে পরিবর্তন করুন** (ঐচ্ছিক; লেগ্যাসি মোড এখনও কাজ করে)
4. সঠিক অ্যাক্সেস নিয়ন্ত্রণ নিশ্চিত করতে **ability visibility** পর্যালোচনা করুন

### আপনি যদি WordPress 7.0+ এ থাকেন

1. **Superdav AI Agent** কে v1.12.0+ এ আপডেট করুন
2. **থার্ড-পার্টি মোড Auto-তে সেট করা আছে কিনা তা যাচাই করুন** (এটি ডিফল্টভাবে থাকা উচিত)
3. সঠিক অ্যাক্সেস নিয়ন্ত্রণ নিশ্চিত করতে **ability visibility** পর্যালোচনা করুন
4. নিশ্চিত করতে **থার্ড-পার্টি ক্ষমতাগুলো পরীক্ষা করুন** যে সেগুলো কাজ করছে

## সমস্যা সমাধান (Troubleshooting)

### auto মোডে ক্ষমতা লোড হচ্ছে না

- যাচাই করুন আপনি WordPress 7.0+ এ আছেন
- নিশ্চিত করুন থার্ড-পার্টি মোড "Auto"-তে সেট করা আছে
- যাচাই করুন যে ক্ষমতা প্রদানকারী প্লাগইনটি সক্রিয় আছে
- রেজিস্ট্রেশন ত্রুটির জন্য WordPress error logs পরীক্ষা করুন

### আমি লেগ্যাসি মোড রাখতে চাই

- **Settings** → **Third-Party Mode** এ যান
- **Legacy** নির্বাচন করুন
- **Save** এ ক্লিক করুন
- লেগ্যাসি মোড কাজ করতে থাকবে

### আমার কাস্টম ক্ষমতাগুলো দেখা যাচ্ছে না

- যাচাই করুন যে সেগুলো WordPress hooks এর মাধ্যমে রেজিস্টার করা হয়েছে
- নিশ্চিত করুন যে তারা সঠিকভাবে Abilities API প্রয়োগ করে
- WordPress error logs পর্যালোচনা করুন
- সমস্ত রেজিস্টার্ড ক্ষমতা দেখতে **Ability Visibility** অ্যাডমিন পেজ ব্যবহার করুন

### আমি "unclassified ability" নোটিশ পাচ্ছি

- এটি নতুন থার্ড-পার্টি ক্ষমতাগুলোর জন্য স্বাভাবিক
- অ্যাডমিন নোটিশে পর্যালোচনা করুন এবং সেগুলোকে শ্রেণীবদ্ধ করুন
- শ্রেণীবদ্ধকরণের বিস্তারিত জানতে **Ability Visibility** দেখুন

## ব্যাকওয়ার্ড সামঞ্জস্যতা (Backward Compatibility)

### বিদ্যমান কনফিগারেশন

আপনার যদি বিদ্যমান থার্ড-পার্টি ক্ষমতা কনফিগারেশন থাকে:

- **Legacy mode**: আপনার কনফিগারেশন কাজ করতে থাকবে
- **Manual mode**: আপনার কনফিগারেশন কাজ করতে থাকবে
- **Auto mode**: আপনার কনফিগারেশন উপেক্ষা করা হবে (auto মোড প্রাধান্য পাবে)

আপনার কাস্টম কনফিগারেশন বজায় রাখতে, Manual বা Legacy মোডে থাকুন।

- সরল ব্যবস্থাপনার জন্য Auto মোডে পরিবর্তন করুন
- Ability Visibility ব্যবহার করে ক্ষমতাগুলো পর্যালোচনা এবং শ্রেণীবদ্ধ করুন

### কাস্টম ক্ষমতাগুলোর জন্য

- WordPress hooks (Abilities API) এর মাধ্যমে ক্ষমতা রেজিস্টার করুন
- কাস্টম ক্ষমতা রেজিস্ট্রি এড়িয়ে চলুন
- Auto mode সহ WordPress 7.0+ এ পরীক্ষা করুন

## পরবর্তী পদক্ষেপ

1. **আপনার WordPress সংস্করণ পরীক্ষা করুন**: Auto mode এর জন্য যাচাই করুন যে আপনি 7.0+ এ আছেন
2. **আপনার থার্ড-পার্টি মোড পর্যালোচনা করুন**: Settings এ যান এবং আপনার বর্তমান মোড পরীক্ষা করুন
3. **প্রয়োজনে আপডেট করুন**: আপনি যদি WordPress 7.0+ এ থাকেন তবে Auto মোডে পরিবর্তন করুন
4. **ক্ষমতাগুলো শ্রেণীবদ্ধ করুন**: কোনো শ্রেণীবদ্ধ না হওয়া ক্ষমতা থাকলে পর্যালোচনা এবং শ্রেণীবদ্ধ করুন
5. **পরীক্ষা করুন**: নিশ্চিত করুন যে আপনার থার্ড-পার্টি ক্ষমতাগুলো সঠিকভাবে কাজ করছে

## সম্পর্কিত বিষয়

- **Ability Visibility**: কোন ক্ষমতাগুলো কোথায় প্রদর্শিত হবে তা নিয়ন্ত্রণ করুন
- **WordPress Abilities API**: নেটিভ WordPress ক্ষমতা রেজিস্ট্রেশন সম্পর্কে জানুন
- **Third-Party Ability Development**: Auto mode এর সাথে কাজ করে এমন ক্ষমতা তৈরি করুন
