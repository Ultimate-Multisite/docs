# Plugin Builder ও Sandbox

Gratis AI Agent v1.5.0-এ **Plugin Builder & Sandbox System** যুক্ত করা হয়েছে। এই সিস্টেমের মাধ্যমে AI অ্যাসিস্ট্যান্ট আপনার নেটওয়ার্কে WordPress প্লাগইন তৈরি করতে, সক্রিয় করতে এবং পরিচালনা করতে পারে—সবকিছু একটি নিরাপদ, বিচ্ছিন্ন (isolated) sandbox পরিবেশে।

## ওভারভিউ

Plugin Builder AI অ্যাসিস্ট্যান্টকে প্রাকৃতিক ভাষার অনুরোধের ভিত্তিতে কাস্টম WordPress প্লাগইন লিখতে সাহায্য করে। যে প্লাগইনগুলো তৈরি হয়, সেগুলো লাইভ সাইটের কার্যকারিতাকে প্রভাবিত করার আগেই একটি sandbox লেয়ারে যাচাই (validate) করা হয়, সংরক্ষণ করা হয় এবং সক্রিয় করা হয়।

এর কিছু ব্যবহার ক্ষেত্র হলো:

- ডেভেলপারদের সরাসরি হস্তক্ষেপ ছাড়াই হালকা ইউটিলিটি প্লাগইন তৈরি করা।
- এমন ফিচারগুলোর প্রোটোটাইপিং করা, যার জন্য WordPress hooks বা কাস্টম পোস্ট টাইপ প্রয়োজন।
- ব্যাচ অপারেশনের জন্য স্বল্পস্থায়ী অটোমেশন স্ক্রিপ্ট তৈরি করা।

## AI ব্যবহার করে প্লাগইন তৈরি করা

একটি প্লাগইন তৈরি করতে, Gratis AI Agent চ্যাট ইন্টারফেসটি খুলুন এবং আপনি কী চান তা বর্ণনা করুন। উদাহরণস্বরূপ:

> "একটি প্লাগইন তৈরি করুন যা ড্যাশবোর্ডে একটি কাস্টম অ্যাডমিন নোটিশ যোগ করে।"

AI নিম্নলিখিত কাজগুলো করবে:

১. স্ট্রাকচার্ড কোড জেনারেশন ব্যবহার করে প্লাগইন PHP কোড তৈরি করবে।
২. সিনট্যাক্স ত্রুটি এবং অনিরাপদ প্যাটার্নগুলোর জন্য আউটপুট যাচাই করবে।
৩. তৈরি করা প্লাগইনটি sandbox স্টোরে সংরক্ষণ করবে।
৪. প্লাগইন স্ল্যাগ সহ একটি নিশ্চিতকরণ (confirmation) এবং একটি **Activate in Sandbox** বাটন ফিরিয়ে দেবে।

সক্রিয় করার আগে আপনি একই কথোপকথন থ্রেডে ফলো-আপ করে ফলাফলটি আরও উন্নত করতে পারেন।

## Sandbox সক্রিয়করণ

sandbox-এ একটি তৈরি প্লাগইন সক্রিয় করা, এটিকে লাইভ নেটওয়ার্কে সক্রিয় করার থেকে আলাদা। sandbox:

- একটি বিচ্ছিন্ন WordPress পরিবেশে (wp-env) প্লাগইনটি চালায়।
- যেকোনো PHP ত্রুটি, সতর্কতা বা hook কনফ্লিক্ট ক্যাপচার করে।
- চ্যাট ইন্টারফেসে সক্রিয়করণের ফলাফল রিপোর্ট করে।

sandbox-এ একটি প্লাগইন সক্রিয় করতে, AI চ্যাট প্রতিক্রিয়ার মধ্যে থাকা **Activate in Sandbox** বাটনে ক্লিক করুন, অথবা স্ল্যাশ কমান্ডটি ব্যবহার করুন:

```
/activate-plugin <plugin-slug>
```

একটি স্ট্যাটাস মেসেজ নিশ্চিত করে যে সক্রিয়করণ সফল হয়েছে নাকি ব্যর্থ হয়েছে। ব্যর্থ হলে, ত্রুটির লগটি চ্যাট থ্রেডে প্রদর্শিত হয়।

## তৈরি প্লাগইন পরিচালনা করা

তৈরি প্লাগইনগুলো **Gratis AI Agent → Plugin Builder → Manage Plugins**-এ তালিকাভুক্ত থাকে। এই স্ক্রিন থেকে আপনি যা করতে পারেন:

| অ্যাকশন | বিবরণ |
|---|---|
| **View source** | সম্পূর্ণ প্লাগইন PHP কোড পর্যালোচনা করুন। |
| **Re-activate in sandbox** | sandbox সক্রিয়করণ পরীক্ষাটি পুনরায় চালান। |
| **Install on network** | প্লাগইনটি লাইভ নেটওয়ার্কে স্থাপন করুন (ম্যানুয়াল নিশ্চিতকরণ প্রয়োজন)। |
| **Update** | AI-এর মাধ্যমে একটি নতুন সংস্করণ সরবরাহ করুন, যা বিদ্যমান কোডকে প্রতিস্থাপন করবে। |
| **Delete** | sandbox স্টোর থেকে প্লাগইনটি সরিয়ে দিন। প্রথমে এটি সমস্ত সাইট থেকে নিষ্ক্রিয় করে দেয়। |

:::warning
**Install on network** আপনার লাইভ WordPress multisite-এ তৈরি প্লাগইনটি স্থাপন করে। এগিয়ে যাওয়ার আগে প্লাগইন কোডটি পর্যালোচনা করুন। লাইভ ইনস্টল সম্পন্ন করার আগে Gratis AI Agent নিশ্চিতকরণের জন্য অনুরোধ করবে।
:::

## নেটওয়ার্কে তৈরি প্লাগইন ইনস্টল করা

যখন আপনি একটি sandboxed প্লাগইন নিয়ে সন্তুষ্ট হবেন, তখন আপনি এটি লাইভ নেটওয়ার্কে ইনস্টল করতে পারেন:

১. **Gratis AI Agent → Plugin Builder → Manage Plugins** এ যান।
২. যে প্লাগইনটি আপনি স্থাপন করতে চান তার পাশে **Install on Network** ক্লিক করুন।
৩. ডায়ালগটি নিশ্চিত করুন। প্লাগইনটি `wp-content/plugins/`-এ ইনস্টল হয় এবং নেটওয়ার্ক-সক্রিয় হয়।

বিকল্পভাবে, চ্যাট ইন্টারফেসে স্ল্যাশ কমান্ড ব্যবহার করুন:

```
/install-plugin <plugin-slug>
```

## প্লাগইন আপডেট

একটি তৈরি প্লাগইন আপডেট করতে, একটি নতুন কথোপকথনে AI অ্যাসিস্ট্যান্টকে পরিবর্তনটি বর্ণনা করুন:

> "dashboard-notice প্লাগইনটি আপডেট করুন যাতে নোটিশটি শুধুমাত্র অ্যাডমিনিস্ট্রেটরদের দেখায়।"

AI একটি নতুন সংস্করণ তৈরি করে, যা বর্তমান সংস্করণের পাশাপাশি sandbox-এ প্রদর্শিত হয়। আপডেট প্রয়োগ করার আগে আপনি diff পর্যালোচনা করে নিশ্চিত করবেন।

## HookScanner ইন্টিগ্রেশন

Plugin Builder প্রতিটি তৈরি প্লাগইন দ্বারা রেজিস্টার করা hooks এবং filters বিশ্লেষণ করার জন্য একটি ইন্টিগ্রেটেড **HookScanner** ব্যবহার করে। HookScanner আউটপুট চ্যাট প্রতিক্রিয়ায় দেখানো হয় এবং এতে যা যা থাকে:

- অ্যাকশন hooks রেজিস্টার করা হয়েছে (`add_action` কল)।
- ফিল্টার hooks রেজিস্টার করা হয়েছে (`add_filter` কল)।
- প্লাগইন নির্ভরতাগুলিতে পাওয়া যেকোনো hooks (এটি `vendor/` এবং `node_modules/` ডিরেক্টরি এড়িয়ে যায়)।

এটি আপনাকে সক্রিয় করার আগে একটি প্লাগইনের আচরণ বুঝতে সাহায্য করে।

## নিরাপত্তা বিবেচনা

- তৈরি প্লাগইনগুলো ম্যানুয়ালি ইনস্টল করা প্লাগইনগুলো থেকে আলাদাভাবে সংরক্ষণ করা হয় এবং আপনি যদি স্পষ্টভাবে সেগুলোকে নেটওয়ার্কে ইনস্টল না করেন, তবে স্ট্যান্ডার্ড WordPress প্লাগইন ম্যানেজমেন্ট স্ক্রিন থেকে এগুলো অ্যাক্সেস করা যায় না।
- প্লাগইন ফাইল লেখার সময় directory traversal প্রতিরোধ করতে sandbox path validation ব্যবহার করে।
- বিপজ্জনক ফাংশন কলযুক্ত প্লাগইনগুলো (যেমন, `eval`, `exec`, `system`) যাচাইকরণের সময় চিহ্নিত করা হয় এবং সেগুলো সক্রিয় করা যাবে না।
