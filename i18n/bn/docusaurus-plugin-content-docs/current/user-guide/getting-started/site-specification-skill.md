# সাইট স্পেসিফিকেশন দক্ষতা

**সাইট স্পেসিফিকেশন দক্ষতা** হলো আপনার সাইটের লক্ষ্য, দর্শক এবং ব্র্যান্ড পরিচয় ধারণ করার একটি সুসংগঠিত পদ্ধতি। এই তথ্যটি আপনার **site_brief** মেমরিতে সংরক্ষণ করা হয়, যা এজেন্টরা বিভিন্ন সেশনে ব্যবহার করে ধারাবাহিক এবং প্রাসঙ্গিক সহায়তা প্রদান করে।

## সাইট স্পেসিফিকেশন কী?

সাইট স্পেসিফিকেশন হলো নিম্নলিখিত বিষয়গুলো নথিভুক্ত করার প্রক্রিয়া:

- **সাইট উদ্দেশ্য (Site purpose)**: আপনার সাইট কী করে এবং কেন এটি বিদ্যমান
- **লক্ষ্য দর্শক (Target audience)**: কারা আপনার সাইট ভিজিট করে এবং তাদের কী প্রয়োজন
- **ব্র্যান্ড পরিচয় (Brand identity)**: আপনার রং, সুর এবং ভিজ্যুয়াল স্টাইল
- **ব্যবসায়িক লক্ষ্য (Business goals)**: আপনার সাইটের জন্য সাফল্য কেমন দেখায়
- **কন্টেন্ট কাঠামো (Content structure)**: আপনার সাইট কীভাবে সংগঠিত

এই স্পেসিফিকেশনটি আপনার **site_brief** হয়ে ওঠে—একটি স্থায়ী মেমরি যা এজেন্টরা আপনার সাইটের প্রেক্ষাপট বুঝতে ব্যবহার করে।

## সাইট স্পেসিফিকেশন কেন ব্যবহার করবেন?

### সেশন জুড়ে ধারাবাহিকতা (Consistency Across Sessions)

site_brief ছাড়া, প্রতিবার নতুন সেশন শুরু করার সময় আপনাকে আপনার সাইটের উদ্দেশ্য পুনরায় ব্যাখ্যা করতে হবে। এটি থাকলে, এজেন্টরা সাথে সাথেই বুঝতে পারে:

- আপনার সাইটের লক্ষ্য এবং দর্শক
- আপনার ব্র্যান্ডের রং এবং সুর
- আপনার কন্টেন্ট কাঠামো
- আপনার ব্যবসায়িক উদ্দেশ্য

### উন্নত সুপারিশ (Better Recommendations)

এজেন্টরা আপনার site_brief ব্যবহার করে:

- আপনার সাইটের উদ্দেশ্যের সাথে সামঞ্জস্যপূর্ণ বৈশিষ্ট্যগুলি পরামর্শ দিতে
- আপনার লক্ষ্যের সাথে মানানসই কন্টেন্ট কাঠামো সুপারিশ করতে
- আপনার ব্র্যান্ডের সাথে সামঞ্জস্যপূর্ণ ডিজাইন প্রস্তাব করতে
- অসামঞ্জস্যপূর্ণ বৈশিষ্ট্যগুলি পরামর্শ দেওয়া এড়াতে

### দ্রুত অনবোর্ডিং (Faster Onboarding)

নতুন এজেন্টরা (বা নতুন সেশনে থাকা এজেন্টরা) স্পষ্টীকরণ প্রশ্ন জিজ্ঞাসা করার পরিবর্তে আপনার site_brief পড়ে দ্রুত কাজ শুরু করতে পারে।

## সাইট স্পেসিফিকেশন শুরু করা

### থিম বিল্ডার অনবোর্ডিং চলাকালীন (During Theme Builder Onboarding)

**Theme Builder onboarding flow** চলাকালীন স্বয়ংক্রিয়ভাবে সাইট স্পেসিফিকেশন দক্ষতা শুরু হয়। Setup Assistant এজেন্ট প্রশ্ন জিজ্ঞাসা করে এবং আপনার site_brief তৈরি করে।

### ম্যানুয়াল শুরু (Manual Initiation)

আপনি যেকোনো সময় সাইট স্পেসিফিকেশন শুরু করতে পারেন:

```
"Let's define my site specification"
```

অথবা

```
"Help me create a site brief"
```

## সাইট স্পেসিফিকেশন প্রক্রিয়া

### ধাপ ১: সাইট উদ্দেশ্য (Site Purpose)

এজেন্ট জিজ্ঞাসা করে:

```
What is your site's primary purpose?
- E-commerce store
- Blog or content site
- Portfolio or showcase
- SaaS application
- Community or forum
- Other: [describe]
```

আপনি একটি ক্যাটাগরি নির্বাচন করতে পারেন বা আপনার নিজস্ব উদ্দেশ্য বর্ণনা করতে পারেন।

### ধাপ ২: লক্ষ্য দর্শক (Target Audience)

```
Who is your primary audience?
- Consumers / general public
- Business professionals
- Developers / technical users
- Students / educators
- Other: [describe]

What are their main needs?
```

### ধাপ ৩: ব্র্যান্ড পরিচয় (Brand Identity)

```
What are your brand colors?
- Primary color: [color picker or hex code]
- Secondary color: [color picker or hex code]
- Accent color: [optional]

How would you describe your brand tone?
- Professional / corporate
- Creative / artistic
- Playful / casual
- Minimal / modern
- Warm / friendly
```

### ধাপ ৪: ব্যবসায়িক লক্ষ্য (Business Goals)

```
What does success look like for your site?
- Generate leads
- Sell products
- Build community
- Share knowledge
- Establish authority
- Other: [describe]

What's your primary metric?
- Revenue
- User engagement
- Content reach
- Conversions
- Other
```

### ধাপ ৫: কন্টেন্ট কাঠামো (Content Structure)

```
How is your content organized?
- Flat (all content at same level)
- Hierarchical (categories and subcategories)
- Chronological (blog-style)
- Product-based (catalog)
- Other: [describe]

What content types do you use?
- Blog posts
- Product pages
- Case studies
- Documentation
- Videos
- Other
```

## আপনার site_brief মেমরি

সাইট স্পেসিফিকেশন সম্পন্ন করার পরে, আপনার তথ্য আপনার এজেন্টের মেমরিতে **site_brief** হিসাবে সংরক্ষণ করা হয়। এটি একটি সুসংগঠিত রেকর্ড যা ধারণ করে:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## আপনার site_brief দেখা এবং আপডেট করা

### আপনার site_brief দেখা

এজেন্টকে জিজ্ঞাসা করুন:

```
"Show me my site brief"
```

অথবা

```
"What do you know about my site?"
```

এজেন্ট আপনার সংরক্ষিত স্পেসিফিকেশনটি প্রদর্শন করবে।

### আপনার site_brief আপডেট করা

যদি আপনার সাইট পরিবর্তিত হয়, তবে আপনি এটি আপডেট করতে পারেন:

```
"Update my site brief: we're now targeting B2B customers"
```

অথবা

```
"Refresh my site specification"
```

এটি আপনার বিদ্যমান তথ্যকে শুরুর বিন্দু হিসেবে ব্যবহার করে সাইট স্পেসিফিকেশন দক্ষতাটি পুনরায় চালাবে।

## এজেন্টরা site_brief কীভাবে ব্যবহার করে

### ডিজাইন সুপারিশ (Design Recommendations)

যখন আপনি ডিজাইন পরিবর্তনের জন্য জিজ্ঞাসা করেন, এজেন্টরা আপনার site_brief-এর রেফারেন্স নেয়:

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### বৈশিষ্ট্য পরামর্শ (Feature Suggestions)

এজেন্টরা আপনার লক্ষ্যের সাথে সামঞ্জস্যপূর্ণ বৈশিষ্ট্যগুলি পরামর্শ দেয়:

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### কন্টেন্ট কাঠামো (Content Structure)

এজেন্টরা আপনার কাঠামোর উপর ভিত্তি করে কন্টেন্ট সংগঠনের প্রস্তাব দেয়:

```
You: "How should I organize my blog?"
Agent: "Since you use a product catalog with blog, I'd suggest
organizing posts by product category and creating 'styling guides'
that showcase multiple products together."
```

## সেরা অনুশীলন (Best Practices)

### নির্দিষ্ট হন (Be Specific)

"সাধারণ দর্শক"-এর পরিবর্তে আপনার প্রকৃত দর্শককে বর্ণনা করুন:

- ✓ "Women aged 25-45, interested in sustainable fashion"
- ✗ "Everyone"

### নিয়মিত আপডেট করুন (Update Regularly)

আপনার সাইট যত বিকশিত হয়, তত আপনার site_brief আপডেট করুন:

- যখন আপনি নতুন দর্শকের দিকে মনোযোগ দেন
- যখন আপনি নতুন পণ্য লাইন যোগ করেন
- যখন আপনার ব্র্যান্ড পরিচয় পরিবর্তিত হয়
- যখন আপনার ব্যবসায়িক লক্ষ্য পরিবর্তিত হয়

### সামঞ্জস্যপূর্ণ পরিভাষা ব্যবহার করুন (Use Consistent Terminology)

সেশন জুড়ে একই শব্দ ব্যবহার করুন:

- ✓ সর্বদা "sustainable jewelry" বলুন (নয় "eco-friendly jewelry" এবং "green products")
- ✓ আপনার দর্শকদের সর্বদা একই উপায়ে উল্লেখ করুন

### প্রেক্ষাপট অন্তর্ভুক্ত করুন (Include Context)

এমন পটভূমি প্রদান করুন যা এজেন্টদের আপনার সিদ্ধান্তগুলি বুঝতে সাহায্য করে:

- "আমরা এমন পেশাদারদের লক্ষ্য করছি যারা দামের চেয়ে গুণমানকে মূল্য দেন"
- "আমাদের দর্শক প্রযুক্তি-সচেতন এবং আধুনিক ডিজাইন আশা করে"
- "আমরা একটি বুটস্ট্র্যাপড স্টার্টআপ, তাই আমাদের সাশ্রয়ী সমাধান প্রয়োজন"

## থিম বিল্ডার অনবোর্ডিং-এর সাথে সম্পর্ক

সাইট স্পেসিফিকেশন দক্ষতাটি **Theme Builder onboarding flow**-এর সাথে একত্রিত। আপনি যখন অনবোর্ডিং সম্পন্ন করেন, তখন আপনি যে তথ্য দিয়েছিলেন তা দিয়ে স্বয়ংক্রিয়ভাবে আপনার site_brief তৈরি হয়।

আপনি যদি চান তবে আপনি স্বাধীনভাবে সাইট স্পেসিফিকেশন চালাতে পারেন:

- প্রাথমিক সেটআপের পরে আপনার স্পেসিফিকেশন পরিমার্জন করতে
- আপনার সাইট বিকশিত হওয়ার সাথে সাথে আপনার site brief আপডেট করতে
- Theme Builder শুরু করার আগে একটি বিস্তারিত স্পেসিফিকেশন তৈরি করতে

## সমস্যা সমাধান (Troubleshooting)

**আমার site_brief ব্যবহার করা হচ্ছে না**
- নিশ্চিত করুন যে এজেন্টের মেমরিতে অ্যাক্সেস আছে
- এজেন্টকে জিজ্ঞাসা করুন "recall my site brief"
- পরীক্ষা করুন যে আপনার সেটিংসে মেমরি সক্ষম করা আছে

**আমি একটি নতুন site_brief দিয়ে শুরু করতে চাই**
- এজেন্টকে জিজ্ঞাসা করুন: "Clear my site brief and start fresh"
- তারপর আবার সাইট স্পেসিফিকেশন চালান

**এজেন্ট আমার site_brief-এর সাথে মিলছে না এমন সুপারিশ করছে**
- এজেন্টকে জিজ্ঞাসা করুন "review my site brief"
- যদি এটি পুরোনো হয় তবে আপনার site_brief আপডেট করুন
- আপনার অনুরোধে অতিরিক্ত প্রেক্ষাপট প্রদান করুন

## পরবর্তী পদক্ষেপ (Next Steps)

আপনার সাইট স্পেসিফিকেশন সংজ্ঞায়িত করার পরে:

1. **Theme Builder ব্যবহার করুন**: আপনার site_brief-এর উপর ভিত্তি করে একটি কাস্টম থিম তৈরি করুন
2. **ডিজাইন পরিমার্জন করুন**: বিস্তারিত ডিজাইনের কাজের জন্য Design System Aesthetics skill ব্যবহার করুন
3. **কন্টেন্ট পরিকল্পনা করুন**: কন্টেন্ট কাঠামোর সুপারিশের জন্য এজেন্টদের জিজ্ঞাসা করুন
4. **বৈশিষ্ট্য তৈরি করুন**: আপনার ব্যবসায়িক লক্ষ্যের সাথে সামঞ্জস্যপূর্ণ বৈশিষ্ট্যগুলি অনুরোধ করুন
