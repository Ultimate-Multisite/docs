# Requisitos do Ultimate Multisite

A primeira coisa que você pode se perguntar antes de comprar o Ultimate Multisite é quais são os requisitos para rodar nosso plugin. Bem, felizmente, existem apenas alguns requisitos básicos para rodar o Ultimate Multisite: um **nome de domínio**, um bom **provedor de hospedagem** e uma instalação do WordPress **Multisite**. É só isso que você precisa!

## Nome de Domínio {#domain-name}

Vamos falar primeiro sobre o nome de domínio.

Ao escolher um nome de domínio, tenha em mente qual é o seu nicho específico e o que você irá criar com o Ultimate Multisite: Você quer criar uma agência? Vender sites tipo loja? Ou talvez hospedagem web WordPress? Tudo é possível com o Ultimate Multisite.

Nosso plugin fornece serviços de assinatura, coleta de pagamentos, formulários de checkout, cupons de desconto e comunicações com clientes. Uma grande parte do trabalho necessário para instalar, configurar e manter um WordPress Multisite é facilitada pelo Ultimate Multisite.

Qualquer nicho que você escolher pode ser gerenciado usando nosso plugin, desde que você venda sites como serviço! Certifique-se de escolher um nome de domínio que combine com o seu plano de negócios.

## Provedor de Hospedagem {#hosting-provider}

Agora que você tem um nome de domínio atraente que representa o seu nicho e negócio, você precisará escolher uma boa empresa de hospedagem para hospedar sua rede com o Ultimate Multisite. Existem algumas boas opções no mercado. Alguns provedores de hospedagem se integram bem com o Ultimate Multisite, o que pode ser um fator decisivo dependendo do que você planeja construir.

O Ultimate Multisite deve funcionar em qualquer provedor de hospedagem. A única diferença é que algumas empresas de hospedagem como _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ e _Runcloud_ têm uma integração mais apertada para mapeamento de domínio. Para outras hospedagens, você pode precisar fazer o mapeamento dos domínios personalizados manualmente. Também é uma boa ideia escolher um provedor de hospedagem que suporte AutoSSL. _Closte_, _Runcloud_ e _cPanel_ suportam isso.

Quando se trata das especificações do servidor, não há uma fórmula específica, pois isso dependerá dos recursos e funções que você oferecer e do tamanho da sua rede. Uma hospedagem compartilhada pode ser um bom começo, mas assim que sua rede começar a crescer, provavelmente você precisará de um VPS dedicado.

## WordPress Multisite {#wordpress-multisite}

Agora, vamos falar um pouco sobre uma instalação de WordPress Multisite.

O WordPress Multisite permite que você tenha uma rede de sites em uma única instalação. Este é um recurso nativo, mas não está ativo por padrão. Você precisa ativar este recurso dentro da sua instalação do WordPress e no painel de controle da hospedagem web.

Ter o WordPress com o Multisite ativado **é a base para o Ultimate Multisite**. Como o principal uso do Ultimate Multisite é ajudar os usuários a gerenciar e vender sites como serviço, isso precisa ser feito no WordPress com o Multisite ativado.

Temos um **artigo detalhado** que ensina passo a passo o que precisa ser feito para ativar o recurso Multisite na sua instalação do WordPress.

Depois de ativar o Multisite no WordPress, é hora de instalar o Ultimate Multisite. Para instalar corretamente o Ultimate Multisite, você deve ter:

  * WordPress: v5.3+ (Recomendado: Última versão estável)

  * PHP: 7.4.x (suporte para > 8.0 em breve)

  * MySQL: v5+ (Recomendado: 5.6, caso você não tenha o 8.0 disponível no seu provedor de hospedagem)

Estes são os únicos requisitos de software para o Ultimate Multisite.

Lembre-se que o Ultimate Multisite pode funcionar com PHP 8.0, mas recomendamos executá-lo com PHP 7.4.x.

Além disso, recomendamos instalá-lo nos **domínios principais**, e não em subdomínios ou subdiretórios. Pode funcionar em subdomínios/subdiretórios, mas pode apresentar alguns erros.
