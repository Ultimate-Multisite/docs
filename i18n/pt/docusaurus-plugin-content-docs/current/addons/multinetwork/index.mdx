# Multi-Network {#multi-network}

O complemento Ultimate Multisite: Multinetwork expande a sua plataforma Ultimate Multisite com poderosa funcionalidade multirrede. Este complemento permite que os seus clientes comprem e façam a gestão de redes multisite WordPress completas, não apenas sites individuais.

## Principais funcionalidades {#key-features}

* **Provisionamento de rede**: Crie e configure automaticamente redes multisite WordPress completas para os seus clientes
* **Modelos de rede**: Permita que os clientes explorem, pré-visualizem e escolham um modelo de rede pré-construído antes do registo
* **Gestão de rede**: Capacidades completas de gestão para ambientes multirrede
* **Redes isoladas**: Cada rede de cliente funciona de forma independente com as suas próprias tabelas de base de dados e configuração
* **SSO entre redes**: Dê aos clientes acesso sem interrupções às suas sub-redes com autenticação por ligação mágica
* **Arquitetura escalável**: Gira várias redes de forma eficiente com alocação de recursos otimizada
* **Integração**: Integra-se perfeitamente com os sistemas de subscrição e faturação do Ultimate Multisite

## Casos de utilização {#use-cases}

* **Alojamento White Label**: Forneça aos clientes as suas próprias redes multisite com marca própria
* **Soluções para agências**: Permita que agências giram várias redes de clientes a partir de uma única plataforma
* **Implementações empresariais**: Suporte grandes organizações com requisitos multirrede complexos
* **Plataformas de educação**: Crie redes separadas para diferentes escolas ou departamentos

## Requisitos {#requirements}

* Ultimate Multisite 2.0.0 ou superior
* Instalação WordPress Multisite
* PHP 7.4 ou superior
* MySQL 5.6 ou superior

## Instalação {#installation}

1. Carregue os ficheiros do plugin para o diretório `/wp-content/plugins/ultimate-multisite-multinetwork`
2. Ative o plugin através do ecrã 'Plugins' no WordPress
3. Configure as definições de multirrede através de Ultimate Multisite → Definições

## Modelos de rede {#network-templates}

A versão 1.3.0 adiciona um fluxo de descoberta e seleção de modelos para clientes que compram novas redes. Use modelos de rede quando quiser que cada nova rede de cliente comece a partir de uma estrutura preparada, design, biblioteca de conteúdo ou configuração específica de um setor.

### Pré-visualizador de modelos {#template-previewer}

O pré-visualizador de modelos de rede permite que os clientes explorem os modelos disponíveis antes do checkout. Cada modelo pode ser aberto num painel de pré-visualização ao vivo, para que os clientes possam comparar layouts e conteúdo antes de comprarem uma rede.

Use o pré-visualizador para ajudar os clientes a:

* Rever o aspeto e a estrutura de cada modelo de rede disponível
* Comparar vários modelos sem sair do fluxo de compra
* Escolher o modelo que melhor corresponde ao seu caso de utilização antes do registo

Mantenha os nomes e descrições dos modelos amigáveis para o cliente. O pré-visualizador é mais útil quando os clientes conseguem entender o caso de utilização pretendido, como redes de agências, redes escolares, redes de franchising ou pacotes de alojamento white-label.

### Seletor de modelos no checkout {#template-picker-at-checkout}

O seletor de modelos pode ser adicionado ao fluxo de checkout de registo como um campo. Quando configurado, os clientes selecionam o modelo de rede que pretendem durante o checkout, e o modelo selecionado é usado quando a sua nova rede é provisionada.

Para usar o seletor num fluxo de checkout:

1. Prepare as redes que pretende oferecer como modelos.
2. Ative o campo de seleção de modelo Multinetwork no formulário de registo ou checkout relevante.
3. Escolha quais modelos devem estar disponíveis para esse fluxo de checkout.
4. Publique o fluxo de checkout e teste um registo para confirmar que o modelo selecionado é aplicado à nova rede.

Se um fluxo de checkout oferecer apenas um produto de rede, o seletor ainda pode ser útil como uma etapa de integração guiada. Para funis com vários planos, exponha apenas os modelos que são compatíveis com o produto ou plano selecionado.

## SSO com ligação mágica entre redes {#cross-network-magic-link-sso}

O SSO com ligação mágica entre redes permite que os clientes se movam entre a sua área de conta principal e sub-redes sem iniciarem sessão manualmente outra vez. Quando um cliente segue uma ligação de acesso entre redes, o Ultimate Multisite gera uma ligação mágica limitada no tempo e autentica o cliente na rede de destino.

Isto melhora a experiência para clientes que gerem mais do que uma rede ou que precisam de aceder a redes de modelos recém-clonadas imediatamente após a compra.

Lista de verificação do administrador:

* Confirme que o complemento Multinetwork está ativo em toda a instalação multisite WordPress.
* Certifique-se de que os clientes têm o acesso de rede correto antes de lhes serem oferecidas ligações entre redes.
* Mantenha os domínios de rede e domínios mapeados corretamente configurados para que as ligações mágicas resolvam para a rede de destino pretendida.
* Teste o painel do cliente e quaisquer botões de acesso pós-checkout depois de alterar definições de rede, domínio ou checkout.

As ligações mágicas devem ser tratadas como ligações de início de sessão. Foram concebidas para acesso contínuo do cliente e não devem ser encaminhadas nem reutilizadas fora da sessão pretendida.

## Perguntas frequentes {#frequently-asked-questions}

### O que é multirrede? {#what-is-multinetwork}

Multirrede é uma funcionalidade do WordPress que permite executar várias redes multisite independentes numa única instalação WordPress.

### Em que é que isto é diferente de multisite normal? {#how-is-this-different-from-regular-multisite}

O multisite normal permite vários sites dentro de uma rede. A multirrede permite várias redes, cada uma com os seus próprios sites, todas dentro de uma única instalação WordPress.

### Isto requer configuração especial do servidor? {#does-this-require-special-server-configuration}

Sim, a multirrede requer configuração e preparação específicas do servidor. Consulte a documentação para obter instruções detalhadas de configuração.
