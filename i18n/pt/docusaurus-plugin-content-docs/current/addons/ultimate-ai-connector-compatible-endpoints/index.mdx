# Ultimate AI Connector para Pontos de Extremidade Compatíveis

O Ultimate AI Connector para Pontos de Extremidade Compatíveis integra APIs de fornecedores de IA compatíveis com OpenAI com a sua rede multisite WordPress. Configure vários fornecedores de IA — cada um com o seu próprio URL de ponto de extremidade, chave de API e modelo predefinido — e encaminhe pedidos com fallback automático quando um fornecedor estiver indisponível.

## Funcionalidades Principais {#key-features}

- **Suporte a vários fornecedores**: Adicione vários cartões de fornecedores de IA, cada um com o seu próprio URL de ponto de extremidade, chave de API e modelo predefinido
- **Arrastar para reordenar**: Organize os fornecedores por prioridade usando a pega em cada cartão; os pedidos usam os fornecedores por ordem
- **Encaminhamento de fallback automático**: Se o fornecedor principal estiver indisponível, os pedidos passam automaticamente para o próximo fornecedor configurado
- **Configuração por fornecedor**: Cada fornecedor é configurado de forma independente com as suas próprias credenciais e seleção de modelo
- **UI de definições baseada em cartões**: Construída sobre componentes Card estáveis do WordPress 6.9+ (CardBody, CardHeader, CardDivider)
- **Expansão automática ao adicionar**: Novos cartões de fornecedores expandem automaticamente para que possa preencher os detalhes de imediato
- **Invalidação da cache de scripts**: Os scripts dos fornecedores têm a cache invalidada na atualização da extensão para garantir que a versão mais recente é carregada

## Requisitos {#requirements}

- WordPress 5.3 ou superior
- PHP 7.4 ou superior
- Extensão Ultimate Multisite (ativa)
- Um ponto de extremidade de API compatível com OpenAI (autoalojado ou de terceiros)

## Instalação {#installation}

1. Carregue os ficheiros do complemento para o seu diretório `/wp-content/plugins/`
2. Ative a extensão através do menu 'Extensões' no WordPress
3. Navegue para **Ultimate Multisite → AI Connector** para configurar o complemento

## Configuração {#configuration}

### Configuração de Vários Fornecedores {#multi-provider-setup}

O ecrã de configuração usa um layout baseado em cartões, em que cada cartão representa um fornecedor de IA. Pode adicionar tantos fornecedores quantos precisar.

#### Adicionar um Fornecedor {#adding-a-provider}

1. Vá a **Ultimate Multisite → AI Connector** na administração da rede
2. Clique em **Adicionar Fornecedor**
3. Aparece um novo cartão de fornecedor e expande automaticamente
4. Preencha os detalhes do fornecedor:
   - **Nome do Fornecedor** — uma etiqueta para identificar este fornecedor (por exemplo, "OpenAI", "Mistral", "Local Ollama")
   - **URL do Ponto de Extremidade** — o URL base da API compatível com OpenAI (por exemplo, `https://api.openai.com/v1`)
   - **Chave de API** — a sua chave de API para este fornecedor
   - **Modelo Predefinido** — o identificador do modelo a usar por predefinição para este fornecedor (por exemplo, `gpt-4o`, `mistral-small`)
5. Clique em **Guardar Definições**

#### Remover um Fornecedor {#removing-a-provider}

Clique no botão **Remover** em qualquer cartão de fornecedor para o eliminar. Guarde as definições para aplicar a alteração.

#### Reordenar Fornecedores {#reordering-providers}

Arraste o ícone de pega (⣿) no lado esquerdo de qualquer cartão de fornecedor para o reordenar. A ordem determina a prioridade de fallback: os pedidos são enviados para o primeiro fornecedor e passam para fornecedores subsequentes se um pedido falhar.

### Encaminhamento de Fallback {#fallback-routing}

Quando um pedido ao fornecedor principal falha — devido a um tempo limite, limite de taxa ou interrupção do serviço — o conector tenta automaticamente novamente com o fornecedor seguinte na lista. Isto continua pela lista até que um fornecedor tenha sucesso ou todos os fornecedores tenham sido tentados.

| Cenário | Comportamento |
|----------|-----------|
| O fornecedor principal responde | O pedido é servido normalmente |
| O fornecedor principal falha | Pedido tentado novamente com o fornecedor seguinte |
| Todos os fornecedores falham | Erro devolvido ao chamador |

O fallback é transparente para a extensão ou tema que chama — recebem uma resposta bem-sucedida ou um erro final, independentemente de quantos fornecedores tenham sido tentados.

### Referência de Configuração do Fornecedor {#provider-configuration-reference}

| Campo | Obrigatório | Descrição |
|-------|----------|-------------|
| Nome do Fornecedor | Sim | Etiqueta de apresentação para este fornecedor |
| URL do Ponto de Extremidade | Sim | URL base da API compatível com OpenAI |
| Chave de API | Sim | Chave de autenticação para este fornecedor |
| Modelo Predefinido | Sim | Identificador do modelo usado quando nenhum modelo é especificado no pedido |

## Resolução de Problemas {#troubleshooting}

### Não há fornecedores listados {#no-providers-are-listed}

Se a lista de fornecedores estiver vazia depois de guardar, confirme que:

- Clicou em **Guardar Definições** depois de adicionar fornecedores
- O seu navegador não bloqueou a gravação da página de definições (verifique a consola do navegador para erros de JavaScript)
- A extensão está ativa e está no ecrã de administração da rede

### Os pedidos estão a falhar para todos os fornecedores {#requests-are-failing-for-all-providers}

- Verifique se o URL do ponto de extremidade de cada fornecedor está acessível a partir do seu servidor (não apenas a partir do seu navegador)
- Verifique se cada chave de API é válida e não expirou
- Reveja o estado do limite de taxa e da quota do fornecedor no respetivo Dashboard
- Ative o registo de depuração do WordPress (`WP_DEBUG_LOG`) para capturar mensagens de erro detalhadas do conector

### Um fornecedor não está a ser usado para fallback {#a-provider-is-not-being-used-for-fallback}

Arrastar para reordenar altera a ordem de fallback. Confirme que o fornecedor que espera que receba pedidos de fallback está corretamente posicionado na lista e que o respetivo URL do ponto de extremidade e chave de API estão corretos.

### Novo cartão de fornecedor não expande automaticamente {#new-provider-card-does-not-expand-automatically}

A expansão automática requer que o JavaScript esteja ativado no seu navegador. Se o cartão não expandir ao clicar, verifique a consola do navegador para erros. O conteúdo do cartão está sempre acessível clicando no cabeçalho do cartão, mesmo que a expansão automática não seja acionada.
