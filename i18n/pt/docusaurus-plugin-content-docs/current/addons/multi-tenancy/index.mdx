# Multi-inquilino

Distribua subsites entre servidores e bases de dados para desempenho e segurança

## Destaques da versão 1.2.0 {#version-120-release-highlights}

Multi-inquilino 1.2.0 adiciona a base de locatário soberano para fluxos de trabalho de isolamento em produção:

- Autologin de locatário sem estado com visitas SSO de curta duração e protegidas contra repetição.
- Isolamento de base de dados e sistema de ficheiros por subsite para locatários soberanos.
- Capacidade multi-inquilino da Hostinger para provisionamento de locatários com reconhecimento do host.
- Portões de verificação de migração soberana para verificações de dados legados e processamento push assíncrono.
- Vinculação configurável do host da base de dados na mesma máquina para instalações Bedrock, FrankenPHP e localhost.
- Melhorias no bootstrap do locatário, escritor da base de dados, bootstrap de esquema e provisionamento de utilizadores.
- Remoção soberana mais segura que limpa as credenciais do locatário.

Consulte os guias do utilizador para [fluxo de trabalho de migração soberana](/user-guide/getting-started/sovereign-migration-workflow), [isolamento multi-inquilino](/user-guide/configuration/multi-tenancy-isolation) e [operações de locatário soberano](/user-guide/administration/sovereign-tenant-operations).

## Requisitos {#requirements}

- WordPress 5.3 ou superior
- PHP 7.4 ou superior
- plugin Multisite Ultimate (ativo)

## Instalação {#installation}

1. Carregue os ficheiros do complemento para o seu diretório `/wp-content/plugins/`
2. Ative o plugin através do menu 'Plugins' no WordPress
3. Configure as definições do complemento na administração de rede

## Configuração {#configuration}

O complemento suporta várias opções de configuração acessíveis através da interface de administração de rede.

### Configuração de locatário soberano {#sovereign-tenant-configuration}

Para locatários soberanos, configure a base de dados do locatário, a raiz do sistema de ficheiros, a vinculação do host da base de dados e a capacidade do host antes de encaminhar tráfego para o locatário. Execute a verificação de migração após cada alteração que afete credenciais da base de dados, vinculação do host, encaminhamento ou provisionamento de utilizadores.

Use `localhost` apenas quando o runtime do locatário e as concessões da base de dados esperarem acesso por socket local ou pela mesma máquina. Use `127.0.0.1` ou um nome de host de serviço quando esse for o host de concessão configurado pelo seu fornecedor.

### Capacidade Hostinger {#hostinger-capability}

A capacidade multi-inquilino da Hostinger funciona com o token de integração Hostinger partilhado. Confirme que o domínio está presente no hPanel, que o DNS aponta para a Account da Hostinger e que as definições da base de dados do locatário correspondem ao hPanel antes da verificação.

## Hooks e filtros {#hooks-and-filters}

### Ações {#actions}

- `ultimate-multisite-addon-template_init` - Acionado após a inicialização do complemento
- `ultimate-multisite-addon-template_loaded` - Acionado após todos os ficheiros do complemento serem carregados

### Filtros {#filters}

- `ultimate-multisite-addon-template_settings` - Modificar definições do complemento
- `ultimate-multisite-addon-template_enabled` - Substituir o estado de ativação/desativação do complemento
