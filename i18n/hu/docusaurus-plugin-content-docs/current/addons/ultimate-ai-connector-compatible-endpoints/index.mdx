# Ultimate AI Connector Compatible Endpontokhoz

Az Ultimate AI Connector Compatible Endpontokhoz integrálja az OpenAI-kompatibilis AI szolgáltató API-endpointjeit a WordPress multisite hálózatába. Konfigurálhat több AI szolgáltatót is — mindegyiknek saját endpoint URL-je, API kulcsa és alapértelmezett modellje van — és automatikus fallback (visszaesítés) mechanizmussal irányítja a kéréseket, ha egy szolgáltató elérhetetlen.

## Főbb funkciók

- **Multi-provider támogatás**: Ad hozzá több AI szolgáltató kártyáját is, mindegyik saját endpoint URL-jával, API kulcsaival és alapértelmezett modelljével.
- **Húzással újrarendelhető**: A szolgáltatókat a kártyán lévő fogantyú segítségével sorrendben rendezheti; a kérések a szolgáltatókat a sorrendben használják.
- **Automatikus fallback (visszaesítés) irányítás**: Ha az elsődleges szolgáltató elérhetetlen, a kérések automatikusan átirányítódnak a következő konfigurált szolgáltatóra.
- **Szolgáltató별 konfiguráció**: Minden szolgáltatót függetlenül konfigurálhatja saját hitelesítő adatai és modellválasztása alapján.
- **Kártyán alapú beállítási felhasználói felület**: Stabil WordPress 6.9+ Card komponensekre épült (CardBody, CardHeader, CardDivider).
- **Automatikus kibővítés hozzáadáskor**: Az új szolgáltató kártyák automatikusan kibővülnek, így azonnal kitölthetők a részletek.
- **Script cache busting**: A szolgáltató scriptjeit frissítéssel (plugin update) cache-bustolják, hogy biztosítsák, hogy a legfrissebb verzió töltődjön be.

## Feltételek

- WordPress 5.3 vagy magasabb
- PHP 7.4 vagy magasabb
- Ultimate Multisite plugin (aktív)
- Egy OpenAI-kompatibilis API endpoint (önként üzemeltetett vagy harmadik fél által biztosított)

## Telepítés

1. Töltse fel az addon fájlokat a `/wp-content/plugins/` könyvtárába.
2. Aktiválja a plugin-t a WordPress 'Plugins' menüpontján keresztül.
3. Navigáljon az **Ultimate Multisite → AI Connector** menüpontra a konfigurációhoz.

## Konfiguráció

### Multi-Provider Beállítás

A konfigurációs képernyő kártyán alapuló elrendezést használ, ahol minden kártya egy AI szolgáltatót képvisel. Annyi szolgáltatót ad hozzá, amennyit szüksége.

#### Szolgáltató hozzáadása

1. Navigáljon az **Ultimate Multisite → AI Connector** menüpontra a hálózat admin felületén.
2. Kattintson az **Add Provider** gombra.
3. Megjelenik és automatikusan kibővül egy új szolgáltató kártyája.
4. Töltse ki a szolgáltató részleteit:
   - **Provider Name** — egy címke, amely azonosítja a szolgáltatót (pl. "OpenAI", "Mistral", "Local Ollama").
   - **Endpoint URL** — az OpenAI-kompatibilis API alap URL-je (pl. `https://api.openai.com/v1`).
   - **API Key** — az Ön API kulcsa ehhez a szolgáltatóhoz.
   - **Default Model** — a modell azonosító, amelyet alapértelmezésként kell használni ehhez a szolgáltatóhoz (pl. `gpt-4o`, `mistral-small`).
5. Kattintson a **Save Settings** gombra.

#### Szolgáltató eltávolítása

Kattintson az **Remove** gombra bármely szolgáltató kártyáján, hogy törölje. Mentse el a beállításokat, hogy alkalmazza a változást.

#### Szolgáltatók újrarendelése

Húzza el a fogantyú ikonot (⣿) bármely szolgáltató kártyájának bal oldaláról, hogy újrarendelje. A sorrend határozza meg a fallback prioritását: a kéréseket az első szolgáltatóhoz küldik, és ha a kérés sikertelen, a következő szolgáltatókra fallback-ot biztosít.

### Fallback (Visszaesítés) Irányítása

Ha a kérés sikertelen a primary (elsődleges) szolgáltatóhoz — például időtúllépés, rate limit vagy szolgáltatás kiesés miatt — a connector automatikusan megpróbálja újra a lista következő szolgáltatójával. Ez a folyamat a listán végig halad, amíg egy szolgáltató sikeres választ ad, vagy amíg minden szolgáltatót megpróbáltak.

| Szénarió | Viselkedés |
|----------|-----------|
| A primary szolgáltató válaszol | A kérés normálisan teljesül |
| A primary szolgáltató sikertelen | A kérést a következő szolgáltatóval próbálják újra |
| Minden szolgáltató sikertelen | Hibaüzenet érkezik a hívóhoz |

A fallback átlátszó a hívó plugin vagy témája számára — függetlenül attól, hogy hány szolgáltatót próbáltak ki, sikeres választ vagy végleges hibaüzenetet kapnak.

### Szolgáltató Konfigurációs Referencia

| Field | Required | Description |
|-------|----------|-------------|
| Provider Name | Yes | Megjelenítő címke ehhez a szolgáltatóhoz |
| Endpoint URL | Yes | Az OpenAI-kompatibilis API alap URL-je |
| API Key | Yes | Hitelesítő kulcs ehhez a szolgáltatóhoz |
| Default Model | Yes | A modell azonosító, amelyet használatban lennak, ha a kérésben nincs megadva modell |

## Hibaelhárítás

### Nincs listázott szolgáltató

Ha az adatok mentése után üres a szolgáltatólista, kérjük, ellenőrizze, hogy:

- Kattintott-e a **Save Settings** gombra a szolgáltatók hozzáadásán utána.
- Nem blokkolta a böngészője a beállítások mentését (nézze meg a böngésző konzolját JavaScript hibaüzenetek után).
- Aktív a plugin, és a hálózat admin felületén van.

### A kérések minden szolgáltatóhoz sikertelenek

- Ellenőrizze, hogy minden szolgáltató endpoint URL-je elérhető-e a szerveréről (nem csak a böngészőjéből).
- Nézze át, hogy minden API kulcs érvényes és nem lejárt.
- Vizsgálja meg a szolgáltató rate-limit és kvóta státuszát a dashboard-jában.
- Aktiválja a WordPress debug logging-ot (`WP_DEBUG_LOG`) a részletes hibaüzenetek rögzítéséhez a connector-től.

### Egy szolgáltatót nem használják fallback-hoz

A húzással újrarendelése megváltoztatja a fallback sorrendet. Kérjük, erősítse meg, hogy a fallback kéréseket váró szolgáltató megfelelően pozícionált a listában, és hogy az endpoint URL-je és API kulcsa helyes.

### Az új szolgáltató kártyája nem bővül automatikusan

Az auto-expand funkció JavaScript elindítását igényel a böngészőjében. Ha a kártya nem bővül kattintással, ellenőrizze a böngésző konzolját hibák szempontjából. A kártya tartalma mindig elérhető a kártya fejlécének kattintásával, még akkor is, ha az auto-expand nem aktiválódik.
