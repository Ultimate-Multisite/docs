# Mailchimp

Multisite Ultimate: Mailchimp ඒකාබද්ධ කිරීම

## API යතුරක් ලබා ගැනීම {#getting-an-api-key}

ලබා ගත හැකි විවිධ සැකසුම් ගැන ආරම්භ කිරීමට පෙර, අපට MailChimp API යතුරක් අවශ්‍ය වේ. මෙය Ultimate Multisite ට MailChimp API සමඟ සන්නිවේදනය කර ලැයිස්තු සහ කණ්ඩායම් ලබා ගැනීමටත්, පරිශීලකයින් ලියාපදිංචි වන විට ඔවුන්ගේ විද්‍යුත් තැපැල් ලිපින ඒ ලැයිස්තු සහ කණ්ඩායම් වෙත එක් කිරීමටත් ඉඩ දෙයි.

ඔබේම MailChimp API යතුර ලබා ගැනීමට මෙම උපදේශනය අනුගමනය කරන්න: [API යතුරු ගැන | MailChimp](https://kb.mailchimp.com/integrations/api-integrations/about-api-keys)

## ඒකාබද්ධ කිරීමේ ප්‍රකාරය {#integration-mode}

MailChimp ඒකාබද්ධ කිරීමේ add-on එක “ඒකාබද්ධ කිරීමේ ප්‍රකාර” දෙකක් ලබා දෙයි. එනම්, විද්‍යුත් තැපැල් ලිපින සමමුහුර්ත කිරීමට ඔබට add-on එක භාවිත කළ හැකි වෙනස් ක්‍රම දෙකක් ඇත.

### ප්‍රකාරය 1: බහු ලැයිස්තු {#mode-1-multiple-lists}

පෙරනිමි ප්‍රකාරය **බහු ලැයිස්තු** භාවිත කරයි. මෙම ප්‍රකාරය මඟින් එක් එක් සැලසුම සඳහා MailChimp ලැයිස්තු කිහිපයක් තෝරා ගැනීමට ඔබට ඉඩ ලැබේ. ඔබේ පරිශීලකයින් වෙනස් ලැයිස්තුවලට බෙදා වෙන් කරන්නේ නම් මෙය හොඳම විකල්පයයි.

උදාහරණයක් ලෙස, ඔබේ MailChimp Account එකේ වෙනස් ලැයිස්තු 3ක් ඇතැයි සිතමු: **පුවත් ලිපිය** , **සැලසුම A** , සහ **සැලසුම B**. ඔවුන්ගේ සැලසුම කුමක් වුවද, සියලු පරිශීලකයින් පුවත් ලිපි ලැයිස්තුවටද ඇතුළත් විය යුතුය. එවිට ඔවුන්ට ඔබේ යාවත්කාලීන කිරීම් මඟ හැරෙන්නේ නැත.

එය ලබා ගැනීමට, ඔබට බහු ලැයිස්තු ප්‍රකාරය භාවිත කර **පුවත් ලිපිය** සහ **සැලසුම A** යන ලැයිස්තු දෙක සැලසුම A සඳහා වූ MailChimp සැකසුම් වෙත එක් කළ හැක. එසේම **පුවත් ලිපිය** සහ **සැලසුම B** සැලසුම B සඳහා වූ MailChimp සැකසුම් වෙත එක් කළ හැක.

### ප්‍රකාරය 2: තනි ලැයිස්තුව + බහු කණ්ඩායම් {#mode-2-single-list--multiple-groups}

MailChimp ඔබේ ලැයිස්තුව **කණ්ඩායම්** ලෙස හැඳින්වෙන පරිශීලකයින්ගේ කුඩා කණ්ඩායම්වලට බෙදා වෙන් කිරීමට ඉඩ දෙයි. මෙයින් ඔබට වැඩි පාලනයක් ලැබේ. එනම්, MailChimp හි එකම ව්‍යාපාරයේ පිටපත් කිහිපයක් සෑදීමකින් තොරව, ඔබේ දායකයින්ගේ නිශ්චිත උපකණ්ඩායමකට ව්‍යාපාරයක් යැවිය හැක. එසේ පිටපත් කිහිපයක් සෑදීමෙන් පසුව එම ව්‍යාපාරයේ මිනුම් විශ්ලේෂණය කිරීම අපහසු වේ.

උදාහරණයක් ලෙස, ඔබට **මගේ ජාල දායකයින්,** ලෙස නම් කළ තනි ලැයිස්තුවක් තිබිය හැකි අතර, එහි ඇතුළත **සැලසුම A** සහ **සැලසුම B** සඳහා කණ්ඩායම් තිබිය හැක.

## පෙරනිමි ලැයිස්තු සහ කණ්ඩායම් {#default-lists-and-groups}

ඔබේ විශේෂිත භාවිත අවස්ථාවට වඩාත් සුදුසු ඒකාබද්ධ කිරීමේ ප්‍රකාරය තෝරා ගත් පසු, පෙරනිමි ලැයිස්තු (සහ ප්‍රකාරය අනුව කණ්ඩායම්) තෝරා ගැනීමට කාලයයි. දී ඇති සැලසුමකට නිශ්චිත ලැයිස්තුවක් තෝරා නැත්නම් පෙරනිමි ලැයිස්තුව භාවිත කෙරේ.

අවලංගු කළ ගිණුම් සඳහා පෙරනිමි ලැයිස්තුවද ඔබට තෝරා ගත හැක. පරිශීලකයෙකු තම දායකත්වය අවසන් කර ඔබේ ජාලයෙන් තම ගිණුම ඉවත් කරන සෑම විටම, ඔහුගේ හෝ ඇයගේ විද්‍යුත් තැපැල් ලිපිනය අවලංගු කළ ගිණුම් සඳහා සකසා ඇති ලැයිස්තුවට ගෙන යනු ඇත.

## ද්විත්ව දායකත්වය සක්‍රිය කිරීම {#enabling-double-subscription}

එම සැකසුම් පිටුවේම, ඔබට **ද්විත්ව දායකත්වය** සක්‍රිය කළ හැක. මෙම විකල්පය පරිශීලකයා පෙරනිමි ලෙස තෝරා ඇති ලැයිස්තුවට සහ එම පරිශීලකයාගේ සැලසුමට තෝරා ඇති ලැයිස්තුවට දෙකටම එක් කරයි. මෙම විකල්පය සක්‍රිය කර නොමැති නම්, පරිශීලකයා එක් කරනු ලබන්නේ පරිශීලකයාගේ සැලසුම සඳහා තෝරා ඇති ලැයිස්තුවට පමණි. එම විශේෂිත සැලසුම සඳහා කිසිවක් සකසා නැත්නම්, පරිශීලකයා තෝරා ඇති පෙරනිමි ලැයිස්තුවට එක් කෙරේ.
