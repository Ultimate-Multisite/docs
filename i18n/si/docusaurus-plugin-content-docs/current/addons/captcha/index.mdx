# Captcha {#captcha}

**බොට් නවත්වන්න. ආදායම ආරක්ෂා කරන්න. සැකසුම් කරදර නැත.**

අභිරුචි ඩොමේන සමඟ බහු-අඩවි ජාලයක් ධාවනය කරනවාද? ඔබට වේදනාව දන්නවා: Google reCAPTCHA සහ hCaptcha වල පරිපාලක කොන්සෝලයේ *සෑම තනි ඩොමේනයක්ම* ලියාපදිංචි කළ යුතුය. නව පාරිභෝගික අඩවියක් එක් කරනවාද? captcha සැකසුම යාවත්කාලීන කරන්න. අභිරුචි ඩොමේනයක් සිතියම්ගත කරනවාද? නැවත යාවත්කාලීන කරන්න. එය කවදාවත් අවසන් නොවේ.

**Ultimate Multisite: Captcha සියල්ල වෙනස් කරයි.**

## Cap Captcha: ස්වයං-සත්කාරක ආරක්ෂාව, සරලව ක්‍රියා කරයි {#cap-captcha-self-hosted-protection-that-just-works}

අපගේ නව **Cap Captcha** සැපයුම්කරු බහු-අඩවි ජාල සඳහා විශාල වෙනසක් ගෙන එයි:

* **සැකසුම් ශූන්‍යයි** — addon සක්‍රීය කරන්න, ඔබ ආරක්ෂිතයි. API keys නැත, Google ගිණුමක් නැත, hCaptcha dashboard නැත, ඩොමේන whitelist කිරීමක් නැත.
* **සෑම ඩොමේනයකම ස්වයංක්‍රීයව ක්‍රියා කරයි** — උපඩොමේන, සිතියම්ගත කළ ඩොමේන, පාරිභෝගික ඩොමේන—සියල්ල ක්ෂණිකව ආරක්ෂා වේ.
* **ස්වයං-සත්කාරක සහ පුද්ගලිකයි** — ඔබේ දත්ත ඔබේ සේවාදායකයෙන් පිටතට කවදාවත් යන්නේ නැත. තෙවන පාර්ශ්ව ලුහුබැඳීමක් නැතිව සම්පූර්ණයෙන් GDPR අනුකූලයි.
* **අදෘශ්‍ය Proof-of-Work** — පරිශීලකයන් පසුබිමේ ගණනීය ප්‍රහේලිකාවක් විසඳයි. ගිනි නිවන නළ ක්ලික් කිරීමක් නැත, කලකිරවන රූප ජාල නැත.
* **සැහැල්ලුයි** — 20KB පමණයි. ඔබේ checkout මන්දගාමී නොකරයි.

## WooCommerce කාඩ් පරීක්ෂණ ආරක්ෂාව {#woocommerce-card-testing-protection}

කාඩ් පරීක්ෂණ ප්‍රහාර වෙළෙන්දන්ට වාර්ෂිකව බිලියන ගණනක වියදම් ඇති කරයි. වංචාකරුවන් සොරකම් කළ ණය කාඩ්පත් ඔබේ WooCommerce checkout හරහා වලංගුදැයි පරීක්ෂා කිරීමට බොට් භාවිත කරයි—සැකසුම් ගාස්තු ඔබ ගෙවයි, ඔවුන්ට වලංගු කාඩ් අංක ලැබේ.

**අපි වැදගත්ම තැන් ආරක්ෂා කරමු:**

* **Store API ආරක්ෂාව** — සාම්ප්‍රදායික පෝරම captchas මඟහැරෙන `/wp-json/wc/store/v1/checkout` මත බොට් ප්‍රහාර අවහිර කරයි
* **PayPal Payments අනුකූලයි** — WooCommerce PayPal Payments සමඟ බාධාවකින් තොරව ක්‍රියා කරයි
* **තත්කාලීන සංඛ්‍යාලේඛන** — ඔබ අවහිර කරන ප්‍රහාර ගණන හරියටම බලන්න

## අනුපාත සීමා කිරීම {#rate-limiting}

Ultimate Multisite: Captcha v1.5.0 ඔබේ ජාලය brute-force සහ කාඩ්-පරීක්ෂණ ප්‍රහාරවලින් ආරක්ෂා කරන දැඩි-නවතුම් අනුපාත සීමාකාරකයක් හඳුන්වා දෙයි.

### එය ක්‍රියා කරන ආකාරය {#how-it-works}

අනුපාත සීමාකාරකය captcha-ආරක්ෂිත පෘෂ්ඨ මත සෑම GET සහ POST ඉල්ලීමක්ම ගණනය කරයි:

* **WordPress පිවිසුම් අවසාන ලක්ෂ්‍ය** — wp-login, register, lost-password, comments
* **WooCommerce checkout** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

අමුත්තෙක් අනුපාත සීමාව ඉක්මවූ විට, plugin මෙසේ ප්‍රතිචාර දක්වයි:

* **HTTP 429 (ඉල්ලීම් වැඩියි)** තත්ත්ව කේතය
* **Retry-After** header, නැවත උත්සාහ කළ යුතු වේලාව පෙන්වයි
* ප්‍රහාරකයන් මන්දගාමී කිරීමට **අහඹු tarpit නින්ද** (පෙරනිමියෙන් තත්පර 1–5, තත්පර 15 ක දැඩි සීමාවක් ඇත)

### සැකසුම {#configuration}

#### Tarpit කාලය {#tarpit-timing}

අනුපාත-සීමා කළ ඉල්ලීම්වලට යෙදෙන අහඹු ප්‍රමාදය පාලනය කරන්න:

* **`cap_rate_limit_tarpit_min`** — අවම නින්ද කාලය තත්පරවලින් (පෙරනිමි: 1)
* **`cap_rate_limit_tarpit_max`** — උපරිම නින්ද කාලය තත්පරවලින් (පෙරනිමි: 5, 15 දක්වා දැඩිව සීමා කර ඇත)

මෙම සැකසුම් **Ultimate Multisite → Settings → Captcha** තුළ තිබේ.

### දිගු කළ හැකියාව {#extensibility}

#### IP Whitelist පෙරහන {#ip-whitelist-filter}

විශ්වාසදායක IP පරාස අනුපාත සීමා කිරීමෙන් නිදහස් කරන්න:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // Whitelist your monitoring service
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### අවහිර කිරීමට පෙර ක්‍රියාව {#pre-block-action}

අනුපාත-සීමා අවහිරයක් යැවීමට වහාම පෙර අභිරුචි තර්කය ක්‍රියාත්මක කරන්න:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // Log the block, notify admins, or update custom metrics
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## Client IP හඳුනාගැනීම සහ විශ්වාසදායක Proxies {#client-ip-detection--trusted-proxies}

Ultimate Multisite: Captcha v1.5.0 ව්‍යාජකරණයට-ප්‍රතිරෝධී client-IP හඳුනාගැනීම හඳුන්වා දෙයි. `Captcha_Core::get_client_ip()` ක්‍රමය දැන් පහත සියල්ල පුරා අමුත්තන්ගේ IP ආරෝපණය සඳහා එකම සත්‍ය මූලාශ්‍රය වේ:

* අනුපාත-සීමා බකට් යතුරු
* කැප්චා සපයන්නාගේ `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* සංඛ්‍යාලේඛන IP හැෂ්

### විශ්වාස ආකෘතිය {#trust-model}

ප්ලගිනය දැඩි IP විශ්වාස ධුරාවලියක් ක්‍රියාත්මක කරයි:

1. **REMOTE_ADDR** — සෑම විටම විශ්වාසයි (ක්ෂණික සම්බන්ධ පාර්ශ්වයේ IP)
2. **CF-Connecting-IP** — පිළිගන්නේ මෙවැනි විට පමණි:
   - ක්ෂණික සම්බන්ධ පාර්ශ්වය වත්මන් Cloudflare IP පරාසයක් තුළ සිටින විට
   - `cap_trust_cloudflare_headers` සක්‍රීය කර ඇති විට (පෙරනිමිය: OFF)
3. **X-Forwarded-For** — පිළිගන්නේ මෙවැනි විට පමණි:
   - ක්ෂණික සම්බන්ධ පාර්ශ්වය ඔබේ පරිපාලක-සැකසූ විශ්වාසිත ප්‍රොක්සි ලැයිස්තුවේ සිටින විට
   - ශීර්ෂකය දකුණේ සිට වමට විග්‍රහ කර, විශ්වාසිත/Cloudflare හොප් මඟහරින විට

### වින්‍යාසය {#configuration-1}

#### Cloudflare ශීර්ෂක විශ්වාසය {#cloudflare-header-trust}

Cloudflare හි `CF-Connecting-IP` ශීර්ෂකය විශ්වාස කිරීම සක්‍රීය කරන්න:

* **`cap_trust_cloudflare_headers`** — `CF-Connecting-IP` විශ්වාසයට ඇතුළත් වීමට ON ලෙස සකසන්න
  - පෙරනිමිය: OFF (අක්‍රීය)
  - ප්ලගිනයට එකතු කළ Cloudflare CIDR ස්නැප්ෂොට් එකක් සමඟ පැමිණේ
  - CIDR පරාස wp-cron හරහා සතිපතා අලුත් කරයි
  - අලුත් කිරීම අසාර්ථක වුවහොත් එකතු කළ fallback භාවිත වේ

#### විශ්වාසිත ප්‍රොක්සි වින්‍යාසය {#trusted-proxy-configuration}

ඔබේ ඉදිරි පෙළ ප්‍රොක්සි සහ load-balancers නිර්වචනය කරන්න:

* **`cap_trusted_proxies`** — CIDR හෝ සරල IP සඳහා පාඨ ප්‍රදේශයක් (පේළියකට එකක්)
  - අදහස් දැක්වීම් අවසර ඇත (`#` සමඟ ආරම්භ වන පේළි)
  - මෙම සැකසුම නැතිනම්, rate limiter සක්‍රීය වුවත් `X-Forwarded-For` නොසලකා හැරේ
  - උදාහරණය:
    ```
    # Our load balancer
    192.0.2.0/24

    # Backup proxy
    198.51.100.50
    ```

#### පළමු සක්‍රීය කිරීමේ ස්වයං හඳුනාගැනීම {#first-enable-auto-detection}

ඔබ rate limiter පළමුවරට සක්‍රීය කරන විට, ප්ලගිනය ඔබේ සම්භාව්‍ය Cloudflare සහ ප්‍රොක්සි තත්ත්වය හඳුනාගෙන, එක්-ක්ලික් "හඳුනාගත් සැකසුම් යොදන්න" පරිපාලක දැනුම්දීමක් පෙන්වයි.

* ප්ලගිනය ඔබ සුරැකි අගයන් **කිසිවිටෙකත් උඩින් ලියන්නේ නැත**
* පසුව එන ගමනාගමනය ඔබේ වින්‍යාසය තවදුරටත් සැබෑ තත්ත්වයට නොගැළපෙන බව පෙන්වන්නේ නම් (උදා., Cloudflare CIDR පරාස වෙනස් කළහොත්), ඉවත් කළ නොහැකි නොගැළපුම් දැනුම්දීමක් නිර්දේශිත යාවත්කාලීනය පෙන්වයි

### මෙය වැදගත් ඇයි {#why-this-matters}

දැඩි IP විශ්වාස ආකෘතියක්, ව්‍යාජ `CF-Connecting-IP` ශීර්ෂකයක් රැගෙන එන සෘජු origin-server ඉල්ලීම් වැනි තීරණාත්මක වංචා මාර්ගයක් වසා දමයි. පෙර එවැනි ඉල්ලීම් සැබෑ සම්බන්ධ පාර්ශ්වය වෙනුවට ව්‍යාජ IP අනුව බකට් කළ හැකි විය. මෙය විශේෂයෙන් වැදගත් වන්නේ:
