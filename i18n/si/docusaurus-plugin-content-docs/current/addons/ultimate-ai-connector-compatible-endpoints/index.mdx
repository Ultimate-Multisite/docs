# අනුකූල අවසාන-ලක්ෂ්‍ය සඳහා Ultimate AI Connector

අනුකූල අවසාන-ලක්ෂ්‍ය සඳහා Ultimate AI Connector ඔබේ WordPress බහු-අඩවි ජාලය සමඟ OpenAI-අනුකූල AI සැපයුම්කරු API ඒකාබද්ධ කරයි. AI සැපයුම්කරුවන් කිහිපයක් සකසන්න — එක් එක් එකට තමන්ගේම අවසාන-ලක්ෂ්‍ය URL, API යතුර, සහ පෙරනිමි ආකෘතිය ඇත — සැපයුම්කරුවෙකු ලබාගත නොහැකි විට ස්වයංක්‍රීය ආපසු-මාරුව සමඟ ඉල්ලීම් යොමු කරන්න.

## ප්‍රධාන විශේෂාංග {#key-features}

- **බහු-සැපයුම්කරු සහාය**: එක් එක් එකට තමන්ගේම අවසාන-ලක්ෂ්‍ය URL, API යතුර, සහ පෙරනිමි ආකෘතිය ඇති AI සැපයුම්කරු කාඩ්පත් කිහිපයක් එක් කරන්න
- **ඇදගෙන-නැවත අනුපිළිවෙළට තැබීම**: එක් එක් කාඩ්පතේ ඇති ග්‍රිප් හැන්ඩලය භාවිතයෙන් සැපයුම්කරුවන් ප්‍රමුඛතාව අනුව සකසන්න; ඉල්ලීම් එම අනුපිළිවෙළට සැපයුම්කරුවන් භාවිත කරයි
- **ස්වයංක්‍රීය ආපසු-මාරු යොමුකිරීම**: ප්‍රධාන සැපයුම්කරු ලබාගත නොහැකි නම්, ඉල්ලීම් ස්වයංක්‍රීයව ඊළඟ සකසා ඇති සැපයුම්කරු වෙත ආපසු මාරු වේ
- **එක් එක් සැපයුම්කරු සඳහා සැකසුම්**: එක් එක් සැපයුම්කරු තමන්ගේම පිවිසුම් තොරතුරු සහ ආකෘති තේරීම සමඟ ස්වාධීනව සකසා ඇත
- **කාඩ්පත්-පදනම් සැකසුම් UI**: ස්ථාවර WordPress 6.9+ Card සංරචක (CardBody, CardHeader, CardDivider) මත නිර්මාණය කර ඇත
- **එක් කිරීමේදී ස්වයංක්‍රීයව විහිදීම**: නව සැපයුම්කරු කාඩ්පත් ස්වයංක්‍රීයව විහිදෙන නිසා ඔබට වහාම විස්තර පිරවිය හැක
- **Script cache busting**: නවතම අනුවාදය පූරණය වීම සහතික කිරීමට plugin යාවත්කාලීන කිරීමේදී සැපයුම්කරු scripts cache-bust කරයි

## අවශ්‍යතා {#requirements}

- WordPress 5.3 හෝ ඉහළ
- PHP 7.4 හෝ ඉහළ
- Ultimate Multisite plugin (සක්‍රිය)
- OpenAI-අනුකූල API අවසාන-ලක්ෂ්‍යයක් (ස්වයං-සත්කාරක හෝ තෙවන පාර්ශ්ව)

## ස්ථාපනය {#installation}

1. addon ගොනු ඔබේ `/wp-content/plugins/` නාමාවලියට උඩුගත කරන්න
2. WordPress හි 'Plugins' මෙනුව හරහා plugin සක්‍රිය කරන්න
3. addon සකස් කිරීමට **Ultimate Multisite → AI Connector** වෙත යන්න

## සැකසුම {#configuration}

### බහු-සැපයුම්කරු සැකසුම {#multi-provider-setup}

සැකසුම් තිරය කාඩ්පත්-පදනම් සැලසුමක් භාවිත කරයි. එහි එක් එක් කාඩ්පත එක් AI සැපයුම්කරුවෙකු නියෝජනය කරයි. ඔබට අවශ්‍ය තරම් සැපයුම්කරුවන් එක් කළ හැක.

#### සැපයුම්කරුවෙකු එක් කිරීම {#adding-a-provider}

1. ජාල පරිපාලකයේ **Ultimate Multisite → AI Connector** වෙත යන්න
2. **සැපයුම්කරු එක් කරන්න** ක්ලික් කරන්න
3. නව සැපයුම්කරු කාඩ්පතක් පෙනී ස්වයංක්‍රීයව විහිදේ
4. සැපයුම්කරු විස්තර පුරවන්න:
   - **සැපයුම්කරු නම** — මෙම සැපයුම්කරු හඳුනා ගැනීමට ලේබලයක් (උදා. "OpenAI", "Mistral", "Local Ollama")
   - **අවසාන-ලක්ෂ්‍ය URL** — OpenAI-අනුකූල API හි මූලික URL (උදා. `https://api.openai.com/v1`)
   - **API යතුර** — මෙම සැපයුම්කරු සඳහා ඔබේ API යතුර
   - **පෙරනිමි ආකෘතිය** — මෙම සැපයුම්කරු සඳහා පෙරනිමියෙන් භාවිත කළ යුතු ආකෘති හඳුනාගැනීමේ නාමය (උදා. `gpt-4o`, `mistral-small`)
5. **සැකසුම් සුරකින්න** ක්ලික් කරන්න

#### සැපයුම්කරුවෙකු ඉවත් කිරීම {#removing-a-provider}

එය මකා දැමීමට ඕනෑම සැපයුම්කරු කාඩ්පතක **ඉවත් කරන්න** බොත්තම ක්ලික් කරන්න. වෙනස යෙදීමට සැකසුම් සුරකින්න.

#### සැපයුම්කරුවන් නැවත අනුපිළිවෙළට තැබීම {#reordering-providers}

එය නැවත අනුපිළිවෙළට තැබීමට ඕනෑම සැපයුම්කරු කාඩ්පතක වම් පැත්තේ ඇති ග්‍රිප් සංකේතය (⣿) ඇදගෙන යන්න. අනුපිළිවෙළ ආපසු-මාරු ප්‍රමුඛතාව තීරණය කරයි: ඉල්ලීම් පළමු සැපයුම්කරු වෙත යවනු ලබන අතර ඉල්ලීමක් අසාර්ථක වුවහොත් පසුව ඇති සැපයුම්කරුවන් වෙත ආපසු මාරු වේ.

### ආපසු-මාරු යොමුකිරීම {#fallback-routing}

ප්‍රධාන සැපයුම්කරු වෙත ඉල්ලීමක් අසාර්ථක වූ විට — කල් ඉකුත්වීමක්, අනුපාත සීමාවක්, හෝ සේවා ඇණහිටීමක් නිසා — connector ලැයිස්තුවේ ඊළඟ සැපයුම්කරු සමඟ ස්වයංක්‍රීයව නැවත උත්සාහ කරයි. සැපයුම්කරුවෙකු සාර්ථක වන තුරු හෝ සියලු සැපයුම්කරුවන් උත්සාහ කර අවසන් වන තුරු මෙය ලැයිස්තුව දිගේ දිගටම සිදුවේ.

|----------|-----------|
| ප්‍රධාන සැපයුම්කරු ප්‍රතිචාර දක්වයි | ඉල්ලීම සාමාන්‍ය ලෙස සේවය කරයි |
| ප්‍රධාන සැපයුම්කරු අසාර්ථක වේ | ඊළඟ සැපයුම්කරු සමඟ ඉල්ලීම නැවත උත්සාහ කරයි |
| සියලු සැපයුම්කරුවන් අසාර්ථක වේ | දෝෂය කැඳවන්නා වෙත ආපසු ලබා දෙයි |

ආපසු-මාරුව කැඳවන plugin හෝ theme වෙත විනිවිද පෙනේ — කොපමණ සැපයුම්කරුවන් උත්සාහ කළත්, ඔවුන්ට සාර්ථක ප්‍රතිචාරයක් හෝ අවසාන දෝෂයක් ලැබේ.

### සැපයුම්කරු සැකසුම් යොමුව {#provider-configuration-reference}

## දෝෂ නිරාකරණය {#troubleshooting}

### සැපයුම්කරුවන් කිසිවක් ලැයිස්තුගත වී නැත {#no-providers-are-listed}

සුරැකීමෙන් පසු සැපයුම්කරු ලැයිස්තුව හිස් නම්, මෙය තහවුරු කරන්න:

- සැපයුම්කරුවන් එක් කිරීමෙන් පසු ඔබ **සැකසුම් සුරකින්න** ක්ලික් කළා
- ඔබේ browser සැකසුම් පිටුව සුරැකීම අවහිර කළේ නැත (JavaScript දෝෂ සඳහා browser console පරීක්ෂා කරන්න)
- plugin සක්‍රිය වන අතර ඔබ ජාල පරිපාලක තිරයේ සිටී

### සියලු සැපයුම්කරුවන් සඳහා ඉල්ලීම් අසාර්ථක වේ {#requests-are-failing-for-all-providers}

- එක් එක් සැපයුම්කරුගේ අවසාන-ලක්ෂ්‍ය URL ඔබේ server වෙතින් ළඟා විය හැකි බව තහවුරු කරන්න (ඔබේ browser වෙතින් පමණක් නොවේ)
- එක් එක් API යතුර වලංගු බව සහ කල් ඉකුත් වී නොමැති බව පරීක්ෂා කරන්න
- ඔවුන්ගේ dashboard තුළ සැපයුම්කරුගේ අනුපාත-සීමා සහ කෝටා තත්ත්වය සමාලෝචනය කරන්න
- connector වෙතින් විස්තරාත්මක දෝෂ පණිවිඩ ලබාගැනීමට WordPress debug logging (`WP_DEBUG_LOG`) සක්‍රිය කරන්න

### සැපයුම්කරුවෙකු ආපසු-මාරුව සඳහා භාවිත නොවේ {#a-provider-is-not-being-used-for-fallback}

ඇදගෙන-නැවත අනුපිළිවෙළට තැබීම ආපසු-මාරු අනුපිළිවෙළ වෙනස් කරයි. ආපසු-මාරු ඉල්ලීම් ලබා ගැනීමට ඔබ බලාපොරොත්තු වන සැපයුම්කරු ලැයිස්තුවේ නිවැරදිව ස්ථානගත වී ඇති බවත්, එහි අවසාන-ලක්ෂ්‍ය URL සහ API යතුර නිවැරදි බවත් තහවුරු කරන්න.

### නව සැපයුම්කරු කාඩ්පත ස්වයංක්‍රීයව විහිදෙන්නේ නැත {#new-provider-card-does-not-expand-automatically}

ස්වයංක්‍රීය විහිදීම සඳහා ඔබේ browser තුළ JavaScript සක්‍රිය විය යුතුය. ක්ලික් කිරීමේදී කාඩ්පත විහිදෙන්නේ නැත්නම්, දෝෂ සඳහා browser console පරීක්ෂා කරන්න. ස්වයංක්‍රීය විහිදීම ක්‍රියාත්මක නොවුණත්, කාඩ්පත් header ක්ලික් කිරීමෙන් කාඩ්පත් අන්තර්ගතය සෑම විටම ප්‍රවේශ විය හැක.
