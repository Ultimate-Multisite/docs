# පාඩම 7: එය ඔබගේම කරගැනීම

ඔබගේ පාරිභෝගිකයන්ට තමන් "කිසියම් WordPress plugin එකක්" භාවිතා කරන බව කිසිවිටෙකත් හැඟෙන්න නොදිය යුතුය. ඔවුන්ට FitSite භාවිතා කරන බව හැඟිය යුතුය -- ඔවුන්ගේ කර්මාන්තය සඳහා ගොඩනැගූ වේදිකාවක්. මෙම පාඩම සන්නාමකරණය, white-labeling, සහ වේදිකාව නිෂ්පාදනයක් ලෙස හැඟෙන පරිදි සකස් කිරීම ආවරණය කරයි.

## අපි නතර වූ තැන {#where-we-left-off}

FitSite සතුව යෝග්‍යතා ස්ටුඩියෝ හිමිකරුවන් සැලසුම් තේරීමේ සිට සජීවී අඩවියක් දක්වා ගෙන යන ක්‍රියාත්මක checkout ප්‍රවාහයක් ඇත. දැන් අපි සම්පූර්ණ අත්දැකීම එකට බැඳුණු, සන්නාමගත නිෂ්පාදනයක් ලෙස හැඟෙන පරිදි සකස් කරමු.

## ඔබගේ වේදිකා වසම {#your-platform-domain}

ඔබගේ සන්නාමයේ පදනම ඔබගේ වසමයි. FitSite සඳහා:

- **ප්‍රධාන වසම**: `fitsite.com` (ඔබගේ අලෙවිකරණ අඩවිය සහ ජාල මූලය)
- **පාරිභෝගික අඩවි**: `studioname.fitsite.com` (උපවසම්)
- **අභිරුචි වසම්**: Growth සහ Pro සැලසුම්වල පාරිභෝගිකයන්ට තමන්ගේම වසමක් සම්බන්ධ කළ හැක

### ඔබගේ වසම සැකසීම {#setting-up-your-domain}

1. ඔබගේ වේදිකා වසම ලියාපදිංචි කරන්න
2. එය ඔබගේ hosting සපයන්නා වෙත යොමු කරන්න
3. පාරිභෝගික උපවසම් සඳහා wildcard DNS (`*.fitsite.com`) වින්‍යාස කරන්න
4. wildcard SSL සක්‍රිය බව තහවුරු කරන්න

විස්තරාත්මක උපදෙස් සඳහා [වසම් සම්බන්ධ කිරීම වින්‍යාස කරන ආකාරය](/user-guide/domain-mapping/how-to-configure-domain-mapping) බලන්න.

## Admin අත්දැකීම White-Label කිරීම {#white-labeling-the-admin-experience}

යෝග්‍යතා ස්ටුඩියෝ හිමිකරුවෙකු තම අඩවි Dashboard වෙත පිවිසෙන විට, ඔවුන්ට WordPress හෝ Ultimate Multisite සන්නාමකරණය නොව ඔබගේ සන්නාමය පෙනිය යුතුය.

### අභිරුචි පිවිසුම් පිටුව {#custom-login-page}

WordPress පිවිසුම් පිටුව පහත දේවල් පෙන්වන පරිදි අභිරුචිකරණය කරන්න:

- ඔබගේ FitSite ලාංඡනය
- යෝග්‍යතාවයට ගැළපෙන පසුබිම් රූප
- ඔබගේ සන්නාම වර්ණ

### Dashboard සන්නාමකරණය {#dashboard-branding}

[Admin Page Creator](/addons/admin-page-creator) addon එක හෝ අභිරුචි CSS භාවිතා කර:

- WordPress ලාංඡනය ඔබගේ FitSite ලාංඡනයෙන් ප්‍රතිස්ථාපනය කරන්න
- ඔබගේ සන්නාමයට ගැළපෙන පරිදි admin වර්ණ රටාව අභිරුචිකරණය කරන්න

### අභිරුචි Admin පිටු {#custom-admin-pages}

යෝග්‍යතා ස්ටුඩියෝ හිමිකරුවන්ට වඩාත් අදාළ ක්‍රියා ඉදිරිපත් කරන අභිරුචි admin පිටු සෑදීම සලකා බලන්න:

- "පුහුණුකරුවන්ගේ පැතිකඩ යාවත්කාලීන කරන්න"
- "ඔබගේ අඩවිය බලන්න"

මෙය සාමාන්‍ය WordPress මෙනුවේ ඒවා සඟවා තැබීම වෙනුවට, නිකේතනයට අදාළ ක්‍රියා ඉදිරියට සහ මධ්‍යයට ගෙන එමින් ඉගෙනීමේ බාධාව අඩු කරයි.

## පාරිභෝගිකයන් සඳහා අභිරුචි වසම {#custom-domain-for-customers}

අභිරුචි වසම් ඇතුළත් සැලසුම්වල සිටින පාරිභෝගිකයන් සඳහා ක්‍රියාවලිය පැහැදිලිව ලේඛනගත කරන්න:

1. පාරිභෝගිකයා තම වසම registrar කෙනෙකු වෙත ලියාපදිංචි කරයි හෝ මාරු කරයි
2. පාරිභෝගිකයා DNS ඔබගේ වේදිකාව වෙත යොමු කරන ලෙස යාවත්කාලීන කරයි (නිශ්චිත records ලබා දෙන්න)
3. Ultimate Multisite වසම් සම්බන්ධ කිරීම සහ SSL හසුරුවයි

මෙම ක්‍රියාවලිය සඳහා විශේෂයෙන්, තාක්ෂණික නොවන යෝග්‍යතා ස්ටුඩියෝ හිමිකරුවන් සඳහා ලියන ලද උපකාරක ලිපියක් හෝ දැනුම් පදනම් ඇතුළත් කිරීමක් සාදන්න.
