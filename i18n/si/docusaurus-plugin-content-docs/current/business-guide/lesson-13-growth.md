# පාඩම 13: විශාල පරිමාණයට වර්ධනය කිරීම {#lesson-13-scaling-up}

ඔබට මුදල් ගෙවන ගනුදෙනුකරුවන් සහිත ක්‍රියාත්මක වේදිකාවක් ඇත. මෙම පාඩම කුඩා මෙහෙයුමකින් තිරසාර ව්‍යාපාරයකට වර්ධනය වන්නේ කෙසේද යන්න ආවරණය කරයි -- යටිතල පහසුකම් පරිමාණය කිරීම, මෙහෙයුම් ස්වයංක්‍රීය කිරීම, සහ ගනුදෙනුකරුවෙකුගෙන් ලැබෙන ආදායම වැඩි කිරීම.

## අප නතර වූ තැන {#where-we-left-off}

FitSite සජීවීව ඇත, ගනුදෙනුකරුවන් ලියාපදිංචි වෙමින් සිටිති, සහ ඔබ දෛනික මෙහෙයුම් පවත්වාගෙන යමින් සිටී. දැන් අපගේ අවධානය වර්ධනය වෙතයි.

## ඔබගේ සංඛ්‍යා දැනගන්න {#know-your-numbers}

පරිමාණය කිරීමට පෙර, ඔබ සිටින තත්ත්වය තේරුම් ගන්න:

### ප්‍රධාන මිනුම් {#key-metrics}

- **MRR (Monthly Recurring Revenue)**: මුළු මාසික subscription ආදායම
- **Customer count**: මුළු සක්‍රීය subscribers සංඛ්‍යාව
- **ARPU (Average Revenue Per User)**: MRR ගනුදෙනුකරු සංඛ්‍යාවෙන් බෙදූ අගය
- **Churn rate**: සෑම මාසයකම අවලංගු කරන ගනුදෙනුකරුවන්ගේ ප්‍රතිශතය
- **LTV (Lifetime Value)**: මුළු subscription කාලය පුරා ගනුදෙනුකරුවෙකුගෙන් ලැබෙන සාමාන්‍ය ආදායම
- **CAC (Customer Acquisition Cost)**: එක් ගනුදෙනුකරුවෙකු ලබාගැනීමට වැය වන සාමාන්‍ය පිරිවැය

### උදාහරණය: ගනුදෙනුකරුවන් 50ක් ඇති FitSite {#example-fitsite-at-50-customers}

| මිනුම | අගය |
|--------|-------|
| ගනුදෙනුකරුවන් | 50 (30 Starter, 15 Growth, 5 Pro) |
| MRR | $4,450 ($1,470 + $1,485 + $995 + $500 order bumps) |
| ARPU | $89/මාසය |
| මාසික churn | 4% (අවලංගු කිරීම් 2ක්/මාසය) |
| LTV | $89 x 25 මාස = $2,225 |

මෙම සංඛ්‍යා ඔබට කුමක් කෙරෙහි අවධානය යොමු කළ යුතුදැයි කියයි. Churn ඉහළද? රඳවාගැනීම සකසන්න. ARPU අඩුද? upgrades ප්‍රවර්ධනය කරන්න. CAC ඉහළද? ලබාගැනීමේ නාලිකා ප්‍රශස්ත කරන්න.

## යටිතල පහසුකම් පරිමාණය කිරීම {#scaling-infrastructure}

### කවදා පරිමාණය කළ යුතුද {#when-to-scale}

Hosting පරිමාණය කරන්න, එවිට:

- පිටු පූරණ කාලය සැලකිය යුතු ලෙස වැඩි වේ
- Server CPU හෝ memory භාවිතය නිතරම 70% ඉක්මවයි
- ඔබ සක්‍රීය sites 100+ වෙත ළඟා වෙමින් සිටී
- වේගය පිළිබඳ ගනුදෙනුකරු පැමිණිලි වැඩි වේ

### කෙසේ පරිමාණය කළ යුතුද {#how-to-scale}

- **Vertical scaling**: විශාල server එකකට upgrade කරන්න (වැඩි CPU, RAM)
- **Caching layers**: object caching සඳහා Redis/Memcached එකතු කරන්න, static content සඳහා page caching
- **CDN**: දැනටමත් Cloudflare හෝ සමාන දෙයක් භාවිත නොකරන්නේ නම්, static assets සඳහා CDN එකක් එකතු කරන්න
- **Database optimization**: network එක වර්ධනය වන විට, database queries මන්දගාමී වේ. tables ප්‍රශස්ත කරන්න, indexes එකතු කරන්න, වෙන් කළ database server එකක් සලකා බලන්න.
- **Separate concerns**: media storage, object storage වෙත මාරු කරන්න (S3-compatible), email transactional email service එකකට offload කරන්න

### Hosting මාරු කිරීම {#hosting-migration}

ඔබගේ වත්මන් host තවදුරටත් පරිමාණය කළ නොහැකි නම්, මාරු කිරීමක් සැලසුම් කරන්න:

1. නව environment එක සකසන්න
2. ඔබගේ network පිටපතක් සමඟ හොඳින් පරීක්ෂා කරන්න
3. traffic අඩු පැයවලදී migration සැලසුම් කරන්න
4. පෙර සිට අවම TTL සමඟ DNS යාවත්කාලීන කරන්න
5. migration පසු සියල්ල ක්‍රියා කරන බව තහවුරු කරන්න

## මෙහෙයුම් ස්වයංක්‍රීය කිරීම {#automating-operations}

ඔබ වර්ධනය වන විට, අතින් කරන ක්‍රියාවලි bottlenecks බවට පත්වේ. හැකි දේ ස්වයංක්‍රීය කරන්න:

### Webhooks සහ Zapier {#webhooks-and-zapier}

ස්වයංක්‍රීය කිරීමට [Webhooks](/user-guide/integrations/webhooks) හෝ [Zapier](/user-guide/integrations/zapier) භාවිතා කරන්න:

- **නව ලියාපදිංචි වීමේ දැනුම්දීම්** → Slack channel හෝ CRM
- **අවලංගු කිරීමේ ඇඟවීම්** → නැවත-දිනාගැනීමේ email sequence එකක් trigger කරන්න
- **ගෙවීම් අසාර්ථකවීම්** → ඔබගේ monitoring tool එකේ ඇඟවීමක්
- **Plan upgrades** → නව feature guide සමඟ සුබපැතුම් email

### Email ස්වයංක්‍රීය කිරීම {#email-automation}

අතින් යවන emails සිට ස්වයංක්‍රීය sequences වෙත මාරු වන්න:

- Onboarding sequence (දැනටමත් පාඩම 8 තුළ සාදා ඇත)
- අක්‍රීය ගනුදෙනුකරුවන් සඳහා re-engagement sequence
- ගනුදෙනුකරුවන් plan limits වෙත ළඟා වන විට upgrade prompts
- වාර්ෂික subscribers සඳහා renewal reminders

### Support ස්වයංක්‍රීය කිරීම {#support-automation}

- පොදු ප්‍රශ්න සඳහා **සූදානම් ප්‍රතිචාර**
- support tickets ලැබුණු බව පිළිගන්නා **ස්වයංක්‍රීය පිළිතුරු**
- ගනුදෙනුකරුවන් පවතින ලිපිවලට ගැළපෙන tickets ඉදිරිපත් කරන විට **දැනුම් පදනම් යෝජනා**

## ආදායම වැඩි කිරීම {#increasing-revenue}

වර්ධනය යනු තවත් ගනුදෙනුකරුවන් පමණක් නොවේ. එය එක් ගනුදෙනුකරුවෙකුගෙන් වැඩි ආදායමක් ලැබීම පිළිබඳවද වේ.

### පවතින ගනුදෙනුකරුවන්ට වැඩි අගයක් විකිණීම {#upselling-existing-customers}

- **Plan upgrades**: Starter ගනුදෙනුකරුවන්ට Growth/Pro features පෙන්වන ඉලක්කගත campaigns
- **Order bumps**: email හරහා පවතින ගනුදෙනුකරුවන්ට add-on products ප්‍රවර්ධනය කරන්න
- **වාර්ෂික පරිවර්තනය**: මාසික ගනුදෙනුකරුවන්ට වාර්ෂික billing වෙත මාරු වීමට discount එකක් ලබා දෙන්න

### නව ආදායම් ධාරා {#new-revenue-streams}

- **ඔබ වෙනුවෙන් සිදු කරන සැකසුම**: ගනුදෙනුකරුවෙකුගේ site එක ඔවුන් වෙනුවෙන් සකසා customize කිරීමට premium එකක් අය කරන්න
- **අභිරුචි design services**: template එකට අමතරව විශේෂ design වැඩ පිරිනමන්න
- **පුහුණු සැසි**: ප්‍රායෝගික උදව් අවශ්‍ය ගනුදෙනුකරුවන් සඳහා ගෙවීම් සහිත එක-එක walkthroughs
- **Premium plugins**: ගෙවීම් සහිත add-ons ලෙස niche-specific premium plugins පිරිනමන්න (උදා., fitness class booking widget එකක්)

### මිල ඉහළ දැමීම {#raising-prices}

ඔබගේ වේදිකාව පරිණත වී අගය එකතු කරන විට:

- පවතින ගනුදෙනුකරුවන් ඔවුන්ගේ වත්මන් මිලටම තබන්න
- නව ලියාපදිංචිවීම් සඳහා මිල ඉහළ දමන්න
- නව features සහ වැඩිදියුණු කිරීම් මඟින් වැඩිවීම් සාධාරණීකරණය කරන්න

## කණ්ඩායමක් ගොඩනැගීම {#building-a-team}

කොයි වෙලාවක හෝ, ඔබට සියල්ල තනිවම කළ නොහැක. සාමාන්‍ය පළමු බඳවාගැනීම්:

1. **Support person**: දෛනික ගනුදෙනුකරු ප්‍රශ්න සලසයි (ආරම්භයේදී අර්ධකාලීන)
2. **Content creator**: දැනුම් පදනම් ලිපි, blog posts, සහ marketing content ලියයි
3. **Designer**: templates වැඩිදියුණු කර නව ඒවා නිර්මාණය කරයි

ඔබට සේවකයින් අවශ්‍ය නැත. Contractors සහ freelancers වේදිකා ව්‍යාපාරයක් සඳහා හොඳින් ක්‍රියා කරයි.

## මෙම පාඩමෙන් අප ගොඩනැගූ දේ {#what-we-built-this-lesson}

- ව්‍යාපාර සෞඛ්‍යය තේරුම් ගැනීමට **මිනුම් රාමුවක්**
- sites දුසිම් ගණනකින් සිය ගණනකට වර්ධනය වීමට **යටිතල පහසුකම් පරිමාණ සැලැස්මක්**
- support, email, සහ මෙහෙයුම් සඳහා **ස්වයංක්‍රීය කිරීමේ උපක්‍රම**
- නව ගනුදෙනුකරුවන් ලබාගැනීම පමණක් නොව ඊට එහා **ආදායම් වර්ධන ක්‍රම**
- තනි මෙහෙයුම ඉක්මවා වැඩෙන විට **කණ්ඩායම් ගොඩනැගීමේ මාර්ගෝපදේශ**
- සෑම අදියරක් සඳහාම අවධානය යොමු කළ යුතු ක්ෂේත්‍ර සමඟ **වර්ධන සන්ධිස්ථාන**
