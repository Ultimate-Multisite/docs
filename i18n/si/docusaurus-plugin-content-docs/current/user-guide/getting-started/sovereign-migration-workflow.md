# ස්වෛරී සංක්‍රමණ වැඩ ප්‍රවාහය

Ultimate Multisite: Multi-Tenancy 1.2.0 මගින් සම්මත උපසයිට් එකක් ස්වෛරී ටෙනන්ට් යටිතල ව්‍යුහයන් තුළ ගෙන යාමේදී සංක්‍රමණ සත්‍යාපන දොරටු (migration verification gates) එක් කරයි.

## ආරම්භ කිරීමට පෙර {#before-you-start}

ඔබ පහත දේ තහවුරු කර ඇති බව තහවුරු කරන්න:

- මූලාශ්‍ර වෙබ් අඩවියේ වත්මන් පිටපතක් (backup).
- Multi-Tenancy addon ක්‍රියාත්මක වීම.
- ගමනාන්ත ටෙනන්ට් දත්ත සමුදාය, ගොනු පද්ධතියේ මුල (filesystem root), සහ ડોමේනය සූදානම් වීම.
- ගමනාගාත පරිසරයට ගැලපෙන දත්ත සමුදා සත්කාරක බැඳීමක් (database host binding).
- ජාලය සඳහා WP-CLI විධාන ක්‍රියාත්මක කිරීමට ප්‍රවේශය.

## නිර්දේශිත වැඩ ප්‍රවාහය {#recommended-workflow}

1. ගමනාන්ත ටෙනන්ට් දත්ත සමුදාය සහ ගොනු පද්ධතිය සූදානම් කරන්න.
2. ටෙනන්ට් වෙන් කිරීමේ සැකසුම් (tenant isolation settings) ලියාපදිංචි කරන්න හෝ යාවත්කාලීන කරන්න.
3. ටෙනන්ට් සංක්‍රමණය (migration) ක්‍රියාත්මක කරන්න.
4. ටෙනන්ට් පරිශීලකයන් සූදානම් කරන්න හෝ තහවුරු කරන්න.
5. අසමකාලීන සංක්‍රමණ රැකියා (async migration jobs) හි ඇති දේ ඉවත් කරන්න (drain).
6. ස්වෛරී සංක්‍රමණ සත්‍යාපනය (sovereign migration verification) ක්‍රියාත්මක කරන්න.
7. SSO සමඟ ටෙනන්ට් එකට පිවිසෙන්න.
8. සත්‍යාපනය සාර්ථක වූ පසු පමණක් DNS හෝ රවුටින් වෙනස් කරන්න.

## සත්‍යාපන දොරටු (Verification gates) {#verification-gates}

සත්‍යාපන වැඩ ප්‍රවාහය සංක්‍රමණය විවිධ කෝණවලින් පරීක්ෂා කරයි:

- ටෙනන්ට් ස්කීමාව (schema) පවතීද සහ එය ටෙනන්ට් දත්ත සමුදායේ ලියන්නට හැකිද.
- වින්‍යාස කළ දත්ත සමුදා සත්කාරකය වල වලංගු අනුමැතියක් තිබේද යන්න.
- ටෙනන්ට් පරිශීලකයන් සිටිනවාද සහ අප බලාපොරොත්තු වන ස්වෛරී ස්ථාපන පරිශීලක ගණනට (expected sovereign install user count) ගැලපේද.
- අසමකාලීන යැවීම් පෝලිම (async push queue) සාර්ථකව ඉවත් කළ හැකිද.
- ස්වෛරී ටෙනන්ට් සඳහා පැරණි ජාල-පහළ දත්ත මාර්ග (Legacy network-side data paths) අවශ්‍ය නොවේ.

සත්‍යාපන අසාර්ථකවීම් ආරම්භ කිරීමට පෙර බාධක ලෙස සලකන්න. වාර්තා වූ දත්ත සමුදා, පරිශීලක, පෝලිම හෝ රවුටින් ගැටලුව නිවැරදි කර, නැවත ටෙනන්ට් එක පාරිභෝගිකයින්ට හෙළීමට පෙර සත්‍යාපනය ක්‍රියාත්මක කරන්න.

## පළමු නිෂ්පාදන පිවිසුම (First production visit) {#first-production-visit}

සත්‍යාපනය සාර්ථක වූ පසු, පළමු පරිපාලක පිවිසුම සඳහා වෙබ් අඩවි කළමනාකරණ තිරයේ ඇති **Visit (SSO)** භාවිතා කරන්න. මෙය එක් පාලිත පියවරකින් ටෙනන්ට් රවුටින්, SSO ටෝකන් හැසිරීම්, ආරම්භක ස්ථානය (origin pinning) සහ ටෙනන්ට්-පහළ පරිශීලක සැපයුම තහවුරු කරයි.
