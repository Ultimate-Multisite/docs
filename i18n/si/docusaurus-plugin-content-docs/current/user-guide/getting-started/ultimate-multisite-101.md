# Ultimate Multisite පිළිබඳ මූලික කරුණු

Ultimate Multisite කියන්නේ WordPress Multisite plugin එකක්. මේකෙන් ඔබට පාරිභෝගිකයින්ට WaaS (Website as a Service) හෝ Websites as a Service ලබා දීමට හැකියාව ලැබෙනවා. අපි මේ ගැන ගැඹුරින් බලන්න කලින්, Ultimate Multisite ඔබේ ව්‍යාපාරයට සහ පාරිභෝගිකයින්ට උදව් කරන්නේ කොහොමද කියලා දැනගන්න නම් අපිට මූලික දැනුමක් අවශ්‍යයි.

## WordPress Multisite එක {#the-wordpress-multisite}

අපි හැමෝම සාමාන්‍ය WordPress ස්ථාපනය ගැන හුරුපුරුදුයි. ඔබ ඒක ඔබේ hosting provider ගේ control panel එකෙන් නිර්මාණය කර ගන්නවා, නැත්නම් නිර්භීතයන් නම් අලුත් web server එකක් සහ database එකක් සකස් කරලා core files download කරලා installation process පටන් ගන්නවා.

මේක ලොව පුරා මිලියන ගණනක් WordPress වෙබ් අඩවි සඳහා වැඩ කරනවා. නමුත් an agency එකක් හෝ hosting provider එකක් දෘෂ්ටිකෝණයෙන් බලද්දී, මේ වෙබ් අඩවි වල ප්‍රමාණය ගැන අපි ටිකක් කතා කරමු.

ඔබේ automated control panel එකකින් එක් WordPress site එකක් හෝ සිය ගණනක් නිර්මාණය කිරීම පහසු වුවත්, මේ වෙබ් අඩවි කළමනාකරණයට පැවරෙන විට ඉක්මනින්ම ගැටළු පෙන්වීමට පටන් ගන්නවා. කළමනාකරණය නොකළොත්, ඒවා මැලවීමට ප්‍රධාන ඉලක්කය වෙනවා malware වලට. කළමනාකරණය කිරීම කියන්නේ උත්සාහයක් සහ සම්පත් අවශ්‍ය වන දෙයක්. WordPress sites කළමනාකරණය සහ පරිපාලනය පහසු කිරීමට බාහිර මෙවලම් සහ plugins තිබුණත්, පාරිභෝගිකයින් පරිපාලන ප්‍රවේශය තබා ගැනීම නිසා මේ උත්සාහයන් ඉතා පහසුවෙන් අභිබවා යා හැකියි.

එහි core එක තුළ WordPress වල 'Multisite' නමින් විශේෂාංගයක් තිබෙනවා. ඒක 2010 දී WordPress 3.0 දියත් කිරීම සමඟ ආරම්භ වූ දෙයක මූලාරම්භය දක්වා දිවෙනවා. එතැන් පස්සේ අලුත් features හඳුන්වා දීම සහ ආරක්ෂාව තහවුරු කිරීම සඳහා සංශෝධන කිහිපයක් සිදු කර තිබෙනවා.

සාරාංශයක් ලෙස, WordPress multisite එකක් මෙලෙස සිතිය හැකියි: විශ්වවිද්‍යාලයක් එක් WordPress ස්ථාපනයක් පවත්වාගෙන යන නමුත් සෑම පීඨයකටම තමන්ගේම WordPress site එකක් පවත්වාගෙන යයි.

### ජාලය (The Network) {#the-network}

WordPress සම්බන්ධයෙන් ගත්තොත්, multisite network එකක් කියන්නේ එක් dashboard එකකින් subsites කිහිපයක් කළමනාකරණය කරන්න පුළුවන් තැනක්. hosting providers අතර multisite network එකක් නිර්මාණය කිරීමේදී වෙනස්කම් තිබුණත්, අවසාන ප්‍රතිඵලය තමයි WordPressට මේ විශේෂිත මාදිලියේ ක්‍රියාත්මක වන බව දැනුම් දීම සඳහා wp-config.php ගොනුවේ අමතර උපදෙස් කිහිපයක් අවශ්‍ය වීම.

multisite network එකක් සහ තනි WordPress ස්ථාපනයක් අතර පැහැදිලි වෙනස්කම් තිබෙනවා, අපි ඒ ගැන කෙටියෙන් සාකච්ඡා කරමු.

#### Subdomain vs. Subdirectory (සබ්ඩොමේන් වුවත් සබ්ඩික්ටරි) {#subdomain-vs-subdirectory}

ඔබ කළ යුතු වඩාත්ම ක්ෂණික තීරණය තමයි multisite installation එක _subdirectories_ (සබ්ඩික්ටරි) මගින්ද නැත්නම් _subdomains_ (සබ්ඩොමේන්) මගින්ද ක්‍රියාත්මක වෙන්නේ කියලා. Ultimate Multisite දෙකම මේ දෙකටම හොඳින් වැඩ කරනවා, නමුත් මේ දෙකේ ව්‍යුහപരമായ වෙනස්කම් තිබෙනවා.

_subdirectory_ configuration එකේදී, network siteleri ප්‍රධාන domain නාමය මත පදනම්ව මාර්ගයක් (path) උරුම කරගන්නවා. උදාහරණයක් ලෙස 'site1' ලෙස නම් කරන ලද network site එකට එහි සම්පූර්ණ URL එක https://domain.com/site1 වනු ඇත. _subdomain_ configuration එකේදී, network site එක ප්‍රධාන domain නාමය මත පදනම්ව තමන්ටම සබ්ඩොමේනයක් (subdomain) ලබා ගනී. එබැවින් 'site1' ලෙස නම් කරන ලද site එකට එහි සම්පූර්ණ URL එක https://site1.domain.com වනු ඇත.

මෙම විකල්ප දෙකම සම්පූර්ණයෙන්ම වලංගු තේරීම් වුවද, _subdomains_ භාවිතා කිරීමෙන් වාසි කිහිපයක් ලැබුණත්, එහි ගෘහ නිර්මාණ ශිල්පයේ වැඩි සැලකිල්ලක් සහ සැලසුම්කරණයක් අවශ්‍ය වේ.

DNS සම්බන්ධයෙන්, _subdirectories_ භාවිතා කිරීම සාපේම ප්‍රශ්නයක් පමණි. ජාල වෙබ් අඩවි මූලික මාර්ගයේ දරුවන් වනු බැවින්, ප්‍රධාන වෙබ් අඩවිය සඳහා එක් ડોමේන් නාම ඇතුළුවීමක් තිබිය යුතුය. _subdomains_ සඳහා වන අභියෝගය තරමක් සංකීර්ණයි. ඒ සඳහා සෑම ජාල අඩවියකටම වෙනම CNAME ඇතුළුවීමක් අවශ්‍ය විය හැකිය, නැතහොදි DNS වාර්තා වල වයිල්ඩ්කාට් (*) ඇතුළුවීමක් භාවිතා කළ හැකිය.

SSL සහ SSL සහතිකපත් නිකුත් කිරීම සහ භාවිතය පිළිබඳ තවත් ප්‍රදේශයක් සැලකිල්ලට ගත යුතුය. _subdirectory_ සැකසුමකදී, ජාල අඩවි මූලික වෙබ් අඩවියේ මාර්ග නිසා එක් ડોමේන් සහතිකයක් භාවිතා කළ හැකිය. එබැවින් domain.com සඳහා වන සහතිකයක් https://domain.com/site1, https://domain.com/site2 සහ ඒ ආදී සියල්ල සඳහා SSL සපයනු ඇත.

_subdomain_ සැකසුමකදී වයිල්ඩ්කාට් SSL සහතිකය භාවිතා කිරීම වඩාත් පොදු විකල්ප වලින් එකකි. මෙම වර්ගයේ SSL සහතිකයක් යනු එක් ડોමේනයක් සහ එහි _subdomains_ සඳහා සංකේතනය (encryption) සපයන දෙයකි. එබැවින්, වයිල්ඩ්කාට් SSL සහතිකයක් https://site1.domain.com, https://site2.domain.com සහ https://domain.com යනාදිය සඳහා සංකේතනය සපයනු ඇත.

වෙනත් විකල්ප තිබුණද, ඒවා බොහෝ විට පරාසයෙන් හා යෙදීමෙන් සීමිත වන අතර සුදුසුකම් සම්බන්ධයෙන් අමතර සැකසුම් සහ සැලකිල්ල අවශ්‍ය වේ.

#### Plugins සහ Themes {#plugins-and-themes}

WordPress ද ලබා දෙන දේ එය ගනු ලබන දේ ද වේ, අවම වශයෙන් පාරිභෝගිකයාගේ දෘෂ්ටිකෝණයෙන්. තනි WordPress ස්ථාපනයක් තුළ වෙබ් අඩවි පරිපාලකවරයෙකු නරක plugin එකක් ස්ථාපනය කළහොත් හෝ තම ස්ථාපනය යාවත්කාලීන කිරීමට අපොහොසත් වුවහොත්, මෙම ක්‍රියාවේ එකම ගොදුර සහ පාඩුව වන්නේ තමාමයි. කෙසේ වෙතත්, බහු අඩවි (multisite) ස්ථාපනයක් තුළ නරක plugin එකක් ස්ථාපනය කරන වෙබ් අඩවි පරිපාලකවරයෙකු ජාලයේ ඇති සෑම අඩවියකම ගොදුර කරනු ඇත.

මේ හේතුව නිසා, ඔබ එය බහු වෙබ් අඩවියක් (multisite) ලෙස සකස් කළ විට, WordPress එම හැකියාව වෙබ් අඩවි පරිපාලකයින්ට ප්ලග්ඉන් සහ තේමා ස්ථාපනය කිරීමට ඇති හැකියාව ඉවත් කර, ඒ වෙනුවට නව ජාල පරිපාලක හෝ 'සුපිරි පරිපාලක' (super admin) භූමිකාවකට එම හැකියාව මාරු කරයි. මෙම විශේෂිත භූමිකාවට ජාල අඩවිවල පරිපාලකයින්ට ඔවුන්ගේ දත්ත පුවරුවේ ඇති ප්ලග්ඉන් මෙනුව බැලීමට හෝ ප්‍රවේශ වීමට අවසර තිබේද යන්න තීරණය කිරීමට සහ එසේ නම්, එවැනි අවසරයන් ප්ලග්ඉන් _ක්‍රියාත්මක කිරීම_ (activating) හෝ _අක්‍රිය කිරීම_ (deactivating) සඳහා ද අදාළ වේද යන්න තීරණය කිරීමට හැකිය.

මෙම ප්‍රමාණයට, ජාල පරිපාලකයා විසින් ජාලයට ප්ලග්ඉන් සහ තේමා ස්ථාපනය කිරීම සහ එම ප්ලග්ඉන් සහ තේමා ජාල අඩවි සඳහා භාවිතා කිරීම සඳහා අවසර දීම භාර ගනී. වෙබ් අඩවි පරිපාලකයින්ට ඔවුන්ගේ අඩවියට අදාළ නොවන හෝ ඔවුන්ට පැවරී නොමැති ප්ලග්ඉන් සහ තේමා ස්ථාපනය කිරීමට හෝ ඒවාට ප්‍රවේශ වීමට නොහැකිය.

#### Media (මධ්‍යස්ථාන) {#media}

WordPress Multisite එකක ජාල වෙබ් අඩවි එකම දත්ත සමුදායක් බෙදා ගන්නා විට, ඒවා මධ්‍යස්ථාන ගොනු සඳහා ගොනු පද්ධතියේ වෙනම මාර්ග තබා ගනී.

ප්‍රමිති WordPress ස්ථානය (wp-content/uploads) එලෙසම පවතී; කෙසේ වෙතත්, එහි මාර්ගය ජාල අඩවියේ අනන්‍ය ID පිළිබිඹු වන පරිදි වෙනස් වේ. එහි ප්‍රතිඵලයක් ලෙස ජාල අඩවියක මධ්‍යස්ථාන ගොනු `wp-contents/uploads/site/[id]` ලෙස පෙනේ.

#### Permalinks (ප්‍රමිතීන්) {#permalinks}

අපි කලින් සඳහන් කළා පරිදි, _subdomain_ සැකසුමක් _subdirectory_ සැකසුමට වඩා වෙනස් වාසි ඇත, ඒවා මෙන්න: මාර්ග (paths).

_subdirectory_ සැකසුමකදී, ජාලය පිහිටුවූ විට නිර්මාණය කරන ප්‍රධාන අඩවිය (පළමු අඩවිය) සහ ජාල උප අඩවි සියල්ලටම වෙබ් අඩවි නාමය (domain name) වෙතින් යන එකම මාර්ගයක් බෙදා ගත යුතුය. මෙය ගැටුම් විශාල ප්‍රමාණයක් ඇති වීමට ඉඩ ඇත.

අනෙකුත් ජාල අඩවි සමඟ ගැටුම් වැළැක්වීම සඳහා, ප්‍රධාන අඩවියට /blog/ මාර්ගය අනිවාර්යයෙන් එකතු කර ඇත. මෙයින් අදහස් කරන්නේ 'Post name' වැනි සාමාන්‍ය permalinks වෙබ් අඩවි නාමය/blog/post-name/ ලෙස ඉදිරිපත් කෙරෙන බවයි.

_subdomain_ සැකසුමකදී මෙම ක්‍රියාව අවශ්‍ය නොවේ, මන්ද සෑම ජාල අඩවියකටම සම්පූර්ණ වෙබ් අඩවි වෙන් කිරීමේ වාසිය ලැබෙන අතර එම නිසා තනි මාර්ගයකට 의존する必要 නොමැත. ඒ වෙනුවට ඒවා තම _subdomain_ මත පදනම්ව තමන්ගේම වෙනස් වූ මාර්ග පවත්වා ගනී.

#### Static Pages (ස්ථාවර පිටු) {#static-pages}

_subdirectory_ සැකසුමේදී, ප්‍රධාන වෙබ් අඩවිය සහ ජාල වෙබ් අඩවි එකම මාර්ගයක් (path) බෙදා ගන්නා නිසා ස්ථාවර පිටු වල නම් ගැටුම් ඇතිවීමේ හැකියාව තිබේ.

මෙය වැළැක්වීම සඳහා, WordPress විසින් සමහර වෙබ් අඩවි නම් කළු ලැයිස්තුවකට (blacklist) ඇතුළත් කිරීමට ක්‍රමයක් සපයයි. එමඟින් ඒවා පළමු වෙබ් අඩවියේ නමට ගැටුමක් ඇති නොවන බව තහවුරු කෙරේ. සාමාන්‍යයෙන් ජාල පරිපාලකයා ප්‍රධාන වෙබ් අඩවියේ පිටු වල මූලික මාර්ග (root paths) ඇතුළත් කරනු ඇත.

_subdomain_ සැකසුමේදී, _subdomain_ එක නිසා නාම ගැටුම් ඇතිවීමේ හැකියාව අවම වේ. මන්ද එය ජාල වෙබ් අඩවියට පමණක් සුවිශේෂී වන අතර ප්‍රධාන වෙබ් අඩවිය හා කිසිදු සම්බන්ධයක් නොමැත.

### ලියාපදිංචිය (Registration) {#registration}

WordPress Multisite හි ජාල සැකසුම් තුළ නව පරිශීලක ලියාපදිංචි කිරීමේ විකල්ප කිහිපයක් තිබෙන අතර, ඒවා මගින් නව සහ දැනට සිටින පරිශීලකයින්ට වෙබ් අඩවි නිර්මාණය කිරීමට ඉඩ සලසයි.

තනි WordPress ස්ථාපනයන් (stand-alone installations) වලට වඩා, ජාල වෙබ් අඩවි පරිශීලක ලියාපදිංචි කිරීම් සඳහා හෝ එම ලියාපදිංචි කිරීම් භූමිකාවන්ට (roles) පැවරීම සඳහා හුරුපුරුදු විකල්ප පවත්වා නොගනී.

පරිශීලක ගිණුම් නිර්මාණය කරන විට, එම ගිණුම් ජාල මට්ටමින් (network level) ජනනය කරනු ලැබේ. එබැවින් ඒවා කිසිදු නිශ්චිත වෙබ් අඩවියකට අයත් වීමට වඩා, ජාලයට අයත් වේ. මෙයට සුවිශේෂී වාසි සහ අවාසි කිහිපයක් ඇත.

උදාහරණයක් ලෙස, ඔබේ WordPress Multisite එක පුවත් සහ තොරතුරු ක්ෂේත්‍රයේ තිබුණා යැයි සිතන්න. එවිට ඔබ එම multisite එක පිහිටුවා, මූල්‍ය, තාක්ෂණය, විනෝදාස්වාදය සහ වෙනත් උනන්දුවක් ඇති අංශ සඳහා ජාල වෙබ් අඩවි (network sites) නිර්මාණය කරමින් ප්ලග්ඉන් සහ තේමා පිළිබඳ සමස්ත පාලනය පවත්වා ගනී. සෑම ජාල වෙබ් අඩවියකටම, කස්ටම් post types හෝ සාමාන්‍ය post categories වලට වඩා එහි පෙනුම සහ පරිශීලක අත්දැකීම (user experience) කෙරෙහි වැඩි පාලනයක් ලැබේ.

මෙම ප්‍රමාණයට, පරිශීලකයෙකු ලොග් වූ විට ඔවුන් ජාලයට ලොග් වන අතර සුමට අත්දැකීමක් ලබා දීම සඳහා අවසානයේ සෑම ජාල වෙබ් අඩවියකටම ලොග් වේ. ඔබේ නව වෙබ් අඩවිය දායකත්ව මත පදනම් වී තිබේ නම්, මෙය වඩාත් සුදුසු විසඳුම සහ ප්‍රතිඵලය වනු ඇත.

කෙසේ වෙතත්, multisite එකේ අදහස් කරන ස්වභාවය සහ අරමුණ එකිනෙකට සම්බන්ධ නොවන විවිධ ජාල වෙබ් අඩවි ලබා දීම නම්, පරිශීලක භූමිකාවන් (user roles) හැසිරවීමට බාහිර හෝ අමතර ප්ලග්ඉන් අවශ්‍ය වීම සෑම විටම සිදුවන දෙයකි.

### Domain සහ SSL {#domain-and-ssl}

අපගේ අවධානයෙන් බොහෝ දේ මගහැරෙන WordPress Multisite ස්ථාපනයක් ගැන කතා කරමු - Wordpress.com. මෙය WordPress multisite හි වඩාත් පුළුල් උදාහරණය වන අතර, එය අරමුණක් ඉටු කිරීම සඳහා එය tùy chỉnh කිරීමට සහ හැඩගැස්වීමට ඇති හැකියාව පෙන්වයි.

අද දින නූතන අන්තර්ජාලයේ SSL භාවිතය පාහේ අත්‍යවශ්‍ය වන අතර WordPress multisite කළමනාකරුවන් ඉක්මනින් මෙම අභියෝගවලට මුහුණ දෙති.

_subdomain_ සැකසුම් මගින් වෙබ් අඩවි මූලික ડોමේන් නාමය මත පදනම්ව නිර්මාණය කෙරේ. එබැවින් ‘site1’ ලෙස නම් කරන ලද වෙබ් අඩවිය ‘site1.domain.com’ ලෙස නිර්මාණය වේ. වයිල්ඩ් කාඩ් SSL සහතිකයක් භාවිතා කිරීමෙන්, ජාල පරිපාලකයෙකුට මෙම අභියෝගය සාර්ථකව විසඳා ගැනීමට හැකි වන අතර ජාලයට SSL සංකේතනය (encryption) හැකියාවන් සැපයිය හැක.

WordPress Multisite වලට ડોમેન મેપિંગ (domain mapping) කියන ක්‍රමයක් ඇතුළත් වෙනවා. මේක හරහා ජාලයේ ඇති වෙබ් අඩවි වලට කස්ටම් ડોමේන් නම් හෝ ජාලයේ මූලික ડોමේනයට වෙනස් වන අනෙකුත් ડોමේන් සමඟ සම්බන්ධ කරන්න පුළුවන්.

ජාල පරිපාලකයින් (network administrators) සඳහා, මේක කියන්නේ ડોමේන් සැකසීම සහ SSL සහතිකපත් (SSL certificates) නිකුත් කිරීම හා නඩත්තු කිරීම යන දෙකෙහිම අමතර සංකීර්ණත්වයක් ඇති කරන දෙයක්.

WordPress Multisite මගින් [www.anotherdomain.com](http://www.anotherdomain.com) වැනි වෙබ් අඩවියක් 'site1' සමඟ සම්බන්ධ කිරීමට ඉඩ දෙන ක්‍රමයක් ලබා දුන්නත්, ජාල පරිපාලකයාට DNS ඇතුළත් කිරීම් (DNS entries) බාහිරව කළමනාකරණය කිරීම සහ SSL සහතිකපත් ක්‍රියාත්මක කිරීම පිළිබඳ අභියෝග තවමත් තිබෙනවා.

## Ultimate Multisite {#ultimate-multisite}

තනි WordPress ස්ථාපනයක් සහ Multisite ස්ථාපනයක ඇති වෙනස්කම් අවබෝධ කරගත් පසු, වෙබ් අඩවි සේවා (Websites as a Service - WaaS) සැපයීම සඳහා Ultimate Multisite යනු විශිෂ්ට මෙවලම බව බලමු.

### හැඳින්වීම {#introduction}

Ultimate Multisite කියන්නේ Website as a Service (WaaS) නිර්මාණය කිරීම සම්බන්ධයෙන් ඔයාගේ ස්විස් ආයුධ කට්ටලය වැනිය. Wix.com, Squarespace, WordPress.com ගැන හිතන්න, ඊට පස්සේ ඔයාගේම සේවාවක් හිමි කරගන්න එක ගැන හිතන්න.

ප්‍රධාන වශයෙන් Ultimate Multisite යනු WordPress Multisite භාවිතා කරනවා, නමුත් එය ජාල පරිපාලකයින් මුහුණ දෙන අභියෝග විසඳන විදිහට පමණක් නොව, පුළුල් පරාසයක භාවිත අවස්ථා සඳහා සහාය වන හැකියාවන් වැඩි දියුණු කරන ආකාරයට ක්‍රියා කරනවා.

අපි පහත කොටස්වල පොදු භාවිත අවස්ථා සහ එම අවස්ථා සඳහා අවශ්‍ය සලකා බැලීම් බලමු.

### භාවිත අවස්ථා (Use Cases) {#use-cases}

#### අවස්ථාව 1: ඒජන්සියක් (An Agency) {#case-1-an-agency}

සාමාන්‍යයෙන් ඒජන්සියක ප්‍රධාන කුසලතා වෙබ් අඩවි නිර්මාණය කිරීම තුළ තිබෙනවා, එහි සත්කාරකත්වය (hosting) හෝ අලෙවිකරණය වැනි දේ අමතර සේවාවන් ලෙස ලැයිස්තුගත කිරීම වැනි කරුණු ඇතුළුව.

एजेन्सीಗಳಿಗೆ अल्टीमेट मल्टीसाइट त्यांच्या एकाच प्लॅटफॉर्मवर अनेक वेबसाइट होस्ट आणि व्यवस्थापित करण्याची क्षमतामुळे एक अविश्वसनीय मूल्य प्रस्ताव (value proposition) सादर करते. विशेषतः ज्या एजेन्सी त्यांच्या डिझाइन्स विशिष्ट थीम्स जसे की GeneratePress, Astra, OceanWP किंवा इतर थीमवर मानकीकृत करतात, ते प्रत्येक नवीन साइटसाठी या थीम्स आपोआप सक्रिय करण्यासाठी अल्टीमेट मल्टीसाइटची क्षमतांचा फायदा घेऊ शकतात.

एजेन्सीच्या किंमत धोरणांसाठी सामान्य आणि लोकप्रिय प्लगइन्ससाठी असलेल्या अनेक डील्सप्रमाणेच, अल्टीमेट मल्टीसाइटचा वापर एजेन्सींना विद्यमान गुंतवणुकीचा फायदा घेण्यास मदत करतो, कारण ते एक सामायिक प्लॅटफॉर्म प्रदान करते जिथून प्लगइन्स स्थापित करता येतात, त्यांची देखभाल करता येते आणि त्यांचा उपयोग करता येतो.

बहुतांशी कॉन्फिगरेशनचा वापर अपेक्षित असेल आणि आनंदाची गोष्ट म्हणजे अल्टीमेट मल्टीसाइट क्लाउडफ्लेअर (Cloudflare) आणि cPanel सारख्या अनेक लोकप्रिय होस्टिंग प्रदात्यांसह तसेच सेवांसाठी डोमेन मॅपिंग आणि SSL प्रमाणपत्रे सुलभ करण्यासाठी त्यांच्या इंटिग्रेशन्समुळे ते खूप सोपे बनवते.

म्हणून, यापैकी एका प्रदात्याचा फायदा घेऊन किंवा अल्टीमेट मल्टीसाइटला क्लाउडफ्लेअरच्या मागे ठेवून डोमेन व्यवस्थापन आणि SSL प्रमाणपत्रे यांसारख्या गोष्टी काहीशी सोप्या होतात.

ज्या एजेन्सींना साइट्सच्या निर्मितीवर कडक नियंत्रण ठेवण्यास प्राधान्य असते, त्यांना अल्टीमेट मल्टीसाइटच्या सुव्यवस्थित इंटरफेसद्वारे साइट्स तयार करणे आणि ग्राहकांशी तसेच योजनांशी जोडणे किती सोपे आहे हे आवडेल.

![Ultimate Multisite site management interface](/img/admin/sites-list.png)

प्रत्येक उत्पादनासाठी प्लगइन्स आणि थीम्सवर कडक नियंत्रण अल्टीमेट मल्टीसाइटच्या सहज इंटरफेसद्वारे राखले जाते, ज्यामुळे नवीन साइटसाठी इंस्टेंशिएट (instantiated) केल्यावर प्लगइन्स आणि थीम्स उपलब्ध करणे किंवा लपवणे तसेच त्यांची सक्रिय स्थिती नियंत्रित करता येते.

තේමා (Themes) වලට සමාන ක්‍රියාකාරීත්වයන් තිබෙනවා, ඒ නිසා වෙබ් අඩවිය නිර්මාණය කිරීමේදී තෝරාගත් තේමාවක් සක්‍රීය කිරීමට හෝ සැඟවීමට හැකියාව ලැබෙනවා.

![Product theme limitations interface](/img/config/product-themes.png)

Ultimate Multisite භාවිතා කිරීමෙන් නියෝජිත ආයතනවලට ඔවුන් දක්ෂතා කරන දේ කළ හැකි බව ගැන සහනයක් ලැබේ - එනම් විශිෂ්ට වෙබ් අඩවි නිර්මාණය කිරීමයි.

#### අවස්ථාව 2: විශේෂිත සපයන්නා (Niche Provider) {#case-2-niche-provider}

"එක දෙයක් හොඳින් කරන්න" කියන පැරණි කියමනක් තිබෙනවා. බොහෝ විශේෂඥයින් සඳහා මෙයින් අදහස් කරන්නේ තනි මූලික අදහසක් වටා නිෂ්පාදනයක් හෝ සේවාවක් නිර්මාණය කිරීමයි.

ඔබ සමාජයකට වෙබ් අඩවි ප්‍රවර්ධනය කරන දැඩි ගොල්ෆ් ක්‍රීඩකයෙක් විය හැකියි, නැතිනම් ක්ලෑන් සඳහා වෙබ් අඩවි සැපයෙන දක්ෂ eSports ක්‍රීඩකයෙකු විය හැකියි. අවන්හල සඳහා වෙන් කිරීමේ සේවාවක් ප්‍රවර්ධනය කරන පුද්ගලයෙක්ද?

බොහෝ හේතු නිසා ඔබට පොදු රාමුවක් සහ වේදිකාවක් මත පදනම් වූ සේවාවන් සැපයීමට අවශ්‍ය වනු ඇත. අවශ්‍ය ක්‍රියාකාරීත්වය ලබා දීම සඳහා ඔබ විසින්ම නිර්මාණය කළ හෝ ආයෝජනය කළ custom plugins තිබිය හැකිය, නැතිනම් කර්මාන්තයේ හොඳම භාවිතයන් සැලසුම් කිරීම සඳහා යම් ආකාරයක ප්‍රමිතිකරණය කරන ලද ප්‍රවේශයක් අවශ්‍ය විය හැකිය.

Ultimate Multisite හි නව්‍ය විශේෂාංගවලින් එකක් වන්නේ template sites භාවිතා කිරීමයි. template site එකක් කියන්නේ තේමාව ස්ථාපනය කර සක්‍රීය කර ඇති, අවශ්‍ය plugins ස්ථාපනය කර සක්‍රීය කර ඇති සහ නියැදි posts හෝ පිටු නිර්මාණය කර ඇති තැනකි. පාරිභෝගිකයෙකු එම template එක මත පදනම්ව නව වෙබ් අඩවියක් නිර්මාණය කරන විට, template එකේ අන්තර්ගතයන් සහ සැකසුම් නව වෙබ් අඩවියට පිටපත් කෙරේ.

විශේෂිත වෙබ් අඩවි සහ සේවාවන් සපයන්නෙකු සඳහා මෙය custom plugins සහ නිර්මාණ සමඟ වහාම භාවිතයට සූදානම් කළ වෙබ් අඩවියක් ක්ෂණිකව නිර්මාණය කිරීමේ හැකියාව පිළිබඳ අසමසම වාසියක් ලබා දෙයි. පාරිභෝගිකයාට සේවාව සම්පූර්ණ කිරීමට අවම දත්ත ප්‍රමාණයක් පමණ සැපයිය යුතුය.

අවශ්‍යතා අනුව _subdirectory_ හෝ _subdomain_ සැකසුම් දෙකම ගැලපිය හැකියි. එවැනි අවස්ථාවලදී, ගෘහ නිර්මාණ ශිල්පය තීරණය වන්නේ _subdirectories_ සඳහා සරල SSL සහ _subdomains_ සඳහා වයිල්ඩ්කාට් (wildcard) SSL සහතිකයක් අතර වෙනසක් මතයි.

#### සිද්ධිය 3: WordPress වෙබ් හොස්ටින් {#case-3-wordpress-web-hosting}

WordPress වෙබ් අඩවි ගොනු කිරීම සඳහා ක්‍රම無කී ප්‍රමාණයක් තිබුණත්, පූර්ව ස්ථාපනය කළ WordPress අනුවාදයක් සමඟ පාරිභෝගිකයාට වෙබ් අවකාශය ලබා දීම තරම් සරල දෙයක් කලාතුරකින් දක්නට ලැබේ. මන්ද, අර්ථවත් සේවාවක් සැපයීමට අවශ්‍ය තීරණ සහ සැලකිලි ගණනාවක් එකට ගෙන ඒම අවශ්‍ය වන බැවිනි.

Ultimate Multisite විසින් WordPress අඩවි හොස්ටින් කිරීම සඳහා සම්පූර්ණ ටර්න්කේට් (turnkey) විසඳුමක් ලබා දීමෙන් මෙම ක්ෂේත්‍රයේ විශිෂ්ටත්වය පෙන්වයි. මෙම විසඳුමට යටින් දාන මූලික යාන්ත්‍රණයන් ඇතුළත් වන්නේ දායකත්ව සේවාවන් සැපයීම, ගෙවීම් එකතු කිරීම, checkout forms (ඇරයුම් පෝරම), වට්ටම් කූපන් සහ පාරිභෝගික සන්නිවේදනයන් සඳහායි.

WordPress Multisite නිවැරදිව ස්ථාපනය කිරීමට, වින්‍යාස කිරීමට සහ නඩත්තු කිරීමට අවශ්‍ය ප්‍රධාන කාර්යයන් බොහෝ දුරට Ultimate Multisite මගින් පහසු කරනු ලබන්නේ ජාල පරිපාලකයින්ට ඔවුන්ගේ සේවාව හෝ විශේෂිත ක්ෂේත්‍ර (උදාහරණයක් ලෙස නිෂ්පාදන ස්ථර, මිල ගණන් සහ සේවා දීමනා) සම්බන්ධයෙන් සලකා බැලිය යුතු අංශ පමණක් සැලකිල්ලට ගත හැකි වන පරිදි ය.

Ultimate Multisite සමඟ ඒකාබද්ධ කිරීමට කැමති සංවර්ධකයින් සඳහා, මෙම විසඳුම RESTful API එකක් සහ සිදුවීම් දැනුම් දීම සඳහා Webhooks ද සපයයි.

බාහිර ප්ලග්ඉන් සහ බලපත්‍ර ගණනාවකට යැපීමකින් තොරව, Ultimate Multisite Wix, Squarespace, WordPress.com වැනි වෙනත් ඒවාට සමාන විශේෂාංග බහුල විසඳුමක් සපයයි.

### ගෘහ නිර්මාණ ශිල්පය පිළිබඳ සලකා බැලීම් {#architecture-considerations}

මෙය සම්පූර්ණ මාර්ගෝපදේශයක් නොවුනත්, Ultimate Multisite ස්ථාපනය සඳහා නිවැරදි තාක්ෂණය තෝරා ගැනීමට මඟ පෙන්වීමක් ලෙස පහත කරුණු භාවිතා කළ හැකිය.

#### බෙදාගත් (Shared) එදිරිව කැප වූ (Dedicated) හොස්ටින් කිරීම {#shared-vs-dedicated-hosting}

අවාසනාවකට, සියලුම හොස්ටිං සපයන්නන් සමාන නැත, සමහරු අතිශය අධික සේවාදායක (server) තදබදයන් සිදු කරයි. අඩු වියදම් සහිත සපයන්නන් සාමාන්‍යයෙන් සේවාදායක තදබදය උපරිම කර ගැනීමෙන් ආදායමක් උපයනවා. එබැවින් ඔබේ Ultimate Multisite ස්ථාපනය එකම සේවාදායකයේ ඇති සිය ගණනක් වෙබ් අඩවි වලින් එකක් පමණක් විය හැකියි.

සපයන්නාගෙන් සුදුසු ආරක්ෂාවන් නොමැතිව, බෙදාගත් සේවාදායකයක් (shared server) මත ඇති අඩවි 'ශබ්දකාරී අසල්වැසි' (noisy neighbour) ගැටලුව අත්විඳියි. එනම්, එකම සේවාදායකයේ අනෙක් අඩවි සඳහා ඉතිරිව ඇති සම්පත් සමඟ තරඟ කිරීමට එම අඩවිය විශාල සම්පත් පරිභෝජනය කිරීමයි. බොහෝ විට මෙය මන්දගාමී වන හෝ නියමිත වේලාවට ප්‍රතිචාර නොදක්වන අඩවි ලෙස පෙන්වනු ලැබේ.

ඔබම වෙබ් හොස්ටිං සපයන්නෙකු ලෙස, මෙම බලපෑම් නිසා ඔබේ පාරිභෝගිකයින් දුර්වල වේගයන්, අඩු පිටු ශ්‍රේණිගත කිරීම් (page rank) සහ ඉහළ ප්‍රතික්ෂේප කිරීමේ අනුපාත (bounce rates) අත්විඳින අතර බොහෝ විට ඔවුන් වෙනත් තැන්වල සේවාවන් සොයන බැවින් පාරිභෝගිකයන් අහිමි වීමට හේතු වේ.

සාරාංශයක් ලෙස, ලාභයි යනු හොඳ නොවේ.

Ultimate Multisite හොඳ හොස්ටිං සපයන්නන් කිහිපයක සමඟ ක්‍රියා කරන බවත් ඒවා ડોමේන් මැපින් (domain mapping) සහ ස්වයංක්‍රීය SSL වැනි කාර්යයන් සැපයීම සඳහා ඔවුන්ගේ පරිසරයට හොඳින් ඒකාබද්ධ වන බැවින් හොඳ හොස්ටිං සපයන්නන් විසින් දන්නා බවත් එය ප්‍රසිද්ධයි. මෙම සපයන්නන් කාර්ය සාධනයට (performance) වටිනාකමක් දෙති සහ බෙදාගත් හොස්ටිං වලට වඩා ඉහළ මට්ටමේ සේවාවක් ලබා දෙති.

සුදුසු අනුකූල සපයන්නන්ගේ ලැයිස්තුවක් සහ ඒ සඳහා සම්පූර්ණ සැකසුම් උපදෙස් (complete set-up instructions) සඳහා කරුණාකර Compatible Providers හි ලේඛන පරීක්ෂා කරන්න.

#### කාර්ය සාධන සලකා බැලීම් (Performance Considerations) {#performance-considerations}

Ultimate Multisite මන්දගාමී යෙදුමක් (application) නොවේ, ඒ වෙනුවට එය 놀랍도록 වේගවත් වේ. කෙසේ වෙතත්, එය පදනම් වූ යෙදුමේ සහ යටිතල පහසුකමේ (infrastructure) හොඳම පමණක් ක්‍රියා කරයි, එමෙන්ම එයට ප්‍රවේශ විය හැකි දේ පමණක් භාවිතා කළ හැකිය.

මෙය සලකා බලන්න: ඔබට 100 අඩවි සහිත Ultimate Multisite ස්ථාපනයක ජාල පරිපාලකයෙකු ලෙස සිටිනවා. ඒ අතරින් සමහර අඩවි හොඳින් ක්‍රියා කරමින් සෑම දිනකම වෙබ් අමුත්තන් ගණනක් ආකර්ෂණය කරයි.

මෙම තත්ත්වය කුඩා පරිමාණයේ, එනම් වෙබ් අඩවි එකක් හෝ පස් දෙනෙකු දක්වා වූ පරිමාණයක වෙනස් විය හැකියි, නමුත් විශාල පරිමාණයේ ගැටළු ඉක්මනින් පැහැදිලි වනු ඇත.

අවධානය යොමු නොකර සිටින විට, තනි Ultimate Multisite වෙබ් අඩවිය සියලුම වෙබ් අඩවිවල නරඹන්නන්ගේ ඉල්ලීම් සපුරාලීමට වගකිව යුතුය. මෙම ඉල්ලීම් ගතික PHP පිටු හෝ CSS, JavaScript හෝ මීඩියා ගොනු වැනි ස්ථිතික දේ විය හැකිය. එක් වෙබ් අඩවියක් හෝ සියයක් තිබුණත්, මෙම කාර්යයන් පුනරාවර්තන, ඒකාකාරී සහ නාස්තිකාරී වේ. සෑම ඉල්ලීමකම ප්‍රතිදානය එකම ස්ථිතික තොරතුරක් නම්, PHP ගොනුව සැකසීමට CPU බලය සහ මතකය භාවිතා කිරීම අත්‍යවශ්‍ය නොවේ.

ඒ හා සමානව, PHP හෝ HTML පිටුවක් සඳහා එක් ඉල්ලීමක් මගින් ස්ක්‍රිප්ට්, CSS සහ රූප ගොනු සඳහා පසු විපරම් ඉල්ලීම් කිහිපයක් ජනනය කරයි. මෙම ඉල්ලීම් සෘජුවම ඔබේ Ultimate Multisite සේවාදායකයට (server) යොමු වේ.

මෙම ගැටලුව පහසුවෙන් විසඳිය හැකි ක්‍රමය වන්නේ සේවාදායකය (server) වැඩිදියුණු කිරීමයි, නමුත් එය ද්විතීයික ගැටළුවක් වන භූගෝලීය ප්‍රමාදයන් (geographic latencies) නිවැරදි නොකරයි. මෙම ගැටලුව නිසි ලෙස විසඳිය හැක්කේ විවිධ ස්ථානවල ඇති බහු සේවාදායකයන් (multiple servers in multiple locations) පමණි.

මෙම හේතුව නිසා බොහෝ ජාල පරිපාලකයින් ස්ථිතික පිටු සඳහා ඉල්ලීම් සපුරාලීම සඳහා front-end caching solutions සහ content distribution networks (CDN) භාවිතා කරති. මෙම ඉල්ලීම් සපුරාලීම සහ සේවාදායකයට පැමිණීමට පෙර දත්ත සැපයීමෙන් සැකසුම් සම්පත් ඉතිරි කරන අතර ප්‍රමාදයන් ඉවත් කරයි, අනවශ්‍ය වැඩිදියුණු කිරීම් මඟ හරියි සහ තාක්ෂණික ආයෝජන උපරිම කර ගනී.

Ultimate Multisite හි ඇති Cloudflare add-on එකක් මගින් ජාල පරිපාලකයින්ට තම ස්ථාපනය (installations) Cloudflare පිටුපස තබා ගැනීමටත් එහි කැෂි කිරීමේ හැකියාවන් පමණක් නොව DNS ආගන්තු කිරීම, SSL සහ ආරක්ෂක යාන්ත්‍රණ ද භාවිතා කිරීමට ඉඩ සලසයි.

#### 백අප් (Backups) {#backups}

백업 පිළිබඳ උපදෙස් සඳහා මිනිසුන් 50 දෙනෙකුගෙන් උපදෙස් ලබාගෙන විවිධ 백අප් උපාය මාර්ග ගැන 50 ක් වෙනස් අදහස් ලැබිය හැකියි. පිළිතුර නම්, එය රඳා පවතින්නේ කුමක් මත දැයි යන්නයි.

ප්‍රතික්ෂේප නොකරන කරුණ නම් බෑක්අප් (backups) අවශ්‍ය බවත්, විශේෂයෙන් කළමනාකරණය කරන ලද සේවාවක් (managed service) ලබා දෙන අය විසින් මේවා කළමනාකරණය නොකරනු ඇතැයි එය අතිශය දුර්ලභ දෙයක් බවයි. එහි ප්‍රතිඵලයක් ලෙස, මෙම සේවාව සැපයීමට සහ කළමනාකරණය කිරීමට පාරිභෝගිකයින් ජාල පරිපාලකයෙකු වෙත යොමු වනු ඇත. එම ජාල පරිපාලකයා කවුරුන්ද යන්න සම්පූර්ණයෙන්ම වෙනස් ගැටලුවකි.

මෙම කොටස සඳහා, බෑක්අප් එකක් යනු බෑක්අප් ආරම්භ කරන අවස්ථාවේ පද්ධතියේ තත්ත්වයේ එක් මොහොතක පිටපතක් බව අපි එකඟ වෙමු. සරලව කිවහොත්, බෑක්අප් එකක වේලාවට පද්ධතියේ තත්ත්වය කුමක් වුවත්, එම තත්ත්වය බෑක්අප් එක තුළ ග්‍රහණය කර ආරක්ෂා කෙරේ.

මෙම අවබෝධයෙන් පසු, බෑක්අප් ලබා ගැනීමට සහ ඔබේ පරිසරයට වඩාත් සුදුසු දේ කුමක්ද යන්න බොහෝ දුරට ඔබගේ අවශ්‍යතා සහ සත්කාරක සපයන්නාගේ (hosting provider) එම අවශ්‍යතා සපුරාලීමේ හැකියාව මත රඳා පවතී. කෙසේ වෙතත්, වඩාත්ම අදහස් සහිත ඒවායේ සිට අඩු අදහස් සහිත ඒවා දක්වා පිළිවෙලින් පහත විකල්ප මඟ පෙන්වීම් ලබා දිය යුතුය.

#### ස්නැප්ෂොට් (Snapshots) {#snapshots}

බෑක්අප් සඳහා ස්නැප්ෂොට් යනු රන් බුලට් වැනි දෙයකි, මන්ද ඒවා සරලයි, සංකීර්ණ නැත (ප්‍රතිපෝෂණය කිරීමට අවශ්‍ය වූ විට හැර), සහ 'ක්‍රියාත්මක වේ'. කෙසේ වෙතත්, ඔබේ සපයන්නාගේ සහාය අවශ්‍ය වන අතර එය බොහෝ දුරට VPS (Virtual Private Server) හෝ ඒ හා සමාන දෙයක් තිබේ නම් පමණක් අදාළ වේ. අපගේ ‘සුදුසු සපයන්නන්’ (Compatible Providers) ලේඛනයේ ලැයිස්තුගත කර ඇති ප්‍රවර්ග කිහිපයක්, ජාල පරිපාලකයාගේ තවත් මැදිහත්වීමකින් හෝ සැලකිල්ලකින් තොරව බෑක්අප් ලබා දෙන ඒවා වේ.

සාම්ප්‍රදායික පිටපත් (backups) ගොනු සහ දත්ත සමුදායන් ඉලක්ක කරන අතර, ස්නැප්ෂොට් එකක් මුළු තැටියම (disk) ඉලක්ක කරයි. මෙයින් අදහස් කරන්නේ ස්නැප්ෂොට් එකේ වෙබ් අඩවියේ දත්ත පමණක් නොව, මෙහෙයුම් පද්ධතිය සහ සැකසුම් ද ග්‍රහණය කර ගන්නා බවයි. බොහෝ දෙනෙකුට මෙය විශාල වාසියක් වන අතර, රෝගී වූ යන්ත්‍රයක් ප්‍රතිස්ථාපනය කිරීමට ක්ෂණිකව ස්නැප්ෂොට් එකකින් නව පද්ධතියක් නිර්මාණය කර ක්‍රියාත්මක කළ හැකිය. ඒ හා සමානව, ගොනු ලබා ගැනීමට අවශ්‍ය සංග්‍රහ කිරීමේ (recovery) ක්‍රියාවලිය සඳහා, දැනට පවතින යන්ත්‍රයක තැටියක් ලෙස ස්නැප්ෂොට් රූපය සම්බන්ධ කිරීම පමණක් ප්‍රමාණවත් වන අතර එමඟින් ගොනු වෙත ප්‍රවේශ වීමට සහ පිටපත් කිරීමට හැකිය.

호스팅 සපයන්නා සමඟ ස්නැප්ෂොට් සඳහා අමතර පිරිවැයක් ඇති විය හැකි නමුත් එය අනතුරු සඳහා කරන රක්ෂණයක් ලෙස සැලකිය හැකිය.

#### Plugins (ප්ලග්ඉන්) {#plugins}

බාහිර ස්ක්‍රිප්ට් කළමනාකරණය කිරීම ඔබට පහසු නොවේ නම්, WordPress සමඟ විසඳිය නොහැකි ගැටළු ඉතා අඩුවක් තිබේ, එවිට ප්ලග් එකක් ඇති දේ තවත් හොඳම විකල්පය විය හැකිය.

ප්ලග්ඉන් වල විවිධ විකල්ප සහ විශේෂාංග තිබුණත්, ඒවායේ ප්‍රධාන කාර්යය එකක්මයි: WordPress ගොනු සහ දත්ත සමුදායේ පිටපත් ගැනීම. ඉන්පසු ක්‍රියාකාරිත්වයන් වෙනස් වේ. සමහර ප්ලග්ඉන් වලට Google Drive හෝ Dropbox වැනි බාහිර සේවාවලට හෝ S3, Wasabi වැනි අනුකූල වස්තු ගබඩා (object storage) සේවාවක් වැනි යම් සමාන සේවාවකට පිටපත් කිරීමට හැකිය. වඩාත් පුළුල් ප්ලග්ඉන් මගින් වෙනස් වූ දත්ත පමණක් පිටපත් කිරීම සඳහා වන විවිධ පසුබැස්ම (differential backups) හෝ වෙනත් උපාය මාර්ග භාවිතා කරයි, එමඟින් බාහිර ගබඩා වියදම් ඉතිරි කරගත හැකිය.

ඔබ ඔබේ ප්ලග්ඉන් තෝරා ගැනීමේදී එය multisite සමඟ ක්‍රියා කරන බව තහවුරු කර ගැනීම වැදගත් වේ. එහි ක්‍රියාකාරිත්වයේ ස්වභාවය නිසා, පසුබැස්ම (backup) ක්‍රියාත්මක වන අතරතුර සේවාදායකයේ (server) తాత్කාලික බරක් (temporary load) තිබිය හැකි බව අපේක්ෂා කළ හැකිය.

#### ડોමේන් සහ SSL {#domain-and-ssl-1}

multisite _subdomain_ මාදිලියේදී ડોමේන් නම් පිළිබඳව බොහෝ දේ සාකච්ඡා කර ඇත. ජාල පරිපාලකයින් සඳහා gần විශ්වීය විසඳුම වන්නේ wildcard DNS entries භාවිතා කිරීමයි.

![Wildcard DNS entry configuration example](/img/config/settings-domain-mapping.png)

මෙම ආකාරයේ DNS entry මගින් ‘site1.domain.com’ සහ ‘site2.domain.com’ වැනි _subdomains_ IP ලිපිනයක් වන 1.2.3.4 වෙත සාර්ථකව විසඳනු ඇත, එමඟින් Ultimate Multisite සහ _subdomain_ මාදිලියේ WordPress Multisite සඳහා සහ වඩාත් පුළුල් වශයෙන් සහාය වේ.

මෙය HTTP සඳහා හොඳින් ක්‍රියා කළ හැකිය, මන්ද ඉලක්ක සේවාදායකය (target host) HTTP headers වලින් කියවනු ලබන නමුත් අද දින ආරක්ෂිත HTTPS ගනුදෙනු අත්‍යවශ්‍ය වන තරමට වෙබ් අඩවි සරල නොවන බැවිනි.

SSL సర్టిఫికేట్‌ల కోసం సులభమైన ఎంపికలు అందుబాటులో ఉన్నాయి. _subdirectory_ మోడ్‌లో ఒక సాధారణ డొమైన్ సర్టిఫికేట్‌ను ఉపయోగించవచ్చు. ఇవి ఉచిత LetsEncrypt సేవ లేదా ఇతర మూలాలను ఉపయోగించే హోస్టింగ్ ప్రొవైడర్ల నుండి త్వరగా మరియు ఉచితంగా లభిస్తాయి. లేదంటే, మీరు సర్టిఫికేట్ సిగ్నేచర్ రిక్వెస్ట్ (CSR) ను రూపొందించగలిగితే, అధికార పరిధి ఉన్న సంస్థల నుండి ఇవి వాణిజ్యపరంగా అందుబాటులో ఉంటాయి.

_subdomain_ మోడ్‌కు, వైల్డ్ కార్డ్ SSL సర్టిఫికేట్‌ను వైల్డ్ కార్డ్ డొమైన్‌తో ఉపయోగించడం వలన అదనపు కాన్ఫిగరేషన్ లేకుండా సర్టిఫికేట్ రూట్ డొమైన్ మరియు అన్ని _subdomains_ కోసం అధికార పరిధిని కలిగి ఉంటుంది.

అయితే, క్లౌడ్ఫ్లర్ వంటి సేవలతో వైల్డ్ కార్డ్ SSL సర్టిఫికేట్‌లు పనిచేయకపోవచ్చు, మీరు ఎంటర్‌ప్రైజ్ ప్లాన్‌లో ఉన్నట్లయితే లేదా ఎంట్రీని DNS మాత్రమే అని సెట్ చేసినట్లయితే, దీని సందర్భంలో అన్ని క్యాషింగ్ మరియు ఆప్టిమైజేషన్ దాటవేయబడతాయి.

Out-of-the-box Ultimate Multisite ఈ సమస్యకు పరిష్కారాన్ని అందిస్తుంది, ఇది WordPress మల్టీసైట్‌ల అవసరాలపై మా విస్తృత అనుభవాన్ని ప్రదర్శిస్తుంది. ఈ సాధారణ అడ్-ఆన్‌ను యాక్టివేట్ చేయడం ద్వారా, Ultimate Multisite మీ క్లౌడ్ఫ్లర్ ఆధారాలను ఉపయోగించి నెట్‌వర్క్ సైట్‌ల కోసం క్లౌడ్ఫ్లర్‌లో DNS ఎంట్రీలను ఆటోమేటిక్‌గా జోడించి వాటి మోడ్‌ను 'proxied' గా సెట్ చేస్తుంది. ఈ విధంగా, ప్రతి నెట్‌వర్క్ సబ్సైట్ సృష్టించబడినప్పుడు SSLతో సహా క్లౌడ్ఫ్లర్ యొక్క పూర్తి రక్షణ మరియు ప్రయోజనాలను కలిగి ఉంటుంది.

మీ Ultimate Multisite ఇన్‌స్టాలేషన్ స్వభావం మరియు ఉద్దేశ్యంపై ఆధారపడి, కస్టమర్‌లు తమ సొంత డొమైన్‌లను ఉపయోగించాల్సిన అవసరం ఉండవచ్చు. ఈ సందర్భంలో, నెట్‌వర్క్ నిర్వాది రెండు సమస్యలను పరిష్కరించడానికి బాధ్యత వహిస్తారు. ఒకటి, డొమైన్ పేరు హోస్టింగ్ మరియు రెండు, ఆ డొమైన్ కోసం SSL సర్టిఫికేట్‌లు.

බොහෝ දෙනාට Cloudflare භාවිතා කිරීම පහසු විකල්පයක් විය හැකියි. පාරිභෝගිකයාට ඔවුන්ගේ ડોමේන් Cloudflare එකේ තැබීමට, Ultimate Multisite හි මූලික ડોමේනයට CNAME එකක් යොමු කිරීමට සහ තම ડોමේනය Ultimate Multisite තුළ සිතියම්ගත කිරීමට අවශ්‍ය වේ. මෙය ආරම්භ කිරීම සඳහා ඔවුන්ගේ කස්ටම් ડોමේන් නාමය ප්‍රයෝජනයට ගැනීමට ඉඩ සලසයි.

මෙයින් ඔබ්බට, DNS සහ SSL සැකසීමේ ක්‍රියාවලිය සාමාන්‍ය දෙයක් නොවන බැවින් වෙනත් විසඳුම් සොයා ගත යුතුය. ඒ නිසා Ultimate Multisite නිර්දේශ කරන්නේ අනුකූල ප්‍රොවයර් (Compatible Providers) ලැයිස්තුවකි. මන්ද, මෙම සැකසුම් කිරීමේ ක්‍රියාවලිය තරමක් සංකීර්ණ විය හැකි නමුත්, Ultimate Multisite මෙම ප්‍රොවයර් සමඟ ඒකාබද්ධ වීම නිසා සංකීර්ණත්වය බොහෝ දුරට අඩු වන අතර ක්‍රියා පටිපාටිය ස්වයංක්‍රීය වේ.

#### Plugins (ප්ලග්ඉන්) {#plugins-1}

ඔබේ පාරිභෝගිකයින්ට හෝ ජාල වෙබ් අඩවි සඳහා පහසුකම් සැපයීම සඳහා ඔබට අමතර plugins අවශ්‍ය වීමට ඉඩ වැඩිය. සියලුම plugins WordPress Multisite සහ Ultimate Multisite සමඟ ක්‍රියා කරන්නේද? එය රඳා පවතින්නේ කුමක් ද යන්න මත රඳා පවතී.

බොහෝ plugins WordPress Multisite වල ස්ථාපනය කළ හැකි වුවද, ඒවායේ සක්‍රීය කිරීම සහ බලපත්‍ර (licensing) කර්තෘ අනුව වෙනස් වේ.

සමහර plugins සඳහා එක් එක් ડોමේනයට බලපත්‍රයක් අවශ්‍ය වන විට බලපත්‍ර යෙදීම සම්බන්ධයෙන් ඇති අභියෝගය වන්නේ මෙහිදීයි. මෙයින් අදහස් කරන්නේ සමහර plugins සඳහා ජාල පරිපාලකයාට සෑම නව වෙබ් අඩවියක් සඳහාම එම plugin එකේ බලපත්‍රය අතින් සක්‍රීය කිරීමට සිදුවන බවයි.

එබැවින්, ඔබේ plugin එක WordPress Multisite සමඟ ක්‍රියා කරන ආකාරය සහ එය බලපත්‍ර ලබා ගැනීම සඳහා අවශ්‍ය විශේෂ අවශ්‍යතා හෝ ක්‍රියා පටිපාටි පිළිබඳව plugin කර්තෘවරයා සමඟ පරීක්ෂා කර බැලීම වඩාත් සුදුසු විය හැකිය.
