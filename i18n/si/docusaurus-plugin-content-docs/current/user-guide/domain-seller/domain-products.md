# ડોમેન නිෂ්පාදන અને કિંમત નિર્ધારણ {#domain-products-and-pricing}

ડોમેન પ્રોડક્ટ્સ એ છે કે તમે કિંમત, TLDs (ટોપ-લેવલ ડોમેન્સ), WHOIS પ્રાઇવસી અને પ્રદાતાની પસંદગીને કેવી રીતે નિયંત્રિત કરો છો. દરેક ડોમેન પ્રોડક્ટ એક સ્ટાન્ડર્ડ Multisite Ultimate પ્રોડક્ટ હોય છે જેમાં વધારાનો **Domain Settings** ટેબ હોય છે.

## ડોમેન પ્રોડક્ટ બનાવવી {#creating-a-domain-product}

૧. **Network Admin › Ultimate Multisite › Products** પર જાઓ
૨. **Add New** પર ક્લિક કરો
૩. પ્રોડક્ટ પ્રકારને **Domain** તરીકે સેટ કરો
૪. **Domain Settings** ટેબ (નીચે જુઓ) ને ગોઠવો
૫. સેવ કરો

ડોમેન પ્રોડક્ટ્સ પ્રોડક્ટ લિસ્ટમાં જાંતીય **Domain** બેજ સાથે દેખાશે અને તેને **Domain Products** ટેબનો ઉપયોગ કરીને ફિલ્ટર કરી શકાય છે.

## ડોમેન સેટિંગ્સ ટેબ {#domain-settings-tab}

### પ્રદાતા (Provider) {#provider}

તમારે કયા રજિસ્ટ્રારને આ પ્રોડક્ટ માટે નોંધણી સંભાળવી તે પસંદ કરો. ડિફોલ્ટ રૂપે Domain Seller સેટિંગ્સમાં સેટ કરેલ વૈશ્વિક **Default provider** પર સેટ હોય છે.

### સમર્થિત TLDs (Supported TLDs) {#supported-tlds}

જો તમે ખાલી છોડી દો, તો એક **catch-all product** બનાવો જે અન્ય કોઈ પ્રોડક્ટ દ્વારા મેચ ન કરાયેલા તમામ TLDs પર લાગુ થશે.

તમારે TLDs ની કોમાથી અલગ કરેલી યાદી દાખલ કરવી પડશે (ઉદાહરણ તરીકે: `.com, .net, .org`) જેથી એક **TLD-specific product** બની શકે જે ફક્ત તે એક્સટેન્શન્સ માટે જ લાગુ પડે છે.

**પ્રોડક્ટ મેચિંગ કેવી રીતે કામ કરે છે:** જ્યારે કોઈ ગ્રાહક ડોમેન શોધે છે, ત્યારે આ એડ-ઓન સૌથી ચોક્કસ મેચ કરતી પ્રોડક્ટ પસંદ કરે છે. તેના TLD લિસ્ટમાં `.com` ધરાવતો પ્રોડક્ટ કેચ-ઓલ પ્રોડક્ટ કરતાં વધુ પ્રાધાન્ય ધરાવે છે. જો કોઈ TLD-વિશિષ્ટ પ્રોડક્ટ મેચ ન થાય, તો કેચ-ઓલનો ઉપયોગ થાય છે. જો કોઈ પ્રોડક્ટ ન હોય, તો ડોમેન શોધ દેખાતી નથી.

### માર્કઅપ પ્રકાર (Markup type) {#markup-type}

તમારી રિટેલ કિંમત વેરહાઉસ કોસ્ટમાંથી કેવી રીતે ગણવામાં આવે છે તે નિયંત્રિત કરવા માટે ત્રણ મોડ છે:

| મોડ | તે કેવી રીતે કામ કરે છે |
|---|---|
| **Percentage** | વેરહાઉસ કોસ્ટ પર એક ટકા વધારો ઉમેરે છે. $10 ના વેરહાઉસ ડોમેન પર 20% માર્કઅપ એટલે $12 મળે છે. |
| **Fixed markup** | એક નિશ્ચિત ડોલર રકમ ઉમેરે છે. $10 ના ડોમેન પર $5 માર્કઅપ એટલે $15 મળે છે. |
| **Fixed price** | વેરહાઉસ કોસ્ટને સંપૂર્ણપણે અવગણે છે. તમે જે રકમ દાખલ કરો છો તે જ ચાર્જ કરે છે. |

### પરિચયક કિંમત નિર્ધારણ (Introductory pricing) {#introductory-pricing}

### WHOIS గోప్యత (WHOIS privacy) {#whois-privacy}

ఈ ఉత్పత్తి ద్వారా నమోదు చేయబడిన డొమైన్‌లకు WHOIS గోప్యతను అందించాలా వద్దా అని నియంత్రిస్తుంది.

| సెట్టింగ్ | ప్రవర్తన |
|---|---|
| **Disabled** | WHOIS గోప్యత ఎప్పుడూ అందించబడదు లేదా ఎనేబుల్ చేయబడదు. |
| **Always Included** | నమోదు సమయంలో స్వయంచాలకంగా WHOIS గోప్యత ఛార్జ్ లేకుండా ఎనేబుల్ చేయబడుతుంది. |
| **Customer Choice** | చెక్అవుట్ సమయంలో ఒక చెక్‌బాక్స్ కనిపిస్తుంది. మీరు **Privacy price** ను సంవత్సరానికి ఛార్జ్ చేయడానికి సెట్ చేయవచ్చు, లేదా ఉచితంగా అందించడానికి దానిని $0 గా వదిలివేయవచ్చు. |

Namecheap కోసం, WHOIS గోప్యత WhoisGuard (ఎల్లప్పుడూ ఉచితం) ను ఉపయోగిస్తుంది. OpenSRS కోసం, ఇది OpenSRS గోప్యత సేవను ఉపయోగిస్తుంది (Wholesale లో ఖర్చు ఉండవచ్చు).

## TLD దిగుమతి మరియు సింక్ (TLD import and sync) {#tld-import-and-sync}

మీ డొమైన్ ఉత్పత్తులు కనెక్ట్ చేయబడిన ప్రొవైడర్ నుండి తీసుకున్న రియల్-టైమ్ హోల్‌సేల్ ధరలను చూపుతాయి. ఇది పనిచేయడానికి, TLD లను దిగుమతి చేసుకోవాలి.

- **Manual sync (మాన్యువల్ సింక్):** Settings › Domain Seller › Sync TLDs (per provider)
- **Automatic sync (ఆటోమేటిక్ సింక్):** అన్ని కాన్ఫిగర్ చేసిన ప్రొవైడర్‌ల ద్వారా రోజుకు ఒక షెడ్యూల్డ్ cron job ద్వారా రన్ అవుతుంది.

సింక్ చేసిన తర్వాత, ఏదైనా డొమైన్ ఉత్పత్తి యొక్క Domain Settings ట్యాబ్‌కు వెళ్లి, అందుబాటులో ఉన్న TLD లను వాటి ప్రస్తుత హోల్‌సేల్ ధరలతో చూడటానికి TLD పిక్కర్‌ను ఉపయోగించండి.

## ఆటో-రీన్యూవల్ (Auto-renewal) {#auto-renewal}

డొమైన్ పునరుద్ధరణలు కస్టమర్ సభ్యత్వ స్థితికి అనుసంధానించబడి ఉంటాయి:

- ఒక సభ్యత్వం పునరుద్ధరించబడినప్పుడు మరియు డొమైన్ అనుసంధానించబడినప్పుడు, డొమైన్ పునరుద్ధరణ ఆటోమేటిక్‌గా వేచి ఉంటుంది.
- పునరుద్ధరణ ప్రయత్నాలు కస్టమర్ యొక్క చురుకైన పేమెంట్ గేట్‌వేను ఉపయోగిస్తాయి.
- విఫలమైన పునరుద్ధరణలు ఎక్స్‌పోనెన్షియల్ బ్యాక్‌ఆఫ్ (exponential backoff) తో ఆటోమేటిక్‌గా మళ్లీ ప్రయత్నించబడతాయి.
- విజయవంతమైన పునరుద్ధరణలు, వైఫల్యాలు మరియు రాబోయే గడువు తేదీల కోసం ఈమెయిల్ నోటిఫికేషన్‌లు పంపబడతాయి.

డొమైన్ లైఫ్‌సైకిల్ ఈవెంట్‌ల కోసం ఈమెయిల్ టెంప్లేట్ ID లు:

| සිදුවීම | Template ID |
|---|---|
| ડોમેન ලියාපදිංචි කිරීම | `domain_registered` |
| ડોમેન අලුත් කිරීම | `domain_renewed` |
| අලුත් කිරීම අසාර්ථක වීම | `domain_renewal_failed` |
| ડોමේනය ඉක්මනින් අවසන් වීම | `domain_expiring_soon` |

## පරිපාලක: අතින් ડોමේන් ලියාපදිංචි කිරීම {#admin-manual-domain-registration}

ගනුදෙනුකරුවෙකු වෙනුවෙන් ඔවුන් Checkout හරහා යන්නේ නැතිව, ඔවුන්ට නම්‍යශීලී ලෙස (manually) ડોමේනයක් ලියාපදිංචි කිරීමට මෙය භාවිතා කරන්න:

1. **Network Admin › Ultimate Multisite › Register Domain** වෙත යන්න
2. ගනුදෙනුකරුවෙකු තෝරාගෙන ડોමේන් නාමය ඇතුළත් කරන්න
3. ලියාපදිංචි කරන්නාගේ සම්බන්ධතා විස්තර (නම, ලිපිනය, දුරකථන අංකය) පුරවන්න
4. **Register** ක්ලික් කරන්න

මෙම අවස්ථාවේදී ડોමේන් වාර්තාව නිර්මාණය වී ගනුදෙනුකරුවාගේ ගිණුමට සම්බන්ධ වේ.
