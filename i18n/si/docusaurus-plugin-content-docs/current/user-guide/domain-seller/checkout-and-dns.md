# චෙක්අවුට් ෆීල්ඩ් සහ පාරිභෝගික DNS කළමනාකරණය

## ડોමේන් තේරීමේ චෙක්අවුට් ෆීල්ඩ් එක

**Domain Selection** ෆීල්ඩ් එක කියන්නේ පාරිභෝගිකයින්ට ඔවුන්ගේ වෙබ් අඩවියේ ડોමේන් ලබා ගැනීමට ඇති විකල්පය දෙන චෙක්අවුට් මූලද්‍රව්‍යයක්. ඔබේ චෙක්අවුට් ෆෝම් එකකට මෙය එකතු කිරීමෙන් ડોමේන් විකිණීම සක්‍රීය කළ හැකියි.

### චෙක්අවුට් ෆෝම් එකකට ෆීල්ඩ් එක එකතු කිරීම

1. **Network Admin › Ultimate Multisite › Checkout Forms** වෙත යන්න
2. චෙක්අවුට් ෆෝම් එකක් විවෘත කරන්න හෝ සාදන්න
3. චෙක්අවුට් සංස්කාරකයේ (editor) **Add Field** ක්ලික් කරන්න
4. ෆීල්ඩ් ලැයිස්තුවෙන් **Domain Selection** තෝරන්න
5. ෆීල්ඩ් විකල්ප වින්‍යාස කරන්න (පහත බලන්න)
6. ෆෝම් එක සුරකින්න

### ෆීල්ඩ් විකල්ප

**Domain modes** — පාරිභෝගිකයාට පෙනෙන ටැබ් (tabs) තෝරා ගන්න. සෑම මාදිලියක්ම ස්වාධීනව සක්‍රීය කිරීමට හෝ අක්‍රීය කිරීමට හැකිය:

| Mode | එය කරන්නේ කුමක්ද |
|---|---|
| **Subdomain** | පාරිභෝගිකයා ඔබේ ජාලයේ නොමිලේ උප-โดමේනයක් භාවිතා කරයි (උදාහරණයක් ලෙස: `mysite.yournetwork.com`). ගෙවීමක් අවශ්‍ය නැත. |
| **Register New Domain** | පාරිභෝගිකයා නව ડોමේනයක් සොයාගෙන ඔබ වින්‍යාස කළ සපයන්නා හරහා එය ලියාපදිංචි කරයි. මිල ගණන් සැකසීම සඳහා අනුරූප වන නිෂ්පාදනය භාවිතා කරයි. |
| **Existing Domain** | පාරිභෝගිකයා දැනටමත් හිමි කරගෙන ඇති ડોමේනයක් මගින් සම්බන්ධ කරයි. ලියාපදිංචි ගාස්තුවක් නැත. එම ડોමේනය ඔවුන්ගේ වෙබ් අඩවියට ස්වයංක්‍රීයව සම්බන්ධ වේ. |

**Default mode** — මෙම මාදිලි තුනම සක්‍රීය වූ විට, කුමන ටැබ් එක මුලින් විවෘත වේද යන්න තීරණය කරන්න. ડોමේන් ලියාපදිංචිය විකල්පයක් ලෙස තබා ගැනීමට **Subdomain** ලෙස හෝ මිලදී ගැනීම් දිරිමත් කිරීමට **Register New Domain** ලෙස සැකසිය හැකිය.

**Domain product** — මෙම ෆීල්ඩ් එක නිශ්චිත ડોමේන් නිෂ්පාදනයකට සම්බන්ධ කිරීමට (pin) ඔබට අවශ්‍ය නම්, එය විකල්පයක් ලෙස කළ හැකිය. මෙය නොමැති නම්, පාරිභෝගිකයා සොයන TLD (Top-Level Domain) මත පදනම්ව අනුරූප නිෂ්පාදනය ස්වයංක්‍රීයව තෝරා ගනී.

### ලියාපදිංචි කරන්නා සම්බන්ධතා ෆීල්ඩ්

පාරිභෝගිකයා **Register New Domain** ටැබ් එක තෝරා ගන්නා විට, චෙක්අවුට් ෆෝම් එකේ පහත සඳහන් දත්ත ස්වයංක්‍රීයව ඇතුළත් වේ:

- නම / මුල් නම
- විද්‍යුත් තැපැල් ලිපිනය (Email address)
- ලිපිනය (வரி 1, නගරය, ප්‍රාන්තය/ප්‍රොvinces, තැපැල් කේතය, රට)
- දුරකථන අංකය

සියලුම ලියාපදිංචි කරන්නන් විසින් මෙය අවශ්‍ය වන අතර ලියාපදිංචිය සඳහා API ඇමතුමක් කිරීමට පෙර එය වලංගු කරනු ලැබේ. දුරකථන අංක ස්වයංක්‍රීයව ලියාපදිංචි කරන්නන් අපේක්ෂා කරන `+CC.NNN` ජාත්‍යන්තර ආකෘතියට හැඩගස්වේ.

### ස්වයංක්‍රීය වෙබ් URL (Auto-generated site URL)

පාරිභෝගිකයෙකු ලියාපදිංචි වූ විට හෝ යම් ડોමේනයක් සිතියම් කළ විට, වෙබ් URL ක්ෂේත්‍රය තෝරාගත් ડોමේനിൽ നിന്ന് ස්වයංක්‍රීයව පිරවිය හැක. පාරිභෝගිකයින්ට වෙනම URL ක්ෂේත්‍රයක් පුරවන්න අවශ්‍ය නොවේ.

### සෙවුම් හැසිරීම (Search behaviour)

- පාරිභෝගිකයා ටයිප් කරන විට AJAX මගින් තත්‍ය කාලයේදී ડોමේන් ලබා ගත හැකි බව පරීක්ෂා කෙරේ
- ප්‍රමුඛ පෙළේ ඇති ડોමේනය නොමැති විට විකල්ප TLD යෝජනා පෙන්වනු ලැබේ
- මිල ගණන් සජීවීව ලබාගෙන පැහැදිලිව පෙන්වනු ලැබේ (ලියාපදිංචි මිල, අලුත් කිරීමේ මිල, විකල්ප WHOIS රහස්‍යතා ගාස්තුව)
- කූපන් කේත ඕනෑම වෙනත් නිෂ්පාදනයකට සමානව ડોමේන් නිෂ්පාදන සඳහා වලංගු වේ

**සෙවුම් ප්‍රතිචාර දැක්වීම (search responsiveness) සකස් කිරීම:**

```php
// API ඇමතුම් අඩු කිරීමට ප්‍රමාදය වැඩි කරන්න (millisecond වලින්) මන්දගාමී සම්බන්ධතා සඳහා
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // පෙරනිමිය: 500
});
```

**ඩොමේන් සෙවුම් පෝරමය සඳහා අභිරුචි ක්ෂේත්‍ර (custom fields) එකතු කිරීම:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'අමතර සටහන්',
    ];
    return $fields;
});
```

## පාරිභෝගික DNS කළමනාකරණය (Customer DNS management)

පාරිභෝගිකයින්ට ඔවුන් ලියාපදිංචි කර ඇති ડોමේන් සඳහා **My Account** පිටුවෙන්, ඔවුන්ගේ ડોමේනයේ ඇතුළත් කිරීම යටතේ DNS වාර්තා කළමනාකරණය කළ හැකිය.

### සහාය දක්වන වාර්තා වර්ග (Supported record types)

| වර්ගය | භාවිතය |
|---|---|
| **A** | නාමය IPv4 ලිපිනයට සම්බන්ධ කිරීම |
| **AAAA** | නාමය IPv6 ලිපිනයට සම්බන්ධ කිරීම |
| **CNAME** | තවත් නාමයකට යොමු කරන විකල්පයක් (alias) නිර්මාණය කිරීම |
| **MX** | විද්‍යුත් තැපැල් හුවමාරු සේවාදායකය සැකසීම |
| **TXT** | SPF, DMARC, ధృవీకరణ හෝ වෙනත් පෙළ වාර්තා එකතු කිරීම |

### DNS කළමනාකරණය සඳහා කුමන සපයන්නන් (providers) සහාය දක්වන්නේද?

DNS කළමනාකරණය (පුවරුවක් එකතු කිරීම, සංස්කරණය කිරීම, මකා දැමීම) සඳහා **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, සහ **Openprovider** සමඟ පහසුකම් ඇත. **Hostinger** ડોමේන් වල නම් සේවාදායකයන් (nameservers) වෙනස් කිරීමට Domain Seller හරහා කළ හැකිය; සත්කාරක කරන ලද (hosted) වෙබ් අඩවි සඳහා DNS වාර්තා කළමනාකරණය කරනු ලබන්නේ ප්‍රධාන Hostinger domain-mapping integration මගිනි. Namecheap, GoDaddy සහ NameSilo ડોමේන් තත්ත්වය සහ කල් ඉකුත්වීමේ තොරතුරු පෙන්වුවද, DNS සේවාදායකයන් (DNS) නියාමක පාලන පැනල තුළ කෙලින්ම කළමනාකරණය කළ යුතුය.

### පෙරනිමි DNS වාර්තා (Default DNS records)

ඔබට යම් ડોමේනයක් ලියාපදිංචි කරන විට ස්වයංක්‍රීයව යොදනු ලබන පෙරනිමි DNS වාර්තා වින්‍යාස කළ හැකිය. **Settings › Domain Seller › Default DNS Records** වෙත යන්න.

පෙරනිමි වාර්තා අගයන්ට ටෝකන් දෙකක් සඳහා සහාය වේ:

| Token | ප්‍රතිස්ථාපනය කරනු ලබන්නේ |
|---|---|
| `{DOMAIN}` | ලියාපදිංචි කරන ලද ડોමේන් නාමය (උදාහරණයක් ලෙස, `example.com`) |
| `{SITE_URL}` | පාරිභෝගිකයාගේ වෙබ් අඩවිය සඳහා WordPress site URL |

**උදාහරණය — ප්‍රධාන ડોමේනය සහ www එක ඔබේ සේවාදායක IP වෙත යොමු කිරීම:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### පරිපාලක: DNS බැලීම සහ සංස්කරණය කිරීම (Admin: viewing and editing DNS)

ජාල පරිපාලකයින්ට ඕනෑම පාරිභෝගික ડોමේනයක DNS වාර්තා බැලීමට සහ සංස්කරණය කිරීමට **Network Admin › Ultimate Multisite › Domains** වෙතින් එම ડોමේනයේ සංස්කරණ පිටුවෙන් කළ හැකිය.
