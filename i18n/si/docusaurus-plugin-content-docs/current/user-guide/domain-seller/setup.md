# ડોમેન સેલર: સેટઅપ અને પ્રોવાઇડર રૂપરેખાંકન {#domain-seller-setup-and-provider-configuration}

ડોમેન સેલર એડ-ઓન એક માર્ગદર્શિત સેટઅપ વિઝાર્ડ સાથે આવે છે જે તમને દરેક જરૂરી પગલાંમાંથી પસાર થવામાં મદદ કરે છે. આ પેજમાં વિઝાર્ડના પ્રવાહ અને પછી પ્રોવાઇડર્સને કેવી રીતે રૂપરેખાંકિત કરવા અથવા ફરીથી રૂપરેખાંકિત કરવા તે વિશે માહિતી આપવામાં આવી છે.

## આવશ્યકતાઓ {#requirements}

- **Multisite Ultimate** v2.4.12 અથવા તેનાથી ઉપર, નેટવર્ક-એક્ટિવેટેડ
- **PHP** 7.4+
- ઓછામાં ઓછી એક સપોર્ટેડ રજિસ્ટ્રાર માટે API ક્રેડેન્શિયલ્સ

## પ્રથમ વખત સેટઅપ વિઝાર્ડ {#first-run-setup-wizard}

સેટઅપ વિઝાર્ડ જ્યારે તમે પહેલીવાર પ્લગઇન નેટવર્ક-એક્ટિવેટ કરો છો ત્યારે આપમેળે શરૂ થાય છે. તે **Network Admin › Ultimate Multisite › Domain Seller Setup** માંથી કોઈપણ સમયે ઉપલબ્ધ હોય છે.

### પગલું ૧ — એક પ્રોવાઇડર પસંદ કરો {#step-1--choose-a-provider}

તમે કયા રજિસ્ટ્રાર સાથે જોડવા માંગો છો તેને પસંદ કરો. સપોર્ટેડ વિકલ્પો:

| પ્રોવાઇડર | DNS મેનેજમેન્ટ | WHOIS પ્રાઇવસી |
|---|---|---|
| OpenSRS | હા | હા |
| Namecheap | ના | હા (WhoisGuard, ફ્રી) |
| HostAfrica | હા | હા (ID સુરક્ષા) |
| Openprovider | હા | હા |
| Hostinger | હોસ્ટેડ ડોમેન્સ માટે મુખ્ય Hostinger ડોમેન મે mapping દ્વારા | હા |
| GoDaddy | ના | ના |
| ResellerClub | હા | ના |
| NameSilo | ના | ના |
| Enom | હા | ના |

### પગલું ૨ — ક્રેડેન્શિયલ્સ દાખલ કરો {#step-2--enter-credentials}

દરેક પ્રોવાઇડર પાસે અલગ-અલગ ક્રેડેન્શિયલ ફીલ્ડ હોય છે:

**OpenSRS** — યુઝરનેમ અને પ્રાઇવેટ કી (OpenSRS રિસેલર કંટ્રોલ પેનલમાંથી)

**Namecheap** — યુઝરનેમ અને API કી (Account › Tools › API Access માંથી)

**HostAfrica** — HostAfrica રિસેલર મોડ્યુલમાંથી ડોમેન્સ રિસેલર API એન્ડપોઇન્ટ અને ક્રેડેન્શિયલ્સ. હાલમાં કોઈ અલગ સેન્ડબોક્સ એન્ડપોઇન્ટ દસ્તાવેજીકૃત નથી; લાઇવ રજિસ્ટ્રેશન ચલાવવા કરતા પહેલા સુરક્ષિત રીડ-ઓનલી ચેક સાથે પરીક્ષણ કરો.

**Openprovider** — API એક્સેસ સક્ષમ યુઝરનેમ અને પાસવર્ડ. વૈકલ્પિક સેન્ડબોક્સ મોડ માટે Openprovider સેન્ડબોક્સ API નો ઉપયોગ થાય છે, અને રજિસ્ટ્રેશન માટે એક વૈકલ્પિક ડિફોલ્ટ ગ્રાહક હેન્ડલનો ફરીથી ઉપયોગ કરી શકાય છે.

**Hostinger** — ප්‍රධාන Hostinger ඒකාබද්ධ කිරීමෙන් ලැබෙන හවුල් Hostinger hPanel API ටෝකන් එක. එම ටෝකනය මගින් ප්‍රධාන ડોමේන් මැපින් සහ Domain Seller ලියාපදිංචි කිරීමේ මෙහෙයුම් සිදු කෙරේ.

**GoDaddy** — API key සහ secret (developer.godaddy.com වෙතින්)

**ResellerClub** — Reseller ID සහ API key (ResellerClub control panel එකෙන්)

**NameSilo** — API key (namesilo.com › Account › API Manager වෙතින්)

**Enom** — Account ID සහ API token

ක්‍රියාත්මක කිරීමට පෙර, ලබා ගත හැකි නම් **Sandbox mode** පරීක්ෂා කරන්න. එසේ කිරීමෙන් සැබෑ පරිසරයට යාමට පෙර සැපයුම්කරුවගේ පරීක්ෂණ පරිසරය සමඟ පරීක්ෂා කළ හැකිය.

### පියවර 3 — සම්බන්ධතාවය පරීක්ෂා කිරීම {#step-3--test-the-connection}

**Test Connection** ක්ලික් කරන්න. මෙම wizard එක මගින් නිවැරදි තොරතුරු සහ සම්බන්ධතාවය තහවුරු කර ගැනීම සඳහා සැහැල්ලු API ඇමතුමක් යවයි. ඉදිරියට යාමට පෙර ඕනෑම තොරතුරු ගැටළු නිවැරදි කරන්න.

### පියවර 4 — TLDs ආනයනය කිරීම (Import) {#step-4--import-tlds}

සම්බන්ධිත සපයන්නාගෙන් සියලුම ලබා ගත හැකි TLDs සහWholesale මිල ගණන් ලබා ගැනීමට **Import TLDs** ක්ලික් කරන්න. මෙය ડોමේන් නිෂ්පාදන සඳහා භාවිතා කරන TLD ලැයිස්තුව පුරවනු ඇත. විශාල TLD දත්ත සමුදායන් සහිත සපයන්නන් සඳහා ආනයනය කිරීමට තත්පර 30-60ක් ගත විය හැකිය.

සැලසුම් කළ cron job එකක් හරහා දිනකට වරක් TLDs ස්වයංක්‍රීයව නැවත සමමුහුර්ත (re-synced) වේ.

### පියවර 5 — ડોමේන් නිෂ්පාදනයක් නිර්මාණය කිරීම {#step-5--create-a-domain-product}

මෙම wizard එක මගින් 10% markup සහිත පෙරනිමි catch-all ડોමේන් නිෂ්පාදනයක් සාදනු ලැබේ. ඔබට මෙම නිෂ්පාදනය වහාම සංස්කරණය කළ හැකිය, නැතහොත් **Ultimate Multisite › Products** යටතේ අතින් නිෂ්පාදන නිර්මාණය කිරීම මගින් එය මඟ හැරිය හැකිය.

සම්පූර්ණ නිෂ්පාදන සැකසුම් මාර්ගෝපදේශය සඳහා [Domain Products and Pricing](./domain-products) බලන්න.

## සපයන්නෙකු නැවත වින්‍යාස කිරීම (Reconfiguring a provider) {#reconfiguring-a-provider}

**Network Admin › Ultimate Multisite › Settings › Domain Seller** වෙත යන්න (හෝ plugin list එකේ **Settings** ක්ලික් කරන්න).

මෙම සැකසුම් පිටුව පහත දේ අඩංගු වේ:

- **Domain විකිණීම සක්‍රීය කිරීම** — මෙම විශේෂාංගය සම්පූර්ණයෙන්ම ක්‍රියාත්මක කිරීමට/ක්‍රියාවිරහිත කිරීමට (on/off) ස්විචය භාවිතා කරන්න.
- **ප්‍රධාන සැපයුම්කරු (Default provider)** — ડોමේන් සෙවීම් සහ නව නිෂ්පාදන සඳහා භාවිතා කරන සැපයුම්කරු.
- **සෑම සෙවුමකට උපරිම TLD ගණන** — පාරිභෝගිකයෙකු සෙවූ විට පරීක්ෂා කළ යුතු TLD ගණන; ඉහළ අගයන් වැඩි විකල්ප පෙන්වන නමුත් ඒවා මන්දගාමී වේ.
- **ප්‍රමාණය තිබීමේ දත්ත (Availability cache duration)** — ලබා ගැනීමේ සහ මිල තොරතුරු ගබඩා කර තබා ගත යුතු කාලය; අඩු අගයන් වඩාත් නිවැරදි වුවද API ඇමතුම් වැඩි කරයි.
- **डोमेन නිෂ්පාදන කළමනාකරණය කිරීම** — නිෂ්පාදන ලැයිස්තුවට ඉක්මන් සබැඳියක්.
- **සැපයුම්කරුවන් වින්‍යාස කිරීම (Configure providers)** — සැපයුම්කරුවන් එකතු කිරීමට හෝ නැවත වින්‍යාස කිරීමට Integration Wizard විවෘත කරයි.

### දෙවන සැපයුම්කරු එකතු කිරීම {#adding-a-second-provider}

**Configure providers** ක්ලික් කර නව ලියාපදිංචි කරන්නා සඳහා wizard එක නැවත ක්‍රියාත්මක කරන්න. ඔබට එකවර බහු සැපයුම්කරුවන් වින්‍යාස කළ හැකිය. සෑම ડોමේන් නිෂ්පාදනයක්ම නිශ්චිත සැපයුම්කරුවකට පැවරීමට හෝ එය පෙරනිමි (default) ලෙස තබා ගත හැකිය.

### TLDs අතින් සමමුහුර්ත කිරීම (Syncing TLDs manually) {#syncing-tlds-manually}

සැපයුම්කරු වින්‍යාස කළ ඕනෑම සැපයුම්කරුවක් අසල **Sync TLDs** ක්ලික් කර නව මිල ගණන් ලබා ගැනීමට භාවිතා කරන්න. සැපයුම්කරු විසින් සිල්ලර මිල (wholesale pricing) යාවත්කාලීන කිරීමෙන් පසු හෝ නව TLDs එකතු කිරීමෙන් පසු මෙය ප්‍රයෝජනවත් වේ.

## සපයන්නා හැකියාවන් පිළිබඳ සටහන් {#provider-capability-notes}

සෑම ලේඛකයෙකුගේම API එකක් එකම ක්‍රියාකාරකම් ප්‍රදර්ශනය කරන්නේ නැහැ. අනුබද්ධ (addon) මගින් සහාය නොලදන ලද ක්‍රියාකාරකම් පැහැදිලි පරිපාලක දෝෂ සහිතව පෙන්වනවා, ඒවා නිහඬව අසාර්ථක වීම වෙනුවට.

- **HostAfrica** යනු සරලම ලොකුම ප්‍රතිලාභ සහිත (live reseller) වැඩ ප්‍රවාහය සඳහා සහාය දක්වන API එකක් වන අතර එය සොයා බැලීම (lookup), TLD/මිල ගණන් සමමුහුර්ත කිරීම, ලියාපදිංචිය, අලුත් කිරීම, මාරු කිරීම, නාමය සේවා යාවත්කාලීන කිරීම, DNS වාර්තා, EPP කේත, ලේඛක අගුළු දැමීම සහ හැඳුනුම්ක ආරක්ෂාව ඇතුළුව සියල්ල ආවරණය කරයි.
- **Openprovider** මගින් ප්‍රතිලාභ සහිත මිලට TLD සමමුහුර්ත කිරීම, ලියාපදිංචිය, අලුත් කිරීම, මාරු කිරීම්, නාමය සේවා යාවත්කාලීන කිරීම, DNS කලාප (zones), EPP කේත, ලේඛක අගුළු දැමීම සහ WHOIS පෞද්ගලිකත්වය සඳහා සහාය දක්වයි. එය කෙටි කාලයක් ක්‍රියාත්මක වන bearer token එකකින් සත්‍යාපනය කරනු ලබන අතර එම add-on මගින් ස්වයංක්‍රීයව නැවුම් කරයි.
- **Hostinger** මගින් හවුල් hPanel API ටෝකනය හරහා ලබා දෙන සොයා බැලීමේ විභවයන් (availability search), ලියාපදිංචිය, කළඹ පරීක්ෂාව (portfolio lookup), නාමය සේවා යාවත්කාලීන කිරීම, ලේඛක අගුළු දැමීම සහ WHOIS පෞද්ගලිකත්වය සඳහා සහාය දක්වයි. Hostinger හි පොදු Domains API මගින් ප්‍රතිලාභ සහිත/Wholesale මිල ගණන්, ආපසු මාරු කිරීම් (inbound transfer), පැහැදිලි අලුත් කිරීම් හෝ EPP කේත ලබා ගැනීම ප්‍රදර්ශනය නොකරයි; අලුත් කිරීම් ස්වයංක්‍රීයව පමණක් සිදු වේ.
