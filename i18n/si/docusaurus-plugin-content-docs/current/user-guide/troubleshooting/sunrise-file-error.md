# Sunrise ගොනුව ස්ථාපනය කිරීමේදී ඇතිවන දෝෂය

sunrise.php කියන්නේ WordPress තමන්ව ආරම්භ කරන (bootstraps) විට සොයා ගන්නා විශේෂ ගොනුවක්. WordPress එකට sunrise.php ගොනුව හඳුනා ගැනීමට හැකි වෙන්න නම්, ඒක **wp-content ෆෝල්ඩරය** ඇතුළේ තිබිය යුතුයි.

ඔබ Ultimate Multisite සක්‍රීය කර, ඔබ ඡායාරූපයේ පෙන්වා ඇති ආකාරයේ setup wizard එක හරහා ගිය විට, Ultimate Multisite අපගේ sunrise.php ගොනුව wp-content ෆෝල්ඩරයට පිටපත් කිරීමට උත්සාහ කරනවා.

බොහෝ වෙලාවට, අපි සාර්ථකව ගොනුව පිටපත් කරලා වැඩ කරන්න පුළුවන්. නමුත් යම් දෙයක් නිසි ලෙස සකස් කර නොමැති නම් (උදාහරණයක් ලෙස ෆෝල්ඩර අවසරයන්), Ultimate Multisite එකට ගොනුව පිටපත් කිරීමට නොහැකි වන තත්වයකට මුහුණ දීමට සිදු විය හැකියි.

Ultimo ඔබට ලබා දෙන error message එක කියවූ විට, මෙහිදී හරියටම සිදුවූ දේ ඔබට පෙනේ: **Sunrise copy failed** (sunrise පිටපත් කිරීම අසාර්ථක විය).

එය නිවැරදි කිරීමට, ඔබට කළ යුත්තේ wp-ultimo plugin ෆෝල්ඩරය තුළ ඇති sunrise.php ගොනුව පිටපත් කර එය ඔබේ wp-content ෆෝල්ඩරයට ඇලවීමයි. ඔබ එය කළ පසු, wizard පිටුව නැවත පටවන්න (reload) සහ පරීක්ෂාවන් සාර්ථක විය යුතුය.

ඕනෑම අවස්ථාවක, අනාගතයේදී Ultimate Multisite වලට පමණක් නොව වෙනත් plugin කිසිවෙකුට හෝ theme වලටද ගැටළු ඇතිවීම වළක්වා ගැනීම සඳහා ඔබේ ෆෝල්ඩර අවසරයන් පිළිබඳව සමස්ත පරීක්ෂාවක් කිරීම සුදුසු විය හැකියි.

WordPress හි අංගයක් වන **Health Check tool** (ඔබට එය ප්‍රධාන site එකේ **admin panel > Tools > Health Check** හරහා පිවිසිය හැකිය) මගින් WordPress වලට ගැටළු ඇති කළ හැකි වටිනාකම් සහිත ෆෝල්ඩර අවසරයන් ඔබ සතු දැයි ඔබට දැන ගැනීමට හැකියාව ඇත.
