# පාරිභෝගික ප්‍රතිපෝෂණ සහ ගැටළු වාර්තා කිරීම

Gratis AI Agent v1.5.0 සමඟ, අවසන් පරිශීලකයන්ට උපකාර වන නොහැකි පිළිතුරු සලකුණු කිරීමට සහ චැට් අතුරුමුහුණතේ සිටම ප්‍රශ්න වාර්තා කිරීමට හැකි අභ්‍යන්තර ප්‍රතිපෝෂණ සහ ගැටළු වාර්තා කිරීමේ පද්ධතියක් හඳුන්වා දී ඇත. මෙම පද්ධතියට කැමැත්ත කළමනාකරණය (consent management), ස්වයංක්‍රීය වාර්තා කිරීමේ විධානයක්, සහ පසුබිමේ AI ආධාරයෙන් වර්ගීකරණය කිරීම (triage) ඇතුළත් වේ.

## අකුරු පහත දැමීමේ බොත්තම (Thumbs-Down Button)

AI සහායකයා විසින් යවන සෑම පණිවිඩයක්ම **අකුරු පහත දැමීමේ** (👎) බොත්තමක් පෙන්වයි. පරිශීලකයෙකු එය ක්ලික් කළ විට, ඔවුන්ට පිළිතුර අනුග්‍රහ නොකරන හෝ වැරදි බව සලකුණු කළ හැකිය.

- එක් එක් සහායක පණිවිඩයක් අසල මූසික චලනය (hover) කරන විට බොත්තම දිස්වේ.
- එය ක්ලික් කිරීමෙන් **ප්‍රතිපෝෂණ කැමැත්ත මොඩලය** (Feedback Consent Modal) විවෘත වේ.
- ප්‍රතිපෝෂණය සංවාද තීරුවට සහ නිශ්චිත පණිවිඩයට සම්බන්ධ වේ.

## ප්‍රතිපෝෂණ කැමැත්ත මොඩලය (Feedback Consent Modal)

පරිශීලකයෙකු අකුරු පහත දැමීමේ බොත්තම ක්ලික් කළ විට, කිසිදු දත්තයක් යැවීමට පෙර කැමැත්ත මොඩලයක් දිස්වේ. මෙම මොඩලය:

- එකතු කරන තොරතුරු (සංවාද උපුටනය, බ්‍රව්සර සන්දර්භය) පැහැදිලි කරයි.
- මෙම දත්ත බෙදා ගැනීමට පරිශීලකයාගේ කැමැත්ත ඇති බව තහවුරු කිරීමට අසයි.
- වැරදුණු දේ විස්තර කිරීමට පරිශීලකයාට නිදහස් පෙළ ක්ෂේත්‍රයක් සපයයි.
- ඉදිරිපත් නොකර ඉවත් කිරීමට **අවලංගු කිරීම** (Cancel) විකල්පයක් ලබා දෙයි.

පරිශීලකයා පැහැදිලිව තහවුරු කරන තුරු කිසිදු ප්‍රතිපෝෂණයක් වාර්තා නොවේ.

## ස්වයංක්‍රීය ප්‍රේරකය සහිත ප්‍රතිපෝෂණ බැනරය (Auto-Prompt Feedback Banner)

සංවාදයක අවසානයේදී, සහායකයා විසින් **ස්වයංක්‍රීය ප්‍රේරකය සහිත ප්‍රතිපෝෂණ බැනරයක්** (auto-prompt feedback banner) පෙන්විය හැකිය — එය සැසිය ප්‍රයෝජනවත් වූයේ දැයි විමසන නොගැඹුරු පණිවිඩයකි.

මෙම බැනරය සංවාදයේ දිග සහ ප්‍රතිඵල පිළිබඳ උපකල්පිත ක්‍රම (heuristics) මත ස්වයංක්‍රීයව දිස්වේ. එය අකුරු පහත දැමීමේ බොත්තමට සමාන ප්‍රතිපෝෂණ ප්‍රවාහයට සම්බන්ධ වේ. ප්‍රතිපෝෂණ ලබා නොදමා බැනරය ඉවත් කළ හැකිය.

## /report-issue සලකුණ විධානය (Slash Command)

චැට් ආදානයේ `/report-issue` ලෙස ටයිප් කිරීමෙන් පරිශීලකයින්ට ප්‍රතිපෝෂණ ප්‍රවාහය කෙලින්ම ආරම්භ කළ හැකිය. මෙම විධානය:

ප්‍රතිචාර දැක්වීමේ මුවා (Feedback Consent Modal) වහාම විවෘත කරයි.
වත්මන් සංවාදය පිළිබඳ සන්දර්භය සහිත විස්තර ක්ෂේත්‍රය පෙර-පිරවිය හැක.
සමర్ప කිරීමට පෙර පරිශීලකයින්ට අමතර විස්තර එක් කිරීමට ඉඩ සලසයි.

`/report-issue` කමාන්ඩ් එක සියලු චැට් මාදිලිවල (inline, floating widget, full-screen) සඳහා ලබා ගත හැකිය.

## AI-ආධාරක වර්ගීකරණය (AI-Assisted Triage)

සමర్పಿಸಿದ ප්‍රතිපෝෂණ වාර්තාව AI වර්ගීකරණ ස්ථරයකට යොමු කෙරේ, එය:

- වාර්තාව වර්ගීකරණය කරයි (තොරතුරු දෝෂ, උපකාරී නොවන පිළිතුර, නුසුදුසු අන්තර්ගතය වැනි).
- අදාළ සංවාද 메ටා දත්ත (metadata) එක් කරයි.
- වර්ගීකරණ සාරාංශය වින්‍යාස කළ **Feedback Endpoint** වෙත යවයි (Settings > Advanced බලන්න [Settings > Advanced]).

මෙමගින් වඩාත්ම තීරණාත්මක ගැටළු මුලින්ම ඉස්මතු කිරීමෙන් අතින් සිදු කරන සමාලෝචන කාලය අඩු කරයි.

## Settings > Advanced {#settings-advanced}

ප්‍රතිපෝෂණ පසුබිම (feedback backend) සක්‍රීය කිරීමට, **Gratis AI Agent → Settings → Advanced** යට ඇති පහත ක්ෂේත්‍ර වින්‍යාස කරන්න:

| ක්ෂේත්‍රය | විස්තරය |
|---|---|
| **Feedback Endpoint URL** | ප්‍රතිපෝෂණ දත්ත (JSON) සහිත POST ඉල්ලීම් ලබා ගන්නා URL එක. |
| **Feedback API Key** | ප්‍රතිපෝෂණ ඉදිරිපත් කිරීම් වල සත්‍යාපනය සඳහා `Authorization` header හි යවන Bearer token එක. |

ප්‍රතිපෝෂණ රැස්කිරීම අක්‍රිය කිරීමට මෙම ක්ෂේත්‍ර දෙකම හිස්ව තබන්න. තනි ප්‍රතිපෝෂණ බොත්තම් පරිශීලකයින්ට ද පෙනෙන නමුත්, ඒවා ඉදිරිපත් නොකරනු ලැබේ.

:::tip
ප්‍රතිපෝෂණ එන්ඩ්පොයින්ට් එක මගින් අවම වශයෙන් `message_id`, `conversation_id`, `feedback_text`, සහ `triage_category` ක්ෂේත්‍රයන් සහිත JSON ශරීරයක් (body) පිළිගත යුතුය. අපේක්ෂිත ස්කීමා (schema) සඳහා ඔබේ එන්ඩ්පොයින්ට් සපයන්නාගේ ලේඛන බලන්න.
:::
