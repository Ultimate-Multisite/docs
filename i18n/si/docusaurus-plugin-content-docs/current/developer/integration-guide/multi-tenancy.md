# බහු-කුලීකරණ ඒකාබද්ධ කිරීම

Ultimate Multisite: බහු-කුලීකරණ 1.2.0, ස්වෛරී කුලීකරුවන්, සංක්‍රමණ සත්‍යාපනය, සහ කුලීකරු ජීවනචක්‍ර ස්වයංක්‍රීකරණය සඳහා ඒකාබද්ධ කිරීමේ ස්පර්ශ-ලක්ෂ්‍ය කිහිපයක් වෙනස් කරයි.

## කුලීකරු ආරම්භක ප්‍රවාහය

කුලීකරුවන් සාදන හෝ වෙනස් කරන ඒකාබද්ධ කිරීම් මෙම අනුපිළිවෙල අනුගමනය කළ යුතුය:

1. කුලීකරු රෙජිස්ට්‍රි වාර්තාව සහ වෙන්කිරීමේ ආකෘතිය විසඳන්න.
2. කුලීකරු දත්තගබඩා ලේඛකයා සාදන්න හෝ සත්‍යාපනය කරන්න.
3. කුලීකරු schema ආරම්භ කරන්න.
4. කුලීකරු පරිශීලකයන් සපයන්න.
5. කුලීකරු routing සහ ගොනු පද්ධති paths ලියාපදිංචි කරන්න.
6. කුලීකරු හෙළි කිරීමට පෙර සංක්‍රමණ සත්‍යාපනය ධාවනය කරන්න.

ස්වෛරී කුලීකරුවෙකුට ජාල දත්තගබඩා සම්බන්ධතාව නැවත භාවිත කළ හැකි බව උපකල්පනය නොකරන්න. addon එක සපයන කුලීකරු රෙජිස්ට්‍රි සහ ලේඛක abstraction භාවිත කරන්න.

## SSO සහ REST hooks

තත්ත්ව-රහිත කුලීකරු ස්වයං-පිවිසුම කෙටි-කාලීන tokens භාවිත කරයි. ඒවාට purpose claim එකක්, JTI replay ආරක්ෂාව, expiry cap එකක්, සහ origin pinning ඇත. පිවිසුම් බොත්තම් හෝ දුරස්ථ කළමනාකරණ links එකතු කරන ඒකාබද්ධ කිරීම්, කුලීකරු පිවිසුම් URLs සෘජුව ගොඩනැගීම වෙනුවට සහාය දක්වන SSO ප්‍රවාහය හරහා කුලීකරු visits ජනනය කළ යුතුය.

ජාල-පාර්ශ්ව API විගණන සිදුවීම් සහ දෛනික සාරාංශ ස්වෛරී කුලීකරු gateways සඳහා ලබාගත හැක. කුලීකරු ජීවනචක්‍ර endpoints අමතන බාහිර පද්ධති නිදොස්කරණය කරන විට එම logs භාවිත කරන්න.

## ස්වෛරී පාරිභෝගික ක්‍රියා URLs

Ultimate Multisite v2.13.0, ස්වෛරී-කුලීකරු පාරිභෝගික ක්‍රියා account, checkout, billing, invoice, අඩවිය, template-switching, සහ domain-mapping ප්‍රවාහ සඳහා ප්‍රධාන අඩවියට ආපසු යොමු කරයි. කුලීකරු-පාර්ශ්ව කළමනාකරණ links පෙන්වන ඒකාබද්ධ කිරීම්, එම ක්‍රියා ප්‍රධාන-අඩවි පාරිභෝගික panel එක වෙත යොමු කළ යුතු අතර, ක්‍රියාව සම්පූර්ණ කිරීමෙන් පසු පරිශීලකයාට කුලීකරු වෙත ආපසු යා හැකි විය යුතු නම් සත්‍යාපිත ආපසු-ගමනාන්තයක් ඇතුළත් කළ යුතුය.

cross-domain කළමනාකරණ links සඳහා core SSO wrapper භාවිත කරන්න:

```php
$url = wu_with_sso($main_site_customer_url);
```

ජනනය කළ URL එක `wu_sso_url` හරහා තවදුරටත් filter කළ හැක. එයට SSO URL, වත්මන් පරිශීලකයා, ඉලක්ක අඩවි ID එක, සහ redirect context ලැබේ. Add-ons වලට එම filter භාවිත කර provider-විශේෂිත context එක් කිරීමට හෝ Ultimate Multisite හි token සත්‍යාපනය රැකගනිමින් broker URL එක ප්‍රතිස්ථාපනය කිරීමට හැක.

ස්වෛරී කුලීකරු තුළ membership, invoice, billing-address, template, හෝ domain-management තත්ත්වය අනුපිටපත් නොකරන්න. කුලීකරු dashboard එක launcher එක ලෙසත්, ප්‍රධාන-අඩවි පාරිභෝගික panel එක කළමනාකරණ ක්‍රියා සඳහා සත්‍ය වාර්තා පද්ධතිය ලෙසත් සලකන්න.

## සංක්‍රමණ සත්‍යාපනය

සංක්‍රමණයක් හෝ ජීවනචක්‍ර ඒකාබද්ධ කිරීමක් කුලීකරු දත්ත වෙනස් කළ පසු, සත්‍යාපන gates ධාවනය කරන්න:

- `wp tenant verify-no-legacy --site=<site-id>` මඟින් කුලීකරු තවදුරටත් legacy ජාල-පාර්ශ්ව paths මත රඳා නොපවතින බව තහවුරු කරයි.
- `wp tenant verify-sovereign-push --site=<site-id>` මඟින් ස්වෛරී push jobs සැකසිය සහ drain කළ හැකි බව තහවුරු කරයි.

ඒකාබද්ධ කිරීම් අසාර්ථක සත්‍යාපනය deployment blocker එකක් ලෙස සැලකිය යුතු අතර, අසාර්ථකත්වය විසඳෙන තුරු කුලීකරුවෙකු live ලෙස සලකුණු කිරීමෙන් වළකින්න.

## කුලීකරු මකාදැමීම

මකාදැමීමේ ප්‍රවාහයන් කුලීකරු දත්තගබඩා credentials පිරිසිදු වන පරිදි addon teardown path එක ඇමතිය යුතුය. Teardown සාර්ථක වූ පසු බාහිර ඒකාබද්ධ කිරීම් provider සම්පත් ඉවත් කළ හැක, නමුත් සත්‍යාපනය හෝ async push jobs තවමත් ධාවනය වන අතරතුර host දත්තගබඩා හෝ folders මකා නොදැමිය යුතුය.

## Deprecated දත්තගබඩා router

Legacy `Database_Router` එක deprecation stub එකකින් ප්‍රතිස්ථාපනය කර ඇත. නව ඒකාබද්ධ කිරීම් පැරණි router class එක මත රඳා නොසිට, වත්මන් අඩවි router සහ කුලීකරු රෙජිස්ට්‍රි APIs හරහා කුලීකරුවන් විසඳිය යුතුය.
