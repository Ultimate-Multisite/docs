# Tutor Multisite Compatibility {#tutor-multisite-compatibility}

Hiji pangaya WordPress anu nambihan perbaikan kasaluyuan pikeun Tutor LMS dina lingkungan multisitus.

## Pedaran {#description}

Pangaya ieu nyayogikeun perbaikan kasaluyuan anu penting pikeun ngajalankeun Tutor LMS dina jaringan multisitus WordPress. Ieu ngatur pendaptaran pamaké, pangelolaan instruktur, watesan kursus, jeung fungsionalitas séjén anu husus pikeun multisitus.

## Fitur {#features}

- **Pangaturan Tabel Database**: Sacara otomatis ngadeteksi jeung nyieun tabel Tutor LMS anu leungit anu gagal kusabab konstrain konci asing multisitus
- **Pendaptaran Pamaké Multisitus**: Pangatur AJAX husus pikeun ngatur pendaptaran pamaké di sakuliah situs jaringan
- **Pangaturan Peran Instruktur**: Panetepan jeung pangaturan peran instruktur sacara otomatis dina lingkungan multisitus
- **Watesan Kursus**: Integrasi jeung Ultimate Multisite pikeun ngatur wates nyieun kursus jeung corong dumasar kana pakét situs
- **Tampilan Admin Husus**: Daptar instruktur jeung antarbeungeut pangaturan anu sadar multisitus
- **Pakakas Admin**: Antarbeungeut pangaturan database pikeun administrator jaringan jeung situs

## Sarat {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Pangaya Tutor LMS (sacara otomatis dipaksa ngaliwatan header `Requires Plugins`)
- Pamasangan WordPress Multisite
- Pangaya Ultimate Multisite (pikeun fungsionalitas watesan kursus)

## Pamasangan {#installation}

1. Unggah payil pangaya ka `/wp-content/plugins/tutor-multisite-compatibility/`
2. Pastikeun Tutor LMS geus dipasang jeung diaktipkeun
3. Aktipkeun pangaya ngaliwatan ménu 'Pangaya' dina WordPress
4. Pangaya bakal sacara otomatis ngahiji jeung Tutor LMS

## Fungsionalitas Dipindahkeun ti MU Plugin {#functionality-moved-from-mu-plugin}

Pangaya ieu ngandung fungsionalitas anu saméméhna aya dina `wp-content/mu-plugins/daves.php`:

- Watesan parobahan status kursus
- Watesan nyieun kursus
- Integrasi jeung wates CartFlows
- Pangaturan peran instruktur Tutor
- Pangaturan pendaptaran pamaké multisitus
- Tampilan ID produk kursus

## Pangaturan Database {#database-management}

### Masalahna {#the-problem}

Dina lingkungan multisitus WordPress, nyieun tabel Tutor LMS gagal pikeun tabel anu boga konstrain konci asing ka tabel pamaké. Ieu kajadian sabab:

- Tabel husus-situs maké préfix saperti `wp_2_tutor_carts`
- Konci asing ngarujuk ka `wp_2_users` (anu teu aya)
- Tabel pamaké sabenerna nyaéta `wp_users` (dibagikeun di sakuliah jaringan)

### Solusina {#the-solution}

Pangaya ieu sacara otomatis:

1. **Ngadeteksi Tabel anu Leungit**: Mariksa tabel anu gagal dijieun kusabab konstrain konci asing
2. **Nyieun Tabel jeung Rujukan anu Bener**: Maké ngaran tabel pamaké anu luyu (`wp_users` dina multisitus)
3. **Ngatur Sababaraha Skenario**:
   - Aktipasi/réaktipasi pangaya
   - Nyieun situs anyar dina jaringan
   - Mariksa tabel sacara manual ngaliwatan antarbeungeut admin

### Tabel anu Kakeunaan {#affected-tables}

- `tutor_coupon_usages` - Ngandung konci asing ka tabel pamaké
- `tutor_carts` - Ngandung konci asing ka tabel pamaké
- `tutor_notification_preferences` - Tabel TutorPro jeung konci asing ka tabel pamaké

### Antarbeungeut Admin {#admin-interface}

Aksés pakakas pangaturan database di:

- **Admin Jaringan**: Setélan → Tutor Multisite DB
- **Admin Situs**: Pakakas → Tutor Multisite DB

Antarbeungeut ngamungkinkeun anjeun pikeun:
- Ningali tabel anu leungit di sakuliah sadaya situs
- Micu nyieun tabel sacara manual
- Mariksa konfigurasi database
- Ningali inpormasi status anu jéntré
