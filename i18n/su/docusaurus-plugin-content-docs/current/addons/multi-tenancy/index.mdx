# Multi-Tenancy

Sebarkeun subsitus ka sababaraha server jeung basis data pikeun kinerja jeung kaamanan

## Sorotan rilis Vérsi 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 nambahkeun pondasi tenant daulat pikeun alur kerja isolasi produksi:

- Autologin tenant tanpa status kalayan kunjungan SSO pondok-umur anu dijaga tina replay.
- Isolasi basis data jeung sistem berkas per-subsitus pikeun tenant daulat.
- Kamampuhan multi-tenancy Hostinger pikeun provisioning tenant anu sadar host.
- Gerbang vérifikasi migrasi daulat pikeun pamariksaan data warisan jeung pamrosésan push async.
- Patalian host basis data mesin-sarua anu bisa dikonfigurasi pikeun instalasi Bedrock, FrankenPHP, jeung localhost.
- Pangaronjat bootstrap tenant, panulis basis data, bootstrap skéma, jeung provisioning pamaké.
- Teardown daulat anu leuwih aman anu ngabersihan kredensial tenant.

Tingali pituduh pamaké pikeun [alur kerja migrasi daulat](/user-guide/getting-started/sovereign-migration-workflow), [isolasi multi-tenancy](/user-guide/configuration/multi-tenancy-isolation), jeung [operasi tenant daulat](/user-guide/administration/sovereign-tenant-operations).

## Sarat {#requirements}

- WordPress 5.3 atawa leuwih luhur
- PHP 7.4 atawa leuwih luhur

2. Aktipkeun plugin ngaliwatan ménu 'Plugins' dina WordPress
3. Konpigurasikeun setélan addon dina admin jaringan

## Konfigurasi {#configuration}

Addon ieu ngarojong rupa-rupa pilihan konfigurasi anu bisa diaksés ngaliwatan antarmuka admin jaringan.

### Konfigurasi tenant daulat {#sovereign-tenant-configuration}

Pikeun tenant daulat, konpigurasikeun basis data tenant, root sistem berkas, patalian host basis data, jeung kamampuhan host saméméh ngarutekan lalu lintas ka tenant. Jalankeun vérifikasi migrasi sanggeus unggal parobahan anu mangaruhan kredensial basis data, patalian host, routing, atawa provisioning pamaké.

Paké `localhost` ngan lamun runtime tenant jeung grant basis data ngarepkeun aksés socket lokal atawa mesin-sarua. Paké `127.0.0.1` atawa hostname jasa lamun éta téh host grant anu dikonpigurasikeun ku panyadia anjeun.

### Kamampuhan Hostinger {#hostinger-capability}

Kamampuhan multi-tenancy Hostinger jalan jeung token integrasi Hostinger babarengan. Pastikeun domain aya dina hPanel, DNS nunjuk ka Account Hostinger, jeung setélan basis data tenant cocog jeung hPanel saméméh vérifikasi.
