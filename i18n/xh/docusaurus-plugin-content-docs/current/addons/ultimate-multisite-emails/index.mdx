# Ii-imeyile

I-addon ye-Ultimate Multisite: Emails ivumela abanini benethiwekhi ukuba babonelele kwaye balawule ii-akhawunti ze-imeyile zabathengi kunye nee-site kunye nobulungu. Inguqulelo 1.1.0 yongeza ulawulo lwee-akhawunti ze-imeyile olujoliswe kubathengi, ukungena kube kanye kwi-webmail, inkxaso yee-forwarder ze-CyberPanel, kunye nokuphuculwa kokubonelela ngexesha le-checkout.

## Iimpawu Ezibalulekileyo {#key-features}

* **I-widget ye-akhawunti ye-imeyile yomthengi**: Abathengi banokudala kwaye balawule iibhokisi ze-imeyile kwiphepha labo le-account emva kokuba benobulungu be-imeyile obusebenzayo.
* **I-Webmail SSO kunye namakhonkco omlingo**: Abathengi banokuvula i-webmail ngqo kwiphepha le-account ngokuqinisekiswa ngonqakrazo olunye xa umboneleli ekuxhasa oko.
* **Ukutshintsha amagama okugqitha**: Abathengi nabalawuli banokuhlaziya amagama okugqitha eebhokisi ze-imeyile kwi-widget yolawulo lwe-imeyile.
* **Ukubonelela ngamalungelo kwi-checkout**: I-checkout idala ubulungu be-imeyile, ibudibanise ne-domain ye-site yomthengi, kwaye yenza izixhobo zebhokisi ye-imeyile zephepha le-account zifumaneke.
* **Ii-forwarder ze-CyberPanel**: Iinethiwekhi ezixhaswa yi-CyberPanel zinokudala kwaye zilawule ii-forwarder ze-imeyile njengenxalenye yenkqubo yenkonzo ye-imeyile.
* **Iisetingi eziqonda umboneleli**: Iphepha leesetingi lihluza ababoneleli libashiye kwiintlanganiso ezisebenzayo nezilungiselelweyo ukuze abalawuli babone kuphela iinketho ezinokusetyenziswa.

## Iimfuno {#requirements}

* Ultimate Multisite 2.0.0 okanye engaphezulu
* Ufakelo lwe-WordPress Multisite
* Ubuncinane intlanganiso yomnikezeli we-imeyile omnye elungiselelweyo
* Iinkcukacha zokuqinisekisa ze-API zomboneleli ezinemvume yokudala iibhokisi ze-imeyile, ukutshintsha amagama okugqitha, nokukhupha amakhonkco okufikelela kwi-webmail apho kuxhaswayo

## Ufakelo {#installation}

1. Layisha iifayile ze-plugin ku-`/wp-content/plugins/ultimate-multisite-emails`.
2. Yenza i-addon isebenze kwinethiwekhi ukusuka ku-**Plugins** kulawulo lwenethiwekhi ye-WordPress.
3. Yiya ku-**Ultimate Multisite** > **Settings** uze uvule iisetingi ze-imeyile ze-addon.
4. Vula iintlanganiso zababoneleli be-imeyile ofuna abathengi bazisebenzise.
5. Gcina iisetingi, emva koko uvavanye ukudalwa kwe-akhawunti ngomthengi ongengowemveliso okanye nge-site yokuvavanya.

## Indlela Esebenza Ngayo Inkqubo Yokuthengisa I-imeyile {#how-the-email-seller-flow-works}

Ukuthengisa i-imeyile kuneemaleko ezimbini ezahlukileyo:

1. **Ilungelo** — imveliso/ubulungu be-imeyile be-Ultimate Multisite obuthi umthengi uvumelekile ukusebenzisa i-imeyile kwi-domain ethile ye-site, umboneleli, isabelo, kunye nomda wee-akhawunti.
2. **Ukudala ibhokisi ye-imeyile** — ibhokisi ye-imeyile yokwenene yomboneleli, efana no-`info@example.com`, edalwa xa umthengi okanye umlawuli wenethiwekhi ekhetha inxalenye yasekuhlaleni kunye negama lokugqitha.

Ngexesha le-checkout, i-addon idala okanye ivuselela **ubulungu be-imeyile** kwaye irekhoda i-domain ye-site yomthengi kunye nomboneleli kobo bulungu. Ayicingeli igama lebhokisi ye-imeyile okanye igama lokugqitha. Emva kokuba ukuthenga kugqityiwe kwaye imeko ye-site/domain ikhona, umthengi unokuvula iphepha le-account aze adale amagama eebhokisi ze-imeyile awafunayo.

Oku kuthetha ukuba inkqubo eqhelekileyo yomthengi yile:

1. Umthengi uthenga isicwangciso se-site aze ongeze imveliso ye-imeyile ngexesha le-checkout.
2. Ultimate Multisite idala ubulungu be-site kunye nobulungu besibini be-imeyile.
3. I-addon ye-Emails idibanisa ubulungu be-imeyile ne-domain ye-site yomthengi kunye nomboneleli.
4. Umthengi uvula **Account** kwi-dashboard yakhe.
5. Iphaneli ethi **Ii-akhawunti ze-imeyile** iyavela kobo bulungu be-imeyile obusebenzayo.
6. Umthengi ucofa **Yenza i-Account**, ufaka inxalenye yasekuhlaleni efana no-`info`, useta igama lokugqitha, kwaye i-addon idala `info@their-domain.com` kumboneleli olungiselelweyo.

## Lungiselela Imveliso Ye-imeyile {#configure-an-email-product}

Dala imveliso ye-imeyile enye kwiphakheji nganye ye-imeyile ofuna ukuyithengisa okanye ukuyinika simahla. Ukuba umtsha ekusetyenzisweni kwemveliso, qala ngo-[Ukudala Imveliso Yakho Yokuqala Yorhumo](../../user-guide/configuration/creating-your-first-subscription-product) uze ubuyele apha kwiisetingi ezithile ze-imeyile.

1. Yiya ku-**Network Admin > Ultimate Multisite > Products > Add Product**.
2. Seta **Uhlobo Lwemveliso** lube **I-imeyile**.
3. Lungiselela amaxabiso. Sebenzisa ixabiso eliphindaphindayo elihlawulwayo kwisicwangciso esithengiswayo, okanye usete imveliso ibe simahla/ixabiso elingu-zero ukuba kufanele ibe sisicwangciso se-imeyile esinikelwayo.
4. Vula icandelo le-**Iisetingi Ze-imeyile**.
5. Khetha **Umboneleli We-imeyile** olungiselelweyo.
6. Seta **Inani Lee-akhawunti**, **Isabelo Sokugcina (MB)**, imida yee-alias, ii-forwarder, kunye ne-autoresponder njengoko kufuneka.
7. Gcina imveliso.

Kuphela ngabakhoneleli abavuliweyo nabalungiselelwe ngokupheleleyo kwiphepha le-**Integrations** abavela kwisikhangeli somboneleli. Ukuba isikhangeli singenanto, lungiselela kwaye uvavanye umboneleli kuqala.

## Yongeza I-imeyile kwi-Checkout {#add-email-to-checkout}

Iimveliso ze-imeyile zihlala zinikelwa njengezongezo kwisicwangciso se-site. Bona [Iifomu ze-Checkout](../../user-guide/configuration/checkout-forms) ukuze ufumane inkqubo jikelele yomhleli we-checkout.

1. Dala okanye uhlele isicwangciso se-site abathengi abasithenga kuqala.
2. Yongeza imveliso ye-imeyile njenge-addon/order bump ekhoyo yaloo nkqubo ye-checkout.
3. Qinisekisa ukuba ifomu ye-checkout isabandakanya amasimi aqhelekileyo afunekayo e-site, ingakumbi umhlaba we-URL/domain ye-site.
4. Qhuba i-checkout yovavanyo ngomthengi we-staging.
5. Emva kwe-checkout, vula [iphepha le-account](../../user-guide/client-management/account-page) lomthengi uze uqinisekise ukuba iphaneli ethi **Ii-akhawunti ze-imeyile** iyavela.

I-domain ye-imeyile ithathwa kwi-domain ye-site yomthengi. Abathengi abafaki i-domain ye-imeyile kwinyathelo lemveliso ye-imeyile.

## Ulawulo Lwe-imeyile Ye-Account Yomthengi {#customer-account-email-management}

Inguqulelo 1.1.0 yongeza i-widget yolawulo lwe-imeyile kwiphepha le-account yomthengi. Yisebenzise xa abathengi kufuneka bazenzele ngokwabo ukudala nokugcina iibhokisi ze-imeyile zee-site zabo.

Kwiphepha le-account, abathengi banako:

1. Ukujonga ii-akhawunti ze-imeyile ezidibene nobulungu babo bangoku be-imeyile/domain ye-site.
2. Ukudala ibhokisi ye-imeyile entsha xa imveliso yabo ye-imeyile ivumela ii-akhawunti ezongezelelweyo.
3. Ukutshintsha igama lokugqitha lebhokisi ye-imeyile ekhoyo.
4. Ukuvula ababoneleli be-webmail abaxhaswayo nge-SSO okanye ngokungena ngekhonkco lomlingo.

I-widget isebenzisa inkqubo ye-currents yephepha le-account ye-Ultimate Multisite, ngoko izenzo zilinganiselwe kumxholo we-account ojongwa ngumthengi. Oku kuthintela abathengi ekulawuleni ii-akhawunti ze-imeyile ezingezobunye ubulungu okanye enye site.

Ukuba umthengi akayiboni iphaneli ye-imeyile, khangela ukuba banobulungu **bemveliso ye-imeyile obusebenzayo**, ukuba ubulungu budityaniswe ne-domain ye-site, kwaye ukuba imveliso inomnikezeli olungiselelweyo.

## Ukunika Umthengi Isicwangciso Se-imeyile Sasimahla {#giving-a-customer-a-free-email-plan}

Ukunika umthengi i-imeyile ngaphandle kokumbiza, nika **ilungelo le-imeyile** kuqala, emva koko udale okanye uvumele umthengi adale ibhokisi ye-imeyile.

Iinketho ezicetyiswayo:

* **Dala imveliso ye-imeyile yasimahla**: seta ixabiso lemveliso ye-imeyile libe simahla/zero, emva koko wongeze ubulungu obusebenzayo baloo mthengi ukusuka ku-**Ultimate Multisite > Memberships > Add Membership**. Khetha imveliso ye-imeyile nomthengi. Ubulungu basimahla bobobomi bonke de butshintshwe okanye burhoxiswe. Ukuba ubulungu be-imeyile benzelwe ukulandela i-site ekhoyo, qinisekisa ukuba budityaniswe kuloo mxholo we-site/domain.
* **Sebenzisa isaphulelo se-100% okanye ikhonkco le-checkout yasimahla**: vumela umthengi adlule kwi-checkout ukuze Ultimate Multisite idale ubulungu be-imeyile kunye nerekhodi lentlawulo ngokuzenzekelayo.
* **Yenza umlawuli adale ibhokisi ye-imeyile**: yiya ku-**Ultimate Multisite > Email Accounts > Add Email Account** ukuze udale ibhokisi ye-imeyile ethile kumboneleli yomthengi. Oku kuluncedo kwinkxaso, kodwa akutshintshi ubulungu bemveliso ye-imeyile obunika amandla izixhobo zomthengi zokuzisebenzela kwiphepha le-account.

Ukuba ufuna umthengi alawule ibhokisi ye-imeyile ngokwakhe kwiphepha le-account, qinisekisa ukuba unobulungu be-imeyile obusebenzayo baloo domain. Ibhokisi ye-imeyile yomboneleli edalwe ngesandla ngaphandle kobo bulungu ingakhona, kodwa umthengi usenokungayiboni iphaneli ye-imeyile yokuzisebenzela yayo.

## Ii-akhawunti Ezidalwe Ngumlawuli kunye "Akukho Nombolo Ye-odolo" {#admin-created-accounts-and-no-order-number}

Iphepha lolawulo le-**Ii-akhawunti ze-imeyile** lidala okanye lihlele ibhokisi ye-imeyile yomboneleli. Ayilonyathelo lokuqala lokuthengisa okanye lokunika isicwangciso se-imeyile.

Xa uhlela i-akhawunti ye-imeyile, umhlaba we-**Ubulungu** ukhangela ubulungu obukhoyo be-Ultimate Multisite ngenombolo yabo yesalathiso/ye-odolo. Ukuba uzama ukwabela i-akhawunti ngaphambi kokuba umthengi abe nobulungu be-imeyile, kusenokungabikho nombolo ye-odolo onokuyikhetha.

Sebenzisa olu luhlu lwemisebenzi endaweni yoko:

1. Dala okanye ufumane ubulungu be-imeyile bomthengi. Kuthengiso, i-checkout iyakwenza oku. Kwisicwangciso esinikelwayo, dala ubulungu ngesandla ukusuka ku-[Memberships](../../user-guide/administration/managing-memberships) okanye uthumele umthengi kwi-checkout yasimahla.
2. Qinisekisa ukuba ubulungu buyasebenza kwaye busebenzisa imveliso ye-imeyile.
3. Qinisekisa ukuba ubulungu be-imeyile budityaniswe ne-domain/umboneleli we-site.
4. Dala ibhokisi ye-imeyile kwiphepha le-account yomthengi okanye ukusuka ku-**Ultimate Multisite > Email Accounts**.
5. Ukuba kuyafuneka, hlela irekhodi lebhokisi ye-imeyile uze wabele umthengi, ubulungu, kunye ne-site.

Ukuba ufuna kuphela ukudala ibhokisi ye-imeyile yomboneleli yomthengi kwaye awudingi ulawulo lokuzisebenzela kwiphepha le-account, i-akhawunti edalwe ngumlawuli inokwanela. Ukuba ufuna umthengi ayibone kwaye ayilawule kwi-Ultimate Multisite, dala nobulungu be-imeyile.

## Webmail SSO kunye Nokungena Ngekhonkco Lomlingo {#webmail-sso-and-magic-link-login}

I-Webmail SSO inika abathengi isenzo esithe ngqo esithi **Vula i-webmail** kwiphepha le-account. Xa ixhaswa ngumboneleli osebenzayo, i-addon icela i-URL yokufikelela yexesha elifutshane ize ithumele umthengi kwi-webmail ngaphandle kokumcela ukuba akope iinkcukacha zokuqinisekisa zebhokisi ye-imeyile.

Ukuziphatha komboneleli kunokwahluka:

* Ababoneleli abanee-API ze-SSO zendalo banokudala iseshoni ye-webmail eqinisekisiweyo ngqo.
* Ababoneleli abanenkxaso yekhonkco lomlingo banokuvelisa ikhonkco lokungena elinomda wexesha.
* Ababoneleli abangenankxaso ye-SSO okanye yekhonkco lomlingo kufuneka basabonisa iinkcukacha zebhokisi ye-imeyile kunye nezenzo zolawulo lwegama lokugqitha, kodwa kungabikho qhosha le-webmail lonqakrazo olunye.

Vula ukufikelela kwi-webmail kuphela kubaboneleli apho ulungiselele iinkcukacha zokuqinisekisa ze-API ezifunekayo. Ukuba isenzo asikho kumboneleli, kuqala qinisekisa ukuba umboneleli uyasebenza, ulungiselelwe, kwaye uxhasa amakhonkco okungena kwi-webmail.

## Ukubonelela nge-Checkout {#checkout-provisioning}

I-addon ibonelela **umxholo wobulungu** be-imeyile ngexesha le-checkout. Irekhoda umboneleli, umda wee-akhawunti, isabelo, kunye ne-domain ye-site ukuze abathengi bakwazi ukudala amagama abo eebhokisi ze-imeyile kwiphepha le-account emva kokuthenga.

Ukukhutshwa okucetyiswayo:

1. Lungiselela umboneleli kunye nemida yemveliso ngaphambi kokunikezela ngemveliso ye-imeyile.
2. Vavanya ukuthenga ngemveliso ye-staging uze uqinisekise ukuba ubulungu be-imeyile buyadalwa.
3. Phonononga iphepha le-account yomthengi ukuze uqinisekise ukuba i-widget ethi **Ii-akhawunti ze-imeyile** iyavela kuloo domain.
4. Dala ibhokisi ye-imeyile yovavanyo kwi-widget.
5. Qinisekisa ukutshintsha kwegama lokugqitha kunye nezenzo ze-webmail ngaphambi kokuvula le nkqubo kwiimveliso zemveliso.

Ukuba ukubonelela kuyasilela, i-account yomthengi kunye nobulungu zisenokuhlala zikhona ngelixa ubulungu be-imeyile okanye ibhokisi ye-imeyile yomboneleli ifuna ukulandelwa ngumlawuli. Khangela iilogi zomsebenzi ze-Ultimate Multisite kunye neelogi zomboneleli ngaphambi kokuzama kwakhona.

## Ii-forwarder Ze-imeyile Ze-CyberPanel {#cyberpanel-email-forwarders}

Kwiinethiwekhi ezixhaswa yi-CyberPanel, inguqulelo 1.1.0 yongeza inkxaso yee-forwarder ze-imeyile. Sebenzisa ii-forwarder xa abathengi befuna iidilesi ezifana no-`info@customer-domain.test` ukuthumela imeyile kwenye inbox ngaphandle kokudala ibhokisi ye-imeyile epheleleyo.

Ngaphambi kokunikezela ngee-forwarder, qinisekisa ukuba intlanganiso ye-CyberPanel ilungiselelwe kwaye iyafikeleleka. Bona isikhokelo se-[Intlanganiso ye-CyberPanel](../../user-guide/administration/cyberpanel-integration) ukuze ufumane i-host, igama lomsebenzisi, igama lokugqitha, kunye neekhonstanti zephakheji ezifunekayo.

## Imibuzo Ebuzwa Rhoqo {#frequently-asked-questions}

### Ngaba abathengi banokudala ii-account zabo ze-imeyile? {#can-customers-create-their-own-email-accounts}

Ewe. Xa umthengi enobulungu be-imeyile obusebenzayo bedomeyini kwaye imveliso ivumela ii-account ezongezelelweyo, abathengi banokudala baze balawule iibhokisi zabo ze-imeyile kwiphepha le-account.

### Ngaba wonke umboneleli uyayixhasa i-webmail SSO? {#does-every-provider-support-webmail-sso}

Hayi. Inkxaso ye-SSO kunye ne-magic-link ixhomekeke kumboneleli we-imeyile osebenzayo. Ababoneleli abangenayo i-access-link API basenokuxhasa ukudalwa kwebhokisi ye-imeyile kunye notshintsho lwegama lokugqithisa.

### Ngaba abalawuli banokudalela abathengi ii-account ze-imeyile? {#can-administrators-create-email-accounts-for-customers}

Ewe. Abalawuli banokudala ii-account ze-imeyile kujongano lomlawuli, kuquka ukuseta igama lokugqithisa lokuqala xa kudalwa i-account. Kulawulo lokuzisebenzela lomthengi, qinisekisa kananjalo ukuba umthengi unobulungu be-imeyile obusebenzayo bedomeyini.

### Ndinganika i-account ye-imeyile simahla? {#can-i-give-an-email-account-away-for-free}

Ewe. Dala imveliso ye-Email yasimahla/enexabiso elinguziro uze uyongeze njengobulungu obusebenzayo bomthengi, okanye vumela umthengi agqibe i-checkout ngesaphulelo se-100%. Emva kokuba ubulungu be-imeyile bukhona, umthengi okanye umlawuli angadala ibhokisi ye-imeyile yokwenene.

### Kutheni ukwabela i-account ye-imeyile kucela inombolo ye-odolo? {#why-does-assigning-an-email-account-ask-for-an-order-number}

Indawo ye-Membership ikhangelela ubulungu obukhoyo be-Ultimate Multisite ngenombolo yayo yesalathiso/ye-odolo. Ukuba umthengi akathenganga imveliso ye-imeyile kwaye awukadalanga ngesandla ubulungu be-imeyile, akukho bulungu/odolo ye-imeyile onokuyabela. Dala ubulungu be-imeyile kuqala, emva koko udale okanye wabele ibhokisi ye-imeyile.

### Ngaba ii-forwarder ziyafana neebhokisi ze-imeyile? {#are-forwarders-the-same-as-mailboxes}

Hayi. Ibhokisi ye-imeyile igcina imeyile kwaye ineziqinisekiso zokungena. I-forwarder ithumela imeyile kwenye idilesi kwaye ayiboneleli nge-inbox eyahlukileyo.
