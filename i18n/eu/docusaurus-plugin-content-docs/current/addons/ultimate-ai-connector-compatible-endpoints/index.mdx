# Ultimate AI Connector amaiera-puntu bateragarrietarako {#ultimate-ai-connector-for-compatible-endpoints}

Ultimate AI Connector amaiera-puntu bateragarrietarako OpenAI-rekin bateragarriak diren AI hornitzaileen APIak zure WordPress multisite sarearekin integratzen ditu. Konfiguratu hainbat AI hornitzaile — bakoitza bere endpoint URLarekin, API gakoarekin eta eredu lehenetsiarekin — eta bideratu eskaerak ordezko bideratze automatikoarekin, hornitzaile bat erabilgarri ez dagoenean.

## Ezaugarri nagusiak {#key-features}

- **Hornitzaile anitzeko euskarria**: Gehitu hainbat AI hornitzaile-txartel, bakoitza bere endpoint URLarekin, API gakoarekin eta eredu lehenetsiarekin
- **Arrastatu berrantolatzeko**: Antolatu hornitzaileak lehentasunaren arabera txartel bakoitzeko heldulekuarekin; eskaerek hornitzaileak ordenan erabiltzen dituzte
- **Ordezko bideratze automatikoa**: Hornitzaile nagusia erabilgarri ez badago, eskaerak automatikoki hurrengo konfiguratutako hornitzailera pasatzen dira
- **Hornitzaile bakoitzeko konfigurazioa**: Hornitzaile bakoitza modu independentean konfiguratzen da, bere kredentzialekin eta eredu-hautaketarekin
- **Txarteletan oinarritutako ezarpenen UIa**: WordPress 6.9+ Card osagai egonkorretan eraikia (CardBody, CardHeader, CardDivider)
- **Gehitzean automatikoki zabaltzea**: Hornitzaile-txartel berriak automatikoki zabaltzen dira, xehetasunak berehala bete ditzazun
- **Script cachea baliogabetzea**: Hornitzaileen scriptetan cachea baliogabetzen da plugin eguneratzean, azken bertsioa kargatzen dela ziurtatzeko

## Eskakizunak {#requirements}

- WordPress 5.3 edo berriagoa
- PHP 7.4 edo berriagoa
- Ultimate Multisite plugin (aktibo)
- OpenAI-rekin bateragarria den API amaiera-puntu bat (norberak ostatatua edo hirugarren batekoa)

## Instalazioa {#installation}

1. Igo addon fitxategiak zure `/wp-content/plugins/` direktoriora
2. Aktibatu plugin WordPress-eko 'Pluginak' menuaren bidez
3. Joan **Ultimate Multisite → AI Connector** atalera addona konfiguratzeko

## Konfigurazioa {#configuration}

### Hornitzaile anitzeko konfigurazioa {#multi-provider-setup}

Konfigurazio-pantailak txarteletan oinarritutako diseinu bat erabiltzen du, non txartel bakoitzak AI hornitzaile bat adierazten duen. Behar adina hornitzaile gehi ditzakezu.

#### Hornitzaile bat gehitzea {#adding-a-provider}

1. Joan sareko administrazioan **Ultimate Multisite → AI Connector** atalera
2. Egin klik **Gehitu hornitzailea** aukeran
3. Hornitzaile-txartel berri bat agertzen da eta automatikoki zabaltzen da
4. Bete hornitzailearen xehetasunak:
   - **Hornitzailearen izena** — hornitzaile hau identifikatzeko etiketa bat (adib. "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URLa** — OpenAI-rekin bateragarria den APIaren oinarrizko URLa (adib. `https://api.openai.com/v1`)
   - **API gakoa** — hornitzaile honetarako zure API gakoa
   - **Eredu lehenetsia** — hornitzaile honetarako lehenespenez erabiliko den eredu-identifikatzailea (adib. `gpt-4o`, `mistral-small`)
5. Egin klik **Gorde ezarpenak** aukeran

#### Hornitzaile bat kentzea {#removing-a-provider}

Egin klik edozein hornitzaile-txarteletako **Kendu** botoian hura ezabatzeko. Gorde ezarpenak aldaketa aplikatzeko.

#### Hornitzaileak berrantolatzea {#reordering-providers}

Arrastatu edozein hornitzaile-txartelen ezkerraldeko helduleku-ikonoa (⣿) hura berrantolatzeko. Ordenak ordezko lehentasuna zehazten du: eskaerak lehen hornitzailera bidaltzen dira eta, eskaerak huts egiten badu, hurrengo hornitzaileetara pasatzen dira.

### Ordezko bideratzea {#fallback-routing}

Hornitzaile nagusirako eskaerak huts egiten duenean — denbora-muga, tasa-muga edo zerbitzuaren eten bat dela eta — konektoreak automatikoki berriro saiatzen da zerrendako hurrengo hornitzailearekin. Horrek zerrendan behera jarraitzen du hornitzaile batek arrakasta izan arte edo hornitzaile guztiak probatu arte.

| Eszenarioa | Portaera |
|----------|-----------|
| Hornitzaile nagusiak erantzuten du | Eskaera normaltasunez zerbitzatzen da |
| Hornitzaile nagusiak huts egiten du | Eskaera hurrengo hornitzailearekin berriro saiatzen da |
| Hornitzaile guztiek huts egiten dute | Errorea deitzaileari itzultzen zaio |

Ordezko bideratzea gardena da deitzen duen plugin edo themearentzat — erantzun arrakastatsu bat edo azken errore bat jasotzen dute, zenbat hornitzaile probatu diren kontuan hartu gabe.

### Hornitzaileen konfigurazio-erreferentzia {#provider-configuration-reference}

| Eremua | Beharrezkoa | Deskribapena |
|-------|----------|-------------|
| Hornitzailearen izena | Bai | Hornitzaile honetarako bistaratze-etiketa |
| Endpoint URLa | Bai | OpenAI-rekin bateragarria den APIaren oinarrizko URLa |
| API gakoa | Bai | Hornitzaile honetarako autentifikazio-gakoa |
| Eredu lehenetsia | Bai | Eskaeran eredurik zehazten ez denean erabiltzen den eredu-identifikatzailea |

## Arazoak konpontzea {#troubleshooting}

### Ez da hornitzailerik zerrendatzen {#no-providers-are-listed}

Gorde ondoren hornitzaileen zerrenda hutsik badago, egiaztatu hau:

- Hornitzaileak gehitu ondoren **Gorde ezarpenak** aukeran klik egin duzula
- Zure nabigatzaileak ez duela ezarpenen orria gordetzea blokeatu (begiratu nabigatzailearen kontsolan JavaScript erroreak dauden)
- Plugin aktibo dagoela eta sareko administrazio-pantailan zaudela

### Eskaerek huts egiten dute hornitzaile guztientzat {#requests-are-failing-for-all-providers}

- Egiaztatu hornitzaile bakoitzaren endpoint URLa zure zerbitzaritik irisgarria dela (ez soilik zure nabigatzailetik)
- Egiaztatu API gako bakoitza baliozkoa dela eta ez dela iraungi
- Berrikusi hornitzailearen tasa-mugaren eta kuotaren egoera haren dashboardean
- Gaitu WordPress arazketa-erregistroa (`WP_DEBUG_LOG`) konektorearen errore-mezu xeheak jasotzeko

### Hornitzaile bat ez da ordezko bideratzerako erabiltzen {#a-provider-is-not-being-used-for-fallback}

Arrastatu berrantolatzeko funtzioak ordezkoen ordena aldatzen du. Egiaztatu ordezko eskaerak jasotzea espero duzun hornitzailea zerrendan behar bezala kokatuta dagoela eta haren endpoint URLa eta API gakoa zuzenak direla.

### Hornitzaile-txartel berria ez da automatikoki zabaltzen {#new-provider-card-does-not-expand-automatically}

Automatikoki zabaltzeko, JavaScript gaituta egon behar da zure nabigatzailean. Txartela klik egitean zabaltzen ez bada, begiratu nabigatzailearen kontsolan errorerik dagoen. Txartelaren edukia beti dago eskuragarri txartelaren goiburuan klik eginez, automatikoki zabaltzea aktibatzen ez bada ere.
