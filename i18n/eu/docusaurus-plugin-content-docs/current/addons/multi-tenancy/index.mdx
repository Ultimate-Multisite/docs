# Multi-Tenancy {#multi-tenancy}

Banatu azpiguneak zerbitzarien eta datu-baseen artean errendimendurako eta segurtasunerako

## 1.2.0 bertsioaren argitalpeneko nabarmenenak {#version-120-release-highlights}

Multi-Tenancy 1.2.0-k ekoizpeneko isolamendu-fluxuetarako maizter subiranoaren oinarria gehitzen du:

- Egoerarik gabeko maizterren autologina, iraupen laburreko eta berrerabilpenetik babestutako SSO bisitekin.
- Azpigune bakoitzeko datu-basearen eta fitxategi-sistemaren isolamendua maizter subiranoentzat.
- Hostinger multi-tenancy gaitasuna, host-a kontuan hartzen duen maizter-hornikuntzarako.
- Migrazio subiranoaren egiaztapen-ateak, datu zaharren egiaztapenetarako eta push prozesamendu asinkronorako.
- Makina bereko datu-basearen host lotura konfiguragarria Bedrock, FrankenPHP eta localhost instalazioetarako.
- Maizterren abioaren, datu-base idazlearen, eskema-abioaren eta erabiltzaileen hornikuntzaren hobekuntzak.
- Maizterren kredentzialak garbitzen dituen desegite subirano seguruagoa.

Ikusi erabiltzaile-gidak: [migrazio subiranoaren lan-fluxua](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy isolamendua](/user-guide/configuration/multi-tenancy-isolation), eta [maizter subiranoen eragiketak](/user-guide/administration/sovereign-tenant-operations).

## Eskakizunak {#requirements}

- WordPress 5.3 edo berriagoa
- PHP 7.4 edo berriagoa
- Multisite Ultimate plugina (aktibo)

## Instalazioa {#installation}

1. Igo gehigarriaren fitxategiak zure `/wp-content/plugins/` direktoriora
2. Aktibatu plugina WordPress-eko 'Pluginak' menuaren bidez
3. Konfiguratu gehigarriaren ezarpenak sareko administrazioan

## Konfigurazioa {#configuration}

Gehigarriak sareko administrazio-interfazearen bidez eskuragarri dauden hainbat konfigurazio-aukera onartzen ditu.

### Maizter subiranoaren konfigurazioa {#sovereign-tenant-configuration}

Maizter subiranoentzat, konfiguratu maizterraren datu-basea, fitxategi-sistemaren erroa, datu-basearen host lotura eta host gaitasuna trafikoa maizterrera bideratu aurretik. Exekutatu migrazioaren egiaztapena datu-basearen kredentzialetan, host loturan, bideratzean edo erabiltzaileen hornikuntzan eragina duen aldaketa bakoitzaren ondoren.

Erabili `localhost` soilik maizterraren exekuzio-inguruneak eta datu-basearen baimenek socket lokala edo makina bereko sarbidea espero dutenean. Erabili `127.0.0.1` edo zerbitzuaren host-izena hori bada zure hornitzaileak konfiguratutako baimen-host-a.

### Hostinger gaitasuna {#hostinger-capability}

Hostinger multi-tenancy gaitasunak partekatutako Hostinger integrazio-tokenarekin funtzionatzen du. Berretsi domeinua hPanel-en dagoela, DNS-k Hostinger account-era apuntatzen duela, eta maizterraren datu-basearen ezarpenak hPanel-ekin bat datozela egiaztapenaren aurretik.

## Hook-ak eta iragazkiak {#hooks-and-filters}

### Ekintzak {#actions}

- `ultimate-multisite-addon-template_init` - Gehigarria hasieratu ondoren abiarazten da
- `ultimate-multisite-addon-template_loaded` - Gehigarriaren fitxategi guztiak kargatu ondoren abiarazten da

### Iragazkiak {#filters}

- `ultimate-multisite-addon-template_settings` - Aldatu gehigarriaren ezarpenak
- `ultimate-multisite-addon-template_enabled` - Gainidatzi gehigarria gaitzeko/desgaitzeko egoera
