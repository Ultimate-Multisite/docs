# Ultimate Multisite eskakizunak

Ultimate Multisite-a eritzeko iridian duan aurretik galdetzeko garrantzitsu da, pluginaren eritzeko beharrezko irudiak zerren dira? Irudi ez da, Ultimate Multisite eritzeko oso garrantzitsuak ezren jakin dira: **domain name** bat, ona **hosting provider** bat eta WordPress-en **Multisite** instalazioa. Horrek dugu behar du!

## Domain Name {#domain-name}

Lehenik, domain name-a eztabaiduon.

Domain name bat ereintzen duenean, zerren dela da irudiak jakinatu: Ultimate Multisitearekin zer egin aurretik duzun eta zer sortu egingo duzun? Agenzia bat ezartzea? Shop-like webstei saldzea? Ondorekin WordPress web hosting? Ultimate Multisitearekin guztiz posible da.

Pluginaren parte batean abonazio serbisuak, mugikorrak, checkout forms, eskaintzak eta kliak komunikazioa ematen du. WordPress Multisite instalatzeko, konfiguratzeko eta mantentzeko beharrezko handi parte bat Ultimate Multisitearekin eragiten da.

Zer nire niche bat ereintzen ari zaitut, websteia serbisua gisa salduz! Zerren domain name bat ereintzen duzu.

## Hosting Provider {#hosting-provider}

Domain name bat eztabaiduon eta niche-n irudiak ematen duen biznearen edo negozio planaren espezifikazioa da, Ultimate Multisitearekin networka hostatzeko ona **web hosting company** bat eritzu behar duzu. Merkian oneko opsioak dituz. Zerren web hosting provider-ek Ultimate Multisitearekin ondore egiten dute, eta hori erabakiak ematen da zer sortzeko planatu duzun bero dagoen.

Ultimate Multisite-ak gure jakrak hosting provider ere lan funtzionatzen da. Arretxoa da, batzuk hosting kompañiak, gabe _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ eta _Runcloud_, domain mapatu behar duen jarduera handiagoa dute. Beste hosting-etan, custom domainak manuel mapatzea beharrezkoa izan daiteke. AutoSSL (Auto SSL) suportatzen duen hosting provider bat hartzea ere ondo ideia da. _Closte_, _Runcloud_ eta _cPanel_ ez duena suportatzen.

Server espezifikazioei dagoen formulazio bat ez da, karena funtzioak eta zerbitzuak zehatzen dituzten eta network-aren diseinazioa dependeratuko du. Shared hosting (hosting-a saltu-saltu) ondo lehenbait izan daiteke, baina network-a handitu gurean hasten den, dedeku VPS dedicated bat behar izango du.

## WordPress Multisite {#wordpress-multisite}

Orain, WordPress Multisite instalazioari gure ezagutzea gure.

WordPress Multisite-ak ziurtatutako installazio batean web-plataforma bat network-a hartzeko aukera ematen du. Hau da funtazioa bereziko da, baina default-ean aktibo ez dago. WordPress-aren eta web hosting control panel-aren beldurarekin beharrezkoa da hau aktiboa ematu.

WordPress-ak Multisite aktiboa duen **da Ultimate Multisite-aren basea**. Ultimate Multisite-aren garrantzia handia da, nahiz eta gure nagusi erabilera erabiltzaileei web-plataformak administrazio eta serbisio gisa saldzea dela, hau WordPress-an Multisite aktiboa duen bitartean egin behar da.

WordPress installazioan Multisite funtazioa aktiboa ematu behar duen jarduera zehatz dena gure **artikulu txiki bat** arau egiten du.

WordPress-an Multisite aktiboa ematen ondoren, Ultimate Multisite instalatzea protokoioa da. Ultimate Multisite ondo instalatzeko, hauek dituzte:

  * WordPress: v5.3+ (Recomendatua: Azken estable versiona)

  * PHP: 7.4.x (8.0-aren suporta iristen da)

  * MySQL: v5+ (Recomendatua: 5.6, hosting provider-ek 8.0 ez dutenean)

Hau da Ultimate Multisite-eko gizarteko espezifikazioak.

Mundia ez du PHP 8.0arekin lan egin dezake, baina PHP 7.4.xarekin jarraitzea ematen dugu.

Bestelako, gure aurreko ematean **main domains** (domain-ek) instalatu beharko dituzte, subdomains edo subdirectories ez du. Subdomains/subdirectories-ekin lan egin dezake, baina erantzunak edo errorrak izan dezak.

Ultimate Multisite instalatzeko detallatuta gidaiaren artean **aipatzi hiri**.
