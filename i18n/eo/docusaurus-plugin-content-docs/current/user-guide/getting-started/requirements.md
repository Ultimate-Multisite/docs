# Requisitoj la Ultimate Multisite

La unua cosa ki vi povas diri memu antaŭ ol iachti Ultimate Multisite estas kio bezonas por fari labori kun nia plugino. Behatko, ĝuste ekzistas nur malfortaj rekvizito por fari labori kun Ultimate Multisite: **domeno**, bona **hostaĝo** (hosting provider) kaj WordPress **Multisite** instalita. Tio estas ĉio necese!

## Domeno {#domain-name}

Unue, partemble pri la domeno.

Kiam vi elektas domenon, memoru, kion estas via specifika nicho kaj kion vi faros kun Ultimate Multisite: Volu vi krei agencon? Vendi ŝop-similajn sitiojn? Aŭ eble WordPress web-hostadon? ĉio estas posible kun Ultimate Multisite.

Nia plugino provizas subskriban servojn, pagamentajn kolektajn, checkout formojn, diskonto vortojn kaj klientkomunikojn. La granda kvanto de laboro necesa por instali, konfigurui kaj mainteni WordPress Multisite estas faciligita de Ultimate Multisite.

Kaj kualia nicho vi elektas povas esti administrita per nia plugino, ĉar vi vendos sitiojn kiel servon! Ĉu certigu vi elektas domenon, kiu korespondas kun via bizneplan.

## Hostaĝo (Hosting Provider) {#hosting-provider}

Nove ke vi havas akta domeno, kiu reprezentas vian nichon kaj bizneson, vi bezonas elekti bonan web-hostaĝcompany por hosti vian reton kun Ultimate Multisite. Ekzistas pli bonej opcioj sur la merkato. Diversaj hostaĝcompany integri bone kun Ultimate Multisite, kio povas esti decida faktoro dependante de tio, kion vi planas konstrui.

Ultimate Multisite devas kun ĉiuj hospedaj proviĝos fari işpon. La unika diferenco estas tio, ke certaj hospedaj kompanio kiel _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ kaj _Runcloud_ havas pli stretan integriĝon por domeno-mapado. Por aliaj hospedaj servoj vi povus bezoni fari la mapadon de la personalaj domenoj manua. Ĝi ankaŭ estas bona idejo akvipli hospedajn servojn, kiuj suportas AutoSSL. _Closte_, _Runcloud_ kaj _cPanel_ tiun funkcion suportas.

Kiam temas pri server specifikoj, ne ekzistas specifika formulo, ĉar ĝi dependos de la funkcio kaj funkcio, kiun vi ofertas kaj de la grandeco de via rano. Partita hospedado povas esti bona komenco, sed kiam via rano komencas kreski, vi probable necesos dedicitan VPS.

## WordPress Multisite {#wordpress-multisite}

Nove, partemble pri instalado de WordPress Multisite.

WordPress Multisite permesas al vi havi reton de sitioj sur unu instalado. Ĝi estas konstruita funkcio, sed ĝi ne estas aktiva por default. Vi bezonas aktivi ĉi tiun funkcion intern en via WordPress instalado kaj web-hospado kontrolpanelo.

Havi WordPress kun Multisite aktivaj **estas baza por Ultimate Multisite**. Ji ĉefe uzu de Ultimate Multisite estas helpi uzantojn administra kaj vendi sitiojn kiel servon, tial tio devas fariĝi sur WordPress kun Multisite aktivaj.

Nove ni havas **profundajn artikolon**, kiu instruas paŝe paŝe kion bezonas fari por aktivi la Multisite funkcion sur via WordPress instalado.

Post kiam vi havis Multisite aktivon en WordPress, estas tempo instali Ultimate Multisite. Por ĝuste instali Ultimate Multisite, vi devas havi:

  * WordPress: v5.3+ (Rekommendas: La plej stabila versio)

  * PHP: 7.4.x (subteno por > 8.0 venos proksime)

  * MySQL: v5+ (Rekommendas: 5.6, en kazo ke vi ne havas 8.0 disponiblajn sur via hospedaj providanto)

Ĉi tiuj estas la unua multaj programaj rakontoj por Ultimate Multisite.

Malkolektu, ke Ultimate Multisite povas fari funkcionigi kun PHP 8.0, sed ni rekomendas diri fari ĝin kun PHP 7.4.x.

Aŭ ankaŭ ni rekomendas instali ĝin sur **main domajnoj**, ne sur subdomajno aŭ subdirektorio. Ĝi povus fari funkcionigi sur subdomajnoj/subdirektorioj, sed ĝi povus prezenti certajn erarojn.

Por detala gvidado pri kiel instali Ultimate Multisite, **klaku ĉi tie**.
