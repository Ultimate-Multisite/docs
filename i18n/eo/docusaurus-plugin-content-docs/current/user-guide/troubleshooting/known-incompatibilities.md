# Konsciente inkompatibilitate kun alia Plugin-oj

Ansekvi instalado kaj aktivaigi Ultimate Multisite, vi povas diri problemo grundu neadresita inkompatibilitate kun aliaj plugin-oj.

Ni laboras aktive por redukti ĉi ti liston ĉe ĉiu nova riliso, sed se plugin estas listita ĉi tie, vi povus bezoni de deaktivado ĝin por havi plene funkcionan Ultimate Multisite Retejo.

Jen lista de ili kaj kion vi devas fari en tiaj okazoj:

**Quelkaj WPMU DEV plugin-oj** Kelkaj plugin-oj, kiuj pli ne estas mantenita de WPMU DEV, estas konsciente kaŭzitaj por interrompi la registriĝfluen de Ultimate Multisite al la lasta paŝo (la uzanto ne povas krei konton). Jen lista de plugin-oj kiuj kaŭzas ti problemo:

  * Signup Language: _Ultimate Multisite havas ekstensilan, kiu ofertas la saman funkcion, nomita Ultimate Multisite: Language Switcher._

  * Set Blog Description on Blog Creation: _Tio estas konstruita en Ultimate Multisite, tiel ne serĉas necesiĝi de ekstra plugino._

  * New Blog Templates: _Tio estas konstruita en Ultimate Multisite, tiel ne serĉas necesiĝi de ekstra plugino._

  * Set Password on Multisite Blog Creation: _Tio estas konstruita en Ultimate Multisite, tiel ne serĉas necesiĝi de ekstra plugino._

  * Signup TOS: _Tio estas konstruita en Ultimate Multisite, tiel ne serĉas necesiĝi de ekstra plugino._

  * Anti-Splog: Eĉ se pli malpotenca, tio ankaŭ estas _konstruita en Ultimate Multisite._

**Notado**: Ĉiuj aliaj WPMU Dev plugin-oj povas esti uzata normalaj kiel kun Ultimate Multisite. Ekzemploj inkluzivas _Smush_, _Forminator_, _Defender_, etc.
