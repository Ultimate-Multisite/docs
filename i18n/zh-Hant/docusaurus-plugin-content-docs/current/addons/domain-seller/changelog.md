# Domain Seller 更新日誌

Version 1.1.0 - 發布日期 2026-05-08
- 新增：為 ResellerClub、Enom 和 OpenSRS 註冊商實作了 DNS 記錄建立功能 (add_dns_record)
- 修復：預設 DNS 記錄解析器現在可以容忍 {DOMAIN} 和 {SITE_URL} 令牌
- 修復：將網域選擇結帳欄位的 slug 進行命名空間化，以防止與 site_url 發生衝突

Version 1.0.8 - 發布日期 2026-05-07
- 修復：ResellerClub 的網域定價現在會從正確的 API 端點獲取即時成本價格

Version 1.0.7 - 發布日期 2026-05-06
* 修復：ResellerClub 的 test_connection 會傳送所需的 tlds 參數 (#224)

Version 1.0.6 - 發布日期 2026-05-05
* 修復：ResellerClub 的網域註冊功能現在能正確運作，並改善了 API 回應處理和註冊商驅動的提供商路由
* 修復：改善了「註冊網域」管理頁面的使用者體驗 (UX)
* 移除：CyberPanel 註冊商整合

Version 1.0.5 - 發布日期 2026-04-02
* 新增：新增 GoDaddy 註冊商整合，用於網域註冊和管理
* 新增：新增 NameSilo 註冊商整合
* 新增：新增 ResellerClub 註冊商整合
* 新增：在網域購買和映射時，自動驗證 SES 發送網域
* 修復：保護 Guard plugin 的常數，防止在測試環境中被重新定義
* 修復：MySQL 標誌在 install-wp-tests.sh 中能正確地進行單詞分割

Version 1.0.4 - 發布日期 2026-03-14
* **修復：** 某些遺失的 CSS 資源
* **修復：** 與無法使用的 tlds 相關的錯誤

Version 1.0.3 - 發布日期 2026-03-09
* **修復：** 當使用帶有結帳短代碼的舊版註冊表單時，Vue 的響應屬性錯誤 (domain_option, selected_domain, domain_provider)
* **修復：** 網域選擇結帳欄位中的子網域輸入欄位錯位和文字過大
* **修復：** 當網域選擇欄位存在時，隱藏「您的 URL 將是」預覽區塊

Version 1.0.2 - 發布日期 2026-03-01
* **改進：** 從設定頁面移除了全域標記法設定 — 定價現在僅按產品計算
* **改進：** 在設定頁面新增了「管理網域產品」連結，方便快速導航
* **改進：** 改善了網域產品設定的欄位描述和工具提示（通用型 vs TLD 特定型、標記類型、入門定價）
* **改進：** 改善了設定頁面各處的描述（搜尋 TLD、續訂、DNS、通知）

* **新增：** TLD 匯入工具，用於批量定價管理

* **新增：** 管理員網域註冊模態框中的註冊人地址欄位，預填充自設定
* **新增：** 包含新增、編輯和刪除記錄支援的客戶 DNS 管理介面
* **新增：** 帶有自動網域映射的「自帶網域」結帳選項
* **新增：** 在結帳時自動從網域名稱生成 site URL
* **新增：** 設定中的預設 nameserver 和 DNS 記錄配置
* **新增：** 核心網域編輯頁面上的網域註冊詳細資訊和 DNS 管理
* **新增：** 設定嚮導會自動建立一個帶有合理預設值的預設網域產品

* **新增：** 「註冊網域」管理頁面，用於手動網域註冊
* **新增：** 透過 Ultimate Multisite update server 自動更新 plugin
* **新增：** 產品列表表中的網域產品類型篩選標籤，帶有紫色徽章樣式
* **新增：** 網域結帳表單上的註冊人聯絡欄位（姓名、地址、城市、州、郵遞區號、國家、電話）
* **新增：** 在呼叫註冊商 API 之前，對註冊人欄位進行驗證，並顯示清晰的錯誤訊息
* **新增：** 針對網域註冊事件的提供商特定日誌通道（例如 domain-seller-namecheap.log）
* **新增：** 在主要註冊/報名結帳表單上的註冊人聯絡欄位（註冊網域時顯示）
* **改進：** 將 Domain Search 結帳欄位替換為統一的網域選擇欄位，支援子網域、註冊和現有網域標籤
* **改進：** 網域產品設定透過核心小工具系統，在產品編輯頁面內嵌渲染
* **改進：** 客戶網域資訊鉤子 (hooks) 整合到核心網域映射小工具，而非獨立的 metabox
* **改進：** TLD 匯入嚮導簡化為從所有提供商一鍵同步
* **改進：** Namecheap 的網域可用性使用批次 API 呼叫，加快搜尋速度
* **改進：** Namecheap 定價 API 使用了正確的參數和回應解析
* **改進：** 將 TLD 儲存集中化到單一的網路選項中
* **改進：** 網域活動日誌記錄 DNS 變更、轉移和配置應用
* **改進：** 使用 IANA 主列表和批次驗證，為 OpenSRS 進行完整的 TLD 同步
* **改進：** 對 Namecheap 進行完整的 TLD 同步，使用分頁 API 請求
* **改進：** 將舊的提供商類別替換為 Integration Registry 模式
* **改進：** 設定面板增加了 DNS 和轉移配置
* **改進：** 電話號碼自動格式化為 +CC.NNN 註冊商格式
* **改進：** 電話欄位驗證在提交前會清除格式化字元
* **改進：** 版本要求提高到 Ultimate Multisite 2.4.12，並有更清晰的通知
* **改進：** CI 工作流程為 addon 和核心 plugin 都使用了適當的結帳機制
* **改進：** prepare_registrant_info() 從結帳儲存的 user meta 讀取，並以帳單地址作為備援
* **修復：** 在結帳時，非登入用戶執行網域搜尋 AJAX 時失敗
* **修復：** 在結帳時，非登入用戶執行網域定價 AJAX 時失敗
* **修復：** 執行 WP-CLI 指令時，Spyc 類別重新宣告導致致命錯誤
* **修復：** Namecheap 沙盒 API 超時時間太短
* **修復：** 網域搜尋 Select 按鈕在綠色背景上不可見
* **修復：** 由於缺少聯絡資訊，網域註冊失敗，出現「RegistrantFirstName is Missing」錯誤
* **修復：** 在結帳時，如果網站尚未存在，網域記錄會被創建為 blog_id=0
* **修復：** 預設 TLD 設定返回字串而非解析後的陣列
* **移除：** 獨立的網域管理管理頁面 — 現在透過核心網域頁面處理
