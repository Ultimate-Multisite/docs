# 電子郵件 {#emails}

Ultimate Multisite: Emails addon 讓網路擁有者能夠在網站與會員資格之外，佈建並管理客戶電子郵件帳號。版本 1.1.0 新增了面向客戶的電子郵件帳號管理、網路郵件單一登入、CyberPanel 轉寄器支援，以及結帳時佈建改進。

## 主要功能 {#key-features}

* **客戶電子郵件帳號小工具**：客戶在擁有有效的電子郵件會員資格後，可以從其 Account 頁面建立並管理信箱。
* **網路郵件 SSO 與魔術連結**：當提供者支援時，客戶可以透過一鍵驗證直接從 Account 頁面開啟網路郵件。
* **密碼變更**：客戶與管理員可以從電子郵件管理小工具更新信箱密碼。
* **結帳權益佈建**：結帳會建立電子郵件會員資格，將其連結到客戶的網站網域，並讓 Account 頁面的信箱工具可供使用。
* **CyberPanel 轉寄器**：以 CyberPanel 為後端的網路可以在電子郵件服務流程中建立並管理電子郵件轉寄器。
* **感知提供者的設定**：設定頁面會將提供者篩選為已啟用且已設定的整合，讓管理員只看到可用的選項。

## 需求 {#requirements}

* Ultimate Multisite 2.0.0 或更高版本
* WordPress Multisite 安裝
* 至少一個已設定的電子郵件提供者整合
* 具備建立信箱、變更密碼，以及在支援時發行網路郵件存取連結權限的提供者 API 憑證

## 安裝 {#installation}

1. 將 plugin 檔案上傳至 `/wp-content/plugins/ultimate-multisite-emails`。
2. 從 WordPress 網路管理員中的 **Plugins** 對 addon 進行網路啟用。
3. 前往 **Ultimate Multisite** > **Settings** 並開啟 addon 電子郵件設定。
4. 啟用你希望客戶使用的電子郵件提供者整合。
5. 儲存設定，然後使用非正式環境客戶或測試網站測試 Account 建立。

## 電子郵件銷售流程的運作方式 {#how-the-email-seller-flow-works}

電子郵件銷售有兩個獨立層級：

1. **權益** — Ultimate Multisite 電子郵件產品／會員資格，表示客戶被允許針對特定網站網域、提供者、配額與帳號限制使用電子郵件。
2. **信箱建立** — 實際的提供者信箱，例如 `info@example.com`，會在客戶或網路管理員選擇本機部分與密碼時建立。

在結帳期間，addon 會建立或啟用**電子郵件會員資格**，並在該會員資格上記錄客戶的網站網域與提供者。它不會猜測信箱名稱或密碼。購買完成且網站／網域情境存在後，客戶可以開啟 Account 頁面並建立他們想要的信箱名稱。

這表示一般客戶流程如下：

1. 客戶購買網站方案，並在結帳期間加入電子郵件產品。
2. Ultimate Multisite 建立網站會員資格與次要電子郵件會員資格。
3. Emails addon 將電子郵件會員資格連結到客戶的網站網域與提供者。
4. 客戶在其 Dashboard 中開啟 **Account**。
5. **電子郵件帳號**面板會針對該有效的電子郵件會員資格出現。
6. 客戶點擊 **建立 Account**，輸入本機部分（例如 `info`）、設定密碼，然後 addon 會在已設定的提供者建立 `info@their-domain.com`。

## 設定電子郵件產品 {#configure-an-email-product}

為你想銷售或贈送的每個電子郵件套裝建立一個電子郵件產品。如果你是第一次設定產品，請從[建立你的第一個訂閱產品](../../user-guide/configuration/creating-your-first-subscription-product)開始，然後回到這裡進行電子郵件專屬設定。

1. 前往 **Network Admin > Ultimate Multisite > Products > Add Product**。
2. 將 **Product Type** 設為 **Email**。
3. 設定價格。若為銷售方案，請使用付費週期性價格；若應為贈送的電子郵件方案，則將產品設為免費／零價格。
4. 開啟 **電子郵件設定**區段。
5. 選取已設定的 **電子郵件提供者**。
6. 視需要設定 **帳號數量**、**儲存配額（MB）**、別名限制、轉寄器與自動回覆。
7. 儲存產品。

只有在 **Integrations** 頁面上已啟用且完整設定的提供者，才會出現在提供者選擇器中。如果選擇器是空的，請先設定並測試提供者。

## 將電子郵件加入結帳 {#add-email-to-checkout}

電子郵件產品通常會作為網站方案的加購項目提供。一般結帳編輯器工作流程請參閱[結帳表單](../../user-guide/configuration/checkout-forms)。

1. 建立或編輯客戶首先購買的網站方案。
2. 將電子郵件產品加入為該結帳流程可用的加購項目／訂單加購。
3. 確認結帳表單仍包含一般必要的網站欄位，尤其是網站 URL／網域欄位。
4. 使用測試環境客戶執行測試結帳。
5. 結帳後，開啟客戶的 [Account 頁面](../../user-guide/client-management/account-page)，並確認**電子郵件帳號**面板出現。

電子郵件網域取自客戶的網站網域。客戶不會在電子郵件產品步驟中輸入電子郵件網域。

## 客戶 Account 電子郵件管理 {#customer-account-email-management}

版本 1.1.0 將電子郵件管理小工具新增到客戶 Account 頁面。當客戶應該能夠自助建立與維護其自身網站的信箱時使用它。

從 Account 頁面，客戶可以：

1. 檢視連結到其目前電子郵件會員資格／網站網域的電子郵件帳號。
2. 在其電子郵件產品允許額外帳號時建立新信箱。
3. 變更現有信箱密碼。
4. 透過 SSO 或魔術連結登入開啟支援的網路郵件提供者。

此小工具使用 Ultimate Multisite Account 頁面目前狀態系統，因此動作會限定在客戶正在檢視的 Account 情境中。這可防止客戶管理屬於其他會員資格或網站的電子郵件帳號。

如果客戶沒有看到電子郵件面板，請確認他們有**有效的電子郵件產品會員資格**、該會員資格已連結到網站網域，且該產品已設定提供者。

## 給客戶免費電子郵件方案 {#giving-a-customer-a-free-email-plan}

若要免費提供客戶電子郵件，請先授予**電子郵件權益**，然後建立信箱，或讓客戶自行建立信箱。

建議選項：

* **建立免費電子郵件產品**：將電子郵件產品價格設為免費/零，然後從 **Ultimate Multisite > 會員資格 > 新增會員資格** 為該客戶新增有效會員資格。選取電子郵件產品與客戶。免費會員資格在變更或取消前皆為終身有效。如果電子郵件會員資格應跟隨現有網站，請確認它已連結到該網站/網域情境。
* **使用 100% 折扣或免費結帳連結**：讓客戶完成結帳，使 Ultimate Multisite 自動建立電子郵件會員資格與付款紀錄。
* **管理員建立信箱**：前往 **Ultimate Multisite > 電子郵件帳號 > 新增電子郵件帳號**，為客戶在提供者端建立特定信箱。這對支援很有用，但它不會取代用來支援客戶自助帳戶頁面工具的電子郵件產品會員資格。

如果你希望客戶能從帳戶頁面自行管理信箱，請確認他們對該網域有有效的電子郵件會員資格。沒有該會員資格而手動建立的提供者信箱可以存在，但客戶可能看不到其自助電子郵件面板。

## 管理員建立的帳號與「沒有訂單號碼」 {#admin-created-accounts-and-no-order-number}

**電子郵件帳號**管理頁面會建立或編輯提供者信箱。它不是銷售或授予電子郵件方案的第一步。

編輯電子郵件帳號時，**會員資格**欄位會依照參考/訂單號碼搜尋現有的 Ultimate Multisite 會員資格。如果你在客戶擁有電子郵件會員資格之前嘗試指派帳號，可能沒有可選取的訂單號碼。

請改用此操作順序：

1. 建立或找到客戶的電子郵件會員資格。若為購買，結帳會完成此動作。若為贈送方案，請從[會員資格](../../user-guide/administration/managing-memberships)手動建立會員資格，或讓客戶進行免費結帳。
2. 確認會員資格為有效，且使用電子郵件產品。
3. 確認電子郵件會員資格已連結到網站網域/提供者。
4. 從客戶帳戶頁面或從 **Ultimate Multisite > 電子郵件帳號** 建立信箱。
5. 如有需要，編輯信箱紀錄並指派客戶、會員資格與網站。

如果你只需要為客戶建立提供者信箱，而不需要帳戶頁面的自助管理，管理員建立的帳號可能就足夠。如果你希望客戶在 Ultimate Multisite 中看到並管理它，也請建立電子郵件會員資格。

## 網頁郵件 SSO 與魔法連結登入 {#webmail-sso-and-magic-link-login}

網頁郵件 SSO 會在帳戶頁面提供客戶直接的**開啟網頁郵件**動作。當有效提供者支援時，此外掛會請求短效存取 URL，並將客戶重新導向至網頁郵件，而不要求他們複製信箱憑證。

提供者行為可能不同：

* 具備原生 SSO API 的提供者可以建立直接驗證的網頁郵件工作階段。
* 支援魔法連結的提供者可以產生有時效限制的登入連結。
* 不支援 SSO 或魔法連結的提供者仍應顯示信箱詳細資料與密碼管理動作，但不會有一鍵網頁郵件按鈕。

僅為你已設定所需 API 憑證的提供者啟用網頁郵件存取。如果某個提供者缺少該動作，請先確認該提供者已啟用、已設定，且支援網頁郵件登入連結。

## 結帳佈建 {#checkout-provisioning}

此外掛會在結帳期間佈建電子郵件**會員資格情境**。它會記錄提供者、帳號限制、配額與網站網域，讓客戶在購買後可從帳戶頁面建立自己的信箱名稱。

建議推出流程：

1. 在提供電子郵件產品之前，先設定提供者與產品限制。
2. 使用測試環境產品測試購買，並確認電子郵件會員資格已建立。
3. 檢查客戶帳戶頁面，確認該網域出現**電子郵件帳號**小工具。
4. 從小工具建立測試信箱。
5. 在為正式產品啟用流程之前，確認變更密碼與網頁郵件動作。

如果佈建失敗，客戶帳號與會員資格仍可能存在，而電子郵件會員資格或提供者信箱需要管理員後續處理。重試之前，請檢查 Ultimate Multisite 活動記錄與提供者記錄。

## CyberPanel 電子郵件轉寄器 {#cyberpanel-email-forwarders}

對於以 CyberPanel 支援的網路，版本 1.1.0 新增電子郵件轉寄器支援。當客戶需要像 `info@customer-domain.test` 這類位址將郵件重新導向到另一個收件匣，而不建立完整信箱時，請使用轉寄器。

在提供轉寄器之前，請確認 CyberPanel 整合已設定且可連線。請參閱 [CyberPanel 整合](../../user-guide/administration/cyberpanel-integration)指南，了解所需的主機、使用者名稱、密碼與套件常數。

## 常見問題 {#frequently-asked-questions}

### 客戶可以建立自己的電子郵件 Account 嗎？ {#can-customers-create-their-own-email-accounts}

可以。當客戶擁有某個 domain 的有效電子郵件 membership，且 product 允許額外 Account 時，客戶可以從 Account 頁面建立並管理自己的信箱。

### 每個提供者都支援 Webmail SSO 嗎？ {#does-every-provider-support-webmail-sso}

不支援。SSO 與 magic-link 支援取決於啟用中的電子郵件提供者。沒有 access-link API 的提供者仍可支援建立信箱與變更密碼。

### 管理員可以為客戶建立電子郵件 Account 嗎？ {#can-administrators-create-email-accounts-for-customers}

可以。管理員可以從管理員介面建立電子郵件 Account，包括在建立 Account 時設定初始密碼。若要讓客戶自助管理，也請確認客戶對該 domain 擁有有效的電子郵件 membership。

### 我可以免費贈送電子郵件 Account 嗎？ {#can-i-give-an-email-account-away-for-free}

可以。建立免費／零價格的 Email product，並將其新增為客戶的有效 membership，或讓客戶使用 100% 折扣完成 checkout。電子郵件 membership 存在後，客戶或管理員即可建立實際信箱。

### 為什麼指派電子郵件 Account 時會要求訂單編號？ {#why-does-assigning-an-email-account-ask-for-an-order-number}

Membership 欄位會依據其 reference/order number 查找既有的 Ultimate Multisite membership。如果客戶未購買電子郵件 product，且你尚未手動建立電子郵件 membership，就沒有可指派的電子郵件 membership/order。請先建立電子郵件 membership，然後再建立或指派信箱。

### 轉寄器和信箱一樣嗎？ {#are-forwarders-the-same-as-mailboxes}

不一樣。信箱會儲存郵件並擁有登入憑證。轉寄器會將郵件重新導向到另一個地址，且不提供獨立收件匣。
