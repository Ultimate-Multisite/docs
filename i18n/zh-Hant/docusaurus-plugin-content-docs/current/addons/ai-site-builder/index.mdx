# AI 網站建構器

ultimate-multisite-ai-site-builder

## 必要條件 {#requirements}

- WordPress 5.3 或更高版本
- PHP 7.4 或更高版本
- Multisite Ultimate 外掛 (需啟用)

## 安裝 {#installation}

1. 將附加元件檔案上傳到您的 `/wp-content/plugins/` 目錄
2. 透過 WordPress 的「外掛」選單啟用該外掛
3. 在網路管理員介面中設定附加元件選項

## 開發 {#development}

### 設定 {#setup}

```bash
# 安裝依賴套件
composer install
npm install

# 執行測試
npm run test

# 執行程式碼標準檢查
vendor/bin/phpcs
vendor/bin/phpstan

# 修復程式碼風格問題
vendor/bin/phpcbf

# 為生產環境建構
npm run build
```

### 專案結構 {#project-structure}

```
ultimate-multisite-ai-site-builder/
├── inc/                          # PHP 類別和包含檔案
│   ├── class-multisite-ultimate-updater.php  # 更新處理器
│   └── class-ai_site_builder-main.php             # 主要功能
├── tests/                        # 單元測試
│   ├── bootstrap.php            # 測試啟動檔案
│   ├── class-*-base.php         # 基礎測試類別
│   └── class-*-test.php         # 測試案例
├── views/                       # 範本檔案
├── assets/                      # 靜態資源
│   ├── css/                     # 樣式表
│   ├── js/                      # JavaScript 檔案
│   └── images/                  # 圖片
├── lang/                        # 語言檔案
├── composer.json                # PHP 依賴套件
├── package.json                 # Node.js 依賴套件
├── phpunit.xml.dist             # PHPUnit 設定
├── .phpcs.xml.dist              # 程式碼標準設定
└── rector.php                   # 程式碼現代化規則
```

### 測試 {#testing}

此附加元件包含一個全面的測試框架：

- **單元測試 (Unit Tests)**：基於 PHPUnit 的測試，並整合了 WordPress 測試套件
- **基礎測試類別 (Base Test Class)**：提供用於常見測試情境的輔助方法
- **測試覆蓋率 (Test Coverage)**：已設定追蹤程式碼覆蓋率
- **多站點測試 (Multisite Testing)**：測試在多站點環境中執行

### 程式碼標準 {#code-standards}

- **WordPress 程式碼標準 (WordPress Coding Standards)**：透過 PHPCS 執行強制檢查
- **PHP 7.4+ 相容性 (Compatibility)**：支援現代 PHP 功能
- **靜態分析 (Static Analysis)**：整合 PHPStan 進行型別檢查
- **程式碼現代化 (Code Modernization)**：使用 Rector 規則進行 PHP 升級

### 建構系統 {#build-system}

- **資源處理 (Asset Processing)**：對 CSS/JS 檔案進行最小化 (Minification)
- **翻譯 (Translation)**：生成 POT 檔案
- **打包 (Packaging)**：建立用於發佈的壓縮檔
- **開發模式 (Development Mode)**：提供未最小化的資源，方便除錯

## 設定 {#configuration}

此附加元件支援多種設定選項，您可以在網路管理員介面中存取這些選項。

## 鉤子與過濾器 {#hooks-and-filters}

### 動作 (Actions) {#actions}

- `ultimate-multisite-ai-site-builder_init` - 在附加元件初始化後觸發
- `ultimate-multisite-ai-site-builder_loaded` - 在所有附加元件檔案載入後觸發

### 過濾器 (Filters) {#filters}

- `ultimate-multisite-ai-site-builder_settings` - 修改附加元件設定
- `ultimate-multisite-ai-site-builder_enabled` - 覆寫附加元件的啟用/停用狀態
