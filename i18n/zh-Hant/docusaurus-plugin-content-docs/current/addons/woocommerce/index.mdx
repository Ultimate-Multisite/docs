# WooCommerce 整合

WooCommerce 整合的運作方式與我們的手動閘道非常相似。它為你的使用者提供大量彈性，讓他們可以選擇你的 WooCommerce 安裝中可用的任何付款表單。而且很簡單：一旦使用者計費週期結束，使用者訂閱就會被暫停。同時，會在網路主網站的 WooCommerce 安裝中建立一筆訂單。接著使用者會在電子郵件中收到發票，其中包含進行付款的連結。一旦收到付款，訂閱就會續訂。

## 功能

- Ultimate Multisite 與 WooCommerce 之間的簡易整合
- 可用多種付款方式
- 使用隨附的訂閱核心進行自動週期性計費，並可選擇使用 WooCommerce Subscriptions plugin 增強功能
- 結帳總額會將 Ultimate Multisite 稅額包含為獨立且不可課稅的 WooCommerce 費用項目，讓客戶可在付款前檢視稅額

## 安裝

1. 將 'ultimate-multisite-woocommerce' 上傳到 '/wp-content/plugins/' 目錄
2. 透過 WordPress 中的 'Plugins' 選單啟用 plugin
3. 依照逐步精靈設定 plugin

## 使用指南

_**重要注意事項：本文適用於 Ultimate Multisite 2.x 版本。**_

_**注意：** Ultimate Multisite: WooCommerce Integration 要求 WooCommerce 至少在你的主網站上啟用。_

我們了解 _Stripe_ 和 _PayPal_ 在某些國家/地區無法使用，這會限制或阻礙 Ultimate Multisite 使用者有效使用我們的 plugin。因此我們建立了一個**用於整合 _WooCommerce_ 的附加元件**。

_WooCommerce_ 是非常受歡迎的電子商務 plugin。世界各地的開發者建立了附加元件，以便將不同的付款閘道整合到其中。我們利用這一點來擴充你可在 Ultimate Multisite 計費系統中使用的付款閘道。

## 設定

首先，你需要安裝 WooCommerce Integration 附加元件，該附加元件**對授權使用者免費**。若要安裝，請在你的超級管理員 Dashboard 上按一下 Ultimate Multisite，然後選擇 **附加元件**。

![附加元件頁面](/img/admin/addons-page.png)

你可以在這裡找到所有 Ultimate Multisite 附加元件。按一下 **Ultimate Multisite: WooCommerce Integration** 附加元件。

![顯示 WooCommerce Integration 的附加元件頁面](/img/admin/addons-page.png)

會彈出一個包含附加元件詳細資訊的視窗。只要按一下 **立即安裝** *。*

<!-- 截圖不可用：WooCommerce Integration 附加元件安裝彈出視窗，含「立即安裝」按鈕 -->

安裝完成後，你將被重新導向到 plugins 頁面。在這裡，只要按一下 **網路啟用**，_WooCommerce_ 附加元件就會在你的網路上啟用。

<!-- 截圖不可用：WP Plugins 頁面顯示 WooCommerce Integration 的「網路啟用」按鈕 -->

啟用後，如果你尚未在網站上安裝並啟用 WooCommerce plugin，**你將收到提醒**。

<!-- 截圖不可用：管理員通知提醒安裝並啟用 WooCommerce plugin -->

安裝完成後，你會在 Ultimate Multisite 付款設定下看到一個額外選項。接著你可以在那裡**啟用** _WooCommerce_ 選項。

![已啟用 WooCommerce 的 Ultimate Multisite 付款閘道設定](/img/addons/woocommerce-payment-gateway.png)

你可以設定一些基本設定，例如顯示在前端的顯示名稱，以及引導使用者完成結帳流程的說明。接著你可以前往你的**主網站 _WooCommerce_ 設定**，以啟用並設定你偏好使用的付款閘道。

### 結帳稅務費用項目

當 Ultimate Multisite 結帳包含稅額時，此整合會將每個 Ultimate Multisite 稅額作為獨立且不可課稅的費用項目加入 WooCommerce 結帳。這會讓稅額總計在付款前保持可見，同時防止 WooCommerce 再次對稅務費用課稅。

## 如何管理付款

WooCommerce Integration 的運作方式與我們的**手動閘道**非常相似。缺點是每筆付款都必須在計費週期結束時手動支付。以下是流程的運作方式：

  * 一旦使用者的計費週期結束，會員資格就會被**暫停**。同時，會在你網路主網站上的 _WooCommerce_ 安裝中建立一筆訂單。

  * 使用者接著會在電子郵件中收到發票，其中包含進行付款的連結（該連結也會出現在其子網站 Dashboard 的計費記錄中）。該連結會將使用者重新導向到 _**WooCommerce**_ **結帳**，使用者可在其中選擇一個**可用的 _WooCommerce_ 閘道**來處理付款。

  * 一旦收到付款，會員資格就會**續訂**，付款會記錄在 Ultimate Multisite 中，訂單也會設定為完成。

### 一些重要注意事項：

  * Ultimate Multisite 的預設功能仍可與此整合形式搭配使用。例如，從**訂閱管理**畫面**發放退款**會與 WooCommerce 通訊，以使用所選付款表單發放退款。

  * Ultimate Multisite: WooCommerce Integration 依賴 WP-Cron 運作。這在大多數時候運作良好，但如果網站流量低，cron 就不會被觸發以檢查排程事件，事件也會被錯過，這可能導致執行延遲。

## WooCommerce Subscriptions Plugin

每個月手動啟動付款對終端使用者而言並不理想，因為訂閱預期會在每個計費週期自動重複。WooCommerce Integration add-on 現在已綑綁核心 subscriptions code，因此定期產品一律使用 subscriptions 進行定期計費。

你**不**需要購買或安裝獨立的 [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) 即可透過此整合銷售定期產品。如果該 plugin 已購買、安裝，並可在你的主網站上使用，Ultimate Multisite 會使用它，而 WooCommerce 會公開額外的訂閱管理功能。若它不可用，綑綁的 subscriptions core 仍會處理定期產品。

![WooCommerce 訂閱整合設定](/img/addons/woocommerce-subscription-integration.png)

就這樣。你的終端使用者將不再需要在其定期產品的計費週期結束時手動付款。

除了會在你的主網站的 _WooCommerce_ 下建立訂單之外，也會為每個帳戶建立一個訂閱。

### 續訂失敗時的停用行為

預設情況下，memberships 會在 WooCommerce Subscriptions 處理其付款失敗重試排程時保持啟用。這會保留正常的 WooCommerce Subscriptions 寬限期，讓恢復的續訂能將訂閱與 membership 重新同步，而不會過早中斷存取權。

想要更嚴格存取控制的管理員，可以在 WooCommerce Integration 設定中啟用 **Suspend Memberships Immediately on Failed Renewals**。啟用這項選擇性設定後，Ultimate Multisite 會在續訂付款失敗時立即暫停相關 membership，而不是等待 WooCommerce Subscriptions 重試處理完成。

開發人員可以使用 `wu_woo_suspend_on_payment_failure` filter 覆寫相同的決策。Hook 參考頁面會從 add-on 原始碼產生，因此本概覽僅指出 filter 名稱，以供自訂暫停政策使用。
