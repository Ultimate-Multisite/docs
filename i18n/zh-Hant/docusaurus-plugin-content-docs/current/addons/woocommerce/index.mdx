# WooCommerce 整合

WooCommerce 整合的功能與我們的手動閘道（Manual Gateway）非常相似。它為您的用戶提供了極大的彈性，讓他們可以選擇您 WooCommerce 安裝中任何可用的付款方式。操作非常簡單：一旦用戶的計費週期結束，用戶的訂閱就會被暫停。同時，系統會在網路主站的 WooCommerce 安裝中創建一筆訂單。用戶隨後會收到一封包含付款連結的電子發票。一旦收到付款，訂閱就會自動續訂。

## 功能特色

- Ultimate Multisite 與 WooCommerce 之間的輕鬆整合
- 提供多種付款方式
- 結合 WooCommerce Subscriptions，實現完全自動化的流程

## 文件說明

**設定 WooCommerce 整合**

**使用 Ultimate Multisite 擴展您的付款選項：WooCommerce 整合**

## 安裝

1. 將 'ultimate-multisite-woocommerce' 上傳到 '/wp-content/plugins/' 目錄
2. 透過 WordPress 的「Plugins」選單啟用此插件
3. 遵循逐步嚮導（Wizard）來設定插件

## 使用指南

_**重要提示：本文檔適用於 Ultimate Multisite 2.x 版本。**_

_**注意：** Ultimate Multisite: WooCommerce Integration 要求您的主站已啟用 WooCommerce。_

我們了解在某些國家，由於限制或阻礙，用戶無法有效使用我們的插件，特別是對於 _Stripe_ 和 _PayPal_ 這類支付方式。因此，我們創建了一個 **附加元件來整合 _WooCommerce_**。

_WooCommerce_ 是一個非常受歡迎的電子商務插件。全球的開發者為它創建了許多附加元件，用來整合不同的付款閘道。我們利用了這一點，擴展了您可以在 Ultimate Multisite 計費系統中使用的付款閘道。

## 設定步驟

首先，您需要安裝 WooCommerce Integration 附加元件，此元件對付費用戶是**免費**的。要安裝它，請點擊超級管理員儀表板上的 Ultimate Multisite，然後選擇 **Add-ons**。

![Add-ons page](/img/admin/addons-page.png)

在這裡，您可以找到所有 Ultimate Multisite 的附加元件。請點擊 **Ultimate Multisite: WooCommerce Integration** 附加元件。

![Add-ons page showing WooCommerce Integration](/img/admin/addons-page.png)

一個包含附加元件詳細資訊的視窗會彈出。只需點擊 **Install Now** *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

安裝完成後，您將被重定向到插件頁面。在這裡，只需點擊 **Network Activate**，_WooCommerce_ 附加元件就會在您的整個網路中啟用。

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

啟用後，如果您的網站尚未安裝並啟用 WooCommerce 插件，**系統會提醒您**。

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

安裝完成後，您會在 Ultimate Multisite 付款設定中看到一個額外的選項。然後您可以從那裡**啟用** _WooCommerce_ 選項。

![Payment gateway settings](/img/config/settings-payment-gateways.png)

您可以配置一些基本設定，例如前端顯示的名稱，以及指導用戶在結帳流程中的說明。然後，您可以前往您的 **主站的 _WooCommerce_ 設定**，啟用並配置您偏好的付款閘道。

## 如何管理付款

WooCommerce Integration 的運作方式與我們的 **手動閘道** 非常相似。缺點是，每次付款都必須在計費週期結束時手動完成。流程如下：

  * 一旦用戶的計費週期結束，會員資格就會被**暫停**。同時，系統會在網路主站的 _WooCommerce_ 安裝中創建一筆訂單。

  * 用戶隨後會收到一封電子發票，其中包含付款連結（該連結也會出現在其子站儀表板的計費歷史記錄中）。該連結會將用戶重定向到 **_WooCommerce_ 結帳** 頁面，用戶可以在那裡選擇**可用的 _WooCommerce_ 閘道**來處理付款。

  * 一旦收到付款，會員資格就會**續訂**，付款記錄會記錄在 Ultimate Multisite 中，並且訂單會被標記為已完成。

### 幾點重要注意事項：

  * Ultimate Multisite 的預設功能仍與此整合形式協同工作。例如，從 **訂閱管理** 畫面**發放退款**時，系統會與 WooCommerce 溝通，使用所選的付款方式來處理退款。

  * Ultimate Multisite: WooCommerce Integration 依賴 WP-Cron 才能正常運作。大多數時候這都能很好地運作，但如果網站流量較低，Cron 就沒有被提示檢查排程事件，導致事件遺漏，從而可能造成執行延遲。

## WooCommerce Subscriptions 插件

每個月手動發起付款對於終端用戶來說並不理想，因為訂閱服務預期會在每個計費週期自動遞延。因此，我們設計了 WooCommerce Integration 附加元件來**支援 WooCommerce Subscriptions 插件**。這將自動化付款流程，讓您的終端用戶更容易操作。

您只需要**安裝** [Woocommerce Subscriptions 插件](https://woocommerce.com/products/woocommerce-subscriptions/)，並在**主站啟用**它。然後返回 Ultimate Multisite 付款設定，**啟用整合**即可。

就是這樣。您的終端用戶將不再需要在計費週期結束時手動付款。

除了在主站的 _WooCommerce_ 下創建一筆訂單外，它還會為每個帳戶創建一個訂閱記錄。
