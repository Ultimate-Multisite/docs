# 多租戶

將子網站分散到多部伺服器與資料庫，以提升效能與安全性

## Version 1.2.0 發布重點

Multi-Tenancy 1.2.0 為正式環境隔離工作流程新增主權租戶基礎：

- 具備短效、重放保護 SSO 造訪的無狀態租戶自動登入。
- 為主權租戶提供每個子網站的資料庫與檔案系統隔離。
- Hostinger 多租戶能力，用於感知主機的租戶佈建。
- 主權遷移驗證關卡，用於舊資料檢查與非同步推送處理。
- 可設定的同機器資料庫主機繫結，適用於 Bedrock、FrankenPHP 與 localhost 安裝。
- 租戶啟動、資料庫寫入器、結構描述啟動與使用者佈建改進。
- 更安全的主權拆除，會清理租戶憑證。

請參閱使用者指南中的[主權遷移工作流程](/user-guide/getting-started/sovereign-migration-workflow)、[多租戶隔離](/user-guide/configuration/multi-tenancy-isolation)與[主權租戶操作](/user-guide/administration/sovereign-tenant-operations)。

## 需求

- WordPress 5.3 或更高版本
- PHP 7.4 或更高版本
- Multisite Ultimate plugin（已啟用）

## 安裝

1. 將 addon 檔案上傳到你的 `/wp-content/plugins/` 目錄
2. 透過 WordPress 中的「Plugins」選單啟用 plugin
3. 在網路管理中設定 addon 設定

## 開發

### 設定

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### 測試

此 addon 包含完整的測試框架：

- **單元測試**：基於 PHPUnit 的測試，搭配 WordPress 測試套件
- **基礎測試類別**：為常見測試情境提供輔助方法
- **測試覆蓋率**：已設定以追蹤程式碼覆蓋率
- **Multisite 測試**：測試在 multisite 環境中執行

### 程式碼標準

- **WordPress Coding Standards**：透過 PHPCS 強制執行
- **PHP 7.4+ 相容性**：支援現代 PHP 功能
- **靜態分析**：整合 PHPStan 進行型別檢查
- **程式碼現代化**：用於 PHP 升級的 Rector 規則

### 建置系統

- **資產處理**：CSS/JS 檔案的最小化
- **翻譯**：POT 檔案產生
- **封裝**：建立用於發佈的封存檔
- **開發模式**：用於偵錯的未最小化資產

## 設定

此 addon 支援可透過網路管理介面存取的各種設定選項。

### 主權租戶設定

對於主權租戶，在將流量路由到租戶之前，請設定租戶資料庫、檔案系統根目錄、資料庫主機繫結與主機能力。每次變更影響資料庫憑證、主機繫結、路由或使用者佈建時，都要執行遷移驗證。

只有在租戶執行環境與資料庫授權預期使用本機 socket 或同機器存取時，才使用 `localhost`。當你的供應商設定的授權主機是 `127.0.0.1` 或服務主機名稱時，請使用它們。

### Hostinger 能力

Hostinger 多租戶能力會與共用的 Hostinger 整合 token 搭配運作。驗證前，請確認網域存在於 hPanel 中、DNS 指向 Hostinger 帳戶，且租戶資料庫設定符合 hPanel。

## Hooks 與 Filters

### Actions

- `ultimate-multisite-addon-template_init` - 在 addon 初始化後觸發
- `ultimate-multisite-addon-template_loaded` - 在所有 addon 檔案載入後觸發

### Filters

- `ultimate-multisite-addon-template_settings` - 修改 addon 設定
- `ultimate-multisite-addon-template_enabled` - 覆寫 addon 啟用／停用狀態
