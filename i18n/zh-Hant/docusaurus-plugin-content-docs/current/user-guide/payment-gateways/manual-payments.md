# 設定手動付款 (v2)

_**重要注意事項：本文適用於 Ultimate Multisite 2.x 版本。**_

如果你的使用者無法使用 **Stripe** 或 **PayPal**，手動付款是一種讓你提供其他付款方式的方法。它可以是電匯、銀行轉帳，或任何你的使用者在當地可用的其他付款方式。

## 如何啟用手動付款 {#how-to-enable-manual-payments}

設定手動付款非常簡單。你只需要在付款閘道下啟用它，並輸入使用者應如何付款的詳細說明。

首先，前往 **Ultimate Multisite > 設定 > 付款**。在 **付款閘道** 下方，開啟 **手動**。你會看到一個 **付款說明** 方塊出現。

在這個方塊中加入你的客戶付款時所需的資訊。例如，可以是你的銀行帳戶詳細資料，以及你的電子郵件，讓客戶可以將付款確認寄給你。

![含有付款說明文字區域的手動付款閘道切換開關](/img/config/manual-gateway-expanded.png)

這是手動閘道設定介面：

![手動閘道設定](/img/config/manual-gateway-settings.png)

設定完成後，只要按一下 **儲存設定** 就完成了。當使用者註冊到你的網路時，他們會看到一則訊息，告知他們將收到你的說明以完成購買。

![註冊確認訊息，告知使用者將收到付款說明](/img/frontend/registration-manual-notice.png)

他們也會在你的 **感謝你** 頁面上收到一則包含你付款說明的訊息。

## 確認手動付款 {#confirming-manual-payments}

若要確認手動付款，請前往左側欄的 **付款** 選單。在那裡，你可以看到網路上的所有付款及其詳細資料，包括它們的 **狀態**。手動付款在你手動變更之前，狀態一律會是 **待處理**。

![付款清單顯示待處理的手動付款](/img/admin/payments-list.png)

按一下 **參考代碼** 進入付款頁面。在此頁面上，你可以看到待處理付款的所有詳細資料，例如參考 ID、產品、時間戳記等。

![付款詳細資料頁面顯示參考代碼、產品與總計](/img/admin/payment-edit.png)

在右側欄中，你可以更改付款狀態。將其變更為 **已完成**，並**切換啟用會員資格** 選項，將會啟用你客戶的網站，並啟用他們的會員資格。

![付款編輯頁面，其中狀態設定為已完成，並開啟啟用會員資格切換開關](/img/admin/payment-activate-membership.png)
