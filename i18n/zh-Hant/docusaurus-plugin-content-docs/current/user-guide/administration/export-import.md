# 匯出與匯入

Ultimate Multisite 2.9.0 在 **工具 > 匯出與匯入** 下新增了一個單站 **匯出與匯入** 工具。當您需要將一個 WordPress 網站打包成 ZIP 檔案、還原該 ZIP 檔案，或在相容的 Ultimate Multisite 和單站 WordPress 安裝之間遷移網站時，請使用此工具。

## 必要權限

您必須以可以存取被匯出或匯入網站 WordPress **工具** 選單的管理員身份登入。在多站網路中，當您從網路層級的 Ultimate Multisite 工具匯出或匯入子站時，請使用網路管理員帳號。

匯出 ZIP 下載是透過經過驗證的下載端點服務的，因此請保持管理員會話活躍，直到下載完成，並且不要公開分享生成的下載 URL。

## 將網站匯出為 ZIP

1. 在您想要複製的網站的 WordPress 管理介面中，前往 **工具 > 匯出與匯入**。
2. 開啟匯出區域，並選擇您想要打包的網站。
3. 選擇包含的選用內容，例如上傳檔案、外掛和主題，前提是這些選項可用。
4. 開始匯出，並等待流程完成。大型網站可能需要在背景完成處理，之後 ZIP 檔案才會準備好。
5. 從匯出清單下載完成的 ZIP 檔案。

請將 ZIP 檔案保存在安全的位置。它可能包含網站內容、設定、媒體檔案和選定的程式碼資產。

## 匯出包含的內容

匯出 ZIP 檔案可以包含：

- 選定網站的資料庫內容和設定。
- 當包含上傳檔案時，上傳的媒體檔案。
- 當選擇了這些選項時，外掛和主題。
- 匯出與匯入工具用來在目標安裝上重建網站的匯入元資料。

確切的 ZIP 大小取決於媒體量、選定的外掛和主題，以及網站資料庫表格的大小。

## 從 ZIP 匯入網站

1. 在目標 WordPress 網站上，前往 **工具 > 匯出與匯入**。
2. 開啟匯入區域，並上傳由匯出與匯入工具建立的 ZIP 檔案。
3. 如果網站需要使用新的網址，請輸入替換 URL；否則，請將欄位留空以保留原始網址。
4. 選擇匯入完成後是否要刪除上傳的 ZIP 檔案。
5. 點擊 **開始匯入**。
6. 監控待處理的匯入，直到它完成。只有當您需要在完成前停止流程時，才使用 **取消匯入**。
7. 在允許正常流量或客戶存取之前，請審核匯入後的網站。

在單站 WordPress 安裝上，匯入 ZIP 檔案會用匯入的網站取代目前的網站。在開始之前，請為目標網站建立完整的備份，並避免同時為同一網站啟動多次匯入。

## 限制與相容性注意事項

- 非常大的上傳目錄或媒體庫可能會產生很大的 ZIP 檔案。在匯出或匯入大型網站之前，請確認 PHP 上傳限制、執行限制、磁碟空間、記憶體和伺服器超時設定。
- 非常大的媒體庫可能需要在流量較低的維護時段移動。
- 目標 WordPress 安裝應運行相容的 WordPress、PHP、Ultimate Multisite、外掛和主題版本。
- 單站匯入會取代目標網站。它不是一個合併工具。
- 支援多站到單站和單站到多站的遷移，僅當目標環境能夠運行匯出網站的外掛、主題、網址和所需的 Ultimate Multisite 組件時才支援。
- 請將 ZIP 檔案保密。它可能包含來自匯出網站的資料庫內容、上傳檔案和設定細節。

如需了解舊版的網路層級匯出流程，請參閱 [網站匯出](/user-guide/administration/site-export)。
