# Ultimate Multisite 系統需求

在購買 Ultimate Multisite 之前，你可能會想知道執行這個 plugin 需要哪些條件。好消息是，執行 Ultimate Multisite 只需要幾個基本要求：一個**網域名稱**、一個優質的**主機服務商**，以及一個 WordPress **Multisite** 安裝環境。就這麼簡單！

## 網域名稱

首先，來談談網域名稱。

在選擇網域名稱時，請考慮你的專業領域，以及你打算用 Ultimate Multisite 來做什麼：你想要經營一間網站代理公司？銷售類似商店的網站？還是提供 WordPress 網站代管服務？使用 Ultimate Multisite，一切皆有可能。

我們的 plugin 提供訂閱服務、收款功能、結帳表單、折扣優惠券和客戶通訊等功能。安裝、設定和維護 WordPress Multisite 所需的大量工作，都可以透過 Ultimate Multisite 輕鬆完成。

只要你是以銷售網站服務為主，任何專業領域都可以使用我們的 plugin 來管理！請務必選擇一個符合你商業計畫的網域名稱。

## 主機服務商

現在你已經有了一個能代表你專業領域和業務的亮眼網域名稱，接下來需要選擇一家優質的網站主機公司來託管你的 Ultimate Multisite 網路。市面上有一些不錯的選擇。部分主機服務商與 Ultimate Multisite 整合得很好，這可能會是你做決定時的關鍵因素，取決於你計畫建立什麼樣的網站。

Ultimate Multisite 應該可以在任何主機服務商上運作。唯一的差別是，像 _WPMU DEV Hosting_、_WP Engine_、_Closte_、_Cloudways_、_Gridpane_ 和 _Runcloud_ 這些主機公司，在網域對應方面有更緊密的整合。對於其他主機，你可能需要手動對應自訂網域。選擇支援 AutoSSL 的主機服務商也是個好主意。_Closte_、_Runcloud_ 和 _cPanel_ 都支援此功能。

至於伺服器規格，沒有固定的公式，因為這取決於你要提供的功能和網路規模。共享主機可能是個不錯的起點，但當你的網路開始成長時，你可能會需要專屬的 VPS。

## WordPress Multisite

現在，來談談 WordPress Multisite 的安裝。

WordPress Multisite 讓你可以在單一安裝環境中擁有多個網站組成的網路。這是內建功能，但預設並未啟用。你需要在 WordPress 安裝環境和網站主機控制台中啟用這項功能。

啟用 WordPress Multisite **是使用 Ultimate Multisite 的基礎**。由於 Ultimate Multisite 的主要用途是幫助使用者管理和銷售網站服務，這必須在已啟用 Multisite 的 WordPress 上進行。

我們有**一篇詳細的文章**，逐步教你如何在 WordPress 安裝環境中啟用 Multisite 功能。

當你在 WordPress 上啟用 Multisite 後，就可以安裝 Ultimate Multisite 了。要正確安裝 Ultimate Multisite，你必須具備：

  * WordPress：v5.3 以上（建議使用：最新穩定版本）

  * PHP：7.4.x（即將支援 > 8.0）

  * MySQL：v5 以上（建議使用：5.6，如果你的主機服務商沒有提供 8.0 版本的話）

以上就是 Ultimate Multisite 的所有軟體需求。

請注意，Ultimate Multisite 可能可以在 PHP 8.0 上運作，但我們建議使用 PHP 7.4.x 來執行。

此外，我們建議將它安裝在**主網域**上，而非子網域或子目錄。它可能可以在子網域或子目錄上運作，但可能會出現一些錯誤。

如需 Ultimate Multisite 的詳細安裝指南，請**點擊這裡**。
