# 版本更新紀錄

## Version 2.12.0 — 發布日期 2026-05-15

- 新增：新增了 Hostinger (hPanel) 作為支援的主機提供者，並整合了網域對映功能
- 新增：Site Exporter 現在能處理網路匯入套件，讓您能更順暢地進行全站點恢復
- 修復：BCC 群發電子郵件現在使用了 undisclosed-recipients 標頭，防止洩露收件人地址
- 修復：當儲存時，如果會員資格沒有提供日期值，到期日將不再損壞
- 修復：Stripe 會員更新現在能正確清除折扣，而無需呼叫已棄用的 deleteDiscount API
- 修復：在網域對映的網站上，SSO 的重定向現在有上限限制，防止無限重定向迴圈發生
- 修復：設定嚮導的圖片選擇器現在能正確更新底層資料模型
- 修復：Site Exporter CLI 現在能保留正確的預設網路網站選擇
- 優化：移除了套件中的 wp-cli，減小了插件的體積

## Version 2.11.0 — 發布日期 2026-05-11

- 新增：網站匯出現在會打包一個自啟動的 `index.php`，這樣即使在沒有單獨安裝插件的新主機上，ZIP 檔案也能成功安裝。
- 新增：網路匯出功能讓管理員可以從 Site Export 管理頁面，將所有子站點匯出到單一的壓縮檔中。
- 新增：現在透過回退鏈（fallback chain）強制執行「允許 Site Templates」的開關，能正確限制範本的可用性，以符合方案限制。
- 新增：結帳表單編輯器會在您新增產品但未設定必填欄位時發出警告。
- 新增：匯入/匯出設定分頁現在清楚描述了其範圍，並直接連結到 Site Export 工具。

## Version 2.10.0 — 發布日期 2026-05-05

- 新增：為 PayPal 提供了輔助設定嚮導，支援手動輸入憑證，並透過 OAuth 旗標閘門，讓閘道配置更順暢。
- 新增：範本切換客戶面板經過重新設計，新增了「目前範本卡片」、「持久化網格」和 **重設目前範本** 按鈕。
- 修復：範本切換功能在 AJAX 失敗時，不再導致使用者介面卡住。
- 修復：範本切換的權限狀態已加強，防止未經授權的存取。
- 修復：網站覆寫輸入欄位在儲存前會進行驗證。
- 修復：當地址為空時，現在會顯示帳單地址提示。
- 修復：解決了 PHP 8.1 的 Null 到字串棄用通知問題。
- 修復：Currents 在 init 鉤子之前進行延遲載入，以防止時間點問題。
- 修復：跨所有登入流程，都會尊重過濾後的 SSO 路徑。
- 修復：空白網站的身份選項在儲存時會被保留。

## Version 2.9.0 — 發布日期 2026-04-30

- 新增：在 **工具 > 匯出與匯入** 下新增了單站點的匯出和匯入功能。
- 修復：匯出 ZIP 檔案現在會透過經過身份驗證的下載端點提供。
- 修復：修正了待處理匯出/匯入查詢中的 SQL 注入風險和查詢問題。
- 修復：當管理員手動驗證客戶電子郵件時，待處理的網站不會被發布。
- 修復：當缺少會員資格時，會清理孤立的 pending_site 記錄。
- 修復：修正了設定導航欄的內邊距和搜尋錨點導航問題。
- 修復：待處理的網站現在會顯示在「所有網站」視圖的第一位。
- 修復：為截圖提供者 (mShots) 新增了 User-Agent 標頭，以防止 403 錯誤。
- 修復：解決了匯入 Cron 排程的循環依賴問題。
- 修復：使用者設定中的 Tour ID 已正規化為底線。
- 優化：現在使用 ZipArchive 代替 Alchemy/Zippy，以獲得更好的相容性。

## Version 2.8.0 — 發布日期 2026-04-29

- 新增：在其他選項設定介面新增了 Enable Jumper 開關。
- 新增：在結帳表單列表表格中新增了狀態欄。
- 新增：為自訂 MU-plugin sunrise 擴充功能，新增了 Addon sunrise 檔案載入器。
- 優化：從設定頁面移除了錯誤報告選擇加入設定。
- 修復：感謝頁的網站卡片 — 圖片現在有約束，且連結樣式正確。
- 修復：截圖提供者已從 thum.io 切換到 WordPress.com mShots。
- 修復：Enable Registration 和 Default Role 在全新安裝時會設定正確的預設值。
- 修復：`get_site_url()` 不再在網域包含埠號時返回空值。
- 修復：當 `copy_media` 設定為空時，媒體檔案現在能正確複製。
- 修復：在網路啟用 sitemeta 寫入後，物件快取會正確失效。
- 修復：對於三段式網域，自訂網域現在會在 DNS 驗證後自動提升為主要網域。
- 修復：當清理過期的付款時，待處理的會員資格會被取消。
- 修復：密碼強度檢查器在內嵌登入提示被關閉後，會重新綁定。
- 修復：當網站已經建立時，感謝頁不會發生無限頁面重新載入。
- 修復：WP core 註冊選項在插件啟用和設定儲存時會同步。
- 修復：在 `calculate_expiration` 中新增了 Null 到期保護機制，以支援 PHP 8.4。
- 修復：當客戶已經擁有活躍會員資格時，會阻止重複註冊。
- 修復：結帳流程中為 `date_expiration` 新增了 Null 檢查。
- 修復：網站配置流程已加固 — 限制、會員資格推斷、網域提升。
- 修復：當檢查失敗時，預先安裝檢查狀態標籤會修正為 NOT Activated。
- 修復：結帳網域用於電子郵件驗證 URL。
- 修復：當沒有密碼欄位時，結帳後會自動登入。
- 修復：免費會員資格不再到期 — 將視為終身有效。
- 修復：電子郵件驗證閘門會暫停網站發布，直到客戶驗證電子郵件為止。
- 修復：SES v2 API 端點基礎路徑和身份路由已修正。
- 修復：`wu_inline_login_error` 鉤子會在 pre-submit 捕獲區塊中發出。
