# 客戶回饋與問題報告

Gratis AI Agent v1.5.0 內建了回饋和問題報告系統，讓終端用戶可以直接在聊天介面中標記無幫助的回應，並報告遇到的問題。此系統包含同意管理、自動報告指令，以及後端的人工智慧輔助分類（triage）。

## 👎 縮小拇指按鈕

AI 助理發送的每一則訊息都會顯示一個**縮小拇指**（👎）按鈕。當用戶點擊它時，可以將該回應標記為無幫助或不正確。

- 此按鈕會在懸停時出現在每個助理訊息旁邊。
- 點擊它會開啟**回饋同意彈窗**（Feedback Consent Modal）。
- 提交的回饋會與整個對話串和特定的訊息關聯。

## 回饋同意彈窗

當用戶點擊縮小拇指按鈕時，在任何資料被傳送之前，會先彈出一個同意彈窗。此彈窗會：

- 解釋將收集哪些資訊（對話摘錄、瀏覽器上下文）。
- 要求用戶確認同意分享這些資料。
- 提供一個文字輸入欄，讓用戶描述問題所在。
- 提供一個**取消**選項，用於不提交而關閉彈窗。

除非用戶明確確認，否則不會記錄任何回饋。

## 自動提示回饋橫幅

在對話結束時，助理可能會顯示一個**自動提示回饋橫幅**（auto-prompt feedback banner）——這是一個不具侵入性的訊息，詢問用戶本次會話是否有幫助。

此橫幅會根據對話長度和結果的判斷規則自動顯示。它連結到與縮小拇指按鈕相同的回饋流程。用戶可以在不提供回饋的情況下關閉此橫幅。

## /report-issue 斜線指令

用戶可以直接在聊天輸入框中輸入 `/report-issue` 來觸發回饋流程。此指令會：

- 立即開啟回饋同意彈窗。
- 將關於當前對話的上下文預先填入描述欄位。
- 允許用戶在提交前添加更多細節。

`/report-issue` 指令在所有聊天模式（內嵌式、浮動小工具、全螢幕）中均可用。

## AI 輔助分類

提交的回饋會被導向一個 AI 分類層，該層會：

- 對報告進行分類（例如：事實錯誤、無幫助的答案、不當內容等）。
- 附加相關的對話元資料。
- 將分類摘要轉發到配置的**回饋端點**（Feedback Endpoint）（參見 [設定 > 進階](#settings-advanced)）。

這能透過優先呈現最關鍵的問題，來減少人工審核的時間。

## 設定 > 進階 {#settings-advanced}

若要啟用回饋後端，請在 **Gratis AI Agent → 設定 → 進階** 下配置以下欄位：

| 欄位 | 描述 |
|---|---|
| **Feedback Endpoint URL** | 接收帶有回饋負載（JSON）POST 請求的 URL。 |
| **Feedback API Key** | 傳送在 `Authorization` 標頭中的 Bearer token，用於驗證回饋提交。 |

若要停用回饋收集，請將兩個欄位留空。雖然個別的回饋按鈕仍會對用戶顯示，但提交的內容將不會被轉發。

:::tip
回饋端點必須接受一個包含至少 `message_id`、`conversation_id`、`feedback_text` 和 `triage_category` 欄位的 JSON 內容。請參閱您的端點提供商的文件，了解預期的結構。
:::
