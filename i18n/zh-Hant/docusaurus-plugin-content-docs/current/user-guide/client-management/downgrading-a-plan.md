# 降級方案（v2）

_**重要注意事項：本文適用於 Ultimate Multisite 版本 2.x。**_

降級方案或訂閱是您的客戶在預算有限，或決定不需要太多資源來執行其子網站時可能會進行的常見動作。

## 如何降級方案

您的客戶可以隨時登入其子網站管理 Dashboard，並在其 Account 頁面下點擊 **變更** 來降級其方案。

![含有您的會員資格卡片與變更按鈕的客戶 Account 頁面](/img/account-page/membership-change-button.png)

點擊 **變更** 按鈕後，使用者／客戶將被重新導向至結帳頁面，在那裡他們可以選擇想要將訂閱變更為哪個方案。

![客戶端的降級方案選項頁面](/img/account-page/downgrade-picker.png)

在此範例中，我們要將方案從 **Premium** 降級為 **Free**。

若要繼續，使用者只需要點擊 **完成結帳** 按鈕。接著會將他們帶回 Account 頁面，並顯示關於會員資格待處理變更的訊息。這些變更將在客戶的**下一個帳單週期**生效。

![顯示待處理會員資格變更橫幅的 Account 頁面](/img/account-page/pending-change.png)

### 使用者降級其方案時會發生什麼事

請務必注意，降級方案不會改變使用者子網站中的現有設定。

它不會自動變更網站範本，因為變更網站範本會完全清除並重設子網站。這是為了避免不必要的資料遺失。因此，磁碟空間、佈景主題、外掛程式等都會保持不變，文章除外。

我們了解您的主要顧慮會是您在每個方案下設定的限制與配額，但我們必須考量如果刪除或變更使用者子網站的任何設定，可能會對其造成的損害。

對於超過方案所設限制的文章，您有 3 個不同選項：**保留文章原樣** *，* **將文章移至回收桶** *，* 或 **將文章移至草稿** *。* 您可以在 Ultimate Multisite 設定中設定此項目。

![Network Admin 設定網站頁面，顯示超過文章限制時的行為選項](/img/account-page/settings-sites-post-limit.png)

### 付款會發生什麼事

在版本 2.0 中，就按比例計費而言，不再需要對付款進行任何調整。

這是因為系統會等待現有會員資格**完成其帳單週期後**，新方案／會員資格才會生效。新會員資格的新帳單金額將會在下一個帳單週期自動套用並收取。
