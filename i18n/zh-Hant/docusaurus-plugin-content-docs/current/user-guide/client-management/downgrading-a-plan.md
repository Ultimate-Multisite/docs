# 降級方案 (v2)

_**重要提醒：本文適用於 Ultimate Multisite 2.x 版本。**_

降級方案或訂閱是您的客戶可能會進行的常見操作，通常是因為預算有限，或是他們認為不需要太多資源來經營子網站。

## 如何降級方案

您的客戶可以隨時登入子網站管理後台，在帳戶頁面點擊 **Change** 來降級方案。

![帳戶頁面，會員資格下方顯示 Change 按鈕](/img/admin/memberships-list.png)

點擊 **Change** 按鈕後，使用者/客戶會被導向結帳頁面，在這裡可以選擇想要變更的訂閱方案。

![結帳頁面顯示可降級的方案選項](/img/admin/memberships-list.png)

在這個範例中，我們將方案從 **Premium** 降級為 **Free**。

使用者只需點擊 **Complete Checkout** 按鈕即可繼續。接著會返回帳戶頁面，顯示會員資格待變更的訊息。變更將在客戶的**下一個帳單週期**生效。

![帳戶頁面顯示會員資格待變更訊息](/img/admin/memberships-list.png)

### 使用者降級方案後會發生什麼事

請注意，降級方案不會改變使用者子網站的現有設定。

系統不會自動變更網站範本，因為變更網站範本會完全清除並重設子網站。這是為了避免不必要的資料遺失。因此，磁碟空間、佈景主題、外掛等都會保持不變，只有文章會受到影響。

我們了解您最關心的可能是每個方案設定的限制和配額，但我們必須考慮到刪除或變更使用者子網站設定可能造成的損害。

對於超過方案限制的文章，您有 3 種不同的處理選項：**保留文章原狀**、**將文章移至垃圾桶**，或**將文章改為草稿**。您可以在 Ultimate Multisite 設定中進行配置。

![Ultimate Multisite 設定中的文章超出限制處理選項](/img/config/settings-sites.png)

### 付款會如何處理

在 2.0 版本中，不再需要進行任何按比例計費的付款調整。

這是因為系統會等待現有會員資格**完成其帳單週期後**，新方案/會員資格才會生效。新會員資格的帳單金額會自動套用，並在下一個帳單週期收取。
