# 第七課：打造屬於您的品牌

您的客戶不應該覺得他們在使用「某個 WordPress 插件」。他們應該覺得他們在使用 FitSite——一個專為他們行業打造的平台。本課將涵蓋品牌化、白牌化，以及如何讓整個平台看起來像一個完整的產品。

## 我們上次停在哪裡

FitSite 已經具備一個完整的結帳流程，能引導健身工作室的擁有者從選擇方案到擁有一個上線的網站。現在，我們要讓整個體驗感覺像一個協調一致、具有品牌風格的產品。

## 您的平台網域

品牌的基礎就是您的網域。對於 FitSite 來說：

- **主網域 (Main domain)**：`fitsite.com` (您的行銷網站和網路根目錄)
- **客戶網站 (Customer sites)**：`studioname.fitsite.com` (子網域)
- **自訂網域 (Custom domains)**：Growth 和 Pro 方案的客戶可以綁定他們自己的網域

### 設定您的網域

1. 註冊您的平台網域
2. 將其指向您的主機提供商
3. 為客戶子網域配置通配符 DNS (`*.fitsite.com`)
4. 確保通配符 SSL 已啟用

請參閱 [如何配置網域映射](/user-guide/domain-mapping/how-to-configure-domain-mapping) 以獲取詳細說明。

## 白牌化管理後台體驗

當健身工作室的擁有者登入他們的網站儀表板時，他們應該看到的是您的品牌，而不是 WordPress 或 Ultimate Multisite 的品牌標識。

### 自訂登入頁面

客製化 WordPress 登入頁面，顯示以下內容：

- 您的 FitSite 標誌
- 適合健身主題的背景圖片
- 您的品牌顏色

### 儀表板品牌化

使用 [Admin Page Creator](/addons/admin-page-creator) 插件或自訂 CSS 來：

- 將 WordPress 標誌替換為您的 FitSite 標誌
- 客製化管理後台的配色方案，使其與您的品牌一致
- 新增一個客製化的儀表板小工具，包含健身相關的快速連結和幫助資源

### 客製化管理頁面

考慮建立客製化的管理頁面，將對健身工作室擁有者最相關的操作放在最顯眼的位置：

- 「編輯您的課程時間表」
- 「更新教練資料」
- 「查看您的網站」

這樣做可以降低學習門檻，將與行業相關的操作放在最前面，而不是將它們深埋在標準的 WordPress 選單中。

## 品牌化您的溝通內容

每一封電子郵件、每一張發票和每一條通知都應該強化您的品牌形象。

### 系統電子郵件

導航至 **Ultimate Multisite > Settings > Emails**，客製化所有系統電子郵件：

- **寄件人名稱 (From name)**：FitSite
- **寄件人電子郵件 (From email)**：hello@fitsite.com
- **電子郵件範本 (Email templates)**：使用您的品牌顏色和標誌
- **語言 (Language)**：全程使用健身相關的用語

需要客製化的關鍵電子郵件：

| 電子郵件 | 通用版本 | FitSite 版本 |
| :--- | :--- | :--- |
| 歡迎信 | "您的新網站已準備就緒" | "您的健身工作室網站已上線" |
| 付款收據 | "已收到付款" | "FitSite 訂閱付款已確認" |
| 試用期結束 | "您的試用期即將結束" | "您的 FitSite 試用期剩餘 3 天——請保持您的工作室網站上線" |

### 發票

使用以下內容客製化發票範本：

- 您的 FitSite 標誌和品牌顏色
- 您的企業詳細資訊
- 健身相關的產品名稱（而非通用的方案 ID）

## 客戶端網站

您的主網域 (`fitsite.com`) 需要一個銷售平台的行銷網站。這與 Ultimate Multisite 的網路管理後台是分開的——它是您業務的公開門面。

關鍵頁面：

- **首頁 (Homepage)**：為健身企業提供清晰的價值主張
- **功能 (Features)**：FitSite 能做什麼，用健身術語來描述
- **定價 (Pricing)**：您的三個方案，並包含行業特定的功能比較
- **範例 (Examples)**：展示使用本平台建立的網站
- **註冊 (Sign Up)**：連結到您的結帳表單

:::tip 您的行銷網站可以是一個網路網站
您可以將行銷網站建立為您自己網路中的一個網站。這樣您就可以從同一個儀表板管理它，同時展示您自己的平台能力。
:::

## 客戶的自訂網域

對於包含自訂網域的方案客戶，請清楚地記錄流程：

1. 客戶將其網域註冊或轉移到註冊商處
2. 客戶更新 DNS，將其指向您的平台（提供確切的記錄）
3. Ultimate Multisite 負責網域映射和 SSL 處理

請為此流程創建一篇幫助文章或知識庫條目，內容應以非技術背景的健身工作室擁有者為目標。

## FitSite 網路目前建置的內容

```
FitSite Network
├── WordPress Multisite (子網域模式)
├── Ultimate Multisite (已配置 + 品牌化)
├── 平台網域 (fitsite.com + 通配符 SSL)
├── 網站範本 (Studio Essential, Gym Pro, Fitness Chain)
├── 產品 (Starter, Growth, Pro + 附加產品)
├── 結帳流程 (行業特定，已測試)
├── 品牌化
│   ├── 客製化登入頁面
│   ├── 品牌化管理儀表板
│   ├── 行業特定系統電子郵件
│   ├── 品牌化發票
│   └── 位於 fitsite.com 的行銷網站
└── 已準備好進行客戶入職流程 (下一課)
```

## 本課所建置的內容

- 為品牌化體驗配置了**平台網域和 DNS**
- 具備 FitSite 品牌化的**白牌管理後台**
- **客製化的溝通內容**——電子郵件、發票和通知全部符合品牌形象
- 一個銷售 FitSite 給健身工作室擁有者的**行銷網站**
- 為希望使用自訂網域的客戶準備了**自訂網域文件**
