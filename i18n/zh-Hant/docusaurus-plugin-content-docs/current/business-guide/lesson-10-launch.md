# 第 10 課：上線日

所有東西都建好了。在您正式開門營業之前，本課將帶您走過預上線檢查清單，確保沒有任何東西出錯、遺漏或令人尷尬的地方。

## 我們的進度

FitSite 已經配置了模板、方案、結帳流程、品牌識別、客戶入職和定價等所有功能。現在，我們需要驗證所有功能是否正常運作，然後正式上線。

## 預上線檢查清單

請逐一檢查清單上的每一項。切勿跳過任何步驟。

### 平台基礎設施

- [ ] 託管服務是否穩定，且在高負載下表現良好
- [ ] Wildcard SSL 是否已啟用，所有子網域是否都透過 HTTPS 服務
- [ ] 網域對應是否正常運作 — 請測試建立一個網站並對應一個自訂網域
- [ ] 是否已配置並測試了備份（請至少還原一次以驗證）
- [ ] 是否已設置監控系統 — 這樣您就能知道平台是否會當機

### 模板

- [ ] 所有三個模板在新網站上是否都能正確載入
- [ ] 佔位內容是否實用且沒有錯別字
- [ ] 所有圖片是否擁有適當的授權（不能有圖庫照片的水印）
- [ ] 所有模板頁面上的行動裝置響應式設計是否正常
- [ ] 頁面載入速度是否可接受（請使用 GTmetrix 或 PageSpeed Insights 等工具進行測試）
- [ ] 任何模板上是否沒有壞鏈接或遺失的資源

### 方案與產品

- [ ] 所有三個方案是否都已啟用且可見
- [ ] 方案描述是否準確且符合特定利基市場
- [ ] 定價是否正確（月付和年付）
- [ ] 是否已在正確的方案上配置了設置費
- [ ] 試用期是否在 Starter 方案上正常運作
- [ ] 訂單加購品（Order bumps）在結帳過程中是否正確顯示
- [ ] 插件和主題的限制是否已根據每個方案正確執行

### 結帳流程

- [ ] 請針對每個方案完成一次完整的測試註冊（使用測試支付模式）
- [ ] 模板選擇是否顯示了每個方案對應的正確模板
- [ ] 支付流程是否成功
- [ ] 註冊後，客戶是否會收到歡迎電子郵件
- [ ] 新網站是否使用正確的模板成功建立
- [ ] 客戶是否可以立即登入其新網站
- [ ] 折扣代碼是否能正確使用

### 品牌識別

- [ ] 登入頁面是否顯示 FitSite 的品牌識別
- [ ] 管理員儀表板（Admin dashboard）是否顯示 FitSite 的品牌識別
- [ ] 所有系統電子郵件是否使用 FitSite 的品牌識別和健身相關的語言
- [ ] 發票是否正確顯示了您的企業詳細資訊
- [ ] 行銷網站是否已上線，並且連結到結帳表單

### 客戶入職

- [ ] 快速入門（Quick Start）小工具是否出現在新客戶的儀表板上
- [ ] 所有快速入門連結是否指向正確的頁面
- [ ] 歡迎電子郵件序列是否已配置並測試
- [ ] 知識庫文章是否已發布且可存取
- [ ] 帳戶頁面是否顯示了正確的方案資訊和升級選項

### 法律與業務

- [ ] 服務條款是否已發布，並從結帳頁面連結到
- [ ] 隱私權政策是否已發布且可存取
- [ ] 退款政策是否已定義並記錄在案
- [ ] 接收付款的企業實體是否已設立
- [ ] 支付閘道（Payment gateway）是否處於正式模式（而非測試模式）
- [ ] 稅務配置是否正確，符合您的司法管轄區

## 軟啟動 vs. 硬啟動

建議採用兩階段的發布策略：

### 第一階段：軟啟動（Soft Launch）

在對公眾發布之前，邀請 5 到 10 位健身工作室老闆註冊。這些是您的 Beta 客戶。您可以提供給他們大幅折扣（例如終身五折，或免費使用三個月），作為交換條件：

- 對註冊和入職體驗提供誠實的反饋
- 允許您將他們的工作室網站作為展示案例
- 報告他們遇到的任何錯誤或問題

這能讓您在向公眾開放之前，獲得真實的客戶反饋和可展示的實體網站。

### 第二階段：公眾發布（Public Launch）

在整合了軟啟動的反饋後：

- 將支付閘道切換到正式模式
- 發布您的行銷網站
- 開始客戶獲取（請參考第 11 課）
- 在相關的健身行業頻道宣布

## 上線日行動清單

在您正式對外開放的那一天：

1. **切換到正式支付** — 在您的支付閘道上停用測試模式
2. **再次驗證** — 使用真實付款進行一次完整的測試註冊（之後請記得退款給自己）
3. **密切監控** — 留意錯誤、註冊失敗或支付問題
4. **保持待命** — 您最初的真實客戶可能需要幫助，快速回應能建立信任
5. **簡短慶祝** — 然後重新投入工作

## 可能出錯的地方

請為以下情況做好準備：

- **支付閘道問題**：準備好您的閘道提供商的支援聯絡方式
- **SSL 憑證問題**：了解如何檢查和續訂憑證
- **電子郵件傳送失敗**：測試電子郵件是否真的送達（檢查垃圾郵件資料夾）
- **負載下的性能問題**：如果您突然流量激增，請知道如何擴展您的託管服務
- **客戶混淆**：準備好您的知識庫和支援管道
