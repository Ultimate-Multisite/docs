# بازخورندې او مسائله رپرتینګ (Customer Feedback & Issue Reporting)

Gratis AI Agent v1.5.0 یو درکار سیستم د بازخورونو او مسائلو رپرت کولو نظام راغlere چې پای کارونکي کولی شي له چټ چیټ رابطو څخه ناکام ځوابونه 신고 کړي او مشکلات مستقیم رپورت کړي. دا سیستم شامل سرلیک کی مدیریت (consent management)، یوه خودکار رپرت کولو فرمان، او پښې کور کې د AI سره مرسته وکړئ (AI-assisted triage).

## ټ姆بس ډاونټ بټن (Thumbs-Down Button)

د AI助手 له خوا هر پیغام چې فرستیا کیږي، یو **ټ姆بس ډاونټ** (👎) بټن ښیي. کله چې کارونکی یې کلیک کوي، هغه کولی شي ځوابونه د ناکام یا غلط په توګه علامت ورکړي.

- بټن د هر助手 پیغام په څنګ کې هوور کولو پر مهال ښکاري.
- کله چې یې کلیک کوئ، **بازخورونو سرلیک موډل** (Feedback Consent Modal) खुوي.
- بازخور د گفتگو ترڅنګ او ځانګړي پیغام سره تړلی دی.

## بازخورونو سرلیک موډل (Feedback Consent Modal)

کله چې کارونکی ټ姆بس ډاونټ بټن کلیک کوي، د کوم د ډیټا فرستولو څخه مخکې یو سرلیک موډل ښکاري. دا موډل:

- توضیح ورکوي چې کوم معلومات ترلاسه کیږي (د گفتگو کاپټ، براوزرسونکي سیاق).
- کارونکي څخه غواړي چې تایید کړي چې دوی له شریکولو د دې ډیټا سره موافقه لري.
- یو آزاد متن خان وړاندې کوي ترڅو کارونکی بیان کړي چې څه غلط شوی و.
- **Cancel** خيار وړاندې کوي چې پرته له رپورت کولو یې ختم شي.

تر کارونکي څرګند تایید नगर्ने وضعیت کې، کوم بازخور ثبت نه کیږي.

## خودکار پرومټ بازخور banner (Auto-Prompt Feedback Banner)

د گفتگو پای ته کله،助手 ممکن یو **خودکار پرومټ بازخور banner** ښکاري — یو غیر مزاحم پیغام چې پوښتنه کوي که آیا هغه جلسه ګټور وه.

دا banner د گفتگو او پایلو اساس له لارې خودکار اخیستل کیږي. دا په ټ姆بس ډاونټ بټن سره هم ورته بازخورไหล منحال دی. کارونکي کولی شي بدون اینکه بازخور ورکړي، banner ختم کړي.

## /report-issue سلاش فرمان (/report-issue Slash Command)

کارونکي کولی شي د چټ چیټ ورودی کې `/report-issue` टाइप गरेर مستقیم بازخورไหล ته تحریک کړي. دا فرمان:

- دکمه‌ی "Feedback Consent Modal" (پنجشینی رضامندی بازخورد) بلافاصله باز می‌شود.
- فیلد توضیحات با اطلاعات مربوط به مکالمه فعلی پر می‌شود.
- به کاربران اجازه می‌دهد قبل از ارسال، جزئیات بیشتری اضافه کنند.

دستور `/report-issue` در تمام حالت‌های چت (inline، ویجت شناور، و صفحه کامل) در دسترس است.

## AI-Assisted Triage (مرتب‌سازی با کمک هوش مصنوعی)

بازخورد ارسالی به یک لایه مرتب‌سازی هوش مصنوعی هدایت می‌شود که:

- گزارش را دسته‌بندی می‌کند (خطای واقعی، پاسخ کمکی نبودن، محتوای نامناسب و غیره).
- داده‌های مرتبط مکالمه را ضمیمه می‌کند.
- خلاصه مرتب‌سازی را به **Feedback Endpoint** پیکربندی شده ارسال می‌کند (به [Settings > Advanced](#settings-advanced) مراجعه کنید).

این کار زمان بازبینی دستی را با نمایش مهم‌ترین مسائل در ابتدا کاهش می‌دهد.

## Settings > Advanced {#settings-advanced} (تنظیمات > پیشرفته)

برای فعال کردن بک‌اند بازخورد، فیلدهای زیر را در بخش **Gratis AI Agent → Settings → Advanced** تنظیم کنید:

| Field (فیلد) | Description (توضیح) |
|---|---|
| **Feedback Endpoint URL** (آدرس نقطه پایانی بازخورد) | آدرسی که درخواست‌های POST حاوی محتوای بازخورد (JSON) را دریافت می‌کند. |
| **Feedback API Key** (کلید API بازخورد) | توکن Bearer که در هدر `Authorization` برای احراز هویت ارسال می‌شود تا ارسال بازخوردها تأیید شوند. |

اگر هر دو فیلد را خالی بگذارید، جمع‌آوری بازخورد غیرفعال می‌شود. دکمه‌های بازخورد فردی همچنان برای کاربران قابل مشاهده خواهند بود، اما ارسال‌ها به جایی منتقل نخواهند شد.

:::tip
نقطه پایانی بازخورد باید یک بدنه JSON با حداقل فیلدهای `message_id`، `conversation_id`، `feedback_text` و `triage_category` را بپذیرد. برای طرح داده مورد انتظار، به مستندات ارائه‌دهنده نقطه پایانی خود مراجعه کنید.
:::
