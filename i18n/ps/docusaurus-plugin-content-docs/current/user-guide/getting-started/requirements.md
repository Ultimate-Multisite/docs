# Ultimate Multisite کی ضروریات

Ultimate Multisite خریدنے سے پہلے آپ اپنے آپ سے سب سے پہلے جو سوال پوچھ سکتے ہیں وہ یہ ہے کہ ہمارے پگ ان (plugin) کو چلانے کے لیے کیا ضروریات ہیں۔ خوش قسمتی سے، Ultimate Multisite چلانے کے لیے صرف چند بنیادی ضروریات ہیں: ایک **ڈومین نیم**، ایک اچھا **ہوسٹنگ پرووائڈر** اور ایک ورڈپریس **Multisite** انسٹالیشن۔ بس اتنا ہی آپ کو چاہیے!

## ڈومین نیم {#domain-name}

سب سے پہلے، آئیے ڈومین نیم کے بارے میں بات کرتے ہیں۔

جب آپ ڈومین نیم کا انتخاب کر رہے ہوں تو اس بات کا خیال رکھیں کہ آپ کی خاص مہارت (niche) کیا ہے اور Ultimate Multisite کے ساتھ آپ کیا بنانے والے ہیں: کیا آپ ایک ایجنسی بنانا چاہتے ہیں؟ دکانوں جیسے ویب سائٹس بیچنا چاہتے ہیں؟ یا شاید ورڈپریس ویب ہوسٹنگ؟ Ultimate Multisite کے ساتھ ہر چیز ممکن ہے۔

ہمارا پگ ان سبسکرپشن سروسز، ادائیگی جمع کرنے، چیک آؤٹ فارمز، رعایت کے وؤچرز اور کسٹمر کمیونیکیشن فراہم کرتا ہے۔ ایک ورڈپریس Multisite کو انسٹال کرنے، ترتیب دینے اور برقرار رکھنے کے لیے بہت زیادہ کام Ultimate Multisite کی مدد سے آسان ہو جاتا ہے۔

آپ جو بھی مہارت منتخب کرتے ہیں اسے ہمارے پگ ان کے ذریعے سنبھالا جا سکتا ہے بشرط کہ آپ ویب سائٹس کو سروس کے طور پر بیچ رہے ہوں! ایسا ڈومین نیم منتخب کریں جو آپ کے بزنس پلان سے میل کھاتا ہو۔

## ہوسٹنگ پرووائڈر {#hosting-provider}

اب جب آپ کا ایک اچھا ڈومین نیم ہے جو آپ کی مہارت اور کاروبار کی نمائندگی کرتا ہے، تو آپ کو Ultimate Multisite کے ساتھ اپنے نیٹ ورک کو ہوسٹ کرنے کے لیے ایک اچھی ویب ہوسٹنگ کمپنی کا انتخاب کرنا ہوگا۔ مارکیٹ میں کچھ اچھے اختیارات ہیں۔ کچھ ہوسٹنگ پرووائڈرز Ultimate Multisite کے ساتھ اچھی طرح سے جڑتے ہیں، جو اس بات پر منحصر ہو سکتا ہے کہ آپ کیا بنانے کی منصوبہ بندی کر رہے ہیں۔

Ultimate Multisite هر ډول هاستنگ پرایدر کې کاروي باید. یوازې فرق دا دی چې یو څه هاستنگ کمپنۍ لکه _WPMU DEV Hosting_، _WP Engine_، _Closte_، _Cloudways_، _Gridpane_ او _Runcloud_ د ډامین مپینګ لپاره سخت تریکي (tighter integration) لري. د نورو هاستنګونو لپاره، ممکن تاسو اړتیا ولرئ چې ځانګړې ډامینونه manuallly مپ کړئ. دا هم ښه فکر دی چې یو ایسی هاستنگ پرایدر ترلاسه کړئ چې AutoSSL ستا apoi ورکوي. _Closte_، _Runcloud_ او _cPanel_ دا کار کوي.

د سرور مشخصات (server specifications) له مخې، هیڅ خاصสูตร نه ده چې دا د هغه خوندونو او کارو په اساس deği mudar کیږي او د ستونټ (network) اندازه هم مهم دی. یو اشتراکی هاستنگ (shared hosting) ښه نقطه شروع کیدو مهرباني دی، तर به چې ستونټ پیل کړي، تاسو احتمالاً یو ځانګړی VPS ته اړتیا لرئ.

## WordPress Multisite {#wordpress-multisite}

خاور اوس، اترای له یوه WordPress Multisite تثبيت په اړه خبرې وکړو.

WordPress Multisite اجازه ورکوي چې تاسو د یو واحد تثبيت کې د سایتونو یو شبکه (network) جوړ کړئ. دا یو内置 (built-in) خصه کار دی، خو معمولاً فعال نه دی. تاسو باید دا کار په خپل WordPress تثبيت او ویب هاستنگ کنټرول پینل کې فعال کړئ.

د Multisite فعال شوې WordPress ته **Ultimate Multisite د بنسټ دی**. ځکه چې Ultimate Multisite اصلي کار دا ده چې کارونکو ته یې سایتونه به مدیریت کولو او د خدمت په توګه به ويلي (as a service) وکاروي، نو دا کار باید په هغه WordPress کې شي چې Multisite فعال وي.

موږ **یو ژور مقاله** لرې چې تاسو ته ګام په ګام نه ښایي چې څنګه ستاسو WordPress تثبيتमा Multisite خصه کار فعال کړئ.

د WordPress کې د Multisite فعال کولو وروسته، وخت راغلی دی چې Ultimate Multisite تثبیت کړئ. Ultimate Multisite په سمه توګه تثبیت کولو لپاره، تاسو باید دا شي لرئ:

  * WordPress: v5.3+ (توصیه: وروستۍ پایدار نسخه)

  * PHP: 7.4.x (د > 8.0 لپاره دعم به راشي وي)

  * MySQL: v5+ (توصیه: 5.6، که ستاسو هاستنگ پرایدر کې 8.0 نه وي)

د دې لپاره یوازې نرم افزار کیتایل (software requirements) دي چې د Ultimate Multisite لپاره اړین دي.

په یاد کې راځئ چې Ultimate Multisite ممکن له PHP 8.0 سره کار کړي، خو موږ توصیه درته کوو چې په PHP 7.4.x سره یې چلائي کړئ.

همره، موږ توصیه درته کوو چې دا په **main domains** (بنیادی د domain) کې هم نصب کړئ، نه په subdomain یا subdirectory کې. دا ممکن د subdomain/subdirectory لپاره کار کړي، خو ممکن یو څه غلطي ورسول شي.

د Ultimate Multisite څنګه نصب কৰل भन्ने बारे مفصل لارښود لپاره، **دلته کلیک وکړئ**.
