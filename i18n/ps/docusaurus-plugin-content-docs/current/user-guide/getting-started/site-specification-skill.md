# مهارت مشخصات سایت (Site Specification Skill)

**مهارت مشخصات سایت (Site Specification skill)** هغه منظم 방법 دی چې تاسو سره مرسته کوي که څنګه موخه، مخاطب او برانډي هویت د ستاسو سایت په اړه معلومات ترلاسه کړئ. دا معلومات په ستاسو **site_brief** کې زیرمه کیږي، چې ایجنټونه په مختلف سیشنونو کې هم له لارې یېを参照وي ترڅو ثابت او سیاقي پوره مرسته ورکړي.

## مشخصات سایت څه دي؟ {#what-is-site-specification}

مشخصات سایت هغه پروسه ده چې دا ثبتوي:

- **سایټको موخه (Site purpose)**: ستاسو سایت څه کوي او ولې دی
- **مخاطب (Target audience)**: څوک ستاسو سایت ته راځي او دوی څه غواړي
- **برانډي هویت (Brand identity)**: ستاسو رنگونه، لهجه او بصري سټایل
- **تجارتي موخه (Business goals)**: د ستاسو سایت لپاره بریالیت څه ښکاري
- **محتووی جوړښت (Content structure)**: ستاسو سایت څنګه منظم دی

دا مشخصات ستاسو **site_brief** ګرځي، یو پایدار حافظه چې ایجنټونه ستاسو سایتको سیاقته په پوه کې لري.

## سایت مشخصات ولې کار اضافه کړئ؟ {#why-use-site-specification}

### سیشنونو کې یکسرهपन (Consistency Across Sessions) {#consistency-across-sessions}

د site_brief پرته، که تاسو هر وخت یو نوی سیشن پیل کړئ، باید ستاسو سایتको موخه بیا توضیح کړئ. د دې سره، ایجنټونه په چټکۍ سره پوهیږي:

- ستاسو سایتको موخه او مخاطب
- ستاسو برانډي رنگونه او لهجه
- ستاسو محتووی جوړښت
- ستاسو تجارتی هدفونه

### ښه وړاندیزونه (Better Recommendations) {#better-recommendations}

ایجنټونه ستاسو site_brief څخه کار کوي ترڅو:

- هغه خیاوې وړاندیز کړي چې د ستاسو سایت موخه سره همخلیږي
- محتووی جوړښتونه وړاندیز کړي چې ستاسو اغلاحې سره سمون وي
- ډیزاینونه وړاندیز کړي چې له برانډي هویت څخه جáveis وي
- هغه خیاوې که نه همخلو، د وړاندیز کولو څخه وایي

### چټک شروع کول (Faster Onboarding) {#faster-onboarding}

نوی ایجنټونه (یا نوي سیشنونو کې ایجنټونه) په ستاسو site_brief لوستل له لارې په چټکۍ سره پوهیږي او نه دا چې وضاحت ورکولو پوښتنې وکړي.

## سایت مشخصات څنګه پیل کړئ؟ {#initiating-site-specification}

### د Theme Builder شروع کولو په وخت کې (During Theme Builder Onboarding) {#during-theme-builder-onboarding}

د **Theme Builder onboarding flow** په جریان کې، Site Specification مهارت له خودي څخه پیل کیږي. Setup Assistant ایجنټ پوښتنې کوي او ستاسو site_brief جوړوي.

### manuall Initiation {#manual-initiation}

تاسو کولی شئیر مشخصات سایت (Site Specification) په هر وخت پیل کړئ:

```
"زه خپل سایت د مشخصات تعریف کولو کې مرسته وکړئ"
```

یا

```
"مې د یو سایت خلاصې جوړول کې مرسته وکړئ"
```

## د سایت مشخصات کی ترتیب (The Site Specification Process) {#the-site-specification-process}

### ګام ۱: د سایت هدف (Site Purpose) {#step-1-site-purpose}

د ایجن پوښتنه کوي:

```
د ستاسو سایت اصلي هدف څه دی؟
- تجارتي پليستي (E-commerce store)
- بلاګ یا محتوا جوړونکی سایت (Blog or content site)
- پورټ فولیو یا ښودل (Portfolio or showcase)
- SaaS اپلیکیشن (SaaS application)
- ټولنې یا فورم (Community or forum)
- نور: [توصیف کړئ]
```

تاسو کولی شئ یو कोटी غوره کړئ او خپل هدف توصیف کړئ.

### ګام ۲: د هدف ډیری (Target Audience) {#step-2-target-audience}

```
د ستاسو اصلي هدف څوک دی؟
- مصرفونکي / عمومي ه လူبند (Consumers / general public)
- کاري متخصصان (Business professionals)
- پراختیاونکي / تکنالوژیک کارونکي (Developers / technical users)
- زده کوونکي / تعلیم ورکونکي (Students / educators)
- نور: [توصیف کړئ]

د دوی اصلي اړتیا څه دي؟
```

### ګام ۳: د برانډ هویت (Brand Identity) {#step-3-brand-identity}

```
د ستاسو برانډ رنگ چیری دي؟
- اصلي رنگ: [رنګ انتخابونکی یا هکس کوډ]
- دوهمي رنگ: [رنګ انتخابونکی یا هکس کوډ]
- اضافه رنگ: [اختیاری]

تاسو خپل برانډ له جهته څنګه توصیف کوئ؟
- حرفي / شرکتي (Professional / corporate)
- خلاقانه / هنري (Creative / artistic)
- لوبې کولو / غیر رسمي (Playful / casual)
- کمینه / عصري (Minimal / modern)
- ګرم / دوستانه (Warm / friendly)
```

### ګام ۴: د کاري ټاکو (Business Goals) {#step-4-business-goals}

```
د ستاسو سایت لپاره بریالیت څه ښکاري؟
- لیدونکي جوړول (Generate leads)
- محصولاتو فرما کول (Sell products)
- ټولنه جوړول (Build community)
- علم شریکول (Share knowledge)
- مرجعیت تثبیت کول (Establish authority)
- نور: [توصیف کړئ]

د ستاسو اصلي معیار څه دی؟
- ترلاسه شوي gelir (Revenue)
- کارونکي ګډون (User engagement)
- محتوا ورسره (Content reach)
- عملونه (Conversions)
- نور
```

### ګام ۵: د محتویات جوړښت (Content Structure) {#step-5-content-structure}

```
د ستاسو محتوا څنګه منظم ده؟
- مسطح (Flat) (ټول محتوا یو سطح کې وي)
- سلسله مراتونی (Hierarchical) (کټګوری او زیرکټګوری)
- زمانی (Chronological) (بلاګ طرز)
- د محصول پر بنسټ (Product-based) (کاتالوگ)
- نور: [توصیف کړئ]

تاسو کوم ډول محتوا کار کوئ؟
- بلاګ پوسټونه (Blog posts)
- د محصول صفحات (Product pages)
- کیسې (Case studies)
- مستندات (Documentation)
- ویډیوونه (Videos)
- نور
```

## ستاسو site_brief حافظه (Your site_brief Memory) {#your-sitebrief-memory}

د سایت مشخصات کی ترتیب پوره کولو وروسته، ستاسو معلومات د **site_brief** په توګه په خپل ایجن حافظه کې زیرمه کیږي. دا یو منظم ثبتیه ده چې موارد زیر شامل دي:

## دۆزین و به‌روزرساندنی Site_brief-ەکەت {#viewing-and-updating-your-sitebrief}

### دۆزینکردنی Site_brief-ەکەت {#view-your-sitebrief}

پێشنیار بکە بە ئا件事ەکە:

```
"Show me my site brief"
```

یان

```
"What do you know about my site?"
```

ئەژمارەکە تایبەتمەندییە تۆم پێشکەش دەکات کە تۆ تۆمار کردووە.

### به‌روزرساندنی Site_brief-ەکەت {#update-your-sitebrief}

ئەگەر سایتت بگۆڕێت، دەتوانی بەهێز بکەیتەوە:

```
"Update my site brief: we're now targeting B2B customers"
```

یان

```
"Refresh my site specification"
```

ئەم کارە توانای Site Specification-ی پاشکەوتنی لەسەر دەدات بە وردەکارییە ئێستای تۆ وەک بنەمای سەرەتایی.

## چۆن Agents (ئەژمارەکان) Site_brief بەکار دێنن {#how-agents-use-sitebrief}

### پێشنیاری دیزاین {#design-recommendations}

کاتێک داواکاریی گۆڕانکاری دیزاینی دەدات، ئەژمارەکان ئاماژە بە Site_brief-ەکەت دەکەن:

```
تۆ: "Redesign my homepage"
ئەژمارەکە: "بنیاد بە Site brief-ەکەت، تۆ خەڵکی ژنانی لە تەمەنی ٢٥-٤٥ ساڵ دەگرێت کە توندی و کارەکتیکییە. من دیزاینێک دروست دەکەم کە گرنگی بە زیوڵە دەستکردەکانت و بەهایە پاشاوتن (sustainable) دەدات."
```

### پێشنیاری تایبەتمەندییەکان {#feature-suggestions}

ئەژمارەکان تایبەتمەندییەک پێشنیار دەکەن کە لەگەڵ ئامانجەکانتدا یەکگرتوو نین:

```
تۆ: "What features should I add?"
ئەژمارەکە: "بۆ ماڵپەڕێکی e-commerce کە گرنگی بە وەفاداریی کڕیار دەدات، من پێشنیار دەکەم: بەرنامەی وەفاداری (loyalty program)، گۆڕینی کڕیاران (customer reviews)، نامەی ئیمەیلی هەواڵەکان (email newsletter)، و پێشنیاری بەرهەم."
```

### پێکهاتەی ناوەڕۆک (Content Structure) {#content-structure}

ئەژمارەکان ڕێکخستنی ناوەڕۆک لەسەر بنەمای پێکهاتەکەت پێشنیار دەکەن:

شما: "من باید بلاگمو څنګه سازمانده کړم؟"
Agent: "ځکه تاسو د محصول کاتو (product catalog) له لاسه ورکړی دی چې بلاګ هم لري، زه وړاندیز کوم چې پوسټونه په محصول کیټګوري (product category) سره منظم کړئ او 'ستايلینګ ګاډی' (styling guides) جوړ کړئ چې ډېر مصنوعي ورته یوځای ښکاره کړي."

## غوره عملونه (Best Practices) {#best-practices}

### دقیق اوسئینه فکر وکړئ (Be Specific) {#be-specific}

د بدل "عمومي وګړي" په بدل کې، خپل د رښتیا هدف مصرفونکي بیان کړئ:

- ✓ "۲۵ تا ۴۵ کلنه ښځې چې پایپالۍ ته علاقه لري"
- ✗ "هغه ټول"

### منظم معلومات تازه کړئ (Update Regularly) {#update-regularly}

د دې لپاره چې ستاسو سایت بدله وي، خپل site_brief تازه کړئ:

- کله چې تاسو به د مصرفونکو په یو نوی هدف پریکړه وکړئ
- کله چې تاسو نوي مصنوعي خطونه اضافه کړئ
- کله چې ستاسو برانډ کیمیتوب بدل شي
- کله چې ستاسو کاري ټاکلونه بدلون ورکړي

### یوځای اصطلاحات प्रयोग کړئ (Use Consistent Terminology) {#use-consistent-terminology}

په ټولګی کې یو او سره اصطلاحات प्रयोग کړئ:

- ✓ هميشه "sustainable jewelry" ووایاست (نه "eco-friendly jewelry" او "green products")
- ✓ خپل مصرفونکي په یو او سره نوم ورکړئ

### سیاق شامل کړئ (Include Context) {#include-context}

دا معلومات ورکړئ چې له لارې د ګډونونکو ته ستاسو پریکړې پیلونه پوه شي:

- "مو زموږ هدف هغه متخصصان دي چې کیفیت ته بولي، نه قیمت ته"
- "مو مصرفونکي تکنالوژۍ ته واقف دي او یې عصري ډیزاین مننه کوي"
- "مو یو کوچنی شرکت دی، نو موږ باید کم لاغری والی راټول کړو"

## له Theme Builder شروع کولو سره اړیکه (Relationship to Theme Builder Onboarding) {#relationship-to-theme-builder-onboarding}

Site Specification مهارت د **Theme Builder onboarding flow** په داخل کې شامل ده. کله چې تاسو onboarding بشپړ کړئ، ستاسو site_brief له هغه معلومات سره د خودکار جوړولو سره جوړیږي چې تاسو یې ورکړی دی.

ستاسو لپاره دا هم ممکن دی چې Site Specification په انفرادي ډول چلالو کړئ که تاسو غواړئ:

- وروسته له پیل کولو وروسته خپل مشخصات اصلاح کړئ
- کله چې ستاسو سایت بدله وي، خپل site brief تازه کړئ
- د Theme Builder شروع کولو دمخه یو مفصل مشخصات جوړ کړئ

## مشکلو حل (Troubleshooting) {#troubleshooting}

**مو site_brief کار نه کیږي**
- ډاغه کړئ چې Agent ته یادونه (memory) ورته رaccesso ورکړی دی
- Agent څخه یې و پوښتنه وکړئ چې "my site brief" بیرته یاد کړي
- وګورئ چې ستاسو تنظیمات کې memory فعال دی

**مخلوط جديد شروع کردن با یک سایت‌بریف جدید می‌خوام**
- از عامل بخواهید: "سایت‌بریف من را پاک کن و از اول شروع کن"
- سپس مشخصات سایت (Site Specification) را دوباره اجرا کنید

**عامل در حال دادن توصیه‌هایی است که با سایت‌بریف من مطابقت ندارد**
- از عامل بخواهید: "سایت‌بریف من را بررسی کن"
- اگر سایت‌بریف شما قدیمی است، آن را به‌روزرسانی کنید
- اطلاعات بیشتری را در درخواست‌های خود ارائه دهید

## مراحل بعدی {#next-steps}

بعد از تعریف مشخصات سایت (Site Specification) خود:

۱. **از Theme Builder استفاده کنید**: یک قالب سفارشی بر اساس سایت‌بریف خود بسازید
۲. **طراحی را اصلاح کنید**: برای کارهای طراحی دقیق، از مهارت Design System Aesthetics استفاده کنید
۳. **برنامه‌ریزی محتوا**: از عامل‌ها برای توصیه‌های ساختار محتوا بخواهید
۴. **ساخت ویژگی‌ها**: درخواست ویژگی‌هایی داشته باشید که با اهداف کسب‌وکار شما همخوانی داشته باشند
