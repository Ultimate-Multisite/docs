# Gratis AI Agent تنظیمات {#gratis-ai-agent-settings}

په Gratis AI Agent کې د **تنظیمات → پرمختللی** پرده د شاته‌پای ادغامونو لپاره د مدیر کچې جوړښت برابروي. دا پاڼه د نظرونو لېږد، د لټون چمتوکوونکي کیلي، د مدیریت شوي Superdav خدمت جوړونه، د Google Calendar کنټرولونه، د TextBee SMS تنظیمات، او د ټولې شبکې د ځانګړتیاوو نښې مستندوي.

## تنظیماتو ته لاسرسی {#accessing-settings}

1. په WordPress مدیر کې، **Gratis AI Agent → تنظیمات** ته لاړ شئ.
2. د **پرمختللی** ټب کېکاږئ.

## د نظرونو endpoint جوړښت {#feedback-endpoint-configuration}

د نظرونو endpoint له AI agent څخه POST غوښتنې ترلاسه کوي کله چې یو کارن د thumbs-down تڼۍ، د اتومات هڅونې بنر، یا د `/report-issue` امر له لارې نظر وسپاري.

| ډګر | تشریح |
|---|---|
| **د نظرونو Endpoint URL** | هغه URL چې د JSON بدن سره د HTTP POST غوښتنو په توګه د نظرونو سپارنې ترلاسه کوي. |
| **Feedback API Key** | یو bearer token چې د هرې نظر غوښتنې په `Authorization` header کې لېږل کېږي. که ستاسو endpoint تصدیق ته اړتیا نه لري، تش پرېږدئ. |

### تمه کېدونکی JSON بار {#expected-json-payload}

ستاسو د نظرونو endpoint باید د لږ تر لږه لاندې ډګرونو لرونکی JSON بدن ومني:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

د خبرو اترو د شرایطو له مخې ښايي په بار کې نور ډګرونه هم موجود وي.

### د `triage_category` ارزښتونه {#triagecategory-values}

د AI د لومړني ارزونې طبقه د بار له لېږدولو مخکې `triage_category` ته له لاندې ارزښتونو څخه یو ټاکي:

| ارزښت | مانا |
|---|---|
| `factual_error` | مرستیال ناسم واقعي معلومات ورکړي وو. |
| `unhelpful_answer` | ځواب له تخنیکي پلوه سم و خو ګټور نه و. |
| `inappropriate_content` | ځواب داسې منځپانګه لرله چې کارنانو ته باید ونه ښودل شي. |
| `other` | نظر له پېژندل شوې کټګورۍ سره برابر نه و. |

### تصدیق {#authentication}

که ستاسو endpoint تصدیق ته اړتیا لري، د **Feedback API Key** ډګر خپل bearer token ته وټاکئ. agent لېږي:

```
Authorization: Bearer <your-api-key>
```

که د **Feedback API Key** ډګر تش وي، هېڅ `Authorization` header نه لېږل کېږي.

### د نظرونو راټولول نافعالول {#disabling-feedback-collection}

د **نظرونو Endpoint URL** او **Feedback API Key** دواړه ډګرونه تش پرېږدئ. د thumbs-down تڼۍ او د نظرونو UI کارنانو ته ښکاره پاتې کېږي، خو سپارنې هېڅ بهرني خدمت ته نه لېږل کېږي.

## Brave Search API Key {#brave-search-api-key}

همدارنګه د **پرمختللی** ټب کې، د **Brave Search API Key** ډګر د [انټرنېټ لټون](../configuration/internet-search) وړتیا فعالوي.

| ډګر | تشریح |
|---|---|
| **Brave Search API Key** | ستاسو API کیلي د Brave Search پراختیاکوونکي Dashboard څخه. په AI مرستیال کې د انټرنېټ لټون فعالولو لپاره اړینه ده. |

د ډګر نښه د Brave Search API د نوملیکنې پاڼې ته د کېکاږلو وړ تړونی لري. د انټرنېټ لټون نافعالولو لپاره یې تش پرېږدئ.

د دې ځانګړتیا په اړه د وروستي کارن مستنداتو لپاره [انټرنېټ لټون](../configuration/internet-search) وګورئ.

## مدیریت شوی Superdav خدمت {#managed-superdav-service}

Superdav AI Agent v1.18.0 د ملاتړ شوو سایټونو لپاره د مدیریت شوي Superdav خدمت endpoints او د اتومات نښلون چمتو کول زیاتوي. دا کنټرولونه وکاروئ کله چې ستاسو سایټ باید د لاس په جوړ شوي خدمت endpoint پر ځای له کوربه شوي چمتوکوونکي سره ونښلول شي.

| ډګر | تشریح |
|---|---|
| **مدیریت شوی Superdav خدمت** | د ملاتړ شوو سایټونو لپاره د کوربه شوي Superdav خدمت نښلون فعالوي. |
| **نښلون چمتو کول** | د endpoint او اعتبارپاڼو اتومات چمتو کول پیلوي. دا وروسته له هغه وکاروئ چې تایید کړئ سایټ باید مدیریت شوی چمتوکوونکی وکاروي. |
| **د خدمت Endpoint / د نښلون حالت** | له چمتو کولو وروسته اوسنی endpoint یا د نښلون حالت ښيي. |

له چمتو کولو وروسته، تنظیمات خوندي کړئ او د مدیریت شوي خدمت پر کاري بهیرونو تر تکیې مخکې د نښلون حالت تایید کړئ. که چمتو کول ناکام شي، ښکاره شوې د بیا هڅې لارښوونه وڅېړئ او تایید کړئ چې سایټ د کوربه شوي چمتوکوونکي د کارولو اجازه لري.

## د Google Calendar جوړښت {#google-calendar-configuration}

کله چې د Superdav AI Agent v1.18.0 کلیزې ځانګړتیاوې فعالې وي، agent کولای شي جوړې شوې کلیزې او د پېښو جزئیات ولولي. د کلیزې وسیلې د لوستلو پر بنسټ دي او د مهال‌وېش-خبر یادونو، د ګډونوالو تعقیب، او د اړیکو برابرولو لپاره ګټورې دي.

| ډګر | تشریح |
|---|---|
| **د Google Calendar اعتبارپاڼې** | هغه اعتبارپاڼې یا token نښلون ساتي چې د کلیزې د معلوماتو لوستلو لپاره اړین وي. |
| **د کلیزې ټاکنه** | محدودوي چې agent کومې جوړې شوې کلیزې وڅېړي. |
| **د کلیزې د نښلون حالت** | تاییدوي چې ایا اوسنۍ اعتبارپاڼې کلیزې او پېښې لوستلای شي که نه. |

د کلیزې اعتبارپاڼې یوازې هغو کلیزو پورې محدودې وساتئ چې agent ورته اړتیا لري. که حالت د ختم شوي token نښه وکړي، اعتبارپاڼې بیا ونښلوئ یا وګرځوئ.

## د TextBee SMS خبرتیاوې {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 د جوړو شوو خبرتیايي کاري بهیرونو لپاره TextBee د SMS چمتوکوونکي په توګه زیاتوي. SMS خبرتیاوې باید د حساسو یا کارن-مخ پیغامونو لپاره د انسان د منظورۍ دروازو سره یوځای وکارول شي.

| ډګر | تشریح |
|---|---|
| **TextBee API Key** | د TextBee SMS چمتوکوونکي ته غوښتنې تصدیقوي. |
| **TextBee وسیله / لېږونکی** | کله چې چمتوکوونکی یې وغواړي، د وتونکو پیغامونو لپاره کارېدونکی TextBee لېږونکی یا وسیله ټاکي. |
| **SMS خبرتیاوې فعالې دي** | منظور شوو کاري بهیرونو ته اجازه ورکوي چې د متن-پیغام خبرتیاوې ولېږي. د SMS لېږلو د مخنیوي لپاره یې نافعال پرېږدئ. |

ازمایښتي پیغام یوازې د مدیر تر ملکیت لاندې شمېرې ته ولېږئ، بیا د مهالویش شوو یا ګډونوالو-مخ یادونو له فعالولو مخکې د منظورۍ-دروازې چلند تایید کړئ.

## د ځانګړتیاوو نښې {#feature-flags}

همدارنګه په v1.9.0 کې معرفي شوی، د **تنظیمات → د ځانګړتیاوو نښې** ټب د اختیاري فعالیت لپاره toggle سویچونه برابروي. هره نښه یا د ټولې شبکې په کچه فعاله وي یا نافعاله؛ دا مهال د هر سایټ لپاره جلا بدلون نشته.

### د ځانګړتیاوو نښو ته لاسرسی {#accessing-feature-flags}

1. په WordPress مدیر کې، **Gratis AI Agent → تنظیمات** ته لاړ شئ.
2. د **د ځانګړتیاوو نښې** ټب کېکاږئ.

### د لاسرسي کنټرول نښې {#access-control-flags}

| نښه | تلواله | تشریح |
|---|---|---|
| **مدیرانو ته محدودول** | بند | کله چې فعال شي، یوازې هغه کاروونکي چې د `administrator` رول لري د AI Agent چټ پینل پرانیستلی شي. نور ټول رولونه پر ځای یې د "له خپل مدیر سره اړیکه ونیسئ" پیغام ګوري. |
| **د شبکې مدیرانو ته محدودول** | بند | کله چې په یوه څو-سایټه شبکه کې فعال شي، یوازې Super Admins کولی شي agent وکاروي. د جلا سایټونو مدیران بندېږي. که دواړه فعال وي، دا د "مدیرانو ته محدودول" څخه لومړیتوب لري. |
| **د ګډون‌وال لاسرسی اجازه ورکول** | بند | کله چې فعال شي، هغه کاروونکي چې د `subscriber` رول لري د چټ انترفېس کارولی شي خو یوازې د لوستلو وړتیاوو پورې محدود وي (د لیکنو جوړول یا د تنظیماتو بدلون نه شي کولی). |
| **د غیر غړو لپاره غیر فعالول** | بند | د Ultimate Multisite غړیتوب حالت سره یوځای کېږي. کله چې فعال شي، چټ د هغو سایټونو لپاره پټېږي چې فعال غړیتوب نه لري. |

### د نښې‌لګونې نښې {#branding-flags}

| نښه | تلواله | تشریح |
|---|---|---|
| **د "Powered by Gratis AI Agent" پښۍ پټول** | بند | د چټ وېجټ په ښکته برخه کې ښودل کېدونکې د نښې‌لګونې منسوبولو کرښه لرې کوي. د سپین-لېبل ځای‌پرځای کولو لپاره سپارښتنه کېږي. |
| **د agent دودیز نوم** | *(تش)* | د چټ سرلیک او د مدیر مینو کې د تلوالې "Gratis AI Agent" نښه ستاسو د خپل محصول نوم سره بدلوي. د تلوالې کارولو لپاره یې تش پرېږدئ. |
| **د agent ټاکونکی پټول** | بند | کله چې فعال شي، کاروونکي نشي کولی د پنځو جوړو دنننیو agents ترمنځ بدل کړي. اوسنی agent په هغه څه ثابت وي چې په Settings → General کې د تلوالې په توګه تنظیم شوی وي. |
| **د سایټ نښه د چټ اوتار په توګه کارول** | بند | د چټ وېجټ په سرلیک کې تلواله AI نښه د WordPress سایټ نښې سره بدلوي (د Appearance → Customize → Site Identity لاندې ټاکل شوې). |

### د اتومات خوندیتوب نښې {#automation-safety-flags}

Superdav AI Agent v1.18.0 د خوندي اتومات چلونو لپاره د انسان د تایید دروازې او د یادونې ریکارډونه معرفي کوي. دا کنټرولونه ښايي د ځانګړتیاوو په نښو یا پرمختللو اتومات تنظیماتو کې ښکاره شي، د نصب شوي کڅوړې پورې اړه لري.

| نښه | تلواله | تشریح |
|---|---|---|
| **د انسان تایید ته اړتیا** | سپارښتل شوی فعال | حساس اتوماتونه دروي تر څو یو مجاز کاروونکی وړاندیز شوی عمل وڅېړي او تایید یې کړي. |
| **د یادونې د تکرار مخنیوی** | فعال | لېږل شوې یادونې ثبتوي څو بیاځلې هڅې یا مهال‌وېشل شوي چلونه تکراري خبرتیاوې ونه لېږي. |
| **د جنتري وسیلې فعالول** | تر تنظیم پورې بند | agent ته اجازه ورکوي چې تنظیم شوې Google جنتريګانې او پېښې ولولي. |
| **د SMS خبرتیاوې فعالول** | تر تنظیم پورې بند | تایید شوو کاري بهیرونو ته اجازه ورکوي چې د اسنادو له خوندي کېدو وروسته TextBee SMS خبرتیاوې ولېږي. |

### د بدلونونو پلي کول {#applying-changes}

د هرې نښې له بدلولو وروسته **تنظیمات خوندي کړئ** کلیک کړئ. بدلونونه سملاسي اغېز کوي — د cache پاکولو یا plugin بیا فعالولو ته اړتیا نشته.
