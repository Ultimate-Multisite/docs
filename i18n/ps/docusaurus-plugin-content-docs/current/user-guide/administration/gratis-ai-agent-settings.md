# رایگان AI Agent تنظیمات

Gratis AI Agent-এর **Settings → Advanced** স্ক্রিনটিতে v1.5.0 সংস্করণে যুক্ত ব্যাকএন্ড ইন্টিগ্রেশনগুলির জন্য অ্যাডমিনিস্ট্রেটর স্তরের কনফিগারেশন পাওয়া যায়। এই পেজটি **Feedback Endpoint** ফিল্ড এবং তাদের প্রত্যাশিত ফরম্যাট সম্পর্কে নথিভুক্ত করে।

## সেটিংস অ্যাক্সেস করা

১. ওয়ার্ডপ্রেস অ্যাডমিন থেকে, যান **Gratis AI Agent → Settings**-এ।
২. **Advanced** ট্যাবটিতে ক্লিক করুন।

## ফিডব্যাক এন্ডপয়েন্ট কনফিগারেশন

AI এজেন্ট যখন কোনো ব্যবহারকারী থাম্বস-ডাউন বা অটো-প্রম্পট ব্যানার মারফত ফিডব্যাক জমা দেন, অথবা `/report-issue` কমান্ড ব্যবহার করেন, তখন ফিডব্যাক এন্ডপয়েন্ট সেই POST অনুরোধগুলি গ্রহণ করে।

| ফিল্ড | বর্ণনা |
|---|---|
| **Feedback Endpoint URL** | যে URL টি HTTP POST অনুরোধ হিসাবে JSON বডি সহ ফিডব্যাক সাবমিশন গ্রহণ করে। |
| **Feedback API Key** | প্রতিটি ফিডব্যাক অনুরোধের `Authorization` হেডার-এ পাঠানো একটি বেয়ারার টোকেন। আপনার এন্ডপয়েন্ট যদি প্রমাণীকরণের প্রয়োজন না হয় তবে এটি খালি রাখুন। |

### প্রত্যাশিত JSON পেলোড

আপনার ফিডব্যাক এন্ডপয়েন্টকে নিম্নলিখিত ফিল্ডগুলি সহ অন্তত একটি JSON বডি গ্রহণ করতে হবে:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "উত্তরটি মূল্যের বিষয়ে ভুল ছিল।",
  "triage_category": "factual_error"
}
```

কথোপকথনের প্রেক্ষাপট অনুযায়ী পেলোডে অতিরিক্ত ফিল্ড থাকতে পারে।

### `triage_category` মানসমূহ

AI ট্রায়াজ স্তর পেলোড ফরোয়ার করার আগে `triage_category`-এর জন্য নিম্নলিখিত মানগুলির মধ্যে একটি নির্ধারণ করে:

| মান | অর্থ |
|---|---|
| `factual_error` | সহকারীটি ভুল তথ্য প্রদান করেছে। |
| `unhelpful_answer` | প্রতিক্রিয়াটি প্রযুক্তিগতভাবে সঠিক ছিল কিন্তু সহায়ক ছিল না। |
| `inappropriate_content` | প্রতিক্রিয়ায় এমন বিষয়বস্তু ছিল যা ব্যবহারকারীদের দেখানো উচিত নয়। |
| `other` | ফিডব্যাক কোনো পরিচিত ক্যাটাগরির সাথে মিল খুঁজে পায়নি। |

### প্রমাণীকরণ

که اگر دات پوینت شما برای احراز هویت نیاز به اعتبار دارد، فیلد **Feedback API Key** را با توکن برر (bearer token) خود تنظیم کنید. عامل این موارد را ارسال می‌کند:

```
Authorization: Bearer <your-api-key>
```

اگر فیلد **Feedback API Key** خالی باشد، هیچ هدر `Authorization` ارسال نخواهد شد.

### غیرفعال کردن جمع‌آوری بازخورد (Disabling Feedback Collection)

هر دو فیلد **Feedback Endpoint URL** و **Feedback API Key** را خالی بگذارید. دکمه انگشت شست پایین و رابط کاربری بازخورد برای کاربران قابل مشاهده باقی می‌ماند، اما ارسال‌ها به هیچ سرویس خارجی منتقل نخواهند شد.

## کلید API جستجوی برِیو (Brave Search API Key)

هم در تب **Advanced**، فیلد **Brave Search API Key** قابلیت [Internet Search](../configuration/internet-search) را فعال می‌کند.

| فیلد | توضیحات |
|---|---|
| **Brave Search API Key** | کلید API شما از داشبورد توسعه‌دهندگان Brave Search. برای فعال کردن جستجوی اینترنت در دستیار هوش مصنوعی مورد نیاز است. |

برچسب فیلد شامل لینکی قابل کلیک به صفحه ثبت‌نام API Brave Search است. خالی بگذارید تا جستجوی اینترنت غیرفعال شود.

برای مستندات کاربر نهایی در مورد این ویژگی، [Internet Search](../configuration/internet-search) را ببینید.

## پرچم‌های ویژگی (Feature Flags)

این قابلیت که در نسخه v1.9.0 معرفی شد، تب **Settings → Feature Flags** سوئیچ‌هایی برای عملکردهای اختیاری فراهم می‌کند. هر پرچم یا فعال است یا غیرفعال در کل شبکه؛ در حال حاضر هیچ تنظیمات خاصی برای سایت وجود ندارد.

### دسترسی به پرچم‌های ویژگی (Accessing Feature Flags)

۱. در پنل مدیریت وردپرس، به **Gratis AI Agent → Settings** بروید.
۲. روی تب **Feature Flags** کلیک کنید.

### پرچم‌های کنترل دسترسی (Access Control Flags)

| Flag | Default | Description |
|---|---|---|
| **Restrict to Administrators** | Off | کله چې فعال شي، یوازې هغه کارندگانی چې `administrator` نقش имат به بتوانن AI Agent چټ چیٹ پینل खोلن کەن۔ بقیه نقشونه "Contact your administrator" پیغام ڏسندا رهيا ڪندا. |
| **Restrict to Network Admins** | Off | جڏهن هڪ ميٽرسائٽ نڪيت ۾ فعال هجي، تڏهن صرف Super Admins ئي ايجنت استعمال ڪري سگهان ٿا. انفرادي سائٽ اڊمن کي روڪيو ويندو آهي. جيڪڏهن ٻئي ٻنهي فعال هوندا آهن ته "Restrict to Administrators" کان وڌيڪ اهميت رکي ٿو. |
| **Allow Subscriber Access** | Off | کله جيئن هي فعال هجي، `subscriber` نقش وارن صارفين کي چټ انٽر فيس کي استعمال ڪرڻ جا حق حاصل هوندو آهي پر انهن جي صلاحيت صرف پڙهڻ تائين محدود هوندي آهي (پوس کري نه سگهندا يا هيٺيون ترتيئون تبديل نه ڪري سگهندا). |
| **Disable for Non-Members** | Off | Ultimate Multisite جي ممبر شڪل کي جوڙي ٿو. جڏهن هي فعال هجي، ان سائٽن لاءِ چټ چھپائي ويندي آهي جن تان ڪا فعال ممبر شڪل نه آهي. |

### Branding Flags

| Flag | Default | Description |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | چټ وِجٽ جي نچي ڏيکاري ويندي برانڊنگ معلومات کي ختم ڪري ڇڏين ٿا. 화이트 ليبل (white-label) منصوبن لاءِ هي سفارش ڪئي ويندي آهي. |
| **Custom Agent Name** | *(blank)* | چټ هڊر ۽ اڊمن مينيٽ ۾ DEFAULT "Gratis AI Agent" جي جاءِ تي پنهنجي پروڊڪٽ جو نالو رکجو. جيڪڏهن هي خالي رهي، ته DEFAULT استعمال ٿيندو. |
| **Hide Agent Picker** | Off | کله فعال هجي، صارف ايجنتن جي وچ ۾ تبديل نه ڪري سگهندا. موجوده ايجنت کي ان طرح ثابت ڪيو ويندو آهي جهڙوڪ Settings → General ۾ جو default رکيو وڃي. |
| **Use Site Icon as Chat Avatar** | Off | چټ وِجٽ هڊر ۾ DEFAULT AI آئڪن جي جاءِ تي WordPress سائٽ جو آئڪن استعمال ڪريو (Appearance → Customize → Site Identity ۾ set ڪيل). |

### Applying Changes

هر هيٺين فليگ کي تبديل ڪرڻ کان پوء **Save Settings** کلي ڇڏيو. تبديلين فوري اثر ٿينديون هونديون آهن — ڪا به cache صاف ڪرڻ يا plugin کي دوبارہ فعال ڪرڻ جي ضرورت نه آهي.
