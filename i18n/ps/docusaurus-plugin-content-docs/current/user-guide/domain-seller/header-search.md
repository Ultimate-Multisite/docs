# د هېډر د ډومېن لټون

دا تنظیم هغه وخت وکاروئ چې تاسو یوه کوچنۍ هېډر فورمه غواړئ چې د ډومېن لټون پیل کړي، بیا پېرودونکي ته اجازه ورکړي چې د Ultimate Multisite د پېر بشپړولو دننه له شته ډومېنونو څخه انتخاب وکړي.

## اړتیاوې

- Ultimate Multisite په شبکه کې فعال وي.
- Multisite Ultimate Domain Seller په شبکه کې فعال وي.
- لږ تر لږه یو فعال د ډومېن ثبت محصول ولرئ چې:
  - `domain_provider` یو تنظیم شوي چمتو کوونکي ته ټاکل شوی وي.
  - ملاتړ شوي TLDs تنظیم شوي وي، د بېلګې په توګه `com`، `net`، او `org`.
- یوه معتبره د پېر بشپړولو فورمه چې د **ډومېن ټاکنه** ډګر ولري.

## د پېر بشپړولو فورمه

1. هغه د پېر بشپړولو فورمه جوړه یا سمون کړئ چې د ثبت پاڼې له خوا کارول کېږي.
2. عادي اړین د پېر بشپړولو/Account ډګرونه ور اضافه کړئ، د **کارن نوم** په ګډون. هغه د پېر بشپړولو فورمه چې یوازې د ډومېن ډګر ولري، د Ultimate Multisite د اعتبارسنجۍ له خوا ردېږي.
3. د **ډومېن ټاکنه** ډګر ور اضافه کړئ.
4. د ډومېن ټاکنې حالت **یوازې ثبت** ته وټاکئ کله چې بهیر باید د وړیا فرعي ډومېنونو یا موجوده ډومېنونو پر ځای پر ثبت شوو ډومېنونو تمرکز وکړي.
5. د ډومېن ثبت محصول هماغه ډګر ته وټاکئ.

د ثبت پاڼه باید د پېر بشپړولو فورمه ښکاره کړي، د بېلګې په توګه:

```text
[wu_checkout slug="domain-form"]
```

## د هېډر فورمه

د سایټ په هېډر کې یوه کوچنۍ `GET` فورمه ور اضافه کړئ چې داخل شوی د لټون عبارت د پېر بشپړولو پاڼې ته د `domain_name` په توګه ولېږي:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

په دودیز هېډر JavaScript کې ډومېن له مخکې مه ټاکئ. هېډر باید یوازې د لټون عبارت ولېږدوي. د Domain Seller د پېر بشپړولو سکرېپټ `?domain_name=example` لولي، د پېر بشپړولو د لټون بکس ډکوي، او د شتون لټون چلوي څو پېرودونکی له راګرځول شوو ډومېنونو څخه انتخاب وکړي.

## تمه کېدونکی چلند

په هېډر کې د `example` لټون باید دا پرانیزي:

```text
/register/?domain_name=example
```

بیا باید د پېر بشپړول د انتخاب وړ پایلې وښيي، لکه:

- `example.com`
- `example.net`
- `example.org` نشته
- نور د چمتو کوونکي له خوا ملاتړ شوي TLDs

له شته پایلې د انتخاب وروسته، د فرمایش لنډیز باید د ډومېن ثبت محصول او ټاکل شوی ډومېن نوم ولري.

## تایید

1. کورپاڼه پرانیزئ.
2. یو ساده نوم ولټوئ، د بېلګې په توګه `example`.
3. تایید کړئ چې د پېر بشپړولو URL کې `?domain_name=example` شامل دی.
4. تایید کړئ چې د پېر بشپړولو د ډومېن ډګر `example` لري.
5. تایید کړئ چې د ډومېن انتخابونو یو لېست ښکاري.
6. د یوه شته ډومېن لپاره **ټاکل** کلیک کړئ.
7. تایید کړئ چې د فرمایش لنډیز کې `Domain Registration - example.com` یا ټاکل شوی ډومېن شته.

## د ستونزو حل

- که هېڅ لېست نه ښکاري، د `admin-ajax.php?action=wu_domain_search` لپاره د براوزر د شبکې ټب وګورئ او تایید کړئ چې تش نه وي `domains` یا `results` راګرځوي.
- که د پېر بشپړولو فورمه د خوندي کولو پر مهال په اعتبارسنجۍ کې ناکامه شي، اړین Account ډګرونه ور اضافه کړئ، لکه **کارن نوم**.
- که د ډومېن ټاکل cart نه تازه کوي، تایید کړئ چې `window.wu_checkout_form` شته او د Domain Seller د پېر بشپړولو شتمنۍ د پېر بشپړولو په پاڼه کې پورته شوې دي.
