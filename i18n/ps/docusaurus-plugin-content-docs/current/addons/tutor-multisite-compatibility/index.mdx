# Tutor Multisite Compatibility

یو WordPress پلګین چې په څو-سایټه چاپېریالونو کې د Tutor LMS لپاره د سازګارۍ سمونې زیاتوي.

## تشریح {#description}

دا پلګین د WordPress څو-سایټه شبکو کې د Tutor LMS د چلولو لپاره اړینې د سازګارۍ سمونې برابروي. دا د کارونکي نوم‌لیکنه، د ښوونکي مدیریت، د کورس محدودیتونه، او نور څو-سایټه ځانګړي فعالیتونه اداره کوي.

## ځانګړتیاوې {#features}

- **د ډیټابېس جدول مدیریت**: په اتومات ډول هغه ورک Tutor LMS جدولونه پېژني او جوړوي چې د څو-سایټه بهرني کلید محدودیتونو له امله ناکامېږي
- **د څو-سایټه کارونکي نوم‌لیکنه**: د شبکې د سایټونو په اوږدو کې د کارونکي نوم‌لیکنې د مدیریت لپاره دودیز AJAX سمبالوونکي
- **د ښوونکي رول مدیریت**: په څو-سایټه چاپېریالونو کې د ښوونکي رول اتومات ټاکنه او مدیریت
- **د کورس محدودیتونه**: د سایټ د پلانونو پر بنسټ د کورس او فنل جوړولو محدودیتونو د مدیریت لپاره له Ultimate Multisite سره ادغام
- **دودیزې اډمین لیدنې**: د څو-سایټه پوه ښوونکي لېست او د مدیریت انترفېس
- **اډمین وسیلې**: د شبکې او سایټ مدیرانو لپاره د ډیټابېس مدیریت انترفېس

## اړتیاوې {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Tutor LMS پلګین (په اتومات ډول د `Requires Plugins` سرلیک له لارې پلي کېږي)
- د WordPress څو-سایټه نصب
- Ultimate Multisite پلګین (د کورس محدودیتونو د فعالیت لپاره)

## نصب {#installation}

1. د پلګین فایلونه `/wp-content/plugins/tutor-multisite-compatibility/` ته پورته کړئ
2. ډاډ ترلاسه کړئ چې Tutor LMS نصب او فعال شوی دی
3. پلګین په WordPress کې د 'Plugins' مینو له لارې فعال کړئ
4. پلګین به په اتومات ډول له Tutor LMS سره ادغام شي

## هغه فعالیت چې له MU plugin څخه لېږدول شوی {#functionality-moved-from-mu-plugin}

دا پلګین هغه فعالیت لري چې پخوا په `wp-content/mu-plugins/daves.php` کې و:

- د کورس د حالت بدلون محدودیتونه
- د کورس جوړولو محدودیتونه
- د CartFlows ادغام او محدودیتونه
- د Tutor ښوونکي رول مدیریت
- د څو-سایټه کارونکي نوم‌لیکنې اداره کول
- د کورس د محصول پېژند display

## د ډیټابېس مدیریت {#database-management}

### ستونزه {#the-problem}

د WordPress څو-سایټه چاپېریالونو کې، د Tutor LMS جدول جوړول د هغو جدولونو لپاره ناکامېږي چې د کاروونکو جدول ته د بهرني کلید محدودیتونه لري. دا ځکه پېښېږي چې:

- د سایټ ځانګړي جدولونه د `wp_2_tutor_carts` په څېر مختاړي کاروي
- بهرني کلیدونه `wp_2_users` ته اشاره کوي (چې شتون نه لري)
- اصلي د کاروونکو جدول `wp_users` دی (چې په ټوله شبکه کې شریک دی)

### حل {#the-solution}

دا پلګین په اتومات ډول:

1. **ورک جدولونه پېژني**: هغه جدولونه ګوري چې د بهرني کلید محدودیتونو له امله جوړ نه شول
2. **جدولونه له سمو اشارو سره جوړوي**: د کارونکي د جدول سم نوم کاروي (په څو-سایټه کې `wp_users`)
3. **څو سناریوګانې اداره کوي**:
   - د پلګین فعالول/بیا فعالول
   - په شبکه کې د نوي سایټ جوړول
   - د اډمین انترفېس له لارې لاسي جدول کتنه

### اغېزمن شوي جدولونه {#affected-tables}

- `tutor_coupon_usages` - د کاروونکو جدول ته بهرنی کلید لري
- `tutor_carts` - د کاروونکو جدول ته بهرنی کلید لري
- `tutor_notification_preferences` - د TutorPro جدول چې د کاروونکو جدول ته بهرنی کلید لري

### اډمین انترفېس {#admin-interface}

د ډیټابېس مدیریت وسیلو ته دلته لاسرسی ومومئ:

- **د شبکې اډمین**: Settings → Tutor Multisite DB
- **د سایټ اډمین**: Tools → Tutor Multisite DB

انترفېس تاسو ته اجازه درکوي چې:
- په ټولو سایټونو کې ورک جدولونه وګورئ
- په لاسي ډول د جدول جوړول پیل کړئ
- د ډیټابېس تشکیلات وګورئ
- تفصیلي حالت معلومات وګورئ
