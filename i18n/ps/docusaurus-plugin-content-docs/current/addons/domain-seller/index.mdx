# Domain Seller

**Multisite Ultimate: Domain Seller** ستاسو د WordPress Multisite شبکه په یوه پیاوړي د ډومېن پلورلو پلېټفارم بدلوي. پېرېدونکو ته اجازه ورکړئ چې د نوم‌لیکنې پر مهال یا له خپل dashboard څخه دودیز ډومېنونه مستقیم ثبت کړي، د ریښتیني وخت د شتون کتنې او روڼو بیو سره.

### ولې Domain Seller؟

د ډومېن ثبت د هر WaaS (Website as a Service) پلېټفارم لپاره له تر ټولو ډېرو غوښتل کېدونکو اضافي پلورونو څخه دی. د دې پر ځای چې خپل پېرېدونکي د درېیم لوري راجستر کوونکي ته ولېږئ، تاسو کولی شئ ډومېنونه مستقیم وړاندې کړئ — په خپله شبکه کې ډېر عاید ساتئ او پېرېدونکو ته بې‌خنډه تجربه ورکوئ.

### مهمې ځانګړتیاوې

#### د څو ډومېن راجستر کوونکو ملاتړ
* **OpenSRS** — بشپړ ثبت، نوي کول، او مدیریت
* **Namecheap** — له WhoisGuard او sandbox ازموینې سره بشپړ ادغام
* **HostAfrica** — د Domains Reseller API له لارې د ډومېن لټون، بیې، ثبت، لېږد، DNS، او د پېژندنې ساتنه
* **Openprovider** — د بیاپلورونکي بیې، ثبت، نوي کول، لېږدونه، DNS، WHOIS محرمیت، او TLD همغږي
* **Hostinger** — د ګډ Hostinger ادغام token په کارولو د ډومېن لټون او ثبت
* **غځېدونکې جوړښت** — د Integration Registry بڼې پر بنسټ جوړ شوی، چې د نورو چمتو کوونکو زیاتول اسانه کوي
* **دننه د نښلون ازموینه** — د ژوندۍ کولو مخکې د چمتو کوونکي اسناد او نښلون تایید کړئ

#### لارښود Setup Wizard
* **په یوه کلیک پیلول** — تاسو د چمتو کوونکي تنظیم، TLD واردولو، او د ډومېن محصول جوړولو له پړاوونو تېروي
* **اتومات TLD همغږي** — له خپلو نښلول شویو چمتو کوونکو څخه ټول موجود TLDs او عمده بیې واردوي
* **د ډیفالټ محصول جوړول** — د هوښیارو markup ډیفالټونو سره د کارولو لپاره چمتو ډومېن محصول جوړوي

#### انعطاف‌منې د ډومېن بیې
* **درې markup حالتونه** — سلنه markup، ثابت markup، یا د هر TLD لپاره ثابته بیه
* **د TLD ځانګړي محصولات** — د بېلابېلو TLD ډلو لپاره جلا ډومېن محصولات جوړ کړئ (.com/.net، .io، د هېواد کوډونه، او نور)
* **پېژندنیزې بیې** — د نوو ثبتونو د جذب لپاره د لومړي کال تخفیف شوې بیه وړاندې کړئ
* **د ریښتیني وخت بیې** — له چمتو کوونکو څخه ژوندۍ عمده بیې د هوښیار caching سره
* **د coupon ملاتړ** — د Multisite Ultimate تخفیف کوډونو سره بشپړه سازګاري

#### د WHOIS محرمیت ساتنه
* **د هر محصول تنظیم** — WHOIS محرمیت Disabled، Always Included، یا Customer Choice ته وټاکئ
* **د پېرېدونکي opt-in** — کله چې Customer Choice ته ټاکل شوی وي، د checkout یو checkbox پېرېدونکو ته اجازه ورکوي چې د محرمیت ساتنه زیاته کړي
* **تنظیمېدونکې بیې** — د محرمیت add-on لپاره د هر کال بیه وټاکئ، یا یې وړیا وړاندې کړئ
* **د راجستر کوونکي ادغام** — د ثبت پر وخت په اتومات ډول WhoisGuard (Namecheap) یا WHOIS privacy (OpenSRS) فعالوي

#### یو موټی Checkout تجربه
* **درې ډومېن حالتونه** — پېرېدونکي د یوه subdomain (وړیا)، د نوي ډومېن ثبتولو، یا د داسې موجود ډومېن mapping ترمنځ انتخاب کوي چې لا له مخکې یې مالکیت لري
* **د ریښتیني وخت ډومېن لټون** — د AJAX په وسیله د شتون کتنه، بې له پاڼې بیاپورته کولو
* **هوښیار وړاندیزونه** — کله چې خوښ شوی ډومېن نیول شوی وي، بدیل TLD وړاندیزونه
* **روڼې بیې** — د ثبت بیې، د نوي کولو بیې، او اختیاري WHOIS محرمیت فیس روښانه ښودنه
* **د ثبت کوونکي د اړیکې ساحې** — د checkout پر مهال اړین د ثبت کوونکي معلومات (نوم، پته، ټیلیفون) په هماغه ځای کې راټولوي
* **اتومات جوړ شوی د site URL** — د site پته په اتومات ډول له ټاکل شوي ډومېن څخه اخیستل کېږي

#### د اتومات نوي کولو سیستم
* **له غړیتوب سره تړلي نوي کول** — ډومېنونه په اتومات ډول نوي کېږي تر څو د پېرېدونکي غړیتوب فعال وي
* **هوښیار مهال‌وېش** — د نوي کولو دندې د ډومېن د پای نېټو پر بنسټ کتار کېږي
* **د بیا هڅې منطق** — د ناکامو نوي کولو هڅو لپاره هوښیارې بیا هڅې
* **برېښنالیک خبرتیاوې** — د راتلونکو نوي کولو، بریالیو نوي کولو، ناکامیو، او پای ته رسېدو لپاره تنظیمېدونکي خبرتیاوې

#### د پېرېدونکي DNS مدیریت
* **ځان‌خدمت DNS** — پېرېدونکي کولی شي له خپل dashboard څخه DNS ریکارډونه (A, AAAA, CNAME, MX, TXT) زیات، سم، او حذف کړي
* **د ډومېن حالت** — د ریښتیني وخت د ثبت حالت، د پای نېټې، او د nameserver معلومات
* **اتومات ډومېن mapping** — ثبت شوي یا موجود ډومېنونه په اتومات ډول د پېرېدونکي site ته map کېږي

#### د Admin وسیلې
* **لاسي د ډومېن ثبت** — له admin panel څخه د پېرېدونکو په استازیتوب ډومېنونه ثبت کړئ
* **د ډومېن څارنه** — د ټولو پېرېدونکو ډومېنونه، حالتونه، او د پای نېټې له یوې پاڼې وګورئ
* **د چمتو کوونکي ځانګړي logs** — د هر راجستر کوونکي لپاره جلا log channels (لکه، domain-seller-namecheap.log)
* **د ثبت جزئیات** — د اصلي ډومېن د سمون پاڼې کې د ثبت کوونکي معلومات او د ثبت ډاټا وګورئ

### ملاتړ شوي چمتو کوونکي

* **OpenSRS** — له ثبت، نوي کولو، DNS مدیریت، او WHOIS محرمیت سره بشپړ ادغام
* **Namecheap** — له ثبت، نوي کولو، batch د شتون کتنې، او وړیا WhoisGuard سره بشپړ ادغام
* **HostAfrica** — ثبت، نوي کول، لېږدونه، DNS مدیریت، د EPP کوډ ترلاسه کول، registrar lock، او د پېژندنې ساتنه
* **Openprovider** — ثبت، نوي کول، لېږدونه، DNS مدیریت، د EPP کوډ ترلاسه کول، registrar lock، WHOIS محرمیت، او د بیاپلورونکي TLD همغږي
* **Hostinger** — ثبت، د ډومېن لټون، nameserver تازه کول، registrar lock، او WHOIS محرمیت؛ نوي کول د Hostinger auto-renew له خوا ترسره کېږي
* **GoDaddy** — ثبت او نوي کول
* **ResellerClub** — ثبت، نوي کول، او DNS مدیریت
* **NameSilo** — ثبت او نوي کول
* **Enom** — ثبت، نوي کول، او DNS مدیریت

### تخنیکي جزئیات

* **اړتیا لري:** PHP 7.4+, Multisite Ultimate 2.4.12+
* **جوړښت:** د Integration Registry بڼه د Domain_Selling_Capability interface سره
* **Autoloading:** Jetpack autoloader د PSR-4 class mapping سره
* **معیارونه:** د WordPress کوډ لیکلو معیارونه، strict typing، prepared statements

## نصب کول

1. ډاډ ترلاسه کړئ چې **Multisite Ultimate v2.4.12+** نصب او په شبکه کې فعال شوی دی
2. plugin دې ځای ته پورته کړئ: `/wp-content/plugins/ultimate-multisite-domain-seller/`
3. plugin د WordPress د شبکې د ادارې له **Plugins** څخه په شبکه کې فعال کړئ
4. **د تنظیم وزرډ** په لومړي فعالولو کې په اوتومات ډول پیلېږي — ګامونه تعقیب کړئ تر څو:
   * خپل د ډومېن ثبتوونکی ونښلوئ
   * اړیکه وازمویئ
   * شته TLDs او عمده بیې واردې کړئ
   * خپل لومړی د ډومېن محصول د بیې له زیاتوالي سره جوړ کړئ
5. د **ډومېن ټاکنه** ساحه خپل د تادیې فورم ته په **Ultimate Multisite > د تادیې فورمونه** کې ورزیاته کړئ

## ډېرې پوښتل شوې پوښتنې

### د Multisite Ultimate کومه نسخه اړینه ده؟

نسخه 2.4.12 یا تر هغې لوړه. addon له WP Ultimo v1.x یا د Ultimate Multisite له 2.4.12 څخه ټیټو نسخو سره سازګار نه دی.

### کوم د ډومېن چمتوکوونکي ملاتړ کېږي؟

نهه ثبتوونکي ملاتړ کېږي: **OpenSRS**، **Namecheap**، **HostAfrica**، **Openprovider**، **Hostinger**، **GoDaddy**، **ResellerClub**، **NameSilo**، او **Enom**. وړتیاوې د چمتوکوونکي له مخې توپیر لري، ځکه د ثبتوونکو API بېلابېل عملیات وړاندې کوي؛ د هر چمتوکوونکي د لنډیز لپاره د تنظیم لارښود وګورئ. د addon د Integration Registry جوړښت د ځانګړو چمتوکوونکو زیاتول ساده کوي — [د پراختیاکوونکي لارښود](/developer/integration-guide/custom-registrar) وګورئ.

### ایا زه د WHOIS د محرمیت ساتنه وړاندې کولی شم؟

هو. هر د ډومېن محصول د WHOIS Privacy تنظیم لري چې درې حالتونه لري: غیرفعال، تل شامل، یا د پېرودونکي انتخاب. کله چې د پېرودونکي انتخاب ته ټاکل شوی وي، پېرودونکي د تادیې پر مهال د بیې سره یو چک‌بکس ویني (یا د Namecheap د WhoisGuard لپاره "وړیا"). د محرمیت نښه د ثبت پر وخت په اوتومات ډول ثبتوونکي ته لېږل کېږي.

### ایا زه د بېلابېلو TLDs لپاره بېلابېلې بیې ټاکلی شم؟

هو. د بېلابېلو TLD ډلو لپاره جلا د ډومېن محصولات جوړ کړئ. هر محصول کولی شي خپل د زیاتوالي ډول (سلنه، ثابته زیاتونه، یا ثابته بیه)، ملاتړ شوي TLDs، او پېژندنیزه بیه ولري.

### پېرودونکي د تادیې پر مهال کوم انتخابونه ویني؟

د ډومېن ټاکنې ساحه پېرودونکو ته درې ټبونه ورکوي: **فرعي ډومېن** (ستاسو پر شبکه وړیا فرعي ډومېن کارول)، **نوی ډومېن ثبتول** (نوی ډومېن لټول او ثبتول)، او **شته ډومېن** (هغه ډومېن نښلول چې دوی یې له مخکې لري). یوازې هغه حالتونه چې تاسو فعال کړي وي ښکاره کېږي.

### ایا دا د اوتومات تمدیدونو ملاتړ کوي؟

هو. ډومېنونه د اوتومات تمدید لپاره ټاکل کېدای شي او د پېرودونکي د غړیتوب له حالت سره تړلي وي. د تمدید پیسې د پېرودونکي د موجوده تادیې دروازې له لارې پروسس کېږي.

### ایا پېرودونکي خپل DNS ریکارډونه اداره کولی شي؟

هو. پېرودونکي په خپل Dashboard کې د DNS د مدیریت انٹرفېس ترلاسه کوي، څو A، AAAA، CNAME، MX، او TXT ریکارډونه زیات، سم، او حذف کړي. د DNS مدیریت له OpenSRS، ResellerClub، Enom، HostAfrica، او Openprovider سره شتون لري. د کوربه شوو ډومېنونو لپاره د Hostinger DNS د اصلي Hostinger د ډومېن-نښلولو ادغام له خوا اداره کېږي.

### ایا د تنظیم وزرډ شته؟

هو. د تنظیم وزرډ په لومړي فعالولو کې په اوتومات ډول چلېږي او تاسو د ثبتوونکي د نښلولو، د TLDs د واردولو، او ستاسو د لومړي ډومېن محصول د جوړولو له لارې رهبري کوي. تاسو کولی شئ وزرډ د addon له تنظیماتو څخه بیا هم وچلوئ.
