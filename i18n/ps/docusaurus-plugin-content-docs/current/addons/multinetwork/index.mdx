# Multi-Network {#multi-network}

د Ultimate Multisite: Multinetwork addon ستاسو Ultimate Multisite پلاتفورم د ځواکمن multinetwork فعالیت سره غځوي. دا addon ستاسو پېرېدونکو ته دا وړتیا ورکوي چې د WordPress بشپړې multisite شبکې وپېري او مدیریت یې کړي، نه یوازې انفرادي سایټونه.

## مهمې ځانګړتیاوې {#key-features}

* **د شبکې برابرول**: ستاسو د پېرېدونکو لپاره په اتومات ډول بشپړې WordPress multisite شبکې جوړې او تنظیموي
* **د شبکې ټېمپلټونه**: پېرېدونکو ته اجازه ورکړئ چې د نوم‌لیکنې مخکې چمتو شوی د شبکې ټېمپلټ وپلټي، مخکتنه یې وکړي، او غوره یې کړي
* **د شبکې مدیریت**: د multinetwork چاپېریالونو لپاره بشپړې مدیریتي وړتیاوې
* **جلا شبکې**: د هر پېرېدونکي شبکه په خپلواکه توګه د خپلو ډیټابېس جدولونو او تنظیماتو سره کار کوي
* **د شبکو ترمنځ SSO**: پېرېدونکو ته د magic-link تصدیق له لارې خپلو فرعي شبکو ته بې‌درنګه لاسرسی ورکړئ
* **د پراخېدو وړ جوړښت**: د غوره شوې سرچینو وېش سره څو شبکې په اغېزمن ډول اداره کړئ
* **ادغام**: د Ultimate Multisite د ګډون او بېلینګ سیسټمونو سره بې‌درنګه یوځای کېږي

## د کارونې موارد {#use-cases}

* **سپین‌لېبل کوربه‌توب**: پېرېدونکو ته د هغوی خپل برانډ لرونکې multisite شبکې برابرې کړئ
* **د اژانس حل‌لارې**: اژانسونو ته اجازه ورکړئ چې د یو واحد پلاتفورم څخه د څو مراجعینو شبکې مدیریت کړي
* **د تصدۍ پلي‌کونې**: له پېچلو څو-شبکوي اړتیاوو لرونکو لویو سازمانونو ملاتړ وکړئ
* **تعلیمي پلاتفورمونه**: د بېلابېلو ښوونځیو یا څانګو لپاره جلا شبکې جوړې کړئ

## اړتیاوې {#requirements}

* Ultimate Multisite 2.0.0 یا لوړه نسخه
* د WordPress Multisite نصب
* PHP 7.4 یا لوړه نسخه
* MySQL 5.6 یا لوړه نسخه

## نصب {#installation}

1. د plugin فایلونه د `/wp-content/plugins/ultimate-multisite-multinetwork` ډایرکټرۍ ته پورته کړئ
2. plugin په WordPress کې د 'Plugins' سکرین له لارې فعال کړئ
3. د Ultimate Multisite → Settings له لارې د multinetwork تنظیمات برابر کړئ

## د شبکې ټېمپلټونه {#network-templates}

نسخه 1.3.0 د هغو پېرېدونکو لپاره چې نوې شبکې پېري، د ټېمپلټ موندنې او ټاکنې بهیر زیاتوي. د شبکې ټېمپلټونه هغه وخت وکاروئ چې غواړئ د هر نوي پېرېدونکي شبکه له چمتو جوړښت، ډیزاین، د منځپانګې کتابتون، یا د ځانګړي سکتور تنظیم څخه پیل شي.

### د ټېمپلټ مخکتونکی {#template-previewer}

د شبکې د ټېمپلټ مخکتونکی پېرېدونکو ته اجازه ورکوي چې له checkout مخکې شته ټېمپلټونه وپلټي. هر ټېمپلټ په ژوندۍ مخکتنې پینل کې پرانیستل کېدای شي، څو پېرېدونکي د شبکې تر پېرلو مخکې بڼې او منځپانګه پرتله کړي.

مخکتونکی وکاروئ څو پېرېدونکو سره مرسته وکړئ:

* د هر شته شبکې ټېمپلټ بڼه او جوړښت وڅېړي
* څو ټېمپلټونه د پېرود بهیر له پرېښودو پرته پرتله کړي
* هغه ټېمپلټ غوره کړي چې د نوم‌لیکنې مخکې د دوی د کارونې مورد سره تر ټولو ښه سمون لري

د ټېمپلټ نومونه او تشریحات د پېرېدونکي لپاره اسانه وساتئ. مخکتونکی هغه وخت ډېر ګټور وي چې پېرېدونکي وکولای شي موخه شوې کارونه درک کړي، لکه د اژانس شبکې، د ښوونځي شبکې، د فرنچایز شبکې، یا د سپین‌لېبل کوربه‌توب بنډلونه.

### په checkout کې د ټېمپلټ ټاکونکی {#template-picker-at-checkout}

د ټېمپلټ ټاکونکی د نوم‌لیکنې checkout بهیر ته د یو ډګر په توګه زیاتېدای شي. کله چې تنظیم شي، پېرېدونکي د checkout پر مهال هغه د شبکې ټېمپلټ ټاکي چې غواړي، او ټاکل شوی ټېمپلټ کارول کېږي کله چې د دوی نوې شبکه برابره شي.

په checkout بهیر کې د ټاکونکي کارولو لپاره:

1. هغه شبکې چمتو کړئ چې غواړئ د ټېمپلټونو په توګه یې وړاندې کړئ.
2. په اړوند نوم‌لیکنه یا checkout فورمه کې د Multinetwork ټېمپلټ ټاکنې ډګر فعال کړئ.
3. وټاکئ چې کوم ټېمپلټونه باید د هغه checkout بهیر لپاره شته وي.
4. د checkout بهیر خپور کړئ او یوه نوم‌لیکنه وازمویئ څو تایید شي چې ټاکل شوی ټېمپلټ نوې شبکې ته پلي کېږي.

که د checkout بهیر یوازې یو د شبکې محصول وړاندې کوي، ټاکونکی بیا هم د لارښوونکې onboarding مرحلې په توګه ګټور کېدای شي. د څو-پلان funnelونو لپاره، یوازې هغه ټېمپلټونه ښکاره کړئ چې له ټاکل شوي محصول یا پلان سره سازګار وي.

## د شبکو ترمنځ Magic Link SSO {#cross-network-magic-link-sso}

د شبکو ترمنځ magic link SSO پېرېدونکو ته اجازه ورکوي چې د خپل لومړني Account سیمې او فرعي شبکو ترمنځ پرته له دې چې بیا په لاس ننوزي، حرکت وکړي. کله چې پېرېدونکی د شبکو ترمنځ د لاسرسي لینک تعقیبوي، Ultimate Multisite یو د وخت محدود magic link جوړوي او پېرېدونکی مقصد شبکې ته تصدیقوي.

دا د هغو پېرېدونکو لپاره تجربه ښه کوي چې له یوې څخه زیاتې شبکې مدیریتوي یا اړتیا لري چې د پېرود سمدستي وروسته نویو کلون شویو ټېمپلټ شبکو ته لاسرسی ومومي.

د Admin چک‌لېست:

* تایید کړئ چې Multinetwork addon د WordPress multisite نصب په اوږدو کې فعال دی.
* ډاډ ترلاسه کړئ چې پېرېدونکي د شبکو ترمنځ لینکونو له وړاندې کېدو مخکې د شبکې سم لاسرسی لري.
* د شبکې ډومېنونه او mapped ډومېنونه سم تنظیم وساتئ څو magic links موخې مقصد شبکې ته حل شي.
* د شبکې، ډومېن، یا checkout تنظیماتو تر بدلولو وروسته د پېرېدونکي dashboard او هر د checkout وروسته د لاسرسي تڼۍ وازمویئ.

Magic links باید د ننوتلو لینکونو په توګه وبلل شي. دا د پېرېدونکي د بې‌درنګه لاسرسي لپاره ډیزاین شوي او باید له موخه شوې ناستې بهر ونه لېږل شي یا بیا ونه کارول شي.

## ډېری پوښتل شوې پوښتنې {#frequently-asked-questions}

### multinetwork څه دی؟ {#what-is-multinetwork}

Multinetwork د WordPress یوه ځانګړتیا ده چې تاسو ته اجازه درکوي په یوه واحد WordPress نصب کې څو خپلواکې multisite شبکې وچلوئ.

### دا له عادي multisite څخه څنګه توپیر لري؟ {#how-is-this-different-from-regular-multisite}

عادي multisite په یوه شبکه کې څو سایټونو ته اجازه ورکوي. Multinetwork څو شبکو ته اجازه ورکوي، چې هره یوه خپلې سایټونه لري، ټول په یوه WordPress نصب کې.

### ایا دا ځانګړي سرور تنظیم ته اړتیا لري؟ {#does-this-require-special-server-configuration}

هو، multinetwork ځانګړي سرور جوړښت او تنظیم ته اړتیا لري. مهرباني وکړئ د تفصیلي جوړښت لارښوونو لپاره مستندات وګورئ.
