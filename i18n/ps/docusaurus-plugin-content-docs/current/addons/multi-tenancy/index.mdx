# څو-کرایه‌والي

د کړنې او امنیت لپاره فرعي سایټونه د سرورونو او ډیټابېسونو ترمنځ ووېشئ

## د نسخې 1.2.0 د خپرېدو مهم ټکي {#version-120-release-highlights}

Multi-Tenancy 1.2.0 د تولید د جلاوالي کاري بهیرونو لپاره د خپلواک کرایه‌دار بنسټ زیاتوي:

- د لنډمهاله، د بیاکارونې-خوندي SSO کتنو سره بې‌حالته کرایه‌دار خپلکار ننوتل.
- د خپلواکو کرایه‌دارانو لپاره د هر فرعي سایټ ډیټابېس او د فایل‌سیسټم جلاوالی.
- د کوربه-خبر کرایه‌دار برابرولو لپاره د Hostinger څو-کرایه‌والي وړتیا.
- د پخوانیو معلوماتو د کتنو او async push پروسس لپاره د خپلواک لېږد د تایید دروازې.
- د Bedrock، FrankenPHP، او localhost نصبونو لپاره د همدې ماشین د ډیټابېس کوربه تړنه چې تنظیمېدلی شي.
- د کرایه‌دار bootstrap، د ډیټابېس لیکونکی، schema bootstrap، او د کارن برابرولو ښه‌والي.
- خوندي‌تر خپلواک teardown چې د کرایه‌دار اسناد پاکوي.

د [خپلواک لېږد کاري بهیر](/user-guide/getting-started/sovereign-migration-workflow)، [څو-کرایه‌والي جلاوالی](/user-guide/configuration/multi-tenancy-isolation)، او [د خپلواک کرایه‌دار عملیات](/user-guide/administration/sovereign-tenant-operations) لپاره د کارن لارښودونه وګورئ.

## اړتیاوې {#requirements}

- WordPress 5.3 یا تر هغه لوړه
- PHP 7.4 یا تر هغه لوړه
- Multisite Ultimate پلگین (فعال)

## نصب {#installation}

1. د اضافې فایلونه خپل `/wp-content/plugins/` ډایرکټرۍ ته پورته کړئ
2. پلگین په WordPress کې د 'پلگینونه' غورنۍ له لارې فعال کړئ
3. د شبکې د مدیر په برخه کې د اضافې تنظیمات عیار کړئ

## پراختیا {#development}

### تنظیم {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### ازموینه {#testing}

اضافه یو بشپړ ازموینې چوکاټ لري:

- **د واحد ازموینې**: د WordPress ازموینې ټولګې سره د PHPUnit پر بنسټ ازموینې
- **بنسټیز ازموینې ټولګی**: د عامو ازموینې سناریوګانو لپاره مرستندویه میتودونه برابروي
- **د ازموینې پوښښ**: د کوډ پوښښ د تعقیب لپاره تنظیم شوی
- **Multisite ازموینه**: ازموینې په multisite چاپېریال کې چلېږي

### د کوډ معیارونه {#code-standards}

- **د WordPress کوډ لیکلو معیارونه**: د PHPCS له لارې پلي کېږي
- **PHP 7.4+ سازګاري**: د PHP عصري ځانګړنې ملاتړ کېږي
- **ثابت تحلیل**: د ډول کتنې لپاره د PHPStan ادغام
- **د کوډ عصري کول**: د PHP لوړولو لپاره د Rector اصول

### د جوړولو سیستم {#build-system}

- **د شتمنیو پروسس**: د CSS/JS فایلونو لپاره کوچنی کول
- **ژباړه**: د POT فایل جوړول
- **بسته بندي**: د وېش لپاره د ارشیف جوړول
- **د پراختیا حالت**: د اشکال موندنې لپاره ناپرې‌کوچني شوې شتمنۍ

## تنظیمات {#configuration}

اضافه د شبکې د مدیر انٹرفېس له لارې لاسرسی‌لرونکي بېلابېل تنظیماتي اختیارونه ملاتړ کوي.

### د خپلواک کرایه‌دار تنظیمات {#sovereign-tenant-configuration}

د خپلواکو کرایه‌دارانو لپاره، د کرایه‌دار ډیټابېس، د فایل‌سیسټم ریښه، د ډیټابېس کوربه تړنه، او د کوربه وړتیا مخکې له دې تنظیم کړئ چې ترافیک کرایه‌دار ته مسیر کړئ. له هر هغه بدلون وروسته د لېږد تایید وچلوئ چې د ډیټابېس اسناد، کوربه تړنه، مسیرونه، یا د کارن برابرول اغېزمنوي.

`localhost` یوازې هغه وخت وکاروئ چې د کرایه‌دار runtime او د ډیټابېس اجازې د محلي socket یا همدې ماشین لاسرسی تمه لري. `127.0.0.1` یا د خدمت کوربه نوم هغه وخت وکاروئ چې هماغه د اجازې کوربه وي چې ستاسو برابروونکي تنظیم کړی وي.

### د Hostinger وړتیا {#hostinger-capability}

د Hostinger څو-کرایه‌والي وړتیا د شریک Hostinger ادغام token سره کار کوي. تایید کړئ چې ډومېن په hPanel کې شته، DNS د Hostinger Account ته اشاره کوي، او د کرایه‌دار ډیټابېس تنظیمات له hPanel سره د تایید مخکې سمون لري.

## Hooks او Filters {#hooks-and-filters}

### کړنې {#actions}

- `ultimate-multisite-addon-template_init` - د اضافې له پیل وروسته چلېږي
- `ultimate-multisite-addon-template_loaded` - وروسته له هغې چلېږي چې د اضافې ټول فایلونه بار شي

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - د اضافې تنظیمات بدل کړئ
- `ultimate-multisite-addon-template_enabled` - د اضافې د فعال/غیرفعال حالت override کړئ
