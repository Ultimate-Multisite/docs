# د سازګارو پای‌ټکو لپاره Ultimate AI نښلوونکی

د سازګارو پای‌ټکو لپاره Ultimate AI نښلوونکی د OpenAI-سازګار AI برابروونکو API ګانې ستاسو د WordPress multisite شبکې سره یوځای کوي. څو AI برابروونکي تنظیم کړئ — هر یو له خپل پای‌ټکي URL، API کیلي، او تلوالیز ماډل سره — او غوښتنې د اتومات بدیل لارې له لارې ولېږدوئ کله چې یو برابروونکی شتون ونه لري.

## مهمې ځانګړتیاوې

- **د څو برابروونکو ملاتړ**: څو AI برابروونکي کارتونه ورزیات کړئ، هر یو له خپل پای‌ټکي URL، API کیلي، او تلوالیز ماډل سره
- **د بیا-ترتیب لپاره راکاږل**: برابروونکي د لومړیتوب له مخې د هر کارت د نیولو لاستي په کارولو تنظیم کړئ؛ غوښتنې برابروونکي په ترتیب کاروي
- **اتومات بدیل لاره موندنه**: که لومړنی برابروونکی شتون ونه لري، غوښتنې په اتومات ډول راتلونکي تنظیم شوي برابروونکي ته ځي
- **د هر برابروونکي تنظیم**: هر برابروونکی په خپلواکه توګه د خپلو اسنادو او ماډل ټاکنې سره تنظیمېږي
- **د کارت پر بنسټ د امستنو UI**: د WordPress 6.9+ پر ثابتو Card برخو جوړ شوی (CardBody, CardHeader, CardDivider)
- **د زیاتولو پر مهال اتومات پرانیستل**: نوي برابروونکي کارتونه په اتومات ډول پرانیستل کېږي څو تاسې سمدستي جزئیات ډک کړئ
- **د سکریپټ cache ماتول**: د plugin د تازه کولو پر مهال د برابروونکي سکریپټونه cache-bust کېږي څو ډاډ ترلاسه شي چې وروستۍ نسخه پورته شوې ده

## اړتیاوې

- WordPress 5.3 یا لوړ
- PHP 7.4 یا لوړ
- Ultimate Multisite plugin (فعال)
- یو OpenAI-سازګار API پای‌ټکی (خپله کوربه‌توب شوی یا د درېیم لوري)

## نصب

1. د addon فایلونه خپل `/wp-content/plugins/` پوښۍ ته پورته کړئ
2. plugin د WordPress د 'Plugins' مېنو له لارې فعال کړئ
3. د addon د تنظیم لپاره **Ultimate Multisite → AI Connector** ته لاړ شئ

## تنظیم

### د څو برابروونکو جوړښت

د تنظیم پرده د کارت پر بنسټ جوړښت کاروي، چې پکې هر کارت یو AI برابروونکی استازیتوب کوي. تاسې د اړتیا په اندازه برابروونکي زیاتولای شئ.

#### د برابروونکي زیاتول

1. د شبکې اډمین کې **Ultimate Multisite → AI Connector** ته لاړ شئ
2. **برابروونکی زیات کړئ** کېکاږئ
3. یو نوی برابروونکي کارت څرګندېږي او په اتومات ډول پرانیستل کېږي
4. د برابروونکي جزئیات ډک کړئ:
   - **د برابروونکي نوم** — د دې برابروونکي د پېژندلو لپاره یو نښان (لکه "OpenAI"، "Mistral"، "Local Ollama")
   - **د پای‌ټکي URL** — د OpenAI-سازګار API بنسټیز URL (لکه `https://api.openai.com/v1`)
   - **API کیلي** — د دې برابروونکي لپاره ستاسو API کیلي
   - **تلوالیز ماډل** — د ماډل پېژندونکی چې د دې برابروونکي لپاره په تلواله توګه کارېږي (لکه `gpt-4o`، `mistral-small`)
5. **امستنې خوندي کړئ** کېکاږئ

#### د برابروونکي لرې کول

د هر برابروونکي کارت پر **لرې کړئ** تڼۍ کېکاږئ څو یې ړنګ کړئ. د بدلون د پلي کولو لپاره امستنې خوندي کړئ.

#### د برابروونکو بیا ترتیبول

د هر برابروونکي کارت کیڼ اړخ کې د نیولو نښه (⣿) راکاږئ څو یې بیا ترتیب کړئ. ترتیب د بدیل لومړیتوب ټاکي: غوښتنې لومړي برابروونکي ته لېږل کېږي او که یوه غوښتنه ناکامه شي، راتلونکو برابروونکو ته بدلېږي.

### بدیل لاره موندنه

کله چې لومړني برابروونکي ته غوښتنه ناکامه شي — د وخت تېرېدو، د کچې محدودیت، یا د خدمت د بندښت له امله — نښلوونکی په اتومات ډول د لست له راتلونکي برابروونکي سره بیا هڅه کوي. دا به د لست تر پایه دوام وکړي تر څو یو برابروونکی بریالی شي یا ټول برابروونکي وازمویل شي.

| سناریو | چلند |
|----------|-----------|
| لومړنی برابروونکی ځواب ورکوي | غوښتنه په عادي ډول ترسره کېږي |
| لومړنی برابروونکی ناکامېږي | غوښتنه له راتلونکي برابروونکي سره بیا ازمویل کېږي |
| ټول برابروونکي ناکامېږي | تېروتنه غوښتونکي ته بېرته ورکول کېږي |

بدیل لاره موندنه د غوښتنه کوونکي plugin یا theme لپاره شفافه ده — هغوی یا بریالی ځواب ترلاسه کوي یا وروستۍ تېروتنه، پرته له دې چې څو برابروونکي ازمویل شوي وي.

### د برابروونکي د تنظیم ماخذ

| ډګر | اړین | تشریح |
|-------|----------|-------------|
| Provider Name | Yes | د دې برابروونکي لپاره د ښودلو نښان |
| Endpoint URL | Yes | د OpenAI-سازګار API بنسټیز URL |
| API Key | Yes | د دې برابروونکي لپاره د کره‌کتنې کیلي |
| Default Model | Yes | د ماډل پېژندونکی چې هغه وخت کارېږي کله چې په غوښتنه کې ماډل نه وي ټاکل شوی |

## ستونزې حل کول

### هېڅ برابروونکي نه دي لست شوي

که د خوندي کولو وروسته د برابروونکو لست تش وي، تایید کړئ چې:

- تاسې د برابروونکو له زیاتولو وروسته **امستنې خوندي کړئ** کېکاږلې دي
- ستاسو براوزر د امستنو پاڼه له خوندي کولو نه ده منع کړې (د JavaScript تېروتنو لپاره د براوزر console وګورئ)
- plugin فعال دی او تاسې د شبکې اډمین پرده کې یاست

### غوښتنې د ټولو برابروونکو لپاره ناکامېږي

- تایید کړئ چې د هر برابروونکي پای‌ټکي URL ستاسو له سرور څخه لاسرسی لري (یوازې ستاسو له براوزر څخه نه)
- وګورئ چې هره API کیلي سمه ده او وخت یې نه دی تېر شوی
- د برابروونکي د rate-limit او quota حالت د هغوی په dashboard کې وڅېړئ
- د WordPress debug logging (`WP_DEBUG_LOG`) فعال کړئ څو له نښلوونکي څخه مفصلې تېروتنې پیغامونه ثبت شي

### یو برابروونکی د بدیل لپاره نه کارېږي

د بیا-ترتیب لپاره راکاږل د بدیل ترتیب بدلوي. تایید کړئ هغه برابروونکی چې تاسې تمه لرئ بدیل غوښتنې ترلاسه کړي، په لست کې په سم ځای کې دی او د هغه پای‌ټکي URL او API کیلي سمې دي.

### نوی برابروونکي کارت په اتومات ډول نه پرانیستل کېږي

اتومات پرانیستل اړتیا لري چې JavaScript ستاسو په براوزر کې فعال وي. که کارت په کلیک کولو نه پرانیستل کېږي، د تېروتنو لپاره د براوزر console وګورئ. د کارت منځپانګې تل د کارت سرلیک په کلیک کولو سره د لاسرسي وړ دي، حتی که اتومات پرانیستل فعال نه شي.
