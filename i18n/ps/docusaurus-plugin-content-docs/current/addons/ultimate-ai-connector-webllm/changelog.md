# د WebLLM لپاره Ultimate AI Connector بدلون‌لړ {#ultimate-ai-connector-for-webllm-changelog}

## نسخه 1.1.0 — په 2026-04-09 خپره شوه {#version-110--released-on-2026-04-09}

* نوی: د اډمین-بار حالت نښې سره لامبو وهونکی د چټ ویجټ — هر ننوتی کاروونکی کولی شي د مخکیني پای څخه نېغ په نېغه د براوزر-اړخ LLM ته پرامپټ ورکړي.
* نوی: SharedWorker رن‌ټایم — د MLC انجن اوس په SharedWorker کې چلېږي، نو څو ټبونه د GPU یوه ناسته شریکوي، د دې پر ځای چې پر هغې سره وجنګېږي.
* نوی: apiFetch منځګړی انټرسیپټر — د WordPress REST غوښتنې چې د AI Client SDK بڼې سره سمون خوري، په شفاف ډول محلي WebLLM بروکر ته لېږدول کېږي، د لوپ‌بک HTTP تګ-راتګ ته اړتیا نشته.
* نوی: د Connector پینل کې د ویجټ امستنو UI، د چټ ویجټ د چالان/بند کولو او د اتومات-پرامپټ چلند د تنظیم لپاره.
* سمونه: د IndexedDB کېش شالید په زور وکاروئ، څو د ماډل وزن ډاونلوډونه د HuggingFace xet CDN له هغو ریدایرکټونو وروسته هم پاتې شي چې د ډیفالټ Cache API لاره ماتوي.
* سمونه: د embedding ماډلونو لپاره د context_window KV-cache override پرېږدئ (هغوی ډیکوډر نه لري او override د رن‌ټایم تېروتنه رامنځته کوله).
* سمونه: مخکې له دې چې د ورکر ټب پورته شي، په /webllm/v1/models کې د سړې-پیل نوماند ماډل اعلان کړئ، څو د SDK کاروونکي سملاسي یو ماډل وویني.
* سمونه: WebLLM د wpai_preferred_text_models filter سره ونښلوئ، څو د AI Experiments ځانګړنه د ماډل د تنظیم په صورت کې د براوزر انجن ته لاره ومومي.
* ښه شوی: د کېش-ماتولو، منځپانګې عادي‌کولو، او د هارډویر-مرجع سمونې چې د پای-تر-پایه ازموینې پر مهال ښکاره شوې.
