# درس ۷: دا ستاسو خپلول {#lesson-7-making-it-yours}

ستاسو پېرودونکي باید هېڅکله داسې احساس ونه کړي چې دوی "کوم WordPress plugin" کاروي. دوی باید داسې احساس وکړي چې دوی FitSite کاروي -- داسې پلېټفارم چې د دوی د صنعت لپاره جوړ شوی. دا درس برانډ جوړونه، white-labeling، او دا پوښي چې څنګه پلېټفارم د یو محصول په څېر احساس شي.

## موږ چېرته پاتې شوي وو {#where-we-left-off}

FitSite یو فعال checkout بهیر لري چې د فټنس سټوډیو مالکان د پلان له ټاکلو څخه تر ژوندي سایټ پورې رسوي. اوس موږ ټوله تجربه داسې جوړوو چې د یو منسجم، برانډ شوي محصول په څېر احساس شي.

## ستاسو د پلېټفارم ډومېن {#your-platform-domain}

ستاسو د برانډ بنسټ ستاسو ډومېن دی. د FitSite لپاره:

- **اصلي ډومېن**: `fitsite.com` (ستاسو د بازارموندنې سایټ او د شبکې ریښه)
- **د پېرودونکو سایټونه**: `studioname.fitsite.com` (فرعي ډومېنونه)
- **Custom domains**: د Growth او Pro پلانونو پېرودونکي کولی شي خپل ډومېن وصل کړي

### ستاسو د ډومېن تنظیمول {#setting-up-your-domain}

1. د خپل پلېټفارم ډومېن ثبت کړئ
2. هغه خپل hosting provider ته وګرځوئ
3. د پېرودونکو فرعي ډومېنونو لپاره wildcard DNS (`*.fitsite.com`) تنظیم کړئ
4. ډاډ ترلاسه کړئ چې wildcard SSL فعال دی

د تفصیلي لارښوونو لپاره [د Domain Mapping د تنظیمولو څرنګوالی](/user-guide/domain-mapping/how-to-configure-domain-mapping) وګورئ.

## د Admin تجربې White-Labeling {#white-labeling-the-admin-experience}

کله چې د فټنس سټوډیو مالک خپل site dashboard ته ننوځي، دوی باید ستاسو برانډ وویني، نه د WordPress یا Ultimate Multisite برانډ.

### دودیز Login پاڼه {#custom-login-page}

د WordPress login پاڼه داسې دودیزه کړئ چې دا وښيي:

- ستاسو د FitSite لوګو
- د فټنس لپاره مناسب شالید انځورونه
- ستاسو د برانډ رنګونه

### د Dashboard برانډ کول {#dashboard-branding}

د [Admin Page Creator](/addons/admin-page-creator) addon یا دودیز CSS وکاروئ چې:

- د WordPress لوګو د خپل FitSite لوګو سره بدل کړئ
- د admin رنګ سکیم داسې دودیز کړئ چې ستاسو له برانډ سره برابر وي
- د فټنس اړوند چټکو لینکونو او مرستې سرچینو سره یو دودیز dashboard widget اضافه کړئ

### دودیزې Admin پاڼې {#custom-admin-pages}

داسې دودیزې admin پاڼې جوړول په پام کې ونیسئ چې د فټنس سټوډیو مالکانو لپاره تر ټولو اړوند کړنې ښکاره کړي:

- "خپل د ټولګیو مهالویش سم کړئ"
- "د روزونکو پروفایلونه تازه کړئ"
- "خپل سایټ وګورئ"

دا د زده کړې بهیر اسانه کوي، ځکه چې د ځانګړي بازار اړوند کړنې د معیاري WordPress مېنو کې د پټولو پر ځای مخې ته او مرکز ته راولي.

## ستاسو اړیکې برانډ کول {#branding-your-communications}

هر برېښنالیک، انوایس، او خبرتیا باید ستاسو برانډ پیاوړی کړي.

### د سیستم برېښنالیکونه {#system-emails}

**Ultimate Multisite > Settings > Emails** ته لاړ شئ او ټول د سیستم برېښنالیکونه دودیز کړئ:

- **From name**: FitSite
- **From email**: hello@fitsite.com
- **Email templates**: د خپل برانډ رنګونه او لوګو وکاروئ
- **ژبه**: په ټوله کې د فټنس اړونده

هغه مهم برېښنالیکونه چې باید دودیز شي:

| برېښنالیک | عمومي بڼه | د FitSite بڼه |
|-------|----------------|-----------------|
| ښه راغلاست | "ستاسو نوی سایټ چمتو دی" | "ستاسو د فټنس سټوډیو وېبپاڼه ژوندۍ ده" |
| د تادیې رسید | "تادیه ترلاسه شوه" | "د FitSite subscription تادیه تایید شوه" |
| د ازمایښتي مودې پای | "ستاسو ازمایښتي موده ژر پای ته رسېږي" | "ستاسو د FitSite ازمایښتي موده په ۳ ورځو کې پای ته رسېږي -- خپل د سټوډیو وېبپاڼه ژوندۍ وساتئ" |

### انوایسونه {#invoices}

د انوایس کينډۍ له دې سره دودیزې کړئ:

- ستاسو د FitSite لوګو او د برانډ رنګونه
- ستاسو د سوداګرۍ جزئیات
- د فټنس اړوند د محصول نومونه (نه عمومي پلان پېژندنې)

## د پېرودونکو مخامخ سایټ {#the-customer-facing-site}

ستاسو اصلي ډومېن (`fitsite.com`) داسې د بازارموندنې سایټ ته اړتیا لري چې پلېټفارم وپلوري. دا د Ultimate Multisite د شبکې admin څخه جلا دی -- دا ستاسو د سوداګرۍ عامه مخ دی.

مهمې پاڼې:

- **کورپاڼه**: د فټنس سوداګریو لپاره روښانه ارزښت وړاندیز
- **ځانګړنې**: FitSite څه کوي، د فټنس په اصطلاحاتو کې
- **بیې**: ستاسو درې پلانونه د ځانګړي بازار د ځانګړنو له پرتلو سره
- **بېلګې**: هغه سایټونه وښایاست چې په پلېټفارم جوړ شوي
- **نوم‌لیکنه**: ستاسو checkout فورم ته لینکونه

:::tip ستاسو د بازارموندنې سایټ د شبکې سایټ کېدای شي
خپل د بازارموندنې سایټ د خپلې شبکې دننه د یو سایټ په توګه جوړ کړئ. دا تاسو ته اجازه درکوي چې هغه له هماغه dashboard څخه اداره کړئ او د خپل پلېټفارم وړتیاوې وښيي.
:::

## د پېرودونکو لپاره Custom Domain {#custom-domain-for-customers}

د هغو پېرودونکو لپاره چې پلانونه یې custom domains لري، بهیر په روښانه ډول مستند کړئ:

1. پېرودونکی خپل ډومېن له registrar سره ثبتوي یا لېږدوي
2. پېرودونکی DNS تازه کوي چې ستاسو پلېټفارم ته اشاره وکړي (دقیق records ورکړئ)
3. Ultimate Multisite د domain mapping او SSL اداره کوي

د همدې بهیر لپاره ځانګړې مرسته‌لیک یا د پوهې بنسټ ننوتنه جوړه کړئ، چې د غیر تخنیکي فټنس سټوډیو مالکانو لپاره لیکل شوې وي.

## تر اوسه د FitSite شبکه {#the-fitsite-network-so-far}

```
FitSite شبکه
├── WordPress Multisite (د فرعي ډومېن حالت)
├── Ultimate Multisite (تنظیم شوی + برانډ شوی)
├── د پلېټفارم ډومېن (fitsite.com + wildcard SSL)
├── د سایټ کينډۍ (Studio Essential, Gym Pro, Fitness Chain)
├── محصولات (Starter, Growth, Pro + Order Bumps)
├── Checkout بهیر (د ځانګړي بازار لپاره، ازمویل شوی)
├── برانډ کول
│   ├── دودیز login پاڼه
│   ├── برانډ شوی admin dashboard
│   ├── د ځانګړي بازار سیستم برېښنالیکونه
│   ├── برانډ شوي انوایسونه
│   └── په fitsite.com د بازارموندنې سایټ
└── د onboarding بهیر لپاره چمتو (بل درس)
```

## په دې درس کې مو څه جوړ کړل {#what-we-built-this-lesson}

- **د پلېټفارم ډومېن او DNS** د برانډ شوې تجربې لپاره تنظیم شول
- **White-labeled admin** د FitSite برانډ سره په ټوله کې
- **دودیزې اړیکې** -- برېښنالیکونه، انوایسونه، او خبرتیاوې ټول د برانډ مطابق
- **یو د بازارموندنې سایټ** چې FitSite د فټنس سټوډیو مالکانو ته پلوري
- **Custom domain مستندات** د هغو پېرودونکو لپاره چې خپل ډومېن غواړي
