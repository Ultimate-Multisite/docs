# درس 13: پراخول

تاسو یوه فعاله پلېټفارم لرئ چې پېرودونکي یې پیسې ورکوي. دا درس پوښي چې څنګه له کوچنیو عملیاتو څخه یوه پایدارې سوداګرۍ ته وده وکړئ -- زېربنا پراخه کړئ، عملیات اتومات کړئ، او د هر پېرودونکي عاید زیات کړئ.

## موږ چېرته پاتې شوي وو

FitSite ژوندی دی، پېرودونکي نوم‌لیکنه کوي، او تاسو ورځني عملیات پرمخ وړئ. اوس موږ پر ودې تمرکز کوو.

## خپلې شمېرې وپېژنئ

له پراخولو مخکې، پوه شئ چې په کوم حالت کې یاست:

### کلیدي سنجونې

- **MRR (میاشتنی تکراري عاید)**: ټول میاشتنی د ګډون عاید
- **د پېرودونکو شمېر**: ټول فعال ګډونوال
- **ARPU (د هر کاروونکي منځنی عاید)**: MRR د پېرودونکو پر شمېر وېشل شوی
- **د پرېښودو کچه**: د پېرودونکو سلنه چې هره میاشت لغوه کوي
- **LTV (د ژوندانه ارزښت)**: د پېرودونکي د ټول ګډون په موده کې منځنی عاید
- **CAC (د پېرودونکي د ترلاسه کولو لګښت)**: د یو پېرودونکي د ترلاسه کولو منځنی لګښت

### بېلګه: FitSite د ۵۰ پېرودونکو سره

| سنجونه | ارزښت |
|--------|-------|
| پېرودونکي | 50 (30 Starter, 15 Growth, 5 Pro) |
| MRR | $4,450 ($1,470 + $1,485 + $995 + $500 اضافي وړاندیزونه) |
| ARPU | $89/میاشت |
| میاشتنی پرېښودل | 4% (2 لغوه کول/میاشت) |
| LTV | $89 x 25 میاشتې = $2,225 |

دا شمېرې تاسو ته وایي چې پر څه تمرکز وکړئ. پرېښودل لوړ دي؟ ساتنه سمه کړئ. ARPU ټیټ دی؟ ارتقاوې وهڅوئ. CAC لوړ دی؟ د ترلاسه کولو چینلونه ښه کړئ.

## د زېربنا پراخول

### کله پراخول پکار دي

hosting هغه وخت پراخه کړئ چې:

- د پاڼې د لوډ وختونه په څرګند ډول زیات شي
- د سرور CPU یا حافظه په منظم ډول له 70% کارونې څخه واوړي
- تاسو 100+ فعالو وېبپاڼو ته نږدې کېږئ
- د سرعت په اړه د پېرودونکو شکایتونه زیات شي

### څنګه پراخول

- **عمودي پراخول**: لوی سرور ته ارتقا وکړئ (زیات CPU، RAM)
- **د cache طبقې**: د object caching لپاره Redis/Memcached زیات کړئ، او د static منځپانګې لپاره page caching
- **CDN**: که لا دمخه Cloudflare یا ورته څه نه کاروئ، د static شتمنیو لپاره CDN زیات کړئ
- **د ډیټابېس ښه‌والی**: لکه څنګه چې شبکه غټېږي، د ډیټابېس پوښتنې ورو کېږي. جدولونه ښه کړئ، indexes زیات کړئ، او ځانګړی ډیټابېس سرور په پام کې ونیسئ.
- **چارو جلا کول**: د رسنیو زېرمه object storage ته ولېږدئ (S3-compatible)، او email یو transactional email خدمت ته وسپارئ

### د hosting کډون

که ستاسو اوسنی کوربه نور نه شي پراخېدای، کډون پلان کړئ:

1. نوی چاپېریال جوړ کړئ
2. د خپلې شبکې له یوې کاپۍ سره یې په بشپړ ډول وازمویئ
3. کډون د کم ترافیک په ساعتونو کې مهالویش کړئ
4. له مخکې DNS د لږ تر لږه TTL سره تازه کړئ
5. له کډون وروسته تایید کړئ چې هر څه کار کوي

## د عملیاتو اتومات کول

لکه څنګه چې تاسو وده کوئ، لاسي پروسې خنډونه ګرځي. هغه څه اتومات کړئ چې کولی شئ:

### Webhooks او Zapier

[Webhooks](/user-guide/integrations/webhooks) یا [Zapier](/user-guide/integrations/zapier) وکاروئ چې اتومات کړئ:

- **د نوې نوم‌لیکنې خبرتیاوې** → Slack چینل یا CRM
- **د لغوه کولو خبرداري** → د بیا راګرځولو email لړۍ پیلوي
- **د تادیې ناکامۍ** → ستاسو د څارنې په وسیله کې خبرداری
- **د پلان ارتقاوې** → د نوې ځانګړنې له لارښود سره د مبارکۍ email

### د email اتومات کول

له لاسي emailونو څخه اتومات لړیو ته لاړ شئ:

- د پیل کولو لړۍ (لا دمخه په ۸ درس کې جوړه شوې)
- د غیرفعالو پېرودونکو لپاره د بیا ښکېلتیا لړۍ
- د ارتقا هڅونې کله چې پېرودونکي د پلان حدودو ته نږدې کېږي
- د کلنیو ګډونوالو لپاره د نوي کولو یادونې

### د ملاتړ اتومات کول

- **چمتو ځوابونه** د عامو پوښتنو لپاره
- **اتومات ځوابونه** چې د ملاتړ ټکټونو ترلاسه کول تاییدوي
- **د پوهې بنسټ وړاندیزونه** کله چې پېرودونکي داسې ټکټونه وړاندې کوي چې له موجودو مقالو سره سمون لري

## د عاید زیاتول

وده یوازې د زیاتو پېرودونکو په اړه نه ده. دا د هر پېرودونکي د زیات عاید په اړه هم ده.

### موجودو پېرودونکو ته لوړ پلور

- **د پلان ارتقاوې**: هدفمند کمپاینونه چې Starter پېرودونکو ته د Growth/Pro ځانګړنې ښيي
- **اضافي وړاندیزونه**: موجودو پېرودونکو ته د email له لارې اضافي محصولات وروپېژنئ
- **کلني بدلون ته اړول**: میاشتنیو پېرودونکو ته تخفیف ورکړئ چې کلني billing ته واوړي

### د عاید نوې لارې

- **ستاسو لپاره بشپړ setup**: د پېرودونکي وېبپاڼه د هغوی لپاره د جوړولو او شخصي کولو په بدل کې لوړ فیس واخلئ
- **د دودیز ډیزاین خدمتونه**: د template تر څنګ ځانګړی ډیزاین کار وړاندې کړئ
- **د روزنې ناستې**: هغو پېرودونکو ته تادیه شوې یو پر یو لارښوونې چې عملي مرسته غواړي
- **Premium plugins**: د ځانګړې ساحې لپاره premium plugins د تادیه شوو add-ons په توګه وړاندې کړئ (لکه، د fitness class booking widget)

### بیې لوړول

لکه څنګه چې ستاسو پلېټفارم پخېږي او ارزښت زیاتوي:

- موجود پېرودونکي د هغوی په اوسني قیمت کې وساتئ
- د نوو نوم‌لیکنو لپاره بیې لوړې کړئ
- زیاتوالی د نوو ځانګړنو او ښه‌والي په وسیله توجیه کړئ

## د ټیم جوړول

په یوه پړاو کې، تاسو هر څه یوازې نه شئ کولای. عام لومړني ګمارنې:

1. **د ملاتړ کس**: د پېرودونکو ورځنۍ پوښتنې سمبالوي (په پیل کې نیمه‌وخت)
2. **د منځپانګې جوړوونکی**: د پوهې بنسټ مقالې، blog پوسټونه، او د بازارموندنې منځپانګه لیکي
3. **ډیزاینر**: templates ښه کوي او نوي جوړوي

تاسو کارکوونکو ته اړتیا نه لرئ. قراردادیان او freelancers د پلېټفارم سوداګرۍ لپاره ښه کار کوي.

## د ودې پړاوونه

| پړاو | اټکلي MRR | تمرکز |
|-----------|-----------------|-------|
| 0-25 پېرودونکي | $0-$2,500 | د محصول او بازار سمون، مستقیمه اړیکه |
| 25-100 پېرودونکي | $2,500-$10,000 | عملیات منظمول، د منځپانګې بازارموندنه |
| 100-250 پېرودونکي | $10,000-$25,000 | ملاتړ ګمارل، بدلون ښه کول، hosting پراخول |
| 250-500 پېرودونکي | $25,000-$50,000 | د ټیم جوړول، د عاید نوې لارې، premium ځانګړنې |
| 500+ پېرودونکي | $50,000+ | د پلېټفارم پختګي، نږدې ساحې، احتمالي وتل |

## په دې درس کې مو څه جوړ کړل

- **د سنجونو چوکاټ** د سوداګرۍ د روغتیا د پوهېدو لپاره
- **د زېربنا د پراخولو پلان** له لسګونو څخه تر سلګونو وېبپاڼو د ودې لپاره
- **د اتومات کولو تګلارې** د ملاتړ، email، او عملیاتو لپاره
- **د عاید د ودې تاکتیکونه** یوازې د نوو پېرودونکو له ترلاسه کولو هاخوا
- **د ټیم جوړولو لارښوونه** د هغه وخت لپاره چې له انفرادي عملیاتو څخه لوی شئ
- **د ودې پړاوونه** د هر پړاو لپاره د تمرکز ساحو سره
