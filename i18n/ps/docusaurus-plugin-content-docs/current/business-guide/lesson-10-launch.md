# لوست 10: د پیل ورځ {#lesson-10-launch-day}

هر څه جوړ شوي دي. مخکې له دې چې دروازې پرانیزئ، دا لوست د پیل څخه مخکې د پلټنې لړلیک درته تېروي، څو ډاډ ترلاسه شي چې هېڅ شی مات، ورک، یا شرموونکی نه دی.

## موږ چېرته پاتې شوي وو {#where-we-left-off}

FitSite کې قالبونه، پلانونه، د تادیې پړاو، نښه‌برندي، د پیل لارښوونه، او بیې ټولې تنظیم شوې دي. اوس موږ تاییدوو چې هر څه کار کوي او ژوندی یې کوو.

## د پیل څخه مخکې د پلټنې لړلیک {#pre-launch-checklist}

هر توکی بشپړ کړئ. هېڅ یو مه پرېږدئ.

### د پلېټفارم بنسټیز جوړښت {#platform-infrastructure}

- [ ] Hosting ثابت دی او د بار لاندې ښه کار کوي
- [ ] Wildcard SSL فعال دی او ټول فرعي ډومېنونه د HTTPS له لارې وړاندې کېږي
- [ ] د ډومېن نښلول کار کوي -- د یو سایټ جوړول او د دودیز ډومېن نښلول وازمویئ
- [ ] بیک‌اپونه تنظیم او ازمویل شوي دي (لږ تر لږه یو بېرته راګرځول وازمویئ)
- [ ] څارنه فعاله ده -- که پلېټفارم ښکته شي، تاسو به خبر شئ

### قالبونه {#templates}

- [ ] ټول درې قالبونه په نویو سایټونو کې سم بارېږي
- [ ] ځای‌ناستی منځپانګه ګټوره ده او د ټایپي تېروتنو څخه پاکه ده
- [ ] ټول انځورونه سم جواز لري (د سټاک انځور د اوبو نښې نه لري)
- [ ] د موبایل ځواب‌ورکوونکې بڼه د هر قالب په هره پاڼه کې کار کوي
- [ ] د پاڼې د بارېدو چټکتیا منلو وړ ده (د GTmetrix یا PageSpeed Insights په څېر وسیلې سره یې وازمویئ)
- [ ] په هېڅ قالب کې مات لینکونه یا ورک شتمنۍ نشته

### پلانونه او محصولات {#plans-and-products}

- [ ] ټول درې پلانونه فعال او ښکاره دي
- [ ] د پلان تشریحات دقیق او د ځانګړې برخې لپاره مناسب دي
- [ ] بیې سمې دي (میاشتنۍ او کلنۍ)
- [ ] د تنظیم فیسونه په سمو پلانونو کې تنظیم شوي دي
- [ ] د آزمایښتي مودې ځانګړنه په Starter پلان کې کار کوي
- [ ] اضافي وړاندیزونه د تادیې پړاو پر مهال سم ښکاري
- [ ] د plugin او theme محدودیتونه د هر پلان لپاره سم پلي کېږي

### د تادیې بهیر {#checkout-flow}

- [ ] په هر پلان کې د بشپړ آزمایښتي نوم‌لیکنې پړاو ترسره کړئ (د آزمایښتي تادیې حالت وکاروئ)
- [ ] د قالب ټاکنه د هر پلان لپاره سم قالبونه ښيي
- [ ] تادیه په بریالیتوب ترسره کېږي
- [ ] پېرودونکی له نوم‌لیکنې وروسته د هرکلي برېښنالیک ترلاسه کوي
- [ ] نوی سایټ د سم قالب سره جوړېږي
- [ ] پېرودونکی سمدستي خپل نوي سایټ ته ننوتلی شي
- [ ] د تخفیف کوډونه سم کار کوي

### نښه‌برندي {#branding}

- [ ] د ننوتلو پاڼه د FitSite نښه‌برندي ښيي
- [ ] د مدیر Dashboard د FitSite نښه‌برندي ښيي
- [ ] ټول سیسټمي برېښنالیکونه د FitSite نښه‌برندي او د فټنس ځانګړې ژبه کاروي
- [ ] رسیدونه ستاسو د سوداګرۍ له جزئیاتو سره سم ښکاري
- [ ] د بازارموندنې سایټ ژوندی دی او د تادیې فورم ته لینک لري

### د پیل لارښوونه {#onboarding}

- [ ] د چټک پیل ویجېټ د نویو پېرودونکو په Dashboardونو کې ښکاري
- [ ] د چټک پیل ټول لینکونه سمو پاڼو ته اشاره کوي
- [ ] د هرکلي برېښنالیکونو لړۍ تنظیم او ازمویل شوې ده
- [ ] د پوهې بنسټ مقالې خپرې شوې او لاسرسی ورته شته
- [ ] د Account پاڼه د پلان سم معلومات او د لوړولو اختیارونه ښيي

### قانوني او سوداګریز {#legal-and-business}

- [ ] د خدمت شرطونه خپاره شوي او د تادیې له پړاو څخه لینک شوي دي
- [ ] د محرمیت تګلاره خپره شوې او لاسرسی ورته شته
- [ ] د پیسو بېرته ورکولو تګلاره تعریف او مستنده شوې ده
- [ ] سوداګریز بنسټ د تادیاتو ترلاسه کولو لپاره جوړ شوی دی
- [ ] د تادیې دروازه په ژوندۍ حالت کې ده (نه په آزمایښتي حالت کې)
- [ ] د مالیې تنظیمات ستاسو د واکمنې سیمې لپاره سم دي

## نرم پیل او سخت پیل {#soft-launch-vs-hard-launch}

د دوه پړاوه پیل په اړه فکر وکړئ:

### پړاو 1: نرم پیل {#phase-1-soft-launch}

د عامه پیل څخه مخکې 5-10 د فټنس سټوډیو مالکانو ته بلنه ورکړئ چې نوم‌لیکنه وکړي. دوی ستاسو بېټا پېرودونکي دي. د دې په بدل کې ورته د پام وړ تخفیف وړاندې کړئ (د ټول عمر لپاره 50% کم، یا 3 میاشتې وړیا):

- د نوم‌لیکنې او د پیل لارښوونې تجربې په اړه رښتینی نظر
- اجازه چې د دوی سایټ د نندارې د بېلګې په توګه وکاروئ
- د هر ډول بګونو یا ستونزو راپور ورکول چې ورسره مخ کېږي

دا تاسو ته رښتینی د پېرودونکو نظر او ژوندي سایټونه درکوي چې د عامو خلکو لپاره له پرانیستلو مخکې یې وښایئ.

### پړاو 2: عامه پیل {#phase-2-public-launch}

کله چې د نرم پیل نظرونه شامل شي:

- د تادیې دروازه ژوندۍ حالت ته واړوئ
- خپل د بازارموندنې سایټ خپور کړئ
- د پېرودونکو جذب پیل کړئ (لوست 11)
- په اړوندو د فټنس صنعت چینلونو کې اعلان وکړئ

## د پیل ورځې کړنې {#launch-day-actions}

په هغه ورځ چې عامو خلکو ته یې پرانیزئ:

1. **ژوندیو تادیاتو ته واړوئ** -- په خپله تادیاتي دروازه کې آزمایښتي حالت غیر فعال کړئ
2. **یو ځل بیا تایید کړئ** -- د رښتینې تادیې سره بشپړه آزمایښتي نوم‌لیکنه ترسره کړئ (وروسته ځان ته پیسې بېرته ورکړئ)
3. **نږدې څارنه وکړئ** -- تېروتنې، ناکامې نوم‌لیکنې، یا د تادیې ستونزې وڅارئ
4. **چمتو اوسئ** -- ستاسو لومړي رښتیني پېرودونکي کېدای شي مرستې ته اړتیا ولري، او چټک ځواب باور جوړوي
5. **لنډه لمانځنه وکړئ** -- بیا بېرته کار ته ستانه شئ

## څه شی غلط کېدای شي {#what-can-go-wrong}

د دې لپاره چمتو اوسئ:

- **د تادیې دروازې ستونزې**: د خپلې دروازې د برابرونکي د ملاتړ اړیکه چمتو ولرئ
- **د SSL سند ستونزې**: پوه شئ چې سندونه څنګه وګورئ او نوي کړئ
- **د برېښنالیک رسولو ناکامۍ**: وازمویئ چې برېښنالیکونه رښتیا رسېږي (د سپم فولډرونه وګورئ)
- **د بار لاندې فعالیت**: که د ترافیک ناڅاپي زیاتوالی ولرئ، پوه شئ چې خپل Hosting څنګه پراخ کړئ
- **د پېرودونکي ګډوډي**: خپل د پوهې بنسټ او د ملاتړ چینلونه چمتو ولرئ
