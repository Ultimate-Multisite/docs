# പാഠം 6: സൈൻഅപ്പ് അനുഭവം

പണം സമ്പാദിക്കാനുള്ള അവസരമായി മാറുന്ന പ്രക്രിയയാണ് చెക്ക്ഔട്ട് ഫ്ലോ. ആശയക്കുഴപ്പമുണ്ടാക്കുന്നതോ പൊതുവായതോ ആയ ഒരു സൈൻഅപ്പ് പ്രക്രിയ ഉപഭോക്താക്കളെ നഷ്ടപ്പെടുത്തും. അവരുടെ ഭാഷ സംസാരിക്കുന്നതും എളുപ്പമുള്ളതുമായ ഒരു പ്രത്യേക മേഖലയിലുള്ള (niche-specific) പ്രവാഹം അവരെ കൺവേർട്ട് ചെയ്യും.

## നമ്മൾ എവിടെ എത്തി {#where-we-left-off}

FitSite-ന് ടെംപ്ലേറ്റുകളും പ്ലാനുകളും സജ്ജീകരിച്ചിട്ടുണ്ട്. ഇപ്പോൾ, ഫിറ്റ്നസ് സ്റ്റുഡിയോ ഉടമകളെ പണം നൽകുന്ന ഉപഭോക്താക്കളാക്കി മാറ്റുന്ന രജിസ്ട്രേഷൻ, చెക്ക്ഔട്ട് അനുഭവം എന്നിവ നമുക്ക് നിർമ്മിക്കണം.

## രജിസ്ട്രേഷൻ പ്രവാഹം മനസ്സിലാക്കൽ {#understanding-the-registration-flow}

Ultimate Multisite-ന്റെ చెക്ക്ഔട്ട് ഫോമുകൾ പൂർണ്ണമായും കസ്റ്റമൈസ് ചെയ്യാവുന്ന മൾട്ടി-സ്റ്റെപ്പ് ഫോമുകളാണ്. FitSite-ന് വേണ്ടത് ഒരു പ്രവാഹമാണ്:

1. ഫിറ്റ്നസ് ബിസിനസ്സുകൾക്കായി നിർമ്മിച്ചതാണെന്ന് തോന്നണം.
2. ആവശ്യമുള്ള വിവരങ്ങൾ മാത്രമേ ശേഖരിക്കാവൂ.
3. ഉപഭോക്താവിനെ കഴിയുന്നത്ര വേഗം പ്രവർത്തിക്കുന്ന സൈറ്റിലേക്ക് എത്തിക്കണം.

പൂർണ്ണമായ സാങ്കേതിക വിവരങ്ങൾക്കായി [The Registration Flow](/user-guide/configuration/the-registration-flow) കാണുക.

## FitSite చెക്ക്ഔട്ട് രൂപകൽപ്പന ചെയ്യൽ {#designing-the-fitsite-checkout}

**Ultimate Multisite > Checkout Forms** എന്നതിലേക്ക് പോവുക, എന്നിട്ട് ഒരു പുതിയ ഫോം സൃഷ്ടിക്കുക.

### ഘട്ടം 1: പ്ലാൻ തിരഞ്ഞെടുക്കൽ (Plan Selection) {#step-1-plan-selection}

ഒരു ഫിറ്റ്നസ് സ്റ്റുഡിയോ ഉടമ ആദ്യം കാണേണ്ടത്, അവർക്ക് മനസ്സിലാകുന്ന ഭാഷയിൽ അവതരിപ്പിച്ച പ്ലാനുകളാണ്.

- ഒരു **Pricing Table** ഫീൽഡ് ചേർക്കുക
- മൂന്ന് FitSite പ്ലാനുകളും കാണിക്കാൻ ഇത് സജ്ജമാക്കുക
- പാഠം 5-ൽ നിങ്ങൾ എഴുതിയ പ്ലാൻ വിവരണങ്ങൾ ഇവിടെ വരും -- അവ സാങ്കേതിക സവിശേഷതകളെക്കുറിച്ചല്ല, മറിച്ച് ഫിറ്റ്നസ് ബിസിനസ് ആവശ്യങ്ങളെക്കുറിച്ച് സംസാരിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക.

:::tip പ്രത്യേക മേഖലയുടെ ഭാഷ പ്രധാനമാണ്
ഒരു ജിം ഉടമയ്ക്ക് "1 GB സ്റ്റോറേജ്" എന്നതൊന്നും അർത്ഥമില്ല. എന്നാൽ "ഒരു പ്രൊഫഷണൽ സ്റ്റുഡിയോ വെബ്സൈറ്റിന് വേണ്ടതെല്ലാം" എന്നത് എല്ലാം ആണ്. നിങ്ങളുടെ ഉപഭോക്താവിന്റെ ഭാഷയിൽ പ്ലാൻ വിവരണങ്ങൾ എഴുതുക.
:::

### ഘട്ടം 2: ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കൽ (Template Selection) {#step-2-template-selection}

ഒരു പ്ലാൻ തിരഞ്ഞെടുത്ത ശേഷം, ഉപഭോക്താവ് അവരുടെ സ്റ്റാർട്ടിംഗ് ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കുന്നു.

- ഒരു **Template Selection** ഫീൽഡ് ചേർക്കുക
- ലഭ്യമായ ടെംപ്ലേറ്റുകൾ അവർ തിരഞ്ഞെടുത്ത പ്ലാനമനുസരിച്ച് ഫിൽട്ടർ ചെയ്യപ്പെടും (പാഠം 5-ൽ സജ്ജീകരിച്ചത്).
- ഓരോ ടെംപ്ലേറ്റിലും ഫിറ്റ്നസ്-നിർദ്ദിഷ്ട രൂപകൽപ്പന കാണിക്കുന്ന ഒരു പ്രിവ്യൂ ഇമേജ് ഉണ്ടായിരിക്കണം.

### ഘട്ടം 3: അക്കൗണ്ട് സൃഷ്ടിക്കൽ (Account Creation) {#step-3-account-creation}

ഇത് ലളിതമായി നിലനിർത്തുക. ഇവ മാത്രം ശേഖരിക്കുക:

- ഇമെയിൽ വിലാസം (Email address)
- പാസ്‌വേഡ് (Password)
- സ്റ്റുഡിയോ/ബിസിനസ് പേര് (ഇതാണ് അവരുടെ സൈറ്റ് പേര് ആകുന്നത്)

സൈൻഅപ്പിന് ആവശ്യമില്ലാത്ത വിവരങ്ങൾ ചോദിക്കരുത്. ഓരോ അധിക ഫീൽഡും കൺവേർഷനുകൾ കുറയ്ക്കും.

### ഘട്ടം 4: സൈറ്റ് സജ്ജീകരണം (Site Setup) {#step-4-site-setup}

- **Site title**: ഘട്ടം 3-ൽ നൽകിയ സ്റ്റുഡിയോ പേരിൽ നിന്ന് മുൻകൂട്ടി പൂരിപ്പിക്കുക (Pre-fill).
- **Site URL**: സ്റ്റുഡിയോ പേരിൽ നിന്ന് ഓട്ടോമാറ്റിക്കായി സൃഷ്ടിക്കുക (ഉദാഹരണത്തിന്, `ironworks.fitsite.com`).

### ഘട്ടം 5: പേയ്‌മെന്റ് (Payment) {#step-5-payment}

- **Payment** ഫീൽഡ് ചേർക്കുക
- നിങ്ങളുടെ പേയ്‌മെന്റ് ഗേറ്റ്‌വേ സജ്ജമാക്കുക ([Stripe](/user-guide/payment-gateways/stripe) സബ്‌സ്‌ക്രിപ്‌ഷൻ ബില്ലിംഗിന് ശുപാർശ ചെയ്യുന്നു)
- പാഠം 5-ൽ നിങ്ങൾ ഓർഡർ ബംപ്സ് സൃഷ്ടിച്ചെങ്കിൽ, പേയ്‌മെന്റ് ഘട്ടത്തിന് മുമ്പ് ഒരു **Order Bump** ഫീൽഡ് ചേർക്കുക.

### ഘട്ടം 6: സ്ഥിരീകരണം (Confirmation) {#step-6-confirmation}

- ഫിറ്റ്നസ്-നിർദ്ദിഷ്ട ഭാഷ ഉപയോഗിച്ച് സ്ഥിരീകരണ സന്ദേശം കസ്റ്റമൈസ് ചെയ്യുക.
- ഉദാഹരണം: "നിങ്ങളുടെ ഫിറ്റ്നസ് സ്റ്റുഡിയോ വെബ്സൈറ്റ് സൃഷ്ടിക്ക Tengah. കുറച്ച് നിമിഷങ്ങൾക്കുള്ളിൽ നിങ്ങളുടെ പുതിയ സൈറ്റിലേക്ക് നിങ്ങളെ റീഡയറക്ട് ചെയ്യും."

## ഒരു പിരീഡ് സെലക്ഷൻ ടോഗിൾ ചേർക്കൽ {#adding-a-period-selection-toggle}

നിങ്ങളുടെ പ്ലാനുകളിൽ വില വ്യതിയാനങ്ങൾ (മാസംതോറുമോ വർഷംതോറുമോ) സജ്ജീകരിച്ചിട്ടുണ്ടെങ്കിൽ, ഉപഭോക്താക്കൾക്ക് ബില്ലിംഗ് കാലയളവുകൾക്കിടയിൽ ടോഗിൾ ചെയ്യാൻ ഒരു **Period Selection** ഫീൽഡ് చెക്ക്ഔട്ട് ഫോമിൽ ചേർക്കുക. നിർദ്ദേശങ്ങൾക്കായി [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle) കാണുക.

## പേയ്‌മെന്റ് സജ്ജമാക്കൽ (Configuring Payment) {#configuring-payment}

നിങ്ങൾ ഇതുവരെ ഒരു പേയ്‌മെന്റ് ഗേറ്റ്‌വേ സജ്ജീകരിച്ചിട്ടില്ലെങ്കിൽ:

1. **Ultimate Multisite > Settings > Payment Gateways** ലേക്ക് പോകുക.
2. [Stripe setup guide](/user-guide/payment-gateways/stripe) അല്ലെങ്കിൽ നിങ്ങൾക്ക് ഇഷ്ടമുള്ള ഗേറ്റ്‌വേ പിന്തുടരുക.
3. ഒരു ടെസ്റ്റ് പേയ്‌മെന്റോടുകൂടിയ പൂർണ്ണമായ చెക്ക്ഔട്ട് പ്രവാഹം പരീക്ഷിക്കുക.

പണം നിങ്ങളുടെ അക്കൗണ്ടിലേക്ക് എങ്ങനെ എത്തുന്നു എന്നതിനെക്കുറിച്ചുള്ള വിശദാംശങ്ങൾക്കായി [Getting Paid](/user-guide/payment-gateways/getting-paid) കാണുക.

## പ്രവാഹം പരീക്ഷിക്കൽ (Testing the Flow) {#testing-the-flow}

തുടരുന്നതിന് മുമ്പ്, ഒരു പൂർണ്ണമായ ടെസ്റ്റ് സൈൻഅപ്പ് പൂർത്തിയാക്കുക:

1. ഒരു ഇൻകോണിറ്റോ/പ്രൈവറ്റ് ബ്രൗസർ വിൻഡോയിൽ చెക്ക്ഔട്ട് ഫോം തുറക്കുക.
2. ഒരു പ്ലാൻ തിരഞ്ഞെടുക്കുക.
3. ഒരു ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കുക.
4. ഒരു അക്കൗണ്ട് സൃഷ്ടിക്കുക.
5. പേയ്‌മെന്റ് പൂർത്തിയാക്കുക (ടെസ്റ്റ് മോഡ് ഉപയോഗിക്കുക).
6. ശരിയായ ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് സൈറ്റ് സൃഷ്ടിക്കപ്പെട്ടിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക.

ഇവ പരിശോധിക്കുക:

- [ ] പ്ലാൻ വിവരണങ്ങൾ വ്യക്തവും പ്രത്യേക മേഖലയ്ക്ക് അനുയോജ്യവുമാണോ
- [ ] ടെംപ്ലേറ്റ് പ്രിവ്യൂകൾ ഫിറ്റ്നസ് അനുയോജ്യമായ ഡിസൈനുകൾ കാണിക്കുന്നുണ്ടോ
- [ ] സൈറ്റ് URL സ്റ്റുഡിയോ പേരിൽ നിന്ന് ശരിയായി സൃഷ്ടിക്കപ്പെടുന്നുണ്ടോ
- [ ] പേയ്‌മെന്റ് വിജയകരമായി നടക്കുന്നുണ്ടോ
- [ ] ഉപഭോക്താവ് തിരഞ്ഞെടുത്ത ടെംപ്ലേറ്റ് ഉപയോഗിച്ച് പ്രവർത്തിക്കുന്ന സൈറ്റിൽ എത്തുന്നുണ്ടോ
- [ ] സ്ഥിരീകരണ ഇമെയിലുകൾ ഫിറ്റ്നസ്-നിർദ്ദിഷ്ട ഭാഷ ഉപയോഗിക്കുന്നുണ്ടോ

## FitSite നെറ്റ്‌വർക്ക് ഇതുവരെ ചെയ്തത് {#the-fitsite-network-so-far}

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow
│   ├── Plan selection with niche-specific descriptions
│   ├── Template selection with fitness previews
│   ├── Minimal account creation
│   ├── Payment via Stripe
│   └── Fitness-specific confirmation
└── Ready for branding (next lesson)
```

## ഈ പാഠത്തിൽ നമ്മൾ നിർമ്മിച്ചത് {#what-we-built-this-lesson}

- ഫിറ്റ്നസ് സ്റ്റുഡിയോ ഉടമകൾക്കായി രൂപകൽപ്പന ചെയ്ത ഒരു മൾട്ടി-സ്റ്റെപ്പ് చెക്ക്ഔട്ട് ഫോം
- സൈൻഅപ്പ് പ്രവാസം മുഴുവൻ പ്രത്യേക മേഖലയുടെ ഭാഷ
- കുറഞ്ഞ തടസ്സം -- അത്യാവശ്യമായ ഫീൽഡുകൾ മാത്രം, പ്രവർത്തിക്കുന്ന സൈറ്റിലേക്കുള്ള വേഗതയേറിയ വഴി
- ടെസ്റ്റ് വെരിഫിക്കേഷനോടെ പേയ്‌മെന്റ് സംയോജനം
- പ്ലാൻ തിരഞ്ഞെടുപ്പിൽ നിന്ന് പ്രവർത്തിക്കുന്ന സൈറ്റ് വരെയുള്ള പരീക്ഷിച്ച എൻഡ്-ടു-എൻഡ് പ്രവാഹം
