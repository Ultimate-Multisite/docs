# പാഠം 3: നിങ്ങളുടെ നെറ്റ്‌വർക്ക് സജ്ജമാക്കൽ

ഇനി നിർമ്മിക്കാനുള്ള സമയമാണ്. ഈ പാഠത്തിൽ, നിങ്ങൾ Ultimate Multisite ഇൻസ്റ്റാൾ ചെയ്യുകയും FitSite നെറ്റ്‌വർക്കിന്റെ അടിത്തറ സജ്ജമാക്കുകയും ചെയ്യും. ഇവിടെ എടുക്കുന്ന ഓരോ തീരുമാനവും ഫിറ്റ്നസ് മേഖലയെ മനസ്സിൽ വെച്ചാണ് എടുക്കുന്നത്.

## എവിടെ വെച്ച് നിർത്തി? {#where-we-left-off}

നമ്മൾ ഫിറ്റ്നസ് സ്റ്റുഡിയോകളെയാണ് നമ്മുടെ niche ആയി തിരഞ്ഞെടുത്തതെന്നും, ആ അവസരം സാധൂകരിച്ചുവെന്നും മനസ്സിലാക്കി. ഇപ്പോൾ ആ ആശയം ഒരു പ്രവർത്തിക്കുന്ന പ്ലാറ്റ്‌ഫോമായി മാറ്റുകയാണ്.

## ഹോസ്റ്റിംഗ് തിരഞ്ഞെടുക്കൽ {#choosing-your-hosting}

ഒരു single വെബ്സൈറ്റിന് വേണ്ടി മാത്രമല്ല, ഒരു niche പ്ലാറ്റ്‌ഫോമിന് വേണ്ടി നിങ്ങളുടെ ഹോസ്റ്റിംഗ് തിരഞ്ഞെടുക്കുന്നത് വളരെ പ്രധാനമാണ്. നിങ്ങൾ ഒരു സൈറ്റ് ഹോസ്റ്റ് ചെയ്യുന്നില്ല — നിങ്ങൾ ഡസൻ കണക്കിന് അല്ലെങ്കിൽ നൂറുകണക്കിന് സൈറ്റുകളിലേക്ക് വളരാൻ പോകുന്ന ഒരു നെറ്റ്‌വർക്ക് ഹോസ്റ്റ് ചെയ്യുകയാണ്.

### എന്ത് ശ്രദ്ധിക്കണം {#what-to-look-for}

- **WordPress Multisite support**: എല്ലാ ഹോസ്റ്റുകളും multisite ശരിയായി കൈകാര്യം ചെയ്യുന്നില്ല.
- **Wildcard SSL**: subdomain അടിസ്ഥാനമാക്കിയുള്ള നെറ്റ്‌വർക്കുകൾക്ക് ഇത് അത്യന്താപേക്ഷിതമാണ്.
- **Scalable resources**: മൈഗ്രേറ്റ് ചെയ്യാതെ വളരാനുള്ള സൗകര്യം നിങ്ങൾക്ക് വേണം.
- **Ultimate Multisite integration**: ഓട്ടോമേറ്റഡ് ഡൊമെയ്ൻ മാപ്പിംഗും SSL-ഉം വലിയ പ്രവർത്തനപരമായ പരിശ്രമം ലാഭിക്കുന്നു.

### ശുപാർശ ചെയ്യുന്ന സമീപനം {#recommended-approach}

[Compatible Providers](/user-guide/host-integrations/closte) ലിസ്റ്റിൽ നിന്ന് ഒരു ഹോസ്റ്റ് തിരഞ്ഞെടുക്കുക. ഇവ Ultimate Multisite ഉപയോഗിച്ച് പരീക്ഷിച്ചവയാണ്, കൂടാതെ ഡൊമെയ്ൻ മാപ്പിംഗിനും SSL ഓട്ടോമേഷനുമുള്ള ആവശ്യമായ ഇന്റഗ്രേഷനുകൾ ഇവ നൽകുന്നു.

FitSite-ന് വേണ്ടി, നമ്മൾ ഒരു subdomain configuration ആണ് ഉപയോഗിക്കാൻ പോകുന്നത്. ഇതിനർത്ഥം, ഉപഭോക്താക്കളുടെ സൈറ്റുകൾ ആദ്യം `studioname.fitsite.com` എന്ന രൂപത്തിൽ കാണപ്പെടും, അതിനുശേഷം അവർക്ക് സ്വന്തമായി ഡൊമെയ്ൻ മാപ്പ് ചെയ്യാൻ ഓപ്ഷൻ ലഭിക്കും.

## WordPress Multisite ഇൻസ്റ്റാൾ ചെയ്യൽ {#installing-wordpress-multisite}

നിങ്ങൾക്ക് ഇതിനകം ഒരു WordPress Multisite ഇൻസ്റ്റാളേഷൻ ഇല്ലെങ്കിൽ:

1. നിങ്ങളുടെ ഹോസ്റ്റിംഗ് പ്രൊവൈഡറിൽ WordPress ഇൻസ്റ്റാൾ ചെയ്യുക.
2. [How to Install WordPress Multisite](/user-guide/getting-started/how-to-install-wordpress-multisite) ഗൈഡ് പിന്തുടരുക.
3. ആവശ്യപ്പെടുമ്പോൾ **subdomain** configuration തിരഞ്ഞെടുക്കുക.

:::tip എന്തുകൊണ്ട് Subdomains?
Subdomains ഓരോ ഉപഭോക്താ സൈറ്റിനും അതിന്റേതായ പ്രത്യേക വിലാസം (`studio.fitsite.com`) നൽകുന്നു, ഒരു path (`fitsite.com/studio`) എന്നതിനേക്കാൾ. ഇത് നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് കൂടുതൽ പ്രൊഫഷണലാണ്, കൂടാതെ permalink conflict-കൾ ഒഴിവാക്കാനും ഇത് സഹായിക്കുന്നു. വിശദമായ താരതമ്യ പഠനത്തിനായി [Ultimate Multisite 101](/user-guide/getting-started/ultimate-multisite-101) കാണുക.
:::

## Ultimate Multisite ഇൻസ്റ്റാൾ ചെയ്യൽ {#installing-ultimate-multisite}

താഴെ പറയുന്ന കാര്യങ്ങൾ ചെയ്യാൻ [Installing Ultimate Multisite](/user-guide/getting-started/installing-ultimate-multisite) ഗൈഡ് പിന്തുടരുക:

1. പ്ലഗിൻ നെറ്റ്‌വർക്ക് മുഴുവൻ അപ്‌ലോഡ് ചെയ്യുകയും ആക്ടിവേറ്റ് ചെയ്യുകയും ചെയ്യുക.
2. [Setup Wizard](/user-guide/getting-started/multisite-setup-wizard) പ്രവർത്തിപ്പിക്കുക.

സെറ്റപ്പ് വിസാർഡിൽ, FitSite niche നെ മനസ്സിൽ വെക്കുക:

- **Currency**: നിങ്ങളുടെ ഫിറ്റ്നസ് സ്റ്റുഡിയോ ഉപഭോക്താക്കൾ ഉപയോഗിക്കുന്ന കറൻസിയിലേക്ക് ഇത് സജ്ജമാക്കുക.
- **Company name**: "FitSite" (അല്ലെങ്കിൽ നിങ്ങൾ തിരഞ്ഞെടുത്ത ബ്രാൻഡ് നാമം).
- **Company logo**: നിങ്ങളുടെ ബ്രാൻഡ് ലോഗോ അപ്‌ലോഡ് ചെയ്യുക -- ഇത് ഇൻവോയ്സുകളിലും ഇമെയിലുകളിലും കാണിക്കും.

## ഫിറ്റ്നസ് niche-ന് വേണ്ടി സജ്ജമാക്കൽ {#configuring-for-the-fitness-niche}

Ultimate Multisite ഇൻസ്റ്റാൾ ചെയ്ത ശേഷം, ഈ niche-ന് പ്രത്യേകമായ ചില കോൺഫിഗറേഷൻ മാറ്റങ്ങൾ വരുത്തുക:

### പൊതുവായ ക്രമീകരണങ്ങൾ (General Settings) {#general-settings}

**Ultimate Multisite > Settings** എന്നതിലേക്ക് പോകുക, അവിടെ താഴെ പറയുന്നവ സജ്ജമാക്കുക:

- **Site name**: FitSite
- **Default role**: Administrator -- ഫിറ്റ്നസ് സ്റ്റുഡിയോ ഉടമകൾക്ക് അവരുടെ സൈറ്റ് ഉള്ളടക്കത്തിന്റെ പൂർണ്ണ നിയന്ത്രണം ആവശ്യമാണ്.
- **Registration**: സ്റ്റുഡിയോ ഉടമകൾക്ക് സ്വയം സൈൻ അപ്പ് ചെയ്യാൻ കഴിയുന്നതിനായി യൂസർ രജിസ്ട്രേഷൻ പ്രവർത്തനക്ഷമമാക്കുക.

### ഇമെയിൽ കോൺഫിഗറേഷൻ (Email Configuration) {#email-configuration}

നിങ്ങളുടെ സിസ്റ്റം ഇമെയിലുകൾ നിങ്ങളുടെ niche-യുടെ ഭാഷ സംസാരിക്കണം. **Ultimate Multisite > Settings > Emails** എന്നതിലേക്ക് പോയി ഇത് കസ്റ്റമൈസ് ചെയ്യുക:

- പൊതുവായ "your new site" ഭാഷയെ ഫിറ്റ്നസ്-നിർദ്ദിഷ്ട സന്ദേശങ്ങളോടെ മാറ്റിസ്ഥാപിക്കുക.
- ഉദാഹരണ സ്വീകാര്യ സബ്ജക്റ്റ്: "നിങ്ങളുടെ ഫിറ്റ്നസ് സ്റ്റുഡിയോ വെബ്സൈറ്റ് തയ്യാറാണ്"
- ഉദാഹരണ സ്വീകാര്യ ബോഡി: അവരുടെ സ്റ്റുഡിയോയെക്കുറിച്ചും, ക്ലാസുകളെക്കുറിച്ചും, അവരുടെ ഫിറ്റ്നസ് സൈറ്റുമായി എങ്ങനെ തുടങ്ങണം എന്നതിനെക്കുറിച്ചും പരാമർശിക്കുക.

ഇത് പാഠം 8-ൽ (Customer Onboarding) നമ്മൾ കൂടുതൽ മെച്ചപ്പെടുത്തും, പക്ഷേ ഇപ്പോൾ തന്നെ ടോൺ സജ്ജമാക്കുന്നത് ആദ്യത്തെ ടെസ്റ്റ് സൈൻ അപ്പുകൾ പോലും niche-specific ആണെന്ന് തോന്നാൻ സഹായിക്കും.

### ഡൊമെയ്ൻ കോൺഫിഗറേഷൻ (Domain Configuration) {#domain-configuration}

ഒരു compatible hosting provider ഉപയോഗിക്കുകയാണെങ്കിൽ, ഡൊമെയ്ൻ മാപ്പിംഗ് ഇപ്പോൾ സജ്ജമാക്കുക:

1. **Ultimate Multisite > Settings > Domain Mapping** എന്നതിലേക്ക് പോകുക.
2. നിങ്ങളുടെ പ്രത്യേക ഹോസ്റ്റിനുള്ള ഇന്റഗ്രേഷൻ ഗൈഡ് പിന്തുടരുക.
3. പുതിയ സബ്സൈറ്റുകൾക്ക് SSL ഓട്ടോമാറ്റിക്കായി ലഭിക്കുന്നുണ്ടോ എന്ന് പരീക്ഷിക്കുക.

ഇത് ഉറപ്പാക്കുന്നു, അടുത്ത പാഠത്തിൽ നമ്മൾ ടെംപ്ലേറ്റുകളും ടെസ്റ്റ് സൈറ്റുകളും ഉണ്ടാക്കാൻ തുടങ്ങുമ്പോൾ, എല്ലാം end to end പ്രവർത്തിക്കുന്നുണ്ടെന്ന്.

## FitSite നെറ്റ്‌വർക്ക് ഇതുവരെ എന്ത്? {#the-fitsite-network-so-far}

ഈ പാഠത്തിന്റെ അവസാനം, നിങ്ങൾക്ക് ലഭിച്ചിരിക്കുന്നത് ഇതാണ്:

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (installed and configured)
├── Hosting with wildcard SSL
├── Domain mapping configured
├── Niche-specific email templates (initial)
└── Ready for site templates (next lesson)
```

## ഈ പാഠത്തിൽ നമ്മൾ എന്ത് നിർമ്മിച്ചു? {#what-we-built-this-lesson}

- **subdomain mode-ൽ പ്രവർത്തിക്കുന്ന ഒരു WordPress Multisite** ഇൻസ്റ്റാളേഷൻ.
- **FitSite ബ്രാൻഡിംഗോടുകൂടി Ultimate Multisite ഇൻസ്റ്റാൾ** ചെയ്ത് കോൺഫിഗർ ചെയ്തു.
- വളരുന്ന നെറ്റ്‌വർക്കിനായി **ഹോസ്റ്റിംഗും SSL-ഉം** സജ്ജമാക്കി.
- നിങ്ങളുടെ ഹോസ്റ്റിംഗ് പ്രൊവൈഡറിനായി **ഡൊമെയ്ൻ മാപ്പിംഗ്** കോൺഫിഗർ ചെയ്തു.
- ആദ്യ ദിവസം മുതൽ **niche-specific ഇമെയിൽ ടോൺ** സ്ഥാപിച്ചു.
