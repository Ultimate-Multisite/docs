# Captcha {#captcha}

**ബോട്ട്‌കളെ തടയൂ. വരുമാനം സംരക്ഷിക്കൂ. കോൺഫിഗറേഷൻ ബുദ്ധിമുട്ടുകൾ പൂജ്യം.**

കസ്റ്റം ഡൊമെയ്‌നുകളോടുകൂടിയ ഒരു മൾട്ടിസൈറ്റ് നെറ്റ്‌വർക്ക് (multisite network) പ്രവർത്തിപ്പിക്കുകയാണോ? നിങ്ങൾക്ക് ഈ ബുദ്ധിമുട്ട് അറിയാം: Google reCAPTCHA, hCaptcha എന്നിവയുടെ അഡ്മിൻ കൺസോളിൽ *ഓരോ ഡൊമെയ്‌നവും* രജിസ്റ്റർ ചെയ്യണമെന്ന് ആവശ്യപ്പെടുന്നു. ഒരു പുതിയ കസ്റ്റമർ സൈറ്റ് കൂട്ടിച്ചേർക്കുമ്പോൾ? കാപ്‌ച കൺഫിഗർ ചെയ്യണം. ഒരു കസ്റ്റം ഡൊമെയ്ൻ മാപ്പ് ചെയ്യുമ്പോൾ? വീണ്ടും അപ്‌ഡേറ്റ് ചെയ്യണം. ഇത് ഒരിക്കലും അവസാനിക്കുന്നില്ല.

**Ultimate Multisite: Captcha എല്ലാം മാറ്റിമറിക്കുന്നു.**

## Cap Captcha: സ്വയം ഹോസ്റ്റ് ചെയ്യുന്ന സംരക്ഷണം, അത് പ്രവർത്തിപ്പിക്കുക മാത്രം ചെയ്താൽ മതി {#cap-captcha-self-hosted-protection-that-just-works}

ഞങ്ങളുടെ പുതിയ **Cap Captcha** പ്രൊവൈഡർ മൾട്ടിസൈറ്റ് നെറ്റ്‌വർക്കുകൾക്ക് ഒരു വഴിത്തിരിവാണ്:

* **സീറോ കോൺഫിഗറേഷൻ** — ഈ ആഡ്‌ഓൺ ആക്ടിവേറ്റ് ചെയ്താൽ മതി, നിങ്ങൾക്ക് സംരക്ഷണം ലഭിക്കും. API കീകളോ, Google അക്കൗണ്ടോ, hCaptcha ഡാഷ്‌ബോർഡോ, ഡൊമെയ്ൻ വൈറ്റ്‌ലിസ്റ്റിംഗോ ആവശ്യമില്ല.
* **എല്ലാ ഡൊമെയ്‌നുകളിലും ഓട്ടോമാറ്റിക്കായി പ്രവർത്തിക്കുന്നു** — സബ്ഡൊമെയ്‌നുകൾ, മാപ്പ് ചെയ്ത ഡൊമെയ്‌നുകൾ, കസ്റ്റമർ ഡൊമെയ്‌നുകൾ—ഇവയെല്ലാം തൽക്ഷണം സംരക്ഷിക്കപ്പെടുന്നു.
* **സ്വയം ഹോസ്റ്റ് ചെയ്തതും സ്വകാര്യവും** — നിങ്ങളുടെ ഡാറ്റ ഒരിക്കലും നിങ്ങളുടെ സെർവറിൽ നിന്ന് പുറത്തേക്ക് പോവില്ല. മൂന്നാം കക്ഷി ട്രാക്കിംഗ് ഇല്ലാതെ പൂർണ്ണമായും GDPR പറ്റി.
* **അദൃശ്യമായ പ്രൂഫ്-ഓഫ്-വർക്ക്** — ഉപയോക്താക്കൾ പശ്ചാത്തലത്തിൽ ഒരു കമ്പ്യൂട്ടേഷണൽ പസിൽ പരിഹരിക്കുന്നു. ഫയർ ഹൈഡ്രന്റുകൾ ക്ലിക്ക് ചെയ്യേണ്ടതില്ല, വിരസമായ ചിത്രങ്ങളുടെ ഗ്രീഡുകൾ കാണേണ്ടതില്ല.
* **ഭാരം കുറഞ്ഞത്** — വെറും 20KB മാത്രം. നിങ്ങളുടെ ചെക്ക്ഔട്ടിനെ മന്ദഗതിയിലാക്കില്ല.

## WooCommerce കാർഡ് ടെസ്റ്റിംഗ് സംരക്ഷണം {#woocommerce-card-testing-protection}

കാർഡ് ടെസ്റ്റിംഗ് ആക്രമണങ്ങൾ വ്യാപാരികൾക്ക് വാർഷികമായി കോടിക്കണക്കിന് രൂപ നഷ്ടമുണ്ടാക്കുന്നു. വ്യാജർമാർ നിങ്ങളുടെ WooCommerce ചെക്ക്ഔട്ടിൽ നിന്ന് മോഷ്ടിച്ച ക്രെഡിറ്റ് കാർഡുകൾ സാധുവാണോ എന്ന് പരിശോധിക്കാൻ ബോട്ടുകൾ ഉപയോഗിക്കുന്നു—നിങ്ങൾ പ്രോസസ്സിംഗ് ഫീസ് നൽകുന്നു, അവർക്ക് സാധുവായ കാർഡ് നമ്പറുകൾ ലഭിക്കുന്നു.

**ഏറ്റവും പ്രധാനപ്പെട്ട സ്ഥലത്താണ് ഞങ്ങൾ സംരക്ഷിക്കുന്നത്:**

* **സ്റ്റോർ API സംരക്ഷണം** — പരമ്പരാഗത ഫോം കാപ്‌ചകൾ മറികടക്കുന്ന `/wp-json/wc/store/v1/checkout` എന്നതിലെ ബോട്ട് ആക്രമണങ്ങൾ തടയുന്നു.
* **PayPal പേയ്‌മെന്റുകളുമായി പൊരുത്തം** — WooCommerce PayPal Payments-മായി തടസ്സമില്ലാതെ പ്രവർത്തിക്കുന്നു.
* **റിയൽ-ടൈം സ്റ്റാറ്റിസ്റ്റിക്സ്** — നിങ്ങൾ എത്ര ആക്രമണങ്ങൾ തടയുന്നു എന്ന് കൃത്യമായി കാണാം.

## Google reCAPTCHA അല്ലെങ്കിൽ hCaptcha ഇപ്പോഴും വേണോ? {#still-need-google-recaptcha-or-hcaptcha}

ഞങ്ങൾ നിങ്ങൾക്ക് സഹായം നൽകുന്നു. ഇവയ്ക്ക് പൂർണ്ണ പിന്തുണയുണ്ട്:

* Google reCAPTCHA v2 (Checkbox)
* Google reCAPTCHA v2 (Invisible)
* Google reCAPTCHA v3 (Score-based)
* hCaptcha (Standard)
* hCaptcha (Invisible)

നിങ്ങളുടെ നെറ്റ്‌വർക്ക് സെറ്റിംഗ്സിൽ നിന്ന് എപ്പോൾ വേണമെങ്കിലും പ്രൊവൈഡർ മാറ്റാൻ കഴിയും.

## വളർച്ചയ്ക്കായി നിർമ്മിച്ചത് (Built for Scale) {#built-for-scale}

* സൈറ്റ് തിരിച്ചുള്ള ഓവർറൈഡുകളോടുകൂടിയ നെറ്റ്‌വർക്ക്-വ്യാപക ക്രമീകരണങ്ങൾ
* സംരക്ഷണം നിരീക്ഷിക്കാനുള്ള സ്റ്റാറ്റിസ്റ്റിക്സ് ഡാഷ്‌ബോർഡ്
* ക്രമീകരിക്കാവുന്ന സുരക്ഷാ തലങ്ങൾ (Fast, Medium, Maximum)
* വിപുലീകരിക്കാവുന്ന പ്രൊവൈഡർ ആർക്കിടെക്ചർ ഉള്ള ഡെവലപ്പർ-കൂഹൃദപൂർവ്വം

## ഇൻസ്റ്റാളേഷൻ {#installation}

1. `ultimate-multisite-captcha` നിങ്ങളുടെ `/wp-content/plugins/` ഡയറക്‌ടറിയിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക
2. നെറ്റ്‌വർക്കിൽ പ്ലഗിൻ ആക്ടിവേറ്റ് ചെയ്യുക
3. **അത്രയേ ഉള്ളൂ. നിങ്ങൾക്ക് സംരക്ഷണം ലഭിച്ചു.**

Cap Captcha ഓട്ടോമാറ്റിക്കായി ആക്ടിവേറ്റ് ചെയ്യും—ക്രമീകരിക്കേണ്ട സെറ്റിംഗുകളോ, നൽകേണ്ട API കീകളോ ഇല്ല. നിങ്ങളുടെ മുഴുവൻ നെറ്റ്‌വർക്കിലുമുള്ള എല്ലാ പിന്തുണയ്ക്കുന്ന എൻഡ്‌പോയിന്റുകളും നിങ്ങൾ ആക്ടിവേറ്റ് ചെയ്യുന്ന നിമിഷം മുതൽ സംരക്ഷിക്കപ്പെടുന്നു.

**ഓപ്ഷണൽ:** പകരം Google reCAPTCHA അല്ലെങ്കിൽ hCaptcha ഉപയോഗിക്കാൻ, **Ultimate Multisite → Settings → Captcha** എന്നതിലേക്ക് പോയി നിങ്ങൾക്ക് ഇഷ്ടമുള്ള പ്രൊവൈഡർ തിരഞ്ഞെടുക്കുക.

## പതിവായി ചോദിക്കുന്ന ചോദ്യങ്ങൾ (Frequently Asked Questions) {#frequently-asked-questions}

## എനിക്ക് എന്തെങ്കിലും കോൺഫിഗർ ചെയ്യേണ്ടതുണ്ടോ? {#do-i-need-to-configure-anything}

ഇല്ല! ആഡ്‌ഓൺ ആക്ടിവേറ്റ് ചെയ്യുമ്പോൾ Cap Captcha ഓട്ടോമാറ്റിക്കായി എനേബിൾ ചെയ്യപ്പെടും. നിങ്ങളുടെ മുഴുവൻ നെറ്റ്‌വർക്കുമൊന്ന് ഉടൻ തന്നെ സംരക്ഷിക്കപ്പെടുന്നു—ക്രമീകരിക്കേണ്ട സെറ്റിംഗുകളോ, നൽകേണ്ട API കീകളോ ഇല്ല.

## Cap Captcha-ക്ക് എനിക്ക് API കീകൾ ആവശ്യമുണ്ടോ? {#do-i-need-api-keys-for-cap-captcha}

ഇല്ല. Cap Captcha പൂർണ്ണമായും സ്വയം ഹോസ്റ്റ് ചെയ്തതാണ്. ബാഹ്യ അക്കൗണ്ടുകളോ, ഡൊമെയ്ൻ രജിസ്ട്രേഷനോ, കോൺഫിഗറേഷൻ തലവേദനകളോ ഇല്ല.

## ഇത് എന്റെ ചെക്ക്ഔട്ടിനെ മന്ദഗതിയിലാക്കുമോ? {#will-this-slow-down-my-checkout}

Cap Captcha വെറും 20KB മാത്രമാണ്, കൂടാതെ ഇത് പശ്ചാത്തലത്തിൽ പ്രൂഫ്-ഓഫ്-വർക്ക് ചലഞ്ചുകൾ പ്രവർത്തിപ്പിക്കുന്നു. നിങ്ങളുടെ ഉപഭോക്താക്കൾ യാതൊരു കാലതാമസവും ശ്രദ്ധിക്കില്ല.

## ഇത് കാർഡ് ടെസ്റ്റിംഗ് ആക്രമണങ്ങൾ തടയുമോ? {#does-this-protect-against-card-testing-attacks}

അതെ! HTML ഫോമുകൾ മാത്രം സംരക്ഷിക്കുന്ന പരമ്പരാഗത കാപ്‌ചകളിൽ നിന്ന് വ്യത്യസ്തമായി, ഞങ്ങളുടെ WooCommerce സംയോജനം (integration) സ്റ്റോർ API-യിലേക്ക് നേരിട്ട് ബന്ധിപ്പിക്കുകയും (hooks) ബോട്ട് ആക്രമണങ്ങൾ API തലത്തിൽ തടയുകയും ചെയ്യുന്നു—ഇവിടെയാണ് മിക്ക കാർഡ് ടെസ്റ്റിംഗ് ആക്രമണങ്ങളും നടക്കുന്നത്.

## വ്യത്യസ്ത സൈറ്റുകളിൽ വ്യത്യസ്ത കാപ്‌ചാ പ്രൊവൈഡർ ഉപയോഗിക്കാമോ? {#can-i-use-different-captcha-providers-on-different-sites}

കാപ്‌ചാ പ്രൊവൈഡർ നെറ്റ്‌വർക്ക് മുഴുവൻ രീതിയിൽ സജ്ജമാക്കിയതാണ്, പക്ഷേ നിങ്ങളുടെ ആവശ്യങ്ങൾക്കനുസരിച്ച് നിങ്ങൾക്ക് കഠിനതാ ക്രമീകരണങ്ങൾ (difficulty settings) ക്രമീകരിക്കാൻ കഴിയും.

## ഇത് GDPR പറ്റിですか? {#is-this-gdpr-compliant}

Cap Captcha 100% സ്വയം ഹോസ്റ്റ് ചെയ്തതാണ്, ബാഹ്യ സേവനങ്ങളിലേക്ക് ഡാറ്റ അയക്കുന്നില്ല. Google reCAPTCHA, hCaptcha എന്നിവയ്ക്ക്, ദയവായി അവയുടെ ബന്ധപ്പെട്ട സ്വകാര്യ നയങ്ങൾ (privacy policies) പരിശോധിക്കുക.

## റേറ്റ് ലിമിറ്റിംഗ് (Rate Limiting) {#rate-limiting}

Ultimate Multisite: Captcha v1.5.0 ഒരു ഹാർഡ്-സ്റ്റോപ്പ് റേറ്റ് ലിമിറ്റർ അവതരിപ്പിക്കുന്നു. ഇത് ബ്രൂട്ട്-ഫോഴ്സ് (brute-force) കൂടാതെ കാർഡ്-ടെസ്റ്റിംഗ് ആക്രമണങ്ങളിൽ നിന്ന് നിങ്ങളുടെ നെറ്റ്‌വർക്കിനെ സംരക്ഷിക്കുന്നു.

### ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു {#how-it-works}

റേറ്റ് ലിമിറ്റർ കാപ്‌ചാ സംരക്ഷിത സറഫേസുകളിലെ (captcha-protected surfaces) എല്ലാ GET, POST അഭ്യർത്ഥനകളെയും (requests) എണ്ണുന്നു:

* **WordPress ലോഗിൻ എൻഡ്‌പോയിന്റുകൾ** — wp-login, register, lost-password, comments
* **WooCommerce ചെക്ക്ഔട്ട്** — my-account, checkout, pay-for-order
* **Ultimate Multisite** — checkout, inline-login

ഒരു സന്ദർശകൻ റേറ്റ് ലിമിറ്റ്超過 ചെയ്യുമ്പോൾ, പ്ലഗിൻ താഴെ പറയുന്നവ ഉപയോഗിച്ച് പ്രതികരിക്കുന്നു:

* **HTTP 429 (Too Many Requests)** സ്റ്റാറ്റസ് കോഡ്
* **Retry-After** ഹെഡർ (എപ്പോൾ വീണ്ടും ശ്രമിക്കണം എന്ന് സൂചിപ്പിക്കുന്നു)
* **Randomised tarpit sleep** (ഡിഫോൾട്ടായി 1–5 സെക്കൻഡ്, പരമാവധി 15 സെക്കൻഡിൽ പരിമിതപ്പെടുത്തി) ആക്രമണകാരികളെ മന്ദഗതിയിലാക്കാൻ.

### കോൺഫിഗറേഷൻ {#configuration}

#### ടാർപിറ്റ് ടൈമിംഗ് (Tarpit Timing) {#tarpit-timing}

റേറ്റ് ലിമിറ്റ് ചെയ്ത അഭ്യർത്ഥനകൾക്ക് ബാധകമാക്കുന്ന ക്രമരഹിതമായ കാലതാമസം നിയന്ത്രിക്കുക:

* **`cap_rate_limit_tarpit_min`** — സെക്കൻഡുകളിലെ കുറഞ്ഞ സ്ലീപ്പ് സമയം (ഡിഫോൾട്ട്: 1)
* **`cap_rate_limit_tarpit_max`** — സെക്കൻഡുകളിലെ കൂടിയ സ്ലീപ്പ് സമയം (ഡിഫോൾട്ട്: 5, പരമാവധി 15)

ഈ ക്രമീകരണങ്ങൾ **Ultimate Multisite → Settings → Captcha** എന്നതിൽ ലഭ്യമാണ്.

### വിപുലീകരിക്കാനുള്ള കഴിവ് (Extensibility) {#extensibility}

#### IP വൈറ്റ്‌ലിസ്റ്റ് ഫിൽട്ടർ (IP Whitelist Filter) {#ip-whitelist-filter}

വിശ്വസ്തരായ IP റേഞ്ചുകളെ റേറ്റ് ലിമിറ്റിംഗിൽ നിന്ന് ഒഴിവാക്കുക:

```php
add_filter( 'wu_cap_rate_limit_whitelist_ip', function( $is_whitelisted, $ip ) {
    // നിങ്ങളുടെ നിരീക്ഷണ സേവനം വൈറ്റ്‌ലിസ്റ്റ് ചെയ്യുക
    if ( $ip === '203.0.113.42' ) {
        return true;
    }
    return $is_whitelisted;
}, 10, 2 );
```

#### പ്രീ-ബ്ലോക്ക് ആക്ഷൻ (Pre-Block Action) {#pre-block-action}

റേറ്റ് ലിമിറ്റ് ബ്ലോക്ക് അയക്കുന്നതിന് തൊട്ടുമുമ്പ് കസ്റ്റം ലോജി എക്സിക്യൂട്ട് ചെയ്യുക:

```php
add_action( 'wu_cap_rate_limit_will_block', function( $ip, $surface ) {
    // ബ്ലോക്ക് രേഖപ്പെടുത്തുക, അഡ്മിൻമാരെ അറിയിക്കുക, അല്ലെങ്കിൽ കസ്റ്റം മെട്രിക്സ് അപ്ഡേറ്റ് ചെയ്യുക
    error_log( "Rate limit triggered for $ip on $surface" );
}, 10, 2 );
```

## ക്ലയിന്റ് IP കണ്ടെത്തലും വിശ്വസ്ത പ്രോക്സികളും (Client IP Detection & Trusted Proxies) {#client-ip-detection--trusted-proxies}

Ultimate Multisite: Captcha v1.5.0 സ്പൂഫ്-റെസിസ്റ്റന്റ് ക്ലയിന്റ്-IP കണ്ടെത്തൽ അവതരിപ്പിക്കുന്നു. `Captcha_Core::get_client_ip()` എന്ന രീതി ഇപ്പോൾ താഴെ പറയുന്നവയിലുടനീളം സന്ദർശക IP ആട്രിബ്യൂഷনের ഏക സത്യ സ്രോതസ്സാണ്:

* റേറ്റ്-ലിമിറ്റ് ബക്കറ്റ് കീകൾ
* കാപ്‌ചാ പ്രൊവൈഡർ `remoteip` (reCAPTCHA, hCaptcha `siteverify`)
* സ്റ്റാറ്റിസ്റ്റിക്സ് IP ഹാഷുകൾ

### വിശ്വാസ മാതൃക (Trust Model) {#trust-model}

പ്ലഗിൻ ഒരു കർശനമായ IP വിശ്വാസ ശ്രേണി നടപ്പിലാക്കുന്നു:

1. **REMOTE_ADDR** — എപ്പോഴും വിശ്വസനീയമാണ് (ഉടനെയുള്ള പിയറിൻ്റെ IP)
2. **CF-Connecting-IP** — ഈ സാഹചര്യങ്ങളിൽ മാത്രം ബഹുമാനിക്കപ്പെടുന്നു:
   - ഉടനെയുള്ള പിയർ നിലവിലെ Cloudflare IP റേഞ്ചിൽ ഉള്ളതാണെങ്കിൽ
   - `cap_trust_cloudflare_headers` എനേബിൾ ചെയ്തിട്ടുണ്ടെങ്കിൽ (ഡിഫോൾട്ട്: OFF)
3. **X-Forwarded-For** — ഈ സാഹചര്യങ്ങളിൽ മാത്രം ബഹുമാനിക്കപ്പെടുന്നു:
   - ഉടനെയുള്ള പിയർ നിങ്ങളുടെ അഡ്മിൻ-കോൺഫിഗർ ചെയ്ത വിശ്വസ്ത പ്രോക്സി ലിസ്റ്റിൽ ആണെങ്കിൽ
   - ഹെഡർ വലതുവശത്തുനിന്ന് ഇടത്തോട്ട് പാർസ് ചെയ്യണം, വിശ്വസ്തമായ/Cloudflare ഹോപ്പുകൾ ഒഴിവാക്കണം

### കോൺഫിഗറേഷൻ {#configuration-1}

#### Cloudflare ഹെഡർ വിശ്വാസം (Cloudflare Header Trust) {#cloudflare-header-trust}

Cloudflare-ൻ്റെ `CF-Connecting-IP` ഹെഡർ വിശ്വസിക്കാൻ എനേബിൾ ചെയ്യുക:

* **`cap_trust_cloudflare_headers`** — `CF-Connecting-IP` വിശ്വാസത്തിലേക്ക് ഓപ്റ്റ് ഇൻ ചെയ്യാൻ ON ആക്കുക
  - ഡിഫോൾട്ട്: OFF (ഡിസേബിൾ)
  - പ്ലഗിൻ ഒരു ബണ്ടിൽ ചെയ്ത Cloudflare CIDR സ്നാപ്പ്ഷോട്ട് സഹിതം വരുന്നു
  - CIDR റേഞ്ചുകൾ wp-cron വഴി ആഴ്ചതോറും പുതുക്കുന്നു
  - റിഫ്രഷ് പരാജയപ്പെട്ടാൽ ബണ്ടിൽ ചെയ്ത ഫോൾബാക്ക് ഉപയോഗിക്കുന്നു

#### വിശ്വസ്ത പ്രോക്സി കോൺഫിഗറേഷൻ (Trusted Proxy Configuration) {#trusted-proxy-configuration}

നിങ്ങളുടെ ഫ്രണ്ട്-ലൈൻ പ്രോക്സികളെയും ലോഡ്-ബാലൻസറുകളെയും നിർവചിക്കുക:

* **`cap_trusted_proxies`** — CIDR-കളോ അല്ലെങ്കിൽ ബെയർ IP-കളോ അടങ്ങിയ ടെക്സ്റ്റ് ഏരിയ (ഓരോ വരിയിലും ഒന്ന്)
  - കമന്റുകൾ അനുവദനീയമാണ് ( `#` తో തുടങ്ങുന്ന വരികൾ)
  - ഈ സെറ്റിംഗ് ഇല്ലെങ്കിൽ, റേറ്റ് ലിമിറ്റർ എനേബിൾ ചെയ്തിട്ടുണ്ടെങ്കിലും `X-Forwarded-For` അവഗണിക്കപ്പെടും.
  - ഉദാഹരണം:
    ```
    # ഞങ്ങളുടെ ലോഡ് ബാലൻസർ
    192.0.2.0/24

    # ബാക്കപ്പ് പ്രോക്സി
    198.51.100.50
    ```

#### ആദ്യമായി എനേബിൾ ചെയ്യുമ്പോൾ ഓട്ടോ-ഡെറ്റക്ഷൻ (First-Enable Auto-Detection) {#first-enable-auto-detection}

നിങ്ങൾ റേറ്റ് ലിമിറ്റർ ആദ്യമായി എനേബിൾ ചെയ്യുമ്പോൾ, പ്ലഗിൻ നിങ്ങളുടെ സാധ്യതയുള്ള Cloudflare, പ്രോക്സി അവസ്ഥകൾ കണ്ടെത്തി ഒരു ക്ലിക്ക്-ഓൺ "Apply detected settings" അഡ്മിൻ നോട്ടീസ് കാണിക്കുന്നു.

* പ്ലഗിൻ നിങ്ങളുടെ സേവ് ചെയ്ത മൂല്യങ്ങൾ **ഒരിക്കലും മാറ്റിസ്ഥാപിക്കില്ല**
* തുടർന്നുള്ള ട്രാഫിക് നിങ്ങളുടെ കോൺഫിഗറേഷൻ യാഥാർത്ഥ്യവുമായി പൊരുത്തപ്പെടുന്നില്ല എന്ന് സൂചിപ്പിക്കുന്നുവെങ്കിൽ (ഉദാഹരണത്തിന്, Cloudflare CIDR റേഞ്ചുകൾ മാറിയെങ്കിൽ), ശുപാർശ ചെയ്യുന്ന അപ്ഡേറ്റ് കാണിക്കുന്ന ഒരു ഒഴിവാക്കാൻ കഴിയാത്ത (non-dismissable) നോട്ടീസ് വരും.

### ഇത് എന്തുകൊണ്ട് പ്രധാനമാണ് (Why This Matters) {#why-this-matters}

ഒരു കർശനമായ IP വിശ്വാസ മാതൃക ഒരു നിർണ്ണായക സ്പൂഫിംഗ് വെക്റ്റർ അടയ്ക്കുന്നു: ഒരു വ്യാജ `CF-Connecting-IP` ഹെഡർ വഹിച്ചുകൊണ്ട് നേരിട്ടുള്ള ഒറിജിൻ-സെർവർ അഭ്യർത്ഥനകൾ മുമ്പ് യഥാർത്ഥ പിയറിന് പകരം സ്പൂഫ് ചെയ്ത IP വഴി ബക്കറ്റ് ചെയ്യപ്പെട്ടിരുന്നു. ഇത് പ്രത്യേകിച്ചും പ്രധാനമാണ്:

* **റേറ്റ് ലിമിറ്റിംഗ്** — IP സ്പൂഫ് ചെയ്ത് ആക്രമണകാരികൾക്ക് ലിമിറ്റുകൾ മറികടക്കാൻ കഴിയില്ല.
* **സ്റ്റാറ്റിസ്റ്റിക്സ്** — നിങ്ങളുടെ ആക്രമണ മെട്രിക്സ് വ്യാജമായവയല്ല, യഥാർത്ഥ സന്ദർശക IP-കളെയാണ് പ്രതിഫലിപ്പിക്കുന്നത്.
* **കാപ്‌ചാ വെരിഫിക്കേഷൻ** — റിസ്ക് വിലയിരുത്തലിനായി പ്രൊവൈഡർമാർക്ക് ശരിയായ സന്ദർശക IP ലഭിക്കുന്നു.
