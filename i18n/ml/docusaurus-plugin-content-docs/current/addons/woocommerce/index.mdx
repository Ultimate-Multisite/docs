# WooCommerce Integration

WooCommerce Integration പ്രവർത്തിക്കുന്നത് ഞങ്ങളുടെ Manual Gateway പ്രവർത്തിക്കുന്ന രീതിക്ക് വളരെ സാമ്യമുള്ളതാണ്. ഇത് നിങ്ങളുടെ ഉപയോക്താക്കൾക്ക് ധാരാളം വഴക്കം നൽകുന്നു, കാരണം നിങ്ങളുടെ WooCommerce ഇൻസ്റ്റാളേഷനിൽ ലഭ്യമായ ഏത് പേയ്‌മെന്റ് ഫോമുകൾ വേണമെങ്കിലും അവർക്ക് തിരഞ്ഞെടുക്കാൻ കഴിയും. ഇത് വളരെ ലളിതമാണ്: ഉപയോക്താവിന്റെ ബില്ലിംഗ് കാലയളവ് അവസാനിക്കുമ്പോൾ, ഉപയോക്താവിന്റെ സബ്‌സ്‌ക്രിപ്ഷൻ 'on hold' ആകും. അതേ സമയം, നെറ്റ്‌വർക്കിന്റെ പ്രധാന സൈറ്റിലെ WooCommerce ഇൻസ്റ്റാളേഷനിൽ ഒരു ഓർഡർ സൃഷ്ടിക്കപ്പെടും. തുടർന്ന്, പേയ്‌മെന്റ് നടത്താനുള്ള ലിങ്ക് സഹിതം ഒരു ഇൻവോയ്സ് ഉപയോക്താവിന്റെ ഇമെയിലിലേക്ക് ലഭിക്കും. പേയ്‌മെന്റ് ലഭിച്ചാൽ, സബ്‌സ്‌ക്രിപ്ഷൻ പുതുക്കപ്പെടും.

## സവിശേഷതകൾ (Features)

- Ultimate Multisite-ഉം WooCommerce-ഉം തമ്മിലുള്ള എളുപ്പത്തിലുള്ള സംയോജനം (Easy integration)
- നിരവധി പേയ്‌മെന്റ് രീതികൾ ലഭ്യമാണ്
- WooCommerce Subscriptions ഉപയോഗിച്ച് പൂർണ്ണമായും ഓട്ടോമേറ്റഡ് ആയ പ്രക്രിയ

## ഡോക്യുമെന്റേഷൻ (Documentation)

**WooCommerce Integration സജ്ജമാക്കൽ**

**Ultimate Multisite ഉപയോഗിച്ച് നിങ്ങളുടെ പേയ്‌മെന്റ് ഓപ്ഷനുകൾ വികസിപ്പിക്കുന്നു: WooCommerce Integration**

## ഇൻസ്റ്റാളേഷൻ (Installation)

1. 'ultimate-multisite-woocommerce' എന്നത് '/wp-content/plugins/' ഡയറക്‌ടറിയിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക
2. WordPress-ലെ 'Plugins' മെനുവിലൂടെ പ്ലഗിൻ ആക്ടിവേറ്റ് ചെയ്യുക
3. പ്ലഗിൻ സജ്ജമാക്കാൻ ഘട്ടം ഘട്ടമായുള്ള Wizard പിന്തുടരുക

## ഉപയോഗ ഗൈഡ് (Usage Guide)

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite പതിപ്പ് 2.x നെക്കുറിച്ചാണ് പരാമർശിക്കുന്നത്.**_

_**ശ്രദ്ധിക്കുക:** Ultimate Multisite: WooCommerce Integration പ്രവർത്തിക്കാൻ നിങ്ങളുടെ പ്രധാന സൈറ്റിൽ കുറഞ്ഞത് WooCommerce കുറഞ്ഞത് ആക്ടിവേറ്റ് ചെയ്തിരിക്കണം._

_Stripe_ ഉം _PayPal_ ഉം ചില രാജ്യങ്ങളിൽ ലഭ്യമല്ലാത്തതിനാൽ, Ultimate Multisite ഉപയോക്താക്കൾക്ക് ഞങ്ങളുടെ പ്ലഗിൻ ഫലപ്രദമായി ഉപയോഗിക്കുന്നതിൽ പരിമിതികളോ തടസ്സങ്ങളോ ഉണ്ടാകുമെന്ന് ഞങ്ങൾ മനസ്സിലാക്കുന്നു. അതിനാൽ, _WooCommerce_ സംയോജിപ്പിക്കാനുള്ള ഒരു **add-on** ഞങ്ങൾ സൃഷ്ടിച്ചു.

_WooCommerce_ എന്നത് വളരെ ജനപ്രിയമായ ഒരു ഇ-കൊമേഴ്‌സ് പ്ലഗിനാണ്. ലോകമെമ്പാടുമുള്ള ഡെവലപ്പർമാർ ഇതിലേക്ക് വിവിധ പേയ്‌മെന്റ് ഗേറ്റ്‌വേകൾ സംയോജിപ്പിക്കാനുള്ള add-on-കൾ ഉണ്ടാക്കിയിട്ടുണ്ട്. Ultimate Multisite ബില്ലിംഗ് സിസ്റ്റത്തിൽ നിങ്ങൾക്ക് ഉപയോഗിക്കാൻ കഴിയുന്ന പേയ്‌മെന്റ് ഗേറ്റ്‌വേകൾ വികസിപ്പിക്കാൻ ഞങ്ങൾ ഇതിനൊരു അവസരം ഉപയോഗിച്ചു.

## ഇത് സജ്ജമാക്കുന്നത് എങ്ങനെ (Setting It Up)

ആദ്യം, നിങ്ങൾ WooCommerce Integration add-on ഇൻസ്റ്റാൾ ചെയ്യേണ്ടതുണ്ട്. ഇത് **ലൈസൻസ് ഉള്ള ഉപയോക്താക്കൾക്ക് സൗജന്യമാണ്**. ഇത് ഇൻസ്റ്റാൾ ചെയ്യാൻ, നിങ്ങളുടെ super admin dashboard-ൽ Ultimate Multisite-ൽ ക്ലിക്ക് ചെയ്യുക, തുടർന്ന് **Add-ons** തിരഞ്ഞെടുക്കുക.

![Add-ons page](/img/admin/addons-page.png)

ഇവിടെ നിങ്ങൾക്ക് എല്ലാ Ultimate Multisite add-on-കളും കാണാൻ കഴിയും. **Ultimate Multisite: WooCommerce Integration** add-on-ൽ ക്ലിക്ക് ചെയ്യുക.

![Add-ons page showing WooCommerce Integration](/img/admin/addons-page.png)

add-on വിശദാംശങ്ങൾ സഹിതം ഒരു വിൻഡോ പോപ്പ് അപ്പ് ചെയ്യും. **Install Now**-ൽ ക്ലിക്ക് ചെയ്താൽ മതി.

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

ഇൻസ്റ്റാളേഷൻ കഴിഞ്ഞാൽ, നിങ്ങൾ പ്ലഗിനുകൾ പേജിലേക്ക് റീഡയറക്ട് ചെയ്യപ്പെടും. ഇവിടെ, **Network Activate**-ൽ ക്ലിക്ക് ചെയ്താൽ _WooCommerce_ add-on നിങ്ങളുടെ നെറ്റ്‌വർക്കിൽ ആക്ടിവേറ്റ് ചെയ്യപ്പെടും.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

ഇത് ആക്ടിവേറ്റ് ചെയ്ത ശേഷം പോലും, നിങ്ങളുടെ വെബ്‌സൈറ്റിൽ WooCommerce പ്ലഗിൻ ഇൻസ്റ്റാൾ ചെയ്യുകയും ആക്ടിവേറ്റ് ചെയ്യുകയും ചെയ്തിട്ടില്ലെങ്കിൽ, **നിങ്ങൾക്ക് ഒരു റിമൈൻഡർ ലഭിക്കും**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

ഇൻസ്റ്റാൾ ചെയ്ത ശേഷം, Ultimate Multisite പേയ്‌മെന്റ് സെറ്റിംഗ്സിൽ നിങ്ങൾക്ക് ഒരു അധിക ഓപ്ഷൻ കാണാൻ കഴിയും. അവിടെ നിന്ന് നിങ്ങൾക്ക് _WooCommerce_ ഓപ്ഷൻ **enable** ചെയ്യാൻ കഴിയും.

![Payment gateway settings](/img/config/settings-payment-gateways.png)

ഫ്രണ്ട് എൻഡിൽ കാണിക്കേണ്ട ഡിസ്‌പ്ലേ നെയിം പോലുള്ള ചില അടിസ്ഥാന ക്രമീകരണങ്ങൾ നിങ്ങൾക്ക് ഇവിടെ സജ്ജമാക്കാൻ കഴിയും, കൂടാതെ നിങ്ങളുടെ ഉപയോക്താക്കളെ അവരുടെ ചെക്ക്ഔട്ട് ഫ്ലോയിൽ നയിക്കാനുള്ള നിർദ്ദേശങ്ങളും നൽകാം. അതിനുശേഷം, നിങ്ങൾ ഉപയോഗിക്കാൻ ഇഷ്ടപ്പെടുന്ന പേയ്‌മെന്റ് ഗേറ്റ്‌വേ എനേബിൾ ചെയ്യാനും സജ്ജമാക്കാനും നിങ്ങളുടെ **പ്രധാന സൈറ്റിലെ _WooCommerce_ സെറ്റിംഗ്സിലേക്ക്** പോകാവുന്നതാണ്.

## പേയ്‌മെന്റ് എങ്ങനെ കൈകാര്യം ചെയ്യണം (How to manage the payment)

WooCommerce Integration പ്രവർത്തിക്കുന്നത് ഞങ്ങളുടെ **Manual Gateway** പ്രവർത്തിക്കുന്ന രീതിക്ക് വളരെ സാമ്യമുള്ളതാണ്. ഇതിലെ പോരായ്മ എന്തെന്നാൽ, ഓരോ പേയ്‌മെന്റും ബില്ലിംഗ് കാലയളവിന്റെ അവസാനം കൈകൊണ്ട് അടയ്‌ക്കേണ്ടതുണ്ട്. പ്രക്രിയ എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്ന് നോക്കാം:

  * ഉപയോക്താവിന്റെ ബില്ലിംഗ് സൈക്കിൾ അവസാനിക്കുമ്പോൾ, മെമ്പർഷിപ്പ് **on hold** ആകും. അതേ സമയം, നെറ്റ്‌വർക്കിന്റെ പ്രധാന സൈറ്റിലെ _WooCommerce_ ഇൻസ്റ്റാളേഷനിൽ ഒരു ഓർഡർ സൃഷ്ടിക്കപ്പെടും.

  * പേയ്‌മെന്റ് നടത്താനുള്ള ലിങ്ക് സഹിതം ഒരു ഇൻവോയ്സ് ഉപയോക്താവിന്റെ ഇമെയിലിലേക്ക് ലഭിക്കും (ഈ ലിങ്ക് അവരുടെ സബ്‌സൈറ്റ് ഡാഷ്‌ബോർഡിലെ ബില്ലിംഗ് ഹിസ്റ്ററിയിലും കാണാം). ആ ലിങ്ക് ഉപയോക്താവിനെ _**WooCommerce**_ **checkout** ലേക്ക് റീഡയറക്ട് ചെയ്യും. അവിടെ ഉപയോക്താക്കൾക്ക് അവരുടെ പേയ്‌മെന്റ് പ്രോസസ്സ് ചെയ്യുന്നതിനായി **ലഭ്യമായ _WooCommerce_ ഗേറ്റ്‌വേകളിൽ** ഒന്ന് തിരഞ്ഞെടുക്കാൻ കഴിയും.

  * പേയ്‌മെന്റ് ലഭിച്ചാൽ, മെമ്പർഷിപ്പ് **പുതുക്കപ്പെടും**, പേയ്‌മെന്റ് Ultimate Multisite-ൽ രേഖപ്പെടുത്തുകയും ഓർഡർ പൂർത്തിയാക്കപ്പെടുകയും ചെയ്യും.

### ചില പ്രധാന കുറിപ്പുകൾ:

  * Ultimate Multisite-ന്റെ ഡിഫോൾട്ട് ഫംഗ്ഷണാലിറ്റി ഈ സംയോജന രൂപത്തോടുകൂടി ഇപ്പോഴും പ്രവർത്തിക്കുന്നു. ഉദാഹരണത്തിന്, **Subscription Management** സ്ക്രീനിൽ നിന്ന് **റീഫണ്ട് നൽകുന്നത്** (issuing refunds) തിരഞ്ഞെടുക്കുന്ന പേയ്‌മെന്റ് ഫോം ഉപയോഗിച്ച് റീഫണ്ട് നൽകുന്നതിനായി WooCommerce യുമായി ആശയവിനിമയം നടത്തും.

  * Ultimate Multisite: WooCommerce Integration പ്രവർത്തിക്കാൻ WP-Cron-നെ ആശ്രയിച്ചിരിക്കുന്നു. മിക്കപ്പോഴും ഇത് വളരെ നന്നായി പ്രവർത്തിക്കുന്നു, പക്ഷേ സൈറ്റിൽ ട്രാഫിക് കുറവാണെങ്കിൽ, ഷെഡ്യൂൾ ചെയ്ത ഇവന്റുകൾ പരിശോധിക്കാൻ ക്രോൺ (cron) പ്രേരിപ്പിക്കപ്പെടില്ല, അതിനാൽ ഇവന്റ് നഷ്ടപ്പെടുകയും അത് എക്സിക്യൂഷൻ കാലതാമസത്തിന് കാരണമാവുകയും ചെയ്യാം.

## Woocommerce Subscriptions Plugin

ഒരു സബ്‌സ്‌ക്രിപ്ഷൻ എന്നത് ഓരോ ബില്ലിംഗ് സൈക്കിളിലും ഓട്ടോമാറ്റിക്കായി ആവർത്തിക്കപ്പെടണം എന്ന് പ്രതീക്ഷിക്കുന്നതിനാൽ, ഓരോ മാസവും പേയ്‌മെന്റ് കൈകൊണ്ട് ആരംഭിക്കുന്നത് അന്തിമ ഉപയോക്താക്കൾക്ക് അനുയോജമല്ല. അതിനാൽ, നിങ്ങളുടെ അന്തിമ ഉപയോക്താക്കൾക്ക് ഇടപാട് എളുപ്പമാക്കാൻ, **WooCommerce Subscriptions plugin** പിന്തുണയ്ക്കുന്നതിനായി ഞങ്ങൾ WooCommerce Integration add-on രൂപകൽപ്പന ചെയ്തിരിക്കുന്നു.

നിങ്ങൾ ചെയ്യേണ്ടത് [Woocommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) **ഇൻസ്റ്റാൾ** ചെയ്യുക, അത് നിങ്ങളുടെ **പ്രധാന സൈറ്റിൽ ആക്ടിവേറ്റ്** ചെയ്യുക എന്നിവ മാത്രമാണ്. അതിനുശേഷം Ultimate Multisite പേയ്‌മെന്റ് സെറ്റിംഗ്സിലേക്ക് പോയി **ഇന്റഗ്രേഷൻ എനേബിൾ** ചെയ്യുക.

അത്രയേ ഉള്ളൂ. നിങ്ങളുടെ അന്തിമ ഉപയോക്താക്കൾക്ക് അവരുടെ ബില്ലിംഗ് സൈക്കിളിന്റെ അവസാനം പേയ്‌മെന്റ് കൈകൊണ്ട് അടയ്‌ക്കേണ്ട ആവശ്യം ഇനി ഉണ്ടാകില്ല.

നിങ്ങളുടെ പ്രധാന സൈറ്റിൽ _WooCommerce_ යටതോട് ഒരു ഓർഡർ സൃഷ്ടിക്കപ്പെടുന്നതിനൊപ്പം, ഓരോ അക്കൗണ്ടിനും ഒരു സബ്‌സ്‌ക്രിപ്ഷനും ഇത് സൃഷ്ടിക്കും.
