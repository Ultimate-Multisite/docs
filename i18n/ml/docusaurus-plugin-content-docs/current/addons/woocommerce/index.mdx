# WooCommerce സംയോജനം

WooCommerce സംയോജനം ഞങ്ങളുടെ മാനുവൽ ഗേറ്റ്വേ പ്രവർത്തിക്കുന്ന രീതിയോട് വളരെ സാമ്യമുള്ള രീതിയിൽ പ്രവർത്തിക്കുന്നു. ഇത് നിങ്ങളുടെ ഉപയോക്താക്കൾക്ക് വലിയ സൗകര്യം നൽകുന്നു; നിങ്ങളുടെ WooCommerce ഇൻസ്റ്റാളിൽ ലഭ്യമായ ഏതെങ്കിലും പേയ്മെന്റ് ഫോമുകൾ തിരഞ്ഞെടുക്കാൻ അവരെ അനുവദിക്കുന്നു. ഇത് ലളിതവുമാണ്: ഉപയോക്താവിന്റെ ബില്ലിംഗ് കാലയളവ് അവസാനിക്കുമ്പോൾ, ഉപയോക്താവിന്റെ സബ്സ്ക്രിപ്ഷൻ ഹോൾഡിലാക്കും. അതേ സമയം, നെറ്റ്‌വർക്കിന്റെ പ്രധാന സൈറ്റിലെ WooCommerce ഇൻസ്റ്റാളിൽ ഒരു ഓർഡർ സൃഷ്ടിക്കും. തുടർന്ന് പേയ്മെന്റ് നടത്താനുള്ള ലിങ്ക് ഉൾപ്പെടെ ഉപയോക്താവിന് ഇമെയിലിൽ ഒരു ഇൻവോയ്സ് ലഭിക്കും. പേയ്മെന്റ് ലഭിച്ചാൽ, സബ്സ്ക്രിപ്ഷൻ പുതുക്കും.

## സവിശേഷതകൾ

- Ultimate Multisite ഉം WooCommerce ഉം തമ്മിലുള്ള ലളിതമായ സംയോജനം
- നിരവധി പേയ്മെന്റ് രീതികൾ ലഭ്യമാണ്
- ഉൾപ്പെടുത്തിയിരിക്കുന്ന സബ്സ്ക്രിപ്ഷൻ കോർ ഉപയോഗിച്ച് ഓട്ടോമേറ്റഡ് ആവർത്തന ബില്ലിംഗ്, ഐച്ഛികമായ WooCommerce Subscriptions പ്ലഗിൻ മെച്ചപ്പെടുത്തലുകളോടെ
- Checkout മൊത്തങ്ങളിൽ Ultimate Multisite നികുതികൾ വേർതിരിച്ച, നികുതിയില്ലാത്ത WooCommerce ഫീസ് ലൈനുകളായി ഉൾപ്പെടും, അതിനാൽ ഉപഭോക്താക്കൾക്ക് പേയ്മെന്റിന് മുമ്പ് നികുതി തുകകൾ പരിശോധിക്കാം

## ഡോക്യുമെന്റേഷൻ

**WooCommerce സംയോജനം സജ്ജീകരിക്കൽ**

**Ultimate Multisite ഉപയോഗിച്ച് നിങ്ങളുടെ പേയ്മെന്റ് ഓപ്ഷനുകൾ വിപുലീകരിക്കൽ: WooCommerce സംയോജനം**

## ഇൻസ്റ്റാളേഷൻ

1. 'ultimate-multisite-woocommerce' '/wp-content/plugins/' ഡയറക്ടറിയിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക
2. WordPress ലെ 'Plugins' മെനുവിലൂടെ പ്ലഗിൻ സജീവമാക്കുക
3. പ്ലഗിൻ സജ്ജമാക്കാൻ ഘട്ടം ഘട്ടമായുള്ള വിസാർഡ് പിന്തുടരുക

## ഉപയോഗ മാർഗ്ഗനിർദ്ദേശം

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite പതിപ്പ് 2.x നെ കുറിച്ചാണ്.**_

_**ശ്രദ്ധിക്കുക:** Ultimate Multisite: WooCommerce സംയോജനത്തിന് കുറഞ്ഞത് നിങ്ങളുടെ പ്രധാന സൈറ്റിൽ WooCommerce സജീവമാക്കിയിരിക്കണം._

ചില രാജ്യങ്ങളിൽ _Stripe_ ഉം _PayPal_ ഉം ലഭ്യമല്ലെന്ന് ഞങ്ങൾ മനസ്സിലാക്കുന്നു. ഇത് Ultimate Multisite ഉപയോക്താക്കൾക്ക് ഞങ്ങളുടെ പ്ലഗിൻ ഫലപ്രദമായി ഉപയോഗിക്കുന്നതിന് പരിമിതിയോ തടസ്സമോ സൃഷ്ടിക്കുന്നു. അതിനാൽ _WooCommerce_ സംയോജിപ്പിക്കാൻ ഞങ്ങൾ ഒരു **ആഡ്-ഓൺ സൃഷ്ടിച്ചു**.

_WooCommerce_ വളരെ ജനപ്രിയമായ ഒരു ഇ-കൊമേഴ്‌സ് പ്ലഗിൻ ആണ്. ലോകമെമ്പാടുമുള്ള ഡെവലപ്പർമാർ വ്യത്യസ്ത പേയ്മെന്റ് ഗേറ്റ്വേകൾ ഇതുമായി സംയോജിപ്പിക്കാൻ ആഡ്-ഓണുകൾ സൃഷ്ടിച്ചു. Ultimate Multisite ബില്ലിംഗ് സിസ്റ്റത്തിൽ നിങ്ങൾക്ക് ഉപയോഗിക്കാവുന്ന പേയ്മെന്റ് ഗേറ്റ്വേകൾ വിപുലീകരിക്കാൻ ഞങ്ങൾ ഇതിന്റെ പ്രയോജനം ഉപയോഗിച്ചു.

## ഇത് സജ്ജീകരിക്കൽ

ആദ്യം, **ലൈസൻസുള്ള ഉപയോക്താക്കൾക്ക് സൗജന്യമായ** WooCommerce സംയോജന ആഡ്-ഓൺ ഇൻസ്റ്റാൾ ചെയ്യണം. ഇത് ഇൻസ്റ്റാൾ ചെയ്യാൻ, നിങ്ങളുടെ സൂപ്പർ അഡ്മിൻ Dashboard ൽ Ultimate Multisite ക്ലിക്ക് ചെയ്ത് **ആഡ്-ഓണുകൾ** തിരഞ്ഞെടുക്കുക.

![ആഡ്-ഓണുകൾ പേജ്](/img/admin/addons-page.png)

ഇവിടെ എല്ലാ Ultimate Multisite ആഡ്-ഓണുകളും കണ്ടെത്താം. **Ultimate Multisite: WooCommerce സംയോജനം** ആഡ്-ഓൺ ക്ലിക്ക് ചെയ്യുക.

![WooCommerce സംയോജനം കാണിക്കുന്ന ആഡ്-ഓണുകൾ പേജ്](/img/admin/addons-page.png)

ആഡ്-ഓൺ വിശദാംശങ്ങളുള്ള ഒരു വിൻഡോ തുറക്കും. **ഇപ്പോൾ ഇൻസ്റ്റാൾ ചെയ്യുക** *.* എന്നതിൽ മാത്രം ക്ലിക്ക് ചെയ്യുക

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

ഇൻസ്റ്റാളേഷൻ പൂർത്തിയായ ശേഷം, നിങ്ങളെ പ്ലഗിനുകൾ പേജിലേക്ക് റീഡയറക്ട് ചെയ്യും. ഇവിടെ **നെറ്റ്‌വർക്ക് സജീവമാക്കുക** എന്നതിൽ മാത്രം ക്ലിക്ക് ചെയ്യുക, _WooCommerce_ ആഡ്-ഓൺ നിങ്ങളുടെ നെറ്റ്‌വർക്കിൽ സജീവമാകും.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

ഇത് സജീവമാക്കിയ ശേഷം, നിങ്ങളുടെ വെബ്‌സൈറ്റിൽ WooCommerce പ്ലഗിൻ ഇൻസ്റ്റാൾ ചെയ്ത് സജീവമാക്കിയിട്ടില്ലെങ്കിൽ, **നിങ്ങൾക്ക് ഒരു ഓർമ്മപ്പെടുത്തൽ ലഭിക്കും**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

ഇൻസ്റ്റാൾ ചെയ്തുകഴിഞ്ഞാൽ, Ultimate Multisite പേയ്മെന്റ് ക്രമീകരണങ്ങൾക്ക് കീഴിൽ ഒരു അധിക ഓപ്ഷൻ കാണും. തുടർന്ന് അവിടെ നിന്ന് _WooCommerce_ ഓപ്ഷൻ **പ്രവർത്തനക്ഷമമാക്കാം**.

![WooCommerce പ്രവർത്തനക്ഷമമാക്കിയ Ultimate Multisite പേയ്മെന്റ് ഗേറ്റ്വേ ക്രമീകരണങ്ങൾ](/img/addons/woocommerce-payment-gateway.png)

ഫ്രണ്ട് എൻഡിൽ കാണിക്കുന്ന പ്രദർശന നാമം, ഉപയോക്താക്കളെ അവരുടെ Checkout പ്രവാഹത്തിൽ നയിക്കുന്ന നിർദ്ദേശങ്ങൾ എന്നിവ പോലുള്ള ചില അടിസ്ഥാന ക്രമീകരണങ്ങൾ നിങ്ങൾക്ക് കോൺഫിഗർ ചെയ്യാം. തുടർന്ന് നിങ്ങൾ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന പേയ്മെന്റ് ഗേറ്റ്വേ പ്രവർത്തനക്ഷമമാക്കാനും കോൺഫിഗർ ചെയ്യാനും നിങ്ങളുടെ **പ്രധാന സൈറ്റിന്റെ _WooCommerce_ ക്രമീകരണങ്ങളിലേക്ക്** പോകാം.

### Checkout നികുതി ഫീസ് ലൈനുകൾ

Ultimate Multisite Checkout ൽ നികുതികൾ ഉൾപ്പെടുമ്പോൾ, സംയോജനം ഓരോ Ultimate Multisite നികുതി തുകയും WooCommerce Checkout ലേക്ക് വേർതിരിച്ച, നികുതിയില്ലാത്ത ഫീസ് ലൈനായി ചേർക്കുന്നു. ഇതിലൂടെ പേയ്മെന്റിന് മുമ്പ് നികുതി മൊത്തം ദൃശ്യമായി നിലനിൽക്കും, കൂടാതെ WooCommerce വീണ്ടും ആ നികുതി ഫീസിന് നികുതി ചുമത്തുന്നത് തടയും.

## പേയ്മെന്റ് എങ്ങനെ നിയന്ത്രിക്കാം

WooCommerce സംയോജനം ഞങ്ങളുടെ **മാനുവൽ ഗേറ്റ്വേ** പ്രവർത്തിക്കുന്ന രീതിയോട് വളരെ സാമ്യമുള്ള രീതിയിൽ പ്രവർത്തിക്കുന്നു. ദോഷം എന്തെന്നാൽ, ഓരോ പേയ്മെന്റും ബില്ലിംഗ് കാലയളവിന്റെ അവസാനം മാനുവലായി അടയ്ക്കണം. പ്രവാഹം ഇങ്ങനെ പ്രവർത്തിക്കുന്നു:

  * ഉപയോക്താവിന്റെ ബില്ലിംഗ് സൈക്കിൾ അവസാനിച്ചാൽ, അംഗത്വം **ഹോൾഡിലാക്കും**. അതേ സമയം, നിങ്ങളുടെ നെറ്റ്‌വർക്കിന്റെ പ്രധാന സൈറ്റിലെ _WooCommerce_ ഇൻസ്റ്റാളിൽ ഒരു ഓർഡർ സൃഷ്ടിക്കും.

  * തുടർന്ന് ഉപയോക്താക്കൾക്ക് പേയ്മെന്റ് നടത്താനുള്ള ലിങ്ക് ഉൾപ്പെടെ ഇമെയിലിൽ ഒരു ഇൻവോയ്സ് ലഭിക്കും (അവരുടെ ഉപസൈറ്റ് Dashboard ലെ ബില്ലിംഗ് ചരിത്രത്തിലും ലിങ്ക് കാണും). ആ ലിങ്ക് ഉപയോക്താവിനെ _**WooCommerce**_ **Checkout** ലേക്ക് റീഡയറക്ട് ചെയ്യും, അവിടെ ഉപയോക്താക്കൾക്ക് പേയ്മെന്റ് പ്രോസസ് ചെയ്യാൻ **ലഭ്യമായ _WooCommerce_ ഗേറ്റ്വേകളിൽ** ഒന്ന് തിരഞ്ഞെടുക്കാം.

  * പേയ്മെന്റ് ലഭിച്ചാൽ, അംഗത്വം **പുതുക്കും**, പേയ്മെന്റ് Ultimate Multisite ൽ രേഖപ്പെടുത്തും, ഓർഡർ പൂർത്തിയായതായി സജ്ജമാക്കും.

### ചില പ്രധാന കുറിപ്പുകൾ:

  * Ultimate Multisite ന്റെ ഡിഫോൾട്ട് പ്രവർത്തനക്ഷമത ഈ സംയോജന ഫോമിനോടൊപ്പം തുടർന്നും പ്രവർത്തിക്കും. ഉദാഹരണത്തിന്, **സബ്സ്ക്രിപ്ഷൻ മാനേജ്മെന്റ്** സ്ക്രീനിൽ നിന്ന് **റിഫണ്ടുകൾ നൽകുന്നത്** തിരഞ്ഞെടുക്കപ്പെട്ട പേയ്മെന്റ് ഫോം ഉപയോഗിച്ച് റിഫണ്ട് നൽകാൻ WooCommerce മായി ആശയവിനിമയം നടത്തും.

  * Ultimate Multisite: WooCommerce സംയോജനം പ്രവർത്തിക്കാൻ WP-Cron നെ ആശ്രയിക്കുന്നു. ഇത് മിക്ക സമയത്തും വളരെ നന്നായി പ്രവർത്തിക്കുന്നു, പക്ഷേ സൈറ്റിൽ കുറഞ്ഞ ട്രാഫിക് മാത്രമാണെങ്കിൽ ക്രോൺ ഷെഡ്യൂൾ ചെയ്ത ഇവന്റുകൾ പരിശോധിക്കാൻ പ്രോംപ്റ്റ് ചെയ്യപ്പെടില്ല, ഇവന്റ് നഷ്ടപ്പെടാം; ഇത് എക്സിക്യൂഷൻ വൈകലിന് കാരണമായേക്കാം.

## WooCommerce Subscriptions പ്ലഗിൻ

ഓരോ മാസവും കൈമാറി പേയ്മെന്റ് ആരംഭിക്കുന്നത് അന്തിമ ഉപയോക്താക്കൾക്ക് അനുയോജ്യമല്ല, കാരണം subscriptions ഓരോ billing cycle-ലും സ്വയമേവ ആവർത്തിക്കുമെന്നാണ് പ്രതീക്ഷിക്കുന്നത്. WooCommerce Integration add-on ഇപ്പോൾ പ്രധാന subscriptions code ഉൾക്കൊള്ളുന്നു, അതിനാൽ recurring products എല്ലായ്പ്പോഴും recurring billing-ന് subscriptions ഉപയോഗിക്കും.

ഈ integration വഴി recurring products വിൽക്കാൻ നിങ്ങൾ വേറിട്ട [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) വാങ്ങുകയോ ഇൻസ്റ്റാൾ ചെയ്യുകയോ ചെയ്യേണ്ടതില്ല. ആ plugin വാങ്ങി, ഇൻസ്റ്റാൾ ചെയ്ത്, നിങ്ങളുടെ main site-ൽ ലഭ്യമാണെങ്കിൽ, Ultimate Multisite അത് ഉപയോഗിക്കും, WooCommerce അധിക subscription management features കാണിക്കും. അത് ലഭ്യമല്ലെങ്കിൽ, ഉൾപ്പെടുത്തിയ subscriptions core recurring products കൈകാര്യം ചെയ്യുന്നത് തുടരും.

![WooCommerce subscription integration ക്രമീകരണങ്ങൾ](/img/addons/woocommerce-subscription-integration.png)

ഇത്രയേ ഉള്ളൂ. recurring products-ക്കായി നിങ്ങളുടെ അന്തിമ ഉപയോക്താക്കൾക്ക് അവരുടെ billing cycle അവസാനത്തിൽ ഇനി കൈമാറി പേയ്മെന്റ് നടത്തേണ്ടതില്ല.

നിങ്ങളുടെ main site-ൽ _WooCommerce_ കീഴിൽ ഒരു order സൃഷ്ടിക്കപ്പെടുന്നതിന് പുറമേ, ഓരോ Account-നും ഒരു subscription-ഉം സൃഷ്ടിക്കും.

### പരാജയപ്പെട്ട renewal suspension പെരുമാറ്റം

സ്ഥിരസ്ഥിതിയായി, WooCommerce Subscriptions അതിന്റെ failed-payment retry schedule പ്രോസസ്സ് ചെയ്യുന്ന സമയത്ത് memberships സജീവമായി തുടരും. ഇതിലൂടെ സാധാരണ WooCommerce Subscriptions grace window നിലനിർത്തപ്പെടുന്നു; അതിനാൽ വീണ്ടെടുക്കപ്പെട്ട renewal-ന് access നേരത്തെ തടസ്സപ്പെടുത്താതെ subscription-ഉം membership-ഉം വീണ്ടും sync ചെയ്യാൻ കഴിയും.

കൂടുതൽ കർശനമായ access control ആഗ്രഹിക്കുന്ന administrators-ക്ക് WooCommerce Integration settings-ൽ **Suspend Memberships Immediately on Failed Renewals** സജീവമാക്കാം. ഈ opt-in setting സജീവമാക്കിയാൽ, WooCommerce Subscriptions retry handling പൂർത്തിയാകുന്നതുവരെ കാത്തിരിക്കാതെ, renewal payment പരാജയപ്പെടുന്ന ഉടനെ Ultimate Multisite ബന്ധപ്പെട്ട membership suspend ചെയ്യും.

Developers-ക്ക് `wu_woo_suspend_on_payment_failure` filter ഉപയോഗിച്ച് അതേ decision override ചെയ്യാം. Hook reference pages add-on source-ൽ നിന്ന് സൃഷ്ടിക്കപ്പെടുന്നതാണ്, അതിനാൽ ഈ overview custom suspension policies-ക്കായി filter name മാത്രം ചൂണ്ടിക്കാണിക്കുന്നു.
