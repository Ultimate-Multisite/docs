# Ultimate AI Connector for WebLLM Changelog

## Version 1.1.0 — Released on 2026-04-09 {#version-110--released-on-2026-04-09}

* പുതിയത്: അഡ്മിൻ-ബാർ സ്റ്റാറ്റസ് ഇൻഡിക്കേറ്റർ സഹിതമുള്ള ഫ്ലോട്ടിംഗ് ചാറ്റ് വിഡ്ജറ്റ് — ലോഗിൻ ചെയ്ത ഏത് ഉപയോക്താവും ഫ്രണ്ട് എൻഡിൽ നിന്ന് നേരിട്ട് ബ്രൗസർ-സൈഡ് LLM-ന് ചോദ്യങ്ങൾ ചോദിക്കാൻ കഴിയും.
* പുതിയത്: SharedWorker റൺടൈം — MLC എഞ്ചിൻ ഇപ്പോൾ ഒരു SharedWorker-ൽ പ്രവർത്തിക്കുന്നു. ഇത് കാരണം ഒന്നിലധികം ടാബുകൾക്ക് GPU സെഷൻ പങ്കിടാൻ കഴിയും, അതിനുവേണ്ടി അവ പരസ്പരം പോരാടേണ്ടി വരുന്നില്ല.
* പുതിയത്: apiFetch middleware interceptor — AI Client SDK പാറ്റേൺ അനുരൂപമായ WordPress REST അഭ്യർത്ഥനകൾ (requests) ലൊക്കൽ WebLLM ബ്രോക്കറിലേക്ക് സുതാര്യമായി (transparently) റൂട്ട് ചെയ്യപ്പെടുന്നു. ഇതിന് loopback HTTP round-trip ആവശ്യമില്ല.
* പുതിയത്: ചാറ്റ് വിഡ്ജറ്റ് ഓണാക്കാനും ഓഫ് ചെയ്യാനും, ഓട്ടോ-പ്രോംപ്റ്റ് സ്വഭാവം (auto-prompt behaviour) സജ്ജമാക്കാനും കണക്റ്റർ പാനലിൽ വിഡ്ജറ്റ് സെറ്റിംഗ്സ് UI ഉൾപ്പെടുത്തി.
* തിരുത്തൽ: IndexedDB കാഷ് ബാക്കെൻഡ് നിർബന്ധമാക്കി. ഇത് HuggingFace xet CDN റീഡയറക്റ്റുകൾ കാരണം തകരാറിലാകുന്ന ഡിഫോൾട്ട് കാഷ് API പാതയിൽ (default Cache API path) മോഡൽ വെയ്റ്റ് ഡൗൺലോഡുകൾ നിലനിർത്താൻ സഹായിക്കുന്നു.
* തിരുത്തൽ: എംബെഡിംഗ് മോഡലുകൾക്കായി context_window KV-cache ഓവർറൈഡ് ഒഴിവാക്കി (ഇവയ്ക്ക് ഡീക്കോഡർ ഇല്ല, ഈ ഓവർറൈഡ് കാരണം റൺടൈം പിശക് ഉണ്ടാകുകയായിരുന്നു).
* തിരുത്തൽ: വർക്കർ ടാബ് ലോഡ് ചെയ്യുന്നതിന് മുമ്പേ /webllm/v1/models-ൽ കോൾഡ്-സ്റ്റാർട്ട് കൻഡിഡേറ്റ് മോഡൽ പ്രഖ്യാപിക്കുന്നു. ഇത് കാരണം SDK ഉപയോക്താക്കൾക്ക് ഒരു മോഡൽ ഉടൻ കാണാൻ കഴിയും.
* തിരുത്തൽ: ഒരു മോഡൽ സജ്ജമാക്കുമ്പോൾ AI Experiments ഫീച്ചർ ബ്രൗസർ എഞ്ചിനിലേക്ക് റൂട്ട് ചെയ്യപ്പെടാൻ WebLLM നെ wpai_preferred_text_models ഫിൽറ്ററിൽ ബന്ധിപ്പിച്ചു (Hook).
* മെച്ചപ്പെടുത്തിയത്: എൻഡ്-ടു-എൻഡ് ടെസ്റ്റിംഗിനിടെ കാഷ്-ബസ്റ്റിംഗ് (Cache-busting), കണ്ടന്റ് നോർമലൈസേഷൻ (content normalisation), ഹാർഡ്‌വെയർ-റെഫറൻസ് (hardware-reference) എന്നിവയുമായി ബന്ധപ്പെട്ട ചില പിശകുകൾ പരിഹരിച്ചു.
