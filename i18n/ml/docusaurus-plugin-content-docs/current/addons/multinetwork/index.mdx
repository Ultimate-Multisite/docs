# Multi-Network {#multi-network}

Ultimate Multisite: Multinetwork add-on നിങ്ങളുടെ Ultimate Multisite പ്ലാറ്റ്‌ഫോമിലേക്ക് ശക്തമായ multinetwork പ്രവർത്തനക്ഷമത കൂട്ടിച്ചേർക്കുന്നു. വ്യക്തിഗത സൈറ്റുകൾ മാത്രമല്ല, മുഴുവൻ WordPress multisite നെറ്റ്‌വർക്കുകൾ വാങ്ങാനും കൈകാര്യം ചെയ്യാനും ഈ add-on നിങ്ങളുടെ ഉപഭോക്താക്കളെ പ്രാപ്തരാക്കുന്നു.

## പ്രധാന സവിശേഷതകൾ (Key Features) {#key-features}

* **Network Provisioning**: നിങ്ങളുടെ ഉപഭോക്താക്കൾക്കായി പൂർണ്ണമായ WordPress multisite നെറ്റ്‌വർക്കുകൾ ഓട്ടോമാറ്റിക്കായി സൃഷ്ടിക്കാനും സജ്ജമാക്കാനും കഴിയും.
* **Network Templates**: സൈൻ അപ്പ് ചെയ്യുന്നതിനുമുമ്പ് ഉപഭോക്താക്കളെ ഒരു മുൻകൂട്ടി തയ്യാറാക്കിയ നെറ്റ്‌വർക്ക് ടെംപ്ലേറ്റ് കാണാനും, അവലോകനം ചെയ്യാനും, തിരഞ്ഞെടുക്കാനും ഇത് അനുവദിക്കുന്നു.
* **Network Management**: multinetwork എൻവയോൺമെന്റുകൾ പൂർണ്ണമായി കൈകാര്യം ചെയ്യാനുള്ള കഴിവുകൾ.
* **Isolated Networks**: ഓരോ ഉപഭോക്താവിന്റെ നെറ്റ്‌വർക്കുകളും അവയുടെ സ്വന്തം ഡാറ്റാബേസ് ടേബിളുകളും കോൺഫിഗറേഷനുകളും സഹിതം സ്വതന്ത്രമായി പ്രവർത്തിക്കുന്നു.
* **Cross-Network SSO**: മാജിക്-ലിങ്ക് ഓതന്റിക്കേഷനിലൂടെ ഉപഭോക്താക്കൾക്ക് അവരുടെ സബ്-നെറ്റ്‌വർക്കുകളിലേക്ക് തടസ്സമില്ലാത്ത പ്രവേശനം നൽകുന്നു.
* **Scalable Architecture**: ഒപ്റ്റിമൈസ് ചെയ്ത റിസോഴ്‌സ് അലോക്കേഷനിലൂടെ ഒന്നിലധികം നെറ്റ്‌വർക്കുകളെ കാര്യക്ഷമമായി കൈകാര്യം ചെയ്യാനുള്ള കഴിവ്.
* **Integration**: Ultimate Multisite-ന്റെ സബ്‌സ്‌ക്രിപ്ഷൻ, ബില്ലിംഗ് സിസ്റ്റം എന്നിവയുമായി തടസ്സമില്ലാതെ സംയോജിക്കുന്നു.

## ഉപയോഗ സാഹചര്യങ്ങൾ (Use Cases) {#use-cases}

* **White Label Hosting**: ഉപഭോക്താക്കൾക്ക് അവരുടെ സ്വന്തം ബ്രാൻഡഡ് multisite നെറ്റ്‌വർക്കുകൾ നൽകാൻ കഴിയും.
* **Agency Solutions**: ഒരു single പ്ലാറ്റ്‌ഫോമിൽ നിന്ന് തന്നെ ഒന്നിലധികം ക്ലയിന്റ് നെറ്റ്‌വർക്കുകൾ കൈകാര്യം ചെയ്യാൻ ഏജൻസികൾക്ക് സാധിക്കുന്നു.
* **Enterprise Deployments**: സങ്കീർണ്ണമായ multi-network ആവശ്യകതകളുള്ള വലിയ സ്ഥാപനങ്ങളെ പിന്തുണയ്ക്കുന്നു.
* **Education Platforms**: വ്യത്യസ്ത സ്കൂളുകൾക്കോ ഡിപ്പാർട്ട്‌മെന്റുകൾക്കോ വേണ്ടി പ്രത്യേക നെറ്റ്‌വർക്കുകൾ സൃഷ്ടിക്കാൻ കഴിയും.

## ആവശ്യകതകൾ (Requirements) {#requirements}

* Ultimate Multisite 2.0.0 അല്ലെങ്കിൽ അതിലും ഉയർന്നത്
* WordPress Multisite ഇൻസ്റ്റാളേഷൻ
* PHP 7.4 അല്ലെങ്കിൽ അതിലും ഉയർന്നത്
* MySQL 5.6 അല്ലെങ്കിൽ അതിലും ഉയർന്നത്

## ഇൻസ്റ്റാളേഷൻ (Installation) {#installation}

1. പ്ലഗിൻ ഫയലുകൾ `/wp-content/plugins/ultimate-multisite-multinetwork` ഡയറക്‌ടറിയിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക.
2. WordPress-ലെ 'Plugins' സ്‌ക്രീനിലൂടെ പ്ലഗിൻ ആക്ടിവേറ്റ് ചെയ്യുക.
3. Ultimate Multisite → Settings വഴി multinetwork ക്രമീകരണങ്ങൾ കോൺഫിഗർ ചെയ്യുക.

## Network Templates {#network-templates}

Version 1.3.0 പുതിയ നെറ്റ്‌വർക്കുകൾ വാങ്ങുന്ന ഉപഭോക്താക്കൾക്കായി ഒരു ടെംപ്ലേറ്റ് ഡിസ്കവറി, സെലക്ഷൻ ഫ്ലോ എന്നിവ ചേർക്കുന്നു. ഓരോ പുതിയ ഉപഭോക്താവിന്റെ നെറ്റ്‌വർക്കും ഒരു തയ്യാറാക്കിയ ഘടനയിൽ നിന്നോ, ഡിസൈനിൽ നിന്നോ, കണ്ടന്റ് ലൈബ്രറിയിൽ നിന്നോ, അല്ലെങ്കിൽ പ്രത്യേക മേഖലയ്ക്കായുള്ള സജ്ജീകരണത്തിൽ നിന്നോ ആരംഭിക്കണമെന്ന് നിങ്ങൾ ആഗ്രഹിക്കുമ്പോൾ നെറ്റ്‌വർക്ക് ടെംപ്ലേറ്റുകൾ ഉപയോഗിക്കുക.

### Template previewer {#template-previewer}

നെറ്റ്‌വർക്ക് ടെംപ്ലേറ്റ് പ്രിവ്യൂവറർ (previewer) ചെക്കൗട്ടിന് മുമ്പ് ലഭ്യമായ ടെംപ്ലേറ്റുകൾ ഉപഭോക്താക്കളെ ब्राउസ് ചെയ്യാൻ അനുവദിക്കുന്നു. ഓരോ ടെംപ്ലേറ്റും ഒരു live preview panel-ൽ തുറക്കാൻ കഴിയുന്നതിനാൽ, നെറ്റ്‌വർക്ക് വാങ്ങുന്നതിനുമുമ്പ് ലേഔട്ടുകളും കണ്ടന്റുകളും താരതമ്യം ചെയ്യാൻ ഉപഭോക്താക്കൾക്ക് കഴിയും.

ഉപഭോക്താക്കളെ സഹായിക്കാൻ പ്രിവ്യൂവറർ ഉപയോഗിക്കുക:

* ലഭ്യമായ ഓരോ നെറ്റ്‌വർക്ക് ടെംപ്ലേറ്റുകളുടെയും രൂപവും ഘടനയും അവലോകനം ചെയ്യാൻ.
* പർച്ചേസ് ഫ്ലോ വിട്ട് പോകാതെ ഒന്നിലധികം ടെംപ്ലേറ്റുകൾ താരതമ്യം ചെയ്യാൻ.
* സൈൻ അപ്പ് ചെയ്യുന്നതിനുമുമ്പ് അവരുടെ ഉപയോഗ സാഹചര്യത്തിന് ഏറ്റവും അനുയോജ്യമായ ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കാൻ.

ടെംപ്ലേറ്റ് പേരുകളും വിവരണങ്ങളും ഉപഭോക്താക്കൾക്ക് എളുപ്പത്തിൽ മനസ്സിലാക്കാവുന്ന രീതിയിൽ സൂക്ഷിക്കുക. ഏജൻസി നെറ്റ്‌വർക്കുകൾ, സ്കൂൾ നെറ്റ്‌വർക്കുകൾ, ഫ്രാഞ്ചൈസി നെറ്റ്‌വർക്കുകൾ, അല്ലെങ്കിൽ white-label hosting bundles പോലുള്ള ഉദ്ദേശിച്ച ഉപയോഗസാഹചര്യങ്ങൾ ഉപഭോക്താക്കൾക്ക് മനസ്സിലാക്കാൻ കഴിയുന്ന സാഹചര്യങ്ങളിലാണ് പ്രിവ്യൂവറർ ഏറ്റവും ഉപകാരപ്രദമാകുന്നത്.

### Template picker at checkout {#template-picker-at-checkout}

ടെംപ്ലേറ്റ് പിക്കർ (Template picker) ഒരു ഫീൽഡായി സൈൻ അപ്പ് ചെക്കൗട്ട് ഫ്ലോയിൽ ചേർക്കാൻ കഴിയും. ഇത് കോൺഫിഗർ ചെയ്യുമ്പോൾ, ഉപഭോക്താക്കൾ ചെക്കൗട്ട സമയത്ത് അവർ ആഗ്രഹിക്കുന്ന നെറ്റ്‌വർക്ക് ടെംപ്ലേറ്റ് തിരഞ്ഞെടുക്കുന്നു, കൂടാതെ അവരുടെ പുതിയ നെറ്റ്‌വർക്ക് സജ്ജമാക്കുമ്പോൾ തിരഞ്ഞെടുത്ത ടെംപ്ലേറ്റ് ഉപയോഗിക്കപ്പെടുന്നു.

ഒരു ചെക്കൗട്ട് ഫ്ലോയിൽ പിക്കർ ഉപയോഗിക്കാൻ:

1. ടെംപ്ലേറ്റുകളായി വാഗ്ദാനം ചെയ്യാൻ നിങ്ങൾ ആഗ്രഹിക്കുന്ന നെറ്റ്‌വർക്കുകൾ തയ്യാറാക്കുക.
2. ബന്ധപ്പെട്ട സൈൻ അപ്പ് അല്ലെങ്കിൽ ചെക്കൗട്ട് ഫോമിൽ Multinetwork ടെംപ്ലേറ്റ് സെലക്ഷൻ ഫീൽഡ് എനേബിൾ ചെയ്യുക.
3. ഏത് ടെംപ്ലേറ്റുകളാണ് ആ ചെക്കൗട്ട് ഫ്ലോയിൽ ലഭ്യമാകേണ്ടതെന്ന് തിരഞ്ഞെടുക്കുക.
4. ചെക്കൗട്ട് ഫ്ലോ പബ്ലിഷ് ചെയ്യുകയും, തിരഞ്ഞെടുത്ത ടെംപ്ലേറ്റ് പുതിയ നെറ്റ്‌വർക്കിലേക്ക് പ്രയോഗിക്കപ്പെടുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കാൻ ഒരു സൈൻ അപ്പ് ടെസ്റ്റ് നടത്തുകയും ചെയ്യുക.

ഒരു ചെക്കൗട്ട് ഫ്ലോയിൽ ഒരു നെറ്റ്‌വർക്ക് ഉൽപ്പന്നം മാത്രമേ നൽകുന്നുള്ളൂ എങ്കിൽ പോലും, ഗൈഡഡ് ഓൺബോർഡിംഗ് ഘട്ടമായി പിക്കർ ഉപയോഗപ്രദമാകും. ഒന്നിലധികം പ്ലാൻ ഫണലുകൾ (multi-plan funnels) ആണെങ്കിൽ, തിരഞ്ഞെടുത്ത ഉൽപ്പന്നവുമായി അല്ലെങ്കിൽ പ്ലാനുമായി പൊരുത്തപ്പെടുന്ന ടെംപ്ലേറ്റുകൾ മാത്രമേ കാണിക്കാവൂ.

## Cross-Network Magic Link SSO {#cross-network-magic-link-sso}

Cross-network magic link SSO ഉപയോഗിച്ച് ഉപഭോക്താക്കൾക്ക് അവരുടെ പ്രധാന അക്കൗണ്ട് ഏരിയയ്ക്കും സബ്-നെറ്റ്‌വർക്കുകളിലേക്കും വീണ്ടും മാനുവലായി സൈൻ ഇൻ ചെയ്യാതെ മാറാൻ കഴിയും. ഒരു ഉപഭോക്താവ് ഒരു cross-network access link പിന്തുടരുമ്പോൾ, Ultimate Multisite ഒരു സമയപരിമിതമായ മാജിക് ലിങ്ക് സൃഷ്ടിക്കുകയും ലക്ഷ്യസ്ഥാന നെറ്റ്‌വർക്കിലേക്ക് ഉപഭോക്താവിനെ ഓതന്റിക്കേറ്റ് ചെയ്യുകയും ചെയ്യുന്നു.

ഒന്നിലധികം നെറ്റ്‌വർക്കുകൾ കൈകാര്യം ചെയ്യുന്നതോ, അല്ലെങ്കിൽ വാങ്ങിയ ഉടൻ തന്നെ പുതിയതായി ക്ലോൺ ചെയ്ത ടെംപ്ലേറ്റ് നെറ്റ്‌വർക്കുകളിലേക്ക് പ്രവേശിക്കേണ്ടതോ ആയ ഉപഭോക്താക്കളുടെ അനുഭവം ഇത് മെച്ചപ്പെടുത്തുന്നു.

Admin checklist:

* WordPress multisite ഇൻസ്റ്റാളേഷന് පුරා Multinetwork add-on സജീവമാണോ എന്ന് ഉറപ്പാക്കുക.
* cross-network ലിങ്കുകൾ വാഗ്ദാനം ചെയ്യുന്നതിനുമുമ്പ് ഉപഭോക്താക്കൾക്ക് ശരിയായ നെറ്റ്‌വർക്ക് ആക്‌സസ് ഉണ്ടെന്ന് ഉറപ്പാക്കുക.
* മാജിക് ലിങ്കുകൾ ഉദ്ദേശിച്ച ലക്ഷ്യസ്ഥാന നെറ്റ്‌വർക്കിലേക്ക് റെസൾവ് ആകുന്നതിനായി നെറ്റ്‌വർക്ക് ഡൊമെയ്‌നുകളും മാപ്പ് ചെയ്ത ഡൊമെയ്‌നുകളും ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക.
* നെറ്റ്‌വർക്ക്, ഡൊമെയ്ൻ, അല്ലെങ്കിൽ ചെക്കൗട്ട് ക്രമീകരണങ്ങൾ മാറ്റിയ ശേഷം ഉപഭോക്തൃ ഡാഷ്‌ബോർഡും ഏതെങ്കിലും post-checkout ആക്‌സസ് ബട്ടണുകളും ടെസ്റ്റ് ചെയ്യുക.

മാജിക് ലിങ്കുകളെ ലോഗിൻ ലിങ്കുകളായി കണക്കാക്കണം. അവ തടസ്സമില്ലാത്ത ഉപഭോക്തൃ പ്രവേശനത്തിനായി രൂപകൽപ്പന ചെയ്തവയാണ്, അതിനാൽ ഉദ്ദേശിച്ച സെഷന് പുറത്തേക്ക് ഇത് ഫോർവേഡ് ചെയ്യുകയോ വീണ്ടും ഉപയോഗിക്കുകയോ ചെയ്യരുത്.

## പതിവായി ചോദിക്കുന്ന ചോദ്യങ്ങൾ (Frequently Asked Questions) {#frequently-asked-questions}

### multinetwork എന്നാൽ എന്ത്? {#what-is-multinetwork}

ഒരു single WordPress ഇൻസ്റ്റാളേഷനിൽ ഒന്നിലധികം സ്വതന്ത്ര multisite നെറ്റ്‌വർക്കുകൾ പ്രവർത്തിപ്പിക്കാൻ കഴിയുന്ന ഒരു WordPress സവിശേഷതയാണ് multinetwork.

### ഇത് സാധാരണ multisite-യിൽ നിന്ന് എങ്ങനെ വ്യത്യാസപ്പെട്ടിരിക്കുന്നു? {#how-is-this-different-from-regular-multisite}

സാധാരണ multisite ഒരു നെറ്റ്‌വർക്കിനുള്ളിൽ ഒന്നിലധികം സൈറ്റുകൾ അനുവദിക്കുന്നു. എന്നാൽ multinetwork, ഒന്നിലധികം നെറ്റ്‌വർക്കുകൾ (ഓരോന്നിനും അതിൻ്റേതായ സൈറ്റുകൾ സഹിതം) ഒരു single WordPress ഇൻസ്റ്റാളേഷനിൽ അനുവദിക്കുന്നു.

### ഇതിന് പ്രത്യേക സെർവർ കോൺഫിഗറേഷൻ ആവശ്യമുണ്ടോ? {#does-this-require-special-server-configuration}

അതെ, multinetwork-ന് പ്രത്യേക സെർവർ സജ്ജീകരണവും കോൺഫിഗറേഷനും ആവശ്യമാണ്. വിശദമായ സജ്ജീകരണ നിർദ്ദേശങ്ങൾക്കായി ദയവായി ഡോക്യുമെന്റേഷൻ പരിശോധിക്കുക.
