# ഇമെയിലുകൾ (Emails)

Ultimate Multisite: Emails addon ഉപയോഗിച്ച് network owners-ന് sites-നൊപ്പം ഉപഭോക്തൃ ഇമെയിൽ അക്കൗണ്ടുകൾ സജ്ജീകരിക്കാനും കൈകാര്യം ചെയ്യാനും കഴിയും. പതിപ്പ് 1.1.0-ൽ ഉപഭോക്താക്കൾക്ക് നേരിട്ട് ഉപയോഗിക്കാവുന്ന ഇമെയിൽ അക്കൗണ്ട് മാനേജ്‌മെന്റ്, webmail single sign-on, CyberPanel forwarder സപ്പോർട്ട്, ചെക്ക്ഔട്ട് സമയത്തെ മെച്ചപ്പെടുത്തിയ അക്കൗണ്ട് സജ്ജീകരണം എന്നിവ ഉൾപ്പെടുത്തിയിരിക്കുന്നു.

## പ്രധാന സവിശേഷതകൾ (Key Features)

* **ഉപഭോക്തൃ ഇമെയിൽ അക്കൗണ്ട് വിഡ്ജറ്റ് (Customer email account widget)**: സപ്പോർട്ട് ടിക്കറ്റ് തുറക്കാതെ തന്നെ, അവരുടെ അക്കൗണ്ട് പേജിൽ നിന്ന് ഉപഭോക്താക്കൾക്ക് ഇമെയിൽ അക്കൗണ്ടുകൾ ഉണ്ടാക്കാനും കൈകാര്യം ചെയ്യാനും കഴിയും.
* **Webmail SSO-യും മാജിക് ലിങ്കുകളും (Webmail SSO and magic links)**: പ്രൊവൈഡർ പിന്തുണയ്ക്കുന്ന സാഹചര്യങ്ങളിൽ, അക്കൗണ്ട് പേജിൽ നിന്ന് ഒറ്റ ക്ലിക്കിൽ ഓതന്റിക്കേഷൻ വഴി ഉപഭോക്താക്കൾക്ക് നേരിട്ട് webmail തുറക്കാൻ കഴിയും.
* **പാസ്‌വേർഡ് മാറ്റങ്ങൾ (Password changes)**: ഇമെയിൽ മാനേജ്‌മെന്റ് വിഡ്ജറ്റിൽ നിന്ന് ഉപഭോക്താക്കൾക്കും അഡ്മിനിസ്ട്രേറ്റർമാർക്കും മെയിൽബോക്‌സ് പാസ്‌വേഡുകൾ അപ്‌ഡേറ്റ് ചെയ്യാൻ കഴിയും.
* **ചെക്ക്ഔട്ട് സജ്ജീകരണം (Checkout provisioning)**: ഉപഭോക്തൃ ഇമെയിൽ അക്കൗണ്ട് സൃഷ്ടിക്കൽ ചെക്ക്ഔട്ടിനിടയിൽ നടക്കാൻ കഴിയും. അതിനാൽ, പർച്ചേസ് പ്രോസസ്സ് പൂർത്തിയാകുമ്പോൾ തന്നെ മെയിൽബോക്‌സ് തയ്യാറായിരിക്കും.
* **CyberPanel forwarders**: CyberPanel പിന്തുണയ്ക്കുന്ന network-കൾക്ക് ഇമെയിൽ സർവീസ് ഫ്ലോയുടെ ഭാഗമായി ഇമെയിൽ ഫോർവേഡറുകൾ ഉണ്ടാക്കാനും കൈകാര്യം ചെയ്യാനും കഴിയും.
* **പ്രൊവൈഡർ-അവബോധമുള്ള ക്രമീകരണങ്ങൾ (Provider-aware settings)**: ക്രമീകരണ പേജ്, സജീവവും കോൺഫിഗർ ചെയ്തതുമായ ഇന്റഗ്രേഷനുകൾ മാത്രമേ ഫിൽട്ടർ ചെയ്യുകയുള്ളൂ. അതിനാൽ, ഉപയോഗിക്കാൻ കഴിയുന്ന ഓപ്ഷനുകൾ മാത്രമേ അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് കാണാൻ കഴിയൂ.

## ആവശ്യകതകൾ (Requirements)

* Ultimate Multisite 2.0.0 അല്ലെങ്കിൽ അതിലും ഉയർന്നത്
* WordPress Multisite ഇൻസ്റ്റാളേഷൻ
* കുറഞ്ഞത് ഒരു കോൺഫിഗർ ചെയ്ത ഇമെയിൽ പ്രൊവൈഡർ ഇന്റഗ്രേഷൻ
* മെയിൽബോക്‌സുകൾ സൃഷ്ടിക്കാനും, പാസ്‌വേഡുകൾ മാറ്റാനും, webmail ആക്‌സസ് ലിങ്കുകൾ നൽകാനും (സാധിക്കുമെങ്കിൽ) അനുമതിയുള്ള പ്രൊവൈഡർ API credentials

## ഇൻസ്റ്റാളേഷൻ (Installation)

1. പ്ലഗിൻ ഫയലുകൾ `/wp-content/plugins/ultimate-multisite-emails` എന്നതിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക.
2. WordPress network admin-ൽ **Plugins** എന്നതിൽ നിന്ന് network-ന് addon ആക്ടിവേറ്റ് ചെയ്യുക.
3. **Ultimate Multisite** > **Settings** എന്നതിലേക്ക് പോയി addon ഇമെയിൽ ക്രമീകരണങ്ങൾ തുറക്കുക.
4. ഉപഭോക്താക്കൾ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന ഇമെയിൽ പ്രൊവൈഡർ ഇന്റഗ്രേഷനുകൾ എനേബിൾ ചെയ്യുക.
5. ക്രമീകരണങ്ങൾ സേവ് ചെയ്യുക, തുടർന്ന് ഒരു പ്രൊഡക്ഷൻ അല്ലാത്ത ഉപഭോക്താവിനെയോ staging സൈറ്റിനെയോ ഉപയോഗിച്ച് അക്കൗണ്ട് സൃഷ്ടിക്കൽ പരീക്ഷിക്കുക.

## ഉപഭോക്തൃ അക്കൗണ്ട് ഇമെയിൽ മാനേജ്‌മെന്റ് (Customer Account Email Management)

പതിപ്പ് 1.1.0 ഉപഭോക്തൃ അക്കൗണ്ട് പേജിലേക്ക് ഒരു ഇമെയിൽ മാനേജ്‌മെന്റ് വിഡ്ജറ്റ് ചേർക്കുന്നു. ഉപഭോക്താക്കൾക്ക് സ്വന്തം സൈറ്റുകൾക്കായി മെയിൽബോക്‌സ് സൃഷ്ടിക്കാനും പരിപാലിക്കാനും സ്വയം സേവനം (self-service) ചെയ്യേണ്ട സാഹചര്യങ്ങളിൽ ഇത് ഉപയോഗിക്കുക.

അക്കൗണ്ട് പേജിൽ നിന്ന്, ഉപഭോക്താക്കൾക്ക്:

1. അവരുടെ നിലവിലെ അംഗത്വത്തെയോ സൈറ്റിനെയോ ബന്ധിപ്പിച്ച ഇമെയിൽ അക്കൗണ്ടുകൾ കാണാൻ കഴിയും.
2. അവരുടെ പ്ലാൻ അധിക അക്കൗണ്ടുകൾ അനുവദിക്കുന്നുണ്ടെങ്കിൽ പുതിയ മെയിൽബോക്‌സ് സൃഷ്ടിക്കാൻ കഴിയും.
3. നിലവിലുള്ള മെയിൽബോക്‌സ് പാസ്‌വേർഡ് മാറ്റാൻ കഴിയും.
4. പിന്തുണയ്ക്കുന്ന webmail പ്രൊവൈഡർമാർ വഴി SSO അല്ലെങ്കിൽ മാജിക്-ലിങ്ക് ലോഗിൻ വഴി തുറക്കാൻ കഴിയും.

ഈ വിഡ്ജറ്റ് Ultimate Multisite account-page currents സിസ്റ്റം ഉപയോഗിക്കുന്നു. അതിനാൽ, ഉപഭോക്താവ് കാണുന്ന അക്കൗണ്ട് സന്ദർഭത്തിന് (account context) മാത്രമായി പ്രവർത്തനങ്ങൾ പരിമിതപ്പെടുത്തിയിരിക്കുന്നു. ഇത് മറ്റൊരു അംഗത്വത്തെയോ സൈറ്റിനെയോ ഉള്ള ഇമെയിൽ അക്കൗണ്ടുകൾ കൈകാര്യം ചെയ്യുന്നത് തടയുന്നു.

## Webmail SSO-യും മാജിക്-ലിങ്ക് ലോഗിനും (Webmail SSO and Magic-Link Login)

Webmail SSO ഉപഭോക്താക്കൾക്ക് അക്കൗണ്ട് പേജിൽ നിന്ന് നേരിട്ട് **Open webmail** ചെയ്യാനുള്ള ഓപ്ഷൻ നൽകുന്നു. സജീവമായ പ്രൊവൈഡർ ഇത് പിന്തുണയ്ക്കുന്നുവെങ്കിൽ, addon ഒരു short-lived access URL ആവശ്യപ്പെടുകയും, ഉപഭോക്താവിനോട് മെയിൽബോക്‌സ് credentials കോപ്പി ചെയ്യാൻ ആവശ്യപ്പെടാതെ webmail-ലേക്ക് റീഡയറക്ട് ചെയ്യുകയും ചെയ്യുന്നു.

പ്രൊവൈഡർമാരുടെ പെരുമാറ്റം വ്യത്യാസപ്പെടാം:

* Native SSO API ഉള്ള പ്രൊവൈഡർമാർക്ക് നേരിട്ടുള്ള ഓതന്റിക്കേറ്റഡ് webmail സെഷൻ സൃഷ്ടിക്കാൻ കഴിയും.
* മാജിക്-ലിങ്ക് സപ്പോർട്ട് ഉള്ള പ്രൊവൈഡർമാർക്ക് സമയപരിമിതമായ ലോഗിൻ ലിങ്ക് സൃഷ്ടിക്കാൻ കഴിയും.
* SSO അല്ലെങ്കിൽ മാജിക്-ലിങ്ക് സപ്പോർട്ട് ഇല്ലാത്ത പ്രൊവൈഡർമാർക്ക് ഇമെയിൽ ബോക്‌സ് വിശദാംശങ്ങളും പാസ്‌വേർഡ് മാനേജ്‌മെന്റ് പ്രവർത്തനങ്ങളും കാണിക്കണം, പക്ഷേ ഒറ്റ ക്ലിക്കുള്ള webmail ബട്ടൺ ഉണ്ടാകില്ല.

നിങ്ങൾ ആവശ്യമായ API credentials കോൺഫിഗർ ചെയ്തിട്ടുള്ള പ്രൊവൈഡർമാർക്ക് മാത്രമേ webmail ആക്‌സസ് എനേബിൾ ചെയ്യുക. ഒരു പ്രൊവൈഡറിന് ആക്ഷൻ കാണുന്നില്ലെങ്കിൽ, ആദ്യം ആ പ്രൊവൈഡർ സജീവമാണോ, കോൺഫിഗർ ചെയ്തിട്ടുണ്ടോ, webmail ലോഗിൻ ലിങ്കുകൾ പിന്തുണയ്ക്കുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കുക.

## ചെക്ക്ഔട്ട് സജ്ജീകരണം (Checkout Provisioning)

ചെക്ക്ഔട്ടിന്റെ ഭാഗമായി addon ഉപഭോക്തൃ ഇമെയിൽ അക്കൗണ്ടുകൾ സൃഷ്ടിക്കാൻ കഴിയും. പ്ലാനിൽ ഡിഫോൾട്ടായി ഒരു മെയിൽബോക്‌സ് ഉൾപ്പെടുന്നുണ്ടെങ്കിലോ, ചെക്ക്ഔട്ട് ഫോം ഇമെയിൽ അക്കൗണ്ട് വിശദാംശങ്ങൾ ശേഖരിക്കുന്നുണ്ടെങ്കിലോ ഇത് ഉപകാരപ്രദമാണ്.

ശുപാർശ ചെയ്യുന്ന രീതി:

1. ചെക്ക്ഔട്ട് സജ്ജീകരണം എനേബിൾ ചെയ്യുന്നതിന് മുമ്പ് പ്രൊവൈഡറും പ്ലാൻ ലിമിറ്റുകളും കോൺഫിഗർ ചെയ്യുക.
2. ഒരു staging ഉൽപ്പന്നവുമായി പർച്ചേസ് നടത്തി, ചെക്ക്ഔട്ടിന് ശേഷം മെയിൽബോക്‌സ് സൃഷ്ടിക്കപ്പെടുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കുക.
3. പുതിയ മെയിൽബോക്‌സ് ഇമെയിൽ വിഡ്ജറ്റിൽ കാണുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കാൻ ഉപഭോക്തൃ അക്കൗണ്ട് പേജ് പരിശോധിക്കുക.
4. പ്രൊഡക്ഷൻ ഉൽപ്പന്നങ്ങൾക്കായി ഈ ഫ്ലോ എനേബിൾ ചെയ്യുന്നതിന് മുമ്പ് പാസ്‌വേർഡ് മാറ്റൽ, webmail പ്രവർത്തനങ്ങൾ എന്നിവ ഉറപ്പാക്കുക.

സജ്ജീകരണം പരാജയപ്പെട്ടാൽ, മെയിൽബോക്‌സിന് അഡ്മിനിസ്ട്രേറ്റർ പിന്തുടരൽ ആവശ്യമാണെങ്കിലും ഉപഭോക്തൃ അക്കൗണ്ടും അംഗത്വവും നിലനിൽക്കാൻ കഴിയും. വീണ്ടും ശ്രമിക്കുന്നതിന് മുമ്പ് Ultimate Multisite activity logs-ഉം പ്രൊവൈഡർ logs-ഉം പരിശോധിക്കുക.

## CyberPanel ഇമെയിൽ ഫോർവേഡറുകൾ (CyberPanel Email Forwarders)

CyberPanel പിന്തുണയ്ക്കുന്ന network-കൾക്ക്, പതിപ്പ് 1.1.0 ഇമെയിൽ ഫോർവേഡർ സപ്പോർട്ട് ചേർക്കുന്നു. `info@customer-domain.test` പോലുള്ള വിലാസങ്ങൾ ഒരു മുഴുവൻ മെയിൽബോക്‌സ് സൃഷ്ടിക്കാതെ മറ്റൊരു ഇൻബോക്‌സിലേക്ക് മെയിൽ റീഡയറക്ട് ചെയ്യേണ്ട സാഹചര്യങ്ങളിൽ ഫോർവേഡറുകൾ ഉപയോഗിക്കുക.

ഫോർവേഡറുകൾ നൽകുന്നതിന് മുമ്പ്, CyberPanel ഇന്റഗ്രേഷൻ കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്നും ലഭ്യമാണെന്നും ഉറപ്പാക്കുക. ആവശ്യമായ host, username, password, package constants എന്നിവയ്ക്കായി [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) ഗൈഡ് കാണുക.

## പതിവായി ചോദിക്കുന്ന ചോദ്യങ്ങൾ (Frequently Asked Questions)

## ഉപഭോക്താക്കൾക്ക് സ്വന്തമായി ഇമെയിൽ അക്കൗണ്ടുകൾ സൃഷ്ടിക്കാൻ കഴിയുമോ?

അതെ. ഉപഭോക്തൃ അക്കൗണ്ട് വിഡ്ജറ്റ് എനേബിൾ ചെയ്യുകയും പ്ലാൻ ഇമെയിൽ അക്കൗണ്ടുകൾ അനുവദിക്കുകയും ചെയ്താൽ, അക്കൗണ്ട് പേജിൽ നിന്ന് ഉപഭോക്താക്കൾക്ക് സ്വന്തം മെയിൽബോക്‌സുകൾ സൃഷ്ടിക്കാനും കൈകാര്യം ചെയ്യാനും കഴിയും.

## എല്ലാ പ്രൊവൈഡർമാരും webmail SSO പിന്തുണയ്ക്കുന്നുണ്ടോ?

ഇല്ല. SSO-യും മാജിക്-ലിങ്ക് സപ്പോർട്ടും സജീവമായ ഇമെയിൽ പ്രൊവൈഡറിനെ ആശ്രയിച്ചിരിക്കുന്നു. ആക്‌സസ്-ലിങ്ക് API ഇല്ലാത്ത പ്രൊവൈഡർമാർക്ക് പോലും മെയിൽബോക്‌സ് സൃഷ്ടിക്കാനും പാസ്‌വേർഡ് മാറ്റാനും കഴിയും.

## അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് ഉപഭോക്താക്കൾക്കായി ഇമെയിൽ അക്കൗണ്ടുകൾ സൃഷ്ടിക്കാൻ കഴിയുമോ?

അതെ. അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് അഡ്മിൻ ഇന്റർഫേസ് വഴി ഇമെയിൽ അക്കൗണ്ടുകൾ സൃഷ്ടിക്കാൻ കഴിയും. അക്കൗണ്ട് സൃഷ്ടിക്കുമ്പോൾ initial പാസ്‌വേർഡ് സജ്ജമാすることも ഇതിൽ ഉൾപ്പെടുന്നു.

## ഫോർവേഡറുകൾ മെയിൽബോക്‌സുകൾ പോലെയാണോ?

ഇല്ല. ഒരു മെയിൽബോക്‌സ് മെയിൽ സംഭരിക്കുകയും ലോഗിൻ credentials നൽകുകയും ചെയ്യുന്നു. ഒരു ഫോർവേഡർ മെയിൽ മറ്റൊരു വിലാസത്തിലേക്ക് റീഡയറക്ട് ചെയ്യുക മാത്രമാണ് ചെയ്യുന്നത്, അതിനാൽ പ്രത്യേക ഇൻബോക്‌സ് ലഭിക്കില്ല.
