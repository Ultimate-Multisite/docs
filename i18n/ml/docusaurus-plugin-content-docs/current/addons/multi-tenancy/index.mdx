# മൾട്ടി-ടെനൻസി

പ്രകടനത്തിനും സുരക്ഷയ്ക്കുമായി ഉപസൈറ്റുകൾ സർവറുകളിലും ഡാറ്റാബേസുകളിലും വിതരണം ചെയ്യുക

## പതിപ്പ് 1.2.0 റിലീസ് പ്രധാനാംശങ്ങൾ {#version-120-release-highlights}

Multi-Tenancy 1.2.0 ഉൽപ്പാദന ഐസൊലേഷൻ പ്രവാഹങ്ങൾക്കായി സ്വയംഭരണ ടെനന്റ് അടിസ്ഥാനം ചേർക്കുന്നു:

- കുറച്ചു സമയം മാത്രം നിലനിൽക്കുന്ന, ആവർത്തനത്തിൽ നിന്ന് സംരക്ഷിതമായ SSO സന്ദർശനങ്ങളോടുകൂടിയ സ്റ്റേറ്റ്ലെസ് ടെനന്റ് ഓട്ടോലോഗിൻ.
- സ്വയംഭരണ ടെനന്റുകൾക്കായി ഓരോ ഉപസൈറ്റിനും പ്രത്യേക ഡാറ്റാബേസ്, ഫയൽസിസ്റ്റം ഐസൊലേഷൻ.
- ഹോസ്റ്റ്-അവെയർ ടെനന്റ് പ്രൊവിഷനിംഗിനായി Hostinger മൾട്ടി-ടെനൻസി ശേഷി.
- പഴയ ഡാറ്റാ പരിശോധനകൾക്കും അസിങ്ക് പുഷ് പ്രോസസ്സിംഗിനുമായി സ്വയംഭരണ മൈഗ്രേഷൻ പരിശോധനാ ഗേറ്റുകൾ.
- Bedrock, FrankenPHP, localhost ഇൻസ്റ്റാളുകൾക്കായി ക്രമീകരിക്കാവുന്ന ഒരേ-മെഷീൻ ഡാറ്റാബേസ് ഹോസ്റ്റ് ബൈൻഡിംഗ്.
- ടെനന്റ് ബൂട്ട്‌സ്ട്രാപ്പ്, ഡാറ്റാബേസ് റൈറ്റർ, സ്കീമ ബൂട്ട്‌സ്ട്രാപ്പ്, ഉപയോക്തൃ പ്രൊവിഷനിംഗ് മെച്ചപ്പെടുത്തലുകൾ.
- ടെനന്റ് ക്രെഡൻഷ്യലുകൾ വൃത്തിയാക്കുന്ന കൂടുതൽ സുരക്ഷിതമായ സ്വയംഭരണ ടിയർഡൗൺ.

[സ്വയംഭരണ മൈഗ്രേഷൻ പ്രവാഹം](/user-guide/getting-started/sovereign-migration-workflow), [മൾട്ടി-ടെനൻസി ഐസൊലേഷൻ](/user-guide/configuration/multi-tenancy-isolation), [സ്വയംഭരണ ടെനന്റ് പ്രവർത്തനങ്ങൾ](/user-guide/administration/sovereign-tenant-operations) എന്നിവയ്ക്കായുള്ള ഉപയോക്തൃ ഗൈഡുകൾ കാണുക.

## ആവശ്യകതകൾ {#requirements}

- WordPress 5.3 അല്ലെങ്കിൽ അതിനുമുകളിൽ
- PHP 7.4 അല്ലെങ്കിൽ അതിനുമുകളിൽ
- Multisite Ultimate പ്ലഗിൻ (സജീവം)

## ഇൻസ്റ്റാളേഷൻ {#installation}

1. അഡോൺ ഫയലുകൾ നിങ്ങളുടെ `/wp-content/plugins/` ഡയറക്ടറിയിലേക്ക് അപ്‌ലോഡ് ചെയ്യുക
2. WordPress-ലെ 'Plugins' മെനുവിലൂടെ പ്ലഗിൻ സജീവമാക്കുക
3. നെറ്റ്‌വർക്ക് അഡ്മിനിൽ അഡോൺ ക്രമീകരണങ്ങൾ കോൺഫിഗർ ചെയ്യുക

## വികസനം {#development}

### സജ്ജീകരണം {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### ടെസ്റ്റിംഗ് {#testing}

അഡോണിൽ സമഗ്രമായ ടെസ്റ്റിംഗ് ഫ്രെയിംവർക്ക് ഉൾപ്പെടുന്നു:

- **യൂണിറ്റ് ടെസ്റ്റുകൾ**: WordPress ടെസ്റ്റ് സ്യൂട്ടിനോടുകൂടിയ PHPUnit അടിസ്ഥാനമാക്കിയുള്ള ടെസ്റ്റുകൾ
- **അടിസ്ഥാന ടെസ്റ്റ് ക്ലാസ്**: പൊതുവായ ടെസ്റ്റ് സാഹചര്യങ്ങൾക്കായി സഹായക രീതികൾ നൽകുന്നു
- **ടെസ്റ്റ് കവറേജ്**: കോഡ് കവറേജ് ട്രാക്ക് ചെയ്യാൻ ക്രമീകരിച്ചിരിക്കുന്നു
- **മൾട്ടിസൈറ്റ് ടെസ്റ്റിംഗ്**: ടെസ്റ്റുകൾ മൾട്ടിസൈറ്റ് പരിസ്ഥിതിയിൽ പ്രവർത്തിക്കുന്നു

### കോഡ് സ്റ്റാൻഡേർഡുകൾ {#code-standards}

- **WordPress കോഡിംഗ് സ്റ്റാൻഡേർഡുകൾ**: PHPCS വഴി നടപ്പിലാക്കുന്നു
- **PHP 7.4+ അനുയോജ്യത**: ആധുനിക PHP സവിശേഷതകൾ പിന്തുണയ്ക്കുന്നു
- **സ്റ്റാറ്റിക് വിശകലനം**: ടൈപ്പ് പരിശോധനയ്ക്കായി PHPStan ഇന്റഗ്രേഷൻ
- **കോഡ് ആധുനികവൽക്കരണം**: PHP അപ്ഗ്രേഡുകൾക്കായുള്ള Rector നിയമങ്ങൾ

### ബിൽഡ് സിസ്റ്റം {#build-system}

- **അസറ്റ് പ്രോസസ്സിംഗ്**: CSS/JS ഫയലുകൾക്കായുള്ള മിനിഫിക്കേഷൻ
- **പരിഭാഷ**: POT ഫയൽ സൃഷ്ടിക്കൽ
- **പാക്കേജിംഗ്**: വിതരണത്തിനായി ആർക്കൈവ് സൃഷ്ടിക്കൽ
- **വികസന മോഡ്**: ഡീബഗിംഗിനായി മിനിഫൈ ചെയ്യാത്ത അസറ്റുകൾ

## കോൺഫിഗറേഷൻ {#configuration}

നെറ്റ്‌വർക്ക് അഡ്മിൻ ഇന്റർഫേസിലൂടെ ലഭ്യമാകുന്ന വിവിധ കോൺഫിഗറേഷൻ ഓപ്ഷനുകൾ ഈ അഡോൺ പിന്തുണയ്ക്കുന്നു.

### സ്വയംഭരണ ടെനന്റ് കോൺഫിഗറേഷൻ {#sovereign-tenant-configuration}

സ്വയംഭരണ ടെനന്റുകൾക്കായി, ടെനന്റിലേക്ക് ട്രാഫിക് റൂട്ട് ചെയ്യുന്നതിന് മുമ്പ് ടെനന്റ് ഡാറ്റാബേസ്, ഫയൽസിസ്റ്റം റൂട്ട്, ഡാറ്റാബേസ് ഹോസ്റ്റ് ബൈൻഡിംഗ്, ഹോസ്റ്റ് ശേഷി എന്നിവ കോൺഫിഗർ ചെയ്യുക. ഡാറ്റാബേസ് ക്രെഡൻഷ്യലുകൾ, ഹോസ്റ്റ് ബൈൻഡിംഗ്, റൂട്ടിംഗ്, ഉപയോക്തൃ പ്രൊവിഷനിംഗ് എന്നിവയെ ബാധിക്കുന്ന ഓരോ മാറ്റത്തിനുശേഷവും മൈഗ്രേഷൻ പരിശോധന നടത്തുക.

ടെനന്റ് റൺടൈമും ഡാറ്റാബേസ് ഗ്രാന്റുകളും ലോക്കൽ സോക്കറ്റ് അല്ലെങ്കിൽ ഒരേ-മെഷീൻ ആക്‌സസ് പ്രതീക്ഷിക്കുമ്പോൾ മാത്രം `localhost` ഉപയോഗിക്കുക. നിങ്ങളുടെ പ്രൊവൈഡർ കോൺഫിഗർ ചെയ്ത ഗ്രാന്റ് ഹോസ്റ്റ് അതാണെങ്കിൽ `127.0.0.1` അല്ലെങ്കിൽ ഒരു സർവീസ് ഹോസ്റ്റ്നെയിം ഉപയോഗിക്കുക.

### Hostinger ശേഷി {#hostinger-capability}

Hostinger മൾട്ടി-ടെനൻസി ശേഷി പങ്കിട്ട Hostinger ഇന്റഗ്രേഷൻ ടോക്കണോടൊപ്പം പ്രവർത്തിക്കുന്നു. ഡൊമെയ്ൻ hPanel-ൽ ഉണ്ടെന്ന്, DNS Hostinger Account-ലേക്ക് പോയിന്റ് ചെയ്യുന്നുവെന്ന്, പരിശോധനയ്ക്കുമുമ്പ് ടെനന്റ് ഡാറ്റാബേസ് ക്രമീകരണങ്ങൾ hPanel-ുമായി പൊരുത്തപ്പെടുന്നുവെന്ന് സ്ഥിരീകരിക്കുക.

## ഹുക്കുകളും ഫിൽറ്ററുകളും {#hooks-and-filters}

### ആക്ഷനുകൾ {#actions}

- `ultimate-multisite-addon-template_init` - അഡോൺ ആരംഭിച്ചതിന് ശേഷം പ്രവർത്തിക്കുന്നു
- `ultimate-multisite-addon-template_loaded` - എല്ലാ അഡോൺ ഫയലുകളും ലോഡ് ചെയ്തതിന് ശേഷം പ്രവർത്തിക്കുന്നു

### ഫിൽറ്ററുകൾ {#filters}

- `ultimate-multisite-addon-template_settings` - അഡോൺ ക്രമീകരണങ്ങൾ മാറ്റുക
- `ultimate-multisite-addon-template_enabled` - അഡോൺ സജീവമാക്കൽ/നിഷ്ക്രിയമാക്കൽ നില ഓവർറൈഡ് ചെയ്യുക
