# ഉപകരണങ്ങളും കോൺഫിഗറേഷനും

നിങ്ങളുടെ WordPress സൈറ്റ് വിശകലനം ചെയ്യാനും രൂപകൽപ്പന ചെയ്യാനും നിർമ്മിക്കാനും Theme Builder ഒരു കൂട്ടം ഉപകരണങ്ങൾ ഉപയോഗിക്കുന്നു. പതിപ്പ് 1.16.0-ൽ, **sd-ai-agent/site-scrape ഇപ്പോൾ Tier 1 ഉപകരണമാണ്**, അതിനാൽ Theme Builder-ൽ ഇത് ഡിഫോൾട്ടായി ലഭ്യമാണ്. Superdav AI Agent v1.18.0 വിശ്വസനീയമായ ഡെവലപ്പർ പ്രവാഹങ്ങൾക്കായി വേർതിരിച്ച് വിതരണം ചെയ്യുന്ന Advanced അനുബന്ധ പാക്കേജും അവതരിപ്പിക്കുന്നു; ആ ഉപകരണങ്ങൾ WordPress.org കോർ പാക്കേജിന്റെ ഭാഗമല്ല, അവ പ്രത്യേകം ഇൻസ്റ്റാൾ ചെയ്ത് അനുമതി നൽകണം.

## ലഭ്യമായ ഉപകരണങ്ങൾ {#available-tools}

### Tier 1 ഉപകരണങ്ങൾ (എപ്പോഴും ലഭ്യം) {#tier-1-tools-always-available}

കൂടുതൽ കോൺഫിഗറേഷൻ ഒന്നുമില്ലാതെ Theme Builder-ൽ Tier 1 ഉപകരണങ്ങൾ ഡിഫോൾട്ടായി ലഭ്യമാണ്.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**ഉദ്ദേശ്യം:** ഡിസൈൻ പ്രചോദനം, ഉള്ളടക്ക ഘടന, പ്രവർത്തനക്ഷമത എന്നിവ വേർതിരിച്ചെടുക്കാൻ നിലവിലുള്ള വെബ്‌സൈറ്റുകൾ വിശകലനം ചെയ്യുക.

**കഴിവുകൾ:**

- **വെബ്‌സൈറ്റ് വിശകലനം** — മത്സരികളുടെ അല്ലെങ്കിൽ പ്രചോദനമായ വെബ്‌സൈറ്റുകൾ സ്ക്രേപ്പ് ചെയ്ത് വിശകലനം ചെയ്യുക
- **ഡിസൈൻ വേർതിരിച്ചെടുക്കൽ** — നിറങ്ങൾ, ഫോണ്ടുകൾ, ലേയൗട്ട് മാതൃകകൾ എന്നിവ തിരിച്ചറിയുക
- **ഉള്ളടക്ക ഘടന** — പേജ് ക്രമീകരണവും ശ്രേണിയും മനസ്സിലാക്കുക
- **സവിശേഷത കണ്ടെത്തൽ** — plugins-യും പ്രവർത്തനക്ഷമതയും തിരിച്ചറിയുക
- **പ്രകടന വിശകലനം** — പേജ് വേഗതയും ഓപ്റ്റിമൈസേഷനും പരിശോധിക്കുക
- **SEO വിശകലനം** — മെറ്റാ ടാഗുകളും ഘടനാപരമായ ഡാറ്റയും പരിശോധിക്കുക

**ഉപയോഗം:**

```
example.com-ന്റെ ഡിസൈൻ വിശകലനം ചെയ്ത് എന്റെ സൈറ്റിനുള്ള പ്രചോദനമായി ഉപയോഗിക്കുക.
```

**ഇത് വേർതിരിച്ചെടുക്കുന്നത്:**

- നിറ പാലറ്റും നിറ ഉപയോഗവും
- ടൈപ്പോഗ്രാഫി (ഫോണ്ടുകളും വലുപ്പങ്ങളും)
- ലേയൗട്ട് ഘടനയും ഇടവേളകളും
- നാവിഗേഷൻ മാതൃകകൾ
- ഉള്ളടക്ക ക്രമീകരണം
- ചിത്രങ്ങളും മീഡിയ ഉപയോഗവും
- ഇന്ററാക്ടീവ് ഘടകങ്ങൾ
- മൊബൈൽ പ്രതികരണക്ഷമത

**പരിമിതികൾ:**

- പാസ്‌വേഡ് സംരക്ഷിത സൈറ്റുകൾ സ്ക്രേപ്പ് ചെയ്യാൻ കഴിയില്ല
- robots.txt-യും സൈറ്റ് നയങ്ങളും മാനിക്കുന്നു
- ഡൈനാമിക് ഉള്ളടക്കം പിടിച്ചെടുക്കാൻ കഴിയണമെന്നില്ല
- JavaScript കൂടുതലുള്ള സൈറ്റുകളിൽ വിശകലനം പരിമിതമായേക്കാം
- വലിയ സൈറ്റുകൾ വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുക്കാം

### Tier 2 ഉപകരണങ്ങൾ (ഐച്ഛികം) {#tier-2-tools-optional}

Theme Builder ക്രമീകരണങ്ങളിൽ വ്യക്തമായി സജ്ജമാക്കിയാൽ Tier 2 ഉപകരണങ്ങൾ ലഭ്യമാണ്.

#### Advanced Analytics {#advanced-analytics}

വിശദമായ പ്രകടന മെട്രിക്കുകൾ നൽകുന്നു:

- പേജ് ലോഡ് സമയം
- Core Web Vitals
- SEO സ്കോർ
- ആക്‌സസിബിലിറ്റി സ്കോർ
- മികച്ച രീതികളുടെ സ്കോർ

#### Content Optimizer {#content-optimizer}

ഇവയ്ക്കുള്ള മെച്ചപ്പെടുത്തലുകൾ വിശകലനം ചെയ്ത് നിർദേശിക്കുന്നു:

- വായനാസൗകര്യം
- SEO ഓപ്റ്റിമൈസേഷൻ
- കീ워ഡ് ഉപയോഗം
- ഉള്ളടക്ക ഘടന
- പ്രവർത്തന ആഹ്വാനത്തിന്റെ സ്ഥാനം

### Tier 3 ഉപകരണങ്ങൾ (Premium) {#tier-3-tools-premium}

Tier 3 ഉപകരണങ്ങൾക്ക് അധിക കോൺഫിഗറേഷനോ Premium സവിശേഷതകളോ ആവശ്യമാണ്.

#### AI Content Generator {#ai-content-generator}

ഇവയ്ക്കായി ഉള്ളടക്കം സൃഷ്ടിക്കുന്നു:

- product വിവരണങ്ങൾ
- സേവന പേജുകൾ
- ബ്ലോഗ് പോസ്റ്റുകൾ
- മെറ്റാ വിവരണങ്ങൾ
- പ്രവർത്തന ആഹ്വാന കോപ്പി

#### Advanced Design Tools {#advanced-design-tools}

മുന്നേറ്റ ഡിസൈൻ കഴിവുകൾ നൽകുന്നു:

- കസ്റ്റം CSS സൃഷ്ടിക്കൽ
- ആനിമേഷൻ സൃഷ്ടിക്കൽ
- ഇന്ററാക്ടീവ് ഘടക ഡിസൈൻ
- മുന്നേറ്റ നിറ സിദ്ധാന്തം
- ടൈപ്പോഗ്രാഫി ഓപ്റ്റിമൈസേഷൻ

### Advanced അനുബന്ധ ഉപകരണങ്ങൾ (വേർതിരിച്ച പാക്കേജ്) {#advanced-companion-tools-separate-package}

Advanced അനുബന്ധ പാക്കേജ് കോർ Superdav AI Agent റിലീസിൽ നിന്ന് വേർതിരിച്ചാണ് വിതരണം ചെയ്യുന്നത്. ഉയർന്ന സ്വാധീനമുള്ള ഉപകരണങ്ങൾ ഉപയോഗിക്കാൻ അഡ്മിനിസ്ട്രേറ്റർമാർ agent-നെ വ്യക്തമായി അനുവദിക്കുന്ന വിശ്വസനീയ ഡെവലപ്പർ, സൈറ്റ് ഉടമ പരിസരങ്ങൾക്കായാണ് ഇത് ഉദ്ദേശിച്ചിരിക്കുന്നത്.

Advanced അനുബന്ധ കഴിവുകളിൽ ഇവ ഉൾപ്പെടാം:

- **ഫയൽ സിസ്റ്റം ഉപകരണങ്ങൾ** — വിശ്വസനീയ പരിസരങ്ങളിൽ അംഗീകരിച്ച ഫയലുകൾ പരിശോധിക്കുകയും മാറ്റുകയും ചെയ്യുക
- **ഡാറ്റാബേസ് ഉപകരണങ്ങൾ** — വ്യക്തമായി അനുമതി നൽകിയാൽ സൈറ്റ് ഡാറ്റ query ചെയ്യുകയോ update ചെയ്യുകയോ ചെയ്യുക
- **WP-CLI ഉപകരണങ്ങൾ** — WordPress പരിപാലനവും പരിശോധനാ കമാൻഡുകളും പ്രവർത്തിപ്പിക്കുക
- **REST dispatcher ഉപകരണങ്ങൾ** — നിയന്ത്രിത dispatch പ്രവാഹങ്ങളിലൂടെ രജിസ്റ്റർ ചെയ്ത REST endpoints വിളിക്കുക
- **Plugin builder ഉപകരണങ്ങൾ** — plugin code scaffold ചെയ്ത് ആവർത്തിച്ച് മെച്ചപ്പെടുത്തുക
- **Git snapshot ഉപകരണങ്ങൾ** — അപകടസാധ്യതയുള്ള ഡെവലപ്പർ പ്രവർത്തനങ്ങൾക്ക് മുമ്പ് snapshots സൃഷ്ടിക്കുക
- **ഉപയോക്തൃ മാനേജ്മെന്റ് ഉപകരണങ്ങൾ** — അനുമതിയുള്ളിടത്ത് ഉപയോക്തൃ രേഖകൾ പരിശോധിക്കാനോ ക്രമീകരിക്കാനോ സഹായിക്കുക
- **ബെഞ്ച്മാർക്ക് ഉപകരണങ്ങൾ** — വികസന അവലോകനത്തിനായി പ്രകടനമോ കഴിവുകളോ സംബന്ധിച്ച ബെഞ്ച്മാർക്കുകൾ പ്രവർത്തിപ്പിക്കുക

ഓരോ ഇൻസ്റ്റാളേഷനും പൊതുവായി ലഭ്യമായ Theme Builder ഉപകരണങ്ങളായി ഇവ രേഖപ്പെടുത്തരുത്. Advanced അനുബന്ധ പാക്കേജ് ഇൻസ്റ്റാൾ ചെയ്ത്, സജ്ജമാക്കി, വിശ്വസനീയ അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് മാത്രം പരിമിതപ്പെടുത്തിയാൽ മാത്രമേ ഇവ ലഭ്യമാകൂ.

## ഉപകരണ കോൺഫിഗറേഷൻ {#tool-configuration}

### ഉപകരണങ്ങൾ സജ്ജമാക്കൽ {#enabling-tools}

Theme Builder-ൽ അധിക ഉപകരണങ്ങൾ സജ്ജമാക്കാൻ:

1. **Gratis AI Agent → Settings** എന്നതിലേക്ക് പോകുക
2. **Theme Builder → Tools** എന്നതിലേക്ക് പോകുക
3. ആവശ്യാനുസരണം ഉപകരണങ്ങൾ ഓൺ/ഓഫ് ചെയ്യുക
4. ക്രമീകരണങ്ങൾ സംരക്ഷിക്കുക

### ഉപകരണ അനുമതികൾ {#tool-permissions}

ചില ഉപകരണങ്ങൾക്ക് അനുമതികൾ ആവശ്യമാണ്:

- **സൈറ്റ് സ്ക്രേപ്പിംഗ്** — ഇന്റർനെറ്റ് ആക്‌സസ് ആവശ്യമാണ്
- **Analytics** — Google Analytics ഇന്റഗ്രേഷൻ ആവശ്യമാണ്
- **ഉള്ളടക്ക സൃഷ്ടിക്കൽ** — API keys ആവശ്യമാണ്
- **Advanced സവിശേഷതകൾ** — Premium subscription ആവശ്യമായേക്കാം
- **Advanced അനുബന്ധ ഉപകരണങ്ങൾ** — വേർതിരിച്ച് വിതരണം ചെയ്യുന്ന Advanced പാക്കേജും വ്യക്തമായ അഡ്മിനിസ്ട്രേറ്റർ വിശ്വാസവും ആവശ്യമാണ്

### API കീകളും യോഗ്യതകളും {#api-keys-and-credentials}

അവ ആവശ്യമായ ഉപകരണങ്ങൾക്കായി API keys കോൺഫിഗർ ചെയ്യുക:

1. **Gratis AI Agent → Settings → API Keys** എന്നതിലേക്ക് പോകുക
2. ഓരോ ഉപകരണത്തിനുമുള്ള യോഗ്യതകൾ നൽകുക
3. കണക്ഷൻ പരിശോധിക്കുക
4. സുരക്ഷിതമായി സംരക്ഷിക്കുക

### Advanced അനുബന്ധ പാക്കേജ് ഇൻസ്റ്റാൾ ചെയ്യൽ {#installing-the-advanced-companion-package}

Advanced അനുബന്ധ പാക്കേജ് WordPress.org കോർ ZIP-ൽ നിന്ന് വേർതിരിച്ചാണ് റിലീസ് ചെയ്യുന്നത്. റിലീസിനായുള്ള ഔദ്യോഗിക project വിതരണ ചാനലിൽ നിന്ന് മാത്രം ഇത് ഇൻസ്റ്റാൾ ചെയ്യുക. തുടർന്ന് വിശ്വസനീയ അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് മാത്രം ആക്‌സസ് പരിമിതപ്പെടുത്തുക. പ്രൊഡക്ഷൻ സൈറ്റിൽ ഫയൽ സിസ്റ്റം, ഡാറ്റാബേസ്, WP-CLI, REST dispatcher, plugin builder, ഉപയോക്തൃ മാനേജ്മെന്റ്, അല്ലെങ്കിൽ ബെഞ്ച്മാർക്ക് ഉപകരണങ്ങൾ സജ്ജമാക്കുന്നതിന് മുമ്പ് മനുഷ്യ അംഗീകാര ഗേറ്റുകളും git snapshot പെരുമാറ്റവും പരിശോധിക്കുക.

## sd-ai-agent/site-scrape ഉപയോഗിക്കൽ {#using-sd-ai-agentsite-scrape}

### അടിസ്ഥാന ഉപയോഗം {#basic-usage}

ഒരു വെബ്‌സൈറ്റ് വിശകലനം ചെയ്യാൻ Theme Builder-നോട് ആവശ്യപ്പെടുക:

```
competitor.com-ൽ ഉള്ള എന്റെ മത്സരിയുടെ സൈറ്റിന്റെ ഡിസൈൻ വിശകലനം ചെയ്ത്
എന്റെ സൈറ്റിനുള്ള ഡിസൈൻ മെച്ചപ്പെടുത്തലുകൾ നിർദേശിക്കുക.
```

### നിർദ്ദിഷ്ട വിശകലനം {#specific-analysis}

നിർദ്ദിഷ്ട തരത്തിലുള്ള വിശകലനങ്ങൾ അഭ്യർത്ഥിക്കുക:

```
example.com-ൽ നിന്ന് നിറ പാലറ്റ് വേർതിരിച്ചെടുത്ത് പ്രചോദനമായി ഉപയോഗിക്കുക.
```

```
example.com-ന്റെ നാവിഗേഷൻ ഘടന വിശകലനം ചെയ്ത് സമാനമായ
ക്രമീകരണം എന്റെ സൈറ്റിൽ പ്രയോഗിക്കുക.
```

```
example.com-ന്റെ മൊബൈൽ പ്രതികരണക്ഷമത പരിശോധിച്ച് എന്റെ സൈറ്റ്
അതുപോലെ പ്രതികരണക്ഷമമാണെന്ന് ഉറപ്പാക്കുക.
```

### താരതമ്യ വിശകലനം {#comparison-analysis}

ഒന്നിലധികം സൈറ്റുകൾ താരതമ്യം ചെയ്യുക:

```
site1.com, site2.com എന്നിവയുടെ ഡിസൈനുകൾ താരതമ്യം ചെയ്ത് രണ്ടിന്റെയും
മികച്ച ഘടകങ്ങൾ ചേർന്ന ഒരു ഡിസൈൻ സൃഷ്ടിക്കുക.
```

## ഉപകരണ പരിമിതികളും പരിഗണനകളും {#tool-limitations-and-considerations}

### നിരക്ക് പരിധി {#rate-limiting}

- സർവർ അമിതഭാരം ഒഴിവാക്കാൻ സ്ക്രാപ്പിംഗിന് നിരക്ക് പരിധിയുണ്ട്
- ഒരേ site-ലേക്കുള്ള പല അഭ്യർത്ഥനകളും നിയന്ത്രിക്കപ്പെടാം
- വലിയ sites വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുക്കാം

### ഉള്ളടക്കത്തിന്റെ കൃത്യത {#content-accuracy}

- ഡൈനാമിക് ഉള്ളടക്കം പൂർണ്ണമായി പിടികൂടാനാകണമെന്നില്ല
- JavaScript ഉപയോഗിച്ച് റെൻഡർ ചെയ്ത ഉള്ളടക്കം അപൂർണ്ണമായിരിക്കാം
- തത്സമയ ഡാറ്റ പ്രതിഫലിക്കണമെന്നില്ല
- ചില ഉള്ളടക്കം പേവാളുകൾക്ക് പിന്നിലായിരിക്കാം

### നിയമപരവും നൈതികവുമായ പരിഗണനകൾ {#legal-and-ethical-considerations}

- robots.txt ഉം site നയങ്ങളും മാനിക്കുക
- പുനരുത്പാദനത്തിനായി പകർപ്പവകാശമുള്ള ഉള്ളടക്കം സ്ക്രാപ്പ് ചെയ്യരുത്
- പകർത്തലിനല്ല, പ്രചോദനത്തിനായി വിശകലനം ഉപയോഗിക്കുക
- എടുത്ത ഉള്ളടക്കം ഉപയോഗിക്കാൻ നിങ്ങൾക്ക് അവകാശങ്ങളുണ്ടെന്ന് സ്ഥിരീകരിക്കുക
- വിശകലനം ചെയ്ത sites-ന്റെ സേവന നിബന്ധനകൾ പാലിക്കുക

### പ്രകടനത്തെ ബാധിക്കുന്നത് {#performance-impact}

- വലിയ sites സ്ക്രാപ്പ് ചെയ്യാൻ സമയം എടുക്കാം
- ഒരേ സമയം ഒന്നിലധികം സ്ക്രാപ്പുകൾ മന്ദഗതിയിലായേക്കാം
- നെറ്റ്‌വർക്ക് കണക്റ്റിവിറ്റി വേഗതയെ ബാധിക്കുന്നു
- വലിയ ഡാറ്റാസെറ്റുകൾക്ക് കൂടുതൽ പ്രോസസ്സിംഗ് ആവശ്യമായേക്കാം

## മികച്ച രീതികൾ {#best-practices}

### Site വിശകലനം ഉപയോഗിക്കൽ {#using-site-analysis}

1. **ഒന്നിലധികം sites വിശകലനം ചെയ്യുക** — പല ഉറവിടങ്ങളിൽ നിന്ന് പ്രചോദനം ശേഖരിക്കുക
2. **ഘടനയിൽ ശ്രദ്ധിക്കുക** — ലേഔട്ട്, ഓർഗനൈസേഷൻ പാറ്റേണുകൾ പഠിക്കുക
3. **നിറങ്ങൾ എടുത്തെടുക്കുക** — നിറ പാലറ്റുകൾ ആരംഭ ബിന്ദുക്കളായി ഉപയോഗിക്കുക
4. **ടൈപ്പോഗ്രാഫി പഠിക്കുക** — നിങ്ങൾക്ക് ഇഷ്ടമുള്ള ഫോണ്ട് കോമ്പിനേഷനുകൾ തിരിച്ചറിയുക
5. **നാവിഗേഷൻ അവലോകനം ചെയ്യുക** — മെനു ഓർഗനൈസേഷൻ മനസ്സിലാക്കുക

### നൈതിക സ്ക്രാപ്പിംഗ് {#ethical-scraping}

1. **robots.txt മാനിക്കുക** — site മാർഗ്ഗനിർദ്ദേശങ്ങൾ പാലിക്കുക
2. **ഉള്ളടക്കം പകർത്തരുത്** — വിശകലനം പ്രചോദനത്തിനായി മാത്രം ഉപയോഗിക്കുക
3. **അവകാശങ്ങൾ സ്ഥിരീകരിക്കുക** — എടുത്ത ഉള്ളടക്കം ഉപയോഗിക്കാനാകുമെന്ന് ഉറപ്പാക്കുക
4. **ഉറവിടങ്ങൾ സൂചിപ്പിക്കുക** — പ്രചോദന ഉറവിടങ്ങൾക്ക് ക്രെഡിറ്റ് നൽകുക
5. **നിബന്ധനകൾ പാലിക്കുക** — site സേവന നിബന്ധനകൾ അനുസരിക്കുക

### ടൂളിന്റെ ഫലപ്രാപ്തി പരമാവധി ആക്കൽ {#maximizing-tool-effectiveness}

1. **വ്യക്തമായിരിക്കുക** — പ്രത്യേക തരത്തിലുള്ള വിശകലനം അഭ്യർത്ഥിക്കുക
2. **സന്ദർഭം നൽകുക** — നിങ്ങളുടെ site-ന്റെ ഉദ്ദേശ്യം വിശദീകരിക്കുക
3. **പ്രതീക്ഷകൾ നിശ്ചയിക്കുക** — നിങ്ങളുടെ ഡിസൈൻ ലക്ഷ്യങ്ങൾ വിവരിക്കുക
4. **ആവർത്തിച്ച് മെച്ചപ്പെടുത്തുക** — ഫലങ്ങളെ അടിസ്ഥാനമാക്കി മെച്ചപ്പെടുത്തുക
5. **ടൂളുകൾ സംയോജിപ്പിക്കുക** — സമഗ്രമായ വിശകലനത്തിനായി ഒന്നിലധികം ടൂളുകൾ ഉപയോഗിക്കുക

## പ്രശ്നപരിഹാരം {#troubleshooting}

### Site സ്ക്രാപ്പ് ചെയ്യാനാകുന്നില്ല {#site-wont-scrape}

- site പൊതുവായി ആക്സസ് ചെയ്യാനാകുന്നതാണോ എന്ന് പരിശോധിക്കുക
- robots.txt സ്ക്രാപ്പിംഗ് അനുവദിക്കുന്നുണ്ടോയെന്ന് സ്ഥിരീകരിക്കുക
- വേറൊരു site ശ്രമിക്കുക
- ഇന്റർനെറ്റ് കണക്ഷൻ പരിശോധിക്കുക
- പ്രശ്നം തുടരുകയാണെങ്കിൽ പിന്തുണയെ ബന്ധപ്പെടുക

### അപൂർണ്ണമായ വിശകലനം {#incomplete-analysis}

- site-ൽ ഡൈനാമിക് ഉള്ളടക്കം ഉണ്ടായിരിക്കാം
- JavaScript പൂർണ്ണമായി റെൻഡർ ചെയ്തിരിക്കണമെന്നില്ല
- വലിയ sites ടൈംഔട്ട് ആകാം
- പകരം നിർദ്ദിഷ്ട പേജുകൾ വിശകലനം ചെയ്യാൻ ശ്രമിക്കുക
- നിർദ്ദിഷ്ട വിശകലന തരങ്ങൾ അഭ്യർത്ഥിക്കുക

### മന്ദഗതിയിലുള്ള പ്രകടനം {#slow-performance}

- വലിയ sites വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുക്കും
- ഒരേ സമയം ഒന്നിലധികം സ്ക്രാപ്പുകൾ മന്ദഗതിയിലാകും
- നെറ്റ്‌വർക്ക് കണക്റ്റിവിറ്റി വേഗതയെ ബാധിക്കുന്നു
- തിരക്കില്ലാത്ത സമയങ്ങളിൽ ശ്രമിക്കുക
- ആദ്യം ചെറിയ ഭാഗങ്ങൾ വിശകലനം ചെയ്യുക
