# ടൂളുകളും കോൺഫിഗറേഷനും

നിങ്ങളുടെ WordPress സൈറ്റ് വിശകലനം ചെയ്യാനും ഡിസൈൻ ചെയ്യാനും നിർമ്മിക്കാനും Theme Builder ഒരു കൂട്ടം ടൂളുകൾ ഉപയോഗിക്കുന്നു. പതിപ്പ് 1.16.0-ൽ, **sd-ai-agent/site-scrape ഇപ്പോൾ ഒരു Tier 1 ടൂളാണ്**, ഇത് Theme Builder-ൽ ഡിഫോൾട്ടായി ലഭ്യമാക്കുന്നു.

## ലഭ്യമായ ടൂളുകൾ

### Tier 1 ടൂളുകൾ (എപ്പോഴും ലഭ്യമാണ്)

Tier 1 ടൂളുകൾ Theme Builder-ൽ അധിക കോൺഫിഗറേഷൻ ഇല്ലാതെ ഡിഫോൾട്ടായി ലഭ്യമാണ്.

#### sd-ai-agent/site-scrape

**ഉദ്ദേശ്യം:** ഡിസൈൻ പ്രചോദനം, ഉള്ളടക്ക ഘടന, പ്രവർത്തനക്ഷമത എന്നിവ വേർതിരിച്ചെടുക്കാൻ നിലവിലുള്ള വെബ്‌സൈറ്റുകൾ വിശകലനം ചെയ്യുന്നു.

**സാധ്യതകൾ:**

- **വെബ്‌സൈറ്റ് വിശകലനം** — എതിരാളികളുടെയോ പ്രചോദന വെബ്‌സൈറ്റുകളുടെയോ സൈപ്പ് ചെയ്യുകയും വിശകലനം ചെയ്യുകയും ചെയ്യുക
- **ഡിസൈൻ വേർതിരിച്ചെടുക്കൽ** — നിറങ്ങൾ, ഫോണ്ടുകൾ, ലേഔട്ട് പാറ്റേണുകൾ എന്നിവ തിരിച്ചറിയുക
- **ഉള്ളടക്ക ഘടന** — പേജ് ഓർഗനൈസേഷനും ഹൈറാർക്കിയും മനസ്സിലാക്കുക
- **ఫీച്ചർ കണ്ടെത്തൽ** — പ്ലഗിനുകളും പ്രവർത്തനക്ഷമതയും തിരിച്ചറിയുക
- **പ്രകടന വിശകലനം** — പേജ് വേഗതയും ഒപ്റ്റിമൈസേഷനും പരിശോധിക്കുക
- **SEO വിശകലനം** — മെറ്റാ ടാഗുകളും ഘടനാപരമായ ഡാറ്റയും (structured data) അവലോകനം ചെയ്യുക

**ഉപയോഗം:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**എന്തൊക്കെ വേർതിരിച്ചെടുക്കുന്നു:**

- കളർ പാലറ്റും നിറങ്ങളുടെ ഉപയോഗവും
- ടൈപ്പോഗ്രാഫി (ഫോണ്ടുകളും വലുപ്പങ്ങളും)
- ലേഔട്ട് ഘടനയും സ്പേസിംഗും
- നാവിഗേഷൻ പാറ്റേണുകൾ
- ഉള്ളടക്ക സംഘടന
- ചിത്രങ്ങളും മീഡിയ ഉപയോഗവും
- ഇന്ററാക്ടീവ് ഘടകങ്ങൾ
- മൊബൈൽ പ്രതികരണശേഷി (Mobile responsiveness)

**പരിമിതികൾ:**

- പാസ്‌വേർഡ് സംരക്ഷിതമായ സൈറ്റുകൾ സൈപ്പ് ചെയ്യാൻ കഴിയില്ല
- robots.txt, സൈറ്റ് നയങ്ങൾ എന്നിവ മാനിക്കുന്നു
- ഡൈനാമിക് ഉള്ളടക്കം (dynamic content) പിടിച്ചെടു নাও കഴിഞ്ഞേക്കും
- JavaScript-heavy ആയ സൈറ്റുകൾക്ക് പരിമിതമായ വിശകലനം ലഭിച്ചേക്കാം
- വലിയ സൈറ്റുകൾ വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുച്ചേക്കാം

### Tier 2 ടൂളുകൾ (ഓപ്ഷണൽ)

Theme Builder സെറ്റിംഗ്സിൽ വ്യക്തമായി പ്രവർത്തനക്ഷമമാക്കുമ്പോൾ Tier 2 ടൂളുകൾ ലഭ്യമാകും.

#### Advanced Analytics

വിശദമായ പ്രകടന മെട്രിക്‌സുകൾ നൽകുന്നു:

- പേജ് ലോഡ് സമയം
- Core Web Vitals
- SEO സ്കോർ
- Accessibility സ്കോർ
- മികച്ച രീതികൾ സ്കോർ (Best practices score)

#### Content Optimizer

ഇവയ്ക്കായി വിശകലനം ചെയ്യുകയും മെച്ചപ്പെടുത്തലുകൾ നിർദ്ദേശിക്കുകയും ചെയ്യുന്നു:

- വായനാക്ഷമത (Readability)
- SEO ഒപ്റ്റിമൈസേഷൻ
- കീവേഡ് ഉപയോഗം
- ഉള്ളടക്ക ഘടന
- Call-to-action സ്ഥാപിക്കാനുള്ള സ്ഥാനം

### Tier 3 ടൂളുകൾ (പ്രീമിയം)

Tier 3 ടൂളുകൾ അധിക കോൺഫിഗറേഷനോ പ്രീമിയം ഫീച്ചറുകളോ ആവശ്യമാണ്.

#### AI Content Generator

ഇവയ്ക്കായി ഉള്ളടക്കം സൃഷ്ടിക്കുന്നു:

- ഉൽപ്പന്ന വിവരണങ്ങൾ
- സേവന പേജുകൾ
- ബ്ലോഗ് പോസ്റ്റുകൾ
- മെറ്റാ വിവരണങ്ങൾ
- Call-to-action കോപ്പി

#### Advanced Design Tools

വിപുലമായ ഡിസൈൻ ശേഷികൾ നൽകുന്നു:

- കസ്റ്റം CSS ജനറേഷൻ
- ആനിമേഷൻ സൃഷ്ടിക്കൽ
- ഇന്ററാക്ടീവ് ഘടകം ഡിസൈൻ
- അഡ്വാൻസ്ഡ് കളർ തിയറി
- ടൈപ്പോഗ്രാഫി ഒപ്റ്റിമൈസേഷൻ

## ടൂൾ കോൺഫിഗറേഷൻ

### ടൂളുകൾ പ്രവർത്തനക്ഷമമാക്കൽ

Theme Builder-ൽ അധിക ടൂളുകൾ പ്രവർത്തനക്ഷമമാക്കാൻ:

1. **Gratis AI Agent → Settings** എന്നതിലേക്ക് പോകുക
2. **Theme Builder → Tools** എന്നതിലേക്ക് പോകുക
3. ആവശ്യമനുസരിച്ച് ടൂളുകൾ ഓൺ/ഓഫ് ചെയ്യുക
4. സെറ്റിംഗുകൾ സേവ് ചെയ്യുക

### ടൂൾ പെർമിഷനുകൾ

ചില ടൂളുകൾക്ക് പെർമിഷനുകൾ ആവശ്യമാണ്:

- **Site scraping** — ഇന്റർനെറ്റ് ആക്‌സസ് ആവശ്യമാണ്
- **Analytics** — Google Analytics സംയോജനം ആവശ്യമാണ്
- **Content generation** — API keys ആവശ്യമാണ്
- **Advanced features** — പ്രീമിയം സബ്‌സ്‌ക്രിപ്ഷൻ ആവശ്യമായി വന്നേക്കാം

### API Keys ഉം Credentials ഉം

അവ ആവശ്യമായ ടൂളുകൾക്കായി API keys കോൺഫിഗർ ചെയ്യുക:

1. **Gratis AI Agent → Settings → API Keys** ലേക്ക് പോകുക
2. ഓരോ ടൂളിനുമുള്ള credentials നൽകുക
3. കണക്ഷൻ ടെസ്റ്റ് ചെയ്യുക
4. സുരക്ഷിതമായി സേവ് ചെയ്യുക

## sd-ai-agent/site-scrape ഉപയോഗിക്കൽ

### അടിസ്ഥാന ഉപയോഗം

ഒരു വെബ്‌സൈറ്റ് വിശകലനം ചെയ്യാൻ Theme Builder-യോട് ആവശ്യപ്പെടുക:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### പ്രത്യേക വിശകലനം

വിശകലനത്തിന്റെ പ്രത്യേക തരങ്ങൾ ആവശ്യപ്പെടുക:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### താരതമ്യ വിശകലനം

複数の സൈറ്റുകൾ താരതമ്യം ചെയ്യുക:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## ടൂൾ പരിമിതികളും പരിഗണനകളും

### റേറ്റ് ലിമിറ്റിംഗ് (Rate Limiting)

- സെർവർ ഓവർലോഡ് തടയാൻ സൈപ്പിംഗിന് റേറ്റ് ലിമിറ്റ് ഉണ്ട്
- ഒരേ സൈറ്റിലേക്കുള്ള ഒന്നിലധികം അഭ്യർത്ഥനകൾ (requests) തടസ്സപ്പെടുത്താൻ സാധ്യതയുണ്ട്
- വലിയ സൈറ്റുകൾ വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുക്കാം

### ഉള്ളടക്ക കൃത്യത (Content Accuracy)

- ഡൈനാമിക് ഉള്ളടക്കം പൂർണ്ണമായി പിടിച്ചെടു নাও കഴിഞ്ഞേക്കും
- JavaScript ഉപയോഗിച്ച് റെൻഡർ ചെയ്യുന്ന ഉള്ളടക്കം അപൂർണ്ണമായേക്കാം
- തത്സമയ ഡാറ്റ (real-time data) പ്രതിഫലിക്കണമെന്നില്ല
- ചില ഉള്ളടക്കങ്ങൾ paywalls-ന് പിന്നിലായിരിക്കാം

### നിയമപരവും ധാർമികവുമായ പരിഗണനകൾ

- robots.txt, സൈറ്റ് നയങ്ങൾ എന്നിവ മാനിക്കുക
- പുനഃസൃഷ്ടിക്കാനായി കോപ്പിറൈറ്റ് ചെയ്ത ഉള്ളടക്കം സൈപ്പ് ചെയ്യരുത്
- പ്രചോദനത്തിനായി വിശകലനം ഉപയോഗിക്കുക, കോപ്പി ചെയ്യാൻ അല്ല
- വേർതിരിച്ചെടുത്ത ഉള്ളടക്കം ഉപയോഗിക്കാൻ നിങ്ങൾക്ക് അവകാശമുണ്ടോ എന്ന് പരിശോധിക്കുക
- വിശകലനം ചെയ്ത സൈറ്റുകളുടെ സേവന നിബന്ധനകൾ പാലിക്കുക

### പ്രകടന സ്വാധീനം (Performance Impact)

- വലിയ സൈറ്റുകൾ സൈപ്പ് ചെയ്യാൻ സമയമെടുക്കും
- ഒന്നിലധികം സമാന്തര സൈപ്പിംഗുകൾ (simultaneous scrapes) മന്ദഗതിയിലായിരിക്കും
- നെറ്റ്‌വർക്ക് കണക്റ്റിവിറ്റി വേഗതയെ ബാധിക്കുന്നു
- വലിയ ഡാറ്റാസെറ്റുകൾക്ക് കൂടുതൽ പ്രോസസ്സിംഗ് ആവശ്യമായി വന്നേക്കാം

## മികച്ച രീതികൾ (Best Practices)

### സൈറ്റ് വിശകലനം ഉപയോഗിക്കുമ്പോൾ

1. **ഒന്നിലധികം സൈറ്റുകൾ വിശകലനം ചെയ്യുക** — നിരവധി സ്രോതസ്സുകളിൽ നിന്ന് പ്രചോദനം ശേഖരിക്കുക
2. **ഘടനയിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുക** — ലേഔട്ട്, സംഘടന പാറ്റേണുകൾ എന്നിവ പഠിക്കുക
3. **നിറങ്ങൾ വേർതിരിച്ചെടുക്കുക** — കളർ പാലറ്റുകൾ ആരംഭ പോയിന്റുകളായി ഉപയോഗിക്കുക
4. **ടൈപ്പോഗ്രാഫി പഠിക്കുക** — നിങ്ങൾക്ക് ഇഷ്ടമുള്ള ഫോണ്ട് കോമ്പിനേഷനുകൾ തിരിച്ചറിയുക
5. **നാവിഗേഷൻ അവലോകനം ചെയ്യുക** — മെനു സംഘടന മനസ്സിലാക്കുക

### ധാർമിക സൈപ്പിംഗ് (Ethical Scraping)

1. **robots.txt മാനിക്കുക** — സൈറ്റ് മാർഗ്ഗനിർദ്ദേശങ്ങൾ പാലിക്കുക
2. **ഉള്ളടക്കം കോപ്പി ചെയ്യരുത്** — പ്രചോദനത്തിനായി മാത്രം വിശകലനം ഉപയോഗിക്കുക
3. **അവകാശങ്ങൾ പരിശോധിക്കുക** — വേർതിരിച്ചെടുത്ത ഉള്ളടക്കം ഉപയോഗിക്കാൻ നിങ്ങൾക്ക് കഴിയുമെന്ന് ഉറപ്പാക്കുക
4. **ഉറവിടങ്ങൾ രേഖപ്പെടുത്തുക** — പ്രചോദന സ്രോതസ്സുകൾക്ക് ക്രെഡിറ്റ് നൽകുക
5. **നിബന്ധനകൾ പാലിക്കുക** — സൈറ്റ് സേവന നിബന്ധനകൾ പാലിക്കുക

### ടൂൾ ഫലപ്രാപ്തി വർദ്ധിപ്പിക്കൽ

1. **കൃത്യമായിരിക്കുക** — പ്രത്യേക തരം വിശകലനം ആവശ്യപ്പെടുക
2. **സന്ദർഭം നൽകുക** — നിങ്ങളുടെ സൈറ്റിന്റെ ഉദ്ദേശ്യം വിശദീകരിക്കുക
3. **പ്രതീക്ഷകൾ സജ്ജമാക്കുക** — നിങ്ങളുടെ ഡിസൈൻ ലക്ഷ്യങ്ങൾ വിവരിക്കുക
4. **ആവർത്തിക്കുക** — ഫലങ്ങളുടെ അടിസ്ഥാനത്തിൽ മെച്ചപ്പെടുത്തുക
5. **ടൂളുകൾ സംയോജിപ്പിക്കുക** — സമഗ്രമായ വിശകലനത്തിനായി ഒന്നിലധികം ടൂളുകൾ ഉപയോഗിക്കുക

## ട്രബിൾഷൂട്ടിംഗ്

### സൈറ്റ് സൈപ്പ് ആകുന്നില്ല

- സൈറ്റ് പൊതുവായി ലഭ്യമാണോ എന്ന് പരിശോധിക്കുക
- robots.txt സൈപ്പിംഗിന് അനുമതി നൽകുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കുക
- മറ്റൊരു സൈറ്റ് പരീക്ഷിക്കുക
- ഇന്റർനെറ്റ് കണക്ഷൻ പരിശോധിക്കുക
- പ്രശ്നം തുടരുകയാണെങ്കിൽ സപ്പോർട്ടുമായി ബന്ധപ്പെടുക

### അപൂർണ്ണമായ വിശകലനം

- സൈറ്റിൽ ഡൈനാമിക് ഉള്ളടക്കം ഉണ്ടാകാം
- JavaScript പൂർണ്ണമായി റെൻഡർ ചെയ്യാ নাও കഴിഞ്ഞേക്കാം
- വലിയ സൈറ്റുകൾക്ക് ടൈമൗട്ട് ആവാം
- പകരം പ്രത്യേക പേജുകൾ വിശകലനം ചെയ്യാൻ ശ്രമിക്കുക
- പ്രത്യേക വിശകലന തരങ്ങൾ ആവശ്യപ്പെടുക

### കുറഞ്ഞ പ്രകടനം (Slow Performance)

- വലിയ സൈറ്റുകൾ വിശകലനം ചെയ്യാൻ കൂടുതൽ സമയം എടുക്കും
- ഒന്നിലധികം സമാന്തര സൈപ്പിംഗുകൾ മന്ദഗതിയിലായിരിക്കും
- നെറ്റ്‌വർക്ക് കണക്റ്റിവിറ്റി വേഗതയെ ബാധിക്കുന്നു
- തിരക്കല്ലാത്ത സമയത്ത് ശ്രമിക്കുക
- ആദ്യം ചെറിയ ഭാഗങ്ങൾ വിശകലനം ചെയ്യാൻ ശ്രമിക്കുക
