# മാനുവൽ പേയ്‌മെന്റുകൾ സജ്ജീകരിക്കുന്നത് (v2)

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite പതിപ്പ് 2.x-നെ പരാമർശിക്കുന്നു.**_

**Stripe** അല്ലെങ്കിൽ **PayPal** നിങ്ങളുടെ ഉപയോക്താക്കൾക്ക് ലഭ്യമല്ലെങ്കിൽ മറ്റ് പേയ്‌മെന്റ് മാർഗ്ഗങ്ങൾ വാഗ്ദാനം ചെയ്യാനുള്ള ഒരു വഴിയാണ് മാനുവൽ പേയ്‌മെന്റുകൾ. ഇത് വയർ ട്രാൻസ്ഫർ, ബാങ്ക് ട്രാൻസ്ഫർ അല്ലെങ്കിൽ നിങ്ങളുടെ ഉപയോക്താക്കൾക്ക് പ്രാദേശികമായി ലഭ്യമായ മറ്റേതെങ്കിലും പേയ്‌മെന്റ് മാർഗ്ഗം ആകാം.

## മാനുവൽ പേയ്‌മെന്റുകൾ എങ്ങനെ പ്രവർത്തനക്ഷമമാക്കാം

മാനുവൽ പേയ്‌മെന്റ് സജ്ജീകരിക്കുന്നത് വളരെ എളുപ്പമാണ്. പേയ്‌മെന്റ് gateway-കളുടെ കീഴിൽ ഇത് പ്രവർത്തനക്ഷമമാക്കുകയും ഉപയോക്താവ് പേയ്‌മെന്റ് എങ്ങനെ അയയ്ക്കണമെന്നതിനെക്കുറിച്ചുള്ള വിശദമായ നിർദ്ദേശങ്ങൾ നൽകുകയും വേണം.

ആദ്യം, **Ultimate Multisite > Settings > Payments** എന്നതിലേക്ക് പോകുക. **Payment Gateways**-ന് താഴെ, **Manual** ഓണാക്കുക. നിങ്ങൾക്കായി ഒരു **Payment Instructions** ബോക്സ് ദൃശ്യമാകുന്നത് കാണാം.

പേയ്‌മെന്റ് നടത്താൻ നിങ്ങളുടെ ഉപഭോക്താവിന് ആവശ്യമായ വിവരങ്ങൾ ഈ ബോക്സിൽ ചേർക്കുക. ഉദാഹരണത്തിന്, നിങ്ങളുടെ ബാങ്ക് അക്കൗണ്ട് വിശദാംശങ്ങളും ഉപഭോക്താവിന് പേയ്‌മെന്റ് സ്ഥിരീകരണം അയയ്ക്കാനുള്ള നിങ്ങളുടെ ഇമെയിലും ആകാം.

![Manual payment gateway toggle-ഉം payment instructions ബോക്സും](/img/config/settings-payment-gateways.png)

സജ്ജീകരിച്ചതിനു ശേഷം, **Save Settings**-ൽ ക്ലിക്ക് ചെയ്താൽ മതി. നിങ്ങളുടെ നെറ്റ്‌വർക്കിൽ ഉപയോക്താക്കൾ രജിസ്റ്റർ ചെയ്യുമ്പോൾ, വാങ്ങൽ പൂർത്തിയാക്കാനുള്ള നിർദ്ദേശങ്ങൾ അവർക്ക് ലഭിക്കുമെന്ന് അറിയിക്കുന്ന ഒരു സന്ദേശം അവർ കാണും.

![രജിസ്ട്രേഷൻ സമയത്ത് കാണിക്കുന്ന മാനുവൽ പേയ്‌മെന്റ് സന്ദേശം](/img/config/settings-payment-gateways.png)

നിങ്ങളുടെ **Thank You** പേജിൽ നിങ്ങളുടെ പേയ്‌മെന്റ് നിർദ്ദേശങ്ങൾ അടങ്ങിയ ഒരു സന്ദേശവും അവർക്ക് ലഭിക്കും.

![പേയ്‌മെന്റ് നിർദ്ദേശങ്ങൾ കാണിക്കുന്ന Thank You പേജ്](/img/config/settings-payment-gateways.png)

## മാനുവൽ പേയ്‌മെന്റുകൾ സ്ഥിരീകരിക്കുന്നത്

ഒരു മാനുവൽ പേയ്‌മെന്റ് സ്ഥിരീകരിക്കാൻ, ഇടതു ബാറിലെ **Payments** മെനുവിലേക്ക് പോകുക. അവിടെ നിങ്ങളുടെ നെറ്റ്‌വർക്കിലെ എല്ലാ പേയ്‌മെന്റുകളും അവയുടെ **status** ഉൾപ്പെടെയുള്ള വിശദാംശങ്ങളും കാണാം. നിങ്ങൾ മാനുവലായി മാറ്റുന്നതുവരെ ഒരു മാനുവൽ പേയ്‌മെന്റിന് എല്ലായ്പ്പോഴും **Pending** status ആയിരിക്കും.

![Pending മാനുവൽ പേയ്‌മെന്റ് കാണിക്കുന്ന പേയ്‌മെന്റുകളുടെ ലിസ്റ്റ്](/img/admin/payments-list.png)

**reference code**-ൽ ക്ലിക്ക് ചെയ്ത് പേയ്‌മെന്റ് പേജിലേക്ക് പ്രവേശിക്കുക. ഈ പേജിൽ reference ID, products, timestamps എന്നിവയും അതിലേറെയും ഉൾപ്പെടെ pending പേയ്‌മെന്റിന്റെ എല്ലാ വിശദാംശങ്ങളും ഉണ്ട്.

![Reference code-ഉം products-ഉം ഉള്ള പേയ്‌മെന്റ് വിശദാംശങ്ങൾ പേജ്](/img/admin/payments-list.png)

വലതു കോളത്തിൽ, നിങ്ങൾക്ക് പേയ്‌മെന്റിന്റെ status മാറ്റാം. ഇത് **Completed** ആയി മാറ്റുകയും **Activate Membership** ഓപ്ഷൻ ടോഗിൾ ചെയ്യുകയും ചെയ്താൽ നിങ്ങളുടെ ഉപഭോക്താവിന്റെ സൈറ്റ് പ്രവർത്തനക്ഷമമാകുകയും അവരുടെ membership സജീവമാകുകയും ചെയ്യും.

![Activate Membership toggle-ഓടെ Completed-ലേക്ക് പേയ്‌മെന്റ് status മാറ്റുന്നത്](/img/admin/payments-list.png)
