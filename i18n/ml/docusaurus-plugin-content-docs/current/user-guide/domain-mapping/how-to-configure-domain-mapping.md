# ഡൊമെയ്ൻ മാപ്പിംഗ് എങ്ങനെ കോൺഫിഗർ ചെയ്യാം (v2)

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite വേർഷൻ 2.x-നെ കുറിച്ചാണ്.**_

ഒരു പ്രീമിയം നെറ്റ്‌വർക്കിന്റെ ഏറ്റവും ശക്തമായ ഫീച്ചറുകളിലൊന്ന്, നമ്മുടെ ക്ലയന്റുകൾക്ക് അവരുടെ സൈറ്റുകളിലേക്ക് ഒരു ടോപ്പ്-ലെവൽ ഡൊമെയ്ൻ അറ്റാച്ച് ചെയ്യാനുള്ള അവസരം നൽകാനുള്ള കഴിവാണ്. ഏതാണ് കൂടുതൽ പ്രൊഫഷണലായി തോന്നുക: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) അതോ [_**joesbikeshop.com**_](http://joesbikeshop.com)? അതുകൊണ്ടാണ് Ultimate Multisite ഈ ഫീച്ചർ ബിൽറ്റ്-ഇൻ ആയി നൽകുന്നത്, തേർഡ്-പാർട്ടി പ്ലഗിനുകൾ ഉപയോഗിക്കേണ്ട ആവശ്യമില്ല.

## ഡൊമെയ്ൻ മാപ്പിംഗ് എന്താണ്?

പേര് സൂചിപ്പിക്കുന്നതുപോലെ, ഡൊമെയ്ൻ മാപ്പിംഗ് എന്നത് ഒരു കസ്റ്റം ഡൊമെയ്നിനുള്ള റിക്വസ്റ്റ് എടുത്ത്, ആ പ്രത്യേക ഡൊമെയ്ൻ അറ്റാച്ച് ചെയ്തിരിക്കുന്ന നെറ്റ്‌വർക്കിലെ അനുബന്ധ സൈറ്റിലേക്ക് ആ റിക്വസ്റ്റ് മാപ്പ് ചെയ്യാനുള്ള Ultimate Multisite-ന്റെ കഴിവാണ്.

### നിങ്ങളുടെ Ultimate Multisite നെറ്റ്‌വർക്കിൽ ഡൊമെയ്ൻ മാപ്പിംഗ് എങ്ങനെ സെറ്റപ്പ് ചെയ്യാം

ഡൊമെയ്ൻ മാപ്പിംഗ് പ്രവർത്തിക്കാൻ നിങ്ങളുടെ ഭാഗത്തുനിന്ന് ചില സെറ്റപ്പുകൾ ആവശ്യമാണ്. നന്ദിയോടെ, Ultimate Multisite ബുദ്ധിമുട്ടുള്ള ജോലികൾ നിങ്ങൾക്കായി ഓട്ടോമേറ്റ് ചെയ്യുന്നു, അതിനാൽ നിങ്ങൾക്ക് ആവശ്യകതകൾ എളുപ്പത്തിൽ പാലിക്കാം.

Ultimate Multisite ഇൻസ്റ്റലേഷൻ സമയത്ത്, വിസാർഡ് സ്വയമേവ **sunrise.php** നിർദ്ദിഷ്ട ഫോൾഡറിലേക്ക് കോപ്പി ചെയ്ത് ഇൻസ്റ്റാൾ ചെയ്യും. **ഈ ഘട്ടം പൂർത്തിയാകുന്നതുവരെ വിസാർഡ് നിങ്ങളെ മുന്നോട്ട് പോകാൻ അനുവദിക്കില്ല**.

![sunrise.php ഘട്ടമുള്ള Ultimate Multisite ഇൻസ്റ്റലേഷൻ വിസാർഡ്](/img/config/settings-domain-mapping.png)

ഇതിനർത്ഥം Ultimate Multisite ഇൻസ്റ്റലേഷൻ വിസാർഡ് നിങ്ങളുടെ നെറ്റ്‌വർക്ക് സെറ്റപ്പ് ചെയ്തുകഴിഞ്ഞാൽ, നിങ്ങൾക്ക് ഉടൻ തന്നെ കസ്റ്റം ഡൊമെയ്ൻ മാപ്പ് ചെയ്യാൻ തുടങ്ങാം.

Ultimate Multisite-ൽ ഡൊമെയ്ൻ മാപ്പിംഗ് നിർബന്ധമല്ല എന്ന് ശ്രദ്ധിക്കുക. WordPress Multisite-ന്റെ നേറ്റീവ് ഡൊമെയ്ൻ മാപ്പിംഗ് ഫംഗ്ഷനോ മറ്റേതെങ്കിലും ഡൊമെയ്ൻ മാപ്പിംഗ് സൊല്യൂഷനോ ഉപയോഗിക്കാനുള്ള ഓപ്ഷൻ നിങ്ങൾക്കുണ്ട്.

മറ്റ് ഡൊമെയ്ൻ മാപ്പിംഗ് സൊല്യൂഷനുകൾക്ക് വഴിയൊരുക്കാൻ Ultimate Multisite ഡൊമെയ്ൻ മാപ്പിംഗ് ഡിസേബിൾ ചെയ്യേണ്ടി വന്നാൽ, **Ultimate Multisite > Settings > Domain Mapping**-ന് കീഴിൽ ഈ ഫീച്ചർ ഡിസേബിൾ ചെയ്യാം.

![എനേബിൾ ടോഗിളുള്ള ഡൊമെയ്ൻ മാപ്പിംഗ് സെറ്റിംഗ്സ്](/img/config/settings-domain-mapping.png)

ഈ ഓപ്ഷന് തൊട്ടുതാഴെ, **Force Admin Redirect** ഓപ്ഷനും കാണാം. ഈ ഓപ്ഷൻ ഉപയോഗിച്ച്, നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് അവരുടെ കസ്റ്റം ഡൊമെയ്നിലും സബ്‌ഡൊമെയ്നിലും അഡ്മിൻ ഡാഷ്‌ബോർഡ് ആക്സസ് ചെയ്യാൻ കഴിയുമോ അതോ ഇവയിലൊന്നിൽ മാത്രമേ ആക്സസ് ചെയ്യാൻ കഴിയൂ എന്ന് നിയന്ത്രിക്കാം.

നിങ്ങൾ **Force redirect to mapped domain** തിരഞ്ഞെടുത്താൽ, നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് അവരുടെ കസ്റ്റം ഡൊമെയ്നുകളിൽ മാത്രമേ അഡ്മിൻ ഡാഷ്‌ബോർഡ് ആക്സസ് ചെയ്യാൻ കഴിയൂ.

**Force redirect to** **network domain** ഓപ്ഷൻ കൃത്യമായി വിപരീതമായി പ്രവർത്തിക്കും - നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് അവരുടെ സബ്‌ഡൊമെയ്നിൽ മാത്രമേ ഡാഷ്‌ബോർഡുകൾ ആക്സസ് ചെയ്യാൻ കഴിയൂ, അവർ കസ്റ്റം ഡൊമെയ്നുകളിൽ സൈൻ ഇൻ ചെയ്യാൻ ശ്രമിച്ചാലും.

**Allow access to the admin by both mapped domain domain and network domain** ഓപ്ഷൻ സബ്‌ഡൊമെയ്നിലും കസ്റ്റം ഡൊമെയ്നിലും അവരുടെ അഡ്മിൻ ഡാഷ്‌ബോർഡുകൾ ആക്സസ് ചെയ്യാൻ അവരെ അനുവദിക്കുന്നു.

![ഡൊമെയ്ൻ മാപ്പിംഗിനുള്ള Force Admin Redirect ഓപ്ഷനുകൾ](/img/config/settings-domain-mapping.png)

ഒരു കസ്റ്റം ഡൊമെയ്ൻ മാപ്പ് ചെയ്യാൻ രണ്ട് വഴികളുണ്ട്. ഒന്നാമത്തേത് super admin ആയി നിങ്ങളുടെ നെറ്റ്‌വർക്ക് അഡ്മിൻ ഡാഷ്‌ബോർഡിൽ നിന്ന് ഡൊമെയ്ൻ നാമം മാപ്പ് ചെയ്യുന്നതാണ്, രണ്ടാമത്തേത് അക്കൗണ്ട് പേജിന് കീഴിലുള്ള സബ്‌സൈറ്റ് അഡ്മിൻ ഡാഷ്‌ബോർഡിലൂടെയാണ്.

എന്നാൽ നിങ്ങളുടെ നെറ്റ്‌വർക്കിലെ ഒരു സബ്‌സൈറ്റിലേക്ക് കസ്റ്റം ഡൊമെയ്ൻ മാപ്പ് ചെയ്യാൻ തുടങ്ങുന്നതിന് മുമ്പ്, ഡൊമെയ്ൻ നാമത്തിന്റെ **DNS സെറ്റിംഗ്സ്** ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കേണ്ടതുണ്ട്.

### 

### ഡൊമെയ്ൻ DNS സെറ്റിംഗ്സ് ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുന്നു

ഒരു മാപ്പിംഗ് പ്രവർത്തിക്കാൻ, നിങ്ങൾ മാപ്പ് ചെയ്യാൻ ഉദ്ദേശിക്കുന്ന ഡൊമെയ്ൻ നിങ്ങളുടെ നെറ്റ്‌വർക്കിന്റെ IP അഡ്രസിലേക്ക് പോയിന്റ് ചെയ്യുന്നുണ്ടെന്ന് ഉറപ്പാക്കേണ്ടതുണ്ട്. നിങ്ങൾക്ക് നെറ്റ്‌വർക്ക് IP അഡ്രസ് ആവശ്യമാണെന്ന് ശ്രദ്ധിക്കുക - Ultimate Multisite ഇൻസ്റ്റാൾ ചെയ്തിരിക്കുന്ന ഡൊമെയ്നിന്റെ IP അഡ്രസ് - നിങ്ങൾ മാപ്പ് ചെയ്യാൻ ആഗ്രഹിക്കുന്ന കസ്റ്റം ഡൊമെയ്നിന്റെ IP അഡ്രസ് അല്ല. ഒരു പ്രത്യേക ഡൊമെയ്നിന്റെ IP അഡ്രസ് കണ്ടെത്താൻ, ഉദാഹരണത്തിന് [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html) സന്ദർശിക്കാൻ ഞങ്ങൾ നിർദ്ദേശിക്കുന്നു.

ഡൊമെയ്ൻ ശരിയായി മാപ്പ് ചെയ്യാൻ, നിങ്ങളുടെ **DNS** കോൺഫിഗറേഷനിൽ ആ **IP അഡ്രസിലേക്ക്** പോയിന്റ് ചെയ്യുന്ന ഒരു **A RECORD** ചേർക്കേണ്ടതുണ്ട്. DNS മാനേജ്മെന്റ് വ്യത്യസ്ത ഡൊമെയ്ൻ രജിസ്ട്രാറുകൾക്കിടയിൽ വളരെ വ്യത്യാസപ്പെട്ടിരിക്കുന്നു, എന്നാൽ "XXXX-ൽ A Record സൃഷ്ടിക്കുന്നു" എന്ന് സെർച്ച് ചെയ്താൽ ഓൺലൈനിൽ ധാരാളം ട്യൂട്ടോറിയലുകൾ ലഭ്യമാണ്, ഇവിടെ XXXX നിങ്ങളുടെ ഡൊമെയ്ൻ രജിസ്ട്രാർ ആണ് (ഉദാ.: "_GoDaddy-ൽ A Record സൃഷ്ടിക്കുന്നു_").

ഇത് പ്രവർത്തിപ്പിക്കാൻ നിങ്ങൾക്ക് ബുദ്ധിമുട്ട് അനുഭവപ്പെടുകയാണെങ്കിൽ, **നിങ്ങളുടെ ഡൊമെയ്ൻ രജിസ്ട്രാർ സപ്പോർട്ടുമായി ബന്ധപ്പെടുക**, ഈ ഭാഗത്ത് അവർക്ക് നിങ്ങളെ സഹായിക്കാൻ കഴിയും.

നിങ്ങളുടെ ക്ലയന്റുകളെ സ്വന്തം ഡൊമെയ്നുകൾ മാപ്പ് ചെയ്യാൻ അനുവദിക്കാൻ നിങ്ങൾ ഉദ്ദേശിക്കുന്നെങ്കിൽ, ഈ ഭാഗത്തെ ജോലി അവർ സ്വയം ചെയ്യേണ്ടിവരും. A Record സൃഷ്ടിക്കാൻ കഴിയുന്നില്ലെങ്കിൽ അവരുടെ രജിസ്ട്രാർ സപ്പോർട്ട് സിസ്റ്റത്തിലേക്ക് അവരെ നയിക്കുക.

### Super Admin ആയി കസ്റ്റം ഡൊമെയ്ൻ നാമം മാപ്പ് ചെയ്യുന്നു

നിങ്ങളുടെ നെറ്റ്‌വർക്കിൽ super admin ആയി ലോഗിൻ ചെയ്തിരിക്കുമ്പോൾ, **Ultimate Multisite > Domains**-ന് കീഴിൽ പോയി കസ്റ്റം ഡൊമെയ്ൻ നാമങ്ങൾ എളുപ്പത്തിൽ ചേർക്കാനും മാനേജ് ചെയ്യാനും കഴിയും.

![Ultimate Multisite-ലെ ഡൊമെയ്ൻസ് ലിസ്റ്റ് പേജ്](/img/admin/domains-list.png)

ഈ പേജിന് കീഴിൽ, മുകളിലുള്ള **Add Domain** ബട്ടണിൽ ക്ലിക്ക് ചെയ്യാം, ഇത് ഒരു മോഡൽ വിൻഡോ കൊണ്ടുവരും, അവിടെ **കസ്റ്റം ഡൊമെയ്ൻ നാമം**, കസ്റ്റം ഡൊമെയ്ൻ നാമം അപ്ലൈ ചെയ്യാൻ ആഗ്രഹിക്കുന്ന **സബ്‌സൈറ്റ്**, ഇത് **പ്രൈമറി ഡൊമെയ്ൻ** നാമമായി സെറ്റ് ചെയ്യണോ വേണ്ടയോ എന്ന് തീരുമാനിക്കാം (**ഒരു സബ്‌സൈറ്റിലേക്ക് ഒന്നിലധികം ഡൊമെയ്ൻ നാമങ്ങൾ** മാപ്പ് ചെയ്യാൻ കഴിയുമെന്ന് ശ്രദ്ധിക്കുക).

![ഡൊമെയ്ൻ നാമവും സൈറ്റ് ഫീൽഡുകളുമുള്ള Add Domain മോഡൽ](/img/admin/domains-list.png)

എല്ലാ വിവരങ്ങളും നൽകിയ ശേഷം, താഴെയുള്ള **Add Existing Domain** ബട്ടണിൽ ക്ലിക്ക് ചെയ്യാം.

ഇത് കസ്റ്റം ഡൊമെയ്നിന്റെ DNS വിവരങ്ങൾ വെരിഫൈ ചെയ്യുന്നതിനും ഫെച്ച് ചെയ്യുന്നതിനുമുള്ള പ്രക്രിയ ആരംഭിക്കും. പേജിന്റെ താഴെ ഒരു ലോഗും കാണാം, അത് കടന്നുപോകുന്ന പ്രക്രിയ ഫോളോ ചെയ്യാൻ. ഈ പ്രക്രിയ പൂർത്തിയാകാൻ കുറച്ച് മിനിറ്റുകൾ എടുത്തേക്കാം.

എല്ലാം ശരിയായി സെറ്റപ്പ് ചെയ്തിട്ടുണ്ടെങ്കിൽ **Stage** അല്ലെങ്കിൽ സ്റ്റാറ്റസ് **Checking DNS**-ൽ നിന്ന് **Ready**-ലേക്ക് മാറും.

![Checking DNS-ൽ നിന്ന് Ready-ലേക്ക് മാറുന്ന ഡൊമെയ്ൻ സ്റ്റേജ്](/img/admin/domains-list.png)

![ഡൊമെയ്ൻസ് ലിസ്റ്റിൽ Ready സ്റ്റാറ്റസുള്ള ഡൊമെയ്ൻ](/img/admin/domains-list.png)

ഡൊമെയ്ൻ നാമത്തിൽ ക്ലിക്ക് ചെയ്താൽ, അതിനുള്ളിൽ ചില ഓപ്ഷനുകൾ കാണാൻ കഴിയും. അവ പെട്ടെന്ന് നോക്കാം:

![Stage, site, active, SSL ഓപ്ഷനുകളുള്ള ഡൊമെയ്ൻ ഡീറ്റെയിൽ പേജ്](/img/admin/domains-list.png)

**Stage:** ഡൊമെയ്ൻ ഏത് ഘട്ടത്തിലാണെന്ന് ഇത് കാണിക്കുന്നു. നിങ്ങൾ ആദ്യം ഡൊമെയ്ൻ ചേർക്കുമ്പോൾ, ഇത് ഒരുപക്ഷേ **Checking DNS** ഘട്ടത്തിലായിരിക്കും. പ്രക്രിയ DNS എൻട്രികൾ പരിശോധിച്ച് അവ ശരിയാണെന്ന് സ്ഥിരീകരിക്കും. തുടർന്ന്, ഡൊമെയ്ൻ **Checking SSL** ഘട്ടത്തിൽ എത്തും. Ultimate Multisite ഡൊമെയ്ന് SSL ഉണ്ടോ ഇല്ലയോ എന്ന് പരിശോധിച്ച് നിങ്ങളുടെ ഡൊമെയ്നെ **Ready** അല്ലെങ്കിൽ **Ready (without SSL)** ആയി വർഗ്ഗീകരിക്കും.

**Site:** ഈ ഡൊമെയ്നുമായി ബന്ധപ്പെട്ട സബ്‌ഡൊമെയ്ൻ. മാപ്പ് ചെയ്ത ഡൊമെയ്ൻ ഈ പ്രത്യേക സൈറ്റിന്റെ കണ്ടന്റ് കാണിക്കും.

**Active:** ഡൊമെയ്ൻ ആക്ടിവേറ്റ് ചെയ്യാനോ ഡീആക്ടിവേറ്റ് ചെയ്യാനോ ഈ ഓപ്ഷൻ ഓൺ അല്ലെങ്കിൽ ഓഫ് ടോഗിൾ ചെയ്യാം.

**Is Primary Domain?:** നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് ഓരോ സൈറ്റിനും ഒന്നിലധികം മാപ്പ് ചെയ്ത ഡൊമെയ്നുകൾ ഉണ്ടായിരിക്കാം. പ്രത്യേക സൈറ്റിന് ഇത് പ്രൈമറി ഡൊമെയ്ൻ ആണോ എന്ന് തിരഞ്ഞെടുക്കാൻ ഈ ഓപ്ഷൻ ഉപയോഗിക്കുക.

**Is Secure?:** ഡൊമെയ്ന് SSL സർട്ടിഫിക്കറ്റ് ഉണ്ടോ ഇല്ലയോ എന്ന് Ultimate Multisite എനേബിൾ ചെയ്യുന്നതിന് മുമ്പ് പരിശോധിക്കുന്നുണ്ടെങ്കിലും, SSL സർട്ടിഫിക്കറ്റോടെയോ അല്ലാതെയോ ഡൊമെയ്ൻ ലോഡ് ചെയ്യാൻ നിങ്ങൾക്ക് മാനുവലായി തിരഞ്ഞെടുക്കാം. വെബ്‌സൈറ്റിന് SSL സർട്ടിഫിക്കറ്റ് ഇല്ലെങ്കിൽ നിങ്ങൾ SSL-ലൂടെ ഫോഴ്സ് ലോഡ് ചെയ്യാൻ ശ്രമിച്ചാൽ, എറർസ് ലഭിച്ചേക്കാം.

### Subsite യൂസർ ആയി കസ്റ്റം ഡൊമെയ്ൻ നാമം മാപ്പ് ചെയ്യുന്നു

Subsite അഡ്മിനിസ്ട്രേറ്റർമാർക്കും അവരുടെ subsite അഡ്മിൻ ഡാഷ്‌ബോർഡിൽ നിന്ന് കസ്റ്റം ഡൊമെയ്ൻ നാമങ്ങൾ മാപ്പ് ചെയ്യാൻ കഴിയും.

ആദ്യം, **Domain mapping** സെറ്റിംഗ്സിന് കീഴിൽ ഈ ഓപ്ഷൻ എനേബിൾ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കണം. താഴെയുള്ള സ്ക്രീൻഷോട്ട് കാണുക.

![Subsite യൂസർമാരെ ഡൊമെയ്നുകൾ മാപ്പ് ചെയ്യാൻ അനുവദിക്കുന്ന ഡൊമെയ്ൻ മാപ്പിംഗ് സെറ്റിംഗ്സ്](/img/config/settings-domain-mapping.png)

**Ultimate Multisite > Products**-ലെ **Plan** ലെവലിലോ പ്രൊഡക്ട് ഓപ്ഷനുകളിലോ ഈ ഓപ്ഷൻ സെറ്റ് ചെയ്യാനോ കോൺഫിഗർ ചെയ്യാനോ കഴിയും.

![പ്രൊഡക്ട് സെറ്റിംഗ്സിലെ പ്രൊഡക്ട് കസ്റ്റം ഡൊമെയ്ൻസ് ഓപ്ഷൻ](/img/config/settings-domain-mapping.png)

ഈ ഓപ്ഷനുകളിൽ ഏതെങ്കിലും എനേബിൾ ചെയ്തിരിക്കുകയും ഒരു subsite യൂസറിന് കസ്റ്റം ഡൊമെയ്ൻ നാമങ്ങൾ മാപ്പ് ചെയ്യാൻ അനുവാദമുണ്ടെങ്കിൽ, subsite യൂസർ **Account** പേജിന് കീഴിൽ **Domains** എന്ന ഒരു metabox കാണണം.

![Subsite അക്കൗണ്ട് പേജിലെ ഡൊമെയ്ൻസ് metabox](/img/admin/domains-list.png)

യൂസർ **Add Domain** ബട്ടണിൽ ക്ലിക്ക് ചെയ്യാം, ഇത് ചില നിർദ്ദേശങ്ങളുള്ള ഒരു മോഡൽ വിൻഡോ കൊണ്ടുവരും.

![Subsite യൂസർമാർക്കുള്ള DNS നിർദ്ദേശങ്ങളുള്ള Add Domain മോഡൽ](/img/admin/domains-list.png)

യൂസർക്ക് **Next Step** ക്ലിക്ക് ചെയ്ത് കസ്റ്റം ഡൊമെയ്ൻ നാമം ചേർക്കാൻ മുന്നോട്ട് പോകാം. ഇത് പ്രൈമറി ഡൊമെയ്ൻ ആയിരിക്കണോ വേണ്ടയോ എന്നും തിരഞ്ഞെടുക്കാം.

![പ്രൈമറി ഡൊമെയ്ൻ ഓപ്ഷനുള്ള കസ്റ്റം ഡൊമെയ്ൻ നാമം ഫീൽഡ് ചേർക്കുക](/img/admin/domains-list.png)

![DNS വെരിഫിക്കേഷൻ ആരംഭിക്കാൻ Add Domain ക്ലിക്ക് ചെയ്യുക](/img/admin/domains-list.png)**Add Domain** ക്ലിക്ക് ചെയ്യുന്നത് കസ്റ്റം ഡൊമെയ്നിന്റെ DNS വിവരങ്ങൾ വെരിഫൈ ചെയ്യുന്നതിനും ഫെച്ച് ചെയ്യുന്നതിനുമുള്ള പ്രക്രിയ ആരംഭിക്കും.

### ഡൊമെയ്ൻ സിങ്കിംഗിനെ കുറിച്ച്

ഡൊമെയ്ൻ സിങ്കിംഗ് എന്നത് **ഡൊമെയ്ൻ മാപ്പിംഗ് പ്രവർത്തിക്കാൻ** Ultimate Multisite കസ്റ്റം ഡൊമെയ്ൻ നാമം നിങ്ങളുടെ ഹോസ്റ്റിംഗ് അക്കൗണ്ടിൽ ആഡ്-ഓൺ ഡൊമെയ്നായി ചേർക്കുന്ന ഒരു പ്രക്രിയയാണ്.

നിങ്ങളുടെ ഹോസ്റ്റിംഗ് പ്രൊവൈഡർക്ക് Ultimate Multisite ഡൊമെയ്ൻ മാപ്പിംഗ് ഫീച്ചറുമായി ഇന്റഗ്രേഷൻ ഉണ്ടെങ്കിൽ ഡൊമെയ്ൻ സിങ്കിംഗ് സ്വയമേവ സംഭവിക്കും. നിലവിൽ, ഈ ഹോസ്റ്റിംഗ് പ്രൊവൈഡർമാർ _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways,_ കൂടാതെ _Cpanel_ എന്നിവയാണ്.

**Integration** ടാബിന് കീഴിലുള്ള Ultimate Multisite സെറ്റിംഗ്സിൽ ഈ ഇന്റഗ്രേഷൻ ആക്ടിവേറ്റ് ചെയ്യേണ്ടതുണ്ട്.
