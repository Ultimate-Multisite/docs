# പ്ലാൻ ഡൗൺഗ്രേഡ് ചെയ്യുന്നത് (v2)

_**പ്രധാന അറിയിപ്പ്: ഈ ലേഖനം Ultimate Multisite version 2.x-നെക്കുറിച്ചാണ്.**_

പരിമിതമായ ബജറ്റ് ഉള്ളപ്പോഴോ അല്ലെങ്കിൽ തങ്ങളുടെ subsite പ്രവർത്തിപ്പിക്കാൻ കൂടുതൽ resources ആവശ്യമില്ലെന്ന് തീരുമാനിക്കുമ്പോഴോ നിങ്ങളുടെ ക്ലയന്റുകൾ സാധാരണയായി ചെയ്യുന്ന ഒരു കാര്യമാണ് പ്ലാൻ അല്ലെങ്കിൽ subscription ഡൗൺഗ്രേഡ് ചെയ്യുക എന്നത്.

## പ്ലാൻ എങ്ങനെ ഡൗൺഗ്രേഡ് ചെയ്യാം

നിങ്ങളുടെ ക്ലയന്റുകൾക്ക് അവരുടെ subsite admin dashboard-ൽ ലോഗിൻ ചെയ്ത് അക്കൗണ്ട് പേജിലെ **Change** ബട്ടൺ ക്ലിക്ക് ചെയ്ത് എപ്പോൾ വേണമെങ്കിലും പ്ലാൻ ഡൗൺഗ്രേഡ് ചെയ്യാവുന്നതാണ്.

![membership-ന് കീഴിൽ Change ബട്ടണുള്ള അക്കൗണ്ട് പേജ്](/img/admin/memberships-list.png)

**Change** ബട്ടൺ ക്ലിക്ക് ചെയ്യുമ്പോൾ, user/client checkout പേജിലേക്ക് redirect ചെയ്യപ്പെടും, അവിടെ അവർക്ക് subscription മാറ്റേണ്ട പ്ലാൻ തിരഞ്ഞെടുക്കാം.

![ഡൗൺഗ്രേഡിനുള്ള പ്ലാൻ ഓപ്ഷനുകൾ കാണിക്കുന്ന Checkout പേജ്](/img/admin/memberships-list.png)

ഈ ഉദാഹരണത്തിൽ, നമ്മൾ പ്ലാൻ **Premium**-ൽ നിന്ന് **Free**-ലേക്ക് ഡൗൺഗ്രേഡ് ചെയ്യുകയാണ്.

മുന്നോട്ട് പോകാൻ user-ന് **Complete Checkout** ബട്ടൺ ക്ലിക്ക് ചെയ്താൽ മതി. അപ്പോൾ membership-ൽ വരാനിരിക്കുന്ന മാറ്റത്തെക്കുറിച്ചുള്ള സന്ദേശം കാണിക്കുന്ന അക്കൗണ്ട് പേജിലേക്ക് തിരികെ എത്തും. മാറ്റങ്ങൾ customer-ന്റെ **അടുത്ത billing cycle**-ൽ പ്രാബല്യത്തിൽ വരും.

![membership മാറ്റം pending ആണെന്ന് കാണിക്കുന്ന അക്കൗണ്ട് പേജ്](/img/admin/memberships-list.png)

### ഒരു user പ്ലാൻ ഡൗൺഗ്രേഡ് ചെയ്യുമ്പോൾ എന്ത് സംഭവിക്കും

പ്ലാൻ ഡൗൺഗ്രേഡ് ചെയ്യുന്നത് user-ന്റെ subsite-ലെ നിലവിലുള്ള configuration-കളെ മാറ്റുന്നില്ല എന്നത് ശ്രദ്ധിക്കേണ്ട കാര്യമാണ്.

Site template സ്വയമേവ മാറുന്നില്ല, കാരണം site template മാറ്റുന്നത് subsite പൂർണ്ണമായും മായ്ച്ച് reset ചെയ്യും. അനാവശ്യമായ data നഷ്ടം ഒഴിവാക്കാനാണ് ഇത്. അതിനാൽ posts ഒഴികെ disk space, themes, plugins മുതലായവ അതേപടി നിലനിൽക്കും.

ഓരോ പ്ലാനിലും നിങ്ങൾ സെറ്റ് ചെയ്ത limits-ഉം quotas-ഉം ആയിരിക്കും നിങ്ങളുടെ പ്രധാന ആശങ്ക എന്ന് ഞങ്ങൾക്കറിയാം, എന്നാൽ user-ന്റെ subsite-ലെ ഏതെങ്കിലും configuration-കൾ ഞങ്ങൾ ഇല്ലാതാക്കുകയോ മാറ്റുകയോ ചെയ്താൽ ഉണ്ടാകുന്ന നാശനഷ്ടങ്ങൾ കൂടി ഞങ്ങൾ പരിഗണിക്കേണ്ടതുണ്ട്.

പ്ലാനിൽ സെറ്റ് ചെയ്ത limit കവിയുന്ന posts-ന്, നിങ്ങൾക്ക് 3 വ്യത്യസ്ത ഓപ്ഷനുകളുണ്ട്: **Posts അതേപടി നിലനിർത്തുക***,* **Posts trash-ലേക്ക് മാറ്റുക***,* അല്ലെങ്കിൽ **Posts draft-ലേക്ക് മാറ്റുക***.* Ultimate Multisite settings-ൽ ഇത് configure ചെയ്യാവുന്നതാണ്.

![Ultimate Multisite settings-ലെ Post limit exceeded ഓപ്ഷനുകൾ](/img/config/settings-sites.png)

### പേയ്മെന്റിന് എന്ത് സംഭവിക്കും

Version 2.0-ൽ, proration-ന്റെ കാര്യത്തിൽ പേയ്മെന്റിൽ ഒരു adjustments-ഉം ആവശ്യമില്ല.

കാരണം, പുതിയ plan/membership പ്രാബല്യത്തിൽ വരുന്നതിന് മുമ്പ് നിലവിലുള്ള membership അതിന്റെ **billing cycle പൂർത്തിയാക്കുന്നത് വരെ** system കാത്തിരിക്കും. പുതിയ membership-നുള്ള പുതിയ billing amount അടുത്ത billing cycle-ൽ സ്വയമേവ ബാധകമാക്കുകയും charge ചെയ്യുകയും ചെയ്യും.
