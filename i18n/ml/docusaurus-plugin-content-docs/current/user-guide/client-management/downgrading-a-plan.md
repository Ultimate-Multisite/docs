# ഒരു പദ്ധതി ഡൗൺഗ്രേഡ് ചെയ്യൽ (v2) {#downgrading-a-plan-v2}

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite പതിപ്പ് 2.x-നെ കുറിച്ചാണ്.**_

പരിമിതമായ ബജറ്റ് ഉള്ളപ്പോൾ, അല്ലെങ്കിൽ അവരുടെ ഉപസൈറ്റ് പ്രവർത്തിപ്പിക്കാൻ അധിക വിഭവങ്ങൾ ആവശ്യമില്ലെന്ന് തീരുമാനിക്കുമ്പോൾ, നിങ്ങളുടെ ക്ലയന്റുകൾ പദ്ധതി അല്ലെങ്കിൽ സബ്സ്ക്രിപ്ഷൻ ഡൗൺഗ്രേഡ് ചെയ്യുന്നത് സാധാരണമാണ്.

## ഒരു പദ്ധതി എങ്ങനെ ഡൗൺഗ്രേഡ് ചെയ്യാം {#how-to-downgrade-a-plan}

നിങ്ങളുടെ ക്ലയന്റുകൾക്ക് അവരുടെ ഉപസൈറ്റ് അഡ്മിൻ Dashboard-ൽ ലോഗിൻ ചെയ്ത് Account പേജിൽ **മാറ്റുക** ക്ലിക്ക് ചെയ്ത് എപ്പോൾ വേണമെങ്കിലും പദ്ധതി ഡൗൺഗ്രേഡ് ചെയ്യാം.

![നിങ്ങളുടെ അംഗത്വ കാർഡും മാറ്റുക ബട്ടണും ഉള്ള ഉപഭോക്തൃ Account പേജ്](/img/account-page/membership-change-button.png)

**മാറ്റുക** ബട്ടൺ ക്ലിക്ക് ചെയ്താൽ, ഉപയോക്താവിനെ/ക്ലയന്റിനെ ചെക്ക്ഔട്ട് പേജിലേക്ക് തിരിച്ചുവിടും. അവിടെ അവർ സബ്സ്ക്രിപ്ഷൻ മാറ്റാൻ ആഗ്രഹിക്കുന്ന പദ്ധതി തിരഞ്ഞെടുക്കാം.

![ഉപഭോക്തൃ ഭാഗത്തെ പദ്ധതി ഡൗൺഗ്രേഡ് ഓപ്ഷനുകളുടെ പേജ്](/img/account-page/downgrade-picker.png)

ഈ ഉദാഹരണത്തിൽ, പദ്ധതി **Premium**-ൽ നിന്ന് **Free**-ലേക്ക് ഡൗൺഗ്രേഡ് ചെയ്യുകയാണ്.

തുടരാൻ ഉപയോക്താവ് **ചെക്ക്ഔട്ട് പൂർത്തിയാക്കുക** ബട്ടൺ ക്ലിക്ക് ചെയ്താൽ മതി. തുടർന്ന് അംഗത്വത്തിൽ കാത്തിരിക്കുന്ന മാറ്റത്തെക്കുറിച്ചുള്ള സന്ദേശം കാണിക്കുന്ന Account പേജിലേക്ക് അവരെ തിരികെ കൊണ്ടുപോകും. മാറ്റങ്ങൾ ഉപഭോക്താവിന്റെ **അടുത്ത ബില്ലിംഗ് സൈക്കിളിൽ** പ്രാബല്യത്തിൽ വരും.

![കാത്തിരിക്കുന്ന അംഗത്വ മാറ്റ ബാനർ കാണിക്കുന്ന Account പേജ്](/img/account-page/pending-change.png)

### ഉപയോക്താവ് പദ്ധതി ഡൗൺഗ്രേഡ് ചെയ്യുമ്പോൾ എന്ത് സംഭവിക്കും {#what-happens-when-a-user-downgrades-their-plan}

പദ്ധതി ഡൗൺഗ്രേഡ് ചെയ്യുന്നത് ഉപയോക്താവിന്റെ ഉപസൈറ്റിലെ നിലവിലുള്ള ക്രമീകരണങ്ങളെ മാറ്റില്ലെന്നത് ശ്രദ്ധിക്കണം.

സൈറ്റ് ടെംപ്ലേറ്റ് മാറ്റുന്നത് ഉപസൈറ്റ് മുഴുവനായി മായ്ച്ച് റീസെറ്റ് ചെയ്യുന്നതിനാൽ, ഇത് സ്വയമേവ സൈറ്റ് ടെംപ്ലേറ്റ് മാറ്റില്ല. അനാവശ്യ ഡാറ്റ നഷ്ടം ഒഴിവാക്കാനാണിത്. അതിനാൽ പോസ്റ്റുകൾ ഒഴികെ ഡിസ്ക് സ്പേസ്, തീമുകൾ, പ്ലഗിനുകൾ തുടങ്ങിയവ അതേപടി നിലനിൽക്കും.

ഓരോ പദ്ധതിയിലും നിങ്ങൾ സജ്ജമാക്കിയ പരിധികളും ക്വോട്ടകളും ആയിരിക്കും നിങ്ങളുടെ പ്രധാന ആശങ്കയെന്ന് ഞങ്ങൾ മനസ്സിലാക്കുന്നു. പക്ഷേ ഉപയോക്താവിന്റെ ഉപസൈറ്റിലെ ഏതെങ്കിലും ക്രമീകരണം ഇല്ലാതാക്കുകയോ മാറ്റുകയോ ചെയ്താൽ ഉണ്ടാകുന്ന ദോഷവും പരിഗണിക്കേണ്ടതുണ്ട്.

പദ്ധതിയിൽ നിശ്ചയിച്ച പരിധി കവിയുന്ന പോസ്റ്റുകൾക്കായി നിങ്ങൾക്ക് 3 വ്യത്യസ്ത ഓപ്ഷനുകളുണ്ട്: **പോസ്റ്റുകൾ അതേപടി നിലനിർത്തുക** *,* **പോസ്റ്റുകൾ ട്രാഷിലേക്ക് മാറ്റുക** *,* അല്ലെങ്കിൽ **പോസ്റ്റുകൾ ഡ്രാഫ്റ്റിലേക്ക് മാറ്റുക** *.* ഇത് Ultimate Multisite ക്രമീകരണങ്ങളിൽ സജ്ജീകരിക്കാം.

![പോസ്റ്റ് പരിധി കവിഞ്ഞാൽ സ്വീകരിക്കുന്ന പെരുമാറ്റ ഓപ്ഷനുകൾ കാണിക്കുന്ന Network Admin ക്രമീകരണങ്ങളുടെ സൈറ്റുകൾ പേജ്](/img/account-page/settings-sites-post-limit.png)

### പേയ്മെന്റിന് എന്ത് സംഭവിക്കും {#what-happens-to-the-payment}

പതിപ്പ് 2.0-ൽ, പ്രോറേഷൻ സംബന്ധിച്ച് പേയ്മെന്റിൽ ഇനി ക്രമീകരണങ്ങൾ ആവശ്യമില്ല.

കാരണം പുതിയ പദ്ധതി/അംഗത്വം പ്രാബല്യത്തിൽ വരുന്നതിന് മുമ്പ് നിലവിലുള്ള അംഗത്വം **അതിന്റെ ബില്ലിംഗ് സൈക്കിൾ പൂർത്തിയാക്കുന്നത് വരെ** സിസ്റ്റം കാത്തിരിക്കും. പുതിയ അംഗത്വത്തിനുള്ള പുതിയ ബില്ലിംഗ് തുക സ്വയമേവ പ്രയോഗിക്കപ്പെടുകയും അടുത്ത ബില്ലിംഗ് സൈക്കിളിൽ ഈടാക്കുകയും ചെയ്യും.
