# ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം (Design System Aesthetics Skill) {#design-system-aesthetics-skill}

**ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം** എന്ന കഴിവ്, നിങ്ങളുടെ സൈറ്റിന്റെ വിഷ്വൽ ഐഡന്റിറ്റി മെച്ചപ്പെടുത്താനുള്ള ഒരു മാർഗ്ഗനിർദ്ദേശ രീതിയാണ്. ടൈപ്പോഗ്രാഫി, കളർ, സ്പേസിംഗ്, മോഷൻ ടോക്കണുകൾ എന്നിവയെക്കുറിച്ച് സ്ഥിരമായ തീരുമാനങ്ങൾ എടുക്കാൻ ഇത് നിങ്ങളെ സഹായിക്കുന്നു, ഇത് നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റത്തെ നിർവചിക്കുന്നു.

## ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം എന്നാൽ എന്ത്? {#what-is-design-system-aesthetics}

ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം എന്നത് താഴെ പറയുന്ന കാര്യങ്ങൾ ഉൾക്കൊള്ളുന്ന ഒരു ഘടനാപരമായ കഴിവാണ്:

- **ടൈപ്പോഗ്രാഫി (Typography)**: ഫോണ്ട് കുടുംബങ്ങൾ, വലുപ്പങ്ങൾ, ഭാരങ്ങൾ (weights), ലൈൻ ഹൈറ്റുകൾ
- **കളർ (Color)**: പ്രൈമറി, സെക്കൻഡറി, ആക്സന്റ്, ന്യൂട്രൽ പാലറ്റുകൾ
- **സ്പേസിംഗ് (Spacing)**: പാഡിംഗ്, മാർജിൻ, ഗ്യാപ്പ് സ്കെയിലുകൾ
- **ബോർഡറുകൾ (Borders)**: റേഡിയസ്, വീതി ടോക്കണുകൾ
- **ഷാഡോകൾ (Shadows)**: എലവേഷൻ, ഡെപ്ത് ടോക്കണുകൾ
- **മോഷൻ (Motion)**: ആനിമേഷനുകളും ട്രാൻസിഷനുകളും

ഈ തീരുമാനങ്ങൾ നിങ്ങളുടെ തീമിന്റെ `theme.json` ഫയലിൽ രേഖപ്പെടുത്തുന്നു, ഇത് ഒരുമിച്ചൊരു വിഷ്വൽ സിസ്റ്റം സൃഷ്ടിക്കുന്നു.

## ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം എന്തിനാണ് ഉപയോഗിക്കുന്നത്? {#why-use-design-system-aesthetics}

### സ്ഥിരത (Consistency) {#consistency}

ഒരു ഡിസൈൻ സിസ്റ്റം ഉറപ്പാക്കുന്നത്:

- എല്ലാ ടെക്സ്റ്റും ഒരേ ടൈപ്പോഗ്രാഫി സ്കെയിൽ ഉപയോഗിക്കുന്നു
- കളറുകൾ സൈറ്റിലുടനീളം സ്ഥിരമായി ഉപയോഗിക്കുന്നു
- സ്പേസിംഗ് ഒരു പ്രവചിക്കാവുന്ന രീതി പിന്തുടരുന്നു
- ആനിമേഷനുകൾ ഒരുമിച്ചൊരു അനുഭവം നൽകുന്നു

### കാര്യക്ഷമത (Efficiency) {#efficiency}

പേജ് ബൈ പേജ് ഡിസൈൻ തീരുമാനങ്ങൾ എടുക്കുന്നതിന് പകരം, നിങ്ങൾക്ക്:

- ടോക്കണുകൾ ഒരിക്കൽ നിർവചിക്കാം
- അവ എല്ലായിടത്തും പ്രയോഗിക്കാം
- ഒരു മൂല്യം മാറ്റിക്കൊണ്ട് ആഗോളതലത്തിൽ അപ്ഡേറ്റ് ചെയ്യാം

### വഴക്കം (Flexibility) {#flexibility}

നിങ്ങൾക്ക്:

- നിങ്ങളുടെ മുഴുവൻ ഡിസൈൻ സിസ്റ്റവും വേഗത്തിൽ ക്രമീകരിക്കാൻ കഴിയും
- വ്യത്യസ്ത സൗന്ദര്യശാസ്ത്രങ്ങളുമായി പരീക്ഷണം നടത്താൻ കഴിയും
- പരിണമിക്കുമ്പോൾ പോലും ബ്രാൻഡ് സ്ഥിരത നിലനിർത്താൻ കഴിയും

## ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം കഴിവ് സജ്ജമാക്കൽ (Triggering) {#triggering-the-design-system-aesthetics-skill}

### മാനുവൽ ആക്ടിവേഷൻ (Manual Activation) {#manual-activation}

നിങ്ങൾക്ക് എപ്പോൾ വേണമെങ്കിലും ഈ കഴിവ് ആരംഭിക്കാം:

```
"എന്റെ ഡിസൈൻ സിസ്റ്റം മെച്ചപ്പെടുത്താൻ എന്നെ സഹായിക്കൂ"
```

അല്ലെങ്കിൽ

```
"എന്റെ സൈറ്റിന്റെ സൗന്ദര്യം മെച്ചപ്പെടുത്താം"
```

അല്ലെങ്കിൽ

```
"ഡിസൈൻ സിസ്റ്റം തീരുമാനങ്ങളിലൂടെ എന്നെ നയിക്കൂ"
```

### ഓട്ടോമാറ്റിക് നിർദ്ദേശങ്ങൾ (Automatic Suggestions) {#automatic-suggestions}

നിങ്ങൾ ചെയ്യുമ്പോൾ ഏജന്റുമാർ ഈ കഴിവ് ഉപയോഗിക്കാൻ നിർദ്ദേശിച്ചേക്കാം:

- ഡിസൈൻ മാറ്റങ്ങൾ ആവശ്യപ്പെടുമ്പോൾ
- ഒരു "ആധുനികമായ" അല്ലെങ്കിൽ "പ്രൊഫഷണൽ" പുനർരൂപകൽപ്പന ആവശ്യപ്പെടുമ്പോൾ
- വിഷ്വൽ സ്ഥിരത മെച്ചപ്പെടുത്താൻ ആഗ്രഹിക്കുമ്പോൾ
- നിങ്ങളുടെ സൈറ്റ് ലോഞ്ച് ചെയ്യാൻ തയ്യാറെടുക്കുമ്പോൾ

## ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം പ്രക്രിയ (The Design System Aesthetics Process) {#the-design-system-aesthetics-process}

### ഘട്ടം 1: ടൈപ്പോഗ്രാഫി (Typography) {#step-1-typography}

ഏജന്റ് നിങ്ങളുടെ ഫോണ്ട് തിരഞ്ഞെടുപ്പുകളെക്കുറിച്ച് ചോദിക്കുന്നു:

```
നിങ്ങളുടെ ഇഷ്ടപ്പെട്ട ടൈപ്പോഗ്രാഫി ശൈലി ഏതാണ്?
- സെറിഫ് (Serif) (പരമ്പരാഗതമായ, മനോഹരമായ)
- സാൻസ്-സെറിഫ് (Sans-serif) (ആധുനികമായ, വൃത്തിയുള്ള)
- മോണോസ്‌പേസ് (Monospace) (സാങ്കേതികമായ, കോഡ്-ഫോക്കസ്ഡ്)

നിങ്ങളുടെ ഹെഡിംഗ് ഫോണ്ടിനായി:
- നിങ്ങൾക്ക് ഒരു പ്രത്യേക ഹെഡിംഗ് ഫോണ്ട് വേണോ, അതോ ബോഡി ടെക്സ്റ്റുമായി ഒരേ ഫോണ്ട് ഉപയോഗിക്കണോ?
- ഇഷ്ടം: ബോൾഡ്, മനോഹരം, കളിയായകം, മിനിമൽ?

നിങ്ങളുടെ ബോഡി ഫോണ്ടിനായി:
- വായനാക്ഷമതയാണ് പ്രധാനം. നിങ്ങൾക്ക് ഏതാണ് ഇഷ്ടം:
  - വലുതും കൂടുതൽ സ്പേസ് ഉള്ള ടെക്സ്റ്റ്
  - കംപാക്റ്റ്, കാര്യക്ഷമമായ ടെക്സ്റ്റ്
  - സ്റ്റാൻഡേർഡ് സൈസിംഗ്
```

തുടർന്ന് ഏജന്റ് നിർവചിക്കുന്നു:

- **ഹെഡിംഗ് ഫോണ്ട്**: ടൈറ്റിലുകൾക്കും ഹെഡിംഗുകൾക്കുമുള്ള പ്രധാന ഫോണ്ട്
- **ബോഡി ഫോണ്ട്**: ഖണ്ഡികകൾക്കും ബോഡി ടെക്സ്റ്റിനുമുള്ള ഫോണ്ട്
- **മോണോസ്‌പേസ് ഫോണ്ട്**: കോഡിനും സാങ്കേതിക ഉള്ളടക്കത്തിനുമുള്ള ഫോണ്ട്
- **സൈസ് സ്കെയിൽ (Size scale)**: മുൻകൂട്ടി നിർവചിച്ച വലുപ്പങ്ങൾ (small, base, large, XL, etc.)
- **വേറ്റ് സ്കെയിൽ (Weight scale)**: ഫോണ്ട് ഭാരങ്ങൾ (regular, medium, bold, etc.)
- **ലൈൻ ഹൈറ്റ് (Line height)**: വായനാക്ഷമതയ്ക്കായി വരികൾ തമ്മിലുള്ള അകലം

### ഘട്ടം 2: കളർ പാലറ്റ് (Color Palette) {#step-2-color-palette}

```
നിങ്ങളുടെ കളർ പാലറ്റ് നമുക്ക് നിർവചിക്കാം.

പ്രൈമറി കളർ (നിങ്ങളുടെ ബ്രാൻഡ് കളർ):
- നിലവിലെ: [നിലവിലുള്ള കളർ കാണിക്കുന്നു]
- മാറ്റാൻ: [കളർ പിക്കർ അല്ലെങ്കിൽ ഹെക്സ് കോഡ്]

സെക്കൻഡറി കളർ (സഹായിക്കുന്ന കളർ):
- നിലവിലെ: [നിലവിലുള്ള കളർ കാണിക്കുന്നു]
- മാറ്റാൻ: [കളർ പിക്കർ അല്ലെങ്കിൽ ഹെക്സ് കോഡ്]

ആക്സന്റ് കളർ (പ്രധാന ഹൈലൈറ്റുകൾ, CTA):
- നിലവിലെ: [നിലവിലുള്ള കളർ കാണിക്കുന്നു]
- മാറ്റാൻ: [കളർ പിക്കർ അല്ലെങ്കിൽ ഹെക്സ് കോഡ്]

ന്യൂട്രൽ പാലറ്റ് (ടെക്സ്റ്റ്, ബോർഡറുകൾ, ബാക്ക്ഗ്രൗണ്ടുകൾക്കായുള്ള ഗ്രേകൾ):
- ലൈറ്റ്: [കളർ]
- മീഡിയം: [കളർ]
- ഡാർക്ക്: [കളർ]
```

ഏജന്റ് താഴെ പറയുന്നവ ഉൾപ്പെടെ ഒരു സമ്പൂർണ്ണ പാലറ്റ് സൃഷ്ടിക്കുന്നു:

- പ്രൈമറി, സെക്കൻഡറി, ആക്സന്റ് കളറുകൾ
- ന്യൂട്രൽ ഗ്രേകൾ (ലൈറ്റ്, മീഡിയം, ഡാർക്ക്)
- സെമാന്റിക് കളറുകൾ (success, warning, error)
- ഹോവർ, ആക്റ്റീവ് സ്റ്റേറ്റുകൾ

### ഘട്ടം 3: സ്പേസിംഗ് (Spacing) {#step-3-spacing}

```
നിങ്ങൾക്ക് എത്രത്തോളം 'ശ്വസിക്കാനുള്ള സ്ഥലം' വേണം?

സ്പേസിംഗ് സ്കെയിൽ ഇഷ്ടം:
- കംപാക്റ്റ് (Compact) (ചെറിയ, കാര്യക്ഷമമായ ലേഔട്ടുകൾ)
- നോർമൽ (Normal) (സന്തുലിതമായ സ്പേസിംഗ്)
- സ്പേഷ്യസ് (Spacious) (വലിയ ഒഴിവുകൾ)

ഇത് ബാധിക്കുന്നത്:
- ബട്ടണുകൾ, കാർഡുകൾ എന്നിവയിലെ പാഡിംഗ്
- സെക്ഷനുകൾ തമ്മിലുള്ള മാർജിനുകൾ
- ഗ്രിഡ് ഇറ്റമുകൾക്കിടയിലുള്ള ഗ്യാപ്പ്
```

ഏജന്റ് സ്പേസിംഗ് ടോക്കണുകൾ നിർവചിക്കുന്നു:

- ബേസ് യൂണിറ്റ് (സാധാരണയായി 4px അല്ലെങ്കിൽ 8px)
- സ്കെയിൽ: xs, sm, md, lg, xl, 2xl
- പാഡിംഗ്, മാർജിൻ, ഗ്യാപ്പ് എന്നിവയ്ക്കുള്ള പ്രത്യേക മൂല്യങ്ങൾ

### ഘട്ടം 4: ബോർഡറുകളും ഷാഡോകളും (Borders and Shadows) {#step-4-borders-and-shadows}

```
വിഷ്വൽ ആഴവും നിർവചനവും:

ബോർഡർ റേഡിയസ് ഇഷ്ടം:
- ഷാർപ്പ് (Sharp) (വളവില്ല)
- സബ്റ്റിൽ (Subtle) (ചെറിയ റേഡിയസ്)
- റൗണ്ടഡ് (Rounded) (മിതമായ റേഡിയസ്)
- വെരി റൗണ്ടഡ് (Very rounded) (വലിയ റേഡിയസ്)

ഷാഡോ ആഴം:
- ഫ്ലാറ്റ് (Flat) (ഷാഡോകളില്ല)
- സബ്റ്റിൽ (Subtle) (ഇളം ഷാഡോകൾ)
- പ്രൊനൗൺസ്ഡ് (Pronounced) (ശക്തമായ ഷാഡോകൾ)
```

ഏജന്റ് സൃഷ്ടിക്കുന്നത്:

- ബോർഡർ റേഡിയസ് ടോക്കണുകൾ (ബട്ടണുകൾ, കാർഡുകൾ, ഇൻപുട്ടുകൾ എന്നിവയ്ക്കായി)
- എലവേഷൻ ലെവലുകൾക്കായുള്ള ഷാഡോ ടോക്കണുകൾ
- ബോർഡർ വീതി ടോക്കണുകൾ

### ഘട്ടം 5: മോഷനും ആനിമേഷനും (Motion and Animation) {#step-5-motion-and-animation}

```
ഇന്ററാക്ടീവായിരിക്കുമ്പോൾ നിങ്ങളുടെ സൈറ്റ് എങ്ങനെ തോന്നണം?

ആനിമേഷൻ ഇഷ്ടം:
- മിനിമൽ (Minimal) (ആനിമേഷനുകളില്ല)
- സബ്റ്റിൽ (Subtle) (മെല്ലെയുള്ള ട്രാൻസിഷനുകൾ)
- പ്ലേഫുൾ (Playful) (ശ്രദ്ധേയമായ ആനിമേഷനുകൾ)

പ്രത്യേക ആനിമേഷനുകൾ:
- പേജ് ട്രാൻസിഷനുകൾ: ഫേഡ്, സ്ലൈഡ്, അതോ ഒന്നുമില്ലേ?
- ബട്ടൺ ഹോവർ: സ്കെയിൽ, കളർ മാറ്റം, അതോ രണ്ടും?
- ലോഡിംഗ് സ്റ്റേറ്റുകൾ: സ്പിന്നർ, скеലെടൺ, അതോ പ്രോഗ്രസ് ബാർ?
```

ഏജന്റ് നിർവചിക്കുന്നു:

- ട്രാൻസിഷൻ ദൈർഘ്യങ്ങൾ (വേഗതയുള്ള, സാധാരണ, മന്ദഗതിയിലുള്ള)
- ഈസിംഗ് ഫംഗ്ഷനുകൾ (ease-in, ease-out, ease-in-out)
- പൊതുവായ ഇടപെടലുകൾക്കായുള്ള ആനിമേഷൻ കീഫ്രെയിമുകൾ

## ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം പ്രയോഗിക്കൽ (Applying Design System Aesthetics) {#applying-design-system-aesthetics}

### ഓട്ടോമാറ്റിക് ആപ്ലിക്കേഷൻ (Automatic Application) {#automatic-application}

നിങ്ങൾ കഴിവ് പൂർത്തിയാക്കിയ ശേഷം, ഏജന്റ്:

1. എല്ലാ ടോക്കണുകളും ഉപയോഗിച്ച് നിങ്ങളുടെ തീമിന്റെ `theme.json` അപ്ഡേറ്റ് ചെയ്യുന്നു
2. ഡിസൈൻ സിസ്റ്റം നിങ്ങളുടെ актив തീമിന് പ്രയോഗിക്കുന്നു
3. പുതിയ സിസ്റ്റവുമായി പൊരുത്തപ്പെടാൻ ബ്ലോക്ക് സ്റ്റൈലുകൾ പുനഃസൃഷ്ടിക്കുന്നു
4. അപ്ഡേറ്റ് ചെയ്ത തീം സജീവമാക്കുന്നു

### മാനുവൽ ആപ്ലിക്കേഷൻ (Manual Application) {#manual-application}

നിങ്ങൾക്ക് `theme.json` നേരിട്ട് എഡിറ്റ് ചെയ്യാനും കഴിയും:

```json
{
  "version": 3,
  "settings": {
    "color": {
      "palette": [
        {
          "color": "#0066CC",
          "name": "Primary",
          "slug": "primary"
        }
      ]
    },
    "typography": {
      "fontFamilies": [
        {
          "fontFamily": "Inter, sans-serif",
          "name": "Body",
          "slug": "body"
        }
      ]
    },
    "spacing": {
      "spacingSizes": [
        {
          "size": "0.5rem",
          "name": "Small",
          "slug": "sm"
        }
      ]
    }
  }
}
```

## നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റം കാണുന്നത് (Viewing Your Design System) {#viewing-your-design-system}

### ഏജന്റിനോട് ചോദിക്കുക {#ask-the-agent}

```
"എന്റെ ഡിസൈൻ സിസ്റ്റം കാണിക്കൂ"
```

അല്ലെങ്കിൽ

```
"എന്റെ നിലവിലെ ഡിസൈൻ ടോക്കണുകൾ എന്തൊക്കെയാണ്?"
```

നിങ്ങളുടെ ടൈപ്പോഗ്രാഫി, കളറുകൾ, സ്പേസിംഗ്, മറ്റ് ടോക്കണുകൾ എന്നിവ ഏജന്റ് പ്രദർശിപ്പിക്കും.

### theme.json കാണുക {#view-themejson}

റാവു ടോക്കൺ നിർവചനങ്ങൾ കാണാൻ `/wp-content/themes/[theme-name]/theme.json` ഒരു ടെക്സ്റ്റ് എഡിറ്ററിൽ തുറക്കുക.

## നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റം അപ്ഡേറ്റ് ചെയ്യൽ (Updating Your Design System) {#updating-your-design-system}

### വേഗത്തിലുള്ള അപ്ഡേറ്റുകൾ (Quick Updates) {#quick-updates}

ഏജന്റിനോട് പ്രത്യേക മാറ്റങ്ങൾ ആവശ്യപ്പെടുക:

```
"പ്രൈമറി കളർ കുറച്ചുകൂടി ഇരുണ്ടതാക്കൂ"
```

അല്ലെങ്കിൽ

```
"സ്പേസിംഗ് സ്കെയിൽ 20% വർദ്ധിപ്പിക്കൂ"
```

അല്ലെങ്കിൽ

```
"ഹെഡിംഗ് ഫോണ്ട് സെറിഫ് ആക്കി മാറ്റൂ"
```

### പൂർണ്ണ പുനർരൂപകൽപ്പന (Full Redesign) {#full-redesign}

ഡിസൈൻ സിസ്റ്റം സൗന്ദര്യം കഴിവ് വീണ്ടും പ്രവർത്തിപ്പിക്കുക:

```
"എന്റെ മുഴുവൻ ഡിസൈൻ സിസ്റ്റവും പുനർരൂപകൽപ്പന ചെയ്യാം"
```

ഇത് നിങ്ങളുടെ നിലവിലെ മൂല്യങ്ങൾ ഉപയോഗിച്ച് എല്ലാ തീരുമാനങ്ങളിലൂടെയും നിങ്ങളെ നയിക്കും.

### ഭാഗിക അപ്ഡേറ്റുകൾ (Partial Updates) {#partial-updates}

പ്രത്യേക വശങ്ങൾ അപ്ഡേറ്റ് ചെയ്യുക:

```
"കളർ പാലറ്റ് മാത്രം അപ്ഡേറ്റ് ചെയ്യൂ, മറ്റെല്ലാം മാറ്റാതെ"
```

## ഡിസൈൻ സിസ്റ്റം മികച്ച രീതികൾ (Design System Best Practices) {#design-system-best-practices}

### സ്ഥിരത (Consistency) {#consistency-1}

- എല്ലായിടത്തും ഒരേ ടോക്കണുകൾ ഉപയോഗിക്കുക
- ഒറ്റത്തവണ ഉപയോഗിക്കാവുന്ന കളറുകളോ വലുപ്പങ്ങളോ ഉണ്ടാക്കരുത്
- മൂല്യങ്ങൾ ഹാർഡ്കോഡ് ചെയ്യുന്നതിന് പകരം ടോക്കണുകളെ റഫർ ചെയ്യുക

### നാമകരണം (Naming) {#naming}

വ്യക്തവും സെമാന്റിക് ആയ പേരുകൾ ഉപയോഗിക്കുക:

- ✓ `primary`, `secondary`, `accent`
- ✗ `blue`, `color-1`, `main`

- ✓ `heading-lg`, `body-sm`
- ✗ `font-size-24`, `text-large`

### സ്കെയിലബിലിറ്റി (Scalability) {#scalability}

വളരാൻ കഴിയുന്ന രീതിയിൽ നിങ്ങളുടെ സിസ്റ്റം രൂപകൽപ്പന ചെയ്യുക:

- പിക്സലുകൾക്ക് പകരം റിലേറ്റീവ് യൂണിറ്റുകൾ (rem, em) ഉപയോഗിക്കുക
- ആർബിട്രറി മൂല്യങ്ങൾക്ക് പകരം സ്കെയിലുകൾ (xs, sm, md, lg, xl) സൃഷ്ടിക്കുക
- ഭാവിയിലെ കൂട്ടിച്ചേർക്കലുകൾക്കായി ആസൂത്രണം ചെയ്യുക

### ഡോക്യുമെന്റേഷൻ (Documentation) {#documentation}

നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റം രേഖപ്പെടുത്തുക:

- നിങ്ങൾ പ്രത്യേക കളറുകൾ തിരഞ്ഞെടുത്തതിന്റെ കാരണം
- ഓരോ ടോക്കൺ എപ്പോൾ ഉപയോഗിക്കണം
- അപവാദങ്ങളും അരിക സാഹചര്യങ്ങളും (Exceptions and edge cases)

## സാധാരണ ഡിസൈൻ സിസ്റ്റം പാറ്റേണുകൾ (Common Design System Patterns) {#common-design-system-patterns}

### മോഡേൺ മിനിമലിസ്റ്റ് (Modern Minimalist) {#modern-minimalist}

- സാൻസ്-സെറിഫ് ടൈപ്പോഗ്രാഫി (Inter, Helvetica)
- പരിമിതമായ കളർ പാലറ്റ് (2-3 കളറുകൾ)
- വലിയ സ്പേസിംഗ്
- സബ്റ്റിൽ ഷാഡോകൾ
- സുഗമവും വേഗതയേറിയതുമായ ആനിമേഷനുകൾ

### വാമും സൗഹൃദപരവും (Warm and Friendly) {#warm-and-friendly}

- സെറിഫ്, സാൻസ്-സെറിഫ് എന്നിവയുടെ മിശ്രിതം
- ഊഷ്മളമായ കളർ പാലറ്റ് (ഓറഞ്ച്, ഇളം ഗ്രേകൾ)
- റൗണ്ടഡ് കോണുകൾ
- മൃദലമായ ഷാഡോകൾ
- കളിയായ ആനിമേഷനുകൾ

### പ്രൊഫഷണൽ കോർപ്പറേറ്റ് (Professional Corporate) {#professional-corporate}

- വൃത്തിയുള്ള സാൻസ്-സെറിഫ് (Roboto, Open Sans)
- ആക്സന്റ് കളർ ഉള്ള ന്യൂട്രൽ പാലറ്റ്
- ഘടനാപരമായ സ്പേസിംഗ്
- മിനിമൽ ഷാഡോകൾ
- സബ്റ്റിൽ ട്രാൻസിഷനുകൾ

### ക്രിയേറ്റീവ് ആർ ബോൾഡ് (Creative and Bold) {#creative-and-bold}

- പ്രത്യേക ടൈപ്പോഗ്രാഫി
- ധീരമായ കളർ പാലറ്റ്
- വൈവിധ്യമാർന്ന സ്പേസിംഗ്
- ശക്തമായ ഷാഡോകൾ
- ശ്രദ്ധേയമായ ആനിമേഷനുകൾ

## ട്രബിൾഷൂട്ടിംഗ് (Troubleshooting) {#troubleshooting}

**എന്റെ ഡിസൈൻ സിസ്റ്റം മാറ്റങ്ങൾ കാണുന്നില്ല**
- നിങ്ങളുടെ ബ്രൗസർ കാഷെ (cache) ക്ലിയർ ചെയ്യുക
- സ്റ്റാറ്റിക് ജനറേറ്റർ ഉപയോഗിക്കുന്നുണ്ടെങ്കിൽ സൈറ്റ് റീബിൽഡ് ചെയ്യുക
- `theme.json` സാധുവായ JSON ആണോ എന്ന് പരിശോധിക്കുക
- തീം സജീവമാണോ എന്ന് ഉറപ്പാക്കുക

**വിവിധ പേജുകളിൽ കളറുകൾ വ്യത്യസ്തമായി കാണപ്പെടുന്നു**
- പ്ലഗിനുകളിൽ CSS വൈരുദ്ധ്യങ്ങൾ (conflicting CSS) ഉണ്ടോയെന്ന് പരിശോധിക്കുക
- എല്ലാ പേജുകളും ഒരേ തീം ഉപയോഗിക്കുന്നുണ്ടോയെന്ന് ഉറപ്പാക്കുക
- ഏതെങ്കിലും കാഷിംഗ് പ്ലഗിനുകൾ ഉണ്ടെങ്കിൽ അത് ക്ലിയർ ചെയ്യുക

**മുമ്പത്തെ ഡിസൈൻ സിസ്റ്റത്തിലേക്ക് തിരികെ പോകണം**
- ഏജന്റിനോട് ചോദിക്കുക: "എന്റെ ഡിസൈൻ സിസ്റ്റം ചരിത്രം കാണിക്കൂ"
- മുമ്പത്തെ മൂല്യങ്ങളിലേക്ക് മാറ്റാൻ theme.jsonを手വഴിയായി എഡിറ്റ് ചെയ്യുക
- വ്യത്യസ്ത തിരഞ്ഞെടുപ്പുകളോടെ കഴിവ് വീണ്ടും പ്രവർത്തിപ്പിക്കുക

## അടുത്ത ഘട്ടങ്ങൾ (Next Steps) {#next-steps}

നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റം നിർവചിച്ച ശേഷം:

1. **നിങ്ങളുടെ സൈറ്റ് അവലോകനം ചെയ്യുക**: പുതിയ ഡിസൈൻ കാണാൻ നിങ്ങളുടെ സൈറ്റ് സന്ദർശിക്കുക
2. **കൂടുതൽ മെച്ചപ്പെടുത്തുക**: കഴിവ് വീണ്ടും ഉപയോഗിച്ച് ക്രമീകരണങ്ങൾ ചെയ്യുക
3. **ടെംപ്ലേറ്റുകൾ സൃഷ്ടിക്കുക**: നിങ്ങളുടെ ഡിസൈൻ ടോക്കണുകൾ ഉപയോഗിച്ച് കസ്റ്റം ബ്ലോക്ക് ടെംപ്ലേറ്റുകൾ നിർമ്മിക്കുക
4. **രേഖപ്പെടുത്തുക**: നിങ്ങളുടെ ഡിസൈൻ സിസ്റ്റം ടീം അംഗങ്ങളുമായി പങ്കിടുക
