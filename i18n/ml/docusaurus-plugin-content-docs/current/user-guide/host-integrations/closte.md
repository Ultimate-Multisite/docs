# Closte ഇന്റഗ്രേഷൻ

## അവലോകനം
Closte എന്നത് Google Cloud ഇൻഫ്രാസ്ട്രക്ചറിൽ നിർമ്മിച്ച ഒരു മാനേജ്ഡ് WordPress ഹോസ്റ്റിംഗ് പ്ലാറ്റ്‌ഫോമാണ്. Ultimate Multisite-ഉം Closte-ഉം തമ്മിൽ ഓട്ടോമാറ്റിക് ഡൊമെയ്ൻ സിൻകിംഗും SSL സർട്ടിഫിക്കറ്റ് മാനേജ്‌മെന്റും സാധ്യമാക്കുന്നതാണ് ഈ ഇന്റഗ്രേഷൻ.

## ഫീച്ചറുകൾ
- ഓട്ടോമാറ്റിക് ഡൊമെയ്ൻ സിൻകിംഗ്
- SSL സർട്ടിഫിക്കറ്റ് മാനേജ്‌മെന്റ്
- Wildcard ഡൊമെയ്ൻ സപ്പോർട്ട്
- Closte-ൽ പ്രവർത്തിക്കുന്നുണ്ടെങ്കിൽ കോൺഫിഗറേഷൻ ആവശ്യമില്ല

## ആവശ്യകതകൾ
നിങ്ങൾ Closte ഉപയോഗിക്കുന്നുണ്ടെങ്കിൽ, താഴെ കാണുന്ന constant നിങ്ങളുടെ `wp-config.php` ഫയലിൽ നിർവചിച്ചിരിക്കണം:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Closte-ൽ ഹോസ്റ്റ് ചെയ്യുന്നുണ്ടെങ്കിൽ സാധാരണയായി ഈ constant മുൻകൂട്ടി നിർവചിച്ചിരിക്കും.

## സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ

### 1. നിങ്ങളുടെ Closte API Key പരിശോധിക്കുക

നിങ്ങൾ Closte-ൽ ഹോസ്റ്റ് ചെയ്യുന്നുണ്ടെങ്കിൽ, `CLOSTE_CLIENT_API_KEY` constant നിങ്ങളുടെ `wp-config.php` ഫയലിൽ മുൻകൂട്ടി നിർവചിച്ചിരിക്കണം. നിങ്ങളുടെ `wp-config.php` ഫയൽ പരിശോധിച്ച് ഇത് ഉറപ്പാക്കാം.

### 2. ഇന്റഗ്രേഷൻ എനേബിൾ ചെയ്യുക

1. നിങ്ങളുടെ WordPress അഡ്മിനിൽ, Ultimate Multisite > Settings എന്നതിലേക്ക് പോകുക
2. "Domain Mapping" ടാബിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക
3. "Host Integrations" എന്നതിലേക്ക് സ്‌ക്രോൾ ചെയ്യുക
4. Closte ഇന്റഗ്രേഷൻ എനേബിൾ ചെയ്യുക
5. "Save Changes" ക്ലിക്ക് ചെയ്യുക

## ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു

Ultimate Multisite-ൽ ഒരു ഡൊമെയ്ൻ മാപ്പ് ചെയ്യുമ്പോൾ:

1. ഇന്റഗ്രേഷൻ നിങ്ങളുടെ ആപ്ലിക്കേഷനിലേക്ക് ഡൊമെയ്ൻ ചേർക്കാൻ Closte-ന്റെ API-ലേക്ക് ഒരു റിക്വസ്റ്റ് അയയ്ക്കുന്നു
2. SSL സർട്ടിഫിക്കറ്റ് provisioning Closte ഓട്ടോമാറ്റിക്കായി കൈകാര്യം ചെയ്യുന്നു
3. ഒരു ഡൊമെയ്ൻ മാപ്പിംഗ് നീക്കം ചെയ്യുമ്പോൾ, ഇന്റഗ്രേഷൻ Closte-ൽ നിന്ന് ഡൊമെയ്ൻ നീക്കം ചെയ്യും

DNS propagation-ഉം SSL സർട്ടിഫിക്കറ്റ് issuance-ഉം എത്ര കൂടെക്കൂടെ സിസ്റ്റം പരിശോധിക്കണമെന്ന് കോൺഫിഗർ ചെയ്യാൻ അനുവദിക്കുന്ന Ultimate Multisite-ലെ DNS check interval സെറ്റിംഗിനൊപ്പവും ഈ ഇന്റഗ്രേഷൻ പ്രവർത്തിക്കുന്നു.

## ഡൊമെയ്ൻ റെക്കോർഡ് ക്രിയേഷൻ

ഒരു സൈറ്റ് സൃഷ്ടിക്കുമ്പോഴോ ഡ്യൂപ്ലിക്കേറ്റ് ചെയ്യുമ്പോഴോ ഒരു ഡൊമെയ്ൻ റെക്കോർഡ് ഓട്ടോമാറ്റിക്കായി സൃഷ്ടിക്കപ്പെടുമെന്ന് ഈ ഇന്റഗ്രേഷൻ ഉറപ്പാക്കുന്നു. Closte ഇന്റഗ്രേഷന് ഇത് പ്രത്യേകിച്ച് പ്രധാനമാണ്, കാരണം ഡൊമെയ്ൻ റെക്കോർഡ് ക്രിയേഷൻ ഡൊമെയ്നും SSL സർട്ടിഫിക്കറ്റും സൃഷ്ടിക്കാൻ Closte API-യെ ട്രിഗർ ചെയ്യുന്നു.

## ട്രബിൾഷൂട്ടിംഗ്

### API കണക്ഷൻ പ്രശ്നങ്ങൾ
- നിങ്ങളുടെ Closte API key ശരിയാണെന്ന് ഉറപ്പാക്കുക
- നിങ്ങളുടെ Closte അക്കൗണ്ടിന് ആവശ്യമായ permissions ഉണ്ടെന്ന് ഉറപ്പാക്കുക

### SSL സർട്ടിഫിക്കറ്റ് പ്രശ്നങ്ങൾ
- SSL സർട്ടിഫിക്കറ്റുകൾ ഇഷ്യൂ ചെയ്യാൻ Closte-ന് കുറച്ച് സമയമെടുത്തേക്കാം (സാധാരണയായി 5-10 മിനിറ്റ്)
- നിങ്ങളുടെ ഡൊമെയ്നുകൾ നിങ്ങളുടെ Closte സെർവറിന്റെ IP അഡ്രസിലേക്ക് ശരിയായി പോയിന്റ് ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
- നിങ്ങളുടെ ഡൊമെയ്നിന്റെ DNS റെക്കോർഡുകൾ ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് പരിശോധിക്കുക

### ഡൊമെയ്ൻ ചേർക്കപ്പെടുന്നില്ല
- എന്തെങ്കിലും error messages ഉണ്ടോ എന്ന് Ultimate Multisite logs പരിശോധിക്കുക
- ഡൊമെയ്ൻ ഇതിനകം Closte-ൽ ചേർത്തിട്ടില്ലെന്ന് ഉറപ്പാക്കുക
- നിങ്ങളുടെ ഡൊമെയ്നിന്റെ DNS റെക്കോർഡുകൾ ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക

### DNS Check Interval
- SSL സർട്ടിഫിക്കറ്റുകൾ ഇഷ്യൂ ചെയ്യാൻ വളരെ കൂടുതൽ സമയമെടുക്കുന്നുണ്ടെങ്കിൽ, Domain Mapping സെറ്റിംഗ്സിൽ DNS check interval ക്രമീകരിക്കാം
- ഡിഫോൾട്ട് interval 300 സെക്കൻഡ് (5 മിനിറ്റ്) ആണ്, പക്ഷേ ടെസ്റ്റിംഗ് സമയത്ത് വേഗത്തിലുള്ള പരിശോധനയ്ക്കായി 10 സെക്കൻഡ് വരെ കുറയ്ക്കാം
