# GridPane ഇന്റഗ്രേഷൻ

## അവലോകനം
GridPane എന്നത് WordPress പ്രൊഫഷണലുകൾക്കായി പ്രത്യേകമായി നിർമ്മിച്ച ഒരു WordPress ഹോസ്റ്റിംഗ് കൺട്രോൾ പാനലാണ്. ഈ ഇന്റഗ്രേഷൻ Ultimate Multisite-ഉം GridPane-ഉം തമ്മിൽ ഓട്ടോമാറ്റിക് ഡൊമെയ്ൻ സിങ്കിംഗും SSL സർട്ടിഫിക്കറ്റ് മാനേജ്മെന്റും സാധ്യമാക്കുന്നു.

## ഫീച്ചറുകൾ
- ഓട്ടോമാറ്റിക് ഡൊമെയ്ൻ സിങ്കിംഗ്
- SSL സർട്ടിഫിക്കറ്റ് മാനേജ്മെന്റ്
- SUNRISE constant-ന്റെ ഓട്ടോമാറ്റിക് കോൺഫിഗറേഷൻ

## ആവശ്യകതകൾ
നിങ്ങളുടെ `wp-config.php` ഫയലിൽ താഴെ പറയുന്ന constants നിർവചിച്ചിരിക്കണം:

```php
define('WU_GRIDPANE', true);
define('WU_GRIDPANE_API_KEY', 'your_api_key');
define('WU_GRIDPANE_SERVER_ID', 'your_server_id');
define('WU_GRIDPANE_APP_ID', 'your_app_id');
```

## സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ

### 1. നിങ്ങളുടെ GridPane API ക്രെഡൻഷ്യലുകൾ നേടുക

1. നിങ്ങളുടെ GridPane dashboard-ൽ ലോഗിൻ ചെയ്യുക
2. "Settings" > "API" എന്നതിലേക്ക് പോകുക
3. നിങ്ങൾക്ക് ഇതിനകം ഒരു API key ഇല്ലെങ്കിൽ ഒന്ന് ജനറേറ്റ് ചെയ്യുക
4. നിങ്ങളുടെ API key കോപ്പി ചെയ്യുക

### 2. നിങ്ങളുടെ Server-ഉം Site ID-കളും നേടുക

1. നിങ്ങളുടെ GridPane dashboard-ൽ "Servers" എന്നതിലേക്ക് പോകുക
2. നിങ്ങളുടെ WordPress multisite ഹോസ്റ്റ് ചെയ്തിരിക്കുന്ന സെർവർ തിരഞ്ഞെടുക്കുക
3. Server ID കുറിച്ചിടുക (URL-ലോ സെർവർ വിശദാംശ പേജിലോ ഇത് കാണാം)
4. "Sites" എന്നതിലേക്ക് പോയി നിങ്ങളുടെ WordPress സൈറ്റ് തിരഞ്ഞെടുക്കുക
5. Site ID കുറിച്ചിടുക (URL-ലോ സൈറ്റ് വിശദാംശ പേജിലോ ഇത് കാണാം)

### 3. wp-config.php-ൽ Constants ചേർക്കുക

നിങ്ങളുടെ `wp-config.php` ഫയലിൽ താഴെ പറയുന്ന constants ചേർക്കുക:

```php
define('WU_GRIDPANE', true);
define('WU_GRIDPANE_API_KEY', 'your_api_key');
define('WU_GRIDPANE_SERVER_ID', 'your_server_id');
define('WU_GRIDPANE_APP_ID', 'your_site_id');
```

### 4. ഇന്റഗ്രേഷൻ പ്രവർത്തനക്ഷമമാക്കുക

1. നിങ്ങളുടെ WordPress admin-ൽ Ultimate Multisite > Settings എന്നതിലേക്ക് പോകുക
2. "Domain Mapping" ടാബിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക
3. "Host Integrations" എന്നതിലേക്ക് സ്ക്രോൾ ചെയ്യുക
4. GridPane ഇന്റഗ്രേഷൻ പ്രവർത്തനക്ഷമമാക്കുക
5. "Save Changes" ക്ലിക്ക് ചെയ്യുക

## ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു

Ultimate Multisite-ൽ ഒരു ഡൊമെയ്ൻ മാപ്പ് ചെയ്യുമ്പോൾ:

1. നിങ്ങളുടെ സൈറ്റിലേക്ക് ഡൊമെയ്ൻ ചേർക്കാൻ ഇന്റഗ്രേഷൻ GridPane-ന്റെ API-യിലേക്ക് ഒരു റിക്വസ്റ്റ് അയയ്ക്കുന്നു
2. GridPane ഓട്ടോമാറ്റിക്കായി SSL സർട്ടിഫിക്കറ്റ് provisioning കൈകാര്യം ചെയ്യുന്നു
3. ഒരു ഡൊമെയ്ൻ മാപ്പിംഗ് നീക്കം ചെയ്യുമ്പോൾ, ഇന്റഗ്രേഷൻ GridPane-ൽ നിന്ന് ഡൊമെയ്ൻ നീക്കം ചെയ്യും

ഡൊമെയ്ൻ മാപ്പിംഗ് ശരിയായി പ്രവർത്തിക്കാൻ ആവശ്യമായ wp-config.php ഫയലിലെ SUNRISE constant-ഉം ഇന്റഗ്രേഷൻ ഓട്ടോമാറ്റിക്കായി കൈകാര്യം ചെയ്യുന്നു.

## SUNRISE Constant മാനേജ്മെന്റ്

GridPane ഇന്റഗ്രേഷന്റെ ഒരു പ്രത്യേക ഫീച്ചർ, GridPane-ന്റെ സ്വന്തം ഡൊമെയ്ൻ മാപ്പിംഗ് സിസ്റ്റവുമായുള്ള conflict-കൾ ഒഴിവാക്കാൻ wp-config.php-ലെ SUNRISE constant ഓട്ടോമാറ്റിക്കായി revert ചെയ്യുന്നു എന്നതാണ്. ഇത് രണ്ട് സിസ്റ്റങ്ങളും പ്രശ്നങ്ങളില്ലാതെ ഒരുമിച്ച് പ്രവർത്തിക്കുന്നത് ഉറപ്പാക്കുന്നു.

## ട്രബിൾഷൂട്ടിംഗ്

### API കണക്ഷൻ പ്രശ്നങ്ങൾ
- നിങ്ങളുടെ API key ശരിയാണോ എന്ന് പരിശോധിക്കുക
- നിങ്ങളുടെ server-ഉം site ID-കളും ശരിയാണെന്ന് ഉറപ്പാക്കുക
- നിങ്ങളുടെ GridPane അക്കൗണ്ടിന് ആവശ്യമായ permissions ഉണ്ടെന്ന് ഉറപ്പാക്കുക

### SSL സർട്ടിഫിക്കറ്റ് പ്രശ്നങ്ങൾ
- SSL സർട്ടിഫിക്കറ്റുകൾ issue ചെയ്യാൻ GridPane-ന് കുറച്ച് സമയമെടുത്തേക്കാം
- നിങ്ങളുടെ ഡൊമെയ്നുകൾ നിങ്ങളുടെ സെർവറിന്റെ IP address-ലേക്ക് ശരിയായി point ചെയ്യുന്നുണ്ടോ എന്ന് പരിശോധിക്കുക
- നിങ്ങളുടെ സൈറ്റിനായുള്ള GridPane SSL settings പരിശോധിക്കുക

### ഡൊമെയ്ൻ ചേർക്കപ്പെട്ടില്ല
- എന്തെങ്കിലും error messages-നായി Ultimate Multisite logs പരിശോധിക്കുക
- ഡൊമെയ്ൻ ഇതിനകം GridPane-ൽ ചേർത്തിട്ടില്ലെന്ന് ഉറപ്പാക്കുക
- നിങ്ങളുടെ ഡൊമെയ്നിന്റെ DNS records ശരിയായി configure ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
