# സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്കിൽ

**സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്കിൽ** എന്നത് നിങ്ങളുടെ സൈറ്റിന്റെ ലക്ഷ്യങ്ങൾ, പ്രേക്ഷകർ, ബ്രാൻഡ് ഐഡന്റിറ്റി എന്നിവ രേഖപ്പെടുത്താനുള്ള ഒരു ഘടനാപരമായ സമീപനമാണ്. ഈ വിവരങ്ങൾ നിങ്ങളുടെ **site_brief** മെമ്മറിയിൽ (memory) സൂക്ഷിക്കപ്പെടുന്നു. ഈ മെമ്മറി ഏജന്റുമാർ സെഷനുകൾക്കിടയിൽ റഫർ ചെയ്ത്, സ്ഥിരവും സന്ദർഭത്തിനനുസരിച്ചുള്ളതുമായ സഹായം നൽകുന്നു.

## സൈറ്റ് സ്പെസിഫിക്കേഷൻ എന്നാൽ എന്ത്?

സൈറ്റ് സ്പെസിഫിക്കേഷൻ എന്നത് താഴെ പറയുന്ന കാര്യങ്ങൾ രേഖപ്പെടുത്തുന്ന പ്രക്രിയയാണ്:

- **സൈറ്റ് ഉദ്ദേശ്യം (Site purpose)**: നിങ്ങളുടെ സൈറ്റ് എന്തിനാണ്, എന്തിനുവേണ്ടിയാണ് നിലനിൽക്കുന്നത്.
- **ലക്ഷ്യമിടുന്ന പ്രേക്ഷകർ (Target audience)**: ആരെയാണ് നിങ്ങളുടെ സൈറ്റ് സന്ദർණය ചെയ്യുന്നത്, അവർക്ക് എന്ത് ആവശ്യമാണ്.
- **ബ്രാൻഡ് ഐഡന്റിറ്റി (Brand identity)**: നിങ്ങളുടെ നിറങ്ങൾ, സ്വരധ്വനി (tone), ദൃശ്യ ശൈലി.
- **ബിസിനസ് ലക്ഷ്യങ്ങൾ (Business goals)**: നിങ്ങളുടെ സൈറ്റിന് വിജയം എന്നാൽ എന്താണ്.
- **ഉള്ളടക്ക ഘടന (Content structure)**: നിങ്ങളുടെ സൈറ്റ് എങ്ങനെയാണ് ക്രമീകരിച്ചിരിക്കുന്നത്.

ഈ സ്പെസിഫിക്കേഷൻ നിങ്ങളുടെ **site_brief** എന്ന സ്ഥിരമായ മെമ്മറിയായി മാറുന്നു. ഏജന്റുമാർ ഇത് ഉപയോഗിച്ച് നിങ്ങളുടെ സൈറ്റിന്റെ സന്ദർഭം മനസ്സിലാക്കുന്നു.

## സൈറ്റ് സ്പെസിഫിക്കേഷൻ എന്തിന് ഉപയോഗിക്കണം?

### സെഷനുകൾക്കിടയിലെ സ്ഥിരത

site_brief ഇല്ലെങ്കിൽ, ഓരോ പുതിയ സെഷൻ തുടങ്ങുമ്പോഴും നിങ്ങളുടെ സൈറ്റിന്റെ ഉദ്ദേശ്യം നിങ്ങൾ വീണ്ടും വിശദീകരിക്കേണ്ടി വരും. site_brief ഉണ്ടെങ്കിൽ, ഏജന്റുമാർ ഉടൻ തന്നെ മനസ്സിലാക്കും:

- നിങ്ങളുടെ സൈറ്റിന്റെ ലക്ഷ്യങ്ങളും പ്രേക്ഷകരെയും
- നിങ്ങളുടെ ബ്രാൻഡ് നിറങ്ങളും സ്വരധ്വനിയും
- നിങ്ങളുടെ ഉള്ളടക്ക ഘടനയും
- നിങ്ങളുടെ ബിസിനസ് ലക്ഷ്യങ്ങളും

### മികച്ച ശുപാർശകൾ

നിങ്ങളുടെ site_brief ഉപയോഗിച്ച് ഏജന്റുമാർ ഇവ ചെയ്യും:

- നിങ്ങളുടെ സൈറ്റിന്റെ ഉദ്ദേശ്യവുമായി യോജിച്ച ഫീച്ചറുകൾ നിർദ്ദേശിക്കുക.
- നിങ്ങളുടെ ലക്ഷ്യങ്ങളുമായി ചേരുന്ന ഉള്ളടക്ക ഘടനകൾ ശുപാർശ ചെയ്യുക.
- നിങ്ങളുടെ ബ്രാൻഡിന് അനുയോജ്യമായ ഡിസൈനുകൾ മുന്നോട്ട് വെക്കുക.
- അനുയോജ്യമല്ലാത്ത ഫീച്ചറുകൾ നിർദ്ദേശിക്കുന്നത് ഒഴിവാക്കുക.

### വേഗത്തിലുള്ള ഓൺബോർഡിംഗ്

പുതിയ ഏജന്റുമാർക്ക് (അല്ലെങ്കിൽ പുതിയ സെഷനുകളിലെ ഏജന്റുമാർക്ക്) വ്യക്തമാക്കുന്ന ചോദ്യങ്ങൾ ചോദിക്കുന്നതിന് പകരം, നിങ്ങളുടെ site_brief വായിച്ചുകൊണ്ട് വേഗത്തിൽ കാര്യങ്ങൾ മനസ്സിലാക്കാൻ കഴിയും.

## സൈറ്റ് സ്പെസിഫിക്കേഷൻ ആരംഭിക്കൽ

### Theme Builder ഓൺബോർഡിംഗിനിടയിൽ

**Theme Builder ഓൺബോർഡിംഗ് ഫ്ലോ** സമയത്താണ് സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്കിൽ ഓട്ടോമാറ്റിക്കായി ആരംഭിക്കുന്നത്. Setup Assistant ഏജന്റ് ചോദ്യങ്ങൾ ചോദിക്കുകയും നിങ്ങളുടെ site_brief നിർമ്മിക്കുകയും ചെയ്യുന്നു.

### സ്വയം ആരംഭിക്കൽ (Manual Initiation)

നിങ്ങൾക്ക് എപ്പോൾ വേണമെങ്കിലും സൈറ്റ് സ്പെസിഫിക്കേഷൻ ആരംഭിക്കാം:

```
"Let's define my site specification"
```

അല്ലെങ്കിൽ

```
"Help me create a site brief"
```

## സൈറ്റ് സ്പെസിഫിക്കേഷൻ പ്രക്രിയ

### ഘട്ടം 1: സൈറ്റ് ഉദ്ദേശ്യം (Site Purpose)

ഏജന്റ് ചോദിക്കുന്നു:

```
What is your site's primary purpose?
- E-commerce store
- Blog or content site
- Portfolio or showcase
- SaaS application
- Community or forum
- Other: [describe]
```

നിങ്ങൾക്ക് ഒരു വിഭാഗം തിരഞ്ഞെടുക്കാനോ സ്വന്തമായി ഒരു ഉദ്ദേശ്യം വിവരിക്കാനോ കഴിയും.

### ഘട്ടം 2: ലക്ഷ്യമിടുന്ന പ്രേക്ഷകർ (Target Audience)

```
Who is your primary audience?
- Consumers / general public
- Business professionals
- Developers / technical users
- Students / educators
- Other: [describe]

What are their main needs?
```

### ഘട്ടം 3: ബ്രാൻഡ് ഐഡന്റിറ്റി (Brand Identity)

```
What are your brand colors?
- Primary color: [color picker or hex code]
- Secondary color: [color picker or hex code]
- Accent color: [optional]

How would you describe your brand tone?
- Professional / corporate
- Creative / artistic
- Playful / casual
- Minimal / modern
- Warm / friendly
```

### ഘട്ടം 4: ബിസിനസ് ലക്ഷ്യങ്ങൾ (Business Goals)

```
What does success look like for your site?
- Generate leads
- Sell products
- Build community
- Share knowledge
- Establish authority
- Other: [describe]

What's your primary metric?
- Revenue
- User engagement
- Content reach
- Conversions
- Other
```

### ഘട്ടം 5: ഉള്ളടക്ക ഘടന (Content Structure)

```
How is your content organized?
- Flat (all content at same level)
- Hierarchical (categories and subcategories)
- Chronological (blog-style)
- Product-based (catalog)
- Other: [describe]

What content types do you use?
- Blog posts
- Product pages
- Case studies
- Documentation
- Videos
- Other
```

## നിങ്ങളുടെ site_brief മെമ്മറി

സൈറ്റ് സ്പെസിഫിക്കേഷൻ പൂർത്തിയാക്കിയ ശേഷം, നിങ്ങളുടെ വിവരങ്ങൾ നിങ്ങളുടെ ഏജന്റ് മെമ്മറിയിൽ **site_brief** ആയി സൂക്ഷിക്കപ്പെടുന്നു. ഇതിൽ താഴെ പറയുന്നവ അടങ്ങിയ ഒരു ഘടനാപരമായ രേഖയുണ്ട്:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## നിങ്ങളുടെ site_brief കാണുന്നതും അപ്ഡേറ്റ് ചെയ്യുന്നതും

### നിങ്ങളുടെ site_brief കാണുക

ഏജന്റിനോട് ചോദിക്കുക:

```
"Show me my site brief"
```

അല്ലെങ്കിൽ

```
"What do you know about my site?"
```

നിങ്ങളുടെ സ്റ്റോർ ചെയ്ത സ്പെസിഫിക്കേഷൻ ഏജന്റ് പ്രദർശിപ്പിക്കും.

### നിങ്ങളുടെ site_brief അപ്ഡേറ്റ് ചെയ്യുക

നിങ്ങളുടെ സൈറ്റ് മാറിക്കൊണ്ടിരിക്കുകയാണെങ്കിൽ, നിങ്ങൾക്ക് ഇത് അപ്ഡേറ്റ് ചെയ്യാം:

```
"Update my site brief: we're now targeting B2B customers"
```

അല്ലെങ്കിൽ

```
"Refresh my site specification"
```

ഇത് നിങ്ങളുടെ നിലവിലുള്ള വിവരങ്ങൾ ഒരു തുടക്കമായി ഉപയോഗിച്ച് സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്കിൽ വീണ്ടും പ്രവർത്തിപ്പിക്കും.

## ഏജന്റുമാർ site_brief എങ്ങനെ ഉപയോഗിക്കുന്നു

### ഡിസൈൻ ശുപാർശകൾ

നിങ്ങൾ ഡിസൈൻ മാറ്റങ്ങൾ ആവശ്യപ്പെടുമ്പോൾ, ഏജന്റുമാർ നിങ്ങളുടെ site_brief റഫർ ചെയ്യുന്നു:

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### ഫീച്ചർ നിർദ്ദേശങ്ങൾ

ഏജന്റുമാർ നിങ്ങളുടെ ലക്ഷ്യങ്ങളുമായി യോജിച്ച ഫീച്ചറുകൾ നിർദ്ദേശിക്കുന്നു:

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### ഉള്ളടക്ക ഘടന

നിങ്ങളുടെ ഘടനയെ അടിസ്ഥാനമാക്കി ഏജന്റുമാർ ഉള്ളടക്ക സംഘടനാരീതികൾ മുന്നോട്ട് വെക്കുന്നു:

```
You: "How should I organize my blog?"
Agent: "Since you use a product catalog with blog, I'd suggest
organizing posts by product category and creating 'styling guides'
that showcase multiple products together."
```

## മികച്ച രീതികൾ (Best Practices)

### വ്യക്തമാക്കുക (Be Specific)

"പൊതുജനം" എന്നതിന് പകരം, നിങ്ങളുടെ യഥാർത്ഥ പ്രേക്ഷകരെ വിവരിക്കുക:

- ✓ "Women aged 25-45, interested in sustainable fashion"
- ✗ "Everyone"

### പതിവായി അപ്ഡേറ്റ് ചെയ്യുക (Update Regularly)

നിങ്ങളുടെ സൈറ്റ് മാറിക്കൊണ്ടിരിക്കുമ്പോൾ, നിങ്ങളുടെ site_brief അപ്ഡേറ്റ് ചെയ്യുക:

- നിങ്ങൾ പുതിയ പ്രേക്ഷകരിലേക്ക് മാറുമ്പോൾ (pivot)
- നിങ്ങൾ പുതിയ ഉൽപ്പന്ന ലൈനുകൾ ചേർക്കുമ്പോൾ
- നിങ്ങളുടെ ബ്രാൻഡ് ഐഡന്റിറ്റി മാറുമ്പോൾ
- നിങ്ങളുടെ ബിസിനസ് ലക്ഷ്യങ്ങൾ മാറുമ്പോൾ

### സ്ഥിരമായ പദാവലികൾ ഉപയോഗിക്കുക (Use Consistent Terminology)

സെഷനുകൾ മുഴുവൻ ഒരേ വാക്കുകൾ ഉപയോഗിക്കുക:

- ✓ എപ്പോഴും "sustainable jewelry" എന്ന് പറയുക ( "eco-friendly jewelry" എന്നും "green products" എന്നും പറയരുത്)
- ✓ നിങ്ങളുടെ പ്രേക്ഷകരെ എപ്പോഴും ഒരേ രീതിയിൽ പരാമർശിക്കുക

### സന്ദർഭം ഉൾപ്പെടുത്തുക (Include Context)

ഏജന്റുമാർക്ക് നിങ്ങളുടെ തീരുമാനങ്ങൾ മനസ്സിലാക്കാൻ സഹായിക്കുന്ന പശ്ചാത്തലം നൽകുക:

- "വിലയേക്കാൾ ഗുണത്തിന് പ്രാധാന്യം നൽകുന്ന പ്രൊഫഷണലുകളെയാണ് ഞങ്ങൾ ലക്ഷ്യമിടുന്നത്"
- "ഞങ്ങളുടെ പ്രേക്ഷകർ സാങ്കേതികവിദ്യയെക്കുറിച്ച് അറിവുള്ളവരും ആധുനിക ഡിസൈൻ പ്രതീക്ഷിക്കുന്നവരുമാണ്"
- "ഞങ്ങൾ ഒരു ബൂട്ട്‌സ്ട്രാപ്പ് സ്റ്റാർട്ടപ്പാണ്, അതിനാൽ നമുക്ക് ചെലവ് കുറഞ്ഞ പരിഹാരങ്ങൾ ആവശ്യമാണ്"

## Theme Builder ഓൺബോർഡിംഗുമായുള്ള ബന്ധം

സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്കിൽ **Theme Builder ഓൺബോർഡിംഗ് ഫ്ലോ** യുമായി സംയോജിപ്പിച്ചിരിക്കുന്നു. നിങ്ങൾ ഓൺബോർഡിംഗ് പൂർത്തിയാക്കുമ്പോൾ, നിങ്ങൾ നൽകിയ വിവരങ്ങൾ ഉപയോഗിച്ച് നിങ്ങളുടെ site_brief ഓട്ടോമാറ്റിക്കായി സൃഷ്ടിക്കപ്പെടുന്നു.

നിങ്ങൾക്ക് വേണമെങ്കിൽ, സൈറ്റ് സ്പെസിഫിക്കേഷൻ സ്വതന്ത്രമായി (independently) പ്രവർത്തിപ്പിക്കാനും കഴിയും:

- ആദ്യ സജ്ജീകരണത്തിന് ശേഷം നിങ്ങളുടെ സ്പെസിഫിക്കേഷൻ മെച്ചപ്പെടുത്താൻ
- നിങ്ങളുടെ സൈറ്റ് മാറിക്കൊണ്ടിരിക്കുമ്പോൾ site_brief അപ്ഡേറ്റ് ചെയ്യാൻ
- Theme Builder തുടങ്ങുന്നതിന് മുമ്പ് വിശദമായ സ്പെസിഫിക്കേഷൻ ഉണ്ടാക്കാൻ

## ട്രബിൾഷൂട്ടിംഗ് (Troubleshooting)

**എന്റെ site_brief ഉപയോഗിക്കുന്നില്ല**
- ഏജന്റിന് മെമ്മറി ആക്സസ് ഉണ്ടോയെന്ന് ഉറപ്പാക്കുക.
- ഏജന്റിനോട് "recall my site brief" എന്ന് ചോദിക്കുക.
- നിങ്ങളുടെ സെറ്റിംഗുകളിൽ മെമ്മറി പ്രവർത്തനക്ഷമമാക്കിയിട്ടുണ്ടോയെന്ന് പരിശോധിക്കുക.

**പുതിയ site_brief ഉപയോഗിച്ച് തുടങ്ങണം**
- ഏജന്റിനോട് ചോദിക്കുക: "Clear my site brief and start fresh"
- അതിനുശേഷം സൈറ്റ് സ്പെസിഫിക്കേഷൻ വീണ്ടും പ്രവർത്തിപ്പിക്കുക.

**എന്റെ site_brief യുമായി യോജിക്കാത്ത ശുപാർശകൾ ഏജന്റ് നൽകുന്നു**
- ഏജന്റിനോട് "review my site brief" എന്ന് ചോദിക്കുക.
- അത് കാലഹരണപ്പെട്ടതാണെങ്കിൽ നിങ്ങളുടെ site_brief അപ്ഡേറ്റ് ചെയ്യുക.
- നിങ്ങളുടെ അഭ്യർത്ഥനകളിൽ കൂടുതൽ സന്ദർഭം നൽകുക.
