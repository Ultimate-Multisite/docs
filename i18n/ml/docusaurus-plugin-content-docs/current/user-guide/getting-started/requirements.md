# Ultimate Multisite ആവശ്യകതകൾ

Ultimate Multisite വാങ്ങുന്നതിന് മുമ്പ് നിങ്ങൾ ആദ്യം ചോദിച്ചേക്കാവുന്ന കാര്യം, ഞങ്ങളുടെ plugin പ്രവർത്തിപ്പിക്കാൻ എന്തൊക്കെ ആവശ്യകതകൾ വേണമെന്നാണ്. ഭാഗ്യവശാൽ, Ultimate Multisite പ്രവർത്തിപ്പിക്കാൻ ചില അടിസ്ഥാന ആവശ്യകതകൾ മാത്രമേ വേണ്ടൂ: ഒരു **domain name**, ഒരു നല്ല **hosting provider**, ഒരു WordPress **Multisite** ഇൻസ്റ്റലേഷൻ. ഇത്രയും മതി!

## Domain Name

ആദ്യം, domain name-നെ കുറിച്ച് സംസാരിക്കാം.

Domain name തിരഞ്ഞെടുക്കുമ്പോൾ, നിങ്ങളുടെ പ്രത്യേക മേഖല എന്താണെന്നും Ultimate Multisite ഉപയോഗിച്ച് നിങ്ങൾ എന്ത് സൃഷ്ടിക്കാനാണ് ഉദ്ദേശിക്കുന്നതെന്നും മനസ്സിൽ വയ്ക്കുക: നിങ്ങൾക്ക് ഒരു ഏജൻസി സൃഷ്ടിക്കണോ? ഷോപ്പ് പോലുള്ള വെബ്‌സൈറ്റുകൾ വിൽക്കണോ? അതോ ഒരു WordPress web hosting? Ultimate Multisite ഉപയോഗിച്ച് എല്ലാം സാധ്യമാണ്.

ഞങ്ങളുടെ plugin subscription services, payment collection, checkout forms, discount vouchers, customer communications എന്നിവ നൽകുന്നു. WordPress Multisite ഇൻസ്റ്റാൾ ചെയ്യുന്നതിനും, കോൺഫിഗർ ചെയ്യുന്നതിനും, പരിപാലിക്കുന്നതിനും ആവശ്യമായ വലിയൊരു ഭാഗം ജോലികൾ Ultimate Multisite എളുപ്പമാക്കുന്നു.

നിങ്ങൾ വെബ്‌സൈറ്റുകൾ ഒരു സേവനമായി വിൽക്കുന്നിടത്തോളം, തിരഞ്ഞെടുക്കുന്ന ഏത് മേഖലയും ഞങ്ങളുടെ plugin ഉപയോഗിച്ച് കൈകാര്യം ചെയ്യാൻ കഴിയും! നിങ്ങളുടെ ബിസിനസ് പ്ലാനുമായി ചേരുന്ന ഒരു domain name തിരഞ്ഞെടുക്കുന്നത് ഉറപ്പാക്കുക.

## Hosting Provider

ഇപ്പോൾ നിങ്ങളുടെ മേഖലയെയും ബിസിനസിനെയും പ്രതിനിധീകരിക്കുന്ന ആകർഷകമായ ഒരു domain name ഉണ്ട്, Ultimate Multisite-ൽ നിങ്ങളുടെ network host ചെയ്യാൻ ഒരു നല്ല web hosting കമ്പനി തിരഞ്ഞെടുക്കേണ്ടതുണ്ട്. വിപണിയിൽ ചില നല്ല ഓപ്ഷനുകൾ ലഭ്യമാണ്. ചില hosting providers Ultimate Multisite-മായി നന്നായി സംയോജിക്കുന്നു, നിങ്ങൾ നിർമ്മിക്കാൻ ഉദ്ദേശിക്കുന്നതിനെ ആശ്രയിച്ച് ഇത് ഒരു നിർണായക ഘടകമായേക്കാം.

Ultimate Multisite ഏത് hosting provider-ലും പ്രവർത്തിക്കും. വ്യത്യാസം ഇത്രമാത്രം: _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_, _Runcloud_ തുടങ്ങിയ ചില hosting കമ്പനികൾക്ക് domain mapping-ന് കൂടുതൽ മികച്ച integration ഉണ്ട്. മറ്റ് hosting-കൾക്ക്, custom domains manual ആയി map ചെയ്യേണ്ടി വന്നേക്കാം. AutoSSL പിന്തുണയ്ക്കുന്ന ഒരു hosting provider തിരഞ്ഞെടുക്കുന്നതും നല്ലൊരു ആശയമാണ്. _Closte_, _Runcloud_, _cPanel_ എന്നിവ ഇത് പിന്തുണയ്ക്കുന്നു.

Server specifications-ന്റെ കാര്യത്തിൽ, പ്രത്യേക ഫോർമുലയൊന്നുമില്ല, കാരണം ഇത് നിങ്ങൾ വാഗ്ദാനം ചെയ്യുന്ന features-നെയും functions-നെയും നിങ്ങളുടെ network-ന്റെ വലുപ്പത്തെയും ആശ്രയിച്ചിരിക്കും. Shared hosting ഒരു നല്ല തുടക്കമായിരിക്കാം, പക്ഷേ നിങ്ങളുടെ network വളരാൻ തുടങ്ങുമ്പോൾ, ഒരു dedicated VPS ആവശ്യമായി വന്നേക്കാം.

## WordPress Multisite

ഇനി, WordPress Multisite installation-നെ കുറിച്ച് കുറച്ച് സംസാരിക്കാം.

WordPress Multisite ഒരു ഇൻസ്റ്റലേഷനിൽ ഒരു sites network ഉണ്ടാക്കാൻ നിങ്ങളെ അനുവദിക്കുന്നു. ഇത് ഒരു built-in feature ആണ്, പക്ഷേ default ആയി active അല്ല. നിങ്ങളുടെ WordPress installation-ലും web hosting control panel-ലും ഈ feature enable ചെയ്യേണ്ടതുണ്ട്.

Multisite enable ചെയ്ത WordPress **Ultimate Multisite-ന്റെ അടിസ്ഥാനമാണ്**. Ultimate Multisite-ന്റെ പ്രധാന ഉപയോഗം users-നെ websites ഒരു service ആയി manage ചെയ്യാനും വിൽക്കാനും സഹായിക്കുക എന്നതായതിനാൽ, ഇത് Multisite enable ചെയ്ത WordPress-ൽ ചെയ്യേണ്ടതുണ്ട്.

നിങ്ങളുടെ WordPress installation-ൽ Multisite feature enable ചെയ്യാൻ എന്തൊക്കെ ചെയ്യണമെന്ന് step by step പഠിപ്പിക്കുന്ന **ഒരു വിശദമായ ലേഖനം** ഞങ്ങളുടെ പക്കലുണ്ട്.

WordPress-ൽ Multisite enable ചെയ്തുകഴിഞ്ഞാൽ, Ultimate Multisite install ചെയ്യാനുള്ള സമയമായി. Ultimate Multisite ശരിയായി install ചെയ്യാൻ, നിങ്ങൾക്ക് ഇവ ഉണ്ടായിരിക്കണം:

  * WordPress: v5.3+ (ശുപാർശ ചെയ്യുന്നത്: അവസാനത്തെ stable version)

  * PHP: 7.4.x (> 8.0-ന്റെ support ഉടൻ വരുന്നു)

  * MySQL: v5+ (ശുപാർശ ചെയ്യുന്നത്: 5.6, നിങ്ങളുടെ hosting provider-ൽ 8.0 ലഭ്യമല്ലെങ്കിൽ)

Ultimate Multisite-ന് ഇവ മാത്രമാണ് software requirements.

Ultimate Multisite PHP 8.0-ൽ പ്രവർത്തിച്ചേക്കാം, പക്ഷേ PHP 7.4.x-ൽ പ്രവർത്തിപ്പിക്കാനാണ് ഞങ്ങൾ ശുപാർശ ചെയ്യുന്നത്.

കൂടാതെ, subdomains അല്ലെങ്കിൽ subdirectories-ൽ അല്ല, **main domains**-ൽ install ചെയ്യാനാണ് ഞങ്ങൾ ശുപാർശ ചെയ്യുന്നത്. Subdomains/subdirectories-ൽ ഇത് പ്രവർത്തിച്ചേക്കാം, പക്ഷേ ചില errors ഉണ്ടായേക്കാം.

Ultimate Multisite എങ്ങനെ install ചെയ്യാമെന്നതിനെ കുറിച്ചുള്ള വിശദമായ ഗൈഡിന്, **ഇവിടെ ക്ലിക്ക് ചെയ്യുക**.
