# മറ്റ് പ്ലഗിനുകളുമായുള്ള അറിയപ്പെടുന്ന അനുയോജ്യതാ പ്രശ്നങ്ങൾ

Ultimate Multisite ഇൻസ്റ്റാൾ ചെയ്ത് ആക്ടിവേറ്റ് ചെയ്ത ശേഷം, മറ്റ് പ്ലഗിനുകളുമായുള്ള പരിഹരിക്കപ്പെടാത്ത അനുയോജ്യതാ പ്രശ്നങ്ങൾ കാരണം നിങ്ങൾക്ക് ബുദ്ധിമുട്ടുകൾ നേരിടേണ്ടി വന്നേക്കാം.

ഓരോ പുതിയ റിലീസിലും ഈ പട്ടിക കുറയ്ക്കാൻ ഞങ്ങൾ സജീവമായി പ്രവർത്തിക്കുന്നു, എന്നാൽ ഒരു പ്ലഗിൻ ഇവിടെ ലിസ്റ്റ് ചെയ്തിട്ടുണ്ടെങ്കിൽ, Ultimate Multisite Network പൂർണ്ണമായി പ്രവർത്തിക്കുന്നതിന് നിങ്ങൾ അത് ഡീആക്ടിവേറ്റ് ചെയ്യേണ്ടി വന്നേക്കാം.

ഇവയുടെ ഒരു പട്ടികയും ഈ സാഹചര്യങ്ങളിൽ നിങ്ങൾ എന്ത് ചെയ്യണമെന്നും താഴെ കൊടുക്കുന്നു:

**ചില WPMU DEV പ്ലഗിനുകൾ** WPMU DEV _ഇനി പരിപാലിക്കാത്ത_ ചില പ്ലഗിനുകൾ Ultimate Multisite രജിസ്ട്രേഷൻ പ്രക്രിയ അവസാന ഘട്ടത്തിൽ നിർത്തിപ്പോകുന്നതിന് കാരണമാകുന്നതായി അറിയപ്പെടുന്നു (ഉപയോക്താവിന് അക്കൗണ്ട് സൃഷ്ടിക്കാൻ കഴിയുന്നില്ല). ഈ പ്രശ്നങ്ങൾക്ക് കാരണമാകുന്ന പ്ലഗിനുകളുടെ പട്ടിക ഇതാ:

  * Signup Language: _Ultimate Multisite-ന് ഇതേ പ്രവർത്തനക്ഷമത നൽകുന്ന ഒരു add-on ഉണ്ട്, അതിന്റെ പേര് Ultimate Multisite: Language Switcher എന്നാണ്._

  * Set Blog Description on Blog Creation: _ഇത് Ultimate Multisite-ൽ തന്നെ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്, അതിനാൽ അധിക പ്ലഗിൻ ആവശ്യമില്ല._

  * New Blog Templates: _ഇത് Ultimate Multisite-ൽ തന്നെ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്, അതിനാൽ അധിക പ്ലഗിൻ ആവശ്യമില്ല._

  * Set Password on Multisite Blog Creation: _ഇത് Ultimate Multisite-ൽ തന്നെ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്, അതിനാൽ അധിക പ്ലഗിൻ ആവശ്യമില്ല._

  * Signup TOS: _ഇത് Ultimate Multisite-ൽ തന്നെ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്, അതിനാൽ അധിക പ്ലഗിൻ ആവശ്യമില്ല._

  * Anti-Splog: കുറച്ച് ശക്തി കുറവാണെങ്കിലും, ഇതും _Ultimate Multisite-ൽ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്._

**കുറിപ്പ്** : മറ്റെല്ലാ WPMU Dev പ്ലഗിനുകളും Ultimate Multisite-നൊപ്പം സാധാരണ രീതിയിൽ ഉപയോഗിക്കാം. ഉദാഹരണങ്ങളിൽ _Smush_ , _Forminator_ , _Defender,_ മുതലായവ ഉൾപ്പെടുന്നു.
