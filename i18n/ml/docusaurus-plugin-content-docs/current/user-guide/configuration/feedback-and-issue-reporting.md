# ഉപയോക്തൃ പ്രതികരണവും പ്രശ്ന റിപ്പോർട്ടിംഗും {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0-ൽ ഒരു ബിൽറ്റ്-ഇൻ ഫീഡ്‌ബാക്ക്, പ്രശ്ന റിപ്പോർട്ടിംഗ് സംവിധാനം ഉൾപ്പെടുത്തിയിരിക്കുന്നു. ഇത് ഉപയോക്താക്കളെ ചാറ്റ് ഇന്റർഫേസിൽ നിന്ന് തന്നെ സഹായകരമല്ലാത്ത മറുപടികൾ അടയാളപ്പെടുത്താനും പ്രശ്നങ്ങൾ റിപ്പോർട്ട് ചെയ്യാനും അനുവദിക്കുന്നു. ഈ സംവിധാനത്തിൽ സമ്മത മാനേജ്‌മെന്റ്, ഓട്ടോമേറ്റഡ് റിപ്പോർട്ടിംഗ് കമാൻഡ്, ബാക്ക്എൻഡിൽ AI സഹായത്തോടെയുള്ള ട്രൈയാജ് എന്നിവ ഉൾപ്പെടുന്നു.

## തള്ളവിരൽ ബട്ടൺ (Thumbs-Down Button) {#thumbs-down-button}

AI അസിസ്റ്റന്റ് അയക്കുന്ന എല്ലാ സന്ദേശങ്ങളിലും ഒരു **തള്ളവിരൽ ബട്ടൺ** (👎) കാണാം. ഒരു ഉപയോക്താവ് ഇത് ക്ലിക്ക് ചെയ്യുമ്പോൾ, അവർക്ക് ഒരു മറുപടി സഹായകരമല്ലാത്തതോ തെറ്റായതോ ആണെന്ന് അടയാളപ്പെടുത്താൻ കഴിയും.

- അസിസ്റ്റന്റ് സന്ദേശങ്ങൾക്കരികിൽ ഹോവർ ചെയ്യുമ്പോൾ ഈ ബട്ടൺ പ്രത്യക്ഷപ്പെടും.
- ഇത് ക്ലിക്ക് ചെയ്യുന്നത് **ഫീഡ്‌ബാക്ക് സമ്മത മോഡൽ** (Feedback Consent Modal) തുറക്കും.
- നൽകുന്ന ഫീഡ്‌ബാക്ക് സംഭാഷണ സ്ട്രീത്തമായും പ്രത്യേക സന്ദേശവുമായും ബന്ധിപ്പിക്കുന്നു.

## ഫീഡ്‌ബാക്ക് സമ്മത മോഡൽ (Feedback Consent Modal) {#feedback-consent-modal}

ഉപയോക്താവ് തള്ളവിരൽ ബട്ടൺ ക്ലിക്ക് ചെയ്യുമ്പോൾ, ഡാറ്റ അയക്കുന്നതിന് മുമ്പ് ഒരു സമ്മത മോഡൽ പ്രത്യക്ഷപ്പെടുന്നു. ഈ മോഡൽ:

- ഏത് വിവരങ്ങളാണ് ശേഖരിക്കുക എന്ന് വിശദീകരിക്കുന്നു (സംഭാഷണ ഭാഗം, ബ്രൗസർ സന്ദർഭം).
- ഈ ഡാറ്റ പങ്കിടാൻ സമ്മതം നൽകുന്നുണ്ടോ എന്ന് ഉപയോക്താവിനോട് സ്ഥിരീകരിക്കാൻ ആവശ്യപ്പെടുന്നു.
- എന്ത് തെറ്റാണ് സംഭവിച്ചതെന്ന് വിവരിക്കാൻ ഒരു ഫ്രീ-ടെക്സ്റ്റ് ഫീൽഡ് നൽകുന്നു.
- സമർപ്പിക്കാതെ ഒഴിവാക്കാൻ ഒരു **Cancel** ഓപ്ഷൻ നൽകുന്നു.

ഉപയോക്താവ് വ്യക്തമായി സ്ഥിരീകരിക്കുന്നതുവരെ ഒരു ഫീഡ്‌ബാക്കും രേഖപ്പെടുത്തുകയില്ല.

## ഓട്ടോ-പ്രോംപ്റ്റ് ഫീഡ്‌ബാക്ക് ബാനർ (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

ഒരു സംഭാഷണം അവസാനിക്കുമ്പോൾ, അസിസ്റ്റന്റ് ഒരു **ഓട്ടോ-പ്രോംപ്റ്റ് ഫീഡ്‌ബാക്ക് ബാനർ** പ്രദർശിപ്പിക്കാൻ സാധ്യതയുണ്ട്. ഇത് ഒരു അലോസരമല്ലാത്ത സന്ദേശമാണ്. സെഷൻ സഹായകരമായിരുന്നോ എന്ന് ഇത് ചോദിക്കുന്നു.

സംഭാഷണത്തിന്റെ ദൈർഘ്യത്തെയും ഫലത്തെയും അടിസ്ഥാനമാക്കിയുള്ള ഹ്യൂറിസ്റ്റിക്സ് അനുസരിച്ച് ഇത് സ്വയമേവ പ്രത്യക്ഷപ്പെടുന്നു. ഇത് തള്ളവിരൽ ബട്ടൺ പോലെ തന്നെ ഫീഡ്‌ബാക്ക് ഫ്ലോയിലേക്ക് ബന്ധിപ്പിച്ചിരിക്കുന്നു. ഫീഡ്‌ബാക്ക് നൽകാതെ ഉപയോക്താവിന് ഈ ബാനർ ഒഴിവാക്കാൻ കഴിയും.

## /report-issue സ്ലാഷ് കമാൻഡ് {#report-issue-slash-command}

ചാറ്റ് ഇൻപുട്ടിൽ `/report-issue` എന്ന് ടൈപ്പ് ചെയ്യുന്നതിലൂടെ ഉപയോക്താക്കൾക്ക് നേരിട്ട് ഫീഡ്‌ബാക്ക് ഫ്ലോ ആരംഭിക്കാൻ കഴിയും. ഈ കമാൻഡ്:

- ഉടൻ തന്നെ ഫീഡ്‌ബാക്ക് സമ്മത മോഡൽ തുറക്കുന്നു.
- നിലവിലെ സംഭാഷണത്തെക്കുറിച്ചുള്ള സന്ദർഭം വിവരണ ഫീൽഡിൽ മുൻകൂട്ടി പൂരിപ്പിക്കുന്നു.
- സമർപ്പിക്കുന്നതിന് മുമ്പ് കൂടുതൽ വിവരങ്ങൾ ചേർക്കാൻ ഉപയോക്താവിനെ അനുവദിക്കുന്നു.

എല്ലാ ചാറ്റ് മോഡുകളിലും (`inline`, `floating widget`, `full-screen`) `/report-issue` കമാൻഡ് ലഭ്യമാണ്.

## AI-സഹായത്തോടെയുള്ള ട്രൈയാജ് (AI-Assisted Triage) {#ai-assisted-triage}

സമർപ്പിക്കപ്പെടുന്ന ഫീഡ്‌ബാക്ക് ഒരു AI ട്രൈയാജ് ലെയറിലേക്ക് റൂട്ട് ചെയ്യപ്പെടുന്നു. ഇത്:

- റിപ്പോർട്ട് തരംതിരിക്കുന്നു (വസ്തുതാപരമായ പിശക്, സഹായകരമല്ലാത്ത മറുപടി, അനുചിതമായ ഉള്ളടക്കം, തുടങ്ങിയവ).
- ബന്ധപ്പെട്ട സംഭാഷണ മെറ്റാഡാറ്റ ഘടിപ്പിക്കുന്നു.
- ട്രൈയാജ് സംഗ്രഹം കോൺഫിഗർ ചെയ്ത **Feedback Endpoint** ലേക്ക് കൈമാറുന്നു (ഇതിനായി [Settings > Advanced](#settings-advanced) കാണുക).

ഇത് ഏറ്റവും നിർണ്ണായകമായ പ്രശ്നങ്ങൾ ആദ്യം കൊണ്ടുവരുന്നതിലൂടെ മാനുവൽ റിവ്യൂ സമയം കുറയ്ക്കുന്നു.

## Settings > Advanced {#settings-advanced}

ഫീഡ്‌ബാക്ക് ബാക്ക്എൻഡ് പ്രവർത്തനക്ഷമമാക്കാൻ, **Gratis AI Agent → Settings → Advanced** എന്നതിന് കീഴിൽ താഴെ പറയുന്ന ഫീൽഡുകൾ കോൺഫിഗർ ചെയ്യുക:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | ഫീഡ്‌ബാക്ക് പേലോഡുകൾ (JSON) സഹിതം POST റിക്വസ്റ്റുകൾ സ്വീകരിക്കുന്ന URL. |
| **Feedback API Key** | ഫീഡ്‌ബാക്ക് സമർപ്പിക്കുമ്പോൾ പ്രാമാണീകരിക്കുന്നതിനായി `Authorization` ഹെഡറിൽ അയയ്‌ക്കേണ്ട Bearer token. |

ഫീഡ്‌ബാക്ക് ശേഖരണം പ്രവർത്തനരഹിതമാക്കാൻ രണ്ട് ഫീൽഡുകളും ഒഴിച്ചിടുക. വ്യക്തിഗത ഫീഡ്‌ബാക്ക് ബട്ടണുകൾ ഉപയോക്താക്കൾക്ക് കാണത്തക്കവിധം തുടരും, പക്ഷേ സമർപ്പിക്കലുകൾ കൈമാറ്റം ചെയ്യപ്പെടില്ല.

:::tip
ഫീഡ്‌ബാക്ക് എൻഡ്‌പോയിന്റ് കുറഞ്ഞത് `message_id`, `conversation_id`, `feedback_text`, `triage_category` എന്നീ ഫീൽഡുകൾ അടങ്ങിയ ഒരു JSON ബോഡി സ്വീകരിക്കണം. പ്രതീക്ഷിക്കുന്ന സ്കീമയ്ക്കായി നിങ്ങളുടെ എൻഡ്‌പോയിന്റ് പ്രൊവൈഡറുടെ ഡോക്യുമെന്റേഷൻ കാണുക.
:::
