# Gratis AI Agent സജ്ജീകരണങ്ങൾ (Settings)

Gratis AI Agent-ലെ **Settings → Advanced** സ്ക്രീനിൽ, v1.5.0-ൽ അവതരിപ്പിച്ച ബാക്കെൻഡ് സംയോജനங்களுக்கான (backend integrations) അഡ്മിനിസ്ട്രേറ്റർ തലത്തിലുള്ള കോൺഫിഗറേഷൻ ലഭ്യമാണ്. ഈ പേജ് **Feedback Endpoint** ഫീൽഡുകളെക്കുറിച്ചും അവയുടെ പ്രതീക്ഷിക്കുന്ന ഫോർമാറ്റിനെക്കുറിച്ചും വിശദീകരിക്കുന്നു.

## സജ്ജീകരണങ്ങൾ ആക്‌സസ് ചെയ്യുന്നത് എങ്ങനെ

1. WordPress അഡ്മിനിൽ, **Gratis AI Agent → Settings** എന്നതിലേക്ക് പോകുക.
2. **Advanced** ടാബിൽ ക്ലിക്ക് ചെയ്യുക.

## Feedback Endpoint കോൺഫിഗറേഷൻ

ഒരു യൂസർ തള്ളവിരൽ താഴേക്ക് (thumbs-down button), ഓട്ടോ-പ്രോംപ്റ്റ് ബാനർ, അല്ലെങ്കിൽ `/report-issue` കമാൻഡ് എന്നിവ വഴി ഫീഡ്ബാക്ക് സമർപ്പിക്കുമ്പോൾ, AI ഏജന്റ് ഈ ഫീഡ്ബാക്ക് പോസ്റ്റ് (POST) റിക്വസ്റ്റുകളായി feedback endpoint-ലേക്ക് അയയ്ക്കുന്നു.

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | JSON ബോഡിയോടെ HTTP POST റിക്വസ്റ്റുകളായി ഫീഡ്ബാക്ക് സമർപ്പിക്കുന്നത് സ്വീകരിക്കുന്ന URL. |
| **Feedback API Key** | ഓരോ ഫീഡ്ബാക്ക് റിക്വസ്റ്റിലും `Authorization` ഹെഡറിൽ അയയ്‌ക്കേണ്ട ഒരു ബിയർയർ ടോക്കൺ (bearer token). നിങ്ങളുടെ endpoint-ന് ഓതന്റിക്കേഷൻ ആവശ്യമില്ലെങ്കിൽ ഇത് ഒഴിച്ചിടുക. |

### പ്രതീക്ഷിക്കുന്ന JSON Payload

നിങ്ങളുടെ ഫീഡ്ബാക്ക് endpoint-ന് കുറഞ്ഞത് താഴെ പറയുന്ന ഫീൽഡുകൾ അടങ്ങിയ ഒരു JSON ബോഡി സ്വീകരിക്കണം:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

സംഭാഷണ സന്ദർഭമനുസരിച്ച്, പേലോഡിൽ (payload) മറ്റ് ഫീൽഡുകൾ ഉണ്ടാകാം.

### `triage_category` മൂല്യങ്ങൾ

പേലോഡ് ഫോർവേഡ് ചെയ്യുന്നതിന് മുമ്പ്, AI ടറിയേജ് ലെയർ (AI triage layer) `triage_category`-ക്ക് താഴെ പറയുന്ന മൂല്യങ്ങളിൽ ഒന്ന് നൽകുന്നു:

| Value | Meaning |
|---|---|
| `factual_error` | അസിസ്റ്റന്റ് തെറ്റായ വസ്തുതാപരമായ വിവരങ്ങൾ നൽകി. |
| `unhelpful_answer` | മറുപടി സാങ്കേതികമായി ശരിയാണെങ്കിലും ഉപയോഗപ്രദമല്ല. |
| `inappropriate_content` | മറുപടിയിൽ യൂസർമാർ കാണാൻ പാടില്ലാത്ത ഉള്ളടക്കം അടങ്ങിയിരിക്കുന്നു. |
| `other` | ഫീഡ്ബാക്ക് അറിയപ്പെടുന്ന വിഭാഗവുമായി പൊരുത്തപ്പെടുന്നില്ല. |

### Authentication (ആധികാരികത)

നിങ്ങളുടെ endpoint-ന് ഓതന്റിക്കേഷൻ ആവശ്യമാണെങ്കിൽ, **Feedback API Key** ഫീൽഡിൽ നിങ്ങളുടെ ബിയർയർ ടോക്കൺ നൽകുക. ഏജന്റ് അയയ്ക്കുന്നത് ഇതാണ്:

```
Authorization: Bearer <your-api-key>
```

**Feedback API Key** ഫീൽഡ് ശൂന്യമാണെങ്കിൽ, യാതൊരു `Authorization` ഹെഡറും അയക്കില്ല.

### ഫീഡ്ബാക്ക് ശേഖരണം പ്രവർത്തനരഹിതമാക്കൽ

**Feedback Endpoint URL** ഉം **Feedback API Key** ഉം എന്നീ രണ്ട് ഫീൽഡുകളും ശൂന്യമായി വിടുക. തള്ളവിരൽ താഴേക്ക് ബട്ടണും ഫീഡ്ബാക്ക് UI-യും യൂസർമാർക്ക് കാണാൻ കഴിയും, പക്ഷേ സമർപ്പിക്കലുകൾ ഒരു ബാഹ്യ സേവനത്തിലേക്ക് ഫോർവേഡ് ചെയ്യില്ല.

## Brave Search API Key

**Advanced** ടാബിൽ, **Brave Search API Key** ഫീൽഡ് [Internet Search](../configuration/internet-search) കഴിവ് പ്രവർത്തനക്ഷമമാക്കുന്നു.

| Field | Description |
|---|---|
| **Brave Search API Key** | Brave Search developer dashboard-ൽ നിന്നുള്ള നിങ്ങളുടെ API key. AI അസിസ്റ്റന്റിൽ ഇന്റർനെറ്റ് സെർച്ച് പ്രവർത്തനക്ഷമമാക്കാൻ ഇത് ആവശ്യമാണ്. |

ഈ ഫീൽഡ് ലേബലിൽ Brave Search API സൈൻ-അപ്പ് പേജിലേക്കുള്ള ക്ലിക്കബിൾ ലിങ്ക് ഉൾപ്പെടുന്നു. ഇന്റർനെറ്റ് സെർച്ച് പ്രവർത്തനരഹിതമാക്കാൻ ഇത് ശൂന്യമായി വിടുക.

ഈ ഫീച്ചറിനെക്കുറിച്ചുള്ള end-user ഡോക്യുമെന്റേഷനായി [Internet Search](../configuration/internet-search) കാണുക.

## Feature Flags (പ്രത്യേക സവിശേഷത ഫ്ലാഗുകൾ)

v1.9.0-ൽ അവതരിപ്പിച്ച **Settings → Feature Flags** ടാബ് ഓപ്ഷണൽ പ്രവർത്തനക്ഷമതകൾക്കുള്ള ടോഗിൾ സ്വിച്ചുകൾ നൽകുന്നു. ഓരോ ഫ്ലാഗും നെറ്റ്‌വർക്ക് മുഴുവൻ പ്രവർത്തനക്ഷമമാക്കുകയോ പ്രവർത്തനരഹിതമാക്കുകയോ ചെയ്യും; നിലവിൽ സൈറ്റ് അടിസ്ഥാനത്തിലുള്ള മാറ്റങ്ങൾ വരുത്താൻ കഴിയില്ല.

### Access Control Flags (പ്രവേശന നിയന്ത്രണ ഫ്ലാഗുകൾ)

| Flag | Default | Description |
|---|---|---|
| **Restrict to Administrators** | Off | ഇത് പ്രവർത്തനക്ഷമമാക്കിയാൽ, `administrator` റോൾ ഉള്ള യൂസർമാർക്ക് മാത്രമേ AI Agent chat panel തുറക്കാൻ കഴിയൂ. മറ്റ് എല്ലാ റോൾ ഉള്ളവർക്കും പകരം "Contact your administrator" എന്ന സന്ദേശം കാണിക്കും. |
| **Restrict to Network Admins** | Off | ഒരു മൾട്ടിസൈറ്റ് നെറ്റ്‌വർക്കിൽ ഇത് പ്രവർത്തനക്ഷമമാക്കിയാൽ, സൂപ്പർ അഡ്മിൻസ്ക്ക് മാത്രമേ ഏജന്റ് ഉപയോഗിക്കാൻ കഴിയൂ. വ്യക്തിഗത സൈറ്റ് അഡ്മിൻമാർക്ക് ഇത് തടസ്സമാകും. രണ്ടും പ്രവർത്തനക്ഷമമാക്കിയാൽ ഇത് "Restrict to Administrators"-നേക്കാൾ മുൻഗണന നൽകും. |
| **Allow Subscriber Access** | Off | ഇത് പ്രവർത്തനക്ഷമമാക്കിയാൽ, `subscriber` റോൾ ഉള്ള യൂസർമാർക്ക് ചാറ്റ് ഇന്റർഫേസ് ഉപയോഗിക്കാൻ കഴിയും, പക്ഷേ അവർക്ക് വായിക്കാൻ മാത്രമേ കഴിയൂ (പോസ്റ്റ് സൃഷ്ടിക്കാനോ സജ്ജീകരണങ്ങൾ മാറ്റാനോ കഴിയില്ല). |
| **Disable for Non-Members** | Off | Ultimate Multisite membership സ്റ്റാറ്റസ്സുമായി സംയോജിക്കുന്നു. ഇത് പ്രവർത്തനക്ഷമമാക്കിയാൽ, സജീവമായ അംഗത്വം ഇല്ലാത്ത സൈറ്റുകൾക്ക് ചാറ്റ് മറഞ്ഞിരിക്കും. |

### Branding Flags (ബ്രാൻഡിംഗ് ഫ്ലാഗുകൾ)

| Flag | Default | Description |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | ചാറ്റ് വിഡ്ജറ്റിന്റെ അടിയിൽ കാണിക്കുന്ന ബ്രാൻഡിംഗ് ആട്രിബ്യൂഷൻ ലൈൻ നീക്കംചെയ്യുന്നു. വൈറ്റ്-ലേബൽ ഡിപ്ലോയ്‌മെന്റുകൾക്ക് ശുപാർശ ചെയ്യുന്നത്. |
| **Custom Agent Name** | *(blank)* | ചാറ്റ് ഹെഡറിലെയും അഡ്മിൻ മെനูกിലെയും ഡിഫോൾട്ട് "Gratis AI Agent" ലേബലിന് പകരം നിങ്ങളുടെ സ്വന്തം ഉൽപ്പന്നത്തിന്റെ പേര് ഉപയോഗിക്കുന്നു. ഡിഫോൾട്ട് ഉപയോഗിക്കാൻ ഇത് ശൂന്യമായി വിടുക. |
| **Hide Agent Picker** | Off | ഇത് പ്രവർത്തനക്ഷമമാക്കിയാൽ, യൂസർമാർക്ക് അഞ്ച് ബിൽറ്റ്-ഇൻ ഏജന്റുമാർക്കിടയിൽ മാറാൻ കഴിയില്ല. നിലവിലെ ഏജന്റ് Settings → General-ൽ ഡിഫോൾട്ടായി സജ്ജീകരിച്ചതിന് തറപ്പായിരിക്കും. |
| **Use Site Icon as Chat Avatar** | Off | ചാറ്റ് വിഡ്ജറ്റ് ഹെഡറിലെ ഡിഫോൾട്ട് AI ഐക്കണിന് പകരം WordPress സൈറ്റ് ഐക്കൺ (Appearance → Customize → Site Identity-യിൽ സജ്ജമാക്കിയത്) ഉപയോഗിക്കുന്നു. |

### Changes Applying ചെയ്യുന്നത് എങ്ങനെ

ഏതെങ്കിലും ഫ്ലാഗ് ടോഗിൾ ചെയ്ത ശേഷം **Save Settings** ക്ലിക്ക് ചെയ്യുക. മാറ്റങ്ങൾ ഉടൻ പ്രാബല്യത്തിൽ വരും — കാഷ് ഫ്ലഷ് ചെയ്യാനോ പ്ലഗിൻ റീആക്ടിവേറ്റ് ചെയ്യാനോ ആവശ്യമില്ല.
