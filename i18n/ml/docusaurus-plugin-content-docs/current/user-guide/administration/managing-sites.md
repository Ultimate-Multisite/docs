# സൈറ്റുകൾ മാനേജ് ചെയ്യൽ

നിങ്ങളുടെ WaaS ബിസിനസിന്റെ അടിസ്ഥാനം സൈറ്റുകൾ (അല്ലെങ്കിൽ subsites) ആണ്. Ultimate Multisite-ൽ 3 തരം സൈറ്റുകൾ ഉണ്ട്:

- **ഉപഭോക്താവിന്റെ ഉടമസ്ഥതയിലുള്ളവ** — നിർദ്ദിഷ്ട ഉപഭോക്താക്കളുടെ സൈറ്റുകൾ
- **Site Templates** — ഉപഭോക്താക്കൾക്ക് തുടക്കമായി തിരഞ്ഞെടുക്കാവുന്ന മുൻകൂട്ടി നിർവചിച്ച സൈറ്റുകൾ
- **Main Site** — നിങ്ങളുടെ പ്രധാന നെറ്റ്‌വർക്ക് സൈറ്റ്

## സൈറ്റുകൾ കാണൽ

നിങ്ങളുടെ നെറ്റ്‌വർക്കിലെ എല്ലാ subsites-ഉം കാണാൻ **Ultimate Multisite → Sites** എന്നതിലേക്ക് പോകുക. ഓരോ സൈറ്റും തരം അനുസരിച്ച് ലേബൽ ചെയ്തിരിക്കും, കൂടാതെ All Sites, Customer-Owned, Templates, അല്ലെങ്കിൽ Pending എന്നിവ അനുസരിച്ച് ഫിൽട്ടർ ചെയ്യാനും കഴിയും.

![സൈറ്റുകളുടെ ലിസ്റ്റ് പേജ്](/img/admin/sites-list.png)

## പുതിയ സൈറ്റ് ചേർക്കൽ

പുതിയ സൈറ്റ് സൃഷ്ടിക്കാൻ **Add Site** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക. നിങ്ങൾ നൽകേണ്ട വിവരങ്ങൾ:

- **Site Title** — പുതിയ സൈറ്റിന്റെ പേര്
- **Site URL** — സൈറ്റിന്റെ domain/path
- **Site Type** — ഇത് customer site ആണോ, template ആണോ, അതോ സാധാരണ സൈറ്റ് ആണോ എന്ന്

**Copy Site** ഓപ്ഷൻ ഉപയോഗിച്ച് നിലവിലുള്ള site template അടിസ്ഥാനമാക്കി പുതിയ സൈറ്റ് സൃഷ്ടിക്കാം. ഇത് പ്രവർത്തനക്ഷമമാക്കുമ്പോൾ, ഏത് template തുടക്കമായി ഉപയോഗിക്കണമെന്ന് തിരഞ്ഞെടുക്കാം. മീഡിയ ഫയലുകൾ ഉൾപ്പെടുത്താൻ **Copy Media on Duplication** പ്രവർത്തനക്ഷമമാക്കിയിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക.

## നിലവിലുള്ള സൈറ്റ് മാനേജ് ചെയ്യൽ

**Edit Site** പേജ് തുറക്കാൻ ഏതെങ്കിലും സൈറ്റിൽ **Manage** ക്ലിക്ക് ചെയ്യുക. ഇവിടെ നിങ്ങൾ കാണുന്നത്:

### അടിസ്ഥാന വിവരങ്ങൾ

സൈറ്റിന്റെ പേര്, തരം, site ID, വിവരണം എന്നിവ. mapped domain, ബന്ധപ്പെട്ട membership, സൈറ്റിന്റെ ഉടമസ്ഥതയുള്ള customer account എന്നിവയും നിങ്ങൾ കാണും.

### സൈറ്റ് ഓപ്ഷനുകൾ

സൈറ്റിന്റെ കഴിവുകളും പരിധികളും ക്രമീകരിക്കുക:

- **Visit limits** — സൈറ്റ് സന്ദർശനങ്ങളുടെ പരമാവധി എണ്ണം
- **User account limits** — ഓരോ user role-നും ഉള്ള പരിധികൾ
- **Disk space** — സൈറ്റിനുള്ള സ്റ്റോറേജ് ക്വോട്ട
- **Custom domain** — ഈ സൈറ്റിന് domain mapping പ്രവർത്തനക്ഷമമാക്കുക
- **Plugin and theme visibility** — ഏതൊക്കെ plugins-ഉം themes-ഉം കാണാം, മറയ്ക്കാം, അല്ലെങ്കിൽ മുൻകൂട്ടി activate ചെയ്യാം എന്ന് നിയന്ത്രിക്കുക

സ്ഥിരസ്ഥിതിയായി, membership ലെവലിൽ സജ്ജീകരിച്ച പരിധികൾ സൈറ്റുകൾ പിന്തുടരുന്നു. സൈറ്റ് ലെവലിൽ പരിധികൾ സജ്ജീകരിക്കുന്നത് membership സെറ്റിംഗുകളെ മറികടക്കും.

### ബന്ധങ്ങൾ

സൈറ്റ് ഓപ്ഷനുകൾക്ക് താഴെ നിങ്ങൾ ഇവയെക്കുറിച്ചുള്ള വിവരങ്ങൾ കാണും:

- സൈറ്റുമായി ബന്ധപ്പെട്ട **Mapped domains**
- സൈറ്റ് ഉൾപ്പെടുന്ന **Membership**
- സൈറ്റുമായി ബന്ധിപ്പിച്ച **Customer account**

### വലത് Sidebar

വലതുവശത്ത് നിങ്ങൾക്ക്:

- Toggle ബട്ടൺ ഉപയോഗിച്ച് **സൈറ്റ് പ്രവർത്തനക്ഷമമാക്കുക/നിഷ്ക്രിയമാക്കുക**
- **സൈറ്റ് തരം മാറ്റുക** അല്ലെങ്കിൽ ഉടമസ്ഥത പുനഃക്രമീകരിക്കുക
- **സൈറ്റ് ഇമേജ്/thumbnail സജ്ജീകരിക്കുക** (site templates-ന് frontend-ൽ പ്രദർശിപ്പിക്കുന്നത്)
- **സൈറ്റ് ശാശ്വതമായി ഡിലീറ്റ് ചെയ്യുക**

:::warning
സൈറ്റ് ഡിലീറ്റ് ചെയ്യുന്നത് പഴയപടിയാക്കാനാവില്ല. സൈറ്റും അതിന്റെ എല്ലാ ഉള്ളടക്കവും ശാശ്വതമായി നീക്കം ചെയ്യപ്പെടും.
:::
