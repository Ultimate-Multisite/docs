# പരമാധികാര ടെനന്റ് പ്രവർത്തനങ്ങൾ (Sovereign Tenant Operations)

Ultimate Multisite: Multi-Tenancy 1.2.0 പരമാധികാര ടെനന്റുകൾക്ക് പ്രവർത്തനപരമായ ടൂളുകൾ കൂട്ടിച്ചേർക്കുന്നു: സ്വന്തമായി ഡാറ്റാബേസ്, ഫയൽ സിസ്റ്റം റൂട്ട്, റൂട്ടിംഗ് കോൺടെക്സ്റ്റ് എന്നിവ ഉപയോഗിച്ച് പ്രവർത്തിക്കുന്ന സബ്സൈറ്റുകൾ, എന്നാൽ നെറ്റ്‌വർക്ക് അഡ്മിനിറ്റിന് അവ കാണാൻ സാധിക്കും.

ഇതുപോലുള്ള സാഹചര്യങ്ങളിൽ ഒറ്റപ്പെട്ട ഉപഭോക്തൃ സൈറ്റുകൾ കൈകാര്യം ചെയ്യുമ്പോഴോ, വിദൂര സൈറ്റുകളിലേക്കുള്ള കൈമാറ്റങ്ങളോ (remote-site handoffs), അല്ലെങ്കിൽ ഒരു സാധാരണ സബ്സൈറ്റിനെ പരമാധികാര ഇൻഫ്രാസ്ട്രക്ചറിലേക്ക് മാറ്റുന്നതിനോ ഇത് ഉപയോഗിക്കുക.

## അഡ്മിനിറ്റേഴ്സിന് എന്തൊക്കെ മാറ്റങ്ങളാണ് വരുന്നത്

- **സ്റ്റേറ്റ്ലസ് ടെനന്റ് ഓട്ടോലോഗിൻ (Stateless tenant autologin)** — ദീർഘകാല സെഷൻ സ്റ്റേറ്റിനെ ആശ്രയിക്കാതെ നെറ്റ്‌വർക്ക് അഡ്മിനിറ്റർമാർക്ക് ഒരു പരമാധികാര ടെനന്റിനെ സന്ദർശിക്കാൻ സാധിക്കും. SSO ടോക്കൺ ഉദ്ദേശ്യത്തിന് അനുയോജ്യമായ രീതിയിൽ (purpose-scoped), ഉറവിടം നിശ്ചയിച്ചുകൊണ്ട് (origin-pinned), പുനരാവിഷ്കരണ സംരക്ഷിതമായി (replay-protected), കുറഞ്ഞ കാലാവധിയോടെ (short expiry window) പരിമിതപ്പെടുത്തിയിരിക്കുന്നു.
- **പരമാധികാര-അറിവുള്ള റൂട്ടിംഗ് (Sovereign-aware routing)** — പഴയ ഒറ്റപ്പെട്ട നെറ്റ്‌വർക്കുകളും പരമാധികാര ടെനന്റുകളും ഒരേ സൈറ്റ് റൂട്ടർ പാത്ത് വഴി പ്രവർത്തിക്കുന്നു, ഇത് പഴയതും പുതിയതുമായ ഒറ്റപ്പെട്ട ഇൻസ്റ്റാളേഷനുകൾ തമ്മിലുള്ള ബൂട്ട്‌സ്‌ട്രോപ്പ് വ്യത്യാസങ്ങൾ കുറയ്ക്കുന്നു.
- **പരിശോധിത മൈഗ്രേഷൻ സ്റ്റേറ്റ് (Verified migration state)** — ഒരു ടെനന്റ് പൂർണ്ണമായി കണക്കാക്കുന്നതിന് മുമ്പ്, മൈഗ്രേഷൻ പരിശോധന ഉപയോക്തൃ പ്രൊവിഷനിംഗ്, ഡാറ്റാബേസ് റൈറ്റർ അനുമതികൾ, ക്യൂ ഡ്രെയിൻ സ്റ്റാറ്റസ്, പഴയ ടേബിളുകളുടെ അഭാവം എന്നിവ പരിശോധിക്കുന്നു.
- **സുരക്ഷിതമായ ഡിറ്ററോഡ് (Safer teardown)** — പരമാധികാര ടെനന്റ് ഡിറ്ററോഡ് ചെയ്യുമ്പോൾ, ഡിലീറ്റ് ചെയ്ത ടെനന്റുകൾക്ക് പഴയ ഡാറ്റാബേസ് ആക്സസ് ബാക്കിവെക്കാതിരിക്കാൻ ടെനന്റ് ക്രെഡൻഷ്യലുകൾ വൃത്തിയായി നീക്കം ചെയ്യുന്നു.

## ഒരു പരമാധികാര ടെനന്റിനെ സന്ദർശിക്കുന്നത്

1. **Network Admin > Ultimate Multisite > Sites** തുറക്കുക.
2. പരമാധികാര ടെനന്റിനെ തിരഞ്ഞെടുക്കുക.
3. പാസ്‌വേഡുകൾ കോപ്പി ചെയ്യുകയോ താൽക്കാലിക അഡ്മിൻ അക്കൗണ്ടുകൾ സൃഷ്ടിക്കുകയോ ചെയ്യുന്നതിന് പകരം ലഭ്യമാണെങ്കിൽ **Visit (SSO)** ഉപയോഗിക്കുക.

ഈ വിസിറ്റ് ഫ്ലോ ആ ടെനന്റിന് ഒരു ഹ്രസ്വകാല ലോഗിൻ ടോക്കൺ ഉണ്ടാക്കുകയും ടെനന്റ് ഓഡിറ്റ് ട്രെയ്‌ലിൽ SSO ഇവന്റ് രേഖപ്പെടുത്തുകയും ചെയ്യുന്നു. ബട്ടൺ പരാജയപ്പെടുകയാണെങ്കിൽ, ടെനന്റ് ഡൊമെയ്ൻ പ്രതീക്ഷിക്കുന്ന ഇൻസ്റ്റാളേഷനുമായി ബന്ധപ്പെട്ടിട്ടുണ്ടെന്നും ടെനന്റിന് നെറ്റ്‌വർക്ക്-സൈഡ് SSO എൻഡ്പോയിന്റിലേക്ക് എത്താൻ കഴിയുന്നുണ്ടെന്നും പരിശോധിക്കുക.

## വിദൂര സൈറ്റ് പ്രവർത്തനങ്ങൾ ചെക്ക്‌ലിസ്റ്റ് (Remote-site operations checklist)

ഒരു പരമാധികാര അല്ലെങ്കിൽ വിദൂര ടെനന്റ് മാറ്റുന്നതിന് മുമ്പ്, താഴെ പറയുന്നവ ഉറപ്പുവരുത്തുക:

## പരമാധികാര ഉള്ള ടെനന്റുകൾ നീക്കം ചെയ്യൽ (Deleting sovereign tenants)

ഒരു പരമാധികാരമുള്ള ടെനന്റ് ഡിലീറ്റ് ചെയ്യുന്നത് ഡാറ്റ നഷ്ടപ്പെടാൻ സാധ്യതയുള്ള ഒരു പ്രക്രിയയാണ്. ആദ്യം ബാക്കപ്പ് എടുക്കുന്നതിനും എക്സ്പോർട്ട് ചെയ്യുന്നതിനും അനുമതി ഉണ്ടെന്ന് ഉറപ്പാക്കിയ ശേഷം, സൈറ്റ് മാനേജ്‌മെന്റ് സ്ക്രീനിൽ നിന്ന് അത് നീക്കം ചെയ്യുക. 1.2.0-ലെ ടെർഡൗൺ ഫ്ലോ (teardown flow) ക്ലീനിംഗിന്റെ ഭാഗമായി ടെനന്റ് ഡാറ്റാബേസ് ക്രെഡൻഷ്യലുകൾ നീക്കം ചെയ്യും, എങ്കിലും ബാഹ്യ ഹോസ്റ്റിംഗ് പാനലുകൾ ഉപയോഗിക്കുമ്പോൾ ഹോസ്റ്റ് ലെവൽ ഡാറ്റാബേസ് യൂസർമാരും ഫോൾഡറുകളും മാഞ്ഞുപോയി എന്ന് അഡ്മിനിസ്ട്രേറ്റർ ഉറപ്പുവരുത്തേണ്ടതുണ്ട്.

:::warning
മൈഗ്രേഷൻ വെരിഫിക്കേഷൻ (migration verification) പ്രവർത്തിക്കുന്നതിനിടയിലോ അല്ലെങ്കിൽ അസിങ്ക് പുഷ് ജോലികൾ (async push jobs) ക്യൂവിൽ നിൽക്കുമ്പോഴോ പരമാധികാരമുള്ള ടെനന്റ് നീക്കം ചെയ്യരുത്. ടെർഡൗൺ പ്രക്രിയ പൂർത്തിയാകുന്നത് വരെ കാത്തിരിക്കുക, കാരണം അത് നിലവിലുള്ള ജോലികൾക്ക് ആവശ്യമായ ക്രെഡൻഷ്യലുകൾ നീക്കം ചെയ്യാതിരിക്കാൻ സഹായിക്കും.
:::
