# കഴിവ് ദൃശ്യതാ നിയന്ത്രണം (Ability Visibility)

Superdav AI Agent v1.12.0, **കഴിവ് ദൃശ്യതാ നിയന്ത്രണം** (Ability Visibility) എന്ന സംവിധാനം അവതരിപ്പിക്കുന്നു. ഇത് ഏത് കഴിവുകൾ എവിടെയെല്ലാം ലഭ്യമാകുമെന്ന് നിയന്ത്രിക്കുന്നു. REST APIs, MCP സെർവറുകൾ, WooCommerce സംയോജിത സംവിധാനങ്ങൾ, മറ്റ് ഇന്റർഫേസുകൾ എന്നിവ വഴി ഏത് ഏജന്റ് കഴിവുകൾ ലഭ്യമാക്കണം എന്ന് അഡ്മിനിസ്ട്രേറ്റർമാർക്ക് ഇത് ക്രമീകരിക്കാൻ സാധിക്കുന്നു.

## കഴിവ് ദൃശ്യതാ നിയന്ത്രണം എന്നാൽ എന്ത്? {#what-is-ability-visibility}

കഴിവ് ദൃശ്യതാ നിയന്ത്രണം എന്നത് ഒരു പെർമിഷൻ സിസ്റ്റമാണ്. ഇത് താഴെ പറയുന്ന കാര്യങ്ങൾ നിയന്ത്രിക്കുന്നു:

- ഏത് **കഴിവുകൾ** ഏജന്റുമാർക്ക് ലഭ്യമാണ്
- അവ **എവിടെയെല്ലാം ലഭ്യമാക്കുന്നു** (REST API, MCP, WooCommerce, തുടങ്ങിയവ)
- **ആർക്ക് പ്രവേശിക്കാൻ കഴിയും** (പങ്കാളി അനുവദ ലിസ്റ്റുകൾ വഴി)
- അവ എങ്ങനെ **വർഗ്ഗീകരിക്കുന്നു** (അംഗീകരിച്ചു vs. വർഗ്ഗീകരിക്കാത്തത്)

ഓരോ കഴിവുകൾക്കും ഒരു ദൃശ്യതാ നിലവാരം (visibility level) ഉണ്ട്. ഇത് വിവിധ സംവിധാനങ്ങളിലുടനീളം അതിന്റെ ലഭ്യത നിർണ്ണയിക്കുന്നു.

## ദൃശ്യതാ നിലവാരങ്ങൾ (Visibility Levels) {#visibility-levels}

### പൊതു (Public) {#public}

**പൊതു കഴിവുകൾ** എല്ലാ സ്ഥലങ്ങളിലും ലഭ്യമാണ്:

- REST API എൻഡ്‌പോയിന്റുകൾ
- MCP സെർവറുകൾ
- WooCommerce സംയോജിത സംവിധാനങ്ങൾ
- ചാറ്റ് ഇന്റർഫേസുകൾ
- എല്ലാ യൂസർ റോളുകൾക്കും (അനുയോജ്യമായ പെർമിഷനുകളോടെ)

ഉദാഹരണം: `scaffold-block-theme`, `activate-theme`, `send-email`

### ആന്തരിക (Internal) {#internal}

**ആന്തരിക കഴിവുകൾ** നിങ്ങളുടെ WordPress ഇൻസ്റ്റാളേഷനുള്ളിൽ മാത്രമേ ലഭ്യമാകൂ:

- ചാറ്റ് ഇന്റർഫേസുകൾ
- അഡ്മിൻ പാനലുകൾ
- ലോഗിൻ ചെയ്ത യൂസർമാർക്ക് മാത്രം
- REST API വഴിയോ ബാഹ്യ സംയോജിത സംവിധാനങ്ങളിലൂടെയോ ലഭ്യമാവില്ല

ഉദാഹരണം: `manage-settings`, `view-analytics`, `export-data`

### പങ്കാളി (Partner) {#partner}

**പങ്കാളി കഴിവുകൾ** അനുവദ ലിസ്റ്റിൽ ഉൾപ്പെടുത്തിയ പങ്കാളികൾക്ക് (whitelisted partners) മാത്രമേ ലഭ്യമാകൂ:

- പങ്കാളി അനുവദ ലിസ്റ്റിൽ വ്യക്തമായി ഉൾപ്പെടുത്തണം
- പങ്കാളി ക്രെഡൻഷ്യലുകൾ ഉപയോഗിച്ച് REST API വഴി ലഭ്യമാകും
- പങ്കാളി ഓതന്റിക്കേഷൻ ഉപയോഗിച്ച് MCP വഴി ലഭ്യമാകും
- സാധാരണ യൂസർമാർക്ക് ലഭ്യമാകില്ല

ഉദാഹരണം: `bulk-import-users`, `modify-billing`, `access-analytics`

### പ്രവർത്തനരഹിതം (Disabled) {#disabled}

**പ്രവർത്തനരഹിതമായ കഴിവുകൾ** എവിടെയും ലഭ്യമാകില്ല:

- ഒരു സംവിധാനത്തിലൂടെയും ലഭ്യമാവില്ല
- ചാറ്റിൽ ലഭ്യമല്ല
- അഡ്മിൻ പാനലുകളിൽ ലഭ്യമല്ല
- കാലഹരണപ്പെട്ടതോ പരീക്ഷണാടിസ്ഥാനത്തിലുള്ളതോ ആയ കഴിവുകൾക്ക് ഇത് ഉപയോഗപ്രദമാണ്

## കഴിവ് ദൃശ്യതാ നിയന്ത്രണം കൈകാര്യം ചെയ്യൽ (Managing Ability Visibility) {#managing-ability-visibility}

### കഴിവ് ദൃശ്യതാ ക്രമീകരണങ്ങൾ ആക്‌സസ് ചെയ്യുന്നത് {#accessing-ability-visibility-settings}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** എന്നതിലേക്ക് പോകുക
2. **Abilities** ടാബിൽ ക്ലിക്ക് ചെയ്യുക
3. ഇൻസ്റ്റാൾ ചെയ്ത എല്ലാ കഴിവുകളുടെയും ഒരു ലിസ്റ്റ്, അവയുടെ ദൃശ്യതാ നിലവാരങ്ങൾ സഹിതം നിങ്ങൾക്ക് കാണാം

### കഴിവ് വിശദാംശങ്ങൾ കാണുന്നത് {#viewing-ability-details}

ഏതെങ്കിലും കഴിവിൽ ക്ലിക്ക് ചെയ്താൽ നിങ്ങൾക്ക് കാണാൻ കഴിയും:

- **Name**: കഴിവ് തിരിച്ചറിയാനുള്ള പേര് (identifier)
- **Description**: കഴിവ് എന്ത് ചെയ്യുന്നു
- **Current visibility**: പൊതു, ആന്തരിക, പങ്കാളി, അല്ലെങ്കിൽ പ്രവർത്തനരഹിതം
- **Partner allow-list**: ആർക്കൊക്കെ പ്രവേശിക്കാൻ കഴിയും (പങ്കാളി ദൃശ്യതാ നിലവാരമാണെങ്കിൽ)
- **Last updated**: ദൃശ്യതാ നിലവാരം അവസാനമായി എപ്പോൾ മാറ്റിയത്
- **Status**: അംഗീകരിച്ചു (Recognized) അതോ വർഗ്ഗീകരിക്കാത്തതോ (Unclassified)

### ദൃശ്യതാ നിലവാരങ്ങൾ മാറ്റുന്നത് {#changing-visibility-levels}

ഒരു കഴിവ് ദൃശ്യതാ നിലവാരം മാറ്റാൻ:

1. ലിസ്റ്റിലുള്ള കഴിവിൽ ക്ലിക്ക് ചെയ്യുക
2. ഡ്രോഡൗൺ മെനുവിൽ നിന്ന് പുതിയ ദൃശ്യതാ നിലവാരം തിരഞ്ഞെടുക്കുക
3. "Partner" തിരഞ്ഞെടുക്കുകയാണെങ്കിൽ, അനുവദ ലിസ്റ്റിൽ പങ്കാളി ഐഡന്റിഫയറുകൾ ചേർക്കുക
4. **Save** ക്ലിക്ക് ചെയ്യുക

ഉദാഹരണം:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### കൂട്ടായ പ്രവർത്തനങ്ങൾ (Bulk Operations) {#bulk-operations}

ഒന്നിലധികം കഴിവുകളുടെ ദൃശ്യതാ നിലവാരം മാറ്റാൻ:

1. കഴിവുകൾക്ക് അടുത്തുള്ള ബോക്സുകൾ ടിക്ക് ചെയ്യുക
2. കൂട്ടായ പ്രവർത്തന ഡ്രോഡൗൺ മെനുവിൽ നിന്ന് പുതിയ ദൃശ്യതാ നിലവാരം തിരഞ്ഞെടുക്കുക
3. **Apply** ക്ലിക്ക് ചെയ്യുക

## പങ്കാളി അനുവദ ലിസ്റ്റ് (Partner Allow-List) {#partner-allow-list}

**പങ്കാളി അനുവദ ലിസ്റ്റ്** എന്നത് ഏത് ബാഹ്യ പങ്കാളികൾക്ക് പങ്കാളി-ലെവൽ കഴിവുകൾ ആക്‌സസ് ചെയ്യാൻ കഴിയുമെന്ന് നിയന്ത്രിക്കുന്നു.

### പങ്കാളികളെ ചേർക്കുന്നത് {#adding-partners}

1. **Superdav AI Agent** → **Settings** → **Partners** എന്നതിലേക്ക് പോകുക
2. **Add Partner** ക്ലിക്ക് ചെയ്യുക
3. പങ്കാളി ഐഡന്റിഫയർ നൽകുക (സാധാരണയായി ഒരു API key അല്ലെങ്കിൽ സ്ഥാപന ഐഡി ആയിരിക്കും)
4. ഓപ്ഷണലായി ഒരു പങ്കാളി പേരും വിവരണവും ചേർക്കുക
5. **Save** ക്ലിക്ക് ചെയ്യുക

### പങ്കാളികൾക്ക് കഴിവുകൾ നൽകുന്നത് {#assigning-abilities-to-partners}

ഒരു പങ്കാളിയെ ചേർത്ത ശേഷം:

1. **Abilities** ടാബിലേക്ക് പോകുക
2. പങ്കാളി ദൃശ്യതാ നിലവാരമുള്ള ഒരു കഴിവ് കണ്ടെത്തുക
3. എഡിറ്റ് ചെയ്യാൻ ക്ലിക്ക് ചെയ്യുക
4. "Partner allow-list" വിഭാഗത്തിൽ, ആർക്കൊക്കെ പ്രവേശനം വേണമെന്ന് ബോക്സുകൾ ടിക്ക് ചെയ്യുക
5. **Save** ക്ലിക്ക് ചെയ്യുക

### പങ്കാളി ആക്‌സസ് പിൻവലിക്കുന്നത് (Revoking Partner Access) {#revoking-partner-access}

ഒരു പങ്കാളിയുടെ ആക്‌സസ് നീക്കം ചെയ്യാൻ:

1. കഴിവ് എഡിറ്റ് ചെയ്യുക
2. അനുവദ ലിസ്റ്റിൽ നിന്ന് പങ്കാളിയുടെ ബോക്സ് ടിക്ക് ചെയ്യാതിരിക്കുക
3. **Save** ക്ലിക്ക് ചെയ്യുക

ഈ കഴിവിലേക്ക് ആ പങ്കാളിക്ക് ഉടൻ തന്നെ പ്രവേശനം നഷ്ടമാകും.

## വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ (Unclassified Abilities) {#unclassified-abilities}

Superdav AI Agent തിരിച്ചറിയാത്ത ഒരു third-party കഴിവ് നിങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുമ്പോൾ, അത് **വർഗ്ഗീകരിക്കാത്തത്** (Unclassified) എന്ന് അടയാളപ്പെടുത്തും.

### വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾക്കുള്ള അഡ്മിൻ അറിയിപ്പ് {#admin-notice-for-unclassified-abilities}

നിങ്ങൾക്ക് ഒരു അഡ്മിൻ അറിയിപ്പ് കാണാം:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ അവലോകനം ചെയ്യുന്നത് {#reviewing-unclassified-abilities}

1. അറിയിപ്പിലെ **Review Abilities** ക്ലിക്ക് ചെയ്യുക
2. ഓരോ വർഗ്ഗീകരിക്കാത്ത കഴിവിലും നിങ്ങൾക്ക് കാണാം:
   - കഴിവ് പേരും വിവരണവും
   - സ്രോതസ്സ് (ഏത് പ്ലഗിൻ/ആഡ്‌ഓൺ ആണ് ഇത് ഇൻസ്റ്റാൾ ചെയ്തത്)
   - നിർദ്ദേശിക്കപ്പെട്ട ദൃശ്യതാ നിലവാരം
   - സുരക്ഷാ വിലയിരുത്തൽ

3. ഒരു ദൃശ്യതാ നിലവാരം തിരഞ്ഞെടുക്കുക:
   - **Public**: കഴിവില വിശ്വസിക്കുന്നു; എല്ലായിടത്തും ലഭ്യമാക്കുക
   - **Internal**: ആന്തരിക ഉപയോഗത്തിന് മാത്രം പരിമിതപ്പെടുത്തുക
   - **Partner**: പ്രത്യേക പങ്കാളികൾക്ക് മാത്രം പരിമിതപ്പെടുത്തുക
   - **Disabled**: ഈ കഴിവ് ഉപയോഗിക്കരുത്

4. നിങ്ങളുടെ തീരുമാനം സേവ് ചെയ്യാൻ **Classify** ക്ലിക്ക് ചെയ്യുക

### വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ എന്തിനാണ് വർഗ്ഗീകരിക്കുന്നത്? {#why-classify-unclassified-abilities}

കഴിവുകൾ വർഗ്ഗീകരിക്കുന്നത്:

- **സുരക്ഷ മെച്ചപ്പെടുത്തുന്നു**: എന്ത് കാര്യമാണ് പുറത്തറിയേണ്ടതെന്ന് നിങ്ങൾ വ്യക്തമായി അംഗീകരിക്കുന്നു
- **അപകടങ്ങൾ തടയുന്നു**: വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ ഡിഫോൾട്ടായി പ്രവർത്തനരഹിതമാണ്
- **പ്രത്യേകതകൾ പ്രവർത്തനക്ഷമമാക്കുന്നു**: വർഗ്ഗീകരിക്കുന്നതോടെ, കഴിവ് ലഭ്യമാകും
- **തീരുമാനങ്ങൾ രേഖപ്പെടുത്തുന്നു**: ഓഡിറ്റിംഗ് ആവശ്യങ്ങൾക്കായി നിങ്ങളുടെ തിരഞ്ഞെടുപ്പുകൾ രേഖപ്പെടുത്തുന്നു

## വിവിധ സംവിധാനങ്ങളിലെ ദൃശ്യതാ നിയന്ത്രണം (Visibility in Different Surfaces) {#visibility-in-different-surfaces}

### REST API {#rest-api}

**പൊതു കഴിവുകൾ** REST എൻഡ്‌പോയിന്റുകൾ വഴി ലഭ്യമാണ്:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**ആന്തരിക, പങ്കാളി കഴിവുകൾ** REST API വഴി ലഭ്യമാകില്ല.

### MCP സെർവറുകൾ {#mcp-servers}

**പൊതു കഴിവുകൾ** MCP വഴി ലഭ്യമാണ്:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**ആന്തരിക കഴിവുകൾ** MCP വഴി പ്രദർശിപ്പിക്കില്ല.

**പങ്കാളി കഴിവുകൾ** പങ്കാളി ക്രെഡൻഷ്യലുകൾ ഉപയോഗിക്കുമ്പോൾ മാത്രമേ ലഭ്യമാകൂ.

### WooCommerce സംയോജിത സംവിധാനം (Integration) {#woocommerce-integration}

WooCommerce സംബന്ധമായ **പൊതു കഴിവുകൾ** ലഭ്യമാണ്:

- ഉൽപ്പന്ന മാനേജ്മെന്റ്
- ഓർഡർ പ്രോസസ്സിംഗ്
- ഉപഭോക്തൃ ആശയവിനിമയം

**ആന്തരിക കഴിവുകൾ** WooCommerce-ന് മുന്നിൽ പ്രദർശിപ്പിക്കില്ല.

### ചാറ്റ് ഇന്റർഫേസ് {#chat-interface}

**എല്ലാ കഴിവുകളും** (പൊതു, ആന്തരിക, പങ്കാളി) യൂസർ പെർമിഷനുകൾ അനുസരിച്ച് ചാറ്റിൽ ലഭ്യമാണ്:

- അഡ്മിനിസ്ട്രേറ്റർമാർ എല്ലാ കഴിവുകളും കാണുന്നു
- സാധാരണ യൂസർമാർ പൊതു കഴിവുകൾ മാത്രമേ കാണൂ
- പങ്കാളികൾ പൊതു + പങ്കാളി കഴിവുകൾ കാണുന്നു (അവർ അനുവദ ലിസ്റ്റിൽ ഉൾപ്പെട്ടെങ്കിൽ)

## സുരക്ഷാ മികച്ച രീതികൾ (Security Best Practices) {#security-best-practices}

### കുറഞ്ഞ പ്രത്യേകാവകാശ തത്വം (Principle of Least Privilege) {#principle-of-least-privilege}

- ഉപയോഗിക്കാൻ കഴിയുന്ന ഏറ്റവും കുറഞ്ഞ ദൃശ്യതാ നിയന്ത്രണം കഴിവുകൾക്ക് സജ്ജമാക്കുക
- സെൻസിറ്റീവായ പ്രവർത്തനങ്ങൾക്കായി പങ്കാളി ദൃശ്യതാ നിലവാരം ഉപയോഗിക്കുക
- ഉപയോഗിക്കാത്ത കഴിവുകൾ പ്രവർത്തനരഹിതമാക്കുക

### പതിവായ ഓഡിറ്റുകൾ (Regular Audits) {#regular-audits}

- കഴിവ് ദൃശ്യതാ നിയന്ത്രണം മാസത്തിലൊരിക്കൽ അവലോകനം ചെയ്യുക
- വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾക്കായി പരിശോധിക്കുക
- ഉപയോഗിക്കാത്ത പങ്കാളികളുടെ ആക്‌സസ് നീക്കം ചെയ്യുക

### ലോഗിംഗും നിരീക്ഷണവും (Logging and Monitoring) {#logging-and-monitoring}

- ഏത് കഴിവുകളാണ് ഏറ്റവും കൂടുതൽ ഉപയോഗിക്കുന്നത് എന്ന് നിരീക്ഷിക്കുക
- പങ്കാളി ആക്‌സസ് രീതികൾ ട്രാക്ക് ചെയ്യുക
- അസാധാരണമായ കഴിവ് ഉപയോഗം ഉണ്ടാകുമ്പോൾ മുന്നറിയിപ്പ് നൽകുക

### third-party കഴിവുകൾ {#third-party-abilities}

- പ്രവർത്തനക്ഷമമാക്കുന്നതിന് മുമ്പ് third-party കഴിവുകൾ അവലോകനം ചെയ്യുക
- അവ വ്യക്തമായി വർഗ്ഗീകരിക്കുക
- ആന്തരിക അല്ലെങ്കിൽ പങ്കാളി ദൃശ്യതാ നിലവാരത്തിൽ നിന്ന് തുടങ്ങുക
- പരിശോധനകൾക്ക് ശേഷം മാത്രമേ പൊതുവായി പ്രോത്സാഹിപ്പിക്കാവൂ

## പ്രശ്നപരിഹാരം (Troubleshooting) {#troubleshooting}

**ഒരു കഴിവ് ലിസ്റ്റിൽ കാണുന്നില്ലെങ്കിൽ**
- കഴിവ് ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടെന്നും സജീവമാണെന്നും ഉറപ്പാക്കുക
- പ്ലഗിൻ/ആഡ്‌ഓൺ പ്രവർത്തനക്ഷമമാണോയെന്നും പരിശോധിക്കുക
- പേജ് റീഫ്രഷ് ചെയ്യുക

**ഒരു കഴിവ് ദൃശ്യതാ നിലവാരം മാറ്റാൻ കഴിയുന്നില്ലെങ്കിൽ**
- നിങ്ങൾക്ക് അഡ്മിനിസ്ട്രേറ്റർ പെർമിഷനുകൾ ഉണ്ടോയെന്ന് ഉറപ്പാക്കുക
- പ്ലഗിൻ കാരണം കഴിവ് ലോക്ക് ചെയ്തിട്ടില്ലെന്ന് പരിശോധിക്കുക
- പരസ്പര വിരുദ്ധമായ പ്ലഗിനുകൾ പ്രവർത്തനരഹിതമാക്കാൻ ശ്രമിക്കുക

**ഒരു പങ്കാളിക്ക് ഒരു കഴിവ് ആക്‌സസ് ചെയ്യാൻ കഴിയുന്നില്ലെങ്കിൽ**
- പങ്കാളി അനുവദ ലിസ്റ്റിൽ ഉൾപ്പെട്ടിട്ടുണ്ടോയെന്ന് ഉറപ്പാക്കുക
- കഴിവ് ദൃശ്യതാ നിലവാരം 'Partner' എന്ന് സജ്ജീകരിച്ചിട്ടുണ്ടോയെന്ന് പരിശോധിക്കുക
- പങ്കാളിയുടെ ക്രെഡൻഷ്യലുകൾ ശരിയാണോയെന്ന് സ്ഥിരീകരിക്കുക
- ഓതന്റിക്കേഷൻ പിശകുകൾക്കായി API ലോഗുകൾ പരിശോധിക്കുക

**വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ കാണിക്കത്തന്നെ തുടരുകയാണെങ്കിൽ**
- അഡ്മിൻ അറിയിപ്പിൽ അവ അവലോകനം ചെയ്ത് വർഗ്ഗീകരിക്കുക
- നിങ്ങളുടെ വർഗ്ഗീകരണം സേവ് ചെയ്തിട്ടുണ്ടോയെന്ന് പരിശോധിക്കുക
- കഴിവ് നൽകുന്ന പ്ലഗിൻ അപ്‌ഡേറ്റാണോയെന്നും ഉറപ്പാക്കുക

## പഴയ രീതിയിൽ നിന്ന് മൈഗ്രേഷൻ (Migration from Legacy Mode) {#migration-from-legacy-mode}

നിങ്ങൾ Superdav AI Agent-ന്റെ പഴയ പതിപ്പിൽ നിന്ന് അപ്ഗ്രേഡ് ചെയ്യുകയാണെങ്കിൽ:

- നിലവിലുള്ള എല്ലാ കഴിവുകളും ഓട്ടോമാറ്റിക്കായി പൊതു (Public) കഴിവുകളായി വർഗ്ഗീകരിക്കും
- ആവശ്യാനുസരണം ദൃശ്യതാ നിയന്ത്രണം അവലോകനം ചെയ്ത് ക്രമീകരിക്കുക
- ആക്‌സസ് പരിമിതപ്പെടുത്താൻ ആഗ്രഹിക്കുന്നില്ലെങ്കിൽ നടപടിയെടുക്കേണ്ടതില്ല

Native Abilities API സംയോജനത്തിലേക്കുള്ള മാറ്റത്തെക്കുറിച്ച് കൂടുതൽ വിവരങ്ങൾക്കായി **Third-Party Mode Migration** കാണുക.

## അടുത്ത ഘട്ടങ്ങൾ (Next Steps) {#next-steps}

കഴിവ് ദൃശ്യതാ നിയന്ത്രണം സജ്ജമാക്കിയ ശേഷം:

1. **നിങ്ങളുടെ കഴിവുകൾ അവലോകനം ചെയ്യുക**: ഇൻസ്റ്റാൾ ചെയ്ത എല്ലാ കഴിവുകളും ഓഡിറ്റ് ചെയ്യുക
2. **വർഗ്ഗീകരിക്കാത്ത കഴിവുകൾ വർഗ്ഗീകരിക്കുക**: അഡ്മിൻ അറിയിപ്പുകൾക്ക് മറുപടി നൽകുക
3. **പങ്കാളികളെ സജ്ജമാക്കുക**: പങ്കാളി ദൃശ്യതാ നിലവാരം ഉപയോഗിക്കുന്നുണ്ടെങ്കിൽ പങ്കാളികളെ ചേർക്കുക
4. **ഉപയോഗം നിരീക്ഷിക്കുക**: ഏത് കഴിവുകളാണ് ഏറ്റവും കൂടുതൽ ഉപയോഗിക്കുന്നത് എന്ന് ട്രാക്ക് ചെയ്യുക
5. **തീരുമാനങ്ങൾ രേഖപ്പെടുത്തുക**: ഓരോ ദൃശ്യതാ നിലവാരവും എന്തിനാണ് തിരഞ്ഞെടുത്തതെന്ന് കുറിപ്പുകൾ സൂക്ഷിക്കുക
