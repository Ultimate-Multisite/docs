# എക്‌സ്‌പോർട്ട് ചെയ്യലും ഇംപോർട്ട് ചെയ്യലും

Ultimate Multisite 2.9.0, **Tools > Export & Import** എന്ന ഭാഗത്ത് ഒരു single-site **Export & Import** ടൂൾ ചേർക്കുന്നു. ഒരു WordPress സൈറ്റിനെ ZIP ഫയലായി പാക്കേജ് ചെയ്യണമെങ്കിൽ, ആ ZIP ഫയൽ പുനഃസ്ഥാപിക്കണമെങ്കിൽ, അല്ലെങ്കിൽ ഒരു സൈറ്റിനെ അനുയോജ്യമായ Ultimate Multisite-ൽ നിന്നും single-site WordPress ഇൻസ്റ്റാളേഷനിലേക്ക് മാറ്റണമെങ്കിൽ ഇത് ഉപയോഗിക്കുക.

## ആവശ്യമായ അനുമതികൾ {#required-permissions}

നിങ്ങൾ എക്‌സ്‌പോർട്ട് ചെയ്യുകയോ ഇംപോർട്ട് ചെയ്യുകയോ ചെയ്യുന്ന സൈറ്റിലെ WordPress **Tools** മെനു ആക്‌സസ് ചെയ്യാൻ കഴിയുന്ന ഒരു അഡ്മിനിസ്ട്രേറ്റർ ആയിരിക്കണം സൈൻ ഇൻ ചെയ്തിരിക്കണം. ഒരു multisite network-ൽ, network-ലെ Ultimate Multisite ടൂളുകൾ ഉപയോഗിച്ച് സബ്സൈറ്റുകൾ എക്‌സ്‌പോർട്ട് ചെയ്യുകയോ ഇംപോർട്ട് ചെയ്യുകയോ ചെയ്യുമ്പോൾ ഒരു network അഡ്മിനിസ്ട്രേറ്റർ അക്കൗണ്ട് ഉപയോഗിക്കുക.

Export ZIP ഡൗൺലോഡുകൾ ഒരു ഓതന്റിക്കേറ്റഡ് ഡൗൺലോഡ് എൻഡ്‌പോയിന്റ് വഴി നൽകുന്നതിനാൽ, ഡൗൺലോഡ് പൂർത്തിയാകുന്നതുവരെ അഡ്മിൻ സെഷൻ സജീവമായി നിലനിർത്തുക, കൂടാതെ ജനറേറ്റുചെയ്ത ഡൗൺലോഡ് URL-കൾ പൊതുവായി പങ്കിടരുത്.

## ഒരു സൈറ്റ് ZIP ആക്കി എക്‌സ്‌പോർട്ട് ചെയ്യൽ {#exporting-a-site-to-a-zip}

1. നിങ്ങൾ കോപ്പി ചെയ്യാൻ ആഗ്രഹിക്കുന്ന സൈറ്റിന്റെ WordPress അഡ്മിനിൽ, **Tools > Export & Import** എന്നതിലേക്ക് പോകുക.
2. എക്‌സ്‌പോർട്ട് ഏരിയ തുറന്ന്, പാക്കേജ് ചെയ്യാൻ ആഗ്രഹിക്കുന്ന സൈറ്റ് തിരഞ്ഞെടുക്കുക.
3. ഓപ്ഷണൽ ആയി ഉൾപ്പെടുത്തേണ്ട ഉള്ളടക്കം (ഉദാഹരണത്തിന്, uploads, plugins, themes) ഉണ്ടെങ്കിൽ അത് തിരഞ്ഞെടുക്കുക.
4. എക്‌സ്‌പോർട്ട് ആരംഭിക്കുകയും പ്രോസസ്സ് പൂർത്തിയാകാൻ കാത്തിരിക്കുകയും ചെയ്യുക. വലിയ സൈറ്റുകൾക്ക് ZIP തയ്യാറാകുന്നതിന് മുമ്പ് ബാക്ക്ഗ്രൗണ്ടിൽ പൂർത്തിയാകാൻ സമയം എടുക്കിയേക്കാം.
5. എക്‌സ്‌പോർട്ടുകളുടെ ലിസ്റ്റിൽ നിന്ന് പൂർത്തിയായ ZIP ഡൗൺലോഡ് ചെയ്യുക.

ഈ ZIP സുരക്ഷിതമായ ഒരിടത്ത് സൂക്ഷിക്കുക. ഇതിൽ സൈറ്റ് ഉള്ളടക്കം, സെറ്റിംഗുകൾ, മീഡിയ ഫയലുകൾ, തിരഞ്ഞെടുത്ത കോഡ് ആസ്തികൾ എന്നിവ അടങ്ങിയിരിക്കാം.

## എക്‌സ്‌പോർട്ടിൽ ഉൾപ്പെടുന്നത് {#what-the-export-includes}

ഒരു എക്‌സ്‌പോർട്ട് ZIP-ൽ ഉൾപ്പെടാൻ സാധ്യതയുള്ളവ:

- തിരഞ്ഞെടുത്ത സൈറ്റിന്റെ ഡാറ്റാബേസ് ഉള്ളടക്കവും കോൺഫിഗറേഷനും.
- uploads ഉൾപ്പെടുത്തുകയാണെങ്കിൽ അപ്‌ലോഡ് ചെയ്ത മീഡിയ ഫയലുകൾ.
- ഈ ഓപ്ഷനുകൾ തിരഞ്ഞെടുക്കുകയാണെങ്കിൽ plugins-ഉം themes-ഉം.
- ടാർഗെറ്റ് ഇൻസ്റ്റാളേഷനിൽ സൈറ്റ് പുനർനിർമ്മിക്കാൻ Export & Import ടൂൾ ഉപയോഗിക്കുന്ന ഇംപോർട്ട് മെറ്റാഡാറ്റ.

കൃത്യമായ ZIP വലുപ്പം മീഡിയയുടെ അളവിനെ, തിരഞ്ഞെടുത്ത plugins, themes, സൈറ്റിന്റെ ഡാറ്റാബേസ് ടേബിളുകളുടെ വലുപ്പത്തെ എന്നിവയെ ആശ്രയിച്ചിരിക്കും.

## ZIP-ൽ നിന്ന് ഒരു സൈറ്റ് ഇംപോർട്ട് ചെയ്യൽ {#importing-a-site-from-a-zip}

1. ലക്ഷ്യസ്ഥാനത്തെ WordPress സൈറ്റിൽ **Tools > Export & Import** എന്നതിലേക്ക് പോകുക.
2. ഇംപോർട്ട് ഏരിയ തുറന്ന്, Export & Import ടൂൾ ഉപയോഗിച്ച് സൃഷ്ടിച്ച ZIP അപ്‌ലോഡ് ചെയ്യുക.
3. സൈറ്റിന് പുതിയ വിലാസം ഉപയോഗിക്കണമെങ്കിൽ ഒരു replacement URL നൽകുക, അല്ലെങ്കിൽ യഥാർത്ഥ URL നിലനിർത്താൻ ഈ ഫീൽഡ് ഒഴിച്ചിടുക.
4. ഇംപോർട്ട് പൂർത്തിയാക്കിയ ശേഷം അപ്‌ലോഡ് ചെയ്ത ZIP ഡിലീറ്റ് ചെയ്യണോ എന്ന് തിരഞ്ഞെടുക്കുക.
5. **Begin Import** ക്ലിക്ക് ചെയ്യുക.
6. ഇംപോർട്ട് പൂർത്തിയാകുന്നതുവരെ പെങ്കിലും ചെയ്യേണ്ട ഇംപോർട്ടിനെ നിരീക്ഷിക്കുക. പ്രോസസ്സ് പൂർത്തിയാകുന്നതിന് മുമ്പ് നിർത്തേണ്ട സാഹചര്യമാകുമ്പോൾ മാത്രമേ **Cancel Import** ഉപയോഗിക്കാവൂ.
7. സാധാരണ ട്രാഫിക് അല്ലെങ്കിൽ കസ്റ്റമർ ആക്‌സസ് അനുവദിക്കുന്നതിന് മുമ്പ് ഇംപോർട്ട് ചെയ്ത സൈറ്റ് പരിശോധിക്കുക.

ഒരു single-site WordPress ഇൻസ്റ്റാളേഷനിൽ, ഒരു ZIP ഇംപോർട്ട് ചെയ്യുന്നത് നിലവിലെ സൈറ്റിനെ ഇംപോർട്ട് ചെയ്ത സൈറ്റുകൊണ്ട് മാറ്റിസ്ഥാപിക്കും. അതിനാൽ, തുടങ്ങുന്നതിന് മുമ്പ് ലക്ഷ്യസ്ഥാന സൈറ്റിന്റെ പൂർണ്ണമായ ബാക്കപ്പ് എടുക്കുക, ഒരേ സമയം ഒരേ സൈറ്റിനായി ഒന്നിലധികം ഇംപോർട്ടുകൾ ആരംഭിക്കുന്നത് ഒഴിവാക്കുക.

## പരിമിതികളും അനുയോജ്യത കുറിപ്പുകളും {#limitations-and-compatibility-notes}

- വളരെ വലിയ uploads ഡയറക്‌ടറികളോ മീഡിയ ലൈബ്രറികളോ വലിയ ZIP ഫയലുകൾ ഉണ്ടാക്കിയേക്കാം. വലിയ സൈറ്റുകൾ എക്‌സ്‌പോർട്ട് ചെയ്യുമ്പോഴോ ഇംപോർട്ട് ചെയ്യുമ്പോഴോ PHP അപ്‌ലോഡ് ലിമിറ്റുകൾ, എക്സിക്യൂഷൻ ലിമിറ്റുകൾ, ഡിസ്ക് സ്പേസ്, മെമ്മറി, സെർവർ ടൈമൗട്ട് സെറ്റിംഗുകൾ എന്നിവ ഉറപ്പാക്കുക.
- വളരെ വലിയ മീഡിയ ലൈബ്രറികൾ കുറഞ്ഞ ട്രാഫിക് ഉള്ള ഒരു മെയിന്റനൻസ് വിൻഡോയിൽ മാറ്റേണ്ടി വന്നേക്കാം.
- ലക്ഷ്യസ്ഥാന WordPress ഇൻസ്റ്റാളേഷൻ അനുയോജ്യമായ WordPress, PHP, Ultimate Multisite, plugin, theme പതിപ്പുകൾ എന്നിവയിൽ പ്രവർത്തിക്കണം.
- ഒരു single-site ഇംപോർട്ട് ചെയ്യുന്നത് ലക്ഷ്യസ്ഥാന സൈറ്റിനെ മാറ്റിസ്ഥാപിക്കുന്നതാണ്. ഇത് ഒരു merge ടൂൾ അല്ല.
- Multisite-ൽ നിന്ന് single-site ലേക്കുള്ള മാറ്റങ്ങളും single-site-ൽ നിന്ന് multisite ലേക്കുള്ള മാറ്റങ്ങളും, ലക്ഷ്യസ്ഥാന അന്തരീക്ഷത്തിന് എക്‌സ്‌പോർട്ട് ചെയ്ത സൈറ്റിന്റെ plugins, themes, URLs, ആവശ്യമായ Ultimate Multisite ഘടകങ്ങൾ എന്നിവ പ്രവർത്തിപ്പിക്കാൻ കഴിയുമ്പോൾ മാത്രമേ പിന്തുണയ്ക്കുകയുള്ളൂ.
- ZIP സ്വകാര്യമായി സൂക്ഷിക്കുക. ഇതിൽ എക്‌സ്‌പോർട്ട് ചെയ്ത സൈറ്റിന്റെ ഡാറ്റാബേസ് ഉള്ളടക്കം, അപ്‌ലോഡ് ചെയ്ത ഫയലുകൾ, കോൺഫിഗറേഷൻ വിശദാംശങ്ങൾ എന്നിവ അടങ്ങിയിരിക്കാം.

പഴയ network-level എക്‌സ്‌പോർട്ട് വർക്ക്‌ഫ്ലോകൾ കാണാൻ [Site Export](/user-guide/administration/site-export) കാണുക.
