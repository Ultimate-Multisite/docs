# ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങളും വിലനിർണ്ണയവും

ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾ വഴി നിങ്ങൾക്ക് വിലനിർണ്ണയം, TLD-കൾ, WHOIS സ്വകാര്യത, പ്രൊവൈഡർ തിരഞ്ഞെടുക്കൽ എന്നിവ നിയന്ത്രിക്കാൻ കഴിയും. ഓരോ ഡൊമെയ്ൻ ഉൽപ്പന്നവും ഒരു സ്റ്റാൻഡേർഡ് Multisite Ultimate ഉൽപ്പന്നമാണ്, അതിനൊപ്പം അധികമായി ഒരു **Domain Settings** ടാബും ഉണ്ടാകും.

## ഒരു ഡൊമെയ്ൻ ഉൽപ്പന്നം സൃഷ്ടിക്കൽ

1. **Network Admin › Ultimate Multisite › Products** എന്നതിലേക്ക് പോകുക
2. **Add New** ക്ലിക്ക് ചെയ്യുക
3. ഉൽപ്പന്ന തരം **Domain** എന്ന് സജ്ജമാക്കുക
4. **Domain Settings** ടാബ് കോൺഫിഗർ ചെയ്യുക (താഴെ കാണുക)
5. സേവ് ചെയ്യുക

ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾ ഉൽപ്പന്ന ലിസ്റ്റിൽ വയലറ്റ് നിറത്തിലുള്ള **Domain** ബാഡ്ജ్‌യോടെ കാണാം. **Domain Products** ടാബ് ഉപയോഗിച്ച് ഇവ ഫിൽട്ടർ ചെയ്യാനും സാധിക്കും.

## ഡൊമെയ്ൻ സെറ്റിംഗ്സ് ടാബ്

### പ്രൊവൈഡർ (Provider)

ഈ ഉൽപ്പന്നത്തിനായി രജിസ്ട്രേഷൻ കൈകാര്യം ചെയ്യുന്ന രജിസ്ട്രാർ ഏതാണെന്ന് തിരഞ്ഞെടുക്കുക. ഇത് സാധാരണയായി Domain Seller സെറ്റിംഗ്സിൽ നൽകിയിട്ടുള്ള ഗ്ലോബൽ **Default provider** ആയിരിക്കും.

### പിന്തുണയ്ക്കുന്ന TLD-കൾ (Supported TLDs)

എല്ലാ TLD-കൾക്കും ബാധകമാകുന്ന ഒരു **catch-all product** സൃഷ്ടിക്കാൻ ഇത് ഒഴിച്ചിടുക.

ഒരു **TLD-specific product** സൃഷ്ടിക്കാൻ, അതായത് ആ പ്രത്യേക എക്സ്റ്റൻഷനുകൾക്ക് മാത്രമേ ബാധകമാകുന്ന ഉൽപ്പന്നം, കോമ ഉപയോഗിച്ച് വേർതിരിച്ച TLD-കളുടെ ലിസ്റ്റ് നൽകുക (ഉദാഹരണത്തിന്, `.com, .net, .org`).

**ഉൽപ്പന്ന മാച്ചിംഗ് എങ്ങനെ പ്രവർത്തിക്കുന്നു:** ഒരു ഉപഭോക്താവ് ഒരു ഡൊമെയ്ൻ തിരയുമ്പോൾ, ആഡ്-ഓൺ ഏറ്റവും അനുയോജ്യമായ ഉൽപ്പന്നം തിരഞ്ഞെടുക്കുന്നു. TLD ലിസ്റ്റിൽ `.com` ഉള്ള ഉൽപ്പന്നത്തിന് ഒരു catch-all ഉൽപ്പന്നത്തേക്കാൾ മുൻഗണന ലഭിക്കും. ഒരു TLD-specific ഉൽപ്പന്നം മാച്ച് ആകുന്നില്ലെങ്കിൽ, catch-all ഉൽപ്പന്നം ഉപയോഗിക്കും. ഉൽപ്പന്നങ്ങൾ ഒന്നും ഇല്ലെങ്കിൽ, ഡൊമെയ്ൻ തിരയൽ കാണിക്കില്ല.

### മാർക്കപ്പ് തരം (Markup type)

വിറ്റഴിക്കാനുള്ള വില (retail price) മൊത്തവിലയിൽ (wholesale cost) നിന്ന് എങ്ങനെ കണക്കാക്കുന്നു എന്ന് നിയന്ത്രിക്കുന്ന മൂന്ന് രീതികളാണ് ഇവ:

| മോഡ് | ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു |
|---|---|
| **Percentage** | മൊത്തവിലയുടെ മുകളിൽ ഒരു ശതമാനം കൂട്ടിച്ചേർക്കുന്നു. $10 മൊത്തവിലയുള്ള ഒരു ഡൊമെയ്‌നിൽ 20% മാർക്കപ്പ് നൽകിയാൽ $12 ലഭിക്കും. |
| **Fixed markup** | ഒരു നിശ്ചിത ഡോളർ തുക കൂട്ടിച്ചേർക്കുന്നു. $10 ഡൊമെയ്‌നിൽ $5 മാർക്കപ്പ് നൽകിയാൽ $15 ലഭിക്കും. |
| **Fixed price** | മൊത്തവിലയെ പൂർണ്ണമായും അവഗണിക്കുന്നു. നിങ്ങൾ നൽകുന്ന തുക എപ്പോഴും ഈടാക്കും. |

### ആമുഖ വിലനിർണ്ണയം (Introductory pricing)

ആദ്യ വർഷത്തേക്ക് കുറഞ്ഞ വില നൽകാൻ ഇത് പ്രവർത്തനക്ഷമമാക്കുക. സാധാരണ **Renewal price** (രണ്ടാം വർഷം മുതലുള്ള വില) എന്നതിനൊപ്പം ഒരു പ്രത്യേക **Introductory price** (ഒന്നാം വർഷത്തെ വില) സജ്ജമാക്കുക. രൂപാന്തരീകരണ സമയത്ത് എന്താണ് പ്രതീക്ഷിക്കേണ്ടതെന്ന് അറിയാൻ ഉപഭോക്താവിന് രണ്ട് വിലകളും കാണാൻ കഴിയും.

### WHOIS സ്വകാര്യത (WHOIS privacy)

ഈ ഉൽപ്പന്നത്തിലൂടെ രജിസ്റ്റർ ചെയ്യുന്ന ഡൊമെയ്‌നുകൾക്ക് WHOIS സ്വകാര്യതാ സംരക്ഷണം നൽകുന്നുണ്ടോ എന്ന് ഇത് നിയന്ത്രിക്കുന്നു.

| സെറ്റിംഗ് | പെരുമാറ്റം |
|---|---|
| **Disabled** | WHOIS സ്വകാര്യത ഒരിക്കലും നൽകുകയോ പ്രവർത്തനക്ഷമമാക്കുകയോ ചെയ്യുന്നില്ല. |
| **Always Included** | WHOIS സ്വകാര്യത രജിസ്ട്രേഷനിൽ യാതൊരു ചാർജുമില്ലാതെ ഓട്ടോമാറ്റിക്കായി പ്രവർത്തനക്ഷമമാക്കുന്നു. |
| **Customer Choice** | ചെക്കൗട്ടിൽ ഒരു ചെക്ക്ബോക്സ് പ്രത്യക്ഷപ്പെടും. ഓരോ വർഷവും ഈടാക്കാൻ **Privacy price** സജ്ജമാക്കുക, അല്ലെങ്കിൽ സൗജന്യമായി നൽകാൻ $0 എന്ന് വിടുക. |

Namecheap-ന്, WHOIS സ്വകാര്യത WhoisGuard ഉപയോഗിക്കുന്നു (എപ്പോഴും സൗജന്യം). OpenSRS-ന്, ഇത് OpenSRS സ്വകാര്യതാ സേവനം ഉപയോഗിക്കുന്നു (മൊത്തവിലയിൽ ചിലവ് വരാം).

## TLD ഇംപോർട്ടും സിൻക്കും (TLD import and sync)

ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾ കണക്ട് ചെയ്ത പ്രൊവൈഡറിൽ നിന്ന് ലഭിക്കുന്ന തത്സമയ മൊത്തവില കാണിക്കുന്നു. ഇത് പ്രവർത്തിക്കണമെങ്കിൽ, TLD-കൾ ഇംപോർട്ട് ചെയ്യണം.

- **മാനുവൽ സിങ്ക്:** Settings › Domain Seller › Sync TLDs (പ്രൊവൈഡർ അനുസരിച്ച്)
- **ഓട്ടോമാറ്റിക് സിങ്ക്:** എല്ലാ കോൺഫിഗർ ചെയ്ത പ്രൊവൈഡർകളിലും ഒരു ഷെഡ്യൂൾഡ് ക്രോൺ ജോബ് വഴി ദിവസവും പ്രവർത്തിക്കുന്നു.

ഒരു സിങ്ക് കഴിഞ്ഞ ശേഷം, ഏത് ഡൊമെയ്ൻ ഉൽപ്പന്നത്തിന്റെ Domain Settings ടാബിലേക്കും പോയി, ലഭ്യമായ TLD-കൾ അവയുടെ നിലവിലെ മൊത്തവിലകളോടൊപ്പം കാണാൻ TLD പിക്കർ ഉപയോഗിക്കുക.

## ഓട്ടോ-റിന്യൂവൽ (Auto-renewal)

ഡൊമെയ്ൻ പുതുക്കലുകൾ ഉപഭോക്താവിന്റെ അംഗത്വ സ്റ്റാറ്റസുമായി ബന്ധപ്പെട്ടിരിക്കുന്നു:

- ഒരു അംഗത്വം പുതുക്കുകയും ഒരു ഡൊമെയ്ൻ ലിങ്ക് ചെയ്യുകയും ചെയ്യുമ്പോൾ, ഡൊമെയ്ൻ പുതുക്കൽ ഓട്ടോമാറ്റിക്കായി ക്യൂവിലാകും
- പുതുക്കൽ ശ്രമങ്ങൾ ഉപഭോക്താവിന്റെ സജീവ പേയ്‌മെന്റ് ഗേറ്റ്‌വേ ഉപയോഗിക്കുന്നു
- പരാജയപ്പെട്ട പുതുക്കലുകൾ എക്സ്പോണൻഷ്യൽ ബാക്ക്ഓഫ് ഉപയോഗിച്ച് ഓട്ടോമാറ്റിക്കായി വീണ്ടും ശ്രമിക്കും
- വിജയകരമായ പുതുക്കലുകൾ, പരാജയങ്ങൾ, വരാനിരിക്കുന്ന കാലാവധി തീരുന്നത് എന്നിവയ്ക്കായി ഇമെയിൽ അറിയിപ്പുകൾ അയയ്ക്കുന്നു

ഡൊമെയ്ൻ ലൈഫ്സൈക്കിൾ ഇവന്റുകൾക്കുള്ള ഇമെയിൽ ടെംപ്ലേറ്റ് ID-കൾ:

| ഇവന്റ് | ടെംപ്ലേറ്റ് ID |
|---|---|
| ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്തു | `domain_registered` |
| ഡൊമെയ്ൻ പുതുക്കി | `domain_renewed` |
| പുതുക്കൽ പരാജയപ്പെട്ടു | `domain_renewal_failed` |
| ഡൊമെയ്ൻ ഉടൻ കാലാവധി തീരുന്നു | `domain_expiring_soon` |

## അഡ്മിൻ: മാന്വൽ ഡൊമെയ്ൻ രജിസ്ട്രേഷൻ (Admin: Manual domain registration)

ഉപഭോക്താവ് ചെക്കൗട്ടിലൂടെ പോകാതെ, അവരുടെ പേരിൽ ഒരു ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യാൻ:

1. **Network Admin › Ultimate Multisite › Register Domain** എന്നതിലേക്ക് പോകുക
2. ഉപഭോക്താവിനെ തിരഞ്ഞെടുക്കുകയും ഡൊമെയ്ൻ പേര് നൽകുകയും ചെയ്യുക
3. രജിസ്ട്രന്റ് കോൺടാക്റ്റ് വിവരങ്ങൾ പൂരിപ്പിക്കുക (പേര്, വിലാസം, ഫോൺ)
4. **Register** ക്ലിക്ക് ചെയ്യുക

ഡൊമെയ്ൻ റെക്കോർഡ് സൃഷ്ടിക്കപ്പെടുകയും ഉപഭോക്താവിന്റെ അക്കൗണ്ടുമായി ബന്ധിപ്പിക്കുകയും ചെയ്യുന്നു.
