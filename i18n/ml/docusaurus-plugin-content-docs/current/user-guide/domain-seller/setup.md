# ഡൊമെയ്ൻ വിൽപ്പന: സജ്ജീകരണവും ദാതൃ കോൺഫിഗറേഷനും

ഡൊമെയ്ൻ വിൽപ്പന ആഡോണിൽ ആവശ്യമായ ഓരോ ഘട്ടത്തിലൂടെയും നിങ്ങളെ നയിക്കുന്ന സജ്ജീകരണ വിസാർഡ് ഉൾപ്പെടുത്തിയിട്ടുണ്ട്. ഈ പേജിൽ വിസാർഡ് പ്രവാഹവും പിന്നീട് ദാതാക്കളെ കോൺഫിഗർ ചെയ്യുന്നതോ വീണ്ടും കോൺഫിഗർ ചെയ്യുന്നതോ എങ്ങനെയെന്നും വിശദീകരിക്കുന്നു.

## ആവശ്യകതകൾ

- **Multisite Ultimate** v2.4.12 അല്ലെങ്കിൽ അതിനുമുകളിൽ, നെറ്റ്‌വർക്ക്-സജീവമാക്കിയിരിക്കുന്നത്
- **PHP** 7.4+
- പിന്തുണയ്ക്കുന്ന കുറഞ്ഞത് ഒരു രജിസ്ട്രാറിനായുള്ള API ക്രെഡൻഷ്യലുകൾ

## ആദ്യ ഉപയോഗത്തിനുള്ള സജ്ജീകരണ വിസാർഡ്

നിങ്ങൾ plugin നെറ്റ്‌വർക്ക്-സജീവമാക്കുന്ന ആദ്യ തവണ സജ്ജീകരണ വിസാർഡ് സ്വയമേവ തുറക്കും. ഇത് എപ്പോൾ വേണമെങ്കിലും **നെറ്റ്‌വർക്ക് അഡ്മിൻ › Ultimate Multisite › ഡൊമെയ്ൻ വിൽപ്പന സജ്ജീകരണം** എന്നിടത്തും ലഭ്യമാണ്.

### ഘട്ടം 1 — ഒരു ദാതാവിനെ തിരഞ്ഞെടുക്കുക

നിങ്ങൾ ബന്ധിപ്പിക്കാൻ ആഗ്രഹിക്കുന്ന രജിസ്ട്രാറിനെ തിരഞ്ഞെടുക്കുക. പിന്തുണയ്ക്കുന്ന ഓപ്ഷനുകൾ:

| ദാതാവ് | DNS മാനേജ്മെന്റ് | WHOIS സ്വകാര്യത |
|---|---|---|
| OpenSRS | ഉണ്ട് | ഉണ്ട് |
| Namecheap | ഇല്ല | ഉണ്ട് (WhoisGuard, സൗജന്യം) |
| HostAfrica | ഉണ്ട് | ഉണ്ട് (ID സംരക്ഷണം) |
| Openprovider | ഉണ്ട് | ഉണ്ട് |
| Hostinger | ഹോസ്റ്റ് ചെയ്ത ഡൊമെയ്‌നുകൾക്കായുള്ള പ്രധാന Hostinger ഡൊമെയ്ൻ മാപ്പിംഗ് വഴി | ഉണ്ട് |
| GoDaddy | ഇല്ല | ഇല്ല |
| ResellerClub | ഉണ്ട് | ഇല്ല |
| NameSilo | ഇല്ല | ഇല്ല |
| Enom | ഉണ്ട് | ഇല്ല |

### ഘട്ടം 2 — ക്രെഡൻഷ്യലുകൾ നൽകുക

ഓരോ ദാതാവിനും വ്യത്യസ്ത ക്രെഡൻഷ്യൽ ഫീൽഡുകൾ ഉണ്ട്:

**OpenSRS** — ഉപയോക്തൃനാമവും സ്വകാര്യ കീയും (OpenSRS റീസെല്ലർ കൺട്രോൾ പാനലിൽ നിന്ന്)

**Namecheap** — ഉപയോക്തൃനാമവും API കീയും (Account › ഉപകരണങ്ങൾ › API ആക്‌സസ് എന്നിടത്ത് നിന്ന്)

**HostAfrica** — HostAfrica റീസെല്ലർ മൊഡ്യൂളിൽ നിന്നുള്ള ഡൊമെയ്‌നുകൾക്കായുള്ള റീസെല്ലർ API endpoint ഉം ക്രെഡൻഷ്യലുകളും. പ്രത്യേക sandbox endpoint ഇപ്പോൾ രേഖപ്പെടുത്തിയിട്ടില്ല; ലൈവ് രജിസ്ട്രേഷനുകൾ നടത്തുന്നതിന് മുമ്പ് സുരക്ഷിതമായ വായന-മാത്രം പരിശോധനകളിലൂടെ പരീക്ഷിക്കുക.

**Openprovider** — API ആക്‌സസ് സജീവമാക്കിയ ഉപയോക്തൃനാമവും പാസ്‌വേഡും. ഐച്ഛിക sandbox മോഡ് Openprovider sandbox API ഉപയോഗിക്കുന്നു. കൂടാതെ, രജിസ്ട്രേഷനുകൾക്കായി ഐച്ഛികമായ ഡിഫോൾട്ട് customer handle വീണ്ടും ഉപയോഗിക്കാം.

**Hostinger** — പ്രധാന Hostinger ഇന്റഗ്രേഷനിൽ നിന്നുള്ള പങ്കിട്ട Hostinger hPanel API token. അതേ token ആണ് പ്രധാന ഡൊമെയ്ൻ മാപ്പിംഗിനും ഡൊമെയ്ൻ വിൽപ്പന രജിസ്ട്രേഷൻ പ്രവർത്തനങ്ങൾക്കും ശക്തി നൽകുന്നത്.

**GoDaddy** — API കീയും രഹസ്യവും (developer.godaddy.com ൽ നിന്ന്)

**ResellerClub** — റീസെല്ലർ IDയും API കീയും (ResellerClub കൺട്രോൾ പാനലിൽ നിന്ന്)

**NameSilo** — API കീ (namesilo.com › Account › API Manager എന്നിടത്ത് നിന്ന്)

**Enom** — Account IDയും API token ഉം

ലൈവിലേക്ക് പോകുന്നതിന് മുമ്പ് ദാതാവിന്റെ ടെസ്റ്റ് പരിസ്ഥിതിയിൽ പരീക്ഷിക്കാൻ ലഭ്യമായിടത്ത് **Sandbox മോഡ്** തിരഞ്ഞെടുക്കുക.

### ഘട്ടം 3 — കണക്ഷൻ പരിശോധിക്കുക

**കണക്ഷൻ പരിശോധിക്കുക** ക്ലിക്ക് ചെയ്യുക. ക്രെഡൻഷ്യലുകളും കണക്റ്റിവിറ്റിയും പരിശോധിക്കാൻ വിസാർഡ് ലഘുവായ API call അയക്കും. തുടരുന്നതിന് മുമ്പ് ക്രെഡൻഷ്യൽ പ്രശ്നങ്ങൾ പരിഹരിക്കുക.

### ഘട്ടം 4 — TLDകൾ ഇമ്പോർട്ട് ചെയ്യുക

ബന്ധിപ്പിച്ച ദാതാവിൽ നിന്ന് ലഭ്യമായ എല്ലാ TLDകളും ഹോൾസെയിൽ വിലകളും എടുക്കാൻ **TLDകൾ ഇമ്പോർട്ട് ചെയ്യുക** ക്ലിക്ക് ചെയ്യുക. ഇത് ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾ ഉപയോഗിക്കുന്ന TLD പട്ടിക പൂരിപ്പിക്കും. വലിയ TLD കാറ്റലോഗുകളുള്ള ദാതാക്കൾക്ക് ഇമ്പോർട്ടിന് 30–60 സെക്കൻഡ് എടുത്തേക്കാം.

ഷെഡ്യൂൾ ചെയ്ത cron ജോബ് വഴി TLDകൾ ദിവസത്തിൽ ഒരിക്കൽ സ്വയമേവ വീണ്ടും സിങ്ക് ചെയ്യപ്പെടും.

### ഘട്ടം 5 — ഒരു ഡൊമെയ്ൻ ഉൽപ്പന്നം സൃഷ്ടിക്കുക

വിസാർഡ് 10% മാർക്കപ്പുള്ള ഒരു ഡിഫോൾട്ട് catch-all ഡൊമെയ്ൻ ഉൽപ്പന്നം സൃഷ്ടിക്കും. നിങ്ങൾക്ക് ഈ ഉൽപ്പന്നം ഉടൻ തിരുത്താനോ ഒഴിവാക്കി **Ultimate Multisite › ഉൽപ്പന്നങ്ങൾ** എന്നിടത്ത് കൈമാറി ഉൽപ്പന്നങ്ങൾ സൃഷ്ടിക്കാനോ കഴിയും.

പൂർണ്ണ ഉൽപ്പന്ന കോൺഫിഗറേഷൻ ഗൈഡിനായി [ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങളും വിലനിർണ്ണയവും](./domain-products) കാണുക.

## ഒരു ദാതാവിനെ വീണ്ടും കോൺഫിഗർ ചെയ്യുക

**നെറ്റ്‌വർക്ക് അഡ്മിൻ › Ultimate Multisite › ക്രമീകരണങ്ങൾ › ഡൊമെയ്ൻ വിൽപ്പന** എന്നിടത്തേക്ക് പോകുക (അല്ലെങ്കിൽ plugin പട്ടികയിൽ **ക്രമീകരണങ്ങൾ** ക്ലിക്ക് ചെയ്യുക).

ക്രമീകരണ പേജിൽ ഉൾപ്പെടുന്നത്:

- **ഡൊമെയ്ൻ വിൽപ്പന സജീവമാക്കുക** — മുഴുവൻ സവിശേഷതയും ഓൺ/ഓഫ് ചെയ്യുക
- **ഡിഫോൾട്ട് ദാതാവ്** — ഡൊമെയ്ൻ തിരയലുകൾക്കും പുതിയ ഉൽപ്പന്നങ്ങൾക്കും ഉപയോഗിക്കുന്ന ദാതാവ്
- **ഓരോ തിരയലിലുമുള്ള പരമാവധി TLDകൾ** — ഒരു ഉപഭോക്താവ് തിരയുമ്പോൾ പരിശോധിക്കേണ്ട TLDകളുടെ എണ്ണം; ഉയർന്ന മൂല്യങ്ങൾ കൂടുതൽ ഓപ്ഷനുകൾ കാണിക്കും, പക്ഷേ വേഗം കുറയും
- **ലഭ്യത cache ദൈർഘ്യം** — ലഭ്യതയും വില ഫലങ്ങളും എത്രകാലം cache ചെയ്യണം; കുറഞ്ഞ മൂല്യങ്ങൾ കൂടുതൽ കൃത്യമാണെങ്കിലും API callകൾ വർധിക്കും
- **ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾ മാനേജുചെയ്യുക** — ഉൽപ്പന്നങ്ങളുടെ പട്ടികയിലേക്കുള്ള ദ്രുത ലിങ്ക്
- **ദാതാക്കളെ കോൺഫിഗർ ചെയ്യുക** — ദാതാക്കളെ ചേർക്കാനോ വീണ്ടും കോൺഫിഗർ ചെയ്യാനോ ഇന്റഗ്രേഷൻ വിസാർഡ് തുറക്കും

### രണ്ടാമത്തെ ദാതാവിനെ ചേർക്കുക

**ദാതാക്കളെ കോൺഫിഗർ ചെയ്യുക** ക്ലിക്ക് ചെയ്ത് പുതിയ രജിസ്ട്രാറിനായി വിസാർഡ് വീണ്ടും പ്രവർത്തിപ്പിക്കുക. നിങ്ങൾക്ക് ഒരേസമയം ഒന്നിലധികം ദാതാക്കളെ കോൺഫിഗർ ചെയ്തിരിക്കാം. ഓരോ ഡൊമെയ്ൻ ഉൽപ്പന്നവും ഒരു നിർദ്ദിഷ്ട ദാതാവിലേക്ക് അസൈൻ ചെയ്യുക, അല്ലെങ്കിൽ ഡിഫോൾട്ടിൽ തന്നെ വിടുക.

### TLDകൾ കൈമാറി സിങ്ക് ചെയ്യുക

ക്രമീകരണ പേജിൽ, ഏറ്റവും പുതിയ വിലകൾ എടുക്കാൻ കോൺഫിഗർ ചെയ്ത ഏതെങ്കിലും ദാതാവിന് സമീപമുള്ള **TLDകൾ സിങ്ക് ചെയ്യുക** ക്ലിക്ക് ചെയ്യുക. ഒരു ദാതാവ് ഹോൾസെയിൽ വില അപ്ഡേറ്റ് ചെയ്തതിനുശേഷമോ പുതിയ TLDകൾ ചേർത്തതിനുശേഷമോ ഇത് ഉപയോഗപ്രദമാണ്.

## ദാതൃ ശേഷി കുറിപ്പുകൾ

എല്ലാ രജിസ്ട്രാർ APIകളും ഒരേ പ്രവർത്തനങ്ങൾ ലഭ്യമാക്കണമെന്നില്ല. പിന്തുണയ്ക്കാത്ത പ്രവർത്തനങ്ങൾ മിണ്ടാതെയുള്ള പരാജയത്തിന് പകരം വ്യക്തമായ അഡ്മിൻ-മുഖമായ പിശകുകളായി addon കാണിക്കുന്നു.

- **HostAfrica** lookup, TLD/pricing sync, registration, renewal, transfer, nameserver updates, DNS records, EPP codes, registrar lock, ID protection എന്നിവ ഉൾപ്പെടുന്ന ഏറ്റവും വിപുലമായ live reseller workflow പിന്തുണയ്ക്കുന്നു.
- **Openprovider** reseller-priced TLD sync, registration, renewal, transfers, nameserver updates, DNS zones, EPP codes, registrar lock, WHOIS privacy എന്നിവ പിന്തുണയ്ക്കുന്നു. ആഡ്‌ഓൺ സ്വയം പുതുക്കുന്ന കുറഞ്ഞ കാലാവധിയുള്ള bearer token ഉപയോഗിച്ചാണ് ഇത് ഓതന്റിക്കേറ്റ് ചെയ്യുന്നത്.
- **Hostinger** പങ്കിട്ട hPanel API token വഴി availability search, registration, portfolio lookup, nameserver updates, registrar lock, WHOIS privacy എന്നിവ പിന്തുണയ്ക്കുന്നു. Hostinger-ന്റെ public Domains API reseller/wholesale pricing, inbound transfer, explicit renewal, EPP-code retrieval എന്നിവ ലഭ്യമാക്കുന്നില്ല; renewals auto-renew മാത്രമാണ്.
