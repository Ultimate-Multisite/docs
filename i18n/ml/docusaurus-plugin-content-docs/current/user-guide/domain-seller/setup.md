# Domain Seller: സജ്ജീകരണവും പ്രൊവൈഡർ കോൺഫിഗറേഷനും

Domain Seller addon ഉപയോഗിക്കുമ്പോൾ, ആവശ്യമായ എല്ലാ ഘട്ടങ്ങളിലൂടെയും നിങ്ങളെ നയിക്കുന്ന ഒരു ഗൈഡഡ് സജ്ജീകരണ വിസാർഡ് (setup wizard) ഇതിനൊപ്പം ലഭിക്കും. ഈ പേജ് വിസാർഡിന്റെ പ്രവർത്തനരീതിയും, അതിനുശേഷം പ്രൊവൈഡർമാരെ എങ്ങനെ കോൺഫിഗർ ചെയ്യാം അല്ലെങ്കിൽ പുനഃക്രമീകരിക്കാം എന്നും വിശദീകരിക്കുന്നു.

## ആവശ്യകതകൾ (Requirements)

- **Multisite Ultimate** v2.4.12 അല്ലെങ്കിൽ അതിലും ഉയർന്ന പതിപ്പ്, നെറ്റ്വർക്ക് ആക്ടിവേറ്റ് ചെയ്തിരിക്കണം
- **PHP** 7.4+
- കുറഞ്ഞത് ഒരു സപ്പോർട്ട് ചെയ്യുന്ന രജിസ്ട്രാർക്കുള്ള API ക്രെഡൻഷ്യലുകൾ

## ആദ്യമായി ഉപയോഗിക്കുമ്പോൾ സജ്ജീകരണ വിസാർഡ് (First-run setup wizard)

പ്ലഗിൻ നെറ്റ്വർക്ക് തലത്തിൽ ആക്ടിവേറ്റ് ചെയ്യുന്ന ആദ്യ തവണ, സജ്ജീകരണ വിസാർഡ് ഓട്ടോമാറ്റിക്കായി ആരംഭിക്കും. ഇത് എപ്പോൾ വേണമെങ്കിലും **Network Admin › Ultimate Multisite › Domain Seller Setup** എന്നതിൽ നിന്ന് ലഭ്യമാണ്.

### ഘട്ടം 1 — ഒരു പ്രൊവൈഡറെ തിരഞ്ഞെടുക്കുക (Step 1 — Choose a provider)

നിങ്ങൾ കണക്ട് ചെയ്യാൻ ആഗ്രഹിക്കുന്ന രജിസ്ട്രാർ തിരഞ്ഞെടുക്കുക. സപ്പോർട്ട് ചെയ്യുന്ന ഓപ്ഷനുകൾ:

| Provider | DNS management | WHOIS privacy |
|---|---|---|
| OpenSRS | Yes | Yes |
| Namecheap | No | Yes (WhoisGuard, free) |
| GoDaddy | No | No |
| ResellerClub | Yes | No |
| NameSilo | No | No |
| Enom | Yes | No |

### ഘട്ടം 2 — ക്രെഡൻഷ്യലുകൾ നൽകുക (Step 2 — Enter credentials)

ഓരോ പ്രൊവൈഡറിനും വ്യത്യസ്തമായ ക്രെഡൻഷ്യൽ ഫീൽഡുകൾ ഉണ്ട്:

**OpenSRS** — യൂസർ നെയിമും പ്രൈവറ്റ് കീയും (OpenSRS Reseller Control Panel-ൽ നിന്ന്)

**Namecheap** — യൂസർ നെയിമും API കീയും (Account › Tools › API Access-ൽ നിന്ന്)

**GoDaddy** — API കീയും സീക്രട്ടും (developer.godaddy.com-ൽ നിന്ന്)

**ResellerClub** — റീസെല്ലർ ID-യും API കീയും (ResellerClub control panel-ൽ നിന്ന്)

**NameSilo** — API കീ (namesilo.com › Account › API Manager-ൽ നിന്ന്)

**Enom** — അക്കൗണ്ട് ID-യും API ടോക്കനും

ലഭ്യമാകുന്നിടത്ത് **Sandbox mode** പരിശോധിക്കുക. ഇത് ലൈവായി ഉപയോഗിക്കുന്നതിന് മുമ്പ് പ്രൊവൈഡറിന്റെ ടെസ്റ്റ് എൻവയോൺമെന്റിൽ പരീക്ഷിക്കാൻ സഹായിക്കും.

### ഘട്ടം 3 — കണക്ഷൻ പരിശോധിക്കുക (Step 3 — Test the connection)

**Test Connection** ക്ലിക്ക് ചെയ്യുക. ക്രെഡൻഷ്യലുകളും കണക്റ്റിവിറ്റിയും പരിശോധിക്കാൻ വിസാർഡ് ഒരു ലഘുവായ API കോൾ അയയ്ക്കുന്നു. മുന്നോട്ട് പോകുന്നതിന് മുമ്പ് ഏതെങ്കിലും ക്രെഡൻഷ്യൽ പ്രശ്നങ്ങൾ പരിഹരിക്കുക.

### ഘട്ടം 4 — TLD-കൾ ഇംപോർട്ട് ചെയ്യുക (Step 4 — Import TLDs)

കണക്ട് ചെയ്ത പ്രൊവൈഡറിൽ നിന്നുള്ള എല്ലാ TLD-കളും ഹോൾസെയിൽ വിലനിർണ്ണയവും (wholesale pricing) വലിച്ചെടുക്കാൻ **Import TLDs** ക്ലിക്ക് ചെയ്യുക. ഇത് domain ഉൽപ്പന്നങ്ങൾ ഉപയോഗിക്കുന്ന TLD ലിസ്റ്റ് നിറയ്ക്കുന്നു. വലിയ TLD കാറ്റലോഗുകളുള്ള പ്രൊവൈഡർമാർക്ക് ഈ ഇംപോർട്ട് ചെയ്യാൻ 30–60 സെക്കൻഡ് സമയം എടുക്കാം.

TLD-കൾ ഒരു ദിവസം ഒരിക്കൽ ഷെഡ്യൂൾ ചെയ്ത cron job വഴി ഓട്ടോമാറ്റിക്കായി പുനഃസമന്വത്തീകരിക്കുകയും (re-synced) ചെയ്യും.

### ഘട്ടം 5 — ഒരു domain ഉൽപ്പന്നം സൃഷ്ടിക്കുക (Step 5 — Create a domain product)

10% മാർക്കപ്പ് സഹിതം ഒരു ഡിഫോൾട്ട് കാച്ച്-ഓൾ (catch-all) domain ഉൽപ്പന്നം വിസാർഡ് സൃഷ്ടിക്കുന്നു. നിങ്ങൾക്ക് ഈ ഉൽപ്പന്നം ഉടൻ തന്നെ എഡിറ്റ് ചെയ്യാം, അല്ലെങ്കിൽ ഒഴിവാക്കി **Ultimate Multisite › Products** എന്നതിൽ നിന്ന് ഉൽപ്പന്നങ്ങൾ സ്വയം സൃഷ്ടിക്കാവുന്നതാണ്.

പൂർണ്ണമായ ഉൽപ്പന്ന കോൺഫിഗറേഷൻ ഗൈഡിനായി [Domain Products and Pricing](./domain-products) കാണുക.

## ഒരു പ്രൊവൈഡർ പുനഃക്രമീകരിക്കുന്നത് (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** എന്നതിലേക്ക് പോകുക (അല്ലെങ്കിൽ പ്ലഗിൻ ലിസ്റ്റിൽ **Settings** ക്ലിക്ക് ചെയ്യുക).

സെറ്റിംഗ്സ് പേജിൽ ഇവ അടങ്ങിയിരിക്കുന്നു:

- **Enable domain selling** — ഈ ഫീച്ചർ ഓൺ/ഓഫ് ചെയ്യാൻ ടോഗിൾ ചെയ്യുക
- **Default provider** — domain തിരയലുകൾക്കും പുതിയ ഉൽപ്പന്നങ്ങൾക്കുമായി ഉപയോഗിക്കുന്ന പ്രൊവൈഡർ
- **Max TLDs per search** — ഒരു customer തിരയുമ്പോൾ എത്ര TLD-കൾ പരിശോധിക്കണം; ഉയർന്ന മൂല്യങ്ങൾ കൂടുതൽ ഓപ്ഷനുകൾ കാണിക്കുന്നു, പക്ഷേ വേഗത കുറവായിരിക്കും
- **Availability cache duration** — ലഭ്യതയുടെയും വിലനിർണ്ണയത്തിന്റെയും ഫലങ്ങൾ എത്രത്തോളം കാഷെ ചെയ്യണം; കുറഞ്ഞ മൂല്യങ്ങൾ കൂടുതൽ കൃത്യമാണ്, പക്ഷേ API കോളുകൾ വർദ്ധിപ്പിക്കും
- **Manage domain products** — Products ലിസ്റ്റിലേക്കുള്ള വേഗത്തിലുള്ള ലിങ്ക്
- **Configure providers** — പ്രൊവൈഡർമാരെ ചേർക്കാനോ പുനഃക്രമീകരിക്കാനോ Integration Wizard തുറക്കുന്നു

### രണ്ടാമത്തെ പ്രൊവൈഡർ ചേർക്കുന്നത് (Adding a second provider)

**Configure providers** ക്ലിക്ക് ചെയ്യുക, എന്നിട്ട് പുതിയ രജിസ്ട്രാറിനായി വിസാർഡ് വീണ്ടും പ്രവർത്തിപ്പിക്കുക. നിങ്ങൾക്ക് ഒരേ സമയം ഒന്നിലധികം പ്രൊവൈഡർമാരെ കോൺഫിഗർ ചെയ്യാൻ കഴിയും. ഓരോ domain ഉൽപ്പന്നത്തെയും ഒരു പ്രത്യേക പ്രൊവൈഡറിന് നൽകുക, അല്ലെങ്കിൽ ഡിഫോൾട്ടിൽ തന്നെ വിടുക.

### TLD-കൾ മാനുവലായി സിങ്ക് ചെയ്യുന്നത് (Syncing TLDs manually)

സെറ്റിംഗ്സ് പേജിൽ, ഏതെങ്കിലും കോൺഫിഗർ ചെയ്ത പ്രൊവൈഡറിന് അടുത്തായി **Sync TLDs** ക്ലിക്ക് ചെയ്ത് ഏറ്റവും പുതിയ വിലനിർണ്ണയങ്ങൾ വലിച്ചെടുക്കുക. ഒരു പ്രൊവൈഡർ ഹോൾസെയിൽ വിലകൾ അപ്ഡേറ്റ് ചെയ്യുകയോ പുതിയ TLD-കൾ ചേർക്കുകയോ ചെയ്തതിന് ശേഷം ഇത് ഉപകാരപ്രദമാണ്.

## Logs

ഓരോ പ്രൊവൈഡറും അതിൻ്റേതായ ഒരു ലോഗ് ചാനലിലേക്ക് എഴുതുന്നു. ലോഗുകൾ **Network Admin › Ultimate Multisite › Logs** എന്നതിൽ നിന്ന് കാണാവുന്നതാണ്:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | എല്ലാ രജിസ്ട്രേഷൻ ശ്രമങ്ങളും (വിജയവും പരാജയവും) |
| `domain-seller-renewal` | പുതുക്കൽ ജോലിയുടെ ഫലങ്ങൾ |
| `domain-seller-opensrs` | Raw OpenSRS API പ്രവർത്തനം |
| `domain-seller-namecheap` | Raw Namecheap API പ്രവർത്തനം |
| `domain-seller-godaddy` | Raw GoDaddy API പ്രവർത്തനം |
| `domain-seller-resellerclub` | Raw ResellerClub API പ്രവർത്തനം |
| `domain-seller-namesilo` | Raw NameSilo API പ്രവർത്തനം |
| `domain-seller-enom` | Raw Enom API പ്രവർത്തനം |
