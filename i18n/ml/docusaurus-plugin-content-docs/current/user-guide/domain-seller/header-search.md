# ഹെഡർ ഡൊമെയിൻ തിരയൽ

ഒരു ഡൊമെയിൻ തിരയൽ ആരംഭിക്കുന്ന ചെറിയ ഹെഡർ ഫോം വേണമെങ്കിൽ ഈ സജ്ജീകരണം ഉപയോഗിക്കുക. തുടർന്ന് Ultimate Multisite ചെക്കൗട്ടിനുള്ളിൽ ലഭ്യമായ ഡൊമെയിനുകളിൽ നിന്ന് ഉപഭോക്താവിന് തിരഞ്ഞെടുക്കാം.

## ആവശ്യകതകൾ

- Ultimate Multisite നെറ്റ്‌വർക്ക്-സജീവമാക്കിയിരിക്കുന്നത്.
- Multisite Ultimate Domain Seller നെറ്റ്‌വർക്ക്-സജീവമാക്കിയിരിക്കുന്നത്.
- കുറഞ്ഞത് ഒരു സജീവ ഡൊമെയിൻ രജിസ്ട്രേഷൻ ഉൽപ്പന്നം, ഇതോടൊപ്പം:
  - `domain_provider` ക്രമീകരിച്ച പ്രൊവൈഡറായി സജ്ജീകരിച്ചിരിക്കുന്നത്.
  - പിന്തുണയ്ക്കുന്ന TLDകൾ ക്രമീകരിച്ചിരിക്കുന്നത്, ഉദാഹരണത്തിന് `com`, `net`, `org`.
- **ഡൊമെയിൻ തിരഞ്ഞെടുപ്പ്** ഫീൽഡ് അടങ്ങിയ സാധുവായ ചെക്കൗട്ട് ഫോം.

## ചെക്കൗട്ട് ഫോം

1. രജിസ്ട്രേഷൻ പേജ് ഉപയോഗിക്കുന്ന ചെക്കൗട്ട് ഫോം സൃഷ്ടിക്കുക അല്ലെങ്കിൽ തിരുത്തുക.
2. **ഉപയോക്തൃനാമം** ഉൾപ്പെടെയുള്ള സാധാരണ നിർബന്ധിത ചെക്കൗട്ട്/Account ഫീൽഡുകൾ ചേർക്കുക. ഡൊമെയിൻ ഫീൽഡ് മാത്രം അടങ്ങിയ ചെക്കൗട്ട് ഫോം Ultimate Multisite സാധൂകരണത്തിൽ നിരസിക്കും.
3. ഒരു **ഡൊമെയിൻ തിരഞ്ഞെടുപ്പ്** ഫീൽഡ് ചേർക്കുക.
4. സൗജന്യ സബ്‌ഡൊമെയിനുകളിലോ നിലവിലുള്ള ഡൊമെയിനുകളിലോ അല്ല, രജിസ്റ്റർ ചെയ്ത ഡൊമെയിനുകളിലാണ് പ്രവാഹം ശ്രദ്ധ കേന്ദ്രീകരിക്കേണ്ടതെങ്കിൽ ഡൊമെയിൻ തിരഞ്ഞെടുപ്പ് മോഡ് **രജിസ്റ്റർ മാത്രം** ആയി സജ്ജീകരിക്കുക.
5. ആ ഫീൽഡിലേക്ക് ഡൊമെയിൻ രജിസ്ട്രേഷൻ ഉൽപ്പന്നം നിയോഗിക്കുക.

രജിസ്ട്രേഷൻ പേജ് ചെക്കൗട്ട് ഫോം റെൻഡർ ചെയ്യണം, ഉദാഹരണത്തിന്:

```text
[wu_checkout slug="domain-form"]
```

## ഹെഡർ ഫോം

സൈറ്റ് ഹെഡറിൽ ചെറിയ `GET` ഫോം ചേർക്കുക. നൽകിയ തിരയൽ പദം `domain_name` ആയി ചെക്കൗട്ട് പേജിലേക്ക് അയയ്ക്കണം:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

കസ്റ്റം ഹെഡർ JavaScriptൽ ഒരു ഡൊമെയിൻ മുൻകൂട്ടി തിരഞ്ഞെടുക്കരുത്. ഹെഡർ തിരയൽ പദം മാത്രം കൈമാറണം. Domain Seller ചെക്കൗട്ട് സ്ക്രിപ്റ്റ് `?domain_name=example` വായിച്ച് ചെക്കൗട്ട് തിരയൽ ബോക്സ് പൂരിപ്പിക്കുകയും ലഭ്യതാ തിരയൽ പ്രവർത്തിപ്പിക്കുകയും ചെയ്യും. അതുവഴി ലഭിച്ച ഡൊമെയിനുകളിൽ നിന്ന് ഉപഭോക്താവിന് തിരഞ്ഞെടുക്കാം.

## പ്രതീക്ഷിക്കുന്ന പെരുമാറ്റം

ഹെഡറിൽ `example` തിരയുമ്പോൾ തുറക്കേണ്ടത്:

```text
/register/?domain_name=example
```

തുടർന്ന് ചെക്കൗട്ട് തിരഞ്ഞെടുക്കാവുന്ന ഫലങ്ങൾ കാണിക്കണം, ഉദാഹരണത്തിന്:

- `example.com`
- `example.net`
- `example.org` ലഭ്യമല്ല
- പ്രൊവൈഡർ പിന്തുണയ്ക്കുന്ന മറ്റ് TLDകൾ

ലഭ്യമായ ഒരു ഫലം തിരഞ്ഞെടുത്തതിന് ശേഷം, ഓർഡർ സംഗ്രഹത്തിൽ ഡൊമെയിൻ രജിസ്ട്രേഷൻ ഉൽപ്പന്നവും തിരഞ്ഞെടുത്ത ഡൊമെയിൻ പേരും ഉൾപ്പെടണം.

## പരിശോധന

1. ഹോം പേജ് തുറക്കുക.
2. ഒരു അടിസ്ഥാന പേര് തിരയുക, ഉദാഹരണത്തിന് `example`.
3. ചെക്കൗട്ട് URLൽ `?domain_name=example` ഉൾപ്പെട്ടിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക.
4. ചെക്കൗട്ട് ഡൊമെയിൻ ഫീൽഡിൽ `example` ഉണ്ടെന്ന് ഉറപ്പാക്കുക.
5. ഡൊമെയിൻ തിരഞ്ഞെടുപ്പുകളുടെ പട്ടിക കാണുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക.
6. ലഭ്യമായ ഒരു ഡൊമെയിനിന് **തിരഞ്ഞെടുക്കുക** ക്ലിക്ക് ചെയ്യുക.
7. ഓർഡർ സംഗ്രഹത്തിൽ `Domain Registration - example.com` അല്ലെങ്കിൽ തിരഞ്ഞെടുത്ത ഡൊമെയിൻ ഉണ്ടെന്ന് ഉറപ്പാക്കുക.

## പ്രശ്നപരിഹാരം

- പട്ടിക ഒന്നും കാണുന്നില്ലെങ്കിൽ, `admin-ajax.php?action=wu_domain_search` എന്നതിനായി ബ്രൗസർ നെറ്റ്‌വർക്ക് ടാബ് പരിശോധിക്കുക. അത് ശൂന്യമല്ലാത്ത `domains` അല്ലെങ്കിൽ `results` മടക്കുന്നതാണോ എന്ന് ഉറപ്പാക്കുക.
- ചെക്കൗട്ട് ഫോം സംരക്ഷിക്കുമ്പോൾ സാധൂകരണം പരാജയപ്പെടുകയാണെങ്കിൽ, **ഉപയോക്തൃനാമം** പോലുള്ള നിർബന്ധിത Account ഫീൽഡുകൾ ചേർക്കുക.
- ഒരു ഡൊമെയിൻ തിരഞ്ഞെടുത്തിട്ടും കാർട്ട് അപ്‌ഡേറ്റ് ചെയ്യുന്നില്ലെങ്കിൽ, `window.wu_checkout_form` നിലവിലുണ്ടെന്നും Domain Seller ചെക്കൗട്ട് ആസറ്റുകൾ ചെക്കൗട്ട് പേജിൽ ലോഡ് ചെയ്തിട്ടുണ്ടെന്നും ഉറപ്പാക്കുക.
