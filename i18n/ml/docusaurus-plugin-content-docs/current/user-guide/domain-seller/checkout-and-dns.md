# ചെക്ക്ഔട്ട് ഫീൽഡ്, കസ്റ്റമർ DNS മാനേജ്‌മെന്റ്

## ഡൊമെയ്ൻ സെലക്ഷൻ ചെക്ക്ഔട്ട് ഫീൽഡ്

**Domain Selection** ഫീൽഡ് എന്നത് ഒരു ചെക്ക്ഔട്ട് ഘടകമാണ്. ഇത് ഉപഭോക്താവിന് അവരുടെ സൈറ്റിന്റെ ഡൊമെയ്ൻ എങ്ങനെ ലഭിക്കണം എന്നതിനെക്കുറിച്ച് തിരഞ്ഞെടുക്കാൻ അവസരം നൽകുന്നു. ഡൊമെയ്ൻ വിൽക്കാനുള്ള സൗകര്യം നൽകാൻ ഇത് ഏത് ചെക്ക്ഔട്ട് ഫോമിലും ചേർക്കാവുന്നതാണ്.

### ചെക്ക്ഔട്ട് ഫോമിലേക്ക് ഫീൽഡ് ചേർക്കൽ

1. **Network Admin › Ultimate Multisite › Checkout Forms** എന്നതിലേക്ക് പോകുക.
2. ഒരു ചെക്ക്ഔട്ട് ഫോം തുറക്കുക അല്ലെങ്കിൽ പുതിയത് ഉണ്ടാക്കുക.
3. ചെക്ക്ഔട്ട് എഡിറ്ററിൽ, **Add Field** ക്ലിക്ക് ചെയ്യുക.
4. ഫീൽഡ് ലിസ്റ്റിൽ നിന്ന് **Domain Selection** തിരഞ്ഞെടുക്കുക.
5. ഫീൽഡ് ഓപ്ഷനുകൾ സജ്ജമാക്കുക (താഴെ കാണുക).
6. ഫോം സേവ് ചെയ്യുക.

### ഫീൽഡ് ഓപ്ഷനുകൾ

**Domain modes** — ഉപഭോക്താവിന് ഏത് ടാബുകളാണ് കാണാൻ കഴിയേണ്ടതെന്ന് ഇവിടെ തിരഞ്ഞെടുക്കാം. ഓരോ മോഡും സ്വതന്ത്രമായി പ്രവർത്തനക്ഷമമാക്കുകയോ പ്രവർത്തനരഹിതമാക്കുകയോ ചെയ്യാം:

| Mode | എന്തുചെയ്യുന്നു |
|---|---|
| **Subdomain** | ഉപഭോക്താവ് നിങ്ങളുടെ നെറ്റ്‌വർക്കിലെ ഒരു സൗജന്യ സബ്ഡൊമെയ്ൻ ഉപയോഗിക്കുന്നു (ഉദാഹരണത്തിന്, `mysite.yournetwork.com`). പണം നൽകേണ്ട ആവശ്യമില്ല. |
| **Register New Domain** | ഉപഭോക്താവ് ഒരു പുതിയ ഡൊമെയ്ൻ തിരയുകയും അത് നിങ്ങളുടെ കോൺഫിഗർ ചെയ്ത പ്രൊവൈഡർ വഴി രജിസ്റ്റർ ചെയ്യുകയും ചെയ്യുന്നു. വിലനിർണ്ണയത്തിനായി മാച്ചിംഗ് ഡൊമെയ്ൻ ഉൽപ്പന്നം ഉപയോഗിക്കുന്നു. |
| **Existing Domain** | ഉപഭോക്താവ് തങ്ങൾ സ്വന്തമായി ഉപയോഗിക്കുന്ന ഒരു ഡൊമെയ്ൻ മാപ്പ് ചെയ്യുന്നു. രജിസ്ട്രേഷൻ ഫീസ് ഇല്ല. ഈ ഡൊമെയ്ൻ അവരുടെ സൈറ്റിലേക്ക് ഓട്ടോമാറ്റിക്കായി മാപ്പ് ചെയ്യപ്പെടും. |

**Default mode** — മൂന്ന് മോഡുകളും പ്രവർത്തനക്ഷമമാക്കിയാൽ, ഏത് ടാബാണ് ആദ്യം തുറക്കുക എന്നത് ഇവിടെ തീരുമാനിക്കാം. ഡൊമെയ്ൻ രജിസ്ട്രേഷൻ ഓപ്ഷണലായി നിലനിർത്താൻ **Subdomain** എന്ന് സജ്ജമാക്കുക, അല്ലെങ്കിൽ വാങ്ങലുകൾ പ്രോത്സാഹിപ്പിക്കാൻ **Register New Domain** എന്ന് സജ്ജമാക്കുക.

**Domain product** — ആവശ്യമെങ്കിൽ ഈ ഫീൽഡ് ഒരു പ്രത്യേക ഡൊമെയ്ൻ ഉൽപ്പന്നവുമായി ബന്ധിപ്പിക്കാം. ഇത് സജ്ജമാക്കാത്തപക്ഷം, ഉപഭോക്താവ് തിരയുന്ന TLD അനുസരിച്ച് ആഡ്-ഓൺ ഓട്ടോമാറ്റിക്കായി മാച്ചിംഗ് ഉൽപ്പന്നം തിരഞ്ഞെടുക്കും.

### രജിസ്ട്രന്റ് കോൺടാക്റ്റ് ഫീൽഡുകൾ

ഒരു ഉപഭോക്താവ് **Register New Domain** ടാബ് തിരഞ്ഞെടുക്കുമ്പോൾ, ചെക്ക്ഔട്ട് ഫോം ഇൻലൈനിൽ രജിസ്ട്രന്റ് കോൺടാക്റ്റ് ഫീൽഡുകൾ ചേർക്കുന്നു:

- First name / Last name
- Email address
- Address (line 1, city, state/province, postal code, country)
- Phone number

ഇവ എല്ലാ രജിസ്ട്രർമാർക്കും നിർബന്ധമാണ്. രജിസ്ട്രേഷൻ API കോൾ ചെയ്യുന്നതിന് മുമ്പ് ഇവ സാധൂകരിക്കുകയും (validate) ചെയ്യും. ഫോൺ നമ്പറുകൾ രജിസ്ട്രർമാർ പ്രതീക്ഷിക്കുന്ന `+CC.NNN` എന്ന അന്താരാഷ്ട്ര ഫോർമാറ്റിലേക്ക് ഓട്ടോമാറ്റിക്കായി ഫോർമാറ്റ് ചെയ്യപ്പെടും.

### ഓട്ടോ-ജനറേറ്റഡ് സൈറ്റ് URL

ഒരു ഉപഭോക്താവ് ഒരു ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുകയോ മാപ്പ് ചെയ്യുകയോ ചെയ്യുമ്പോൾ, തിരഞ്ഞെടുക്കപ്പെട്ട ഡൊമെയ്നിൽ നിന്ന് സൈറ്റ് URL ഫീൽഡ് ഓട്ടോമാറ്റിക്കായി പൂരിപ്പിക്കപ്പെടും. ഉപഭോക്താക്കൾക്ക് പ്രത്യേക URL ഫീൽഡ് പൂരിപ്പിക്കേണ്ട ആവശ്യമില്ല.

### തിരയൽ സ്വഭാവം (Search behaviour)

- ഉപഭോക്താവ് ടൈപ്പ് ചെയ്യുമ്പോൾ AJAX ഉപയോഗിച്ച് ഡൊമെയ്ൻ ലഭ്യത തത്സമയം പരിശോധിക്കപ്പെടുന്നു.
- ഇഷ്ടപ്പെട്ട ഡൊമെയ്ൻ ലഭ്യമല്ലെങ്കിൽ, ബദൽ TLD നിർദ്ദേശങ്ങൾ കാണിക്കുന്നു.
- വിലനിർണ്ണയം തത്സമയം ശേഖരിക്കുകയും വ്യക്തമായി പ്രദർശിപ്പിക്കുകയും ചെയ്യുന്നു (രജിസ്ട്രേഷൻ വില, പുതുക്കൽ വില, ഓപ്ഷണൽ WHOIS പ്രൈവസി ഫീസ്).
- കൂപ്പൺ കോഡുകൾ മറ്റ് ഏതൊരു ഉൽപ്പന്നത്തെപ്പോലെ തന്നെ ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾക്കും ബാധകമാകും.

**തിരയൽ പ്രതികരണശേഷി മെച്ചപ്പെടുത്തൽ (Tuning search responsiveness):**

```php
// API കോളുകൾ കുറയ്ക്കുന്നതിനായി debounce delay (മില്ലിസെക്കൻഡ്) വർദ്ധിപ്പിക്കുക
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ഡൊമെയ്ൻ തിരയൽ ഫോമിലേക്ക് കസ്റ്റം ഫീൽഡുകൾ ചേർക്കൽ:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## കസ്റ്റമർ DNS മാനേജ്‌മെന്റ്

രജിസ്റ്റർ ചെയ്ത ഡൊമെയ്‌നുകളുടെ DNS റെക്കോർഡുകൾ ഉപഭോക്താവിന് **My Account** പേജിൽ, അവരുടെ ഡൊമെയ്ൻ എൻട്രിക്ക് കീഴിൽ നിന്ന് മാനേജ് ചെയ്യാൻ കഴിയും.

### പിന്തുണയ്ക്കുന്ന റെക്കോർഡ് തരങ്ങൾ

| Type | ഉപയോഗം |
|---|---|
| **A** | ഹോസ്റ്റ്നെയിમને IPv4 വിലാസവുമായി മാപ്പ് ചെയ്യുന്നു |
| **AAAA** | ഹോസ്റ്റ്നെയിમને IPv6 വിലാസവുമായി മാപ്പ് ചെയ്യുന്നു |
| **CNAME** | മറ്റൊരു ഹോസ്റ്റ്നെയിമിനെ ചൂണ്ടിക്കാണിക്കുന്ന ഒരു ആലിസായ (alias) സൃഷ്ടിക്കുന്നു |
| **MX** | മെയിൽ എക്സ്ചേഞ്ച് സെർവർ സജ്ജമാക്കുന്നു |
| **TXT** | SPF, DMARC, വെരിഫിക്കേഷൻ അല്ലെങ്കിൽ മറ്റ് ടെക്സ്റ്റ് റെക്കോർഡുകൾ ചേർക്കുന്നു |

### ഏത് പ്രൊവൈഡർമാരാണ് DNS മാനേജ്‌മെന്റ് പിന്തുണയ്ക്കുന്നത്?

DNS മാനേജ്‌മെന്റ് (റെക്കോർഡുകൾ ചേർക്കുക, എഡിറ്റ് ചെയ്യുക, ഡിലീറ്റ് ചെയ്യുക) **OpenSRS**, **ResellerClub**, **Enom** എന്നിവയിൽ ലഭ്യമാണ്. Namecheap, GoDaddy, NameSilo ഡൊമെയ്‌നുകൾ സ്റ്റാറ്റസ്, കാലഹരണപ്പെട്ട വിവരങ്ങൾ എന്നിവ കാണിക്കുന്നുണ്ടെങ്കിലും, DNS റെക്കോർഡുകൾ രജിസ്ട്രറുടെ കൺട്രോൾ പാനലിൽ നേരിട്ട് മാനേജ് ചെയ്യേണ്ടതുണ്ട്.

### ഡിഫോൾട്ട് DNS റെക്കോർഡുകൾ

ഒരു ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുമ്പോൾ ഓട്ടോമാറ്റിക്കായി લાગુമാകുന്ന ഡിഫോൾട്ട് DNS റെക്കോർഡുകൾ നിങ്ങൾക്ക് സജ്ജമാക്കാൻ കഴിയും. **Settings › Domain Seller › Default DNS Records** എന്നതിലേക്ക് പോകുക.

ഡിഫോൾട്ട് റെക്കോർഡ് മൂല്യങ്ങൾ രണ്ട് ടോക്കണുകൾ പിന്തുണയ്ക്കുന്നു:

| Token | എന്തിനാക്കി മാറ്റിസ്ഥാപിക്കപ്പെടുന്നു |
|---|---|
| `{DOMAIN}` | രജിസ്റ്റർ ചെയ്ത ഡൊമെയ്ൻ നാമം (ഉദാഹരണത്തിന്, `example.com`) |
| `{SITE_URL}` | ഉപഭോക്താവിന്റെ സൈറ്റിനായുള്ള WordPress സൈറ്റ് URL |

**ഉദാഹരണം — ഏപ്‌ക്സ് ഡൊമെയ്‌നையும் www-യെയും നിങ്ങളുടെ സെർവർ IP-യിലേക്ക് പോയിന്റ് ചെയ്യുക:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### അഡ്മിൻ: DNS കാണുന്നതും എഡിറ്റ് ചെയ്യുന്നതും

**Network Admin › Ultimate Multisite › Domains** എന്നതിലെ ഡൊമെയ്ൻ എഡിറ്റ് പേജിൽ നിന്ന് നെറ്റ്‌വർക്ക് അഡ്മിന് ഏത് ഉപഭോക്താവിന്റെ ഡൊമെയ്‌നുകളുടെയും DNS റെക്കോർഡുകൾ കാണാനും എഡിറ്റ് ചെയ്യാനും കഴിയും.
