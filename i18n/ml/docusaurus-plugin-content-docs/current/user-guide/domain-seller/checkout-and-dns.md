# ചെക്ക്ഔട്ട് ഫീൽഡും ഉപഭോക്തൃ DNS മാനേജ്മെന്റും

## ഡൊമെയ്ൻ തിരഞ്ഞെടുക്കൽ ചെക്ക്ഔട്ട് ഫീൽഡ്

**ഡൊമെയ്ൻ തിരഞ്ഞെടുക്കൽ** ഫീൽഡ് ഉപഭോക്താക്കൾക്ക് അവരുടെ സൈറ്റിന്റെ ഡൊമെയ്ൻ എങ്ങനെ നേടാമെന്ന് തിരഞ്ഞെടുക്കാൻ അവസരം നൽകുന്ന ഒരു ചെക്ക്ഔട്ട് ഘടകമാണ്. ഡൊമെയ്ൻ വിൽപ്പന സജീവമാക്കാൻ ഇത് ഏതെങ്കിലും ചെക്ക്ഔട്ട് ഫോമിൽ ചേർക്കുക.

### ചെക്ക്ഔട്ട് ഫോമിലേക്ക് ഫീൽഡ് ചേർക്കൽ

1. **നെറ്റ്‌വർക്ക് അഡ്മിൻ › Ultimate Multisite › ചെക്ക്ഔട്ട് ഫോമുകൾ** എന്നതിലേക്ക് പോകുക
2. ഒരു ചെക്ക്ഔട്ട് ഫോം തുറക്കുക അല്ലെങ്കിൽ സൃഷ്ടിക്കുക
3. ചെക്ക്ഔട്ട് എഡിറ്ററിൽ **ഫീൽഡ് ചേർക്കുക** ക്ലിക്ക് ചെയ്യുക
4. ഫീൽഡ് ലിസ്റ്റിൽ നിന്ന് **ഡൊമെയ്ൻ തിരഞ്ഞെടുക്കൽ** തിരഞ്ഞെടുക്കുക
5. ഫീൽഡ് ഓപ്ഷനുകൾ ക്രമീകരിക്കുക (താഴെ കാണുക)
6. ഫോം സംരക്ഷിക്കുക

### ഫീൽഡ് ഓപ്ഷനുകൾ

**ഡൊമെയ്ൻ മോഡുകൾ** — ഉപഭോക്താവ് കാണുന്ന ടാബുകൾ തിരഞ്ഞെടുക്കുക. ഓരോ മോഡും വേർതിരിച്ച് സജീവമാക്കുകയോ നിർജ്ജീവമാക്കുകയോ ചെയ്യാം:

| മോഡ് | ഇത് ചെയ്യുന്നത് |
|---|---|
| **സബ്ഡൊമെയ്ൻ** | ഉപഭോക്താവ് നിങ്ങളുടെ നെറ്റ്‌വർക്കിലെ സൗജന്യ സബ്ഡൊമെയ്ൻ ഉപയോഗിക്കുന്നു (ഉദാ., `mysite.yournetwork.com`). പേയ്‌മെന്റ് ആവശ്യമില്ല. |
| **പുതിയ ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുക** | ഉപഭോക്താവ് പുതിയ ഡൊമെയ്ൻ തിരയുകയും നിങ്ങൾ ക്രമീകരിച്ച പ്രൊവൈഡർ വഴി അത് രജിസ്റ്റർ ചെയ്യുകയും ചെയ്യുന്നു. വിലനിർണ്ണയത്തിന് പൊരുത്തപ്പെടുന്ന ഡൊമെയ്ൻ ഉൽപ്പന്നം ഉപയോഗിക്കുന്നു. |
| **നിലവിലുള്ള ഡൊമെയ്ൻ** | ഉപഭോക്താവ് ഇതിനകം ഉടമസ്ഥതയിലുള്ള ഒരു ഡൊമെയ്ൻ മാപ്പ് ചെയ്യുന്നു. രജിസ്ട്രേഷൻ ഫീസ് ഇല്ല. ഡൊമെയ്ൻ സ്വയമേവ അവരുടെ സൈറ്റിലേക്ക് മാപ്പ് ചെയ്യും. |

**ഡിഫോൾട്ട് മോഡ്** — മൂന്ന് മോഡുകളും സജീവമാക്കിയിരിക്കുമ്പോൾ ആദ്യം തുറക്കുന്ന ടാബ്. ഡൊമെയ്ൻ രജിസ്ട്രേഷൻ ഐച്ഛികമായി നിലനിർത്താൻ **സബ്ഡൊമെയ്ൻ** ആയി സജ്ജമാക്കുക, അല്ലെങ്കിൽ വാങ്ങലുകൾ പ്രോത്സാഹിപ്പിക്കാൻ **പുതിയ ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുക** ആയി സജ്ജമാക്കുക.

**ഡൊമെയ്ൻ ഉൽപ്പന്നം** — ആവശ്യമെങ്കിൽ ഈ ഫീൽഡ് ഒരു നിർദ്ദിഷ്ട ഡൊമെയ്ൻ ഉൽപ്പന്നത്തിലേക്ക് പിൻ ചെയ്യുക. സജ്ജമാക്കിയിട്ടില്ലെങ്കിൽ, ഉപഭോക്താവ് തിരയുന്ന TLD അടിസ്ഥാനമാക്കി addon സ്വയമേവ പൊരുത്തപ്പെടുന്ന ഉൽപ്പന്നം തിരഞ്ഞെടുക്കും.

### രജിസ്ട്രന്റ് ബന്ധപ്പെടൽ ഫീൽഡുകൾ

ഉപഭോക്താവ് **പുതിയ ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുക** ടാബ് തിരഞ്ഞെടുക്കുമ്പോൾ, ചെക്ക്ഔട്ട് ഫോം രജിസ്ട്രന്റ് ബന്ധപ്പെടൽ ഫീൽഡുകൾ അതിനുള്ളിൽ ചേർക്കും:

- പേര് / കുടുംബപ്പേര്
- ഇമെയിൽ വിലാസം
- വിലാസം (വരി 1, നഗരം, സംസ്ഥാനം/പ്രവിശ്യ, പോസ്റ്റൽ കോഡ്, രാജ്യം)
- ഫോൺ നമ്പർ

എല്ലാ രജിസ്ട്രാർമാർക്കും ഇവ ആവശ്യമാണ്. രജിസ്ട്രേഷൻ API കോൾ നടത്തുന്നതിന് മുമ്പ് ഇവ സാധൂകരിക്കും. രജിസ്ട്രാർമാർ പ്രതീക്ഷിക്കുന്ന `+CC.NNN` അന്താരാഷ്ട്ര ഫോർമാറ്റിലേക്ക് ഫോൺ നമ്പറുകൾ സ്വയമേവ ഫോർമാറ്റ് ചെയ്യും.

### സ്വയമേവ സൃഷ്ടിക്കുന്ന സൈറ്റ് URL

ഉപഭോക്താവ് ഒരു ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുകയോ മാപ്പ് ചെയ്യുകയോ ചെയ്യുമ്പോൾ, തിരഞ്ഞെടുത്ത ഡൊമെയ്ൻ ഉപയോഗിച്ച് സൈറ്റ് URL ഫീൽഡ് സ്വയമേവ പൂരിപ്പിക്കും. ഉപഭോക്താക്കൾ പ്രത്യേക URL ഫീൽഡ് പൂരിപ്പിക്കേണ്ടതില്ല.

### തിരയൽ പെരുമാറ്റം

- ഉപഭോക്താവ് ടൈപ്പ് ചെയ്യുമ്പോൾ AJAX ഉപയോഗിച്ച് ഡൊമെയ്ൻ ലഭ്യത തത്സമയം പരിശോധിക്കും
- ഇഷ്ടപ്പെട്ട ഡൊമെയ്ൻ ലഭ്യമല്ലെങ്കിൽ പകരം TLD നിർദ്ദേശങ്ങൾ കാണിക്കും
- വില തത്സമയം ലഭ്യമാക്കി വ്യക്തമായി പ്രദർശിപ്പിക്കും (രജിസ്ട്രേഷൻ വില, പുതുക്കൽ വില, ഐച്ഛിക WHOIS സ്വകാര്യത ഫീസ്)
- മറ്റ് ഏത് ഉൽപ്പന്നത്തെയും പോലെ ഡൊമെയ്ൻ ഉൽപ്പന്നങ്ങൾക്കും കൂപ്പൺ കോഡുകൾ ബാധകമാണ്

**തിരയൽ പ്രതികരണക്ഷമത ക്രമീകരിക്കൽ:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ഡൊമെയ്ൻ തിരയൽ ഫോമിലേക്ക് ഇഷ്ടാനുസൃത ഫീൽഡുകൾ ചേർക്കൽ:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ഉപഭോക്തൃ DNS മാനേജ്മെന്റ്

ഉപഭോക്താക്കൾക്ക് **എന്റെ Account** പേജിൽ, അവരുടെ ഡൊമെയ്ൻ എൻട്രിക്ക് കീഴിൽ, രജിസ്റ്റർ ചെയ്ത ഡൊമെയ്‌നുകളുടെ DNS റെക്കോർഡുകൾ മാനേജ് ചെയ്യാം.

### പിന്തുണയ്ക്കുന്ന റെക്കോർഡ് തരങ്ങൾ

| തരം | ഉപയോഗം |
|---|---|
| **A** | ഹോസ്റ്റ്‌നെയിം IPv4 വിലാസത്തിലേക്ക് മാപ്പ് ചെയ്യുക |
| **AAAA** | ഹോസ്റ്റ്‌നെയിം IPv6 വിലാസത്തിലേക്ക് മാപ്പ് ചെയ്യുക |
| **CNAME** | മറ്റൊരു ഹോസ്റ്റ്‌നെയിമിലേക്ക് ചൂണ്ടുന്ന ഒരു അപരനാമം സൃഷ്ടിക്കുക |
| **MX** | മെയിൽ എക്സ്ചേഞ്ച് സെർവർ സജ്ജമാക്കുക |
| **TXT** | SPF, DMARC, സ്ഥിരീകരണം, അല്ലെങ്കിൽ മറ്റ് ടെക്സ്റ്റ് റെക്കോർഡുകൾ ചേർക്കുക |

### DNS മാനേജ്മെന്റിനെ ഏത് പ്രൊവൈഡർമാർ പിന്തുണയ്ക്കുന്നു?

DNS മാനേജ്മെന്റ് (റെക്കോർഡുകൾ ചേർക്കൽ, എഡിറ്റ് ചെയ്യൽ, നീക്കംചെയ്യൽ) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, **Openprovider** എന്നിവയിൽ ലഭ്യമാണ്. **Hostinger** ഡൊമെയ്‌നുകൾക്ക് Domain Seller വഴി നെയിംസെർവറുകൾ അപ്ഡേറ്റ് ചെയ്യാം; ഹോസ്റ്റ് ചെയ്ത ഡൊമെയ്‌നുകളുടെ DNS റെക്കോർഡുകൾ പ്രധാന Hostinger ഡൊമെയ്ൻ-മാപ്പിംഗ് ഇന്റഗ്രേഷൻ ആണ് മാനേജ് ചെയ്യുന്നത്. Namecheap, GoDaddy, NameSilo ഡൊമെയ്‌നുകൾ സ്റ്റാറ്റസും കാലഹരണ തീയതിയും കാണിക്കും, പക്ഷേ DNS രജിസ്ട്രാറിന്റെ കൺട്രോൾ പാനലിൽ നേരിട്ട് മാനേജ് ചെയ്യണം.

### ഡിഫോൾട്ട് DNS റെക്കോർഡുകൾ

ഒരു ഡൊമെയ്ൻ രജിസ്റ്റർ ചെയ്യുമ്പോൾ സ്വയമേവ പ്രയോഗിക്കുന്ന ഡിഫോൾട്ട് DNS റെക്കോർഡുകൾ നിങ്ങൾക്ക് ക്രമീകരിക്കാം. **ക്രമീകരണങ്ങൾ › Domain Seller › ഡിഫോൾട്ട് DNS റെക്കോർഡുകൾ** എന്നതിലേക്ക് പോകുക.

ഡിഫോൾട്ട് റെക്കോർഡ് മൂല്യങ്ങൾ രണ്ട് ടോക്കണുകൾ പിന്തുണയ്ക്കുന്നു:

| ടോക്കൺ | ഇതുകൊണ്ട് മാറ്റിസ്ഥാപിക്കും |
|---|---|
| `{DOMAIN}` | രജിസ്റ്റർ ചെയ്ത ഡൊമെയ്ൻ പേര് (ഉദാ., `example.com`) |
| `{SITE_URL}` | ഉപഭോക്താവിന്റെ സൈറ്റിനുള്ള WordPress സൈറ്റ് URL |

**ഉദാഹരണം — അപെക്സ് ഡൊമെയ്‌നും www-യും നിങ്ങളുടെ സെർവർ IP-യിലേക്ക് ചൂണ്ടുക:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### അഡ്മിൻ: DNS കാണലും എഡിറ്റ് ചെയ്യലും

നെറ്റ്‌വർക്ക് അഡ്മിൻമാർക്ക് ഏത് ഉപഭോക്തൃ ഡൊമെയ്‌നിന്റെയും DNS റെക്കോർഡുകൾ **നെറ്റ്‌വർക്ക് അഡ്മിൻ › Ultimate Multisite › ഡൊമെയ്‌നുകൾ** എന്നതിലെ ഡൊമെയ്ൻ എഡിറ്റ് പേജിൽ നിന്ന് കാണാനും എഡിറ്റ് ചെയ്യാനും കഴിയും.
