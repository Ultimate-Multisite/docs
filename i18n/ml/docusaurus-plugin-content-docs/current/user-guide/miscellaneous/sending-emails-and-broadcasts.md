# ഇമെയിലുകളും ബ്രോഡ്കാസ്റ്റുകളും അയക്കുന്നത് (v2)

_**പ്രധാന കുറിപ്പ്: ഈ ലേഖനം Ultimate Multisite 2.x പതിപ്പിനെ കുറിച്ചാണ്.**_

Ultimate Multisite-ൽ ഒരു പ്രത്യേക ഉപയോക്താവിലേക്കോ ഒരു കൂട്ടം ഉപയോക്താക്കളിലേക്കോ ഇമെയിൽ അയച്ചും അവരുടെ admin dashboard-ൽ അറിയിപ്പുകൾ പ്രദർശിപ്പിച്ചും നിങ്ങളുടെ ഉപഭോക്താക്കളുമായി ആശയവിനിമയം നടത്താൻ സഹായിക്കുന്ന ഒരു ഫീച്ചർ ഉണ്ട്.

## ബ്രോഡ്കാസ്റ്റുകൾ ഉപയോഗിച്ച് നിങ്ങളുടെ ഉപഭോക്താക്കളുടെ dashboard-ൽ admin notices ചേർക്കുക

Ultimate Multisite broadcast ഫീച്ചർ ഉപയോഗിച്ച്, നിങ്ങളുടെ ഉപയോക്താവിന്റെ subsite admin dashboard-ൽ **admin notices** ചേർക്കാൻ സാധിക്കും.

സിസ്റ്റം മെയിന്റനൻസ് പോലുള്ള അറിയിപ്പുകൾ നൽകാനോ നിലവിലുള്ള ഉപയോക്താക്കൾക്ക് പുതിയ ഉൽപ്പന്നങ്ങളോ സേവനങ്ങളോ വാഗ്ദാനം ചെയ്യാനോ ഇത് വളരെ ഉപകാരപ്രദമാണ്. നിങ്ങളുടെ ഉപയോക്താവിന്റെ dashboard-ൽ admin notice ഇങ്ങനെയാണ് കാണപ്പെടുക.

![ഉപഭോക്താവിന്റെ dashboard-ൽ കാണിക്കുന്ന Admin notice broadcast](/img/admin/broadcasts-list.png)

Admin notice ആരംഭിക്കാൻ, നിങ്ങളുടെ network admin dashboard-ലേക്ക് പോയി **Ultimate Multisite** മെനുവിന് കീഴിൽ **Broadcasts** ഓപ്ഷൻ കാണാം.

![Ultimate Multisite admin-ലെ Broadcasts മെനു](/img/admin/broadcasts-list.png)

ഈ പേജിൽ നിന്ന്, മുകളിലുള്ള **Add Broadcast** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

ഇത് Add broadcast modal window തുറക്കും, അവിടെ നിങ്ങൾക്ക് ഏത് തരം broadcast അയക്കണമെന്ന് തിരഞ്ഞെടുക്കാം.

**Message** സെലക്ട് ചെയ്ത് **Next Step** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

![Message type സെലക്ട് ചെയ്ത Add broadcast modal](/img/admin/broadcasts-list.png)

അടുത്ത വിൻഡോയിൽ **Target customer** അല്ലെങ്കിൽ **Target product** ചോദിക്കും. ഒന്നിലധികം ഉപയോക്താക്കളെയോ ഉൽപ്പന്നങ്ങളെയോ തിരഞ്ഞെടുക്കാമെന്ന് ഓർക്കുക.

ഒരു ഉപയോക്തൃ അക്കൗണ്ടോ ഉൽപ്പന്നമോ തിരയാൻ ഫീൽഡിനുള്ളിൽ കീവേഡ് ടൈപ്പ് ചെയ്യാൻ തുടങ്ങുക.

**Message type** ഫീൽഡിന് കീഴിൽ, notice-ന്റെ നിറം തിരഞ്ഞെടുക്കാം. ഇത് നിങ്ങളുടെ സന്ദേശത്തിന്റെ അടിയന്തിരത എടുത്തുകാണിക്കും.

അതിനുശേഷം **Next Step** ക്ലിക്ക് ചെയ്യാം.

![Broadcast-നുള്ള Target customer, product തിരഞ്ഞെടുക്കൽ](/img/admin/broadcasts-list.png)

അടുത്ത വിൻഡോയിൽ subject-ഉം ഉപയോക്താക്കൾക്ക് broadcast ചെയ്യേണ്ട content/message-ഉം നൽകി നിങ്ങളുടെ സന്ദേശം രചിക്കാം.

![Broadcast message subject, content editor](/img/admin/broadcasts-list.png)

സന്ദേശം തയ്യാറാക്കിയ ശേഷം, **Send** ബട്ടൺ അമർത്തുക.

അത്രമാത്രം. Admin notice ഉടൻ തന്നെ നിങ്ങളുടെ ഉപയോക്താവിന്റെ dashboard-ൽ കാണിക്കും.

## നിങ്ങളുടെ ഉപഭോക്താക്കൾക്ക് ഇമെയിലുകൾ അയക്കുക

Ultimate Multisite broadcast ഫീച്ചർ ഉപയോഗിച്ച്, നിങ്ങളുടെ ഉപയോക്താക്കൾക്ക് ഇമെയിൽ അയക്കാം. പ്രത്യേക ഉപയോക്താക്കൾക്ക് മാത്രം ഇമെയിൽ അയക്കാനോ അവർ സബ്സ്ക്രൈബ് ചെയ്ത product അല്ലെങ്കിൽ plan അടിസ്ഥാനമാക്കി ഒരു പ്രത്യേക ഉപയോക്തൃ ഗ്രൂപ്പിനെ ടാർഗറ്റ് ചെയ്യാനോ ഉള്ള ഓപ്ഷനുണ്ട്.

ഒരു email broadcast ആരംഭിക്കാൻ, നിങ്ങളുടെ network admin dashboard-ലേക്ക് പോയി Ultimate Multisite മെനുവിന് കീഴിൽ Broadcast ഓപ്ഷൻ കാണാം.

![Ultimate Multisite admin-ലെ Broadcasts പേജ്](/img/admin/broadcasts-list.png)

ഈ പേജിൽ നിന്ന്, മുകളിലുള്ള **Add broadcast** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

ഇത് Add broadcast modal window തുറക്കും, അവിടെ നിങ്ങൾക്ക് ഏത് തരം broadcast അയക്കണമെന്ന് തിരഞ്ഞെടുക്കാം. **Email** സെലക്ട് ചെയ്ത് **Next Step** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

![Email type സെലക്ട് ചെയ്ത Add broadcast modal](/img/admin/broadcasts-list.png)

അടുത്ത വിൻഡോയിൽ **Target customer** അല്ലെങ്കിൽ **Target product** ചോദിക്കും. ഒന്നിലധികം ഉപയോക്താക്കളെയോ ഉൽപ്പന്നങ്ങളെയോ തിരഞ്ഞെടുക്കാമെന്ന് ഓർക്കുക.

ഒരു ഉപയോക്തൃ അക്കൗണ്ടോ ഉൽപ്പന്നമോ തിരയാൻ ഫീൽഡിനുള്ളിൽ കീവേഡ് ടൈപ്പ് ചെയ്യാൻ തുടങ്ങുക.

ടാർഗറ്റ് ഓഡിയൻസ് തിരഞ്ഞെടുത്തുകഴിഞ്ഞാൽ, **Next Step** ക്ലിക്ക് ചെയ്യാം.

![Email broadcast-നുള്ള Target customer, product തിരഞ്ഞെടുക്കൽ](/img/admin/broadcasts-list.png)

അടുത്ത വിൻഡോയിൽ subject-ഉം ഉപയോക്താക്കൾക്ക് അയക്കേണ്ട content/message-ഉം നൽകി നിങ്ങളുടെ ഇമെയിൽ രചിക്കാം.

![Email broadcast subject, content editor](/img/admin/broadcasts-list.png)

സന്ദേശം തയ്യാറാക്കിയ ശേഷം, **Send** ബട്ടൺ അമർത്തുക.

Broadcast ഫീച്ചർ ഉപയോഗിച്ച് നിങ്ങളുടെ എൻഡ്-യൂസേഴ്സിലേക്ക് ഇമെയിൽ അയക്കുന്നത് ഇത്ര എളുപ്പമാണ്.

## സിസ്റ്റം ഇമെയിലുകൾ

Ultimate Multisite-ലെ സിസ്റ്റം ഇമെയിലുകൾ എന്നത് registration, payment, domain mapping തുടങ്ങിയ ചില പ്രവർത്തനങ്ങൾക്ക് ശേഷം സിസ്റ്റം അയക്കുന്ന **ഓട്ടോമാറ്റിക് അറിയിപ്പുകളാണ്**. ഈ ഇമെയിലുകൾ Ultimate Multisite settings-ൽ നിന്ന് എഡിറ്റ് ചെയ്യാനോ മാറ്റം വരുത്താനോ കഴിയും. മറ്റൊരു Ultimate Multisite installation-ൽ നിന്ന് നിലവിലുള്ള settings reset ചെയ്യാനും import ചെയ്യാനും അനുവദിക്കുന്ന ഒരു ഫീച്ചറും ഇതിനൊപ്പമുണ്ട്.

### Reset ചെയ്യലും Import ചെയ്യലും

പുതിയ Ultimate Multisite versions-ഉം add-ons-ഉം ഇടയ്ക്കിടെ പുതിയ ഇമെയിലുകൾ register ചെയ്യാറുണ്ട്.

conflicts-ഉം മറ്റ് പ്രശ്നങ്ങളും ഒഴിവാക്കാൻ, **ഒരു പ്രത്യേക ഫീച്ചറിന്റെ ശരിയായ പ്രവർത്തനത്തിന് അത്യാവശ്യമല്ലെങ്കിൽ, പുതിയ email templates നിങ്ങളുടെ install-ൽ System Emails ആയി ഞങ്ങൾ സ്വയമേവ ചേർക്കില്ല**.

എന്നിരുന്നാലും, super admins-നും agents-നും importer tool വഴി പുതുതായി register ചെയ്ത ഇമെയിലുകൾ import ചെയ്യാം. ഈ പ്രക്രിയ പുതിയ email template-ന്റെ content-ഉം configuration-ഉം ഉള്ള ഒരു പുതിയ system email സൃഷ്ടിക്കും, super admin-ന് ആവശ്യമായ മാറ്റങ്ങൾ വരുത്താനോ അവ അതേപടി നിലനിർത്താനോ സാധിക്കും.

#### സിസ്റ്റം ഇമെയിലുകൾ എങ്ങനെ import ചെയ്യാം

നിങ്ങളുടെ Ultimate Multisite Settings പേജിലേക്ക് പോയി **Emails** tab-ലേക്ക് പോകുക.

![Ultimate Multisite settings-ലെ Emails tab](/img/config/settings-emails.png)

അതിനുശേഷം, sidebar-ൽ **Customize System Emails** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

![Sidebar-ലെ Customize System Emails ബട്ടൺ](/img/config/settings-emails.png)

System Emails പേജിൽ, മുകളിൽ **Reset & Import** action button കാണാം. ആ ബട്ടൺ ക്ലിക്ക് ചെയ്താൽ import and reset modal window തുറക്കും.

![System Emails പേജിലെ Reset and Import action button](/img/config/settings-emails.png)

അതിനുശേഷം, import ചെയ്യാൻ ലഭ്യമായ system emails ഏതൊക്കെയെന്ന് കാണാൻ Import Emails options toggle ചെയ്യാം.

![Import ചെയ്യാൻ ലഭ്യമായ system emails കാണിക്കുന്ന Import Emails options](/img/config/settings-emails.png)

#### സിസ്റ്റം ഇമെയിലുകൾ Reset ചെയ്യുന്നത്

ചിലപ്പോൾ, ഒരു email template-ൽ നിങ്ങൾ വരുത്തിയ മാറ്റങ്ങൾ ഇനി നിങ്ങൾക്ക് അനുയോജ്യമല്ലെന്ന് മനസ്സിലാകുകയും അത് **default state**-ലേക്ക് reset ചെയ്യാൻ ആഗ്രഹിക്കുകയും ചെയ്യാം.

അത്തരം സാഹചര്യങ്ങളിൽ, നിങ്ങൾക്ക് രണ്ട് ഓപ്ഷനുകളുണ്ട്: system email ഡിലീറ്റ് ചെയ്ത് (മുകളിലുള്ള നിർദ്ദേശങ്ങൾ ഉപയോഗിച്ച്) വീണ്ടും import ചെയ്യാം - ഇത് send metrics-ഉം മറ്റ് കാര്യങ്ങളും മായ്ക്കും, അതുകൊണ്ട് ഈ രീതി കുറച്ച് പ്രിയപ്പെട്ടതാണ്.

അല്ലെങ്കിൽ ആ email template reset ചെയ്യാൻ **Reset & Import tool** ഉപയോഗിക്കാം.

ഒരു email template reset ചെയ്യാൻ, Reset & Import tool-ലേക്ക് എത്തുന്നതുവരെ മുകളിലുള്ള steps പിന്തുടരുക, അതിനുശേഷം **Reset** option toggle ചെയ്ത് default content-ലേക്ക് reset ചെയ്യേണ്ട emails തിരഞ്ഞെടുക്കുക.

![Email templates defaults-ലേക്ക് restore ചെയ്യാനുള്ള Reset option](/img/config/settings-emails.png)
