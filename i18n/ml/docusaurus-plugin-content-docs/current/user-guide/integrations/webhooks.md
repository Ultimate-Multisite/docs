# Webhooks (v2) സംബന്ധിച്ച ആദ്യ പരിചയം

_**ശ്രദ്ധിക്കുക: ഈ സവിശേഷതയോ ലേഖനമോ പരിചയസമ്പന്നരായ ഉപയോക്താക്കൾക്കുള്ളതാണ്.**_

Ultimate Multisite പോലുള്ള ഒരു ആപ്പ് അല്ലെങ്കിൽ സോഫ്റ്റ്‌വെയർ മറ്റ് അപ്ലിക്കേഷനുകൾക്ക് തത്സമയ വിവരം നൽകുന്നതിനുള്ള മാർഗമാണ് **webhook**. എന്തെങ്കിലും സംഭവിക്കുന്ന നിമിഷം തന്നെ ഒരു webhook മറ്റ് അപ്ലിക്കേഷനുകളിലേക്ക് ഡാറ്റയോ പേലോഡുകളോ എത്തിക്കുന്നു. അതായത് നിങ്ങൾക്ക് **ഡാറ്റ ഉടൻ ലഭിക്കും.**

ഒരു ഇവന്റ് പ്രവർത്തനക്ഷമമാകുന്ന ഓരോ തവണയും Ultimate Multisite-ൽ നിന്ന് മറ്റൊരു CRM-ലേക്കോ സിസ്റ്റത്തിലേക്കോ ചില ഡാറ്റ സംയോജിപ്പിക്കാനോ കൈമാറാനോ വേണമെങ്കിൽ ഇത് സഹായകരമാണ്. ഉദാഹരണത്തിന്, പുതിയ ഒരു ഉപയോക്തൃ Account സൃഷ്ടിക്കുമ്പോൾ ഓരോ തവണയും ഉപയോക്താവിന്റെ പേരും ഇമെയിൽ വിലാസവും ഒരു മെയിലിംഗ് ലിസ്റ്റിലേക്ക് അയക്കേണ്ടിവരും.

## ഒരു webhook എങ്ങനെ സൃഷ്ടിക്കാം {#how-to-create-a-webhook}

ഒരു webhook സൃഷ്ടിക്കാൻ, നിങ്ങളുടെ നെറ്റ്‌വർക്ക് അഡ്മിൻ Dashboard-ലേക്ക് പോകുക. **Ultimate Multisite > Webhooks > Add New Webhook** ക്ലിക്ക് ചെയ്യുക.

![Add New Webhook ബട്ടൺ ഉള്ള ശൂന്യമായ Webhooks പട്ടിക പേജ്](/img/admin/webhooks-list-empty.png)

തുടർന്ന് നിങ്ങൾക്ക് webhook ക്രമീകരണം തിരുത്താം:

![Name, Event, URL ഫീൽഡുകൾ ഉള്ള Add New Webhook ഫോം](/img/admin/webhook-add-modal.png)

പുതിയ webhook സൃഷ്ടിക്കുമ്പോൾ **Name, URL,** **Event** പോലുള്ള വിവരങ്ങൾ നിങ്ങളോട് ചോദിക്കും. നിങ്ങളുടെ webhook-ിന് നിങ്ങൾക്ക് ഇഷ്ടമുള്ള ഏതെങ്കിലും പേര് ഉപയോഗിക്കാം. ഏറ്റവും പ്രധാനപ്പെട്ട ഫീൽഡുകൾ URL ഉം Event ഉം ആണ്.

![URL ഫീൽഡും പേലോഡ് പ്രിവ്യൂവും കാണിക്കുന്ന webhook തിരുത്തൽ ഇന്റർഫേസ്](/img/admin/webhook-url-field.png)

Ultimate Multisite **പേലോഡോ ഡാറ്റയോ** അയക്കുന്ന **അവസാനബിന്ദുവോ ലക്ഷ്യസ്ഥാനമോ** ആണ് URL. ഡാറ്റ സ്വീകരിക്കുന്ന അപ്ലിക്കേഷനാണ് ഇത്.

മൂന്നാംകക്ഷി അപ്ലിക്കേഷനുകളുമായുള്ള സംയോജനം എളുപ്പമാക്കാൻ ഉപയോക്താക്കൾ സാധാരണയായി ഉപയോഗിക്കുന്ന പരിഹാരമാണ് Zapier. Zapier പോലുള്ള ഒരു പ്ലാറ്റ്ഫോം ഇല്ലെങ്കിൽ, ഡാറ്റ പിടിച്ചെടുക്കുകയും പ്രോസസ്സ് ചെയ്യുകയും ചെയ്യുന്ന ഒരു ഇഷ്ടാനുസൃത ഫംഗ്ഷൻ നിങ്ങൾ സ്വയം സൃഷ്ടിക്കേണ്ടിവരും. **Zapier-നൊപ്പം Ultimate Multisite webhook എങ്ങനെ ഉപയോഗിക്കാം** എന്നതിനെക്കുറിച്ചുള്ള ഈ ലേഖനം കാണുക.

ഈ ലേഖനത്തിൽ, ഒരു webhook എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്നതിന്റെ അടിസ്ഥാന ആശയവും Ultimate Multisite-ൽ ലഭ്യമായ ഇവന്റുകളും നോക്കാം. [requestbin.com](https://requestbin.com/) എന്ന മൂന്നാംകക്ഷി സൈറ്റ് നാം ഉപയോഗിക്കും. കോഡിംഗ് ഒന്നും ചെയ്യാതെ ഒരു അവസാനബിന്ദു സൃഷ്ടിക്കാനും പേലോഡ് പിടിച്ചെടുക്കാനും ഈ സൈറ്റ് നമ്മെ അനുവദിക്കും. _**നിരാകരണം: ഡാറ്റ ലഭിച്ചുവെന്ന് കാണിക്കുക മാത്രമാണ് ഇത് ചെയ്യുക.**_ പേലോഡിൽ പ്രോസസ്സിംഗോ മറ്റേതെങ്കിലും തരത്തിലുള്ള നടപടിയോ നടത്തുകയില്ല.

[requestbin.com](https://requestbin.com/) ലേക്ക് പോയി Create Request Bin ക്ലിക്ക് ചെയ്യുക.

ആ ബട്ടൺ ക്ലിക്ക് ചെയ്തതിന് ശേഷം, നിങ്ങൾക്ക് ഇതിനകം Account ഉണ്ടെങ്കിൽ ലോഗിൻ ചെയ്യാനോ അല്ലെങ്കിൽ സൈൻ അപ്പ് ചെയ്യാനോ ആവശ്യപ്പെടും. നിങ്ങൾക്ക് ഇതിനകം Account ഉണ്ടെങ്കിൽ, അത് നിങ്ങളെ നേരിട്ട് അവരുടെ Dashboard-ലേക്ക് നയിക്കും. അവരുടെ Dashboard-ൽ, നിങ്ങളുടെ Ultimate Multisite webhook സൃഷ്ടിക്കുമ്പോൾ ഉപയോഗിക്കാവുന്ന അവസാനബിന്ദുവോ URL-ഓ ഉടൻ കാണാം.

URL പകർത്തി Ultimate Multisite-ലേക്ക് തിരികെ പോകുക. URL ഫീൽഡിൽ അവസാനബിന്ദു നൽകുകയും ഡ്രോപ്പ്ഡൗണിൽ നിന്ന് ഒരു ഇവന്റ് തിരഞ്ഞെടുക്കുകയും ചെയ്യുക. ഈ ഉദാഹരണത്തിൽ, നാം **Payment Received** തിരഞ്ഞെടുക്കും.

ഒരു ഉപയോക്താവ് പേയ്മെന്റ് ചെയ്യുമ്പോഴെല്ലാം ഈ ഇവന്റ് പ്രവർത്തനക്ഷമമാകും. ലഭ്യമായ എല്ലാ ഇവന്റുകളും, അവയുടെ വിവരണവും പേലോഡുകളും പേജിന്റെ അടിയിൽ നൽകിയിരിക്കുന്നു. webhook സംരക്ഷിക്കാൻ **Add New Webhook** ബട്ടൺ ക്ലിക്ക് ചെയ്യുക.

![Payment Received തിരഞ്ഞെടുത്തിരിക്കുന്ന webhook ഇവന്റ് ഡ്രോപ്പ്ഡൗൺ](/img/admin/webhook-event-picker.png)

നാം സൃഷ്ടിച്ച webhook പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് കാണാൻ, ഇപ്പോൾ അവസാനബിന്ദുവിലേക്ക് ഒരു പരീക്ഷണ ഇവന്റ് അയക്കാം. നാം സൃഷ്ടിച്ച webhook-ന്റെ കീഴിലുള്ള **Send Test Event** ക്ലിക്ക് ചെയ്താൽ ഇത് ചെയ്യാം.

![ക്രമീകരിച്ച ഒരു webhook ഉം Send Test നടപടിയും കാണിക്കുന്ന Webhooks പട്ടിക](/img/admin/webhooks-list-populated.png)

പരീക്ഷണം വിജയിച്ചതായി പറയുന്ന സ്ഥിരീകരണ ജാലകം ഇത് കാണിക്കുന്നു.

![പരീക്ഷണ പേലോഡ് അയച്ചതിന് ശേഷമുള്ള webhook പരീക്ഷണ ഇവന്റ് ഫലം](/img/admin/webhook-test-result.png)

ഇപ്പോൾ നാം _Requestbin_ സൈറ്റിലേക്ക് മടങ്ങിയാൽ, ചില പരീക്ഷണ ഡാറ്റ ഉൾപ്പെടുത്തിയ പേലോഡ് ലഭിച്ചതായി കാണാം.

webhook-കളും അവസാനബിന്ദുക്കളും എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്നതിന്റെ അടിസ്ഥാന തത്വം ഇതാണ്. നിങ്ങൾ ഒരു ഇഷ്ടാനുസൃത അവസാനബിന്ദു സൃഷ്ടിക്കുകയാണെങ്കിൽ, Ultimate Multisite-ൽ നിന്ന് നിങ്ങൾക്ക് ലഭിക്കുന്ന ഡാറ്റ പ്രോസസ്സ് ചെയ്യാൻ ഒരു ഇഷ്ടാനുസൃത ഫംഗ്ഷൻ സൃഷ്ടിക്കേണ്ടിവരും.
