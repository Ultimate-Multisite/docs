# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency तुमच्या नेटवर्कला अनेक चलनांमध्ये पेमेंट स्वीकारण्यास सक्षम करते. हे आपोआप लाईव्ह किंवा मॅन्युअल एक्सचेंज रेट्स वापरून किंमती रूपांतरित करते, तुम्हाला कोणत्याही चलनासाठी उत्पादनांच्या अचूक किंमती सेट करण्याची मुभा देते, आणि भू-स्थान (geolocation) वापरून प्रत्येक भेट देणाऱ्या व्यक्तीचे पसंतीचे चलन ओळखते — हे सर्व एकाच ॲडमिन डॅशबोर्डमधून व्यवस्थापित केले जाते.

## मुख्य वैशिष्ट्ये (Key Features) {#key-features}

- **स्वयंचलित एक्सचेंज रेट रूपांतरण (Automatic exchange rate conversion)** — तुमच्या बेस चलनातील किंमती तुमच्या निवडलेल्या प्रदात्याकडून (provider) मिळालेल्या रेट्स वापरून रूपांतरित केल्या जातात.
- **उत्पादनानुसार किंमत बदलणे (Per-product pricing overrides)** — स्वयंचलित रूपांतरण टाळण्यासाठी कोणत्याही उत्पादनावर चलन-विशिष्ट अचूक किंमती सेट करा.
- **चलन निवडक साइनअप फील्ड (Currency selector signup field)** — ग्राहकांना चेकआउट/साइनअप दरम्यान त्यांचे पसंतीचे चलन निवडण्याची सुविधा देते.
- **अनेक एक्सचेंज रेट प्रदाते (Multiple exchange rate providers)** — ECB, Frankfurter, Stripe, किंवा पूर्णपणे मॅन्युअल रेट्स.
- **भू-स्थान-आधारित चलन ओळख (Geolocation-based currency detection)** — भेट देणाऱ्या व्यक्तीच्या देशाशी जुळणारे चलन आपोआप पूर्व-निवड (pre-select) करते.
- **ॲडमिन डॅशबोर्ड (Admin dashboard)** — सक्रिय चलने, एक्सचेंज रेट्स आणि प्रदाता सेटिंग्ज एकाच ठिकाणाहून व्यवस्थापित करा.

## आवश्यकता (Requirements) {#requirements}

- Ultimate Multisite 2.0.0 किंवा त्याहून अधिक
- WordPress Multisite इंस्टॉलेशन
- PHP 7.4 किंवा त्याहून अधिक

## इंस्टॉलेशन (Installation) {#installation}

1. प्लगइन फाइल्स `/wp-content/plugins/ultimate-multisite-multi-currency` येथे अपलोड करा.
2. वर्डप्रेसमधील **Plugins** स्क्रीनमधून प्लगइन सक्रिय करा (Network Activate).
3. ॲडमिन डॅशबोर्ड उघडण्यासाठी **Ultimate Multisite → Multi-Currency** वर जा.
4. जी चलने तुम्हाला स्वीकारायची आहेत ती जोडा आणि तुमचा एक्सचेंज रेट प्रदाता (exchange rate provider) कॉन्फिगर करा.

## कॉन्फिगरेशन (Configuration) {#configuration}

### ॲडमिन डॅशबोर्ड (Admin Dashboard) {#admin-dashboard}

Multi-Currency ॲडमिन डॅशबोर्ड (**Ultimate Multisite → Multi-Currency**) हे खालील गोष्टींसाठी मध्यवर्ती ठिकाण आहे:

- चलने जोडणे, सक्षम करणे किंवा अक्षम करणे.
- एक्सचेंज रेट प्रदाता निवडणे आणि मागणीनुसार रेट्स रिफ्रेश करणे.
- सर्व सक्रिय चलनांसाठी सध्याचे रेट्स तपासणे.
- जेव्हा स्वयंचलित प्रदाते योग्य नसतात तेव्हा मॅन्युअल रेट्स सेट करणे.

### एक्सचेंज रेट प्रदाते (Exchange Rate Providers) {#exchange-rate-providers}

| Provider | Description |
|---|---|
| **ECB** | European Central Bank — EUR-आधारित नेटवर्कसाठी विनामूल्य दैनिक रेट्स. |
| **Frankfurter** | ओपन-सोर्स ECB मिरर API — विनामूल्य, API की आवश्यक नाही. |
| **Stripe** | Stripe च्या लाईव्ह FX रेट्सचा वापर करतो — सक्रिय Stripe इंटिग्रेशन आवश्यक आहे. |
| **Manual** | तुम्ही थेट रेट्स प्रविष्ट करता; कोणतेही बाह्य API कॉल नाही. |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** अंतर्गत तुमचा प्रदाता निवडा. रेट्स कधीही मॅन्युअली रिफ्रेश केले जाऊ शकतात किंवा आपोआप अपडेट करण्यासाठी शेड्यूल केले जाऊ शकतात.

### उत्पादनानुसार किंमत बदलणे (Per-Product Pricing Overrides) {#per-product-pricing-overrides}

जेव्हा स्वयंचलित रूपांतरण पुरेसे अचूक नसते, तेव्हा तुम्ही कोणत्याही उत्पादनावर विशिष्ट चलनासाठी अचूक किंमत सेट करू शकता:

1. उत्पादन एडिटर उघडा (**Ultimate Multisite → Products → Edit**).
2. **Currency Pricing** विभागात खाली स्क्रोल करा.
3. ज्या प्रत्येक चलनासाठी तुम्हाला ओव्हरराईड करायचे आहे, त्याची अचूक किंमत प्रविष्ट करा.
4. उत्पादन सेव्ह करा.

ओव्हरराईड नसलेल्या उत्पादनांमध्ये तुमच्या बेस चलनातील रूपांतरित किंमत वापरली जाते.

### चलन निवडक साइनअप फील्ड (Currency Selector Signup Field) {#currency-selector-signup-field}

ग्राहकांना नोंदणी करताना त्यांचे पसंतीचे चलन निवडण्याची सुविधा देण्यासाठी चलन निवडक फील्ड तुमच्या साइनअप/चेकआउट फ्लोमध्ये जोडा जाऊ शकते:

1. **Ultimate Multisite → Multi-Currency → Settings** वर जा.
2. **Show currency selector on signup** सक्षम करा.
3. पर्यायीरित्या एक डिफॉल्ट चलन सेट करा किंवा भू-स्थानला पूर्व-निवड मूल्य ठरू द्या.

निवडलेले चलन सबस्क्रिप्शनसोबत साठवले जाते आणि पुढील सर्व बिलिंगसाठी वापरले जाते.

### भू-स्थान-आधारित चलन ओळख (Geolocation-Based Currency Detection) {#geolocation-based-currency-detection}

जेव्हा भू-स्थान ओळख सक्षम असते, तेव्हा हे ॲडॉन भेट देणाऱ्या व्यक्तीच्या देशाशी चलन मॅप करते आणि चलन निवडक फील्डमध्ये ते पूर्व-निवड करते. सक्षम करण्यासाठी:

1. **Ultimate Multisite → Multi-Currency → Settings** वर जा.
2. **Detect currency from visitor location** सक्षम करा.
3. **Geolocation Mapping** टेबलमध्ये देशांना चलनांशी मॅप करा.

जर भेट देणाऱ्या व्यक्तीच्या देशासाठी कोणतेही मॅपिंग नसेल, तर नेटवर्कचे बेस चलन फॉलबॅक म्हणून वापरले जाते.

## वारंवार विचारले जाणारे प्रश्न (Frequently Asked Questions) {#frequently-asked-questions}

### जर एक्सचेंज रेट प्रदाता अनुपलब्ध असेल तर काय होते? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

ॲडॉन शेवटचे यशस्वीरित्या प्राप्त झालेले रेट्स वापरते. जर कधीही रेट्स प्राप्त झाले नसतील, तर ते तुम्ही कॉन्फिगर केलेले मॅन्युअल रेट्स वापरते. सुरक्षिततेसाठी मॅन्युअल रेट्स कॉन्फिगर करण्याची शिफारस केली जाते.

### मी वेगवेगळ्या चलनांसाठी वेगवेगळे प्रदाते वापरू शकतो का? {#can-i-use-different-providers-for-different-currencies}

नाही — सर्व चलनांसाठी एका वेळी एकच प्रदाता सक्रिय असतो. जर तुम्हाला वैयक्तिक रेट्सवर पूर्ण नियंत्रण हवे असेल, तर **Manual** वापरा.

### भू-स्थानसाठी वेगळे प्लगइन आवश्यक आहे का? {#does-geolocation-require-a-separate-plugin}

नाही, बाह्य भू-स्थान प्लगइन आवश्यक नाही. ॲडॉन वर्डप्रेसच्या अंगभूत IP-टू-देश ओळख (IP-to-country detection) वापरते. अधिक अचूकतेसाठी, प्रदान केलेल्या फिल्टर हुक्सद्वारे समर्पित भू-स्थान सेवा समाकलित (integrate) केली जाऊ शकते.

### उत्पादनानुसार ओव्हरराईड्स कर (tax) पूर्वी लागू होतात की नंतर? {#are-per-product-overrides-applied-before-or-after-tax}

उत्पादनानुसार ओव्हरराईड्स लक्ष्यित चलनात कर-पूर्व किंमत सेट करतात. तुमच्या विद्यमान कर कॉन्फिगरेशनचा वापर करून ओव्हरराईड किमतीवर कर मोजला जातो.
