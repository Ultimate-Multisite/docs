# साधने आणि संरचना {#tools-and-configuration}

Theme Builder तुमची WordPress साईट विश्लेषित करण्यासाठी, डिझाइन करण्यासाठी आणि तयार करण्यासाठी साधनांचा संच वापरते. आवृत्ती 1.16.0 मध्ये, **sd-ai-agent/site-scrape आता Tier 1 साधन आहे**, त्यामुळे ते Theme Builder मध्ये डीफॉल्टने उपलब्ध आहे. Superdav AI Agent v1.18.0 विश्वासार्ह विकासक कार्यप्रवाहांसाठी स्वतंत्रपणे वितरित केलेले Advanced सहचर पॅकेजही सादर करते; ती साधने WordPress.org मुख्य पॅकेजचा भाग नाहीत आणि ती स्वतंत्रपणे स्थापित व अधिकृत करावी लागतात.

## उपलब्ध साधने {#available-tools}

### Tier 1 साधने (नेहमी उपलब्ध) {#tier-1-tools-always-available}

Tier 1 साधने कोणत्याही अतिरिक्त संरचनेशिवाय Theme Builder मध्ये डीफॉल्टने उपलब्ध असतात.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**उद्देश:** डिझाइन प्रेरणा, सामग्री रचना आणि कार्यक्षमता काढण्यासाठी विद्यमान वेबसाइट्सचे विश्लेषण करणे.

**क्षमता:**

- **वेबसाइट विश्लेषण** — स्पर्धक किंवा प्रेरणादायी वेबसाइट्स scrape करून विश्लेषित करा
- **डिझाइन निष्कर्षण** — रंग, फॉन्ट आणि मांडणी नमुने ओळखा
- **सामग्री रचना** — पृष्ठ संघटना आणि श्रेणीक्रम समजून घ्या
- **वैशिष्ट्य शोध** — plugins आणि कार्यक्षमता ओळखा
- **कार्यक्षमता विश्लेषण** — पृष्ठ वेग आणि ऑप्टिमायझेशन तपासा
- **SEO विश्लेषण** — meta tags आणि structured data पुनरावलोकन करा

**वापर:**

```
example.com चे डिझाइन विश्लेषित करा आणि माझ्या साईटसाठी ते प्रेरणा म्हणून वापरा.
```

**ते काय काढते:**

- रंग पॅलेट आणि रंग वापर
- टायपोग्राफी (फॉन्ट आणि आकार)
- मांडणी रचना आणि अंतर
- नेव्हिगेशन नमुने
- सामग्री संघटना
- प्रतिमा आणि मीडिया वापर
- परस्परसंवादी घटक
- मोबाईल प्रतिसादक्षमता

**मर्यादा:**

- पासवर्ड-संरक्षित साईट्स scrape करू शकत नाही
- robots.txt आणि साईट धोरणांचा आदर करते
- गतिशील सामग्री कदाचित पकडू शकणार नाही
- JavaScript-प्रधान साईट्सचे विश्लेषण मर्यादित असू शकते
- मोठ्या साईट्सचे विश्लेषण करण्यास जास्त वेळ लागू शकतो

### Tier 2 साधने (पर्यायी) {#tier-2-tools-optional}

Tier 2 साधने Theme Builder सेटिंग्जमध्ये स्पष्टपणे सक्षम केल्यावर उपलब्ध असतात.

#### प्रगत विश्लेषण {#advanced-analytics}

तपशीलवार कार्यक्षमता मेट्रिक्स प्रदान करते:

- पृष्ठ लोड वेळा
- Core Web Vitals
- SEO गुण
- प्रवेशयोग्यता गुण
- सर्वोत्तम पद्धती गुण

#### सामग्री ऑप्टिमायझर {#content-optimizer}

यासाठी विश्लेषण करून सुधारणा सुचवते:

- वाचनीयता
- SEO ऑप्टिमायझेशन
- कीवर्ड वापर
- सामग्री रचना
- कॉल-टू-अ‍ॅक्शन ठेवण्याचे स्थान

### Tier 3 साधने (Premium) {#tier-3-tools-premium}

Tier 3 साधनांसाठी अतिरिक्त संरचना किंवा premium वैशिष्ट्ये आवश्यक असतात.

#### AI सामग्री जनरेटर {#ai-content-generator}

यासाठी सामग्री तयार करते:

- उत्पादन वर्णने
- सेवा पृष्ठे
- ब्लॉग पोस्ट्स
- Meta descriptions
- कॉल-टू-अ‍ॅक्शन मजकूर

#### प्रगत डिझाइन साधने {#advanced-design-tools}

प्रगत डिझाइन क्षमता प्रदान करते:

- सानुकूल CSS निर्मिती
- अ‍ॅनिमेशन निर्मिती
- परस्परसंवादी घटक डिझाइन
- प्रगत रंग सिद्धांत
- टायपोग्राफी ऑप्टिमायझेशन

### प्रगत सहचर साधने (स्वतंत्र पॅकेज) {#advanced-companion-tools-separate-package}

Advanced सहचर पॅकेज मुख्य Superdav AI Agent प्रकाशनापासून स्वतंत्रपणे वितरित केले जाते. हे विश्वासार्ह विकासक आणि साईट-मालक वातावरणांसाठी आहे, जिथे प्रशासक एजंटला उच्च-प्रभाव साधने वापरण्याची स्पष्ट परवानगी देतात.

Advanced सहचर क्षमतांमध्ये समाविष्ट असू शकते:

- **फाइलसिस्टम साधने** — विश्वासार्ह वातावरणात मंजूर फाइल्स तपासा आणि बदला
- **डेटाबेस साधने** — स्पष्टपणे अधिकृत केल्यावर साईट डेटा क्वेरी किंवा अपडेट करा
- **WP-CLI साधने** — WordPress देखभाल आणि तपासणी कमांड्स चालवा
- **REST dispatcher साधने** — नियंत्रित dispatch कार्यप्रवाहांद्वारे नोंदणीकृत REST endpoints कॉल करा
- **Plugin builder साधने** — plugin code scaffold करा आणि त्यावर पुनरावृत्ती करा
- **Git snapshot साधने** — जोखमीच्या विकासक क्रियांपूर्वी snapshots तयार करा
- **वापरकर्ता-व्यवस्थापन साधने** — परवानगी असल्यास वापरकर्ता नोंदी पुनरावलोकन किंवा समायोजित करण्यात मदत करा
- **बेंचमार्क साधने** — विकास पुनरावलोकनासाठी कार्यक्षमता किंवा क्षमता बेंचमार्क चालवा

ही साधने प्रत्येक स्थापनेसाठी सर्वसाधारणपणे उपलब्ध Theme Builder साधने म्हणून दस्तऐवजीकरण करू नका. ती केवळ Advanced सहचर पॅकेज स्थापित, सक्षम आणि विश्वासार्ह प्रशासकांपुरते मर्यादित असताना उपलब्ध असतात.

## साधन संरचना {#tool-configuration}

### साधने सक्षम करणे {#enabling-tools}

Theme Builder मध्ये अतिरिक्त साधने सक्षम करण्यासाठी:

1. **Gratis AI Agent → Settings** येथे जा
2. **Theme Builder → Tools** येथे जा
3. गरजेनुसार साधने चालू/बंद करा
4. सेटिंग्ज जतन करा

### साधन परवानग्या {#tool-permissions}

काही साधनांना परवानग्या आवश्यक असतात:

- **साईट scraping** — इंटरनेट प्रवेश आवश्यक आहे
- **विश्लेषण** — Google Analytics एकत्रीकरण आवश्यक आहे
- **सामग्री निर्मिती** — API keys आवश्यक आहेत
- **प्रगत वैशिष्ट्ये** — premium subscription आवश्यक असू शकते
- **Advanced सहचर साधने** — स्वतंत्रपणे वितरित Advanced पॅकेज आणि स्पष्ट प्रशासक विश्वास आवश्यक आहे

### API Keys आणि प्रमाणपत्रे {#api-keys-and-credentials}

ज्या साधनांना API keys आवश्यक आहेत त्यांच्यासाठी त्या संरचीत करा:

1. **Gratis AI Agent → Settings → API Keys** येथे जा
2. प्रत्येक साधनासाठी प्रमाणपत्रे प्रविष्ट करा
3. कनेक्शन चाचणी करा
4. सुरक्षितपणे जतन करा

### Advanced सहचर पॅकेज स्थापित करणे {#installing-the-advanced-companion-package}

Advanced सहचर पॅकेज WordPress.org मुख्य ZIP पासून स्वतंत्रपणे प्रकाशित केले जाते. ते फक्त त्या प्रकाशनासाठीच्या अधिकृत प्रकल्प वितरण चॅनलवरून स्थापित करा, नंतर प्रवेश विश्वासार्ह प्रशासकांपुरता मर्यादित करा. उत्पादन साईटवर फाइलसिस्टम, डेटाबेस, WP-CLI, REST dispatcher, plugin builder, वापरकर्ता-व्यवस्थापन किंवा बेंचमार्क साधने सक्षम करण्यापूर्वी मानवी मंजुरी गेट्स आणि git snapshot वर्तनाचे पुनरावलोकन करा.

## sd-ai-agent/site-scrape वापरणे {#using-sd-ai-agentsite-scrape}

### मूलभूत वापर {#basic-usage}

Theme Builder ला एखाद्या वेबसाइटचे विश्लेषण करण्यास सांगा:

```
competitor.com वरील माझ्या स्पर्धकाच्या साईटचे डिझाइन विश्लेषित करा
आणि माझ्या साईटसाठी डिझाइन सुधारणा सुचवा.
```

### विशिष्ट विश्लेषण {#specific-analysis}

विशिष्ट प्रकारचे विश्लेषण मागवा:

```
example.com मधून रंग पॅलेट काढा आणि ते प्रेरणा म्हणून वापरा.
```

```
example.com ची नेव्हिगेशन रचना विश्लेषित करा आणि तशीच
संघटना माझ्या साईटवर लागू करा.
```

```
example.com ची मोबाईल प्रतिसादक्षमता तपासा आणि माझी साईट
तितकीच प्रतिसादक्षम असल्याची खात्री करा.
```

### तुलना विश्लेषण {#comparison-analysis}

अनेक साईट्सची तुलना करा:

```
site1.com आणि site2.com यांच्या डिझाइन्सची तुलना करा आणि दोन्हीतील सर्वोत्तम घटक
एकत्र करणारे डिझाइन तयार करा.
```

## साधन मर्यादा आणि विचार {#tool-limitations-and-considerations}

### दर मर्यादा {#rate-limiting}

- सर्व्हरवर अति भार येऊ नये म्हणून Scraping वर दर-मर्यादा आहे
- त्याच site साठी अनेक विनंत्या थ्रॉटल केल्या जाऊ शकतात
- मोठ्या sites चे विश्लेषण करण्यास अधिक वेळ लागू शकतो

### सामग्रीची अचूकता {#content-accuracy}

- गतिशील सामग्री पूर्णपणे पकडली जाऊ शकत नाही
- JavaScript द्वारे रेंडर केलेली सामग्री अपूर्ण असू शकते
- रिअल-टाइम डेटा प्रतिबिंबित झाला नसेल
- काही सामग्री paywalls मागे असू शकते

### कायदेशीर आणि नैतिक विचार {#legal-and-ethical-considerations}

- robots.txt आणि site धोरणांचा आदर करा
- पुनरुत्पादनासाठी कॉपीराइट असलेली सामग्री scrape करू नका
- विश्लेषण प्रेरणेसाठी वापरा, कॉपी करण्यासाठी नाही
- काढलेली सामग्री वापरण्याचे हक्क आपल्याकडे आहेत याची पडताळणी करा
- विश्लेषित sites च्या सेवा अटींचे पालन करा

### कार्यप्रदर्शनावर परिणाम {#performance-impact}

- मोठ्या sites चे scraping करण्यास वेळ लागू शकतो
- एकाच वेळी अनेक scrapes धीमे असू शकतात
- नेटवर्क कनेक्टिव्हिटी गतीवर परिणाम करते
- मोठ्या datasets साठी अधिक प्रक्रिया आवश्यक असू शकते

## सर्वोत्तम पद्धती {#best-practices}

### Site विश्लेषण वापरणे {#using-site-analysis}

1. **अनेक sites चे विश्लेषण करा** — अनेक स्रोतांकडून प्रेरणा गोळा करा
2. **रचनेवर लक्ष केंद्रित करा** — लेआउट आणि संघटन नमुने शिका
3. **रंग काढा** — रंग पॅलेट्स प्रारंभिक बिंदू म्हणून वापरा
4. **टायपोग्राफीचा अभ्यास करा** — आपल्याला आवडणारे फॉन्ट संयोजन ओळखा
5. **नेव्हिगेशनचे पुनरावलोकन करा** — मेनू संघटना समजून घ्या

### नैतिक Scraping {#ethical-scraping}

1. **robots.txt चा आदर करा** — site मार्गदर्शक तत्त्वांचे पालन करा
2. **सामग्री कॉपी करू नका** — विश्लेषण फक्त प्रेरणेसाठी वापरा
3. **हक्कांची पडताळणी करा** — काढलेली सामग्री आपण वापरू शकता याची खात्री करा
4. **स्रोतांना श्रेय द्या** — प्रेरणा स्रोतांना श्रेय द्या
5. **अटींचे पालन करा** — site च्या सेवा अटींचे पालन करा

### साधनांची परिणामकारकता कमाल करणे {#maximizing-tool-effectiveness}

1. **विशिष्ट रहा** — विशिष्ट प्रकारच्या विश्लेषणाची विनंती करा
2. **संदर्भ द्या** — आपल्या site चा उद्देश समजावून सांगा
3. **अपेक्षा निश्चित करा** — आपले डिझाइन उद्दिष्टे वर्णन करा
4. **पुनरावृत्ती करा** — निकालांवर आधारित सुधारणा करा
5. **साधने एकत्र वापरा** — सर्वसमावेशक विश्लेषणासाठी अनेक साधने वापरा

## समस्या निवारण {#troubleshooting}

### Site Scrape होत नाही {#site-wont-scrape}

- site सार्वजनिकरीत्या प्रवेशयोग्य आहे का ते तपासा
- robots.txt scraping ला परवानगी देते का ते पडताळा
- वेगळी site वापरून पहा
- इंटरनेट कनेक्शन तपासा
- समस्या कायम राहिल्यास support शी संपर्क साधा

### अपूर्ण विश्लेषण {#incomplete-analysis}

- site वर गतिशील सामग्री असू शकते
- JavaScript पूर्णपणे रेंडर झालेले नसू शकते
- मोठ्या sites timeout होऊ शकतात
- त्याऐवजी विशिष्ट पृष्ठांचे विश्लेषण करून पहा
- विशिष्ट विश्लेषण प्रकारांची विनंती करा

### धीमे कार्यप्रदर्शन {#slow-performance}

- मोठ्या sites चे विश्लेषण करण्यास अधिक वेळ लागतो
- एकाच वेळी अनेक scrapes धीमे असतात
- नेटवर्क कनेक्टिव्हिटी गतीवर परिणाम करते
- कमी गर्दीच्या वेळेत प्रयत्न करा
- आधी लहान विभागांचे विश्लेषण करा
