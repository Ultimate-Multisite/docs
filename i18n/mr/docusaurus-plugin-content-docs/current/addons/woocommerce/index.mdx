# WooCommerce एकत्रीकरण

WooCommerce एकत्रीकरण आमच्या Manual Gateway ज्या प्रकारे काम करते त्याच्याशी खूपच समान प्रकारे काम करते. हे तुमच्या वापरकर्त्यांना मोठ्या प्रमाणात लवचिकता देते, ज्यामुळे ते तुमच्या WooCommerce स्थापनेवर उपलब्ध असलेल्या कोणत्याही पेमेंट फॉर्मची निवड करू शकतात. आणि ते सोपे आहे: वापरकर्त्याचा बिलिंग कालावधी संपल्यानंतर, वापरकर्त्याची सदस्यता होल्डवर ठेवली जाते. त्याच वेळी, नेटवर्कच्या मुख्य साइटवरील WooCommerce स्थापनेवर एक ऑर्डर तयार केली जाईल. त्यानंतर वापरकर्त्याला त्यांच्या ईमेलमध्ये एक इनव्हॉइस मिळतो, ज्यात पेमेंट करण्यासाठी एक लिंक असते. पेमेंट मिळाल्यानंतर, सदस्यता नूतनीकरण केली जाते.

## वैशिष्ट्ये {#features}

- Ultimate Multisite आणि WooCommerce यांच्यात सोपे एकत्रीकरण
- अनेक पेमेंट पद्धती उपलब्ध
- बंडल केलेल्या सदस्यता कोअरचा वापर करून स्वयंचलित आवर्ती बिलिंग, ऐच्छिक WooCommerce Subscriptions plugin सुधारणांसह
- Checkout एकूणांमध्ये Ultimate Multisite कर स्वतंत्र, करपात्र नसलेल्या WooCommerce फी ओळी म्हणून समाविष्ट असतात, जेणेकरून ग्राहक पेमेंटपूर्वी कर रकमेचे पुनरावलोकन करू शकतील

## दस्तऐवजीकरण {#documentation}

**WooCommerce एकत्रीकरण सेट करणे**

**Ultimate Multisite सह तुमचे पेमेंट पर्याय विस्तारित करणे: WooCommerce एकत्रीकरण**

## स्थापना {#installation}

1. 'ultimate-multisite-woocommerce' हे '/wp-content/plugins/' निर्देशिकेत अपलोड करा
2. WordPress मधील 'Plugins' मेनूद्वारे plugin सक्रिय करा
3. plugin सेट करण्यासाठी चरण-दर-चरण Wizard चे अनुसरण करा

## वापर मार्गदर्शक {#usage-guide}

_**महत्त्वाची नोंद: हा लेख Ultimate Multisite आवृत्ती 2.x संदर्भित करतो.**_

_**लक्ष द्या:** Ultimate Multisite: WooCommerce एकत्रीकरणासाठी WooCommerce किमान तुमच्या मुख्य साइटवर सक्रिय असणे आवश्यक आहे._

आम्हाला समजते की _Stripe_ आणि _PayPal_ काही देशांमध्ये उपलब्ध नाहीत, ज्यामुळे Ultimate Multisite वापरकर्त्यांना आमचा plugin प्रभावीपणे वापरणे मर्यादित किंवा अडथळादायक होते. म्हणून आम्ही **_WooCommerce_ एकत्रित करण्यासाठी add-on** तयार केला.

_WooCommerce_ हा एक अतिशय लोकप्रिय ईकॉमर्स plugin आहे. जगभरातील विकासकांनी त्यात वेगवेगळे पेमेंट गेटवे एकत्रित करण्यासाठी add-ons तयार केले. Ultimate Multisite बिलिंग प्रणालीमध्ये तुम्ही वापरू शकणारे पेमेंट गेटवे विस्तारित करण्यासाठी आम्ही याचा फायदा घेतला.

## ते सेट करणे {#setting-it-up}

प्रथम, तुम्हाला WooCommerce Integration add-on स्थापित करणे आवश्यक आहे, जो **परवानाधारक वापरकर्त्यांसाठी मोफत** आहे. तो स्थापित करण्यासाठी, तुमच्या super admin dashboard वर Ultimate Multisite वर क्लिक करा आणि **Add-ons** निवडा.

![Add-ons पृष्ठ](/img/admin/addons-page.png)

येथे तुम्हाला सर्व Ultimate Multisite add-ons सापडतील. **Ultimate Multisite: WooCommerce Integration** add-on वर क्लिक करा.

![WooCommerce Integration दाखवणारे Add-ons पृष्ठ](/img/admin/addons-page.png)

add-on तपशीलांसह एक विंडो उघडेल. फक्त **Install Now** वर क्लिक करा *.*

<!-- स्क्रीनशॉट उपलब्ध नाही: Install Now बटणासह WooCommerce Integration addon स्थापना पॉपअप -->

स्थापना पूर्ण झाल्यानंतर, तुम्हाला plugins पृष्ठावर पुनर्निर्देशित केले जाईल. येथे, फक्त **Network Activate** वर क्लिक करा आणि _WooCommerce_ add-on तुमच्या नेटवर्कवर सक्रिय केला जाईल.

<!-- स्क्रीनशॉट उपलब्ध नाही: WooCommerce Integration साठी Network Activate बटण दाखवणारे WP Plugins पृष्ठ -->

ते सक्रिय केल्यानंतर, जर तुमच्या वेबसाइटवर WooCommerce plugin अद्याप स्थापित आणि सक्रिय नसेल, तर **तुम्हाला एक स्मरणपत्र मिळेल**.

<!-- स्क्रीनशॉट उपलब्ध नाही: WooCommerce plugin स्थापित आणि सक्रिय करण्याची आठवण करून देणारी Admin सूचना -->

स्थापित झाल्यानंतर, तुम्हाला Ultimate Multisite पेमेंट सेटिंग्जखाली एक अतिरिक्त पर्याय दिसेल. त्यानंतर तुम्ही तेथून _WooCommerce_ पर्याय **सक्षम** करू शकता.

![WooCommerce सक्षम असलेली Ultimate Multisite पेमेंट गेटवे सेटिंग्ज](/img/addons/woocommerce-payment-gateway.png)

तुम्ही फ्रंट एंडमध्ये दिसणारे प्रदर्शन नाव आणि तुमच्या वापरकर्त्यांना त्यांच्या checkout प्रवाहात मार्गदर्शन करण्यासाठी सूचना यांसारख्या काही मूलभूत सेटिंग्ज कॉन्फिगर करू शकता. त्यानंतर तुम्ही वापरायला प्राधान्य देत असलेला पेमेंट गेटवे सक्षम आणि कॉन्फिगर करण्यासाठी तुमच्या **मुख्य साइटच्या _WooCommerce_ सेटिंग्ज** मध्ये जाऊ शकता.

### Checkout कर फी ओळी {#checkout-tax-fee-lines}

जेव्हा Ultimate Multisite checkout मध्ये कर समाविष्ट असतात, तेव्हा एकत्रीकरण प्रत्येक Ultimate Multisite कर रक्कम WooCommerce checkout मध्ये स्वतंत्र, करपात्र नसलेली फी ओळ म्हणून जोडते. यामुळे पेमेंटपूर्वी कर एकूण दृश्यमान राहतो, तसेच WooCommerce ने त्या कर फीवर पुन्हा कर लावणे टाळले जाते.

## पेमेंट कसे व्यवस्थापित करावे {#how-to-manage-the-payment}

WooCommerce एकत्रीकरण आमच्या **Manual Gateway** ज्या प्रकारे काम करते त्याच्याशी खूपच समान प्रकारे काम करते. तोटा असा आहे की प्रत्येक पेमेंट बिलिंग कालावधीच्या शेवटी मॅन्युअली भरावे लागते. प्रवाह कसा काम करतो ते येथे आहे:

  * वापरकर्त्याचे बिलिंग चक्र संपल्यानंतर, सदस्यत्व **होल्डवर** ठेवले जाते. त्याच वेळी, तुमच्या नेटवर्कच्या मुख्य साइटवरील _WooCommerce_ स्थापनेवर एक ऑर्डर तयार केली जाईल.

  * त्यानंतर वापरकर्त्यांना त्यांच्या ईमेलमध्ये एक इनव्हॉइस मिळतो, ज्यात पेमेंट करण्यासाठी एक लिंक असते (ती लिंक त्यांच्या सबसाइट dashboard मधील त्यांच्या बिलिंग इतिहासातही दिसते). ती लिंक वापरकर्त्याला _**WooCommerce**_ **checkout** कडे पुनर्निर्देशित करते, जिथे वापरकर्ते त्यांचे पेमेंट प्रक्रिया करण्यासाठी **उपलब्ध _WooCommerce_ गेटवे** पैकी एक निवडू शकतील.

  * पेमेंट मिळाल्यानंतर, सदस्यत्व **नूतनीकरण** केले जाते, पेमेंट Ultimate Multisite वर लॉग केले जाते आणि ऑर्डर पूर्ण म्हणून सेट केली जाते.

### काही महत्त्वाच्या नोंदी: {#some-important-notes}

  * Ultimate Multisite ची डीफॉल्ट कार्यक्षमता या एकत्रीकरण फॉर्मसह अजूनही काम करते. उदाहरणार्थ, **Subscription Management** स्क्रीनवरून **परतावे जारी करणे** निवडलेल्या पेमेंट फॉर्मचा वापर करून परतावा जारी करण्यासाठी WooCommerce शी संवाद साधेल.

  * Ultimate Multisite: WooCommerce एकत्रीकरण काम करण्यासाठी WP-Cron वर अवलंबून असते. हे बहुतेक वेळा खूप चांगले काम करते, परंतु साइटवर कमी ट्रॅफिक असल्यास cron ला नियोजित इव्हेंट तपासण्यास प्रवृत्त केले जात नाही आणि इव्हेंट चुकतो, ज्यामुळे अंमलबजावणीत विलंब होऊ शकतो.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

दर महिन्याला हाताने पेमेंट सुरू करणे अंतिम वापरकर्त्यांसाठी आदर्श नाही, कारण subscriptions प्रत्येक billing cycle मध्ये आपोआप पुन्हा होण्याची अपेक्षा असते. WooCommerce Integration add-on आता मुख्य subscriptions code सोबत बंडल केलेले आहे, त्यामुळे recurring products नेहमी recurring billing साठी subscriptions वापरतात.

या integration द्वारे recurring products विकण्यासाठी तुम्हाला स्वतंत्र [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) खरेदी किंवा install करण्याची **गरज नाही**. जर तो plugin खरेदी केलेला, install केलेला आणि तुमच्या मुख्य साइटवर उपलब्ध असेल, तर Ultimate Multisite तो वापरेल आणि WooCommerce अतिरिक्त subscription व्यवस्थापन वैशिष्ट्ये उपलब्ध करून देईल. तो उपलब्ध नसल्यास, बंडल केलेला subscriptions core तरीही recurring products हाताळतो.

![WooCommerce subscription integration सेटिंग्ज](/img/addons/woocommerce-subscription-integration.png)

आणि एवढेच. तुमच्या अंतिम वापरकर्त्यांना recurring products साठी त्यांच्या billing cycle च्या शेवटी हाताने पेमेंट करण्याची गरज राहणार नाही.

तुमच्या मुख्य साइटवर _WooCommerce_ अंतर्गत ऑर्डर तयार होण्याव्यतिरिक्त, प्रत्येक Account साठी subscription देखील तयार होईल.

### अयशस्वी renewal suspension वर्तन {#failed-renewal-suspension-behavior}

पूर्वनिर्धारितपणे, WooCommerce Subscriptions त्याचे failed-payment retry schedule प्रक्रिया करत असताना memberships सक्रिय राहतात. यामुळे सामान्य WooCommerce Subscriptions grace window जतन होते, जेणेकरून पुनर्प्राप्त झालेला renewal subscription आणि membership पुन्हा समक्रमित करू शकतो, प्रवेश अकाली खंडित न करता.

अधिक कठोर प्रवेश नियंत्रण हवे असलेले प्रशासक WooCommerce Integration सेटिंग्जमध्ये **Failed Renewals वर Memberships तत्काळ Suspend करा** सक्षम करू शकतात. हे opt-in सेटिंग सक्षम केले असताना, WooCommerce Subscriptions retry handling पूर्ण होण्याची वाट पाहण्याऐवजी Ultimate Multisite renewal payment अयशस्वी होताच संबंधित membership suspend करते.

Developers हाच निर्णय `wu_woo_suspend_on_payment_failure` filter ने override करू शकतात. Hook reference pages add-on source मधून generated केले जातात, त्यामुळे हा overview custom suspension policies साठी फक्त filter name नमूद करतो.
