# बहु-टेनन्सी {#multi-tenancy}

कार्यक्षमता आणि सुरक्षिततेसाठी उप-साइट्स सर्व्हर आणि डेटाबेसमध्ये वितरित करा

## आवृत्ती 1.2.0 प्रकाशनातील ठळक मुद्दे {#version-120-release-highlights}

Multi-Tenancy 1.2.0 उत्पादन विलगीकरण कार्यप्रवाहांसाठी सार्वभौम टेनंट पाया जोडते:

- अल्पायुषी, रीप्ले-संरक्षित SSO भेटींसह स्टेटलेस टेनंट स्वयंचलित लॉगिन.
- सार्वभौम टेनंटसाठी प्रति-उप-साइट डेटाबेस आणि फाइलसिस्टम विलगीकरण.
- होस्ट-जागरूक टेनंट पुरवठ्यासाठी Hostinger बहु-टेनन्सी क्षमता.
- जुना डेटा तपासणी आणि असिंक पुश प्रक्रियेसाठी सार्वभौम स्थलांतर पडताळणी गेट्स.
- Bedrock, FrankenPHP, आणि localhost इंस्टॉलसाठी कॉन्फिगर करण्यायोग्य समान-मशीन डेटाबेस होस्ट बाइंडिंग.
- टेनंट बूटस्ट्रॅप, डेटाबेस रायटर, स्कीमा बूटस्ट्रॅप, आणि वापरकर्ता पुरवठा सुधारणा.
- टेनंट क्रेडेन्शियल्स साफ करणारे अधिक सुरक्षित सार्वभौम टिअरडाउन.

[सार्वभौम स्थलांतर कार्यप्रवाह](/user-guide/getting-started/sovereign-migration-workflow), [बहु-टेनन्सी विलगीकरण](/user-guide/configuration/multi-tenancy-isolation), आणि [सार्वभौम टेनंट ऑपरेशन्स](/user-guide/administration/sovereign-tenant-operations) यांसाठी वापरकर्ता मार्गदर्शिका पहा.

## आवश्यकता {#requirements}

- WordPress 5.3 किंवा उच्च
- PHP 7.4 किंवा उच्च
- Multisite Ultimate प्लगइन (सक्रिय)

## स्थापना {#installation}

1. अ‍ॅडऑन फाइल्स तुमच्या `/wp-content/plugins/` निर्देशिकेत अपलोड करा
2. WordPress मधील 'प्लगइन्स' मेनूद्वारे प्लगइन सक्रिय करा
3. नेटवर्क प्रशासनात अ‍ॅडऑन सेटिंग्ज कॉन्फिगर करा

## विकास {#development}

### सेटअप {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs

# Fix code style issues
vendor/bin/phpcbf
```

### चाचणी {#testing}

अ‍ॅडऑनमध्ये सर्वसमावेशक चाचणी फ्रेमवर्क समाविष्ट आहे:

- **युनिट चाचण्या**: WordPress चाचणी संचासह PHPUnit-आधारित चाचण्या
- **बेस चाचणी वर्ग**: सामान्य चाचणी परिस्थितींसाठी सहाय्यक पद्धती पुरवतो
- **चाचणी कव्हरेज**: कोड कव्हरेज ट्रॅक करण्यासाठी कॉन्फिगर केलेले
- **Multisite चाचणी**: चाचण्या multisite वातावरणात चालतात

### कोड मानके {#code-standards}

- **WordPress कोडिंग मानके**: PHPCS द्वारे अंमलात आणलेली
- **PHP 7.4+ सुसंगतता**: आधुनिक PHP वैशिष्ट्ये समर्थित
- **स्थिर विश्लेषण**: प्रकार तपासणीसाठी PHPStan एकत्रीकरण
- **कोड आधुनिकीकरण**: PHP अपग्रेडसाठी Rector नियम

### बिल्ड प्रणाली {#build-system}

- **अ‍ॅसेट प्रक्रिया**: CSS/JS फाइल्ससाठी मिनिफिकेशन
- **भाषांतर**: POT फाइल निर्मिती
- **पॅकेजिंग**: वितरणासाठी आर्काइव्ह निर्मिती
- **विकास मोड**: डीबगिंगसाठी अनमिनिफाइड अ‍ॅसेट्स

## कॉन्फिगरेशन {#configuration}

अ‍ॅडऑन नेटवर्क प्रशासन इंटरफेसद्वारे उपलब्ध विविध कॉन्फिगरेशन पर्यायांना समर्थन देते.

### सार्वभौम टेनंट कॉन्फिगरेशन {#sovereign-tenant-configuration}

सार्वभौम टेनंटसाठी, टेनंटकडे ट्रॅफिक रूट करण्यापूर्वी टेनंट डेटाबेस, फाइलसिस्टम रूट, डेटाबेस होस्ट बाइंडिंग, आणि होस्ट क्षमता कॉन्फिगर करा. डेटाबेस क्रेडेन्शियल्स, होस्ट बाइंडिंग, रूटिंग, किंवा वापरकर्ता पुरवठ्यावर परिणाम करणाऱ्या प्रत्येक बदलानंतर स्थलांतर पडताळणी चालवा.

टेनंट रनटाइम आणि डेटाबेस ग्रँट्स स्थानिक सॉकेट किंवा समान-मशीन प्रवेश अपेक्षित असतील तेव्हाच `localhost` वापरा. तुमच्या प्रदात्याने कॉन्फिगर केलेला ग्रँट होस्ट तो असेल तेव्हा `127.0.0.1` किंवा सेवा होस्टनाव वापरा.

### Hostinger क्षमता {#hostinger-capability}

Hostinger बहु-टेनन्सी क्षमता सामायिक Hostinger एकत्रीकरण टोकनसोबत कार्य करते. पडताळणीपूर्वी डोमेन hPanel मध्ये उपस्थित आहे, DNS Hostinger खात्याकडे निर्देशित करतो, आणि टेनंट डेटाबेस सेटिंग्ज hPanel शी जुळतात याची पुष्टी करा.

## Hooks आणि Filters {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - अ‍ॅडऑन प्रारंभीकरणानंतर फायर केले जाते
- `ultimate-multisite-addon-template_loaded` - सर्व अ‍ॅडऑन फाइल्स लोड झाल्यानंतर फायर केले जाते

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - अ‍ॅडऑन सेटिंग्ज सुधारा
- `ultimate-multisite-addon-template_enabled` - अ‍ॅडऑन सक्षम/अक्षम स्थिती ओव्हरराइड करा
