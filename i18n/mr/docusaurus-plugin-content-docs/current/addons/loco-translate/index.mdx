# Loco Translate

Ultimate Multisite: Loco Translate Integration addon तुमच्या WordPress multisite network ला साइट-विशिष्ट भाषांतर क्षमता (site-specific translation capabilities) घेऊन येतो. जर तुम्हाला कधी गरज पडली असेल की तुमच्या नेटवर्कमधील वेगवेगळ्या साइट्सला एकाच प्लगइन किंवा थीमचे वेगवेगळे भाषांतर हवे आहे, तर हे addon सर्वोत्तम उपाय आहे.

**हे addon काय करते**

हे addon Ultimate Multisite ला लोकप्रिय Loco Translate प्लगइनसोबत सहजपणे जोडते, ज्यामुळे तुमच्या नेटवर्कमधील प्रत्येक सबसाइटला स्वतःचे कस्टम भाषांतर (custom translations) राखता येते. तुम्ही जर मल्टी-रिजनल प्लॅटफॉर्म चालवत असाल, वेगवेगळ्या भाषा समुदायांना सेवा देत असाल, किंवा फक्त तुमच्या नेटवर्कमध्ये भाषांतराची लवचिकता (translation flexibility) हवी असेल, तर हे addon काम सोपे करते.

**कोणासाठी उत्तम:**

* विविध भाषांतराच्या गरजा असलेल्या मल्टी-रिजनल व्यवसायांसाठी
* विविध समुदायांना सेवा देणाऱ्या शैक्षणिक संस्थांसाठी
* बहुभाषिक समर्थन (multilingual support) देणाऱ्या SaaS प्लॅटफॉर्मसाठी
* अनेक क्लायंट साइट्स व्यवस्थापित करणाऱ्या वेब एजन्सीसाठी
* भाषांतराच्या लवचिकतेची आवश्यकता असलेल्या कोणत्याही multisite network साठी

## इन्स्टॉलेशन {#installation}

1. Ultimate Multisite इन्स्टॉल आणि नेटवर्क सक्रिय (network activated) असल्याची खात्री करा.
2. Loco Translate प्लगइन इन्स्टॉल आणि सक्रिय करा.
3. हे addon तुमच्या प्लगइन्स डिरेक्टरीमध्ये अपलोड करा.
4. Loco Translate Integration addon नेटवर्क स्तरावर सक्रिय करा.
5. कोणत्याही सबसाइटवर Loco Translate वापरून साइट-विशिष्ट भाषांतर तयार करण्यास सुरुवात करा.

बस! हे एकत्रीकरण (integration) कोणत्याही अतिरिक्त कॉन्फिगरेशनशिवाय आपोआप काम करते.

## वारंवार विचारले जाणारे प्रश्न (Frequently Asked Questions) {#frequently-asked-questions}

## इन्स्टॉलेशननंतर काही कॉन्फिगर करावे लागेल का? {#do-i-need-to-configure-anything-after-installation}

नाही! हे एकत्रीकरण आपोआप होते. एकदा addon आणि Loco Translate दोन्ही सक्रिय झाल्यावर, सबसाइट्स लगेच स्वतःचे कस्टम भाषांतर तयार करू शकतात.

## जर सबसाइटने कस्टम भाषांतर तयार केले नाही तर काय होईल? {#what-happens-if-a-subsite-doesnt-create-custom-translations}

साइट आपोआप नेटवर्क-व्यापी भाषांतरे (network-wide translations) डीफॉल्ट पर्याय म्हणून वापर करेल. काहीही बिघडणार नाही आणि सर्व काही सहजपणे काम करेल.

## सबसाइट्स नेटवर्क-व्यापी भाषांतरे ॲक्सेस करू शकतात का? {#can-subsites-access-network-wide-translations}

सामान्य साइट ॲडमिन (site admins) फक्त त्यांच्या स्वतःच्या साइटचे भाषांतर पाहू आणि संपादित करू शकतात. सुपर ॲडमिन (Super admins) ते कोणत्या साइटवर पाहत आहेत यावर अवलंबून, नेटवर्क-व्यापी आणि साइट-विशिष्ट दोन्ही भाषांतरे ॲक्सेस करू शकतात.

## हे प्लगइन्स आणि थीम्स दोघांसोबत काम करते का? {#does-this-work-with-both-plugins-and-themes}

होय! साइट-विशिष्ट भाषांतर WordPress प्लगइन्स आणि थीम्स दोघांसाठीही काम करते.

## यामुळे माझ्या multisite network ला मंदता येईल का? {#will-this-slow-down-my-multisite-network}

नाही. हे addon कार्यक्षम फाइल लोडिंग (efficient file loading) वापरते, जे फक्त गरज पडल्यास साइट-विशिष्ट भाषांतरे तपासते. कार्यक्षमतेवर कोणताही लक्षणीय परिणाम होत नाही.

## मी नेटवर्क भाषांतरे एका विशिष्ट साइटसाठी सुरुवातीच्या बिंदू म्हणून कॉपी करू शकतो का? {#can-i-copy-network-translations-to-a-specific-site-as-a-starting-point}

होय! Loco Translate मध्ये भाषांतर संपादित करताना, तुम्ही नेटवर्क भाषांतरापासून सुरुवात करू शकता आणि प्रत्येक साइटसाठी त्यांना कस्टमाइझ करू शकता.

## हे WooCommerce आणि इतर प्लगइन्ससोबत काम करते का? {#does-this-work-with-woocommerce-and-other-plugins}

नक्कीच! कोणतेही प्लगइन किंवा थीम जे मानक WordPress भाषांतर फंक्शन्स (standard WordPress translation functions) वापरते, ते साइट-विशिष्ट भाषांतरासोबत काम करेल.
