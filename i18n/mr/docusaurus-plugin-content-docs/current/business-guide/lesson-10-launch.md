# Lesson 10: Launch Day

सर्व काही तयार आहे. तुम्ही दरवाजे उघडण्यापूर्वी, ही Lesson तुम्हाला Pre-launch checklist मधून घेऊन जाईल, जेणेकरून काहीही तुटलेले नाही, काहीतरी गहाळ नाही किंवा काही लाजिरवाणे नाही याची खात्री करता येईल.

## Where We Left Off {#where-we-left-off}

FitSite मध्ये templates, plans, checkout, branding, onboarding आणि pricing सर्व काही कॉन्फिगर केले आहे. आता आपण सर्व काही व्यवस्थित काम करत आहे की नाही हे तपासावे आणि लाईव्ह (live) व्हावे.

## Pre-Launch Checklist {#pre-launch-checklist}

प्रत्येक आयटमवर काम करा. कोणताही आयटम वगळू नका.

### Platform Infrastructure {#platform-infrastructure}

- [ ] Hosting stable आहे आणि loadखाली चांगले काम करत आहे
- [ ] Wildcard SSL active आहे आणि सर्व subdomains HTTPS वर serve होतात
- [ ] Domain mapping काम करते की नाही -- एक site तयार करून आणि custom domain map करून तपासा
- [ ] Backups कॉन्फिगर आणि test केलेले आहेत (verify करण्यासाठी किमान एक restore करा)
- [ ] Monitoring व्यवस्थित आहे -- प्लॅटफॉर्म बंद झाला तर तुम्हाला कळेल

### Templates {#templates}

- [ ] नवीन sites वर तिन्ही templates योग्यरित्या लोड होतात
- [ ] Placeholder content उपयुक्त आहे आणि त्यात स्पेलिंगच्या चुका नाहीत
- [ ] सर्व images योग्यरित्या licensed आहेत (कोणतेही stock photo watermarks नाहीत)
- [ ] प्रत्येक template page वर Mobile responsiveness काम करते
- [ ] Page load speed स्वीकारार्ह आहे (GTmetrix किंवा PageSpeed Insights सारख्या टूलने test करा)
- [ ] कोणत्याही template वर broken links किंवा गहाळ assets नाहीत

### Plans and Products {#plans-and-products}

- [ ] तिन्ही plans active आणि visible आहेत
- [ ] Plan descriptions अचूक आणि niche-specific आहेत
- [ ] Pricing बरोबर आहे (monthly आणि annual)
- [ ] Setup fees योग्य plans वर कॉन्फिगर केले आहेत
- [ ] Starter plan वर Trial period काम करतो
- [ ] Checkout दरम्यान Order bumps योग्यरित्या दिसतात
- [ ] Plan नुसार Plugin आणि theme limitations योग्यरित्या लागू होतात

### Checkout Flow {#checkout-flow}

- [ ] प्रत्येक plan वर पूर्ण test signup करा (test payment mode वापरा)
- [ ] Template selection plan नुसार योग्य templates दाखवते
- [ ] Payment यशस्वीरित्या प्रक्रिया होते
- [ ] signup नंतर ग्राहकाला welcome email मिळतो
- [ ] नवीन site योग्य template सह तयार होते
- [ ] ग्राहक लगेच त्यांच्या नवीन साइटवर login करू शकतो
- [ ] Discount codes योग्यरित्या काम करतात

### Branding {#branding}

- [ ] Login page वर FitSite branding दिसते
- [ ] Admin dashboard वर FitSite branding दिसते
- [ ] सर्व system emails मध्ये FitSite branding आणि fitness-specific language वापरली जाते
- [ ] Invoices मध्ये तुमच्या बिझनेसचे तपशील योग्यरित्या दिसतात
- [ ] Marketing site live आहे आणि checkout form ला link करते

### Onboarding {#onboarding}

- [ ] Quick Start widget नवीन customer dashboards वर दिसतो
- [ ] सर्व Quick Start links योग्य pages कडे पॉइंट करतात
- [ ] Welcome email sequence कॉन्फिगर आणि test केलेले आहे
- [ ] Knowledge base articles प्रकाशित आणि उपलब्ध आहेत
- [ ] Account page वर योग्य plan माहिती आणि upgrade options दिसतात

### Legal and Business {#legal-and-business}

- [ ] Terms of service प्रकाशित आणि checkout मधून link केलेले आहेत
- [ ] Privacy policy प्रकाशित आणि उपलब्ध आहे
- [ ] Refund policy परिभाषित आणि दस्तऐवजीकरण केलेली आहे
- [ ] पेमेंट स्वीकारण्यासाठी बिझनेस entity सेट केलेली आहे
- [ ] Payment gateway live mode मध्ये आहे (test mode मध्ये नाही)
- [ ] Tax configuration तुमच्या क्षेत्रासाठी योग्य आहे

## Soft Launch vs. Hard Launch {#soft-launch-vs-hard-launch}

दोन टप्प्यांचा launch विचार करा:

### Phase 1: Soft Launch {#phase-1-soft-launch}

सार्वजनिक लॉन्च करण्यापूर्वी 5-10 फिटनेस स्टुडिओ मालकांना साइन अप करण्यासाठी आमंत्रित करा. हे तुमचे beta ग्राहक आहेत. त्यांना खालील गोष्टींच्या बदल्यात मोठा सवलत (आयुष्यभर 50% सूट, किंवा 3 महिने मोफत) ऑफर करा:

- साइनअप आणि onboarding अनुभवावर प्रामाणिक अभिप्राय
- त्यांचे site एक showcase example म्हणून वापरण्याची परवानगी
- त्यांना येणाऱ्या कोणत्याही bugs किंवा समस्यांची तक्रार करणे

यामुळे तुम्हाला वास्तविक ग्राहक अभिप्राय आणि सार्वजनिक करण्यासाठी live sites मिळतात.

### Phase 2: Public Launch {#phase-2-public-launch}

Soft launch चा अभिप्राय समाविष्ट केल्यानंतर:

- Payment gateway live mode वर बदला
- तुमचा marketing site प्रकाशित करा
- ग्राहक मिळवण्यास सुरुवात करा (Lesson 11)
- संबंधित फिटनेस उद्योगाच्या चॅनेलवर घोषणा करा

## Launch Day Actions {#launch-day-actions}

ज्या दिवशी तुम्ही सार्वजनिक होणार आहात:

1. **live payments वर बदला** -- तुमच्या payment gateway वर test mode अक्षम करा
2. **पुन्हा एकदा तपासा** -- वास्तविक पेमेंटसह पूर्ण test signup करा (नंतर स्वतःला refund करा)
3. **जवळजवळ निरीक्षण करा** -- चुका, अयशस्वी signups किंवा पेमेंट समस्यांवर लक्ष ठेवा
4. **उपलब्ध रहा** -- तुमच्या पहिल्या वास्तविक ग्राहकांना मदतीची गरज भासू शकते, आणि जलद प्रतिसाद विश्वास निर्माण करतो
5. **थोड्यासाठी साजरा करा** -- आणि मग कामावर परत या

## What Can Go Wrong {#what-can-go-wrong}

यासाठी तयार रहा:

- **Payment gateway issues**: तुमच्या gateway provider चा support contact तयार ठेवा
- **SSL certificate problems**: सर्टिफिकेट कसे तपासावे आणि renew करावे हे जाणून घ्या
- **Email delivery failures**: ईमेल खरोखर पोहोचतात की नाही हे test करा (spam folders तपासा)
- **Performance under load**: जर तुम्हाला traffic spike मिळाला, तर तुमचे hosting कसे scale करायचे हे जाणून घ्या
- **Customer confusion**: तुमचा knowledge base आणि support channels तयार ठेवा
