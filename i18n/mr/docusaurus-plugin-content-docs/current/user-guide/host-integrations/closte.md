# Closte इंटिग्रेशन

## आढावा
Closte हे Google Cloud इन्फ्रास्ट्रक्चरवर तयार केलेले मॅनेज्ड WordPress होस्टिंग प्लॅटफॉर्म आहे. हे इंटिग्रेशन Ultimate Multisite आणि Closte यांच्यामध्ये आपोआप डोमेन सिंकिंग आणि SSL सर्टिफिकेट मॅनेजमेंट सक्षम करते.

## वैशिष्ट्ये
- आपोआप डोमेन सिंकिंग
- SSL सर्टिफिकेट मॅनेजमेंट
- वाइल्डकार्ड डोमेन सपोर्ट
- Closte वर चालवत असल्यास कोणत्याही कॉन्फिगरेशनची गरज नाही

## आवश्यकता
तुम्ही Closte वापरत असल्यास तुमच्या `wp-config.php` फाइलमध्ये खालील constant डिफाइन करणे आवश्यक आहे:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

तुम्ही Closte वर होस्टिंग करत असल्यास हा constant साधारणपणे आधीच डिफाइन केलेला असतो.

## सेटअप सूचना

### 1. तुमची Closte API Key व्हेरिफाय करा

तुम्ही Closte वर होस्टिंग करत असल्यास, `CLOSTE_CLIENT_API_KEY` constant तुमच्या `wp-config.php` फाइलमध्ये आधीच डिफाइन केलेला असायला हवा. तुमची `wp-config.php` फाइल तपासून तुम्ही हे व्हेरिफाय करू शकता.

### 2. इंटिग्रेशन सक्षम करा

1. तुमच्या WordPress admin मध्ये, Ultimate Multisite > Settings वर जा
2. "Domain Mapping" टॅबवर नेव्हिगेट करा
3. "Host Integrations" पर्यंत खाली स्क्रोल करा
4. Closte इंटिग्रेशन सक्षम करा
5. "Save Changes" वर क्लिक करा

## हे कसे काम करते

Ultimate Multisite मध्ये डोमेन मॅप केल्यावर:

1. इंटिग्रेशन तुमच्या अॅप्लिकेशनमध्ये डोमेन जोडण्यासाठी Closte च्या API ला रिक्वेस्ट पाठवते
2. Closte आपोआप SSL सर्टिफिकेट प्रोव्हिजनिंग हाताळते
3. डोमेन मॅपिंग काढून टाकल्यावर, इंटिग्रेशन Closte मधून डोमेन काढून टाकते

हे इंटिग्रेशन Ultimate Multisite मधील DNS चेक इंटरव्हल सेटिंगसोबत देखील काम करते, ज्यामुळे तुम्ही DNS प्रोपगेशन आणि SSL सर्टिफिकेट इश्यूअन्ससाठी सिस्टम किती वेळा तपासणी करते हे कॉन्फिगर करू शकता.

## डोमेन रेकॉर्ड तयार करणे

हे इंटिग्रेशन हे सुनिश्चित करते की साइट तयार केल्यावर किंवा डुप्लिकेट केल्यावर, डोमेन रेकॉर्ड आपोआप तयार होतो. Closte इंटिग्रेशनसाठी हे विशेषतः महत्त्वाचे आहे, कारण डोमेन रेकॉर्ड तयार झाल्यावर Closte API ला डोमेन आणि SSL सर्टिफिकेट तयार करण्यासाठी ट्रिगर होते.

## समस्यानिवारण

### API कनेक्शन समस्या
- तुमची Closte API key बरोबर आहे का ते व्हेरिफाय करा
- तुमच्या Closte अकाउंटला आवश्यक परवानग्या आहेत का याची खात्री करा

### SSL सर्टिफिकेट समस्या
- Closte ला SSL सर्टिफिकेट्स इश्यू करण्यासाठी थोडा वेळ लागू शकतो (साधारणपणे 5-10 मिनिटे)
- तुमचे डोमेन्स तुमच्या Closte सर्व्हरच्या IP अॅड्रेसकडे योग्यरित्या पॉइंट करत आहेत का ते व्हेरिफाय करा
- तुमच्या डोमेनसाठी DNS रेकॉर्ड्स योग्यरित्या कॉन्फिगर केले आहेत का ते तपासा

### डोमेन जोडले गेले नाही
- कोणत्याही एरर मेसेजसाठी Ultimate Multisite लॉग्स तपासा
- डोमेन आधीच Closte मध्ये जोडलेले नाही याची खात्री करा
- तुमच्या डोमेनचे DNS रेकॉर्ड्स योग्यरित्या कॉन्फिगर केले आहेत याची खात्री करा

### DNS चेक इंटरव्हल
- SSL सर्टिफिकेट्स इश्यू होण्यासाठी खूप वेळ लागत असल्यास, तुम्ही Domain Mapping सेटिंग्जमध्ये DNS चेक इंटरव्हल अॅडजस्ट करू शकता
- डीफॉल्ट इंटरव्हल 300 सेकंद (5 मिनिटे) आहे, परंतु टेस्टिंग दरम्यान जलद तपासणीसाठी तुम्ही ते 10 सेकंदांपर्यंत कमी करू शकता
