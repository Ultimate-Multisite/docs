# चेकआउट क्षेत्र आणि ग्राहक DNS व्यवस्थापन

## डोमेन निवड चेकआउट क्षेत्र {#the-domain-selection-checkout-field}

**डोमेन निवड** क्षेत्र हा एक चेकआउट घटक आहे जो ग्राहकांना त्यांच्या साइटचा डोमेन कसा मिळवायचा याचा पर्याय देतो. डोमेन विक्री सक्षम करण्यासाठी ते कोणत्याही चेकआउट फॉर्ममध्ये जोडा.

### चेकआउट फॉर्ममध्ये क्षेत्र जोडणे {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › चेकआउट फॉर्म** येथे जा
2. चेकआउट फॉर्म उघडा किंवा तयार करा
3. चेकआउट संपादकात, **क्षेत्र जोडा** क्लिक करा
4. क्षेत्र यादीतून **डोमेन निवड** निवडा
5. क्षेत्र पर्याय कॉन्फिगर करा (खाली पहा)
6. फॉर्म जतन करा

### क्षेत्र पर्याय {#field-options}

**डोमेन मोड** — ग्राहकाला कोणते टॅब दिसतील ते निवडा. प्रत्येक मोड स्वतंत्रपणे सक्षम किंवा अक्षम करता येतो:

| मोड | ते काय करते |
|---|---|
| **उपडोमेन** | ग्राहक आपल्या नेटवर्कवरील मोफत उपडोमेन वापरतो (उदा., `mysite.yournetwork.com`). पेमेंट आवश्यक नाही. |
| **नवीन डोमेन नोंदणी करा** | ग्राहक नवीन डोमेन शोधतो आणि आपल्या कॉन्फिगर केलेल्या प्रदात्यामार्फत ते नोंदणी करतो. किंमत ठरवण्यासाठी जुळणारे डोमेन उत्पादन वापरते. |
| **विद्यमान डोमेन** | ग्राहक स्वतःकडे आधीपासून असलेला डोमेन मॅप करतो. नोंदणी शुल्क नाही. डोमेन आपोआप त्यांच्या साइटशी मॅप केला जातो. |

**डीफॉल्ट मोड** — तिन्ही मोड सक्षम असताना, कोणता टॅब आधी उघडेल. डोमेन नोंदणी ऐच्छिक ठेवण्यासाठी **उपडोमेन** सेट करा, किंवा खरेदीला प्रोत्साहन देण्यासाठी **नवीन डोमेन नोंदणी करा** सेट करा.

**डोमेन उत्पादन** — वैकल्पिकरित्या हे क्षेत्र एखाद्या विशिष्ट डोमेन उत्पादनाशी पिन करा. सेट केलेले नसल्यास, ग्राहकाने शोधलेल्या TLD च्या आधारे अ‍ॅडऑन आपोआप जुळणारे उत्पादन निवडतो.

### नोंदणीधारक संपर्क क्षेत्रे {#registrant-contact-fields}

ग्राहकाने **नवीन डोमेन नोंदणी करा** टॅब निवडल्यावर, चेकआउट फॉर्म नोंदणीधारक संपर्क क्षेत्रे इनलाइन जोडतो:

- नाव / आडनाव
- ईमेल पत्ता
- पत्ता (ओळ १, शहर, राज्य/प्रांत, पोस्टल कोड, देश)
- फोन क्रमांक

ही सर्व नोंदणी संस्थांकडून आवश्यक असतात आणि नोंदणी API कॉल करण्यापूर्वी त्यांची पडताळणी केली जाते. फोन क्रमांक नोंदणी संस्थांना अपेक्षित असलेल्या `+CC.NNN` आंतरराष्ट्रीय स्वरूपात आपोआप फॉरमॅट केले जातात.

### आपोआप तयार होणारा साइट URL {#auto-generated-site-url}

ग्राहक डोमेन नोंदणी करतो किंवा मॅप करतो तेव्हा, साइट URL क्षेत्र निवडलेल्या डोमेनवरून आपोआप भरले जाते. ग्राहकांना स्वतंत्र URL क्षेत्र भरण्याची गरज नसते.

### शोध वर्तन {#search-behaviour}

- ग्राहक टाइप करत असताना AJAX सह डोमेन उपलब्धता रिअल-टाइममध्ये तपासली जाते
- पसंतीचा डोमेन उपलब्ध नसल्यास पर्यायी TLD सूचना दाखवल्या जातात
- किंमत थेट आणली जाते आणि स्पष्टपणे दाखवली जाते (नोंदणी किंमत, नूतनीकरण किंमत, वैकल्पिक WHOIS गोपनीयता शुल्क)
- कूपन कोड इतर कोणत्याही उत्पादनाप्रमाणेच डोमेन उत्पादनांवर लागू होतात

**शोध प्रतिसादक्षमता ट्यून करणे:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**डोमेन शोध फॉर्ममध्ये सानुकूल क्षेत्रे जोडणे:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ग्राहक DNS व्यवस्थापन {#customer-dns-management}

ग्राहक त्यांच्या नोंदणीकृत डोमेनसाठी DNS रेकॉर्ड **माझे Account** पृष्ठावरून, त्यांच्या डोमेनच्या नोंदीखाली व्यवस्थापित करू शकतात.

### समर्थित रेकॉर्ड प्रकार {#supported-record-types}

| प्रकार | वापर |
|---|---|
| **A** | होस्टनेम IPv4 पत्त्याशी मॅप करा |
| **AAAA** | होस्टनेम IPv6 पत्त्याशी मॅप करा |
| **CNAME** | दुसऱ्या होस्टनेमकडे निर्देश करणारे उपनाव तयार करा |
| **MX** | मेल एक्स्चेंज सर्व्हर सेट करा |
| **TXT** | SPF, DMARC, पडताळणी किंवा इतर मजकूर रेकॉर्ड जोडा |

### कोणते प्रदाते DNS व्यवस्थापनास समर्थन देतात? {#which-providers-support-dns-management}

DNS व्यवस्थापन (रेकॉर्ड जोडणे, संपादित करणे, हटवणे) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, आणि **Openprovider** सह उपलब्ध आहे. **Hostinger** डोमेन Domain Seller मार्फत नेमसर्व्हर अपडेट करू शकतात; होस्ट केलेल्या डोमेनसाठी DNS रेकॉर्ड मुख्य Hostinger डोमेन-मॅपिंग एकत्रीकरणाद्वारे व्यवस्थापित केले जातात. Namecheap, GoDaddy, आणि NameSilo डोमेन स्थिती आणि कालबाह्यता माहिती दाखवतात परंतु DNS थेट नोंदणी संस्थेच्या नियंत्रण पॅनेलमध्ये व्यवस्थापित करणे आवश्यक आहे.

### डीफॉल्ट DNS रेकॉर्ड {#default-dns-records}

डोमेन नोंदणी झाल्यावर आपोआप लागू होणारे डीफॉल्ट DNS रेकॉर्ड आपण कॉन्फिगर करू शकता. **सेटिंग्ज › Domain Seller › डीफॉल्ट DNS रेकॉर्ड** येथे जा.

डीफॉल्ट रेकॉर्ड मूल्ये दोन टोकनना समर्थन देतात:

| टोकन | याने बदलले जाते |
|---|---|
| `{DOMAIN}` | नोंदणीकृत डोमेन नाव (उदा., `example.com`) |
| `{SITE_URL}` | ग्राहकाच्या साइटसाठी WordPress साइट URL |

**उदाहरण — apex डोमेन आणि www आपल्या सर्व्हर IP कडे निर्देशित करा:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### प्रशासक: DNS पाहणे आणि संपादित करणे {#admin-viewing-and-editing-dns}

नेटवर्क प्रशासक **Network Admin › Ultimate Multisite › डोमेन** मधील डोमेनच्या संपादन पृष्ठावरून कोणत्याही ग्राहक डोमेनसाठी DNS रेकॉर्ड पाहू आणि संपादित करू शकतात.
