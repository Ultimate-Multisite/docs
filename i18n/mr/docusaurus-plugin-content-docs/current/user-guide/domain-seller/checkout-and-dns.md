# चेकआउट फील्ड आणि ग्राहक DNS व्यवस्थापन

## डोमेन निवड चेकआउट फील्ड

**Domain Selection** फील्ड हे एक चेकआउट घटक (checkout element) आहे जे ग्राहकांना त्यांच्या साइटचे डोमेन कसे मिळवायचे याबद्दल पर्याय देते. डोमेन विक्री सक्षम करण्यासाठी ते कोणत्याही चेकआउट फॉर्ममध्ये जोडा.

### चेकआउट फॉर्ममध्ये फील्ड जोडणे

1. **Network Admin › Ultimate Multisite › Checkout Forms** वर जा.
2. एक चेकआउट फॉर्म उघडा किंवा तयार करा.
3. चेकआउट एडिटरमध्ये, **Add Field** वर क्लिक करा.
4. फील्ड यादीतून **Domain Selection** निवडा.
5. फील्डचे पर्याय कॉन्फिगर करा (खाली पहा).
6. फॉर्म सेव्ह करा.

### फील्ड पर्याय

**Domain modes** — ग्राहक कोणते टॅब पाहतील हे निवडा. प्रत्येक मोड स्वतंत्रपणे सक्षम (enabled) किंवा अक्षम (disabled) केला जाऊ शकतो:

| Mode | काय करते |
|---|---|
| **Subdomain** | ग्राहक तुमच्या नेटवर्कवर एक मोफत सबडोमेन वापरतो (उदा. `mysite.yournetwork.com`). यासाठी पेमेंट करण्याची गरज नाही. |
| **Register New Domain** | ग्राहक नवीन डोमेन शोधतो आणि तो तुमच्या कॉन्फिगर केलेल्या प्रदात्याद्वारे (provider) नोंदणीकृत करतो. किंमत निश्चित करण्यासाठी जुळणारे डोमेन उत्पादन वापरले जाते. |
| **Existing Domain** | ग्राहक स्वतःचे मालकीचे असलेले डोमेन मॅप करतो. यासाठी कोणतेही नोंदणी शुल्क नाही. हे डोमेन आपोआप त्यांच्या साइटशी मॅप केले जाते. |

**Default mode** — जेव्हा तिन्ही मोड सक्षम असतात, तेव्हा कोणते टॅब प्रथम उघडेल. डोमेन नोंदणी ऐच्छिक ठेवण्यासाठी **Subdomain** वर सेट करा, किंवा खरेदीला प्रोत्साहन देण्यासाठी **Register New Domain** वर सेट करा.

**Domain product** — पर्यायीरित्या हे फील्ड विशिष्ट डोमेन उत्पादनाशी (domain product) जोडा. जर ते सेट केले नसेल, तर addon आपोआप ग्राहक ज्या TLD (Top-Level Domain) शोधतो त्यानुसार जुळणारे उत्पादन निवडतो.

### Registrant contact fields

जेव्हा ग्राहक **Register New Domain** टॅब निवडतो, तेव्हा चेकआउट फॉर्ममध्ये खालील नोंदणी संपर्क फील्ड (registrant contact fields) आपोआप जोडले जातात:

- पहिले नाव / आडनाव
- ईमेल पत्ता
- पत्ता (लाइन १, शहर, राज्य/प्रांत, पिन कोड, देश)
- फोन नंबर

हे सर्व रजिस्ट्रारसाठी आवश्यक आहेत आणि नोंदणी API कॉल करण्यापूर्वी त्यांची पडताळणी (validated) केली जाते. फोन नंबर आपोआप रजिस्ट्रारद्वारे अपेक्षित असलेल्या `+CC.NNN` आंतरराष्ट्रीय स्वरूपात स्वरूपित (formatted) केले जातात.

### Auto-generated site URL

जेव्हा ग्राहक डोमेन नोंदणीकृत करतो किंवा मॅप करतो, तेव्हा साइट URL फील्ड निवडलेल्या डोमेनमधून आपोआप भरले जाते. ग्राहकांना वेगळे URL फील्ड भरण्याची गरज नसते.

### Search behaviour

- ग्राहक टाइप करत असताना डोमेनची उपलब्धता AJAX सह रिअल-टाइममध्ये तपासली जाते.
- पसंतीचे डोमेन अनुपलब्ध असल्यास पर्यायी TLD सूचना दाखवल्या जातात.
- किंमत थेट मिळवली जाते आणि स्पष्टपणे प्रदर्शित केली जाते (नोंदणी किंमत, नूतनीकरण किंमत, पर्यायी WHOIS गोपनीयता शुल्क).
- कूपन कोड कोणत्याही इतर उत्पादनाप्रमाणेच डोमेन उत्पादनांना लागू होतात.

**शोध प्रतिसाद (search responsiveness) ट्यून करणे:**

```php
// API कॉल कमी करण्यासाठी debounce delay (milliseconds) वाढवा
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**डोमेन शोध फॉर्ममध्ये कस्टम फील्ड जोडणे:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ग्राहक DNS व्यवस्थापन

ग्राहक त्यांच्या नोंदणीकृत डोमेनसाठी **My Account** पृष्ठावरून, त्यांच्या डोमेनच्या एंट्रीखाली DNS रेकॉर्ड्स (DNS records) व्यवस्थापित करू शकतात.

### समर्थित रेकॉर्ड प्रकार

| Type | उपयोग |
|---|---|
| **A** | होस्टनेमला IPv4 पत्त्याशी मॅप करणे |
| **AAAA** | होस्टनेमला IPv6 पत्त्याशी मॅप करणे |
| **CNAME** | दुसऱ्या होस्टनेमकडे पॉइंट करणारा उपनाव (alias) तयार करणे |
| **MX** | मेल एक्सचेंज सर्व्हर सेट करणे |
| **TXT** | SPF, DMARC, पडताळणी किंवा इतर मजकूर रेकॉर्ड जोडणे |

### कोणते प्रदाते DNS व्यवस्थापन समर्थन देतात?

DNS व्यवस्थापन (रेकॉर्ड जोडणे, संपादित करणे, हटवणे) **OpenSRS**, **ResellerClub**, आणि **Enom** सह उपलब्ध आहे. Namecheap, GoDaddy, आणि NameSilo डोमेन स्थिती आणि समाप्ती माहिती दर्शवतात, परंतु DNS रजिस्ट्रारच्या कंट्रोल पॅनेलमध्ये थेट व्यवस्थापित करणे आवश्यक आहे.

### डीफॉल्ट DNS रेकॉर्ड

जेव्हा डोमेन नोंदणीकृत होते तेव्हा आपोआप लागू होणारे डीफॉल्ट DNS रेकॉर्ड तुम्ही कॉन्फिगर करू शकता. **Settings › Domain Seller › Default DNS Records** वर जा.

डीफॉल्ट रेकॉर्ड मूल्यांना दोन टोकन (tokens) समर्थन आहे:

| Token | कशाने बदलले जाते |
|---|---|
| `{DOMAIN}` | नोंदणीकृत डोमेनचे नाव (उदा. `example.com`) |
| `{SITE_URL}` | ग्राहकाच्या साइटसाठी WordPress साइट URL |

**उदाहरण — ॲपेक्स डोमेन आणि www ला तुमच्या सर्व्हर IP कडे पॉइंट करणे:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ॲडमिन: DNS पाहणे आणि संपादित करणे

नेटवर्क ॲडमिन कोणत्याही ग्राहक डोमेनचे DNS रेकॉर्ड **Network Admin › Ultimate Multisite › Domains** मधील डोमेनच्या एडिट पृष्ठावरून पाहू आणि संपादित करू शकतात.
