# Site Specification Skill

**Site Specification skill** ही एक व्यवस्थित पद्धत आहे, जी तुमच्या साइटचे उद्दिष्ट (goals), प्रेक्षक (audience) आणि ब्रँडची ओळख (brand identity) गोळा करण्यासाठी वापरली जाते. ही माहिती तुमच्या **site_brief** मेमरीमध्ये साठवली जाते, ज्याचा वापर एजंट्स (agents) वेगवेगळ्या सत्रांमध्ये (sessions) सातत्यपूर्ण आणि गरजेनुसार मदत देण्यासाठी करतात.

## Site Specification म्हणजे काय? {#what-is-site-specification}

Site Specification म्हणजे खालील गोष्टींचे दस्तऐवजीकरण (documenting) करणे:

- **साइटचा उद्देश (Site purpose)**: तुमची साइट काय करते आणि ती का अस्तित्वात आहे.
- **लक्ष्यित प्रेक्षक (Target audience)**: तुमच्या साइटला कोण भेट देते आणि त्यांना कशाची गरज आहे.
- **ब्रँड ओळख (Brand identity)**: तुमचे रंग, बोलण्याची पद्धत (tone) आणि व्हिज्युअल स्टाईल.
- **व्यवसायाचे उद्दिष्ट (Business goals)**: तुमच्या साइटसाठी यश कशाचे रूप आहे.
- **सामग्रीची रचना (Content structure)**: तुमची साइट कशी आयोजित केली आहे.

ही Specification तुमच्या **site_brief** बनते, जी एक कायमस्वरूपी मेमरी आहे, जी एजंट्स तुमच्या साइटच्या संदर्भाची (context) माहिती घेण्यासाठी वापरतात.

## Site Specification का वापरावे? {#why-use-site-specification}

### सत्रांमध्ये सातत्य (Consistency Across Sessions) {#consistency-across-sessions}

site_brief शिवाय, तुम्हाला प्रत्येक वेळी नवीन सत्र सुरू करताना तुमच्या साइटचा उद्देश पुन्हा समजावून सांगण्याची गरज पडेल. site_brief मुळे, एजंट्स लगेच समजतात:

- तुमच्या साइटची उद्दिष्ट्ये आणि प्रेक्षक
- तुमचे ब्रँड रंग आणि बोलण्याची पद्धत
- तुमची सामग्रीची रचना
- तुमचे व्यावसायिक उद्दिष्ट्ये

### चांगली शिफारसी (Better Recommendations) {#better-recommendations}

एजंट्स तुमच्या site_brief चा वापर खालील गोष्टींसाठी करतात:

- तुमच्या साइटच्या उद्देशानुसार वैशिष्ट्ये (features) सुचवणे
- तुमच्या उद्दिष्टांशी जुळणारी सामग्रीची रचना सुचवणे
- तुमच्या ब्रँडशी सुसंगत डिझाइन प्रस्तावित करणे
- विसंगत वैशिष्ट्ये सुचवणे टाळणे

### जलद ऑनबोर्डिंग (Faster Onboarding) {#faster-onboarding}

नवीन एजंट्स (किंवा नवीन सत्रातील एजंट्स) स्पष्टीकरण प्रश्न विचारण्याऐवजी तुमचा site_brief वाचून लवकर कामाला लागतात.

## Site Specification सुरू करणे {#initiating-site-specification}

### Theme Builder Onboarding दरम्यान {#during-theme-builder-onboarding}

Site Specification skill आपोआप **Theme Builder onboarding flow** दरम्यान सुरू होते. Setup Assistant एजंट प्रश्न विचारतो आणि तुमचा site_brief तयार करतो.

### मॅन्युअल सुरुवात (Manual Initiation) {#manual-initiation}

तुम्ही कधीही Site Specification सुरू करू शकता:

```
"Let's define my site specification"
```

किंवा

```
"Help me create a site brief"
```

## Site Specification प्रक्रिया {#the-site-specification-process}

### पायरी १: साइटचा उद्देश (Site Purpose) {#step-1-site-purpose}

एजंट विचारतो:

```
What is your site's primary purpose?
- E-commerce store
- Blog or content site
- Portfolio or showcase
- SaaS application
- Community or forum
- Other: [describe]
```

तुम्ही एक श्रेणी निवडू शकता किंवा तुमचा स्वतःचा उद्देश वर्णन करू शकता.

### पायरी २: लक्ष्यित प्रेक्षक (Target Audience) {#step-2-target-audience}

```
Who is your primary audience?
- Consumers / general public
- Business professionals
- Developers / technical users
- Students / educators
- Other: [describe]

What are their main needs?
```

### पायरी ३: ब्रँड ओळख (Brand Identity) {#step-3-brand-identity}

```
What are your brand colors?
- Primary color: [color picker or hex code]
- Secondary color: [color picker or hex code]
- Accent color: [optional]

How would you describe your brand tone?
- Professional / corporate
- Creative / artistic
- Playful / casual
- Minimal / modern
- Warm / friendly
```

### पायरी ४: व्यावसायिक उद्दिष्ट्ये (Business Goals) {#step-4-business-goals}

```
What does success look like for your site?
- Generate leads
- Sell products
- Build community
- Share knowledge
- Establish authority
- Other: [describe]

What's your primary metric?
- Revenue
- User engagement
- Content reach
- Conversions
- Other
```

### पायरी ५: सामग्रीची रचना (Content Structure) {#step-5-content-structure}

```
How is your content organized?
- Flat (all content at same level)
- Hierarchical (categories and subcategories)
- Chronological (blog-style)
- Product-based (catalog)
- Other: [describe]

What content types do you use?
- Blog posts
- Product pages
- Case studies
- Documentation
- Videos
- Other
```

## तुमचा site_brief मेमरी {#your-sitebrief-memory}

Site Specification पूर्ण केल्यानंतर, तुमची माहिती तुमच्या एजंट मेमरीमध्ये **site_brief** म्हणून साठवली जाते. यात खालील माहिती असलेला एक व्यवस्थित रेकॉर्ड असतो:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## तुमचा site_brief पाहणे आणि अद्यतनित करणे (Viewing and Updating) {#viewing-and-updating-your-sitebrief}

### तुमचा site_brief पहा {#view-your-sitebrief}

एजंटला विचारा:

```
"Show me my site brief"
```

किंवा

```
"What do you know about my site?"
```

एजंट तुम्हाला तुमची साठवलेली Specification दाखवेल.

### तुमचा site_brief अद्यतनित करा (Update) {#update-your-sitebrief}

जर तुमची साइट विकसित झाली असेल, तर तुम्ही ती अद्यतनित करू शकता:

```
"Update my site brief: we're now targeting B2B customers"
```

किंवा

```
"Refresh my site specification"
```

यामुळे Site Specification skill पुन्हा चालेल आणि तुमच्या सध्याच्या माहितीचा वापर प्रारंभिक बिंदू म्हणून केला जाईल.

## एजंट्स site_brief चा वापर कसे करतात {#how-agents-use-sitebrief}

### डिझाइन शिफारसी (Design Recommendations) {#design-recommendations}

जेव्हा तुम्ही डिझाइन बदलांची मागणी करता, तेव्हा एजंट्स तुमच्या site_brief चा संदर्भ घेतात:

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### वैशिष्ट्ये सुचवणे (Feature Suggestions) {#feature-suggestions}

एजंट्स तुमच्या उद्दिष्टांशी जुळणारी वैशिष्ट्ये सुचवतात:

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### सामग्रीची रचना (Content Structure) {#content-structure}

एजंट्स तुमच्या रचनेवर आधारित सामग्रीचे आयोजन प्रस्तावित करतात:

```
You: "How should I organize my blog?"
Agent: "Since you use a product catalog with blog, I'd suggest
organizing posts by product category and creating 'styling guides'
that showcase multiple products together."
```

## सर्वोत्तम पद्धती (Best Practices) {#best-practices}

### विशिष्ट रहा (Be Specific) {#be-specific}

"सामान्य प्रेक्षक" याऐवजी, तुमच्या वास्तविक प्रेक्षकांचे वर्णन करा:

- ✓ "Women aged 25-45, interested in sustainable fashion"
- ✗ "Everyone"

### नियमितपणे अद्यतनित करा (Update Regularly) {#update-regularly}

तुमची साइट विकसित होत असताना, तुमचा site_brief अद्यतनित करा:

- जेव्हा तुम्ही नवीन प्रेक्षकांकडे वळता (pivot)
- जेव्हा तुम्ही नवीन उत्पादन श्रेणी जोडता
- जेव्हा तुमची ब्रँड ओळख बदलते
- जेव्हा तुमची व्यावसायिक उद्दिष्ट्ये बदलतात

### सुसंगत शब्दजगाचा वापर करा (Use Consistent Terminology) {#use-consistent-terminology}

सत्रामध्ये समान शब्दांचा वापर करा:

- ✓ नेहमी "sustainable jewelry" म्हणा (न की "eco-friendly jewelry" आणि "green products")
- ✓ तुमच्या प्रेक्षकांचा नेहमी त्याच प्रकारे उल्लेख करा

### संदर्भ द्या (Include Context) {#include-context}

एजंट्सना तुमचे निर्णय समजण्यास मदत करणारी पार्श्वभूमी माहिती द्या:

- "आम्ही अशा व्यावसायिकांना लक्ष्य करत आहोत ज्यांना किमतीपेक्षा गुणवत्ता जास्त महत्त्वाची वाटते."
- "आमचे प्रेक्षक तंत्रज्ञान-जाणकार आहेत आणि त्यांना आधुनिक डिझाइनची अपेक्षा आहे."
- "आम्ही एक स्व-वित्तपोषित स्टार्टअप आहोत, त्यामुळे आम्हाला कमी खर्चाचे उपाय लागतील."

## Theme Builder Onboarding शी संबंध {#relationship-to-theme-builder-onboarding}

Site Specification skill **Theme Builder onboarding flow** मध्ये समाविष्ट आहे. जेव्हा तुम्ही ऑनबोर्डिंग पूर्ण करता, तेव्हा तुम्ही दिलेल्या माहितीसह तुमचा site_brief आपोआप तयार होतो.

तुम्हाला स्वतंत्रपणे Site Specification चालवायचे असल्यास, तुम्ही ते करू शकता:

- प्रारंभिक सेटअपनंतर तुमची Specification सुधारण्यासाठी
- तुमची साइट विकसित होत असताना तुमचा site brief अद्यतनित करण्यासाठी
- Theme Builder सुरू करण्यापूर्वी तपशीलवार Specification तयार करण्यासाठी

## समस्या निवारण (Troubleshooting) {#troubleshooting}

**माझा site_brief वापरला जात नाहीये**
- तपासा की एजंटला मेमरीमध्ये प्रवेश आहे की नाही.
- एजंटला "recall my site brief" (माझा site brief आठवा) असे विचारण्यास सांगा.
- तपासा की तुमच्या सेटिंग्जमध्ये मेमरी सक्षम (enabled) आहे की नाही.

**मला नवीन site_brief सह सुरुवात करायची आहे**
- एजंटला विचारा: "Clear my site brief and start fresh" (माझा site brief साफ करा आणि नवीन सुरुवात करा).
- त्यानंतर Site Specification पुन्हा चालवा.

**एजंट माझ्या site_brief शी जुळणारे शिफारसी करत नाहीये**
- एजंटला "review my site brief" (माझा site brief तपासा) असे विचारण्यास सांगा.
- जर तो जुना झाला असेल, तर तुमचा site_brief अद्यतनित करा.
- तुमच्या विनंत्यांमध्ये अतिरिक्त संदर्भ द्या.

## पुढील पाऊले (Next Steps) {#next-steps}

तुमची साइट Specification परिभाषित केल्यानंतर:

1. **Theme Builder वापरा**: तुमच्या site_brief वर आधारित एक कस्टम थीम तयार करा.
2. **डिझाइन सुधारित करा**: तपशीलवार डिझाइन कामासाठी Design System Aesthetics skill वापरा.
3. **सामग्रीची योजना करा**: सामग्रीच्या रचनेसाठी एजंट्सकडून शिफारसी विचारा.
4. **वैशिष्ट्ये तयार करा**: तुमच्या व्यावसायिक उद्दिष्टांशी जुळणारी वैशिष्ट्ये विनंती करा.
