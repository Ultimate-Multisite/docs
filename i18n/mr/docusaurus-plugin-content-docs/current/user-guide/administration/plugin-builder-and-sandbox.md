# Plugin Builder आणि Sandbox

Gratis AI Agent v1.5.0 मध्ये **Plugin Builder & Sandbox System** सादर केले आहे. या सिस्टीममुळे AI असिस्टंटला तुमच्या नेटवर्कवर WordPress प्लगइन्स तयार करण्याची, सक्रिय करण्याची आणि व्यवस्थापित करण्याची क्षमता मिळते — आणि हे सर्व एका सुरक्षित, वेगळ्या (isolated) sandbox वातावरणातून होते.

## विहंगावलोकन (Overview)

Plugin Builder AI असिस्टंटला नैसर्गिक भाषेतील विनंत्यांच्या आधारावर कस्टम WordPress प्लगइन्स लिहिण्यास सक्षम करतो. तयार केलेले प्लगइन्स थेट लाईव्ह साइटच्या कार्यक्षमतेवर परिणाम करण्यापूर्वी, ते व्हॅलिडेट केले जातात, साठवले जातात आणि एका sandbox लेयरमध्ये सक्रिय केले जातात.

याची काही उपयुक्त उदाहरणे (Use cases) खालीलप्रमाणे आहेत:

- डेव्हलपरची मदत न घेता हलके (lightweight) युटिलिटी प्लगइन्स तयार करणे.
- WordPress hooks किंवा custom post types ची आवश्यकता असलेल्या फीचर्सचे प्रोटोटाइप बनवणे.
- बॅच ऑपरेशन्ससाठी अल्पकाळ टिकणारे ऑटोमेशन स्क्रिप्ट्स तयार करणे.

## AI द्वारे प्लगइन तयार करणे (Generating a Plugin via AI)

प्लगइन तयार करण्यासाठी, Gratis AI Agent च्या chat इंटरफेसवर जा आणि तुम्हाला काय हवे आहे याचे वर्णन करा. उदाहरणार्थ:

> "डॅशबोर्डवर एक कस्टम ॲडमिन नोटीस जोडणारे प्लगइन तयार करा."

AI खालील गोष्टी करेल:

1. स्ट्रक्चर्ड कोड जनरेशन वापरून प्लगइनचे PHP कोड तयार करेल.
2. सिंटॅक्स त्रुटी (syntax errors) आणि असुरक्षित पॅटर्नसाठी आउटपुटची व्हॅलिडेशन करेल.
3. तयार केलेले प्लगइन sandbox स्टोअरमध्ये सेव्ह करेल.
4. प्लगइन स्लग् (slug) आणि **Activate in Sandbox** बटण असलेले पुष्टीकरण (confirmation) परत करेल.

सक्रिय करण्यापूर्वी तुम्ही त्याच संभाषण थ्रेडमध्ये (conversation thread) फॉलो-अप करून परिणाम सुधारू शकता.

## Sandbox सक्रिय करणे (Sandbox Activation)

sandbox मध्ये तयार केलेले प्लगइन सक्रिय करणे, ते लाईव्ह नेटवर्कवर सक्रिय करण्यापेक्षा वेगळे आहे. हे sandbox:

- प्लगइन एका वेगळ्या WordPress वातावरणात (wp-env) चालवते.
- कोणतेही PHP त्रुटी (errors), चेतावणी (warnings), किंवा hook संघर्ष (hook conflicts) कॅप्चर करते.
- सक्रिय करण्याचे परिणाम chat इंटरफेसमध्ये रिपोर्ट करते.

sandbox मध्ये प्लगइन सक्रिय करण्यासाठी, AI chat response मधील **Activate in Sandbox** बटणावर क्लिक करा, किंवा स्लॅश कमांड (slash command) वापरा:

```
/activate-plugin <plugin-slug>
```

एक स्टेटस मेसेज (status message) सक्रियण यशस्वी झाले की अयशस्वी झाले याची पुष्टी करतो. अयशस्वी झाल्यास, त्रुटीचा लॉग (error log) chat थ्रेडमध्ये प्रदर्शित होतो.

## तयार प्लगइन्सचे व्यवस्थापन (Managing Generated Plugins)

तयार केलेले प्लगइन्स **Gratis AI Agent → Plugin Builder → Manage Plugins** येथे सूचीबद्ध केले जातात. या स्क्रीनवरून तुम्ही खालील गोष्टी करू शकता:

| कृती (Action) | वर्णन (Description) |
|---|---|
| **View source** | संपूर्ण प्लगइन PHP कोड तपासा. |
| **Re-activate in sandbox** | sandbox सक्रियण तपासणी पुन्हा चालवा. |
| **Install on network** | प्लगइन लाईव्ह नेटवर्कवर तैनात (deploy) करा (मॅन्युअल पुष्टीकरण आवश्यक). |
| **Update** | AI द्वारे नवीन आवृत्ती (version) द्या, जी विद्यमान कोडला बदलेल. |
| **Delete** | प्लगइन sandbox स्टोअरमधून काढा. प्रथम सर्व साइट्सवरून ते निष्क्रिय (deactivate) करते. |

:::warning
**Install on network** हे तयार केलेले प्लगइन तुमच्या लाईव्ह WordPress multisite वर तैनात करते. पुढे जाण्यापूर्वी प्लगइन कोड तपासा. लाईव्ह इन्स्टॉल पूर्ण करण्यापूर्वी Gratis AI Agent पुष्टीकरणासाठी विचारणा करेल.
:::

## नेटवर्कवर प्लगइन इन्स्टॉल करणे (Installing a Generated Plugin on the Network)

जेव्हा तुम्ही sandbox मध्ये प्लगइनने समाधान मिळवले आहे, तेव्हा तुम्ही ते लाईव्ह नेटवर्कवर इन्स्टॉल करू शकता:

1. **Gratis AI Agent → Plugin Builder → Manage Plugins** वर जा.
2. जे प्लगइन तुम्हाला तैनात करायचे आहे, त्याच्या बाजूला **Install on Network** वर क्लिक करा.
3. डायलॉगची पुष्टी करा. प्लगइन `wp-content/plugins/` मध्ये इन्स्टॉल होते आणि नेटवर्कवर सक्रिय होते.

पर्यायीरित्या, chat इंटरफेसमध्ये स्लॅश कमांड वापरा:

```
/install-plugin <plugin-slug>
```

## प्लगइन अपडेट्स (Plugin Updates)

तयार केलेल्या प्लगइनला अपडेट करण्यासाठी, नवीन संभाषणात (new conversation) AI असिस्टंटला बदलाचे वर्णन करा:

> "dashboard-notice प्लगइन अपडेट करा जेणेकरून नोटीस फक्त ॲडमिनिस्ट्रेटरला दिसेल."

AI एक नवीन आवृत्ती तयार करते, जी सध्याच्या आवृत्तीसोबत sandbox मध्ये दिसते. अपडेट लागू करण्यापूर्वी तुम्ही diff तपासा आणि पुष्टी करा.

## HookScanner इंटिग्रेशन (HookScanner Integration)

Plugin Builder प्रत्येक तयार केलेल्या प्लगइनद्वारे नोंदवलेल्या hooks आणि filters चे विश्लेषण करण्यासाठी एक इंटिग्रेटेड **HookScanner** वापरतो. HookScanner चा आउटपुट chat response मध्ये दाखवला जातो आणि त्यात खालील गोष्टींचा समावेश असतो:

- नोंदवलेले Action hooks (`add_action` कॉल्स).
- नोंदवलेले Filter hooks (`add_filter` कॉल्स).
- प्लगइन डिपेंडन्सीमध्ये आढळलेले कोणतेही hooks (यामध्ये `vendor/` आणि `node_modules/` डिरेक्टरी वगळल्या जातात).

हे प्लगइन सक्रिय करण्यापूर्वी त्याचे वर्तन (behaviour) समजून घेण्यास मदत करते.

## सुरक्षा विचार (Security Considerations)

- तयार केलेले प्लगइन्स मॅन्युअली इन्स्टॉल केलेल्या प्लगइन्सपासून वेगळे साठवले जातात आणि तुम्ही ते नेटवर्कवर स्पष्टपणे इन्स्टॉल करेपर्यंत ते मानक WordPress प्लगइन व्यवस्थापन स्क्रीनवरून उपलब्ध नसतात.
- प्लगइन फाईल्स लिहिताना डिरेक्टरी ट्रॅव्हर्सल (directory traversal) रोखण्यासाठी sandbox path validation वापरते.
- धोकादायक फंक्शन कॉल्स (उदा. `eval`, `exec`, `system`) असलेल्या प्लगइन्स व्हॅलिडेशन दरम्यान फ्लॅग केले जातात आणि ते सक्रिय केले जाणार नाहीत.
