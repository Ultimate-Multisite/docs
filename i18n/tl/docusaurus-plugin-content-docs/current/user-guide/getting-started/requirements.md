# Mga Kinakailangan para sa Ultimate Multisite

Ang unang bagay na maaaring itanong mo sa sarili bago bumili ng Ultimate Multisite ay ano ang mga requirements para tumakbo ang ating plugin. Mabuti naman, iilan lang ang pangunahing kailangan para tumakbo ang Ultimate Multisite: isang **domain name**, isang magandang **hosting provider**, at isang WordPress **Multisite** installation. Iyan lang ang kailangan mo!

## Domain Name

Una, pag-usapan natin ang domain name.

Kapag pumipili ng domain name, isipin mo kung ano ang iyong specific niche at kung ano ang gagawin mo gamit ang Ultimate Multisite: Gusto mo bang gumawa ng agency? Magbenta ng mga website na parang shop? O baka naman WordPress web hosting? Lahat ay posible sa Ultimate Multisite.

Ang plugin namin ay nagbibigay ng mga serbisyo para sa subscription, koleksyon ng bayad, checkout forms, discount vouchers, at komunikasyon sa customer. Maraming trabaho ang kailangan para i-install, i-configure, at mapanatili ang WordPress Multisite, pero ginagawang madali ito ng Ultimate Multisite.

Anumang niche na pipiliin mo ay magagamit gamit ang aming plugin basta nagbebenta ka ng mga website bilang serbisyo! Siguraduhin lang na pumili ka ng domain name na tugma sa plano ng iyong negosyo.

## Hosting Provider

Ngayon na mayroon ka nang magandang pangalan ng domain na kumakatawan sa iyong niche at negosyo, kailangan mo nang pumili ng magandang web hosting company para i-host ang iyong network gamit ang Ultimate Multisite. May ilang magagandang opsyon sa merkado. Ang ilang mga hosting provider ay mahusay na nakakasama (integrate) sa Ultimate Multisite, na maaaring maging malaking salik depende sa kung ano ang plano mong itayo.

Ang Ultimate Multisite ay gumagana sa anumang hosting provider. Ang tanging pagkakaiba ay ang ilang hosting company tulad ng _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ at _Runcloud_ ay may mas mahigpit na integrasyon para sa domain mapping. Para sa ibang mga hosting, maaaring kailanganin mong i-map nang mano-mano (manually) ang iyong custom domains. Magandang ideya rin na pumili ng hosting provider na sumusuporta sa AutoSSL. Sinusuportahan ito ng _Closte_, _Runcloud_ at _cPanel_.

Pagdating sa mga detalye ng server specifications, walang eksaktong formula dahil nakadepende ito sa mga features at functions na inaalok mo at sa laki ng network mo. Ang shared hosting ay maaaring maging magandang simula, pero kapag nagsimulang lumaki ang iyong network, malamang kailangan mo nang dedicated VPS.

## WordPress Multisite

Ngayon, pag-usapan natin nang kaunti ang tungkol sa isang WordPress Multisite installation.

Pinapayagan ng WordPress Multisite kang magkaroon ng isang network ng mga site sa iisang installation. Ito ay isang built-in feature, pero hindi ito naka-activate by default. Kailangan mong i-enable ang feature na ito sa loob ng iyong WordPress installation at web hosting control panel.

Ang paggamit ng WordPress na may Multisitng enabled ay ang pundasyon para sa **Ultimate Multisite**. Dahil ang pangunahing gamit ng Ultimate Multisite ay tumulong sa mga user na mag-manage at magbenta ng mga website bilang serbisyo, kailangan itong gawin sa WordPress na may Multisitng enabled.

Mayroon tayong **malalim na artikulo** na nagtuturo step-by-step kung ano ang kailangang gawin para i-enable ang Multisite feature sa iyong WordPress installation.

Pagkatapos mong ma-enable ang Multisite sa WordPress, oras na para mag-install ng Ultimate Multisite. Para maayos na ma-install ang Ultimate Multisite, dapat ay mayroon ka ng:

*   WordPress: v5.3+ (Inirerekomenda: Huling stable version)
*   PHP: 7.4.x (Magkakaroon ng suporta para sa > 8.0 sa lalong madaling panahon)
*   MySQL: v5+ (Inirerekomenda: 5.6, kung wala kang 8.0 na available sa iyong hosting provider)

Ito lang ang mga software requirements para sa Ultimate Multisite.

Tandaan na ang Ultimate Multisite ay maaaring gumana sa PHP 8.0, pero inirerekomenda namin na tumakbo ito gamit ang PHP 7.4.x.

Inirerekomenda din namin na i-install ito sa mga **main domains**, hindi sa subdomains o subdirectories. Maaaring gumana ito sa subdomains/subdirectories, pero baka magkaroon ng ilang error.

Para sa detalyadong gabay kung paano i-install ang Ultimate Multisite, **mag-click dito**.
