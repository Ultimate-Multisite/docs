# Multi-Tenancy

Ipamahagi ang mga sub-sityo sa iba't ibang server at base ng datos para sa pagganap at seguridad

## Mga tampok na inilabas sa bersyon 1.2.0

Nagdaragdag ang Multi-Tenancy 1.2.0 ng pundasyong tenant na soberano para sa mga workflow ng isolasyon sa produksiyon:

- Walang-estadong awtomatikong pag-login ng tenant na may maikli ang bisa at protektado laban sa replay na mga pagbisita sa SSO.
- Bawat-sub-sityong isolasyon ng base ng datos at file system para sa mga tenant na soberano.
- Kakayahang multi-tenancy ng Hostinger para sa provisioning ng tenant na may kamalayan sa host.
- Mga gate ng beripikasyon ng migrasyong soberano para sa mga pagsusuri ng lumang datos at async na pagpoproseso ng push.
- Nako-configure na pagbubuklod ng host ng base ng datos sa parehong makina para sa mga install ng Bedrock, FrankenPHP, at localhost.
- Mga pagpapahusay sa bootstrap ng tenant, tagasulat ng base ng datos, bootstrap ng schema, at provisioning ng user.
- Mas ligtas na pagbuwag na soberano na naglilinis ng mga kredensyal ng tenant.

Tingnan ang mga gabay ng user para sa [workflow ng migrasyong soberano](/user-guide/getting-started/sovereign-migration-workflow), [isolasyon ng multi-tenancy](/user-guide/configuration/multi-tenancy-isolation), at [mga operasyon ng tenant na soberano](/user-guide/administration/sovereign-tenant-operations).

## Mga Kinakailangan

- WordPress 5.3 o mas mataas
- PHP 7.4 o mas mataas
- Multisite Ultimate plugin (aktibo)

## Pag-install

1. I-upload ang mga file ng add-on sa iyong direktoryong `/wp-content/plugins/`
2. I-activate ang plugin sa pamamagitan ng menu na 'Plugins' sa WordPress
3. I-configure ang mga setting ng add-on sa admin ng network

### Pagsusuri

Kasama sa add-on ang isang komprehensibong framework ng pagsusuri:

- **Mga Unit Test**: Mga test na nakabatay sa PHPUnit na may test suite ng WordPress
- **Base na Class ng Test**: Nagbibigay ng mga helper method para sa mga karaniwang senaryo ng test
- **Saklaw ng Test**: Naka-configure upang subaybayan ang saklaw ng code
- **Pagsusuri sa Multisite**: Tumatakbo ang mga test sa kapaligirang multisite

### Mga Pamantayan ng Code

- **Mga Pamantayan sa Pag-code ng WordPress**: Ipinapatupad sa pamamagitan ng PHPCS
- **Pagiging Compatible sa PHP 7.4+**: Sinusuportahan ang mga modernong feature ng PHP
- **Static na Pagsusuri**: Integrasyon ng PHPStan para sa pagsusuri ng uri
- **Pagmodernisa ng Code**: Mga panuntunan ng Rector para sa mga upgrade ng PHP

### Sistema ng Build

- **Pagproseso ng Asset**: Minification para sa mga CSS/JS file
- **Pagsasalin**: Pagbuo ng POT file
- **Pagpa-package**: Paglikha ng archive para sa distribusyon
- **Mode ng Pag-develop**: Mga hindi naka-minify na asset para sa pag-debug

## Configuration

Sinusuportahan ng add-on ang iba't ibang opsyon sa configuration na maa-access sa pamamagitan ng interface ng admin ng network.

### Configuration ng tenant na soberano

Para sa mga tenant na soberano, i-configure ang base ng datos ng tenant, ugat ng file system, pagbubuklod ng host ng base ng datos, at kakayahan ng host bago iruta ang trapiko sa tenant. Patakbuhin ang beripikasyon ng migrasyon pagkatapos ng bawat pagbabago na nakaaapekto sa mga kredensyal ng base ng datos, pagbubuklod ng host, pagruruta, o provisioning ng user.

Gamitin ang `localhost` lamang kapag inaasahan ng runtime ng tenant at mga grant ng base ng datos ang lokal na socket o access sa parehong makina. Gamitin ang `127.0.0.1` o hostname ng serbisyo kapag iyon ang grant host na na-configure ng iyong provider.

### Kakayahan ng Hostinger

Gumagana ang kakayahang multi-tenancy ng Hostinger kasama ang nakabahaging integration token ng Hostinger. Kumpirmahing naroroon ang domain sa hPanel, nakaturo ang DNS sa Hostinger account, at tumutugma ang mga setting ng base ng datos ng tenant sa hPanel bago ang beripikasyon.

## Mga Hook at Filter

### Mga Action

- `ultimate-multisite-addon-template_init` - Pinapagana pagkatapos ng initialization ng add-on
- `ultimate-multisite-addon-template_loaded` - Pinapagana pagkatapos ma-load ang lahat ng file ng add-on

### Mga Filter

- `ultimate-multisite-addon-template_settings` - Baguhin ang mga setting ng add-on
- `ultimate-multisite-addon-template_enabled` - I-override ang status ng pag-enable/pag-disable ng add-on
