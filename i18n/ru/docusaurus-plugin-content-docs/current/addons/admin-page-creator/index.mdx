# Admin Page Creator

## Content
Admin Page Creator позволит вам легко добавлять новые административные страницы и подпункты в админ‑панели WordPress. Он может использоваться для создания страницы приветствия или учебной страницы, которые могут служить целевой страницей для новых пользователей. Вы можете использовать конструктор страниц, такой как Beaver Builder, Elementor, Brizy и Oxygen, чтобы без труда разрабатывать страницы.

Он также поддерживает пользовательский HTML, CSS и JavaScript, а также контроль доступа на основе плана и роли пользователя!

## Features
* Поддержка пользовательского HTML и CSS
* Поддержка JavaScript
* PHP Content Mode
* Разрешения на основе роли пользователя
* Поддержка конструкторов страниц

## Installation

1. Загрузите **ultimate-multisite-admin-page-creator** в каталог **/wp-content/plugins/**
2. Активируйте плагин через меню **Plugins** в админке сети WordPress

## Usage Guide

Хотя WordPress делает добавление контента на фронтенд сайтов действительно простым, создание страниц и подпунктов на админ‑стороне требует некоторых базовых навыков программирования. Ultimate Multisite: Admin Page Creator стремится перенести ту же простоту использования фронтенда на бэкенд ваших сетевых сайтов.

## Why use Admin Page Creator?

Возможно, вы хотите предложить клиентам пользовательскую страницу в их админ‑панели с учебными материалами, а может быть, вы хотите добавить пользовательскую страницу с FAQ и виджетом контактов. Возможности с Admin Page Creator безграничны, и именно это мы разберем в сегодняшнем руководстве.

## Creating your first Custom Admin Page

После установки и активации Ultimate Multisite: Admin Page Creator перейдите в меню сети админки и откройте раздел Admin Pages.

![Custom Admin Pages list on Network Admin](/img/admin/admin-pages-list.png)

_Список пользовательских административных страниц в сети админки_

Вы увидите список всех административных страниц, которые вы создали до сих пор (поскольку это ваш первый визит, список будет пустым). Нажмите **Add new Admin Page**. Вы будете перенаправлены в редактор страницы.

## Menu Options

С боковой панели, в мета‑боксе Общие параметры, вы сможете настроить тип меню, метку и многое другое.

### Menu Types

Новая пользовательская административная страница может иметь два типа меню: верхний уровень меню и подпункт.

Страница верхнего уровня меню, как следует из названия, добавит вашу новую страницу в меню админки рядом со стандартными страницами WordPress.

Если вы выберете вариант верхнего уровня, вы сможете выбрать порядок меню и значок меню.

Если вы хотите сделать эту страницу подпунктом меню, вы сможете выбрать родительскую страницу, к которой будет привязана эта страница. Вы можете добавить свою пользовательскую страницу к стандартным страницам WordPress или к вашим собственным страницам верхнего уровня.

## Advanced Options

Вы должны иметь возможность добавлять различные типы контента на ваши пользовательские административные страницы. Это включает правила и файлы CSS, JavaScript и внешние библиотеки. Вы можете сделать это, используя вкладку Расширенные параметры.

### Custom CSS

На вкладке CSS вы можете добавить свой собственный CSS‑код, а также импортировать CSS‑файлы с удаленных сайтов (например, CDN, Google Fonts и т.д.).

То же самое справедливо для JavaScript. Вы можете добавить внешние библиотеки для использования в вашем пользовательском JavaScript‑коде.

## Permissions

Также в разделе Расширенные параметры вы сможете выбрать, какие под‑сайты будут иметь доступ к этой странице. Это полезно, если вы планируете предоставлять разный эксклюзивный контент разным уровням планов, например.

<!-- Screenshot unavailable: Admin Page Creator permissions settings for plan and role-based access control -->

_У вас полный контроль над тем, где будет отображаться эта страница_

**Важно** : Для отображения страницы необходимо выполнить оба условия. Это означает, что если пользователь является клиентом определенного плана, но не имеет одной из выбранных ролей (или наоборот), страница не появится в их панели.

## Active

Если вы хотите отключить пользовательскую административную страницу без удаления или изменения разрешений, просто снимите галочку с опции Активно.

Отключение страницы верхнего уровня автоматически отключит все подпункты, привязанные к этой родительской странице.
