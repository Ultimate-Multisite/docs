# Тарифы с оплатой по счетчику {#metered-plans}

Аддон Ultimate Multisite: Тарифы с оплатой по счетчику меняет подход к выставлению счетов вашим клиентам, добавляя гибкие возможности тарификации по фактическому использованию. Вместо жестких лимитов, которые блокируют пользователей при превышении, включите оплату перерасхода, чтобы автоматически выставлять клиентам счета за дополнительное использование.

## Ключевые возможности {#key-features}

* **Тарификация по счетчику**: взимайте плату с клиентов на основе фактического использования, а не фиксированных уровней
* **Управление перерасходом**: автоматически выставляйте счета за использование сверх лимитов тарифного плана
* **Отслеживание использования**: отслеживайте потребление ресурсов клиентами в реальном времени
* **Гибкое ценообразование**: задавайте собственные ставки для разных ресурсов (хранилище, пропускная способность, сайты и т. д.)
* **Оплата AI-токенов**: отслеживайте и выставляйте счета за входные и выходные токены для использования AI на клиентских подсайтах
* **Автоматическое выставление счетов**: автоматически создавайте счета при превышении порогов использования
* **Отчеты об использовании**: подробная аналитика и отчеты об использовании для клиентов и администраторов

## Идеально подходит для {#perfect-for}

* **Хостинг-провайдеров**: предлагайте масштабируемые тарифы, которые растут вместе с потребностями клиентов
* **SaaS-платформ**: выставляйте счета на основе фактического потребления ресурсов
* **Агентских сетей**: гибкие модели ценообразования для клиентских проектов
* **Реселлерского хостинга**: дайте партнерам возможность предлагать ценообразование на основе использования

## Как это работает {#how-it-works}

1. Настройте ресурсы с оплатой по счетчику (дисковое пространство, пропускная способность, количество сайтов и т. д.)
2. Задайте базовые лимиты тарифного плана и ставки за перерасход
3. Система автоматически отслеживает использование по всей вашей сети
4. Когда клиенты превышают лимиты, рассчитывается плата за перерасход
5. Счета создаются автоматически на основе платежного цикла
6. Клиенты могут просматривать свое использование и предстоящие списания в реальном времени

## Оплата AI-токенов {#ai-token-billing}

Версия 1.1.0 добавляет оплату AI-токенов для сетей, которые предлагают AI-возможности через WordPress AI Client SDK. Тарифы с оплатой по счетчику отслеживают завершенные запросы AI-генерации, записывают количество входных и выходных токенов для текущего подсайта и преобразуют эти токены в оплачиваемые расходы на AI за текущий расчетный период клиента.

Используйте оплату AI-токенов, когда хотите предлагать AI-инструменты на клиентских сайтах, не беря на себя непредсказуемые затраты на модели. Каждый запрос связывается с подсайтом, клиентом, членством, провайдером и моделью, чтобы администраторы могли проверять использование, а клиенты — видеть, как их потребление AI влияет на их account.

### Настройка ставок за токены {#configure-per-token-rates}

AI-ставки настраиваются отдельно для каждой модели как отдельные ставки за входные и выходные токены. Ставки рассчитываются за 1 миллион токенов, что соответствует типичным страницам цен AI-провайдеров.

1. В админке сети откройте **Multisite Ultimate → Настройки → Тарифы с оплатой по счетчику**.
2. Включите учет расходов на AI для продукта или членства, по которому должно выставляться счет за использование AI.
3. Добавьте записи ставок моделей, используя ID моделей, возвращаемые вашим коннектором, например `gpt-4o`, `gpt-4o-mini` или другой OpenAI-совместимый идентификатор модели.
4. Введите отдельные значения **Ставка за входные токены** и **Ставка за выходные токены** в валюте вашего сайта за 1 миллион токенов.
5. Задайте включенный лимит расходов на AI для клиента и поведение при перерасходе для тарифного плана.

Если точный ID модели не настроен, Тарифы с оплатой по счетчику могут сопоставлять по префиксу модели. Например, ставка, настроенная для `claude-sonnet-4`, может применяться к ID моделей с временной меткой, которые начинаются с этого префикса. Разработчики также могут изменить итоговую таблицу ставок с помощью фильтра [`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates).

### Поддерживаемые AI-коннекторы {#supported-ai-connectors}

Оплата AI-токенов работает с коннекторами, которые отправляют события генерации WordPress AI Client SDK и включают использование токенов в метаданные ответа. Сюда входит семейство AI-коннекторов Ultimate Multisite, документированное здесь:

* [Ultimate AI Connector для совместимых конечных точек](../ultimate-ai-connector-compatible-endpoints/) для OpenAI-совместимых API, включая OpenAI, Mistral, Ollama-совместимые шлюзы и других провайдеров, предоставляющих совместимые метаданные использования токенов
* [Ultimate AI Connector для WebLLM](../ultimate-ai-connector-webllm/) для использования WebLLM, встроенного в браузер, когда запросы проходят через поток событий AI Client SDK
* AI-коннекторы провайдеров, используемые [Gratis AI Agent](../gratis-ai-agent/), когда выбранный провайдер сообщает количество токенов запроса и завершения через SDK

Пользовательские коннекторы могут участвовать, вызывая стандартные события `wp_ai_client_before_generate_result` и `wp_ai_client_after_generate_result` с метаданными модели, провайдера и использования токенов. Настройки коннектора принудительно применяются с основного сайта к подсайтам, чтобы клиентские сайты использовали одобренных сетью провайдеров и лимиты.

### Использование в account клиента {#customer-account-usage}

Клиенты видят использование AI рядом с другими ресурсами с оплатой по счетчику в своей области account. Строка расходов на AI показывает текущую сумму за расчетный период, рассчитанную на основе записанных токенов запроса и завершения, настроенных ставок модели и любого включенного лимита в тарифном плане.

Если у клиента есть сохраненный способ оплаты, использование сверх включенного лимита считается оплачиваемым перерасходом. Если тарифный план настроен на жесткое отключение и у клиента нет доступного способа оплаты, AI-запросы могут быть заблокированы после достижения лимита, а клиенту будет предложено добавить платежные данные перед продолжением.

## Поддерживаемые ресурсы {#supported-resources}

* Дисковое хранилище
* Пропускная способность/передача данных
* Количество сайтов
* Количество пользователей
* Использование AI-токенов и расходы на AI
* Пользовательские типы ресурсов

## Требования {#requirements}

* Multisite Ultimate 2.0.0 или выше
* Установка WordPress Multisite
* PHP 7.4 или выше

## Установка {#installation}

1. Загрузите файлы плагина в `/wp-content/plugins/ultimate-multisite-metered-plans`
2. Активируйте плагин через экран «Плагины» в WordPress
3. Перейдите в Multisite Ultimate → Настройки → Тарифы с оплатой по счетчику
4. Настройте ресурсы с оплатой по счетчику и цены
5. Включите тарификацию по счетчику для продуктов в редакторе продукта

## Часто задаваемые вопросы {#frequently-asked-questions}

## Что происходит, когда клиент превышает свой лимит? {#what-happens-when-a-customer-exceeds-their-limit}

Вместо блокировки доступа система отслеживает перерасход и добавляет его в следующий счет клиента на основе настроенных вами тарифов.

## Могу ли я установить разные тарифы за перерасход для разных продуктов? {#can-i-set-different-overage-rates-for-different-products}

Да, у каждого продукта могут быть собственные измеряемые ресурсы и цены за перерасход.

## Как клиенты уведомляются об использовании? {#how-are-customers-notified-about-usage}

Клиенты могут просматривать текущее использование в своем Dashboard и получать необязательные уведомления при приближении к лимитам или их превышении.

## Как рассчитывается стоимость использования AI-токенов? {#how-is-ai-token-usage-priced}

Стоимость использования AI-токенов рассчитывается на основе настроенных для модели тарифов за ввод и вывод на 1 миллион токенов. Metered Plans умножает количество токенов prompt и completion в каждом запросе на эти тарифы, добавляет результат к текущим расходам клиента на AI за расчетный период и применяет правила allowance или перерасхода для плана.

## Могу ли я по-прежнему использовать жесткие лимиты? {#can-i-still-use-hard-limits}

Да, биллинг по измеряемому использованию необязателен для каждого продукта. Вы можете сочетать традиционные продукты с жесткими лимитами с продуктами с измеряемым использованием.
