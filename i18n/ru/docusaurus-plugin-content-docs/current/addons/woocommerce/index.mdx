# Интеграция WooCommerce

Интеграция WooCommerce работает очень похоже на то, как работает наш Manual Gateway. Она предоставляет вашим пользователям большую гибкость, позволяя им выбирать любую из форм оплаты, доступных в вашей установке WooCommerce. И всё просто: как только период оплаты пользователя заканчивается, подписка пользователя приостанавливается. Одновременно в установке WooCommerce главного сайта сети будет создан заказ. Затем пользователь получает счёт по электронной почте со ссылкой для выполнения оплаты. После получения оплаты подписка продлевается.

## Возможности

- Простая интеграция между Ultimate Multisite и WooCommerce
- Доступно несколько способов оплаты
- Автоматическое регулярное выставление счетов с использованием встроенного ядра подписок, с необязательными улучшениями плагина WooCommerce Subscriptions
- Итоговые суммы checkout включают налоги Ultimate Multisite как отдельные, не облагаемые налогом строки сборов WooCommerce, чтобы клиенты могли просмотреть суммы налогов перед оплатой

## Документация

**Настройка интеграции WooCommerce**

**Расширение ваших вариантов оплаты с Ultimate Multisite: интеграция WooCommerce**

## Установка

1. Загрузите 'ultimate-multisite-woocommerce' в директорию '/wp-content/plugins/'
2. Активируйте плагин через меню 'Plugins' в WordPress
3. Следуйте пошаговому мастеру, чтобы настроить плагин

## Руководство по использованию

_**ВАЖНОЕ ПРИМЕЧАНИЕ: эта статья относится к Ultimate Multisite версии 2.x.**_

_**ВНИМАНИЕ:** Ultimate Multisite: интеграция WooCommerce требует, чтобы WooCommerce был активирован как минимум на вашем главном сайте._

Мы понимаем, что _Stripe_ и _PayPal_ недоступны в некоторых странах, что ограничивает или мешает пользователям Ultimate Multisite эффективно использовать наш плагин. Поэтому мы создали **дополнение для интеграции _WooCommerce_**.

_WooCommerce_ — очень популярный плагин электронной коммерции. Разработчики по всему миру создали дополнения для интеграции с ним различных платёжных шлюзов. Мы воспользовались этим, чтобы расширить платёжные шлюзы, которые вы можете использовать в системе выставления счетов Ultimate Multisite.

## Настройка

Сначала вам нужно установить дополнение интеграции WooCommerce, которое **бесплатно для лицензированных пользователей**. Чтобы установить его, нажмите Ultimate Multisite на Dashboard суперадминистратора и выберите **Дополнения**.

![Страница дополнений](/img/admin/addons-page.png)

Здесь вы можете найти все дополнения Ultimate Multisite. Нажмите на дополнение **Ultimate Multisite: интеграция WooCommerce**.

![Страница дополнений с интеграцией WooCommerce](/img/admin/addons-page.png)

Появится окно с подробностями дополнения. Просто нажмите **Установить сейчас** *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

После завершения установки вы будете перенаправлены на страницу плагинов. Здесь просто нажмите **Активировать для сети**, и дополнение _WooCommerce_ будет активировано в вашей сети.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

После его активации, если плагин WooCommerce всё ещё не установлен и не активирован на вашем веб-сайте, **вы получите напоминание**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

После установки вы увидите дополнительную опцию в настройках платежей Ultimate Multisite. Затем вы можете **включить** опцию _WooCommerce_ оттуда.

![Настройки платёжного шлюза Ultimate Multisite с включённым WooCommerce](/img/addons/woocommerce-payment-gateway.png)

Вы можете настроить некоторые базовые параметры, например отображаемое имя, которое будет показано во фронтенде, и инструкции, чтобы направлять пользователей в процессе checkout. Затем вы можете перейти в **настройки _WooCommerce_ вашего главного сайта**, чтобы включить и настроить платёжный шлюз, который предпочитаете использовать.

### Строки налоговых сборов checkout

Когда checkout Ultimate Multisite включает налоги, интеграция добавляет каждую сумму налога Ultimate Multisite в checkout WooCommerce как отдельную, не облагаемую налогом строку сбора. Это сохраняет видимость общей суммы налога перед оплатой и предотвращает повторное налогообложение налогового сбора со стороны WooCommerce.

## Как управлять оплатой

Интеграция WooCommerce работает очень похоже на то, как работает наш **Manual Gateway**. Недостаток в том, что каждый платёж должен быть оплачен вручную в конце расчётного периода. Вот как работает процесс:

  * Как только расчётный цикл пользователя заканчивается, membership переводится **в режим ожидания**. Одновременно в установке _WooCommerce_ на главном сайте вашей сети будет создан заказ.

  * Затем пользователи получают счёт по электронной почте со ссылкой для выполнения оплаты (ссылка также отображается в их истории выставления счетов из Dashboard их подсайта). Эта ссылка перенаправляет пользователя на _**WooCommerce**_ **checkout** , где пользователи смогут выбрать один из **доступных шлюзов _WooCommerce_**, чтобы обработать свой платёж.

  * После получения оплаты membership **продлевается** , платёж регистрируется в Ultimate Multisite, а заказ помечается как завершённый.

### Некоторые важные примечания:

  * Стандартная функциональность Ultimate Multisite всё ещё работает с этой формой интеграции. Например, **оформление возвратов** с экрана **Управление подпиской** будет взаимодействовать с WooCommerce, чтобы выполнить возврат с использованием выбранной формы оплаты.

  * Ultimate Multisite: интеграция WooCommerce полагается на WP-Cron для работы. В большинстве случаев это работает очень хорошо, но если у сайта низкий трафик, cron не запускается для проверки запланированных событий, и событие пропускается, что может вызвать задержки выполнения.

## Плагин WooCommerce Subscriptions

Ручной запуск платежа каждый месяц не идеален для конечных пользователей, поскольку ожидается, что подписки будут автоматически повторяться в каждом платежном цикле. Дополнение WooCommerce Integration теперь включает основной код подписок, поэтому повторяющиеся продукты всегда используют подписки для повторяющихся платежей.

Вам **не** нужно покупать или устанавливать отдельный [плагин WooCommerce Subscriptions](https://woocommerce.com/products/woocommerce-subscriptions/), чтобы продавать повторяющиеся продукты через эту интеграцию. Если этот плагин куплен, установлен и доступен на вашем основном сайте, Ultimate Multisite будет использовать его, а WooCommerce предоставит дополнительные функции управления подписками. Если он недоступен, встроенная основа подписок всё равно обрабатывает повторяющиеся продукты.

![Настройки интеграции подписок WooCommerce](/img/addons/woocommerce-subscription-integration.png)

Вот и всё. Вашим конечным пользователям больше не потребуется вручную совершать платеж в конце своего платежного цикла за повторяющиеся продукты.

Помимо создания заказа на вашем основном сайте в разделе _WooCommerce_, для каждого Account также будет создана подписка.

### Поведение при приостановке после неудачного продления

По умолчанию членства остаются активными, пока WooCommerce Subscriptions обрабатывает свой график повторных попыток после неудачного платежа. Это сохраняет обычное льготное окно WooCommerce Subscriptions, чтобы восстановленное продление могло снова синхронизировать подписку и членство без преждевременного прерывания доступа.

Администраторы, которым нужен более строгий контроль доступа, могут включить **Suspend Memberships Immediately on Failed Renewals** в настройках WooCommerce Integration. Когда эта настройка включена, Ultimate Multisite приостанавливает связанное членство сразу после сбоя платежа за продление, вместо того чтобы ждать завершения обработки повторных попыток WooCommerce Subscriptions.

Разработчики могут переопределить это же решение с помощью фильтра `wu_woo_suspend_on_payment_failure`. Страницы справки по хукам генерируются из исходного кода дополнения, поэтому в этом обзоре указано только имя фильтра для пользовательских политик приостановки.
