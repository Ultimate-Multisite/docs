# Электронная почта

Аддон Ultimate Multisite: Emails позволяет владельцам сети создавать и управлять учетными записями электронной почты клиентов вместе с сайтами и членством. В версии 1.1.0 добавлены функции управления учетными записями электронной почты для клиентов, единый вход (SSO) для веб-почты, поддержка пересыльщиков CyberPanel и улучшения процесса создания учетных записей во время оформления заказа.

## Основные функции

* **Виджет учетной записи электронной почты клиента**: Клиенты могут создавать и управлять учетными записями электронной почты на странице своей учетной записи, не создавая при этом тикет в службу поддержки.
* **SSO для веб-почты и магия-ссылки**: Клиенты могут открыть веб-почту прямо со страницы учетной записи с аутентификацией в один клик, если это поддерживает провайдер.
* **Смена паролей**: Клиенты и администраторы могут обновлять пароли почтовых ящиков через виджет управления электронной почтой.
* **Создание учетных записей при оформлении заказа**: Создание учетной записи электронной почты клиента может происходить во время оформления заказа, чтобы почтовый ящик был готов после завершения процесса покупки.
* **Пересыльщики CyberPanel**: Сети, работающие на CyberPanel, могут создавать и управлять пересыльщиками электронной почты в рамках процесса предоставления услуг электронной почты.
* **Настройки с учетом провайдера**: Страница настроек фильтрует провайдеров, показывая только активные и настроенные интеграции, чтобы администраторы видели только рабочие варианты.

## Требования

* Ultimate Multisite 2.0.0 или выше
* Установка WordPress Multisite
* Минимум одна настроенная интеграция провайдера электронной почты
* Учетные данные API провайдера с разрешениями на создание почтовых ящиков, смену паролей и выдачу ссылок доступа к веб-почте (где это поддерживается)

## Установка

1. Загрузите файлы плагина в `/wp-content/plugins/ultimate-multisite-emails`.
2. Активируйте аддон на уровне сети в **Плагины** в административной панели WordPress.
3. Перейдите в **Ultimate Multisite** > **Настройки** и откройте настройки аддона электронной почты.
4. Включите интеграции провайдеров электронной почты, которые вы хотите, чтобы клиенты использовали.
5. Сохраните настройки, а затем протестируйте создание учетной записи с помощью нерабочего клиента или на тестовом сайте.

## Управление электронной почтой клиента

В версии 1.1.0 добавлен виджет управления электронной почтой на страницу учетной записи клиента. Используйте его, когда клиенты должны самостоятельно создавать и поддерживать почтовые ящики для своих сайтов.

На странице учетной записи клиенты могут:

1. Просматривать учетные записи электронной почты, связанные с их текущим членством или сайтом.
2. Создавать новый почтовый ящик, если их тарифный план позволяет дополнительные учетные записи.
3. Изменять пароль существующего почтового ящика.
4. Открывать поддерживаемые веб-почтовые провайдеры через SSO или вход по магия-ссылке.

Виджет использует систему учетной записи Ultimate Multisite, поэтому действия ограничиваются контекстом учетной записи, которую просматривает клиент. Это не позволяет клиентам управлять учетными записями электронной почты, принадлежащими другому членству или сайту.

## SSO для веб-почты и вход по магия-ссылке

SSO для веб-почты предоставляет клиентам прямую возможность **Открыть веб-почту** со страницы учетной записи. Если это поддерживается активным провайдером, аддон запрашивает временную ссылку доступа и перенаправляет клиента в веб-почту, не заставляя его копировать учетные данные почтового ящика.

Поведение провайдеров может различаться:

* Провайдеры с нативными API SSO могут создать прямую аутентифицированную сессию веб-почты.
* Провайдеры с поддержкой магия-ссылок могут генерировать ограниченную по времени ссылку для входа.
* Провайдеры без поддержки SSO или магия-ссылок должны по-прежнему отображать детали почтового ящика и действия по управлению паролем, но без кнопки веб-почты в один клик.

Включайте доступ к веб-почте только для тех провайдеров, для которых вы настроили необходимые учетные данные API. Если действие отсутствует для провайдера, сначала убедитесь, что провайдер активен, настроен и поддерживает ссылки входа в веб-почту.

## Создание учетных записей при оформлении заказа

Аддон может создавать учетные записи электронной почты клиентов в рамках процесса оформления заказа. Это полезно, когда план по умолчанию включает почтовый ящик или когда форма оформления заказа собирает данные об учетной записи электронной почты.

Рекомендуемый порядок внедрения:

1. Настройте лимиты провайдера и плана, прежде чем включать создание учетных записей при оформлении заказа.
2. Протестируйте покупку с помощью тестового продукта и убедитесь, что почтовый ящик создан после оформления заказа.
3. Проверьте страницу учетной записи клиента, чтобы убедиться, что новый почтовый ящик появился в виджете электронной почты.
4. Подтвердите действия по смене пароля и входа в веб-почту, прежде чем включать этот процесс для продакшн-продуктов.

Если создание учетной записи не удалось, учетная запись и членство клиента могут существовать, но для почтового ящика потребуется вмешательство администратора. Прежде чем повторять попытку, проверьте журналы активности Ultimate Multisite и журналы провайдера.

## Пересыльщики электронной почты CyberPanel

Для сетей, работающих на CyberPanel, в версии 1.1.0 добавлена поддержка пересыльщиков электронной почты. Используйте пересыльщики, когда клиентам нужны адреса типа `info@customer-domain.test` для перенаправления почты на другой ящик без создания полноценного почтового ящика.

Прежде чем предлагать пересыльщики, убедитесь, что интеграция CyberPanel настроена и доступна. См. руководство [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) для получения необходимых констант хоста, имени пользователя, пароля и пакета.

## Часто задаваемые вопросы

## Могут ли клиенты создавать собственные учетные записи электронной почты?

Да. Когда включен виджет учетной записи клиента и план позволяет учетные записи электронной почты, клиенты могут создавать и управлять своими почтовыми ящиками прямо со страницы учетной записи.

## Поддерживает ли каждый провайдер SSO для веб-почты?

Нет. Поддержка SSO и магия-ссылок зависит от активного провайдера электронной почты. Провайдеры, у которых нет API для ссылок доступа, все равно могут поддерживать создание почтовых ящиков и смену паролей.

## Могут ли администраторы создавать учетные записи электронной почты для клиентов?

Да. Администраторы могут создавать учетные записи электронной почты из административного интерфейса, включая установку начального пароля при создании учетной записи.

## Являются ли пересыльщики тем же, что и почтовые ящики?

Нет. Почтовый ящик хранит почту и имеет учетные данные для входа. Пересыльщик перенаправляет почту на другой адрес и не предоставляет отдельный почтовый ящик.
