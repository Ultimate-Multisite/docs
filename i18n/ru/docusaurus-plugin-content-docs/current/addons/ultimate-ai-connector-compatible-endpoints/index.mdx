# Ultimate AI Connector for Compatible Endpoints {#ultimate-ai-connector-for-compatible-endpoints}

Ultimate AI Connector for Compatible Endpoints интегрирует API провайдеров ИИ, совместимых с OpenAI, с вашей мультисайтовой сетью WordPress. Настройте несколько ИИ-провайдеров — каждый со своим URL конечной точки (endpoint), ключом API и моделью по умолчанию — и маршрутизируйте запросы с автоматическим резервированием (fallback), если один из провайдеров недоступен.

## Ключевые возможности {#key-features}

- **Поддержка нескольких провайдеров**: Добавьте несколько карточек провайдеров ИИ, каждая со своим URL конечной точки, ключом API и моделью по умолчанию.
- **Изменение порядка перетаскиванием**: Расположите провайдеров в порядке приоритета, используя ручку захвата на каждой карточке; запросы будут использовать провайдеров в указанном порядке.
- **Автоматическое резервирование маршрутизации**: Если основной провайдер недоступен, запросы автоматически перенаправляются к следующему настроенному провайдеру.
- **Конфигурация для каждого провайдера**: Каждый провайдер настраивается независимо, используя свои учетные данные и выбор модели.
- **Интерфейс настроек на основе карточек**: Построен на стабильных компонентах Card WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **Автоматическое расширение при добавлении**: Новые карточки провайдеров автоматически расширяются, чтобы вы могли сразу заполнить детали.
- **Обход кэша скриптов**: Скрипты провайдеров очищаются от кэша при обновлении плагина, чтобы гарантировать загрузку самой последней версии.

## Требования {#requirements}

- WordPress 5.3 или выше
- PHP 7.4 или выше
- Плагин Ultimate Multisite (активен)
- Конечная точка API, совместимая с OpenAI (самостоятельно размещенная или сторонний сервис)

## Установка {#installation}

1. Загрузите файлы аддона в каталог `/wp-content/plugins/`
2. Активируйте плагин через меню «Плагины» в WordPress
3. Перейдите в **Ultimate Multisite → AI Connector**, чтобы настроить аддон

## Настройка {#configuration}

### Настройка нескольких провайдеров {#multi-provider-setup}

Экран настройки использует макет на основе карточек, где каждая карточка представляет одного ИИ-провайдера. Вы можете добавить столько провайдеров, сколько вам потребуется.

#### Добавление провайдера {#adding-a-provider}

1. Перейдите в **Ultimate Multisite → AI Connector** в административной панели сети.
2. Нажмите **Add Provider** (Добавить провайдера).
3. Появится и автоматически расширится новая карточка провайдера.
4. Заполните детали провайдера:
   - **Provider Name** (Название провайдера) — метка для идентификации этого провайдера (например, "OpenAI", "Mistral", "Local Ollama").
   - **Endpoint URL** (URL конечной точки) — базовый URL API, совместимого с OpenAI (например, `https://api.openai.com/v1`).
   - **API Key** (Ключ API) — ваш ключ API для этого провайдера.
   - **Default Model** (Модель по умолчанию) — идентификатор модели, который будет использоваться по умолчанию для этого провайдера (например, `gpt-4o`, `mistral-small`).
5. Нажмите **Save Settings** (Сохранить настройки).

#### Удаление провайдера {#removing-a-provider}

Нажмите кнопку **Remove** (Удалить) на любой карточке провайдера, чтобы удалить ее. Сохраните настройки, чтобы применить изменения.

#### Изменение порядка провайдеров {#reordering-providers}

Перетащите значок захвата (⣿) в левой части любой карточки провайдера, чтобы изменить ее порядок. Порядок определяет приоритет резервирования: запросы отправляются первому провайдеру и перенаправляются к последующим провайдерам, если запрос не удался.

### Резервирование маршрутизации (Fallback Routing) {#fallback-routing}

Если запрос к основному провайдеру не удается — из-за таймаута, превышения лимита запросов или сбоя сервиса — коннектор автоматически повторяет запрос с использованием следующего провайдера в списке. Это продолжается по списку до тех пор, пока один из провайдеров не ответит успешно или пока не будут проверены все провайдеры.

| Сценарий | Поведение |
|----------|-----------|
| Основной провайдер отвечает | Запрос обрабатывается в обычном режиме |
| Основной провайдер не отвечает | Запрос повторяется с использованием следующего провайдера |
| Все провайдеры не отвечают | Обработка ошибки возвращается вызывающей стороне |

Резервирование происходит прозрачно для вызывающего плагина или темы — они получают либо успешный ответ, либо окончательную ошибку, независимо от того, сколько провайдеров было проверено.

### Справочник конфигурации провайдера {#provider-configuration-reference}

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| Provider Name | Да | Отображаемая метка для этого провайдера |
| Endpoint URL | Да | Базовый URL API, совместимого с OpenAI |
| API Key | Да | Ключ аутентификации для этого провайдера |
| Default Model | Да | Идентификатор модели, используемый, если в запросе не указана модель |

## Устранение неполадок {#troubleshooting}

### Провайдеры не отображаются {#no-providers-are-listed}

Если список провайдеров пуст после сохранения, убедитесь, что:

- Вы нажали **Save Settings** (Сохранить настройки) после добавления провайдеров.
- Ваш браузер не заблокировал сохранение страницы настроек (проверьте консоль браузера на наличие ошибок JavaScript).
- Плагин активен и вы находитесь на экране административной панели сети.

### Запросы не проходят для всех провайдеров {#requests-are-failing-for-all-providers}

- Убедитесь, что URL конечной точки каждого провайдера доступен с вашего сервера (а не только из вашего браузера).
- Проверьте, что каждый ключ API действителен и не истек.
- Просмотрите статус лимитов и квот провайдера в его панели управления (dashboard).
- Включите отладку WordPress (`WP_DEBUG_LOG`), чтобы захватить подробные сообщения об ошибках от коннектора.

### Провайдер не используется для резервирования {#a-provider-is-not-being-used-for-fallback}

Изменение порядка перетаскиванием меняет порядок резервирования. Убедитесь, что провайдер, который, по вашему мнению, должен получать запросы резервирования, расположен правильно в списке, и что его URL конечной точки и ключ API верны.

### Новая карточка провайдера не расширяется автоматически {#new-provider-card-does-not-expand-automatically}

Автоматическое расширение требует включения JavaScript в вашем браузере. Если карточка не расширяется при нажатии, проверьте консоль браузера на наличие ошибок. Содержимое карточки всегда доступно нажатием на заголовок карточки, даже если автоматическое расширение не срабатывает.
