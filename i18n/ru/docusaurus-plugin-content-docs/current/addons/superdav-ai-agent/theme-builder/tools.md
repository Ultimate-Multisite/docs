# Инструменты и конфигурация {#tools-and-configuration}

Theme Builder использует набор инструментов для анализа, проектирования и создания вашего WordPress сайта. В версии 1.16.0 **sd-ai-agent/site-scrape теперь является инструментом Tier 1**, что делает его доступным по умолчанию в Theme Builder. Superdav AI Agent v1.18.0 также вводит отдельно распространяемый расширенный сопутствующий пакет для доверенных рабочих процессов разработчиков; эти инструменты не являются частью основного пакета WordPress.org и должны быть установлены и авторизованы отдельно.

## Доступные инструменты {#available-tools}

### Инструменты Tier 1 (всегда доступны) {#tier-1-tools-always-available}

Инструменты Tier 1 доступны по умолчанию в Theme Builder без дополнительной конфигурации.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**Назначение:** Анализировать существующие веб-сайты для извлечения дизайнерского вдохновения, структуры содержимого и функциональности.

**Возможности:**

- **Анализ веб-сайтов** — сканировать и анализировать сайты конкурентов или источники вдохновения
- **Извлечение дизайна** — определять цвета, шрифты и шаблоны макета
- **Структура содержимого** — понимать организацию и иерархию страниц
- **Обнаружение функций** — определять plugins и функциональность
- **Анализ производительности** — проверять скорость страниц и оптимизацию
- **SEO-анализ** — просматривать meta-теги и структурированные данные

**Использование:**

```
Проанализируй дизайн example.com и используй его как вдохновение для моего сайта.
```

**Что он извлекает:**

- Цветовая палитра и использование цветов
- Типографика (шрифты и размеры)
- Структура макета и отступы
- Шаблоны навигации
- Организация содержимого
- Использование изображений и медиа
- Интерактивные элементы
- Адаптивность для мобильных устройств

**Ограничения:**

- Не может сканировать сайты, защищенные паролем
- Соблюдает robots.txt и политики сайта
- Может не захватывать динамическое содержимое
- Сайты с большим количеством JavaScript могут иметь ограниченный анализ
- Анализ больших сайтов может занять больше времени

### Инструменты Tier 2 (необязательные) {#tier-2-tools-optional}

Инструменты Tier 2 доступны при явном включении в настройках Theme Builder.

#### Расширенная аналитика {#advanced-analytics}

Предоставляет подробные метрики производительности:

- Время загрузки страниц
- Core Web Vitals
- SEO-оценка
- Оценка доступности
- Оценка лучших практик

#### Оптимизатор содержимого {#content-optimizer}

Анализирует и предлагает улучшения для:

- Удобочитаемости
- SEO-оптимизации
- Использования ключевых слов
- Структуры содержимого
- Размещения призыва к действию

### Инструменты Tier 3 (премиум) {#tier-3-tools-premium}

Инструменты Tier 3 требуют дополнительной конфигурации или премиум-функций.

#### AI-генератор содержимого {#ai-content-generator}

Генерирует содержимое для:

- Описаний продуктов
- Страниц услуг
- Записей блога
- Meta-описаний
- Текстов призыва к действию

#### Расширенные инструменты дизайна {#advanced-design-tools}

Предоставляет расширенные возможности дизайна:

- Генерация пользовательского CSS
- Создание анимаций
- Дизайн интерактивных элементов
- Расширенная теория цвета
- Оптимизация типографики

### Расширенные сопутствующие инструменты (отдельный пакет) {#advanced-companion-tools-separate-package}

Расширенный сопутствующий пакет распространяется отдельно от основного выпуска Superdav AI Agent. Он предназначен для доверенных сред разработчиков и владельцев сайтов, где администраторы явно разрешают агенту использовать инструменты с высоким воздействием.

Возможности расширенного сопутствующего пакета могут включать:

- **Инструменты файловой системы** — проверять и изменять одобренные файлы в доверенных средах
- **Инструменты базы данных** — запрашивать или обновлять данные сайта при явной авторизации
- **Инструменты WP-CLI** — выполнять команды обслуживания и проверки WordPress
- **Инструменты REST-диспетчера** — вызывать зарегистрированные REST endpoints через контролируемые рабочие процессы диспетчеризации
- **Инструменты создания plugins** — создавать каркас и итеративно дорабатывать код plugin
- **Инструменты Git-снимков** — создавать снимки перед рискованными операциями разработчика
- **Инструменты управления пользователями** — помогать проверять или корректировать записи пользователей, где это разрешено
- **Инструменты бенчмаркинга** — выполнять тесты производительности или возможностей для проверки разработки

Не документируйте их как общедоступные инструменты Theme Builder для каждой установки. Они доступны только когда расширенный сопутствующий пакет установлен, включен и ограничен доверенными администраторами.

## Конфигурация инструментов {#tool-configuration}

### Включение инструментов {#enabling-tools}

Чтобы включить дополнительные инструменты в Theme Builder:

1. Перейдите в **Gratis AI Agent → Settings**
2. Перейдите в **Theme Builder → Tools**
3. Включите или отключите инструменты по необходимости
4. Сохраните настройки

### Разрешения инструментов {#tool-permissions}

Некоторым инструментам требуются разрешения:

- **Сканирование сайтов** — требует доступа к интернету
- **Аналитика** — требует интеграции с Google Analytics
- **Генерация содержимого** — требует API-ключей
- **Расширенные функции** — могут требовать премиум-подписку
- **Расширенные сопутствующие инструменты** — требуют отдельно распространяемый расширенный пакет и явное доверие администратора

### API-ключи и учетные данные {#api-keys-and-credentials}

Настройте API-ключи для инструментов, которым они требуются:

1. Перейдите в **Gratis AI Agent → Settings → API Keys**
2. Введите учетные данные для каждого инструмента
3. Проверьте подключение
4. Надежно сохраните

### Установка расширенного сопутствующего пакета {#installing-the-advanced-companion-package}

Расширенный сопутствующий пакет выпускается отдельно от основного ZIP WordPress.org. Устанавливайте его только из официального канала распространения проекта для данного выпуска, затем ограничьте доступ доверенными администраторами. Проверьте шлюзы одобрения человеком и поведение Git-снимков перед включением инструментов файловой системы, базы данных, WP-CLI, REST-диспетчера, создания plugins, управления пользователями или бенчмаркинга на производственном сайте.

## Ограничения и соображения по инструментам {#tool-limitations-and-considerations}

### Ограничение частоты запросов {#rate-limiting}

- Скрапинг ограничивается по частоте, чтобы предотвратить перегрузку сервера
- Несколько запросов к одному и тому же сайту могут ограничиваться
- Анализ крупных сайтов может занять больше времени

### Точность контента {#content-accuracy}

- Динамический контент может быть захвачен не полностью
- Контент, отрисованный JavaScript, может быть неполным
- Данные в реальном времени могут не отражаться
- Часть контента может находиться за платным доступом

### Юридические и этические соображения {#legal-and-ethical-considerations}

- Уважайте robots.txt и политики сайта
- Не скрапьте защищённый авторским правом контент для воспроизведения
- Используйте анализ для вдохновения, а не для копирования
- Убедитесь, что у вас есть права на использование извлечённого контента
- Соблюдайте условия обслуживания анализируемых сайтов

### Влияние на производительность {#performance-impact}

- Скрапинг крупных сайтов может занять время
- Несколько одновременных скрапингов могут выполняться медленнее
- Сетевое подключение влияет на скорость
- Большие наборы данных могут потребовать дополнительной обработки

## Рекомендации {#best-practices}

### Использование анализа сайта {#using-site-analysis}

1. **Анализируйте несколько сайтов** — собирайте вдохновение из нескольких источников
2. **Сосредоточьтесь на структуре** — изучайте шаблоны макета и организации
3. **Извлекайте цвета** — используйте цветовые палитры как отправные точки
4. **Изучайте типографику** — определяйте сочетания шрифтов, которые вам нравятся
5. **Проверяйте навигацию** — понимайте организацию меню

### Этичный скрапинг {#ethical-scraping}

1. **Уважайте robots.txt** — следуйте рекомендациям сайта
2. **Не копируйте контент** — используйте анализ только для вдохновения
3. **Проверяйте права** — убедитесь, что можете использовать извлечённый контент
4. **Указывайте источники** — отмечайте источники вдохновения
5. **Соблюдайте условия** — выполняйте условия обслуживания сайта

### Максимальная эффективность инструмента {#maximizing-tool-effectiveness}

1. **Будьте конкретны** — запрашивайте конкретные типы анализа
2. **Предоставляйте контекст** — объясняйте назначение вашего сайта
3. **Задавайте ожидания** — описывайте свои цели дизайна
4. **Итерируйте** — уточняйте на основе результатов
5. **Комбинируйте инструменты** — используйте несколько инструментов для комплексного анализа

## Устранение неполадок {#troubleshooting}

### Сайт не скрапится {#site-wont-scrape}

- Проверьте, доступен ли сайт публично
- Убедитесь, что robots.txt разрешает скрапинг
- Попробуйте другой сайт
- Проверьте подключение к интернету
- Обратитесь в поддержку, если проблема сохраняется

### Неполный анализ {#incomplete-analysis}

- На сайте может быть динамический контент
- JavaScript может быть отрисован не полностью
- На крупных сайтах может истечь время ожидания
- Попробуйте вместо этого анализировать конкретные страницы
- Запрашивайте конкретные типы анализа

### Низкая производительность {#slow-performance}

- Анализ крупных сайтов занимает больше времени
- Несколько одновременных скрапингов выполняются медленнее
- Сетевое подключение влияет на скорость
- Попробуйте в часы меньшей нагрузки
- Сначала анализируйте меньшие разделы
