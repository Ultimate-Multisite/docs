# Changelog

## 1.9.0 — Выпущено 2026-04-28

### Новое

- **Возможность `create_contact_form`** — создает контактную форму, используя активный плагин форм (Contact Form 7, WPForms, Fluent Forms или Gravity Forms), и возвращает шорткод, готовый для вставки в любую запись или страницу.
- **Возможность `set_featured_image`** — присваивает главное изображение записи или страницы, используя существующий ID в Медиабиблиотеке или удаленный URL; автоматически импортирует изображение, если предоставлен URL.
- **Возможность `batch_create_posts`** — позволяет создавать несколько записей за один вызов возможности. Поддерживает те же параметры, что и `create_post`, сообщает об успехе/неудаче для каждой записи и имеет опциональный режим `stop_on_error` (остановка при ошибке).
- **Параметр `page_template`** — добавлен в `create_post` и `update_post`. Позволяет присвоить файл шаблона страницы PHP (например, `page-full-width.php`) во время создания или обновления. Для отката к шаблону по умолчанию темы в `update_post` передайте пустую строку.
- **Возможности скриншотов на стороне клиента** — `capture_screenshot`, `compare_screenshots` и `review_page_design`. Позволяют делать скриншоты страниц в реальном времени (полные или только область просмотра) через серверный headless-браузер, сравнивать два скриншота и получать анализ дизайна от ИИ, который охватывает макет, типографику, цвет и доступность.
- **Пять встроенных агентов** — Content Writer, Site Builder, Design Studio, Plugin Manager и Support Assistant. Каждый агент имеет выделенный набор инструментов, настроенный системный промпт и стартовые подсказки. Переключение происходит через новый **Agent Picker** в заголовке чата. Подробнее см. [Built-in Agents](../../user-guide/configuration/built-in-agents).
- **Флаги функций (Feature flags)** — новая вкладка **Настройки → Флаги функций** с переключателями контроля доступа (ограничить администраторами, ограничить администраторами сети, доступ подписчиков, отключить для не-членов) и опциями брендинга (скрыть атрибуцию в футере, пользовательское имя агента, скрыть агент-пикер, использовать иконку сайта в качестве аватара чата). Подробнее см. [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings).
- **Восстановление последней сессии** — панель чата теперь автоматически перезагружает самую последнюю беседу при загрузке страницы и при открытии виджета, поэтому контекст никогда не теряется при навигации по страницам.
- **Ссылки на действия плагинов** — быстрые ссылки к Настройкам и Реестру возможностей теперь появляются на экране **Плагины → Установленные плагины** WordPress под описанием плагина.

### Улучшено

- **Повторная попытка источника изображения** — агент теперь повторяет попытку использования всех настроенных бесплатных источников изображений, прежде чем использовать сгенерированное ИИ изображение в случае сбоя загрузки.
- **Панель информации о модели** — всегда видна в заголовке чата; больше не скрывается после первого сообщения.
- **Поведение автопрокрутки** — автопрокрутка приостанавливается, когда пользователь прокручивает страницу вверх для чтения; появляется плавающая кнопка **Scroll to bottom**, которая автоматически исчезает, когда пользователь достигает последнего сообщения.
- **UI Agent Picker** — переработан в оверлей в формате таблица-форма с иконками для каждого агента, что упрощает их идентификацию и переключение с первого взгляда.
- **Ленивая загрузка JS-чанков** — начальный JavaScript-бандл виджета чата теперь разделен на лениво загружаемые чанки, что уменьшает начальный размер бандла на 75–90%.
- **Реконструкция виджета чата** — унифицированная иконка ИИ заменила предыдущий пользовательский аватар; она соответствует системе встроенных агентов.
- **Преобразование URL-ссылок** — URL-адреса, появляющиеся в системных сообщениях и пузырях с ошибками, теперь отображаются как кликабельные ссылки.

### Исправлено

- **Обнаруживаемость возможностей** — исправлено описание, ссылки в системном промпте и выравнивание пространства имен, чтобы все возможности надежно отображались в списке инструментов агента.
- **Кэш провайдеров** — провайдеры теперь кэшируются на уровне всего сайта с помощью счетчика версий, что предотвращает проблемы со устаревшими провайдерами в сетях multisite.
- **`ability_invalid_output`** — устранена в 12 обработчиках возможностей; все теперь возвращают правильно структурированные JSON-ответы.
- **Конвейер `pending_client_tool_calls`** — настроен от начала до конца, чтобы вызовы инструментов на стороне клиента завершались корректно и возвращали результаты модели.
- **Ящик истории** — изменения, которые нельзя отменить, исключены из списка отката; ссылка **View full history** теперь работает корректно.
- **Система изменений/отката** — исправлено пять отдельных ошибок и объединено в новый интерфейс администратора.
- **Уведомление сохранения настроек** — всплывающее уведомление (snackbar) теперь надежно появляется после нажатия **Save Settings**.
- **Контекстное меню корзины** — добавлена опция **Delete Permanently**, чтобы элементы можно было удалить навсегда, не оставаясь в корзине.
- **Редактирование и повторная отправка** — нажатие на иконку редактирования теперь переводит в режим редактирования только выбранное сообщение, а не все сообщения в ветке.
- **Высота макета чата** — панель чата адаптирует свою высоту, когда над страницей появляются плагином внедренные элементы (уведомления администратора, баннеры), не позволяя области ввода уйти за пределы экрана.

## 1.4.0 — Выпущено 2026-04-09

### Новое

- **Команда бенчмаркинга WP-CLI** (`wp gratis-ai-agent benchmark`) — запускает набор тестов Agent Capabilities v1 из командной строки для CI-пайплайнов и рабочих процессов оценки моделей. Поддерживает запуск по вопросам, переопределение провайдеров/моделей и вывод в формате JSON/CSV.
- **Набор тестов Agent Capabilities v1** — структурированный набор сложных многоэтапных промптов, которые проверяют весь спектр возможностей, с подсчетом баллов и отчетов по токенам/времени.
- **Возможности для пользовательских типов записей (Custom Post Type)** — `register_post_type`, `list_post_types`, `delete_post_type`. Регистрации сохраняются между перезапусками через таблицу опций.
- **Возможности для пользовательских таксономий (Custom Taxonomy)** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. Поддерживает иерархические и плоские таксономии с опциональными слагами перезаписи.
- **Возможности системы дизайна (Design System)** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. Включает пять готовых пресетов: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **Возможности глобальных стилей (Global Styles)** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. Читает и записывает значения `theme.json` через API глобальных стилей WordPress.
- **Возможность управления навигационным меню** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. Поддерживает вложенные меню и назначение локаций темы.
- **Возможность управления опциями** — `get_option`, `set_option`, `delete_option`, `list_options`. Включает встроенный список запрещенных опций, защищающий критически важные опции WordPress от изменения.
- **Реестр устанавливаемых возможностей (Installable Abilities Registry)** — `list_available_abilities`, `install_ability`, `recommend_plugin`. Обнаруживает и активирует пакеты возможностей, распределенные в виде плагинов WordPress.
- **Оркестрация Site Builder v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. Многофазные планы создания сайта с обнаружением плагинов, ссылками на результаты шагов, отслеживанием прогресса и автономным восстановлением ошибок.
- **Тестовый вопрос для сайта ресторана** (`q-restaurant-website`) — сквозное тестирование, охватывающее регистрацию CPT, систему дизайна, навигацию и обнаружение плагинов.
- **Плагины-коннекторы для провайдеров ИИ** добавлены в чертежи WordPress Playground для более быстрой локальной настройки разработки.

### Улучшено

- README обновлен с документацией и инструкциями по настройке плагинов-коннекторов для провайдеров ИИ.

### Исправлено

- Устранено 25 сбоев тестов PHPUnit в ветке `main`.
- Исправлен формат URL релиза GitHub в `blueprint.json`.
- Перенумерованы идентификаторы задач, чтобы избежать конфликтов со старыми ID.
