# Gratis AI Agent {#gratis-ai-agent}

Gratis AI Agent — это автономный AI-помощник для WordPress, который может планировать, создавать и управлять полноценными сайтами WordPress, используя обычные инструкции на естественном языке. Он предоставляет структурированный набор **возможностей** (abilities) — дискретных, тестируемых действий, которые агент может вызвать. Эти возможности охватывают всё: от управления контентом до кастомизации дизайн-системы. В версии 1.9.0 добавлены возможности создания контента, визуальный обзор страниц через скриншоты с клиентской стороны, пять встроенных профилей агентов и флаги функций для контроля доступа и брендинга.

## Ключевые возможности {#key-features}

- **Система возможностей (Abilities system)** — модульный, расширяемый каталог действий, которые агент может выполнить в вашей установке WordPress
- **Управление контентом (Content Management)** — создание постов и страниц (включая назначение шаблонов страниц), пакетное создание нескольких постов за один запрос, установка изображений в качестве главного, создание контактных форм
- **Визуальный обзор (Visual Review)** — захват скриншотов живых страниц, сравнение состояний «до/после» и получение AI-отчета о дизайне, который охватывает макет, типографику, цвет и доступность
- **Пять встроенных агентов** — Контент-писатель (Content Writer), Создатель сайтов (Site Builder), Дизайн-студия (Design Studio), Менеджер плагинов (Plugin Manager) и Помощник поддержки (Support Assistant); переключение происходит через «Выбор агента» (Agent Picker) в шапке чата
- **Флаги функций (Feature flags)** — переключатели для контроля доступа и брендинга в Настройки → Флаги функций (Settings → Feature Flags) для ограничения ролей, белой этикетки (white-label naming) и кастомизации виджетов
- **Управление кастомными типами записей (Custom Post Type management)** — регистрация, перечисление и удаление кастомных типов записей с сохранением данных после перезапуска
- **Управление кастомными таксономиями (Custom Taxonomy management)** — регистрация, перечисление и удаление кастомных таксономий с сохранением данных
- **Возможности дизайн-системы (Design System abilities)** — внедрение кастомного CSS, управление подобранными паттернами блоков, установка логотипа сайта и применение пресетов theme.json
- **Управление глобальными стилями (Global Styles management)** — чтение и запись значений theme.json для палитр цветов, типографики, отступов и прочего
- **Управление навигационными меню (Navigation Menu management)** — программное создание и обновление навигационных меню и их элементов
- **Управление опциями (Options Management)** — чтение и запись опций WordPress с встроенным блоком безопасности для защиты критически важных настроек
- **Реестр устанавливаемых возможностей (Installable Abilities Registry)** — обнаружение и активация дополнительных пакетов возможностей без изменения файлов ядра плагина
- **Оркестрация Site Builder v2 (Site Builder Orchestration v2)** — многоэтапные планы создания сайта с обнаружением плагинов, отслеживанием прогресса и восстановлением после ошибок
- **Команда бенчмаркинга WP-CLI (WP-CLI benchmark command)** — запуск набора тестов производительности агента из командной строки для CI и оценки моделей

## Требования {#requirements}

- WordPress 6.0 или выше
- PHP 8.1 или выше
- WordPress Multisite (активировано на уровне сети) или установка для одного сайта
- Активный плагин-коннектор для AI-провайдера (Anthropic, OpenAI или совместимый)

## Установка {#installation}

1. Загрузите файлы плагина в `/wp-content/plugins/gratis-ai-agent`
2. Активируйте плагин на экране **Плагины** в WordPress
3. Перейдите в **Gratis AI Agent → Настройки** и подключите вашего AI-провайдера
4. (Необязательно) Просмотрите **Реестр возможностей (Abilities Registry)** для установки дополнительных пакетов возможностей

## Быстрый старт {#quick-start}

После активации и подключения к AI-провайдеру откройте панель **Gratis AI Agent** в боковой панели администратора WordPress. Введите инструкцию на естественном языке, и агент спланирует и выполнит необходимые шаги, используя доступные ему возможности.

**Примеры инструкций:**

```
Создай кастомный тип записи «Портфолио» с таксономией «Категория проекта».
```

```
Внедрить палитру цветов для тёмного режима в тему и установить основной цвет на #1a1a2e.
```

```
Создать сайт ресторана с онлайн-меню, формой бронирования и страницей контактов.
```

## Обзор возможностей (Abilities Overview) {#abilities-overview}

Возможности — это атомарные действия, которые может выполнить агент. Каждая возможность — это зарегистрированный PHP-класс, который предоставляет схему (параметры и типы возвращаемых значений), которую агент считывает во время выполнения. Возможности сгруппированы по функциональным областям:

| Область | Возможности |
|---|---|
| **Управление контентом** | `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form` |
| **Визуальный обзор** | `capture_screenshot`, `compare_screenshots`, `review_page_design` |
| **Кастомные типы записей** | `register_post_type`, `list_post_types`, `delete_post_type` |
| **Кастомные таксономии** | `register_taxonomy`, `list_taxonomies`, `delete_taxonomy` |
| **Дизайн-система** | `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset` |
| **Глобальные стили** | `get_global_styles`, `set_global_styles`, `reset_global_styles` |
| **Навигационные меню** | `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus` |
| **Управление опциями** | `get_option`, `set_option`, `delete_option`, `list_options` |
| **Устанавливаемые возможности** | `list_available_abilities`, `install_ability`, `recommend_plugin` |
| **Site Builder** | `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error` |

См. [Abilities Reference](./abilities) для полной документации по параметрам и примерам использования.

## WP-CLI {#wp-cli}

Gratis AI Agent включает семейство команд `wp gratis-ai-agent` для запуска тестов производительности и управления возможностями из командной строки. Полный список команд см. в [WP-CLI Reference](./wp-cli).

```bash
# Запустить полный набор тестов производительности
wp gratis-ai-agent benchmark

# Показать установленные возможности
wp gratis-ai-agent abilities list

# Установить пакет возможностей
wp gratis-ai-agent abilities install <slug>
```

## Конфигурация {#configuration}

Все настройки находятся в **Gratis AI Agent → Настройки** в админ-панели WordPress:

| Настройка | Описание |
|---|---|
| **AI Provider** | Выбор и настройка активного коннектора AI-провайдера |
| **Model** | Выбор языковой модели для сессий агента |
| **Max Tokens** | Бюджет токенов на один запрос (по умолчанию 4096) |
| **Ability Timeout** | Максимальное время, в течение которого может работать отдельная возможность (по умолчанию 30) |
| **Debug Logging** | Логирование вызовов возможностей и рассуждений агента в журнал отладки WordPress |

## Блок безопасности управления опциями (Options Management Safety Blocklist) {#options-management-safety-blocklist}

Возможность управления опциями включает встроенный блок безопасности, который не позволяет агенту читать или записывать конфиденциальные опции WordPress. По умолчанию блок безопасности охватывает:

- `admin_email`, `blogadmin_email`
- `siteurl`, `home` (опции URL — для них используйте специализированный инструмент миграции)
- `auth_key`, `secure_auth_key`, `logged_in_key`, `nonce_key` (секретные ключи)
- `active_plugins`, `template` (активация плагина/темы — используйте специализированные возможности)
- Учетные данные и API-ключи платежных шлюзов, зарегистрированные установленными плагинами

Дополнительные записи можно добавить через фильтр `gratis_ai_agent_options_blocklist`.

## Хуки и фильтры (Hooks and Filters) {#hooks-and-filters}

### Действия (Actions) {#actions}

- `gratis_ai_agent_ability_registered` — срабатывает после регистрации возможности, получает имя класса возможности
- `gratis_ai_agent_plan_started` — срабатывает, когда начинается выполнение плана Site Builder
- `gratis_ai_agent_plan_completed` — срабатывает, когда план успешно завершен
- `gratis_ai_agent_plan_failed` — срабатывает, когда план сталкивается с невосстановимой ошибкой

### Фильтры (Filters) {#filters}

- `gratis_ai_agent_abilities` — изменение списка зарегистрированных возможностей до того, как агент его загрузит
- `gratis_ai_agent_options_blocklist` — расширение или сокращение блока безопасности опций
- `gratis_ai_agent_benchmark_questions` — добавление кастомных вопросов для набора тестов производительности
- `gratis_ai_agent_plan_plugins` — изменение списка обнаруженных плагинов во время генерации плана сайта
