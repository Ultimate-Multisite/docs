# Мультитенантность

Распределяйте подсайты по серверам и базам данных для производительности и безопасности

## Основные моменты выпуска версии 1.2.0

Multi-Tenancy 1.2.0 добавляет основу суверенного тенанта для рабочих процессов изоляции в production:

- Автовход тенанта без сохранения состояния с краткосрочными, защищенными от повторного воспроизведения SSO-визитами.
- Изоляция базы данных и файловой системы для каждого подсайта для суверенных тенантов.
- Возможность мультитенантности Hostinger для подготовки тенантов с учетом хоста.
- Проверочные шлюзы суверенной миграции для проверок устаревших данных и асинхронной обработки отправки.
- Настраиваемая привязка хоста базы данных на той же машине для установок Bedrock, FrankenPHP и localhost.
- Улучшения начальной загрузки тенанта, записи в базу данных, начальной загрузки схемы и подготовки пользователей.
- Более безопасное удаление суверенного тенанта, очищающее учетные данные тенанта.

См. руководства пользователя по [рабочему процессу суверенной миграции](/user-guide/getting-started/sovereign-migration-workflow), [изоляции мультитенантности](/user-guide/configuration/multi-tenancy-isolation) и [операциям с суверенными тенантами](/user-guide/administration/sovereign-tenant-operations).

## Требования

- WordPress 5.3 или выше
- PHP 7.4 или выше
- Multisite Ultimate plugin (активен)

## Установка

1. Загрузите файлы дополнения в ваш каталог `/wp-content/plugins/`
2. Активируйте plugin через меню 'Plugins' в WordPress
3. Настройте параметры дополнения в сетевом администрировании

### Тестирование

Дополнение включает комплексную среду тестирования:

- **Модульные тесты**: тесты на основе PHPUnit с набором тестов WordPress
- **Базовый класс тестов**: предоставляет вспомогательные методы для распространенных тестовых сценариев
- **Покрытие тестами**: настроено для отслеживания покрытия кода
- **Тестирование Multisite**: тесты выполняются в среде multisite

### Стандарты кода

- **Стандарты кодирования WordPress**: применяются через PHPCS
- **Совместимость с PHP 7.4+**: поддерживаются современные возможности PHP
- **Статический анализ**: интеграция PHPStan для проверки типов
- **Модернизация кода**: правила Rector для обновлений PHP

### Система сборки

- **Обработка ресурсов**: минификация файлов CSS/JS
- **Перевод**: генерация POT-файла
- **Упаковка**: создание архива для распространения
- **Режим разработки**: неминифицированные ресурсы для отладки

## Конфигурация

Дополнение поддерживает различные параметры конфигурации, доступные через интерфейс сетевого администрирования.

### Конфигурация суверенного тенанта

Для суверенных тенантов настройте базу данных тенанта, корневой каталог файловой системы, привязку хоста базы данных и возможность хоста перед маршрутизацией трафика к тенанту. Запускайте проверку миграции после каждого изменения, которое влияет на учетные данные базы данных, привязку хоста, маршрутизацию или подготовку пользователей.

Используйте `localhost` только тогда, когда среда выполнения тенанта и разрешения базы данных ожидают доступ через локальный сокет или с той же машины. Используйте `127.0.0.1` или имя хоста сервиса, когда это хост разрешения, настроенный вашим провайдером.

### Возможность Hostinger

Возможность мультитенантности Hostinger работает с общим токеном интеграции Hostinger. Перед проверкой убедитесь, что домен присутствует в hPanel, DNS указывает на Account Hostinger, а параметры базы данных тенанта соответствуют hPanel.

## Хуки и фильтры

### Действия

- `ultimate-multisite-addon-template_init` - Срабатывает после инициализации дополнения
- `ultimate-multisite-addon-template_loaded` - Срабатывает после загрузки всех файлов дополнения

### Фильтры

- `ultimate-multisite-addon-template_settings` - Изменить параметры дополнения
- `ultimate-multisite-addon-template_enabled` - Переопределить статус включения/отключения дополнения
