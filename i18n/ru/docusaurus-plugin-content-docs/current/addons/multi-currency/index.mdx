# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency позволяет вашей сети принимать платежи в нескольких валютах. Он автоматически конвертирует цены, используя актуальные или ручные обменные курсы, дает возможность устанавливать точные цены для каждого продукта в любой валюте и определяет предпочтительную валюту посетителя с помощью геолокации — и всё это управляется с одной административной панели.

## Основные функции {#key-features}

- **Автоматическая конвертация обменных курсов** — цены конвертируются из вашей базовой валюты с использованием курсов, полученных от выбранного вами провайдера.
- **Переопределение цен для конкретных продуктов** — вы можете установить точные цены для конкретной валюты на любой продукт, чтобы обойти автоматическую конвертацию.
- **Поле выбора валюты при регистрации** — позволяет клиентам выбрать предпочитаемую валюту во время оформления заказа/регистрации.
- **Несколько провайдеров обменных курсов** — ECB, Frankfurter, Stripe или полностью ручные курсы.
- **Определение валюты на основе геолокации** — автоматически выбирает валюту, соответствующую стране посетителя.
- **Административная панель** — управляйте активными валютами, обменными курсами и настройками провайдеров из одного места.

## Требования {#requirements}

- Ultimate Multisite 2.0.0 или выше
- Установка WordPress Multisite
- PHP 7.4 или выше

## Установка {#installation}

1. Загрузите файлы плагина в `/wp-content/plugins/ultimate-multisite-multi-currency`
2. Активируйте плагин на экране **Плагины** в WordPress (Активация для всей сети)
3. Перейдите в **Ultimate Multisite → Multi-Currency**, чтобы открыть административную панель
4. Добавьте валюты, которые вы хотите принимать, и настройте провайдера обменных курсов

## Настройка {#configuration}

### Административная панель {#admin-dashboard}

Административная панель Multi-Currency (**Ultimate Multisite → Multi-Currency**) — это центральное место для:

- Добавления, активации или деактивации валют
- Выбора провайдера обменных курсов и обновления курсов по требованию
- Просмотра текущих курсов для всех активных валют
- Установки ручных курсов, когда автоматические провайдеры не подходят

### Провайдеры обменных курсов {#exchange-rate-providers}

| Провайдер | Описание |
|---|---|
| **ECB** | Европейский центральный банк — бесплатные ежедневные курсы для сетей, основанных на EUR |
| **Frankfurter** | Открытый API-зеркало ECB — бесплатно, не требуется ключ API |
| **Stripe** | Использует актуальные курсы валют Stripe — требует активной интеграции со Stripe |
| **Ручной** | Вы вводите курсы напрямую; не требуется вызов внешнего API |

Выберите провайдера в разделе **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider**. Курсы можно обновить вручную в любое время или запланировать автоматическое обновление.

### Переопределение цен для конкретных продуктов {#per-product-pricing-overrides}

Если автоматическая конвертация недостаточно точна, вы можете установить точную цену для конкретной валюты на любой продукт:

1. Откройте редактор продукта (**Ultimate Multisite → Products → Edit**)
2. Прокрутите до раздела **Currency Pricing**
3. Введите точную цену для каждой валюты, которую вы хотите переопределить
4. Сохраните продукт

Продукты без переопределения используют конвертированную цену из вашей базовой валюты.

### Поле выбора валюты при регистрации {#currency-selector-signup-field}

Поле выбора валюты можно добавить в процесс регистрации/оформления заказа, чтобы клиенты выбирали предпочитаемую валюту при регистрации:

1. Перейдите в **Ultimate Multisite → Multi-Currency → Settings**
2. Активируйте **Show currency selector on signup**
3. Опционально установите валюту по умолчанию или позвольте геолокации определить предварительно выбранное значение

Выбранная валюта сохраняется с подпиской и используется для всего последующего выставления счетов.

### Определение валюты на основе геолокации {#geolocation-based-currency-detection}

Когда включено определение геолокации, плагин сопоставляет страну посетителя с валютой и предварительно выбирает ее в поле выбора валюты. Чтобы включить:

1. Перейдите в **Ultimate Multisite → Multi-Currency → Settings**
2. Активируйте **Detect currency from visitor location**
3. Сопоставьте страны с валютами в таблице **Geolocation Mapping**

Если для страны посетителя нет сопоставления, используется базовая валюта сети в качестве запасного варианта.

## Часто задаваемые вопросы {#frequently-asked-questions}

### Что произойдет, если провайдер обменных курсов недоступен? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

Плагин переключится на последние успешно полученные курсы. Если курсы никогда не были получены, он переключится на ручные курсы, которые вы настроили. Рекомендуется настроить ручные курсы в качестве страховки.

### Могу ли я использовать разные провайдеры для разных валют? {#can-i-use-different-providers-for-different-currencies}

Нет — для всех валют активен только один провайдер. Используйте **Manual**, если вам нужен полный контроль над отдельными курсами.

### Требуется ли для геолокации отдельный плагин? {#does-geolocation-require-a-separate-plugin}

Внешний плагин геолокации не требуется. Плагин использует встроенное в WordPress определение IP-адреса по стране. Для более высокой точности можно интегрировать выделенный сервис геолокации через предоставленные хуки фильтра.

### Переопределения цен для продуктов применяются до или после налога? {#are-per-product-overrides-applied-before-or-after-tax}

Переопределения цен для продуктов устанавливают цену до налога в целевой валюте. Налог рассчитывается поверх цены переопределения с использованием вашей существующей конфигурации налогов.
