# Урок 7: Делаем это своим

Ваши клиенты никогда не должны чувствовать, что они используют «какой-то WordPress plugin». Они должны чувствовать, что используют FitSite — платформу, созданную специально для их отрасли. В этом уроке мы рассмотрим брендинг, white-labeling и то, как сделать платформу похожей на готовый продукт.

## Откуда мы начали {#where-we-left-off}

FitSite имеет рабочий процесс оформления заказа, который ведет владельцев фитнес-студий от выбора плана до запуска сайта. Теперь мы заставим весь этот опыт выглядеть как единый, брендированный продукт.

## Домен вашей платформы {#your-platform-domain}

Основой вашего бренда является ваш домен. Для FitSite:

- **Основной домен**: `fitsite.com` (ваш маркетинговый сайт и корневой узел сети)
- **Сайты клиентов**: `studioname.fitsite.com` (поддомены)
- **Пользовательские домены**: Клиенты на планах Growth и Pro могут подключить свой собственный домен

### Настройка домена {#setting-up-your-domain}

1. Зарегистрируйте домен вашей платформы
2. Укажите его на вашего хостинг-провайдера
3. Настройте wildcard DNS (`*.fitsite.com`) для поддоменов клиентов
4. Убедитесь, что активен wildcard SSL

Подробные инструкции см. в [How to Configure Domain Mapping](/user-guide/domain-mapping/how-to-configure-domain-mapping).

## White-labeling админ-опыта {#white-labeling-the-admin-experience}

Когда владелец фитнес-студии входит в личный кабинет своего сайта, он должен видеть ваш бренд, а не брендинг WordPress или Ultimate Multisite.

### Пользовательская страница входа {#custom-login-page}

Настройте страницу входа WordPress так, чтобы она отображала:

- Логотип FitSite
- Фоновые изображения, подходящие для фитнеса
- Цвета вашего бренда

### Брендинг в админ-панели {#dashboard-branding}

Используйте аддон [Admin Page Creator](/addons/admin-page-creator) или пользовательский CSS, чтобы:

- Заменить логотип WordPress на логотип FitSite
- Настроить цветовую схему админки в соответствии с вашим брендом
- Добавить пользовательский виджет в админ-панель с быстрыми ссылками и ресурсами помощи, специфичными для фитнеса

### Пользовательские страницы админки {#custom-admin-pages}

Рассмотрите возможность создания пользовательских страниц админки, которые выводят наиболее релевантные действия для владельцев фитнес-студий:

- «Редактировать расписание занятий»
- «Обновить профили тренеров»
- «Посмотреть свой сайт»

Это снижает кривую обучения, выводя нишево-релевантные действия на первый план, вместо того чтобы прятать их в стандартное меню WordPress.

## Брендирование коммуникаций {#branding-your-communications}

Каждое письмо, каждый счет и каждое уведомление должны усиливать ваш бренд.

### Системные письма {#system-emails}

Перейдите в **Ultimate Multisite > Settings > Emails** и настройте все системные письма:

- **От кого**: FitSite
- **Email от**: hello@fitsite.com
- **Шаблоны писем**: Используйте цвета и логотип вашего бренда
- **Язык**: Фитнес-специфичный во всех частях

Ключевые письма для настройки:

| Email | Generic Version | FitSite Version |
|-------|----------------|-----------------|
| Welcome | "Ваш новый сайт готов" | "Ваш фитнес-сайт запущен" |
| Payment receipt | "Оплата получена" | "Подписка FitSite подтверждена" |
| Trial ending | "Ваша пробная подписка скоро закончится" | "Ваша пробная подписка FitSite заканчивается через 3 дня — сохраните свой фитнес-сайт активным" |

### Счета-фактуры {#invoices}

Настройте шаблоны счетов-фактур, включив:

- Логотип FitSite и цвета вашего бренда
- Ваши бизнес-данные
- Названия продуктов, специфичные для фитнеса (а не общие ID планов)

## Сайт для клиентов {#the-customer-facing-site}

Ваш основной домен (`fitsite.com`) должен иметь маркетинговый сайт, который продает платформу. Это отделено от админки сети Ultimate Multisite — это публичное лицо вашего бизнеса.

Ключевые страницы:

- **Главная страница**: Четкое ценностное предложение для фитнес-бизнеса
- **Функции**: Что делает FitSite, но в фитнес-терминах
- **Цены**: Ваши три плана с сравнением функций, специфичных для ниши
- **Примеры**: Демонстрация сайтов, созданных на платформе
- **Регистрация**: Ссылки на вашу форму оформления заказа

:::tip Ваш маркетинговый сайт может быть сайтом сети
Создайте свой маркетинговый сайт как сайт в собственной сети. Это позволит вам управлять им из одной и той же админки и продемонстрирует возможности вашей собственной платформы.
:::

## Пользовательский домен для клиентов {#custom-domain-for-customers}

Для клиентов, чьи планы включают пользовательские домены, четко документируйте процесс:

1. Клиент регистрирует или переносит свой домен к регистратору
2. Клиент обновляет DNS, чтобы он указывал на вашу платформу (предоставьте точные записи)
3. Ultimate Multisite обрабатывает маппинг домена и SSL

Создайте статью в справке или запись в базе знаний, специально посвященную этому процессу, написанную для нетехнических владельцев фитнес-студий.

```
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow (niche-specific, tested)
├── Branding
│   ├── Custom login page
│   ├── Branded admin dashboard
│   ├── Niche-specific system emails
│   ├── Branded invoices
│   └── Marketing site on fitsite.com
└── Ready for onboarding flow (next lesson)
```

## Что мы построили в этом уроке {#what-we-built-this-lesson}

- **Домен платформы и DNS** настроены для брендированного опыта
- **White-labeled админка** с брендингом FitSite на всех уровнях
- **Настроенные коммуникации** — письма, счета и уведомления полностью соответствуют бренду
- **Маркетинговый сайт**, который продает FitSite владельцам фитнес-студий
