# Урок 6: Опыт регистрации

Процесс оформления заказа — это место, где интерес превращается в доход. Непонятный или слишком общий процесс регистрации отпугивает клиентов. А вот воронка, адаптированная под конкретную нишу и написанная на языке этой ниши, кажется естественной и легко конвертирует клиентов.

## Где мы остановились

В FitSite настроены шаблоны и тарифные планы. Теперь нам нужно создать процесс регистрации и оформления заказа, который превратит владельцев фитнес-студий в платящих клиентов.

## Понимание процесса регистрации

Формы оформления заказа Ultimate Multisite — это полностью настраиваемые многошаговые формы. Для FitSite нам нужна воронка, которая:

1. Кажется, будто она создана специально для фитнес-бизнеса
2. Собирает только необходимую информацию
3. Позволяет клиенту как можно быстрее получить работающий сайт

Подробное техническое описание процесса можно посмотреть в [The Registration Flow](/user-guide/configuration/the-registration-flow).

## Проектирование оформления заказа FitSite

Перейдите в **Ultimate Multisite > Checkout Forms** и создайте новую форму.

### Шаг 1: Выбор плана

Первое, что должен увидеть владелец фитнес-студии, — это тарифные планы, представленные на языке, который он понимает.

- Добавьте поле **Pricing Table** (Таблица тарифов)
- Настройте его так, чтобы отображались все три плана FitSite
- Описания планов, которые вы написали в Уроке 5, должны появиться здесь — убедитесь, что они говорят о потребностях фитнес-бизнеса, а не о технических функциях

:::tip Язык ниши имеет значение
«1 ГБ хранилища» ничего не значит для владельца спортзала. А «Всё необходимое для профессионального сайта студии» — значит всё. Пишите описания планов на языке вашего клиента.
:::

### Шаг 2: Выбор шаблона

После выбора плана клиент выбирает свой стартовый шаблон.

- Добавьте поле **Template Selection** (Выбор шаблона)
- Доступные шаблоны фильтруются в зависимости от выбранного плана (это было настроено в Уроке 5)
- Каждый шаблон должен иметь превью-изображение, демонстрирующее дизайн, специфичный для фитнеса

### Шаг 3: Создание аккаунта

Сделайте этот шаг максимально простым. Собирайте только:

- Адрес электронной почты
- Пароль
- Название студии/бизнеса (это станет названием их сайта)

Не запрашивайте информацию, которая вам не нужна на этапе регистрации. Каждое лишнее поле снижает конверсию.

### Шаг 4: Настройка сайта

- **Site title** (Название сайта): Автозаполните из названия студии, введенного на Шаге 3
- **Site URL** (URL сайта): Автоматически генерируется из названия студии (например, `ironworks.fitsite.com`)

### Шаг 5: Оплата

- Добавьте поле **Payment** (Оплата)
- Настройте ваш платежный шлюз ([Stripe](/user-guide/payment-gateways/stripe) рекомендуется для выставления счетов по подписке)
- Если вы создавали дополнительные товары (order bumps) в Уроке 5, добавьте поле **Order Bump** перед шагом оплаты

### Шаг 6: Подтверждение

- Настройте сообщение об успешном завершении, используя язык, специфичный для фитнеса
- Пример: «Ваш сайт фитнес-студии создается. Через несколько секунд вы будете перенаправлены на ваш новый сайт.»

## Добавление переключателя выбора периода

Если вы настроили вариации цен в своих планах (ежемесячно против ежегодно), добавьте поле **Period Selection** (Выбор периода) в форму оформления заказа, чтобы клиенты могли переключаться между периодами оплаты. Инструкции смотрите в [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle).

## Настройка оплаты

Если вы еще не настроили платежный шлюз:

1. Перейдите в **Ultimate Multisite > Settings > Payment Gateways**
2. Следуйте [руководству по настройке Stripe](/user-guide/payment-gateways/stripe) или используйте свой предпочтительный шлюз
3. Протестируйте полный процесс оформления заказа с тестовой оплатой

Подробнее о том, как платежи поступают на ваш счет, смотрите в [Getting Paid](/user-guide/payment-gateways/getting-paid).

## Тестирование процесса

Прежде чем двигаться дальше, выполните полный тестовый процесс регистрации:

1. Откройте форму оформления заказа в режиме инкогнито/приватном окне браузера
2. Выберите план
3. Выберите шаблон
4. Создайте аккаунт
5. Оплатите (используйте тестовый режим)
6. Проверьте, что сайт создан с правильным шаблоном

Убедитесь, что:

- [ ] Описания планов ясные и специфичные для ниши
- [ ] Превью шаблонов показывают дизайны, подходящие для фитнеса
- [ ] URL сайта генерируется корректно из названия студии
- [ ] Оплата проходит успешно
- [ ] Клиент попадает на работающий сайт с выбранным шаблоном
- [ ] Письма-подтверждения используют язык, специфичный для фитнеса

## Сеть FitSite на данный момент

```
FitSite Network
├── WordPress Multisite (режим поддоменов)
├── Ultimate Multisite (настроен)
├── Хостинг с wildcard SSL + маппинг доменов
├── Шаблоны сайтов (Studio Essential, Gym Pro, Fitness Chain)
├── Продукты (Starter, Growth, Pro + Order Bumps)
├── Процесс оформления заказа
│   ├── Выбор плана с описаниями, специфичными для ниши
│   ├── Выбор шаблона с превью для фитнеса
│   ├── Минимальное создание аккаунта
│   ├── Оплата через Stripe
│   └── Подтверждение, специфичное для фитнеса
└── Готов к брендированию (следующий урок)
```

## Что мы построили в этом уроке

- **Многошаговая форма оформления заказа**, адаптированная для владельцев фитнес-студий
- **Язык, специфичный для ниши**, на протяжении всего процесса регистрации
- **Минимальное трение** — только необходимые поля, быстрый путь к работающему сайту
- **Интеграция оплаты** с тестовой проверкой
- **Протестированный сквозной процесс** от выбора плана до работающего сайта
