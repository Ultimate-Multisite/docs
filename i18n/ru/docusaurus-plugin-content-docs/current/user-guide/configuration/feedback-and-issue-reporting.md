# Обратная связь и отчеты о проблемах

Gratis AI Agent v1.5.0 включает встроенную систему обратной связи и отчетности о проблемах. Она позволяет конечным пользователям отмечать не полезные ответы и сообщать о проблемах прямо из интерфейса чата. Эта система включает управление согласием, автоматизированную команду для отчетности и предварительную обработку данных (triage) с помощью AI на бэкенде.

## Кнопка «Палец вниз»

Каждое сообщение, отправленное AI-помощником, отображает кнопку **«Палец вниз»** (👎). Нажав на нее, пользователь может пометить ответ как бесполезный или неверный.

- Кнопка появляется при наведении курсора рядом с каждым сообщением помощника.
- Нажатие открывает **Модальное окно согласия на обратную связь**.
- Обратная связь привязывается к ветке разговора и к конкретному сообщению.

## Модальное окно согласия на обратную связь

Когда пользователь нажимает кнопку «Палец вниз», появляется модальное окно согласия, прежде чем будут отправлены какие-либо данные. В этом окне:

- Объясняется, какая информация будет собрана (фрагмент разговора, контекст браузера).
- Пользователю предлагается подтвердить согласие на передачу этих данных.
- Предоставляется поле для ввода текста, где пользователь может описать, что пошло не так.
- Предлагается опция **Отмена** для закрытия окна без отправки данных.

Обратная связь записывается только после явного подтверждения пользователем.

## Баннер обратной связи по авто-промпту

В конце разговора помощник может отобразить **баннер обратной связи по авто-промпту** — ненавязчивое сообщение, спрашивающее, была ли сессия полезной.

Этот баннер появляется автоматически на основе длины и результатов разговора (по эвристике). Он ведет к тому же потоку обратной связи, что и кнопка «Палец вниз». Пользователи могут закрыть баннер, не предоставляя обратную связь.

## Команда /report-issue

Пользователи могут запустить процесс предоставления обратной связи напрямую, введя `/report-issue` в поле ввода чата. Эта команда:

- Немедленно открывает Модальное окно согласия на обратную связь.
- Предзаполняет поле описания контекстом текущего разговора.
- Позволяет пользователям добавить дополнительные детали перед отправкой.

Команда `/report-issue` доступна во всех режимах чата (встроенный, плавающий виджет, полноэкранный).

## Предварительная обработка данных с помощью AI (AI-Assisted Triage)

Отправленная обратная связь направляется в слой предварительной обработки данных (triage), который:

- Категоризирует отчет (фактическая ошибка, бесполезный ответ, неприемлемый контент и т. д.).
- Прикрепляет соответствующие метаданные разговора.
- Пересылает сводку предварительной обработки данных в настроенный **Endpoint обратной связи** (см. [Настройки > Дополнительно](#settings-advanced)).

Это сокращает время ручной проверки, выделяя наиболее критические проблемы в первую очередь.

## Настройки > Дополнительно {#settings-advanced}

Чтобы включить бэкенд обратной связи, настройте следующие поля в разделе **Gratis AI Agent → Настройки → Дополнительно**:

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** | URL, который принимает POST-запросы с полезной нагрузкой обратной связи (JSON). |
| **Feedback API Key** | Bearer-токен, отправляемый в заголовке `Authorization` для аутентификации отправки обратной связи. |

Оставьте оба поля пустыми, чтобы отключить сбор обратной связи. Индивидуальные кнопки обратной связи остаются видимыми для пользователей, но отправка данных не будет осуществляться.

:::tip
Endpoint обратной связи должен принимать тело JSON с полями как минимум `message_id`, `conversation_id`, `feedback_text` и `triage_category`. См. документацию вашего провайдера endpoint для ожидаемой схемы.
:::
