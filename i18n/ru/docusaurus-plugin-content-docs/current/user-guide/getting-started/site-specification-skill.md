# Навык определения спецификации сайта {#site-specification-skill}

**Навык определения спецификации сайта** — это структурированный подход к фиксации целей вашего сайта, целевой аудитории и фирменного стиля. Эта информация сохраняется в вашей памяти **site_brief**, к которой агенты обращаются в течение сессий, чтобы предоставлять последовательную и учитывающую контекст помощь.

## Что такое спецификация сайта? {#what-is-site-specification}

Спецификация сайта — это процесс документирования:

- **Цель сайта**: Что делает ваш сайт и зачем он нужен
- **Целевая аудитория**: Кто посещает ваш сайт и что ему нужно
- **Фирменный стиль**: Ваши цвета, тон и визуальное оформление
- **Бизнес-цели**: Как выглядит успех для вашего сайта
- **Структура контента**: Как организован ваш сайт

Эта спецификация становится вашим **site_brief** — постоянной памятью, которую агенты используют для понимания контекста вашего сайта.

## Зачем использовать спецификацию сайта? {#why-use-site-specification}

### Последовательность в течение сессий {#consistency-across-sessions}

Без site_brief вам придется заново объяснять цель вашего сайта каждый раз, когда вы начинаете новую сессию. С ним агенты сразу понимают:

- Цели и аудиторию вашего сайта
- Ваши фирменные цвета и тон
- Структуру вашего контента
- Ваши бизнес-задачи

### Лучшие рекомендации {#better-recommendations}

Агенты используют ваш site_brief, чтобы:

- Предлагать функции, соответствующие цели вашего сайта
- Рекомендовать структуры контента, соответствующие вашим целям
- Предлагать дизайны, соответствующие вашему бренду
- Избегать предложения несовместимых функций

### Более быстрое освоение {#faster-onboarding}

Новые агенты (или агенты в новых сессиях) могут быстро войти в курс дела, прочитав ваш site_brief, вместо того чтобы задавать уточняющие вопросы.

## Инициирование спецификации сайта {#initiating-site-specification}

### Во время настройки Theme Builder {#during-theme-builder-onboarding}

Навык определения спецификации сайта автоматически запускается в процессе **onboarding Theme Builder**. Агент Setup Assistant задает вопросы и создает ваш site_brief.

### Ручное инициирование {#manual-initiation}

Вы можете запустить определение спецификации сайта в любое время:

```
"Давай определим спецификацию моего сайта"
```

или

```
"Помоги мне создать site brief"
```

## Процесс определения спецификации сайта {#the-site-specification-process}

### Шаг 1: Цель сайта {#step-1-site-purpose}

Агент спрашивает:

```
Какова основная цель вашего сайта?
- Интернет-магазин
- Блог или контентный сайт
- Портфолио или витрина
- SaaS-приложение
- Сообщество или форум
- Другое: [опишите]
```

Вы можете выбрать категорию или описать свою собственную цель.

### Шаг 2: Целевая аудитория {#step-2-target-audience}

```
Кто ваша основная аудитория?
- Потребители / широкая публика
- Бизнес-профессионалы
- Разработчики / технические пользователи
- Студенты / преподаватели
- Другое: [опишите]

Каковы их основные потребности?
```

### Шаг 3: Фирменный стиль {#step-3-brand-identity}

```
Какие цвета вашего бренда?
- Основной цвет: [выбор цвета или HEX-код]
- Вторичный цвет: [выбор цвета или HEX-код]
- Акцентный цвет: [необязательно]

Как бы вы описали тон вашего бренда?
- Профессиональный / корпоративный
- Креативный / художественный
- Игривый / неформальный
- Минималистичный / современный
- Теплый / дружелюбный
```

### Шаг 4: Бизнес-цели {#step-4-business-goals}

```
Как выглядит успех для вашего сайта?
- Генерация лидов
- Продажа продуктов
- Построение сообщества
- Обмен знаниями
- Установление авторитета
- Другое: [опишите]

Каков ваш основной показатель?
- Выручка
- Вовлеченность пользователей
- Охват контента
- Конверсии
- Другое
```

### Шаг 5: Структура контента {#step-5-content-structure}

```
Как организован ваш контент?
- Плоский (весь контент на одном уровне)
- Иерархический (категории и подкатегории)
- Хронологический (в стиле блога)
- По продуктам (каталог)
- Другое: [опишите]

Какие типы контента вы используете?
- Посты в блоге
- Страницы продуктов
- Кейсы
- Документация
- Видео
- Другое
```

## Ваша память site_brief {#your-sitebrief-memory}

После завершения определения спецификации сайта ваша информация сохраняется как **site_brief** в памяти вашего агента. Это структурированная запись, содержащая:

```json
{
  "site_purpose": "Интернет-магазин украшений ручной работы",
  "target_audience": "Женщины 25-45 лет, интересующиеся устойчивой модой",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Теплый, ремесленный, устойчивый",
  "business_goals": ["Генерация продаж", "Построение лояльности бренда"],
  "primary_metric": "Выручка",
  "content_structure": "Каталог продуктов с блогом",
  "content_types": ["Страницы продуктов", "Посты в блоге", "Истории клиентов"]
}
```

## Просмотр и обновление site_brief {#viewing-and-updating-your-sitebrief}

### Просмотр site_brief {#view-your-sitebrief}

Спросите агента:

```
"Покажи мне мой site brief"
```

или

```
"Что ты знаешь о моем сайте?"
```

Агент отобразит вашу сохраненную спецификацию.

### Обновление site_brief {#update-your-sitebrief}

Если ваш сайт меняется, вы можете обновить site_brief:

```
"Обнови мой site brief: теперь мы нацелены на B2B-клиентов"
```

или

```
"Обновить спецификацию моего сайта"
```

Это повторно запустит навык определения спецификации сайта, используя вашу существующую информацию в качестве отправной точки.

## Как агенты используют site_brief {#how-agents-use-sitebrief}

### Рекомендации по дизайну {#design-recommendations}

Когда вы запрашиваете изменения в дизайне, агенты обращаются к вашему site_brief:

```
Вы: "Переработай главную страницу"
Агент: "Исходя из вашего site brief, вы нацелены на женщин 25-45 лет
с теплым, ремесленным тоном. Я создам дизайн, который подчеркнет
ваши украшения ручной работы и ценности устойчивого развития."
```

### Предложения функций {#feature-suggestions}

Агенты предлагают функции, соответствующие вашим целям:

```
Вы: "Какие функции мне добавить?"
Агент: "Для интернет-магазина, ориентированного на лояльность клиентов,
я бы порекомендовал: программа лояльности, отзывы клиентов, рассылку по электронной почте
и рекомендации продуктов."
```

### Структура контента {#content-structure}

Агенты предлагают организацию контента, исходя из вашей структуры:

```
Вы: "Как мне организовать блог?"
Агент: "Поскольку вы используете каталог продуктов с блогом, я бы предложил
организовать посты по категориям продуктов и создать 'гиды по стилю',
которые демонстрируют несколько продуктов вместе."
```

## Лучшие практики {#best-practices}

### Будьте конкретны {#be-specific}

Вместо "широкая аудитория" опишите вашу реальную аудиторию:

- ✓ "Женщины 25-45 лет, интересующиеся устойчивой модой"
- ✗ "Все"

### Обновляйте регулярно {#update-regularly}

По мере развития вашего сайта обновляйте site_brief:

- Когда вы меняете целевую аудиторию
- Когда добавляете новые линейки продуктов
- Когда меняется ваш фирменный стиль
- Когда смещаются ваши бизнес-цели

### Используйте последовательную терминологию {#use-consistent-terminology}

Используйте одни и те же термины в течение сессий:

- ✓ Всегда говорите "экологичные украшения" (а не "эко-дружественные украшения" и "зеленые продукты")
- ✓ Последовательно называйте свою аудиторию одинаково

### Учитывайте контекст {#include-context}

Предоставляйте предысторию, которая помогает агентам понять ваши решения:

- "Мы нацелены на профессионалов, которые ценят качество выше цены"
- "Наша аудитория технически подкована и ожидает современного дизайна"
- "Мы стартап с ограниченным бюджетом, поэтому нам нужны экономичные решения"

## Связь с настройкой Theme Builder {#relationship-to-theme-builder-onboarding}

Навык определения спецификации сайта интегрирован в процесс **onboarding Theme Builder**. Когда вы завершаете настройку, ваш site_brief автоматически создается с предоставленной вами информацией.

Вы также можете запустить определение спецификации сайта независимо, если захотите:

- Уточнить спецификацию после первоначальной настройки
- Обновить site_brief по мере развития сайта
- Создать подробную спецификацию до начала работы с Theme Builder

## Устранение неполадок {#troubleshooting}

**Мой site_brief не используется**
- Убедитесь, что у агента есть доступ к памяти
- Попросите агента "вспомнить мой site brief"
- Проверьте, что память включена в ваших настройках

**Я хочу начать с нуля с новым site_brief**
- Попросите агента: "Очистить мой site brief и начать заново"
- Затем снова запустите определение спецификации сайта

**Агент дает рекомендации, не соответствующие моему site_brief**
- Попросите агента "проверить мой site brief"
- Обновите свой site_brief, если он устарел
- Предоставьте дополнительный контекст в своих запросах

## Следующие шаги {#next-steps}

После определения спецификации сайта:

1. **Используйте Theme Builder**: Создайте кастомную тему на основе вашего site_brief
2. **Уточните дизайн**: Используйте навык Design System Aesthetics для детальной проработки дизайна
3. **Планируйте контент**: Попросите агентов рекомендации по структуре контента
4. **Создавайте функции**: Запросите функции, соответствующие вашим бизнес-целям
