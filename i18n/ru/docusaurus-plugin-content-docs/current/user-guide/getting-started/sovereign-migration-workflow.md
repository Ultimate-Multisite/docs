# Рабочий процесс миграции в суверенный режим

Ultimate Multisite: Multi-Tenancy 1.2.0 добавляет ворота проверки миграции для переноса стандартного субсайта в инфраструктуру суверенного арендатора (tenant).

## Перед началом

Убедитесь, что у вас есть:

- Текущая резервная копия исходного сайта.
- Добавленный плагин Multi-Tenancy активен.
- База данных целевого арендатора, корневая файловая система и домен готовы.
- Привязка хоста базы данных, соответствующая целевой среде.
- Доступ для выполнения команд WP-CLI для сети.

## Рекомендуемый рабочий процесс

1. Подготовьте базу данных и файловую систему целевого арендатора.
2. Зарегистрируйте или обновите настройки изоляции арендаторов (tenant isolation settings).
3. Выполните миграцию арендатора.
4. Разверните или проверьте пользователей арендатора.
5. Очистите очередь асинхронных миграционных задач.
6. Запустите проверку суверенной миграции.
7. Посетите арендатора с помощью SSO.
8. Измените DNS или маршрутизацию только после успешного прохождения проверки.

## Ворота проверки

Рабочий процесс проверки проверяет миграцию с нескольких сторон:

- Схема арендатора существует и может быть записана писателем базы данных арендатора.
- Настроенный хост базы данных имеет действительные права доступа (grants).
- Пользователи арендатора присутствуют и соответствуют ожидаемому количеству пользователей установки суверенного режима.
- Очередь асинхно отправляемых задач может быть успешно очищена.
- Устаревшие пути к данным на стороне сети больше не требуются для суверенного арендатора.

Рассматривайте сбои проверки как блокирующие факторы перед запуском. Исправьте сообщаемую проблему с базой данных, пользователями, очередью или маршрутизацией, а затем повторите проверку, прежде чем открывать арендатора для клиентов.

## Первый посещение в продакшене

После успешного прохождения проверки используйте **Посетить (SSO)** из экрана управления сайтом для первого административного посещения. Это подтверждает маршрутизацию арендатора, обработку токенов SSO, привязку источника и настройку пользователей на стороне арендатора за один контролируемый шаг.
