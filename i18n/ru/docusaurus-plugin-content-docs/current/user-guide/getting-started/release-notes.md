# Примечания к выпуску

## Версия 2.12.0 — Выпущено 2026-05-15

- Новое: Добавлен Hostinger (hPanel) в список поддерживаемых хостинг-провайдеров с интеграцией маппинга доменов.
- Новое: Site Exporter теперь обрабатывает пакеты импорта сети для упрощенного восстановления сайта в масштабах всей сети.
- Исправлено: Рассылки BCC теперь используют заголовок `undisclosed-recipients`, чтобы предотвратить раскрытие адресов получателей.
- Исправлено: Дата истечения срока членства больше не повреждается при сохранении с не-датой значением.
- Исправлено: Обновления членства Stripe теперь корректно сбрасывают скидки, не вызывая устаревший API `deleteDiscount`.
- Исправлено: Перенаправления SSO на сайтах с маппингом домена теперь ограничены, чтобы предотвратить бесконечные циклы перенаправлений.
- Исправлено: Выбор изображения в мастере настройки теперь корректно обновляет базовую модель данных.
- Исправлено: CLI Site Exporter теперь сохраняет правильный выбор сайта по умолчанию для сети.
- Улучшено: Удален встроенный `wp-cli` из пакета плагина, что уменьшило размер плагина.

## Версия 2.11.0 — Выпущено 2026-05-11

- Новое: Экспорт сайтов теперь включает самозапускающийся `index.php`, чтобы ZIP-архив можно было установить на свежий хостинг без отдельной установки плагина.
- Новое: Экспорт сети позволяет администраторам экспортировать все дочерние сайты в одном архиве из админ-страницы Site Export.
- Новое: Переключатель плана Site Templates теперь принудительно контролируется через цепочку запасных вариантов, корректно ограничивая доступность шаблонов в зависимости от лимитов плана.
- Новое: Редактор формы оформления заказа предупреждает, если продукт добавлен без настроенного обязательного поля.
- Новое: Вкладка настроек Импорт/Экспорт теперь четко описывает свою область действия и напрямую ведет к инструменту Site Export.

## Версия 2.10.0 — Выпущено 2026-05-05

- Новое: Пошаговый мастер настройки PayPal для ручного ввода учетных данных с шлюзом флага OAuth для бесшовной настройки платежного шлюза.
- Новое: Панель клиента для переключения шаблонов переработана и включает карточку текущего шаблона, постоянную сетку и кнопку **Сбросить текущий шаблон**.
- Исправлено: Переключение шаблонов больше не вызывает зависание интерфейса при сбое AJAX.
- Исправлено: Статусы разрешений на переключение шаблонов защищены от несанкционированного доступа.
- Исправлено: Введенные данные для переопределения сайта проходят проверку перед сохранением.
- Исправлено: Запрос биллингового адреса теперь отображается, если адрес пуст.
- Исправлено: Устранены предупреждения об устаревании `null-to-string` в PHP 8.1.
- Исправлено: Currents загружается через хук `before_init` с отложенной загрузкой, чтобы предотвратить проблемы со временем.
- Исправлено: Отфильтрованный путь SSO соблюдается во всех процессах входа.
- Исправлено: Опции идентификации для пустого сайта сохраняются при сохранении.

## Версия 2.9.0 — Выпущено 2026-04-30

- Новое: Добавлен экспорт и импорт одного сайта в разделе **Инструменты > Экспорт и импорт**.
- Исправлено: ZIP-файлы экспорта теперь обслуживаются через аутентифицированную конечную точку загрузки.
- Исправлено: Устранены риски SQL-инъекции и проблемы с запросами в запросах на ожидающий экспорт/импорт.
- Исправлено: Временный сайт не публикуется, когда администратор вручную подтверждает адрес электронной почты клиента.
- Исправлено: Потерянные записи `pending_site` очищаются, если отсутствует членство.
- Исправлено: Исправлен отступ навигации настроек и навигация по якорям поиска.
- Исправлено: Временные сайты теперь отображаются первыми в режиме «Все сайты».
- Исправлено: Добавлен заголовок User-Agent поставщика скриншотов (mShots), чтобы предотвратить ошибки 403.
- Исправлена циклическая зависимость расписания импорта cron.
- Исправлено: ID тура нормализованы до подчеркиваний в ключах настроек пользователя.
- Улучшено: Теперь используется ZipArchive вместо Alchemy/Zippy для лучшей совместимости.

## Версия 2.8.0 — Выпущено 2026-04-29

- Новое: В настройках «Другие опции» добавлен переключатель Jumper.
- Новое: В список форм оформления заказа добавлена колонка статуса.
- Новое: Загрузчик файлов Addon sunrise для пользовательских расширений MU-plugin sunrise.
- Улучшено: Удалена опция включения отчетов об ошибках со страницы настроек.
- Исправлено: Карточка сайта на странице благодарности — изображение теперь ограничено, а ссылки стилизованы правильно.
- Исправлено: Поставщик скриншотов изменен с thum.io на WordPress.com mShots.
- Исправлено: «Включить регистрацию» и «Роль по умолчанию» теперь устанавливают правильные значения по умолчанию при новой установке.
- Исправлено: `get_site_url()` больше не возвращает пустую строку, когда домен включает порт.
- Исправлено: Клонирование файлов медиа теперь копируется корректно, когда настройка `copy_media` была пустой.
- Исправлено: Кэш объектов обнуляется корректно после записи sitemeta в режиме network-activate.
- Исправлено: Пользовательский домен автоматически повышается до основного при проверке DNS для трехчастных доменов.
- Исправлено: Временное членство отменяется при очистке истекшего платежа.
- Исправлено: Проверка силы пароля восстанавливается после закрытия всплывающего окна входа.
- Исправлено: Прекращена бесконечная перезагрузка страницы на странице благодарности, если сайт уже создан.
- Исправлено: Опция регистрации WP core синхронизирована при активации плагина и сохранении настроек.
- Исправлено: Добавлена защита от нулевого срока в `calculate_expiration` для совместимости с PHP 8.4.
- Исправлено: Заблокирована повторная регистрация, если у клиента уже есть активное членство.
- Исправлено: Добавлена проверка на null для `date_expiration` в оформлении заказа.
- Исправлено: Усилено предоставление сайта — ограничения, вывод информации о членстве, повышение домена.
- Исправлено: Статус метки предварительной установки исправлен на «НЕ активирован», если проверка не удалась.
- Исправлено: Для верификации электронной почты используется домен оформления заказа.
- Исправлено: Автоматический вход после оформления заказа, если нет поля для пароля.
- Исправлено: Бесплатное членство больше не истекает — считается пожизненным.
- Исправлено: Шлюз проверки электронной почты удерживает публикацию сайта до подтверждения адреса электронной почты клиентом.
- Исправлено: Исправлен базовый путь конечной точки API SES v2 и маршрут идентификации.
- Исправлено: Хук `wu_inline_login_error` испускается в блоке перехвата `pre-submit`.
