# Примечания к выпуску {#release-notes}

## Версия 2.13.0 — выпущена 2026-06-05 {#version-2130--released-on-2026-06-05}

- Новое: добавлена поддержка суверенных арендаторов для потоков клиентского Account, checkout, выставления счетов, сайта, инвойса, переключения шаблонов и сопоставления доменов, чтобы сети арендаторов могли направлять клиентов обратно на основной сайт для управляемых действий.
- Новое: добавлены проверки учетных данных продления для регулярных membership, чтобы шлюзы могли отключать автопродление, если отсутствует сохраненное соглашение об оплате, subscription или vault token.
- Новое: добавлена публикация через loopback с проверкой HMAC для ожидающего создания сайта, чтобы provisioning от checkout к сайту был надежнее на хостах, где фоновые задания задерживаются.
- Новое: добавлены точки расширения для разработчиков для SSO URL, базовых доменов формы checkout и автоматического создания записей домена.
- Исправление: SSO стал надежнее на сопоставленных доменах, при анонимных посещениях брокера, выходе из системы и конфликтах зависимостей между plugin.
- Исправление: ожидающее создание сайта теперь восстанавливается после устаревших флагов публикации и не оставляет клиентов застрявшими на экране создания сайта.
- Исправление: записи домена больше не создаются для общих базовых доменов формы checkout, а неиспользуемые фоновые задания провайдера хостинга пропускаются, когда нет активной интеграции.
- Исправление: пограничные случаи checkout, платежного адреса, сброса пароля, подтверждения email, переключения шаблонов, туров и клиентского dashboard больше не блокируют обычные клиентские потоки.
- Исправление: массовые email-рассылки теперь сохраняют получателей приватными, избегая фатальных ошибок SMTP/plugin при сбое списков получателей или почтовых транспортов.
- Исправление: пограничные случаи продлений membership, отображения срока действия и сбора платежей теперь избегают немедленных истечений срока, сбоев или пропущенных обязательных платежей.
- Улучшено: совместимость с WordPress протестирована до версии 7.0, production Vue-ресурсы пересобраны из npm-источников, а покрытие Cypress end-to-end теперь проверяет больше потоков checkout, настройки, SSO и шлюзов.

## Версия 2.12.0 — выпущена 2026-05-15 {#version-2120--released-on-2026-05-15}

- Новое: добавлен Hostinger (hPanel) как поддерживаемый провайдер хостинга с интеграцией сопоставления доменов
- Новое: Site Exporter теперь обрабатывает пакеты импорта сети для упрощенного восстановления сайтов в масштабе всей сети
- Исправление: BCC email-рассылки теперь используют заголовок undisclosed-recipients, чтобы предотвратить раскрытие адресов получателей
- Исправление: дата истечения membership больше не повреждается при сохранении с недатовым значением
- Исправление: обновления Stripe membership теперь корректно очищают скидки без вызова устаревшего deleteDiscount API
- Исправление: перенаправления SSO на сайтах с сопоставленными доменами теперь ограничиваются, чтобы предотвратить бесконечные циклы перенаправлений
- Исправление: выбор в пикере изображений мастера настройки теперь корректно обновляет базовую модель данных
- Исправление: Site Exporter CLI теперь сохраняет правильный выбор сайта сети по умолчанию
- Улучшено: встроенный wp-cli удален из пакета plugin, что уменьшило размер plugin

## Версия 2.11.0 — выпущена 2026-05-11 {#version-2110--released-on-2026-05-11}

- Новое: экспорты сайта теперь включают самозагружающийся `index.php`, чтобы ZIP можно было установить на свежем хосте без отдельной установки plugin.
- Новое: экспорт сети позволяет администраторам экспортировать все подсайты в один архив со страницы администрирования Site Export.
- Новое: переключатель плана Allow Site Templates теперь применяется через цепочку fallback, корректно ограничивая доступность шаблонов в соответствии с лимитами плана.
- Новое: редактор формы checkout предупреждает, когда продукт добавляется без настроенного обязательного поля.
- Новое: вкладка настроек Import/Export теперь четко описывает свою область действия и напрямую ссылается на инструмент Site Export.

## Версия 2.10.0 — выпущена 2026-05-05 {#version-2100--released-on-2026-05-05}

- Новое: мастер управляемой настройки PayPal для ручного ввода учетных данных с флагом OAuth для бесшовной настройки шлюза.
- Новое: клиентская панель переключения шаблонов переработана с карточкой текущего шаблона, постоянной сеткой и кнопкой **Сбросить текущий шаблон**.
- Исправление: переключение шаблонов больше не зависает в UI при сбое AJAX.
- Исправление: состояния разрешений переключения шаблонов защищены от несанкционированного доступа.
- Исправление: поля переопределения сайта проверяются перед сохранением.
- Исправление: запрос платежного адреса теперь показывается, когда адрес пуст.
- Исправление: устранены уведомления PHP 8.1 об устаревшем преобразовании null в строку.
- Исправление: Currents загружается отложенно до init hook, чтобы предотвратить проблемы синхронизации.
- Исправление: отфильтрованный путь SSO учитывается во всех потоках входа.
- Исправление: пустые параметры идентичности сайта сохраняются при сохранении.

- Исправление: осиротевшие записи pending_site очищаются, когда отсутствует membership.
- Исправление: исправлены отступы навигации настроек и навигация по якорю поиска.
- Исправление: ожидающие сайты теперь показываются первыми в представлении All Sites.
- Исправление: добавлен заголовок User-Agent для провайдера скриншотов (mShots), чтобы предотвратить ошибки 403.
- Исправление: устранена циклическая зависимость расписания cron импорта.
- Исправление: ID туров нормализуются до подчеркиваний в ключах пользовательских настроек.
- Улучшено: теперь используется ZipArchive вместо Alchemy/Zippy для лучшей совместимости.
