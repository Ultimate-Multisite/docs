# Мультиаренда на Hostinger

Ultimate Multisite: Multi-Tenancy 1.2.0 добавляет возможность мультиаренды от Hostinger, позволяя размещенным доменам участвовать в предоставлении суверенных арендных ресурсов параллельно с существующей интеграцией сопоставления доменов Hostinger.

Используйте эту возможность, когда управляете доменами арендаторов и изолированной инфраструктурой арендаторов через hPanel от Hostinger.

## Примечания по настройке

1. Настройте основную интеграцию Hostinger в разделе **Ultimate Multisite > Settings > Host Integrations**.
2. Убедитесь, что токен API Hostinger может управлять целевым доменом или поддоменом.
3. Включите дополнение Multi-Tenancy (Мультиаренда).
4. Настройте стратегию изоляции арендаторов перед публикацией арендатора.
5. Запустите рабочий процесс проверки миграции перед отправкой производственного трафика арендатору.

Возможность Hostinger использует общий соединение Hostinger для операций на стороне хоста. DNS все равно должен указывать на правильный аккаунт Hostinger, и лимиты аккаунта hPanel по-прежнему применяются.

## Изменения, специфичные для возможностей

- Суверенные арендаторы могут создаваться с операциями над доменом, осведомленными о хосте.
- Строки имен хостов баз данных на одной машине нормализуются перед проверкой прав доступа.
- Арендаторы, управляемые Hostinger, должны использовать значение хоста базы данных, показанное в hPanel, если только среда WordPress не требует локального переопределения.
- Посещения через SSO (единый вход) зависят от того, что домен арендатора разрешается в арендатора, размещенного на Hostinger.

## Устранение неполадок с арендаторами Hostinger

- Если установка арендатора не удалась, проверьте, прикреплен ли домен уже к аккаунту Hostinger.
- Если проверка базы данных не удается, сравните имя пользователя БД арендатора, имя базы данных и привязку хоста с hPanel.
- Если **Visit (SSO)** не работает, подтвердите, что DNS и SSL активны для домена арендатора.
- Если при демонтаже остаются ресурсы хоста, удалите любые оставшиеся базы данных, пользователей или папки из hPanel после подтверждения резервных копий.
