# Настройки Gratis AI Agent

Экран **Настройки → Расширенные** (Settings → Advanced) в Gratis AI Agent предоставляет административную конфигурацию для бэкенд-интеграций, добавленных в версии v1.5.0. На этой странице описаны поля **Конечной точки обратной связи** (Feedback Endpoint) и ожидаемый формат данных.

## Доступ к настройкам

1. В админ-панели WordPress перейдите в **Gratis AI Agent → Настройки**.
2. Нажмите вкладку **Расширенные** (Advanced).

## Конфигурация конечной точки обратной связи

Конечная точка обратной связи получает POST-запросы от AI-агента каждый раз, когда пользователь отправляет отзыв с помощью кнопки "палец вниз", баннера с автоматическим запросом или команды `/report-issue`.

| Поле | Описание |
|---|---|
| **URL конечной точки обратной связи** (Feedback Endpoint URL) | URL, который принимает отзывы в виде HTTP POST-запросов с телом в формате JSON. |
| **API ключ обратной связи** (Feedback API Key) | Bearer-токен, который отправляется в заголовке `Authorization` с каждым запросом отзыва. Оставьте поле пустым, если ваша конечная точка не требует аутентификации. |

### Ожидаемый полезный груз JSON (Expected JSON Payload)

Ваша конечная точка обратной связи должна принимать тело в формате JSON, содержащее как минимум следующие поля:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Ответ был неверным относительно ценообразования.",
  "triage_category": "factual_error"
}
```

В полезный груз могут быть добавлены дополнительные поля в зависимости от контекста разговора.

### Значения `triage_category`

Слой триажа AI присваивает одно из следующих значений полю `triage_category` перед пересылкой полезного груза:

| Значение | Значение |
|---|---|
| `factual_error` | Ассистент предоставил неверную фактическую информацию. |
| `unhelpful_answer` | Ответ был технически правильным, но бесполезным. |
| `inappropriate_content` | Ответ содержал контент, который не должен быть показан пользователям. |
| `other` | Отзыв не соответствует известной категории. |

### Аутентификация

Если ваша конечная точка требует аутентификации, установите в поле **API ключ обратной связи** ваш bearer-токен. Агент отправляет:

```
Authorization: Bearer <your-api-key>
```

Если поле **API ключ обратной связи** пустое, заголовок `Authorization` не отправляется.

### Отключение сбора обратной связи

Оставьте поля **URL конечной точки обратной связи** и **API ключ обратной связи** пустыми. Кнопка "палец вниз" и интерфейс обратной связи остаются видимыми для пользователей, но отправка данных не пересылается ни в какую внешнюю службу.

## API ключ Brave Search

Также на вкладке **Расширенные** (Advanced) поле **API ключ Brave Search** (Brave Search API Key) активирует возможность [Поиска в Интернете](../configuration/internet-search).

| Поле | Описание |
|---|---|
| **API ключ Brave Search** (Brave Search API Key) | Ваш API-ключ из панели разработчика Brave Search. Требуется для включения поиска в Интернете в AI-ассистенте. |

Подпись поля включает кликабельную ссылку на страницу регистрации API Brave Search. Оставьте поле пустым, чтобы отключить поиск в Интернете.

Подробную документацию для конечных пользователей по этой функции см. в [Поиск в Интернете](../configuration/internet-search).

## Флаги функций (Feature Flags)

Также введенные в v1.9.0, вкладка **Настройки → Флаги функций** (Settings → Feature Flags) предоставляет переключатели для опциональной функциональности. Каждый флаг может быть включен или выключен для всей сети; на данный момент нет возможности переопределить настройки для конкретного сайта.

### Доступ к флагам функций

1. В админ-панели WordPress перейдите в **Gratis AI Agent → Настройки**.
2. Нажмите вкладку **Флаги функций** (Feature Flags).

### Флаги контроля доступа

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Ограничить администраторам** (Restrict to Administrators) | Выкл. | При включении только пользователи с ролью `administrator` могут открыть панель чата AI Agent. Все остальные роли увидят сообщение "Свяжитесь с вашим администратором". |
| **Ограничить администраторами сети** (Restrict to Network Admins) | Выкл. | При включении в сети multisite только Суперадмины могут использовать агент. Индивидуальные администраторы сайтов блокируются. Имеет приоритет над "Ограничить администраторам", если оба включены. |
| **Разрешить доступ подписчикам** (Allow Subscriber Access) | Выкл. | При включении пользователи с ролью `subscriber` могут использовать чат-интерфейс, но их возможности ограничены только чтением (невозможно создавать записи или изменять настройки). |
| **Отключить для не-членов** (Disable for Non-Members) | Выкл. | Интегрируется со статусом членства Ultimate Multisite. При включении чат скрывается для сайтов, у которых нет активного членства. |

### Флаги брендинга

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Скрыть подвал "Powered by Gratis AI Agent"** (Hide "Powered by Gratis AI Agent" Footer) | Выкл. | Удаляет строку указания бренда, отображаемую в нижней части виджета чата. Рекомендуется для развертывания с белым лейблом (white-label). |
| **Пользовательское имя агента** (Custom Agent Name) | *(пусто)* | Заменяет стандартное название "Gratis AI Agent" в заголовке чата и меню администратора на название вашего продукта. Оставьте пустым, чтобы использовать значение по умолчанию. |
| **Скрыть выбор агента** (Hide Agent Picker) | Выкл. | При включении пользователи не могут переключаться между пятью встроенными агентами. Текущий агент фиксируется на том, который установлен в качестве значения по умолчанию в Настройки → Общие. |
| **Использовать иконку сайта в качестве аватара чата** (Use Site Icon as Chat Avatar) | Выкл. | Заменяет стандартную иконку AI в заголовке виджета чата на иконку сайта WordPress (устанавливается в Внешний вид → Настроить → Идентичность сайта). |

### Применение изменений

Нажмите **Сохранить настройки** (Save Settings) после переключения любого флага. Изменения вступают в силу немедленно — очистка кэша или повторная активация плагина не требуются.
