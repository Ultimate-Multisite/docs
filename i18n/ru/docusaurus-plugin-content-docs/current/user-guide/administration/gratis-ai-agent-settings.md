# Настройки Gratis AI Agent

Экран **Settings → Advanced** в Gratis AI Agent предоставляет конфигурацию уровня администратора для серверных интеграций. На этой странице описаны пересылка обратной связи, ключи поставщиков поиска, настройка управляемого сервиса Superdav, элементы управления Google Calendar, настройки TextBee SMS и сетевые флаги функций.

## Доступ к настройкам {#accessing-settings}

1. В админ-панели WordPress перейдите в **Gratis AI Agent → Settings**.
2. Нажмите вкладку **Advanced**.

## Конфигурация endpoint обратной связи {#feedback-endpoint-configuration}

Endpoint обратной связи получает POST-запросы от AI-агента всякий раз, когда пользователь отправляет обратную связь через кнопку с опущенным большим пальцем, баннер автоматического запроса или команду `/report-issue`.

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** | URL, который получает отправки обратной связи в виде HTTP POST-запросов с JSON-телом. |
| **Feedback API Key** | bearer token, отправляемый в `Authorization` header каждого запроса обратной связи. Оставьте пустым, если вашему endpoint не требуется аутентификация. |

### Ожидаемая JSON-нагрузка {#expected-json-payload}

Ваш endpoint обратной связи должен принимать JSON-тело как минимум со следующими полями:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

В нагрузке могут присутствовать дополнительные поля в зависимости от контекста разговора.

### Значения `triage_category` {#triagecategory-values}

Слой AI-сортировки назначает одно из следующих значений для `triage_category` перед пересылкой нагрузки:

| Значение | Значение |
|---|---|
| `factual_error` | Ассистент предоставил неверную фактическую информацию. |
| `unhelpful_answer` | Ответ был технически корректным, но не полезным. |
| `inappropriate_content` | Ответ содержал контент, который не следует показывать пользователям. |
| `other` | Обратная связь не соответствовала известной категории. |

### Аутентификация {#authentication}

Если вашему endpoint требуется аутентификация, задайте поле **Feedback API Key** равным вашему bearer token. Агент отправляет:

```
Authorization: Bearer <your-api-key>
```

Если поле **Feedback API Key** пустое, `Authorization` header не отправляется.

### Отключение сбора обратной связи {#disabling-feedback-collection}

Оставьте поля **Feedback Endpoint URL** и **Feedback API Key** пустыми. Кнопка с опущенным большим пальцем и интерфейс обратной связи остаются видимыми для пользователей, но отправки не пересылаются ни в какой внешний сервис.

## Brave Search API Key {#brave-search-api-key}

Также на вкладке **Advanced** поле **Brave Search API Key** включает возможность [Интернет-поиск](../configuration/internet-search).

| Поле | Описание |
|---|---|
| **Brave Search API Key** | Ваш API-ключ из панели разработчика Brave Search. Требуется для включения интернет-поиска в AI-ассистенте. |

Метка поля включает кликабельную ссылку на страницу регистрации Brave Search API. Оставьте пустым, чтобы отключить интернет-поиск.

См. [Интернет-поиск](../configuration/internet-search) для документации для конечных пользователей по этой функции.

## Управляемый сервис Superdav {#managed-superdav-service}

Superdav AI Agent v1.18.0 добавляет управляемые endpoint сервиса Superdav и автоматическую подготовку подключения для поддерживаемых сайтов. Используйте эти элементы управления, когда ваш сайт должен подключаться к размещённому поставщику вместо вручную настроенного endpoint сервиса.

| Поле | Описание |
|---|---|
| **Managed Superdav Service** | Включает подключение к размещённому сервису Superdav для поддерживаемых сайтов. |
| **Provision Connection** | Запускает автоматическую подготовку endpoint и учётных данных. Используйте это после подтверждения, что сайт должен использовать управляемого поставщика. |
| **Service Endpoint / Connection Status** | Показывает текущий endpoint или состояние подключения после подготовки. |

После подготовки сохраните настройки и проверьте статус подключения, прежде чем полагаться на рабочие процессы управляемого сервиса. Если подготовка не удалась, просмотрите отображаемые рекомендации по повторной попытке и подтвердите, что у сайта есть разрешение использовать размещённого поставщика.

## Конфигурация Google Calendar {#google-calendar-configuration}

Когда функции календаря Superdav AI Agent v1.18.0 включены, агент может читать настроенные календари и сведения о событиях. Инструменты календаря ориентированы на чтение и полезны для напоминаний с учётом расписания, последующего взаимодействия с участниками и сопоставления контактов.

| Поле | Описание |
|---|---|
| **Google Calendar Credentials** | Хранит учётные данные или подключение токена, необходимое для чтения данных календаря. |
| **Calendar Selection** | Ограничивает, какие настроенные календари агент может просматривать. |
| **Calendar Connection Status** | Подтверждает, могут ли текущие учётные данные читать календари и события. |

Ограничьте учётные данные календаря только теми календарями, которые нужны агенту. Повторно подключите или замените учётные данные, если статус указывает на истёкший токен.

## TextBee SMS-уведомления {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 добавляет TextBee как SMS-поставщика для настроенных рабочих процессов уведомлений. SMS-уведомления следует сочетать с контрольными точками человеческого подтверждения для конфиденциальных или предназначенных для пользователей сообщений.

| Поле | Описание |
|---|---|
| **TextBee API Key** | Аутентифицирует запросы к SMS-поставщику TextBee. |
| **TextBee Device / Sender** | Выбирает отправителя или устройство TextBee, используемое для исходящих сообщений, когда это требуется поставщиком. |
| **SMS Notifications Enabled** | Позволяет утверждённым рабочим процессам отправлять уведомления текстовыми сообщениями. Оставьте отключённым, чтобы предотвратить отправку SMS. |

Отправьте тестовое сообщение только на номер, принадлежащий администратору, затем подтвердите поведение контрольной точки утверждения перед включением запланированных напоминаний или напоминаний для участников.

## Флаги функций {#feature-flags}

Также представленная в v1.9.0 вкладка **Settings → Feature Flags** предоставляет переключатели для необязательной функциональности. Каждый флаг либо включён, либо отключён на уровне всей сети; на данный момент переопределения для отдельного сайта нет.

### Доступ к флагам функций {#accessing-feature-flags}

1. В админ-панели WordPress перейдите в **Gratis AI Agent → Settings**.
2. Нажмите вкладку **Feature Flags**.

### Флаги контроля доступа {#access-control-flags}

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Restrict to Administrators** | Off | Если включено, только пользователи с ролью `administrator` могут открыть панель чата AI Agent. Все остальные роли вместо этого видят сообщение «Свяжитесь с вашим администратором». |
| **Restrict to Network Admins** | Off | Если включено в мультисайтовой сети, только Super Admins могут использовать агента. Администраторы отдельных сайтов блокируются. Имеет приоритет над «Restrict to Administrators», если включены оба параметра. |
| **Allow Subscriber Access** | Off | Если включено, пользователи с ролью `subscriber` могут использовать интерфейс чата, но ограничены возможностями только для чтения (без создания записей или изменения настроек). |
| **Disable for Non-Members** | Off | Интегрируется со статусом членства Ultimate Multisite. Если включено, чат скрыт для сайтов, у которых нет активного членства. |

### Флаги брендинга {#branding-flags}

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | Удаляет строку атрибуции бренда, отображаемую внизу виджета чата. Рекомендуется для white-label-развертываний. |
| **Custom Agent Name** | *(пусто)* | Заменяет стандартную метку «Gratis AI Agent» в заголовке чата и меню администратора на название вашего продукта. Оставьте пустым, чтобы использовать значение по умолчанию. |
| **Hide Agent Picker** | Off | Если включено, пользователи не могут переключаться между пятью встроенными агентами. Текущий агент фиксируется тем, который настроен как используемый по умолчанию в Settings → General. |
| **Use Site Icon as Chat Avatar** | Off | Заменяет стандартный значок AI в заголовке виджета чата на значок сайта WordPress (задается в Appearance → Customize → Site Identity). |

### Флаги безопасности автоматизации {#automation-safety-flags}

Superdav AI Agent v1.18.0 вводит шлюзы одобрения человеком и записи напоминаний для более безопасных запусков автоматизации. Эти элементы управления могут отображаться во флагах функций или расширенных настройках автоматизации, в зависимости от установленного пакета.

| Флаг | По умолчанию | Описание |
|---|---|---|
| **Require Human Approval** | Рекомендуется включить | Приостанавливает чувствительные автоматизации до тех пор, пока авторизованный пользователь не просмотрит и не подтвердит предлагаемое действие. |
| **Reminder Deduplication** | On | Записывает отправленные напоминания, чтобы повторные попытки или запланированные запуски не отправляли дублирующиеся уведомления. |
| **Enable Calendar Tools** | Off до настройки | Позволяет агенту читать настроенные Google-календари и события. |
| **Enable SMS Notifications** | Off до настройки | Позволяет одобренным рабочим процессам отправлять TextBee SMS-уведомления после сохранения учетных данных. |

### Применение изменений {#applying-changes}

Нажмите **Save Settings** после переключения любого флага. Изменения вступают в силу немедленно — очистка кэша или повторная активация plugin не требуется.
