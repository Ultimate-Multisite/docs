# Экспорт сайта {#site-export}

Страница администратора **Экспорт сайта** позволяет администраторам сети упаковать один сайт или всю сеть в загружаемый архив для миграции, резервного копирования или передачи другому лицу.

## Экспорт одного сайта {#exporting-one-site}

Перейдите в **Ultimate Multisite > Экспорт сайта** и выберите **Сгенерировать новый экспорт сайта**. Выберите дочерний сайт, который вы хотите экспортировать, а затем укажите, должен ли архив включать загруженные файлы, плагины и темы.

После завершения экспорта загрузите ZIP-архив из списка **Существующие экспорты**. Теперь ZIP-архивы экспорта включают самозапускающийся `index.php` и `readme.txt`, поэтому архив можно загрузить на новый хостинг и запустить, не устанавливая предварительно отдельный плагин импортера.

## Экспорт всей сети {#exporting-the-whole-network}

Используйте **Экспорт сети** на странице «Экспорт сайта», когда вам нужен единый архив, содержащий все дочерние сайты в сети. Это полезно перед миграцией хостинга, учебными тревогами по восстановлению после катастрофы или перестройкой тестовой среды, когда каждый дочерний сайт должен перемещаться вместе.

Поскольку экспорт всей сети может быть намного больше, чем экспорт одного сайта, выполняйте его в период низкой нагрузки и убедитесь, что целевое хранилище имеет достаточно свободного места для загруженных файлов, плагинов, тем и сгенерированных ZIP-архивов.

### Пакеты импорта сети {#network-import-bundles}

Начиная с Ultimate Multisite 2.12.0, Site Exporter может генерировать **пакеты импорта сети** — специализированные архивы, предназначенные для оптимизированного восстановления всей сети. Пакет импорта сети содержит все необходимые файлы и метаданные для восстановления нескольких сайтов на новой сетевой установке.

#### Генерация пакета импорта сети {#generating-a-network-import-bundle}

1. Перейдите в **Ultimate Multisite > Экспорт сайта**
2. Нажмите **Сгенерировать новый экспорт сети**
3. Выберите **Пакет импорта сети** в качестве типа экспорта
4. Укажите, нужно ли включать загруженные файлы, плагины и темы
5. Нажмите **Сгенерировать**
6. Загрузите ZIP-архив пакета из списка **Существующие экспорты**

#### Восстановление из пакета импорта сети {#restoring-from-a-network-import-bundle}

Чтобы восстановить сайты из пакета импорта сети:

1. Установите Ultimate Multisite на целевом хостинге
2. Завершите мастер настройки мультисайта
3. Перейдите в **Ultimate Multisite > Экспорт сайта** на новой сети
4. Нажмите **Импортировать пакет сети**
5. Загрузите ZIP-архив пакета импорта сети
6. Следуйте инструкциям по импорту на экране
7. Процесс импорта восстановит все сайты, их контент и настройки

Пакеты импорта сети сохраняют:
- Контент сайта (записи, страницы, пользовательские типы записей)
- Настройки и опции сайта
- Роли и права пользователей
- Плагины и темы (если включены в пакет)
- Загруженные медиафайлы (если включены в пакет)
- Пользовательские таблицы и данные базы данных

## Установка самозапускающегося ZIP-архива экспорта {#installing-a-self-booting-export-zip}

Чтобы восстановить самозапускающийся ZIP-архив на новом хостинге:

1. Загрузите содержимое экспортированного ZIP-архива в корневую папку сайта.
2. Откройте загруженный `index.php` в браузере.
3. Следуйте инструкциям установщика на экране из пакета экспорта.
4. Ознакомьтесь с файлом `readme.txt` в пакете на предмет заметок, специфичных для экспорта, прежде чем удалять временные файлы.

Подробную информацию об установке и импорте для конкретных аддонов см. в [документации аддона Site Exporter](/addons/site-exporter).

Для инструмента для одного сайта, добавленного в Ultimate Multisite 2.9.0, см. [Экспорт и импорт](/user-guide/administration/export-import).
