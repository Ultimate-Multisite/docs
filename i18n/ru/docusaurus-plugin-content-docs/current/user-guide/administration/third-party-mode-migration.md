# Миграция в режим сторонних плагинов

Superdav AI Agent v1.12.0 меняет подход к работе со сторонними возможностями (abilities). **Режим сторонних плагинов теперь по умолчанию установлен в «автоматический»**, что позволяет интегрировать нативные возможности WordPress Abilities API в WordPress 7.0+ без ручной настройки.

## Что изменилось? {#what-changed}

### До v1.12.0 {#before-v1120}

Для работы со сторонними возможностями требовалась ручная настройка:

- Необходимо было явно включить «режим сторонних плагинов»
- Возможности загружались из пользовательского реестра
- Интеграция с WordPress Abilities API была необязательной
- По умолчанию использовался устаревший режим (Legacy mode)

### После v1.12.0 {#after-v1120}

Сторонние возможности работают автоматически:

- Режим сторонних плагинов по умолчанию установлен в «автоматический»
- Возможности интегрируются нативно с WordPress Abilities API
- На WordPress 7.0+ не требуется ручная настройка
- Устаревший режим (Legacy mode) по-прежнему доступен для более старых версий WordPress

## Кого это касается? {#who-is-affected}

### Новые установки (WordPress 7.0+) {#new-installations-wordpress-70}

**Действия не требуются.** Режим сторонних плагинов автоматически установлен в «автоматический», и возможности работают сразу из коробки.

### Существующие установки {#existing-installations}

**Ваши настройки сохранены.** Если вы использовали:

- **Устаревший режим (Legacy mode)**: Вы остаетесь в устаревшем режиме (изменений нет)
- **Ручной режим сторонних плагинов (Manual third-party mode)**: Вы остаетесь в ручном режиме (изменений нет)
- **Автоматический режим (Auto mode)**: Вы продолжаете работать в автоматическом режиме (изменений нет)

### Версии WordPress до 7.0 {#wordpress-versions-before-70}

**Устаревший режим по-прежнему доступен.** Если вы используете WordPress 6.x или более раннюю версию:

- Режим сторонних плагинов по умолчанию установлен в «устаревший» (legacy)
- Вы можете вручную включить режим сторонних плагинов, если это необходимо
- Обновитесь до WordPress 7.0+ для использования нативного Abilities API

## Понимание режимов {#understanding-the-modes}

### Автоматический режим (Auto Mode) (Новое значение по умолчанию) {#auto-mode-new-default}

**Автоматический режим** использует нативную интеграцию с WordPress Abilities API:

- Возможности регистрируются через хуки WordPress
- Полная совместимость с Abilities API WordPress 7.0+
- Автоматическое обнаружение сторонних возможностей
- Не требуется ручная настройка

**Когда использовать**: WordPress 7.0+ со сторонними возможностями

### Ручной режим (Manual Mode) {#manual-mode}

**Ручной режим** требует явной настройки:

- Вы указываете, какие сторонние возможности нужно загрузить
- Полезно для тестирования или выборочной загрузки возможностей
- Требует редактирования файлов конфигурации
- Больше контроля, но и больше настроек

**Когда использовать**: Тестирование, выборочная загрузка возможностей или пользовательские конфигурации

### Устаревший режим (Legacy Mode) {#legacy-mode}

**Устаревший режим** использует старую систему сторонних возможностей:

- Пользовательский реестр возможностей (не WordPress Abilities API)
- Обратная совместимость со старыми версиями WordPress
- Нет нативной интеграции с WordPress
- Устаревший, но все еще поддерживаемый

**Когда использовать**: WordPress 6.x или более ранние версии, или когда вам нужна обратная совместимость

## Проверка текущего режима {#checking-your-current-mode}

### Через панель администратора {#via-admin-panel}

1. Перейдите в **Админ-панель WordPress** → **Superdav AI Agent** → **Настройки**
2. Найдите настройку **Режим сторонних плагинов (Third-Party Mode)**
3. Вы увидите текущий режим и опции для его изменения

### Через код {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', или 'legacy'
```

## Изменение режима {#changing-your-mode}

### Переключение на Автоматический режим (Auto Mode) {#switch-to-auto-mode}

Если вы используете WordPress 7.0+ и хотите использовать автоматический режим:

1. Перейдите в **Superdav AI Agent** → **Настройки**
2. Найдите **Режим сторонних плагинов (Third-Party Mode)**
3. Выберите **Авто (WordPress Abilities API)**
4. Нажмите **Сохранить**

Superdav AI Agent автоматически обнаружит и зарегистрирует сторонние возможности.

### Переключение на Ручной режим (Manual Mode) {#switch-to-manual-mode}

Если вы хотите вручную контролировать, какие возможности загружаются:

1. Перейдите в **Superdav AI Agent** → **Настройки**
2. Найдите **Режим сторонних плагинов (Third-Party Mode)**
3. Выберите **Ручной (Manual)**
4. Нажмите **Сохранить**
5. Отредактируйте свой файл конфигурации, чтобы указать, какие возможности нужно загрузить

### Переключение на Устаревший режим (Legacy Mode) {#switch-to-legacy-mode}

Если вам нужна обратная совместимость:

1. Перейдите в **Superdav AI Agent** → **Настройки**
2. Найдите **Режим сторонних плагинов (Third-Party Mode)**
3. Выберите **Устаревший (Legacy)**
4. Нажмите **Сохранить**

## Преимущества Автоматического режима (Auto Mode) {#benefits-of-auto-mode}

### Автоматическое обнаружение {#automatic-discovery}

Возможности автоматически обнаруживаются из:

- Установленных плагинов
- Активной темы
- Плагинов must-use
- Плагинов drop-in

Не требуется ручная регистрация.

### Нативная интеграция {#native-integration}

Возможности интегрируются с WordPress Abilities API:

- Соответствует ядру WordPress
- Работает в админ-панели WordPress
- Совместимо с другими плагинами, использующими Abilities API
- Обеспечивает защиту от устаревания по мере развития WordPress

### Упрощенное управление {#simplified-management}

- Не нужно редактировать файлы конфигурации
- Не требуется ручная регистрация возможностей
- Управление видимостью возможностей работает автоматически
- Уведомления в админ-панели оповещают вас о неклассифицированных возможностях

### Лучшая производительность {#better-performance}

- Возможности кэшируются
- Загружаются по требованию (lazy-loaded)
- Оптимизировано для WordPress 7.0+

## Путь миграции {#migration-path}

### Если вы используете WordPress 6.x {#if-youre-on-wordpress-6x}

1. **Обновитесь до WordPress 7.0+** (когда будете готовы)
2. **Обновите Superdav AI Agent** до v1.12.0+
3. **Измените режим сторонних плагинов на Автоматический** (необязательно; устаревший режим по-прежнему работает)
4. **Проверьте видимость возможностей**, чтобы обеспечить правильный контроль доступа

### Если вы используете WordPress 7.0+ {#if-youre-on-wordpress-70}

1. **Обновите Superdav AI Agent** до v1.12.0+
2. **Убедитесь, что режим сторонних плагинов установлен в «Автоматический»** (по умолчанию он должен быть таким)
3. **Проверьте видимость возможностей**, чтобы обеспечить правильный контроль доступа
4. **Протестируйте сторонние возможности**, чтобы убедиться, что они работают

## Устранение неполадок {#troubleshooting}

### Возможности не загружаются в автоматическом режиме {#abilities-arent-loading-in-auto-mode}

- Убедитесь, что вы используете WordPress 7.0+
- Проверьте, что режим сторонних плагинов установлен в «Автоматический»
- Убедитесь, что плагин, предоставляющий возможность, активен
- Проверьте журналы ошибок WordPress на наличие ошибок регистрации

### Я хочу сохранить устаревший режим (Legacy mode) {#i-want-to-keep-legacy-mode}

- Перейдите в **Настройки** → **Режим сторонних плагинов (Third-Party Mode)**
- Выберите **Устаревший (Legacy)**
- Нажмите **Сохранить**
- Устаревший режим продолжит работать

### Мои пользовательские возможности не отображаются {#my-custom-abilities-arent-showing}

- Убедитесь, что они зарегистрированы через хуки WordPress
- Проверьте, что они правильно реализуют Abilities API
- Просмотрите журналы ошибок WordPress
- Используйте страницу админ-панели **Видимость возможностей (Ability Visibility)**, чтобы увидеть все зарегистрированные возможности

### Я получаю уведомления о «неклассифицированной возможности» {#im-getting-unclassified-ability-notices}

- Это нормально для новых сторонних возможностей
- Просмотрите и классифицируйте их в уведомлении админ-панели
- Посмотрите **Видимость возможностей (Ability Visibility)**, чтобы узнать подробности классификации

## Обратная совместимость {#backward-compatibility}

### Существующие конфигурации {#existing-configurations}

Если у вас есть существующие конфигурации сторонних возможностей:

- **Устаревший режим (Legacy mode)**: Ваша конфигурация продолжает работать
- **Ручной режим (Manual mode)**: Ваша конфигурация продолжает работать
- **Автоматический режим (Auto mode)**: Ваша конфигурация игнорируется (автоматический режим берет верх)

Чтобы сохранить вашу пользовательскую конфигурацию, оставайтесь в ручном или устаревшем режиме.

### График устаревания {#deprecation-timeline}

- **v1.12.0**: Устаревший и ручной режимы по-прежнему полностью поддерживаются
- **v1.13.0+**: В устаревшем режиме могут появиться уведомления об устаревании
- **v2.0.0**: Устаревший режим может быть удален (ТСО)

## Лучшие практики {#best-practices}

### Для новых установок {#for-new-installations}

- Используйте автоматический режим (это значение по умолчанию)
- Позвольте Superdav AI Agent автоматически обнаруживать возможности
- Используйте Видимость возможностей для контроля доступа

### Для существующих установок {#for-existing-installations}

- Обновитесь до WordPress 7.0+, когда это возможно
- Переключитесь на автоматический режим для упрощения управления
- Просмотрите и классифицируйте возможности с помощью Видимости возможностей

### Для пользовательских возможностей {#for-custom-abilities}

- Регистрируйте возможности через хуки WordPress (Abilities API)
- Избегайте пользовательских реестров возможностей
- Тестируйте на WordPress 7.0+ в автоматическом режиме

## Следующие шаги {#next-steps}

1. **Проверьте версию WordPress**: Убедитесь, что вы используете 7.0+ для автоматического режима
2. **Проверьте режим сторонних плагинов**: Перейдите в Настройки и проверьте текущий режим
3. **Обновите при необходимости**: Переключитесь на автоматический режим, если вы используете WordPress 7.0+
4. **Классифицируйте возможности**: Просмотрите и классифицируйте любые неклассифицированные возможности
5. **Протестируйте**: Убедитесь, что ваши сторонние возможности работают правильно
