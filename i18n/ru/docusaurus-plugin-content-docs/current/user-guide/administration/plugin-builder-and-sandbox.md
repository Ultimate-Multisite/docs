# Plugin Builder & Sandbox

Gratis AI Agent v1.5.0 представляет **Plugin Builder & Sandbox System** — систему, которая позволяет AI-помощнику генерировать, активировать и управлять плагинами WordPress на вашей сети через безопасную, изолированную среду sandbox.

## Обзор {#overview}

Plugin Builder позволяет AI-помощнику писать кастомные плагины WordPress в ответ на запросы на естественном языке. Сгенерированные плагины проходят проверку, сохраняются и активируются внутри слоя sandbox, прежде чем смогут повлиять на функциональность рабочего сайта.

Сценарии использования включают:

- Генерацию легковесных утилитарных плагинов без привлечения разработчиков.
- Прототипирование функций, требующих использования хуков WordPress или кастомных типов записей.
- Создание краткосрочных скриптов автоматизации для пакетных операций.

## Генерация плагина с помощью AI {#generating-a-plugin-via-ai}

Чтобы сгенерировать плагин, откройте чат-интерфейс Gratis AI Agent и опишите, что вам нужно. Например:

> "Создай плагин, который добавит кастомное уведомление администратора на дашборд."

AI выполнит следующие действия:

1. Сгенерирует PHP-код плагина с использованием структурированной генерации кода.
2. Проверит вывод на наличие синтаксических ошибок и небезопасных шаблонов.
3. Сохранит сгенерированный плагин в хранилище sandbox.
4. Вернет подтверждение со слагом плагина и кнопкой **Activate in Sandbox** (Активировать в Sandbox).

Вы можете уточнить результат, задав дополнительные вопросы в той же ветке разговора, прежде чем активировать плагин.

## Активация в Sandbox {#sandbox-activation}

Активация сгенерированного плагина в sandbox отличается от его активации на рабочей сети. Sandbox:

- Запускает плагин в изолированной среде WordPress (wp-env).
- Перехватывает любые ошибки PHP, предупреждения или конфликты хуков.
- Сообщает о результате активации прямо в чат-интерфейсе.

Чтобы активировать плагин в sandbox, нажмите кнопку **Activate in Sandbox** в ответе AI в чате или используйте команду слэш:

```
/activate-plugin <plugin-slug>
```

Сообщение о статусе подтвердит, удалось ли активировать плагин или нет. В случае сбоя журнал ошибок будет отображен в ветке чата.

## Управление сгенерированными плагинами {#managing-generated-plugins}

Сгенерированные плагины перечислены в разделе **Gratis AI Agent → Plugin Builder → Manage Plugins**. На этом экране вы можете:

| Действие | Описание |
|---|---|
| **View source** | Просмотреть полный PHP-код плагина. |
| **Re-activate in sandbox** | Повторно запустить проверку активации в sandbox. |
| **Install on network** | Развернуть плагин на рабочую сеть (требуется ручное подтверждение). |
| **Update** | Предоставить новую версию через AI, которая заменит существующий код. |
| **Delete** | Удалить плагин из хранилища sandbox. Сначала деактивирует его на всех сайтах. |

:::warning
**Install on network** развертывает сгенерированный плагин на вашу рабочую мультисайт-сеть WordPress. Обязательно просмотрите код плагина перед продолжением. Gratis AI Agent запросит подтверждение перед завершением живой установки.
:::

## Установка сгенерированного плагина на сеть {#installing-a-generated-plugin-on-the-network}

Когда вы удовлетворены плагином, протестированным в sandbox, вы можете установить его на рабочую сеть:

1. Перейдите в **Gratis AI Agent → Plugin Builder → Manage Plugins**.
2. Нажмите **Install on Network** рядом с плагином, который вы хотите развернуть.
3. Подтвердите диалоговое окно. Плагин будет установлен в `wp-content/plugins/` и активирован на уровне сети.

Альтернативно, используйте команду слэш в чат-интерфейсе:

```
/install-plugin <plugin-slug>
```

## Обновление плагинов {#plugin-updates}

Чтобы обновить сгенерированный плагин, опишите изменения AI-помощнику в новом разговоре:

> "Обнови плагин dashboard-notice так, чтобы уведомление отображалось только администраторам."

AI сгенерирует новую версию, которая появится в sandbox рядом с текущей. Вы просмотрите различия (diff) и подтвердите, прежде чем обновление будет применено.

## Интеграция HookScanner {#hookscanner-integration}

Plugin Builder использует встроенный **HookScanner** для анализа хуков и фильтров, зарегистрированных каждым сгенерированным плагином. Результаты HookScanner отображаются в ответе чата и включают:

- Зарегистрированные action hooks (вызовы `add_action`).
- Зарегистрированные filter hooks (вызовы `add_filter`).
- Любые хуки, найденные в зависимостях плагина (пропускаются каталоги `vendor/` и `node_modules/`).

Это помогает понять поведение плагина до его активации.

## Соображения безопасности {#security-considerations}

- Сгенерированные плагины хранятся отдельно от плагинов, установленных вручную, и не доступны через стандартный экран управления плагинами WordPress, пока вы явно не установите их на сеть.
- Sandbox использует проверку путей (path validation), чтобы предотвратить обход каталогов при записи файлов плагинов.
- Плагины с опасными вызовами функций (например, `eval`, `exec`, `system`) помечаются во время проверки и не будут активированы.
