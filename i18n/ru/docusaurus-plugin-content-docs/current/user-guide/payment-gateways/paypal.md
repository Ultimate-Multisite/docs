# Настройка шлюза PayPal (v2)

_**ВАЖНОЕ ПРИМЕЧАНИЕ: эта статья относится к Ultimate Multisite версии 2.x.**_

Вы можете активировать до четырех способов оплаты на нашей странице настроек оплаты: Stripe, Stripe Checkout, PayPal и Manual. В этой статье мы рассмотрим, как выполнить интеграцию с **PayPal**.

Как и Stripe, PayPal широко используется для онлайн-платежей, особенно на сайтах WordPress. Эта статья поможет вам использовать PayPal как способ оплаты, доступный в вашей сети.

Обратите внимание, что для получения учетных данных API, необходимых для этой интеграции, вам нужно иметь **бизнес-аккаунт PayPal**.

## Включение PayPal в вашей сети

Чтобы включить PayPal как доступный способ оплаты в вашей сети, перейдите на вкладку **Ultimate Multisite > Settings > Payments** и установите флажок рядом с PayPal.

![Включение PayPal в активных платежных шлюзах](/img/config/settings-payment-gateways.png)

## Использование мастера настройки

Ultimate Multisite 2.10.0 добавляет мастер настройки PayPal в настройки платежного шлюза. После включения PayPal используйте мастер в **Ultimate Multisite > Settings > Payments**, чтобы выбрать, как вы хотите подключить шлюз, и подтвердить, какие учетные данные всё еще нужны перед сохранением.

Мастер поддерживает два пути настройки:

* **Ручной ввод учетных данных**: используйте этот путь, если у вас уже есть учетные данные PayPal API, если настройка OAuth недоступна для вашего аккаунта или если вы предпочитаете самостоятельно скопировать учетные данные из PayPal. Введите API Username, API Password и API Signature в поля PayPal, затем сохраните настройки оплаты.
* **Шлюз подключения OAuth**: используйте этот путь только тогда, когда опция OAuth доступна и включена для вашей установки. Мастер показывает поток OAuth за флагом функции, поэтому сети без этого флага продолжают использовать поля ручного ввода учетных данных.

Если вы не видите опцию OAuth в мастере, выполните описанный ниже процесс ручного ввода учетных данных. Шлюз работает с теми же учетными данными PayPal Business API, что и предыдущие выпуски Ultimate Multisite 2.x.

## Получение учетных данных PayPal API

После включения PayPal в качестве платежного шлюза вам нужно будет заполнить поля для PayPal API **Username** , PayPal API **Password** и PayPal API **Signature**.

Вы можете получить их, войдя в свой аккаунт PayPal [Live](https://www.paypal.com/home) или [Sandbox](https://www.sandbox.paypal.com/home).

(Помните, что вы можете использовать **режим песочницы** для тестирования платежей и проверки правильности настройки шлюза. Просто включите соответствующий раздел.)

![Поля учетных данных PayPal API и переключатель режима песочницы](/img/config/settings-payment-gateways.png)

Чтобы запросить учетные данные API Signature или Certificate для вашего аккаунта PayPal:

  1. Перейдите в [настройки аккаунта](https://www.paypal.com/businessmanage/account/accountAccess).

  2. В разделе **доступ API** нажмите **Обновить**.
![Настройки аккаунта PayPal с разделом доступа API](/img/config/settings-payment-gateways.png)

  3. В разделе **Интеграция NVP/SOAP API (Classic)** нажмите **Управлять учетными данными API**.
![Интеграция PayPal NVP/SOAP API Управлять учетными данными API](/img/config/settings-payment-gateways.png)

     * Если вы уже сгенерировали API Signature или Certificate, вы будете перенаправлены на страницу, где сможете найти свои учетные данные.

     * _**Примечание:** если вам предложат подтвердить ваш аккаунт PayPal, следуйте инструкциям на экране._

  4. Выберите _один_ из следующих вариантов, затем нажмите **Согласиться и отправить**.

     * **Запросить API Signature** – выберите для аутентификации API Signature.

     * **Запросить API Certificate** – выберите для аутентификации API Certificate.

  5. PayPal генерирует ваши учетные данные API следующим образом:
![Сгенерированные PayPal учетные данные API](/img/config/settings-payment-gateways.png)

     * **Учетные данные API Signature** включают API Username, API Password и Signature, срок действия которых не истекает. Эти значения по умолчанию скрыты для дополнительной безопасности. Нажмите **Показать/скрыть**, чтобы включать и отключать их отображение. По завершении нажмите **Готово**.

     * **Учетные данные API Certificate** включают API Username, API Password и Certificate, срок действия которого автоматически истекает через три года. Нажмите **Скачать Certificate**, чтобы сохранить API Certificate на свой рабочий стол.

Готово, ваша интеграция платежей PayPal завершена!

Если у вас есть вопросы по настройкам PayPal, вы можете обратиться в [Справочный центр](https://www.paypal.com/br/smarthelp/home) PayPal.
