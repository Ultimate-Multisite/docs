# Продажа доменов: Настройка и конфигурация провайдеров

Addon Domain Seller поставляется с пошаговым мастером настройки, который проведет вас через каждый необходимый шаг. На этой странице мы расскажем о процессе работы мастера и о том, как настроить или перенастроить провайдеров после первоначальной установки.

## Требования

- **Ultimate Multisite** v2.4.12 или выше, активированный на уровне сети (network-activated)
- **PHP** 7.4+
- Учетные данные API для как минимум одного поддерживаемого регистратора

## Мастер настройки при первом запуске

Мастер настройки запускается автоматически при первой активации плагина на уровне сети. Вы также можете запустить его в любое время в разделе **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Шаг 1 — Выбор провайдера

Выберите регистратора, с которым вы хотите подключиться. Поддерживаемые опции:

| Провайдер | Управление DNS | Конфиденциальность WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Нет | Да (WhoisGuard, бесплатно) |
| GoDaddy | Нет | Нет |
| ResellerClub | Да | Нет |
| NameSilo | Нет | Нет |
| Enom | Да | Нет |

### Шаг 2 — Ввод учетных данных

У каждого провайдера свои поля для ввода учетных данных:

**OpenSRS** — Имя пользователя и приватный ключ (из OpenSRS Reseller Control Panel)

**Namecheap** — Имя пользователя и API ключ (из Account › Tools › API Access)

**GoDaddy** — API ключ и секрет (из developer.godaddy.com)

**ResellerClub** — Reseller ID и API ключ (из панели управления ResellerClub)

**NameSilo** — API ключ (из namesilo.com › Account › API Manager)

**Enom** — Account ID и API токен

Если есть такая возможность, проверьте **Sandbox mode**, чтобы протестировать подключение в тестовой среде провайдера перед запуском в рабочем режиме.

### Шаг 3 — Тестирование подключения

Нажмите **Test Connection**. Мастер отправляет легкий API-запрос для проверки учетных данных и возможности подключения. Обязательно исправьте любые проблемы с учетными данными, прежде чем продолжить.

### Шаг 4 — Импорт TLD

Нажмите **Import TLDs**, чтобы загрузить все доступные TLD и оптовые цены от подключенного провайдера. Это заполняет список TLD, который используется продуктами доменов. Импорт может занять от 30 до 60 секунд для провайдеров с большим каталогом TLD.

TLD также автоматически синхронизируются один раз в день с помощью запланированной задачи cron.

### Шаг 5 — Создание продукта домена

Мастер создает по умолчанию универсальный продукт домена с наценкой в 10%. Вы можете отредактировать этот продукт сразу или пропустить этот шаг и создать продукты вручную в разделе **Ultimate Multisite › Products**.

См. [Domain Products and Pricing](./domain-products) для полного руководства по настройке продуктов.

## Перенастройка провайдера

Перейдите в **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или нажмите **Settings** в списке плагинов).

На странице настроек вы найдете:

- **Enable domain selling** — переключатель для включения/выключения всей функции
- **Default provider** — провайдер, используемый для поиска доменов и создания новых продуктов
- **Max TLDs per search** — сколько TLD проверять при поиске клиента; более высокие значения показывают больше вариантов, но работают медленнее
- **Availability cache duration** — как долго кэшировать результаты проверки доступности и цены; более низкие значения точнее, но увеличивают количество API-запросов
- **Manage domain products** — быстрая ссылка на список продуктов
- **Configure providers** — открывает мастер интеграции для добавления или перенастройки провайдеров

### Добавление второго провайдера

Нажмите **Configure providers** и снова запустите мастер для нового регистратора. Вы можете настроить несколько провайдеров одновременно. Назначьте каждый продукт домена конкретному провайдеру или оставьте его на настройку по умолчанию.

### Ручная синхронизация TLD

На странице настроек нажмите **Sync TLDs** рядом с любым настроенным провайдером, чтобы загрузить последние цены. Это полезно, если провайдер обновил оптовые цены или добавил новые TLD.

## Логи

Каждый провайдер записывает данные в свой собственный канал логов. Просмотреть логи можно в разделе **Network Admin › Ultimate Multisite › Logs**:

| Лог-канал | Содержимое |
|---|---|
| `domain-seller-registration` | Все попытки регистрации (успешные и неудачные) |
| `domain-seller-renewal` | Результаты задачи по продлению |
| `domain-seller-opensrs` | Сырая активность API OpenSRS |
| `domain-seller-namecheap` | Сырая активность API Namecheap |
| `domain-seller-godaddy` | Сырая активность API GoDaddy |
| `domain-seller-resellerclub` | Сырая активность API ResellerClub |
| `domain-seller-namesilo` | Сырая активность API NameSilo |
| `domain-seller-enom` | Сырая активность API Enom |
