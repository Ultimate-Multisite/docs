# Domain Seller: настройка и конфигурация провайдера

Аддон Domain Seller поставляется с пошаговым мастером настройки, который проводит вас через каждый необходимый шаг. Эта страница описывает процесс работы мастера и то, как затем настроить или перенастроить провайдеров.

## Требования

- **Multisite Ultimate** v2.4.12 или выше, активированная в сети
- **PHP** 7.4+
- Учетные данные API как минимум для одного поддерживаемого регистратора

## Мастер настройки при первом запуске

Мастер настройки запускается автоматически при первой сетевой активации плагина. Он также доступен в любое время из **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Шаг 1 — выберите провайдера

Выберите регистратора, которого хотите подключить. Поддерживаемые варианты:

| Провайдер | Управление DNS | Конфиденциальность WHOIS |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Нет | Да (WhoisGuard, бесплатно) |
| HostAfrica | Да | Да (защита ID) |
| Openprovider | Да | Да |
| Hostinger | Через базовое сопоставление доменов Hostinger для размещенных доменов | Да |
| GoDaddy | Нет | Нет |
| ResellerClub | Да | Нет |
| NameSilo | Нет | Нет |
| Enom | Да | Нет |

### Шаг 2 — введите учетные данные

У каждого провайдера разные поля учетных данных:

**OpenSRS** — имя пользователя и приватный ключ (из OpenSRS Reseller Control Panel)

**Namecheap** — имя пользователя и API-ключ (из Account › Tools › API Access)

**HostAfrica** — endpoint Domains Reseller API и учетные данные из реселлерского модуля HostAfrica. Отдельный sandbox endpoint в настоящее время не задокументирован; перед запуском реальных регистраций протестируйте с помощью безопасных проверок только для чтения.

**Openprovider** — имя пользователя и пароль с включенным доступом к API. Необязательный sandbox mode использует Openprovider sandbox API, а необязательный стандартный customer handle можно повторно использовать для регистраций.

**Hostinger** — общий токен Hostinger hPanel API из базовой интеграции Hostinger. Тот же токен обеспечивает базовое сопоставление доменов и операции регистрации Domain Seller.

**GoDaddy** — API-ключ и секрет (с developer.godaddy.com)

**ResellerClub** — ID реселлера и API-ключ (из панели управления ResellerClub)

**NameSilo** — API-ключ (из namesilo.com › Account › API Manager)

**Enom** — Account ID и API-токен

Отметьте **Sandbox mode**, где он доступен, чтобы протестировать работу с тестовой средой провайдера перед выходом в рабочий режим.

### Шаг 3 — протестируйте подключение

Нажмите **Test Connection**. Мастер отправляет легкий API-вызов для проверки учетных данных и подключения. Исправьте любые проблемы с учетными данными, прежде чем продолжить.

### Шаг 4 — импортируйте TLD

Нажмите **Import TLDs**, чтобы получить все доступные TLD и оптовые цены от подключенного провайдера. Это заполняет список TLD, используемый доменными продуктами. Импорт может занять 30–60 секунд у провайдеров с большими каталогами TLD.

TLD также автоматически повторно синхронизируются один раз в день через запланированную cron-задачу.

### Шаг 5 — создайте доменный продукт

Мастер создает стандартный универсальный доменный продукт с наценкой 10%. Вы можете сразу отредактировать этот продукт или пропустить шаг и создать продукты вручную в **Ultimate Multisite › Products**.

См. [Доменные продукты и ценообразование](./domain-products) для полного руководства по конфигурации продукта.

## Перенастройка провайдера

Перейдите в **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или нажмите **Settings** в списке плагинов).

Страница настроек содержит:

- **Enable domain selling** — включает или выключает всю функцию
- **Default provider** — провайдер, используемый для поиска доменов и новых продуктов
- **Max TLDs per search** — сколько TLD проверять при поиске клиентом; более высокие значения показывают больше вариантов, но работают медленнее
- **Availability cache duration** — как долго кэшировать результаты доступности и цен; более низкие значения точнее, но увеличивают количество API-вызовов
- **Manage domain products** — быстрая ссылка на список Products
- **Configure providers** — открывает Integration Wizard для добавления или перенастройки провайдеров

### Добавление второго провайдера

Нажмите **Configure providers** и снова запустите мастер для нового регистратора. Можно одновременно настроить несколько провайдеров. Назначьте каждый доменный продукт конкретному провайдеру или оставьте провайдера по умолчанию.

### Ручная синхронизация TLD

На странице настроек нажмите **Sync TLDs** рядом с любым настроенным провайдером, чтобы получить последние цены. Это полезно после того, как провайдер обновляет оптовые цены или добавляет новые TLD.

## Журналы

Каждый провайдер пишет в свой собственный канал журнала. Журналы доступны для просмотра в **Network Admin › Ultimate Multisite › Logs**:

| Канал журнала | Содержимое |
|---|---|
| `domain-seller-registration` | Все попытки регистрации (успешные и неудачные) |
| `domain-seller-renewal` | Результаты задач продления |
| `domain-seller-opensrs` | Необработанная активность OpenSRS API |
| `domain-seller-namecheap` | Необработанная активность Namecheap API |
| `domain-seller-hostafrica` | Необработанная активность HostAfrica API |
| `domain-seller-openprovider` | Необработанная активность Openprovider API |
| `domain-seller-hostinger` | Необработанная активность Hostinger API |
| `domain-seller-godaddy` | Необработанная активность GoDaddy API |
| `domain-seller-resellerclub` | Необработанная активность ResellerClub API |
| `domain-seller-namesilo` | Необработанная активность NameSilo API |
| `domain-seller-enom` | Необработанная активность Enom API |

## Примечания о возможностях провайдеров

Не каждый API регистратора предоставляет одни и те же операции. Аддон показывает неподдерживаемые операции с понятными ошибками для администратора, вместо того чтобы завершаться сбоем без уведомления.

- **HostAfrica** поддерживает самый широкий рабочий процесс для реселлеров в реальном времени, включая поиск, синхронизацию TLD/цен, регистрацию, продление, перенос, обновления nameserver, DNS-записи, EPP-коды, registrar lock и ID protection.
- **Openprovider** поддерживает синхронизацию TLD с реселлерскими ценами, регистрацию, продление, переносы, обновления nameserver, DNS-зоны, EPP-коды, registrar lock и WHOIS privacy. Он аутентифицируется с помощью краткоживущего bearer token, который addon обновляет автоматически.
- **Hostinger** поддерживает поиск доступности, регистрацию, просмотр портфолио, обновления nameserver, registrar lock и WHOIS privacy через общий API-токен hPanel. Публичный Domains API от Hostinger не предоставляет реселлерские/оптовые цены, входящий перенос, явное продление или получение EPP-кода; продления выполняются только с автообновлением.
