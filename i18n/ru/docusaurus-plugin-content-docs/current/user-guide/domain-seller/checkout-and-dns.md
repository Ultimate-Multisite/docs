# Поле оформления заказа и управление DNS клиента {#checkout-field-and-customer-dns-management}

## Поле оформления заказа «Выбор домена» {#the-domain-selection-checkout-field}

Поле **Выбор домена** — это элемент оформления заказа, который дает клиентам выбор, как получить домен для своего сайта. Добавьте его в любую форму оформления заказа, чтобы включить продажу доменов.

### Добавление поля в форму оформления заказа {#adding-the-field-to-a-checkout-form}

1. Перейдите в **Network Admin › Ultimate Multisite › Формы оформления заказа**
2. Откройте или создайте форму оформления заказа
3. В редакторе оформления заказа нажмите **Добавить поле**
4. Выберите **Выбор домена** из списка полей
5. Настройте параметры поля (см. ниже)
6. Сохраните форму

### Параметры поля {#field-options}

**Режимы домена** — выберите, какие вкладки видит клиент. Каждый режим можно включать или отключать независимо:

| Режим | Что он делает |
|---|---|
| **Поддомен** | Клиент использует бесплатный поддомен в вашей сети (например, `mysite.yournetwork.com`). Оплата не требуется. |
| **Зарегистрировать новый домен** | Клиент ищет новый домен и регистрирует его через настроенного вами провайдера. Для расчета цены используется соответствующий доменный продукт. |
| **Существующий домен** | Клиент подключает домен, которым уже владеет. Плата за регистрацию не взимается. Домен автоматически подключается к его сайту. |

**Режим по умолчанию** — когда включены все три режима, какая вкладка открывается первой. Установите **Поддомен**, чтобы регистрация домена оставалась необязательной, или **Зарегистрировать новый домен**, чтобы стимулировать покупки.

**Доменный продукт** — при необходимости привяжите это поле к конкретному доменному продукту. Если не задано, аддон автоматически выбирает соответствующий продукт на основе TLD, который ищет клиент.

### Поля контактных данных регистранта {#registrant-contact-fields}

Когда клиент выбирает вкладку **Зарегистрировать новый домен**, форма оформления заказа добавляет встроенные поля контактных данных регистранта:

- Имя / фамилия
- Адрес электронной почты
- Адрес (строка 1, город, штат/провинция, почтовый индекс, страна)
- Номер телефона

Они требуются всеми регистраторами и проверяются до выполнения вызова API регистрации. Номера телефонов автоматически форматируются в международный формат `+CC.NNN`, ожидаемый регистраторами.

### Автоматически сгенерированный URL сайта {#auto-generated-site-url}

Когда клиент регистрирует или подключает домен, поле URL сайта автоматически заполняется выбранным доменом. Клиентам не нужно заполнять отдельное поле URL.

### Поведение поиска {#search-behaviour}

- Доступность домена проверяется в реальном времени через AJAX по мере ввода клиентом
- Если предпочтительный домен недоступен, отображаются альтернативные варианты TLD
- Цены загружаются в реальном времени и отображаются понятно (цена регистрации, цена продления, необязательная плата за конфиденциальность WHOIS)
- Коды купонов применяются к доменным продуктам так же, как к любому другому продукту

**Настройка отзывчивости поиска:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Добавление пользовательских полей в форму поиска домена:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Управление DNS клиента {#customer-dns-management}

Клиенты могут управлять DNS-записями для своих зарегистрированных доменов со страницы **Мой Account**, в записи соответствующего домена.

### Поддерживаемые типы записей {#supported-record-types}

| Тип | Использование |
|---|---|
| **A** | Сопоставить имя хоста с IPv4-адресом |
| **AAAA** | Сопоставить имя хоста с IPv6-адресом |
| **CNAME** | Создать псевдоним, указывающий на другое имя хоста |
| **MX** | Задать сервер обмена почтой |
| **TXT** | Добавить SPF, DMARC, проверочные или другие текстовые записи |

### Какие провайдеры поддерживают управление DNS? {#which-providers-support-dns-management}

Управление DNS (добавление, редактирование, удаление записей) доступно с **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** и **Openprovider**. Домены **Hostinger** могут обновлять серверы имен через Domain Seller; DNS-записи для размещенных доменов управляются основной интеграцией сопоставления доменов Hostinger. Домены Namecheap, GoDaddy и NameSilo отображают сведения о статусе и сроке действия, но DNS необходимо управлять напрямую в панели управления регистратора.

### DNS-записи по умолчанию {#default-dns-records}

Вы можете настроить DNS-записи по умолчанию, которые применяются автоматически при регистрации домена. Перейдите в **Settings › Domain Seller › DNS-записи по умолчанию**.

Значения записей по умолчанию поддерживают два токена:

| Токен | Заменяется на |
|---|---|
| `{DOMAIN}` | Зарегистрированное доменное имя (например, `example.com`) |
| `{SITE_URL}` | URL сайта WordPress для сайта клиента |

**Пример — направить корневой домен и www на IP вашего сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: просмотр и редактирование DNS {#admin-viewing-and-editing-dns}

Сетевые администраторы могут просматривать и редактировать DNS-записи для любого домена клиента со страницы редактирования домена в **Network Admin › Ultimate Multisite › Домены**.
