# Управление полями оформления заказа и DNS для клиентов

## Поле выбора домена

Поле **Выбор домена** — это элемент оформления заказа, который дает клиентам возможность выбрать, как они хотят получить домен для своего сайта. Добавьте его в любую форму оформления заказа, чтобы включить продажу доменов.

### Добавление поля в форму оформления заказа

1. Перейдите в **Network Admin › Ultimate Multisite › Checkout Forms**
2. Откройте или создайте форму оформления заказа
3. В редакторе оформления заказа нажмите **Add Field**
4. Выберите **Domain Selection** из списка полей
5. Настройте параметры поля (см. ниже)
6. Сохраните форму

### Параметры поля

**Режимы доменов** — Выберите вкладки, которые увидит клиент. Каждый режим можно включать или отключать независимо:

| Режим | Что он делает |
|---|---|
| **Subdomain** | Клиент использует бесплатный поддомен в вашей сети (например, `mysite.yournetwork.com`). Оплата не требуется. |
| **Register New Domain** | Клиент ищет и регистрирует новый домен через вашего настроенного провайдера. Для расчета стоимости используется соответствующий продукт домена. |
| **Existing Domain** | Клиент привязывает домен, которым он уже владеет. Плата за регистрацию не взимается. Домен автоматически привязывается к его сайту. |

**Режим по умолчанию** — Когда включены все три режима, какая вкладка открывается первой. Установите **Subdomain**, чтобы оставить регистрацию домена необязательной, или **Register New Domain**, чтобы стимулировать покупки.

**Продукт домена** — Опционально привяжите это поле к конкретному продукту домена. Если не указано, аддон автоматически выберет соответствующий продукт на основе TLD, который вводит клиент.

### Поля контактных данных владельца

Когда клиент выбирает вкладку **Register New Domain**, форма оформления заказа автоматически добавляет поля контактных данных владельца:

- Имя / Фамилия
- Адрес электронной почты
- Адрес (улица 1, город, штат/провинция, почтовый индекс, страна)
- Номер телефона

Эти данные обязательны для всех регистраторов и проверяются перед отправкой запроса к API регистрации. Номера телефонов автоматически форматируются в международный формат `+CC.NNN`, который ожидают регистраторы.

### Автоматически генерируемый URL сайта

Когда клиент регистрирует или привязывает домен, поле URL сайта автоматически заполняется на основе выбранного домена. Клиентам не нужно вводить отдельное поле для URL.

### Поведение поиска

- Доступность домена проверяется в реальном времени с помощью AJAX по мере ввода данных клиентом
- При недоступности предпочтительного домена показываются альтернативные предложения TLD
- Стоимость запрашивается и четко отображается (цена регистрации, цена продления, необязательная плата за конфиденциальность WHOIS)
- Купоны применяются к продуктам доменов так же, как и к любым другим продуктам

**Настройка отзывчивости поиска:**

```php
// Увеличить задержку дебаунса (миллисекунды), чтобы уменьшить количество вызовов API при медленном соединении
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // по умолчанию: 500
});
```

**Добавление пользовательских полей в форму поиска домена:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Дополнительные примечания',
    ];
    return $fields;
});
```

## Управление DNS для клиентов

Клиенты могут управлять записями DNS для своих зарегистрированных доменов на странице **My Account** в разделе, посвященном их домену.

### Поддерживаемые типы записей

| Тип | Использование |
|---|---|
| **A** | Привязывает имя хоста к IP-адресу IPv4 |
| **AAAA** | Привязывает имя хоста к IP-адресу IPv6 |
| **CNAME** | Создает псевдоним, указывающий на другой хост |
| **MX** | Устанавливает почтовый обменный сервер |
| **TXT** | Добавляет текстовые записи SPF, DMARC, верификации или другие текстовые записи |

### Какие провайдеры поддерживают управление DNS?

Управление DNS (добавление, редактирование, удаление записей) доступно с **OpenSRS**, **ResellerClub** и **Enom**. Домены Namecheap, GoDaddy и NameSilo отображают статус и информацию о сроке действия, но DNS необходимо управлять непосредственно в панели управления регистратора.

### Записи DNS по умолчанию

Вы можете настроить записи DNS по умолчанию, которые будут автоматически применены при регистрации домена. Перейдите в **Settings › Domain Seller › Default DNS Records**.

Значения записей по умолчанию поддерживают два токена:

| Токен | Заменяется на |
|---|---|
| `{DOMAIN}` | Зарегистрированное имя домена (например, `example.com`) |
| `{SITE_URL}` | URL сайта WordPress для клиента |

**Пример — привязка корневого домена и www к IP-адресу вашего сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: просмотр и редактирование DNS

Администраторы сети могут просматривать и редактировать записи DNS для любого домена клиента на странице редактирования домена в **Network Admin › Ultimate Multisite › Domains**.
