# Продукты доменов и ценообразование

Продукты доменов — это инструмент, который позволяет вам контролировать ценообразование, TLD, конфиденциальность WHOIS и выбор провайдера. Каждый продукт домена — это стандартный продукт Ultimate Multisite с дополнительной вкладкой **Настройки домена** (Domain Settings).

## Создание продукта домена {#creating-a-domain-product}

1. Перейдите в **Админ-панель сети › Ultimate Multisite › Продукты** (Network Admin › Ultimate Multisite › Products)
2. Нажмите **Добавить новый** (Add New)
3. Установите тип продукта на **Домен** (Domain)
4. Настройте вкладку **Настройки домена** (Domain Settings) (см. ниже)
5. Сохраните

Продукты доменов отображаются с фиолетовым значком **Домен** (Domain) в списке продуктов и могут быть отфильтрованы с помощью вкладки **Продукты доменов** (Domain Products).

## Вкладка настроек домена {#domain-settings-tab}

### Провайдер (Provider) {#provider}

Выберите регистратора, который будет обрабатывать регистрацию для этого продукта. По умолчанию используется глобальный **Провайдер по умолчанию** (Default provider), установленный в настройках Domain Seller.

### Поддерживаемые TLD {#supported-tlds}

Оставьте поле пустым, чтобы создать **универсальный продукт** (catch-all product), который будет применяться ко всем TLD, не совпадающим с другими продуктами.

Введите через запятую список TLD (например, `.com, .net, .org`), чтобы создать **продукт для конкретного TLD**, который будет применяться только к этим расширениям.

**Как работает сопоставление продуктов:** Когда клиент ищет домен, аддон выбирает наиболее подходящий продукт. Продукт с `.com` в списке TLD имеет приоритет над универсальным продуктом. Если ни один продукт, специфичный для TLD, не подходит, используется универсальный продукт. Если продукты не настроены, поиск домена не отображается.

### Тип наценки (Markup type) {#markup-type}

Три режима определяют, как рассчитывается ваша розничная цена на основе оптовой стоимости:

| Режим | Как это работает |
|---|---|
| **Процент** (Percentage) | Добавляет процент к оптовой стоимости. Наценка 20% на домен стоимостью 10 долларов США дает 12 долларов США. |
| **Фиксированная наценка** (Fixed markup) | Добавляет фиксированную сумму в долларах. Наценка в 5 долларов США на домен стоимостью 10 долларов США дает 15 долларов США. |
| **Фиксированная цена** (Fixed price) | Полностью игнорирует оптовую стоимость. Всегда взимает сумму, которую вы указали. |

### Вводная цена (Introductory pricing) {#introductory-pricing}

Включите эту опцию, чтобы предложить скидку на первый год. Установите отдельную **Вводную цену** (Introductory price) (цена на первый год) вместе с обычной **Ценой продления** (Renewal price) (цена со второго года и далее). Клиент увидит обе цены во время оформления заказа, чтобы знать, чего ожидать при продлении.

### Конфиденциальность WHOIS (WHOIS privacy) {#whois-privacy}

Определяет, предлагается ли защита конфиденциальности WHOIS для доменов, зарегистрированных через этот продукт.

| Настройка | Поведение |
|---|---|
| **Отключено** (Disabled) | Конфиденциальность WHOIS никогда не предлагается и не включается. |
| **Всегда включено** (Always Included) | Конфиденциальность WHOIS автоматически включается при регистрации бесплатно. |
| **Выбор клиента** (Customer Choice) | Во время оформления заказа появляется флажок. Установите **Цену за конфиденциальность** (Privacy price) для взимания платы за год, или оставьте ее на $0, чтобы предложить услугу бесплатно. |

Для Namecheap конфиденциальность WHOIS использует WhoisGuard (всегда бесплатно). Для OpenSRS используется служба конфиденциальности OpenSRS (может иметь стоимость на оптовом уровне).

## Импорт и синхронизация TLD {#tld-import-and-sync}

Продукты доменов отображают оптовую цену в реальном времени, полученную от подключенного провайдера. Для этого необходимо импортировать TLD.

- **Ручная синхронизация:** Настройки › Domain Seller › Синхронизация TLD (по провайдеру)
- **Автоматическая синхронизация:** Происходит ежедневно с помощью запланированной задачи cron для всех настроенных провайдеров

После синхронизации перейдите на вкладку Настройки домена любого продукта домена и используйте выбор TLD, чтобы увидеть доступные TLD с их текущими оптовыми ценами.

## Автоматическое продление {#auto-renewal}

Продление доменов привязано к статусу членства клиента:

- Когда членство продлевается и домен связан, продление домена автоматически ставится в очередь
- Попытки продления используют активный платежный шлюз клиента
- Неудачные продления автоматически повторяются с экспоненциальной задержкой (exponential backoff)
- Уведомления по электронной почте отправляются об успешном продлении, сбоях и предстоящих истечениях срока действия

ID шаблонов электронной почты для событий жизненного цикла домена:

| Событие | ID шаблона |
|---|---|
| Домен зарегистрирован | `domain_registered` |
| Домен продлен | `domain_renewed` |
| Продление не удалось | `domain_renewal_failed` |
| Домен скоро истекает | `domain_expiring_soon` |

## Админ: Ручная регистрация домена {#admin-manual-domain-registration}

Чтобы зарегистрировать домен от имени клиента, не заставляя его проходить процесс оформления заказа:

1. Перейдите в **Админ-панель сети › Ultimate Multisite › Регистрация домена** (Network Admin › Ultimate Multisite › Register Domain)
2. Выберите клиента и введите имя домена
3. Заполните контактные данные регистратора (имя, адрес, телефон)
4. Нажмите **Зарегистрировать** (Register)

Запись домена создается и привязывается к учетной записи клиента.
