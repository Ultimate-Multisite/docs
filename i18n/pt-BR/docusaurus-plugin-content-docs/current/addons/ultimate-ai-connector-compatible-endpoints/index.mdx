# Conector Ultimate de IA para Endpoints Compatíveis

O Conector Ultimate de IA para Endpoints Compatíveis integra APIs de provedores de IA compatíveis com o OpenAI à sua rede multisite do WordPress. Configure múltiplos provedores de IA — cada um com sua própria URL de endpoint, chave de API e modelo padrão — e roteie as requisições com fallback automático quando um provedor estiver indisponível.

## Principais Recursos {#key-features}

- **Suporte a múltiplos provedores**: Adicione vários cartões de provedores de IA, cada um com sua própria URL de endpoint, chave de API e modelo padrão.
- **Reordenar arrastando**: Organize os provedores por prioridade usando o ícone de pega (grip handle) em cada cartão; as requisições usarão os provedores em ordem.
- **Roteamento automático de fallback**: Se o provedor primário estiver indisponível, as requisições automaticamente retornam (fallback) para o próximo provedor configurado.
- **Configuração por provedor**: Cada provedor é configurado de forma independente com suas próprias credenciais e seleção de modelo.
- **Interface de configurações baseada em cartões**: Construído com os componentes Card estáveis do WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **Expansão automática ao adicionar**: Novos cartões de provedores se expandem automaticamente para que você possa preencher os detalhes imediatamente.
- **Quebra de cache de scripts**: Os scripts dos provedores são "cache-busted" (quebra de cache) na atualização do plugin para garantir que a versão mais recente seja carregada.

## Requisitos {#requirements}

- WordPress 5.3 ou superior
- PHP 7.4 ou superior
- Plugin Ultimate Multisite (ativo)
- Um endpoint de API compatível com OpenAI (auto-hospedado ou de terceiros)

## Instalação {#installation}

1. Faça o upload dos arquivos do addon para o seu diretório `/wp-content/plugins/`
2. Ative o plugin através do menu 'Plugins' no WordPress
3. Navegue até **Ultimate Multisite → AI Connector** para configurar o addon

## Configuração {#configuration}

### Configuração de Múltiplos Provedores {#multi-provider-setup}

A tela de configuração usa um layout baseado em cartões, onde cada cartão representa um provedor de IA. Você pode adicionar quantos provedores precisar.

#### Adicionando um Provedor {#adding-a-provider}

1. Vá para **Ultimate Multisite → AI Connector** no administrador da rede.
2. Clique em **Add Provider** (Adicionar Provedor).
3. Um novo cartão de provedor aparece e se expande automaticamente.
4. Preencha os detalhes do provedor:
   - **Provider Name** (Nome do Provedor) — um rótulo para identificar este provedor (ex: "OpenAI", "Mistral", "Local Ollama").
   - **Endpoint URL** (URL do Endpoint) — a URL base da API compatível com OpenAI (ex: `https://api.openai.com/v1`).
   - **API Key** (Chave de API) — sua chave de API para este provedor.
   - **Default Model** (Modelo Padrão) — o identificador do modelo a ser usado por padrão para este provedor (ex: `gpt-4o`, `mistral-small`).
5. Clique em **Save Settings** (Salvar Configurações).

#### Removendo um Provedor {#removing-a-provider}

Clique no botão **Remove** (Remover) em qualquer cartão de provedor para excluí-lo. Salve as configurações para aplicar a mudança.

#### Reordenando Provedores {#reordering-providers}

Arraste o ícone de pega (⣿) no lado esquerdo de qualquer cartão de provedor para reordená-lo. A ordem determina a prioridade de fallback: as requisições são enviadas para o primeiro provedor e retornam (fallback) para os provedores subsequentes caso uma requisição falhe.

### Roteamento de Fallback {#fallback-routing}

Quando uma requisição para o provedor primário falha — devido a um tempo limite, limite de taxa (rate limit) ou interrupção do serviço — o conector tenta automaticamente com o próximo provedor da lista. Isso continua pela lista até que um provedor tenha sucesso ou todos os provedores tenham sido tentados.

| Cenário | Comportamento |
|----------|-----------|
| Provedor primário responde | A requisição é atendida normalmente |
| Provedor primário falha | A requisição é retentada com o próximo provedor |
| Todos os provedores falham | Erro retornado ao chamador |

O fallback é transparente para o plugin ou tema chamador — eles recebem ou uma resposta de sucesso ou um erro final, independentemente de quantos provedores foram tentados.

### Referência de Configuração do Provedor {#provider-configuration-reference}

| Campo | Obrigatório | Descrição |
|-------|----------|-------------|
| Provider Name | Sim | Rótulo de exibição para este provedor |
| Endpoint URL | Sim | URL base da API compatível com OpenAI |
| API Key | Sim | Chave de autenticação para este provedor |
| Default Model | Sim | Identificador de modelo usado quando nenhum modelo é especificado na requisição |

## Solução de Problemas {#troubleshooting}

### Nenhum provedor é listado {#no-providers-are-listed}

Se a lista de provedores estiver vazia após salvar, confirme que:

- Você clicou em **Save Settings** (Salvar Configurações) após adicionar os provedores.
- Seu navegador não bloqueou o salvamento da página de configurações (verifique o console do navegador em busca de erros JavaScript).
- O plugin está ativo e você está na tela do administrador da rede.

### As requisições estão falhando para todos os provedores {#requests-are-failing-for-all-providers}

- Verifique se a URL do endpoint de cada provedor é alcançável a partir do seu servidor (e não apenas do seu navegador).
- Verifique se cada chave de API é válida e não expirou.
- Revise o status de limite de taxa e cota do provedor no painel deles.
- Ative o log de depuração do WordPress (`WP_DEBUG_LOG`) para capturar mensagens de erro detalhadas do conector.

### Um provedor não está sendo usado para fallback {#a-provider-is-not-being-used-for-fallback}

Reordenar arrastando altera a ordem de fallback. Confirme se o provedor que você espera receber as requisições de fallback está posicionado corretamente na lista e se sua URL de endpoint e chave de API estão corretas.

### O novo cartão de provedor não se expande automaticamente {#new-provider-card-does-not-expand-automatically}

A expansão automática requer que o JavaScript esteja ativado no seu navegador. Se o cartão não expandir ao clicar, verifique o console do navegador em busca de erros. O conteúdo do cartão está sempre acessível ao clicar no cabeçalho do cartão, mesmo que a expansão automática não seja acionada.
