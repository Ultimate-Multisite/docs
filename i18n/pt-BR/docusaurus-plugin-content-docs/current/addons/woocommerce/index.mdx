# Integração WooCommerce {#woocommerce-integration}

A Integração WooCommerce funciona de uma forma muito semelhante à maneira como nosso Manual Gateway funciona. Ela oferece muita flexibilidade aos seus usuários, permitindo que escolham qualquer uma das formas de pagamento disponíveis na sua instalação do WooCommerce. E é simples: quando o período de cobrança do usuário termina, a assinatura do usuário é colocada em espera. Ao mesmo tempo, um pedido será criado na instalação do WooCommerce do site principal da rede. O usuário então recebe uma fatura em seu e-mail, com um link para realizar o pagamento. Assim que o pagamento é recebido, a assinatura é renovada.

## Recursos {#features}

- Integração fácil entre Ultimate Multisite e WooCommerce
- Diversos métodos de pagamento disponíveis
- Cobrança recorrente automatizada usando o núcleo de assinaturas incluído, com melhorias opcionais do plugin WooCommerce Subscriptions
- Os totais do checkout incluem os impostos do Ultimate Multisite como linhas separadas de taxas não tributáveis do WooCommerce, para que os clientes possam revisar os valores dos impostos antes do pagamento

## Guia de uso {#usage-guide}

_**OBSERVAÇÃO IMPORTANTE: Este artigo se refere à versão 2.x do Ultimate Multisite.**_

_**ATENÇÃO:** Ultimate Multisite: Integração WooCommerce exige que o WooCommerce esteja ativado pelo menos no seu site principal._

Entendemos que _Stripe_ e _PayPal_ não estão disponíveis em alguns países, o que limita ou dificulta que os usuários do Ultimate Multisite usem efetivamente nosso plugin. Por isso, criamos um **add-on para integrar o _WooCommerce_**.

_WooCommerce_ é um plugin de eCommerce muito popular. Desenvolvedores ao redor do mundo criaram add-ons para integrar diferentes gateways de pagamento a ele. Aproveitamos isso para ampliar os gateways de pagamento que você pode usar no sistema de cobrança do Ultimate Multisite.

## Configurando {#setting-it-up}

Primeiro, você precisa instalar o add-on WooCommerce Integration, que é **gratuito para usuários licenciados**. Para instalá-lo, clique em Ultimate Multisite no seu dashboard de super admin e selecione **Add-ons**.

![Página de Add-ons](/img/admin/addons-page.png)

Aqui você encontra todos os add-ons do Ultimate Multisite. Clique no add-on **Ultimate Multisite: WooCommerce Integration**.

![Página de Add-ons mostrando WooCommerce Integration](/img/admin/addons-page.png)

Uma janela aparecerá com os detalhes do add-on. Basta clicar em **Install Now** *.*

Após a instalação ser concluída, você será redirecionado para a página de plugins. Aqui, basta clicar em **Network Activate** e o add-on _WooCommerce_ será ativado na sua rede.

Depois de instalado, você verá uma opção adicional nas configurações de pagamentos do Ultimate Multisite. Então você poderá **ativar** a opção _WooCommerce_ ali.

Depois, você pode acessar as **configurações do _WooCommerce_ do seu site principal** para ativar e configurar o gateway de pagamento que prefere usar.

### Linhas de taxas de imposto no checkout {#checkout-tax-fee-lines}

Quando um checkout do Ultimate Multisite inclui impostos, a integração adiciona cada valor de imposto do Ultimate Multisite ao checkout do WooCommerce como uma linha separada de taxa não tributável. Isso mantém o total de impostos visível antes do pagamento, evitando que o WooCommerce tribute novamente a taxa de imposto.

## Como gerenciar o pagamento {#how-to-manage-the-payment}

A Integração WooCommerce funciona de uma forma muito semelhante à maneira como nosso **Manual Gateway** funciona. A desvantagem é que todo pagamento deve ser pago manualmente ao final do período de cobrança. Veja como o fluxo funciona:

  * Quando o ciclo de cobrança do usuário termina, a assinatura é colocada **em espera**. Ao mesmo tempo, um pedido será criado na instalação do _WooCommerce_ no site principal da sua rede.

  * Os usuários então recebem uma fatura em seu e-mail, com um link para realizar o pagamento (o link também aparece no histórico de cobrança deles no dashboard do subsite). Esse link redireciona o usuário para o **checkout** do _**WooCommerce**_, onde os usuários poderão selecionar um dos **gateways _WooCommerce_ disponíveis** para processar o pagamento.

  * Assim que o pagamento é recebido, a assinatura é **renovada**, o pagamento é registrado no Ultimate Multisite e o pedido é definido como concluído.

### Algumas observações importantes: {#some-important-notes}

  * A funcionalidade padrão do Ultimate Multisite ainda funciona com esta forma de integração. Por exemplo, **emitir reembolsos** pela tela de **Gerenciamento de Assinaturas** se comunicará com o WooCommerce para emitir o reembolso usando a forma de pagamento selecionada.

  * Ultimate Multisite: WooCommerce Integration depende do WP-Cron para funcionar. Isso funciona muito bem na maioria das vezes, mas, se o site tiver pouco tráfego, o cron não é acionado para verificar eventos agendados e o evento é perdido, o que pode causar atrasos na execução.

## Plugin WooCommerce Subscriptions {#woocommerce-subscriptions-plugin}

Iniciar manualmente um pagamento a cada mês não é ideal para usuários finais, porque espera-se que as assinaturas recorram automaticamente a cada ciclo de cobrança. O add-on WooCommerce Integration agora inclui o código principal de assinaturas, então produtos recorrentes sempre usam assinaturas para cobrança recorrente.

Você **não** precisa comprar nem instalar o [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) separado para vender produtos recorrentes por meio desta integração. Se esse plugin for comprado, instalado e estiver disponível no seu site principal, Ultimate Multisite o usará e WooCommerce exibirá recursos adicionais de gerenciamento de assinaturas. Quando ele não estiver disponível, o núcleo de assinaturas incluído ainda lida com produtos recorrentes.

![Configurações de integração de assinatura do WooCommerce](/img/addons/woocommerce-subscription-integration.png)

E é isso. Seus usuários finais não precisarão mais fazer manualmente um pagamento no fim do ciclo de cobrança para produtos recorrentes.

Além de um pedido ser criado no seu site principal em _WooCommerce_, também será criada uma assinatura para cada Account.

### Comportamento de suspensão em renovação com falha {#failed-renewal-suspension-behavior}

Por padrão, as associações permanecem ativas enquanto WooCommerce Subscriptions processa seu cronograma de novas tentativas por pagamento com falha. Isso preserva a janela de carência normal do WooCommerce Subscriptions, para que uma renovação recuperada possa trazer a assinatura e a associação de volta à sincronização sem interromper prematuramente o acesso.

Administradores que desejam controle de acesso mais rígido podem ativar **Suspend Memberships Immediately on Failed Renewals** nas configurações de WooCommerce Integration. Quando esta configuração opcional está ativada, Ultimate Multisite suspende a associação relacionada assim que um pagamento de renovação falha, em vez de esperar que o tratamento de novas tentativas do WooCommerce Subscriptions termine.

Desenvolvedores podem substituir a mesma decisão com o filtro `wu_woo_suspend_on_payment_failure`. As páginas de referência de hooks são geradas a partir do código-fonte do add-on, então esta visão geral apenas destaca o nome do filtro para políticas personalizadas de suspensão.
