# Multitenancy

Distribua subsites entre servidores e bancos de dados para desempenho e segurança

## Destaques do lançamento da versão 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 adiciona a base de locatário soberano para fluxos de trabalho de isolamento em produção:

- Autologin de locatário sem estado com visitas SSO de curta duração e protegidas contra repetição.
- Isolamento de banco de dados e sistema de arquivos por subsite para locatários soberanos.
- Capacidade de multitenancy Hostinger para provisionamento de locatários com reconhecimento de host.
- Portões de verificação de migração soberana para verificações de dados legados e processamento push assíncrono.
- Vinculação configurável de host de banco de dados na mesma máquina para instalações Bedrock, FrankenPHP e localhost.
- Melhorias no bootstrap de locatário, gravador de banco de dados, bootstrap de esquema e provisionamento de usuários.
- Desmontagem soberana mais segura que limpa as credenciais do locatário.

Consulte os guias do usuário para [fluxo de trabalho de migração soberana](/user-guide/getting-started/sovereign-migration-workflow), [isolamento de multitenancy](/user-guide/configuration/multi-tenancy-isolation) e [operações de locatário soberano](/user-guide/administration/sovereign-tenant-operations).

## Requisitos {#requirements}

- WordPress 5.3 ou superior
- PHP 7.4 ou superior
- plugin Multisite Ultimate (ativo)

## Instalação {#installation}

1. Envie os arquivos do complemento para o diretório `/wp-content/plugins/`
2. Ative o plugin pelo menu 'Plugins' no WordPress
3. Configure as definições do complemento no administrador da rede

## Configuração {#configuration}

O complemento oferece suporte a várias opções de configuração acessíveis pela interface de administração da rede.

### Configuração de locatário soberano {#sovereign-tenant-configuration}

Para locatários soberanos, configure o banco de dados do locatário, a raiz do sistema de arquivos, a vinculação do host do banco de dados e a capacidade do host antes de rotear tráfego para o locatário. Execute a verificação de migração após cada alteração que afete credenciais de banco de dados, vinculação de host, roteamento ou provisionamento de usuários.

Use `localhost` somente quando o runtime do locatário e as concessões do banco de dados esperarem acesso por socket local ou na mesma máquina. Use `127.0.0.1` ou um nome de host de serviço quando esse for o host de concessão configurado pelo seu provedor.

### Capacidade Hostinger {#hostinger-capability}

A capacidade de multitenancy Hostinger funciona com o token de integração compartilhado da Hostinger. Confirme que o domínio está presente no hPanel, que o DNS aponta para a Account Hostinger e que as configurações do banco de dados do locatário correspondem ao hPanel antes da verificação.

- `ultimate-multisite-addon-template_enabled` - Substitui o status de ativação/desativação do complemento
