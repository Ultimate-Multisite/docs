# Gratis AI Agent {#gratis-ai-agent}

Gratis AI Agent yaiku asisten AI WordPress otonom sing bisa ngrancang, mbangun, lan ngatur situs WordPress lengkap liwat pandhuan basa alami. Iki nyedhiyakake sakumpulan **kabisan** terstruktur — tumindak kapisah sing bisa dites lan bisa dijaluk dening agent — nyakup kabeh saka manajemen konten nganti kustomisasi sistem desain. Versi 1.9.0 nambahake kabisan nggawe konten, tinjauan kaca visual liwat screenshot sisih klien, limang profil agent bawaan, lan feature flags kanggo kontrol akses lan branding.

## Fitur Utama {#key-features}

- **Sistem kabisan** — katalog tumindak modular lan bisa dikembangake sing bisa dijaluk agent ing instalasi WordPress panjenengan
- **Manajemen Konten** — nggawe kiriman lan kaca (kalebu nemtokake template kaca), nggawe pirang-pirang kiriman sekaligus ing siji panggilan, nyetel gambar unggulan, lan mbangun formulir kontak
- **Tinjauan Visual** — njupuk screenshot kaca langsung, mbandhingake kahanan sadurunge/sawise, lan entuk tinjauan desain sing digawe AI nyakup tata letak, tipografi, warna, lan aksesibilitas
- **Limang agent bawaan** — Content Writer, Site Builder, Design Studio, Plugin Manager, lan Support Assistant; bisa diganti liwat Agent Picker ing header chat
- **Feature flags** — tombol kontrol akses lan branding ing Settings → Feature Flags kanggo watesan peran, jeneng white-label, lan kustomisasi widget
- **Manajemen Custom Post Type** — ndhaptar, nampilake dhaptar, lan mbusak custom post type kanthi persistensi liwat restart
- **Manajemen Custom Taxonomy** — ndhaptar, nampilake dhaptar, lan mbusak custom taxonomy kanthi persistensi
- **Kabisan Design System** — nyuntikake CSS kustom, ngatur pola blok sing dikurasi, nyetel logo situs, lan ngetrapake preset theme.json
- **Manajemen Global Styles** — maca lan nulis nilai theme.json kanggo palet warna, tipografi, spasi, lan liyane
- **Manajemen Navigation Menu** — nggawe lan nganyari menu navigasi lan item-itemé kanthi programatis
- **Manajemen Options** — maca lan nulis opsi WordPress nganggo blocklist keamanan bawaan kanggo nglindhungi setelan kritis
- **Installable Abilities Registry** — nemokake lan ngaktifake paket kabisan tambahan tanpa ngowahi berkas plugin inti
- **Site Builder Orchestration v2** — rencana mbangun situs multi-langkah kanthi panemuan plugin, pelacakan kemajuan, lan pemulihan kesalahan
- **Prentah benchmark WP-CLI** — mbukak suite benchmark kapabilitas agent saka command line kanggo CI lan evaluasi model

## Syarat {#requirements}

- WordPress 6.0 utawa luwih dhuwur
- PHP 8.1 utawa luwih dhuwur
- WordPress Multisite (diaktifake jaringan) utawa instalasi situs tunggal
- Plugin konektor panyedhiya AI aktif (Anthropic, OpenAI, utawa kompatibel)

## Instalasi {#installation}

1. Unggah berkas plugin menyang `/wp-content/plugins/gratis-ai-agent`
2. Aktifake plugin liwat layar **Plugins** ing WordPress
3. Navigasi menyang **Gratis AI Agent → Settings** lan sambungake panyedhiya AI panjenengan
4. (Opsional) Jelajahi **Abilities Registry** kanggo masang paket kabisan tambahan

## Miwiti Cepet {#quick-start}

Sawise diaktifake lan disambungake menyang panyedhiya AI, bukak panel **Gratis AI Agent** ing sidebar admin WordPress. Ketik pandhuan basa alami lan agent bakal ngrancang lan nindakake langkah-langkah sing dibutuhake nganggo kabisan sing kasedhiya.

**Tuladha pandhuan:**

```
Gawe custom post type "Portfolio" nganggo taxonomy "Project Category".
```

```
Suntikake palet warna dark-mode menyang theme lan setel warna utama dadi #1a1a2e.
```

```
Bangun situs web restoran nganggo menu online, formulir booking, lan kaca kontak.
```

## Ringkesan Kabisan {#abilities-overview}

Kabisan yaiku tumindak atomik sing bisa ditindakake agent. Saben kabisan yaiku class PHP sing kadhaptar sing nyedhiyakake skema (parameter lan jinis bali) sing diwaca agent nalika runtime. Kabisan diklompokake dadi wilayah fungsional:

| Wilayah | Kabisan |
|---|---|
| **Manajemen Konten** | `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form` |
| **Tinjauan Visual** | `capture_screenshot`, `compare_screenshots`, `review_page_design` |
| **Custom Post Types** | `register_post_type`, `list_post_types`, `delete_post_type` |
| **Custom Taxonomies** | `register_taxonomy`, `list_taxonomies`, `delete_taxonomy` |
| **Design System** | `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset` |
| **Global Styles** | `get_global_styles`, `set_global_styles`, `reset_global_styles` |
| **Navigation Menus** | `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus` |
| **Manajemen Options** | `get_option`, `set_option`, `delete_option`, `list_options` |
| **Installable Abilities** | `list_available_abilities`, `install_ability`, `recommend_plugin` |
| **Site Builder** | `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error` |

Delengen [Referensi Kabisan](./abilities) kanggo dokumentasi parameter lengkap lan tuladha panggunaan.

## WP-CLI {#wp-cli}

Gratis AI Agent nggawa kulawarga prentah `wp gratis-ai-agent` kanggo mbukak benchmark lan ngatur kabisan saka command line. Delengen [Referensi WP-CLI](./wp-cli) kanggo dhaptar prentah lengkap.

```bash
# Run the full benchmark suite
wp gratis-ai-agent benchmark

# List installed abilities
wp gratis-ai-agent abilities list

# Install an ability pack
wp gratis-ai-agent abilities install <slug>
```

## Konfigurasi {#configuration}

Kabeh setelan ana ing **Gratis AI Agent → Settings** ing admin WordPress:

| Setelan | Katrangan |
|---|---|
| **AI Provider** | Pilih lan konfigurasi konektor panyedhiya AI aktif |
| **Model** | Pilih model basa sing arep digunakake kanggo sesi agent |
| **Max Tokens** | Anggaran token saben panjaluk (gawan 4096) |
| **Ability Timeout** | Detik maksimum kabisan individu bisa mlaku (gawan 30) |
| **Debug Logging** | Cathet panggilan kabisan lan penalaran agent menyang log debug WordPress |

## Blocklist Keamanan Manajemen Options {#options-management-safety-blocklist}

Kemampuan Manajemen Pilihan kalebu blocklist bawaan sing nyegah agent maca utawa nulis pilihan WordPress sing sensitif. Blocklist standar nyakup:

- `admin_email`, `blogadmin_email`
- `siteurl`, `home` (pilihan URL — gunakna piranti migrasi khusus kanggo iki)
- `auth_key`, `secure_auth_key`, `logged_in_key`, `nonce_key` (kunci rahasia)
- `active_plugins`, `template` (aktivasi plugin/theme — gunakna kemampuan khusus)
- Kredensial gateway pembayaran lan kunci API sing didaftaraké déning plugin sing wis dipasang

Entri tambahan bisa ditambahaké liwat filter `gratis_ai_agent_options_blocklist`.

## Hook lan Filter {#hooks-and-filters}

### Aksi {#actions}

- `gratis_ai_agent_ability_registered` — dijalanké sawisé kemampuan didaftaraké, nampa jeneng class kemampuan
- `gratis_ai_agent_plan_started` — dijalanké nalika rencana site builder wiwit dieksekusi
- `gratis_ai_agent_plan_completed` — dijalanké nalika rencana rampung kanthi sukses
- `gratis_ai_agent_plan_failed` — dijalanké nalika rencana nemoni galat sing ora bisa dipulihaké

### Filter {#filters}

- `gratis_ai_agent_abilities` — ngowahi dhaptar kemampuan sing didaftaraké sadurungé agent ngemot
- `gratis_ai_agent_options_blocklist` — njembarake utawa nyuda blocklist keamanan pilihan
- `gratis_ai_agent_benchmark_questions` — nambah pitakon benchmark khusus menyang rangkaian evaluasi
- `gratis_ai_agent_plan_plugins` — ngowahi dhaptar panemuan plugin nalika nggawe rencana site
