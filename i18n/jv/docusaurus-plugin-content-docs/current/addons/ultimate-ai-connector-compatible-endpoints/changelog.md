* Anyar: Dhukungan multi-provider — atur pirang-pirang AI endpoint lan arahake panyuwunan nganggo fallback otomatis ing antarane provider.
* Ndandani: Integrasi SDK multi-provider nganggo provider ID, URL registrasi, lan scoping filter HTTP sing bener saben provider.
* Ndandani: Kertu provider anyar saiki otomatis ngembang nalika ditambahake; script cache busting nalika plugin dianyari.
* Ndandani: Namespace kelas provider dinamis kanggo eval() lan prilaku otomatis ngembang kanggo provider sing anyar ditambahake.
* Ndandani: Nggunakake komponen Card/CardBody/CardHeader/CardDivider sing stabil (ora maneh eksperimental) kanggo kompatibilitas WordPress 6.9+.
* Ndandani: Ngganti DragHandle sing ora kadéfinisi nganggo ikon grip unicode kanggo provider drag-to-reorder.
* Ndandani: Ngilangi panyuwunan HTTP sing mblokir sing mlaku saben muatan kaca.
* Ditingkatake: Alur kerja GitHub Actions ditingkatake menyang Node.js 24.
