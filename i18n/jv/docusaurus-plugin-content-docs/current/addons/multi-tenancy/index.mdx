# Multi-Tenancy {#multi-tenancy}

Sebarake subsitus ing antarane server lan basis data kanggo kinerja lan keamanan

## Sorotan rilis Versi 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 nambah dhasar panyewa berdaulat kanggo alur kerja isolasi produksi:

- Autologin panyewa tanpa status nganggo kunjungan SSO sing umure cekak lan dilindhungi saka replay.
- Isolasi basis data lan sistem berkas saben subsitus kanggo panyewa berdaulat.
- Kapabilitas multi-tenancy Hostinger kanggo panyedhiyan panyewa sing sadar host.
- Gerbang verifikasi migrasi berdaulat kanggo pamriksan data warisan lan pangolahan push async.
- Pangiket host basis data ing mesin sing padha sing bisa dikonfigurasi kanggo instalasi Bedrock, FrankenPHP, lan localhost.
- Dandan bootstrap panyewa, panulis basis data, bootstrap skema, lan panyedhiyan pangguna.
- Mbongkar berdaulat sing luwih aman sing ngresiki kredensial panyewa.

Delengen pandhuan pangguna kanggo [alur kerja migrasi berdaulat](/user-guide/getting-started/sovereign-migration-workflow), [isolasi multi-tenancy](/user-guide/configuration/multi-tenancy-isolation), lan [operasi panyewa berdaulat](/user-guide/administration/sovereign-tenant-operations).

## Sarat {#requirements}

- WordPress 5.3 utawa luwih dhuwur
- PHP 7.4 utawa luwih dhuwur
- Plugin Multisite Ultimate (aktif)

## Instalasi {#installation}

1. Unggah berkas tambahan menyang direktori `/wp-content/plugins/` sampeyan
2. Aktifake plugin liwat menu 'Plugins' ing WordPress
3. Konfigurasi setelan tambahan ing admin jaringan

## Konfigurasi {#configuration}

Tambahan iki ndhukung macem-macem pilihan konfigurasi sing bisa diakses liwat antarmuka admin jaringan.

### Konfigurasi panyewa berdaulat {#sovereign-tenant-configuration}

Kanggo panyewa berdaulat, konfigurasi basis data panyewa, oyod sistem berkas, pangiket host basis data, lan kapabilitas host sadurunge ngarahake lalu lintas menyang panyewa. Jalanake verifikasi migrasi sawisé saben owah-owahan sing mengaruhi kredensial basis data, pangiket host, routing, utawa panyedhiyan pangguna.

Gunakake `localhost` mung nalika runtime panyewa lan grant basis data ngarepake soket lokal utawa akses mesin sing padha. Gunakake `127.0.0.1` utawa hostname layanan nalika kuwi host grant sing dikonfigurasi dening panyedhiya sampeyan.

### Kapabilitas Hostinger {#hostinger-capability}

Kapabilitas multi-tenancy Hostinger bisa digunakake karo token integrasi Hostinger bareng. Priksa manawa domain ana ing hPanel, DNS nuduhake menyang akun Hostinger, lan setelan basis data panyewa cocog karo hPanel sadurunge verifikasi.

## Hook lan Filter {#hooks-and-filters}

- `ultimate-multisite-addon-template_enabled` - Nglindhih status aktif/nonaktif tambahan
