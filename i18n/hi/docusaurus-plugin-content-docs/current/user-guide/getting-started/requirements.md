# Ultimate Multisite की आवश्यकताएं

Ultimate Multisite खरीदने से पहले सबसे पहला सवाल जो आपके मन में आ सकता है वो है कि हमारे plugin को चलाने के लिए क्या-क्या चाहिए। खुशखबरी यह है कि Ultimate Multisite चलाने के लिए बस कुछ बुनियादी चीज़ें चाहिए: एक **domain name**, एक अच्छा **hosting provider** और WordPress **Multisite** installation। बस इतना ही!

## Domain Name

सबसे पहले, domain name के बारे में बात करते हैं।

Domain name चुनते समय यह ध्यान रखें कि आपकी खास niche क्या है और आप Ultimate Multisite से क्या बनाना चाहते हैं: क्या आप एक agency बनाना चाहते हैं? Shop जैसी websites बेचना चाहते हैं? या शायद WordPress web hosting? Ultimate Multisite के साथ सब कुछ संभव है।

हमारा plugin subscription services, payment collection, checkout forms, discount vouchers और customer communications की सुविधा देता है। WordPress Multisite को install करने, configure करने और maintain करने में लगने वाली बहुत सारी मेहनत Ultimate Multisite के ज़रिए आसान हो जाती है।

आप जो भी niche चुनें, हमारे plugin से उसे manage किया जा सकता है, बशर्ते आप websites as a service बेच रहे हों! ऐसा domain name ज़रूर चुनें जो आपकी business plan से मेल खाता हो।

## Hosting Provider

अब जब आपके पास एक आकर्षक domain name है जो आपकी niche और business को दर्शाता है, तो आपको Ultimate Multisite के साथ अपने network को host करने के लिए एक अच्छी web hosting company चुननी होगी। Market में कई अच्छे विकल्प मौजूद हैं। कुछ hosting providers Ultimate Multisite के साथ बेहतर तरीके से integrate होते हैं, जो आप क्या बनाने की सोच रहे हैं इसके हिसाब से एक महत्वपूर्ण factor हो सकता है।

Ultimate Multisite किसी भी hosting provider में काम करना चाहिए। बस फ़र्क इतना है कि _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ और _Runcloud_ जैसी कुछ hosting companies में domain mapping के लिए बेहतर integration मिलता है। दूसरी hosting के लिए आपको custom domains manually map करने पड़ सकते हैं। ऐसा hosting provider लेना भी अच्छा रहेगा जो AutoSSL support करता हो। _Closte_, _Runcloud_ और _cPanel_ इसे support करते हैं।

Server specifications की बात करें तो कोई fixed formula नहीं है क्योंकि यह इस पर निर्भर करेगा कि आप कौन-कौन से features और functions offer करेंगे और आपके network का size कितना है। Shared hosting शुरुआत के लिए ठीक हो सकती है, लेकिन जैसे-जैसे आपका network बढ़ेगा, आपको शायद dedicated VPS की ज़रूरत पड़ेगी।

## WordPress Multisite

अब, WordPress Multisite installation के बारे में थोड़ी बात करते हैं।

WordPress Multisite आपको एक ही installation पर sites का network रखने की सुविधा देता है। यह एक built-in feature है, लेकिन यह default रूप से active नहीं होता। आपको अपने WordPress installation और web hosting control panel में इस feature को enable करना होगा।

Multisite enabled WordPress **Ultimate Multisite की नींव है**। चूंकि Ultimate Multisite का मुख्य उपयोग users को websites as a service manage करने और बेचने में मदद करना है, इसलिए यह Multisite enabled WordPress पर ही होना चाहिए।

हमारे पास **एक विस्तृत लेख** है जो step by step बताता है कि अपने WordPress installation में Multisite feature enable करने के लिए क्या करना होगा।

WordPress पर Multisite enable करने के बाद, Ultimate Multisite install करने का समय है। Ultimate Multisite को सही तरीके से install करने के लिए, आपके पास होना चाहिए:

  * WordPress: v5.3+ (Recommended: Latest stable version)

  * PHP: 7.4.x (> 8.0 के लिए support जल्द आ रहा है)

  * MySQL: v5+ (Recommended: 5.6, अगर आपके hosting provider पर 8.0 उपलब्ध नहीं है)

Ultimate Multisite के लिए बस ये ही software requirements हैं।

ध्यान रखें कि Ultimate Multisite PHP 8.0 पर काम कर सकता है लेकिन हम इसे PHP 7.4.x पर चलाने की सलाह देते हैं।

साथ ही, हम इसे **main domains** पर install करने की सलाह देते हैं, subdomains या subdirectories पर नहीं। यह subdomains/subdirectories पर काम कर सकता है, लेकिन कुछ errors आ सकते हैं।

Ultimate Multisite install करने की विस्तृत guide के लिए, **यहाँ click करें**।
