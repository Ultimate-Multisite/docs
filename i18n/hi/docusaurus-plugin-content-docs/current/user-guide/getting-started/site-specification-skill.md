# साइट स्पेसिफिकेशन कौशल {#site-specification-skill}

**साइट स्पेसिफिकेशन कौशल** एक व्यवस्थित तरीका है जिसका उपयोग आपके साइट के लक्ष्यों, दर्शकों और ब्रांड पहचान को कैप्चर करने के लिए किया जाता है। यह जानकारी आपकी **site_brief** मेमोरी में स्टोर की जाती है, जिसका उपयोग एजेंट विभिन्न सत्रों में लगातार और संदर्भ के अनुसार सहायता प्रदान करने के लिए करते हैं।

## साइट स्पेसिफिकेशन क्या है? {#what-is-site-specification}

साइट स्पेसिफिकेशन वह प्रक्रिया है जिसमें निम्नलिखित का दस्तावेजीकरण (documenting) किया जाता है:

- **साइट का उद्देश्य (Site purpose)**: आपका साइट क्या करता है और इसका अस्तित्व क्यों है।
- **लक्षित दर्शक (Target audience)**: आपके साइट पर कौन आता है और उन्हें क्या चाहिए।
- **ब्रांड पहचान (Brand identity)**: आपके रंग, लहजा (tone), और विज़ुअल स्टाइल।
- **व्यावसायिक लक्ष्य (Business goals)**: आपके साइट के लिए सफलता कैसी दिखती है।
- **सामग्री संरचना (Content structure)**: आपका साइट कैसे व्यवस्थित है।

यह स्पेसिफिकेशन आपकी **site_brief** बन जाती है, जो एक स्थायी मेमोरी है जिसका उपयोग एजेंट आपके साइट के संदर्भ को समझने के लिए करते हैं।

## साइट स्पेसिफिकेशन का उपयोग क्यों करें? {#why-use-site-specification}

### सत्रों में निरंतरता (Consistency Across Sessions) {#consistency-across-sessions}

site_brief के बिना, आपको हर बार जब आप नया सत्र शुरू करते हैं तो अपने साइट के उद्देश्य को फिर से समझाना पड़ेगा। इसके साथ, एजेंट तुरंत समझ जाते हैं:

- आपके साइट के लक्ष्य और दर्शक
- आपके ब्रांड के रंग और लहजा
- आपकी सामग्री संरचना
- आपके व्यावसायिक उद्देश्य

### बेहतर सुझाव (Better Recommendations) {#better-recommendations}

एजेंट आपके site_brief का उपयोग निम्न के लिए करते हैं:

- आपके साइट के उद्देश्य के अनुरूप फीचर्स का सुझाव देना
- आपके लक्ष्यों से मेल खाने वाली सामग्री संरचना की सिफारिश करना
- आपके ब्रांड के अनुरूप डिज़ाइन प्रस्तावित करना
- असंगत फीचर्स का सुझाव देने से बचना

### तेज़ ऑनबोर्डिंग (Faster Onboarding) {#faster-onboarding}

नए एजेंट (या नए सत्रों में एजेंट) स्पष्टीकरण वाले प्रश्न पूछने के बजाय आपके site_brief को पढ़कर जल्दी से काम करना सीख सकते हैं।

## साइट स्पेसिफिकेशन शुरू करना {#initiating-site-specification}

### थीम बिल्डर ऑनबोर्डिंग के दौरान {#during-theme-builder-onboarding}

साइट स्पेसिफिकेशन कौशल **Theme Builder onboarding flow** के दौरान स्वचालित रूप से शुरू होता है। सेटअप असिस्टेंट एजेंट प्रश्न पूछता है और आपकी site_brief बनाता है।

### मैन्युअल शुरुआत {#manual-initiation}

आप किसी भी समय साइट स्पेसिफिकेशन शुरू कर सकते हैं:

```
"Let's define my site specification"
```

या

```
"Help me create a site brief"
```

## साइट स्पेसिफिकेशन प्रक्रिया {#the-site-specification-process}

### चरण 1: साइट का उद्देश्य (Site Purpose) {#step-1-site-purpose}

एजेंट पूछता है:

```
आपके साइट का प्राथमिक उद्देश्य क्या है?
- ई-कॉमर्स स्टोर
- ब्लॉग या कंटेंट साइट
- पोर्टफोलियो या शोकेस
- SaaS एप्लीकेशन
- समुदाय या फ़ोरम
- अन्य: [विवरण दें]
```

आप एक श्रेणी चुन सकते हैं या अपना उद्देश्य बता सकते हैं।

### चरण 2: लक्षित दर्शक (Target Audience) {#step-2-target-audience}

```
आपके प्राथमिक दर्शक कौन हैं?
- उपभोक्ता / सामान्य जनता
- व्यावसायिक पेशेवर
- डेवलपर / तकनीकी उपयोगकर्ता
- छात्र / शिक्षक
- अन्य: [विवरण दें]

उनकी मुख्य ज़रूरतें क्या हैं?
```

### चरण 3: ब्रांड पहचान (Brand Identity) {#step-3-brand-identity}

```
आपके ब्रांड के रंग क्या हैं?
- प्राथमिक रंग: [कलर पिकर या हेक्स कोड]
- द्वितीयक रंग: [कलर पिकर या हेक्स कोड]
- एक्सेंट रंग: [वैकल्पिक]

आप अपने ब्रांड के लहजे का वर्णन कैसे करेंगे?
- पेशेवर / कॉर्पोरेट
- रचनात्मक / कलात्मक
- चंचल / कैज़ुअल
- न्यूनतम / आधुनिक
- गर्मजोशी भरा / दोस्ताना
```

### चरण 4: व्यावसायिक लक्ष्य (Business Goals) {#step-4-business-goals}

```
आपके साइट के लिए सफलता कैसी दिखती है?
- लीड जनरेट करना
- उत्पाद बेचना
- समुदाय बनाना
- ज्ञान साझा करना
- अधिकार स्थापित करना
- अन्य: [विवरण दें]

आपका प्राथमिक मीट्रिक क्या है?
- राजस्व (Revenue)
- उपयोगकर्ता जुड़ाव (User engagement)
- कंटेंट पहुंच (Content reach)
- कन्वर्ज़न (Conversions)
- अन्य
```

### चरण 5: सामग्री संरचना (Content Structure) {#step-5-content-structure}

```
आपकी सामग्री कैसे व्यवस्थित है?
- फ्लैट (सभी सामग्री एक ही स्तर पर)
- पदानुक्रमित (श्रेणियाँ और उपश्रेणियाँ)
- कालानुक्रमिक (ब्लॉग-शैली)
- उत्पाद-आधारित (कैटलॉग)
- अन्य: [विवरण दें]

आप किस प्रकार की सामग्री का उपयोग करते हैं?
- ब्लॉग पोस्ट
- उत्पाद पेज
- केस स्टडी
- डॉक्यूमेंटेशन
- वीडियो
- अन्य
```

## आपकी site_brief मेमोरी {#your-sitebrief-memory}

साइट स्पेसिफिकेशन पूरा करने के बाद, आपकी जानकारी आपके एजेंट मेमोरी में **site_brief** के रूप में स्टोर हो जाती है। यह एक संरचित रिकॉर्ड है जिसमें निम्नलिखित शामिल है:

```json
{
  "site_purpose": "हस्तनिर्मित गहनों के लिए ई-कॉमर्स स्टोर",
  "target_audience": "25-45 वर्ष की महिलाएं, जो टिकाऊ फैशन में रुचि रखती हैं",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "गर्मजोशी भरा, कारीगरी वाला, टिकाऊ",
  "business_goals": ["बिक्री उत्पन्न करना", "ब्रांड वफादारी बनाना"],
  "primary_metric": "राजस्व",
  "content_structure": "ब्लॉग के साथ उत्पाद कैटलॉग",
  "content_types": ["उत्पाद पेज", "ब्लॉग पोस्ट", "ग्राहक की कहानियाँ"]
}
```

## अपनी site_brief देखना और अपडेट करना {#viewing-and-updating-your-sitebrief}

### अपनी site_brief देखें {#view-your-sitebrief}

एजेंट से पूछें:

```
"Show me my site brief"
```

या

```
"What do you know about my site?"
```

एजेंट आपकी संग्रहीत स्पेसिफिकेशन प्रदर्शित करेगा।

### अपनी site_brief अपडेट करें {#update-your-sitebrief}

यदि आपका साइट विकसित होता है, तो आप इसे अपडेट कर सकते हैं:

```
"Update my site brief: we're now targeting B2B customers"
```

या

```
"Refresh my site specification"
```

यह आपके मौजूदा जानकारी के साथ साइट स्पेसिफिकेशन कौशल को फिर से चलाएगा।

## एजेंट site_brief का उपयोग कैसे करते हैं {#how-agents-use-sitebrief}

### डिज़ाइन सुझाव (Design Recommendations) {#design-recommendations}

जब आप डिज़ाइन में बदलाव के लिए पूछते हैं, तो एजेंट आपकी site_brief का संदर्भ लेते हैं:

```
आप: "मेरी होमपेज को रीडिज़ाइन करें"
एजेंट: "आपके site brief के आधार पर, आप 25-45 वर्ष की महिलाओं को लक्षित कर रहे हैं
जिनका लहजा गर्मजोशी भरा और कारीगरी वाला है। मैं एक ऐसा डिज़ाइन बनाऊंगा जो
आपके हस्तनिर्मित गहनों और टिकाऊ मूल्यों पर ज़ोर देता है।"
```

### फीचर सुझाव (Feature Suggestions) {#feature-suggestions}

एजेंट आपके लक्ष्यों के अनुरूप फीचर्स का सुझाव देते हैं:

```
आप: "मुझे कौन से फीचर्स जोड़ने चाहिए?"
एजेंट: "ग्राहक वफादारी पर केंद्रित ई-कॉमर्स स्टोर के लिए,
मैं सुझाव दूंगा: लॉयल्टी प्रोग्राम, ग्राहक समीक्षाएं, ईमेल न्यूज़लेटर,
और उत्पाद सिफारिशें।"
```

### सामग्री संरचना (Content Structure) {#content-structure}

एजेंट आपकी संरचना के आधार पर सामग्री संगठन का प्रस्ताव देते हैं:

```
आप: "मुझे अपने ब्लॉग को कैसे व्यवस्थित करना चाहिए?"
एजेंट: "चूंकि आप एक उत्पाद कैटलॉग के साथ ब्लॉग का उपयोग करते हैं, मैं सुझाव दूंगा
कि पोस्ट को उत्पाद श्रेणी के अनुसार व्यवस्थित किया जाए और 'स्टाइलिंग गाइड'
बनाए जाएं जो एक साथ कई उत्पादों को प्रदर्शित करते हैं।"
```

## सर्वोत्तम अभ्यास (Best Practices) {#best-practices}

### विशिष्ट रहें (Be Specific) {#be-specific}

"सामान्य दर्शक" के बजाय, अपने वास्तविक दर्शकों का वर्णन करें:

- ✓ "25-45 वर्ष की महिलाएं, जो टिकाऊ फैशन में रुचि रखती हैं"
- ✗ "हर कोई"

### नियमित रूप से अपडेट करें (Update Regularly) {#update-regularly}

जैसे-जैसे आपका साइट विकसित होता है, अपनी site_brief को अपडेट करें:

- जब आप नए दर्शकों की ओर मुड़ते हैं
- जब आप नई उत्पाद लाइनें जोड़ते हैं
- जब आपकी ब्रांड पहचान बदलती है
- जब आपके व्यावसायिक लक्ष्य बदलते हैं

### सुसंगत शब्दावली का उपयोग करें (Use Consistent Terminology) {#use-consistent-terminology}

सत्रों में समान शब्दों का उपयोग करें:

- ✓ हमेशा "टिकाऊ गहने" कहें (न कि "पर्यावरण-अनुकूल गहने" और "हरित उत्पाद")
- ✓ अपने दर्शकों का लगातार एक ही तरीके से उल्लेख करें

### संदर्भ शामिल करें (Include Context) {#include-context}

ऐजेंटों को आपके निर्णय समझने में मदद करने के लिए पृष्ठभूमि प्रदान करें:

- "हम पेशेवरों को लक्षित कर रहे हैं जो कीमत से अधिक गुणवत्ता को महत्व देते हैं"
- "हमारे दर्शक तकनीकी रूप से जानकार हैं और आधुनिक डिज़ाइन की उम्मीद करते हैं"
- "हम एक बूटस्ट्रैप्ड स्टार्टअप हैं, इसलिए हमें लागत प्रभावी समाधानों की आवश्यकता है"

## थीम बिल्डर ऑनबोर्डिंग से संबंध {#relationship-to-theme-builder-onboarding}

साइट स्पेसिफिकेशन कौशल **Theme Builder onboarding flow** में एकीकृत है। जब आप ऑनबोर्डिंग पूरा करते हैं, तो आपके द्वारा प्रदान की गई जानकारी के साथ आपकी site_brief स्वचालित रूप से बन जाती है।

यदि आप चाहें तो आप साइट स्पेसिफिकेशन को स्वतंत्र रूप से भी चला सकते हैं:

- प्रारंभिक सेटअप के बाद अपनी स्पेसिफिकेशन को परिष्कृत करना
- जैसे-जैसे आपका साइट विकसित होता है, अपनी site_brief को अपडेट करना
- थीम बिल्डर शुरू करने से पहले एक विस्तृत स्पेसिफिकेशन बनाना

## समस्या निवारण (Troubleshooting) {#troubleshooting}

**मेरी site_brief का उपयोग नहीं किया जा रहा है**
- पुष्टि करें कि एजेंट के पास मेमोरी तक पहुंच है
- एजेंट से पूछें कि "recall my site brief"
- जांच करें कि सेटिंग्स में मेमोरी सक्षम है

**मैं एक नई site_brief के साथ शुरू करना चाहता हूँ**
- एजेंट से पूछें: "Clear my site brief and start fresh"
- फिर साइट स्पेसिफिकेशन को फिर से चलाएं

**एजेंट ऐसे सुझाव दे रहा है जो मेरी site_brief से मेल नहीं खाते**
- एजेंट से पूछें कि "review my site brief"
- यदि यह पुराना है तो अपनी site_brief को अपडेट करें
- अपने अनुरोधों में अतिरिक्त संदर्भ प्रदान करें

## अगले कदम (Next Steps) {#next-steps}

अपनी साइट स्पेसिफिकेशन को परिभाषित करने के बाद:

1. **थीम बिल्डर का उपयोग करें**: अपनी site_brief के आधार पर एक कस्टम थीम बनाएं
2. **डिज़ाइन को परिष्कृत करें**: विस्तृत डिज़ाइन कार्य के लिए Design System Aesthetics skill का उपयोग करें
3. **सामग्री की योजना बनाएं**: सामग्री संरचना के सुझावों के लिए एजेंटों से पूछें
4. **फीचर्स बनाएं**: अपने व्यावसायिक लक्ष्यों के अनुरूप फीचर्स का अनुरोध करें
