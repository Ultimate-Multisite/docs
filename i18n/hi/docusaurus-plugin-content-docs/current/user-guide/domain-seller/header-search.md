# हेडर डोमेन खोज

इस सेटअप का उपयोग तब करें जब आपको एक छोटा हेडर फ़ॉर्म चाहिए जो डोमेन खोज शुरू करे, फिर ग्राहक को Ultimate Multisite चेकआउट के अंदर उपलब्ध डोमेन में से चुनने दे।

## आवश्यकताएँ

- Ultimate Multisite नेटवर्क-सक्रिय।
- Multisite Ultimate Domain Seller नेटवर्क-सक्रिय।
- कम से कम एक सक्रिय डोमेन पंजीकरण उत्पाद जिसमें:
  - `domain_provider` किसी कॉन्फ़िगर किए गए प्रदाता पर सेट हो।
  - समर्थित TLDs कॉन्फ़िगर हों, उदाहरण के लिए `com`, `net`, और `org`।
- **डोमेन चयन** फ़ील्ड वाला एक मान्य चेकआउट फ़ॉर्म।

## चेकआउट फ़ॉर्म

1. पंजीकरण पेज द्वारा उपयोग किए जाने वाला चेकआउट फ़ॉर्म बनाएँ या संपादित करें।
2. सामान्य आवश्यक चेकआउट/Account फ़ील्ड जोड़ें, जिनमें **उपयोगकर्ता नाम** शामिल हो। केवल डोमेन फ़ील्ड वाला चेकआउट फ़ॉर्म Ultimate Multisite सत्यापन द्वारा अस्वीकार कर दिया जाता है।
3. एक **डोमेन चयन** फ़ील्ड जोड़ें।
4. जब प्रवाह का ध्यान मुफ़्त उपडोमेन या मौजूदा डोमेन के बजाय पंजीकृत डोमेन पर होना चाहिए, तो Domain Selection मोड को **केवल पंजीकरण** पर सेट करें।
5. उस फ़ील्ड को डोमेन पंजीकरण उत्पाद असाइन करें।

पंजीकरण पेज को चेकआउट फ़ॉर्म रेंडर करना चाहिए, उदाहरण के लिए:

```text
[wu_checkout slug="domain-form"]
```

## हेडर फ़ॉर्म

साइट हेडर में एक छोटा `GET` फ़ॉर्म जोड़ें जो दर्ज किए गए खोज शब्द को `domain_name` के रूप में चेकआउट पेज पर भेजता है:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

कस्टम हेडर JavaScript में डोमेन पहले से चयनित न करें। हेडर को केवल खोज शब्द पास करना चाहिए। Domain Seller चेकआउट स्क्रिप्ट `?domain_name=example` पढ़ती है, चेकआउट खोज बॉक्स भरती है, और उपलब्धता खोज चलाती है ताकि ग्राहक लौटाए गए डोमेन में से चुन सके।

## अपेक्षित व्यवहार

हेडर में `example` खोजने पर यह खुलना चाहिए:

```text
/register/?domain_name=example
```

इसके बाद चेकआउट में चुनने योग्य परिणाम दिखने चाहिए, जैसे:

- `example.com`
- `example.net`
- `example.org` अनुपलब्ध
- अन्य प्रदाता-समर्थित TLDs

उपलब्ध परिणाम चुनने के बाद, ऑर्डर सारांश में डोमेन पंजीकरण उत्पाद और चुना गया डोमेन नाम शामिल होना चाहिए।

## सत्यापन

1. होम पेज खोलें।
2. एक साधारण नाम खोजें, उदाहरण के लिए `example`।
3. पुष्टि करें कि चेकआउट URL में `?domain_name=example` शामिल है।
4. पुष्टि करें कि चेकआउट डोमेन फ़ील्ड में `example` है।
5. पुष्टि करें कि डोमेन विकल्पों की सूची दिखाई देती है।
6. उपलब्ध डोमेन के लिए **चुनें** पर क्लिक करें।
7. पुष्टि करें कि ऑर्डर सारांश में `Domain Registration - example.com` या चुना गया डोमेन शामिल है।

## समस्या निवारण

- यदि कोई सूची दिखाई नहीं देती है, तो `admin-ajax.php?action=wu_domain_search` के लिए ब्राउज़र नेटवर्क टैब जाँचें और पुष्टि करें कि यह खाली नहीं `domains` या `results` लौटाता है।
- यदि सेव करते समय चेकआउट फ़ॉर्म सत्यापन में विफल होता है, तो **उपयोगकर्ता नाम** जैसे आवश्यक Account फ़ील्ड जोड़ें।
- यदि डोमेन चुनने पर कार्ट अपडेट नहीं होता है, तो पुष्टि करें कि `window.wu_checkout_form` मौजूद है और Domain Seller चेकआउट एसेट्स चेकआउट पेज पर लोड हैं।
