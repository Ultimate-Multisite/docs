# Domain Seller: सेटअप और प्रदाता कॉन्फ़िगरेशन {#domain-seller-setup-and-provider-configuration}

Domain Seller ऐडऑन एक मार्गदर्शित सेटअप विज़ार्ड के साथ आता है, जो आपको हर ज़रूरी चरण से गुज़ारता है। यह पेज विज़ार्ड प्रवाह और बाद में प्रदाताओं को कॉन्फ़िगर या फिर से कॉन्फ़िगर करने का तरीका बताता है।

## आवश्यकताएँ {#requirements}

- **Multisite Ultimate** v2.4.12 या उससे ऊपर, नेटवर्क-सक्रिय
- **PHP** 7.4+
- कम से कम एक समर्थित रजिस्ट्रार के लिए API क्रेडेंशियल

## पहली बार चलने वाला सेटअप विज़ार्ड {#first-run-setup-wizard}

जब आप पहली बार plugin को नेटवर्क-सक्रिय करते हैं, तो सेटअप विज़ार्ड अपने-आप शुरू हो जाता है। यह किसी भी समय **Network Admin › Ultimate Multisite › Domain Seller Setup** से भी उपलब्ध है।

### चरण 1 — प्रदाता चुनें {#step-1--choose-a-provider}

वह रजिस्ट्रार चुनें जिसे आप जोड़ना चाहते हैं। समर्थित विकल्प:

| प्रदाता | DNS प्रबंधन | WHOIS गोपनीयता |
|---|---|---|
| OpenSRS | हाँ | हाँ |
| Namecheap | नहीं | हाँ (WhoisGuard, मुफ़्त) |
| HostAfrica | हाँ | हाँ (ID सुरक्षा) |
| Openprovider | हाँ | हाँ |
| Hostinger | होस्ट किए गए डोमेन के लिए मुख्य Hostinger डोमेन मैपिंग के ज़रिए | हाँ |
| GoDaddy | नहीं | नहीं |
| ResellerClub | हाँ | नहीं |
| NameSilo | नहीं | नहीं |
| Enom | हाँ | नहीं |

### चरण 2 — क्रेडेंशियल दर्ज करें {#step-2--enter-credentials}

हर प्रदाता के क्रेडेंशियल फ़ील्ड अलग होते हैं:

**OpenSRS** — उपयोगकर्ता नाम और निजी कुंजी (OpenSRS Reseller Control Panel से)

**Namecheap** — उपयोगकर्ता नाम और API key (Account › Tools › API Access से)

**HostAfrica** — HostAfrica रिसेलर मॉड्यूल से Domains Reseller API endpoint और क्रेडेंशियल। अभी कोई अलग sandbox endpoint दस्तावेज़ित नहीं है; लाइव पंजीकरण चलाने से पहले सुरक्षित रीड-ओनली जाँचों से परीक्षण करें।

**Openprovider** — API पहुँच सक्षम उपयोगकर्ता नाम और पासवर्ड। वैकल्पिक सैंडबॉक्स मोड Openprovider sandbox API का उपयोग करता है, और वैकल्पिक डिफ़ॉल्ट ग्राहक हैंडल को पंजीकरणों के लिए फिर से इस्तेमाल किया जा सकता है।

**Hostinger** — मुख्य Hostinger इंटीग्रेशन से साझा Hostinger hPanel API टोकन। यही टोकन मुख्य डोमेन मैपिंग और Domain Seller पंजीकरण कार्यों को चलाता है।

**GoDaddy** — API key और सीक्रेट (developer.godaddy.com से)

**ResellerClub** — रिसेलर ID और API key (ResellerClub control panel से)

**NameSilo** — API key (namesilo.com › Account › API Manager से)

**Enom** — Account ID और API टोकन

लाइव जाने से पहले प्रदाता के परीक्षण परिवेश में परीक्षण करने के लिए, जहाँ उपलब्ध हो वहाँ **सैंडबॉक्स मोड** चुनें।

### चरण 3 — कनेक्शन जाँचें {#step-3--test-the-connection}

**कनेक्शन जाँचें** पर क्लिक करें। विज़ार्ड क्रेडेंशियल और कनेक्टिविटी सत्यापित करने के लिए हल्की API कॉल भेजता है। आगे बढ़ने से पहले किसी भी क्रेडेंशियल समस्या को ठीक करें।

### चरण 4 — TLD आयात करें {#step-4--import-tlds}

जुड़े हुए प्रदाता से सभी उपलब्ध TLD और थोक मूल्य खींचने के लिए **TLD आयात करें** पर क्लिक करें। इससे डोमेन उत्पादों द्वारा उपयोग की जाने वाली TLD सूची भर जाती है। बड़े TLD कैटलॉग वाले प्रदाताओं के लिए आयात में 30–60 सेकंड लग सकते हैं।

TLD को निर्धारित cron जॉब के ज़रिए दिन में एक बार अपने-आप फिर से सिंक भी किया जाता है।

### चरण 5 — डोमेन उत्पाद बनाएँ {#step-5--create-a-domain-product}

विज़ार्ड 10% मार्कअप के साथ एक डिफ़ॉल्ट कैच-ऑल डोमेन उत्पाद बनाता है। आप इस उत्पाद को तुरंत संपादित कर सकते हैं या छोड़कर **Ultimate Multisite › Products** के अंतर्गत उत्पाद मैन्युअल रूप से बना सकते हैं।

पूरी उत्पाद कॉन्फ़िगरेशन गाइड के लिए [डोमेन उत्पाद और मूल्य निर्धारण](./domain-products) देखें।

## प्रदाता को फिर से कॉन्फ़िगर करना {#reconfiguring-a-provider}

**Network Admin › Ultimate Multisite › Settings › Domain Seller** पर जाएँ (या plugin सूची में **Settings** पर क्लिक करें)।

सेटिंग्स पेज में शामिल है:

- **डोमेन बिक्री सक्षम करें** — पूरी सुविधा को चालू/बंद करें
- **डिफ़ॉल्ट प्रदाता** — डोमेन खोजों और नए उत्पादों के लिए उपयोग किया जाने वाला प्रदाता
- **प्रति खोज अधिकतम TLD** — ग्राहक के खोजने पर कितने TLD जाँचे जाएँ; अधिक मान ज़्यादा विकल्प दिखाते हैं, लेकिन धीमे होते हैं
- **उपलब्धता कैश अवधि** — उपलब्धता और मूल्य परिणामों को कितनी देर कैश करना है; कम मान अधिक सटीक होते हैं, लेकिन API कॉल बढ़ाते हैं
- **डोमेन उत्पाद प्रबंधित करें** — उत्पाद सूची का त्वरित लिंक
- **प्रदाताओं को कॉन्फ़िगर करें** — प्रदाताओं को जोड़ने या फिर से कॉन्फ़िगर करने के लिए Integration Wizard खोलता है

### दूसरा प्रदाता जोड़ना {#adding-a-second-provider}

**प्रदाताओं को कॉन्फ़िगर करें** पर क्लिक करें और नए रजिस्ट्रार के लिए विज़ार्ड फिर से चलाएँ। आपके पास एक साथ कई प्रदाता कॉन्फ़िगर हो सकते हैं। हर डोमेन उत्पाद को किसी विशिष्ट प्रदाता को असाइन करें, या उसे डिफ़ॉल्ट पर छोड़ दें।

### TLD को मैन्युअल रूप से सिंक करना {#syncing-tlds-manually}

सेटिंग्स पेज में, नवीनतम मूल्य खींचने के लिए किसी भी कॉन्फ़िगर किए गए प्रदाता के आगे **TLD सिंक करें** पर क्लिक करें। यह तब उपयोगी है जब कोई प्रदाता थोक मूल्य अपडेट करता है या नए TLD जोड़ता है।

## प्रदाता क्षमता नोट्स {#provider-capability-notes}

हर रजिस्ट्रार API समान कार्य उपलब्ध नहीं कराता। ऐडऑन असमर्थित कार्यों को चुपचाप विफल करने के बजाय स्पष्ट व्यवस्थापक-समक्ष त्रुटियों के साथ दिखाता है।

- **HostAfrica** सबसे व्यापक लाइव पुनर्विक्रेता कार्यप्रवाह का समर्थन करता है, जिसमें खोज, TLD/मूल्य समन्वयन, पंजीकरण, नवीनीकरण, स्थानांतरण, नेमसर्वर अपडेट, DNS रिकॉर्ड, EPP कोड, रजिस्ट्रार लॉक और ID सुरक्षा शामिल हैं।
- **Openprovider** पुनर्विक्रेता-मूल्य वाले TLD समन्वयन, पंजीकरण, नवीनीकरण, स्थानांतरण, नेमसर्वर अपडेट, DNS ज़ोन, EPP कोड, रजिस्ट्रार लॉक और WHOIS गोपनीयता का समर्थन करता है। यह कम अवधि वाले bearer token से प्रमाणीकरण करता है, जिसे ऐड-ऑन अपने-आप ताज़ा करता है।
- **Hostinger** साझा hPanel API token के माध्यम से उपलब्धता खोज, पंजीकरण, पोर्टफोलियो खोज, नेमसर्वर अपडेट, रजिस्ट्रार लॉक और WHOIS गोपनीयता का समर्थन करता है। Hostinger की सार्वजनिक Domains API पुनर्विक्रेता/थोक मूल्य, इनबाउंड स्थानांतरण, स्पष्ट नवीनीकरण या EPP-code पुनर्प्राप्ति उपलब्ध नहीं कराती; नवीनीकरण केवल अपने-आप नवीनीकृत होते हैं।
