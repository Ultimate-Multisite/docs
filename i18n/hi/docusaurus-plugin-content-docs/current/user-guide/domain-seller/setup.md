# डोमेन सेलर: सेटअप और प्रोवाइडर कॉन्फ़िगरेशन

डोमेन सेलर ऐडऑन एक गाइडेड सेटअप विज़ार्ड के साथ आता है जो आपको हर ज़रूरी कदम से गुज़ारता है। यह पेज विज़ार्ड के फ्लो और बाद में प्रोवाइडर्स को कैसे कॉन्फ़िगर या रीकॉन्फ़िगर करना है, इसके बारे में बताता है।

## आवश्यकताएँ (Requirements)

- **Multisite Ultimate** v2.4.12 या इससे नया, नेटवर्क-एक्टिवेटेड
- **PHP** 7.4+
- कम से कम एक समर्थित रजिस्ट्रार के लिए API क्रेडेंशियल्स

## पहले रन का सेटअप विज़ार्ड (First-run setup wizard)

यह सेटअप विज़ार्ड पहली बार जब आप प्लगइन को नेटवर्क-एक्टिवेट करते हैं, तो अपने आप शुरू हो जाता है। यह **Network Admin › Ultimate Multisite › Domain Seller Setup** से किसी भी समय उपलब्ध है।

### स्टेप 1 — प्रोवाइडर चुनें (Choose a provider)

वह रजिस्ट्रार चुनें जिसे आप कनेक्ट करना चाहते हैं। समर्थित विकल्प:

| प्रोवाइडर | DNS प्रबंधन (DNS management) | WHOIS गोपनीयता (WHOIS privacy) |
|---|---|---|
| OpenSRS | हाँ (Yes) | हाँ (Yes) |
| Namecheap | नहीं (No) | हाँ (Yes) (WhoisGuard, मुफ़्त) |
| GoDaddy | नहीं (No) | नहीं (No) |
| ResellerClub | हाँ (Yes) | नहीं (No) |
| NameSilo | नहीं (No) | नहीं (No) |
| Enom | हाँ (Yes) | नहीं (No) |

### स्टेप 2 — क्रेडेंशियल्स दर्ज करें (Enter credentials)

हर प्रोवाइडर के लिए क्रेडेंशियल फ़ील्ड अलग-अलग होते हैं:

**OpenSRS** — यूज़रनेम और प्राइवेट की (OpenSRS Reseller Control Panel से)

**Namecheap** — यूज़रनेम और API की (Account › Tools › API Access से)

**GoDaddy** — API की और सीक्रेट (developer.godaddy.com से)

**ResellerClub** — रीसेलर ID और API की (ResellerClub कंट्रोल पैनल से)

**NameSilo** — API की (namesilo.com › Account › API Manager से)

**Enom** — अकाउंट ID और API टोकन

लाइव होने से पहले प्रोवाइडर के टेस्ट एनवायरनमेंट के खिलाफ टेस्ट करने के लिए जहाँ उपलब्ध हो, **Sandbox mode** को चेक करें।

### स्टेप 3 — कनेक्शन का परीक्षण करें (Test the connection)

**Test Connection** पर क्लिक करें। विज़ार्ड क्रेडेंशियल्स और कनेक्टिविटी को वेरिफाई करने के लिए एक हल्का API कॉल भेजता है। आगे बढ़ने से पहले किसी भी क्रेडेंशियल समस्या को ठीक करें।

### स्टेप 4 — TLDs आयात करें (Import TLDs)

कनेक्टेड प्रोवाइडर से सभी उपलब्ध TLDs और थोक मूल्य निर्धारण (wholesale pricing) खींचने के लिए **Import TLDs** पर क्लिक करें। यह डोमेन उत्पादों द्वारा उपयोग की जाने वाली TLD सूची को भरता है। बड़े TLD कैटलॉग वाले प्रोवाइडर्स के लिए इस आयात में 30–60 सेकंड लग सकते हैं।

TLDs को एक बार में स्वचालित रूप से एक शेड्यूल किए गए क्रॉन जॉब (scheduled cron job) के माध्यम से री-सिंक भी किया जाता है।

### स्टेप 5 — एक डोमेन उत्पाद बनाएँ (Create a domain product)

विज़ार्ड 10% मार्कअप के साथ एक डिफ़ॉल्ट कैच-ऑल डोमेन उत्पाद बनाता है। आप इस उत्पाद को तुरंत एडिट कर सकते हैं या इसे छोड़ सकते हैं और **Ultimate Multisite › Products** के तहत मैन्युअल रूप से उत्पाद बना सकते हैं।

पूरे उत्पाद कॉन्फ़िगरेशन गाइड के लिए [Domain Products and Pricing](./domain-products) देखें।

## प्रोवाइडर को रीकॉन्फ़िगर करना (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** पर जाएँ (या प्लगइन लिस्ट में **Settings** पर क्लिक करें)।

सेटिंग्स पेज में ये चीज़ें शामिल हैं:

- **Enable domain selling** — पूरी सुविधा को ऑन/ऑफ करने के लिए टॉगल करें
- **Default provider** — डोमेन खोज और नए उत्पादों के लिए उपयोग किया जाने वाला प्रोवाइडर
- **Max TLDs per search** — जब कोई ग्राहक खोज करता है तो कितने TLDs की जाँच की जाएगी; उच्च मान अधिक विकल्प दिखाते हैं लेकिन धीमे होते हैं
- **Availability cache duration** — उपलब्धता और मूल्य निर्धारण परिणामों को कितनी देर तक कैश करना है; कम मान अधिक सटीक होते हैं लेकिन API कॉल बढ़ाते हैं
- **Manage domain products** — उत्पादों की सूची का त्वरित लिंक
- **Configure providers** — प्रोवाइडर्स को जोड़ने या रीकॉन्फ़िगर करने के लिए इंटीग्रेशन विज़ार्ड खोलता है

### दूसरा प्रोवाइडर जोड़ना (Adding a second provider)

**Configure providers** पर क्लिक करें और नए रजिस्ट्रार के लिए विज़ार्ड को फिर से चलाएँ। आप एक साथ कई प्रोवाइडर्स कॉन्फ़िगर कर सकते हैं। प्रत्येक डोमेन उत्पाद को एक विशिष्ट प्रोवाइडर असाइन करें, या इसे डिफ़ॉल्ट पर छोड़ दें।

### TLDs को मैन्युअल रूप से सिंक करना (Syncing TLDs manually)

सेटिंग्स पेज पर, किसी भी कॉन्फ़िगर किए गए प्रोवाइडर के बगल में **Sync TLDs** पर क्लिक करें ताकि नवीनतम मूल्य निर्धारण प्राप्त किया जा सके। यह तब उपयोगी है जब कोई प्रोवाइडर थोक मूल्य निर्धारण अपडेट करता है या नए TLDs जोड़ता है।

## लॉग्स (Logs)

हर प्रोवाइडर अपने लॉग चैनल में लिखता है। लॉग्स **Network Admin › Ultimate Multisite › Logs** के तहत देखे जा सकते हैं:

| लॉग चैनल (Log channel) | सामग्री (Contents) |
|---|---|
| `domain-seller-registration` | सभी पंजीकरण प्रयास (सफलता और विफलता) |
| `domain-seller-renewal` | नवीनीकरण जॉब के परिणाम |
| `domain-seller-opensrs` | कच्ची OpenSRS API गतिविधि |
| `domain-seller-namecheap` | कच्ची Namecheap API गतिविधि |
| `domain-seller-godaddy` | कच्ची GoDaddy API गतिविधि |
| `domain-seller-resellerclub` | कच्ची ResellerClub API गतिविधि |
| `domain-seller-namesilo` | कच्ची NameSilo API गतिविधि |
| `domain-seller-enom` | कच्ची Enom API गतिविधि |
