# चेकआउट फ़ील्ड और कस्टमर DNS प्रबंधन

## डोमेन चयन चेकआउट फ़ील्ड

**Domain Selection** फ़ील्ड एक चेकआउट एलिमेंट है जो ग्राहकों को यह चुनने का विकल्प देता है कि वे अपने साइट का डोमेन कैसे प्राप्त करेंगे। डोमेन बेचने की सुविधा सक्षम करने के लिए इसे किसी भी चेकआउट फॉर्म में जोड़ें।

### चेकआउट फॉर्म में फ़ील्ड जोड़ना

1. **Network Admin › Ultimate Multisite › Checkout Forms** पर जाएं
2. एक चेकआउट फॉर्म खोलें या बनाएं
3. चेकआउट एडिटर में, **Add Field** पर क्लिक करें
4. फ़ील्ड सूची से **Domain Selection** चुनें
5. फ़ील्ड विकल्पों को कॉन्फ़िगर करें (नीचे देखें)
6. फॉर्म को सेव करें

### फ़ील्ड विकल्प

**Domain modes** — चुनें कि ग्राहक कौन से टैब देख सकते हैं। प्रत्येक मोड को स्वतंत्र रूप से सक्षम या अक्षम किया जा सकता है:

| Mode | यह क्या करता है |
|---|---|
| **Subdomain** | ग्राहक आपके नेटवर्क पर एक मुफ्त सबडोमेन का उपयोग करता है (उदाहरण के लिए, `mysite.yournetwork.com`)। इसके लिए भुगतान की आवश्यकता नहीं है। |
| **Register New Domain** | ग्राहक एक नया डोमेन खोजता है और इसे आपके कॉन्फ़िगर किए गए प्रोवाइडर के माध्यम से रजिस्टर करता है। मूल्य निर्धारण के लिए मिलान करने वाले डोमेन उत्पाद का उपयोग करता है। |
| **Existing Domain** | ग्राहक एक ऐसे डोमेन को मैप करता है जो पहले से ही उसके पास है। कोई पंजीकरण शुल्क नहीं। डोमेन स्वचालित रूप से उसकी साइट से मैप हो जाता है। |

**Default mode** — जब तीनों मोड सक्षम होते हैं, तो कौन सा टैब पहले खुलेगा। डोमेन पंजीकरण को वैकल्पिक रखने के लिए इसे **Subdomain** पर सेट करें, या खरीदारी को प्रोत्साहित करने के लिए **Register New Domain** पर सेट करें।

**Domain product** — वैकल्पिक रूप से इस फ़ील्ड को किसी विशिष्ट डोमेन उत्पाद से पिन करें। यदि सेट नहीं किया गया है, तो ऐडऑन स्वचालित रूप से ग्राहक द्वारा खोजे गए TLD के आधार पर मिलान करने वाले उत्पाद का चयन करता है।

### रजिस्ट्रेंट संपर्क फ़ील्ड

जब कोई ग्राहक **Register New Domain** टैब का चयन करता है, तो चेकआउट फॉर्म इनलाइन रजिस्ट्रेंट संपर्क फ़ील्ड जोड़ता है:

- पहला नाम / अंतिम नाम
- ईमेल पता
- पता (लाइन 1, शहर, राज्य/प्रांत, पिन कोड, देश)
- फ़ोन नंबर

ये सभी रजिस्ट्रारों द्वारा आवश्यक होते हैं और पंजीकरण API कॉल किए जाने से पहले मान्य किए जाते हैं। फ़ोन नंबरों को स्वचालित रूप से रजिस्ट्रारों द्वारा अपेक्षित `+CC.NNN` अंतर्राष्ट्रीय प्रारूप में फॉर्मेट किया जाता है।

### ऑटो-जनरेटेड साइट URL

जब कोई ग्राहक डोमेन रजिस्टर करता है या मैप करता है, तो साइट URL फ़ील्ड स्वचालित रूप से चुने गए डोमेन से पॉप्युलेट हो जाता है। ग्राहकों को अलग से URL फ़ील्ड भरने की आवश्यकता नहीं होती है।

### खोज व्यवहार

- ग्राहक द्वारा टाइप करते समय डोमेन की उपलब्धता AJAX के साथ वास्तविक समय में जाँची जाती है
- जब पसंदीदा डोमेन अनुपलब्ध होता है तो वैकल्पिक TLD सुझाव दिखाए जाते हैं
- मूल्य निर्धारण लाइव प्राप्त किया जाता है और स्पष्ट रूप से प्रदर्शित होता है (पंजीकरण मूल्य, नवीनीकरण मूल्य, वैकल्पिक WHOIS गोपनीयता शुल्क)
- कूपन कोड डोमेन उत्पादों पर किसी अन्य उत्पाद की तरह लागू होते हैं

**खोज प्रतिक्रियाशीलता को ट्यून करना:**

```php
// धीमी कनेक्शन पर API कॉल कम करने के लिए डिबाउंस डिले (मिलीसेकंड) बढ़ाएँ
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**डोमेन खोज फॉर्म में कस्टम फ़ील्ड जोड़ना:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## कस्टमर DNS प्रबंधन

ग्राहक **My Account** पेज से, अपने डोमेन की एंट्री के तहत, अपने पंजीकृत डोमेन के लिए DNS रिकॉर्ड प्रबंधित कर सकते हैं।

### समर्थित रिकॉर्ड प्रकार

| Type | उपयोग |
|---|---|
| **A** | होस्टनाम को IPv4 पते से मैप करें |
| **AAAA** | होस्टनाम को IPv6 पते से मैप करें |
| **CNAME** | एक अन्य होस्टनाम की ओर इंगित करने वाला उपनाम (alias) बनाएं |
| **MX** | मेल एक्सचेंज सर्वर सेट करें |
| **TXT** | SPF, DMARC, सत्यापन, या अन्य टेक्स्ट रिकॉर्ड जोड़ें |

### कौन से प्रोवाइडर DNS प्रबंधन का समर्थन करते हैं?

DNS प्रबंधन (रिकॉर्ड जोड़ना, संपादित करना, हटाना) **OpenSRS**, **ResellerClub**, और **Enom** के साथ उपलब्ध है। Namecheap, GoDaddy, और NameSilo डोमेन स्थिति और समाप्ति जानकारी दिखाते हैं, लेकिन DNS को रजिस्ट्रार के कंट्रोल पैनल में सीधे प्रबंधित किया जाना चाहिए।

### डिफ़ॉल्ट DNS रिकॉर्ड

आप डिफ़ॉल्ट DNS रिकॉर्ड कॉन्फ़िगर कर सकते हैं जो डोमेन पंजीकृत होने पर स्वचालित रूप से लागू हो जाते हैं। **Settings › Domain Seller › Default DNS Records** पर जाएं।

डिफ़ॉल्ट रिकॉर्ड मान दो टोकन का समर्थन करते हैं:

| Token | किससे बदला जाएगा |
|---|---|
| `{DOMAIN}` | पंजीकृत डोमेन नाम (उदाहरण के लिए, `example.com`) |
| `{SITE_URL}` | ग्राहक की साइट के लिए WordPress साइट URL |

**उदाहरण — एपेक्स डोमेन और www को आपके सर्वर IP पर पॉइंट करना:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### एडमिन: DNS देखना और संपादित करना

नेटवर्क एडमिन **Network Admin › Ultimate Multisite › Domains** में डोमेन की एडिट पेज से किसी भी ग्राहक डोमेन के DNS रिकॉर्ड देख और संपादित कर सकते हैं।
