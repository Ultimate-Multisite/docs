# चेकआउट क्षेत्र और ग्राहक DNS प्रबंधन {#checkout-field-and-customer-dns-management}

## डोमेन चयन चेकआउट क्षेत्र {#the-domain-selection-checkout-field}

**डोमेन चयन** क्षेत्र एक चेकआउट तत्व है, जो ग्राहकों को यह चुनने देता है कि वे अपनी साइट का डोमेन कैसे पाएँ। डोमेन बेचने की सुविधा चालू करने के लिए इसे किसी भी चेकआउट फ़ॉर्म में जोड़ें।

### चेकआउट फ़ॉर्म में क्षेत्र जोड़ना {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › Checkout Forms** पर जाएँ
2. कोई चेकआउट फ़ॉर्म खोलें या बनाएँ
3. चेकआउट संपादक में, **क्षेत्र जोड़ें** पर क्लिक करें
4. क्षेत्र सूची से **डोमेन चयन** चुनें
5. क्षेत्र विकल्प कॉन्फ़िगर करें (नीचे देखें)
6. फ़ॉर्म सहेजें

### क्षेत्र विकल्प {#field-options}

**डोमेन मोड** — चुनें कि ग्राहक कौन-से टैब देखेगा। हर मोड को अलग से चालू या बंद किया जा सकता है:

| मोड | यह क्या करता है |
|---|---|
| **सबडोमेन** | ग्राहक आपके नेटवर्क पर मुफ़्त सबडोमेन इस्तेमाल करता है (जैसे, `mysite.yournetwork.com`)। भुगतान की ज़रूरत नहीं। |
| **नया डोमेन पंजीकृत करें** | ग्राहक नया डोमेन खोजता है और आपके कॉन्फ़िगर किए गए प्रदाता के ज़रिए उसे पंजीकृत करता है। कीमत तय करने के लिए मेल खाते डोमेन उत्पाद का उपयोग होता है। |
| **मौजूदा डोमेन** | ग्राहक ऐसा डोमेन मैप करता है जिसका वह पहले से मालिक है। कोई पंजीकरण शुल्क नहीं। डोमेन अपने-आप उसकी साइट से मैप हो जाता है। |

**डिफ़ॉल्ट मोड** — जब तीनों मोड चालू हों, तो कौन-सा टैब पहले खुले। डोमेन पंजीकरण को वैकल्पिक रखने के लिए **सबडोमेन** सेट करें, या खरीदारी को बढ़ावा देने के लिए **नया डोमेन पंजीकृत करें** सेट करें।

**डोमेन उत्पाद** — वैकल्पिक रूप से इस क्षेत्र को किसी खास डोमेन उत्पाद से जोड़ें। अगर सेट नहीं है, तो ऐडऑन ग्राहक द्वारा खोजे गए TLD के आधार पर अपने-आप मेल खाता उत्पाद चुनता है।

### पंजीकरणकर्ता संपर्क क्षेत्र {#registrant-contact-fields}

जब ग्राहक **नया डोमेन पंजीकृत करें** टैब चुनता है, तो चेकआउट फ़ॉर्म पंजीकरणकर्ता संपर्क क्षेत्र वहीं जोड़ देता है:

- पहला नाम / अंतिम नाम
- ईमेल पता
- पता (पंक्ति 1, शहर, राज्य/प्रांत, डाक कोड, देश)
- फ़ोन नंबर

ये सभी रजिस्ट्रारों के लिए ज़रूरी हैं और पंजीकरण API कॉल होने से पहले मान्य किए जाते हैं। फ़ोन नंबर अपने-आप रजिस्ट्रारों द्वारा अपेक्षित `+CC.NNN` अंतरराष्ट्रीय प्रारूप में फ़ॉर्मैट किए जाते हैं।

### अपने-आप बनी साइट URL {#auto-generated-site-url}

जब ग्राहक कोई डोमेन पंजीकृत या मैप करता है, तो साइट URL क्षेत्र चुने गए डोमेन से अपने-आप भर जाता है। ग्राहकों को अलग URL क्षेत्र भरने की ज़रूरत नहीं होती।

### खोज व्यवहार {#search-behaviour}

- ग्राहक के टाइप करते समय AJAX से डोमेन उपलब्धता वास्तविक समय में जाँची जाती है
- पसंदीदा डोमेन उपलब्ध न होने पर वैकल्पिक TLD सुझाव दिखाए जाते हैं
- कीमत लाइव लाई जाती है और साफ़ दिखाई जाती है (पंजीकरण कीमत, नवीनीकरण कीमत, वैकल्पिक WHOIS गोपनीयता शुल्क)
- कूपन कोड डोमेन उत्पादों पर वैसे ही लागू होते हैं जैसे किसी अन्य उत्पाद पर

**खोज की प्रतिक्रिया गति समायोजित करना:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**डोमेन खोज फ़ॉर्म में कस्टम क्षेत्र जोड़ना:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ग्राहक DNS प्रबंधन {#customer-dns-management}

ग्राहक अपने पंजीकृत डोमेन के DNS रिकॉर्ड **मेरा Account** पेज से, अपने डोमेन की प्रविष्टि के तहत प्रबंधित कर सकते हैं।

### समर्थित रिकॉर्ड प्रकार {#supported-record-types}

| प्रकार | उपयोग |
|---|---|
| **A** | होस्टनाम को IPv4 पते से मैप करें |
| **AAAA** | होस्टनाम को IPv6 पते से मैप करें |
| **CNAME** | किसी दूसरे होस्टनाम की ओर इशारा करने वाला उपनाम बनाएँ |
| **MX** | मेल एक्सचेंज सर्वर सेट करें |
| **TXT** | SPF, DMARC, सत्यापन या अन्य टेक्स्ट रिकॉर्ड जोड़ें |

### कौन-से प्रदाता DNS प्रबंधन का समर्थन करते हैं? {#which-providers-support-dns-management}

DNS प्रबंधन (रिकॉर्ड जोड़ना, संपादित करना, हटाना) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, और **Openprovider** के साथ उपलब्ध है। **Hostinger** डोमेन Domain Seller के ज़रिए नेमसर्वर अपडेट कर सकते हैं; होस्ट किए गए डोमेन के DNS रिकॉर्ड मुख्य Hostinger डोमेन-मैपिंग एकीकरण से प्रबंधित होते हैं। Namecheap, GoDaddy, और NameSilo डोमेन स्थिति और समाप्ति जानकारी दिखाते हैं, लेकिन DNS को सीधे रजिस्ट्रार के नियंत्रण पैनल में प्रबंधित करना होगा।

### डिफ़ॉल्ट DNS रिकॉर्ड {#default-dns-records}

आप डिफ़ॉल्ट DNS रिकॉर्ड कॉन्फ़िगर कर सकते हैं, जो डोमेन पंजीकृत होने पर अपने-आप लागू होते हैं। **Settings › Domain Seller › Default DNS Records** पर जाएँ।

डिफ़ॉल्ट रिकॉर्ड मान दो टोकन का समर्थन करते हैं:

| टोकन | इससे बदला जाएगा |
|---|---|
| `{DOMAIN}` | पंजीकृत डोमेन नाम (जैसे, `example.com`) |
| `{SITE_URL}` | ग्राहक की साइट के लिए WordPress साइट URL |

**उदाहरण — एपेक्स डोमेन और www को अपने सर्वर IP पर इंगित करें:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Admin: DNS देखना और संपादित करना {#admin-viewing-and-editing-dns}

नेटवर्क Admin किसी भी ग्राहक डोमेन के DNS रिकॉर्ड को डोमेन के संपादन पेज से **Network Admin › Ultimate Multisite › Domains** में देख और संपादित कर सकते हैं।
