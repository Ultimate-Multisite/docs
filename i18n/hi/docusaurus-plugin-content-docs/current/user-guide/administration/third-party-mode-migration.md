# थर्ड-पार्टी मोड माइग्रेशन {#third-party-mode-migration}

Superdav AI Agent v1.12.0 थर्ड-पार्टी क्षमताओं (abilities) को संभालने के तरीके में बदलाव करता है। **थर्ड-पार्टी मोड अब डिफ़ॉल्ट रूप से ऑटो पर सेट होता है**, जिससे WordPress 7.0+ पर मैन्युअल कॉन्फ़िगरेशन के बिना ही नेटिव WordPress Abilities API इंटीग्रेशन संभव हो जाता है।

## क्या बदला है? {#what-changed}

### v1.12.0 से पहले {#before-v1120}

थर्ड-पार्टी क्षमताओं के लिए मैन्युअल कॉन्फ़िगरेशन की आवश्यकता होती थी:

- आपको स्पष्ट रूप से "थर्ड-पार्टी मोड" को सक्षम करना पड़ता था
- क्षमताओं को एक कस्टम रजिस्ट्री से लोड किया जाता था
- WordPress Abilities API के साथ इंटीग्रेशन वैकल्पिक था
- डिफ़ॉल्ट मोड लेगेसी (Legacy) था

### v1.12.0 के बाद {#after-v1120}

थर्ड-पार्टी क्षमताएं अपने आप काम करती हैं:

- थर्ड-पार्टी मोड डिफ़ॉल्ट रूप से "ऑटो" पर सेट होता है
- क्षमताएं सीधे WordPress Abilities API के साथ इंटीग्रेट होती हैं
- WordPress 7.0+ पर किसी मैन्युअल कॉन्फ़िगरेशन की ज़रूरत नहीं है
- पुराने WordPress संस्करणों के लिए लेगेसी मोड अभी भी उपलब्ध है

## किसे प्रभावित करता है? {#who-is-affected}

### नई इंस्टॉलेशन (WordPress 7.0+) {#new-installations-wordpress-70}

**आपको कुछ भी करने की ज़रूरत नहीं है।** थर्ड-पार्टी मोड स्वचालित रूप से "ऑटो" पर सेट हो जाता है, और क्षमताएं तुरंत काम करना शुरू कर देती हैं।

### मौजूदा इंस्टॉलेशन {#existing-installations}

**आपकी सेटिंग्स सुरक्षित हैं।** यदि आप पहले उपयोग कर रहे थे:

- **लेगेसी मोड (Legacy mode)**: आप लेगेसी मोड में ही रहेंगे (कोई बदलाव नहीं)
- **मैनुअल थर्ड-पार्टी मोड (Manual third-party mode)**: आप मैनुअल मोड में ही रहेंगे (कोई बदलाव नहीं)
- **ऑटो मोड (Auto mode)**: आप ऑटो मोड के साथ जारी रखेंगे (कोई बदलाव नहीं)

### 7.0 से पहले के WordPress संस्करण {#wordpress-versions-before-70}

**लेगेसी मोड अभी भी उपलब्ध है।** यदि आप WordPress 6.x या उससे पहले के संस्करण पर हैं:

- थर्ड-पार्टी मोड डिफ़ॉल्ट रूप से "लेगेसी" पर सेट होता है
- यदि आप चाहें तो आप थर्ड-पार्टी मोड को मैन्युअल रूप से सक्षम कर सकते हैं
- नेटिव Abilities API का उपयोग करने के लिए WordPress 7.0+ पर अपग्रेड करें

## मोड को समझना {#understanding-the-modes}

### ऑटो मोड (नया डिफ़ॉल्ट) {#auto-mode-new-default}

**ऑटो मोड** नेटिव WordPress Abilities API इंटीग्रेशन का उपयोग करता है:

- क्षमताएं WordPress हुक्स (hooks) के माध्यम से रजिस्टर होती हैं
- WordPress 7.0+ Abilities API के साथ पूर्ण अनुकूलता
- थर्ड-पार्टी क्षमताओं का स्वचालित पता लगाना (Automatic discovery)
- किसी मैन्युअल कॉन्फ़िगरेशन की ज़रूरत नहीं

**कब उपयोग करें**: WordPress 7.0+ के साथ थर्ड-पार्टी क्षमताओं के लिए

### मैनुअल मोड {#manual-mode}

**मैनुअल मोड** के लिए स्पष्ट कॉन्फ़िगरेशन की आवश्यकता होती है:

- आप निर्दिष्ट करते हैं कि कौन सी थर्ड-पार्टी क्षमताएं लोड होनी चाहिए
- यह टेस्टिंग या चुनिंदा क्षमता लोड करने के लिए उपयोगी है
- इसके लिए कॉन्फ़िगरेशन फ़ाइलों को एडिट करना ज़रूरी है
- अधिक नियंत्रण, लेकिन अधिक सेटअप

**कब उपयोग करें**: टेस्टिंग, चुनिंदा क्षमता लोड करना, या कस्टम कॉन्फ़िगरेशन के लिए

### लेगेसी मोड {#legacy-mode}

**लेगेसी मोड** पुराने थर्ड-पार्टी क्षमता प्रणाली का उपयोग करता है:

- कस्टम क्षमता रजिस्ट्री (WordPress Abilities API नहीं)
- पुराने WordPress संस्करणों के साथ बैकवर्ड कंपैटिबल
- कोई नेटिव WordPress इंटीग्रेशन नहीं
- अप्रचलित (Deprecated) लेकिन अभी भी समर्थित

**कब उपयोग करें**: WordPress 6.x या उससे पहले, या जब आपको लेगेसी कंपैटिबिलिटी की ज़रूरत हो

## अपना वर्तमान मोड जांचना {#checking-your-current-mode}

### एडमिन पैनल के माध्यम से {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** पर जाएं
2. **Third-Party Mode** सेटिंग देखें
3. आपको अपना वर्तमान मोड और इसे बदलने के विकल्प दिखाई देंगे

### कोड के माध्यम से {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', या 'legacy'
```

## अपना मोड बदलना {#changing-your-mode}

### ऑटो मोड पर स्विच करना {#switch-to-auto-mode}

यदि आप WordPress 7.0+ पर हैं और ऑटो मोड का उपयोग करना चाहते हैं:

1. **Superdav AI Agent** → **Settings** पर जाएं
2. **Third-Party Mode** ढूंढें
3. **Auto (WordPress Abilities API)** चुनें
4. **Save** पर क्लिक करें

Superdav AI Agent स्वचालित रूप से थर्ड-पार्टी क्षमताओं का पता लगाएगा और उन्हें रजिस्टर करेगा।

### मैनुअल मोड पर स्विच करना {#switch-to-manual-mode}

यदि आप मैन्युअल रूप से नियंत्रित करना चाहते हैं कि कौन सी क्षमताएं लोड हों:

1. **Superdav AI Agent** → **Settings** पर जाएं
2. **Third-Party Mode** ढूंढें
3. **Manual** चुनें
4. **Save** पर क्लिक करें
5. कौन सी क्षमताएं लोड करनी हैं, यह निर्दिष्ट करने के लिए अपनी कॉन्फ़िगरेशन फ़ाइल को एडिट करें

### लेगेसी मोड पर स्विच करना {#switch-to-legacy-mode}

यदि आपको लेगेसी कंपैटिबिलिटी की ज़रूरत है:

1. **Superdav AI Agent** → **Settings** पर जाएं
2. **Third-Party Mode** ढूंढें
3. **Legacy** चुनें
4. **Save** पर क्लिक करें

## ऑटो मोड के लाभ {#benefits-of-auto-mode}

### स्वचालित पता लगाना (Automatic Discovery) {#automatic-discovery}

क्षमताएं स्वचालित रूप से निम्न स्रोतों से पता लगाई जाती हैं:

- इंस्टॉल किए गए प्लगइन्स
- सक्रिय थीम
- मस्ट-यूज़ प्लगइन्स
- ड्रॉप-इन प्लगइन्स

मैन्युअल रजिस्ट्रेशन की ज़रूरत नहीं।

### नेटिव इंटीग्रेशन {#native-integration}

क्षमताएं WordPress Abilities API के साथ इंटीग्रेट होती हैं:

- WordPress कोर के साथ सुसंगत
- WordPress एडमिन के साथ काम करता है
- Abilities API का उपयोग करने वाले अन्य प्लगइन्स के साथ संगत
- जैसे-जैसे WordPress विकसित होता है, भविष्य के लिए तैयार

### सरल प्रबंधन {#simplified-management}

- एडिट करने के लिए कोई कॉन्फ़िगरेशन फ़ाइल नहीं
- कोई मैन्युअल क्षमता रजिस्ट्रेशन नहीं
- क्षमता विजिबिलिटी (Ability Visibility) नियंत्रण स्वचालित रूप से काम करते हैं
- एडमिन नोटिस आपको बिना वर्गीकृत क्षमताओं के बारे में सचेत करते हैं

### बेहतर प्रदर्शन {#better-performance}

- क्षमताओं को कैश किया जाता है
- ज़रूरत पड़ने पर आलसी-लोड (Lazy-loaded) होता है
- WordPress 7.0+ के लिए अनुकूलित

## माइग्रेशन पाथ {#migration-path}

### यदि आप WordPress 6.x पर हैं {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ पर अपग्रेड करें** (जब तैयार हों)
2. **Superdav AI Agent** को v1.12.0+ पर अपडेट करें
3. **थर्ड-पार्टी मोड को ऑटो पर बदलें** (वैकल्पिक; लेगेसी मोड अभी भी काम करता है)
4. उचित एक्सेस कंट्रोल सुनिश्चित करने के लिए **क्षमता विजिबिलिटी (ability visibility)** की समीक्षा करें

### यदि आप WordPress 7.0+ पर हैं {#if-youre-on-wordpress-70}

1. **Superdav AI Agent** को v1.12.0+ पर अपडेट करें
2. **पुष्टि करें कि थर्ड-पार्टी मोड ऑटो पर सेट है** (यह डिफ़ॉल्ट रूप से होना चाहिए)
3. उचित एक्सेस कंट्रोल सुनिश्चित करने के लिए **क्षमता विजिबिलिटी (ability visibility)** की समीक्षा करें
4. पुष्टि करने के लिए **थर्ड-पार्टी क्षमताओं का परीक्षण करें** कि वे काम करती हैं

## समस्या निवारण (Troubleshooting) {#troubleshooting}

### ऑटो मोड में क्षमताएं लोड नहीं हो रही हैं {#abilities-arent-loading-in-auto-mode}

- पुष्टि करें कि आप WordPress 7.0+ पर हैं
- जांचें कि थर्ड-पार्टी मोड "ऑटो" पर सेट है
- पुष्टि करें कि क्षमता प्रदान करने वाला प्लगइन सक्रिय है
- रजिस्ट्रेशन त्रुटियों के लिए WordPress एरर लॉग जांचें

### मैं लेगेसी मोड रखना चाहता हूँ {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** पर जाएं
- **Legacy** चुनें
- **Save** पर क्लिक करें
- लेगेसी मोड काम करना जारी रखेगा

### मेरी कस्टम क्षमताएं दिखाई नहीं दे रही हैं {#my-custom-abilities-arent-showing}

- पुष्टि करें कि वे WordPress हुक्स के माध्यम से रजिस्टर की गई हैं
- जांचें कि वे Abilities API को सही ढंग से लागू करती हैं
- WordPress एरर लॉग की समीक्षा करें
- सभी पंजीकृत क्षमताओं को देखने के लिए **Ability Visibility** एडमिन पेज का उपयोग करें

### मुझे "unclassified ability" नोटिस मिल रहे हैं {#im-getting-unclassified-ability-notices}

- यह नई थर्ड-पार्टी क्षमताओं के लिए सामान्य है
- एडमिन नोटिस में उनकी समीक्षा करें और उन्हें वर्गीकृत करें
- वर्गीकरण पर विवरण के लिए **Ability Visibility** देखें

## बैकवर्ड कंपैटिबिलिटी {#backward-compatibility}

### मौजूदा कॉन्फ़िगरेशन {#existing-configurations}

यदि आपके पास मौजूदा थर्ड-पार्टी क्षमता कॉन्फ़िगरेशन हैं:

- **लेगेसी मोड**: आपकी कॉन्फ़िगरेशन काम करना जारी रखेगी
- **मैनुअल मोड**: आपकी कॉन्फ़िगरेशन काम करना जारी रखेगी
- **ऑटो मोड**: आपकी कॉन्फ़िगरेशन को अनदेखा किया जाता है (ऑटो मोड नियंत्रण ले लेता है)

अपनी कस्टम कॉन्फ़िगरेशन को बनाए रखने के लिए, मैनुअल या लेगेसी मोड में रहें।

### अप्रचलन समयरेखा (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: लेगेसी और मैनुअल मोड अभी भी पूरी तरह समर्थित हैं
- **v1.13.0+**: लेगेसी मोड में अप्रचलन नोटिस दिखाई दे सकते हैं
- **v2.0.0**: लेगेसी मोड हटा दिया जा सकता है (TBD)

## सर्वोत्तम अभ्यास (Best Practices) {#best-practices}

### नई इंस्टॉलेशन के लिए {#for-new-installations}

- ऑटो मोड का उपयोग करें (यह डिफ़ॉल्ट है)
- Superdav AI Agent को क्षमताओं का स्वचालित रूप से पता लगाने दें
- एक्सेस को नियंत्रित करने के लिए Ability Visibility का उपयोग करें

### मौजूदा इंस्टॉलेशन के लिए {#for-existing-installations}

- जहाँ संभव हो, WordPress 7.0+ पर अपग्रेड करें
- सरल प्रबंधन के लिए ऑटो मोड पर स्विच करें
- Ability Visibility का उपयोग करके क्षमताओं की समीक्षा और वर्गीकरण करें

### कस्टम क्षमताओं के लिए {#for-custom-abilities}

- WordPress हुक्स (Abilities API) के माध्यम से क्षमताएं रजिस्टर करें
- कस्टम क्षमता रजिस्ट्री से बचें
- ऑटो मोड के साथ WordPress 7.0+ पर परीक्षण करें

## अगले कदम {#next-steps}

1. **अपना WordPress संस्करण जांचें**: ऑटो मोड के लिए पुष्टि करें कि आप 7.0+ पर हैं
2. **अपने थर्ड-पार्टी मोड की समीक्षा करें**: सेटिंग्स पर जाएं और अपना वर्तमान मोड जांचें
3. **आवश्यकतानुसार अपडेट करें**: यदि आप WordPress 7.0+ पर हैं तो ऑटो मोड पर स्विच करें
4. **क्षमताओं का वर्गीकरण करें**: किसी भी बिना वर्गीकृत क्षमता की समीक्षा करें और उन्हें वर्गीकृत करें
5. **परीक्षण करें**: पुष्टि करें कि आपकी थर्ड-पार्टी क्षमताएं सही ढंग से काम करती हैं

## संबंधित विषय {#related-topics}

- **Ability Visibility**: नियंत्रित करें कि कौन सी क्षमताएं कहाँ उजागर होती हैं
- **WordPress Abilities API**: नेटिव WordPress क्षमता रजिस्ट्रेशन के बारे में जानें
- **Third-Party Ability Development**: ऐसी क्षमताएं बनाएं जो ऑटो मोड के साथ काम करें
