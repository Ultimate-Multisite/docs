# एबिलिटी विजिबिलिटी (Ability Visibility)

Superdav AI Agent v1.12.0 में **एबिलिटी विजिबिलिटी** कंट्रोल पेश किए गए हैं, जो यह नियंत्रित करते हैं कि कौन से प्लेटफॉर्म (surfaces) कौन सी क्षमताएं (abilities) दिखाएंगे। यह एडमिनिस्ट्रेटर को यह ठीक से सेट करने की अनुमति देता है कि REST APIs, MCP सर्वर, WooCommerce इंटीग्रेशन, और अन्य इंटरफेस के माध्यम से कौन सी एजेंट क्षमताएं उपलब्ध होंगी।

## एबिलिटी विजिबिलिटी क्या है? {#what-is-ability-visibility}

एबिलिटी विजिबिलिटी एक परमिशन सिस्टम है जो नियंत्रित करता है:

- **कौन सी क्षमताएं** एजेंटों के लिए उपलब्ध हैं
- **वे कहाँ दिखाई देती हैं** (REST API, MCP, WooCommerce, आदि)
- **कौन उन्हें एक्सेस कर सकता है** (पार्टनर की अनुमति सूची/allow-lists के माध्यम से)
- **उनका वर्गीकरण कैसे होता है** (पहचाना गया/recognized बनाम अनक्लासिफाइड/unclassified)

हर क्षमता के साथ एक विजिबिलिटी लेवल जुड़ा होता है जो यह निर्धारित करता है कि वह अलग-अलग प्लेटफॉर्म पर कितनी उपलब्ध होगी।

## विजिबिलिटी लेवल (Visibility Levels) {#visibility-levels}

### पब्लिक (Public) {#public}

**पब्लिक एबिलिटीज** हर जगह उपलब्ध होती हैं:

- REST API एंडपॉइंट्स
- MCP सर्वर
- WooCommerce इंटीग्रेशन
- चैट इंटरफेस
- सभी यूजर रोल्स (उपयुक्त अनुमतियों के साथ)

उदाहरण: `scaffold-block-theme`, `activate-theme`, `send-email`

### इंटरनल (Internal) {#internal}

**इंटरनल एबिलिटीज** केवल आपकी WordPress इंस्टॉलेशन के अंदर उपलब्ध होती हैं:

- चैट इंटरफेस
- एडमिन पैनल
- केवल लॉग-इन किए हुए यूजर्स के लिए
- REST API या बाहरी इंटीग्रेशन के माध्यम से उजागर नहीं होती हैं

उदाहरण: `manage-settings`, `view-analytics`, `export-data`

### पार्टनर (Partner) {#partner}

**पार्टनर एबिलिटीज** केवल व्हाइटलिस्ट किए गए पार्टनर्स के लिए उपलब्ध होती हैं:

- इसके लिए स्पष्ट पार्टनर की अनुमति सूची में एंट्री की आवश्यकता होती है
- पार्टनर क्रेडेंशियल्स के साथ REST API के माध्यम से उपलब्ध
- पार्टनर ऑथेंटिकेशन के साथ MCP के माध्यम से उपलब्ध
- सामान्य यूजर्स के लिए उपलब्ध नहीं

उदाहरण: `bulk-import-users`, `modify-billing`, `access-analytics`

### डिसेबल्ड (Disabled) {#disabled}

**डिसेबल्ड एबिलिटीज** कहीं भी उपलब्ध नहीं होती हैं:

- किसी भी प्लेटफॉर्म के माध्यम से उजागर नहीं होती हैं
- चैट में उपलब्ध नहीं
- एडमिन पैनल में उपलब्ध नहीं
- पुरानी या प्रायोगिक (experimental) एबिलिटीज के लिए उपयोगी

## एबिलिटी विजिबिलिटी मैनेज करना {#managing-ability-visibility}

### एबिलिटी विजिबिलिटी सेटिंग्स तक पहुँचना {#accessing-ability-visibility-settings}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** पर जाएँ
2. **Abilities** टैब पर क्लिक करें
3. आपको सभी इंस्टॉल की गई एबिलिटीज की सूची उनके विजिबिलिटी लेवल के साथ दिखाई देगी

### एबिलिटी विवरण देखना {#viewing-ability-details}

किसी भी एबिलिटी पर क्लिक करके देखें:

- **Name**: एबिलिटी का पहचानकर्ता (identifier)
- **Description**: एबिलिटी क्या करती है
- **Current visibility**: पब्लिक, इंटरनल, पार्टनर, या डिसेबल्ड
- **Partner allow-list**: कौन से पार्टनर्स एक्सेस कर सकते हैं (यदि पार्टनर विजिबिलिटी है)
- **Last updated**: जब विजिबिलिटी आखिरी बार बदली गई थी
- **Status**: पहचाना गया (Recognized) या अनक्लासिफाइड (Unclassified)

### विजिबिलिटी लेवल बदलना {#changing-visibility-levels}

किसी एबिलिटी की विजिबिलिटी बदलने के लिए:

1. सूची में एबिलिटी पर क्लिक करें
2. ड्रॉपडाउन से एक नया विजिबिलिटी लेवल चुनें
3. यदि "Partner" चुन रहे हैं, तो अनुमति सूची (allow-list) में पार्टनर आइडेंटिफायर्स जोड़ें
4. **Save** पर क्लिक करें

उदाहरण:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### बल्क ऑपरेशन (Bulk Operations) {#bulk-operations}

कई एबिलिटीज के लिए विजिबिलिटी बदलने के लिए:

1. एबिलिटीज के बगल में चेकबॉक्स चुनें
2. बल्क एक्शन ड्रॉपडाउन से एक नया विजिबिलिटी लेवल चुनें
3. **Apply** पर क्लिक करें

## पार्टनर अलाउ-लिस्ट (Partner Allow-List) {#partner-allow-list}

**पार्टनर अलाउ-लिस्ट** यह नियंत्रित करती है कि कौन से बाहरी पार्टनर्स पार्टनर-लेवल की एबिलिटीज को एक्सेस कर सकते हैं।

### पार्टनर्स जोड़ना {#adding-partners}

1. **Superdav AI Agent** → **Settings** → **Partners** पर जाएँ
2. **Add Partner** पर क्लिक करें
3. पार्टनर आइडेंटिफायर दर्ज करें (आमतौर पर एक API कुंजी या संगठन ID)
4. वैकल्पिक रूप से एक पार्टनर नाम और विवरण जोड़ें
5. **Save** पर क्लिक करें

### पार्टनर्स को एबिलिटीज असाइन करना {#assigning-abilities-to-partners}

एक पार्टनर जोड़ने के बाद:

1. **Abilities** टैब पर जाएँ
2. पार्टनर विजिबिलिटी वाली कोई एबिलिटी खोजें
3. एडिट करने के लिए क्लिक करें
4. "Partner allow-list" सेक्शन में, उन पार्टनर्स के लिए चेकबॉक्स चुनें जिन्हें एक्सेस होना चाहिए
5. **Save** पर क्लिक करें

### पार्टनर एक्सेस रद्द करना (Revoking Partner Access) {#revoking-partner-access}

किसी पार्टनर का एक्सेस हटाने के लिए:

1. एबिलिटी को एडिट करें
2. अलाउ-लिस्ट में पार्टनर के बॉक्स को अनचेक करें
3. **Save** पर क्लिक करें

पार्टनर को तुरंत उस एबिलिटी का एक्सेस नहीं मिलेगा।

## अनक्लासिफाइड एबिलिटीज (Unclassified Abilities) {#unclassified-abilities}

जब आप कोई थर्ड-पार्टी एबिलिटी इंस्टॉल करते हैं जिसे Superdav AI Agent नहीं पहचानता है, तो उसे **अनक्लासिफाइड** के रूप में चिह्नित किया जाता है।

### अनक्लासिफाइड एबिलिटीज के लिए एडमिन नोटिस {#admin-notice-for-unclassified-abilities}

आपको एक एडमिन नोटिस दिखाई देगा:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### अनक्लासिफाइड एबिलिटीज की समीक्षा करना {#reviewing-unclassified-abilities}

1. नोटिस में **Review Abilities** पर क्लिक करें
2. प्रत्येक अनक्लासिफाइड एबिलिटी के लिए, आपको यह दिखाई देगा:
   - एबिलिटी का नाम और विवरण
   - स्रोत (किस प्लगइन/एडऑन ने इसे इंस्टॉल किया)
   - सुझाया गया विजिबिलिटी लेवल
   - सुरक्षा मूल्यांकन (Security assessment)

3. एक विजिबिलिटी लेवल चुनें:
   - **Public**: एबिलिटी पर भरोसा है; इसे हर जगह उजागर करें
   - **Internal**: केवल आंतरिक उपयोग तक सीमित करें
   - **Partner**: विशिष्ट पार्टनर्स तक सीमित करें
   - **Disabled**: इस एबिलिटी का उपयोग न करें

4. अपना निर्णय सहेजने के लिए **Classify** पर क्लिक करें

### अनक्लासिफाइड एबिलिटीज को क्लासिफाई क्यों करें? {#why-classify-unclassified-abilities}

एबिलिटीज को क्लासिफाई करने से:

- **सुरक्षा में सुधार होता है**: आप स्पष्ट रूप से अनुमोदित करते हैं कि क्या उजागर किया गया है
- **दुर्घटनाओं से बचाव होता है**: अनक्लासिफाइड एबिलिटीज डिफ़ॉल्ट रूप से डिसेबल होती हैं
- **फीचर्स सक्षम होते हैं**: एक बार क्लासिफाई होने के बाद, एबिलिटी उपलब्ध हो जाती है
- **निर्णयों का दस्तावेजीकरण होता है**: आपके विकल्प ऑडिट उद्देश्यों के लिए लॉग किए जाते हैं

## विभिन्न प्लेटफॉर्म पर विजिबिलिटी {#visibility-in-different-surfaces}

### REST API {#rest-api}

**पब्लिक एबिलिटीज** REST एंडपॉइंट्स के माध्यम से उपलब्ध होती हैं:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**इंटरनल और पार्टनर एबिलिटीज** REST API के माध्यम से उपलब्ध नहीं होती हैं।

### MCP सर्वर {#mcp-servers}

**पब्लिक एबिलिटीज** MCP के माध्यम से उपलब्ध होती हैं:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**इंटरनल एबिलिटीज** MCP के माध्यम से उजागर नहीं होती हैं।

**पार्टनर एबिलिटीज** केवल पार्टनर क्रेडेंशियल्स के साथ उपलब्ध होती हैं।

### WooCommerce इंटीग्रेशन {#woocommerce-integration}

WooCommerce से संबंधित **पब्लिक एबिलिटीज** उपलब्ध होती हैं:

- प्रोडक्ट प्रबंधन
- ऑर्डर प्रोसेसिंग
- ग्राहक संचार

**इंटरनल एबिलिटीज** WooCommerce को उजागर नहीं होती हैं।

### चैट इंटरफेस {#chat-interface}

**सभी एबिलिटीज** (पब्लिक, इंटरनल, पार्टनर) उपयोगकर्ता की अनुमतियों के आधार पर चैट में उपलब्ध होती हैं:

- एडमिनिस्ट्रेटर सभी एबिलिटीज देखते हैं
- सामान्य यूजर्स केवल पब्लिक एबिलिटीज देखते हैं
- पार्टनर्स पब्लिक + पार्टनर एबिलिटीज देखते हैं (यदि व्हाइटलिस्टेड हैं)

## सुरक्षा सर्वोत्तम अभ्यास (Security Best Practices) {#security-best-practices}

### सबसे कम विशेषाधिकार का सिद्धांत (Principle of Least Privilege) {#principle-of-least-privilege}

- एबिलिटीज को सबसे प्रतिबंधात्मक विजिबिलिटी पर सेट करें जो उनके उपयोग की अनुमति देती हो
- संवेदनशील ऑपरेशंस के लिए पार्टनर विजिबिलिटी का उपयोग करें
- उन एबिलिटीज को डिसेबल करें जिनका आप उपयोग नहीं करते हैं

### नियमित ऑडिट (Regular Audits) {#regular-audits}

- एबिलिटी विजिबिलिटी की मासिक समीक्षा करें
- अनक्लासिफाइड एबिलिटीज की जाँच करें
- अनुपयोगी पार्टनर्स के लिए एक्सेस हटा दें

### लॉगिंग और मॉनिटरिंग (Logging and Monitoring) {#logging-and-monitoring}

- मॉनिटर करें कि कौन सी एबिलिटीज सबसे अधिक उपयोग की जाती हैं
- पार्टनर एक्सेस पैटर्न ट्रैक करें
- असामान्य एबिलिटी उपयोग पर अलर्ट करें

### थर्ड-पार्टी एबिलिटीज (Third-Party Abilities) {#third-party-abilities}

- सक्षम करने से पहले थर्ड-पार्टी एबिलिटीज की समीक्षा करें
- उन्हें स्पष्ट रूप से क्लासिफाई करें
- इंटरनल या पार्टनर विजिबिलिटी से शुरुआत करें
- केवल जांच के बाद पब्लिक में बढ़ावा दें

## समस्या निवारण (Troubleshooting) {#troubleshooting}

**सूची में कोई एबिलिटी दिखाई नहीं दे रही है**
- सत्यापित करें कि एबिलिटी इंस्टॉल और सक्रिय है
- जांचें कि प्लगइन/एडऑन सक्षम है
- पेज को रीफ़्रेश करें

**मैं किसी एबिलिटी की विजिबिलिटी नहीं बदल पा रहा हूँ**
- सत्यापित करें कि आपके पास एडमिनिस्ट्रेटर अनुमतियाँ हैं
- जांचें कि एबिलिटी किसी प्लगइन द्वारा लॉक तो नहीं है
- विरोधाभासी (conflicting) प्लगइन्स को डिसेबल करने का प्रयास करें

**एक पार्टनर एबिलिटी एक्सेस नहीं कर पा रहा है**
- सत्यापित करें कि पार्टनर अलाउ-लिस्ट में है
- जांचें कि एबिलिटी की विजिबिलिटी पार्टनर पर सेट है
- पुष्टि करें कि पार्टनर के क्रेडेंशियल्स सही हैं
- प्रमाणीकरण त्रुटियों (authentication errors) के लिए API लॉग की जाँच करें

**अनक्लासिफाइड एबिलिटीज दिखाई देती रहती हैं**
- एडमिन नोटिस में उनकी समीक्षा करें और उन्हें क्लासिफाई करें
- जांचें कि आपका वर्गीकरण सहेजा गया था
- सत्यापित करें कि एबिलिटी प्रदान करने वाला प्लगइन अपडेट है

## लेगेसी मोड से माइग्रेशन (Migration from Legacy Mode) {#migration-from-legacy-mode}

यदि आप Superdav AI Agent के पुराने संस्करण से अपग्रेड कर रहे हैं:

- सभी मौजूदा एबिलिटीज स्वचालित रूप से पब्लिक के रूप में क्लासिफाई हो जाती हैं
- आवश्यकतानुसार विजिबिलिटी की समीक्षा करें और समायोजित करें
- कोई कार्रवाई आवश्यक नहीं है जब तक कि आप एक्सेस को प्रतिबंधित नहीं करना चाहते हैं

ट्रांजिशन के बारे में अधिक विवरण के लिए **Third-Party Mode Migration** देखें।

## अगले कदम (Next Steps) {#next-steps}

एबिलिटी विजिबिलिटी कॉन्फ़िगर करने के बाद:

1. **अपनी एबिलिटीज की समीक्षा करें**: सभी इंस्टॉल की गई एबिलिटीज का ऑडिट करें
2. **अनक्लासिफाइड एबिलिटीज को क्लासिफाई करें**: एडमिन नोटिस का जवाब दें
3. **पार्टनर्स सेट करें**: यदि पार्टनर विजिबिलिटी का उपयोग कर रहे हैं तो पार्टनर्स जोड़ें
4. **उपयोग की निगरानी करें**: ट्रैक करें कि कौन सी एबिलिटीज सबसे अधिक उपयोग की जाती हैं
5. **निर्णयों का दस्तावेजीकरण करें**: यह नोट्स रखें कि आपने प्रत्येक विजिबिलिटी लेवल क्यों चुना
