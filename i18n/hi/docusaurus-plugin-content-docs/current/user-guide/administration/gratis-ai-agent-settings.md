# Gratis AI Agent सेटिंग्स

Gratis AI Agent में **Settings → Advanced** स्क्रीन बैकएंड इंटीग्रेशन के लिए प्रशासक-स्तर का कॉन्फ़िगरेशन देती है। यह पेज फ़ीडबैक फ़ॉरवर्डिंग, खोज प्रदाता कुंजियाँ, प्रबंधित Superdav सेवा सेटअप, Google Calendar नियंत्रण, TextBee SMS सेटिंग्स, और पूरे नेटवर्क के फ़ीचर फ़्लैग का दस्तावेज़ है।

## सेटिंग्स तक पहुँचना

1. WordPress एडमिन में, **Gratis AI Agent → Settings** पर जाएँ।
2. **Advanced** टैब पर क्लिक करें।

## फ़ीडबैक Endpoint कॉन्फ़िगरेशन

फ़ीडबैक endpoint AI agent से POST अनुरोध प्राप्त करता है जब भी कोई उपयोगकर्ता thumbs-down बटन, ऑटो-प्रॉम्प्ट बैनर, या `/report-issue` कमांड के ज़रिए फ़ीडबैक जमा करता है।

| फ़ील्ड | विवरण |
|---|---|
| **फ़ीडबैक Endpoint URL** | वह URL जो JSON बॉडी वाले HTTP POST अनुरोधों के रूप में फ़ीडबैक सबमिशन प्राप्त करता है। |
| **Feedback API Key** | प्रत्येक फ़ीडबैक अनुरोध के `Authorization` header में भेजा गया bearer token। यदि आपके endpoint को प्रमाणीकरण की आवश्यकता नहीं है, तो खाली छोड़ दें। |

### अपेक्षित JSON पेलोड

आपके फ़ीडबैक endpoint को कम-से-कम निम्नलिखित फ़ील्ड वाली JSON बॉडी स्वीकार करनी होगी:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

बातचीत के संदर्भ के आधार पर पेलोड में अतिरिक्त फ़ील्ड मौजूद हो सकते हैं।

### `triage_category` मान

AI ट्रायज परत पेलोड फ़ॉरवर्ड करने से पहले `triage_category` को निम्न में से एक मान देती है:

| मान | अर्थ |
|---|---|
| `factual_error` | assistant ने गलत तथ्यात्मक जानकारी दी। |
| `unhelpful_answer` | जवाब तकनीकी रूप से सही था, लेकिन उपयोगी नहीं था। |
| `inappropriate_content` | जवाब में ऐसा सामग्री था जिसे उपयोगकर्ताओं को नहीं दिखाया जाना चाहिए। |
| `other` | फ़ीडबैक किसी ज्ञात श्रेणी से मेल नहीं खाता था। |

### प्रमाणीकरण

यदि आपके endpoint को प्रमाणीकरण की आवश्यकता है, तो **Feedback API Key** फ़ील्ड को अपने bearer token पर सेट करें। agent भेजता है:

```
Authorization: Bearer <your-api-key>
```

यदि **Feedback API Key** फ़ील्ड खाली है, तो कोई `Authorization` header नहीं भेजा जाता।

### फ़ीडबैक संग्रह बंद करना

**फ़ीडबैक Endpoint URL** और **Feedback API Key** दोनों फ़ील्ड खाली छोड़ दें। thumbs-down बटन और फ़ीडबैक UI उपयोगकर्ताओं को दिखाई देते रहेंगे, लेकिन सबमिशन किसी बाहरी सेवा को फ़ॉरवर्ड नहीं किए जाएँगे।

## Brave Search API Key

**Advanced** टैब पर ही, **Brave Search API Key** फ़ील्ड [इंटरनेट खोज](../configuration/internet-search) क्षमता सक्षम करता है।

| फ़ील्ड | विवरण |
|---|---|
| **Brave Search API Key** | Brave Search डेवलपर Dashboard से आपकी API कुंजी। AI assistant में इंटरनेट खोज सक्षम करने के लिए आवश्यक। |

फ़ील्ड लेबल में Brave Search API साइन-अप पेज का क्लिक करने योग्य लिंक शामिल है। इंटरनेट खोज बंद करने के लिए खाली छोड़ दें।

इस फ़ीचर पर अंतिम-उपयोगकर्ता दस्तावेज़ के लिए [इंटरनेट खोज](../configuration/internet-search) देखें।

## प्रबंधित Superdav सेवा

Superdav AI Agent v1.18.0 समर्थित साइटों के लिए प्रबंधित Superdav सेवा endpoints और स्वचालित कनेक्शन प्रावधान जोड़ता है। इन नियंत्रणों का उपयोग तब करें जब आपकी साइट को मैन्युअल रूप से कॉन्फ़िगर किए गए सेवा endpoint के बजाय होस्टेड प्रदाता से जुड़ना चाहिए।

| फ़ील्ड | विवरण |
|---|---|
| **प्रबंधित Superdav सेवा** | समर्थित साइटों के लिए होस्टेड Superdav सेवा कनेक्शन सक्षम करता है। |
| **कनेक्शन प्रावधान** | स्वचालित endpoint और क्रेडेंशियल प्रावधान शुरू करता है। साइट को प्रबंधित प्रदाता का उपयोग करना चाहिए, इसकी पुष्टि करने के बाद इसका उपयोग करें। |
| **सेवा Endpoint / कनेक्शन स्थिति** | प्रावधान के बाद मौजूदा endpoint या कनेक्शन स्थिति दिखाता है। |

प्रावधान के बाद, सेटिंग्स सहेजें और प्रबंधित-सेवा वर्कफ़्लो पर निर्भर होने से पहले कनेक्शन स्थिति सत्यापित करें। यदि प्रावधान विफल होता है, तो दिखाई गई पुनः प्रयास मार्गदर्शिका की समीक्षा करें और पुष्टि करें कि साइट को होस्टेड प्रदाता का उपयोग करने की अनुमति है।

## Google Calendar कॉन्फ़िगरेशन

जब Superdav AI Agent v1.18.0 कैलेंडर फ़ीचर सक्षम होते हैं, तो agent कॉन्फ़िगर किए गए कैलेंडर और इवेंट विवरण पढ़ सकता है। कैलेंडर टूल पढ़ने-केंद्रित हैं और शेड्यूल-सचेत रिमाइंडर, सहभागी फ़ॉलो-अप, और संपर्क मिलान के लिए उपयोगी हैं।

| फ़ील्ड | विवरण |
|---|---|
| **Google Calendar क्रेडेंशियल** | कैलेंडर डेटा पढ़ने के लिए आवश्यक क्रेडेंशियल या token कनेक्शन संग्रहीत करता है। |
| **कैलेंडर चयन** | सीमित करता है कि agent कौन-से कॉन्फ़िगर किए गए कैलेंडर देख सकता है। |
| **कैलेंडर कनेक्शन स्थिति** | पुष्टि करता है कि मौजूदा क्रेडेंशियल कैलेंडर और इवेंट पढ़ सकते हैं या नहीं। |

कैलेंडर क्रेडेंशियल को केवल उन कैलेंडर तक सीमित रखें जिनकी agent को आवश्यकता है। यदि स्थिति समाप्त token दिखाती है, तो क्रेडेंशियल को फिर से कनेक्ट या रोटेट करें।

## TextBee SMS सूचनाएँ

Superdav AI Agent v1.18.0 कॉन्फ़िगर किए गए सूचना वर्कफ़्लो के लिए TextBee को SMS प्रदाता के रूप में जोड़ता है। संवेदनशील या उपयोगकर्ता-सामने संदेशों के लिए SMS सूचनाओं को मानव अनुमोदन गेट के साथ जोड़ा जाना चाहिए।

| फ़ील्ड | विवरण |
|---|---|
| **TextBee API Key** | TextBee SMS प्रदाता को भेजे गए अनुरोधों को प्रमाणित करता है। |
| **TextBee डिवाइस / प्रेषक** | प्रदाता द्वारा आवश्यक होने पर, आउटगोइंग संदेशों के लिए उपयोग किया जाने वाला TextBee प्रेषक या डिवाइस चुनता है। |
| **SMS सूचनाएँ सक्षम** | स्वीकृत वर्कफ़्लो को टेक्स्ट-संदेश सूचनाएँ भेजने देता है। SMS भेजने से रोकने के लिए बंद रखें। |

केवल प्रशासक-स्वामित्व वाले नंबर पर परीक्षण संदेश भेजें, फिर निर्धारित या सहभागी-सामने रिमाइंडर सक्षम करने से पहले अनुमोदन-गेट व्यवहार की पुष्टि करें।

## फ़ीचर फ़्लैग

v1.9.0 में भी पेश किया गया, **Settings → Feature Flags** टैब वैकल्पिक कार्यक्षमता के लिए टॉगल स्विच देता है। प्रत्येक फ़्लैग पूरे नेटवर्क में या तो सक्षम होता है या बंद; इस समय प्रति-साइट ओवरराइड नहीं है।

### फ़ीचर फ़्लैग तक पहुँचना

1. WordPress एडमिन में, **Gratis AI Agent → Settings** पर जाएँ।
2. **Feature Flags** टैब पर क्लिक करें।

### पहुँच नियंत्रण फ़्लैग

| फ़्लैग | डिफ़ॉल्ट | विवरण |
|---|---|---|
| **प्रशासकों तक सीमित करें** | बंद | सक्षम होने पर, केवल `administrator` भूमिका वाले उपयोगकर्ता AI Agent चैट पैनल खोल सकते हैं। अन्य सभी भूमिकाओं को इसके बजाय "अपने प्रशासक से संपर्क करें" संदेश दिखाई देता है। |
| **Network Admins तक सीमित करें** | बंद | किसी Multisite नेटवर्क पर सक्षम होने पर, केवल Super Admins एजेंट का उपयोग कर सकते हैं। अलग-अलग साइट प्रशासक अवरुद्ध रहते हैं। यदि दोनों सक्षम हों, तो "प्रशासकों तक सीमित करें" पर इसे प्राथमिकता मिलती है। |
| **Subscriber पहुँच की अनुमति दें** | बंद | सक्षम होने पर, `subscriber` भूमिका वाले उपयोगकर्ता चैट इंटरफ़ेस का उपयोग कर सकते हैं, लेकिन वे केवल पढ़ने की क्षमताओं तक सीमित रहते हैं (पोस्ट निर्माण या सेटिंग बदलाव नहीं)। |
| **गैर-सदस्यों के लिए बंद करें** | बंद | Ultimate Multisite सदस्यता स्थिति के साथ एकीकृत होता है। सक्षम होने पर, जिन साइटों की सक्रिय सदस्यता नहीं है, उनके लिए चैट छिपा दी जाती है। |

### ब्रांडिंग फ़्लैग

| फ़्लैग | डिफ़ॉल्ट | विवरण |
|---|---|---|
| **"Powered by Gratis AI Agent" फ़ुटर छिपाएँ** | बंद | चैट विजेट के नीचे दिखाई जाने वाली ब्रांडिंग श्रेय पंक्ति हटाता है। व्हाइट-लेबल डिप्लॉयमेंट के लिए अनुशंसित। |
| **कस्टम एजेंट नाम** | *(खाली)* | चैट हेडर और admin मेन्यू में डिफ़ॉल्ट "Gratis AI Agent" लेबल को आपके अपने उत्पाद नाम से बदलता है। डिफ़ॉल्ट उपयोग करने के लिए खाली छोड़ें। |
| **एजेंट पिकर छिपाएँ** | बंद | सक्षम होने पर, उपयोगकर्ता पाँच बिल्ट-इन एजेंटों के बीच स्विच नहीं कर सकते। मौजूदा एजेंट वही रहता है जो Settings → General में डिफ़ॉल्ट के रूप में कॉन्फ़िगर किया गया है। |
| **चैट अवतार के रूप में साइट आइकन उपयोग करें** | बंद | चैट विजेट हेडर में डिफ़ॉल्ट AI आइकन को WordPress साइट आइकन से बदलता है (Appearance → Customize → Site Identity के अंतर्गत सेट)। |

### स्वचालन सुरक्षा फ़्लैग

Superdav AI Agent v1.18.0 अधिक सुरक्षित स्वचालन चलाने के लिए मानव अनुमोदन गेट और रिमाइंडर रिकॉर्ड पेश करता है। इंस्टॉल किए गए पैकेज के आधार पर, ये नियंत्रण सुविधा फ़्लैग या उन्नत स्वचालन सेटिंग्स में दिखाई दे सकते हैं।

| फ़्लैग | डिफ़ॉल्ट | विवरण |
|---|---|---|
| **मानव अनुमोदन आवश्यक** | चालू अनुशंसित | संवेदनशील स्वचालन को तब तक रोकता है जब तक कोई अधिकृत उपयोगकर्ता प्रस्तावित कार्रवाई की समीक्षा और पुष्टि नहीं कर देता। |
| **रिमाइंडर दोहराव रोकना** | चालू | भेजे गए रिमाइंडर रिकॉर्ड करता है ताकि पुनर्प्रयास या निर्धारित रन डुप्लिकेट सूचनाएँ न भेजें। |
| **कैलेंडर टूल सक्षम करें** | कॉन्फ़िगर होने तक बंद | एजेंट को कॉन्फ़िगर किए गए Google कैलेंडर और इवेंट पढ़ने की अनुमति देता है। |
| **SMS सूचनाएँ सक्षम करें** | कॉन्फ़िगर होने तक बंद | क्रेडेंशियल सहेजे जाने के बाद स्वीकृत वर्कफ़्लो को TextBee SMS सूचनाएँ भेजने की अनुमति देता है। |

### बदलाव लागू करना

किसी भी फ़्लैग को बदलने के बाद **सेटिंग्स सहेजें** पर क्लिक करें। बदलाव तुरंत प्रभावी हो जाते हैं — कोई कैश फ़्लश या plugin पुनःसक्रियण आवश्यक नहीं है।
