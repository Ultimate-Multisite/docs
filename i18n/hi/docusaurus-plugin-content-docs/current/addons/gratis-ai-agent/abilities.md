# क्षमताओं का संदर्भ {#abilities-reference}

क्षमताएँ वे परमाणु क्रियाएँ (atomic actions) हैं जिन्हें Gratis AI Agent आपके वर्डप्रेस इंस्टॉलेशन पर इस्तेमाल कर सकता है। हर क्षमता एक पंजीकृत PHP क्लास होती है जो एक JSON स्कीमा को प्रदर्शित करती है — एजेंट रनटाइम पर इस स्कीमा को पढ़ता है ताकि यह समझ सके कि किन मापदंडों की आवश्यकता है और क्षमता क्या लौटाएगी।

यह पेज Gratis AI Agent v1.9.0 के साथ आने वाली सभी क्षमताओं का दस्तावेजीकरण करता है।

---

## कस्टम पोस्ट टाइप (Custom Post Types) {#custom-post-types}

ये क्षमताएँ एजेंट के माध्यम से पंजीकृत कस्टम पोस्ट टाइप्स (CPTs) को प्रबंधित करती हैं। पंजीकरण वर्डप्रेस ऑप्शंस टेबल में स्थायी रूप से सहेजे जाते हैं ताकि वे प्लगइन निष्क्रिय होने और पुनः सक्रिय होने पर भी बने रहें।

### `register_post_type` {#registerposttype}

एक नया कस्टम पोस्ट टाइप रजिस्टर करता है।

**पैरामीटर्स**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | पोस्ट टाइप की कुंजी (अधिकतम 20 वर्ण, कोई अपरकेस नहीं, कोई स्पेस नहीं) |
| `singular_label` | string | हाँ | मानव-पठनीय एकवचन नाम, जैसे: `Portfolio Item` |
| `plural_label` | string | हाँ | मानव-पठनीय बहुवचन नाम, जैसे: `Portfolio Items` |
| `public` | boolean | नहीं | क्या पोस्ट टाइप सार्वजनिक रूप से सुलभ है। डिफ़ॉल्ट `true` |
| `supports` | array | नहीं | समर्थित सुविधाएँ: `title`, `editor`, `thumbnail`, `excerpt`, `comments`, `revisions`, `custom-fields`। डिफ़ॉल्ट `["title","editor"]` |
| `has_archive` | boolean | नहीं | क्या पोस्ट टाइप आर्काइव पेज सक्षम है। डिफ़ॉल्ट `false` |
| `menu_icon` | string | नहीं | एडमिन मेनू आइकन के लिए Dashicons क्लास या URL। डिफ़ॉल्ट `"dashicons-admin-post"` |
| `rewrite_slug` | string | नहीं | पोस्ट टाइप के लिए URL स्लग। डिफ़ॉल्ट `slug` है |

**उदाहरण**

```json
{
  "slug": "portfolio",
  "singular_label": "Portfolio Item",
  "plural_label": "Portfolio Items",
  "public": true,
  "supports": ["title", "editor", "thumbnail"],
  "has_archive": true,
  "menu_icon": "dashicons-portfolio"
}
```

**वापसी** `{ "success": true, "slug": "portfolio" }`

---

### `list_post_types` {#listposttypes}

एजेंट द्वारा पंजीकृत सभी कस्टम पोस्ट टाइप्स लौटाता है।

**पैरामीटर** — कोई नहीं

**वापसी**

```json
{
  "post_types": [
    {
      "slug": "portfolio",
      "singular_label": "Portfolio Item",
      "plural_label": "Portfolio Items",
      "public": true
    }
  ]
}
```

### `delete_post_type` {#deleteposttype}

यह एजेंट द्वारा पहले पंजीकृत किसी कस्टम पोस्ट टाइप को अनरजिस्टर (unregister) करता है। उस प्रकार के मौजूदा पोस्ट डेटाबेस में बने रहते हैं लेकिन अब पोस्ट टाइप के माध्यम से एक्सेस नहीं किए जा सकते।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | हटाने के लिए पोस्ट टाइप की कुंजी (key) |

**रिटर्न** `{ "success": true, "slug": "portfolio" }`

## कस्टम टैक्सोनॉमीज़ (Custom Taxonomies) {#custom-taxonomies}

ये क्षमताएं कस्टम टैक्सोनॉमीज़ को मैनेज करती हैं। CPTs की तरह ही, टैक्सोनॉमी रजिस्ट्रेशन भी स्थायी (persisted) रहते हैं।

### `register_taxonomy` {#registertaxonomy}

एक नई कस्टम टैक्सोनॉमी रजिस्टर करता है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | टैक्सोनॉमी की कुंजी (key) (अधिकतम 32 वर्ण) |
| `singular_label` | string | हाँ | मानव-पठनीय एकल नाम, जैसे: `Project Category` |
| `plural_label` | string | हाँ | मानव-पठनीय बहुवचन नाम, जैसे: `Project Categories` |
| `post_types` | array | हाँ | यह टैक्सोनॉमी किन पोस्ट टाइप से जुड़ी होगी |
| `hierarchical` | boolean | नहीं | श्रेणी-शैली (category-style) के लिए `true`, टैग-शैली (tag-style) के लिए `false`। डिफ़ॉल्ट `true` है। |
| `public` | boolean | नहीं | यह निर्धारित करता है कि टर्म्स सार्वजनिक रूप से एक्सेस किए जा सकते हैं या नहीं। डिफ़ॉल्ट `true` है। |
| `rewrite_slug` | string | नहीं | टैक्सोनॉमी के लिए URL स्लग। डिफ़ॉल्ट `slug` होता है। |

**उदाहरण**

```json
{
  "slug": "project-category",
  "singular_label": "Project Category",
  "plural_label": "Project Categories",
  "post_types": ["portfolio"],
  "hierarchical": true
}
```

**रिटर्न** `{ "success": true, "slug": "project-category" }`

### `list_taxonomies` {#listtaxonomies}

यह एजेंट द्वारा पंजीकृत सभी कस्टम टैक्सोनॉमी (custom taxonomies) को लौटाता है।

**पैरामीटर** — कोई नहीं

**रिटर्न**

```json
{
  "taxonomies": [
    {
      "slug": "project-category",
      "singular_label": "Project Category",
      "post_types": ["portfolio"],
      "hierarchical": true
    }
  ]
}
```

---

### `delete_taxonomy` {#deletetaxonomy}

एजेंट द्वारा पहले पंजीकृत किसी कस्टम टैक्सोनॉमी को अनरजिस्टर करता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | हटाने के लिए टैक्सोनॉमी की कुंजी (key) |

**रिटर्न** `{ "success": true, "slug": "project-category" }`

## डिज़ाइन सिस्टम {#design-system}

डिज़ाइन सिस्टम क्षमताएं वर्डप्रेस साइट के विज़ुअल प्रेजेंटेशन को बदलती हैं — कस्टम CSS से लेकर ब्लॉक पैटर्न और साइट लोगो तक।

### `inject_custom_css` {#injectcustomcss}

यह `wp_add_inline_style` के माध्यम से साइट के `<head>` में CSS जोड़ता है। CSS को `gratis_ai_agent_custom_css` विकल्प में स्टोर किया जाता है और क्षमता रीसेट होने पर साफ़ तरीके से हटा दिया जाता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `css` | string | हाँ | इंजेक्ट करने के लिए मान्य CSS |
| `label` | string | नहीं | इस CSS ब्लॉक के लिए मानव-पठनीय लेबल, जिसका उपयोग डिबग लॉग में होता है। डिफ़ॉल्ट `"agent-injected"` |
| `replace` | boolean | नहीं | यदि यह `true` है, तो यह पहले इंजेक्ट किए गए सभी CSS को बदल देगा। डिफ़ॉल्ट `false` (जोड़ना) |

**उदाहरण**

```json
{
  "css": ":root { --primary: #1a1a2e; --accent: #e94560; } body { font-family: 'Inter', sans-serif; }",
  "label": "brand-colours",
  "replace": false
}
```

**वापसी** `{ "success": true, "bytes": 96 }`

---

### `add_block_pattern` {#addblockpattern}

वर्डप्रेस पैटर्न लाइब्रेरी में एक दोबारा इस्तेमाल होने वाले ब्लॉक पैटर्न को रजिस्टर करता है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | पैटर्न की पहचान, जैसे: `gratis/hero-dark` |
| `title` | string | हाँ | एडिटर में दिखने वाला मानव-पठनीय पैटर्न का नाम |
| `content` | string | हाँ | पैटर्न के लिए सीरियलाइज़ किया गया ब्लॉक मार्कअप (HTML) |
| `categories` | array | नहीं | पैटर्न कैटेगरी स्लग, जैसे: `["featured", "hero"]` |
| `description` | string | नहीं | पैटर्न पिकर में दिखने वाला संक्षिप्त विवरण |
| `keywords` | array | नहीं | खोज कीवर्ड्स |

**वापसी** `{ "success": true, "slug": "gratis/hero-dark" }`

---

### `list_block_patterns` {#listblockpatterns}

एजेंट द्वारा पंजीकृत सभी ब्लॉक पैटर्न को सूचीबद्ध करता है।

**पैरामीटर्स** — कोई नहीं

**वापसी**

```json
{
  "patterns": [
    {
      "slug": "gratis/hero-dark",
      "title": "Dark Hero",
      "categories": ["hero"]
    }
  ]
}
```

---

### `set_site_logo` {#setsitelogo}

वर्डप्रेस साइट लोगो को किसी दिए गए अटैचमेंट आईडी या एक रिमोट इमेज URL पर सेट करता है। जब कोई URL प्रदान की जाती है, तो छवि डाउनलोड हो जाती है और मीडिया लाइब्रेरी में आयात कर ली जाती है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `attachment_id` | integer | नहीं | मौजूदा मीडिया लाइब्रेरी अटैचमेंट की आईडी |
| `url` | string | नहीं | लोगो के रूप में आयात करने और सेट करने के लिए रिमोट इमेज URL |

`attachment_id` या `url` में से कोई एक प्रदान करना होगा।

**वापसी** `{ "success": true, "attachment_id": 42 }`

---

### `apply_theme_json_preset` {#applythemejsonpreset}

यह सक्रिय थीम की `theme.json` (या `global-styles`) में एक नामित रंग/टाइपोग्राफी प्रीसेट को लागू करता है। प्रीसेट्स Gratis AI एजेंट टीम द्वारा बनाए गए क्यूरेटेड बंडल्स हैं।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `preset` | string | हाँ | प्रीसेट का नाम, जैसे: `minimal-dark`, `warm-editorial`, `corporate-blue` |
| `merge` | boolean | नहीं | यदि यह `true` है, तो मौजूदा मानों को बदलने के बजाय उनमें मर्ज किया जाएगा। डिफ़ॉल्ट `false` |

**उपलब्ध प्रीसेट्स**

| प्रीसेट | विवरण |
|---|---|
| `minimal-dark` | लगभग काला बैकग्राउंड, सफेद टेक्स्ट, एक एकल एक्सेंट रंग |
| `warm-editorial` | गर्म ऑफ-व्हाइट बैकग्राउंड, सेरिफ़ हेडिंग, मिट्टी जैसे एक्सेंट रंग |
| `corporate-blue` | नेवी और सफेद पैलेट के साथ पेशेवर टाइपोग्राफी |
| `vibrant-startup` | चमकीले ग्रेडिएंट्स, गोल कोने, आधुनिक सैन्स-सेरिफ़ टाइप |
| `classic-blog` | न्यूट्रल ग्रे, आरामदायक लाइन-हाइट, पारंपरिक लेआउट स्पेसिंग |

**वापसी** `{ "success": true, "preset": "minimal-dark" }`

## ग्लोबल स्टाइल्स (Global Styles) {#global-styles}

ग्लोबल स्टाइल्स की क्षमताएं वर्डप्रेस ग्लोबल स्टाइल्स API के माध्यम से थीम.json मानों को पढ़ती और लिखती हैं, जो साइट पर सभी ब्लॉक्स और टेम्प्लेट को प्रभावित करती हैं।

### `get_global_styles` {#getglobalstyles}

वर्तमान ग्लोबल स्टाइल्स कॉन्फ़िगरेशन लौटाता है।

**पैरामीटर्स**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `path` | string | नहीं | किसी विशिष्ट मान (value) का JSON पॉइंटर, जैसे `/color/palette` या `/typography/fontSizes`. यदि इसे छोड़ दिया जाए तो पूरी ऑब्जेक्ट वापस कर दी जाती है। |

**वापस करता है** पूरी ग्लोबल स्टाइल्स ऑब्जेक्ट या `path` पर मान।

---

### `set_global_styles` {#setglobalstyles}

ग्लोबल स्टाइल्स कॉन्फ़िगरेशन में एक या एक से अधिक मानों को अपडेट करता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `path` | string | हाँ | सेट करने के लिए मान का JSON पॉइंटर, जैसे `/color/palette` |
| `value` | any | हाँ | नया मान |

**उदाहरण** — पैलेट में एक रंग जोड़ना

```json
{
  "path": "/color/palette",
  "value": [
    { "slug": "primary", "color": "#1a1a2e", "name": "Primary" },
    { "slug": "accent",  "color": "#e94560", "name": "Accent" }
  ]
}
```

**वापस करता है** `{ "success": true, "path": "/color/palette" }`

---

### `reset_global_styles` {#resetglobalstyles}

सभी एजेंट-लागू किए गए ग्लोबल स्टाइल्स परिवर्तनों को रीसेट करता है, जिससे थीम के डिफ़ॉल्ट वापस आ जाते हैं।

**पैरामीटर** — कोई नहीं

**वापस करता है** `{ "success": true }`

## नेविगेशन मेनू {#navigation-menus}

नेविगेशन मेनू क्षमताएं वर्डप्रेस नेविगेशन मेनू और उनके आइटम्स को बनाती हैं और प्रबंधित करती हैं।

### `create_menu` {#createmenu}

एक नया वर्डप्रेस नेविगेशन मेनू बनाता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `name` | string | हाँ | मेनू का नाम, जैसे `Primary Navigation` |
| `location` | string | नहीं | इस मेनू को असाइन करने के लिए थीम लोकेशन, जैसे `primary` |

**वापस करता है** `{ "success": true, "menu_id": 7 }`

### `update_menu` {#updatemenu}

किसी मेनू का नाम बदलने या उसे किसी थीम लोकेशन पर फिर से असाइन (reassign) करने के लिए।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `menu_id` | integer | हाँ | अपडेट किए जाने वाले मेनू की ID |
| `name` | string | नहीं | नया मेनू नाम |
| `location` | string | नहीं | असाइन करने या फिर से सौंपने के लिए थीम लोकेशन |

**रिटर्न** `{ "success": true, "menu_id": 7 }`

---

### `add_menu_item` {#addmenuitem}

किसी मौजूदा नेविगेशन मेनू में एक आइटम जोड़ने के लिए।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `menu_id` | integer | हाँ | लक्षित मेनू की ID |
| `type` | string | हाँ | आइटम का प्रकार: `custom`, `post_type`, या `taxonomy` |
| `title` | string | नहीं | मेनू आइटम के लिए लेबल (custom टाइप के लिए आवश्यक) |

## विकल्पों का प्रबंधन (Options Management) {#options-management}

ऑप्शंस की क्षमताएं `get_option` / `update_option` के माध्यम से वर्डप्रेस ऑप्शंस को पढ़ती और लिखती हैं। एक बिल्ट-इन सुरक्षा ब्लॉकलिस्ट महत्वपूर्ण सेटिंग्स में गलती से बदलाव करने से रोकती है।

### `get_option` {#getoption}

एक वर्डप्रेस ऑप्शन को पढ़ता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `option_name` | string | हाँ | ऑप्शन की कुंजी, जैसे कि `blogname` |

**वापस (Returns)** `{ "option_name": "blogname", "value": "My Site" }`

यदि `option_name` सुरक्षा ब्लॉकलिस्ट पर है तो त्रुटि वापस करता है।

---

### `set_option` {#setoption}

एक वर्डप्रेस ऑप्शन लिखता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `option_name` | string | हाँ | ऑप्शन की कुंजी |
| `value` | any | हाँ | नया मान (एरे/ऑब्जेक्ट के लिए स्वचालित रूप से सीरियलाइज़ किया जाता है) |
| `autoload` | string | नहीं | `"yes"` या `"no"`. डिफ़ॉल्ट मौजूदा autoload सेटिंग को बनाए रखता है |

यदि `option_name` सुरक्षा ब्लॉकलिस्ट पर है तो त्रुटि वापस करता है।

**वापस (Returns)** `{ "success": true, "option_name": "blogname" }`

### `delete_option` {#deleteoption}

एक वर्डप्रेस ऑप्शन को हटाता है।

**पैरामीटर**

|---|---|---|---|
| `option_name` | string | हाँ | हटाने के लिए ऑप्शन की कुंजी |

### `list_options` {#listoptions}

यह फ़ंक्शन किसी पैटर्न से मेल खाने वाले वर्डप्रेस विकल्पों की सूची दिखाता है।

## कंटेंट मैनेजमेंट {#content-management}

कंटेंट मैनेजमेंट क्षमताएं वर्डप्रेस पोस्ट और पेज बनाते हैं और संपादित करती हैं। पोस्ट आईडी वापस की जाती है ताकि मल्टी-एबिलिटी योजनाओं में बनाई गई सामग्री का संदर्भ लिया जा सके।

### `create_post` {#createpost}

एक नया वर्डप्रेस पोस्ट, पेज या कस्टम पोस्ट टाइप एंट्री बनाता है।

**पैरामीटर्स**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `title` | string | हाँ | पोस्ट का शीर्षक |
| `content` | string | नहीं | पोस्ट की सामग्री — प्लेन टेक्स्ट, HTML, या सीरियलाइज़्ड ब्लॉक मार्कअप स्वीकार करता है |
| `status` | string | नहीं | `draft`, `publish`, `pending`, `private`. डिफ़ॉल्ट `draft` |
| `post_type` | string | नहीं | पोस्ट टाइप स्लग, जैसे `post`, `page`, या कोई भी पंजीकृत CPT। डिफ़ॉल्ट `post` |
| `excerpt` | string | नहीं | आर्काइव और खोज परिणामों में दिखाई देने वाला संक्षिप्त सारांश |
| `categories` | array | नहीं | असाइन करने के लिए श्रेणी नामों या आईडी की ऐरे |
| `tags` | array | नहीं | असाइन करने के लिए टैग नामों या आईडी की ऐरे |
| `author` | integer | नहीं | पोस्ट लेखक के रूप में सेट करने के लिए वर्डप्रेस उपयोगकर्ता आईडी। डिफ़ॉल्ट वर्तमान उपयोगकर्ता |
| `date` | string | नहीं | ISO 8601 प्रारूप में प्रकाशन तिथि, जैसे `2026-05-01T09:00:00` |
| `page_template` | string | नहीं | इस पोस्ट या पेज को असाइन करने के लिए टेम्पलेट फ़ाइल, जैसे `page-full-width.php`। केवल तभी सार्थक जब `post_type` `page` हो या कोई ऐसा CPT जो पेज टेम्पलेट्स का समर्थन करता हो। |

**उदाहरण**

```json
{
  "title": "हमारे नए साइट में आपका स्वागत है",
  "content": "<!-- wp:paragraph --><p>नमस्ते दुनिया!</p><!-- /wp:paragraph -->",
  "status": "publish",
  "post_type": "page",
  "page_template": "page-full-width.php"
}
```

**वापसी** `{ "success": true, "post_id": 42, "permalink": "https://example.com/welcome/" }`

### `set_featured_image`

### `create_contact_form`

यह प्लगइन सक्रिय फॉर्म प्लगइन (जैसे Contact Form 7, WPForms, Fluent Forms, या Gravity Forms में से जो भी इंस्टॉल हो) का उपयोग करके एक संपर्क फ़ॉर्म बनाता है। यह एक शॉर्टकोड लौटाता है जिसे आप किसी भी पोस्ट या पेज में एम्बेड कर सकते हैं।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `title` | string | हाँ | फॉर्म प्लगइन के एडमिन में दिखने वाला फॉर्म का नाम |
| `fields` | array | हाँ | फॉर्म फ़ील्ड्स की क्रमबद्ध सूची (नीचे Field ऑब्जेक्ट देखें) |

| `subject` | string | नहीं | ईमेल विषय पंक्ति। Contact Form 7 का उपयोग करते समय `[your-name]` और `[your-subject]` प्लेसहोल्डर्स का समर्थन करता है। |
| `confirmation_message` | string | नहीं | सफल सबमिशन के बाद प्रदर्शित होने वाला संदेश। डिफ़ॉल्ट: `"Thank you for your message. We'll be in touch soon."` |

**फ़ील्ड ऑब्जेक्ट**

| `label` | string | हाँ | फॉर्म पर दिखने वाला मानव-पठनीय लेबल |
| `type` | string | हाँ | `text`, `email`, `tel`, `textarea`, `select`, `checkbox`, `radio`, `file`, `date` |
| `required` | boolean | नहीं | सबमिशन से पहले फ़ील्ड को भरने की आवश्यकता है। डिफ़ॉल्ट `false` |
| `options` | array | नहीं | `select`, `checkbox`, और `radio` फ़ील्ड के लिए विकल्प |
| `placeholder` | string | नहीं | टेक्स्ट-प्रकार इनपुट के लिए प्लेसहोल्डर टेक्स्ट |

  "success": true,

## विज़ुअल रिव्यू (Visual Review)

विज़ुअल रिव्यू की सुविधा आपको लाइव पेजों के स्क्रीनशॉट लेने और उनका विश्लेषण करने की अनुमति देती है, जिससे बिना किसी ब्राउज़र एक्सटेंशन के स्वचालित डिज़ाइन रिव्यू, पहले/बाद की तुलना, और विज़ुअल रिग्रेशन चेक किए जा सकते हैं।

### `capture_screenshot`

यह एक सर्वर-साइड हेडलेस ब्राउज़र का उपयोग करके किसी दिए गए URL पर वर्डप्रेस पेज का स्क्रीनशॉट लेता है। इमेज को मीडिया लाइब्रेरी में सेव किया जाता है और एक CDN URL वापस कर दी जाती है।

**पैरामीटर्स (Parameters)**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `url` | string | हाँ | स्क्रीनशॉट लेने वाले पेज का पूरा URL, जैसे: `https://example.com/about/` |
| `width` | integer | नहीं | पिक्सेल में व्यूपोर्ट की चौड़ाई। डिफ़ॉल्ट `1280` |
| `height` | integer | नहीं | पिक्सेल में व्यूपोर्ट की ऊँचाई। डिफ़ॉल्ट `800` |
| `full_page` | boolean | नहीं | केवल व्यूपोर्ट के बजाय पूरे स्क्रॉल करने योग्य पेज को कैप्चर करें। डिफ़ॉल्ट `false` |
| `delay_ms` | integer | नहीं | पेज लोड होने के बाद कैप्चर करने से पहले मिलीसेकंड में प्रतीक्षा समय, एनिमेटेड सामग्री के लिए उपयोगी। डिफ़ॉल्ट `500` |
| `label` | string | नहीं | मीडिया लाइब्रेरी में अटैचमेंट के साथ संग्रहीत मानव-पठनीय लेबल |

**वापसी (Returns)**

```json
{
  "success": true,
  "attachment_id": 88,
  "url": "https://example.com/wp-content/uploads/2026/04/screenshot-about.png",
  "width": 1280,
  "height": 800
}
```

### `compare_screenshots`

दो स्क्रीनशॉट लेता है और एक विज़ुअल डिफ़रेंस स्कोर के साथ एक डिफ़रेंस इमेज लौटाता है जो बदले गए क्षेत्रों को हाइलाइट करती है। यह पुष्टि करने के लिए उपयोगी है कि डिज़ाइन परिवर्तन अपेक्षित परिणाम दे रहा है या अनपेक्षित समस्याओं का पता लगाने के लिए।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `before_url` | string | हाँ | उस पेज का URL जिसे "पहले" स्थिति के रूप में कैप्चर किया जाएगा |
| `after_url` | string | हाँ | उस पेज का URL जिसे "बाद" की स्थिति के रूप में कैप्चर किया जाएगा। यदि समय के साथ तुलना कर रहे हैं तो यह समान URL हो सकता है |
| `width` | integer | नहीं | दोनों कैप्चर के लिए व्यूपोर्ट चौड़ाई। डिफ़ॉल्ट `1280` |
| `threshold` | float | नहीं | पिक्सेल-अंतर सीमा (0.0–1.0)। इस सहनशीलता के भीतर वाले पिक्सेल अपरिवर्तित माने जाते हैं। डिफ़ॉल्ट `0.1` |

**वापसी (Returns)**

```json
{
  "success": true,
  "diff_score": 0.04,
  "changed_pixels": 2340,
  "total_pixels": 1024000,
  "diff_attachment_id": 91,
  "diff_url": "https://example.com/wp-content/uploads/2026/04/diff-about.png"
}
```

`diff_score` का `0.0` का मतलब है कि कोई दृश्य परिवर्तन नहीं है; `1.0` का मतलब है कि हर पिक्सेल बदल गया है।

### `review_page_design`

यह एक पेज का स्क्रीनशॉट लेता है और विज़ुअल विश्लेषण के लिए इसे भाषा मॉडल को भेजता है। यह लेआउट, टाइपोग्राफी, रंग उपयोग और एक्सेसिबिलिटी संबंधी चिंताओं को कवर करने वाले एक संरचित मूल्यांकन (structured assessment) को लौटाता है।

**पैरामीटर**

| पैरामीटर | प्रकार | आवश्यक | विवरण |
|---|---|---|---|
| `url` | string | हाँ | समीक्षा के लिए पेज का पूरा URL |
| `focus` | string | नहीं | उन समीक्षा क्षेत्रों की कॉमा-सेपरेटेड सूची जिन पर जोर देना है: `layout`, `typography`, `colour`, `accessibility`, `mobile`। डिफ़ॉल्ट: सभी क्षेत्र |
| `width` | integer | नहीं | व्यूपोर्ट चौड़ाई। डिफ़ॉल्ट `1280` |

**वापसी (Returns)**

## इंस्टॉल करने योग्य क्षमताएं (Installable Abilities)

इंस्टाल करने योग्य एबिलिटीज रजिस्ट्री आपको वर्डप्रेस प्लगइन्स के रूप में वितरित अतिरिक्त एबिलिटी पैक के साथ एजेंट को बढ़ाने की सुविधा देती है। प्रत्येक पैक मानक एबिलिटी API का उपयोग करके एक या अधिक क्षमताओं को रजिस्टर करता है।

### `list_available_abilities`

यह रजिस्ट्री से इंस्टॉल करने के लिए उपलब्ध एबिलिटी पैक की सूची लौटाता है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `category` | string | नहीं | श्रेणी के आधार पर फ़िल्टर करें: `ecommerce`, `seo`, `media`, `social`, `developer` |

**रिटर्न**

### `install_ability`

यह रजिस्ट्री से किसी एबिलिटी पैक को डाउनलोड और एक्टिवेट करता है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `slug` | string | हाँ | एबिलिटी पैक प्लगइन स्लग |

**रिटर्न** `{ "success": true, "slug": "gratis-ai-agent-woocommerce", "abilities_added": 3 }`

---

### `recommend_plugin`

यह सर्वोत्तम प्लगइन खोजने के लिए एबिलिटी रजिस्ट्री को क्वेरी करता है जो बताए गए उपयोग के मामले के लिए सबसे उपयुक्त हो और वैकल्पिक रूप से उसे इंस्टॉल भी कर सकता है।

**पैरामीटर्स**

| पैरामीटर | टाइप | आवश्यक | विवरण |
|---|---|---|---|
| `description` | string | हाँ | वांछित कार्यक्षमता का प्राकृतिक भाषा में विवरण |
| `install` | boolean | नहीं | यदि यह `true` है, तो अनुशंसित प्लगइन तुरंत इंस्टॉल हो जाएगा। डिफ़ॉल्ट `false` होता है |

**उदाहरण**

```json
{
  "description": "मुझे फ़ाइल अपलोड समर्थन और स्पैम सुरक्षा के साथ एक संपर्क फ़ॉर्म चाहिए",
  "install": false
}
```

**रिटर्न**

```json
{
  "recommendation": {
    "slug": "contact-form-7",
    "name": "Contact Form 7",
    "reason": "व्यापक रूप से अपनाया गया, फ़ाइल अपलोड का समर्थन करता है, और स्पैम फ़िल्टरिंग के लिए Akismet के साथ एकीकृत होता है।",
    "alternatives": ["wpforms-lite", "fluent-forms"]
  }
}
```
