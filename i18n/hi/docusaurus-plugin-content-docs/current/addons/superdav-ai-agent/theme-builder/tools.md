# टूल्स और कॉन्फ़िगरेशन

Theme Builder आपके WordPress साइट का विश्लेषण करने, डिज़ाइन करने और बनाने के लिए कई टूल्स का उपयोग करता है। वर्जन 1.16.0 में, **sd-ai-agent/site-scrape अब एक Tier 1 टूल है**, जिससे यह Theme Builder में डिफ़ॉल्ट रूप से उपलब्ध है।

## उपलब्ध टूल्स

### Tier 1 टूल्स (हमेशा उपलब्ध)

Tier 1 टूल्स Theme Builder में बिना किसी अतिरिक्त कॉन्फ़िगरेशन के डिफ़ॉल्ट रूप से उपलब्ध होते हैं।

#### sd-ai-agent/site-scrape

**उद्देश्य:** डिज़ाइन प्रेरणा, कंटेंट स्ट्रक्चर और कार्यक्षमता निकालने के लिए मौजूदा वेबसाइटों का विश्लेषण करना।

**क्षमताएं:**

- **वेबसाइट विश्लेषण** — प्रतियोगी या प्रेरणादायक वेबसाइटों को स्क्रैप और विश्लेषण करना
- **डिज़ाइन निष्कर्षण** — रंगों, फ़ॉन्ट्स और लेआउट पैटर्न की पहचान करना
- **कंटेंट स्ट्रक्चर** — पेज के संगठन और पदानुक्रम को समझना
- **फ़ीचर डिटेक्शन** — प्लगइन्स और कार्यक्षमता की पहचान करना
- **परफॉर्मेंस विश्लेषण** — पेज की गति और ऑप्टिमाइज़ेशन की जाँच करना
- **SEO विश्लेषण** — मेटा टैग और स्ट्रक्चर्ड डेटा की समीक्षा करना

**उपयोग:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**यह क्या निकालता है:**

- रंग पैलेट और रंग का उपयोग
- टाइपोग्राफी (फ़ॉन्ट्स और साइज़)
- लेआउट स्ट्रक्चर और स्पेसिंग
- नेविगेशन पैटर्न
- कंटेंट का संगठन
- इमेज और मीडिया का उपयोग
- इंटरैक्टिव तत्व
- मोबाइल रिस्पॉन्सिवनेस

**सीमाएं:**

- पासवर्ड से सुरक्षित साइटों को स्क्रैप नहीं कर सकता
- robots.txt और साइट नीतियों का सम्मान करता है
- डायनामिक कंटेंट को कैप्चर नहीं कर सकता है
- JavaScript-भारी साइटों का विश्लेषण सीमित हो सकता है
- बड़ी साइटों का विश्लेषण करने में अधिक समय लग सकता है

### Tier 2 टूल्स (वैकल्पिक)

Tier 2 टूल्स Theme Builder सेटिंग्स में स्पष्ट रूप से सक्षम करने पर उपलब्ध होते हैं।

#### Advanced Analytics

विस्तृत परफॉर्मेंस मेट्रिक्स प्रदान करता है:

- पेज लोड समय
- Core Web Vitals
- SEO स्कोर
- एक्सेसिबिलिटी स्कोर
- बेस्ट प्रैक्टिसेज स्कोर

#### Content Optimizer

निम्नलिखित के लिए विश्लेषण करता है और सुधार सुझाता है:

- पठनीयता (Readability)
- SEO ऑप्टिमाइज़ेशन
- कीवर्ड का उपयोग
- कंटेंट स्ट्रक्चर
- कॉल-टू-एक्शन प्लेसमेंट

### Tier 3 टूल्स (प्रीमियम)

Tier 3 टूल्स के लिए अतिरिक्त कॉन्फ़िगरेशन या प्रीमियम सुविधाओं की आवश्यकता होती है।

#### AI Content Generator

निम्नलिखित के लिए कंटेंट उत्पन्न करता है:

- प्रोडक्ट डिस्क्रिप्शन
- सर्विस पेज
- ब्लॉग पोस्ट
- मेटा डिस्क्रिप्शन
- कॉल-टू-एक्शन कॉपी

#### Advanced Design Tools

उन्नत डिज़ाइन क्षमताएं प्रदान करता है:

- कस्टम CSS जनरेशन
- एनिमेशन क्रिएशन
- इंटरैक्टिव तत्व डिज़ाइन
- उन्नत रंग सिद्धांत (Advanced color theory)
- टाइपोग्राफी ऑप्टिमाइज़ेशन

## टूल कॉन्फ़िगरेशन

### टूल्स को सक्षम करना

Theme Builder में अतिरिक्त टूल्स को सक्षम करने के लिए:

1. **Gratis AI Agent → Settings** पर जाएँ
2. **Theme Builder → Tools** पर जाएँ
3. ज़रूरत के हिसाब से टूल्स को ऑन/ऑफ करें
4. सेटिंग्स सेव करें

### टूल परमिशन

कुछ टूल्स को परमिशन की आवश्यकता होती है:

- **Site scraping** — इंटरनेट एक्सेस की आवश्यकता है
- **Analytics** — Google Analytics इंटीग्रेशन की आवश्यकता है
- **Content generation** — API keys की आवश्यकता है
- **Advanced features** — प्रीमियम सब्सक्रिप्शन की आवश्यकता हो सकती है

### API Keys और Credentials

उन टूल्स के लिए API keys कॉन्फ़िगर करें जिन्हें उनकी आवश्यकता है:

1. **Gratis AI Agent → Settings → API Keys** पर जाएँ
2. प्रत्येक टूल के लिए क्रेडेंशियल्स दर्ज करें
3. कनेक्शन का परीक्षण करें
4. सुरक्षित रूप से सेव करें

## sd-ai-agent/site-scrape का उपयोग करना

### बुनियादी उपयोग

Theme Builder से किसी वेबसाइट का विश्लेषण करने के लिए कहें:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### विशिष्ट विश्लेषण

विशिष्ट प्रकार के विश्लेषण का अनुरोध करें:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### तुलनात्मक विश्लेषण

कई साइटों की तुलना करें:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## टूल सीमाएं और विचार

### रेट लिमिटिंग (Rate Limiting)

- सर्वर ओवरलोड को रोकने के लिए स्क्रैपिंग रेट-लिमिटेड होती है
- एक ही साइट पर कई अनुरोधों को धीमा किया जा सकता है (throttled)
- बड़ी साइटों का विश्लेषण करने में अधिक समय लग सकता है

### कंटेंट सटीकता (Content Accuracy)

- डायनामिक कंटेंट पूरी तरह से कैप्चर नहीं हो सकता है
- JavaScript-रेंडर किया गया कंटेंट अधूरा हो सकता है
- रियल-टाइम डेटा प्रतिबिंबित नहीं हो सकता है
- कुछ कंटेंट पेवॉल के पीछे हो सकता है

### कानूनी और नैतिक विचार

- robots.txt और साइट नीतियों का सम्मान करें
- पुनरुत्पादन के लिए कॉपीराइटेड कंटेंट स्क्रैप न करें
- विश्लेषण का उपयोग प्रेरणा के लिए करें, नकल करने के लिए नहीं
- सत्यापित करें कि आपके पास निकाले गए कंटेंट का उपयोग करने का अधिकार है
- विश्लेषण की गई साइटों की सेवा की शर्तों का पालन करें

### परफॉर्मेंस प्रभाव

- बड़ी साइटों को स्क्रैप करने में समय लग सकता है
- कई एक साथ स्क्रैप करने से गति धीमी हो सकती है
- नेटवर्क कनेक्टिविटी गति को प्रभावित करती है
- बड़े डेटासेट के लिए अधिक प्रोसेसिंग की आवश्यकता हो सकती है

## सर्वोत्तम अभ्यास (Best Practices)

### साइट विश्लेषण का उपयोग करना

1. **कई साइटों का विश्लेषण करें** — कई स्रोतों से प्रेरणा इकट्ठा करें
2. **स्ट्रक्चर पर ध्यान दें** — लेआउट और संगठन पैटर्न सीखें
3. **रंग निकालें** — कलर पैलेट को शुरुआती बिंदु के रूप में उपयोग करें
4. **टाइपोग्राफी का अध्ययन करें** — उन फ़ॉन्ट कॉम्बिनेशन की पहचान करें जो आपको पसंद हैं
5. **नेविगेशन की समीक्षा करें** — मेनू के संगठन को समझें

### नैतिक स्क्रैपिंग

1. **robots.txt का सम्मान करें** — साइट दिशानिर्देशों का पालन करें
2. **कंटेंट कॉपी न करें** — विश्लेषण का उपयोग केवल प्रेरणा के लिए करें
3. **अधिकार सत्यापित करें** — सुनिश्चित करें कि आप निकाले गए कंटेंट का उपयोग कर सकते हैं
4. **स्रोत का श्रेय दें** — प्रेरणा के स्रोतों को क्रेडिट दें
5. **शर्तों का पालन करें** — साइट की सेवा की शर्तों का पालन करें

### टूल की प्रभावशीलता को अधिकतम करना

1. **विशिष्ट रहें** — विशिष्ट प्रकार के विश्लेषण का अनुरोध करें
2. **संदर्भ प्रदान करें** — अपनी साइट के उद्देश्य की व्याख्या करें
3. **अपेक्षाएं निर्धारित करें** — अपने डिज़ाइन लक्ष्यों का वर्णन करें
4. **दोहराएं (Iterate)** — परिणामों के आधार पर सुधार करें
5. **टूल्स को मिलाएं** — व्यापक विश्लेषण के लिए कई टूल्स का उपयोग करें

## समस्या निवारण (Troubleshooting)

### साइट स्क्रैप नहीं हो रही

- जाँच करें कि साइट सार्वजनिक रूप से सुलभ है या नहीं
- सत्यापित करें कि robots.txt स्क्रैपिंग की अनुमति देता है
- कोई दूसरी साइट आज़माएं
- इंटरनेट कनेक्शन की जाँच करें
- यदि समस्या बनी रहती है तो सपोर्ट से संपर्क करें

### अधूरा विश्लेषण

- साइट में डायनामिक कंटेंट हो सकता है
- JavaScript पूरी तरह से रेंडर नहीं हो सकता है
- बड़ी साइटें टाइमआउट हो सकती हैं
- इसके बजाय विशिष्ट पेजों का विश्लेषण करने का प्रयास करें
- विशिष्ट विश्लेषण प्रकार का अनुरोध करें

### धीमी परफॉर्मेंस

- बड़ी साइटों का विश्लेषण करने में अधिक समय लगता है
- कई एक साथ स्क्रैप करने से गति धीमी होती है
- नेटवर्क कनेक्टिविटी गति को प्रभावित करती है
- ऑफ-पीक घंटों के दौरान प्रयास करें
- पहले छोटे हिस्सों का विश्लेषण करें
