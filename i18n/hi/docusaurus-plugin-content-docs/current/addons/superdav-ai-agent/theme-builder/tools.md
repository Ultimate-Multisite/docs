# टूल और कॉन्फ़िगरेशन

Theme Builder आपके WordPress साइट का विश्लेषण, डिज़ाइन और निर्माण करने के लिए टूल के एक सेट का उपयोग करता है। संस्करण 1.16.0 में, **sd-ai-agent/site-scrape अब Tier 1 टूल है**, इसलिए यह Theme Builder में डिफ़ॉल्ट रूप से उपलब्ध है। Superdav AI Agent v1.18.0 भरोसेमंद डेवलपर कार्यप्रवाहों के लिए अलग से वितरित Advanced साथी पैकेज भी पेश करता है; वे टूल WordPress.org कोर पैकेज का हिस्सा नहीं हैं और उन्हें अलग से इंस्टॉल और अधिकृत करना होगा।

## उपलब्ध टूल {#available-tools}

### Tier 1 टूल (हमेशा उपलब्ध) {#tier-1-tools-always-available}

Tier 1 टूल Theme Builder में बिना किसी अतिरिक्त कॉन्फ़िगरेशन के डिफ़ॉल्ट रूप से उपलब्ध होते हैं।

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**उद्देश्य:** डिज़ाइन प्रेरणा, सामग्री संरचना और कार्यक्षमता निकालने के लिए मौजूदा वेबसाइटों का विश्लेषण करना।

**क्षमताएँ:**

- **वेबसाइट विश्लेषण** — प्रतिस्पर्धी या प्रेरणा वाली वेबसाइटों को scrape और विश्लेषित करें
- **डिज़ाइन निष्कर्षण** — रंगों, फ़ॉन्ट और लेआउट पैटर्न की पहचान करें
- **सामग्री संरचना** — पेज संगठन और पदानुक्रम समझें
- **फ़ीचर पहचान** — plugin और कार्यक्षमता की पहचान करें
- **प्रदर्शन विश्लेषण** — पेज गति और अनुकूलन जाँचें
- **SEO विश्लेषण** — meta tags और संरचित डेटा की समीक्षा करें

**उपयोग:**

```
example.com के डिज़ाइन का विश्लेषण करें और उसे मेरे साइट के लिए प्रेरणा के रूप में उपयोग करें।
```

**यह क्या निकालता है:**

- रंग पैलेट और रंग उपयोग
- टाइपोग्राफ़ी (फ़ॉन्ट और आकार)
- लेआउट संरचना और स्पेसिंग
- नेविगेशन पैटर्न
- सामग्री संगठन
- छवियाँ और मीडिया उपयोग
- इंटरैक्टिव तत्व
- मोबाइल अनुकूलता

**सीमाएँ:**

- पासवर्ड-सुरक्षित साइटों को scrape नहीं कर सकता
- robots.txt और साइट नीतियों का सम्मान करता है
- डायनेमिक सामग्री कैप्चर नहीं कर सकता
- JavaScript-भारी साइटों का विश्लेषण सीमित हो सकता है
- बड़ी साइटों के विश्लेषण में अधिक समय लग सकता है

### Tier 2 टूल (वैकल्पिक) {#tier-2-tools-optional}

Tier 2 टूल तब उपलब्ध होते हैं जब उन्हें Theme Builder सेटिंग्स में स्पष्ट रूप से सक्षम किया जाता है।

#### उन्नत विश्लेषिकी {#advanced-analytics}

विस्तृत प्रदर्शन मेट्रिक्स प्रदान करता है:

- पेज लोड समय
- Core Web Vitals
- SEO स्कोर
- पहुँच-योग्यता स्कोर
- सर्वोत्तम प्रथाओं का स्कोर

#### सामग्री अनुकूलक {#content-optimizer}

इनके लिए सुधारों का विश्लेषण और सुझाव देता है:

- पठनीयता
- SEO अनुकूलन
- कीवर्ड उपयोग
- सामग्री संरचना
- कॉल-टू-एक्शन का स्थान

### Tier 3 टूल (प्रीमियम) {#tier-3-tools-premium}

Tier 3 टूल के लिए अतिरिक्त कॉन्फ़िगरेशन या प्रीमियम फ़ीचर की आवश्यकता होती है।

#### AI सामग्री जनरेटर {#ai-content-generator}

इनके लिए सामग्री बनाता है:

- product विवरण
- सेवा पेज
- ब्लॉग पोस्ट
- meta descriptions
- कॉल-टू-एक्शन कॉपी

#### उन्नत डिज़ाइन टूल {#advanced-design-tools}

उन्नत डिज़ाइन क्षमताएँ प्रदान करता है:

- कस्टम CSS जनरेशन
- एनीमेशन निर्माण
- इंटरैक्टिव तत्व डिज़ाइन
- उन्नत रंग सिद्धांत
- टाइपोग्राफ़ी अनुकूलन

### Advanced साथी टूल (अलग पैकेज) {#advanced-companion-tools-separate-package}

Advanced साथी पैकेज कोर Superdav AI Agent रिलीज़ से अलग वितरित किया जाता है। यह भरोसेमंद डेवलपर और साइट-स्वामी परिवेशों के लिए है, जहाँ व्यवस्थापक agent को उच्च-प्रभाव वाले टूलिंग का उपयोग करने की स्पष्ट अनुमति देते हैं।

Advanced साथी क्षमताओं में ये शामिल हो सकते हैं:

- **फ़ाइल सिस्टम टूल** — भरोसेमंद परिवेशों में स्वीकृत फ़ाइलों की जाँच और संशोधन करें
- **डेटाबेस टूल** — स्पष्ट अनुमति होने पर साइट डेटा क्वेरी या अपडेट करें
- **WP-CLI टूल** — WordPress रखरखाव और निरीक्षण कमांड चलाएँ
- **REST डिस्पैचर टूल** — नियंत्रित डिस्पैच कार्यप्रवाहों के ज़रिए पंजीकृत REST endpoint कॉल करें
- **Plugin बिल्डर टूल** — plugin code का scaffold बनाएँ और उस पर iteration करें
- **Git स्नैपशॉट टूल** — जोखिम भरे डेवलपर ऑपरेशनों से पहले स्नैपशॉट बनाएँ
- **यूज़र-प्रबंधन टूल** — अनुमति मिलने पर यूज़र रिकॉर्ड की समीक्षा या समायोजन में मदद करें
- **बेंचमार्क टूल** — विकास समीक्षा के लिए प्रदर्शन या क्षमता बेंचमार्क चलाएँ

इन्हें हर इंस्टॉलेशन के लिए सामान्य रूप से उपलब्ध Theme Builder टूल के रूप में दस्तावेज़ित न करें। ये केवल तब उपलब्ध होते हैं जब Advanced साथी पैकेज इंस्टॉल, सक्षम और भरोसेमंद व्यवस्थापकों तक सीमित हो।

## टूल कॉन्फ़िगरेशन {#tool-configuration}

### टूल सक्षम करना {#enabling-tools}

Theme Builder में अतिरिक्त टूल सक्षम करने के लिए:

1. **Gratis AI Agent → Settings** पर जाएँ
2. **Theme Builder → Tools** पर जाएँ
3. आवश्यकता के अनुसार टूल चालू/बंद करें
4. सेटिंग्स सहेजें

### टूल अनुमतियाँ {#tool-permissions}

कुछ टूल को अनुमतियों की आवश्यकता होती है:

- **साइट scraping** — इंटरनेट पहुँच की आवश्यकता होती है
- **विश्लेषिकी** — Google Analytics इंटीग्रेशन की आवश्यकता होती है
- **सामग्री जनरेशन** — API keys की आवश्यकता होती है
- **उन्नत फ़ीचर** — प्रीमियम subscription की आवश्यकता हो सकती है
- **Advanced साथी टूल** — अलग से वितरित Advanced पैकेज और स्पष्ट व्यवस्थापक भरोसे की आवश्यकता होती है

### API Keys और क्रेडेंशियल {#api-keys-and-credentials}

जिन टूल को आवश्यकता हो, उनके लिए API keys कॉन्फ़िगर करें:

1. **Gratis AI Agent → Settings → API Keys** पर जाएँ
2. प्रत्येक टूल के लिए क्रेडेंशियल दर्ज करें
3. कनेक्शन जाँचें
4. सुरक्षित रूप से सहेजें

### Advanced साथी पैकेज इंस्टॉल करना {#installing-the-advanced-companion-package}

Advanced साथी पैकेज WordPress.org कोर ZIP से अलग रिलीज़ किया जाता है। इसे केवल रिलीज़ के आधिकारिक प्रोजेक्ट वितरण चैनल से इंस्टॉल करें, फिर पहुँच को भरोसेमंद व्यवस्थापकों तक सीमित करें। उत्पादन साइट पर फ़ाइल सिस्टम, डेटाबेस, WP-CLI, REST डिस्पैचर, plugin बिल्डर, यूज़र-प्रबंधन या बेंचमार्क टूल सक्षम करने से पहले मानवीय स्वीकृति गेट और git स्नैपशॉट व्यवहार की समीक्षा करें।

## sd-ai-agent/site-scrape का उपयोग {#using-sd-ai-agentsite-scrape}

### बुनियादी उपयोग {#basic-usage}

Theme Builder से किसी वेबसाइट का विश्लेषण करने को कहें:

```
competitor.com पर मेरे प्रतिस्पर्धी के साइट के डिज़ाइन का विश्लेषण करें
और मेरे साइट के लिए डिज़ाइन सुधार सुझाएँ।
```

### विशिष्ट विश्लेषण {#specific-analysis}

विशिष्ट प्रकार के विश्लेषण का अनुरोध करें:

```
example.com से रंग पैलेट निकालें और उसे प्रेरणा के रूप में उपयोग करें।
```

```
example.com की नेविगेशन संरचना का विश्लेषण करें और मेरे साइट पर मिलती-जुलती
व्यवस्था लागू करें।
```

```
example.com की मोबाइल अनुकूलता जाँचें और सुनिश्चित करें कि मेरा साइट
उतना ही अनुकूल हो।
```

### तुलना विश्लेषण {#comparison-analysis}

कई साइटों की तुलना करें:

```
site1.com और site2.com के डिज़ाइन की तुलना करें और ऐसा डिज़ाइन बनाएँ
जो दोनों के सबसे अच्छे तत्वों को मिलाए।
```

## टूल सीमाएँ और विचार {#tool-limitations-and-considerations}

### दर सीमा {#rate-limiting}

- सर्वर पर अधिक भार रोकने के लिए Scraping की दर सीमित की जाती है
- उसी साइट पर कई अनुरोधों को धीमा किया जा सकता है
- बड़ी साइटों का विश्लेषण करने में अधिक समय लग सकता है

### सामग्री की सटीकता {#content-accuracy}

- गतिशील सामग्री पूरी तरह कैप्चर नहीं हो सकती
- JavaScript से रेंडर की गई सामग्री अधूरी हो सकती है
- रीयल-टाइम डेटा दिखाई नहीं दे सकता
- कुछ सामग्री paywalls के पीछे हो सकती है

### कानूनी और नैतिक विचार {#legal-and-ethical-considerations}

- robots.txt और साइट नीतियों का सम्मान करें
- कॉपीराइट वाली सामग्री को पुनर्प्रकाशन के लिए scrape न करें
- विश्लेषण का उपयोग प्रेरणा के लिए करें, न कि नकल के लिए
- सुनिश्चित करें कि निकाली गई सामग्री का उपयोग करने के आपके पास अधिकार हैं
- विश्लेषित साइटों की सेवा की शर्तों का पालन करें

### प्रदर्शन पर प्रभाव {#performance-impact}

- बड़ी साइटों को scrape करने में समय लग सकता है
- एक साथ कई scrapes धीमे हो सकते हैं
- नेटवर्क कनेक्टिविटी गति को प्रभावित करती है
- बड़े डेटा सेट के लिए अधिक प्रोसेसिंग की आवश्यकता हो सकती है

## सर्वोत्तम अभ्यास {#best-practices}

### साइट विश्लेषण का उपयोग {#using-site-analysis}

1. **कई साइटों का विश्लेषण करें** — कई स्रोतों से प्रेरणा इकट्ठा करें
2. **संरचना पर ध्यान दें** — लेआउट और संगठन के पैटर्न सीखें
3. **रंग निकालें** — रंग पैलेट को शुरुआती बिंदु के रूप में उपयोग करें
4. **टाइपोग्राफी का अध्ययन करें** — अपनी पसंद के फ़ॉन्ट संयोजनों की पहचान करें
5. **नेविगेशन की समीक्षा करें** — मेन्यू संगठन को समझें

### नैतिक Scraping {#ethical-scraping}

1. **robots.txt का सम्मान करें** — साइट दिशानिर्देशों का पालन करें
2. **सामग्री कॉपी न करें** — विश्लेषण का उपयोग केवल प्रेरणा के लिए करें
3. **अधिकार सत्यापित करें** — सुनिश्चित करें कि आप निकाली गई सामग्री का उपयोग कर सकते हैं
4. **स्रोतों का उल्लेख करें** — प्रेरणा के स्रोतों को श्रेय दें
5. **शर्तों का पालन करें** — साइट की सेवा की शर्तों का पालन करें

### टूल की प्रभावशीलता बढ़ाना {#maximizing-tool-effectiveness}

1. **विशिष्ट रहें** — विश्लेषण के खास प्रकारों का अनुरोध करें
2. **संदर्भ दें** — अपनी साइट का उद्देश्य समझाएँ
3. **अपेक्षाएँ तय करें** — अपने डिज़ाइन लक्ष्यों का वर्णन करें
4. **दोहराएँ** — परिणामों के आधार पर सुधार करें
5. **टूल मिलाएँ** — व्यापक विश्लेषण के लिए कई टूल का उपयोग करें

## समस्या निवारण {#troubleshooting}

### साइट Scrape नहीं हो रही {#site-wont-scrape}

- जाँचें कि साइट सार्वजनिक रूप से सुलभ है या नहीं
- सत्यापित करें कि robots.txt scraping की अनुमति देता है
- कोई दूसरी साइट आज़माएँ
- इंटरनेट कनेक्शन जाँचें
- समस्या बनी रहे तो सहायता से संपर्क करें

### अधूरा विश्लेषण {#incomplete-analysis}

- साइट में गतिशील सामग्री हो सकती है
- JavaScript पूरी तरह रेंडर नहीं हो सकता
- बड़ी साइटों का समय समाप्त हो सकता है
- इसके बजाय खास पृष्ठों का विश्लेषण करने की कोशिश करें
- विशिष्ट विश्लेषण प्रकारों का अनुरोध करें

### धीमा प्रदर्शन {#slow-performance}

- बड़ी साइटों का विश्लेषण करने में अधिक समय लगता है
- एक साथ कई scrapes धीमे होते हैं
- नेटवर्क कनेक्टिविटी गति को प्रभावित करती है
- कम व्यस्त समय में कोशिश करें
- पहले छोटे भागों का विश्लेषण करें
