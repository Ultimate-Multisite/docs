# Ultimate AI Connector for WebLLM परिवर्तनों का विवरण (Changelog)

## Version 1.1.0 — 2026-04-09 को जारी किया गया {#version-110--released-on-2026-04-09}

* **नया:** एडमिन-बार स्टेटस इंडिकेटर के साथ फ्लोटिंग चैट विजेट — अब कोई भी लॉग-इन उपयोगकर्ता फ्रंट एंड से सीधे ब्राउज़र-साइड LLM को प्रॉम्प्ट कर सकता है।
* **नया:** SharedWorker रनटाइम — अब MLC इंजन SharedWorker में चलता है, जिससे कई टैब एक-दूसरे से GPU के लिए प्रतिस्पर्धा करने के बजाय एक ही GPU सेशन साझा कर पाते हैं।
* **नया:** apiFetch मिडलवेयर इंटरसेप्टर — WordPress REST अनुरोध जो AI Client SDK पैटर्न से मेल खाते हैं, उन्हें स्वचालित रूप से स्थानीय WebLLM ब्रोकर पर रूट किया जाता है। इसके लिए कोई लूपबैक HTTP राउंड-ट्रिप की आवश्यकता नहीं होती।
* **नया:** कनेक्टर पैनल में विजेट सेटिंग्स UI — चैट विजेट को टॉगल करने और ऑटो-प्रॉम्प्ट व्यवहार को कॉन्फ़िगर करने के लिए।
* **सुधार:** IndexedDB कैश बैकएंड को फोर्स किया गया है, ताकि मॉडल वेट डाउनलोड HuggingFace xet CDN रीडायरेक्ट से बच सकें, जो डिफ़ॉल्ट Cache API पाथ को तोड़ देते थे।
* **सुधार:** एम्बेडिंग मॉडल के लिए context_window KV-cache ओवरराइड को छोड़ दिया गया है (क्योंकि इनका कोई डीकोडर नहीं है और इस ओवरराइड के कारण रनटाइम त्रुटि हो रही थी)।
* **सुधार:** वर्कर टैब लोड होने से पहले ही /webllm/v1/models पर कोल्ड-स्टार्ट कैंडिडेट मॉडल को विज्ञापन दिया जाता है, ताकि SDK उपभोक्ता तुरंत एक मॉडल देख सकें।
* **सुधार:** WebLLM को wpai_preferred_text_models फ़िल्टर में हुक किया गया है, ताकि AI Experiments सुविधा मॉडल कॉन्फ़िगर होने पर ब्राउज़र इंजन पर रूट हो सके।
* **सुधार:** एंड-टू-एंड टेस्टिंग के दौरान कैश-बस्टिंग, कंटेंट नॉर्मलाइज़ेशन, और हार्डवेयर-रेफरेंस में सुधार किए गए।
