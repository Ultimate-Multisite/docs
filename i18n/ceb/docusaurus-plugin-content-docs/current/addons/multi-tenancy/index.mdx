# Multi-Tenancy {#multi-tenancy}

I-apod-apod ang mga subsite sa mga server ug database alang sa performance ug seguridad

## Mga destaque sa release sa Version 1.2.0 {#version-120-release-highlights}

Ang Multi-Tenancy 1.2.0 nagdugang sa sovereign tenant nga pundasyon alang sa production isolation workflows:

- Stateless tenant autologin nga adunay mubo-og-kinabuhi, replay-protected nga SSO visits.
- Per-subsite nga database ug filesystem isolation alang sa sovereign tenants.
- Hostinger multi-tenancy capability alang sa host-aware nga tenant provisioning.
- Sovereign migration verification gates alang sa legacy data checks ug async push processing.
- Configurable same-machine database host binding alang sa Bedrock, FrankenPHP, ug localhost installs.
- Mga pagpaayo sa tenant bootstrap, database writer, schema bootstrap, ug user provisioning.
- Mas luwas nga sovereign teardown nga naglimpyo sa tenant credentials.

Tan-awa ang mga giya sa tiggamit alang sa [sovereign migration workflow](/user-guide/getting-started/sovereign-migration-workflow), [multi-tenancy isolation](/user-guide/configuration/multi-tenancy-isolation), ug [sovereign tenant operations](/user-guide/administration/sovereign-tenant-operations).

## Mga Kinahanglanon {#requirements}

- WordPress 5.3 o mas taas
- PHP 7.4 o mas taas
- Multisite Ultimate plugin (aktibo)

## Pag-instalar {#installation}

1. I-upload ang mga addon file ngadto sa imong `/wp-content/plugins/` directory
2. I-activate ang plugin pinaagi sa 'Plugins' menu sa WordPress
3. I-configure ang addon settings sa network admin

## Configuration {#configuration}

Ang addon nagsuporta sa lain-laing configuration options nga ma-access pinaagi sa network admin interface.

### Sovereign tenant configuration {#sovereign-tenant-configuration}

Alang sa sovereign tenants, i-configure ang tenant database, filesystem root, database host binding, ug host capability sa wala pa i-route ang traffic ngadto sa tenant. Padagana ang migration verification human sa matag kausaban nga makaapekto sa database credentials, host binding, routing, o user provisioning.

Gamita ang `localhost` lamang kung ang tenant runtime ug database grants nagdahom og local socket o same-machine access. Gamita ang `127.0.0.1` o usa ka service hostname kung kana mao ang grant host nga gi-configure sa imong provider.

### Hostinger capability {#hostinger-capability}

Ang Hostinger multi-tenancy capability molihok uban sa shared Hostinger integration token. Kumpirmaha nga ang domain naa sa hPanel, ang DNS nagpunting sa Hostinger account, ug ang tenant database settings motakdo sa hPanel sa wala pa ang verification.
