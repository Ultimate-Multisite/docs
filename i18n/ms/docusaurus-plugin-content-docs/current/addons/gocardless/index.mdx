# GoCardless {#gocardless}

Ultimate Multisite: Pembayaran GoCardless

Benarkan pelanggan anda membayar keahlian dan produk Ultimate Multisite menggunakan Direct Debit dengan GoCardless!

## Pemasangan {#installation}

1. Muat naik 'ultimate-multisite-gocardless' ke direktori '/wp-content/plugins/'
2. Aktifkan plugin melalui menu 'Plugins' dalam WordPress
3. Ikuti Wizard langkah demi langkah untuk menyediakan plugin

## Panduan Penggunaan {#usage-guide}

_GoCardless_ ialah payment gateway yang tersedia di **Eropah dan di UK** yang membolehkan pembayaran **Direct Debit**.

**Ultimate Multisite: GoCardless Gateway** menambah sokongan _GoCardless_ kepada Ultimate Multisite, membolehkan anda mengenakan caj kepada pelanggan anda di rantau tersebut menggunakan Direct Debit.

Tutorial ini akan menerangkan cara menyediakan _GoCardless_ untuk mula menerima pembayaran melalui rangkaian Ultimate Multisite anda.

_**Nota: GoCardless hanya berfungsi dengan mata wang berikut:**_ **EUR, GBP, AUD, SEK, dan DKK.**

## Memasang Add-on {#installing-the-add-on}

Pertama sekali, anda perlu memasang add-on GoCardless untuk Ultimate Multisite. Anda boleh melakukannya dengan pergi ke **Ultimate Multisite > Settings > Add-ons (di sidebar)**. Kemudian pilih add-on GoCardless dalam senarai tersebut.

Selepas add-on dipasang dan diaktifkan, pergi ke **Ultimate Multisite > Settings > Payments**, hidupkan pilihan GoCardless, dan simpan.

![Payment gateway settings](/img/config/settings-payment-gateways.png)

## Mendapatkan Access Token {#getting-an-access-token}

Buat akaun di _GoCardless_ (atau di _GoCardless_ sandbox, jika anda ingin **mengujinya dahulu**, yang disyorkan). Kemudian, selepas log masuk, navigasi ke item menu **Developers**.

<!-- Screenshot unavailable: GoCardless dashboard showing Developers menu position -->

Di halaman **Developers**, klik butang **Create** di sudut kanan atas skrin, kemudian klik pada pilihan **Access Token**.

<!-- Screenshot unavailable: GoCardless Developers page with Create button and Access Token option -->

Ini akan membuka tetingkap modal baharu, membolehkan anda mencipta token baharu. Beri nama yang mudah dikenali dan pastikan anda memilih pilihan **Read-write access** dalam menu **Scope**.

<!-- Screenshot unavailable: GoCardless modal for creating new access token with Read-write scope -->

Access token akan dipaparkan di skrin. Salin token itu dan tampalkannya ke menu **Ultimate Multisite > Settings > Payments > GoCardless** anda.

Pastikan anda meletakkannya di medan yang betul. Jika ia adalah **live token**, hidupkan **sandbox mode off** sebelum menampal token.

![Payment gateway settings with GoCardless](/img/config/settings-payment-gateways.png)

<!-- Screenshot unavailable: GoCardless token fields in Ultimate Multisite payment settings -->

Itu sahaja untuk langkah pertama. Seterusnya, anda perlu menyediakan pendengar webhook.

## Menyediakan Pendengar Webhook {#setting-up-the-webhook-listener}

Apabila berurusan dengan langganan, Ultimate Multisite perlu mengetahui tentang perubahan status langganan, pembayaran baharu yang dibuat, dan perkara serupa.

Cara payment gateway memberitahu Ultimate Multisite tentang perubahan tersebut adalah menggunakan **panggilan webhook**. Oleh sebab itu, untuk integrasi berfungsi 100%, kita perlu memberitahu _GoCardless_ URL mana yang perlu dipanggil dengan semua maklumat tersebut.

Kembali ke panel **GoCardless > Developers** anda dan klik butang Create sekali lagi, tetapi kali ini, pilih pilihan **Webhook endpoint**.

<!-- Screenshot unavailable: GoCardless Developers page with Webhook endpoint option -->A tetingkap baharu akan terbuka dengan pilihan webhook endpoint. Ia meminta URL webhook, jadi kembali ke **Ultimate Multisite settings > Payments > GoCardless** anda dan salin URL webhook yang disenaraikan di sana:

<!-- Screenshot unavailable: Ultimate Multisite GoCardless settings showing webhook URL -->

Selain itu, anda boleh memasukkan **webhook secret** yang akan digunakan untuk menandatangani panggilan webhook anda. Apa sahaja yang anda masukkan di medan itu, anda juga perlu memasukkannya di medan **Ultimate Multisite Webhook Secret**.

<!-- Screenshot unavailable: GoCardless webhook endpoint configuration form with URL and secret fields -->

<!-- Screenshot unavailable: Ultimate Multisite GoCardless webhook secret field -->

Selesaikan dengan menyimpan endpoint dengan mengklik butang **Create webhook endpoint** dan kemudian **simpan tetapan Ultimate Multisite anda juga**.

Anda sepatutnya sudah bersedia untuk menerima _GoCardless_ sekarang!
