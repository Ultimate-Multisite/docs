# Berbilang Penyewa

Agihkan sublaman merentas pelayan dan pangkalan data untuk prestasi dan keselamatan

## Sorotan keluaran versi 1.2.0 {#version-120-release-highlights}

Berbilang Penyewa 1.2.0 menambah asas penyewa berdaulat untuk aliran kerja pengasingan produksi:

- Autologin penyewa tanpa keadaan dengan lawatan SSO jangka pendek yang dilindungi daripada main semula.
- Pengasingan pangkalan data dan sistem fail mengikut sublaman untuk penyewa berdaulat.
- Keupayaan berbilang penyewa Hostinger untuk peruntukan penyewa yang sedar hos.
- Pintu pengesahan migrasi berdaulat untuk semakan data warisan dan pemprosesan tolakan tak segerak.
- Pengikatan hos pangkalan data mesin yang sama yang boleh dikonfigurasi untuk pemasangan Bedrock, FrankenPHP, dan localhost.
- Penambahbaikan bootstrap penyewa, penulis pangkalan data, bootstrap skema, dan peruntukan pengguna.
- Pembongkaran berdaulat yang lebih selamat yang membersihkan kelayakan penyewa.

Lihat panduan pengguna untuk [aliran kerja migrasi berdaulat](/user-guide/getting-started/sovereign-migration-workflow), [pengasingan berbilang penyewa](/user-guide/configuration/multi-tenancy-isolation), dan [operasi penyewa berdaulat](/user-guide/administration/sovereign-tenant-operations).

## Keperluan {#requirements}

- WordPress 5.3 atau lebih tinggi
- PHP 7.4 atau lebih tinggi
- pemalam Multisite Ultimate (aktif)

## Pemasangan {#installation}

1. Muat naik fail tambahan ke direktori `/wp-content/plugins/` anda
2. Aktifkan pemalam melalui menu 'Pemalam' dalam WordPress
3. Konfigurasikan tetapan tambahan dalam pentadbir rangkaian

## Pembangunan {#development}

### Persediaan {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Pengujian {#testing}

Tambahan ini merangkumi rangka kerja pengujian yang menyeluruh:

- **Ujian Unit**: Ujian berasaskan PHPUnit dengan suite ujian WordPress
- **Kelas Ujian Asas**: Menyediakan kaedah pembantu untuk senario ujian biasa
- **Liputan Ujian**: Dikonfigurasikan untuk menjejak liputan kod
- **Pengujian Multisite**: Ujian dijalankan dalam persekitaran multisite

### Piawaian Kod {#code-standards}

- **Piawaian Pengekodan WordPress**: Dikuatkuasakan melalui PHPCS
- **Keserasian PHP 7.4+**: Ciri PHP moden disokong
- **Analisis Statik**: Penyepaduan PHPStan untuk semakan jenis
- **Pemodenan Kod**: Peraturan Rector untuk peningkatan PHP

### Sistem Binaan {#build-system}

- **Pemprosesan Aset**: Pengecilan untuk fail CSS/JS
- **Terjemahan**: Penjanaan fail POT
- **Pembungkusan**: Penciptaan arkib untuk pengedaran
- **Mod Pembangunan**: Aset tidak dikecilkan untuk penyahpepijatan

## Konfigurasi {#configuration}

Tambahan ini menyokong pelbagai pilihan konfigurasi yang boleh diakses melalui antara muka pentadbir rangkaian.

### Konfigurasi penyewa berdaulat {#sovereign-tenant-configuration}

Untuk penyewa berdaulat, konfigurasikan pangkalan data penyewa, akar sistem fail, pengikatan hos pangkalan data, dan keupayaan hos sebelum menghalakan trafik kepada penyewa. Jalankan pengesahan migrasi selepas setiap perubahan yang menjejaskan kelayakan pangkalan data, pengikatan hos, penghalaan, atau peruntukan pengguna.

Gunakan `localhost` hanya apabila masa jalan penyewa dan geran pangkalan data menjangkakan akses soket setempat atau mesin yang sama. Gunakan `127.0.0.1` atau nama hos perkhidmatan apabila itu ialah hos geran yang dikonfigurasikan oleh penyedia anda.

### Keupayaan Hostinger {#hostinger-capability}

Keupayaan berbilang penyewa Hostinger berfungsi dengan token penyepaduan Hostinger yang dikongsi. Sahkan domain hadir dalam hPanel, DNS menunjuk kepada Account Hostinger, dan tetapan pangkalan data penyewa sepadan dengan hPanel sebelum pengesahan.

## Hook dan Penapis {#hooks-and-filters}

### Tindakan {#actions}

- `ultimate-multisite-addon-template_init` - Dicetuskan selepas pemulaan tambahan
- `ultimate-multisite-addon-template_loaded` - Dicetuskan selepas semua fail tambahan dimuatkan

### Penapis {#filters}

- `ultimate-multisite-addon-template_settings` - Ubah suai tetapan tambahan
- `ultimate-multisite-addon-template_enabled` - Pintas status dayakan/nyahdayakan tambahan
