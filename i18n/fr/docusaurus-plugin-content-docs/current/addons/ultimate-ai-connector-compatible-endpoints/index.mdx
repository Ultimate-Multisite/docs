# Connecteur IA Ultimate pour Endpoints Compatibles

Le Connecteur IA Ultimate pour Endpoints Compatibles intègre les API de fournisseurs d'IA compatibles OpenAI avec votre réseau WordPress multisite. Configurez plusieurs fournisseurs d'IA — chacun avec son propre URL de point de terminaison (endpoint), clé API et modèle par défaut — et acheminez les requêtes avec un mécanisme de secours automatique lorsqu'un fournisseur est indisponible.

## Fonctionnalités Clés

- **Prise en charge multi-fournisseurs** : Ajoutez plusieurs cartes de fournisseurs d'IA, chacune avec son propre URL de point de terminaison, clé API et modèle par défaut.
- **Réorganisation par glisser-déposer** : Organisez les fournisseurs par ordre de priorité en utilisant la poignée de chaque carte ; les requêtes utilisent les fournisseurs dans l'ordre.
- **Mécanisme de secours automatique** : Si le fournisseur principal est indisponible, les requêtes sont automatiquement transférées au fournisseur configuré suivant.
- **Configuration par fournisseur** : Chaque fournisseur est configuré indépendamment avec ses propres identifiants et la sélection de modèle.
- **Interface utilisateur basée sur des cartes** : Construit sur les composants Card stables de WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **Expansion automatique lors de l'ajout** : Les nouvelles cartes de fournisseurs s'expandent automatiquement afin que vous puissiez saisir les détails immédiatement.
- **Anti-cache des scripts** : Les scripts des fournisseurs sont anti-cachés lors de la mise à jour du plugin pour garantir que la dernière version est chargée.

## Prérequis

- WordPress 5.3 ou supérieur
- PHP 7.4 ou supérieur
- Plugin Ultimate Multisite (actif)
- Un point de terminaison API compatible OpenAI (auto-hébergé ou tiers)

## Installation

1. Téléchargez les fichiers de l'addon dans votre répertoire `/wp-content/plugins/`
2. Activez le plugin via le menu 'Plugins' de WordPress
3. Accédez à **Ultimate Multisite → Connecteur IA** pour configurer l'addon

## Configuration

### Configuration Multi-Fournisseurs

L'écran de configuration utilise une disposition basée sur des cartes où chaque carte représente un fournisseur d'IA. Vous pouvez ajouter autant de fournisseurs que nécessaire.

#### Ajouter un Fournisseur

1. Allez dans **Ultimate Multisite → Connecteur IA** dans l'administration du réseau.
2. Cliquez sur **Ajouter un fournisseur** (Add Provider).
3. Une nouvelle carte de fournisseur apparaît et s'expande automatiquement.
4. Remplissez les détails du fournisseur :
   - **Nom du fournisseur** (Provider Name) — une étiquette pour identifier ce fournisseur (ex. : "OpenAI", "Mistral", "Local Ollama").
   - **URL du point de terminaison** (Endpoint URL) — l'URL de base de l'API compatible OpenAI (ex. : `https://api.openai.com/v1`).
   - **Clé API** (API Key) — votre clé API pour ce fournisseur.
   - **Modèle par défaut** (Default Model) — l'identifiant de modèle à utiliser par défaut pour ce fournisseur (ex. : `gpt-4o`, `mistral-small`).
5. Cliquez sur **Enregistrer les paramètres** (Save Settings).

#### Supprimer un Fournisseur

Cliquez sur le bouton **Supprimer** (Remove) sur n'importe quelle carte de fournisseur pour la supprimer. Enregistrez les paramètres pour appliquer le changement.

#### Réorganiser les Fournisseurs

Faites glisser l'icône de poignée (⣿) sur le côté gauche de n'importe quelle carte de fournisseur pour la réorganiser. L'ordre détermine la priorité de secours : les requêtes sont envoyées au premier fournisseur et sont transférées aux fournisseurs suivants si une requête échoue.

### Mécanisme de Secours (Fallback Routing)

Lorsqu'une requête au fournisseur principal échoue — en raison d'un délai d'attente, d'une limite de débit ou d'une panne de service — le connecteur tente automatiquement la requête avec le fournisseur suivant de la liste. Ce processus se poursuit dans la liste jusqu'à ce qu'un fournisseur réussisse ou que tous les fournisseurs aient été essayés.

| Scénario | Comportement |
|----------|-----------|
| Le fournisseur principal répond | La requête est traitée normalement |
| Le fournisseur principal échoue | La requête est retentée avec le fournisseur suivant |
| Tous les fournisseurs échouent | Une erreur est renvoyée à l'appelant |

Le mécanisme de secours est transparent pour le plugin ou le thème appelant — ils reçoivent soit une réponse réussie, soit une erreur finale, quelle que soit la quantité de fournisseurs essayés.

### Référence de Configuration des Fournisseurs

| Champ | Requis | Description |
|-------|----------|-------------|
| Provider Name | Oui | Étiquette d'affichage pour ce fournisseur |
| Endpoint URL | Oui | URL de base de l'API compatible OpenAI |
| API Key | Oui | Clé d'authentification pour ce fournisseur |
| Default Model | Oui | Identifiant de modèle utilisé lorsqu'aucun modèle n'est spécifié dans la requête |

## Dépannage

### Aucun fournisseur n'est listé

Si la liste des fournisseurs est vide après l'enregistrement, veuillez vérifier que :

- Vous avez cliqué sur **Enregistrer les paramètres** après avoir ajouté les fournisseurs.
- Votre navigateur n'a pas bloqué l'enregistrement de la page de paramètres (vérifiez la console du navigateur pour les erreurs JavaScript).
- Le plugin est actif et que vous êtes sur l'écran d'administration du réseau.

### Les requêtes échouent pour tous les fournisseurs

- Vérifiez que l'URL du point de terminaison de chaque fournisseur est accessible depuis votre serveur (et pas seulement depuis votre navigateur).
- Assurez-vous que chaque clé API est valide et n'a pas expiré.
- Examinez le statut de la limite de débit et du quota du fournisseur dans son tableau de bord.
- Activez le journal de débogage WordPress (`WP_DEBUG_LOG`) pour capturer des messages d'erreur détaillés provenant du connecteur.

### Un fournisseur n'est pas utilisé pour le mécanisme de secours

Le glisser-déposer modifie l'ordre de secours. Confirmez que le fournisseur que vous attendez pour recevoir les requêtes de secours est positionné correctement dans la liste et que son URL de point de terminaison et sa clé API sont correctes.

### La nouvelle carte de fournisseur ne s'expande pas automatiquement

L'expansion automatique nécessite que JavaScript soit activé dans votre navigateur. Si la carte ne s'expande pas au clic, vérifiez la console du navigateur pour des erreurs. Le contenu de la carte est toujours accessible en cliquant sur l'en-tête de la carte même si l'expansion automatique ne se déclenche pas.
