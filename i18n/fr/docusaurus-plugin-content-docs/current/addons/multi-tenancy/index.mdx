# Multi-tenance

Répartissez les sous-sites entre des serveurs et des bases de données pour les performances et la sécurité

## Points forts de la version 1.2.0 {#version-120-release-highlights}

Multi-tenance 1.2.0 ajoute la fondation de tenants souverains pour les workflows d’isolation en production :

- Connexion automatique de tenant sans état avec des visites SSO de courte durée, protégées contre la relecture.
- Isolation par sous-site de la base de données et du système de fichiers pour les tenants souverains.
- Capacité de multi-tenance Hostinger pour le provisionnement de tenants tenant compte de l’hôte.
- Portes de vérification de migration souveraine pour les contrôles de données héritées et le traitement push asynchrone.
- Liaison configurable de l’hôte de base de données sur la même machine pour les installations Bedrock, FrankenPHP et localhost.
- Améliorations de l’amorçage des tenants, du rédacteur de base de données, de l’amorçage du schéma et du provisionnement des utilisateurs.
- Démantèlement souverain plus sûr qui nettoie les identifiants des tenants.

Consultez les guides utilisateur pour le [workflow de migration souveraine](/user-guide/getting-started/sovereign-migration-workflow), l’[isolation de multi-tenance](/user-guide/configuration/multi-tenancy-isolation) et les [opérations de tenants souverains](/user-guide/administration/sovereign-tenant-operations).

## Exigences {#requirements}

- WordPress 5.3 ou supérieur
- PHP 7.4 ou supérieur
- Extension Multisite Ultimate (active)

## Installation {#installation}

1. Téléversez les fichiers du module complémentaire dans votre répertoire `/wp-content/plugins/`
2. Activez l’extension via le menu « Extensions » dans WordPress
3. Configurez les réglages du module complémentaire dans l’administration du réseau

Le module complémentaire inclut un framework de tests complet :

- **Tests unitaires** : tests basés sur PHPUnit avec la suite de tests WordPress
- **Classe de test de base** : fournit des méthodes d’aide pour les scénarios de test courants
- **Couverture de test** : configurée pour suivre la couverture du code
- **Tests multisite** : les tests s’exécutent dans un environnement multisite

- **Standards de codage WordPress** : appliqués via PHPCS
- **Compatibilité PHP 7.4+** : fonctionnalités PHP modernes prises en charge

## Configuration {#configuration}

Le module complémentaire prend en charge diverses options de configuration accessibles via l’interface d’administration du réseau.

### Configuration des tenants souverains {#sovereign-tenant-configuration}

Pour les tenants souverains, configurez la base de données du tenant, la racine du système de fichiers, la liaison de l’hôte de base de données et la capacité de l’hôte avant d’acheminer le trafic vers le tenant. Exécutez la vérification de migration après chaque changement qui affecte les identifiants de base de données, la liaison de l’hôte, le routage ou le provisionnement des utilisateurs.

Utilisez `localhost` uniquement lorsque l’environnement d’exécution du tenant et les autorisations de base de données attendent un socket local ou un accès sur la même machine. Utilisez `127.0.0.1` ou un nom d’hôte de service lorsque c’est l’hôte d’autorisation configuré par votre fournisseur.

### Capacité Hostinger {#hostinger-capability}

La capacité de multi-tenance Hostinger fonctionne avec le jeton d’intégration Hostinger partagé. Confirmez que le domaine est présent dans hPanel, que le DNS pointe vers le Account Hostinger et que les réglages de base de données du tenant correspondent à hPanel avant la vérification.

## Hooks et filtres {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - Déclenché après l’initialisation du module complémentaire
- `ultimate-multisite-addon-template_loaded` - Déclenché après le chargement de tous les fichiers du module complémentaire

### Filtres {#filters}

- `ultimate-multisite-addon-template_settings` - Modifier les réglages du module complémentaire
- `ultimate-multisite-addon-template_enabled` - Remplacer l’état d’activation/désactivation du module complémentaire
