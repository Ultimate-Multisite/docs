# Multi-Network {#multi-network}

L’addon Ultimate Multisite: Multinetwork étend votre plateforme Ultimate Multisite avec de puissantes fonctionnalités multinetwork. Cet addon permet à vos clients d’acheter et de gérer des réseaux multisites WordPress entiers, et pas seulement des sites individuels.

## Fonctionnalités clés {#key-features}

* **Provisionnement de réseau**: Créez et configurez automatiquement des réseaux multisites WordPress complets pour vos clients
* **Modèles de réseau**: Permettez aux clients de parcourir, prévisualiser et choisir un modèle de réseau prédéfini avant l’inscription
* **Gestion de réseau**: Capacités complètes de gestion pour les environnements multinetwork
* **Réseaux isolés**: Chaque réseau client fonctionne indépendamment avec ses propres tables de base de données et sa propre configuration
* **SSO inter-réseaux**: Offrez aux clients un accès fluide à leurs sous-réseaux avec une authentification par lien magique
* **Architecture évolutive**: Gérez efficacement plusieurs réseaux grâce à une allocation des ressources optimisée
* **Intégration**: S’intègre de manière transparente aux systèmes d’abonnement et de facturation d’Ultimate Multisite

## Cas d’utilisation {#use-cases}

* **Hébergement en marque blanche**: Fournissez aux clients leurs propres réseaux multisites à leur marque
* **Solutions pour agences**: Permettez aux agences de gérer plusieurs réseaux de clients depuis une seule plateforme
* **Déploiements d’entreprise**: Prenez en charge les grandes organisations ayant des exigences multi-réseaux complexes
* **Plateformes éducatives**: Créez des réseaux séparés pour différentes écoles ou différents départements

## Exigences {#requirements}

* Ultimate Multisite 2.0.0 ou supérieur
* Installation WordPress Multisite
* PHP 7.4 ou supérieur
* MySQL 5.6 ou supérieur

## Installation {#installation}

1. Téléversez les fichiers du plugin dans le répertoire `/wp-content/plugins/ultimate-multisite-multinetwork`
2. Activez le plugin via l’écran « Plugins » dans WordPress
3. Configurez les réglages multinetwork via Ultimate Multisite → Réglages

## Modèles de réseau {#network-templates}

La version 1.3.0 ajoute un flux de découverte et de sélection de modèles pour les clients qui achètent de nouveaux réseaux. Utilisez des modèles de réseau lorsque vous souhaitez que chaque nouveau réseau client démarre à partir d’une structure, d’un design, d’une bibliothèque de contenu ou d’une configuration spécifique à un secteur déjà préparés.

### Outil de prévisualisation des modèles {#template-previewer}

L’outil de prévisualisation des modèles de réseau permet aux clients de parcourir les modèles disponibles avant le checkout. Chaque modèle peut être ouvert dans un panneau de prévisualisation en direct, afin que les clients puissent comparer les mises en page et le contenu avant d’acheter un réseau.

Utilisez l’outil de prévisualisation pour aider les clients à :

* Examiner l’apparence et la structure de chaque modèle de réseau disponible
* Comparer plusieurs modèles sans quitter le flux d’achat
* Choisir le modèle qui correspond le mieux à leur cas d’utilisation avant l’inscription

Gardez les noms et descriptions de modèles faciles à comprendre pour les clients. L’outil de prévisualisation est plus utile lorsque les clients peuvent comprendre le cas d’utilisation prévu, comme les réseaux d’agence, les réseaux scolaires, les réseaux de franchise ou les offres d’hébergement en marque blanche.

### Sélecteur de modèle au checkout {#template-picker-at-checkout}

Le sélecteur de modèle peut être ajouté au flux de checkout d’inscription en tant que champ. Lorsqu’il est configuré, les clients sélectionnent le modèle de réseau qu’ils souhaitent pendant le checkout, et le modèle sélectionné est utilisé lorsque leur nouveau réseau est provisionné.

Pour utiliser le sélecteur dans un flux de checkout :

1. Préparez les réseaux que vous souhaitez proposer comme modèles.
2. Activez le champ de sélection de modèle Multinetwork dans le formulaire d’inscription ou de checkout pertinent.
3. Choisissez quels modèles doivent être disponibles pour ce flux de checkout.
4. Publiez le flux de checkout et testez une inscription pour confirmer que le modèle sélectionné est appliqué au nouveau réseau.

Si un flux de checkout ne propose qu’un seul produit de réseau, le sélecteur peut tout de même être utile comme étape d’onboarding guidée. Pour les tunnels multi-plans, exposez uniquement les modèles compatibles avec le produit ou le plan sélectionné.

## SSO par lien magique inter-réseaux {#cross-network-magic-link-sso}

Le SSO par lien magique inter-réseaux permet aux clients de passer de leur zone de compte principale à leurs sous-réseaux sans avoir à se connecter manuellement à nouveau. Lorsqu’un client suit un lien d’accès inter-réseaux, Ultimate Multisite génère un lien magique à durée limitée et authentifie le client dans le réseau de destination.

Cela améliore l’expérience des clients qui gèrent plus d’un réseau ou qui doivent accéder à des réseaux de modèles nouvellement clonés immédiatement après l’achat.

Liste de vérification pour l’administrateur :

* Confirmez que l’addon Multinetwork est actif sur toute l’installation multisite WordPress.
* Assurez-vous que les clients disposent de l’accès réseau correct avant que des liens inter-réseaux ne leur soient proposés.
* Gardez les domaines de réseau et les domaines mappés correctement configurés afin que les liens magiques se résolvent vers le réseau de destination prévu.
* Testez le Dashboard client et tous les boutons d’accès post-checkout après avoir modifié les réglages de réseau, de domaine ou de checkout.

Les liens magiques doivent être traités comme des liens de connexion. Ils sont conçus pour un accès client fluide et ne doivent pas être transférés ni réutilisés en dehors de la session prévue.

## Questions fréquentes {#frequently-asked-questions}

### Qu’est-ce que multinetwork ? {#what-is-multinetwork}

Multinetwork est une fonctionnalité WordPress qui vous permet d’exécuter plusieurs réseaux multisites indépendants sur une seule installation WordPress.

### En quoi est-ce différent du multisite classique ? {#how-is-this-different-from-regular-multisite}

Le multisite classique permet plusieurs sites au sein d’un seul réseau. Multinetwork permet plusieurs réseaux, chacun avec ses propres sites, tous au sein d’une seule installation WordPress.

### Cela nécessite-t-il une configuration serveur spéciale ? {#does-this-require-special-server-configuration}

Oui, multinetwork nécessite une installation et une configuration serveur spécifiques. Veuillez consulter la documentation pour obtenir des instructions de configuration détaillées.
