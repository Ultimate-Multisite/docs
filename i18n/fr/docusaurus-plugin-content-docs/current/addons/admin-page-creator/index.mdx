# Admin Page Creator

## Contenu
Admin page creator vous permettra d'ajouter facilement de nouvelles pages d'administration et sous-pages dans le tableau de bord d'administration WordPress. Il peut être utilisé pour créer une page d'accueil ou une page tutoriel qui peut servir de page d'atterrissage pour les nouveaux utilisateurs. Vous pouvez utiliser des constructeurs de pages tels que Beaver Builder, Elementor, Brizy et Oxygen pour concevoir les pages sans tracas.

Il offre également un support HTML, CSS et JavaScript personnalisés, ainsi qu'un contrôle d'accès basé sur le plan et le rôle de l'utilisateur !

## Fonctionnalités
* Support HTML et CSS personnalisés
* Support JavaScript
* Mode de contenu PHP
* Permission basée sur le rôle de l'utilisateur
* Support aux constructeurs de pages

## Installation

1. Téléchargez **ultimate-multisite-admin-page-creator** dans le répertoire **/wp-content/plugins/**
2. Activez le plugin via le menu **Plugins** dans votre réseau d'administration WordPress

## Guide d'utilisation

Alors que WordPress rend vraiment facile l'ajout de contenu à l'interface publique des sites, la création de pages et de sous-pages côté administration nécessite certaines compétences de base en codage. Ultimate Multisite: Admin Page Creator vise à apporter cette même facilité d'utilisation de l'interface publique au côté administration de vos sites réseau.

## Pourquoi utiliser Admin Page Creator ?

Peut-être souhaitez-vous proposer une page personnalisée sur le panneau d'administration de vos clients avec des tutoriels, ou bien ajouter une page personnalisée avec des FAQ et un widget de contact. Les possibilités sont infinies avec Admin Page Creator, et c'est ce que nous allons explorer dans le tutoriel d'aujourd'hui.

## Créer votre première page d'administration personnalisée

Après avoir installé et activé Ultimate Multisite: Admin Page Creator, accédez à votre menu d'administration réseau et allez dans Pages d'administration.

![Custom Admin Pages list on Network Admin](/img/admin/admin-pages-list.png)

Vous verrez une liste de toutes les pages d'administration que vous avez créées jusqu'à présent (puisque c'est votre première visite, la liste sera vide). Cliquez sur **Add new Admin Page**. Vous serez redirigé vers l'éditeur de page d'administration.

### Titre de la page

C'est le titre qui sera affiché en haut de la page, et non l'étiquette du menu d'administration.

<!-- Screenshot unavailable: Admin Page editor showing Page Title field -->

### Sélecteur de type de contenu

Ultimate Multisite: Admin Page Creator vous propose deux options concernant le contenu de la page : vous pouvez soit utiliser l'éditeur WYSIWYG par défaut de WordPress, soit passer à un éditeur HTML.

Si vous n'êtes pas familier avec HTML, restez sur l'éditeur WordPress par défaut. Vous pourrez modifier le contenu de la page comme vous le feriez avec un article ou une page normale.

## Options de menu

Sur le côté, dans la boîte méta Options générales, vous pourrez configurer le type de menu, l'étiquette et bien plus encore.

### Types de menu

Votre nouvelle page d'administration personnalisée peut avoir deux types de menu différents : menu de premier niveau et sous-menu.

Une page de menu de premier niveau, comme son nom l'indique, ajoutera votre page nouvellement créée au menu d'administration aux côtés des pages WordPress par défaut.

Si vous choisissez l'option de premier niveau, vous pourrez également sélectionner l'ordre du menu et une icône de menu.

Si vous souhaitez faire de cette page un sous-menu, vous pourrez sélectionner une page parente à laquelle attacher cette page. Vous pouvez ajouter votre page personnalisée aux pages WordPress par défaut ou à vos pages d'administration personnalisées de premier niveau.

## Options avancées

Vous devriez pouvoir ajouter toutes sortes de contenus à vos pages d'administration personnalisées. Cela inclut des règles et fichiers CSS, JavaScript et des bibliothèques externes. Vous pouvez le faire en utilisant l'onglet Options avancées.

### CSS personnalisé

Dans l'onglet CSS, vous pouvez ajouter votre propre code CSS personnalisé, ainsi qu'importer des fichiers CSS depuis des sites distants (tels que les CDN, Google Fonts, etc.).

<!-- Screenshot unavailable: Admin Page Creator custom CSS editor with external stylesheet import -->

### JavaScript personnalisé

Il en va de même pour JavaScript. Vous pouvez ajouter des bibliothèques externes à utiliser dans votre code JavaScript personnalisé.

<!-- Screenshot unavailable: Admin Page Creator custom JavaScript editor with external library import -->

## Permissions

Également dans les Options avancées, vous pourrez sélectionner les sous-sites qui auront accès à cette page. Cela est utile si vous prévoyez de fournir du contenu exclusif différent à différents niveaux de plan, par exemple.

<!-- Screenshot unavailable: Admin Page Creator permissions settings for plan and role-based access control -->

**Important** : Les deux conditions doivent être remplies pour que la page soit affichée. Cela signifie que si l'utilisateur est client d'un plan donné, mais ne possède pas l'un des rôles sélectionnés (ou vice-versa), la page n'apparaîtra pas sur son panneau.

## Actif

Si vous souhaitez désactiver une page d'administration personnalisée sans la supprimer ou modifier les permissions, décochez simplement l'option Actif.

Désactiver une page de premier niveau désactivera automatiquement toutes les sous-pages attachées à cette page parente.
