# Admin Page Creator

## Contenu {#content}
Admin Page Creator vous permettra d’ajouter facilement de nouvelles pages d’administration et sous-pages dans le Dashboard d’administration WordPress. Il peut être utilisé pour créer une page de bienvenue ou une page de tutoriel pouvant servir de page d’atterrissage pour les nouveaux utilisateurs. Vous pouvez utiliser des page builders comme Beaver Builder, Elementor, Brizy et Oxygen pour concevoir les pages sans difficulté.

Il offre également la prise en charge du HTML, CSS et JavaScript personnalisés, ainsi qu’un contrôle d’accès basé sur le plan et le rôle utilisateur !

## Fonctionnalités {#features}
* Prise en charge du HTML et du CSS personnalisés
* Prise en charge de JavaScript
* Mode de contenu PHP
* Autorisation basée sur le rôle utilisateur
* Prise en charge des page builders

## Installation {#installation}

1. Téléversez **ultimate-multisite-admin-page-creator** dans le répertoire **/wp-content/plugins/**
2. Activez le plugin via le menu **Plugins** dans votre administration du réseau WordPress

## Guide d’utilisation {#usage-guide}

Bien que WordPress permette très facilement d’ajouter du contenu à l’interface publique des sites, la création de pages et de sous-pages côté administration nécessite quelques compétences de base en codage. Ultimate Multisite: Admin Page Creator vise à apporter cette même facilité d’utilisation de l’interface publique à l’interface d’administration des sites de votre réseau.

## Pourquoi utiliser Admin Page Creator ? {#why-use-admin-page-creator}

Peut-être souhaitez-vous proposer une page personnalisée sur le panneau d’administration de vos clients avec des tutoriels, ou peut-être voulez-vous ajouter une page personnalisée avec une FAQ et un widget de contact. Les possibilités sont infinies avec Admin Page Creator, et c’est ce que nous allons explorer dans le tutoriel d’aujourd’hui.

## Créer votre première page d’administration personnalisée {#creating-your-first-custom-admin-page}

Après avoir installé et activé Ultimate Multisite: Admin Page Creator, accédez au menu d’administration de votre réseau et allez dans Pages d’administration.

![Liste des pages d’administration personnalisées dans l’administration du réseau](/img/admin/admin-pages-list.png)

_Liste des pages d’administration personnalisées dans l’administration du réseau_

Vous verrez une liste de toutes les pages d’administration que vous avez créées jusqu’à présent (comme il s’agit de votre première visite, la liste sera vide). Cliquez sur **Ajouter une nouvelle page d’administration**. Vous serez redirigé vers l’éditeur de page d’administration.

## Les éléments d’une page d’administration personnalisée {#the-elements-of-a-custom-admin-page}

Dans l’éditeur de page d’administration, vous disposerez de plusieurs options différentes pour personnaliser votre page.

### Titre de la page {#page-title}

Il s’agit du titre qui sera affiché en haut de la page, et non du libellé du menu d’administration.

<!-- Screenshot unavailable: Admin Page editor showing Page Title field -->

_Saisissez un titre de page_

### Sélecteur de type de contenu {#content-type-selector}

Ultimate Multisite: Admin Page Creator vous propose deux options concernant le contenu de la page : vous pouvez soit utiliser l’éditeur WYSIWYG WordPress par défaut, soit passer à un éditeur HTML.

Si vous n’êtes pas familier avec le HTML, restez sur l’éditeur WordPress par défaut. Vous pourrez modifier le contenu de la page exactement comme vous le feriez avec une publication ou une page normale.

## Options de menu {#menu-options}

Sur le côté, dans la méta-boîte Options générales, vous pourrez configurer le type de menu, le libellé et bien plus encore.

### Types de menus {#menu-types}

Votre nouvelle page d’administration personnalisée peut avoir deux types de menus différents : menu de premier niveau et sous-menu.

Une page de menu de premier niveau, comme son nom l’indique, ajoutera votre page nouvellement créée au menu d’administration aux côtés des pages WordPress par défaut.

Si vous sélectionnez l’option de premier niveau, vous pourrez également sélectionner l’ordre du menu et une icône de menu.

Si vous souhaitez faire de cette page une page de sous-menu, vous pourrez sélectionner une page parente à laquelle rattacher cette page. Vous pouvez ajouter votre page personnalisée aux pages WordPress par défaut ou à vos pages d’administration personnalisées de premier niveau.

## Options avancées {#advanced-options}

Vous devriez pouvoir ajouter toutes sortes de contenu à vos pages d’administration personnalisées. Cela inclut également des règles et fichiers CSS, JavaScript et des bibliothèques externes. Vous pouvez le faire à l’aide de l’onglet Options avancées.

### CSS personnalisé {#custom-css}

Dans l’onglet CSS, vous pouvez ajouter votre propre code CSS personnalisé, ainsi qu’importer des fichiers CSS depuis des sites distants (comme des CDN, Google Fonts, etc.).

_Le CSS personnalisé est également pris en charge. Vous pouvez aussi inclure des fichiers de style externes !_

Il en va de même pour JavaScript. Vous pouvez ajouter des bibliothèques externes à utiliser dans votre code JavaScript personnalisé.

## Autorisations {#permissions}

Toujours dans les Options avancées, vous pourrez sélectionner les sous-sites qui auront accès à cette page. C’est utile si vous prévoyez de proposer différents contenus exclusifs à différents niveaux de plan, par exemple.

<!-- Screenshot unavailable: Admin Page Creator permissions settings for plan and role-based access control -->

_Vous avez un contrôle total sur l’endroit où cette page va s’afficher_

**Important** : Les deux conditions doivent être remplies pour que la page soit affichée. Cela signifie que si l’utilisateur est un client d’un plan donné, mais n’a pas l’un des rôles sélectionnés (ou inversement), la page n’apparaîtra pas dans son panneau.

## Actif {#active}

Si vous souhaitez désactiver une page d’administration personnalisée sans la supprimer ni modifier les autorisations, décochez simplement l’option Active.

La désactivation d’une page de premier niveau désactivera automatiquement toutes les sous-pages rattachées à cette page parente.
