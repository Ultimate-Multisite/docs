# GoCardless

Ultimate Multisite : paiement GoCardless

Permettez à vos clients de payer les memberships et produits Ultimate Multisite par prélèvement automatique avec GoCardless !

## Installation {#installation}

1. Téléversez 'ultimate-multisite-gocardless' dans le répertoire '/wp-content/plugins/'
2. Activez le plugin via le menu 'Plugins' dans WordPress
3. Suivez l’assistant étape par étape pour configurer le plugin

## Guide d’utilisation {#usage-guide}

_GoCardless_ est une passerelle de paiement disponible en **Europe et au Royaume-Uni** qui permet les paiements par **prélèvement automatique**.

La **passerelle Ultimate Multisite : GoCardless** ajoute la prise en charge de _GoCardless_ à Ultimate Multisite, vous permettant de facturer vos clients dans cette région par prélèvement automatique.

Ce tutoriel expliquera comment configurer _GoCardless_ afin de commencer à accepter des paiements via celui-ci sur votre réseau Ultimate Multisite.

_**Remarque : GoCardless ne fonctionne qu’avec les devises suivantes :**_ **EUR, GBP, AUD, SEK et DKK.**

## Installation de l’add-on {#installing-the-add-on}

Tout d’abord, vous devrez installer l’add-on GoCardless pour Ultimate Multisite. Vous pouvez le faire en allant dans **Ultimate Multisite > Réglages > Add-ons (dans la barre latérale)**. Puis en choisissant l’add-on GoCardless dans cette liste.

Une fois l’add-on installé et activé, allez dans **Ultimate Multisite > Réglages > Paiements**, activez l’option GoCardless, puis enregistrez.

![Réglages de passerelle de paiement](/img/config/settings-payment-gateways.png)

## Obtenir un jeton d’accès {#getting-an-access-token}

Créez un compte sur _GoCardless_ (ou sur le sandbox _GoCardless_, si vous voulez **le tester d’abord**, ce qui est recommandé). Ensuite, après vous être connecté, accédez à l’élément de menu **Développeurs**.

<!-- Capture d’écran indisponible : Dashboard GoCardless montrant la position du menu Développeurs -->

Sur la page **Développeurs**, cliquez sur le bouton **Créer** dans le coin supérieur droit de l’écran, puis cliquez sur l’option **Jeton d’accès**.

<!-- Capture d’écran indisponible : page Développeurs de GoCardless avec le bouton Créer et l’option Jeton d’accès -->

Cela ouvrira une nouvelle fenêtre modale, vous permettant de créer un nouveau jeton. Donnez-lui un nom reconnaissable et assurez-vous de sélectionner l’option **Accès en lecture-écriture** dans le menu **Portée**.

<!-- Capture d’écran indisponible : fenêtre modale GoCardless pour créer un nouveau jeton d’accès avec une portée lecture-écriture -->

Le jeton d’accès apparaîtra à l’écran. Copiez-le et collez-le dans votre menu **Ultimate Multisite > Réglages > Paiements > GoCardless**.

Assurez-vous de le placer dans le bon champ. S’il s’agit d’un **jeton live**, désactivez le **mode sandbox** avant de coller le jeton.

![Réglages de passerelle de paiement avec GoCardless](/img/config/settings-payment-gateways.png)

<!-- Capture d’écran indisponible : champs de jeton GoCardless dans les réglages de paiement Ultimate Multisite -->

C’est tout pour la première étape. Ensuite, vous devez configurer un écouteur webhook.

## Configuration de l’écouteur webhook {#setting-up-the-webhook-listener}

Lorsqu’il s’agit d’abonnements, Ultimate Multisite doit être informé des changements de statut d’abonnement, des nouveaux paiements effectués et d’autres éléments de ce type.

La manière dont les passerelles de paiement notifient Ultimate Multisite de ces changements consiste à utiliser des **appels webhook**. Pour cette raison, afin que l’intégration fonctionne à 100 %, nous devons indiquer à _GoCardless_ quelle URL appeler avec toutes ces informations.

Retournez à votre panneau **GoCardless > Développeurs** et cliquez à nouveau sur le bouton Créer, mais cette fois, sélectionnez l’option **endpoint webhook**.

<!-- Capture d’écran indisponible : page Développeurs de GoCardless avec l’option endpoint webhook -->Une nouvelle fenêtre s’ouvrira avec les options de l’endpoint webhook. Elle demande l’URL du webhook, donc retournez à vos **réglages Ultimate Multisite > Paiements > GoCardless** et copiez l’URL du webhook qui y est indiquée :

<!-- Capture d’écran indisponible : réglages GoCardless d’Ultimate Multisite montrant l’URL du webhook -->

De plus, vous pouvez saisir un **secret webhook** qui sera utilisé pour signer vos appels webhook. Quelle que soit la valeur que vous saisissez dans ce champ, vous devrez également la saisir dans le **champ Secret webhook d’Ultimate Multisite**.

<!-- Capture d’écran indisponible : formulaire de configuration de l’endpoint webhook GoCardless avec champs URL et secret -->

<!-- Capture d’écran indisponible : champ de secret webhook GoCardless dans Ultimate Multisite -->

Terminez en enregistrant l’endpoint en cliquant sur le bouton **Créer un endpoint webhook**, puis **enregistrez également vos réglages Ultimate Multisite**.

Vous devriez maintenant être prêt à accepter _GoCardless_ !
