# Incompatibilités connues avec d'autres plugins

Après l'installation et l'activation d'Ultimate Multisite, vous pourriez rencontrer des problèmes dus à des incompatibilités non résolues avec d'autres plugins.

Nous travaillons activement à réduire cette liste à chaque nouvelle version, mais si un plugin est répertorié ici, vous devrez peut-être le désactiver pour disposer d'un réseau Ultimate Multisite pleinement fonctionnel.

Voici une liste de ces plugins et ce que vous devez faire dans ces cas :

**Certains plugins WPMU DEV** Certains plugins _non_ maintenus par WPMU DEV sont connus pour provoquer l'arrêt du flux d'inscription d'Ultimate Multisite à la dernière étape (l'utilisateur ne peut pas créer de compte). Voici une liste des plugins qui causent ces problèmes :

* Langue d'inscription : _Ultimate Multisite possède un module complémentaire qui offre la même fonctionnalité, appelé Ultimate Multisite : Language Switcher._
* Définir la description du blog lors de la création du blog : _Ceci est intégré à Ultimate Multisite, il n'est donc pas nécessaire d'utiliser un plugin supplémentaire._
* Nouveaux modèles de blog : _Ceci est intégré à Ultimate Multisite, il n'est donc pas nécessaire d'utiliser un plugin supplémentaire._
* Définir le mot de passe lors de la création d'un blog multisite : _Ceci est intégré à Ultimate Multisite, il n'est donc pas nécessaire d'utiliser un plugin supplémentaire._
* Conditions d'utilisation d'inscription : _Ceci est intégré à Ultimate Multisite, il n'est donc pas nécessaire d'utiliser un plugin supplémentaire._
* Anti-Splog : Bien qu'il soit moins puissant, il est également _intégré à Ultimate Multisite._

**Remarque** : Tous les autres plugins WPMU Dev peuvent être utilisés normalement en parallèle d'Ultimate Multisite. Des exemples incluent _Smush_ , _Forminator_ , _Defender,_ etc.
