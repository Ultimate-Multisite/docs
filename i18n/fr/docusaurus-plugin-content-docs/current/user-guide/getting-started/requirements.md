# Ultimate Multisite Exigences

La première chose que vous pourriez vous demander avant d'acheter Ultimate Multisite est quelles sont les exigences pour faire fonctionner notre plugin. Eh bien, heureusement, il n'y a que quelques exigences de base pour exécuter Ultimate Multisite : un **domain name**, un bon **hosting provider** et une installation WordPress **Multisite**. C'est tout ce dont vous avez besoin !

## Nom de domaine

Tout d'abord, parlons du nom de domaine.

Lorsque vous choisissez un nom de domaine, gardez à l'esprit votre créneau spécifique et ce que vous allez créer avec Ultimate Multisite : voulez-vous créer une agence ? Vendre des sites web de type boutique ? Ou peut-être un hébergement web WordPress ? Tout est possible avec Ultimate Multisite.

Notre plugin fournit des services d'abonnement, la collecte de paiements, des formulaires de paiement, des bons de réduction et des communications client. Une grande partie du travail nécessaire pour installer, configurer et maintenir un WordPress Multisite est facilitée par Ultimate Multisite.

N'importe quel créneau que vous choisissez peut être géré à l'aide de notre plugin tant que vous vendez des sites web en tant que service ! Assurez-vous de choisir un nom de domaine qui correspond à votre plan d'affaires.

## Fournisseur d'hébergement

Maintenant que vous avez un nom de domaine accrocheur qui représente votre créneau et votre entreprise, vous devrez choisir une bonne société d'hébergement web pour héberger votre réseau avec Ultimate Multisite. Il existe de bonnes options sur le marché. Certains fournisseurs d'hébergement s'intègrent bien avec Ultimate Multisite, ce qui peut être un facteur décisif selon ce que vous prévoyez de construire.

Ultimate Multisite devrait fonctionner sur n'importe quel fournisseur d'hébergement. La seule différence est que certains fournisseurs d'hébergement comme _WPMU DEV Hosting_, _WP Engine_, _Closte_, _Cloudways_, _Gridpane_ et _Runcloud_ ont une intégration plus étroite pour la cartographie de domaine. Pour d'autres hébergements, vous devrez peut-être mapper les domaines personnalisés manuellement. Il est également judicieux de choisir un fournisseur d'hébergement qui prend en charge AutoSSL. _Closte_, _Runcloud_ et _cPanel_ le supportent.

En ce qui concerne les spécifications du serveur, il n'y a pas de formule spécifique car cela dépendra des fonctionnalités et des fonctions que vous offrirez ainsi que de la taille de votre réseau. Un hébergement partagé peut être un bon point de départ, mais dès que votre réseau commence à se développer, vous aurez probablement besoin d'un VPS dédié.

## WordPress Multisite

Maintenant, parlons un peu d'une installation WordPress Multisite.

WordPress Multisite vous permet d'avoir un réseau de sites sur une seule installation. Il s'agit d'une fonctionnalité intégrée, mais elle n'est pas activée par défaut. Vous devez activer cette fonctionnalité dans votre installation WordPress et le panneau de contrôle de votre hébergement web.

Avoir WordPress avec Multisite activé **est la base pour Ultimate Multisite**. Étant donné que l'utilisation principale d'Ultimate Multisite est d'aider les utilisateurs à gérer et vendre des sites web en tant que service, cela doit être fait sur WordPress avec Multisite activé.

Nous avons **un article approfondi** expliquant étape par étape ce qu'il faut faire pour activer la fonctionnalité Multisite sur votre installation WordPress.

Une fois que vous avez activé Multisite sur WordPress, il est temps d'installer Ultimate Multisite. Pour installer correctement Ultimate Multisite, vous devez disposer de :

* WordPress : v5.3+ (Recommandé : dernière version stable)

* PHP : 7.4.x (support vers > 8.0 prochainement)

* MySQL : v5+ (Recommandé : 5.6, au cas où vous n'auriez pas la version 8.0 disponible chez votre fournisseur d'hébergement)

Ce sont les seules exigences logicielles pour Ultimate Multisite.

Gardez à l'esprit qu'Ultimate Multisite peut fonctionner avec PHP 8.0 mais nous recommandons de l'exécuter avec PHP 7.4.x.

De plus, nous recommandons de l'installer sur **main domains**, pas sur des sous-domaines ou sous-répertoires. Cela peut fonctionner sur des sous-domaines/sous-répertoires, mais cela peut présenter des erreurs.
