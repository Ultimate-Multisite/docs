# کۆتاییه و قفلکردنی سایت {#membership-expiration-and-site-blocking}

ئەم ڕێنماییە ڕوون دەکاتەوە کە Ultimate Multisite چۆن بەکارهێنانی ماوەی ئەستۆ، کۆتاییه بوونی تریالەکان، و قفلکردنی سایت لە پێشەوە (frontend) دەگرێتەوە. ئەمە گۆڕانکارییەکانی ژیانێکی ئەستۆیی لەسەر ئاستی یەکێک لەو سیستەمە، ئەو ڕێکخراوانەی کە دیاریکراون چۆن قفل دەکرێن، و چی دەبێت چاودێری بکەیت کاتێک سایتەکان دوای کۆتاییه بوونی ئەستۆ بە ئاسانی دەبینرێن.

## ژیانکردنی دۆخی ئەستۆ (Membership Status Lifecycle) {#membership-status-lifecycle}

هەر یەک لەو ماوەی ئەستۆییانەی لە Ultimate Multisite دا، یەکێک لەم دۆخە هەیە:

:::note ئەستۆکانی بێ بەرامبەر (Free memberships) ژیانێکی بەردەوامن
ئەستۆکانی بێ بەرامبەر بە شێوەیەکی ئۆتۆماتیک کۆتایی نییە. Ultimate Multisite ئەوان وەک دەستگەیشتنی بەردەوام دادەنێت، بۆیە لە چاودێرییەکانی کۆتاییه د Neto نین، تەنها ئەگەر بەڕێوەبەری (administrator) ئەو دۆخە بگۆڕێت یان کڕیارەکە بۆ بەرنامەیەکی دیکە بدات.
:::

| دۆخ (Status) | مانای کارکردن |
|---|---|
| **Pending** | چاوەڕوان پێشەم پێدانی یەکەم بارکردن یان ڕاگەیاندنی ئیمەیڵ |
| **Trialing** | ماوەیەکی تریالی چالاکە، هێشتا پارە وەربگرتو نییە |
| **Active** | پارە کراو و ئێستا چالاکە |
| **On Hold** | پارەی نوێکردنەوە چاوەڕوانە (فاکچۆری دروستکراوە، چاوەڕوان دەکرێت) |
| **Expired** | لەو کاتەدا کە ماوەی کۆتاییه و ماوەیەکی بەردەوام نییە بۆ نوێکردنەوە |
| **Cancelled** | بە شێوەیەکی ڕاستەوخۆ لەلایەن کڕیار یان بەڕێوەبەری دیاریکراوە |

### چۆنیەتی گواستنەوەی ئەستۆکان بۆ دۆخی کۆتاییه (Expired) {#how-memberships-transition-to-expired}

Ultimate Multisite **هەر کات ساڵێک** پشکنینی پشتگیریدار (background check) ئەنجام دەدات کە لێکۆڵینێت بۆ ئەو ئەستۆکانەی کە پێویستە بە دۆخی کۆتاییه بن. ئەم پشکنینە لەسەر [Action Scheduler](https://actionscheduler.org/) کار دەکات (نەبێت ڕاستەوخۆ WP-Cron بێت) و وەک اکشنی زمانە `wu_membership_check` دەچێتەبەردەست.

پشکنینی کۆتاییه **ماوەیەکی بەردەوام لە 3 ڕۆژ** بە شێوەیەکی پیش‌دەکەوت (grace period). یەکێک لەو ئەستۆیانە تا ٣ ڕۆژی دوای ئەوەی `date_expiration` دەگوازێت، نیشانەی `expired` دەبێت. ئەمەش کڕیارەکان دەفرۆشداری دەدات بۆ تەواوکردنی پارەی دڵەڕاوێ پێش ئەوەی دۆخی ئەستۆکانیان بگۆڕێت.

:::info
پیرودەی مهلت برای انقضا (grace period) سه روزه از تنظیمات دوره مهلت بلاک کردن بخش فرانت‌اند که در زیر توضیح داده شده، جداست. دوره‌ی مهلت انقضا کنترل می‌کند که وضعیت از فعال/در انتظار به منقضی شده تغییر کند. دوره‌ی مهلت بخش فرانت‌اند کنترل می‌کند که بعد از اینکه وضعیت تغییر کرد، سایت چه زمانی مسدود می‌شود.
:::

#### عضویت‌های خودکار تمدیدشونده در مقابل غیرخودکار تمدیدشونده {#auto-renewing-vs-non-auto-renewing-memberships}

این تمایز برای درک رفتار انقضا خیلی مهم است:

- **عضویت‌های غیرخودکار تمدیدشونده** (`auto_renew = false`): کارهای زمان‌بندی ساعتی (hourly cron job) کل چرخه عمر را مدیریت می‌کند -- پرداخت‌های تجدید را ایجاد می‌کند، عضویت را به حالت در انتظار تغییر می‌دهد و در نهایت اگر پرداخت دریافت نشود، آن را منقضی شده علامت‌گذاری می‌کند.

- **عضویت‌های خودکار تمدیدشونده** (`auto_renew = true`): بررسی انقضا توسط کرون (cron) **این موارد را کاملاً نادیده می‌گیرد**. انتظار می‌رود درگاه پرداخت (Stripe، PayPal و غیره) زمانی که اشتراک با شکست مواجه می‌شود یا لغو می‌گردد، از طریق webhook به Ultimate Multisite اطلاع دهد. اگر webhook دریافت نشود -- به دلیل تنظیم نادرست endpoint، قطع شدن درگاه پرداخت، یا لغو اشتراک خارج از سیستم -- عضویت ممکن است حتی بعد از گذشته بودن تاریخ انقضا تا ابد `active` باقی بماند.

### پایان دوره‌های آزمایشی (Trials) {#how-trials-end}

وقتی دوره آزمایشی یک عضویت آزمایشی به پایان می‌رسد، سیستم موارد زیر را انجام می‌دهد:

1. یک پرداخت تمدید در انتظار با مبلغ اشتراک کامل ایجاد می‌کند
2. وضعیت عضویت را از `trialing` به `on-hold` تغییر می‌دهد
3. ایمیل اطلاع‌رسانی پرداخت تمدید را برای مشتری ارسال می‌کند

این فرآیند با همان برنامه ساعتی بررسی انقضا انجام می‌شود، اما **فقط برای عضویت‌های غیرخودکار تمدیدشونده**. برای دوره‌های آزمایشی خودکار تمدیدشونده، درگاه پرداخت مسئول انتقال از دوره آزمایشی به اشتراک پولی است.

## مسدود کردن دسترسی فرانت‌اند {#block-frontend-access}

بشێوەیەکی پیش‌دۆل، کاتێک یەکەم ئەندازیاری ئەستۆی ئەندازیار (membership) کۆتایی دێت یان لە دوورەوە دەچێت، **فقط داشبۆردی wp-admin ڕێگری دەکرێت**. پێشەنگەی جیهانی سایتەکە بۆ بینەران بە وردی دەمێنێت. بۆ ئەوەی دەستکاریی جیهانیتیش ڕابگرین، دەبێت تنظیمەکەی **Block Frontend Access** چالاک بکەیت.

### گۆڕینی ئەو تنظیمە {#configuring-the-setting}

بڕۆ بۆ **Ultimate Multisite > Settings > Memberships** و **Block Frontend Access**-ی چالاک بکە.

![Block Frontend Access settings showing the toggle enabled, a grace period of 7 days, and a Frontend Block Page selection](/img/config/settings-membership-block-frontend.png)

لێرەدا بینینی تەواوی لاپەڕەی تنظیمەکانی ئەندازیار دەبینیت:

![Membership settings full page](/img/config/settings-membership-full.png)

سێ تنظیمێکی پەیوەندیدار بەم ڕێکارەوە کۆنترۆڵ دەکەن:

| Setting | Description | Default |
|---|---|---|
| **Block Frontend Access** | گۆڕینی سەرەکی (Master toggle). کاتێک چالاک دەکرێت، لە کاتی ئەوەی ئەندازیاری تۆڕەکە زیاتر نییە، پێشەنگەی جیهانی سایتەکانی تۆڕەکە ڕێگری دەکات. | Off |
| **Frontend Block Grace Period** | ژمارەی ڕۆژە بۆ چاوەڕێکردن دوای ئەوەی ئەندازیارەکە غیر چالاک بێت پێش ئەوەی ڕابگرێت. بۆ `0` تنظیم بکە بۆ ڕابگرتنی دەستبەجێ. | 0 |
| **Frontend Block Page** | لاپەڕەیەکی لەسەر سایت سەرەکی کە بینەران بەرەو دەڕواندەوە کاتێک سایتەکە ڕابگراوە. ئەگەر نەکراوە، بینەران پەیامی گشتی "Site not available" دەبینن. | None |

### چی بینەران دەبینن کاتێک سایتەکە ڕابگرێت؟ {#what-visitors-see-when-a-site-is-blocked}

کاتێک دەستکاریی جیهانیت ڕێگری دەکرێت، بینەرانی سایتەکە یەکێک لەم دوۆیەدا ئەنجام دەدەن:

1. **ڕەواندرێن** بەرەو لاپەڕەیەک کە تۆ لە **Frontend Block Page** دا هەڵبژاردووە (ئەگەر دیاریکراوە)، یان
2. **پەیامی پیش‌دۆل ببینن**: "This site is not available at the moment." بە لینکێک بۆ لاپەڕەی Login بۆ ئیدارەی سایتەکە.

ئیدارەکانی سایت 여هانی دەکەن بتوانن Login بکەن -- لاپەڕەی Login هەرگیز ڕابگراوە.

### چی ڕابگرێت و کێ؟ {#what-gets-blocked-and-when}

ڕێکارکردنی ڕابگرتنی پەیوەستە بە **بۆرتەندی ئەندازیارەکە**:

| وضعیت عضویت | آیا فرانت‌اند مسدود است؟ | آیا دوره مهلت اعمال شده است؟ |
|---|---|---|
| فعال (Active) | خیر | -- |
| آزمایشی (Trialing) | **خیر** (به نکته زیر مراجعه کنید) | -- |
| در حالت تعلیق (On Hold) | فعال در نظر گرفته می‌شود -- مسدود نیست | -- |
| منقضی شده (Expired) | **بله**، اگر Block Frontend Access روشن باشد | بله |
| لغو شده (Cancelled) | **بله**، همیشه (مستقل از تنظیمات) | **خیر** -- بلافاصله مسدود می‌شود |
| در انتظار (Pending) | از طریق بررسی عضویت مسدود نمی‌شود | -- |

:::warning اعضای آزمایشی مسدود نمی‌شوند
حتی اگر دوره آزمایشی به پایان رسیده باشد، یک عضویت با وضعیت `trialing` در فرانت‌اند **مسدود نخواهد شد**. ابتدا باید این آزمون به وضعیت دیگری تغییر کند (معمولاً از طریق cron job به `on-hold`، سپس اگر پرداخت نشده باشد به `expired`). اگر مشاهده می‌کنید اعضای آزمایشی هنوز تغییر وضعیت نداده‌اند، لطفاً بخش عیب‌یابی زیر را بررسی کنید.
:::

:::note اعضای لغو شده از دوره مهلت معاف هستند
اعضای لغو شده همیشه پس از گذشت تاریخ انقضا مسدود می‌شوند، صرف نظر از اینکه Block Frontend Access فعال باشد یا نه. دوره مهلت مسدود کردن فرانت‌اند برای اعضای لغو شده اعمال **نمی‌شود**.
:::

## عیب‌یابی: سایت‌هایی که پس از انقضا همچنان قابل دسترسی هستند {#troubleshooting-sites-remaining-accessible-after-expiration}

اگر سایت‌ها پس از منقضی شدن عضویت همچنان عمومی در دسترس هستند، لطفاً این بررسی‌ها را به ترتیب انجام دهید:

### ۱. تأیید کنید تنظیم Block Frontend Access فعال است {#1-verify-the-block-frontend-access-setting-is-enabled}

به مسیر **Ultimate Multisite > Settings > Memberships** بروید و مطمئن شوید که کلید **Block Frontend Access** روشن است. این تنظیم **پیش‌فرض خاموش** است، به این معنی که فقط وقتی عضویت غیرفعال می‌شود، دسترسی به wp-admin محدود می‌شود.

### ۲. بررسی دوره مهلت مسدود کردن فرانت‌اند {#2-check-the-frontend-block-grace-period}

لە هەمان لاپەڕەی تنظیمات، **Frontend Block Grace Period** بەشی دەستکاری بکە. ئەگەر ئەمە بۆ ٧ ڕۆژ دیاریکراوە، بۆ نموونە، پێشتر لەو کاتەی ماوەی ئەندازیاری ئەلێمبەرەکە کۆتایی دێت، فرۆنت ئێڵپەینەکە تا ٧ ڕۆژی دوای ئەو ڕێکەوتەی کەسربوونەکە کۆتایی دەچێت بە بلاک نەکاتەوە - تەنانەت ئەگەر ماوەی ئەندازیاری لە حالەتدا `expired` بێت.

ئەمە بۆ `0` دیاری بکە ئەگەر دەتەوێت بلاککردن بە شێوەیەکی فوري دوای ئەوەی ئەندازیاریەکە ناتوانێت بەکاربهێنرێت دەستپێبکات.

### ٣. دڵنیابە کە ڕاستەقینەی گۆڕانکاری لە حالەتدا هاتووە {#3-confirm-the-membership-status-has-actually-changed}

گەرەوە بۆ **Ultimate Multisite > Memberships** و حالەتی ئەندازیاریی پەیوەندیدار بزانە. ئەگەر بەهۆی ئەو کاتەی کارپێکردنەکە کۆتایی هاتنی، 여وڵا بە `active` دەردەکەوێت، ئەوا گۆڕانکاری لە حالەتدا ڕوویداو نییە. هۆکارە باوەکان:

- **ئەندازیارییەکە خۆی نوێ دەکات (auto-renewing)**: فیلدی `auto_renew` لە لاپەڕەی دەستکاری ئەندازیارییەکە بزانە. ئەگەر ئۆتۆڕینەوە یەکگرتووە، کرۆنێکی کۆتایی هاتنی ئەم ئەندازیارە پاشکەوتێت - ئەو پشت بە گۆڕینی پارە (payment gateway) دەبەستێت بۆ ڕاپۆرتکردنی شکست. داشبۆردی گۆڕینی خۆت (Stripe, PayPal) بزانە بۆ ئەوەی دڵنیابیت کە حالەتەکە لەگەڵ ئەوەی Ultimate Multisite نیشان دەدات یەک دەبێت.

- **کرۆن جابووری (cron job) بەڕێوە نەچوو**: هەنگاوەکانی دوای ئەمە بزانە.

### ٤. دڵنیابە کە Action Scheduler کار دەکات {#4-verify-action-scheduler-is-running}

Ultimate Multisite لەسەر Action Scheduler بۆ کرۆن جابوورەکانی خۆی بەکاردێت. بچۆ بۆ **Tools > Scheduled Actions** لە Network admin و لێی بگەڕێ بۆ:

- **`wu_membership_check`** -- ئەمە دەبێت وەک کارێکی داریکراو هەموو کاتژمێرێک دابکرێتەوە. ئەگەر نەبووە، چاککردنەوەی ئەندازیارییەکان بە شێوەیەکی داریکراو نیشان پێدات.
- **`wu_async_mark_membership_as_expired`** -- ئەمە کارە تاکەکانی بۆ دیاریکردنی ئەندازیارییە تایبەت و ڕاگەیاندنی کۆتایی هاتنیان. ئەگەر هەڵگرتنی کارە شکستخواز لەم شوێنەدا بزانین، دەتوانن پەیامەکانی هەڵە نیشان بدەن کە ڕوون دەکەنەوە بۆچی شکستە.

ئەگەر کارەکان بەستراو یان شکستخواز بینی، ئەوا دەتوانیت کێشەیەکی Action Scheduler هەبێت. هۆکارە باوەکان:

- **`DISABLE_WP_CRON` لە `wp-config.php`-دا بە `true` دیاریکراوە** بدون ئەوەی جۆری سیستەم (system-level cron) جێگرەوەی هەبێت
- **کەمەتر بوونی ترافیکی سایتەکە** -- WP-Cron تەنها کاتێک کار دەکات کە کەسێک سەردانی سایتەکە بکات

ئەو کە بەدوای ئەوەی جۆری کارکردن (cron) بە شێوەیەکی پێکەوتوو و بەردەوام بێت، ئامێرێکی cron سیستەم ڕێک بخەرەوە:

```bash
# هەموو ٥ خولەکێک بە wget چاودێری بکات
*/5 * * * * wget -q -O /dev/null "https://your-network-url.com/wp-cron.php?doing_wp_cron"

# یان لە ڕێگەی WP-CLI
*/5 * * * * cd /path/to/wordpress && wp cron event run --due-now --url=https://your-network-url.com
```

### ٥. دۆزینەوەی کێشەکانی Gateway Webhook (Auto-Renewing Memberships) {#5-check-for-gateway-webhook-issues-auto-renewing-memberships}

ئەگەر ئەندامەکە بە شێوەیەکی خۆکار دەکرێتەوە و چوارچووی پیشکش (gateway subscription) لێی لەناوچووبوو یان شکستپێدات، بەڵام Ultimate Multisite 여전히 وەک `active` نیشان دەدات:

- **Stripe**: بگە به داشبۆرد Stripe > Customers و دۆخی پیشکشەکە بزانە. پاشان چاودن بکە کە endpointی webhook لە خوارەوە لە Developers > Webhooks فعالە. ئەو endpointە دەبێت ئاماژەی بە سایتەکەت بدات و پێداتنی سەرکەوتوو نیشان بدات.
- **PayPal**: دۆخی پیشکشەکە لە هەژمار بازرگانی PayPal یەکەت بزانە و چاودن بکە کە IPN/webhook delivery ڕووبوونە.

ئەگەر gateway ئەو پیشکشە بە لێچوو (cancelled) نیشان بدات بەڵام Ultimate Multisite نیشان نادات، ئەوا ئاماژەی webhook لەدەستداوە. دەتوانیت دۆخی ئەندامەکە بە شێوەیەکی دەستکرد لە **Ultimate Multisite > Memberships > [Edit Membership]** بگۆڕیت.

### ٦. چاودنکردنی ماوەی بەهێزبوون (Expiration Grace Period) (لە ئاستی Cron) {#6-check-the-expiration-grace-period-cron-level}

پشکنینی cron خۆی ماوەیەکی بەهێزبوون هەیە (دۆخی پیش‌تەکی: ٣ ڕۆژ) پێش ئەوەی یەکێک لە پیشکەکان وەک کۆتایی نیشان بکرێت کە منقەوت بووە. ئەمە جیاوازە لە ماوەی بەهێزبوونەکە بۆ بلوککردنی فرۆنت-ئندی (frontend block grace period). کاتێکی کۆی گشتی پێش ئەوەی سایتەکە بڵاو بکرێت دەتوانرێت بێت:

**ماوەی بەهێزبوون بۆ کۆتایی (٣ ڕۆژ)** + **ماوەی بەهێزبوون بۆ بلوککردنی فرۆنت-ئندی (دیاریکراوی تۆ)** = کاتێکی کۆی گشتی

بۆ نموونە، لەگەڵ ڕێکخستنی سەرەتایی و پۆلۆیەکی ٧ ڕۆژ بۆ ماوەی پێشەوە (grace period)، دەتوانێت تا ١٠ ڕۆژ دوای `date_expiration` سایتەکە بە ڕاستی بڕدرێت.

### ٧. بە شێوەیەکی دەستکرد مێنبەرێک کۆتایی بکە {#7-manually-expire-a-membership}
ئەگەر پێویستت بەوێ کە بە خێرایی سایتێک بڕدرێت و چاوەڕێی cikli cron نەکات، دەتوانیت دۆخی مێنبەرەکە بە شێوەیەکی دەستکرد بگۆڕیت:

١. گەهارت بۆ **Ultimate Multisite > Memberships**
٢. لەسەر مێنبەرەکە کە کاریگەریی لەسەردا هەیە، 클릭 بکە (Click)
٣. دۆخی (status) بە **Expired** یان **Cancelled** بگۆڕە
٤. **Save** بکە

بڕدرانی پێشەوە (frontend block) لە بارکردنەوەی پەیجەکەی دوایەکە دەست پێدەکات (بە پێی ماوەی پێشەوەی بڕدرانی پێشەوە بۆ مێنبەرەکانی کۆتایی کراو، یان بە خێرایی بۆ مێنبەرەکانی لێکراو).

## پوختەکان {#summary}

سێبۆری تەواوی لە ڕێکەوتنی کۆتایی تا بڕدرانی سایتەکە:

```text
date_expiration دەگاتە سەر
       |
       v
  [ماوەی پێشەوەی cronی ٣ ڕۆژ] <-- مێنبەرەکە 여전히 وەک چالاک/لە چاوەڕوانیدا دەردەکەوێت
       |
       v
  Cron مێنبەرەکە بە "expired" دەبنە ئاماژە
       |
       v
  [ماوەی پێشەوەی بڕدرانی پێشەوە] <-- لە Settings > Memberships دا دیاریکراوە
       |
       v
  پێشەوەی سایتەکە بڕدرێت
```

بۆ مێنبەرەکانی لێکراو، ڕێگایەکە کورتترە:

```text
  Membership cancelled دەکرێت
       |
       v
  date_expiration دەگاتە سەر (ماوەی پێشەوە نییە)
       |
       v
  پێشەوەی سایتەکە بە خێرایی بڕدرێت
```

## ئاماژەی پەرەپێدەران (Developer Reference) {#developer-reference}

ئەم hooks و filters یان ڕاستەوخۆکان دەتوانیت بە شێوەیەکی تایبەت ئەو کارەکتەرە بگۆڕیت کە کۆتایی پێهێنانی و بڕدرانی سایتەکە:

| Hook/Filter | Description |
|---|---|
| `wu_membership_grace_period_days` | گۆڕانکاریی ئەو ڕۆژانەی پێش کۆتایی هاتنی ئەندازی membership دەکرێت (دفتار: ٣) |
| `wu_schedule_membership_check_interval` | گۆڕانکاری لە نێوان چاودنەکانی membership (دفتار: ١ کاتژمێر) |
| `wu_membership_renewal_days_before_expiring` | گۆڕانکاریی ئەو ڕۆژانەی پێش کۆتایی هاتنی بۆ دروستکردنی پارەدانەوەی نوێبوون (دفتار: ٣) |
| `wu_blocked_site_reactivation_url` | گۆڕانکاری بۆ دابینکردنی URLێکی نوێبوون کە تایبەت بێت کاتێک سایتەکە ڕێگری دەکرێت |
| `wu_membership_is_active` | گۆڕانکاری بۆ دیاریکردنی ئەوەی membership چالاکە (active) یە |

| `wu_membership_trial_check_query_params` | گۆڕانکاریی پارامێترەکانی پرسیار بۆ دۆزینەوەی trial-ی کۆتایی هاتوو |
