# زیپ کردن سایت (Site Export) {#site-export}

صفحهٔ مدیریت **Site Export** به مدیران شبکه اجازه می‌ده که یک سایت، یا کل شبکه را برای مهاجرت، بک‌آپ گرفتن، یا انتقال کارها در قالب یک آرشیو قابل دانلود بسته‌بندی کنند.

## خروجی گرفتن از یک سایت {#exporting-one-site}

به مسیر **Ultimate Multisite > Site Export** بروید و گزینه **Generate new Site Export** (ایجاد خروجی جدید سایت) را انتخاب کنید. زیرسایتی که می‌خواهید صادر شود را انتخاب کنید، سپس تصمیم بگیرید که آیا آرشیو باید شامل آپلودها، پلاگین‌ها و تم‌ها باشد یا خیر.

وقتی خروجی گرفتن تمام شد، فایل ZIP را از لیست **Existing Exports** (خروجی‌های موجود) دانلود کنید. زیپ‌های خروجی اکنون شامل یک `index.php` خود-راه‌انداز و یک فایل `readme.txt` هستند، بنابراین می‌توان آرشیو را روی هاست جدید آپلود کرد و بدون نیاز به نصب افزونه ایمپورتر جداگانه، کار را شروع نمود.

## خروجی گرفتن از کل شبکه {#exporting-the-whole-network}

وقتی به یک آرشیو واحد شامل تمام زیرسایت‌های شبکه نیاز دارید، در صفحه Site Export از گزینه **Network Export** (خروجی شبکه) استفاده کنید. این قابلیت قبل از مهاجرت هاست‌ها، تمرین‌های بازیابی بلایا یا بازسازی محیط‌های آزمایشی که در آن هر زیرسایت باید به صورت یکجا منتقل شود، بسیار مفید است.

از آنجایی که خروجی گرفتن از کل شبکه می‌تواند بسیار بزرگتر از خروجی گرفتن از یک سایت واحد باشد، آن را در دوره‌ای با ترافیک کم اجرا کنید و مطمئن شوید فضای ذخیره‌سازی مقصد برای آپلودها، پلاگین‌ها، تم‌ها و فایل‌های ZIP تولید شده کافی است.

### بسته‌های وارد کردن شبکه (Network Import Bundles) {#network-import-bundles}

با نسخه Ultimate Multisite 2.12.0، Site Exporter می‌تواند **network import bundles** (بسته‌های وارد کردن شبکه) را ایجاد کند — آرشیوهای تخصصی که برای بازیابی یک سایت در کل شبکه به شکلی ساده طراحی شده‌اند. یک بسته وارد کردن شبکه شامل تمام فایل‌ها و متادیتای لازم برای بازگرداندن چندین سایت به یک نصب جدید شبکه است.

#### ایجاد یک بسته وارد کردن شبکه {#generating-a-network-import-bundle}

١. به **Ultimate Multisite > Site Export** برەچە بکە
٢. بۆ **Generate new Network Export** کلیک بکە
٣. وەک جۆری دابەشکردن (export type) **Network Import Bundle** هەڵبژێرە
٤. دیاری بکە ئایا دەتوانیت فایلەکان (uploads)، پڵگینەکان (plugins) و قالبەکان (themes) لەگەڵدا بگرێیت یان نا؟
٥. بۆ **Generate** کلیک بکە
٦. ZIPی دابەشکردنی (bundle) لە لایستی **Existing Exports** دا دابەست بکە

#### گەڕاندەوە بە شێوەیەکی Network Import Bundle {#restoring-from-a-network-import-bundle}

بۆ گەڕاندەوەی سایتەکان لە یەکێک لە دابەشکردنی Network Import bundle:

١. Ultimate Multisite لە هاستگاری نێوانت (target host) دامەزرێنە
٢. کارپۆڵەکە بۆ سیستمی multisite تەواو بکە
٣. بۆ تۆڕی نوێ، به **Ultimate Multisite > Site Export** بڕۆ
٤. بۆ **Import Network Bundle** کلیک بکە
٥. فایلەکەی ZIPی network import bundle بە ئۆردوو بکەبارە (upload)
٦. ڕێنماییەکانی گەڕاندەوەی لەسەر شاشەکە بەدوای بکە

دابەشکردنی network bundles ئەم شتەکان دەپارێزێت:
- ناوەڕۆکی سایت (پۆستەکان، صفحات، custom post types)
- تنظیمات و هەڵبژاردنەکانی سایت
- ڕۆڵە یان ڕێگەدانانی بەکارهێنەران (User roles and permissions)
- پڵگینەکان و قالبەکان (ئەگەر لە bundleـدا بگرون)
- دابەشکردنی میدیا (Media uploads) (ئەگەر لە bundleـدا بگرون)
- خشتەی داتابێس و داتاکانی تایبەت

## دامەزراندنی ZIPی دابەشکردنی خۆکار (self-booting export ZIP) {#installing-a-self-booting-export-zip}

بۆ گەڕاندەوەی ZIPی خۆکار لە هاستگاری نوێ:

١. ناوەڕۆکی ZIPی دابەشکراوەکە بۆ ڕیشەی وێب (web root) یەکێک لە ئامانجەکان بەڵا بکەبارە
٢. `index.php` کە بەدوایەکە، لەناو browser-ەکەدا بکەرەوە
٣. ڕێنماییەکانی دامەزراندنی لەسەر شاشەکە لە پەککەی دابەشکردنی (export package) بەدوای بکە.
٤. پێش ئەوەی فایلە موقتەکان دەدات، `readme.txt` کە لە bundleـدا هەیە بۆ تێبینی تایبەت بە دابەشکردن، بزانە و پشکنین بکە.

بۆ وردەکاری دامەزراندنی تایبەت بە addon و ڕێنماییەکانی import، سەردانی [Site Exporter addon documentation](/addons/site-exporter) بکە.

بۆ ئامرازێکی single-site کە لە Ultimate Multisite 2.9.0 دا زیادکراوە، سەردانی [Export & Import](/user-guide/administration/export-import) بکە.
