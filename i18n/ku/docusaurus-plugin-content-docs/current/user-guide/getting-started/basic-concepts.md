# پێکهات بنەڕەتی (Basic Concepts)

ئەگەر تۆ یەکەم جار بەکارهێنەر بیت لە WordPress Multisite و کەسێک بوو کە تازە دەستپێکرد بە بەکارهێنانی Ultimate Multisite، دەتوانیت سەرەتادا زۆر وشە و دەستەواژەی نوێ بگرێت. فێربوونیان کارێکی گرنگە چونکە پێویستە تێگەیشتن لە پلاتفۆرمەکە و ئەوەی چۆن بە گشتی کار دەکات هەبێت.

لە ئەم بابەتدا، هەوڵ دەبینین هەندێک لە پێکهاتە سەرەکییەکان لە WordPress دیاری بکەین و ڕوون بکەینەوە. هەندێکیان زیاتر بۆ بەکارهێنەران گرنگن، هەندێکی تر بۆ دزنوێنەران (developers)، و هەندێکیان بۆ هەردووکیان.

## WordPress Multisite {#wordpress-multisite}

**Multisite** ی WordPress جۆرێکە لە دامەزراندنی WordPress کە ڕێگە دەدات بە دروستکردن و بەڕێوەبردنی تۆڕی چەندین شوێن (websites) لەناو داشبۆردێکی تاک. دەتوانیت هەموو شتەکان بەڕێوەبەیت، لەوانەش ژمارەی شوێنەکان، تایبەتمەندییەکان، Theme ها و ڕۆڵەکانی بەکارهێنەر. ئەگەری هەیە صد و هەزاران شوێن بەڕێوەبەر بکەیت.

## تۆڕ (Network) {#network}

لە ڕووی WordPress، تۆڕی Multisite ئەو شوێنەیە کە دەتوانیت ژمارەیەک **subsite** لە داشبۆردێکی تاک بەڕێوەبەرت. سەرەڕای ئەوەی دروستکردنی تۆڕی Multisite لە نێوان پڕۆفایڵە هاوسازییەکانی (hosting providers) جیاوازە، ئەنجامی کۆتایی بە شێوەیەکی ئاسایی چەند ڕێنمایی زیاتری لە فایلەکەدا هەیە **wp-config.php** بۆ ئەوەی WordPress بزانێت کە لەم شێوازی تایبەتدا کار دەکات.

جیاوازی زۆر هەیە نێوان تۆڕی Multisite و دامەزراندنی وردی (stand-alone) ی WordPress کە کورت دەبینین.

## داتابەیس (Database) {#database}

داتابەیس کۆمەڵێک داتای ڕێکخراو و ڕێکخراوەیە. لە شێوازی زانستی کۆمپیوتەردا، داتابەیس ئاماژەیە بۆ نەرمووشی (software) بەکارهێنانی داتاکان و ڕێکخستنیان. بیکاتە ئەو ناوە کە دەتوانیت داتاکان لە بەشە جیاوازەکانی پێکهاتووە بەناوی **tables** (سەرچاوەکان).

WordPress Multisite یەک داتابەیس بەکاردەهێنێت و هەر subsite ی تایبەت بەتاوەری خۆی دەبێت، کە ID-ی بلاگەکە لە پیشوڕدا (prefix) دەنووسرێت. بۆیە کاتێک دامەزراندنی تۆڕی Multisite دروست دەکەیت و subsite ی دروست دەکەیت، پێویستە ئەم tablesان هەبێت:

_wp_1_options_ \- تابلوی تنظیمات بۆ زیرساخت فرعی اول

_wp_2_options_ \- تابلوی تنظیمات برای زیرساخت فرعی دوم

## ارائه‌دهنده هاستینگ (Hosting provider) {#hosting-provider}

ارائه‌دهنده هاستینگ شرکتی است که به کسب‌وکارها و افراد اجازه می‌دهد وب‌سایت‌های خود را از طریق وب جهانی در دسترس قرار دهند. خدماتی که ارائه‌دهندگان هاستینگ ارائه می‌دهند متفاوت هستند، اما معمولاً شامل طراحی سایت، فضای ذخیره‌سازی روی هاست و اتصال به اینترنت می‌شود.

## دامنه (Domain) {#domain}

نام دامنه آدرس‌هایی است که مردم برای بازدید از سایت شما استفاده می‌کنند. این نام به مرورگر وب می‌گوید که باید به کجا برای یافتن سایت شما مراجعه کند. درست مثل یک آدرس خیابانی، دامنه روشی است که مردم به سایت شما آنلاین دسترسی پیدا می‌کنند. و همچنین مثل داشتن یک تابلوی راهنمایی جلوی مغازه شماست. اگر مایلید از وب‌سایت ما بازدید کنید، باید آدرس وب ما را در نوار آدرس مرورگر خود تایپ کنید که این آدرس [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) است و [**ultimatemultisite.com**](http://ultimatemultisite.com) نام دامنه است.

## زیردامنه (Subdomain) {#subdomain}

زیردامنه نوعی سلسله مراتب وب تحت دامنه اصلی است، اما به جای استفاده از پوشه‌ها برای سازماندهی محتوا در یک وب‌سایت، نوعی وب‌سایت مستقل ایجاد می‌کند. این ساختار به صورت [**https://site1.domain.com/**](https://site1.domain.com/) نمایش داده می‌شود که در آن _site1_ نام زیردامنه و [_domain.com_](http://domain.com) دامنه اصلی است.

## زیردایرکتوری (Subdirectory) {#subdirectory}

زیردایرکتوری نوع دیگری از سلسله مراتب وب تحت یک دامنه ریشه است که از پوشه‌ها برای سازماندهی محتوا در یک وب‌سایت استفاده می‌کند. زیردایرکتوری مثل یک پوشه فرعی است و می‌توان از نام‌های آن‌ها به جای هم استفاده کرد. این ساختار به صورت [**https://domain.com/site1**](https://domain.com/site1) نمایش داده می‌شود که در آن _site1_ نام زیردایرکتوری و [_domain.com_](http://domain.com) دامنه اصلی است.

## زیرسایت (Subsite) {#subsite}

Subsite یەک سایت چیاکە (child site) یە کە لە تۆڕی Multisite دروست دەکەیت. بەپێی ئەوەی جۆری دامەزراندنی WordPress Multisite-ەکەت چۆن ڕێکخستووە، دەتوانێت **subdomain** بێت یان **subdirectory**.

## Super Admin (سەر ئیدارەکار) {#super-admin}

Super Admin ی WordPress ڕۆڵێکی بەکارهێنەرە کە توانای تەواویی پێداویستی هەیە بۆ بەڕێوەبردنی هەموو سایتەکانی subsite لە تۆڕی Multisite دا. بۆ بەکارهێنەرانی Multisite، ئەمە **سەر ئاستی دەستگەیشتن**ە کە دەتوانیت بە دامەزراندنی WordPress-ەکەت پێ بدات.

## Plugin (پڵگین) {#plugin}

بۆیە، پڵگین بە گشتی کۆمەڵێک کۆدە کە تایبەتمەندی زیاتری بۆ سایتەکەت زیاد دەکات. ئەمە دەتوانێت تا ببێت بە وردی گۆڕینی لوگۆی Login یان زیادکردنی تایبەتمەندی بازرگانی (e-commerce) بێت. _Woocommerce and Contact Form_ نموونەی پڵگینن.

لە WordPress Multisite، پڵگینەکان تەنها دەتوانرێت لە داشبۆردی بەڕێوەبردنی تۆڕەکە بە کارکردنەوەی Super Admin دامەزرێن بکرێن. ئیدارەکانی subsite تەنها دەتوانن پڵگینەکان لە ناو subsite خۆیان چالاک و غیر چالاک بکەن.

## Theme (تیم) {#themes}

Theme ی WordPress کۆمەڵێک فایلە (_دیزاین، style sheets، و کۆد_) نۆ بە شێوەیەکی گشتی ڕووی سەرنجی سایتەکە دیاری دەکات. ئەو هەموو تایبەتمەندی دیزاینی پێشەوە (front-end) دەدات، وەک دیزاینی فۆنت، بچووککردنەوەی لایناکان، و ڕەنگەکان و هتد.

وەک پڵگینەکان، Theme لە WordPress Multisite تەنها دەتوانرێت لەلایەن Super Admin دامەزرێن بکرێت و دەتوانرێت لە ئاستی subsite لەلایەن ئیدارەکانی subsite چالاک بکرێنەوە یان غیر چالاک بکرێن.

## Site Template (نمۆری سایت) {#site-template}

**Site Template** سایتێکی بنەڕەتییە کە دەتوانیت بەکاری بهێنیت وەک بنەمایدا بۆ دروستکردنی سایتە نوێکان لە تۆڕەکەتدا.

ئەمەش واتای ئەوەیە کە دەتوانیت یەک سایتێکی بنەڕەتی دروست بکەیت، پڵگین جیاواز چالاک بکەیت، Theme ی چالاک دابنێیت، و بە هەر شێوەی خۆت تایبەتمەندی بکەیت. دواتر، کاتێک کڕیارەکەت هەژمارێکی نوێ دروست دەکات، لەبری ئەوەی سایتێکی WordPressی باو و بدون ناوەڕۆکی گرنگی وەربگرێت، ئەو کۆپی سایت بنەڕەتی تۆ دەپێێن کە هەموو تایبەتمەندییەکان و ناوەڕۆکەکان پێشتر جێگیرکراون.

## Domain Mapping (مەپکردنی دامێنی) {#domain-mapping}

**Domain mapping** و WordPress یەکێکە لەو ڕێگەیە کە بەکارهێنەران دەڕواندێت بۆ سەر میزبینی دروستکراوەکان، لە ڕێگەی پیتەکانی ئەو شوێنە. لە WordPress Multisite دا، سایتەکانی بچووک (subsites) بە بەکارهێنانی یەک دایرێکتۆری زیرەکی (subdirectory) یان دووبارە (subdomain) دروست دەکرێن. Domain mapping ئەوەی ڕێگە دەدات بە بەکارهێنەرانی subsite بۆ بەکارهێنانی domainێکی سەرەوە وەک [**joesbikeshop.com**](http://joesbikeshop.com) بۆ ئەوەی شوێنی سایتەکەی زیاتر پڕۆفیشناڵتر بێت.

## SSL {#ssl}

SSL ناوی **Secure Sockets Layer**ە. ئەمە بەڵگەی دیجیتاڵی (digital certificate)یە کە ناسنامەی یەکێتی وێبسایتی پارێزراوە و پەیوەندییەکی پاشکۆڕکراو (encrypted connection) دەدات. لە ئێستادا، ئەوە بە تەکنەلۆژیای ستاندارد بۆ پاراستنی پەیوەندیی ئینتەرنێت و پاراستنی هەر داتای هەستی کە لە نێوان دوو سیستەمدا گواستار دەبێت، ڕێدەگرێت لە ئەفسەران لێکردن و دەستکاری هەر زانیارییەکی گواستراو، لەوانەش وردەکاریی تایبەت. Browserە مۆدێرنەکان پێویستیان بە SSL هەیە کە ئەمەش گرنگە کاتێک سایتێک دروست دەکەیت و دەڕێژیتەوە.

## Media {#media}

Media بریتییە لە وێنە، ئاودا، ڤیدیۆ و فایلەکانی دیکە کە یەکێک لە بنەمای دروستکردنی سایتێک دەبن.

سایتە تۆڕییەکان (Network sites) داتابەیسێکی یەک بەشاری بەشاریان هەیە لە WordPress Multisite دا، ئەوان ڕێگاکانی جیاواز لە فایڵسیستمەکەدا بۆ فایلەکانی Media پاراستن دەکەن.

شوێنی ستاندارد ی WordPress (wp-content/uploads) دەمێنێتەوە؛ بەڵام ڕێگای ئەو شوێنە دەگۆڕدرێت بۆ ئەوەی ئاماژەی بدات بە ID تایبەتی سایتەکە. لە ئەنجامدا، فایلەکانی Media بۆ Network site وەک wp-contents/uploads/site/[id] دەبن.

## Permalinks {#permalinks}

Permalinks پیتەماوەکانی جێگیر (permanent URLs) یان شوێنەکانن لە ناو سایتێکدا بۆ هەر بابەت یان لاپەڕەیەکی تایبەت بە تۆ. Permalinks بە **pretty links**یش ناسراون. بە شێوەیەکی پیش‌دۆز، URLەکانی WordPress بە شێوازی query string دەبن کە وشەیەک وەک ئەمە دەبینرێت:

[**http://www.example.com/registration**](http://www.example.com/registration)

## Ultimate Multisite {#ultimate-multisite}

Ultimate Multisite ووردپرس پلاگینی یە، کە بۆ نصبکردنی WordPress Multisite دروست کراوە، و ئەو نصبەی ووردپرسە دەگۆڕێت بۆ تۆڕێکی پریمیم لە شوێنەکان – وەک [WordPress.com](https://WordPress.com) – کە ڕێگە بە کڕیارەکان دەدات شوێن دروست بکەن بە نرخەکانی مانگانە، چوارچێوەیەکی، یان ساڵان (دەتوانیت پلانی بێ بەرامبەرشیشیش دروست بکەیت).

## Checkout Form (فۆرمی دابینکردن) {#checkout-form}

Checkout Form فۆرمی داواکاری تەنها یەک بەشی یان چەند بەشی پێکدایە کە پەیوەستە بە دروستکردنی subsite، ئەندازیاری ئەلێمنتی (membership)، و خێرخوردنی کاربری لە ڕێگەی تۆمارکردنەوەی Ultimate Multisite. ئەم فۆرمە لەخۆدەگرێت چەندین فیلد و فۆرمی پایتەخۆیی هەیە کە بەکارهێنەر دەبێت لە ماوەی تۆمارکردنەوەدا پێشکەش بکات.

## Webhook (وێب‌هۆک) {#webhook}

Webhook (کە هەندێک جار وەک web callback یان HTTP push APIیش ناودەبێت) ڕێگەیە بۆ ئەوەی بەرنامەیەکی دیاریکراو زانیارییەکی ڕاستەوخۆ بە بەرنامەکانی دیکە بدات. Webhook داتا لە کاتی ڕووداوەکەدا دەدات بە بەرنامەکان، واتە تۆ دەبێت بە خێرایی داتایەکە وەربگریت.

**Ultimate Multisite webhooks** دەرفەتێکی نامۆیان دروست دەکەنەوە، کە ڕێگە بە ئیدارەکانی تۆڕ دەدات هەموو جۆرە گۆڕانکارییە دیاریکراوەکان بکەن، بە تایبەتی ئەگەر لەگەڵ خزمەتگوزارییەک وەک _Zapier_ و _IFTTT_ بەکاربهێنرێت.

## Events (ڕووداوەکان) {#events}

Event ئەو کارەیە کە بەهۆی کردارێکی بەکارهێنەر یان سەرچاوەیەکی دیکەی کاری پێ ئەنجام دەدرێت، وەک کلیککردن لەسەر ماوس. Ultimate Multisite تۆمار دەکات هەموو ڕووداوەکان و تۆماری ئەو کارانەی کە لە چوارچێوەی تەواوی تۆڕەکەت ڕوودەدەن. ئەم سیستەمە چالاکیی جیاوازەکانی ناو multisite یەکەت، وەک گۆڕینی پلاندانان (plan changes)، بەدوای تۆ دەبێت.
