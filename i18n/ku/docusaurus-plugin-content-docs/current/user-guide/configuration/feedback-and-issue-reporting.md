# بازخورد مشتری و گزارش مشکلات {#customer-feedback--issue-reporting}

Gratis AI Agent v1.5.0 سیستمی داخلی برای بازخورد و گزارش مشکل اضافه کرده که به کاربران نهایی اجازه می‌دهد پاسخ‌های کمکی را علامت‌گذاری کنند و مشکلات را مستقیماً از رابط چت گزارش دهند. این سیستم شامل مدیریت رضایت، یک دستور گزارش خودکار و طبقه‌بندی اولیه با کمک هوش مصنوعی در بک‌اند است.

## دکمه انگشت شست پایین (Thumbs-Down Button) {#thumbs-down-button}

هر پیامی که دستیار هوش مصنوعی ارسال می‌کند، دارای یک دکمه **انگشت شست پایین** (👎) است. وقتی کاربر روی آن کلیک می‌کند، می‌تواند پاسخی را به عنوان کمکی یا اشتباه علامت‌گذاری کند.

- این دکمه هنگام نگه داشتن ماوس کنار هر پیام دستیار ظاهر می‌شود.
- کلیک بر روی آن پنجره **Modal رضایت بازخورد** را باز می‌کند.
- بازخورد با رشته گفتگو و پیام خاص مرتبط است.

## Modal رضایت بازخورد (Feedback Consent Modal) {#feedback-consent-modal}

وقتی کاربر روی دکمه انگشت شست پایین کلیک می‌کند، قبل از ارسال هر داده‌ای یک پنجره رضایت ظاهر می‌شود. این پنجره:

- توضیح می‌دهد که چه اطلاعاتی جمع‌آوری خواهد شد (بخش گفتگو، زمینه مرورگر).
- از کاربر می‌پرسد که آیا با اشتراک‌گذاری این داده‌ها موافق است یا خیر.
- یک فیلد متن آزاد برای کاربر فراهم می‌کند تا مشکل چیست را توصیف کند.
- گزینه‌ای به نام **لغو** (Cancel) ارائه می‌دهد تا بدون ارسال، آن را ببندید.

تا زمانی که کاربر صراحتاً تأیید نکند، هیچ بازخوردی ثبت نمی‌شود.

## بنر بازخورد خودکار (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

در پایان یک گفتگو، دستیار ممکن است یک **بنر بازخورد خودکار** نمایش دهد — پیامی غیر مزاحم که می‌پرسد آیا جلسه مفید بوده است یا خیر.

این بنر بر اساس طول گفتگو و معیارهای نتیجه به صورت خودکار ظاهر می‌شود. این بنر به همان جریان بازخورد دکمه انگشت شست پایین لینک شده است. کاربران می‌توانند بدون ارائه بازخورد، این بنر را ببندند.

## دستور اسلش /report-issue (/report-issue Slash Command) {#report-issue-slash-command}

کاربران می‌توانند مستقیماً با تایپ `/report-issue` در ورودی چت، جریان بازخورد را فعال کنند. این دستور:

- دەستکرد بە شۆڕشی ڕەزامەندی (Feedback Consent Modal) دەگرێتەوە.
- فیلدی وەسفەکە بە وردەکارییەکانی ئەو گفتوگۆیەی کە ئێستا دەکەیت، پێشوەخت پڕ دەکاتەوە.
- دەبێت بڕستنی وردەکاری زیاتر بۆ ئەوەی بڵێیت لەسەردا یان پێش دانانی داواکارییەکە ڕێ>\<be بدات.

لێرەدا، فرمانـ `/report-issue` لە هەموو جۆرەکانی چات (inline، floating widget، full-screen) بەکار دێت.

## AI-Assisted Triage (پێکهێنانی پۆلسی بە یارمەتی AI) {#ai-assisted-triage}

ڕاپۆرتە پێشکراوەکانی کارگێڕان دەچن بۆ لایەیەکی پۆلسیی AI کە:

- ڕاپۆرتەکە پۆلسی دەکاتەوە (خاڵێکی هەڵە، وەڵامێکی نادەستکار، ناوەڕۆکی نادەستکار، و هتد).
- زانیارییە پەیوەندیدارەکانی گفتوگۆکە دەبەستێت.
- پوختەی پۆلسییەکە بۆ **Feedback Endpoint** کە ڕێکخراوە (ببینەرەوە لە [Settings > Advanced](#settings-advanced)) ڕادەگرێت.

ئەم کارە کاتی چاودێری دەستکرد کەم دەکاتەوە بەو شێوەیە کە گرنگترین قەبارەکان یەکەمدا دەداتە سەرنجی.

## Settings > Advanced {#settings-advanced} (ڕێکخراوەکان > پێشکەوتوو) {#settings--advanced}

بۆ چالاککردنی بک‌اندی فیدبک، ئەو فیلەکانە لە خوارەوە بەرزان بکەرەوە لە ژێر **Gratis AI Agent → Settings → Advanced**:

| Field | Description (وەرگێڕان) |
|---|---|
| **Feedback Endpoint URL** | ئەو URLەیە کە داواکارییەکانی POST لەگەڵ بارデータの فیدبکەکە (JSON) دەگرێت. |
| **Feedback API Key** | توکنێکی Bearer کە لە سەردار `Authorization` header دەنێردرێت بۆ ڕاستکردنەوەی پێشکراوەکانی فیدبک. |

ئەو دوو فیلدیش پەت بکەرەوە بۆ ئەوەی کۆکردنەوەی فیدبکەکە بەکار نهارێت. بٹنی فیدبکەکانی تاکەکەسی بەکار دێن، بەڵام داواکارییەکان ڕادەگرن.

:::tip
پوختەی فیدبک دەبێت تێدا بێت JSON body یەک هەبێت لەگەڵ فیلدەکانی `message_id`، `conversation_id`، `feedback_text` و `triage_category`. بۆ پێکهاتنی شێوازی داتاکە کە دەبێت بێت، بەدوای dokumantasi پڕۆڤایەرەکەت بگەڕێ.
:::
