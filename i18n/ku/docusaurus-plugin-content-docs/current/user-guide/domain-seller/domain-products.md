# پێداویستییە دامێنی و نرخەکان (Domain Products and Pricing) {#domain-products-and-pricing}

پێدافەوداڵەکانی دامێنی ئەو شێوازانن کە بەڕێوەبردنی نرخ، TLDs (ناونیشانی ناوچەیی)، تایبەتمەندی پنهانکردنی WHOIS، و هەڵبژاردنی پڕۆڤایدر دەکرێت. هەر پێداویستییەکی دامێنی پڕۆدۆکٹێکی ستاندارد Multisite Ultimate بێت کە تابلۆی زیاتری **Domain Settings** (سەرکردایەتیی دامێنی) لەخۆدەگرێت.

## دروستکردنی پێداویستییەکی دامێنی {#creating-a-domain-product}

1. بگواشبە **Network Admin › Ultimate Multisite › Products**.
2. **Add New** (زیادکردنی نوێ) بگرە.
3. جۆری پڕۆدۆکٹەکە بۆ **Domain** (دامێنی) دیاری بکە.
4. تابلۆی **Domain Settings** (سەرکردایەتیی دامێنی) ڕێک بخەرەوە (لێرەدا دەبینیت).
5. سیوکردن (Save) بکە.

پێداویستییەکی دامێنی لە لیستەکەی پڕۆدۆکٹ بە نشانێکی پۆپۆر purple **Domain** دەردەکەوێت و دەتوانیت بە تابلۆی **Domain Products** فیلتر بکەیت.

## تابلۆی Domain settings (سەرکردایەتیی دامێنی) {#domain-settings-tab}

### Provider (پڕۆڤایدر) {#provider}

بۆ دیاریکردنی ئەو پڕۆڤایدرانەی کە تۆماری ئەم پڕۆدۆکٹە دەگرن، هەڵبژێرە. بە شێوەیەکی پیش‌تەوا، بۆ **Default provider** (پڕۆڤایدرێکی سەرەکی) لە ڕێکخستنی Domain Seller دێت.

### Supported TLDs (TLDs پشتیوانی) {#supported-tlds}

ئەم فیلدەیانە خالی دەبێت تا پڕۆدۆکٹێکی **catch-all product** (پڕۆدۆکتێکی گشتی کە بۆ هەموو TLDیەکانی دیکە کار دەکات) دروست بکەیت کە بۆ هەموو TLDیەکانی تر کە لەلایەن پڕۆدۆکٹێکی دیکە نەبێت، بکاربهێنرێت.

ئەگەر لیستێکی دووخۆری (comma-separated list) ی TLDs بنێریت (وەک `.com, .net, .org`) بۆ دروستکردنی **TLD-specific product** (پڕۆدۆکٹێکی تایبەت بە TLD)، کە تەنها بۆ ئەو کۆتایانە کار دەکات.

**چۆنیەتی یەکگرتووکردنی پڕۆدۆکٹ:** کاتێک کڕیارێک دامێنییەکی دەگەڕێت، ئەپڵۆگینەکە (addon) پڕۆدۆکٹێکی وردترین یەک لەو پڕۆدۆکتان هەڵدەبژێرێت. پڕۆدۆکٹێک کە `.com` لە لیستەکەی TLDدا هەیە پێشترە لە پڕۆدۆکٹێکی catch-all. ئەگەر هیچ پڕۆدۆکٹێکی تایبەت بە TLD نەبێت، پڕۆدۆکٹێکی catch-all بەکاردێت. ئەگەر هیچ پڕۆدۆکتێک نەبێت، گەڕان بۆ دامێنی نیشان ناکرێت.

### Markup type (جۆری مارکەوە) {#markup-type}

سێ شێواز هەیە کە دیاری دەکەن چۆن نرخێکی خریداریکردنی ئێوە لەو زیانە پاشخریدەدا (wholesale cost) هەژمار دەکرێت:

| Mode (شێواز) | How it works (چۆنیەتی کارکردن) |
|---|---|
| **Percentage** (پەربڵاو) | پێکەنینیشێکی پۆست بەسەر زیانە پاشخریدەکە زیاد دەکات. مارکەوەی ٢٠٪ لەسەر دامێنی یەک ١٠ دووخۆری، ١٢ دەبێت. |
| **Fixed markup** (مارکەوەی دیاریکراو) | نیشاندنی ببێت بە شتێکی پارەی دیاریکراو. مارکەوەی ٥ دووخۆری لەسەر دامێنی یەک ١٠، ١٥ دەبێت. |
| **Fixed price** (نرخی دیاریکراو) | زیانە پاشخریدەکە بە تەواوی نادەگرێت. هەمیشە ئەو پارەی کە تۆ دەنووسیت، وەردەگرێت. |

### Introductory pricing (نرخی پێشکەشکردن) {#introductory-pricing}

قیمتی ویژه برای سال اول ارائه دهید. یک **قیمت مقدماتی** (قیمت سال اول) جداگانه در کنار **قیمت تجدید** معمولی (سال دوم به بعد) تنظیم کنید. مشتری هر دو قیمت را در طول فرآیند خرید می‌بیند تا بداند هنگام تمدید چه انتظاری داشته باشد.

### حریم خصوصی WHOIS {#whois-privacy}

کنترل می‌کند که آیا برای دامنه‌های ثبت شده از طریق این محصول، حفاظت از حریم خصوصی WHOIS ارائه می‌شود یا نه.

| تنظیم | رفتار |
|---|---|
| **غیرفعال** | حریم خصوصی WHOIS هرگز ارائه یا فعال نمی‌شود. |
| **همیشه شامل** | حریم خصوصی WHOIS به صورت خودکار هنگام ثبت نام بدون هزینه فعال می‌شود. |
| **انتخاب مشتری** | یک چک‌باکس در طول فرآیند خرید ظاهر می‌شود. **قیمت حریم خصوصی** را برای دریافت هزینه سالانه تنظیم کنید، یا آن را روی $0 بگذارید تا رایگان ارائه شود. |

برای Namecheap، حریم خصوصی WHOIS از WhoisGuard استفاده می‌کند (همیشه رایگان است). برای OpenSRS، از سرویس حریم خصوصی OpenSRS استفاده می‌کند (ممکن است در فروش عمده هزینه داشته باشد).

## وارد کردن و همگام‌سازی TLDها {#tld-import-and-sync}

محصولات دامنه قیمت‌های عمده لحظه‌ای را که از ارائه‌دهنده متصل دریافت می‌شوند، نمایش می‌دهند. برای اینکه این کار انجام شود، باید TLDها (نوع دامنه) وارد شوند.

- **همگام‌سازی دستی:** Settings › Domain Seller › Sync TLDs (برای هر ارائه‌دهنده)
- **همگام‌سازی خودکار:** روزانه از طریق یک کارهای زمان‌بندی شده (scheduled cron job) در تمام ارائه‌دهندگان پیکربندی شده اجرا می‌شود.

بعد از همگام‌سازی، به تب Domain Settings هر محصول دامنه بروید و از انتخابگر TLD برای دیدن TLDهای موجود با قیمت‌های عمده فعلی استفاده کنید.

## تمدید خودکار (Auto-renewal) {#auto-renewal}

تمدید دامنه‌ها به وضعیت عضویت مشتری وابسته است:

- وقتی یک عضویت تمدید می‌شود و یک دامنه به آن متصل می‌گردد، تمدید دامنه به صورت خودکار در صف قرار می‌گیرد.
- تلاش‌های تمدید از دروازه پرداخت فعال مشتری استفاده می‌کنند.
- تمدیدهای ناموفق به صورت خودکار با عقب‌نشینی نمایی (exponential backoff) دوباره امتحان می‌شوند.
- برای تمدیدهای موفق، ناموفق‌ها و تاریخ انقضاها، اعلان‌های ایمیلی ارسال می‌شود.

شناسه‌های قالب ایمیل برای رویدادهای چرخه حیات دامنه:

| Event | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## ئیدارەکار: تۆمارکردنی دۆمەین بە شێوەی دەستکرد (Manual domain registration) {#admin-manual-domain-registration}

بۆ ئەوەی دۆمەیەک لە بەرامبەر کڕیارێک تۆمار بکەیت، بەبێ ئەوەی ئەو کڕیارە ڕێگای پاشەکەوت (checkout) بگرێت:

1. بگوا به **Network Admin › Ultimate Multisite › Register Domain**
2. کڕیارەکەت هەڵبژێرە و ناوی دۆمەکە بنووسە
3. وردەکاری پەیوەندیی تۆمارکار (ناوی، پیت، ژمارەی تەلەفۆن) پڕ بکەرەوە
4. بۆ **Register** بگەڕێ

رکوردکردنی دۆمەکە دروست دەبێت و بە شتکۆی کڕیارەکە پەیوەست دەبێت.
