# Admin Page Creator {#admin-page-creator}

## Naverok {#content}
Admin page creator dihêle hûn bi hêsanî rûpelên rêveberiyê û bin-rûpelên nû di nav Dashboard-a rêveberiya WordPress de zêde bikin. Ew dikare ji bo çêkirina rûpelek bixêrhatinê an rûpelek fêrkariyê were bikaranîn ku ji bo bikarhênerên nû wekî rûpela destpêkê kar bike. Hûn dikarin çêkerên rûpelan wekî Beaver Builder, Elementor, Brizy û Oxygen bikar bînin da ku rûpelan bê zehmetî sêwirînin.

Ew her wiha piştgirîya HTML, CSS û JavaScript-a taybet, û kontrola gihîştinê ya li gorî plan û rola bikarhênerê jî pêşkêş dike!

## Taybetmendî {#features}
* Piştgirî ji HTML û CSS-a taybet re
* Piştgirîya JavaScript
* Moda Naveroka PHP
* Destûr li gorî rola bikarhênerê
* Piştgirî ji çêkerên rûpelan re

## Sazkirin {#installation}

1. **ultimate-multisite-admin-page-creator** bar bikin nav pelrêça **/wp-content/plugins/**
2. Pêvekê bi rêya menûya **Plugins** di WordPress Network Admin-a xwe de çalak bikin

## Rêbera Bikaranînê {#usage-guide}

Her çend WordPress zêdekirina naverokê li beşa pêşiya malperan pir hêsan dike, çêkirina rûpel û bin-rûpelan li aliyê rêveberiyê hin jêhatîbûnên bingehîn ên kodnivîsînê dixwaze. Ultimate Multisite: Admin Page Creator armanc dike ku heman hêsaniya bikaranîna beşa pêşiyê bîne beşa paşê ya malperên tora we.

## Çima Admin Page Creator bikar bînin? {#why-use-admin-page-creator}

Dibe ku hûn bixwazin li panela rêveberiya kiryarên xwe rûpelek taybet bi fêrkariyan pêşkêş bikin, an jî rûpelek taybet bi pirsyarên gelemperî û widget-ek têkiliyê zêde bikin. Bi Admin Page Creator re derfet bêdawî ne, û em ê di fêrkariyê îro de vê yekê vekolin.

## Çêkirina yekem Rûpela Rêveberiya Taybet a xwe {#creating-your-first-custom-admin-page}

Piştî sazkirin û çalakkirina Ultimate Multisite: Admin Page Creator, biçin menûya rêveberiya tora xwe û biçin Admin Pages.

![Lîsteya Rûpelên Rêveberiya Taybet li ser Rêveberiya Torê](/img/admin/admin-pages-list.png)

_Lîsteya Rûpelên Rêveberiya Taybet li ser Rêveberiya Torê_

Hûn ê lîsteyek ji hemû rûpelên rêveberiyê yên ku heta niha we çêkirine bibînin (ji ber ku ev serdana weya yekem e, lîste vala be). Berdewam bikin û li **Rûpela Rêveberiya Nû Zêde Bike** bitikînin. Hûn ê werin veguhestin ser edîtora Rûpela Rêveberiyê.

## Hêmanên Rûpelek Rêveberiya Taybet {#the-elements-of-a-custom-admin-page}

Li ser edîtora Rûpela Rêveberiyê, hûn ê gelek vebijarkên cuda hebin da ku rûpela xwe taybet bikin.

### Sernavê Rûpelê {#page-title}

Ev sernav e ku li serê rûpelê dê were nîşandan, ne etîketa menûya rêveberiyê.

### Hilbijêrê Cûreyê Naverokê {#content-type-selector}

Ultimate Multisite: Admin Page Creator dema ku mijar naveroka rûpelê be du vebijarkan dide we: hûn dikarin an edîtora standard a WordPress WYSIWYG bikar bînin an jî veguherin edîtorek HTML.

Ger hûn bi HTML nas ne, bi edîtora standard a WordPress re bimînin. Hûn ê karibin naveroka rûpelê biguherînin, wekî ku hûn bi Post an Page-ek normal re dikin.

## Vebijarkên Menûyê {#menu-options}

Li aliyê kêlekê, di meta-box-a General Options de, hûn ê karibin cûreyê menûyê, etîket, û gelek tiştên din saz bikin.

### Cûreyên Menûyê {#menu-types}

Rûpela rêveberiya taybet a nû ya we dikare du cûreyên menûyê hebe: menûya asta jor û bin-menû.

Rûpelek menûya asta jor, wekî ku navê wê jî dibêje, rûpela nû çêkirî ya we dê li kêleka rûpelên standard ên WordPress zêde bike nav menûya rêveberiyê.

Ger hûn vebijarka asta jor hilbijêrin, hûn ê karibin rêza menûyê û her wiha îkonek menûyê jî hilbijêrin.

Ger hûn bixwazin vê rûpelê bikin rûpelek bin-menû, hûn ê karibin rûpelek dêûbav hilbijêrin da ku vê rûpelê pê ve girêdin. Hûn dikarin rûpela xwe ya taybet zêde bikin ser rûpelên standard ên WordPress an ser rûpelên rêveberiya taybet ên asta jor ên xwe.

## Vebijarkên Pêşketî {#advanced-options}

Divê hûn bikaribin her cûre naverokê zêde bikin ser rûpelên rêveberiya taybet ên xwe. Ev qaîde û pelên CSS, JavaScript, û pirtûkxaneyên derveyî jî dihewîne. Hûn dikarin vê yekê bi Tab-a Advanced Options bikin.

### CSS-a Taybet {#custom-css}

Li ser tab-a CSS, hûn dikarin koda CSS-a xwe ya taybet zêde bikin, her wiha pelên CSS ji malperên dûr bînin (wekî CDN, Google Fonts, hwd).

<!-- Screenshot unavailable: Admin Page Creator custom CSS editor with external stylesheet import -->

_CSS-a taybet jî tê piştgirîkirin. Hûn dikarin pelên şêwazê yên derveyî jî têxin navê!_

### JavaScript-a Taybet {#custom-javascript}

Heman tişt ji bo JavaScript jî derbasdar e. Hûn dikarin pirtûkxaneyên derveyî zêde bikin da ku di koda JavaScript-a xwe ya taybet de bikar bînin.

<!-- Screenshot unavailable: Admin Page Creator custom JavaScript editor with external library import -->

_Koda JavaScript-a xwe zêde bikin!_

## Destûr {#permissions}

Her wiha di Advanced Options de, hûn ê karibin hilbijêrin kîjan bin-malper dê gihîştinê bi vê rûpelê re hebe. Ev kêrhatî ye ger hûn plan dikin ku naveroka taybet a cuda ji asta planên cuda re pêşkêş bikin, wekî mînak.

<!-- Screenshot unavailable: Admin Page Creator permissions settings for plan and role-based access control -->

_Hûn kontrola tevahî hene li ser ku ev rûpel li ku derê dê xuya bibe_

**Girîng** : Divê her du merc jî bêne bicihanîn da ku rûpel were nîşandan. Ev tê wê wateyê ku heke bikarhêner xerîdarê planek diyarkirî be, lê yek ji rolên hilbijartî tune be (an berevajî), rûpel li ser panela wî/ê xuya nabe.

## Çalak {#active}

Heke tu dixwazî rûpeleke rêveberiya taybet bêyî jêbirinê an destdanê bi destûran re neçalak bikî, tenê vebijêrka Çalak rake.

Neçalakkirina rûpeleke asta-serî dê bixweber hemû bin-rûpelên girêdayî wê rûpela dêûbav jî neçalak bike.
