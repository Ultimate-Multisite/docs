# 멀티 테넌시

성능과 보안을 위해 하위 사이트를 서버와 데이터베이스에 분산합니다

## Version 1.2.0 릴리스 주요 내용

Multi-Tenancy 1.2.0은 프로덕션 격리 워크플로를 위한 독립 테넌트 기반을 추가합니다:

- 짧은 수명과 재생 공격 방지 기능을 갖춘 SSO 방문을 사용하는 상태 비저장 테넌트 자동 로그인.
- 독립 테넌트를 위한 하위 사이트별 데이터베이스 및 파일시스템 격리.
- 호스트 인식 테넌트 프로비저닝을 위한 Hostinger 멀티 테넌시 기능.
- 레거시 데이터 검사 및 비동기 푸시 처리를 위한 독립 마이그레이션 검증 게이트.
- Bedrock, FrankenPHP 및 localhost 설치를 위한 구성 가능한 동일 머신 데이터베이스 호스트 바인딩.
- 테넌트 부트스트랩, 데이터베이스 작성기, 스키마 부트스트랩 및 사용자 프로비저닝 개선.
- 테넌트 자격 증명을 정리하는 더 안전한 독립 테넌트 해체.

사용자 가이드는 [독립 마이그레이션 워크플로](/user-guide/getting-started/sovereign-migration-workflow), [멀티 테넌시 격리](/user-guide/configuration/multi-tenancy-isolation), [독립 테넌트 운영](/user-guide/administration/sovereign-tenant-operations)을 참조하세요.

## 요구 사항

- WordPress 5.3 이상
- PHP 7.4 이상
- Multisite Ultimate plugin(활성화됨)

## 설치

1. 애드온 파일을 `/wp-content/plugins/` 디렉터리에 업로드합니다
2. WordPress의 'Plugins' 메뉴를 통해 plugin을 활성화합니다
3. 네트워크 관리자에서 애드온 설정을 구성합니다

## 개발

### 설정

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### 테스트

이 애드온에는 포괄적인 테스트 프레임워크가 포함되어 있습니다:

- **단위 테스트**: WordPress 테스트 스위트를 사용하는 PHPUnit 기반 테스트
- **기본 테스트 클래스**: 일반적인 테스트 시나리오를 위한 도우미 메서드 제공
- **테스트 범위**: 코드 범위를 추적하도록 구성됨
- **멀티사이트 테스트**: 테스트는 멀티사이트 환경에서 실행됨

### 코드 표준

- **WordPress 코딩 표준**: PHPCS를 통해 적용됨
- **PHP 7.4+ 호환성**: 최신 PHP 기능 지원
- **정적 분석**: 유형 검사를 위한 PHPStan 통합
- **코드 현대화**: PHP 업그레이드를 위한 Rector 규칙

### 빌드 시스템

- **자산 처리**: CSS/JS 파일 축소
- **번역**: POT 파일 생성
- **패키징**: 배포용 아카이브 생성
- **개발 모드**: 디버깅용 축소되지 않은 자산

## 구성

이 애드온은 네트워크 관리자 인터페이스를 통해 접근할 수 있는 다양한 구성 옵션을 지원합니다.

### 독립 테넌트 구성

독립 테넌트의 경우, 테넌트로 트래픽을 라우팅하기 전에 테넌트 데이터베이스, 파일시스템 루트, 데이터베이스 호스트 바인딩 및 호스트 기능을 구성하세요. 데이터베이스 자격 증명, 호스트 바인딩, 라우팅 또는 사용자 프로비저닝에 영향을 주는 모든 변경 후에는 마이그레이션 검증을 실행하세요.

테넌트 런타임과 데이터베이스 권한이 로컬 소켓 또는 동일 머신 접근을 예상하는 경우에만 `localhost`를 사용하세요. 제공업체가 구성한 권한 부여 호스트가 그것인 경우 `127.0.0.1` 또는 서비스 호스트 이름을 사용하세요.

### Hostinger 기능

Hostinger 멀티 테넌시 기능은 공유 Hostinger 통합 토큰과 함께 작동합니다. 검증 전에 도메인이 hPanel에 있는지, DNS가 Hostinger 계정을 가리키는지, 테넌트 데이터베이스 설정이 hPanel과 일치하는지 확인하세요.

## 훅 및 필터

### 액션

- `ultimate-multisite-addon-template_init` - 애드온 초기화 후 실행됨
- `ultimate-multisite-addon-template_loaded` - 모든 애드온 파일이 로드된 후 실행됨

### 필터

- `ultimate-multisite-addon-template_settings` - 애드온 설정 수정
- `ultimate-multisite-addon-template_enabled` - 애드온 활성화/비활성화 상태 재정의
