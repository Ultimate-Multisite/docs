# 다중 통화 (Multi-Currency)

Ultimate Multisite: Multi-Currency는 네트워크에서 여러 통화로 결제를 받을 수 있도록 지원합니다. 이 플러그인은 실시간 또는 수동 환율을 사용하여 가격을 자동으로 변환하고, 모든 통화에 대해 상품별 정확한 가격을 설정할 수 있게 하며, 지리적 위치(geolocation)를 통해 방문자가 선호하는 통화를 감지합니다. 이 모든 기능은 단일 관리자 대시보드에서 관리할 수 있습니다.

## 주요 기능

- **자동 환율 변환** — 선택한 제공업체에서 가져온 환율을 사용하여 기본 통화의 가격을 변환합니다.
- **상품별 가격 재정의** — 자동 변환을 우회하여 모든 상품에 대해 통화별로 정확한 가격을 설정할 수 있습니다.
- **통화 선택기 가입 필드** — 고객이 결제/가입 과정에서 선호하는 통화를 직접 선택할 수 있게 합니다.
- **다중 환율 제공업체** — ECB, Frankfurter, Stripe 또는 완전히 수동으로 환율을 설정할 수 있습니다.
- **지리적 위치 기반 통화 감지** — 방문자의 국가에 맞춰 통화를 자동으로 미리 선택합니다.
- **관리자 대시보드** — 활성화된 통화, 환율, 제공업체 설정을 한 곳에서 관리합니다.

## 요구 사항

- Ultimate Multisite 2.0.0 이상
- WordPress Multisite 설치
- PHP 7.4 이상

## 설치

1. 플러그인 파일을 `/wp-content/plugins/ultimate-multisite-multi-currency`에 업로드합니다.
2. WordPress의 **플러그인** 화면에서 플러그인을 활성화합니다 (네트워크 활성화).
3. **Ultimate Multisite → Multi-Currency**로 이동하여 관리자 대시보드를 엽니다.
4. 받고 싶은 통화를 추가하고 환율 제공업체를 구성합니다.

## 설정

### 관리자 대시보드

Multi-Currency 관리자 대시보드 (**Ultimate Multisite → Multi-Currency**)는 다음 작업을 수행하는 중앙 허브입니다.

- 통화 추가, 활성화 또는 비활성화
- 환율 제공업체를 선택하고 필요할 때 환율을 새로고침
- 모든 활성 통화의 현재 환율 검토
- 자동 제공업체가 적합하지 않을 때 수동 환율 설정

### 환율 제공업체

| 제공업체 | 설명 |
|---|---|
| **ECB** | 유럽 중앙은행 — EUR 기반 네트워크를 위한 무료 일일 환율 |
| **Frankfurter** | 오픈 소스 ECB 미러 API — 무료이며 API 키가 필요하지 않습니다. |
| **Stripe** | Stripe의 실시간 FX 환율을 사용합니다 — 활성 Stripe 통합이 필요합니다. |
| **수동(Manual)** | 사용자가 직접 환율을 입력합니다. 외부 API 호출이 없습니다. |

**Ultimate Multisite → Multi-Currency → 설정 → 환율 제공업체**에서 제공업체를 선택합니다. 환율은 언제든지 수동으로 새로고침하거나 자동으로 업데이트되도록 예약할 수 있습니다.

### 상품별 가격 재정의

자동 변환이 충분히 정확하지 않은 경우, 모든 상품에 대해 특정 통화의 정확한 가격을 설정할 수 있습니다.

1. 상품 편집기 열기 (**Ultimate Multisite → 상품 → 편집**)
2. **통화 가격 책정(Currency Pricing)** 섹션으로 스크롤합니다.
3. 재정의하려는 각 통화에 대한 정확한 가격을 입력합니다.
4. 상품을 저장합니다.

재정의가 없는 상품은 기본 통화에서 변환된 가격을 사용합니다.

### 통화 선택기 가입 필드

통화 선택기 필드를 가입/결제 흐름에 추가하여 고객이 등록 시 선호하는 통화를 선택하도록 할 수 있습니다.

1. **Ultimate Multisite → Multi-Currency → 설정**으로 이동합니다.
2. **가입 시 통화 선택기 표시(Show currency selector on signup)**를 활성화합니다.
3. 선택적으로 기본 통화를 설정하거나 지리적 위치가 미리 선택되는 값을 결정하도록 할 수 있습니다.

선택된 통화는 구독 정보와 함께 저장되며 이후 모든 청구에 사용됩니다.

### 지리적 위치 기반 통화 감지

지리적 위치 감지가 활성화되면, 애드온은 방문자의 국가를 통화에 매핑하고 통화 선택기 필드에서 이를 미리 선택합니다. 활성화하려면:

1. **Ultimate Multisite → Multi-Currency → 설정**으로 이동합니다.
2. **방문자 위치에서 통화 감지(Detect currency from visitor location)**를 활성화합니다.
3. **지리적 위치 매핑(Geolocation Mapping)** 테이블에서 국가를 통화에 매핑합니다.

방문자의 국가에 대한 매핑이 없는 경우, 네트워크의 기본 통화가 대체(fallback)로 사용됩니다.

## 자주 묻는 질문

### 환율 제공업체가 사용할 수 없게 되면 어떻게 되나요?

애드온은 마지막으로 성공적으로 가져온 환율로 대체됩니다. 환율을 가져온 적이 없다면, 설정한 수동 환율로 대체됩니다. 안전장치로 수동 환율을 구성하는 것이 좋습니다.

### 통화별로 다른 제공업체를 사용할 수 있나요?

아니요. 모든 통화에 대해 한 번에 하나의 제공업체만 활성화됩니다. 개별 환율에 대한 완전한 제어가 필요하면 **수동(Manual)**을 사용하세요.

### 지리적 위치 감지에 별도의 플러그인이 필요한가요?

외부 지리적 위치 플러그인은 필요하지 않습니다. 애드온은 WordPress의 내장 IP-to-country 감지 기능을 사용합니다. 더 높은 정확도를 위해서는 제공된 필터 훅을 통해 전용 지리적 위치 서비스를 통합할 수 있습니다.

### 상품별 재정의는 세금 부과 전인가요, 후인가요?

상품별 재정의는 대상 통화의 세금 제외 가격을 설정합니다. 세금은 기존 세금 설정을 사용하여 재정의 가격 위에 계산됩니다.
