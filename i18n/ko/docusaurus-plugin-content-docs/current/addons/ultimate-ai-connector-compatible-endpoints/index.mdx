# Ultimate AI Connector for Compatible Endpoints {#ultimate-ai-connector-for-compatible-endpoints}

Ultimate AI Connector for Compatible Endpoints는 OpenAI와 호환되는 AI 제공업체 API를 WordPress 멀티사이트 네트워크에 통합합니다. 여러 AI 제공업체를 설정할 수 있으며, 각 제공업체는 고유한 엔드포인트 URL, API 키, 기본 모델을 가집니다. 또한, 특정 제공업체가 사용 불가능할 경우 자동으로 요청을 다른 제공업체로 전환(폴백)하는 기능도 제공합니다.

## 주요 기능 {#key-features}

- **다중 제공업체 지원**: 각 제공업체별 엔드포인트 URL, API 키, 기본 모델을 가진 여러 AI 제공업체 카드를 추가할 수 있습니다.
- **드래그하여 순서 변경**: 각 카드에 있는 손잡이(grip handle)를 사용하여 제공업체의 우선순위에 따라 순서를 배열할 수 있습니다. 요청은 이 순서대로 처리됩니다.
- **자동 폴백 라우팅**: 기본 제공업체가 사용 불가능할 경우, 요청은 자동으로 다음으로 설정된 제공업체로 전환됩니다.
- **제공업체별 설정**: 각 제공업체는 고유한 자격 증명과 모델 선택을 통해 독립적으로 구성됩니다.
- **카드 기반 설정 UI**: 안정적인 WordPress 6.9+ Card 컴포넌트(CardBody, CardHeader, CardDivider)를 기반으로 구축되었습니다.
- **추가 시 자동 확장**: 새로운 제공업체 카드를 추가하면 자동으로 확장되어 즉시 세부 정보를 입력할 수 있습니다.
- **스크립트 캐시 무효화**: 플러그인 업데이트 시 제공업체 스크립트가 캐시 무효화되어 항상 최신 버전이 로드되도록 보장합니다.

## 요구 사항 {#requirements}

- WordPress 5.3 이상
- PHP 7.4 이상
- Ultimate Multisite 플러그인 (활성화됨)
- OpenAI와 호환되는 API 엔드포인트 (자체 호스팅 또는 타사)

## 설치 {#installation}

1. 애드온 파일을 `/wp-content/plugins/` 디렉토리에 업로드합니다.
2. WordPress의 '플러그인' 메뉴를 통해 플러그인을 활성화합니다.
3. **Ultimate Multisite → AI Connector**로 이동하여 애드온을 구성합니다.

## 설정 {#configuration}

### 다중 제공업체 설정 {#multi-provider-setup}

설정 화면은 카드 기반 레이아웃을 사용하며, 각 카드는 하나의 AI 제공업체를 나타냅니다. 필요한 만큼 많은 제공업체를 추가할 수 있습니다.

#### 제공업체 추가하기 {#adding-a-provider}

1. 네트워크 관리자에서 **Ultimate Multisite → AI Connector**로 이동합니다.
2. **Add Provider**를 클릭합니다.
3. 새로운 제공업체 카드가 나타나고 자동으로 확장됩니다.
4. 제공업체 세부 정보를 입력합니다:
   - **Provider Name**: 이 제공업체를 식별하는 레이블 (예: "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL**: OpenAI와 호환되는 API의 기본 URL (예: `https://api.openai.com/v1`)
   - **API Key**: 이 제공업체에 대한 API 키
   - **Default Model**: 이 제공업체에서 기본적으로 사용할 모델 식별자 (예: `gpt-4o`, `mistral-small`)
5. **Save Settings**를 클릭합니다.

#### 제공업체 제거하기 {#removing-a-provider}

제거하려는 제공업체 카드에서 **Remove** 버튼을 클릭하여 삭제합니다. 변경 사항을 적용하려면 설정을 저장하세요.

#### 제공업체 순서 변경하기 {#reordering-providers}

제공업체 카드의 왼쪽 측면에 있는 손잡이 아이콘(⣿)을 드래그하여 순서를 변경할 수 있습니다. 이 순서가 폴백 우선순위를 결정합니다. 요청은 첫 번째 제공업체로 전송되며, 요청이 실패하면 다음 제공업체로 폴백됩니다.

### 폴백 라우팅 {#fallback-routing}

기본 제공업체로의 요청이 시간 초과, 속도 제한 또는 서비스 중단 등의 이유로 실패하면, 커넥터는 목록의 다음 제공업체로 자동으로 재시도합니다. 이 과정은 제공업체가 성공하거나 모든 제공업체를 시도할 때까지 목록을 따라 계속됩니다.

| 시나리오 | 동작 |
|----------|-----------|
| 기본 제공업체가 응답함 | 요청이 정상적으로 처리됩니다. |
| 기본 제공업체가 실패함 | 다음 제공업체로 요청이 재시도됩니다. |
| 모든 제공업체가 실패함 | 호출자에게 오류가 반환됩니다. |

폴백 기능은 호출하는 플러그인이나 테마에게는 투명합니다. 몇 개의 제공업체가 시도되었는지와 관계없이, 성공적인 응답 또는 최종 오류 중 하나를 받게 됩니다.

### 제공업체 설정 참조 {#provider-configuration-reference}

| 필드 | 필수 여부 | 설명 |
|-------|----------|-------------|
| Provider Name | 예 | 이 제공업체를 위한 표시 레이블 |
| Endpoint URL | 예 | OpenAI와 호환되는 API의 기본 URL |
| API Key | 예 | 이 제공업체를 위한 인증 키 |
| Default Model | 예 | 요청에 모델이 지정되지 않았을 때 사용되는 모델 식별자 |

## 문제 해결 {#troubleshooting}

### 제공업체가 목록에 표시되지 않음 {#no-providers-are-listed}

제공업체 목록을 저장한 후 비어 있다면 다음 사항을 확인하세요.

- 제공업체 추가 후 **Save Settings**를 클릭했는지 확인합니다.
- 브라우저가 설정 페이지 저장을 차단하지 않았는지 확인합니다 (브라우저 콘솔에서 JavaScript 오류를 확인하세요).
- 플러그인이 활성화되어 있고 네트워크 관리자 화면에 있는지 확인합니다.

### 모든 제공업체에서 요청 실패 {#requests-are-failing-for-all-providers}

- 각 제공업체의 엔드포인트 URL이 서버에서 도달 가능한지 확인합니다 (브라우저에서만 가능한 것이 아닌지).
- 각 API 키가 유효하고 만료되지 않았는지 확인합니다.
- 제공업체의 대시보드에서 속도 제한 및 할당량 상태를 검토합니다.
- 커넥터의 상세 오류 메시지를 캡처하기 위해 WordPress 디버그 로깅(`WP_DEBUG_LOG`)을 활성화합니다.

### 폴백에 특정 제공업체가 사용되지 않음 {#a-provider-is-not-being-used-for-fallback}

드래그하여 순서를 변경하면 폴백 순서가 변경됩니다. 폴백 요청을 받을 것으로 예상되는 제공업체가 목록에 올바르게 배치되어 있는지, 그리고 해당 제공업체의 엔드포인트 URL과 API 키가 정확한지 확인하세요.

### 새로운 제공업체 카드가 자동으로 확장되지 않음 {#new-provider-card-does-not-expand-automatically}

자동 확장은 브라우저에서 JavaScript가 활성화되어 있어야 합니다. 카드가 클릭 시 확장되지 않으면 브라우저 콘솔에서 오류를 확인하세요. 자동 확장이 작동하지 않더라도 카드 헤더를 클릭하면 카드 내용은 항상 접근할 수 있습니다.
