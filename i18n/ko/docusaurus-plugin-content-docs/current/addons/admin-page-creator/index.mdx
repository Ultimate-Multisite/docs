# Admin Page Creator {#admin-page-creator}

## 콘텐츠 {#content}
Admin page creator를 사용하면 WordPress 관리자 Dashboard 내에 새 관리자 페이지와 하위 페이지를 쉽게 추가할 수 있습니다. 새 사용자를 위한 랜딩 페이지 역할을 할 수 있는 환영 페이지나 튜토리얼 페이지를 만드는 데 사용할 수 있습니다. Beaver Builder, Elementor, Brizy, Oxygen 같은 페이지 빌더를 사용하여 번거로움 없이 페이지를 디자인할 수 있습니다.

또한 사용자 지정 HTML, CSS 및 JavaScript 지원뿐만 아니라 plan 및 사용자 역할 기반 접근 제어도 제공합니다!

## 기능 {#features}
* 사용자 지정 HTML 및 CSS 지원
* JavaScript 지원
* PHP 콘텐츠 모드
* 사용자 역할 기반 권한
* 페이지 빌더 지원

## 문서 {#documentation}
[Multisite Ultimate: Admin Page Creator를 사용하여 사용자 지정 관리자 페이지 및 하위 페이지 만들기](https://help.multisiteultimate.com/article/305-creating-custom-admin-pages-and-sub-pages-using-ultimate-multisite-admin-page-creator)

## 설치 {#installation}

1. **ultimate-multisite-admin-page-creator**를 **/wp-content/plugins/** 디렉터리에 업로드합니다
2. WordPress Network Admin의 **플러그인** 메뉴를 통해 plugin을 활성화합니다

## 사용 가이드 {#usage-guide}

WordPress는 사이트의 프런트엔드에 콘텐츠를 추가하는 일을 매우 쉽게 만들어 주지만, 관리자 쪽에서 페이지와 하위 페이지를 만들려면 약간의 기본 코딩 기술이 필요합니다. Ultimate Multisite: Admin Page Creator는 프런트엔드와 같은 사용 편의성을 네트워크 사이트의 백엔드에도 제공하는 것을 목표로 합니다.

## Admin Page Creator를 사용하는 이유는 무엇인가요? {#why-use-admin-page-creator}

## 메뉴 옵션 {#menu-options}

측면의 일반 옵션 메타 박스에서 메뉴 유형, 레이블 등 다양한 항목을 구성할 수 있습니다.

### 메뉴 유형 {#menu-types}

새 사용자 지정 관리자 페이지에는 두 가지 메뉴 유형이 있을 수 있습니다. 최상위 메뉴와 하위 메뉴입니다.

최상위 메뉴 페이지는 이름에서 알 수 있듯이 새로 만든 페이지를 기본 WordPress 페이지와 함께 관리자 메뉴에 추가합니다.

최상위 옵션을 선택하면 메뉴 순서와 메뉴 아이콘도 선택할 수 있습니다.

이 페이지를 하위 메뉴 페이지로 만들고 싶다면 이 페이지를 연결할 상위 페이지를 선택할 수 있습니다. 사용자 지정 페이지를 기본 WordPress 페이지 또는 최상위 사용자 지정 관리자 페이지에 추가할 수 있습니다.

_이 사용자 지정 페이지의 메뉴 항목을 완전히 사용자 지정하세요_

## 고급 옵션 {#advanced-options}

사용자 지정 관리자 페이지에 모든 종류의 콘텐츠를 추가할 수 있어야 합니다. 여기에는 CSS 규칙과 파일, JavaScript, 외부 라이브러리도 포함됩니다. Advanced Options 탭을 사용하여 이를 수행할 수 있습니다.

### 사용자 지정 CSS {#custom-css}

CSS 탭에서는 자체 사용자 지정 CSS 코드를 추가할 수 있을 뿐만 아니라 원격 사이트(예: CDN, Google Fonts 등)에서 CSS 파일을 가져올 수도 있습니다.

<!-- 스크린샷 없음: 외부 스타일시트 가져오기가 있는 Admin Page Creator 사용자 지정 CSS 편집기 -->

_사용자 지정 CSS도 지원됩니다. 외부 스타일 파일도 포함할 수 있습니다!_

### 사용자 지정 JavaScript {#custom-javascript}

JavaScript도 마찬가지입니다. 사용자 지정 JavaScript 코드에서 사용할 외부 라이브러리를 추가할 수 있습니다.

## 권한 {#permissions}

Advanced Options에서도 어떤 하위 사이트가 이 페이지에 접근할 수 있는지 선택할 수 있습니다. 예를 들어 서로 다른 plan 등급에 서로 다른 독점 콘텐츠를 제공하려는 경우 유용합니다.

<!-- 스크린샷 없음: plan 및 역할 기반 접근 제어를 위한 Admin Page Creator 권한 설정 -->

_이 페이지가 어디에 표시될지 완전히 제어할 수 있습니다_

**중요** : 페이지가 표시되려면 두 조건이 모두 충족되어야 합니다. 즉, 사용자가 특정 요금제의 클라이언트이지만 선택된 역할 중 하나를 가지고 있지 않거나(또는 그 반대인 경우) 해당 페이지는 그들의 패널에 나타나지 않습니다.

## 활성 {#active}

사용자 지정 관리자 페이지를 삭제하거나 권한을 건드리지 않고 비활성화하려면 Active 옵션의 선택을 해제하면 됩니다.

상위 수준 페이지를 비활성화하면 해당 상위 페이지에 연결된 모든 하위 페이지가 자동으로 비활성화됩니다.
