# 변경 로그

## 1.9.0 — 2026-04-28 출시

### 새로 추가된 기능

- **`create_contact_form` 기능** — 활성화된 문의 양식 플러그인(Contact Form 7, WPForms, Fluent Forms, 또는 Gravity Forms)을 사용하여 문의 양식을 생성하고, 어떤 포스트나 페이지에도 삽입할 수 있는 쇼트코드를 반환합니다.
- **`set_featured_image` 기능** — 기존 미디어 라이브러리 첨부 ID 또는 원격 URL을 사용하여 포스트나 페이지에 대표 이미지를 지정합니다. URL을 제공하면 자동으로 이미지를 가져옵니다.
- **`batch_create_posts` 기능** — 단일 기능 호출로 여러 개의 포스트를 생성할 수 있습니다. `create_post`와 동일한 매개변수를 지원하며, 포스트별 성공/실패 여부를 보고하고, 선택적으로 `stop_on_error` 모드를 지원합니다.
- **`page_template` 매개변수** — `create_post`와 `update_post`에 추가되었습니다. 포스트를 생성하거나 업데이트할 때 PHP 페이지 템플릿 파일(예: `page-full-width.php`)을 지정할 수 있습니다. `update_post`에 빈 문자열을 전달하면 테마 기본값으로 되돌아갑니다.
- **클라이언트 측 스크린샷 기능** — `capture_screenshot`, `compare_screenshots`, `review_page_design` 기능이 추가되었습니다. 서버 측 headless 브라우저를 통해 실제 페이지의 전체 또는 뷰포트 스크린샷을 캡처하고, 두 스크린샷을 비교하며, 레이아웃, 타이포그래피, 색상, 접근성을 포함하는 AI 기반 디자인 검토 보고서를 받습니다.
- **다섯 가지 내장 에이전트** — 콘텐츠 작성기(Content Writer), 사이트 빌더(Site Builder), 디자인 스튜디오(Design Studio), 플러그인 관리자(Plugin Manager), 지원 도우미(Support Assistant)가 추가되었습니다. 각 에이전트는 전용 도구 세트, 맞춤 시스템 프롬프트, 시작 제안을 가지고 있습니다. 채팅 헤더의 새로운 **에이전트 선택기(Agent Picker)**를 통해 전환할 수 있습니다. [내장 에이전트](../../user-guide/configuration/built-in-agents)를 참조하세요.
- **기능 플래그(Feature flags)** — **설정 → 기능 플래그**에 새로운 탭이 추가되었으며, 접근 제어 토글(관리자에게 제한, 네트워크 관리자에게 제한, 구독자 접근, 비회원 비활성화)과 브랜딩 옵션(푸터 출처 표시 숨기기, 사용자 지정 에이전트 이름, 에이전트 선택기 숨기기, 사이트 아이콘을 채팅 아바타로 사용)을 설정할 수 있습니다. [Gratis AI Agent 설정](../../user-guide/administration/gratis-ai-agent-settings)를 참조하세요.
- **최근 세션 복원** — 채팅 패널이 페이지 로드 시와 위젯 열림 시 가장 최근 대화를 자동으로 다시 로드하므로, 페이지 이동 중에도 대화 맥락이 끊기지 않습니다.
- **플러그인 액션 링크** — WordPress **플러그인 → 설치된 플러그인** 화면의 플러그인 설명 아래에 설정 및 기능 레지스트리로의 빠른 링크가 표시됩니다.

### 개선된 기능

- **이미지 소스 재시도** — 에이전트가 다운로드 실패 시 AI 생성 이미지로 대체하기 전에 구성된 모든 무료 이미지 소스를 재시도합니다.
- **모델 정보 패널** — 채팅 헤더에 항상 표시됩니다. 더 이상 첫 번째 메시지 이후에 숨겨지지 않습니다.
- **자동 스크롤 동작** — 사용자가 읽기 위해 위로 스크롤하면 자동 스크롤이 일시 중지됩니다. 부유하는 **맨 아래로 스크롤** 버튼이 나타나며, 사용자가 최신 메시지에 도달하면 자동으로 사라집니다.
- **에이전트 선택기 UI** — 에이전트별 아이콘이 있는 폼-테이블 오버레이로 재설계되어, 한눈에 에이전트를 식별하고 전환하기가 더 쉬워졌습니다.
- **지연 로딩 JS 청크** — 채팅 위젯의 초기 JavaScript 번들이 지연 로딩 청크로 분할되어 초기 번들 크기가 75~90% 감소했습니다.
- **채팅 위젯 디자인 개선** — 통일된 AI 아이콘이 이전의 사용자 지정 아바타를 대체했습니다. 내장 에이전트 시스템과 일관성을 유지합니다.
- **URL 링크화** — 시스템 메시지 및 오류 메시지 버블에 나타나는 URL이 이제 클릭 가능한 링크로 렌더링됩니다.

### 수정된 기능

- **기능 검색 가능성** — 설명, 시스템 프롬프트 참조, 네임스페이스 정렬을 수정하여 모든 기능이 에이전트의 도구 목록에 안정적으로 표시되도록 했습니다.
- **프로바이더 캐시** — 프로바이더가 버전 카운터를 통해 사이트 전체에 캐시되어, 멀티사이트 네트워크에서 오래된 프로바이더 문제가 발생하는 것을 방지합니다.
- **`ability_invalid_output`** — 12개 기능 핸들러 전반에 걸쳐 해결되었습니다. 모든 핸들러가 올바르게 구조화된 JSON 응답을 반환합니다.
- **`pending_client_tool_calls` 파이프라인** — 엔드투엔드로 연결되어 클라이언트 측 도구 호출이 올바르게 완료되고 모델에 결과를 반환하도록 했습니다.
- **히스토리 드로어** — 되돌릴 수 없는 변경 사항은 되돌리기 목록에서 제외됩니다. **전체 히스토리 보기** 링크가 이제 올바르게 작동합니다.
- **변경/되돌리기 시스템** — 다섯 가지 별도의 버그가 수정되었으며 새로운 관리자 인터페이스 아래에 통합되었습니다.
- **설정 저장 토스트** — **설정 저장**을 클릭한 후 스낵바 알림이 이제 안정적으로 표시됩니다.
- **휴지통 컨텍스트 메뉴** — **영구 삭제** 옵션이 추가되어 휴지통 보기에서 항목을 남기지 않고 하드 삭제할 수 있습니다.
- **편집 및 재전송** — 편집 아이콘을 클릭하면 스레드의 모든 메시지가 아닌 클릭된 메시지에 대해서만 편집 모드로 진입합니다.
- **채팅 레이아웃 높이** — 플러그인 삽입 콘텐츠(관리자 알림, 배너)가 페이지 상단에 나타날 때 채팅 패널이 높이를 조정하여, 입력 영역이 화면 밖으로 밀려나지 않도록 합니다.

## 1.4.0 — 2026-04-09 출시

### 새로 추가된 기능

- **WP-CLI 벤치마크 명령어** (`wp gratis-ai-agent benchmark`) — CI 파이프라인 및 모델 평가 워크플로우를 위해 명령줄에서 Agent Capabilities v1 벤치마크 스위트를 실행합니다. 질문별 실행, 프로바이더/모델 재정의, JSON/CSV 출력을 지원합니다.
- **Agent Capabilities v1 벤치마크 스위트** — 전체 기능 표면을 테스트하는 구조화된 복잡한 다단계 프롬프트 세트로, 점수 및 토큰/지속 시간 보고서를 제공합니다.
- **Custom Post Type 기능** — `register_post_type`, `list_post_types`, `delete_post_type`. 등록 정보는 옵션 테이블을 통해 재시작 시에도 유지됩니다.
- **Custom Taxonomy 기능** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. 계층적 및 평면적 분류를 지원하며, 선택적 리라이트 슬러그를 포함합니다.
- **디자인 시스템 기능** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`의 다섯 가지 큐레이션된 프리셋을 포함합니다.
- **글로벌 스타일 기능** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. WordPress Global Styles API를 통해 theme.json 값을 읽고 씁니다.
- **내비게이션 메뉴 관리 기능** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. 중첩 메뉴 및 테마 위치 할당을 지원합니다.
- **옵션 관리 기능** — `get_option`, `set_option`, `delete_option`, `list_options`. 중요한 WordPress 옵션이 수정되는 것을 보호하는 내장 안전 블랙리스트를 포함합니다.
- **설치 가능한 기능 레지스트리** — `list_available_abilities`, `install_ability`, `recommend_plugin`. WordPress 플러그인으로 배포되는 기능 팩을 검색하고 활성화합니다.
- **사이트 빌더 오케스트레이션 v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. 플러그인 검색, 단계별 출력 참조, 진행 상황 추적, 자율 오류 복구를 포함하는 다단계 사이트 빌드 계획입니다.
- **레스토랑 웹사이트 벤치마크 질문** (`q-restaurant-website`) — CPT 등록, 디자인 시스템, 내비게이션, 플러그인 검색을 포괄하는 엔드투엔드 테스트입니다.
- **AI 프로바이더 커넥터 플러그인** — 로컬 개발 환경 설정을 빠르게 하기 위해 WordPress Playground 청사진에 추가되었습니다.

### 개선된 기능

- README에 AI 프로바이더 커넥터 문서 및 설정 지침이 업데이트되었습니다.

### 수정된 기능

- `main` 브랜치에서 25개의 PHPUnit 테스트 실패가 해결되었습니다.
- `blueprint.json`의 GitHub 릴리스 URL 형식이 수정되었습니다.
- 오래된 ID와의 충돌을 피하기 위해 태스크 ID가 재번호 매김되었습니다.
