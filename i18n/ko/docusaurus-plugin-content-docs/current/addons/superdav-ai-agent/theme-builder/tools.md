# 도구 및 구성

Theme Builder는 WordPress 사이트를 분석, 디자인, 구축하기 위해 도구 모음을 사용합니다. 버전 1.16.0에서는 **sd-ai-agent/site-scrape가 이제 Tier 1 도구**가 되어 Theme Builder에서 기본적으로 사용할 수 있습니다. Superdav AI Agent v1.18.0은 신뢰할 수 있는 개발자 워크플로를 위해 별도로 배포되는 Advanced 동반 패키지도 도입합니다. 이러한 도구는 WordPress.org 핵심 패키지의 일부가 아니며 별도로 설치하고 승인해야 합니다.

## 사용 가능한 도구 {#available-tools}

### Tier 1 도구(항상 사용 가능) {#tier-1-tools-always-available}

Tier 1 도구는 추가 구성 없이 Theme Builder에서 기본적으로 사용할 수 있습니다.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**목적:** 기존 웹사이트를 분석하여 디자인 영감, 콘텐츠 구조, 기능을 추출합니다.

**기능:**

- **웹사이트 분석** — 경쟁사 또는 영감이 되는 웹사이트를 스크레이프하고 분석
- **디자인 추출** — 색상, 글꼴, 레이아웃 패턴 식별
- **콘텐츠 구조** — 페이지 구성과 계층 구조 이해
- **기능 감지** — plugin 및 기능 식별
- **성능 분석** — 페이지 속도와 최적화 확인
- **SEO 분석** — 메타 태그와 구조화된 데이터 검토

**사용법:**

```
example.com의 디자인을 분석하고 내 사이트의 영감으로 사용하세요.
```

**추출하는 항목:**

- 색상 팔레트와 색상 사용 방식
- 타이포그래피(글꼴 및 크기)
- 레이아웃 구조와 간격
- 내비게이션 패턴
- 콘텐츠 구성
- 이미지와 미디어 사용
- 상호작용 요소
- 모바일 반응성

**제한 사항:**

- 비밀번호로 보호된 사이트는 스크레이프할 수 없음
- robots.txt와 사이트 정책을 준수
- 동적 콘텐츠를 캡처하지 못할 수 있음
- JavaScript가 많은 사이트는 분석이 제한될 수 있음
- 대규모 사이트는 분석에 더 오래 걸릴 수 있음

### Tier 2 도구(선택 사항) {#tier-2-tools-optional}

Tier 2 도구는 Theme Builder 설정에서 명시적으로 활성화했을 때 사용할 수 있습니다.

#### 고급 분석 {#advanced-analytics}

상세한 성능 지표를 제공합니다:

- 페이지 로드 시간
- Core Web Vitals
- SEO 점수
- 접근성 점수
- 모범 사례 점수

#### 콘텐츠 최적화 도구 {#content-optimizer}

다음 항목을 분석하고 개선 사항을 제안합니다:

- 가독성
- SEO 최적화
- 키워드 사용
- 콘텐츠 구조
- 행동 유도 문구 배치

### Tier 3 도구(프리미엄) {#tier-3-tools-premium}

Tier 3 도구는 추가 구성 또는 프리미엄 기능이 필요합니다.

#### AI 콘텐츠 생성기 {#ai-content-generator}

다음 콘텐츠를 생성합니다:

- 제품 설명
- 서비스 페이지
- 블로그 글
- 메타 설명
- 행동 유도 문구 카피

#### 고급 디자인 도구 {#advanced-design-tools}

고급 디자인 기능을 제공합니다:

- 사용자 지정 CSS 생성
- 애니메이션 생성
- 상호작용 요소 디자인
- 고급 색상 이론
- 타이포그래피 최적화

### Advanced 동반 도구(별도 패키지) {#advanced-companion-tools-separate-package}

Advanced 동반 패키지는 핵심 Superdav AI Agent 릴리스와 별도로 배포됩니다. 이는 관리자가 agent가 영향력이 큰 도구를 사용하도록 명시적으로 허용하는, 신뢰할 수 있는 개발자 및 사이트 소유자 환경을 위한 것입니다.

Advanced 동반 기능에는 다음이 포함될 수 있습니다:

- **파일 시스템 도구** — 신뢰할 수 있는 환경에서 승인된 파일 검사 및 수정
- **데이터베이스 도구** — 명시적으로 승인된 경우 사이트 데이터 쿼리 또는 업데이트
- **WP-CLI 도구** — WordPress 유지관리 및 검사 명령 실행
- **REST 디스패처 도구** — 제어된 디스패치 워크플로를 통해 등록된 REST endpoint 호출
- **Plugin 빌더 도구** — plugin code 스캐폴드 및 반복 개선
- **Git 스냅샷 도구** — 위험한 개발자 작업 전에 스냅샷 생성
- **사용자 관리 도구** — 허용된 경우 사용자 기록 검토 또는 조정 지원
- **벤치마크 도구** — 개발 검토를 위한 성능 또는 기능 벤치마크 실행

이 도구들을 모든 설치에서 일반적으로 사용할 수 있는 Theme Builder 도구로 문서화하지 마세요. Advanced 동반 패키지가 설치되고, 활성화되며, 신뢰할 수 있는 관리자에게 제한된 경우에만 사용할 수 있습니다.

## 도구 구성 {#tool-configuration}

### 도구 활성화 {#enabling-tools}

Theme Builder에서 추가 도구를 활성화하려면:

1. **Gratis AI Agent → 설정**으로 이동
2. **Theme Builder → 도구**로 이동
3. 필요에 따라 도구 켜기/끄기 전환
4. 설정 저장

### 도구 권한 {#tool-permissions}

일부 도구에는 권한이 필요합니다:

- **사이트 스크레이핑** — 인터넷 접근 필요
- **분석** — Google Analytics 통합 필요
- **콘텐츠 생성** — API 키 필요
- **고급 기능** — 프리미엄 구독이 필요할 수 있음
- **Advanced 동반 도구** — 별도로 배포되는 Advanced 패키지와 명시적인 관리자 신뢰 필요

### API 키 및 자격 증명 {#api-keys-and-credentials}

필요한 도구에 대한 API 키를 구성하세요:

1. **Gratis AI Agent → 설정 → API 키**로 이동
2. 각 도구의 자격 증명 입력
3. 연결 테스트
4. 안전하게 저장

### Advanced 동반 패키지 설치 {#installing-the-advanced-companion-package}

Advanced 동반 패키지는 WordPress.org 핵심 ZIP과 별도로 릴리스됩니다. 해당 릴리스의 공식 프로젝트 배포 채널에서만 설치한 다음, 신뢰할 수 있는 관리자에게 접근을 제한하세요. 프로덕션 사이트에서 파일 시스템, 데이터베이스, WP-CLI, REST 디스패처, plugin 빌더, 사용자 관리 또는 벤치마크 도구를 활성화하기 전에 사람의 승인 게이트와 git 스냅샷 동작을 검토하세요.

## sd-ai-agent/site-scrape 사용 {#using-sd-ai-agentsite-scrape}

### 기본 사용법 {#basic-usage}

Theme Builder에게 웹사이트를 분석하도록 요청하세요:

```
competitor.com에 있는 내 경쟁사 사이트의 디자인을 분석하고
내 사이트를 위한 디자인 개선 사항을 제안하세요.
```

### 특정 분석 {#specific-analysis}

특정 유형의 분석을 요청하세요:

```
example.com에서 색상 팔레트를 추출하고 이를 영감으로 사용하세요.
```

```
example.com의 내비게이션 구조를 분석하고 비슷한
구성을 내 사이트에 적용하세요.
```

```
example.com의 모바일 반응성을 확인하고 내 사이트도
동일하게 반응형이 되도록 하세요.
```

### 비교 분석 {#comparison-analysis}

여러 사이트를 비교하세요:

```
site1.com과 site2.com의 디자인을 비교하고 두 사이트의 가장 좋은 요소를
결합한 디자인을 만드세요.
```

## 도구 제한 사항 및 고려 사항 {#tool-limitations-and-considerations}

### 속도 제한 {#rate-limiting}

- 스크래핑은 서버 과부하를 방지하기 위해 속도 제한됩니다
- 동일한 사이트에 대한 여러 요청은 제한될 수 있습니다
- 대규모 사이트는 분석하는 데 더 오래 걸릴 수 있습니다

### 콘텐츠 정확성 {#content-accuracy}

- 동적 콘텐츠가 완전히 캡처되지 않을 수 있습니다
- JavaScript로 렌더링된 콘텐츠가 불완전할 수 있습니다
- 실시간 데이터가 반영되지 않을 수 있습니다
- 일부 콘텐츠는 유료 장벽 뒤에 있을 수 있습니다

### 법적 및 윤리적 고려 사항 {#legal-and-ethical-considerations}

- robots.txt와 사이트 정책을 존중하세요
- 저작권이 있는 콘텐츠를 복제 목적으로 스크래핑하지 마세요
- 분석은 복사가 아니라 영감을 얻기 위해 사용하세요
- 추출한 콘텐츠를 사용할 권리가 있는지 확인하세요
- 분석한 사이트의 서비스 약관을 따르세요

### 성능 영향 {#performance-impact}

- 대규모 사이트를 스크래핑하는 데 시간이 걸릴 수 있습니다
- 동시에 여러 스크래핑을 수행하면 더 느릴 수 있습니다
- 네트워크 연결 상태가 속도에 영향을 줍니다
- 대규모 데이터세트는 더 많은 처리가 필요할 수 있습니다

## 모범 사례 {#best-practices}

### 사이트 분석 사용 {#using-site-analysis}

1. **여러 사이트 분석** — 여러 출처에서 영감을 수집하세요
2. **구조에 집중** — 레이아웃과 구성 패턴을 익히세요
3. **색상 추출** — 색상 팔레트를 출발점으로 사용하세요
4. **타이포그래피 연구** — 마음에 드는 글꼴 조합을 파악하세요
5. **탐색 검토** — 메뉴 구성을 이해하세요

### 윤리적 스크래핑 {#ethical-scraping}

1. **robots.txt 존중** — 사이트 지침을 따르세요
2. **콘텐츠 복사 금지** — 분석은 영감을 얻는 용도로만 사용하세요
3. **권리 확인** — 추출한 콘텐츠를 사용할 수 있는지 확인하세요
4. **출처 표시** — 영감의 출처에 크레딧을 표기하세요
5. **약관 준수** — 사이트 서비스 약관을 준수하세요

### 도구 효과 극대화 {#maximizing-tool-effectiveness}

1. **구체적으로 요청** — 특정 유형의 분석을 요청하세요
2. **맥락 제공** — 사이트의 목적을 설명하세요
3. **기대치 설정** — 디자인 목표를 설명하세요
4. **반복 개선** — 결과를 바탕으로 다듬으세요
5. **도구 조합** — 종합적인 분석을 위해 여러 도구를 사용하세요

## 문제 해결 {#troubleshooting}

### 사이트를 스크래핑할 수 없음 {#site-wont-scrape}

- 사이트가 공개적으로 접근 가능한지 확인하세요
- robots.txt가 스크래핑을 허용하는지 확인하세요
- 다른 사이트를 시도해 보세요
- 인터넷 연결을 확인하세요
- 문제가 지속되면 지원팀에 문의하세요

### 불완전한 분석 {#incomplete-analysis}

- 사이트에 동적 콘텐츠가 있을 수 있습니다
- JavaScript가 완전히 렌더링되지 않을 수 있습니다
- 대규모 사이트는 시간이 초과될 수 있습니다
- 대신 특정 페이지를 분석해 보세요
- 특정 분석 유형을 요청하세요

### 느린 성능 {#slow-performance}

- 대규모 사이트는 분석하는 데 더 오래 걸립니다
- 동시에 여러 스크래핑을 수행하면 더 느립니다
- 네트워크 연결 상태가 속도에 영향을 줍니다
- 사용량이 적은 시간대에 시도해 보세요
- 먼저 더 작은 섹션을 분석하세요
