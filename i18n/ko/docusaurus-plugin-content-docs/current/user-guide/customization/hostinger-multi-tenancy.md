# 호스팅터 멀티테넌시

Ultimate Multisite: Multi-Tenancy 1.2.0은 호스팅된 도메인이 기존의 호스팅터 도메인 매핑 통합과 함께 주권 테넌트 프로비저닝에 참여할 수 있도록 호스팅터 멀티테넌시 기능을 추가했습니다.

이 기능은 테넌트 도메인과 격리된 테넌트 인프라를 Hostinger hPanel을 통해 관리하는 경우 사용하세요.

## 설정 참고 사항 {#setup-notes}

1. **Ultimate Multisite > Settings > Host Integrations**에서 핵심 호스팅터 통합을 구성합니다.
2. 호스팅터 API 토큰이 대상 도메인 또는 서브도메인을 관리할 수 있는지 확인합니다.
3. 멀티테넌시 애드온을 활성화합니다.
4. 테넌트를 게시하기 전에 테넌트 격리 전략을 구성합니다.
5. 프로덕션 트래픽을 테넌트로 보내기 전에 마이그레이션 검증 워크플로우를 실행합니다.

호스팅터 기능은 호스트 측 작업을 위해 공유된 호스팅터 연결을 사용합니다. DNS는 여전히 올바른 호스팅터 계정을 가리켜야 하며, hPanel 계정 제한이 계속 적용됩니다.

## 기능별 변경 사항 {#capability-specific-changes}

- 주권 테넌트는 호스트 인식을 기반으로 한 도메인 작업을 통해 생성될 수 있습니다.
- 동일한 머신 데이터베이스 호스트 문자열은 승인 확인 전에 정규화됩니다.
- WordPress 런타임이 로컬 오버라이드를 요구하지 않는 한, Hostinger에서 관리하는 테넌트는 hPanel에 표시된 데이터베이스 호스트 값을 사용해야 합니다.
- SSO 방문은 테넌트 도메인이 호스팅터에서 호스팅되는 테넌트로 해석되는 것에 의존합니다.

## 호스팅터 테넌트 문제 해결 {#troubleshooting-hostinger-tenants}

- 테넌트 설치가 실패하면, 해당 도메인이 이미 호스팅터 계정에 연결되어 있는지 확인하세요.
- 데이터베이스 확인이 실패하면, hPanel과 테넌트 DB 사용자 이름, 데이터베이스 이름 및 호스트 바인딩을 비교해 보세요.
- **방문(SSO)**에 실패하면, 테넌트 도메인에 대해 DNS와 SSL이 활성화되어 있는지 확인하세요.
- 정리 시 호스트 리소스를 남기는 경우, 백업을 확인한 후 hPanel에서 남아 있는 데이터베이스, 사용자 또는 폴더를 제거하세요.
