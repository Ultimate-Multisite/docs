# 주권 테넌트 운영

Ultimate Multisite 1.2.0 버전에서는 네트워크 관리자가 볼 수 있지만 자체 데이터베이스, 파일 시스템 루트 및 라우팅 컨텍스트를 사용하는 서브사이트(sovereign tenants)를 위한 운영 도구를 추가했습니다.

격리된 고객 사이트 관리, 원격 사이트 인수인계 또는 표준 서브사이트를 주권 인프라로 이전하는 작업을 할 때 이 페이지를 사용하세요.

## 관리자에게 변경되는 사항

- **무상태 테넌트 자동 로그인 (Stateless tenant autologin)** — 네트워크 관리자는 장기간 유지되는 공유 세션 상태에 의존하지 않고도 주권 테넌트에 접속할 수 있습니다. SSO 토큰은 목적 범위 지정(purpose-scoped), 오리진 고정(origin-pinned), 재사용 방지(replay-protected)되며 짧은 만료 시간으로 제한됩니다.
- **주권 인식 라우팅 (Sovereign-aware routing)** — 이전의 격리된 네트워크와 주권 테넌트는 동일한 사이트 라우터 경로를 통해 해결되므로, 오래된 설치와 새로운 격리 설치 간의 부트스트랩 차이가 줄어듭니다.
- **검증된 마이그레이션 상태 (Verified migration state)** — 테넌트를 완료로 처리하기 전에 사용자 프로비저닝, 데이터베이스 쓰기 권한, 큐 드레인 상태 및 레거시 테이블 부재 여부를 확인합니다.
- **더 안전한 삭제 (Safer teardown)** — 주권 삭제 시 이제 테넌트 자격 증명을 깨끗하게 제거하여 삭제된 테넌트가 오래된 데이터베이스 접근 권한을 남기지 않도록 합니다.

## 주권 테넌트 방문하기

1. **네트워크 관리 > Ultimate Multisite > 사이트(Sites)**로 이동합니다.
2. 해당 주권 테넌트를 선택합니다.
3. 비밀번호를 복사하거나 임시 관리자 계정을 생성하는 대신, 사용 가능한 경우 **방문 (SSO)** 기능을 사용하세요.

방문 흐름은 해당 테넌트에 대한 단기 로그인 토큰을 생성하고 테넌트 감사 추적(audit trail)에 SSO 이벤트를 기록합니다. 버튼이 작동하지 않으면, 해당 테넌트 도메인이 예상되는 설치 경로로 해결되는지 확인하고 테넌트가 네트워크 측 SSO 엔드포인트에 도달할 수 있는지 확인하세요.

## 원격 사이트 운영 체크리스트

주권 또는 원격 테넌트를 변경하기 전에 다음 사항을 확인하세요:

## 주권 테넌트 삭제하기

주권 테넌트를 삭제하는 것은 되돌릴 수 없는 작업입니다. 먼저 백업 및 내보내기 상태를 확인한 후 사이트 관리 화면에서 삭제해야 합니다. 1.2.0의 정리 흐름(teardown flow)은 정리 과정의 일부로 테넌트 데이터베이스 자격 증명을 제거하지만, 외부 호스팅 패널을 사용하는 경우 호스트 수준의 데이터베이스 사용자 및 폴더가 완전히 제거되었는지 관리자가 직접 확인해야 합니다.

:::warning
마이그레이션 검증이 실행 중이거나 비동기 푸시 작업이 대기 중일 때는 주권 테넌트를 삭제하지 마세요. 정리 과정에서 필요한 자격 증명을 제거할 수 있으므로 검증이 완료될 때까지 기다려주세요.
:::
