# 자주 묻는 질문

다음은 지원 요청 중 자주 발생하는 **자주 묻는 질문**입니다.

  * **Ultimate Multisite란 무엇인가요?**

Ultimate Multisite는 WordPress 멀티사이트 설치용으로 제작된 WordPress 플러그인으로, 귀하의 WordPress 설치를 프리미엄 사이트 네트워크로 변환합니다 – 예를 들어 [WordPress.com](https://WordPress.com)처럼 – 클라이언트가 월간, 분기별 또는 연간 요금으로 사이트를 생성할 수 있도록 합니다 (무료 플랜도 만들 수 있습니다).

  * **Ultimate Multisite가 멀티사이트 없이 동작하나요?**

아니요. Ultimate Multisite는 WordPress 멀티사이트, 서브디렉터리, 또는 서브도메인이 필요합니다.

  * **Ultimate Multisite의 요구 사항은 무엇인가요?**

Ultimate Multisite는 다음을 필요로 합니다:

**WordPress Version** : v5.3+ (권장: 최신 안정 버전)  
**PHP Version** : 7.4.x (8.0 이상 지원 예정)  
**MySQL Version** : v5+ (권장: 버전 5.6, 호스팅 제공업체에서 8.0을 사용할 수 없을 경우)

_Multisite (서브도메인 또는 서브디렉터리)도 활성화되어야 합니다._ [WordPress 멀티사이트 설치 방법](/docs/user-guide/getting-started/how-to-install-wordpress-multisite).

  * **Ultimate Multisite가 서브디렉터리 멀티사이트와 함께 동작하나요?**

네. Ultimate Multisite는 서브디렉터리 설치와 서브디렉터리 설치를 모두 지원합니다.

  * **Ultimate Multisite가 서브도메인 멀티사이트와 함께 동작하나요?**

네. Ultimate Multisite는 서브디렉터리 설치와 서브도메인 설치를 모두 지원합니다.

  * **Ultimate Multisite를 사용해 사용자가 사용할 수 있는 플러그인과 테마를 제한할 수 있나요?**

네. Ultimate Multisite를 사용하면 네트워크의 각 플랜에 사용 가능한 플러그인과 테마를 선택할 수 있습니다.

  * **Ultimate Multisite를 사용해 사용자가 자신의 사이트에서 만들 수 있는 게시물 수를 제한할 수 있나요?**

네. Ultimate Multisite를 사용하면 네트워크의 **모든 사용 가능한 게시물 유형**에 대해 게시물 할당량을 설정할 수 있습니다. 또한 관리자가 초대할 수 있는 사용자 수, **디스크 공간 양**, 그리고 더 많은 항목을 제한할 수 있습니다.

  * **Ultimate Multisite가 사이트 템플릿을 지원하나요?**

네. Ultimate Multisite를 사용하면 원하는 만큼 많은 사이트 템플릿을 만들 수 있습니다. 사이트 템플릿은 테마가 아니라 네트워크의 일반 사이트입니다. 플러그인, 테마를 활성화하고 원하는 대로 사이트 템플릿을 커스터마이즈할 수 있습니다. 사용자가 해당 템플릿을 선택하면, 그 사용자의 사이트가 해당 템플릿의 복사본으로 생성되며 모든 커스터마이징이 적용됩니다.

또한 특정 플랜에 대해 특정 템플릿을 설정할 수 있습니다. 그렇게 하면 해당 플랜에 가입한 모든 사용자는 그 사이트가 해당 사이트 템플릿의 복제본으로 생성됩니다.

  * **Ultimate Multisite가 WooCommerce와 통합되나요?**

네!

네트워크 사용자가 WooCommerce를 사용할 수 있도록 하고, 매장을 설정할 수 있게 하려면 Ultimate Multisite를 사용하면 가능합니다. 또한 각 플랜 단계에서 고객이 만들 수 있는 제품 수를 제한할 수도 있습니다.

Ultimate Multisite의 구독 결제와 관련하여, [Ultimate Multisite: WooCommerce Integration Add-on](https://ultimatemultisite.com/addons/wp-ultimo-woocommerce/)을 사용하면 메인 사이트에서 WooCommerce 게이트웨이를 통해 Ultimate Multisite 클라이언트가 결제할 수 있습니다. 이렇게 하면 WooCommerce용으로 개발된 방대한 결제 게이트웨이 라이브러리를 활용해 Ultimate Multisite 클라이언트에게 청구할 수 있습니다.

  * **WPMUDEV Pro Sites에서 구독을 마이그레이션할 수 있는 방법이 있나요?**

네, Ultimate Multisite는 이제 [Pro Sites 네트워크를 마이그레이션할 수 있는 유용한 도구](https://ultimatemultisite.com/addons/wp-ultimo-pro-sites-migrator/)를 제공합니다.

  * **내 플랜 편집 페이지에서 “Allowed Plugins” 탭이 비어 있는 이유는 무엇인가요?**

Ultimate Multisite는 네트워크에서 네트워크 활성화되지 않은 플러그인에 대해 허용하거나 거부하도록만 허용합니다. 플러그인이 네트워크 활성화되면, 본질적으로 네트워크의 모든 사이트에서 기본적으로 로드됩니다.

“Allowed Plugins” 탭에 플러그인이 표시되지 않으면, 원하는 플러그인이 네트워크 활성화되어 있는지 확인하세요.

  * **내 자체 가격표 디자인을 사용할 수 있나요?**

네. 버전 1.4.0부터 Ultimate Multisite는 템플릿 오버라이드를 지원합니다.

  * **Ultimate Multisite가 WP Engine과 함께 동작하나요?**

네. Ultimate Multisite는 항상 WP Engine에서 지원되었으며, 도메인 매핑을 작동시키려면 관리자가 수동으로 추가 단계(도메인을 WP Engine 패널에 추가)를 수행해야 했습니다. **버전 1.6.0 이후에는 더 이상 그렇지 않습니다.** Ultimate Multisite는 WP Engine API와 직접 통합되며, 매핑 후 도메인이 패널에 자동으로 추가됩니다.

  * **어떤 호스팅 제공업체를 추천하시나요?**

Ultimate Multisite는 모든 호스팅 제공업체에서 작동해야 합니다. 차이점은 Closte, Serverpilot, WPMUDEV, WP Engine, Cloudways, Gridpane 및 [Runcloud.io](http://Runcloud.io)와 같은 일부 호스팅 회사가 도메인 매핑 기능에 대해 더 긴밀한 통합을 제공하므로 바로 사용할 수 있다는 점입니다. 다른 호스팅의 경우 도메인 매핑을 수동으로 설정해야 할 수 있습니다.
