# 사용자 피드백 및 문제 신고

Gratis AI Agent v1.5.0에는 사용자가 도움이 되지 않는 답변을 표시하고 채팅 인터페이스에서 직접 문제를 신고할 수 있는 내장 피드백 및 문제 신고 시스템이 추가되었습니다. 이 시스템에는 동의 관리, 자동 신고 명령어, 그리고 백엔드에서의 AI 기반 분류(triage) 기능이 포함되어 있습니다.

## 👎 (싫어요) 버튼 {#thumbs-down-button}

AI 비서가 전송하는 모든 메시지에는 **👎 (싫어요)** 버튼이 표시됩니다. 사용자가 이 버튼을 클릭하면 해당 답변을 도움이 되지 않거나 부정확하다고 표시할 수 있습니다.

- 이 버튼은 각 비서 메시지 옆에 마우스를 올리면 나타납니다.
- 클릭하면 **피드백 동의 모달**이 나타납니다.
- 피드백은 대화 스레드와 특정 메시지에 연결됩니다.

## 피드백 동의 모달 {#feedback-consent-modal}

사용자가 👎 버튼을 클릭하면 데이터가 전송되기 전에 동의 모달이 나타납니다. 이 모달은 다음 기능을 수행합니다.

- 어떤 정보가 수집될지 설명합니다 (대화 발췌 내용, 브라우저 컨텍스트 등).
- 사용자에게 이 데이터를 공유하는 것에 동의하는지 확인하도록 요청합니다.
- 사용자가 문제가 무엇이었는지 설명할 수 있는 자유 텍스트 필드를 제공합니다.
- 제출하지 않고 닫을 수 있는 **취소** 옵션을 제공합니다.

사용자가 명시적으로 확인하기 전까지는 어떠한 피드백도 기록되지 않습니다.

## 자동 프롬프트 피드백 배너 {#auto-prompt-feedback-banner}

대화가 끝날 무렵, 비서가 **자동 프롬프트 피드백 배너**를 표시할 수 있습니다. 이는 세션이 도움이 되었는지 묻는 방해되지 않는 메시지입니다.

이 배너는 대화 길이와 결과에 대한 휴리스틱(경향성)을 기반으로 자동으로 나타납니다. 👎 버튼과 동일한 피드백 흐름으로 연결됩니다. 사용자는 피드백을 제공하지 않고도 이 배너를 닫을 수 있습니다.

## /report-issue 슬래시 명령어 {#report-issue-slash-command}

사용자는 채팅 입력창에 `/report-issue`를 입력하여 피드백 흐름을 직접 시작할 수 있습니다. 이 명령어는 다음을 수행합니다.

- 즉시 피드백 동의 모달을 엽니다.
- 현재 대화에 대한 컨텍스트로 설명 필드를 미리 채웁니다.
- 사용자가 제출하기 전에 추가 세부 정보를 추가할 수 있도록 합니다.

`/report-issue` 명령어는 모든 채팅 모드(인라인, 플로팅 위젯, 전체 화면)에서 사용할 수 있습니다.

## AI 기반 분류(Triage) {#ai-assisted-triage}

제출된 피드백은 AI 분류 계층으로 라우팅되어 다음 작업을 수행합니다.

- 보고서를 분류합니다 (사실 오류, 도움이 되지 않는 답변, 부적절한 콘텐츠 등).
- 관련 대화 메타데이터를 첨부합니다.
- 분류 요약을 설정된 **피드백 엔드포인트**로 전달합니다 ( [설정 > 고급](#settings-advanced) 참조).

이를 통해 가장 중요한 문제들을 먼저 파악하여 수동 검토 시간을 줄여줍니다.

## 설정 > 고급 {#settings-advanced}

피드백 백엔드를 활성화하려면 **Gratis AI Agent → 설정 → 고급**에서 다음 필드를 구성해야 합니다.

| 필드 | 설명 |
|---|---|
| **Feedback Endpoint URL** | 피드백 페이로드(JSON)를 POST 요청으로 받는 URL입니다. |
| **Feedback API Key** | 피드백 제출을 인증하기 위해 `Authorization` 헤더에 전송되는 Bearer 토큰입니다. |

두 필드를 모두 비워두면 피드백 수집 기능이 비활성화됩니다. 개별 피드백 버튼은 사용자에게 계속 표시되지만, 제출된 내용은 전송되지 않습니다.

:::tip
피드백 엔드포인트는 최소한 `message_id`, `conversation_id`, `feedback_text`, `triage_category` 필드를 가진 JSON 본문을 수락해야 합니다. 예상 스키마는 엔드포인트 제공업체의 문서를 참조하세요.
:::
