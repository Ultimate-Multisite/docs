# 다른 플러그인과의 알려진 호환성 문제

Ultimate Multisite를 설치하고 활성화한 후, 다른 플러그인과의 해결되지 않은 호환성 문제로 인해 문제가 발생할 수 있습니다.

우리는 매번 새로운 릴리스마다 이 목록을 줄이기 위해 적극적으로 노력하고 있지만, 여기 나열된 플러그인이 있다면 완전하게 기능하는 Ultimate Multisite Network를 위해 해당 플러그인을 비활성화해야 할 수 있습니다.

다음은 해당 플러그인 목록과 이러한 경우에 수행해야 할 작업입니다:

**Some WPMU DEV plugins** WPMU DEV에서 더 이상 유지보수되지 않는 일부 플러그인은 Ultimate Multisite 등록 흐름이 마지막 단계에서 중단되는 원인으로 알려져 있습니다 (사용자가 계정을 만들 수 없습니다). 다음은 이러한 문제를 일으키는 플러그인 목록입니다:

  * Signup Language: _Ultimate Multisite에는 동일한 기능을 제공하는 부가 기능이 있으며, 이름은 Ultimate Multisite: Language Switcher입니다._

  * Set Blog Description on Blog Creation: _Ultimate Multisite에 내장되어 있으므로 추가 플러그인이 필요하지 않습니다._

  * New Blog Templates: _Ultimate Multisite에 내장되어 있으므로 추가 플러그인이 필요하지 않습니다._

  * Set Password on Multisite Blog Creation: _Ultimate Multisite에 내장되어 있으므로 추가 플러그인이 필요하지 않습니다._

  * Signup TOS: _Ultimate Multisite에 내장되어 있으므로 추가 플러그인이 필요하지 않습니다._

  * Anti-Splog: 비록 덜 강력하지만, 이것도 _Ultimate Multisite에 내장되어 있습니다._

**Note** : 다른 모든 WPMU Dev 플러그인은 Ultimate Multisite와 함께 정상적으로 사용할 수 있습니다. 예시로는 _Smush_ , _Forminator_ , _Defender,_ 등입니다.
