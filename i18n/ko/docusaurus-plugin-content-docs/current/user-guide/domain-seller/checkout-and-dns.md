# 체크아웃 필드 및 고객 DNS 관리

## 도메인 선택 체크아웃 필드

**도메인 선택(Domain Selection)** 필드는 고객이 사이트 도메인을 어떻게 얻을지 선택할 수 있게 해주는 체크아웃 요소입니다. 이 필드를 체크아웃 양식에 추가하여 도메인 판매 기능을 활성화할 수 있습니다.

### 체크아웃 양식에 필드 추가하기

1. **네트워크 관리자(Network Admin) › Ultimate Multisite › 체크아웃 양식(Checkout Forms)**으로 이동합니다.
2. 체크아웃 양식을 열거나 새로 만듭니다.
3. 체크아웃 편집기에서 **필드 추가(Add Field)**를 클릭합니다.
4. 필드 목록에서 **도메인 선택(Domain Selection)**을 선택합니다.
5. 필드 옵션을 구성합니다 (아래 참조).
6. 양식을 저장합니다.

### 필드 옵션

**도메인 모드(Domain modes)** — 고객에게 어떤 탭을 보여줄지 선택합니다. 각 모드는 독립적으로 활성화하거나 비활성화할 수 있습니다.

| 모드 | 기능 |
|---|---|
| **서브도메인(Subdomain)** | 고객이 네트워크 내의 무료 서브도메인(예: `mysite.yournetwork.com`)을 사용합니다. 결제가 필요 없습니다. |
| **새 도메인 등록(Register New Domain)** | 고객이 새로운 도메인을 검색하고 설정된 제공업체를 통해 등록합니다. 가격 책정 시 일치하는 도메인 상품을 사용합니다. |
| **기존 도메인(Existing Domain)** | 고객이 이미 소유한 도메인을 연결(매핑)합니다. 등록 비용이 들지 않습니다. 도메인은 자동으로 고객의 사이트에 매핑됩니다. |

**기본 모드(Default mode)** — 세 가지 모드 모두가 활성화된 경우, 어떤 탭이 먼저 열릴지 결정합니다. 도메인 등록을 선택 사항으로 유지하려면 **서브도메인**으로 설정하고, 구매를 장려하려면 **새 도메인 등록**으로 설정합니다.

**도메인 상품(Domain product)** — 이 필드를 특정 도메인 상품에 선택적으로 고정할 수 있습니다. 설정하지 않은 경우, 애드온이 고객이 검색한 TLD를 기반으로 일치하는 상품을 자동으로 선택합니다.

### 등록자 연락처 필드

고객이 **새 도메인 등록(Register New Domain)** 탭을 선택하면, 체크아웃 양식에 등록자 연락처 필드가 인라인으로 추가됩니다.

- 이름 / 성
- 이메일 주소
- 주소 (1줄, 시, 주/도, 우편번호, 국가)
- 전화번호

이 필드들은 모든 등록 기관에서 필수이며, 등록 API 호출이 이루어지기 전에 유효성 검사를 거칩니다. 전화번호는 등록 기관에서 요구하는 `+CC.NNN` 국제 형식으로 자동 포맷됩니다.

### 자동 생성 사이트 URL

고객이 도메인을 등록하거나 매핑할 때, 사이트 URL 필드는 선택된 도메인으로부터 자동으로 채워집니다. 고객이 별도의 URL 필드를 채울 필요가 없습니다.

### 검색 동작 방식

- 고객이 입력하는 동안 AJAX를 사용하여 도메인 사용 가능 여부를 실시간으로 확인합니다.
- 선호하는 도메인이 사용 불가능할 경우 대체 TLD 제안을 보여줍니다.
- 가격은 실시간으로 가져와 명확하게 표시됩니다 (등록 가격, 갱신 가격, 선택적 WHOIS 개인 정보 보호 비용).
- 쿠폰 코드는 다른 모든 상품과 마찬가지로 도메인 상품에도 적용됩니다.

**검색 응답성 조정:**

```php
// 느린 연결에서 API 호출을 줄이려면 디바운스 지연 시간(밀리초)을 늘립니다.
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // 기본값: 500
});
```

**도메인 검색 양식에 사용자 지정 필드 추가:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => '추가 메모',
    ];
    return $fields;
});
```

## 고객 DNS 관리

고객은 **마이 계정(My Account)** 페이지에서 자신의 도메인 항목 아래에 있는 곳에서 등록된 도메인의 DNS 레코드를 관리할 수 있습니다.

### 지원되는 레코드 유형

| 유형 | 용도 |
|---|---|
| **A** | 호스트 이름을 IPv4 주소에 매핑합니다. |
| **AAAA** | 호스트 이름을 IPv6 주소에 매핑합니다. |
| **CNAME** | 다른 호스트를 가리키는 별칭을 만듭니다. |
| **MX** | 메일 교환 서버를 설정합니다. |
| **TXT** | SPF, DMARC, 인증 또는 기타 텍스트 레코드를 추가합니다. |

### DNS 관리를 지원하는 제공업체는?

DNS 관리(레코드 추가, 편집, 삭제)는 **OpenSRS**, **ResellerClub**, **Enom**에서 사용할 수 있습니다. Namecheap, GoDaddy, NameSilo 도메인은 상태 및 만료 정보를 표시하지만, DNS는 등록 기관의 제어판에서 직접 관리해야 합니다.

### 기본 DNS 레코드

도메인이 등록될 때 자동으로 적용되는 기본 DNS 레코드를 구성할 수 있습니다. **설정(Settings) › 도메인 판매자(Domain Seller) › 기본 DNS 레코드(Default DNS Records)**로 이동하세요.

기본 레코드 값은 두 가지 토큰을 지원합니다.

| 토큰 | 대체되는 값 |
|---|---|
| `{DOMAIN}` | 등록된 도메인 이름 (예: `example.com`) |
| `{SITE_URL}` | 고객 사이트의 WordPress 사이트 URL |

**예시 — 에이펙스 도메인과 www를 서버 IP에 연결:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### 관리자: DNS 보기 및 편집

네트워크 관리자는 **네트워크 관리자(Network Admin) › Ultimate Multisite › 도메인(Domains)**에서 해당 도메인의 편집 페이지를 통해 모든 고객 도메인의 DNS 레코드를 보고 편집할 수 있습니다.
