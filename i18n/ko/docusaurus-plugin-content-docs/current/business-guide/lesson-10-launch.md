# 레슨 10: 런칭 당일

모든 것이 완성되었습니다. 문을 열기 전에, 이 레슨에서는 아무것도 고장 나거나, 빠지거나, 민망한 부분이 없는지 확인하는 사전 런칭 체크리스트를 안내합니다.

## 지금까지 진행한 내용

FitSite는 템플릿, 플랜, 결제, 브랜딩, 온보딩, 가격 책정까지 모든 것이 설정되었습니다. 이제 모든 것이 제대로 작동하는지 확인하고 정식으로 서비스를 시작할 차례입니다.

## 사전 런칭 체크리스트

모든 항목을 순서대로 확인하세요. 건너뛰는 항목은 없어야 합니다.

### 플랫폼 인프라

- [ ] 호스팅이 안정적이며 부하가 걸려도 성능이 좋은지 확인
- [ ] 와일드카드 SSL이 활성화되었고 모든 서브도메인이 HTTPS로 서비스되는지 확인
- [ ] 도메인 매핑이 제대로 작동하는지 확인 -- 사이트 생성 및 커스텀 도메인 매핑 테스트
- [ ] 백업이 설정되고 테스트되었는지 확인 (최소한 하나는 복원하여 검증)
- [ ] 모니터링 시스템이 갖춰져 있는지 확인 -- 플랫폼 장애 발생 시 알 수 있어야 함

### 템플릿

- [ ] 세 가지 템플릿 모두 새 사이트에 올바르게 로드되는지 확인
- [ ] 자리 표시자 콘텐츠가 유용하고 오타가 없는지 확인
- [ ] 모든 이미지가 적절하게 라이선스가 부여되었는지 확인 (스톡 사진 워터마크 없음)
- [ ] 모든 템플릿 페이지에서 모바일 반응형 디자인이 작동하는지 확인
- [ ] 페이지 로드 속도가 허용 가능한지 확인 (GTmetrix 또는 PageSpeed Insights 같은 도구로 테스트)
- [ ] 모든 템플릿에 깨진 링크나 누락된 에셋이 없는지 확인

### 플랜 및 제품

- [ ] 세 가지 플랜 모두 활성화되어 보이고 있는지 확인
- [ ] 플랜 설명이 정확하고 틈새 시장에 특화되어 있는지 확인
- [ ] 가격 책정이 정확한지 확인 (월간 및 연간)
- [ ] 설정 비용이 적절한 플랜에 구성되었는지 확인
- [ ] 스타터 플랜의 체험 기간이 제대로 작동하는지 확인
- [ ] 결제 과정 중 오더 범프(Order bumps)가 올바르게 표시되는지 확인
- [ ] 플랜별로 플러그인 및 테마 제한이 정확하게 적용되는지 확인

### 결제 흐름 (Checkout Flow)

- [ ] 각 플랜별로 전체 테스트 가입을 완료 (테스트 결제 모드 사용)
- [ ] 템플릿 선택 시 플랜별로 올바른 템플릿이 표시되는지 확인
- [ ] 결제가 성공적으로 처리되는지 확인
- [ ] 가입 후 고객이 환영 이메일을 받는지 확인
- [ ] 올바른 템플릿으로 새 사이트가 생성되는지 확인
- [ ] 고객이 즉시 새 사이트에 로그인할 수 있는지 확인
- [ ] 할인 코드가 올바르게 작동하는지 확인

### 브랜딩

- [ ] 로그인 페이지에 FitSite 브랜딩이 표시되는지 확인
- [ ] 관리자 대시보드에 FitSite 브랜딩이 표시되는지 확인
- [ ] 모든 시스템 이메일에 FitSite 브랜딩과 피트니스 관련 언어가 사용되는지 확인
- [ ] 청구서에 비즈니스 상세 정보와 함께 올바르게 표시되는지 확인
- [ ] 마케팅 사이트가 활성화되어 있고 결제 양식으로 연결되는지 확인

### 온보딩

- [ ] 신규 고객 대시보드에 Quick Start 위젯이 나타나는지 확인
- [ ] 모든 Quick Start 링크가 올바른 페이지를 가리키는지 확인
- [ ] 환영 이메일 시퀀스가 구성되고 테스트되었는지 확인
- [ ] 지식 기반(Knowledge base) 아티클이 게시되고 접근 가능한지 확인
- [ ] 계정 페이지에 올바른 플랜 정보와 업그레이드 옵션이 표시되는지 확인

### 법률 및 비즈니스

- [ ] 서비스 약관이 게시되고 결제 과정에서 링크되는지 확인
- [ ] 개인정보 보호정책이 게시되고 접근 가능한지 확인
- [ ] 환불 정책이 정의되고 문서화되었는지 확인
- [ ] 결제 수령을 위한 비즈니스 법인 등록이 완료되었는지 확인
- [ ] 결제 게이트웨이가 라이브 모드(테스트 모드 아님)인지 확인
- [ ] 세금 설정이 관할 지역에 맞게 정확한지 확인

## 소프트 런칭 vs. 하드 런칭

두 단계의 런칭을 고려해 보세요.

### 1단계: 소프트 런칭 (Soft Launch)

공개 런칭 전에 5~10명의 피트니스 스튜디오 사장님을 초대하여 가입하도록 합니다. 이들이 베타 고객입니다. 이들에게 다음을 제공하는 대가로 상당한 할인(평생 50% 할인 또는 3개월 무료)을 제공하세요.

- 가입 및 온보딩 경험에 대한 솔직한 피드백
- 그들의 사이트를 쇼케이스 예시로 사용할 허가
- 발견한 버그나 문제 보고

이를 통해 대중에게 공개하기 전에 실제 고객 피드백과 쇼케이스할 라이브 사이트를 얻을 수 있습니다.

### 2단계: 공개 런칭 (Public Launch)

소프트 런칭 피드백을 반영한 후:

- 결제 게이트웨이를 라이브 모드로 전환
- 마케팅 사이트를 게시
- 고객 확보 활동 시작 (레슨 11)
- 관련 피트니스 산업 채널에 공지

## 런칭 당일 조치 사항

대중에게 공개하는 날:

1. **라이브 결제로 전환** -- 결제 게이트웨이의 테스트 모드를 비활성화합니다.
2. **다시 한번 검증** -- 실제 결제를 사용하여 전체 테스트 가입을 진행합니다 (나중에 환불하세요).
3. **면밀히 모니터링** -- 오류, 가입 실패, 결제 문제 등을 주시합니다.
4. **대기 상태 유지** -- 첫 실제 고객들이 도움이 필요할 수 있으며, 빠른 응대는 신뢰를 쌓습니다.
5. **잠시 축하** -- 그리고 다시 업무에 복귀합니다.

## 문제가 생길 수 있는 경우

다음 사항에 대비하세요:

- **결제 게이트웨이 문제**: 게이트웨이 제공업체의 지원팀 연락처를 준비해 두세요.
- **SSL 인증서 문제**: 인증서를 확인하고 갱신하는 방법을 알아두세요.
- **이메일 전송 실패**: 이메일이 실제로 도착하는지 테스트하세요 (스팸 폴더 확인).
- **부하에 따른 성능 저하**: 트래픽이 급증할 경우, 호스팅을 확장하는 방법을 알아두세요.
- **고객 혼란**: 지식 기반과 지원 채널을 준비해 두세요.

## 이번 레슨에서 배운 것

- 인프라, 콘텐츠, 결제, 법률을 다루는 포괄적인 사전 런칭 체크리스트
