# Multi-Tenancy {#multi-tenancy}

Разпределяйте подсайтове между сървъри и бази данни за производителност и сигурност

## Акценти от изданието версия 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 добавя основата за суверенни наематели за работни процеси с изолация в продукционна среда:

- Stateless автоматично влизане за наематели с краткотрайни, защитени от повторно използване SSO посещения.
- Изолация на база данни и файлова система за всеки подсайт за суверенни наематели.
- Възможност за многонаемност на Hostinger за създаване на наематели с отчитане на хоста.
- Контролни точки за проверка на суверенна миграция за проверки на наследени данни и асинхронна push обработка.
- Конфигурируемо обвързване на хост на база данни на същата машина за инсталации на Bedrock, FrankenPHP и localhost.
- Подобрения в началното зареждане на наемател, записващия компонент за базата данни, началното зареждане на схемата и създаването на потребители.
- По-безопасно премахване на суверенен наемател, което изчиства идентификационните данни на наемателя.

Вижте ръководствата за потребители за [работен процес за суверенна миграция](/user-guide/getting-started/sovereign-migration-workflow), [изолация при многонаемност](/user-guide/configuration/multi-tenancy-isolation) и [операции със суверенни наематели](/user-guide/administration/sovereign-tenant-operations).

## Изисквания {#requirements}

- WordPress 5.3 или по-нова версия
- PHP 7.4 или по-нова версия
- Плъгин Multisite Ultimate (активен)

## Инсталация {#installation}

1. Качете файловете на добавката във вашата директория `/wp-content/plugins/`
2. Активирайте плъгина чрез менюто „Plugins“ в WordPress
3. Конфигурирайте настройките на добавката в мрежовата администрация

## Разработка {#development}

### Настройка {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Тестване {#testing}

Добавката включва цялостна рамка за тестване:

- **Unit тестове**: Тестове, базирани на PHPUnit, с тестовия пакет на WordPress
- **Базов тестов клас**: Осигурява помощни методи за често срещани тестови сценарии
- **Покритие на тестовете**: Конфигурирано за проследяване на покритието на кода
- **Тестване на Multisite**: Тестовете се изпълняват в среда с multisite

### Стандарти за код {#code-standards}

- **Стандарти за кодиране на WordPress**: Налагат се чрез PHPCS
- **Съвместимост с PHP 7.4+**: Поддържат се съвременни PHP функционалности
- **Статичен анализ**: Интеграция с PHPStan за проверка на типове
- **Модернизация на кода**: Правила на Rector за надграждания на PHP

### Система за билдване {#build-system}

- **Обработка на ресурси**: Минификация за CSS/JS файлове
- **Превод**: Генериране на POT файл
- **Пакетиране**: Създаване на архив за разпространение
- **Режим за разработка**: Неминифицирани ресурси за дебъгване

## Конфигурация {#configuration}

Добавката поддържа различни опции за конфигурация, достъпни чрез интерфейса за мрежова администрация.

### Конфигурация на суверенен наемател {#sovereign-tenant-configuration}

За суверенни наематели конфигурирайте базата данни на наемателя, корена на файловата система, обвързването на хоста на базата данни и възможността на хоста, преди да насочите трафика към наемателя. Изпълнявайте проверка на миграцията след всяка промяна, която засяга идентификационните данни за базата данни, обвързването на хоста, маршрутизацията или създаването на потребители.

Използвайте `localhost` само когато средата за изпълнение на наемателя и правата за базата данни очакват локален сокет или достъп на същата машина. Използвайте `127.0.0.1` или име на хост на услуга, когато това е хостът за права, конфигуриран от вашия доставчик.

### Възможност на Hostinger {#hostinger-capability}

Възможността за многонаемност на Hostinger работи със споделения интеграционен токен на Hostinger. Потвърдете, че домейнът присъства в hPanel, DNS сочи към Hostinger акаунта и настройките на базата данни на наемателя съвпадат с hPanel преди проверката.

## Hooks и филтри {#hooks-and-filters}

### Действия {#actions}

- `ultimate-multisite-addon-template_init` - Задейства се след инициализацията на добавката
- `ultimate-multisite-addon-template_loaded` - Задейства се след зареждането на всички файлове на добавката

### Филтри {#filters}

- `ultimate-multisite-addon-template_settings` - Променя настройките на добавката
- `ultimate-multisite-addon-template_enabled` - Замества състоянието за активиране/деактивиране на добавката
