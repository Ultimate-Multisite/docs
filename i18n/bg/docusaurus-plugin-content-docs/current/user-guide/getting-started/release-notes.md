# Бележки към изданията {#release-notes}

## Версия 2.13.0 — Издадена на 2026-06-05 {#version-2130--released-on-2026-06-05}

- Ново: Добавена е поддръжка за sovereign-tenant за потоци за customer Account, checkout, billing, сайт, фактура, превключване на шаблони и domain mapping, така че мрежите на наематели да могат да насочват клиентите обратно към основния сайт за управлявани действия.
- Ново: Добавени са проверки на данни за подновяване за recurring membership-и, така че gateway-ите да могат да деактивират автоматичното подновяване, когато липсва запазено споразумение за billing, subscription или vault token.
- Ново: Добавено е HMAC-verified loopback публикуване за предстоящо създаване на сайт, за да се направи provisioning-ът от checkout към сайт по-надежден при хостове, където background jobs се забавят.
- Ново: Добавени са разширителни точки за разработчици за SSO URL-и, базови домейни на checkout формуляри и автоматично създаване на domain record-и.
- Поправка: SSO е по-надежден при mapped domains, анонимни посещения чрез broker, logout и конфликти на зависимости между plugin-и.
- Поправка: Предстоящото създаване на сайт вече се възстановява от остарели publish flags и избягва клиентите да остават блокирани на екрана за създаване на сайт.
- Поправка: Domain records вече не се създават за споделени базови домейни на checkout формуляри, а неизползваните background jobs на host provider се пропускат, когато няма активна интеграция.
- Поправка: Крайни случаи при checkout, адрес за billing, нулиране на парола, имейл верификация, превключване на шаблони, tours и customer dashboard вече не блокират нормалните клиентски потоци.
- Поправка: Broadcast имейлите вече запазват получателите поверителни, като същевременно избягват фатални SMTP/plugin грешки, когато списъци с получатели или mail transports се провалят.
- Поправка: Крайни случаи при подновявания на membership, показване на изтичане и събиране на плащания вече избягват незабавни изтичания, сривове или пропуснати задължителни плащания.
- Подобрено: Съвместимостта с WordPress е тествана до 7.0, production Vue assets са rebuild-нати от npm sources, а Cypress end-to-end покритието вече упражнява повече потоци за checkout, setup, SSO и gateway.

## Версия 2.12.0 — Издадена на 2026-05-15 {#version-2120--released-on-2026-05-15}

- Ново: Добавен е Hostinger (hPanel) като поддържан host provider с интеграция за domain mapping
- Ново: Site Exporter вече обработва network import bundles за рационализирано възстановяване на сайтове в цялата мрежа
- Поправка: BCC broadcast имейлите вече използват undisclosed-recipients header, за да предотвратят разкриване на адресите на получателите
- Поправка: Датата на изтичане на membership вече не се поврежда при записване със стойност, която не е дата
- Поправка: Stripe membership updates вече правилно изчистват отстъпките, без да извикват остарелия deleteDiscount API
- Поправка: SSO redirects на сайтове с domain mapping вече са ограничени, за да се предотвратят безкрайни redirect loops
- Поправка: Изборът в image picker на setup wizard вече правилно актуализира основния data model
- Поправка: Site Exporter CLI вече запазва правилния default network site selection
- Подобрено: Премахнат е включеният wp-cli от пакета на plugin-а, което намалява размера на plugin-а

## Версия 2.11.0 — Издадена на 2026-05-11 {#version-2110--released-on-2026-05-11}

- Ново: Експортите на сайт вече включват self-booting `index.php`, така че ZIP-ът да може да бъде инсталиран на нов хост без отделна инсталация на plugin.
- Ново: Network export позволява на администраторите да експортират всички подсайтове в един архив от администраторската страница Site Export.
- Ново: Превключвателят Allow Site Templates plan вече се прилага чрез fallback chain, като правилно ограничава наличността на шаблони според лимитите на plan-а.
- Ново: Редакторът на checkout формуляри предупреждава, когато product е добавен без конфигурирано задължително поле.
- Ново: Разделът за настройки Import/Export вече ясно описва своя обхват и води директно към инструмента Site Export.

## Версия 2.10.0 — Издадена на 2026-05-05 {#version-2100--released-on-2026-05-05}

- Ново: PayPal guided setup wizard за ръчно въвеждане на credentials с OAuth flag gate за безпроблемна конфигурация на gateway.
- Ново: Клиентският панел за превключване на шаблон е преработен с карта на текущия шаблон, постоянна grid подредба и бутон **Нулиране на текущия шаблон**.
- Поправка: Превключването на шаблон вече не блокира UI при AJAX грешка.
- Поправка: Състоянията на разрешенията за превключване на шаблон са защитени срещу неоторизиран достъп.
- Поправка: Полетата за override на сайт се валидират преди записване.
- Поправка: Подканата за адрес за billing вече се показва, когато адресът е празен.
- Поправка: PHP 8.1 null-to-string deprecation notices са разрешени.
- Поправка: Currents се lazy-load-ват преди init hook, за да се предотвратят timing issues.
- Поправка: Филтрираният SSO path се спазва във всички login flows.
- Поправка: Празните опции за идентичност на сайт се запазват при запис.

- Поправка: Pending sites вече се показват първи в изгледа All Sites.
- Поправка: Добавен е User-Agent header за screenshot provider (mShots), за да се предотвратят 403 грешки.

- Поправка: Tour IDs са нормализирани до underscores в keys на потребителските настройки.
- Подобрено: ZipArchive вече се използва вместо Alchemy/Zippy за по-добра съвместимост.
