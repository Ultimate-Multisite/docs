# Процес на запознаване с Theme Builder

Superdav AI Agent v1.12.0 въвежда ръководен **процес на запознаване с Theme Builder**, който ви помага да създадете персонализирана блокова тема по време на първоначалната настройка. Това замества остарелия режим Site Builder с по-гъвкав подход, асистиран от agent.

## Какво е Процесът на запознаване с Theme Builder?

Процесът на запознаване с Theme Builder е интерактивен помощник за настройка, който:

- Ви ръководи през дизайнерски решения (цветове, типография, разположение)
- Улавя предпочитанията ви за визуална идентичност на сайта
- Генерира персонализирана блокова тема, създадена специално за нуждите ви
- Автоматично активира темата при завършване

Този процес работи с **Setup Assistant agent**, който задава уточняващи въпроси и изгражда темата ви стъпка по стъпка.

## Започване на Процеса на запознаване с Theme Builder

### Настройка при първоначалното използване

Когато за първи път стартирате Superdav AI Agent на нова WordPress инсталация, ще видите:

```
Welcome to Superdav AI Agent!

What would you like to do?
1. Build a custom theme (Theme Builder)
2. Use an existing theme
3. Skip setup for now
```

Изберете **"Build a custom theme"**, за да влезете в процеса на запознаване с Theme Builder.

### Ръчна активация

Можете също да започнете процеса на запознаване с Theme Builder по всяко време, като поискате:

```
"Start the Theme Builder onboarding"
```

или

```
"Help me create a custom theme"
```

## Стъпките на запознаването

### Стъпка 1: Избор на режим

Setup Assistant agent ви пита за предпочитанията ви:

```
How would you like to build your theme?
- Guided (I'll ask questions and build it for you)
- Hands-on (I'll show you options and you decide)
```

**Guided mode** (Ръководен режим) се препоръчва за повечето потребители; agent прави дизайнерски препоръки въз основа на индустрията и целите ви.

### Стъпка 2: Спецификация на сайта

Ще бъдете попитани за сайта си:

- **Site purpose**: E-commerce, блог, портфолио, SaaS и др.
- **Target audience**: Кои са вашите посетители?
- **Brand colors**: Основни и второстепенни цветове (или "Не съм сигурен")
- **Tone**: Професионален, креативен, забавен, минималистичен и др.

Тази информация се съхранява в паметта **site_brief** на сайта ви, която agent използва в бъдещи сесии.

### Стъпка 3: Решения за дизайн система

Agent ви ръководи през избора на дизайнерски токени:

- **Typography**: Семейство на шрифта (serif, sans-serif, monospace) и мащаб на размера
- **Color palette**: Основен, второстепенен, акцентен и неутрален цветове
- **Spacing**: Компактни, нормални или просторни разположения
- **Motion**: Анимации и преходи (ако е желателно)

### Стъпка 4: Генериране на темата

Setup Assistant agent създава структурата на вашата персонализирана блокова тема, включваща:

- `theme.json`, съдържащ всички ваши дизайнерски токени
- Блокови шаблони за често срещани разположения (головна страница, блог, контакт)
- Персонализирани стилове на блокове, съответстващи на дизайн системата ви
- Метаданни на темата и декларации за поддръжка на WordPress

### Стъпка 5: Активация и верификация

Темата се активира автоматично и ще видите:

```
✓ Your custom theme is now live!
  Theme name: [Your Site Name] Theme
  Colors: [Primary] / [Secondary]
  Typography: [Font Family]

  Visit your site to see the new design.
```

След това можете да посетите сайта си, за да проверите дали темата се показва правилно.

## Спецификация на сайта и паметта site_brief

По време на процеса на запознаване, agent улавя спецификацията на сайта ви в категория **site_brief** памет. Тя включва:

- Цел и цели на сайта
- Целева аудитория
- Цветове и тон на марката
- Дизайнерски предпочитания
- Структура на съдържанието

### Защо е важна site_brief

В бъдещи сесии, agent използва site_brief, за да:

- Поддържа дизайнерска консистентност при промени
- Предлага функции, съобразени с целта на сайта ви
- Осигурява препоръки, базирани на контекста
- Избягва повтарянето на въпроси за настройка

### Преглед на site_brief

Можете да попитате agent:

```
"Show me my site brief"
```

или

```
"What do you know about my site?"
```

Agent ще покаже съхранената спецификация на сайта ви.

## Персонализиране след запознаването

След завършване на процеса на запознаване с Theme Builder, можете да:

### Използвате Design System Aesthetics Skill

Поискайте финиране на дизайна:

```
"Refine the typography to be more modern"
```

или

```
"Adjust the color palette to be warmer"
```

**Design System Aesthetics skill** ви ръководи през целенасочени дизайнерски актуализации.

### Редактирайте theme.json директно

За напреднали потребители, редактирайте `/wp-content/themes/[theme-name]/theme.json`, за да настроите:

- Цветови токени
- Мащаби на типографията
- Стойности за разстояние
- Дефиниции на рамки и сенки

### Създайте персонализирани блокови шаблони

Използвайте блоковия редактор на WordPress, за да създадете персонализирани шаблони за:

- Разположения на началната страница
- Страници с публикации в блога
- Продуктови страници
- Контактни форми

## Сравнение: Старо срещу Ново запознаване

| Feature | Site Builder (Legacy) | Theme Builder (New) |
|---------|----------------------|-------------------|
| Setup method | Wizard-based form | Agent-guided conversation |
| Theme generation | Limited templates | Custom scaffolding |
| Design tokens | Manual entry | Guided decisions |
| Flexibility | Fixed options | Customizable |
| Future updates | Not referenced | Stored in site_brief |

## Отстраняване на неизправности

**Процесът на запознаване не завърши**
- Рестартирайте процеса: "Start the Theme Builder onboarding"
- Проверете дали вашата WordPress инсталация е актуална
- Потвърдете, че Setup Assistant agent е активиран

**site_brief ми не се използва**
- Потвърдете, че agent има достъп до паметта
- Попитайте agent да "recall my site brief"
- Проверете дали паметта е активирана в настройките ви

**Генерираната тема не съответства на предпочитанията ми**
- Използвайте Design System Aesthetics skill, за да я финизирате
- Попитайте agent да "regenerate the theme with [specific changes]"
- Редактирайте theme.json директно за прецизен контрол

## Следващи стъпки

След завършване на процеса на запознаване с Theme Builder:

1. **Проверете сайта си**: Посетете сайта, за да видите новата тема
2. **Доработете дизайна**: Използвайте Design System Aesthetics skill за корекции
3. **Добавете съдържание**: Започнете да изграждате съдържанието на сайта си
4. **Изследвайте възможности**: Научете за други agent възможности като scaffold-block-theme и activate-theme
