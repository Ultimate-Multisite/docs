# Domain Seller: Настройка и конфигурация на доставчик {#domain-seller-setup-and-provider-configuration}

Добавката Domain Seller идва с воден съветник за настройка, който ви превежда през всяка задължителна стъпка. Тази страница обхваща потока на съветника и как да конфигурирате или преконфигурирате доставчици след това.

## Изисквания {#requirements}

- **Multisite Ultimate** v2.4.12 или по-нова, активирана в мрежата
- **PHP** 7.4+
- API удостоверения за поне един поддържан регистратор

## Съветник за първоначална настройка {#first-run-setup-wizard}

Съветникът за настройка се стартира автоматично първия път, когато активирате plugin в мрежата. Той също е достъпен по всяко време от **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Стъпка 1 — Изберете доставчик {#step-1--choose-a-provider}

Изберете регистратора, който искате да свържете. Поддържани опции:

| Доставчик | Управление на DNS | WHOIS поверителност |
|---|---|---|
| OpenSRS | Да | Да |
| Namecheap | Не | Да (WhoisGuard, безплатно) |
| HostAfrica | Да | Да (защита на ID) |
| Openprovider | Да | Да |
| Hostinger | Чрез основното Hostinger съпоставяне на домейни за хоствани домейни | Да |
| GoDaddy | Не | Не |
| ResellerClub | Да | Не |
| NameSilo | Не | Не |
| Enom | Да | Не |

### Стъпка 2 — Въведете удостоверения {#step-2--enter-credentials}

Всеки доставчик има различни полета за удостоверения:

**OpenSRS** — Потребителско име и частен ключ (от OpenSRS Reseller Control Panel)

**Namecheap** — Потребителско име и API ключ (от Account › Tools › API Access)

**HostAfrica** — Domains Reseller API endpoint и удостоверения от HostAfrica reseller модула. Понастоящем не е документиран отделен sandbox endpoint; тествайте с безопасни проверки само за четене, преди да изпълнявате реални регистрации.

**Openprovider** — Потребителско име и парола с активиран API достъп. Незадължителният sandbox режим използва Openprovider sandbox API, а незадължителен клиентски handle по подразбиране може да се използва повторно за регистрации.

**Hostinger** — Споделеният Hostinger hPanel API token от основната Hostinger интеграция. Същият token захранва основното съпоставяне на домейни и операциите по регистрация на Domain Seller.

**GoDaddy** — API ключ и secret (от developer.godaddy.com)

**ResellerClub** — Reseller ID и API ключ (от ResellerClub control panel)

**NameSilo** — API ключ (от namesilo.com › Account › API Manager)

**Enom** — Account ID и API token

Отметнете **Sandbox mode**, когато е налично, за да тествате спрямо тестовата среда на доставчика, преди да преминете на живо.

### Стъпка 3 — Тествайте връзката {#step-3--test-the-connection}

Щракнете върху **Test Connection**. Съветникът изпраща лека API заявка, за да провери удостоверенията и свързаността. Поправете всички проблеми с удостоверенията, преди да продължите.

### Стъпка 4 — Импортирайте TLDs {#step-4--import-tlds}

Щракнете върху **Import TLDs**, за да изтеглите всички налични TLDs и цени на едро от свързания доставчик. Това попълва списъка с TLD, използван от домейн продуктите. Импортът може да отнеме 30–60 секунди за доставчици с големи каталози с TLD.

TLDs също се синхронизират отново автоматично веднъж дневно чрез планирана cron задача.

### Стъпка 5 — Създайте домейн продукт {#step-5--create-a-domain-product}

Съветникът създава домейн продукт по подразбиране, обхващащ всички случаи, с 10% надценка. Можете да редактирате този продукт веднага или да пропуснете и да създадете продукти ръчно под **Ultimate Multisite › Products**.

Вижте [Домейн продукти и ценообразуване](./domain-products) за пълното ръководство за конфигурация на продукт.

## Преконфигуриране на доставчик {#reconfiguring-a-provider}

Отидете на **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или щракнете върху **Settings** в списъка с plugin).

Страницата с настройки съдържа:

- **Enable domain selling** — включване/изключване на цялата функционалност
- **Default provider** — доставчикът, използван за търсения на домейни и нови продукти
- **Max TLDs per search** — колко TLDs да се проверяват, когато клиент търси; по-високите стойности показват повече опции, но са по-бавни
- **Availability cache duration** — колко дълго да се кешират резултатите за наличност и цени; по-ниските стойности са по-точни, но увеличават API заявките
- **Manage domain products** — бърза връзка към списъка Products
- **Configure providers** — отваря Integration Wizard за добавяне или преконфигуриране на доставчици

### Добавяне на втори доставчик {#adding-a-second-provider}

Щракнете върху **Configure providers** и стартирайте съветника отново за новия регистратор. Можете да имате няколко доставчици, конфигурирани едновременно. Задайте всеки домейн продукт към конкретен доставчик или го оставете на настройката по подразбиране.

### Ръчно синхронизиране на TLDs {#syncing-tlds-manually}

На страницата с настройки щракнете върху **Sync TLDs** до който и да е конфигуриран доставчик, за да изтеглите най-новите цени. Това е полезно, след като доставчик актуализира цените на едро или добави нови TLDs.

## Журнали {#logs}

Всеки доставчик записва в собствен канал за журнал. Журналите могат да се преглеждат под **Network Admin › Ultimate Multisite › Logs**:

| Канал на журнала | Съдържание |
|---|---|
| `domain-seller-registration` | Всички опити за регистрация (успешни и неуспешни) |
| `domain-seller-renewal` | Резултати от задачата за подновяване |
| `domain-seller-opensrs` | Сурова OpenSRS API активност |
| `domain-seller-namecheap` | Сурова Namecheap API активност |
| `domain-seller-hostafrica` | Сурова HostAfrica API активност |
| `domain-seller-openprovider` | Сурова Openprovider API активност |
| `domain-seller-hostinger` | Сурова Hostinger API активност |
| `domain-seller-godaddy` | Сурова GoDaddy API активност |
| `domain-seller-resellerclub` | Сурова ResellerClub API активност |
| `domain-seller-namesilo` | Сурова NameSilo API активност |
| `domain-seller-enom` | Сурова Enom API активност |

## Бележки за възможностите на доставчиците {#provider-capability-notes}

Не всеки API на регистратор предоставя едни и същи операции. Добавката показва неподдържаните операции с ясни грешки, предназначени за администратори, вместо да се проваля безшумно.

- **HostAfrica** поддържа най-широкия live reseller работен процес, включително търсене, синхронизиране на TLD/цени, регистрация, подновяване, трансфер, актуализации на nameserver, DNS записи, EPP кодове, registrar lock и ID защита.
- **Openprovider** поддържа синхронизиране на TLD с reseller цени, регистрация, подновяване, трансфери, актуализации на nameserver, DNS зони, EPP кодове, registrar lock и WHOIS поверителност. Удостоверява се с краткосрочен bearer token, който добавката обновява автоматично.
- **Hostinger** поддържа търсене за наличност, регистрация, преглед на портфолио, актуализации на nameserver, registrar lock и WHOIS поверителност чрез споделения hPanel API токен. Публичният Domains API на Hostinger не предоставя reseller/wholesale ценообразуване, входящ трансфер, изрично подновяване или извличане на EPP код; подновяванията са само с автоматично подновяване.
