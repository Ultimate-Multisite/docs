# Domain Seller: Настройка и Конфигурация на Поставчика

Addon за Domain Seller идва с ръководен инсталационен асистент, който ви превежда през всяка необходима стъпка. Тази страница обхваща работния процес на асистента и как да конфигурирате или преконфигурирате доставчиците след това.

## Изисквания

- **Multisite Ultimate** v2.4.12 или по-висок, активиран на ниво мрежа (network-activated)
- **PHP** 7.4+
- API данни за потребителски имена за поне един поддържан регистратор

## Инсталационен асистент при първоначалното пускане

Асистентът се стартира автоматично, когато активирате плагинато на ниво мрежа за първи път. Той е достъпен по всяко време и от **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Стъпка 1 — Избор на доставчик

Изберете регистратора, с който искате да се свържете. Поддържани опции:

| Provider | DNS management | WHOIS privacy |
|---|---|---|
| OpenSRS | Yes | Yes |
| Namecheap | No | Yes (WhoisGuard, free) |
| GoDaddy | No | No |
| ResellerClub | Yes | No |
| NameSilo | No | No |
| Enom | Yes | No |

### Стъпка 2 — Въвеждане на данни за вход

Всеки доставчик има различни полета за данни за вход:

**OpenSRS** — Потребителско име и личен ключ (от OpenSRS Reseller Control Panel)

**Namecheap** — Потребителско име и API ключ (от Account › Tools › API Access)

**GoDaddy** — API ключ и секретен ключ (от developer.godaddy.com)

**ResellerClub** — Reseller ID и API ключ (от контролния панел на ResellerClub)

**NameSilo** — API ключ (от namesilo.com › Account › API Manager)

**Enom** — Account ID и API токен

Проверете **Sandbox mode**, когато е налично, за тестване срещу тестовата среда на доставчика преди пускане в експлоатация.

### Стъпка 3 — Тестване на връзката

Кликнете върху **Test Connection**. Асистентът изпраща лек API заявка, за да провери данните за вход и свързаността. Поправете всички проблеми с данните за вход, преди да продължите.

### Стъпка 4 — Импортиране на TLDs

Кликнете върху **Import TLDs**, за да изтеглите всички налични TLDs и търговски цени от свързания доставчик. Това попълва списъка с TLDs, използван от продуктите за домейни. Импортирането може да отнеме 30–60 секунди за доставчици с голям каталог TLDs.

TLDs също се синхронизират автоматично веднъж дневно чрез планиран cron job.

### Стъпка 5 — Създаване на продукт за домейни

Асистентът създава по подразбиране продукт за домейни "catch-all" с марж от 10%. Можете да редактирате този продукт веднага или да пропуснете и да създадете продукти ръчно в **Ultimate Multisite › Products**.

Вижте [Domain Products and Pricing](./domain-products) за пълното ръководство за конфигурация на продуктите.

## Преконфигуриране на доставчик

Отидете на **Network Admin › Ultimate Multisite › Settings › Domain Seller** (или кликнете върху **Settings** в списъка с плагини).

Страницата за настройки съдържа:

- **Enable domain selling** — превключва цялата функция във включено/изключено
- **Default provider** — доставчикът, използван за търсене на домейни и нови продукти
- **Max TLDs per search** — колко TLDs да се проверят, когато клиент търси; по-високите стойности показват повече опции, но са по-бавни
- **Availability cache duration** — колко дълго да се кешират резултатите от наличност и цени; по-ниските стойности са по-точни, но увеличават API заявките
- **Manage domain products** — бърза връзка към списъка с продукти
- **Configure providers** — отваря Интеграционния асистент (Integration Wizard) за добавяне или преконфигуриране на доставчици

### Добавяне на втори доставчик

Кликнете върху **Configure providers** и пуснете асистента отново за новия регистратор. Можете да имате конфигурирани множество доставчици едновременно. Назначете всеки продукт за домейни на конкретен доставчик или го оставете на дефолтен.

### Ръчна синхронизация на TLDs

На страницата за настройки, кликнете върху **Sync TLDs** до всеки конфигуриран доставчик, за да изтеглите най-актуалните цени. Това е полезно, когато доставчикът обнови търговските цени или добави нови TLDs.

## Лог файлове (Logs)

Всеки доставчик пише във своя собствен лог канал. Лог файловете са достъпни под **Network Admin › Ultimate Multisite › Logs**:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | Всички опити за регистрация (успешни и неуспешни) |
| `domain-seller-renewal` | Резултати от работата по обновяване |
| `domain-seller-opensrs` | Сурови API дейности на OpenSRS |
| `domain-seller-namecheap` | Сурови API дейности на Namecheap |
| `domain-seller-godaddy` | Сурови API дейности на GoDaddy |
| `domain-seller-resellerclub` | Сурови API дейности на ResellerClub |
| `domain-seller-namesilo` | Сурови API дейности на NameSilo |
| `domain-seller-enom` | Сурови API дейности на Enom |
