# Управление на полета за домейн и DNS на клиентите

## Полето за избор на домейн

Полето **Domain Selection** (Избор на домейн) е елемент в чекаута, който дава на клиентите избор как да получат домейна за своя сайт. Добавете го към всяка форма за чекаута, за да активирате продажбата на домейни.

### Добавяне на полето към форма за чекаута

1. Отидете на **Network Admin › Ultimate Multisite › Checkout Forms**
2. Отворете или създайте форма за чекаута
3. В редактора на чекаута кликнете върху **Add Field** (Добави поле)
4. Изберете **Domain Selection** от списъка с полета
5. Настройте опциите на полето (вижте по-долу)
6. Запазете формата

### Опции на полето

**Domain modes** (Режими за домейн) — Изберете кои табове ще виждат клиентите. Всеки режим може да бъде активиран или деактивиран независимо:

| Режим | Какво прави |
|---|---|
| **Subdomain** | Клиентът използва безплатен поддомейн в вашата мрежа (напр. `mysite.yournetwork.com`). Не е нужна плащане. |
| **Register New Domain** | Клиентът търси нов домейн и го регистрира чрез вашия конфигуриран доставчик. Използва се продукта за съответния домейн за ценообразуване. |
| **Existing Domain** | Клиентът свързва домейн, който вече притежава. Няма такса за регистрация. Домейнът автоматично се свързва със сайта му. |

**Default mode** (По подразбиране режим) — Кой таб се отваря първи, когато всички три режима са активирани. Настройте го на **Subdomain**, за да оставите регистрацията на домейн необвързана с плащане, или на **Register New Domain**, за да стимулирате покупки.

**Domain product** (Продукт за домейн) — По избор, свържете това поле към конкретен продукт за домейн. Ако не е зададено, добавката автоматично избира съответния продукт въз основа на TLD, който клиентът търси.

### Полета за контакт на регистранта

Когато клиентът избере таба **Register New Domain**, формата за чекаута добавя полета за контакт на регистранта в линия:

- Име / Фамилия
- Имейл адрес
- Адрес (линия 1, град, държава/провинция, пощенски код, държава)
- Телефонен номер

Те са задължителни за всички регистратори и се валидират преди да бъде изпратен API заявка за регистрация. Телефонните номера автоматично се форматират в международния формат `+CC.NNN`, който очакванят регистраторите.

### Автоматично генериран URL на сайта

Когато клиентът регистрира или свързва домейн, полето за URL на сайта се попълва автоматично от избрания домейн. Клиентите не се нуждаят от попълване на отделно поле за URL.

### Поведение при търсене

- Наличността на домейн се проверява в реално време с AJAX, докато клиентът пише
- Алтернативни предложения за TLD се показват, когато предпочитаният домейн не е наличен
- Цените се извличат на живо и се показват ясно (цена за регистрация, цена за подновяване, опционална такса за конфиденциалност WHOIS)
- Купонните кодове се прилагат към продукти за домейни по същия начин като към всеки друг продукт

**Настройка на отзивчивостта на търсенето:**

```php
// Увеличете закъснението на debounce (милисекунди), за да намалите API заявките при бавни връзки
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Добавяне на персонализирани полета към формата за търсене на домейн:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Допълнителни бележки',
    ];
    return $fields;
});
```

## Управление на DNS на клиентите

Клиентите могат да управляват DNS записи за регистрираните си домейни от страницата **My Account** (Моят профил), в секцията на домейна си.

### Поддържани типове записи

| Тип | Използване |
|---|---|
| **A** | Свързва име на домейн към IPv4 адрес |
| **AAAA** | Свързва име на домейн към IPv6 адрес |
| **CNAME** | Създава псевдоним, насочващ към друг име на домейн |
| **MX** | Задава сървър за измяна на поща (mail exchange server) |
| **TXT** | Добавя SPF, DMARC, верификация или други текстови записи |

### Кои доставчици поддържат управление на DNS?

Управлението на DNS (добавяне, редактиране, изтриване на записи) е налично с **OpenSRS**, **ResellerClub** и **Enom**. Домейните от Namecheap, GoDaddy и NameSilo показват статуса и информация за изтичане, но DNS трябва да се управлява директно в контролния панел на регистратора.

### По подразбиране DNS записи

Можете да конфигурирате записи за DNS по подразбиране, които се прилагат автоматично, когато домейнът бъде регистриран. Отидете на **Settings › Domain Seller › Default DNS Records**.

Стойностите на записи по подразбиране поддържат два токена:

| Токен | Замества се с |
|---|---|
| `{DOMAIN}` | Регистрираното име на домейна (напр. `example.com`) |
| `{SITE_URL}` | URL на сайта на клиента в WordPress |

**Пример — насочване на основния домейн и www към IP адреса на вашия сървър:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: преглед и редактиране на DNS

Администраторите на мрежата могат да преглеждат и редактират DNS записи за всеки домейн на клиент от страницата за редактиране на домейна в **Network Admin › Ultimate Multisite › Domains**.
