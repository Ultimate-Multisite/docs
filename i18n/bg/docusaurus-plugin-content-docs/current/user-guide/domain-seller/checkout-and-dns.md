# Поле за плащане и управление на DNS на клиентите

## Полето за плащане „Избор на домейн“ {#the-domain-selection-checkout-field}

Полето **Избор на домейн** е елемент за плащане, който дава на клиентите избор как да получат домейна на своя сайт. Добавете го към всеки формуляр за плащане, за да активирате продажбата на домейни.

### Добавяне на полето към формуляр за плащане {#adding-the-field-to-a-checkout-form}

1. Отидете на **Администриране на мрежата › Ultimate Multisite › Формуляри за плащане**
2. Отворете или създайте формуляр за плащане
3. В редактора за плащане щракнете върху **Добавяне на поле**
4. Изберете **Избор на домейн** от списъка с полета
5. Конфигурирайте опциите на полето (вижте по-долу)
6. Запазете формуляра

### Опции на полето {#field-options}

**Режими на домейн** — Изберете кои табове вижда клиентът. Всеки режим може да бъде активиран или деактивиран независимо:

| Режим | Какво прави |
|---|---|
| **Поддомейн** | Клиентът използва безплатен поддомейн във вашата мрежа (напр. `mysite.yournetwork.com`). Не е необходимо плащане. |
| **Регистриране на нов домейн** | Клиентът търси нов домейн и го регистрира чрез конфигурирания от вас доставчик. Използва съответния домейн продукт за ценообразуване. |
| **Съществуващ домейн** | Клиентът свързва домейн, който вече притежава. Няма такса за регистрация. Домейнът се свързва автоматично към неговия сайт. |

**Режим по подразбиране** — Когато и трите режима са активирани, кой таб се отваря първи. Задайте **Поддомейн**, за да запазите регистрацията на домейн като незадължителна, или **Регистриране на нов домейн**, за да насърчите покупките.

**Домейн продукт** — По желание фиксирайте това поле към конкретен домейн продукт. Ако не е зададено, разширението автоматично избира съответния продукт въз основа на TLD, който клиентът търси.

### Полета за контакт на регистранта {#registrant-contact-fields}

Когато клиент избере таба **Регистриране на нов домейн**, формулярът за плащане добавя полета за контакт на регистранта директно във формуляра:

- Име / Фамилия
- Имейл адрес
- Адрес (ред 1, град, щат/провинция, пощенски код, държава)
- Телефонен номер

Те се изискват от всички регистратори и се валидират, преди да бъде направено извикването към API за регистрация. Телефонните номера се форматират автоматично в международния формат `+CC.NNN`, очакван от регистраторите.

### Автоматично генериран URL на сайта {#auto-generated-site-url}

Когато клиент регистрира или свърже домейн, полето за URL на сайта се попълва автоматично от избрания домейн. Клиентите не трябва да попълват отделно поле за URL.

### Поведение при търсене {#search-behaviour}

- Наличността на домейна се проверява в реално време с AJAX, докато клиентът пише
- Показват се алтернативни предложения за TLD, когато предпочитаният домейн не е наличен
- Цените се извличат на живо и се показват ясно (цена за регистрация, цена за подновяване, незадължителна такса за WHOIS поверителност)
- Кодовете за отстъпка се прилагат към домейн продукти по същия начин, както към всеки друг продукт

**Настройване на отзивчивостта на търсенето:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Добавяне на персонализирани полета към формуляра за търсене на домейн:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Управление на DNS от клиентите {#customer-dns-management}

Клиентите могат да управляват DNS записи за регистрираните си домейни от страницата **Моят Account**, под записа на своя домейн.

### Поддържани типове записи {#supported-record-types}

| Тип | Употреба |
|---|---|
| **A** | Свързва име на хост с IPv4 адрес |
| **AAAA** | Свързва име на хост с IPv6 адрес |
| **CNAME** | Създава псевдоним, сочещ към друго име на хост |
| **MX** | Задава сървър за обмен на поща |
| **TXT** | Добавя SPF, DMARC, верификационни или други текстови записи |

### Кои доставчици поддържат управление на DNS? {#which-providers-support-dns-management}

Управлението на DNS (добавяне, редактиране, изтриване на записи) е налично с **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** и **Openprovider**. Домейните на **Hostinger** могат да актуализират nameservers чрез Domain Seller; DNS записите за хоствани домейни се управляват от основната интеграция на Hostinger за свързване на домейни. Домейните на Namecheap, GoDaddy и NameSilo показват информация за статус и изтичане, но DNS трябва да се управлява директно в контролния панел на регистратора.

### DNS записи по подразбиране {#default-dns-records}

Можете да конфигурирате DNS записи по подразбиране, които се прилагат автоматично, когато домейн бъде регистриран. Отидете на **Настройки › Domain Seller › DNS записи по подразбиране**.

Стойностите на записите по подразбиране поддържат два токена:

| Токен | Заменя се с |
|---|---|
| `{DOMAIN}` | Името на регистрирания домейн (напр. `example.com`) |
| `{SITE_URL}` | URL адресът на WordPress сайта за сайта на клиента |

**Пример — насочване на apex домейна и www към IP адреса на вашия сървър:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Администратор: преглед и редактиране на DNS {#admin-viewing-and-editing-dns}

Мрежовите администратори могат да преглеждат и редактират DNS записи за всеки клиентски домейн от страницата за редактиране на домейна в **Администриране на мрежата › Ultimate Multisite › Домейни**.
