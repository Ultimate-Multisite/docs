# Интеграция с WP Engine

## Преглед {#overview}
WP Engine е премиум управлявана хостинг платформа за WordPress, която осигурява оптимизирана производителност, сигурност и мащабируемост за сайтове, работещи с WordPress. Тази интеграция позволява автоматично синхронизиране на домейн имената между Ultimate Multisite и WP Engine.

## Характеристики {#features}
- Автоматично синхронизиране на домейн имената
- Поддръжка на поддомейни за мултисайт инсталации
- Лесно свързване с вече съществуващите системи на WP Engine

## Изисквания {#requirements}
Интеграцията автоматично открива дали хоствате на WP Engine и използва вградения WP Engine API. Не е необходима допълнителна конфигурация, ако plugin на WP Engine е активен и правилно настроен.

Въпреки това, ако трябва ръчно да настроите интеграцията, можете да дефинирате една от тези константи във вашия файл `wp-config.php`:

```php
define('WPE_APIKEY', 'your_api_key'); // Предпочитан метод
// ИЛИ
define('WPE_API', 'your_api_key'); // Алтернативен метод
```

## Инструкции за настройка {#setup-instructions}

### 1. Проверете plugin на WP Engine {#1-verify-wp-engine-plugin}

Ако хоствате на WP Engine, plugin на WP Engine вече трябва да е инсталиран и активиран. Проверете дали:

1. Plugin на WP Engine е активен
2. Файлът `wp-content/mu-plugins/wpengine-common/class-wpeapi.php` съществува

### 2. Активирайте интеграцията {#2-enable-the-integration}

1. В административния панел на WordPress си, отидете на Ultimate Multisite > Settings
2. Навигирайте до раздела "Domain Mapping"
3. Извишете до "Host Integrations"
4. Активирайте интеграцията за WP Engine
5. Натиснете "Save Changes"

## Как работи {#how-it-works}

### Синхронизиране на домейн имената {#domain-syncing}

Когато домейн име е мапиран в Ultimate Multisite:

1. Интеграцията използва WP Engine API, за да добави домейна към вашата инсталация на WP Engine
2. WP Engine управлява конфигурацията на домейна и издаването на SSL сертификат
3. Когато мапирането на домейн бъде премахнато, интеграцията ще премахне домейна от WP Engine

### Поддръжка на поддомейни {#subdomain-support}

За мултисайт инсталации с поддомейни:

1. Интеграцията добавя всеки поддомейн към WP Engine, когато се създава нов сайт
2. WP Engine управлява конфигурацията на поддомейна
3. Когато сайтът бъде изтрит, интеграцията ще премахне поддомейна от WP Engine

## Важни бележки {#important-notes}

### Wildcard домейн имена {#wildcard-domains}
За мултисайт инсталации с поддомейни, се препоръчва да се свържете с поддръжката на WP Engine, за да поискате конфигурация на wildcard домейн. Това позволява всички поддомейни да работят автоматично, без да е необходимо да добавяте всеки един поотделно.

### SSL сертификати {#ssl-certificates}
WP Engine автоматично управлява издаването и обновяването на SSL сертификати за всички домейни, добавени чрез тази интеграция. Не е необходима допълнителна конфигурация.

## Отстраняване на неизправности {#troubleshooting}

### Проблеми с връзката към API {#api-connection-issues}
- Проверете дали plugin на WP Engine е активен и правилно настроен
- Ако сте дефинирали ръчно API ключ, проверете дали е коректен
- Свържете се с поддръжката на WP Engine, ако имате проблеми с API

### Домейнът не е добавен {#domain-not-added}
- Проверете лог файловете на Ultimate Multisite за съобщения за грешки
- Проверете дали домейнът вече не е добавен към WP Engine
- Уверете се, че планът ви за WP Engine поддържа броя домейни, които добавяте

### Проблеми с поддомейни {#subdomain-issues}
- Ако поддомейните не работят, свържете се с поддръжката на WP Engine, за да поискате конфигурация на wildcard домейн
- Проверете дали настройките на DNS са правилно конфигурирани за основния домейн и поддомейните
