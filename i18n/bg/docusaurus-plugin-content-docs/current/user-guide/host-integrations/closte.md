# Интеграция с Closte

## Преглед
Closte е управлявана платформа за хостинг на WordPress, работеща върху инфраструктура на Google Cloud. Тази интеграция позволява автоматично синхронизиране на домейни и управление на SSL сертификати между Ultimate Multisite и Closte.

## Характеристики
- Автоматично синхронизиране на домейни
- Управление на SSL сертификати
- Поддръжка на Wildcard домейни
- Не е необходима конфигурация, ако работите с Closte

## Изисквания
Следващата константа трябва да бъде дефинирана във файла `wp-config.php`, ако използвате Closte:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Тази константа обикновено вече е дефинирана, ако хоствате с Closte.

## Инструкции за настройка

### 1. Проверете API ключа на Closte
Ако хоствате с Closte, константата `CLOSTE_CLIENT_API_KEY` вече трябва да е дефинирана във файла `wp-config.php`. Можете да проверите това, като прегледате файла си `wp-config.php`.

### 2. Активирайте интеграцията

1. В административния панел на WordPress, отидете на Ultimate Multisite > Settings
2. Навигирайте до раздела "Domain Mapping"
3. Извишете до "Host Integrations"
4. Активирайте интеграцията Closte
5. Натиснете "Save Changes"

## Как работи
Когато домейн бъде мапиран в Ultimate Multisite:

1. Интеграцията изпраща заявка към API на Closte, за да добави домейна към вашето приложение
2. Closte автоматично управлява предоставянето на SSL сертификати
3. Когато мапирането на домейн бъде премахнато, интеграцията ще премахне домейна от Closte

Интеграцията също работи с настройката на интервал за проверка на DNS в Ultimate Multisite, което ви позволява да конфигурирате колко често системата проверява разпространението на DNS и издаването на SSL сертификати.

## Създаване на домейн записи
Тази интеграция гарантира, че когато сайт бъде създаден или дублиран, домейн запис се създава автоматично. Това е особено важно за интеграцията с Closte, тъй като създаването на домейн запис активира API на Closte да създаде домейна и SSL сертификата.

## Отстраняване на неизправности

### Проблеми с връзката към API
- Проверете дали API ключът на Closte е коректен
- Уверете се, че акаунтът ви в Closte има необходимите права

### Проблеми с SSL сертификати
- Closte може да отнеме известно време за издаване на SSL сертификати (обикновено 5-10 минути)
- Проверете дали домейните ви показват правилно към IP адреса на сървъра ви в Closte
- Проверете DNS записите за домейна, за да се уверите, че са правилно конфигурирани

### Домейнът не е добавен
- Проверете лог файловете на Ultimate Multisite за съобщения за грешки
- Уверете се, че домейнът вече не е добавен към Closte
- Уверете се, че DNS записите на домейна ви са правилно конфигурирани

### Интервал за проверка на DNS
- Ако SSL сертификатите отнемат твърде много време за издаване, можете да коригирате интервала за проверка на DNS в настройките Domain Mapping
- По подразбиране интервалът е 300 секунди (5 минути), но можете да го зададете дори на 10 секунди за по-бърза проверка по време на тестване
