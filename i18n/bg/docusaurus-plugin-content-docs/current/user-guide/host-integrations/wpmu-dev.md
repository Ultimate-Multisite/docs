# Интеграция с WPMU DEV

## Преглед {#overview}
WPMU DEV е цялостна платформа за WordPress, която предлага хостинг, плагини и услуги за сайтове, работещи с WordPress. Тази интеграция позволява автоматично синхронизиране на домейни и управление на SSL сертификати между Ultimate Multisite и хостинга на WPMU DEV.

## Характеристики {#features}
- Автоматично синхронизиране на домейни
- Управление на SSL сертификати
- Разширени опити за проверка на SSL сертификати

## Изисквания {#requirements}
Интеграцията автоматично проверява дали хоствате на WPMU DEV и използва вградения API. Ако хоствате на WPMU DEV, не е необходима допълнителна конфигурация.

Интеграцията проверява за наличието на константата `WPMUDEV_HOSTING_SITE_ID`, която се дефинира автоматично, когато хоствате на WPMU DEV.

## Инструкции за настройка {#setup-instructions}

### 1. Проверка на хостинга на WPMU DEV {#1-verify-wpmu-dev-hosting}

Ако хоствате на WPMU DEV, необходимите константи вече трябва да са дефинирани. Проверете дали:

1. Константата `WPMUDEV_HOSTING_SITE_ID` е дефинирана във вашето потребителско пространство (environment)
2. Имате активна членска подписка за WPMU DEV с достъп до API

### 2. Активиране на интеграцията {#2-enable-the-integration}

1. В административния панел на WordPress си отидете на Ultimate Multisite > Settings
2. Навигирайте до раздела "Domain Mapping"
3. Извишете до "Host Integrations"
4. Активирайте интеграцията с WPMU DEV
5. Натиснете "Save Changes"

## Как работи {#how-it-works}

### Синхронизиране на домейни {#domain-syncing}

Когато домейн е мапиран в Ultimate Multisite:

1. Интеграцията използва WPMU DEV API, за да добави домейна към вашия хостинг акаунт
2. Автоматично добавя и www версията на домейна
3. WPMU DEV управлява конфигурацията на домейна и издаването на SSL сертификата

### Управление на SSL сертификати {#ssl-certificate-management}

Интеграцията е конфигурирана да увеличи броя на опита за проверка на SSL сертификата за хостинг на WPMU DEV, тъй като издаването и инсталирането на SSL сертификати може да отнеме известно време. По подразбиране, тя ще опита до 10 пъти за проверка на SSL сертификата, което е в сравнение със стандартните 5 опита.

## Важни бележки {#important-notes}

### Премахване на домейн {#domain-removal}
В момента WPMU DEV API не предоставя начин за премахване на домейни. Когато мапирането на домейн бъде премахнато в Ultimate Multisite, домейнът ще остане в вашия хостинг акаунт на WPMU DEV. Ако е необходимо, трябва ръчно да го премахнете от панела за управление (dashboard) на WPMU DEV.

### Аутентикация на API {#api-authentication}
Интеграцията използва API ключ на WPMU DEV, който е запазен в базата данни на WordPress ви като опция `wpmudev_apikey`. Това се настройва автоматично, когато свържете сайта си с WPMU DEV.

## Отстраняване на неизправности {#troubleshooting}

### Проблеми с връзката към API {#api-connection-issues}
- Проверете дали сайтът ви е правилно свързан с WPMU DEV
- Проверете дали опцията `wpmudev_apikey` е зададена в базата данни на WordPress
- Уверете се, че членството ви за WPMU DEV е активно

### Проблеми с SSL сертификати {#ssl-certificate-issues}
- WPMU DEV може да отнеме известно време за издаване на SSL сертификати (обикновено 5-15 минути)
- Интеграцията е конфигурирана да проверява до 10 пъти за SSL сертификати
- Ако SSL сертификатите все още не се издават след множество опити, свържете се с поддръжката на WPMU DEV

### Домейнът не е добавен {#domain-not-added}
- Проверете лог файловете на Ultimate Multisite за съобщения за грешки
- Проверете дали домейнът вече не е добавен към WPMU DEV
- Уверете се, че плана за хостинг на WPMU DEV поддържа броя домейни, които добавяте
