# Настройки на Gratis AI Agent

Екранът **Settings → Advanced** в Gratis AI Agent предоставя конфигурация на администраторско ниво за backend интеграции. Тази страница документира препращане на обратна връзка, ключове за доставчици на търсене, настройка на управлявана Superdav услуга, контроли за Google Calendar, SMS настройки за TextBee и feature flags за цялата мрежа.

## Достъп до настройките {#accessing-settings}

1. В WordPress администрацията отидете на **Gratis AI Agent → Settings**.
2. Щракнете върху таба **Advanced**.

## Конфигурация на endpoint за обратна връзка {#feedback-endpoint-configuration}

Endpoint-ът за обратна връзка получава POST заявки от AI агента всеки път, когато потребител изпрати обратна връзка чрез бутона с палец надолу, банера за автоматична подкана или командата `/report-issue`.

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** | URL адресът, който получава изпращанията на обратна връзка като HTTP POST заявки с JSON тяло. |
| **Feedback API Key** | bearer token, изпращан в `Authorization` header на всяка заявка за обратна връзка. Оставете празно, ако вашият endpoint не изисква удостоверяване. |

### Очакван JSON payload {#expected-json-payload}

Вашият endpoint за обратна връзка трябва да приема JSON тяло с поне следните полета:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

В payload-а може да присъстват допълнителни полета в зависимост от контекста на разговора.

### Стойности на `triage_category` {#triagecategory-values}

AI triage слоят присвоява една от следните стойности на `triage_category`, преди да препрати payload-а:

| Стойност | Значение |
|---|---|
| `factual_error` | Асистентът е предоставил неправилна фактическа информация. |
| `unhelpful_answer` | Отговорът е бил технически правилен, но не е бил полезен. |
| `inappropriate_content` | Отговорът е съдържал съдържание, което не трябва да се показва на потребителите. |
| `other` | Обратната връзка не съответства на известна категория. |

### Удостоверяване {#authentication}

Ако вашият endpoint изисква удостоверяване, задайте полето **Feedback API Key** на вашия bearer token. Агентът изпраща:

```
Authorization: Bearer <your-api-key>
```

Ако полето **Feedback API Key** е празно, не се изпраща `Authorization` header.

### Деактивиране на събирането на обратна връзка {#disabling-feedback-collection}

Оставете полетата **Feedback Endpoint URL** и **Feedback API Key** празни. Бутонът с палец надолу и UI за обратна връзка остават видими за потребителите, но изпращанията не се препращат към външна услуга.

## Brave Search API Key {#brave-search-api-key}

Също в таба **Advanced**, полето **Brave Search API Key** активира възможността [Интернет търсене](../configuration/internet-search).

| Поле | Описание |
|---|---|
| **Brave Search API Key** | Вашият API ключ от developer dashboard-а на Brave Search. Изисква се за активиране на интернет търсене в AI асистента. |

Етикетът на полето включва връзка с възможност за щракване към страницата за регистрация за Brave Search API. Оставете празно, за да деактивирате интернет търсенето.

Вижте [Интернет търсене](../configuration/internet-search) за документация за крайни потребители относно тази функция.

## Управлявана Superdav услуга {#managed-superdav-service}

Superdav AI Agent v1.18.0 добавя управлявани Superdav service endpoints и автоматично осигуряване на връзка за поддържани сайтове. Използвайте тези контроли, когато вашият сайт трябва да се свърже с хоствания доставчик вместо с ръчно конфигуриран service endpoint.

| Поле | Описание |
|---|---|
| **Managed Superdav Service** | Активира връзката към хостваната Superdav услуга за поддържани сайтове. |
| **Provision Connection** | Стартира автоматично осигуряване на endpoint и идентификационни данни. Използвайте това, след като потвърдите, че сайтът трябва да използва управлявания доставчик. |
| **Service Endpoint / Connection Status** | Показва текущия endpoint или състоянието на връзката след осигуряване. |

След осигуряване запазете настройките и проверете състоянието на връзката, преди да разчитате на работни процеси с управлявана услуга. Ако осигуряването е неуспешно, прегледайте показаните насоки за повторен опит и потвърдете, че сайтът има разрешение да използва хоствания доставчик.

## Конфигурация на Google Calendar {#google-calendar-configuration}

Когато calendar функциите на Superdav AI Agent v1.18.0 са активирани, агентът може да чете конфигурирани календари и подробности за събития. Calendar инструментите са ориентирани към четене и са полезни за напомняния, съобразени с графика, последваща комуникация с участници и съпоставяне на контакти.

| Поле | Описание |
|---|---|
| **Google Calendar Credentials** | Съхранява идентификационните данни или token връзката, необходими за четене на calendar данни. |
| **Calendar Selection** | Ограничава кои конфигурирани календари агентът може да преглежда. |
| **Calendar Connection Status** | Потвърждава дали текущите идентификационни данни могат да четат календари и събития. |

Ограничете calendar идентификационните данни само до календарите, от които агентът се нуждае. Свържете отново или сменете идентификационните данни, ако състоянието показва изтекъл token.

## TextBee SMS известия {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 добавя TextBee като SMS доставчик за конфигурирани работни процеси за известия. SMS известията трябва да бъдат съчетани с стъпки за човешко одобрение за чувствителни или насочени към потребители съобщения.

| Поле | Описание |
|---|---|
| **TextBee API Key** | Удостоверява заявки към TextBee SMS доставчика. |
| **TextBee Device / Sender** | Избира TextBee подателя или устройството, използвано за изходящи съобщения, когато това се изисква от доставчика. |
| **SMS Notifications Enabled** | Позволява на одобрени работни процеси да изпращат известия чрез текстови съобщения. Оставете деактивирано, за да предотвратите изпращане на SMS. |

Изпратете тестово съобщение само до номер, притежаван от администратор, след което потвърдете поведението на стъпките за одобрение, преди да активирате планирани напомняния или напомняния, насочени към участници.

## Feature Flags {#feature-flags}

Също въведен в v1.9.0, табът **Settings → Feature Flags** предоставя превключватели за незадължителна функционалност. Всеки flag е или активиран, или деактивиран за цялата мрежа; в момента няма override за отделен сайт.

### Достъп до Feature Flags {#accessing-feature-flags}

1. В WordPress администрацията отидете на **Gratis AI Agent → Settings**.
2. Щракнете върху таба **Feature Flags**.

### Flags за контрол на достъпа {#access-control-flags}

| Флаг | По подразбиране | Описание |
|---|---|---|
| **Ограничаване до администратори** | Изключено | Когато е включено, само потребители с ролята `administrator` могат да отварят чат панела на AI Agent. Всички други роли вместо това виждат съобщение „Свържете се с вашия администратор“. |
| **Ограничаване до Network Admins** | Изключено | Когато е включено в multisite мрежа, само Super Admins могат да използват агента. Администраторите на отделни сайтове са блокирани. Има предимство пред „Ограничаване до администратори“, ако и двете са включени. |
| **Разрешаване на достъп за абонати** | Изключено | Когато е включено, потребители с ролята `subscriber` могат да използват чат интерфейса, но са ограничени до възможности само за четене (без създаване на публикации или промени в настройките). |
| **Изключване за нечленове** | Изключено | Интегрира се със статуса на членство в Ultimate Multisite. Когато е включено, чатът е скрит за сайтове, които нямат активно членство. |

### Флагове за брандиране {#branding-flags}

| Флаг | По подразбиране | Описание |
|---|---|---|
| **Скриване на долния текст "Powered by Gratis AI Agent"** | Изключено | Премахва реда за бранд атрибуция, показан в долната част на чат widget-а. Препоръчва се за white-label внедрявания. |
| **Персонализирано име на агента** | *(празно)* | Заменя стандартния етикет "Gratis AI Agent" в заглавката на чата и администраторското меню със собственото ви име на продукт. Оставете празно, за да използвате стандартното. |
| **Скриване на избора на агент** | Изключено | Когато е включено, потребителите не могат да превключват между петте вградени агента. Текущият агент е фиксиран към това, което е конфигурирано като стандартно в Settings → General. |
| **Използване на иконата на сайта като чат аватар** | Изключено | Заменя стандартната AI икона в заглавката на чат widget-а с иконата на WordPress сайта (зададена под Appearance → Customize → Site Identity). |

### Флагове за безопасност на автоматизацията {#automation-safety-flags}

Superdav AI Agent v1.18.0 въвежда портали за човешко одобрение и записи за напомняния за по-безопасни автоматизационни изпълнения. Тези контроли може да се появяват във feature flags или разширените настройки за автоматизация, в зависимост от инсталирания пакет.

| Флаг | По подразбиране | Описание |
|---|---|---|
| **Изискване на човешко одобрение** | Препоръчително включено | Спира на пауза чувствителни автоматизации, докато упълномощен потребител не прегледа и потвърди предложеното действие. |
| **Дедупликация на напомняния** | Включено | Записва изпратените напомняния, така че повторните опити или планираните изпълнения да не изпращат дублирани известия. |
| **Активиране на календарни инструменти** | Изключено до конфигуриране | Позволява на агента да чете конфигурирани Google календари и събития. |
| **Активиране на SMS известия** | Изключено до конфигуриране | Позволява на одобрени работни потоци да изпращат TextBee SMS известия след запазване на идентификационните данни. |

### Прилагане на промени {#applying-changes}

Щракнете върху **Save Settings** след превключване на който и да е флаг. Промените влизат в сила незабавно — не се изисква изчистване на кеша или повторно активиране на plugin.
