# Настройки на AI Агента Gratis

Екранът **Settings → Advanced** в Gratis AI Agent предоставя конфигурация на ниво администратор за интеграции на бекенда, въведени в версия v1.5.0. Тази страница документира полетата на **Feedback Endpoint** и очаквания формат им.

## Достъп до настройките

1. В административната част на WordPress отидете на **Gratis AI Agent → Settings**.
2. Кликнете върху таба **Advanced**.

## Конфигурация на Feedback Endpoint

Feedback endpoint получава POST заявки от AI агента всеки път, когато потребител подава обратна връзка чрез бутона "показалец надолу", банера за автоматични заявки или командата `/report-issue`.

| Поле | Описание |
|---|---|
| **Feedback Endpoint URL** | URL адресът, който получава подадените обратни връзки като HTTP POST заявки с JSON тяло. |
| **Feedback API Key** | Bearer токен, изпратен в заглавието `Authorization` на всяка заявка за обратна връзка. Оставете празно, ако вашият endpoint не изисква удостоверяване. |

### Очакван JSON Payload

Вашият feedback endpoint трябва да приема JSON тяло с поне следните полета:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Отговорът беше грешен относно ценообразуването.",
  "triage_category": "factual_error"
}
```

Могат да бъдат налични допълнителни полета в payload, в зависимост от контекста на разговора.

### Стойности за `triage_category`

Слойът за триаж на AI присвоява една от следните стойности на `triage_category` преди препращането на payload:

| Стойност | Значение |
|---|---|
| `factual_error` | Асистентът е предоставил невярна фактическа информация. |
| `unhelpful_answer` | Отговорът е технически коректен, но не е полезен. |
| `inappropriate_content` | Отговорът съдържа съдържание, което не трябва да се показва на потребителите. |
| `other` | Обратната връзка не съответства на известен клас. |

### Удостоверяване (Authentication)

Ако вашият endpoint изисква удостоверяване, задайте полето **Feedback API Key** на вашия bearer токен. Агентът изпраща:

```
Authorization: Bearer <your-api-key>
```

Ако полето **Feedback API Key** е празно, няма да се изпраща `Authorization` заглавие.

### Деактивиране на събирането на обратна връзка

Оставете полетата **Feedback Endpoint URL** и **Feedback API Key** празни. Бутонът "показалец надолу" и потребителският интерфейс за обратна връзка остават видими за потребителите, но подаденият материал не се препраща към никаква външна услуга.

## Brave Search API Key

Също в таба **Advanced**, полето **Brave Search API Key** активира способността за [Internet Search](../configuration/internet-search).

| Поле | Описание |
|---|---|
| **Brave Search API Key** | Вашият API ключ от Brave Search developer dashboard. Необходим за активиране на интернет търсене в AI асистента. |

Етикетът на полето включва кликателна връзка към страницата за регистрация на Brave Search API. Оставете празно, за да деактивирате интернет търсенето.

Вижте [Internet Search](../configuration/internet-search) за документация за крайните потребители относно тази функция.

## Feature Flags

Също въведени в v1.9.0, табът **Settings → Feature Flags** предоставя превключватели за опционална функционалност. Всеки флаг е или активиран, или деактивиран за цялата мрежа; в момента няма преоразбиране на ниво сайт.

### Достъпни флагове (Access Control Flags)

| Флаг | По подразбиране | Описание |
|---|---|---|
| **Restrict to Administrators** | Off | Когато е активиран, само потребителите с роля `administrator` могат да отварят чат панела на AI Агента. Всички други роли виждат съобщението "Contact your administrator". |
| **Restrict to Network Admins** | Off | Когато е активиран в мултисайт мрежа, само Super Admins могат да използват агента. Индивидуалните администратори на сайтовете са блокирани. При двете настройки да бъдат активирани, тази има преоритет пред "Restrict to Administrators". |
| **Allow Subscriber Access** | Off | Когато е активиран, потребителите с роля `subscriber` могат да използват чат интерфейса, но са ограничени до режим на четене (няма създаване на публикации или промяна на настройките). |
| **Disable for Non-Members** | Off | Интегрира се с статуса на членство на Ultimate Multisite. Когато е активиран, чатът се крие за сайтове, които нямат активно членство. |

### Флагове за брендинг (Branding Flags)

| Флаг | По подразбиране | Описание |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | Премахва линията с атрибуция на марката, показана в долната част на чат виджeта. Препоръчително за white-label внедрявания. |
| **Custom Agent Name** | *(blank)* | Замества стандартното име "Gratis AI Agent" в заглавието на чата и менюто на администратора с името на вашия продукт. Оставете празно, за да използвате стандартното. |
| **Hide Agent Picker** | Off | Когато е активиран, потребителите не могат да превключват между петте вградени агента. Текущият агент е фиксиран на това, което е конфигурирано като по подразбиране в Settings → General. |
| **Use Site Icon as Chat Avatar** | Off | Замества стандартната икона на AI в заглавието на чат виджeта с иконата на сайта на WordPress (зададена в Appearance → Customize → Site Identity). |

### Прилагане на промените

Кликнете върху **Save Settings** след промяна на който и да е флаг. Промените влизат в сила незабавно — не е необходимо изпразване на кеш или реактивиране на плагин.
