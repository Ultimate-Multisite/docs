# Changelog

## 1.9.0 — Випущено 2026-04-28 {#190--released-on-2026-04-28}

### Новинки {#new}

- **Функціонал `create_contact_form`** — створює контактну форму, використовуючи активний плагін для форм (Contact Form 7, WPForms, Fluent Forms або Gravity Forms), і повертає шорткод, готовий для вбудовування в будь-який пост чи сторінку.
- **Функціонал `set_featured_image`** — призначає зображення-обкладинку до посту чи сторінки за існуючим ID в Медіабібліотеці або за віддаленою URL-адресою; автоматично імпортує зображення, якщо надано URL.
- **Функціонал `batch_create_posts`** — створює кілька постів за один виклик функціоналу. Підтримує ті ж параметри, що й `create_post`, звітує про успіх/невдачу для кожного посту та має необов'язковий режим `stop_on_error`.
- **Параметр `page_template`** — додано до `create_post` та `update_post`. Призначає файл шаблону сторінки PHP (наприклад, `page-full-width.php`) під час створення або оновлення. Для `update_post` передайте порожній рядок, щоб повернутися до стандартного шаблону теми.
- **Функціонал скріншотів на клієнтській стороні** — `capture_screenshot`, `compare_screenshots` та `review_page_design`. Робить повні або viewport-скріншоти живих сторінок через серверний headless-браузер, порівнює два скріншоти та надає огляд дизайну, згенерований ШІ, який охоплює макет, типографіку, колір та доступність.
- **П'ять вбудованих агентів** — Content Writer, Site Builder, Design Studio, Plugin Manager та Support Assistant. Кожен агент має власний набір інструментів, адаптований системний промпт та стартові підказки. Перемикаються за допомогою нового **Agent Picker** у заголовку чату. Дивіться [Built-in Agents](../../user-guide/configuration/built-in-agents).
- **Прапорці функцій (Feature flags)** — нова вкладка **Settings → Feature Flags** з перемикачами контролю доступу (обмежено адміністраторами, обмежено адміністраторами мережі, доступ для підписників, вимкнено для нечленів) та опціями брендингу (приховати атрибуцію у футері, користувацька назва агента, приховати агент-пікер, використовувати іконку сайту як аватар чату). Дивіться [Gratis AI Agent Settings](../../user-guide/administration/gratis-ai-agent-settings).
- **Відновлення останньої сесії** — панель чату тепер автоматично перезавантажує останню розмову при завантаженні сторінки та відкритті віджета, тому контекст ніколи не втрачається під час навігації по сторінках.
- **Посилання на дії плагінів** — швидкі посилання до Налаштувань та Реєстру функціоналу тепер з'являються на екрані **Plugins → Installed Plugins** WordPress під описом плагіна.

### Покращено {#improved}

- **Повторна спроба джерела зображення** — агент тепер повторює всі налаштовані безкоштовні джерела зображень, перш ніж перейти до генерації зображення ШІ у разі збою завантаження.
- **Панель інформації про модель** — завжди видима у заголовку чату; більше не ховається після першого повідомлення.
- **Поведінка автоматичного прокручування** — автоматичне прокручування призупиняється, коли користувач прокручує вгору для читання; з'являється плаваючий кнопка **Scroll to bottom** і автоматично зникає, коли користувач досягає останнього повідомлення.
- **UI Agent Picker** — перероблено як оверлей у форматі таблиці-форми з іконками для кожного агента, що полегшує ідентифікацію та перемикання агентів на перший погляд.
- **JS-чанки з відкладеним завантаженням** — початковий JavaScript-пакет чат-віджета тепер розділено на чанки з відкладеним завантаженням, що зменшує розмір початкового пакета на 75–90%.
- **Редизайн чат-віджета** — уніфікована іконка ШІ замінила попередню користувацьку аватарку; це узгоджено з системою вбудованих агентів.
- **Посилання URL-адрес** — URL-адреси, що з'являються в системних повідомленнях та бульбашках повідомлень про помилки, тепер відображаються як клікабельні посилання.

### Виправлено {#fixed}

- **Виявлення функціоналу** — виправлено описи, посилання в системних промптах та вирівнювання просторів імен, щоб усі функції надійно відображалися у списку інструментів агента.
- **Кеш провайдерів** — провайдери тепер кешуються на рівні всього сайту за допомогою лічильника версій, запобігаючи проблемам зі старими даними провайдерів у мережах multisite.
- **`ability_invalid_output`** — виправлено у 12 обробниках функціоналу; усі тепер повертають належним чином структуровані відповіді JSON.
- **Конвеєр `pending_client_tool_calls`** — підключено від початку до кінця, щоб виклики інструментів на клієнтській стороні завершувалися коректно та повертали результати моделі.
- **Шuflік історії** — зміни, які не можна відкотити, виключені зі списку відкоту; посилання **View full history** тепер працює коректно.
- **Система змін/відкоту** — виправлено п'ять окремих помилок та об'єднано їх у новому інтерфейсі адміністратора.
- **Toast "Збереження налаштувань"** — сповіщення snackbar тепер з'являється надійно після натискання **Save Settings**.
- **Контекстне меню "Кошик"** — додано опцію **Delete Permanently**, щоб елементи можна було видалити безповоротно, не залишаючи їх у перегляді кошика.
- **Редагування та повторна відправка** — натискання на іконку редагування тепер переводить у режим редагування лише вибране повідомлення, а не всі повідомлення в потоці.
- **Висота макета чату** — панель чату адаптує свою висоту, коли над сторінкою з'являється вміст, впроваджений плагіном (сповіщення адміністратора, банери), запобігаючи витісненню області введення за екран.

## 1.4.0 — Випущено 2026-04-09 {#140--released-on-2026-04-09}

### Новинки {#new-1}

- **Команда бенчмарку WP-CLI** (`wp gratis-ai-agent benchmark`) — запускає набір бенчмарків Agent Capabilities v1 з командного рядка для конвеєрів CI та робочих процесів оцінки моделей. Підтримує виконання за окремими питаннями, перевизначення провайдера/моделі та вивід у форматах JSON/CSV.
- **Набір бенчмарків Agent Capabilities v1** — структурований набір складних багатоетапних промптів, які перевіряють повну поверхню функціоналу, з оцінюванням та звітуванням про токени/тривалість.
- **Функціонал для користувацьких типів записів (Custom Post Type)** — `register_post_type`, `list_post_types`, `delete_post_type`. Реєстрації зберігаються між перезапусками через таблицю опцій.
- **Функціонал для користувацьких таксономій (Custom Taxonomy)** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. Підтримує ієрархічні та плоскі таксономії з необов'язковими слагами перезапису.
- **Функціонал системи дизайну (Design System)** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. Включає п'ять курованих пресетів: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **Функціонал глобальних стилів (Global Styles)** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. Зчитує та записує значення theme.json через API глобальних стилів WordPress.
- **Функціонал керування навігаційним меню** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. Підтримує вкладені меню та призначення місць теми.
- **Функціонал керування опціями** — `get_option`, `set_option`, `delete_option`, `list_options`. Включає вбудований блок-список безпеки, який захищає критичні опції WordPress від зміни.
- **Реєстр функціоналу, що встановлюється (Installable Abilities Registry)** — `list_available_abilities`, `install_ability`, `recommend_plugin`. Виявляйте та активуйте пакети функціоналу, розподілені як плагіни WordPress.
- **Оркестрація Site Builder v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. Багатофазні плани створення сайту з виявленням плагінів, посиланнями на результати кроків, відстеженням прогресу та автономним відновленням після помилок.
- **Тестове питання для ресторанного вебсайту** (`q-restaurant-website`) — end-to-end тест, що охоплює реєстрацію CPT, систему дизайну, навігацію та виявлення плагінів.
- **Плагіни-коннектори для провайдерів ШІ** додано до blueprint'ів WordPress Playground для швидшого локального налаштування розробки.

### Покращено {#improved-1}

- README оновлено з документацією та інструкціями з налаштування плагінів-коннекторів для провайдерів ШІ.

### Виправлено {#fixed-1}

- Виправлено 25 збоїв тестів PHPUnit на гілці `main`.
- Виправлено формат URL-адреси випуску GitHub у `blueprint.json`.
- Перенумеровані ідентифікатори завдань, щоб уникнути колізій зі старими ID.
