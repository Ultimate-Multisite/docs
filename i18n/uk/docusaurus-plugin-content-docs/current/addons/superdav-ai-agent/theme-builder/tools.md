# Інструменти та конфігурація {#tools-and-configuration}

Theme Builder використовує набір інструментів для аналізу, проєктування та створення вашого WordPress сайту. У версії 1.16.0 **sd-ai-agent/site-scrape тепер є інструментом Tier 1**, що робить його доступним за замовчуванням у Theme Builder. Superdav AI Agent v1.18.0 також представляє окремо розповсюджуваний розширений супровідний пакет для довірених робочих процесів розробників; ці інструменти не є частиною основного пакета WordPress.org і мають бути встановлені та авторизовані окремо.

## Доступні інструменти {#available-tools}

### Інструменти Tier 1 (завжди доступні) {#tier-1-tools-always-available}

Інструменти Tier 1 доступні за замовчуванням у Theme Builder без додаткової конфігурації.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**Призначення:** Аналізувати наявні вебсайти, щоб отримати натхнення для дизайну, структуру вмісту та функціональність.

**Можливості:**

- **Аналіз вебсайту** — збирати й аналізувати вебсайти конкурентів або джерела натхнення
- **Витягування дизайну** — визначати кольори, шрифти та шаблони макета
- **Структура вмісту** — розуміти організацію сторінок та ієрархію
- **Виявлення функцій** — визначати плагіни та функціональність
- **Аналіз продуктивності** — перевіряти швидкість сторінки та оптимізацію
- **SEO-аналіз** — переглядати метатеги та структуровані дані

**Використання:**

```
Проаналізуй дизайн example.com і використай його як натхнення для мого сайту.
```

**Що він витягує:**

- Палітру кольорів і використання кольорів
- Типографіку (шрифти та розміри)
- Структуру макета та відступи
- Шаблони навігації
- Організацію вмісту
- Використання зображень і медіа
- Інтерактивні елементи
- Адаптивність для мобільних пристроїв

**Обмеження:**

- Не може збирати дані із сайтів, захищених паролем
- Дотримується robots.txt і політик сайту
- Може не захоплювати динамічний вміст
- Сайти з інтенсивним використанням JavaScript можуть мати обмежений аналіз
- Аналіз великих сайтів може займати більше часу

### Інструменти Tier 2 (необов’язкові) {#tier-2-tools-optional}

Інструменти Tier 2 доступні, коли їх явно ввімкнено в налаштуваннях Theme Builder.

#### Розширена аналітика {#advanced-analytics}

Надає детальні метрики продуктивності:

- Час завантаження сторінки
- Core Web Vitals
- SEO-оцінку
- Оцінку доступності
- Оцінку найкращих практик

#### Оптимізатор вмісту {#content-optimizer}

Аналізує та пропонує покращення для:

- Читабельності
- SEO-оптимізації
- Використання ключових слів
- Структури вмісту
- Розміщення закликів до дії

### Інструменти Tier 3 (преміум) {#tier-3-tools-premium}

Інструменти Tier 3 потребують додаткової конфігурації або преміум-функцій.

#### Генератор вмісту AI {#ai-content-generator}

Генерує вміст для:

- Описів товарів
- Сторінок послуг
- Дописів блогу
- Метаописів
- Текстів заклику до дії

#### Розширені інструменти дизайну {#advanced-design-tools}

Надає розширені можливості дизайну:

- Генерація користувацького CSS
- Створення анімацій
- Дизайн інтерактивних елементів
- Розширена теорія кольору
- Оптимізація типографіки

### Розширені супровідні інструменти (окремий пакет) {#advanced-companion-tools-separate-package}

Розширений супровідний пакет розповсюджується окремо від основного випуску Superdav AI Agent. Він призначений для довірених середовищ розробників і власників сайтів, де адміністратори явно дозволяють агенту використовувати інструменти з високим впливом.

Розширені супровідні можливості можуть включати:

- **Інструменти файлової системи** — перевіряти та змінювати схвалені файли в довірених середовищах
- **Інструменти бази даних** — виконувати запити або оновлювати дані сайту, коли це явно авторизовано
- **Інструменти WP-CLI** — запускати команди обслуговування та перевірки WordPress
- **Інструменти REST-диспетчера** — викликати зареєстровані REST-ендпоінти через контрольовані робочі процеси диспетчеризації
- **Інструменти створення плагінів** — створювати каркас і поступово вдосконалювати код плагіна
- **Інструменти Git-знімків** — створювати знімки перед ризикованими операціями розробника
- **Інструменти керування користувачами** — допомагати переглядати або коригувати записи користувачів там, де це дозволено
- **Інструменти бенчмаркінгу** — запускати бенчмарки продуктивності або можливостей для огляду під час розробки

Не документуйте їх як загальнодоступні інструменти Theme Builder для кожної інсталяції. Вони доступні лише тоді, коли розширений супровідний пакет установлено, увімкнено та обмежено для довірених адміністраторів.

## Конфігурація інструментів {#tool-configuration}

### Увімкнення інструментів {#enabling-tools}

Щоб увімкнути додаткові інструменти в Theme Builder:

1. Перейдіть до **Gratis AI Agent → Налаштування**
2. Перейдіть до **Theme Builder → Інструменти**
3. Увімкніть або вимкніть інструменти за потреби
4. Збережіть налаштування

### Дозволи інструментів {#tool-permissions}

Деякі інструменти потребують дозволів:

- **Збирання даних із сайту** — потребує доступу до інтернету
- **Аналітика** — потребує інтеграції з Google Analytics
- **Генерація вмісту** — потребує API-ключів
- **Розширені функції** — можуть потребувати преміум-підписки
- **Розширені супровідні інструменти** — потребують окремо розповсюджуваного розширеного пакета та явної довіри адміністратора

### API-ключі та облікові дані {#api-keys-and-credentials}

Налаштуйте API-ключі для інструментів, які їх потребують:

1. Перейдіть до **Gratis AI Agent → Налаштування → API-ключі**
2. Введіть облікові дані для кожного інструмента
3. Перевірте з’єднання
4. Надійно збережіть

### Установлення розширеного супровідного пакета {#installing-the-advanced-companion-package}

Розширений супровідний пакет випускається окремо від основного ZIP-пакета WordPress.org. Установлюйте його лише з офіційного каналу розповсюдження проєкту для цього випуску, а потім обмежте доступ довіреними адміністраторами. Перегляньте шлюзи людського схвалення та поведінку Git-знімків перед увімкненням інструментів файлової системи, бази даних, WP-CLI, REST-диспетчера, створення плагінів, керування користувачами або бенчмаркінгу на робочому сайті.

## Використання sd-ai-agent/site-scrape {#using-sd-ai-agentsite-scrape}

### Базове використання {#basic-usage}

Попросіть Theme Builder проаналізувати вебсайт:

```
Проаналізуй дизайн сайту мого конкурента за адресою competitor.com
і запропонуй покращення дизайну для мого сайту.
```

### Конкретний аналіз {#specific-analysis}

Запитуйте конкретні типи аналізу:

```
Витягни палітру кольорів із example.com і використай її як натхнення.
```

```
Проаналізуй структуру навігації example.com і застосуй подібну
організацію до мого сайту.
```

```
Перевір адаптивність example.com для мобільних пристроїв і забезпеч, щоб мій сайт
був так само адаптивним.
```

### Порівняльний аналіз {#comparison-analysis}

Порівнюйте кілька сайтів:

```
Порівняй дизайни site1.com і site2.com та створи дизайн,
який поєднує найкращі елементи обох.
```

## Обмеження інструментів і міркування {#tool-limitations-and-considerations}

### Обмеження частоти запитів {#rate-limiting}

- Scraping має обмеження частоти, щоб запобігти перевантаженню сервера
- Кілька запитів до того самого сайту можуть обмежуватися
- Великі сайти можуть потребувати більше часу для аналізу

### Точність вмісту {#content-accuracy}

- Динамічний вміст може бути захоплений не повністю
- Вміст, відрендерений JavaScript, може бути неповним
- Дані в реальному часі можуть не відображатися
- Деякий вміст може бути за платними стінами

### Юридичні та етичні міркування {#legal-and-ethical-considerations}

- Поважайте robots.txt і політики сайту
- Не виконуйте scraping захищеного авторським правом вмісту для відтворення
- Використовуйте аналіз для натхнення, а не для копіювання
- Переконайтеся, що маєте права на використання витягнутого вмісту
- Дотримуйтеся умов обслуговування проаналізованих сайтів

### Вплив на продуктивність {#performance-impact}

- Scraping великих сайтів може займати час
- Кілька одночасних scraping можуть бути повільнішими
- Мережеве з’єднання впливає на швидкість
- Великі набори даних можуть потребувати більше обробки

## Найкращі практики {#best-practices}

### Використання аналізу сайту {#using-site-analysis}

1. **Аналізуйте кілька сайтів** — збирайте натхнення з кількох джерел
2. **Зосередьтеся на структурі** — вивчайте шаблони макета й організації
3. **Витягуйте кольори** — використовуйте колірні палітри як відправні точки
4. **Вивчайте типографіку** — визначайте поєднання шрифтів, які вам подобаються
5. **Переглядайте навігацію** — розумійте організацію меню

### Етичний scraping {#ethical-scraping}

1. **Поважайте robots.txt** — дотримуйтеся вказівок сайту
2. **Не копіюйте вміст** — використовуйте аналіз лише для натхнення
3. **Перевіряйте права** — переконайтеся, що можете використовувати витягнутий вміст
4. **Вказуйте джерела** — зазначайте джерела натхнення
5. **Дотримуйтеся умов** — виконуйте умови обслуговування сайту

### Максимізація ефективності інструменту {#maximizing-tool-effectiveness}

1. **Будьте конкретними** — запитуйте конкретні типи аналізу
2. **Надавайте контекст** — пояснюйте призначення вашого сайту
3. **Задавайте очікування** — описуйте свої цілі дизайну
4. **Ітеруйте** — уточнюйте на основі результатів
5. **Поєднуйте інструменти** — використовуйте кілька інструментів для комплексного аналізу

## Усунення несправностей {#troubleshooting}

### Сайт не вдається просканувати {#site-wont-scrape}

- Перевірте, чи сайт є публічно доступним
- Переконайтеся, що robots.txt дозволяє scraping
- Спробуйте інший сайт
- Перевірте інтернет-з’єднання
- Зверніться до підтримки, якщо проблема не зникає

### Неповний аналіз {#incomplete-analysis}

- Сайт може мати динамічний вміст
- JavaScript може бути відрендерений не повністю
- Великі сайти можуть перевищувати час очікування
- Спробуйте натомість аналізувати конкретні сторінки
- Запитайте конкретні типи аналізу

### Низька продуктивність {#slow-performance}

- Великі сайти потребують більше часу для аналізу
- Кілька одночасних scraping є повільнішими
- Мережеве з’єднання впливає на швидкість
- Спробуйте в години меншого навантаження
- Спочатку аналізуйте менші розділи
