# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency дозволяє вашій мережі приймати платежі в кількох валютах. Він автоматично конвертує ціни, використовуючи актуальні або ручні обмінні курси, дозволяє встановлювати точні ціни для кожного продукту в будь-якій валюті та визначає бажану валюту кожного відвідувача за допомогою геолокації — і все це керується з однієї панелі адміністратора.

## Основні функції {#key-features}

- **Автоматична конвертація обмінних курсів** — ціни конвертуються з вашої базової валюти за курсами, отриманими від обраного вами провайдера.
- **Перевизначення цін на продукт** — встановлюйте точні ціни для конкретної валюти на будь-якому продукті, щоб обійти автоматичну конвертацію.
- **Поле вибору валюти при реєстрації** — дозволяє клієнтам вибрати бажану валюту під час оформлення замовлення/реєстрації.
- **Кілька провайдерів обмінних курсів** — ECB, Frankfurter, Stripe або повністю ручні курси.
- **Виявлення валюти на основі геолокації** — автоматично попередньо вибирає валюту, яка відповідає країні відвідувача.
- **Панель адміністратора** — керуйте активними валютами, обмінними курсами та налаштуваннями провайдерів з одного місця.

## Вимоги {#requirements}

- Ultimate Multisite 2.0.0 або вище
- Встановлення WordPress Multisite
- PHP 7.4 або вище

## Встановлення {#installation}

1. Завантажте файли плагіна до `/wp-content/plugins/ultimate-multisite-multi-currency`
2. Активуйте плагін на екрані **Плагіни** у WordPress (Активувати для мережі)
3. Перейдіть до **Ultimate Multisite → Multi-Currency**, щоб відкрити панель адміністратора
4. Додайте валюти, які ви хочете приймати, та налаштуйте провайдера обмінних курсів

## Налаштування {#configuration}

### Панель адміністратора {#admin-dashboard}

Панель адміністратора Multi-Currency (**Ultimate Multisite → Multi-Currency**) — це центральне місце для:

- Додавання, активації або деактивації валют
- Вибору провайдера обмінних курсів та оновлення курсів за запитом
- Перегляду поточних курсів для всіх активних валют
- Встановлення ручних курсів, коли автоматичні провайдери не підходять

### Провайдери обмінних курсів {#exchange-rate-providers}

| Провайдер | Опис |
|---|---|
| **ECB** | Європейський центральний банк — безкоштовні щоденні курси для мереж, базуваних на EUR |
| **Frankfurter** | Відкритий API-дзеркало ECB — безкоштовно, не потрібен API-ключ |
| **Stripe** | Використовує актуальні курси валют Stripe — вимагає активної інтеграції зі Stripe |
| **Manual** | Ви вводите курси напряму; не потрібні зовнішні виклики API |

Виберіть свого провайдера у розділі **Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider**. Курси можна оновити вручну в будь-який час або запланувати автоматичне оновлення.

### Перевизначення цін на продукт {#per-product-pricing-overrides}

Коли автоматична конвертація недостатньо точна, ви можете встановити точну ціну для конкретної валюти на будь-якому продукті:

1. Відкрийте редактор продукту (**Ultimate Multisite → Products → Edit**)
2. Прокрутіть до розділу **Currency Pricing**
3. Введіть точну ціну для кожної валюти, яку ви хочете перевизначити
4. Збережіть продукт

Продукти без перевизначення використовують конвертовану ціну з вашої базової валюти.

### Поле вибору валюти при реєстрації {#currency-selector-signup-field}

Поле вибору валюти можна додати до вашого потоку реєстрації/оформлення замовлення, щоб клієнти обирали бажану валюту під час реєстрації:

1. Перейдіть до **Ultimate Multisite → Multi-Currency → Settings**
2. Активуйте **Show currency selector on signup**
3. За бажанням встановіть валюту за замовчуванням або дозвольте геолокації визначити попередньо вибране значення

Обрана валюта зберігається з підпискою та використовується для всіх подальших платежів.

### Виявлення валюти на основі геолокації {#geolocation-based-currency-detection}

Коли увімкнено виявлення геолокації, плагін зіставляє країну відвідувача з валютою та попередньо вибирає її у полі вибору валюти. Щоб увімкнути:

1. Перейдіть до **Ultimate Multisite → Multi-Currency → Settings**
2. Активуйте **Detect currency from visitor location**
3. Зіставте країни з валютами у таблиці **Geolocation Mapping**

Якщо для країни відвідувача немає зіставлення, як запасний варіант використовується базова валюта мережі.

## Часто задавані питання {#frequently-asked-questions}

### Що станеться, якщо провайдер обмінних курсів буде недоступний? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

Плагін повернеться до останніх успішно отриманих курсів. Якщо курси ніколи не отримувалися, він повернеться до ручних курсів, які ви налаштували. Рекомендується налаштувати ручні курси як запобіжний захід.

### Чи можу я використовувати різні провайдери для різних валют? {#can-i-use-different-providers-for-different-currencies}

Ні — для всіх валют активний лише один провайдер. Використовуйте **Manual**, якщо вам потрібен повний контроль над окремими курсами.

### Чи вимагає геолокація окремого плагіна? {#does-geolocation-require-a-separate-plugin}

Не потрібен зовнішній плагін геолокації. Плагін використовує вбудоване виявлення IP-адреси до країни WordPress. Для вищої точності можна інтегрувати спеціалізований сервіс геолокації через надані хуки фільтрації.

### Чи застосовуються перевизначення цін на продукт до чи після податків? {#are-per-product-overrides-applied-before-or-after-tax}

Перевизначення цін на продукт встановлює ціну до податків у цільовій валюті. Податок розраховується поверх перевизначеної ціни, використовуючи вашу існуючу конфігурацію податків.
