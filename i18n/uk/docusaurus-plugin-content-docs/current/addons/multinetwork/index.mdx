# Multi-Network

Додаток Ultimate Multisite: Multinetwork розширює вашу платформу Ultimate Multisite потужною функціональністю мультимереж. Цей додаток дозволяє вашим клієнтам купувати та керувати цілими мережами WordPress multisite, а не лише окремими сайтами.

## Основні функції

* **Провизіонування мереж (Network Provisioning)**: Автоматично створює та налаштовує повні мережі WordPress multisite для ваших клієнтів.
* **Шаблони мереж (Network Templates)**: Дозволяє клієнтам переглядати, попередньо переглядати та обирати готовий шаблон мережі перед реєстрацією.
* **Керування мережами (Network Management)**: Повні можливості керування середовищами мультимереж.
* **Ізольовані мережі (Isolated Networks)**: Кожна мережа клієнта працює незалежно, маючи власні таблиці бази даних та конфігурацію.
* **SSO між мережами (Cross-Network SSO)**: Забезпечує клієнтам безшовний доступ до їхніх підмереж за допомогою автентифікації за магічним посиланням (magic-link).
* **Масштабована архітектура (Scalable Architecture)**: Ефективно обробляє кілька мереж за допомогою оптимізованого розподілу ресурсів.
* **Інтеграція (Integration)**: Бездоганно інтегрується з системами підписки та білінгу Ultimate Multisite.

## Сценарії використання

* **Хостинг з білим лейблом (White Label Hosting)**: Надайте клієнтам власні брендовані мережі multisite.
* **Рішення для агентств (Agency Solutions)**: Дозволяє агентствам керувати кількома мережами клієнтів з однієї платформи.
* **Корпоративні розгортання (Enterprise Deployments)**: Підтримує великі організації зі складними вимогами до кількох мереж.
* **Освітні платформи (Education Platforms)**: Створюйте окремі мережі для різних шкіл або відділів.

## Вимоги

* Ultimate Multisite 2.0.0 або вище
* Встановлення WordPress Multisite
* PHP 7.4 або вище
* MySQL 5.6 або вище

## Встановлення

1. Завантажте файли плагіна до каталогу `/wp-content/plugins/ultimate-multisite-multinetwork`
2. Активуйте плагін на екрані «Плагіни» у WordPress
3. Налаштуйте параметри мультимереж через Ultimate Multisite → Settings

## Шаблони мереж

Версія 1.3.0 додає процес пошуку та вибору шаблонів для клієнтів, які купують нові мережі. Використовуйте шаблони мереж, коли ви хочете, щоб кожна нова мережа клієнта починалася з підготовленої структури, дизайну, бібліотеки контенту або спеціалізованого для вертикалі налаштування.

### Попередній перегляд шаблону

Попередній переглядач мережевих шаблонів дозволяє клієнтам переглядати доступні шаблони перед оформленням замовлення. Кожен шаблон можна відкрити у панелі попереднього перегляду в режимі реального часу, щоб клієнти могли порівнювати макети та контент перед покупкою мережі.

Використовуйте переглядач, щоб допомогти клієнтам:

* Ознайомитися зі виглядом та структурою кожного доступного шаблону мережі
* Порівняти кілька шаблонів, не виходячи з процесу покупки
* Обрати шаблон, який найкраще відповідає їхньому сценарію використання перед реєстрацією

Зберігайте назви та описи шаблонів дружніми для клієнтів. Переглядач найбільш корисний, коли клієнти можуть зрозуміти передбачуваний сценарій використання, наприклад, мережі агентств, шкільні мережі, франчайзингові мережі або пакети хостингу з білим лейблом.

### Вибір шаблону при оформленні замовлення

Вибір шаблону можна додати до процесу оформлення замовлення (checkout flow) як поле. Після налаштування клієнти вибирають шаблон мережі, який їм потрібен, під час оформлення замовлення, і обраний шаблон використовується, коли провизіонується їхня нова мережа.

Щоб використовувати вибір у процесі оформлення замовлення:

1. Підготуйте мережі, які ви хочете запропонувати як шаблони.
2. Увімкніть поле вибору шаблону мультимережі у відповідній формі реєстрації або оформлення замовлення.
3. Виберіть, які шаблони мають бути доступні для цього процесу оформлення замовлення.
4. Опублікуйте процес оформлення замовлення та протестуйте реєстрацію, щоб підтвердити, що обраний шаблон застосовується до нової мережі.

Якщо процес оформлення замовлення пропонує лише один продукт мережі, вибір шаблону все одно може бути корисним як керований етап онбордингу. Для воронок із кількома планами виводьте лише ті шаблони, які сумісні з обраним продуктом або планом.

## SSO за магічним посиланням між мережами

SSO за магічним посиланням між мережами дозволяє клієнтам переходити між своєю основною зоною облікового запису та підмережами без повторного ручного входу. Коли клієнт переходить за посиланням доступу між мережами, Ultimate Multisite генерує обмежене за часом магічне посилання та автентифікує клієнта у цільовій мережі.

Це покращує досвід для клієнтів, які керують більше ніж однією мережею, або яким потрібно негайно отримати доступ до щойно клонованих мереж-шаблонів після покупки.

Чекліст адміністратора:

* Підтвердіть, що додаток Multinetwork активний у всій установці WordPress multisite.
* Переконайтеся, що клієнти мають правильний доступ до мережі, перш ніж їм будуть запропоновані посилання між мережами.
* Зберігайте коректну конфігурацію доменів мереж та відображених доменів, щоб магічні посилання вели до передбачуваної цільової мережі.
* Протестуйте панель клієнта та будь-які кнопки доступу після оформлення замовлення після зміни налаштувань мережі, домену чи оформлення замовлення.

Магічні посилання слід розглядати як посилання для входу. Вони розроблені для безшовного доступу клієнта і не повинні пересилатися чи повторно використовуватися поза межами запланованої сесії.

## Часто задавані питання

### Що таке multinetwork?

Multinetwork — це функція WordPress, яка дозволяє запускати кілька незалежних мереж multisite на одній установці WordPress.

### Чим це відрізняється від звичайного multisite?

Звичайний multisite дозволяє мати кілька сайтів у межах однієї мережі. Multinetwork дозволяє мати кілька мереж, кожна зі своїми сайтами, все в межах однієї установки WordPress.

### Чи вимагає це спеціальної конфігурації сервера?

Так, multinetwork вимагає специфічного налаштування та конфігурації сервера. Будь ласка, зверніться до документації для отримання детальних інструкцій зі встановлення.
