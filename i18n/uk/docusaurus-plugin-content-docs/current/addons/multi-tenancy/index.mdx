# Мультиорендність

Розподіляйте підсайти між серверами та базами даних для продуктивності й безпеки

## Основні новини випуску версії 1.2.0

Multi-Tenancy 1.2.0 додає основу суверенного орендаря для робочих процесів ізоляції у продакшені:

- Автовхід орендаря без збереження стану з короткочасними, захищеними від повторного відтворення SSO-візитами.
- Ізоляція бази даних і файлової системи для кожного підсайту для суверенних орендарів.
- Можливість мультиорендності Hostinger для підготовки орендарів з урахуванням хоста.
- Контрольні шлюзи перевірки суверенної міграції для перевірок застарілих даних і асинхронної обробки надсилання.
- Налаштовуване прив’язування хоста бази даних на тій самій машині для встановлень Bedrock, FrankenPHP і localhost.
- Покращення початкового завантаження орендаря, записувача бази даних, початкового завантаження схеми та підготовки користувачів.
- Безпечніше суверенне видалення, яке очищає облікові дані орендаря.

Дивіться посібники користувача щодо [робочого процесу суверенної міграції](/user-guide/getting-started/sovereign-migration-workflow), [ізоляції мультиорендності](/user-guide/configuration/multi-tenancy-isolation) та [операцій із суверенними орендарями](/user-guide/administration/sovereign-tenant-operations).

## Вимоги

- WordPress 5.3 або вище
- PHP 7.4 або вище
- Плагін Multisite Ultimate (активний)

## Встановлення

1. Завантажте файли додатка до вашого каталогу `/wp-content/plugins/`
2. Активуйте плагін через меню «Плагіни» у WordPress
3. Налаштуйте параметри додатка в адмініструванні мережі

## Розробка

### Налаштування

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Тестування

Додаток містить комплексний фреймворк тестування:

- **Модульні тести**: тести на основі PHPUnit з набором тестів WordPress
- **Базовий клас тестів**: надає допоміжні методи для поширених сценаріїв тестування
- **Покриття тестами**: налаштовано для відстеження покриття коду
- **Тестування Multisite**: тести виконуються в середовищі multisite

### Стандарти коду

- **Стандарти кодування WordPress**: застосовуються через PHPCS
- **Сумісність із PHP 7.4+**: підтримуються сучасні можливості PHP
- **Статичний аналіз**: інтеграція PHPStan для перевірки типів
- **Модернізація коду**: правила Rector для оновлень PHP

### Система збирання

- **Обробка ресурсів**: мініфікація файлів CSS/JS
- **Переклад**: генерація файлу POT
- **Пакування**: створення архіву для розповсюдження
- **Режим розробки**: немініфіковані ресурси для налагодження

## Конфігурація

Додаток підтримує різні параметри конфігурації, доступні через інтерфейс адміністрування мережі.

### Конфігурація суверенного орендаря

Для суверенних орендарів налаштуйте базу даних орендаря, корінь файлової системи, прив’язування хоста бази даних і можливість хоста перед маршрутизацією трафіку до орендаря. Запускайте перевірку міграції після кожної зміни, що впливає на облікові дані бази даних, прив’язування хоста, маршрутизацію або підготовку користувачів.

Використовуйте `localhost` лише тоді, коли середовище виконання орендаря та дозволи бази даних очікують локальний сокет або доступ на тій самій машині. Використовуйте `127.0.0.1` або ім’я хоста сервісу, коли саме це є хостом дозволів, налаштованим вашим провайдером.

### Можливість Hostinger

Можливість мультиорендності Hostinger працює зі спільним інтеграційним токеном Hostinger. Переконайтеся, що домен присутній у hPanel, DNS вказує на обліковий запис Hostinger, а налаштування бази даних орендаря відповідають hPanel перед перевіркою.

## Хуки та фільтри

### Дії

- `ultimate-multisite-addon-template_init` - Спрацьовує після ініціалізації додатка
- `ultimate-multisite-addon-template_loaded` - Спрацьовує після завантаження всіх файлів додатка

### Фільтри

- `ultimate-multisite-addon-template_settings` - Змінити налаштування додатка
- `ultimate-multisite-addon-template_enabled` - Перевизначити стан увімкнення/вимкнення додатка
