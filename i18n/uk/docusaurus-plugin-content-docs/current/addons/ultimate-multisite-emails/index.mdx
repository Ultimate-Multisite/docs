# Електронна пошта

Додаток Ultimate Multisite: Emails дозволяє власникам мереж надавати та керувати обліковими записами електронної пошти клієнтів поряд із сайтами та членствами. У версії 1.1.0 додано керування обліковими записами електронної пошти для клієнтів, вхід до вебпошти за єдиним обліковим записом (SSO), підтримку перенаправлення CyberPanel та покращення функціоналу створення облікових записів під час оформлення замовлення.

## Основні функції

* **Віджет облікового запису електронної пошти клієнта**: Клієнти можуть створювати та керувати обліковими записами електронної пошти зі своєї сторінки облікового запису, не створюючи тікет підтримки.
* **SSO вебпошти та магічні посилання**: Клієнти можуть відкривати вебпошту безпосередньо зі сторінки облікового запису за один клік, якщо це підтримує провайдер.
* **Зміна паролів**: Клієнти та адміністратори можуть оновлювати паролі поштових скриньок через віджет керування електронною поштою.
* **Створення облікових записів під час оформлення замовлення**: Створення облікового запису електронної пошти клієнта може відбуватися під час оформлення замовлення, щоб поштова скринька була готовою після завершення процесу покупки.
* **Перенаправлення CyberPanel**: Мережі на базі CyberPanel можуть створювати та керувати перенаправленням електронної пошти як частиною процесу надання послуг електронної пошти.
* **Налаштування з урахуванням провайдера**: Сторінка налаштувань фільтрує провайдерів, відображаючи лише активні та налаштовані інтеграції, щоб адміністратори бачили лише робочі варіанти.

## Вимоги

* Ultimate Multisite 2.0.0 або вище
* Встановлення WordPress Multisite
* Принаймні одна налаштована інтеграція провайдера електронної пошти
* Облікові дані API провайдера з дозволом на створення поштових скриньок, зміну паролів та видачу посилань до вебпошти, де це підтримується

## Встановлення

1. Завантажте файли плагіна до `/wp-content/plugins/ultimate-multisite-emails`.
2. Активуйте додаток на рівні мережі у **Plugins** в адмін-панелі мережі WordPress.
3. Перейдіть до **Ultimate Multisite** > **Settings** та відкрийте налаштування електронної пошти додатка.
4. Увімкніть інтеграції провайдерів електронної пошти, які ви хочете, щоб клієнти використовували.
5. Збережіть налаштування, а потім протестуйте створення облікового запису з непродакшн клієнтом або на тестовому сайті.

## Керування електронною поштою облікового запису клієнта

У версії 1.1.0 додано віджет керування електронною поштою на сторінку облікового запису клієнта. Використовуйте його, коли клієнти повинні самостійно створювати та підтримувати поштові скриньки для власних сайтів.

На сторінці облікового запису клієнти можуть:

1. Переглядати облікові записи електронної пошти, пов'язані з їхнім поточним членством або сайтом.
2. Створювати нову поштову скриньку, якщо їхній план дозволяє додаткові облікові записи.
3. Змінювати пароль існуючої поштової скриньки.
4. Відкривати підтримувані провайдери вебпошти через SSO або вхід за магічним посиланням.

Віджет використовує поточну систему сторінки облікового запису Ultimate Multisite, тому дії обмежуються контекстом облікового запису, який переглядає клієнт. Це запобігає керуванню клієнтами обліковими записами електронної пошти, які належать до іншого членства чи сайту.

## SSO вебпошти та вхід за магічним посиланням

SSO вебпошта надає клієнтам пряму дію **Open webmail** зі сторінки облікового запису. Якщо це підтримує активний провайдер, додаток запитує короткочасне URL-посилання доступу та перенаправляє клієнта до вебпошти, не вимагаючи від нього копіювання облікових даних поштової скриньки.

Поведінка провайдерів може відрізнятися:

* Провайдери з нативними API SSO можуть створити пряму автентифіковану сесію вебпошти.
* Провайдери з підтримкою магічних посилань можуть генерувати обмежене за часом посилання для входу.
* Провайдери без підтримки SSO або магічних посилань все одно повинні відображати деталі поштової скриньки та дії керування паролем, але без кнопки вебпошти за один клік.

Увімкніть доступ до вебпошти лише для тих провайдерів, для яких ви налаштували необхідні облікові дані API. Якщо дія відсутня для провайдера, спочатку підтвердіть, що провайдер активний, налаштований та підтримує посилання входу до вебпошти.

## Створення облікових записів під час оформлення замовлення

Додаток може створювати облікові записи електронної пошти клієнтів як частину процесу оформлення замовлення. Це корисно, коли план за замовчуванням включає поштову скриньку або коли форма оформлення замовлення збирає деталі облікового запису електронної пошти.

Рекомендований порядок впровадження:

1. Налаштуйте ліміти провайдера та плану перед увімкненням створення облікових записів під час оформлення замовлення.
2. Протестуйте покупку з тестовим продуктом і підтвердіть, що поштова скринька створюється після оформлення замовлення.
3. Перегляньте сторінку облікового запису клієнта, щоб підтвердити, що нова поштова скринька з'явилася у віджеті електронної пошти.
4. Підтвердіть дії зміни пароля та вебпошти, перш ніж вмикати цей процес для продакшн-продуктів.

Якщо створення облікового запису не вдалося, обліковий запис та членство клієнта все одно можуть існувати, тоді як поштова скринька вимагатиме подальшої уваги адміністратора. Перевірте журнали активності Ultimate Multisite та журнали провайдера, перш ніж повторювати спробу.

## Перенаправлення електронної пошти CyberPanel

Для мереж на базі CyberPanel у версії 1.1.0 додано підтримку перенаправлення електронної пошти. Використовуйте перенаправлення, коли клієнтам потрібні адреси, такі як `info@customer-domain.test`, щоб перенаправляти пошту на іншу скриньку, не створюючи при цьому повноцінну поштову скриньку.

Перш ніж пропонувати перенаправлення, підтвердіть, що інтеграція CyberPanel налаштована та доступна. Зверніться до [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) для вивчення необхідних констант хоста, імені користувача, пароля та пакета.

## Часто задавані питання

## Чи можуть клієнти створювати власні облікові записи електронної пошти?

Так. Коли увімкнено віджет облікового запису клієнта і план дозволяє облікові записи електронної пошти, клієнти можуть створювати та керувати власними поштовими скриньками зі сторінки облікового запису.

## Чи підтримує кожен провайдер SSO вебпошти?

Ні. Підтримка SSO та магічних посилань залежить від активного провайдера електронної пошти. Провайдери без API для посилань доступу все одно можуть підтримувати створення поштових скриньок та зміну паролів.

## Чи можуть адміністратори створювати облікові записи електронної пошти для клієнтів?

Так. Адміністратори можуть створювати облікові записи електронної пошти з інтерфейсу адміністратора, включаючи встановлення початкового пароля під час створення облікового запису.

## Чи є перенаправлення тим самим, що поштові скриньки?

Ні. Поштова скринька зберігає пошту та має облікові дані для входу. Перенаправлення перенаправляє пошту на іншу адресу і не надає окремої скриньки.
