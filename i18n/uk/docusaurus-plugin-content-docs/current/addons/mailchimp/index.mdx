# Mailchimp

Ultimate Multisite: Інтеграція з Mailchimp

Підключіть вашу мережу Ultimate Multisite до Mailchimp. Ви можете пов'язати певні плани з списками Mailchimp та автоматично додавати підписників до цих списків під час реєстрації.

## Встановлення {#installation}

1. Завантажте 'ultimate-multisite-mailchimp' до каталогу '/wp-content/plugins/'
2. Активуйте плагін через меню 'Plugins' у вашому WordPress Network Admin

## Посібник із використання {#usage-guide}

Синхронізація електронних адрес підписників зі списками MailChimp надзвичайно проста за допомогою нашого плагіна Ultimate Multisite: MailChimp Integration Add-on. Цей посібник допоможе вам налаштувати все!

## Отримання API ключа {#getting-an-api-key}

Перш ніж ми перейдемо до різних доступних налаштувань, нам потрібен API ключ MailChimp. Це дозволить Ultimate Multisite спілкуватися з API MailChimp для отримання списків, груп та додавання електронних адрес до цих списків і груп, коли користувачі реєструються.

Дотримуйтесь цього посібника, щоб отримати власний API ключ MailChimp: [About API Keys | Mailchimp](https://kb.mailchimp.com/integrations/api-integrations/about-api-keys)

<!-- Screenshot unavailable: MailChimp API key settings in Ultimate Multisite Add-on Settings -->

_Додайте ваш API ключ MailChimp у_ _**Налаштування Ultimate Multisite - > Налаштування Add-on -> MailChimp**_

## Режим інтеграції {#integration-mode}

Add-on інтеграції MailChimp пропонує два «режими інтеграції», що означає, що є два різні способи використання цього аддону для синхронізації електронних адрес.

<!-- Screenshot unavailable: MailChimp integration mode selection with Multiple Lists and Single List options -->

_Два режими інтеграції для більшої гнучкості!_

### Режим 1: Кілька списків (Multiple Lists) {#mode-1-multiple-lists}

Режим за замовчуванням використовує **Кілька списків (Multiple Lists)**. Цей режим дозволить вам вибрати кілька списків MailChimp для кожного плану. Це найкращий варіант, якщо ви сегментуєте своїх користувачів у різні списки.

Наприклад, припустимо, у вас є 3 різні списки на обліковому записі MailChimp: **Newsletter** , **Plan A** та **Plan B**. Ви хочете, щоб усі користувачі, незалежно від їхнього плану, також були включені до списку Newsletter, щоб вони не пропустили ваші оновлення.

Щоб це реалізувати, ви можете використати режим Multiple Lists, щоб додати обидва списки **Newsletter** та **Plan A** до налаштувань MailChimp для Plan A, а **Newsletter** та **Plan B** — до налаштувань MailChimp для Plan B.

<!-- Screenshot unavailable: MailChimp default lists selection for new accounts and canceled accounts -->

_Виберіть списки за замовчуванням для нових та скасованих облікових записів_

### Режим 2: Один список + Кілька груп (Single List + Multiple Groups) {#mode-2-single-list--multiple-groups}

MailChimp дозволяє вам сегментувати ваш список на менші групи користувачів, які називаються **Групами (Groups)**. Це дає вам більше контролю, оскільки ви можете надіслати кампанію конкретній підгрупі своїх підписників, не створюючи при цьому кілька копій однієї й тієї ж кампанії в MailChimp (що ускладнює аналіз метрик цієї кампанії згодом).

Наприклад, ви можете мати один список під назвою **My Network Subscribers**, і всередині нього мати групи для **Plan A** та **Plan B**.

<!-- Screenshot unavailable: MailChimp single list and groups selection for default lists -->

_Виберіть списки та групи за замовчуванням, якщо ви використовуєте режим інтеграції «один список + кілька груп»_

## Списки та групи за замовчуванням {#default-lists-and-groups}

Після того, як ви визначите, який режим інтеграції більше підходить для вашого конкретного випадку використання, настав час вибрати списки за замовчуванням (і групи, залежно від режиму). Список за замовчуванням буде використано, коли заданий план не має вибраного конкретного Списку.

Ви також можете вибрати список за замовчуванням для скасованих облікових записів. Щоразу, коли користувач припиняє свою підписку та видаляє свій обліковий запис із вашої мережі, його електронна адреса буде переміщена до списку, призначеного для скасованих облікових записів.

## Увімкнення подвійної підписки (Double Subscription) {#enabling-double-subscription}

На тій самій сторінці налаштувань ви зможете увімкнути **Double Subscription**. Ця опція додасть користувача як до списку за замовчуванням, так і до списку, вибраного для плану цього користувача. Якщо ця опція не увімкнена, користувач буде доданий лише до списку, вибраного для плану. Якщо для цього конкретного плану нічого не встановлено, то користувач буде доданий до списку за замовчуванням.

<!-- Screenshot unavailable: MailChimp double subscription toggle setting -->

_Ваші користувачі можуть бути підписані як на список за замовчуванням, так і на список плану при реєстрації_

## Вибір списків та груп для кожного плану {#selecting-lists-and-groups-for-each-of-the-plans}

Перейдіть до своїх **Plans** та виберіть план, який ви хочете відредагувати. Зверніть увагу, що в мета-боксах **Advanced Options** з'явиться нова вкладка під назвою MailChimp.

На цій вкладці ви зможете вибрати кілька списків або один список та кілька груп (залежно від режиму інтеграції) для цього плану.

<!-- Screenshot unavailable: MailChimp tab on plan edit page showing lists and groups selection -->

_Виберіть Списки та групи, які ви хочете використовувати для кожного зі своїх планів_

Коли новий користувач реєструється за цим планом, його електронна адреса буде додана до вибраних списків та груп!
