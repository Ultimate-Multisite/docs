# Domain Seller

**Multisite Ultimate: Domain Seller** перетворює вашу мережу WordPress Multisite на потужну платформу продажу доменів. Дозвольте клієнтам реєструвати власні домени безпосередньо під час реєстрації або з їхнього Dashboard, з перевіркою доступності в реальному часі та прозорим ціноутворенням.

### Чому Domain Seller?

Реєстрація доменів — один із найпопулярніших додаткових продажів для будь-якої платформи WaaS (Website as a Service). Замість того щоб скеровувати ваших клієнтів до стороннього реєстратора, ви можете пропонувати домени безпосередньо — зберігаючи більше доходу у своїй мережі та водночас забезпечуючи клієнтам безперешкодний досвід.

### Ключові можливості

#### Підтримка кількох реєстраторів доменів
* **OpenSRS** — Повна реєстрація, поновлення та керування
* **Namecheap** — Повна інтеграція з WhoisGuard і тестуванням у sandbox
* **HostAfrica** — Пошук доменів, ціноутворення, реєстрація, перенесення, DNS і захист ID через Domains Reseller API
* **Openprovider** — Реселерські ціни, реєстрація, поновлення, перенесення, DNS, конфіденційність WHOIS і синхронізація TLD
* **Hostinger** — Пошук і реєстрація доменів із використанням спільного токена інтеграції Hostinger
* **Розширювана архітектура** — Побудована на шаблоні Integration Registry, що спрощує додавання нових провайдерів
* **Вбудоване тестування підключення** — Перевіряйте облікові дані провайдера та підключення перед запуском

#### Майстер налаштування з підказками
* **Підключення одним кліком** — Проводить вас через конфігурацію провайдера, імпорт TLD і створення доменного продукту
* **Автоматична синхронізація TLD** — Імпортує всі доступні TLD і оптові ціни з підключених провайдерів
* **Створення типового продукту** — Створює готовий до використання доменний продукт із розумними типовими націнками

#### Гнучке ціноутворення доменів
* **Три режими націнки** — Відсоткова націнка, фіксована націнка або фіксована ціна за TLD
* **Продукти для окремих TLD** — Створюйте окремі доменні продукти для різних груп TLD (.com/.net, .io, коди країн тощо)
* **Вступне ціноутворення** — Пропонуйте знижену ціну за перший рік, щоб залучати нові реєстрації
* **Ціноутворення в реальному часі** — Актуальні оптові ціни від провайдерів з інтелектуальним кешуванням
* **Підтримка купонів** — Повна сумісність із кодами знижок Multisite Ultimate

#### Захист конфіденційності WHOIS
* **Конфігурація для кожного продукту** — Установіть конфіденційність WHOIS як Disabled, Always Included або Customer Choice
* **Згода клієнта** — Якщо встановлено Customer Choice, прапорець під час оформлення замовлення дозволяє клієнтам додати захист конфіденційності
* **Налаштовуване ціноутворення** — Установіть ціну за рік для доповнення конфіденційності або пропонуйте його безкоштовно
* **Інтеграція з реєстратором** — Автоматично вмикає WhoisGuard (Namecheap) або конфіденційність WHOIS (OpenSRS) під час реєстрації

#### Єдиний досвід оформлення замовлення
* **Три режими домену** — Клієнти обирають між субдоменом (безкоштовно), реєстрацією нового домену або прив’язкою наявного домену, яким вони вже володіють
* **Пошук домену в реальному часі** — Перевірка доступності на основі AJAX без перезавантаження сторінки
* **Розумні пропозиції** — Альтернативні пропозиції TLD, коли бажаний домен зайнятий
* **Прозоре ціноутворення** — Чітке відображення ціни реєстрації, ціни поновлення та необов’язкової плати за конфіденційність WHOIS
* **Поля контактних даних реєстранта** — Збирає необхідну інформацію про реєстранта (ім’я, адресу, телефон) безпосередньо під час оформлення замовлення
* **Автоматично згенерована URL-адреса сайту** — Адреса сайту автоматично формується з обраного домену

#### Система автопоновлення
* **Поновлення, прив’язані до членства** — Домени поновлюються автоматично, доки членство клієнта активне
* **Розумне планування** — Завдання поновлення ставляться в чергу на основі дат закінчення терміну дії доменів
* **Логіка повторних спроб** — Інтелектуальні повторні спроби для невдалих спроб поновлення
* **Email-сповіщення** — Налаштовувані сповіщення про майбутні поновлення, успішні поновлення, збої та закінчення терміну дії

#### Керування DNS для клієнтів
* **Самообслуговування DNS** — Клієнти можуть додавати, редагувати й видаляти записи DNS (A, AAAA, CNAME, MX, TXT) зі свого Dashboard
* **Статус домену** — Статус реєстрації, дати закінчення терміну дії та інформація про nameserver у реальному часі
* **Автоматична прив’язка домену** — Зареєстровані або наявні домени автоматично прив’язуються до сайту клієнта

#### Інструменти адміністратора
* **Ручна реєстрація домену** — Реєструйте домени від імені клієнтів з панелі адміністратора
* **Моніторинг доменів** — Переглядайте всі домени клієнтів, статуси та дати закінчення терміну дії на одній сторінці
* **Журналювання для окремих провайдерів** — Окремі канали журналів для кожного реєстратора (наприклад, domain-seller-namecheap.log)
* **Деталі реєстрації** — Переглядайте інформацію про реєстранта та дані реєстрації на основній сторінці редагування домену

### Підтримувані провайдери

* **OpenSRS** — Повна інтеграція з реєстрацією, поновленням, керуванням DNS і конфіденційністю WHOIS
* **Namecheap** — Повна інтеграція з реєстрацією, поновленням, пакетною перевіркою доступності та безкоштовним WhoisGuard
* **HostAfrica** — Реєстрація, поновлення, перенесення, керування DNS, отримання коду EPP, блокування реєстратора та захист ID
* **Openprovider** — Реєстрація, поновлення, перенесення, керування DNS, отримання коду EPP, блокування реєстратора, конфіденційність WHOIS і синхронізація реселерських TLD
* **Hostinger** — Реєстрація, пошук домену, оновлення nameserver, блокування реєстратора та конфіденційність WHOIS; поновлення обробляються автопоновленням Hostinger
* **GoDaddy** — Реєстрація та поновлення
* **ResellerClub** — Реєстрація, поновлення та керування DNS
* **NameSilo** — Реєстрація та поновлення
* **Enom** — Реєстрація, поновлення та керування DNS

### Технічні деталі

* **Потребує:** PHP 7.4+, Multisite Ultimate 2.4.12+
* **Архітектура:** шаблон Integration Registry з інтерфейсом Domain_Selling_Capability
* **Автозавантаження:** автозавантажувач Jetpack із зіставленням класів PSR-4
* **Стандарти:** стандарти кодування WordPress, сувора типізація, підготовлені запити

## Встановлення

1. Переконайтеся, що **Multisite Ultimate v2.4.12+** встановлено та активовано в мережі
2. Завантажте плагін до `/wp-content/plugins/ultimate-multisite-domain-seller/`
3. Активуйте плагін у мережі з розділу **Плагіни** в WordPress Network Admin
4. **Майстер налаштування** запускається автоматично під час першої активації — виконайте кроки, щоб:
   * Підключити вашого реєстратора доменів
   * Перевірити з’єднання
   * Імпортувати доступні TLD та оптові ціни
   * Створити ваш перший доменний продукт із ціновою націнкою
5. Додайте поле **Вибір домену** до вашої форми оформлення замовлення в **Ultimate Multisite > Форми оформлення замовлення**

## Часті запитання

### Яка версія Multisite Ultimate потрібна?

Версія 2.4.12 або новіша. Додаток несумісний із WP Ultimo v1.x або версіями Ultimate Multisite нижче 2.4.12.

### Які провайдери доменів підтримуються?

Підтримуються дев’ять реєстраторів: **OpenSRS**, **Namecheap**, **HostAfrica**, **Openprovider**, **Hostinger**, **GoDaddy**, **ResellerClub**, **NameSilo** і **Enom**. Можливості відрізняються залежно від провайдера, оскільки API реєстраторів надають різні операції; див. посібник із налаштування для огляду по кожному провайдеру. Архітектура Integration Registry цього додатка спрощує додавання власних провайдерів — див. [посібник для розробників](/developer/integration-guide/custom-registrar).

### Чи можу я пропонувати захист приватності WHOIS?

Так. Кожен доменний продукт має налаштування приватності WHOIS із трьома режимами: вимкнено, завжди включено або вибір клієнта. Якщо встановлено «Вибір клієнта», клієнти бачать прапорець під час оформлення замовлення з ціною (або "Free" для WhoisGuard від Namecheap). Прапорець приватності автоматично передається реєстратору під час реєстрації.

### Чи можу я встановити різні ціни для різних TLD?

Так. Створюйте окремі доменні продукти для різних груп TLD. Кожен продукт може мати власний тип націнки (відсоток, фіксована націнка або фіксована ціна), підтримувані TLD та вступну ціну.

### Які варіанти оформлення замовлення бачать клієнти?

Поле «Вибір домену» надає клієнтам три вкладки: **Субдомен** (використати безкоштовний субдомен у вашій мережі), **Зареєструвати новий домен** (знайти та зареєструвати новий домен) і **Наявний домен** (прив’язати домен, яким вони вже володіють). Відображатимуться лише режими, які ви ввімкнете.

### Чи підтримує це автоматичні продовження?

Так. Домени можна налаштувати на автопродовження, і вони прив’язані до статусу членства клієнта. Платежі за продовження обробляються через наявний платіжний шлюз клієнта.

### Чи можуть клієнти керувати своїми DNS-записами?

Так. Клієнти отримують інтерфейс керування DNS у своєму dashboard для додавання, редагування та видалення записів A, AAAA, CNAME, MX і TXT. Керування DNS доступне з OpenSRS, ResellerClub, Enom, HostAfrica і Openprovider. DNS Hostinger для розміщених доменів обробляється основною інтеграцією зіставлення доменів Hostinger.

### Чи є майстер налаштування?

Так. Майстер налаштування запускається автоматично під час першої активації та проводить вас через підключення реєстратора, імпорт TLD і створення вашого першого доменного продукту. Ви також можете повторно запустити майстер із налаштувань додатка.
