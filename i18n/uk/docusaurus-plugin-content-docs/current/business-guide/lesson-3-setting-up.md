# Урок 3: Налаштування вашої мережі

Настав час будувати. У цьому уроці ви встановите Ultimate Multisite та налаштуєте основу мережі FitSite. Кожне рішення тут приймається з урахуванням ніші фітнесу.

## З чого ми почали {#where-we-left-off}

Ми обрали фітнес-студії як нашу нішу та підтвердили цю можливість. Тепер ми перетворюємо цю ідею на робочу платформу.

## Вибір хостингу {#choosing-your-hosting}

Вабір хостингу має значення для нішевої платформи більше, ніж для одного вебсайту. Ви не розміщуєте один сайт — ви розміщуєте мережу, яка може вирости до десятків чи сотень сайтів.

### На що звернути увагу {#what-to-look-for}

- **Підтримка WordPress Multisite**: Не всі хости добре працюють з multisite
- **Wildcard SSL**: Життєво необхідний для мереж на основі піддоменів
- **Масштабовані ресурси**: Вам потрібен запас для зростання без необхідності міграції
- **Інтеграція Ultimate Multisite**: Автоматичне відображення доменів та SSL значно економить операційні зусилля

### Рекомендований підхід {#recommended-approach}

Оберіть хостинг із списку [Сумісних провайдерів](/user-guide/host-integrations/closte). Вони були протестовані з Ultimate Multisite та надають інтеграції, необхідні для автоматичного відображення доменів та SSL.

Для FitSite ми використаємо конфігурацію піддоменів. Це означає, що сайти клієнтів спочатку виглядатимуть як `studioname.fitsite.com`, перш ніж вони опціонально прив'яжуть власний домен.

## Встановлення WordPress Multisite {#installing-wordpress-multisite}

Якщо у вас ще немає встановленого WordPress Multisite:

1. Встановіть WordPress у вашого хостинг-провайдера
2. Дотримуйтесь інструкцій у [Як встановити WordPress Multisite](/user-guide/getting-started/how-to-install-wordpress-multisite)
3. Оберіть конфігурацію **піддоменів** (subdomain) за запитом

:::tip Чому піддомени?
Піддомени надають кожному сайту клієнта власну унікальну адресу (`studio.fitsite.com`), а не шлях (`fitsite.com/studio`). Це виглядає професійніше для ваших клієнтів і запобігає конфліктам пермалінків. Детальніше порівняння дивіться у [Ultimate Multisite 101](/user-guide/getting-started/ultimate-multisite-101).
:::

## Встановлення Ultimate Multisite {#installing-ultimate-multisite}

Дотримуйтесь інструкцій у [Встановлення Ultimate Multisite](/user-guide/getting-started/installing-ultimate-multisite), щоб:

1. Завантажити та активувати плагін на рівні всієї мережі
2. Запустити [Майстер налаштування](/user-guide/getting-started/multisite-setup-wizard)

Під час роботи з майстром налаштування пам'ятайте про нішу FitSite:

- **Валюта (Currency)**: Встановіть валюту, яку використовують ваші клієнти-власники фітнес-студій
- **Назва компанії (Company name)**: "FitSite" (або назва вашого бренду)
- **Логотип компанії (Company logo)**: Завантажте логотип вашого бренду — він з'явиться у рахунках-фактурах та електронних листах

## Налаштування для ніші фітнесу {#configuring-for-the-fitness-niche}

Після встановлення Ultimate Multisite виконайте такі налаштування, специфічні для ніші:

### Загальні налаштування (General Settings) {#general-settings}

Перейдіть до **Ultimate Multisite > Settings** та налаштуйте:

- **Назва сайту (Site name)**: FitSite
- **Роль за замовчуванням (Default role)**: Administrator — власники фітнес-студій потребують повного контролю над контентом свого сайту
- **Реєстрація (Registration)**: Увімкніть реєстрацію користувачів, щоб власники студій могли зареєструватися самостійно

### Налаштування електронної пошти (Email Configuration) {#email-configuration}

Ваші системні електронні листи мають говорити мовою вашої ніші. Перейдіть до **Ultimate Multisite > Settings > Emails** та налаштуйте:

- Замініть загальні фрази "ваш новий сайт" на повідомлення, специфічні для фітнесу
- Приклад теми вітання: "Ваш сайт фітнес-студії готовий"
- Приклад тіла вітання: Згадайте їхню студію, класи та початок роботи з їхнім фітнес-сайтом

Ми деталізуємо це пізніше, в Уроці 8 (Онбординг клієнтів), але встановлення тону зараз гарантує, що навіть ранні тестові реєстрації будуть нішево-специфічними.

### Налаштування доменів (Domain Configuration) {#domain-configuration}

Якщо ви використовуєте сумісний хостинг-провайдер, налаштуйте відображення доменів зараз:

1. Перейдіть до **Ultimate Multisite > Settings > Domain Mapping**
2. Дотримуйтесь інструкцій інтеграції для вашого конкретного хоста
3. Перевірте, що нові підсайти автоматично отримують SSL

Це гарантує, що коли ми почнемо створювати шаблони та тестові сайти в наступному уроці, все працюватиме від початку до кінця.

## Мережа FitSite наразі {#the-fitsite-network-so-far}

Наприкінці цього уроку ви маєте:

```
FitSite Network
├── WordPress Multisite (режим піддоменів)
├── Ultimate Multisite (встановлено та налаштовано)
├── Хостинг із Wildcard SSL
├── Налаштоване відображення доменів
├── Нішево-специфічні шаблони електронної пошти (початкові)
└── Готовність до шаблонів сайтів (наступний урок)
```

## Що ми створили в цьому уроці {#what-we-built-this-lesson}

- **Робоча інсталяція WordPress Multisite** у режимі піддоменів
- **Встановлений та налаштований Ultimate Multisite** з брендингом FitSite
- **Хостинг та SSL** налаштовані для зростаючої мережі
- **Відображення доменів** налаштовано для вашого хостинг-провайдера
- **Нішево-специфічний тон електронної пошти** встановлено з першого дня
