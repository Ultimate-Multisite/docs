# Урок 6: Досвід реєстрації {#lesson-6-the-signup-experience}

Процес оформлення замовлення — це місце, де інтерес перетворюється на дохід. Заплутаний або загальний процес реєстрації втрачає клієнтів. Нішево-орієнтований процес, який говорить їхньою мовою та здається легким, конвертує їх.

## З чого ми зупинилися {#where-we-left-off}

FitSite має налаштовані шаблони та плани. Тепер ми створюємо процес реєстрації та оформлення замовлення, який перетворює власників фітнес-студій на платячих клієнтів.

## Розуміння потоку реєстрації {#understanding-the-registration-flow}

Форми оформлення замовлення Ultimate Multisite — це повністю налаштовувані багатоетапні форми. Для FitSite нам потрібен потік, який:

1. Здається, що він створений спеціально для фітнес-бізнесу
2. Збирає лише необхідну інформацію
3. Якнайшвидше доводить клієнта до робочого сайту

Дивіться [The Registration Flow](/user-guide/configuration/the-registration-flow) для повного технічного посилання.

## Дизайн оформлення замовлення FitSite {#designing-the-fitsite-checkout}

Перейдіть до **Ultimate Multisite > Checkout Forms** та створіть нову форму.

### Крок 1: Вибір плану {#step-1-plan-selection}

Перше, що має побачити власник фітнес-студії, — це плани, представлені такими термінами, які він розуміє.

- Додайте поле **Pricing Table**
- Налаштуйте його так, щоб відображалися всі три плани FitSite
- Описи планів, які ви написали в Уроці 5, з'являються тут — переконайтеся, що вони звертаються до потреб фітнес-бізнесу, а не до технічних функцій

:::tip Важлива нішева мова
"1 ГБ сховища" нічого не означає для власника спортзалу. "Все, що потрібно для професійного сайту студії" означає все. Пишіть описи планів мовою вашого клієнта.
:::

### Крок 2: Вибір шаблону {#step-2-template-selection}

Після вибору плану клієнт обирає свій стартовий шаблон.

- Додайте поле **Template Selection**
- Доступні шаблони фільтруються за обраним планом (налаштовано в Уроці 5)
- Кожен шаблон повинен мати попередній перегляд зображення, що демонструє дизайн, специфічний для фітнесу

### Крок 3: Створення облікового запису {#step-3-account-creation}

Зробіть це мінімальним. Збирайте лише:

- Адреса електронної пошти
- Пароль
- Назва студії/бізнесу (це стає назвою їхнього сайту)

Не просіть інформацію, яка вам не потрібна при реєстрації. Кожне додаткове поле зменшує конверсію.

### Крок 4: Налаштування сайту {#step-4-site-setup}

- **Site title**: Автозаповнення з назви студії, вказаної на Кроці 3
- **Site URL**: Автогенерація з назви студії (наприклад, `ironworks.fitsite.com`)

### Крок 5: Оплата {#step-5-payment}

- Додайте поле **Payment**
- Налаштуйте ваш платіжний шлюз ([Stripe](/user-guide/payment-gateways/stripe) рекомендується для білінгу підписки)
- Якщо ви створювали доповнення до замовлення (order bumps) в Уроці 5, додайте поле **Order Bump** перед кроком оплати

### Крок 6: Підтвердження {#step-6-confirmation}

- Налаштуйте повідомлення підтвердження фітнес-специфічною мовою
- Приклад: "Ваш сайт фітнес-студії створюється. Ви будете перенаправлені на ваш новий сайт через кілька секунд."

## Додавання перемикача вибору періоду {#adding-a-period-selection-toggle}

Якщо ви налаштували варіації цін у своїх планах (щомісячно проти щорічно), додайте поле **Period Selection** до форми оформлення замовлення, щоб клієнти могли перемикатися між періодами виставлення рахунків. Дивіться [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle) для інструкцій.

## Налаштування оплати {#configuring-payment}

Якщо ви ще не налаштували платіжний шлюз:

1. Перейдіть до **Ultimate Multisite > Settings > Payment Gateways**
2. Дотримуйтесь [посібника зі встановлення Stripe](/user-guide/payment-gateways/stripe) або свого обраного шлюзу
3. Протестуйте повний процес оформлення замовлення за допомогою тестової оплати

Дивіться [Getting Paid](/user-guide/payment-gateways/getting-paid) для деталей про те, як платежі надходять на ваш рахунок.

## Тестування потоку {#testing-the-flow}

Перш ніж рухатися далі, виконайте повне тестове оформлення замовлення:

1. Відкрийте форму оформлення замовлення у вікні інкогніто/приватному браузері
2. Виберіть план
3. Оберіть шаблон
4. Створіть обліковий запис
5. Завершіть оплату (використовуйте тестовий режим)
6. Перевірте, чи створено сайт із правильним шаблоном

Перевірте, що:

- [ ] Описи планів чіткі та специфічні для ніші
- [ ] Попередній перегляд шаблонів демонструє дизайни, відповідні фітнесу
- [ ] URL-адреса сайту генерується правильно з назви студії
- [ ] Оплата проходить успішно
- [ ] Клієнт потрапляє на робочий сайт із вибраним шаблоном
- [ ] Електронні листи підтвердження використовують фітнес-специфічну мову

## Мережа FitSite наразі {#the-fitsite-network-so-far}

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow
│   ├── Plan selection with niche-specific descriptions
│   ├── Template selection with fitness previews
│   ├── Minimal account creation
│   ├── Payment via Stripe
│   └── Fitness-specific confirmation
└── Ready for branding (next lesson)
```

## Що ми створили в цьому уроці {#what-we-built-this-lesson}

- **Багатоетапна форма оформлення замовлення**, адаптована для власників фітнес-студій
- **Нішево-специфічна мова** протягом усього потоку реєстрації
- **Мінімальне тертя** — лише необхідні поля, швидкий шлях до робочого сайту
- **Інтеграція оплати** з тестовою перевіркою
- **Протестований потік від початку до кінця** — від вибору плану до робочого сайту
