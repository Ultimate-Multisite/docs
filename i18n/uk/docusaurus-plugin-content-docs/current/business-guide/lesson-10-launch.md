# Урок 10: День запуску {#lesson-10-launch-day}

Все створено. Перш ніж відкрити двері, цей урок проходить перелік перевірок перед запуском, щоб переконатися, що нічого не зламано, не випало і не викличе ганьби.

## З чого ми зупинилися {#where-we-left-off}

FitSite має шаблони, плани, оформлення замовлення, брендинг, онбординг та ціноутворення — усе налаштовано. Тепер ми перевіряємо, чи все працює, і виходимо в ефір.

## Чекліст перед запуском {#pre-launch-checklist}

Пройдіть кожний пункт. Не пропускайте жодного.

### Інфраструктура платформи {#platform-infrastructure}

- [ ] Хостинг стабільний і добре працює під навантаженням
- [ ] Wildcard SSL активний, і всі піддомени обслуговуються через HTTPS
- [ ] Мапування доменів працює — протестуйте створення сайту та мапування користувацького домену
- [ ] Резервні копії налаштовані та протестовані (відновіть принаймні одну, щоб перевірити)
- [ ] Моніторинг на місці — ви будете знати, якщо платформа вийде з ладу

### Шаблони {#templates}

- [ ] Усі три шаблони коректно завантажуються на нових сайтах
- [ ] Заповнювальний контент корисний і не містить помилок
- [ ] Усі зображення належним чином ліцензовані (без водяних знаків стокових фото)
- [ ] Адаптивність для мобільних пристроїв працює на кожній сторінці шаблону
- [ ] Швидкість завантаження сторінки прийнятна (протестуйте за допомогою інструменту, як-от GTmetrix або PageSpeed Insights)
- [ ] На жодному шаблоні немає зламаних посилань чи відсутніх ресурсів

### Плани та продукти {#plans-and-products}

- [ ] Усі три плани активні та видимі
- [ ] Описи планів точні та специфічні для ніші
- [ ] Ціноутворення правильне (щомісячно та щорічно)
- [ ] Встановлювальні збори налаштовані для правильних планів
- [ ] Випробуційний період працює для плану Starter
- [ ] Додаткові товари (Order bumps) коректно відображаються під час оформлення замовлення
- [ ] Обмеження за планом для плагінів та тем правильно застосовані

### Потік оформлення замовлення (Checkout Flow) {#checkout-flow}

- [ ] Проведіть повне тестове реєстрацію для кожного плану (використовуйте тестовий режим оплати)
- [ ] Вибір шаблону показує правильні шаблони для кожного плану
- [ ] Оплата проходить успішно
- [ ] Клієнт отримує вітальне email після реєстрації
- [ ] Створюється новий сайт із правильним шаблоном
- [ ] Клієнт може негайно увійти на свій новий сайт
- [ ] Коди знижок працюють коректно

### Брендинг {#branding}

- [ ] Сторінка входу відображає брендинг FitSite
- [ ] Адмін-панель (Admin dashboard) відображає брендинг FitSite
- [ ] Усі системні email використовують брендинг FitSite та мову, специфічну для фітнесу
- [ ] Рахунки-фактури відображаються правильно з вашими бізнес-даними
- [ ] Маркетинговий сайт активний і посилається на форму оформлення замовлення

### Онбординг {#onboarding}

- [ ] Віджет Quick Start з'являється на панелях управління нових клієнтів
- [ ] Усі посилання Quick Start ведуть на правильні сторінки
- [ ] Послідовність вітальних email налаштована та протестована
- [ ] Статті знань (Knowledge base articles) опубліковані та доступні
- [ ] Сторінка облікового запису відображає правильну інформацію про план та опції оновлення

### Юридичні та бізнес-питання {#legal-and-business}

- [ ] Умови обслуговування опубліковані та посилання на них є в checkout
- [ ] Політика конфіденційності опублікована та доступна
- [ ] Політика повернення коштів визначена та документована
- [ ] Бізнес-суб'єкт створений для отримання платежів
- [ ] Платіжний шлюз перебуває в режимі live (не в тестовому)
- [ ] Конфігурація податків правильна для вашої юрисдикції

## М'який запуск vs. Повномасштабний запуск {#soft-launch-vs-hard-launch}

Розгляньте запуск у два етапи:

### Етап 1: М'який запуск (Soft Launch) {#phase-1-soft-launch}

Запросіть 5–10 власників фітнес-студій для реєстрації до публічного запуску. Це ваші бета-клієнти. Запропонуйте їм значну знижку (знижка 50% на все життя або 3 місяці безкоштовно) в обмін на:

- Чесний відгук про процес реєстрації та онбордингу
- Дозвіл використовувати їхній сайт як демонстраційний приклад
- Звіт про будь-які помилки чи проблеми, які вони виявляють

Це дає вам реальний відгук від клієнтів та живі сайти для демонстрації перед публічним відкриттям.

### Етап 2: Повномасштабний запуск (Public Launch) {#phase-2-public-launch}

Після інтеграції відгуків з м'якого запуску:

- Переведіть платіжний шлюз у режим live
- Опублікуйте свій маркетинговий сайт
- Почніть залучення клієнтів (Урок 11)
- Оголосіть у відповідних каналах фітнес-індустрії

## Дії в день запуску {#launch-day-actions}

У день, коли ви виходите в публічний доступ:

1. **Переключіться на реальні платежі** — вимкніть тестовий режим на платіжному шлюзі
2. **Перевірте ще раз** — проведіть повну тестову реєстрацію з реальними платежами (повернуться самі)
3. **Спостерігайте уважно** — стежте за помилками, невдалими реєстраціями чи проблемами з оплатою
4. **Будьте доступні** — вашим першим реальним клієнтам може знадобитися допомога, а швидка відповідь будує довіру
5. **Відсвяткуйте коротко** — а потім поверніться до роботи

## Що може піти не так {#what-can-go-wrong}

Будьте готові до:

- **Проблеми з платіжним шлюзом**: Тримайте під рукою контакт підтримки вашого провайдера шлюзу
- **Проблеми з SSL-сертифікатом**: Знайте, як перевіряти та поновлювати сертифікати
- **Збої доставки email**: Протестуйте, що email дійсно надходять (перевірте папки спаму)
- **Продуктивність під навантаженням**: Якщо ви отримаєте сплеск трафіку, знайте, як масштабувати хостинг
- **Плутанина клієнтів**: Тримайте готові вашу базу знань та канали підтримки
