# Мультитенант Hostinger

Ultimate Multisite: Multi-Tenancy 1.2.0 додає функціональність мультитенант Hostinger, що дозволяє розміщеним доменам брати участь у наданні суверенних арендаторів паралельно з існуючою інтеграцією мапінгу доменів Hostinger.

Використовуйте цю можливість, коли ви керуєте доменами орендодавців та ізольованою інфраструктурою орендарів через hPanel Hostinger.

## Примітки щодо налаштування

1. Налаштуйте основну інтеграцію Hostinger у розділі **Ultimate Multisite > Settings > Host Integrations**.
2. Переконайтеся, що токен API Hostinger може керувати цільовим доменом або піддоменом.
3. Увімкніть додаток Multi-Tenancy.
4. Налаштуйте стратегію ізоляції орендарів до публікації орендаря.
5. Запустіть робочий процес перевірки міграції перед відправкою продакшн-трафіку на орендаря.

Функціональність Hostinger використовує спільне з'єднання Hostinger для операцій на стороні хоста. DNS все одно має вказувати на правильний обліковий запис Hostinger, а обмеження облікового запису hPanel залишаються чинними.

## Зміни залежно від функціональності

- Суверенних орендарів можна створювати з доменами, що враховують хост.
- Рядки для хостів баз даних на одній машині нормалізуються перед підтвердженням прав доступу.
- Орендарі, керовані Hostinger, повинні використовувати значення хоста бази даних, показане в hPanel, якщо середовище WordPress не вимагає локального перевизначення.
- Відвідування через SSO залежить від того, що домен орендаря розв'язується на хостинговий орендар Hostinger.

## Усунення несправностей з орендарями Hostinger

- Якщо встановлення орендаря не вдалося, перевірте, чи додано домен вже до облікового запису Hostinger.
- Якщо підтвердження бази даних не вдається, порівняйте ім'я користувача БД орендаря, назву бази даних та прив'язку хоста з hPanel.
- Якщо **Visit (SSO)** не працює, переконайтеся, що DNS та SSL активні для домену орендаря.
- Якщо ви видаляєте ресурси хоста після підтвердження резервних копій, видаліть будь-які залишки баз даних, користувачів або папок з hPanel.
