# Налаштування ручних платежів (v2)

_**ВАЖЛИВА ПРИМІТКА: Ця стаття стосується Ultimate Multisite версії 2.x.**_

Ручні платежі дозволяють пропонувати альтернативні способи оплати, якщо **Stripe** або **PayPal** недоступні для ваших користувачів. Це може бути банківський переказ або будь-який інший спосіб оплати, доступний вашим користувачам локально.

## Як увімкнути ручні платежі

Налаштувати ручні платежі дуже просто. Вам потрібно лише увімкнути цю опцію в розділі платіжних шлюзів і додати детальні інструкції щодо того, як користувач має здійснити оплату.

Спочатку перейдіть до **Ultimate Multisite > Settings > Payments**. У розділі **Payment Gateways** увімкніть перемикач **Manual**. Після цього з'явиться поле **Payment Instructions**.

Додайте в це поле інформацію, яка знадобиться вашому клієнту для здійснення оплати. Наприклад, це можуть бути реквізити вашого банківського рахунку та електронна пошта, щоб клієнт міг надіслати вам підтвердження оплати.

![Перемикач ручного платіжного шлюзу та поле з інструкціями щодо оплати](/img/config/settings-payment-gateways.png)

Після налаштування просто натисніть **Save Settings** — і готово. Коли користувачі реєструватимуться у вашій мережі, вони побачать повідомлення про те, що отримають ваші інструкції для завершення покупки.

![Повідомлення про ручну оплату під час реєстрації](/img/config/settings-payment-gateways.png)

Також вони отримають повідомлення з вашими інструкціями щодо оплати на сторінці **Thank You**.

![Сторінка Thank You з інструкціями щодо оплати](/img/config/settings-payment-gateways.png)

## Підтвердження ручних платежів

Щоб підтвердити ручний платіж, перейдіть до меню **Payments** на лівій панелі. Там ви побачите всі платежі у вашій мережі та їхні деталі, зокрема **статус**. Ручний платіж завжди матиме статус **Pending**, доки ви не зміните його вручну.

![Список платежів із ручним платежем у статусі очікування](/img/admin/payments-list.png)

Перейдіть на сторінку платежу, натиснувши на **reference code**. На цій сторінці ви знайдете всі деталі платежу, що очікує: reference ID, продукти, часові позначки тощо.

![Сторінка деталей платежу з reference code та продуктами](/img/admin/payments-list.png)

У правій колонці ви можете змінити статус платежу. Якщо змінити його на **Completed** і **увімкнути опцію Activate Membership**, сайт вашого клієнта буде активовано, а його членство стане активним.

![Зміна статусу платежу на Completed з перемикачем Activate Membership](/img/admin/payments-list.png)
