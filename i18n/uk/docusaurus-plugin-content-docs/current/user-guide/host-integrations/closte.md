# Інтеграція з Closte {#closte-integration}

## Огляд {#overview}
Closte — це керований хостинг для WordPress, побудований на інфраструктурі Google Cloud. Ця інтеграція забезпечує автоматичну синхронізацію доменів та керування SSL-сертифікатами між Ultimate Multisite і Closte.

## Можливості {#features}
- Автоматична синхронізація доменів
- Керування SSL-сертифікатами
- Підтримка wildcard-доменів
- Не потребує налаштування, якщо сайт працює на Closte

## Вимоги {#requirements}
Якщо ви використовуєте Closte, у файлі `wp-config.php` має бути визначена така константа:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

Зазвичай ця константа вже визначена, якщо ваш сайт розміщено на Closte.

## Інструкція з налаштування {#setup-instructions}

### 1. Перевірте API-ключ Closte {#1-verify-your-closte-api-key}

Якщо ваш сайт розміщено на Closte, константа `CLOSTE_CLIENT_API_KEY` вже має бути визначена у файлі `wp-config.php`. Щоб переконатися в цьому, перевірте вміст файлу `wp-config.php`.

### 2. Увімкніть інтеграцію {#2-enable-the-integration}

1. В адмінпанелі WordPress перейдіть до Ultimate Multisite > Settings
2. Відкрийте вкладку "Domain Mapping"
3. Прокрутіть до розділу "Host Integrations"
4. Увімкніть інтеграцію з Closte
5. Натисніть "Save Changes"

## Як це працює {#how-it-works}

Коли домен прив'язується в Ultimate Multisite:

1. Інтеграція надсилає запит до API Closte, щоб додати домен до вашого застосунку
2. Closte автоматично випускає SSL-сертифікат
3. Коли прив'язку домену видаляють, інтеграція видаляє домен із Closte

Інтеграція також працює з налаштуванням інтервалу перевірки DNS в Ultimate Multisite, що дозволяє вказати, як часто система перевіряє поширення DNS та випуск SSL-сертифіката.

## Створення записів домену {#domain-record-creation}

Ця інтеграція гарантує, що під час створення або дублювання сайту автоматично створюється запис домену. Це особливо важливо для інтеграції з Closte, оскільки створення запису домену запускає API Closte для створення домену та SSL-сертифіката.

## Вирішення проблем {#troubleshooting}

### Проблеми з підключенням до API {#api-connection-issues}
- Переконайтеся, що ваш API-ключ Closte правильний
- Перевірте, чи має ваш обліковий запис Closte необхідні дозволи

### Проблеми з SSL-сертифікатом {#ssl-certificate-issues}
- Closte може потребувати деякий час для випуску SSL-сертифікатів (зазвичай 5–10 хвилин)
- Переконайтеся, що ваші домени правильно вказують на IP-адресу вашого сервера Closte
- Перевірте DNS-записи вашого домену, щоб переконатися в їх правильності

### Домен не додано {#domain-not-added}
- Перевірте логи Ultimate Multisite на наявність повідомлень про помилки
- Переконайтеся, що домен ще не додано до Closte
- Перевірте, чи правильно налаштовані DNS-записи вашого домену

### Інтервал перевірки DNS {#dns-check-interval}
- Якщо випуск SSL-сертифікатів займає надто багато часу, ви можете змінити інтервал перевірки DNS у налаштуваннях Domain Mapping
- Типовий інтервал — 300 секунд (5 хвилин), але для швидшої перевірки під час тестування його можна зменшити до 10 секунд
