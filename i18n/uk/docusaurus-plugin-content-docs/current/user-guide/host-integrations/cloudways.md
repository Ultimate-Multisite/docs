# Інтеграція Cloudways

## Огляд
Cloudways — це керована платформа хмарного хостингу, яка дає змогу розгортати WordPress сайти на різних хмарних провайдерах, таких як DigitalOcean, AWS, Google Cloud тощо. Ця інтеграція забезпечує автоматичну синхронізацію доменів і керування SSL-сертифікатами між Ultimate Multisite та Cloudways.

## Можливості
- Автоматична синхронізація доменів
- Керування SSL-сертифікатами
- Підтримка додаткових доменів
- DNS-перевірка для SSL-сертифікатів

## Вимоги
У вашому файлі `wp-config.php` мають бути визначені такі константи:

```php
define('WU_CLOUDWAYS_EMAIL', 'your_cloudways_email');
define('WU_CLOUDWAYS_API_KEY', 'your_api_key');
define('WU_CLOUDWAYS_SERVER_ID', 'your_server_id');
define('WU_CLOUDWAYS_APP_ID', 'your_app_id');
```

За бажанням ви також можете визначити:

```php
define('WU_CLOUDWAYS_EXTRA_DOMAINS', 'comma,separated,list,of,domains');
```

## Інструкції з налаштування

### 1. Отримайте свої облікові дані Cloudways API

1. Увійдіть до своєї панелі Cloudways
2. Перейдіть до "Account" > "API Keys"
3. Згенеруйте API-ключ, якщо у вас його ще немає
4. Скопіюйте свою електронну адресу та API-ключ

### 2. Отримайте ідентифікатори сервера та застосунку

1. У своїй панелі Cloudways перейдіть до "Servers"
2. Виберіть сервер, на якому розміщено ваш WordPress мультисайт
3. Server ID відображається в URL: `https://platform.cloudways.com/server/{SERVER_ID}`
4. Перейдіть до "Applications" і виберіть свій WordPress застосунок
5. App ID відображається в URL: `https://platform.cloudways.com/server/{SERVER_ID}/application/{APP_ID}`

### 3. Додайте константи до wp-config.php

Додайте такі константи до свого файлу `wp-config.php`:

```php
define('WU_CLOUDWAYS_EMAIL', 'your_cloudways_email');
define('WU_CLOUDWAYS_API_KEY', 'your_api_key');
define('WU_CLOUDWAYS_SERVER_ID', 'your_server_id');
define('WU_CLOUDWAYS_APP_ID', 'your_app_id');
```

Якщо у вас є додаткові **зовнішні** домени (поза вашою мережею мультисайту), які завжди мають залишатися у списку псевдонімів Cloudways:

```php
define('WU_CLOUDWAYS_EXTRA_DOMAINS', 'extradomain1.com,extradomain2.com');
```

:::warning Не включайте wildcard власної мережі
**Не** додавайте `*.your-network.com` (або будь-який шаблон піддоменів власної мережі) до
`WU_CLOUDWAYS_EXTRA_DOMAINS`. Див. [Важливо — пастка wildcard SSL](#important--wildcard-ssl-pitfall)
нижче, щоб дізнатися, чому це заважає видачі SSL-сертифікатів для окремих орендарів.
:::

### 4. Увімкніть інтеграцію

1. У вашій адмінпанелі WordPress перейдіть до Ultimate Multisite > Налаштування
2. Перейдіть на вкладку "Зіставлення доменів"
3. Прокрутіть униз до "Інтеграції хостингу"
4. Увімкніть інтеграцію Cloudways
5. Натисніть "Зберегти зміни"

## Як це працює

### Синхронізація доменів

Коли домен зіставляється в Ultimate Multisite:

1. Інтеграція отримує всі наразі зіставлені домени
2. Вона додає новий домен до списку (разом із версією www, якщо застосовно)
3. Вона надсилає повний список до Cloudways через API
4. Cloudways оновлює псевдоніми доменів для вашого застосунку

Примітка: Cloudways API вимагає надсилати повний список доменів щоразу, а не просто додавати чи видаляти окремі домени.

### Керування SSL-сертифікатами

Після синхронізації доменів:

1. Інтеграція перевіряє, які домени мають дійсні DNS-записи, що вказують на ваш сервер
2. Вона надсилає запит до Cloudways на встановлення SSL-сертифікатів Let's Encrypt для цих доменів
3. Cloudways обробляє випуск і встановлення SSL-сертифікатів

Інтеграція завжди запитує **стандартні** (не wildcard) сертифікати Let's Encrypt у
Cloudways. Якщо в `WU_CLOUDWAYS_EXTRA_DOMAINS` передано шаблон wildcard, початковий
`*.` видаляється перед SSL-запитом — сам wildcard ніколи не встановлюється цією
інтеграцією. Щоб використовувати wildcard-сертифікат на Cloudways, вам доведеться встановити його
вручну, але це блокує видачу Let's Encrypt для кожного домену зіставлених користувацьких доменів
(див. пастку нижче).

## Додаткові домени

Константа `WU_CLOUDWAYS_EXTRA_DOMAINS` дає змогу вказати додаткові **зовнішні**
домени, які завжди мають залишатися у списку псевдонімів застосунку Cloudways. Використовуйте її для:

- Зовнішніх доменів, якими не керує Ultimate Multisite (наприклад, окремий маркетинговий сайт, що використовує той самий застосунок Cloudways)
- Припаркованих або staging-доменів, які ви хочете залишати у списку псевдонімів застосунку

**Не** використовуйте цю константу для wildcard піддоменів власної мережі
(наприклад, `*.your-network.com`). Див. пастку wildcard SSL нижче.

## Важливо — пастка wildcard SSL

Поширена помилка під час дотримання стандартного налаштування Cloudways — додати wildcard на кшталт
`*.your-network.com` до `WU_CLOUDWAYS_EXTRA_DOMAINS` або вручну встановити Cloudways
wildcard SSL-сертифікат для цього wildcard.

**Якщо ви це зробите, Cloudways відмовиться видавати сертифікати Let's Encrypt для
користувацьких доменів окремих орендарів, які зіставляє Ultimate Multisite.** Cloudways замінює активний
SSL-сертифікат у застосунку щоразу, а вже наявний wildcard-сертифікат у
застосунку блокує видачу Let's Encrypt для кожного домену, на яку спирається інтеграція.

### Рекомендоване налаштування Cloudways SSL для мережі Ultimate Multisite

1. На вкладці **SSL Certificate** застосунку Cloudways встановіть **стандартний
   сертифікат Let's Encrypt**, що покриває лише `your-network.com` і `www.your-network.com`
   — **не** wildcard.
2. **Не** додавайте `*.your-network.com` (або будь-який шаблон піддоменів власної мережі) до
   `WU_CLOUDWAYS_EXTRA_DOMAINS`. Зарезервуйте цю константу лише для **зовнішніх** доменів.
3. Створіть wildcard піддоменів для окремих орендарів лише на рівні **DNS** (запис `A` для
   `*.your-network.com`, що вказує на IP-адресу вашого сервера Cloudways), щоб підсайти розв’язувалися. SSL
   для окремих зіставлених користувацьких доменів потім автоматично випускається інтеграцією
   через Let's Encrypt.

Якщо власні домени ваших орендарів зависли без SSL, перевірте вкладку SSL у Cloudways. Якщо там активний
wildcard-сертифікат, видаліть його, повторно випустіть стандартний сертифікат Let's Encrypt
лише для основного домену мережі та видаліть будь-які wildcard-записи з
`WU_CLOUDWAYS_EXTRA_DOMAINS`. Потім повторно запустіть зіставлення домену (або дочекайтеся наступного),
і інтеграція знову почне випускати сертифікати для кожного домену.

## Усунення несправностей

### Проблеми з підключенням API
- Переконайтеся, що ваша електронна пошта та API key правильні
- Перевірте, що ваші ідентифікатори сервера й застосунку правильні
- Переконайтеся, що ваш обліковий запис Cloudways має необхідні дозволи

### Проблеми із сертифікатами SSL
- Cloudways вимагає, щоб домени мали дійсні DNS-записи, які вказують на ваш сервер, перед випуском SSL-сертифікатів
- Інтеграція перевіряє DNS-записи перед запитом SSL-сертифікатів
- Якщо SSL-сертифікати не випускаються, перевірте, чи ваші домени правильно вказують на IP-адресу вашого сервера
- **Власні домени орендарів зависли без SSL?** Перевірте вкладку SSL Certificate застосунку Cloudways. Якщо wildcard-сертифікат (встановлений вручну або такий, що покриває `*.your-network.com`) активний, Cloudways не випускатиме сертифікати Let's Encrypt для окремо зіставлених власних доменів. Замініть його стандартним сертифікатом Let's Encrypt, що покриває лише основний домен мережі (`your-network.com`, `www.your-network.com`), і видаліть будь-які wildcard-записи з `WU_CLOUDWAYS_EXTRA_DOMAINS`. Потім повторно запустіть зіставлення домену (або дочекайтеся наступного), і інтеграція запросить сертифікати для кожного домену.

### Домен не додано
- Перевірте журнали Ultimate Multisite на наявність повідомлень про помилки
- Переконайтеся, що домен ще не додано до Cloudways
- Переконайтеся, що ваш план Cloudways підтримує кількість доменів, які ви додаєте
