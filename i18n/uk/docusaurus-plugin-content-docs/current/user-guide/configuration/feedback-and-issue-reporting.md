# Зворотний зв'язок та повідомлення про проблеми

Gratis AI Agent v1.5.0 запроваджує вбудовану систему збору зворотного зв'язку та повідомлень про проблеми, яка дозволяє кінцевим користувачам позначати незручні відповіді та повідомляти про проблеми безпосередньо з інтерфейсу чату. Ця система включає управління згодою, автоматизовану команду для звітів та допомогу ШІ для попередньої фільтрації (triage) на бекенді.

## Кнопка «Палець вниз»

Кожне повідомлення, надіслане помічником ШІ, відображає кнопку **«Палець вниз»** (👎). Коли користувач натискає на неї, він може позначити відповідь як незручну або неправильну.

- Кнопка з'являється при наведенні курсору біля кожного повідомлення помічника.
- Натискання відкриває **Модальне вікно збору згоди на зворотний зв'язок** (Feedback Consent Modal).
- Зворотний зв'язок прив'язується до потоку розмови та конкретного повідомлення.

## Модальне вікно збору згоди на зворотний зв'язок

Коли користувач натискає кнопку «Палець вниз», перед відправкою будь-яких даних з'являється модальне вікно згоди. У цьому вікні:

- Пояснюється, яка інформація буде зібрана (уривок розмови, контекст браузера).
- Просить користувача підтвердити свою згоду на обмін цими даними.
- Надається поле для введення тексту, де користувач може описати, що пішло не так.
- Пропонується опція **Скасувати** для закриття без надсилання.

Жоден зворотний зв'язок не реєструється, доки користувач явно не підтвердить свою згоду.

## Банер зворотного зв'язку за авто-підказкою

В кінці розмови помічник може відобразити **банер зворотного зв'язку за авто-підказкою** — ненав'язливе повідомлення, яке запитує, чи була сесія корисною.

Цей банер з'являється автоматично на основі евристики, що базується на тривалості та результаті розмови. Він веде до того ж потоку збору зворотного зв'язку, що й кнопка «Палець вниз». Користувачі можуть закрити банер, не надаючи зворотного зв'язку.

## Команда `/report-issue`

Користувачі можуть ініціювати процес збору зворотного зв'язку, ввівши `/report-issue` у поле введення чату. Ця команда:

- Негайно відкриває Модальне вікно збору згоди на зворотний зв'язок.
- Заповнює поле опису контекстом поточної розмови.
- Дозволяє користувачам додати додаткові деталі перед надсиланням.

Команда `/report-issue` доступна в усіх режимах чату (inline, floating widget, full-screen).

## Попереднє фільтрування (Triage) за допомогою ШІ

Надісланий зворотний зв'язок направляється до шару попереднього фільтрування (triage) ШІ, який:

- Категоризує звіт (фактична помилка, незручна відповідь, недоречний контент тощо).
- Прикріплює відповідні метадані розмови.
- Пересилає підсумок попереднього фільтрування до налаштованого **Endpoint для зворотного зв'язку** (див. [Settings > Advanced](#settings-advanced)).

Це зменшує час ручного огляду, висвітлюючи найбільш критичні проблеми в першу чергу.

## Settings > Advanced {#settings-advanced}

Щоб увімкнути бекенд збору зворотного зв'язку, налаштуйте наступні поля у розділі **Gratis AI Agent → Settings → Advanced**:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | URL, який отримує POST-запити з вантажами зворотного зв'язку (JSON). |
| **Feedback API Key** | Bearer token, який надсилається в заголовку `Authorization` для автентифікації надсилання зворотного зв'язку. |

Залиште обидва поля порожніми, щоб вимкнути збір зворотного зв'язку. Індивідуальні кнопки зворотного зв'язку залишаються видимими для користувачів, але надсилання не відбуватиметься.

:::tip
Endpoint для зворотного зв'язку повинен приймати JSON-тіло з такими полями мінімум: `message_id`, `conversation_id`, `feedback_text` та `triage_category`. Зверніться до документації вашого провайдера endpoint для очікуваної схеми.
:::
