# Domain Seller: Налаштування та конфігурація провайдерів

Addon Domain Seller постачається з покроковим майстром налаштування, який проведе вас через кожен необхідний етап. На цій сторінці описано робочий процес майстра та те, як налаштувати або переналаштувати провайдерів після цього.

## Вимоги

- **Multisite Ultimate** v2.4.12 або вище, активований на рівні мережі (network-activated)
- **PHP** 7.4+
- Кредиріали API для принаймні одного підтримуваного реєстратора

## Майстер налаштування при першому запуску

Майстер налаштування запускається автоматично вперше, коли ви активовано на рівні мережі плагін. Його також можна запустити в будь-який час через **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Крок 1 — Вибір провайдера

Виберіть реєстратора, з яким ви хочете підключитися. Підтримувані варіанти:

| Провайдер | Управління DNS | Приватність WHOIS |
|---|---|---|
| OpenSRS | Так | Так |
| Namecheap | Ні | Так (WhoisGuard, безкоштовно) |
| GoDaddy | Ні | Ні |
| ResellerClub | Так | Ні |
| NameSilo | Ні | Ні |
| Enom | Так | Ні |

### Крок 2 — Введення облікових даних

Кожен провайдер має різні поля для облікових даних:

**OpenSRS** — Ім'я користувача та приватний ключ (з OpenSRS Reseller Control Panel)

**Namecheap** — Ім'я користувача та API ключ (з Account › Tools › API Access)

**GoDaddy** — API ключ та секрет (з developer.godaddy.com)

**ResellerClub** — Reseller ID та API ключ (з панелі керування ResellerClub)

**NameSilo** — API ключ (з namesilo.com › Account › API Manager)

**Enom** — Account ID та API токен

Перевірте **Sandbox mode**, де це можливо, щоб протестувати підключення на тестовому середовищі провайдера перед запуском у робочий режим.

### Крок 3 — Тестування підключення

Натисніть **Test Connection**. Майстер надсилає легкий API-запит для перевірки облікових даних та зв'язку. Виправте будь-які проблеми з обліковими даними, перш ніж продовжувати.

### Крок 4 — Імпорт TLDs

Натисніть **Import TLDs**, щоб завантажити всі доступні TLD та оптові ціни від підключеного провайдера. Це заповнить список TLD, який використовується для доменних продуктів. Імпорт може зайняти 30–60 секунд для провайдерів із великим каталогом TLD.

TLD також автоматично синхронізуються один раз на день за допомогою запланованого cron job.

### Крок 5 — Створення доменного продукту

Майстер створює стандартний "catch-all" доменний продукт із націнкою 10%. Ви можете відредагувати цей продукт негайно або пропустити цей крок і створити продукти вручну у розділі **Ultimate Multisite › Products**.

Дивіться [Domain Products and Pricing](./domain-products) для повного посібника з конфігурації продуктів.

## Переналаштування провайдера

Перейдіть до **Network Admin › Ultimate Multisite › Settings › Domain Seller** (або натисніть **Settings** у списку плагінів).

На сторінці налаштувань ви знайдете:

- **Enable domain selling** — перемикач для вмикання/вимикання всієї функції
- **Default provider** — провайдер, який використовується для пошуку доменів та нових продуктів
- **Max TLDs per search** — скільки TLD перевіряється, коли клієнт здійснює пошук; вищі значення показують більше варіантів, але працюють повільніше
- **Availability cache duration** — як довго кешуються результати доступності та ціноутворення; нижчі значення є точнішими, але збільшують кількість API-запитів
- **Manage domain products** — швидке посилання на список продуктів
- **Configure providers** — відкриває Integration Wizard для додавання або переналаштування провайдерів

### Додавання другого провайдера

Натисніть **Configure providers** і повторіть запуск майстра для нового реєстратора. Ви можете налаштувати кілька провайдерів одночасно. Призначте кожен доменний продукт конкретному провайдеру або залиште його на значенні за замовчуванням.

### Ручна синхронізація TLDs

На сторінці налаштувань натисніть **Sync TLDs** поруч із будь-яким налаштованим провайдером, щоб завантажити останні ціни. Це корисно після того, як провайдер оновлює оптові ціни або додає нові TLD.

## Логи

Кожен провайдер пише у власний канал логів. Логи можна переглянути за шляхом **Network Admin › Ultimate Multisite › Logs**:

| Лог-канал | Зміст |
|---|---|
| `domain-seller-registration` | Усі спроби реєстрації (успішні та невдалі) |
| `domain-seller-renewal` | Результати роботи з оновлення |
| `domain-seller-opensrs` | Сира активність API OpenSRS |
| `domain-seller-namecheap` | Сира активність API Namecheap |
| `domain-seller-godaddy` | Сира активність API GoDaddy |
| `domain-seller-resellerclub` | Сира активність API ResellerClub |
| `domain-seller-namesilo` | Сира активність API NameSilo |
| `domain-seller-enom` | Сира активність API Enom |
