# Domain Seller: налаштування та конфігурація провайдерів

Аддон Domain Seller постачається з керованим майстром налаштування, який проводить вас через кожен обов’язковий крок. На цій сторінці описано перебіг роботи майстра та те, як налаштувати або переналаштувати провайдерів згодом.

## Вимоги

- **Ultimate Multisite** v2.4.12 або новіша, активована в мережі
- **PHP** 7.4+
- Облікові дані API принаймні для одного підтримуваного реєстратора

## Майстер першого запуску

Майстер налаштування запускається автоматично під час першої мережевої активації плагіна. Він також доступний будь-коли за шляхом **Network Admin › Ultimate Multisite › Domain Seller Setup**.

### Крок 1 — Виберіть провайдера

Виберіть реєстратора, якого хочете підключити. Підтримувані варіанти:

| Провайдер | Керування DNS | Конфіденційність WHOIS |
|---|---|---|
| OpenSRS | Так | Так |
| Namecheap | Ні | Так (WhoisGuard, безкоштовно) |
| HostAfrica | Так | Так (захист ID) |
| Openprovider | Так | Так |
| Hostinger | Через основне зіставлення доменів Hostinger для розміщених доменів | Так |
| GoDaddy | Ні | Ні |
| ResellerClub | Так | Ні |
| NameSilo | Ні | Ні |
| Enom | Так | Ні |

### Крок 2 — Введіть облікові дані

Кожен провайдер має різні поля облікових даних:

**OpenSRS** — ім’я користувача та приватний ключ (з OpenSRS Reseller Control Panel)

**Namecheap** — ім’я користувача та ключ API (з Account › Tools › API Access)

**HostAfrica** — endpoint Domains Reseller API та облікові дані з реселерського модуля HostAfrica. Окремий sandbox endpoint наразі не задокументовано; тестуйте за допомогою безпечних перевірок лише для читання перед запуском реальних реєстрацій.

**Openprovider** — ім’я користувача та пароль з увімкненим доступом до API. Необов’язковий sandbox-режим використовує sandbox API Openprovider, а необов’язковий стандартний ідентифікатор клієнта можна повторно використовувати для реєстрацій.

**Hostinger** — спільний API-токен Hostinger hPanel з основної інтеграції Hostinger. Той самий токен забезпечує основне зіставлення доменів і операції реєстрації Domain Seller.

**GoDaddy** — ключ API та секрет (з developer.godaddy.com)

**ResellerClub** — ID реселера та ключ API (з панелі керування ResellerClub)

**NameSilo** — ключ API (з namesilo.com › Account › API Manager)

**Enom** — ID Account і API-токен

Увімкніть **Sandbox mode** там, де доступно, щоб протестувати роботу з тестовим середовищем провайдера перед переходом у production.

### Крок 3 — Перевірте з’єднання

Натисніть **Test Connection**. Майстер надсилає легкий виклик API, щоб перевірити облікові дані та підключення. Виправте будь-які проблеми з обліковими даними, перш ніж продовжити.

### Крок 4 — Імпортуйте TLD

Натисніть **Import TLDs**, щоб отримати всі доступні TLD і оптові ціни від підключеного провайдера. Це заповнює список TLD, який використовується доменними продуктами. Імпорт може тривати 30–60 секунд для провайдерів із великими каталогами TLD.

TLD також автоматично повторно синхронізуються раз на день через заплановане завдання cron.

### Крок 5 — Створіть доменний продукт

Майстер створює стандартний універсальний доменний продукт із націнкою 10%. Ви можете одразу відредагувати цей продукт або пропустити цей крок і створити продукти вручну в **Ultimate Multisite › Products**.

Див. [Доменні продукти та ціноутворення](./domain-products), щоб отримати повний посібник із налаштування продуктів.

## Переналаштування провайдера

Перейдіть до **Network Admin › Ultimate Multisite › Settings › Domain Seller** (або натисніть **Settings** у списку плагінів).

Сторінка налаштувань містить:

- **Enable domain selling** — увімкнути або вимкнути всю функцію
- **Default provider** — провайдер, який використовується для пошуку доменів і нових продуктів
- **Max TLDs per search** — скільки TLD перевіряти, коли клієнт виконує пошук; вищі значення показують більше варіантів, але працюють повільніше
- **Availability cache duration** — як довго кешувати результати доступності та ціноутворення; нижчі значення точніші, але збільшують кількість викликів API
- **Manage domain products** — швидке посилання на список продуктів
- **Configure providers** — відкриває Integration Wizard для додавання або переналаштування провайдерів

### Додавання другого провайдера

Натисніть **Configure providers** і знову запустіть майстер для нового реєстратора. Ви можете мати кілька провайдерів, налаштованих одночасно. Призначте кожен доменний продукт конкретному провайдеру або залиште стандартного.

### Ручна синхронізація TLD

На сторінці налаштувань натисніть **Sync TLDs** поруч із будь-яким налаштованим провайдером, щоб отримати найновіші ціни. Це корисно після того, як провайдер оновлює оптові ціни або додає нові TLD.

## Журнали

Кожен провайдер записує дані у власний канал журналу. Журнали можна переглянути в **Network Admin › Ultimate Multisite › Logs**:

| Канал журналу | Вміст |
|---|---|
| `domain-seller-registration` | Усі спроби реєстрації (успішні та невдалі) |
| `domain-seller-renewal` | Результати завдань поновлення |
| `domain-seller-opensrs` | Необроблена активність API OpenSRS |
| `domain-seller-namecheap` | Необроблена активність API Namecheap |
| `domain-seller-hostafrica` | Необроблена активність API HostAfrica |
| `domain-seller-openprovider` | Необроблена активність API Openprovider |
| `domain-seller-hostinger` | Необроблена активність API Hostinger |
| `domain-seller-godaddy` | Необроблена активність API GoDaddy |
| `domain-seller-resellerclub` | Необроблена активність API ResellerClub |
| `domain-seller-namesilo` | Необроблена активність API NameSilo |
| `domain-seller-enom` | Необроблена активність API Enom |

## Примітки щодо можливостей провайдерів

Не кожен API реєстратора надає ті самі операції. Аддон показує непідтримувані операції з чіткими помилками для адміністраторів замість того, щоб мовчки завершуватися невдало.

- **HostAfrica** підтримує найширший робочий процес live-реселера, зокрема пошук, синхронізацію TLD/цін, реєстрацію, поновлення, перенесення, оновлення nameserver, DNS-записи, EPP-коди, registrar lock і захист ID.
- **Openprovider** підтримує синхронізацію TLD за реселерськими цінами, реєстрацію, поновлення, перенесення, оновлення nameserver, DNS-зони, EPP-коди, registrar lock і конфіденційність WHOIS. Він автентифікується за допомогою короткострокового bearer token, який addon автоматично оновлює.
- **Hostinger** підтримує пошук доступності, реєстрацію, перегляд портфоліо, оновлення nameserver, registrar lock і конфіденційність WHOIS через спільний API-токен hPanel. Публічний Domains API Hostinger не надає реселерських/гуртових цін, вхідного перенесення, явного поновлення або отримання EPP-кодів; поновлення виконуються лише через автопоновлення.
