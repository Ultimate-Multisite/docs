# Доменні продукти та ціноутворення

Доменні продукти — це спосіб контролю ціноутворення, TLD (доменних розширень), конфіденційності WHOIS та вибору провайдера. Кожен доменний продукт є стандартним продуктом Ultimate Multisite із додатковою вкладкою **Налаштування домену** (Domain Settings).

## Створення доменного продукту

1. Перейдіть до **Network Admin › Ultimate Multisite › Products**
2. Натисніть **Add New**
3. Встановіть тип продукту на **Domain**
4. Налаштуйте вкладку **Domain Settings** (див. нижче)
5. Зберегти

Доменні продукти відображаються з фіолетовою позначкою **Domain** у списку продуктів і можуть бути відфільтровані за допомогою вкладки **Domain Products**.

## Вкладка налаштувань домену

### Провайдер

Виберіть реєстратора, який обробляє реєстрацію для цього продукту. За замовчуванням встановлюється глобальний **Default provider**, заданий у налаштуваннях Domain Seller.

### Підтримувані TLD

Залиште поле порожнім, щоб створити **універсальний продукт** (catch-all product), який застосовується до всіх TLD, не покритих іншим продуктом.

Введіть список TLD, розділений комами (наприклад, `.com, .net, .org`), щоб створити **продукт, специфічний для TLD**, який застосовується лише до цих розширень.

**Як працює зіставлення продуктів:** Коли клієнт шукає домен, аддон вибирає найбільш специфічний відповідний продукт. Продукт із `.com` у своєму списку TLD має пріоритет над універсальним продуктом. Якщо жоден продукт, специфічний для TLD, не підходить, використовується універсальний продукт. Якщо продукти не існує, пошук домену не відображається.

### Тип націнки (Markup type)

Три режими контролюють, як розраховується ваша роздрібна ціна на основі оптової вартості:

| Режим | Як це працює |
|---|---|
| **Відсоток (Percentage)** | Додає відсоток до оптової вартості. Націнка 20% на домен за оптовій вартістю $10 дає $12. |
| **Фіксована націнка (Fixed markup)** | Додає фіксовану доларову суму. Націнка $5 на домен за оптовій вартістю $10 дає $15. |
| **Фіксована ціна (Fixed price)** | Повністю ігнорує оптову вартість. Завжди стягує ту суму, яку ви вводите. |

### Вступна ціна (Introductory pricing)

Увімкніть, щоб запропонувати знижену ціну на перший рік. Встановіть окрему **Вступну ціну** (ціна на рік 1) поряд із звичайною **Ціною поновлення** (Renewal price, рік 2+). Клієнт бачить обидві ціни під час оформлення замовлення, щоб знати, чого очікувати при поновленні.

### Конфіденційність WHOIS

Контролює, чи пропонується захист конфіденційності WHOIS для доменів, зареєстрованих через цей продукт.

| Налаштування | Поведінка |
|---|---|
| **Disabled** | Конфіденційність WHOIS ніколи не пропонується чи не вмикається. |
| **Always Included** | Конфіденційність WHOIS автоматично вмикається при реєстрації безкоштовно. |
| **Customer Choice** | Під час оформлення замовлення з'являється прапорець. Встановіть **Privacy price**, щоб стягувати плату за рік, або залиште його на $0, щоб запропонувати це безкоштовно. |

Для Namecheap конфіденційність WHOIS використовує WhoisGuard (завжди безкоштовно). Для OpenSRS використовується служба конфіденційності OpenSRS (може мати вартість на оптовому рівні).

## Імпорт та синхронізація TLD

Доменні продукти відображають оптову вартість у режимі реального часу, отриману від підключеного провайдера. Щоб це працювало, необхідно імпортувати TLD.

- **Ручна синхронізація:** Settings › Domain Seller › Sync TLDs (за провайдером)
- **Автоматична синхронізація:** Виконується щодня за допомогою запланованого cron job для всіх налаштованих провайдерів

Після синхронізації перейдіть на вкладку Domain Settings будь-якого доменного продукту та використайте TLD picker, щоб побачити доступні TLD з їхньою поточною оптовою вартістю.

## Автоматичне поновлення

Поновлення доменів прив'язане до статусу членства клієнта:

- Коли членство поновлюється, і домен прив'язаний, поновлення домену автоматично ставиться в чергу
- Спроби поновлення використовують активний платіжний шлюз клієнта
- Невиконані поновлення автоматично повторюються з експоненційним відкатом (exponential backoff)
- Електронні повідомлення надсилаються про успішне поновлення, збої та майбутні закінчення терміну дії

ID шаблонів електронних листів для подій життєвого циклу домену:

| Подія | Template ID |
|---|---|
| Домен зареєстровано | `domain_registered` |
| Домен поновлено | `domain_renewed` |
| Поновлення не вдалося | `domain_renewal_failed` |
| Домен скоро закінчиться | `domain_expiring_soon` |

## Адмін: Ручна реєстрація домену

Щоб зареєструвати домен від імені клієнта, не змушуючи його проходити через оформлення замовлення:

1. Перейдіть до **Network Admin › Ultimate Multisite › Register Domain**
2. Виберіть клієнта та введіть ім'я домену
3. Заповніть контактні дані реєстратора (ім'я, адреса, телефон)
4. Натисніть **Register**

Запис домену створюється та прив'язується до облікового запису клієнта.
