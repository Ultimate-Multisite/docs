# Поле оформлення замовлення та керування DNS клієнта

## Поле вибору домену під час оформлення замовлення {#the-domain-selection-checkout-field}

Поле **Вибір домену** — це елемент оформлення замовлення, який дає клієнтам вибір, як отримати домен для свого сайту. Додайте його до будь-якої форми оформлення замовлення, щоб увімкнути продаж доменів.

### Додавання поля до форми оформлення замовлення {#adding-the-field-to-a-checkout-form}

1. Перейдіть до **Network Admin › Ultimate Multisite › Форми оформлення замовлення**
2. Відкрийте або створіть форму оформлення замовлення
3. У редакторі оформлення замовлення натисніть **Додати поле**
4. Виберіть **Вибір домену** зі списку полів
5. Налаштуйте параметри поля (див. нижче)
6. Збережіть форму

### Параметри поля {#field-options}

**Режими домену** — Виберіть, які вкладки бачить клієнт. Кожен режим можна вмикати або вимикати незалежно:

| Режим | Що він робить |
|---|---|
| **Субдомен** | Клієнт використовує безкоштовний субдомен у вашій мережі (наприклад, `mysite.yournetwork.com`). Оплата не потрібна. |
| **Зареєструвати новий домен** | Клієнт шукає новий домен і реєструє його через налаштованого вами провайдера. Використовує відповідний доменний продукт для ціноутворення. |
| **Наявний домен** | Клієнт підключає домен, яким уже володіє. Реєстраційний збір відсутній. Домен автоматично підключається до його сайту. |

**Режим за замовчуванням** — Коли ввімкнено всі три режими, яка вкладка відкривається першою. Установіть **Субдомен**, щоб реєстрація домену залишалася необов’язковою, або **Зареєструвати новий домен**, щоб заохотити покупки.

**Доменний продукт** — За бажанням закріпіть це поле за конкретним доменним продуктом. Якщо не задано, addon автоматично вибирає відповідний продукт на основі TLD, який шукає клієнт.

### Поля контактних даних реєстранта {#registrant-contact-fields}

Коли клієнт вибирає вкладку **Зареєструвати новий домен**, форма оформлення замовлення додає вбудовані поля контактних даних реєстранта:

- Ім’я / Прізвище
- Адреса електронної пошти
- Адреса (рядок 1, місто, штат/провінція, поштовий індекс, країна)
- Номер телефону

Вони потрібні всім реєстраторам і перевіряються перед виконанням виклику registration API. Номери телефонів автоматично форматуються в міжнародний формат `+CC.NNN`, якого очікують реєстратори.

### Автоматично згенерований URL сайту {#auto-generated-site-url}

Коли клієнт реєструє або підключає домен, поле URL сайту автоматично заповнюється з вибраного домену. Клієнтам не потрібно заповнювати окреме поле URL.

### Поведінка пошуку {#search-behaviour}

- Доступність домену перевіряється в реальному часі через AJAX, коли клієнт вводить текст
- Альтернативні пропозиції TLD показуються, коли бажаний домен недоступний
- Ціни отримуються наживо та відображаються чітко (ціна реєстрації, ціна поновлення, необов’язкова плата за WHOIS privacy)
- Коди купонів застосовуються до доменних продуктів так само, як і до будь-якого іншого продукту

**Налаштування швидкості реакції пошуку:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Додавання користувацьких полів до форми пошуку домену:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Керування DNS клієнта {#customer-dns-management}

Клієнти можуть керувати DNS-записами для своїх зареєстрованих доменів зі сторінки **My Account**, у записі свого домену.

### Підтримувані типи записів {#supported-record-types}

| Тип | Використання |
|---|---|
| **A** | Зіставити ім’я хоста з IPv4-адресою |
| **AAAA** | Зіставити ім’я хоста з IPv6-адресою |
| **CNAME** | Створити псевдонім, що вказує на інше ім’я хоста |
| **MX** | Установити сервер обміну поштою |
| **TXT** | Додати SPF, DMARC, запис підтвердження або інші текстові записи |

### Які провайдери підтримують керування DNS? {#which-providers-support-dns-management}

Керування DNS (додавання, редагування, видалення записів) доступне з **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** і **Openprovider**. Домени **Hostinger** можуть оновлювати сервери імен через Domain Seller; DNS-записами для розміщених доменів керує основна інтеграція зіставлення доменів Hostinger. Домени Namecheap, GoDaddy і NameSilo відображають інформацію про статус і дату завершення строку дії, але DNS потрібно керувати безпосередньо в панелі керування реєстратора.

### DNS-записи за замовчуванням {#default-dns-records}

Ви можете налаштувати DNS-записи за замовчуванням, які застосовуються автоматично під час реєстрації домену. Перейдіть до **Налаштування › Domain Seller › DNS-записи за замовчуванням**.

Значення записів за замовчуванням підтримують два токени:

| Токен | Замінюється на |
|---|---|
| `{DOMAIN}` | Зареєстроване доменне ім’я (наприклад, `example.com`) |
| `{SITE_URL}` | URL сайту WordPress для сайту клієнта |

**Приклад — спрямуйте apex-домен і www на IP-адресу вашого сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Адміністратор: перегляд і редагування DNS {#admin-viewing-and-editing-dns}

Адміністратори мережі можуть переглядати й редагувати DNS-записи для будь-якого домену клієнта зі сторінки редагування домену в **Network Admin › Ultimate Multisite › Домени**.
