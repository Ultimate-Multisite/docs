# Управління полями оформлення замовлення та DNS для клієнтів

## Поле вибору домену

Поле **Domain Selection** (Вибір домену) — це елемент оформлення замовлення, який дає клієнтам можливість обрати, як вони хочуть отримати доменне ім'я для свого сайту. Додайте його до будь-якої форми оформлення замовлення, щоб увімкнути продаж доменів.

### Додавання поля до форми оформлення замовлення

1. Перейдіть до **Network Admin › Ultimate Multisite › Checkout Forms**
2. Відкрийте або створіть форму оформлення замовлення
3. В редакторі оформлення замовлення натисніть **Add Field** (Додати поле)
4. Виберіть **Domain Selection** зі списку полів
5. Налаштуйте параметри поля (див. нижче)
6. Збережіть форму

### Параметри поля

**Domain modes** (Режими домену) — Виберіть вкладки, які бачить клієнт. Кожен режим можна вмикати або вимикати незалежно:

| Режим | Що робить |
|---|---|
| **Subdomain** | Клієнт використовує безкоштовний піддомен у вашій мережі (наприклад, `mysite.yournetwork.com`). Оплата не потрібна. |
| **Register New Domain** | Клієнт шукає новий домен і реєструє його через вашого конфігурованого провайдера. Використовує продукт відповідного домену для розрахунку вартості. |
| **Existing Domain** | Клієнт прив'язує домен, який йому вже належить. Плата за реєстрацію не стягується. Домен автоматично прив'язується до його сайту. |

**Default mode** (Режим за замовчуванням) — Коли всі три режими увімкнено, яка вкладка відкривається першою. Встановіть **Subdomain**, щоб зберегти реєстрацію домену необов'язковою, або **Register New Domain**, щоб заохотити покупки.

**Domain product** (Продукт домену) — За бажанням прив'яжіть це поле до конкретного продукту домену. Якщо не встановлено, аддон автоматично вибирає відповідний продукт на основі TLD, який вводить клієнт.

### Поля контактних даних власника (Registrant contact fields)

Коли клієнт вибирає вкладку **Register New Domain**, форма оформлення замовлення додає поля контактних даних власника прямо в ряд:

- Ім'я / Прізвище
- Адреса електронної пошти
- Адреса (рядок 1, місто, штат/провінція, поштовий індекс, країна)
- Номер телефону

Ці дані є обов'язковими для всіх реєстраторів і валідуються перед викликом API реєстрації. Номери телефонів автоматично форматуються до міжнародного формату `+CC.NNN`, який очікують реєстратори.

### Автоматично згенерований URL сайту

Коли клієнт реєструє або прив'язує домен, поле URL сайту автоматично заповнюється на основі обраного домену. Клієнтам не потрібно вводити окреме поле URL.

### Поведінка пошуку

- Доступність домену перевіряється в режимі реального часу за допомогою AJAX, поки клієнт друкує
- Альтернативні пропозиції TLD відображаються, якщо бажаний домен недоступний
- Ціна завантажується в реальному часі та чітко відображається (ціна реєстрації, ціна поновлення, необов'язкова плата за конфіденційність WHOIS)
- Купони застосовуються до продуктів доменів так само, як і до будь-яких інших продуктів

**Налаштування чутливості пошуку:**

```php
// Збільште затримку debounce (мілісекунди), щоб зменшити кількість викликів API на повільних з'єднаннях
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Додавання користувацьких полів до форми пошуку домену:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Додаткові примітки',
    ];
    return $fields;
});
```

## Управління DNS для клієнтів

Клієнти можуть керувати DNS-записами для своїх зареєстрованих доменів на сторінці **My Account** (Мій обліковий запис), у розділі, присвяченому їхньому домену.

### Підтримувані типи записів

| Тип | Використання |
|---|---|
| **A** | Прив'язка імені хоста до IP-адреси IPv4 |
| **AAAA** | Прив'язка імені хоста до IP-адреси IPv6 |
| **CNAME** | Створення псевдоніма, що вказує на інший хост |
| **MX** | Встановлення сервера обміну електронною поштою |
| **TXT** | Додавання записів SPF, DMARC, верифікації або інших текстових даних |

### Які провайдери підтримують управління DNS?

Управління DNS (додавання, редагування, видалення записів) доступне з **OpenSRS**, **ResellerClub** та **Enom**. Домени Namecheap, GoDaddy та NameSilo відображають статус та інформацію про термін дії, але DNS необхідно керувати безпосередньо в панелі керування реєстратора.

### Записи DNS за замовчуванням

Ви можете налаштувати записи DNS за замовчуванням, які автоматично застосовуються при реєстрації домену. Перейдіть до **Settings › Domain Seller › Default DNS Records**.

Значення записів за замовчуванням підтримують два токени:

| Токен | Замінюється на |
|---|---|
| `{DOMAIN}` | Назва зареєстрованого домену (наприклад, `example.com`) |
| `{SITE_URL}` | URL сайту WordPress для клієнта |

**Приклад — вказівка кореневого домену та www на IP-адресу вашого сервера:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Адмін: перегляд та редагування DNS

Адміністратори мережі можуть переглядати та редагувати записи DNS для будь-якого домену клієнта на сторінці редагування домену у **Network Admin › Ultimate Multisite › Domains**.
