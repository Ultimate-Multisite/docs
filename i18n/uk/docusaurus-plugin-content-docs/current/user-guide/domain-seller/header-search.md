# Пошук домену в заголовку

Використовуйте це налаштування, коли потрібна невелика форма в заголовку, що запускає пошук домену, а потім дає клієнту змогу вибрати з доступних доменів усередині checkout Ultimate Multisite.

## Вимоги

- Ultimate Multisite активний у мережі.
- Multisite Ultimate Domain Seller активний у мережі.
- Принаймні один активний продукт реєстрації домену з:
  - `domain_provider`, установленим на налаштованого провайдера.
  - Налаштованими підтримуваними TLD, наприклад `com`, `net` і `org`.
- Дійсна форма checkout, що містить поле **Вибір домену**.

## Форма checkout

1. Створіть або відредагуйте форму checkout, яку використовує сторінка реєстрації.
2. Додайте звичайні обов’язкові поля checkout/Account, зокрема **Ім’я користувача**. Форма checkout, що містить лише поле домену, відхиляється валідацією Ultimate Multisite.
3. Додайте поле **Вибір домену**.
4. Установіть режим Domain Selection на **Лише реєстрація**, коли потік має зосереджуватися на зареєстрованих доменах, а не на безкоштовних субдоменах чи наявних доменах.
5. Призначте продукт реєстрації домену цьому полю.

Сторінка реєстрації має відображати форму checkout, наприклад:

```text
[wu_checkout slug="domain-form"]
```

## Форма в заголовку

Додайте невелику форму `GET` у заголовку сайту, яка надсилає введений пошуковий термін на сторінку checkout як `domain_name`:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

Не вибирайте домен заздалегідь у користувацькому JavaScript заголовка. Заголовок має лише передавати пошуковий термін. Скрипт checkout Domain Seller зчитує `?domain_name=example`, заповнює поле пошуку checkout і запускає пошук доступності, щоб клієнт міг вибрати з повернених доменів.

## Очікувана поведінка

Пошук `example` у заголовку має відкрити:

```text
/register/?domain_name=example
```

Після цього checkout має показати результати, які можна вибрати, наприклад:

- `example.com`
- `example.net`
- `example.org` недоступний
- інші TLD, що підтримуються провайдером

Після вибору доступного результату підсумок замовлення має містити продукт реєстрації домену та вибране доменне ім’я.

## Перевірка

1. Відкрийте домашню сторінку.
2. Знайдіть голе ім’я, наприклад `example`.
3. Переконайтеся, що URL checkout містить `?domain_name=example`.
4. Переконайтеся, що поле домену checkout містить `example`.
5. Переконайтеся, що з’являється список варіантів домену.
6. Натисніть **Вибрати** для доступного домену.
7. Переконайтеся, що підсумок замовлення містить `Domain Registration - example.com` або вибраний домен.

## Усунення несправностей

- Якщо список не з’являється, перевірте вкладку мережі браузера для `admin-ajax.php?action=wu_domain_search` і переконайтеся, що вона повертає непорожні `domains` або `results`.
- Якщо форма checkout не проходить валідацію під час збереження, додайте обов’язкові поля Account, як-от **Ім’я користувача**.
- Якщо вибір домену не оновлює кошик, переконайтеся, що `window.wu_checkout_form` існує і що ресурси checkout Domain Seller завантажені на сторінці checkout.
