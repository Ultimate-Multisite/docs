# Міграція в режим сторонніх розробників

Superdav AI Agent v1.12.0 змінює обробку можливостей сторонніх розробників (third-party abilities). **Режим сторонніх розробників тепер за замовчуванням встановлюється на "auto"**, що дозволяє інтегрувати нативну Abilities API WordPress на WordPress 7.0+ без ручної конфігурації.

## Що змінилося? {#what-changed}

### До v1.12.0 {#before-v1120}

Для використання можливостей сторонніх розробників потрібна була ручна конфігурація:

- Вам потрібно було явно увімкнути "third-party mode"
- Можливості завантажувалися з користувацького реєстру
- Інтеграція з WordPress Abilities API була необов'язковою
- За замовчуванням використовувався режим "Legacy"

### Після v1.12.0 {#after-v1120}

Можливості сторонніх розробників працюють автоматично:

- Режим сторонніх розробників за замовчуванням встановлюється на "auto"
- Можливості інтегруються нативно з WordPress Abilities API
- На WordPress 7.0+ не потрібна ручна конфігурація
- Режим "Legacy" все ще доступний для старіших версій WordPress

## Кого це стосується? {#who-is-affected}

### Нові встановлення (WordPress 7.0+) {#new-installations-wordpress-70}

**Дія не потрібна.** Режим сторонніх розробників автоматично встановлюється на "auto", і можливості працюють "з коробки".

### Існуючі встановлення {#existing-installations}

**Ваші налаштування збережено.** Якщо ви використовували:

- **Legacy mode**: Ви залишаєтеся в режимі "Legacy" (без змін)
- **Manual third-party mode**: Ви залишаєтеся в ручному режимі (без змін)
- **Auto mode**: Ви продовжуєте використовувати авторежим (без змін)

### Версії WordPress до 7.0 {#wordpress-versions-before-70}

**Режим "Legacy" все ще доступний.** Якщо ви використовуєте WordPress 6.x або раніше:

- Режим сторонніх розробників за замовчуванням встановлюється на "legacy"
- Ви можете вручну увімкнути режим сторонніх розробників за бажанням
- Оновіться до WordPress 7.0+ для використання нативного Abilities API

## Розуміння режимів {#understanding-the-modes}

### Auto Mode (Новий стандарт) {#auto-mode-new-default}

**Auto mode** використовує нативну інтеграцію WordPress Abilities API:

- Можливості реєструються через WordPress hooks
- Повна сумісність з Abilities API WordPress 7.0+
- Автоматичне виявлення можливостей сторонніх розробників
- Не потрібна ручна конфігурація

**Коли використовувати**: WordPress 7.0+ з можливостями сторонніх розробників

### Manual Mode {#manual-mode}

**Manual mode** вимагає явного налаштування:

- Ви вказуєте, які можливості сторонніх розробників завантажити
- Корисно для тестування або вибіркового завантаження можливостей
- Потрібно редагувати файли конфігурації
- Більше контролю, але більше налаштування

**Коли використовувати**: Тестування, вибіркове завантаження можливостей або користувацькі конфігурації

### Legacy Mode {#legacy-mode}

**Legacy mode** використовує стару систему можливостей сторонніх розробників:

- Користувацький реєстр можливостей (не WordPress Abilities API)
- Зворотна сумісність зі старішими версіями WordPress
- Немає нативної інтеграції з WordPress
- Застаріло, але все ще підтримується

**Коли використовувати**: WordPress 6.x або раніше, або коли потрібна зворотна сумісність

## Перевірка поточного режиму {#checking-your-current-mode}

### Через панель адміністратора {#via-admin-panel}

1. Перейдіть до **WordPress Admin** → **Superdav AI Agent** → **Settings**
2. Знайдіть налаштування **Third-Party Mode**
3. Ви побачите поточний режим та опції для його зміни

### Через код {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', або 'legacy'
```

## Зміна режиму {#changing-your-mode}

### Перехід в Auto Mode {#switch-to-auto-mode}

Якщо ви на WordPress 7.0+ і хочете використовувати авторежим:

1. Перейдіть до **Superdav AI Agent** → **Settings**
2. Знайдіть **Third-Party Mode**
3. Виберіть **Auto (WordPress Abilities API)**
4. Натисніть **Save**

Superdav AI Agent автоматично виявить та зареєструє можливості сторонніх розробників.

### Перехід в Manual Mode {#switch-to-manual-mode}

Якщо ви хочете вручну контролювати, які можливості завантажуються:

1. Перейдіть до **Superdav AI Agent** → **Settings**
2. Знайдіть **Third-Party Mode**
3. Виберіть **Manual**
4. Натисніть **Save**
5. Відредагуйте свій файл конфігурації, щоб вказати, які можливості завантажити

### Перехід в Legacy Mode {#switch-to-legacy-mode}

Якщо вам потрібна зворотна сумісність:

1. Перейдіть до **Superdav AI Agent** → **Settings**
2. Знайдіть **Third-Party Mode**
3. Виберіть **Legacy**
4. Натисніть **Save**

## Переваги Auto Mode {#benefits-of-auto-mode}

### Автоматичне виявлення {#automatic-discovery}

Можливості автоматично виявляються з:

- Встановлених плагінів
- Активної теми
- Must-use плагінів
- Drop-in плагінів

Не потрібна ручна реєстрація.

### Нативна інтеграція {#native-integration}

Можливості інтегруються з WordPress Abilities API:

- Сумісно з ядром WordPress
- Працює з адмін-панеллю WordPress
- Сумісно з іншими плагінами, які використовують Abilities API
- Забезпечує захист від змін у майбутньому, оскільки WordPress розвивається

### Спрощене управління {#simplified-management}

- Немає файлів конфігурації для редагування
- Немає ручної реєстрації можливостей
- Налаштування видимості можливостей працюють автоматично
- Повідомлення в адмін-панелі попереджають про некласифіковані можливості

### Краща продуктивність {#better-performance}

- Можливості кешуються
- Завантажуються "на вимогу" (lazy-loaded)
- Оптимізовано для WordPress 7.0+

## Шлях міграції {#migration-path}

### Якщо ви на WordPress 6.x {#if-youre-on-wordpress-6x}

1. **Оновіться до WordPress 7.0+** (коли будете готові)
2. **Оновіть Superdav AI Agent** до v1.12.0+
3. **Змініть режим сторонніх розробників на Auto** (необов'язково; режим "Legacy" все ще працює)
4. **Перегляньте видимість можливостей**, щоб забезпечити належний контроль доступу

### Якщо ви на WordPress 7.0+ {#if-youre-on-wordpress-70}

1. **Оновіть Superdav AI Agent** до v1.12.0+
2. **Перевірте, чи встановлено режим сторонніх розробників на Auto** (за замовчуванням має бути так)
3. **Перегляньте видимість можливостей**, щоб забезпечити належний контроль доступу
4. **Протестуйте можливості сторонніх розробників**, щоб підтвердити їх роботу

## Усунення несправностей {#troubleshooting}

### Можливості не завантажуються в авторежимі {#abilities-arent-loading-in-auto-mode}

- Перевірте, що ви на WordPress 7.0+
- Переконайтеся, що режим сторонніх розробників встановлено на "Auto"
- Перевірте, чи активний плагін, що надає можливість
- Перевірте журнали помилок WordPress на наявність помилок реєстрації

### Я хочу зберегти режим "Legacy" {#i-want-to-keep-legacy-mode}

- Перейдіть до **Settings** → **Third-Party Mode**
- Виберіть **Legacy**
- Натисніть **Save**
- Режим "Legacy" продовжить працювати

### Мої користувацькі можливості не відображаються {#my-custom-abilities-arent-showing}

- Перевірте, що вони зареєстровані через WordPress hooks
- Переконайтеся, що вони правильно реалізують Abilities API
- Перегляньте журнали помилок WordPress
- Використовуйте сторінку адмін-панелі **Ability Visibility**, щоб побачити всі зареєстровані можливості

### Я отримую повідомлення "unclassified ability" {#im-getting-unclassified-ability-notices}

- Це нормально для нових можливостей сторонніх розробників
- Перегляньте та класифікуйте їх у повідомленні в адмін-панелі
- Дивіться **Ability Visibility** для деталей класифікації

## Зворотна сумісність {#backward-compatibility}

### Існуючі конфігурації {#existing-configurations}

Якщо у вас є існуючі конфігурації можливостей сторонніх розробників:

- **Legacy mode**: Ваша конфігурація продовжує працювати
- **Manual mode**: Ваша конфігурація продовжує працювати
- **Auto mode**: Ваша конфігурація ігнорується (авторежим бере пріоритет)

Щоб зберегти вашу користувацьку конфігурацію, залишайтеся в режимі Manual або Legacy.

### Графік виведення з ладу (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: Режими Legacy та Manual все ще повністю підтримуються
- **v1.13.0+**: У режимі Legacy можуть з'являтися повідомлення про виведення з ладу
- **v2.0.0**: Режим Legacy може бути видалено (TBD)

## Найкращі практики {#best-practices}

### Для нових встановлень {#for-new-installations}

- Використовуйте Auto mode (це стандарт)
- Дозвольте Superdav AI Agent автоматично виявляти можливості
- Використовуйте Ability Visibility для контролю доступу

### Для існуючих встановлень {#for-existing-installations}

- Оновлюйте до WordPress 7.0+, коли це можливо
- Перейдіть в Auto mode для спрощення управління
- Перегляньте та класифікуйте можливості за допомогою Ability Visibility

### Для користувацьких можливостей {#for-custom-abilities}

- Реєструйте можливості через WordPress hooks (Abilities API)
- Уникайте користувацьких реєстрів можливостей
- Тестуйте на WordPress 7.0+ з Auto mode

## Наступні кроки {#next-steps}

1. **Перевірте версію WordPress**: Переконайтеся, що ви на 7.0+ для Auto mode
2. **Перегляньте режим сторонніх розробників**: Перейдіть до Settings та перевірте поточний режим
3. **Оновіть за потреби**: Перейдіть в Auto mode, якщо ви на WordPress 7.0+
4. **Класифікуйте можливості**: Перегляньте та класифікуйте будь-які некласифіковані можливості
5. **Тестування**: Переконайтеся, що ваші можливості сторонніх розробників працюють коректно
