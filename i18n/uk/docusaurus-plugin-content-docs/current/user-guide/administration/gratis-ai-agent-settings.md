# Налаштування Gratis AI Agent

Екран **Settings → Advanced** у Gratis AI Agent надає адміністративну конфігурацію для інтеграцій бекенду, запроваджених у версії v1.5.0. На цій сторінці описано поля **Feedback Endpoint** та очікуваний формат даних.

## Доступ до налаштувань

1. У адмін-панелі WordPress перейдіть до **Gratis AI Agent → Settings**.
2. Натисніть вкладку **Advanced**.

## Конфігурація Feedback Endpoint

Endpoint для відгуків отримує POST-запити від AI-агента щоразу, коли користувач надсилає відгук за допомогою кнопки "палець вниз", банера автоматичного промпту або команди `/report-issue`.

| Поле | Опис |
|---|---|
| **Feedback Endpoint URL** | URL, який отримує відгуки як HTTP POST-запити з тілом у форматі JSON. |
| **Feedback API Key** | Bearer-токен, який надсилається в заголовку `Authorization` кожного запиту відгуку. Залиште поле порожнім, якщо ваш endpoint не вимагає автентифікації. |

### Очікуваний JSON Payload

Ваш endpoint для відгуків повинен приймати тіло у форматі JSON, що містить щонайменше такі поля:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Відповідь була невірною щодо ціноутворення.",
  "triage_category": "factual_error"
}
```

Додаткові поля можуть бути присутні в payload залежно від контексту розмови.

### Значення `triage_category`

Шар AI-триажу призначає одне з таких значень для `triage_category` перед переадретацією payload:

| Значення | Значення |
|---|---|
| `factual_error` | Асистент надав неправдиву фактичну інформацію. |
| `unhelpful_answer` | Відповідь була технічно правильною, але не корисною. |
| `inappropriate_content` | Відповідь містила контент, який не слід показувати користувачам. |
| `other` | Відгук не збігся з жодною відомою категорією. |

### Автентифікація

Якщо ваш endpoint вимагає автентифікації, встановіть у полі **Feedback API Key** ваш bearer-токен. Агент надсилає:

```
Authorization: Bearer <your-api-key>
```

Якщо поле **Feedback API Key** порожнє, заголовок `Authorization` не надсилається.

### Вимкнення збору відгуків

Залиште поля **Feedback Endpoint URL** та **Feedback API Key** порожніми. Кнопка "палець вниз" та інтерфейс відгуків залишаються видимими для користувачів, але відправлення даних не перенаправляється жодним зовнішнім сервісом.

## Brave Search API Key

Також на вкладці **Advanced** поле **Brave Search API Key** активує можливість [Internet Search](../configuration/internet-search).

| Поле | Опис |
|---|---|
| **Brave Search API Key** | Ваш API-ключ із панелі розробника Brave Search. Необхідний для активації пошуку в інтернеті в AI-асистента. |

Підпис поля містить клікабельне посилання на сторінку реєстрації Brave Search API. Залиште поле порожнім, щоб вимкнути пошук в інтернеті.

Дивіться [Internet Search](../configuration/internet-search) для документації для кінцевих користувачів щодо цієї функції.

## Feature Flags

Також запроваджені у v1.9.0, вкладка **Settings → Feature Flags** надає перемикачі для необов'язкової функціональності. Кожен прапорець може бути увімкнений або вимкнений на рівні всієї мережі; наразі немає можливості перевизначення на рівні окремого сайту.

### Доступ до Feature Flags

1. У адмін-панелі WordPress перейдіть до **Gratis AI Agent → Settings**.
2. Натисніть вкладку **Feature Flags**.

### Прапори контролю доступу (Access Control Flags)

| Flag | Default | Description |
|---|---|---|
| **Restrict to Administrators** | Off | При увімкненні лише користувачі з роллю `administrator` можуть відкривати чат-панель AI Agent. Усі інші ролі бачитимуть повідомлення "Зверніться до свого адміністратора". |
| **Restrict to Network Admins** | Off | При увімкненні в мережі multisite лише Суперадміністратори можуть користуватися агентом. Індивідуальні адміністратори сайтів блокуються. Має пріоритет над "Restrict to Administrators", якщо обидва увімкнені. |
| **Allow Subscriber Access** | Off | При увімкненні користувачі з роллю `subscriber` можуть використовувати інтерфейс чату, але їх функції обмежені лише читанням (немає створення постів чи зміни налаштувань). |
| **Disable for Non-Members** | Off | Інтегрується зі статусом членства Ultimate Multisite. При увімкненні чат приховується для сайтів, які не мають активного членства. |

### Прапори брендування (Branding Flags)

| Flag | Default | Description |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | Видаляє рядок атрибуції бренду, який відображається внизу віджету чату. Рекомендовано для розгортання під власним брендом (white-label). |
| **Custom Agent Name** | *(blank)* | Замінює стандартну назву "Gratis AI Agent" у заголовку чату та меню адміністратора на назву вашого продукту. Залиште порожнім, щоб використовувати стандартну. |
| **Hide Agent Picker** | Off | При увімкненні користувачі не можуть перемикатися між п'ятьма вбудованими агентами. Поточний агент фіксується на тому, який налаштовано як стандартний у Settings → General. |
| **Use Site Icon as Chat Avatar** | Off | Замінює стандартну іконку AI у заголовку віджету чату на іконку сайту WordPress (встановлюється у Appearance → Customize → Site Identity). |

### Застосування змін

Натисніть **Save Settings** після перемикання будь-якого прапорця. Зміни набувають чинності негайно — не потрібно очищати кеш чи реактивувати плагін.
