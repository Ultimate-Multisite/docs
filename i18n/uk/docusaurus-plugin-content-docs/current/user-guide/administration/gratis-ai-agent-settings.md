# Налаштування Gratis AI Agent

Екран **Settings → Advanced** у Gratis AI Agent надає конфігурацію рівня адміністратора для backend-інтеграцій. На цій сторінці задокументовано пересилання відгуків, ключі постачальників пошуку, налаштування керованого сервісу Superdav, елементи керування Google Calendar, налаштування TextBee SMS і мережеві прапорці функцій.

## Доступ до налаштувань

1. В адмінпанелі WordPress перейдіть до **Gratis AI Agent → Settings**.
2. Натисніть вкладку **Advanced**.

## Конфігурація кінцевої точки відгуків

Кінцева точка відгуків отримує POST-запити від AI агента щоразу, коли користувач надсилає відгук через кнопку з опущеним великим пальцем, банер автоматичного запиту або команду `/report-issue`.

| Поле | Опис |
|---|---|
| **Feedback Endpoint URL** | URL, який отримує надіслані відгуки як HTTP POST-запити з тілом JSON. |
| **Feedback API Key** | bearer token, що надсилається в `Authorization` header кожного запиту відгуку. Залиште порожнім, якщо ваша кінцева точка не потребує автентифікації. |

### Очікуване JSON-навантаження

Ваша кінцева точка відгуків має приймати тіло JSON щонайменше з такими полями:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

У навантаженні можуть бути присутні додаткові поля залежно від контексту розмови.

### Значення `triage_category`

Шар AI-сортування призначає одне з наведених нижче значень для `triage_category` перед пересиланням навантаження:

| Значення | Значення |
|---|---|
| `factual_error` | Асистент надав неправильну фактичну інформацію. |
| `unhelpful_answer` | Відповідь була технічно правильною, але некорисною. |
| `inappropriate_content` | Відповідь містила контент, який не слід показувати користувачам. |
| `other` | Відгук не відповідав відомій категорії. |

### Автентифікація

Якщо ваша кінцева точка потребує автентифікації, задайте в полі **Feedback API Key** ваш bearer token. Агент надсилає:

```
Authorization: Bearer <your-api-key>
```

Якщо поле **Feedback API Key** порожнє, `Authorization` header не надсилається.

### Вимкнення збору відгуків

Залиште поля **Feedback Endpoint URL** і **Feedback API Key** порожніми. Кнопка з опущеним великим пальцем та інтерфейс відгуків залишаються видимими для користувачів, але надсилання не пересилаються до жодного зовнішнього сервісу.

## Brave Search API Key

Також на вкладці **Advanced** поле **Brave Search API Key** вмикає можливість [інтернет-пошуку](../configuration/internet-search).

| Поле | Опис |
|---|---|
| **Brave Search API Key** | Ваш API-ключ із панелі розробника Brave Search. Потрібний для ввімкнення інтернет-пошуку в AI асистенті. |

Мітка поля містить клікабельне посилання на сторінку реєстрації Brave Search API. Залиште порожнім, щоб вимкнути інтернет-пошук.

Див. [Інтернет-пошук](../configuration/internet-search) для документації щодо цієї функції для кінцевих користувачів.

## Керований сервіс Superdav

Superdav AI Agent v1.18.0 додає керовані кінцеві точки сервісу Superdav та автоматичне створення підключення для підтримуваних сайтів. Використовуйте ці елементи керування, коли ваш сайт має підключатися до хостингового постачальника замість кінцевої точки сервісу, налаштованої вручну.

| Поле | Опис |
|---|---|
| **Managed Superdav Service** | Вмикає підключення до хостингового сервісу Superdav для підтримуваних сайтів. |
| **Provision Connection** | Запускає автоматичне створення кінцевої точки та облікових даних. Використовуйте це після підтвердження, що сайт має використовувати керованого постачальника. |
| **Service Endpoint / Connection Status** | Показує поточну кінцеву точку або стан підключення після створення. |

Після створення збережіть налаштування та перевірте стан підключення, перш ніж покладатися на робочі процеси керованого сервісу. Якщо створення не вдалося, перегляньте відображені вказівки щодо повторної спроби та підтвердьте, що сайт має дозвіл використовувати хостингового постачальника.

## Конфігурація Google Calendar

Коли функції календаря Superdav AI Agent v1.18.0 увімкнені, агент може читати налаштовані календарі та деталі подій. Інструменти календаря орієнтовані на читання та корисні для нагадувань з урахуванням розкладу, подальшого зв’язку з учасниками та зіставлення контактів.

| Поле | Опис |
|---|---|
| **Google Calendar Credentials** | Зберігає облікові дані або token-з’єднання, потрібні для читання даних календаря. |
| **Calendar Selection** | Обмежує, які налаштовані календарі агент може переглядати. |
| **Calendar Connection Status** | Підтверджує, чи поточні облікові дані можуть читати календарі та події. |

Обмежуйте облікові дані календаря лише тими календарями, які потрібні агенту. Повторно підключіть або змініть облікові дані, якщо стан вказує на прострочений token.

## TextBee SMS-сповіщення

Superdav AI Agent v1.18.0 додає TextBee як постачальника SMS для налаштованих робочих процесів сповіщень. SMS-сповіщення слід поєднувати з етапами людського схвалення для чутливих або спрямованих на користувачів повідомлень.

| Поле | Опис |
|---|---|
| **TextBee API Key** | Автентифікує запити до постачальника TextBee SMS. |
| **TextBee Device / Sender** | Вибирає відправника або пристрій TextBee, що використовується для вихідних повідомлень, коли цього вимагає постачальник. |
| **SMS Notifications Enabled** | Дозволяє схваленим робочим процесам надсилати сповіщення текстовими повідомленнями. Залиште вимкненим, щоб запобігти надсиланню SMS. |

Надішліть тестове повідомлення лише на номер, що належить адміністратору, а потім підтвердьте поведінку етапу схвалення перед увімкненням запланованих нагадувань або нагадувань для учасників.

## Прапорці функцій

Також запроваджена у v1.9.0 вкладка **Settings → Feature Flags** надає перемикачі для необов’язкової функціональності. Кожен прапорець увімкнений або вимкнений на рівні всієї мережі; наразі немає перевизначення для окремого сайту.

### Доступ до прапорців функцій

1. В адмінпанелі WordPress перейдіть до **Gratis AI Agent → Settings**.
2. Натисніть вкладку **Feature Flags**.

### Прапорці керування доступом

| Прапорець | За замовчуванням | Опис |
|---|---|---|
| **Обмежити для адміністраторів** | Вимкнено | Якщо ввімкнено, лише користувачі з роллю `administrator` можуть відкрити панель чату AI Agent. Усі інші ролі натомість бачать повідомлення "Зверніться до свого адміністратора". |
| **Обмежити для Network Admins** | Вимкнено | Якщо ввімкнено в мультисайтовій мережі, лише Super Admins можуть використовувати агента. Адміністратори окремих сайтів блокуються. Має пріоритет над "Обмежити для адміністраторів", якщо ввімкнено обидва параметри. |
| **Дозволити доступ передплатникам** | Вимкнено | Якщо ввімкнено, користувачі з роллю `subscriber` можуть використовувати інтерфейс чату, але обмежені можливостями лише для читання (без створення записів або зміни налаштувань). |
| **Вимкнути для тих, хто не є учасниками** | Вимкнено | Інтегрується зі статусом членства Ultimate Multisite. Якщо ввімкнено, чат приховано для сайтів, які не мають активного членства. |

### Прапорці брендингу

| Прапорець | За замовчуванням | Опис |
|---|---|---|
| **Приховати футер "Powered by Gratis AI Agent"** | Вимкнено | Видаляє рядок атрибуції брендингу, що відображається внизу віджета чату. Рекомендовано для розгортань із white-label. |
| **Власна назва агента** | *(порожньо)* | Замінює стандартну мітку "Gratis AI Agent" у заголовку чату та меню адміністратора на назву вашого власного продукту. Залиште порожнім, щоб використовувати стандартну. |
| **Приховати вибір агента** | Вимкнено | Якщо ввімкнено, користувачі не можуть перемикатися між п’ятьма вбудованими агентами. Поточний агент фіксується на тому, що налаштовано як стандартний у Settings → General. |
| **Використовувати іконку сайту як аватар чату** | Вимкнено | Замінює стандартну іконку AI у заголовку віджета чату на іконку сайту WordPress (налаштовується в Appearance → Customize → Site Identity). |

### Прапорці безпеки автоматизації

Superdav AI Agent v1.18.0 запроваджує шлюзи схвалення людиною та записи нагадувань для безпечніших запусків автоматизації. Ці елементи керування можуть з’являтися у прапорцях функцій або розширених налаштуваннях автоматизації, залежно від встановленого пакета.

| Прапорець | За замовчуванням | Опис |
|---|---|---|
| **Вимагати схвалення людиною** | Рекомендовано ввімкнути | Призупиняє чутливі автоматизації, доки авторизований користувач не перегляне та не підтвердить запропоновану дію. |
| **Дедуплікація нагадувань** | Увімкнено | Записує надіслані нагадування, щоб повторні спроби або заплановані запуски не надсилали дублікати сповіщень. |
| **Увімкнути інструменти календаря** | Вимкнено, доки не налаштовано | Дозволяє агенту читати налаштовані календарі та події Google. |
| **Увімкнути SMS-сповіщення** | Вимкнено, доки не налаштовано | Дозволяє схваленим робочим процесам надсилати SMS-сповіщення TextBee після збереження облікових даних. |

### Застосування змін

Натисніть **Зберегти налаштування** після перемикання будь-якого прапорця. Зміни набирають чинності негайно — очищення кешу або повторна активація plugin не потрібні.
