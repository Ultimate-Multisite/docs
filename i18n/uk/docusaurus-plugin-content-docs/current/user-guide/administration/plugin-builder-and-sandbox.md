# Plugin Builder & Sandbox

Gratis AI Agent v1.5.0 представляє **Plugin Builder & Sandbox System** — систему, яка дозволяє AI-асистенту генерувати, активувати та керувати WordPress плагінами у вашій мережі, використовуючи безпечне, ізольоване середовище sandbox.

## Overview

Plugin Builder дозволяє AI-асистенту створювати кастомні WordPress плагіни на основі природної мови. Згенеровані плагіни валідуються, зберігаються та активуються всередині шару sandbox, перш ніж вони зможуть вплинути на функціональність робочого сайту.

Сценарії використання включають:

- Генерацію легких утилітарних плагінів без залучення розробників.
- Протопіювання функцій, які вимагають використання WordPress hooks або кастомних типів записів (custom post types).
- Створення короткоживучих скриптів автоматизації для пакетних операцій.

## Generating a Plugin via AI

Щоб згенерувати плагін, відкрийте чат-інтерфейс Gratis AI Agent та опишіть, що вам потрібно. Наприклад:

> "Створи плагін, який додасть кастомне сповіщення адміністратору на дашборд."

AI виконає наступне:

1. Створить PHP-код плагіна за допомогою структурованої генерації коду.
2. Валідує результат на наявність синтаксичних помилок та небезпечних шаблонів.
3. Збереже згенерований плагін у сховищі sandbox.
4. Поверне підтвердження зі слагом плагіна та кнопкою **Activate in Sandbox**.

Ви можете вдосконалити результат, продовживши діалог у тій самій гілці розмови, перш ніж активувати плагін.

## Sandbox Activation

Активувати згенерований плагін у sandbox — це зовсім не те саме, що активувати його в робочій мережі. Sandbox:

- Запускає плагін в ізольованому середовищі WordPress (wp-env).
- Фіксує будь-які помилки PHP, попередження або конфлікти хуків.
- Повідомляє про результат активації назад у чат-інтерфейс.

Щоб активувати плагін у sandbox, натисніть кнопку **Activate in Sandbox** у відповіді AI у чаті або використайте команду:

```
/activate-plugin <plugin-slug>
```

Статусний повідомлення підтвердить, чи вдалася активація, чи ні. У разі збою журнал помилок буде відображено у чат-потоці.

## Managing Generated Plugins

Згенеровані плагіни перелічені у розділі **Gratis AI Agent → Plugin Builder → Manage Plugins**. З цього екрана ви можете:

| Action | Description |
|---|---|
| **View source** | Переглянути повний PHP-код плагіна. |
| **Re-activate in sandbox** | Повторно запустити перевірку активації в sandbox. |
| **Install on network** | Розгорнути плагін у робочу мережу (потрібне ручне підтвердження). |
| **Update** | Надати нову версію через AI, замінивши існуючий код. |
| **Delete** | Видалити плагін зі сховища sandbox. Спочатку деактивує його з усіх сайтів. |

:::warning
**Install on network** розгортає згенерований плагін у вашу робочу WordPress multisite. Перегляньте код плагіна перед продовженням. Gratis AI Agent запросить підтвердження перед завершенням live-інсталяції.
:::

## Installing a Generated Plugin on the Network

Коли ви задоволені плагіном, протестованим у sandbox, ви можете встановити його у робочу мережу:

1. Перейдіть до **Gratis AI Agent → Plugin Builder → Manage Plugins**.
2. Натисніть **Install on Network** поруч із плагіном, який ви хочете розгорнути.
3. Підтвердіть діалог. Плагін встановлюється у `wp-content/plugins/` та активується на рівні мережі.

Альтернативно, використайте команду у чат-інтерфейсі:

```
/install-plugin <plugin-slug>
```

## Plugin Updates

Щоб оновити згенерований плагін, опишіть зміни AI-асистенту в новій розмові:

> "Онови плагін dashboard-notice так, щоб сповіщення бачили лише адміністратори."

AI згенерує нову версію, яка з'явиться у sandbox поруч із поточною версією. Ви переглядаєте відмінності (diff) та підтверджуєте, перш ніж оновлення буде застосовано.

## HookScanner Integration

Plugin Builder використовує інтегрований **HookScanner** для аналізу хуків та фільтрів, зареєстрованих кожним згенерованим плагіном. Результати HookScanner відображаються у відповіді чату та включають:

- Зареєстровані action hooks (`add_action` виклики).
- Зареєстровані filter hooks (`add_filter` виклики).
- Будь-які хуки, знайдені у залежностях плагіна (ігнорує каталоги `vendor/` та `node_modules/`).

Це допомагає вам зрозуміти поведінку плагіна перед його активацією.

## Security Considerations

- Згенеровані плагіни зберігаються окремо від вручну встановлених плагінів і не доступні через стандартний екран керування плагінами WordPress, доки ви явно не встановите їх у мережу.
- Sandbox використовує валідацію шляхів для запобігання обходу каталогів під час запису файлів плагінів.
- Плагіни з небезпечними викликами функцій (наприклад, `eval`, `exec`, `system`) позначаються під час валідації та не будуть активовані.
