# Експорт сайту {#site-export}

Сторінка адміністрування **Експорт сайту** дозволяє адміністраторам мережі пакувати один сайт або всю мережу в архів для міграції, резервного копіювання чи передачі роботи.

## Експорт одного сайту {#exporting-one-site}

Перейдіть до **Ultimate Multisite > Site Export** та виберіть **Generate new Site Export** (Створити новий експорт сайту). Виберіть підсайт, який ви хочете експортувати, а потім вкажіть, чи має архів включати завантаження, плагіни та теми.

Після завершення експорту завантажте ZIP-архів зі списку **Existing Exports** (Існуючі експорти). Експортні ZIP-архіви тепер містять самозавантажувальний `index.php` та `readme.txt`, завдяки чому архів можна завантажити на новий хостинг і запустити без попередньої установки окремого плагіна-імпортера.

## Експорт всієї мережі {#exporting-the-whole-network}

Використовуйте **Network Export** (Експорт мережі) на сторінці Site Export, коли вам потрібен єдиний архів, що містить усі підсайти мережі. Це корисно перед міграцією хостингу, навчальними тренуваннями з відновлення після катастрофи або переналаштуванням середовища, де кожен підсайт має переміщатися разом.

Оскільки експорт мережі може бути значно більшим, ніж експорт одного сайту, виконуйте його у період низького трафіку та переконайтеся, що цільове сховище має достатньо вільного місця для завантажень, плагінів, тем та згенерованих ZIP-архівів.

### Пакети імпорту мережі {#network-import-bundles}

Починаючи з Ultimate Multisite 2.12.0, Site Exporter може створювати **network import bundles** (пакети імпорту мережі) — спеціалізовані архіви, розроблені для спрощеного відновлення сайтів на рівні всієї мережі. Пакет імпорту мережі містить усі необхідні файли та метадані для відновлення кількох сайтів на новій інсталяції мережі.

#### Створення пакета імпорту мережі {#generating-a-network-import-bundle}

1. Перейдіть до **Ultimate Multisite > Site Export**
2. Натисніть **Generate new Network Export** (Створити новий експорт мережі)
3. Виберіть **Network Import Bundle** як тип експорту
4. Виберіть, чи включати завантаження, плагіни та теми
5. Натисніть **Generate** (Створити)
6. Завантажте ZIP-пакет зі списку **Existing Exports** (Існуючі експорти)

#### Відновлення з пакета імпорту мережі {#restoring-from-a-network-import-bundle}

Щоб відновити сайти з пакета імпорту мережі:

1. Встановіть Ultimate Multisite на ваш цільовий хостинг
2. Завершіть майстер налаштування multisite
3. Перейдіть до **Ultimate Multisite > Site Export** на новій мережі
4. Натисніть **Import Network Bundle** (Імпортувати пакет мережі)
5. Завантажте ZIP-файл пакета імпорту мережі
6. Дотримуйтесь інструкцій імпорту на екрані
7. Процес імпорту відновить усі сайти, їхній контент та налаштування

Пакети імпорту мережі зберігають:
- Контент сайту (дописи, сторінки, користувацькі типи записів)
- Налаштування та опції сайту
- Ролі користувачів та дозволи
- Плагіни та теми (якщо включено в пакет)
- Медіа-завантаження (якщо включено в пакет)
- Користувацькі таблиці та дані бази даних

## Встановлення самозавантажувального ZIP-архіву експорту {#installing-a-self-booting-export-zip}

Щоб відновити самозавантажувальний ZIP-архів на новому хостингу:

1. Завантажте вміст експортованого ZIP-архіву до кореневого каталогу веб-сайту.
2. Відкрийте завантажений `index.php` у браузері.
3. Дотримуйтесь інструкцій встановлення на екрані з пакета експорту.
4. Ознайомтеся з пакетованим `readme.txt`, щоб знати особливості експорту, перш ніж видаляти тимчасові файли.

Щоб дізнатися деталі встановлення та імпорту для конкретного аддону, перегляньте [Site Exporter addon documentation](/addons/site-exporter).

Для інструменту односайтового експорту, доданого в Ultimate Multisite 2.9.0, перегляньте [Export & Import](/user-guide/administration/export-import).
