# Wielodzierżawność {#multi-tenancy}

Dystrybuuj podwitryny między serwerami i bazami danych w celu zwiększenia wydajności i bezpieczeństwa

## Najważniejsze informacje o wydaniu wersji 1.2.0 {#version-120-release-highlights}

Multi-Tenancy 1.2.0 dodaje fundament suwerennego dzierżawcy dla przepływów pracy izolacji produkcyjnej:

- Bezstanowe automatyczne logowanie dzierżawcy z krótkotrwałymi, zabezpieczonymi przed powtórzeniem wizytami SSO.
- Izolacja bazy danych i systemu plików dla każdej podwitryny w przypadku suwerennych dzierżawców.
- Możliwość wielodzierżawności Hostinger do obsługi udostępniania dzierżawców z uwzględnieniem hosta.
- Bramki weryfikacji suwerennej migracji do kontroli danych starszych systemów i asynchronicznego przetwarzania wypychania.
- Konfigurowalne wiązanie hosta bazy danych na tej samej maszynie dla instalacji Bedrock, FrankenPHP i localhost.
- Ulepszenia inicjalizacji dzierżawcy, zapisu bazy danych, inicjalizacji schematu i udostępniania użytkowników.
- Bezpieczniejsze usuwanie suwerennego dzierżawcy, które czyści dane uwierzytelniające dzierżawcy.

Zobacz przewodniki użytkownika dotyczące [przepływu pracy suwerennej migracji](/user-guide/getting-started/sovereign-migration-workflow), [izolacji wielodzierżawności](/user-guide/configuration/multi-tenancy-isolation) oraz [operacji suwerennego dzierżawcy](/user-guide/administration/sovereign-tenant-operations).

## Wymagania {#requirements}

- WordPress 5.3 lub nowszy
- PHP 7.4 lub nowszy
- plugin Multisite Ultimate (aktywny)

## Instalacja {#installation}

1. Prześlij pliki dodatku do katalogu `/wp-content/plugins/`
2. Aktywuj plugin przez menu „Pluginy” w WordPress
3. Skonfiguruj ustawienia dodatku w panelu administratora sieci

### Testowanie {#testing}

Dodatek zawiera kompleksowy framework testowy:

- **Testy jednostkowe**: Testy oparte na PHPUnit z zestawem testowym WordPress
- **Bazowa klasa testowa**: Zapewnia metody pomocnicze dla typowych scenariuszy testowych
- **Pokrycie testami**: Skonfigurowane do śledzenia pokrycia kodu
- **Testowanie Multisite**: Testy uruchamiane w środowisku multisite

### Standardy kodu {#code-standards}

- **WordPress Coding Standards**: Wymuszane przez PHPCS
- **Zgodność z PHP 7.4+**: Obsługa nowoczesnych funkcji PHP
- **Analiza statyczna**: Integracja PHPStan do sprawdzania typów
- **Modernizacja kodu**: Reguły Rector dla aktualizacji PHP

### System budowania {#build-system}

- **Przetwarzanie zasobów**: Minifikacja plików CSS/JS
- **Tłumaczenie**: Generowanie pliku POT
- **Pakietowanie**: Tworzenie archiwum do dystrybucji
- **Tryb deweloperski**: Niezminifikowane zasoby do debugowania

## Konfiguracja {#configuration}

Dodatek obsługuje różne opcje konfiguracji dostępne przez interfejs administratora sieci.

### Konfiguracja suwerennego dzierżawcy {#sovereign-tenant-configuration}

W przypadku suwerennych dzierżawców skonfiguruj bazę danych dzierżawcy, katalog główny systemu plików, wiązanie hosta bazy danych oraz możliwość hosta przed skierowaniem ruchu do dzierżawcy. Uruchamiaj weryfikację migracji po każdej zmianie, która wpływa na dane uwierzytelniające bazy danych, wiązanie hosta, routing lub udostępnianie użytkowników.

Używaj `localhost` tylko wtedy, gdy środowisko wykonawcze dzierżawcy i uprawnienia bazy danych oczekują dostępu przez lokalne gniazdo lub dostęp z tej samej maszyny. Używaj `127.0.0.1` albo nazwy hosta usługi, gdy jest to host uprawnień skonfigurowany przez Twojego dostawcę.

### Możliwość Hostinger {#hostinger-capability}

Możliwość wielodzierżawności Hostinger działa ze współdzielonym tokenem integracji Hostinger. Potwierdź, że domena jest obecna w hPanel, DNS wskazuje na Account Hostinger, a ustawienia bazy danych dzierżawcy są zgodne z hPanel przed weryfikacją.

## Hooki i filtry {#hooks-and-filters}

### Akcje {#actions}

- `ultimate-multisite-addon-template_init` - Wywoływane po inicjalizacji dodatku
- `ultimate-multisite-addon-template_loaded` - Wywoływane po załadowaniu wszystkich plików dodatku

### Filtry {#filters}

- `ultimate-multisite-addon-template_settings` - Modyfikuj ustawienia dodatku
- `ultimate-multisite-addon-template_enabled` - Nadpisz stan włączenia/wyłączenia dodatku
