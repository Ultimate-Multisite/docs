# WooCommerce Integration

Gumagana ang WooCommerce Integration sa paraang halos kapareho ng paggana ng aming Manu-manong Gateway. Nag-aalok ito ng napakaraming flexibility sa iyong mga user, na nagpapahintulot sa kanila na pumili ng alinman sa mga form ng pagbabayad na available sa iyong WooCommerce install. At simple ito: kapag natapos na ang panahon ng pagsingil ng user, ang suskripsyon ng user ay ilalagay muna sa hold. Kasabay nito, gagawa ng order sa WooCommerce install ng pangunahing site ng network. Pagkatapos ay makatatanggap ang user ng invoice sa kanilang email, na may link para isagawa ang pagbabayad. Kapag natanggap na ang pagbabayad, mare-renew ang suskripsyon.

## Mga Tampok {#features}

- Madaling integrasyon sa pagitan ng Ultimate Multisite at WooCommerce
- Ilang paraan ng pagbabayad na available
- Awtomatikong umuulit na pagsingil gamit ang kasamang subscriptions core, na may opsiyonal na mga pagpapahusay mula sa WooCommerce Subscriptions plugin
- Kasama sa kabuuan sa pag-checkout ang mga buwis ng Ultimate Multisite bilang magkakahiwalay at hindi nabubuwisang linya ng bayarin sa WooCommerce upang masuri ng mga customer ang mga halaga ng buwis bago magbayad

## Gabay sa Paggamit {#usage-guide}

_**MAHALAGANG TALA: Tumutukoy ang artikulong ito sa Ultimate Multisite bersyon 2.x.**_

_**PAALALA:** Kinakailangan ng Ultimate Multisite: WooCommerce Integration na naka-activate ang WooCommerce kahit man lang sa iyong pangunahing site._

Nauunawaan namin na ang _Stripe_ at _PayPal_ ay hindi available sa ilang bansa, na naglilimita o humahadlang sa mga user ng Ultimate Multisite na magamit nang epektibo ang aming plugin. Kaya gumawa kami ng **add-on upang i-integrate ang _WooCommerce_**.

Ang _WooCommerce_ ay isang napakapopular na plugin para sa eCommerce. Gumawa ang mga developer sa buong mundo ng mga add-on upang i-integrate dito ang iba’t ibang gateway ng pagbabayad. Sinamantala namin ito upang palawakin ang mga gateway ng pagbabayad na maaari mong gamitin sa sistema ng pagsingil ng Ultimate Multisite.

Una, kailangan mong i-install ang WooCommerce Integration add-on na **libre para sa mga lisensyadong user**. Upang i-install ito, mag-click sa Ultimate Multisite sa iyong super admin dashboard at piliin ang **Add-ons**.

Kapag na-install na, makakakita ka ng karagdagang opsiyon sa ilalim ng mga setting ng pagbabayad ng Ultimate Multisite. Pagkatapos ay maaari mong **i-enable** ang opsiyong _WooCommerce_ mula roon.

### Mga linya ng bayarin sa buwis sa pag-checkout {#checkout-tax-fee-lines}

Kapag may kasamang buwis ang pag-checkout sa Ultimate Multisite, idinaragdag ng integration ang bawat halaga ng buwis ng Ultimate Multisite sa pag-checkout ng WooCommerce bilang hiwalay at hindi nabubuwisang linya ng bayarin. Pinananatili nitong nakikita ang kabuuang buwis bago magbayad habang pinipigilan ang WooCommerce na buwisan muli ang bayarin sa buwis.

## Paano pamahalaan ang pagbabayad {#how-to-manage-the-payment}

Gumagana ang WooCommerce Integration sa paraang halos kapareho ng paggana ng aming **Manu-manong Gateway**. Ang disbentahe ay kailangang bayaran nang manu-mano ang bawat pagbabayad sa katapusan ng panahon ng pagsingil. Ganito gumagana ang daloy:

  * Kapag natapos na ang billing cycle ng user, ang membership ay ilalagay **sa hold**. Kasabay nito, gagawa ng order sa _WooCommerce_ install sa pangunahing site ng iyong network.

  * Pagkatapos ay makatatanggap ang mga user ng invoice sa kanilang email, na may link para isagawa ang pagbabayad (lumalabas din ang link sa kanilang billing history mula sa kanilang dashboard ng subsite). Ire-redirect ng link na iyon ang user sa _**WooCommerce**_ **pag-checkout** , kung saan makakapili ang mga user ng isa sa mga **available na _WooCommerce_ gateway** upang iproseso ang kanilang pagbabayad.

  * Kapag natanggap na ang pagbabayad, ang membership ay **mare-renew** , mala-log ang pagbabayad sa Ultimate Multisite at itatakda ang order bilang kumpleto.

### Ilang Mahahalagang Tala: {#some-important-notes}

  * Gumagana pa rin ang default na functionality ng Ultimate Multisite sa form ng integration na ito. Halimbawa, ang **pag-isyu ng mga refund** mula sa screen ng **Pamamahala ng Suskripsyon** ay makikipag-ugnayan sa WooCommerce upang i-isyu ang refund gamit ang napiling form ng pagbabayad.

  * Umaasa ang Ultimate Multisite: WooCommerce Integration sa WP-Cron upang gumana. Gumagana ito nang napakahusay sa karamihan ng pagkakataon, ngunit kung mababa ang traffic ng site, hindi nata-trigger ang cron na tingnan ang mga naka-schedule na event at nalalampasan ang event, na maaaring magdulot ng mga pagkaantala sa pagpapatupad.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

Ang mano-manong pagsisimula ng bayad bawat buwan ay hindi mainam para sa mga end-user dahil inaasahang awtomatikong nauulit ang mga subscription sa bawat billing cycle. Kasama na ngayon sa WooCommerce Integration add-on ang core subscriptions code, kaya palaging gumagamit ng subscriptions ang mga umuulit na produkto para sa umuulit na billing.

**Hindi** mo kailangang bilhin o i-install ang hiwalay na [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) para magbenta ng mga umuulit na produkto sa pamamagitan ng integrasyong ito. Kung nabili, na-install, at available ang plugin na iyon sa iyong pangunahing site, gagamitin ito ng Ultimate Multisite at magpapakita ang WooCommerce ng karagdagang mga feature sa pamamahala ng subscription. Kapag hindi ito available, ang kasamang subscriptions core pa rin ang hahawak ng mga umuulit na produkto.

![Mga setting ng integrasyon ng subscription ng WooCommerce](/img/addons/woocommerce-subscription-integration.png)

At iyon na iyon. Hindi na kakailanganin ng iyong mga end-user na mano-manong magbayad sa katapusan ng kanilang billing cycle para sa mga umuulit na produkto.

Bukod sa paggawa ng isang order sa iyong pangunahing site sa ilalim ng _WooCommerce_, gagawa rin ito ng subscription para sa bawat account.

### Gawi sa pagsususpinde kapag nabigong pag-renew {#failed-renewal-suspension-behavior}

Bilang default, nananatiling aktibo ang mga membership habang pinoproseso ng WooCommerce Subscriptions ang iskedyul nito ng muling pagsubok para sa nabigong bayad. Pinapanatili nito ang normal na grace window ng WooCommerce Subscriptions upang maibalik ng isang na-recover na pag-renew ang subscription at membership sa pagkakasync nang hindi napaaga ang pagkaantala ng access.

Maaaring i-enable ng mga administrator na nais ng mas mahigpit na kontrol sa access ang **Suspend Memberships Immediately on Failed Renewals** sa mga setting ng WooCommerce Integration. Kapag naka-enable ang opt-in na setting na ito, sinususpinde ng Ultimate Multisite ang kaugnay na membership sa sandaling mabigo ang bayad sa pag-renew sa halip na hintayin matapos ang retry handling ng WooCommerce Subscriptions.

Maaaring i-override ng mga developer ang parehong desisyon gamit ang `wu_woo_suspend_on_payment_failure` filter. Ang mga pahina ng hook reference ay ginagawa mula sa source ng add-on, kaya binabanggit lamang ng overview na ito ang pangalan ng filter para sa mga custom na patakaran sa pagsususpinde.
