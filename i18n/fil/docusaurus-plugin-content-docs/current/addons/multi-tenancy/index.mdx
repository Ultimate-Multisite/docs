# Multi-Tenancy

Ipamahagi ang mga subsite sa mga server at database para sa performance at seguridad

## Mga tampok sa release ng Version 1.2.0

Nagdaragdag ang Multi-Tenancy 1.2.0 ng pundasyon para sa sovereign tenant para sa mga workflow ng production isolation:

- Stateless tenant autologin na may panandalian at protektado laban sa replay na mga pagbisita sa SSO.
- Per-subsite na isolation ng database at filesystem para sa mga sovereign tenant.
- Kakayahan ng Hostinger multi-tenancy para sa host-aware na tenant provisioning.
- Mga gate ng beripikasyon ng sovereign migration para sa mga legacy data check at async push processing.
- Nako-configure na same-machine database host binding para sa mga install ng Bedrock, FrankenPHP, at localhost.
- Mga pagpapahusay sa tenant bootstrap, database writer, schema bootstrap, at user provisioning.
- Mas ligtas na sovereign teardown na naglilinis ng mga kredensyal ng tenant.

Tingnan ang mga gabay ng user para sa [workflow ng sovereign migration](/user-guide/getting-started/sovereign-migration-workflow), [isolation ng multi-tenancy](/user-guide/configuration/multi-tenancy-isolation), at [mga operasyon ng sovereign tenant](/user-guide/administration/sovereign-tenant-operations).

## Mga Kinakailangan

- WordPress 5.3 o mas mataas
- PHP 7.4 o mas mataas
- Multisite Ultimate plugin (aktibo)

## Pag-install

1. I-upload ang mga file ng addon sa iyong direktoryong `/wp-content/plugins/`
2. I-activate ang plugin sa pamamagitan ng menu na 'Mga Plugin' sa WordPress
3. I-configure ang mga setting ng addon sa network admin

## Configuration

Sinusuportahan ng addon ang iba't ibang opsyon sa configuration na maa-access sa pamamagitan ng network admin interface.

### Configuration ng sovereign tenant

Para sa mga sovereign tenant, i-configure ang database ng tenant, filesystem root, database host binding, at kakayahan ng host bago i-route ang traffic sa tenant. Patakbuhin ang beripikasyon ng migration pagkatapos ng bawat pagbabagong nakakaapekto sa mga kredensyal ng database, host binding, routing, o user provisioning.

Gamitin ang `localhost` lamang kapag inaasahan ng tenant runtime at database grants ang local socket o same-machine access. Gamitin ang `127.0.0.1` o isang service hostname kapag iyon ang grant host na naka-configure ng iyong provider.

### Kakayahan ng Hostinger

Gumagana ang kakayahan ng Hostinger multi-tenancy sa shared Hostinger integration token. Kumpirmahing naroon ang domain sa hPanel, nakaturo ang DNS sa Hostinger account, at tumutugma ang mga setting ng database ng tenant sa hPanel bago ang beripikasyon.

## Mga Hook at Filter

### Mga Action

- `ultimate-multisite-addon-template_init` - Pinaandar pagkatapos ng initialization ng addon
- `ultimate-multisite-addon-template_loaded` - Pinaandar pagkatapos ma-load ang lahat ng file ng addon

### Mga Filter

- `ultimate-multisite-addon-template_settings` - Baguhin ang mga setting ng addon
- `ultimate-multisite-addon-template_enabled` - I-override ang status ng pag-enable/pag-disable ng addon
