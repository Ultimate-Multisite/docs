# Μια πρώτη ματιά στα Webhooks (v2)

_**ΠΡΟΣΟΧΗ: Σημειώστε ότι αυτή η λειτουργία ή το άρθρο προορίζεται για προχωρημένους χρήστες.**_

Ένα **webhook** είναι ένας τρόπος ώστε μια εφαρμογή ή λογισμικό όπως το Ultimate Multisite να παρέχει σε άλλες εφαρμογές πληροφορίες σε πραγματικό χρόνο. Ένα webhook παραδίδει δεδομένα ή payloads σε άλλες εφαρμογές τη στιγμή που συμβαίνει κάτι, πράγμα που σημαίνει ότι **λαμβάνετε δεδομένα αμέσως.**

Αυτό είναι χρήσιμο αν χρειάζεται να ενσωματώσετε ή να μεταβιβάσετε συγκεκριμένα δεδομένα από το Ultimate Multisite προς ένα άλλο CRM ή σύστημα κάθε φορά που ενεργοποιείται ένα συμβάν. Για παράδειγμα, χρειάζεται να στείλετε το όνομα και τη διεύθυνση email του χρήστη σε μια λίστα αλληλογραφίας κάθε φορά που δημιουργείται ένας νέος λογαριασμός χρήστη.

## Πώς να δημιουργήσετε ένα webhook {#how-to-create-a-webhook}

Για να δημιουργήσετε ένα webhook, μεταβείτε στο dashboard διαχείρισης του δικτύου σας. Κάντε κλικ στο **Ultimate Multisite > Webhooks > Add New Webhook.**

![Κενή σελίδα λίστας Webhooks με κουμπί Add New Webhook](/img/admin/webhooks-list-empty.png)

Στη συνέχεια μπορείτε να επεξεργαστείτε τη διαμόρφωση του webhook:

![Φόρμα Add New Webhook με πεδία Name, Event και URL](/img/admin/webhook-add-modal.png)

Κατά τη δημιουργία ενός νέου webhook θα σας ζητηθούν πληροφορίες όπως **Name, URL,** και **Event**. Μπορείτε να χρησιμοποιήσετε οποιοδήποτε όνομα θέλετε για το webhook σας. Τα πιο σημαντικά πεδία είναι το URL και το Event.

![Διεπαφή επεξεργασίας webhook που εμφανίζει το πεδίο URL και την προεπισκόπηση payload](/img/admin/webhook-url-field.png)

Το URL είναι το **endpoint ή ο προορισμός** στον οποίο το Ultimate Multisite θα στείλει το **payload ή τα δεδομένα**. Αυτή είναι η εφαρμογή που θα λάβει τα δεδομένα.

Το Zapier είναι η πιο συνηθισμένη λύση που χρησιμοποιούν οι χρήστες για να κάνουν ευκολότερη την ενσωμάτωση με εφαρμογή τρίτου μέρους. Χωρίς μια πλατφόρμα όπως το Zapier, θα χρειαστεί να δημιουργήσετε χειροκίνητα μια προσαρμοσμένη συνάρτηση που θα συλλαμβάνει τα δεδομένα και θα τα επεξεργάζεται. Δείτε αυτό το άρθρο για το **πώς να χρησιμοποιήσετε το Ultimate Multisite webhook με το Zapier.**

Σε αυτό το άρθρο, θα εξετάσουμε τη βασική έννοια του πώς λειτουργεί ένα webhook και τα συμβάντα που είναι διαθέσιμα στο Ultimate Multisite. Θα χρησιμοποιήσουμε έναν ιστότοπο τρίτου μέρους που ονομάζεται [requestbin.com](https://requestbin.com/). Αυτός ο ιστότοπος θα μας επιτρέψει να δημιουργήσουμε ένα endpoint και να συλλάβουμε το payload χωρίς να γράψουμε καθόλου κώδικα. _**Αποποίηση ευθύνης: το μόνο που θα κάνει είναι να μας δείξει ότι τα δεδομένα έχουν ληφθεί.**_ Δεν θα γίνει καμία επεξεργασία ή οποιοδήποτε είδος ενέργειας στο payload.

Μεταβείτε στο [requestbin.com](https://requestbin.com/) και κάντε κλικ στο Create Request Bin.

Αφού κάνετε κλικ σε αυτό το κουμπί, θα σας ζητήσει να συνδεθείτε αν έχετε ήδη λογαριασμό ή να εγγραφείτε. Αν έχετε ήδη λογαριασμό, θα σας οδηγήσει απευθείας στο dashboard τους. Στο dashboard τους, θα δείτε αμέσως το endpoint ή το URL που μπορείτε να χρησιμοποιήσετε για τη δημιουργία του Ultimate Multisite webhook σας.

Προχωρήστε και αντιγράψτε το URL και επιστρέψτε στο Ultimate Multisite. Τοποθετήστε το endpoint στο πεδίο URL και επιλέξτε ένα συμβάν από το αναπτυσσόμενο μενού. Σε αυτό το παράδειγμα, θα επιλέξουμε **Payment Received**.

Αυτό το συμβάν ενεργοποιείται κάθε φορά που ένας χρήστης κάνει μια πληρωμή. Όλα τα διαθέσιμα συμβάντα, η περιγραφή τους και τα payloads παρατίθενται στο κάτω μέρος της σελίδας. Κάντε κλικ στο κουμπί **Add New Webhook** για να αποθηκεύσετε το webhook.

![Αναπτυσσόμενο μενού συμβάντων webhook με επιλεγμένο το Payment Received](/img/admin/webhook-event-picker.png)

Τώρα μπορούμε να στείλουμε ένα δοκιμαστικό συμβάν στο endpoint για να δούμε αν το webhook που δημιουργήσαμε λειτουργεί. Μπορούμε να το κάνουμε αυτό κάνοντας κλικ στο **Send Test Event** κάτω από το webhook που δημιουργήσαμε.

![Λίστα Webhooks που εμφανίζει ένα διαμορφωμένο webhook και την ενέργεια Send Test](/img/admin/webhooks-list-populated.png)

Αυτό εμφανίζει ένα παράθυρο επιβεβαίωσης που λέει ότι η δοκιμή ήταν επιτυχής.

![Αποτέλεσμα δοκιμαστικού συμβάντος webhook μετά την αποστολή δοκιμαστικού payload](/img/admin/webhook-test-result.png)

Τώρα, αν επιστρέψουμε στον ιστότοπο _Requestbin_, θα δούμε ότι το payload έχει ληφθεί και περιέχει ορισμένα δοκιμαστικά δεδομένα.

Αυτή είναι η βασική αρχή του πώς λειτουργούν τα webhook και τα endpoints. Αν πρόκειται να δημιουργήσετε ένα προσαρμοσμένο endpoint, θα χρειαστεί να δημιουργήσετε μια προσαρμοσμένη συνάρτηση για να επεξεργάζεται τα δεδομένα που λαμβάνετε από το Ultimate Multisite.
