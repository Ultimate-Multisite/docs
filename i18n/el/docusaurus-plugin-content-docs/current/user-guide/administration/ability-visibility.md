# Ορατότητα Ικανοτήτων (Ability Visibility) {#ability-visibility}

Το Superdav AI Agent v1.12.0 εισάγει τους ελέγχους **Ορατότητας Ικανοτήτων (Ability Visibility)** που καθορίζουν ποιες επιφάνειες εκθέτουν κάθε ικανότητα. Αυτό επιτρέπει στους διαχειριστές να προσαρμόσουν με ακρίβεια ποιες δυνατότητες των agents είναι διαθέσιμες μέσω REST APIs, MCP servers, ενσωμαρώσεων WooCommerce και άλλων διεπαφών.

## Τι είναι η Ορατότητα Ικανοτήτων; {#what-is-ability-visibility}

Η Ορατότητα Ικανοτήτων είναι ένα σύστημα δικαιωμάτων που ελέγχει:

- **Ποιες ικανότητες** είναι διαθέσιμες στους agents
- **Πού εκτίθενται** (REST API, MCP, WooCommerce, κ.λπ.)
- **Ποιος μπορεί να έχει πρόσβαση** σε αυτές (μέσω λίστας επιτρεπόμενων συνεργατών)
- **Πώς τα ταξινομούνται** (αναγνωρισμένες έναντι μη ταξινομημένων)

Κάθε ικανότητα φέρει ένα επίπεδο ορατότητας που καθορίζει τη διαθεσιμότητά της σε διαφορετικές επιφάνειες.

## Επίπεδα Ορατότητας {#visibility-levels}

### Δημόσια (Public) {#public}

Οι **Δημόσιες ικανότητες** είναι διαθέσιμες παντού:

- Στα endpoints του REST API
- Στους MCP servers
- Στις ενσωμαρώσεις WooCommerce
- Στις διεπαφές chat
- Σε όλους τους ρόλους χρηστών (με τα κατάλληλα δικαιώματα)

Παράδειγμα: `scaffold-block-theme`, `activate-theme`, `send-email`

### Εσωτερικά (Internal) {#internal}

Οι **Εσωτερικές ικανότητες** είναι διαθέσιμες μόνο εντός της εγκατάστασής σας WordPress:

- Στις διεπαφές chat
- Στα πάνελ διαχειριστών (Admin panels)
- Μόνο σε συνδεδεμένους χρήστες
- Δεν εκτίθενται μέσω REST API ή εξωτερικών ενσωματώσεων

Παράδειγμα: `manage-settings`, `view-analytics`, `export-data`

### Συνεργάτη (Partner) {#partner}

Οι **Συνεργατικές ικανότητες** είναι διαθέσιμες μόνο σε συνεργάτες που έχουν καταχωρηθεί στη λίστα επιτρεπόμενων:

- Απαιτείται ρητή καταχώριση στη λίστα επιτρεπόμενων συνεργατών
- Διαθέσιμες μέσω REST API με τα στοιχεία ταυτοποίησης του συνεργάτη
- Διαθέσιμες μέσω MCP με τα στοιχεία ταυτοποίησης του συνεργάτη
- Δεν είναι διαθέσιμες στους κανονικούς χρήστες

Παράδειγμα: `bulk-import-users`, `modify-billing`, `access-analytics`

### Απενεργοποιημένες (Disabled) {#disabled}

Οι **Απενεργοποιημένες ικανότητες** δεν είναι διαθέσιμες πουθενά:

- Δεν εκτίθενται μέσω καποιασδήποτε επιφάνειας
- Δεν είναι διαθέσιμες στο chat
- Δεν είναι διαθέσιμες στα πάνελ διαχειριστών
- Χρήσιμες για ξεπερασμένες ή πειραματικές ικανότητες

## Διαχείριση Ορατότητας Ικανοτήτων {#managing-ability-visibility}

### Πρόσβαση στις Ρυθμίσεις Ορατότητας Ικανοτήτων {#accessing-ability-visibility-settings}

1. Μεταβείτε στο **WordPress Admin** → **Superdav AI Agent** → **Settings**
2. Κάντε κλικ στην καρτέλα **Abilities**
3. Θα δείτε μια λίστα με όλες τις εγκατεστημένες ικανότητες με τα αντίστοιχα επίπεδα ορατότητας

### Προβολή Λεπτομερειών Ικανότητας {#viewing-ability-details}

Κάντε κλικ σε οποιαδήποτε ικανότητα για να δείτε:

- **Name**: Ο αναγνωριστικός κωδικός της ικανότητας
- **Description**: Τι κάνει η ικανότητα
- **Current visibility**: Δημόσια, Εσωτερικά, Συνεργατικά ή Απενεργοποιημένη
- **Partner allow-list**: Σε ποιους συνεργάτες μπορεί να έχει πρόσβαση (αν είναι Συνεργατική ορατότητα)
- **Last updated**: Πότε άλλαξε τελευταία η ορατότητα
- **Status**: Αναγνωρισμένη ή Μη ταξινομημένη

### Αλλαγή Επιπέδων Ορατότητας {#changing-visibility-levels}

Για να αλλάξετε την ορατότητα μιας ικανότητας:

1. Κάντε κλικ στην ικανότητα στη λίστα
2. Επιλέξτε ένα νέο επίπεδο ορατότητας από το αναπτυσσόμενο μενού
3. Εάν επιλέξετε "Partner", προσθέστε τους αναγνωριστικούς κωδικούς των συνεργατών στη λίστα επιτρεπόμενων
4. Κάντε κλικ στο **Save**

Παράδειγμα:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### Συλλογικές Λειτουργίες (Bulk Operations) {#bulk-operations}

Για να αλλάξετε την ορατότητα για πολλαπλές ικανότητες:

1. Επιλέξτε τα κουτάκια δίπλα στις ικανότητες
2. Επιλέξτε ένα νέο επίπεδο ορατότητας από το αναπτυσσόμενο μενού των συλλογικών ενεργειών
3. Κάντε κλικ στο **Apply**

## Λίστα Επιτρεπόμενων Συνεργατών (Partner Allow-List) {#partner-allow-list}

Η **λίστα επιτρεπόμενων συνεργατών** ελέγχει ποιοι εξωτερικοί συνεργάτες μπορούν να έχουν πρόσβαση σε ικανότητες επιπέδου Partner.

### Προσθήκη Συνεργατών {#adding-partners}

1. Μεταβείτε στο **Superdav AI Agent** → **Settings** → **Partners**
2. Κάντε κλικ στο **Add Partner**
3. Εισάγετε τον αναγνωριστικό κωδικό του συνεργάτη (συνήθως ένα API key ή έναν κωδικό οργάνωσης)
4. Προαιρετικά προσθέστε ένα όνομα και μια περιγραφή συνεργάτη
5. Κάντε κλικ στο **Save**

### Καταχρισμός Ικανοτήτων στους Συνεργάτες {#assigning-abilities-to-partners}

Μετά την προσθήκη ενός συνεργάτη:

1. Μεταβείτε στην καρτέλα **Abilities**
2. Βρείτε μια ικανότητα με Συνεργατική ορατότητα
3. Κάντε κλικ για επεξεργασία
4. Στην ενότητα "Partner allow-list", επιλέξτε τα κουτάκια για τους συνεργάτες που πρέπει να έχουν πρόσβαση
5. Κάντε κλικ στο **Save**

### Απαίρεση Πρόσβασης Συνεργάτη {#revoking-partner-access}

Για να αφαιρέσετε την πρόσβαση ενός συνεργάτη:

1. Επεξεργαστείτε την ικανότητα
2. Απολέξτε το κουτάκι του συνεργάτη από τη λίστα επιτρεπόμενων
3. Κάντε κλικ στο **Save**

Ο συνεργάτης θα χάσει άμεσα την πρόσβαση σε αυτήν την ικανότητα.

## Μη Ταξινομημένες Ικανότητες (Unclassified Abilities) {#unclassified-abilities}

Όταν εγκαθιστάτε μια ικανότητα τρίτου μέρους που το Superdav AI Agent δεν αναγνωρίζει, αυτή σημειώνεται ως **Μη Ταξινομημένη (Unclassified)**.

### Ειδοποίηση Διαχειριστή για Μη Ταξινομημένες Ικανότητες {#admin-notice-for-unclassified-abilities}

Θα δείτε μια ειδοποίηση διαχειριστή:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### Επισκόπηση Μη Ταξινομημένων Ικανοτήτων {#reviewing-unclassified-abilities}

1. Κάντε κλικ στο **Review Abilities** στην ειδοποίηση
2. Για κάθε μη ταξινομημένη ικανότητα, θα δείτε:
   - Όνομα και περιγραφή της ικανότητας
   - Πηγή (ποιο plugin/addon την εγκατέστησε)
   - Προτεινόμενο επίπεδο ορατότητας
   - Αξιολόγηση ασφαλείας

3. Επιλέξτε ένα επίπεδο ορατότητας:
   - **Public**: Εμπιστευτείτε την ικανότητα· την εκθέτετε παντού
   - **Internal**: Περιορίζετε μόνο για εσωτερική χρήση
   - **Partner**: Περιορίζετε σε συγκεκριμένους συνεργάτες
   - **Disabled**: Μην χρησιμοποιείτε αυτήν την ικανότητα

4. Κάντε κλικ στο **Classify** για να αποθηκεύσετε την απόφαση σας

### Γιατί να Ταξινομήσετε τις Μη Ταξινομημένες Ικανότητες; {#why-classify-unclassified-abilities}

Η ταξινόμηση των ικανοτήτων:

- **Βελτιώνει την ασφάλεια**: Εγκρίνετε ρητά τι εκτίθεται
- **Προλαμβάνει ατυχήματα**: Οι μη ταξινομημένες ικανότητες είναι απενεργοποιημένες εξ ορισμού
- **Ενεργοποιεί λειτουργίες**: Μόλις ταξινομηθούν, η ικανότητα γίνεται διαθέσιμη
- **Καταγράφει αποφάσεις**: Οι επιλογές σας καταγράφονται για σκοπούς ελέγχου (audit)

## Ορατότητα σε Διαφορετικές Επιφάνειες {#visibility-in-different-surfaces}

### REST API {#rest-api}

Οι **Δημόσιες ικανότητες** είναι διαθέσιμες μέσω endpoints του REST:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

Οι **Εσωτερικές και Συνεργατικές ικανότητες** δεν είναι διαθέσιμες μέσω REST API.

### MCP Servers {#mcp-servers}

Οι **Δημόσιες ικανότητες** είναι διαθέσιμες μέσω MCP:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

Οι **Εσωτερικές ικανότητες** δεν εκτίθενται μέσω MCP.

Οι **Συνεργατικές ικανότητες** είναι διαθέσιμες μόνο με τα στοιχεία ταυτοποίησης του συνεργάτη.

### Ενσωμάτωση WooCommerce {#woocommerce-integration}

Οι **Δημόσιες ικανότητες** που σχετίζονται με το WooCommerce είναι διαθέσιμες:

- Διαχείριση προϊόντων
- Επεξεργασία παραγγελιών
- Επικοινωνία με πελάτες

Οι **Εσωτερικές ικανότητες** δεν εκτίθενται στο WooCommerce.

### Διεπαφή Chat {#chat-interface}

**Όλες οι ικανότητες** (Δημόσιες, Εσωτερικές, Συνεργατικές) είναι διαθέσιμες στο chat, ανάλογα με τα δικαιώματα του χρήστη:

- Οι διαχειριστές βλέπουν όλες τις ικανότητες
- Οι κανονικοί χρήστες βλέπουν μόνο τις Δημόσιες ικανότητες
- Οι συνεργάτες βλέπουν τις Δημόσιες + Συνεργατικές ικανότητες (εάν είναι στη λίστα επιτρεπόμενων)

## Καλές Πρακτικές Ασφαλείας {#security-best-practices}

### Αρχή της Ελάχιστης Προνομιαίας Πρόσβασης (Principle of Least Privilege) {#principle-of-least-privilege}

- Ρυθμίστε τις ικανότητες στο πιο περιοριστικό επίπεδο ορατότητας που επιτρέπει ακόμα τη χρήση τους
- Χρησιμοποιήστε την ορατότητα Partner για ευαίσθης λειτουργίες
- Απενεργοποιήστε τις ικανότητες που δεν χρησιμοποιείτε

### Τακτικές Ελέγχους (Regular Audits) {#regular-audits}

- Ελέγχετε την ορατότητα των ικανοτήτων μηνιαίως
- Ψάξτε για μη ταξινομημένες ικανότητες
- Αφαιρέστε την πρόσβαση για τους συνεργάτες που δεν χρησιμοποιούνται

### Καتمادή και Παρακολούθηση (Logging and Monitoring) {#logging-and-monitoring}

- Παρακολουθήστε ποιες ικανότητες χρησιμοποιούνται περισσότερο
- Τηκίστε τα μοτίβα πρόσβασης των συνεργατών
- Στείλτε συναγερμό σε ασυνήθιστη χρήση ικανοτήτων

### Ικανότητες Τρίτου Μέρους (Third-Party Abilities) {#third-party-abilities}

- Ελέγξτε τις ικανότητες τρίτων πριν τις ενεργοποιήσετε
- Ταξινομήστε τις ρητά
- Ξεκινήστε με Εσωτερική ή Συνεργατική ορατότητα
- Προωθήστε σε Δημόσια μόνο αφού τις ελέγξετε πλήρως

## Αντιμετώπιση Προβλημάτων (Troubleshooting) {#troubleshooting}

**Μια ικανότητα δεν εμφανίζεται στη λίστα**
- Επιβεβαιώστε ότι η ικανότητα είναι εγκατεστημένη και ενεργή
- Ελέγξτε ότι το plugin/addon είναι ενεργοποιημένο
- Φρεσκάρετε τη σελίδα

**Δεν μπορώ να αλλάξω την ορατότητα μιας ικανότητας**
- Επιβεβαιώστε ότι έχετε δικαιώματα διαχειριστή
- Ελέγξτε ότι η ικανότητα δεν είναι κλειδωμένη από ένα plugin
- Δοκιμάστε να απενεργοποιήσετε συγκρουόμενα plugins

**Ένας συνεργάτης δεν μπορεί να έχει πρόσβαση σε μια ικανότητα**
- Επιβεβαιώστε ότι ο συνεργάτης βρίσκεται στη λίστα επιτρεπόμενων
- Ελέγξτε ότι η ορατότητα της ικανότητας είναι ρυθμισμένη σε Partner
- Επιβεβαιώστε ότι τα στοιχεία ταυτοποίησης του συνεργάτη είναι σωστά
- Ελέγξτε τα logs του API για σφάλματα ταυτοποίησης

**Οι μη ταξινομημένες ικανότητες συνεχίζουν να εμφανίζονται**
- Ελέγξτε και ταξινομήστε τις στην ειδοποίηση διαχειριστή
- Επιβεβαιώστε ότι η ταξινόμησή σας αποθηκεύτηκε
- Ελέγξτε ότι το plugin που παρέχει την ικανότητα είναι ενημερωμένο

## Μετανάστευση από το Legacy Mode {#migration-from-legacy-mode}

Εάν κάνετε αναβάθμιση από μια παλαιότερη έκδοση του Superdav AI Agent:

- Όλες οι υπάρχουσες ικανότητες ταξινομούνται αυτόματα ως Δημόσιες
- Ελέγξτε και προσαρμόστε την ορατότητα όπως χρειάζεται
- Δεν απαιτείται ενέργεια εκτός αν θέλετε να περιορίσετε την πρόσβαση

Δείτε τη **Μετανάστευση από το Third-Party Mode** για περισσότερες λεπτομέρειες σχετικά με τη μετάβαση στην ενσωμάτωση του native Abilities API.

## Επόμενα Βήματα {#next-steps}

Μετά τη ρύθμιση της ορατότητας των ικανοτήτων:

1. **Ελέγξτε τις ικανότητές σας**: Ελέγξτε όλες τις εγκατεστημένες ικανότητες
2. **Ταξινομήστε τις μη ταξινομημένες ικανότητες**: Απαντήστε στις ειδοποιήσεις διαχειριστή
3. **Ρυθμίστε τους συνεργάτες**: Προσθέστε συνεργάτες εάν χρησιμοποιείτε Συνεργατική ορατότητα
4. **Παρακολουθήστε τη χρήση**: Τηκίστε ποιες ικανότητες χρησιμοποιούνται περισσότερο
5. **Καταγράψτε τις αποφάσεις**: Κρατήστε σημειώσεις για το γιατί επιλέξατε κάθε επίπεδο ορατότητας
