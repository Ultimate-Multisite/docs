# Ρυθμίσεις AI Agent Gratis

Η οθόνη **Ρυθμίσεις → Προηγμένα** του Gratis AI Agent παρέχει διαμόρφωση επιπέδου διαχειριστή για τις ενσωματώσεις backend που εισήχθησαν στην έκδοση v1.5.0. Αυτή η σελίδα τεκμηριώνει τα πεδία του **Feedback Endpoint** και το αναμενόμενο τους μορφές τους.

## Πρόσβαση στις Ρυθμίσεις

1. Στο admin του WordPress, μεταβείτε στο **Gratis AI Agent → Ρυθμίσεις**.
2. Κάντε κλικ στην καρτέλα **Advanced**.

## Διαμόρφωση Feedback Endpoint

Το feedback endpoint λαμβάνει αιτήματα POST από το AI agent κάθε φορά που ένας χρήστης υποβάλλει σχόλιο μέσω του κουμπιού "thumbs-down", του banner αυτο-προτροπής ή της εντολής `/report-issue`.

| Πεδίο | Περιγραφή |
|---|---|
| **Feedback Endpoint URL** | Η URL που λαμβάνει τις υποβολές σχολίων ως αιτήματα HTTP POST με σώμα JSON. |
| **Feedback API Key** | Ένα bearer token που αποστέλεται στο `Authorization` header κάθε αιτήματος σχολίου. Αφήστε το κενό αν το endpoint σας δεν απαιτεί πιστοποίηση. |

### Αναμενόμενο JSON Payload

Το feedback endpoint σας πρέπει να αποδέχεται ένα σώμα JSON με τουλάχιστον τα ακόλουθα πεδία:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "Η απάντηση ήταν λανθασμένη σχετικά με τις τιμές.",
  "triage_category": "factual_error"
}
```

Μπορεί να υπάρχουν επιπλέον πεδία στο payload ανάλογα με το πλαίσιο της συνομιλίας.

### Τιμές `triage_category`

Το επίπεδο triage του AI εκχωρεί μία από τις ακόλουθες τιμές στο `triage_category` πριν περάσει το payload:

| Τιμή | Σημασία |
|---|---|
| `factual_error` | Ο βοηθός παρείχε λανθασμένες πραγματικές πληροφορίες. |
| `unhelpful_answer` | Η απάντηση ήταν τεχνικά σωστή, αλλά όχι χρήσιμη. |
| `inappropriate_content` | Η απάντηση περιείχε περιεχόμενο που δεν πρέπει να εμφανίζεται στους χρήστες. |
| `other` | Το σχόλιο δεν αντιστοίχισε σε μια γνωστή κατηγορία. |

### Πιστοποίηση (Authentication)

Εάν το endpoint σας απαιτεί πιστοποίηση, ρυθμίστε το πεδίο **Feedback API Key** με το bearer token σας. Το agent αποστέλνει:

```
Authorization: Bearer <your-api-key>
```

Εάν το πεδίο **Feedback API Key** είναι κενό, δεν αποστέλεται κανένα `Authorization` header.

### Απενεργοποίηση Συλλογής Σχολίων

Αφήστε τα πεδία **Feedback Endpoint URL** και **Feedback API Key** κενά. Το κουμπί "thumbs-down" και η διεπαφή σχολίων παραμένουν ορατοί στους χρήστες, αλλά οι υποβολές δεν περνούν σε κανένα εξωτερικό service.

## Brave Search API Key

Επίσης στην καρτέλα **Advanced**, το πεδίο **Brave Search API Key** ενεργοποιεί την ικανότητα [Internet Search](../configuration/internet-search).

| Πεδίο | Περιγραφή |
|---|---|
| **Brave Search API Key** | Το API key σας από το developer dashboard της Brave Search. Απαιτείται για την ενεργοποίηση της αναζήτησης στο διαδίκτυο στον AI βοηθό. |

Η ετικέτα του πεδίου περιλαμβάνει έναν κάνσιμο κλικ σύνδεσμο στη σελίδα εγγραφής Brave Search API. Αφήστε το κενό για να απενεργοποιήσετε την αναζήτηση στο διαδίκτυο.

Για τεκμηρίωση για τους τελικούς χρήστες σχετικά με αυτό το χαρακτηριστικό, δείτε [Internet Search](../configuration/internet-search).

## Feature Flags

Επίσης εισαχθέν στην έκδοση v1.9.0, η καρτέλα **Settings → Feature Flags** παρέχει διακόπτες για προαιρετικές λειτουργίες. Κάθε flag είναι είτε ενεργοποιημένο είτε απενεργοποιημένο σε όλο το δίκτυο· δεν υπάρχει επί του παρόντος παρακάμπτση ανά site.

### Πρόσβαση στα Feature Flags

1. Στο admin του WordPress, μεταβείτε στο **Gratis AI Agent → Ρυθμίσεις**.
2. Κάντε κλικ στην καρτέλα **Feature Flags**.

### Flags Περιορισμού Πρόσβασης (Access Control Flags)

| Flag | Default | Περιγραφή |
|---|---|---|
| **Restrict to Administrators** | Off | Όταν είναι ενεργοποιημένο, μόνο οι χρήστες με το ρόλο `administrator` μπορούν να ανοίξουν το chat panel του AI Agent. Όλοι οι άλλοι ρόλοι βλέπουν αντίθετα μήνυμα: "Επικοινωνήστε με τον διαχειριστή σας". |
| **Restrict to Network Admins** | Off | Όταν είναι ενεργοποιημένο σε ένα multisite network, μόνο οι Super Admins μπορούν να χρησιμοποιήσουν το agent. Οι διαχειριστές των μεμονωμένων sites μπλοκάρονται. Έχει προτεραιότητα έναντι του "Restrict to Administrators" εάν και τα δύο είναι ενεργοποιημένα. |
| **Allow Subscriber Access** | Off | Όταν είναι ενεργοποιημένο, οι χρήστες με το ρόλο `subscriber` μπορούν να χρησιμοποιήσουν τη διεπαφή chat, αλλά περιορίζονται σε δυνατότητες μόνο για ανάγνωση (δεν μπορούν να δημιουργήσουν posts ή να αλλάξουν ρυθμίσεις). |
| **Disable for Non-Members** | Off | Ενσωματώνεται με την κατάσταση μέλους του Ultimate Multisite. Όταν είναι ενεργοποιημένο, το chat κρύβεται για sites που δεν έχουν ενεργό μέλος. |

### Flags Branding (Branding Flags)

| Flag | Default | Περιγραφή |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | Αφαιρεί τη γραμμή αναγνώρισης της επωνυμίας που εμφανίζεται στο κάτω μέρος του chat widget. Συνιστάται για white-label deployments. |
| **Custom Agent Name** | *(blank)* | Αντικαθιστά την προεπιλεγμένη ετικέτα "Gratis AI Agent" στην κεφαλίδα του chat και στο μενού διαχειριστή με το δικό σας όνομα προϊόντος. Αφήστε το κενό για να χρησιμοποιήσετε το προεπιλεγμένο. |
| **Hide Agent Picker** | Off | Όταν είναι ενεργοποιημένο, οι χρήστες δεν μπορούν να εναλλάσσονται μεταξύ των πέντε ενσωματωμένων agents. Το τρέχον agent είναι σταθερό στο όποιο έχει ρυθμιστεί ως προεπιλεγμένο στις Ρυθμίσεις → Γενικά. |
| **Use Site Icon as Chat Avatar** | Off | Αντικαθιστά το προεπιλεγμένο εικονίδιο AI στην κεφαλίδα του chat widget με το εικονίδιο του site του WordPress (ρυθμίζεται κάτω από Εμφάνιση → Προσαρμογή → Ταυτότητα Site). |

### Εφαρμογή Αλλαγών

Κάντε κλικ στο **Save Settings** μετά την ενεργοποίηση ή απενεργοποίηση οποιουδήποτε flag. Οι αλλαγές τίθενται σε λειτουργία αμέσως — δεν απαιτείται καθαρισμός cache ή επανενεργοποίηση plugin.
