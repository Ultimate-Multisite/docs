# Λειτουργίες Tenant Εξουσιώδους (Sovereign Tenant Operations)

Ultimate Multisite: Multi-Tenancy 1.2.0 προσθέτει εργαλεία λειτουργίας για εξουσιώδη tenants: υποσύλλογları που τρέχουν με τη δική τους βάση δεδομένων, ρίζα συστήματος αρχείων και πλαίσιο διαρροής (routing context), ενώ παραμένουν ορατοί από τον διαχειριστή δικτύου.

Χρησιμοποιήστε αυτήν σελίδα όταν διαχειρίζεστε απομονωμένα κλαδιά πελατών, μεταφορές σε εξωτερικά sites ή μετασχηματισμούς που μετακινούν ένα τυπικό υποσύλλογο σε εξουσιώδη υποδομή.

## Τι αλλάζει για τους διαχειριστές

- **Stateless tenant autologin** — Οι διαχειριστές δικτύου μπορούν να επισκεφτούν έναν εξουσιώδη tenant χωρίς να βασίζονται σε διαρκή κατάσταση συνεδρίας κοινής χρήσης. Το token SSO είναι περιορισμένο στο σκοπό, δεσμευμένο στην πηγή (origin-pinned), προστατευμένο από επαναλήψεις και έχει όριο λήξης σε μικρό χρονικό διάστημα.
- **Sovereign-aware routing** — Τα παλιά απομονωμένα δίκτυα και οι εξουσιώδεις tenants επιλύονται μέσω του ίδιου διαδρομής router του site, μειώνοντας τις διαφορές εκκίνησης μεταξύ των παλαιών και των νέων απομονωμένων εγκαταστάσεων.
- **Verified migration state** — Η επαλήθευση της μεταφοράς ελέγχει την προμήθεια χρήστη, τις επιτρεπές ρυθμίσεις του γράφτη βάσης δεδομένων (database writer permissions), το κατάσταση تخلیσης ουρών (queue drain status) και την απουσία παλαιών σωληνώσεων πριν ο tenant θεωρηθεί ολοκληρωμένος.
- **Safer teardown** — Η διαγραφή εξουσιών τώρα αφαιρεί τα στοιχεία του tenant καθαρά ώστε οι διαγραμμένες tenants να μην αφήσουν πίσω τους παλιά πρόσβαση στη βάση δεδομένων.

## Επισκεφθείτε έναν εξουσιώδη tenant

1. Ανοίξτε **Network Admin > Ultimate Multisite > Sites**.
2. Επιλέξτε τον εξουσιώδη tenant.
3. Χρησιμοποιήστε την επιλογή **Visit (SSO)** όταν είναι διαθέσιμη αντί να αντιγράφετε κωδικούς πρόσβασης ή να δημιουργείτε προσωρινές λογαριασμούς διαχειριστή.

Η ροή επισκεψιμότητας δημιουργεί ένα σύντομο token εισόδου για αυτόν τον tenant και καταγράφει το γεγονός SSO στο αρχείο 감사 (audit trail) του tenant. Εάν η επιλογή αποτύχει, ελέγξτε ότι ο домен του tenant αντιστοιχεί στην αναμενόμενη εγκατάσταση και ότι ο tenant μπορεί να φτάσει στον endpoint SSO της πλευράς δικτύου.

## Λίστα ελέγχου λειτουργιών σε εξωτερικό site (Remote-site operations checklist)

Πριν αλλάξετε έναν εξουσιώδη ή εξωτερικό tenant, επιβεβαιώστε:

- Το домен του ενοικιαζόμενου (tenant domain) δείχνει στον φιλοξενούμενο που διατηρεί το αρχείο του συστήματος αρχείων του ενοικιαζόμενου (tenant filesystem).
- Ο φιλοξενούμενος της βάσης δεδομένων του ενοικιαζόμενου ταιριάζει με τη διαμορφωμένη σύνδεση φιλοξενίας για αυτή την εγκατάσταση.
- Οι εντολές επαλήθευσης της μετεгриผ่าน (migration verification commands) περνούν για τον ενοικιαζόμενο.
- Οι ουρές μετεгриผ่าน ασύγχρονης μεταφοράς (Async migration queues) αποσυμπιέζονται πριν γίνει οποιαδήποτε αλλαγή DNS ή ιδιοκτησίας.
- Ο χρήστης διαχειριστής του ενοικιαζόμενου δημιουργήθηκε κατά τη διάρκεια της μετεгриผ่าน και μπορεί να συνδεθεί μέσω SSO.

## Διαγραφή κυριότερων ενοικιαζομένων (Deleting sovereign tenants)

Η διαγραφή ενός κυριότερου ενοικιαζόμενου είναι καταστροφική. Επιβεβαιώστε πρώτα το κατάσταση του backup και της εξαγωγής, και μετά διαγράψτε από την οθόνη διαχείρισης του site. Ο ροή αποσύνδεσης (teardown flow) 1.2.0 αφαιρεί τα στοιχεία πρόσβασης στη βάση δεδομένων του ενοικιαζόμενου ως μέρος της καθαρισμού, αλλά οι διαχειριστές πρέπει να επαληθεύσουν ότι τα χρήστες βάσης δεδομένων και τα φακέλια σε επίπεδο φιλοξενούμενου εξαφανίζονται όταν χρησιμοποιούν external hosting panels.

:::warning
Μην διαγράφετε ένα κυριότερο ενοικιαζόμενο ενώ η επαλήθευση της μετεгриผ่าน είναι ακόμα ενεργή ή ενώ υπάρχουν ουρές push jobs ασύγχρονων μεταφορών (async push jobs). Περιμένετε να ολοκληρωθεί η επαλήθευση ώστε το teardown να μην αφαιρεί στοιχεία που απαιτούνται από τα ενδεχόμενα job.
:::
