# Μεταβατική Λειτουργία Τρίτων Μερών

Το Superdav AI Agent v1.12.0 αλλάζει τον τρόπο διαχείρισης των ικανοτήτων (abilities) τρίτων μερών. **Η λειτουργία τρίτων μερών (Third-party mode) είναι πλέον αυτόματα (auto) εξ ορισμού**, επιτρέποντας την ενσωμάτωση με το native WordPress Abilities API σε εκδόσεις WordPress 7.0 και νεότερες χωρίς χειροκίνητη ρύθμιση.

## Τι άλλαξε;

### Πριν το v1.12.0

Οι ικανότητες τρίτων μερών απαιτούσαν χειροκίνητη ρύθμιση:

- Έπρεπε να ενεργοποιήσετε ρητά τη «λειτουργία τρίτων μερών» (third-party mode).
- Οι ικανότητες φορτώνονταν από ένα προσαρμοσμένο registry.
- Η ενσωμάτωση με το WordPress Abilities API ήταν προαιρετική.
- Η λειτουργία κληρονομικότητας (Legacy mode) ήταν η προεπιλεγμένη.

### Μετά το v1.12.0

Οι ικανότητες τρίτων μερών λειτουργούν αυτόματα:

- Η λειτουργία τρίτων μερών είναι αυτόματα «auto» εξ ορισμού.
- Οι ικανότητες ενσωματώνονται native με το WordPress Abilities API.
- Δεν απαιτείται χειροκίνητη ρύθμιση σε WordPress 7.0 και νεότερες.
- Η λειτουργία κληρονομικότητας (Legacy mode) παραμένει διαθέσιμη για παλαιότερες εκδόσεις WordPress.

## Ποιος επηρεάζεται;

### Νέες Εγκαταστάσεις (WordPress 7.0+)

**Δεν απαιτείται καμία ενέργεια.** Η λειτουργία τρίτων μερών ρυθμίζεται αυτόματα σε «auto», και οι ικανότητες λειτουργούν «out of the box».

### Υπάρχουσες Εγκαταστάσεις

**Οι ρυθμίσεις σας διατηρούνται.** Αν χρησιμοποιούσατε:

- **Legacy mode**: Παραμένετε σε λειτουργία κληρονομικότητας (κανένα]),
- **Χειροκίνητη λειτουργία τρίτων μερών (Manual third-party mode)**: Παραμένετε σε χειροκίνητη λειτουργία (κανένα]),
- **Auto mode**: Συνεχίζετε με την αυτόματη λειτουργία (κανένα).

### Εκδόσεις WordPress Πριν το 7.0

**Η λειτουργία κληρονομικότητας (Legacy mode) παραμένει διαθέσιμη.** Αν χρησιμοποιείτε WordPress 6.x ή παλαιότερες:

- Η λειτουργία τρίτων μερών είναι αυτόματα «legacy» εξ ορισμού.
- Μπορείτε να ενεργοποιήσετε χειροκίνητα τη λειτουργία τρίτων μερών αν το επιθυμείτε.
- Αναβαθμίστε σε WordPress 7.0+ για να χρησιμοποιήσετε το native Abilities API.

## Κατανόηση των Λειτουργιών

### Auto Mode (Νέα Προεπιλογή)

Το **Auto mode** χρησιμοποιεί την native ενσωμάτωση με το WordPress Abilities API:

- Οι ικανότητες καταγράφονται μέσω WordPress hooks.
- Πλήρης συμβατότητα με το WordPress 7.0+ Abilities API.
- Αυτόματη ανακάλυψη ικανοτήτων τρίτων μερών.
- Δεν απαιτείται χειροκίνητη ρύθμιση.

**Πότε να χρησιμοποιήσετε**: WordPress 7.0+ με ικανότητες τρίτων μερών.

### Manual Mode

Το **Manual mode** απαιτεί ρητή ρύθμιση:

- Εspecifies ποιες ικανότητες τρίτων μερών πρέπει να φορτωθούν.
- Χρήσιμο για δοκιμές ή επιλεγμένη φόρτωση ικανοτήτων.
- Απαιτεί επεξεργασία αρχείων διαμόρφωσης.
- Περισσότερος έλεγχος, αλλά και περισσότερη ρύθμιση.

**Πότε να χρησιμοποιήσετε**: Δοκιμές, επιλεγμένη φόρτωση ικανοτήτων ή προσαρμοσμένες διαμορφώσεις.

### Legacy Mode

Το **Legacy mode** χρησιμοποιεί το παрое σύστημα ικανοτήτων τρίτων μερών:

- Προσαρμοσμένο registry ικανοτήτων (όχι το WordPress Abilities API).
- Συμβατότητα με παλαιότερες εκδόσεις WordPress.
- Καμία native ενσωμάτωση με το WordPress.
- Αποθησαρεμένο (Deprecated) αλλά εξακολουθεί να υποστηρίζεται.

**Πότε να χρησιμοποιήσετε**: WordPress 6.x ή παλαιότερες, ή όταν χρειάζεστε συμβατότητα με παλαιότερες εκδόσεις.

## Έλεγχος της Τρέχουσας Λειτουργίας σας

### Μέσω του Admin Panel

1. Μεταβείτε στο **WordPress Admin** → **Superdav AI Agent** → **Settings**.
2. Ψάξτε για τη ρύθμιση **Third-Party Mode**.
3. Θα δείτε την τρέχουσα λειτουργία σας και τις επιλογές για να την αλλάξετε.

### Μέσω Κώδικα

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', ή 'legacy'
```

## Αλλαγή της Λειτουργίας σας

### Μετάβαση στο Auto Mode

Εάν βρίσκεστε σε WordPress 7.0+ και θέλετε να χρησιμοποιήσετε το auto mode:

1. Μεταβείτε στο **Superdav AI Agent** → **Settings**.
2. Βρείτε το **Third-Party Mode**.
3. Επιλέξτε **Auto (WordPress Abilities API)**.
4. Κάντε κλικ στο **Save**.

Το Superdav AI Agent θα ανακαλύψει και θα καταγράψει αυτόματα τις ικανότητες τρίτων μερών.

### Μετάβαση στο Manual Mode

Εάν θέλετε να ελέγχετε χειροκίνητα ποιες ικανότητες φορτώνονται:

1. Μεταβείτε στο **Superdav AI Agent** → **Settings**.
2. Βρείτε το **Third-Party Mode**.
3. Επιλέξτε **Manual**.
4. Κάντε κλικ στο **Save**.
5. Επεξεργαστείτε το αρχείο διαμόρφωσης σας για να καθορίσετε ποιες ικανότητες πρέπει να φορτωθούν.

### Μετάβαση στο Legacy Mode

Εάν χρειάζεστε συμβατότητα με παλαιότερες εκδόσεις:

1. Μεταβείτε στο **Superdav AI Agent** → **Settings**.
2. Βρείτε το **Third-Party Mode**.
3. Επιλέξτε **Legacy**.
4. Κάντε κλικ στο **Save**.

## Οφέλη του Auto Mode

### Αυτόματη Ανακάλυψη

Οι ικανότητες ανακαλύπτονται αυτόματα από:

- Εγκατεστημένα plugins.
- Ενεργό theme.
- Must-use plugins.
- Drop-in plugins.

Δεν απαιτείται χειροκίνητη καταγραφή.

### Native Ενσωμάτωση

Οι ικανότητες ενσωματώνονται με το WordPress Abilities API:

- Συνεπείς με το WordPress core.
- Λειτουργούν με το WordPress admin.
- Συμβατές με άλλα plugins που χρησιμοποιούν το Abilities API.
- Προστατευτικές για το μέλλον καθώς εξελίσσεται το WordPress.

### Απλοποιημένη Διαχείριση

- Δεν υπάρχουν αρχεία διαμόρφωσης που πρέπει να επεξεργαστείτε.
- Καμία χειροκίνητη καταγραφή ικανοτήτων.
- Οι ρυθμίσεις ορατότητας ικανοτήτων (Ability Visibility) λειτουργούν αυτόματα.
- Οι ειδοποιήσεις του admin σας ενημερώνουν για μη ταξινομημένες ικανότητες.

### Καλύτερη Απόδοση

- Οι ικανότητες αποθηκεύονται σε cache.
- Φορτίζονται «lazy» (κατόπιν ανάγκης).
- Βελτιστοποιημένες για το WordPress 7.0+.

## Διαδρομή Μεταβίβασης

### Εάν βρίσκεστε σε WordPress 6.x

1. **Αναβαθμίστε σε WordPress 7.0+** (όταν είστε έτοιμοι).
2. **Ενημερώστε το Superdav AI Agent** σε v1.12.0+.
3. **Αλλάξτε τη λειτουργία τρίτων μερών σε Auto** (προαιρετικό· η λειτουργία legacyยัง λειτουργεί).
4. **Ελέγξτε την ορατότητα των ικανοτήτων** για να διασφαλίσετε τις σωστές ρυθμίσεις πρόσβασης.

### Εάν βρίσκεστε σε WordPress 7.0+

1. **Ενημερώστε το Superdav AI Agent** σε v1.12.0+.
2. **Επιβεβαιώστε ότι η λειτουργία τρίτων μερών είναι ρυθμισμένη σε Auto** (πρέπει να είναι αυτόματα).
3. **Ελέγξτε την ορατότητα των ικανοτήτων** για να διασφαλίσετε τις σωστές ρυθμίσεις πρόσβασης.
4. **Δοκιμάστε τις ικανότητες τρίτων μερών** για να επιβεβαιώσετε ότι λειτουργούν.

## Αντιμετώπιση Προβλημάτων

### Οι ικανότητες δεν φορτώνουν σε auto mode

- Επιβεβαιώστε ότι βρίσκεστε σε WordPress 7.0+.
- Ελέγξτε ότι η λειτουργία τρίτων μερών είναι ρυθμισμένη σε «Auto».
- Επιβεβαιώστε ότι το plugin που παρέχει την ικανότητα είναι ενεργό.
- Ελέγξτε τα αρχεία καταγραφής σφαλμάτων (error logs) του WordPress για λάθη καταγραφής.

### Θέλω να διατηρήσω το legacy mode

- Μεταβείτε στο **Settings** → **Third-Party Mode**.
- Επιλέξτε **Legacy**.
- Κάντε κλικ στο **Save**.
- Η λειτουργία legacy θα συνεχίσει να λειτουργεί.

### Οι προσαρμοσμένες ικανότητές μου δεν εμφανίζονται

- Επιβεβαιώστε ότι καταγράφονται μέσω WordPress hooks.
- Ελέγξτε ότι υλοποιούν σωστά το Abilities API.
- Ελέγξτε τα αρχεία καταγραφής σφαλμάτων του WordPress.
- Χρησιμοποιήστε τη σελίδα admin **Ability Visibility** για να δείτε όλες τις καταγεγραμμένες ικανότητες.

### Παίρνω ειδοποιήσεις «unclassified ability»

- Αυτό είναι φυσιολογικό για νέες ικανότητες τρίτων μερών.
- Ελέγξτε και ταξινομήστε τις στην ειδοποίηση του admin.
- Δείτε το **Ability Visibility** για λεπτομέρειες σχετικά με την ταξινόμηση.

## Συμβατότητα με Παλαιότερες Εκδόσεις

### Υπάρχουσες Διαμορφώσεις

Εάν έχετε υπάρχουσες διαμορφώσεις ικανοτήτων τρίτων μερών:

- **Legacy mode**: Η διαμόρφωσή σας συνεχίζει να λειτουργεί.
- **Manual mode**: Η διαμόρφωσή σας συνεχίζει να λειτουργεί.
- **Auto mode**: Η διαμόρφωσή σας αγνοείται (το auto mode αναλαμβάνει).

Για να διατηρήσετε τη δική σας προσαρμοσμένη διαμόρφωση, παραμείνετε σε Manual ή Legacy mode.

### Χρονοδιάγραμμα Αποθησαρείας (Deprecation Timeline)

- **v1.12.0**: Οι λειτουργίες Legacy και Manual υποστηρίζονται πλήρως.
- **v1.13.0+**: Η λειτουργία Legacy μπορεί να εμφανίσει ειδοποιήσεις αποθησαρείας.
- **v2.0.0**: Η λειτουργία Legacy ενδέχεται να αφαιρεθεί (TBD).

## Βέλτιστες Πρακτικές

### Για Νέες Εγκαταστάσεις

- Χρησιμοποιήστε το Auto mode (είναι η προεπιλογή).
- Αφήστε το Superdav AI Agent να ανακαλύψει αυτόματα τις ικανότητες.
- Χρησιμοποιήστε το Ability Visibility για τον έλεγχο πρόσβασης.

### Για Υπάρχουσες Εγκαταστάσεις

- Αναβαθμίστε σε WordPress 7.0+ όταν είναι δυνατόν.
- Μεταβείτε στο Auto mode για απλοποιημένη διαχείριση.
- Ελέγξτε και ταξινομήστε τις ικανότητες χρησιμοποιώντας το Ability Visibility.

### Για Προσαρμοσμένες Ικανότητες

- Καταγράψτε τις ικανότητες μέσω WordPress hooks (Abilities API).
- Αποφύγετε τα προσαρμοσμένα registries ικανοτήτων.
- Δοκιμάστε σε WordPress 7.0+ με Auto mode.

## Επόμενα Βήματα

1. **Ελέγξτε την έκδοση WordPress σας**: Επιβεβαιώστε ότι βρίσκεστε σε 7.0+ για το Auto mode.
2. **Ελέγξτε τη λειτουργία τρίτων μερών σας**: Μεταβείτε στις Settings και ελέγξτε την τρέχουσα λειτουργία σας.
3. **Ενημερώστε αν χρειάζεται**: Μεταβείτε στο Auto mode εάν βρίσκεστε σε WordPress 7.0+.
4. **Ταξινομήστε τις ικανότητες**: Ελέγξτε και ταξινομήστε οποιεσδήποτε μη ταξινομημένες ικανότητες.
5. **Δοκιμάστε**: Επιβεβαιώστε ότι οι ικανότητες τρίτων μερών λειτουργούν σωστά.

## Σχετικά Θέματα

- **Ability Visibility**: Ελέγξτε ποιες ικανότητες εκτίθενται και πού.
- **WordPress Abilities API**: Μάθετε για την native καταγραφή ικανοτήτων του WordPress.
- **Third-Party Ability Development**: Δημιουργήστε ικανότητες που λειτουργούν με το Auto mode.
