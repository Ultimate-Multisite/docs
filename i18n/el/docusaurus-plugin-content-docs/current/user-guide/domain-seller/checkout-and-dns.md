# Πεδίο ολοκλήρωσης αγοράς και διαχείριση DNS πελατών

## Το πεδίο ολοκλήρωσης αγοράς Επιλογή τομέα

Το πεδίο **Επιλογή τομέα** είναι ένα στοιχείο ολοκλήρωσης αγοράς που δίνει στους πελάτες επιλογή για το πώς θα αποκτήσουν τον τομέα του ιστοτόπου τους. Προσθέστε το σε οποιαδήποτε φόρμα ολοκλήρωσης αγοράς για να ενεργοποιήσετε την πώληση τομέων.

### Προσθήκη του πεδίου σε φόρμα ολοκλήρωσης αγοράς

1. Μεταβείτε στο **Διαχείριση Δικτύου › Ultimate Multisite › Φόρμες Ολοκλήρωσης Αγοράς**
2. Ανοίξτε ή δημιουργήστε μια φόρμα ολοκλήρωσης αγοράς
3. Στον επεξεργαστή ολοκλήρωσης αγοράς, κάντε κλικ στο **Προσθήκη Πεδίου**
4. Επιλέξτε **Επιλογή τομέα** από τη λίστα πεδίων
5. Διαμορφώστε τις επιλογές του πεδίου (δείτε παρακάτω)
6. Αποθηκεύστε τη φόρμα

### Επιλογές πεδίου

**Λειτουργίες τομέα** — Επιλέξτε ποιες καρτέλες βλέπει ο πελάτης. Κάθε λειτουργία μπορεί να ενεργοποιηθεί ή να απενεργοποιηθεί ανεξάρτητα:

| Λειτουργία | Τι κάνει |
|---|---|
| **Υποτομέας** | Ο πελάτης χρησιμοποιεί έναν δωρεάν υποτομέα στο δίκτυό σας (π.χ. `mysite.yournetwork.com`). Δεν απαιτείται πληρωμή. |
| **Καταχώριση Νέου Τομέα** | Ο πελάτης αναζητά έναν νέο τομέα και τον καταχωρίζει μέσω του διαμορφωμένου παρόχου σας. Χρησιμοποιεί το αντίστοιχο προϊόν τομέα για την τιμολόγηση. |
| **Υφιστάμενος Τομέας** | Ο πελάτης αντιστοιχίζει έναν τομέα που ήδη κατέχει. Δεν υπάρχει χρέωση καταχώρισης. Ο τομέας αντιστοιχίζεται αυτόματα στον ιστότοπό του. |

**Προεπιλεγμένη λειτουργία** — Όταν είναι ενεργοποιημένες και οι τρεις λειτουργίες, ποια καρτέλα ανοίγει πρώτη. Ορίστε το σε **Υποτομέας** για να διατηρήσετε την καταχώριση τομέα προαιρετική ή σε **Καταχώριση Νέου Τομέα** για να ενθαρρύνετε αγορές.

**Προϊόν τομέα** — Προαιρετικά καρφιτσώστε αυτό το πεδίο σε ένα συγκεκριμένο προϊόν τομέα. Αν δεν οριστεί, το addon επιλέγει αυτόματα το αντίστοιχο προϊόν με βάση το TLD που αναζητά ο πελάτης.

### Πεδία επικοινωνίας καταχωρίζοντος

Όταν ένας πελάτης επιλέγει την καρτέλα **Καταχώριση Νέου Τομέα**, η φόρμα ολοκλήρωσης αγοράς προσθέτει ενσωματωμένα πεδία επικοινωνίας καταχωρίζοντος:

- Όνομα / Επώνυμο
- Διεύθυνση email
- Διεύθυνση (γραμμή 1, πόλη, πολιτεία/επαρχία, ταχυδρομικός κώδικας, χώρα)
- Αριθμός τηλεφώνου

Αυτά απαιτούνται από όλους τους καταχωρητές και επικυρώνονται πριν γίνει η κλήση API καταχώρισης. Οι αριθμοί τηλεφώνου μορφοποιούνται αυτόματα στη διεθνή μορφή `+CC.NNN` που αναμένουν οι καταχωρητές.

### Αυτόματα δημιουργημένο URL ιστοτόπου

Όταν ένας πελάτης καταχωρίζει ή αντιστοιχίζει έναν τομέα, το πεδίο URL ιστοτόπου συμπληρώνεται αυτόματα από τον επιλεγμένο τομέα. Οι πελάτες δεν χρειάζεται να συμπληρώσουν ξεχωριστό πεδίο URL.

### Συμπεριφορά αναζήτησης

- Η διαθεσιμότητα τομέα ελέγχεται σε πραγματικό χρόνο με AJAX καθώς πληκτρολογεί ο πελάτης
- Εμφανίζονται προτάσεις εναλλακτικών TLD όταν ο προτιμώμενος τομέας δεν είναι διαθέσιμος
- Η τιμολόγηση ανακτάται ζωντανά και εμφανίζεται καθαρά (τιμή καταχώρισης, τιμή ανανέωσης, προαιρετική χρέωση απορρήτου WHOIS)
- Οι κωδικοί κουπονιών εφαρμόζονται στα προϊόντα τομέα όπως και σε οποιοδήποτε άλλο προϊόν

**Ρύθμιση της απόκρισης αναζήτησης:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**Προσθήκη προσαρμοσμένων πεδίων στη φόρμα αναζήτησης τομέα:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## Διαχείριση DNS πελατών

Οι πελάτες μπορούν να διαχειρίζονται εγγραφές DNS για τους καταχωρισμένους τομείς τους από τη σελίδα **My Account**, κάτω από την καταχώριση του τομέα τους.

### Υποστηριζόμενοι τύποι εγγραφών

| Τύπος | Χρήση |
|---|---|
| **A** | Αντιστοίχιση ονόματος host σε διεύθυνση IPv4 |
| **AAAA** | Αντιστοίχιση ονόματος host σε διεύθυνση IPv6 |
| **CNAME** | Δημιουργία ψευδωνύμου που δείχνει σε άλλο όνομα host |
| **MX** | Ορισμός διακομιστή ανταλλαγής αλληλογραφίας |
| **TXT** | Προσθήκη SPF, DMARC, επαλήθευσης ή άλλων εγγραφών κειμένου |

### Ποιοι πάροχοι υποστηρίζουν διαχείριση DNS;

Η διαχείριση DNS (προσθήκη, επεξεργασία, διαγραφή εγγραφών) είναι διαθέσιμη με **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** και **Openprovider**. Οι τομείς **Hostinger** μπορούν να ενημερώνουν nameservers μέσω του Domain Seller· οι εγγραφές DNS για φιλοξενούμενους τομείς διαχειρίζονται από την κύρια ενσωμάτωση αντιστοίχισης τομέων του Hostinger. Οι τομείς Namecheap, GoDaddy και NameSilo εμφανίζουν πληροφορίες κατάστασης και λήξης, αλλά το DNS πρέπει να διαχειρίζεται απευθείας στον πίνακα ελέγχου του καταχωρητή.

### Προεπιλεγμένες εγγραφές DNS

Μπορείτε να διαμορφώσετε προεπιλεγμένες εγγραφές DNS που εφαρμόζονται αυτόματα όταν καταχωρίζεται ένας τομέας. Μεταβείτε στο **Ρυθμίσεις › Domain Seller › Προεπιλεγμένες Εγγραφές DNS**.

Οι προεπιλεγμένες τιμές εγγραφών υποστηρίζουν δύο tokens:

| Token | Αντικαθίσταται με |
|---|---|
| `{DOMAIN}` | Το καταχωρισμένο όνομα τομέα (π.χ. `example.com`) |
| `{SITE_URL}` | Το URL WordPress ιστοτόπου για τον ιστότοπο του πελάτη |

**Παράδειγμα — κατευθύνετε τον apex τομέα και το www στη διεύθυνση IP του διακομιστή σας:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Διαχειριστής: προβολή και επεξεργασία DNS

Οι διαχειριστές δικτύου μπορούν να βλέπουν και να επεξεργάζονται εγγραφές DNS για οποιονδήποτε τομέα πελάτη από τη σελίδα επεξεργασίας του τομέα στο **Διαχείριση Δικτύου › Ultimate Multisite › Τομείς**.
