# Workflow Μεταφοράς Εξουσιώδους Περιβάλλοντος

Ultimate Multisite: Multi-Tenancy 1.2.0 προσθέτει προστατευτικά σημεία επαλήθευσης μεταφοράς για τη μετακίνηση ενός τυπικού υποσύßματος σε υποδομή εξουσιοδοτημένου περιβάλλοντος (sovereign tenant).

## Πριν ξεκινήσετε

Επιβεβαιώστε ότι έχετε:

- Ένα τρέχον backup του αρχικού site.
- Το addon Multi-Tenancy ενεργό.
- Η βάση δεδομένων, το root του filesystem και το domain του προορισμού tenant έτοιμα.
- Μια σύνδεση host της βάσης δεδομένων που ταιριάζει με το περιβάλλον προορισμού.
- Πρόσβαση για να εκτελέσετε εντολές WP-CLI για το network.

## Προτεινόμενη ροή εργασίας

1. Προετοιμάστε τη βάση δεδομένων και το filesystem του tenant προορισμού.
2. Εγγραφή ή ενημέρωση των ρυθμίσεων απομόνωσης του tenant.
3. Εκτέλεση της μεταφοράς του tenant.
4. Provisioning (προμήθεια) ή επαλήθευση των χρηστών του tenant.
5. Αποφόρτωση (drain) των jobs με ασύγχρονη μεταφορά.
6. Εκτέλεση της επαλήθευσης της εξουσιώδους μεταφοράς.
7. Επισκεφθείτε τον tenant με SSO.
8. Αλλαγή DNS ή διαδρομής μόνο αφού περάσει η επαλήθευση.

## Προστατευτικά Σημεία Επαλήθευσης (Verification gates)

Η ροή εργασίας επαλήθευσης ελέγχει τη μεταφορά από διάφορες πτυχές:

- Το schema του tenant υπάρχει και μπορεί να γράφει από τον δημιουργό της βάσης δεδομένων του tenant.
- Ο διαμορφωμένος host της βάσης δεδομένων έχει έγκυρα δικαιώματα πρόσβασης (grants).
- Οι χρήστες του tenant είναι παρόντες και ταιριάζουν με τον αναμενόμενο αριθμό χρηστών εγκατάστασης εξουσιοδοτημένου περιβάλλοντος.
- Η ουρά push ασύγχρονης μεταφοράς μπορεί να αποφόρτωσει επιτυχώς.
- Οι παλαιότερες διαδρομές δεδομένων στο network δεν απαιτούν πλέον για τον εξουσιώδη tenant.

Διαχειριστεί τις αποτυχημένες επαληθεύσεις ως εμπόδια πριν την εκκίνηση. Διορθώστε το ζητούμενο πρόβλημα της βάσης δεδομένων, του χρήστη, της ουράς ή της διαδρομής και εκτελέστε ξανά την επαλήθευση πριν προфакίνα τον tenant στους πελάτες.

## Πρώτη επισκεφθείσα παραγωγή (First production visit)

Μετά την επιτυχή ολοκλήρωση της επαλήθευσης, χρησιμοποιήστε τη λειτουργία **Visit (SSO)** από την οθόνη διαχείρισης του site για την πρώτη πρόσβαση διαχειριστή. Αυτό επιβεβαιώνει τη διαδρομή του tenant, τη διαχείριση των tokens SSO, το origin pinning και την προμήθεια χρηστών στο πλευρό του tenant σε μία kontrollημένη ενέργεια.
