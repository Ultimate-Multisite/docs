# Πώς να ρυθμίσετε το Domain Mapping (v2)

_**ΣΗΜΑΝΤΙΚΗ ΣΗΜΕΙΩΣΗ: Αυτό το άρθρο αναφέρεται στην έκδοση 2.x του Ultimate Multisite.**_

Μία από τις πιο ισχυρές δυνατότητες ενός premium δικτύου είναι η δυνατότητα να προσφέρουμε στους πελάτες μας τη δυνατότητα να συνδέσουν ένα top-level domain στους ιστότοπούς τους. Άλλωστε, τι φαίνεται πιο επαγγελματικό: [_**joesbikeshop.yournetwork.com**_](http://joesbikeshop.yournetwork.com) ή [_**joesbikeshop.com**_](http://joesbikeshop.com); Γι' αυτό το Ultimate Multisite προσφέρει αυτή τη δυνατότητα ενσωματωμένη, χωρίς να χρειάζεται να χρησιμοποιήσετε plugins τρίτων.

## Τι είναι το domain mapping;

Όπως υποδηλώνει το όνομά του, το domain mapping είναι η δυνατότητα που προσφέρει το Ultimate Multisite να λαμβάνει ένα αίτημα για ένα προσαρμοσμένο domain και να αντιστοιχίζει αυτό το αίτημα στον αντίστοιχο ιστότοπο του δικτύου που έχει συνδεδεμένο αυτό το συγκεκριμένο domain.

### Πώς να ρυθμίσετε το domain mapping στο δίκτυο Ultimate Multisite σας

Το domain mapping απαιτεί κάποιες ρυθμίσεις από την πλευρά σας για να λειτουργήσει. Ευτυχώς, το Ultimate Multisite αυτοματοποιεί τη δύσκολη δουλειά για εσάς, ώστε να μπορείτε εύκολα να καλύψετε τις απαιτήσεις.

Κατά την εγκατάσταση του Ultimate Multisite, ο οδηγός εγκατάστασης θα αντιγράψει και θα εγκαταστήσει αυτόματα το **sunrise.php** στον καθορισμένο φάκελο. **Ο οδηγός δεν θα σας επιτρέψει να προχωρήσετε μέχρι να ολοκληρωθεί αυτό το βήμα**.

![Οδηγός εγκατάστασης Ultimate Multisite με το βήμα sunrise.php](/img/config/settings-domain-mapping.png)

Αυτό σημαίνει ότι μόλις ο οδηγός εγκατάστασης του Ultimate Multisite ολοκληρώσει τη ρύθμιση του δικτύου σας, μπορείτε να αρχίσετε να αντιστοιχίζετε το προσαρμοσμένο domain αμέσως.

Σημειώστε ότι το domain mapping στο Ultimate Multisite δεν είναι υποχρεωτικό. Έχετε την επιλογή να χρησιμοποιήσετε τη native λειτουργία domain mapping του WordPress Multisite ή οποιαδήποτε άλλη λύση domain mapping.

Εάν χρειαστεί να απενεργοποιήσετε το domain mapping του Ultimate Multisite για να δώσετε χώρο σε άλλες λύσεις domain mapping, μπορείτε να απενεργοποιήσετε αυτή τη δυνατότητα στο **Ultimate Multisite > Settings > Domain Mapping**.

![Ρυθμίσεις Domain Mapping με διακόπτη ενεργοποίησης](/img/config/settings-domain-mapping.png)

Ακριβώς κάτω από αυτή την επιλογή, μπορείτε επίσης να δείτε την επιλογή **Force Admin Redirect**. Αυτή η επιλογή σας επιτρέπει να ελέγχετε αν οι πελάτες σας θα μπορούν να έχουν πρόσβαση στο admin dashboard τους τόσο στο προσαρμοσμένο domain όσο και στο subdomain ή μόνο σε ένα από τα δύο.

Αν επιλέξετε **Force redirect to mapped domain**, οι πελάτες σας θα μπορούν να έχουν πρόσβαση στο admin dashboard τους μόνο στα προσαρμοσμένα domains τους.

Η επιλογή **Force redirect to network domain** θα κάνει ακριβώς το αντίθετο - οι πελάτες σας θα μπορούν να έχουν πρόσβαση στα dashboards τους μόνο στο subdomain τους, ακόμα κι αν προσπαθήσουν να συνδεθούν από τα προσαρμοσμένα domains τους.

Και η επιλογή **Allow access to the admin by both mapped domain domain and network domain** τους επιτρέπει να έχουν πρόσβαση στα admin dashboards τους τόσο στο subdomain όσο και στο προσαρμοσμένο domain.

![Επιλογές Force Admin Redirect για το domain mapping](/img/config/settings-domain-mapping.png)

Υπάρχουν δύο τρόποι για να αντιστοιχίσετε ένα προσαρμοσμένο domain. Ο πρώτος είναι αντιστοιχίζοντας το όνομα domain από το admin dashboard του δικτύου σας ως super admin και ο δεύτερος είναι μέσω του admin dashboard του subsite στη σελίδα λογαριασμού.

Αλλά πριν αρχίσετε να αντιστοιχίζετε το προσαρμοσμένο domain σε ένα από τα subsites του δικτύου σας, θα πρέπει να βεβαιωθείτε ότι οι **ρυθμίσεις DNS** του ονόματος domain είναι σωστά ρυθμισμένες.

### Βεβαιωθείτε ότι οι ρυθμίσεις DNS του domain είναι σωστά ρυθμισμένες

Για να λειτουργήσει η αντιστοίχιση, πρέπει να βεβαιωθείτε ότι το domain που σκοπεύετε να αντιστοιχίσετε δείχνει στη διεύθυνση IP του Δικτύου σας. Σημειώστε ότι χρειάζεστε τη διεύθυνση IP του Δικτύου - τη διεύθυνση IP του domain όπου είναι εγκατεστημένο το Ultimate Multisite - όχι τη διεύθυνση IP του προσαρμοσμένου domain που θέλετε να αντιστοιχίσετε. Για να αναζητήσετε τη διεύθυνση IP ενός συγκεκριμένου domain, προτείνουμε να πάτε στο [Site24x7](https://www.site24x7.com/find-ip-address-of-web-site.html), για παράδειγμα.

Για να αντιστοιχίσετε σωστά το domain, πρέπει να προσθέσετε ένα **A RECORD** στη ρύθμιση του **DNS** σας που να δείχνει σε αυτή τη **διεύθυνση IP**. Η διαχείριση DNS διαφέρει σημαντικά μεταξύ διαφορετικών domain registrars, αλλά υπάρχουν πολλά tutorials online που το καλύπτουν αν αναζητήσετε "_Creating A Record on XXXX_" όπου XXXX είναι ο domain registrar σας (π.χ.: "_Creating A Record on GoDaddy_").

Αν δυσκολεύεστε να το κάνετε αυτό να λειτουργήσει, **επικοινωνήστε με την υποστήριξη του domain registrar σας** και θα μπορέσουν να σας βοηθήσουν με αυτό το μέρος.

Αν σκοπεύετε να επιτρέψετε στους πελάτες σας να αντιστοιχίζουν τα δικά τους domains, θα πρέπει να κάνουν τη δουλειά σε αυτό το μέρος μόνοι τους. Κατευθύνετέ τους προς το σύστημα υποστήριξης του registrar τους αν δυσκολεύονται να δημιουργήσουν το A Record.

### Αντιστοίχιση προσαρμοσμένου ονόματος domain ως Super Admin

Όταν είστε συνδεδεμένοι ως super admin στο δίκτυό σας, μπορείτε εύκολα να προσθέσετε και να διαχειριστείτε προσαρμοσμένα ονόματα domain πηγαίνοντας στο **Ultimate Multisite > Domains**.

![Σελίδα λίστας Domains στο Ultimate Multisite](/img/admin/domains-list.png)

Σε αυτή τη σελίδα, μπορείτε να κάνετε κλικ στο κουμπί **Add Domain** στην κορυφή και αυτό θα εμφανίσει ένα παράθυρο modal όπου μπορείτε να ορίσετε και να συμπληρώσετε το **προσαρμοσμένο όνομα domain**, το **subsite** στο οποίο θέλετε να εφαρμόσετε το προσαρμοσμένο όνομα domain, και να αποφασίσετε αν θέλετε να το ορίσετε ως **κύριο domain** ή όχι (σημειώστε ότι μπορείτε να αντιστοιχίσετε **πολλαπλά ονόματα domain σε ένα subsite**).

![Modal Προσθήκης Domain με πεδία ονόματος domain και ιστότοπου](/img/admin/domains-list.png)

Αφού συμπληρώσετε όλες τις πληροφορίες, μπορείτε στη συνέχεια να κάνετε κλικ στο κουμπί **Add Existing Domain** στο κάτω μέρος.

Αυτό θα ξεκινήσει τη διαδικασία επαλήθευσης και ανάκτησης των πληροφοριών DNS του προσαρμοσμένου domain. Θα δείτε επίσης ένα log στο κάτω μέρος της σελίδας για να παρακολουθείτε τη διαδικασία που εκτελείται. Αυτή η διαδικασία μπορεί να διαρκέσει μερικά λεπτά για να ολοκληρωθεί.

Το **Stage** ή η κατάσταση θα πρέπει να αλλάξει από **Checking DNS** σε **Ready** αν όλα είναι σωστά ρυθμισμένα.

![Το Stage του domain αλλάζει από Checking DNS σε Ready](/img/admin/domains-list.png)

![Domain με κατάσταση Ready στη λίστα domains](/img/admin/domains-list.png)

Αν κάνετε κλικ στο όνομα domain, θα μπορείτε να δείτε κάποιες επιλογές μέσα σε αυτό. Ας ρίξουμε μια γρήγορη ματιά σε αυτές:

![Σελίδα λεπτομερειών domain με επιλογές stage, site, active και SSL](/img/admin/domains-list.png)

**Stage:** Αυτό είναι το στάδιο στο οποίο βρίσκεται το domain. Όταν προσθέτετε πρώτα το domain, πιθανότατα θα είναι στο στάδιο **Checking DNS**. Η διαδικασία θα ελέγξει τις εγγραφές DNS και θα επιβεβαιώσει ότι είναι σωστές. Στη συνέχεια, το domain θα τοποθετηθεί στο στάδιο **Checking SSL**. Το Ultimate Multisite θα ελέγξει αν το domain έχει SSL ή όχι και θα κατηγοριοποιήσει το domain σας ως **Ready** ή **Ready (without SSL)**.

**Site:** Το subdomain που σχετίζεται με αυτό το domain. Το αντιστοιχισμένο domain θα εμφανίζει το περιεχόμενο αυτού του συγκεκριμένου ιστότοπου.

**Active:** Μπορείτε να ενεργοποιήσετε ή να απενεργοποιήσετε αυτή την επιλογή για να ενεργοποιήσετε ή να απενεργοποιήσετε το domain.

**Is Primary Domain?:** Οι πελάτες σας μπορούν να έχουν περισσότερα από ένα αντιστοιχισμένα domains για κάθε ιστότοπο. Χρησιμοποιήστε αυτή την επιλογή για να επιλέξετε αν αυτό είναι το κύριο domain για τον συγκεκριμένο ιστότοπο.

**Is Secure?:** Παρόλο που το Ultimate Multisite ελέγχει αν το domain έχει πιστοποιητικό SSL ή όχι πριν το ενεργοποιήσει, μπορείτε να επιλέξετε χειροκίνητα να φορτώσετε το domain με ή χωρίς πιστοποιητικό SSL. Σημειώστε ότι αν ο ιστότοπος δεν έχει πιστοποιητικό SSL και προσπαθήσετε να το φορτώσετε με SSL, μπορεί να σας δώσει σφάλματα.

### Αντιστοίχιση προσαρμοσμένου ονόματος domain ως χρήστης Subsite

Οι διαχειριστές subsite μπορούν επίσης να αντιστοιχίζουν προσαρμοσμένα ονόματα domain από το admin dashboard του subsite τους.

Πρώτα, πρέπει να βεβαιωθείτε ότι ενεργοποιείτε αυτή την επιλογή στις ρυθμίσεις **Domain mapping**. Δείτε το παρακάτω screenshot.

![Ρυθμίσεις domain mapping που επιτρέπουν στους χρήστες subsite να αντιστοιχίζουν domains](/img/config/settings-domain-mapping.png)

Μπορείτε επίσης να ορίσετε ή να ρυθμίσετε αυτή την επιλογή σε επίπεδο **Plan** ή στις επιλογές προϊόντος στο **Ultimate Multisite > Products**.

![Επιλογή προσαρμοσμένων domains προϊόντος στις ρυθμίσεις προϊόντος](/img/config/settings-domain-mapping.png)

Όταν οποιαδήποτε από αυτές τις επιλογές είναι ενεργοποιημένη και ένας χρήστης subsite έχει άδεια να αντιστοιχίζει προσαρμοσμένα ονόματα domain, ο χρήστης subsite θα πρέπει να βλέπει ένα metabox στη σελίδα **Account** που ονομάζεται **Domains**.

![Metabox Domains στη σελίδα λογαριασμού subsite](/img/admin/domains-list.png)

Ο χρήστης μπορεί να κάνει κλικ στο κουμπί **Add Domain** και θα εμφανιστεί ένα παράθυρο modal με κάποιες οδηγίες.

![Modal Προσθήκης Domain με οδηγίες DNS για χρήστες subsite](/img/admin/domains-list.png)

Ο χρήστης μπορεί στη συνέχεια να κάνει κλικ στο **Next Step** και να προχωρήσει στην προσθήκη του προσαρμοσμένου ονόματος domain. Μπορεί επίσης να επιλέξει αν αυτό θα είναι το κύριο domain ή όχι.

![Πεδίο προσθήκης προσαρμοσμένου ονόματος domain με επιλογή κύριου domain](/img/admin/domains-list.png)

![Κάντε κλικ στο Add Domain για να ξεκινήσει η επαλήθευση DNS](/img/admin/domains-list.png)Κάνοντας κλικ στο **Add Domain** θα ξεκινήσει η διαδικασία επαλήθευσης και ανάκτησης των πληροφοριών DNS του προσαρμοσμένου domain.

### Σχετικά με το Domain Syncing

Το Domain Syncing είναι μια διαδικασία όπου το Ultimate Multisite προσθέτει το προσαρμοσμένο όνομα domain στον λογαριασμό hosting σας ως add-on domain **για να λειτουργήσει το domain mapping**.

Το domain syncing συμβαίνει αυτόματα αν ο πάροχος hosting σας έχει ενσωμάτωση με τη δυνατότητα domain mapping του Ultimate Multisite. Προς το παρόν, αυτοί οι πάροχοι hosting είναι οι _Runcloud, Closte, WP Engine, Gridpane, WPMU Dev, Cloudways_ και _Cpanel_.

Θα πρέπει να ενεργοποιήσετε αυτή την ενσωμάτωση στις ρυθμίσεις του Ultimate Multisite στην καρτέλα **Integration**.

![Καρτέλα Integration στις ρυθμίσεις Ultimate Multisite](/img/config/settings-domain-mapping.png)

![Ρυθμίσεις ενσωμάτωσης παρόχου hosting για domain syncing](/img/config/settings-domain-mapping.png)

_Σημειώστε ότι αν ο πάροχος hosting σας δεν είναι ένας από τους παρόχους που αναφέρθηκαν παραπάνω, **θα πρέπει να συγχρονίσετε ή να προσθέσετε χειροκίνητα το όνομα domain** στον λογαριασμό hosting σας._
