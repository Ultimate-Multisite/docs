# Ανατροφοδότηση & Αναφορά Προβλημάτων από Χρήστες {#customer-feedback--issue-reporting}

Το Gratis AI Agent v1.5.0 εισάγει ένα ενσωματωμένο σύστημα ανατροφοδότησης και αναφοράς προβλημάτων που επιτρέπει στους τελικούς χρήστες να σημειώνουν απαντήσεις που δεν ήταν χρήσιμες και να αναφέρουν προβλήματα απευθείας από τη διεπαφή συνομίας. Αυτό το σύστημα περιλαμβάνει διαχείριση συγκατάθεσης, μια αυτοματοποιημένη εντολή αναφοράς και τριάζαρισμα (triage) με υποβοήθηση AI στο backend.

## Κουμπί «Μικρή Άγκυρα» (Thumbs-Down Button) {#thumbs-down-button}

Κάθε μήνυμα που αποστέλλεται από το AI βοηθό εμφανίζει ένα κουμπί **«μικρή άγκυρα»** (👎). Όταν ένας χρήστης κάνει κλικ σε αυτό, μπορεί να σημειώσει μια απάντηση ως μη χρήσιμη ή λανθασμένη.

- Το κουμπί εμφανίζεται κατά το hover δίπλα σε κάθε μήνυμα βοηθού.
- Το κλικ ανοίγει το **Modal Συγκατάθεσης Ανατροφοδότησης** (Feedback Consent Modal).
- Η ανατροφοδότηση συνδέεται με τη ροή συνομίας και το συγκεκριμένο μήνυμα.

## Modal Συγκατάθεσίας Ανατροφοδότησης {#feedback-consent-modal}

Όταν ένας χρήστης κάνει κλικ στο κουμπί «μικρή άγκυρα», εμφανίζεται ένα modal συγκατάθεσης πριν αποσταλεί οποιοδήποτε δεδομένο. Το modal:

- Εξηγεί ποια πληροφορία θα συλλεχθεί (αποσπάσματα συνομίας, πλαίσιο του browser).
- Ζητά από τον χρήστη να επιβεβαιώσει ότι συγκατατίθεται στην κοινή χρήση αυτών των δεδομένων.
- Παρέχει ένα πεδίο κειμένου για να περιγράψει ο χρήστης τι πήγε στραβά.
- Προσφέρει μια επιλογή **Ακύρωση** για να κλείσει χωρίς να υποβάλει.

Δεν καταγράφεται καμία ανατροφοδότηση μέχρι να επιβεβαιώσει ο χρήστης ρητά.

## Banner Ανατροφοδότησης Αυτο-Προτροπής (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

Στο τέλος μιας συνομίας, το βοηθό μπορεί να εμφανίσει ένα **banner ανατροφοδότησης αυτο-προτροπής** — ένα μη επιθετικό μήνυμα που ρωτά αν η συνεδρία ήταν χρήσιμη.

Αυτό το banner εμφανίζεται αυτόματα με βάση τις εκτιμήσεις μήκους και αποτελέσματος της συνομίας. Συνδέεται με τη ίδια ροή ανατροφοδότησης που έχει το κουμπί «μικρή άγκυρα». Οι χρήστες μπορούν να αγνοήσουν το banner χωρίς να παρέχουν ανατροφοδότηση.

## Εντολή Slash /report-issue {#report-issue-slash-command}

Οι χρήστες μπορούν να ενεργοποιήσουν απευθείας τη ροή ανατροφοδότησης πληκτρολογώντας `/report-issue` στην είσοδο συνομίας. Αυτή η εντολή:

- Ανοίγει αμέσως το Modal Συγκατάθεσης Ανατροφοδότησης.
- Συμπληρώνει προεπιλεγμένα το πεδίο περιγραφής με το πλαίσιο σχετικά με την τρέχουσα συνομιλία.
- Επιτρέπει στους χρήστες να προσθέσουν επιπλέον λεπτομέρειες πριν την υποβολή.

Η εντολή `/report-issue` είναι διαθέσιμη σε όλες τις λειτουργίες συνομίας (inline, floating widget, full-screen).

## Τριάζαρισμα με Υποβοήθηση AI {#ai-assisted-triage}

Η υποβληθείσα ανατροφοδότηση направляется σε ένα επίπεδο τριάζαρισμού AI που:

- Κατηγοριοποιεί την αναφορά (υποfactικό λάθος, μη χρήσιμη απάντηση, ανappropriate περιεχόμενο, κ.λπ.).
- Συμπληρώνει σχετικά μεταδεδομένα της συνομίας.
- Μεταforward το περίληψη τριάζαρισματος στο ρυθμισμένο **Endpoint Ανατροφοδότησης** (βλ. [Settings > Advanced](#settings-advanced)).

Αυτό μειώνει τον χρόνο χειροκίνητης αναθεώρησης καθώς αναδεικνύει πρώτα τα πιο κρίσιμα ζητήματα.

## Settings > Advanced {#settings-advanced}

Για να ενεργοποιήσετε το backend ανατροφοδότησης, ρυθμίστε τα ακόλουθα πεδία κάτω από **Gratis AI Agent → Settings → Advanced**:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | Η URL που λαμβάνει POST αιτήματα με payloads ανατροφοδότησης (JSON). |
| **Feedback API Key** | Bearer token που αποστέλλεται στο header `Authorization` για την πιστοποίηση των υποβολών ανατροφοδότησης. |

Αφήστε και τα δύο πεδία κενά για να απενεργοποιήσετε τη συλλογή ανατροφοδότησης. Τα μεμονωμένα κουμπιά ανατροφοδότησης παραμένουν ορατά στους χρήστες, αλλά οι υποβολές δεν θα μεταforwardίζονται.

:::tip
Το endpoint ανατροφοδότησης πρέπει να αποδέχεται ένα σώμα JSON με τουλάχιστον τα πεδία `message_id`, `conversation_id`, `feedback_text` και `triage_category`. Δείτε την τεκμηρίωση του παρόχου endpoint σας για το αναμενόμενο schema.
:::
