# Ultimate AI Connector for Compatible Endpoints

Το Ultimate AI Connector for Compatible Endpoints ενσωματώνει API παρόχων AI συμβατών με το OpenAI με το multisite network του WordPress σας. Διαμορφώστε πολλαπλούς παρόχους AI — ο καθένας με τη δική του διεύθυνση endpoint, κλειδί API και προεπιλεγμένο μοντέλο — και διαδρομικάστε τις αιτήσεις με αυτόματη επαναφορά (fallback) όταν ένας πάροχος είναι недоступен.

## Βασικά Χαρακτηριστικά

- **Υποστήριξη πολλαπλών παρόχων**: Προσθέστε πολλαπλές κάρτες παρόχων AI, η καθεμία με τη δική της διεύθυνση endpoint, κλειδί API και προεπιλεγμένο μοντέλο.
- **Σύλληψη και αναδιάταξη (Drag-to-reorder)**: Διάταξτε τους παρόχους βάσει προτεραιότητας χρησιμοποιώντας την λαβή (grip handle) σε κάθε κάρτα· οι αιτήσεις χρησιμοποιούν τους παρόχους με σειρά.
- **Αυτόματη διαδρομική επαναφορά (Automatic fallback routing)**: Εάν ο κύριος πάροχος είναι недоступен, οι αιτήσεις επαναφέρονται αυτόματα στον επόμενο διαμορφωμένο πάροχο.
- **Διαμόρφωση ανά πάροχο**: Κάθε πάροχος διαμορφώνεται ανεξάρτητα με τα δικά του στοιχεία και την επιλογή μοντέλου.
- **Διεπαφή χρήστη με κάρτες (Card-based settings UI)**: Κατασκευασμένη με σταθερές κάρτες του WordPress 6.9+ (CardBody, CardHeader, CardDivider).
- **Αυτόματη επέκταση κατά την προσθήκη**: Οι νέες κάρτες παρόχων επεκτείνονται αυτόματα, ώστε να μπορείτε να συμπληρώσετε τις λεπτομέρειες αμέσως.
- **Cache busting για scripts**: Οι scripts των παρόχων αποσταθεροποιούνται από το cache κατά την ενημέρωση του plugin, για να διασφαλιστεί ότι φορτώνεται η τελευταία έκδοση.

## Απαιτήσεις

- WordPress 5.3 ή νεότερη
- PHP 7.4 ή νεότερη
- Plugin Ultimate Multisite (ενεργοποιημένο)
- Ένα endpoint API συμβατό με το OpenAI (self-hosted ή τρίτου μέρους)

## Εγκατάσταση

1. Ανεβάστε τα αρχεία του addon στον κατάλογο `/wp-content/plugins/`
2. Ενεργοποιήστε το plugin μέσω του μενού 'Plugins' στο WordPress
3. Μεταβείτε στο **Ultimate Multisite → AI Connector** για να διαμορφώσετε το addon

## Διαμόρφωση

### Ρύθμιση Πολλαπλών Παρόχων

Η οθόνη διαμόρφωσης χρησιμοποιεί ένα layout με κάρτες, όπου κάθε κάρτα αντιπροσωπεύει έναν πάροχο AI. Μπορείτε να προσθέσετε όσους παρόχους χρειάζεστε.

#### Προσθήκη Παρόχου

1. Μεταβείτε στο **Ultimate Multisite → AI Connector** στο network admin
2. Κάντε κλικ στο **Add Provider**
3. Εμφανίζεται και επεκτείνεται αυτόματα μια νέα κάρτα παρόχου
4. Συμπληρώστε τις λεπτομέρειες του παρόχου:
   - **Provider Name** — μια ετικέτα για την αναγνώριση αυτού του παρόχου (π.χ. "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL** — η βασική διεύθυνση URL του API συμβατού με το OpenAI (π.χ. `https://api.openai.com/v1`)
   - **API Key** — το κλειδί API σας για αυτόν τον πάροχο
   - **Default Model** — ο αναγνωριστικός κωδικός του μοντέλου που χρησιμοποιείται ως προεπιλογή για αυτόν τον πάροχο (π.χ. `gpt-4o`, `mistral-small`)
5. Κάντε κλικ στο **Save Settings**

#### Αφαίρεση Παρόχου

Κάντε κλικ στο κουμπί **Remove** σε οποιαδήποτε κάρτα παρόχου για να τον διαγράψετε. Αποθηκεύστε τις ρυθμίσεις για να εφαρμόσετε την αλλαγή.

#### Αναδιάταξη Παρόχων

Σύρετε το εικονίδιο λαβής (⣿) στην αριστερή πλευρά οποιασδήποτε κάρτας παρόχου για να την αναδιατάξετε. Η σειρά καθορίζει την προτεραιότητα επαναφοράς: οι αιτήσεις αποστέλλονται στον πρώτο πάροχο και επαναφέρονται στους επόμενους παρόχους εάν αποτύχει μια αίτηση.

### Διαδρομική Επαναφορά (Fallback Routing)

Όταν μια αίτηση στον κύριο πάροχο αποτύχει — λόγω timeout, rate limit ή διακοπής υπηρεσίας — το connector προσπαθεί αυτόματα ξανά με τον επόμενο πάροχο της λίστας. Αυτό συνεχίζεται κατά τη λίστα μέχρι να επιτύχει ένας πάροχος ή να δοκιμαστούν όλοι οι πάροχοι.

| Σενάριο | Συμπεριφορά |
|----------|-----------|
| Ο κύριος πάροχος ανταποκρίνεται | Η αίτηση εξυπηρετείται κανονικά |
| Ο κύριος πάροχος αποτυγχάνει | Η αίτηση επαναφέρεται με τον επόμενο πάροχο |
| Όλοι οι πάροχοι αποτυγχάνουν | Επιστρέφεται μήνυμα σφάλματος στον κλήση |

Η επαναφορά είναι διαφανής για το plugin ή το theme που καλεί — λαμβάνουν είτε μια επιτυχή απάντηση είτε ένα τελικό σφάλμα, ανεξάρτητα από το πόσοι πάροχοι δοκιμάστηκαν.

### Αναφορά Διαμόρφωσης Παρόχου

| Πεδίο | Απαιτείται | Περιγραφή |
|-------|----------|-------------|
| Provider Name | Yes | Ετικέτα εμφάνισης για αυτόν τον πάροχο |
| Endpoint URL | Yes | Βασική διεύθυνση URL του API συμβατού με το OpenAI |
| API Key | Yes | Κλειδί πιστοποίησης για αυτόν τον πάροχο |
| Default Model | Yes | Αναγνωριστικός κωδικός μοντέλου που χρησιμοποιείται όταν δεν ορίζεται μοντέλο στην αίτηση |

## Αντιμετώπιση Προβλημάτων

### Δεν εμφανίζονται παρόχοι

Εάν η λίστα των παρόχων είναι κενή μετά την αποθήκευση, επιβεβαιώστε ότι:

- Κάνατε κλικ στο **Save Settings** μετά την προσθήκη των παρόχων
- Το browser σας δεν απέκλεισε την αποθήκευση της σελίδας ρυθμίσεων (ελέγξτε την κονσόλα του browser για σφάλματα JavaScript)
- Το plugin είναι ενεργό και βρίσκεστε στην οθόνη του network admin

### Οι αιτήσεις αποτυγχάνουν για όλους τους παρόχους

- Επιβεβαιώστε ότι η endpoint URL κάθε παρόχου είναι προσβάσιμη από τον server σας (και όχι μόνο από το browser σας)
- Ελέγξτε ότι κάθε κλειδί API είναι έγκυρο και δεν έχει λήξει
- Εξέρετε την κατάσταση rate-limit και quota του παρόχου στο dashboard του
- Ενεργοποιήστε το debug logging του WordPress (`WP_DEBUG_LOG`) για να καταγράψετε λεπτομερή μηνύματα σφάλματος από το connector

### Ένας πάροχος δεν χρησιμοποιείται για fallback

Η αναδιάταξη με σύρσιμο αλλάζει τη σειρά επαναφοράς. Επιβεβαιώστε ότι ο πάροχος που αναμένετε να λάβει αιτήσεις fallback είναι τοποθετημένος σωστά στη λίστα και ότι η endpoint URL και το API key του είναι σωστά.

### Η νέα κάρτα παρόχου δεν επεκτείνεται αυτόματα

Η αυτόματη επέκταση απαιτεί να είναι ενεργοποιημένο το JavaScript στο browser σας. Εάν η κάρτα δεν επεκτείνεται με ένα κλικ, ελέγξτε την κονσόλα του browser για σφάλματα. Το περιεχόμενο της κάρτας είναι πάντα προσβάσιμο κάνοντας κλικ στο tiêu đề της κάρτας, ακόμα κι αν η αυτόματη επέκταση δεν ενεργοποιηθεί.
