# Συμβατότητα Tutor Multisite {#tutor-multisite-compatibility}

Ένα plugin WordPress που προσθέτει διορθώσεις συμβατότητας για το Tutor LMS σε περιβάλλοντα multisite.

## Περιγραφή {#description}

Αυτό το plugin παρέχει απαραίτητες διορθώσεις συμβατότητας για τη λειτουργία του Tutor LMS σε δίκτυα WordPress multisite. Διαχειρίζεται την εγγραφή χρηστών, τη διαχείριση εκπαιδευτών, τους περιορισμούς μαθημάτων και άλλες λειτουργίες ειδικές για multisite.

## Χαρακτηριστικά {#features}

- **Διαχείριση Πινάκων Βάσης Δεδομένων**: Ανακαλύπτει και δημιουργεί αυτόματα τα λείποντα πινάκια του Tutor LMS που αποτυγχάνουν λόγω περιορισμών ξένης κλειδιά (foreign key) του multisite.
- **Εγγραφή Χρηστών Multisite**: Προσαρμοσμένοι AJAX handlers για τη διαχείριση της εγγραφής χρηστών σε όλα τα sites του δικτύου.
- **Διαχείριση Ρόλων Εκπαιδευτών**: Αυτόματη ανάθεση και διαχείριση ρόλων εκπαιδευτών σε περιβάλλοντα multisite.
- **Περιορισμοί Μαθημάτων**: Ενσωμάτωση με το Ultimate Multisite για τη διαχείριση των περιορισμών δημιουργίας μαθημάτων και funnels βάσει των σχεδίων των sites.
- **Προσαρμοσμένες Προβολές Διαχειριστή**: Διεπαφή καταλόγου και διαχείρισης εκπαιδευτών που λαμβάνει υπόψη το multisite.
- **Εργαλεία Διαχειριστή**: Διεπαφή διαχείρισης βάσης δεδομένων για τους διαχειριστές του δικτύου και του site.

## Απαιτήσεις {#requirements}

- WordPress 5.0+
- PHP 7.4+
- Plugin Tutor LMS (επιβάλλεται αυτόματα μέσω του κεφαλίδου `Requires Plugins`)
- Εγκατάσταση WordPress Multisite
- Plugin Ultimate Multisite (για τη λειτουργικότητα περιορισμών μαθημάτων)

## Εγκατάσταση {#installation}

1. Ανεβάστε τα αρχεία του plugin στο `/wp-content/plugins/tutor-multisite-compatibility/`
2. Βεβαιωθείτε ότι το Tutor LMS είναι εγκατεστημένο και ενεργοποιημένο
3. Ενεργοποιήστε το plugin μέσω του μενού 'Plugins' στο WordPress
4. Το plugin θα ενσωματωθεί αυτόματα με το Tutor LMS

## Λειτουργικότητα Μετατοπισμένη από το MU Plugin {#functionality-moved-from-mu-plugin}

Αυτό το plugin περιέχει λειτουργικότητα που βρισκόταν προηγουμένως στο `wp-content/mu-plugins/daves.php`:

- Περιορισμοί αλλαγής κατάστασης μαθήματος
- Περιορισμοί δημιουργίας μαθήματος
- Ενσωμάτωση και περιορισμοί CartFlows
- Διαχείριση ρόλων εκπαιδευτών Tutor
- Διαχείριση εγγραφής χρηστών multisite
- Отображение ID προϊόντος μαθήματος

## Διαχείριση Βάσης Δεδομένων {#database-management}

### Το Πρόβλημα {#the-problem}

Στα περιβάλλοντα WordPress multisite, η δημιουργία πινάκων του Tutor LMS αποτυγχάνει για πινάκια με περιορισμούς ξένης κλειδιά (foreign key) στον πίνακα χρηστών. Αυτό συμβαίνει επειδή:

- Τα πινάκια ειδικά για το site χρησιμοποιούν προфиксы όπως `wp_2_tutor_carts`
- Τα foreign keys αναφέρονται στο `wp_2_users` (που δεν υπάρχει)
- Ο πραγματικός πίνακας χρηστών είναι το `wp_users` (κοινό σε όλο το δίκτυο)

### Η Λύση {#the-solution}

Αυτό το plugin αυτόματα:

1. **Ανακαλύπτει Απείρετα Πίνακα**: Ελέγχει για πινάκια που αποτύχουν να δημιουργηθούν λόγω περιορισμών ξένης κλειδιά.
2. **Δημιουργεί Πίνακες με Σωστούς Αναφορές**: Χρησιμοποιεί το σωστό όνομα πίνακα χρηστών (`wp_users` σε multisite).
3. **Διαχειρίζεται Πολλαπλούς Σενarios**:
   - Ενεργοποίηση/ανα활성화 του plugin
   - Δημιουργία νέου site στο δίκτυο
   - Χειροκίνηικός έλεγχος πινάκων μέσω της διεπαφής διαχειριστή

### Πληθαρχικά Πίνακα {#affected-tables}

- `tutor_coupon_usages` - Περιέχει foreign key στον πίνακα χρηστών
- `tutor_carts` - Περιέχει foreign key στον πίνακα χρηστών
- `tutor_notification_preferences` - Πίνακας TutorPro με foreign key στον πίνακα χρηστών

### Διεπαφή Διαχειριστή {#admin-interface}

Μεταβείτε στα εργαλεία διαχείρισης βάσης δεδομένων στη διεύθυνση:

- **Διαχειριστής Δικτύου**: Settings → Tutor Multisite DB
- **Διαχειριστής Site**: Tools → Tutor Multisite DB

Η διεπαφή σας επιτρέπει να:
- Προβάλετε τα λείποντα πινάκια σε όλα τα sites
- Ενεργοποιήσετε χειροκίνητα τη δημιουργία πινάκων
- Ελέγχετε τη διαμόρφωση της βάσης δεδομένων
- Προβάλετε λεπτομερείς πληροφορίες κατάστασης
