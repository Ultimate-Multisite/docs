# Ενσωμάτωση πολυμίσθωσης

Ultimate Multisite: Η πολυμίσθωση 1.2.0 αλλάζει αρκετά σημεία επαφής ενσωμάτωσης για κυρίαρχους μισθωτές, επαλήθευση μετεγκατάστασης και αυτοματοποίηση κύκλου ζωής μισθωτή.

## Ροή αρχικής εκκίνησης μισθωτή {#tenant-bootstrap-flow}

Οι ενσωματώσεις που δημιουργούν ή τροποποιούν μισθωτές θα πρέπει να ακολουθούν αυτήν τη σειρά:

1. Επίλυση της εγγραφής μητρώου μισθωτή και του μοντέλου απομόνωσης.
2. Δημιουργία ή επαλήθευση του writer της βάσης δεδομένων μισθωτή.
3. Αρχική εκκίνηση του σχήματος μισθωτή.
4. Παροχή χρηστών μισθωτή.
5. Καταχώριση δρομολόγησης μισθωτή και διαδρομών συστήματος αρχείων.
6. Εκτέλεση επαλήθευσης μετεγκατάστασης πριν από την έκθεση του μισθωτή.

Μην υποθέτετε ότι ένας κυρίαρχος μισθωτής μπορεί να επαναχρησιμοποιήσει τη σύνδεση βάσης δεδομένων του δικτύου. Χρησιμοποιήστε το μητρώο μισθωτή και τις αφαιρέσεις writer που παρέχονται από το πρόσθετο.

## SSO και REST hooks {#sso-and-rest-hooks}

Η χωρίς κατάσταση αυτόματη σύνδεση μισθωτή χρησιμοποιεί βραχύβια tokens με purpose claim, προστασία επανάληψης JTI, όριο λήξης και καρφίτσωμα προέλευσης. Οι ενσωματώσεις που προσθέτουν κουμπιά σύνδεσης ή συνδέσμους απομακρυσμένης διαχείρισης θα πρέπει να δημιουργούν επισκέψεις μισθωτή μέσω της υποστηριζόμενης ροής SSO αντί να κατασκευάζουν απευθείας URL σύνδεσης μισθωτή.

Τα συμβάντα ελέγχου API από την πλευρά του δικτύου και οι ημερήσιες περιλήψεις είναι διαθέσιμα για πύλες κυρίαρχων μισθωτών. Χρησιμοποιήστε αυτά τα αρχεία καταγραφής κατά την αποσφαλμάτωση εξωτερικών συστημάτων που καλούν τελικά σημεία κύκλου ζωής μισθωτή.

## URL ενεργειών κυρίαρχου πελάτη {#sovereign-customer-action-urls}

Το Ultimate Multisite v2.13.0 δρομολογεί τις ενέργειες πελατών κυρίαρχων μισθωτών πίσω στον κύριο ιστότοπο για ροές Account, ολοκλήρωσης αγοράς, χρέωσης, τιμολογίου, ιστότοπου, αλλαγής προτύπου και αντιστοίχισης domain. Οι ενσωματώσεις που αποδίδουν συνδέσμους διαχείρισης από την πλευρά του μισθωτή θα πρέπει να κατευθύνουν αυτές τις ενέργειες στον πίνακα πελάτη του κύριου ιστότοπου και να περιλαμβάνουν έναν επικυρωμένο στόχο επιστροφής όταν ο χρήστης θα πρέπει να μπορεί να πλοηγηθεί πίσω στον μισθωτή μετά την ολοκλήρωση της ενέργειας.

Χρησιμοποιήστε το βασικό wrapper SSO για συνδέσμους διαχείρισης μεταξύ domain:

```php
$url = wu_with_sso($main_site_customer_url);
```

Το παραγόμενο URL παραμένει φιλτράρισιμο μέσω του `wu_sso_url`, το οποίο λαμβάνει το SSO URL, τον τρέχοντα χρήστη, το ID του στοχευόμενου ιστότοπου και το πλαίσιο ανακατεύθυνσης. Τα πρόσθετα μπορούν να χρησιμοποιούν αυτό το φίλτρο για να προσαρτούν πλαίσιο ειδικό για τον πάροχο ή για να αντικαθιστούν το URL του broker διατηρώντας παράλληλα την επικύρωση token του Ultimate Multisite.

Μην αντιγράφετε την κατάσταση συνδρομής, τιμολογίου, διεύθυνσης χρέωσης, προτύπου ή διαχείρισης domain μέσα στον κυρίαρχο μισθωτή. Αντιμετωπίστε το dashboard του μισθωτή ως εκκινητή και τον πίνακα πελάτη του κύριου ιστότοπου ως το σύστημα καταγραφής για διαχειριζόμενες ενέργειες.

## Επαλήθευση μετεγκατάστασης {#migration-verification}

Αφού μια μετεγκατάσταση ή ενσωμάτωση κύκλου ζωής αλλάξει δεδομένα μισθωτή, εκτελέστε τις πύλες επαλήθευσης:

- `wp tenant verify-no-legacy --site=<site-id>` επιβεβαιώνει ότι ο μισθωτής δεν εξαρτάται πλέον από παλαιού τύπου διαδρομές από την πλευρά του δικτύου.
- `wp tenant verify-sovereign-push --site=<site-id>` επιβεβαιώνει ότι οι εργασίες προώθησης κυρίαρχου μισθωτή μπορούν να επεξεργαστούν και να αδειάσουν την ουρά.

Οι ενσωματώσεις θα πρέπει να αντιμετωπίζουν την αποτυχημένη επαλήθευση ως blocker ανάπτυξης και να αποφεύγουν να επισημαίνουν έναν μισθωτή ως live μέχρι να επιλυθεί η αποτυχία.

## Διαγραφή μισθωτή {#tenant-deletion}

Οι ροές διαγραφής θα πρέπει να καλούν τη διαδρομή teardown του πρόσθετου ώστε να καθαρίζονται τα διαπιστευτήρια βάσης δεδομένων μισθωτή. Οι εξωτερικές ενσωματώσεις μπορούν να αφαιρούν πόρους παρόχου αφού ολοκληρωθεί επιτυχώς το teardown, αλλά δεν θα πρέπει να διαγράφουν βάσεις δεδομένων ή φακέλους host ενώ η επαλήθευση ή οι ασύγχρονες εργασίες προώθησης εξακολουθούν να εκτελούνται.

## Καταργημένος δρομολογητής βάσης δεδομένων {#deprecated-database-router}

Το παλαιού τύπου `Database_Router` έχει αντικατασταθεί από ένα stub κατάργησης. Οι νέες ενσωματώσεις θα πρέπει να επιλύουν μισθωτές μέσω των τρεχόντων API δρομολογητή ιστότοπου και μητρώου μισθωτή αντί να εξαρτώνται από την παλιά κλάση δρομολογητή.
