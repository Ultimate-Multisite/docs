# બહુ-ટેનન્સી

કાર્યક્ષમતા અને સુરક્ષા માટે ઉપસાઇટ્સને સર્વર્સ અને ડેટાબેઝમાં વિતરિત કરો

## આવૃત્તિ 1.2.0 રિલીઝના મુખ્ય મુદ્દા {#version-120-release-highlights}

Multi-Tenancy 1.2.0 ઉત્પાદન આઇસોલેશન વર્કફ્લોઝ માટે સ્વાયત્ત ટેનન્ટ આધાર ઉમેરે છે:

- ટૂંકા સમય માટે માન્ય, રીપ્લે-સુરક્ષિત SSO મુલાકાતો સાથે સ્ટેટલેસ ટેનન્ટ સ્વચાલિત લોગિન.
- સ્વાયત્ત ટેનન્ટ્સ માટે પ્રતિ-ઉપસાઇટ ડેટાબેઝ અને ફાઇલસિસ્ટમ આઇસોલેશન.
- હોસ્ટ-જાગૃત ટેનન્ટ પ્રોવિઝનિંગ માટે Hostinger બહુ-ટેનન્સી ક્ષમતા.
- લેગસી ડેટા તપાસો અને async પુશ પ્રોસેસિંગ માટે સ્વાયત્ત માઇગ્રેશન ચકાસણી ગેટ્સ.
- Bedrock, FrankenPHP, અને localhost ઇન્સ્ટોલ્સ માટે રૂપરેખાંકિત કરી શકાય તેવું સમાન-મશીન ડેટાબેઝ હોસ્ટ બાઇન્ડિંગ.
- ટેનન્ટ બૂટસ્ટ્રેપ, ડેટાબેઝ રાઇટર, સ્કીમા બૂટસ્ટ્રેપ, અને વપરાશકર્તા પ્રોવિઝનિંગ સુધારાઓ.
- ટેનન્ટ ઓળખપત્રોને સાફ કરતું વધુ સુરક્ષિત સ્વાયત્ત ટિયરડાઉન.

[સ્વાયત્ત માઇગ્રેશન વર્કફ્લો](/user-guide/getting-started/sovereign-migration-workflow), [બહુ-ટેનન્સી આઇસોલેશન](/user-guide/configuration/multi-tenancy-isolation), અને [સ્વાયત્ત ટેનન્ટ કામગીરીઓ](/user-guide/administration/sovereign-tenant-operations) માટે વપરાશકર્તા માર્ગદર્શિકાઓ જુઓ.

## આવશ્યકતાઓ {#requirements}

- WordPress 5.3 અથવા ઉચ્ચતર
- PHP 7.4 અથવા ઉચ્ચતર
- Multisite Ultimate plugin (સક્રિય)

## ઇન્સ્ટોલેશન {#installation}

1. addon ફાઇલો તમારી `/wp-content/plugins/` ડિરેક્ટરીમાં અપલોડ કરો
2. WordPress માં 'Plugins' મેનુ દ્વારા plugin સક્રિય કરો
3. નેટવર્ક એડમિનમાં addon સેટિંગ્સ રૂપરેખાંકિત કરો

## વિકાસ {#development}

### સેટઅપ {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### પરીક્ષણ {#testing}

addon માં વ્યાપક પરીક્ષણ ફ્રેમવર્ક શામેલ છે:

- **યુનિટ ટેસ્ટ્સ**: WordPress ટેસ્ટ સ્યુટ સાથે PHPUnit-આધારિત ટેસ્ટ્સ
- **આધાર ટેસ્ટ ક્લાસ**: સામાન્ય ટેસ્ટ પરિસ્થિતિઓ માટે સહાયક પદ્ધતિઓ પ્રદાન કરે છે
- **ટેસ્ટ કવરેજ**: કોડ કવરેજ ટ્રેક કરવા માટે રૂપરેખાંકિત
- **Multisite પરીક્ષણ**: ટેસ્ટ્સ multisite પર્યાવરણમાં ચાલે છે

### કોડ ધોરણો {#code-standards}

- **WordPress કોડિંગ ધોરણો**: PHPCS દ્વારા અમલમાં મૂકાયેલા
- **PHP 7.4+ સુસંગતતા**: આધુનિક PHP સુવિધાઓ સમર્થિત
- **સ્થિર વિશ્લેષણ**: પ્રકાર ચકાસણી માટે PHPStan ઇન્ટિગ્રેશન
- **કોડ આધુનિકીકરણ**: PHP અપગ્રેડ્સ માટે Rector નિયમો

### બિલ્ડ સિસ્ટમ {#build-system}

- **એસેટ પ્રોસેસિંગ**: CSS/JS ફાઇલો માટે મિનિફિકેશન
- **અનુવાદ**: POT ફાઇલ જનરેશન
- **પેકેજિંગ**: વિતરણ માટે આર્કાઇવ બનાવવું
- **વિકાસ મોડ**: ડિબગિંગ માટે અનમિનિફાઇડ એસેટ્સ

## રૂપરેખાંકન {#configuration}

addon નેટવર્ક એડમિન ઇન્ટરફેસ દ્વારા ઉપલબ્ધ વિવિધ રૂપરેખાંકન વિકલ્પોને સમર્થન આપે છે.

### સ્વાયત્ત ટેનન્ટ રૂપરેખાંકન {#sovereign-tenant-configuration}

સ્વાયત્ત ટેનન્ટ્સ માટે, ટેનન્ટ ડેટાબેઝ, ફાઇલસિસ્ટમ રૂટ, ડેટાબેઝ હોસ્ટ બાઇન્ડિંગ, અને હોસ્ટ ક્ષમતાને ટેનન્ટ તરફ ટ્રાફિક રૂટ કરતા પહેલા રૂપરેખાંકિત કરો. ડેટાબેઝ ઓળખપત્રો, હોસ્ટ બાઇન્ડિંગ, રૂટિંગ, અથવા વપરાશકર્તા પ્રોવિઝનિંગને અસર કરતા દરેક ફેરફાર પછી માઇગ્રેશન ચકાસણી ચલાવો.

`localhost` નો ઉપયોગ માત્ર ત્યારે કરો જ્યારે ટેનન્ટ રનટાઇમ અને ડેટાબેઝ grants સ્થાનિક સોકેટ અથવા સમાન-મશીન ઍક્સેસની અપેક્ષા રાખે. જ્યારે તમારા પ્રદાતા દ્વારા રૂપરેખાંકિત grant host તે હોય ત્યારે `127.0.0.1` અથવા સર્વિસ hostname નો ઉપયોગ કરો.

### Hostinger ક્ષમતા {#hostinger-capability}

Hostinger બહુ-ટેનન્સી ક્ષમતા શેર કરેલા Hostinger ઇન્ટિગ્રેશન token સાથે કામ કરે છે. ડોમેન hPanel માં હાજર છે, DNS Hostinger Account તરફ પોઇન્ટ કરે છે, અને ચકાસણી પહેલાં ટેનન્ટ ડેટાબેઝ સેટિંગ્સ hPanel સાથે મેળ ખાય છે તેની પુષ્ટિ કરો.

## હૂક્સ અને ફિલ્ટર્સ {#hooks-and-filters}

### ક્રિયાઓ {#actions}

- `ultimate-multisite-addon-template_init` - addon આરંભ પછી ચલાવવામાં આવે છે
- `ultimate-multisite-addon-template_loaded` - તમામ addon ફાઇલો લોડ થયા પછી ચલાવવામાં આવે છે

### ફિલ્ટર્સ {#filters}

- `ultimate-multisite-addon-template_settings` - addon સેટિંગ્સમાં ફેરફાર કરો
- `ultimate-multisite-addon-template_enabled` - addon સક્ષમ/અક્ષમ સ્થિતિને ઓવરરાઇડ કરો
