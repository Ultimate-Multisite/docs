# ઇમેઇલ્સ

Ultimate Multisite: Emails એડઓન નેટવર્ક માલિકોને સાઇટ્સ અને મેમ્બરશિપની સાથે ગ્રાહકોના ઇમેઇલ એકાઉન્ટ્સ સેટ કરવા અને તેનું સંચાલન કરવાની મંજૂરી આપે છે. વર્ઝન 1.1.0 માં ગ્રાહક-સામનો ઇમેઇલ એકાઉન્ટ મેનેજમેન્ટ, વેબમેઇલ single sign-on, CyberPanel ફોરવર્ડર સપોર્ટ અને ચેકઆઉટ-સમયના સેટઅપમાં સુધારા ઉમેરવામાં આવ્યા છે.

## મુખ્ય વિશેષતાઓ

* **ગ્રાહક ઇમેઇલ એકાઉન્ટ વિજેટ**: ગ્રાહકો હવે સપોર્ટ ટિકિટ ખોલ્યા વિના તેમના એકાઉન્ટ પેજ પરથી ઇમેઇલ એકાઉન્ટ બનાવી અને તેનું સંચાલન કરી શકે છે.
* **વેબમેઇલ SSO અને મેજિક લિંક્સ**: જ્યારે પ્રદાતા (provider) સપોર્ટ કરે છે, ત્યારે ગ્રાહકો એક-ક્લિક ઓથેન્ટિકેશન સાથે એકાઉન્ટ પેજ પરથી સીધા વેબમેઇલ ખોલી શકે છે.
* **પાસવર્ડ બદલવા**: ગ્રાહકો અને એડમિનિસ્ટ્રેટર્સ ઇમેઇલ મેનેજમેન્ટ વિજેટમાંથી મેઇલબોક્સ પાસવર્ડ અપડેટ કરી શકે છે.
* **ચેકઆઉટ સેટઅપ**: ગ્રાહક ઇમેઇલ એકાઉન્ટની રચના ચેકઆઉટ દરમિયાન થઈ શકે છે, જેથી ખરીદી પ્રક્રિયા પૂર્ણ થયા પછી મેઇલબોક્સ તૈયાર હોય.
* **CyberPanel ફોરવર્ડર્સ**: CyberPanel-આધારિત નેટવર્ક્સ ઇમેઇલ સેવા પ્રવાહના ભાગરૂપે ઇમેઇલ ફોરવર્ડર બનાવી અને તેનું સંચાલન કરી શકે છે.
* **પ્રદાતા-સભાન સેટિંગ્સ**: સેટિંગ્સ પેજ એડમિનિસ્ટ્રેટર્સને ફક્ત ઉપયોગમાં લઈ શકાય તેવા વિકલ્પો બતાવે છે, કારણ કે તે સક્રિય અને કન્ફિગર કરેલા ઇન્ટિગ્રેશન્સ દ્વારા પ્રદાતાઓને ફિલ્ટર કરે છે.

## જરૂરિયાતો

* Ultimate Multisite 2.0.0 અથવા તેથી વધુ
* WordPress Multisite ઇન્સ્ટોલેશન
* ઓછામાં ઓછું એક કન્ફિગર કરેલ ઇમેઇલ પ્રદાતા ઇન્ટિગ્રેશન
* પ્રદાતા API ક્રેડેન્શિયલ્સ જેમાં મેઇલબોક્સ બનાવવા, પાસવર્ડ બદલવા અને વેબમેઇલ એક્સેસ લિંક્સ આપવાની પરવાનગી હોય (જ્યાં સપોર્ટેડ હોય)

## ઇન્સ્ટોલેશન

1. પ્લગઇન ફાઇલોને `/wp-content/plugins/ultimate-multisite-emails` પર અપલોડ કરો.
2. WordPress નેટવર્ક એડમિનમાંથી **Plugins** માંથી એડઓન નેટવર્ક સ્તરે સક્રિય કરો.
3. **Ultimate Multisite** > **Settings** પર જાઓ અને એડઓન ઇમેઇલ સેટિંગ્સ ખોલો.
4. તે ઇમેઇલ પ્રદાતા ઇન્ટિગ્રેશન્સને સક્ષમ કરો જે તમે ગ્રાહકોને વાપરવા આપવા માંગો છો.
5. સેટિંગ્સ સેવ કરો, અને પછી કોઈ બિન-ઉત્પાદન ગ્રાહક અથવા સ્ટેજિંગ સાઇટ સાથે એકાઉન્ટ બનાવવાનો ટેસ્ટ કરો.

## ગ્રાહક એકાઉન્ટ ઇમેઇલ મેનેજમેન્ટ

વર્ઝન 1.1.0 માં ગ્રાહક એકાઉન્ટ પેજ પર ઇમેઇલ મેનેજમેન્ટ વિજેટ ઉમેરવામાં આવ્યું છે. તેનો ઉપયોગ ત્યારે કરો જ્યારે ગ્રાહકોને તેમના પોતાના સાઇટ્સ માટે મેઇલબોક્સ બનાવવાની અને જાળવણી કરવાની સ્વ-સેવા (self-service) કરવાની જરૂર હોય.

એકાઉન્ટ પેજ પરથી, ગ્રાહકો નીચેના કાર્યો કરી શકે છે:

1. તેમની વર્તમાન મેમ્બરશિપ અથવા સાઇટ સાથે જોડાયેલા ઇમેઇલ એકાઉન્ટ્સ જોવી.
2. જ્યારે તેમની યોજના વધારાના એકાઉન્ટ્સની મંજૂરી આપે છે ત્યારે નવો મેઇલબોક્સ બનાવવો.
3. હાલના મેઇલબોક્સનો પાસવર્ડ બદલવો.
4. SSO અથવા મેજિક-લિંક લોગિન દ્વારા સપોર્ટેડ વેબમેઇલ પ્રદાતા ખોલવા.

આ વિજેટ Ultimate Multisite એકાઉન્ટ-પેજ કરન્ટ્સ સિસ્ટમનો ઉપયોગ કરે છે, તેથી ક્રિયાઓ ગ્રાહક જે એકાઉન્ટ જોઈ રહ્યો છે તેના સંદર્ભમાં મર્યાદિત હોય છે. આ ગ્રાહકોને અન્ય મેમ્બરશિપ અથવા સાઇટના ઇમેઇલ એકાઉન્ટ્સનું સંચાલન કરતા અટકાવે છે.

## વેબમેઇલ SSO અને મેજિક-લિંક લોગિન

વેબમેઇલ SSO ગ્રાહકોને એકાઉન્ટ પેજ પરથી સીધું **Open webmail** એક્શન આપે છે. જ્યારે સક્રિય પ્રદાતા દ્વારા સપોર્ટેડ હોય, ત્યારે એડઓન એક ટૂંકા ગાળાના એક્સેસ URL માટે વિનંતી કરે છે અને ગ્રાહકને વેબમેઇલમાં રીડાયરેક્ટ કરે છે, જ્યારે તેમને મેઇલબોક્સ ક્રેડેન્શિયલ્સ કોપી કરવાની જરૂર પડતી નથી.

પ્રદાતાનું વર્તન અલગ હોઈ શકે છે:

* જે પ્રદાતા પાસે native SSO APIs હોય છે તેઓ સીનૈંતર ઓથેન્ટિક વેબમેઇલ સેશન બનાવી શકે છે.
* જે પ્રદાતા પાસે મેજિક-લિંક સપોર્ટ હોય છે તેઓ સમય મર્યાદિત લોગિન લિંક જનરેટ કરી શકે છે.
* જે પ્રદાતા પાસે SSO અથવા મેજિક-લિંક સપોર્ટ નથી તેઓએ હજુ પણ મેઇલબોક્સની વિગતો અને પાસવર્ડ-મેનેજમેન્ટ ક્રિયાઓ બતાવવી જોઈએ, પરંતુ એક-ક્લિક વેબમેઇલ બટન નહીં.

વેબમેઇલ એક્સેસ ફક્ત તે પ્રદાતા માટે સક્ષમ કરો જ્યાં તમે જરૂરી API ક્રેડેન્શિયલ્સ કન્ફિગર કર્યા હોય. જો કોઈ પ્રદાતા માટે એક્શન ખૂટે છે, તો પહેલા ખાતરી કરો કે પ્રદાતા સક્રિય, કન્ફિગર કરેલ છે અને વેબમેઇલ લોગિન લિંક્સને સપોર્ટ કરે છે.

## ચેકઆઉટ સેટઅપ

એડઓન ચેકઆઉટના ભાગરૂપે ગ્રાહક ઇમેઇલ એકાઉન્ટ બનાવી શકે છે. આ ત્યારે ઉપયોગી છે જ્યારે યોજનામાં ડિફોલ્ટ તરીકે મેઇલબોક્સ શામેલ હોય અથવા જ્યારે ચેકઆઉટ ફોર્મ ઇમેઇલ-એકાઉન્ટની વિગતો એકત્રિત કરે.

ભલામણ કરેલ અમલ:

1. ચેકઆઉટ સેટઅપ સક્ષમ કરતા પહેલા પ્રદાતા અને યોજનાની મર્યાદાઓ કન્ફિગર કરો.
2. સ્ટેજિંગ પ્રોડક્ટ સાથે ખરીદીનો ટેસ્ટ કરો અને ખાતરી કરો કે ચેકઆઉટ પછી મેઇલબોક્સ બનાવવામાં આવ્યું છે.
3. નવા મેઇલબોક્સની ખાતરી કરવા માટે ગ્રાહક એકાઉન્ટ પેજની સમીક્ષા કરો જે ઇમેઇલ વિજેટમાં દેખાય છે.
4. પ્રોડક્શન પ્રોડક્ટ્સ માટે ફ્લો સક્ષમ કરતા પહેલા પાસવર્ડ-બદલ અને વેબમેઇલ ક્રિયાઓની ખાતરી કરો.

જો સેટઅપ નિષ્ફળ જાય, તો પણ ગ્રાહક એકાઉન્ટ અને મેમ્બરશિપ અસ્તિત્વમાં રહી શકે છે જ્યારે મેઇલબોક્સ માટે એડમિનિસ્ટ્રેટરના ફોલો-અપની જરૂર હોય છે. ફરી પ્રયાસ કરતા પહેલા Ultimate Multisite activity logs અને પ્રદાતા logs તપાસો.

## CyberPanel ઇમેઇલ ફોરવર્ડર્સ

CyberPanel-આધારિત નેટવર્ક્સ માટે, વર્ઝન 1.1.0 માં ઇમેઇલ ફોરવર્ડર સપોર્ટ ઉમેરવામાં આવ્યો છે. ફોરવર્ડર્સનો ઉપયોગ ત્યારે કરો જ્યારે ગ્રાહકોને `info@customer-domain.test` જેવા એડ્રેસની જરૂર હોય જે સંપૂર્ણ મેઇલબોક્સ બનાવ્યા વિના મેઇલને અન્ય ઇનબોક્સ પર રીડાયરેક્ટ કરે.

ફોરવર્ડર્સ ઓફર કરતા પહેલા, ખાતરી કરો કે CyberPanel ઇન્ટિગ્રેશન કન્ફિગર અને એક્સેસિબલ છે. જરૂરી હોસ્ટ, યુઝરનેમ, પાસવર્ડ અને પેકેજ કોન્સ્ટન્ટ્સ માટે [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) ગાઇડ જુઓ.

## વારંવાર પૂછાતા પ્રશ્નો

## શું ગ્રાહકો પોતાના ઇમેઇલ એકાઉન્ટ બનાવી શકે છે?

હા. જ્યારે ગ્રાહક એકાઉન્ટ વિજેટ સક્ષમ હોય અને યોજના ઇમેઇલ એકાઉન્ટ્સની મંજૂરી આપે છે, ત્યારે ગ્રાહકો એકાઉન્ટ પેજ પરથી તેમના પોતાના મેઇલબોક્સ બનાવી અને તેનું સંચાલન કરી શકે છે.

## શું દરેક પ્રદાતા વેબમેઇલ SSOને સપોર્ટ કરે છે?

ના. SSO અને મેજિક-લિંક સપોર્ટ સક્રિય ઇમેઇલ પ્રદાતા પર આધાર રાખે છે. જે પ્રદાતા પાસે એક્સેસ-લિંક API નથી તેઓ પણ મેઇલબોક્સ બનાવટ અને પાસવર્ડ ફેરફારોને સપોર્ટ કરી શકે છે.

## શું એડમિનિસ્ટ્રેટર્સ ગ્રાહકો માટે ઇમેઇલ એકાઉન્ટ બનાવી શકે છે?

હા. એડમિનિસ્ટ્રેટર્સ એડમિન ઇન્ટરફેસમાંથી ઇમેઇલ એકાઉન્ટ બનાવી શકે છે, જેમાં એકાઉન્ટ બનાવતી વખતે પ્રારંભિક પાસવર્ડ સેટ કરવાનો સમાવેશ થાય છે.

## શું ફોરવર્ડર્સ મેઇલબોક્સ જેવા છે?

ના. મેઇલબોક્સ મેઇલ સ્ટોર કરે છે અને તેમાં લોગિન ક્રેડેન્શિયલ્સ હોય છે. ફોરવર્ડર મેઇલને અન્ય એડ્રેસ પર રીડાયરેક્ટ કરે છે અને અલગ ઇનબોક્સ પ્રદાન કરતું નથી.
