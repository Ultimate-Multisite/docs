# ઇમેઇલ્સ {#emails}

Ultimate Multisite: Emails એડઑન નેટવર્ક માલિકોને સાઇટ્સ અને મેમ્બરશિપ્સની સાથે ગ્રાહક ઇમેઇલ Account પ્રોવિઝન અને મેનેજ કરવાની મંજૂરી આપે છે. વર્ઝન 1.1.0 ગ્રાહક-મુખી ઇમેઇલ Account મેનેજમેન્ટ, વેબમેઇલ સિંગલ સાઇન-ઑન, CyberPanel ફૉર્વર્ડર સપોર્ટ અને ચેકઆઉટ-સમય પ્રોવિઝનિંગ સુધારાઓ ઉમેરે છે.

## મુખ્ય સુવિધાઓ {#key-features}

* **ગ્રાહક ઇમેઇલ Account વિજેટ**: ગ્રાહકો પાસે સક્રિય ઇમેઇલ મેમ્બરશિપ હોય તે પછી તેઓ તેમના Account પેજ પરથી મેઇલબૉક્સ બનાવી અને મેનેજ કરી શકે છે.
* **વેબમેઇલ SSO અને મેજિક લિંક્સ**: પ્રોવાઇડર તેને સપોર્ટ કરે ત્યારે ગ્રાહકો એક-ક્લિક ઓથેન્ટિકેશન સાથે Account પેજ પરથી સીધું વેબમેઇલ ખોલી શકે છે.
* **પાસવર્ડ ફેરફારો**: ગ્રાહકો અને એડમિનિસ્ટ્રેટર્સ ઇમેઇલ મેનેજમેન્ટ વિજેટમાંથી મેઇલબૉક્સ પાસવર્ડ અપડેટ કરી શકે છે.
* **ચેકઆઉટ એન્ટાઇટલમેન્ટ પ્રોવિઝનિંગ**: ચેકઆઉટ ઇમેઇલ મેમ્બરશિપ બનાવે છે, તેને ગ્રાહકના સાઇટ ડોમેન સાથે લિંક કરે છે, અને Account-પેજ મેઇલબૉક્સ ટૂલ્સ ઉપલબ્ધ બનાવે છે.
* **CyberPanel ફૉર્વર્ડર્સ**: CyberPanel-આધારિત નેટવર્ક્સ ઇમેઇલ સેવા પ્રવાહના ભાગરૂપે ઇમેઇલ ફૉર્વર્ડર્સ બનાવી અને મેનેજ કરી શકે છે.
* **પ્રોવાઇડર-જાગૃત સેટિંગ્સ**: સેટિંગ્સ પેજ પ્રોવાઇડર્સને સક્રિય, કન્ફિગર કરેલા ઇન્ટિગ્રેશન્સ સુધી ફિલ્ટર કરે છે જેથી એડમિનિસ્ટ્રેટર્સ માત્ર ઉપયોગી વિકલ્પો જ જુએ.

## આવશ્યકતાઓ {#requirements}

* Ultimate Multisite 2.0.0 અથવા વધુ
* WordPress Multisite ઇન્સ્ટૉલેશન
* ઓછામાં ઓછું એક કન્ફિગર કરેલું ઇમેઇલ પ્રોવાઇડર ઇન્ટિગ્રેશન
* મેઇલબૉક્સ બનાવવા, પાસવર્ડ બદલવા, અને જ્યાં સપોર્ટેડ હોય ત્યાં વેબમેઇલ ઍક્સેસ લિંક્સ જારી કરવાની પરવાનગી ધરાવતા પ્રોવાઇડર API ક્રેડેન્શિયલ્સ

## ઇન્સ્ટૉલેશન {#installation}

1. plugin ફાઇલો `/wp-content/plugins/ultimate-multisite-emails` પર અપલોડ કરો.
2. WordPress નેટવર્ક એડમિનમાં **Plugins** માંથી એડઑન નેટવર્ક ઍક્ટિવેટ કરો.
3. **Ultimate Multisite** > **Settings** પર જાઓ અને એડઑન ઇમેઇલ સેટિંગ્સ ખોલો.
4. તમે ગ્રાહકોને ઉપયોગ કરવા દેવા માંગતા હો તે ઇમેઇલ પ્રોવાઇડર ઇન્ટિગ્રેશન્સ સક્ષમ કરો.
5. સેટિંગ્સ સાચવો, પછી નૉન-પ્રોડક્શન ગ્રાહક અથવા સ્ટેજિંગ સાઇટ સાથે Account બનાવવાનું પરીક્ષણ કરો.

## ઇમેઇલ વેચાણ પ્રવાહ કેવી રીતે કામ કરે છે {#how-the-email-seller-flow-works}

ઇમેઇલ વેચાણમાં બે અલગ સ્તરો હોય છે:

1. **એન્ટાઇટલમેન્ટ** — Ultimate Multisite ઇમેઇલ ઉત્પાદન/મેમ્બરશિપ જે કહે છે કે ગ્રાહકને કોઈ ચોક્કસ સાઇટ ડોમેન, પ્રોવાઇડર, ક્વોટા અને Account મર્યાદા માટે ઇમેઇલનો ઉપયોગ કરવાની મંજૂરી છે.
2. **મેઇલબૉક્સ બનાવવું** — વાસ્તવિક પ્રોવાઇડર મેઇલબૉક્સ, જેમ કે `info@example.com`, જે ગ્રાહક અથવા નેટવર્ક એડમિનિસ્ટ્રેટર લોકલ ભાગ અને પાસવર્ડ પસંદ કરે ત્યારે બનાવવામાં આવે છે.

ચેકઆઉટ દરમિયાન, એડઑન **ઇમેઇલ મેમ્બરશિપ** બનાવે છે અથવા સક્રિય કરે છે અને તે મેમ્બરશિપ પર ગ્રાહકનું સાઇટ ડોમેન અને પ્રોવાઇડર રેકોર્ડ કરે છે. તે મેઇલબૉક્સ નામ અથવા પાસવર્ડનો અંદાજ લગાવતું નથી. ખરીદી પૂર્ણ થયા પછી અને સાઇટ/ડોમેન સંદર્ભ અસ્તિત્વમાં આવ્યા પછી, ગ્રાહક Account પેજ ખોલી શકે છે અને તેઓ ઇચ્છતા મેઇલબૉક્સ નામો બનાવી શકે છે.

આનો અર્થ એ છે કે સામાન્ય ગ્રાહક પ્રવાહ આ છે:

1. ગ્રાહક સાઇટ પ્લાન ખરીદે છે અને ચેકઆઉટ દરમિયાન ઇમેઇલ ઉત્પાદન ઉમેરે છે.
2. Ultimate Multisite સાઇટ મેમ્બરશિપ અને દ્વિતીય ઇમેઇલ મેમ્બરશિપ બનાવે છે.
3. Emails એડઑન ઇમેઇલ મેમ્બરશિપને ગ્રાહકના સાઇટ ડોમેન અને પ્રોવાઇડર સાથે લિંક કરે છે.
4. ગ્રાહક તેમના Dashboard માં **Account** ખોલે છે.
5. તે સક્રિય ઇમેઇલ મેમ્બરશિપ માટે **ઇમેઇલ Accounts** પેનલ દેખાય છે.
6. ગ્રાહક **Account બનાવો** પર ક્લિક કરે છે, `info` જેવા લોકલ ભાગ દાખલ કરે છે, પાસવર્ડ સેટ કરે છે, અને એડઑન કન્ફિગર કરેલા પ્રોવાઇડર પર `info@their-domain.com` બનાવે છે.

## ઇમેઇલ ઉત્પાદન કન્ફિગર કરો {#configure-an-email-product}

તમે વેચવા અથવા ભેટમાં આપવા માંગતા દરેક ઇમેઇલ પેકેજ માટે એક ઇમેઇલ ઉત્પાદન બનાવો. જો તમે ઉત્પાદન સેટઅપમાં નવા હો, તો [તમારું પ્રથમ સબ્સ્ક્રિપ્શન ઉત્પાદન બનાવવું](../../user-guide/configuration/creating-your-first-subscription-product) થી શરૂ કરો અને પછી ઇમેઇલ-વિશિષ્ટ સેટિંગ્સ માટે અહીં પાછા આવો.

1. **Network Admin > Ultimate Multisite > Products > Add Product** પર જાઓ.
2. **Product Type** ને **Email** પર સેટ કરો.
3. કિંમત કન્ફિગર કરો. વેચાયેલા પ્લાન માટે ચૂકવેલ પુનરાવર્તિત કિંમત વાપરો, અથવા જો તે કોમ્પ્લિમેન્ટરી ઇમેઇલ પ્લાન હોવો જોઈએ તો ઉત્પાદનને મફત/શૂન્ય-કિંમત પર સેટ કરો.
4. **Email Settings** વિભાગ ખોલો.
5. કન્ફિગર કરેલું **Email Provider** પસંદ કરો.
6. જરૂરી હોય તેમ **Number of Accounts**, **Storage Quota (MB)**, alias મર્યાદાઓ, ફૉર્વર્ડર્સ અને ઑટોરિસ્પોન્ડર્સ સેટ કરો.
7. ઉત્પાદન સાચવો.

માત્ર તે પ્રોવાઇડર્સ જ પ્રોવાઇડર સિલેક્ટરમાં દેખાય છે જે **Integrations** પેજ પર સક્ષમ અને સંપૂર્ણ રીતે કન્ફિગર કરેલા હોય છે. જો સિલેક્ટર ખાલી હોય, તો પહેલા પ્રોવાઇડર કન્ફિગર અને પરીક્ષણ કરો.

## ચેકઆઉટમાં ઇમેઇલ ઉમેરો {#add-email-to-checkout}

ઇમેઇલ ઉત્પાદનો સામાન્ય રીતે સાઇટ પ્લાનમાં એડ-ઑન તરીકે ઑફર કરવામાં આવે છે. સામાન્ય ચેકઆઉટ એડિટર વર્કફ્લો માટે [ચેકઆઉટ ફોર્મ્સ](../../user-guide/configuration/checkout-forms) જુઓ.

1. ગ્રાહકો પ્રથમ ખરીદે છે તે સાઇટ પ્લાન બનાવો અથવા સંપાદિત કરો.
2. તે ચેકઆઉટ પ્રવાહ માટે ઇમેઇલ ઉત્પાદનને ઉપલબ્ધ એડ-ઑન/ઓર્ડર બમ્પ તરીકે ઉમેરો.
3. ખાતરી કરો કે ચેકઆઉટ ફોર્મમાં હજી પણ સામાન્ય જરૂરી સાઇટ ફીલ્ડ્સ, ખાસ કરીને સાઇટ URL/ડોમેન ફીલ્ડ, સામેલ છે.
4. સ્ટેજિંગ ગ્રાહક સાથે પરીક્ષણ ચેકઆઉટ ચલાવો.
5. ચેકઆઉટ પછી, ગ્રાહકનું [Account પેજ](../../user-guide/client-management/account-page) ખોલો અને **ઇમેઇલ Accounts** પેનલ દેખાય છે તેની પુષ્ટિ કરો.

ઇમેઇલ ડોમેન ગ્રાહકના સાઇટ ડોમેનમાંથી લેવામાં આવે છે. ગ્રાહકો ઇમેઇલ ઉત્પાદન પગલામાં ઇમેઇલ ડોમેન દાખલ કરતા નથી.

## ગ્રાહક Account ઇમેઇલ મેનેજમેન્ટ {#customer-account-email-management}

વર્ઝન 1.1.0 ગ્રાહક Account પેજમાં ઇમેઇલ મેનેજમેન્ટ વિજેટ ઉમેરે છે. જ્યારે ગ્રાહકોએ પોતાની સાઇટ્સ માટે મેઇલબૉક્સ બનાવવું અને જાળવણી જાતે સેવા તરીકે કરવી જોઈએ ત્યારે તેનો ઉપયોગ કરો.

Account પેજ પરથી, ગ્રાહકો કરી શકે છે:

1. તેમની વર્તમાન ઇમેઇલ મેમ્બરશિપ/સાઇટ ડોમેન સાથે જોડાયેલા ઇમેઇલ Accounts જોઈ શકે છે.
2. જ્યારે તેમનું ઇમેઇલ ઉત્પાદન વધારાના Accountsની મંજૂરી આપે ત્યારે નવું મેઇલબૉક્સ બનાવી શકે છે.
3. અસ્તિત્વમાં રહેલા મેઇલબૉક્સનો પાસવર્ડ બદલી શકે છે.
4. SSO અથવા મેજિક-લિંક લૉગિન દ્વારા સપોર્ટેડ વેબમેઇલ પ્રોવાઇડર્સ ખોલી શકે છે.

વિજેટ Ultimate Multisite Account-પેજ currents સિસ્ટમનો ઉપયોગ કરે છે, તેથી ક્રિયાઓ ગ્રાહક જોઈ રહ્યો હોય તે Account સંદર્ભ સુધી મર્યાદિત રહે છે. આ ગ્રાહકોને બીજી મેમ્બરશિપ અથવા સાઇટના ઇમેઇલ Accounts મેનેજ કરવાથી અટકાવે છે.

જો ગ્રાહકને email પેનલ ન દેખાય, તો તપાસો કે તેમની પાસે **સક્રિય Email product membership** છે, membership કોઈ site domain સાથે linked છે, અને product પાસે configured provider છે.

## ગ્રાહકને મફત Email plan આપવો {#giving-a-customer-a-free-email-plan}

ગ્રાહકને ચાર્જ કર્યા વિના email આપવા માટે, પહેલા **email entitlement** grant કરો, પછી mailbox બનાવો અથવા ગ્રાહકને બનાવવા દો.

ભલામણ કરેલા વિકલ્પો:

* **મફત email product બનાવો**: email product price ને free/zero પર સેટ કરો, પછી **Ultimate Multisite > Memberships > Add Membership** માંથી તે ગ્રાહક માટે active membership ઉમેરો. email product અને ગ્રાહક પસંદ કરો. Free memberships બદલાય અથવા cancel થાય ત્યાં સુધી lifetime હોય છે. જો email membership કોઈ existing site ને follow કરવા માટે હોય, તો ખાતરી કરો કે તે તે site/domain context સાથે linked છે.
* **100% discount અથવા free checkout link વાપરો**: ગ્રાહકને checkoutમાંથી પસાર થવા દો જેથી Ultimate Multisite email membership અને payment record આપમેળે બનાવે.
* **Admin દ્વારા mailbox બનાવો**: ગ્રાહક માટે provider પર ચોક્કસ mailbox બનાવવા **Ultimate Multisite > Email Accounts > Add Email Account** પર જાઓ. આ support માટે ઉપયોગી છે, પરંતુ તે ગ્રાહકના self-service account-page tools ને શક્તિ આપતી email product membership ને replace કરતું નથી.

જો તમે ઇચ્છો છો કે ગ્રાહક account page માંથી પોતે mailbox manage કરે, તો ખાતરી કરો કે તેમની પાસે domain માટે active email membership છે. તે membership વગર manually created provider mailbox હોઈ શકે છે, પરંતુ ગ્રાહકને તેના માટે self-service email panel ન દેખાઈ શકે.

## Admin દ્વારા બનાવેલા Accounts અને "No Order Number" {#admin-created-accounts-and-no-order-number}

**Email Accounts** admin page provider mailbox બનાવે અથવા edit કરે છે. Email plan વેચવા અથવા grant કરવા માટે આ પ્રથમ step નથી.

Email account edit કરતી વખતે, **Membership** field existing Ultimate Multisite memberships ને તેમના reference/order number દ્વારા search કરે છે. જો તમે ગ્રાહક પાસે email membership હોય તે પહેલાં account assign કરવાનો પ્રયાસ કરો છો, તો select કરવા માટે order number ન હોઈ શકે.

તેના બદલે operations નો આ order વાપરો:

1. ગ્રાહકની email membership બનાવો અથવા શોધો. Purchases માટે, checkout આ કરે છે. Complimentary plan માટે, [Memberships](../../user-guide/administration/managing-memberships) માંથી membership manually બનાવો અથવા ગ્રાહકને free checkout મારફતે મોકલો.
2. Confirm કરો કે membership active છે અને Email product વાપરે છે.
3. Confirm કરો કે email membership site domain/provider સાથે linked છે.
4. Customer account page અથવા **Ultimate Multisite > Email Accounts** માંથી mailbox બનાવો.
5. જરૂર હોય તો, mailbox record edit કરો અને customer, membership, અને site assign કરો.

જો તમને customer માટે ફક્ત provider mailbox બનાવવાની જરૂર હોય અને self-service account-page management ની જરૂર ન હોય, તો admin-created account પૂરતું હોઈ શકે. જો તમે ઇચ્છો છો કે customer તેને Ultimate Multisite માં જુએ અને manage કરે, તો email membership પણ બનાવો.

## Webmail SSO અને Magic-Link Login {#webmail-sso-and-magic-link-login}

Webmail SSO ગ્રાહકોને account pageમાંથી સીધી **Open webmail** action આપે છે. Active provider દ્વારા supported હોય ત્યારે, addon short-lived access URL request કરે છે અને ગ્રાહકને mailbox credentials copy કરવાનું કહ્યા વગર webmailમાં redirect કરે છે.

Provider behaviour બદલાઈ શકે છે:

* Native SSO APIs ધરાવતા providers સીધું authenticated webmail session બનાવી શકે છે.
* Magic-link support ધરાવતા providers time-limited login link generate કરી શકે છે.
* SSO અથવા magic-link support વગરના providers એ mailbox details અને password-management actions તો હજુ પણ બતાવવી જોઈએ, પરંતુ one-click webmail button નહીં.

Webmail access ફક્ત તે providers માટે enable કરો જ્યાં તમે જરૂરી API credentials configured કર્યા છે. જો કોઈ provider માટે action missing હોય, તો પહેલા confirm કરો કે provider active, configured છે અને webmail login links support કરે છે.

## Checkout Provisioning {#checkout-provisioning}

addon checkout દરમિયાન email **membership context** provision કરે છે. તે provider, account limit, quota, અને site domain record કરે છે જેથી ગ્રાહકો purchase પછી account pageમાંથી પોતાના mailbox names બનાવી શકે.

ભલામણ કરેલો rollout:

1. Email product offer કરતા પહેલાં provider અને product limits configure કરો.
2. Staging product સાથે purchase test કરો અને confirm કરો કે email membership બનાવાઈ છે.
3. Domain માટે **Email Accounts** widget દેખાય છે તેની ખાતરી કરવા customer account page review કરો.
4. Widget માંથી test mailbox બનાવો.
5. Production products માટે flow enable કરતા પહેલાં password-change અને webmail actions confirm કરો.

જો provisioning fail થાય, તો customer account અને membership હજી પણ exist કરી શકે છે જ્યારે email membership અથવા provider mailbox માટે administrator follow-up જરૂરી હોય. Retry કરતા પહેલાં Ultimate Multisite activity logs અને provider logs check કરો.

## CyberPanel Email Forwarders {#cyberpanel-email-forwarders}

CyberPanel-backed networks માટે, version 1.1.0 email forwarder support ઉમેરે છે. જ્યારે ગ્રાહકોને `info@customer-domain.test` જેવા addresses ને full mailbox બનાવ્યા વગર બીજા inbox પર mail redirect કરવા જરૂરી હોય ત્યારે forwarders વાપરો.

Forwarders offer કરતા પહેલાં, confirm કરો કે CyberPanel integration configured અને reachable છે. Required host, username, password, અને package constants માટે [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) guide જુઓ.

## વારંવાર પૂછાતા પ્રશ્નો {#frequently-asked-questions}

### શું ગ્રાહકો પોતાના ઇમેઇલ Accounts બનાવી શકે છે? {#can-customers-create-their-own-email-accounts}

હા. જ્યારે ગ્રાહક પાસે domain માટે active ઇમેઇલ membership હોય અને product વધારાના Accountsની મંજૂરી આપે, ત્યારે ગ્રાહકો Account પેજ પરથી પોતાના mailboxes બનાવી અને manage કરી શકે છે.

### શું દરેક provider webmail SSO સપોર્ટ કરે છે? {#does-every-provider-support-webmail-sso}

ના. SSO અને magic-link સપોર્ટ active ઇમેઇલ provider પર આધારિત છે. access-link API વગરના providers હજુ પણ mailbox બનાવટ અને password ફેરફારો સપોર્ટ કરી શકે છે.

### શું administrators ગ્રાહકો માટે ઇમેઇલ Accounts બનાવી શકે છે? {#can-administrators-create-email-accounts-for-customers}

હા. Administrators admin interfaceમાંથી ઇમેઇલ Accounts બનાવી શકે છે, જેમાં Account બનાવતી વખતે initial password સેટ કરવાનો સમાવેશ થાય છે. ગ્રાહક self-service management માટે, ખાતરી કરો કે ગ્રાહક પાસે domain માટે active ઇમેઇલ membership પણ છે.

### શું હું મફતમાં ઇમેઇલ Account આપી શકું? {#can-i-give-an-email-account-away-for-free}

હા. મફત/શૂન્ય-કિંમતવાળો Email product બનાવો અને તેને ગ્રાહક માટે active membership તરીકે ઉમેરો, અથવા ગ્રાહકને 100% discount સાથે checkout પૂર્ણ કરવા દો. ઇમેઇલ membership અસ્તિત્વમાં આવ્યા પછી, ગ્રાહક અથવા administrator actual mailbox બનાવી શકે છે.

### ઇમેઇલ Account સોંપતી વખતે order number કેમ પૂછાય છે? {#why-does-assigning-an-email-account-ask-for-an-order-number}

Membership field તેના reference/order number દ્વારા અસ્તિત્વમાં રહેલી Ultimate Multisite membership શોધે છે. જો ગ્રાહકે ઇમેઇલ product ખરીદ્યો નથી અને તમે હાથથી ઇમેઇલ membership બનાવી નથી, તો સોંપવા માટે કોઈ ઇમેઇલ membership/order નથી. પહેલાં ઇમેઇલ membership બનાવો, પછી mailbox બનાવો અથવા સોંપો.

### શું forwarders mailboxes જેવા જ છે? {#are-forwarders-the-same-as-mailboxes}

ના. mailbox mail સંગ્રહે છે અને તેમાં login credentials હોય છે. forwarder mail ને બીજા address પર redirect કરે છે અને અલગ inbox આપતું નથી.
