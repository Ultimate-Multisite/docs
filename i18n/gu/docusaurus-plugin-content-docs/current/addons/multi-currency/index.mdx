# Multi-Currency {#multi-currency}

Ultimate Multisite: Multi-Currency તમારા નેટવર્કને બહુવિધ ચલણોમાં પેમેન્ટ સ્વીકારવા સક્ષમ બનાવે છે. તે લાઇવ અથવા મેન્યુઅલ એક્સચેન્જ રેટ્સનો ઉપયોગ કરીને આપમેળે કિંમતોમાં રૂપાંતર કરે છે, તમને કોઈપણ ચલણ માટે ઉત્પાદન દીઠ ચોક્કસ કિંમતો સેટ કરવાની મંજૂરી આપે છે, અને ભૌગોલિક સ્થાન (geolocation) દ્વારા દરેક વિઝિટરનું પસંદ કરેલ ચલણ શોધી કાઢે છે — આ બધું એક જ એડમિન ડેશબોર્ડમાંથી સંચાલિત થાય છે.

## મુખ્ય વિશેષતાઓ (Key Features) {#key-features}

- **ઓટોમેટિક એક્સચેન્જ રેટ કન્વર્ઝન** — તમારી મુખ્ય ચલણ (base currency) માંથી કિંમતોને તમારા પસંદ કરેલા પ્રદાતા (provider) પાસેથી મેળવેલા રેટ્સનો ઉપયોગ કરીને રૂપાંતરિત કરવામાં આવે છે.
- **ઉત્પાદન દીઠ કિંમત ઓવરરાઇડ્સ** — ઓટોમેટિક કન્વર્ઝનને બાયપાસ કરવા માટે કોઈપણ ઉત્પાદન પર ચલણ-વિશિષ્ટ ચોક્કસ કિંમતો સેટ કરો.
- **કરન્સી સિલેક્ટર સાઇનઅપ ફીલ્ડ** — ગ્રાહકોને ચેકઆઉટ/સાઇનઅપ દરમિયાન તેમનું પસંદ કરેલ ચલણ પસંદ કરવાની મંજૂરી આપે છે.
- **બહુવિધ એક્સચેન્જ રેટ પ્રદાતાઓ** — ECB, Frankfurter, Stripe, અથવા સંપૂર્ણપણે મેન્યુઅલ રેટ્સ.
- **ભૌગોલિક સ્થાન આધારિત ચલણ શોધ** — આપમેળે તે ચલણની પૂર્વ-સિલેક્શન કરે છે જે વિઝિટરના દેશ સાથે મેળ ખાય છે.
- **એડમિન ડેશબોર્ડ** — એક જ જગ્યાએ સક્રિય ચલણો, એક્સચેન્જ રેટ્સ અને પ્રદાતા સેટિંગ્સનું સંચાલન કરો.

## જરૂરિયાતો (Requirements) {#requirements}

- Ultimate Multisite 2.0.0 અથવા તેથી વધુ
- WordPress Multisite ઇન્સ્ટોલેશન
- PHP 7.4 અથવા તેથી વધુ

## ઇન્સ્ટોલેશન (Installation) {#installation}

1. પ્લગઇન ફાઇલોને `/wp-content/plugins/ultimate-multisite-multi-currency` પર અપલોડ કરો.
2. WordPress માં **Plugins** સ્ક્રીન દ્વારા પ્લગઇન એક્ટિવેટ કરો (Network Activate).
3. એડમિન ડેશબોર્ડ ખોલવા માટે **Ultimate Multisite → Multi-Currency** પર જાઓ.
4. જે ચલણો તમે સ્વીકારવા માંગો છો તે ઉમેરો અને તમારા એક્સચેન્જ રેટ પ્રદાતાને કન્ફિગર કરો.

## કન્ફિગરેશન (Configuration) {#configuration}

### એડમિન ડેશબોર્ડ (Admin Dashboard) {#admin-dashboard}

Multi-Currency એડમિન ડેશબોર્ડ (**Ultimate Multisite → Multi-Currency**) એ નીચેની બાબતો માટેનું કેન્દ્રિય સ્થળ છે:

- ચલણો ઉમેરવા, સક્ષમ કરવા અથવા અક્ષમ કરવા.
- એક્સચેન્જ રેટ પ્રદાતા પસંદ કરવો અને જરૂર પડ્યે રેટ્સને રિફ્રેશ કરવા.
- તમામ સક્રિય ચલણો માટે વર્તમાન રેટ્સની સમીક્ષા કરવી.
- જ્યારે ઓટોમેટિક પ્રદાતાઓ યોગ્ય ન હોય ત્યારે મેન્યુઅલ રેટ્સ સેટ કરવા.

### એક્સચેન્જ રેટ પ્રદાતાઓ (Exchange Rate Providers) {#exchange-rate-providers}

| પ્રદાતા (Provider) | વર્ણન (Description) |
|---|---|
| **ECB** | યુરોપિયન સેન્ટ્રલ બેંક — EUR-આધારિત નેટવર્ક્સ માટે મફત દૈનિક રેટ્સ. |
| **Frankfurter** | ઓપન-સોર્સ ECB મિરર API — મફત, કોઈ API કીની જરૂર નથી. |
| **Stripe** | Stripe ના લાઇવ FX રેટ્સનો ઉપયોગ કરે છે — સક્રિય Stripe ઇન્ટિગ્રેશન જરૂરી છે. |
| **Manual** | તમે સીધા રેટ્સ દાખલ કરો છો; કોઈ બાહ્ય API કોલ નથી. |

**Ultimate Multisite → Multi-Currency → Settings → Exchange Rate Provider** હેઠળ તમારા પ્રદાતાનો ઉપયોગ કરો. રેટ્સને ગમે ત્યારે મેન્યુઅલી રિફ્રેશ કરી શકાય છે અથવા આપમેળે અપડેટ થવા માટે શેડ્યૂલ કરી શકાય છે.

### ઉત્પાદન દીઠ કિંમત ઓવરરાઇડ્સ (Per-Product Pricing Overrides) {#per-product-pricing-overrides}

જ્યારે ઓટોમેટિક કન્વર્ઝન પૂરતું ચોક્કસ ન હોય, ત્યારે તમે કોઈપણ ઉત્પાદન પર ચોક્કસ ચલણ માટે ચોક્કસ કિંમત સેટ કરી શકો છો:

1. ઉત્પાદન એડિટર ખોલો (**Ultimate Multisite → Products → Edit**).
2. **Currency Pricing** વિભાગ સુધી સ્ક્રોલ કરો.
3. દરેક ચલણ માટે ચોક્કસ કિંમત દાખલ કરો જેને તમે ઓવરરાઇડ કરવા માંગો છો.
4. ઉત્પાદન સેવ કરો.

ઓવરરાઇડ વિનાના ઉત્પાદનો તમારી મુખ્ય ચલણમાંથી રૂપાંતરિત કિંમતનો ઉપયોગ કરે છે.

### કરન્સી સિલેક્ટર સાઇનઅપ ફીલ્ડ (Currency Selector Signup Field) {#currency-selector-signup-field}

ગ્રાહકોને રજીસ્ટ્રેશન દરમિયાન તેમનું પસંદ કરેલ ચલણ પસંદ કરવા માટે કરન્સી સિલેક્ટર ફીલ્ડને તમારા સાઇનઅપ/ચેકઆઉટ ફ્લોમાં ઉમેરી શકાય છે:

1. **Ultimate Multisite → Multi-Currency → Settings** પર જાઓ.
2. **Show currency selector on signup** ને સક્ષમ કરો.
3. વૈકલ્પિક રીતે ડિફોલ્ટ ચલણ સેટ કરો અથવા ભૌગોલિક સ્થાનને પૂર્વ-સિલેક્ટેડ મૂલ્ય નક્કી કરવા દો.

પસંદ કરેલ ચલણ સબ્સ્ક્રિપ્શન સાથે સ્ટોર કરવામાં આવે છે અને તમામ અનુગામી બિલિંગ માટે વપરાય છે.

### ભૌગોલિક સ્થાન આધારિત ચલણ શોધ (Geolocation-Based Currency Detection) {#geolocation-based-currency-detection}

જ્યારે ભૌગોલિક સ્થાન શોધ સક્ષમ હોય છે, ત્યારે એડઓન વિઝિટરના દેશને ચલણ સાથે મેપ કરે છે અને તેને કરન્સી સિલેક્ટર ફીલ્ડમાં પૂર્વ-સિલેક્ટ કરે છે. તેને સક્ષમ કરવા માટે:

1. **Ultimate Multisite → Multi-Currency → Settings** પર જાઓ.
2. **Detect currency from visitor location** ને સક્ષમ કરો.
3. **Geolocation Mapping** ટેબલમાં દેશોને ચલણો સાથે મેપ કરો.

જો વિઝિટરના દેશ માટે કોઈ મેપિંગ ન હોય, તો નેટવર્કનું મુખ્ય ચલણ (base currency) ફૉલબેક તરીકે વપરાય છે.

## વારંવાર પૂછાતા પ્રશ્નો (Frequently Asked Questions) {#frequently-asked-questions}

### જો એક્સચેન્જ રેટ પ્રદાતા ઉપલબ્ધ ન હોય તો શું થાય છે? {#what-happens-if-an-exchange-rate-provider-is-unavailable}

એડઓન છેલ્લા સફળતાપૂર્વક મેળવેલા રેટ્સ પર પાછું ફરે છે. જો ક્યારેય કોઈ રેટ મેળવવામાં ન આવ્યો હોય, તો તે તમારા દ્વારા કન્ફિગર કરેલા મેન્યુઅલ રેટ્સ પર પાછું ફરે છે. સલામતી નેટ તરીકે મેન્યુઅલ રેટ્સ કન્ફિગર કરવાની ભલામણ કરવામાં આવે છે.

### શું હું વિવિધ ચલણો માટે વિવિધ પ્રદાતાઓનો ઉપયોગ કરી શકું છું? {#can-i-use-different-providers-for-different-currencies}

ના — તમામ ચલણો માટે એક પ્રદાતા એક સમયે સક્રિય હોય છે. જો તમને વ્યક્તિગત રેટ્સ પર સંપૂર્ણ નિયંત્રણ જોઈતું હોય તો **Manual** નો ઉપયોગ કરો.

### શું ભૌગોલિક સ્થાન માટે અલગ પ્લગઇન જરૂરી છે? {#does-geolocation-require-a-separate-plugin}

કોઈ બાહ્ય ભૌગોલિક સ્થાન પ્લગઇન જરૂરી નથી. એડઓન WordPress ના બિલ્ટ-ઇન IP-થી-દેશ શોધનો ઉપયોગ કરે છે. વધુ ચોકસાઈ માટે, પ્રદાન કરેલ ફિલ્ટર હૂક્સ દ્વારા સમર્પિત ભૌગોલિક સ્થાન સેવાને ઇન્ટિગ્રેટ કરી શકાય છે.

### શું ઉત્પાદન દીઠ ઓવરરાઇડ્સ ટેક્સ પહેલા કે પછી લાગુ થાય છે? {#are-per-product-overrides-applied-before-or-after-tax}

ઉત્પાદન દીઠ ઓવરરાઇડ્સ લક્ષ્ય ચલણમાં ટેક્સ પહેલાની કિંમત સેટ કરે છે. તમારા હાલના ટેક્સ કન્ફિગરેશનનો ઉપયોગ કરીને ઓવરરાઇડ કિંમત પર ટેક્સની ગણતરી કરવામાં આવે છે.
