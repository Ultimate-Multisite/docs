# ડોમેન પ્રોડક્ટ્સ અને પ્રાઇસિંગ {#domain-products-and-pricing}

ડોમેન પ્રોડક્ટ્સ એ એક એવો રસ્તો છે જેના દ્વારા તમે પ્રાઇસિંગ, TLDs, WHOIS પ્રાઇવસી અને પ્રોવાઇડરની પસંદગીને નિયંત્રિત કરો છો. દરેક ડોમેન પ્રોડક્ટ એક સ્ટાન્ડર્ડ Multisite Ultimate પ્રોડક્ટ છે જેમાં એક વધારાની **Domain Settings** ટેબ હોય છે.

## ડોમેન પ્રોડક્ટ બનાવવી {#creating-a-domain-product}

1. **Network Admin › Ultimate Multisite › Products** પર જાઓ
2. **Add New** પર ક્લિક કરો
3. પ્રોડક્ટનો પ્રકાર **Domain** સેટ કરો
4. **Domain Settings** ટેબને configure કરો (નીચે જુઓ)
5. સેવ કરો

ડોમેન પ્રોડક્ટ્સ પ્રોડક્ટ લિસ્ટમાં વાજાપા રંગના **Domain** બેજ સાથે દેખાય છે અને તમે **Domain Products** ટેબનો ઉપયોગ કરીને તેને ફિલ્ટર કરી શકો છો.

## ડોમેન સેટિંગ્સ ટેબ {#domain-settings-tab}

### Provider {#provider}

તમે કયા રજિસ્ટ્રાર (registrar) આ પ્રોડક્ટ માટે રજીસ્ટ્રેશન સંભાળે છે તે પસંદ કરો. તે ડિફોલ્ટ રૂપે Domain Seller settings માં સેટ કરેલ ગ્લોબલ **Default provider** પર રહે છે.

### Supported TLDs {#supported-tlds}

જો તમે ખાલી છોડી દો, તો એક **catch-all product** બનશે જે અન્ય કોઈ પ્રોડક્ટ દ્વારા મેચ ન થતા તમામ TLDs પર લાગુ થશે.

એક કોમા-થી અલગ કરેલી TLDs ની લિસ્ટ (દા.ત., `.com, .net, .org`) દાખલ કરો જેથી એક **TLD-specific product** બને જે ફક્ત તે એક્સટેન્શન પર જ લાગુ પડે.

**પ્રોડક્ટ મેચિંગ કેવી રીતે કામ કરે છે:** જ્યારે કોઈ ગ્રાહક ડોમેન સર્ચ કરે છે, ત્યારે એડ-ઓન સૌથી ચોક્કસ મેચિંગ પ્રોડક્ટ પસંદ કરે છે. જે પ્રોડક્ટની TLD લિસ્ટમાં `.com` હોય, તેને catch-all પ્રોડક્ટ કરતાં વધુ પ્રાધાન્ય (precedence) મળે છે. જો કોઈ TLD-specific પ્રોડક્ટ મેચ ન થાય, તો catch-all નો ઉપયોગ થાય છે. જો કોઈ પ્રોડક્ટ ન હોય, તો ડોમેન સર્ચ બતાવવામાં આવતો નથી.

### Markup type {#markup-type}

ત્રણ મોડ્સ નિયંત્રિત કરે છે કે તમારી રિટેલ કિંમત (retail price) હોલસેલ કિંમત (wholesale cost) માંથી કેવી રીતે ગણવામાં આવશે:

| Mode | તે કેવી રીતે કામ કરે છે |
|---|---|
| **Percentage** | હોલસેલ કિંમત પર ટકાવારી ઉમેરે છે. $10 ના હોલસેલ ડોમેન પર 20% માર્કઅપ $12 આપે છે. |
| **Fixed markup** | એક નિશ્ચિત ડોલર રકમ ઉમેરે છે. $10 ના ડોમેન પર $5 નો માર્કઅપ $15 આપે છે. |
| **Fixed price** | હોલસેલ કિંમતને સંપૂર્ણપણે અવગણે છે. હંમેશા તે રકમ લે છે જે તમે દાખલ કરો છો. |

### Introductory pricing {#introductory-pricing}

ડિસ્કાઉન્ટેડ પ્રથમ વર્ષની કિંમત ઓફર કરવા માટે તેને enable કરો. નિયમિત **Renewal price** (વર્ષ 2+) ની સાથે એક અલગ **Introductory price** (વર્ષ-1 ની કિંમત) સેટ કરો. ગ્રાહકને બંને કિંમતો ચેકઆઉટ દરમિયાન દેખાય છે જેથી તેઓ રિન્યુઅલ વખતે શું અપેક્ષા રાખવી તે જાણી શકે.

### WHOIS privacy {#whois-privacy}

આ પ્રોડક્ટ દ્વારા રજીસ્ટર કરાયેલા ડોમેન માટે WHOIS પ્રાઇવસી પ્રોટેક્શન ઓફર કરવામાં આવે છે કે નહીં તે નિયંત્રિત કરે છે.

| Setting | Behaviour |
|---|---|
| **Disabled** | WHOIS પ્રાઇવસી ક્યારેય ઓફર કે enable થતી નથી. |
| **Always Included** | WHOIS પ્રાઇવસી રજીસ્ટ્રેશન સમયે આપમેળે અને કોઈ ચાર્જ વગર enable થાય છે. |
| **Customer Choice** | ચેકબોક્સ ચેકઆઉટ દરમિયાન દેખાય છે. **Privacy price** સેટ કરો જેથી તે પ્રતિ વર્ષ ચાર્જ થાય, અથવા તેને $0 પર છોડી દો જેથી તે મફત ઓફર થાય. |

Namecheap માટે, WHOIS પ્રાઇવસી WhoisGuard (હંમેશા મફત) નો ઉપયોગ કરે છે. OpenSRS માટે, તે OpenSRS privacy service (હોલસેલ પર ખર્ચ થઈ શકે છે) નો ઉપયોગ કરે છે.

## TLD import અને sync {#tld-import-and-sync}

ડોમેન પ્રોડક્ટ્સ કનેક્ટેડ પ્રોવાઇડર પાસેથી મેળવેલ રીઅલ-ટાઇમ હોલસેલ પ્રાઇસિંગ દર્શાવે છે. આ કામ કરવા માટે, TLDs import થયેલા હોવા જરૂરી છે.

- **Manual sync:** Settings › Domain Seller › Sync TLDs (પ્રાપડર મુજબ)
- **Automatic sync:** બધા configure કરેલ પ્રોવાઇડર્સ પર નિયમિત ક્રોન જોબ દ્વારા દરરોજ ચાલે છે

sync કર્યા પછી, કોઈપણ ડોમેન પ્રોડક્ટના Domain Settings ટેબ પર જાઓ અને ઉપલબ્ધ TLDs અને તેમની વર્તમાન હોલસેલ કિંમતો જોવા માટે TLD picker નો ઉપયોગ કરો.

## Auto-renewal {#auto-renewal}

ડોમેન રિન્યુઅલ ગ્રાહકના મેમ્બરશિપ સ્ટેટસ સાથે જોડાયેલું છે:

- જ્યારે મેમ્બરશિપ રિન્યુઅલ થાય અને ડોમેન લિંક હોય, ત્યારે ડોમેન રિન્યુઅલ આપમેળે ક્યુ (queued) થઈ જાય છે
- રિન્યુઅલ પ્રયાસો ગ્રાહકના active payment gateway નો ઉપયોગ કરે છે
- નિષ્ફળ રિન્યુઅલ આપમેળે એક્સપોનેન્શિયલ બેકઓફ સાથે ફરીથી પ્રયાસ કરવામાં આવે છે
- સફળ રિન્યુઅલ, નિષ્ફળતા અને આવનારા સમાપ્તિઓ માટે ઈમેલ નોટિફિકેશન મોકલવામાં આવે છે

ડોમેન જીવનચક્રની ઘટનાઓ માટે ઈમેલ ટેમ્પલેટ ID:

| Event | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## Admin: Manual domain registration {#admin-manual-domain-registration}

ગ્રાહકને ચેકઆઉટમાંથી પસાર થવા આપ્યા વિના તેમની વતી ડોમેન રજીસ્ટર કરવા માટે:

1. **Network Admin › Ultimate Multisite › Register Domain** પર જાઓ
2. ગ્રાહક પસંદ કરો અને ડોમેનનું નામ દાખલ કરો
3. રજીસ્ટ્રન્ટના સંપર્ક વિગતો (નામ, સરનામું, ફોન) ભરો
4. **Register** પર ક્લિક કરો

ડોમેન રેકોર્ડ બનાવવામાં આવે છે અને ગ્રાહકના એકાઉન્ટ સાથે લિંક થઈ જાય છે.
