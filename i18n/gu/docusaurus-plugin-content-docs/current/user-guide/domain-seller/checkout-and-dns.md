# ચેકઆઉટ ક્ષેત્ર અને ગ્રાહક DNS વ્યવસ્થાપન

## Domain Selection ચેકઆઉટ ક્ષેત્ર {#the-domain-selection-checkout-field}

**Domain Selection** ક્ષેત્ર એ ચેકઆઉટ ઘટક છે જે ગ્રાહકોને તેમની સાઇટનો ડોમેન કેવી રીતે મેળવવો તેની પસંદગી આપે છે. ડોમેન વેચાણ સક્ષમ કરવા માટે તેને કોઈપણ ચેકઆઉટ ફોર્મમાં ઉમેરો.

### ચેકઆઉટ ફોર્મમાં ક્ષેત્ર ઉમેરવું {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › Checkout Forms** પર જાઓ
2. ચેકઆઉટ ફોર્મ ખોલો અથવા બનાવો
3. ચેકઆઉટ સંપાદકમાં, **Add Field** પર ક્લિક કરો
4. ક્ષેત્ર સૂચિમાંથી **Domain Selection** પસંદ કરો
5. ક્ષેત્ર વિકલ્પો કૉન્ફિગર કરો (નીચે જુઓ)
6. ફોર્મ સાચવો

### ક્ષેત્ર વિકલ્પો {#field-options}

**ડોમેન મોડ્સ** — ગ્રાહકને કયા ટૅબ્સ દેખાય તે પસંદ કરો. દરેક મોડને સ્વતંત્ર રીતે સક્ષમ અથવા અક્ષમ કરી શકાય છે:

| મોડ | તે શું કરે છે |
|---|---|
| **સબડોમેન** | ગ્રાહક તમારા નેટવર્ક પર મફત સબડોમેનનો ઉપયોગ કરે છે (દા.ત., `mysite.yournetwork.com`). ચુકવણી જરૂરી નથી. |
| **નવો ડોમેન નોંધાવો** | ગ્રાહક નવો ડોમેન શોધે છે અને તમારા કૉન્ફિગર કરેલા પ્રદાતા દ્વારા તેને નોંધાવે છે. કિંમત માટે મેળ ખાતા ડોમેન ઉત્પાદનનો ઉપયોગ કરે છે. |
| **હાલનો ડોમેન** | ગ્રાહક પોતાનો પહેલેથી માલિકીનો ડોમેન મૅપ કરે છે. નોંધણી ફી નથી. ડોમેન આપમેળે તેમની સાઇટ સાથે મૅપ થાય છે. |

**ડિફૉલ્ટ મોડ** — જ્યારે ત્રણેય મોડ સક્ષમ હોય, ત્યારે કયો ટૅબ પહેલા ખૂલશે. ડોમેન નોંધણી વૈકલ્પિક રાખવા માટે **સબડોમેન** પર સેટ કરો, અથવા ખરીદીઓને પ્રોત્સાહિત કરવા માટે **નવો ડોમેન નોંધાવો** પર સેટ કરો.

**ડોમેન ઉત્પાદન** — વૈકલ્પિક રીતે આ ક્ષેત્રને ચોક્કસ ડોમેન ઉત્પાદન સાથે પિન કરો. જો સેટ ન હોય, તો addon ગ્રાહક જે TLD શોધે છે તેના આધારે મેળ ખાતું ઉત્પાદન આપમેળે પસંદ કરે છે.

### નોંધણીકર્તા સંપર્ક ક્ષેત્રો {#registrant-contact-fields}

જ્યારે ગ્રાહક **નવો ડોમેન નોંધાવો** ટૅબ પસંદ કરે છે, ત્યારે ચેકઆઉટ ફોર્મ નોંધણીકર્તા સંપર્ક ક્ષેત્રો ઇનલાઇન ઉમેરે છે:

- પ્રથમ નામ / છેલ્લું નામ
- ઇમેઇલ સરનામું
- સરનામું (લાઇન 1, શહેર, રાજ્ય/પ્રાંત, પોસ્ટલ કોડ, દેશ)
- ફોન નંબર

આ બધા રજિસ્ટ્રાર દ્વારા જરૂરી છે અને નોંધણી API કૉલ કરવામાં આવે તે પહેલાં માન્ય કરવામાં આવે છે. ફોન નંબરો રજિસ્ટ્રાર અપેક્ષિત `+CC.NNN` આંતરરાષ્ટ્રીય ફોર્મેટમાં આપમેળે ફોર્મેટ થાય છે.

### આપમેળે જનરેટ થયેલ સાઇટ URL {#auto-generated-site-url}

જ્યારે ગ્રાહક ડોમેન નોંધાવે અથવા મૅપ કરે છે, ત્યારે સાઇટ URL ક્ષેત્ર પસંદ કરેલા ડોમેનમાંથી આપમેળે ભરાય છે. ગ્રાહકોને અલગ URL ક્ષેત્ર ભરવાની જરૂર નથી.

### શોધ વર્તન {#search-behaviour}

- ગ્રાહક ટાઇપ કરે તેમ AJAX સાથે રિયલ-ટાઇમમાં ડોમેન ઉપલબ્ધતા ચકાસવામાં આવે છે
- પસંદ કરેલો ડોમેન ઉપલબ્ધ ન હોય ત્યારે વૈકલ્પિક TLD સૂચનો બતાવવામાં આવે છે
- કિંમત લાઇવ મેળવવામાં આવે છે અને સ્પષ્ટ રીતે દર્શાવવામાં આવે છે (નોંધણી કિંમત, નવીનીકરણ કિંમત, વૈકલ્પિક WHOIS ગોપનીયતા ફી)
- કૂપન કોડ ડોમેન ઉત્પાદનો પર અન્ય કોઈપણ ઉત્પાદનની જેમ જ લાગુ પડે છે

**શોધ પ્રતિસાદક્ષમતા ટ્યુન કરવી:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ડોમેન શોધ ફોર્મમાં કસ્ટમ ક્ષેત્રો ઉમેરવા:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ગ્રાહક DNS વ્યવસ્થાપન {#customer-dns-management}

ગ્રાહકો **My Account** પેજમાંથી, તેમના ડોમેનની એન્ટ્રી હેઠળ, તેમના નોંધાયેલા ડોમેન્સ માટે DNS રેકોર્ડ્સનું વ્યવસ્થાપન કરી શકે છે.

### સમર્થિત રેકોર્ડ પ્રકારો {#supported-record-types}

| પ્રકાર | ઉપયોગ |
|---|---|
| **A** | હોસ્ટનામને IPv4 સરનામા સાથે મૅપ કરો |
| **AAAA** | હોસ્ટનામને IPv6 સરનામા સાથે મૅપ કરો |
| **CNAME** | બીજા હોસ્ટનામ તરફ નિર્દેશ કરતું ઉપનામ બનાવો |
| **MX** | મેઇલ એક્સચેન્જ સર્વર સેટ કરો |
| **TXT** | SPF, DMARC, ચકાસણી, અથવા અન્ય ટેક્સ્ટ રેકોર્ડ્સ ઉમેરો |

### કયા પ્રદાતાઓ DNS વ્યવસ્થાપનને સમર્થન આપે છે? {#which-providers-support-dns-management}

DNS વ્યવસ્થાપન (રેકોર્ડ્સ ઉમેરવા, સંપાદિત કરવા, કાઢી નાખવા) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, અને **Openprovider** સાથે ઉપલબ્ધ છે. **Hostinger** ડોમેન્સ Domain Seller દ્વારા નેમસર્વર્સ અપડેટ કરી શકે છે; હોસ્ટેડ ડોમેન્સ માટે DNS રેકોર્ડ્સ મુખ્ય Hostinger ડોમેન-મૅપિંગ ઇન્ટિગ્રેશન દ્વારા સંચાલિત થાય છે. Namecheap, GoDaddy, અને NameSilo ડોમેન્સ સ્થિતિ અને સમાપ્તિ માહિતી દર્શાવે છે પરંતુ DNS સીધું રજિસ્ટ્રારના કન્ટ્રોલ પેનલમાં સંચાલિત કરવું આવશ્યક છે.

### ડિફૉલ્ટ DNS રેકોર્ડ્સ {#default-dns-records}

તમે ડિફૉલ્ટ DNS રેકોર્ડ્સ કૉન્ફિગર કરી શકો છો જે ડોમેન નોંધાય ત્યારે આપમેળે લાગુ થાય છે. **Settings › Domain Seller › Default DNS Records** પર જાઓ.

ડિફૉલ્ટ રેકોર્ડ મૂલ્યો બે ટોકન્સને સમર્થન આપે છે:

| ટોકન | આથી બદલાય છે |
|---|---|
| `{DOMAIN}` | નોંધાયેલ ડોમેન નામ (દા.ત., `example.com`) |
| `{SITE_URL}` | ગ્રાહકની સાઇટ માટે WordPress સાઇટ URL |

**ઉદાહરણ — એપેક્સ ડોમેન અને www ને તમારા સર્વર IP તરફ નિર્દેશ કરો:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### એડમિન: DNS જોવું અને સંપાદિત કરવું {#admin-viewing-and-editing-dns}

નેટવર્ક એડમિન્સ **Network Admin › Ultimate Multisite › Domains** માં ડોમેનના સંપાદન પેજ પરથી કોઈપણ ગ્રાહક ડોમેન માટે DNS રેકોર્ડ્સ જોઈ અને સંપાદિત કરી શકે છે.
