# ચેકઆઉટ ફીલ્ડ અને ગ્રાહક DNS સંચાલન

## ડોમેન સિલેક્શન ચેકઆઉટ ફીલ્ડ

**Domain Selection** ફીલ્ડ એક ચેકઆઉટ એલિમેન્ટ છે જે ગ્રાહકોને તેમના સાઇટના ડોમેન કેવી રીતે મેળવશે, તે પસંદગી આપે છે. ડોમેન વેચવાની સુવિધા આપવા માટે તેને કોઈપણ ચેકઆઉટ ફોર્મમાં ઉમેરો.

### ચેકઆઉટ ફોર્મમાં ફીલ્ડ ઉમેરવું

1. **Network Admin › Ultimate Multisite › Checkout Forms** પર જાઓ
2. એક ચેકઆઉટ ફોર્મ ખોલો અથવા બનાવો
3. ચેકઆઉટ એડિટરમાં, **Add Field** પર ક્લિક કરો
4. ફીલ્ડ લિસ્ટમાંથી **Domain Selection** પસંદ કરો
5. ફીલ્ડ વિકલ્પો ગોઠવો (નીચે જુઓ)
6. ફોર્મ સેવ કરો

### ફીલ્ડ વિકલ્પો

**Domain modes** — ગ્રાહક કયા ટેબ જુએ છે તે પસંદ કરો. દરેક મોડને સ્વતંત્ર રીતે enable અથવા disable કરી શકાય છે:

| Mode | શું કરે છે |
|---|---|
| **Subdomain** | ગ્રાહક તમારા નેટવર્ક પરના મફત સબડોમેનનો ઉપયોગ કરે છે (દા.ત., `mysite.yournetwork.com`). કોઈ પેમેન્ટની જરૂર નથી. |
| **Register New Domain** | ગ્રાહક નવું ડોમેન સર્ચ કરે છે અને તમારા કન્ફિગર કરેલા પ્રદાતા (provider) દ્વારા તેનું રજીસ્ટ્રેશન કરાવે છે. કિંમત માટે મેચિંગ ડોમેન પ્રોડક્ટનો ઉપયોગ થાય છે. |
| **Existing Domain** | ગ્રાહક જે ડોમેન પહેલેથી જ માલિકીનું છે, તેને મેપ કરે છે. કોઈ રજીસ્ટ્રેશન ફી નથી. ડોમેન આપમેળે તેમની સાઇટ સાથે મેપ થઈ જાય છે. |

**Default mode** — જ્યારે ત્રણેય મોડ્સ enable હોય, ત્યારે કયું ટેબ પહેલા ખુલશે. ડોમેન રજીસ્ટ્રેશનને વૈકલ્પિક રાખવા માટે તેને **Subdomain** પર સેટ કરો, અથવા ખરીદીને પ્રોત્સાહિત કરવા માટે **Register New Domain** પર સેટ કરો.

**Domain product** — વૈકલ્પિક રીતે આ ફીલ્ડને ચોક્કસ ડોમેન પ્રોડક્ટ સાથે પિન કરો. જો સેટ ન કરવામાં આવે, તો એડઓન આપમેળે TLD ના આધારે મેચિંગ પ્રોડક્ટ પસંદ કરે છે જે ગ્રાહક સર્ચ કરે છે.

### Registrant contact fields

જ્યારે ગ્રાહક **Register New Domain** ટેબ પસંદ કરે છે, ત્યારે ચેકઆઉટ ફોર્મમાં ઇનલાઇન રજીસ્ટ્રન્ટ કોન્ટેક્ટ ફીલ્ડ્સ ઉમેરાય છે:

- First name / Last name
- Email address
- Address (line 1, city, state/province, postal code, country)
- Phone number

આ બધા રજિસ્ટ્રાર દ્વારા જરૂરી છે અને રજીસ્ટ્રેશન API કોલ કરવામાં આવે તે પહેલાં તેની માન્યતા (validated) કરવામાં આવે છે. ફોન નંબરોને રજિસ્ટ્રાર દ્વારા અપેક્ષિત `+CC.NNN` આંતરરાષ્ટ્રીય ફોર્મેટમાં આપમેળે ફોર્મેટ કરવામાં આવે છે.

### Auto-generated site URL

જ્યારે ગ્રાહક ડોમેન રજીસ્ટર કરે છે અથવા મેપ કરે છે, ત્યારે સાઇટ URL ફીલ્ડ પસંદ કરેલા ડોમેનમાંથી આપમેળે ભરાઈ જાય છે. ગ્રાહકોએ અલગ URL ફીલ્ડ ભરવાની જરૂર નથી.

### Search behaviour

- ગ્રાહક ટાઇપ કરે છે ત્યારે ડોમેનની ઉપલબ્ધતા AJAX સાથે રીઅલ-ટાઇમમાં તપાસવામાં આવે છે
- જ્યારે પસંદ કરેલ ડોમેન ઉપલબ્ધ ન હોય ત્યારે વૈકલ્પિક TLD સૂચનો બતાવવામાં આવે છે
- કિંમત લાઇવ મેળવવામાં આવે છે અને સ્પષ્ટ રીતે પ્રદર્શિત થાય છે (રજીસ્ટ્રેશન કિંમત, રિન્યુઅલ કિંમત, વૈકલ્પિક WHOIS પ્રાઇવસી ફી)
- કૂપન કોડ્સ અન્ય કોઈપણ પ્રોડક્ટની જેમ ડોમેન પ્રોડક્ટ્સ પર લાગુ પડે છે

**Search responsiveness ટ્યુન કરવી:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ડોમેન સર્ચ ફોર્મમાં કસ્ટમ ફીલ્ડ્સ ઉમેરવા:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## ગ્રાહક DNS સંચાલન

ગ્રાહકો તેમના રજીસ્ટર કરેલા ડોમેન માટે DNS રેકોર્ડ્સનું સંચાલન **My Account** પેજ પર, તેમના ડોમેનની એન્ટ્રી હેઠળ કરી શકે છે.

### સપોર્ટેડ રેકોર્ડ પ્રકારો

| Type | ઉપયોગ |
|---|---|
| **A** | હોસ્ટનેમને IPv4 એડ્રેસ સાથે મેપ કરે છે |
| **AAAA** | હોસ્ટનેમને IPv6 એડ્રેસ સાથે મેપ કરે છે |
| **CNAME** | અન્ય હોસ્ટનેમ તરફ પોઇન્ટ કરતો ઉપનામ (alias) બનાવે છે |
| **MX** | મેઇલ એક્સચેન્જ સર્વર સેટ કરે છે |
| **TXT** | SPF, DMARC, વેરિફિકેશન, અથવા અન્ય ટેક્સ્ટ રેકોર્ડ્સ ઉમેરે છે |

### કયા પ્રદાતાઓ DNS સંચાલનને સપોર્ટ કરે છે?

DNS સંચાલન (રેકોર્ડ્સ ઉમેરવા, સંપાદિત કરવા, કાઢી નાખવા) **OpenSRS**, **ResellerClub**, અને **Enom** સાથે ઉપલબ્ધ છે. Namecheap, GoDaddy, અને NameSilo ડોમેન સ્ટેટસ અને સમાપ્તિની માહિતી દર્શાવે છે, પરંતુ DNS નું સંચાલન રજિસ્ટ્રારના કંટ્રોલ પેનલમાંથી સીધું કરવું આવશ્યક છે.

### Default DNS records

તમે ડિફોલ્ટ DNS રેકોર્ડ્સ કન્ફિગર કરી શકો છો જે ડોમેન રજીસ્ટર થાય ત્યારે આપમેળે લાગુ થઈ જાય છે. **Settings › Domain Seller › Default DNS Records** પર જાઓ.

ડિફોલ્ટ રેકોર્ડ વેલ્યુ બે ટોકન્સને સપોર્ટ કરે છે:

| Token | શાનાથી બદલાશે |
|---|---|
| `{DOMAIN}` | રજીસ્ટર થયેલ ડોમેનનું નામ (દા.ત., `example.com`) |
| `{SITE_URL}` | ગ્રાહકની સાઇટ માટેનું WordPress સાઇટ URL |

**ઉદાહરણ — એપેક્સ ડોમેન અને www ને તમારા સર્વર IP તરફ પોઇન્ટ કરવો:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### Admin: DNS જોવું અને સંપાદિત કરવું

Network admins ને **Network Admin › Ultimate Multisite › Domains** માં ડોમેનના એડિટ પેજ પરથી કોઈપણ ગ્રાહકના ડોમેન માટે DNS રેકોર્ડ્સ જોવા અને સંપાદિત કરવાની મંજૂરી છે.
