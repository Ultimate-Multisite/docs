# હોસ્ટિંગર મલ્ટી-ટેનન્સી

Ultimate Multisite: Multi-Tenancy 1.2.0 માં એક Hostinger મલ્ટી-ટેનન્સી ક્ષમતા ઉમેરવામાં આવી છે જેથી હોસ્ટેડ ડોમેન્સ હાલના Hostinger ડોમેન-મેપિંગ ઇન્ટિગ્રેશન સાથે સાર્વભૌમ ટેนન્ટ પ્રોવિઝનિંગમાં ભાગ લઈ શકે.

જ્યારે તમે ટેนન્ટ ડોમેન્સ અને અલગ ટેนન્ટ ઇન્ફ્રાસ્ટ્રક્ચરને Hostinger hPanel દ્વારા મેનેજ કરો ત્યારે આ ક્ષમતાનો ઉપયોગ કરો.

## સેટઅપ નોંધો {#setup-notes}

1. **Ultimate Multisite > Settings > Host Integrations** હેઠળ કોર Hostinger ઇન્ટિગ્રેશનને કન્ફિગર કરો.
2. ખાતરી કરો કે Hostinger API ટોકન લક્ષ્ય ડોમેન અથવા સબડોમેનનું સંચાલન કરી શકે છે.
3. મલ્ટી-ટેનન્સી એડ-ઓન (addon) ને એનિબલ કરો.
4. ટેนન્ટ પ્રકાશિત કરતા પહેલા ટેนન્ટ ઇસોલેશન વ્યૂહરચનાને કન્ફિગર કરો.
5. ટેนન્ટ પર પ્રોડક્શન ટ્રાફિક મોકલતા પહેલા માઇગ્રેશન વેરિફિકેશન વર્કફ્લો ચલાવો.

Hostinger ક્ષમતા હોસ્ટ-સાઇડ ઓપરેશન્સ માટે શેર કરેલ Hostinger કનેક્શનનો ઉપયોગ કરે છે. DNS હજુ પણ સાચા Hostinger એકાઉન્ટ તરફ નિર્દેશ કરવો જોઈએ, અને hPanel એકાઉન્ટની મર્યાદાઓ હજી પણ લાગુ પડે છે.

## ક્ષમતા-વિશિષ્ટ ફેરફારો {#capability-specific-changes}

- સાર્વભૌમ ટેนન્ટ્સને હોસ્ટ-અવેર ડોમેન ઓપરેશન્સ સાથે બનાવી શકાય છે.
- ગ્રાન્ટ વેરિફિકેશન પહેલાSame-machine database host strings ને સામાન્ય બનાવવામાં આવે છે.
- Hostinger દ્વારા સંચાલિત ટેนન્ટ્સ hPanel માં દર્શાવેલ ડેટાબેઝ હોસ્ટ વેલ્યુનો ઉપયોગ કરવા જોઈએ, સિવાય કે WordPress runtime ને સ્થાનિક ઓવરરાઇડની જરૂર હોય.
- SSO મુલાકાતો ટેนન્ટ ડોમેન Hostinger-હોસ્ટેડ ટેนન્ટ પર રિઝોલ્વ થવા પર આધાર રાખે છે.

## Hostinger ટેนન્ટ્સમાં ટ્રબલશૂટિંગ {#troubleshooting-hostinger-tenants}

- જો ટેนન્ટ ઇન્સ્ટોલેશન નિષ્ફળ જાય, તો ચકાસો કે ડોમેન પહેલેથી જ Hostinger એકાઉન્ટ સાથે જોડાયેલ છે.
- જો ડેટાબેઝ વેરિફિકેશન નિષ્ફળ જાય, તો hPanel સાથે ટેนન્ટ DB યુઝરનેમ, ડેટાબેઝનું નામ અને હોસ્ટ બાઇન્ડિંગની સરખામણી કરો.
- જો **Visit (SSO)** નિષ્ફળ જાય, તો ટેนન્ટ ડોમેન માટે DNS અને SSL સક્રિય છે તેની પુષ્ટિ કરો.
- જો ટેન્ડઆઉનમાં હોસ્ટ સંસાધનો પાછળ રહી જાય, તો બેકઅપની પુષ્ટિ કર્યા પછી hPanel માં બાકી રહેલા કોઈપણ ડેટાબેઝ, યુઝર અથવા ફોલ્ડર્સ દૂર કરો.
