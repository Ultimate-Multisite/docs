# થર્ડ-પાર્ટી મોડ માઇગ્રેશન

Superdav AI Agent v1.12.0 થર્ડ-પાર્ટી એબિલિટીઝ (abilities) ને કેવી રીતે હેન્ડલ કરવામાં આવે છે તેમાં ફેરફાર કરે છે. **થર્ડ-પાર્ટી મોડ હવે આપમેળે (auto) પર ડિફોલ્ટ થાય છે**, જે WordPress 7.0+ પર મેન્યુઅલ કન્ફિગરેશન વિના નેટિવ WordPress Abilities API ઇન્ટિગ્રેશનને સક્ષમ બનાવે છે.

## શું બદલાયું છે? {#what-changed}

### v1.12.0 પહેલાં {#before-v1120}

થર્ડ-પાર્ટી એબિલિટીઝ માટે મેન્યુઅલ કન્ફિગરેશનની જરૂર હતી:

- તમારે સ્પષ્ટપણે "થર્ડ-પાર્ટી મોડ" સક્ષમ કરવો પડતો હતો
- એબિલિટીઝ એક કસ્ટમ રજિસ્ટ્રીમાંથી લોડ થતી હતી
- WordPress Abilities API સાથેનું ઇન્ટિગ્રેશન વૈકલ્પિક હતું
- લેગસી મોડ ડિફોલ્ટ હતો

### v1.12.0 પછી {#after-v1120}

થર્ડ-પાર્ટી એબિલિટીઝ આપોઆપ કામ કરે છે:

- થર્ડ-પાર્ટી મોડ "auto" પર ડિફોલ્ટ થાય છે
- એબિલિટીઝ WordPress Abilities API સાથે નેટિવ રીતે ઇન્ટિગ્રેટ થાય છે
- WordPress 7.0+ પર કોઈ મેન્યુઅલ કન્ફિગરેશનની જરૂર નથી
- જૂના WordPress વર્ઝન માટે લેગસી મોડ હજી પણ ઉપલબ્ધ છે

## કોને અસર થશે? {#who-is-affected}

### નવી ઇન્સ્ટોલેશન્સ (WordPress 7.0+) {#new-installations-wordpress-70}

**કોઈ કાર્યવાહીની જરૂર નથી.** થર્ડ-પાર્ટી મોડ આપમેળે "auto" પર સેટ થઈ જાય છે, અને એબિલિટીઝ બિલકુલ તૈયાર (out of the box) કામ કરે છે.

### હાલની ઇન્સ્ટોલેશન્સ {#existing-installations}

**તમારી સેટિંગ્સ જળવાઈ રહે છે.** જો તમે ઉપયોગ કરી રહ્યા હતા:

- **લેગસી મોડ**: તમે લેગસી મોડમાં જ રહેશો (કોઈ ફેરફાર નહીં)
- **મેન્યુઅલ થર્ડ-પાર્ટી મોડ**: તમે મેન્યુઅલ મોડમાં જ રહેશો (કોઈ ફેરફાર નહીં)
- **ઓટો મોડ**: તમે ઓટો મોડ સાથે ચાલુ રહેશો (કોઈ ફેરફાર નહીં)

### 7.0 પહેલાના WordPress વર્ઝન {#wordpress-versions-before-70}

**લેગસી મોડ હજી પણ ઉપલબ્ધ છે.** જો તમે WordPress 6.x અથવા તેનાથી જૂના વર્ઝન પર છો:

- થર્ડ-પાર્ટી મોડ "legacy" પર ડિફોલ્ટ થાય છે
- જો તમે ઈચ્છો તો તમે થર્ડ-પાર્ટી મોડ મેન્યુઅલી સક્ષમ કરી શકો છો
- નેટિવ Abilities API નો ઉપયોગ કરવા માટે WordPress 7.0+ પર અપગ્રેડ કરો

## મોડ્સ સમજવા {#understanding-the-modes}

### ઓટો મોડ (નવો ડિફોલ્ટ) {#auto-mode-new-default}

**ઓટો મોડ** નેટિવ WordPress Abilities API ઇન્ટિગ્રેશનનો ઉપયોગ કરે છે:

- એબિલિટીઝ WordPress hooks દ્વારા રજિસ્ટર થાય છે
- WordPress 7.0+ Abilities API સાથે સંપૂર્ણ સુસંગતતા
- થર્ડ-પાર્ટી એબિલિટીઝની આપોઆપ શોધ
- કોઈ મેન્યુઅલ કન્ફિગરેશનની જરૂર નથી

**ક્યારે ઉપયોગ કરવો**: થર્ડ-પાર્ટી એબિલિટીઝ સાથે WordPress 7.0+

### મેન્યુઅલ મોડ {#manual-mode}

**મેન્યુઅલ મોડ** માટે સ્પષ્ટ કન્ફિગરેશનની જરૂર છે:

- તમે કઈ થર્ડ-પાર્ટી એબિલિટીઝ લોડ કરવી તે સ્પષ્ટ કરો છો
- પરીક્ષણ (testing) અથવા પસંદગીયુક્ત એબિલિટી લોડિંગ માટે ઉપયોગી
- કન્ફિગરેશન ફાઇલોમાં એડિટ કરવાની જરૂર છે
- વધુ નિયંત્રણ, પરંતુ વધુ સેટઅપ

**ક્યારે ઉપયોગ કરવો**: પરીક્ષણ, પસંદગીયુક્ત એબિલિટી લોડિંગ, અથવા કસ્ટમ કન્ફિગરેશન્સ

### લેગસી મોડ {#legacy-mode}

**લેગસી મોડ** જૂની થર્ડ-પાર્ટી એબિલિટી સિસ્ટમનો ઉપયોગ કરે છે:

- કસ્ટમ એબિલિટી રજિસ્ટ્રી (WordPress Abilities API નહીં)
- જૂના WordPress વર્ઝન સાથે પાછળની સુસંગતતા (Backward compatible)
- કોઈ નેટિવ WordPress ઇન્ટિગ્રેશન નહીં
- Deprecated છે પરંતુ હજી પણ સપોર્ટેડ છે

**ક્યારે ઉપયોગ કરવો**: WordPress 6.x અથવા તેનાથી જૂના, અથવા જ્યારે તમને લેગસી સુસંગતતાની જરૂર હોય

## તમારો વર્તમાન મોડ તપાસવો {#checking-your-current-mode}

### એડમિન પેનલ દ્વારા {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** પર જાઓ
2. **Third-Party Mode** સેટિંગ શોધો
3. તમને તમારો વર્તમાન મોડ અને તેને બદલવાના વિકલ્પો દેખાશે

### કોડ દ્વારા {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', અથવા 'legacy'
```

## તમારો મોડ બદલવો {#changing-your-mode}

### ઓટો મોડ પર સ્વિચ કરો {#switch-to-auto-mode}

જો તમે WordPress 7.0+ પર છો અને ઓટો મોડનો ઉપયોગ કરવા માંગો છો:

1. **Superdav AI Agent** → **Settings** પર જાઓ
2. **Third-Party Mode** શોધો
3. **Auto (WordPress Abilities API)** પસંદ કરો
4. **Save** પર ક્લિક કરો

Superdav AI Agent આપમેળે થર્ડ-પાર્ટી એબિલિટીઝ શોધી અને રજિસ્ટર કરશે.

### મેન્યુઅલ મોડ પર સ્વિચ કરો {#switch-to-manual-mode}

જો તમે કઈ એબિલિટીઝ લોડ કરવી તે મેન્યુઅલી નિયંત્રિત કરવા માંગો છો:

1. **Superdav AI Agent** → **Settings** પર જાઓ
2. **Third-Party Mode** શોધો
3. **Manual** પસંદ કરો
4. **Save** પર ક્લિક કરો
5. કઈ એબિલિટીઝ લોડ કરવી તે સ્પષ્ટ કરવા માટે તમારી કન્ફિગરેશન ફાઇલ એડિટ કરો

### લેગસી મોડ પર સ્વિચ કરો {#switch-to-legacy-mode}

જો તમને લેગસી સુસંગતતાની જરૂર હોય:

1. **Superdav AI Agent** → **Settings** પર જાઓ
2. **Third-Party Mode** શોધો
3. **Legacy** પસંદ કરો
4. **Save** પર ક્લિક કરો

## ઓટો મોડના ફાયદા {#benefits-of-auto-mode}

### આપોઆપ શોધ (Automatic Discovery) {#automatic-discovery}

એબિલિટીઝ આપોઆપ નીચેનામાંથી શોધી લેવામાં આવે છે:

- ઇન્સ્ટોલ કરેલ plugins
- active theme
- must-use plugins
- drop-in plugins

મેન્યુઅલ રજિસ્ટ્રેશનની જરૂર નથી.

### નેટિવ ઇન્ટિગ્રેશન {#native-integration}

એબિલિટીઝ WordPress Abilities API સાથે ઇન્ટિગ્રેટ થાય છે:

- WordPress core સાથે સુસંગત
- WordPress admin સાથે કામ કરે છે
- Abilities API નો ઉપયોગ કરતા અન્ય plugins સાથે સુસંગત
- જેમ જેમ WordPress વિકસિત થાય છે તેમ ભવિષ્ય માટે તૈયાર

### સરળ સંચાલન (Simplified Management) {#simplified-management}

- એડિટ કરવા માટે કોઈ કન્ફિગરેશન ફાઇલો નથી
- કોઈ મેન્યુઅલ એબિલિટી રજિસ્ટ્રેશન નથી
- Ability Visibility કંટ્રોલ્સ આપોઆપ કામ કરે છે
- એડમિન નોટિસેસ તમને વર્ગીકૃત ન થયેલી એબિલિટીઝ વિશે ચેતવણી આપે છે

### સારું પ્રદર્શન (Better Performance) {#better-performance}

- એબિલિટીઝ કેશ થાય છે
- જરૂર પડ્યે લેઝી-લોડ થાય છે
- WordPress 7.0+ માટે ઓપ્ટિમાઇઝ કરેલ

## માઇગ્રેશન પાથ {#migration-path}

### જો તમે WordPress 6.x પર છો {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ પર અપગ્રેડ કરો** (જ્યારે તૈયાર હોવ)
2. **Superdav AI Agent** ને v1.12.0+ પર અપડેટ કરો
3. **થર્ડ-પાર્ટી મોડને Auto પર બદલો** (વૈકલ્પિક; લેગસી મોડ હજી પણ કામ કરે છે)
4. યોગ્ય એક્સેસ કંટ્રોલ્સ સુનિશ્ચિત કરવા માટે **એબિલિટી વિઝિબિલિટી**ની સમીક્ષા કરો

### જો તમે WordPress 7.0+ પર છો {#if-youre-on-wordpress-70}

1. **Superdav AI Agent** ને v1.12.0+ પર અપડેટ કરો
2. **તપાસો કે થર્ડ-પાર્ટી મોડ Auto પર સેટ છે** (તે ડિફોલ્ટ હોવો જોઈએ)
3. યોગ્ય એક્સેસ કંટ્રોલ્સ સુનિશ્ચિત કરવા માટે **એબિલિટી વિઝિબિલિટી**ની સમીક્ષા કરો
4. ખાતરી કરવા માટે **થર્ડ-પાર્ટી એબિલિટીઝનું પરીક્ષણ** કરો કે તે કામ કરે છે

## ટ્રબલશૂટિંગ {#troubleshooting}

### ઓટો મોડમાં એબિલિટીઝ લોડ થતી નથી {#abilities-arent-loading-in-auto-mode}

- ચકાસો કે તમે WordPress 7.0+ પર છો
- તપાસો કે થર્ડ-પાર્ટી મોડ "Auto" પર સેટ છે
- ચકાસો કે એબિલિલીટી પ્રદાન કરતું plugin active છે
- રજિસ્ટ્રેશન એરર માટે WordPress error logs તપાસો

### હું લેગસી મોડ જાળવી રાખવા માંગુ છું {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** પર જાઓ
- **Legacy** પસંદ કરો
- **Save** પર ક્લિક કરો
- લેગસી મોડ કામ કરવાનું ચાલુ રાખશે

### મારી કસ્ટમ એબિલિટીઝ દેખાતી નથી {#my-custom-abilities-arent-showing}

- ચકાસો કે તે WordPress hooks દ્વારા રજિસ્ટર થયેલ છે
- તપાસો કે તેઓ Abilities API ને યોગ્ય રીતે અમલમાં મૂકે છે
- WordPress error logsની સમીક્ષા કરો
- તમામ રજિસ્ટર થયેલી એબિલિટીઝ જોવા માટે **Ability Visibility** એડમિન પેજનો ઉપયોગ કરો

### મને "unclassified ability" નોટિસેસ મળી રહ્યા છે {#im-getting-unclassified-ability-notices}

- આ નવી થર્ડ-પાર્ટી એબિલિટીઝ માટે સામાન્ય છે
- એડમિન નોટિસમાં તેની સમીક્ષા કરો અને વર્ગીકૃત કરો
- વર્ગીકરણ પર વિગતો માટે **Ability Visibility** જુઓ

## પાછળની સુસંગતતા (Backward Compatibility) {#backward-compatibility}

### હાલની કન્ફિગરેશન્સ {#existing-configurations}

જો તમારી પાસે હાલની થર્ડ-પાર્ટી એબિલિટી કન્ફિગરેશન્સ છે:

- **લેગસી મોડ**: તમારી કન્ફિગરેશન કામ કરવાનું ચાલુ રાખશે
- **મેન્યુઅલ મોડ**: તમારી કન્ફિગરેશન કામ કરવાનું ચાલુ રાખશે
- **ઓટો મોડ**: તમારી કન્ફિગરેશનને અવગણવામાં આવશે (ઓટો મોડ નિયંત્રણ લેશે)

તમારી કસ્ટમ કન્ફિગરેશન જાળવી રાખવા માટે, મેન્યુઅલ અથવા લેગસી મોડમાં રહો.

### Deprecation Timeline {#deprecation-timeline}

- **v1.12.0**: લેગસી અને મેન્યુઅલ મોડ હજી પણ સંપૂર્ણપણે સપોર્ટેડ છે
- **v1.13.0+**: લેગસી મોડમાં Deprecation નોટિસેસ દેખાઈ શકે છે
- **v2.0.0**: લેગસી મોડ દૂર થઈ શકે છે (TBD)

## શ્રેષ્ઠ પદ્ધતિઓ (Best Practices) {#best-practices}

### નવી ઇન્સ્ટોલેશન્સ માટે {#for-new-installations}

- ઓટો મોડનો ઉપયોગ કરો (તે ડિફોલ્ટ છે)
- Superdav AI Agent ને એબિલિટીઝ આપોઆપ શોધ કરવા દો
- એક્સેસ નિયંત્રણ માટે Ability Visibility નો ઉપયોગ કરો

### હાલની ઇન્સ્ટોલેશન્સ માટે {#for-existing-installations}

- શક્ય હોય ત્યારે WordPress 7.0+ પર અપગ્રેડ કરો
- સરળ સંચાલન માટે ઓટો મોડ પર સ્વિચ કરો
- Ability Visibility નો ઉપયોગ કરીને એબિલિટીઝની સમીક્ષા અને વર્ગીકરણ કરો

### કસ્ટમ એબિલિટીઝ માટે {#for-custom-abilities}

- WordPress hooks (Abilities API) દ્વારા એબિલિટીઝ રજિસ્ટર કરો
- કસ્ટમ એબિલિટી રજિસ્ટ્રી ટાળો
- ઓટો મોડ સાથે WordPress 7.0+ પર પરીક્ષણ કરો

## આગળના પગલાં {#next-steps}

1. **તમારું WordPress વર્ઝન તપાસો**: ઓટો મોડ માટે ખાતરી કરો કે તમે 7.0+ પર છો
2. **તમારા થર્ડ-પાર્ટી મોડની સમીક્ષા કરો**: સેટિંગ્સ પર જાઓ અને તમારો વર્તમાન મોડ તપાસો
3. **જરૂર પડ્યે અપડેટ કરો**: જો તમે WordPress 7.0+ પર છો તો ઓટો મોડ પર સ્વિચ કરો
4. **એબિલિટીઝનું વર્ગીકરણ કરો**: કોઈપણ વર્ગીકૃત ન થયેલી એબિલિટીઝની સમીક્ષા કરો અને વર્ગીકૃત કરો
5. **પરીક્ષણ કરો**: ખાતરી કરો કે તમારી થર્ડ-પાર્ટી એબિલિટીઝ યોગ્ય રીતે કામ કરે છે

## સંબંધિત વિષયો {#related-topics}

- **Ability Visibility**: કઈ એબિલિટીઝ ક્યાં ખુલ્લી છે તે નિયંત્રિત કરો
- **WordPress Abilities API**: નેટિવ WordPress એબિલિટી રજિસ્ટ્રેશન વિશે જાણો
- **Third-Party Ability Development**: એબિલિટીઝ બનાવો જે ઓટો મોડ સાથે કામ કરે
