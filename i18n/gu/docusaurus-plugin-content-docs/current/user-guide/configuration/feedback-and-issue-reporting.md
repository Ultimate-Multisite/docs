# ગ્રાહક પ્રતિસાદ અને સમસ્યા રિપોર્ટિંગ

Gratis AI Agent v1.5.0 માં એક બિલ્ટ-ઇન (built-in) પ્રતિસાદ અને સમસ્યા રિપોર્ટિંગ સિસ્ટમ રજૂ કરવામાં આવી છે. આ સિસ્ટમ અંતિમ વપરાશકર્તાઓને (end users) ચેટ ઇન્ટરફેસમાંથી જ બિન-મદદરૂપ જવાબોને ફ્લેગ (flag) કરવા અને સમસ્યાઓ રિપોર્ટ કરવા દે છે. આ સિસ્ટમમાં સંમતિ વ્યવસ્થાપન (consent management), એક સ્વચાલિત રિપોર્ટિંગ કમાન્ડ, અને બેકએન્ડ પર AI-સહાયિત ટ્રાઇએજ (triage) શામેલ છે.

## થમ્બ્સ-ડાઉન બટન (Thumbs-Down Button) {#thumbs-down-button}

AI સહાયક દ્વારા મોકલેલા દરેક મેસેજ સાથે એક **થમ્બ્સ-ડાઉન** (👎) બટન દેખાય છે. જ્યારે વપરાશકર્તા તેના પર ક્લિક કરે છે, ત્યારે તેઓ કોઈ જવાબને બિન-મદદરૂપ અથવા ખોટો તરીકે માર્ક કરી શકે છે.

- આ બટન દરેક સહાયક મેસેજની બાજુમાં હોવર (hover) કરવા પર દેખાય છે.
- તેના પર ક્લિક કરવાથી **Feedback Consent Modal** ખુલે છે.
- આ પ્રતિસાદ વાતચીત થ્રેડ (conversation thread) અને ચોક્કસ મેસેજ સાથે જોડાયેલો હોય છે.

## ફીડબેક કન્સન્ટ મોડલ (Feedback Consent Modal) {#feedback-consent-modal}

જ્યારે વપરાશકર્તા થમ્બ્સ-ડાઉન બટન પર ક્લિક કરે છે, ત્યારે કોઈ ડેટા મોકલવામાં આવે તે પહેલાં એક કન્સન્ટ મોડલ દેખાય છે. આ મોડલ:

- સમજાવે છે કે કઈ માહિતી એકત્રિત કરવામાં આવશે (વાતચીતનો ટુકડો, બ્રાઉઝરનો સંદર્ભ).
- વપરાશકર્તાને પૂછે છે કે શું તેઓ આ ડેટા શેર કરવા સંમતિ આપે છે.
- વપરાશકર્તાને જે ખોટું થયું તે વર્ણવવાનો ફ્રી-ટેક્સ્ટ ફીલ્ડ (free-text field) પ્રદાન કરે છે.
- સબમિટ કર્યા વિના બંધ કરવા માટે **Cancel** વિકલ્પ આપે છે.

વપરાશકર્તા સ્પષ્ટપણે પુષ્ટિ ન કરે ત્યાં સુધી કોઈ પ્રતિસાદ રેકોર્ડ થતો નથી.

## ઓટો-પ્રોમ્પ્ટ ફીડબેક બેનર (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

વાતચીતના અંતે, સહાયક એક **ઓટો-પ્રોમ્પ્ટ ફીડબેક બેનર** બતાવી શકે છે — જે એક બિન-વ્યસ્ત (non-intrusive) મેસેજ હોય છે જે પૂછે છે કે સત્ર મદદરૂપ હતું કે નહીં.

આ બેનર વાતચીતની લંબાઈ અને પરિણામના હેયુરિસ્ટિક્સ (heuristics) ના આધારે આપમેળે દેખાય છે. તે થમ્બ્સ-ડાઉન બટન જેટલો જ ફીડબેક ફ્લો (feedback flow) સાથે જોડાયેલો છે. વપરાશકર્તાઓ ફીડબેક આપ્યા વિના આ બેનરને બંધ કરી શકે છે.

## /report-issue સ્લેશ કમાન્ડ (Slash Command) {#report-issue-slash-command}

વપરાશકર્તાઓ ચેટ ઇનપુટમાં `/report-issue` ટાઇપ કરીને સીધો ફીડબેક ફ્લો શરૂ કરી શકે છે. આ કમાન્ડ:

- તરત જ ફીડબેક કન્સન્ટ મોડલ ખોલે છે.
- વર્તમાન વાતચીત વિશે સંદર્ભ સાથે ડિસ્ક્રિપ્શન ફીલ્ડને પ્રી-પૉપ્યુલેટ (pre-populate) કરે છે.
- સબમિટ કરતા પહેલા વપરાશકર્તાને વધારાની વિગતો ઉમેરવાની મંજૂરી આપે છે.

`/report-issue` કમાન્ડ તમામ ચેટ મોડ્સ (inline, floating widget, full-screen) માં ઉપલબ્ધ છે.

## AI-સહાયિત ટ્રાઇએજ (AI-Assisted Triage) {#ai-assisted-triage}

સબમિટ કરેલો પ્રતિસાદ એક AI ટ્રાઇએજ લેયર (triage layer) પર મોકલવામાં આવે છે જે:

- રિપોર્ટને વર્ગીકૃત કરે છે (તથ્યગત ભૂલ, બિન-મદદરૂપ જવાબ, અયોગ્ય સામગ્રી, વગેરે).
- સંબંધિત વાતચીત મેટાડેટા જોડે છે.
- ટ્રાઇએજ સારાંશને કન્ફિગરેડ **Feedback Endpoint** (જુઓ [Settings > Advanced](#settings-advanced)) પર ફોરવર્ડ કરે છે.

આનાથી સૌથી ગંભીર સમસ્યાઓને પહેલા બહાર લાવીને મેન્યુઅલ રિવ્યૂનો સમય ઘટાડે છે.

## Settings > Advanced {#settings-advanced}

ફીડબેક બેકએન્ડને સક્ષમ કરવા માટે, **Gratis AI Agent → Settings → Advanced** હેઠળ નીચેના ફીલ્ડ્સને કન્ફિગર કરો:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | જે URL ફીડબેક પેલોડ્સ (JSON) સાથે POST રિક્વેસ્ટ મેળવે છે. |
| **Feedback API Key** | ફીડબેક સબમિશનને ઓથેન્ટિકેટ કરવા માટે `Authorization` હેડરમાં મોકલેલો બેરર ટોકન (Bearer token). |

ફીડબેક કલેક્શનને ડિસેબલ કરવા માટે બંને ફીલ્ડ્સને ખાલી રાખો. વ્યક્તિગત ફીડબેક બટનો વપરાશકર્તાઓ માટે દેખાતા રહેશે, પરંતુ સબમિશન ફોરવર્ડ થશે નહીં.

:::tip
ફીડબેક એન્ડપોઇન્ટમાં ઓછામાં ઓછા `message_id`, `conversation_id`, `feedback_text`, અને `triage_category` ફીલ્ડ્સ સાથેનું JSON બોડી સ્વીકારવું આવશ્યક છે. અપેક્ષિત સ્કીમા (schema) માટે તમારા એન્ડપોઇન્ટ પ્રદાતાના દસ્તાવેજીકરણ (documentation) જુઓ.
:::
