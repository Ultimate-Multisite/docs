# Site Specification Skill

**Site Specification skill** એ તમારા સાઇટના લક્ષ્યો, ઓડિયન્સ અને બ્રાન્ડની ઓળખને એક વ્યવસ્થિત રીતે કેપ્ચર કરવાની એક રીત છે. આ માહિતી તમારી **site_brief** મેમરીમાં સ્ટોર થાય છે, જે એજન્ટ્સ સેશન્સ દરમિયાન સંદર્ભ તરીકે ઉપયોગ કરે છે જેથી સતત અને સંદર્ભને સમજતી મદદ આપી શકાય.

## Site Specification શું છે? {#what-is-site-specification}

Site Specification એ નીચેની બાબતોનું દસ્તાવેજીકરણ કરવાની પ્રક્રિયા છે:

- **Site purpose**: તમારું સાઇટ શું કરે છે અને તે શા માટે અસ્તિત્વમાં છે
- **Target audience**: તમારા સાઇટની મુલાકાત કોણ લે છે અને તેમને શું જરૂર છે
- **Brand identity**: તમારા રંગો, સ્વર (tone) અને વિઝ્યુઅલ સ્ટાઇલ
- **Business goals**: તમારા સાઇટ માટે સફળતા કેવી દેખાય છે
- **Content structure**: તમારું સાઇટ કેવી રીતે ગોઠવાયેલું છે

આ સ્પષ્ટીકરણ તમારી **site_brief** બની જાય છે, જે એક કાયમી મેમરી છે જે એજન્ટ્સ તમારા સાઇટના સંદર્ભને સમજવા માટે ઉપયોગ કરે છે.

## Site Specification શા માટે વાપરવું? {#why-use-site-specification}

### સેશન્સમાં સુસંગતતા (Consistency Across Sessions) {#consistency-across-sessions}

site_brief વિના, તમારે દર વખતે જ્યારે તમે નવું સેશન શરૂ કરો ત્યારે તમારા સાઇટના હેતુ વિશે ફરીથી સમજાવવું પડશે. site_brief સાથે, એજન્ટ્સ તરત જ સમજી જાય છે:

- તમારા સાઇટના લક્ષ્યો અને ઓડિયન્સ
- તમારા બ્રાન્ડના રંગો અને સ્વર
- તમારી કન્ટેન્ટ સ્ટ્રક્ચર
- તમારા બિઝનેસ ઓબ્જેક્ટિવ્સ

### વધુ સારી ભલામણો (Better Recommendations) {#better-recommendations}

એજન્ટ્સ તમારા site_brief નો ઉપયોગ નીચેના માટે કરે છે:

- તમારા સાઇટના હેતુ સાથે સુસંગત ફીચર્સ સૂચવવા
- તમારા લક્ષ્યો સાથે મેળ ખાતી કન્ટેન્ટ સ્ટ્રક્ચરની ભલામણ કરવી
- તમારા બ્રાન્ડ સાથે સુસંગત ડિઝાઇન પ્રસ્તાવિત કરવી
- અસંગત ફીચર્સ સૂચવવાનું ટાળવું

### ઝડપી ઓનબોર્ડિંગ (Faster Onboarding) {#faster-onboarding}

નવા એજન્ટ્સ (અથવા નવા સેશન્સમાં એજન્ટ્સ) સ્પષ્ટતાના પ્રશ્નો પૂછવાને બદલે તમારા site_brief ને વાંચીને ઝડપથી તૈયાર થઈ શકે છે.

## Site Specification શરૂ કરવું {#initiating-site-specification}

### Theme Builder Onboarding દરમિયાન {#during-theme-builder-onboarding}

Site Specification skill આપમેળે **Theme Builder onboarding flow** દરમિયાન શરૂ થાય છે. Setup Assistant એજન્ટ પ્રશ્નો પૂછે છે અને તમારું site_brief બનાવે છે.

### મેન્યુઅલ શરૂઆત (Manual Initiation) {#manual-initiation}

તમે ગમે ત્યારે Site Specification શરૂ કરી શકો છો:

```
"Let's define my site specification"
```

અથવા

```
"Help me create a site brief"
```

## The Site Specification Process {#the-site-specification-process}

### Step 1: Site Purpose {#step-1-site-purpose}

એજન્ટ પૂછે છે:

```
What is your site's primary purpose?
- E-commerce store
- Blog or content site
- Portfolio or showcase
- SaaS application
- Community or forum
- Other: [describe]
```

તમે એક કેટેગરી પસંદ કરી શકો છો અથવા તમારો પોતાનો હેતુ વર્ણવી શકો છો.

### Step 2: Target Audience {#step-2-target-audience}

```
Who is your primary audience?
- Consumers / general public
- Business professionals
- Developers / technical users
- Students / educators
- Other: [describe]

What are their main needs?
```

### Step 3: Brand Identity {#step-3-brand-identity}

```
What are your brand colors?
- Primary color: [color picker or hex code]
- Secondary color: [color picker or hex code]
- Accent color: [optional]

How would you describe your brand tone?
- Professional / corporate
- Creative / artistic
- Playful / casual
- Minimal / modern
- Warm / friendly
```

### Step 4: Business Goals {#step-4-business-goals}

```
What does success look like for your site?
- Generate leads
- Sell products
- Build community
- Share knowledge
- Establish authority
- Other: [describe]

What's your primary metric?
- Revenue
- User engagement
- Content reach
- Conversions
- Other
```

### Step 5: Content Structure {#step-5-content-structure}

```
How is your content organized?
- Flat (all content at same level)
- Hierarchical (categories and subcategories)
- Chronological (blog-style)
- Product-based (catalog)
- Other: [describe]

What content types do you use?
- Blog posts
- Product pages
- Case studies
- Documentation
- Videos
- Other
```

## તમારું site_brief મેમરી {#your-sitebrief-memory}

Site Specification પૂર્ણ કર્યા પછી, તમારી માહિતી તમારા એજન્ટની મેમરીમાં **site_brief** તરીકે સ્ટોર થાય છે. આ એક structured record છે જેમાં નીચેની વિગતો હોય છે:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## site_brief જોવું અને અપડેટ કરવું {#viewing-and-updating-your-sitebrief}

### તમારું site_brief જુઓ {#view-your-sitebrief}

એજન્ટને પૂછો:

```
"Show me my site brief"
```

અથવા

```
"What do you know about my site?"
```

એજન્ટ તમારી સ્ટોર કરેલી સ્પષ્ટીકરણ પ્રદર્શિત કરશે.

### તમારું site_brief અપડેટ કરો {#update-your-sitebrief}

જો તમારું સાઇટ વિકસિત થાય, તો તમે તેને અપડેટ કરી શકો છો:

```
"Update my site brief: we're now targeting B2B customers"
```

અથવા

```
"Refresh my site specification"
```

આ તમારી હાલની માહિતીને શરૂઆતનો મુદ્દો બનાવીને Site Specification skill ને ફરીથી ચલાવશે.

## એજન્ટ્સ site_brief નો ઉપયોગ કેવી રીતે કરે છે {#how-agents-use-sitebrief}

### ડિઝાઇન ભલામણો (Design Recommendations) {#design-recommendations}

જ્યારે તમે ડિઝાઇન ફેરફારો માટે પૂછો છો, ત્યારે એજન્ટ્સ તમારા site_brief નો સંદર્ભ લે છે:

```
You: "Redesign my homepage"
Agent: "Based on your site brief, you're targeting women aged 25-45
with a warm, artisanal tone. I'll create a design that emphasizes
your handmade jewelry and sustainable values."
```

### ફીચર સૂચનો (Feature Suggestions) {#feature-suggestions}

એજન્ટ્સ તમારા લક્ષ્યો સાથે સુસંગત ફીચર્સ સૂચવે છે:

```
You: "What features should I add?"
Agent: "For an e-commerce store focused on customer loyalty,
I'd recommend: loyalty program, customer reviews, email newsletter,
and product recommendations."
```

### કન્ટેન્ટ સ્ટ્રક્ચર (Content Structure) {#content-structure}

એજન્ટ્સ તમારી સ્ટ્રક્ચરના આધારે કન્ટેન્ટ ઓર્ગેનાઇઝેશન પ્રસ્તાવિત કરે છે:

```
You: "How should I organize my blog?"
Agent: "Since you use a product catalog with blog, I'd suggest
organizing posts by product category and creating 'styling guides'
that showcase multiple products together."
```

## શ્રેષ્ઠ પદ્ધતિઓ (Best Practices) {#best-practices}

### ચોક્કસ રહો (Be Specific) {#be-specific}

"general audience" ને બદલે, તમારા વાસ્તવિક ઓડિયન્સનું વર્ણન કરો:

- ✓ "Women aged 25-45, interested in sustainable fashion"
- ✗ "Everyone"

### નિયમિત અપડેટ કરો (Update Regularly) {#update-regularly}

જેમ જેમ તમારું સાઇટ વિકસિત થાય, તેમ તેમ તમારા site_brief ને અપડેટ કરો:

- જ્યારે તમે નવા ઓડિયન્સ તરફ વળો (pivot)
- જ્યારે તમે નવી પ્રોડક્ટ લાઈન ઉમેરો
- જ્યારે તમારી બ્રાન્ડની ઓળખ બદલાય
- જ્યારે તમારા બિઝનેસ લક્ષ્યો બદલાય

### સુસંગત પરિભાષાનો ઉપયોગ કરો (Use Consistent Terminology) {#use-consistent-terminology}

સેશન્સમાં સમાન શબ્દોનો ઉપયોગ કરો:

- ✓ હંમેશા "sustainable jewelry" કહો (નહીં કે "eco-friendly jewelry" અને "green products")
- ✓ તમારા ઓડિયન્સનો સતત એક જ રીતે ઉલ્લેખ કરો

### સંદર્ભ શામેલ કરો (Include Context) {#include-context}

એજન્ટ્સને તમારા નિર્ણયો સમજવામાં મદદ કરતો બેકગ્રાઉન્ડ પ્રદાન કરો:

- "We're targeting professionals who value quality over price"
- "Our audience is tech-savvy and expects modern design"
- "We're a bootstrapped startup, so we need cost-effective solutions"

## Theme Builder Onboarding સાથે સંબંધ {#relationship-to-theme-builder-onboarding}

Site Specification skill **Theme Builder onboarding flow** માં સંકલિત છે. જ્યારે તમે ઓનબોર્ડિંગ પૂર્ણ કરો છો, ત્યારે તમે જે માહિતી આપી છે તેના આધારે તમારું site_brief આપમેળે બનાવવામાં આવે છે.

જો તમે ઈચ્છો તો તમે સ્વતંત્ર રીતે Site Specification ચલાવી શકો છો:

- પ્રારંભિક સેટઅપ પછી તમારા સ્પષ્ટીકરણને સુધારવા માટે
- જેમ જેમ તમારું સાઇટ વિકસિત થાય તેમ તમારું site brief અપડેટ કરવા માટે
- Theme Builder શરૂ કરતા પહેલા વિગતવાર સ્પષ્ટીકરણ બનાવવા માટે

## ટ્રબલશૂટિંગ (Troubleshooting) {#troubleshooting}

**મારું site_brief વપરાતું નથી**
- પુષ્ટિ કરો કે એજન્ટ પાસે મેમરીની ઍક્સેસ છે
- એજન્ટને "recall my site brief" કહેવા કહો
- તપાસો કે તમારી સેટિંગ્સમાં મેમરી એનેબલ છે

**હું એક નવા site_brief થી શરૂ કરવા માંગુ છું**
- એજન્ટને પૂછો: "Clear my site brief and start fresh"
- પછી ફરીથી Site Specification ચલાવો

**એજન્ટ જે ભલામણો કરી રહ્યો છે તે મારા site_brief સાથે મેળ ખાતી નથી**
- એજન્ટને "review my site brief" કહેવા કહો
- જો તે જૂનું હોય તો તમારું site_brief અપડેટ કરો
- તમારા વિનંતીઓમાં વધારાનો સંદર્ભ પ્રદાન કરો

## આગળના પગલાં (Next Steps) {#next-steps}

તમારા સાઇટનું સ્પષ્ટીકરણ વ્યાખ્યાયિત કર્યા પછી:

1. **Theme Builder નો ઉપયોગ કરો**: તમારા site_brief પર આધારિત કસ્ટમ થીમ બનાવો
2. **ડિઝાઇન સુધારો**: વિગતવાર ડિઝાઇન કામ માટે Design System Aesthetics skill નો ઉપયોગ કરો
3. **કન્ટેન્ટનું આયોજન કરો**: કન્ટેન્ટ સ્ટ્રક્ચર ભલામણો માટે એજન્ટ્સને પૂછો
4. **ફીચર્સ બનાવો**: તમારા બિઝનેસ લક્ષ્યો સાથે સંરેખિત ફીચર્સની વિનંતી કરો
