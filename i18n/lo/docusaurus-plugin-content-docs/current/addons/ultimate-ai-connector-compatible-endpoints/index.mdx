# ຕົວເຊື່ອມຕໍ່ AI ຂອງ Ultimate ສຳລັບ endpoint ທີ່ເຂົ້າກັນໄດ້ {#ultimate-ai-connector-for-compatible-endpoints}

ຕົວເຊື່ອມຕໍ່ AI ຂອງ Ultimate ສຳລັບ endpoint ທີ່ເຂົ້າກັນໄດ້ ຜະສານ API ຂອງຜູ້ໃຫ້ບໍລິການ AI ທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI ເຂົ້າກັບເຄືອຂ່າຍ WordPress multisite ຂອງທ່ານ. ກຳນົດຄ່າຜູ້ໃຫ້ບໍລິການ AI ຫຼາຍລາຍ — ແຕ່ລະລາຍມີ endpoint URL, API key ແລະ model ຄ່າເລີ່ມຕົ້ນຂອງຕົນເອງ — ແລະສົ່ງເສັ້ນທາງຄຳຂໍພ້ອມການ fallback ອັດຕະໂນມັດເມື່ອຜູ້ໃຫ້ບໍລິການໃດໜຶ່ງບໍ່ພ້ອມໃຊ້ງານ.

## ຄຸນສົມບັດຫຼັກ {#key-features}

- **ຮອງຮັບຫຼາຍຜູ້ໃຫ້ບໍລິການ**: ເພີ່ມ card ຜູ້ໃຫ້ບໍລິການ AI ຫຼາຍລາຍ, ແຕ່ລະລາຍມີ endpoint URL, API key ແລະ model ຄ່າເລີ່ມຕົ້ນຂອງຕົນເອງ
- **ລາກເພື່ອຈັດລຳດັບໃໝ່**: ຈັດຮຽງຜູ້ໃຫ້ບໍລິການຕາມລຳດັບຄວາມສຳຄັນໂດຍໃຊ້ດ້າມຈັບໃນແຕ່ລະ card; ຄຳຂໍຈະໃຊ້ຜູ້ໃຫ້ບໍລິການຕາມລຳດັບ
- **ການສົ່ງເສັ້ນທາງ fallback ອັດຕະໂນມັດ**: ຖ້າຜູ້ໃຫ້ບໍລິການຫຼັກບໍ່ພ້ອມໃຊ້ງານ, ຄຳຂໍຈະ fallback ໄປຫາຜູ້ໃຫ້ບໍລິການຖັດໄປທີ່ກຳນົດຄ່າໄວ້ໂດຍອັດຕະໂນມັດ
- **ການກຳນົດຄ່າຕໍ່ຜູ້ໃຫ້ບໍລິການ**: ແຕ່ລະຜູ້ໃຫ້ບໍລິການຖືກກຳນົດຄ່າແຍກອິດສະຫຼະດ້ວຍ credentials ແລະການເລືອກ model ຂອງຕົນເອງ
- **UI ການຕັ້ງຄ່າແບບ card**: ສ້າງຂຶ້ນຈາກຄອມໂພເນັນ Card ທີ່ສະຖຽນຂອງ WordPress 6.9+ (CardBody, CardHeader, CardDivider)
- **ຂະຫຍາຍອັດຕະໂນມັດເມື່ອເພີ່ມ**: card ຜູ້ໃຫ້ບໍລິການໃໝ່ຈະຂະຫຍາຍອັດຕະໂນມັດ ເພື່ອໃຫ້ທ່ານກອກລາຍລະອຽດໄດ້ທັນທີ
- **ການລ້າງ cache ຂອງ script**: script ຂອງຜູ້ໃຫ້ບໍລິການຈະຖືກ cache-busted ເມື່ອອັບເດດ plugin ເພື່ອຮັບປະກັນວ່າໂຫຼດເວີຊັນຫຼ້າສຸດແລ້ວ

## ຂໍ້ກຳນົດ {#requirements}

- WordPress 5.3 ຫຼືສູງກວ່າ
- PHP 7.4 ຫຼືສູງກວ່າ
- plugin Ultimate Multisite (ເປີດໃຊ້ງານ)
- endpoint API ທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI (ໂຮສເອງ ຫຼື ບຸກຄົນທີສາມ)

## ການຕິດຕັ້ງ {#installation}

1. ອັບໂຫຼດໄຟລ໌ addon ໄປທີ່ໄດເຣັກທໍຣີ `/wp-content/plugins/` ຂອງທ່ານ
2. ເປີດໃຊ້ງານ plugin ຜ່ານເມນູ 'Plugins' ໃນ WordPress
3. ໄປທີ່ **Ultimate Multisite → AI Connector** ເພື່ອກຳນົດຄ່າ addon

## ການກຳນົດຄ່າ {#configuration}

### ການຕັ້ງຄ່າຫຼາຍຜູ້ໃຫ້ບໍລິການ {#multi-provider-setup}

ໜ້າຈໍການກຳນົດຄ່າໃຊ້ layout ແບບ card ໂດຍທີ່ແຕ່ລະ card ແທນຜູ້ໃຫ້ບໍລິການ AI ໜຶ່ງລາຍ. ທ່ານສາມາດເພີ່ມຜູ້ໃຫ້ບໍລິການໄດ້ຫຼາຍເທົ່າທີ່ຕ້ອງການ.

#### ການເພີ່ມຜູ້ໃຫ້ບໍລິການ {#adding-a-provider}

1. ໄປທີ່ **Ultimate Multisite → AI Connector** ໃນ network admin
2. ຄລິກ **ເພີ່ມຜູ້ໃຫ້ບໍລິການ**
3. card ຜູ້ໃຫ້ບໍລິການໃໝ່ຈະປາກົດ ແລະຂະຫຍາຍອັດຕະໂນມັດ
4. ກອກລາຍລະອຽດຜູ້ໃຫ້ບໍລິການ:
   - **ຊື່ຜູ້ໃຫ້ບໍລິການ** — ປ້າຍຊື່ເພື່ອລະບຸຜູ້ໃຫ້ບໍລິການນີ້ (ເຊັ່ນ "OpenAI", "Mistral", "Local Ollama")
   - **Endpoint URL** — URL ພື້ນຖານຂອງ API ທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI (ເຊັ່ນ `https://api.openai.com/v1`)
   - **API Key** — API key ຂອງທ່ານສຳລັບຜູ້ໃຫ້ບໍລິການນີ້
   - **Model ຄ່າເລີ່ມຕົ້ນ** — ຕົວລະບຸ model ທີ່ໃຊ້ເປັນຄ່າເລີ່ມຕົ້ນສຳລັບຜູ້ໃຫ້ບໍລິການນີ້ (ເຊັ່ນ `gpt-4o`, `mistral-small`)
5. ຄລິກ **ບັນທຶກການຕັ້ງຄ່າ**

#### ການລຶບຜູ້ໃຫ້ບໍລິການ {#removing-a-provider}

ຄລິກປຸ່ມ **ລຶບ** ຢູ່ໃນ card ຜູ້ໃຫ້ບໍລິການໃດກໍໄດ້ເພື່ອລຶບມັນ. ບັນທຶກການຕັ້ງຄ່າເພື່ອນຳໃຊ້ການປ່ຽນແປງ.

#### ການຈັດລຳດັບຜູ້ໃຫ້ບໍລິການໃໝ່ {#reordering-providers}

ລາກໄອຄອນດ້າມຈັບ (⣿) ຢູ່ດ້ານຊ້າຍຂອງ card ຜູ້ໃຫ້ບໍລິການໃດກໍໄດ້ເພື່ອຈັດລຳດັບໃໝ່. ລຳດັບຈະກຳນົດຄວາມສຳຄັນຂອງ fallback: ຄຳຂໍຖືກສົ່ງໄປຫາຜູ້ໃຫ້ບໍລິການທຳອິດ ແລະ fallback ໄປຫາຜູ້ໃຫ້ບໍລິການຖັດໄປຖ້າຄຳຂໍລົ້ມເຫຼວ.

### ການສົ່ງເສັ້ນທາງ Fallback {#fallback-routing}

ເມື່ອຄຳຂໍໄປຫາຜູ້ໃຫ້ບໍລິການຫຼັກລົ້ມເຫຼວ — ເນື່ອງຈາກ timeout, rate limit, ຫຼື ການຂັດຂ້ອງຂອງບໍລິການ — ຕົວເຊື່ອມຕໍ່ຈະລອງໃໝ່ກັບຜູ້ໃຫ້ບໍລິການຖັດໄປໃນລາຍການໂດຍອັດຕະໂນມັດ. ການນີ້ຈະດຳເນີນຕໍ່ໄປຕາມລາຍການຈົນກວ່າຜູ້ໃຫ້ບໍລິການໃດໜຶ່ງສຳເລັດ ຫຼືຜູ້ໃຫ້ບໍລິການທັງໝົດຖືກລອງແລ້ວ.

| ສະຖານະການ | ພຶດຕິກຳ |
|----------|-----------|
| ຜູ້ໃຫ້ບໍລິການຫຼັກຕອບກັບ | ຄຳຂໍຖືກໃຫ້ບໍລິການຕາມປົກກະຕິ |
| ຜູ້ໃຫ້ບໍລິການຫຼັກລົ້ມເຫຼວ | ຄຳຂໍຖືກລອງໃໝ່ກັບຜູ້ໃຫ້ບໍລິການຖັດໄປ |
| ຜູ້ໃຫ້ບໍລິການທັງໝົດລົ້ມເຫຼວ | ສົ່ງຄືນຂໍ້ຜິດພາດໃຫ້ຜູ້ເອີ້ນ |

Fallback ໂປ່ງໃສຕໍ່ plugin ຫຼື theme ທີ່ເອີ້ນໃຊ້ — ພວກເຂົາຈະໄດ້ຮັບຄຳຕອບທີ່ສຳເລັດ ຫຼືຂໍ້ຜິດພາດສຸດທ້າຍ, ບໍ່ວ່າຈະລອງຜູ້ໃຫ້ບໍລິການໄປແລ້ວຈັກລາຍກໍຕາມ.

### ອ້າງອີງການກຳນົດຄ່າຜູ້ໃຫ້ບໍລິການ {#provider-configuration-reference}

| ຊ່ອງຂໍ້ມູນ | ຈຳເປັນ | ຄຳອະທິບາຍ |
|-------|----------|-------------|
| ຊື່ຜູ້ໃຫ້ບໍລິການ | ແມ່ນ | ປ້າຍຊື່ທີ່ສະແດງສຳລັບຜູ້ໃຫ້ບໍລິການນີ້ |
| Endpoint URL | ແມ່ນ | URL ພື້ນຖານຂອງ API ທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI |
| API Key | ແມ່ນ | ຄີຢືນຢັນຕົວຕົນສຳລັບຜູ້ໃຫ້ບໍລິການນີ້ |
| Model ຄ່າເລີ່ມຕົ້ນ | ແມ່ນ | ຕົວລະບຸ model ທີ່ໃຊ້ເມື່ອບໍ່ໄດ້ລະບຸ model ໃນຄຳຂໍ |

## ການແກ້ໄຂບັນຫາ {#troubleshooting}

### ບໍ່ມີຜູ້ໃຫ້ບໍລິການຢູ່ໃນລາຍການ {#no-providers-are-listed}

ຖ້າລາຍການຜູ້ໃຫ້ບໍລິການຫວ່າງເປົ່າຫຼັງຈາກບັນທຶກ, ໃຫ້ຢືນຢັນວ່າ:

- ທ່ານໄດ້ຄລິກ **ບັນທຶກການຕັ້ງຄ່າ** ຫຼັງຈາກເພີ່ມຜູ້ໃຫ້ບໍລິການ
- browser ຂອງທ່ານບໍ່ໄດ້ບລັອກໜ້າການຕັ້ງຄ່າບໍ່ໃຫ້ບັນທຶກ (ກວດເບິ່ງ browser console ສຳລັບຂໍ້ຜິດພາດ JavaScript)
- plugin ເປີດໃຊ້ງານຢູ່ ແລະທ່ານຢູ່ໃນໜ້າຈໍ network admin

### ຄຳຂໍລົ້ມເຫຼວສຳລັບຜູ້ໃຫ້ບໍລິການທັງໝົດ {#requests-are-failing-for-all-providers}

- ກວດຢືນຢັນວ່າ endpoint URL ຂອງແຕ່ລະຜູ້ໃຫ້ບໍລິການສາມາດເຂົ້າເຖິງຈາກ server ຂອງທ່ານ (ບໍ່ແມ່ນພຽງແຕ່ຈາກ browser ຂອງທ່ານ)
- ກວດສອບວ່າ API key ແຕ່ລະອັນຖືກຕ້ອງ ແລະບໍ່ໄດ້ໝົດອາຍຸ
- ທົບທວນສະຖານະ rate-limit ແລະ quota ຂອງຜູ້ໃຫ້ບໍລິການໃນ dashboard ຂອງພວກເຂົາ
- ເປີດໃຊ້ WordPress debug logging (`WP_DEBUG_LOG`) ເພື່ອບັນທຶກຂໍ້ຄວາມຂໍ້ຜິດພາດລະອຽດຈາກຕົວເຊື່ອມຕໍ່

### ຜູ້ໃຫ້ບໍລິການບາງລາຍບໍ່ຖືກໃຊ້ສຳລັບ fallback {#a-provider-is-not-being-used-for-fallback}

ການລາກເພື່ອຈັດລຳດັບໃໝ່ຈະປ່ຽນລຳດັບ fallback. ຢືນຢັນວ່າຜູ້ໃຫ້ບໍລິການທີ່ທ່ານຄາດວ່າຈະຮັບຄຳຂໍ fallback ຖືກວາງຢູ່ໃນຕຳແໜ່ງທີ່ຖືກຕ້ອງໃນລາຍການ ແລະ endpoint URL ກັບ API key ຂອງມັນຖືກຕ້ອງ.

### card ຜູ້ໃຫ້ບໍລິການໃໝ່ບໍ່ຂະຫຍາຍອັດຕະໂນມັດ {#new-provider-card-does-not-expand-automatically}

ການຂະຫຍາຍອັດຕະໂນມັດຕ້ອງໃຫ້ເປີດໃຊ້ JavaScript ໃນ browser ຂອງທ່ານ. ຖ້າ card ບໍ່ຂະຫຍາຍເມື່ອຄລິກ, ໃຫ້ກວດເບິ່ງ browser console ສຳລັບຂໍ້ຜິດພາດ. ເນື້ອໃນຂອງ card ສາມາດເຂົ້າເຖິງໄດ້ສະເໝີໂດຍການຄລິກຫົວ card ແມ່ນແຕ່ຖ້າການຂະຫຍາຍອັດຕະໂນມັດບໍ່ຖືກກະຕຸ້ນ.
