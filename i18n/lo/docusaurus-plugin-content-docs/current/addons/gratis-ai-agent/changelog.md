# ບັນທຶກການປ່ຽນແປງ

## 1.9.0 — ປ່ອຍໃຊ້ໃນວັນທີ 2026-04-28

### ໃໝ່

- **ຄວາມສາມາດ `create_contact_form`** — ສ້າງຟອມຕິດຕໍ່ໂດຍໃຊ້ plugin ຟອມທີ່ເປີດໃຊ້ຢູ່ (Contact Form 7, WPForms, Fluent Forms, ຫຼື Gravity Forms) ແລະສົ່ງຄືນ shortcode ທີ່ພ້ອມຝັງໃນໂພສ ຫຼື ໜ້າໃດກໍໄດ້.
- **ຄວາມສາມາດ `set_featured_image`** — ກຳນົດຮູບເດັ່ນໃຫ້ໂພສ ຫຼື ໜ້າ ຈາກ ID ໄຟລ໌ແນບ Media Library ທີ່ມີຢູ່ ຫຼື URL ຈາກພາຍນອກ; ນຳເຂົ້າຮູບອັດຕະໂນມັດເມື່ອມີການໃຫ້ URL.
- **ຄວາມສາມາດ `batch_create_posts`** — ສ້າງຫຼາຍໂພສໃນການເອີ້ນຄວາມສາມາດຄັ້ງດຽວ. ຮອງຮັບພາລາມິເຕີດຽວກັນກັບ `create_post`, ລາຍງານຄວາມສຳເລັດ/ລົ້ມເຫຼວແຍກຕາມແຕ່ລະໂພສ, ແລະມີໂໝດ `stop_on_error` ເປັນທາງເລືອກ.
- **ພາລາມິເຕີ `page_template`** — ເພີ່ມເຂົ້າໃນ `create_post` ແລະ `update_post`. ກຳນົດໄຟລ໌ແມ່ແບບໜ້າ PHP (ເຊັ່ນ `page-full-width.php`) ໃນເວລາສ້າງ ຫຼື ອັບເດດ. ສົ່ງສະຕຣິງວ່າງໄປທີ່ `update_post` ເພື່ອກັບຄືນເປັນຄ່າເລີ່ມຕົ້ນຂອງ theme.
- **ຄວາມສາມາດຖ່າຍພາບໜ້າຈໍຝັ່ງ client** — `capture_screenshot`, `compare_screenshots`, ແລະ `review_page_design`. ຈັບພາບໜ້າຈໍແບບເຕັມໜ້າ ຫຼື ແບບ viewport ຂອງໜ້າສົດຜ່ານ headless browser ຝັ່ງ server, ປຽບທຽບຄວາມແຕກຕ່າງຂອງພາບໜ້າຈໍສອງຮູບ, ແລະຮັບການວິຈານດ້ານການອອກແບບທີ່ສ້າງໂດຍ AI ເຊິ່ງຄອບຄຸມໂຄງຮ່າງ, ຕົວອັກສອນ, ສີ, ແລະ ການເຂົ້າເຖິງ.
- **agent ໃນຕົວຫ້າຕົວ** — ຜູ້ຂຽນເນື້ອຫາ, ຜູ້ສ້າງເວັບໄຊ, ສະຕູດິໂອອອກແບບ, ຜູ້ຈັດການ Plugin, ແລະ ຜູ້ຊ່ວຍສະໜັບສະໜູນ. agent ແຕ່ລະຕົວມີຊຸດເຄື່ອງມືສະເພາະ, system prompt ທີ່ປັບໃຫ້ເໝາະ, ແລະຄຳແນະນຳເລີ່ມຕົ້ນ. ສາມາດສະຫຼັບໄດ້ຜ່ານ **ຕົວເລືອກ Agent** ໃໝ່ໃນສ່ວນຫົວຂອງແຊັດ. ເບິ່ງ [Agent ໃນຕົວ](../../user-guide/configuration/built-in-agents).
- **ທຸງຟີເຈີ** — ແທັບ **ການຕັ້ງຄ່າ → ທຸງຟີເຈີ** ໃໝ່ ພ້ອມສະວິດຄວບຄຸມການເຂົ້າເຖິງ (ຈຳກັດເຉພາະຜູ້ບໍລິຫານ, ຈຳກັດເຉພາະຜູ້ບໍລິຫານເຄືອຂ່າຍ, ການເຂົ້າເຖິງຂອງຜູ້ສະໝັກສະມາຊິກ, ປິດໃຊ້ສຳລັບຜູ້ທີ່ບໍ່ແມ່ນສະມາຊິກ) ແລະຕົວເລືອກການສ້າງແບຣນ (ເຊື່ອງການອ້າງອີງໃນສ່ວນທ້າຍ, ຊື່ agent ກຳນົດເອງ, ເຊື່ອງຕົວເລືອກ agent, ໃຊ້ໄອຄອນເວັບໄຊເປັນ avatar ຂອງແຊັດ). ເບິ່ງ [ການຕັ້ງຄ່າ Gratis AI Agent](../../user-guide/administration/gratis-ai-agent-settings).
- **ກູ້ຄືນເຊດຊັນຫຼ້າສຸດ** — ຕອນນີ້ແຜງແຊັດໂຫຼດການສົນທະນາຫຼ້າສຸດຄືນໂດຍອັດຕະໂນມັດເມື່ອໂຫຼດໜ້າ ແລະ ເມື່ອເປີດ widget, ດັ່ງນັ້ນບໍລິບົດຈະບໍ່ສູນຫາຍລະຫວ່າງການນຳທາງຂ້າມໜ້າ.
- **ລິ້ງການກະທຳຂອງ Plugin** — ຕອນນີ້ລິ້ງດ່ວນໄປຫາການຕັ້ງຄ່າ ແລະ ທະບຽນຄວາມສາມາດ ປາກົດໃນໜ້າ WordPress **Plugins → Plugins ທີ່ຕິດຕັ້ງແລ້ວ** ຢູ່ລຸ່ມຄຳອະທິບາຍ plugin.

### ປັບປຸງ

- **ລອງໃໝ່ແຫຼ່ງຮູບພາບ** — ຕອນນີ້ agent ລອງໃໝ່ກັບແຫຼ່ງຮູບພາບຟຣີທີ່ກຳນົດໄວ້ທັງໝົດ ກ່ອນຈະຖອຍໄປໃຊ້ຮູບພາບທີ່ສ້າງໂດຍ AI ເມື່ອດາວໂຫຼດລົ້ມເຫຼວ.
- **ແຜງຂໍ້ມູນ model** — ສະແດງຢູ່ໃນສ່ວນຫົວແຊັດສະເໝີ; ບໍ່ຖືກເຊື່ອງຫຼັງຈາກຂໍ້ຄວາມທຳອິດອີກຕໍ່ໄປ.
- **ພຶດຕິກຳການເລື່ອນອັດຕະໂນມັດ** — ການເລື່ອນອັດຕະໂນມັດຈະຢຸດຊົ່ວຄາວເມື່ອຜູ້ໃຊ້ເລື່ອນຂຶ້ນເພື່ອອ່ານ; ປຸ່ມລອຍ **ເລື່ອນໄປທ້າຍສຸດ** ຈະປາກົດ ແລະຫາຍໄປອັດຕະໂນມັດເມື່ອຜູ້ໃຊ້ໄປຮອດຂໍ້ຄວາມຫຼ້າສຸດ.
- **UI ຕົວເລືອກ Agent** — ອອກແບບໃໝ່ເປັນ overlay ແບບຕາຕະລາງຟອມ ພ້ອມໄອຄອນແຍກຕາມ agent, ເຮັດໃຫ້ລະບຸ ແລະສະຫຼັບ agent ໄດ້ງ່າຍຂຶ້ນໃນພາບລວມ.
- **ຊິ້ນສ່ວນ JS ທີ່ໂຫຼດແບບ lazy-loaded** — ຕອນນີ້ JavaScript bundle ເລີ່ມຕົ້ນຂອງ chat widget ຖືກແຍກເປັນຊິ້ນສ່ວນທີ່ໂຫຼດແບບ lazy-loaded, ຫຼຸດຂະໜາດ bundle ເລີ່ມຕົ້ນລົງ 75–90%.
- **ອອກແບບ chat widget ໃໝ່** — ໄອຄອນ AI ແບບລວມເຂົ້າກັນແທນ avatar ກຳນົດເອງກ່ອນໜ້າ; ສອດຄ່ອງກັບລະບົບ agent ໃນຕົວ.
- **ການປ່ຽນ URL ເປັນລິ້ງ** — ຕອນນີ້ URL ທີ່ປາກົດໃນຂໍ້ຄວາມລະບົບ ແລະ ຟອງຂໍ້ຄວາມຜິດພາດ ຖືກສະແດງເປັນລິ້ງທີ່ຄລິກໄດ້.

### ແກ້ໄຂ

- **ການຄົ້ນພົບຄວາມສາມາດ** — ແກ້ໄຂຄຳອະທິບາຍ, ການອ້າງອີງ system prompt, ແລະ ການຈັດ namespace ໃຫ້ກົງກັນ ເພື່ອໃຫ້ຄວາມສາມາດທັງໝົດປາກົດໃນລາຍຊື່ເຄື່ອງມືຂອງ agent ໄດ້ຢ່າງເຊື່ອຖື.
- **cache ຂອງ providers** — ຕອນນີ້ providers ຖືກ cache ທົ່ວທັງເວັບໄຊຜ່ານຕົວນັບເວີຊັນ, ປ້ອງກັນບັນຫາ provider ຄ້າງໃນເຄືອຂ່າຍ multisite.
- **`ability_invalid_output`** — ແກ້ໄຂໃນ 12 ຕົວຈັດການຄວາມສາມາດ; ທັງໝົດສົ່ງຄືນຄຳຕອບ JSON ທີ່ມີໂຄງສ້າງຖືກຕ້ອງ.
- **pipeline `pending_client_tool_calls`** — ເຊື່ອມຕໍ່ຄົບຕັ້ງແຕ່ຕົ້ນຮອດທ້າຍ ເພື່ອໃຫ້ການເອີ້ນເຄື່ອງມືຝັ່ງ client ສຳເລັດຢ່າງຖືກຕ້ອງ ແລະສົ່ງຜົນກັບໄປຫາ model.
- **ລິ້ນຊັກປະຫວັດ** — ການປ່ຽນແປງທີ່ຍ້ອນກັບບໍ່ໄດ້ຖືກຕັດອອກຈາກລາຍຊື່ຍ້ອນກັບ; ຕອນນີ້ລິ້ງ **ເບິ່ງປະຫວັດເຕັມ** ເຮັດວຽກຖືກຕ້ອງ.
- **ລະບົບການປ່ຽນແປງ/ຍ້ອນກັບ** — ແກ້ໄຂບັກແຍກກັນຫ້າຢ່າງ ແລະຮວມໃຫ້ເປັນໜຶ່ງພາຍໃຕ້ອິນເຕີເຟດ admin ໃໝ່.
- **toast ບັນທຶກການຕັ້ງຄ່າ** — ຕອນນີ້ snackbar notification ປາກົດຢ່າງເຊື່ອຖືຫຼັງຈາກຄລິກ **ບັນທຶກການຕັ້ງຄ່າ**.
- **ເມນູບໍລິບົດຖັງຂີ້ເຫຍື້ອ** — ເພີ່ມຕົວເລືອກ **ລຶບຖາວອນ** ເພື່ອໃຫ້ລາຍການສາມາດຖືກລຶບແບບຖາວອນໄດ້ໂດຍບໍ່ຕ້ອງອອກຈາກມຸມມອງຖັງຂີ້ເຫຍື້ອ.
- **ແກ້ໄຂ ແລະ ສົ່ງອີກຄັ້ງ** — ຕອນນີ້ການຄລິກໄອຄອນແກ້ໄຂຈະເຂົ້າໂໝດແກ້ໄຂສຳລັບຂໍ້ຄວາມທີ່ຄລິກເທົ່ານັ້ນ, ບໍ່ແມ່ນທຸກຂໍ້ຄວາມໃນ thread.
- **ຄວາມສູງຂອງໂຄງຮ່າງແຊັດ** — ແຜງແຊັດປັບຄວາມສູງເມື່ອເນື້ອຫາທີ່ plugin ສອດແຊກເຂົ້າມາ (ແຈ້ງເຕືອນ admin, ແບນເນີ) ປາກົດຢູ່ເທິງໜ້າ, ປ້ອງກັນບໍລິເວນປ້ອນຂໍ້ມູນບໍ່ໃຫ້ຖືກດັນອອກນອກຈໍ.

## 1.4.0 — ປ່ອຍໃຊ້ໃນວັນທີ 2026-04-09

### ໃໝ່

- **ຄຳສັ່ງ benchmark ຂອງ WP-CLI** (`wp gratis-ai-agent benchmark`) — ເປີດໃຊ້ຊຸດ benchmark Agent Capabilities v1 ຈາກ command line ສຳລັບ CI pipelines ແລະ workflows ການປະເມີນ model. ຮອງຮັບການເປີດໃຊ້ແບບລາຍຄຳຖາມ, ການປ່ຽນແທນ provider/model, ແລະຜົນລັບ JSON/CSV.
- **ຊຸດ benchmark Agent Capabilities v1** — ຊຸດພ້ອມທີ່ມີໂຄງສ້າງຂອງ prompt ທີ່ຊັບຊ້ອນ ແລະຫຼາຍຂັ້ນຕອນ ເພື່ອທົດສອບຂອບເຂດຄວາມສາມາດທັງໝົດ, ພ້ອມການໃຫ້ຄະແນນ ແລະລາຍງານ token/ໄລຍະເວລາ.
- **ຄວາມສາມາດ Custom Post Type** — `register_post_type`, `list_post_types`, `delete_post_type`. ການລົງທະບຽນຈະຄົງຢູ່ຂ້າມການ restart ຜ່ານຕາຕະລາງ options.
- **ຄວາມສາມາດ Custom Taxonomy** — `register_taxonomy`, `list_taxonomies`, `delete_taxonomy`. ຮອງຮັບ taxonomies ແບບລຳດັບຊັ້ນ ແລະແບບແປ ພ້ອມ rewrite slugs ແບບເລືອກໄດ້.
- **ຄວາມສາມາດລະບົບການອອກແບບ** — `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset`. ລວມ preset ທີ່ຄັດເລືອກມາຫ້າອັນ: `minimal-dark`, `warm-editorial`, `corporate-blue`, `vibrant-startup`, `classic-blog`.
- **ຄວາມສາມາດ Global Styles** — `get_global_styles`, `set_global_styles`, `reset_global_styles`. ອ່ານ ແລະຂຽນຄ່າ theme.json ຜ່ານ WordPress Global Styles API.
- **ຄວາມສາມາດຈັດການເມນູນຳທາງ** — `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus`. ຮອງຮັບເມນູຊ້ອນກັນ ແລະການກຳນົດຕຳແໜ່ງ theme.
- **ຄວາມສາມາດຈັດການ Options** — `get_option`, `set_option`, `delete_option`, `list_options`. ລວມ blocklist ຄວາມປອດໄພໃນຕົວ ເພື່ອປົກປ້ອງ options WordPress ທີ່ສຳຄັນຈາກການຖືກແກ້ໄຂ.
- **ທະບຽນຄວາມສາມາດທີ່ຕິດຕັ້ງໄດ້** — `list_available_abilities`, `install_ability`, `recommend_plugin`. ຄົ້ນພົບ ແລະເປີດໃຊ້ ability packs ທີ່ແຈກຢາຍເປັນ WordPress plugins.
- **Site Builder Orchestration v2** — `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error`. ແຜນການສ້າງເວັບໄຊຫຼາຍໄລຍະ ພ້ອມການຄົ້ນພົບ plugin, ການອ້າງອີງຜົນລັບຂັ້ນຕອນ, ການຕິດຕາມຄວາມຄືບໜ້າ, ແລະການຟື້ນຟູຂໍ້ຜິດພາດແບບອັດຕະໂນມັດ.
- **ຄຳຖາມ benchmark ເວັບໄຊຮ້ານອາຫານ** (`q-restaurant-website`) — ການທົດສອບແບບ end-to-end ທີ່ຄອບຄຸມການລົງທະບຽນ CPT, ລະບົບການອອກແບບ, ການນຳທາງ, ແລະການຄົ້ນພົບ plugin.
- **plugins ຕົວເຊື່ອມຕໍ່ AI provider** ເພີ່ມເຂົ້າໃນ WordPress Playground blueprints ສຳລັບການຕັ້ງຄ່າພັດທະນາໃນເຄື່ອງທ້ອງຖິ່ນທີ່ໄວຂຶ້ນ.
