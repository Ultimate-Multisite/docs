# Ultimate AI Connector for WebLLM ບັນທຶກການປ່ຽນແປງ

## ເວີຊັນ 1.1.0 — ປ່ອຍໃຊ້ໃນວັນທີ 2026-04-09

* ໃໝ່: ວິດເຈັດແຊັດແບບລອຍພ້ອມຕົວຊີ້ສະຖານະໃນແຖບຜູ້ດູແລ — ຜູ້ໃຊ້ທີ່ລັອກອິນແລ້ວທຸກຄົນສາມາດສົ່ງຄຳສັ່ງໃຫ້ LLM ຝັ່ງເບຣາວເຊີໄດ້ໂດຍກົງຈາກດ້ານໜ້າ.
* ໃໝ່: SharedWorker runtime — ຕອນນີ້ເຄື່ອງຈັກ MLC ເຮັດວຽກໃນ SharedWorker ເພື່ອໃຫ້ຫຼາຍແຖບໃຊ້ GPU session ດຽວກັນ ແທນທີ່ຈະແຍ່ງກັນໃຊ້.
* ໃໝ່: apiFetch middleware interceptor — ຄຳຂໍ WordPress REST ທີ່ກົງກັບແພັດເທີນ AI Client SDK ຈະຖືກສົ່ງໄປຫາ WebLLM broker ແບບ local ຢ່າງໂປ່ງໃສ, ບໍ່ຕ້ອງມີການໄປ-ກັບ HTTP ແບບ loopback.
* ໃໝ່: UI ການຕັ້ງຄ່າວິດເຈັດໃນແຜງ Connector ສຳລັບເປີດປິດວິດເຈັດແຊັດ ແລະກຳນົດຄ່າພຶດຕິກຳຄຳສັ່ງອັດຕະໂນມັດ.
* ແກ້ໄຂ: ບັງຄັບໃຊ້ IndexedDB cache backend ເພື່ອໃຫ້ການດາວໂຫຼດນ້ຳໜັກໂມເດວຢູ່ລອດຫຼັງການປ່ຽນທາງ HuggingFace xet CDN ທີ່ເຮັດໃຫ້ເສັ້ນທາງ Cache API ຄ່າເລີ່ມຕົ້ນເສຍ.
* ແກ້ໄຂ: ຂ້າມການ override context_window KV-cache ສຳລັບໂມເດວ embedding (ພວກມັນບໍ່ມີ decoder ແລະ override ເຮັດໃຫ້ເກີດ runtime error).
* ແກ້ໄຂ: ໂຄສະນາໂມເດວຜູ້ສະໝັກເລີ່ມຕົ້ນເຢັນໃນ /webllm/v1/models ກ່ອນແຖບ worker ຈະໂຫຼດແລ້ວ, ເພື່ອໃຫ້ຜູ້ບໍລິໂພກ SDK ເຫັນໂມເດວທັນທີ.
* ແກ້ໄຂ: ຜູກ WebLLM ເຂົ້າກັບຟິລເຕີ wpai_preferred_text_models ເພື່ອໃຫ້ຟີເຈີ AI Experiments ສົ່ງເສັ້ນທາງໄປຫາເຄື່ອງຈັກເບຣາວເຊີເມື່ອມີການກຳນົດຄ່າໂມເດວໄວ້.
* ປັບປຸງ: ການລ້າງແຄຊ, ການປັບເນື້ອຫາໃຫ້ເປັນມາດຕະຖານ, ແລະການແກ້ໄຂການອ້າງອີງຮາດແວທີ່ປາກົດໃນລະຫວ່າງການທົດສອບແບບຈາກຕົ້ນຈົນຈົບ.
