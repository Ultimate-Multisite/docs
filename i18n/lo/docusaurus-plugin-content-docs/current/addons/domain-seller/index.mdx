# ຜູ້ຂາຍໂດເມນ

**Multisite Ultimate: Domain Seller** ປ່ຽນເຄືອຂ່າຍ WordPress Multisite ຂອງທ່ານໃຫ້ເປັນແພລດຟອມຂາຍໂດເມນທີ່ມີພະລັງ. ໃຫ້ລູກຄ້າລົງທະບຽນໂດເມນກຳນົດເອງໄດ້ໂດຍກົງລະຫວ່າງການລົງຊື່ ຫຼືຈາກ Dashboard ຂອງພວກເຂົາ, ພ້ອມການກວດສອບຄວາມພ້ອມໃຊ້ງານແບບເວລາຈິງ ແລະລາຄາທີ່ໂປ່ງໃສ.

### ເປັນຫຍັງຈຶ່ງໃຊ້ Domain Seller? {#why-domain-seller}

ການລົງທະບຽນໂດເມນແມ່ນໜຶ່ງໃນການຂາຍເພີ່ມທີ່ຖືກຮ້ອງຂໍຫຼາຍທີ່ສຸດສຳລັບແພລດຟອມ WaaS (ເວັບໄຊເປັນບໍລິການ) ໃດໆ. ແທນທີ່ຈະສົ່ງລູກຄ້າຂອງທ່ານໄປຫາຜູ້ຈົດທະບຽນພາກສ່ວນທີສາມ, ທ່ານສາມາດສະເໜີໂດເມນໄດ້ໂດຍກົງ — ເກັບລາຍຮັບໄວ້ໃນເຄືອຂ່າຍຂອງທ່ານໄດ້ຫຼາຍຂຶ້ນ ພ້ອມກັບມອບປະສົບການທີ່ລື່ນໄຫຼໃຫ້ລູກຄ້າ.

### ຄຸນສົມບັດຫຼັກ {#key-features}

#### ຮອງຮັບຜູ້ຈົດທະບຽນໂດເມນຫຼາຍລາຍ {#multiple-domain-registrar-support}
* **OpenSRS** — ການລົງທະບຽນ, ການຕໍ່ອາຍຸ ແລະການຈັດການແບບຄົບຖ້ວນ
* **Namecheap** — ການຜະສານລວມທີ່ສົມບູນກັບ WhoisGuard ແລະການທົດສອບ sandbox
* **HostAfrica** — ການຄົ້ນຫາໂດເມນ, ການຕັ້ງລາຄາ, ການລົງທະບຽນ, ການໂອນຍ້າຍ, DNS ແລະການປົກປ້ອງ ID ຜ່ານ Domains Reseller API
* **Openprovider** — ລາຄາຕົວແທນຈຳໜ່າຍ, ການລົງທະບຽນ, ການຕໍ່ອາຍຸ, ການໂອນຍ້າຍ, DNS, ຄວາມເປັນສ່ວນຕົວ WHOIS ແລະການຊິງຄ໌ TLD
* **Hostinger** — ການຄົ້ນຫາໂດເມນ ແລະການລົງທະບຽນໂດຍໃຊ້ໂທເຄັນການຜະສານລວມ Hostinger ແບບແບ່ງປັນ
* **ສະຖາປັດຕະຍະກຳທີ່ຂະຫຍາຍໄດ້** — ສ້າງຢູ່ເທິງຮູບແບບ Integration Registry, ເຮັດໃຫ້ເພີ່ມຜູ້ໃຫ້ບໍລິການໄດ້ງ່າຍ
* **ການທົດສອບການເຊື່ອມຕໍ່ໃນຕົວ** — ກວດຢືນຢັນຂໍ້ມູນຮັບຮອງ ແລະການເຊື່ອມຕໍ່ຂອງຜູ້ໃຫ້ບໍລິການກ່ອນເປີດໃຊ້ງານຈິງ

#### ຕົວຊ່ວຍຕັ້ງຄ່າແບບນຳທາງ {#guided-setup-wizard}
* **ການເລີ່ມໃຊ້ງານຄລິກດຽວ** — ນຳທ່ານຜ່ານການກຳນົດຄ່າຜູ້ໃຫ້ບໍລິການ, ການນຳເຂົ້າ TLD ແລະການສ້າງຜະລິດຕະພັນໂດເມນ
* **ການຊິງຄ໌ TLD ອັດຕະໂນມັດ** — ນຳເຂົ້າ TLD ທີ່ພ້ອມໃຊ້ງານທັງໝົດ ແລະລາຄາຂາຍສົ່ງຈາກຜູ້ໃຫ້ບໍລິການທີ່ທ່ານເຊື່ອມຕໍ່ໄວ້
* **ການສ້າງຜະລິດຕະພັນເລີ່ມຕົ້ນ** — ສ້າງຜະລິດຕະພັນໂດເມນທີ່ພ້ອມໃຊ້ງານ ພ້ອມຄ່າເລີ່ມຕົ້ນການບວກລາຄາທີ່ເໝາະສົມ

#### ການຕັ້ງລາຄາໂດເມນທີ່ຍືດຫຍຸ່ນ {#flexible-domain-pricing}
* **ສາມໂໝດການບວກລາຄາ** — ການບວກລາຄາແບບເປີເຊັນ, ການບວກລາຄາຄົງທີ່ ຫຼືລາຄາຄົງທີ່ຕໍ່ TLD
* **ຜະລິດຕະພັນສະເພາະ TLD** — ສ້າງຜະລິດຕະພັນໂດເມນແຍກຕ່າງຫາກສຳລັບກຸ່ມ TLD ທີ່ແຕກຕ່າງກັນ (.com/.net, .io, ລະຫັດປະເທດ ແລະອື່ນໆ)
* **ລາຄາແນະນຳເບື້ອງຕົ້ນ** — ສະເໜີລາຄາປີທຳອິດແບບຫຼຸດລາຄາເພື່ອດຶງດູດການລົງທະບຽນໃໝ່
* **ລາຄາແບບເວລາຈິງ** — ລາຄາຂາຍສົ່ງສົດຈາກຜູ້ໃຫ້ບໍລິການ ພ້ອມການແຄຊທີ່ສະຫຼາດ
* **ຮອງຮັບຄູປອງ** — ເຂົ້າກັນໄດ້ເຕັມທີ່ກັບລະຫັດສ່ວນຫຼຸດ Multisite Ultimate

#### ການປົກປ້ອງຄວາມເປັນສ່ວນຕົວ WHOIS {#whois-privacy-protection}
* **ການກຳນົດຄ່າຕໍ່ຜະລິດຕະພັນ** — ຕັ້ງຄ່າຄວາມເປັນສ່ວນຕົວ WHOIS ເປັນ ປິດໃຊ້ງານ, ຮວມໄວ້ສະເໝີ ຫຼື ລູກຄ້າເລືອກ
* **ລູກຄ້າເລືອກເຂົ້າໃຊ້** — ເມື່ອຕັ້ງເປັນ ລູກຄ້າເລືອກ, ຊ່ອງກາໝາຍໃນການຊຳລະເງິນຈະໃຫ້ລູກຄ້າເພີ່ມການປົກປ້ອງຄວາມເປັນສ່ວນຕົວໄດ້
* **ລາຄາທີ່ກຳນົດຄ່າໄດ້** — ຕັ້ງລາຄາຕໍ່ປີສຳລັບສ່ວນເສີມຄວາມເປັນສ່ວນຕົວ ຫຼືສະເໜີໃຫ້ຟຣີ
* **ການຜະສານລວມກັບຜູ້ຈົດທະບຽນ** — ເປີດໃຊ້ WhoisGuard (Namecheap) ຫຼືຄວາມເປັນສ່ວນຕົວ WHOIS (OpenSRS) ອັດຕະໂນມັດໃນເວລາລົງທະບຽນ

#### ປະສົບການຊຳລະເງິນແບບລວມສູນ {#unified-checkout-experience}
* **ສາມໂໝດໂດເມນ** — ລູກຄ້າເລືອກລະຫວ່າງໂດເມນຍ່ອຍ (ຟຣີ), ການລົງທະບຽນໂດເມນໃໝ່ ຫຼືການແມບໂດເມນທີ່ມີຢູ່ແລ້ວທີ່ພວກເຂົາເປັນເຈົ້າຂອງ
* **ການຄົ້ນຫາໂດເມນແບບເວລາຈິງ** — ການກວດສອບຄວາມພ້ອມໃຊ້ງານທີ່ຂັບເຄື່ອນດ້ວຍ AJAX ໂດຍບໍ່ຕ້ອງໂຫຼດໜ້າຄືນ
* **ຄຳແນະນຳທີ່ສະຫຼາດ** — ຄຳແນະນຳ TLD ທາງເລືອກເມື່ອໂດເມນທີ່ຕ້ອງການຖືກໃຊ້ແລ້ວ
* **ລາຄາທີ່ໂປ່ງໃສ** — ສະແດງລາຄາລົງທະບຽນ, ລາຄາຕໍ່ອາຍຸ ແລະຄ່າທຳນຽມຄວາມເປັນສ່ວນຕົວ WHOIS ທີ່ເປັນທາງເລືອກຢ່າງຊັດເຈນ
* **ຊ່ອງຂໍ້ມູນຕິດຕໍ່ຜູ້ລົງທະບຽນ** — ເກັບຂໍ້ມູນຜູ້ລົງທະບຽນທີ່ຈຳເປັນ (ຊື່, ທີ່ຢູ່, ໂທລະສັບ) ໃນໜ້າຊຳລະເງິນໂດຍກົງ
* **URL ເວັບໄຊທີ່ສ້າງອັດຕະໂນມັດ** — ທີ່ຢູ່ເວັບໄຊຖືກສ້າງຂຶ້ນອັດຕະໂນມັດຈາກໂດເມນທີ່ເລືອກ

#### ລະບົບຕໍ່ອາຍຸອັດຕະໂນມັດ {#auto-renewal-system}
* **ການຕໍ່ອາຍຸທີ່ຜູກກັບສະມາຊິກພາບ** — ໂດເມນຕໍ່ອາຍຸອັດຕະໂນມັດໃນຂະນະທີ່ສະມາຊິກພາບຂອງລູກຄ້າຍັງເປີດໃຊ້ຢູ່
* **ການຈັດກຳນົດການທີ່ສະຫຼາດ** — ງານຕໍ່ອາຍຸຖືກຈັດຄິວຕາມວັນໝົດອາຍຸຂອງໂດເມນ
* **ຕັກກະການລອງໃໝ່** — ການລອງໃໝ່ທີ່ສະຫຼາດສຳລັບຄວາມພະຍາຍາມຕໍ່ອາຍຸທີ່ລົ້ມເຫຼວ
* **ການແຈ້ງເຕືອນອີເມວ** — ການແຈ້ງເຕືອນທີ່ກຳນົດຄ່າໄດ້ສຳລັບການຕໍ່ອາຍຸທີ່ຈະມາເຖິງ, ການຕໍ່ອາຍຸສຳເລັດ, ຄວາມລົ້ມເຫຼວ ແລະການໝົດອາຍຸ

#### ການຈັດການ DNS ຂອງລູກຄ້າ {#customer-dns-management}
* **DNS ແບບບໍລິການຕົນເອງ** — ລູກຄ້າສາມາດເພີ່ມ, ແກ້ໄຂ ແລະລຶບບັນທຶກ DNS (A, AAAA, CNAME, MX, TXT) ຈາກ Dashboard ຂອງພວກເຂົາ
* **ສະຖານະໂດເມນ** — ສະຖານະການລົງທະບຽນແບບເວລາຈິງ, ວັນໝົດອາຍຸ ແລະຂໍ້ມູນ nameserver
* **ການແມບໂດເມນອັດຕະໂນມັດ** — ໂດເມນທີ່ລົງທະບຽນແລ້ວ ຫຼືທີ່ມີຢູ່ແລ້ວຈະຖືກແມບໄປຫາເວັບໄຊຂອງລູກຄ້າອັດຕະໂນມັດ

#### ເຄື່ອງມືຜູ້ດູແລ {#admin-tools}
* **ການລົງທະບຽນໂດເມນດ້ວຍຕົນເອງ** — ລົງທະບຽນໂດເມນແທນລູກຄ້າຈາກແຜງຜູ້ດູແລ
* **ການຕິດຕາມໂດເມນ** — ເບິ່ງໂດເມນ, ສະຖານະ ແລະວັນໝົດອາຍຸຂອງລູກຄ້າທັງໝົດຈາກໜ້າດຽວ
* **ການບັນທຶກລັອກສະເພາະຜູ້ໃຫ້ບໍລິການ** — ຊ່ອງລັອກແຍກຕ່າງຫາກສຳລັບຜູ້ຈົດທະບຽນແຕ່ລະລາຍ (ເຊັ່ນ domain-seller-namecheap.log)
* **ລາຍລະອຽດການລົງທະບຽນ** — ເບິ່ງຂໍ້ມູນຜູ້ລົງທະບຽນ ແລະຂໍ້ມູນການລົງທະບຽນໃນໜ້າແກ້ໄຂໂດເມນຫຼັກ

### ຜູ້ໃຫ້ບໍລິການທີ່ຮອງຮັບ {#supported-providers}

* **OpenSRS** — ການຜະສານລວມແບບຄົບຖ້ວນກັບການລົງທະບຽນ, ການຕໍ່ອາຍຸ, ການຈັດການ DNS ແລະຄວາມເປັນສ່ວນຕົວ WHOIS
* **Namecheap** — ການຜະສານລວມແບບຄົບຖ້ວນກັບການລົງທະບຽນ, ການຕໍ່ອາຍຸ, ການກວດສອບຄວາມພ້ອມໃຊ້ງານແບບຊຸດ ແລະ WhoisGuard ຟຣີ
* **HostAfrica** — ການລົງທະບຽນ, ການຕໍ່ອາຍຸ, ການໂອນຍ້າຍ, ການຈັດການ DNS, ການດຶງລະຫັດ EPP, registrar lock ແລະການປົກປ້ອງ ID
* **Openprovider** — ການລົງທະບຽນ, ການຕໍ່ອາຍຸ, ການໂອນຍ້າຍ, ການຈັດການ DNS, ການດຶງລະຫັດ EPP, registrar lock, ຄວາມເປັນສ່ວນຕົວ WHOIS ແລະການຊິງຄ໌ TLD ຂອງຕົວແທນຈຳໜ່າຍ
* **Hostinger** — ການລົງທະບຽນ, ການຄົ້ນຫາໂດເມນ, ການອັບເດດ nameserver, registrar lock ແລະຄວາມເປັນສ່ວນຕົວ WHOIS; ການຕໍ່ອາຍຸຖືກຈັດການໂດຍການຕໍ່ອາຍຸອັດຕະໂນມັດຂອງ Hostinger
* **GoDaddy** — ການລົງທະບຽນ ແລະການຕໍ່ອາຍຸ
* **ResellerClub** — ການລົງທະບຽນ, ການຕໍ່ອາຍຸ ແລະການຈັດການ DNS
* **NameSilo** — ການລົງທະບຽນ ແລະການຕໍ່ອາຍຸ
* **Enom** — ການລົງທະບຽນ, ການຕໍ່ອາຍຸ ແລະການຈັດການ DNS

### ລາຍລະອຽດທາງເຕັກນິກ {#technical-details}

* **ຕ້ອງການ:** PHP 7.4+, Multisite Ultimate 2.4.12+
* **ສະຖາປັດຕະຍະກຳ:** ຮູບແບບ Integration Registry ພ້ອມອິນເຕີເຟດ Domain_Selling_Capability
* **ການໂຫຼດອັດຕະໂນມັດ:** Jetpack autoloader ພ້ອມການແມບຄລາສ PSR-4
* **ມາດຕະຖານ:** ມາດຕະຖານການຂຽນໂຄດ WordPress, ການກຳນົດປະເພດແບບເຂັ້ມງວດ, prepared statements

## ການຕິດຕັ້ງ {#installation}

1. ໃຫ້ແນ່ໃຈວ່າ **Multisite Ultimate v2.4.12+** ຖືກຕິດຕັ້ງ ແລະເປີດໃຊ້ງານໃນລະດັບເຄືອຂ່າຍແລ້ວ
2. ອັບໂຫຼດປັກອິນໄປທີ່ `/wp-content/plugins/ultimate-multisite-domain-seller/`
3. ເປີດໃຊ້ງານປັກອິນໃນລະດັບເຄືອຂ່າຍຈາກ **ປັກອິນ** ໃນ WordPress Network Admin
4. **ຕົວຊ່ວຍຕັ້ງຄ່າ** ຈະເປີດຂຶ້ນອັດຕະໂນມັດໃນການເປີດໃຊ້ງານຄັ້ງທຳອິດ — ໃຫ້ເຮັດຕາມຂັ້ນຕອນເພື່ອ:
   * ເຊື່ອມຕໍ່ຜູ້ຈົດທະບຽນໂດເມນຂອງທ່ານ
   * ທົດສອບການເຊື່ອມຕໍ່
   * ນຳເຂົ້າ TLD ທີ່ມີໃຫ້ໃຊ້ ແລະລາຄາຂາຍສົ່ງ
   * ສ້າງຜະລິດຕະພັນໂດເມນທຳອິດຂອງທ່ານພ້ອມການບວກກຳໄລລາຄາ
5. ເພີ່ມຊ່ອງຂໍ້ມູນ **ການເລືອກໂດເມນ** ໃສ່ແບບຟອມຊຳລະເງິນຂອງທ່ານໃນ **Ultimate Multisite > ແບບຟອມຊຳລະເງິນ**

## ຄຳຖາມທີ່ພົບເລື້ອຍ {#frequently-asked-questions}

### ຕ້ອງໃຊ້ Multisite Ultimate ເວີຊັນໃດ? {#what-version-of-multisite-ultimate-is-required}

ເວີຊັນ 2.4.12 ຫຼືສູງກວ່າ. ສ່ວນເສີມນີ້ບໍ່ຮອງຮັບ WP Ultimo v1.x ຫຼື Ultimate Multisite ເວີຊັນຕ່ຳກວ່າ 2.4.12.

### ຜູ້ໃຫ້ບໍລິການໂດເມນໃດແດ່ທີ່ຮອງຮັບ? {#which-domain-providers-are-supported}

ຮອງຮັບຜູ້ຈົດທະບຽນ 9 ລາຍ: **OpenSRS**, **Namecheap**, **HostAfrica**, **Openprovider**, **Hostinger**, **GoDaddy**, **ResellerClub**, **NameSilo**, ແລະ **Enom**. ຄວາມສາມາດຈະແຕກຕ່າງກັນຕາມຜູ້ໃຫ້ບໍລິການ ເພາະ API ຂອງຜູ້ຈົດທະບຽນເປີດໃຫ້ໃຊ້ການດຳເນີນການທີ່ຕ່າງກັນ; ເບິ່ງຄູ່ມືການຕັ້ງຄ່າສຳລັບພາບລວມແບບແຍກຕາມຜູ້ໃຫ້ບໍລິການ. ສະຖາປັດຕະຍະກຳ Integration Registry ຂອງສ່ວນເສີມເຮັດໃຫ້ການເພີ່ມຜູ້ໃຫ້ບໍລິການແບບກຳນົດເອງເຮັດໄດ້ງ່າຍ — ເບິ່ງ [ຄູ່ມືນັກພັດທະນາ](/developer/integration-guide/custom-registrar).

### ຂ້ອຍສາມາດສະເໜີການປົກປ້ອງຄວາມເປັນສ່ວນຕົວ WHOIS ໄດ້ບໍ? {#can-i-offer-whois-privacy-protection}

ໄດ້. ແຕ່ລະຜະລິດຕະພັນໂດເມນມີການຕັ້ງຄ່າ WHOIS Privacy ພ້ອມ 3 ໂໝດ: ປິດໃຊ້ງານ, ລວມໃຫ້ສະເໝີ, ຫຼື ລູກຄ້າເລືອກ. ເມື່ອຕັ້ງເປັນລູກຄ້າເລືອກ, ລູກຄ້າຈະເຫັນກ່ອງເລືອກລະຫວ່າງການຊຳລະເງິນພ້ອມລາຄາ (ຫຼື "ຟຣີ" ສຳລັບ WhoisGuard ຂອງ Namecheap). ສັນຍາລັກຄວາມເປັນສ່ວນຕົວຈະຖືກສົ່ງໄປຫາຜູ້ຈົດທະບຽນອັດຕະໂນມັດໃນເວລາຈົດທະບຽນ.

### ຂ້ອຍສາມາດຕັ້ງລາຄາຕ່າງກັນສຳລັບ TLD ທີ່ຕ່າງກັນໄດ້ບໍ? {#can-i-set-different-pricing-for-different-tlds}

ໄດ້. ສ້າງຜະລິດຕະພັນໂດເມນແຍກຕ່າງຫາກສຳລັບກຸ່ມ TLD ທີ່ຕ່າງກັນ. ແຕ່ລະຜະລິດຕະພັນສາມາດມີປະເພດການບວກກຳໄລຂອງຕົນເອງ (ເປີເຊັນ, ການບວກກຳໄລຄົງທີ່, ຫຼື ລາຄາຄົງທີ່), TLD ທີ່ຮອງຮັບ, ແລະລາຄາແນະນຳເບື້ອງຕົ້ນ.

### ລູກຄ້າຈະເຫັນຕົວເລືອກການຊຳລະເງິນໃດແດ່? {#what-checkout-options-do-customers-see}

ຊ່ອງຂໍ້ມູນການເລືອກໂດເມນໃຫ້ແທັບ 3 ຢ່າງແກ່ລູກຄ້າ: **ໂດເມນຍ່ອຍ** (ໃຊ້ໂດເມນຍ່ອຍຟຣີໃນເຄືອຂ່າຍຂອງທ່ານ), **ຈົດທະບຽນໂດເມນໃໝ່** (ຄົ້ນຫາ ແລະຈົດທະບຽນໂດເມນໃໝ່), ແລະ **ໂດເມນທີ່ມີຢູ່ແລ້ວ** (ເຊື່ອມແຜນທີ່ໂດເມນທີ່ພວກເຂົາມີຢູ່ແລ້ວ). ຈະສະແດງພຽງແຕ່ໂໝດທີ່ທ່ານເປີດໃຊ້ງານເທົ່ານັ້ນ.

### ຮອງຮັບການຕໍ່ອາຍຸອັດຕະໂນມັດບໍ? {#does-it-support-automatic-renewals}

ໄດ້. ໂດເມນສາມາດຕັ້ງໃຫ້ຕໍ່ອາຍຸອັດຕະໂນມັດໄດ້ ແລະຈະຜູກກັບສະຖານະສະມາຊິກຂອງລູກຄ້າ. ການຊຳລະເງິນຄ່າຕໍ່ອາຍຸຈະຖືກປະມວນຜົນຜ່ານຊ່ອງທາງຊຳລະເງິນທີ່ລູກຄ້າໃຊ້ຢູ່ແລ້ວ.

### ລູກຄ້າສາມາດຈັດການບັນທຶກ DNS ຂອງຕົນໄດ້ບໍ? {#can-customers-manage-their-dns-records}

ໄດ້. ລູກຄ້າຈະໄດ້ຮັບອິນເຕີເຟດຈັດການ DNS ໃນ Dashboard ຂອງພວກເຂົາ ເພື່ອເພີ່ມ, ແກ້ໄຂ, ແລະລຶບບັນທຶກ A, AAAA, CNAME, MX, ແລະ TXT. ການຈັດການ DNS ມີໃຫ້ໃຊ້ກັບ OpenSRS, ResellerClub, Enom, HostAfrica, ແລະ Openprovider. DNS ຂອງ Hostinger ສຳລັບໂດເມນທີ່ຖືກໂຮສໄວ້ ຈະຖືກຈັດການໂດຍການຜະສານງານການເຊື່ອມແຜນທີ່ໂດເມນຫຼັກຂອງ Hostinger.

### ມີຕົວຊ່ວຍຕັ້ງຄ່າບໍ? {#is-there-a-setup-wizard}

ມີ. ຕົວຊ່ວຍຕັ້ງຄ່າຈະເຮັດວຽກອັດຕະໂນມັດໃນການເປີດໃຊ້ງານຄັ້ງທຳອິດ ແລະພາທ່ານຜ່ານຂັ້ນຕອນການເຊື່ອມຕໍ່ຜູ້ຈົດທະບຽນ, ນຳເຂົ້າ TLD, ແລະສ້າງຜະລິດຕະພັນໂດເມນທຳອິດຂອງທ່ານ. ທ່ານຍັງສາມາດເປີດໃຊ້ຕົວຊ່ວຍນີ້ອີກຄັ້ງຈາກການຕັ້ງຄ່າສ່ວນເສີມໄດ້.
