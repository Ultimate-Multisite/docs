# Mailchimp

Multisite Ultimate: ການເຊື່ອມຕໍ່ Mailchimp

ເຊື່ອມຕໍ່ເຄືອຂ່າຍ Multisite Ultimate ຂອງທ່ານກັບ Mailchimp ຂອງທ່ານ. ທ່ານສາມາດເຊື່ອມບາງແຜນກັບລາຍຊື່ Mailchimp ແລະເພີ່ມຜູ້ສະໝັກເຂົ້າໃນລາຍຊື່ເຫຼົ່ານັ້ນໂດຍອັດຕະໂນມັດເມື່ອລົງທະບຽນ.

## ການຕິດຕັ້ງ

1. ອັບໂຫຼດ 'ultimate-multisite-mailchimp' ໄປຍັງໄດເຣັກທໍຣີ '/wp-content/plugins/'
2. ເປີດໃຊ້ປລັກອິນຜ່ານເມນູ 'ປລັກອິນ' ໃນ WordPress ຜູ້ຄຸ້ມຄອງເຄືອຂ່າຍຂອງທ່ານ

## ຄູ່ມືການໃຊ້ງານ

ການຊິງອີເມວຂອງຜູ້ສະໝັກກັບລາຍຊື່ MailChimp ຂອງທ່ານເຮັດໄດ້ງ່າຍຫຼາຍດ້ວຍສ່ວນເສີມ Ultimate Multisite: MailChimp Integration ຂອງພວກເຮົາ. ບົດແນະນຳນີ້ມີເປົ້າໝາຍຊ່ວຍໃຫ້ທ່ານຕັ້ງຄ່າທຸກຢ່າງໃຫ້ພ້ອມ!

## ການຮັບ API key

ກ່ອນທີ່ພວກເຮົາຈະເລີ່ມເຂົ້າໄປເບິ່ງການຕັ້ງຄ່າຕ່າງໆທີ່ມີ, ພວກເຮົາຕ້ອງມີ MailChimp API key. ນີ້ຈະຊ່ວຍໃຫ້ Ultimate Multisite ສາມາດສື່ສານກັບ MailChimp API ເພື່ອດຶງລາຍຊື່, ກຸ່ມ, ແລະເພີ່ມທີ່ຢູ່ອີເມວເຂົ້າໃນລາຍຊື່ແລະກຸ່ມເຫຼົ່ານັ້ນເມື່ອຜູ້ໃຊ້ລົງທະບຽນ.

ເຮັດຕາມບົດແນະນຳນີ້ເພື່ອຮັບ MailChimp API key ຂອງທ່ານເອງ: [ກ່ຽວກັບ API Keys | MailChimp](https://kb.mailchimp.com/integrations/api-integrations/about-api-keys)

## ໂໝດການເຊື່ອມຕໍ່

ສ່ວນເສີມ MailChimp Integration ມີ “ໂໝດການເຊື່ອມຕໍ່” ສອງແບບ, ໝາຍຄວາມວ່າມີສອງວິທີທີ່ແຕກຕ່າງກັນທີ່ທ່ານສາມາດໃຊ້ສ່ວນເສີມເພື່ອຊິງທີ່ຢູ່ອີເມວ.

<!-- Screenshot unavailable: MailChimp integration mode selection with Multiple Lists and Single List options -->

_ສອງໂໝດການເຊື່ອມຕໍ່ເພື່ອຄວາມຍືດຫຍຸ່ນທີ່ຫຼາຍຂຶ້ນ!_

### ໂໝດ 1: ຫຼາຍລາຍຊື່

ໂໝດເລີ່ມຕົ້ນໃຊ້ **ຫຼາຍລາຍຊື່**. ໂໝດນີ້ຈະໃຫ້ທ່ານເລືອກລາຍຊື່ MailChimp ຫຼາຍລາຍຊື່ສຳລັບແຕ່ລະແຜນໄດ້. ນີ້ເປັນຕົວເລືອກທີ່ດີທີ່ສຸດຖ້າທ່ານແບ່ງກຸ່ມຜູ້ໃຊ້ຂອງທ່ານໄວ້ໃນລາຍຊື່ທີ່ແຕກຕ່າງກັນ.

ຕົວຢ່າງ, ສົມມຸດວ່າທ່ານມີ 3 ລາຍຊື່ທີ່ແຕກຕ່າງກັນໃນ Account MailChimp ຂອງທ່ານ: **ຈົດໝາຍຂ່າວ** , **ແຜນ A** , ແລະ **ແຜນ B**. ທ່ານຕ້ອງການໃຫ້ຜູ້ໃຊ້ທັງໝົດ, ບໍ່ວ່າແຜນຂອງເຂົາເຈົ້າຈະເປັນຫຍັງ, ຖືກລວມເຂົ້າໃນລາຍຊື່ຈົດໝາຍຂ່າວນຳ, ເພື່ອບໍ່ໃຫ້ພາດຂ່າວອັບເດດຂອງທ່ານ.

ເພື່ອບັນລຸສິ່ງນັ້ນ, ທ່ານສາມາດໃຊ້ໂໝດຫຼາຍລາຍຊື່ເພື່ອເພີ່ມທັງລາຍຊື່ **ຈົດໝາຍຂ່າວ** ແລະ **ແຜນ A** ເຂົ້າໃນການຕັ້ງຄ່າ MailChimp ຂອງແຜນ A, ແລະ **ຈົດໝາຍຂ່າວ** ແລະ **ແຜນ B** ເຂົ້າໃນການຕັ້ງຄ່າ MailChimp ຂອງແຜນ B.

### ໂໝດ 2: ລາຍຊື່ດຽວ + ຫຼາຍກຸ່ມ

MailChimp ຊ່ວຍໃຫ້ທ່ານແບ່ງລາຍຊື່ຂອງທ່ານເປັນກຸ່ມຜູ້ໃຊ້ຂະໜາດນ້ອຍກວ່າທີ່ເອີ້ນວ່າ **ກຸ່ມ**. ນີ້ຊ່ວຍໃຫ້ທ່ານຄວບຄຸມໄດ້ຫຼາຍຂຶ້ນ ເພາະທ່ານສາມາດສົ່ງແຄມເປນໄປຫາກຸ່ມຍ່ອຍສະເພາະຂອງຜູ້ສະໝັກຂອງທ່ານໄດ້ ໂດຍບໍ່ຕ້ອງສ້າງສຳເນົາຫຼາຍຊຸດຂອງແຄມເປນດຽວກັນໃນ MailChimp (ເຊິ່ງເຮັດໃຫ້ວິເຄາະຕົວຊີ້ວັດຂອງແຄມເປນນັ້ນພາຍຫຼັງໄດ້ຍາກ).

ຕົວຢ່າງ, ທ່ານສາມາດມີລາຍຊື່ດຽວຊື່ **ຜູ້ສະໝັກໃນເຄືອຂ່າຍຂອງຂ້ອຍ,** ແລະພາຍໃນນັ້ນມີກຸ່ມສຳລັບ **ແຜນ A** ແລະ **ແຜນ B**.

_ເລືອກລາຍຊື່ແລະກຸ່ມເລີ່ມຕົ້ນ, ຖ້າໃຊ້ໂໝດການເຊື່ອມຕໍ່ “ລາຍຊື່ດຽວ + ຫຼາຍກຸ່ມ”_

## ລາຍຊື່ແລະກຸ່ມເລີ່ມຕົ້ນ

ຫຼັງຈາກທ່ານເລືອກໂໝດການເຊື່ອມຕໍ່ທີ່ເໝາະສົມກັບກໍລະນີການໃຊ້ສະເພາະຂອງທ່ານຫຼາຍກວ່າແລ້ວ, ກໍຮອດເວລາເລືອກລາຍຊື່ເລີ່ມຕົ້ນ (ແລະກຸ່ມ, ຂຶ້ນກັບໂໝດ). ລາຍຊື່ເລີ່ມຕົ້ນຈະຖືກໃຊ້ເມື່ອແຜນທີ່ກຳນົດໃດໜຶ່ງບໍ່ໄດ້ມີລາຍຊື່ສະເພາະຖືກເລືອກໄວ້.

ທ່ານຍັງສາມາດເລືອກລາຍຊື່ເລີ່ມຕົ້ນສຳລັບ Account ທີ່ຖືກຍົກເລີກໄດ້. ເມື່ອໃດກໍຕາມທີ່ຜູ້ໃຊ້ຢຸດການສະໝັກຂອງຕົນ ແລະລຶບ Account ຂອງຕົນອອກຈາກເຄືອຂ່າຍຂອງທ່ານ, ທີ່ຢູ່ອີເມວຂອງລາວຈະຖືກຍ້າຍໄປຍັງລາຍຊື່ທີ່ຕັ້ງໄວ້ສຳລັບ Account ທີ່ຖືກຍົກເລີກ.

## ການເປີດໃຊ້ການສະໝັກແບບຄູ່

ໃນໜ້າການຕັ້ງຄ່າດຽວກັນ, ທ່ານຈະສາມາດເປີດໃຊ້ **ການສະໝັກແບບຄູ່**. ຕົວເລືອກນີ້ຈະເພີ່ມຜູ້ໃຊ້ເຂົ້າໄປທັງໃນລາຍຊື່ເລີ່ມຕົ້ນທີ່ເລືອກໄວ້ ແລະລາຍຊື່ທີ່ເລືອກໄວ້ສຳລັບແຜນຂອງຜູ້ໃຊ້ນັ້ນ. ຖ້າບໍ່ໄດ້ເປີດໃຊ້ຕົວເລືອກນີ້, ຜູ້ໃຊ້ຈະຖືກເພີ່ມເຂົ້າສະເພາະໃນລາຍຊື່ທີ່ເລືອກໄວ້ສຳລັບແຜນຂອງຜູ້ໃຊ້ເທົ່ານັ້ນ. ຖ້າບໍ່ມີການຕັ້ງຄ່າໄວ້ສຳລັບແຜນນັ້ນໂດຍສະເພາະ, ຜູ້ໃຊ້ຈະຖືກເພີ່ມເຂົ້າໃນລາຍຊື່ເລີ່ມຕົ້ນທີ່ເລືອກໄວ້.

## ການເລືອກລາຍຊື່ແລະກຸ່ມສຳລັບແຕ່ລະແຜນ

ໄປທີ່ **ແຜນ** ຂອງທ່ານ ແລະເລືອກແຜນທີ່ທ່ານຕ້ອງການແກ້ໄຂ. ໃຫ້ສັງເກດວ່າໃນ meta-box **ຕົວເລືອກຂັ້ນສູງ**, ແທັບໃໝ່ທີ່ຊື່ MailChimp ຈະມີໃຫ້ໃຊ້.

ໃນແທັບນັ້ນ, ທ່ານຈະສາມາດເລືອກຫຼາຍລາຍຊື່ ຫຼືລາຍຊື່ດຽວແລະຫຼາຍກຸ່ມ (ຂຶ້ນກັບໂໝດການເຊື່ອມຕໍ່) ສຳລັບແຜນນັ້ນ.

<!-- Screenshot unavailable: MailChimp tab on plan edit page showing lists and groups selection -->

_ເລືອກລາຍຊື່ແລະກຸ່ມທີ່ທ່ານຕ້ອງການໃຊ້ສຳລັບແຕ່ລະແຜນຂອງທ່ານ_

ເມື່ອຜູ້ໃຊ້ໃໝ່ລົງທະບຽນສຳລັບແຜນນັ້ນ, ທີ່ຢູ່ອີເມວຂອງລາວຈະຖືກເພີ່ມເຂົ້າໃນລາຍຊື່ແລະກຸ່ມທີ່ເລືອກໄວ້!
