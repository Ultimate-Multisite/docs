# ແຜນທີ່ວັດຕາມການໃຊ້ງານ

ສ່ວນເສີມ Ultimate Multisite: ແຜນທີ່ວັດຕາມການໃຊ້ງານ ປ່ຽນວິທີທີ່ທ່ານຄິດເງິນລູກຄ້າ ໂດຍນໍາໃຊ້ຄວາມສາມາດການຄິດເງິນແບບວັດຕາມການໃຊ້ງານທີ່ຍືດຫຍຸ່ນ. ແທນທີ່ຈະໃຊ້ຂີດຈໍາກັດແຂງຕາຍທີ່ບລັອກຜູ້ໃຊ້ເມື່ອໃຊ້ເກີນ, ໃຫ້ເປີດໃຊ້ຄ່າເກີນກໍານົດທີ່ຄິດເງິນລູກຄ້າອັດຕະໂນມັດສໍາລັບການໃຊ້ງານເພີ່ມເຕີມ.

## ຄຸນສົມບັດຫຼັກ {#key-features}

* **ການຄິດເງິນແບບວັດຕາມການໃຊ້ງານ**: ຄິດເງິນລູກຄ້າຕາມການໃຊ້ງານຈິງ ແທນທີ່ຈະເປັນລະດັບລາຄາຄົງທີ່
* **ການຈັດການການໃຊ້ເກີນ**: ຄິດເງິນອັດຕະໂນມັດສໍາລັບການໃຊ້ງານທີ່ເກີນຂີດຈໍາກັດຂອງແຜນ
* **ການຕິດຕາມການໃຊ້ງານ**: ກວດຕິດຕາມການໃຊ້ຊັບພະຍາກອນຂອງລູກຄ້າແບບເວລາຈິງ
* **ການກໍານົດລາຄາທີ່ຍືດຫຍຸ່ນ**: ກໍານົດອັດຕາເອງສໍາລັບຊັບພະຍາກອນຕ່າງໆ (ພື້ນທີ່ຈັດເກັບ, ແບນວິດ, ເວັບໄຊ, ແລະອື່ນໆ)
* **ການຄິດເງິນ AI Token**: ຕິດຕາມແລະຄິດເງິນ token ຂາເຂົ້າແລະຂາອອກສໍາລັບການໃຊ້ AI ທົ່ວເວັບໄຊຍ່ອຍຂອງລູກຄ້າ
* **ການອອກໃບແຈ້ງໜີ້ອັດຕະໂນມັດ**: ສ້າງໃບແຈ້ງໜີ້ອັດຕະໂນມັດເມື່ອເກີນເກນການໃຊ້ງານ
* **ລາຍງານການໃຊ້ງານ**: ການວິເຄາະແລະລາຍງານການໃຊ້ງານຢ່າງລະອຽດສໍາລັບລູກຄ້າແລະຜູ້ບໍລິຫານ

## ເໝາະຢ່າງຍິ່ງສໍາລັບ {#perfect-for}

* **ຜູ້ໃຫ້ບໍລິການໂຮສຕິງ**: ສະເໜີແຜນທີ່ຂະຫຍາຍໄດ້ຕາມຄວາມຕ້ອງການຂອງລູກຄ້າ
* **ແພລດຟອມ SaaS**: ຄິດເງິນຕາມການໃຊ້ຊັບພະຍາກອນຈິງ
* **ເຄືອຂ່າຍເອເຈນຊີ**: ແບບຈໍາລອງການກໍານົດລາຄາທີ່ຍືດຫຍຸ່ນສໍາລັບໂຄງການລູກຄ້າ
* **ໂຮສຕິງສໍາລັບຜູ້ຂາຍຕໍ່**: ໃຫ້ພາດເນີສາມາດສະເໜີການກໍານົດລາຄາຕາມການໃຊ້ງານ

## ວິທີການເຮັດວຽກ {#how-it-works}

1. ກໍານົດຄ່າຊັບພະຍາກອນທີ່ວັດຕາມການໃຊ້ງານ (ພື້ນທີ່ດິສກ໌, ແບນວິດ, ຈໍານວນເວັບໄຊ, ແລະອື່ນໆ)
2. ກໍານົດຂີດຈໍາກັດແຜນພື້ນຖານ ແລະອັດຕາຄ່າເກີນກໍານົດ
3. ລະບົບຈະຕິດຕາມການໃຊ້ງານທົ່ວເຄືອຂ່າຍຂອງທ່ານອັດຕະໂນມັດ
4. ເມື່ອລູກຄ້າໃຊ້ເກີນຂີດຈໍາກັດ, ຄ່າເກີນກໍານົດຈະຖືກຄໍານວນ
5. ໃບແຈ້ງໜີ້ຈະຖືກສ້າງອັດຕະໂນມັດຕາມຮອບການຄິດເງິນ
6. ລູກຄ້າສາມາດເບິ່ງການໃຊ້ງານ ແລະຄ່າໃຊ້ຈ່າຍທີ່ຈະເກີດຂຶ້ນແບບເວລາຈິງ

## ການຄິດເງິນ AI Token {#ai-token-billing}

ເວີຊັນ 1.1.0 ເພີ່ມການຄິດເງິນ AI token ສໍາລັບເຄືອຂ່າຍທີ່ສະເໜີຄຸນສົມບັດ AI ຜ່ານ WordPress AI Client SDK. ແຜນທີ່ວັດຕາມການໃຊ້ງານຈະຟັງຄໍາຂໍການສ້າງ AI ທີ່ສໍາເລັດແລ້ວ, ບັນທຶກຈໍານວນ token ຂາເຂົ້າແລະຂາອອກສໍາລັບເວັບໄຊຍ່ອຍປັດຈຸບັນ, ແລະແປງ token ເຫຼົ່ານັ້ນເປັນຍອດໃຊ້ AI ທີ່ຄິດເງິນໄດ້ສໍາລັບຮອບການຄິດເງິນປັດຈຸບັນຂອງລູກຄ້າ.

ໃຊ້ການຄິດເງິນ AI token ເມື່ອທ່ານຕ້ອງການສະເໜີເຄື່ອງມື AI ຢູ່ໃນເວັບໄຊຂອງລູກຄ້າ ໂດຍບໍ່ຕ້ອງຮັບພາລະຕົ້ນທຶນແບບຈໍາລອງທີ່ຄາດເດົາຍາກ. ແຕ່ລະຄໍາຂໍຈະຖືກຜູກກັບເວັບໄຊຍ່ອຍ, ລູກຄ້າ, ສະມາຊິກພາບ, ຜູ້ໃຫ້ບໍລິການ, ແລະແບບຈໍາລອງ ເພື່ອໃຫ້ຜູ້ບໍລິຫານສາມາດກວດສອບການໃຊ້ງານໄດ້ ໃນຂະນະທີ່ລູກຄ້າສາມາດເຫັນວ່າການໃຊ້ AI ຂອງພວກເຂົາມີຜົນຕໍ່ Account ຂອງພວກເຂົາແນວໃດ.

### ກໍານົດອັດຕາຕໍ່ Token {#configure-per-token-rates}

ອັດຕາ AI ຖືກກໍານົດຕາມແຕ່ລະແບບຈໍາລອງ ເປັນອັດຕາຂາເຂົ້າແລະຂາອອກແຍກກັນ. ອັດຕາຖືກຄໍານວນຕໍ່ 1 ລ້ານ token, ເຊິ່ງກົງກັບໜ້າລາຄາຂອງຜູ້ໃຫ້ບໍລິການ AI ທົ່ວໄປ.

1. ໃນຜູ້ບໍລິຫານເຄືອຂ່າຍ, ເປີດ **Multisite Ultimate → Settings → Metered Plans**.
2. ເປີດໃຊ້ການວັດຍອດໃຊ້ AI ສໍາລັບຜະລິດຕະພັນ ຫຼືສະມາຊິກພາບທີ່ຄວນຄິດເງິນການໃຊ້ AI.
3. ເພີ່ມລາຍການອັດຕາແບບຈໍາລອງໂດຍໃຊ້ ID ແບບຈໍາລອງທີ່ connector ຂອງທ່ານສົ່ງກັບມາ, ເຊັ່ນ `gpt-4o`, `gpt-4o-mini`, ຫຼືຕົວລະບຸແບບຈໍາລອງອື່ນທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI.
4. ປ້ອນຄ່າ **ອັດຕາຂາເຂົ້າ** ແລະ **ອັດຕາຂາອອກ** ແຍກກັນໃນສະກຸນເງິນຂອງເວັບໄຊທ່ານ ຕໍ່ 1 ລ້ານ token.
5. ກໍານົດວົງເງິນຍອດໃຊ້ AI ທີ່ລວມຢູ່ຂອງລູກຄ້າ ແລະພຶດຕິກໍາການໃຊ້ເກີນຂອງແຜນ.

ເມື່ອບໍ່ໄດ້ກໍານົດ ID ແບບຈໍາລອງທີ່ກົງກັນແນ່ນອນ, ແຜນທີ່ວັດຕາມການໃຊ້ງານສາມາດຈັບຄູ່ຕາມຄໍານໍາໜ້າຂອງແບບຈໍາລອງໄດ້. ຕົວຢ່າງ, ອັດຕາທີ່ກໍານົດສໍາລັບ `claude-sonnet-4` ສາມາດນໍາໃຊ້ກັບ ID ແບບຈໍາລອງທີ່ມີເວລາກໍາກັບ ແລະເລີ່ມຕົ້ນດ້ວຍຄໍານໍາໜ້ານັ້ນ. ນັກພັດທະນາຍັງສາມາດປັບຕາຕະລາງອັດຕາສຸດທ້າຍດ້ວຍຕົວກັ່ນຕອງ [`wu_ai_token_rates`](./hooks/Filters/wu_ai_token_rates).

### AI Connectors ທີ່ຮອງຮັບ {#supported-ai-connectors}

ການຄິດເງິນ AI token ໃຊ້ງານໄດ້ກັບ connector ທີ່ສົ່ງເຫດການການສ້າງຂອງ WordPress AI Client SDK ແລະລວມຂໍ້ມູນການໃຊ້ token ໄວ້ໃນ metadata ຂອງຄໍາຕອບ. ນີ້ລວມເຖິງກຸ່ມ connector AI ຂອງ Ultimate Multisite ທີ່ບັນທຶກໄວ້ທີ່ນີ້:

* [Ultimate AI Connector ສໍາລັບ Endpoint ທີ່ເຂົ້າກັນໄດ້](../ultimate-ai-connector-compatible-endpoints/) ສໍາລັບ API ທີ່ເຂົ້າກັນໄດ້ກັບ OpenAI, ລວມທັງ OpenAI, Mistral, gateway ທີ່ເຂົ້າກັນໄດ້ກັບ Ollama, ແລະຜູ້ໃຫ້ບໍລິການອື່ນໆທີ່ເປີດເຜີຍ metadata ການໃຊ້ token ທີ່ເຂົ້າກັນໄດ້
* [Ultimate AI Connector ສໍາລັບ WebLLM](../ultimate-ai-connector-webllm/) ສໍາລັບການໃຊ້ WebLLM ແບບດັ້ງເດີມໃນເບຣາວເຊີ ເມື່ອຄໍາຂໍຖືກສົ່ງເສັ້ນທາງຜ່ານກະແສເຫດການ AI Client SDK
* connector ຜູ້ໃຫ້ບໍລິການ AI ທີ່ໃຊ້ໂດຍ [Gratis AI Agent](../gratis-ai-agent/) ເມື່ອຜູ້ໃຫ້ບໍລິການທີ່ເລືອກລາຍງານຈໍານວນ token ຂອງ prompt ແລະ completion ຜ່ານ SDK

connector ແບບກໍານົດເອງສາມາດເຂົ້າຮ່ວມໄດ້ໂດຍການຍິງເຫດການມາດຕະຖານ `wp_ai_client_before_generate_result` ແລະ `wp_ai_client_after_generate_result` ພ້ອມກັບ metadata ຂອງແບບຈໍາລອງ, ຜູ້ໃຫ້ບໍລິການ, ແລະການໃຊ້ token. ການຕັ້ງຄ່າ connector ຖືກບັງຄັບໃຊ້ຈາກເວັບໄຊຫຼັກໄປຫາເວັບໄຊຍ່ອຍ ເພື່ອໃຫ້ເວັບໄຊລູກຄ້າໃຊ້ຜູ້ໃຫ້ບໍລິການແລະຂີດຈໍາກັດທີ່ເຄືອຂ່າຍອະນຸມັດ.

### ການໃຊ້ງານ Account ຂອງລູກຄ້າ {#customer-account-usage}

ລູກຄ້າຈະເຫັນການໃຊ້ AI ຄຽງຄູ່ກັບຊັບພະຍາກອນອື່ນໆທີ່ວັດຕາມການໃຊ້ງານໃນພື້ນທີ່ Account ຂອງພວກເຂົາ. ແຖວຍອດໃຊ້ AI ສະແດງຍອດລວມຂອງຮອບການຄິດເງິນປັດຈຸບັນທີ່ມາຈາກ token prompt ແລະ completion ທີ່ບັນທຶກໄວ້, ອັດຕາແບບຈໍາລອງທີ່ກໍານົດ, ແລະວົງເງິນທີ່ລວມຢູ່ໃນແຜນ.

ຖ້າລູກຄ້າມີວິທີຊໍາລະເງິນຢູ່ໃນໄຟລ໌, ການໃຊ້ງານເກີນວົງເງິນທີ່ລວມຢູ່ຈະຖືກຖືວ່າເປັນຄ່າເກີນກໍານົດທີ່ຄິດເງິນໄດ້. ຖ້າແຜນຖືກກໍານົດໃຫ້ຕັດຂາດທັນທີ ແລະລູກຄ້າບໍ່ມີວິທີຊໍາລະເງິນທີ່ພ້ອມໃຊ້, ຄໍາຂໍ AI ອາດຖືກບລັອກເມື່ອເຖິງວົງເງິນ ແລະລູກຄ້າຈະຖືກແຈ້ງໃຫ້ເພີ່ມລາຍລະອຽດການຊໍາລະເງິນກ່ອນດໍາເນີນຕໍ່.

## ການຕິດຕັ້ງ {#installation}

1. ອັບໂຫຼດໄຟລ໌ plugin ໄປທີ່ `/wp-content/plugins/ultimate-multisite-metered-plans`
2. ເປີດໃຊ້ plugin ຜ່ານໜ້າ 'Plugins' ໃນ WordPress
3. ໄປທີ່ Multisite Ultimate → Settings → Metered Plans
4. ກໍານົດຄ່າຊັບພະຍາກອນທີ່ວັດຕາມການໃຊ້ງານ ແລະການກໍານົດລາຄາຂອງທ່ານ
5. ເປີດໃຊ້ການຄິດເງິນແບບວັດຕາມການໃຊ້ງານສໍາລັບຜະລິດຕະພັນໃນຕົວແກ້ໄຂຜະລິດຕະພັນ

## ຄໍາຖາມທີ່ພົບເລື້ອຍ {#frequently-asked-questions}

## ຈະເກີດຫຍັງຂຶ້ນເມື່ອລູກຄ້າໃຊ້ເກີນຂີດຈໍາກັດຂອງຕົນ? {#what-happens-when-a-customer-exceeds-their-limit}

ແທນທີ່ຈະບລັອກການເຂົ້າໃຊ້, ລະບົບຈະຕິດຕາມສ່ວນທີ່ໃຊ້ເກີນ ແລະເພີ່ມມັນເຂົ້າໃນໃບແຈ້ງໜີ້ຄັ້ງຕໍ່ໄປຂອງພວກເຂົາ ຕາມອັດຕາທີ່ທ່ານໄດ້ຕັ້ງຄ່າໄວ້.

## ຂ້ອຍສາມາດຕັ້ງອັດຕາການໃຊ້ເກີນທີ່ແຕກຕ່າງກັນສຳລັບສິນຄ້າຕ່າງກັນໄດ້ບໍ? {#can-i-set-different-overage-rates-for-different-products}

ໄດ້, ແຕ່ລະສິນຄ້າສາມາດມີຊັບພະຍາກອນແບບວັດຕາມການໃຊ້ ແລະລາຄາການໃຊ້ເກີນທີ່ເປັນເອກະລັກໄດ້.

## ລູກຄ້າຈະໄດ້ຮັບແຈ້ງກ່ຽວກັບການໃຊ້ງານແນວໃດ? {#how-are-customers-notified-about-usage}

ລູກຄ້າສາມາດເບິ່ງການໃຊ້ງານປັດຈຸບັນຂອງຕົນໃນແດຊບອດຂອງຕົນ ແລະຮັບການແຈ້ງເຕືອນທາງເລືອກເມື່ອໃກ້ຈະຮອດ ຫຼືເກີນຂີດຈຳກັດ.

## ການໃຊ້ AI token ຖືກຄິດລາຄາແນວໃດ? {#how-is-ai-token-usage-priced}

ການໃຊ້ AI token ຖືກຄິດລາຄາຈາກອັດຕາ input ແລະ output ທີ່ກຳນົດໄວ້ຂອງໂມເດວ ຕໍ່ 1 ລ້ານ token. ແຜນແບບວັດຕາມການໃຊ້ຈະຄູນຈຳນວນ prompt token ແລະ completion token ຂອງແຕ່ລະຄຳຂໍດ້ວຍອັດຕາເຫຼົ່ານັ້ນ, ເພີ່ມຜົນລວມເຂົ້າໃນຍອດໃຊ້ຈ່າຍ AI ຂອງລູກຄ້າໃນຮອບບິນປັດຈຸບັນ, ແລະນຳໃຊ້ກົດຂອງໂຄຕາທີ່ອະນຸຍາດຂອງແຜນ ຫຼືກົດການໃຊ້ເກີນ.

## ຂ້ອຍຍັງສາມາດໃຊ້ຂີດຈຳກັດແບບເຂັ້ມງວດໄດ້ບໍ? {#can-i-still-use-hard-limits}

ໄດ້, ການເກັບເງິນແບບວັດຕາມການໃຊ້ເປັນທາງເລືອກຕໍ່ແຕ່ລະສິນຄ້າ. ທ່ານສາມາດປະສົມສິນຄ້າແບບຂີດຈຳກັດເຂັ້ມງວດແບບດັ້ງເດີມ ກັບສິນຄ້າແບບວັດຕາມການໃຊ້ໄດ້.
