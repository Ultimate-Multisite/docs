# ຫຼາຍເຄືອຂ່າຍ

addon Ultimate Multisite: Multinetwork ຂະຫຍາຍແພລດຟອມ Ultimate Multisite ຂອງທ່ານດ້ວຍຟັງຊັນຫຼາຍເຄືອຂ່າຍທີ່ມີພະລັງ. addon ນີ້ຊ່ວຍໃຫ້ລູກຄ້າຂອງທ່ານຊື້ ແລະຈັດການເຄືອຂ່າຍ WordPress multisite ທັງໝົດໄດ້, ບໍ່ແມ່ນພຽງແຕ່ເວັບໄຊແຍກຕ່າງຫາກ.

## ຄຸນສົມບັດຫຼັກ {#key-features}

* **ການຈັດຕຽມເຄືອຂ່າຍ**: ສ້າງ ແລະກຳນົດຄ່າເຄືອຂ່າຍ WordPress multisite ທີ່ສົມບູນໃຫ້ລູກຄ້າຂອງທ່ານໂດຍອັດຕະໂນມັດ
* **ແມ່ແບບເຄືອຂ່າຍ**: ໃຫ້ລູກຄ້າເລືອກເບິ່ງ, ດູຕົວຢ່າງ, ແລະເລືອກແມ່ແບບເຄືອຂ່າຍທີ່ສ້າງໄວ້ກ່ອນ ກ່ອນລົງທະບຽນ
* **ການຈັດການເຄືອຂ່າຍ**: ຄວາມສາມາດຈັດການຄົບຖ້ວນສຳລັບສະພາບແວດລ້ອມຫຼາຍເຄືອຂ່າຍ
* **ເຄືອຂ່າຍທີ່ແຍກກັນ**: ເຄືອຂ່າຍຂອງລູກຄ້າແຕ່ລະອັນເຮັດວຽກຢ່າງເອກະລາດ ພ້ອມຕາຕະລາງຖານຂໍ້ມູນ ແລະການກຳນົດຄ່າຂອງຕົນເອງ
* **Cross-Network SSO**: ໃຫ້ລູກຄ້າເຂົ້າເຖິງເຄືອຂ່າຍຍ່ອຍຂອງພວກເຂົາໄດ້ຢ່າງລື່ນໄຫຼດ້ວຍການຢືນຢັນຕົວຕົນແບບລິ້ງວິເສດ
* **ສະຖາປັດຕະຍະກຳທີ່ຂະຫຍາຍໄດ້**: ຈັດການເຄືອຂ່າຍຫຼາຍອັນໄດ້ຢ່າງມີປະສິດທິພາບ ດ້ວຍການຈັດສັນຊັບພະຍາກອນທີ່ປັບໃຫ້ເໝາະສົມ
* **ການເຊື່ອມຕໍ່**: ເຊື່ອມຕໍ່ຢ່າງລື່ນໄຫຼກັບລະບົບສະມາຊິກ ແລະການຮຽກເກັບເງິນຂອງ Ultimate Multisite

## ກໍລະນີການນຳໃຊ້ {#use-cases}

* **ໂຮສຕິ້ງແບບໄວທ໌ເລເບວ**: ມອບເຄືອຂ່າຍ multisite ທີ່ມີແບຣນຂອງລູກຄ້າເອງໃຫ້ພວກເຂົາ
* **ວິທີແກ້ໄຂສຳລັບເອເຈນຊີ**: ໃຫ້ເອເຈນຊີຈັດການເຄືອຂ່າຍຂອງລູກຄ້າຫຼາຍອັນຈາກແພລດຟອມດຽວ
* **ການນຳໃຊ້ລະດັບອົງກອນ**: ຮອງຮັບອົງກອນຂະໜາດໃຫຍ່ທີ່ມີຄວາມຕ້ອງການຫຼາຍເຄືອຂ່າຍທີ່ຊັບຊ້ອນ
* **ແພລດຟອມການສຶກສາ**: ສ້າງເຄືອຂ່າຍແຍກຕ່າງຫາກສຳລັບໂຮງຮຽນ ຫຼືພະແນກຕ່າງໆ

## ຂໍ້ກຳນົດ {#requirements}

* Ultimate Multisite 2.0.0 ຫຼືສູງກວ່າ
* ການຕິດຕັ້ງ WordPress Multisite
* PHP 7.4 ຫຼືສູງກວ່າ
* MySQL 5.6 ຫຼືສູງກວ່າ

## ການຕິດຕັ້ງ {#installation}

1. ອັບໂຫຼດໄຟລ໌ປລັກອິນໄປທີ່ໄດເຣັກທໍຣີ `/wp-content/plugins/ultimate-multisite-multinetwork`
2. ເປີດໃຊ້ປລັກອິນຜ່ານໜ້າຈໍ 'ປລັກອິນ' ໃນ WordPress
3. ກຳນົດຄ່າການຕັ້ງຄ່າຫຼາຍເຄືອຂ່າຍຜ່ານ Ultimate Multisite → ການຕັ້ງຄ່າ

## ແມ່ແບບເຄືອຂ່າຍ {#network-templates}

ເວີຊັນ 1.3.0 ເພີ່ມຂະບວນການຄົ້ນພົບ ແລະເລືອກແມ່ແບບສຳລັບລູກຄ້າທີ່ຊື້ເຄືອຂ່າຍໃໝ່. ໃຊ້ແມ່ແບບເຄືອຂ່າຍເມື່ອທ່ານຕ້ອງການໃຫ້ເຄືອຂ່າຍໃໝ່ຂອງລູກຄ້າແຕ່ລະຄົນເລີ່ມຈາກໂຄງສ້າງ, ການອອກແບບ, ຄັງເນື້ອຫາ, ຫຼືການຕັ້ງຄ່າເຉພາະສາຍງານທີ່ຕຽມໄວ້ແລ້ວ.

### ຕົວຢ່າງແມ່ແບບ {#template-previewer}

ຕົວຢ່າງແມ່ແບບເຄືອຂ່າຍໃຫ້ລູກຄ້າເລືອກເບິ່ງແມ່ແບບທີ່ມີກ່ອນ checkout. ແມ່ແບບແຕ່ລະອັນສາມາດເປີດໃນແຜງຕົວຢ່າງສົດໄດ້, ເພື່ອໃຫ້ລູກຄ້າສາມາດປຽບທຽບໂຄງຮ່າງ ແລະເນື້ອຫາກ່ອນທີ່ພວກເຂົາຈະຊື້ເຄືອຂ່າຍ.

ໃຊ້ຕົວຢ່າງເພື່ອຊ່ວຍລູກຄ້າ:

* ທົບທວນຮູບລັກ ແລະໂຄງສ້າງຂອງແມ່ແບບເຄືອຂ່າຍທີ່ມີແຕ່ລະອັນ
* ປຽບທຽບແມ່ແບບຫຼາຍອັນໂດຍບໍ່ຕ້ອງອອກຈາກຂະບວນການຊື້
* ເລືອກແມ່ແບບທີ່ກົງກັບກໍລະນີການນຳໃຊ້ຂອງພວກເຂົາທີ່ສຸດກ່ອນລົງທະບຽນ

ຮັກສາຊື່ ແລະຄຳອະທິບາຍແມ່ແບບໃຫ້ເປັນມິດຕໍ່ລູກຄ້າ. ຕົວຢ່າງຈະມີປະໂຫຍດຫຼາຍທີ່ສຸດເມື່ອລູກຄ້າສາມາດເຂົ້າໃຈກໍລະນີການນຳໃຊ້ທີ່ຕັ້ງໃຈໄວ້, ເຊັ່ນ ເຄືອຂ່າຍເອເຈນຊີ, ເຄືອຂ່າຍໂຮງຮຽນ, ເຄືອຂ່າຍແຟຣນໄຊສ໌, ຫຼືຊຸດໂຮສຕິ້ງແບບໄວທ໌ເລເບວ.

### ຕົວເລືອກແມ່ແບບໃນ checkout {#template-picker-at-checkout}

ຕົວເລືອກແມ່ແບບສາມາດເພີ່ມໃສ່ຂະບວນການ checkout ສຳລັບການລົງທະບຽນເປັນຊ່ອງຂໍ້ມູນໄດ້. ເມື່ອກຳນົດຄ່າແລ້ວ, ລູກຄ້າເລືອກແມ່ແບບເຄືອຂ່າຍທີ່ຕ້ອງການໃນລະຫວ່າງ checkout, ແລະແມ່ແບບທີ່ເລືອກຈະຖືກໃຊ້ເມື່ອເຄືອຂ່າຍໃໝ່ຂອງພວກເຂົາຖືກຈັດຕຽມ.

ເພື່ອໃຊ້ຕົວເລືອກໃນຂະບວນການ checkout:

1. ຕຽມເຄືອຂ່າຍທີ່ທ່ານຕ້ອງການສະເໜີເປັນແມ່ແບບ.
2. ເປີດໃຊ້ຊ່ອງເລືອກແມ່ແບບ Multinetwork ໃນແບບຟອມລົງທະບຽນ ຫຼື checkout ທີ່ກ່ຽວຂ້ອງ.
3. ເລືອກວ່າແມ່ແບບໃດຄວນມີໃຫ້ສຳລັບຂະບວນການ checkout ນັ້ນ.
4. ເຜີຍແຜ່ຂະບວນການ checkout ແລະທົດສອບການລົງທະບຽນເພື່ອຢືນຢັນວ່າແມ່ແບບທີ່ເລືອກຖືກນຳໃຊ້ກັບເຄືອຂ່າຍໃໝ່.

ຖ້າຂະບວນການ checkout ສະເໜີພຽງຜະລິດຕະພັນເຄືອຂ່າຍອັນດຽວ, ຕົວເລືອກກໍຍັງສາມາດເປັນປະໂຫຍດໃນຖານະຂັ້ນຕອນ onboarding ແບບມີການແນະນຳ. ສຳລັບ funnel ຫຼາຍແຜນ, ໃຫ້ສະແດງສະເພາະແມ່ແບບທີ່ເຂົ້າກັນໄດ້ກັບຜະລິດຕະພັນ ຫຼືແຜນທີ່ເລືອກ.

## Cross-Network Magic Link SSO {#cross-network-magic-link-sso}

Cross-network magic link SSO ໃຫ້ລູກຄ້າຍ້າຍໄປມາລະຫວ່າງພື້ນທີ່ບັນຊີຫຼັກຂອງພວກເຂົາ ແລະເຄືອຂ່າຍຍ່ອຍໂດຍບໍ່ຕ້ອງລົງຊື່ເຂົ້າໃໝ່ດ້ວຍຕົນເອງ. ເມື່ອລູກຄ້າຕາມລິ້ງເຂົ້າເຖິງຂ້າມເຄືອຂ່າຍ, Ultimate Multisite ຈະສ້າງລິ້ງວິເສດທີ່ຈຳກັດເວລາ ແລະຢືນຢັນຕົວຕົນລູກຄ້າເຂົ້າສູ່ເຄືອຂ່າຍປາຍທາງ.

ສິ່ງນີ້ປັບປຸງປະສົບການສຳລັບລູກຄ້າທີ່ຈັດການຫຼາຍກວ່າໜຶ່ງເຄືອຂ່າຍ ຫຼືຜູ້ທີ່ຕ້ອງເຂົ້າເຖິງເຄືອຂ່າຍແມ່ແບບທີ່ຖືກໂຄລນໃໝ່ທັນທີຫຼັງການຊື້.

ລາຍການກວດສອບຂອງຜູ້ດູແລ:

* ຢືນຢັນວ່າ addon Multinetwork ເປີດໃຊ້ງານທົ່ວການຕິດຕັ້ງ WordPress multisite.
* ໃຫ້ແນ່ໃຈວ່າລູກຄ້າມີສິດເຂົ້າເຖິງເຄືອຂ່າຍທີ່ຖືກຕ້ອງກ່ອນຈະມີລິ້ງຂ້າມເຄືອຂ່າຍໃຫ້ພວກເຂົາ.
* ຮັກສາໂດເມນເຄືອຂ່າຍ ແລະໂດເມນທີ່ແມັບໄວ້ໃຫ້ກຳນົດຄ່າຢ່າງຖືກຕ້ອງ ເພື່ອໃຫ້ລິ້ງວິເສດແກ້ໄປຫາເຄືອຂ່າຍປາຍທາງທີ່ຕັ້ງໃຈໄວ້.
* ທົດສອບ dashboard ຂອງລູກຄ້າ ແລະປຸ່ມເຂົ້າເຖິງຫຼັງ checkout ໃດໆ ຫຼັງຈາກປ່ຽນການຕັ້ງຄ່າເຄືອຂ່າຍ, ໂດເມນ, ຫຼື checkout.

ລິ້ງວິເສດຄວນຖືກປະຕິບັດເໝືອນລິ້ງລັອກອິນ. ພວກມັນຖືກອອກແບບມາເພື່ອການເຂົ້າເຖິງຂອງລູກຄ້າຢ່າງລື່ນໄຫຼ ແລະບໍ່ຄວນຖືກສົ່ງຕໍ່ ຫຼືນຳໃຊ້ຊ້ຳນອກເຊສຊັນທີ່ຕັ້ງໃຈໄວ້.

## ຄຳຖາມທີ່ພົບເລື້ອຍ {#frequently-asked-questions}

### ຫຼາຍເຄືອຂ່າຍແມ່ນຫຍັງ? {#what-is-multinetwork}

ຫຼາຍເຄືອຂ່າຍແມ່ນຄຸນສົມບັດຂອງ WordPress ທີ່ອະນຸຍາດໃຫ້ທ່ານໃຊ້ງານເຄືອຂ່າຍ multisite ອິດສະຫຼະຫຼາຍອັນຢູ່ໃນການຕິດຕັ້ງ WordPress ດຽວ.

### ສິ່ງນີ້ແຕກຕ່າງຈາກ multisite ປົກກະຕິແນວໃດ? {#how-is-this-different-from-regular-multisite}

multisite ປົກກະຕິອະນຸຍາດໃຫ້ມີເວັບໄຊຫຼາຍອັນພາຍໃນເຄືອຂ່າຍດຽວ. ຫຼາຍເຄືອຂ່າຍອະນຸຍາດໃຫ້ມີເຄືອຂ່າຍຫຼາຍອັນ, ແຕ່ລະອັນມີເວັບໄຊຂອງຕົນເອງ, ທັງໝົດຢູ່ພາຍໃນການຕິດຕັ້ງ WordPress ດຽວ.

### ສິ່ງນີ້ຕ້ອງການການກຳນົດຄ່າເຊີບເວີພິເສດບໍ? {#does-this-require-special-server-configuration}

ແມ່ນ, ຫຼາຍເຄືອຂ່າຍຕ້ອງການການຕັ້ງຄ່າ ແລະການກຳນົດຄ່າເຊີບເວີສະເພາະ. ກະລຸນາປຶກສາເອກະສານສຳລັບຄຳແນະນຳການຕັ້ງຄ່າແບບລະອຽດ.
