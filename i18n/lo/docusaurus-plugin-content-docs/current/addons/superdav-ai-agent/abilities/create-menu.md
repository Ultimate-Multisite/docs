# ສ້າງເມນູ {#create-menu}

ຄວາມສາມາດ **ສ້າງເມນູ** ສ້າງເມນູການນຳທາງຂອງ WordPress ໂດຍຮອງຮັບປ້າຍກຳກັບການນຳທາງທີ່ແຍກຕ່າງຫາກຈາກຊື່ໜ້າ.

## ພາບລວມ {#overview}

ຄວາມສາມາດນີ້ຂະຫຍາຍຟັງຊັນການສ້າງເມນູມາດຕະຖານ ດ້ວຍຄວາມສາມາດໃນການລະບຸພາລາມິເຕີ `navigation_label`. ສິ່ງນີ້ຊ່ວຍໃຫ້ທ່ານສ້າງເມນູທີ່ປ້າຍກຳກັບທີ່ສະແດງໃນການນຳທາງແຕກຕ່າງຈາກຊື່ໜ້າ, ເພີ່ມຄວາມຍືດຫຍຸ່ນໃນໂຄງສ້າງເວັບໄຊ ແລະປະສົບການຜູ້ໃຊ້.

## ພາລາມິເຕີ {#parameters}

| ພາລາມິເຕີ | ປະເພດ | ຈຳເປັນ | ຄຳອະທິບາຍ |
|-----------|------|----------|-------------|
| `name` | string | ແມ່ນ | ຊື່ເມນູ, ເຊັ່ນ `Primary Navigation` |
| `location` | string | ບໍ່ | ຕຳແໜ່ງຂອງທີມທີ່ຈະກຳນົດເມນູນີ້ໃຫ້, ເຊັ່ນ `primary` |
| `navigation_label` | string | ບໍ່ | ປ້າຍກຳກັບທີ່ຈະສະແດງໃນການນຳທາງ (ແຍກຕ່າງຫາກຈາກຊື່ໜ້າ) |

## ຄ່າທີ່ສົ່ງກັບ {#return-value}

```json
{
  "success": true,
  "menu_id": 7,
  "name": "Primary Navigation",
  "navigation_label": "Main Menu"
}
```

## ປ້າຍກຳກັບການນຳທາງ ທຽບກັບ ຊື່ໜ້າ {#navigation-label-vs-page-title}

ພາລາມິເຕີ `navigation_label` ຊ່ວຍໃຫ້ທ່ານແຍກຊື່ເມນູພາຍໃນອອກຈາກປ້າຍກຳກັບທີ່ສະແດງໃຫ້ຜູ້ໃຊ້ເຫັນ:

- **`name`** — ຕົວລະບຸເມນູພາຍໃນທີ່ WordPress ໃຊ້ (ເຊັ່ນ "Primary Navigation")
- **`navigation_label`** — ປ້າຍກຳກັບທີ່ສະແດງໃຫ້ຜູ້ເຂົ້າຊົມເວັບໄຊເຫັນໃນການນຳທາງ (ເຊັ່ນ "Main Menu")

ສິ່ງນີ້ມີປະໂຫຍດເມື່ອ:
- ຫຼັກການຕັ້ງຊື່ພາຍໃນຂອງທ່ານແຕກຕ່າງຈາກປ້າຍກຳກັບທີ່ຜູ້ໃຊ້ເຫັນ
- ທ່ານຕ້ອງການປ້າຍກຳກັບທີ່ສັ້ນກວ່າໃນການນຳທາງ ກ່ວາໃນແຜງຜູ້ດູແລ
- ທ່ານຕ້ອງຮອງຮັບຫຼາຍພາສາທີ່ມີຄວາມຍາວຂອງປ້າຍກຳກັບແຕກຕ່າງກັນ
- ທ່ານກຳລັງສ້າງເມນູສຳລັບພື້ນທີ່ ຫຼືກຸ່ມຜູ້ໃຊ້ສະເພາະ

## ຕົວຢ່າງການໃຊ້ງານ {#usage-examples}

### ຕົວຢ່າງ 1: ເມນູງ່າຍໆທີ່ມີປ້າຍກຳກັບການນຳທາງ {#example-1-simple-menu-with-navigation-label}

**ຄຳສັ່ງ:**
```
ສ້າງເມນູການນຳທາງຫຼັກຊື່ "Main Navigation" ພ້ອມປ້າຍກຳກັບການນຳທາງ "Menu".
```

**ຜົນລັບ:**
```json
{
  "success": true,
  "menu_id": 7,
  "name": "Main Navigation",
  "navigation_label": "Menu"
}
```

### ຕົວຢ່າງ 2: ເມນູສຳລັບຕຳແໜ່ງທີມສະເພາະ {#example-2-menu-for-specific-theme-location}

**ຄຳສັ່ງ:**
```
ສ້າງເມນູສ່ວນທ້າຍຊື່ "Footer Links" ພ້ອມປ້າຍກຳກັບການນຳທາງ "Quick Links" ແລະກຳນົດໃຫ້ມັນໄປທີ່ຕຳແໜ່ງສ່ວນທ້າຍ.
```

**ຜົນລັບ:**
```json
{
  "success": true,
  "menu_id": 8,
  "name": "Footer Links",
  "navigation_label": "Quick Links",
  "location": "footer"
}
```

## ການເຊື່ອມລວມກັບ Theme Builder {#integration-with-theme-builder}

ເມື່ອໃຊ້ Theme Builder, ຄວາມສາມາດສ້າງເມນູຈະ:

1. ກວດພົບຕຳແໜ່ງເມນູຂອງທີມທີ່ມີຢູ່ໂດຍອັດຕະໂນມັດ
2. ສ້າງເມນູພ້ອມປ້າຍກຳກັບການນຳທາງທີ່ເໝາະສົມກັບການອອກແບບຂອງທ່ານ
3. ກຳນົດເມນູໄປທີ່ຕຳແໜ່ງທີມທີ່ຖືກຕ້ອງ
4. ຮອງຮັບການເພີ່ມລາຍການເມນູຫຼັງຈາກສ້າງແລ້ວ

## ຄວາມສາມາດທີ່ກ່ຽວຂ້ອງ {#related-abilities}

- **`add_menu_item`** — ເພີ່ມລາຍການເຂົ້າໃນເມນູທີ່ມີຢູ່
- **`update_menu`** — ປ່ຽນຊື່ເມນູ ຫຼືກຳນົດໃໝ່ໃຫ້ໄປທີ່ຕຳແໜ່ງທີມ
- **`delete_menu`** — ລຶບເມນູອອກຈາກເວັບໄຊຂອງທ່ານ

## ແນວປະຕິບັດທີ່ດີທີ່ສຸດ {#best-practices}

- **ໃຊ້ປ້າຍກຳກັບການນຳທາງທີ່ຊັດເຈນ** — ຮັກສາປ້າຍກຳກັບໃຫ້ກະຊັບ ແລະອະທິບາຍໄດ້ດີສຳລັບຜູ້ໃຊ້
- **ໃຫ້ກົງກັບຕຳແໜ່ງທີມ** — ກຳນົດເມນູໄປທີ່ຕຳແໜ່ງທີມທີ່ຖືກຕ້ອງເພື່ອການສະແດງຜົນທີ່ເໝາະສົມ
- **ວາງແຜນໂຄງສ້າງເມນູ** — ຕັດສິນໃຈລຳດັບຊັ້ນເມນູຂອງທ່ານກ່ອນສ້າງລາຍການ
- **ທົດສອບການຕອບສະໜອງ** — ກວດສອບວ່າເມນູສະແດງຖືກຕ້ອງໃນອຸປະກອນມືຖື
- **ແປປ້າຍກຳກັບໃຫ້ເໝາະກັບພາສາ** — ໃຊ້ປ້າຍກຳກັບການນຳທາງທີ່ແຕກຕ່າງກັນສຳລັບເວີຊັນພາສາຕ່າງໆ

## ຂໍ້ຈຳກັດ {#limitations}

- ປ້າຍກຳກັບການນຳທາງເປັນພຽງການສະແດງຜົນ; `name` ພາຍໃນຍັງຄົງຖືກໃຊ້ສຳລັບການລະບຸຕົວຕົນໂດຍ WordPress
- ການຮອງຮັບຂອງທີມແຕກຕ່າງກັນ; ບໍ່ແມ່ນທຸກທີມຈະຮອງຮັບຕຳແໜ່ງເມນູທັງໝົດ
- ຕ້ອງເພີ່ມລາຍການເມນູແຍກຕ່າງຫາກຫຼັງຈາກສ້າງເມນູແລ້ວ
- ການປ່ຽນປ້າຍກຳກັບການນຳທາງຕ້ອງອັບເດດເມນູ
