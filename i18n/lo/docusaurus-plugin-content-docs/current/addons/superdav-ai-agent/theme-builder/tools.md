# ເຄື່ອງມື ແລະ ການກຳນົດຄ່າ {#tools-and-configuration}

Theme Builder ໃຊ້ຊຸດເຄື່ອງມືເພື່ອວິເຄາະ, ອອກແບບ, ແລະ ສ້າງ WordPress ຂອງທ່ານ. ໃນເວີຊັນ 1.16.0, **sd-ai-agent/site-scrape ຕອນນີ້ເປັນເຄື່ອງມືລະດັບ 1**, ເຮັດໃຫ້ມັນພ້ອມໃຊ້ງານໂດຍຄ່າເລີ່ມຕົ້ນໃນ Theme Builder. Superdav AI Agent v1.18.0 ຍັງແນະນຳແພັກເກດ companion ຂັ້ນສູງທີ່ແຈກຈ່າຍແຍກຕ່າງຫາກສຳລັບກະແສວຽກຂອງນັກພັດທະນາທີ່ໄວ້ໃຈໄດ້; ເຄື່ອງມືເຫຼົ່ານັ້ນບໍ່ແມ່ນສ່ວນໜຶ່ງຂອງແພັກເກດຫຼັກ WordPress.org ແລະ ຕ້ອງຕິດຕັ້ງ ແລະ ອະນຸຍາດແຍກຕ່າງຫາກ.

## ເຄື່ອງມືທີ່ພ້ອມໃຊ້ງານ {#available-tools}

### ເຄື່ອງມືລະດັບ 1 (ພ້ອມໃຊ້ສະເໝີ) {#tier-1-tools-always-available}

ເຄື່ອງມືລະດັບ 1 ພ້ອມໃຊ້ງານໂດຍຄ່າເລີ່ມຕົ້ນໃນ Theme Builder ໂດຍບໍ່ຕ້ອງກຳນົດຄ່າເພີ່ມເຕີມໃດໆ.

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**ຈຸດປະສົງ:** ວິເຄາະເວັບໄຊທີ່ມີຢູ່ແລ້ວເພື່ອດຶງແຮງບັນດານໃຈດ້ານການອອກແບບ, ໂຄງສ້າງເນື້ອຫາ, ແລະ ຟັງຊັນການເຮັດວຽກ.

**ຄວາມສາມາດ:**

- **ການວິເຄາະເວັບໄຊ** — scrape ແລະ ວິເຄາະເວັບໄຊຄູ່ແຂ່ງ ຫຼື ເວັບໄຊເພື່ອແຮງບັນດານໃຈ
- **ການດຶງຂໍ້ມູນການອອກແບບ** — ລະບຸສີ, ຟອນ, ແລະ ຮູບແບບການຈັດວາງ
- **ໂຄງສ້າງເນື້ອຫາ** — ເຂົ້າໃຈການຈັດອົງປະກອບໜ້າ ແລະ ລຳດັບຊັ້ນ
- **ການກວດຫາຄຸນສົມບັດ** — ລະບຸ plugin ແລະ ຟັງຊັນການເຮັດວຽກ
- **ການວິເຄາະປະສິດທິພາບ** — ກວດຄວາມໄວໜ້າ ແລະ ການປັບໃຫ້ເໝາະສົມ
- **ການວິເຄາະ SEO** — ກວດທົບທວນ meta tags ແລະ ຂໍ້ມູນແບບມີໂຄງສ້າງ

**ການນຳໃຊ້:**

```
ວິເຄາະການອອກແບບຂອງ example.com ແລະ ໃຊ້ມັນເປັນແຮງບັນດານໃຈສຳລັບ site ຂອງຂ້ອຍ.
```

**ສິ່ງທີ່ມັນດຶງອອກມາ:**

- ຊຸດສີ ແລະ ການໃຊ້ສີ
- ຕົວອັກສອນ (ຟອນ ແລະ ຂະໜາດ)
- ໂຄງສ້າງການຈັດວາງ ແລະ ໄລຍະຫ່າງ
- ຮູບແບບການນຳທາງ
- ການຈັດອົງປະກອບເນື້ອຫາ
- ການໃຊ້ຮູບພາບ ແລະ ສື່
- ອົງປະກອບໂຕ້ຕອບ
- ການຕອບສະໜອງບົນມືຖື

**ຂໍ້ຈຳກັດ:**

- ບໍ່ສາມາດ scrape site ທີ່ປ້ອງກັນດ້ວຍລະຫັດຜ່ານ
- ເຄົາລົບ robots.txt ແລະ ນະໂຍບາຍ site
- ອາດຈັບເນື້ອຫາແບບໄດນາມິກບໍ່ໄດ້
- site ທີ່ໃຊ້ JavaScript ຫຼາຍອາດມີການວິເຄາະຈຳກັດ
- site ຂະໜາດໃຫຍ່ອາດໃຊ້ເວລາວິເຄາະດົນຂຶ້ນ

### ເຄື່ອງມືລະດັບ 2 (ທາງເລືອກ) {#tier-2-tools-optional}

ເຄື່ອງມືລະດັບ 2 ພ້ອມໃຊ້ງານເມື່ອເປີດໃຊ້ຢ່າງຊັດເຈນໃນການຕັ້ງຄ່າ Theme Builder.

#### ການວິເຄາະຂັ້ນສູງ {#advanced-analytics}

ໃຫ້ຕົວຊີ້ວັດປະສິດທິພາບລະອຽດ:

- ເວລາໂຫຼດໜ້າ
- Core Web Vitals
- ຄະແນນ SEO
- ຄະແນນການເຂົ້າເຖິງ
- ຄະແນນແນວປະຕິບັດທີ່ດີທີ່ສຸດ

#### ເຄື່ອງມືປັບເນື້ອຫາໃຫ້ເໝາະສົມ {#content-optimizer}

ວິເຄາະ ແລະ ແນະນຳການປັບປຸງສຳລັບ:

- ຄວາມອ່ານງ່າຍ
- ການປັບ SEO ໃຫ້ເໝາະສົມ
- ການໃຊ້ຄຳສຳຄັນ
- ໂຄງສ້າງເນື້ອຫາ
- ການວາງຕຳແໜ່ງຄຳເຊີນໃຫ້ດຳເນີນການ

### ເຄື່ອງມືລະດັບ 3 (Premium) {#tier-3-tools-premium}

ເຄື່ອງມືລະດັບ 3 ຕ້ອງການການກຳນົດຄ່າເພີ່ມເຕີມ ຫຼື ຄຸນສົມບັດ Premium.

#### ເຄື່ອງມືສ້າງເນື້ອຫາ AI {#ai-content-generator}

ສ້າງເນື້ອຫາສຳລັບ:

- ຄຳອະທິບາຍ product
- ໜ້າບໍລິການ
- ໂພສບລັອກ
- ຄຳອະທິບາຍ meta
- ຂໍ້ຄວາມຄຳເຊີນໃຫ້ດຳເນີນການ

#### ເຄື່ອງມືອອກແບບຂັ້ນສູງ {#advanced-design-tools}

ໃຫ້ຄວາມສາມາດດ້ານການອອກແບບຂັ້ນສູງ:

- ການສ້າງ CSS ກຳນົດເອງ
- ການສ້າງອະນິເມຊັນ
- ການອອກແບບອົງປະກອບໂຕ້ຕອບ
- ທິດສະດີສີຂັ້ນສູງ
- ການປັບຕົວອັກສອນໃຫ້ເໝາະສົມ

### ເຄື່ອງມື Companion ຂັ້ນສູງ (ແພັກເກດແຍກ) {#advanced-companion-tools-separate-package}

ແພັກເກດ companion ຂັ້ນສູງຖືກແຈກຈ່າຍແຍກຕ່າງຫາກຈາກການປ່ອຍ Superdav AI Agent ຫຼັກ. ມັນຖືກຕັ້ງໃຈໃຫ້ໃຊ້ສຳລັບສະພາບແວດລ້ອມນັກພັດທະນາ ແລະ ເຈົ້າຂອງ site ທີ່ໄວ້ໃຈໄດ້ ເຊິ່ງຜູ້ດູແລອະນຸຍາດຢ່າງຊັດເຈນໃຫ້ agent ໃຊ້ເຄື່ອງມືທີ່ມີຜົນກະທົບສູງ.

ຄວາມສາມາດ companion ຂັ້ນສູງອາດລວມເຖິງ:

- **ເຄື່ອງມືລະບົບໄຟລ໌** — ກວດເບິ່ງ ແລະ ແກ້ໄຂໄຟລ໌ທີ່ອະນຸມັດແລ້ວໃນສະພາບແວດລ້ອມທີ່ໄວ້ໃຈໄດ້
- **ເຄື່ອງມືຖານຂໍ້ມູນ** — ສອບຖາມ ຫຼື ອັບເດດຂໍ້ມູນ site ເມື່ອຖືກອະນຸຍາດຢ່າງຊັດເຈນ
- **ເຄື່ອງມື WP-CLI** — ຮັນຄຳສັ່ງບຳລຸງຮັກສາ ແລະ ກວດກາ WordPress
- **ເຄື່ອງມື REST dispatcher** — ເອີ້ນ REST endpoints ທີ່ລົງທະບຽນຜ່ານກະແສວຽກ dispatch ທີ່ຄວບຄຸມ
- **ເຄື່ອງມືສ້າງ plugin** — ສ້າງໂຄງຮ່າງ ແລະ ປັບປຸງຊ້ຳໆບົນໂຄດ plugin
- **ເຄື່ອງມື Git snapshot** — ສ້າງ snapshot ກ່ອນການດຳເນີນງານນັກພັດທະນາທີ່ມີຄວາມສ່ຽງ
- **ເຄື່ອງມືຈັດການຜູ້ໃຊ້** — ຊ່ວຍກວດທົບທວນ ຫຼື ປັບບັນທຶກຜູ້ໃຊ້ໃນບ່ອນທີ່ອະນຸຍາດ
- **ເຄື່ອງມື benchmark** — ຮັນ benchmark ດ້ານປະສິດທິພາບ ຫຼື ຄວາມສາມາດສຳລັບການທົບທວນການພັດທະນາ

ຢ່າບັນທຶກເຄື່ອງມືເຫຼົ່ານີ້ວ່າເປັນເຄື່ອງມື Theme Builder ທີ່ພ້ອມໃຊ້ໂດຍທົ່ວໄປສຳລັບທຸກການຕິດຕັ້ງ. ພວກມັນພ້ອມໃຊ້ງານພຽງເມື່ອແພັກເກດ companion ຂັ້ນສູງຖືກຕິດຕັ້ງ, ເປີດໃຊ້, ແລະ ຈຳກັດໄວ້ສຳລັບຜູ້ດູແລທີ່ໄວ້ໃຈໄດ້ເທົ່ານັ້ນ.

## ການກຳນົດຄ່າເຄື່ອງມື {#tool-configuration}

### ການເປີດໃຊ້ເຄື່ອງມື {#enabling-tools}

ເພື່ອເປີດໃຊ້ເຄື່ອງມືເພີ່ມເຕີມໃນ Theme Builder:

1. ໄປທີ່ **Gratis AI Agent → ການຕັ້ງຄ່າ**
2. ໄປທີ່ **Theme Builder → ເຄື່ອງມື**
3. ສະຫຼັບເປີດ/ປິດເຄື່ອງມືຕາມຕ້ອງການ
4. ບັນທຶກການຕັ້ງຄ່າ

### ສິດອະນຸຍາດເຄື່ອງມື {#tool-permissions}

ບາງເຄື່ອງມືຕ້ອງການສິດອະນຸຍາດ:

- **ການ scrape site** — ຕ້ອງການການເຂົ້າເຖິງອິນເຕີເນັດ
- **ການວິເຄາະ** — ຕ້ອງການການຜະສານລວມ Google Analytics
- **ການສ້າງເນື້ອຫາ** — ຕ້ອງການ API keys
- **ຄຸນສົມບັດຂັ້ນສູງ** — ອາດຕ້ອງການ subscription Premium
- **ເຄື່ອງມື companion ຂັ້ນສູງ** — ຕ້ອງການແພັກເກດຂັ້ນສູງທີ່ແຈກຈ່າຍແຍກຕ່າງຫາກ ແລະ ຄວາມໄວ້ໃຈຈາກຜູ້ດູແລຢ່າງຊັດເຈນ

### API Keys ແລະ ຂໍ້ມູນຢືນຢັນຕົວຕົນ {#api-keys-and-credentials}

ກຳນົດຄ່າ API keys ສຳລັບເຄື່ອງມືທີ່ຕ້ອງການ:

1. ໄປທີ່ **Gratis AI Agent → ການຕັ້ງຄ່າ → API Keys**
2. ໃສ່ຂໍ້ມູນຢືນຢັນສຳລັບແຕ່ລະເຄື່ອງມື
3. ທົດສອບການເຊື່ອມຕໍ່
4. ບັນທຶກຢ່າງປອດໄພ

### ການຕິດຕັ້ງແພັກເກດ Companion ຂັ້ນສູງ {#installing-the-advanced-companion-package}

ແພັກເກດ companion ຂັ້ນສູງຖືກປ່ອຍແຍກຕ່າງຫາກຈາກ ZIP ຫຼັກຂອງ WordPress.org. ຕິດຕັ້ງມັນຈາກຊ່ອງທາງແຈກຈ່າຍທາງການຂອງໂຄງການສຳລັບການປ່ອຍນັ້ນເທົ່ານັ້ນ, ຈາກນັ້ນຈຳກັດການເຂົ້າເຖິງໃຫ້ກັບຜູ້ດູແລທີ່ໄວ້ໃຈໄດ້. ກວດທົບທວນດ່ານອະນຸມັດໂດຍມະນຸດ ແລະ ພຶດຕິກຳ git snapshot ກ່ອນເປີດໃຊ້ລະບົບໄຟລ໌, ຖານຂໍ້ມູນ, WP-CLI, REST dispatcher, ຕົວສ້າງ plugin, ການຈັດການຜູ້ໃຊ້, ຫຼື ເຄື່ອງມື benchmark ໃນ site ການຜະລິດ.

```
ວິເຄາະການອອກແບບຂອງ site ຄູ່ແຂ່ງຂອງຂ້ອຍທີ່ competitor.com
```

```
ດຶງຊຸດສີຈາກ example.com ແລະ ໃຊ້ມັນເປັນແຮງບັນດານໃຈ.
```

```
ວິເຄາະໂຄງສ້າງການນຳທາງຂອງ example.com ແລະ ນຳໃຊ້
ການຈັດອົງປະກອບທີ່ຄ້າຍກັນກັບ site ຂອງຂ້ອຍ.
```

```
ກວດກາການຕອບສະໜອງບົນມືຖືຂອງ example.com ແລະ ຮັບປະກັນວ່າ site ຂອງຂ້ອຍ
ຕອບສະໜອງໄດ້ດີເທົ່າກັນ.
```

## ຂໍ້ຈຳກັດ ແລະ ຂໍ້ພິຈາລະນາຂອງເຄື່ອງມື {#tool-limitations-and-considerations}

### ການຈຳກັດອັດຕາ {#rate-limiting}

- ການ Scraping ຖືກຈຳກັດອັດຕາເພື່ອປ້ອງກັນ server ໂຫຼດເກີນ
- ຄຳຂໍຫຼາຍຄັ້ງໄປຍັງເວັບໄຊດຽວກັນອາດຖືກຊະລໍ
- ເວັບໄຊຂະໜາດໃຫຍ່ອາດໃຊ້ເວລານານກວ່າໃນການວິເຄາະ

### ຄວາມຖືກຕ້ອງຂອງເນື້ອຫາ {#content-accuracy}

- ເນື້ອຫາແບບໄດນາມິກອາດບໍ່ຖືກຈັບໄດ້ຄົບຖ້ວນ
- ເນື້ອຫາທີ່ສະແດງຜ່ານ JavaScript ອາດບໍ່ສົມບູນ
- ຂໍ້ມູນແບບເວລາຈິງອາດບໍ່ຖືກສະທ້ອນ
- ເນື້ອຫາບາງຢ່າງອາດຢູ່ຫຼັງ paywalls

### ຂໍ້ພິຈາລະນາດ້ານກົດໝາຍແລະຈັນຍາບັນ {#legal-and-ethical-considerations}

- ເຄົາລົບ robots.txt ແລະນະໂຍບາຍຂອງເວັບໄຊ
- ຢ່າ scrape ເນື້ອຫາທີ່ມີລິຂະສິດເພື່ອນຳໄປຜະລິດຊ້ຳ
- ໃຊ້ການວິເຄາະເພື່ອແຮງບັນດານໃຈ ບໍ່ແມ່ນເພື່ອຄັດລອກ
- ກວດສອບວ່າທ່ານມີສິດໃນການໃຊ້ເນື້ອຫາທີ່ດຶງອອກມາ
- ປະຕິບັດຕາມ terms of service ຂອງເວັບໄຊທີ່ຖືກວິເຄາະ

### ຜົນກະທົບຕໍ່ປະສິດທິພາບ {#performance-impact}

- ການ Scraping ເວັບໄຊຂະໜາດໃຫຍ່ອາດໃຊ້ເວລາ
- ການ scrape ຫຼາຍລາຍການພ້ອມກັນອາດຊ້າກວ່າ
- ການເຊື່ອມຕໍ່ network ມີຜົນຕໍ່ຄວາມໄວ
- ຊຸດຂໍ້ມູນຂະໜາດໃຫຍ່ອາດຕ້ອງການການປະມວນຜົນເພີ່ມເຕີມ

## ແນວປະຕິບັດທີ່ດີທີ່ສຸດ {#best-practices}

### ການໃຊ້ການວິເຄາະເວັບໄຊ {#using-site-analysis}

1. **ວິເຄາະຫຼາຍເວັບໄຊ** — ເກັບຮວບຮວມແຮງບັນດານໃຈຈາກຫຼາຍແຫຼ່ງ
2. **ເນັ້ນໂຄງສ້າງ** — ຮຽນຮູ້ຮູບແບບ layout ແລະການຈັດອົງກອນ
3. **ດຶງສີອອກມາ** — ໃຊ້ color palettes ເປັນຈຸດເລີ່ມຕົ້ນ
4. **ສຶກສາການຈັດຕົວອັກສອນ** — ລະບຸການຜະສົມ font ທີ່ທ່ານມັກ
5. **ທົບທວນ navigation** — ເຂົ້າໃຈການຈັດອົງກອນເມນູ

### ການ Scraping ຢ່າງມີຈັນຍາບັນ {#ethical-scraping}

1. **ເຄົາລົບ robots.txt** — ປະຕິບັດຕາມຄຳແນະນຳຂອງເວັບໄຊ
2. **ຢ່າຄັດລອກເນື້ອຫາ** — ໃຊ້ການວິເຄາະເພື່ອແຮງບັນດານໃຈເທົ່ານັ້ນ
3. **ກວດສອບສິດ** — ໃຫ້ແນ່ໃຈວ່າທ່ານສາມາດໃຊ້ເນື້ອຫາທີ່ດຶງອອກມາໄດ້
4. **ລະບຸແຫຼ່ງທີ່ມາ** — ໃຫ້ເຄຣດິດແຫຼ່ງແຮງບັນດານໃຈ
5. **ປະຕິບັດຕາມ terms** — ປະຕິບັດຕາມ terms of service ຂອງເວັບໄຊ

### ການເພີ່ມປະສິດຜົນຂອງເຄື່ອງມືໃຫ້ສູງສຸດ {#maximizing-tool-effectiveness}

1. **ໃຫ້ຈຳເພາະ** — ຂໍປະເພດການວິເຄາະທີ່ຈຳເພາະ
2. **ໃຫ້ບໍລິບົດ** — ອະທິບາຍຈຸດປະສົງຂອງເວັບໄຊຂອງທ່ານ
3. **ຕັ້ງຄວາມຄາດຫວັງ** — ອະທິບາຍເປົ້າໝາຍການອອກແບບຂອງທ່ານ
4. **ເຮັດຊ້ຳແລະປັບປຸງ** — ປັບແຕ່ງຕາມຜົນລັບ
5. **ປະສົມຜະສານເຄື່ອງມື** — ໃຊ້ຫຼາຍເຄື່ອງມືເພື່ອການວິເຄາະຢ່າງຄົບຖ້ວນ

## ການແກ້ໄຂບັນຫາ {#troubleshooting}

### ເວັບໄຊບໍ່ສາມາດ Scrape ໄດ້ {#site-wont-scrape}

- ກວດສອບວ່າເວັບໄຊເຂົ້າເຖິງໄດ້ສາທາລະນະຫຼືບໍ່
- ຢືນຢັນວ່າ robots.txt ອະນຸຍາດໃຫ້ scraping
- ລອງເວັບໄຊອື່ນ
- ກວດສອບການເຊື່ອມຕໍ່ internet
- ຕິດຕໍ່ຝ່າຍສະໜັບສະໜູນຖ້າບັນຫາຍັງຄົງຢູ່

### ການວິເຄາະບໍ່ສົມບູນ {#incomplete-analysis}

- ເວັບໄຊອາດມີເນື້ອຫາແບບໄດນາມິກ
- JavaScript ອາດບໍ່ຖືກ render ຢ່າງຄົບຖ້ວນ
- ເວັບໄຊຂະໜາດໃຫຍ່ອາດ timeout
- ລອງວິເຄາະໜ້າທີ່ຈຳເພາະແທນ
- ຂໍປະເພດການວິເຄາະທີ່ຈຳເພາະ

### ປະສິດທິພາບຊ້າ {#slow-performance}

- ເວັບໄຊຂະໜາດໃຫຍ່ໃຊ້ເວລານານກວ່າໃນການວິເຄາະ
- ການ scrape ຫຼາຍລາຍການພ້ອມກັນຈະຊ້າກວ່າ
- ການເຊື່ອມຕໍ່ network ມີຜົນຕໍ່ຄວາມໄວ
- ລອງໃນຊ່ວງເວລາທີ່ຄົນໃຊ້ໜ້ອຍ
- ວິເຄາະສ່ວນທີ່ນ້ອຍກວ່າກ່ອນ
