# การหมดอายุสมาชิกและการบล็อกเว็บไซต์

คู่มือนี้จะอธิบายว่า Ultimate Multisite จัดการกับการหมดอายุของสมาชิก การสิ้นสุดช่วงทดลองใช้ และการบล็อกเว็บไซต์ส่วนหน้าอย่างไร ครอบคลุมวงจรชีวิตของสมาชิกตั้งแต่ใช้งานอยู่จนถึงหมดอายุ การตั้งค่าที่ควบคุมว่าจะบล็อกเว็บไซต์หรือไม่ และสิ่งที่ต้องตรวจสอบเมื่อเว็บไซต์ยังคงเข้าถึงได้หลังจากสมาชิกหมดอายุ

## วงจรชีวิตสถานะสมาชิก {#membership-status-lifecycle}

สมาชิกทุกรายใน Ultimate Multisite จะมีสถานะอย่างใดอย่างหนึ่งดังต่อไปนี้:

:::note สมาชิกแบบฟรีจะมีอายุตลอดชีพ
สมาชิกแบบฟรีจะไม่หมดอายุโดยอัตโนมัติ Ultimate Multisite ถือว่าเป็นการเข้าถึงตลอดชีพ ดังนั้นจึงไม่รวมในการตรวจสอบการหมดอายุ เว้นแต่ผู้ดูแลระบบจะเปลี่ยนสถานะหรือย้ายลูกค้าไปยังผลิตภัณฑ์อื่น
:::

| สถานะ | ความหมาย |
|---|---|
| **Pending** (รอดำเนินการ) | รอการชำระเงินครั้งแรกหรือการยืนยันอีเมล |
| **Trialing** (ทดลองใช้) | ช่วงทดลองใช้งานที่ยังไม่มีการเก็บค่าธรรมเนียมใดๆ |
| **Active** (ใช้งานอยู่) | ชำระเงินแล้วและเป็นปัจจุบัน |
| **On Hold** (พักไว้) | การชำระเงินเพื่อต่ออายุรอดำเนินการ (มีการสร้างใบแจ้งหนี้ รอการชำระเงิน) |
| **Expired** (หมดอายุ) | ผ่านวันที่หมดอายุและช่วงเวลาผ่อนผันโดยไม่มีการต่ออายุ |
| **Cancelled** (ยกเลิก) | ถูกยกเลิกอย่างชัดเจนโดยลูกค้าหรือผู้ดูแลระบบ |

### การเปลี่ยนสถานะสมาชิกเป็นหมดอายุ {#how-memberships-transition-to-expired}

Ultimate Multisite จะทำการตรวจสอบเบื้องหลัง **ทุกชั่วโมง** เพื่อค้นหาสมาชิกที่ควรถูกทำเครื่องหมายว่าหมดอายุ การตรวจสอบนี้ใช้ [Action Scheduler](https://actionscheduler.org/) (ไม่ใช่ WP-Cron โดยตรง) และทำงานในรูปแบบของ scheduled action ชื่อ `wu_membership_check`

การตรวจสอบการหมดอายุมี **ช่วงเวลาผ่อนผันโดยค่าเริ่มต้น 3 วัน** สมาชิกจะไม่ถูกทำเครื่องหมายว่า `expired` จนกว่าจะผ่านไป 3 วันหลังจากที่ `date_expiration` ของสมาชิกนั้นผ่านไป สิ่งนี้จะให้เวลาลูกค้าในการชำระเงินล่าช้าให้เสร็จสิ้นก่อนที่สถานะของพวกเขาจะเปลี่ยนแปลง

:::info
ช่วงเวลาผ่อนผันการหมดอายุ 3 วันนั้นแยกต่างหากจากการตั้งค่า Grace Period ของ Frontend Block ที่อธิบายไว้ด้านล่าง ช่วงเวลาผ่อนผันการหมดอายุจะควบคุมว่า **สถานะจะเปลี่ยน** จาก active/on-hold เป็น expired เมื่อไหร่ ส่วนช่วงเวลาผ่อนผันของ frontend block จะควบคุมว่า **เว็บไซต์จะถูกบล็อก** หลังจากที่สถานะได้เปลี่ยนไปแล้ว
:::

#### การสมัครสมาชิกแบบต่ออายุอัตโนมัติ กับ แบบไม่ต่ออายุอัตโนมัติ {#auto-renewing-vs-non-auto-renewing-memberships}

ความแตกต่างนี้สำคัญมากในการทำความเข้าใจพฤติกรรมการหมดอายุ:

- **การสมัครสมาชิกแบบไม่ต่ออายุอัตโนมัติ** (`auto_renew = false`): งาน cron ทุกชั่วโมงจะจัดการวงจรชีวิตทั้งหมด -- มันจะสร้างการชำระเงินเพื่อต่ออายุ, เปลี่ยนสถานะการเป็น on-hold และในที่สุดจะทำเครื่องหมายว่าหมดอายุหากไม่ได้รับการชำระเงิน
- **การสมัครสมาชิกแบบต่ออายุอัตโนมัติ** (`auto_renew = true`): การตรวจสอบวันหมดอายุของ cron จะ **ข้ามสิ่งเหล่านี้ไปทั้งหมด** ระบบคาดหวังว่า Payment gateway (Stripe, PayPal ฯลฯ) จะแจ้ง Ultimate Multisite ผ่าน webhook เมื่อการสมัครสมาชิกไม่สำเร็จหรือถูกยกเลิก หากไม่ได้รับ webhook -- ไม่ว่าจะเกิดจาก endpoint ที่ตั้งค่าผิดพลาด, ปัญหาของ gateway หรือการยกเลิกการสมัครสมาชิกนอกระบบ -- การเป็นสมาชิกอาจยังคงเป็น `active` ไปเรื่อยๆ แม้ว่าวันหมดอายุจะผ่านไปแล้วก็ตาม

### วิธีการสิ้นสุดช่วงทดลองใช้ (Trials End) {#how-trials-end}

เมื่อช่วงทดลองใช้ของการเป็นสมาชิกแบบ trialing สิ้นสุดลง ระบบจะดำเนินการดังนี้:

1. สร้างรายการชำระเงินเพื่อต่ออายุที่รอดำเนินการด้วยจำนวนค่าบริการเต็ม
2. เปลี่ยนสถานะการเป็นสมาชิกจาก `trialing` เป็น `on-hold`
3. ส่งอีเมลแจ้งเตือนการชำระเงินเพื่อต่ออายุไปยังลูกค้า

กระบวนการนี้จะทำงานตามตารางเวลาทุกชั่วโมงเดียวกับการตรวจสอบวันหมดอายุปกติ แต่ **เฉพาะสำหรับการสมัครสมาชิกแบบไม่ต่ออายุอัตโนมัติเท่านั้น** สำหรับช่วงทดลองใช้แบบต่ออายุอัตโนมัติ Payment gateway จะจัดการการเปลี่ยนสถานะจากการทดลองใช้ไปเป็นการสมัครสมาชิกแบบชำระเงิน

โดยปกติแล้ว เมื่อสมาชิกหมดอายุหรือถูกระงับ **จะมีแค่แดชบอร์ด wp-admin เท่านั้นที่ถูกจำกัดการเข้าถึง** ส่วนหน้าเว็บไซต์สาธารณะยังคงเปิดให้ผู้เยี่ยมชมเข้าถึงได้อยู่ หากต้องการบล็อกการเข้าถึงสาธารณะด้วย คุณต้องเปิดใช้งานการตั้งค่า **Block Frontend Access**

### การตั้งค่าการตั้งค่า {#block-frontend-access}

ไปที่ **Ultimate Multisite > Settings > Memberships** แล้วเปิดใช้งาน **Block Frontend Access**

![Block Frontend Access settings showing the toggle enabled, a grace period of 7 days, and a Frontend Block Page selection](/img/config/settings-membership-block-frontend.png)

นี่คือภาพรวมเต็มของหน้าการตั้งค่าสมาชิก:

![Membership settings full page](/img/config/settings-membership-full.png)

การตั้งค่าที่เกี่ยวข้องสามอย่างนี้จะควบคุมพฤติกรรมนี้:

| การตั้งค่า | คำอธิบาย | ค่าเริ่มต้น |
|---|---|---|
| **Block Frontend Access** | สวิตช์หลัก เมื่อเปิดใช้งาน จะบล็อกส่วนหน้าสาธารณะของเว็บไซต์ในเครือเมื่อสมาชิกหมดอายุ | ปิด (Off) |
| **Frontend Block Grace Period** | จำนวนวันที่ต้องรอหลังจากที่สถานะสมาชิกไม่ทำงานก่อนที่จะทำการบล็อก ตั้งค่าเป็น `0` เพื่อบล็อกทันที | 0 |
| **Frontend Block Page** | หน้าบนเว็บไซต์หลักที่จะเปลี่ยนเส้นทางผู้เยี่ยมชมไปเมื่อมีการบล็อก หากไม่ได้ตั้งค่า ผู้เยี่ยมชมจะเห็นข้อความทั่วไปว่า "ไซต์นี้ไม่พร้อมใช้งานในขณะนี้" พร้อมลิงก์ไปยังหน้าเข้าสู่ระบบสำหรับผู้ดูแลระบบไซต์ | ไม่มี (None) |

### สิ่งที่ผู้เยี่ยมชมจะเห็นเมื่อเว็บไซต์ถูกบล็อก {#configuring-the-setting}

เมื่อการเข้าถึงส่วนหน้าถูกบล็อก ผู้เยี่ยมชมเว็บไซต์จะ:

1. **ถูกเปลี่ยนเส้นทาง** ไปยังหน้าที่คุณเลือกไว้ใน **Frontend Block Page** (ถ้ามีการตั้งค่า), หรือ
2. **เห็นข้อความเริ่มต้น**: "ไซต์นี้ไม่พร้อมใช้งานในขณะนี้" พร้อมลิงก์ไปยังหน้าเข้าสู่ระบบสำหรับผู้ดูแลระบบไซต์

ผู้ดูแลระบบเว็บไซต์ยังสามารถเข้าสู่ระบบได้ -- หน้าเข้าสู่ระบบจะไม่ถูกบล็อกเลย

### อะไรจะถูกบล็อกและเมื่อไหร่ {#what-visitors-see-when-a-site-is-blocked}

พฤติกรรมการบล็อกขึ้นอยู่กับสถานะสมาชิก:

| สถานะสมาชิก | บล็อกส่วนหน้าเปิดอยู่หรือไม่? | มีช่วงเวลาผ่อนผันหรือไม่? |
|---|---|---|
| Active (ใช้งานอยู่) | ไม่ | -- |
| Trialing (ทดลองใช้) | **ไม่** (ดูหมายเหตุข้างล่าง) | -- |
| On Hold (พักไว้) | ถือว่าใช้งานอยู่ -- ไม่ถูกบล็อก | -- |
| Expired (หมดอายุ) | **ใช่**, หากตั้งค่า Block Frontend Access เป็นเปิด | ใช่ |
| Cancelled (ยกเลิก) | **ใช่**, เสมอ (ไม่ว่าจะตั้งค่าอย่างไร) | **ไม่** -- ถูกบล็อกทันที |
| Pending (รอดำเนินการ) | ไม่ถูกบล็อกจากการตรวจสอบสมาชิก | -- |

:::warning สมาชิกประเภท Trialing จะไม่ถูกบล็อก
แม้ว่าช่วงทดลองใช้จะสิ้นสุดลงแล้ว แต่สมาชิกที่มีสถานะ `trialing` จะ**ไม่**ถูกบล็อกที่ส่วนหน้าบ้าน (frontend) จนกว่าสถานะจะเปลี่ยนไปเป็นสถานะอื่นก่อน (โดยปกติคือ `on-hold` ผ่าน cron job จากนั้นจึงจะเป็น `expired` หากยังไม่ได้ชำระเงิน) หากคุณเห็นสมาชิกที่เป็น trialing ที่ยังไม่เปลี่ยนสถานะ ให้ตรวจสอบในส่วนการแก้ไขปัญหาด้านล่างนี้
:::

:::note สมาชิกที่ถูกยกเลิกจะข้ามช่วงเวลาผ่อนผัน
สมาชิกที่ถูกยกเลิกจะถูกบล็อกทันทีเมื่อวันหมดอายุผ่านไปแล้ว ไม่ว่าคุณจะเปิดใช้งาน Block Frontend Access หรือไม่ก็ตาม ช่วงเวลาผ่อนผันของ Frontend Block จะ**ไม่**ใช้กับสมาชิกที่ถูกยกเลิก
:::

## การแก้ไขปัญหา: เว็บไซต์ยังเข้าถึงได้หลังจากหมดอายุ {#what-gets-blocked-and-when}

หากเว็บไซต์ยังสามารถเข้าถึงได้สาธารณะหลังจากสมาชิกหมดอายุ ให้ลองตรวจสอบตามลำดับดังนี้:

### 1. ตรวจสอบว่าการตั้งค่า Block Frontend Access เปิดอยู่หรือไม่ {#troubleshooting-sites-remaining-accessible-after-expiration}

ไปที่ **Ultimate Multisite > Settings > Memberships** และยืนยันว่าปุ่มสลับ (toggle) **Block Frontend Access** นั้นเปิดอยู่ การตั้งค่านี้จะ**ปิดเป็นค่าเริ่มต้น** ซึ่งหมายความว่าจะมีเพียง wp-admin เท่านั้นที่ถูกจำกัดเมื่อสถานะสมาชิกไม่ใช้งาน

### 2. ตรวจสอบช่วงเวลาผ่อนผันของ Frontend Block {#1-verify-the-block-frontend-access-setting-is-enabled}

ในหน้าการตั้งค่าเดียวกัน ให้ตรวจสอบค่า **Frontend Block Grace Period** (ระยะเวลาผ่อนผันการบล็อกส่วนหน้า) หากตั้งค่าเป็น 7 วัน ตัวอย่างเช่น ส่วนหน้าเว็บจะไม่ถูกบล็อกจนกว่าจะครบ 7 วันหลังจากวันที่หมดอายุของสมาชิก แม้ว่าสถานะสมาชิกจะเป็น `expired` แล้วก็ตาม

ให้ตั้งค่าเป็น `0` หากคุณต้องการให้มีการบล็อกทันทีหลังจากสมาชิกไม่ใช้งานแล้ว

### 3. ยืนยันว่าสถานะสมาชิกได้เปลี่ยนแปลงจริง {#2-check-the-frontend-block-grace-period}

ไปที่ **Ultimate Multisite > Memberships** และตรวจสอบสถานะของสมาชิกที่ได้รับผลกระทบ หากยังแสดงเป็น `active` ทั้งๆ ที่วันที่หมดอายุผ่านไปแล้ว แสดงว่าการเปลี่ยนสถานะยังไม่ได้เกิดขึ้น สาเหตุทั่วไป:

- **สมาชิกรีเนวเวิร์คโดยอัตโนมัติ**: ตรวจสอบฟิลด์ `auto_renew` ในหน้าแก้ไขสมาชิก หากเปิดใช้งาน auto-renew ตัว cron job การหมดอายุจะข้ามสมาชิกนี้ไป โดยจะพึ่งพา payment gateway เพื่อรายงานความล้มเหลว โปรดตรวจสอบแดชบอร์ดของเกตเวย์ของคุณ (Stripe, PayPal) เพื่อยืนยันว่าสถานะการสมัครสมาชิกตรงกับที่ Ultimate Multisite แสดง

- **cron job ยังไม่ได้ทำงาน**: ดูขั้นตอนถัดไป

### 4. ตรวจสอบว่า Action Scheduler กำลังทำงานอยู่หรือไม่ {#3-confirm-the-membership-status-has-actually-changed}

Ultimate Multisite ใช้ Action Scheduler สำหรับ cron jobs ของมัน ไปที่ **Tools > Scheduled Actions** ในส่วน network admin แล้วมองหา:

- **`wu_membership_check`** -- สิ่งนี้ควรปรากฏเป็น action ที่ทำงานซ้ำทุกชั่วโมง หากไม่มี แสดงว่าการตรวจสอบสมาชิกไม่ได้ถูกตั้งเวลาไว้
- **`wu_async_mark_membership_as_expired`** -- นี่คือ task รายบุคคลเพื่อทำเครื่องหมายสมาชิกบางรายว่าหมดอายุ หากคุณเห็น action ที่ล้มเหลวที่นี่ อาจมีข้อความแสดงข้อผิดพลาดอธิบายสาเหตุได้

หากคุณเห็น action ที่ค้างหรือล้มเหลว คุณอาจมีปัญหาเกี่ยวกับ Action Scheduler สาเหตุทั่วไป:

- **`DISABLE_WP_CRON` ถูกตั้งค่าเป็น `true`** ในไฟล์ `wp-config.php` โดยที่ไม่มีการแทนที่ด้วย cron ระดับระบบ

- **ทราฟฟิกเว็บไซต์ต่ำ** -- WP-Cron จะทำงานก็ต่อเมื่อมีคนเข้าชมเว็บไซต์เท่านั้น

เพื่อให้แน่ใจว่า cron ทำงานได้อย่างน่าเชื่อถือ ให้ตั้งค่า cron job ของระบบ:

```bash
# รันทุก 5 นาทีผ่าน wget
*/5 * * * * wget -q -O /dev/null "https://your-network-url.com/wp-cron.php?doing_wp_cron"

# หรือผ่าน WP-CLI
*/5 * * * * cd /path/to/wordpress && wp cron event run --due-now --url=https://your-network-url.com
```

### 5. ตรวจสอบปัญหา Gateway Webhook (การต่ออายุสมาชิกอัตโนมัติ) {#4-verify-action-scheduler-is-running}

หากการเป็นสมาชิกมีการต่ออายุอัตโนมัติ แต่การสมัครสมาชิกของ gateway ถูกยกเลิกหรือล้มเหลว แต่ Ultimate Multisite ยังแสดงสถานะเป็น `active` อยู่:

- **Stripe**: ไปที่ Stripe Dashboard > Customers แล้วตรวจสอบสถานะการสมัครสมาชิก จากนั้นตรวจสอบว่า webhook endpoint ทำงานอยู่ภายใต้ Developers > Webhooks หรือไม่ Endpoint นั้นควรชี้ไปยังเว็บไซต์ของคุณและแสดงการส่งข้อมูลที่สำเร็จ
- **PayPal**: ตรวจสอบสถานะการสมัครสมาชิกในบัญชีธุรกิจ PayPal ของคุณ และตรวจสอบการส่ง IPN/webhook

หาก gateway แสดงว่าการสมัครสมาชิกถูกยกเลิก แต่ Ultimate Multisite ไม่แสดงเช่นนั้น เป็นไปได้ว่าการแจ้งเตือน webhook สูญหายไป คุณสามารถเปลี่ยนสถานะการเป็นสมาชิกด้วยตนเองใน **Ultimate Multisite > Memberships > [Edit Membership]** ได้

### 6. ตรวจสอบช่วงเวลาผ่อนผันก่อนหมดอายุ (ระดับ Cron) {#5-check-for-gateway-webhook-issues-auto-renewing-memberships}

การตรวจสอบ cron มีช่วงเวลาผ่อนผันของตัวเอง (ค่าเริ่มต้น: 3 วัน) ก่อนที่จะทำเครื่องหมายว่าการเป็นสมาชิกหมดอายุ ซึ่งแยกต่างหากจากช่วงเวลาผ่อนผันของการบล็อกส่วนหน้าบ้าน ระยะเวลารวมก่อนที่เว็บไซต์จะถูกบล็อกคือ:

**ช่วงเวลาผ่อนผันก่อนหมดอายุ (3 วัน)** + **ช่วงเวลาผ่อนผันในการบล็อกส่วนหน้าบ้าน (การตั้งค่าของคุณ)** = ความล่าช้าโดยรวม

ຕົວຢ່າງ, ຖ້າຕັ້ງຄ່າເລີ່ມຕົ້ນ ແລະ ມີໄລຍະເວລາຜ່ານໄປ 7 ວັນສຳລັບສ່ວນໜ້າ (frontend grace period), ອາດຈະໃຊ້ເວລາເຖິງ 10 ມື້ຫຼັງຈາກ `date_expiration` ກ່ອນທີ່ເວັບໄຊຈະຖືກປິດ.

### 7. ການໝົດອາຍຸສະມາຊິກດ້ວຍຕົນເອງ (Manually Expire a Membership) {#6-check-the-expiration-grace-period-cron-level}

ຖ້າທ່ານຕ້ອງການປິດເວັບໄຊທັນທີໂດຍບໍ່ຕ້ອງລໍຖ້າວົງຈອນ cron, ທ່ານສາມາດປ່ຽນສະຖານະຂອງສະມາຊິກໄດ້ດ້ວຍຕົນເອງ:

1. ໄປທີ່ **Ultimate Multisite > Memberships**
2. ເລືອກສະມາຊິກທີ່ຕ້ອງການປິດ
3. ປ່ຽນສະຖານະເປັນ **Expired** (ໝົດອາຍຸ) ຫຼື **Cancelled** (ຍົກເລີກ)
4. ກົດ **Save**

ການປິດສ່ວນໜ້າຈະມີຜົນໃນການໂຫຼດໜ້າຕໍ່ໄປ (ຂຶ້ນກັບໄລຍະເວລາຜ່ານຂອງ Frontend Block Grace Period ສຳລັບສະມາຊິກທີ່ໝົດອາຍຸ, ຫຼື ທັນທີສຳລັບສະມາຊິກທີ່ຖືກຍົກເລີກ).

## ສະຫຼຸບ {#7-manually-expire-a-membership}

ໄລຍະເວລາເຕັມຕັ້ງແຕ່ມື້ໝົດອາຍຸຈົນເຖິງການປິດເວັບໄຊ:

```text
date_expiration ຜ່ານໄປ
       |
       v
  [ໄລຍະເວລາ grace period 3 ວັນຂອງ cron]     <-- ສະມາຊິກຍັງສະແດງວ່າ active/on-hold
       |
       v
  Cron ໝາຍໄວ້ວ່າສະມາຊິກ "expired" (ໝົດອາຍຸ)
       |
       v
  [ໄລຍະເວລາ grace period Frontend Block]  <-- ຕັ້ງຄ່າໃນ Settings > Memberships
       |
       v
  ສ່ວນໜ້າຂອງເວັບໄຊຖືກປິດ
```

ສຳລັບສະມາຊິກທີ່ຖືກຍົກເລີກ, ຂັ້ນທາງຈະສັ້ນກວ່າ:

```text
  Membership cancelled (ສະມາຊິກຖືກຍົກເລີກ)
       |
       v
  date_expiration ຜ່ານໄປ (ບໍ່ມີໄລຍະເວລາ grace period)
       |
       v
  ສ່ວນໜ້າຂອງເວັບໄຊຖືກປິດທັນທີ
```

## ອ້າງອີງສຳລັບນັກພັດທະນາ (Developer Reference) {#summary}

Hooks ແລະ Filters ຂ້າງລຸ່ມນີ້ຊ່ວຍໃຫ້ທ່ານສາມາດປັບແຕ່ງພຶດຕິກຳການໝົດອາຍຸ ແລະ ການປິດເວັບໄຊໄດ້:

| Hook/Filter | คำอธิบาย |
|---|---|
| `wu_membership_grace_period_days` | กรองจำนวนวันพัก (grace period days) ก่อนที่จะทำเครื่องหมายว่าสมาชิกหมดอายุ (ค่าเริ่มต้น: 3 วัน) |
| `wu_schedule_membership_check_interval` | กรองช่วงเวลาการตรวจสอบสถานะสมาชิก (interval) ระหว่างการตรวจสอบ (ค่าเริ่มต้น: 1 ชั่วโมง) |
| `wu_membership_renewal_days_before_expiring` | กรองจำนวนวันที่ต้องแจ้งเตือนก่อนที่สมาชิกจะหมดอายุ เพื่อสร้างการชำระเงินต่ออายุ (default: 3 วัน) |
| `wu_blocked_site_reactivation_url` | กรองเพื่อกำหนด URL สำหรับการเปิดใช้งานใหม่ (reactivation URL) แบบกำหนดเอง เมื่อเว็บไซต์ถูกบล็อก |
| `wu_membership_is_active` | กรองว่าสถานะสมาชิกนั้นถือว่า "ใช้งานอยู่" หรือไม่ |
| `wu_membership_expired_check_query_params` | กรองพารามิเตอร์ของคำสั่ง (query parameters) ที่ใช้ในการค้นหาสมาชิกที่หมดอายุ |
| `wu_membership_trial_check_query_params` | กรองพารามิเตอร์ของคำสั่ง (query parameters) ที่ใช้ในการค้นหาช่วงทดลองใช้ (trials) ที่หมดอายุ |
