# ດຶງຂໍ້ມູນອອກ ແລະ ນຳເຂົ້າ (Export & Import)

Ultimate Multisite 2.9.0 ເພີ່ມເຄື່ອງມື **Export & Import** (ດຶງຂໍ້ມູນອອກ ແລະ ນຳເຂົ້າ) ໜຶ່ງຊຸດຢູ່ພາຍໃຕ້ເມນູ **Tools > Export & Import**. ໃຊ້ເຄື່ອງມືນີ້ເວລາທີ່ທ່ານຕ້ອງການເກັບໂຄງການ WordPress site ດຽວເປັນໄຟລ໌ ZIP, ກູ້ຄືນໄຟລ໌ ZIP ນັ້ນ, ຫຼື ຍ້າຍ site ລະຫວ່າງການຕິດຕັ້ງ Ultimate Multisite ແລະ single-site WordPress ທີ່ເຂົ້າກັນໄດ້.

## ສິດທິທີ່ຕ້ອງການ (Required permissions)

ທ່ານຕ້ອງເຂົ້າສູ່ລະບົບໃນຖານະຜູ້ບໍລິຫານ (administrator) ທີ່ມີສິດເຂົ້າເຖິງເມນູ **Tools** ໃນເວັບໄຊທີ່ກຳລັງຈະດຶງຂໍ້ມູນອອກ ຫຼື ນຳເຂົ້າ. ໃນເຄືອຂ່າຍ multisite, ໃຫ້ໃຊ້ບັນຊີຜູ້ບໍລິຫານຂອງເຄືອຂ່າຍ ເມື່ອດຶງຂໍ້ມູນອອກ ຫຼື ນຳເຂົ້າ subsites ຈາກເຄື່ອງມື Ultimate Multisite ໃນລະດັບເຄືອຂ່າຍ.

ການດາວໂຫຼດ ZIP ທີ່ດຶງຂໍ້ມູນອອກຈະຖືກສະໜອງຜ່ານ endpoint ການດາວໂຫຼດທີ່ມີການຢັ້ງຢືນ (authenticated download endpoint), ສະນັ້ນຄວນຮັກສາ session ຂອງ admin ໃຫ້ເປີດຢູ່ຈົນກວ່າການດາວໂຫຼດຈະສຳເລັດ ແລະ ຫ້າມແບ່ງປັນ URL ການດາວໂຫຼດທີ່ສ້າງຂຶ້ນອອກໄປໃນທີ່ສາທາລະນະ.

## ການດຶງຂໍ້ມູນ site ໄປເປັນ ZIP (Exporting a site to a ZIP)

1. ໃນ WordPress admin ຂອງ site ທີ່ທ່ານຕ້ອງການຄັດລອກ, ໃຫ້ໄປທີ່ **Tools > Export & Import**.
2. ເປີດພື້ນທີ່ດຶງຂໍ້ມູນອອກ ແລະ ເລືອກ site ທີ່ທ່ານຕ້ອງການເກັບໄວ້ໃນໄຟລ໌.
3. ເລືອກເນື້ອຫາທາງເລືອກທີ່ຈະໃສ່ເຂົ້າໄປ, ເຊັ່ນ: ການອັບໂຫຼດ (uploads), plugins, ແລະ themes, ເມື່ອມີຕົວເລືອກເຫຼົ່ານັ້ນສະແດງຂຶ້ນມາ.
4. ເລີ່ມການດຶງຂໍ້ມູນອອກ ແລະ ລໍຖ້າໃຫ້ຂະບວນການສຳເລັດ. site ຂະໜາດໃຫຍ່ອາດຈະຕ້ອງໃຊ້ເວລາໃນການເຮັດວຽກຢູ່ເບື້ອງຫຼັງກ່ອນທີ່ໄຟລ໌ ZIP ຈະພ້ອມ.
5. ດາວໂຫຼດໄຟລ໌ ZIP ທີ່ສຳເລັດຈາກລາຍການ exports.

ຄວນເກັບໄຟລ໌ ZIP ນັ້ນໄວ້ໃນຕຳແໜ່ງທີ່ປອດໄພ. ມັນອາດຈະມີເນື້ອຫາຂອງ site, ການຕັ້ງຄ່າ (settings), ໄຟລ໌ media, ແລະ ຊຸດ code ທີ່ເລືອກໄວ້.

## ສິ່ງທີ່ການດຶງຂໍ້ມູນອອກເກັບໄວ້ (What the export includes)

ໄຟລ໌ ZIP ທີ່ດຶງຂໍ້ມູນອອກສາມາດມີສິ່ງເຫຼົ່ານີ້ໄດ້:

- ເນື້ອໃນ database ແລະ ການຕັ້ງຄ່າຂອງ site ທີ່ເລືອກ.
- ໄຟລ໌ media ທີ່ອັບໂຫຼດເຂົ້າໄປ ເມື່ອໄດ້ເລືອກຕົວເລືອກການອັບໂຫຼດ.
- plugins ແລະ themes ເມື່ອໄດ້ເລືອກຕົວເລືອກເຫຼົ່ານັ້ນ.
- metadata ທີ່ຈະນຳໃຊ້ໃນການນຳເຂົ້າ (Import) ໂດຍເຄື່ອງມື Export & Import ເພື່ອສ້າງ site ຄືນໃໝ່ໃນການຕິດຕັ້ງປາຍທາງ (target installation).

ขนาดไฟล์ ZIP ที่แน่นอนจะขึ้นอยู่กับปริมาณสื่อ (media), ปลั๊กอินและธีมที่เลือก รวมถึงขนาดของตารางฐานข้อมูลของเว็บไซต์ด้วย

## การนำเข้าเว็บไซต์จากไฟล์ ZIP

1. ไปที่ **Tools > Export & Import** บนเว็บไซต์ WordPress ปลายทางของคุณ
2. เปิดพื้นที่สำหรับนำเข้า และอัปโหลดไฟล์ ZIP ที่สร้างโดยเครื่องมือ Export & Import
3. ใส่ URL ทดแทน (replacement URL) หากเว็บไซต์ควรใช้ที่อยู่ใหม่ หรือปล่อยช่องนี้ว่างไว้เพื่อเก็บ URL เดิม
4. เลือกว่าจะลบไฟล์ ZIP ที่อัปโหลดหลังจากนำเข้าเสร็จหรือไม่
5. คลิก **Begin Import**
6. ติดตามการนำเข้าที่รอดำเนินการจนกว่าจะเสร็จสิ้น ใช้ **Cancel Import** ก็ต่อเมื่อคุณต้องการหยุดกระบวนการก่อนที่จะเสร็จสมบูรณ์เท่านั้น
7. ตรวจสอบเว็บไซต์ที่นำเข้าแล้วก่อนอนุญาตให้มีการเข้าชมปกติหรือการเข้าถึงของลูกค้า

ในการติดตั้ง WordPress แบบเว็บไซต์เดียว (single-site) การนำเข้าไฟล์ ZIP จะแทนที่เว็บไซต์ปัจจุบันด้วยเว็บไซต์ที่นำเข้า สร้างสำรองข้อมูลแบบเต็มของเว็บไซต์เป้าหมายก่อนที่คุณจะเริ่ม และหลีกเลี่ยงการเริ่มนำเข้าหลายครั้งสำหรับเว็บไซต์เดียวกันพร้อมกัน

โฟลเดอร์อัปโหลดขนาดใหญ่มากหรือไลบรารีสื่ออาจสร้างไฟล์ ZIP ขนาดใหญ่ได้ โปรดตรวจสอบขีดจำกัดการอัปโหลดของ PHP, ขีดจำกัดการประมวลผล, พื้นที่ดิสก์, หน่วยความจำ และการตั้งค่า timeout ของเซิร์ฟเวอร์ ก่อนที่จะทำการส่งออก (export) หรือนำเข้า (import) เว็บไซต์ขนาดใหญ่

ไลบรารีสื่อที่มีขนาดใหญ่มากอาจจำเป็นต้องย้ายระหว่างช่วงเวลาบำรุงรักษาที่มีผู้ใช้งานน้อย

การติดตั้ง WordPress ปลายทางควรใช้ WordPress, PHP, Ultimate Multisite, plugin และ theme เวอร์ชันที่เข้ากันได้

การนำเข้าเว็บไซต์แบบ single-site จะแทนที่เว็บไซต์ปลายทาง ไม่ใช่เครื่องมือสำหรับรวมข้อมูล (merge tool)

การย้ายจาก Multisite ไปเป็น single-site และจาก single-site ไปเป็น Multisite จะรองรับก็ต่อเมื่อสภาพแวดล้อมปลายทางสามารถรัน plugin, theme, URL และส่วนประกอบ Ultimate Multisite ที่จำเป็นของเว็บไซต์ที่ส่งออกได้เท่านั้น

โปรดเก็บไฟล์ ZIP เป็นความลับ เพราะอาจมีเนื้อหาฐานข้อมูล, ไฟล์ที่อัปโหลด และรายละเอียดการตั้งค่าจากเว็บไซต์ที่ส่งออกอยู่ด้วย

สำหรับเวิร์กโฟลว์การส่งออกระดับเครือข่ายแบบเก่า โปรดดูที่ [Site Export](/user-guide/administration/site-export)
