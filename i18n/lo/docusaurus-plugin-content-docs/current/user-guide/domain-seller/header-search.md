# ການຄົ້ນຫາໂດເມນໃນສ່ວນຫົວ

ໃຊ້ການຕັ້ງຄ່ານີ້ເມື່ອທ່ານຕ້ອງການຟອມສ່ວນຫົວຂະໜາດນ້ອຍທີ່ເລີ່ມການຄົ້ນຫາໂດເມນ, ແລ້ວໃຫ້ລູກຄ້າເລືອກຈາກໂດເມນທີ່ມີຢູ່ພາຍໃນການຊຳລະເງິນຂອງ Ultimate Multisite.

## ຂໍ້ກຳນົດ {#requirements}

- Ultimate Multisite ເປີດໃຊ້ງານລະດັບເຄືອຂ່າຍ.
- Multisite Ultimate Domain Seller ເປີດໃຊ້ງານລະດັບເຄືອຂ່າຍ.
- ຢ່າງໜ້ອຍມີສິນຄ້າລົງທະບຽນໂດເມນທີ່ເປີດໃຊ້ຢູ່ໜຶ່ງລາຍການພ້ອມກັບ:
  - `domain_provider` ຖືກຕັ້ງໃຫ້ເປັນຜູ້ໃຫ້ບໍລິການທີ່ກຳນົດຄ່າແລ້ວ.
  - TLD ທີ່ຮອງຮັບຖືກກຳນົດຄ່າແລ້ວ, ຕົວຢ່າງ `com`, `net`, ແລະ `org`.
- ຟອມຊຳລະເງິນທີ່ຖືກຕ້ອງເຊິ່ງມີຊ່ອງ **ການເລືອກໂດເມນ**.

## ຟອມຊຳລະເງິນ {#checkout-form}

1. ສ້າງ ຫຼື ແກ້ໄຂຟອມຊຳລະເງິນທີ່ໃຊ້ໂດຍໜ້າລົງທະບຽນ.
2. ເພີ່ມຊ່ອງຊຳລະເງິນ/ບັນຊີປົກກະຕິທີ່ຈຳເປັນ, ລວມທັງ **ຊື່ຜູ້ໃຊ້**. ຟອມຊຳລະເງິນທີ່ມີແຕ່ຊ່ອງໂດເມນຢ່າງດຽວຈະຖືກການກວດສອບຂອງ Ultimate Multisite ປະຕິເສດ.
3. ເພີ່ມຊ່ອງ **ການເລືອກໂດເມນ**.
4. ຕັ້ງໂໝດການເລືອກໂດເມນເປັນ **ລົງທະບຽນເທົ່ານັ້ນ** ເມື່ອຂັ້ນຕອນຄວນເນັ້ນໃສ່ໂດເມນທີ່ລົງທະບຽນ ແທນທີ່ຈະເປັນໂດເມນຍ່ອຍຟຣີ ຫຼື ໂດເມນທີ່ມີຢູ່ແລ້ວ.
5. ກຳນົດສິນຄ້າລົງທະບຽນໂດເມນໃຫ້ກັບຊ່ອງນັ້ນ.

ໜ້າລົງທະບຽນຄວນສະແດງຟອມຊຳລະເງິນ, ຕົວຢ່າງ:

```text
[wu_checkout slug="domain-form"]
```

## ຟອມສ່ວນຫົວ {#header-form}

ເພີ່ມຟອມ `GET` ຂະໜາດນ້ອຍໃນສ່ວນຫົວຂອງເວັບໄຊທີ່ສົ່ງຄຳຄົ້ນຫາທີ່ປ້ອນໄປຫາໜ້າຊຳລະເງິນເປັນ `domain_name`:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

ຢ່າເລືອກໂດເມນໄວ້ລ່ວງໜ້າໃນ JavaScript ສ່ວນຫົວແບບກຳນົດເອງ. ສ່ວນຫົວຄວນພຽງແຕ່ສົ່ງຄຳຄົ້ນຫາ. ສະຄຣິບຊຳລະເງິນຂອງ Domain Seller ອ່ານ `?domain_name=example`, ຕື່ມກ່ອງຄົ້ນຫາການຊຳລະເງິນ, ແລະ ເຮັດການຄົ້ນຫາຄວາມພ້ອມໃຊ້ ເພື່ອໃຫ້ລູກຄ້າເລືອກຈາກໂດເມນທີ່ສົ່ງກັບມາ.

## ພຶດຕິກຳທີ່ຄາດຫວັງ {#expected-behaviour}

ການຄົ້ນຫາ `example` ໃນສ່ວນຫົວຄວນເປີດ:

```text
/register/?domain_name=example
```

ຈາກນັ້ນການຊຳລະເງິນຄວນສະແດງຜົນລັບທີ່ເລືອກໄດ້ເຊັ່ນ:

- `example.com`
- `example.net`
- `example.org` ບໍ່ພ້ອມໃຊ້
- TLD ອື່ນໆທີ່ຜູ້ໃຫ້ບໍລິການຮອງຮັບ

ຫຼັງຈາກເລືອກຜົນລັບທີ່ພ້ອມໃຊ້, ສະຫຼຸບຄຳສັ່ງຊື້ຄວນລວມມີສິນຄ້າລົງທະບຽນໂດເມນ ແລະ ຊື່ໂດເມນທີ່ເລືອກ.

## ການກວດສອບ {#verification}

1. ເປີດໜ້າຫຼັກ.
2. ຄົ້ນຫາຊື່ເປົ່າ, ຕົວຢ່າງ `example`.
3. ຢືນຢັນວ່າ URL ການຊຳລະເງິນມີ `?domain_name=example`.
4. ຢືນຢັນວ່າຊ່ອງໂດເມນການຊຳລະເງິນມີ `example`.
5. ຢືນຢັນວ່າລາຍຊື່ຕົວເລືອກໂດເມນປາກົດຂຶ້ນ.
6. ຄລິກ **ເລືອກ** ສຳລັບໂດເມນທີ່ພ້ອມໃຊ້.
7. ຢືນຢັນວ່າສະຫຼຸບຄຳສັ່ງຊື້ມີ `Domain Registration - example.com` ຫຼື ໂດເມນທີ່ເລືອກ.

## ການແກ້ໄຂບັນຫາ {#troubleshooting}

- ຖ້າບໍ່ມີລາຍຊື່ປາກົດ, ກວດເບິ່ງແຖບເຄືອຂ່າຍຂອງເບຣາວເຊີສຳລັບ `admin-ajax.php?action=wu_domain_search` ແລະ ຢືນຢັນວ່າມັນສົ່ງກັບ `domains` ຫຼື `results` ທີ່ບໍ່ຫວ່າງ.
- ຖ້າຟອມຊຳລະເງິນກວດສອບບໍ່ຜ່ານໃນຂະນະບັນທຶກ, ເພີ່ມຊ່ອງບັນຊີທີ່ຈຳເປັນເຊັ່ນ **ຊື່ຜູ້ໃຊ້**.
- ຖ້າການເລືອກໂດເມນບໍ່ອັບເດດກະຕ່າ, ຢືນຢັນວ່າ `window.wu_checkout_form` ມີຢູ່ ແລະ ຊັບສິນການຊຳລະເງິນຂອງ Domain Seller ຖືກໂຫຼດໃນໜ້າຊຳລະເງິນ.
