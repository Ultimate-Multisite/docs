# ผลิตภัณฑ์โดเมนและการตั้งราคา

ผลิตภัณฑ์โดเมนคือวิธีที่คุณจะควบคุมเรื่องราคา, TLDs (ประเภทของชื่อโดเมน), ความเป็นส่วนตัว WHOIS และการเลือกผู้ให้บริการ แต่ละผลิตภัณฑ์โดเมนจะเป็นผลิตภัณฑ์มาตรฐานของ Ultimate Multisite พร้อมแท็บ **Domain Settings** เพิ่มเติม

## การสร้างผลิตภัณฑ์โดเมน

1. ไปที่ **Network Admin › Ultimate Multisite › Products**
2. คลิก **Add New** (เพิ่มใหม่)
3. ตั้งค่าประเภทผลิตภัณฑ์เป็น **Domain** (โดเมน)
4. กำหนดค่าแท็บ **Domain Settings** (ดูด้านล่าง)
5. บันทึก

ผลิตภัณฑ์โดเมนจะปรากฏพร้อมป้าย **Domain** สีม่วงในรายการผลิตภัณฑ์ และสามารถกรองได้โดยใช้แท็บ **Domain Products**

## แท็บ Domain settings

### ผู้ให้บริการ (Provider)

เลือกผู้จดทะเบียนที่จะจัดการการลงทะเบียนสำหรับผลิตภัณฑ์นี้ โดยค่าเริ่มต้นจะเป็น **Default provider** ทั่วโลกที่ตั้งไว้ในการตั้งค่า Domain Seller

### TLDs ที่รองรับ (Supported TLDs)

ปล่อยว่างไว้เพื่อสร้าง **catch-all product** (ผลิตภัณฑ์ครอบคลุมทั้งหมด) ที่ใช้กับทุก TLD ที่ไม่ตรงกับผลิตภัณฑ์อื่น ๆ

ใส่รายการ TLDs คั่นด้วยเครื่องหมายจุลภาค (เช่น `.com, .net, .org`) เพื่อสร้าง **TLD-specific product** (ผลิตภัณฑ์เฉพาะ TLD) ที่ใช้ได้เฉพาะส่วนขยายเหล่านั้นเท่านั้น

**วิธีการจับคู่ผลิตภัณฑ์:** เมื่อลูกค้าค้นหาโดเมน ส่วนเสริมนี้จะเลือกผลิตภัณฑ์ที่ตรงกันมากที่สุด ผลิตภัณฑ์ที่มี `.com` ในรายการ TLD จะมีความสำคัญเหนือกว่าผลิตภัณฑ์แบบ catch-all หากไม่มีผลิตภัณฑ์เฉพาะ TLD ใดตรงกัน ระบบจะใช้แบบ catch-all หากไม่มีผลิตภัณฑ์ใดเลย การค้นหาโดเมนจะไม่แสดงผล

### ประเภทการตั้งราคา (Markup type)

มีสามโหมดที่ควบคุมวิธีการคำนวณราคาขายปลีกของคุณจากต้นทุนส่ง (wholesale cost):

| โหมด | วิธีการทำงาน |
|---|---|
| **Percentage** (เปอร์เซ็นต์) | บวกเปอร์เซ็นต์เข้าไปในต้นทุนส่ง เช่น การบวกเพิ่ม 20% จากต้นทุนโดเมนที่ 10 ดอลลาร์ จะได้ราคา 12 ดอลลาร์ |
| **Fixed markup** (การบวกคงที่) | บวกจำนวนเงินที่เป็นตัวเลขคงที่ เช่น การบวกเพิ่ม 5 ดอลลาร์จากโดเมนราคา 10 ดอลลาร์ จะได้ราคา 15 ดอลลาร์ |
| **Fixed price** (ราคาคงที่) | ไม่สนใจต้นทุนส่งเลย จะคิดตามจำนวนที่คุณป้อนเสมอ |

### ราคาแนะนำ (Introductory pricing)

เปิดใช้งานเพื่อเสนอราคาพิเศษสำหรับปีแรก. ตั้งค่า **Introductory price** (ราคาปีที่ 1) แยกต่างหากจาก **Renewal price** (ราคาต่ออายุ ปีที่ 2 ขึ้นไป) ปกติ ลูกค้าจะเห็นทั้งสองราคาแสดงระหว่างการชำระเงิน เพื่อให้พวกเขาทราบว่าจะคาดหวังอะไรเมื่อถึงเวลาต่ออายุ.

### ความเป็นส่วนตัว WHOIS

ควบคุมว่ามีการเสนอการป้องกันความเป็นส่วนตัวของ WHOIS สำหรับโดเมนที่ลงทะเบียนผ่านผลิตภัณฑ์นี้หรือไม่.

| การตั้งค่า | พฤติกรรม |
|---|---|
| **Disabled** (ปิดใช้งาน) | ไม่มีการเสนอหรือเปิดใช้งานการป้องกันความเป็นส่วนตัวของ WHOIS เลย. |
| **Always Included** (รวมเสมอ) | การป้องกันความเป็นส่วนตัวของ WHOIS จะถูกเปิดใช้งานโดยอัตโนมัติเมื่อลงทะเบียนโดยไม่มีค่าใช้จ่าย. |
| **Customer Choice** (ทางเลือกของลูกค้า) | จะมีช่องทำเครื่องหมายปรากฏขึ้นระหว่างการชำระเงิน ตั้งค่า **Privacy price** ให้คิดเป็นรายปี หรือปล่อยไว้ที่ $0 เพื่อเสนอให้ฟรี. |

สำหรับ Namecheap, WHOIS privacy ใช้ WhoisGuard (ฟรีเสมอ). สำหรับ OpenSRS ใช้วิธีบริการความเป็นส่วนตัวของ OpenSRS (อาจมีค่าใช้จ่ายเมื่อซื้อแบบขายส่ง).

## การนำเข้าและซิงค์ TLD

ผลิตภัณฑ์โดเมนจะแสดงราคาขายส่งแบบเรียลไทม์ที่ดึงมาจากผู้ให้บริการที่เชื่อมต่ออยู่. เพื่อให้ทำงานได้ โดเมน TLD จะต้องถูกนำเข้า (import) เข้ามาด้วย.

- **Manual sync** (ซิงค์ด้วยตนเอง): Settings › Domain Seller › Sync TLDs (ตามผู้ให้บริการแต่ละราย)
- **Automatic sync** (ซิงค์อัตโนมัติ): ทำงานทุกวันผ่าน cron job ที่ตั้งเวลาไว้ในผู้ให้บริการทั้งหมดที่กำหนดค่าไว้.

หลังจากซิงค์แล้ว ให้ไปที่แท็บ Domain Settings ของผลิตภัณฑ์โดเมนใดก็ได้ และใช้ตัวเลือก TLD picker เพื่อดู TLD ที่มีให้พร้อมกับราคาขายส่งปัจจุบันของมัน.

## การต่ออายุอัตโนมัติ (Auto-renewal)

การต่ออายุโดเมนจะผูกอยู่กับสถานะสมาชิกของลูกค้า:

- เมื่อสมาชิกมีการต่ออายุและเชื่อมโยงโดเมนแล้ว การต่ออายุโดเมนจะถูกจัดคิวโดยอัตโนมัติ.
- ความพยายามในการต่ออายุจะใช้ Payment gateway ที่ลูกค้าใช้งานอยู่.
- การต่ออายุที่ล้มเหลวจะถูกลองใหม่โดยอัตโนมัติพร้อมการหน่วงเวลาแบบทวีคูณ (exponential backoff).
- จะมีการส่งอีเมลแจ้งเตือนสำหรับการต่ออายุที่สำเร็จ, ความล้มเหลว และการหมดอายุที่กำลังจะมาถึง.

ID เทมเพลตอีเมลสำหรับเหตุการณ์วงจรชีวิตโดเมน:

| Event | Template ID |
|---|---|
| ໂດເມນລົງທະບຽນ | `domain_registered` |
| ຕໍ່ອາຍຸໂດເມນ | `domain_renewed` |
| ການຕໍ່ອາຍຸບໍ່ສຳເລັດ | `domain_renewal_failed` |
| ໂດເມນກຳລັງໝົດອາຍຸ | `domain_expiring_soon` |

## ແດມິນ: ການລົງທະບຽນໂດເມນດ້ວຍຕົນເອງ (Manual domain registration)

ເພື່ອລົງທະບຽນໂດເມນໃນນາມຂອງລູກຄ້າໂດຍບໍ່ໃຫ້ເຂົາເຈົ້າຜ່ານຂັ້ນຕອນການຊື້ເຄື່ອງ (checkout):

1. ໄປທີ່ **Network Admin › Ultimate Multisite › Register Domain**
2. ເລືອກລູກຄ້າ ແລະ ໃສ່ຊື່ໂດເມນ
3. ຕື່ມຂໍ້ມູນຕິດຕໍ່ຂອງຜູ້ລົງທະບຽນ (ຊື່, ທີ່ຢູ່, ໂທລະສັບ)
4. ກົດ **Register**

ຫຼັງຈາກນີ້, ບັນທຶກໂດເມນຈະຖືກສ້າງຂຶ້ນ ແລະ ເຊື່ອມຕໍ່ກັບບັນຊີຂອງລູກຄ້າແລ້ວ.
