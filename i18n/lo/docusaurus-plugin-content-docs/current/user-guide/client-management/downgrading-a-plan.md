# การลดระดับแผน (v2)

_**ข้อควรระวัง: บทความนี้อ้างถึง Ultimate Multisite เวอร์ชัน 2.x เท่านั้น**_

การลดระดับแผนหรือการสมัครสมาชิกเป็นสิ่งที่ลูกค้าของคุณอาจทำได้ หากพวกเขามีงบประมาณจำกัด หรือตัดสินใจว่าพวกเขาไม่จำเป็นต้องใช้ทรัพยากรมากในการดูแลเว็บไซต์ย่อย (subsite) ของตนเอง

## วิธีการลดระดับแผน {#how-to-downgrade-a-plan}

ลูกค้าสามารถลดระดับแผนได้ทุกเมื่อ โดยการเข้าสู่ระบบแอดมินของ subsite และคลิกที่ **Change** (เปลี่ยน) ภายใต้หน้าบัญชีของพวกเขา

![Customer account page with Your Membership card and Change button](/img/account-page/membership-change-button.png)

เมื่อคลิกปุ่ม **Change** ผู้ใช้/ลูกค้าจะถูกนำไปยังหน้าชำระเงิน (checkout page) ซึ่งสามารถเลือกแผนที่ต้องการเปลี่ยนการสมัครสมาชิกได้

![Downgrade plan options page on the customer side](/img/account-page/downgrade-picker.png)

ในตัวอย่างนี้ เรากำลังลดระดับแผนจาก **Premium** เป็น **Free**

เพื่อให้ดำเนินการต่อ ผู้ใช้เพียงแค่ต้องคลิกปุ่ม **Complete Checkout** (ดำเนินการชำระเงินให้เสร็จสิ้น) จากนั้นระบบจะนำกลับไปยังหน้าบัญชี ซึ่งจะแสดงข้อความเกี่ยวกับการเปลี่ยนแปลงที่รอดำเนินการสำหรับสมาชิก การเปลี่ยนแปลงเหล่านี้จะมีผลใน **รอบบิลถัดไป** ของลูกค้า

![Account page showing pending membership change banner](/img/account-page/pending-change.png)

### สิ่งที่จะเกิดขึ้นเมื่อผู้ใช้ลดระดับแผน {#what-happens-when-a-user-downgrades-their-plan}

สิ่งสำคัญที่ต้องจำไว้คือ การลดระดับแผนจะไม่เปลี่ยนแปลงการตั้งค่าที่มีอยู่เดิมใน subsite ของผู้ใช้

มันจะไม่เปลี่ยนเทมเพลตของเว็บไซต์โดยอัตโนมัติ เนื่องจากหากมีการเปลี่ยนเทมเพลตเว็บไซต์ จะเป็นการลบและรีเซ็ต subsite ทั้งหมด ซึ่งเราทำเพื่อหลีกเลี่ยงการสูญเสียข้อมูลที่ไม่จำเป็น ดังนั้น พื้นที่ดิสก์ ธีม ปลั๊กอิน ฯลฯ จะยังคงอยู่ครบถ้วน ยกเว้นโพสต์ (posts)

ພວກເຮົາເຂົ້າໃຈວ່າຄວາມกังวลหลักຂອງທ່ານແມ່ນข้อจำกัดและโควต้าที่คุณตั้งไว้ภายใต้แต่ละแพลน แต่เราต้องคำนึงถึงผลกระทบที่จะเกิดขึ้นกับเว็บไซต์ย่อยของผู้ใช้งาน หากเราลบหรือเปลี่ยนแปลงการตั้งค่าใดๆ ของมัน

สำหรับโพสต์ที่เกินขีดจำกัดที่กำหนดในแพลน คุณมี 3 ทางเลือก: **เก็บโพสต์ไว้ตามเดิม** *,* **ย้ายโพสต์ไปถังขยะ** *,* หรือ **ย้ายโพสต์ไปเป็นฉบับร่าง** *. คุณสามารถตั้งค่าสิ่งนี้ได้ภายในการตั้งค่าของ Ultimate Multisite

![Network Admin Settings Sites page showing post-limit-exceeded behaviour options](/img/account-page/settings-sites-post-limit.png)

### เกิดอะไรขึ้นกับการชำระเงิน {#what-happens-to-the-payment}

ในเวอร์ชัน 2.0 ไม่จำเป็นต้องมีการปรับเปลี่ยนใดๆ เกี่ยวกับการคำนวณส่วนที่เหลือ (proration) อีกต่อไป

นี่เป็นเพราะระบบจะรอให้สมาชิกที่มีอยู่ **ดำเนินการรอบบิลให้เสร็จสิ้นก่อน** ที่แพลน/การเป็นสมาชิกใหม่จะมีผล การเรียกเก็บเงินสำหรับสมาชิกใหม่จะถูกนำไปใช้และเรียกเก็บโดยอัตโนมัติในรอบบิลถัดไป
