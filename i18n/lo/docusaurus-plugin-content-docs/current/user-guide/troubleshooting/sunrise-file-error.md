# កំហុសក្នុងការដំឡើង Sunrise File {#error-installing-the-sunrise-file}

ឯកសារ sunrise.php គឺជាឯកសារពិសេសមួយដែល WordPress ស្វែងរកពេលវាចាប់ផ្តើមដំណើរការ។ ដើម្បីឱ្យ WordPress អាចរកឃើញឯកសារ sunrise.php បាន វាត្រូវតែស្ថិតនៅក្នុងថត **wp-content**។

នៅពេលអ្នកបើក Ultimate Multisite ហើយធ្វើតាមដំណើរការដំឡើងដូចក្នុងរូបភាពដែលអ្នកបានឃើញ Ultimate Multisite ព្យាយាមចម្លងឯកសារ sunrise.php របស់យើងទៅក្នុងថត wp-content។

<!-- រូបភាពមិនអាចបង្ហាញបានទេ៖ ទំព័រ wizard ដែលបង្ហាញពីជំហានដំឡើង sunrise.php -->

ភាគច្រើន យើងអាចចម្លងឯកសារនោះដោយជោគជ័យ ហើយអ្វីៗដំណើរការធម្មតា។ ប៉ុន្តែ បើមានអ្វីមួយមិនត្រូវបានកំណត់ត្រឹមត្រូវ (ដូចជាការអនុញ្ញាតថត folder ជាដើម) អ្នកប្រហែលជាជួបស្ថានភាពដែល Ultimate Multisite មិនអាចចម្លងឯកសារនោះបានទេ។

ប្រសិនបើអ្នកអានសារកំហុស Ultimo ផ្តល់ឱ្យអ្នក អ្នកនឹងឃើញថាវាកើតឡើងពិតប្រាកដដូចនេះ៖ **Sunrise copy failed** (ការចម្លង Sunrise របរាជ័យ)។

<!-- រូបភាពមិនអាចបង្ហាញបានទេ៖ សារកំហុសដែលបង្ហាញពី Sunrise copy failed -->

ដើម្បីដោះស្រាយបញ្ហានេះ អ្នកគ្រាន់តែចម្លងឯកសារ sunrise.php ចូលទៅក្នុងថត plugin wp-ultimo ហើយបិទភ្ជាប់វាទៅក្នុងថត wp-content របស់អ្នក។ បន្ទាប់ពីធ្វើរួច សូមផ្ទុកទំព័រ wizard ឡើងវិញ ហើយការត្រួតពិនិត្យគួរតែឆ្លងកាត់ដោយជោគជ័យ។

<!-- រូបភាពមិនអាចបង្ហាញបានទេ៖ file manager ដែលបង្ហាញ sunrise.php នៅខាងក្នុងថត plugin wp-ultimo --> ទោះជាយ៉ាងណាក៏ដោយ ការធ្វើបែបនេះអាចទាមទារឱ្យអ្នកពិនិត្យមើលការអនុញ្ញាតថត folder របស់អ្នកឱ្យបានទូទៅ ដើម្បីជៀសវាងបញ្ហាអនាគត (មិនត្រឹមតែជាមួយ Ultimate Multisite ប៉ុណ្ណោះទេ ប៉ុន្តែថែមទាំងជាមួយ plugin និង theme ផ្សេងទៀតផងដែរ)។

**Health Check tool** ដែលជាផ្នែកមួយនៃ WordPress (អ្នកអាចចូលប្រើវាបានតាមរយៈ **admin panel > Tools > Health Check** នៅលើ site chínhរបស់អ្នក) មានសមត្ថភាពក្នុងការប្រាប់អ្នកថា តើការអនុញ្ញាតថត folder របស់អ្នកត្រូវបានកំណត់ត្រឹមតម្លៃដែលអាចបង្កបញ្ហាដល់ WordPress ឬអត់។
