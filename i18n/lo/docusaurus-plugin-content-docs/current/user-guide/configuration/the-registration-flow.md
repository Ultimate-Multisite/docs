# กระบวนการลงทะเบียน (v2) {#the-registration-flow-v2}

_**ข้อควรระวัง: บทความนี้อ้างถึง Ultimate Multisite เวอร์ชัน 2.x เท่านั้น**_

ผู้ใช้งานสามารถลงทะเบียนเข้าสู่เครือข่ายของคุณได้หลายวิธี พวกเขาสามารถใช้แบบฟอร์มลงทะเบียน หรือลิงก์ที่แชร์ไปยังแผนที่เลือกไว้ล่วงหน้าได้ ในที่นี้ เราจะแสดงให้ดูว่าลูกค้าของคุณสามารถลงทะเบียนบนเครือข่ายของคุณได้อย่างไร โดยใช้เส้นทางที่มีอยู่ และเกิดอะไรขึ้นหลังจากที่พวกเขาลงทะเบียนบนเครือข่ายของคุณ

## การใช้แบบฟอร์มลงทะเบียน: {#using-the-registration-form}

นี่คือกระบวนการลงทะเบียนมาตรฐาน คุณจะต้องสร้างหน้าลงทะเบียนพร้อมกับ **checkout form** (แบบฟอร์มชำระเงิน) ซึ่งเป็นที่ที่ลูกค้าของคุณจะไปเพื่อลงทะเบียนบนเครือข่ายของคุณและสมัครแผนต่างๆ คุณสามารถมีหลายหน้าสำหรับลงทะเบียนได้ โดยแต่ละหน้าจะมีแบบฟอร์มลงทะเบียนที่แตกต่างกันหากคุณต้องการ

หน้าเริ่มต้นสำหรับการลงทะเบียนคือ [_**yourdomain.com/register**_](http://yourdomain.com/register) แต่คุณสามารถเปลี่ยนหน้านี้ได้ทุกเมื่อใน **Ultimate Multisite > Settings > Login & Registration > Default Registration Page**.

หลังจากผู้ใช้มาถึงหน้าลงทะเบียนของคุณ (โดยปกติคือการคลิกปุ่ม **Sign in** หรือ **Buy now**) พวกเขาจะเห็นแบบฟอร์มลงทะเบียนของคุณที่นั่น

![Registration form displayed on the registration page](/img/frontend/registration-form.png)

นี่คือตัวอย่างของ checkout form ในหน้า frontend:

![Frontend checkout registration form](/img/config/checkout-frontend-registration.png)

สิ่งที่พวกเขาต้องทำก็คือ กรอกข้อมูลในช่องที่จำเป็นทั้งหมด เช่น อีเมล ชื่อผู้ใช้ รหัสผ่าน ฯลฯ และชำระเงินสำหรับแผนนั้น หรือยืนยันที่อยู่อีเมลของตนหากกำลังลงทะเบียนสำหรับแผนฟรี หรือแผนแบบเสียเงินที่มีช่วงทดลองใช้โดยไม่มีข้อมูลการชำระเงิน

ในหน้า "ขอบคุณ" (Thank you page) พวกเขาจะเห็นข้อความแจ้งว่าพวกเขาจำเป็นต้องยืนยันที่อยู่อีเมลหรือไม่ หรือเว็บไซต์ของพวกเขาก็เปิดใช้งานแล้วและสามารถเริ่มใช้งานได้เลย

![Thank You page after registration](/img/frontend/registration-thank-you.png)

ຖ້າຕ້ອງການຢືນຢັນທີ່ຢູ່ອີເມວ, ພວກເຂົາจะต้องเข้าไปในกล่องจดหมายอีเมลของตัวเองแล้วคลิกที่ลิงก์ยืนยันครับ เว็บไซต์ของพวกเขาจะไม่เปิดใช้งานถ้าที่อยู่อีเมลไม่ได้รับการยืนยัน

ถ้าพวกเขาลงทะเบียนด้วยแผนแบบชำระเงิน หรือการยืนยันอีเมลไม่ใช่ข้อบังคับสำหรับเครือข่ายของคุณ พวกเขาจะสามารถเปิดใช้งานเว็บไซต์ได้ทันทีหลังจากชำระเงิน และจะเห็นลิงก์เพื่อเข้าสู่ระบบแดชบอร์ดของตนเองครับ

![Site activated with link to sign in to dashboard](/img/frontend/site-activated.png)

## การใช้ลิงก์ที่แชร์ได้ (Shareable Link): {#using-a-shareable-link}

ขั้นตอนการลงทะเบียนโดยใช้ลิงก์ที่แชร์ได้นั้น หลักๆ แล้วเหมือนกับการกรอกฟอร์มลงทะเบียนเลยครับ ความแตกต่างเดียวคือ เมื่อใช้ลิงก์ที่แชร์ได้ ลูกค้าของคุณสามารถเลือกสินค้าหรือเทมเพลตเว็บไซต์ไว้ล่วงหน้าในแบบฟอร์มชำระเงิน (ดูส่วนการเลือกสินค้าและเทมเพลตล่วงหน้าผ่าน URL parameters) หรืออาจจะมีการเพิ่มโค้ดคูปองเข้าไปด้วย (ดูส่วนการใช้ URL Parameters)

กระบวนการลงทะเบียนจะเหมือนเดิมครับ พวกเขาจะต้องกรอกชื่อ, ชื่อผู้ใช้, ที่อยู่อีเมล, ชื่อเว็บไซต์ และชื่อเรื่อง ฯลฯ... แต่แผนหรือเทมเพลตเว็บไซต์จะถูกเลือกไว้ให้พวกเขาแล้ว

### การลงทะเบียนโดยใช้การชำระเงินแบบแมนวล (Manual Payments): {#registering-using-manual-payments}

ถ้าคุณไม่ต้องการใช้ PayPal, Stripe หรือเกตเวย์การชำระเงินอื่น ๆ ที่ Ultimate Multisite หรือส่วนเสริมของมันมีให้ คุณสามารถใช้การชำระเงินแบบแมนวลสำหรับลูกค้าของคุณได้ วิธีนี้จะทำให้คุณสามารถสร้างใบแจ้งหนี้เพื่อเรียกเก็บเงินจากพวกเขาผ่านโปรเซสเซอร์การชำระเงินที่คุณเลือก หลังจากที่พวกเขาลงทะเบียนในเครือข่ายของคุณแล้ว

กระบวนการลงทะเบียนจะเหมือนกับข้างบนทุกประการ แต่ในหน้าลงทะเบียน ลูกค้าของคุณจะเห็นข้อความแจ้งว่าพวกเขาจะได้รับอีเมลพร้อมคำแนะนำเพิ่มเติมเพื่อดำเนินการชำระเงินให้เสร็จสมบูรณ์ครับ

![ຂໍ້ຄວາມການຈ່າຍເງິນດ້ວຍມືໃນລະຫວ່າງການລົງທະບຽນ](/img/frontend/registration-manual-notice.png)

ແລະ ຫຼັງຈາກທີ່ການລົງທະບຽນສຳເລັດແລ້ວ, ພວກເຂົາຈະເຫັນຄຳແນະນຳການຈ່າຍເງິນທີ່ທ່ານໄດ້ຕັ້ງໄວ້ (ແລະ ຍັງໄດ້ຮັບມັນໃນອີເມວຂອງພວກເຂົາອີກດ້ວຍ).

![ຄຳແນະນຳການຈ່າຍເງິນສະແດງຫຼັງຈາກລົງທະບຽນ](/img/frontend/registration-payment-instructions.png)

ທ່ານສາມາດປ່ຽນຄຳແນະນຳການຈ່າຍເງິນໄດ້ໃນ **Ultimate Multisite > Settings > Payments** ຫຼັງຈາກທີ່ທ່ານເປີດໃຊ້ຕົວເລືອກການຈ່າຍເງິນແບບ **Manual**:

![ປິດ/ເປີດຕົວເລືອກການຈ່າຍເງິນດ້ວຍມືພ້ອມຊ່ອງໃສ່ຄຳແນະນຳການຈ່າຍເງິນ](/img/config/manual-gateway-settings.png)

ຫຼັງຈາກທີ່ລູກຄ້າຂອງທ່ານເຮັດການຈ່າຍເງິນແບບ Manual ແລະ ສົ່ງການຢືນຢັນໃຫ້ທ່ານແລ້ວ, ທ່ານຕ້ອງ **ຢືນຢັນການຈ່າຍເງິນດ້ວຍຕົນເອງ** ເພື່ອເປີດໃຊ້ສະມາຊິກຂອງລູກຄ້າ ແລະ ເວັບໄຊທ໌.

ເພື່ອເຮັດສິ່ງນີ້, ໄປທີ່ **Ultimate Multisite > Payments** ແລະ ຊອກຫາການຈ່າຍເງິນຂອງລູກຄ້າ. ມັນຄວນຈະສະແດງສະຖານະເປັນ **Pending** (ລໍຖ້າ).

![ລາຍການການຈ່າຍເງິນທີ່ມີການຈ່າຍເງິນແບບ Manual ລໍຖ້າ](/img/admin/payments-list.png)

ໃຫ້ຄລິກໃສ່ເລກການຈ່າຍເງິນ ແລະ ທ່ານຈະສາມາດປ່ຽນສະຖານະຂອງມັນເປັນ **Completed** (ສຳເລັດ).

![ໜ້າລາຍລະອຽດການຈ່າຍເງິນ](/img/admin/payment-edit.png)

![ການປ່ຽນສະຖານະການຈ່າຍເງິນເປັນ Completed](/img/admin/payment-status-completed.png)

ຫຼັງຈາກທີ່ທ່ານປ່ຽນສະຖານະເປັນ **Completed** ແລ້ວ, ທ່ານຄວນຈະເຫັນຂໍ້ຄວາມ **Activate membership** (ເປີດໃຊ້ສະມາຊິກ). ໃຫ້ປິດຕົວເລືອກນີ້ເປັນ **on** ເພື່ອເປີດໃຊ້ສະມາຊິກ ແລະ ເວັບໄຊທ໌ທີ່ກ່ຽວຂ້ອງກັບລູກຄ້ານີ້. ຈາກນັ້ນ, ຄລິກເພື່ອ **Save Payment** (ບັນທຶກການຈ່າຍເງິນ).

![ປິດ/ເປີດຕົວເລືອກ Activate membership ແລະ ແປ້ນສະໝັກ Save Payment](/img/admin/payment-activate-membership.png)

ລູກຄ້າຂອງທ່ານຄວນຈະສາມາດເຂົ້າເຖິງ dashboard ແລະ ຄຸນສົມບັດທັງໝົດທີ່ພວກເຂົາໄດ້ສະໝັກໄວ້ແລ້ວ.
