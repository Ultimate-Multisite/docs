# การรายงานข้อเสนอแนะและปัญหาจากลูกค้า

Gratis AI Agent v1.5.0 มีระบบการแจ้งข้อเสนอแนะและรายงานปัญหาในตัว ที่ช่วยให้ผู้ใช้ปลายทางสามารถทำเครื่องหมายคำตอบที่ไม่เป็นประโยชน์หรือรายงานปัญหาได้โดยตรงจากหน้าต่างแชท ระบบนี้รวมถึงการจัดการความยินยอม (consent management), คำสั่งรายงานอัตโนมัติ, และการคัดแยกปัญหาโดย AI ในส่วนหลังบ้าน

## ปุ่มกดไม่เห็นด้วย (Thumbs-Down Button) {#thumbs-down-button}

ทุกข้อความที่ส่งโดยผู้ช่วย AI จะมีปุ่ม **thumbs-down** (👎) แสดงอยู่ เมื่อผู้ใช้คลิกปุ่มนี้ พวกเขาสามารถทำเครื่องหมายคำตอบนั้นว่าไม่เป็นประโยชน์หรือไม่ถูกต้องได้

- ปุ่มจะปรากฏขึ้นเมื่อนำเมาส์ไปวางข้างข้อความของผู้ช่วยแต่ละข้อความ
- การคลิกปุ่มจะเปิด **Feedback Consent Modal** (หน้าต่างขอความยินยอมเพื่อแสดงความคิดเห็น)
- ข้อเสนอแนะนี้จะถูกผูกไว้กับสายการสนทนาและข้อความนั้นๆ โดยเฉพาะ

## Feedback Consent Modal {#feedback-consent-modal}

เมื่อผู้ใช้คลิกปุ่ม thumbs-down จะมีหน้าต่างขอความยินยอมปรากฏขึ้นก่อนที่จะมีการส่งข้อมูลใดๆ หน้าต่างนี้:

- อธิบายว่าจะมีข้อมูลอะไรบ้างที่ถูกเก็บรวบรวม (ส่วนหนึ่งของการสนทนา, บริบทของเบราว์เซอร์)
- ขอให้ผู้ใช้ยืนยันว่าตกลงที่จะแบ่งปันข้อมูลนี้หรือไม่
- มีช่องสำหรับพิมพ์ข้อความอิสระเพื่อให้ผู้ใช้สามารถอธิบายสิ่งที่ผิดพลาดได้
- มีตัวเลือก **Cancel** เพื่อปิดโดยไม่ต้องส่งข้อมูล

จะไม่มีการบันทึกข้อเสนอแนะใดๆ จนกว่าผู้ใช้จะยืนยันอย่างชัดเจน

## แบนเนอร์แจ้งข้อเสนอแนะอัตโนมัติ (Auto-Prompt Feedback Banner) {#auto-prompt-feedback-banner}

เมื่อการสนทนาสิ้นสุดลง ผู้ช่วยอาจแสดง **auto-prompt feedback banner** ซึ่งเป็นข้อความที่ไม่รบกวนซึ่งถามว่าเซสชันนั้นมีประโยชน์หรือไม่

แบนเนอร์นี้จะปรากฏขึ้นโดยอัตโนมัติตามความยาวและการประเมินผลของการสนทนา มันเชื่อมโยงไปยังขั้นตอนการให้ข้อเสนอแนะแบบเดียวกับปุ่ม thumbs-down ผู้ใช้สามารถปิดแบนเนอร์ได้โดยไม่ต้องให้ข้อเสนอแนะ

## คำสั่ง /report-issue Slash Command {#report-issue-slash-command}

ผู้ใช้สามารถเริ่มกระบวนการให้ข้อเสนอแนะได้โดยตรงโดยการพิมพ์ `/report-issue` ในช่องป้อนข้อมูลแชท คำสั่งนี้:

- ເປີດ Modal ຂໍຄວາມເຫັນ (Feedback Consent) ທັນທີ.
- ຕື່ມຂໍ້ມູນໃນຊ່ອງ description ລ່ວງໜ້າດ້ວຍບໍລິບົດຂອງການສົນທະນາປັດຈຸບັນ.
- ອະນຸຍາດໃຫ້ຜູ້ໃຊ້ເພີ່ມລາຍລະອຽດເພີ່ມເຕີມກ່ອນທີ່ຈະສົ່ງ.

ຄຳສັ່ງ `/report-issue` ສາມາດໃຊ້ໄດ້ໃນທຸກໂໝດແຊັດ (inline, floating widget, full-screen).

## AI-Assisted Triage (ການຈັດປະເພດໂດຍ AI ຊ່ວຍ) {#ai-assisted-triage}

ຂໍ້ສະເໜີຄວາມຄິດເຫັນທີ່ສົ່ງມາຈະຖືກສົ່ງໄປຫາ layer ຂອງ AI triage ເພື່ອ:

- ແບ່ງປະເພດຂອງລາຍງານ (ຂໍ້ຜິດພາດດ້ານຂໍ້ມູນ, ຄຳຕອບທີ່ບໍ່ເປັນປະໂຫຍດ, ເນື້ອໃນທີ່ບໍ່ເໝາະສົມ, ແລະ ອື່ນໆ).
- ເພີ່ມຂໍ້ມູນເພີ່ມເຕີມທີ່ກ່ຽວຂ້ອງກັບການສົນທະນາ.
- ສົ່ງສະຫຼຸບການຈັດປະເພດໄປຫາ **Feedback Endpoint** ທີ່ໄດ້ຕັ້ງຄ່າໄວ້ (ເບິ່ງ [Settings > Advanced](#settings-advanced)).

ສິ່ງນີ້ຊ່ວຍຫຼຸດເວລາໃນການກວດສອບດ້ວຍມື ເພາະມັນຈະបង្ហាញບັນຫາທີ່ສຳຄັນທີ່ສຸດກ່ອນ.

## Settings > Advanced {#settings-advanced}

ເພື່ອເປີດໃຊ້ backend ຂໍ້ສະເໜີຄວາມຄິດເຫັນ, ກະລຸນາຕັ້ງຄ່າຊ່ອງຕໍ່ໄປນີ້ພາຍໃຕ້ **Gratis AI Agent → Settings → Advanced**:

| Field | Description (ຄຳອະທິບາຍ) |
|---|---|
| **Feedback Endpoint URL** | URL ທີ່ຈະຮັບ POST requests ທີ່ມີ payload ຂໍ້ສະເໜີຄວາມຄິດເຫັນ (JSON). |
| **Feedback API Key** | Bearer token ທີ່ສົ່ງໄປໃນ header `Authorization` ເພື່ອຢັ້ງຢືນການສົ່ງຂໍ້ສະເໜີຄວາມຄິດເຫັນ. |

ປ່ອຍຊ່ອງທັງສອງໄວ້ຫວ່າງເພື່ອປິດການເກັບກຳຂໍ້ສະເໜີຄວາມຄິດເຫັນ. ແຕ່ປຸ່ມ feedback ແຕ່ລະອັນຍັງຈະເຫັນຢູ່ຕໍ່ໜ້າຜູ້ໃຊ້, ແຕ່ການສົ່ງຂໍ້ມູນໄປຫາ endpoint ຈະບໍ່ຖືກໂອນຕໍ່.

:::tip
endpoint ຂໍ້ສະເໜີຄວາມຄິດເຫັນຕ້ອງຮັບ body ເປັນ JSON ທີ່ມີຢ່າງໜ້ອຍສາມຟິລດແມ່ນ `message_id`, `conversation_id`, `feedback_text`, ແລະ `triage_category`. ເບິ່ງ documentation ຂອງ endpoint provider ຂອງທ່ານເພື່ອເບິ່ງ schema ທີ່ຄາດຫວັງ.
:::
