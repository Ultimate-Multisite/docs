# ข้อกำหนดของ Ultimate Multisite {#ultimate-multisite-requirements}

สิ่งแรกที่คุณอาจจะถามตัวเองก่อนซื้อ Ultimate Multisite ก็คือ ต้องมีข้อกำหนดอะไรบ้างในการใช้งานปลั๊กอินของเรา? โชคดีนะ มีข้อกำหนดพื้นฐานไม่กี่อย่างสำหรับการใช้งาน Ultimate Multisite คือ **ชื่อโดเมน** (domain name), **ผู้ให้บริการโฮสติ้ง** (hosting provider) ที่ดี และการติดตั้ง WordPress **Multisite**

แค่นั้นก็เพียงพอแล้ว!

## ชื่อโดเมน (Domain Name) {#domain-name}

มาเริ่มกันที่เรื่องชื่อโดเมนก่อนนะ

เวลาเลือกชื่อโดเมน ให้จำไว้ว่าคุณกำลังทำธุรกิจเฉพาะด้านอะไร และจะสร้างอะไรด้วย Ultimate Multisite: คุณอยากสร้างเอเจนซี่? ขายเว็บไซต์แบบร้านค้าออนไลน์? หรืออาจจะเป็นบริการโฮสติ้ง WordPress? ทุกอย่างเป็นไปได้ด้วย Ultimate Multisite เลย

ปลั๊กอินของเรามีบริการสมัครสมาชิก, การเก็บเงิน, ฟอร์มชำระเงิน, คูปองส่วนลด และการสื่อสารกับลูกค้า งานที่ต้องทำเยอะมากในการติดตั้ง ตั้งค่า และดูแลรักษา WordPress Multisite จะง่ายขึ้นมากด้วย Ultimate Multisite

ไม่ว่าคุณจะเลือกตลาดเฉพาะทางไหน คุณก็สามารถจัดการมันได้โดยใช้ปลั๊กอินของเรา ตราบใดที่คุณขายเว็บไซต์เป็นบริการ! อย่าลืมเลือกชื่อโดเมนที่เข้ากับแผนธุรกิจของคุณนะ

## ผู้ให้บริการโฮสติ้ง (Hosting Provider) {#hosting-provider}

เมื่อคุณมีชื่อโดเมนที่น่าสนใจซึ่งสะท้อนถึงตลาดเฉพาะทางและธุรกิจของคุณแล้ว คุณจะต้องเลือกบริษัทโฮสติ้งเว็บที่ดีเพื่อใช้เก็บเครือข่ายของคุณด้วย Ultimate Multisite มีตัวเลือกดีๆ ในตลาดอยู่หลายเจ้า ผู้ให้บริการโฮสติ้งบางรายทำงานร่วมกับ Ultimate Multisite ได้ดี ซึ่งอาจเป็นปัจจัยชี้ขาด ขึ้นอยู่กับสิ่งที่คุณวางแผนจะสร้างขึ้น

Ultimate Multisite អាចដំណើរការได้กับผู้ให้บริการโฮสติ้งทุกราย ความแตกต่างเพียงอย่างเดียวคือ ผู้ให้บริการโฮสติ้งบางราย เช่น _WPMU DEV Hosting_ , _WP Engine_ , _Closte_ , _Cloudways_ , _Gridpane_ และ _Runcloud_ จะมีการเชื่อมต่อที่แน่นหนากว่าสำหรับการแมปโดเมน สำหรับผู้ให้บริการโฮสติ้งรายอื่น คุณอาจต้องทำการแมปโดเมนแบบกำหนดเองด้วยตนเอง นอกจากนี้ การเลือกผู้ให้บริการโฮสติ้งที่รองรับ AutoSSL ก็เป็นความคิดที่ดีเช่นกัน _Closte_ , _Runcloud_ และ _cPanel_ รองรับฟีเจอร์นี้

สำหรับข้อกำหนดของเซิร์ฟเวอร์ ไม่มีสูตรตายตัว เพราะมันจะขึ้นอยู่กับฟีเจอร์และฟังก์ชันที่คุณจะนำเสนอ รวมถึงขนาดเครือข่ายของคุณ การใช้ shared hosting อาจเป็นจุดเริ่มต้นที่ดี แต่เมื่อเครือข่ายของคุณเริ่มเติบโต คุณน่าจะต้องเปลี่ยนไปใช้ dedicated VPS

## WordPress Multisite {#wordpress-multisite}

ทีนี้ เรามาพูดถึงการติดตั้ง WordPress Multisite กันสักหน่อยนะ

WordPress Multisite ช่วยให้คุณสามารถมีเว็บไซต์หลายๆ เว็บไซต์อยู่ในการติดตั้งเดียวได้ ฟีเจอร์นี้มีมาให้ในตัว แต่โดยปกติแล้วมันจะยังไม่ได้เปิดใช้งาน คุณต้องเปิดใช้งานฟีเจอร์นี้ทั้งในส่วนของ WordPress และใน control panel ของ web hosting ด้วย

การมี WordPress ที่เปิดใช้งาน Multisite **คือพื้นฐานสำหรับ Ultimate Multisite** เนื่องจากวัตถุประสงค์หลักของ Ultimate Multisite คือการช่วยให้ผู้ใช้จัดการและขายเว็บไซต์เป็นบริการ (as a service) ดังนั้นสิ่งนี้จึงต้องทำบน WordPress ที่เปิดใช้งาน Multisite แล้ว

เรามี **บทความเชิงลึก** ที่สอนทีละขั้นตอนว่าต้องทำอะไรบ้างเพื่อเปิดใช้งานฟีเจอร์ Multisite บนการติดตั้ง WordPress ของคุณ

หลังจากที่คุณเปิดใช้งาน Multisite บน WordPress ก็ถึงเวลาที่จะติดตั้ง Ultimate Multisite เพื่อให้ติดตั้ง Ultimate Multisite ได้อย่างถูกต้อง คุณต้องมีสิ่งเหล่านี้:

  * WordPress: v5.3+ (แนะนำ: เวอร์ชันที่เสถียรล่าสุด)

  * PHP: 7.4.x (จะมีการรองรับเวอร์ชัน > 8.0 เร็วๆ นี้)

  * MySQL: v5+ (แนะนำ: 5.6 ในกรณีที่คุณไม่มี 8.0 ให้ใช้บนผู้ให้บริการโฮสติ้งของคุณ)

ນີ້ແມ່ນຂໍ້ກຳນົດດ້ານຊອບແວຣ໌ທີ່ຈຳເປັນສຳລັບ Ultimate Multisite ເທົ່ານັ້ນ.

ຄວນຈື່ໄວ້ວ່າ Ultimate Multisite ອາດຈະເຮັດວຽກກັບ PHP 8.0 ແຕ່ພວກເຮົາແນະນຳໃຫ້ໃຊ້ກັບ PHP 7.4.x.

ຍັງແນະນຳໃຫ້ຕິດຕັ້ງມັນຢູ່ເທິງ **main domains** (ໂດເມນຫຼັກ), ບໍ່ແມ່ນ subdomain ຫຼື subdirectory. ມັນອາດຈະເຮັດວຽກກັບ subdomain/subdirectory, ແຕ່ມັນອາດຈະເກີດຂໍ້ຜິດພາດບາງຢ່າງໄດ້.

ສຳລັບຄູ່ມືລາຍລະອຽດກ່ຽວກັບວິທີຕິດຕັ້ງ Ultimate Multisite, **ກະລຸນາคลิกທີ່ນີ້**.
