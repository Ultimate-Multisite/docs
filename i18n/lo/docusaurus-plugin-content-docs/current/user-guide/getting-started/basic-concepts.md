# แนวคิดพื้นฐาน

สำหรับผู้ใช้ WordPress Multisite มือใหม่ และคนที่เพิ่งเริ่มใช้ Ultimate Multisite อาจมีคำศัพท์และวลีใหม่ๆ ให้เรียนรู้เยอะในช่วงแรก การเรียนรู้สิ่งเหล่านี้เป็นเรื่องสำคัญมาก เพราะคุณจะต้องเข้าใจแพลตฟอร์มและวิธีการทำงานโดยรวมของมัน

ในบทความนี้ เราจะพยายามอธิบายและให้ความหมายของแนวคิดหลักๆ ใน WordPress บางส่วน ซึ่งบางอย่างอาจเกี่ยวข้องกับผู้ใช้ บางอย่างสำหรับนักพัฒนา และบางอย่างสำหรับทั้งสองกลุ่ม

## WordPress Multisite {#wordpress-multisite}

**Multisite** ของ WordPress คือรูปแบบการติดตั้ง WordPress ที่ช่วยให้คุณสามารถสร้างและจัดการเครือข่ายเว็บไซต์หลายแห่งได้จากแดชบอร์ด WordPress เพียงที่เดียว คุณสามารถจัดการทุกอย่าง รวมถึงจำนวนไซต์ ฟีเจอร์ ธีม และบทบาทของผู้ใช้ด้วย เป็นไปได้ที่จะจัดการเว็บไซต์หลายร้อยและหลายพันเว็บไซต์

## เครือข่าย (Network) {#network}

ในแง่ของ WordPress เครือข่าย Multisite คือส่วนที่คุณสามารถจัดการ **subsites** หลายๆ แห่งได้จากแดชบอร์ดเดียว แม้ว่าวิธีการสร้างเครือข่าย Multisite จะแตกต่างกันไปตามผู้ให้บริการโฮสติ้ง แต่ผลลัพธ์สุดท้ายมักจะเป็นการเพิ่มคำสั่งเล็กน้อยในไฟล์ **wp-config.php** เพื่อบอกให้ WordPress รู้ว่ากำลังทำงานในโหมดนี้

มีความแตกต่างที่ชัดเจนหลายอย่างระหว่างเครือข่าย Multisite กับการติดตั้ง WordPress แบบเดี่ยว ซึ่งเราจะมาพูดถึงโดยย่อกัน

## ฐานข้อมูล (Database) {#database}

ฐานข้อมูลคือชุดข้อมูลที่มีโครงสร้างและมีการจัดระเบียบ ในศัพท์คอมพิวเตอร์ ฐานข้อมูลหมายถึงซอฟต์แวร์ที่ใช้ในการจัดเก็บและจัดการข้อมูล ลองนึกภาพว่ามันเป็นตู้เก็บเอกสารที่คุณเก็บข้อมูลไว้ในส่วนต่างๆ ที่เรียกว่าตาราง (tables)

WordPress Multisite ใช้ฐานข้อมูลเดียว และแต่ละ subsites จะได้ตารางของตัวเองโดยมี blog id เป็น prefix ดังนั้นเมื่อคุณติดตั้งการติดตั้งเครือข่ายแล้วและสร้าง subsites คุณควรจะมีตารางเหล่านี้:

_wp_1_options_ - ຕາຕະລາງການຕັ້ງຄ່າ (options table) ສຳລັບ subsites ແຫ່ງທຳອິດ

_wp_2_options_ - ຕາຕະລາງການຕັ້ງຄ່າ (options table) ສຳລັບ subsites ແຫ່ງທີສອງ

##ຜູ້ໃຫ້ບໍລິການໂຮງHosting

ຜູ້ໃຫ້ບໍລິການໂຮງHosting ແມ່ນບໍລິສັດທີ່ຊ່ວຍໃຫ້ທຸລະກິດ ແລະ ບຸກຄົນສາມາດເປີດເວັບໄຊຂອງຕົນໃຫ້ເຂົ້າເຖິງໄດ້ຜ່ານ World Wide Web. ການບໍລິການທີ່ຜູ້ໃຫ້ບໍລິການໂຮງHosting ສະເໜີຈະແຕກຕ່າງກັນໄປ ແຕ່ໂດຍປົກກະຕິແລ້ວມັນຈະປະກອບມີການອອກແບບເວັບໄຊ, ພື້ນທີ່ເກັບຂໍ້ມູນໃນ Host, ແລະ ການເຊື່ອມຕໍ່ກັບອິນເຕີເນັດ.

##Domain (ໂດເມນ)

Domain name ແມ່ນທີ່ຢູ່ທີ່ຄົນໃຊ້ເພື່ອເຂົ້າເຖິງເວັບໄຊຂອງເຈົ້າ. ມັນບອກໃຫ້ web browser ຮູ້ວ່າຄວນໄປຫາເວັບໄຊຂອງເຈົ້າຢູ່ໃສ. ຄືກັບທີ່ຢູ່ສະຖານທີ່, domain ແມ່ນວິທີທີ່ຄົນເຂົ້າເຖິງເວັບໄຊຂອງເຈົ້າທາງອອນລາຍ. ແລະ ຄືກັບການມີປ້າຍສະຕໍຢູ່ໜ້າຮ້ານຂອງເຈົ້າ. ຖ້າເຈົ້າຕ້ອງການເຂົ້າເຖິງເວັບໄຊຂອງພວກເຮົາ, ເຈົ້າຈະຕ້ອງພິມທີ່ຢູ່ web ຂອງພວກເຮົາໃນ address bar ຂອງ browser ເຊິ່ງແມ່ນ [_www.ultimatemultisite.com_](http://www.ultimatemultisite.com) _,_ ໂດຍ[_ultimatemultisite.com_](http://ultimatemultisite.com) ແມ່ນ domain name.

##Subdomain (ຊັບໂດເມນ)

Subdomain ແມ່ນປະເພດຂອງໂຄງສ້າງເວັບໄຊທີ່ຢູ່ພາຍໃຕ້ domain ຫຼັກ, ແຕ່ແທນທີ່ຈະໃຊ້ folder ເພື່ອຈັດລະບຽບເນື້ອໃນໃນເວັບໄຊ, ມັນກໍຄືການມີເວັບໄຊຂອງຕົນເອງ. ມັນຈະສະແດງອອກເປັນ [**https://site1.domain.com/**](https://site1.domain.com/) ໂດຍ _site1_ ແມ່ນຊື່ subdomain ແລະ [_domain.com_](http://domain.com) ແມ່ນ domain ຫຼັກ.

##Subdirectory (ຊັບໂຟເल्ເດຣີ)

Subdirectory ແມ່ນປະເພດຂອງໂຄງສ້າງເວັບໄຊທີ່ຢູ່ພາຍໃຕ້ root domain ທີ່ໃຊ້ folder ເພື່ອຈັດລະບຽບເນື້ອໃນໃນເວັບໄຊ. Subdirectory ແມ່ນຄືກັບ subfolder ແລະ ຊື່ສາມາດໃຊ້ແທນກັນໄດ້. ມັນຈະສະແດງອອກເປັນ [**https://domain.com/site1**](https://domain.com/site1) ໂດຍ _site1_ ແມ່ນຊື່ subdirectory ແລະ [_domain.com_](http://domain.com) ແມ່ນ domain ຫຼັກ.

##Subsite

Subsite ແມ່ນ subsite ເຊິ່ງເປັນເວັບໄຊທີ່ທ່ານສ້າງຂຶ້ນໃນເຄືອຂ່າຍ Multisite. ມັນສາມາດເປັນໄດ້ທັງ **subdomain** ຫຼື **subdirectory** ຂຶ້ນຢູ່ກັບວິທີການຕັ້ງຄ່າ WordPress Multisite ຂອງທ່ານ.

## Super Admin {#hosting-provider}

WordPress Super Admin ແມ່ນໜ້າທີ່ຂອງຜູ້ໃຊ້ທີ່ມີສິດເຕັມທີ່ໃນການຈັດການ subsites ທັງໝົດໃນເຄືອຂ່າຍ Multisite. ສຳລັບຜູ້ໃຊ້ Multisite, ນີ້ແມ່ນ **ລະດັບການເຂົ້າເຖິງສູງສຸດ** ທີ່ທ່ານສາມາດໃຫ້ກັບ WordPress installation ຂອງທ່ານໄດ້.

## Plugin {#domain}

ໂດຍທົ່ວໄປແລ້ວ, plugin ແມ່ນຊຸດຂອງ code ທີ່ເພີ່ມຟັງຊັນພິເສດໃສ່ເວັບໄຊ WordPress ຂອງທ່ານ. ສິ່ງນີ້ອາດຈະງ່າຍປານການປ່ຽນໂລໂກ້ເຂົ້າສູ່ລະບົບ ຫຼື ຊັບຊ້ອນປານການເພີ່ມຟັງຊັນອີຄອມເມີຊ (e-commerce). _Woocommerce ແລະ Contact Form_ ແມ່ນຕົວຢ່າງຂອງ plugin.

ໃນ WordPress Multisite, plugin ສາມາດຕິດຕັ້ງໄດ້ຈາກ dashboard ຂອງ network admin ໂດຍ Super Admin ເທົ່ານັ້ນ. Subsite Admins ສາມາດເປີດໃຊ້ (activate) ແລະ ປິດການໃຊ້ງານ (deactivate) plugin ພາຍໃນ subsite ຂອງຕົນເທົ່ານັ້ນ.

## Themes {#subdomain}

WordPress theme ແມ່ນກຸ່ມຂອງໄຟລ໌ (_ຮູບພາບ, style sheets, ແລະ code_) ທີ່ກຳນົດຮູບແບບໂດຍລວມຂອງເວັບໄຊ. ມັນຈະໃຫ້ການຈັດຮູບແບບດ້ານໜ້າທັງໝົດ ເຊັ່ນ: ການຈັດຮູບແບບຕົວອັກສອນ, ແຜນຜັງໜ້າ, ສີຕ່າງໆ ແລະ ອື່ນໆ.

ຄືກັບ plugin, themes ໃນ WordPress Multisite ສາມາດຕິດຕັ້ງໄດ້ໂດຍ Super Admin ເທົ່ານັ້ນ ແລະ ສາມາດເປີດໃຊ້ໃນລະດັບ subsite ໂດຍ subsite admins.

## Site Template {#subdirectory}

**Site Template** ແມ່ນເວັບໄຊແມ່ (boilerplate site) ທີ່ສາມາດນຳໃຊ້ເປັນພື້ນຖານເມື່ອສ້າງ subsites ໃໝ່ໃນເຄືອຂ່າຍຂອງທ່ານໄດ້.

ໝາຍຄວາມວ່າທ່ານສາມາດສ້າງ subsite ແມ່, ເປີດໃຊ້ plugin ແຕກຕ່າງກັນ, ກໍານົດ theme ທີ່ເປີດໃຊ້ ແລະ ປັບແຕ່ງມັນໄດ້ຕາມໃຈມັກ. ແລ້ວເມື່ອລູກຄ້າຂອງທ່ານສ້າງບັນຊີໃໝ່, ແທນທີ່ຈະໄດ້ subsite WordPress ປົກກະຕິທີ່ບໍ່ມີເນື້ອໃນທີ່ມີຄວາມໝາຍພາຍໃນ, ລາວຈະໄດ້ຮັບການຄັດລອກຂອງ subsite ແມ່ຂອງທ່ານທີ່ມີການປັບແຕ່ງ ແລະ ເນື້ອໃນທັງໝົດສຳເລັດຢູ່ແລ້ວ.

## Domain Mapping {#subsite}

การแมปโดเมน (Domain mapping) กับ WordPress คือวิธีที่เราจะเปลี่ยนเส้นทางผู้ใช้งานไปยังโฮสต์ที่ถูกต้อง ผ่านที่อยู่ของเว็บไซต์ ในระบบ WordPress Multisite นั้น เราสามารถสร้าง subsites ได้โดยใช้ทั้งโฟลเดอร์ย่อย (subdirectory) หรือ subdomain การทำ domain mapping ช่วยให้ผู้ใช้ใน subsites สามารถใช้โดเมนระดับบน เช่น [**joesbikeshop.com**](http://joesbikeshop.com) เพื่อทำให้ที่อยู่เว็บไซต์ดูเป็นมืออาชีพมากขึ้น

## SSL {#super-admin}

SSL ย่อมาจาก **Secure Sockets Layer** มันคือใบรับรองดิจิทัลที่ใช้ยืนยันตัวตนของเว็บไซต์และเปิดใช้งานการเชื่อมต่อที่เข้ารหัส ในปัจจุบัน มันถูกใช้เป็นเทคโนโลยีมาตรฐานสำหรับการรักษาความปลอดภัยของการเชื่อมต่ออินเทอร์เน็ต และปกป้องข้อมูลที่ละเอียดอ่อนใดๆ ที่ถูกส่งระหว่างสองระบบ ป้องกันไม่ให้ผู้ไม่หวังดีอ่านหรือแก้ไขข้อมูลที่ถูกถ่ายโอน รวมถึงรายละเอียดส่วนตัวที่เป็นไปได้ เว็บเบราว์เซอร์สมัยใหม่ต้องการ SSL ซึ่งทำให้มันสำคัญมากเมื่อสร้างและรันเว็บไซต์

## Media (สื่อ) {#plugin}

Media คือรูปภาพ เสียง วิดีโอ และไฟล์อื่นๆ ที่ทำให้เว็บไซต์มีหน้าตาขึ้นมา

เว็บไซต์แบบเครือข่าย (Network sites) จะแชร์ฐานข้อมูลชุดเดียวใน WordPress Multisite แต่จะเก็บไฟล์สื่อไว้ในที่อยู่ (path) แยกกันบนระบบไฟล์ ตำแหน่งมาตรฐานของ WordPress (wp-content/uploads) ยังคงเหมือนเดิม เพียงแต่เส้นทางของมันจะถูกเปลี่ยนให้สะท้อนถึง ID เฉพาะของเว็บไซต์เครือข่ายนั้นๆ ดังนั้น ไฟล์สื่อสำหรับเว็บไซต์เครือข่ายจะปรากฏเป็น wp-contents/uploads/site/[id]

## Permalinks (ลิงก์ถาวร) {#themes}

Permalinks คือ URL ถาวรของแต่ละโพสต์หรือหน้าบล็อกของคุณภายในเว็บไซต์ของคุณ Permalinks ยังถูกเรียกว่า **pretty links** โดยค่าเริ่มต้น URL ของ WordPress จะใช้รูปแบบ query string ซึ่งมีลักษณะคล้ายกับสิ่งนี้:

## Ultimate Multisite {#site-template}

Ultimate Multisite ແມ່ນ plugin ຂອງ WordPress, ເຊິ່ງສ້າງຂຶ້ນມາເພື່ອເຮັດໃຫ້ການຕິດຕັ້ງ WordPress ຂອງທ່ານກາຍເປັນເຄືອຂ່າຍເວັບໄຊທີ່ພຣີມຽມ – ຄືກັບ [WordPress.com](https://WordPress.com) – ເຮັດໃຫ້ລູກຄ້າສາມາດສ້າງເວັບໄຊໄດ້ຜ່ານຄ່າທຳນຽບປະຈຳເດືອນ, ປະຈຳໄຕມາດ ຫຼື ປະຈຳປີ (ທ່ານຍັງສາມາດສ້າງແຜນຟຣີໄດ້ອີກດ້ວຍ).

## Checkout Form {#domain-mapping}

Checkout Form ແມ່ນແບບຟອມສັ່ງຊື້ທີ່ເປັນແບບຄັ້ງດຽວ ຫຼື ແບບຫຼາຍຂັ້ນ, ເຊິ່ງກ່ຽວຂ້ອງກັບການສ້າງ subsites (ເວັບໄຊຍ່ອຍ), ການເປັນສະມາຊິກ ແລະ ບັນຊີຜູ້ໃຊ້ຜ່ານການລົງທະບຽນ Ultimate Multisite. ມັນປະກອບດ້ວຍແບບຟອມຕ່າງໆ ແລະ ແບບຟອມຈ່າຍເງິນທີ່ຜູ້ໃຊ້ຕ້ອງສົ່ງໃນລະຫວ່າງຂັ້ນຕອນການລົງທະບຽນ.

## Webhook {#ssl}

Webhook (ຫຼືເອີ້ນວ່າ web callback ຫຼື HTTP push API) ແມ່ນວິທີທີ່ແອັບພລິເຄຊັນໜຶ່ງສາມາດສະໜອງຂໍ້ມູນແບບ Real-time ໃຫ້ກັບແອັບພລິເຄຊັນອື່ນໆໄດ້. Webhook ສົ່ງຂໍ້ມູນໄປຫາແອັບພລິເຄຊັນອື່ນໃນທັນທີ, ຊຶ່ງໝາຍຄວາມວ່າທ່ານຈະໄດ້ຮັບຂໍ້ມູນທັນທີ.

**Ultimate Multisite webhooks** ເປີດໂອກາດທີ່ບໍ່ຈຳກັດ, ໂດຍເຮັດໃຫ້ຜູ້ບໍລິຫານເຄືອຂ່າຍສາມາດເຮັດການເຊື່ອມຕໍ່ທີ່ແປກໆ ແຕ່ມີປະໂຫຍດໄດ້ທຸກຢ່າງ, ໂດຍສະເພາະເມື່ອໃຊ້ຮ່ວມກັບບໍລິການຕ່າງໆ ເຊັ່ນ _Zapier ແລະ IFTTT_.

## Events {#media}

Event ແມ່ນການກະທຳທີ່ເກີດຂຶ້ນເປັນຜົນມາຈາກການກະທຳຂອງຜູ້ໃຊ້ ຫຼື ແຫຼ່ງຂໍ້ມູນອື່ນ, ເຊັ່ນ: ການຄລິກເມົ້າ. Ultimate Multisite ຈະເກັບບັນທຶກທຸກ Event ແລະ Log ທີ່เกิดขึ้นພາຍໃນເຄືອຂ່າຍທັງໝົດຂອງທ່ານ. ມັນຕິດຕາມກິດຈະກຳຕ່າງໆທີ່ເກີດຂຶ້ນໃນ multisite ຂອງທ່ານ, ເຊັ່ນ: ການປ່ຽນແປງແຜນການ.
