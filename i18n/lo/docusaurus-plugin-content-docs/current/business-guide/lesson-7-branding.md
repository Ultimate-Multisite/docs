# ບົດຮຽນ 7: ເຮັດໃຫ້ມັນເປັນຂອງທ່ານ

ລູກຄ້າຂອງທ່ານບໍ່ຄວນຮູ້ສຶກເລີຍວ່າພວກເຂົາກຳລັງໃຊ້ "ປລັກອິນ WordPress ບາງຢ່າງ." ພວກເຂົາຄວນຮູ້ສຶກວ່າກຳລັງໃຊ້ FitSite -- ແພລດຟອມທີ່ສ້າງຂຶ້ນສຳລັບອຸດສາຫະກຳຂອງພວກເຂົາ. ບົດຮຽນນີ້ຄອບຄຸມການສ້າງແບຣນ, ການເຮັດ white-label, ແລະການເຮັດໃຫ້ແພລດຟອມຮູ້ສຶກເປັນຜະລິດຕະພັນ.

## ຈຸດທີ່ພວກເຮົາຢຸດໄວ້ {#where-we-left-off}

FitSite ມີຂະບວນການ checkout ທີ່ໃຊ້ງານໄດ້ ເຊິ່ງພາເຈົ້າຂອງສະຕູດິໂອຟິດເນສຈາກການເລືອກແຜນໄປສູ່ເວັບໄຊທີ່ເປີດໃຊ້ງານຈິງ. ຕອນນີ້ພວກເຮົາຈະເຮັດໃຫ້ປະສົບການທັງໝົດຮູ້ສຶກກົມກຽວ ແລະເປັນຜະລິດຕະພັນທີ່ມີແບຣນຊັດເຈນ.

## ໂດເມນແພລດຟອມຂອງທ່ານ {#your-platform-domain}

ຮາກຖານຂອງແບຣນຂອງທ່ານແມ່ນໂດເມນຂອງທ່ານ. ສຳລັບ FitSite:

- **ໂດເມນຫຼັກ**: `fitsite.com` (ເວັບໄຊການຕະຫຼາດຂອງທ່ານ ແລະຮາກເຄືອຂ່າຍ)
- **ເວັບໄຊລູກຄ້າ**: `studioname.fitsite.com` (subdomains)
- **ໂດເມນກຳນົດເອງ**: ລູກຄ້າໃນແຜນ Growth ແລະ Pro ສາມາດແມັບໂດເມນຂອງຕົນເອງໄດ້

### ການຕັ້ງຄ່າໂດເມນຂອງທ່ານ {#setting-up-your-domain}

1. ຈົດທະບຽນໂດເມນແພລດຟອມຂອງທ່ານ
2. ຊີ້ໂດເມນນັ້ນໄປຫາຜູ້ໃຫ້ບໍລິການ hosting ຂອງທ່ານ
3. ກຳນົດຄ່າ wildcard DNS (`*.fitsite.com`) ສຳລັບ subdomains ຂອງລູກຄ້າ
4. ໃຫ້ແນ່ໃຈວ່າ wildcard SSL ເປີດໃຊ້ງານຢູ່

ເບິ່ງ [ວິທີກຳນົດຄ່າ Domain Mapping](/user-guide/domain-mapping/how-to-configure-domain-mapping) ສຳລັບຄຳແນະນຳລະອຽດ.

## ການເຮັດ White-Label ໃຫ້ປະສົບການ Admin {#white-labeling-the-admin-experience}

ເມື່ອເຈົ້າຂອງສະຕູດິໂອຟິດເນສເຂົ້າສູ່ລະບົບໃນ Dashboard ຂອງເວັບໄຊພວກເຂົາ, ພວກເຂົາຄວນເຫັນແບຣນຂອງທ່ານ, ບໍ່ແມ່ນ WordPress ຫຼືແບຣນ Ultimate Multisite.

### ໜ້າເຂົ້າສູ່ລະບົບກຳນົດເອງ {#custom-login-page}

ປັບແຕ່ງໜ້າເຂົ້າສູ່ລະບົບ WordPress ເພື່ອສະແດງ:

- ໂລໂກ້ FitSite ຂອງທ່ານ
- ຮູບພື້ນຫຼັງທີ່ເໝາະກັບຟິດເນສ
- ສີແບຣນຂອງທ່ານ

### ການສ້າງແບຣນໃຫ້ Dashboard {#dashboard-branding}

ໃຊ້ addon [ຕົວສ້າງໜ້າ Admin](/addons/admin-page-creator) ຫຼື CSS ກຳນົດເອງເພື່ອ:

- ແທນທີ່ໂລໂກ້ WordPress ດ້ວຍໂລໂກ້ FitSite ຂອງທ່ານ
- ປັບແຕ່ງໂທນສີ admin ໃຫ້ກົງກັບແບຣນຂອງທ່ານ
- ເພີ່ມ widget Dashboard ກຳນົດເອງທີ່ມີລິ້ງດ່ວນ ແລະຊັບພະຍາກອນຊ່ວຍເຫຼືອສະເພາະດ້ານຟິດເນສ

### ໜ້າ Admin ກຳນົດເອງ {#custom-admin-pages}

ພິຈາລະນາການສ້າງໜ້າ admin ກຳນົດເອງທີ່ສະແດງການດຳເນີນການທີ່ກ່ຽວຂ້ອງທີ່ສຸດສຳລັບເຈົ້າຂອງສະຕູດິໂອຟິດເນສ:

- "ແກ້ໄຂຕາຕະລາງຄລາສຂອງທ່ານ"
- "ອັບເດດໂປຣໄຟລ໌ຄູຝຶກ"
- "ເບິ່ງເວັບໄຊຂອງທ່ານ"

ສິ່ງນີ້ຫຼຸດຄວາມຊັນຂອງການຮຽນຮູ້ໂດຍການວາງການດຳເນີນການທີ່ກ່ຽວກັບນິດຊ໌ໄວ້ດ້ານໜ້າແລະກາງ, ແທນທີ່ຈະຝັງໄວ້ໃນເມນູ WordPress ມາດຕະຖານ.

## ການສ້າງແບຣນໃຫ້ການສື່ສານຂອງທ່ານ {#branding-your-communications}

ທຸກອີເມວ, ໃບແຈ້ງໜີ້, ແລະການແຈ້ງເຕືອນຄວນຕອກຢ້ຳແບຣນຂອງທ່ານ.

### ອີເມວລະບົບ {#system-emails}

ໄປທີ່ **Ultimate Multisite > Settings > Emails** ແລະປັບແຕ່ງອີເມວລະບົບທັງໝົດ:

- **ຊື່ຜູ້ສົ່ງ**: FitSite
- **ອີເມວຜູ້ສົ່ງ**: hello@fitsite.com
- **ແມ່ແບບອີເມວ**: ໃຊ້ສີແບຣນ ແລະໂລໂກ້ຂອງທ່ານ
- **ພາສາ**: ສະເພາະດ້ານຟິດເນສຕະຫຼອດທັງໝົດ

ອີເມວສຳຄັນທີ່ຄວນປັບແຕ່ງ:

| ອີເມວ | ເວີຊັນທົ່ວໄປ | ເວີຊັນ FitSite |
|-------|----------------|-----------------|
| ຕ້ອນຮັບ | "ເວັບໄຊໃໝ່ຂອງທ່ານພ້ອມແລ້ວ" | "ເວັບໄຊສະຕູດິໂອຟິດເນສຂອງທ່ານເປີດໃຊ້ງານແລ້ວ" |
| ໃບຮັບເງິນການຊຳລະ | "ໄດ້ຮັບການຊຳລະແລ້ວ" | "ຢືນຢັນການຊຳລະ subscription FitSite ແລ້ວ" |
| ການທົດລອງໃຊ້ໃກ້ສິ້ນສຸດ | "ການທົດລອງໃຊ້ຂອງທ່ານຈະສິ້ນສຸດໄວໆນີ້" | "ການທົດລອງໃຊ້ FitSite ຂອງທ່ານຈະສິ້ນສຸດໃນ 3 ມື້ -- ຮັກສາເວັບໄຊສະຕູດິໂອຂອງທ່ານໃຫ້ເປີດໃຊ້ງານຕໍ່ໄປ" |

### ໃບແຈ້ງໜີ້ {#invoices}

ປັບແຕ່ງແມ່ແບບໃບແຈ້ງໜີ້ດ້ວຍ:

- ໂລໂກ້ FitSite ແລະສີແບຣນຂອງທ່ານ
- ລາຍລະອຽດທຸລະກິດຂອງທ່ານ
- ຊື່ຜະລິດຕະພັນສະເພາະດ້ານຟິດເນສ (ບໍ່ແມ່ນ ID ແຜນທົ່ວໄປ)

## ເວັບໄຊຝັ່ງລູກຄ້າ {#the-customer-facing-site}

ໂດເມນຫຼັກຂອງທ່ານ (`fitsite.com`) ຕ້ອງມີເວັບໄຊການຕະຫຼາດທີ່ຂາຍແພລດຟອມ. ນີ້ແຍກອອກຈາກ network admin ຂອງ Ultimate Multisite -- ມັນແມ່ນໜ້າສາທາລະນະຂອງທຸລະກິດຂອງທ່ານ.

ໜ້າສຳຄັນ:

- **ໜ້າຫຼັກ**: ຂໍ້ສະເໜີຄຸນຄ່າທີ່ຊັດເຈນສຳລັບທຸລະກິດຟິດເນສ
- **ຄຸນສົມບັດ**: FitSite ເຮັດຫຍັງໄດ້, ໃນຄຳສັບດ້ານຟິດເນສ
- **ລາຄາ**: ແຜນສາມລະດັບຂອງທ່ານພ້ອມການປຽບທຽບຄຸນສົມບັດສະເພາະນິດຊ໌
- **ຕົວຢ່າງ**: ສະແດງເວັບໄຊທີ່ສ້າງເທິງແພລດຟອມ
- **ລົງທະບຽນ**: ລິ້ງໄປຫາແບບຟອມ checkout ຂອງທ່ານ

:::tip ເວັບໄຊການຕະຫຼາດຂອງທ່ານສາມາດເປັນເວັບໄຊເຄືອຂ່າຍໄດ້
ສ້າງເວັບໄຊການຕະຫຼາດຂອງທ່ານເປັນເວັບໄຊໜຶ່ງພາຍໃນເຄືອຂ່າຍຂອງທ່ານເອງ. ສິ່ງນີ້ໃຫ້ທ່ານຈັດການມັນໄດ້ຈາກ Dashboard ດຽວກັນ ແລະສະແດງຄວາມສາມາດຂອງແພລດຟອມຂອງທ່ານເອງ.
:::

## ໂດເມນກຳນົດເອງສຳລັບລູກຄ້າ {#custom-domain-for-customers}

ສຳລັບລູກຄ້າໃນແຜນທີ່ລວມໂດເມນກຳນົດເອງ, ຈົ່ງບັນທຶກຂະບວນການໃຫ້ຊັດເຈນ:

1. ລູກຄ້າຈົດທະບຽນ ຫຼືໂອນໂດເມນຂອງຕົນໄປຫາຜູ້ຮັບຈົດທະບຽນ
2. ລູກຄ້າອັບເດດ DNS ໃຫ້ຊີ້ໄປຫາແພລດຟອມຂອງທ່ານ (ລະບຸ records ທີ່ແນ່ນອນ)
3. Ultimate Multisite ຈັດການ domain mapping ແລະ SSL

ສ້າງບົດຄວາມຊ່ວຍເຫຼືອ ຫຼືລາຍການຖານຄວາມຮູ້ໂດຍສະເພາະສຳລັບຂະບວນການນີ້, ຂຽນສຳລັບເຈົ້າຂອງສະຕູດິໂອຟິດເນສທີ່ບໍ່ແມ່ນສາຍເຕັກນິກ.

## ເຄືອຂ່າຍ FitSite ຈົນເຖິງຕອນນີ້ {#the-fitsite-network-so-far}

```
ເຄືອຂ່າຍ FitSite
├── WordPress Multisite (ໂໝດ subdomain)
├── Ultimate Multisite (ກຳນົດຄ່າແລ້ວ + ມີແບຣນແລ້ວ)
├── ໂດເມນແພລດຟອມ (fitsite.com + wildcard SSL)
├── ແມ່ແບບເວັບໄຊ (Studio Essential, Gym Pro, Fitness Chain)

├── ຂະບວນການ Checkout (ສະເພາະນິດຊ໌, ທົດສອບແລ້ວ)
├── ການສ້າງແບຣນ
│   ├── ໜ້າເຂົ້າສູ່ລະບົບກຳນົດເອງ
│   ├── Dashboard admin ທີ່ມີແບຣນ
│   ├── ອີເມວລະບົບສະເພາະນິດຊ໌
│   ├── ໃບແຈ້ງໜີ້ທີ່ມີແບຣນ
│   └── ເວັບໄຊການຕະຫຼາດເທິງ fitsite.com
└── ພ້ອມສຳລັບຂະບວນການ onboarding (ບົດຮຽນຕໍ່ໄປ)
```

- **ໂດເມນແພລດຟອມ ແລະ DNS** ຖືກກຳນົດຄ່າສຳລັບປະສົບການທີ່ມີແບຣນ
- **Admin ແບບ white-label** ພ້ອມແບຣນ FitSite ຕະຫຼອດທັງໝົດ
- **ການສື່ສານທີ່ປັບແຕ່ງແລ້ວ** -- ອີເມວ, ໃບແຈ້ງໜີ້, ແລະການແຈ້ງເຕືອນທັງໝົດຢູ່ໃນແບຣນ
- **ເວັບໄຊການຕະຫຼາດ** ທີ່ຂາຍ FitSite ໃຫ້ເຈົ້າຂອງສະຕູດິໂອຟິດເນສ
- **ເອກະສານໂດເມນກຳນົດເອງ** ສຳລັບລູກຄ້າທີ່ຕ້ອງການໂດເມນຂອງຕົນເອງ
