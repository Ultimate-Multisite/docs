# ບົດຮຽນທີ 3: ການຕັ້ງຄ່າເຄືອຂ່າຍຂອງທ່ານ

ຮອດເວລາສ້າງແລ້ວ. ໃນບົດຮຽນນີ້ ທ່ານຈະຕິດຕັ້ງ Ultimate Multisite ແລະກຳນົດຄ່າພື້ນຖານຂອງເຄືອຂ່າຍ FitSite. ທຸກການຕັດສິນໃຈຢູ່ນີ້ຖືກເຮັດໂດຍຄຳນຶງເຖິງຕະຫຼາດສະເພາະດ້ານຟິດເນສ.

## ຈຸດທີ່ພວກເຮົາຄ້າງໄວ້ {#where-we-left-off}

ພວກເຮົາໄດ້ເລືອກສະຕູດິໂອຟິດເນສເປັນຕະຫຼາດສະເພາະຂອງເຮົາ ແລະກວດສອບໂອກາດແລ້ວ. ຕອນນີ້ພວກເຮົາຈະປ່ຽນແນວຄິດນັ້ນໃຫ້ເປັນແພລດຟອມທີ່ໃຊ້ງານໄດ້ຈິງ.

## ການເລືອກໂຮສຕິ້ງຂອງທ່ານ {#choosing-your-hosting}

ການເລືອກໂຮສຕິ້ງຂອງທ່ານສຳຄັນກັບແພລດຟອມຕະຫຼາດສະເພາະຫຼາຍກວ່າກັບເວັບໄຊດຽວ. ທ່ານບໍ່ໄດ້ໂຮສຕ໌ພຽງແຕ່ໜຶ່ງເວັບໄຊ -- ທ່ານກຳລັງໂຮສຕ໌ເຄືອຂ່າຍທີ່ຈະເຕີບໂຕໄປເຖິງຫຼາຍສິບ ຫຼືຫຼາຍຮ້ອຍເວັບໄຊ.

### ສິ່ງທີ່ຄວນມອງຫາ {#what-to-look-for}

- **ການຮອງຮັບ WordPress Multisite**: ບໍ່ແມ່ນທຸກໂຮສຕ໌ຈະຈັດການ multisite ໄດ້ດີ
- **Wildcard SSL**: ຈຳເປັນສຳລັບເຄືອຂ່າຍທີ່ໃຊ້ subdomain
- **ຊັບພະຍາກອນທີ່ຂະຫຍາຍໄດ້**: ທ່ານຕ້ອງມີພື້ນທີ່ໃຫ້ເຕີບໂຕໂດຍບໍ່ຕ້ອງຍ້າຍ
- **ການຜະສານກັບ Ultimate Multisite**: ການຈັບຄູ່ໂດເມນແລະ SSL ແບບອັດຕະໂນມັດຊ່ວຍປະຫຍັດແຮງງານດ້ານປະຕິບັດການໄດ້ຫຼາຍ

### ແນວທາງທີ່ແນະນຳ {#recommended-approach}

ເລືອກໂຮສຕ໌ຈາກລາຍຊື່ [ຜູ້ໃຫ້ບໍລິການທີ່ເຂົ້າກັນໄດ້](/user-guide/host-integrations/closte). ຜູ້ໃຫ້ບໍລິການເຫຼົ່ານີ້ໄດ້ຖືກທົດສອບກັບ Ultimate Multisite ແລ້ວ ແລະມີການຜະສານທີ່ທ່ານຕ້ອງການສຳລັບການຈັບຄູ່ໂດເມນແລະການອັດຕະໂນມັດ SSL.

ສຳລັບ FitSite, ພວກເຮົາຈະໃຊ້ການກຳນົດຄ່າແບບ subdomain. ນີ້ໝາຍຄວາມວ່າເວັບໄຊຂອງລູກຄ້າຈະປາກົດໃນຕອນແລກເປັນ `studioname.fitsite.com` ກ່ອນທີ່ພວກເຂົາຈະເລືອກຈັບຄູ່ໂດເມນຂອງຕົນເອງໄດ້.

## ການຕິດຕັ້ງ WordPress Multisite {#installing-wordpress-multisite}

ຖ້າທ່ານຍັງບໍ່ມີການຕິດຕັ້ງ WordPress Multisite:

1. ຕິດຕັ້ງ WordPress ໃນຜູ້ໃຫ້ບໍລິການໂຮສຕິ້ງຂອງທ່ານ
2. ເຮັດຕາມຄູ່ມື [ວິທີຕິດຕັ້ງ WordPress Multisite](/user-guide/getting-started/how-to-install-wordpress-multisite)
3. ເລືອກການກຳນົດຄ່າແບບ **subdomain** ເມື່ອຖືກຖາມ

:::tip ເປັນຫຍັງຈຶ່ງໃຊ້ Subdomains?
Subdomains ເຮັດໃຫ້ເວັບໄຊຂອງລູກຄ້າແຕ່ລະຄົນມີທີ່ຢູ່ທີ່ແຍກຊັດເຈນຂອງຕົນເອງ (`studio.fitsite.com`) ແທນທີ່ຈະເປັນພາດ (`fitsite.com/studio`). ວິທີນີ້ດູເປັນມືອາຊີບຫຼາຍກວ່າສຳລັບລູກຄ້າຂອງທ່ານ ແລະຫຼີກລ້ຽງຄວາມຂັດແຍ່ງຂອງ permalink. ເບິ່ງ [Ultimate Multisite 101](/user-guide/getting-started/ultimate-multisite-101) ສຳລັບການປຽບທຽບຢ່າງລະອຽດ.
:::

## ການຕິດຕັ້ງ Ultimate Multisite {#installing-ultimate-multisite}

ເຮັດຕາມຄູ່ມື [ການຕິດຕັ້ງ Ultimate Multisite](/user-guide/getting-started/installing-ultimate-multisite) ເພື່ອ:

1. ອັບໂຫຼດແລະເປີດໃຊ້ plugin ທົ່ວທັງເຄືອຂ່າຍ
2. ແລ່ນ [ຕົວຊ່ວຍຕັ້ງຄ່າ](/user-guide/getting-started/multisite-setup-wizard)

ໃນລະຫວ່າງຕົວຊ່ວຍຕັ້ງຄ່າ, ໃຫ້ຄຳນຶງເຖິງຕະຫຼາດສະເພາະ FitSite:

- **ສະກຸນເງິນ**: ຕັ້ງເປັນສະກຸນເງິນທີ່ລູກຄ້າສະຕູດິໂອຟິດເນສຂອງທ່ານໃຊ້
- **ຊື່ບໍລິສັດ**: "FitSite" (ຫຼືຊື່ແບຣນທີ່ທ່ານເລືອກ)
- **ໂລໂກ້ບໍລິສັດ**: ອັບໂຫຼດໂລໂກ້ແບຣນຂອງທ່ານ -- ສິ່ງນີ້ຈະປາກົດໃນໃບແຈ້ງໜີ້ແລະອີເມວ

## ການກຳນົດຄ່າສຳລັບຕະຫຼາດສະເພາະດ້ານຟິດເນສ {#configuring-for-the-fitness-niche}

ເມື່ອຕິດຕັ້ງ Ultimate Multisite ແລ້ວ, ໃຫ້ເລືອກການກຳນົດຄ່າສະເພາະຕະຫຼາດນີ້:

### ການຕັ້ງຄ່າທົ່ວໄປ {#general-settings}

ໄປທີ່ **Ultimate Multisite > Settings** ແລະກຳນົດຄ່າ:

- **ຊື່ເວັບໄຊ**: FitSite
- **ບົດບາດເລີ່ມຕົ້ນ**: ຜູ້ບໍລິຫານ -- ເຈົ້າຂອງສະຕູດິໂອຟິດເນສຕ້ອງການການຄວບຄຸມເນື້ອຫາເວັບໄຊຂອງຕົນຢ່າງເຕັມທີ່
- **ການລົງທະບຽນ**: ເປີດໃຊ້ການລົງທະບຽນຜູ້ໃຊ້ ເພື່ອໃຫ້ເຈົ້າຂອງສະຕູດິໂອສາມາດສະໝັກໄດ້ເອງ

### ການກຳນົດຄ່າອີເມວ {#email-configuration}

ອີເມວຂອງລະບົບທ່ານຄວນໃຊ້ພາສາທີ່ເຂົ້າກັບຕະຫຼາດສະເພາະຂອງທ່ານ. ໄປທີ່ **Ultimate Multisite > Settings > Emails** ແລະປັບແຕ່ງ:

- ແທນທີ່ຂໍ້ຄວາມທົ່ວໄປເຊັ່ນ "ເວັບໄຊໃໝ່ຂອງທ່ານ" ດ້ວຍຂໍ້ຄວາມສະເພາະດ້ານຟິດເນສ
- ຕົວຢ່າງຫົວຂໍ້ຕ້ອນຮັບ: "ເວັບໄຊສະຕູດິໂອຟິດເນສຂອງທ່ານພ້ອມແລ້ວ"
- ຕົວຢ່າງເນື້ອຫາຕ້ອນຮັບ: ອ້າງອີງເຖິງສະຕູດິໂອຂອງພວກເຂົາ, ຄລາສຕ່າງໆ, ແລະການເລີ່ມຕົ້ນກັບເວັບໄຊຟິດເນສຂອງພວກເຂົາ

ພວກເຮົາຈະປັບປຸງສິ່ງເຫຼົ່ານີ້ໃຫ້ດີຂຶ້ນອີກໃນບົດຮຽນທີ 8 (ການຕ້ອນຮັບລູກຄ້າ), ແຕ່ການກຳນົດໂທນຕອນນີ້ຈະຮັບປະກັນວ່າແມ່ນແຕ່ການສະໝັກທົດສອບໃນຊ່ວງແລກກໍຍັງຮູ້ສຶກສະເພາະກັບຕະຫຼາດນີ້.

### ການກຳນົດຄ່າໂດເມນ {#domain-configuration}

ຖ້າໃຊ້ຜູ້ໃຫ້ບໍລິການໂຮສຕິ້ງທີ່ເຂົ້າກັນໄດ້, ໃຫ້ກຳນົດຄ່າການຈັບຄູ່ໂດເມນຕອນນີ້:

1. ໄປທີ່ **Ultimate Multisite > Settings > Domain Mapping**
2. ເຮັດຕາມຄູ່ມືການຜະສານສຳລັບໂຮສຕ໌ສະເພາະຂອງທ່ານ
3. ທົດສອບວ່າ subsite ໃໝ່ໄດ້ຮັບ SSL ອັດຕະໂນມັດ

ສິ່ງນີ້ຮັບປະກັນວ່າເມື່ອພວກເຮົາເລີ່ມສ້າງເທມເພລດແລະເວັບໄຊທົດສອບໃນບົດຮຽນຕໍ່ໄປ, ທຸກຢ່າງຈະໃຊ້ງານໄດ້ຕັ້ງແຕ່ຕົ້ນຈົນຈົບ.

## ເຄືອຂ່າຍ FitSite ຈົນເຖິງຕອນນີ້ {#the-fitsite-network-so-far}

ໃນຕອນທ້າຍຂອງບົດຮຽນນີ້, ນີ້ຄືສິ່ງທີ່ທ່ານມີ:

```
ເຄືອຂ່າຍ FitSite
├── WordPress Multisite (ໂໝດ subdomain)
├── Ultimate Multisite (ຕິດຕັ້ງແລະກຳນົດຄ່າແລ້ວ)
├── ໂຮສຕິ້ງພ້ອມ wildcard SSL
├── ກຳນົດຄ່າການຈັບຄູ່ໂດເມນແລ້ວ
├── ເທມເພລດອີເມວສະເພາະຕະຫຼາດ (ເບື້ອງຕົ້ນ)
└── ພ້ອມສຳລັບເທມເພລດເວັບໄຊ (ບົດຮຽນຕໍ່ໄປ)
```

## ສິ່ງທີ່ພວກເຮົາສ້າງໃນບົດຮຽນນີ້ {#what-we-built-this-lesson}

- ການຕິດຕັ້ງ **WordPress Multisite ທີ່ໃຊ້ງານໄດ້** ໃນໂໝດ subdomain
- **ຕິດຕັ້ງ Ultimate Multisite ແລ້ວ** ແລະກຳນົດຄ່າພ້ອມການສ້າງແບຣນ FitSite
- ຕັ້ງຄ່າ **ໂຮສຕິ້ງແລະ SSL** ສຳລັບເຄືອຂ່າຍທີ່ກຳລັງເຕີບໂຕ
- ກຳນົດຄ່າ **ການຈັບຄູ່ໂດເມນ** ສຳລັບຜູ້ໃຫ້ບໍລິການໂຮສຕິ້ງຂອງທ່ານ
- ສ້າງ **ໂທນອີເມວສະເພາະຕະຫຼາດ** ຕັ້ງແຕ່ມື້ທຳອິດ
