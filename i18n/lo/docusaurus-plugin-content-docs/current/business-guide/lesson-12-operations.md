# ບົດຮຽນທີ 12: ການດຳເນີນທຸລະກິດ {#lesson-12-running-the-business}

ແພລດຟອມບໍ່ແມ່ນໂຄງການທີ່ເຈົ້າເຮັດໃຫ້ສຳເລັດ -- ມັນແມ່ນທຸລະກິດທີ່ເຈົ້າດຳເນີນ. ບົດຮຽນນີ້ກວມເອົາການດຳເນີນງານປະຈຳວັນໃນການຈັດການ FitSite: ການຊ່ວຍເຫຼືອ, ການເກັບເງິນ, ການບຳລຸງຮັກສາ, ແລະການຮັກສາໃຫ້ລູກຄ້າພໍໃຈ.

## ຈຸດທີ່ພວກເຮົາຄ້າງໄວ້ {#where-we-left-off}

FitSite ເປີດໃຊ້ງານແລ້ວ ແລະລູກຄ້າກຳລັງລົງທະບຽນ. ຕອນນີ້ເຈົ້າຕ້ອງດຳເນີນການໃຫ້ຍືນຍົງ.

## ການດຳເນີນງານປະຈຳວັນ {#daily-operations}

### ການຕິດຕາມ {#monitoring}

ກວດສອບສິ່ງເຫຼົ່ານີ້ທຸກມື້ (ຫຼືຕັ້ງການແຈ້ງເຕືອນ):

- **ເວລາໃຊ້ງານບໍ່ຂາດຕອນ**: ແພລດຟອມເຂົ້າເຖິງໄດ້ບໍ? ໃຊ້ບໍລິການຕິດຕາມ uptime.
- **ການລົງທະບຽນໃໝ່**: ມື້ນີ້ມີລູກຄ້າໃໝ່ລົງທະບຽນຈັກຄົນ?
- **ການຊຳລະເງິນລົ້ມເຫຼວ**: ມີການຊຳລະເງິນລົ້ມເຫຼວທີ່ຕ້ອງໃສ່ໃຈບໍ?
- **ຄຳຮ້ອງຂໍການຊ່ວຍເຫຼືອ**: ມີຄຳຖາມຈາກລູກຄ້າທີ່ຍັງບໍ່ໄດ້ຕອບບໍ?

### ການຊ່ວຍເຫຼືອລູກຄ້າ {#customer-support}

ການເນັ້ນຕະຫຼາດສະເພາະຂອງເຈົ້າເປັນຂໍ້ໄດ້ປຽບໃນຈຸດນີ້. ເພາະລູກຄ້າທັງໝົດຂອງເຈົ້າແມ່ນສະຕູດິໂອຟິດເນັດ, ເຈົ້າຈະເຫັນຄຳຖາມເດີມໆ ຊ້ຳໆ:

**ຄຳຖາມທົ່ວໄປທີ່ເຈົ້າຈະໄດ້ຮັບ:**

- "ຂ້ອຍຈະອັບເດດຕາຕະລາງຄລາສຂອງຂ້ອຍແນວໃດ?"
- "ຂ້ອຍປ່ຽນສີຂອງເວັບໄຊຂອງຂ້ອຍໄດ້ບໍ?"
- "ຂ້ອຍຈະເພີ່ມຄູຝຶກໃໝ່ໃສ່ເວັບໄຊຂອງຂ້ອຍແນວໃດ?"
- "ໂດເມນຂອງຂ້ອຍໃຊ້ງານບໍ່ໄດ້"
- "ຂ້ອຍຈະຍົກເລີກ/ອັບເກຣດແຜນຂອງຂ້ອຍແນວໃດ?"

ສ້າງຖານຄວາມຮູ້ຂອງເຈົ້າ (ທີ່ເລີ່ມໃນບົດຮຽນທີ 8) ໂດຍອີງຕາມຄຳຖາມທີ່ເກີດຊ້ຳໆເຫຼົ່ານີ້. ທຸກ ticket ການຊ່ວຍເຫຼືອທີ່ສາມາດເຄີຍເປັນບົດຄວາມຖານຄວາມຮູ້ໄດ້ ແມ່ນສັນຍານໃຫ້ຂຽນບົດຄວາມນັ້ນ.

### ລະດັບການຊ່ວຍເຫຼືອ {#support-tiers}

ເມື່ອເຈົ້າເຕີບໃຫຍ່ຂຶ້ນ, ໃຫ້ຈັດໂຄງສ້າງການຊ່ວຍເຫຼືອຕາມແຜນ:

| ແຜນ | ລະດັບການຊ່ວຍເຫຼືອ | ເວລາຕອບກັບ |
|------|--------------|---------------|
| Starter | ຖານຄວາມຮູ້ + ອີເມວ | 48 ຊົ່ວໂມງ |
| Growth | ການຊ່ວຍເຫຼືອທາງອີເມວ | 24 ຊົ່ວໂມງ |
| Pro | ອີເມວລຳດັບສຳຄັນ + ການໂທ onboarding | 4 ຊົ່ວໂມງ |

[ສ່ວນເສີມ Support Tickets](/addons/support-tickets) ສາມາດຊ່ວຍຈັດການຄຳຮ້ອງຂໍການຊ່ວຍເຫຼືອພາຍໃນແພລດຟອມໄດ້.

## ການດຳເນີນງານດ້ານການເກັບເງິນ {#billing-operations}

### ການຊຳລະເງິນແບບປະຈຳ {#recurring-payments}

Ultimate Multisite ຈັດການການເກັບເງິນແບບປະຈຳອັດຕະໂນມັດຜ່ານ payment gateway ຂອງເຈົ້າ. ໜ້າທີ່ຂອງເຈົ້າຄືການຕິດຕາມ:

- **ການຊຳລະເງິນລົ້ມເຫຼວ**: ຕິດຕາມຕໍ່ຢ່າງທັນເວລາ. ຄວາມລົ້ມເຫຼວສ່ວນໃຫຍ່ແມ່ນບັດໝົດອາຍຸ, ບໍ່ແມ່ນການຍົກເລີກໂດຍຕັ້ງໃຈ.
- **Dunning**: ຕັ້ງ logic ລອງຊຳລະໃໝ່ອັດຕະໂນມັດຜ່ານ payment gateway ຂອງເຈົ້າ (Stripe ຈັດການສິ່ງນີ້ໄດ້ດີ)
- **ຄຳຮ້ອງຂໍຍົກເລີກ**: ເຂົ້າໃຈວ່າເປັນຫຍັງລູກຄ້າຈຶ່ງອອກໄປ. ທຸກການຍົກເລີກແມ່ນຄຳຕິຊົມ.

### ການຈັດການສະມາຊິກ {#managing-memberships}

ໄປທີ່ **Ultimate Multisite > Memberships** ເພື່ອ:

- ເບິ່ງ subscription ທີ່ໃຊ້ງານຢູ່ທັງໝົດ
- ຈັດການຄຳຮ້ອງຂໍອັບເກຣດ ແລະດາວນ໌ເກຣດ
- ດຳເນີນການຄືນເງິນເມື່ອຈຳເປັນ
- ຈັດການການໝົດອາຍຸຊ່ວງທົດລອງ

ເບິ່ງ [ການຈັດການສະມາຊິກ](/user-guide/administration/managing-memberships) ສຳລັບເອກະສານອ້າງອີງສົມບູນ.

### ການອອກໃບແຈ້ງໜີ້ {#invoicing}

ຮັບປະກັນວ່າໃບແຈ້ງໜີ້ຖືກສ້າງຢ່າງຖືກຕ້ອງສຳລັບທຸກການຊຳລະເງິນ. ລູກຄ້າອາດຕ້ອງການໃບແຈ້ງໜີ້ເພື່ອລາຍງານຄ່າໃຊ້ຈ່າຍທາງທຸລະກິດ. ເບິ່ງ [ການຈັດການການຊຳລະເງິນ ແລະໃບແຈ້ງໜີ້](/user-guide/administration/managing-payments-and-invoices).

## ການບຳລຸງຮັກສາແພລດຟອມ {#platform-maintenance}

### ການອັບເດດ WordPress ແລະ Plugin {#wordpress-and-plugin-updates}

ໃນຖານະຜູ້ບໍລິຫານເຄືອຂ່າຍ, ເຈົ້າຮັບຜິດຊອບຕໍ່:

- **ການອັບເດດ WordPress core**: ທົດສອບໃນເວັບໄຊ staging ກ່ອນນຳໃຊ້ກັບ production
- **ການອັບເດດ Plugin**: ເຊັ່ນດຽວກັນ -- ທົດສອບກ່ອນ, ແລ້ວຈຶ່ງນຳໃຊ້ທົ່ວເຄືອຂ່າຍ
- **ການອັບເດດ Theme**: ກວດຢືນຢັນວ່າ template ຍັງເບິ່ງຖືກຕ້ອງຫຼັງຈາກອັບເດດ theme
- **ການອັບເດດ Ultimate Multisite**: ຕິດຕາມ changelog ແລະທົດສອບກ່ອນອັບເດດ

:::warning ຢ່າອັບເດດໃນ Production ໂດຍບໍ່ໄດ້ທົດສອບ
ການອັບເດດທີ່ເສຍຈະກະທົບທຸກເວັບໄຊລູກຄ້າໃນເຄືອຂ່າຍຂອງເຈົ້າ. ຈົ່ງທົດສອບການອັບເດດໃນສຳເນົາ staging ຂອງເຄືອຂ່າຍເຈົ້າກ່ອນສະເໝີ.
:::

### ຄວາມປອດໄພ {#security}

- ອັບເດດ software ທັງໝົດໃຫ້ທັນສະໄໝ
- ໃຊ້ລະຫັດຜ່ານທີ່ແຂງແຮງ ແລະ two-factor authentication ສຳລັບບັນຊີ admin
- ຕິດຕາມກິດຈະກຳທີ່ໜ້າສົງໄສ
- ມີແຜນສຳລັບເຫດການດ້ານຄວາມປອດໄພ

### ປະສິດທິພາບ {#performance}

ເມື່ອເຄືອຂ່າຍຂອງເຈົ້າເຕີບໃຫຍ່ຂຶ້ນ, ໃຫ້ຕິດຕາມ:

- **ເວລາໂຫຼດໜ້າ**: ເວັບໄຊລູກຄ້າໄວບໍ?
- **ການໃຊ້ຊັບພະຍາກອນ server**: CPU, memory, disk space
- **ປະສິດທິພາບ database**: ເຄືອຂ່າຍຂະໜາດໃຫຍ່ຕ້ອງການການປັບປຸງ database ຕາມເວລາ

ພິຈາລະນານຳ caching (page cache, object cache) ແລະ CDN ມາໃຊ້ ຖ້າເຈົ້າຍັງບໍ່ໄດ້ເຮັດ. [ການເຊື່ອມຕໍ່ Cloudflare](/user-guide/host-integrations/cloudflare) ຈັດການສ່ວນໃຫຍ່ຂອງສິ່ງນີ້.

## ການຈັດການວົງຈອນຊີວິດລູກຄ້າ {#customer-lifecycle-management}

### ການຫຼຸດ Churn {#reducing-churn}

Churn ແມ່ນເປີເຊັນຂອງລູກຄ້າທີ່ຍົກເລີກໃນແຕ່ລະເດືອນ. ສຳລັບທຸລະກິດ subscription, ການຫຼຸດ churn ມີຄວາມສຳຄັນເທົ່າກັບການໄດ້ລູກຄ້າໃໝ່.

**ເຫດຜົນທົ່ວໄປທີ່ລູກຄ້າສະຕູດິໂອຟິດເນັດຍົກເລີກ:**

- ພວກເຂົາບໍ່ຮູ້ວິທີໃຊ້ແພລດຟອມ → ປັບປຸງ onboarding
- ເວັບໄຊເບິ່ງບໍ່ດີພໍ → ປັບປຸງ template
- ພວກເຂົາບໍ່ເຫັນຄຸນຄ່າ → ປັບປຸງຟີເຈີ ຫຼືການສື່ສານ
- ພວກເຂົາພົບທາງເລືອກທີ່ຖືກກວ່າ → ເນັ້ນຢ້ຳຄຸນຄ່າຕະຫຼາດສະເພາະຂອງເຈົ້າ
- ທຸລະກິດຂອງພວກເຂົາປິດ → ຫຼີກລ້ຽງບໍ່ໄດ້, ແຕ່ຕິດຕາມສິ່ງນີ້ແຍກຕ່າງຫາກ

### ການກະຕຸ້ນໃຫ້ອັບເກຣດ {#encouraging-upgrades}

ລູກຄ້າໃນ Starter ທີ່ກຳລັງປະສົບຜົນສຳເລັດຄວນຖືກກະຕຸ້ນໃຫ້ອັບເກຣດ:

- ເມື່ອພວກເຂົາເຖິງຂີດຈຳກັດຂອງແຜນ (ເວັບໄຊ, storage), ໃຫ້ສະແດງຂໍ້ຄວາມຊວນອັບເກຣດ
- ສົ່ງອີເມວເຈາະຈົງທີ່ເນັ້ນຟີເຈີຂອງແຜນ Growth ທີ່ພວກເຂົາຈະໄດ້ປະໂຫຍດ
- ນຳສະເໜີສິ່ງທີ່ລູກຄ້າ Growth/Pro ໄດ້ສ້າງ

### ແຄມເປນດຶງລູກຄ້າກັບຄືນ {#win-back-campaigns}

ເມື່ອລູກຄ້າຍົກເລີກ:

1. ຖາມວ່າເປັນຫຍັງ (ສຳຫຼວດຕອນອອກ ຫຼືອີເມວ)
2. ແກ້ໄຂຂໍ້ກັງວົນຂອງພວກເຂົາຖ້າເປັນໄປໄດ້
3. ສະເໜີສ່ວນຫຼຸດເພື່ອກັບຄືນມາ (30-60 ມື້ຫຼັງຈາກຍົກເລີກ)

## ກິດຈະວັດປະຈຳອາທິດ ແລະປະຈຳເດືອນ {#weekly-and-monthly-routines}

### ປະຈຳອາທິດ {#weekly}

- ທົບທວນການລົງທະບຽນໃໝ່ ແລະການຍົກເລີກ
- ຕອບ ticket ການຊ່ວຍເຫຼືອທີ່ເປີດຢູ່ທັງໝົດ
- ກວດສອບປະສິດທິພາບແພລດຟອມ ແລະ uptime
- ທົບທວນການຊຳລະເງິນລົ້ມເຫຼວໃດໆ

### ປະຈຳເດືອນ {#monthly}

- ວິເຄາະຕົວຊີ້ວັດສຳຄັນ (MRR, churn, ລູກຄ້າໃໝ່, ການອັບເກຣດ)
- ນຳໃຊ້ການອັບເດດ WordPress ແລະ plugin (ຫຼັງຈາກທົດສອບ staging)
- ທົບທວນ ແລະອັບເດດຖານຄວາມຮູ້ໂດຍອີງຕາມຮູບແບບການຊ່ວຍເຫຼືອ
- ສົ່ງ newsletter ຫຼືອັບເດດໄປຫາລູກຄ້າ (ຟີເຈີໃໝ່, ເຄັດລັບ, ຂ່າວວົງການຟິດເນັດ)

### ປະຈຳໄຕມາດ {#quarterly}

- ທົບທວນລາຄາເມື່ອທຽບກັບຄູ່ແຂ່ງ ແລະຄຳຕິຊົມລູກຄ້າ
- ປະເມີນການອອກແບບ template -- ຈຳເປັນຕ້ອງປັບໃໝ່ບໍ?
- ປະເມີນຄວາມສາມາດຮອງຮັບຂອງ hosting -- ເຈົ້າຕ້ອງຂະຫຍາຍບໍ?
- ທົບທວນ ແລະປັບປຸງ onboarding ໂດຍອີງຕາມຂໍ້ມູນ activation

## ສິ່ງທີ່ພວກເຮົາສ້າງໃນບົດຮຽນນີ້ {#what-we-built-this-lesson}

- **ຂັ້ນຕອນການຕິດຕາມປະຈຳວັນ** ສຳລັບ uptime, ການລົງທະບຽນ, ການຊຳລະເງິນ, ແລະ ການສະໜັບສະໜູນ
- **ໂຄງສ້າງການສະໜັບສະໜູນເປັນຫຼາຍລະດັບ** ທີ່ສອດຄ່ອງກັບລະດັບແຜນ
- **ການດຳເນີນງານດ້ານການເກັບເງິນ** ລວມເຖິງການຕິດຕາມເກັບເງິນຄ້າງຊຳລະ ແລະ ການຈັດການການຍົກເລີກ
- **ຂັ້ນຕອນການບຳລຸງຮັກສາ** ສຳລັບການອັບເດດ, ຄວາມປອດໄພ, ແລະ ປະສິດທິພາບ
- **ກົນຍຸດການຫຼຸດ churn** ທີ່ສະເພາະກັບກຸ່ມ fitness
- **ຂັ້ນຕອນການດຳເນີນງານປະຈຳອາທິດ, ປະຈຳເດືອນ, ແລະ ປະຈຳໄຕມາດ**
