# ບົດຮຽນ 6: ປະສົບການການສະໝັກ

ຂັ້ນຕອນການຊຳລະເງິນແມ່ນຈຸດທີ່ຄວາມສົນໃຈກາຍເປັນລາຍຮັບ. ຂະບວນການສະໝັກທີ່ສັບສົນ ຫຼື ທົ່ວໄປເກີນໄປຈະເຮັດໃຫ້ສູນເສຍລູກຄ້າ. ຂັ້ນຕອນທີ່ເຈາະຈົງຕາມກຸ່ມຕະຫຼາດ, ເວົ້າພາສາຂອງພວກເຂົາ ແລະ ຮູ້ສຶກງ່າຍດາຍ ຈະປ່ຽນພວກເຂົາເປັນລູກຄ້າ.

## ຈຸດທີ່ພວກເຮົາຢຸດໄວ້

FitSite ມີແມ່ແບບ ແລະ ແຜນທີ່ຕັ້ງຄ່າແລ້ວ. ຕອນນີ້ພວກເຮົາຈະສ້າງປະສົບການການລົງທະບຽນ ແລະ ການຊຳລະເງິນ ທີ່ປ່ຽນເຈົ້າຂອງສະຕູດິໂອຟິດເນສໃຫ້ເປັນລູກຄ້າທີ່ຈ່າຍເງິນ.

## ເຂົ້າໃຈຂັ້ນຕອນການລົງທະບຽນ

ແບບຟອມຊຳລະເງິນຂອງ Ultimate Multisite ແມ່ນແບບຟອມຫຼາຍຂັ້ນຕອນທີ່ປັບແຕ່ງໄດ້ຢ່າງເຕັມທີ່. ສຳລັບ FitSite, ພວກເຮົາຕ້ອງການຂັ້ນຕອນທີ່:

1. ຮູ້ສຶກເໝືອນຖືກສ້າງມາສຳລັບທຸລະກິດຟິດເນສ
2. ເກັບກຳເທົ່ານັ້ນສິ່ງທີ່ຈຳເປັນ
3. ພາລູກຄ້າໄປຫາເວັບໄຊທີ່ໃຊ້ງານໄດ້ໄວທີ່ສຸດ

ເບິ່ງ [ຂັ້ນຕອນການລົງທະບຽນ](/user-guide/configuration/the-registration-flow) ສຳລັບເອກະສານອ້າງອີງດ້ານເຕັກນິກສະບັບເຕັມ.

## ອອກແບບການຊຳລະເງິນຂອງ FitSite

ໄປທີ່ **Ultimate Multisite > ແບບຟອມຊຳລະເງິນ** ແລະ ສ້າງແບບຟອມໃໝ່.

### ຂັ້ນຕອນ 1: ການເລືອກແຜນ

ສິ່ງທຳອິດທີ່ເຈົ້າຂອງສະຕູດິໂອຟິດເນສເຫັນຄວນເປັນແຜນຕ່າງໆ, ນຳສະເໜີດ້ວຍຄຳສັບທີ່ພວກເຂົາເຂົ້າໃຈ.

- ເພີ່ມຊ່ອງຂໍ້ມູນ **ຕາຕະລາງລາຄາ**
- ຕັ້ງຄ່າໃຫ້ສະແດງແຜນ FitSite ທັງສາມແຜນ
- ຄຳອະທິບາຍແຜນທີ່ທ່ານຂຽນໃນບົດຮຽນ 5 ຈະປາກົດຢູ່ນີ້ -- ໃຫ້ແນ່ໃຈວ່າມັນເວົ້າເຖິງຄວາມຕ້ອງການຂອງທຸລະກິດຟິດເນສ, ບໍ່ແມ່ນຄຸນສົມບັດທາງເຕັກນິກ

:::tip ພາສາຂອງກຸ່ມຕະຫຼາດສຳຄັນ
"ພື້ນທີ່ເກັບຂໍ້ມູນ 1 GB" ບໍ່ມີຄວາມໝາຍຫຍັງສຳລັບເຈົ້າຂອງຍິມ. "ທຸກສິ່ງທີ່ທ່ານຕ້ອງການສຳລັບເວັບໄຊສະຕູດິໂອມືອາຊີບ" ໝາຍເຖິງທຸກຢ່າງ. ຂຽນຄຳອະທິບາຍແຜນດ້ວຍພາສາຂອງລູກຄ້າຂອງທ່ານ.
:::

### ຂັ້ນຕອນ 2: ການເລືອກແມ່ແບບ

ຫຼັງຈາກເລືອກແຜນແລ້ວ, ລູກຄ້າເລືອກແມ່ແບບເລີ່ມຕົ້ນຂອງພວກເຂົາ.

- ເພີ່ມຊ່ອງຂໍ້ມູນ **ການເລືອກແມ່ແບບ**
- ແມ່ແບບທີ່ມີຢູ່ຈະຖືກກັ່ນຕອງຕາມແຜນທີ່ພວກເຂົາເລືອກ (ຕັ້ງຄ່າໃນບົດຮຽນ 5)
- ແມ່ແບບແຕ່ລະອັນຄວນມີຮູບຕົວຢ່າງທີ່ສະແດງການອອກແບບເຈາະຈົງສຳລັບຟິດເນສ

### ຂັ້ນຕອນ 3: ການສ້າງ Account

ໃຫ້ສັ້ນກະຊັບ. ເກັບກຳເທົ່ານີ້:

- ທີ່ຢູ່ອີເມວ
- ລະຫັດຜ່ານ
- ຊື່ສະຕູດິໂອ/ທຸລະກິດ (ອັນນີ້ຈະກາຍເປັນຊື່ເວັບໄຊຂອງພວກເຂົາ)

ຢ່າຖາມຂໍ້ມູນທີ່ທ່ານບໍ່ຈຳເປັນຕ້ອງໃຊ້ໃນຕອນສະໝັກ. ຊ່ອງຂໍ້ມູນເພີ່ມແຕ່ລະຊ່ອງຈະຫຼຸດການປ່ຽນເປັນລູກຄ້າ.

### ຂັ້ນຕອນ 4: ການຕັ້ງຄ່າເວັບໄຊ

- **ຊື່ເວັບໄຊ**: ຕື່ມລ່ວງໜ້າຈາກຊື່ສະຕູດິໂອທີ່ປ້ອນໃນຂັ້ນຕອນ 3
- **URL ຂອງເວັບໄຊ**: ສ້າງອັດຕະໂນມັດຈາກຊື່ສະຕູດິໂອ (ເຊັ່ນ `ironworks.fitsite.com`)

### ຂັ້ນຕອນ 5: ການຊຳລະເງິນ

- ເພີ່ມຊ່ອງຂໍ້ມູນ **ການຊຳລະເງິນ**
- ຕັ້ງຄ່າປະຕູຊຳລະເງິນຂອງທ່ານ ([Stripe](/user-guide/payment-gateways/stripe) ແມ່ນແນະນຳສຳລັບການຮຽກເກັບເງິນແບບສະໝັກສະມາຊິກ)
- ຖ້າທ່ານສ້າງຂໍ້ສະເໜີເພີ່ມໃນຄຳສັ່ງຊື້ໃນບົດຮຽນ 5, ໃຫ້ເພີ່ມຊ່ອງຂໍ້ມູນ **ຂໍ້ສະເໜີເພີ່ມໃນຄຳສັ່ງຊື້** ກ່ອນຂັ້ນຕອນການຊຳລະເງິນ

### ຂັ້ນຕອນ 6: ການຢືນຢັນ

- ປັບແຕ່ງຂໍ້ຄວາມຢືນຢັນດ້ວຍພາສາເຈາະຈົງສຳລັບຟິດເນສ
- ຕົວຢ່າງ: "ເວັບໄຊສະຕູດິໂອຟິດເນສຂອງທ່ານກຳລັງຖືກສ້າງ. ທ່ານຈະຖືກນຳໄປຫາເວັບໄຊໃໝ່ຂອງທ່ານໃນອີກບໍ່ກີ່ວິນາທີ."

## ການເພີ່ມປຸ່ມສະຫຼັບເລືອກຊ່ວງເວລາ

ຖ້າທ່ານຕັ້ງຄ່າຮູບແບບລາຄາໃນແຜນຂອງທ່ານ (ລາຍເດືອນ ແລະ ລາຍປີ), ໃຫ້ເພີ່ມຊ່ອງຂໍ້ມູນ **ການເລືອກຊ່ວງເວລາ** ໃສ່ແບບຟອມຊຳລະເງິນ ເພື່ອໃຫ້ລູກຄ້າສາມາດສະຫຼັບລະຫວ່າງຊ່ວງເວລາການຮຽກເກັບເງິນໄດ້. ເບິ່ງ [ແບບຟອມຊຳລະເງິນ](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle) ສຳລັບຄຳແນະນຳ.

## ການຕັ້ງຄ່າການຊຳລະເງິນ

ຖ້າທ່ານຍັງບໍ່ໄດ້ຕັ້ງຄ່າປະຕູຊຳລະເງິນ:

1. ໄປທີ່ **Ultimate Multisite > ການຕັ້ງຄ່າ > ປະຕູຊຳລະເງິນ**
2. ເຮັດຕາມ [ຄູ່ມືການຕັ້ງຄ່າ Stripe](/user-guide/payment-gateways/stripe) ຫຼື ປະຕູທີ່ທ່ານມັກ
3. ທົດສອບຂັ້ນຕອນຊຳລະເງິນຄົບຖ້ວນດ້ວຍການຊຳລະເງິນທົດສອບ

ເບິ່ງ [ການຮັບເງິນ](/user-guide/payment-gateways/getting-paid) ສຳລັບລາຍລະອຽດວ່າການຊຳລະເງິນໄຫຼເຂົ້າບັນຊີຂອງທ່ານແນວໃດ.

## ການທົດສອບຂັ້ນຕອນ

ກ່ອນຈະໄປຕໍ່, ໃຫ້ເຮັດການສະໝັກທົດສອບຄົບຖ້ວນ:

1. ເປີດແບບຟອມຊຳລະເງິນໃນໜ້າຕ່າງບຣາວເຊີແບບບໍ່ລະບຸຕົວຕົນ/ສ່ວນຕົວ
2. ເລືອກແຜນ
3. ເລືອກແມ່ແບບ
4. ສ້າງ Account
5. ຊຳລະເງິນໃຫ້ສຳເລັດ (ໃຊ້ໂໝດທົດສອບ)
6. ກວດສອບວ່າເວັບໄຊຖືກສ້າງດ້ວຍແມ່ແບບທີ່ຖືກຕ້ອງ

ກວດສອບວ່າ:

- [ ] ຄຳອະທິບາຍແຜນຊັດເຈນ ແລະ ເຈາະຈົງຕາມກຸ່ມຕະຫຼາດ
- [ ] ຕົວຢ່າງແມ່ແບບສະແດງການອອກແບບທີ່ເໝາະກັບຟິດເນສ
- [ ] URL ຂອງເວັບໄຊສ້າງໄດ້ຖືກຕ້ອງຈາກຊື່ສະຕູດິໂອ
- [ ] ຂະບວນການຊຳລະເງິນສຳເລັດ
- [ ] ລູກຄ້າໄປຮອດເວັບໄຊທີ່ໃຊ້ງານໄດ້ພ້ອມແມ່ແບບທີ່ເລືອກ
- [ ] ອີເມວຢືນຢັນໃຊ້ພາສາເຈາະຈົງສຳລັບຟິດເນສ

## ສິ່ງທີ່ພວກເຮົາສ້າງໃນບົດຮຽນນີ້

- **ແບບຟອມຊຳລະເງິນຫຼາຍຂັ້ນຕອນ** ທີ່ປັບໃຫ້ເໝາະກັບເຈົ້າຂອງສະຕູດິໂອຟິດເນສ
- **ພາສາເຈາະຈົງຕາມກຸ່ມຕະຫຼາດ** ຕະຫຼອດຂັ້ນຕອນການສະໝັກ
- **ຄວາມຕິດຂັດຕ່ຳສຸດ** -- ມີແຕ່ຊ່ອງຂໍ້ມູນທີ່ຈຳເປັນ, ເສັ້ນທາງໄວໄປຫາເວັບໄຊທີ່ໃຊ້ງານໄດ້
- **ການເຊື່ອມຕໍ່ການຊຳລະເງິນ** ພ້ອມການຢືນຢັນແບບທົດສອບ
- **ຂັ້ນຕອນຕັ້ງແຕ່ຕົ້ນຈົນຈົບທີ່ທົດສອບແລ້ວ** ຈາກການເລືອກແຜນໄປຫາເວັບໄຊທີ່ໃຊ້ງານໄດ້
