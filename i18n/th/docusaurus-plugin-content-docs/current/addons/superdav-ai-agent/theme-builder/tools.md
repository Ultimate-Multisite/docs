# เครื่องมือและการตั้งค่า

Theme Builder ใช้ชุดเครื่องมือในการวิเคราะห์ ออกแบบ และสร้างเว็บไซต์ WordPress ของคุณ ในเวอร์ชัน 1.16.0, **sd-ai-agent/site-scrape ได้กลายเป็นเครื่องมือ Tier 1** ทำให้สามารถใช้งานได้โดยค่าเริ่มต้นใน Theme Builder

## เครื่องมือที่มีให้ใช้

### เครื่องมือ Tier 1 (ใช้งานได้เสมอ)

เครื่องมือ Tier 1 สามารถใช้งานได้โดยค่าเริ่มต้นใน Theme Builder โดยไม่จำเป็นต้องตั้งค่าเพิ่มเติม

#### sd-ai-agent/site-scrape

**วัตถุประสงค์:** ใช้ในการวิเคราะห์เว็บไซต์ที่มีอยู่ เพื่อดึงแรงบันดาลใจด้านการออกแบบ โครงสร้างเนื้อหา และฟังก์ชันการทำงาน

**ความสามารถ:**

- **การวิเคราะห์เว็บไซต์** — ดึงข้อมูลและวิเคราะห์เว็บไซต์คู่แข่งหรือเว็บไซต์ที่ต้องการแรงบันดาลใจ
- **การดึงองค์ประกอบการออกแบบ** — ระบุสี ฟอนต์ และรูปแบบการจัดวาง
- **โครงสร้างเนื้อหา** — ทำความเข้าใจการจัดระเบียบและลำดับชั้นของหน้าเพจ
- **การตรวจจับฟีเจอร์** — ระบุปลั๊กอินและฟังก์ชันการทำงาน
- **การวิเคราะห์ประสิทธิภาพ** — ตรวจสอบความเร็วของหน้าเว็บและการปรับปรุงให้เหมาะสม
- **การวิเคราะห์ SEO** — ตรวจสอบ meta tags และ structured data

**วิธีการใช้งาน:**

```
Analyze the design of example.com and use it as inspiration for my site.
```

**สิ่งที่ดึงข้อมูลออกมาได้:**

- ชุดสีและวิธีการใช้สี
- ตัวอักษร (ฟอนต์และขนาด)
- โครงสร้างการจัดวางและระยะห่าง
- รูปแบบการนำทาง (Navigation patterns)
- การจัดระเบียบเนื้อหา
- การใช้รูปภาพและสื่อต่างๆ
- องค์ประกอบแบบโต้ตอบ (Interactive elements)
- การรองรับการแสดงผลบนมือถือ (Mobile responsiveness)

**ข้อจำกัด:**

- ไม่สามารถดึงข้อมูลจากเว็บไซต์ที่ป้องกันด้วยรหัสผ่านได้
- เคารพ robots.txt และนโยบายของเว็บไซต์
- อาจไม่สามารถจับเนื้อหาแบบไดนามิกได้
- เว็บไซต์ที่ใช้ JavaScript หนักๆ อาจมีการวิเคราะห์ที่จำกัด
- เว็บไซต์ขนาดใหญ่อาจใช้เวลานานในการวิเคราะห์

### เครื่องมือ Tier 2 (ทางเลือก)

เครื่องมือ Tier 2 จะสามารถใช้งานได้เมื่อเปิดใช้งานอย่างชัดเจนในการตั้งค่า Theme Builder

#### Advanced Analytics

ให้เมตริกประสิทธิภาพโดยละเอียด:

- เวลาในการโหลดหน้าเว็บ
- Core Web Vitals
- คะแนน SEO
- คะแนนการเข้าถึง (Accessibility score)
- คะแนนแนวทางปฏิบัติที่ดีที่สุด (Best practices score)

#### Content Optimizer

วิเคราะห์และแนะนำการปรับปรุงสำหรับ:

- ความสามารถในการอ่าน (Readability)
- การปรับปรุง SEO
- การใช้คีย์เวิร์ด
- โครงสร้างเนื้อหา
- ตำแหน่งของ Call-to-action

### เครื่องมือ Tier 3 (พรีเมียม)

เครื่องมือ Tier 3 ต้องมีการตั้งค่าเพิ่มเติมหรือฟีเจอร์แบบพรีเมียม

#### AI Content Generator

สร้างเนื้อหาสำหรับ:

- คำอธิบายผลิตภัณฑ์
- หน้าบริการ
- บทความบล็อก
- Meta descriptions
- ข้อความ Call-to-action

#### Advanced Design Tools

ให้ความสามารถด้านการออกแบบขั้นสูง:

- การสร้าง Custom CSS
- การสร้างแอนิเมชัน
- การออกแบบองค์ประกอบแบบโต้ตอบ
- ทฤษฎีสีขั้นสูง
- การปรับปรุงตัวอักษร (Typography optimization)

## การตั้งค่าเครื่องมือ

### การเปิดใช้งานเครื่องมือ

ในการเปิดใช้งานเครื่องมือเพิ่มเติมใน Theme Builder:

1. ไปที่ **Gratis AI Agent → Settings**
2. ไปที่ **Theme Builder → Tools**
3. เปิด/ปิดเครื่องมือตามที่ต้องการ
4. บันทึกการตั้งค่า

### สิทธิ์การใช้งานเครื่องมือ

เครื่องมือบางตัวจำเป็นต้องมีสิทธิ์การใช้งาน:

- **Site scraping** — ต้องมีการเข้าถึงอินเทอร์เน็ต
- **Analytics** — ต้องมีการเชื่อมต่อกับ Google Analytics
- **Content generation** — ต้องใช้ API keys
- **Advanced features** — อาจต้องมีการสมัครสมาชิกแบบพรีเมียม

### API Keys และ Credentials

กำหนดค่า API keys สำหรับเครื่องมือที่ต้องการ:

1. ไปที่ **Gratis AI Agent → Settings → API Keys**
2. ป้อน credentials สำหรับแต่ละเครื่องมือ
3. ทดสอบการเชื่อมต่อ
4. บันทึกอย่างปลอดภัย

## การใช้ sd-ai-agent/site-scrape

### การใช้งานพื้นฐาน

ขอให้ Theme Builder วิเคราะห์เว็บไซต์:

```
Analyze the design of my competitor's site at competitor.com
and suggest design improvements for my site.
```

### การวิเคราะห์เฉพาะเจาะจง

ร้องขอการวิเคราะห์ประเภทเฉพาะ:

```
Extract the color palette from example.com and use it as inspiration.
```

```
Analyze the navigation structure of example.com and apply similar
organization to my site.
```

```
Check the mobile responsiveness of example.com and ensure my site
is equally responsive.
```

### การวิเคราะห์เปรียบเทียบ

เปรียบเทียบหลายเว็บไซต์:

```
Compare the designs of site1.com and site2.com and create a design
that combines the best elements of both.
```

## ข้อจำกัดและข้อควรพิจารณาของเครื่องมือ

### Rate Limiting

- การดึงข้อมูลถูกจำกัดอัตราเพื่อป้องกันเซิร์ฟเวอร์ล่ม
- การร้องขอหลายครั้งไปยังเว็บไซต์เดียวกันอาจถูกจำกัด
- เว็บไซต์ขนาดใหญ่อาจใช้เวลานานในการวิเคราะห์

### ความแม่นยำของเนื้อหา

- เนื้อหาแบบไดนามิกอาจไม่ถูกจับทั้งหมด
- เนื้อหาที่เรนเดอร์ด้วย JavaScript อาจไม่สมบูรณ์
- ข้อมูลแบบเรียลไทม์อาจไม่สะท้อน
- เนื้อหาบางส่วนอาจอยู่หลัง paywalls

### ข้อควรพิจารณาทางกฎหมายและจริยธรรม

- เคารพ robots.txt และนโยบายของเว็บไซต์
- ห้ามดึงข้อมูลเนื้อหาที่มีลิขสิทธิ์เพื่อนำไปทำซ้ำ
- ใช้การวิเคราะห์เพื่อเป็นแรงบันดาลใจ ไม่ใช่การคัดลอก
- ตรวจสอบว่าคุณมีสิทธิ์ในการใช้เนื้อหาที่ดึงมาหรือไม่
- ปฏิบัติตามข้อกำหนดในการให้บริการของเว็บไซต์ที่วิเคราะห์

### ผลกระทบต่อประสิทธิภาพ

- การดึงข้อมูลเว็บไซต์ขนาดใหญ่อาจใช้เวลา
- การดึงข้อมูลพร้อมกันหลายครั้งอาจช้ากว่า
- การเชื่อมต่อเครือข่ายส่งผลต่อความเร็ว
- ชุดข้อมูลขนาดใหญ่อาจต้องใช้การประมวลผลมากขึ้น

## แนวทางปฏิบัติที่ดีที่สุด

### การใช้การวิเคราะห์เว็บไซต์

1. **วิเคราะห์หลายเว็บไซต์** — รวบรวมแรงบันดาลใจจากหลายแหล่ง
2. **เน้นที่โครงสร้าง** — เรียนรู้รูปแบบการจัดวางและการจัดระเบียบ
3. **ดึงสี** — ใช้ชุดสีเป็นจุดเริ่มต้น
4. **ศึกษาตัวอักษร** — ระบุการผสมผสานฟอนต์ที่คุณชอบ
5. **ทบทวนการนำทาง** — ทำความเข้าใจการจัดระเบียบเมนู

### การดึงข้อมูลอย่างมีจริยธรรม

1. **เคารพ robots.txt** — ปฏิบัติตามแนวทางของเว็บไซต์
2. **ห้ามคัดลอกเนื้อหา** — ใช้การวิเคราะห์เพื่อเป็นแรงบันดาลใจเท่านั้น
3. **ตรวจสอบสิทธิ์** — ตรวจสอบให้แน่ใจว่าคุณสามารถใช้เนื้อหาที่ดึงมาได้
4. **ระบุแหล่งที่มา** — ให้เครดิตแหล่งที่มาของแรงบันดาลใจ
5. **ปฏิบัติตามข้อกำหนด** — ปฏิบัติตามข้อกำหนดในการให้บริการของเว็บไซต์

### การเพิ่มประสิทธิภาพเครื่องมือ

1. **ระบุให้ชัดเจน** — ร้องขอการวิเคราะห์ประเภทเฉพาะ
2. **ให้บริบท** — อธิบายวัตถุประสงค์ของเว็บไซต์ของคุณ
3. **กำหนดความคาดหวัง** — อธิบายเป้าหมายการออกแบบของคุณ
4. **ทำซ้ำ** — ปรับปรุงตามผลลัพธ์ที่ได้
5. **รวมเครื่องมือ** — ใช้หลายเครื่องมือเพื่อการวิเคราะห์ที่ครอบคลุม

## การแก้ไขปัญหา

### เว็บไซต์ไม่สามารถดึงข้อมูลได้

- ตรวจสอบว่าเว็บไซต์สามารถเข้าถึงได้ในที่สาธารณะหรือไม่
- ตรวจสอบว่า robots.txt อนุญาตให้ดึงข้อมูลหรือไม่
- ลองใช้เว็บไซต์อื่น
- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ติดต่อฝ่ายสนับสนุนหากปัญหายังคงอยู่

### การวิเคราะห์ไม่สมบูรณ์

- เว็บไซต์อาจมีเนื้อหาแบบไดนามิก
- JavaScript อาจไม่ได้ถูกเรนเดอร์อย่างสมบูรณ์
- เว็บไซต์ขนาดใหญ่อาจหมดเวลา (timeout)
- ลองวิเคราะห์เฉพาะหน้าเพจแทน
- ร้องขอประเภทการวิเคราะห์ที่เฉพาะเจาะจง

### ประสิทธิภาพช้า

- เว็บไซต์ขนาดใหญ่อาจใช้เวลานานในการวิเคราะห์
- การดึงข้อมูลพร้อมกันหลายครั้งจะช้ากว่า
- การเชื่อมต่อเครือข่ายส่งผลต่อความเร็ว
- ลองทำในช่วงเวลาที่คนใช้น้อย
- วิเคราะห์ส่วนย่อยๆ ก่อน
