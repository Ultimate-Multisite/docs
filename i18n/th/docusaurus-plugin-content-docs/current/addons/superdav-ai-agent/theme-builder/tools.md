# เครื่องมือและการกำหนดค่า

Theme Builder ใช้ชุดเครื่องมือเพื่อวิเคราะห์ ออกแบบ และสร้างเว็บไซต์ WordPress ของคุณ ในเวอร์ชัน 1.16.0, **sd-ai-agent/site-scrape ตอนนี้เป็นเครื่องมือ Tier 1** ทำให้พร้อมใช้งานตามค่าเริ่มต้นใน Theme Builder นอกจากนี้ Superdav AI Agent v1.18.0 ยังเปิดตัวแพ็กเกจ Advanced companion ที่แจกจ่ายแยกต่างหากสำหรับเวิร์กโฟลว์ของนักพัฒนาที่เชื่อถือได้ เครื่องมือเหล่านั้นไม่ได้เป็นส่วนหนึ่งของแพ็กเกจหลักจาก WordPress.org และต้องติดตั้งและอนุญาตแยกต่างหาก

## เครื่องมือที่พร้อมใช้งาน {#available-tools}

### เครื่องมือ Tier 1 (พร้อมใช้งานเสมอ) {#tier-1-tools-always-available}

เครื่องมือ Tier 1 พร้อมใช้งานตามค่าเริ่มต้นใน Theme Builder โดยไม่ต้องกำหนดค่าเพิ่มเติมใด ๆ

#### sd-ai-agent/site-scrape {#sd-ai-agentsite-scrape}

**วัตถุประสงค์:** วิเคราะห์เว็บไซต์ที่มีอยู่เพื่อดึงแรงบันดาลใจด้านการออกแบบ โครงสร้างเนื้อหา และฟังก์ชันการทำงาน

**ความสามารถ:**

- **การวิเคราะห์เว็บไซต์** — ดึงข้อมูลและวิเคราะห์เว็บไซต์คู่แข่งหรือเว็บไซต์ที่ใช้เป็นแรงบันดาลใจ
- **การดึงข้อมูลการออกแบบ** — ระบุสี ฟอนต์ และรูปแบบเลย์เอาต์
- **โครงสร้างเนื้อหา** — ทำความเข้าใจการจัดระเบียบหน้าและลำดับชั้น
- **การตรวจจับฟีเจอร์** — ระบุ plugin และฟังก์ชันการทำงาน
- **การวิเคราะห์ประสิทธิภาพ** — ตรวจสอบความเร็วหน้าและการปรับแต่ง
- **การวิเคราะห์ SEO** — ตรวจทานเมตาแท็กและข้อมูลที่มีโครงสร้าง

**การใช้งาน:**

```
วิเคราะห์การออกแบบของ example.com และใช้เป็นแรงบันดาลใจสำหรับเว็บไซต์ของฉัน
```

**สิ่งที่เครื่องมือนี้ดึงข้อมูล:**

- ชุดสีและการใช้สี
- ตัวอักษร (ฟอนต์และขนาด)
- โครงสร้างเลย์เอาต์และระยะห่าง
- รูปแบบการนำทาง
- การจัดระเบียบเนื้อหา
- การใช้รูปภาพและสื่อ
- องค์ประกอบแบบโต้ตอบ
- การตอบสนองบนมือถือ

**ข้อจำกัด:**

- ไม่สามารถดึงข้อมูลเว็บไซต์ที่ป้องกันด้วยรหัสผ่าน
- เคารพ robots.txt และนโยบายของเว็บไซต์
- อาจไม่สามารถจับเนื้อหาแบบไดนามิกได้
- เว็บไซต์ที่ใช้ JavaScript อย่างหนักอาจมีการวิเคราะห์ที่จำกัด
- เว็บไซต์ขนาดใหญ่อาจใช้เวลาวิเคราะห์นานขึ้น

### เครื่องมือ Tier 2 (ไม่บังคับ) {#tier-2-tools-optional}

เครื่องมือ Tier 2 พร้อมใช้งานเมื่อเปิดใช้งานอย่างชัดเจนในการตั้งค่า Theme Builder

#### การวิเคราะห์ขั้นสูง {#advanced-analytics}

ให้เมตริกประสิทธิภาพโดยละเอียด:

- เวลาโหลดหน้า
- Core Web Vitals
- คะแนน SEO
- คะแนนการเข้าถึง
- คะแนนแนวทางปฏิบัติที่ดีที่สุด

#### ตัวปรับแต่งเนื้อหา {#content-optimizer}

วิเคราะห์และแนะนำการปรับปรุงสำหรับ:

- ความอ่านง่าย
- การปรับแต่ง SEO
- การใช้คำหลัก
- โครงสร้างเนื้อหา
- การวางตำแหน่งคำกระตุ้นให้ดำเนินการ

### เครื่องมือ Tier 3 (Premium) {#tier-3-tools-premium}

เครื่องมือ Tier 3 ต้องการการกำหนดค่าเพิ่มเติมหรือฟีเจอร์แบบ premium

#### ตัวสร้างเนื้อหา AI {#ai-content-generator}

สร้างเนื้อหาสำหรับ:

- คำอธิบาย product
- หน้า service
- บทความบล็อก
- คำอธิบายเมตา
- ข้อความคำกระตุ้นให้ดำเนินการ

#### เครื่องมือออกแบบขั้นสูง {#advanced-design-tools}

ให้ความสามารถด้านการออกแบบขั้นสูง:

- การสร้าง CSS แบบกำหนดเอง
- การสร้างแอนิเมชัน
- การออกแบบองค์ประกอบแบบโต้ตอบ
- ทฤษฎีสีขั้นสูง
- การปรับแต่งตัวอักษร

### เครื่องมือ Advanced Companion (แพ็กเกจแยกต่างหาก) {#advanced-companion-tools-separate-package}

แพ็กเกจ Advanced companion แจกจ่ายแยกต่างหากจากรุ่นแกนหลักของ Superdav AI Agent มีไว้สำหรับสภาพแวดล้อมของนักพัฒนาและเจ้าของเว็บไซต์ที่เชื่อถือได้ ซึ่งผู้ดูแลระบบอนุญาตอย่างชัดเจนให้ agent ใช้เครื่องมือที่มีผลกระทบสูง

ความสามารถของ Advanced companion อาจรวมถึง:

- **เครื่องมือระบบไฟล์** — ตรวจสอบและแก้ไขไฟล์ที่ได้รับอนุมัติในสภาพแวดล้อมที่เชื่อถือได้
- **เครื่องมือฐานข้อมูล** — สืบค้นหรืออัปเดตข้อมูลเว็บไซต์เมื่อได้รับอนุญาตอย่างชัดเจน
- **เครื่องมือ WP-CLI** — เรียกใช้คำสั่งบำรุงรักษาและตรวจสอบ WordPress
- **เครื่องมือตัวกระจาย REST** — เรียก endpoint ของ REST ที่ลงทะเบียนผ่านเวิร์กโฟลว์การกระจายที่ควบคุมได้
- **เครื่องมือสร้าง plugin** — สร้างโครงร่างและทำซ้ำบนโค้ด plugin
- **เครื่องมือสแนปช็อต Git** — สร้างสแนปช็อตก่อนการดำเนินการของนักพัฒนาที่มีความเสี่ยง
- **เครื่องมือจัดการผู้ใช้** — ช่วยตรวจทานหรือปรับระเบียนผู้ใช้เมื่อได้รับอนุญาต
- **เครื่องมือ benchmark** — เรียกใช้ benchmark ด้านประสิทธิภาพหรือความสามารถเพื่อการตรวจทานในการพัฒนา

อย่าจัดทำเอกสารเครื่องมือเหล่านี้ว่าเป็นเครื่องมือ Theme Builder ที่พร้อมใช้งานโดยทั่วไปสำหรับทุกการติดตั้ง เครื่องมือเหล่านี้พร้อมใช้งานเฉพาะเมื่อแพ็กเกจ Advanced companion ได้รับการติดตั้ง เปิดใช้งาน และจำกัดไว้สำหรับผู้ดูแลระบบที่เชื่อถือได้เท่านั้น

## การกำหนดค่าเครื่องมือ {#tool-configuration}

### การเปิดใช้งานเครื่องมือ {#enabling-tools}

เพื่อเปิดใช้งานเครื่องมือเพิ่มเติมใน Theme Builder:

1. ไปที่ **Gratis AI Agent → Settings**
2. ไปที่ **Theme Builder → Tools**
3. เปิด/ปิดเครื่องมือตามต้องการ
4. บันทึกการตั้งค่า

### สิทธิ์ของเครื่องมือ {#tool-permissions}

เครื่องมือบางอย่างต้องการสิทธิ์:

- **การดึงข้อมูลเว็บไซต์** — ต้องการการเข้าถึงอินเทอร์เน็ต
- **การวิเคราะห์** — ต้องการการผสานรวม Google Analytics
- **การสร้างเนื้อหา** — ต้องการคีย์ API
- **ฟีเจอร์ขั้นสูง** — อาจต้องมี subscription แบบ premium
- **เครื่องมือ Advanced companion** — ต้องการแพ็กเกจ Advanced ที่แจกจ่ายแยกต่างหากและความไว้วางใจจากผู้ดูแลระบบอย่างชัดเจน

### คีย์ API และข้อมูลประจำตัว {#api-keys-and-credentials}

กำหนดค่าคีย์ API สำหรับเครื่องมือที่ต้องใช้:

1. ไปที่ **Gratis AI Agent → Settings → API Keys**
2. ป้อนข้อมูลประจำตัวสำหรับแต่ละเครื่องมือ
3. ทดสอบการเชื่อมต่อ
4. บันทึกอย่างปลอดภัย

### การติดตั้งแพ็กเกจ Advanced Companion {#installing-the-advanced-companion-package}

แพ็กเกจ Advanced companion เผยแพร่แยกต่างหากจาก ZIP แกนหลักของ WordPress.org ติดตั้งเฉพาะจากช่องทางการแจกจ่ายอย่างเป็นทางการของโครงการสำหรับรุ่นนั้น จากนั้นจำกัดการเข้าถึงให้เฉพาะผู้ดูแลระบบที่เชื่อถือได้ ตรวจทานเกตการอนุมัติจากมนุษย์และพฤติกรรมสแนปช็อต git ก่อนเปิดใช้งานเครื่องมือระบบไฟล์ ฐานข้อมูล WP-CLI ตัวกระจาย REST ตัวสร้าง plugin การจัดการผู้ใช้ หรือ benchmark บนเว็บไซต์ production

## การใช้ sd-ai-agent/site-scrape {#using-sd-ai-agentsite-scrape}

### การใช้งานพื้นฐาน {#basic-usage}

ขอให้ Theme Builder วิเคราะห์เว็บไซต์:

```
วิเคราะห์การออกแบบของเว็บไซต์คู่แข่งของฉันที่ competitor.com
และแนะนำการปรับปรุงการออกแบบสำหรับเว็บไซต์ของฉัน
```

### การวิเคราะห์เฉพาะเจาะจง {#specific-analysis}

ขอประเภทการวิเคราะห์ที่เฉพาะเจาะจง:

```
ดึงชุดสีจาก example.com และใช้เป็นแรงบันดาลใจ
```

```
วิเคราะห์โครงสร้างการนำทางของ example.com และใช้การจัดระเบียบ
ที่คล้ายกันกับเว็บไซต์ของฉัน
```

```
ตรวจสอบการตอบสนองบนมือถือของ example.com และตรวจให้แน่ใจว่าเว็บไซต์ของฉัน
ตอบสนองได้ดีเท่ากัน
```

### การวิเคราะห์เปรียบเทียบ {#comparison-analysis}

เปรียบเทียบหลายเว็บไซต์:

```
เปรียบเทียบการออกแบบของ site1.com และ site2.com และสร้างการออกแบบ
ที่รวมองค์ประกอบที่ดีที่สุดของทั้งสองเข้าด้วยกัน
```

## ข้อจำกัดและข้อควรพิจารณาของเครื่องมือ {#tool-limitations-and-considerations}

### การจำกัดอัตรา {#rate-limiting}

- การ scraping ถูกจำกัดอัตราเพื่อป้องกันไม่ให้เซิร์ฟเวอร์ทำงานหนักเกินไป
- คำขอหลายรายการไปยังเว็บไซต์เดียวกันอาจถูกชะลอ
- เว็บไซต์ขนาดใหญ่อาจใช้เวลาวิเคราะห์นานขึ้น

### ความถูกต้องของเนื้อหา {#content-accuracy}

- เนื้อหาแบบไดนามิกอาจไม่ถูกจับภาพได้ครบถ้วน
- เนื้อหาที่เรนเดอร์ด้วย JavaScript อาจไม่สมบูรณ์
- ข้อมูลแบบเรียลไทม์อาจไม่ถูกสะท้อน
- เนื้อหาบางส่วนอาจอยู่หลังกำแพงจ่ายเงิน

### ข้อพิจารณาทางกฎหมายและจริยธรรม {#legal-and-ethical-considerations}

- เคารพ robots.txt และนโยบายของเว็บไซต์
- อย่า scrape เนื้อหาที่มีลิขสิทธิ์เพื่อทำซ้ำ
- ใช้การวิเคราะห์เพื่อเป็นแรงบันดาลใจ ไม่ใช่เพื่อคัดลอก
- ตรวจสอบว่าคุณมีสิทธิ์ใช้เนื้อหาที่ดึงออกมา
- ปฏิบัติตามข้อกำหนดการให้บริการของเว็บไซต์ที่วิเคราะห์

### ผลกระทบต่อประสิทธิภาพ {#performance-impact}

- การ scrape เว็บไซต์ขนาดใหญ่อาจใช้เวลา
- การ scrape พร้อมกันหลายรายการอาจช้าลง
- การเชื่อมต่อเครือข่ายส่งผลต่อความเร็ว
- ชุดข้อมูลขนาดใหญ่อาจต้องใช้การประมวลผลมากขึ้น

## แนวปฏิบัติที่ดีที่สุด {#best-practices}

### การใช้การวิเคราะห์เว็บไซต์ {#using-site-analysis}

1. **วิเคราะห์หลายเว็บไซต์** — รวบรวมแรงบันดาลใจจากหลายแหล่ง
2. **มุ่งเน้นที่โครงสร้าง** — เรียนรู้รูปแบบเลย์เอาต์และการจัดระเบียบ
3. **ดึงสีออกมา** — ใช้ชุดสีเป็นจุดเริ่มต้น
4. **ศึกษาการใช้ตัวอักษร** — ระบุการจับคู่ฟอนต์ที่คุณชอบ
5. **ตรวจสอบการนำทาง** — ทำความเข้าใจการจัดระเบียบเมนู

### การ Scraping อย่างมีจริยธรรม {#ethical-scraping}

1. **เคารพ robots.txt** — ปฏิบัติตามแนวทางของเว็บไซต์
2. **อย่าคัดลอกเนื้อหา** — ใช้การวิเคราะห์เพื่อเป็นแรงบันดาลใจเท่านั้น
3. **ตรวจสอบสิทธิ์** — ตรวจสอบให้แน่ใจว่าคุณสามารถใช้เนื้อหาที่ดึงออกมาได้
4. **ระบุแหล่งที่มา** — ให้เครดิตแหล่งแรงบันดาลใจ
5. **ปฏิบัติตามข้อกำหนด** — ปฏิบัติตามข้อกำหนดการให้บริการของเว็บไซต์

### การเพิ่มประสิทธิผลของเครื่องมือให้สูงสุด {#maximizing-tool-effectiveness}

1. **ระบุให้เฉพาะเจาะจง** — ขอประเภทการวิเคราะห์ที่เฉพาะเจาะจง
2. **ให้บริบท** — อธิบายวัตถุประสงค์ของเว็บไซต์ของคุณ
3. **ตั้งความคาดหวัง** — อธิบายเป้าหมายการออกแบบของคุณ
4. **ทำซ้ำและปรับปรุง** — ปรับปรุงตามผลลัพธ์
5. **ใช้เครื่องมือร่วมกัน** — ใช้หลายเครื่องมือเพื่อการวิเคราะห์ที่ครอบคลุม

## การแก้ไขปัญหา {#troubleshooting}

### ไม่สามารถ Scrape เว็บไซต์ได้ {#site-wont-scrape}

- ตรวจสอบว่าเว็บไซต์เข้าถึงได้แบบสาธารณะหรือไม่
- ตรวจสอบว่า robots.txt อนุญาตให้ scraping หรือไม่
- ลองใช้เว็บไซต์อื่น
- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ติดต่อฝ่ายสนับสนุนหากปัญหายังคงอยู่

### การวิเคราะห์ไม่สมบูรณ์ {#incomplete-analysis}

- เว็บไซต์อาจมีเนื้อหาแบบไดนามิก
- JavaScript อาจไม่ถูกเรนเดอร์ครบถ้วน
- เว็บไซต์ขนาดใหญ่อาจหมดเวลา
- ลองวิเคราะห์หน้าที่เฉพาะเจาะจงแทน
- ขอประเภทการวิเคราะห์ที่เฉพาะเจาะจง

### ประสิทธิภาพช้า {#slow-performance}

- เว็บไซต์ขนาดใหญ่ใช้เวลาวิเคราะห์นานขึ้น
- การ scrape พร้อมกันหลายรายการช้าลง
- การเชื่อมต่อเครือข่ายส่งผลต่อความเร็ว
- ลองทำในช่วงเวลาที่ไม่ใช่ช่วงพีค
- วิเคราะห์ส่วนที่เล็กกว่าก่อน
