# Multi-Tenancy

กระจายไซต์ย่อยไปยังเซิร์ฟเวอร์และฐานข้อมูลต่าง ๆ เพื่อประสิทธิภาพและความปลอดภัย

## ไฮไลต์การเปิดตัวเวอร์ชัน 1.2.0

Multi-Tenancy 1.2.0 เพิ่มรากฐานเทนแนนต์แบบอธิปไตยสำหรับเวิร์กโฟลว์การแยกสภาพแวดล้อมการใช้งานจริง:

- การเข้าสู่ระบบอัตโนมัติของเทนแนนต์แบบไร้สถานะ พร้อมการเข้าชม SSO อายุสั้นที่ป้องกันการเล่นซ้ำ
- การแยกฐานข้อมูลและระบบไฟล์รายไซต์ย่อยสำหรับเทนแนนต์แบบอธิปไตย
- ความสามารถด้าน multi-tenancy ของ Hostinger สำหรับการจัดเตรียมเทนแนนต์ที่รับรู้โฮสต์
- ประตูตรวจสอบการย้ายแบบอธิปไตยสำหรับการตรวจสอบข้อมูลเดิมและการประมวลผลแบบ push แบบอะซิงโครนัส
- การผูกโฮสต์ฐานข้อมูลบนเครื่องเดียวกันที่กำหนดค่าได้สำหรับการติดตั้ง Bedrock, FrankenPHP และ localhost
- การปรับปรุงการบูตสแตรปเทนแนนต์ ตัวเขียนฐานข้อมูล การบูตสแตรปสคีมา และการจัดเตรียมผู้ใช้
- การรื้อถอนแบบอธิปไตยที่ปลอดภัยยิ่งขึ้นซึ่งล้างข้อมูลรับรองของเทนแนนต์

ดูคู่มือผู้ใช้สำหรับ [เวิร์กโฟลว์การย้ายแบบอธิปไตย](/user-guide/getting-started/sovereign-migration-workflow), [การแยก multi-tenancy](/user-guide/configuration/multi-tenancy-isolation) และ [การดำเนินงานเทนแนนต์แบบอธิปไตย](/user-guide/administration/sovereign-tenant-operations)

## ข้อกำหนด

- WordPress 5.3 หรือสูงกว่า
- PHP 7.4 หรือสูงกว่า
- ปลั๊กอิน Multisite Ultimate (เปิดใช้งานอยู่)

## การติดตั้ง

1. อัปโหลดไฟล์ส่วนเสริมไปยังไดเรกทอรี `/wp-content/plugins/` ของคุณ
2. เปิดใช้งานปลั๊กอินผ่านเมนู 'Plugins' ใน WordPress
3. กำหนดค่าการตั้งค่าส่วนเสริมในผู้ดูแลระบบเครือข่าย

## การพัฒนา

### การตั้งค่า

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### การทดสอบ

ส่วนเสริมนี้มีเฟรมเวิร์กการทดสอบที่ครอบคลุม:

- **การทดสอบหน่วย**: การทดสอบที่ใช้ PHPUnit พร้อมชุดทดสอบ WordPress
- **คลาสทดสอบพื้นฐาน**: มีเมธอดตัวช่วยสำหรับสถานการณ์ทดสอบทั่วไป
- **ความครอบคลุมการทดสอบ**: กำหนดค่าให้ติดตามความครอบคลุมของโค้ด
- **การทดสอบ Multisite**: การทดสอบทำงานในสภาพแวดล้อม multisite

### มาตรฐานโค้ด

- **มาตรฐานการเขียนโค้ดของ WordPress**: บังคับใช้ผ่าน PHPCS
- **ความเข้ากันได้กับ PHP 7.4+**: รองรับฟีเจอร์ PHP สมัยใหม่
- **การวิเคราะห์แบบสแตติก**: การผสานรวม PHPStan สำหรับการตรวจสอบประเภท
- **การปรับโค้ดให้ทันสมัย**: กฎ Rector สำหรับการอัปเกรด PHP

### ระบบบิลด์

- **การประมวลผลแอสเซต**: การย่อขนาดไฟล์ CSS/JS
- **การแปล**: การสร้างไฟล์ POT
- **การแพ็กเกจ**: การสร้างอาร์ไคฟ์สำหรับการแจกจ่าย
- **โหมดการพัฒนา**: แอสเซตที่ไม่ถูกย่อขนาดสำหรับการดีบัก

## การกำหนดค่า

ส่วนเสริมรองรับตัวเลือกการกำหนดค่าต่าง ๆ ที่เข้าถึงได้ผ่านอินเทอร์เฟซผู้ดูแลระบบเครือข่าย

### การกำหนดค่าเทนแนนต์แบบอธิปไตย

สำหรับเทนแนนต์แบบอธิปไตย ให้กำหนดค่าฐานข้อมูลเทนแนนต์ รากระบบไฟล์ การผูกโฮสต์ฐานข้อมูล และความสามารถของโฮสต์ก่อนกำหนดเส้นทางทราฟฟิกไปยังเทนแนนต์ เรียกใช้การตรวจสอบการย้ายหลังจากการเปลี่ยนแปลงทุกครั้งที่ส่งผลต่อข้อมูลรับรองฐานข้อมูล การผูกโฮสต์ การกำหนดเส้นทาง หรือการจัดเตรียมผู้ใช้

ใช้ `localhost` เฉพาะเมื่อรันไทม์เทนแนนต์และสิทธิ์ฐานข้อมูลคาดหวังการเข้าถึงผ่านซ็อกเก็ตภายในเครื่องหรือเครื่องเดียวกัน ใช้ `127.0.0.1` หรือชื่อโฮสต์บริการเมื่อสิ่งนั้นเป็นโฮสต์สิทธิ์ที่ผู้ให้บริการของคุณกำหนดค่าไว้

### ความสามารถของ Hostinger

ความสามารถด้าน multi-tenancy ของ Hostinger ทำงานร่วมกับโทเค็นการผสานรวม Hostinger แบบใช้ร่วมกัน ยืนยันว่าโดเมนมีอยู่ใน hPanel, DNS ชี้ไปยัง Account ของ Hostinger และการตั้งค่าฐานข้อมูลเทนแนนต์ตรงกับ hPanel ก่อนการตรวจสอบ

## Hooks และ Filters

### Actions

- `ultimate-multisite-addon-template_init` - เรียกใช้งานหลังจากการเริ่มต้นส่วนเสริม
- `ultimate-multisite-addon-template_loaded` - เรียกใช้งานหลังจากโหลดไฟล์ส่วนเสริมทั้งหมดแล้ว

### Filters

- `ultimate-multisite-addon-template_settings` - แก้ไขการตั้งค่าส่วนเสริม
- `ultimate-multisite-addon-template_enabled` - แทนที่สถานะเปิด/ปิดใช้งานส่วนเสริม
