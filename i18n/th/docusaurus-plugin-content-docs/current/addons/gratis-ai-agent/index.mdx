# Gratis AI Agent {#gratis-ai-agent}

Gratis AI Agent คือผู้ช่วย AI อัตโนมัติสำหรับ WordPress ที่สามารถวางแผน สร้าง และจัดการเว็บไซต์ WordPress ทั้งหมดได้ เพียงแค่ใช้คำสั่งที่เป็นภาษาธรรมชาติ มันเปิดเผยชุด **ความสามารถ (abilities)** ที่มีโครงสร้าง ซึ่งเป็นชุดการกระทำที่แยกส่วนและสามารถทดสอบได้ที่เอเจนต์สามารถเรียกใช้ได้ ครอบคลุมทุกอย่างตั้งแต่การจัดการเนื้อหาไปจนถึงการปรับแต่งระบบดีไซน์ (design system) เวอร์ชัน 1.9.0 ได้เพิ่มความสามารถในการสร้างเนื้อหา การรีวิวหน้าเว็บด้วยภาพหน้าจอจากฝั่งไคลเอนต์ (client-side screenshots) โปรไฟล์เอเจนต์ในตัว 5 โปรไฟล์ และ feature flags สำหรับการควบคุมการเข้าถึงและการสร้างแบรนด์

## คุณสมบัติหลัก (Key Features) {#key-features}

- **ระบบความสามารถ (Abilities system)** — แคตตาล็อกการกระทำที่ปรับเปลี่ยนได้และขยายได้ ซึ่งเอเจนต์สามารถเรียกใช้กับ WordPress ของคุณ
- **การจัดการเนื้อหา (Content Management)** — สร้างโพสต์และหน้า (รวมถึงการกำหนดเทมเพลตหน้า), สร้างโพสต์จำนวนมากพร้อมกันในการเรียกใช้ครั้งเดียว, กำหนดรูปภาพเด่น (featured images), และสร้างแบบฟอร์มติดต่อ
- **การรีวิวด้วยภาพ (Visual Review)** — จับภาพหน้าจอของหน้าเว็บจริง, เปรียบเทียบสถานะก่อน/หลัง (before/after), และรับการรีวิวการออกแบบที่สร้างโดย AI ซึ่งครอบคลุมเรื่องเลย์เอาต์, ตัวอักษร, สี, และการเข้าถึง (accessibility)
- **เอเจนต์ในตัว 5 ตัว (Five built-in agents)** — Content Writer, Site Builder, Design Studio, Plugin Manager, และ Support Assistant; สามารถสลับได้ผ่าน Agent Picker ที่ส่วนหัวของแชท
- **Feature flags** — ตัวสลับสำหรับควบคุมการเข้าถึงและการสร้างแบรนด์ใน Settings → Feature Flags สำหรับการจำกัดบทบาท (role restrictions), การตั้งชื่อแบบ white-label, และการปรับแต่งวิดเจ็ต
- **การจัดการ Custom Post Type** — ลงทะเบียน, แสดงรายการ, และลบ custom post types พร้อมการคงอยู่ของข้อมูลแม้จะรีสตาร์ท
- **การจัดการ Custom Taxonomy** — ลงทะเบียน, แสดงรายการ, และลบ custom taxonomies พร้อมการคงอยู่ของข้อมูล
- **ความสามารถของ Design System** — แทรก CSS แบบกำหนดเอง, จัดการ block patterns ที่คัดสรรมาแล้ว, กำหนดโลโก้เว็บไซต์, และใช้ theme.json presets
- **การจัดการ Global Styles** — อ่านและเขียนค่า theme.json สำหรับพาเลทสี, ตัวอักษร, ระยะห่าง, และอื่น ๆ
- **การจัดการเมนูนำทาง (Navigation Menu management)** — สร้างและอัปเดตเมนูนำทางและรายการต่าง ๆ ผ่านโค้ด (programmatically)
- **การจัดการ Options** — อ่านและเขียน WordPress options พร้อม blocklist ความปลอดภัยในตัวเพื่อป้องกันการตั้งค่าที่สำคัญ
- **Installable Abilities Registry** — ค้นหาและเปิดใช้งานชุดความสามารถเพิ่มเติมโดยไม่ต้องแก้ไขไฟล์ปลั๊กอินหลัก
- **Site Builder Orchestration v2** — แผนการสร้างเว็บไซต์หลายขั้นตอน พร้อมการค้นหาปลั๊กอิน, การติดตามความคืบหน้า, และการกู้คืนข้อผิดพลาด
- **WP-CLI benchmark command** — รันชุดทดสอบความสามารถของเอเจนต์จาก command line สำหรับ CI และการประเมินโมเดล

## ข้อกำหนด (Requirements) {#requirements}

- WordPress 6.0 หรือสูงกว่า
- PHP 8.1 หรือสูงกว่า
- WordPress Multisite (เปิดใช้งานระดับเครือข่าย) หรือการติดตั้งแบบ single-site
- ปลั๊กอินตัวเชื่อมต่อ AI ที่ใช้งานได้ (Anthropic, OpenAI, หรือที่เข้ากันได้)

## การติดตั้ง (Installation) {#installation}

1. อัปโหลดไฟล์ปลั๊กอินไปยัง `/wp-content/plugins/gratis-ai-agent`
2. เปิดใช้งานปลั๊กอินผ่านหน้า **Plugins** ใน WordPress
3. ไปที่ **Gratis AI Agent → Settings** และเชื่อมต่อ AI provider ของคุณ
4. (ทางเลือก) เรียกดู **Abilities Registry** เพื่อติดตั้งชุดความสามารถเพิ่มเติม

## เริ่มต้นใช้งานอย่างรวดเร็ว (Quick Start) {#quick-start}

เมื่อเปิดใช้งานและเชื่อมต่อกับ AI provider แล้ว ให้เปิดแผง **Gratis AI Agent** ในแถบด้านข้างของ WordPress admin พิมพ์คำสั่งที่เป็นภาษาธรรมชาติ แล้วเอเจนต์จะวางแผนและดำเนินการตามขั้นตอนที่จำเป็นโดยใช้ความสามารถที่มีอยู่

**ตัวอย่างคำสั่ง:**

```
Create a "Portfolio" custom post type with a "Project Category" taxonomy.
```

```
Inject a dark-mode colour palette into the theme and set the primary colour to #1a1a2e.
```

```
Build a restaurant website with an online menu, booking form, and contact page.
```

## ภาพรวมความสามารถ (Abilities Overview) {#abilities-overview}

ความสามารถคือการกระทำระดับอะตอมที่เอเจนต์สามารถทำได้ แต่ละความสามารถคือ PHP class ที่ลงทะเบียน ซึ่งเปิดเผย schema (พารามิเตอร์และประเภทค่าที่ส่งคืน) ที่เอเจนต์อ่านในขณะรันไทม์ ความสามารถเหล่านี้ถูกจัดกลุ่มตามฟังก์ชันการทำงาน:

| Area | Abilities |
|---|---|
| **Content Management** | `create_post`, `update_post`, `batch_create_posts`, `set_featured_image`, `create_contact_form` |
| **Visual Review** | `capture_screenshot`, `compare_screenshots`, `review_page_design` |
| **Custom Post Types** | `register_post_type`, `list_post_types`, `delete_post_type` |
| **Custom Taxonomies** | `register_taxonomy`, `list_taxonomies`, `delete_taxonomy` |
| **Design System** | `inject_custom_css`, `add_block_pattern`, `list_block_patterns`, `set_site_logo`, `apply_theme_json_preset` |
| **Global Styles** | `get_global_styles`, `set_global_styles`, `reset_global_styles` |
| **Navigation Menus** | `create_menu`, `update_menu`, `add_menu_item`, `remove_menu_item`, `list_menus` |
| **Options Management** | `get_option`, `set_option`, `delete_option`, `list_options` |
| **Installable Abilities** | `list_available_abilities`, `install_ability`, `recommend_plugin` |
| **Site Builder** | `create_site_plan`, `execute_site_plan`, `get_plan_progress`, `handle_plan_error` |

ดู [Abilities Reference](./abilities) สำหรับเอกสารพารามิเตอร์ฉบับเต็มและตัวอย่างการใช้งาน

## WP-CLI {#wp-cli}

Gratis AI Agent มาพร้อมกับชุดคำสั่ง `wp gratis-ai-agent` สำหรับการรัน benchmark และการจัดการความสามารถจาก command line ดู [WP-CLI Reference](./wp-cli) สำหรับรายการคำสั่งทั้งหมด

```bash
# Run the full benchmark suite
wp gratis-ai-agent benchmark

# List installed abilities
wp gratis-ai-agent abilities list

# Install an ability pack
wp gratis-ai-agent abilities install <slug>
```

## การตั้งค่า (Configuration) {#configuration}

การตั้งค่าทั้งหมดอยู่ใน **Gratis AI Agent → Settings** ใน WordPress admin:

| Setting | Description |
|---|---|
| **AI Provider** | เลือกและกำหนดค่าตัวเชื่อมต่อ AI provider ที่ใช้งานอยู่ |
| **Model** | เลือกโมเดลภาษาที่จะใช้สำหรับเซสชันของเอเจนต์ |
| **Max Tokens** | งบประมาณโทเค็นต่อการเรียกใช้ (ค่าเริ่มต้น 4096) |
| **Ability Timeout** | ระยะเวลาสูงสุดที่ความสามารถแต่ละตัวสามารถทำงานได้ (ค่าเริ่มต้น 30) |
| **Debug Logging** | บันทึกการเรียกใช้ความสามารถและการให้เหตุผลของเอเจนต์ลงใน WordPress debug log |

## Options Management Safety Blocklist {#options-management-safety-blocklist}

ความสามารถ Options Management มี blocklist ในตัวที่ป้องกันไม่ให้เอเจนต์อ่านหรือเขียน WordPress options ที่มีความอ่อนไหว blocklist เริ่มต้นครอบคลุม:

- `admin_email`, `blogadmin_email`
- `siteurl`, `home` (options URL — ควรใช้เครื่องมือย้ายข้อมูลเฉพาะสำหรับสิ่งเหล่านี้)
- `auth_key`, `secure_auth_key`, `logged_in_key`, `nonce_key` (คีย์ลับ)
- `active_plugins`, `template` (การเปิดใช้งานปลั๊กอิน/ธีม — ควรใช้ความสามารถเฉพาะ)
- ข้อมูลรับรองเกตเวย์การชำระเงินและ API keys ที่ลงทะเบียนโดยปลั๊กอินที่ติดตั้ง

สามารถเพิ่มรายการเพิ่มเติมได้ผ่าน filter `gratis_ai_agent_options_blocklist`

## Hooks and Filters {#hooks-and-filters}

### Actions {#actions}

- `gratis_ai_agent_ability_registered` — ถูกเรียกใช้หลังจากที่ความสามารถถูกลงทะเบียน โดยจะรับชื่อ class ของความสามารถ
- `gratis_ai_agent_plan_started` — ถูกเรียกใช้เมื่อแผนการสร้างเว็บไซต์เริ่มดำเนินการ
- `gratis_ai_agent_plan_completed` — ถูกเรียกใช้เมื่อแผนเสร็จสมบูรณ์
- `gratis_ai_agent_plan_failed` — ถูกเรียกใช้เมื่อแผนพบข้อผิดพลาดที่ไม่สามารถกู้คืนได้

### Filters {#filters}

- `gratis_ai_agent_abilities` — แก้ไขรายการความสามารถที่ลงทะเบียนก่อนที่เอเจนต์จะโหลด
- `gratis_ai_agent_options_blocklist` — ขยายหรือลด blocklist ความปลอดภัยของ options
- `gratis_ai_agent_benchmark_questions` — เพิ่มคำถาม benchmark แบบกำหนดเองให้กับชุดการประเมิน
- `gratis_ai_agent_plan_plugins` — แก้ไขรายการการค้นหาปลั๊กอินระหว่างการสร้างแผนเว็บไซต์
