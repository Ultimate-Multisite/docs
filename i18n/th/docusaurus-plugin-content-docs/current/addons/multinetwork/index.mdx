# Multi-Network {#multi-network}

Addon Ultimate Multisite: Multinetwork ช่วยขยายขีดความสามารถของแพลตฟอร์ม Ultimate Multisite ของคุณด้วยฟังก์ชัน multinetwork ที่ทรงพลัง Addon นี้ช่วยให้ลูกค้าของคุณสามารถซื้อและจัดการเครือข่าย WordPress multisite ทั้งระบบ ไม่ใช่แค่เว็บไซต์เดี่ยว ๆ เท่านั้น

## Key Features {#key-features}

* **Network Provisioning**: สร้างและตั้งค่าเครือข่าย WordPress multisite ที่สมบูรณ์ให้ลูกค้าของคุณโดยอัตโนมัติ
* **Network Templates**: ให้ลูกค้าสามารถเรียกดู ดูตัวอย่าง และเลือกเทมเพลตเครือข่ายที่สร้างไว้ล่วงหน้าก่อนลงทะเบียน
* **Network Management**: ความสามารถในการจัดการระบบสำหรับสภาพแวดล้อม multinetwork อย่างเต็มรูปแบบ
* **Isolated Networks**: แต่ละเครือข่ายของลูกค้าจะทำงานอย่างอิสระ โดยมีตารางฐานข้อมูลและการตั้งค่าของตัวเอง
* **Cross-Network SSO**: ให้ลูกค้าเข้าถึง sub-network ต่าง ๆ ได้อย่างราบรื่นด้วยการยืนยันตัวตนแบบ magic-link
* **Scalable Architecture**: จัดการหลายเครือข่ายได้อย่างมีประสิทธิภาพด้วยการจัดสรรทรัพยากรที่เหมาะสม
* **Integration**: ผสานรวมกับระบบการสมัครสมาชิกและการเรียกเก็บเงินของ Ultimate Multisite ได้อย่างราบรื่น

## Use Cases {#use-cases}

* **White Label Hosting**: ให้บริการเครือข่าย multisite ที่มีแบรนด์ของลูกค้าเอง
* **Agency Solutions**: ช่วยให้เอเจนซี่สามารถจัดการเครือข่ายของลูกค้าหลายรายจากแพลตฟอร์มเดียว
* **Enterprise Deployments**: รองรับองค์กรขนาดใหญ่ที่มีความต้องการเครือข่ายหลายรูปแบบที่ซับซ้อน
* **Education Platforms**: สร้างเครือข่ายแยกต่างหากสำหรับโรงเรียนหรือแผนกที่แตกต่างกัน

## Requirements {#requirements}

* Ultimate Multisite 2.0.0 หรือสูงกว่า
* WordPress Multisite installation
* PHP 7.4 หรือสูงกว่า
* MySQL 5.6 หรือสูงกว่า

## Installation {#installation}

1. อัปโหลดไฟล์ปลั๊กอินไปยังไดเรกทอรี `/wp-content/plugins/ultimate-multisite-multinetwork`
2. เปิดใช้งานปลั๊กอินผ่านหน้า 'Plugins' ใน WordPress
3. กำหนดค่า multinetwork settings ผ่าน Ultimate Multisite → Settings

## Network Templates {#network-templates}

เวอร์ชัน 1.3.0 ได้เพิ่มขั้นตอนการค้นหาและเลือกเทมเพลตสำหรับลูกค้าที่ซื้อเครือข่ายใหม่ คุณควรใช้ network templates เมื่อคุณต้องการให้เครือข่ายลูกค้าใหม่แต่ละแห่งเริ่มต้นจากโครงสร้าง การออกแบบ คลังเนื้อหา หรือการตั้งค่าเฉพาะอุตสาหกรรมที่เตรียมไว้แล้ว

### Template previewer {#template-previewer}

เครื่องมือดูตัวอย่าง network template ช่วยให้ลูกค้าสามารถเรียกดูเทมเพลตที่มีอยู่ได้ก่อนชำระเงิน แต่ละเทมเพลตสามารถเปิดดูในแผงตัวอย่างแบบสด (live preview panel) ได้ ทำให้ลูกค้าสามารถเปรียบเทียบเลย์เอาต์และเนื้อหาก่อนที่จะซื้อเครือข่าย

ใช้ previewer เพื่อช่วยลูกค้า:

* ตรวจสอบรูปลักษณ์และโครงสร้างของ network template ที่มีให้เลือกแต่ละแบบ
* เปรียบเทียบเทมเพลตหลายแบบได้โดยไม่ต้องออกจากขั้นตอนการซื้อ
* เลือกเทมเพลตที่เหมาะสมกับกรณีใช้งานของพวกเขาที่สุดก่อนลงทะเบียน

ควรตั้งชื่อและคำอธิบายเทมเพลตให้เป็นมิตรกับลูกค้าที่สุด เครื่องมือดูตัวอย่างจะมีประโยชน์ที่สุดเมื่อลูกค้าสามารถเข้าใจกรณีการใช้งานที่ตั้งใจไว้ เช่น เครือข่ายของเอเจนซี่ เครือข่ายของโรงเรียน เครือข่ายแฟรนไชส์ หรือชุดบริการ white-label hosting

### Template picker at checkout {#template-picker-at-checkout}

Template picker สามารถเพิ่มเข้าไปในขั้นตอนการชำระเงิน (checkout flow) ของการลงทะเบียนได้ในรูปแบบของฟิลด์ เมื่อตั้งค่าแล้ว ลูกค้าจะเลือก network template ที่ต้องการในระหว่างการชำระเงิน และเทมเพลตที่เลือกจะถูกนำไปใช้เมื่อมีการสร้างเครือข่ายใหม่

วิธีใช้ picker ในขั้นตอนการชำระเงิน:

1. เตรียมเครือข่ายที่คุณต้องการนำเสนอเป็นเทมเพลต
2. เปิดใช้งานฟิลด์การเลือก Multinetwork template ในแบบฟอร์มการลงทะเบียนหรือชำระเงินที่เกี่ยวข้อง
3. เลือกเทมเพลตที่ควรมีให้ใช้สำหรับขั้นตอนการชำระเงินนั้น ๆ
4. เผยแพร่ขั้นตอนการชำระเงินและทดสอบการลงทะเบียนเพื่อยืนยันว่าเทมเพลตที่เลือกถูกนำไปใช้กับเครือข่ายใหม่

หากขั้นตอนการชำระเงินมีผลิตภัณฑ์เครือข่ายเพียงรายการเดียว picker ก็ยังคงมีประโยชน์ในฐานะขั้นตอนการเริ่มต้นใช้งานที่แนะนำ (guided onboarding step) สำหรับ funnels แบบหลายแผน (multi-plan funnels) ให้แสดงเฉพาะเทมเพลตที่เข้ากันได้กับผลิตภัณฑ์หรือแผนที่เลือกเท่านั้น

## Cross-Network Magic Link SSO {#cross-network-magic-link-sso}

Cross-network magic link SSO ช่วยให้ลูกค้าสามารถย้ายระหว่างพื้นที่บัญชีหลักและ sub-network ได้โดยไม่ต้องลงชื่อเข้าใช้ด้วยตนเองอีกครั้ง เมื่อลูกค้าคลิกลิงก์เข้าถึงข้ามเครือข่าย (cross-network access link) Ultimate Multisite จะสร้าง magic link ที่มีระยะเวลาจำกัดและทำการยืนยันตัวตนของลูกค้าไปยังเครือข่ายปลายทาง

สิ่งนี้ช่วยปรับปรุงประสบการณ์สำหรับลูกค้าที่จัดการเครือข่ายมากกว่าหนึ่งเครือข่าย หรือผู้ที่จำเป็นต้องเข้าถึงเครือข่ายเทมเพลตที่เพิ่งโคลนมาทันทีหลังการซื้อ

รายการตรวจสอบสำหรับผู้ดูแลระบบ (Admin checklist):

* ยืนยันว่า Addon Multinetwork เปิดใช้งานอยู่ทั่วทั้งการติดตั้ง WordPress multisite
* ตรวจสอบให้แน่ใจว่าลูกค้ามีการเข้าถึงเครือข่ายที่ถูกต้องก่อนที่จะมีการเสนอลิงก์ข้ามเครือข่าย
* ตั้งค่าโดเมนเครือข่ายและโดเมนที่แมปไว้อย่างถูกต้อง เพื่อให้ magic link สามารถแก้ไขไปยังเครือข่ายปลายทางที่ตั้งใจไว้
* ทดสอบ dashboard ของลูกค้าและปุ่มเข้าถึงใด ๆ หลังการชำระเงินหลังจากเปลี่ยนการตั้งค่าเครือข่าย โดเมน หรือการชำระเงิน

Magic links ควรถูกจัดการเหมือนกับลิงก์สำหรับเข้าสู่ระบบ (login links) โดยออกแบบมาเพื่อให้ลูกค้าเข้าถึงได้อย่างราบรื่นและไม่ควรส่งต่อหรือนำกลับมาใช้ใหม่นอกเซสชันที่ตั้งใจไว้

## Frequently Asked Questions {#frequently-asked-questions}

### What is multinetwork? {#what-is-multinetwork}

Multinetwork คือฟีเจอร์ของ WordPress ที่ช่วยให้คุณสามารถรันเครือข่าย multisite ที่เป็นอิสระหลายเครือข่ายบนการติดตั้ง WordPress เดียว

### How is this different from regular multisite? {#how-is-this-different-from-regular-multisite}

Regular multisite อนุญาตให้มีหลายเว็บไซต์ภายในเครือข่ายเดียว ส่วน Multinetwork อนุญาตให้มีหลายเครือข่าย ซึ่งแต่ละเครือข่ายมีเว็บไซต์ของตัวเองทั้งหมด ภายใต้การติดตั้ง WordPress เดียวกัน

### Does this require special server configuration? {#does-this-require-special-server-configuration}

ใช่ multinetwork ต้องการการตั้งค่าและกำหนดค่าเซิร์ฟเวอร์ที่เฉพาะเจาะจง โปรดดูเอกสารประกอบเพื่อดูคำแนะนำการตั้งค่าโดยละเอียด
