# WooCommerce Integration {#woocommerce-integration}

WooCommerce Integration ทำงานในลักษณะที่คล้ายกับวิธีที่ Manual Gateway ของเราทำงานมาก โดยมอบความยืดหยุ่นอย่างมากให้ผู้ใช้ของคุณ ทำให้พวกเขาสามารถเลือกแบบฟอร์มการชำระเงินใดก็ได้ที่มีอยู่ในการติดตั้ง WooCommerce ของคุณ และใช้งานง่าย: เมื่อรอบการเรียกเก็บเงินของผู้ใช้สิ้นสุดลง การสมัครสมาชิกของผู้ใช้จะถูกพักไว้ ในเวลาเดียวกัน จะมีการสร้างคำสั่งซื้อบนการติดตั้ง WooCommerce ของเว็บไซต์หลักของเครือข่าย จากนั้นผู้ใช้จะได้รับใบแจ้งหนี้ในอีเมล พร้อมลิงก์สำหรับดำเนินการชำระเงิน เมื่อได้รับการชำระเงินแล้ว การสมัครสมาชิกจะได้รับการต่ออายุ

## คุณสมบัติ {#features}

- การผสานรวมที่ง่ายระหว่าง Ultimate Multisite และ WooCommerce
- มีวิธีการชำระเงินหลายแบบให้ใช้งาน
- การเรียกเก็บเงินแบบเกิดซ้ำอัตโนมัติโดยใช้แกนหลักการสมัครสมาชิกที่มาพร้อมกัน พร้อมการปรับปรุงเพิ่มเติมจาก WooCommerce Subscriptions plugin ที่เป็นทางเลือก
- ยอดรวม checkout รวมภาษี Ultimate Multisite เป็นบรรทัดค่าธรรมเนียม WooCommerce แยกต่างหากที่ไม่ต้องเสียภาษี เพื่อให้ลูกค้าสามารถตรวจสอบจำนวนภาษีก่อนชำระเงินได้

## การติดตั้ง {#installation}

1. อัปโหลด 'ultimate-multisite-woocommerce' ไปยังไดเรกทอรี '/wp-content/plugins/'
2. เปิดใช้งาน plugin ผ่านเมนู 'Plugins' ใน WordPress
3. ทำตาม Wizard ทีละขั้นตอนเพื่อตั้งค่า plugin

## คู่มือการใช้งาน {#usage-guide}

_**หมายเหตุสำคัญ: บทความนี้อ้างอิงถึง Ultimate Multisite เวอร์ชัน 2.x**_

_**โปรดทราบ:** Ultimate Multisite: WooCommerce Integration ต้องเปิดใช้งาน WooCommerce อย่างน้อยบนเว็บไซต์หลักของคุณ_

เราเข้าใจว่า _Stripe_ และ _PayPal_ ไม่สามารถใช้งานได้ในบางประเทศ ซึ่งจำกัดหรือเป็นอุปสรรคต่อผู้ใช้ Ultimate Multisite ในการใช้งาน plugin ของเราอย่างมีประสิทธิภาพ ดังนั้นเราจึงสร้าง **add-on เพื่อผสานรวม _WooCommerce_**

_WooCommerce_ เป็น plugin eCommerce ที่ได้รับความนิยมมาก นักพัฒนาทั่วโลกได้สร้าง add-on เพื่อผสานรวม gateway การชำระเงินต่าง ๆ เข้ากับมัน เราใช้ประโยชน์จากสิ่งนี้เพื่อขยาย gateway การชำระเงินที่คุณสามารถใช้ในระบบเรียกเก็บเงินของ Ultimate Multisite

## การตั้งค่า {#setting-it-up}

ก่อนอื่น คุณต้องติดตั้ง add-on WooCommerce Integration ซึ่ง **ฟรีสำหรับผู้ใช้ที่มีไลเซนส์** หากต้องการติดตั้ง ให้คลิก Ultimate Multisite บน super admin dashboard ของคุณ แล้วเลือก **Add-ons**

![หน้า Add-ons](/img/admin/addons-page.png)

ที่นี่คุณจะพบ add-on ทั้งหมดของ Ultimate Multisite คลิกที่ add-on **Ultimate Multisite: WooCommerce Integration**

![หน้า Add-ons ที่แสดง WooCommerce Integration](/img/admin/addons-page.png)

หน้าต่างจะปรากฏขึ้นพร้อมรายละเอียด add-on เพียงคลิก **Install Now** *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

หลังจากการติดตั้งเสร็จสิ้น คุณจะถูกเปลี่ยนเส้นทางไปยังหน้า plugins ที่นี่ เพียงคลิก **Network Activate** แล้ว add-on _WooCommerce_ จะถูกเปิดใช้งานบนเครือข่ายของคุณ

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

หลังจากเปิดใช้งานแล้ว หากคุณยังไม่ได้ติดตั้งและเปิดใช้งาน WooCommerce plugin บนเว็บไซต์ของคุณ **คุณจะได้รับการแจ้งเตือน**

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

เมื่อติดตั้งแล้ว คุณจะเห็นตัวเลือกเพิ่มเติมภายใต้การตั้งค่าการชำระเงินของ Ultimate Multisite จากนั้นคุณสามารถ **เปิดใช้งาน** ตัวเลือก _WooCommerce_ ได้จากที่นั่น

![การตั้งค่า payment gateway ของ Ultimate Multisite โดยเปิดใช้งาน WooCommerce](/img/addons/woocommerce-payment-gateway.png)

คุณสามารถกำหนดค่าการตั้งค่าพื้นฐานบางอย่าง เช่น ชื่อที่แสดงซึ่งจะแสดงที่ front end และคำแนะนำเพื่อนำทางผู้ใช้ของคุณในขั้นตอน checkout จากนั้นคุณสามารถไปที่ **การตั้งค่า _WooCommerce_ ของเว็บไซต์หลักของคุณ** เพื่อเปิดใช้งานและกำหนดค่า payment gateway ที่คุณต้องการใช้

### บรรทัดค่าธรรมเนียมภาษีของ checkout {#checkout-tax-fee-lines}

เมื่อ checkout ของ Ultimate Multisite รวมภาษี การผสานรวมจะเพิ่มจำนวนภาษี Ultimate Multisite แต่ละรายการไปยัง checkout ของ WooCommerce เป็นบรรทัดค่าธรรมเนียมแยกต่างหากที่ไม่ต้องเสียภาษี วิธีนี้ทำให้ยอดภาษียังคงมองเห็นได้ก่อนชำระเงิน ในขณะที่ป้องกันไม่ให้ WooCommerce คิดภาษีกับค่าธรรมเนียมภาษีซ้ำอีกครั้ง

## วิธีจัดการการชำระเงิน {#how-to-manage-the-payment}

WooCommerce Integration ทำงานในลักษณะที่คล้ายกับวิธีที่ **Manual Gateway** ของเราทำงานมาก ข้อเสียคือการชำระเงินทุกรายการต้องชำระด้วยตนเองเมื่อสิ้นสุดรอบการเรียกเก็บเงิน นี่คือวิธีการทำงานของขั้นตอน:

  * เมื่อรอบการเรียกเก็บเงินของผู้ใช้สิ้นสุดลง membership จะถูกพักไว้ **on hold** ในเวลาเดียวกัน จะมีการสร้างคำสั่งซื้อบนการติดตั้ง _WooCommerce_ บนเว็บไซต์หลักของเครือข่ายของคุณ

  * จากนั้นผู้ใช้จะได้รับใบแจ้งหนี้ในอีเมล พร้อมลิงก์สำหรับดำเนินการชำระเงิน (ลิงก์ยังปรากฏในประวัติการเรียกเก็บเงินจาก dashboard ของ subsite ของพวกเขาด้วย) ลิงก์นั้นจะเปลี่ยนเส้นทางผู้ใช้ไปยัง _**WooCommerce**_ **checkout** ซึ่งผู้ใช้จะสามารถเลือกหนึ่งใน **gateway _WooCommerce_ ที่พร้อมใช้งาน** เพื่อดำเนินการชำระเงินของตน

  * เมื่อได้รับการชำระเงินแล้ว membership จะได้รับการ **ต่ออายุ** การชำระเงินจะถูกบันทึกใน Ultimate Multisite และคำสั่งซื้อจะถูกตั้งค่าเป็นเสร็จสมบูรณ์

### หมายเหตุสำคัญบางประการ: {#some-important-notes}

  * ฟังก์ชันเริ่มต้นของ Ultimate Multisite ยังคงทำงานได้กับรูปแบบการผสานรวมนี้ ตัวอย่างเช่น **การออกเงินคืน** จากหน้าจอ **Subscription Management** จะสื่อสารกับ WooCommerce เพื่อออกเงินคืนโดยใช้แบบฟอร์มการชำระเงินที่เลือก

  * Ultimate Multisite: WooCommerce Integration พึ่งพา WP-Cron ในการทำงาน ซึ่งโดยส่วนใหญ่ทำงานได้ดีมาก แต่หากเว็บไซต์มีทราฟฟิกต่ำ cron จะไม่ถูกเรียกให้ตรวจสอบเหตุการณ์ที่กำหนดเวลาไว้ และเหตุการณ์จะถูกพลาดไป ซึ่งอาจทำให้เกิดความล่าช้าในการดำเนินการ

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

การเริ่มชำระเงินด้วยตนเองทุกเดือนไม่เหมาะสำหรับผู้ใช้ปลายทาง เพราะคาดว่าสมาชิกแบบ recurring จะเกิดซ้ำโดยอัตโนมัติในทุกรอบการเรียกเก็บเงิน add-on WooCommerce Integration ตอนนี้รวม core subscriptions code ไว้แล้ว ดังนั้นสินค้าที่เป็น recurring จะใช้ subscriptions สำหรับการเรียกเก็บเงินแบบ recurring เสมอ

คุณ **ไม่** จำเป็นต้องซื้อหรือติดตั้ง [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) แยกต่างหากเพื่อขายสินค้าที่เป็น recurring ผ่าน integration นี้ หาก plugin นั้นถูกซื้อ ติดตั้ง และพร้อมใช้งานบนเว็บไซต์หลักของคุณ Ultimate Multisite จะใช้งาน และ WooCommerce จะแสดงฟีเจอร์การจัดการ subscription เพิ่มเติม เมื่อไม่พร้อมใช้งาน bundled subscriptions core จะยังคงจัดการสินค้าที่เป็น recurring

![การตั้งค่า WooCommerce subscription integration](/img/addons/woocommerce-subscription-integration.png)

และเท่านั้นเอง ผู้ใช้ปลายทางของคุณจะไม่จำเป็นต้องชำระเงินด้วยตนเองเมื่อสิ้นสุดรอบการเรียกเก็บเงินสำหรับสินค้าที่เป็น recurring อีกต่อไป

นอกจากจะมีการสร้างคำสั่งซื้อบนเว็บไซต์หลักของคุณภายใต้ _WooCommerce_ แล้ว ยังจะสร้าง subscription สำหรับแต่ละ Account ด้วย

### พฤติกรรมการระงับเมื่อต่ออายุไม่สำเร็จ {#failed-renewal-suspension-behavior}

โดยค่าเริ่มต้น membership จะยังคงใช้งานอยู่ในขณะที่ WooCommerce Subscriptions ประมวลผลกำหนดการลองชำระเงินซ้ำหลังการชำระเงินล้มเหลว วิธีนี้จะรักษาช่วงผ่อนผันปกติของ WooCommerce Subscriptions เพื่อให้การต่ออายุที่กู้คืนได้สามารถทำให้ subscription และ membership กลับมาซิงค์กันโดยไม่ขัดจังหวะการเข้าถึงก่อนเวลาอันควร

ผู้ดูแลระบบที่ต้องการควบคุมการเข้าถึงอย่างเข้มงวดยิ่งขึ้นสามารถเปิดใช้งาน **Suspend Memberships Immediately on Failed Renewals** ในการตั้งค่า WooCommerce Integration ได้ เมื่อเปิดใช้งานการตั้งค่าแบบ opt-in นี้ Ultimate Multisite จะระงับ membership ที่เกี่ยวข้องทันทีที่การชำระเงินต่ออายุล้มเหลว แทนที่จะรอให้การจัดการการลองซ้ำของ WooCommerce Subscriptions เสร็จสิ้น

นักพัฒนาสามารถ override การตัดสินใจเดียวกันได้ด้วย filter `wu_woo_suspend_on_payment_failure` หน้าอ้างอิง hook สร้างจากซอร์สของ add-on ดังนั้นภาพรวมนี้จึงกล่าวถึงเฉพาะชื่อ filter สำหรับนโยบายการระงับแบบกำหนดเอง
