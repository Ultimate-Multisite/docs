# อีเมล (Emails)

Ultimate Multisite: Emails addon ช่วยให้เจ้าของเครือข่ายสามารถจัดเตรียมและจัดการบัญชีอีเมลของลูกค้าได้พร้อมกับการจัดการเว็บไซต์และการเป็นสมาชิก

เวอร์ชัน 1.1.0 ได้เพิ่มฟังก์ชันการจัดการบัญชีอีเมลที่ลูกค้าสามารถจัดการได้ด้วยตนเอง, การเข้าสู่ระบบเว็บเมลแบบ Single Sign-On (SSO), การรองรับ CyberPanel forwarder, และการปรับปรุงการจัดเตรียมบัญชีในช่วงขั้นตอนการชำระเงิน

## คุณสมบัติหลัก (Key Features)

* **วิดเจ็ตบัญชีอีเมลของลูกค้า (Customer email account widget)**: ลูกค้าสามารถสร้างและจัดการบัญชีอีเมลได้จากหน้าบัญชีของตนเอง โดยไม่จำเป็นต้องเปิดตั๋วสนับสนุน (support ticket)
* **Webmail SSO และ magic links**: ลูกค้าสามารถเข้าถึงเว็บเมลได้โดยตรงจากหน้าบัญชีด้วยการยืนยันตัวตนด้วยคลิกเดียว หากผู้ให้บริการรองรับ
* **การเปลี่ยนรหัสผ่าน**: ทั้งลูกค้าและผู้ดูแลระบบสามารถอัปเดตรหัสผ่านกล่องจดหมายได้จากวิดเจ็ตจัดการอีเมล
* **การจัดเตรียมบัญชีเมื่อชำระเงิน (Checkout provisioning)**: สามารถสร้างบัญชีอีเมลของลูกค้าได้ในระหว่างขั้นตอนการชำระเงิน ทำให้กล่องจดหมายพร้อมใช้งานทันทีที่กระบวนการซื้อเสร็จสมบูรณ์
* **CyberPanel forwarders**: เครือข่ายที่ใช้ CyberPanel สามารถสร้างและจัดการอีเมล forwarder ได้เป็นส่วนหนึ่งของบริการอีเมล
* **การตั้งค่าที่คำนึงถึงผู้ให้บริการ (Provider-aware settings)**: หน้าการตั้งค่าจะกรองผู้ให้บริการให้แสดงเฉพาะการเชื่อมต่อที่ใช้งานได้และตั้งค่าเรียบร้อยแล้ว เพื่อให้ผู้ดูแลระบบเห็นเฉพาะตัวเลือกที่ใช้งานได้จริง

## ข้อกำหนดเบื้องต้น (Requirements)

* Ultimate Multisite 2.0.0 หรือสูงกว่า
* การติดตั้ง WordPress Multisite
* ต้องมีการเชื่อมต่อผู้ให้บริการอีเมลที่ตั้งค่าไว้แล้วอย่างน้อย 1 รายการ
* ข้อมูลรับรอง API ของผู้ให้บริการ (Provider API credentials) ที่มีสิทธิ์ในการสร้างกล่องจดหมาย, เปลี่ยนรหัสผ่าน, และออกลิงก์เข้าถึงเว็บเมลหากมีการรองรับ

## การติดตั้ง (Installation)

1. อัปโหลดไฟล์ปลั๊กอินไปยัง `/wp-content/plugins/ultimate-multisite-emails`
2. เปิดใช้งาน Addon ในระดับเครือข่าย (Network activate) จากเมนู **Plugins** ในหน้าผู้ดูแลระบบเครือข่าย WordPress
3. ไปที่ **Ultimate Multisite** > **Settings** และเปิดการตั้งค่าอีเมลของ Addon
4. เปิดใช้งานการเชื่อมต่อผู้ให้บริการอีเมลที่คุณต้องการให้ลูกค้าใช้
5. บันทึกการตั้งค่า จากนั้นทดสอบการสร้างบัญชีด้วยลูกค้าที่ไม่ใช่ระบบจริง (non-production customer) หรือบนเว็บไซต์ Staging

## การจัดการอีเมลบัญชีลูกค้า (Customer Account Email Management)

เวอร์ชัน 1.1.0 ได้เพิ่มวิดเจ็ตจัดการอีเมลลงในหน้าบัญชีลูกค้า ใช้ฟังก์ชันนี้เมื่อลูกค้าควรจัดการการสร้างและบำรุงรักษากล่องจดหมายสำหรับเว็บไซต์ของตนเองด้วยตนเอง

จากหน้าบัญชี ลูกค้าสามารถ:

1. ดูบัญชีอีเมลที่เชื่อมโยงกับสมาชิกภาพหรือเว็บไซต์ปัจจุบันของตน
2. สร้างกล่องจดหมายใหม่ เมื่อแผนของพวกเขาสามารถรองรับบัญชีเพิ่มเติมได้
3. เปลี่ยนรหัสผ่านของกล่องจดหมายที่มีอยู่
4. เข้าถึงผู้ให้บริการเว็บเมลที่รองรับผ่าน SSO หรือการเข้าสู่ระบบด้วย magic-link

วิดเจ็ตนี้ใช้ระบบบัญชีหน้าของ Ultimate Multisite ดังนั้นการดำเนินการทั้งหมดจึงจำกัดขอบเขตอยู่เฉพาะบริบทของบัญชีที่ลูกค้ากำลังดูอยู่เท่านั้น ซึ่งช่วยป้องกันไม่ให้ลูกค้าจัดการบัญชีอีเมลที่เป็นของสมาชิกภาพหรือเว็บไซต์อื่น

## Webmail SSO และการเข้าสู่ระบบด้วย Magic-Link

Webmail SSO ให้ลูกค้าสามารถดำเนินการ **Open webmail** ได้โดยตรงจากหน้าบัญชี เมื่อผู้ให้บริการที่ใช้งานรองรับ Addon จะร้องขอ URL การเข้าถึงที่มีอายุสั้น และเปลี่ยนเส้นทางลูกค้าเข้าสู่เว็บเมลโดยไม่ต้องให้ลูกค้าคัดลอกข้อมูลรับรองกล่องจดหมาย

พฤติกรรมของผู้ให้บริการอาจแตกต่างกันไป:

* ผู้ให้บริการที่มี API SSO แบบ Native สามารถสร้างเซสชันเว็บเมลที่ได้รับการยืนยันตัวตนได้โดยตรง
* ผู้ให้บริการที่รองรับ magic-link สามารถสร้างลิงก์เข้าสู่ระบบที่มีระยะเวลาจำกัดได้
* ผู้ให้บริการที่ไม่มีการรองรับ SSO หรือ magic-link ควรยังคงแสดงรายละเอียดกล่องจดหมายและการดำเนินการจัดการรหัสผ่าน แต่จะไม่มีปุ่มเว็บเมลแบบคลิกเดียว

ให้เปิดใช้งานการเข้าถึงเว็บเมลเฉพาะสำหรับผู้ให้บริการที่คุณได้กำหนดค่าข้อมูลรับรอง API ที่จำเป็นแล้วเท่านั้น หากไม่มีการดำเนินการสำหรับผู้ให้บริการรายใดรายหนึ่ง ให้ตรวจสอบก่อนว่าผู้ให้บริการนั้นเปิดใช้งานและตั้งค่าเรียบร้อยแล้ว และรองรับลิงก์เข้าสู่ระบบเว็บเมลหรือไม่

## การจัดเตรียมเมื่อชำระเงิน (Checkout Provisioning)

Addon สามารถสร้างบัญชีอีเมลของลูกค้าได้เป็นส่วนหนึ่งของขั้นตอนการชำระเงิน ซึ่งมีประโยชน์เมื่อแผนบริการรวมกล่องจดหมายมาให้โดยค่าเริ่มต้น หรือเมื่อแบบฟอร์มการชำระเงินมีการเก็บรายละเอียดบัญชีอีเมล

ขั้นตอนที่แนะนำ:

1. กำหนดค่าผู้ให้บริการและขีดจำกัดของแผนก่อนที่จะเปิดใช้งานการจัดเตรียมเมื่อชำระเงิน
2. ทดสอบการซื้อด้วยผลิตภัณฑ์ Staging และยืนยันว่ากล่องจดหมายถูกสร้างขึ้นหลังจากการชำระเงิน
3. ตรวจสอบหน้าบัญชีลูกค้าเพื่อยืนยันว่ากล่องจดหมายใหม่ปรากฏในวิดเจ็ตอีเมล
4. ยืนยันการดำเนินการเปลี่ยนรหัสผ่านและเว็บเมลก่อนที่จะเปิดใช้งานกระบวนการนี้สำหรับผลิตภัณฑ์จริง

หากการจัดเตรียมล้มเหลว บัญชีลูกค้าและการเป็นสมาชิกยังคงอยู่ได้ แต่กล่องจดหมายจำเป็นต้องได้รับการติดตามจากผู้ดูแลระบบ ให้ตรวจสอบบันทึกกิจกรรมของ Ultimate Multisite และบันทึกของผู้ให้บริการก่อนที่จะลองอีกครั้ง

## CyberPanel Email Forwarders

สำหรับเครือข่ายที่ใช้ CyberPanel เวอร์ชัน 1.1.0 ได้เพิ่มการรองรับอีเมล forwarder ใช้ forwarder เมื่อลูกค้าต้องการที่อยู่อีเมล เช่น `info@customer-domain.test` เพื่อเปลี่ยนเส้นทางอีเมลไปยังกล่องจดหมายอื่นโดยไม่จำเป็นต้องสร้างกล่องจดหมายเต็มรูปแบบ

ก่อนที่จะเสนอ forwarder ให้ยืนยันว่าการเชื่อมต่อ CyberPanel ได้รับการตั้งค่าและสามารถเข้าถึงได้ ดูคู่มือ [CyberPanel Integration](../../user-guide/administration/cyberpanel-integration) สำหรับค่าคงที่ของโฮสต์, ชื่อผู้ใช้, รหัสผ่าน, และแพ็กเกจที่จำเป็น

## คำถามที่พบบ่อย (Frequently Asked Questions)

## ลูกค้าสามารถสร้างบัญชีอีเมลของตัวเองได้หรือไม่?

ได้ค่ะ เมื่อเปิดใช้งานวิดเจ็ตบัญชีลูกค้าและแผนบริการอนุญาตให้มีบัญชีอีเมล ลูกค้าสามารถสร้างและจัดการกล่องจดหมายของตนเองได้จากหน้าบัญชี

## ผู้ให้บริการทุกรายรองรับ Webmail SSO หรือไม่?

ไม่ค่ะ การรองรับ SSO และ magic-link ขึ้นอยู่กับผู้ให้บริการอีเมลที่ใช้งาน ผู้ให้บริการที่ไม่มี API สำหรับลิงก์เข้าถึงยังคงสามารถรองรับการสร้างกล่องจดหมายและการเปลี่ยนรหัสผ่านได้

## ผู้ดูแลระบบสามารถสร้างบัญชีอีเมลให้ลูกค้าได้หรือไม่?

ได้ค่ะ ผู้ดูแลระบบสามารถสร้างบัญชีอีเมลจากหน้าผู้ดูแลระบบ รวมถึงการตั้งรหัสผ่านเริ่มต้นเมื่อสร้างบัญชี

## Forwarder กับ Mailbox เหมือนกันหรือไม่?

ไม่ค่ะ Mailbox จะเก็บอีเมลและมีข้อมูลรับรองในการเข้าสู่ระบบ ส่วน Forwarder จะเปลี่ยนเส้นทางอีเมลไปยังที่อยู่อื่นและไม่ได้ให้กล่องจดหมายแยกต่างหาก
