# บันทึกการเปลี่ยนแปลง (Changelog) ของระบบ Support Tickets

### 1.0.4 - 2026-05-05
* ปรับปรุง: ลบโฟลเดอร์ vendor/ ออกจากการติดตามของ Git (ซึ่ง .gitignore จัดการอยู่แล้ว) เพื่อลดขนาดของ repository
* ปรับปรุง: ทดสอบการใช้งานได้ถึง WordPress 7.0

### 1.0.3 - 2026-05-01
* แก้ไข: ยอมรับค่า null ในตัวตั้งค่า (setters) ของโมเดลตั๋วที่อนุญาตให้เป็นค่าว่าง (nullable)
* แก้ไข: จัดเก็บตั๋วทั้งหมดในตารางระดับเครือข่าย (global network-wide tables) เพื่อรองรับระบบ multisite อย่างถูกต้อง
* แก้ไข: ซ่อนฟิลด์สำหรับเจ้าหน้าที่ และแก้ไขปัญหาการเปลี่ยนเส้นทางไปยังหน้าว่างเปล่าในฟอร์มสร้างตั๋วใหม่ของลูกค้า
* แก้ไข: แทนที่การเรียกใช้ add_meta() ที่ไม่ได้กำหนดค่า ด้วย update_meta() ที่ถูกต้องสำหรับการจัดเก็บ metadata
* แก้ไข: แทนที่การตรวจสอบ capability ที่ไม่ได้ลงทะเบียน ด้วย wu_view_all_support_tickets ที่ถูกต้อง
* แก้ไข: เพิ่ม AJAX handlers ที่ขาดหายไปสำหรับสถานะตั๋ว การมอบหมายงาน และการแก้ไขด่วน (quick-edit)
* แก้ไข: แก้ไขชื่อเมธอดสำหรับการตรวจจับการตอบกลับของเจ้าหน้าที่ในหน้าดูตั๋ว
* แก้ไข: รวมตัวจัดการการตอบกลับที่ซ้ำซ้อน และจัดแนวชื่อ action ของ nonce
* แก้ไข: เพิ่มหน้าแสดงผลส่วนหน้า (frontend view) ที่ขาดหายไปสำหรับ shortcode [wu_submit_ticket]
* แก้ไข: เพิ่มคอลัมน์ user_id ที่ขาดหายไป และแก้ไขเมธอด Support_Ticket::get_user_id()
* แก้ไข: ลบเครื่องหมายอัญประกาศคู่ที่เกินมาในแท็กตัวเลือกตัวกรองความสำคัญ (priority filter option tag)
* แก้ไข: เพิ่มแผงการจัดการตั๋วสำหรับผู้ดูแลระบบเครือข่าย (network admin) สำหรับ super admins
* ปรับปรุง: รวม CSS ของแอดมินให้อยู่ในไฟล์สไตล์ชีตภายนอกเพียงไฟล์เดียว
* ปรับปรุง: ลบเมนูย่อย Settings ออกจากเมนูแอดมินของ subsite
* ปรับปรุง: โหลด assets ส่วนหน้าแบบมีเงื่อนไขเฉพาะในหน้าตั๋วสนับสนุนเท่านั้น
* ปรับปรุง: ข้ามการทำงานของ plugin autoloader เมื่อ Bedrock root autoloader โหลด dependencies ไปแล้ว

### 1.0.2 - 2025-12-11
* เพิ่ม: ฟังก์ชันการดูตั๋วแบบเต็มรูปแบบสำหรับแอดมินและส่วนหน้า
* เพิ่ม: AJAX handler สำหรับการส่งคำตอบกลับตั๋ว
* เพิ่ม: รองรับฟังก์ชันการตอบกลับตั๋วพร้อมการจัดการฟอร์มที่เหมาะสม
* เพิ่ม: การแสดงข้อความแจ้งเตือน (notice) ที่เหมาะสมสำหรับการส่งตั๋วและการตอบกลับในส่วนแอดมิน
* เพิ่ม: การเชื่อมโยงตั๋วโดยอัตโนมัติกับผู้ใช้ปัจจุบันสำหรับการส่งตั๋วด้วยตัวเองของลูกค้า
* เพิ่ม: การเสริมความปลอดภัยเพื่อป้องกันไม่ให้ลูกค้าสามารถเปลี่ยนแปลงความเป็นเจ้าของตั๋วได้
* เพิ่ม: ฟังก์ชันตัวช่วย (helper functions) ที่ขาดหายไป (เช่น wu_format_date, wu_user_can_view_ticket, ฯลฯ)
* เพิ่ม: การดาวน์โหลดและการจัดการไฟล์แนบที่เหมาะสม
* เพิ่ม: ระบบแจ้งเตือนทางอีเมลสำหรับการตอบกลับและสถานะที่เปลี่ยนแปลงของตั๋ว
* แก้ไข: ฟอร์มตอบกลับตั๋วตอนนี้รวม ticket ID ที่จำเป็นแล้ว
* แก้ไข: ปัญหาการมองเห็นข้อความแจ้งเตือนในหน้าดูตั๋วของแอดมิน
* แก้ไข: ข้อผิดพลาดทางไวยากรณ์ในฟังก์ชันตั๋วสนับสนุน
* ปรับปรุง: โครงสร้างฐานข้อมูลด้วยคลาส query และ schema ที่เหมาะสมสำหรับคำตอบและการแนบไฟล์
* ปรับปรุง: การย้ายไปใช้คุณสมบัติ (properties) จริงแทนการใช้ attributes สำหรับโมเดลตั๋ว
