# ผลิตภัณฑ์โดเมนและราคา

ผลิตภัณฑ์โดเมนคือส่วนที่คุณใช้ควบคุมเรื่องราคา, TLDs, WHOIS privacy, และการเลือกผู้ให้บริการ (provider) ผลิตภัณฑ์โดเมนแต่ละรายการเป็นผลิตภัณฑ์มาตรฐานของ Ultimate Multisite ที่มีแท็บ **Domain Settings** เพิ่มเติม

## การสร้างผลิตภัณฑ์โดเมน

1. ไปที่ **Network Admin › Ultimate Multisite › Products**
2. คลิก **Add New**
3. ตั้งค่าประเภทผลิตภัณฑ์เป็น **Domain**
4. กำหนดค่าในแท็บ **Domain Settings** (ดูด้านล่าง)
5. บันทึก

ผลิตภัณฑ์โดเมนจะปรากฏพร้อมป้าย **Domain** สีม่วงในรายการผลิตภัณฑ์ และสามารถกรองได้โดยใช้แท็บ **Domain Products**

## แท็บการตั้งค่าโดเมน (Domain settings tab)

### ผู้ให้บริการ (Provider)

เลือกผู้จดทะเบียน (registrar) ที่จะจัดการการจดทะเบียนสำหรับผลิตภัณฑ์นี้ โดยค่าเริ่มต้นจะใช้ **Default provider** ทั่วโลกที่ตั้งค่าไว้ใน Domain Seller settings

### TLD ที่รองรับ (Supported TLDs)

ปล่อยว่างไว้เพื่อสร้าง **ผลิตภัณฑ์ที่ครอบคลุมทุก TLD (catch-all product)** ซึ่งจะใช้ได้กับ TLDs ทั้งหมดที่ไม่มีผลิตภัณฑ์อื่นจับคู่

ป้อนรายการ TLDs ที่คั่นด้วยเครื่องหมายจุลภาค (เช่น `.com, .net, .org`) เพื่อสร้าง **ผลิตภัณฑ์ที่เฉพาะเจาะจง TLD (TLD-specific product)** ซึ่งจะใช้ได้กับส่วนขยายเหล่านั้นเท่านั้น

**วิธีการจับคู่ผลิตภัณฑ์:** เมื่อลูกค้าค้นหาโดเมน ระบบเสริม (addon) จะเลือกผลิตภัณฑ์ที่จับคู่ได้เฉพาะเจาะจงที่สุด ผลิตภัณฑ์ที่มี `.com` ในรายการ TLDs จะมีความสำคัญกว่าผลิตภัณฑ์ที่ครอบคลุมทุก TLD หากไม่มีผลิตภัณฑ์เฉพาะ TLD ใดที่ตรงกัน ระบบจะใช้ผลิตภัณฑ์ที่ครอบคลุมทุก TLD หากไม่มีผลิตภัณฑ์ใดเลย การค้นหาโดเมนจะไม่แสดงผล

### ประเภทการบวกราคา (Markup type)

มี 3 โหมดที่ใช้ควบคุมวิธีการคำนวณราคาส่ง (retail price) จากต้นทุนราคาส่ง (wholesale cost):

| โหมด | วิธีการทำงาน |
|---|---|
| **Percentage** | เพิ่มเปอร์เซ็นต์จากต้นทุนราคาส่ง การบวกกำไร 20% สำหรับโดเมนราคาส่ง $10 จะได้ราคา $12 |
| **Fixed markup** | เพิ่มจำนวนเงินคงที่ การบวกกำไร $5 สำหรับโดเมนราคา $10 จะได้ราคา $15 |
| **Fixed price** | ไม่สนใจต้นทุนราคาส่งเลย จะเรียกเก็บตามจำนวนเงินที่คุณป้อนเสมอ |

### ราคาแนะนำ (Introductory pricing)

เปิดใช้งานเพื่อเสนอราคาปีแรกที่ลดพิเศษ กำหนด **Introductory price** (ราคาปีที่ 1) แยกต่างหากจาก **Renewal price** (ราคาปี 2 เป็นต้นไป) ลูกค้าจะเห็นราคาทั้งสองแบบนี้ในหน้าชำระเงิน เพื่อให้ทราบว่าเมื่อถึงรอบต่ออายุจะต้องจ่ายเท่าไหร่

### WHOIS privacy

ควบคุมว่ามีการเสนอการป้องกัน WHOIS privacy สำหรับโดเมนที่จดทะเบียนผ่านผลิตภัณฑ์นี้หรือไม่

| การตั้งค่า | พฤติกรรม |
|---|---|
| **Disabled** | จะไม่มีการเสนอหรือเปิดใช้งาน WHOIS privacy เลย |
| **Always Included** | WHOIS privacy จะถูกเปิดใช้งานโดยอัตโนมัติเมื่อจดทะเบียนโดยไม่มีค่าใช้จ่าย |
| **Customer Choice** | จะมีช่องทำเครื่องหมายปรากฏในหน้าชำระเงิน กำหนด **Privacy price** เพื่อเรียกเก็บเงินต่อปี หรือปล่อยไว้ที่ $0 เพื่อเสนอให้ฟรี |

สำหรับ Namecheap, WHOIS privacy ใช้ WhoisGuard (ฟรีเสมอ) สำหรับ OpenSRS, ใช้บริการ OpenSRS privacy (อาจมีค่าใช้จ่ายในราคาส่ง)

## การนำเข้าและซิงค์ TLD

ผลิตภัณฑ์โดเมนจะแสดงราคาส่งแบบเรียลไทม์ที่ดึงมาจากผู้ให้บริการที่เชื่อมต่อ เพื่อให้สิ่งนี้ทำงานได้ TLDs จะต้องถูกนำเข้า (imported)

- **Manual sync:** Settings › Domain Seller › Sync TLDs (ต่อผู้ให้บริการ)
- **Automatic sync:** ทำงานทุกวันผ่าน cron job ที่ตั้งเวลาไว้สำหรับผู้ให้บริการที่กำหนดค่าไว้ทั้งหมด

หลังจากซิงค์แล้ว ให้ไปที่แท็บ Domain Settings ของผลิตภัณฑ์โดเมนใดก็ได้ และใช้ TLD picker เพื่อดู TLDs ที่พร้อมใช้งานพร้อมราคาส่งปัจจุบัน

## การต่ออายุอัตโนมัติ (Auto-renewal)

การต่ออายุโดเมนจะผูกติดกับสถานะสมาชิกของลูกค้า:

- เมื่อมีการต่ออายุสมาชิกและมีการเชื่อมโยงโดเมน การต่ออายุโดเมนจะถูกจัดคิวโดยอัตโนมัติ
- การพยายามต่ออายุจะใช้ payment gateway ที่ใช้งานอยู่ของลูกค้า
- การต่ออายุที่ล้มเหลวจะถูกลองใหม่โดยอัตโนมัติด้วยการหน่วงเวลาแบบ exponential backoff
- จะมีการส่งอีเมลแจ้งเตือนสำหรับการต่ออายุที่สำเร็จ, ความล้มเหลว, และการหมดอายุที่กำลังจะมาถึง

รหัสแม่แบบอีเมลสำหรับเหตุการณ์วงจรชีวิตโดเมน:

| เหตุการณ์ | Template ID |
|---|---|
| Domain registered | `domain_registered` |
| Domain renewed | `domain_renewed` |
| Renewal failed | `domain_renewal_failed` |
| Domain expiring soon | `domain_expiring_soon` |

## สำหรับผู้ดูแลระบบ: การจดทะเบียนโดเมนด้วยตนเอง

หากต้องการจดทะเบียนโดเมนในนามของลูกค้าโดยที่ลูกค้าไม่ต้องผ่านหน้าชำระเงิน:

1. ไปที่ **Network Admin › Ultimate Multisite › Register Domain**
2. เลือกลูกค้าและป้อนชื่อโดเมน
3. กรอกรายละเอียดผู้จดทะเบียน (ชื่อ, ที่อยู่, โทรศัพท์)
4. คลิก **Register**

ระบบจะสร้างบันทึกโดเมนและเชื่อมโยงกับบัญชีของลูกค้า
