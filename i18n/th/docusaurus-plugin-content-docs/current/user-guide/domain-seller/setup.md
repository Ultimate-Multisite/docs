# Domain Seller: การตั้งค่าและการกำหนดค่าผู้ให้บริการ {#domain-seller-setup-and-provider-configuration}

Addon Domain Seller มาพร้อมกับตัวช่วยตั้งค่าแบบมีคำแนะนำที่จะพาคุณผ่านทุกขั้นตอนที่จำเป็น หน้านี้ครอบคลุมลำดับการทำงานของตัวช่วยตั้งค่าและวิธีกำหนดค่าหรือกำหนดค่าผู้ให้บริการใหม่ในภายหลัง

## ข้อกำหนด {#requirements}

- **Multisite Ultimate** v2.4.12 หรือสูงกว่า เปิดใช้งานในเครือข่าย
- **PHP** 7.4+
- ข้อมูลรับรอง API สำหรับ registrar ที่รองรับอย่างน้อยหนึ่งราย

## ตัวช่วยตั้งค่าเมื่อใช้งานครั้งแรก {#first-run-setup-wizard}

ตัวช่วยตั้งค่าจะเริ่มทำงานโดยอัตโนมัติครั้งแรกที่คุณเปิดใช้งาน plugin ในเครือข่าย และยังสามารถเข้าถึงได้ทุกเมื่อจาก **Network Admin › Ultimate Multisite › Domain Seller Setup**

### ขั้นตอนที่ 1 — เลือกผู้ให้บริการ {#step-1--choose-a-provider}

เลือก registrar ที่คุณต้องการเชื่อมต่อ ตัวเลือกที่รองรับ:

| ผู้ให้บริการ | การจัดการ DNS | ความเป็นส่วนตัว WHOIS |
|---|---|---|
| OpenSRS | ใช่ | ใช่ |
| Namecheap | ไม่ | ใช่ (WhoisGuard, ฟรี) |
| HostAfrica | ใช่ | ใช่ (การป้องกัน ID) |
| Openprovider | ใช่ | ใช่ |
| Hostinger | ผ่านการแมปโดเมน Hostinger หลักสำหรับโดเมนที่โฮสต์ | ใช่ |
| GoDaddy | ไม่ | ไม่ |
| ResellerClub | ใช่ | ไม่ |
| NameSilo | ไม่ | ไม่ |
| Enom | ใช่ | ไม่ |

### ขั้นตอนที่ 2 — ป้อนข้อมูลรับรอง {#step-2--enter-credentials}

ผู้ให้บริการแต่ละรายมีช่องข้อมูลรับรองที่แตกต่างกัน:

**OpenSRS** — ชื่อผู้ใช้และคีย์ส่วนตัว (จาก OpenSRS Reseller Control Panel)

**Namecheap** — ชื่อผู้ใช้และคีย์ API (จาก Account › Tools › API Access)

**HostAfrica** — endpoint API ของ Domains Reseller และข้อมูลรับรองจากโมดูลตัวแทนจำหน่าย HostAfrica ขณะนี้ยังไม่มีการบันทึก endpoint sandbox แยกต่างหาก; ทดสอบด้วยการตรวจสอบแบบอ่านอย่างเดียวที่ปลอดภัยก่อนดำเนินการจดทะเบียนจริง

**Openprovider** — ชื่อผู้ใช้และรหัสผ่านที่เปิดใช้งานการเข้าถึง API โหมด sandbox ที่เป็นตัวเลือกใช้ Openprovider sandbox API และสามารถใช้ handle ลูกค้าเริ่มต้นที่เป็นตัวเลือกซ้ำสำหรับการจดทะเบียนได้

**Hostinger** — โทเค็น Hostinger hPanel API ที่ใช้ร่วมกันจากการผสานรวม Hostinger หลัก โทเค็นเดียวกันนี้ขับเคลื่อนการแมปโดเมนหลักและการดำเนินการจดทะเบียนของ Domain Seller

**GoDaddy** — คีย์ API และ secret (จาก developer.godaddy.com)

**ResellerClub** — ID ตัวแทนจำหน่ายและคีย์ API (จากแผงควบคุม ResellerClub)

**NameSilo** — คีย์ API (จาก namesilo.com › Account › API Manager)

**Enom** — Account ID และโทเค็น API

ทำเครื่องหมาย **โหมด Sandbox** เมื่อมีให้ใช้งาน เพื่อทดสอบกับสภาพแวดล้อมทดสอบของผู้ให้บริการก่อนเปิดใช้งานจริง

### ขั้นตอนที่ 3 — ทดสอบการเชื่อมต่อ {#step-3--test-the-connection}

คลิก **ทดสอบการเชื่อมต่อ** ตัวช่วยตั้งค่าจะส่งการเรียก API แบบเบาเพื่อตรวจสอบข้อมูลรับรองและการเชื่อมต่อ แก้ไขปัญหาข้อมูลรับรองใด ๆ ก่อนดำเนินการต่อ

### ขั้นตอนที่ 4 — นำเข้า TLD {#step-4--import-tlds}

คลิก **นำเข้า TLD** เพื่อดึง TLD ทั้งหมดที่พร้อมใช้งานและราคาขายส่งจากผู้ให้บริการที่เชื่อมต่ออยู่ การดำเนินการนี้จะเติมรายการ TLD ที่ใช้โดย product โดเมน การนำเข้าอาจใช้เวลา 30–60 วินาทีสำหรับผู้ให้บริการที่มีแค็ตตาล็อก TLD ขนาดใหญ่

TLD ยังถูกซิงค์ใหม่โดยอัตโนมัติวันละครั้งผ่านงาน cron ตามกำหนดเวลา

### ขั้นตอนที่ 5 — สร้าง product โดเมน {#step-5--create-a-domain-product}

ตัวช่วยตั้งค่าจะสร้าง product โดเมนแบบครอบคลุมทั้งหมดเริ่มต้นพร้อมส่วนเพิ่มราคา 10% คุณสามารถแก้ไข product นี้ได้ทันที หรือข้ามและสร้าง product ด้วยตนเองภายใต้ **Ultimate Multisite › Products**

ดู [Product โดเมนและการกำหนดราคา](./domain-products) สำหรับคู่มือการกำหนดค่า product ฉบับเต็ม

## การกำหนดค่าผู้ให้บริการใหม่ {#reconfiguring-a-provider}

ไปที่ **Network Admin › Ultimate Multisite › Settings › Domain Seller** (หรือคลิก **Settings** ในรายการ plugin)

หน้าการตั้งค่าประกอบด้วย:

- **เปิดใช้งานการขายโดเมน** — เปิด/ปิดฟีเจอร์ทั้งหมด
- **ผู้ให้บริการเริ่มต้น** — ผู้ให้บริการที่ใช้สำหรับการค้นหาโดเมนและ product ใหม่
- **จำนวน TLD สูงสุดต่อการค้นหา** — จำนวน TLD ที่จะตรวจสอบเมื่อลูกค้าค้นหา; ค่าที่สูงกว่าจะแสดงตัวเลือกมากขึ้นแต่ช้ากว่า
- **ระยะเวลาแคชความพร้อมใช้งาน** — ระยะเวลาที่จะแคชผลลัพธ์ความพร้อมใช้งานและราคา; ค่าที่ต่ำกว่าจะถูกต้องแม่นยำกว่าแต่เพิ่มการเรียก API
- **จัดการ product โดเมน** — ลิงก์ด่วนไปยังรายการ Products
- **กำหนดค่าผู้ให้บริการ** — เปิด Integration Wizard เพื่อเพิ่มหรือกำหนดค่าผู้ให้บริการใหม่

### การเพิ่มผู้ให้บริการรายที่สอง {#adding-a-second-provider}

คลิก **กำหนดค่าผู้ให้บริการ** และเรียกใช้ตัวช่วยตั้งค่าอีกครั้งสำหรับ registrar ใหม่ คุณสามารถกำหนดค่าผู้ให้บริการหลายรายพร้อมกันได้ กำหนด product โดเมนแต่ละรายการให้กับผู้ให้บริการเฉพาะ หรือปล่อยไว้เป็นค่าเริ่มต้น

### การซิงค์ TLD ด้วยตนเอง {#syncing-tlds-manually}

ในหน้าการตั้งค่า คลิก **ซิงค์ TLD** ถัดจากผู้ให้บริการที่กำหนดค่าไว้เพื่อดึงราคาล่าสุด สิ่งนี้มีประโยชน์หลังจากผู้ให้บริการอัปเดตราคาขายส่งหรือเพิ่ม TLD ใหม่

|---|---|
| `domain-seller-registration` | ความพยายามจดทะเบียนทั้งหมด (สำเร็จและล้มเหลว) |
| `domain-seller-renewal` | ผลลัพธ์งานต่ออายุ |
| `domain-seller-opensrs` | กิจกรรม API ดิบของ OpenSRS |
| `domain-seller-namecheap` | กิจกรรม API ดิบของ Namecheap |
| `domain-seller-hostafrica` | กิจกรรม API ดิบของ HostAfrica |
| `domain-seller-openprovider` | กิจกรรม API ดิบของ Openprovider |
| `domain-seller-hostinger` | กิจกรรม API ดิบของ Hostinger |
| `domain-seller-godaddy` | กิจกรรม API ดิบของ GoDaddy |
| `domain-seller-resellerclub` | กิจกรรม API ดิบของ ResellerClub |
| `domain-seller-namesilo` | กิจกรรม API ดิบของ NameSilo |
| `domain-seller-enom` | กิจกรรม API ดิบของ Enom |

## หมายเหตุเกี่ยวกับความสามารถของผู้ให้บริการ {#provider-capability-notes}

ไม่ใช่ API ของ registrar ทุกแห่งที่จะเปิดเผยการดำเนินการเดียวกัน Addon จะแสดงการดำเนินการที่ไม่รองรับด้วยข้อผิดพลาดที่ชัดเจนสำหรับผู้ดูแล แทนที่จะล้มเหลวโดยเงียบ ๆ

- **HostAfrica** รองรับเวิร์กโฟลว์ตัวแทนจำหน่ายแบบใช้งานจริงที่ครอบคลุมที่สุด รวมถึงการค้นหา, การซิงก์ TLD/ราคา, การจดทะเบียน, การต่ออายุ, การโอนย้าย, การอัปเดต nameserver, ระเบียน DNS, โค้ด EPP, registrar lock และการปกป้อง ID
- **Openprovider** รองรับการซิงก์ TLD ตามราคาตัวแทนจำหน่าย, การจดทะเบียน, การต่ออายุ, การโอนย้าย, การอัปเดต nameserver, โซน DNS, โค้ด EPP, registrar lock และความเป็นส่วนตัวของ WHOIS โดยยืนยันตัวตนด้วย bearer token อายุสั้นที่ส่วนเสริมจะรีเฟรชให้อัตโนมัติ
- **Hostinger** รองรับการค้นหาความพร้อมใช้งาน, การจดทะเบียน, การค้นหาพอร์ตโฟลิโอ, การอัปเดต nameserver, registrar lock และความเป็นส่วนตัวของ WHOIS ผ่าน hPanel API token ที่ใช้ร่วมกัน Domains API สาธารณะของ Hostinger ไม่เปิดเผยราคาตัวแทนจำหน่าย/ขายส่ง, การโอนย้ายขาเข้า, การต่ออายุแบบระบุชัดเจน หรือการดึงโค้ด EPP; การต่ออายุจะเป็นแบบต่ออายุอัตโนมัติเท่านั้น
