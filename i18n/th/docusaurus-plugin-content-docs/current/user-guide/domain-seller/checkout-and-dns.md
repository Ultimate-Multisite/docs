# ฟิลด์ Checkout และการจัดการ DNS ของลูกค้า

## ฟิลด์ Checkout การเลือกโดเมน

ฟิลด์ **Domain Selection** เป็นองค์ประกอบ Checkout ที่ให้ลูกค้าเลือกวิธีรับโดเมนของ site ของตน เพิ่มฟิลด์นี้ลงในแบบฟอร์ม Checkout ใดก็ได้เพื่อเปิดใช้งานการขายโดเมน

### การเพิ่มฟิลด์ลงในแบบฟอร์ม Checkout

1. ไปที่ **Network Admin › Ultimate Multisite › Checkout Forms**
2. เปิดหรือสร้างแบบฟอร์ม Checkout
3. ในตัวแก้ไข Checkout ให้คลิก **Add Field**
4. เลือก **Domain Selection** จากรายการฟิลด์
5. กำหนดค่าตัวเลือกของฟิลด์ (ดูด้านล่าง)
6. บันทึกแบบฟอร์ม

### ตัวเลือกฟิลด์

**โหมดโดเมน** — เลือกแท็บที่ลูกค้าจะเห็น แต่ละโหมดสามารถเปิดหรือปิดใช้งานได้อย่างอิสระ:

| โหมด | สิ่งที่ทำ |
|---|---|
| **Subdomain** | ลูกค้าใช้ subdomain ฟรีบนเครือข่ายของคุณ (เช่น `mysite.yournetwork.com`) ไม่ต้องชำระเงิน |
| **Register New Domain** | ลูกค้าค้นหาโดเมนใหม่และจดทะเบียนผ่านผู้ให้บริการที่คุณกำหนดค่าไว้ ใช้ product โดเมนที่ตรงกันสำหรับการกำหนดราคา |
| **Existing Domain** | ลูกค้าแมปโดเมนที่ตนมีอยู่แล้ว ไม่มีค่าธรรมเนียมการจดทะเบียน โดเมนจะถูกแมปไปยัง site ของพวกเขาโดยอัตโนมัติ |

**โหมดเริ่มต้น** — เมื่อเปิดใช้งานทั้งสามโหมด แท็บใดจะเปิดขึ้นก่อน ตั้งเป็น **Subdomain** เพื่อให้การจดทะเบียนโดเมนเป็นตัวเลือกเสริม หรือ **Register New Domain** เพื่อกระตุ้นการซื้อ

**Product โดเมน** — เลือกผูกฟิลด์นี้กับ product โดเมนเฉพาะได้ หากไม่ได้ตั้งค่า addon จะเลือก product ที่ตรงกันโดยอัตโนมัติตาม TLD ที่ลูกค้าค้นหา

### ฟิลด์ข้อมูลติดต่อผู้จดทะเบียน

เมื่อลูกค้าเลือกแท็บ **Register New Domain** แบบฟอร์ม Checkout จะเพิ่มฟิลด์ข้อมูลติดต่อผู้จดทะเบียนไว้ในบรรทัดเดียวกัน:

- ชื่อ / นามสกุล
- อีเมล
- ที่อยู่ (บรรทัดที่ 1, เมือง, รัฐ/จังหวัด, รหัสไปรษณีย์, ประเทศ)
- หมายเลขโทรศัพท์

ฟิลด์เหล่านี้เป็นสิ่งที่ผู้รับจดทะเบียนทุกรายต้องการ และจะถูกตรวจสอบความถูกต้องก่อนเรียกใช้ registration API หมายเลขโทรศัพท์จะถูกจัดรูปแบบอัตโนมัติเป็นรูปแบบสากล `+CC.NNN` ที่ผู้รับจดทะเบียนคาดหวัง

### URL ของ site ที่สร้างอัตโนมัติ

เมื่อลูกค้าจดทะเบียนหรือแมปโดเมน ฟิลด์ URL ของ site จะถูกกรอกโดยอัตโนมัติจากโดเมนที่เลือก ลูกค้าไม่จำเป็นต้องกรอกฟิลด์ URL แยกต่างหาก

### พฤติกรรมการค้นหา

- ตรวจสอบความพร้อมใช้งานของโดเมนแบบเรียลไทม์ด้วย AJAX ขณะที่ลูกค้าพิมพ์
- แสดงคำแนะนำ TLD ทางเลือกเมื่อโดเมนที่ต้องการไม่พร้อมใช้งาน
- ดึงราคาสดและแสดงอย่างชัดเจน (ราคาจดทะเบียน, ราคาต่ออายุ, ค่าธรรมเนียม WHOIS privacy แบบเลือกได้)
- รหัสคูปองใช้กับ product โดเมนได้เหมือน product อื่น ๆ

**การปรับแต่งการตอบสนองของการค้นหา:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**การเพิ่มฟิลด์กำหนดเองลงในแบบฟอร์มค้นหาโดเมน:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## การจัดการ DNS ของลูกค้า

ลูกค้าสามารถจัดการระเบียน DNS สำหรับโดเมนที่จดทะเบียนไว้ได้จากหน้า **My Account** ใต้รายการของโดเมนนั้น

### ประเภทระเบียนที่รองรับ

| ประเภท | การใช้งาน |
|---|---|
| **A** | แมปชื่อโฮสต์กับที่อยู่ IPv4 |
| **AAAA** | แมปชื่อโฮสต์กับที่อยู่ IPv6 |
| **CNAME** | สร้างนามแฝงที่ชี้ไปยังชื่อโฮสต์อื่น |
| **MX** | ตั้งค่าเซิร์ฟเวอร์แลกเปลี่ยนเมล |
| **TXT** | เพิ่ม SPF, DMARC, การยืนยัน หรือระเบียนข้อความอื่น ๆ |

### ผู้ให้บริการใดรองรับการจัดการ DNS?

การจัดการ DNS (เพิ่ม แก้ไข ลบระเบียน) พร้อมใช้งานกับ **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica** และ **Openprovider** โดเมน **Hostinger** สามารถอัปเดต nameserver ผ่าน Domain Seller ได้; ระเบียน DNS สำหรับโดเมนที่โฮสต์จะได้รับการจัดการโดยการผสานการทำงาน domain-mapping หลักของ Hostinger โดเมน Namecheap, GoDaddy และ NameSilo แสดงสถานะและข้อมูลวันหมดอายุ แต่ต้องจัดการ DNS โดยตรงในแผงควบคุมของผู้รับจดทะเบียน

### ระเบียน DNS เริ่มต้น

คุณสามารถกำหนดค่าระเบียน DNS เริ่มต้นที่จะถูกนำไปใช้โดยอัตโนมัติเมื่อจดทะเบียนโดเมน ไปที่ **Settings › Domain Seller › Default DNS Records**

ค่าระเบียนเริ่มต้นรองรับสองโทเค็น:

| โทเค็น | แทนที่ด้วย |
|---|---|
| `{DOMAIN}` | ชื่อโดเมนที่จดทะเบียน (เช่น `example.com`) |
| `{SITE_URL}` | URL ของ site WordPress สำหรับ site ของลูกค้า |

**ตัวอย่าง — ชี้ apex domain และ www ไปยัง IP เซิร์ฟเวอร์ของคุณ:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ผู้ดูแลระบบ: การดูและแก้ไข DNS

ผู้ดูแลระบบเครือข่ายสามารถดูและแก้ไขระเบียน DNS สำหรับโดเมนของลูกค้ารายใดก็ได้จากหน้าแก้ไขของโดเมนใน **Network Admin › Ultimate Multisite › Domains**.
