# การจัดการฟิลด์ในหน้าชำระเงินและการจัดการ DNS ของลูกค้า

## ฟิลด์เลือกโดเมนในหน้าชำระเงิน

ฟิลด์ **Domain Selection** เป็นองค์ประกอบในหน้าชำระเงินที่ให้ลูกค้าเลือกวิธีการได้โดเมนสำหรับเว็บไซต์ของพวกเขา คุณสามารถเพิ่มฟิลด์นี้ลงในหน้าชำระเงินใดก็ได้เพื่อเปิดใช้งานการขายโดเมน

### วิธีเพิ่มฟิลด์ลงในแบบฟอร์มชำระเงิน

1. ไปที่ **Network Admin › Ultimate Multisite › Checkout Forms**
2. เปิดหรือสร้างแบบฟอร์มชำระเงิน
3. ในตัวแก้ไขหน้าชำระเงิน ให้คลิก **Add Field**
4. เลือก **Domain Selection** จากรายการฟิลด์
5. กำหนดค่าตัวเลือกของฟิลด์ (ดูด้านล่าง)
6. บันทึกแบบฟอร์ม

### ตัวเลือกของฟิลด์

**Domain modes** — เลือกแท็บที่ลูกค้าจะเห็น แต่ละโหมดสามารถเปิดหรือปิดได้โดยอิสระ:

| Mode | สิ่งที่ทำ |
|---|---|
| **Subdomain** | ลูกค้าใช้ซับโดเมนฟรีบนเครือข่ายของคุณ (เช่น `mysite.yournetwork.com`) ไม่ต้องชำระเงิน |
| **Register New Domain** | ลูกค้าค้นหาโดเมนใหม่และลงทะเบียนผ่านผู้ให้บริการที่คุณตั้งค่าไว้ ใช้ผลิตภัณฑ์โดเมนที่ตรงกันสำหรับการกำหนดราคา |
| **Existing Domain** | ลูกค้าเชื่อมโยงโดเมนที่ตนเองเป็นเจ้าของอยู่แล้ว ไม่ต้องเสียค่าลงทะเบียน โดเมนจะถูกแมปไปยังเว็บไซต์ของลูกค้าโดยอัตโนมัติ |

**Default mode** — เมื่อเปิดใช้งานทั้งสามโหมด แท็บใดจะเปิดขึ้นมาก่อน กำหนดเป็น **Subdomain** เพื่อให้การลงทะเบียนโดเมนเป็นทางเลือก หรือกำหนดเป็น **Register New Domain** เพื่อกระตุ้นการซื้อ

**Domain product** — คุณสามารถกำหนดให้ฟิลด์นี้ผูกกับผลิตภัณฑ์โดเมนเฉพาะได้ตามต้องการ หากไม่ได้ตั้งค่า ส่วนเสริมจะเลือกผลิตภัณฑ์ที่ตรงกันโดยอัตโนมัติโดยอิงจาก TLD ที่ลูกค้าค้นหา

### ฟิลด์ข้อมูลติดต่อผู้ลงทะเบียน

เมื่อลูกค้าเลือกแท็บ **Register New Domain** แบบฟอร์มชำระเงินจะเพิ่มฟิลด์ข้อมูลติดต่อผู้ลงทะเบียนแบบอินไลน์:

- ชื่อ / นามสกุล
- ที่อยู่อีเมล
- ที่อยู่ (บรรทัดที่ 1, เมือง, รัฐ/จังหวัด, รหัสไปรษณีย์, ประเทศ)
- หมายเลขโทรศัพท์

ข้อมูลเหล่านี้เป็นสิ่งที่ผู้รับจดทะเบียนทุกคนต้องการ และจะถูกตรวจสอบความถูกต้องก่อนที่จะมีการเรียกใช้ API สำหรับการลงทะเบียน หมายเลขโทรศัพท์จะถูกจัดรูปแบบเป็นรูปแบบสากล `+CC.NNN` โดยอัตโนมัติ ซึ่งเป็นรูปแบบที่ผู้รับจดทะเบียนคาดหวัง

### URL ของเว็บไซต์ที่สร้างโดยอัตโนมัติ

เมื่อลูกค้าลงทะเบียนหรือแมปโดเมน ฟิลด์ URL ของเว็บไซต์จะถูกเติมข้อมูลโดยอัตโนมัติจากโดเมนที่เลือก ลูกค้าจึงไม่จำเป็นต้องกรอกฟิลด์ URL แยกต่างหาก

### พฤติกรรมการค้นหา

- ตรวจสอบความพร้อมใช้งานของโดเมนแบบเรียลไทม์ด้วย AJAX ขณะที่ลูกค้าพิมพ์
- แสดงคำแนะนำ TLD ทางเลือกเมื่อโดเมนที่ต้องการไม่พร้อมใช้งาน
- ดึงราคาแบบสดและแสดงอย่างชัดเจน (ราคาลงทะเบียน, ราคาต่ออายุ, ค่าความเป็นส่วนตัว WHOIS ที่อาจมี)
- คูปองโค้ดใช้ได้กับผลิตภัณฑ์โดเมนเช่นเดียวกับผลิตภัณฑ์อื่น ๆ

**การปรับความเร็วในการตอบสนองของการค้นหา:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**การเพิ่มฟิลด์ที่กำหนดเองลงในแบบฟอร์มค้นหาโดเมน:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## การจัดการ DNS ของลูกค้า

ลูกค้าสามารถจัดการระเบียน DNS สำหรับโดเมนที่ลงทะเบียนได้จากหน้า **My Account** ภายใต้รายการโดเมนของพวกเขา

### ประเภทระเบียนที่รองรับ

| Type | การใช้งาน |
|---|---|
| **A** | แมปชื่อโฮสต์ไปยังที่อยู่ IPv4 |
| **AAAA** | แมปชื่อโฮสต์ไปยังที่อยู่ IPv6 |
| **CNAME** | สร้างชื่อแทน (alias) ที่ชี้ไปยังชื่อโฮสต์อื่น |
| **MX** | กำหนดเซิร์ฟเวอร์แลกเปลี่ยนอีเมล (mail exchange server) |
| **TXT** | เพิ่มระเบียนข้อความ เช่น SPF, DMARC, การยืนยัน หรือระเบียนข้อความอื่น ๆ |

### ผู้ให้บริการรายใดที่รองรับการจัดการ DNS?

การจัดการ DNS (เพิ่ม แก้ไข ลบระเบียน) สามารถทำได้กับ **OpenSRS**, **ResellerClub**, และ **Enom** โดเมนจาก Namecheap, GoDaddy, และ NameSilo จะแสดงสถานะและข้อมูลวันหมดอายุ แต่ DNS จะต้องจัดการโดยตรงในแผงควบคุมของผู้รับจดทะเบียน

### ระเบียน DNS เริ่มต้น

คุณสามารถกำหนดระเบียน DNS เริ่มต้นที่จะถูกนำไปใช้โดยอัตโนมัติเมื่อมีการลงทะเบียนโดเมน ไปที่ **Settings › Domain Seller › Default DNS Records**

ค่าระเบียนเริ่มต้นรองรับโทเค็นสองตัว:

| Token | แทนที่ด้วย |
|---|---|
| `{DOMAIN}` | ชื่อโดเมนที่ลงทะเบียน (เช่น `example.com`) |
| `{SITE_URL}` | URL ของเว็บไซต์ WordPress สำหรับเว็บไซต์ของลูกค้า |

**ตัวอย่าง — ชี้โดเมนหลักและ www ไปยัง IP เซิร์ฟเวอร์ของคุณ:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ผู้ดูแลระบบ: การดูและแก้ไข DNS

ผู้ดูแลระบบเครือข่ายสามารถดูและแก้ไขระเบียน DNS สำหรับโดเมนของลูกค้าใดก็ได้จากหน้าแก้ไขโดเมนใน **Network Admin › Ultimate Multisite › Domains**
