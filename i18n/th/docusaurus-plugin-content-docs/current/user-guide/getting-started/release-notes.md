# บันทึกการเผยแพร่

## เวอร์ชัน 2.12.0 — เผยแพร่เมื่อ 2026-05-15

- ใหม่: เพิ่ม Hostinger (hPanel) เป็นผู้ให้บริการโฮสต์ที่รองรับ พร้อมการผสานรวมการแมปโดเมน
- ใหม่: Site Exporter สามารถจัดการชุดข้อมูลนำเข้าเครือข่าย (network import bundles) ได้แล้ว เพื่อการกู้คืนเว็บไซต์ทั่วทั้งเครือข่ายที่ง่ายขึ้น
- แก้ไข: อีเมลที่ส่งแบบ BCC broadcast จะใช้ header ชื่อ undisclosed-recipients เพื่อป้องกันการเปิดเผยที่อยู่ผู้รับ
- แก้ไข: วันหมดอายุสมาชิกจะไม่เสียหายอีกต่อไปเมื่อบันทึกด้วยค่าที่ไม่ใช่รูปแบบวันที่
- แก้ไข: การอัปเดตสมาชิกของ Stripe จะล้างส่วนลดได้อย่างถูกต้องแล้ว โดยไม่ต้องเรียกใช้ API deleteDiscount ที่เลิกใช้แล้ว
- แก้ไข: การเปลี่ยนเส้นทาง SSO บนไซต์ที่แมปโดเมน จะถูกจำกัดจำนวนครั้งเพื่อป้องกันการวนลูปเปลี่ยนเส้นทางไม่รู้จบ
- แก้ไข: การเลือกรูปภาพใน Setup wizard จะอัปเดต data model พื้นฐานได้อย่างถูกต้อง
- แก้ไข: Site Exporter CLI จะคงการเลือกไซต์เครือข่ายเริ่มต้นที่ถูกต้อง
- ปรับปรุง: ลบ wp-cli ที่มาพร้อมกับแพ็กเกจปลั๊กอินออกแล้ว ทำให้ขนาดของปลั๊กอินเล็กลง

## เวอร์ชัน 2.11.0 — เผยแพร่เมื่อ 2026-05-11

- ใหม่: การส่งออกไซต์ (Site exports) จะรวมไฟล์ `index.php` ที่สามารถบูตได้ด้วยตัวเอง ทำให้สามารถติดตั้ง ZIP บนโฮสต์ใหม่ได้โดยไม่ต้องติดตั้งปลั๊กอินแยก
- ใหม่: การส่งออกเครือข่าย (Network export) ช่วยให้ผู้ดูแลระบบสามารถส่งออกซับไซต์ทั้งหมดได้ในไฟล์เก็บถาวรเดียวจากหน้า Site Export admin page
- ใหม่: การเปิด/ปิด Site Templates จะถูกบังคับใช้ผ่าน fallback chain ทำให้จำกัดความพร้อมใช้งานของเทมเพลตตามขีดจำกัดของแผนได้อย่างถูกต้อง
- ใหม่: ตัวแก้ไขฟอร์ม Checkout จะแจ้งเตือนเมื่อมีการเพิ่มผลิตภัณฑ์โดยที่ไม่ได้กำหนดค่าฟิลด์ที่จำเป็น
- ใหม่: แท็บการตั้งค่า Import/Export จะอธิบายขอบเขตการใช้งานอย่างชัดเจน และลิงก์โดยตรงไปยังเครื่องมือ Site Export

## เวอร์ชัน 2.10.0 — เผยแพร่เมื่อ 2026-05-05

- ใหม่: ระบบตั้งค่าแบบมีไกด์ (guided setup wizard) สำหรับ PayPal เพื่อการป้อนข้อมูล Credentials ด้วยตนเอง พร้อมระบบ OAuth flag gate สำหรับการตั้งค่า Gateway ที่ราบรื่น
- ใหม่: หน้าลูกค้าสำหรับสลับเทมเพลต (Template switch customer panel) ได้รับการออกแบบใหม่ พร้อมการ์ดเทมเพลตปัจจุบัน (current-template card), grid แบบถาวร, และปุ่ม **Reset current template**
- แก้ไข: การสลับเทมเพลตจะไม่ทำให้ UI ค้างเมื่อเกิดข้อผิดพลาด AJAX
- แก้ไข: สถานะสิทธิ์ในการสลับเทมเพลตได้รับการรักษาความปลอดภัยจากการเข้าถึงโดยไม่ได้รับอนุญาต
- แก้ไข: ข้อมูล Site override จะได้รับการตรวจสอบความถูกต้องก่อนบันทึก
- แก้ไข: ข้อความแจ้งเตือนที่อยู่สำหรับเรียกเก็บเงิน (Billing address prompt) จะแสดงขึ้นเมื่อที่อยู่ว่างเปล่า
- แก้ไข: แก้ไขการแจ้งเตือนการเลิกใช้ (deprecation notices) null-to-string ของ PHP 8.1
- แก้ไข: Currents จะถูกโหลดแบบ lazy-loaded ก่อน hook init เพื่อป้องกันปัญหาด้านเวลา
- แก้ไข: เคารพเส้นทาง SSO ที่ถูกกรอง (Filtered SSO path) ในทุกขั้นตอนการเข้าสู่ระบบ
- แก้ไข: ตัวเลือก identity ของไซต์ที่ว่างเปล่าจะถูกเก็บรักษาไว้เมื่อบันทึก

## เวอร์ชัน 2.9.0 — เผยแพร่เมื่อ 2026-04-30

- ใหม่: เพิ่มการส่งออกและนำเข้าสำหรับไซต์เดียว (Single-site) ภายใต้เมนู **Tools > Export & Import**
- แก้ไข: ไฟล์ ZIP ที่ส่งออกจะถูกให้บริการผ่าน endpoint การดาวน์โหลดที่ต้องมีการยืนยันตัวตน
- แก้ไข: แก้ไขความเสี่ยง SQL injection และปัญหา Query ในการส่งออก/นำเข้าที่รอดำเนินการ
- แก้ไข: ไซต์ที่รอดำเนินการจะไม่ถูกเผยแพร่เมื่อผู้ดูแลระบบยืนยันอีเมลลูกค้าด้วยตนเอง
- แก้ไข: ทำความสะอาด record ของ pending_site ที่เป็นปัญหาเมื่อไม่มีข้อมูลสมาชิก
- แก้ไข: แก้ไข padding ของเมนูการตั้งค่าและ navigation anchor search
- แก้ไข: ไซต์ที่รอดำเนินการจะแสดงเป็นรายการแรกในมุมมอง All Sites
- แก้ไข: เพิ่ม User-Agent header สำหรับผู้ให้บริการ Screenshot (mShots) เพื่อป้องกันข้อผิดพลาด 403
- แก้ไข: แก้ไขการพึ่งพาแบบวงกลม (circular dependency) ของตารางเวลา Import cron
- แก้ไข: ปรับค่า Tour IDs ให้เป็น underscores ในคีย์การตั้งค่าผู้ใช้
- ปรับปรุง: ใช้ ZipArchive แทน Alchemy/Zippy เพื่อความเข้ากันได้ที่ดีขึ้น

## เวอร์ชัน 2.8.0 — เผยแพร่เมื่อ 2026-04-29

- ใหม่: เพิ่มสวิตช์ Enable Jumper ในส่วนการตั้งค่า Other Options
- ใหม่: เพิ่มคอลัมน์สถานะ (Status column) ในตารางรายการฟอร์ม Checkout
- ใหม่: เพิ่มไฟล์ loader sunrise สำหรับ Addon เพื่อขยาย MU-plugin sunrise แบบกำหนดเอง
- ปรับปรุง: ลบตัวเลือกการเปิดใช้งานการรายงานข้อผิดพลาด (error-reporting opt-in) ออกจากหน้าการตั้งค่า
- แก้ไข: การ์ดไซต์บนหน้าขอบคุณ (Thank-you page) — ปรับให้รูปภาพถูกจำกัดขนาดและจัดรูปแบบลิงก์อย่างถูกต้อง
- แก้ไข: ผู้ให้บริการ Screenshot เปลี่ยนจาก thum.io เป็น WordPress.com mShots
- แก้ไข: Enable Registration และ Default Role จะตั้งค่า default ที่ถูกต้องเมื่อติดตั้งใหม่
- แก้ไข: `get_site_url()` จะไม่คืนค่าว่างอีกต่อไปเมื่อโดเมนมี port
- แก้ไข: ไฟล์สื่อที่ถูกโคลนจะถูกคัดลอกอย่างถูกต้องเมื่อการตั้งค่า `copy_media` ว่างเปล่า
- แก้ไข: Object cache จะถูกทำให้เป็นโมฆะอย่างถูกต้องหลังจากเขียน sitemeta ของ network-activate
- แก้ไข: โดเมนที่กำหนดเองจะถูกโปรโมตเป็นหลักโดยอัตโนมัติเมื่อมีการตรวจสอบ DNS สำหรับโดเมน 3 ส่วน
- แก้ไข: สมาชิกภาพที่รอดำเนินการจะถูกยกเลิกเมื่อมีการล้างการชำระเงินที่หมดอายุ
- แก้ไข: Password strength checker จะถูกผูกใหม่หลังจากปิด inline login prompt
- แก้ไข: การรีโหลดหน้าเว็บไม่รู้จบจะหยุดลงบนหน้าขอบคุณเมื่อไซต์ถูกสร้างขึ้นแล้ว
- แก้ไข: ตัวเลือกการลงทะเบียนของ WP core จะซิงค์เมื่อเปิดใช้งานปลั๊กอินและบันทึกการตั้งค่า
- แก้ไข: เพิ่มการป้องกัน null expiration ใน `calculate_expiration` เพื่อความเข้ากันได้กับ PHP 8.4
- แก้ไข: บล็อกการลงทะเบียนซ้ำเมื่อลูกค้ามีสมาชิกภาพที่ใช้งานอยู่แล้ว
- แก้ไข: เพิ่มการตรวจสอบ null สำหรับ `date_expiration` ใน checkout
- แก้ไข: การจัดเตรียมไซต์ (Site provisioning) มีความแข็งแกร่งขึ้น — จำกัดขอบเขต, การอนุมานสมาชิกภาพ, การโปรโมตโดเมน
- แก้ไข: ป้ายสถานะการตรวจสอบก่อนติดตั้ง (Pre-install check status label) ถูกแก้ไขเป็น NOT Activated เมื่อการตรวจสอบล้มเหลว
- แก้ไข: ใช้โดเมน Checkout สำหรับ URL การยืนยันอีเมล
- แก้ไข: การเข้าสู่ระบบอัตโนมัติหลัง checkout เมื่อไม่มีฟิลด์รหัสผ่าน
- แก้ไข: สมาชิกภาพฟรีจะไม่หมดอายุอีกต่อไป — ถือเป็นตลอดชีพ
- แก้ไข: ระบบยืนยันอีเมลจะระงับการเผยแพร่ไซต์จนกว่าลูกค้าจะยืนยันอีเมล
- แก้ไข: แก้ไข base path และ identity route ของ endpoint API SES v2
- แก้ไข: hook `wu_inline_login_error` ถูกปล่อยใน pre-submit catch block
