# การผสานการทำงานกับ Closte

## ภาพรวม
Closte เป็นแพลตฟอร์ม managed WordPress hosting ที่สร้างบนโครงสร้างพื้นฐานของ Google Cloud การผสานการทำงานนี้ช่วยให้สามารถซิงค์โดเมนและจัดการใบรับรอง SSL ระหว่าง Ultimate Multisite และ Closte ได้โดยอัตโนมัติ

## คุณสมบัติ
- ซิงค์โดเมนอัตโนมัติ
- จัดการใบรับรอง SSL
- รองรับ wildcard domain
- ไม่ต้องตั้งค่าใดๆ หากใช้งานบน Closte

## ข้อกำหนด
ต้องกำหนดค่าคงที่ต่อไปนี้ในไฟล์ `wp-config.php` ของคุณ หากคุณใช้ Closte:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

โดยปกติค่าคงที่นี้จะถูกกำหนดไว้แล้วหากคุณโฮสต์บน Closte

## คำแนะนำการตั้งค่า

### 1. ตรวจสอบ API Key ของ Closte

หากคุณโฮสต์บน Closte ค่าคงที่ `CLOSTE_CLIENT_API_KEY` ควรถูกกำหนดไว้ในไฟล์ `wp-config.php` ของคุณแล้ว คุณสามารถตรวจสอบได้โดยเปิดดูไฟล์ `wp-config.php`

### 2. เปิดใช้งานการผสานการทำงาน

1. ในหน้า WordPress admin ไปที่ Ultimate Multisite > Settings
2. ไปที่แท็บ "Domain Mapping"
3. เลื่อนลงไปที่ "Host Integrations"
4. เปิดใช้งานการผสานการทำงานกับ Closte
5. คลิก "Save Changes"

## วิธีการทำงาน

เมื่อมีการแมปโดเมนใน Ultimate Multisite:

1. ระบบจะส่งคำขอไปยัง API ของ Closte เพื่อเพิ่มโดเมนในแอปพลิเคชันของคุณ
2. Closte จะจัดการออกใบรับรอง SSL ให้โดยอัตโนมัติ
3. เมื่อลบการแมปโดเมน ระบบจะลบโดเมนออกจาก Closte

การผสานการทำงานนี้ยังทำงานร่วมกับการตั้งค่าช่วงเวลาตรวจสอบ DNS ใน Ultimate Multisite ซึ่งช่วยให้คุณกำหนดความถี่ในการตรวจสอบการเผยแพร่ DNS และการออกใบรับรอง SSL ได้

## การสร้างระเบียนโดเมน

การผสานการทำงานนี้ทำให้มั่นใจได้ว่าเมื่อสร้างหรือทำซ้ำเว็บไซต์ ระเบียนโดเมนจะถูกสร้างขึ้นโดยอัตโนมัติ นี่เป็นสิ่งสำคัญอย่างยิ่งสำหรับการผสานการทำงานกับ Closte เนื่องจากการสร้างระเบียนโดเมนจะกระตุ้นให้ Closte API สร้างโดเมนและใบรับรอง SSL

## การแก้ไขปัญหา

### ปัญหาการเชื่อมต่อ API
- ตรวจสอบว่า API key ของ Closte ถูกต้อง
- ตรวจสอบให้แน่ใจว่าบัญชี Closte ของคุณมีสิทธิ์ที่จำเป็น

### ปัญหาใบรับรอง SSL
- Closte อาจใช้เวลาสักครู่ในการออกใบรับรอง SSL (โดยปกติ 5-10 นาที)
- ตรวจสอบว่าโดเมนของคุณชี้ไปยังที่อยู่ IP ของเซิร์ฟเวอร์ Closte อย่างถูกต้อง
- ตรวจสอบระเบียน DNS ของโดเมนเพื่อให้แน่ใจว่าได้รับการกำหนดค่าอย่างถูกต้อง

### โดเมนไม่ถูกเพิ่ม
- ตรวจสอบบันทึกของ Ultimate Multisite เพื่อดูข้อความแสดงข้อผิดพลาด
- ตรวจสอบว่าโดเมนยังไม่ได้ถูกเพิ่มใน Closte
- ตรวจสอบให้แน่ใจว่าระเบียน DNS ของโดเมนได้รับการกำหนดค่าอย่างถูกต้อง

### ช่วงเวลาตรวจสอบ DNS
- หากใบรับรอง SSL ใช้เวลานานเกินไปในการออก คุณสามารถปรับช่วงเวลาตรวจสอบ DNS ได้ในการตั้งค่า Domain Mapping
- ช่วงเวลาเริ่มต้นคือ 300 วินาที (5 นาที) แต่คุณสามารถตั้งค่าต่ำสุดที่ 10 วินาทีเพื่อการตรวจสอบที่รวดเร็วขึ้นระหว่างการทดสอบ
