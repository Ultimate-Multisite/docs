# การดาวน์เกรดแผน (v2)

_**หมายเหตุสำคัญ: บทความนี้อ้างอิงถึง Ultimate Multisite เวอร์ชัน 2.x**_

การดาวน์เกรดแผนหรือการสมัครสมาชิกเป็นการดำเนินการทั่วไปที่ลูกค้าของคุณอาจทำ หากพวกเขามีงบประมาณจำกัด หรือพวกเขาตัดสินใจว่าไม่จำเป็นต้องใช้ทรัพยากรมากมายเพื่อใช้งานซับไซต์ของตน

## วิธีดาวน์เกรดแผน

ลูกค้าของคุณสามารถดาวน์เกรดแผนได้ทุกเมื่อ โดยเข้าสู่ระบบไปยัง Dashboard ผู้ดูแลระบบของซับไซต์ และคลิก **เปลี่ยน** ใต้หน้า Account ของพวกเขา

![หน้า Account ของลูกค้าพร้อมการ์ดการเป็นสมาชิกของคุณและปุ่มเปลี่ยน](/img/account-page/membership-change-button.png)

เมื่อคลิกปุ่ม **เปลี่ยน** ผู้ใช้/ลูกค้าจะถูกนำไปยังหน้าชำระเงิน ซึ่งพวกเขาสามารถเลือกแผนที่ต้องการเปลี่ยนการสมัครสมาชิกไปใช้ได้

![หน้าตัวเลือกการดาวน์เกรดแผนฝั่งลูกค้า](/img/account-page/downgrade-picker.png)

ในตัวอย่างนี้ เรากำลังดาวน์เกรดแผนจาก **Premium** เป็น **Free**

เพื่อดำเนินการต่อ ผู้ใช้เพียงต้องคลิกปุ่ม **ดำเนินการชำระเงินให้เสร็จสมบูรณ์** จากนั้นระบบจะนำพวกเขากลับไปยังหน้า Account พร้อมแสดงข้อความเกี่ยวกับการเปลี่ยนแปลงที่รอดำเนินการสำหรับการเป็นสมาชิก การเปลี่ยนแปลงจะมีผลใน **รอบการเรียกเก็บเงินถัดไป** ของลูกค้า

![หน้า Account ที่แสดงแบนเนอร์การเปลี่ยนแปลงการเป็นสมาชิกที่รอดำเนินการ](/img/account-page/pending-change.png)

### จะเกิดอะไรขึ้นเมื่อผู้ใช้ดาวน์เกรดแผนของตน

สิ่งสำคัญที่ควรทราบคือ การดาวน์เกรดแผนจะไม่เปลี่ยนแปลงการกำหนดค่าที่มีอยู่ในซับไซต์ของผู้ใช้

ระบบจะไม่เปลี่ยนเทมเพลตของไซต์โดยอัตโนมัติ เนื่องจากการเปลี่ยนเทมเพลตของไซต์จะลบและรีเซ็ตซับไซต์ทั้งหมด ทั้งนี้เพื่อหลีกเลี่ยงการสูญเสียข้อมูลที่ไม่จำเป็น ดังนั้นพื้นที่ดิสก์ ธีม ปลั๊กอิน ฯลฯ จะยังคงอยู่ ยกเว้นโพสต์

เราเข้าใจว่าข้อกังวลหลักของคุณคือขีดจำกัดและโควตาที่คุณตั้งไว้ในแต่ละแผน แต่เราต้องคำนึงถึงความเสียหายที่จะเกิดขึ้นกับซับไซต์ของผู้ใช้ หากเราลบหรือเปลี่ยนแปลงการกำหนดค่าใด ๆ ของซับไซต์นั้น

สำหรับโพสต์ที่เกินขีดจำกัดที่ตั้งไว้ในแผน คุณมี 3 ตัวเลือกที่แตกต่างกัน: **เก็บโพสต์ไว้ตามเดิม** *,* **ย้ายโพสต์ไปยังถังขยะ** *,* หรือ **ย้ายโพสต์ไปยังฉบับร่าง** *.* คุณสามารถกำหนดค่านี้ได้ในการตั้งค่า Ultimate Multisite

![หน้าไซต์ในการตั้งค่า Network Admin ที่แสดงตัวเลือกพฤติกรรมเมื่อเกินขีดจำกัดโพสต์](/img/account-page/settings-sites-post-limit.png)

### จะเกิดอะไรขึ้นกับการชำระเงิน

ในเวอร์ชัน 2.0 จะไม่จำเป็นต้องมีการปรับใด ๆ ในการชำระเงินในแง่ของการคำนวณตามสัดส่วนอีกต่อไป

เนื่องจากระบบจะรอให้การเป็นสมาชิกที่มีอยู่ **ครบตามรอบการเรียกเก็บเงินก่อน** แผน/การเป็นสมาชิกใหม่จึงจะมีผล จำนวนเงินเรียกเก็บใหม่สำหรับการเป็นสมาชิกใหม่จะถูกนำไปใช้และเรียกเก็บโดยอัตโนมัติในรอบการเรียกเก็บเงินถัดไป
