# การปรับแต่งฟอร์มลงทะเบียนของคุณ

เพื่อให้เครือข่ายของคุณดูแตกต่างจาก SaaS อื่นๆ ที่สร้างบนแพลตฟอร์ม WordPress Ultimate Multisite ช่วยให้คุณปรับแต่งหน้าลงทะเบียนและหน้าเข้าสู่ระบบได้ด้วยฟีเจอร์ **Checkout Forms** ของเรา

แม้ว่าฟีเจอร์นี้จะเป็นวิธีที่ง่ายและยืดหยุ่นสำหรับทดลองแนวทางต่างๆ ในการเปลี่ยนผู้เยี่ยมชมให้เป็นลูกค้า แต่ส่วนใหญ่จะใช้สำหรับสร้างฟอร์มลงทะเบียนแบบเฉพาะตัว บทความนี้จะแสดงให้คุณเห็นว่าสามารถทำได้อย่างไร

## หน้าเข้าสู่ระบบและหน้าลงทะเบียน:

เมื่อติดตั้ง Ultimate Multisite ระบบจะสร้างหน้าเข้าสู่ระบบและหน้าลงทะเบียนแบบกำหนดเองบนเว็บไซต์หลักของคุณโดยอัตโนมัติ คุณสามารถเปลี่ยนหน้าเริ่มต้นเหล่านี้ได้ทุกเมื่อโดยไปที่ **Ultimate Multisite > Settings > Login & Registration**

![หน้าตั้งค่า Login and Registration](/img/config/settings-general.png)

มาดูตัวเลือกแต่ละอย่างที่คุณสามารถปรับแต่งได้ในหน้า **Login & Registration** กัน:

  * **Enable registration:** ตัวเลือกนี้จะเปิดหรือปิดการลงทะเบียนบนเครือข่ายของคุณ หากปิดไว้ ลูกค้าของคุณจะไม่สามารถลงทะเบียนและสมัครใช้ผลิตภัณฑ์ของคุณได้

  * **Enable email verification:** หากเปิดตัวเลือกนี้ไว้ ลูกค้าที่สมัครแผนฟรีหรือแผนชำระเงินที่มีช่วงทดลองใช้จะได้รับอีเมลยืนยัน และจะต้องคลิกลิงก์ยืนยันเพื่อให้เว็บไซต์ของพวกเขาถูกสร้างขึ้น

  * **Default registration page:** นี่คือหน้าเริ่มต้นสำหรับการลงทะเบียน หน้านี้จะต้องเผยแพร่บนเว็บไซต์ของคุณและมีฟอร์มลงทะเบียน (หรือที่เรียกว่า checkout form) ซึ่งลูกค้าของคุณจะสมัครใช้ผลิตภัณฑ์ คุณสามารถสร้างหน้าลงทะเบียนและ checkout form ได้มากเท่าที่ต้องการ เพียงอย่าลืมใส่ shortcode ของ checkout form ในหน้าลงทะเบียน ไม่เช่นนั้นฟอร์มจะไม่แสดง

  * **Use custom login page:** ตัวเลือกนี้ช่วยให้คุณใช้หน้าเข้าสู่ระบบแบบกำหนดเองแทนหน้า wp-login.php เริ่มต้น หากเปิดตัวเลือกนี้ไว้ คุณสามารถเลือกหน้าที่จะใช้สำหรับเข้าสู่ระบบได้ในตัวเลือก **Default login page** (ด้านล่าง)

  * **Obfuscate the original login url (wp-login.php):** หากคุณต้องการซ่อน URL เข้าสู่ระบบเดิม คุณสามารถเปิดตัวเลือกนี้ได้ สิ่งนี้มีประโยชน์ในการป้องกันการโจมตีแบบ brute-force หากเปิดตัวเลือกนี้ไว้ Ultimate Multisite จะแสดงข้อผิดพลาด 404 เมื่อผู้ใช้พยายามเข้าถึงลิงก์ wp-login.php เดิม

  * **Force synchronous site publication:** หลังจากลูกค้าสมัครใช้ผลิตภัณฑ์บนเครือข่าย เว็บไซต์ที่รอดำเนินการใหม่จะต้องถูกแปลงเป็นเว็บไซต์เครือข่ายจริง กระบวนการเผยแพร่จะเกิดขึ้นผ่าน Job Queue แบบ asynchronous เปิดตัวเลือกนี้เพื่อบังคับให้การเผยแพร่เกิดขึ้นใน request เดียวกันกับการสมัคร

ตอนนี้มาดูตัวเลือกอื่นๆ ที่ยังเกี่ยวข้องกับกระบวนการเข้าสู่ระบบและลงทะเบียน ซึ่งอยู่ด้านล่าง **Other options** ในหน้า Login & registration เดียวกัน:

  * **Default role:** นี่คือบทบาทที่ลูกค้าของคุณจะมีบนเว็บไซต์ของพวกเขาหลังจากกระบวนการสมัคร

  * **Add users to the main site as well:** การเปิดตัวเลือกนี้จะเพิ่มผู้ใช้ไปยังเว็บไซต์หลักของเครือข่ายด้วยหลังจากกระบวนการสมัคร หากคุณเปิดตัวเลือกนี้ ตัวเลือกสำหรับตั้ง **default role** ของผู้ใช้เหล่านี้บนเว็บไซต์ของคุณจะปรากฏด้านล่างด้วย

  * **Enable multiple accounts:** อนุญาตให้ผู้ใช้มีบัญชีในเว็บไซต์ต่างๆ ของเครือข่ายด้วยอีเมลเดียวกัน หากปิดตัวเลือกนี้ไว้ ลูกค้าของคุณจะไม่สามารถสร้างบัญชีบนเว็บไซต์อื่นๆ ที่ทำงานบนเครือข่ายของคุณด้วยอีเมลเดียวกันได้

และนั่นคือตัวเลือกทั้งหมดที่เกี่ยวข้องกับการเข้าสู่ระบบและลงทะเบียนที่คุณสามารถปรับแต่งได้! อย่าลืมบันทึกการตั้งค่าหลังจากแก้ไขเสร็จ

## การใช้ฟอร์มลงทะเบียนหลายฟอร์ม:

Ultimate Multisite 2.0 มีตัวแก้ไข checkout form ที่ช่วยให้คุณสร้างฟอร์มได้มากเท่าที่ต้องการ พร้อมฟิลด์ที่แตกต่างกัน ผลิตภัณฑ์ที่เสนอ และอื่นๆ

ทั้งหน้าเข้าสู่ระบบและหน้าลงทะเบียนจะฝัง shortcode ไว้: **[wu_login_form]** บนหน้าเข้าสู่ระบบ และ **[wu_checkout]** สำหรับหน้าลงทะเบียน คุณสามารถปรับแต่งหน้าลงทะเบียนเพิ่มเติมได้โดยการสร้าง checkout form

หากต้องการเข้าถึงฟีเจอร์นี้ ให้ไปที่เมนู **Checkout Forms** ที่แถบด้านซ้าย

![เมนู Checkout Forms ในแถบด้านข้าง](/img/config/checkout-forms-list.png)

ในหน้านี้ คุณจะเห็น checkout form ทั้งหมดที่คุณมี

หากต้องการสร้างฟอร์มใหม่ เพียงคลิก **Add Checkout Form** ที่ด้านบนของหน้า

คุณสามารถเลือกหนึ่งในสามตัวเลือกนี้เป็นจุดเริ่มต้น: single step, multi-step หรือ blank จากนั้นคลิก **Go to the Editor**

![Add Checkout Form พร้อมตัวเลือก single step, multi-step หรือ blank](/img/config/checkout-forms-list.png)

หรือคุณสามารถแก้ไขหรือทำสำเนาฟอร์มที่มีอยู่แล้วได้โดยคลิกที่ตัวเลือกด้านล่างชื่อฟอร์ม ที่นั่นคุณจะพบตัวเลือกสำหรับคัดลอก shortcode ของฟอร์มหรือลบฟอร์มด้วย

![การดำเนินการเมื่อวางเมาส์บน checkout form พร้อมตัวเลือกแก้ไข ทำสำเนา และลบ](/img/config/checkout-form-hover-actions.png)

หากคุณเลือก single step หรือ multi-step checkout form จะถูกเติมขั้นตอนพื้นฐานที่จำเป็นให้โดยอัตโนมัติ จากนั้นหากต้องการ คุณสามารถเพิ่มขั้นตอนเพิ่มเติมได้

### การแก้ไข Checkout Form:

ดังที่เราได้กล่าวไว้ก่อนหน้านี้ คุณสามารถสร้าง checkout form สำหรับวัตถุประสงค์ต่างๆ ในตัวอย่างนี้เราจะทำงานกับฟอร์มลงทะเบียน

หลังจากไปยังตัวแก้ไข checkout form ให้ตั้งชื่อฟอร์มของคุณ (ที่จะใช้สำหรับการอ้างอิงภายในเท่านั้น) และ slug (ใช้สำหรับสร้าง shortcode เป็นต้น)

![ตัวแก้ไข checkout form พร้อมฟิลด์ชื่อและ slug](/img/config/checkout-form-editor.png)

ฟอร์มประกอบด้วยขั้นตอนและฟิลด์ คุณสามารถเพิ่มขั้นตอนใหม่ได้โดยคลิก **Add New Checkout Step**

![ปุ่ม Add New Checkout Step](/img/config/checkout-form-add-step.png)

ในแท็บแรกของหน้าต่าง modal ให้กรอกเนื้อหาของขั้นตอนฟอร์ม ตั้ง ID ชื่อ และคำอธิบาย รายการเหล่านี้ส่วนใหญ่ใช้ภายใน

![แท็บเนื้อหาขั้นตอน checkout พร้อม ID ชื่อ และคำอธิบาย](/img/config/checkout-form-step.png)

ถัดไป ตั้งค่าการมองเห็นของขั้นตอน คุณสามารถเลือกระหว่าง **Always show**, **Only show for logged in users** หรือ **Only show for guests**

![ตัวเลือกการมองเห็นขั้นตอน checkout](/img/config/checkout-form-step.png)

สุดท้าย กำหนดค่าสไตล์ของขั้นตอน เหล่านี้เป็นฟิลด์ที่ไม่บังคับ

![การกำหนดค่าสไตล์ขั้นตอน checkout](/img/config/checkout-form-step.png)

ตอนนี้ถึงเวลาเพิ่มฟิลด์ในขั้นตอนแรก เพียงคลิก **Add New Field** และเลือกประเภทส่วนที่คุณต้องการ

![ปุ่ม Add New Field](/img/config/checkout-form-editor.png)![dropdown เลือกประเภทฟิลด์](/img/config/checkout-form-step.png)

ฟิลด์แต่ละอันมีพารามิเตอร์ต่างๆ ที่ต้องกรอก สำหรับรายการแรกนี้ เราจะเลือกฟิลด์ **Username**

![การกำหนดค่าฟิลด์ Username](/img/config/checkout-form-step.png)![พารามิเตอร์ฟิลด์ Username](/img/config/checkout-form-step.png)![การตั้งค่าเพิ่มเติมของฟิลด์ Username](/img/config/checkout-form-step.png)

คุณสามารถเพิ่มขั้นตอนและฟิลด์ได้มากเท่าที่ต้องการ หากต้องการแสดงผลิตภัณฑ์ให้ลูกค้าเลือก ให้ใช้ฟิลด์ Pricing Table หากต้องการให้ลูกค้าเลือก template ให้เพิ่มฟิลด์ Template Selection และอื่นๆ

_**หมายเหตุ:** หากคุณสร้างผลิตภัณฑ์หลังจากสร้าง checkout form คุณจะต้องเพิ่มผลิตภัณฑ์ในส่วน Pricing table หากคุณไม่เพิ่ม ผลิตภัณฑ์จะไม่แสดงให้ลูกค้าเห็นในหน้าลงทะเบียน_

_**หมายเหตุ 2:** username, email, password, site title, site URL, order summary, payment และ submit button เป็นฟิลด์บังคับสำหรับการสร้าง checkout form_

ขณะที่คุณทำงานกับ checkout form คุณสามารถใช้ปุ่ม Preview เพื่อดูว่าลูกค้าจะเห็นฟอร์มอย่างไร คุณยังสามารถสลับระหว่างดูในฐานะผู้ใช้ที่มีอยู่หรือผู้เยี่ยมชม

![ปุ่ม Preview บนตัวแก้ไข checkout form](/img/config/checkout-form-editor.png)![ตัวอย่าง checkout form ในฐานะผู้เยี่ยมชมหรือผู้ใช้ที่มีอยู่](/img/config/checkout-form-editor.png)

สุดท้าย ใน **Advanced Options** คุณสามารถกำหนดค่าข้อความสำหรับหน้า **Thank You** เพิ่ม snippet สำหรับติดตาม conversion เพิ่ม CSS แบบกำหนดเองให้ checkout form หรือจำกัดให้ใช้ได้เฉพาะบางประเทศ

![Advanced Options พร้อมหน้า Thank You, การติดตาม conversion และ CSS แบบกำหนดเอง](/img/config/checkout-form-advanced.png)

คุณยังสามารถเปิดหรือปิด checkout form ได้ด้วยตนเองโดยสลับตัวเลือกนี้ที่คอลัมน์ขวา หรือลบฟอร์มอย่างถาวร

![ปุ่มสลับ Active และตัวเลือกลบสำหรับ checkout form](/img/config/checkout-form-active.png)

อย่าลืมบันทึก checkout form ของคุณ!

![ปุ่ม Save Checkout Form](/img/config/checkout-form-save.png)

หากต้องการรับ shortcode ของฟอร์ม ให้คลิก **Generate Shortcode** และคัดลอกผลลัพธ์ที่แสดงในหน้าต่าง modal

![modal Generate Shortcode พร้อม shortcode ให้คัดลอก](/img/config/checkout-form-editor.png)

_**หมายเหตุ:** คุณจะต้องเพิ่ม shortcode นี้ไปยังหน้าลงทะเบียนของคุณเพื่อให้ checkout form นี้ถูกเพิ่มเข้าไป_

## การเลือกผลิตภัณฑ์และ template ล่วงหน้าผ่านพารามิเตอร์ URL:

หากคุณต้องการสร้างตารางราคาแบบกำหนดเองสำหรับผลิตภัณฑ์ของคุณและเลือกล่วงหน้าใน checkout form ว่าผลิตภัณฑ์หรือ template ใดที่ลูกค้าเลือกจากตารางราคาหรือหน้า template คุณสามารถใช้พารามิเตอร์ URL สำหรับสิ่งนี้

### **สำหรับแผน:**

ไปที่ **Ultimate Multisite > Products > Select a plan** คุณควรเห็นปุ่ม **Click to copy Shareable Link** ที่ด้านบนของหน้า นี่คือลิงก์ที่คุณสามารถใช้เพื่อเลือกแผนนี้ล่วงหน้าใน checkout form ของคุณ

![หน้าผลิตภัณฑ์พร้อมปุ่ม shareable link](/img/config/products-list.png)

โปรดทราบว่า shareable link นี้ใช้ได้กับ **Plans** เท่านั้น คุณไม่สามารถใช้ shareable link สำหรับ package หรือ service

### สำหรับ template:

หากคุณต้องการเลือก site template ล่วงหน้าใน checkout form คุณสามารถใช้พารามิเตอร์: **?template_id=X** บน URL หน้าลงทะเบียนของคุณ "X" จะต้องถูกแทนที่ด้วย **หมายเลข ID ของ site template** หากต้องการรับหมายเลขนี้ ให้ไปที่ **Ultimate Multisite > Sites**

คลิก **Manage** ด้านล่าง site template ที่คุณต้องการใช้ คุณจะเห็นหมายเลข SITE ID เพียงใช้หมายเลขนี้สำหรับ site template นี้เพื่อให้ถูกเลือกล่วงหน้าใน checkout form ของคุณ ในกรณีของเรา พารามิเตอร์ URL จะเป็น **?template_id=2**

![รายการ Sites แสดง ID ของ site template](/img/config/site-templates-list.png)

สมมติว่าเว็บไซต์เครือข่ายของเราคือ [**www.mynetwork.com**](http://www.mynetwork.com) และหน้าลงทะเบียนพร้อม checkout form ของเราอยู่ที่หน้า **/register** URL ทั้งหมดที่มี site template ถูกเลือกล่วงหน้าจะมีลักษณะเป็น [**www.mynetwork.com/register/?template**](http://www.mynetwork.com/register/?template)**_id=2**

และหากคุณต้องการ คุณสามารถเลือกทั้งผลิตภัณฑ์และ template ล่วงหน้าใน checkout form ของคุณ สิ่งที่คุณต้องทำคือคัดลอก shareable link ของแผนและวางพารามิเตอร์ template ไว้ท้าย จะมีลักษณะเป็น [**www.mynetwork.com/register/premium-plan/?template**](http://www.mynetwork.com/register/premium-plan/?template)**_id=2**
