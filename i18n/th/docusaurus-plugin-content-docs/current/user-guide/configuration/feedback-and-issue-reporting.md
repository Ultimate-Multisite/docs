# การให้ข้อเสนอแนะและการรายงานปัญหาของลูกค้า

Gratis AI Agent v1.5.0 ได้เพิ่มระบบการให้ข้อเสนอแนะและการรายงานปัญหาที่มาพร้อมกับตัวแอปพลิเคชัน ซึ่งช่วยให้ผู้ใช้ปลายทางสามารถแจ้งได้ว่าคำตอบใดไม่เป็นประโยชน์ หรือรายงานปัญหาต่างๆ ได้โดยตรงจากหน้าต่างแชท ระบบนี้ประกอบด้วยการจัดการความยินยอม (consent management) คำสั่งรายงานอัตโนมัติ และการคัดแยกปัญหาเบื้องต้นด้วย AI (AI-assisted triage) ในส่วนหลังบ้าน

## ปุ่ม 👍 (Thumbs-Down Button) {#thumbs-down-button}

ทุกข้อความที่ผู้ช่วย AI ส่งออกไปจะมีปุ่ม **👎 (Thumbs-Down)** แสดงอยู่ เมื่อผู้ใช้คลิกที่ปุ่มนี้ พวกเขาสามารถทำเครื่องหมายว่าคำตอบนั้นไม่เป็นประโยชน์หรือไม่ถูกต้อง

- ปุ่มจะปรากฏขึ้นเมื่อนำเมาส์ไปวางเหนือข้อความของผู้ช่วยแต่ละข้อความ
- การคลิกจะเปิด **Feedback Consent Modal**
- ข้อเสนอแนะที่ส่งเข้ามาจะถูกเชื่อมโยงกับเธรดการสนทนาและข้อความที่เฉพาะเจาะจงนั้นๆ

## Feedback Consent Modal {#feedback-consent-modal}

เมื่อผู้ใช้คลิกปุ่ม thumbs-down จะมีหน้าต่าง Modal ขอความยินยอมปรากฏขึ้นก่อนที่จะมีการส่งข้อมูลใดๆ Modal นี้จะ:

- อธิบายว่าข้อมูลใดบ้างที่จะถูกรวบรวม (เช่น ส่วนของบทสนทนา, บริบทของเบราว์เซอร์)
- ขอให้ผู้ใช้ยืนยันว่ายินยอมที่จะแชร์ข้อมูลเหล่านี้
- มีช่องข้อความให้ผู้ใช้บรรยายว่าเกิดข้อผิดพลาดอะไรขึ้น
- มีตัวเลือก **Cancel** เพื่อยกเลิกโดยไม่ส่งข้อมูล

จะไม่มีการบันทึกข้อเสนอแนะใดๆ จนกว่าผู้ใช้จะยืนยันด้วยตนเอง

## Auto-Prompt Feedback Banner {#auto-prompt-feedback-banner}

เมื่อจบการสนทนา ผู้ช่วยอาจแสดง **auto-prompt feedback banner** ซึ่งเป็นข้อความที่ไม่รบกวนสายตา เพื่อสอบถามว่าเซสชันนี้มีประโยชน์หรือไม่

แบนเนอร์นี้จะปรากฏขึ้นโดยอัตโนมัติตามความยาวและผลลัพธ์ของการสนทนา โดยจะเชื่อมโยงไปยังขั้นตอนการให้ข้อเสนอแนะเดียวกับปุ่ม thumbs-down ผู้ใช้สามารถปิดแบนเนอร์นี้ได้โดยไม่ต้องให้ข้อเสนอแนะ

## คำสั่ง /report-issue Slash Command {#report-issue-slash-command}

ผู้ใช้สามารถเรียกใช้ขั้นตอนการให้ข้อเสนอแนะได้โดยตรงด้วยการพิมพ์ `/report-issue` ในช่องแชท คำสั่งนี้จะ:

- เปิด Feedback Consent Modal ทันที
- ใส่ข้อมูลบริบทเกี่ยวกับการสนทนาปัจจุบันลงในช่องคำอธิบายล่วงหน้า
- อนุญาตให้ผู้ใช้เพิ่มรายละเอียดเพิ่มเติมก่อนส่ง

คำสั่ง `/report-issue` สามารถใช้ได้ในโหมดแชททั้งหมด (inline, floating widget, full-screen)

## AI-Assisted Triage {#ai-assisted-triage}

ข้อเสนอแนะที่ส่งเข้ามาจะถูกส่งไปยังชั้นการคัดแยกปัญหาด้วย AI (AI triage layer) ซึ่งจะ:

- จัดหมวดหมู่รายงาน (เช่น ข้อเท็จจริงผิดพลาด, คำตอบไม่เป็นประโยชน์, เนื้อหาไม่เหมาะสม ฯลฯ)
- แนบข้อมูลเมตา (metadata) ของการสนทนาที่เกี่ยวข้อง
- ส่งสรุปการคัดแยกไปยัง **Feedback Endpoint** ที่กำหนดค่าไว้ (ดูที่ [Settings > Advanced](#settings-advanced))

สิ่งนี้ช่วยลดเวลาในการตรวจสอบด้วยตนเอง โดยการนำปัญหาที่สำคัญที่สุดขึ้นมาให้เห็นก่อน

## Settings > Advanced {#settings-advanced}

ในการเปิดใช้งานระบบหลังบ้านสำหรับการให้ข้อเสนอแนะ ให้กำหนดค่าฟิลด์ต่อไปนี้ภายใต้ **Gratis AI Agent → Settings → Advanced**:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | URL ที่รับ POST requests ที่มี payload ของข้อเสนอแนะ (JSON) |
| **Feedback API Key** | Bearer token ที่ส่งในส่วน `Authorization` header สำหรับการยืนยันการส่งข้อเสนอแนะ |

ปล่อยให้ทั้งสองฟิลด์ว่างไว้เพื่อปิดการเก็บข้อเสนอแนะ ปุ่มให้ข้อเสนอแนะแต่ละปุ่มยังคงมองเห็นได้สำหรับผู้ใช้ แต่การส่งข้อมูลจะไม่ถูกส่งต่อไป

:::tip
Feedback endpoint ต้องยอมรับ JSON body ที่มีอย่างน้อย `message_id`, `conversation_id`, `feedback_text`, และ `triage_category` ดูเอกสารของผู้ให้บริการ endpoint ของคุณสำหรับ schema ที่คาดหวัง
:::
