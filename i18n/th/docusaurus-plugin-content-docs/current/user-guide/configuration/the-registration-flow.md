# ขั้นตอนการลงทะเบียน (v2) {#the-registration-flow-v2}

_**หมายเหตุสำคัญ: บทความนี้อ้างอิงถึง Ultimate Multisite เวอร์ชัน 2.x**_

ผู้ใช้สามารถลงทะเบียนเข้าสู่เครือข่ายของคุณได้หลายวิธี พวกเขาสามารถใช้แบบฟอร์มลงทะเบียนของคุณ หรือลิงก์ที่แชร์ได้ไปยังแผนที่เลือกไว้ล่วงหน้า ที่นี่เราจะแสดงให้คุณเห็นว่าลูกค้าของคุณสามารถลงทะเบียนบนเครือข่ายของคุณได้อย่างไรผ่านเส้นทางที่มีให้ใช้ และจะเกิดอะไรขึ้นหลังจากพวกเขาลงทะเบียนบนเครือข่ายของคุณ

## การใช้แบบฟอร์มลงทะเบียน: {#using-the-registration-form}

นี่คือกระบวนการลงทะเบียนมาตรฐาน คุณสร้างหน้าลงทะเบียนพร้อม **แบบฟอร์มชำระเงิน** และหน้านี้จะเป็นที่ที่ลูกค้าของคุณจะไปเพื่อลงทะเบียนบนเครือข่ายของคุณและสมัครใช้งานแผน คุณสามารถมีหน้าลงทะเบียนได้หลายหน้า โดยแต่ละหน้ามีแบบฟอร์มลงทะเบียนที่แตกต่างกันได้หากต้องการ

หน้าเริ่มต้นสำหรับการลงทะเบียนคือ [_**yourdomain.com/register**_](http://yourdomain.com/register) แต่คุณสามารถเปลี่ยนได้ทุกเมื่อที่ **Ultimate Multisite > Settings > Login & Registration > Default Registration Page**

หลังจากผู้ใช้มาถึงหน้าลงทะเบียนของคุณ (โดยปกติคือคลิกปุ่ม **เข้าสู่ระบบ** หรือ **ซื้อเลย**) พวกเขาจะเห็นแบบฟอร์มลงทะเบียนของคุณที่นั่น

![แบบฟอร์มลงทะเบียนที่แสดงบนหน้าลงทะเบียน](/img/frontend/registration-form.png)

นี่คือตัวอย่างของแบบฟอร์มชำระเงินตามที่ปรากฏบนส่วนหน้า:

![แบบฟอร์มลงทะเบียนชำระเงินบนส่วนหน้า](/img/config/checkout-frontend-registration.png)

สิ่งที่พวกเขาต้องทำคือกรอกช่องบังคับทั้งหมด - อีเมล ชื่อผู้ใช้ รหัสผ่าน ฯลฯ - และชำระเงินสำหรับแผน หรือยืนยันที่อยู่อีเมลของตน หากพวกเขากำลังลงทะเบียนแผนฟรีหรือแผนแบบชำระเงินที่มีช่วงทดลองใช้โดยไม่มีข้อมูลการชำระเงิน

บนหน้า "ขอบคุณ" พวกเขาจะเห็นข้อความแจ้งว่าพวกเขาจำเป็นต้องยืนยันที่อยู่อีเมลหรือไม่ หรือเว็บไซต์ของพวกเขาเปิดใช้งานแล้วและสามารถเริ่มใช้งานได้

![หน้าขอบคุณหลังจากลงทะเบียน](/img/frontend/registration-thank-you.png)

หากจำเป็นต้องยืนยันที่อยู่อีเมล พวกเขาจะต้องไปที่กล่องจดหมายอีเมลของตนและคลิกลิงก์ยืนยัน เว็บไซต์ของพวกเขาจะไม่ถูกเปิดใช้งานหากที่อยู่อีเมลของพวกเขาไม่ได้รับการยืนยัน

หากพวกเขาลงทะเบียนในแผนแบบชำระเงิน หรือการยืนยันอีเมลไม่ใช่ข้อบังคับบนเครือข่ายของคุณ เว็บไซต์ของพวกเขาจะถูกเปิดใช้งานทันทีหลังจากการชำระเงิน และจะแสดงลิงก์ให้เข้าสู่ระบบ Dashboard

![เว็บไซต์เปิดใช้งานพร้อมลิงก์เพื่อเข้าสู่ระบบ Dashboard](/img/frontend/site-activated.png)

## การใช้ลิงก์ที่แชร์ได้: {#using-a-shareable-link}

กระบวนการลงทะเบียนโดยใช้ลิงก์ที่แชร์ได้นั้นโดยพื้นฐานแล้วเหมือนกับแบบฟอร์มลงทะเบียน ความแตกต่างเพียงอย่างเดียวคือเมื่อใช้ลิงก์ที่แชร์ได้ ลูกค้าของคุณสามารถมีผลิตภัณฑ์หรือแม่แบบเว็บไซต์ที่ถูกเลือกไว้ล่วงหน้าบนแบบฟอร์มชำระเงิน (ดูหัวข้อ การเลือกผลิตภัณฑ์และแม่แบบไว้ล่วงหน้าผ่านพารามิเตอร์ URL) หรืออาจมีการเพิ่มรหัสคูปองไว้แล้ว (ดูหัวข้อ การใช้พารามิเตอร์ URL)

กระบวนการลงทะเบียนจะเหมือนเดิม: พวกเขาจะต้องกรอกชื่อ ชื่อผู้ใช้ ที่อยู่อีเมล ชื่อเว็บไซต์และชื่อเรื่อง ฯลฯ แต่แผนหรือแม่แบบเว็บไซต์จะถูกเลือกไว้ล่วงหน้าให้แล้ว

### การลงทะเบียนโดยใช้การชำระเงินด้วยตนเอง: {#registering-using-manual-payments}

หากคุณไม่ต้องการใช้ PayPal, Stripe หรือช่องทางการชำระเงินอื่นใดที่ Ultimate Multisite หรือการผสานรวมส่วนเสริมของมันมีให้ คุณสามารถใช้การชำระเงินด้วยตนเองสำหรับลูกค้าของคุณได้ วิธีนี้ คุณสามารถสร้างใบแจ้งหนี้ให้พวกเขาชำระเงินผ่านตัวประมวลผลการชำระเงินที่คุณต้องการหลังจากที่พวกเขาลงทะเบียนบนเครือข่ายของคุณ

กระบวนการลงทะเบียนจะเหมือนกับด้านบนทุกประการ แต่บนหน้าลงทะเบียน ลูกค้าของคุณจะเห็นข้อความระบุว่าพวกเขาจะได้รับอีเมลพร้อมคำแนะนำเพิ่มเติมเพื่อดำเนินการชำระเงินให้เสร็จสมบูรณ์

![ข้อความการชำระเงินด้วยตนเองระหว่างการลงทะเบียน](/img/frontend/registration-manual-notice.png)

และหลังจากการลงทะเบียนเสร็จสมบูรณ์ พวกเขาจะเห็นคำแนะนำการชำระเงินที่คุณตั้งค่าไว้ (และจะได้รับทางอีเมลด้วย)

![คำแนะนำการชำระเงินที่แสดงหลังจากลงทะเบียน](/img/frontend/registration-payment-instructions.png)

สามารถเปลี่ยนคำแนะนำการชำระเงินได้ที่ **Ultimate Multisite > Settings > Payments** หลังจากเปิดตัวเลือกการชำระเงิน **Manual**:

![สวิตช์การชำระเงิน Manual พร้อมช่องคำแนะนำการชำระเงิน](/img/config/manual-gateway-settings.png)

หลังจากลูกค้าของคุณชำระเงินด้วยตนเองเสร็จสิ้นและส่งการยืนยันให้คุณ คุณต้อง **ยืนยันการชำระเงินด้วยตนเอง** เพื่อเปิดใช้งานสมาชิกภาพและเว็บไซต์ของลูกค้า

ในการทำเช่นนี้ ให้ไปที่ **Ultimate Multisite > Payments** และค้นหาการชำระเงินของลูกค้า ควรยังคงแสดงสถานะ **Pending**

![รายการการชำระเงินพร้อมการชำระเงินด้วยตนเองที่รอดำเนินการ](/img/admin/payments-list.png)

คลิกที่หมายเลขการชำระเงิน แล้วคุณจะสามารถเปลี่ยนสถานะเป็น **Completed** ได้

![หน้ารายละเอียดการชำระเงิน](/img/admin/payment-edit.png)

![การเปลี่ยนสถานะการชำระเงินเป็น Completed](/img/admin/payment-status-completed.png)

หลังจากเปลี่ยนสถานะเป็น **Completed** แล้ว คุณควรเห็นข้อความ **Activate membership** เปิดตัวเลือกนี้เป็น **เปิด** เพื่อเปิดใช้งานสมาชิกภาพและเว็บไซต์ที่เชื่อมโยงกับลูกค้ารายนี้ จากนั้นคลิก **Save Payment**

![สวิตช์ Activate membership และปุ่ม Save Payment](/img/admin/payment-activate-membership.png)

ตอนนี้ลูกค้าของคุณควรสามารถเข้าถึง Dashboard และฟีเจอร์ทั้งหมดที่พวกเขาสมัครใช้งานได้แล้ว.
