# การตั้งค่า Gratis AI Agent

หน้าจอ **การตั้งค่า → ขั้นสูง** ใน Gratis AI Agent มีการกำหนดค่าระดับผู้ดูแลระบบสำหรับการผสานการทำงานฝั่งแบ็กเอนด์ หน้านี้อธิบายการส่งต่อข้อเสนอแนะ คีย์ของผู้ให้บริการค้นหา การตั้งค่าบริการ Superdav แบบจัดการ การควบคุม Google Calendar การตั้งค่า SMS ของ TextBee และแฟล็กฟีเจอร์ทั่วทั้งเครือข่าย

## การเข้าถึงการตั้งค่า

1. ในผู้ดูแลระบบ WordPress ให้ไปที่ **Gratis AI Agent → การตั้งค่า**
2. คลิกแท็บ **ขั้นสูง**

## การกำหนดค่า Endpoint สำหรับข้อเสนอแนะ

Endpoint สำหรับข้อเสนอแนะรับคำขอ POST จาก AI agent เมื่อใดก็ตามที่ผู้ใช้ส่งข้อเสนอแนะผ่านปุ่มนิ้วหัวแม่มือลง แบนเนอร์แจ้งเตือนอัตโนมัติ หรือคำสั่ง `/report-issue`

| ฟิลด์ | คำอธิบาย |
|---|---|
| **URL ของ Endpoint สำหรับข้อเสนอแนะ** | URL ที่รับการส่งข้อเสนอแนะเป็นคำขอ HTTP POST พร้อมเนื้อหา JSON |
| **Feedback API Key** | bearer token ที่ส่งใน `Authorization` header ของคำขอข้อเสนอแนะแต่ละรายการ เว้นว่างไว้หาก endpoint ของคุณไม่ต้องการการยืนยันตัวตน |

### เพย์โหลด JSON ที่คาดหวัง

Endpoint สำหรับข้อเสนอแนะของคุณต้องยอมรับเนื้อหา JSON ที่มีฟิลด์ต่อไปนี้อย่างน้อย:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

อาจมีฟิลด์เพิ่มเติมในเพย์โหลด โดยขึ้นอยู่กับบริบทของการสนทนา

### ค่า `triage_category`

ชั้นคัดแยกของ AI จะกำหนดค่าหนึ่งในค่าต่อไปนี้ให้กับ `triage_category` ก่อนส่งต่อเพย์โหลด:

| ค่า | ความหมาย |
|---|---|
| `factual_error` | ผู้ช่วยให้ข้อมูลข้อเท็จจริงที่ไม่ถูกต้อง |
| `unhelpful_answer` | คำตอบถูกต้องในเชิงเทคนิค แต่ไม่เป็นประโยชน์ |
| `inappropriate_content` | คำตอบมีเนื้อหาที่ไม่ควรแสดงต่อผู้ใช้ |
| `other` | ข้อเสนอแนะไม่ตรงกับหมวดหมู่ที่รู้จัก |

### การยืนยันตัวตน

หาก endpoint ของคุณต้องการการยืนยันตัวตน ให้ตั้งค่าฟิลด์ **Feedback API Key** เป็น bearer token ของคุณ Agent จะส่ง:

```
Authorization: Bearer <your-api-key>
```

หากฟิลด์ **Feedback API Key** ว่างเปล่า จะไม่มีการส่ง `Authorization` header

### การปิดใช้งานการเก็บรวบรวมข้อเสนอแนะ

เว้นทั้งฟิลด์ **URL ของ Endpoint สำหรับข้อเสนอแนะ** และ **Feedback API Key** ไว้ให้ว่าง ปุ่มนิ้วหัวแม่มือลงและ UI สำหรับข้อเสนอแนะยังคงมองเห็นได้สำหรับผู้ใช้ แต่การส่งข้อเสนอแนะจะไม่ถูกส่งต่อไปยังบริการภายนอกใด ๆ

## Brave Search API Key

ในแท็บ **ขั้นสูง** เช่นกัน ฟิลด์ **Brave Search API Key** จะเปิดใช้งานความสามารถ [การค้นหาบนอินเทอร์เน็ต](../configuration/internet-search)

| ฟิลด์ | คำอธิบาย |
|---|---|
| **Brave Search API Key** | API key ของคุณจาก Dashboard สำหรับนักพัฒนาของ Brave Search จำเป็นต้องมีเพื่อเปิดใช้งานการค้นหาบนอินเทอร์เน็ตในผู้ช่วย AI |

ป้ายกำกับฟิลด์มีลิงก์ที่คลิกได้ไปยังหน้าสมัครใช้งาน Brave Search API เว้นว่างไว้เพื่อปิดใช้งานการค้นหาบนอินเทอร์เน็ต

ดู [การค้นหาบนอินเทอร์เน็ต](../configuration/internet-search) สำหรับเอกสารสำหรับผู้ใช้ปลายทางเกี่ยวกับฟีเจอร์นี้

## บริการ Superdav แบบจัดการ

Superdav AI Agent v1.18.0 เพิ่ม endpoint ของบริการ Superdav แบบจัดการและการจัดเตรียมการเชื่อมต่ออัตโนมัติสำหรับเว็บไซต์ที่รองรับ ใช้การควบคุมเหล่านี้เมื่อเว็บไซต์ของคุณควรเชื่อมต่อกับผู้ให้บริการที่โฮสต์ไว้แทน endpoint ของบริการที่กำหนดค่าด้วยตนเอง

| ฟิลด์ | คำอธิบาย |
|---|---|
| **บริการ Superdav แบบจัดการ** | เปิดใช้งานการเชื่อมต่อบริการ Superdav ที่โฮสต์ไว้สำหรับเว็บไซต์ที่รองรับ |
| **จัดเตรียมการเชื่อมต่อ** | เริ่มการจัดเตรียม endpoint และข้อมูลประจำตัวโดยอัตโนมัติ ใช้สิ่งนี้หลังจากยืนยันว่าเว็บไซต์ควรใช้ผู้ให้บริการแบบจัดการ |
| **Endpoint บริการ / สถานะการเชื่อมต่อ** | แสดง endpoint ปัจจุบันหรือสถานะการเชื่อมต่อหลังการจัดเตรียม |

หลังการจัดเตรียม ให้บันทึกการตั้งค่าและตรวจสอบสถานะการเชื่อมต่อก่อนพึ่งพาเวิร์กโฟลว์บริการแบบจัดการ หากการจัดเตรียมล้มเหลว ให้ตรวจสอบคำแนะนำการลองใหม่ที่แสดง และยืนยันว่าเว็บไซต์มีสิทธิ์ใช้ผู้ให้บริการที่โฮสต์ไว้

## การกำหนดค่า Google Calendar

เมื่อเปิดใช้งานฟีเจอร์ปฏิทินของ Superdav AI Agent v1.18.0 Agent สามารถอ่านปฏิทินที่กำหนดค่าไว้และรายละเอียดกิจกรรมได้ เครื่องมือปฏิทินมุ่งเน้นการอ่าน และมีประโยชน์สำหรับการแจ้งเตือนที่รับรู้ตารางเวลา การติดตามผู้เข้าร่วม และการจับคู่รายชื่อติดต่อ

| ฟิลด์ | คำอธิบาย |
|---|---|
| **ข้อมูลประจำตัว Google Calendar** | จัดเก็บข้อมูลประจำตัวหรือการเชื่อมต่อโทเค็นที่จำเป็นในการอ่านข้อมูลปฏิทิน |
| **การเลือกปฏิทิน** | จำกัดว่าปฏิทินที่กำหนดค่าไว้รายการใดที่ Agent สามารถตรวจสอบได้ |
| **สถานะการเชื่อมต่อปฏิทิน** | ยืนยันว่าข้อมูลประจำตัวปัจจุบันสามารถอ่านปฏิทินและกิจกรรมได้หรือไม่ |

จำกัดข้อมูลประจำตัวของปฏิทินไว้เฉพาะปฏิทินที่ Agent ต้องการ เชื่อมต่อใหม่หรือหมุนเวียนข้อมูลประจำตัวหากสถานะระบุว่าโทเค็นหมดอายุ

## การแจ้งเตือน SMS ของ TextBee

Superdav AI Agent v1.18.0 เพิ่ม TextBee เป็นผู้ให้บริการ SMS สำหรับเวิร์กโฟลว์การแจ้งเตือนที่กำหนดค่าไว้ การแจ้งเตือน SMS ควรจับคู่กับจุดตรวจการอนุมัติโดยมนุษย์สำหรับข้อความที่ละเอียดอ่อนหรือข้อความที่แสดงต่อผู้ใช้

| ฟิลด์ | คำอธิบาย |
|---|---|
| **TextBee API Key** | ยืนยันตัวตนคำขอไปยังผู้ให้บริการ SMS ของ TextBee |
| **อุปกรณ์ / ผู้ส่ง TextBee** | เลือกผู้ส่งหรืออุปกรณ์ของ TextBee ที่ใช้สำหรับข้อความขาออก เมื่อผู้ให้บริการกำหนดให้ต้องมี |
| **เปิดใช้งานการแจ้งเตือน SMS** | อนุญาตให้เวิร์กโฟลว์ที่ได้รับอนุมัติส่งการแจ้งเตือนทางข้อความ เว้นปิดใช้งานไว้เพื่อป้องกันการส่ง SMS |

ส่งข้อความทดสอบไปยังหมายเลขที่ผู้ดูแลระบบเป็นเจ้าของเท่านั้น จากนั้นยืนยันพฤติกรรมของจุดตรวจการอนุมัติก่อนเปิดใช้งานการแจ้งเตือนตามกำหนดเวลาหรือการแจ้งเตือนที่แสดงต่อผู้เข้าร่วม

## แฟล็กฟีเจอร์

แท็บ **การตั้งค่า → แฟล็กฟีเจอร์** ซึ่งเปิดตัวใน v1.9.0 เช่นกัน มีสวิตช์เปิดปิดสำหรับฟังก์ชันเสริม แต่ละแฟล็กจะเปิดใช้งานหรือปิดใช้งานทั่วทั้งเครือข่าย และในขณะนี้ยังไม่มีการแทนที่แยกตามเว็บไซต์

### การเข้าถึงแฟล็กฟีเจอร์

1. ในผู้ดูแลระบบ WordPress ให้ไปที่ **Gratis AI Agent → การตั้งค่า**
2. คลิกแท็บ **แฟล็กฟีเจอร์**

### แฟล็กควบคุมการเข้าถึง

| แฟล็ก | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| **จำกัดเฉพาะผู้ดูแลระบบ** | ปิด | เมื่อเปิดใช้งาน เฉพาะผู้ใช้ที่มีบทบาท `administrator` เท่านั้นที่สามารถเปิดแผงแชท AI Agent ได้ บทบาทอื่นทั้งหมดจะเห็นข้อความ "ติดต่อผู้ดูแลระบบของคุณ" แทน |
| **จำกัดเฉพาะผู้ดูแลเครือข่าย** | ปิด | เมื่อเปิดใช้งานบนเครือข่ายหลายไซต์ เฉพาะ Super Admins เท่านั้นที่สามารถใช้เอเจนต์ได้ ผู้ดูแลไซต์แต่ละรายจะถูกบล็อก มีลำดับความสำคัญเหนือ "จำกัดเฉพาะผู้ดูแลระบบ" หากเปิดใช้งานทั้งสองอย่าง |
| **อนุญาตการเข้าถึงของ Subscriber** | ปิด | เมื่อเปิดใช้งาน ผู้ใช้ที่มีบทบาท `subscriber` สามารถใช้อินเทอร์เฟซแชทได้ แต่จะถูกจำกัดให้มีความสามารถแบบอ่านอย่างเดียว (ไม่สามารถสร้างโพสต์หรือเปลี่ยนแปลงการตั้งค่าได้) |
| **ปิดใช้งานสำหรับผู้ที่ไม่ใช่สมาชิก** | ปิด | ผสานรวมกับสถานะสมาชิกของ Ultimate Multisite เมื่อเปิดใช้งาน แชทจะถูกซ่อนสำหรับไซต์ที่ไม่มีสมาชิกภาพที่ใช้งานอยู่ |

### แฟล็กการสร้างแบรนด์

| แฟล็ก | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| **ซ่อนส่วนท้าย "Powered by Gratis AI Agent"** | ปิด | ลบบรรทัดการระบุแบรนด์ที่แสดงที่ด้านล่างของวิดเจ็ตแชท แนะนำสำหรับการติดตั้งใช้งานแบบไวต์เลเบล |
| **ชื่อเอเจนต์แบบกำหนดเอง** | *(ว่าง)* | แทนที่ป้ายกำกับเริ่มต้น "Gratis AI Agent" ในส่วนหัวแชทและเมนูผู้ดูแลระบบด้วยชื่อผลิตภัณฑ์ของคุณเอง เว้นว่างไว้เพื่อใช้ค่าเริ่มต้น |
| **ซ่อนตัวเลือกเอเจนต์** | ปิด | เมื่อเปิดใช้งาน ผู้ใช้จะไม่สามารถสลับระหว่างเอเจนต์ในตัวทั้งห้าตัวได้ เอเจนต์ปัจจุบันจะถูกกำหนดคงที่ตามสิ่งที่ตั้งค่าเป็นค่าเริ่มต้นใน Settings → General |
| **ใช้ไอคอนไซต์เป็นอวตารแชท** | ปิด | แทนที่ไอคอน AI เริ่มต้นในส่วนหัววิดเจ็ตแชทด้วยไอคอนไซต์ WordPress (ตั้งค่าใต้ Appearance → Customize → Site Identity) |

### แฟล็กความปลอดภัยของระบบอัตโนมัติ

Superdav AI Agent v1.18.0 แนะนำจุดตรวจอนุมัติโดยมนุษย์และบันทึกการแจ้งเตือนสำหรับการทำงานอัตโนมัติที่ปลอดภัยยิ่งขึ้น การควบคุมเหล่านี้อาจปรากฏในแฟล็กฟีเจอร์หรือการตั้งค่าระบบอัตโนมัติขั้นสูง ขึ้นอยู่กับแพ็กเกจที่ติดตั้ง

| แฟล็ก | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| **ต้องได้รับการอนุมัติโดยมนุษย์** | แนะนำให้เปิด | หยุดระบบอัตโนมัติที่ละเอียดอ่อนไว้ชั่วคราวจนกว่าผู้ใช้ที่ได้รับอนุญาตจะตรวจสอบและยืนยันการดำเนินการที่เสนอ |
| **การลดการแจ้งเตือนซ้ำ** | เปิด | บันทึกการแจ้งเตือนที่ส่งแล้ว เพื่อให้การลองใหม่หรือการทำงานตามกำหนดเวลาไม่ส่งการแจ้งเตือนซ้ำ |
| **เปิดใช้งานเครื่องมือปฏิทิน** | ปิดจนกว่าจะกำหนดค่า | อนุญาตให้เอเจนต์อ่านปฏิทินและเหตุการณ์ Google ที่กำหนดค่าไว้ |
| **เปิดใช้งานการแจ้งเตือน SMS** | ปิดจนกว่าจะกำหนดค่า | อนุญาตให้เวิร์กโฟลว์ที่ได้รับอนุมัติส่งการแจ้งเตือน TextBee SMS หลังจากบันทึกข้อมูลประจำตัวแล้ว |

### การนำการเปลี่ยนแปลงไปใช้

คลิก **บันทึกการตั้งค่า** หลังจากสลับแฟล็กใด ๆ การเปลี่ยนแปลงจะมีผลทันที — ไม่จำเป็นต้องล้างแคชหรือเปิดใช้งาน plugin ใหม่
