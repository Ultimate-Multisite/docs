# การหมดอายุสมาชิกและการบล็อกเว็บไซต์

คู่มือนี้จะอธิบายว่า Ultimate Multisite จัดการกับการหมดอายุสมาชิก การสิ้นสุดช่วงทดลองใช้ และการบล็อกเว็บไซต์ที่ส่วนหน้า (frontend) อย่างไร โดยครอบคลุมวงจรชีวิตของสมาชิกตั้งแต่สถานะใช้งานได้จนถึงหมดอายุ รวมถึงการตั้งค่าที่ควบคุมว่าเว็บไซต์จะถูกบล็อกหรือไม่ และสิ่งที่ควรตรวจสอบเมื่อเว็บไซต์ยังคงเข้าถึงได้แม้ว่าสมาชิกจะหมดอายุแล้ว

## วงจรชีวิตสถานะสมาชิก (Membership Status Lifecycle) {#membership-status-lifecycle}

สมาชิกทุกคนใน Ultimate Multisite จะมีสถานะใดสถานะหนึ่งดังต่อไปนี้:

:::note สมาชิกฟรีมีอายุการใช้งานตลอดชีพ
สมาชิกฟรีจะไม่หมดอายุโดยอัตโนมัติ Ultimate Multisite ถือว่าเป็นการเข้าถึงตลอดชีพ ดังนั้นจึงไม่รวมอยู่ในขั้นตอนการตรวจสอบการหมดอายุ เว้นแต่ผู้ดูแลระบบจะเปลี่ยนสถานะหรือย้ายลูกค้าไปยังผลิตภัณฑ์อื่น
:::

| สถานะ | ความหมาย |
|---|---|
| **Pending** | กำลังรอการชำระเงินครั้งแรกหรือการยืนยันอีเมล |
| **Trialing** | ช่วงทดลองใช้ที่ใช้งานอยู่ ยังไม่ได้มีการเก็บเงิน |
| **Active** | ชำระเงินแล้วและยังใช้งานได้ |
| **On Hold** | กำลังรอการชำระเงินต่ออายุ (มีการสร้างใบแจ้งหนี้ รอการชำระเงิน) |
| **Expired** | พ้นวันหมดอายุและช่วงผ่อนผันโดยไม่ได้ต่ออายุ |
| **Cancelled** | ถูกยกเลิกอย่างชัดเจนโดยลูกค้าหรือผู้ดูแลระบบ |

### สมาชิกเปลี่ยนเป็นสถานะหมดอายุได้อย่างไร {#how-memberships-transition-to-expired}

Ultimate Multisite จะทำการตรวจสอบเบื้องหลัง **ทุกชั่วโมง** เพื่อค้นหาสมาชิกที่ควรถูกทำเครื่องหมายว่าหมดอายุ การตรวจสอบนี้ใช้ [Action Scheduler](https://actionscheduler.org/) (ไม่ใช่ WP-Cron โดยตรง) และทำงานในรูปแบบ action ที่กำหนดเวลาไว้ชื่อ `wu_membership_check`

การตรวจสอบการหมดอายุมี **ช่วงผ่อนผันในตัว (built-in grace period) 3 วัน** เป็นค่าเริ่มต้น สมาชิกจะไม่ถูกทำเครื่องหมายว่า `expired` จนกว่าจะผ่านไป 3 วันหลังจาก `date_expiration` ซึ่งช่วยให้ลูกค้ามีเวลาชำระเงินที่ล่าช้าก่อนที่สถานะจะเปลี่ยนแปลง

:::info
ช่วงผ่อนผันการหมดอายุ 3 วันนี้แยกต่างหากจากการตั้งค่าช่วงผ่อนผันการบล็อกส่วนหน้า (Frontend Block Grace Period) ที่อธิบายไว้ด้านล่าง ช่วงผ่อนผันการหมดอายุจะควบคุมว่า **สถานะจะเปลี่ยน** จาก active/on-hold เป็น expired เมื่อใด ส่วนช่วงผ่อนผันการบล็อกส่วนหน้าจะควบคุมว่า **เว็บไซต์จะถูกบล็อก** เมื่อใดหลังจากที่สถานะได้เปลี่ยนไปแล้ว
:::

#### สมาชิกที่ต่ออายุอัตโนมัติ เทียบกับ สมาชิกที่ไม่ต่ออายุอัตโนมัติ {#auto-renewing-vs-non-auto-renewing-memberships}

ความแตกต่างนี้มีความสำคัญอย่างยิ่งต่อการทำความเข้าใจพฤติกรรมการหมดอายุ:

- **สมาชิกที่ไม่ต่ออายุอัตโนมัติ** (`auto_renew = false`): งาน cron รายชั่วโมงจะจัดการวงจรชีวิตทั้งหมด — มันจะสร้างการชำระเงินต่ออายุ เปลี่ยนสถานะสมาชิกเป็น on-hold และในที่สุดก็จะทำเครื่องหมายว่าหมดอายุหากไม่ได้รับการชำระเงิน

- **สมาชิกที่ต่ออายุอัตโนมัติ** (`auto_renew = true`): การตรวจสอบการหมดอายุของ cron **จะข้ามสมาชิกเหล่านี้ไปโดยสิ้นเชิง** คาดว่าเกตเวย์การชำระเงิน (Stripe, PayPal ฯลฯ) จะแจ้งให้ Ultimate Multisite ทราบผ่าน webhooks เมื่อการสมัครสมาชิกล้มเหลวหรือถูกยกเลิก หากไม่ได้รับ webhook — เนื่องจาก endpoint ที่ตั้งค่าผิดพลาด การขัดข้องของเกตเวย์ หรือการยกเลิกการสมัครสมาชิกนอกระบบ — สมาชิกอาจยังคงสถานะ `active` ได้อย่างไม่มีกำหนดแม้ว่าวันหมดอายุจะผ่านไปแล้วก็ตาม

### ช่วงทดลองใช้สิ้นสุดลงอย่างไร {#how-trials-end}

เมื่อช่วงทดลองใช้ของสมาชิกสิ้นสุดลง ระบบจะ:

1. สร้างการชำระเงินต่ออายุที่รอดำเนินการด้วยจำนวนเงินเต็มของการสมัครสมาชิก
2. เปลี่ยนสถานะสมาชิกจาก `trialing` เป็น `on-hold`
3. ส่งอีเมลแจ้งการชำระเงินต่ออายุไปยังลูกค้า

กระบวนการนี้ทำงานตามตารางเวลาชั่วโมงเดียวกับที่ตรวจสอบการหมดอายุปกติ แต่ **เฉพาะสำหรับสมาชิกที่ไม่ต่ออายุอัตโนมัติเท่านั้น** สำหรับการทดลองใช้แบบต่ออายุอัตโนมัติ เกตเวย์การชำระเงินจะเป็นผู้จัดการการเปลี่ยนจากช่วงทดลองใช้เป็นการสมัครสมาชิกแบบชำระเงิน

## การบล็อกการเข้าถึงส่วนหน้า (Block Frontend Access) {#block-frontend-access}

โดยค่าเริ่มต้น เมื่อสมาชิกหมดอายุหรืออยู่ในสถานะ on hold **จะมีการจำกัดเฉพาะแดชบอร์ด wp-admin เท่านั้น** ส่วนหน้าสาธารณะของเว็บไซต์ยังคงเข้าถึงได้สำหรับผู้เยี่ยมชม หากต้องการบล็อกการเข้าถึงสาธารณะด้วย คุณต้องเปิดใช้งานการตั้งค่า **Block Frontend Access**

### การตั้งค่า (Configuring the Setting) {#configuring-the-setting}

ไปที่ **Ultimate Multisite > Settings > Memberships** และเปิดใช้งาน **Block Frontend Access**

![Block Frontend Access settings showing the toggle enabled, a grace period of 7 days, and a Frontend Block Page selection](/img/config/settings-membership-block-frontend.png)

นี่คือมุมมองเต็มของหน้าการตั้งค่าสมาชิก:

![Membership settings full page](/img/config/settings-membership-full.png)

การตั้งค่าที่เกี่ยวข้องสามรายการควบคุมพฤติกรรมนี้:

| การตั้งค่า | คำอธิบาย | ค่าเริ่มต้น |
|---|---|---|
| **Block Frontend Access** | สวิตช์หลัก เมื่อเปิดใช้งาน จะบล็อกส่วนหน้าสาธารณะของเว็บไซต์เครือข่ายเมื่อสมาชิกไม่ใช้งานแล้ว | ปิด |
| **Frontend Block Grace Period** | จำนวนวันที่ต้องรอหลังจากที่สมาชิกไม่ใช้งานก่อนที่จะบล็อก ตั้งค่าเป็น `0` เพื่อบล็อกทันที | 0 |
| **Frontend Block Page** | หน้าบนเว็บไซต์หลักที่จะเปลี่ยนเส้นทางผู้เยี่ยมชมไปเมื่อเว็บไซต์ถูกบล็อก หากไม่ได้ตั้งค่า ผู้เยี่ยมชมจะเห็นข้อความ "Site not available" ทั่วไป | ไม่มี |

### ผู้เยี่ยมชมเห็นอะไรเมื่อเว็บไซต์ถูกบล็อก {#what-visitors-see-when-a-site-is-blocked}

เมื่อการเข้าถึงส่วนหน้าถูกบล็อก ผู้เยี่ยมชมเว็บไซต์จะ:

1. **ถูกเปลี่ยนเส้นทาง** ไปยังหน้าที่คุณเลือกใน **Frontend Block Page** (หากมีการตั้งค่า) หรือ
2. **เห็นข้อความเริ่มต้น**: "This site is not available at the moment." พร้อมลิงก์ไปยังหน้าเข้าสู่ระบบสำหรับผู้ดูแลระบบเว็บไซต์

ผู้ดูแลระบบเว็บไซต์ยังคงสามารถเข้าสู่ระบบได้ — หน้าเข้าสู่ระบบจะไม่ถูกบล็อก

### อะไรถูกบล็อกและเมื่อไหร่ {#what-gets-blocked-and-when}

พฤติกรรมการบล็อกขึ้นอยู่กับสถานะสมาชิก:

| สถานะสมาชิก | บล็อกส่วนหน้าหรือไม่? | มีช่วงผ่อนผันหรือไม่? |
|---|---|---|
| Active | ไม่ | -- |
| Trialing | **ไม่** (ดูหมายเหตุด้านล่าง) | -- |
| On Hold | ถือว่าใช้งานได้ -- ไม่ถูกบล็อก | -- |
| Expired | **ใช่** หากเปิดใช้งาน Block Frontend Access | ใช่ |
| Cancelled | **ใช่** เสมอ (ไม่ว่าการตั้งค่าจะเป็นอย่างไร) | **ไม่** -- ถูกบล็อกทันที |
| Pending | ไม่ถูกบล็อกผ่านการตรวจสอบสมาชิก | -- |

:::warning สมาชิกช่วงทดลองใช้จะไม่ถูกบล็อก
แม้ว่าช่วงทดลองใช้จะสิ้นสุดลง สมาชิกที่มีสถานะ `trialing` จะ **ไม่** ถูกบล็อกที่ส่วนหน้า การทดลองใช้จะต้องเปลี่ยนสถานะเป็นสถานะอื่นก่อน (โดยทั่วไปคือ `on-hold` ผ่าน cron job จากนั้น `expired` หากไม่ชำระเงิน) หากคุณเห็นสมาชิกช่วงทดลองใช้ที่ยังไม่ได้เปลี่ยนสถานะ ให้ตรวจสอบส่วนการแก้ไขปัญหาด้านล่าง
:::

:::note สมาชิกที่ถูกยกเลิกจะข้ามช่วงผ่อนผัน
สมาชิกที่ถูกยกเลิกจะถูกบล็อกเสมอเมื่อวันหมดอายุผ่านไป ไม่ว่า Block Frontend Access จะเปิดใช้งานอยู่หรือไม่ก็ตาม ช่วงผ่อนผันการบล็อกส่วนหน้าจะไม่ใช้กับสมาชิกที่ถูกยกเลิก
:::

## การแก้ไขปัญหา: เว็บไซต์ยังคงเข้าถึงได้หลังหมดอายุ {#troubleshooting-sites-remaining-accessible-after-expiration}

หากเว็บไซต์ยังคงเข้าถึงได้โดยสาธารณะหลังจากที่สมาชิกหมดอายุ ให้ดำเนินการตรวจสอบเหล่านี้ตามลำดับ:

### 1. ตรวจสอบว่าได้เปิดใช้งานการตั้งค่า Block Frontend Access แล้ว {#1-verify-the-block-frontend-access-setting-is-enabled}

ไปที่ **Ultimate Multisite > Settings > Memberships** และยืนยันว่าสวิตช์ **Block Frontend Access** เปิดอยู่ การตั้งค่านี้ **ปิดโดยค่าเริ่มต้น** ซึ่งหมายความว่าเมื่อสมาชิกไม่ใช้งาน จะมีการจำกัดเฉพาะ wp-admin เท่านั้น

### 2. ตรวจสอบ Frontend Block Grace Period {#2-check-the-frontend-block-grace-period}

ในหน้าการตั้งค่าเดียวกัน ให้ตรวจสอบค่า **Frontend Block Grace Period** หากตั้งค่าเป็น 7 วัน ตัวอย่างเช่น ส่วนหน้าจะไม่ถูกบล็อกจนกว่าจะผ่านไป 7 วันหลังจากวันหมดอายุของสมาชิก แม้ว่าสถานะสมาชิกจะ `expired` ไปแล้วก็ตาม

ให้ตั้งค่าเป็น `0` หากคุณต้องการบล็อกทันทีหลังจากที่สมาชิกไม่ใช้งาน

### 3. ยืนยันว่าสถานะสมาชิกได้เปลี่ยนไปจริง {#3-confirm-the-membership-status-has-actually-changed}

ไปที่ **Ultimate Multisite > Memberships** และตรวจสอบสถานะของสมาชิกที่ได้รับผลกระทบ หากยังคงแสดงเป็น `active` แม้ว่าวันหมดอายุจะผ่านไปแล้ว นั่นหมายความว่าการเปลี่ยนสถานะยังไม่เกิดขึ้น สาเหตุทั่วไป:

- **สมาชิกนั้นต่ออายุอัตโนมัติ**: ตรวจสอบฟิลด์ `auto_renew` ในหน้าแก้ไขสมาชิก หากเปิดใช้งานการต่ออายุอัตโนมัติ cron การหมดอายุจะข้ามสมาชิกนี้ไป — มันอาศัยเกตเวย์การชำระเงินในการรายงานความล้มเหลว ตรวจสอบแดชบอร์ดเกตเวย์ของคุณ (Stripe, PayPal) เพื่อยืนยันว่าสถานะการสมัครสมาชิกตรงกับที่ Ultimate Multisite แสดงหรือไม่

- **cron job ยังไม่ได้ทำงาน**: ดูขั้นตอนถัดไป

### 4. ตรวจสอบว่า Action Scheduler กำลังทำงานอยู่ {#4-verify-action-scheduler-is-running}

Ultimate Multisite ใช้ Action Scheduler สำหรับงาน cron job ของมัน ไปที่ **Tools > Scheduled Actions** ใน network admin และมองหา:

- **`wu_membership_check`** -- สิ่งนี้ควรปรากฏเป็น action ที่เกิดขึ้นซ้ำทุกชั่วโมง หากไม่มี แสดงว่าการตรวจสอบสมาชิกไม่ได้ถูกกำหนดเวลาไว้
- **`wu_async_mark_membership_as_expired`** -- เหล่านี้เป็นงานรายบุคคลเพื่อทำเครื่องหมายสมาชิกให้หมดอายุ หากคุณเห็น action ที่ล้มเหลวที่นี่ อาจมีข้อความแสดงข้อผิดพลาดที่อธิบายว่าทำไม

หากคุณเห็น action ที่ค้างหรือล้มเหลว คุณอาจมีปัญหา Action Scheduler สาเหตุทั่วไป:

- **`DISABLE_WP_CRON` ถูกตั้งค่าเป็น `true`** ใน `wp-config.php` โดยไม่มีการแทนที่ cron ระดับระบบ
- **ปริมาณการเข้าชมเว็บไซต์ต่ำ** -- WP-Cron จะทำงานก็ต่อเมื่อมีคนเข้าชมเว็บไซต์เท่านั้น

เพื่อให้แน่ใจว่า cron ทำงานได้อย่างน่าเชื่อถือ ให้ตั้งค่าระบบ cron job:

```bash
# Run every 5 minutes via wget
*/5 * * * * wget -q -O /dev/null "https://your-network-url.com/wp-cron.php?doing_wp_cron"

# Or via WP-CLI
*/5 * * * * cd /path/to/wordpress && wp cron event run --due-now --url=https://your-network-url.com
```

### 5. ตรวจสอบปัญหา Webhook ของเกตเวย์ (สำหรับสมาชิกที่ต่ออายุอัตโนมัติ) {#5-check-for-gateway-webhook-issues-auto-renewing-memberships}

หากสมาชิกต่ออายุอัตโนมัติ และการสมัครสมาชิกของเกตเวย์ถูกยกเลิกหรือล้มเหลว แต่ Ultimate Multisite ยังคงแสดงว่าเป็น `active`:

- **Stripe**: ไปที่ Stripe Dashboard > Customers และตรวจสอบสถานะการสมัครสมาชิก จากนั้นตรวจสอบว่า endpoint webhook เปิดใช้งานอยู่ภายใต้ Developers > Webhooks endpoint ควรชี้ไปยังเว็บไซต์ของคุณและแสดงการส่งที่สำเร็จ
- **PayPal**: ตรวจสอบสถานะการสมัครสมาชิกในบัญชีธุรกิจ PayPal ของคุณ และตรวจสอบการส่ง IPN/webhook

หากเกตเวย์แสดงว่าการสมัครสมาชิกถูกยกเลิก แต่ Ultimate Multisite ไม่ได้แสดง คุณอาจสูญเสียการแจ้งเตือน webhook คุณสามารถเปลี่ยนสถานะสมาชิกด้วยตนเองใน **Ultimate Multisite > Memberships > [Edit Membership]**

### 6. ตรวจสอบช่วงผ่อนผันการหมดอายุ (ระดับ Cron) {#6-check-the-expiration-grace-period-cron-level}

การตรวจสอบ cron มีช่วงผ่อนผันของตัวเอง (ค่าเริ่มต้น: 3 วัน) ก่อนที่จะทำเครื่องหมายสมาชิกว่าหมดอายุ ซึ่งแยกต่างหากจากช่วงผ่อนผันการบล็อกส่วนหน้า เวลาทั้งหมดก่อนที่เว็บไซต์จะถูกบล็อกอาจเป็น:

**ช่วงผ่อนผันการหมดอายุ (3 วัน)** + **ช่วงผ่อนผันการบล็อกส่วนหน้า (ค่าที่คุณตั้ง)** = ความล่าช้าทั้งหมด

ตัวอย่างเช่น ด้วยการตั้งค่าเริ่มต้นและช่วงผ่อนผันส่วนหน้า 7 วัน อาจใช้เวลานานถึง 10 วันหลังจาก `date_expiration` ก่อนที่เว็บไซต์จะถูกบล็อกจริง

### 7. ทำการหมดอายุสมาชิกด้วยตนเอง {#7-manually-expire-a-membership}

หากคุณต้องการบล็อกเว็บไซต์ทันทีโดยไม่ต้องรอรอบ cron คุณสามารถเปลี่ยนสถานะสมาชิกด้วยตนเองได้:

1. ไปที่ **Ultimate Multisite > Memberships**
2. คลิกที่สมาชิกที่ได้รับผลกระทบ
3. เปลี่ยนสถานะเป็น **Expired** หรือ **Cancelled**
4. คลิก **Save**

การบล็อกส่วนหน้าจะมีผลในการโหลดหน้าถัดไป (ขึ้นอยู่กับช่วงผ่อนผันการบล็อกส่วนหน้าสำหรับสมาชิกที่หมดอายุ หรือทันทีสำหรับสมาชิกที่ถูกยกเลิก)

## สรุป {#summary}

ไทม์ไลน์ทั้งหมดตั้งแต่วันหมดอายุจนถึงการบล็อกเว็บไซต์:

```text
date_expiration ผ่านไป
       |
       v
  [ช่วงผ่อนผัน cron 3 วัน]     <-- สมาชิกยังคงแสดงเป็น active/on-hold
       |
       v
  Cron ทำเครื่องหมายสมาชิกเป็น "expired"
       |
       v
  [ช่วงผ่อนผันการบล็อกส่วนหน้า]  <-- กำหนดค่าใน Settings > Memberships
       |
       v
  ส่วนหน้าเว็บไซต์ถูกบล็อก
```

สำหรับสมาชิกที่ถูกยกเลิก เส้นทางจะสั้นกว่า:

```text
  สมาชิกถูกยกเลิก
       |
       v
  date_expiration ผ่านไป (ไม่มีช่วงผ่อนผัน)
       |
       v
  ส่วนหน้าเว็บไซต์ถูกบล็อกทันที
```

## เอกสารอ้างอิงสำหรับนักพัฒนา (Developer Reference) {#developer-reference}

hook และ filter ต่อไปนี้ช่วยให้คุณสามารถปรับแต่งพฤติกรรมการหมดอายุและการบล็อกได้:

| Hook/Filter | คำอธิบาย |
|---|---|
| `wu_membership_grace_period_days` | กรองจำนวนวันของช่วงผ่อนผันก่อนที่จะทำเครื่องหมายสมาชิกว่าหมดอายุ (ค่าเริ่มต้น: 3) |
| `wu_schedule_membership_check_interval` | กรองช่วงเวลาของการตรวจสอบสมาชิก (ค่าเริ่มต้น: 1 ชั่วโมง) |
| `wu_membership_renewal_days_before_expiring` | กรองจำนวนวันที่ต้องทำก่อนหมดอายุเพื่อสร้างการชำระเงินต่ออายุ (ค่าเริ่มต้น: 3) |
| `wu_blocked_site_reactivation_url` | กรองเพื่อระบุ URL การเปิดใช้งานใหม่ที่กำหนดเองเมื่อเว็บไซต์ถูกบล็อก |
| `wu_membership_is_active` | กรองว่าสมาชิกถือว่าใช้งานได้หรือไม่ |
| `wu_membership_expired_check_query_params` | กรองพารามิเตอร์คิวรีที่ใช้ค้นหาสมาชิกที่หมดอายุ |
| `wu_membership_trial_check_query_params` | กรองพารามิเตอร์คิวรีที่ใช้ค้นหาช่วงทดลองใช้ที่หมดอายุ |
