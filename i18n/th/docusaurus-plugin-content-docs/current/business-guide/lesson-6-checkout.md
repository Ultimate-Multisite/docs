# บทที่ 6: ประสบการณ์การสมัครสมาชิก

ขั้นตอนการชำระเงินคือจุดที่ความสนใจจะเปลี่ยนเป็นรายได้ ถ้าขั้นตอนการสมัครสมาชิกสับสนหรือดูทั่วไปเกินไป ก็จะทำให้ลูกค้าหนีไปได้ แต่ถ้าเราออกแบบขั้นตอนให้เจาะจงกับกลุ่มเป้าหมายและรู้สึกง่ายดาย ก็จะช่วยเปลี่ยนพวกเขาให้กลายเป็นลูกค้าที่จ่ายเงินได้

## จุดที่เราทำค้างไว้

FitSite มีเทมเพลตและแผนบริการที่ตั้งค่าไว้แล้ว ตอนนี้เราจะมาสร้างประสบการณ์การลงทะเบียนและชำระเงิน เพื่อเปลี่ยนเจ้าของสตูดิโอฟิตเนสให้กลายเป็นลูกค้าที่จ่ายเงินได้จริง

## ทำความเข้าใจขั้นตอนการลงทะเบียน

ฟอร์มการชำระเงินของ Ultimate Multisite เป็นฟอร์มแบบหลายขั้นตอนที่ปรับแต่งได้เต็มที่ สำหรับ FitSite เราต้องการขั้นตอนที่:

1. ให้ความรู้สึกว่าถูกสร้างมาเพื่อธุรกิจฟิตเนสโดยเฉพาะ
2. เก็บข้อมูลเท่าที่จำเป็นเท่านั้น
3. ทำให้ลูกค้าสามารถเข้าถึงเว็บไซต์ที่ใช้งานได้เร็วที่สุด

ดู [The Registration Flow](/user-guide/configuration/the-registration-flow) สำหรับข้อมูลอ้างอิงทางเทคนิคฉบับเต็ม

## การออกแบบหน้าชำระเงินสำหรับ FitSite

ไปที่ **Ultimate Multisite > Checkout Forms** แล้วสร้างฟอร์มใหม่

### ขั้นตอนที่ 1: การเลือกแผนบริการ (Plan Selection)

สิ่งแรกที่เจ้าของสตูดิโอฟิตเนสควรเห็นคือแผนบริการต่างๆ โดยนำเสนอในภาษาที่พวกเขาเข้าใจ

- เพิ่มฟิลด์ **Pricing Table**
- ตั้งค่าให้แสดงแผนบริการ FitSite ทั้งสามแผน
- คำอธิบายแผนบริการที่คุณเขียนไว้ในบทที่ 5 จะปรากฏที่นี่ — ตรวจสอบให้แน่ใจว่าคำอธิบายนั้นพูดถึงความต้องการทางธุรกิจฟิตเนส ไม่ใช่แค่คุณสมบัติทางเทคนิค

:::tip ภาษาเฉพาะกลุ่มมีความสำคัญ
"พื้นที่เก็บข้อมูล 1 GB" ไม่มีความหมายอะไรสำหรับเจ้าของยิม แต่ "ทุกสิ่งที่คุณต้องการสำหรับเว็บไซต์สตูดิโอระดับมืออาชีพ" มีความหมายทุกอย่าง จงเขียนคำอธิบายแผนบริการด้วยภาษาของลูกค้าคุณ
:::

### ขั้นตอนที่ 2: การเลือกเทมเพลต (Template Selection)

หลังจากเลือกแผนบริการแล้ว ลูกค้าจะเลือกเทมเพลตเริ่มต้น

- เพิ่มฟิลด์ **Template Selection**
- เทมเพลตที่มีให้เลือกจะถูกกรองตามแผนที่พวกเขาเลือก (ซึ่งตั้งค่าไว้ในบทที่ 5)
- แต่ละเทมเพลตควรมีรูปภาพตัวอย่างที่แสดงการออกแบบที่เกี่ยวข้องกับฟิตเนสโดยเฉพาะ

### ขั้นตอนที่ 3: การสร้างบัญชี (Account Creation)

ให้เก็บข้อมูลให้น้อยที่สุด เก็บเพียง:

- ที่อยู่อีเมล (Email address)
- รหัสผ่าน (Password)
- ชื่อสตูดิโอ/ธุรกิจ (Studio/business name) (สิ่งนี้จะกลายเป็นชื่อเว็บไซต์ของพวกเขา)

อย่าขอข้อมูลที่คุณไม่จำเป็นต้องใช้ในการสมัครสมาชิก ทุกฟิลด์ที่เพิ่มเข้ามาจะลดอัตราการเปลี่ยนเป็นลูกค้า (conversions)

### ขั้นตอนที่ 4: การตั้งค่าเว็บไซต์ (Site Setup)

- **ชื่อเว็บไซต์ (Site title)**: ดึงข้อมูลล่วงหน้าจากชื่อสตูดิโอที่กรอกในขั้นตอนที่ 3
- **URL เว็บไซต์ (Site URL)**: สร้างอัตโนมัติจากชื่อสตูดิโอ (เช่น `ironworks.fitsite.com`)

### ขั้นตอนที่ 5: การชำระเงิน (Payment)

- เพิ่มฟิลด์ **Payment**
- ตั้งค่า Payment gateway ของคุณ (แนะนำ [Stripe](/user-guide/payment-gateways/stripe) สำหรับการเรียกเก็บเงินแบบสมัครสมาชิก)
- หากคุณสร้าง Order bumps ในบทที่ 5 ให้เพิ่มฟิลด์ **Order Bump** ก่อนขั้นตอนการชำระเงิน

### ขั้นตอนที่ 6: การยืนยัน (Confirmation)

- ปรับแต่งข้อความยืนยันด้วยภาษาที่เจาะจงกับฟิตเนส
- ตัวอย่าง: "เว็บไซต์สตูดิโอฟิตเนสของคุณกำลังถูกสร้างขึ้น คุณจะถูกเปลี่ยนเส้นทางไปยังเว็บไซต์ใหม่ในอีกไม่กี่วินาที"

## การเพิ่มตัวสลับการเลือกช่วงเวลา (Period Selection Toggle)

หากคุณตั้งค่าความแตกต่างของราคาในแผนบริการ (รายเดือนเทียบกับรายปี) ให้เพิ่มฟิลด์ **Period Selection** ในหน้าชำระเงิน เพื่อให้ลูกค้าสามารถสลับระหว่างช่วงเวลาการเรียกเก็บเงิน ดูคำแนะนำได้ที่ [Checkout Forms](/user-guide/configuration/checkout-forms#adding-a-period-selection-toggle)

## การตั้งค่าการชำระเงิน (Configuring Payment)

หากคุณยังไม่ได้ตั้งค่า payment gateway:

1. ไปที่ **Ultimate Multisite > Settings > Payment Gateways**
2. ทำตาม [คู่มือการตั้งค่า Stripe](/user-guide/payment-gateways/stripe) หรือ gateway ที่คุณต้องการ
3. ทดสอบขั้นตอนการชำระเงินทั้งหมดด้วยการชำระเงินแบบทดสอบ (test payment)

ดู [Getting Paid](/user-guide/payment-gateways/getting-paid) สำหรับรายละเอียดเกี่ยวกับวิธีการที่การชำระเงินจะเข้าบัญชีของคุณ

## การทดสอบขั้นตอน (Testing the Flow)

ก่อนจะไปต่อ ให้ทำการทดสอบการสมัครสมาชิกแบบเต็มรูปแบบ:

1. เปิดฟอร์มชำระเงินในหน้าต่างเบราว์เซอร์แบบ Incognito/Private
2. เลือกแผนบริการ
3. เลือกเทมเพลต
4. สร้างบัญชี
5. ชำระเงินให้เสร็จสมบูรณ์ (ใช้โหมดทดสอบ)
6. ตรวจสอบว่าเว็บไซต์ถูกสร้างขึ้นด้วยเทมเพลตที่ถูกต้อง

ตรวจสอบให้แน่ใจว่า:

- [ ] คำอธิบายแผนบริการชัดเจนและเจาะจงกลุ่มเป้าหมาย
- [ ] ตัวอย่างเทมเพลตแสดงการออกแบบที่เหมาะสมกับฟิตเนส
- [ ] URL เว็บไซต์สร้างขึ้นอย่างถูกต้องจากชื่อสตูดิโอ
- [ ] การชำระเงินดำเนินการได้สำเร็จ
- [ ] ลูกค้าเข้าสู่เว็บไซต์ที่ใช้งานได้ด้วยเทมเพลตที่เลือก
- [ ] อีเมลยืนยันใช้ภาษาที่เจาะจงกับฟิตเนส

## ภาพรวมเครือข่าย FitSite จนถึงตอนนี้

```
FitSite Network
├── WordPress Multisite (subdomain mode)
├── Ultimate Multisite (configured)
├── Hosting with wildcard SSL + domain mapping
├── Site Templates (Studio Essential, Gym Pro, Fitness Chain)
├── Products (Starter, Growth, Pro + Order Bumps)
├── Checkout Flow
│   ├── Plan selection with niche-specific descriptions
│   ├── Template selection with fitness previews
│   ├── Minimal account creation
│   ├── Payment via Stripe
│   └── Fitness-specific confirmation
└── Ready for branding (next lesson)
```

## สิ่งที่เราสร้างในบทนี้

- **ฟอร์มชำระเงินแบบหลายขั้นตอน** ที่ปรับให้เหมาะกับเจ้าของสตูดิโอฟิตเนส
- **ภาษาที่เจาะจงกลุ่มเป้าหมาย** ตลอดขั้นตอนการสมัครสมาชิก
- **ลดความยุ่งยาก** — มีเพียงฟิลด์ที่จำเป็นเท่านั้น และมีเส้นทางที่รวดเร็วไปยังเว็บไซต์ที่ใช้งานได้
- **การเชื่อมต่อระบบชำระเงิน** พร้อมการตรวจสอบด้วยโหมดทดสอบ
- **ขั้นตอนแบบ End-to-end ที่ผ่านการทดสอบ** ตั้งแต่การเลือกแผนบริการจนถึงเว็บไซต์ที่ใช้งานได้
