# Монархлик Тенент Операциялари

Ultimate Multisite: Multi-Tenancy 1.2.0 монархлик тенентлар учун операцион воситаларни қўшилди: бу субсайтлар ўз базаси, файлосининг асосий кўриниши ва маршрутизация контексти билан ишлайди, лекин улар тармоқ администраторидан кўриниб туришида бўлади.

Стандарт субсайтни монархлик инфратузилмасига ўтказиш ёки изолятланган мижоз сайтларини бошқариш вазиятларида бу саҳани фойдаланг.

## Администраторлар учун ўзгаришлари

- **Татбиқсиз тенент автологини** — Тармоқ администраторлари узоқ муддатли умумий сессия ҳолатига таянмай, монархлик тенентидан фойдаланиш мумкин. SSO токени мақсадга мослаштирилган, манбага белгиланган, қайта ишлашдан ҳимояланган ва қисқа вақтни ўтган чекингга солиш билан чекланган.
- **Монархликка эътиборли маршрутизация** — Қалган изолятланган тармоқлар ва монархлик тенентлари бир хил сайт маршрутизатор йўлидан ҳалланади, бу эски ва янги изолятланган ўрнатилмалар орасидаги бошланғич фарқларни камарат.
- **Тартибга солинган миграция ҳолати** — Миграцияни текшириш чеклари тенентнинг тартибга созилиши, базаси ёзувчи рухсати, кутуқ (queue) вазиятлари ва эски жадваллар мавжудлигини текширади, бундан кейинтенент тўлиқ деб ҳисобланади.
- **Хавфсизроқ ютиш** — Монархлик тенентидан фойдаланиш энди тенент маълумотларини тоза чизиб олиб ташлайди, шунинг учун ўчирилган тенентлар эски базасига кимёсий киритилма қолдирмайди.

## Монархлик тенентидан фойдаланиш

1. **Network Admin > Ultimate Multisite > Sites** бўлимига киринг.
2. Монархлик тенентини танланг.
3. Паролларни кўчириш ёки вақтинча администратор ҳисоботлари яратиш ўрнига, мавжуд бўлган ҳолда **Visit (SSO)** функциясини ишлатинг.

Бу визит жараёни ушбу тенент учун қисқа муддатли логин токенини яратади ва SSO воқеатини тенент аудитор йўлида қайд этиб қўяди. Агар тугма номужда бўлса, тенент домени кутилган ўрнатилма билан мос келмалигини ва тенент тармоқ томонидаги SSO конектига етишишини текширинг.

## Масофавий сайт операциялари чеклиги

Монархлик ёки масофавий тененти ўзгартиришдан олдин қуйидагиларни тасдиқланг:

- Локидатель домен хостинг к хосту, который владеет файловой системой арендатора.
- Хост базы данных арендатора соответствует настроенному привязке хоста для этой установки.
- Команды проверки миграции проходят для арендатора.
- Очереди асинхронной миграции опорожняются перед внесением изменений в DNS или владение.
- Пользователь администратора арендатора был предоставлен во время миграции и может войти через SSO.

## Удаление суверенных арендаторов (sovereign tenants)

Удаление суверенного арендатора является разрушительной операцией. Сначала подтвердите статус резервной копии и экспорта, а затем удалите его на экране управления сайтом. В процессе развертывания версии 1.2.0 учетные данные базы данных арендатора удаляются как часть очистки, но администраторы все равно должны убедиться, что при использовании внешних панелей хостинга удалены пользователи и папки уровня хоста в базе данных.

:::warning
Не удаляйте суверенного арендатора во время выполнения проверки миграции или пока в очереди находятся асинхронные задания на отправку (async push jobs). Подождите завершения проверки, чтобы развертывание не удалило учетные данные, необходимые для ожидающих заданий.
:::
