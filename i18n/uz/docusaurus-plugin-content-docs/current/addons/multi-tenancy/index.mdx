# Ko‘p ijarachilik

Unumdorlik va xavfsizlik uchun quyi saytlarni serverlar va ma’lumotlar bazalari bo‘ylab taqsimlang

## Version 1.2.0 relizining asosiy yangiliklari

Multi-Tenancy 1.2.0 ishlab chiqarishdagi izolyatsiya ish jarayonlari uchun suveren ijarachi asosini qo‘shadi:

- Qisqa muddatli, qayta ijrodan himoyalangan SSO tashriflari bilan holatsiz ijarachi autologin.
- Suveren ijarachilar uchun har bir quyi sayt bo‘yicha ma’lumotlar bazasi va fayl tizimi izolyatsiyasi.
- Hostdan xabardor ijarachi tayyorlash uchun Hostinger ko‘p ijarachilik imkoniyati.
- Eski ma’lumotlarni tekshirish va asinxron push qayta ishlash uchun suveren migratsiyani tasdiqlash darvozalari.
- Bedrock, FrankenPHP va localhost o‘rnatishlari uchun sozlanadigan bir xil mashinadagi ma’lumotlar bazasi host bog‘lanishi.
- Ijarachini bootstrap qilish, ma’lumotlar bazasi yozuvchisi, sxemani bootstrap qilish va foydalanuvchi tayyorlash yaxshilanishlari.
- Ijarachi hisob ma’lumotlarini tozalaydigan xavfsizroq suveren olib tashlash.

[suveren migratsiya ish jarayoni](/user-guide/getting-started/sovereign-migration-workflow), [ko‘p ijarachilik izolyatsiyasi](/user-guide/configuration/multi-tenancy-isolation) va [suveren ijarachi operatsiyalari](/user-guide/administration/sovereign-tenant-operations) uchun foydalanuvchi qo‘llanmalariga qarang.

## Talablar

- WordPress 5.3 yoki undan yuqori
- PHP 7.4 yoki undan yuqori
- Multisite Ultimate plugin (faol)

2. Pluginni WordPress ichidagi 'Plugins' menyusi orqali faollashtiring
3. Addon sozlamalarini tarmoq administratori bo‘limida sozlang

## Konfiguratsiya

Addon tarmoq administratori interfeysi orqali mavjud bo‘lgan turli konfiguratsiya variantlarini qo‘llab-quvvatlaydi.

### Suveren ijarachi konfiguratsiyasi

Suveren ijarachilar uchun trafikni ijarachiga yo‘naltirishdan oldin ijarachi ma’lumotlar bazasini, fayl tizimi ildizini, ma’lumotlar bazasi host bog‘lanishini va host imkoniyatini sozlang. Ma’lumotlar bazasi hisob ma’lumotlari, host bog‘lanishi, marshrutlash yoki foydalanuvchi tayyorlashga ta’sir qiladigan har bir o‘zgarishdan keyin migratsiya tekshiruvini ishga tushiring.

`localhost` dan faqat ijarachi runtime va ma’lumotlar bazasi ruxsatlari lokal soket yoki bir xil mashinadan kirishni kutganda foydalaning. Provayderingiz tomonidan sozlangan ruxsat hosti shu bo‘lsa, `127.0.0.1` yoki xizmat host nomidan foydalaning.

### Hostinger imkoniyati

Hostinger ko‘p ijarachilik imkoniyati umumiy Hostinger integratsiya tokeni bilan ishlaydi. Tekshiruvdan oldin domen hPanel ichida mavjudligini, DNS Hostinger accountga yo‘naltirilganini va ijarachi ma’lumotlar bazasi sozlamalari hPanel bilan mos kelishini tasdiqlang.

## Hooklar va filtrlar

### Amallar

- `ultimate-multisite-addon-template_init` - Addon ishga tushirilgandan keyin ishga tushiriladi
- `ultimate-multisite-addon-template_loaded` - Barcha addon fayllari yuklangandan keyin ishga tushiriladi

### Filtrlar

- `ultimate-multisite-addon-template_settings` - Addon sozlamalarini o‘zgartirish
- `ultimate-multisite-addon-template_enabled` - Addonni yoqish/o‘chirish holatini bekor qilish
