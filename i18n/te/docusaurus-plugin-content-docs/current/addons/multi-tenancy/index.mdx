# బహుళ-అద్దెదారితనం

పనితీరు మరియు భద్రత కోసం ఉప-సైట్‌లను సర్వర్‌లు మరియు డేటాబేస్‌లలో పంపిణీ చేయండి

## Version 1.2.0 విడుదల ముఖ్యాంశాలు

Multi-Tenancy 1.2.0 ఉత్పత్తి ఐసోలేషన్ వర్క్‌ఫ్లోల కోసం స్వతంత్ర అద్దెదారు పునాదిని జోడిస్తుంది:

- తక్కువ కాలం ఉండే, మళ్లీ వినియోగం నుంచి రక్షించబడిన SSO సందర్శనలతో స్థితి-రహిత అద్దెదారు ఆటో-లాగిన్.
- స్వతంత్ర అద్దెదారుల కోసం ప్రతి ఉప-సైట్‌కు ప్రత్యేక డేటాబేస్ మరియు ఫైల్‌సిస్టమ్ ఐసోలేషన్.
- హోస్ట్‌ను గుర్తించే అద్దెదారు ప్రొవిజనింగ్ కోసం Hostinger బహుళ-అద్దెదారితనం సామర్థ్యం.
- పాత డేటా తనిఖీలు మరియు async పుష్ ప్రాసెసింగ్ కోసం స్వతంత్ర మైగ్రేషన్ ధృవీకరణ గేట్లు.
- Bedrock, FrankenPHP, మరియు localhost ఇన్‌స్టాల్‌ల కోసం కాన్ఫిగర్ చేయగల అదే-యంత్ర డేటాబేస్ హోస్ట్ బైండింగ్.
- అద్దెదారు బూట్‌స్ట్రాప్, డేటాబేస్ రైటర్, స్కీమా బూట్‌స్ట్రాప్, మరియు వినియోగదారు ప్రొవిజనింగ్ మెరుగుదలలు.
- అద్దెదారు క్రెడెన్షియల్‌లను శుభ్రం చేసే మరింత సురక్షిత స్వతంత్ర తొలగింపు.

[స్వతంత్ర మైగ్రేషన్ వర్క్‌ఫ్లో](/user-guide/getting-started/sovereign-migration-workflow), [బహుళ-అద్దెదారితనం ఐసోలేషన్](/user-guide/configuration/multi-tenancy-isolation), మరియు [స్వతంత్ర అద్దెదారు కార్యకలాపాలు](/user-guide/administration/sovereign-tenant-operations) కోసం వినియోగదారు మార్గదర్శకాలను చూడండి.

## అవసరాలు

- WordPress 5.3 లేదా అంతకంటే ఎక్కువ
- PHP 7.4 లేదా అంతకంటే ఎక్కువ
- Multisite Ultimate ప్లగిన్ (సక్రియంగా)

## ఇన్‌స్టాలేషన్

1. అనుబంధ ఫైల్‌లను మీ `/wp-content/plugins/` డైరెక్టరీకి అప్‌లోడ్ చేయండి
2. WordPress‌లోని 'ప్లగిన్‌లు' మెనూ ద్వారా ప్లగిన్‌ను సక్రియం చేయండి
3. నెట్‌వర్క్ అడ్మిన్‌లో అనుబంధ సెట్టింగ్‌లను కాన్ఫిగర్ చేయండి

## అభివృద్ధి

### సెటప్

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### పరీక్ష

అనుబంధంలో సమగ్ర పరీక్ష ఫ్రేమ్‌వర్క్ ఉంది:

- **యూనిట్ పరీక్షలు**: WordPress పరీక్ష సూట్‌తో PHPUnit-ఆధారిత పరీక్షలు
- **ప్రాథమిక పరీక్ష క్లాస్**: సాధారణ పరీక్ష సందర్భాల కోసం సహాయక పద్ధతులను అందిస్తుంది
- **పరీక్ష కవరేజ్**: కోడ్ కవరేజ్‌ను ట్రాక్ చేయడానికి కాన్ఫిగర్ చేయబడింది
- **బహుళ-సైట్ పరీక్ష**: పరీక్షలు బహుళ-సైట్ పరిసరంలో నడుస్తాయి

### కోడ్ ప్రమాణాలు

- **WordPress కోడింగ్ ప్రమాణాలు**: PHPCS ద్వారా అమలు చేయబడతాయి
- **PHP 7.4+ అనుకూలత**: ఆధునిక PHP ఫీచర్‌లకు మద్దతు ఉంది
- **స్టాటిక్ విశ్లేషణ**: టైప్ తనిఖీ కోసం PHPStan ఇంటిగ్రేషన్
- **కోడ్ ఆధునీకరణ**: PHP అప్‌గ్రేడ్‌ల కోసం Rector నియమాలు

### బిల్డ్ సిస్టమ్

- **అసెట్ ప్రాసెసింగ్**: CSS/JS ఫైల్‌ల కోసం మినిఫికేషన్
- **అనువాదం**: POT ఫైల్ సృష్టి
- **ప్యాకేజింగ్**: పంపిణీ కోసం ఆర్కైవ్ సృష్టి
- **అభివృద్ధి మోడ్**: డీబగ్గింగ్ కోసం మినిఫై చేయని అసెట్‌లు

## కాన్ఫిగరేషన్

నెట్‌వర్క్ అడ్మిన్ ఇంటర్‌ఫేస్ ద్వారా అందుబాటులో ఉండే వివిధ కాన్ఫిగరేషన్ ఎంపికలకు ఈ అనుబంధం మద్దతు ఇస్తుంది.

### స్వతంత్ర అద్దెదారు కాన్ఫిగరేషన్

స్వతంత్ర అద్దెదారుల కోసం, అద్దెదారుకు ట్రాఫిక్‌ను రూట్ చేయడానికి ముందు అద్దెదారు డేటాబేస్, ఫైల్‌సిస్టమ్ రూట్, డేటాబేస్ హోస్ట్ బైండింగ్, మరియు హోస్ట్ సామర్థ్యాన్ని కాన్ఫిగర్ చేయండి. డేటాబేస్ క్రెడెన్షియల్‌లు, హోస్ట్ బైండింగ్, రూటింగ్, లేదా వినియోగదారు ప్రొవిజనింగ్‌ను ప్రభావితం చేసే ప్రతి మార్పు తర్వాత మైగ్రేషన్ ధృవీకరణను నడపండి.

అద్దెదారు రన్‌టైమ్ మరియు డేటాబేస్ గ్రాంట్‌లు స్థానిక సాకెట్ లేదా అదే-యంత్ర యాక్సెస్‌ను ఆశించినప్పుడే `localhost` ఉపయోగించండి. మీ ప్రొవైడర్ కాన్ఫిగర్ చేసిన గ్రాంట్ హోస్ట్ అదే అయితే `127.0.0.1` లేదా సేవ హోస్ట్‌నేమ్‌ను ఉపయోగించండి.

### Hostinger సామర్థ్యం

Hostinger బహుళ-అద్దెదారితనం సామర్థ్యం షేర్ చేసిన Hostinger ఇంటిగ్రేషన్ టోకెన్‌తో పనిచేస్తుంది. ధృవీకరణకు ముందు డొమైన్ hPanel‌లో ఉందని, DNS Hostinger Account‌కి చూపుతుందని, మరియు అద్దెదారు డేటాబేస్ సెట్టింగ్‌లు hPanel‌తో సరిపోతున్నాయని నిర్ధారించండి.

## హుక్‌లు మరియు ఫిల్టర్‌లు

### చర్యలు

- `ultimate-multisite-addon-template_init` - అనుబంధ ప్రారంభీకరణ తర్వాత ఫైర్ అవుతుంది
- `ultimate-multisite-addon-template_loaded` - అన్ని అనుబంధ ఫైల్‌లు లోడ్ అయిన తర్వాత ఫైర్ అవుతుంది

### ఫిల్టర్‌లు

- `ultimate-multisite-addon-template_settings` - అనుబంధ సెట్టింగ్‌లను మార్చండి
- `ultimate-multisite-addon-template_enabled` - అనుబంధాన్ని సక్రియం/నిష్క్రియం చేసే స్థితిని ఓవర్‌రైడ్ చేయండి
