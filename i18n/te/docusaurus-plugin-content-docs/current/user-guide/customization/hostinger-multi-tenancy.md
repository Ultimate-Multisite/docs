# హోస్టింగ్マルチテナンシー

Ultimate Multisite: Multi-Tenancy 1.2.0 లో హోస్టింగ్ డొమైన్‌లు ఇప్పటికే ఉన్న Hostinger డొమైన్ మ్యాపింగ్ ఇంటిగ్రేషన్‌తో పాటు సార్వభౌమ టెనెంట్ ప్రావిజనింగ్‌లో పాల్గొనేలా ఒక Hostingerマルチテナンసీ సామర్థ్యాన్ని జోడిస్తుంది.

టెనెంట్ డొమైన్‌లు మరియు వేరు చేయబడిన టెనెంట్ ఇన్‌ఫ్రాస్ట్రక్చర్‌ను Hostinger hPanel ద్వారా నిర్వహించినప్పుడు ఈ సామర్థ్యాన్ని ఉపయోగించండి.

## సెటప్ నోట్స్ {#setup-notes}

1. **Ultimate Multisite > Settings > Host Integrations** లో కోర్ Hostinger ఇంటిగ్రేషన్‌ను కాన్ఫిగర్ చేయండి.
2. టార్గెట్ డొమైన్ లేదా సబ్-డొమైన్‌ను నిర్వహించడానికి Hostinger API టోకెన్ ఉపయోగించగలదని నిర్ధారించుకోండి.
3. Multi-Tenancy addon ను ఎనేబుల్ చేయండి.
4. టెనెంట్‌ను పబ్లిష్ చేయడానికి ముందు టెనెంట్ ఐసోలేషన్ వ్యూహాన్ని కాన్ఫిగర్ చేయండి.
5. టెనెంట్‌కు ప్రొడక్షన్ ట్రాఫిక్‌ను పంపే ముందు మైగ్రేషన్ వెరిఫికేషన్ వర్క్‌ఫ్లోను రన్ చేయండి.

Hostinger సామర్థ్యం హోస్ట్-సైడ్ ఆపరేషన్ల కోసం షేర్డ్ Hostinger కనెక్షన్‌ను ఉపయోగిస్తుంది. DNS ఇప్పటికీ సరైన Hostinger ఖాతాను సూచించాలి, మరియు hPanel ఖాతా పరిమితులు ఇంకా వర్తిస్తాయి.

## సామర్థ్య-నిర్దిష్ట మార్పులు {#capability-specific-changes}

- హోస్ట్-అవేర్ డొమైన్ ఆపరేషన్లతో సార్వభౌమ టెనెంట్‌లను సృష్టించవచ్చు.
- గ్రాంట్ వెరిఫికేషన్ ఇచ్చే ముందు సెమీ-మెషిన్ డేటాబేస్ హోస్ట్ స్ట్రింగ్‌లను సాధారణీ (normalize) చేస్తారు.
- WordPress రన్‌టైమ్ స్థానిక ఓవర్‌రైడ్ అవసరమైతే తప్ప, Hostinger ద్వారా నిర్వహించబడే టెనెంట్‌లు hPanel లో చూపబడిన డేటాబేస్ హోస్ట్ విలువను ఉపయోగించాలి.
- SSO సందర్శనలు టెనెంట్ డొమైన్ Hostinger-హోస్ట్ చేసిన టెనెంట్‌కు రిజాల్వింగ్ (resolve) పై ఆధారపడి ఉంటాయి.

## Hostinger టెనెంట్‌లను ట్రబుల్షూటింగ్ చేయడం {#troubleshooting-hostinger-tenants}

- టెనెంట్ ఇన్‌స్టాలేషన్ విఫలమైతే, డొమైన్ ఇప్పటికే Hostinger ఖాతాకు జోడించబడిందో లేదో తనిఖీ చేయండి.
- డేటాబేస్ వెరిఫికేషన్ విఫలమైతే, hPanel తో టెనెంట్ DB యూజర్ పేరు, డేటాబేస్ పేరు మరియు హోస్ట్ బైండింగ్‌ను పోల్చండి.
- **Visit (SSO)** విఫలమైతే, టెనెంట్ డొమైన్ కోసం DNS మరియు SSL యాక్టివ్‌గా ఉన్నాయని నిర్ధారించుకోండి.
- టెయర్‌డౌన్ చేసిన తర్వాత హోస్ట్ వనరులను వెనుక వదిలివేస్తే, బ్యాకప్‌లను కన్ఫర్మ్ చేసిన తర్వాత hPanel నుండి మిగిలిన డేటాబేస్‌లు, యూజర్లు లేదా ఫోల్డర్‌లను తొలగించండి.
