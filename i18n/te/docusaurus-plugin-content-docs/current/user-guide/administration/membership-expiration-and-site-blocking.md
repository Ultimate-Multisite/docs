# Membership Expiration and Site Blocking

ఈ గైడ్ Ultimate Multisite సభ్యత్వం గడువు ముగియడం, ట్రయల్స్ ముగియడం మరియు ఫ్రంటెండ్ సైట్ బ్లాకింగ్ ఎలా చేస్తుందో వివరిస్తుంది. సభ్యత్వం క్రియాశీలక స్థితి నుండి గడువు ముగిసిన స్థితి వరకు దాని జీవితచక్రం (lifecycle), సైట్‌లు బ్లాక్ అవుతాయో లేదో నియంత్రించే సెట్టింగ్‌లు, మరియు సభ్యత్వం గడువు ముగిసిన తర్వాత సైట్‌లు అందుబాటులో ఉంటే ఏమి తనిఖీ చేయాలో ఇది కవర్ చేస్తుంది.

## Membership Status Lifecycle

Ultimate Multisite లోని ప్రతి సభ్యత్వానికి ఈ క్రింది స్టేటస్‌లలో ఒకటి ఉంటుంది:

:::note Free memberships are lifetime
Free memberships గడువు ముగియవు. Ultimate Multisite వీటిని జీవితకాల యాక్సెస్ (lifetime access)గా పరిగణిస్తుంది, కాబట్టి అడ్మినిస్ట్రేటర్ వాటి స్టేటస్‌ను మార్చకపోతే లేదా కస్టమర్‌ను వేరే ఉత్పత్తికి తరలించకపోతే, అవి గడువు ముగింపు తనిఖీలలో చేర్చబడవు.
:::

| Status | Meaning |
|---|---|
| **Pending** | మొదటి చెల్లింపు లేదా ఈమెయిల్ వెరిఫికేషన్ కోసం వేచి ఉంది |
| **Trialing** | క్రియాశీల ట్రయల్ పీరియడ్, ఇంకా చెల్లింపు తీసుకోలేదు |
| **Active** | చెల్లించి, ప్రస్తుతమైనది |
| **On Hold** | పునరుద్ధరణ చెల్లింపు కోసం వేచి ఉంది (ఇన్వాయిస్ సృష్టించబడింది, చెల్లింపు కోసం వేచి ఉంది) |
| **Expired** | గడువు ముగింపు తేదీ మరియు గ్రేస్ పీరిడ్ దాటిన తర్వాత పునరుద్ధరించకపోవడం |
| **Cancelled** | కస్టమర్ లేదా అడ్మిన్ ద్వారా స్పష్టంగా రద్దు చేయబడింది |

### How Memberships Transition to Expired

Ultimate Multisite అనేది **ప్రతి గంటకు** ఒక బ్యాక్‌గ్రౌండ్ చెక్‌ను నడుపుతుంది, ఇది గడువు ముగియాల్సిన సభ్యత్వాలను వెతుకుతుంది. ఈ చెక్ [Action Scheduler](https://actionscheduler.org/) (నేరుగా WP-Cron కాదు) ను ఉపయోగిస్తుంది మరియు `wu_membership_check` అనే షెడ్యూల్డ్ యాక్షన్ ద్వారా నడుస్తుంది.

ఈ గడువు ముగింపు చెక్‌లో డిఫాల్ట్‌గా **3 రోజుల అంతర్నిర్మిత గ్రేస్ పీరిడ్** ఉంటుంది. ఒక సభ్యత్వం దాని `date_expiration` దాటిన తర్వాత 3 రోజులు గడిచే వరకు `expired` అని మార్క్ చేయబడదు. ఇది కస్టమర్‌లకు వారి స్టేటస్ మారడానికి ముందు ఆలస్యమైన చెల్లింపును పూర్తి చేయడానికి సమయం ఇస్తుంది.

:::info
3-రోజుల గడువు ముగింపు గ్రేస్ పీరిడ్ అనేది కింద వివరించిన Frontend Block Grace Period సెట్టింగ్‌ నుండి వేరుగా ఉంటుంది. గడువు ముగింపు గ్రేస్ పీరిడ్ అనేది **స్టేటస్** క్రియాశీలక/ఆన్-హోల్డ్ నుండి గడువు ముగిసిన స్థితికి మారే సమయాన్ని నియంత్రిస్తుంది. ఫ్రంటెండ్ బ్లాక్ గ్రేస్ పీరిడ్ అనేది స్టేటస్ ఇప్పటికే మారిన తర్వాత **సైట్** ఎప్పుడు బ్లాక్ అవుతుందో నియంత్రిస్తుంది.
:::

#### Auto-Renewing vs. Non-Auto-Renewing Memberships

ఈ తేడా గడువు ముగింపు ప్రవర్తనను అర్థం చేసుకోవడానికి చాలా కీలకం:

- **Non-auto-renewing memberships** (`auto_renew = false`): ఈ గంటకు నడిచే క్రోన్ జాబ్ మొత్తం జీవితచక్రాన్ని నిర్వహిస్తుంది -- ఇది పునరుద్ధరణ చెల్లింపులను సృష్టిస్తుంది, సభ్యత్వాన్ని ఆన్-హోల్డ్‌కు మారుస్తుంది, మరియు చెల్లింపు అందకపోతే చివరికి దానిని గడువు ముగిసినట్లుగా మార్క్ చేస్తుంది.

- **Auto-renewing memberships** (`auto_renew = true`): ఈ క్రోన్ గడువు చెక్ వీటిని **పూర్తిగా విస్మరిస్తుంది**. చెల్లింపు గేట్‌వే (Stripe, PayPal, మొదలైనవి) సబ్‌స్క్రిప్షన్ విఫలమైనా లేదా రద్దు అయినా వెబ్‌హుక్స్ (webhooks) ద్వారా Ultimate Multisite కి నోటిఫికేషన్ ఇవ్వాలని ఆశించబడుతుంది. వెబ్‌హుక్ అందకపోతే -- తప్పుగా కాన్ఫిగర్ చేసిన ఎండ్‌పాయింట్ కారణంగా, గేట్‌వే అంతరాయం కారణంగా, లేదా సిస్టమ్ వెలుపల రద్దు చేయబడిన సబ్‌స్క్రిప్షన్ కారణంగా -- గడువు తేదీ దాటిన తర్వాత కూడా సభ్యత్వం శాశ్వతంగా `active`గా ఉండవచ్చు.

### How Trials End

ఒక ట్రయల్ సభ్యత్వం యొక్క ట్రయల్ పీరియడ్ ముగిసినప్పుడు, సిస్టమ్:

1. పూర్తి సబ్‌స్క్రిప్షన్ మొత్తంతో పెండింగ్ పునరుద్ధరణ చెల్లింపును సృష్టిస్తుంది
2. సభ్యత్వ స్టేటస్‌ను `trialing` నుండి `on-hold` కు మారుస్తుంది
3. కస్టమర్‌కు పునరుద్ధరణ చెల్లింపు నోటిఫికేషన్ ఈమెయిల్ పంపుతుంది

ఈ ప్రక్రియ సాధారణ గడువు చెక్ వలె అదే గంటకు షెడ్యూల్‌లో నడుస్తుంది, కానీ **non-auto-renewing memberships** కోసం మాత్రమే. auto-renewing trials కోసం, చెల్లింపు గేట్‌వే ట్రయల్ నుండి చెల్లించిన సబ్‌స్క్రిప్షన్‌కు మారడాన్ని నిర్వహిస్తుంది.

## Block Frontend Access

డిఫాల్ట్‌గా, ఒక సభ్యత్వం గడువు ముగిసినా లేదా ఆన్-హోల్డ్‌కి వెళ్లినా, **wp-admin డాష్‌బోర్డ్ మాత్రమే పరిమితం చేయబడుతుంది**. సైట్ యొక్క పబ్లిక్ ఫ్రంటెండ్ సందర్శకులకు అందుబాటులో ఉంటుంది. పబ్లిక్ యాక్సెస్‌ను కూడా బ్లాక్ చేయాలంటే, మీరు **Block Frontend Access** సెట్టింగ్‌ను ఎనేబుల్ చేయాలి.

### Configuring the Setting

**Ultimate Multisite > Settings > Memberships**కి నావిగేట్ చేసి, **Block Frontend Access** ను ఎనేబుల్ చేయండి.

![Block Frontend Access settings showing the toggle enabled, a grace period of 7 days, and a Frontend Block Page selection](/img/config/settings-membership-block-frontend.png)

ఇది సభ్యత్వ సెట్టింగ్ పేజీ యొక్క పూర్తి వీక్షణ:

![Membership settings full page](/img/config/settings-membership-full.png)

ఈ ప్రవర్తనను నియంత్రించే మూడు సంబంధిత సెట్టింగ్‌లు ఉన్నాయి:

| Setting | Description | Default |
|---|---|---|
| **Block Frontend Access** | మాస్టర్ టోగుల్. ఇది ఎనేబుల్ చేయబడితే, సభ్యత్వం క్రియాశీలకంగా లేనప్పుడు నెట్‌వర్క్ సైట్‌ల పబ్లిక్ ఫ్రంటెండ్‌ను బ్లాక్ చేస్తుంది. | Off |
| **Frontend Block Grace Period** | సభ్యత్వం నిష్క్రియం అయిన తర్వాత బ్లాక్ చేయడానికి వేచి ఉండే రోజుల సంఖ్య. వెంటనే బ్లాక్ చేయాలంటే `0`కి సెట్ చేయండి. | 0 |
| **Frontend Block Page** | సైట్ బ్లాక్ అయినప్పుడు సందర్శకులను మళ్లించడానికి ప్రధాన సైట్‌లోని ఒక పేజీ. ఇది సెట్ చేయకపోతే, సందర్శకులు సాధారణ "Site not available" సందేశాన్ని చూస్తారు. | None |

### What Visitors See When a Site Is Blocked

ఫ్రంటెండ్ యాక్సెస్ బ్లాక్ అయినప్పుడు, సైట్‌కు సందర్శకులు ఈ క్రింది వాటిలో ఒకటి చూస్తారు:

1. **Frontend Block Page** లో మీరు ఎంచుకున్న పేజీకి **రీడైరెక్ట్** చేయబడతారు (కాన్ఫిగర్ చేస్తే), లేదా
2. **డిఫాల్ట్ సందేశం**: "This site is not available at the moment." మరియు సైట్ అడ్మిన్ కోసం లాగిన్ పేజీకి లింక్‌తో.

సైట్ అడ్మిన్‌లు ఇప్పటికీ లాగిన్ కావచ్చు -- లాగిన్ పేజీ ఎప్పుడూ బ్లాక్ చేయబడదు.

### What Gets Blocked and When

బ్లాకింగ్ ప్రవర్తన సభ్యత్వ స్టేటస్‌పై ఆధారపడి ఉంటుంది:

| Membership Status | Frontend Blocked? | Grace Period Applied? |
|---|---|---|
| Active | No | -- |
| Trialing | **No** (see note below) | -- |
| On Hold | Considered active -- not blocked | -- |
| Expired | **Yes**, if Block Frontend Access is on | Yes |
| Cancelled | **Yes**, always (regardless of setting) | **No** -- blocked immediately |
| Pending | Not blocked via membership check | -- |

:::warning Trialing memberships are not blocked
ట్రయల్ సభ్యత్వాలు బ్లాక్ చేయబడవు
ట్రయల్ పీరియడ్ ముగిసినప్పటికీ, `trialing` స్టేటస్‌తో ఉన్న సభ్యత్వం ఫ్రంటెండ్‌లో **బ్లాక్ చేయబడదు**. ట్రయల్ మొదట మరొక స్టేటస్‌కు (సాధారణంగా క్రోన్ జాబ్ ద్వారా `on-hold`, ఆపై చెల్లింపు లేకపోతే `expired`) మారాలి. మీరు ట్రయలింగ్ సభ్యత్వాలు మారలేదని చూస్తున్నట్లయితే, క్రింది ట్రబుల్‌షూటింగ్ సెక్షన్‌ను తనిఖీ చేయండి.
:::

:::note Cancelled memberships bypass the grace period
రద్దు చేయబడిన సభ్యత్వాలు గ్రేస్ పీరిడ్‌ను దాటిపోతాయి
Block Frontend Access ఎనేబుల్ చేయబడిందా అనే దానితో సంబంధం లేకుండా, గడువు తేదీ దాటిన తర్వాత రద్దు చేయబడిన సభ్యత్వాలు ఎల్లప్పుడూ బ్లాక్ చేయబడతాయి. Frontend Block Grace Period అనేది రద్దు చేయబడిన సభ్యత్వాలకు వర్తించదు.
:::

## Troubleshooting: Sites Remaining Accessible After Expiration

సభ్యత్వం గడువు ముగిసిన తర్వాత సైట్‌లు పబ్లిక్‌గా అందుబాటులో ఉంటే, ఈ తనిఖీలను క్రమంలో చేయండి:

### 1. Verify the Block Frontend Access Setting Is Enabled

**Ultimate Multisite > Settings > Memberships**కి వెళ్లి, **Block Frontend Access** టోగుల్ ఆన్ చేయబడిందో లేదో నిర్ధారించుకోండి. ఈ సెట్టింగ్ **డిఫాల్ట్‌గా ఆఫ్** చేయబడి ఉంటుంది, అంటే సభ్యత్వం నిష్క్రియం అయినప్పుడు wp-admin మాత్రమే పరిమితం చేయబడుతుంది.

### 2. Check the Frontend Block Grace Period

అదే సెట్టింగ్ పేజీలో, **Frontend Block Grace Period** విలువను తనిఖీ చేయండి. ఉదాహరణకు, ఇది 7 రోజులుగా సెట్ చేయబడితే, సభ్యత్వ స్టేటస్ ఇప్పటికే `expired` అయినప్పటికీ, ఫ్రంటెండ్ 7 రోజుల వరకు బ్లాక్ చేయబడదు.

సభ్యత్వం నిష్క్రియం అయిన తర్వాత వెంటనే బ్లాక్ చేయాలనుకుంటే దీనిని `0`కి సెట్ చేయండి.

### 3. Confirm the Membership Status Has Actually Changed

**Ultimate Multisite > Memberships**కి వెళ్లి, ప్రభావితమైన సభ్యత్వ స్థితిని తనిఖీ చేయండి. గడువు తేదీ దాటిపోయినా ఇది ఇంకా `active`గా కనిపిస్తే, స్టేటస్ ట్రాన్సిషన్ జరగలేదు. సాధారణ కారణాలు:

- **సభ్యత్వం ఆటో-రిన్యూ అవుతోంది**: సభ్యత్వం ఎడిట్ పేజీలో `auto_renew` ఫీల్డ్‌ను తనిఖీ చేయండి. ఆటో-రిన్యూ ఎనేబుల్ చేయబడితే, గడువు క్రోన్ ఈ సభ్యత్వాన్ని దాటవేస్తుంది -- ఇది విఫలమవడాన్ని నివేదించడానికి చెల్లింపు గేట్‌వేపై ఆధారపడుతుంది. Ultimate Multisite చూపినదానితో సబ్‌స్క్రిప్షన్ స్టేటస్ సరిపోలుతుందో లేదో నిర్ధారించడానికి మీ గేట్‌వే డాష్‌బోర్డ్‌ను (Stripe, PayPal) తనిఖీ చేయండి.

- **క్రోన్ జాబ్ నడపలేదు**: తదుపరి దశను చూడండి.

### 4. Verify Action Scheduler Is Running

Ultimate Multisite తన క్రోన్ జాబ్‌ల కోసం Action Scheduler ను ఉపయోగిస్తుంది. నెట్‌వర్క్ అడ్మిన్ లో **Tools > Scheduled Actions**కి వెళ్లి, వీటి కోసం చూడండి:

- **`wu_membership_check`** -- ఇది ప్రతి గంట నడుస్తున్న పునరావృత చర్యగా కనిపించాలి. ఇది లేకపోతే, సభ్యత్వ చెక్‌లు షెడ్యూల్ చేయబడటం లేదు.
- **`wu_async_mark_membership_as_expired`** -- ఇవి నిర్దిష్ట సభ్యత్వాలను గడువు ముగిసినట్లుగా మార్క్ చేయడానికి వ్యక్తిగత పనులు. ఇక్కడ మీరు విఫలమైన చర్యలను చూస్తే, ఎందుకు అనే వివరణ ఇచ్చే లోపాల సందేశాలు ఉండవచ్చు.

మీరు ఇరుక్కుపోయిన లేదా విఫలమైన చర్యలను చూస్తే, మీకు Action Scheduler సమస్య ఉండవచ్చు. సాధారణ కారణాలు:

- `wp-config.php` లో `DISABLE_WP_CRON` ను `true`కి సెట్ చేయడం, కానీ సిస్టమ్-స్థాయి క్రోన్ రీప్లేస్‌మెంట్ లేకపోవడం
- తక్కువ సైట్ ట్రాఫిక్ -- WP-Cron ఎవరైనా సైట్‌ను సందర్శించినప్పుడు మాత్రమే నడుస్తుంది

విశ్వసనీయమైన క్రోన్ ఎగ్జిక్యూషన్‌ను నిర్ధారించడానికి, సిస్టమ్ క్రోన్ జాబ్‌ను సెటప్ చేయండి:

```bash
# Run every 5 minutes via wget
*/5 * * * * wget -q -O /dev/null "https://your-network-url.com/wp-cron.php?doing_wp_cron"

# Or via WP-CLI
*/5 * * * * cd /path/to/wordpress && wp cron event run --due-now --url=https://your-network-url.com
```

### 5. Check for Gateway Webhook Issues (Auto-Renewing Memberships)

సభ్యత్వం ఆటో-రిన్యూ అవుతూ, గేట్‌వే సబ్‌స్క్రిప్షన్ రద్దు చేయబడినా లేదా విఫలమైనా, కానీ Ultimate Multisite ఇప్పటికీ దానిని `active`గా చూపుతుంటే:

- **Stripe**: Stripe Dashboard > Customers కి వెళ్లి సబ్‌స్క్రిప్షన్ స్టేటస్‌ను తనిఖీ చేయండి. ఆపై Developers > Webhooks కింద వెబ్‌హుక్ ఎండ్‌పాయింట్ క్రియాశీలంగా ఉందో లేదో ధృవీకరించండి. ఎండ్‌పాయింట్ మీ సైట్‌ను చూపించాలి మరియు విజయవంతమైన డెలివరీలను చూపించాలి.
- **PayPal**: మీ PayPal బిజినెస్ ఖాతాలో సబ్‌స్క్రిప్షన్ స్టేటస్‌ను తనిఖీ చేయండి మరియు IPN/వెబ్‌హుక్ డెలివరీని ధృవీకరించండి.

గేట్‌వే సబ్‌స్క్రిప్షన్‌ను రద్దు చేసినట్లు చూపించినా Ultimate Multisite చూపకపోతే, వెబ్‌హుక్ నోటిఫికేషన్ కోల్పోయి ఉండవచ్చు. మీరు **Ultimate Multisite > Memberships > [Edit Membership]** లో సభ్యత్వ స్థితిని మాన్యువల్‌గా మార్చవచ్చు.

### 6. Check the Expiration Grace Period (Cron Level)

క్రోన్ చెక్‌కు దాని స్వంత గ్రేస్ పీరిడ్ ఉంటుంది (డిఫాల్ట్: 3 రోజులు) సభ్యత్వాన్ని గడువు ముగిసినట్లుగా మార్క్ చేయడానికి ముందు. ఇది ఫ్రంటెండ్ బ్లాక్ గ్రేస్ పీరిడ్ నుండి వేరుగా ఉంటుంది. సైట్ బ్లాక్ కావడానికి మొత్తం సమయం ఇలా ఉండవచ్చు:

**Expiration grace period (3 days)** + **Frontend block grace period (your setting)** = Total delay

ఉదాహరణకు, డిఫాల్ట్ సెట్టింగ్‌లు మరియు 7-రోజుల ఫ్రంటెండ్ గ్రేస్ పీరిడ్‌తో, `date_expiration` తర్వాత సైట్ నిజంగా బ్లాక్ కావడానికి 10 రోజుల వరకు పట్టవచ్చు.

### 7. Manually Expire a Membership

క్రోన్ సైకిల్‌ కోసం వేచి ఉండకుండా సైట్‌ను తక్షణమే బ్లాక్ చేయవలసి వస్తే, మీరు సభ్యత్వ స్థితిని మాన్యువల్‌గా మార్చవచ్చు:

1. **Ultimate Multisite > Memberships**కి వెళ్లండి
2. ప్రభావితమైన సభ్యత్వంపై క్లిక్ చేయండి
3. స్టేటస్‌ను **Expired** లేదా **Cancelled**కి మార్చండి
4. **Save** పై క్లిక్ చేయండి

ఫ్రంటెండ్ బ్లాక్ తదుపరి పేజీ లోడ్ అయినప్పుడు అమలులోకి వస్తుంది (గడువు ముగిసిన సభ్యత్వాల కోసం ఫ్రంటెండ్ బ్లాక్ గ్రేస్ పీరిడ్ ప్రకారం, లేదా రద్దు చేయబడిన సభ్యత్వాల కోసం తక్షణమే).

## Summary

గడువు తేదీ నుండి సైట్ బ్లాకింగ్ వరకు పూర్తి టైమ్‌లైన్:

```text
date_expiration passes
       |
       v
  [3-day cron grace period]     <-- membership still shows as active/on-hold
       |
       v
  Cron marks membership as "expired"
       |
       v
  [Frontend Block Grace Period]  <-- configured in Settings > Memberships
       |
       v
  Site frontend is blocked
```

రద్దు చేయబడిన సభ్యత్వాల కోసం, మార్గం చిన్నది:

```text
  Membership cancelled
       |
       v
  date_expiration passes (no grace period)
       |
       v
  Site frontend is blocked immediately
```

## Developer Reference

కింది హుక్స్ మరియు ఫిల్టర్‌లు గడువు ముగింపు మరియు బ్లాకింగ్ ప్రవర్తనను అనుకూలీకరించడానికి మీకు అనుమతిస్తాయి:

| Hook/Filter | Description |
|---|---|
| `wu_membership_grace_period_days` | సభ్యత్వాన్ని గడువు ముగిసినట్లుగా మార్క్ చేయడానికి ముందు గ్రేస్ పీరిడ్ రోజుల సంఖ్యను ఫిల్టర్ చేయండి (డిఫాల్ట్: 3) |
| `wu_schedule_membership_check_interval` | సభ్యత్వ చెక్‌ల మధ్య అంతరాన్ని ఫిల్టర్ చేయండి (డిఫాల్ట్: 1 గంట) |
| `wu_membership_renewal_days_before_expiring` | పునరుద్ధరణ చెల్లింపును సృష్టించడానికి గడువు ముగియడానికి ఎన్ని రోజుల ముందు ఫిల్టర్ చేయండి (డిఫాల్ట్: 3) |
| `wu_blocked_site_reactivation_url` | సైట్ బ్లాక్ అయినప్పుడు కస్టమ్ రియాక్టివేషన్ URL అందించడానికి ఫిల్టర్ చేయండి |
| `wu_membership_is_active` | సభ్యత్వం క్రియాశీలకంగా పరిగణించబడుతుందో లేదో ఫిల్టర్ చేయండి |
| `wu_membership_expired_check_query_params` | గడువు ముగిసిన సభ్యత్వాలను కనుగొనడానికి ఉపయోగించే క్వెరీ పారామితులను ఫిల్టర్ చేయండి |
