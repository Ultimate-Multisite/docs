# థర్డ్-పార్టీ మోడ్ మైగ్రేషన్ {#third-party-mode-migration}

Superdav AI Agent v1.12.0 ద్వారా థర్డ్-పార్టీ సామర్థ్యాలను (abilities) నిర్వహించే విధానంలో మార్పులు వచ్చాయి. **థర్డ్-పార్టీ మోడ్ ఇప్పుడు ఆటో (auto) మోడ్‌కి డిఫాల్ట్‌గా మారుతుంది**, దీని వలన మాన్యువల్ కాన్ఫిగరేషన్ అవసరం లేకుండా WordPress 7.0+ లోనే native WordPress Abilities API integration పనిచేస్తుంది.

## ఏమ మారింది? {#what-changed}

### v1.12.0 కంటే ముందు {#before-v1120}

థర్డ్-పార్టీ సామర్థ్యాలను మాన్యువల్‌గా కాన్ఫిగర్ చేయవలసి వచ్చేది:

- మీరు ప్రత్యేకంగా "third-party mode" ను ఎనేబుల్ చేయాల్సి వచ్చేది
- సామర్థ్యాలను ఒక కస్టమ్ రిజిస్ట్రీ నుండి లోడ్ చేసేవి
- WordPress Abilities API తో ఇంటిగ్రేషన్ ఐచ్ఛికం (optional) మాత్రమే
- డిఫాల్ట్‌గా లెగసీ మోడ్ (Legacy mode) ఉండేది

### v1.12.0 తర్వాత {#after-v1120}

థర్డ్-పార్టీ సామర్థ్యాలు ఆటోమేటిక్‌గా పనిచేస్తాయి:

- థర్డ్-పార్టీ మోడ్ "auto" కి డిఫాల్ట్‌గా మారుతుంది
- సామర్థ్యాలు WordPress Abilities API తో native గా ఇంటిగ్రేట్ అవుతాయి
- WordPress 7.0+ లో మాన్యువల్ కాన్ఫిగరేషన్ అవసరం లేదు
- పాత WordPress వెర్షన్ల కోసం లెగసీ మోడ్ ఇప్పటికీ అందుబాటులో ఉంది

## ఎవరికి ప్రభావం ఉంటుంది? {#who-is-affected}

### కొత్త ఇన్‌స్టాలేషన్లు (WordPress 7.0+) {#new-installations-wordpress-70}

**మీరు ఏమీ చేయనవసరం లేదు.** థర్డ్-పార్టీ మోడ్ ఆటోమేటిక్‌గా "auto" కి సెట్ చేయబడుతుంది, మరియు సామర్థ్యాలు వెంటనే పనిచేస్తాయి.

### ఇప్పటికే ఉన్న ఇన్‌స్టాలేషన్లు {#existing-installations}

**మీ సెట్టింగ్‌లు చెక్కుచెదరవు.** మీరు వీటిని ఉపయోగిస్తున్నట్లయితే:

- **లెగసీ మోడ్ (Legacy mode)**: మీరు లెగసీ మోడ్‌లోనే ఉంటారు (మార్పు లేదు)
- **మాన్యువల్ థర్డ్-పార్టీ మోడ్ (Manual third-party mode)**: మీరు మాన్యువల్ మోడ్‌లోనే ఉంటారు (మార్పు లేదు)
- **ఆటో మోడ్ (Auto mode)**: మీరు ఆటో మోడ్‌లోనే కొనసాగుతారు (మార్పు లేదు)

### 7.0 కంటే పాత WordPress వెర్షన్లు {#wordpress-versions-before-70}

**లెగసీ మోడ్ ఇప్పటికీ అందుబాటులో ఉంది.** మీరు WordPress 6.x లేదా అంతకన్నా పాత వెర్షన్‌లో ఉంటే:

- థర్డ్-పార్టీ మోడ్ "legacy" కి డిఫాల్ట్‌గా మారుతుంది
- కావాలనుకుంటే మీరు మాన్యువల్‌గా థర్డ్-పార్టీ మోడ్‌ను ఎనేబుల్ చేయవచ్చు
- native Abilities API ఉపయోగించడానికి WordPress 7.0+ కి అప్‌గ్రేడ్ చేయండి

## మోడ్‌లను అర్థం చేసుకోవడం {#understanding-the-modes}

### ఆటో మోడ్ (Auto Mode) (కొత్త డిఫాల్ట్) {#auto-mode-new-default}

**ఆటో మోడ్** native WordPress Abilities API integration ను ఉపయోగిస్తుంది:

- సామర్థ్యాలు WordPress hooks ద్వారా రిజిస్టర్ చేయబడతాయి
- WordPress 7.0+ Abilities API తో పూర్తి అనుకూలత (Full compatibility)
- థర్డ్-పార్టీ సామర్థ్యాల ఆటోమేటిక్ డిస్కవరీ
- మాన్యువల్ కాన్ఫిగరేషన్ అవసరం లేదు

**ఎప్పుడు ఉపయోగించాలి**: WordPress 7.0+ తో థర్డ్-పార్టీ సామర్థ్యాలు ఉన్నప్పుడు

### మాన్యువల్ మోడ్ (Manual Mode) {#manual-mode}

**మాన్యువల్ మోడ్** ప్రత్యేక కాన్ఫిగరేషన్ అవసరం:

- ఏ థర్డ్-పార్టీ సామర్థ్యాలను లోడ్ చేయాలో మీరు పేర్కొంటారు
- టెస్టింగ్ లేదా ఎంపిక చేసిన సామర్థ్యాలను లోడ్ చేయడానికి ఉపయోగపడుతుంది
- కాన్ఫిగరేషన్ ఫైళ్లను ఎడిట్ చేయాలి
- ఎక్కువ నియంత్రణ ఉంటుంది, కానీ సెటప్ ఎక్కువ

**ఎప్పుడు ఉపయోగించాలి**: టెస్టింగ్, ఎంపిక చేసిన సామర్థ్యాలను లోడ్ చేయడం, లేదా కస్టమ్ కాన్ఫిగరేషన్ల కోసం

### లెగసీ మోడ్ (Legacy Mode) {#legacy-mode}

**లెగసీ మోడ్** పాత థర్డ్-పార్టీ సామర్థ్య వ్యవస్థను ఉపయోగిస్తుంది:

- కస్టమ్ అబిలిటీ రిజిస్ట్రీ (WordPress Abilities API కాదు)
- పాత WordPress వెర్షన్లతో వెనుకకు అనుకూలత (Backward compatible)
- native WordPress ఇంటిగ్రేషన్ లేదు
- డిప్రికేట్ అయినప్పటికీ, ఇప్పటికీ మద్దతు ఇస్తున్నారు

**ఎప్పుడు ఉపయోగించాలి**: WordPress 6.x లేదా అంతకన్నా పాత వెర్షన్ల కోసం, లేదా లెగసీ అనుకూలత అవసరమైనప్పుడు

## మీ ప్రస్తుత మోడ్‌ను తనిఖీ చేయడం {#checking-your-current-mode}

### అడ్మిన్ ప్యానెల్ ద్వారా {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** కి వెళ్లండి
2. **Third-Party Mode** సెట్టింగ్‌ కోసం చూడండి
3. మీరు మీ ప్రస్తుత మోడ్‌ను మరియు దానిని మార్చడానికి ఉన్న ఆప్షన్‌లను చూస్తారు

### కోడ్ ద్వారా {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', or 'legacy'
```

## మీ మోడ్‌ను మార్చడం {#changing-your-mode}

### ఆటో మోడ్‌కు మారడం (Switch to Auto Mode) {#switch-to-auto-mode}

మీరు WordPress 7.0+ లో ఉన్నట్లయితే మరియు ఆటో మోడ్‌ను ఉపయోగించాలనుకుంటే:

1. **Superdav AI Agent** → **Settings** కి వెళ్లండి
2. **Third-Party Mode** కనుగొనండి
3. **Auto (WordPress Abilities API)** ఎంచుకోండి
4. **Save** పై క్లిక్ చేయండి

Superdav AI Agent స్వయంచాలకంగా థర్డ్-పార్టీ సామర్థ్యాలను కనుగొని, రిజిస్టర్ చేస్తుంది.

### మాన్యువల్ మోడ్‌కు మారడం (Switch to Manual Mode) {#switch-to-manual-mode}

ఏ సామర్థ్యాలు లోడ్ చేయాలో మీరు మాన్యువల్‌గా నియంత్రించాలనుకుంటే:

1. **Superdav AI Agent** → **Settings** కి వెళ్లండి
2. **Third-Party Mode** కనుగొనండి
3. **Manual** ఎంచుకోండి
4. **Save** పై క్లిక్ చేయండి
5. ఏ సామర్థ్యాలను లోడ్ చేయాలో పేర్కొనడానికి మీ కాన్ఫిగరేషన్ ఫైల్‌ను ఎడిట్ చేయండి

### లెగసీ మోడ్‌కు మారడం (Switch to Legacy Mode) {#switch-to-legacy-mode}

మీకు లెగసీ అనుకూలత అవసరమైతే:

1. **Superdav AI Agent** → **Settings** కి వెళ్లండి
2. **Third-Party Mode** కనుగొనండి
3. **Legacy** ఎంచుకోండి
4. **Save** పై క్లిక్ చేయండి

## ఆటో మోడ్ ప్రయోజనాలు {#benefits-of-auto-mode}

### ఆటోమేటిక్ డిస్కవరీ (Automatic Discovery) {#automatic-discovery}

సామర్థ్యాలు వీటి నుండి ఆటోమేటిక్‌గా కనుగొనబడతాయి:

- ఇన్‌స్టాల్ చేయబడిన ప్లగిన్‌లు (Installed plugins)
- యాక్టివ్ థీమ్ (Active theme)
- తప్పనిసరి ప్లగిన్‌లు (Must-use plugins)
- డ్రాప్-ఇన్ ప్లగిన్‌లు (Drop-in plugins)

మాన్యువల్ రిజిస్ట్రేషన్ అవసరం లేదు.

### native ఇంటిగ్రేషన్ (Native Integration) {#native-integration}

సామర్థ్యాలు WordPress Abilities API తో ఇంటిగ్రేట్ అవుతాయి:

- WordPress core తో స్థిరంగా ఉంటుంది
- WordPress admin తో పనిచేస్తుంది
- Abilities API ఉపయోగించే ఇతర ప్లగిన్‌లతో అనుకూలంగా ఉంటుంది
- WordPress అభివృద్ధి చెందుతున్న కొద్దీ భవిష్యత్తుకు సిద్ధంగా ఉంటుంది

### సరళమైన నిర్వహణ (Simplified Management) {#simplified-management}

- ఎడిట్ చేయాల్సిన కాన్ఫిగరేషన్ ఫైళ్లు లేవు
- మాన్యువల్ సామర్థ్య రిజిస్ట్రేషన్ లేదు
- Ability Visibility కంట్రోల్స్ ఆటోమేటిక్‌గా పనిచేస్తాయి
- అడ్మిన్ నోటిఫికేషన్‌లు వర్గీకరించబడని సామర్థ్యాల గురించి మీకు హెచ్చరిస్తాయి

### మెరుగైన పనితీరు (Better Performance) {#better-performance}

- సామర్థ్యాలు కాష్ చేయబడతాయి (cached)
- అవసరమైనప్పుడు లేజీ-లోడ్ చేయబడతాయి (Lazy-loaded)
- WordPress 7.0+ కోసం ఆప్టిమైజ్ చేయబడింది

## మైగ్రేషన్ మార్గం (Migration Path) {#migration-path}

### మీరు WordPress 6.x లో ఉంటే {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ కి అప్‌గ్రేడ్ చేయండి** (సిద్ధమైనప్పుడు)
2. **Superdav AI Agent** ను v1.12.0+ కి అప్‌డేట్ చేయండి
3. **థర్డ్-పార్టీ మోడ్‌ను ఆటో (Auto) కి మార్చండి** (ఐచ్ఛికం; లెగసీ మోడ్ ఇప్పటికీ పనిచేస్తుంది)
4. సరైన యాక్సెస్ కంట్రోల్స్ కోసం **ability visibility** ను సమీక్షించండి

### మీరు WordPress 7.0+ లో ఉంటే {#if-youre-on-wordpress-70}

1. **Superdav AI Agent** ను v1.12.0+ కి అప్‌డేట్ చేయండి
2. **థర్డ్-పార్టీ మోడ్ ఆటో (Auto) కి సెట్ చేయబడిందో లేదో ధృవీకరించండి** (ఇది డిఫాల్ట్‌గా ఉండాలి)
3. సరైన యాక్సెస్ కంట్రోల్స్ కోసం **ability visibility** ను సమీక్షించండి
4. అవి పనిచేస్తున్నాయో లేదో నిర్ధారించడానికి **థర్డ్-పార్టీ సామర్థ్యాలను పరీక్షించండి**

## ట్రబుల్‌షూటింగ్ (Troubleshooting) {#troubleshooting}

### ఆటో మోడ్‌లో సామర్థ్యాలు లోడ్ కావడం లేదు {#abilities-arent-loading-in-auto-mode}

- మీరు WordPress 7.0+ లో ఉన్నారో లేదో ధృవీకరించండి
- థర్డ్-పార్టీ మోడ్ "Auto" కి సెట్ చేయబడిందో లేదో తనిఖీ చేయండి
- సామర్థ్యాన్ని అందించే ప్లగిన్ యాక్టివ్‌గా ఉందో లేదో తనిఖీ చేయండి
- రిజిస్ట్రేషన్ లోపాల కోసం WordPress error logs ను తనిఖీ చేయండి

### నేను లెగసీ మోడ్‌ను కొనసాగించాలనుకుంటున్నాను {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** కి వెళ్లండి
- **Legacy** ఎంచుకోండి
- **Save** పై క్లిక్ చేయండి
- లెగసీ మోడ్ పనిచేయడం కొనసాగిస్తుంది

### నా కస్టమ్ సామర్థ్యాలు కనిపించడం లేదు {#my-custom-abilities-arent-showing}

- అవి WordPress hooks ద్వారా రిజిస్టర్ చేయబడ్డాయో లేదో ధృవీకరించండి
- అవి Abilities API ని సరిగ్గా అమలు చేస్తున్నాయో లేదో తనిఖీ చేయండి
- WordPress error logs ను సమీక్షించండి
- అన్ని రిజిస్టర్ చేయబడిన సామర్థ్యాలను చూడటానికి **Ability Visibility** అడ్మిన్ పేజీని ఉపయోగించండి

### నాకు "unclassified ability" నోటిఫికేషన్‌లు వస్తున్నాయి {#im-getting-unclassified-ability-notices}

- ఇది కొత్త థర్డ్-పార్టీ సామర్థ్యాలకు సాధారణం
- అడ్మిన్ నోటిస్‌లో వాటిని సమీక్షించి, వర్గీకరించండి (classify)
- వర్గీకరణపై వివరాల కోసం **Ability Visibility** చూడండి

## వెనుకకు అనుకూలత (Backward Compatibility) {#backward-compatibility}

### ఇప్పటికే ఉన్న కాన్ఫిగరేషన్‌లు {#existing-configurations}

మీకు ఇప్పటికే థర్డ్-పార్టీ సామర్థ్య కాన్ఫిగరేషన్‌లు ఉంటే:

- **లెగసీ మోడ్**: మీ కాన్ఫిగరేషన్ పనిచేయడం కొనసాగిస్తుంది
- **మాన్యువల్ మోడ్**: మీ కాన్ఫిగరేషన్ పనిచేయడం కొనసాగిస్తుంది
- **ఆటో మోడ్**: మీ కాన్ఫిగరేషన్ విస్మరించబడుతుంది (auto మోడ్ దీనిని అధిగమిస్తుంది)

మీ కస్టమ్ కాన్ఫిగరేషన్‌ను ఉంచడానికి, మాన్యువల్ లేదా లెగసీ మోడ్‌లోనే ఉండండి.

### డిప్రికేషన్ టైమ్‌లైన్ (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: లెగసీ మరియు మాన్యువల్ మోడ్‌లు ఇంకా పూర్తిగా మద్దతు ఇస్తాయి
- **v1.13.0+**: లెగసీ మోడ్‌లో డిప్రికేషన్ నోటిఫికేషన్‌లు కనిపించవచ్చు
- **v2.0.0**: లెగసీ మోడ్ తొలగించబడవచ్చు (TBD)

## ఉత్తమ పద్ధతులు (Best Practices) {#best-practices}

### కొత్త ఇన్‌స్టాలేషన్ల కోసం {#for-new-installations}

- ఆటో మోడ్‌ను ఉపయోగించండి (ఇది డిఫాల్ట్)
- Superdav AI Agent సామర్థ్యాలను ఆటోమేటిక్‌గా కనుగొనేందుకు వదిలేయండి
- యాక్సెస్‌ను నియంత్రించడానికి Ability Visibility ఉపయోగించండి

### ఇప్పటికే ఉన్న ఇన్‌స్టాలేషన్ల కోసం {#for-existing-installations}

- వీలైనంత త్వరగా WordPress 7.0+ కి అప్‌గ్రేడ్ చేయండి
- సరళమైన నిర్వహణ కోసం ఆటో మోడ్‌కు మారండి
- Ability Visibility ఉపయోగించి సామర్థ్యాలను సమీక్షించి, వర్గీకరించండి

### కస్టమ్ సామర్థ్యాల కోసం {#for-custom-abilities}

- సామర్థ్యాలను WordPress hooks ద్వారా రిజిస్టర్ చేయండి (Abilities API)
- కస్టమ్ అబిలిటీ రిజిస్ట్రీలను నివారించండి
- ఆటో మోడ్‌తో WordPress 7.0+ పై పరీక్షించండి

5. **పరీక్షించండి**: మీ థర్డ్-పార్టీ సామర్థ్యాలు సరిగ్గా పనిచేస్తున్నాయో లేదో ధృవీకరించండి
