# Gratis AI Agent సెట్టింగ్‌లు

Gratis AI Agent లోని **Settings → Advanced** స్క్రీన్, v1.5.0 లో పరిచయ చేయబడిన బ్యాకెండ్ ఇంటిగ్రేషన్‌ల కోసం అడ్మినిస్ట్రేటర్-స్థాయి కాన్ఫిగరేషన్‌ను అందిస్తుంది. ఈ పేజీ **Feedback Endpoint** ఫీల్డ్‌లు మరియు వాటి ఆశించిన ఫార్మాట్‌ను డాక్యుమెంట్ చేస్తుంది.

## సెట్టింగ్‌లను యాక్సెస్ చేయడం

1. WordPress అడ్మిన్ లో, **Gratis AI Agent → Settings** కి వెళ్లండి.
2. **Advanced** ట్యాబ్‌పై క్లిక్ చేయండి.

## Feedback Endpoint కాన్ఫిగరేషన్

వినియోగదారుడు థంబ్స్-డౌన్ బటన్, ఆటో-ప్రాంప్ట్ బ్యానర్, లేదా `/report-issue` కమాండ్ ద్వారా ఫీడ్‌బ్యాక్ సమర్పించిన ప్రతిసారీ, ఈ ఫీడ్‌బ్యాక్ ఎండ్ పాయింట్ (feedback endpoint) నుండి POST రిక్వెస్ట్‌లు వస్తాయి.

| ఫీల్డ్ | వివరణ |
|---|---|
| **Feedback Endpoint URL** | JSON బాడీతో HTTP POST రిక్వెస్ట్‌లుగా ఫీడ్‌బ్యాక్ సమర్పణలను స్వీకరించే URL. |
| **Feedback API Key** | ప్రతి ఫీడ్‌బ్యాక్ రిక్వెస్ట్‌లో `Authorization` హెడర్‌లో పంపబడే బేరర్ టోకెన్ (bearer token). మీ ఎండ్ పాయింట్‌కు ఆథెంటికేషన్ అవసరం లేకపోతే దీనిని ఖాళీగా వదిలేయండి. |

### ఆశించిన JSON పేలోడ్ (Expected JSON Payload)

మీ ఫీడ్‌బ్యాక్ ఎండ్ పాయింట్ కనీసం ఈ క్రింది ఫీల్డ్‌లతో కూడిన JSON బాడీని అంగీకరించాలి:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "ధరల గురించి సమాధానం తప్పుగా ఉంది.",
  "triage_category": "factual_error"
}
```

సంభాషణ సందర్భం (conversation context) ఆధారంగా పేలోడ్‌లో అదనపు ఫీల్డ్‌లు ఉండవచ్చు.

### `triage_category` విలువలు

పేలోడ్‌ను ఫార్వార్డ్ చేయడానికి ముందు, AI ట్రియాజ్ లేయర్ (AI triage layer) `triage_category` కు ఈ క్రింది విలువల్లో ఒకదాన్ని కేటాయిస్తుంది:

| విలువ | అర్థం |
|---|---|
| `factual_error` | అసిస్టెంట్ తప్పుడు వాస్తవ సమాచారాన్ని అందించింది. |
| `unhelpful_answer` | ప్రతిస్పందన సాంకేతికంగా సరైనది కానీ ఉపయోగకరంగా లేదు. |
| `inappropriate_content` | ప్రతిస్పందనలో వినియోగదారులకు చూపకూడని కంటెంట్ ఉంది. |
| `other` | ఫీడ్‌బ్యాక్ తెలిసిన కేటగిరీకి సరిపోలేదు. |

### ఆథెంటికేషన్ (Authentication)

మీ ఎండ్ పాయింట్‌కు ఆథెంటికేషన్ అవసరమైతే, **Feedback API Key** ఫీల్డ్‌లో మీ బేరర్ టోకెన్‌ను సెట్ చేయండి. ఏజెంట్ ఈ విధంగా పంపుతుంది:

```
Authorization: Bearer <your-api-key>
```

**Feedback API Key** ఫీల్డ్ ఖాళీగా ఉంటే, ఏ `Authorization` హెడర్ కూడా పంపబడదు.

### ఫీడ్‌బ్యాక్ సేకరణను నిలిపివేయడం (Disabling Feedback Collection)

**Feedback Endpoint URL** మరియు **Feedback API Key** రెండింటినీ ఖాళీగా వదిలేయండి. థంబ్స్-డౌన్ బటన్ మరియు ఫీడ్‌బ్యాక్ UI వినియోగదారులకు కనిపించినప్పటికీ, సమర్పణలు ఏ బాహ్య సేవకు కూడా ఫార్వార్డ్ చేయబడవు.

## Brave Search API Key

**Advanced** ట్యాబ్‌లో కూడా, **Brave Search API Key** ఫీల్డ్ [Internet Search](../configuration/internet-search) సామర్థ్యాన్ని ఎనేబుల్ చేస్తుంది.

| ఫీల్డ్ | వివరణ |
|---|---|
| **Brave Search API Key** | Brave Search developer dashboard నుండి మీ API కీ. AI అసిస్టెంట్‌లో ఇంటర్నెట్ సెర్చ్‌ను ఎనేబుల్ చేయడానికి ఇది అవసరం. |

ఈ ఫీల్డ్ లేబుల్‌లో Brave Search API సైన్-అప్ పేజీకి క్లిక్ చేయదగిన లింక్ ఉంటుంది. ఇంటర్నెట్ సెర్చ్‌ను డిసేబుల్ చేయడానికి దీనిని ఖాళీగా వదిలేయండి.

ఈ ఫీచర్‌పై ఎండ్-యూజర్ డాక్యుమెంటేషన్ కోసం [Internet Search](../configuration/internet-search) చూడండి.

## ఫీచర్ ఫ్లాగ్‌లు (Feature Flags)

v1.9.0 లో కూడా పరిచయ చేయబడిన **Settings → Feature Flags** ట్యాబ్, ఐచ్ఛిక కార్యాచరణ కోసం టోగుల్ స్విచ్‌లను అందిస్తుంది. ప్రతి ఫ్లాగ్ నెట్‌వర్క్-వ్యాప్తంగా ఎనేబుల్ చేయబడిందా లేదా డిసేబుల్ చేయబడిందా అని ఉంటుంది; ప్రస్తుతం దీనికి per-site ఓవర్‌రైడ్ లేదు.

### ఫీచర్ ఫ్లాగ్‌లను యాక్సెస్ చేయడం

1. WordPress అడ్మిన్ లో, **Gratis AI Agent → Settings** కి వెళ్లండి.
2. **Feature Flags** ట్యాబ్‌పై క్లిక్ చేయండి.

### యాక్సెస్ కంట్రోల్ ఫ్లాగ్‌లు (Access Control Flags)

| ఫ్లాగ్ | డిఫాల్ట్ | వివరణ |
|---|---|---|
| **Restrict to Administrators** | Off | ఇది ఎనేబుల్ చేయబడినప్పుడు, `administrator` రోల్ ఉన్న వినియోగదారులు మాత్రమే AI Agent chat panel ను తెరవగలరు. మిగిలిన అన్ని రోల్స్ "దయచేసి మీ అడ్మినిస్ట్రేటర్‌ను సంప్రదించండి" అనే సందేశాన్ని చూస్తాయి. |
| **Restrict to Network Admins** | Off | మల్టీసైట్ నెట్‌వర్క్‌లో ఇది ఎనేబుల్ చేయబడినప్పుడు, సూపర్ అడ్మిన్స్ మాత్రమే ఏజెంట్‌ను ఉపయోగించగలరు. వ్యక్తిగత సైట్ అడ్మిన్స్‌కు యాక్సెస్ బ్లాక్ చేయబడుతుంది. రెండూ ఎనేబుల్ చేయబడితే, ఇది "Restrict to Administrators" కంటే ఎక్కువ ప్రాధాన్యతనిస్తుంది. |
| **Allow Subscriber Access** | Off | ఇది ఎనేబుల్ చేయబడినప్పుడు, `subscriber` రోల్ ఉన్న వినియోగదారులు చాట్ ఇంటర్‌ఫేస్‌ను ఉపయోగించగలరు, కానీ వారు రీడ్-ఓన్లీ సామర్థ్యాలకు పరిమితమవుతారు (పోస్ట్ సృష్టించడం లేదా సెట్టింగ్‌లను మార్చడం చేయలేరు). |
| **Disable for Non-Members** | Off | Ultimate Multisite membership status తో ఇంటిగ్రేట్ అవుతుంది. ఇది ఎనేబుల్ చేయబడినప్పుడు, చురుకైన membership లేని సైట్‌ల కోసం చాట్ దాచబడుతుంది. |

### బ్రాండింగ్ ఫ్లాగ్‌లు (Branding Flags)

| ఫ్లాగ్ | డిఫాల్ట్ | వివరణ |
|---|---|---|
| **Hide "Powered by Gratis AI Agent" Footer** | Off | చాట్ విడ్జెట్ దిగువన చూపబడే బ్రాండింగ్ అట్రిబ్యూషన్ లైన్‌ను తొలగిస్తుంది. వైట్-లేబుల్ డిప్లాయ్‌మెంట్‌లకు సిఫార్సు చేయబడింది. |
| **Custom Agent Name** | *(blank)* | చాట్ హెడర్‌లో మరియు అడ్మిన్ మెనూలో డిఫాల్ట్ "Gratis AI Agent" లేబుల్‌ను మీ స్వంత ఉత్పత్తి పేరుతో భర్తీ చేస్తుంది. డిఫాల్ట్‌ను ఉపయోగించడానికి ఖాళీగా వదిలేయండి. |
| **Hide Agent Picker** | Off | ఇది ఎనేబుల్ చేయబడినప్పుడు, వినియోగదారులు ఐదు అంతర్నిర్మిత ఏజెంట్ల మధ్య మారలేరు. ప్రస్తుత ఏజెంట్ Settings → General లో డిఫాల్ట్‌గా కాన్ఫిగర్ చేయబడినదే స్థిరంగా ఉంటుంది. |
| **Use Site Icon as Chat Avatar** | Off | చాట్ విడ్జెట్ హెడర్‌లో డిఫాల్ట్ AI ఐకాన్‌ను WordPress సైట్ ఐకాన్‌తో భర్తీ చేస్తుంది (Appearance → Customize → Site Identity కింద సెట్ చేయబడింది). |

### మార్పులు వర్తింపజేయడం (Applying Changes)

ఏ ఫ్లాగ్‌ను టోగుల్ చేసిన తర్వాత **Save Settings** పై క్లిక్ చేయండి. మార్పులు తక్షణమే అమలులోకి వస్తాయి — కాష్ ఫ్లష్ లేదా ప్లగిన్ రియాక్టివేషన్ అవసరం లేదు.
