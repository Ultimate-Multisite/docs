# Gratis AI Agent సెట్టింగ్‌లు

Gratis AI Agent లోని **సెట్టింగ్‌లు → అధునాతన** స్క్రీన్, బ్యాకెండ్ అనుసంధానాల కోసం నిర్వాహక స్థాయి ఆకృతీకరణను అందిస్తుంది. ఈ పేజీ అభిప్రాయ ఫార్వార్డింగ్, శోధన ప్రదాత కీలు, నిర్వహిత Superdav సేవ సెటప్, Google Calendar నియంత్రణలు, TextBee SMS సెట్టింగ్‌లు, మరియు నెట్‌వర్క్-వ్యాప్త ఫీచర్ ఫ్లాగ్‌లను వివరిస్తుంది.

## సెట్టింగ్‌లను తెరవడం {#accessing-settings}

1. WordPress నిర్వాహకంలో, **Gratis AI Agent → సెట్టింగ్‌లు** కు వెళ్లండి.
2. **అధునాతన** ట్యాబ్‌ను క్లిక్ చేయండి.

## అభిప్రాయ ఎండ్‌పాయింట్ ఆకృతీకరణ {#feedback-endpoint-configuration}

వినియోగదారు థంబ్స్-డౌన్ బటన్, ఆటో-ప్రాంప్ట్ బ్యానర్, లేదా `/report-issue` కమాండ్ ద్వారా అభిప్రాయం సమర్పించినప్పుడల్లా, అభిప్రాయ ఎండ్‌పాయింట్ AI ఏజెంట్ నుండి POST అభ్యర్థనలను స్వీకరిస్తుంది.

| ఫీల్డ్ | వివరణ |
|---|---|
| **అభిప్రాయ ఎండ్‌పాయింట్ URL** | JSON బాడీతో HTTP POST అభ్యర్థనలుగా అభిప్రాయ సమర్పణలను స్వీకరించే URL. |
| **Feedback API Key** | ప్రతి అభిప్రాయ అభ్యర్థనలోని `Authorization` header లో పంపబడే bearer token. మీ ఎండ్‌పాయింట్‌కు ప్రామాణీకరణ అవసరం లేకపోతే ఖాళీగా వదిలేయండి. |

### ఆశించే JSON పేలోడ్ {#expected-json-payload}

మీ అభిప్రాయ ఎండ్‌పాయింట్ కనీసం క్రింది ఫీల్డ్‌లతో JSON బాడీని అంగీకరించాలి:

```json
{
  "message_id": "msg_abc123",
  "conversation_id": "conv_xyz789",
  "feedback_text": "The answer was incorrect about pricing.",
  "triage_category": "factual_error"
}
```

సంభాషణ సందర్భాన్ని బట్టి పేలోడ్‌లో అదనపు ఫీల్డ్‌లు ఉండవచ్చు.

### `triage_category` విలువలు {#triagecategory-values}

పేలోడ్‌ను ఫార్వార్డ్ చేయడానికి ముందు AI ట్రయాజ్ లేయర్ `triage_category` కి క్రింది విలువలలో ఒకదాన్ని కేటాయిస్తుంది:

| విలువ | అర్థం |
|---|---|
| `factual_error` | అసిస్టెంట్ తప్పు వాస్తవ సమాచారాన్ని అందించింది. |
| `unhelpful_answer` | స్పందన సాంకేతికంగా సరైనదే కానీ ఉపయోగకరంగా లేదు. |
| `inappropriate_content` | స్పందనలో వినియోగదారులకు చూపకూడని కంటెంట్ ఉంది. |
| `other` | అభిప్రాయం తెలిసిన వర్గానికి సరిపోలలేదు. |

### ప్రామాణీకరణ {#authentication}

మీ ఎండ్‌పాయింట్‌కు ప్రామాణీకరణ అవసరమైతే, **Feedback API Key** ఫీల్డ్‌ను మీ bearer token గా సెట్ చేయండి. ఏజెంట్ పంపేది:

```
Authorization: Bearer <your-api-key>
```

**Feedback API Key** ఫీల్డ్ ఖాళీగా ఉంటే, `Authorization` header పంపబడదు.

### అభిప్రాయ సేకరణను నిలిపివేయడం {#disabling-feedback-collection}

**అభిప్రాయ ఎండ్‌పాయింట్ URL** మరియు **Feedback API Key** ఫీల్డ్‌లను రెండింటినీ ఖాళీగా వదిలేయండి. థంబ్స్-డౌన్ బటన్ మరియు అభిప్రాయ UI వినియోగదారులకు కనిపిస్తూనే ఉంటాయి, కానీ సమర్పణలు ఏ బాహ్య సేవకూ ఫార్వార్డ్ చేయబడవు.

## Brave Search API Key {#brave-search-api-key}

**అధునాతన** ట్యాబ్‌లోనే, **Brave Search API Key** ఫీల్డ్ [ఇంటర్నెట్ శోధన](../configuration/internet-search) సామర్థ్యాన్ని ప్రారంభిస్తుంది.

| ఫీల్డ్ | వివరణ |
|---|---|
| **Brave Search API Key** | Brave Search డెవలపర్ డ్యాష్‌బోర్డ్ నుండి మీ API కీ. AI అసిస్టెంట్‌లో ఇంటర్నెట్ శోధనను ప్రారంభించడానికి అవసరం. |

ఫీల్డ్ లేబుల్‌లో Brave Search API నమోదు పేజీకి క్లిక్ చేయగల లింక్ ఉంటుంది. ఇంటర్నెట్ శోధనను నిలిపివేయడానికి ఖాళీగా వదిలేయండి.

ఈ ఫీచర్‌పై తుది వినియోగదారుల పత్రాల కోసం [ఇంటర్నెట్ శోధన](../configuration/internet-search) చూడండి.

## నిర్వహిత Superdav సేవ {#managed-superdav-service}

Superdav AI Agent v1.18.0 మద్దతు ఉన్న సైట్‌ల కోసం నిర్వహిత Superdav సేవ ఎండ్‌పాయింట్‌లను మరియు స్వయంచాలక కనెక్షన్ ప్రొవిజనింగ్‌ను జోడిస్తుంది. మీ సైట్ చేతితో ఆకృతీకరించిన సేవ ఎండ్‌పాయింట్‌కు బదులుగా హోస్ట్ చేసిన ప్రదాతకు కనెక్ట్ కావాల్సినప్పుడు ఈ నియంత్రణలను ఉపయోగించండి.

| ఫీల్డ్ | వివరణ |
|---|---|
| **నిర్వహిత Superdav సేవ** | మద్దతు ఉన్న సైట్‌ల కోసం హోస్ట్ చేసిన Superdav సేవ కనెక్షన్‌ను ప్రారంభిస్తుంది. |
| **కనెక్షన్ ప్రొవిజన్ చేయండి** | స్వయంచాలక ఎండ్‌పాయింట్ మరియు క్రెడెన్షియల్ ప్రొవిజనింగ్‌ను ప్రారంభిస్తుంది. సైట్ నిర్వహిత ప్రదాతను ఉపయోగించాలి అని నిర్ధారించిన తర్వాత దీన్ని ఉపయోగించండి. |
| **సేవ ఎండ్‌పాయింట్ / కనెక్షన్ స్థితి** | ప్రొవిజనింగ్ తర్వాత ప్రస్తుత ఎండ్‌పాయింట్ లేదా కనెక్షన్ స్థితిని చూపుతుంది. |

ప్రొవిజనింగ్ తర్వాత, సెట్టింగ్‌లను సేవ్ చేసి, నిర్వహిత-సేవ వర్క్‌ఫ్లోలపై ఆధారపడే ముందు కనెక్షన్ స్థితిని ధృవీకరించండి. ప్రొవిజనింగ్ విఫలమైతే, చూపించిన మళ్లీ ప్రయత్నించే మార్గదర్శకాన్ని సమీక్షించి, హోస్ట్ చేసిన ప్రదాతను ఉపయోగించడానికి సైట్‌కు అనుమతి ఉందని నిర్ధారించండి.

## Google Calendar ఆకృతీకరణ {#google-calendar-configuration}

Superdav AI Agent v1.18.0 క్యాలెండర్ ఫీచర్‌లు ప్రారంభించబడినప్పుడు, ఏజెంట్ ఆకృతీకరించిన క్యాలెండర్‌లు మరియు ఈవెంట్ వివరాలను చదవగలదు. క్యాలెండర్ టూల్‌లు చదవడానికే ఉద్దేశించబడ్డాయి. షెడ్యూల్‌ను పరిగణించే రిమైండర్‌లు, హాజరుల ఫాలో-అప్, మరియు పరిచయ సరిపోలికలకు ఇవి ఉపయోగకరంగా ఉంటాయి.

| ఫీల్డ్ | వివరణ |
|---|---|
| **Google Calendar క్రెడెన్షియల్‌లు** | క్యాలెండర్ డేటాను చదవడానికి అవసరమైన క్రెడెన్షియల్‌లు లేదా token కనెక్షన్‌ను నిల్వ చేస్తుంది. |
| **క్యాలెండర్ ఎంపిక** | ఏ ఆకృతీకరించిన క్యాలెండర్‌లను ఏజెంట్ పరిశీలించవచ్చో పరిమితం చేస్తుంది. |
| **క్యాలెండర్ కనెక్షన్ స్థితి** | ప్రస్తుత క్రెడెన్షియల్‌లు క్యాలెండర్‌లు మరియు ఈవెంట్‌లను చదవగలవో లేదో నిర్ధారిస్తుంది. |

క్యాలెండర్ క్రెడెన్షియల్‌లను ఏజెంట్‌కు అవసరమైన క్యాలెండర్‌లకే పరిమితం చేయండి. స్థితి token గడువు ముగిసిందని సూచిస్తే, క్రెడెన్షియల్‌లను మళ్లీ కనెక్ట్ చేయండి లేదా మార్చండి.

## TextBee SMS నోటిఫికేషన్‌లు {#textbee-sms-notifications}

Superdav AI Agent v1.18.0 ఆకృతీకరించిన నోటిఫికేషన్ వర్క్‌ఫ్లోల కోసం TextBee ని SMS ప్రదాతగా జోడిస్తుంది. సున్నితమైన లేదా వినియోగదారులకు కనిపించే సందేశాల కోసం SMS నోటిఫికేషన్‌లను మానవ ఆమోద దశలతో జత చేయాలి.

| ఫీల్డ్ | వివరణ |
|---|---|
| **TextBee API Key** | TextBee SMS ప్రదాతకు వెళ్లే అభ్యర్థనలను ప్రామాణీకరిస్తుంది. |
| **TextBee పరికరం / పంపేవారు** | ప్రదాతకు అవసరమైనప్పుడు, బయటికెళ్లే సందేశాలకు ఉపయోగించే TextBee పంపేవారిని లేదా పరికరాన్ని ఎంచుకుంటుంది. |
| **SMS నోటిఫికేషన్‌లు ప్రారంభించబడ్డాయి** | ఆమోదించిన వర్క్‌ఫ్లోలు టెక్స్ట్-సందేశ నోటిఫికేషన్‌లను పంపడానికి అనుమతిస్తుంది. SMS పంపడాన్ని నివారించడానికి నిలిపివేసి ఉంచండి. |

పరీక్ష సందేశాన్ని నిర్వాహకుడి సొంత నంబర్‌కే పంపండి. ఆపై షెడ్యూల్ చేసిన లేదా హాజరులకు కనిపించే రిమైండర్‌లను ప్రారంభించే ముందు ఆమోద-దశ ప్రవర్తనను నిర్ధారించండి.

## ఫీచర్ ఫ్లాగ్‌లు {#feature-flags}

v1.9.0 లో కూడా పరిచయం చేసిన **సెట్టింగ్‌లు → ఫీచర్ ఫ్లాగ్‌లు** ట్యాబ్ ఐచ్చిక కార్యాచరణ కోసం టాగుల్ స్విచ్‌లను అందిస్తుంది. ప్రతి ఫ్లాగ్ నెట్‌వర్క్-వ్యాప్తంగా ప్రారంభించబడుతుంది లేదా నిలిపివేయబడుతుంది; ప్రస్తుతం ప్రతి సైట్‌కు విడిగా ఓవర్‌రైడ్ లేదు.

### ఫీచర్ ఫ్లాగ్‌లను తెరవడం {#accessing-feature-flags}

1. WordPress నిర్వాహకంలో, **Gratis AI Agent → సెట్టింగ్‌లు** కు వెళ్లండి.
2. **ఫీచర్ ఫ్లాగ్‌లు** ట్యాబ్‌ను క్లిక్ చేయండి.

### యాక్సెస్ నియంత్రణ ఫ్లాగ్‌లు {#access-control-flags}

### మార్పులను వర్తింపజేయడం {#applying-changes}

ఏదైనా ఫ్లాగ్‌ను మార్చిన తర్వాత **సెట్టింగ్‌లను సేవ్ చేయండి** క్లిక్ చేయండి. మార్పులు వెంటనే అమలులోకి వస్తాయి — క్యాష్ ఫ్లష్ లేదా plugin మళ్లీ యాక్టివేషన్ అవసరం లేదు.
