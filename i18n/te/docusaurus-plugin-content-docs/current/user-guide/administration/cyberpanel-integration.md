# CyberPanel Integration

ఈ గైడ్ Ultimate Multisite CyberPanel ఇంటిగ్రేషన్‌ను ఎలా కాన్ఫిగర్ చేయాలో వివరిస్తుంది, తద్వారా మీ నెట్‌వర్క్‌లోని మ్యాప్ చేయబడిన డొమైన్‌లు CyberPanelలో వర్చువల్ హోస్ట్‌లుగా ఆటోమేటిక్‌గా జోడించబడతాయి (మరియు తొలగించబడతాయి). దీనితో పాటు, Let's Encrypt ద్వారా ఆప్షనల్ ఆటో-SSL ప్రొవిజనింగ్ కూడా అందుబాటులో ఉంటుంది.

## ఇది ఏమి చేస్తుంది

- Ultimate Multisiteలో ఒక డొమైన్ మ్యాప్ చేయబడినప్పుడు, ఈ ఇంటిగ్రేషన్ ఆ డొమైన్‌ కోసం వర్చువల్ హోస్ట్ సృష్టించడానికి CyberPanel APIని కాల్ చేస్తుంది.
- డొమైన్ మ్యాపింగ్ తొలగించబడినప్పుడు, సంబంధిత వర్చువల్ హోస్ట్‌ను తొలగించడానికి ఇంటిగ్రేషన్ APIని కాల్ చేస్తుంది.
- ఆటో-SSL ఎనేబుల్ చేయబడితే, వర్చువల్ హోస్ట్ సృష్టించబడిన వెంటనే, ఇంటిగ్రేషన్ Let's Encrypt సర్టిఫికేట్ జారీని ట్రిగ్గర్ చేస్తుంది.
- డొమైన్ మ్యాపింగ్ సెట్టింగ్‌లలోని "Auto-create www subdomain" సెట్టింగ్‌ను బట్టి, ఇది ఆప్షనల్‌గా `www.`ని జోడించడం/తొలగించడం చేస్తుంది.

## ముందస్తు అవసరాలు (Prerequisites)

- రన్ అవుతున్న CyberPanel ఇన్‌స్టాన్స్ (v2.3 లేదా అంతకన్నా ఎక్కువ సిఫార్సు చేయబడింది), ఇది మీ WordPress సర్వర్ నుండి యాక్సబుల్గా ఉండాలి.
- మీ WordPress నెట్‌వర్క్ రూట్‌ను ఇప్పటికే సర్వ్ చేస్తున్న CyberPanelలో ఒక ఇప్పటికే ఉన్న వెబ్‌సైట్ అవసరం. ఈ ఇంటిగ్రేషన్ కొత్త వర్చువల్ హోస్ట్‌లను ఈ సర్వర్‌కు జత చేస్తుంది.
- CyberPanel API యాక్సెస్ ఎనేబుల్ చేయబడి ఉండాలి. ప్రామాణీకరణ (Authentication) కోసం మీ CyberPanel అడ్మిన్ యూజర్‌నేమ్ మరియు పాస్‌వర్డ్‌ను ఉపయోగిస్తారు.
- ఆటో-SSL చెల్లుబాటు అయ్యే సర్టిఫికేట్‌ను జారీ చేయడానికి, మ్యాప్ చేయబడిన డొమైన్‌ల కోసం మీ DNS రికార్డులు ఇప్పటికే మీ సర్వర్ యొక్క IP చిరునామాను చూపించాలి.

## అవసరాలు (Requirements)

కింది కాన్స్టాంట్‌లను మీ `wp-config.php` ఫైల్‌లో నిర్వచించాలి:

```php
define('WU_CYBERPANEL_HOST', 'https://your-cyberpanel-host:8090');
define('WU_CYBERPANEL_USERNAME', 'your_admin_username');
define('WU_CYBERPANEL_PASSWORD', 'your_admin_password');
define('WU_CYBERPANEL_PACKAGE', 'Default');
```

ఐచ్ఛికంగా, మీరు వీటిని కూడా నిర్వచించవచ్చు:

```php
define('WU_CYBERPANEL_AUTO_SSL', true);          // డిఫాల్ట్: true — డొమైన్ సృష్టి తర్వాత Let's Encrypt SSL జారీ చేస్తుంది
define('WU_CYBERPANEL_PHP_VERSION', 'PHP 8.2');  // డిఫాల్ట్: PHP 8.2
define('WU_CYBERPANEL_EMAIL', 'admin@yourdomain.com'); // SSL సర్టిఫికేట్ కాంటాక్ట్ కోసం ఉపయోగించబడుతుంది
```

## సెటప్ సూచనలు (Setup Instructions)

### 1. CyberPanel APIని ఎనేబుల్ చేయండి

1. అడ్మినిస్ట్రేటర్‌గా మీ CyberPanel డాష్‌బోర్డ్‌లోకి లాగిన్ అవ్వండి.
2. **Security** > **SSL**కి వెళ్లి, CyberPanel ఇంటర్‌ఫేస్‌లోనే SSL యాక్టివ్‌గా ఉందని నిర్ధారించుకోండి (సురక్షిత API కాల్‌లకు ఇది అవసరం).
3. CyberPanel API డిఫాల్ట్‌గా `https://your-server-ip:8090/api/` వద్ద అందుబాటులో ఉంటుంది. దీనిని ఎనేబుల్ చేయడానికి అదనపు దశలు అవసరం లేదు — అడ్మిన్ యూజర్‌ల కోసం ఇది డిఫాల్ట్‌గా ఆన్‌లో ఉంటుంది.

### 2. wp-config.php లో కాన్స్టాంట్‌లను జోడించండి

కింది కాన్స్టాంట్‌లను `/* That's all, stop editing! */` లైన్ కంటే ముందు మీ `wp-config.php` ఫైల్‌లో జోడించండి:

```php
define('WU_CYBERPANEL_HOST', 'https://your-cyberpanel-host:8090');
define('WU_CYBERPANEL_USERNAME', 'admin');
define('WU_CYBERPANEL_PASSWORD', 'your_secure_password');
define('WU_CYBERPANEL_PACKAGE', 'Default');
```

ఆటో-SSLని ఎనేబుల్ చేయడానికి (సిఫార్సు చేయబడింది):

```php
define('WU_CYBERPANEL_AUTO_SSL', true);
define('WU_CYBERPANEL_EMAIL', 'admin@yourdomain.com');
```

### 3. ఇంటిగ్రేషన్‌ను ఎనేబుల్ చేయండి

1. మీ WordPress నెట్‌వర్క్ అడ్మిన్లో, **Ultimate Multisite** > **Settings**కి వెళ్లండి.
2. **Domain Mapping** ట్యాబ్‌కి నావిగేట్ చేయండి.
3. **Host Integrations** వరకు స్క్రోల్ చేయండి.
4. **CyberPanel** ఇంటిగ్రేషన్‌ను ఎనేబుల్ చేయండి.
5. **Save Changes** క్లిక్ చేయండి.

### 4. కనెక్టివిటీని ధృవీకరించండి (Verify Connectivity)

సెట్టింగ్‌ల విజర్డ్‌లోని అంతర్నిర్మిత కనెక్షన్ పరీక్షను ఉపయోగించండి:

1. **Ultimate Multisite** > **Settings** > **Domain Mapping** > **Host Integrations** > **CyberPanel**కి వెళ్లండి.
2. **Test Connection** క్లిక్ చేయండి.
3. ఒక విజయ సందేశం (success message) ప్లగిన్‌కు CyberPanel APIకి చేరుకోగలదని మరియు సరిగ్గా ప్రామాణీకరించగలదని నిర్ధారిస్తుంది.

## ఇది ఎలా పనిచేస్తుంది (How It Works)

### డొమైన్ మ్యాపింగ్ (Domain Mapping)

Ultimate Multisiteలో ఒక డొమైన్ మ్యాప్ చేయబడినప్పుడు:

1. ఇంటిగ్రేషన్ మీ CyberPanel హోస్ట్‌లోని `/api/createWebsite` కు ఒక `POST` రిక్వెస్ట్‌ను పంపుతుంది.
2. CyberPanel కాన్ఫిగర్ చేయబడిన ప్యాకేజీ కింద ఆ డొమైన్ కోసం కొత్త వర్చువల్ హోస్ట్‌ను సృష్టిస్తుంది.
3. డాక్యుమెంట్ రూట్ మీ WordPress నెట్‌వర్క్ రూట్ డైరెక్టరీని చూపించడానికి సెట్ చేయబడుతుంది.
4. డొమైన్ మ్యాపింగ్ తొలగించబడినప్పుడు, వర్చువల్ హోస్ట్‌ను క్లీన్ చేయడానికి ఇంటిగ్రేషన్ `/api/deleteWebsite`ని కాల్ చేస్తుంది.

### ఆటో-SSL (Auto-SSL)

`WU_CYBERPANEL_AUTO_SSL` `true` అయినప్పుడు:

1. వర్చువల్ హోస్ట్ సృష్టించబడిన తర్వాత, ఇంటిగ్రేషన్ డొమైన్ కోసం `/api/issueSSL`ని కాల్ చేస్తుంది.
2. CyberPanel ACME HTTP-01 ఛాలెంజ్‌ని ఉపయోగించి Let's Encrypt సర్టిఫికేట్‌ను అభ్యర్థిస్తుంది.
3. సర్టిఫికేట్ గడువు ముగియడానికి ముందు CyberPanel ద్వారా ఆటోమేటిక్‌గా పునరుద్ధరించబడుతుంది.

> **ముఖ్య గమనిక:** Let's Encrypt డొమైన్‌ను ధృవీకరించడానికి DNS మీ సర్వర్ IP చిరునామాకు పూర్తిగా ప్రొపగేట్ (propagate) అయి ఉండాలి. మ్యాపింగ్ అయిన వెంటనే SSL జారీ విఫలమైతే, DNS ప్రొపగేషన్ కోసం వేచి ఉండండి మరియు CyberPanel డాష్‌బోర్డ్‌లో **SSL** > **Manage SSL** కింద SSLని మళ్లీ ట్రిగ్గర్ చేయండి.

### www సబ్-డొమైన్ (www Subdomain)

మీ డొమైన్ మ్యాపింగ్ సెట్టింగ్‌లలో **Auto-create www subdomain** ఎనేబుల్ చేయబడితే, ఇంటిగ్రేషన్ `www.<domain>` కోసం వర్చువల్ హోస్ట్ మారుపేరును (alias) కూడా సృష్టిస్తుంది మరియు ఆటో-SSL ఆన్‌లో ఉన్నప్పుడు, అపెక్స్ మరియు www వేరియంట్‌ల రెండింటినీ కవర్ చేసే సర్టిఫికేట్‌ను జారీ చేస్తుంది.

### ఇమెయిల్ ఫార్వర్డర్‌లు (Email Forwarders)

[Ultimate Multisite: Emails](../../addons/ultimate-multisite-emails/) యాడ్‌ఆన్ యాక్టివ్‌గా ఉన్నప్పుడు, CyberPanel కస్టమర్ ఇమెయిల్ ఫార్వర్డర్‌లను కూడా అందించగలదు. ఫార్వర్డర్‌లు పూర్తి మెయిల్‌బాక్స్ సృష్టించకుండానే, ఒక డొమైన్ చిరునామా నుండి మరొక ఇన్‌బాక్స్‌కు సందేశాలను రూట్ చేస్తాయి. ఇది `info@customer-domain.test` లేదా `support@customer-domain.test` వంటి మారుపేర్లకు ఉపయోగపడుతుంది.

కస్టమర్ల కోసం ఫార్వర్డర్‌లను ఎనేబుల్ చేయడానికి ముందు:

1. పైన పేర్కొన్న CyberPanel కాన్స్టాంట్‌లు కాన్ఫిగర్ చేయబడి ఉన్నాయని మరియు కనెక్షన్ పరీక్ష పాస్ అయినట్లు నిర్ధారించుకోండి.
2. ఇమెయిల్స్ యాడ్‌ఆన్ సెట్టింగ్‌లలో CyberPanel ఇమెయిల్ ప్రొవైడర్‌ను ఎనేబుల్ చేయండి.
3. ఫార్వర్డర్‌ను సృష్టించే ముందు కస్టమర్ డొమైన్ CyberPanelలో ఇప్పటికే ఉందో లేదో నిర్ధారించుకోండి.
4. ప్రొడక్షన్ ప్లాన్‌లలో ఈ ఫీచర్‌ను అందించడానికి ముందు, ఒక టెస్ట్ ఫార్వర్డర్‌ను సృష్టించి, దాని ద్వారా ఒక సందేశాన్ని పంపండి.

ఫార్వర్డర్ సృష్టి విఫలమైతే, ముందుగా Ultimate Multisite యాక్టివిటీ లాగ్‌లను తనిఖీ చేయండి, ఆపై CyberPanelలో సోర్స్ డొమైన్ ఉందో లేదో మరియు API యూజర్‌కు ఇమెయిల్-మేనేజ్‌మెంట్ పర్మిషన్స్ ఉన్నాయో లేదో నిర్ధారించుకోండి.

## కాన్ఫిగరేషన్ రిఫరెన్స్ (Configuration Reference)

| Constant | Required | Default | Description |
|---|---|---|---|
| `WU_CYBERPANEL_HOST` | Yes | — | మీ CyberPanel ఇన్‌స్టాన్స్‌కు పోర్ట్‌తో సహా పూర్తి URL, ఉదా. `https://cp.example.com:8090` |
| `WU_CYBERPANEL_USERNAME` | Yes | — | CyberPanel అడ్మిన్ యూజర్‌నేమ్ |
| `WU_CYBERPANEL_PASSWORD` | Yes | — | CyberPanel అడ్మిన్ పాస్‌వర్డ్ |
| `WU_CYBERPANEL_PACKAGE` | Yes | `Default` | కొత్త వర్చువల్ హోస్ట్‌లకు కేటాయించాల్సిన CyberPanel హోస్టింగ్ ప్యాకేజ్ |
| `WU_CYBERPANEL_AUTO_SSL` | No | `true` | డొమైన్ సృష్టి తర్వాత Let's Encrypt SSL సర్టిఫికేట్‌ను జారీ చేయండి |
| `WU_CYBERPANEL_PHP_VERSION` | No | `PHP 8.2` | కొత్త వర్చువల్ హోస్ట్‌ల కోసం PHP వెర్షన్ (CyberPanelలో ఇన్‌స్టాల్ చేయబడిన వెర్షన్‌తో సరిపోలాలి) |
| `WU_CYBERPANEL_EMAIL` | No | — | SSL సర్టిఫికేట్ రిజిస్ట్రేషన్ కోసం కాంటాక్ట్ ఇమెయిల్ |

## ముఖ్య గమనికలు (Important Notes)

- CyberPanel యొక్క API సెషన్-ఆధారిత టోకెన్ ప్రామాణీకరణను ఉపయోగిస్తుంది. ప్రతి API కాల్ వద్ద ఇంటిగ్రేషన్ టోకెన్ పొందడాన్ని ఆటోమేటిక్‌గా నిర్వహిస్తుంది.
- మీ CyberPanel అడ్మిన్ ఖాతాకు వెబ్‌సైట్‌లను సృష్టించడానికి మరియు తొలగించడానికి అనుమతులు ఉండాలి.
- CyberPanel డిఫాల్ట్‌గా పోర్ట్ `8090` పై నడుస్తుంది. మీ సర్వర్ ఫైర్‌వాల్‌ను ఉపయోగిస్తే, ఈ పోర్ట్ WordPress అప్లికేషన్ సర్వర్ నుండి యాక్సబుల్గా ఉందని నిర్ధారించుకోండి.
- ఈ ఇంటిగ్రేషన్ DNS రికార్డులను నిర్వహించదు. Ultimate Multisiteలో డొమైన్‌ను మ్యాప్ చేయడానికి ముందు మీరు డొమైన్ DNSని మీ సర్వర్ IP చిరునామాకు చూపించాలి.
- మీరు OpenLiteSpeed (OLS) ఉపయోగిస్తే, వర్చువల్ హోస్ట్ మార్పుల తర్వాత గ్రేస్ఫుల్ రీస్టార్ట్ ఆటోమేటిక్‌గా ట్రిగ్గర్ అవుతుంది. మాన్యువల్ జోక్యం అవసరం లేదు.

## ట్రబుల్‌షూటింగ్ (Troubleshooting)

### API కనెక్షన్ తిరస్కరించబడింది (API Connection Refused)

- మీ సర్వర్ ఫైర్‌వాల్‌లో పోర్ట్ `8090` ఓపెన్ అయ్యిందో లేదో ధృవీకరించండి.
- `WU_CYBERPANEL_HOST` విలువ సరైన ప్రోటోకాల్ (`https://`) మరియు పోర్ట్‌ను కలిగి ఉందని నిర్ధారించుకోండి.
- మీ CyberPanel SSL సర్టిఫికేట్ చెల్లుబాటు అవుతుందో లేదో తనిఖీ చేయండి; సెల్ఫ్-సైన్డ్ సర్టిఫికేట్‌లు TLS ధృవీకరణ వైఫల్యాలకు కారణం కావచ్చు. విశ్వసనీయ ప్రైవేట్ నెట్‌వర్క్ వాతావరణాలలో మాత్రమే `WU_CYBERPANEL_VERIFY_SSL`ని `false`కి సెట్ చేయండి.

### ప్రామాణీకరణ లోపాలు (Authentication Errors)

- CyberPanelలోకి నేరుగా లాగిన్ అవ్వడం ద్వారా మీ `WU_CYBERPANEL_USERNAME` మరియు `WU_CYBERPANEL_PASSWORD` సరైనవో కాదో నిర్ధారించుకోండి.
- పదేపదే లాగిన్ ప్రయత్నాలు విఫలమైతే CyberPanel ఖాతాలను లాక్ చేస్తుంది. లాక్‌అవుట్‌లు సంభవిస్తే CyberPanelలో **Security** > **Brute Force Monitor**ని తనిఖీ చేయండి.

### డొమైన్ సృష్టించబడలేదు (Domain Not Created)

- API లోపం సందేశాల కోసం Ultimate Multisite యాక్టివిటీ లాగ్‌లను (**Ultimate Multisite** > **Activity Logs**) తనిఖీ చేయండి.
- `WU_CYBERPANEL_PACKAGE`లో నిర్వచించబడిన ప్యాకేజీ CyberPanelలో ఉందో లేదో ధృవీకరించండి (**Packages** > **List Packages**).
- డొమైన్ CyberPanelలో ఇప్పటికే వెబ్‌సైట్‌గా నమోదు చేయబడలేదని నిర్ధారించుకోండి — డూప్లికేట్ వెబ్‌సైట్ సృష్టి లోపాన్ని తిరిగి ఇస్తుంది.

### SSL సర్టిఫికేట్ జారీ కాలేదు (SSL Certificate Not Issued)

- DNS పూర్తిగా ప్రొపగేట్ అయిందని నిర్ధారించుకోండి: `dig +short your-domain.com` మీ సర్వర్ IPని తిరిగి ఇవ్వాలి.
- Let's Encrypt రేట్ లిమిట్‌లను అమలు చేస్తుంది. మీరు ఇటీవల అదే డొమైన్ కోసం అనేక సర్టిఫికేట్‌లను జారీ చేసి ఉంటే, మళ్లీ ప్రయత్నించే ముందు వేచి ఉండండి.
- సర్టిఫికేట్ జారీ వైఫల్యాల వివరాల కోసం **Logs** > **Error Logs** కింద CyberPanel SSL లాగ్‌లను తనిఖీ చేయండి.
- బ్యాకప్‌గా, మీరు CyberPanel నుండి మాన్యువల్‌గా SSLని జారీ చేయవచ్చు: **SSL** > **Manage SSL** > డొమైన్‌ను ఎంచుకోండి > **Issue SSL**.
