# కస్టమర్ ఫీడ్‌బ్యాక్ & సమస్య రిపోర్టింగ్

Gratis AI Agent v1.5.0 లో అంతర్నిర్మిత (built-in) ఫీడ్‌బ్యాక్ మరియు సమస్య రిపోర్టింగ్ సిస్టమ్ జోడించబడింది. దీని ద్వారా వినియోగదారులు సహాయకారి కాని ప్రతిస్పందనలను గుర్తించి, చాట్ ఇంటర్‌ఫేస్ నుండి నేరుగా సమస్యలను రిపోర్ట్ చేయవచ్చు. ఈ సిస్టమ్‌లో సమ్మతి నిర్వహణ (consent management), ఆటోమేటెడ్ రిపోర్టింగ్ కమాండ్ మరియు బ్యాకెండ్‌లో AI సహాయంతో తనిఖీ (triage) చేయడం వంటివి ఉంటాయి.

## థంబ్స్-డౌన్ బటన్ {#thumbs-down-button}

AI అసిస్టెంట్ పంపే ప్రతి సందేశం పక్కన ఒక **థంబ్స్-డౌన్** (👎) బటన్ కనిపిస్తుంది. వినియోగదారు దీనిపై క్లిక్ చేసినప్పుడు, వారు ఆ ప్రతిస్పందన సహాయకారి కాదని లేదా తప్పు అని గుర్తించవచ్చు.

- ఈ బటన్ ప్రతి అసిస్టెంట్ సందేశం పక్కన మౌస్ హోవర్ చేసినప్పుడు కనిపిస్తుంది.
- దీనిపై క్లిక్ చేయడం వలన **ఫీడ్‌బ్యాక్ సమ్మతి మోడల్** (Feedback Consent Modal) ఓపెన్ అవుతుంది.
- ఈ ఫీడ్‌బ్యాక్ సంభాషణ థ్రెడ్‌కు మరియు నిర్దిష్ట సందేశానికి అనుసంధానించబడుతుంది.

## ఫీడ్‌బ్యాక్ సమ్మతి మోడల్ {#feedback-consent-modal}

వినియోగదారు థంబ్స్-డౌన్ బటన్‌పై క్లిక్ చేసినప్పుడు, ఏ డేటా కూడా పంపబడకముందే ఒక సమ్మతి మోడల్ కనిపిస్తుంది. ఈ మోడల్:

- ఏ సమాచారం సేకరించబడుతుందో వివరిస్తుంది (సంభాషణ భాగం, బ్రౌజర్ సందర్భం).
- ఈ డేటాను పంచుకోవడానికి వినియోగదారు సమ్మతిని నిర్ధారించమని అడుగుతుంది.
- వినియోగదారు ఏమి తప్పు జరిగిందో వివరించడానికి ఒక ఫ్రీ-టెక్స్ట్ ఫీల్డ్‌ను అందిస్తుంది.
- సమర్పించకుండా మూసివేయడానికి **Cancel** ఆప్షన్‌ను అందిస్తుంది.

వినియోగదారు స్పష్టంగా నిర్ధారించే వరకు ఎటువంటి ఫీడ్‌బ్యాక్ రికార్డ్ చేయబడదు.

## ఆటో-ప్రాంప్ట్ ఫీడ్‌బ్యాక్ బ్యానర్ {#auto-prompt-feedback-banner}

ఒక సంభాషణ ముగిసిన తర్వాత, అసిస్టెంట్ **ఆటో-ప్రాంప్ట్ ఫీడ్‌బ్యాక్ బ్యానర్** ప్రదర్శించవచ్చు — ఇది సెషన్ సహాయకరంగా ఉందా అని అడిగే ఒక అంతరాయం కలిగించని సందేశం.

ఈ బ్యానర్ సంభాషణ పొడవు మరియు ఫలితాల ఆధారంగా ఆటోమేటిక్‌గా కనిపిస్తుంది. ఇది థంబ్స్-డౌన్ బటన్‌తో సమానమైన ఫీడ్‌బ్యాక్ ఫ్లోకు లింక్ అవుతుంది. వినియోగదారులు ఫీడ్‌బ్యాక్ ఇవ్వకుండానే ఈ బ్యానర్‌ను మూసివేయవచ్చు.

## /report-issue స్లాష్ కమాండ్ {#report-issue-slash-command}

వినియోగదారులు చాట్ ఇన్‌పుట్‌లో `/report-issue` అని టైప్ చేయడం ద్వారా నేరుగా ఫీడ్‌బ్యాక్ ఫ్లోను ట్రిగ్గర్ చేయవచ్చు. ఈ కమాండ్:

- వెంటనే ఫీడ్‌బ్యాక్ సమ్మతి మోడల్‌ను ఓపెన్ చేస్తుంది.
- ప్రస్తుత సంభాషణ గురించి సందర్భంతో (context) డిస్క్రిప్షన్ ఫీల్డ్‌ను ముందుగానే నింపుతుంది (pre-populates).
- సమర్పించడానికి ముందు వినియోగదారులు అదనపు వివరాలను జోడించడానికి అనుమతిస్తుంది.

`/report-issue` కమాండ్ అన్ని చాట్ మోడ్‌లలో (inline, floating widget, full-screen) అందుబాటులో ఉంటుంది.

## AI-అసిస్టెడ్ తనిఖీ (AI-Assisted Triage) {#ai-assisted-triage}

సమర్పించిన ఫీడ్‌బ్యాక్ అనేది AI తనిఖీ పొరకు (triage layer) పంపబడుతుంది. ఇది:

- రిపోర్ట్‌ను వర్గీకరిస్తుంది (వాస్తవ లోపం, సహాయకారి కాని సమాధానం, తగని కంటెంట్, మొదలైనవి).
- సంబంధిత సంభాషణ మెటాడేటాను జత చేస్తుంది.
- తనిఖీ సారాంశాన్ని కాన్ఫిగర్ చేయబడిన **Feedback Endpoint** కు ఫార్వార్డ్ చేస్తుంది (ఎక్కడ చూడాలి: [Settings > Advanced](#settings-advanced)).

ఇది అత్యంత ముఖ్యమైన సమస్యలను ముందుగా గుర్తించడం ద్వారా మాన్యువల్ రివ్యూ సమయాన్ని తగ్గిస్తుంది.

## Settings > Advanced {#settings-advanced}

ఫీడ్‌బ్యాక్ బ్యాకెండ్‌ను ఎనేబుల్ చేయడానికి, **Gratis AI Agent → Settings → Advanced** కింద ఈ క్రింది ఫీల్డ్‌లను కాన్ఫిగర్ చేయండి:

| Field | Description |
|---|---|
| **Feedback Endpoint URL** | ఫీడ్‌బ్యాక్ పేలోడ్‌లతో (JSON) POST రిక్వెస్ట్‌లను స్వీకరించే URL. |
| **Feedback API Key** | ఫీడ్‌బ్యాక్ సమర్పణలను ధృవీకరించడానికి `Authorization` హెడర్‌లో పంపబడే Bearer టోకెన్. |

ఫీడ్‌బ్యాక్ సేకరణను డిసేబుల్ చేయడానికి రెండు ఫీల్డ్‌లను ఖాళీగా వదిలేయండి. వ్యక్తిగత ఫీడ్‌బ్యాక్ బటన్‌లు వినియోగదారులకు కనిపించినప్పటికీ, సమర్పణలు ఫార్వార్డ్ చేయబడవు.

:::tip
ఫీడ్‌బ్యాక్ ఎండ్‌పాయింట్ కనీసం `message_id`, `conversation_id`, `feedback_text`, మరియు `triage_category` ఫీల్డ్‌లతో కూడిన JSON బాడీని అంగీకరించాలి. ఆశించిన స్కీమా కోసం మీ ఎండ్‌పాయింట్ ప్రొవైడర్ డాక్యుమెంటేషన్‌ను చూడండి.
:::
