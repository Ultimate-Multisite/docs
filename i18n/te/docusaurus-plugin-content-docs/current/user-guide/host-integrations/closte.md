# Closte ఇంటిగ్రేషన్

## అవలోకనం
Closte అనేది Google Cloud ఇన్‌ఫ్రాస్ట్రక్చర్ పై నిర్మించబడిన మేనేజ్డ్ WordPress హోస్టింగ్ ప్లాట్‌ఫారమ్. ఈ ఇంటిగ్రేషన్ Ultimate Multisite మరియు Closte మధ్య ఆటోమేటిక్ డొమైన్ సింకింగ్ మరియు SSL సర్టిఫికేట్ మేనేజ్‌మెంట్‌ను అనుమతిస్తుంది.

## ఫీచర్లు
- ఆటోమేటిక్ డొమైన్ సింకింగ్
- SSL సర్టిఫికేట్ మేనేజ్‌మెంట్
- వైల్డ్‌కార్డ్ డొమైన్ సపోర్ట్
- Closte పై రన్ అవుతున్నట్లయితే కాన్ఫిగరేషన్ అవసరం లేదు

## అవసరాలు
మీరు Closte ఉపయోగిస్తుంటే, మీ `wp-config.php` ఫైల్‌లో కింది constant నిర్వచించబడి ఉండాలి:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

మీరు Closte పై హోస్ట్ చేస్తుంటే ఈ constant సాధారణంగా ఇప్పటికే నిర్వచించబడి ఉంటుంది.

## సెటప్ సూచనలు

### 1. మీ Closte API Key ను వెరిఫై చేయండి

మీరు Closte పై హోస్ట్ చేస్తుంటే, `CLOSTE_CLIENT_API_KEY` constant ఇప్పటికే మీ `wp-config.php` ఫైల్‌లో నిర్వచించబడి ఉండాలి. మీ `wp-config.php` ఫైల్‌ను తనిఖీ చేయడం ద్వారా మీరు దీనిని వెరిఫై చేయవచ్చు.

### 2. ఇంటిగ్రేషన్‌ను ఎనేబుల్ చేయండి

1. మీ WordPress admin లో, Ultimate Multisite > Settings కి వెళ్ళండి
2. "Domain Mapping" ట్యాబ్‌కు నావిగేట్ చేయండి
3. "Host Integrations" కు స్క్రోల్ చేయండి
4. Closte ఇంటిగ్రేషన్‌ను ఎనేబుల్ చేయండి
5. "Save Changes" క్లిక్ చేయండి

## ఇది ఎలా పని చేస్తుంది

Ultimate Multisite లో డొమైన్ మ్యాప్ చేయబడినప్పుడు:

1. ఇంటిగ్రేషన్ మీ అప్లికేషన్‌కు డొమైన్‌ను జోడించడానికి Closte యొక్క API కి రిక్వెస్ట్ పంపుతుంది
2. Closte ఆటోమేటిక్‌గా SSL సర్టిఫికేట్ ప్రొవిజనింగ్‌ను హ్యాండిల్ చేస్తుంది
3. డొమైన్ మ్యాపింగ్ తొలగించబడినప్పుడు, ఇంటిగ్రేషన్ Closte నుండి డొమైన్‌ను తొలగిస్తుంది

DNS ప్రొపగేషన్ మరియు SSL సర్టిఫికేట్ జారీ కోసం సిస్టమ్ ఎంత తరచుగా తనిఖీ చేయాలో కాన్ఫిగర్ చేయడానికి, Ultimate Multisite లోని DNS check interval సెట్టింగ్‌తో కూడా ఇంటిగ్రేషన్ పని చేస్తుంది.

## డొమైన్ రికార్డ్ క్రియేషన్

సైట్ క్రియేట్ చేయబడినప్పుడు లేదా డూప్లికేట్ చేయబడినప్పుడు, డొమైన్ రికార్డ్ ఆటోమేటిక్‌గా క్రియేట్ అవుతుందని ఈ ఇంటిగ్రేషన్ నిర్ధారిస్తుంది. Closte ఇంటిగ్రేషన్ కోసం ఇది చాలా ముఖ్యం, ఎందుకంటే డొమైన్ రికార్డ్ క్రియేషన్ Closte API ని డొమైన్ మరియు SSL సర్టిఫికేట్‌ను క్రియేట్ చేయడానికి ట్రిగ్గర్ చేస్తుంది.

## ట్రబుల్షూటింగ్

### API కనెక్షన్ సమస్యలు
- మీ Closte API key సరిగ్గా ఉందో వెరిఫై చేయండి
- మీ Closte అకౌంట్‌కు అవసరమైన పర్మిషన్లు ఉన్నాయని నిర్ధారించుకోండి

### SSL సర్టిఫికేట్ సమస్యలు
- SSL సర్టిఫికేట్‌లను జారీ చేయడానికి Closte కొంత సమయం తీసుకోవచ్చు (సాధారణంగా 5-10 నిమిషాలు)
- మీ డొమైన్‌లు మీ Closte సర్వర్ యొక్క IP అడ్రస్‌కు సరిగ్గా పాయింట్ అవుతున్నాయో వెరిఫై చేయండి
- DNS రికార్డ్‌లు సరిగ్గా కాన్ఫిగర్ చేయబడ్డాయో నిర్ధారించడానికి మీ డొమైన్ కోసం వాటిని తనిఖీ చేయండి

### డొమైన్ జోడించబడలేదు
- ఏదైనా ఎర్రర్ మెసేజ్‌ల కోసం Ultimate Multisite లాగ్‌లను తనిఖీ చేయండి
- డొమైన్ ఇప్పటికే Closte కు జోడించబడలేదని వెరిఫై చేయండి
- మీ డొమైన్ యొక్క DNS రికార్డ్‌లు సరిగ్గా కాన్ఫిగర్ చేయబడ్డాయని నిర్ధారించుకోండి

### DNS Check Interval
- SSL సర్టిఫికేట్‌లు జారీ చేయడానికి చాలా ఎక్కువ సమయం తీసుకుంటున్నట్లయితే, Domain Mapping సెట్టింగ్స్‌లో DNS check interval ను సర్దుబాటు చేయవచ్చు
- డిఫాల్ట్ ఇంటర్వల్ 300 సెకన్లు (5 నిమిషాలు), కానీ టెస్టింగ్ సమయంలో వేగవంతమైన తనిఖీ కోసం మీరు దీనిని 10 సెకన్ల వరకు తగ్గించవచ్చు
