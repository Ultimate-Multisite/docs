# Checkout క్షేత్రం మరియు వినియోగదారు DNS నిర్వహణ

## డొమైన్ ఎంపిక Checkout క్షేత్రం {#the-domain-selection-checkout-field}

**డొమైన్ ఎంపిక** క్షేత్రం అనేది వినియోగదారులు తమ సైట్ డొమైన్‌ను ఎలా పొందాలో ఎంచుకునే Checkout అంశం. డొమైన్ అమ్మకాన్ని ప్రారంభించడానికి దీన్ని ఏ Checkout ఫారమ్‌కైనా జోడించండి.

### Checkout ఫారమ్‌కు క్షేత్రాన్ని జోడించడం {#adding-the-field-to-a-checkout-form}

1. **నెట్‌వర్క్ అడ్మిన్ › Ultimate Multisite › Checkout ఫారమ్‌లు** కు వెళ్లండి
2. Checkout ఫారమ్‌ను తెరవండి లేదా సృష్టించండి
3. Checkout ఎడిటర్‌లో, **క్షేత్రాన్ని జోడించు** పై క్లిక్ చేయండి
4. క్షేత్రాల జాబితా నుండి **డొమైన్ ఎంపిక** ను ఎంచుకోండి
5. క్షేత్ర ఎంపికలను కాన్ఫిగర్ చేయండి (క్రింద చూడండి)
6. ఫారమ్‌ను సేవ్ చేయండి

### క్షేత్ర ఎంపికలు {#field-options}

**డొమైన్ మోడ్‌లు** — వినియోగదారుకు ఏ ట్యాబ్‌లు కనిపించాలో ఎంచుకోండి. ప్రతి మోడ్‌ను విడిగా ప్రారంభించవచ్చు లేదా నిలిపివేయవచ్చు:

| మోడ్ | ఇది ఏమి చేస్తుంది |
|---|---|
| **సబ్‌డొమైన్** | వినియోగదారు మీ నెట్‌వర్క్‌లో ఉచిత సబ్‌డొమైన్‌ను ఉపయోగిస్తారు (ఉదా., `mysite.yournetwork.com`). చెల్లింపు అవసరం లేదు. |
| **కొత్త డొమైన్ నమోదు** | వినియోగదారు కొత్త డొమైన్ కోసం వెతికి, మీరు కాన్ఫిగర్ చేసిన ప్రొవైడర్ ద్వారా దాన్ని నమోదు చేస్తారు. ధర కోసం సరిపోలే డొమైన్ ఉత్పత్తిని ఉపయోగిస్తుంది. |
| **ఇప్పటికే ఉన్న డొమైన్** | వినియోగదారు ఇప్పటికే తమ వద్ద ఉన్న డొమైన్‌ను మ్యాప్ చేస్తారు. నమోదు రుసుము లేదు. డొమైన్ వారి సైట్‌కు స్వయంచాలకంగా మ్యాప్ చేయబడుతుంది. |

**డిఫాల్ట్ మోడ్** — మూడు మోడ్‌లు ప్రారంభించబడినప్పుడు, ముందుగా ఏ ట్యాబ్ తెరుచుకోవాలి. డొమైన్ నమోదును ఐచ్ఛికంగా ఉంచడానికి **సబ్‌డొమైన్** కు సెట్ చేయండి, లేదా కొనుగోళ్లను ప్రోత్సహించడానికి **కొత్త డొమైన్ నమోదు** కు సెట్ చేయండి.

**డొమైన్ ఉత్పత్తి** — ఐచ్ఛికంగా ఈ క్షేత్రాన్ని నిర్దిష్ట డొమైన్ ఉత్పత్తికి పిన్ చేయండి. సెట్ చేయకపోతే, వినియోగదారు వెతికే TLD ఆధారంగా addon సరిపోలే ఉత్పత్తిని స్వయంచాలకంగా ఎంచుకుంటుంది.

### నమోదుదారు సంప్రదింపు క్షేత్రాలు {#registrant-contact-fields}

వినియోగదారు **కొత్త డొమైన్ నమోదు** ట్యాబ్‌ను ఎంచుకున్నప్పుడు, Checkout ఫారమ్ నమోదుదారు సంప్రదింపు క్షేత్రాలను అదే చోట జోడిస్తుంది:

- మొదటి పేరు / చివరి పేరు
- ఇమెయిల్ చిరునామా
- చిరునామా (పంక్తి 1, నగరం, రాష్ట్రం/ప్రావిన్స్, పోస్టల్ కోడ్, దేశం)
- ఫోన్ నంబర్

ఇవి అన్ని రిజిస్ట్రార్‌లకు అవసరం. నమోదు API కాల్ చేయడానికి ముందు ఇవి ధృవీకరించబడతాయి. ఫోన్ నంబర్లు రిజిస్ట్రార్‌లు ఆశించే `+CC.NNN` అంతర్జాతీయ ఆకృతికి స్వయంచాలకంగా ఫార్మాట్ చేయబడతాయి.

### స్వయంచాలకంగా సృష్టించబడే సైట్ URL {#auto-generated-site-url}

వినియోగదారు డొమైన్‌ను నమోదు చేసినప్పుడు లేదా మ్యాప్ చేసినప్పుడు, సైట్ URL క్షేత్రం ఎంచుకున్న డొమైన్‌తో స్వయంచాలకంగా నింపబడుతుంది. వినియోగదారులు వేరే URL క్షేత్రాన్ని పూరించాల్సిన అవసరం లేదు.

### శోధన ప్రవర్తన {#search-behaviour}

- వినియోగదారు టైప్ చేస్తున్నప్పుడు AJAX తో డొమైన్ అందుబాటు నిజ సమయంలో తనిఖీ చేయబడుతుంది
- ఇష్టపడిన డొమైన్ అందుబాటులో లేకపోతే ప్రత్యామ్నాయ TLD సూచనలు చూపబడతాయి
- ధర ప్రత్యక్షంగా తెచ్చి స్పష్టంగా చూపబడుతుంది (నమోదు ధర, పునరుద్ధరణ ధర, ఐచ్ఛిక WHOIS గోప్యతా రుసుము)
- కూపన్ కోడ్‌లు ఇతర ఉత్పత్తుల మాదిరిగానే డొమైన్ ఉత్పత్తులకు వర్తిస్తాయి

**శోధన స్పందన వేగాన్ని సర్దుబాటు చేయడం:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**డొమైన్ శోధన ఫారమ్‌కు అనుకూల క్షేత్రాలను జోడించడం:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## వినియోగదారు DNS నిర్వహణ {#customer-dns-management}

వినియోగదారులు తమ నమోదు చేసిన డొమైన్‌ల DNS రికార్డులను **My Account** పేజీలో, సంబంధిత డొమైన్ ఎంట్రీ కింద నిర్వహించవచ్చు.

### మద్దతు ఉన్న రికార్డు రకాలు {#supported-record-types}

| రకం | ఉపయోగం |
|---|---|
| **A** | హోస్ట్‌నేమ్‌ను IPv4 చిరునామాకు మ్యాప్ చేయండి |
| **AAAA** | హోస్ట్‌నేమ్‌ను IPv6 చిరునామాకు మ్యాప్ చేయండి |
| **CNAME** | మరో హోస్ట్‌నేమ్‌ను సూచించే అలియాస్‌ను సృష్టించండి |
| **MX** | మెయిల్ ఎక్స్చేంజ్ సర్వర్‌ను సెట్ చేయండి |
| **TXT** | SPF, DMARC, ధృవీకరణ, లేదా ఇతర టెక్స్ట్ రికార్డులను జోడించండి |

### ఏ ప్రొవైడర్లు DNS నిర్వహణకు మద్దతు ఇస్తాయి? {#which-providers-support-dns-management}

DNS నిర్వహణ (రికార్డులను జోడించడం, సవరించడం, తొలగించడం) **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, మరియు **Openprovider** తో అందుబాటులో ఉంటుంది. **Hostinger** డొమైన్‌లు Domain Seller ద్వారా నేమ్‌సర్వర్‌లను నవీకరించవచ్చు; హోస్ట్ చేసిన డొమైన్‌ల DNS రికార్డులు ప్రధాన Hostinger డొమైన్-మ్యాపింగ్ ఇంటిగ్రేషన్ ద్వారా నిర్వహించబడతాయి. Namecheap, GoDaddy, మరియు NameSilo డొమైన్‌లు స్థితి మరియు గడువు ముగింపు సమాచారాన్ని చూపిస్తాయి, కానీ DNS ను రిజిస్ట్రార్ నియంత్రణ ప్యానెల్‌లో నేరుగా నిర్వహించాలి.

### డిఫాల్ట్ DNS రికార్డులు {#default-dns-records}

డొమైన్ నమోదు చేయబడినప్పుడు స్వయంచాలకంగా వర్తించే డిఫాల్ట్ DNS రికార్డులను మీరు కాన్ఫిగర్ చేయవచ్చు. **సెట్టింగ్‌లు › Domain Seller › డిఫాల్ట్ DNS రికార్డులు** కు వెళ్లండి.

డిఫాల్ట్ రికార్డు విలువలు రెండు టోకెన్‌లకు మద్దతు ఇస్తాయి:

| టోకెన్ | దీనితో భర్తీ చేయబడుతుంది |
|---|---|
| `{DOMAIN}` | నమోదు చేసిన డొమైన్ పేరు (ఉదా., `example.com`) |
| `{SITE_URL}` | వినియోగదారి సైట్ కోసం WordPress సైట్ URL |

**ఉదాహరణ — అపెక్స్ డొమైన్ మరియు www ను మీ సర్వర్ IP కు చూపించడం:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### అడ్మిన్: DNS ను చూడడం మరియు సవరించడం {#admin-viewing-and-editing-dns}

నెట్‌వర్క్ అడ్మిన్‌లు **నెట్‌వర్క్ అడ్మిన్ › Ultimate Multisite › డొమైన్‌లు** లోని డొమైన్ సవరణ పేజీ నుండి ఏ వినియోగదారు డొమైన్‌కైనా DNS రికార్డులను చూడవచ్చు మరియు సవరించవచ్చు.
