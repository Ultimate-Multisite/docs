# డొమైన్ ఉత్పత్తులు మరియు ధర నిర్ణయం

డొమైన్ ఉత్పత్తులు అంటే మీరు ధర నిర్ణయం, TLDలు, WHOIS ప్రైవసీ మరియు ప్రొవైడర్ ఎంపికను నియంత్రించే విధానం. ప్రతి డొమైన్ ఉత్పత్తి అనేది అదనపు **Domain Settings** ట్యాబ్‌తో కూడిన స్టాండర్డ్ Multisite Ultimate ఉత్పత్తి.

## డొమైన్ ఉత్పత్తిని సృష్టించడం

1. **Network Admin › Ultimate Multisite › Products**కి వెళ్లండి
2. **Add New** క్లిక్ చేయండి
3. ఉత్పత్తి రకాన్ని **Domain**గా సెట్ చేయండి
4. **Domain Settings** ట్యాబ్‌ను కాన్ఫిగర్ చేయండి (కింద చూడండి)
5. సేవ్ చేయండి

డొమైన్ ఉత్పత్తులు ఉత్పత్తి జాబితాలో ఊదా రంగు **Domain** బ్యాడ్జ్‌తో కనిపిస్తాయి మరియు **Domain Products** ట్యాబ్‌ని ఉపయోగించి ఫిల్టర్ చేయవచ్చు.

## డొమైన్ సెట్టింగ్‌లు ట్యాబ్

### ప్రొవైడర్ (Provider)

ఈ ఉత్పత్తి కోసం రిజిస్ట్రేషన్‌ను ఏ రిజిస్ట్రార్ నిర్వహిస్తుందో ఎంచుకోండి. ఇది డొమైన్ సెల్లర్ సెట్టింగ్‌లలో సెట్ చేయబడిన గ్లోబల్ **Default provider**కి డిఫాల్ట్‌గా ఉంటుంది.

### సపోర్టెడ్ TLDలు (Supported TLDs)

అన్ని TLDలకు వర్తించే **catch-all product**ని సృష్టించడానికి ఖాళీగా వదిలేయండి.

కేవలం ఆ ఎక్స్‌టెన్షన్‌లకు మాత్రమే వర్తించే **TLD-specific product**ని సృష్టించడానికి కామాతో వేరు చేయబడిన TLDల జాబితాను నమోదు చేయండి (ఉదాహరణకు, `.com, .net, .org`).

**ఉత్పత్తి మ్యాచింగ్ ఎలా పనిచేస్తుంది:** కస్టమర్ ఒక డొమైన్ కోసం శోధించినప్పుడు, అడాన్ అత్యంత నిర్దిష్టంగా సరిపోయే ఉత్పత్తిని ఎంచుకుంటుంది. దాని TLD జాబితాలో `.com` ఉన్న ఉత్పత్తి, catch-all ఉత్పత్తి కంటే ఎక్కువ ప్రాధాన్యతనిస్తుంది. ఏ TLD-specific ఉత్పత్తి సరిపోలకపోతే, catch-all ఉపయోగించబడుతుంది. ఏ ఉత్పత్తులు లేకపోతే, డొమైన్ శోధన చూపబడదు.

### మార్కప్ రకం (Markup type)

మీ రిటైల్ ధరను హోల్‌సేల్ ఖర్చు నుండి ఎలా లెక్కించాలో మూడు మోడ్‌లు నియంత్రిస్తాయి:

| మోడ్ | ఇది ఎలా పనిచేస్తుంది |
|---|---|
| **Percentage** | హోల్‌సేల్ ఖర్చుపై శాతం జోడిస్తుంది. $10 హోల్‌సేల్ డొమైన్‌పై 20% మార్కప్ అంటే $12 అవుతుంది. |
| **Fixed markup** | స్థిరమైన డాలర్ మొత్తాన్ని జోడిస్తుంది. $10 డొమైన్‌పై $5 మార్కప్ అంటే $15 అవుతుంది. |
| **Fixed price** | హోల్‌సేల్ ఖర్చును పూర్తిగా విస్మరిస్తుంది. మీరు నమోదు చేసిన మొత్తాన్ని ఎల్లప్పుడూ వసూలు చేస్తుంది. |

### పరిచయ ధర (Introductory pricing)

తగ్గించిన మొదటి సంవత్సరం ధరను అందించడానికి ఎనేబుల్ చేయండి. సాధారణ **Renewal price** (సంవత్సరం 2+) పక్కన వేరే **Introductory price** (సంవత్సరం 1 ధర) సెట్ చేయండి. కస్టమర్ చెక్అవుట్ సమయంలో రెండు ధరలు ప్రదర్శించబడతాయి, తద్వారా పునరుద్ధరణ సమయంలో ఏమి ఆశించాలో వారికి తెలుస్తుంది.

### WHOIS ప్రైవసీ (WHOIS privacy)

ఈ ఉత్పత్తి ద్వారా రిజిస్టర్ చేయబడిన డొమైన్‌ల కోసం WHOIS ప్రైవసీ రక్షణ అందించబడుతుందో లేదో నియంత్రిస్తుంది.

| సెట్టింగ్ | ప్రవర్తన |
|---|---|
| **Disabled** | WHOIS ప్రైవసీ ఎప్పుడూ అందించబడదు లేదా ఎనేబుల్ చేయబడదు. |
| **Always Included** | WHOIS ప్రైవసీ రిజిస్ట్రేషన్‌లో ఎటువంటి ఛార్జీ లేకుండా ఆటోమేటిక్‌గా ఎనేబుల్ చేయబడుతుంది. |
| **Customer Choice** | చెక్అవుట్ సమయంలో ఒక చెక్‌బాక్స్ కనిపిస్తుంది. ప్రతి సంవత్సరం వసూలు చేయడానికి **Privacy price**ను సెట్ చేయండి, లేదా ఉచితంగా అందించడానికి దానిని $0 వద్ద వదిలేయండి. |

Namecheap కోసం, WHOIS ప్రైవసీ WhoisGuardని ఉపయోగిస్తుంది (ఎల్లప్పుడూ ఉచితం). OpenSRS కోసం, ఇది OpenSRS ప్రైవసీ సేవను ఉపయోగిస్తుంది (హోల్‌సేల్‌లో ఖర్చు ఉండవచ్చు).

## TLD దిగుమతి మరియు సింక్ (TLD import and sync)

డొమైన్ ఉత్పత్తులు కనెక్ట్ చేయబడిన ప్రొవైడర్ నుండి తీసుకురాబడిన రియల్-టైమ్ హోల్‌సేల్ ధరలను ప్రదర్శిస్తాయి. ఇది పని చేయడానికి, TLDలను దిగుమతి చేసుకోవాలి.

- **మాన్యువల్ సింక్:** Settings › Domain Seller › Sync TLDs (ప్రొవైడర్ ప్రకారం)
- **ఆటోమేటిక్ సింక్:** కాన్ఫిగర్ చేయబడిన అన్ని ప్రొవైడర్‌లలో షెడ్యూల్ చేయబడిన క్రోన్ జాబ్ ద్వారా రోజువారీగా నడుస్తుంది

సింక్ అయిన తర్వాత, ఏదైనా డొమైన్ ఉత్పత్తి యొక్క Domain Settings ట్యాబ్‌కి వెళ్లి, అందుబాటులో ఉన్న TLDలను వాటి ప్రస్తుత హోల్‌సేల్ ధరలతో చూడటానికి TLD పికర్‌ను ఉపయోగించండి.

## ఆటో-రిన్యూవల్ (Auto-renewal)

డొమైన్ పునరుద్ధరణ కస్టమర్ యొక్క membership స్థితికి ముడిపడి ఉంటుంది:

- membership పునరుద్ధరించబడి మరియు డొమైన్ లింక్ చేయబడితే, డొమైన్ పునరుద్ధరణ ఆటోమేటిక్‌గా క్యూలో చేరుతుంది
- పునరుద్ధరణ ప్రయత్నాలు కస్టమర్ యొక్క యాక్టివ్ పేమెంట్ గేట్‌వేను ఉపయోగిస్తాయి
- విఫలమైన పునరుద్ధరణలు ఎక్స్‌పోనెన్షియల్ బ్యాక్‌ఆఫ్‌తో ఆటోమేటిక్‌గా మళ్లీ ప్రయత్నిస్తాయి
- విజయవంతమైన పునరుద్ధరణలు, వైఫల్యాలు మరియు రాబోయే గడువు ముగియడం కోసం ఈమెయిల్ నోటిఫికేషన్‌లు పంపబడతాయి

డొమైన్ జీవితచక్రా సంఘటనల కోసం ఈమెయిల్ టెంప్లేట్ IDలు:

| సంఘటన | టెంప్లేట్ ID |
|---|---|
| డొమైన్ రిజిస్టర్ చేయబడింది | `domain_registered` |
| డొమైన్ పునరుద్ధరించబడింది | `domain_renewed` |
| పునరుద్ధరణ విఫలమైంది | `domain_renewal_failed` |
| డొమైన్ త్వరలో గడువు ముగుస్తుంది | `domain_expiring_soon` |

## అడ్మిన్: మాన్యువల్ డొమైన్ రిజిస్ట్రేషన్

కస్టమర్ చెక్అవుట్ ద్వారా వెళ్లకుండా వారి తరపున డొమైన్‌ను రిజిస్టర్ చేయడానికి:

1. **Network Admin › Ultimate Multisite › Register Domain**కి వెళ్లండి
2. కస్టమర్‌ను ఎంచుకుని, డొమైన్ పేరును నమోదు చేయండి
3. రిజిస్ట్రెంట్ కాంటాక్ట్ వివరాలను పూరించండి (పేరు, చిరునామా, ఫోన్)
4. **Register** క్లిక్ చేయండి

డొమైన్ రికార్డ్ సృష్టించబడి కస్టమర్ ఖాతాకు లింక్ చేయబడుతుంది.
