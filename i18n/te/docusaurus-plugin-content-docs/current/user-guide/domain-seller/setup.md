# డొమైన్ విక్రేత: సెటప్ మరియు ప్రొవైడర్ కాన్ఫిగరేషన్

డొమైన్ విక్రేత యాడాన్‌లో మార్గదర్శక సెటప్ విజార్డ్ ఉంటుంది. అది అవసరమైన ప్రతి దశలో మీకు దారి చూపుతుంది. ఈ పేజీ విజార్డ్ ప్రవాహాన్ని, తరువాత ప్రొవైడర్లను ఎలా కాన్ఫిగర్ చేయాలి లేదా మళ్లీ కాన్ఫిగర్ చేయాలి అనే విషయాలను వివరిస్తుంది.

## అవసరాలు

- **Multisite Ultimate** v2.4.12 లేదా అంతకంటే ఎక్కువ, నెట్‌వర్క్‌లో యాక్టివేట్ చేయబడినది
- **PHP** 7.4+
- మద్దతు ఉన్న కనీసం ఒక రిజిస్ట్రార్‌కు API ఆధారాలు

## మొదటి అమలు సెటప్ విజార్డ్

మీరు pluginను మొదటిసారి నెట్‌వర్క్‌లో యాక్టివేట్ చేసినప్పుడు సెటప్ విజార్డ్ స్వయంగా ప్రారంభమవుతుంది. ఇది ఎప్పుడైనా **నెట్‌వర్క్ అడ్మిన్ › Ultimate Multisite › డొమైన్ విక్రేత సెటప్** నుండి కూడా అందుబాటులో ఉంటుంది.

### దశ 1 — ప్రొవైడర్‌ను ఎంచుకోండి

మీరు కనెక్ట్ చేయాలనుకునే రిజిస్ట్రార్‌ను ఎంచుకోండి. మద్దతు ఉన్న ఎంపికలు:

| ప్రొవైడర్ | DNS నిర్వహణ | WHOIS గోప్యత |
|---|---|---|
| OpenSRS | అవును | అవును |
| Namecheap | లేదు | అవును (WhoisGuard, ఉచితం) |
| HostAfrica | అవును | అవును (ID రక్షణ) |
| Openprovider | అవును | అవును |
| Hostinger | హోస్ట్ చేసిన డొమైన్‌ల కోసం ప్రధాన Hostinger డొమైన్ మ్యాపింగ్ ద్వారా | అవును |
| GoDaddy | లేదు | లేదు |
| ResellerClub | అవును | లేదు |
| NameSilo | లేదు | లేదు |
| Enom | అవును | లేదు |

### దశ 2 — ఆధారాలను నమోదు చేయండి

ప్రతి ప్రొవైడర్‌కు వేర్వేరు ఆధార ఫీల్డ్‌లు ఉంటాయి:

**OpenSRS** — వినియోగదారు పేరు మరియు ప్రైవేట్ కీ (OpenSRS రీసెల్లర్ కంట్రోల్ ప్యానెల్ నుండి)

**Namecheap** — వినియోగదారు పేరు మరియు API కీ (Account › Tools › API Access నుండి)

**HostAfrica** — HostAfrica రీసెల్లర్ మాడ్యూల్ నుండి డొమైన్‌ల రీసెల్లర్ API ముగింపు బిందువు మరియు ఆధారాలు. ప్రత్యేక సాండ్‌బాక్స్ ముగింపు బిందువు ప్రస్తుతం డాక్యుమెంట్ చేయబడలేదు; ప్రత్యక్ష రిజిస్ట్రేషన్‌లు నడపడానికి ముందు సురక్షితమైన చదవడానికి మాత్రమే పరిమితమైన తనిఖీలతో పరీక్షించండి.

**Openprovider** — API యాక్సెస్ ప్రారంభించిన వినియోగదారు పేరు మరియు పాస్‌వర్డ్. ఐచ్ఛిక సాండ్‌బాక్స్ మోడ్ Openprovider సాండ్‌బాక్స్ APIను ఉపయోగిస్తుంది. ఐచ్ఛిక డిఫాల్ట్ కస్టమర్ హ్యాండిల్‌ను రిజిస్ట్రేషన్‌లకు మళ్లీ ఉపయోగించవచ్చు.

**Hostinger** — ప్రధాన Hostinger ఇంటిగ్రేషన్ నుండి భాగస్వామ్య Hostinger hPanel API టోకెన్. అదే టోకెన్ ప్రధాన డొమైన్ మ్యాపింగ్ మరియు డొమైన్ విక్రేత రిజిస్ట్రేషన్ కార్యకలాపాలను నడుపుతుంది.

**GoDaddy** — API కీ మరియు సీక్రెట్ (developer.godaddy.com నుండి)

**ResellerClub** — రీసెల్లర్ ID మరియు API కీ (ResellerClub కంట్రోల్ ప్యానెల్ నుండి)

**NameSilo** — API కీ (namesilo.com › Account › API Manager నుండి)

**Enom** — Account ID మరియు API టోకెన్

ప్రత్యక్షంగా ప్రారంభించే ముందు ప్రొవైడర్ పరీక్ష వాతావరణంలో పరీక్షించడానికి, అందుబాటులో ఉన్న చోట **సాండ్‌బాక్స్ మోడ్**ను ఎంచుకోండి.

### దశ 3 — కనెక్షన్‌ను పరీక్షించండి

**కనెక్షన్‌ను పరీక్షించండి** క్లిక్ చేయండి. ఆధారాలు మరియు కనెక్టివిటీని ధృవీకరించడానికి విజార్డ్ తేలికైన API కాల్‌ను పంపుతుంది. కొనసాగించే ముందు ఆధారాలకు సంబంధించిన ఏవైనా సమస్యలను సరిచేయండి.

### దశ 4 — TLDలను దిగుమతి చేయండి

కనెక్ట్ చేసిన ప్రొవైడర్ నుండి అందుబాటులో ఉన్న అన్ని TLDలు మరియు హోల్‌సేల్ ధరలను పొందడానికి **TLDలను దిగుమతి చేయండి** క్లిక్ చేయండి. ఇది డొమైన్ ఉత్పత్తులు ఉపయోగించే TLD జాబితాను నింపుతుంది. పెద్ద TLD క్యాటలాగ్‌లు ఉన్న ప్రొవైడర్లకు దిగుమతి 30–60 సెకన్లు పట్టవచ్చు.

షెడ్యూల్ చేసిన క్రాన్ జాబ్ ద్వారా TLDలు రోజుకు ఒకసారి స్వయంగా మళ్లీ సింక్ అవుతాయి.

### దశ 5 — డొమైన్ ఉత్పత్తిని సృష్టించండి

విజార్డ్ 10% మార్కప్‌తో డిఫాల్ట్ క్యాచ్-ఆల్ డొమైన్ ఉత్పత్తిని సృష్టిస్తుంది. మీరు ఈ ఉత్పత్తిని వెంటనే సవరించవచ్చు లేదా దాటవేసి **Ultimate Multisite › ఉత్పత్తులు** కింద ఉత్పత్తులను మాన్యువల్‌గా సృష్టించవచ్చు.

పూర్తి ఉత్పత్తి కాన్ఫిగరేషన్ మార్గదర్శిని కోసం [డొమైన్ ఉత్పత్తులు మరియు ధరలు](./domain-products) చూడండి.

## ప్రొవైడర్‌ను మళ్లీ కాన్ఫిగర్ చేయడం

**నెట్‌వర్క్ అడ్మిన్ › Ultimate Multisite › సెట్టింగ్‌లు › డొమైన్ విక్రేత**కు వెళ్లండి (లేదా plugin జాబితాలో **సెట్టింగ్‌లు** క్లిక్ చేయండి).

సెట్టింగ్‌ల పేజీలో ఇవి ఉంటాయి:

- **డొమైన్ విక్రయాన్ని ప్రారంభించండి** — మొత్తం ఫీచర్‌ను ఆన్/ఆఫ్ చేయండి
- **డిఫాల్ట్ ప్రొవైడర్** — డొమైన్ శోధనలు మరియు కొత్త ఉత్పత్తుల కోసం ఉపయోగించే ప్రొవైడర్
- **ఒక్కో శోధనకు గరిష్ఠ TLDలు** — కస్టమర్ శోధించినప్పుడు ఎన్ని TLDలను తనిఖీ చేయాలో; ఎక్కువ విలువలు మరిన్ని ఎంపికలను చూపుతాయి కానీ నెమ్మదిగా ఉంటాయి
- **లభ్యత క్యాష్ వ్యవధి** — లభ్యత మరియు ధర ఫలితాలను ఎంతసేపు క్యాష్ చేయాలో; తక్కువ విలువలు మరింత ఖచ్చితంగా ఉంటాయి కానీ API కాల్‌లను పెంచుతాయి
- **డొమైన్ ఉత్పత్తులను నిర్వహించండి** — ఉత్పత్తుల జాబితాకు త్వరిత లింక్
- **ప్రొవైడర్లను కాన్ఫిగర్ చేయండి** — ప్రొవైడర్లను జోడించడానికి లేదా మళ్లీ కాన్ఫిగర్ చేయడానికి ఇంటిగ్రేషన్ విజార్డ్‌ను తెరుస్తుంది

### రెండవ ప్రొవైడర్‌ను జోడించడం

**ప్రొవైడర్లను కాన్ఫిగర్ చేయండి** క్లిక్ చేసి, కొత్త రిజిస్ట్రార్ కోసం విజార్డ్‌ను మళ్లీ నడపండి. మీరు ఒకేసారి అనేక ప్రొవైడర్లను కాన్ఫిగర్ చేసి ఉంచవచ్చు. ప్రతి డొమైన్ ఉత్పత్తిని నిర్దిష్ట ప్రొవైడర్‌కు కేటాయించండి లేదా డిఫాల్ట్‌లోనే ఉంచండి.

### TLDలను మాన్యువల్‌గా సింక్ చేయడం

సెట్టింగ్‌ల పేజీలో, తాజా ధరలను పొందడానికి కాన్ఫిగర్ చేసిన ఏ ప్రొవైడర్ పక్కనైనా **TLDలను సింక్ చేయండి** క్లిక్ చేయండి. ప్రొవైడర్ హోల్‌సేల్ ధరలను నవీకరించిన తర్వాత లేదా కొత్త TLDలను జోడించిన తర్వాత ఇది ఉపయోగకరంగా ఉంటుంది.

## ప్రొవైడర్ సామర్థ్య గమనికలు

ప్రతి రిజిస్ట్రార్ API ఒకే కార్యకలాపాలను అందించదు. మద్దతు లేని కార్యకలాపాల విషయంలో యాడాన్ నిశ్శబ్దంగా విఫలమవకుండా, అడ్మిన్‌కు స్పష్టంగా కనిపించే లోపాలను చూపిస్తుంది.

- **HostAfrica** లుకప్, TLD/ధరల సమకాలీకరణ, రిజిస్ట్రేషన్, రిన్యూవల్, ట్రాన్స్‌ఫర్, nameserver నవీకరణలు, DNS రికార్డులు, EPP కోడ్‌లు, registrar lock, ID రక్షణతో సహా అత్యంత విస్తృతమైన ప్రత్యక్ష రీసెల్లర్ వర్క్‌ఫ్లోకు మద్దతు ఇస్తుంది.
- **Openprovider** రీసెల్లర్ ధరల TLD సమకాలీకరణ, రిజిస్ట్రేషన్, రిన్యూవల్, ట్రాన్స్‌ఫర్‌లు, nameserver నవీకరణలు, DNS జోన్‌లు, EPP కోడ్‌లు, registrar lock, WHOIS గోప్యతకు మద్దతు ఇస్తుంది. ఇది స్వల్పకాలిక bearer token తో ధృవీకరిస్తుంది; addon దాన్ని స్వయంచాలకంగా రిఫ్రెష్ చేస్తుంది.
- **Hostinger** భాగస్వామ్య hPanel API token ద్వారా లభ్యత శోధన, రిజిస్ట్రేషన్, పోర్ట్‌ఫోలియో లుకప్, nameserver నవీకరణలు, registrar lock, WHOIS గోప్యతకు మద్దతు ఇస్తుంది. Hostinger యొక్క public Domains API రీసెల్లర్/హోల్‌సేల్ ధరలు, ఇన్‌బౌండ్ ట్రాన్స్‌ఫర్, స్పష్టమైన రిన్యూవల్, లేదా EPP-code పొందడాన్ని అందించదు; రిన్యూవల్స్ ఆటో-రిన్యూ మాత్రమే.
