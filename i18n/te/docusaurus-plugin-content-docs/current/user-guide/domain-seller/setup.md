# Domain Seller: సెటప్ మరియు ప్రొవైడర్ కాన్ఫిగరేషన్

Domain Seller యాడ్‌ఆన్ అనేది ఒక సులభమైన సెటప్ విజర్డ్‌తో వస్తుంది. ఈ విజర్డ్ ద్వారా అవసరమైన ప్రతి స్టెప్‌లో మిమ్మల్ని నడిపిస్తుంది. ఈ పేజీలో విజర్డ్ ఎలా పనిచేస్తుందో, ఆ తర్వాత ప్రొవైడర్లను ఎలా కాన్ఫిగర్ చేయాలి లేదా మళ్లీ కాన్ఫిగర్ చేయాలి అనే వివరాలు ఉన్నాయి.

## అవసరాలు (Requirements)

- **Multisite Ultimate** v2.4.12 లేదా అంతకంటే ఎక్కువ, నెట్‌వర్క్-యాక్టివేట్ చేయబడి ఉండాలి
- **PHP** 7.4+
- కనీసం ఒక సపోర్టెడ్ రిజిస్ట్రార్ కోసం API క్రెడెన్షియల్స్

## మొదటిసారి సెటప్ విజర్డ్ (First-run setup wizard)

ఈ సెటప్ విజర్డ్ మీరు మొదటిసారి ప్లగిన్‌ను నెట్‌వర్క్-యాక్టివేట్ చేసినప్పుడు ఆటోమేటిక్‌గా మొదలవుతుంది. ఇది ఎప్పుడైనా **Network Admin › Ultimate Multisite › Domain Seller Setup** నుండి కూడా అందుబాటులో ఉంటుంది.

### స్టెప్ 1 — ప్రొవైడర్‌ను ఎంచుకోండి (Choose a provider)

మీరు కనెక్ట్ చేయాలనుకుంటున్న రిజిస్ట్రార్‌ను ఎంచుకోండి. సపోర్టెడ్ ఆప్షన్లు:

| Provider | DNS management | WHOIS privacy |
|---|---|---|
| OpenSRS | Yes | Yes |
| Namecheap | No | Yes (WhoisGuard, free) |
| GoDaddy | No | No |
| ResellerClub | Yes | No |
| NameSilo | No | No |
| Enom | Yes | No |

### స్టెప్ 2 — క్రెడెన్షియల్స్ నమోదు చేయండి (Enter credentials)

ప్రతి ప్రొవైడర్ యొక్క క్రెడెన్షియల్ ఫీల్డ్‌లు వేరుగా ఉంటాయి:

**OpenSRS** — యూజర్‌నేమ్ మరియు ప్రైవేట్ కీ (OpenSRS Reseller Control Panel నుండి)

**Namecheap** — యూజర్‌నేమ్ మరియు API కీ (Account › Tools › API Access నుండి)

**GoDaddy** — API కీ మరియు సీక్రెట్ (developer.godaddy.com నుండి)

**ResellerClub** — రిసెల్లర్ ID మరియు API కీ (ResellerClub control panel నుండి)

**NameSilo** — API కీ (namesilo.com › Account › API Manager నుండి)

**Enom** — అకౌంట్ ID మరియు API టోకెన్

లైవ్ (live)గా వెళ్లడానికి ముందు, అందుబాటులో ఉన్న చోట **Sandbox mode**ని చెక్ చేయండి. దీనివల్ల ప్రొవైడర్ యొక్క టెస్ట్ ఎన్విరాన్‌మెంట్‌లో టెస్ట్ చేయవచ్చు.

### స్టెప్ 3 — కనెక్షన్‌ను పరీక్షించండి (Test the connection)

**Test Connection** పై క్లిక్ చేయండి. క్రెడెన్షియల్స్ మరియు కనెక్టివిటీని ధృవీకరించడానికి విజర్డ్ ఒక చిన్న API కాల్ పంపుతుంది. కొనసాగించే ముందు ఏవైనా క్రెడెన్షియల్ సమస్యలు ఉంటే సరిచేయండి.

### స్టెప్ 4 — TLDలను ఇంపోర్ట్ చేయండి (Import TLDs)

కనెక్ట్ అయిన ప్రొవైడర్ నుండి అందుబాటులో ఉన్న అన్ని TLDలు మరియు హోల్‌సేల్ ధరలను తీసుకోవడానికి **Import TLDs** పై క్లిక్ చేయండి. ఇది డొమైన్ ఉత్పత్తులు ఉపయోగించే TLD జాబితాను నింపుతుంది. పెద్ద TLD కేటలాగ్ ఉన్న ప్రొవైడర్ల కోసం ఈ ఇంపోర్ట్ ప్రక్రియకు 30–60 సెకన్లు పట్టవచ్చు.

TLDలు రోజుకు ఒకసారి షెడ్యూల్ చేయబడిన క్రోన్ జాబ్ ద్వారా ఆటోమేటిక్‌గా రీ-సింక్ అవుతాయి.

### స్టెప్ 5 — డొమైన్ ఉత్పత్తిని సృష్టించండి (Create a domain product)

విజర్డ్ 10% మార్కప్‌తో డిఫాల్ట్ క్యాట్చ్-ఆల్ డొమైన్ ఉత్పత్తిని సృష్టిస్తుంది. మీరు ఈ ఉత్పత్తిని వెంటనే ఎడిట్ చేయవచ్చు లేదా స్కిప్ చేసి, **Ultimate Multisite › Products** కింద మాన్యువల్‌గా ఉత్పత్తులను సృష్టించవచ్చు.

పూర్తి ఉత్పత్తి కాన్ఫిగరేషన్ గైడ్ కోసం [Domain Products and Pricing](./domain-products) చూడండి.

## ప్రొవైడర్‌ను మళ్లీ కాన్ఫిగర్ చేయడం (Reconfiguring a provider)

**Network Admin › Ultimate Multisite › Settings › Domain Seller** కి వెళ్లండి (లేదా ప్లగిన్ జాబితాలో **Settings** పై క్లిక్ చేయండి).

సెట్టింగ్స్ పేజీలో ఈ వివరాలు ఉంటాయి:

- **Enable domain selling** — మొత్తం ఫీచర్‌ను ఆన్/ఆఫ్ చేయడానికి టోగుల్ చేయండి
- **Default provider** — డొమైన్ సెర్చ్‌లు మరియు కొత్త ఉత్పత్తుల కోసం ఉపయోగించే ప్రొవైడర్
- **Max TLDs per search** — కస్టమర్ సెర్చ్ చేసినప్పుడు ఎన్ని TLDలను చెక్ చేయాలో; ఎక్కువ విలువలు ఎక్కువ ఆప్షన్లను చూపుతాయి కానీ నెమ్మదిగా ఉంటాయి
- **Availability cache duration** — లభ్యత (availability) మరియు ధరల ఫలితాలను ఎంతకాలం క్యాష్ చేయాలి; తక్కువ విలువలు మరింత ఖచ్చితమైనవిగా ఉంటాయి, కానీ API కాల్స్ పెరుగుతాయి
- **Manage domain products** — ఉత్పత్తుల జాబితాకు త్వరిత లింక్
- **Configure providers** — ప్రొవైడర్లను జోడించడానికి లేదా మళ్లీ కాన్ఫిగర్ చేయడానికి ఇంటిగ్రేషన్ విజర్డ్‌ను తెరుస్తుంది

### రెండవ ప్రొవైడర్‌ను జోడించడం (Adding a second provider)

**Configure providers** పై క్లిక్ చేసి, కొత్త రిజిస్ట్రార్ కోసం విజర్డ్‌ను మళ్లీ రన్ చేయండి. మీరు ఏకకాలంలో బహుళ ప్రొవైడర్లను కాన్ఫిగర్ చేయవచ్చు. ప్రతి డొమైన్ ఉత్పత్తిని ఒక నిర్దిష్ట ప్రొవైడర్‌కు కేటాయించండి, లేదా డిఫాల్ట్‌లోనే ఉంచండి.

### TLDలను మాన్యువల్‌గా సింక్ చేయడం (Syncing TLDs manually)

సెట్టింగ్స్ పేజీలో, ఏదైనా కాన్ఫిగర్ చేయబడిన ప్రొవైడర్ పక్కన ఉన్న **Sync TLDs** పై క్లిక్ చేయడం ద్వారా తాజా ధరలను తీసుకోవచ్చు. ఒక ప్రొవైడర్ హోల్‌సేల్ ధరలను అప్‌డేట్ చేసినా లేదా కొత్త TLDలను జోడించినా ఇది ఉపయోగపడుతుంది.

## లాగ్‌లు (Logs)

ప్రతి ప్రొవైడర్ దాని స్వంత లాగ్ ఛానెల్‌లో వ్రాస్తుంది. లాగ్‌లను **Network Admin › Ultimate Multisite › Logs** కింద చూడవచ్చు:

| Log channel | Contents |
|---|---|
| `domain-seller-registration` | అన్ని రిజిస్ట్రేషన్ ప్రయత్నాలు (విజయం మరియు వైఫల్యం) |
| `domain-seller-renewal` | రిన్యూవల్ జాబ్ ఫలితాలు |
| `domain-seller-opensrs` | ముడి OpenSRS API యాక్టివిటీ |
| `domain-seller-namecheap` | ముడి Namecheap API యాక్టివిటీ |
| `domain-seller-godaddy` | ముడి GoDaddy API యాక్టివిటీ |
| `domain-seller-resellerclub` | ముడి ResellerClub API యాక్టివిటీ |
| `domain-seller-namesilo` | ముడి NameSilo API యాక్టివిటీ |
| `domain-seller-enom` | ముడి Enom API యాక్టివిటీ |
