# WooCommerce İnteqrasiyası {#woocommerce-integration}

WooCommerce İnteqrasiyası bizim Manual Gateway işləmə üsuluna çox bənzər şəkildə işləyir. O, istifadəçilərinizə böyük çeviklik təqdim edir və onlara WooCommerce quraşdırmanızda mövcud olan istənilən ödəniş formasını seçməyə imkan verir. Və bu sadədir: istifadəçinin hesablaşma dövrü bitdikdən sonra istifadəçi abunəliyi gözləməyə alınır. Eyni zamanda, şəbəkənin əsas saytının WooCommerce quraşdırmasında bir sifariş yaradılacaq. Sonra istifadəçi e-poçtunda ödənişi həyata keçirmək üçün link olan bir faktura alır. Ödəniş qəbul edildikdən sonra abunəlik yenilənir.

## Xüsusiyyətlər {#features}

- Ultimate Multisite və WooCommerce arasında asan inteqrasiya
- Bir neçə ödəniş üsulu mövcuddur
- Daxil edilmiş abunəliklər nüvəsindən istifadə edərək avtomatlaşdırılmış təkrarlanan hesablaşma, istəyə bağlı WooCommerce Subscriptions plugin təkmilləşdirmələri ilə
- Checkout cəmləri Ultimate Multisite vergilərini ayrıca, vergiyə cəlb olunmayan WooCommerce rüsum sətirləri kimi daxil edir ki, müştərilər ödənişdən əvvəl vergi məbləğlərini nəzərdən keçirə bilsinlər

## Sənədləşdirmə {#documentation}

**WooCommerce İnteqrasiyasının Qurulması**

**Ultimate Multisite ilə Ödəniş Seçimlərinizi Genişləndirmək: WooCommerce İnteqrasiyası**

## Quraşdırma {#installation}

1. 'ultimate-multisite-woocommerce' faylını '/wp-content/plugins/' qovluğuna yükləyin
2. Plugin-i WordPress-də 'Plugins' menyusu vasitəsilə aktivləşdirin
3. Plugin-i qurmaq üçün addım-addım Wizard-ı izləyin

## İstifadə Bələdçisi {#usage-guide}

_**VACİB QEYD: Bu məqalə Ultimate Multisite 2.x versiyasına aiddir.**_

_**DİQQƏT:** Ultimate Multisite: WooCommerce İnteqrasiyası WooCommerce-in ən azı əsas saytınızda aktivləşdirilməsini tələb edir._

Biz anlayırıq ki, _Stripe_ və _PayPal_ bəzi ölkələrdə mövcud deyil və bu, Ultimate Multisite istifadəçilərinin plugin-imizdən effektiv istifadə etməsini məhdudlaşdırır və ya çətinləşdirir. Buna görə də _WooCommerce_ inteqrasiyası üçün **add-on yaratdıq**.

_WooCommerce_ çox populyar eCommerce plugin-dir. Dünyanın müxtəlif yerlərindəki tərtibatçılar müxtəlif ödəniş gateway-lərini ona inteqrasiya etmək üçün add-on-lar yaratdılar. Biz Ultimate Multisite hesablaşma sistemində istifadə edə biləcəyiniz ödəniş gateway-lərini genişləndirmək üçün bundan yararlandıq.

## Onun Qurulması {#setting-it-up}

Əvvəlcə **lisenziyalı istifadəçilər üçün pulsuz** olan WooCommerce Integration add-on-unu quraşdırmalısınız. Onu quraşdırmaq üçün super admin dashboard-unuzda Ultimate Multisite üzərinə klikləyin və **Add-ons** seçin.

![Add-ons səhifəsi](/img/admin/addons-page.png)

Burada bütün Ultimate Multisite add-on-larını tapa bilərsiniz. **Ultimate Multisite: WooCommerce İnteqrasiyası** add-on-una klikləyin.

![WooCommerce İnteqrasiyasını göstərən Add-ons səhifəsi](/img/admin/addons-page.png)

Add-on təfərrüatları ilə bir pəncərə açılacaq. Sadəcə **Install Now** üzərinə klikləyin *.*

<!-- Screenshot unavailable: WooCommerce Integration addon install popup with Install Now button -->

Quraşdırma tamamlandıqdan sonra plugins səhifəsinə yönləndiriləcəksiniz. Burada sadəcə **Network Activate** üzərinə klikləyin və _WooCommerce_ add-on-u şəbəkənizdə aktivləşdiriləcək.

<!-- Screenshot unavailable: WP Plugins page showing Network Activate button for WooCommerce Integration -->

Onu aktivləşdirdikdən sonra, əgər hələ də vebsaytınızda WooCommerce plugin quraşdırılmayıb və aktivləşdirilməyibsə, **siz xatırlatma alacaqsınız**.

<!-- Screenshot unavailable: Admin notice reminding to install and activate WooCommerce plugin -->

Quraşdırıldıqdan sonra Ultimate Multisite ödəniş ayarları altında əlavə bir seçim görəcəksiniz. Sonra oradan _WooCommerce_ seçimini **aktivləşdirə** bilərsiniz.

![WooCommerce aktivləşdirilmiş Ultimate Multisite ödəniş gateway ayarları](/img/addons/woocommerce-payment-gateway.png)

Front end-də görünəcək ekran adını və istifadəçilərinizi checkout axınında istiqamətləndirmək üçün təlimatları kimi bəzi əsas ayarları konfiqurasiya edə bilərsiniz. Sonra istifadə etməyi üstün tutduğunuz ödəniş gateway-ini aktivləşdirmək və konfiqurasiya etmək üçün **Əsas saytınızın _WooCommerce_ ayarlarına** keçə bilərsiniz.

### Checkout vergi rüsumu sətirləri {#checkout-tax-fee-lines}

Ultimate Multisite checkout-a vergilər daxil olduqda, inteqrasiya hər Ultimate Multisite vergi məbləğini WooCommerce checkout-a ayrıca, vergiyə cəlb olunmayan rüsum sətri kimi əlavə edir. Bu, WooCommerce-in vergi rüsumunu yenidən vergiyə cəlb etməsinin qarşısını alarkən, vergi cəminin ödənişdən əvvəl görünən qalmasını təmin edir.

## Ödənişi necə idarə etmək olar {#how-to-manage-the-payment}

WooCommerce İnteqrasiyası bizim **Manual Gateway** işləmə üsuluna çox bənzər şəkildə işləyir. Mənfi tərəfi odur ki, hər ödəniş hesablaşma dövrünün sonunda əl ilə ödənilməlidir. Axın belə işləyir:

  * İstifadəçinin hesablaşma dövrü bitdikdən sonra membership **gözləməyə alınır**. Eyni zamanda, şəbəkənizin əsas saytındakı _WooCommerce_ quraşdırmasında bir sifariş yaradılacaq.

  * Sonra istifadəçilər e-poçtlarında ödənişi həyata keçirmək üçün link olan bir faktura alırlar (link həmçinin onların subsite dashboard-undakı hesablaşma tarixçəsində də görünür). Həmin link istifadəçini _**WooCommerce**_ **checkout** səhifəsinə yönləndirir; burada istifadəçilər ödənişlərini emal etmək üçün **mövcud _WooCommerce_ gateway-lərindən** birini seçə biləcəklər.

  * Ödəniş qəbul edildikdən sonra membership **yenilənir** , ödəniş Ultimate Multisite-da qeydə alınır və sifariş tamamlanmış kimi təyin edilir.

### Bəzi Vacib Qeydlər: {#some-important-notes}

  * Ultimate Multisite-ın standart funksionallığı bu inteqrasiya forması ilə hələ də işləyir. Məsələn, **Subscription Management** ekranından **geri ödənişlərin verilməsi** seçilmiş ödəniş formasından istifadə edərək geri ödənişi vermək üçün WooCommerce ilə əlaqə yaradacaq.

  * Ultimate Multisite: WooCommerce İnteqrasiyası işləmək üçün WP-Cron-a əsaslanır. Bu, çox vaxt çox yaxşı işləyir, lakin saytın trafiki azdırsa, cron planlaşdırılmış tədbirləri yoxlamaq üçün işə salınmır və tədbir qaçırılır; bu da icra gecikmələrinə səbəb ola bilər.

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

Hər ay ödənişi əl ilə başlatmaq son istifadəçilər üçün ideal deyil, çünki subscription-ların hər billing dövründə avtomatik təkrarlanması gözlənilir. WooCommerce Integration add-on-u indi əsas subscriptions kodunu özündə birləşdirir, buna görə recurring məhsullar recurring billing üçün həmişə subscriptions-dan istifadə edir.

Bu inteqrasiya vasitəsilə recurring məhsullar satmaq üçün ayrıca [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) almağa və ya quraşdırmağa ehtiyacınız **yoxdur**. Əgər həmin plugin alınmış, quraşdırılmış və əsas saytınızda əlçatandırsa, Ultimate Multisite ondan istifadə edəcək və WooCommerce əlavə subscription idarəetmə xüsusiyyətlərini göstərəcək. O əlçatan olmadıqda, daxilə daxil edilmiş subscriptions əsası yenə də recurring məhsulları idarə edir.

![WooCommerce subscription inteqrasiya ayarları](/img/addons/woocommerce-subscription-integration.png)

Və bu qədər. Son istifadəçiləriniz artıq recurring məhsullar üçün billing dövrünün sonunda əl ilə ödəniş etməli olmayacaqlar.

Əsas saytınızda _WooCommerce_ altında sifariş yaradılmaqla yanaşı, hər Account üçün də subscription yaradılacaq.

### Uğursuz yeniləmə dayandırma davranışı {#failed-renewal-suspension-behavior}

Varsayılan olaraq, WooCommerce Subscriptions uğursuz ödəniş təkrar cəhd cədvəlini emal edərkən membership-lər aktiv qalır. Bu, normal WooCommerce Subscriptions güzəşt pəncərəsini qoruyur ki, bərpa edilmiş yeniləmə subscription və membership-i girişə vaxtından əvvəl müdaxilə etmədən yenidən sinxronlaşdıra bilsin.

Daha sərt giriş nəzarəti istəyən administratorlar WooCommerce Integration ayarlarında **Uğursuz yeniləmələrdə Membership-ləri dərhal dayandır** seçimini aktivləşdirə bilərlər. Bu opt-in ayarı aktiv olduqda, Ultimate Multisite renewal ödənişi uğursuz olan kimi, WooCommerce Subscriptions təkrar cəhd emalının bitməsini gözləmək əvəzinə əlaqəli membership-i dayandırır.

Developer-lər eyni qərarı `wu_woo_suspend_on_payment_failure` filter-i ilə override edə bilərlər. Hook istinad səhifələri add-on mənbəyindən yaradılır, buna görə bu icmal custom dayandırma siyasətləri üçün yalnız filter adını qeyd edir.
