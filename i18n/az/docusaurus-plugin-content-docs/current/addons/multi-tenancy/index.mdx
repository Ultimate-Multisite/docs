# Çoxicarəçilik

Performans və təhlükəsizlik üçün alt saytları serverlər və verilənlər bazaları arasında paylayın

## Versiya 1.2.0 buraxılışının əsas məqamları {#version-120-release-highlights}

Multi-Tenancy 1.2.0 istehsal izolyasiyası iş axınları üçün suveren icarəçi əsasını əlavə edir:

- Qısaömürlü, təkrar istifadədən qorunan SSO ziyarətləri ilə vəziyyətsiz icarəçi avtomatik girişi.
- Suveren icarəçilər üçün hər alt sayt üzrə verilənlər bazası və fayl sistemi izolyasiyası.
- Host-aware icarəçi təminatı üçün Hostinger çoxicarəçilik imkanı.
- Köhnə məlumat yoxlamaları və asinxron push emalı üçün suveren miqrasiya doğrulama qapıları.
- Bedrock, FrankenPHP və localhost quraşdırmaları üçün konfiqurasiya edilə bilən eyni maşın verilənlər bazası host bağlanması.
- İcarəçi bootstrap-u, verilənlər bazası yazıcısı, sxem bootstrap-u və istifadəçi təminatı təkmilləşdirmələri.
- İcarəçi etimadnamələrini təmizləyən daha təhlükəsiz suveren sökmə.

İstifadəçi təlimatlarında [suveren miqrasiya iş axını](/user-guide/getting-started/sovereign-migration-workflow), [çoxicarəçilik izolyasiyası](/user-guide/configuration/multi-tenancy-isolation) və [suveren icarəçi əməliyyatları](/user-guide/administration/sovereign-tenant-operations) bölmələrinə baxın.

## Tələblər {#requirements}

- WordPress 5.3 və ya daha yuxarı
- PHP 7.4 və ya daha yuxarı
- Multisite Ultimate plugin-i (aktiv)

## Quraşdırma {#installation}

1. Əlavə fayllarını `/wp-content/plugins/` kataloqunuza yükləyin
2. Plugin-i WordPress-də 'Plugins' menyusu vasitəsilə aktivləşdirin
3. Əlavə parametrlərini şəbəkə adminində konfiqurasiya edin

## Hazırlanma {#development}

### Quraşdırma {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### Test {#testing}

Əlavəyə hərtərəfli test çərçivəsi daxildir:

- **Vahid testləri**: WordPress test dəsti ilə PHPUnit əsaslı testlər
- **Baza test sinfi**: Ümumi test ssenariləri üçün köməkçi metodlar təmin edir
- **Test əhatəsi**: Kod əhatəsini izləmək üçün konfiqurasiya edilib
- **Multisite testi**: Testlər multisite mühitində işləyir

### Kod standartları {#code-standards}

- **WordPress kodlaşdırma standartları**: PHPCS vasitəsilə tətbiq edilir
- **PHP 7.4+ uyğunluğu**: Müasir PHP xüsusiyyətləri dəstəklənir
- **Statik analiz**: Tip yoxlaması üçün PHPStan inteqrasiyası
- **Kod modernləşdirilməsi**: PHP yüksəltmələri üçün Rector qaydaları

### Yığım sistemi {#build-system}

- **Resurs emalı**: CSS/JS faylları üçün minimallaşdırma
- **Tərcümə**: POT faylının yaradılması
- **Paketləmə**: Paylama üçün arxiv yaradılması
- **Hazırlanma rejimi**: Sazlama üçün minimallaşdırılmamış resurslar

## Konfiqurasiya {#configuration}

Əlavə şəbəkə admin interfeysi vasitəsilə əlçatan olan müxtəlif konfiqurasiya seçimlərini dəstəkləyir.

### Suveren icarəçi konfiqurasiyası {#sovereign-tenant-configuration}

Suveren icarəçilər üçün trafiki icarəçiyə yönləndirməzdən əvvəl icarəçi verilənlər bazasını, fayl sistemi kökünü, verilənlər bazası host bağlanmasını və host imkanını konfiqurasiya edin. Verilənlər bazası etimadnamələrinə, host bağlanmasına, yönləndirməyə və ya istifadəçi təminatına təsir edən hər dəyişiklikdən sonra miqrasiya doğrulamasını işə salın.

`localhost`-dan yalnız icarəçi runtime-ı və verilənlər bazası icazələri lokal soket və ya eyni maşın girişini gözlədikdə istifadə edin. Provayderiniz tərəfindən konfiqurasiya edilmiş icazə host-u bu olduqda `127.0.0.1` və ya servis host adından istifadə edin.

### Hostinger imkanı {#hostinger-capability}

Hostinger çoxicarəçilik imkanı paylaşılan Hostinger inteqrasiya token-i ilə işləyir. Doğrulamadan əvvəl domenin hPanel-də mövcud olduğunu, DNS-in Hostinger hesabına yönəldiyini və icarəçi verilənlər bazası parametrlərinin hPanel ilə uyğun olduğunu təsdiqləyin.

## Hook-lar və filtrlər {#hooks-and-filters}

### Əməliyyatlar {#actions}

- `ultimate-multisite-addon-template_init` - Əlavə başladıldıqdan sonra işə düşür
- `ultimate-multisite-addon-template_loaded` - Bütün əlavə faylları yükləndikdən sonra işə düşür

### Filtrlər {#filters}

- `ultimate-multisite-addon-template_settings` - Əlavə parametrlərini dəyişdirin
- `ultimate-multisite-addon-template_enabled` - Əlavənin aktiv/deaktiv statusunu üstələyin
