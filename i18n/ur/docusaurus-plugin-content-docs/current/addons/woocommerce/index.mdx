# WooCommerce Integration

WooCommerce Integration بالکل اسی طرح کام کرتی ہے جیسے ہمارا Manual Gateway کام کرتا ہے۔ یہ آپ کے صارفین کو بہت زیادہ لچک فراہم کرتی ہے، جس سے وہ آپ کی WooCommerce تنصیب پر دستیاب ادائیگی کے کسی بھی فارم کو منتخب کر سکتے ہیں۔ اور یہ آسان ہے: جیسے ہی صارف کی بلنگ مدت ختم ہوتی ہے، صارف کی رکنیت روک دی جاتی ہے۔ اسی وقت، نیٹ ورک کی مرکزی سائٹ کی WooCommerce تنصیب پر ایک آرڈر بنایا جائے گا۔ پھر صارف کو اپنی ای میل میں ایک رسید موصول ہوتی ہے، جس میں ادائیگی کرنے کا ایک لنک ہوتا ہے۔ جیسے ہی ادائیگی موصول ہوتی ہے، رکنیت کی تجدید ہو جاتی ہے۔

## خصوصیات {#features}

- Ultimate Multisite اور WooCommerce کے درمیان آسان انضمام
- ادائیگی کے کئی طریقے دستیاب
- شامل شدہ subscriptions core استعمال کرتے ہوئے خودکار بار بار بلنگ، اختیاری WooCommerce Subscriptions plugin بہتریوں کے ساتھ
- Checkout کل رقم میں Ultimate Multisite ٹیکسز الگ، غیر قابلِ ٹیکس WooCommerce فیس لائنز کے طور پر شامل ہوتے ہیں تاکہ صارفین ادائیگی سے پہلے ٹیکس رقوم کا جائزہ لے سکیں

## تنصیب {#installation}

1. 'ultimate-multisite-woocommerce' کو '/wp-content/plugins/' ڈائریکٹری میں اپ لوڈ کریں
2. WordPress میں 'Plugins' مینو کے ذریعے plugin کو فعال کریں
3. plugin کو ترتیب دینے کے لیے مرحلہ وار Wizard کی پیروی کریں

## استعمال کا رہنما {#usage-guide}

_**اہم نوٹ: یہ مضمون Ultimate Multisite ورژن 2.x سے متعلق ہے۔**_

_**توجہ:** Ultimate Multisite: WooCommerce Integration کے لیے ضروری ہے کہ WooCommerce کم از کم آپ کی مرکزی سائٹ پر فعال ہو۔_

ہم سمجھتے ہیں کہ _Stripe_ اور _PayPal_ کچھ ممالک میں دستیاب نہیں ہیں، جو Ultimate Multisite صارفین کو ہمارے plugin کو مؤثر طور پر استعمال کرنے سے محدود یا متاثر کرتا ہے۔ اس لیے ہم نے _WooCommerce_ کو ضم کرنے کے لیے ایک **add-on** بنایا۔

_WooCommerce_ ایک بہت مقبول eCommerce plugin ہے۔ دنیا بھر کے developers نے مختلف ادائیگی gateways کو اس کے ساتھ ضم کرنے کے لیے add-ons بنائے۔ ہم نے اس کا فائدہ اٹھاتے ہوئے ان payment gateways کو بڑھایا جنہیں آپ Ultimate Multisite بلنگ سسٹم میں استعمال کر سکتے ہیں۔

### Checkout ٹیکس فیس لائنز {#checkout-tax-fee-lines}

جب Ultimate Multisite checkout میں ٹیکس شامل ہوں، تو integration ہر Ultimate Multisite ٹیکس رقم کو WooCommerce checkout میں ایک الگ، غیر قابلِ ٹیکس فیس لائن کے طور پر شامل کرتی ہے۔ اس سے ادائیگی سے پہلے ٹیکس کی کل رقم واضح رہتی ہے جبکہ WooCommerce کو ٹیکس فیس پر دوبارہ ٹیکس لگانے سے روکا جاتا ہے۔

## ادائیگی کا انتظام کیسے کریں {#how-to-manage-the-payment}

WooCommerce Integration بالکل اسی طرح کام کرتی ہے جیسے ہمارا **Manual Gateway** کام کرتا ہے۔ کمی یہ ہے کہ ہر ادائیگی بلنگ مدت کے اختتام پر دستی طور پر ادا کرنی ہوتی ہے۔ یہ flow اس طرح کام کرتا ہے:

  * جیسے ہی صارف کا بلنگ cycle ختم ہوتا ہے، رکنیت **روک دی جاتی ہے**۔ اسی وقت، آپ کے نیٹ ورک کی مرکزی سائٹ پر _WooCommerce_ تنصیب میں ایک آرڈر بنایا جائے گا۔

  * پھر صارفین کو اپنی ای میل میں ایک رسید موصول ہوتی ہے، جس میں ادائیگی کرنے کا ایک لنک ہوتا ہے (یہ لنک ان کے subsite dashboard سے ان کی بلنگ history میں بھی ظاہر ہوتا ہے)۔ وہ لنک صارف کو _**WooCommerce**_ **checkout** پر منتقل کرتا ہے، جہاں صارفین اپنی ادائیگی process کرنے کے لیے **دستیاب _WooCommerce_ gateways** میں سے ایک منتخب کر سکیں گے۔

  * جیسے ہی ادائیگی موصول ہوتی ہے، رکنیت کی **تجدید** ہو جاتی ہے، ادائیگی Ultimate Multisite پر log ہو جاتی ہے اور آرڈر کو مکمل پر سیٹ کر دیا جاتا ہے۔

### کچھ اہم نوٹس: {#some-important-notes}

  * Ultimate Multisite کی default functionality اب بھی اس integration form کے ساتھ کام کرتی ہے۔ مثال کے طور پر، **refunds جاری کرنا** **Subscription Management** screen سے WooCommerce کے ساتھ رابطہ کرے گا تاکہ منتخب payment form استعمال کرتے ہوئے refund جاری کیا جا سکے۔

  * Ultimate Multisite: WooCommerce Integration کام کرنے کے لیے WP-Cron پر انحصار کرتی ہے۔ یہ اکثر اوقات بہت اچھی طرح کام کرتا ہے، لیکن اگر سائٹ پر traffic کم ہو تو cron کو scheduled events چیک کرنے کا اشارہ نہیں ملتا اور event چھوٹ جاتا ہے، جس سے execution delays ہو سکتے ہیں۔

## WooCommerce Subscriptions Plugin {#woocommerce-subscriptions-plugin}

ہر ماہ دستی طور پر ادائیگی شروع کرنا آخری صارفین کے لیے مثالی نہیں ہے کیونکہ subscriptions سے توقع کی جاتی ہے کہ وہ ہر billing cycle میں خودکار طور پر دہرائی جائیں۔ WooCommerce Integration add-on اب بنیادی subscriptions code کو شامل کرتا ہے، اس لیے recurring products ہمیشہ recurring billing کے لیے subscriptions استعمال کرتے ہیں۔

آپ کو اس integration کے ذریعے recurring products فروخت کرنے کے لیے الگ [WooCommerce Subscriptions plugin](https://woocommerce.com/products/woocommerce-subscriptions/) خریدنے یا install کرنے کی **ضرورت نہیں** ہے۔ اگر وہ plugin خریدا گیا، install کیا گیا، اور آپ کی main site پر دستیاب ہے، تو Ultimate Multisite اسے استعمال کرے گا اور WooCommerce اضافی subscription management خصوصیات ظاہر کرے گا۔ جب یہ دستیاب نہ ہو، تو شامل شدہ subscriptions core پھر بھی recurring products کو سنبھالتا ہے۔

![WooCommerce subscription integration settings](/img/addons/woocommerce-subscription-integration.png)

اور بس یہی ہے۔ آپ کے آخری صارفین کو recurring products کے لیے اپنے billing cycle کے اختتام پر دستی طور پر ادائیگی کرنے کی مزید ضرورت نہیں ہوگی۔

آپ کی main site پر _WooCommerce_ کے تحت ایک order بننے کے علاوہ، یہ ہر Account کے لیے ایک subscription بھی بنائے گا۔

### ناکام renewal پر suspension کا رویہ {#failed-renewal-suspension-behavior}

بطور default، memberships اس وقت تک active رہتی ہیں جب تک WooCommerce Subscriptions اپنے failed-payment retry schedule کو process کرتا ہے۔ یہ عام WooCommerce Subscriptions grace window کو برقرار رکھتا ہے تاکہ recover ہونے والی renewal subscription اور membership کو قبل از وقت access میں خلل ڈالے بغیر دوبارہ sync میں لا سکے۔

وہ administrators جو access control کو زیادہ سخت چاہتے ہیں، WooCommerce Integration settings میں **Suspend Memberships Immediately on Failed Renewals** کو enable کر سکتے ہیں۔ جب یہ opt-in setting enable ہو، تو Ultimate Multisite متعلقہ membership کو renewal payment fail ہوتے ہی suspend کر دیتا ہے، بجائے اس کے کہ WooCommerce Subscriptions retry handling کے ختم ہونے کا انتظار کرے۔

Developers اسی decision کو `wu_woo_suspend_on_payment_failure` filter کے ذریعے override کر سکتے ہیں۔ Hook reference pages add-on source سے generate کیے جاتے ہیں، اس لیے یہ overview custom suspension policies کے لیے صرف filter name کی نشاندہی کرتا ہے۔
