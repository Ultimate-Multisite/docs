# کثیر کرایہ داری {#multi-tenancy}

کارکردگی اور سیکیورٹی کے لیے ذیلی سائٹس کو سرورز اور ڈیٹابیسز میں تقسیم کریں

## Version 1.2.0 ریلیز کی جھلکیاں {#version-120-release-highlights}

Multi-Tenancy 1.2.0 پیداواری علیحدگی کے ورک فلو کے لیے خودمختار کرایہ دار کی بنیاد شامل کرتا ہے:

- مختصر مدت والی، دوبارہ استعمال سے محفوظ SSO وزٹس کے ساتھ بے حالت کرایہ دار خودکار لاگ اِن۔
- خودمختار کرایہ داروں کے لیے فی ذیلی سائٹ ڈیٹابیس اور فائل سسٹم کی علیحدگی۔
- میزبان سے باخبر کرایہ دار فراہمی کے لیے Hostinger کثیر کرایہ داری صلاحیت۔
- پرانے ڈیٹا کی جانچوں اور غیر ہم وقت پُش پروسیسنگ کے لیے خودمختار منتقلی کی توثیقی گیٹس۔
- Bedrock، FrankenPHP، اور localhost تنصیبات کے لیے قابلِ ترتیب اسی مشین کا ڈیٹابیس میزبان بائنڈنگ۔
- کرایہ دار بوٹسٹریپ، ڈیٹابیس رائٹر، اسکیما بوٹسٹریپ، اور صارف فراہمی میں بہتریاں۔
- زیادہ محفوظ خودمختار خاتمہ جو کرایہ دار کی اسناد صاف کرتا ہے۔

[خودمختار منتقلی ورک فلو](/user-guide/getting-started/sovereign-migration-workflow)، [کثیر کرایہ داری علیحدگی](/user-guide/configuration/multi-tenancy-isolation)، اور [خودمختار کرایہ دار آپریشنز](/user-guide/administration/sovereign-tenant-operations) کے لیے صارف رہنما دیکھیں۔

## تقاضے {#requirements}

- WordPress 5.3 یا اس سے زیادہ
- PHP 7.4 یا اس سے زیادہ
- Multisite Ultimate پلگ اِن (فعال)

## تنصیب {#installation}

1. addon فائلوں کو اپنی `/wp-content/plugins/` ڈائریکٹری میں اپ لوڈ کریں
2. WordPress میں 'پلگ اِنز' مینو کے ذریعے پلگ اِن فعال کریں
3. نیٹ ورک منتظم میں addon ترتیبات تشکیل دیں

## ترقی {#development}

### سیٹ اپ {#setup}

```bash
# Install dependencies
composer install
npm install

# Run tests
npm run test

# Run code standards checks
vendor/bin/phpcs
vendor/bin/phpstan

# Fix code style issues
vendor/bin/phpcbf

# Build for production
npm run build
```

### ٹیسٹنگ {#testing}

addon میں ایک جامع ٹیسٹنگ فریم ورک شامل ہے:

- **یونٹ ٹیسٹس**: WordPress ٹیسٹ سوئٹ کے ساتھ PHPUnit پر مبنی ٹیسٹس
- **بنیادی ٹیسٹ کلاس**: عام ٹیسٹ منظرناموں کے لیے مددگار طریقے فراہم کرتی ہے
- **ٹیسٹ کوریج**: کوڈ کوریج کو ٹریک کرنے کے لیے تشکیل شدہ
- **Multisite ٹیسٹنگ**: ٹیسٹس multisite ماحول میں چلتے ہیں

### کوڈ معیارات {#code-standards}

- **WordPress کوڈنگ معیارات**: PHPCS کے ذریعے نافذ
- **PHP 7.4+ مطابقت**: جدید PHP خصوصیات کی معاونت
- **جامد تجزیہ**: قسم کی جانچ کے لیے PHPStan انضمام
- **کوڈ جدید کاری**: PHP اپ گریڈز کے لیے Rector قواعد

### بلڈ سسٹم {#build-system}

- **اثاثہ پروسیسنگ**: CSS/JS فائلوں کے لیے اختصار
- **ترجمہ**: POT فائل تخلیق
- **پیکیجنگ**: تقسیم کے لیے آرکائیو تخلیق
- **ترقیاتی موڈ**: ڈیبگنگ کے لیے غیر مختصر اثاثے

## تشکیل {#configuration}

addon مختلف تشکیل اختیارات کی معاونت کرتا ہے جو نیٹ ورک منتظم انٹرفیس کے ذریعے قابلِ رسائی ہیں۔

### خودمختار کرایہ دار تشکیل {#sovereign-tenant-configuration}

خودمختار کرایہ داروں کے لیے، کرایہ دار ڈیٹابیس، فائل سسٹم روٹ، ڈیٹابیس میزبان بائنڈنگ، اور میزبان صلاحیت کو کرایہ دار کی طرف ٹریفک روٹ کرنے سے پہلے تشکیل دیں۔ ہر اس تبدیلی کے بعد منتقلی کی توثیق چلائیں جو ڈیٹابیس اسناد، میزبان بائنڈنگ، روٹنگ، یا صارف فراہمی کو متاثر کرتی ہو۔

`localhost` صرف اس وقت استعمال کریں جب کرایہ دار رن ٹائم اور ڈیٹابیس گرانٹس مقامی ساکٹ یا اسی مشین تک رسائی کی توقع کریں۔ `127.0.0.1` یا سروس میزبان نام استعمال کریں جب وہی آپ کے فراہم کنندہ کے ذریعے تشکیل شدہ گرانٹ میزبان ہو۔

### Hostinger صلاحیت {#hostinger-capability}

Hostinger کثیر کرایہ داری صلاحیت مشترکہ Hostinger انضمام ٹوکن کے ساتھ کام کرتی ہے۔ توثیق سے پہلے تصدیق کریں کہ ڈومین hPanel میں موجود ہے، DNS Hostinger اکاؤنٹ کی طرف اشارہ کرتا ہے، اور کرایہ دار ڈیٹابیس ترتیبات hPanel سے مطابقت رکھتی ہیں۔

## ہکس اور فلٹرز {#hooks-and-filters}

### Actions {#actions}

- `ultimate-multisite-addon-template_init` - addon کی ابتدا کے بعد چلایا جاتا ہے
- `ultimate-multisite-addon-template_loaded` - تمام addon فائلیں لوڈ ہونے کے بعد چلایا جاتا ہے

### Filters {#filters}

- `ultimate-multisite-addon-template_settings` - addon ترتیبات میں ترمیم کریں
- `ultimate-multisite-addon-template_enabled` - addon کو فعال/غیرفعال کرنے کی حیثیت کو اوور رائیڈ کریں
