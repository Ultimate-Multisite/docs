# Ultimate AI Connector for WebLLM Changelog {#ultimate-ai-connector-for-webllm-changelog}

## Version 1.1.0 — Released on 2026-04-09 {#version-110--released-on-2026-04-09}

* **نیا:** ایڈمن بار اسٹیٹس انڈیکیٹر کے ساتھ تیرتا ہوا چیٹ وِجٹ — اب کوئی بھی لاگ ان صارف فرنٹ اینڈ سے براہ راست براؤزر سائیڈ LLM کو پرامپٹ دے سکتا ہے۔
* **نیا:** SharedWorker رن ٹائم — اب MLC engine ایک SharedWorker میں چلتا ہے، جس کی وجہ سے متعدد ٹیبز ایک ہی GPU سیشن شیئر کرتے ہیں بجائے اس کے کہ وہ اس پر ایک دوسرے سے تصادم کریں۔
* **نیا:** apiFetch مڈل ویئر انٹرسیپٹر — WordPress REST کی درخواستیں جو AI Client SDK کے پیٹرن سے مطابقت رکھتی ہیں، انہیں شفاف طریقے سے مقامی WebLLM بروکر کو بھیجا جاتا ہے، جس کے لیے لوپ بیک HTTP راؤنڈ ٹرپ کی ضرورت نہیں ہے۔
* **نیا:** وِجٹ سیٹنگز UI جو کنیکٹر پینل میں شامل کیا گیا ہے، تاکہ چیٹ وِجٹ کو آن/آف کیا جا سکے اور خودکار پرامپٹ کے رویے کو ترتیب دیا جا سکے۔
* **فکس:** IndexedDB کیش بیک اینڈ کو مجبور کیا گیا تاکہ ماڈل ویٹ ڈاؤن لوڈز HuggingFace xet CDN ری ڈائریکٹس سے بچ سکیں جو ڈیفالٹ Cache API پاتھ کو خراب کر دیتے ہیں۔
* **فکس:** ایمبیڈنگ ماڈلز کے لیے context_window KV-cache اووررائڈ کو چھوڑ دیا گیا (ان میں کوئی ڈیکوڈر نہیں ہے اور اس اووررائڈ کی وجہ سے رن ٹائم ایرر آ رہا تھا)۔
* **فکس:** ورکر ٹیب لوڈ ہونے سے پہلے /webllm/v1/models پر کولڈ-سٹارٹ امیدوار ماڈل کا اشتہار دیا جاتا ہے، تاکہ SDK صارفین کو فوری طور پر ماڈل نظر آئے۔
* **فکس:** WebLLM کو wpai_preferred_text_models فلٹر میں جوڑا گیا تاکہ AI Experiments فیچر، جب کوئی ماڈل ترتیب دیا جائے، تو براؤزر انجن کی طرف ریٹ کرے۔
* **بہتر کیا گیا:** اینڈ-ٹو-اینڈ ٹیسٹنگ کے دوران کیش-باستنگ، کنٹینٹ نارملائزیشن، اور ہارڈویئر-ریفرنس کے مسائل کو حل کیا گیا۔
