# سبق 10: لانچ کا دن {#lesson-10-launch-day}

سب کچھ تیار ہے۔ دروازے کھولنے سے پہلے، یہ سبق ایک پری-لانچ چیک لسٹ کے ذریعے چلتا ہے تاکہ یہ یقینی بنایا جا سکے کہ کچھ بھی خراب، غائب، یا شرمناک نہ ہو۔

## ہم کہاں تھے {#where-we-left-off}

FitSite میں ٹیمپلیٹس، پلانز، چیک آؤٹ، برانڈنگ، آن بورڈنگ، اور پرائسنگ سب کچھ سیٹ ہو چکا ہے۔ اب ہم تصدیق کرتے ہیں کہ ہر چیز کام کر رہی ہے اور آن لائن جا رہے ہیں۔

## پری-لانچ چیک لسٹ {#pre-launch-checklist}

ہر آئٹم پر کام کریں۔ کوئی بھی چھوڑیں نہیں۔

### پلیٹ فارم انفراسٹرکچر {#platform-infrastructure}

- [ ] ہوسٹنگ stable ہے اور بوجھ (load) کے تحت اچھا پرفارم کر رہی ہے
- [ ] Wildcard SSL فعال ہے اور تمام سب ڈومینز HTTPS پر سروس دے رہے ہیں
- [ ] Domain mapping کام کرتا ہے -- ایک سائٹ بنا کر اور کسٹم ڈومین میپ کر کے ٹیسٹ کریں
- [ ] بیک اپس سیٹ ہو چکے ہیں اور ٹیسٹ کیے گئے ہیں (تصدیق کے لیے کم از کم ایک کو restore کریں)
- [ ] نگرانی (Monitoring) کی سہولت موجود ہے -- آپ کو معلوم ہوگا کہ اگر پلیٹ فارم ڈاؤن ہوتا ہے

### ٹیمپلیٹس {#templates}

- [ ] تمام تینوں ٹیمپلیٹس نئی سائٹس پر صحیح طریقے سے لوڈ ہوتے ہیں
- [ ] Placeholder content مددگار ہے اور ٹائپو سے پاک ہے
- [ ] تمام تصاویر مناسب طریقے سے لائسنس شدہ ہیں (کوئی اسٹاک فوٹو واٹر مارک نہیں)
- [ ] ہر ٹیمپلیٹ پیج پر Mobile responsiveness کام کرتا ہے
- [ ] پیج لوڈ کی رفتار قابل قبول ہے (GTmetrix یا PageSpeed Insights جیسے ٹول سے ٹیسٹ کریں)
- [ ] کسی بھی ٹیمپلیٹ پر کوئی ٹوٹی ہوئی لنک یا غائب اثاثہ (missing assets) نہیں ہے

### پلانز اور پروڈکٹس {#plans-and-products}

- [ ] تمام تینوں پلانز فعال اور نظر آنے والے ہیں
- [ ] پلان کی تفصیلات درست اور مخصوص (niche-specific) ہیں
- [ ] پرائسنگ صحیح ہے (ماہانہ اور سالانہ)
- [ ] سیٹ اپ فیس صحیح پلانز پر سیٹ ہو چکی ہے
- [ ] ٹرائل پیریڈ Starter پلان پر کام کرتا ہے
- [ ] چیک آؤٹ کے دوران Order bumps صحیح طریقے سے ظاہر ہوتے ہیں
- [ ] ہر پلان کے لحاظ سے Plugin اور theme کی حدود (limitations) صحیح طریقے سے نافذ کی گئی ہیں

### چیک آؤٹ فلو {#checkout-flow}

- [ ] ہر پلان پر مکمل ٹیسٹ سائن اپ کریں (ٹیسٹ پیمنٹ موڈ استعمال کریں)
- [ ] ٹیمپلیٹ سلیکشن ہر پلان کے لیے صحیح ٹیمپلیٹس دکھاتا ہے
- [ ] ادائیگی کامیابی سے ہوتی ہے
- [ ] سائن اپ کے بعد کسٹمر کو خوش آمدید ای میل ملتی ہے
- [ ] صحیح ٹیمپلیٹ کے ساتھ نئی سائٹ بنائی جاتی ہے
- [ ] کسٹمر فوری طور پر اپنی نئی سائٹ میں لاگ ان کر سکتا ہے
- [ ] ڈسکاؤنٹ کوڈز صحیح طریقے سے کام کرتے ہیں

### برانڈنگ {#branding}

- [ ] لاگ ان پیج پر FitSite کی برانڈنگ نظر آتی ہے
- [ ] ایڈمن ڈیش بورڈ پر FitSite کی برانڈنگ نظر آتی ہے
- [ ] تمام سسٹم ای میلز میں FitSite کی برانڈنگ اور فٹنس سے متعلق زبان استعمال ہوتی ہے
- [ ] انوائسز میں آپ کی بزنس تفصیلات کے ساتھ صحیح طریقے سے دکھایا جاتا ہے
- [ ] مارکیٹنگ سائٹ live ہے اور چیک آؤٹ فارم سے منسلک ہے

### آن بورڈنگ {#onboarding}

- [ ] Quick Start widget نئے کسٹمر ڈیش بورڈز پر ظاہر ہوتا ہے
- [ ] تمام Quick Start لنکس صحیح پیجز کی طرف جاتے ہیں
- [ ] خوش آمدید ای میل سیریز سیٹ ہو چکی ہے اور ٹیسٹ کی گئی ہے
- [ ] Knowledge base کے آرٹیکلز شائع اور قابل رسائی ہیں
- [ ] اکاؤنٹ پیج پر صحیح پلان کی معلومات اور اپ گریڈ کے آپشنز دکھائے جاتے ہیں

### قانونی اور کاروباری {#legal-and-business}

- [ ] سروس کی شرائط (Terms of service) شائع ہیں اور چیک آؤٹ سے منسلک ہیں
- [ ] پرائیویسی پالیسی (Privacy policy) شائع اور قابل رسائی ہے
- [ ] ریفنڈ پالیسی (Refund policy) متعین اور دستاویز شدہ ہے
- [ ] ادائیگی وصول کرنے کے لیے بزنس اینٹیٹی سیٹ ہو چکی ہے
- [ ] پیمنٹ گیٹ وے live موڈ میں ہے (ٹیسٹ موڈ میں نہیں)
- [ ] ٹیکس کی ترتیب آپ کے دائرہ اختیار کے لیے درست ہے

## Soft Launch بمقابلہ Hard Launch {#soft-launch-vs-hard-launch}

ایک دو مرحلوں والے لانچ پر غور کریں:

### مرحلہ 1: Soft Launch {#phase-1-soft-launch}

عوامی لانچ سے پہلے 5-10 فٹنس اسٹوڈیو مالکان کو سائن اپ کرنے کی دعوت دیں۔ یہ آپ کے بیٹا کسٹمر ہیں۔ ان کو یہ دینے کے بدلے میں ایک بڑا چھوٹ (زندگی بھر 50% چھوٹ، یا 3 ماہ مفت) پیش کریں:

- سائن اپ اور آن بورڈنگ کے تجربے پر ایماندار رائے
- ان کی سائٹ کو ایک نمائش کے طور پر استعمال کرنے کی اجازت
- وہ جو کوئی بھی بگ یا مسئلہ محسوس کریں، اس کی رپورٹنگ

یہ آپ کو حقیقی کسٹمر کی رائے اور عوامی ہونے سے پہلے دکھانے کے لیے live سائٹس فراہم کرتا ہے۔

### مرحلہ 2: Public Launch {#phase-2-public-launch}

Soft launch کی رائے شامل کرنے کے بعد:

- پیمنٹ گیٹ وے کو live موڈ پر سوئچ کریں
- اپنی مارکیٹنگ سائٹ شائع کریں
- کسٹمر حاصل کرنا شروع کریں (Lesson 11)
- متعلقہ فٹنس انڈسٹری چینلز پر اعلان کریں

## لانچ کے دن کے اقدامات {#launch-day-actions}

جس دن آپ عوامی ہوتے ہیں:

1. **live پیمنٹس پر سوئچ کریں** -- اپنے پیمنٹ گیٹ وے پر ٹیسٹ موڈ کو غیر فعال کریں
2. **ایک بار پھر تصدیق کریں** -- ایک حقیقی ادائیگی کے ساتھ مکمل ٹیسٹ سائن اپ کریں (بعد میں خود کو ریفنڈ کر لیں)
3. **قریباً نگرانی کریں** -- غلطیوں، ناکام سائن اپ، یا ادائیگی کے مسائل پر نظر رکھیں
4. **موجود رہیں** -- آپ کے پہلے حقیقی کسٹمروں کو مدد کی ضرورت ہو سکتی ہے، اور تیز ردعمل اعتماد پیدا کرتا ہے
5. **مختصر جشن منائیں** -- پھر کام پر واپس آ جائیں

## کیا غلط ہو سکتا ہے {#what-can-go-wrong}

ان چیزوں کے لیے تیار رہیں:

- **پیمنٹ گیٹ وے کے مسائل**: اپنے گیٹ وے فراہم کنندہ کے سپورٹ کا رابطہ تیار رکھیں
- **SSL سرٹیفکیٹ کے مسائل**: سرٹیفکیٹس کو کیسے چیک اور ری نیو کرنا ہے، یہ جانیں
- **ای میل کی ترسیل میں ناکامی**: ٹیسٹ کریں کہ ای میلز واقعی پہنچتی ہیں (اسپم فولڈرز چیک کریں)
- **بوجھ کے تحت کارکردگی**: اگر آپ کو ٹریفک میں اضافہ ہوتا ہے، تو اپنی ہوسٹنگ کو scale کرنا کیسے ہے، یہ جانیں
- **کسٹمر کی الجھن**: اپنی نالج بیس اور سپورٹ چینلز تیار رکھیں
