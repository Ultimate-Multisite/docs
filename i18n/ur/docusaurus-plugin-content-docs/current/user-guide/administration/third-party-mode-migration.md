# تھرڈ- پارٹی موڈ کی منتقلی (Migration) {#third-party-mode-migration}

Superdav AI Agent v1.12.0 نے تھرڈ- پارٹی صلاحیتوں (abilities) کو ہینڈل کرنے کے طریقے میں تبدیلی کی ہے۔ **تھرڈ- پارٹی موڈ اب ڈیفالٹ طور پر آٹو (auto) پر سیٹ ہوتا ہے،** جس سے دستی ترتیب (manual configuration) کے بغیر WordPress 7.0+ پر مقامی WordPress Abilities API انٹیگریشن ممکن ہو جاتی ہے۔

## کیا تبدیل ہوا؟ {#what-changed}

### v1.12.0 سے پہلے {#before-v1120}

تھرڈ- پارٹی صلاحیتوں کے لیے دستی ترتیب کی ضرورت ہوتی تھی:

- آپ کو واضح طور پر "تھرڈ- پارٹی موڈ" کو فعال کرنا پڑتا تھا۔
- صلاحیتوں کو ایک کسٹم رجسٹری سے لوڈ کیا جاتا تھا۔
- WordPress Abilities API کے ساتھ انٹیگریشن اختیاری (optional) تھا۔
- لیگیسی موڈ (Legacy mode) ڈیفالٹ تھا۔

### v1.12.0 کے بعد {#after-v1120}

تھرڈ- پارٹی صلاحیتیں خود بخود کام کرتی ہیں:

- تھرڈ- پارٹی موڈ ڈیفالٹ طور پر "آٹو" پر ہوتا ہے۔
- صلاحیتیں WordPress Abilities API کے ساتھ مقامی طور پر انٹیگریٹ ہوتی ہیں۔
- WordPress 7.0+ پر کسی دستی ترتیب کی ضرورت نہیں ہے۔
- پرانے WordPress ورژن کے لیے لیگیسی موڈ اب بھی دستیاب ہے۔

## کس کو متاثر کرتا ہے؟ {#who-is-affected}

### نئی انسٹالیشنز (WordPress 7.0+) {#new-installations-wordpress-70}

**کوئی کارروائی درکار نہیں۔** تھرڈ- پارٹی موڈ خود بخود "آٹو" پر سیٹ ہو جاتا ہے، اور صلاحیتیں بغیر کسی پریشانی کے کام کرتی ہیں۔

### موجودہ انسٹالیشنز {#existing-installations}

**آپ کی سیٹنگز محفوظ ہیں۔** اگر آپ استعمال کر رہے تھے:

- **لیگیسی موڈ (Legacy mode)**: آپ لیگیسی موڈ میں ہی رہیں گے (کوئی تبدیلی نہیں)
- **دستی تھرڈ- پارٹی موڈ (Manual third-party mode)**: آپ دستی موڈ میں ہی رہیں گے (کوئی تبدیلی نہیں)
- **آٹو موڈ (Auto mode)**: آپ آٹو موڈ کے ساتھ جاری رکھیں گے (کوئی تبدیلی نہیں)

### WordPress ورژن 7.0 سے پہلے {#wordpress-versions-before-70}

**لیگیسی موڈ اب بھی دستیاب ہے۔** اگر آپ WordPress 6.x یا اس سے پہلے کے ورژن پر ہیں:

- تھرڈ- پارٹی موڈ ڈیفالٹ طور پر "لیگیسی" پر ہوتا ہے۔
- اگر آپ چاہیں تو آپ تھرڈ- پارٹی موڈ کو دستی طور پر فعال کر سکتے ہیں۔
- مقامی Abilities API استعمال کرنے کے لیے WordPress 7.0+ پر اپ گریڈ کریں۔

## موڈز کو سمجھنا {#understanding-the-modes}

### آٹو موڈ (Auto Mode) (نیا ڈیفالٹ) {#auto-mode-new-default}

**آٹو موڈ** مقامی WordPress Abilities API انٹیگریشن استعمال کرتا ہے:

- صلاحیتیں WordPress hooks کے ذریعے رجسٹر ہوتی ہیں۔
- WordPress 7.0+ Abilities API کے ساتھ مکمل مطابقت (compatibility)۔
- تھرڈ- پارٹی صلاحیتوں کی خودکار دریافت (Automatic discovery)۔
- کسی دستی ترتیب کی ضرورت نہیں۔

**کب استعمال کریں**: WordPress 7.0+ کے ساتھ تھرڈ- پارٹی صلاحیتوں کے لیے

### دستی موڈ (Manual Mode) {#manual-mode}

**دستی موڈ** کے لیے واضح ترتیب کی ضرورت ہوتی ہے:

- آپ مخصوص تھرڈ- پارٹی صلاحیتوں کو لوڈ کرنے کی وضاحت کرتے ہیں۔
- ٹیسٹنگ یا منتخب صلاحیتوں کو لوڈ کرنے کے لیے مفید ہے۔
- اس کے لیے کنفیگریشن فائلوں میں ترمیم کی ضرورت ہوتی ہے۔
- زیادہ کنٹرول، لیکن زیادہ سیٹ اپ۔

**کب استعمال کریں**: ٹیسٹنگ، منتخب صلاحیتوں کو لوڈ کرنا، یا کسٹم کنفیگریشنز کے لیے

### لیگیسی موڈ (Legacy Mode) {#legacy-mode}

**لیگیسی موڈ** پرانے تھرڈ- پارٹی صلاحیتوں کے سسٹم کا استعمال کرتا ہے:

- کسٹم ایبلٹی رجسٹری (WordPress Abilities API نہیں)۔
- پرانے WordPress ورژن کے ساتھ پیچھے کی طرف مطابقت (Backward compatible)۔
- کوئی مقامی WordPress انٹیگریشن نہیں۔
- ختم کیا جا چکا ہے لیکن اب بھی سپورٹ کیا جاتا ہے۔

**کب استعمال کریں**: WordPress 6.x یا اس سے پہلے، یا جب آپ کو لیگیسی مطابقت کی ضرورت ہو۔

## آپ کا موجودہ موڈ چیک کرنا {#checking-your-current-mode}

### ایڈمن پینل کے ذریعے {#via-admin-panel}

1. **WordPress Admin** → **Superdav AI Agent** → **Settings** پر جائیں۔
2. **Third-Party Mode** سیٹنگ کو تلاش کریں۔
3. آپ کو اپنا موجودہ موڈ اور اسے تبدیل کرنے کے آپشنز نظر آئیں گے۔

### کوڈ کے ذریعے {#via-code}

```php
$mode = get_option( 'superdav_third_party_mode' );
echo $mode; // 'auto', 'manual', یا 'legacy'
```

## اپنا موڈ تبدیل کرنا {#changing-your-mode}

### آٹو موڈ پر سوئچ کرنا {#switch-to-auto-mode}

اگر آپ WordPress 7.0+ پر ہیں اور آٹو موڈ استعمال کرنا چاہتے ہیں:

1. **Superdav AI Agent** → **Settings** پر جائیں۔
2. **Third-Party Mode** تلاش کریں۔
3. **Auto (WordPress Abilities API)** کا انتخاب کریں۔
4. **Save** پر کلک کریں۔

Superdav AI Agent خود بخود تھرڈ- پارٹی صلاحیتوں کو دریافت اور رجسٹر کر دے گا۔

### دستی موڈ پر سوئچ کرنا {#switch-to-manual-mode}

اگر آپ کنٹرول کرنا چاہتے ہیں کہ کون سی صلاحیتیں لوڈ ہوں:

1. **Superdav AI Agent** → **Settings** پر جائیں۔
2. **Third-Party Mode** تلاش کریں۔
3. **Manual** کا انتخاب کریں۔
4. **Save** پر کلک کریں۔
5. کون سی صلاحیتیں لوڈ کرنی ہیں، یہ بتانے کے لیے اپنی کنفیگریشن فائل میں ترمیم کریں۔

### لیگیسی موڈ پر سوئچ کرنا {#switch-to-legacy-mode}

اگر آپ کو لیگیسی مطابقت کی ضرورت ہے:

1. **Superdav AI Agent** → **Settings** پر جائیں۔
2. **Third-Party Mode** تلاش کریں۔
3. **Legacy** کا انتخاب کریں۔
4. **Save** پر کلک کریں۔

## آٹو موڈ کے فوائد {#benefits-of-auto-mode}

### خودکار دریافت (Automatic Discovery) {#automatic-discovery}

صلاحیتیں خود بخود مندرجہ ذیل جگہوں سے دریافت ہوتی ہیں:

- انسٹال شدہ plugins
- فعال theme
- must-use plugins
- drop-in plugins

کسی دستی رجسٹریشن کی ضرورت نہیں۔

### مقامی انٹیگریشن (Native Integration) {#native-integration}

صلاحیتیں WordPress Abilities API کے ساتھ انٹیگریٹ ہوتی ہیں:

- WordPress core کے ساتھ مستقل۔
- WordPress admin کے ساتھ کام کرتا ہے۔
- Abilities API استعمال کرنے والے دیگر plugins کے ساتھ مطابقت رکھتا ہے۔
- جیسے جیسے WordPress ارتقا پذیر ہوتا ہے، مستقبل کے لیے تیار۔

### آسان انتظام (Simplified Management) {#simplified-management}

- ترمیم کرنے کے لیے کوئی کنفیگریشن فائل نہیں۔
- کوئی دستی صلاحیت رجسٹریشن نہیں۔
- Ability Visibility کنٹرول خود بخود کام کرتے ہیں۔
- ایڈمن نوٹس آپ کو غیر درجہ بندی شدہ صلاحیتوں کے بارے میں الرٹ کرتے ہیں۔

### بہتر کارکردگی (Better Performance) {#better-performance}

- صلاحیتوں کو کیش کیا جاتا ہے۔
- ضرورت پڑنے پر Lazy-loaded ہوتا ہے۔
- WordPress 7.0+ کے لیے بہتر بنایا گیا ہے۔

## منتقلی کا راستہ (Migration Path) {#migration-path}

### اگر آپ WordPress 6.x پر ہیں {#if-youre-on-wordpress-6x}

1. **WordPress 7.0+ پر اپ گریڈ کریں** (جب تیار ہوں)
2. **Superdav AI Agent** کو v1.12.0+ پر اپ ڈیٹ کریں۔
3. **تھرڈ- پارٹی موڈ کو آٹو پر تبدیل کریں** (اختیاری؛ لیگیسی موڈ اب بھی کام کرتا ہے)
4. مناسب رسائی کنٹرول کو یقینی بنانے کے لیے **صلاحیت کی مرئیت (ability visibility)** کا جائزہ لیں۔

### اگر آپ WordPress 7.0+ پر ہیں {#if-youre-on-wordpress-70}

1. **Superdav AI Agent** کو v1.12.0+ پر اپ ڈیٹ کریں۔
2. **تصدیق کریں کہ تھرڈ- پارٹی موڈ آٹو پر سیٹ ہے** (یہ ڈیفالٹ ہونا چاہیے)۔
3. مناسب رسائی کنٹرول کو یقینی بنانے کے لیے **صلاحیت کی مرئیت** کا جائزہ لیں۔
4. تصدیق کے لیے **تھرڈ- پارٹی صلاحیتوں** کا ٹیسٹ کریں۔

## مسائل کا حل (Troubleshooting) {#troubleshooting}

### آٹو موڈ میں صلاحیتیں لوڈ نہیں ہو رہیں {#abilities-arent-loading-in-auto-mode}

- تصدیق کریں کہ آپ WordPress 7.0+ پر ہیں۔
- چیک کریں کہ تھرڈ- پارٹی موڈ "آٹو" پر سیٹ ہے۔
- تصدیق کریں کہ جو plugin صلاحیت فراہم کر رہا ہے وہ فعال ہے۔
- رجسٹریشن کی غلطیوں کے لیے WordPress error logs چیک کریں۔

### میں لیگیسی موڈ رکھنا چاہتا ہوں {#i-want-to-keep-legacy-mode}

- **Settings** → **Third-Party Mode** پر جائیں۔
- **Legacy** کا انتخاب کریں۔
- **Save** پر کلک کریں۔
- لیگیسی موڈ کام کرتا رہے گا۔

### میری کسٹم صلاحیتیں نظر نہیں آ رہیں {#my-custom-abilities-arent-showing}

- تصدیق کریں کہ وہ WordPress hooks کے ذریعے رجسٹر ہیں۔
- چیک کریں کہ وہ Abilities API کو صحیح طریقے سے نافذ کرتے ہیں۔
- WordPress error logs کا جائزہ لیں۔
- تمام رجسٹرڈ صلاحیتیں دیکھنے کے لیے **Ability Visibility** ایڈمن پیج استعمال کریں۔

### مجھے "unclassified ability" نوٹس مل رہے ہیں {#im-getting-unclassified-ability-notices}

- یہ نئی تھرڈ- پارٹی صلاحیتوں کے لیے عام ہے۔
- ایڈمن نوٹس میں ان کا جائزہ لیں اور انہیں درجہ بندی (classify) کریں۔
- درجہ بندی کے تفصیلات کے لیے **Ability Visibility** دیکھیں۔

## پیچھے کی طرف مطابقت (Backward Compatibility) {#backward-compatibility}

### موجودہ کنفیگریشنز {#existing-configurations}

اگر آپ کے پاس پہلے سے تھرڈ- پارٹی صلاحیتوں کی کنفیگریشن موجود ہے:

- **لیگیسی موڈ**: آپ کی کنفیگریشن کام کرتی رہے گی۔
- **دستی موڈ**: آپ کی کنفیگریشن کام کرتی رہے گی۔
- **آٹو موڈ**: آپ کی کنفیگریشن کو نظر انداز کیا جائے گا (آٹو موڈ کا حکم چلے گا)۔

اپنی کسٹم کنفیگریشن کو برقرار رکھنے کے لیے، دستی یا لیگیسی موڈ میں رہیں۔

### ختم ہونے کا ٹائم لائن (Deprecation Timeline) {#deprecation-timeline}

- **v1.12.0**: لیگیسی اور دستی موڈ مکمل طور پر سپورٹ کیے جاتے ہیں۔
- **v1.13.0+**: لیگیسی موڈ پر ختم ہونے کے نوٹس نظر آ سکتے ہیں۔
- **v2.0.0**: لیگیسی موڈ کو ہٹایا جا سکتا ہے (TBD)۔

## بہترین طریقے (Best Practices) {#best-practices}

### نئی انسٹالیشنز کے لیے {#for-new-installations}

- آٹو موڈ استعمال کریں (یہ ڈیفالٹ ہے)۔
- Superdav AI Agent کو صلاحیتوں کو خود بخود دریافت کرنے دیں۔
- رسائی کو کنٹرول کرنے کے لیے Ability Visibility استعمال کریں۔

### موجودہ انسٹالیشنز کے لیے {#for-existing-installations}

- جہاں ممکن ہو وہاں WordPress 7.0+ پر اپ گریڈ کریں۔
- آسان انتظام کے لیے آٹو موڈ پر سوئچ کریں۔
- Ability Visibility کا استعمال کرتے ہوئے صلاحیتوں کا جائزہ لیں اور انہیں درجہ بندی کریں۔

### کسٹم صلاحیتوں کے لیے {#for-custom-abilities}

- صلاحیتوں کو WordPress hooks (Abilities API) کے ذریعے رجسٹر کریں۔
- کسٹم ایبلٹی رجسٹری سے گریز کریں۔
- آٹو موڈ کے ساتھ WordPress 7.0+ پر ٹیسٹ کریں۔

## اگلے اقدامات (Next Steps) {#next-steps}

1. **اپنا WordPress ورژن چیک کریں**: آٹو موڈ کے لیے تصدیق کریں کہ آپ 7.0+ پر ہیں۔
2. **اپنے تھرڈ- پارٹی موڈ کا جائزہ لیں**: سیٹنگز پر جائیں اور اپنا موجودہ موڈ چیک کریں۔
3. **اگر ضرورت ہو تو اپ ڈیٹ کریں**: اگر آپ WordPress 7.0+ پر ہیں تو آٹو موڈ پر سوئچ کریں۔
4. **صلاحیتوں کو درجہ بندی کریں**: کسی بھی غیر درجہ بندی شدہ صلاحیتوں کا جائزہ لیں اور انہیں درجہ بندی کریں۔
5. **ٹیسٹ کریں**: تصدیق کریں کہ آپ کی تھرڈ- پارٹی صلاحیتیں صحیح طریقے سے کام کر رہی ہیں۔
