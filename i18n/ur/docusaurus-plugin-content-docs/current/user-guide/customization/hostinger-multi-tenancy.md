# ہوسٹنگر ملٹی ٹیننسی

Ultimate Multisite: Multi-Tenancy 1.2.0 میں ایک Hostinger ملٹی ٹیننسی کی صلاحیت شامل کی گئی ہے تاکہ ہوسٹڈ ڈومینز موجودہ Hostinger ڈومین میپنگ کے ساتھ ساتھ خودمختار ٹینٹنٹ پروویژن (sovereign tenant provisioning) میں حصہ لے سکیں۔

جب آپ ٹینٹنٹ ڈومینز اور الگ تھلگ ٹینٹنٹ انفراسٹرکچر کو Hostinger hPanel کے ذریعے سنبھال رہے ہوں تو اس صلاحیت کا استعمال کریں۔

## سیٹ اپ کے نوٹس

1. **Ultimate Multisite > Settings > Host Integrations** کے تحت کور ہوسٹنگر انٹیگریشن کو کنفیگر کریں۔
2. تصدیق کریں کہ Hostinger API ٹوکن مطلوبہ ڈومین یا سبڈومین کو مینج کر سکتا ہے۔
3. ملٹی ٹیننسی ایڈ آن کو فعال (Enable) کریں۔
4. ٹینٹنٹ شائع کرنے سے پہلے ٹینٹنٹ کی تنہائی (isolation) کی حکمت عملی کو کنفیگر کریں۔
5. ٹینٹنٹ پر پروڈکشن ٹریفک بھیجنے سے پہلے مائریگیشن تصدیق ورک فلو چلائیں۔

Hostinger کی صلاحیت ہوسٹ سائیڈ آپریشنز کے لیے مشترکہ Hostinger کنکشن کا استعمال کرتی ہے۔ DNS کو اب بھی صحیح Hostinger اکاؤنٹ کی طرف پوائنٹ کرنا چاہیے، اور hPanel اکاؤنٹ کی حدود لاگو ہوتی رہتی ہیں۔

## صلاحیت سے متعلق تبدیلیاں

- ہوٹ سائیڈ کے علم رکھنے والے ڈومین آپریشنز کے ساتھ خودمختار ٹینٹنٹ بنائے جا سکتے ہیں۔
- تصدیق دینے سے پہلے میزبانی (grant) کی تصدیق کرنے سے پہلے مشین پر موجود ڈیٹا بیس ہوسٹ سٹرنگز کو نارملائز کیا جاتا ہے۔
- Hostinger کے ذریعے مینج کیے جانے والے ٹینٹنٹ کو hPanel میں دکھائے گئے ڈیٹا بیس ہوسٹ ویلیو کا استعمال کرنا چاہیے جب تک کہ WordPress رن ٹائم کو مقامی اووررائڈ کی ضرورت نہ ہو۔
- SSO وزٹس اس بات پر منحصر ہیں کہ ٹینٹنٹ ڈومین Hostinger کے ہوسٹ کیے گئے ٹینٹنٹ پر کیسے ریزولف ہوتا ہے۔

## Hostinger ٹینٹنٹ کے لیے مسائل حل کرنا (Troubleshooting)

- اگر کوئی ٹینٹنٹ انسٹال ہونا ناکام ہو جائے، تو تصدیق کریں کہ ڈومین پہلے سے ہی Hostinger اکاؤنٹ سے منسلک ہے۔
- اگر ڈیٹا بیس کی تصدیق فیل ہو جائے، تو hPanel کے ساتھ ٹینٹنٹ DB یوزر نیم، ڈیٹا بیس کا نام اور ہوسٹ باائنڈنگ کا موازنہ کریں۔
- اگر **Visit (SSO)** ناکام ہو جائے، تو اس بات کی تصدیق کریں کہ ٹینٹنٹ ڈومین کے لیے DNS اور SSL فعال ہیں۔
- اگر ڈیٹیلنگ (teardown) سے ہوسٹ کے وسائل پیچھے رہ جاتے ہیں، تو بیک اپ کی تصدیق کرنے کے بعد hPanel سے باقی تمام ڈیٹا بیسز، صارفین یا فولڈرز کو ہٹا دیں۔
