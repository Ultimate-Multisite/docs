# Closte انضمام

## جائزہ
Closte ایک منظم WordPress ہوسٹنگ پلیٹ فارم ہے جو Google Cloud انفراسٹرکچر پر بنایا گیا ہے۔ یہ انضمام Ultimate Multisite اور Closte کے درمیان خودکار ڈومین مطابقت پذیری اور SSL سرٹیفکیٹ کے انتظام کو ممکن بناتا ہے۔

## خصوصیات
- خودکار ڈومین مطابقت پذیری
- SSL سرٹیفکیٹ کا انتظام
- Wildcard ڈومین کی سہولت
- اگر آپ Closte پر چل رہے ہیں تو کسی ترتیب کی ضرورت نہیں

## تقاضے
اگر آپ Closte استعمال کر رہے ہیں تو درج ذیل constant آپ کی `wp-config.php` فائل میں متعین ہونا ضروری ہے:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

اگر آپ Closte پر ہوسٹنگ کر رہے ہیں تو یہ constant عام طور پر پہلے سے متعین ہوتا ہے۔

## سیٹ اپ کی ہدایات

### 1. اپنی Closte API Key کی تصدیق کریں

اگر آپ Closte پر ہوسٹنگ کر رہے ہیں تو `CLOSTE_CLIENT_API_KEY` constant آپ کی `wp-config.php` فائل میں پہلے سے متعین ہونا چاہیے۔ آپ اپنی `wp-config.php` فائل چیک کر کے اس کی تصدیق کر سکتے ہیں۔

### 2. انضمام کو فعال کریں

1. اپنے WordPress ایڈمن میں، Ultimate Multisite > Settings پر جائیں
2. "Domain Mapping" ٹیب پر جائیں
3. نیچے سکرول کر کے "Host Integrations" تک جائیں
4. Closte انضمام کو فعال کریں
5. "Save Changes" پر کلک کریں

## یہ کیسے کام کرتا ہے

جب Ultimate Multisite میں کوئی ڈومین میپ کیا جاتا ہے:

1. انضمام Closte کی API کو ایک درخواست بھیجتا ہے تاکہ ڈومین کو آپ کی ایپلیکیشن میں شامل کیا جا سکے
2. Closte خودکار طور پر SSL سرٹیفکیٹ کی فراہمی کا انتظام کرتا ہے
3. جب کوئی ڈومین میپنگ ہٹائی جاتی ہے تو انضمام اس ڈومین کو Closte سے ہٹا دیتا ہے

یہ انضمام Ultimate Multisite میں DNS چیک وقفہ کی ترتیب کے ساتھ بھی کام کرتا ہے، جو آپ کو یہ ترتیب دینے کی اجازت دیتا ہے کہ سسٹم کتنی بار DNS پروپیگیشن اور SSL سرٹیفکیٹ جاری ہونے کی جانچ کرے۔

## ڈومین ریکارڈ کی تخلیق

یہ انضمام یقینی بناتا ہے کہ جب کوئی سائٹ بنائی یا نقل کی جاتی ہے تو ایک ڈومین ریکارڈ خودکار طور پر بن جاتا ہے۔ یہ Closte انضمام کے لیے خاص طور پر اہم ہے، کیونکہ ڈومین ریکارڈ کی تخلیق Closte API کو ڈومین اور SSL سرٹیفکیٹ بنانے کے لیے متحرک کرتی ہے۔

## مسائل کا حل

### API کنکشن کے مسائل
- تصدیق کریں کہ آپ کی Closte API key درست ہے
- یقینی بنائیں کہ آپ کے Closte اکاؤنٹ کے پاس ضروری اجازتیں ہیں

### SSL سرٹیفکیٹ کے مسائل
- Closte کو SSL سرٹیفکیٹ جاری کرنے میں کچھ وقت لگ سکتا ہے (عام طور پر 5-10 منٹ)
- تصدیق کریں کہ آپ کے ڈومینز آپ کے Closte سرور کے IP ایڈریس کی طرف صحیح طور پر اشارہ کر رہے ہیں
- اپنے ڈومین کے DNS ریکارڈز چیک کریں تاکہ یقینی بنایا جا سکے کہ وہ درست طریقے سے ترتیب دیے گئے ہیں

### ڈومین شامل نہیں ہوا
- کسی بھی خرابی کے پیغامات کے لیے Ultimate Multisite لاگز چیک کریں
- تصدیق کریں کہ ڈومین پہلے سے Closte میں شامل نہیں ہے
- یقینی بنائیں کہ آپ کے ڈومین کے DNS ریکارڈز درست طریقے سے ترتیب دیے گئے ہیں

### DNS چیک وقفہ
- اگر SSL سرٹیفکیٹس جاری ہونے میں بہت زیادہ وقت لگ رہا ہے تو آپ Domain Mapping کی ترتیبات میں DNS چیک وقفہ کو ایڈجسٹ کر سکتے ہیں
- پہلے سے طے شدہ وقفہ 300 سیکنڈ (5 منٹ) ہے، لیکن آپ ٹیسٹنگ کے دوران تیز تر جانچ کے لیے اسے 10 سیکنڈ تک کم کر سکتے ہیں
