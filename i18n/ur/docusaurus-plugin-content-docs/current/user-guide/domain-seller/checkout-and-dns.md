# چیک آؤٹ فیلڈ اور کسٹمر DNS انتظام

## ڈومین انتخاب کا چیک آؤٹ فیلڈ {#the-domain-selection-checkout-field}

**ڈومین انتخاب** فیلڈ ایک چیک آؤٹ عنصر ہے جو صارفین کو یہ انتخاب دیتا ہے کہ وہ اپنی سائٹ کا ڈومین کیسے حاصل کریں۔ ڈومین فروخت فعال کرنے کے لیے اسے کسی بھی چیک آؤٹ فارم میں شامل کریں۔

### چیک آؤٹ فارم میں فیلڈ شامل کرنا {#adding-the-field-to-a-checkout-form}

1. **Network Admin › Ultimate Multisite › چیک آؤٹ فارمز** پر جائیں
2. چیک آؤٹ فارم کھولیں یا بنائیں
3. چیک آؤٹ ایڈیٹر میں، **فیلڈ شامل کریں** پر کلک کریں
4. فیلڈ فہرست سے **ڈومین انتخاب** منتخب کریں
5. فیلڈ اختیارات ترتیب دیں (نیچے دیکھیں)
6. فارم محفوظ کریں

### فیلڈ اختیارات {#field-options}

**ڈومین موڈز** — منتخب کریں کہ کسٹمر کون سے ٹیبز دیکھتا ہے۔ ہر موڈ کو الگ سے فعال یا غیر فعال کیا جا سکتا ہے:

| موڈ | یہ کیا کرتا ہے |
|---|---|
| **سب ڈومین** | کسٹمر آپ کے نیٹ ورک پر مفت سب ڈومین استعمال کرتا ہے (مثلاً، `mysite.yournetwork.com`)۔ ادائیگی کی ضرورت نہیں۔ |
| **نیا ڈومین رجسٹر کریں** | کسٹمر نئے ڈومین کی تلاش کرتا ہے اور اسے آپ کے ترتیب شدہ فراہم کنندہ کے ذریعے رجسٹر کرتا ہے۔ قیمت کے لیے مطابقت رکھنے والی ڈومین پراڈکٹ استعمال کرتا ہے۔ |
| **موجودہ ڈومین** | کسٹمر ایک ایسا ڈومین میپ کرتا ہے جس کا وہ پہلے سے مالک ہے۔ کوئی رجسٹریشن فیس نہیں۔ ڈومین خودکار طور پر اس کی سائٹ سے میپ ہو جاتا ہے۔ |

**ڈیفالٹ موڈ** — جب تینوں موڈز فعال ہوں، تو کون سا ٹیب پہلے کھلے۔ ڈومین رجسٹریشن کو اختیاری رکھنے کے لیے **سب ڈومین** پر سیٹ کریں، یا خریداریوں کی حوصلہ افزائی کے لیے **نیا ڈومین رجسٹر کریں** پر۔

**ڈومین پراڈکٹ** — اختیاری طور پر اس فیلڈ کو کسی مخصوص ڈومین پراڈکٹ سے پن کریں۔ اگر سیٹ نہ ہو، تو addon خودکار طور پر اس TLD کی بنیاد پر مطابقت رکھنے والی پراڈکٹ منتخب کرتا ہے جسے کسٹمر تلاش کرتا ہے۔

### رجسٹرنٹ رابطہ فیلڈز {#registrant-contact-fields}

جب کسٹمر **نیا ڈومین رجسٹر کریں** ٹیب منتخب کرتا ہے، تو چیک آؤٹ فارم رجسٹرنٹ رابطہ فیلڈز کو inline شامل کرتا ہے:

- پہلا نام / آخری نام
- ای میل پتہ
- پتہ (لائن 1، شہر، ریاست/صوبہ، پوسٹل کوڈ، ملک)
- فون نمبر

یہ تمام رجسٹرارز کے لیے مطلوب ہیں اور رجسٹریشن API کال کیے جانے سے پہلے ان کی توثیق کی جاتی ہے۔ فون نمبرز کو خودکار طور پر رجسٹرارز کے متوقع `+CC.NNN` بین الاقوامی فارمیٹ میں ڈھالا جاتا ہے۔

### خودکار طور پر تیار کردہ سائٹ URL {#auto-generated-site-url}

جب کسٹمر ڈومین رجسٹر یا میپ کرتا ہے، تو سائٹ URL فیلڈ خودکار طور پر منتخب کردہ ڈومین سے پُر ہو جاتی ہے۔ صارفین کو الگ URL فیلڈ بھرنے کی ضرورت نہیں ہوتی۔

### تلاش کا رویہ {#search-behaviour}

- کسٹمر کے ٹائپ کرتے ہی AJAX کے ساتھ ڈومین دستیابی حقیقی وقت میں چیک کی جاتی ہے
- ترجیحی ڈومین دستیاب نہ ہونے پر متبادل TLD تجاویز دکھائی جاتی ہیں
- قیمت براہِ راست حاصل کی جاتی ہے اور واضح طور پر دکھائی جاتی ہے (رجسٹریشن قیمت، تجدید قیمت، اختیاری WHOIS رازداری فیس)
- کوپن کوڈز ڈومین پراڈکٹس پر اسی طرح لاگو ہوتے ہیں جیسے کسی بھی دوسری پراڈکٹ پر

**تلاش کی جوابی رفتار بہتر بنانا:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ڈومین تلاش فارم میں حسبِ ضرورت فیلڈز شامل کرنا:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## کسٹمر DNS انتظام {#customer-dns-management}

کسٹمر اپنے رجسٹرڈ ڈومینز کے DNS ریکارڈز **My Account** صفحے سے، اپنے ڈومین کے اندراج کے تحت، منظم کر سکتے ہیں۔

### معاون ریکارڈ اقسام {#supported-record-types}

| قسم | استعمال |
|---|---|
| **A** | میزبان نام کو IPv4 پتے سے میپ کریں |
| **AAAA** | میزبان نام کو IPv6 پتے سے میپ کریں |
| **CNAME** | کسی دوسرے میزبان نام کی طرف اشارہ کرنے والا alias بنائیں |
| **MX** | میل ایکسچینج سرور سیٹ کریں |
| **TXT** | SPF، DMARC، توثیق، یا دیگر متنی ریکارڈز شامل کریں |

### کون سے فراہم کنندگان DNS انتظام کی حمایت کرتے ہیں؟ {#which-providers-support-dns-management}

DNS انتظام (ریکارڈز شامل کرنا، ترمیم کرنا، حذف کرنا) **OpenSRS**، **ResellerClub**، **Enom**، **HostAfrica**، اور **Openprovider** کے ساتھ دستیاب ہے۔ **Hostinger** ڈومینز Domain Seller کے ذریعے nameservers اپ ڈیٹ کر سکتے ہیں؛ hosted ڈومینز کے DNS ریکارڈز core Hostinger ڈومین میپنگ integration کے ذریعے منظم کیے جاتے ہیں۔ Namecheap، GoDaddy، اور NameSilo ڈومینز اسٹیٹس اور میعاد ختم ہونے کی معلومات دکھاتے ہیں لیکن DNS کو براہِ راست رجسٹرار کے کنٹرول پینل میں منظم کرنا ضروری ہے۔

### ڈیفالٹ DNS ریکارڈز {#default-dns-records}

آپ ڈیفالٹ DNS ریکارڈز ترتیب دے سکتے ہیں جو ڈومین رجسٹر ہونے پر خودکار طور پر لاگو ہوتے ہیں۔ **Settings › Domain Seller › Default DNS Records** پر جائیں۔

ڈیفالٹ ریکارڈ اقدار دو tokens کی حمایت کرتی ہیں:

| Token | جس سے تبدیل کیا جائے گا |
|---|---|
| `{DOMAIN}` | رجسٹرڈ ڈومین نام (مثلاً، `example.com`) |
| `{SITE_URL}` | کسٹمر کی سائٹ کے لیے WordPress سائٹ URL |

**مثال — apex ڈومین اور www کو اپنے سرور IP کی طرف اشارہ کرائیں:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ایڈمن: DNS دیکھنا اور ترمیم کرنا {#admin-viewing-and-editing-dns}

Network admins کسی بھی کسٹمر ڈومین کے DNS ریکارڈز کو ڈومین کے ترمیمی صفحے سے **Network Admin › Ultimate Multisite › Domains** میں دیکھ اور ترمیم کر سکتے ہیں۔
