# چیک آؤٹ فیلڈ اور کسٹمر DNS مینجمنٹ

## ڈومین سلیکشن چیک آؤٹ فیلڈ

**Domain Selection** فیلڈ ایک چیک آؤٹ ایلیمنٹ ہے جو کسٹمر کو یہ انتخاب دیتا ہے کہ وہ اپنے سائٹ کا ڈومین کیسے حاصل کرنا چاہتے ہیں۔ ڈومین بیچنے کی سہولت شامل کرنے کے لیے اسے کسی بھی چیک آؤٹ فارم میں شامل کریں۔

### چیک آؤٹ فارم میں فیلڈ شامل کرنا

1. **Network Admin › Ultimate Multisite › Checkout Forms** پر جائیں
2. ایک چیک آؤٹ فارم کھولیں یا بنائیں
3. چیک آؤٹ ایڈیٹر میں، **Add Field** پر کلک کریں
4. فیلڈ کی فہرست میں سے **Domain Selection** کا انتخاب کریں
5. فیلڈ کے آپشنز ترتیب دیں (نیچے دیکھیں)
6. فارم کو محفوظ (Save) کریں

### فیلڈ آپشنز

**Domain modes** — وہ ٹیبز منتخب کریں جو کسٹمر کو نظر آئیں گے۔ ہر موڈ کو آزادانہ طور پر فعال یا غیر فعال کیا جا سکتا ہے:

| Mode | یہ کیا کرتا ہے |
|---|---|
| **Subdomain** | کسٹمر آپ کے نیٹ ورک پر ایک مفت سب ڈومین استعمال کرتا ہے (مثلاً، `mysite.yournetwork.com`)۔ ادائیگی کی ضرورت نہیں ہے۔ |
| **Register New Domain** | کسٹمر ایک نیا ڈومین تلاش کرتا ہے اور اسے آپ کے ترتیب کردہ فراہم کنندہ (provider) کے ذریعے رجسٹر کرتا ہے۔ قیمت کے لیے میچنگ ڈومین پروڈکٹ استعمال ہوتی ہے۔ |
| **Existing Domain** | کسٹمر ایک ایسا ڈومین استعمال کرتا ہے جو وہ پہلے سے ملکیت رکھتا ہے۔ کوئی رجسٹریشن فیس نہیں۔ ڈومین خود بخود اس کی سائٹ سے منسلک (map) ہو جاتا ہے۔ |

**Default mode** — جب تینوں موڈز فعال ہوں، تو کون سا ٹیب پہلے کھلے گا۔ ڈومین رجسٹریشن کو اختیاری رکھنے کے لیے اسے **Subdomain** پر سیٹ کریں، یا خریداری کی حوصلہ افزائی کے لیے **Register New Domain** پر سیٹ کریں۔

**Domain product** — اختیاری طور پر اس فیلڈ کو کسی مخصوص ڈومین پروڈکٹ سے جوڑ دیں۔ اگر سیٹ نہ کیا جائے، تو ایڈ آن خود بخود اس پروڈکٹ کا انتخاب کر لے گا جو کسٹمر کی تلاش کے TLD پر مبنی ہو۔

### رجسٹرنٹ رابطہ فیلڈز

جب کوئی کسٹمر **Register New Domain** ٹیب کا انتخاب کرتا ہے، تو چیک آؤٹ فارم میں ان لائن رجسٹرنٹ رابطہ فیلڈز شامل ہو جاتے ہیں:

- فرسٹ نیم / لاسٹ نیم
- ای میل ایڈریس
- پتہ (line 1, شہر، ریاست/صوبہ، پوسٹل کوڈ، ملک)
- فون نمبر

یہ تمام رجسٹرارز کے لیے لازمی ہیں اور رجسٹریشن API کال کیے جانے سے پہلے ان کی جانچ (validated) کی جاتی ہے۔ فون نمبرز کو خود بخود `+CC.NNN` فارمیٹ میں فارمیٹ کیا جاتا ہے جو رجسٹرارز کے ذریعہ متوقع بین الاقوامی فارمیٹ ہوتا ہے۔

### خودکار طور پر تیار کردہ سائٹ URL

جب کوئی کسٹمر ڈومین رجسٹر کرتا ہے یا اسے منسلک (map) کرتا ہے، تو سائٹ URL فیلڈ منتخب کردہ ڈومین سے خود بخود بھر جاتا ہے۔ کسٹمر کو الگ سے URL فیلڈ بھرنے کی ضرورت نہیں ہوتی۔

### تلاش کا رویہ (Search behaviour)

- کسٹمر ٹائپ کرتے وقت AJAX کے ذریعے ڈومین کی دستیابی کی حقیقی وقت میں جانچ کی جاتی ہے۔
- جب پسند کردہ ڈومین دستیاب نہیں ہوتا تو متبادل TLD کی تجاویز دکھائی جاتی ہیں۔
- قیمتیں لائیو حاصل کی جاتی ہیں اور واضح طور پر دکھائی جاتی ہیں (رجسٹریشن کی قیمت، تجدید کی قیمت، اختیاری WHOIS پرائیویسی فیس)۔
- کوپن کوڈز ڈومین پروڈکٹس پر بھی لاگو ہوتے ہیں جیسا کہ کسی دوسرے پروڈکٹ پر ہوتے ہیں۔

**تلاش کی رسپانسیونیس کو بہتر بنانا (Tuning search responsiveness):**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**ڈومین تلاش فارم میں کسٹم فیلڈز شامل کرنا:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## کسٹمر DNS مینجمنٹ

کسٹمرز اپنے رجسٹرڈ ڈومینز کے لیے **My Account** پیج پر، اپنے ڈومین کی انٹری کے تحت، DNS ریکارڈز کو مینج کر سکتے ہیں۔

### سپورٹ شدہ ریکارڈ کی اقسام

| Type | استعمال |
|---|---|
| **A** | ہوسٹ نام کو IPv4 ایڈریس سے جوڑنا (Map) |
| **AAAA** | ہوسٹ نام کو IPv6 ایڈریس سے جوڑنا (Map) |
| **CNAME** | ایک ایسا عرفیت (alias) بنانا جو کسی دوسرے ہوسٹ نام کی طرف اشارہ کرتا ہو |
| **MX** | میل ایکسچینج سرور سیٹ کرنا |
| **TXT** | SPF، DMARC، تصدیق، یا دیگر ٹیکسٹ ریکارڈ شامل کرنا |

### کون سے فراہم کنندہ DNS مینجمنٹ کو سپورٹ کرتے ہیں؟

DNS مینجمنٹ (ریکارڈز شامل کرنا، ایڈٹ کرنا، ڈیلیٹ کرنا) **OpenSRS**, **ResellerClub**, اور **Enom** کے ساتھ دستیاب ہے۔ Namecheap, GoDaddy, اور NameSilo ڈومینز اسٹیٹس اور ایکسپائری کی معلومات دکھاتے ہیں، لیکن DNS کو رجسٹرار کے کنٹرول پینل میں براہ راست مینج کیا جانا چاہیے۔

### ڈیفالٹ DNS ریکارڈز

آپ ڈیفالٹ DNS ریکارڈز ترتیب دے سکتے ہیں جو ڈومین رجسٹر ہونے پر خود بخود لاگو ہو جاتے ہیں۔ **Settings › Domain Seller › Default DNS Records** پر جائیں۔

ڈیفالٹ ریکارڈ ویلیوز دو ٹوکنز کو سپورٹ کرتی ہیں:

| Token | کس سے تبدیل ہوگا |
|---|---|
| `{DOMAIN}` | رجسٹرڈ ڈومین کا نام (مثلاً، `example.com`) |
| `{SITE_URL}` | کسٹمر کی سائٹ کے لیے WordPress سائٹ کا URL |

**مثال — ایپیکس ڈومین اور www کو آپ کے سرور IP کی طرف پوائنٹ کرنا:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### ایڈمن: DNS دیکھنا اور ایڈٹ کرنا

نیٹ ورک ایڈمنز کو **Network Admin › Ultimate Multisite › Domains** میں ڈومین کی ایڈیٹ پیج سے کسی بھی کسٹمر ڈومین کے DNS ریکارڈز دیکھنے اور ایڈٹ کرنے کی سہولت حاصل ہے۔
