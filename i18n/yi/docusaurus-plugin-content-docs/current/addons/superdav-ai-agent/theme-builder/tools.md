# געצייג און קאָנפֿיגוראַציע

Theme Builder ניצט אַ זאַמלונג געצייג צו אַנאַליזירן, דיזײַנען און בויען אײַער WordPress וועבזײַטל. אין ווערסיע 1.16.0, **sd-ai-agent/site-scrape איז איצט אַ Tier 1 געצייג**, וואָס מאַכט עס בנימצא לויטן פֿעליקייט אין Theme Builder. Superdav AI Agent v1.18.0 פֿירט אויך אַרײַן אַ באַזונדער פֿאַרשפּרייטן Advanced באַגלייט־פּעקל פֿאַר פֿאַרטרויענסווערטע אַנטוויקלער־אַרבעטסלויפֿן; יענע געצייג זענען נישט טייל פֿון דעם WordPress.org קערן־פּעקל און מוזן אינסטאַלירט און דערלויבט ווערן באַזונדער.

## בנימצא געצייג

### Tier 1 געצייג (שטענדיק בנימצא)

Tier 1 געצייג זענען בנימצא לויטן פֿעליקייט אין Theme Builder אָן קיין נאָך־קאָנפֿיגוראַציע.

#### sd-ai-agent/site-scrape

**ציל:** אַנאַליזירן עקזיסטירנדיקע וועבזײַטן כּדי אַרויסצוציען דיזײַן־אינספּיראַציע, אינהאַלט־סטרוקטור און פֿונקציאָנאַליטעט.

**פֿעיִקייטן:**

- **וועבזײַט־אַנאַליז** — אָפּקראַצן און אַנאַליזירן קאָנקורענטישע אָדער אינספּיראַציע־וועבזײַטן
- **דיזײַן־אויסציען** — דערקענען פֿאַרבן, שריפֿטן און אויסלייג־מוסטערן
- **אינהאַלט־סטרוקטור** — פֿאַרשטיין זײַט־אָרגאַניזאַציע און היראַרכיע
- **פֿונקציע־דערקענונג** — דערקענען plugins און פֿונקציאָנאַליטעט
- **פֿאָרשטעלונג־אַנאַליז** — קאָנטראָלירן זײַט־גיכקייט און אָפּטימיזאַציע
- **SEO אַנאַליז** — איבערקוקן מעטאַ־טאַגס און סטרוקטורירטע דאַטן

**באַניץ:**

```
אַנאַליזיר דעם דיזײַן פֿון example.com און ניצ עס ווי אינספּיראַציע פֿאַר מײַן וועבזײַטל.
```

**וואָס עס ציט אַרויס:**

- פֿאַרב־פּאַלעט און פֿאַרב־באַניץ
- טיפּאָגראַפֿיע (שריפֿטן און גרייסן)
- אויסלייג־סטרוקטור און אָפּשטאַנדן
- נאַוויגאַציע־מוסטערן
- אינהאַלט־אָרגאַניזאַציע
- בילדער און מעדיע־באַניץ
- אינטעראַקטיווע עלעמענטן
- מאָביל־רעספּאָנסיווקייט

**באַגרענעצונגען:**

- קען נישט אָפּקראַצן פּאַראָל־באַשיצטע וועבזײַטן
- רעספּעקטירט robots.txt און וועבזײַטל־פּאָליטיקן
- קען אפֿשר נישט כאַפּן דינאַמישן אינהאַלט
- JavaScript־שווערע וועבזײַטן קענען האָבן באַגרענעצטע אַנאַליז
- גרויסע וועבזײַטן קענען נעמען לענגער צו אַנאַליזירן

### Tier 2 געצייג (אָפּציאָנעל)

Tier 2 געצייג זענען בנימצא ווען זיי ווערן בפֿירוש אײַנגעשלאָסן אין Theme Builder אײַנשטעלונגען.

#### פֿאָרטגעשריטענע אַנאַליטיק

שטעלט צו פּרטימדיקע פֿאָרשטעלונג־מעטריקן:

- זײַט־לאָדן־צײַטן
- Core Web Vitals
- SEO צאָל
- צוטריטלעכקייט־צאָל
- בעסטע־פּראַקטיקעס־צאָל

#### אינהאַלט־אָפּטימיזירער

אַנאַליזירט און פֿאָרשלאָגט פֿאַרבעסערונגען פֿאַר:

- לייענבאַרקייט
- SEO אָפּטימיזאַציע
- שליסלוואָרט־באַניץ
- אינהאַלט־סטרוקטור
- אָרטירונג פֿון רוף־צו־אַקציע

### Tier 3 געצייג (פּרעמיום)

Tier 3 געצייג דאַרפֿן נאָך־קאָנפֿיגוראַציע אָדער פּרעמיום־פֿעיִקייטן.

#### AI אינהאַלט־גענעראַטאָר

שאַפֿט אינהאַלט פֿאַר:

- פּראָדוקט־באַשרײַבונגען
- דינסט־זײַטן
- בלאָג־אַרטיקלען
- מעטאַ־באַשרײַבונגען
- רוף־צו־אַקציע טעקסט

#### פֿאָרטגעשריטענע דיזײַן־געצייג

שטעלט צו פֿאָרטגעשריטענע דיזײַן־פֿעיִקייטן:

- אייגענע CSS שאַפֿונג
- אַנימאַציע־שאַפֿונג
- אינטעראַקטיווע עלעמענט־דיזײַן
- פֿאָרטגעשריטענע פֿאַרב־טעאָריע
- טיפּאָגראַפֿיע־אָפּטימיזאַציע

### פֿאָרטגעשריטענע באַגלייט־געצייג (באַזונדער פּעקל)

דאָס Advanced באַגלייט־פּעקל ווערט פֿאַרשפּרייט באַזונדער פֿון דער קערן־אַרויסגאַבע פֿון Superdav AI Agent. עס איז געמיינט פֿאַר פֿאַרטרויענסווערטע אַנטוויקלער־און וועבזײַטל־באַזיצער־סבֿיבֿות, וווּ אַדמיניסטראַטאָרן דערלויבן בפֿירוש דעם agent צו נוצן געצייג מיט גרויסער השפּעה.

Advanced באַגלייט־פֿעיִקייטן קענען אַרײַננעמען:

- **טעקע־סיסטעם־געצייג** — דורכקוקן און ענדערן באשטעטיקטע טעקעס אין פֿאַרטרויענסווערטע סבֿיבֿות
- **דאַטאַבאַזע־געצייג** — אָנפֿרעגן אָדער דערהײַנטיקן וועבזײַטל־דאַטן ווען בפֿירוש דערלויבט
- **WP-CLI געצייג** — לויפֿן WordPress וישאַלט־און דורכקוק־באַפֿעלן
- **REST פֿאַרשיקער־געצייג** — רופֿן רעגיסטרירטע REST endpoints דורך קאָנטראָלירטע פֿאַרשיק־אַרבעטסלויפֿן
- **plugin בויער־געצייג** — אויסשטעלן אַ ראַם און איבעראַרבעטן plugin קאָד
- **Git מאָמענטבילד־געצייג** — שאַפֿן מאָמענטבילדער פֿאַר ריזיקאַנטע אַנטוויקלער־אָפּעראַציעס
- **באַניצער־פֿאַרוואַלטונג־געצייג** — העלפֿן איבערקוקן אָדער צופּאַסן באַניצער־רעקאָרדס וווּ דערלויבט
- **בענטשמאַרק־געצייג** — לויפֿן פֿאָרשטעלונג־אָדער פֿעיִקייט־בענטשמאַרקס פֿאַר אַנטוויקלונג־איבערקוק

דאָקומענטירט זיי נישט ווי אַלגעמיין בנימצא Theme Builder געצייג פֿאַר יעדער אינסטאַלאַציע. זיי זענען בנימצא נאָר ווען דאָס Advanced באַגלייט־פּעקל איז אינסטאַלירט, אײַנגעשלאָסן און באַגרענעצט צו פֿאַרטרויענסווערטע אַדמיניסטראַטאָרן.

## געצייג־קאָנפֿיגוראַציע

### אײַנשליסן געצייג

כּדי אײַנצושליסן נאָך געצייג אין Theme Builder:

1. נאַוויגירט צו **Gratis AI Agent → אײַנשטעלונגען**
2. גייט צו **Theme Builder → געצייג**
3. שלט געצייג אָן/אויס לויטן באַדערף
4. היט די אײַנשטעלונגען

### געצייג־דערלויבענישן

עטלעכע געצייג דאַרפֿן דערלויבענישן:

- **וועבזײַטל־אָפּקראַצן** — דאַרף אינטערנעץ־צוטריט
- **אַנאַליטיק** — דאַרף Google Analytics אינטעגראַציע
- **אינהאַלט־שאַפֿונג** — דאַרף API שליסלען
- **פֿאָרטגעשריטענע פֿעיִקייטן** — קען דאַרפֿן אַ פּרעמיום subscription
- **פֿאָרטגעשריטענע באַגלייט־געצייג** — דאַרפֿן דאָס באַזונדער פֿאַרשפּרייטע Advanced פּעקל און בפֿירושן אַדמיניסטראַטאָר־צוטרוי

### API שליסלען און צוטריט־דאַטן

קאָנפֿיגורירט API שליסלען פֿאַר געצייג וואָס דאַרפֿן זיי:

1. גייט צו **Gratis AI Agent → אײַנשטעלונגען → API שליסלען**
2. גיט אַרײַן צוטריט־דאַטן פֿאַר יעדן געצייג
3. פּרוּווט די פֿאַרבינדונג
4. היט זיכער

### אינסטאַלירן דאָס Advanced באַגלייט־פּעקל

דאָס Advanced באַגלייט־פּעקל ווערט אַרויסגעלאָזט באַזונדער פֿון דעם WordPress.org קערן־ZIP. אינסטאַלירט עס נאָר פֿון דעם אָפֿיציעלן פּראָיעקט־פֿאַרשפּרייטונג־קאַנאַל פֿאַר דער אַרויסגאַבע, דערנאָך באַגרענעצט צוטריט צו פֿאַרטרויענסווערטע אַדמיניסטראַטאָרן. קוקט איבער מענטשלעכע באשטעטיקונג־טויערן און git מאָמענטבילד־פֿירונג איידער איר שליסט אײַן טעקע־סיסטעם, דאַטאַבאַזע, WP-CLI, REST פֿאַרשיקער, plugin בויער, באַניצער־פֿאַרוואַלטונג, אָדער בענטשמאַרק־געצייג אויף אַ פּראָדוקציע־וועבזײַטל.

## ניצן sd-ai-agent/site-scrape

### גרונט־באַניץ

בעט Theme Builder צו אַנאַליזירן אַ וועבזײַט:

```
אַנאַליזיר דעם דיזײַן פֿון מײַנס אַ קאָנקורענטס וועבזײַטל אויף competitor.com
און פֿאָרשלאָג דיזײַן־פֿאַרבעסערונגען פֿאַר מײַן וועבזײַטל.
```

### ספּעציפֿישע אַנאַליז

בעטן ספּעציפֿישע מינים אַנאַליז:

```
צי אַרויס די פֿאַרב־פּאַלעט פֿון example.com און ניצ עס ווי אינספּיראַציע.
```

```
אַנאַליזיר די נאַוויגאַציע־סטרוקטור פֿון example.com און לייג אָן ענלעכע
אָרגאַניזאַציע אויף מײַן וועבזײַטל.
```

```
קאָנטראָליר די מאָביל־רעספּאָנסיווקייט פֿון example.com און מאַך זיכער אַז מײַן וועבזײַטל
איז גלײַך אַזוי רעספּאָנסיוו.
```

### פֿאַרגלײַך־אַנאַליז

פֿאַרגלײַכט עטלעכע וועבזײַטן:

```
פֿאַרגלײַך די דיזײַנען פֿון site1.com און site2.com און שאַף אַ דיזײַן
וואָס פֿאַרבינדט די בעסטע עלעמענטן פֿון ביידע.
```

## געצייג־באַגרענעצונגען און באַטראַכטונגען

### קורס־באַגרענעצונג

- Scraping איז באַגרענעצט לויט קורס, כּדי צו פֿאַרמיידן איבערלאַסטונג פֿונעם סערווער
- קייפּלדיקע בקשות צו דער זעלביקער וועבזײַט קענען ווערן פֿאַרפּאַמעלעכט
- גרויסע וועבזײַטן קענען נעמען לענגער צו אַנאַליזירן

### פּינקטלעכקייט פֿון אינהאַלט

- דינאַמישער אינהאַלט קען נישט ווערן גאָר אַרײַנגעכאַפּט
- אינהאַלט וואָס ווערט אויסגעפֿירט דורך JavaScript קען זײַן נישט גאַנץ
- רעאַל־צײַט־דאַטן קען נישט ווערן אָפּגעשפּיגלט
- טייל אינהאַלט קען זײַן הינטער צאָל־ווענט

### לעגאַלע און עטישע באַטראַכטונגען

- רעספּעקטירט robots.txt און פּאָליטיקעס פֿון וועבזײַטן
- זאָלט נישט Scrape־ן אינהאַלט מיט דרוקרעכט פֿאַר ווידער־פּראָדוקציע
- ניצט אַנאַליז פֿאַר אינספּיראַציע, נישט פֿאַר קאָפּירן
- באַשטעטיקט אַז איר האָט רעכטן צו ניצן דעם אַרויסגענומענעם אינהאַלט
- פֿאָלגט די באַדינגונגען פֿון סערוויס פֿון אַנאַליזירטע וועבזײַטן

### השפּעה אויף פאָרשטעלונג

- Scraping פֿון גרויסע וועבזײַטן קען נעמען צײַט
- קייפּלדיקע גלײַכצײַטיקע Scrapes קענען זײַן פּאַמעלעכער
- נעץ־פֿאַרבינדונג ווירקט אויף גיכקייט
- גרויסע דאַטן־זאַמלונגען קענען דאַרפֿן מער פּראָצעסירונג

## בעסטע פּראַקטיקן

### ניצן וועבזײַט־אַנאַליז

1. **אַנאַליזירט קייפּלדיקע וועבזײַטן** — זאַמלט אינספּיראַציע פֿון עטלעכע קוואַלן
2. **פֿאָקוסירט זיך אויף סטרוקטור** — לערנט אויסלייג־ און אָרגאַניזאַציע־מוסטערן
3. **ציעט אַרויס פֿאַרבן** — ניצט פֿאַרב־פּאַלעטן ווי אָנהייב־פּונקטן
4. **שטודירט טיפאָגראַפֿיע** — דערקענט שריפֿט־קאָמבינאַציעס וואָס איר האָט ליב
5. **איבערקוקט נאַוויגאַציע** — פֿאַרשטייט מעניו־אָרגאַניזאַציע

### עטישע Scraping

1. **רעספּעקטירט robots.txt** — פֿאָלגט וועבזײַט־הנחיות
2. **קאָפּירט נישט קיין אינהאַלט** — ניצט אַנאַליז נאָר פֿאַר אינספּיראַציע
3. **באַשטעטיקט רעכטן** — מאַכט זיכער אַז איר קענט ניצן אַרויסגענומענעם אינהאַלט
4. **צייכנט אָן קוואַלן** — גיט קרעדיט צו אינספּיראַציע־קוואַלן
5. **פֿאָלגט באַדינגונגען** — האַלטט זיך צו די באַדינגונגען פֿון סערוויס פֿון דער וועבזײַט

### מאַקסימיזירן עפעקטיווקייט פֿון מכשירים

1. **זײַט ספּעציפֿיש** — בעט ספּעציפֿישע טיפּן אַנאַליז
2. **גיט קאָנטעקסט** — דערקלערט דעם ציל פֿון אייער וועבזײַט
3. **שטעלט ערוואַרטונגען** — באַשרײַבט אייערע פּלאַן־צילן
4. **איטערירט** — פֿאַרפֿײַנערט לויט די רעזולטאַטן
5. **קאָמבינירט מכשירים** — ניצט קייפּלדיקע מכשירים פֿאַר אַ פולשטענדיקן אַנאַליז

## פּראָבלעמען־לייזונג

### וועבזײַט קען נישט ווערן Scrape־ד

- קאָנטראָלירט צי די וועבזײַט איז עפֿנטלעך צוטריטלעך
- באַשטעטיקט אַז robots.txt דערלויבט Scraping
- פּרוּווט אַן אַנדערע וועבזײַט
- קאָנטראָלירט אינטערנעץ־פֿאַרבינדונג
- קאָנטאַקטירט שטיצע אויב דער פּראָבלעם בלײַבט ווײַטער

### אומפֿולשטענדיקער אַנאַליז

- די וועבזײַט קען האָבן דינאַמישן אינהאַלט
- JavaScript קען נישט ווערן גאַנץ אויסגעפֿירט
- גרויסע וועבזײַטן קענען איבערשרײַטן צײַט־גרענעץ
- פּרוּווט אַנאַליזירן ספּעציפֿישע בלעטער אַנשטאָט
- בעט ספּעציפֿישע טיפּן אַנאַליז

### פּאַמעלעכע פאָרשטעלונג

- גרויסע וועבזײַטן נעמען לענגער צו אַנאַליזירן
- קייפּלדיקע גלײַכצײַטיקע Scrapes זײַנען פּאַמעלעכער
- נעץ־פֿאַרבינדונג ווירקט אויף גיכקייט
- פּרוּווט אין שעהן מחוץ שפּיץ־צײַט
- אַנאַליזירט קלענערע אָפּטיילונגען ערשט
