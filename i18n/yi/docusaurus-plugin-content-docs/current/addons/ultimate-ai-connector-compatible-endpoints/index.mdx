# Ultimate AI Connector פֿאַר קאָמפּאַטיבלע ענדפּוינטן

דער Ultimate AI Connector פֿאַר קאָמפּאַטיבלע ענדפּוינטן אינטעגרירט OpenAI-קאָמפּאַטיבלע AI פּראַוויידער APIs מיט אײַער WordPress מולטיסייט־נעץ. קאָנפֿיגורירט עטלעכע AI פּראַוויידערס — יעדער איינער מיט זײַן אייגענעם ענדפּוינט־URL, API שליסל, און פעליקייט־מאָדעל — און רוטירט בקשות מיט אויטאָמאַטישן צוריקפֿאַל ווען אַ פּראַוויידער איז נישט בנימצא.

## הויפּט־אייגנשאַפֿטן

- **שטיצע פֿאַר עטלעכע פּראַוויידערס**: לייגט צו עטלעכע AI פּראַוויידער־קאַרטלעך, יעדער מיט זײַן אייגענעם ענדפּוינט־URL, API שליסל, און פעליקייט־מאָדעל
- **ציען־צו־איבעראָרדענען**: אָרדנט פּראַוויידערס לויט פּריאָריטעט דורך ניצן דעם גריפּ־הענטל אויף יעדן קאַרטל; בקשות נוצן פּראַוויידערס לויטן סדר
- **אויטאָמאַטישע צוריקפֿאַל־רוטירונג**: אויב דער הויפּט־פּראַוויידער איז נישט בנימצא, פֿאַלן בקשות אויטאָמאַטיש צוריק צום קומענדיקן קאָנפֿיגורירטן פּראַוויידער
- **קאָנפֿיגוראַציע פּער פּראַוויידער**: יעדער פּראַוויידער ווערט אומאָפּהענגיק קאָנפֿיגורירט מיט זײַנע אייגענע קרעדענציאַלן און מאָדעל־אויסקלייב
- **קאַרטל־באַזירטע אײַנשטעלונגען־UI**: געבויט אויף סטאַבילע WordPress 6.9+ Card קאָמפּאָנענטן (CardBody, CardHeader, CardDivider)
- **אויטאָ־אויסברייטערן בײַם צולייגן**: נײַע פּראַוויידער־קאַרטלעך ברייטן זיך אויס אויטאָמאַטיש כּדי איר זאָלט קענען גלײַך אַרײַנפֿילן די פּרטים
- **סקריפּט־קאַש אויפֿרײַסן**: פּראַוויידער־סקריפּטן ווערן קאַש־אויפֿגעריסן בײַ אַ פּלוגין־דערהײַנטיקונג כּדי צו פֿאַרזיכערן אַז די לעצטע ווערסיע ווערט געלאָדן

## פֿאָדערונגען

- WordPress 5.3 אָדער העכער
- PHP 7.4 אָדער העכער
- Ultimate Multisite פּלוגין (אַקטיוו)
- אַן OpenAI-קאָמפּאַטיבלע API ענדפּוינט (אַליין־געהאָסט אָדער פֿון אַ דריטן צד)

## אינסטאַלאַציע

1. אַרויפֿלאָדן די אַדאָן־טעקעס אין אײַער `/wp-content/plugins/` וועגווײַזער
2. אַקטיווירן דעם פּלוגין דורך דעם 'Plugins' מעניו אין WordPress
3. גיין צו **Ultimate Multisite → AI Connector** כּדי צו קאָנפֿיגורירן דעם אַדאָן

## קאָנפֿיגוראַציע

### אויפֿשטעל פֿאַר עטלעכע פּראַוויידערס

דער קאָנפֿיגוראַציע־עקראַן ניצט אַ קאַרטל־באַזירטן אויסלייג, וווּ יעדער קאַרטל רעפּרעזענטירט איין AI פּראַוויידער. איר קענט צולייגן אַזוי פֿיל פּראַוויידערס ווי איר דאַרפֿט.

#### צולייגן אַ פּראַוויידער

1. גייט צו **Ultimate Multisite → AI Connector** אין דער נעץ־אַדמיניסטראַציע
2. קוועטשט **צולייגן פּראַוויידער**
3. אַ נײַער פּראַוויידער־קאַרטל באַווײַזט זיך און ברייטערט זיך אויטאָמאַטיש אויס
4. פֿילט אַרײַן די פּראַוויידער־פּרטים:
   - **פּראַוויידער־נאָמען** — אַן עטיקעט צו אידענטיפֿיצירן דעם פּראַוויידער (למשל "OpenAI", "Mistral", "Local Ollama")
   - **ענדפּוינט־URL** — דער באַזע־URL פֿון דער OpenAI-קאָמפּאַטיבלע API (למשל `https://api.openai.com/v1`)
   - **API שליסל** — אײַער API שליסל פֿאַר דעם פּראַוויידער
   - **פעליקייט־מאָדעל** — דער מאָדעל־אידענטיפֿיקאַטאָר צו נוצן לויט פעליקייט פֿאַר דעם פּראַוויידער (למשל `gpt-4o`, `mistral-small`)
5. קוועטשט **אויפֿהיטן אײַנשטעלונגען**

#### אַוועקנעמען אַ פּראַוויידער

קוועטשט דעם **אַוועקנעמען** קנעפּל אויף יעדן פּראַוויידער־קאַרטל כּדי אים צו אויסמעקן. היט אויף די אײַנשטעלונגען כּדי אָנווענדן די ענדערונג.

#### איבעראָרדענען פּראַוויידערס

שלעפּט דעם גריפּ־אייקאָן (⣿) אויף דער לינקער זײַט פֿון יעדן פּראַוויידער־קאַרטל כּדי אים איבערצואָרדענען. דער סדר באַשטימט די צוריקפֿאַל־פּריאָריטעט: בקשות ווערן געשיקט צום ערשטן פּראַוויידער און פֿאַלן צוריק צו ווײַטערדיקע פּראַוויידערס אויב אַ בקשה דורכפֿאַלט.

### צוריקפֿאַל־רוטירונג

ווען אַ בקשה צום הויפּט־פּראַוויידער דורכפֿאַלט — צוליב אַ צייט־אויסלויף, ראַטע־גרענעץ, אָדער דינסט־אויספֿאַל — פּרוּווט דער קאָנעקטאָר אויטאָמאַטיש ווידער מיטן קומענדיקן פּראַוויידער אין דער רשימה. דאָס גייט ווײַטער אַראָפּ אין דער רשימה ביז אַ פּראַוויידער איז מצליח אָדער אַלע פּראַוויידערס זענען שוין פּרוּווט געוואָרן.

| סצענאַר | אויפֿפֿירונג |
|----------|-----------|
| הויפּט־פּראַוויידער ענטפֿערט | די בקשה ווערט נאָרמאַל באַדינט |
| הויפּט־פּראַוויידער דורכפֿאַלט | די בקשה ווערט ווידער פּרוּווט מיטן קומענדיקן פּראַוויידער |
| אַלע פּראַוויידערס דורכפֿאַלן | אַ פֿעלער ווערט צוריקגעגעבן צום רופֿער |

צוריקפֿאַל איז טראַנספּאַרענט פֿאַרן רופֿנדיקן פּלוגין אָדער טעמע — זיי באַקומען אָדער אַ מצליחדיקן ענטפֿער אָדער אַ לעצטן פֿעלער, אומאָפּהענגיק פֿון וויפֿל פּראַוויידערס מע האָט פּרוּווט.

## פּראָבלעם־לייזונג

### קיין פּראַוויידערס ווערן נישט אויסגערעכנט

אויב די פּראַוויידער־רשימה איז ליידיק נאָך אויפֿהיטן, באַשטעטיקט אַז:

- איר האָט געקוועטשט **אויפֿהיטן אײַנשטעלונגען** נאָך צולייגן פּראַוויידערס
- אײַער בלעטערער האָט נישט בלאָקירט די אײַנשטעלונגען־בלאַט פֿון אויפֿהיטן (קוקט אין דער בלעטערער־קאָנסאָל פֿאַר JavaScript פֿעלערן)
- דער פּלוגין איז אַקטיוו און איר זענט אויפֿן נעץ־אַדמיניסטראַציע־עקראַן

### בקשות דורכפֿאַלן פֿאַר אַלע פּראַוויידערס

- באַשטעטיקט אַז יעדנס פּראַוויידערס ענדפּוינט־URL איז דערגרייכלעך פֿון אײַער סערווער (נישט נאָר פֿון אײַער בלעטערער)
- קאָנטראָלירט אַז יעדער API שליסל איז גילטיק און נישט אויסגעלאָפֿן
- איבערקוקט דעם פּראַוויידערס ראַטע־גרענעץ און קוואָטע־סטאַטוס אין זייער Dashboard
- אַקטיווירט WordPress דעבאַג־לאָגירונג (`WP_DEBUG_LOG`) כּדי צו כאַפּן דעטאַלירטע פֿעלער־מעלדונגען פֿון דעם קאָנעקטאָר

### אַ פּראַוויידער ווערט נישט געניצט פֿאַר צוריקפֿאַל

ציען־צו־איבעראָרדענען ענדערט דעם צוריקפֿאַל־סדר. באַשטעטיקט אַז דער פּראַוויידער וואָס איר דערוואַרט זאָל באַקומען צוריקפֿאַל־בקשות איז ריכטיק פּאָזיציאָנירט אין דער רשימה, און אַז זײַן ענדפּוינט־URL און API שליסל זענען ריכטיק.

### נײַער פּראַוויידער־קאַרטל ברייטערט זיך נישט אויס אויטאָמאַטיש

אויטאָ־אויסברייטערן פֿאָדערט אַז JavaScript זאָל זײַן אַקטיווירט אין אײַער בלעטערער. אויב דער קאַרטל ברייטערט זיך נישט אויס בײַם קוועטשן, קוקט אין דער בלעטערער־קאָנסאָל פֿאַר פֿעלערן. די קאַרטל־אינהאַלטן זענען שטענדיק צוטריטלעך דורך קוועטשן דעם קאַרטל־קעפּל, אַפֿילו אויב אויטאָ־אויסברייטערן ווערט נישט אויסגעלייזט.
