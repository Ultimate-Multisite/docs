# סאַוורענע מיגראַציע־וועריפֿיקאַציע

Ultimate Multisite: Multi-Tenancy 1.2.0 נעמט אַרײַן WP-CLI וועריפֿיקאַציע־באַפֿעלן פֿאַר סאַוורענע טענאַנט־מיגראַציעס. ניצט זיי ווען אַ טענאַנט־מיגראַציע, SSO־באַזוך, אָדער איזאָלירטע אינסטאַלאַציע פֿירט זיך נישט ווי דערוואַרט.

## באַפֿעלן צו לויפֿן

לויפֿט וועריפֿיקאַציע פֿון דער נעץ־WordPress אינסטאַלאַציע:

```bash
wp tenant verify-no-legacy --site=<site-id>
wp tenant verify-sovereign-push --site=<site-id>
```

ניצט דעם פּלאַץ־ID פֿאַר דעם טענאַנט וואָס איר מיגרירט. דער ערשטער באַפֿעל קאָנטראָלירט אַז דער טענאַנט איז מער נישט אָפענגיק אויף אַלטע נעץ־זײַטיקע דאַטן. דער צווייטער באַפֿעל באַשטעטיקט אַז סאַוורענע push־אַרבעטן קענען פּראָצעסירן און זיך אויסליידיקן.

## געוויינטלעכע דורכפֿאַלן

### דאַטאַבאַזע־דערלויבונגען שטימען נישט מיטן host

אויב די וועריפֿיקאַציע מעלדט דורכפֿאַלן פֿון דערלויבונגען אָדער writer-user, קאָנטראָלירט דעם קאָנפֿיגורירטן דאַטאַבאַזע־host. `localhost`, `127.0.0.1`, און אַ קאַנטיינער־דינסט־נאָמען זענען אַנדערע MySQL דערלויבונג־hosts. דערהײַנטיקט דעם טענאַנט־host־פֿאַרבינד אָדער די דאַטאַבאַזע־דערלויבונגען, און דערנאָך לויפֿט ווידער די וועריפֿיקאַציע.

### Bedrock אָדער לאָקאַלע אינסטאַלאַציעס קענען זיך נישט פֿאַרבינדן

Bedrock און לאָקאַלע socket־אינסטאַלאַציעס קענען מעלדן די דאַטאַבאַזע ווי `localhost`, בשעת דער runtime פֿאַרבינדט זיך דורך אַ נאָרמאַליזירטן אַדרעס. ווערסיע 1.2.0 נאָרמאַליזירט host־שטריקלעך פֿון דער זעלבער מאַשין, אָבער אייגענע host־overrideס קענען נאָך אַלץ זיך שלאָגן מיט דאַטאַבאַזע־דערלויבונגען.

### אַסינכראָנע push־ריי ווערט נישט אויסגעליידיקט

אויב `verify-sovereign-push` ענדיקט זיך נישט, קאָנטראָלירט Action Scheduler אָדער דעם קאָנפֿיגורירטן אַסינכראָנעם runner. רייניקט דורכגעפֿאַלענע אַרבעטן בלויז נאָך דעם ווי איר באַשטעטיקט אַז זיי זענען זיכער ווידער צו פּרוּוון אָדער אַוועקצוּוואַרפֿן.

### טענאַנט־באַניצער־צאָל איז נישט ריכטיק

די מיגראַציע זאָל צושטעלן באַניצער פֿאַר דעם סאַוורענעם טענאַנט. אויב דער דערוואַרטער אינסטאַלאַציע־באַניצער פֿעלט, לויפֿט ווידער דעם באַניצער־צושטעל־שריט איידער איר פּרוּווט SSO ווידער.

### SSO־באַזוך ווערט אָפּגעוואָרפֿן

Stateless טענאַנט־autologin דאַרף אַז דער טענאַנט־דאָמיין, origin pin, token purpose, nonce, און expiry זאָלן שטימען. באַשטעטיקט אַז דער טענאַנט־URL איז ריכטיק און אַז מען פּרוּווט זיך אַרײַנצולאָגירן באַלד נאָך שאַפֿן דעם SSO־באַזוך.

## ווען ווידער צו פּרוּוון

פּרוּווט ווידער וועריפֿיקאַציע נאָך יעדער אינפֿראַסטרוקטור־ענדערונג. באַשטעלט נישט איבער פּראָדוקציע־טראַפיק, מעקט נישט אויס די מקור־דאַטן, און נעמט נישט אַוועק מיגראַציע־קרעדענשאַלז ביז אַלע וועריפֿיקאַציע־קאָנטראָלן גייען דורך.
