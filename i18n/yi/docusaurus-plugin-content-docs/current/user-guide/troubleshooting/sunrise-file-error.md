# טעות בײַם אינסטאַלירן די Sunrise טעקע {#error-installing-the-sunrise-file}

די sunrise.php טעקע איז אַ ספּעציעלע טעקע וואָס WordPress זוכט בעת עס שטאַרט זיך. כּדי WordPress זאָל קענען דערקענען די sunrise.php טעקע, דאַרף זי זיך געפֿינען אין דעם **wp-content טעקע־קאַטאַלאָג**.

ווען איר אַקטיווירט Ultimate Multisite און גייט דורך דעם אײַנשטעל־וויזאַרד ווי דער וואָס איר האָט אויפֿן סקרינשוט, פּרוּווט Ultimate Multisite קאַפּירן אונדזער sunrise.php טעקע אין דעם wp-content טעקע־קאַטאַלאָג.

<!-- סקרינשוט נישט בנימצא: אײַנשטעל־וויזאַרד בלאַט וואָס ווײַזט דעם sunrise.php אינסטאַלאַציע־שריט -->

אין רובֿ פֿאַלן קענען מיר מיט הצלחה קאַפּירן די טעקע און אַלץ אַרבעט. אָבער אויב עפּעס איז נישט ריכטיק אײַנגעשטעלט (טעקע־קאַטאַלאָג רעכט, למשל), קענט איר טרעפֿן אַ סצענאַר וווּ Ultimate Multisite קען נישט קאַפּירן די טעקע.

אויב איר לייענט די טעות־מעלדונג וואָס Ultimo גיט אײַך, וועט איר זען אַז פּונקט דאָס איז דאָ געשען: **Sunrise copy failed**.

<!-- סקרינשוט נישט בנימצא: טעות־מעלדונג וואָס ווײַזט Sunrise copy failed -->

כּדי דאָס צו פֿאַרריכטן, קענט איר פּשוט קאַפּירן די sunrise.php טעקע אין דעם wp-ultimo plugin טעקע־קאַטאַלאָג און זי אַרײַנלייגן אין אייער wp-content טעקע־קאַטאַלאָג. נאָכדעם וואָס איר טוט דאָס, לאָדט איבער דעם וויזאַרד־בלאַט און די קאָנטראָלן זאָלן דורכגיין.

<!-- סקרינשוט נישט בנימצא: טעקע־פֿאַרוואַלטער וואָס ווײַזט sunrise.php אין דעם wp-ultimo plugin טעקע־קאַטאַלאָג --> אין יעדן פֿאַל, דאָס קען פֿאַרדינען אַ אַלגעמיינע קאָנטראָל פֿון אייערע טעקע־קאַטאַלאָג רעכט כּדי צו פֿאַרמײַדן פּראָבלעמען אין דער צוקונפֿט (נישט נאָר מיט Ultimate Multisite נאָר אויך מיט אַנדערע plugins און themes).

די **Health Check געצייַג** וואָס איז טייל פֿון WordPress (איר קענט צוטריטן דערצו דורך אייער הויפּט־זייט **אַדמין־פּאַנעל > געצייַגן > Health Check**) קען אײַך מעלדן אויב אייערע טעקע־קאַטאַלאָג רעכט זענען באַשטימט אויף ווערטן וואָס קענען גורם זײַן פּראָבלעמען מיט WordPress.
