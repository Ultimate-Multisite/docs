# פּלוגין־בויער & זאַמדקעסטל

Gratis AI Agent v1.5.0 פֿירט אַרײַן דעם **פּלוגין־בויער & זאַמדקעסטל־סיסטעם**, וואָס לאָזט דעם AI־אַסיסטענט שאַפֿן, אַקטיווירן, און פֿאַרוואַלטן WordPress פּלוגינס אויף אײַער נעץ — אַלץ דורך אַ זיכערער, אָפּגעזונדערטער זאַמדקעסטל־סבֿיבֿה.

## איבערבליק

דער פּלוגין־בויער דערמעגלעכט דעם AI־אַסיסטענט צו שרײַבן אייגענע WordPress פּלוגינס לויט בקשות אין נאַטירלעכער שפּראַך. געשאַפֿענע פּלוגינס ווערן וואַלידירט, אָפּגעהיט, און אַקטיווירט אינעם זאַמדקעסטל־שיכט איידער זיי האָבן אַמאָל אַ השפּעה אויף לעבעדיקע וועבזײַטל־פֿונקציאָנאַליטעט.

באַניץ־פֿאַלן אַרײַננעמען:

- שאַפֿן לײַכטוועגיקע נוצ־פּלוגינס אָן אַנטוויקלער־באַטייליקונג.
- פּראָטאָטיפּירן פֿעיִקייטן וואָס דאַרפֿן WordPress האָקס אָדער אייגענע פּאָסט־טיפּן.
- שאַפֿן קורצלעביקע אויטאָמאַציע־סקריפּטן פֿאַר באַטש־אָפּעראַציעס.

## שאַפֿן אַ פּלוגין דורך AI

כּדי צו שאַפֿן אַ פּלוגין, עפֿנט דעם Gratis AI Agent שמועס־צובינד און באַשרײַבט וואָס איר דאַרפֿט. למשל:

> "שאַף אַ פּלוגין וואָס לייגט צו אַן אייגענע אַדמין־מעלדונג אויפֿן Dashboard."

דער AI וועט:

1. פּראָדוצירן דעם פּלוגין PHP קאָד ניצנדיק סטרוקטורירטע קאָד־שאַפֿונג.
2. וואַלידירן דעם רעזולטאַט אויף סינטאַקס־טעותים און אומזיכערע מוסטערן.
3. אָפּהיטן דעם געשאַפֿענעם פּלוגין אין דער זאַמדקעסטל־קראָם.
4. צוריקגעבן אַ באַשטעטיקונג מיטן פּלוגין־שלאַג און אַ **אַקטיווירן אין זאַמדקעסטל** קנעפּל.

איר קענט פֿאַרפֿײַנערן דעם רעזולטאַט דורך נאָכפֿאָלגן אין דער זעלבער שמועס־פֿאָדעם איידער אַקטיווירן.

## זאַמדקעסטל־אַקטיוואַציע

אַקטיווירן אַ געשאַפֿענעם פּלוגין אינעם זאַמדקעסטל איז באַזונדער פֿון אַקטיווירן אים אויף דער לעבעדיקער נעץ. דער זאַמדקעסטל:

- לויפֿט דעם פּלוגין אין אַן אָפּגעזונדערטער WordPress סבֿיבֿה (wp-env).
- כאַפּט יעדע PHP טעותים, וואָרענונגען, אָדער האָק־קאָנפֿליקטן.
- באַריכטעט דעם אַקטיוואַציע־רעזולטאַט צוריק אינעם שמועס־צובינד.

כּדי צו אַקטיווירן אַ פּלוגין אינעם זאַמדקעסטל, דריקט דעם **אַקטיווירן אין זאַמדקעסטל** קנעפּל אין דער AI שמועס־ענטפֿער, אָדער ניצט דעם שלעש־באַפֿעל:

```
/activate-plugin <plugin-slug>
```

אַ סטאַטוס־מעלדונג באַשטעטיקט צי די אַקטיוואַציע איז געלונגען אָדער דורכגעפֿאַלן. בײַ דורכפֿאַל ווערט דער טעות־לאָג געוויזן אינעם שמועס־פֿאָדעם.

## פֿאַרוואַלטן געשאַפֿענע פּלוגינס

געשאַפֿענע פּלוגינס ווערן אויסגעליסטעט אין **Gratis AI Agent → פּלוגין־בויער → פֿאַרוואַלטן פּלוגינס**. פֿון דעם עקראַן קענט איר:

| אַקציע | באַשרײַבונג |
|---|---|
| **זען מקור** | איבערקוקן דעם פולן פּלוגין PHP קאָד. |
| **ווידער־אַקטיווירן אין זאַמדקעסטל** | ווידער דורכפֿירן דעם זאַמדקעסטל־אַקטיוואַציע־טשעק. |
| **אינסטאַלירן אויף נעץ** | אַרויסשטעלן דעם פּלוגין אויף דער לעבעדיקער נעץ (דאַרפֿט האַנטיקע באַשטעטיקונג). |
| **דערהײַנטיקן** | צושטעלן אַ נײַע ווערסיע דורך דעם AI, וואָס פֿאַרבײַט דעם עקזיסטירנדיקן קאָד. |
| **מעקן** | אַראָפּנעמען דעם פּלוגין פֿון דער זאַמדקעסטל־קראָם. דעאַקטיווירט אים קודם פֿון אַלע וועבזײַטן. |

:::warning
**אינסטאַלירן אויף נעץ** שטעלט אַרויס דעם געשאַפֿענעם פּלוגין אויף אײַער לעבעדיקער WordPress multisite. קוקט איבער דעם פּלוגין־קאָד איידער איר גייט ווײַטער. Gratis AI Agent וועט בעטן באַשטעטיקונג איידער ער פֿאַרענדיקט אַ לעבעדיקע אינסטאַלאַציע.
:::

## אינסטאַלירן אַ געשאַפֿענעם פּלוגין אויף דער נעץ

ווען איר זענט צופֿרידן מיט אַ פּלוגין אינעם זאַמדקעסטל, קענט איר אים אינסטאַלירן אויף דער לעבעדיקער נעץ:

1. גייט צו **Gratis AI Agent → פּלוגין־בויער → פֿאַרוואַלטן פּלוגינס**.
2. דריקט **אינסטאַלירן אויף נעץ** לעבן דעם פּלוגין וואָס איר ווילט אַרויסשטעלן.
3. באַשטעטיקט דעם דיאַלאָג. דער פּלוגין ווערט אינסטאַלירט אין `wp-content/plugins/` און נעץ־אַקטיווירט.

אַלטערנאַטיוו, ניצט דעם שלעש־באַפֿעל אינעם שמועס־צובינד:

```
/install-plugin <plugin-slug>
```

## פּלוגין־דערהײַנטיקונגען

כּדי צו דערהײַנטיקן אַ געשאַפֿענעם פּלוגין, באַשרײַבט די ענדערונג צום AI־אַסיסטענט אין אַ נײַעם שמועס:

> "דערהײַנטיק דעם dashboard-notice פּלוגין אַזוי אַז ער זאָל ווײַזן די מעלדונג בלויז צו אַדמיניסטראַטאָרן."

דער AI שאַפֿט אַ נײַע ווערסיע, וואָס דערשײַנט אינעם זאַמדקעסטל לעבן דער איצטיקער ווערסיע. איר קוקט איבער דעם אונטערשייד און באַשטעטיקט איידער די דערהײַנטיקונג ווערט אָנגעווענדט.

## HookScanner אינטעגראַציע

דער פּלוגין־בויער ניצט אַן אינטעגרירטן **HookScanner** צו אַנאַליזירן די האָקס און פֿילטערס רעגיסטרירט דורך יעדן געשאַפֿענעם פּלוגין. HookScanner רעזולטאַט ווערט געוויזן אין דער שמועס־ענטפֿער און נעמט אַרײַן:

- רעגיסטרירטע אַקציע־האָקס (`add_action` רופֿן).
- רעגיסטרירטע פֿילטער־האָקס (`add_filter` רופֿן).
- יעדע האָקס געפֿונען אין פּלוגין־אָפענגיקייטן (איבערהיפּט `vendor/` און `node_modules/` דירעקטאָריעס).

דאָס העלפֿט אײַך פֿאַרשטיין אַ פּלוגינס אויפֿפֿירונג איידער איר אַקטיווירט אים.

## זיכערהייט־באַטראַכטונגען

- געשאַפֿענע פּלוגינס ווערן אָפּגעהיט באַזונדער פֿון האַנטיק אינסטאַלירטע פּלוגינס און זענען נישט צוטריטלעך דורך דעם נאָרמאַלן WordPress פּלוגין־פֿאַרוואַלטונג־עקראַן ביז איר אינסטאַלירט זיי בפֿירוש אויף דער נעץ.
- דער זאַמדקעסטל ניצט דרך־וואַלידאַציע צו פֿאַרמײַדן דורכגיין דורך דירעקטאָריעס בײַם שרײַבן פּלוגין־טעקעס.
- פּלוגינס מיט סכּנהדיקע פֿונקציע־רופֿן (למשל, `eval`, `exec`, `system`) ווערן אָנגעצייכנט בעת וואַלידאַציע און וועלן נישט אַקטיווירט ווערן.
