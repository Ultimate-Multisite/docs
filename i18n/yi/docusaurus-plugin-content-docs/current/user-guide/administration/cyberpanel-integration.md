# CyberPanel אינטעגראַציע

דער וועגווייזער דערקלערט ווי צו קאָנפֿיגורירן די Ultimate Multisite CyberPanel אינטעגראַציע, אַזוי אַז צוגעפּאַסטע דאָמענס אין אייער נעץ ווערן אויטאָמאַטיש צוגעגעבן (און אַוועקגענומען) ווי ווירטועלע באַלעבאָסן אין CyberPanel, מיט אָפּציאָנעלער אויטאָ-SSL צושטעלונג דורך Let's Encrypt.

## וואָס עס טוט {#what-it-does}

- ווען אַ דאָמען ווערט צוגעפּאַסט אין Ultimate Multisite, רופֿט די אינטעגראַציע דעם CyberPanel API צו שאַפֿן אַ ווירטועלן באַלעבאָס פֿאַר יענעם דאָמען.
- ווען אַ דאָמען-צופּאַסונג ווערט אַוועקגענומען, רופֿט די אינטעגראַציע דעם API צו אויסמעקן דעם פּאַסיקן ווירטועלן באַלעבאָס.
- ווען אויטאָ-SSL איז אַקטיווירט, טריגערט די אינטעגראַציע Let's Encrypt סערטיפֿיקאַט-אַרויסגעבן גלייך נאָך דעם וואָס דער ווירטועלער באַלעבאָס ווערט געשאַפֿן.
- לייגט אָפּציאָנעל צו/נעמט אַוועק דעם `www.` אַליאַס לויט אייער "אויטאָמאַטיש שאַפֿן www אונטערדאָמען" באַשטעטיקונג אין דאָמען-צופּאַסונג באַשטעטיקונגען.

## פֿאָרויסבאַדינגונגען {#prerequisites}

- אַ לויפֿנדיקע CyberPanel אינסטאַנץ (v2.3 אָדער שפּעטער רעקאָמענדירט) דערגרייכלעך פֿון אייער WordPress סערווער.
- אַן עקזיסטירנדיקע וועבזײַט אין CyberPanel וואָס דינט שוין אייער WordPress נעץ-וואָרצל. די אינטעגראַציע הענגט אָן נײַע ווירטועלע באַלעבאָסן צו דעם סערווער.
- CyberPanel API צוטריט אַקטיווירט. אָטענטיפֿיקאַציע ניצט אייער CyberPanel אַדמין באַניצערנאָמען און פּאַראָל.
- אייערע DNS רעקאָרדס פֿאַר צוגעפּאַסטע דאָמענס מוזן שוין ווײַזן אויף אייער סערווער'ס IP אַדרעס איידער אויטאָ-SSL קען אַרויסגעבן אַ גילטיקן סערטיפֿיקאַט.

## רעקווירעמענטן {#requirements}

די פֿאָלגנדיקע קאָנסטאַנטן מוזן זײַן דעפֿינירט אין אייער `wp-config.php` טעקע:

```php
define('WU_CYBERPANEL_HOST', 'https://your-cyberpanel-host:8090');
define('WU_CYBERPANEL_USERNAME', 'your_admin_username');
define('WU_CYBERPANEL_PASSWORD', 'your_admin_password');
define('WU_CYBERPANEL_PACKAGE', 'Default');
```

אָפּציאָנעל קענט איר אויך דעפֿינירן:

```php
define('WU_CYBERPANEL_AUTO_SSL', true);          // Default: true — issue Let's Encrypt SSL after domain creation
define('WU_CYBERPANEL_PHP_VERSION', 'PHP 8.2');  // Default: PHP 8.2
define('WU_CYBERPANEL_EMAIL', 'admin@yourdomain.com'); // Used for SSL certificate contact
```

## אויפֿשטעל־אנווייזונגען {#setup-instructions}

### 1. אַקטיווירן דעם CyberPanel API {#1-enable-the-cyberpanel-api}

1. לאָגט זיך אַרײַן אין אייער CyberPanel Dashboard ווי אַן אַדמיניסטראַטאָר.
2. גייט צו **זיכערהייט** > **SSL** און באַשטעטיקט אַז SSL איז אַקטיוו אויף דער CyberPanel צובינד גופֿא (נויטיק פֿאַר זיכערע API רופֿן).
3. דער CyberPanel API איז פֿאַראַן בײַ `https://your-server-ip:8090/api/` לויטן סטאַנדאַרט. קיין נאָך שריט זענען נישט נויטיק עס צו אַקטיווירן — עס איז אָן לויטן סטאַנדאַרט פֿאַר אַדמין באַניצערס.

### 2. צולייגן קאָנסטאַנטן צו wp-config.php {#2-add-constants-to-wp-configphp}

לייגט צו די פֿאָלגנדיקע קאָנסטאַנטן צו אייער `wp-config.php` טעקע פֿאַר דער `/* That's all, stop editing! */` שורה:

```php
define('WU_CYBERPANEL_HOST', 'https://your-cyberpanel-host:8090');
define('WU_CYBERPANEL_USERNAME', 'admin');
define('WU_CYBERPANEL_PASSWORD', 'your_secure_password');
define('WU_CYBERPANEL_PACKAGE', 'Default');
```

צו אַקטיווירן אויטאָ-SSL (רעקאָמענדירט):

```php
define('WU_CYBERPANEL_AUTO_SSL', true);
define('WU_CYBERPANEL_EMAIL', 'admin@yourdomain.com');
```

### 3. אַקטיווירן די אינטעגראַציע {#3-enable-the-integration}

1. אין אייער WordPress נעץ-אַדמין, גייט צו **Ultimate Multisite** > **באַשטעטיקונגען**.
2. נאַוויגירט צום **דאָמען-צופּאַסונג** קוויטל.
3. ראָלט אַראָפּ צו **באַלעבאָס-אינטעגראַציעס**.
4. אַקטיווירט די **CyberPanel** אינטעגראַציע.
5. קוועטשט **היטן ענדערונגען**.

### 4. באַשטעטיקן פֿאַרבינדלעכקייט {#4-verify-connectivity}

ניצט דעם אַרײַנגעבויטן פֿאַרבינדונג-טעסט אין דעם באַשטעטיקונגען-וויזאַרד:

1. גייט צו **Ultimate Multisite** > **באַשטעטיקונגען** > **דאָמען-צופּאַסונג** > **באַלעבאָס-אינטעגראַציעס** > **CyberPanel**.
2. קוועטשט **טעסטן פֿאַרבינדונג**.
3. אַ דערפֿאָלג-אָנזאָג באַשטעטיקט אַז דער plugin קען דערגרייכן דעם CyberPanel API און זיך אָטענטיפֿיצירן ריכטיק.

## ווי עס אַרבעט {#how-it-works}

### דאָמען-צופּאַסונג {#domain-mapping}

ווען אַ דאָמען ווערט צוגעפּאַסט אין Ultimate Multisite:

1. די אינטעגראַציע שיקט אַ `POST` בקשה צו `/api/createWebsite` אויף אייער CyberPanel באַלעבאָס.
2. CyberPanel שאַפֿט אַ נײַעם ווירטועלן באַלעבאָס פֿאַר דעם דאָמען אונטער דעם קאָנפֿיגורירטן פּעקל.
3. דער דאָקומענט-וואָרצל ווערט באַשטימט צו ווײַזן אויף אייער WordPress נעץ-וואָרצל טעקע־אָרט.
4. ווען די דאָמען-צופּאַסונג ווערט אַוועקגענומען, רופֿט די אינטעגראַציע `/api/deleteWebsite` צו אויפֿרייניקן דעם ווירטועלן באַלעבאָס.

### אויטאָ-SSL {#auto-ssl}

ווען `WU_CYBERPANEL_AUTO_SSL` איז `true`:

1. נאָך דעם וואָס דער ווירטועלער באַלעבאָס ווערט געשאַפֿן, רופֿט די אינטעגראַציע `/api/issueSSL` פֿאַר דעם דאָמען.
2. CyberPanel בעט אַ Let's Encrypt סערטיפֿיקאַט מיטן ACME HTTP-01 אַרויסרוף.
3. דער סערטיפֿיקאַט ווערט אויטאָמאַטיש באַנײַט דורך CyberPanel איידער ער לויפֿט אויס.

> **וויכטיק:** DNS מוז זײַן גאָר פֿאַרשפּרייט צו אייער סערווער'ס IP אַדרעס איידער Let's Encrypt קען וואַלידירן דעם דאָמען. אויב SSL אַרויסגעבן פֿעלט אויס גלייך נאָך צופּאַסונג, וואַרט אויף DNS פֿאַרשפּרייטונג און טריגערט ווידער SSL פֿון דעם CyberPanel Dashboard אונטער **SSL** > **פֿאַרוואַלטן SSL**.

### www אונטערדאָמען {#www-subdomain}

אויב **אויטאָמאַטיש שאַפֿן www אונטערדאָמען** איז אַקטיווירט אין אייערע דאָמען-צופּאַסונג באַשטעטיקונגען, שאַפֿט די אינטעגראַציע אויך אַ ווירטועלן באַלעבאָס-אַליאַס פֿאַר `www.<domain>` און, ווען אויטאָ-SSL איז אָן, גיט אַרויס אַ סערטיפֿיקאַט וואָס דעקט ביידע די הויפּט- און www וואַריאַנטן.

### בליצפּאָסט ווײַטערפֿירער {#email-forwarders}

ווען דער [Ultimate Multisite: בליצפּאָסטן](../../addons/ultimate-multisite-emails/) addon איז אַקטיוו, קען CyberPanel אויך צושטעלן קונה בליצפּאָסט-ווײַטערפֿירער. ווײַטערפֿירער רוטירן אָנזאָגן פֿון אַ דאָמען-אַדרעס צו אַן אַנדער אינקעסטל אָן שאַפֿן אַ פולן בריווקאַסטן, וואָס איז נוצלעך פֿאַר אַליאַסן ווי `info@customer-domain.test` אָדער `support@customer-domain.test`.

איידער איר אַקטיווירט ווײַטערפֿירער פֿאַר קונים:

1. באַשטעטיקט אַז די CyberPanel קאָנסטאַנטן אויבן זענען קאָנפֿיגורירט און דער פֿאַרבינדונג-טעסט גייט דורך.
2. אַקטיווירט דעם CyberPanel בליצפּאָסט-צושטעלער אין די Emails addon באַשטעטיקונגען.
3. באַשטעטיקט אַז דער קונה-דאָמען עקזיסטירט שוין אין CyberPanel איידער איר שאַפֿט דעם ווײַטערפֿירער.
4. שאַפֿט אַ פּראָבע-ווײַטערפֿירער און שיקט דורך אים אַן אָנזאָג איידער איר אָפֿערט די פֿעיִקייט אויף פּראָדוקציע פּלענער.

אויב שאַפֿן אַ ווײַטערפֿירער פֿעלט אויס, קאָנטראָלירט ערשט די Ultimate Multisite אַקטיוויטעט-לאָגס, דערנאָך באַשטעטיקט אין CyberPanel אַז דער מקור-דאָמען עקזיסטירט און אַז דער API באַניצער האָט בליצפּאָסט-פֿאַרוואַלטונג דערלויבענישן.

|---|---|---|---|
| `WU_CYBERPANEL_HOST` | יאָ | — | פֿולער URL צו אייער CyberPanel אינסטאַנץ אַרײַנגערעכנט פּאָרט, למשל `https://cp.example.com:8090` |
| `WU_CYBERPANEL_USERNAME` | יאָ | — | CyberPanel אַדמין באַניצערנאָמען |
| `WU_CYBERPANEL_PASSWORD` | יאָ | — | CyberPanel אַדמין פּאַראָל |
| `WU_CYBERPANEL_PACKAGE` | יאָ | `Default` | CyberPanel האָסטינג-פּעקל צו צוטיילן צו נײַע ווירטועלע באַלעבאָסן |
| `WU_CYBERPANEL_AUTO_SSL` | ניין | `true` | אַרויסגעבן אַ Let's Encrypt SSL סערטיפֿיקאַט נאָך דאָמען-שאַפֿן |
| `WU_CYBERPANEL_PHP_VERSION` | ניין | `PHP 8.2` | PHP ווערסיע פֿאַר נײַע ווירטועלע באַלעבאָסן (מוז שטימען מיט אַ ווערסיע אינסטאַלירט אין CyberPanel) |
| `WU_CYBERPANEL_EMAIL` | ניין | — | קאָנטאַקט-בליצפּאָסט פֿאַר SSL סערטיפֿיקאַט רעגיסטראַציע |

## וויכטיקע באַמערקונגען {#important-notes}

- CyberPanel'ס API ניצט סעסיע-באַזירטע token אָטענטיפֿיקאַציע. די אינטעגראַציע האַנדלט token קריגן אויטאָמאַטיש בײַ יעדן API רוף.
- אייער CyberPanel אַדמין Account מוז האָבן דערלויבענישן צו שאַפֿן און אויסמעקן וועבזײַטן.
- CyberPanel לויפֿט אויף פּאָרט `8090` לויטן סטאַנדאַרט. אויב אייער סערווער ניצט אַ פֿײַערוואַנט, זאָרגט אַז דער פּאָרט זאָל זײַן דערגרייכלעך פֿון דעם WordPress אַפּליקאַציע-סערווער.
- די אינטעגראַציע פֿאַרוואַלט נישט DNS רעקאָרדס. איר מוזט ווײַזן דאָמען-DNS אויף אייער סערווער'ס IP אַדרעס איידער איר פּאַסט צו דעם דאָמען אין Ultimate Multisite.
- אויב איר ניצט OpenLiteSpeed (OLS), ווערט אַ גראַציעזער ווידער-אָנהייב אויטאָמאַטיש געטריגערט נאָך ווירטועלער באַלעבאָס ענדערונגען. קיין מאַנועלע אַרײַנמישונג איז נישט נויטיק.

## פּראָבלעם־לייזונג {#troubleshooting}

### API פֿאַרבינדונג אָפּגעזאָגט {#api-connection-refused}

- באַשטעטיקט אַז פּאָרט `8090` איז אָפֿן אין אייער סערווער פֿײַערוואַנט.
- באַשטעטיקט אַז דער `WU_CYBERPANEL_HOST` ווערט נעמט אַרײַן דעם ריכטיקן פּראָטאָקאָל (`https://`) און פּאָרט.
- קאָנטראָלירט אַז אייער CyberPanel SSL סערטיפֿיקאַט איז גילטיק; זעלבסט-אונטערגעשריבענע סערטיפֿיקאַטן קענען גורם זײַן TLS וועריפֿיקאַציע דורכפֿאַלן. שטעלט `WU_CYBERPANEL_VERIFY_SSL` אויף `false` נאָר אין פֿאַרטרויענסווערטע פּריוואַטע נעץ-סביבות.

### אָטענטיפֿיקאַציע גרײַזן {#authentication-errors}

- באַשטעטיקט אַז אייער `WU_CYBERPANEL_USERNAME` און `WU_CYBERPANEL_PASSWORD` זענען ריכטיק דורך זיך אַרײַנלאָגירן גלייך אין CyberPanel.
- CyberPanel שליסט Accounts נאָך איבערגעחזרטע דורכגעפֿאַלענע לאָגין-פּרוּוון. קאָנטראָלירט **זיכערהייט** > **Brute Force Monitor** אין CyberPanel אויב שליסונגען פּאַסירן.

### דאָמען נישט געשאַפֿן {#domain-not-created}

- קאָנטראָלירט דעם Ultimate Multisite אַקטיוויטעט-לאָג (**Ultimate Multisite** > **אַקטיוויטעט-לאָגס**) פֿאַר API גרײַז-אָנזאָגן.
- באַשטעטיקט אַז דאָס פּעקל דעפֿינירט אין `WU_CYBERPANEL_PACKAGE` עקזיסטירט אין CyberPanel (**פּעקלעך** > **רשימה פֿון פּעקלעך**).
- זאָרגט אַז דער דאָמען איז נישט שוין רעגיסטרירט ווי אַ וועבזײַט אין CyberPanel — דופּליקאַט וועבזײַט-שאַפֿן גיט צוריק אַ גרײַז.

### SSL סערטיפֿיקאַט נישט אַרויסגעגעבן {#ssl-certificate-not-issued}

- באַשטעטיקט אַז DNS איז גאָר פֿאַרשפּרייט: `dig +short your-domain.com` זאָל צוריקגעבן אייער סערווער'ס IP.
- Let's Encrypt דורכפֿירט קורס-גרענעצן. אויב איר האָט לעצטנס אַרויסגעגעבן עטלעכע סערטיפֿיקאַטן פֿאַר דעם זעלבן דאָמען, וואַרט איידער איר פּרוּווט ווידער.
- קאָנטראָלירט די CyberPanel SSL לאָגס אונטער **לאָגס** > **גרײַז-לאָגס** פֿאַר פּרטים וועגן סערטיפֿיקאַט-אַרויסגעבן דורכפֿאַלן.
- ווי אַ צוריקפֿאַל, קענט איר אַרויסגעבן SSL מאַנועל פֿון CyberPanel: **SSL** > **פֿאַרוואַלטן SSL** > אויסקלײַבן דעם דאָמען > **אַרויסגעבן SSL**.
