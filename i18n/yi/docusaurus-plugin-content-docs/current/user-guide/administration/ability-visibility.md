# פֿעיִקייט־זעבאַרקייט {#ability-visibility}

Superdav AI Agent v1.12.0 ברענגט אַרײַן **פֿעיִקייט־זעבאַרקייט** קאָנטראָלן וואָס באַשטימען וועלכע אויבערפֿלעכן שטעלן אַרויס יעדע פֿעיִקייט. דאָס לאָזט אַדמיניסטראַטאָרן פֿײַן־צופּאַסן וועלכע אַגענט־מעגלעכקייטן זענען בנימצא דורך REST APIs, MCP סערווערס, WooCommerce אינטעגראַציעס, און אַנדערע צובינדן.

## וואָס איז פֿעיִקייט־זעבאַרקייט? {#what-is-ability-visibility}

פֿעיִקייט־זעבאַרקייט איז אַ דערלויבעניש־סיסטעם וואָס קאָנטראָלירט:

- **וועלכע פֿעיִקייטן** זענען בנימצא פֿאַר אַגענטן
- **וווּ זיי ווערן אַרויסגעשטעלט** (REST API, MCP, WooCommerce, אאַז״וו)
- **ווער קען צוטריט באַקומען צו זיי** (דורך פּאַרטנער־דערלויבט־רשימות)
- **ווי זיי ווערן קלאַסיפֿיצירט** (דערקענט קעגן נישט־קלאַסיפֿיצירט)

יעדע פֿעיִקייט טראָגט אַ זעבאַרקייט־מדרגה וואָס באַשטימט איר בנימצאקייט איבער פֿאַרשידענע אויבערפֿלעכן.

## זעבאַרקייט־מדרגות {#visibility-levels}

### עפֿנטלעך {#public}

**עפֿנטלעכע פֿעיִקייטן** זענען בנימצא אומעטום:

- REST API ענדפּוינטן
- MCP סערווערס
- WooCommerce אינטעגראַציעס
- שמועס־צובינדן
- אַלע באַניצער־ראָלעס (מיט פּאַסיקע דערלויבענישן)

בײַשפּיל: `scaffold-block-theme`, `activate-theme`, `send-email`

### אינערלעך {#internal}

**אינערלעכע פֿעיִקייטן** זענען בנימצא בלויז אין אײַער WordPress אינסטאַלאַציע:

- שמועס־צובינדן
- אַדמין־פּאַנעלן
- נאָר אײַנגעקלאָצטע באַניצער
- נישט אַרויסגעשטעלט דורך REST API אָדער אויסערלעכע אינטעגראַציעס

בײַשפּיל: `manage-settings`, `view-analytics`, `export-data`

### פּאַרטנער {#partner}

**פּאַרטנער־פֿעיִקייטן** זענען בנימצא בלויז פֿאַר פּאַרטנערס אויף אַ דערלויבט־רשימה:

- דאַרף אַן אויסדריקלעכן פּאַרטנער־דערלויבט־רשימה אײַנטראָג
- בנימצא דורך REST API מיט פּאַרטנער־קרעדענציעלן
- בנימצא דורך MCP מיט פּאַרטנער־אָטענטיפֿיקאַציע
- נישט בנימצא פֿאַר געוויינטלעכע באַניצער

בײַשפּיל: `bulk-import-users`, `modify-billing`, `access-analytics`

### אויסגעלאָשן {#disabled}

**אויסגעלאָשענע פֿעיִקייטן** זענען נישט בנימצא אין ערגעץ:

- נישט אַרויסגעשטעלט דורך קיין שום אויבערפֿלעך
- נישט בנימצא אין שמועס
- נישט בנימצא אין אַדמין־פּאַנעלן
- נוצלעך פֿאַר פֿאַרעלטערטע אָדער עקספּערימענטעלע פֿעיִקייטן

## פֿאַרוואַלטן פֿעיִקייט־זעבאַרקייט {#managing-ability-visibility}

### צוטריט באַקומען צו פֿעיִקייט־זעבאַרקייט־אײַנשטעלונגען {#accessing-ability-visibility-settings}

1. גייט צו **WordPress אַדמין** → **Superdav AI Agent** → **אײַנשטעלונגען**
2. קליקט דעם **פֿעיִקייטן** קוויטל
3. איר וועט זען אַ רשימה פֿון אַלע אינסטאַלירטע פֿעיִקייטן מיט זייערע זעבאַרקייט־מדרגות

### באַטראַכטן פֿעיִקייט־פּרטים {#viewing-ability-details}

קליקט אויף סיי וועלכער פֿעיִקייט צו זען:

- **נאָמען**: דער אידענטיפֿיקאַטאָר פֿון דער פֿעיִקייט
- **באַשרײַבונג**: וואָס די פֿעיִקייט טוט
- **איצטיקע זעבאַרקייט**: עפֿנטלעך, אינערלעך, פּאַרטנער, אָדער אויסגעלאָשן
- **פּאַרטנער־דערלויבט־רשימה**: וועלכע פּאַרטנערס קענען באַקומען צוטריט (אויב פּאַרטנער־זעבאַרקייט)
- **לעצטע דערהײַנטיקונג**: ווען די זעבאַרקייט איז צום לעצטן מאָל געביטן געוואָרן
- **סטאַטוס**: דערקענט אָדער נישט־קלאַסיפֿיצירט

### טוישן זעבאַרקייט־מדרגות {#changing-visibility-levels}

צו טוישן די זעבאַרקייט פֿון אַ פֿעיִקייט:

1. קליקט די פֿעיִקייט אין דער רשימה
2. קלײַבט אויס אַ נײַע זעבאַרקייט־מדרגה פֿון דער אַראָפּפֿאַל־רשימה
3. אויב איר קלײַבט אויס „פּאַרטנער“, לייגט צו פּאַרטנער־אידענטיפֿיקאַטאָרן צו דער דערלויבט־רשימה
4. קליקט **אָפּהיטן**

בײַשפּיל:

```
Ability: bulk-import-users
Current visibility: Public
Change to: Partner
Partner allow-list: partner-123, partner-456
```

### מאַסן־אָפּעראַציעס {#bulk-operations}

צו טוישן זעבאַרקייט פֿאַר עטלעכע פֿעיִקייטן:

1. צייכנט אָן די קעסטלעך לעבן די פֿעיִקייטן
2. קלײַבט אויס אַ נײַע זעבאַרקייט־מדרגה פֿון דער מאַסן־אַקציע אַראָפּפֿאַל־רשימה
3. קליקט **צולייגן**

## פּאַרטנער־דערלויבט־רשימה {#partner-allow-list}

די **פּאַרטנער־דערלויבט־רשימה** קאָנטראָלירט וועלכע אויסערלעכע פּאַרטנערס קענען באַקומען צוטריט צו פּאַרטנער־מדרגה פֿעיִקייטן.

### צולייגן פּאַרטנערס {#adding-partners}

1. גייט צו **Superdav AI Agent** → **אײַנשטעלונגען** → **פּאַרטנערס**
2. קליקט **צולייגן פּאַרטנער**
3. גיט אַרײַן דעם פּאַרטנער־אידענטיפֿיקאַטאָר (געוויינטלעך אַן API שליסל אָדער אָרגאַניזאַציע־ID)
4. לייגט אפֿשר צו אַ פּאַרטנער־נאָמען און באַשרײַבונג
5. קליקט **אָפּהיטן**

### צוטיילן פֿעיִקייטן צו פּאַרטנערס {#assigning-abilities-to-partners}

נאָך צולייגן אַ פּאַרטנער:

1. גייט צום **פֿעיִקייטן** קוויטל
2. געפֿינט אַ פֿעיִקייט מיט פּאַרטנער־זעבאַרקייט
3. קליקט צו רעדאַקטירן
4. אין דער „פּאַרטנער־דערלויבט־רשימה“ אָפּטיילונג, צייכנט אָן די קעסטלעך פֿאַר פּאַרטנערס וואָס זאָלן האָבן צוטריט
5. קליקט **אָפּהיטן**

### אָפּנעמען פּאַרטנער־צוטריט {#revoking-partner-access}

צו אַוועקנעמען דעם צוטריט פֿון אַ פּאַרטנער:

1. רעדאַקטירט די פֿעיִקייט
2. נעמט אַוועק דעם צייכן פֿון דעם פּאַרטנערס קעסטל אין דער דערלויבט־רשימה
3. קליקט **אָפּהיטן**

דער פּאַרטנער וועט גלײַך פֿאַרלירן צוטריט צו יענער פֿעיִקייט.

## נישט־קלאַסיפֿיצירטע פֿעיִקייטן {#unclassified-abilities}

ווען איר אינסטאַלירט אַ דריט־פּאַרטיי פֿעיִקייט וואָס Superdav AI Agent דערקענט נישט, ווערט זי באַצייכנט ווי **נישט־קלאַסיפֿיצירט**.

### אַדמין־מעלדונג פֿאַר נישט־קלאַסיפֿיצירטע פֿעיִקייטן {#admin-notice-for-unclassified-abilities}

איר וועט זען אַן אַדמין־מעלדונג:

```
⚠️ Unclassified Abilities Detected

The following abilities were installed but are not recognized:
- custom-import-tool
- external-api-wrapper

These abilities may pose security or compatibility risks.
Please review and classify them.

[Review Abilities] [Dismiss]
```

### איבערקוקן נישט־קלאַסיפֿיצירטע פֿעיִקייטן {#reviewing-unclassified-abilities}

1. קליקט **איבערקוקן פֿעיִקייטן** אין דער מעלדונג
2. פֿאַר יעדער נישט־קלאַסיפֿיצירטער פֿעיִקייט, וועט איר זען:
   - פֿעיִקייט־נאָמען און באַשרײַבונג
   - מקור (וועלכער plugin/צולייג האָט עס אינסטאַלירט)
   - פֿאָרגעשלאָגענע זעבאַרקייט־מדרגה
   - זיכערהייט־אָפּשאַצונג

3. קלײַבט אויס אַ זעבאַרקייט־מדרגה:
   - **עפֿנטלעך**: איר פֿאַרטרויט די פֿעיִקייט; שטעלט זי אַרויס אומעטום
   - **אינערלעך**: באַגרענעצט בלויז צו אינערלעכן באַניץ
   - **פּאַרטנער**: באַגרענעצט צו ספּעציפֿישע פּאַרטנערס
   - **אויסגעלאָשן**: ניצט נישט די פֿעיִקייט

4. קליקט **קלאַסיפֿיצירן** צו אָפּהיטן אײַער באַשלוס

### פֿאַר וואָס קלאַסיפֿיצירן נישט־קלאַסיפֿיצירטע פֿעיִקייטן? {#why-classify-unclassified-abilities}

קלאַסיפֿיצירן פֿעיִקייטן:

- **פֿאַרבעסערט זיכערהייט**: איר באַשטעטיקט בפֿירוש וואָס ווערט אַרויסגעשטעלט
- **פֿאַרמײַדט אומגליקן**: נישט־קלאַסיפֿיצירטע פֿעיִקייטן זענען אויסגעלאָשן לויטן סטאַנדאַרט
- **דערמעגלעכט פֿעיִקייטן**: אַ מאָל קלאַסיפֿיצירט, ווערט די פֿעיִקייט בנימצא
- **דאָקומענטירט באַשלוסן**: אײַערע ברירות ווערן פֿאַרשריבן פֿאַר קאָנטראָל־צוועקן

## זעבאַרקייט אין פֿאַרשידענע אויבערפֿלעכן {#visibility-in-different-surfaces}

### REST API {#rest-api}

**עפֿנטלעכע פֿעיִקייטן** זענען בנימצא דורך REST ענדפּוינטן:

```bash
curl -X POST https://yoursite.com/wp-json/superdav/v1/abilities/scaffold-block-theme \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"theme_name": "My Theme"}'
```

**אינערלעכע און פּאַרטנער־פֿעיִקייטן** זענען נישט בנימצא דורך REST API.

### MCP סערווערס {#mcp-servers}

**עפֿנטלעכע פֿעיִקייטן** זענען בנימצא דורך MCP:

```
MCP Server: Superdav AI Agent
Available abilities:
- scaffold-block-theme
- activate-theme
- send-email
```

**אינערלעכע פֿעיִקייטן** ווערן נישט אַרויסגעשטעלט דורך MCP.

**פּאַרטנער־פֿעיִקייטן** זענען בנימצא בלויז מיט פּאַרטנער־קרעדענציעלן.

### WooCommerce אינטעגראַציע {#woocommerce-integration}

**עפֿנטלעכע פֿעיִקייטן** פֿאַרבונדן מיט WooCommerce זענען בנימצא:

- פֿאַרוואַלטונג פֿון פּראָדוקטן
- באַאַרבעטונג פֿון באַשטעלונגען
- קאָמוניקאַציע מיט קונים

**אינערלעכע פֿעיִקייטן** ווערן נישט אַרויסגעשטעלט צו WooCommerce.

### שמועס־צובינד {#chat-interface}

**אַלע פֿעיִקייטן** (עפֿנטלעך, אינערלעך, פּאַרטנער) זענען בנימצא אין שמועס, לויט באַניצער־דערלויבענישן:

- אַדמיניסטראַטאָרן זען אַלע פֿעיִקייטן
- געוויינטלעכע באַניצער זען בלויז עפֿנטלעכע פֿעיִקייטן
- פּאַרטנערס זען עפֿנטלעכע + פּאַרטנער־פֿעיִקייטן (אויב אויף דער דערלויבט־רשימה)

## בעסטע זיכערהייט־פּראַקטיקן {#security-best-practices}

### פּרינציפּ פֿון קלענסטער פּריווילעגיע {#principle-of-least-privilege}

- שטעלט פֿעיִקייטן אויף דער מערסט באַגרענעצטער זעבאַרקייט וואָס ערלויבט נאָך אַלץ זייער באַניץ
- ניצט פּאַרטנער־זעבאַרקייט פֿאַר שפּירעוודיקע אָפּעראַציעס
- לעשט אויס פֿעיִקייטן וואָס איר ניצט נישט

### רעגולערע קאָנטראָלן {#regular-audits}

- איבערקוקט פֿעיִקייט־זעבאַרקייט יעדן חודש
- קאָנטראָלירט פֿאַר נישט־קלאַסיפֿיצירטע פֿעיִקייטן
- נעמט אַוועק צוטריט פֿאַר נישט־געניצטע פּאַרטנערס

### פֿאַרשרײַבן און מאָניטאָרן {#logging-and-monitoring}

- מאָניטאָרט וועלכע פֿעיִקייטן ווערן מערסט געניצט
- נאָכפֿאָלגט פּאַרטנער־צוטריט מוסטערן
- שיקט אַ ווארענונג בײַ אומגעוויינטלעכן פֿעיִקייט־באַניץ

### דריט־פּאַרטיי פֿעיִקייטן {#third-party-abilities}

- איבערקוקט דריט־פּאַרטיי פֿעיִקייטן איידער איר דערמעגלעכט זיי
- קלאַסיפֿיצירט זיי בפֿירוש
- הייבט אָן מיט אינערלעכער אָדער פּאַרטנער־זעבאַרקייט
- העכערט צו עפֿנטלעך נאָר נאָך דורכקוק

## פּראָבלעם־לייזונג {#troubleshooting}

**אַ פֿעיִקייט ווײַזט זיך נישט אין דער רשימה**
- באַשטעטיקט אַז די פֿעיִקייט איז אינסטאַלירט און אַקטיוו
- קאָנטראָלירט אַז דער plugin/צולייג איז דערמעגלעכט
- דערפֿרישט די זײַט

**איך קען נישט טוישן די זעבאַרקייט פֿון אַ פֿעיִקייט**
- באַשטעטיקט אַז איר האָט אַדמיניסטראַטאָר־דערלויבענישן
- קאָנטראָלירט אַז די פֿעיִקייט איז נישט פֿאַרשלאָסן דורך אַ plugin
- פּרוּווט אויסלעשן קאָנפֿליקטירנדיקע plugins

**אַ פּאַרטנער קען נישט באַקומען צוטריט צו אַ פֿעיִקייט**
- באַשטעטיקט אַז דער פּאַרטנער איז אין דער דערלויבט־רשימה
- קאָנטראָלירט אַז די זעבאַרקייט פֿון דער פֿעיִקייט איז געשטעלט אויף פּאַרטנער
- באַשטעטיקט אַז די קרעדענציעלן פֿון פּאַרטנער זענען ריכטיק
- קאָנטראָלירט API רעקאָרדן פֿאַר אָטענטיפֿיקאַציע־טעותן

**נישט־קלאַסיפֿיצירטע פֿעיִקייטן האַלטן אין זיך ווײַזן**
- איבערקוקט און קלאַסיפֿיצירט זיי אין דער אַדמין־מעלדונג
- קאָנטראָלירט אַז אײַער קלאַסיפֿיקאַציע איז אָפּגעהיט געוואָרן
- באַשטעטיקט אַז דער plugin וואָס גיט די פֿעיִקייט איז דערהײַנטיקט

## מיגראַציע פֿון אַלטן מאָדוס {#migration-from-legacy-mode}

אויב איר גראַדירט אַרויף פֿון אַן עלטערער ווערסיע פֿון Superdav AI Agent:

- אַלע עקזיסטירנדיקע פֿעיִקייטן ווערן אויטאָמאַטיש קלאַסיפֿיצירט ווי עפֿנטלעך
- איבערקוקט און פּאַסט צו זעבאַרקייט לויט נויט
- קיין אַקציע איז נישט נייטיק, סײַדן איר ווילט באַגרענעצן צוטריט

זעט **דריט־פּאַרטיי מאָדוס מיגראַציע** פֿאַר מער פּרטים וועגן דעם איבערגאַנג צו געבוירענער Abilities API אינטעגראַציע.

## קומענדיקע שריט {#next-steps}

נאָך קאָנפֿיגורירן פֿעיִקייט־זעבאַרקייט:

1. **איבערקוקט אײַערע פֿעיִקייטן**: קאָנטראָלירט אַלע אינסטאַלירטע פֿעיִקייטן
2. **קלאַסיפֿיצירט נישט־קלאַסיפֿיצירטע פֿעיִקייטן**: רעאַגירט אויף אַדמין־מעלדונגען
3. **שטעלט אײַן פּאַרטנערס**: לייגט צו פּאַרטנערס אויב איר ניצט פּאַרטנער־זעבאַרקייט
4. **מאָניטאָרט באַניץ**: נאָכפֿאָלגט וועלכע פֿעיִקייטן ווערן מערסט געניצט
5. **דאָקומענטירט באַשלוסן**: האַלט הערות פֿאַר וואָס איר האָט אויסגעקליבן יעדע זעבאַרקייט־מדרגה
