# אינטעגרירן Ultimate Multisite מיט Zapier

אין איינעם פֿון די אַרטיקלען האָבן מיר באַהאַנדלט [Webhooks](webhooks.md) און ווי זיי קענען ווערן גענוצט צו אינטעגרירן מיט דריט־פּאַרטיי אַפּליקאַציעס.

ניצן webhooks איז אַ ביסל קאָמפּליצירט, ווײַל עס דאַרף פֿאָרגעשריטענע קענטעניש אין קאָדירן און כאַפּן payloads. ניצן **Zapier** איז אַ וועג פֿאַר אײַך דאָס צו אַרומגיין.

Zapier האָט אינטעגראַציע מיט איבער 5000+ אַפּס, וואָס מאַכט קאָמוניקאַציע צווישן פֿאַרשיידענע אַפּליקאַציעס גרינגער.

איר קענט שאַפֿן **Triggers** וואָס וועלן זיך אויסלייזן ווען געשעענישן פּאַסירן אויף אײַער נעץ (למשל, אַ Account ווערט געשאַפֿן און לייזט אויס די account_create געשעעניש), אָדער שאַפֿן **Actions** אויף אײַער נעץ ווי אַ רעאַקציע אויף אויסערלעכע געשעענישן (למשל, שאַפֿן אַ נײַע Account מיטגלידערשאַפֿט אין אײַער Ultimate Multisite נעץ).

דאָס איז מעגלעך ווײַל **Ultimate Multisite Zapier's triggers** און actions ווערן געטריבן דורך דער [REST API](https://developer.ultimatemultisite.com/api/docs/).

## ווי צו אָנהייבן

ערשט, זוכט Ultimate Multisite אין דער Zapier אַפּ־ליסטע. אַלטערנאַטיוו, קענט איר קליקן [דעם לינק](https://zapier.com/apps/wp-ultimo/integrations).

גייט צו אײַער Dashboard און דריקט דעם **+** **Create Zap** קנעפּל אויף דער לינקער זײַטן־באַר צו אויפֿשטעלן אַ נײַעם Zap.

![Zapier Dashboard מיט Create Zap קנעפּל](/img/admin/webhooks-list.png)

איר וועט ווערן ווײַטערגעפֿירט צו דער Zap־שאַפֿונג זײַט.

אין דעם זוך־קעסטל טיפּט "wp ultimo". קליקט צו אויסקלײַבן די **Beta** ווערסיע אָפּציע.

![זוכן WP Ultimo אין דער Zapier אַפּ־ליסטע](/img/admin/webhooks-list.png)

נאָך אויסקלײַבן אונדזער אַפּ, קלײַבט אויס די בנימצא געשעעניש: **נײַע Ultimate Multisite געשעעניש**.

![אויסקלײַבן New Ultimate Multisite Event trigger](/img/admin/webhooks-list.png)

איצט דאַרפֿן מיר געבן Zapier צוטריט צו **אײַער נעץ**. קליקן אויף **Sign in** וועט עפֿענען אַ נײַ פֿענצטער וואָס פֿאַרלאַנגט די **API קרעדענציאַלן**.

![Zapier Sign in פֿאָרדערונג פֿאַר API קרעדענציאַלן](/img/admin/webhooks-list.png)

גייט צו אײַער נעץ־אַדמין פּאַנעל און נאַוויגירט צו **Ultimate Multisite > Settings** > **API & Webhooks** און זוכט דעם API Settings אָפּטייל.

קלײַבט אויס די **Enable API** אָפּציע, ווײַל זי איז נויטיק כּדי די פֿאַרבינדונג זאָל אַרבעטן.

![API און Webhooks אײַנשטעלונגען מיט API Settings און Enable API אָפּציעס](/img/admin/settings-api-webhooks.png)

ניצט דעם **Copy to Clipboard** בילדל אויף די API Key און API Secret פֿעלדער און קלעפּט אַרײַן יענע ווערטן אויף דעם אינטעגראַציע־עקראַן.

אין דעם URL פֿעלד, לייגט אַרײַן אײַער נעץ־גאַנצן URL, אַרײַנגערעכנט דעם פּראָטאָקאָל (HTTP אָדער HTTPS).

![Zapier אינטעגראַציע־עקראַן מיט API Key, Secret, און URL פֿעלדער](/img/admin/webhooks-list.png)

קליקט דעם **Yes, Continue** קנעפּל צו גיין ווײַטער צום קומענדיקן שריט. אויב אַלץ אַרבעט זיך אויס, זאָלט איר ווערן באַגריסט דורך אײַער נײַ פֿאַרבונדן Account! קליקט **Continue** צו שאַפֿן אַ נײַעם trigger.

## ווי צו שאַפֿן אַ נײַעם Trigger

איצט אַז אײַער Account איז פֿאַרבונדן, קענט איר זען בנימצא געשעענישן. לאָמיר אויסקלײַבן די **payment_received** געשעעניש פֿאַר דעם טוטאָריאַל.

![אויסקלײַבן payment_received געשעעניש אין Zapier trigger](/img/admin/webhooks-list.png)

אַמאָל די געשעעניש איז אויסגעקליבן און איר קליקט **continue** , וועט אַ **פּרוביר־שריט** דערשײַנען.

![Zapier פּרוביר־שריט פֿאַר דעם trigger](/img/admin/webhooks-list.png)

אין דעם שטאַפּל וועט Zapier פּרובירן צי אײַער Zap קען **ברענגען דעם ספּעציפֿישן payload צו יענער געשעעניש**. אין צוקונפֿטיקע געשעענישן פֿון דעם זעלבן טיפּ, וועט אינפֿאָרמאַציע מיט דער זעלבער סטרוקטור ווערן געשיקט.

![Zapier trigger פּרובירונג פֿאַרענדיקט הצלחהדיק מיט payload](/img/admin/webhooks-list.png)

אין אונדזער טוטאָריאַל איז די פּרובירונג **הצלחהדיק פֿאַרענדיקט** געוואָרן און האָט צוריקגעגעבן די payload בײַשפּיל־אינפֿאָרמאַציע. די בײַשפּיל־אינפֿאָרמאַציע וועט זיין נוצלעך אונדז צו פירן בײַם שאַפֿן actions. אײַער trigger איז איצט געשאַפֿן און גרייט צו ווערן פֿאַרבונדן מיט אַנדערע אַפּליקאַציעס.

## ווי צו שאַפֿן Actions

Actions ניצן אינפֿאָרמאַציע פֿון אַנדערע triggers צו שאַפֿן נײַע איינטראָגן אין אײַער נעץ.

אין דעם **שאַפֿן אַן action שריט** וועט איר אויסקלײַבן די Ultimate Multisite **Beta** און די אָפּציע פֿון **Create Items on Ultimate Multisite**.

![שאַפֿן אַן action מיט Create Items on Ultimate Multisite](/img/admin/webhooks-list.png)

אין דעם קומענדיקן שריט וועט איר אָדער שאַפֿן אײַער באַשטעטיקונג, פּונקט ווי מיר האָבן געטאָן אין **ווי צו אָנהייבן** , אָדער אויסקלײַבן אַ געשאַפֿענע באַשטעטיקונג. אין דעם טוטאָריאַל וועלן מיר אויסקלײַבן די זעלבע באַשטעטיקונג וואָס איז פֿריִער געשאַפֿן געוואָרן.

![אויסקלײַבן באַשטעטיקונג פֿאַר דער Zapier action](/img/admin/webhooks-list.png)

### אויפֿשטעלן די Action

דאָס איז דער **הויפּט־שריט פֿון דער action** און דאָ זענען די זאַכן אַ ביסל אַנדערש. די ערשטע אינפֿאָרמאַציע וואָס איר וועט אויסקלײַבן איז דער **Item**. Item איז דער **אינפֿאָרמאַציע־מאָדעל** פֿון אײַער נעץ, אַזאַ ווי **Customers, Payments, Sites, Emails** און אַנדערע.

![אויסקלײַבן Item טיפּ פֿאַר דער Zapier action](/img/admin/webhooks-list.png)

ווען מען קלײַבט אויס אַן item, וועט די פֿאָרעם **זיך איבעראָרדענען כּדי צו ברענגען די נויטיקע און אָפּציאָנעלע פֿעלדער** פֿאַר דעם אויסגעקליבענעם item.

למשל, ווען מען קלײַבט אויס דעם item **Customer** , וועלן די פֿאָרעם־פֿעלדער ברענגען אַלץ וואָס איז נייטיק אויסצופֿילן כּדי צו שאַפֿן אַ נײַעם Customer אין דער נעץ.

![Customer item פֿעלדער אין Zapier action אויפֿשטעלונג](/img/admin/webhooks-list.png)

נאָך אויספֿילן אַלע פֿעלדער וואָס זענען באַצייכנט ווי **נויטיק** און קליקן אויף continue, וועט אַ לעצטער עקראַן אײַך ווײַזן די אויסגעפֿילטע פֿעלדער און די פֿעלדער וואָס זענען געבליבן נישט אויסגעפֿילט.

![Zapier action פּרובירונג וואָס ווײַזט אויסגעפֿילטע און נישט אויסגעפֿילטע פֿעלדער](/img/admin/webhooks-list.png)

ווי באַלד ווי אײַער פּרובירונג פֿאַרענדיקט זיך און איז הצלחהדיק, איז אײַער action קאָנפֿיגורירט. עס איז אויך וויכטיק צו קאָנטראָלירן אויף אײַער נעץ צי דער item איז געשאַפֿן געוואָרן מיט דער פּרובירונג פֿון אײַער action.
