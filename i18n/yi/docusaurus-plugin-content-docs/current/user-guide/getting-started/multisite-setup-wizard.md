# Multisite אויפֿשטעלונג־וועגווייזער {#multisite-setup-wizard}

Ultimate Multisite נעמט אַרײַן אַן אײַנגעבויטן וועגווייזער וואָס קאָנווערטירט אויטאָמאַטיש אַ נאָרמאַלע WordPress אינסטאַלאַציע אין אַ WordPress Multisite נעץ. דאָס נעמט אַוועק די נויט צו רעדאַגירן האַנטלעך `wp-config.php` אָדער לויפֿן דאַטאַבאַזע־באַפֿעלן.

:::tip
אויב אײַער WordPress אינסטאַלאַציע לויפֿט שוין ווי אַ Multisite נעץ, קענט איר איבערשפּרינגען דעם שריט אינגאַנצן. דער וועגווייזער דערשײַנט נאָר ווען Multisite איז נאָך נישט אַקטיווירט.
:::

## ווען דערשײַנט דער וועגווייזער? {#when-does-the-wizard-appear}

ווען איר אַקטיווירט Ultimate Multisite אויף אַ נאָרמאַלער (נישט-Multisite) WordPress אינסטאַלאַציע, דערקענט דער plugin אַז Multisite איז נישט אַקטיווירט און ווײַזט אײַך אויטאָמאַטיש איבער צום Multisite אויפֿשטעלונג־וועגווייזער אַנשטאָט דעם געוויינטלעכן אויפֿשטעלונג־וועגווייזער.

איר קענט אויך צוקומען דערצו דירעקט בײַ **WP Admin > Ultimate Multisite > Multisite Setup**.

## פֿאָרויסבאַדינגונגען {#prerequisites}

איידער איר לויפֿט דעם וועגווייזער, מאַכט זיכער:

- איר האָט **אַדמיניסטראַטאָר־צוטריט** צו אײַער WordPress אינסטאַלאַציע
- די `wp-config.php` טעקע פֿון אײַער סערווער איז **שרײַבבאַר** דורך דעם וועב־סערווער
- איר האָט אַ **לעצטן באַקאַפּ** פֿון אײַערע טעקעס און דאַטאַבאַזע

:::warning
דער וועגווייזער ענדערט אײַער `wp-config.php` טעקע און שאַפֿט נײַע דאַטאַבאַזע־טאַבעלעס. שאַפֿט שטענדיק אַ באַקאַפּ איידער איר גייט ווײַטער.
:::

## שריט 1: באַגריסונג {#step-1-welcome}

דער ערשטער עקראַן דערקלערט וואָס WordPress Multisite איז און וואָס דער וועגווייזער וועט טאָן:

- אַקטיווירן די Multisite פֿונקציע אין אײַער WordPress קאָנפֿיגוראַציע
- שאַפֿן די נייטיקע נעץ־דאַטאַבאַזע־טאַבעלעס
- צולייגן די נויטיקע multisite קאָנסטאַנטן צו `wp-config.php`
- נעץ־אַקטיווירן Ultimate Multisite איבער דער נעץ

![Multisite אויפֿשטעלונג־וועגווייזער - באַגריסונג־שריט](/img/installation/multisite-wizard/wizard-welcome.png)

קליקט **ווײַטער** כּדי ווײַטערצוגיין.

## שריט 2: נעץ־קאָנפֿיגוראַציע {#step-2-network-configuration}

דער שריט בעט אײַך קאָנפֿיגורירן אײַערע נעץ־איינשטעלונגען.

### פּלאַץ־סטרוקטור {#site-structure}

קלײַבט ווי אַזוי אײַערע נעץ־פּלאַץ וועלן ווערן אָרגאַניזירט:

- **סוב־דאָמיינס** (רעקאָמענדירט) — פּלאַץ באַקומען זייער אייגענעם סובדאָמעין, למשל `site1.yourdomain.com`
- **סוב־טעקעס־קאַטאַלאָגן** — פּלאַץ ווערן געשאַפֿן ווי פּאַטס, למשל `yourdomain.com/site1`

:::note
אויב איר קלײַבט סוב־דאָמיינס, וועט איר דאַרפֿן קאָנפֿיגורירן **wildcard DNS** און אַ **wildcard SSL certificate** פֿאַר אײַער דאָמעין. רובֿ פאַרוואַלטעטע WordPress האָסטס באַהאַנדלען דאָס אויטאָמאַטיש. זעט [Ultimate Multisite 101](./ultimate-multisite-101) פֿאַר אַ פּרטימדיקן פֿאַרגלײַך.
:::

### נעץ־טיטל {#network-title}

אַרײַנגעבן אַ נאָמען פֿאַר אײַער נעץ. לויט דער פֿעליקייט איז דאָס אײַער איצטיקער פּלאַץ־טיטל מיט "Network" צוגעלייגט. איר קענט דאָס שפּעטער טוישן פֿון די נעץ־איינשטעלונגען.

### נעץ־אַדמין אימעיל {#network-admin-email}

די אימעיל־אַדרעס געניצט פֿאַר נעץ־אַדמיניסטראַציע־מעלדונגען. לויט דער פֿעליקייט איז דאָס די אימעיל־אַדרעס פֿון דעם איצטיקן באַניצער.

![Multisite אויפֿשטעלונג־וועגווייזער - נעץ־קאָנפֿיגוראַציע](/img/installation/multisite-wizard/wizard-network-configuration.png)

נאָך אויספֿילן די פֿעלדער, קליקט **ווײַטער** כּדי ווײַטערצוגיין.

## שריט 3: אינסטאַלאַציע {#step-3-installation}

קליקט דעם **אינסטאַלירן** קנעפּל כּדי אָנצוהייבן. דער וועגווייזער פֿירט אויס פֿינף אויטאָמאַטישע שריט אין סדר, יעדער ווײַזנדיק זײַן פֿאָרשריט אין רעאַלער צײַט:

![Multisite אויפֿשטעלונג־וועגווייזער - אינסטאַלאַציע ווארטנדיק](/img/installation/multisite-wizard/wizard-installation-pending.png)

| שריט | באַשרײַבונג |
|------|-------------|
| **אַקטיווירן Multisite** | לייגט צו דעם `WP_ALLOW_MULTISITE` קאָנסטאַנט צו `wp-config.php` |
| **שאַפֿן נעץ** | שאַפֿט די multisite דאַטאַבאַזע־טאַבעלעס (`wp_site`, `wp_sitemeta`, `wp_blogs`, אאַז״ו) און באַפֿילט זיי מיט אײַער נעץ־קאָנפֿיגוראַציע |
| **דערהײַנטיקן קאָנפֿיגוראַציע** | לייגט צו די לעצטע multisite קאָנסטאַנטן צו `wp-config.php` (`MULTISITE`, `SUBDOMAIN_INSTALL`, `DOMAIN_CURRENT_SITE`, אאַז״ו) |
| **פֿאַרריכטן קיכלעך** | מאַכט זיכער אַז דער פּלאַץ־URL איז ריכטיק אין די נעץ־מעטאַדאַטן כּדי צו פֿאַרמײַדן אויטענטיפֿיקאַציע־פּראָבלעמען נאָך אַקטיוואַציע |
| **נעץ־אַקטיווירן Plugin** | נעץ־אַקטיווירט Ultimate Multisite אַזוי אַז עס לויפֿט איבער דער גאַנצער נעץ |

יעדער שריט ווײַזט איינעם פֿון די דאָזיקע סטאַטוסן:

- **ווארטנדיק** — וואַרט צו ווערן באַאַרבעט
- **אינסטאַלירט...** — לויפֿט איצט
- **הצלחה!** — געענדיקט מיט הצלחה
- **פֿעלער־מעלדונג** — אַ פֿעלער איז פֿאָרגעקומען (די מעלדונג וועט באַשרײַבן דעם ענין)

ווען אַלע שריט ענדיקן זיך מיט הצלחה, וועט איר זען אַ גרינעם "הצלחה!" סטאַטוס פֿאַר יעדער פּונקט:

![Multisite אויפֿשטעלונג־וועגווייזער - אינסטאַלאַציע פֿאַרענדיקט](/img/installation/multisite-wizard/wizard-installation-complete.png)

דער וועגווייזער וועט דערנאָך אויטאָמאַטיש ווײַטערגיין צום פֿאַרענדיקונג־עקראַן.

## שריט 4: פֿאַרענדיקט {#step-4-complete}

ווען די אינסטאַלאַציע איז פֿאַרענדיקט, וועט איר זען אַ הצלחה־מעלדונג וואָס באַשטעטיקט אַז WordPress Multisite איז אַקטיווירט געוואָרן.

![Multisite אויפֿשטעלונג־וועגווייזער - אויפֿשטעלונג פֿאַרענדיקט](/img/installation/multisite-wizard/wizard-complete.png)

איר קענט איצט ווײַטערגיין מיטן Ultimate Multisite אויפֿשטעלונג־וועגווייזער כּדי צו קאָנפֿיגורירן אײַער WaaS פּלאַטפֿאָרמע (פֿירמע־דעטאַלן, פֿעליק־אינהאַלט, צאָלונג־גייטווייז, אאַז״ו).

:::note
נאָך דעם ווי די multisite אינסטאַלאַציע ענדיקט זיך, וועט אײַער בלעטערער איבערפֿירן דורך דעם נײַ־אַקטיווירטן נעץ־אַדמין. איר מעגט דאַרפֿן זיך ווידער אַרײַנלאָגירן, ווײַל די אויטענטיפֿיקאַציע־קיכלעך ווערן דערהײַנטיקט פֿאַר דער multisite סבֿיבֿה.
:::

## האַנטלעכער אויפֿשטעלונג־צוריקפֿאַל {#manual-setup-fallback}

אויב דער וועגווייזער קען נישט שרײַבן צו אײַער `wp-config.php` טעקע (צוליב טעקע־דערלויבענישן אָדער סערווער־באַגרענעצונגען), וועט ער ווײַזן דעם פּינקטלעכן code וואָס איר דאַרפֿט צולייגן האַנטלעך:

1. די **wp-config.php קאָנסטאַנטן** וואָס דאַרפֿן ווערן צוגעלייגט אויבן דער `/* That's all, stop editing! */` שורה
2. די **.htaccess איבערשרײַב־כללים** וואָס פּאַסן צו אײַער אויסגעקליבענער פּלאַץ־סטרוקטור (סובדאָמעין אָדער סובקאַטאַלאָג)

נאָך מאַכן די האַנטלעכע ענדערונגען, דערפֿרישט די זײַט און דער וועגווייזער וועט דערקענען אַז multisite איז איצט אַקטיוו.

## פּראָבלעם־לייזונג {#troubleshooting}

### דער וועגווייזער זאָגט אַז wp-config.php איז נישט שרײַבבאַר {#the-wizard-says-wp-configphp-is-not-writable}

אײַער וועב־סערווער־פּראָצעס דאַרף שרײַב־דערלויבעניש צו דער `wp-config.php` טעקע. איר קענט אָדער:

- צײַטווײַליק טוישן די טעקע־דערלויבענישן צו `644` אָדער `666`
- נוצן די האַנטלעכע אויפֿשטעלונג־צוריקפֿאַל־אינסטרוקציעס צוגעשטעלט דורך דעם וועגווייזער
- בעטן אײַער האָסטינג־צושטעלער פֿאַר הילף

### פּלאַץ זענען נישט צוטריטלעך נאָך אויפֿשטעלונג (סובדאָמיינס) {#sites-are-not-accessible-after-setup-subdomains}

אויב איר האָט אויסגעקליבן די סובדאָמעין־סטרוקטור, דאַרפֿט איר קאָנפֿיגורירן **wildcard DNS** פֿאַר אײַער דאָמעין. לייגט צו אַ DNS רעקאָרד:

```
Type: A (or CNAME)
Host: *
Value: [your server IP]
```

פֿרעגט בײַ אײַער האָסטינג־פּראָוויידער אויב איר זענט נישט זיכער ווי אַזוי דאָס צו קאָנפֿיגורירן.

### אָטענטיפֿיקאַציע־פּראָבלעמען נאָך דער אײַנשטעלונג {#authentication-issues-after-setup}

אויב איר ווערט אַרויסגעלאָגט אָדער איר דערלעבט cookie־טעותים נאָך דער multisite־אײַנשטעלונג:

1. מעקט אויס אײַערע בלעטערער־cookies פֿאַר דעם פּלאַץ
2. לאָגט זיך ווידער אַרײַן בײַ `yourdomain.com/wp-login.php`
3. אויב דער פּראָבלעם בלײַבט, קאָנטראָלירט אַז אײַער `wp-config.php` האָט נישט `COOKIE_DOMAIN` באַשטימט אויף `false` — דאָס איז אַ באַקאַנטער פּראָבלעם אויף subdomain multisite־אינסטאַלאַציעס

### אַ שריט איז דורכגעפֿאַלן בעת דער אינסטאַלאַציע {#a-step-failed-during-installation}

אויב איינער פֿון די אינסטאַלאַציע־שריט ווײַזט אַ טעות:

1. פֿאַרמערקט די געוויזענע טעות־מעלדונג
2. גייט צוריק צום קאָנפֿיגוראַציע־שריט און פּרוּווט ווידער
3. אויב דער טעות בלײַבט, קאָנטראָלירט אײַער סערווער'ס PHP טעות־לאָג פֿאַר מער פּרטים
4. שריט וואָס זענען שוין פֿאַרענדיקט געוואָרן וועלן איבערגעהיפּט ווערן בײַ ווײַטערדיקע לויפֿן (דער אינסטאַלירער איז idempotent)
