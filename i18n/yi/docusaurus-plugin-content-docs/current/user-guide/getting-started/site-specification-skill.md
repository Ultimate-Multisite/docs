# זײַט־ספּעציפיקאַציע־פֿעיִקייט

די **זײַט־ספּעציפיקאַציע־פֿעיִקייט** איז אַ סטרוקטורירטער צוגאַנג צו כאַפּן אײַער זײַטס צילן, עולם, און בראַנד־אידענטיטעט. די אינפֿאָרמאַציע ווערט אויפֿגעהיט אין אײַער **site_brief** זכּרון, אויף וועלכן agents באַציִען זיך איבער סעסיעס כּדי צו צושטעלן קאָנסיסטענטע, קאָנטעקסט־באַוווּסטע הילף.

## וואָס איז זײַט־ספּעציפיקאַציע? {#what-is-site-specification}

זײַט־ספּעציפיקאַציע איז דער פּראָצעס פֿון דאָקומענטירן:

- **זײַט־ציל**: וואָס אײַער זײַט טוט און פֿאַר וואָס זי עקזיסטירט
- **ציל־עולם**: ווער באַזוכט אײַער זײַט און וואָס זיי דאַרפֿן
- **בראַנד־אידענטיטעט**: אײַערע קאָלירן, טאָן, און וויזועלער סטיל
- **געשעפֿט־צילן**: ווי הצלחה זעט אויס פֿאַר אײַער זײַט
- **אינהאַלט־סטרוקטור**: ווי אײַער זײַט איז אָרגאַניזירט

די ספּעציפיקאַציע ווערט אײַער **site_brief**, אַ שטענדיקער זכּרון וואָס agents ניצן צו פֿאַרשטיין דעם קאָנטעקסט פֿון אײַער זײַט.

## פֿאַר וואָס נוצן זײַט־ספּעציפיקאַציע? {#why-use-site-specification}

### קאָנסיסטענץ איבער סעסיעס {#consistency-across-sessions}

אָן site_brief וואָלט איר געדאַרפֿט ווידער דערקלערן דעם ציל פֿון אײַער זײַט יעדעס מאָל ווען איר הייבט אָן אַ נײַע סעסיע. דערמיט פֿאַרשטייען agents גלײַך:

- די צילן און עולם פֿון אײַער זײַט
- אײַערע בראַנד־קאָלירן און טאָן
- אײַער אינהאַלט־סטרוקטור
- אײַערע געשעפֿט־אָביעקטיוון

### בעסערע רעקאָמענדאַציעס {#better-recommendations}

Agents נוצן אײַער site_brief צו:

- פֿאָרשלאָגן פֿעיִקייטן וואָס פּאַסן צום ציל פֿון אײַער זײַט
- רעקאָמענדירן אינהאַלט־סטרוקטורן וואָס שטימען מיט אײַערע צילן
- פֿאָרלייגן דיזײַנס וואָס זענען קאָנסיסטענט מיט אײַער בראַנד
- אויסמײַדן פֿאָרשלאָגן אומפּאַסיקע פֿעיִקייטן

### גיכערער אַרײַנפֿיר {#faster-onboarding}

נײַע agents (אָדער agents אין נײַע סעסיעס) קענען זיך גיך אַרײַנאַרבעטן דורך לייענען אײַער site_brief אַנשטאָט צו פֿרעגן קלאַריפֿיצירנדיקע פֿראַגעס.

## אָנהייבן זײַט־ספּעציפיקאַציע {#initiating-site-specification}

### בעת Theme Builder אַרײַנפֿיר {#during-theme-builder-onboarding}

די זײַט־ספּעציפיקאַציע־פֿעיִקייט ווערט אויטאָמאַטיש אָנגעהויבן בעת דעם **Theme Builder onboarding flow**. דער Setup Assistant agent פֿרעגט פֿראַגעס און בויט אײַער site_brief.

### מאַנועלער אָנהייב {#manual-initiation}

איר קענט אָנהייבן זײַט־ספּעציפיקאַציע אין יעדן מאָמענט:

```
"לאָמיר דעפֿינירן מײַן זײַט־ספּעציפיקאַציע"
```

אָדער

```
"העלף מיר שאַפֿן אַ זײַט־בריוו"
```

## דער זײַט־ספּעציפיקאַציע־פּראָצעס {#the-site-specification-process}

### שריט 1: זײַט־ציל {#step-1-site-purpose}

דער agent פֿרעגט:

```
וואָס איז דער הויפּטציל פֿון אײַער זײַט?
- אינטערנעץ־האַנדל־קראָם
- בלאָג אָדער אינהאַלט־זײַט
- פּאָרטפֿאָליאָ אָדער אויסשטעלונג
- SaaS אַפּליקאַציע
- קהילה אָדער פֿאָרום
- אַנדערש: [באַשרײַבט]
```

איר קענט אויסקלײַבן אַ קאַטעגאָריע אָדער באַשרײַבן אײַער אייגענעם ציל.

### שריט 2: ציל־עולם {#step-2-target-audience}

```
ווער איז אײַער הויפּט־עולם?
- קאָנסומענטן / אַלגעמיינער עולם
- געשעפֿט־פּראָפֿעסיאָנאַלן
- אַנטוויקלער / טעכנישע באַניצער
- סטודענטן / מחנכים
- אַנדערש: [באַשרײַבט]

וואָס זענען זייערע הויפּט־באַדערפֿענישן?
```

### שריט 3: בראַנד־אידענטיטעט {#step-3-brand-identity}

```
וואָס זענען אײַערע בראַנד־קאָלירן?
- הויפּט־קאָליר: [קאָליר־אויסקלײַבער אָדער hex code]
- צווייטיקע קאָליר: [קאָליר־אויסקלײַבער אָדער hex code]
- אַקצענט־קאָליר: [אָפּציאָנעל]

ווי וואָלט איר באַשריבן אײַער בראַנד־טאָן?
- פּראָפֿעסיאָנעל / קאָרפּאָראַטיוו
- שעפֿעריש / קינסטלעריש
- שפּילעוודיק / אומפֿאָרמעל
- מינימאַל / מאָדערן
- וואַרעם / פֿרײַנדלעך
```

### שריט 4: געשעפֿט־צילן {#step-4-business-goals}

```
ווי זעט הצלחה אויס פֿאַר אײַער זײַט?
- גענערירן לידס
- פֿאַרקויפֿן פּראָדוקטן
- בויען אַ קהילה
- טיילן וויסן
- באַשטעטיקן אויטאָריטעט
- אַנדערש: [באַשרײַבט]

וואָס איז אײַער הויפּט־מעטריק?
- הכנסה
- באַניצער־באַטייליקונג
- אינהאַלט־דערגרייכונג
- קאָנווערסיעס
- אַנדערש
```

### שריט 5: אינהאַלט־סטרוקטור {#step-5-content-structure}

```
ווי איז אײַער אינהאַלט אָרגאַניזירט?
- פֿלאַך (אַלער אינהאַלט אויף דעם זעלבן מדרגה)
- היעראַרכיש (קאַטעגאָריעס און אונטערקאַטעגאָריעס)
- כראָנאָלאָגיש (בלאָג־סטיל)
- פּראָדוקט־באַזירט (קאַטאַלאָג)
- אַנדערש: [באַשרײַבט]

וואָס פֿאַר אינהאַלט־טיפּן ניצט איר?
- בלאָג־אַרטיקלען
- פּראָדוקט־זײַטן
- פֿאַל־שטודיעס
- דאָקומענטאַציע
- ווידעאָס
- אַנדערש
```

## אײַער site_brief זכּרון {#your-sitebrief-memory}

נאָכן פֿאַרענדיקן זײַט־ספּעציפיקאַציע, ווערט אײַער אינפֿאָרמאַציע אויפֿגעהיט ווי **site_brief** אין אײַער agent זכּרון. דאָס איז אַ סטרוקטורירטער רעקאָרד וואָס אַנטהאַלט:

```json
{
  "site_purpose": "E-commerce store for handmade jewelry",
  "target_audience": "Women aged 25-45, interested in sustainable fashion",
  "brand_colors": {
    "primary": "#8B4513",
    "secondary": "#D2B48C",
    "accent": "#2F4F4F"
  },
  "brand_tone": "Warm, artisanal, sustainable",
  "business_goals": ["Generate sales", "Build brand loyalty"],
  "primary_metric": "Revenue",
  "content_structure": "Product catalog with blog",
  "content_types": ["Product pages", "Blog posts", "Customer stories"]
}
```

## באַקוקן און דערהייַנטיקן אײַער site_brief {#viewing-and-updating-your-sitebrief}

### באַקוקט אײַער site_brief {#view-your-sitebrief}

פֿרעגט דעם agent:

```
"ווײַז מיר מײַן זײַט־בריוו"
```

אָדער

```
"וואָס ווייסטו וועגן מײַן זײַט?"
```

דער agent וועט ווײַזן אײַער אויפֿגעהיטענע ספּעציפיקאַציע.

### דערהייַנטיקט אײַער site_brief {#update-your-sitebrief}

אויב אײַער זײַט אַנטוויקלט זיך, קענט איר עס דערהייַנטיקן:

```
"דערהייַנטיק מײַן זײַט־בריוו: מיר צילן איצט אויף B2B קונים"
```

אָדער

```
"דערפֿריש מײַן זײַט־ספּעציפיקאַציע"
```

דאָס וועט ווידער דורכפֿירן די זײַט־ספּעציפיקאַציע־פֿעיִקייט מיט אײַער עקזיסטירנדיקער אינפֿאָרמאַציע ווי אַ אָנהייב־פּונקט.

## ווי agents נוצן site_brief {#how-agents-use-sitebrief}

### דיזײַן־רעקאָמענדאַציעס {#design-recommendations}

ווען איר בעט דיזײַן־ענדערונגען, באַציִען זיך agents אויף אײַער site_brief:

```
איר: "דיזײַן איבער מײַן היים־זײַט"
Agent: "באַזירט אויף אײַער זײַט־בריוו, צילט איר אויף פֿרויען אין עלטער 25-45
מיט אַ וואַרעמען, האַנטווערקלעכן טאָן. איך וועל שאַפֿן אַ דיזײַן וואָס אונטערשטרײַכט
אײַער האַנטגעמאַכטע צירונג און אײַערע סאַסטיינאַבעלע ווערטן."
```

### פֿעיִקייט־פֿאָרשלאָגן {#feature-suggestions}

Agents פֿאָרשלאָגן פֿעיִקייטן וואָס פּאַסן מיט אײַערע צילן:

```
איר: "וואָס פֿאַר פֿעיִקייטן זאָל איך צולייגן?"
Agent: "פֿאַר אַן אינטערנעץ־האַנדל־קראָם פֿאָקוסירט אויף קונה־לויאַליטעט,
וואָלט איך רעקאָמענדירט: לויאַליטעט־פּראָגראַם, קונה־באריכטן, בליצפּאָסט־נײַעסבריוו,
און פּראָדוקט־רעקאָמענדאַציעס."
```

### אינהאַלט־סטרוקטור {#content-structure}

Agents פֿאָרלייגן אינהאַלט־אָרגאַניזאַציע באַזירט אויף אײַער סטרוקטור:

```
איר: "ווי זאָל איך אָרגאַניזירן מײַן בלאָג?"
Agent: "ווײַל איר ניצט אַ פּראָדוקט־קאַטאַלאָג מיט אַ בלאָג, וואָלט איך פֿאָרגעשלאָגן
צו אָרגאַניזירן אַרטיקלען לויט פּראָדוקט־קאַטעגאָריע און שאַפֿן 'סטילינג־גוידעס'
וואָס ווײַזן עטלעכע פּראָדוקטן צוזאַמען."
```

## בעסטע פּראַקטיקעס {#best-practices}

### זײַט ספּעציפֿיש {#be-specific}

אַנשטאָט "אַלגעמיינער עולם," באַשרײַבט אײַער אמתן עולם:

- ✓ "פֿרויען אין עלטער 25-45, אינטערעסירט אין סאַסטיינאַבעלער מאָדע"
- ✗ "אַלעמען"

### דערהייַנטיקט רעגולער {#update-regularly}

ווי אײַער זײַט אַנטוויקלט זיך, דערהייַנטיקט אײַער site_brief:

- ווען איר דרייט זיך צו אַ נײַעם עולם
- ווען איר לייגט צו נײַע פּראָדוקט־ליניעס
- ווען אײַער בראַנד־אידענטיטעט ענדערט זיך
- ווען אײַערע געשעפֿט־צילן פֿאַרשיבן זיך

### ניצט קאָנסיסטענטע טערמינאָלאָגיע {#use-consistent-terminology}

ניצט די זעלבע טערמינען איבער סעסיעס:

- ✓ זאָגט שטענדיק "sustainable jewelry" (נישט "eco-friendly jewelry" און "green products")
- ✓ באַציט זיך קאָנסעקווענט צו אײַער עולם אויף דער זעלבער אופֿן

### אַרײַננעמען קאָנטעקסט {#include-context}

גיט הינטערגרונט וואָס העלפֿט agents פֿאַרשטיין אײַערע באַשלוסן:

- "מיר צילן אויף פּראָפֿעסיאָנאַלן וואָס שעצן קוואַליטעט העכער פֿון פּרײַז"
- "אונדזער עולם איז טעכנאָלאָגיש-באַהאַוונט און דערוואַרט מאָדערנעם פּלאַן"
- "מיר זענען אַ bootstrapped startup, דעריבער דאַרפֿן מיר קאָסטן-עפֿעקטיווע לייזונגען"

## באַציִונג צו Theme Builder אָנבאָרדינג {#relationship-to-theme-builder-onboarding}

די Site Specification פֿעיִקייט איז אינטעגרירט אין דעם **Theme Builder אָנבאָרדינג־פֿלוס**. ווען איר פֿאַרענדיקט אָנבאָרדינג, ווערט אײַער site_brief אויטאָמאַטיש געשאַפֿן מיט דער אינפֿאָרמאַציע וואָס איר האָט צוגעשטעלט.

איר קענט אויך לויפֿן Site Specification אומאָפּהענגיק אויב איר ווילט:

- פֿאַרפֿײַנערן אײַער ספּעציפֿיקאַציע נאָך דער ערשט־אײַנשטעלונג
- דערהײַנטיקן אײַער site brief ווען אײַער וועבזײַט אַנטוויקלט זיך

## פּראָבלעמען־לייזונג {#troubleshooting}

**מײַן site_brief ווערט נישט געניצט**

**דער agent מאַכט רעקאָמענדאַציעס וואָס פּאַסן נישט צו מײַן site_brief**
