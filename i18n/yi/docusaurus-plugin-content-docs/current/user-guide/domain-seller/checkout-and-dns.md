# ויסשרײַב־פֿעלד און קונה DNS־פֿאַרוואַלטונג {#checkout-field-and-customer-dns-management}

## דאָס דאָמיין־אויסקלייב ויסשרײַב־פֿעלד {#the-domain-selection-checkout-field}

דאָס **דאָמיין־אויסקלייב** פֿעלד איז אַ ויסשרײַב־עלעמענט וואָס גיט קונים אַן אויסקלייב ווי אַזוי צו באַקומען זייער פּלאַץ־דאָמיין. לייגט עס צו צו יעדן ויסשרײַב־פֿאָרעם כּדי צו דערמעגלעכן דאָמיין־פֿאַרקויף.

### צולייגן דאָס פֿעלד צו אַ ויסשרײַב־פֿאָרעם {#adding-the-field-to-a-checkout-form}

1. גייט צו **Network Admin › Ultimate Multisite › ויסשרײַב־פֿאָרעמען**
2. עפֿנט אָדער שאַפֿט אַ ויסשרײַב־פֿאָרעם
3. אינעם ויסשרײַב־רעדאַקטאָר, קוועטשט **לייג צו פֿעלד**
4. קלײַבט **דאָמיין־אויסקלייב** פֿון דער פֿעלד־ליסטע
5. קאָנפֿיגורירט די פֿעלד־אָפּציעס (זעט אונטן)
6. היט דעם פֿאָרעם

### פֿעלד־אָפּציעס {#field-options}

**דאָמיין־מאָדעס** — קלײַבט וועלכע קוויטלעך דער קונה זעט. יעדער מאָדע קען באַזונדער דערמעגלעכט אָדער אויסגעלאָשן ווערן:

| מאָדע | וואָס עס טוט |
|---|---|
| **סובדאָמיין** | דער קונה ניצט אַ בחינם סובדאָמיין אויף אײַער נעץ (למשל, `mysite.yournetwork.com`). קיין צאָלונג איז נישט נייטיק. |
| **רעגיסטרירן נײַעם דאָמיין** | דער קונה זוכט אַ נײַעם דאָמיין און רעגיסטרירט אים דורך אײַער קאָנפֿיגורירטן צושטעלער. ניצט דעם פּאַסיקן דאָמיין־פּראָדוקט פֿאַר פּרײַזן. |
| **עקזיסטירנדיקער דאָמיין** | דער קונה פֿאַרבינדט אַ דאָמיין וואָס ער פֿאַרמאָגט שוין. קיין רעגיסטראַציע־אָפּצאָל. דער דאָמיין ווערט אויטאָמאַטיש פֿאַרבונדן מיט זײַן פּלאַץ. |

**פעליקייַט־מאָדע** — ווען אַלע דרײַ מאָדעס זענען דערמעגלעכט, וועלכער קוויטל עפֿנט זיך ערשט. שטעלט אויף **סובדאָמיין** כּדי דאָמיין־רעגיסטראַציע זאָל בלײַבן אָפּציאָנעל, אָדער **רעגיסטרירן נײַעם דאָמיין** כּדי צו מוטיקן קויפֿן.

**דאָמיין־פּראָדוקט** — איר קענט אָפּציאָנעל צוטשעפּען דאָס פֿעלד צו אַ ספּעציפֿישן דאָמיין־פּראָדוקט. אויב נישט באַשטימט, קלײַבט דער צוגאָב אויטאָמאַטיש דעם פּאַסיקן פּראָדוקט לויט דעם TLD וואָס דער קונה זוכט.

### רעגיסטראַנט־קאָנטאַקט־פֿעלדער {#registrant-contact-fields}

ווען אַ קונה קלײַבט דעם **רעגיסטרירן נײַעם דאָמיין** קוויטל, לייגט דער ויסשרײַב־פֿאָרעם צו רעגיסטראַנט־קאָנטאַקט־פֿעלדער גלײַך אינעווייניק:

- ערשטער נאָמען / לעצטער נאָמען
- בליצפּאָסט־אַדרעס
- אַדרעס (שורה 1, שטאָט, שטאַט/פּראָווינץ, פּאָסטקאָד, לאַנד)
- טעלעפֿאָן־נומער

די ווערן געפֿאָדערט דורך אַלע רעגיסטראַרן און וואַלידירט איידער דער רעגיסטראַציע API־רוף ווערט געמאַכט. טעלעפֿאָן־נומערן ווערן אויטאָמאַטיש פֿאָרמאַטירט צום `+CC.NNN` אינטערנאַציאָנאַלן פֿאָרמאַט וואָס רעגיסטראַרן דערוואַרטן.

### אויטאָמאַטיש געשאַפֿענע פּלאַץ־URL {#auto-generated-site-url}

ווען אַ קונה רעגיסטרירט אָדער פֿאַרבינדט אַ דאָמיין, ווערט דאָס פּלאַץ־URL פֿעלד אויטאָמאַטיש אָנגעפֿילט פֿון דעם אויסגעקליבענעם דאָמיין. קונים דאַרפֿן נישט אָנפֿילן קיין באַזונדער URL־פֿעלד.

### זוך־פֿירונג {#search-behaviour}

- דאָמיין־פֿאַראַנענקייט ווערט אָפּגעקוקט אין אמת־צײַט מיט AJAX בשעת דער קונה טיפּט
- אַלטערנאַטיווע TLD־פֿאָרשלאָגן ווערן געוויזן ווען דער געוואונטשענער דאָמיין איז נישט פֿאַראַן
- פּרײַזן ווערן געבראַכט לעבן און קלאָר געוויזן (רעגיסטראַציע־פּרײַז, באַנײַונג־פּרײַז, אָפּציאָנעלע WHOIS פּריוואַטקייט־אָפּצאָל)
- קופּאָן־קאָדן גילטן אויף דאָמיין־פּראָדוקטן אַזוי ווי אויף יעדן אַנדערן פּראָדוקט

**אײַנשטעלן זוך־רעאַקציע־גיכקייט:**

```php
// Increase debounce delay (milliseconds) to reduce API calls on slow connections
add_filter('wu_domain_seller_search_delay', function($delay) {
    return 800; // default: 500
});
```

**צולייגן אייגענע פֿעלדער צום דאָמיין־זוך־פֿאָרעם:**

```php
add_filter('wu_checkout_form_register_domain_form_fields', function($fields) {
    $fields['custom_note'] = [
        'type'  => 'text',
        'label' => 'Additional notes',
    ];
    return $fields;
});
```

## קונה DNS־פֿאַרוואַלטונג {#customer-dns-management}

קונים קענען פֿאַרוואַלטן DNS־רעקאָרדס פֿאַר זייערע רעגיסטרירטע דאָמיינס פֿון דער **מײַן Account** זײַט, אונטערן אײַנטראָג פֿון זייער דאָמיין.

### געשטיצטע רעקאָרד־טיפּן {#supported-record-types}

| טיפּ | באַניץ |
|---|---|
| **A** | פֿאַרבינדן האָסטנאָמען מיט IPv4 אַדרעס |
| **AAAA** | פֿאַרבינדן האָסטנאָמען מיט IPv6 אַדרעס |
| **CNAME** | שאַפֿן אַן אַליאַס וואָס ווײַזט אויף אַן אַנדערן האָסטנאָמען |
| **MX** | שטעלן בליצפּאָסט־אויסטויש־סערווער |
| **TXT** | צולייגן SPF, DMARC, באַשטעטיקונג, אָדער אַנדערע טעקסט־רעקאָרדס |

### וועלכע צושטעלער שטיצן DNS־פֿאַרוואַלטונג? {#which-providers-support-dns-management}

DNS־פֿאַרוואַלטונג (צולייגן, רעדאַקטירן, אויסמעקן רעקאָרדס) איז פֿאַראַן מיט **OpenSRS**, **ResellerClub**, **Enom**, **HostAfrica**, און **Openprovider**. **Hostinger** דאָמיינס קענען דערהײַנטיקן נאָמען־סערווערס דורך דאָמיין־פֿאַרקויפֿער; DNS־רעקאָרדס פֿאַר געהאָסטעטע דאָמיינס ווערן פֿאַרוואַלטט דורך דער קערן Hostinger דאָמיין־פֿאַרבינדונג־אינטעגראַציע. Namecheap, GoDaddy, און NameSilo דאָמיינס ווײַזן סטאַטוס און אויסלויף־אינפֿאָרמאַציע, אָבער DNS מוז פֿאַרוואַלטט ווערן דירעקט אין דעם קאָנטראָל־פּאַנעל פֿונעם רעגיסטראַר.

### פעליקייַט DNS־רעקאָרדס {#default-dns-records}

איר קענט קאָנפֿיגורירן פעליקייַט DNS־רעקאָרדס וואָס ווערן אויטאָמאַטיש אָנווענדט ווען אַ דאָמיין ווערט רעגיסטרירט. גייט צו **אײַנשטעלונגען › דאָמיין־פֿאַרקויפֿער › פעליקייַט DNS־רעקאָרדס**.

פעליקייַט־רעקאָרד־ווערטן שטיצן צוויי טאָקענס:

| טאָקען | פֿאַרביטן מיט |
|---|---|
| `{DOMAIN}` | דער רעגיסטרירטער דאָמיין־נאָמען (למשל, `example.com`) |
| `{SITE_URL}` | דער WordPress פּלאַץ־URL פֿאַרן קונהס פּלאַץ |

**בײַשפּיל — ווײַזט דעם apex דאָמיין און www אויף אײַער סערווער־IP:**

```
Type: A
Name: {DOMAIN}
Value: 203.0.113.10

Type: CNAME
Name: www.{DOMAIN}
Value: {DOMAIN}
```

### אַדמין: אָנקוקן און רעדאַקטירן DNS {#admin-viewing-and-editing-dns}

נעץ־אַדמינען קענען אָנקוקן און רעדאַקטירן DNS־רעקאָרדס פֿאַר יעדן קונה־דאָמיין פֿון דער דאָמיין־רעדאַקטיר־זײַט אין **Network Admin › Ultimate Multisite › דאָמיינס**.
