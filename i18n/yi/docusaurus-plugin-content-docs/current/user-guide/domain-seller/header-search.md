# קעפּל־דאָמיין־זוכן

ניצט דעם אויפֿשטעל ווען איר ווילט אַ קליינע פֿאָרעם אין קעפּל וואָס הייבט אָן אַ דאָמיין־זוכן, און לאָזט דערנאָך דעם קונה אויסקלייבן פֿון פֿאַראַנענע דאָמיינס אינעם Ultimate Multisite אויסקויף.

## באַדערפֿענישן {#requirements}

- Ultimate Multisite נעץ־אַקטיוו.
- Multisite Ultimate Domain Seller נעץ־אַקטיוו.
- כאָטש איין אַקטיווע דאָמיין־רעגיסטראַציע־פּראָדוקט מיט:
  - `domain_provider` באַשטימט צו אַ קאָנפֿיגורירטן צושטעלער.
  - געשטיצטע TLDs קאָנפֿיגורירט, למשל `com`, `net`, און `org`.
- אַ גילטיקע אויסקויף־פֿאָרעם וואָס אַנטהאַלט דאָס **דאָמיין־אויסקלייב** פֿעלד.

## אויסקויף־פֿאָרעם {#checkout-form}

1. שאַפֿט אָדער רעדאַקטירט די אויסקויף־פֿאָרעם וואָס די רעגיסטראַציע־בלאַט ניצט.
2. לייגט צו די נאָרמאַלע נייטיקע אויסקויף/Account פֿעלדער, אַרײַנגערעכנט **באַניצערנאָמען**. אַ אויסקויף־פֿאָרעם וואָס אַנטהאַלט בלויז דאָס דאָמיין־פֿעלד ווערט אָפּגעוואָרפֿן דורך Ultimate Multisite וואַלידאַציע.
3. לייגט צו אַ **דאָמיין־אויסקלייב** פֿעלד.
4. שטעלט דעם דאָמיין־אויסקלייב מאָדוס אויף **בלויז רעגיסטרירן** ווען דער פֿלוס זאָל זיך קאָנצענטרירן אויף רעגיסטרירטע דאָמיינס אַנשטאָט פֿרײַע סובדאָמיינס אָדער עקזיסטירנדיקע דאָמיינס.
5. צוטיילט דעם דאָמיין־רעגיסטראַציע־פּראָדוקט צו יענעם פֿעלד.

די רעגיסטראַציע־בלאַט זאָל ווײַזן די אויסקויף־פֿאָרעם, למשל:

```text
[wu_checkout slug="domain-form"]
```

## קעפּל־פֿאָרעם {#header-form}

לייגט צו אַ קליינע `GET` פֿאָרעם אינעם וועבזײַטל־קעפּל וואָס שיקט דעם אַרײַנגעפֿירטן זוך־טערמין צום אויסקויף־בלאַט ווי `domain_name`:

```html
<form class="ums-header-domain-search" action="/register/" method="get">
  <label class="screen-reader-text" for="ums-header-domain-search-input">Search for a domain</label>
  <input id="ums-header-domain-search-input" name="domain_name" type="text" placeholder="Find your domain" autocomplete="off">
  <button type="submit">Search</button>
</form>
```

קלייבט נישט פֿאָרויס אויס קיין דאָמיין אין אייגענעם קעפּל JavaScript. דער קעפּל זאָל בלויז איבערגעבן דעם זוך־טערמין. דער Domain Seller אויסקויף־סקריפּט לייענט `?domain_name=example`, פֿילט אויס דאָס זוך־קעסטל אין אויסקויף, און לויפֿט די פֿאַראַנענקייט־זוכן כּדי דער קונה זאָל קענען אויסקלייבן פֿון די צוריקגעגעבענע דאָמיינס.

## דערוואַרטע אויפֿפֿירונג {#expected-behaviour}

זוכן `example` אינעם קעפּל זאָל עפֿענען:

```text
/register/?domain_name=example
```

דער אויסקויף זאָל דערנאָך ווײַזן אויסקלייבבאַרע רעזולטאַטן אַזעלכע ווי:

- `example.com`
- `example.net`
- `example.org` נישט פֿאַראַן
- אַנדערע דורך דעם צושטעלער געשטיצטע TLDs

נאָך אויסקלייבן אַ פֿאַראַנענעם רעזולטאַט, זאָל דער באַשטעל־קיצער אַנטהאַלטן דעם דאָמיין־רעגיסטראַציע־פּראָדוקט און דעם אויסגעקליבענעם דאָמיין־נאָמען.

## באַשטעטיקונג {#verification}

1. עפֿנט די היים־בלאַט.
2. זוכט אַ פּשוטן נאָמען, למשל `example`.
3. באַשטעטיקט אַז דער אויסקויף URL אַנטהאַלט `?domain_name=example`.
4. באַשטעטיקט אַז דאָס דאָמיין־פֿעלד אין אויסקויף אַנטהאַלט `example`.
5. באַשטעטיקט אַז אַ רשימה פֿון דאָמיין־ברירות דערשײַנט.
6. קליקט **אויסקלייבן** פֿאַר אַ פֿאַראַנענעם דאָמיין.
7. באַשטעטיקט אַז דער באַשטעל־קיצער אַנטהאַלט `Domain Registration - example.com` אָדער דעם אויסגעקליבענעם דאָמיין.

## פּראָבלעמען־לייזונג {#troubleshooting}

- אויב קיין רשימה דערשײַנט נישט, קאָנטראָלירט דעם בלעטערער־נעץ־טאַב פֿאַר `admin-ajax.php?action=wu_domain_search` און באַשטעטיקט אַז עס גיט צוריק נישט־ליידיקע `domains` אָדער `results`.
- אויב די אויסקויף־פֿאָרעם פֿאַלט דורך וואַלידאַציע בעתן אויפֿהיטן, לייגט צו די נייטיקע Account פֿעלדער אַזעלכע ווי **באַניצערנאָמען**.
- אויב אויסקלייבן אַ דאָמיין דערהײַנטיקט נישט דעם קאָרב, באַשטעטיקט אַז `window.wu_checkout_form` עקזיסטירט און אַז די Domain Seller אויסקויף־אַסעטן ווערן געלאָדן אויף דער אויסקויף־בלאַט.
