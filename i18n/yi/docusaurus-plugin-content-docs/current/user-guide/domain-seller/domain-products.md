# Domain פּראָדוקטן און פּרײַזן {#domain-products-and-pricing}

Domain פּראָדוקטן זענען דער אופֿן ווי איר קאָנטראָלירט פּרײַזן, TLDs, WHOIS פּריוואַטקייט, און אויסקלײַב פֿון צושטעלער. יעדער Domain פּראָדוקט איז אַ סטאַנדאַרטער Ultimate Multisite פּראָדוקט מיט אַן עקסטרע **Domain Settings** קוויטל.

## שאַפֿן אַ Domain פּראָדוקט {#creating-a-domain-product}

1. גייט צו **Network Admin › Ultimate Multisite › Products**
2. קליקט **Add New**
3. שטעלט דעם פּראָדוקט־טיפּ אויף **Domain**
4. קאָנפֿיגורירט דעם **Domain Settings** קוויטל (זעט אונטן)
5. היט

Domain פּראָדוקטן דערשײַנען מיט אַ לילאַ **Domain** צייכן אין דער פּראָדוקט־ליסטע און מע קען זיי פֿילטערן ניצנדיק דעם **Domain Products** קוויטל.

## Domain settings קוויטל {#domain-settings-tab}

### צושטעלער {#provider}

קלײַבט אויס וועלכער רעגיסטראַר באַהאַנדלט רעגיסטראַציע פֿאַר דעם פּראָדוקט. עס נעמט לויט פעליקייט דעם גלאבאלן **Default provider** באַשטימט אין Domain Seller סעטטינגס.

### געשטיצטע TLDs {#supported-tlds}

לאָזט ליידיק כּדי צו שאַפֿן אַ **אַלגעמיינער־אויפֿכאַפּ־פּראָדוקט** וואָס גילט פֿאַר אַלע TLDs וואָס ווערן נישט געפּאַסט דורך אַן אַנדער פּראָדוקט.

גיט אַרײַן אַ מיט קאָמעס אָפּגעטיילטע ליסטע פֿון TLDs (למשל, `.com, .net, .org`) כּדי צו שאַפֿן אַ **TLD-ספּעציפֿישן פּראָדוקט** וואָס גילט נאָר פֿאַר יענע עקסטענסיעס.

**ווי פּראָדוקט־פּאַסונג אַרבעט:** ווען אַ קונה זוכט אַ Domain, קלײַבט דער צולייג דעם מערסט ספּעציפֿיש פּאַסיקן פּראָדוקט. אַ פּראָדוקט מיט `.com` אין זײַן TLD ליסטע האָט פּריאָריטעט איבער אַן אַלגעמיינעם־אויפֿכאַפּ־פּראָדוקט. אויב קיין TLD-ספּעציפֿישער פּראָדוקט פּאַסט נישט, ווערט געניצט דער אַלגעמיינער־אויפֿכאַפּ. אויב קיין פּראָדוקטן עקזיסטירן נישט, ווערט די Domain זוכן נישט געוויזן.

### אויפֿשלאָג־טיפּ {#markup-type}

דרײַ מאָדעס קאָנטראָלירן ווי אײַער לאַכקאָדעשדיקער פּרײַז ווערט אויסגערעכנט פֿון דער גראָסיר־קאָסט:

| מאָדע | ווי עס אַרבעט |
|---|---|
| **פּראָצענט** | לייגט צו אַ פּראָצענט אויף דער גראָסיר־קאָסט. אַ 20% אויפֿשלאָג אויף אַ $10 גראָסיר Domain גיט $12. |
| **פֿעסטער אויפֿשלאָג** | לייגט צו אַ פֿעסטן דאָלאַר־סכום. אַ $5 אויפֿשלאָג אויף אַ $10 Domain גיט $15. |
| **פֿעסטער פּרײַז** | איגנאָרירט די גראָסיר־קאָסט אינגאַנצן. רעכנט תּמיד דעם סכום וואָס איר גיט אַרײַן. |

### אַרײַנפֿיר־פּרײַזן {#introductory-pricing}

אַקטיווירט כּדי אָנצובאָטן אַ אַראָפּגעזעצטן ערשט־יאָריקן פּרײַז. שטעלט אַ באַזונדערן **אַרײַנפֿיר־פּרײַז** (דער פּרײַז פֿאַר יאָר 1) צוזאַמען מיטן רעגולערן **באַנײַונג־פּרײַז** (יאָר 2+). דער קונה זעט ביידע פּרײַזן געוויזן בעת checkout, כּדי זיי זאָלן וויסן וואָס צו דערוואַרטן בײַ באַנײַונג.

### WHOIS פּריוואַטקייט {#whois-privacy}

קאָנטראָלירט צי WHOIS פּריוואַטקייט־שוץ ווערט אָנגעבאָטן פֿאַר Domains רעגיסטרירט דורך דעם פּראָדוקט.

| באַשטעטיקונג | אויפֿפֿירונג |
|---|---|
| **פֿאַרקריפּלט** | WHOIS פּריוואַטקייט ווערט קיינמאָל נישט אָנגעבאָטן אָדער אַקטיווירט. |
| **תּמיד אַרײַנגערעכנט** | WHOIS פּריוואַטקייט ווערט אויטאָמאַטיש אַקטיווירט בײַ רעגיסטראַציע אָן קיין אָפּצאָל. |
| **קונה־אויסקלײַב** | אַ checkbox דערשײַנט בעת checkout. שטעלט דעם **פּריוואַטקייט־פּרײַז** כּדי אָפּצורעכענען פּער יאָר, אָדער לאָזט עס אויף $0 כּדי עס אָנצובאָטן בחינם. |

פֿאַר Namecheap, WHOIS פּריוואַטקייט ניצט WhoisGuard (תּמיד בחינם). פֿאַר OpenSRS, ניצט עס דעם OpenSRS פּריוואַטקייט־דינסט (קען האָבן אַ קאָסט בײַם גראָסיר).

## TLD אַרײַנפֿיר און סינק {#tld-import-and-sync}

Domain פּראָדוקטן ווײַזן רעאַל־צײַטיקע גראָסיר־פּרײַזן געפֿעשט פֿון דעם פֿאַרבונדן צושטעלער. כּדי דאָס זאָל אַרבעטן, מוזן TLDs ווערן אַרײַנגעפֿירט.

- **מאַנועלער סינק:** Settings › Domain Seller › Sync TLDs (פּער צושטעלער)
- **אויטאָמאַטישער סינק:** לויפֿט טעגלעך דורך אַ פּלאַנירטע cron אַרבעט איבער אַלע קאָנפֿיגורירטע צושטעלער

נאָך אַ סינק, גייט צו וועלכן Domain פּראָדוקטס Domain Settings קוויטל און ניצט דעם TLD אויסקלײַבער כּדי צו זען פֿאַראַנענע TLDs מיט זייערע איצטיקע גראָסיר־פּרײַזן.

## אויטאָ־באַנײַונג {#auto-renewal}

Domain באַנײַונגען זענען פֿאַרבונדן מיטן קונהס membership סטאַטוס:

- ווען אַ membership ווערט באַנײַט און אַ Domain איז פֿאַרבונדן, ווערט די Domain באַנײַונג אויטאָמאַטיש אַרײַנגעשטעלט אין ריי
- באַנײַונג־פּרוּוון ניצן דעם קונהס אַקטיוון צאָלונג־גייטוויי
- דורכגעפֿאַלענע באַנײַונגען ווערן אויטאָמאַטיש ווידער־פּרוּווט מיט עקספּאָנענציעלן backoff
- בליצבריוו־מעלדונגען ווערן געשיקט פֿאַר מצליחדיקע באַנײַונגען, דורכפֿאַלן, און נאָענטע אויסלויפֿן

בליצבריוו־מוסטער IDs פֿאַר Domain לעבן־ציקל געשעענישן:

| געשעעניש | מוסטער ID |
|---|---|
| Domain רעגיסטרירט | `domain_registered` |
| Domain באַנײַט | `domain_renewed` |
| באַנײַונג דורכגעפֿאַלן | `domain_renewal_failed` |
| Domain לויפֿט באַלד אויס | `domain_expiring_soon` |

## Admin: מאַנועלע Domain רעגיסטראַציע {#admin-manual-domain-registration}

כּדי צו רעגיסטרירן אַ Domain אין נאָמען פֿון אַ קונה אָן אַז זיי זאָלן דורכגיין checkout:

1. גייט צו **Network Admin › Ultimate Multisite › Register Domain**
2. קלײַבט אויס דעם קונה און גיט אַרײַן דעם Domain נאָמען
3. פֿילט אַרײַן רעגיסטראַנט קאָנטאַקט־פּרטים (נאָמען, אַדרעס, טעלעפֿאָן)
4. קליקט **Register**

דער Domain רעקאָרד ווערט געשאַפֿן און פֿאַרבונדן מיטן קונהס Account.
