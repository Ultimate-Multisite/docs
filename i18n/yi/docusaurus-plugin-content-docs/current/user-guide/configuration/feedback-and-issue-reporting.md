# קונה־באַמערקונגען און מעלדן פּראָבלעמען

Gratis AI Agent v1.5.0 פֿירט אַרײַן אַן אײַנגעבויטע סיסטעם פֿאַר באַמערקונגען און מעלדן פּראָבלעמען, וואָס לאָזט סוף־באַניצער אָנווײַזן אומניצלעכע ענטפֿערס און מעלדן פּראָבלעמען גלײַך פֿון דער שמועס־צובינד. די סיסטעם נעמט אַרײַן צושטימונג־פֿאַרוואַלטונג, אַן אויטאָמאַטישן מעלדונג־באַפֿעל, און טריאַזש מיט הילף פֿון קינסטלעך־אינטעליגענץ אויפֿן באַקענד.

## דוימען־אַראָפּ קנעפּל {#thumbs-down-button}

יעדע מעלדונג געשיקט דורך דעם קינסטלעך־אינטעליגענץ אַסיסטענט ווײַזט אַ **דוימען־אַראָפּ** (👎) קנעפּל. ווען אַ באַניצער קליקט דערויף, קען ער באַצייכענען אַן ענטפֿער ווי אומניצלעך אָדער נישט ריכטיק.

- דער קנעפּל דערשײַנט בײַם איבערפֿאָרן דערנעבן יעדער אַסיסטענט־מעלדונג.
- אַ קליק דערויף עפֿנט דעם **צושטימונג־מאָדאַל פֿאַר באַמערקונגען**.
- די באַמערקונג ווערט פֿאַרבונדן מיטן שמועס־פֿאָדעם און דער ספּעציפֿישער מעלדונג.

## צושטימונג־מאָדאַל פֿאַר באַמערקונגען {#feedback-consent-modal}

ווען אַ באַניצער קליקט דעם דוימען־אַראָפּ קנעפּל, דערשײַנט אַ צושטימונג־מאָדאַל איידער סיי וועלכע דאַטן ווערן געשיקט. דער מאָדאַל:

- דערקלערט וועלכע אינפֿאָרמאַציע וועט ווערן געזאַמלט (שמועס־אויסצוג, בלעטערער־קאָנטעקסט).
- בעט דעם באַניצער צו באַשטעטיקן אַז ער שטימט צו צו טיילן די דאַטן.
- גיט אַ פֿרײַ־טעקסט פֿעלד פֿאַרן באַניצער צו באַשרײַבן וואָס איז געגאַנגען נישט ריכטיק.
- אָפערט אַן **אַנולירן** אָפּציע צו פֿאַרמאַכן אָן אַרײַנשיקן.

קיין באַמערקונג ווערט נישט רעקאָרדירט ביז דער באַניצער באַשטעטיקט אויסדריקלעך.

## אויטאָ־אָנפֿרעג באַמערקונגען־באַנער {#auto-prompt-feedback-banner}

בײַם סוף פֿון אַ שמועס קען דער אַסיסטענט ווײַזן אַן **אויטאָ־אָנפֿרעג באַמערקונגען־באַנער** — אַן אומשטערנדיקע מעלדונג וואָס פֿרעגט צי די סעסיע איז געווען נוצלעך.

דער באַנער דערשײַנט אויטאָמאַטיש לויט שמועס־לענג און רעזולטאַט־היוריסטיקעס. ער פֿאַרבינדט צו דעם זעלבן באַמערקונגען־פֿלוס ווי דער דוימען־אַראָפּ קנעפּל. באַניצער קענען פֿאַרמאַכן דעם באַנער אָן צושטעלן באַמערקונגען.

## /report-issue שטריך־באַפֿעל {#report-issue-slash-command}

באַניצער קענען אויסלייזן דעם באַמערקונגען־פֿלוס גלײַך דורך אַרײַנטיפּן `/report-issue` אין דעם שמועס־אַרײַנפֿיר. דער באַפֿעל:

- עפֿנט דעם צושטימונג־מאָדאַל פֿאַר באַמערקונגען גלײַך.
- פֿילט פֿאָרויס דאָס באַשרײַבונג־פֿעלד מיט קאָנטעקסט וועגן דעם איצטיקן שמועס.
- לאָזט באַניצער צולייגן נאָך פּרטים איידער אַרײַנשיקן.

דער `/report-issue` באַפֿעל איז בנימצא אין אַלע שמועס־מאָדעס (אינלײַן, שוועבנדיקער ווידזשעט, גאַנצער־עקראַן).

## טריאַזש מיט הילף פֿון קינסטלעך־אינטעליגענץ {#ai-assisted-triage}

אַרײַנגעשיקטע באַמערקונגען ווערן איבערגעגעבן צו אַ טריאַזש־שיכט מיט קינסטלעך־אינטעליגענץ, וואָס:

- קאַטעגאָריזירט דעם מעלדונג (פֿאַקטישער טעות, אומניצלעכע ענטפֿער, אומפּאַסיקער אינהאַלט, אאַז״ו).
- הענגט אָן באַטעפֿנדיקע שמועס־מעטאַדאַטן.
- שיקט ווײַטער דעם טריאַזש־קיצור צו דעם קאָנפֿיגורירטן **באַמערקונגען־Endpoint** (זעט [אײַנשטעלונגען > פֿאָרגעשריטן](#settings-advanced)).

דאָס רעדוצירט די צײַט פֿון מאַנועלער איבערקוק דורך אַרויסברענגען די מערסט קריטישע פּראָבלעמען ערשט.

## אײַנשטעלונגען > פֿאָרגעשריטן {#settings-advanced}

כדי צו אַקטיווירן דעם באַמערקונגען־באַקענד, קאָנפֿיגורירט די ווײַטערדיקע פֿעלדער אונטער **Gratis AI Agent → אײַנשטעלונגען → פֿאָרגעשריטן**:

| פֿעלד | באַשרײַבונג |
|---|---|
| **באַמערקונגען־Endpoint URL** | דער URL וואָס נעמט אָן POST בקשות מיט באַמערקונגען־פּיילאָודס (JSON). |
| **Feedback API Key** | Bearer token געשיקט אין דעם `Authorization` header פֿאַר אָטענטיפֿיצירן אַרײַנשיקונגען פֿון באַמערקונגען. |

לאָזט ביידע פֿעלדער ליידיק כדי אויסצושאַלטן זאַמלונג פֿון באַמערקונגען. יחידישע באַמערקונגען־קנעפּלעך בלײַבן זעעוודיק פֿאַר באַניצער, אָבער אַרײַנשיקונגען וועלן נישט ווערן ווײַטערגעשיקט.

:::tip
דער באַמערקונגען־endpoint מוז אָננעמען אַ JSON גוף מיט לפּחות `message_id`, `conversation_id`, `feedback_text`, און `triage_category` פֿעלדער. זעט די דאָקומענטאַציע פֿון אײַער endpoint־צושטעלער פֿאַר דער דערוואַרטע סכעמע.
:::
