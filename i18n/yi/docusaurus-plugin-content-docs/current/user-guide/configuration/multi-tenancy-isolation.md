# Multi-Tenancy איזאָלאַציע {#multi-tenancy-isolation}

Ultimate Multisite: Multi-Tenancy 1.2.0 שטיצט פּער-אונטער־פּלאַץ דאַטאַבאַזע־און טעקעסיסטעם־איזאָלאַציע פֿאַר זעלבשטענדיקע טענאַנץ. דאָס האַלט טענאַנט־דאַטן באַזונדער, בעת עס בלײַבט אויפֿהאַלטן נעצווערק־מדרגה פּראָוויזשאַנינג, בילינג, און אַדמיניסטראַציע.

## איזאָלאַציע־סטראַטעגיע {#isolation-strategy}

ניצן זעלבשטענדיקע איזאָלאַציע פֿאַר קאַסטאַמערס וואָס דאַרפֿן שטאַרקערע דאַטן־אָפּטיילונג, באַזונדערע טעקעסיסטעם־סטאָרידזש, אָדער אַ באַזונדערע האָסט־גרענעץ.

יעדער זעלבשטענדיקער טענאַנט זאָל האָבן:

- אַ באַזונדערע טענאַנט־דאַטאַבאַזע אָדער דאַטאַבאַזע־פּרעפֿיקס־סטראַטעגיע וואָס איז באוויליקט פֿאַרן האָסט.
- אַ באַזונדערן טענאַנט־טעקעסיסטעם־וואָרצל.
- אַ טענאַנט־רעגיסטרי־איינטראָג וואָס מאַפּט דעם פּלאַץ צו זײַן דאַטאַבאַזע, וואָרצל־דרך, האָסטנאָמען, און איזאָלאַציע־מאָדעל.
- אַ מיגראַציע־וועריפֿיקאַציע־רעזולטאַט איידער דער טענאַנט ווערט באַטראַכט ווי לעבן.

## דאַטאַבאַזע־האָסט־פֿאַרבינדונג {#database-host-binding}

ווערסיע 1.2.0 ענדערט די סטאַנדאַרט זעלבע־מאַשין האָסט־פֿאַרבינדונג־אופֿפֿירונג פֿאַר זעלבשטענדיקע אינסטאַלאַציעס. זעלבע־מאַשין ווערטן אַזאַ ווי `localhost` ווערן נאָרמאַליזירט כּדי Bedrock, FrankenPHP, און קאָנטיינעריזירטע WordPress אינסטאַלאַציעס זאָלן קענען דערלויבן און וועריפֿיצירן פּערמישאַנז קעגן דעם האָסט־שטריקל וואָס MySQL זעט טאַקע.

בײַם קאָנפֿיגורירן אַ זעלבשטענדיקן טענאַנט:

1. שטעל דעם דאַטאַבאַזע־האָסט אויף דעם ווערט וואָס דער טענאַנט־ראַנטײַם דאַרף.
2. ניצט `localhost` פֿאַר לאָקאַלע סאָקעט־אינסטאַלאַציעס ווען דער האָסט דערוואַרט לאָקאַלע פֿאַרבינדונגען.
3. ניצט `127.0.0.1` אָדער אַ סערוויס־האָסטנאָמען נאָר ווען דער דאַטאַבאַזע־סערווער גיט פּריווילעגיעס צו יענעם האָסט.
4. לויפֿט מיגראַציע־וועריפֿיקאַציע נאָך ענדערן די האָסט־פֿאַרבינדונג.

אויב וועריפֿיקאַציע מעלדט גראַנט־דורכפֿאַלן, פֿאַרגלײַכט די טענאַנט DB באַניצער־גראַנטן מיט דער קאָנפֿיגורירטער האָסט־פֿאַרבינדונג. אַ באַניצער געגראַנט פֿאַר `user@localhost` איז אַנדערש פֿון `user@127.0.0.1` אָדער `user@%`.

## טעקעסיסטעם־וואָרצל {#filesystem-root}

דער טענאַנט־וואָרצל זאָל זײַן סטאַביל איבער רעסטאַרטן און דעפּלוימענטן. פֿאַרמײַדט צײַטווײַליקע מאָונט־דרכים. פֿאַר Bedrock-סטיל אינסטאַלאַציעס, באַשטעטיקט אַז דער טענאַנט־וואָרצל ווײַזט אויף דעם WordPress וועב־וואָרצל וואָס דער טענאַנט־בוטסטראַפּ דערוואַרט, ניט בלויז אויף דעם פּראָיעקט־וואָרצל.

## פּראָוויזשאַנינג־סדר {#provisioning-order}

פֿאַר נײַע זעלבשטענדיקע טענאַנץ, ניצט דעם סדר:

1. שאַפֿט דעם טענאַנט־רעגיסטרי־איינטראָג.
2. שאַפֿט די טענאַנט־דאַטאַבאַזע און דאַטאַבאַזע־באַניצער.
3. בוטסטראַפּט דעם טענאַנט־סכעמע.
4. פּראָוויזשאַנט טענאַנט־באַניצער.
5. קאָנפֿיגורירט טענאַנט־טעקעסיסטעם־דרכים.
6. לויפֿט מיגראַציע־וועריפֿיקאַציע.
7. בײַט רוטינג אָדער DNS נאָך דעם וואָס וועריפֿיקאַציע גייט דורך.

דער סדר פֿאַרמײַדט אַז טיילווײַז איזאָלירטע טענאַנץ זאָלן באַקומען טראַפיק איידער דער דאַטאַבאַזע־שרײַבער, באַניצער, און טעקעסיסטעם זענען גרייט.

## זעלבשטענדיקע קאַסטאַמער־פֿאַרוואַלטונג־פֿליסן {#sovereign-customer-management-flows}

Ultimate Multisite v2.13.0 האַלט קאַסטאַמער־פֿאַרוואַלטונג־אַקציעס אויף דעם הויפּטפּלאַץ ווען זעלבשטענדיקער מאָדוס איז אַקטיווירט. אַ טענאַנט קען נאָך אַלץ לויפֿן ווי אַן איזאָלירטע WordPress אינסטאַלאַציע, אָבער קאַסטאַמער־געווידמעטע אַקציעס וואָס זענען אָפענגיק אויף נעצווערק־בילינג, מיטגלידערשאַפֿט, אָדער געטיילטע Account־דאַטן זאָלן צוריקשיקן דעם קאַסטאַמער צום הויפּטפּלאַץ אַנשטאָט צו פּרוּוון פֿאַרענדיקן די אַקציע אינעם טענאַנט־ראַנטײַם.

דער הויפּטפּלאַץ־פֿלוס גילט פֿאַר:

- Checkout און פּלאַן־ענדערונגען.
- Account איבערבליק און קאַסטאַמער־פּראָפֿיל־אַקציעס.
- בילינג־אַדרעס־דערהײַנטיקונגען און צאָלונג־פֿאַרוואַלטונג־עקראַנען.
- רעכענונג־און צאָלונג־געשיכטע־בליקן.
- פּלאַץ־פֿאַרוואַלטונג־אַקציעס אַזאַ ווי צולייגן פּלאַץן אָדער אויסמעקן אַ פּלאַץ.
- מוסטער־איבערבײַטן.
- דאָמיין־מאַפּינג און הויפּט־דאָמיין־ענדערונגען.

ווען דער קאַסטאַמער הייבט אָן איינע פֿון די אַקציעס פֿון אַ זעלבשטענדיקן טענאַנט, בויט Ultimate Multisite דעם קאָרעספּאָנדירנדיקן הויפּטפּלאַץ־URL און האַלט אָפּ דעם מקור־טענאַנט ווי אַ צוריקקער־ציל ווען עס איז זיכער דאָס צו טאָן. דאָס לאָזט קאַסטאַמערס פֿאַרענדיקן די פֿאַרוואַלטעטע אַקציע קעגן די נעצווערק־רעקאָרדס, און דערנאָך צוריקקומען צום טענאַנט־קאָנטעקסט אָן דופּליקירן בילינג־אָדער מיטגלידערשאַפֿט־צושטאַנד אין דער זעלבשטענדיקער דאַטאַבאַזע.

פֿאַר אָפּעראַטאָרן איז די פּראַקטישע כּלל: האַלט בילינג, Account, Checkout, רעכענונג, מוסטער, און דאָמיין־פֿאַרוואַלטונג־בלעטער בנימצא אויפֿן הויפּטפּלאַץ פֿאַר זעלבשטענדיקע נעצווערקן. טענאַנט Dashboard קען פֿאַרלינקען צו יענע בלעטער, אָבער דער הויפּטפּלאַץ בלײַבט דער מקור פֿון אמת פֿאַר דער אַקציע.
