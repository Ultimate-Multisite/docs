# Closte אינטעגראַציע

## איבערבליק
Closte איז אַ פאַרוואַלטעטע WordPress האָסטינג־פּלאַטפֿאָרמע געבויט אויף Google Cloud אינפֿראַסטרוקטור. די אינטעגראַציע דערמעגלעכט אויטאָמאַטישע דאָמיין־סינכראָניזאַציע און SSL סערטיפיקאַט־פאַרוואַלטונג צווישן Ultimate Multisite און Closte.

## פֿעיִקייטן
- אויטאָמאַטישע דאָמיין־סינכראָניזאַציע
- SSL סערטיפיקאַט־פאַרוואַלטונג
- שטיצע פֿאַר ווילדקאַרד־דאָמעינען
- קיין קאָנפֿיגוראַציע איז ניט נייטיק אויב עס לויפֿט אויף Closte

## פֿאָדערונגען
די פֿאָלגנדיקע קאָנסטאַנטע מוז זײַן דעפֿינירט אין אײַער `wp-config.php` טעקע אויב איר ניצט Closte:

```php
define('CLOSTE_CLIENT_API_KEY', 'your_api_key');
```

די קאָנסטאַנטע איז טיפּיש שוין דעפֿינירט אויב איר האָסט אויף Closte.

## אײַנשטעל־אנווײַזונגען

### 1. באַשטעטיקט אײַער Closte API Key

אויב איר האָסט אויף Closte, זאָל די `CLOSTE_CLIENT_API_KEY` קאָנסטאַנטע שוין זײַן דעפֿינירט אין אײַער `wp-config.php` טעקע. איר קענט דאָס באַשטעטיקן דורך קאָנטראָלירן אײַער `wp-config.php` טעקע.

### 2. אַקטיווירט די אינטעגראַציע

1. אין אײַער WordPress אַדמין, גייט צו Ultimate Multisite > Settings
2. נאַוויגירט צום "דאָמיין־מאַפּינג" קוויטל
3. מעגילט אַראָפּ צו "האָסט־אינטעגראַציעס"
4. אַקטיווירט די Closte אינטעגראַציע
5. גיט אַ קליק "היט ענדערונגען"

## ווי עס אַרבעט

ווען אַ דאָמיין ווערט געמאַפּט אין Ultimate Multisite:

1. די אינטעגראַציע שיקט אַן אָנפֿרעג צו Closte'ס API כּדי צוצולייגן דעם דאָמיין צו אײַער אַפּליקאַציע
2. Closte באַהאַנדלט אויטאָמאַטיש SSL סערטיפיקאַט־צושטעלונג
3. ווען אַ דאָמיין־מאַפּינג ווערט אַוועקגענומען, וועט די אינטעגראַציע אַוועקנעמען דעם דאָמיין פֿון Closte

די אינטעגראַציע אַרבעט אויך מיט דער DNS קאָנטראָל־אינטערוואַל אײַנשטעלונג אין Ultimate Multisite, וואָס דערלויבט אײַך צו קאָנפֿיגורירן ווי אָפֿט דער סיסטעם קאָנטראָלירט אויף DNS פֿאַרשפּרייטונג און SSL סערטיפיקאַט־אַרויסגעבן.

## שאַפֿן דאָמיין־רעקאָרד

די אינטעגראַציע זיכערט אַז ווען אַ פּלאַץ ווערט געשאַפֿן אָדער דופּליקירט, ווערט אַ דאָמיין־רעקאָרד אויטאָמאַטיש געשאַפֿן. דאָס איז באַזונדערס וויכטיק פֿאַר דער Closte אינטעגראַציע, ווײַל דאָס שאַפֿן דעם דאָמיין־רעקאָרד טריגערט דעם Closte API צו שאַפֿן דעם דאָמיין און SSL סערטיפיקאַט.

## טראָובלעשוּטינג

### API פֿאַרבינדונג־פּראָבלעמען
- באַשטעטיקט אַז אײַער Closte API Key איז ריכטיק
- זיכערט אַז אײַער Closte Account האָט די נייטיקע דערלויבענישן

### SSL סערטיפיקאַט־פּראָבלעמען
- Closte קען נעמען אַ ביסל צײַט אַרויסצוגעבן SSL סערטיפיקאַטן (געוויינטלעך 5-10 מינוט)
- באַשטעטיקט אַז אײַערע דאָמעינען ווײַזן ריכטיק אויף אײַער Closte סערווער'ס IP אַדרעס
- קאָנטראָלירט די DNS רעקאָרדס פֿאַר אײַער דאָמיין כּדי צו פֿאַרזיכערן אַז זיי זײַנען ריכטיק קאָנפֿיגורירט

### דאָמיין ניט צוגעלייגט
- קאָנטראָלירט די Ultimate Multisite לאָגן פֿאַר וועלכע־ניט־איז טעות־מעלדונגען
- באַשטעטיקט אַז דער דאָמיין איז ניט שוין צוגעלייגט צו Closte
- זיכערט אַז אײַער דאָמיין'ס DNS רעקאָרדס זײַנען ריכטיק קאָנפֿיגורירט

### DNS קאָנטראָל־אינטערוואַל
- אויב SSL סערטיפיקאַטן נעמען צו לאַנג אַרויסצוגעבן, קענט איר צופּאַסן דעם DNS קאָנטראָל־אינטערוואַל אין די דאָמיין־מאַפּינג אײַנשטעלונגען
- דער פעליקייַט־אינטערוואַל איז 300 סעקונדעס (5 מינוט), אָבער איר קענט אים שטעלן ביז אַזוי נידעריק ווי 10 סעקונדעס פֿאַר שנעלערער קאָנטראָל בעת טעסטינג
