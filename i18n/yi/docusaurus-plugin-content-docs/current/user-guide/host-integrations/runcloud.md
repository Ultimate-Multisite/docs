# RunCloud אינטעגראַציע

## איבערבליק {#overview}
RunCloud איז אַ וואָלקן־באזירטע סערווער־פאַרוואַלטונג־פּלאַטפֿאָרמע וואָס לאָזט אײַך גרינג אויסשטעלן און פאַרוואַלטן וועב־אַפּליקאַציעס אויף אײַערע אייגענע וואָלקן־סערווערס. די אינטעגראַציע דערמעגלעכט אויטאָמאַטישע דאָמיין־סינכראָניזאַציע און SSL־סערטיפיקאַט־פאַרוואַלטונג צווישן Ultimate Multisite און RunCloud.

## פֿעיִקייטן {#features}
- אויטאָמאַטישע דאָמיין־סינכראָניזאַציע
- SSL־סערטיפיקאַט־פאַרוואַלטונג
- דאָמיין־באַזײַטיקונג ווען מאַפּינגס ווערן אויסגעמעקט

## פֿאָדערונגען {#requirements}
די ווײַטערדיקע קאָנסטאַנטן מוזן ווערן דעפֿינירט אין אײַער `wp-config.php` טעקע:

```php
define('WU_RUNCLOUD_API_KEY', 'your_api_key');
define('WU_RUNCLOUD_API_SECRET', 'your_api_secret');
define('WU_RUNCLOUD_SERVER_ID', 'your_server_id');
define('WU_RUNCLOUD_APP_ID', 'your_app_id');
```

## אויפֿשטעל־אינסטרוקציעס {#setup-instructions}

### 1. באַקומען אײַערע RunCloud API קרעדענציאַלן {#1-get-your-runcloud-api-credentials}

1. לאָגט זיך אַרײַן אין אײַער RunCloud Dashboard
2. גייט צו "באַניצער־פּראָפֿיל" (קליקט אויף אײַער פּראָפֿיל־בילד אין דער אויבער־רעכטער ווינקל)
3. קלייבט אויס "API" פֿון דער מעניו
4. קליקט "גענערירן API שליסל" אויב איר האָט נאָך נישט איינעם
5. קאַפּירט אײַער API שליסל און API סוד

### 2. באַקומען אײַערע סערווער־ און אַפּ־IDס {#2-get-your-server-and-app-ids}

1. אין אײַער RunCloud Dashboard, גייט צו "סערווערס"
2. קלייבט אויס דעם סערווער וווּ אײַער WordPress מולטיסײַט איז געהאָסטעט
3. דער סערווער־ID איז זעבאַר אין דער URL: `https://manage.runcloud.io/servers/{SERVER_ID}`
4. גייט צו "וועב־אַפּליקאַציעס" און קלייבט אויס אײַער WordPress אַפּליקאַציע
5. דער אַפּ־ID איז זעבאַר אין דער URL: `https://manage.runcloud.io/servers/{SERVER_ID}/apps/{APP_ID}`

### 3. צולייגן קאָנסטאַנטן צו wp-config.php {#3-add-constants-to-wp-configphp}

לייגט צו די ווײַטערדיקע קאָנסטאַנטן צו אײַער `wp-config.php` טעקע:

```php
define('WU_RUNCLOUD_API_KEY', 'your_api_key');
define('WU_RUNCLOUD_API_SECRET', 'your_api_secret');
define('WU_RUNCLOUD_SERVER_ID', 'your_server_id');
define('WU_RUNCLOUD_APP_ID', 'your_app_id');
```

### 4. אַקטיווירן די אינטעגראַציע {#4-enable-the-integration}

1. אין אײַער WordPress אַדמין, גייט צו Ultimate Multisite > סעטינגס
2. נאַוויגירט צו דעם "דאָמיין־מאַפּינג" קוויטל
3. ראָלט אַראָפּ צו "האָסט־אינטעגראַציעס"
4. אַקטיווירט די RunCloud אינטעגראַציע
5. קליקט "אויפֿהיטן ענדערונגען"

## ווי עס אַרבעט {#how-it-works}

ווען אַ דאָמיין ווערט געמאַפּט אין Ultimate Multisite:

1. די אינטעגראַציע שיקט אַ בקשה צו RunCloud'ס API כּדי צוצולייגן דעם דאָמיין צו אײַער אַפּליקאַציע
2. אויב דער דאָמיין ווערט הצלחהדיק צוגעלייגט, וועט די אינטעגראַציע אויך ווידער־אויסשטעלן SSL־סערטיפיקאַטן
3. ווען אַ דאָמיין־מאַפּינג ווערט באַזײַטיקט, וועט די אינטעגראַציע באַזײַטיקן דעם דאָמיין פֿון RunCloud

פֿאַר סאָבדאָמיין־אינסטאַלאַציעס וועט די אינטעגראַציע אויטאָמאַטיש באַהאַנדלען דאָס שאַפֿן פֿון סאָבדאָמיינס אין RunCloud ווען נײַע זייטלעך ווערן צוגעלייגט צו אײַער נעצווערק.

## טראָבלשוטינג {#troubleshooting}

### API פֿאַרבינדונג־פּראָבלעמען {#api-connection-issues}
- באַשטעטיקט אַז אײַערע API קרעדענציאַלן זענען ריכטיק
- קאָנטראָלירט אַז אײַערע סערווער־ און אַפּ־IDס זענען ריכטיק
- מאַכט זיכער אַז אײַער RunCloud Account האָט די נייטיקע דערלויבענישן

### SSL סערטיפיקאַט־פּראָבלעמען {#ssl-certificate-issues}
- RunCloud קען נעמען אַ ביסל צײַט אַרויסצוגעבן SSL־סערטיפיקאַטן
- באַשטעטיקט אַז אײַערע דאָמיינס ווײַזן געהעריק אויף דעם IP־אַדרעס פֿון אײַער סערווער
- קאָנטראָלירט די RunCloud SSL סעטינגס פֿאַר אײַער אַפּליקאַציע

### דאָמיין נישט צוגעלייגט {#domain-not-added}
- קאָנטראָלירט די Ultimate Multisite לאָגס פֿאַר סיי וועלכע טעות־מעלדונגען
- באַשטעטיקט אַז דער דאָמיין איז נישט שוין צוגעלייגט צו RunCloud
- מאַכט זיכער אַז אײַער RunCloud פּלאַן שטיצט קייפֿל דאָמיינס
