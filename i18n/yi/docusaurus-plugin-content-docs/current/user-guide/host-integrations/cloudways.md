# Cloudways אינטעגראַציע

## איבערבליק {#overview}
Cloudways איז אַ גערירטע קלאוד־האָסטינג פּלאַטפאָרמע וואָס ערלויבט אײַך אַרויסצושטעלן WordPress וועבזײַטן אויף פֿאַרשיידענע קלאוד־צושטעלער ווי DigitalOcean, AWS, Google Cloud, און מער. די אינטעגראַציע דערמעגלעכט אויטאָמאַטישע דאָמיין־סינכראָניזאַציע און SSL־סערטיפיקאַט־פֿאַרוואַלטונג צווישן Ultimate Multisite און Cloudways.

## פֿעיִקייטן {#features}
- אויטאָמאַטישע דאָמיין־סינכראָניזאַציע
- SSL־סערטיפיקאַט־פֿאַרוואַלטונג
- שטיצע פֿאַר עקסטרע דאָמיינס
- DNS־וואַלידאַציע פֿאַר SSL־סערטיפיקאַטן

## פֿאָדערונגען {#requirements}
די ווײַטערדיקע קאָנסטאַנטן מוזן זײַן דעפֿינירט אין אײַער `wp-config.php` טעקע:

```php
define('WU_CLOUDWAYS_EMAIL', 'your_cloudways_email');
define('WU_CLOUDWAYS_API_KEY', 'your_api_key');
define('WU_CLOUDWAYS_SERVER_ID', 'your_server_id');
define('WU_CLOUDWAYS_APP_ID', 'your_app_id');
```

אָפּציאָנעל קענט איר אויך דעפֿינירן:

```php
define('WU_CLOUDWAYS_EXTRA_DOMAINS', 'comma,separated,list,of,domains');
```

## אויפֿשטעל־אנווײַזונגען {#setup-instructions}

### 1. באַקומען אײַערע Cloudways API קרעדענציאַלן {#1-get-your-cloudways-api-credentials}

1. לאָגט זיך אַרײַן אין אײַער Cloudways Dashboard
2. גייט צו "Account" > "API Keys"
3. דזשענערירט אַן API key אויב איר האָט נאָך נישט קיין איינעם
4. קאָפּירט אײַער בליצפּאָסט און API key

### 2. באַקומען אײַערע סערווער־ און אַפּליקאַציע־IDs {#2-get-your-server-and-application-ids}

1. אין אײַער Cloudways Dashboard, גייט צו "Servers"
2. קלײַבט אויס דעם סערווער וווּ אײַער WordPress מולטיסײַט איז געהאָסטעט
3. דער Server ID איז זעבאַר אין דעם URL: `https://platform.cloudways.com/server/{SERVER_ID}`
4. גייט צו "Applications" און קלײַבט אויס אײַער WordPress אַפּליקאַציע
5. דער App ID איז זעבאַר אין דעם URL: `https://platform.cloudways.com/server/{SERVER_ID}/application/{APP_ID}`

### 3. לייגט צו קאָנסטאַנטן צו wp-config.php {#3-add-constants-to-wp-configphp}

לייגט צו די ווײַטערדיקע קאָנסטאַנטן צו אײַער `wp-config.php` טעקע:

```php
define('WU_CLOUDWAYS_EMAIL', 'your_cloudways_email');
define('WU_CLOUDWAYS_API_KEY', 'your_api_key');
define('WU_CLOUDWAYS_SERVER_ID', 'your_server_id');
define('WU_CLOUDWAYS_APP_ID', 'your_app_id');
```

אויב איר האָט נאָך **אויסערלעכע** דאָמיינס (אויסער אײַער מולטיסײַט־נעץ) וואָס זאָלן שטענדיק בלײַבן אויף דער Cloudways אַליאַסן־רשימה:

```php
define('WU_CLOUDWAYS_EXTRA_DOMAINS', 'extradomain1.com,extradomain2.com');
```

:::warning נעמט נישט אַרײַן אײַער אייגענעם נעץ־ווילדקאַרד
לייגט **נישט** צו `*.your-network.com` (אָדער קיין שום סובדאָמיין־מוסטער פֿון אײַער אייגענעם נעץ) צו
`WU_CLOUDWAYS_EXTRA_DOMAINS`. זעט [וויכטיק — ווילדקאַרד SSL־שטרויכלונג](#important--wildcard-ssl-pitfall)
אונטן פֿאַרוואָס דאָס פֿאַרמײַדט אַרויסגעבן פּער־טענאַנט SSL־סערטיפיקאַטן.
:::

### 4. אַקטיווירט די אינטעגראַציע {#4-enable-the-integration}

1. אין אײַער WordPress אַדמין, גייט צו Ultimate Multisite > Settings
2. נאַוויגירט צום "Domain Mapping" קוויטל
3. רוקט אַראָפּ צו "Host Integrations"
4. אַקטיווירט די Cloudways אינטעגראַציע
5. קוועטשט "Save Changes"

## ווי עס אַרבעט {#how-it-works}

### דאָמיין־סינכראָניזאַציע {#domain-syncing}

ווען אַ דאָמיין ווערט צוגעמאַפּט אין Ultimate Multisite:

1. די אינטעגראַציע נעמט אַרויס אַלע איצט צוגעמאַפּטע דאָמיינס
2. זי לייגט צו דעם נײַעם דאָמיין צו דער רשימה (צוזאַמען מיט אַ www ווערסיע אויב עס פּאַסט)
3. זי שיקט די גאַנצע רשימה צו Cloudways דורך דעם API
4. Cloudways דערהײַנטיקט די דאָמיין־אַליאַסן פֿאַר אײַער אַפּליקאַציע

באַמערקונג: דער Cloudways API פֿאָדערט צו שיקן די גאַנצע רשימה פֿון דאָמיינס יעדעס מאָל, נישט בלויז צוצולייגן אָדער אַראָפּצונעמען באַזונדערע דאָמיינס.

### SSL־סערטיפיקאַט־פֿאַרוואַלטונג {#ssl-certificate-management}

נאָך דעם ווי דאָמיינס ווערן סינכראָניזירט:

1. די אינטעגראַציע קאָנטראָלירט וועלכע דאָמיינס האָבן גילטיקע DNS רעקאָרדס וואָס ווײַזן אויף אײַער סערווער
2. זי שיקט אַ בקשה צו Cloudways צו אינסטאַלירן Let's Encrypt SSL סערטיפיקאַטן פֿאַר יענע דאָמיינס
3. Cloudways באַהאַנדלט דאָס אַרויסגעבן און אינסטאַלירן פֿון די SSL סערטיפיקאַטן

די אינטעגראַציע בעט שטענדיק **סטאַנדאַרטע** (נישט־ווילדקאַרד) Let's Encrypt סערטיפיקאַטן פֿון
Cloudways. אויב אַ ווילדקאַרד־מוסטער ווערט צוגעשטעלט אין `WU_CLOUDWAYS_EXTRA_DOMAINS`, ווערט דער אָנהייבנדיקער
`*.` אָפּגעשטרייפּט פֿאַר דער SSL בקשה — דער ווילדקאַרד אַליין ווערט קיינמאָל נישט אינסטאַלירט דורך דער
אינטעגראַציע. כּדי צו ניצן אַ ווילדקאַרד־סערטיפיקאַט אויף Cloudways וואָלט איר עס געדאַרפֿט אינסטאַלירן
האַנטיש, אָבער דאָס בלאָקירט פּער־דאָמיין Let's Encrypt אַרויסגעבן פֿאַר צוגעמאַפּטע אייגענע דאָמיינס
(זעט די שטרויכלונג אונטן).

## עקסטרע דאָמיינס {#extra-domains}

די `WU_CLOUDWAYS_EXTRA_DOMAINS` קאָנסטאַנטע ערלויבט אײַך אָנצוגעבן נאָך **אויסערלעכע**
דאָמיינס וואָס זאָלן שטענדיק בלײַבן אויף דער Cloudways אַפּליקאַציע־אַליאַסן־רשימה. ניצט עס פֿאַר:

- אויסערלעכע דאָמיינס וואָס ווערן נישט פֿאַרוואַלטעט דורך Ultimate Multisite (למשל, אַ באַזונדערע מאַרקעטינג־וועבזײַט וואָס טיילט זיך מיט דער זעלבער Cloudways אַפּליקאַציע)
- געפּאַרקטע אָדער סטיידזשינג־דאָמיינס וואָס איר ווילט האַלטן אויף דער אַפּליקאַציע־אַליאַסן־רשימה

ניצט **נישט** די קאָנסטאַנטע פֿאַר אײַער אייגענעם נעץ־סובדאָמיין־ווילדקאַרד
(למשל `*.your-network.com`). זעט די ווילדקאַרד SSL־שטרויכלונג אונטן.

## וויכטיק — ווילדקאַרד SSL־שטרויכלונג {#important--wildcard-ssl-pitfall}

אַ געוויינטלעכער טעות בײַם נאָכפֿאָלגן Cloudways' סטאַנדאַרט־אויפֿשטעל איז צוצולייגן אַ ווילדקאַרד אַזאַ ווי
`*.your-network.com` צו `WU_CLOUDWAYS_EXTRA_DOMAINS`, אָדער האַנטיש אינסטאַלירן אַ Cloudways
ווילדקאַרד SSL סערטיפיקאַט פֿאַר יענעם ווילדקאַרד.

**אויב איר טוט דאָס, וועט Cloudways אָפּזאָגן אַרויסצוגעבן Let's Encrypt סערטיפיקאַטן פֿאַר די
פּער־טענאַנט אייגענע דאָמיינס וואָס Ultimate Multisite מאַפּט.** Cloudways פֿאַרבײַט דעם אַקטיוון
SSL סערטיפיקאַט אויף דער אַפּליקאַציע יעדעס מאָל, און אַ פֿריִער עקזיסטירנדיקער ווילדקאַרד־סערטיפיקאַט אויף
דער אַפּליקאַציע בלאָקירט די פּער־דאָמיין Let's Encrypt אַרויסגעבן אויף וועלכער די אינטעגראַציע פֿאַרלאָזט זיך.

### רעקאָמענדירטע Cloudways SSL אויפֿשטעל פֿאַר אַן Ultimate Multisite נעץ {#recommended-cloudways-ssl-setup-for-an-ultimate-multisite-network}

1. אין דער Cloudways אַפּליקאַציעס **SSL Certificate** קוויטל, אינסטאַלירט אַ **סטאַנדאַרטן
   Let's Encrypt סערטיפיקאַט** וואָס דעקט בלויז `your-network.com` און `www.your-network.com`
   — **נישט** קיין ווילדקאַרד.
2. לייגט **נישט** `*.your-network.com` (אָדער קיין שום סובדאָמיין־מוסטער פֿון אײַער אייגענעם נעץ) אין
   `WU_CLOUDWAYS_EXTRA_DOMAINS`. רעזערווירט די קאָנסטאַנטע בלויז פֿאַר **אויסערלעכע** דאָמיינס.
3. שאַפֿט דעם פּער־טענאַנט סובדאָמיין־ווילדקאַרד בלויז אויף דער **DNS** מדרגה (אַן `A` רעקאָרד פֿאַר
   `*.your-network.com` וואָס ווײַזט אויף אײַער Cloudways סערווער IP) כּדי סובזײַטן זאָלן זיך אויפֿלייזן. SSL
   פֿאַר יחידישע צוגעמאַפּטע אייגענע דאָמיינס ווערט דאַן אויטאָמאַטיש אַרויסגעגעבן דורך דער אינטעגראַציע
   דורך Let's Encrypt.

אויב אייערע לאָקאַטאָרן׳ס אייגענע דאָומיינס בלייבן אָן SSL, קאָנטראָלירט דעם Cloudways SSL קוויטל. אויב אַ
וויילדקאַרד־צערטיפיקאַט איז דאָרטן אַקטיוו, נעמט אים אַראָפּ, גיט ווידער אַרויס אַ סטאַנדאַרט Let's Encrypt
צערטיפיקאַט בלויז פֿאַר דעם הויפּט נעצווערק־דאָומיין, און נעמט אַרויס אַלע וויילדקאַרד־אײַנטראָגן פֿון
`WU_CLOUDWAYS_EXTRA_DOMAINS`. דערנאָך טריגערט ווידער אַ דאָומיין־מאַפּינג (אָדער וואַרט ביז דער קומענדיקער)
און די אינטעגראַציע וועט ווידער אָנהייבן אַרויסגעבן צערטיפיקאַטן פּער דאָומיין.

## פּראָבלעם־לייזונג {#troubleshooting}

### API פֿאַרבינדונג־פּראָבלעמען {#api-connection-issues}
- באַשטעטיקט אַז אייער בליצפּאָסט און API key זענען ריכטיק
- קאָנטראָלירט אַז אייער סערווער־ און אַפּליקאַציע־IDs זענען ריכטיק
- מאַכט זיכער אַז אייער Cloudways Account האָט די נויטיקע דערלויבענישן

### SSL צערטיפיקאַט־פּראָבלעמען {#ssl-certificate-issues}
- Cloudways דאַרף אַז דאָומיינס זאָלן האָבן גילטיקע DNS רעקאָרדס וואָס ווײַזן אויף אייער סערווער איידער מען גיט אַרויס SSL צערטיפיקאַטן
- די אינטעגראַציע וואַלידירט DNS רעקאָרדס איידער זי בעט SSL צערטיפיקאַטן
- אויב SSL צערטיפיקאַטן ווערן נישט אַרויסגעגעבן, קאָנטראָלירט אַז אייערע דאָומיינס ווײַזן געהעריק אויף אייער סערווער׳ס IP אַדרעס
- **אייגענע דאָומיינס פּער לאָקאַטאָר בלייבן אָן SSL?** קאָנטראָלירט דעם Cloudways אַפּליקאַציע׳ס SSL Certificate קוויטל. אויב אַ וויילדקאַרד־צערטיפיקאַט (מאַנועל אינסטאַלירט, אָדער וואָס דעקט `*.your-network.com`) איז אַקטיוו, וועט Cloudways נישט אַרויסגעבן Let's Encrypt צערטיפיקאַטן פֿאַר יחיד־געמאַפּטע אייגענע דאָומיינס. פֿאַרבײַט אים מיט אַ סטאַנדאַרט Let's Encrypt צערטיפיקאַט וואָס דעקט בלויז דעם הויפּט נעצווערק־דאָומיין (`your-network.com`, `www.your-network.com`) און נעמט אַרויס אַלע וויילדקאַרד־אײַנטראָגן פֿון `WU_CLOUDWAYS_EXTRA_DOMAINS`. דערנאָך טריגערט ווידער אַ דאָומיין־מאַפּינג (אָדער וואַרט ביז דער קומענדיקער) און די אינטעגראַציע וועט בעטן צערטיפיקאַטן פּער דאָומיין.

### דאָומיין נישט צוגעגעבן {#domain-not-added}
- קאָנטראָלירט די Ultimate Multisite לאָגס פֿאַר סיי וועלכע טעות־מעלדונגען
- באַשטעטיקט אַז דער דאָומיין איז נאָך נישט צוגעגעבן צו Cloudways
- מאַכט זיכער אַז אייער Cloudways פּלאַן שטיצט די צאָל דאָומיינס וואָס איר לייגט צו
