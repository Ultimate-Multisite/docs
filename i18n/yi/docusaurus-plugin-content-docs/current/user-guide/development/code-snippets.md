# קאד־אויסשניטן פֿאַר v2

אין גרונט, ווערן קאד־אויסשניטן פֿאַר **WordPress** גענוצט צו טאָן געוויסע אַקציעס וואָס וואָלטן אַנדערש געקענט דאַרפֿן אַ באַזונדערן קלענערן פּלוגין. אַזעלכע קאד־אויסשניטן ווערן אַרײַנגעשטעלט אין איינע פֿון די WordPress קערן־ אָדער טעמע־טעקעס (בדרך־כּלל די functions.php טעקע פֿון אײַער טעמע) אָדער זיי קענען ווערן גענוצט ווי אַ MU plugin.

אין דעם אַרטיקל וועלן מיר אײַך ווײַזן דרײַ קאד־אויסשניטן וואָס קענען ווערן גענוצט מיט **Ultimate Multisite v2** :

## טוישן די פּאָזיציע פֿון דעם Account מעניו־איינטראָג

כדי צו טוישן די פּאָזיציע פֿון דעם Account מעניו־איינטראָג אויף אײַער קליענטס Dashboard, לייגט נאָר צו דעם ווײַטערדיקן קאד־אויסשניט אין די functions.php פֿון דער אַקטיווער טעמע פֿון אײַער הויפּט־זײַטל. איר קענט אויך אַרײַנלייגן דעם אויסשניט אין איינעם פֿון אײַערע mu-plugins אָדער צוגעפּאַסטע פּלוגינס.

add_filter('wu_my_account_menu_position', function() { return 10; // Tweak this value to place the menu in the desired position.

## ווי צו קאָנטראָלירן צי דער באַניצער איז אונטער אַ געוויסן פּלאַן און/אָדער האָט אַן אַקטיווע אַבאָנעמענט

ווי אַ נעץ־אַדמין, קענט איר דאַרפֿן צו שאַפֿן צוגעפּאַסטע פֿונקציעס וואָס וועלן דורכפֿירן יסודותדיקע אַקציעס אָדער מאַכן אַ דינסט/אייגנשאַפֿט בנימצא פֿאַר אַ אויסגעקליבענע גרופּע פֿון אַבאָנענטן אָדער סוף־באַניצערס, באַזירט אויף דעם סטאַטוס פֿון זייער אַבאָנעמענט און דעם פּלאַן אונטער וועלכן זיי זענען אַבאָנירט.

די Ultimate Multisite געבוירענע פֿונקציעס וועלן אײַך העלפֿן דערמיט.

כדי צו קאָנטראָלירן צי דער באַניצער איז אַ מיטגליד פֿון אַ געוויסן פּלאַן, קענט איר נוצן די פֿונקציע:

wu_has_plan($user_id, $plan_id)

כדי צו קאָנטראָלירן צי די אַבאָנעמענט איז אַקטיוו, קענט איר נוצן די פֿונקציע:

wu_is_active_subscriber($user_id)

אונטן איז אַ בײַשפּיל־אויסשניט וואָס קאָנטראָלירט צי דער איצטיקער באַניצער איז אונטער אַ ספּעציפֿישן פּלאַן (_Plan ID 50_) און צי דער באַניצערס אַבאָנעמענט איז אַקטיוו.

$user_id = get_current_user_id();$plan_id = 50;if (wu_has_plan($user_id, $plan_id) && wu_is_active_subscriber($user_id)) { // USER IS MEMBER OF PLAN AND HIS SUBSCRIPTION IS ACTIVE, DO STUFF} else { // USER IS NOT A MEMBER OF PLAN -- OR -- HIS SUBSCRIPTION IS NOT ACTIVE, DO OTHER STUFF} // end if;

באַמערקט אַז _**wu_has_plan**_ דאַרף אַ "Plan ID" כּדי עס זאָל פֿונקציאָנירן.

כדי צו באַקומען דעם ID פֿון אַ פּלאַן, קענט איר גיין צו **Ultimate Multisite > Products**. דער ID פֿון יעדער פּראָדוקט וועט ווערן געוויזן רעכטס פֿון דער טאַבעלע.

באַמערקט אַז באַניצערס קענען נאָר זײַן אַבאָנירט צו אַ **Plan** , נישט צו אַ פּעקל אָדער דינסט, ווײַל זיי זענען בלויז צוגאָבן פֿאַר אַ **Plan**.

## פֿאַרריכטן CORS פּראָבלעמען מיט Font-Icons אין צוגעפּאַסטע דאָמיינען
## פֿאַרריכטן CORS פּראָבלעמען מיט Font-Icons אין צוגעפּאַסטע דאָמיינען

נאָך מאַפּן אַ דאָמיין צו אַ אונטער־זײַטל, קענט איר אַנטדעקן אַז דאָס זײַטל האָט שוועריקייטן צו לאָדן צוגעפּאַסטע שריפֿטן. דאָס ווערט געפֿירט דורך אַ קראָס־אָרידזשין בלאָק אין אײַערע סערווער־איינשטעלונגען.

ווײַל שריפֿט־טעקעס ווערן כּמעט שטענדיק געלאָדן גלײַך פֿון CSS, קען אונדזער דאָמיין־מאַפּינג־פּלוגין נישט איבערשרײַבן די URLs צו נוצן דעם צוגעפּאַסטן דאָמיין אַנשטאָט דעם אָריגינעלן, דעריבער, כּדי צו פֿאַרריכטן דעם פּראָבלעם, דאַרפֿט איר ענדערן אײַערע סערווער־קאָנפֿיגוראַציע־טעקעס.

אונטן זענען קאד־אויסשניטן צו פֿאַרריכטן דעם פּראָבלעם פֿאַר Apache און NGINX. די ענדערונגען דאַרפֿן פֿאָרגעשריטענע קענטענישן פֿון סערווער־קאָנפֿיגוראַציע־טעקעס (.htaccess טעקעס און NGINX config טעקעס). אויב איר פֿילט זיך נישט באַקוועם צו מאַכן די ענדערונגען אַליין, שיקט די זײַט צו די שטיצע־אַגענטן פֿון אײַער האָסטינג־צושטעלער ווען איר דאַרפֿט הילף.

### Apache

אין אײַער .htaccess טעקע, לייגט צו:

<FilesMatch “.(ttf|ttc|otf|eot|woff|font.css|css)$”> Header set Access-Control-Allow-Origin “*”

### NGINX

אין אײַער סערווער config טעקע (דער אָרט בײַט זיך פֿון סערווער צו סערווער), לייגט צו:

location ~ .(ttf|ttc|otf|eot|woff|font.css|css)$ { add_header Access-Control-Allow-Origin “*”;}
