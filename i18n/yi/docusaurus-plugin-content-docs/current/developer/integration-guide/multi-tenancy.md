# Multi-Tenancy אינטעגראַציע {#multi-tenancy-integration}

Ultimate Multisite: Multi-Tenancy 1.2.0 ענדערט עטלעכע אינטעגראַציע־רירפּונקטן פֿאַר זעלבסטשטענדיקע לאָקאַטאָרן, מיגראַציע־וועריפֿיקאַציע, און אויטאָמאַציע פֿון לאָקאַטאָר־לעבנסציקל.

## לאָקאַטאָר bootstrap־פֿלוס {#tenant-bootstrap-flow}

אינטעגראַציעס וואָס שאַפֿן אָדער מאָדיפֿיצירן לאָקאַטאָרן זאָלן נאָכגיין דעם סדר:

1. לייזט אויף דעם לאָקאַטאָר־רעגיסטרי־רעקאָרד און איזאָלאַציע־מאָדעל.
2. שאַפֿט אָדער וועריפֿיצירט דעם לאָקאַטאָר־דאַטנבאַנק־שרײַבער.
3. פֿירט bootstrap אויס פֿאַר דער לאָקאַטאָר־סטשעמע.
4. שטעלט צו לאָקאַטאָר־באַניצער.
5. רעגיסטרירט לאָקאַטאָר־רוטינג און פֿיילסיסטעם־וועגן.
6. פֿירט מיגראַציע־וועריפֿיקאַציע אויס איידער מען מאַכט דעם לאָקאַטאָר צוטריטלעך.

נעמט נישט אָן אַז אַ זעלבסטשטענדיקער לאָקאַטאָר קען ווידער־נוצן די נעצווערק־דאַטנבאַנק־פֿאַרבינדונג. ניצט די לאָקאַטאָר־רעגיסטרי און שרײַבער־אַבסטראקציעס וואָס דער addon שטעלט צו.

## SSO און REST hooks {#sso-and-rest-hooks}

סטייטלאָזער לאָקאַטאָר־אויטאָלאָגין ניצט קורץ־לעבעדיקע tokens מיט אַ purpose claim, JTI replay protection, אַן expiry cap, און origin pinning. אינטעגראַציעס וואָס לייגן צו לאָגין־קנעפּלעך אָדער ווײַטע פֿאַרוואַלטונג־לינקען זאָלן גענערירן לאָקאַטאָר־באַזוכן דורך דעם געשטיצטן SSO־פֿלוס אַנשטאָט צו בויען לאָקאַטאָר־לאָגין־URLs דירעקט.

נעצווערק־זײַטיקע API־אודיט־געשעענישן און טעגלעכע קיצורים זענען פֿאַראַן פֿאַר זעלבסטשטענדיקע לאָקאַטאָר־גייטווייז. ניצט יענע לאָגס בײַם דיבאַגירן עקסטערנע סיסטעמען וואָס רופֿן לאָקאַטאָר־לעבנסציקל endpoints.

## זעלבסטשטענדיקע קונה־אַקציע־URLs {#sovereign-customer-action-urls}

Ultimate Multisite v2.13.0 רוטעט זעלבסטשטענדיקע־לאָקאַטאָר קונה־אַקציעס צוריק צו דער הויפּטזײַט פֿאַר Account, checkout, billing, invoice, זײַט, template־איבערבײַט, און domain-mapping פֿלוסן. אינטעגראַציעס וואָס רענדערן לאָקאַטאָר־זײַטיקע פֿאַרוואַלטונג־לינקען זאָלן צילן יענע אַקציעס צום הויפּטזײַט־קונה־פּאַנעל און אַרײַננעמען אַ וועריפֿיצירטן צוריקקער־ציל ווען דער באַניצער דאַרף קענען נאַוויגירן צוריק צום לאָקאַטאָר נאָך פֿאַרענדיקן די אַקציע.

ניצט דעם core SSO־wrapper פֿאַר קראָס־דאָמיין פֿאַרוואַלטונג־לינקען:

```php
$url = wu_with_sso($main_site_customer_url);
```

די גענערירטע URL בלײַבט פֿילטערבאַר דורך `wu_sso_url`, וואָס באַקומען די SSO URL, דעם איצטיקן באַניצער, ציל־זײַט ID, און redirect־קאָנטעקסט. Add-ons קענען ניצן יענעם פֿילטער צו צולייגן provider־ספּעציפֿישן קאָנטעקסט אָדער צו פֿאַרבײַטן דעם broker URL בשעת מען היט אָפּ Ultimate Multisite'ס token־וועריפֿיקאַציע.

דוּפּליצירט נישט membership, invoice, billing-address, template, אָדער domain-management צושטאַנד אינעווייניק אינעם זעלבסטשטענדיקן לאָקאַטאָר. באַהאַנדלט דעם לאָקאַטאָר Dashboard ווי דעם launcher און דעם הויפּטזײַט־קונה־פּאַנעל ווי די סיסטעם פֿון רעקאָרד פֿאַר פֿאַרוואַלטעטע אַקציעס.

## מיגראַציע־וועריפֿיקאַציע {#migration-verification}

נאָך אַ מיגראַציע אָדער לעבנסציקל־אינטעגראַציע ענדערט לאָקאַטאָר־דאַטן, פֿירט אויס די וועריפֿיקאַציע־טויערן:

- `wp tenant verify-no-legacy --site=<site-id>` באַשטעטיקט אַז דער לאָקאַטאָר איז מער נישט אָפֿהענגיק פֿון אַלטע נעצווערק־זײַטיקע וועגן.
- `wp tenant verify-sovereign-push --site=<site-id>` באַשטעטיקט אַז זעלבסטשטענדיקע push־אַרבעטן קענען פּראָצעסירן און אויסליידיקן.

אינטעגראַציעס זאָלן באַהאַנדלען דורכגעפֿאַלענע וועריפֿיקאַציע ווי אַ deployment־בלאָקירער און אויסמײַדן צו מאַרקירן אַ לאָקאַטאָר ווי live ביז דער דורכפֿאַל ווערט געלייזט.

## לאָקאַטאָר־אויסמעקונג {#tenant-deletion}

אויסמעקונג־פֿלוסן זאָלן רופֿן דעם addon teardown־וועג כּדי לאָקאַטאָר־דאַטנבאַנק credentials זאָלן ווערן אָפּגערייניקט. עקסטערנע אינטעגראַציעס מעגן אַראָפּנעמען provider־רעסורסן נאָך דעם ווי teardown געלינגט, אָבער זאָלן נישט אויסמעקן host־דאַטנבאַנקן אָדער טעקעס־קעסטלעך בשעת וועריפֿיקאַציע אָדער async push־אַרבעטן לויפֿן נאָך.

## אָפּגעשאַפטער דאַטנבאַנק־router {#deprecated-database-router}

דער אַלטער `Database_Router` איז פאַרביטן געוואָרן דורך אַ deprecation stub. נײַע אינטעגראַציעס זאָלן רעזאָלווירן לאָקאַטאָרן דורך די איצטיקע זײַט־router און לאָקאַטאָר־רעגיסטרי APIs אַנשטאָט זיך צו פֿאַרלאָזן אויף דער אַלטער router־קלאַס.
